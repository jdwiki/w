Java EE nedir? #

Java EE, Java Enterprise Edition anlamına gelir. Java EE, Java SE’yi (Java Standard Edition anlamına gelir) genişletir. Java EE, büyük ölçekli kurumsal uygulamaların geliştirilmesine yönelik bir dizi teknoloji ve ilgili belirtimdir. Java EE, topluluk odaklı bir süreçte geliştirilmiştir. Şimdiye kadar Java EE’nin aşağıdaki sürümleri yayınlandı:

  • J2EE 1.2 (12 Aralık 1999)
  • J2EE 1.3 (24 Eylül 2001)
  • J2EE 1.4 (11 Kasım 2003)
  • Java EE 5 (11 Mayıs 2006)
  • Java EE 6 (10 Aralık 2009)
  • Java EE 7 (5 Nisan 2013)

Ve Java EE 8’in 2017’nin ilk yarısında piyasaya sürülmesi bekleniyor.

Java EE’nin anahtar kavramı, her Java EE sürümünün bir dizi belirli teknolojiden oluşmasıdır. Bu teknolojiler belirli JSR’leri (Java Spesifikasyon İstekleri) ele alır. Bir programcının bu teknolojileri kullanabilmesi için Java EE teknoloji belirtimlerinin bir uygulamasını indirmesi gerekir. Java Topluluğu, her teknoloji için bir referans uygulaması sağlar, ancak diğer Java EE uyumlu teknolojiler geliştirilir ve kullanılabilir. Topluluk, bir JSR uygulamasının geliştiricileri tarafından JSR ile uyumlu olup olmadığını kontrol etmek için kullanılabilecek Java Uyumluluk Kiti (JCK) adlı bir dizi test sağlar. Kurulum #

Her şeyden önce, Java EE’yi “yükleyemezsiniz”. Java EE bir dizi spesifikasyondan oluşur. Ancak bu spesifikasyonların uygulamalarını yükleyebilirsiniz.

İhtiyaçlarınıza bağlı olarak, birçok olasılık var. Belirtimlerin çoğunu (veya tümünü) yüklemek için Java EE 7 uyumlu bir Uygulama Sunucusu seçebilirsiniz. İhtiyaçlarınıza bağlı olarak, web profilini uygulayan uygulama sunucuları veya tam profili uygulayan uygulama sunucuları arasında seçim yapabilirsiniz. Java EE7 uyumlu uygulama sunucularının listesi için bkz. Java EE Uyumluluğu.

Payara Sunucu Kurulumu Full #

Ön koşullar

  • JDK 1.7 veya üstü yüklü. Oracle JDK’ları buradan bulabilirsiniz.

Adımlar

  • Payara Sunucusu Dolu dosyasını indirin.
  • Bilgisayarınızda bir yerde Payara Sunucusunu açın. Windows kullanıcıları için INSTALL_DIR olarak C:\payara41 ve Linux/Mac kullanıcıları için /payara41 kullanacağız.

Payara’yı komut isteminden başlatma/durdurma

  • Windows: Payara’yı başlatmak/durdurmak için bir komut istemi açın ve aşağıdaki komutu yürütün:

    "C:\payara41\bin\asadmin" start-domain

    "C:\payara41\bin\asadmin" stop-domain

  • Linux/Max: Bir terminal açın ve Payara’yı başlatmak/durdurmak için aşağıdaki komutu çalıştırın:

    /payara41/bin/asadmin start-domain

    /payara41/bin/asadmin stop-domain

Payara’yı Netbeans’ten başlatmak

  • Payara sunucusunu Netbeans’e ekleyin (önceki bölüme bakın)
  • ‘Hizmetler’ sekmesine gidin (Windows -> Hizmetler).
  • ‘Sunucular’ öğesini genişletin.
  • Payara sunucusuna sağ tıklayın ve ‘Başlat’ı seçin.
  • (Opsiyonel) Konsola gitmek için Payara sunucusuna sağ tıklayın ve ‘Alan Yönetici Konsolunu Görüntüle’yi seçin.

Uygulama Sunucusunu çalıştırdığınızı kontrol etmek için bir tarayıcı açın ve Payara Sunucu Konsolunu görmek için http://localhost:4848 adresine gidin.

İşte! Şimdi JavaEE kullanarak ilk uygulamanızı oluşturma ve sunucunuza dağıtma zamanı!

İlk JavaEE Uygulamamı Oluşturma (Merhaba Dünya) #

Bir şeyi anlayalım. JavaEE bir dizi spesifikasyondan oluşur. Bir Uygulama Sunucusu (örneğin Payara) kurduğunuzda, tüm özellikleri bir kerede kurarsınız. Örneğin, JPA (Java Persistence API) adlı bir ORM belirtimi, JSF (Java Sunucu Yüzleri) adlı Bileşen Tabanlı Web Uygulamaları oluşturmaya yönelik bir belirtim, REST web hizmetleri oluşturmaya yönelik bir belirtim ve JAX-RS adlı istemciler.

Tahmin edebileceğiniz gibi, JavaEE’de basit bir Hello World uygulaması oluşturmanın tek bir yolu yoktur.

İkinci olarak, JavaEE spesifikasyonu şuna benzeyen (basitleştirilmiş) belirli bir klasör yapısına sahiptir:

/projectname/src/main/java
/projectname/src/main/resources
/projectname/src/main/resources/META-INF
/projectname/src/main/webapp
/projectname/src/main/webapp/WEB-INF

/projectname/src/main/java içine ihtiyacımız olan tüm java sınıflarını koyuyoruz.

/projectname/src/main/webapp içine html dosyaları, css dosyaları, javascript dosyaları vb. koyuyoruz.

/projectname/src/main/webapp/WEB-INF dosyasının içinde web.xml ve beans.xml gibi bazı isteğe bağlı yapılandırma dosyaları bulunur.

Basit olması için ilk JavaEE Uygulamamızı oluşturmak için NetBeans IDE’yi (ücretsizdir) kullanacağız. [Burada] bulabilirsiniz.]1 JavaEE sürümünü seçin ve yükleyin.

Yeni proje oluştur #

  • NetBeans IDE’yi açın
  • Dosya > Yeni Proje > Maven > Web Uygulaması’na gidin ve İleri’ye tıklayın.
  • proje adını HelloJavaEE olarak değiştirin, ardından İleri ve Bitir’e tıklayın.

Projeyi temizleyin ve inşa edin #

  • Çalıştır > Projeyi Temizle ve Oluştur’a (HelloJavaEE) gidin.

WAR dosyasını dağıtın #

  • Bir tarayıcıda http://localhost:4848 adresine gidin (payara sunucusunu kurmak için talimatları izleyin).
  • Uygulamalar’a gidin > Dağıt’a tıklayın, Dosya Seç’e tıklayın ve `../NetBeansProjects/HelloJavaEE/target’ altında savaş dosyanızı (HelloJavaEE-1.0-SNAPSHOT.war) seçin.

WAR dosyasını doğrudan Netbeans’ten dağıtın #

  • Payara’yı kurun (bir sonraki bölüme bakın).
  • Netbeans’te ‘Hizmetler’ sekmesine gidin (görmüyorsanız Pencere-> Hizmetler).
  • Sunucular’a sağ tıklayın ve ‘Sunucu Ekle…’ seçeneğini seçin.
  • ‘GlassFish Sunucusu’nu seçin, ona bir ad verin ve ileri’ye tıklayın.
  • Yerel Payara kurulumunuzun üzerine gelin, ‘Yerel Etki Alanı’nı seçin ve ileri’ye tıklayın.
  • Etki alanı konum ayarlarını olduğu gibi bırakın (Alan Adı: etki alanı1, Ana Bilgisayar: localhost, DAS Bağlantı Noktası: 4848, HTTP Bağlantı Noktası: 8080).
  • ‘Projeler’ sekmesine gidin (Pencereler -> Projeler).
  • Projenize sağ tıklayın ve ‘Özellikler’i seçin.
  • Sol bölmede, ‘Çalıştır’a gidin ve yeni eklediğiniz Sunucuyu seçin.
  • (İsteğe bağlı) Bağlam yolunu değiştirin. Bağlam yolunu ‘/MyFirstApplication’ olarak ayarlarsanız, varsayılan URL ‘http://localhost:8080/MyFirstApplication’ olacaktır.

Sonuçları Görüntüle #

  • Bir tarayıcıda http://localhost:8080/HelloJavaEE-1.0-SNAPSHOT adresine gidin

İşte! Bu, JavaEE teknolojisini kullanan ilk uygulamanız. Şimdi JPA, EJB, JAX-RS, JavaBatch, vb. gibi farklı özellikleri kullanarak başka “Merhaba Dünya” uygulamaları oluşturmaya başlamalısınız…
