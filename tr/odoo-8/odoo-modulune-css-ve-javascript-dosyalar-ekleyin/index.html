Odoo modülüne CSS ve Javascript dosyaları ekleyin

Sözdizimi #

  • XML ​​sözdizimi hakkında not: Kayıt XML dosyasının içinde yapıldığından, düz bir HTML’de olduğu gibi hiçbir etiketi kapatmadan bırakamazsınız, örneğin: <link rel=‘stylesheet’ href="…" ** >**, Bunun yerine bağlantı etiketini kapatın, örneğin:
  • <link rel=‘stylesheet’ href="…" />

Parametreler #

inherit_id parametresinin olası değerleri anlam
web.assets_backend Yalnızca dahili sayfalarda kullanılır, herkese açık bir web sitesine dahil DEĞİLDİR.
website.assets_frontend Yalnızca herkese açık bir web sitesinde kullanılır ("website" modülü aracılığıyla).
web.assets_common Hem genel web sitesinde hem de dahili sayfalarda kullanılır.

Hangi seçeneğin sizin için uygun olduğundan emin değilseniz, çoğu durumda kullanıldığı gibi ilk seçeneği (arka uç) deneyin ve “web sitesi” modülünü yüklemediyseniz hemen hemen her durumda. Odoo, “arka uç” ve “ön uç” varlıkları arasında ayrım yapar, çünkü “web sitesi” modülü tarafından sağlanan genel web sitesi, ERP görevleri için kullanılması amaçlanan dahili sayfalardan farklı stil ve JS kodu kullanır, yani “ön uç” genel web sitesi ile ilişkilendirilir ve “arka uç”, ERP için dahili sayfalarla ilişkilidir (“ön uç” ve “arka uç” anlamları burada Odoo’ya özgüdür, ancak her ikisi de daha genel anlamda “ön uç"tur).

Seçeneklerden sadece birini seçip kullanabileceğiniz gibi, aynı modülde herhangi bir kombinasyonunu (ikisi veya tümü) kullanabilirsiniz. DRY’ye daha iyi uymak ve genel web sitesinde ve dahili sayfalarda uygun koda sahip olmak için bir arka uç, bir ön uç ve ortak bir JS/CSS kodunu ayrı dosyalara ayırın.

‘openerp.py’ bildirimindeki bağımlılık listesine “web” (seçenek 1 kullanırken) veya “web sitesi” (seçenek 2 kullanırken) eklemeyi unutmayın.

CSS ve JS dosyalarını Odoo modülünde doğru şekilde depolayın #

CSS ve JS dosyaları, modülün kök dizinindeki ‘statik’ dizinin altında bulunmalıdır (‘statik’ altındaki alt dizin ağacının geri kalanı isteğe bağlı bir kuraldır):

  • statik/src/css/your_file.css
  • statik/src/js/your_file.js

Ardından, aşağıdaki örneklerde listelenen 3 yoldan birini kaldırmadan bu dosyalara bağlantılar ekleyin.

Seçenek 1: [BACKEND] Dahili sayfalarda kullanmak için CSS ve Javascript dosyaları ekleyin #

Odoo v8.0 yolu, XML dosyasına karşılık gelen kaydı eklemektir:

  • ​Bildirime XML dosyası ekleyin (yani, __openerp__.py dosyası.):

    …
    ‘data’ : [ ‘your_file.xml’ ],
    ​…

  • Ardından ``your_file.xml’` içine aşağıdaki kaydı ekleyin:

    <openerp>
        <data>  
            <template id="assets_backend" name="your_module_name assets" inherit_id="web.assets_backend">  
                <xpath expr="." position="inside">  
                    <link rel='stylesheet' href="/your_module_name/static/src/css/your_file.css"/>  
                    <script type="text/javascript" src="/your_module_name/static/src/js/your_file.js"></script>  
                </xpath>  
            </template>  
        ....  
        ....  
        </data>  
     </openerp>

Seçenek 2: [ÖNDEN] Herkese açık bir web sitesinde kullanmak için CSS ve Javascript dosyaları ekleyin #

Not: Bir “web sitesi” modülü yüklediyseniz ve genel bir web siteniz varsa bu yolu kullanmalısınız.

  • ``your_file.xml’` içine aşağıdaki kaydı ekleyin:
    <openerp>
        <data>  

            <template id="assets_frontend" name="your_module_name assets" inherit_id="website.assets_frontend">  
                <xpath expr="link[last()]" position="after">  
                    <link rel='stylesheet' href="/your_module_name/static/src/css/your_file.css"/>  
                </xpath>  
                <xpath expr="script[last()]" position="after">  
                    <script type="text/javascript" src="/your_module_name/static/src/js/your_file.js"></script>  
                </xpath>  
            </template>  

        </data>  
     </openerp>

Seçenek 3: [ORTAK] Tüm sayfalarda kullanmak üzere CSS ve Javascript dosyaları ekleyin (arka uç ve ön uç) #

  • ``your_file.xml’` içine aşağıdaki kaydı ekleyin:
    <openerp>
        <data>  

            <template id="assets_common" name="your_module_name assets" inherit_id="web.assets_common">  
                <xpath expr="." position="inside">  
                    <link rel='stylesheet' href="/your_module_name/static/src/css/your_file.css"/>  
                    <script type="text/javascript" src="/your_module_name/static/src/js/your_file.js"></script>  
                </xpath>  
            </template>  
  
        </data>  
     </openerp>
