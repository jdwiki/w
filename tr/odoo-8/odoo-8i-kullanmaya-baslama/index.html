Odoo nedir? #

Odoo (önceden OpenERP ve ondan önce TinyERP olarak biliniyordu), bir açık çekirdekli kurumsal yönetim uygulamaları paketidir. Her büyüklükteki şirketi hedefleyen uygulama paketi, Web Sitesi/E-ticaretten üretime, envantere ve muhasebeye kadar tüm iş ihtiyaçlarını sorunsuz bir şekilde entegre eder. İlk defa bir yazılım editörü bu kadar işlevsel bir kapsama ulaşmayı başardı. Odoo, dünyadaki en yüklü iş yazılımıdır. Odoo, dünya çapında çok küçük şirketlerden (1 kullanıcı) çok büyük şirketlere (300.000 kullanıcı) kadar değişen 2.000.000’den fazla kullanıcı tarafından kullanılmaktadır.

OpenObject çerçevesi ve çekirdek ERP (kurumsal kaynak planlaması) modüllerinin kaynak kodu, Belçika merkezli Odoo S.A tarafından küratörlüğünü yapıyor. Ayrıca, özelleştirilmiş programlama, destek ve diğer hizmetler, aktif bir küresel topluluk ve 500 resmi ortaktan oluşan bir ağ tarafından sağlanmaktadır. Ana Odoo bileşenleri, OpenObject çerçevesi, yaklaşık 30 çekirdek modül (resmi modüller olarak da adlandırılır) ve 3000’den fazla topluluk modülüdür.

Odoo, üniversite derslerinin bir bileşeni olarak kullanılmıştır. Deneyimsel öğrenme üzerine bir araştırma, OpenERP’nin öğretimi desteklemek için tescilli sistemlere uygun bir alternatif sunduğunu ileri sürdü.

Odoo hakkında, bazıları muhasebe veya geliştirme gibi belirli alanları kapsayan birkaç kitap yazılmıştır.

Odoo, Trends Gazelle ve BOSSIE Ödülleri de dahil olmak üzere üç yıl üst üste ödüller aldı.

Veritabanı olarak Python komut dosyası ve PostgreSQL kullanır. Topluluk sürümü, kullanıcı başına yıllık 240 USD/- Enterprise sürümü ve ticari olarak desteklenen bir çevrimiçi sürüm ile desteklenir. Geliştirme deposu GitHub’dadır.

2013 yılında, Odoo S.A.’nın çalışmalarını desteklemek için Odoo topluluk sürümlerinin ve modüllerinin sürekli tanıtımını ve bakımını sağlamak için kar amacı gütmeyen Odoo Community Association kuruldu.

Kurmak #

Odoo üç farklı şekilde kurulabilir:

  1. Paket yükleyiciler (en kolay, daha az esnek)
  2. Kaynak yükleme (kurulum biraz zaman alır, çok esnektir)
  3. docker.com adresinden resmi bir liman işçisi görüntüsü

İlgili tüm bağımlılık gereksinimlerine sahip resmi paketler odoo.com adresinde mevcuttur.

Pencereler

Yükleyici‘yi indirin ve çalıştırın.

Not: Windows 8’de “Windows bilgisayarınızı korudu” başlıklı bir uyarı görebilirsiniz. Daha Fazla Bilgi’yi ve ardından Yine de Çalıştır’ı tıklayın. UAC istemini kabul edin ve çeşitli kurulum adımlarını uygulayın. Odoo, kurulumun sonunda otomatik olarak başlatılacaktır.

Yapılandırma: #

Yapılandırma dosyası %PROGRAMFILES%\Odoo 8.0-id\server\openerp-server.conf konumunda bulunabilir. (id, sistem kullanıcı adınızdır)

Yapılandırma dosyası, uzak bir Postgresql’ye bağlanmak, dosya konumlarını düzenlemek veya bir dbfilter ayarlamak için düzenlenebilir. Yapılandırma dosyasını yeniden yüklemek için Odoo hizmetini Services ‣ odoo sunucusu aracılığıyla yeniden başlatın.

Linux

Debian tabanlı dağıtımlar

Odoo 8.0’ı Debian tabanlı dağıtıma kurmak için aşağıdaki komutları root olarak yürütün:

# wget -O - https://nightly.odoo.com/odoo.key | apt-key add -
# echo "deb http://nightly.odoo.com/8.0/nightly/deb/ ./" >> /etc/apt/sources.list
# apt-get update && apt-get install odoo

Bu, tüm bağımlılıkları otomatik olarak kuracak, Odoo’nun kendisini bir arka plan programı olarak kuracak ve otomatik olarak başlatacaktır.

Dikkat

PDF raporlarını yazdırmak için wkhtmltopdf’i kendiniz yüklemelisiniz: debian depolarında bulunan wkhtmltopdf sürümü üstbilgileri ve altbilgileri desteklemediğinden otomatik olarak yüklenemez. Önerilen sürüm 0.12.1’dir ve wkhtmltopdf indirme sayfasında arşiv bölümünde mevcuttur. Debian Jessie için resmi bir sürüm olmadığı için paketi http://nightly.odoo.com/extra/ adresinde bulabilirsiniz. veya wkhtmltopdf’in indirme sayfasından bu şekilde indirip kurabilirsiniz.

# wget https://bitbucket.org/wkhtmltopdf/wkhtmltopdf/downloads/{path to correct distro and system architecture}
# sudo dpkg -i {.deb package}
# sudo cp /usr/local/bin/wkhtmlto* /usr/bin/

Yapılandırma dosyası /etc/odoo/openerp-server.conf adresinde bulunabilir.

Yapılandırma dosyası düzenlendiğinde, Odoo hizmeti kullanılarak yeniden başlatılmalıdır:

$ sudo hizmeti odoo yeniden başlatma odoo’yu yeniden başlatma: tamam

RPM tabanlı dağılımlar

RHEL tabanlı dağıtımlarda (RHEL, CentOS, Scientific Linux), Odoo’nun tüm bağımlılıklarının kullanılabilir olması için dağıtım havuzlarına EPEL eklenmelidir. CentOS için:

$ sudo yum install -y epel-release

Diğer RHEL tabanlı dağıtım için EPEL belgelerine bakın.

Aşağıda kurulum adımları yer almaktadır.

$ sudo yum install -y postgresql-server
$ sudo postgresql-setup initdb
$ sudo systemctl enable postgresql
$ sudo systemctl start postgresql
$ sudo yum-config-manager --add-repo=https://nightly.odoo.com/8.0/nightly/rpm/odoo.repo
$ sudo yum install -y odoo
$ sudo systemctl enable odoo
$ sudo systemctl start odoo

Dikkat

PDF raporlarını yazdırmak için wkhtmltopdf’i kendiniz yüklemelisiniz: Fedora/CentOS depolarında bulunan wkhtmltopdf sürümü, üstbilgileri ve altbilgileri desteklemediğinden otomatik olarak yüklenemez. wkhtmltopdf indirme sayfasında bulunan sürümü kullanın. Yapılandırma, debian’a benzer şekilde kurulabilir

wget https://bitbucket.org/wkhtmltopdf/wkhtmltopdf/downloads/{path to correct distro and system architecture}
sudo rpm -i  {.rpm package}
sudo cp /usr/local/bin/wkhtmlto* /usr/bin/

Yapılandırma dosyası /etc/odoo/openerp-server.conf adresinde bulunabilir.

Yapılandırma dosyası düzenlendiğinde, Odoo, Systemd aracılığıyla yeniden başlatılmalıdır:

$ sudo systemctl restart odoo

Kaynak Kurulumu

Odoo zip https://nightly.odoo.com/8.0/nightly/src/odoo_8.0.latest.zip adresinden indirilebilir, daha sonra zip dosyasının içeriğini kullanabilmek için sıkıştırılmamış olması gerekir

Git, Odoo’nun farklı sürümleri arasında daha basit güncelleme ve daha kolay geçiş sağlar. Ayrıca modül olmayan yamaların ve katkıların bakımını da kolaylaştırır. Git’in birincil dezavantajı, Odoo projesinin tüm geçmişini içerdiği için bir tarball’dan önemli ölçüde daha büyük olmasıdır.

Git deposu https://github.com/odoo/odoo.git.

Sonra depoyu ile klonlayabilirsiniz.

$ git clone https://github.com/odoo/odoo.git

Bağımlılıkları yükleme

Kaynak yükleme, bağımlılıkların manuel olarak yüklenmesini gerektirir:

Python 2.7. varsayılan olarak dahil olan Linux ve OS X’te

Windows’ta resmi Python 2.7.9 yükleyicisini kullanın.

Python zaten kuruluysa, 2.7.9 olduğundan, önceki sürümlerin daha az kullanışlı olduğundan ve 3.x sürümlerinin Odoo ile uyumlu olmadığından emin olun.

PostgreSQL’i yapılandırma

Kurulumdan sonra bir postgres kullanıcısı oluşturmanız gerekecek: varsayılan olarak tek kullanıcı postgres’tir ve Odoo postgres olarak bağlanmayı yasaklar.

Linux’ta dağıtımınızın paketini kullanın, ardından girişiniz gibi adlı bir postgres kullanıcısı oluşturun:

$ sudo su - postgres -c "createuser -s $USER"

Rol girişi, unix girişiniz ile aynı olduğundan, unix soketleri şifresiz olarak kullanılabilir. OS X’te postgres.app, başlamanın en basit yoludur, ardından Linux’ta olduğu gibi bir postgres kullanıcısı oluşturun

Windows’ta, pencereler için PostgreSQL’i kullanın, ardından PostgreSQL’in bin dizinini (varsayılan: C:\Program Files\PostgreSQL\9.4\bin) PATH’inize ekleyin

pg admin gui’yi kullanarak bir parola ile bir postgres kullanıcısı oluşturun: pgAdminIII’i açın, bir bağlantı oluşturmak için sunucuya çift tıklayın, Düzenle ‣ Yeni Nesne ‣ Yeni Oturum Açma Rolü’nü seçin, Rol Adı alanına kullanıcı adını girin (örn. odoo), ardından Tanım sekmesini açın ve parolayı girin (örn. odoo), ardından Tamam’a tıklayın.

Kullanıcı ve parola, -w ve -r seçenekleri veya yapılandırma dosyası kullanılarak Odoo’ya iletilmelidir.

Gereksinimler.txt dosyasında listelenen Python bağımlılıkları.

Linux’ta python bağımlılıkları, sistemin paket yöneticisi veya pip kullanılarak kurulabilir.

Yerel kod (Pillow, lxml, greenlet, gevent, psycopg2, ldap) kullanan kitaplıklar için, pip’in bağımlılıkları kendilerinin kurabilmesi için geliştirme araçlarını ve yerel bağımlılıkları kurmak gerekebilir. Bunlar Python, Postgres, libxml2, libxslt, libevent, libsasl2 ve libldap2 için -dev veya -devel paketlerinde mevcuttur. Ardından Python bağımlılıkları kurulabilir:

$ pip install -r requirements.txt

OS X’te, Python olmayan bağımlılıkları kurmak için Komut Satırı Araçlarını (xcode-select –install) kurmanız ve ardından seçtiğiniz bir paket yöneticisini (homebrew, macports) indirip kurmanız gerekecektir. pip daha sonra Python bağımlılıklarını Linux’ta olduğu gibi kurmak için kullanılabilir:

$ pip install -r requirements.txt

Windows’ta bazı bağımlılıkları manuel olarak yüklemeniz, gereksinimler.txt dosyasında ince ayar yapmanız ve ardından kalanları yüklemek için pip çalıştırmanız gerekir.

Install psycopg using the installer here http://www.stickpeople.com/projects/python/win-psycopg/

Ardından gereksinimleri.txt dosyasını düzenleyin: psycopg2’yi zaten sahip olduğunuz şekilde kaldırın. derleme gerektirdiğinden isteğe bağlı python-ldap, gevent ve psutil’i kaldırın. Windows altında gerekli olduğundan pypiwin32 ekleyin.

Ardından, bir cmd.exe isteminden aşağıdaki komutu kullanarak bağımlılıkları yüklemek için pip kullanın (\YourOdooPath’i Odoo’yu indirdiğiniz asıl yolla değiştirin):

C:\> cd \YourOdooPath
C:\YourOdooPath> C:\Python27\Scripts\pip.exe install -r requirements.txt

nodejs aracılığıyla daha az CSS

Linux’ta nodejs ve npm’yi kurmak için dağıtımınızın paket yöneticisini kullanın.

Dikkat

Debian wheezy ve Ubuntu 13.10’da ve nodejs’yi manuel olarak kurmanız gerekmeden önce:

$ wget -qO- https://deb.nodesource.com/setup | bash -
$ apt-get install -y nodejs

Daha sonraki debian sürümlerinde (>jessie) ve ubuntu (>14.04) npm paketleri düğümü çağırırken bir sembolik bağlantı eklemeniz gerekebilir, ancak debian ikili düğümleri çağırır

$ apt-get install -y npm
$ sudo ln -s /usr/bin/nodejs /usr/bin/node

Once npm is installed, use it to install less and less-plugin-clean-css:

$ sudo npm install -g less less-plugin-clean-css

on OS X, install nodejs via your preferred package manager (homebrew, macports) then install less and less-plugin-clean-css:

$ sudo npm install -g less less-plugin-clean-css

Windows’ta “nodejs"yi kurun, yeniden başlatın (PATH’i güncellemek için) ve “less” ve “less-plugin-clean-css"yi kurun:

C:\> npm install -g less less-plugin-clean-css

Odoo’yu Çalıştırmak

Tüm bağımlılıklar ayarlandıktan sonra Odoo, odoo.py çalıştırılarak başlatılabilir.

Yapılandırma, komut satırı argümanları veya bir yapılandırma dosyası aracılığıyla sağlanabilir.

Ortak gerekli yapılandırmalar şunlardır:

PostgreSQL host, port, user and password.

Odoo’nun psycopg2’nin varsayılanlarının ötesinde bir varsayılanı yoktur: 5432 numaralı bağlantı noktasındaki bir UNIX soketi üzerinden geçerli kullanıcı ve parola olmadan bağlanır. Varsayılan olarak bu, Linux ve OS X üzerinde çalışmalıdır, ancak UNIX soketlerini desteklemediği için pencerelerde çalışmayacaktır. Custom addons path beyond the defaults, to load your own modules

Windows altında odoo’yu çalıştırmanın tipik bir yolu şudur:

C:\YourOdooPath> python odoo.py -w odoo -r odoo --addons-path=addons,../mymodules --db-filter=mydb$

odoo, odoo postgresql oturum açma ve parola olduğunda, ../mymodules ek eklentiler içeren bir dizin ve mydb localhost’ta hizmet vermek için varsayılan db:8069

*nix sistemlerinde odoo’yu çalıştırmanın tipik bir yolu şudur:

$ ./odoo.py --addons-path=addons,../mymodules --db-filter=mydb$Packaged installers
