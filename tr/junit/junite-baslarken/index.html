Beklenen istisnayı yakala #

Herhangi bir “try catch” bloğu olmadan istisnayı kolayca yakalamak mümkündür.

public class ListTest {
  private final List<Object> list = new ArrayList<>();

  @Test(expected = IndexOutOfBoundsException.class)
  public void testIndexOutOfBoundsException() {
    list.get(0);
  }
}

Yukarıdaki örnek, atılan istisna tarafından taşınan mesajı kontrol etmek istemediğiniz/kontrol etmenize gerek olmadığında, daha basit durumlar için yeterli olacaktır.

İstisna hakkındaki bilgileri kontrol etmek istiyorsanız, try/catch bloğunu kullanmak isteyebilirsiniz:

@Test
public void testIndexOutOfBoundsException() {
    try {
        list.get(0);
        Assert.fail("Should throw IndexOutOfBoundException");
    } catch (IndexOutOfBoundsException ex) {
        Assert.assertEquals("Index: 0, Size: 0", ex.getMessage());
    }
}

Bu örnekte, hiçbir İstisna atılmadığında testin başarısız olmasını sağlamak için her zaman Assert.fail() eklemeniz gerektiğinin farkında olmalısınız.

Daha ayrıntılı durumlar için, JUnit, bu bilgiyi de test edebilen ve aşağıdaki şekilde kullanılan ExpectedException``@Rule özelliğine sahiptir:

public class SimpleExpectedExceptionTest {
     @Rule
     public ExpectedException expectedException = ExpectedException.none();

     @Test
     public void throwsNothing() {
         // no exception expected, none thrown: passes.
     }

     @Test
     public void throwsExceptionWithSpecificType() {
         expectedException.expect(NullPointerException.class);

         throw new NullPointerException();
     }

     @Test
     public void throwsExceptionWithSpecificTypeAndMessage() {
         expectedException.expect(IllegalArgumentException.class);
         expectedException.expectMessage("Wanted a donut.");

         throw new IllegalArgumentException("Wanted a donut.");
     }
}

JUnit5’te istisnaları test etme #

Aynısını JUnit 5’te elde etmek için tamamen yeni bir mekanizma kullanırsınız:

test edilmiş yöntem #

public class Calculator {
    public double divide(double a, double b) {
        if (b == 0.0) {
            throw new IllegalArgumentException("Divider must not be 0");
        }
        return a/b;
    }
}

Test yöntemi #

public class CalculatorTest {
    @Test
    void triangularMinus5() { // The test method does not have to be public in JUnit5
        Calculator calc = new Calculator();

        IllegalArgumentException thrown = assertThrows(
            IllegalArgumentException.class, 
            () -> calculator.divide(42.0, 0.0));
        // If the exception has not been thrown, the above test has failed.

        // And now you may further inspect the returned exception...
        // ...e.g. like this:
        assertEquals("Divider must not be 0", thrown.getMessage());
}

Testleri Yoksaymak #

Bir testi yok saymak için, teste “@Ignore” notunu eklemeniz ve isteğe bağlı olarak açıklamaya bir parametre eklemeniz yeterlidir.

@Ignore("Calculator add not implemented yet.")
@Test
public void testPlus() {
    assertEquals(5, calculator.add(2,3));
}

Testi yorumlamaya veya “@Test” notunu kaldırmaya kıyasla, test çalıştırıcısı yine de bu testi rapor edecek ve yok sayıldığını not edecektir.

JUnit varsayımlar kullanılarak bir test senaryosunu koşullu olarak yok saymak da mümkündür. Örnek bir kullanım durumu, test senaryosunu yalnızca bir geliştirici tarafından belirli bir hata düzeltildikten sonra çalıştırmak olacaktır. Örnek:

import org.junit.Assume;
import org.junit.Assert;
...

@Test 
public void testForBug1234() {

    Assume.assumeTrue(isBugFixed(1234));//will not run this test until the bug 1234 is fixed

    Assert.assertEquals(5, calculator.add(2,3));
}

Varsayılan koşucu, başarısız varsayımlara sahip testleri yok sayılmış olarak ele alır. Diğer koşucuların farklı davranması mümkündür; geçtikleri gibi davranın.

JUnit – Temel açıklama örnekleri #

İşte anlamanız gereken bazı temel JUnit ek açıklamaları: ================================================= =========

@BeforeClass – Run once before any of the test methods in the class, public static void 
@AfterClass – Run once after all the tests in the class has been run, public static void
@Before – Run before @Test, public void
@After – Run after @Test, public void
@Test – This is the test method to run, public void

Kurulum veya Kurulum #

JUnit bir Java kütüphanesi olduğundan, onu yüklemek için tek yapmanız gereken Java projenizin sınıf yoluna birkaç JAR dosyası eklemek ve artık hazırsınız.

Bu iki JAR dosyasını manuel olarak indirebilirsiniz: junit.jar & hamcrest-core.jar.

Maven kullanıyorsanız, basitçe “pom.xml” dosyanıza bir bağımlılık ekleyebilirsiniz:

<dependency>
  <groupId>junit</groupId>
  <artifactId>junit</artifactId>
  <version>4.12</version>
  <scope>test</scope>
</dependency>

Veya Gradle kullanıyorsanız, “build.gradle” dosyanıza bir bağımlılık ekleyin:

apply plugin: 'java'

dependencies {
    testCompile 'junit:junit:4.12'
}

Bundan sonra ilk test sınıfınızı oluşturabilirsiniz:

import static org.junit.Assert.assertEquals;

import org.junit.Test;

public class MyTest {
    @Test
    public void onePlusOneShouldBeTwo() {
        int sum = 1 + 1;
        assertEquals(2, sum);
    }
}

ve komut satırından çalıştırın:

  • Windows java -cp .;junit-X.YY.jar;hamcrest-core-X.Y.jar org.junit.runner.JUnitCore MyTest
  • Linux veya OsX java -cp .:junit-X.YY.jar:hamcrest-core-X.Y.jar org.junit.runner.JUnitCore MyTest

veya Maven ile: mvn testi

Temel birim testi örneği #

Bu örnek, junit kullanarak StringBuilder.toString() öğesinin birim testi için temel bir kurulumdur.

import static org.junit.Assert.assertEquals;

import org.junit.Test;

public class StringBuilderTest {

    @Test
    public void stringBuilderAppendShouldConcatinate()  {
        StringBuilder stringBuilder = new StringBuilder();
        stringBuilder.append("String");
        stringBuilder.append("Builder");
        stringBuilder.append("Test");

        assertEquals("StringBuilderTest", stringBuilder.toString());
    }

}

@Önce sonra #

‘@Before’ ile açıklamalı bir yöntem, ‘@Test’ yöntemlerinin her yürütülmesinden önce yürütülür. Benzer bir “@After” açıklamalı yöntem, her “@Test” yönteminden sonra yürütülür. Bu, tekrar tekrar bir Test ayarı yapmak ve her testten sonra temizlemek için kullanılabilir. Bu nedenle testler bağımsızdır ve hazırlık kodu @Test yöntemi içinde kopyalanmaz.

Örnek:

import static org.junit.Assert.assertEquals;

import java.util.ArrayList;
import java.util.List;

import org.junit.After;
import org.junit.Before;
import org.junit.Test;

public class DemoTest {

    private List<Integer> list;

    @Before
    public void setUp() {
        list = new ArrayList<>();
        list.add(3);
        list.add(1);
        list.add(4);
        list.add(1);
        list.add(5);
        list.add(9);
    }

    @After
    public void tearDown() {
        list.clear();
    }

    @Test
    public void shouldBeOkToAlterTestData() {
        list.remove(0); // Remove first element of list.
        assertEquals(5, list.size()); // Size is down to five
    }

    @Test
    public void shouldBeIndependentOfOtherTests() {
        assertEquals(6, list.size());
    }
}

‘@Before’ veya ‘@After’ ile açıklama eklenen yöntemler, ‘public void’ ve sıfır bağımsız değişken içermelidir.
