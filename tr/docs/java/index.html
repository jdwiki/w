Java Dili Eğitimi

Java programlama dili…

  • Genel amaçlı: Çok çeşitli uygulama alanlarında yazılım yazmak için kullanılmak üzere tasarlanmıştır ve herhangi bir özel alan için özel özelliklerden yoksundur.

  • Sınıf tabanlı: Nesne yapısı sınıflarda tanımlanır. Sınıf örnekleri her zaman sınıf tanımlarında belirtilen alanlara ve yöntemlere sahiptir (bkz. Sınıflar ve Nesneler). Bu, JavaScript gibi sınıf tabanlı olmayan dillerin aksine.

  • Statik olarak yazılmış: derleyici, derleme zamanında değişken türlerine uyulup uyulmadığını kontrol eder. Örneğin, bir yöntem ‘String’ türünde bir argüman bekliyorsa, yöntem çağrıldığında bu argüman aslında bir dize olmalıdır.

  • Nesne yönelimli: Java programında çoğu şey sınıf örnekleridir, yani durum (alanlar) ve davranış demetleridir (veriler üzerinde çalışan ve nesnenin dış dünyayla arayüzünü oluşturan yöntemler).

  • Taşınabilir: javac ile herhangi bir platformda derlenebilir ve elde edilen sınıf dosyaları JVM’ye sahip herhangi bir platformda çalışabilir.

Java, uygulama geliştiricilerinin “bir kez yaz, her yerde çalıştır” (WORA) sağlamasına olanak tanır; bu, derlenmiş Java kodunun yeniden derlemeye gerek kalmadan Java’yı destekleyen tüm platformlarda çalışabileceği anlamına gelir.

Java kodu, sırayla Java Sanal Makinesi (JVM) tarafından yorumlanan bayt koduna (".class" dosyaları) derlenir. Teoride, bir Java derleyicisi tarafından oluşturulan bayt kodu, farklı türde bir bilgisayarda bile herhangi bir JVM’de aynı şekilde çalışmalıdır. JVM, bayt kodunun sıklıkla yürütülen kısımlarını yerel makine komutlarında derlemeyi seçebilir (ve gerçek dünyadaki programlarda olacaktır). Buna “Tam zamanında (JIT) derleme” denir.

Java Sürümleri ve Sürümleri #

Sun / Oracle tarafından tanımlanan Java’nın üç “sürüm"ü vardır:

  • Java Standard Edition (SE), genel kullanım için tasarlanmış sürümdür.
  • Java Enterprise Edition (EE), Java’da “kurumsal düzeyde” hizmetler oluşturmak için bir dizi olanak ekler. Java EE ayrı olarak ele alınmaktadır.
  • Java Micro Edition (ME), Java SE’nin bir alt kümesini temel alır ve sınırlı kaynaklara sahip küçük cihazlarda kullanılmak üzere tasarlanmıştır.

Java SE / EE / ME sürümleri hakkında ayrı bir konu var.

Her sürümün birden fazla sürümü vardır. Java SE sürümleri aşağıda listelenmiştir.

Java’yı yükleme #

Java Kurulumu (Standart Sürüm) hakkında ayrı bir konu var.

Java programlarının derlenmesi ve çalıştırılması #

Şu konularda ayrı başlıklar var:

  • Java kaynak kodu derleniyor
  • Java dağıtımı JAR dosyaları oluşturma dahil
  • Java uygulamalarını çalıştırma
  • Sınıf Yolu

Sıradaki ne? #

Java programlama dilini öğrenmeye ve anlamaya devam etmek için konuların bağlantıları burada. Bu konular, başlamanıza yardımcı olacak Java programlamanın temelleridir.

  • Java’daki İlkel Veri Türleri
  • Java’daki Operatörler
  • Java’daki Dizeler
  • Java’daki Temel Kontrol Yapıları
  • Java’daki Sınıflar ve Nesneler
  • Java’daki Diziler
  • Java kodu standartları

Test yapmak #

Java’nın standart kitaplıkta test için herhangi bir desteği olmasa da, testi desteklemek için tasarlanmış 3. taraf kitaplıkları vardır. En popüler iki birim test kitaplığı şunlardır:

  • JUnit (Resmi Site)
  • TestNG (Resmi Site)

Başka #

  • Java için tasarım kalıpları Tasarım Kalıpları kapsamında ele alınmaktadır.
  • Android için programlama Android kapsamındadır.
  • Java Enterprise Edition teknolojileri Java EE kapsamındadır.
  • Oracle JavaFX teknolojileri JavaFX kapsamındadır.

1. Sürümler bölümünde ömrünün sonu (ücretsiz) tarihi, Oracle’ın genel indirme sitelerine Java SE’nin daha fazla güncellemesini göndermeyi bırakacağı tarihtir. Java SE için genel bakımın yanı sıra kritik hata düzeltmelerine ve güvenlik düzeltmelerine sürekli erişime ihtiyaç duyan müşteriler, Oracle Java SE Desteği aracılığıyla uzun vadeli destek alabilirler.
