android-gradle nedir #

“android-gradle”, Google Tools geliştirici ekibi tarafından resmi olarak sürdürülen bir [“gradle” eklentisidir1 ve 16 Mayıs 2013’te Google I/O’da duyurulmasından bu yana resmi oluşturma aracıdır.

Derlemenizi Gradle ile yapılandırın bölümünü okuyarak temel bilgileri öğrenin.

Ana Özellikler #

Android Gradle Eklentisinin ana özellikleri şunlardır:

  • Bağımlılık yönetimi
  • Kütüphaneli Modüler Projeler
  • Lezzetler ve Yapı Türleri aracılığıyla Varyantlar
  • IDE bağımsız yapılar

genel bakış #

  1. Android Studio indirin ve kurun
  2. açın ve tüm varsayılan ayarlarla yeni bir proje oluşturun

Teoride gradle’ı doğrudan kurabilir, yapılandırma dosyalarını ve dizin yapısını kendiniz oluşturabilirsiniz. pratikte kimse yapmaz bu.

‘android-gradle’ Eklentisi #

Bir gradle projesi genellikle her biri özel bir derleme komut dosyası içeren alt projeye veya * modüllere * bölünür.

Eklenti bağımlılığı genellikle ana / üst düzey build.gradle dosyasında bildirilir:

buildscript {
    // maven repositories for dependencies
    repositories {
        jcenter()
    }
    // build script dependencies
    dependencies {
        // this is the dependency to the android build tools
        classpath 'com.android.tools.build:gradle:2.1.2'
    }
}

allprojects {
    // maven repositories for all sub-project / modules
    repositories {
        jcenter()
    }
}

Bu örnekte ‘android-gradle’ eklenti sürümü, bu satırdan da görebileceğiniz gibi ‘2.1.2’dir:

classpath 'com.android.tools.build:gradle:2.1.2'

Modüller #

Proje, her biri özel bir “build.gradle” komut dosyası içeren modüllere bölünmüştür. settings.gradle dosyası şu modülleri listeler:

include ':app'

İki nokta üst üste : bir şekilde klasör sınırlayıcı olarak kullanılır.

Eklentiyi kullanmak için her modülün “build.gradle” dosyasının en üstüne uygulanması gerekir (örnekte “app”).

Bir Android Uygulaması için:

apply plugin: 'com.android.application'

Android Kitaplığı için:

apply plugin: 'com.android.library'

Ve sonra “android” etiketinde yapılandırılır:

android {
  // gradle-android plugin configuration
}

Temel Android uygulaması Yapılandırması #

Bir uygulama için Android Studio tarafından oluşturulan ‘build.gradle’ şuna benzer:

apply plugin: 'com.android.application'

android {
    // setup which version of the SDK to build against and
    // with what version of the build tools
    compileSdkVersion 23
    buildToolsVersion "23.0.2"

    // default app configurations
    defaultConfig {
        // this is your app unique ID
        applicationId "com.example.myapp"
        
        // devices with lower SDK version can't install the app
        minSdkVersion 14
        // target SDK version, should be the last available one and
        // match the compile one
        targetSdkVersion 23
        
        // integer and string version of your app
        versionCode 1
        versionName "1.0"
    }
    
    // default build types are "debug" and "release"
    buildTypes {
        release {
            // enable / disable proguard optimization
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}

// app dependencies
dependencies {
    // any jar file in the libs folder
    compile fileTree(dir: 'libs', include: ['*.jar'])
    // test dependency
    testCompile 'junit:junit:4.12'
    // runtime dependency on the support library
    compile 'com.android.support:appcompat-v7:24.0.0'
}

Derlemenizi Gradle ile yapılandırın size daha gelişmiş Android Gradle Eklentisi ayarlarını ve seçeneklerini öğretin ve bu ayarın anlamında daha derine inin.

Ürün Lezzetleri ile geçersiz kılınabileceğinden defaultConfig bu şekilde adlandırılır.

‘buildTypes’ etiketi, optimizasyonu etkinleştirerek (proguard gibi) uygulamanızı nasıl oluşturacağınızı ayarlamanıza olanak tanır, Yapı Türleri bölümünü okuyarak daha fazla bilgi edinebilirsiniz. Uygulamanızın imzalanmasını ayarlamak için de kullanılabilir.

Ayrıca Bağımlılıkları Bildirmek hakkında daha fazla bilgi edinmelisiniz. Gördüğünüz gibi ‘bağımlılıklar’ etiketi ‘android’ etiketinin dışında: bu, Android eklentisi tarafından tanımlanmadığı, ancak standart ‘gradle’ olduğu anlamına gelir.

Gradle Sarıcı #

Android Studio ayrıca varsayılan olarak bir gradle wrapper kuracaktır. Bu, doğrudan komut satırından çalıştırabileceğiniz bir araçtır ve ilk kez çalıştırdığınızda yerel bir gradle sürümünü indirecektir.

Uygulamayı derlemeyi başlatmak için daha sonra gradle sarmalayıcıyı başlatabilirsiniz.

Linux / Mac:

./gradlew assemble

Pencereler:

gradlew assemble

Komut dosyası, projenizin kök dizininde bir “gradle” klasöründe bulunan sarmalayıcıyı başlatır:

  • gradle-wrapper.jar: gradle’ı indirmek ve yürütmek için sarmalayıcının kodu
  • ‘gradle-wrapper.properties’, sarmalayıcının hangi gradle sürümünü indirmesi gerektiğini tanımlar
