Selam Dünya #

Gereksinimler #

  1. Haxe araç setinin bir sürümü kurulmalıdır
  2. Sistem yolunuzda Haxe bulunmalıdır
  3. Komut satırı erişilebilir olmalıdır

Kod #

İstediğiniz proje dizinine gidin ve aşağıdaki içeriğe sahip bir “Test.hx” kaynak dosyası oluşturun:

class Test {
    static function main() {
        trace("Hello world");
    }
}

Haxe kaynak dosyalarına modüller denir. Bir modül, modül adıyla aynı tanımlayıcıya sahip bir tür (“soyut”, “sınıf”, “enum”, “arayüz” veya “typedef”) tanımlamalıdır - bu durumda “Test” sınıfı. Bu gereksinim karşılandığında, bir modül isteğe bağlı sayıda farklı tür tanımlayabilir.

Haxe programları, statik “ana” işleviyle belirtildiği gibi bir giriş noktası gerektirir. Giriş noktasını uygulayan sınıf, başlangıç ​​sınıfı veya ana sınıftır. Yine bu durumda ana sınıf “Test” sınıfıdır.

’trace()’ işlevi, kolaylık sağlamak için genel ad alanına maruz kalan genel amaçlı bir günlük tutma işlevidir. Hedef dilin standart çıktı tutamacına çıktı verir (ör. JavaScript için tarayıcı konsolu, C++ için komut satırı). Daha fazla bilgi için API belgelerine bakın.

Uygulamak #

Komut satırınızdan proje klasörüne gidin. Şunu arayarak Haxe’nin ortamınızda yapılandırılıp yapılandırılmadığını test edin:

“haxe –help”

Haxe yorumlayıcısı, herhangi bir belirli hedef dil API’sine dayanmayan kodu test etmek için kullanılabilir. Aşağıdakileri arayarak tercümanı kullanın:

``haxe -main Test –interp''

_Unutmayın, ‘Test’ modülü ‘Test’ başlangıç ​​sınıfını içerir, bu nedenle ‘-main Test’ derleyiciye geçirilir.

Haxe kaynakları (transpile) birkaç farklı dildeki kaynaklara / bayt kodlarına derlenebilir. Aşağıdaki tablo, hedef dili, derleyici bayrağını, bağımsız değişken türünü ve derleme sonucunu görüntüler. Şunu arayarak kullanın:

haxe -main Testi [işaret] [argüman].

Dil bayrak argüman Sonuç
ActionScript 3 -as3 Dizin Kaynak
C# -cs Dizin Kaynak + isteğe bağlı bayt kodu (.exe)
C++ -cpp Dizin Kaynak + isteğe bağlı ikili (yerel)
Flaş -swf Dosya Bayt kodu (.swf)
HL -hl Dosya Kaynak
İkinci -iki Dosya Kaynak
Java -java Dizin Kaynak + isteğe bağlı bayt kodu (.jar)
JavaScript -js Dosya Kaynak
Neko -neko Dosya Bayt kodu (.n)
PHP -php Dizin Kaynak
piton -piton Dosya Kaynak
HashLink -hl Dosya Bayt kodu (.hl)

Buradaki yol argümanlarının “haxe"nin çağrıldığı yola göre olduğuna dikkat edin. İsteğe bağlı bayt kodu/ikili çıktılar, hedef dilin derleyicisinin çağrılmasını içeren ek bir derleme adımından kaçınmak için “-D derlemesiz” bayrakları eklenerek devre dışı bırakılabilir.

Kurulum #

Haxe Windows, Linux ve OS X üzerinde mevcut. İki şekilde dağıtılır:

  • isteğe bağlı bir Neko VM bağımlılığı sağlayan ve “haxe” ve “haxelib” ortam değişkenlerini yapılandıran bir kurucu olarak;
  • ikili dosyalar olarak, yalnızca Haxe derleyicisini ve paket yöneticisini sağlar.

Pencereler #

Yükleyici ve ikili dosyalar Haxe web sitesinde mevcuttur.

Linux #

İkili dosyalar (32-bit ve 64-bit), Haxe web sitesinde mevcuttur.

Haxe Vakfı ayrıca popüler Linux dağıtımları için Haxe ve Neko paketlerinin bakımına resmi olarak katılır. Varsa bu paketleri kullanmanız önerilir.

Ubuntu #

Şu anda desteklenen tüm Ubuntu sürümleri için en son Haxe ve Neko sürümlerini sağlayan Haxe PPA kullanılması önerilir. PPA, Ubuntu tabanlı dağıtımlar için de kullanılabilir.

sudo add-apt-repository ppa:haxe/releases -y
sudo apt-get update
sudo apt-get install haxe -y
mkdir ~/haxelib && haxelib setup ~/haxelib

Neko’nun bir Haxe bağımlılığı olarak kurulduğunu unutmayın.

Debian #

Mevcut kararlı sürümleri yüklemek için aşağıdaki komutları çalıştırın:

sudo apt-get install haxe -y
mkdir ~/haxelib && haxelib setup ~/haxelib

Neko’nun bir Haxe bağımlılığı olarak kurulacağını unutmayın.

Kararsız kanaldan daha yeni sürümleri yüklemek için aşağıdakileri yapın:

  1. /etc/apt/sources.list içine şunu ekleyin

    deb http://httpredir.debian.org/debian unstable main contrib non-free

  2. /etc/apt/preferences.d/ içinde, aşağıdaki içeriğe sahip unstable adlı yeni bir dosya oluşturun:

    Package: *
Pin: release a=unstable
Pin-Priority: 100

Package: haxe neko libneko*
Pin: release a=unstable
Pin-Priority: 999

  3. Yeni eklenen kaynaktan paket dizin dosyalarını çekin:

    sudo apt-get update

  4. Haxe’yi (ve Neko’yu) kurun:

    sudo apt-get install haxe -y

fötr şapka #

Haxe Vakfı, Haxe ve Neko RPM paketlerini Fedora deposunda tutar. Paketler çoğu zaman günceldir. Ancak, yeni bir Haxe sürümü yayınlandığında, güncellenmiş bir paketi Fedora’nın kararlı sürümlerine aktarmak birkaç gün, 2 haftaya kadar sürebilir. Güncelleme etkinlikleri Bodhi Fedora Güncelleme Sisteminde izlenebilir.

Şu anda mevcut olan Haxe ve Neko sürümlerini yüklemek için aşağıdaki komutları çalıştırın:

sudo dnf install haxe -y
mkdir ~/haxelib && haxelib setup ~/haxelib

Neko’nun bir Haxe bağımlılığı olarak kurulduğunu unutmayın.

openSuse #

Haxe Vakfı, openSUSE:Factory deposunda Haxe ve Neko RPM paketlerini tutar. Paketler çoğu zaman günceldir. Ancak, yeni bir Haxe sürümü yayınlandığında, openSUSE:Factory tarafından kabul edilmesi birkaç gün, 2 haftaya kadar sürecektir.

Şu anda mevcut olan Haxe ve Neko sürümlerini yüklemek için aşağıdaki komutları çalıştırın:

sudo zypper install haxe
mkdir ~/haxelib && haxelib setup ~/haxelib

Neko’nun bir Haxe bağımlılığı olarak kurulduğunu unutmayın.

openSUSE:Factory veya bir openSUSE sürümü için mevcut olmayabilecek en son Haxe sürümünü almak için openSUSE Build Service’deki devel:languages:haxe projesini kullanın. Haxe paketi sayfasını ziyaret edin, sağ üst köşedeki “Paketi indir"e tıklayın ve talimatları izleyin. Yine Neko, Haxe’nin bir bağımlılığı olarak da kurulacak.

Arch Linux #

Arch Linux topluluk deposunda Haxe ve Neko paketleri var. Haxe Vakfı, paketlerin güncel kalmasına yardımcı olmaya devam edecek. Ancak, Haxe’nin yeni bir sürümü yayınlandığında, paketin güncellenmesi, paket sağlayıcının uygunluğuna bağlı olarak zaman alacaktır.

Haxe ve Neko’nun şu anda mevcut sürümleri için aşağıdaki sayfaları kontrol edin:

  • Arch Linux’ta Haxe
  • Arch Linux’ta Neko

Şu anda mevcut olan Haxe ve Neko sürümlerini yüklemek için aşağıdaki komutları çalıştırın:

sudo pacman -S haxe
mkdir ~/haxelib && haxelib setup ~/haxelib

Neko’nun bir Haxe bağımlılığı olarak kurulduğunu unutmayın.

OS X #

Yükleyici ve ikili dosyalar Haxe web sitesinde mevcuttur.

Mevcut kararlı Haxe sürümünü Brew paket yöneticisi aracılığıyla kurmak da mümkündür.

brew install haxe
