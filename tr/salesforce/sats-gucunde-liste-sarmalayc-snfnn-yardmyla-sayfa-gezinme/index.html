Satış gücünde liste sarmalayıcı sınıfının yardımıyla Sayfa Gezinme.

  • Sayfalandırma Denetleyicisi

Bu sayfada

  • Sayfalandırma Denetleyicisi

Slaesforce StandardSetController yalnızca sObject Listesini tutar, sarmalayıcı sınıf nesnesinin listesini tutmaz. Aşağıdaki kaynak kodu, satış gücünde Wrapper Class kullanarak Paginating’in kullanımını gösterir.

Sayfalandırma Denetleyicisi #

Kod örneği: Şimdi Sayfalandırma Denetleyicisini Oluşturmaya Başlayın

public with sharing class Pagination {

}

tüm kişileri sayfalama şeklinde göstereceğim ve kişiler üzerinde silme işlemi gerçekleştirmek için seçilen veya seçimi kaldırılan her kişi için bir onay kutusu olmalıdır. Bu yüzden, teması tutmak için bir sarmalayıcı sınıfı ve seçim için Boole Değişkeni oluşturmam gerekiyor. İlk önce denetleyici Sayfalandırması için Wrapper sınıfı oluşturun Aşağıdaki kodu sayfalama denetleyicisine ekleyin.

 public class contactWrapper{
        public Contact cont {get;set;}
        public Boolean isSelected{get;set;}
        public contactWrapper(contact c,Boolean s)
        {
            cont=c;
            isSelected=s;
        }
    }

Şimdi StandardSetController’ın yardımıyla Apex’ten Sayfalandırmaya Veri Alma. StandardSetController, Visualforce sayfalarınızdaki özel kodu büyük ölçüde basitleştirmek için kullanabileceğiniz yerleşik işlevselliğe sahip son derece güçlü bir araçtır. Sunucu yalnızca istenen sayfanın verilerini döndürdüğünden, StandardSetController, özellikle SOQL kullanırken alacağınız görünüm durumuna kıyasla, görünüm durumunu önemli ölçüde azaltabilir.

 Public Integer noOfRecords{get; set;} // Future reference in Visual force Page
    Public Integer size{get;set;}  // Future reference in Visual force Page
    public final Integer Page_Size=10; // Number records in a Page should be displayed 

 public ApexPages.StandardSetController setCon {
        get{
            if(setCon == null){
                size=Page_Size;
                string queryString = 'Select Id,Name, Email, Birthdate, Phone, MobilePhone from Contact order by Name';
                setCon = new ApexPages.StandardSetController(Database.getQueryLocator(queryString));
                setCon.setPageSize(size);
                noOfRecords = setCon.getResultSize();
            }
            return setCon;
        }set;
    }

Artık setCon Değişkeninde ilgili kişilere sahipsiniz, setCon.getRecords() için istediğiniz zaman setCon’dan ilk 10 kişi kaydını alacaktır. Burada size demoyu göstermek için basit bir sarmalayıcı sınıfı oluşturdum. Gereksinimlere göre kendi sarmalayıcı sınıfınızı oluşturabilirsiniz. Ancak her zaman ApexPages.StandardSetController’ın yalnızca List of sObject’i tuttuğunun farkında olmalısınız, sarmalayıcı sınıf object listesini tutmaz. Bu geleceği farklı bir şekilde gerçekleştirmek için aşağıda fazladan kod yazmamın nedeni budur. Aşağıdaki kod, kişi listesini sarmalayıcı sınıf nesneleri listesine dönüştürür, böylece görsel güç sayfasındaki kişileri aradığınızda, sarmalayıcı sınıf nesnesinin listesini alır.

 public list<contactWrapper> contWpr{get;set;} 
    public set<id> selectedContactIds{ get;private set;} // to maintain state of the selected contact
                                                         // through out paginating 

    public Pagination() {
       selectedContactIds=new  set<id>();
    }

      Public list<contactWrapper> getContacts(){
         contWpr =new list<contactWrapper>();
        for(Contact c: (List<Contact>)setCon.getRecords())
            if(selectedContactIds.contains(c.id))
                contWpr.add(new contactWrapper(c,true));
                else
                contWpr.add(new contactWrapper(c,false));
        return contWpr;
    }

Artık sonuç üretmek için kod yazdınız. Peki sayfada nasıl gezinilir? Bu, ApexPages.StandardSetController ile kolay bir adımla yapılabilir. StandardSetController’ın aşağıdaki kod güzelliğine bakın, sayfa numarası, ofset ve limit vb. korumaya gerek yok. Sadece StandardSetController yöntemlerini kullanın. Kişileri sayfalandırmayı neredeyse bitirdiniz. Tüm sayfa işlevselliğimi yerine getirmek için eklediğim son birkaç yöntem. Daha önce de belirttiğim gibi, kişi seçmek ve seçilen kişiler üzerinde silme işlemi gerçekleştirmek için ek onay kutumuz var.

public void contactSelection()
    {
        Id id=(Id)ApexPages.currentPage().getParameters().get('cId');
        if(selectedContactIds.contains(id))
            selectedContactIds.remove(id);
            else
            selectedContactIds.add(id);
    }

 public void deleteContacts()
    {
        List<contact> contactToDelete=[select id from contact where id in :selectedContactIds];
        if(contactToDelete.size()!=0)  //   if(!contactToDelete.isEmpty()) // Best Practice 
           { 
                try {  delete contactToDelete; }  // You may get Exception if you try to delete the 
                                                // related contact ,include try block to avoid error.
                catch(exception ex){ System.debug(ex); }
                refresh();
           }
    }   

 public pageReference refresh() {
        setCon = null;
        selectedContactIds=new set<id>();
        getContacts();
        setCon.setPageNumber(1);
        return null;
    }
