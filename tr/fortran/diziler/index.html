Dizi oluşturucuları #

Sözdizimi ile bir array yapıcı kullanılarak bir rank-1 dizi değeri oluşturulabilir.

(/ ... /)
[ ... ]

[...] formu Fortran 2003’te tanıtıldı ve özellikle karmaşık ifadelerde genellikle okunması daha net olarak kabul edilir. Bu form yalnızca bu örnekte kullanılmıştır.

Bir dizi yapıcısında bulunan değerler, skaler değerler, dizi değerleri veya ima edilen-do döngüleri olabilir.

Oluşturulan dizinin tür ve tür parametreleri, dizi yapıcısındaki değerlerle eşleşir.

[1, 2, 3]      ! A rank-1 length-3 array of default integer type
[1., 2., 3.]   ! A rank-1 length-3 array of default real type
["A", "B"]     ! A rank-1 length-2 array of default character type

integer, parameter :: A = [2, 4]
[1, A, 3]      ! A rank-1 length-4 array of default integer type, with A's elements

integer i
[1, (i, i=2, 5), 6]  ! A rank-1 length-6 array of default integer type with an implied-do

Yukarıdaki formlarda verilen tüm değerler aynı tip ve tip parametresinde olmalıdır. Türlerin veya tür parametrelerinin karıştırılmasına izin verilmez. Aşağıdaki örnekler geçerli değil

[1, 2.]      ! INVALID: Mixing integer and default real
[1e0, 2d0]   ! INVALID: Mixing default real and double precision
[1., 2._dp]  ! INVALID: Allowed only if kind `dp` corresponds to default real
["Hello", "Frederick"]  ! INVALID: Different length parameters

Farklı türler kullanarak bir dizi oluşturmak için dizi için bir tür belirtimi verilmelidir.

[integer :: 1, 2., 3d0]    ! A default integer array
[real(dp) :: 1, 2, 3._sp]  ! A real(dp) array
[character(len=9) :: "Hello", "Frederick"]  ! A length-2 array of length-9 characters

Karakter dizileri için bu son biçim, alternatif olarak boşluk doldurmayı önlemek için özellikle uygundur.

["Hello    ", "Frederick"]  ! A length-2 array of length-9 characters

Sabit adlı bir dizinin boyutu, değerini ayarlamak için kullanılan dizi oluşturucu tarafından ima edilebilir.

integer, parameter :: ids(*) = [1, 2, 3, 4]

ve uzunluk parametreli tipler için uzunluk parametresi kabul edilebilir

character(len=*), parameter :: names(*) = [character(3) :: "Me", "You", "Her"]

Tip belirtimi, sıfır uzunluklu dizilerin yapımında da gereklidir. İtibaren

[ ] ! Not a valid array constructor

tip ve tip parametreleri var olmayan değer kümesinden belirlenemez. Sıfır uzunluklu bir varsayılan tamsayı dizisi oluşturmak için:

[integer :: ]

Dizi oluşturucuları yalnızca sıra-1 dizileri oluşturur. Adlandırılmış bir sabitin değerini ayarlamak gibi zamanlarda, bir ifadede daha yüksek dereceli diziler de gereklidir. İnşa edilmiş bir sıra-1 dizisi ile “yeniden şekillendirme"nin sonucundan daha yüksek sıralı diziler alınabilir

integer, parameter :: multi_rank_ids(2,2) = RESHAPE([1,2,3,4], shape=[2,2])

Bir dizi oluşturucusunda, değer listesindeki herhangi bir dizi ile dizinin değerleri, sanki tek tek öğeler kendilerine dizi öğesi sırasına göre verilmiş gibi olur. Böylece, önceki örnek

integer, parameter :: A = [2, 4]
[1, A, 3]      ! A rank-1 length-4 array of default integer type, with A's elements

eşdeğerdir

[1, 2, 4, 3]   ! With the array written out in array element order

Genellikle yapıcıdaki değerler, iç içe dizi oluşturucuları dahil olmak üzere isteğe bağlı ifadeler olabilir. Böyle bir dizi oluşturucusunun sabit veya belirtim ifadesi olması gibi belirli koşulları karşılaması için, kurucu değerlere kısıtlamalar uygulanır.

Bir dizi oluşturucu olmasa da, belirli dizi değerleri, “spread” içsel işlevi kullanılarak uygun bir şekilde oluşturulabilir. Örneğin

[(0, i=1,10)]  ! An array with 10 default integers each of value 0

ayrıca işlev başvurusunun sonucudur

SPREAD(0, 1, 10)

Temel gösterim #

Herhangi bir tür, ya boyut özniteliği kullanılarak ya da doğrudan dizinin ‘boyutlarını’ (boyutlarını) belirterek bir dizi olarak bildirilebilir:

! One dimensional array with 4 elements
integer, dimension(4) :: foo

! Two dimensional array with 4 rows and 2 columns
real, dimension(4, 2) :: bar

! Three dimensional array
type(mytype), dimension(6, 7, 8) :: myarray

! Same as above without using the dimension keyword
integer :: foo2(4)
real :: bar2(4, 2)
type(mytype) :: myarray2(6, 7, 8)

Çok boyutlu diziyi bildirmenin ikinci yolu, aynı tür farklı sıra/boyutlu dizilerin aşağıdaki gibi bir satırda bildirilmesine izin verir.

real :: pencil(5), plate(3,-2:4), cuboid(0:3,-10:5,6)

İzin verilen maksimum sıralama (boyut sayısı) Fortran 2008 standardında 15’tir ve daha önce 7 idi.

Fortran, dizileri sütun-ana düzende saklar. Yani, “bar"ın elemanları bellekte şu şekilde depolanır:

bar(1, 1), bar(2, 1), bar(3, 1), bar(4, 1), bar(1, 2), bar(2, 2), ...

Fortran’da dizi numaralandırma, 0 ile başlayan C’nin aksine, varsayılan olarak 1 ile başlar. Aslında Fortran’da her boyut için üst ve alt sınırları açıkça belirtebilirsiniz:

integer, dimension(7:12, -3:-1) :: geese

Bu, ilk öğesi ‘geese(7, -3)’ olan ‘(6, 3)’ şeklinde bir dizi bildirir.

2 (veya daha fazla) boyut boyunca alt ve üst sınırlara, “ubound” ve “lbound” içsel işlevleriyle erişilebilir. Gerçekten de ’lbound(geese,2)’ ‘-3’ döndürürken, ‘ubound(geese,1)’ ‘12’ döndürür.

Bir dizinin boyutuna, “size” içsel işleviyle erişilebilir. Örneğin, size(geese, dim = 1), ilk boyutun 6 olan boyutunu döndürür.

Tüm diziler, dizi öğeleri ve dizi bölümleri #

olarak bildirilen diziyi düşünün

real x(10)

O zaman üç ilgi yönümüz var:

  1. “x” dizisinin tamamı;
  2. “x(1)” gibi dizi öğeleri;
  3. Dizi bölümleri, “x(2:6)” gibi.

Bütün diziler #

Çoğu durumda, “x” dizisinin tamamı, dizinin tüm öğelerini tek bir varlık olarak ifade eder. “print *, SUM(x)”, “print *, SIZE(x)” veya “x=1” gibi yürütülebilir ifadelerde görünebilir.

Bütün bir dizi, açıkça şekillendirilmemiş dizilere başvurabilir (yukarıdaki “x” gibi):

function f(y)
  real, intent(out) :: y(:)
  real, allocatable :: z(:)

  y = 1.         ! Intrinsic assignment for the whole array
  z = [1., 2.,]  ! Intrinsic assignment for the whole array, invoking allocation
end function

Varsayılan boyutlu bir dizi, tüm dizi olarak da görünebilir, ancak yalnızca sınırlı durumlarda (başka bir yerde belgelenecektir).

dizi öğeleri #

Bir dizi elemanına, dizinin her sırası için bir tane olmak üzere, tüm dizideki konumu gösteren tamsayı dizinleri verdiği belirtilir:

real x(5,2)
x(1,1) = 0.2
x(2,4) = 0.3

Bir dizi öğesi bir skalerdir.

Dizi bölümleri #

Bir dizi bölümü, iki nokta üst üste içeren bir sözdizimi kullanarak, tüm dizinin bir dizi öğesine (belki de yalnızca bir tanesine) yapılan bir başvurudur:

real x(5,2)
x(:,1) = 0.         ! Referring to x(1,1), x(2,1), x(3,1), x(4,1) and x(5,1)
x(2,:) = 0.         ! Referring to x(2,1), x(2,2)
x(2:4,1) = 0.       ! Referring to x(2,1), x(3,1) and x(4,1)
x(2:3,1:2) = 0.     ! Referring to x(2,1), x(3,1), x(2,2) and x(3,2)
x(1:1,1) = 0.       ! Referring to x(1,1)
x([1,3,5],2) = 0.   ! Referring to x(1,2), x(3,2) and x(5,2)

Yukarıdaki son form bir [vector alt simgesi](https://www.wikiod.com/tr/fortran/diziler#Bütün diziler, dizi elemanları ve dizi bölümleri) kullanır. Bu, diğer dizi bölümlerinin ötesinde bir dizi kısıtlamaya tabidir.

Her dizi bölümü, yalnızca bir öğeye başvurulsa bile bir dizidir. Bu, “x(1:1,1)” sıra 1 dizisidir ve “x(1:1,1:1)” sıra 2 dizisidir.

Dizi bölümleri genel olarak tüm dizinin bir özniteliğine sahip değildir. Özellikle, nerede

real, allocatable :: x(:)
x = [1,2,3]     ! x is allocated as part of the assignment
x = [1,2,3,4]   ! x is dealloacted then allocated to a new shape in the assignment

görev

x(:) = [1,2,3,4,5]   ! This is bad when x isn't the same shape as the right-hand side

izin verilmez: “x(:)”, ancak “x"in tüm öğelerini içeren bir dizi bölümü ayrılabilir bir dizi değildir.

x(:) = [5,6,7,8]

“x” sağ tarafın şeklinde olduğunda iyidir.

Dizilerin dizi bileşenleri #

type t
   real y(5)
end type t

type(t) x(2)

Daha karmaşık ayarlarda tüm dizilere, dizi öğelerine ve dizi bölümlerine de başvurabiliriz.

Yukarıdan, “x” tam bir dizidir. Ayrıca buna sahibiz

x(1)%y        ! A whole array
x(1)%y(1)     ! An array element
x%y(1)        ! An array section
x(1)%y(:)     ! An array section
x([1,2]%y(1)  ! An array section
x(1)%y(1:1)   ! An array section

Bu gibi durumlarda, referansın bir dizi 1’den oluşan birden fazla kısmına sahip olmamıza izin verilmez. Örneğin aşağıdakilere izin verilmez.

x%y             ! Both the x and y parts are arrays
x(1:1)%y(1:1)   ! Recall that each part is still an array section

Ayrılabilir diziler #

Diziler ayrılabilir özniteliğine sahip olabilir:

! One dimensional allocatable array
integer, dimension(:), allocatable :: foo
! Two dimensional allocatable array
real, dimension(:,:), allocatable :: bar

Bu, değişkeni bildirir ancak bunun için herhangi bir alan ayırmaz.

! We can specify the bounds as usual
allocate(foo(3:5))

! It is an error to allocate an array twice
! so check it has not been allocated first
if (.not. allocated(foo)) then
  allocate(bar(10, 2))
end if

Bir değişkene artık ihtiyaç duyulmadığında, serbest bırakılabilir:

deallocate(foo)

Herhangi bir nedenle bir ‘allocate’ ifadesi başarısız olursa, program duracaktır. Durum ‘stat’ anahtar sözcüğü ile kontrol edilirse bu önlenebilir:

real, dimension(:), allocatable :: geese
integer :: status

allocate(geese(17), stat=status)
if (stat /= 0) then
  print*, "Something went wrong trying to allocate 'geese'"
  stop 1
end if

“deallocate” ifadesi de “stat” anahtar kelimesine sahiptir:

deallocate (geese, stat=status)

“status”, ayırma veya ayırma başarılıysa değeri 0 olan bir tamsayı değişkenidir.

Dizi doğası belirtimi: sıra ve şekil #

Bir nesne üzerindeki ‘boyut’ özelliği, o nesnenin bir dizi olduğunu belirtir. Fortran 2008’de beş dizi yapısı vardır:1

  • açık şekil
  • varsayılan şekil
  • varsayılan boyut
  • ertelenmiş şekil
  • ima edilen şekil

Üç sıra-1 diziyi alın2

integer a, b, c
dimension(5) a    ! Explicit shape (default lower bound 1), extent 5
dimension(:) b    ! Assumed or deferred shape
dimension(*) c    ! Assumed size or implied shape array

Bunlarla, bir dizinin doğasını tam olarak belirlemek için daha fazla bağlamın gerekli olduğu görülebilir.

açık şekil #

Açık bir şekil dizisi her zaman bildiriminin şeklidir. Dizi bir alt programa veya “blok” yapısına yerel olarak bildirilmediği sürece, şekli tanımlayan sınırlar sabit ifadeler olmalıdır. Diğer durumlarda, açık bir şekil dizisi, bir alt programın veya “blok” un her çağrışımına göre değişebilen uzantıları kullanan otomatik bir nesne olabilir.

subroutine sub(n)
  integer, intent(in) :: n
  integer a(5)   ! A local explicit shape array with constant bound
  integer b(n)   ! A local explicit shape array, automatic object
end subroutine

varsayılan şekil #

Varsayılan bir şekil dizisi, “ayrılabilir” veya “işaretçi” özniteliği olmayan boş bir argümandır. Böyle bir dizi, şeklini ilişkili olduğu gerçek argümandan alır.

integer a(5), b(10)
call sub(a)   ! In this call the dummy argument is like x(5)
call sub(b)   ! In this call the dummy argument is like x(10)

contains

  subroutine sub(x)
    integer x(:)    ! Assumed shape dummy argument
  end subroutine sub

end

Sahte bir argüman şekil aldığında, prosedürü referans alan kapsamın, bu prosedür için açık bir arayüze sahip olması gerekir.

varsayılan boyut #

Varsayılan bir boyut dizisi, boyutunun gerçek bağımsız değişkeninden varsayıldığı yapay bir bağımsız değişkendir.

subroutine sub(x)
  integer x(*)   ! Assumed size array
end subroutine

Varsayılan boyutlu diziler, varsayılan şekil dizilerinden çok farklı davranır ve bu farklılıklar başka yerlerde belgelenmiştir.

ertelenmiş şekil #

Ertelenmiş şekil dizisi, “ayrılabilir” veya “işaretçi” özniteliğine sahip bir dizidir. Böyle bir dizinin şekli, [allocation](https://www.wikiod.com/tr/fortran/diziler#Ayrılabilir diziler) veya işaretçi ataması ile belirlenir.

integer, allocatable :: a(:)
integer, pointer :: b(:)

ima edilen şekil #

Bir örtülü şekil dizisi, şeklini değerini belirlemek için kullanılan ifadeden alan adlandırılmış bir sabittir.

integer, parameter :: a(*) = [1,2,3,4]

Bu dizi bildirimlerinin yapay argümanlar üzerindeki etkileri başka bir yerde belgelenecektir.

1Fortran 2008’i genişleten bir Teknik Spesifikasyon, altıncı bir dizi niteliği ekler: varsayılan sıralama. Bu burada kapsanmamıştır.

2 Bunlar eşdeğer olarak şu şekilde yazılabilir:

integer, dimension(5) :: a
integer, dimension(:) :: b
integer, dimension(*) :: c

veya

integer a(5)
integer b(:)
integer c(*)

Dizi işlemleri #

Hesaplama hedefleri nedeniyle, dizilerdeki matematiksel işlemler Fortran’da basittir.

Toplama ve çıkarma #

Aynı şekil ve büyüklükteki diziler üzerindeki işlemler matris cebirine çok benzer. Tüm dizinleri döngülerle çalıştırmak yerine, toplama (ve çıkarma) yazılabilir:

real, dimension(2,3) :: A, B, C
real, dimension(5,6,3) :: D
A    = 3.    ! Assigning single value to the whole array
B    = 5.    ! Equivalent writing for assignment
C    = A + B ! All elements of C now have value 8.
D    = A + B ! Compiler will raise an error. The shapes and dimensions are not the same

Dilimleme dizileri de geçerlidir:

integer :: i, j
real, dimension(3,2) :: Mat = 0.
real, dimension(3)   :: Vec1 = 0., Vec2 = 0., Vec3 = 0.
i = 0
j = 0
do i = 1,3
  do j = 1,2
    Mat(i,j) = i+j
  enddo
enddo
Vec1 = Mat(:,1)
Vec2 = Mat(:,2)
Vec3 = Mat(1:2,1) + Mat(2:3,2)

İşlev #

Aynı şekilde, çoğu içsel işlev, bileşen bazında işlem varsayılarak dolaylı olarak kullanılabilir (bu sistematik olmasa da):

real, dimension(2) :: A, B
A(1) = 6
A(2) = 44 ! Random values
B    = sin(A) ! Identical to B(1) = sin(6), B(2) = sin(44).

Çarpma ve bölme #

Ürün ve bölme ile ilgili dikkatli olunmalıdır: * ve / sembollerini kullanan içsel işlemler eleman bazındadır:

real, dimension(2) :: A, B, C
A(1) = 2
A(2) = 4
B(1) = 1
B(2) = 3
C = A*B ! Returns C(1) = 2*1 and C(2) = 4*3

Bu, matris işlemleriyle karıştırılmamalıdır (aşağıya bakın).

Matris işlemleri #

Matris işlemleri içsel prosedürlerdir. Örneğin bir önceki bölümdeki dizilerin matris çarpımı şu şekilde yazılır:

real, dimension(2,1) :: A, B
real, dimension(1,1) :: C
A(1) = 2
A(2) = 4
B(1) = 1
B(2) = 3
C = matmul(transpose(A),B) ! Returns the scalar product of vectors A and B

Karmaşık işlemler, geçici diziler oluşturarak işlevlerin kapsüllenmesine izin verir. Bazı derleyiciler ve derleme seçenekleri tarafından izin verilirken, bu tavsiye edilmez. Örneğin, matris devrik içeren bir çarpım şu şekilde yazılabilir:

real, dimension(3,3) :: A, B, C
A(:) = 4
B(:) = 5
C = matmul(transpose(A),matmul(B,matmul(A,transpose(B)))) ! Equivalent to A^t.B.A.B^T

Gelişmiş dizi bölümleri: alt simge üçlüleri ve vektör alt simgeleri #

[Başka bir örnekte](https://www.wikiod.com/tr/fortran/diziler#Bütün diziler, dizi elemanları ve dizi bölümleri) belirtildiği gibi, bir dizinin öğelerinin bir alt kümesi olarak adlandırılır. bir dizi bölümü referans alınabilir. Bu örnekten sahip olabiliriz

real x(10)
x(:)   = 0.
x(2:6) = 1.
x(3:4) = [3., 5.]

Dizi bölümleri bundan daha genel olabilir. Alt simge üçlüleri veya vektör alt simgeleri şeklini alabilirler.

Alt simge üçüzleri #

Bir alt simge üçlüsü “[bound1]:[bound2][:adım]” biçimini alır. Örneğin

real x(10)
x(1:10) = ...   ! Elements x(1), x(2), ..., x(10)
x(1:) = ...     ! The omitted second bound is equivalent to the upper, same as above
x(:10) = ...    ! The omitted first bound is equivalent to the lower, same as above
x(1:6:2) = ...  ! Elements x(1), x(3), x(5)
x(5:1) = ...    ! No elements: the lower bound is greater than the upper
x(5:1:-1) = ... ! Elements x(5), x(4), x(3), x(2), x(1)
x(::3) = ...    ! Elements x(1), x(4), x(7), x(10), assuming omitted bounds
x(::-3) = ...   ! No elements: the bounds are assumed with the first the lower, negative stride

Bir adım (sıfır olmamalıdır) belirtildiğinde, öğelerin sırası belirtilen ilk sınırla başlar. Adım pozitif (veya negatif) ise, bir diziyi takip eden seçili elemanlar, ikinci sınırdan daha büyük olmayan (veya daha küçük) son eleman alınana kadar adım adım artırılır (veya azaltılır). Adım atlanırsa, bir adım olarak kabul edilir.

İlk sınır, ikinci sınırdan daha büyükse ve adım pozitifse, hiçbir öğe belirtilmez. İlk sınır ikinci sınırdan küçükse ve adım negatifse, hiçbir öğe belirtilmez.

“x” öğesinin her bir öğesi her iki durumda da görünse bile “x(10:1:-1)” ile “x(1:10:1)” aynı olmadığı belirtilmelidir.

Vektör abonelikleri #

Bir vektör alt indisi, bir sıra-1 tamsayı dizisidir. Bu, dizinin değerlerine karşılık gelen bir dizi öğeyi belirtir.

real x(10)
integer i
x([1,6,4]) = ...     ! Elements x(1), x(6), x(4)
x([(i,i=2,4)]) = ... ! Elements x(2), x(3) and x(4)
print*, x([2,5,2])   ! Elements x(2), x(5) and x(2)

Vektör alt simgeli bir dizi bölümü, nasıl kullanılabileceğiyle sınırlıdır:

  • prosedürde tanımlanan sahte bir argümanla ilişkili argüman olmayabilir;
  • bir işaretçi atama ifadesinde hedef olmayabilir;
  • bir veri aktarım bildiriminde dahili bir dosya olmayabilir.

Ayrıca, böyle bir dizi bölümü, aynı eleman iki kez seçildiğinde tanımını içeren bir ifadede görünmeyebilir. Yukardan:

print*, x([2,5,2])   ! Elements x(2), x(5) and x(2) are printed
x([2,5,2]) = 1.      ! Not permitted: x(2) appears twice in this definition

Daha yüksek dereceli dizi bölümleri #

real x(5,2)
print*, x(::2,2:1:-1)  ! Elements x(1,2), x(3,2), x(5,2), x(1,1), x(3,1), x(5,1)
