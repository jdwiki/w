Dahili alt programlar #

Dahili bir alt program olmayan bir program birimi, dahili alt programlar olarak adlandırılan diğer program birimlerini içerebilir.

program prog
  implicit none
contains
  function f()
  end function f
  subroutine g()
  end subroutine g
end program

Böyle bir dahili alt programın bir dizi özelliği vardır:

  • alt programdaki varlıklar ile dış programdaki varlıklar arasında ana bilgisayar ilişkisi var
  • örtük yazım kuralları miras alınır (“örtük yok”, yukarıdaki “f” içinde geçerlidir)
  • dahili alt programlar, ana bilgisayarda bulunan açık bir arayüze sahiptir

Modül alt programları ve harici alt programlar, aşağıdaki gibi dahili alt programlara sahip olabilir:

module mod
  implicit none
contains
  function f()
  contains
    subroutine s()
    end subroutine s
  end function f
end module mod

Fortran programları #

Eksiksiz bir Fortran programı, bir dizi farklı program biriminden oluşur. Program birimleri şunlardır:

  • ana program
  • fonksiyon veya alt program alt programı
  • modül veya alt modül
  • blok veri program birimi

Ana program ve bazı prosedür (fonksiyon veya alt program) alt programları Fortran dışında bir dil tarafından sağlanabilir. Örneğin, bir C ana programı, bir Fortran işlevi alt programı tarafından tanımlanan bir işlevi çağırabilir veya bir Fortran ana programı, C tarafından tanımlanan bir prosedürü çağırabilir.

Bu Fortran program birimlerine ayrı dosyalar veya tek bir dosya içinde verilebilir.

Örneğin, iki dosyayı görebiliriz:

prog.f90

program main
  use mod
end program main

mod.f90

module mod
end module mod

Ve derleyici (doğru çağrıldığında) ana programı modül ile ilişkilendirebilecektir.

Tek dosya birçok program birimi içerebilir

her şey.f90

module mod
end module mod

program prog
  use mod
end program prog

function f()
end function f()

Ancak bu durumda, ana program ve modül söz konusu olduğunda ‘f’ fonksiyonunun hala bir harici fonksiyon olduğuna dikkat edilmelidir. Ancak modüle ana program tarafından erişilebilir.

Yazma kapsamı kuralları, içinde bulundukları dosyaya değil, her bir program birimine uygulanır. Örneğin, her bir kapsam biriminin örtük yazımının olmamasını istiyorsak, yukarıdaki dosyanın şu şekilde yazılması gerekir:

module mod
  implicit none
end module mod

program prog
  use mod
  implicit none
end program prog

function f()
  implicit none
  <type> f
end function f

Modüller ve alt modüller #

Modüller başka bir yerde belgelenmiştir.

Derleyiciler genellikle sözde module dosyaları oluşturur: genellikle aşağıdakileri içeren dosya

module my_module
end module

derleyici tarafından “my_module.mod” gibi bir dosyaya yol açacaktır. Bu gibi durumlarda, bir modüle bir program birimi tarafından erişilebilmesi için, bu modül dosyasının bu sonraki program birimi işlenmeden önce görünür olması gerekir.

Harici prosedürler #

Harici bir prosedür, başka bir program biriminin dışında veya Fortran dışında bir yolla tanımlanan bir prosedürdür.

Gibi bir dosyada bulunan işlev

integer function f()
  implicit none
end function f

harici bir fonksiyondur.

Harici prosedürler için, bunların varlığı bir arayüz bloğu kullanılarak bildirilebilir (belirli bir arayüze verilen)

program prog
  implicit none
  interface
    integer function f()
  end interface
end program prog

veya örtük bir arayüz vermek için bir bildirim ifadesi ile

program prog
  implicit none
  integer, external :: f
end program prog

ya da

program prog
  implicit none
  integer f
  external f
end program prog

“harici” özniteliği gerekli değildir:

program prog
  implicit none
  integer i
  integer f
  i = f()   ! f is now an external function
end program prog

Kaynak kod dosyaları #

Kaynak kod dosyası, derleyici tarafından işlenecek (genellikle) bir düz metin dosyasıdır. Bir kaynak kod dosyası, en fazla bir ana program ve herhangi bir sayıda modül ve harici alt program içerebilir. Örneğin, bir kaynak kod dosyası aşağıdakileri içerebilir:

module mod1
end module mod1

module mod2
end module mod2

function func1()    ! An external function
end function func1

subroutine sub1()   ! An external subroutine
end subroutine sub1

program prog        ! The main program starts here...
end program prog    ! ... and ends here

function func2()    ! An external function
end function func2

Burada hatırlamalıyız ki, harici alt programlar modüller ve ana program ile aynı dosyada verilmiş olsa da, harici alt programlar başka herhangi bir bileşen tarafından açıkça bilinmez.

Alternatif olarak, tek tek bileşenler birden çok dosyaya yayılabilir ve hatta farklı zamanlarda derlenebilir. Derleyici belgeleri, birden çok dosyanın tek bir programda nasıl birleştirileceğine ilişkin okunmalıdır.

Tek bir kaynak kod dosyası, [sabit form](https://www.wikiod.com/tr/fortran/tarihi-ozelliklere-modern-alternatifler#Sabit Kaynak Formu) içerebilir veya serbest biçimli kaynak kodu: bunlar karıştırılamaz, ancak derleme zamanında birleştirilen birden çok dosyanın farklı stilleri olabilir.

Derleyiciye kaynak formu belirtmek için genellikle iki seçenek vardır:

  • dosya adı son eki seçimi
  • derleyici bayraklarının kullanımı

Sabit veya serbest biçimli kaynağı belirtmek için derleme zamanı bayrağı, derleyicinin belgelerinde bulunabilir.

Önemli dosya adı sonekleri de derleyicinin belgelerinde bulunabilir, ancak genel bir kural olarak ‘file.f90’ adlı bir dosyanın serbest biçimli kaynak içerdiği, ‘file.f’ dosyasının ise sabit biçimli olduğu kabul edilir. kaynak.

Serbest biçimli kaynağı belirtmek için “.f90” son ekinin kullanılması (Fortran 90 standardında tanıtılmıştır) genellikle programcıyı kaynak kodunun uyduğu dil standardını belirtmek için son eki kullanmaya teşvik eder. Örneğin, “.f03” veya “.f08” sonekleri olan dosyaları görebiliriz. Bu genellikle kaçınılmalıdır: Fortran 2003 kaynaklarının çoğu aynı zamanda Fortran 77, Fortran 90/5 ve Fortran 2008 ile de uyumludur. Ayrıca, birçok derleyici bu tür son ekleri otomatik olarak dikkate almaz.

Derleyiciler ayrıca genellikle yerleşik bir kod ön işlemcisi sunar (genellikle cpp’ye dayanır). Yine, önişlemcinin derlemeden önce çalıştırılması gerektiğini belirtmek için bir derleme zamanı bayrağı kullanılabilir, ancak kaynak kod dosyası soneki de bu tür ön işleme gereksinimini gösterebilir.

Büyük/küçük harfe duyarlı dosya sistemleri için ‘file.F’ dosyası genellikle önceden işlenecek sabit biçimli bir kaynak dosya ve ‘file.F90’ ön işlenecek serbest biçimli bir kaynak dosya olarak alınır. Daha önce olduğu gibi, bu tür bayraklar ve dosya sonekleri için derleyicinin belgelerine başvurulmalıdır.
