Kayan noktalı sayıların kesinliği #

“real” türündeki kayan noktalı sayıların gerçek değeri olamaz. Belirli bir ondalık basamağa kadar gerçek sayıları temsil edebilirler.

FORTRAN 77 iki kayan nokta türünü garanti eder ve daha yeni standartlar en az iki gerçek türü garanti eder. Gerçek değişkenler olarak bildirilebilir

real x
double precision y

Buradaki “x” varsayılan türden bir gerçeldir ve “y”, “x"den daha büyük ondalık kesinliği olan bir tür gerçelidir. Fortran 2008’de, “y"nin ondalık kesinliği en az 10 ve ondalık üs aralığı en az 37’dir.

 real, parameter             :: single = 1.12345678901234567890
 double precision, parameter :: double = 1.12345678901234567890d0
 
 print *, single
 print *, double

baskılar

   1.12345684    
   1.1234567890123457

varsayılan yapılandırmayı kullanan ortak derleyicilerde.

Çift kesinlik sabitindeki “d0"a dikkat edin. Üssü belirtmek için ’e’ yerine ’d’ içeren gerçek bir hazır bilgi, çift kesinliği belirtmek için kullanılır.

! Default single precision constant
1.23e45
! Double precision constant
1.23d45

Fortran 90, türler kullanarak parametreli “gerçek” türleri tanıttı. Gerçek türün türü, sabit veya değişmez sabit adlı bir tamsayıdır:

real(kind=real_kind) :: x

ya da sadece

real(real_kind) :: x

Bu ifade, x’in, “gerçek_tür” değerine bağlı olarak belirli bir kesinlikle “gerçek” türünde olduğunu bildirir.

Kayan nokta değişmezleri, bir sonek kullanılarak belirli bir türle bildirilebilir

1.23456e78_real_kind

“gerçek_tür"ün tam değeri standartlaştırılmamıştır ve derleyiciden derleyiciye farklılık gösterir. Herhangi bir gerçek değişken veya sabitin türünü sorgulamak için tür () işlevi kullanılabilir:

print *, kind(1.0), kind(1.d0)

tipik olarak yazdırır

“4 8”

veya

“1 2”

derleyiciye göre değişir.

Tür numaraları birkaç şekilde ayarlanabilir:

  1. Tek (varsayılan) ve çift kesinlik:

    integer, parameter :: single_kind = kind(1.)
integer, parameter :: double_kind = kind(1.d0)

  2. Gerekli ondalık kesinliği belirtmek için ‘selected_real_kind([p, r])’ içsel işlevini kullanma. Döndürülen tür, en az “p” basamaklı kesinliğe sahiptir ve en az “r” üssüne izin verir.

    integer, parameter :: single_kind = selected_real_kind( p=6, r=37 )
integer, parameter :: double_kind = selected_real_kind( p=15, r=200 )

  3. Fortran 2003’ten başlayarak, gerçek türlerin eşlik eden C derleyicisinin “float”, “double” veya “long_double” türleriyle birlikte çalışabilir olmasını sağlamak için “ISO_C_Binding” iç modülü aracılığıyla önceden tanımlanmış sabitler kullanılabilir:

    use ISO_C_Binding

integer, parameter :: single_kind = c_float
integer, parameter :: double_kind = c_double
integer, parameter :: long_kind = c_long_double

  4. Fortran 2008’den başlayarak, önceden tanımlanmış sabitler, ISO_Fortran_env iç modülü aracılığıyla kullanılabilir. Bu sabitler, bit cinsinden belirli depolama boyutuna sahip gerçek türler sağlar.

    use ISO_Fortran_env

integer, parameter :: single_kind = real32
integer, parameter :: double_kind = real64
integer, parameter :: quadruple_kind = real128

Derleyicide belirli bir tür mevcut değilse, ‘selected_real_kind()’ tarafından döndürülen değer veya tamsayı sabitinin değeri ‘-1’ olur.

İçsel türler #

Aşağıdakiler Fortran’a intrinsic veri türleridir:

integer
real
character
complex
logical

“integer”, “real” ve “complex” sayısal türlerdir.

“karakter”, karakter dizilerini depolamak için kullanılan bir türdür.

“mantıksal”, “.true.” veya “.false.” ikili değerlerini depolamak için kullanılır.

Tüm sayısal ve mantıksal içsel türler, türler kullanılarak parametrelendirilir.

integer(kind=specific_kind)

ya da sadece

integer(specific_kind)

burada “belirli_tür” sabit adlı bir tam sayıdır.

Karakter değişkenleri tür parametresine sahip olmanın yanı sıra bir uzunluk parametresine de sahiptir:

character char

‘char’ öğesini varsayılan türden bir uzunluk-1 karakter değişkeni olarak bildirir, oysa

character(len=len) name

’name’nin varsayılan tür ve uzunluk ’len’ olan bir karakter değişkeni olduğunu bildirir. Tür de belirtilebilir

character(len=len, kind=specific_kind) name
character(kind=specific_kind) char

“name"nin “tür” ve uzunluk “len” türünde bir karakter olduğunu bildirir. “char”, “tür” türünde bir uzunluk-1 karakteridir.

Alternatif olarak, karakter bildirimi için kullanılmayan form

character*len  name

“name"nin “len” uzunluğunda ve varsayılan karakter türünde olduğunu bildiren eski kodda görülebilir.

İçsel tipte bir değişkenin bildirimi yukarıdaki biçimde olabilir, ancak ’type(…)’ biçimini de kullanabilir:

integer i
real x
double precision y

eşdeğerdir (ancak büyük ölçüde tercih edilir)

type(integer) i
type(real) x
type(double precision) y

Türetilmiş veri türleri #

Yeni bir tür tanımlayın, “mytype”:

type :: mytype
  integer :: int
  real    :: float
end type mytype

mytype: türünde bir değişken bildirin

type(mytype) :: foo

Türetilmiş bir türün bileşenlerine % operatörü1 ile erişilebilir:

foo%int = 4
foo%float = 3.142

Bir Fortran 2003 özelliği (henüz tüm derleyiciler tarafından uygulanmamıştır), parametreli veri türlerini tanımlamaya izin verir:

type, public :: matrix(rows, cols, k)
  integer, len :: rows, cols
  integer, kind :: k = kind(0.0)
  real(kind = k), dimension(rows, cols) :: values
end type matrix

Türetilmiş tür “matris”, tür adından sonra parantez içinde listelenen üç tür parametresine sahiptir (bunlar “satırlar”, “sütunlar” ve “k"dir). Her tür parametresinin bildiriminde, tür (’tür’) veya uzunluk (’len’) türü parametreleri olup olmadığı belirtilmelidir.

İçsel türlerinki gibi tür türü parametreleri sabit ifadeler olmalıdır, oysa içsel karakter değişkeninin uzunluğu gibi uzunluk türü parametreleri yürütme sırasında değişebilir.

“k” parametresinin varsayılan bir değere sahip olduğuna dikkat edin, bu nedenle “matris” türünde bir değişken bildirildiğinde aşağıdaki gibi sağlanabilir veya atlanabilir.

type (matrix (55, 65, kind=double)) :: b, c ! default parameter provided
type (matrix (rows=40, cols=50)     :: m    ! default parameter omitted

Türetilmiş bir türün adı “çift kesinlik” olamaz veya herhangi bir içsel türle aynı olamaz.

  1. Birçok kişi, Fortran’ın bileşen erişim operatörü olarak neden daha yaygın olan ‘.’ yerine ‘%’ kullandığını merak ediyor. Bunun nedeni, . öğesinin operatör sözdizimi tarafından alınmış olmasıdır, yani .not., .and., .my_own_operator..

Değişmez sabitler #

Program birimleri genellikle değişmez sabitleri kullanır. Bunlar gibi bariz durumları kapsar

print *, "Hello", 1, 1.0

Bir durum dışında, her değişmez sabit, sözdizimi tarafından verilen tür, tür parametreleri ve değeri olan bir skalerdir.

Tamsayı değişmez sabitleri şu şekildedir:

1
-1
-1_1   ! For valid kind parameter 1
1_ik   ! For the named constant ik being a valid kind paramter

Gerçek değişmez sabitler şu şekildedir:

1.0    ! Default real
1e0    ! Default real using exponent format
1._1   ! Real with kind parameter 1 (if valid)
1.0_sp ! Real with kind paramter named constant sp
1d0    ! Double precision real using exponent format
1e0_dp ! Real with kind named constant dp using exponent format

Karmaşık değişmez sabitler şu şekildedir:

(1, 1.)       ! Complex with integer and real components, literal constants
(real, imag)  ! Complex with named constants as components

Gerçek ve sanal bileşenlerin her ikisi de tamsayıysa, karmaşık değişmez sabit, varsayılan karmaşıktır ve tamsayı bileşenleri varsayılan gerçeğe dönüştürülür. Bir bileşen gerçekse, karmaşık değişmez sabitin tür parametresi gerçeğin parametresidir (ve tamsayı bileşeni bu gerçek türe dönüştürülür). Her iki bileşen de gerçekse, karmaşık değişmez sabit, en büyük kesinlikle gerçek türdendir.

Mantıksal değişmez sabitler

.TRUE.     ! Default kind, with true value
.FALSE.    ! Default kind, with false value
.TRUE._1   ! Of kind 1 (if valid), with true value
.TRUE._lk  ! Of kind named constant lk (if valid), with true value

Karakter değişmez değerleri kavram olarak biraz farklıdır, çünkü tür belirteci değerden önce gelir

"Hello"       ! Character value of default kind
'Hello'       ! Character value of default kind
ck_"Hello"    ! Character value of kind ck
"'Bye"        ! Default kind character with a '
'''Bye'       ! Default kind character with a '
""            ! A zero-length character of default kind

Yukarıda önerildiği gibi, karakter değişmez sabitleri kesme işaretleri veya tırnak işaretleri ile sınırlandırılmalı ve başlangıç ​​ve bitiş işaretçisi eşleşmelidir. Değişmez kesme işaretleri, tırnak işaretleri içinde bulunarak veya çift görünerek dahil edilebilir. Tırnak işaretleri için de aynı.

BOZ sabitleri, yalnızca bir değer belirtmeleri bakımından yukarıdakilerden farklıdır: tür veya tür parametresi yoktur. BOZ sabiti bir bit modelidir ve şu şekilde belirtilir:

B'00000'    ! A binary bit pattern
B"01010001" ! A binary bit pattern
O'012517'   ! An octal bit pattern
O"1267671"  ! An octal bit pattern
Z'0A4F'     ! A hexadecimal bit pattern
Z"FFFFFF"   ! A hexadecimal bit pattern

BOZ değişmez sabitleri, göründükleri yerde sınırlıdır: “veri” ifadelerinde sabitler ve bir dizi içsel prosedür olarak.

Varsayılan ve ertelenmiş uzunluk tipi parametreleri #

Uzunluk parametreli karakter türü veya türetilmiş türdeki değişkenler, varsayılan veya ertelenmiş uzunluk parametresine sahip olabilir. ’name’ karakter değişkeni

character(len=len) name

yürütme boyunca “len” uzunluğundadır. Tersine, uzunluk belirteci şunlardan biri olabilir:

character(len=*) ...  ! Assumed length

veya

character(len=:) ...  ! Deferred length

Varsayılan uzunluk değişkenleri, uzunluklarını başka bir varlıktan alır.

fonksiyonda

function f(dummy_name)
  character(len=*) dummy_name
end function f

kukla bağımsız değişken “kukla_adı” gerçek bağımsız değişkenin uzunluğuna sahiptir.

içinde adlandırılmış sabit ‘const_name’

character(len=*), parameter :: const_name = 'Name from which length is assumed'

sağ taraftaki sabit ifade ile verilen uzunluğa sahiptir.

Ertelenen uzunluk türü parametreleri yürütme sırasında değişebilir. Ertelenmiş uzunluğa sahip bir değişken, “ayrılabilir” veya “işaretçi” özniteliğine sahip olmalıdır

character(len=:), allocatable :: alloc_name
character(len=:), pointer :: ptr_name

Böyle bir değişkenin uzunluğu aşağıdaki yollardan herhangi biriyle ayarlanabilir

allocate(character(len=5) :: alloc_name, ptr_name)
alloc_name = 'Name'         ! Using allocation on intrinsic assignment
ptr_name => another_name    ! For given target

Uzunluk parametreli türetilmiş türler için sözdizimi benzerdir

  type t(len)
    integer, len :: len
    integer i(len)
  end type t

  type(t(:)), allocatable :: t1
  type(t(5)) t2

  call sub(t2)
  allocate(type(t(5)) :: t1)

contains

  subroutine sub(t2)
    type(t(*)), intent(out) :: t2
  end subroutine sub

end

Karakter alt dizelerine erişme #

Karakter varlığı için

character(len=5), parameter :: greeting = "Hello"

sözdizimi ile bir alt dizeye başvurulabilir

greeting(2:4)  ! "ell"

Tek bir harfe ulaşmak için yazmak yeterli değil

greeting(1)    ! This isn't the letter "H"

ancak

greeting(1:1)  ! This is "H"

Bir karakter dizisi için

character(len=5), parameter :: greeting(2) = ["Hello", "Yo!  "]

gibi alt dize erişimimiz var

greeting(1)(2:4)  ! "ell"

ancak bitişik olmayan karakterlere başvuramayız

greeting(:)(2:4)  ! The parent string here is an array

Değişmez sabitlerin alt dizilerine bile erişebiliriz

"Hello"(2:4)

Bir karakter değişkeninin bir kısmı, değişken olarak bir alt dizgi kullanılarak da tanımlanabilir. Örneğin

integer :: i=1
character :: filename = 'file000.txt'

filename(9:11) = 'dat'
write(filename(5:7), '(I3.3)') i

Karmaşık bileşenlere erişme #

karmaşık varlık

complex, parameter :: x = (1., 4.)

gerçek kısım ‘1.’ ve karmaşık kısım ‘4.’ var. Bu bireysel bileşenlere şu şekilde erişebiliriz:

real(x)  ! The real component
aimag(x) ! The complex component
x%re     ! The real component
y%im     ! The complex component

x%.. formu Fortran 2008 için yenidir ve derleyicilerde yaygın olarak desteklenmez. Ancak bu form, karmaşık bir değişkenin bireysel bileşenlerini doğrudan ayarlamak için kullanılabilir.

complex y
y%re = 0.
y%im = 1.

Bildirim ve özellikler #

Buradaki konular ve örnekler boyunca birçok değişken, fonksiyon vb. bildirimleri göreceğiz.

Veri nesnelerinin adlarının yanı sıra öznitelikleri olabilir. Bu konuda kapsanan aşağıdaki gibi beyan ifadeleridir.

integer, parameter :: single_kind = kind(1.)

bu, “single_kind” nesnesine “parametre” niteliğini verir (bunu adlandırılmış bir sabit yapar).

gibi daha birçok özellik vardır.

  • “hedef”
  • “işaretçi”
  • “isteğe bağlı”
  • “kaydet”

Nitelikler, öznitelik belirtim ifadeleri olarak adlandırılanlarla belirtilebilir.

integer i    ! i is an integer (of default kind)...
pointer i    ! ... with the POINTER attribute...
optional i   ! ... and the OPTIONAL attribute

Ancak, genellikle bu öznitelik belirtim ifadelerini kullanmaktan kaçınmanın daha iyi olduğu kabul edilir. Netlik için, nitelikler tek bir bildirimin parçası olarak belirtilebilir.

integer, pointer, optional :: i

Bu aynı zamanda örtük yazma kullanma eğilimini de azaltır.

Çoğu durumda bu Fortran belgelerinde bu tek bildirim ifadesi tercih edilir.
