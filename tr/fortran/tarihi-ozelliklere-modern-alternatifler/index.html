Aritmetik if ifadesi #

Aritmetik “if” ifadesi, bir aritmetik ifadenin sonucuna bağlı olarak üç dalın kullanılmasına izin verir.

if (arith_expr) label1, label2, label3

Bu “if” ifadesi, kontrol akışını bir koddaki etiketlerden birine aktarır. ‘arith_expr’ sonucu negatif ise ’label1’, sonuç sıfır ise ’label2’, sonuç pozitif ise son ’label3’ uygulanır. Aritmetik ‘if’, üç etiketi de gerektirir, ancak etiketlerin yeniden kullanılmasına izin verir, bu nedenle bu ifade iki dallı bir ‘if’ olarak basitleştirilebilir.

Örnekler:

if (N * N - N / 2) 130, 140, 130

if (X) 100, 110, 120

Şimdi bu özellik, ‘if’ ifadesi ve ‘if-else’ yapısı tarafından sunulan aynı işlevsellik ile eskidir. Örneğin, fragman

    if (X) 100, 110, 120
100 print*, "Negative"
    goto 200
110 print*, "Zero"
    goto 200
120 print*, "Positive"
200 continue

“if-else” yapısı olarak yazılabilir

if (X<0) then
  print*, "Negative"
else if (X==0) then
  print*, "Zero"
else
  print*, "Positive"
end if

için bir “if” ifadesinin değiştirilmesi

    if (X) 100, 100, 200
100 print *, "Negative or zero"
200 continue

belki

if (X<=0) print*, "Negative or zero"

Örtülü değişken türleri #

Sabit Kaynak Formu #

Fortran başlangıçta 80 sütunlu bir delikli karta dayalı olarak sabit biçim formu için tasarlandı:

Evet: Bu, yazarın kendi kodunun bir satırıdır

Bunlar, şuna çok benzeyen bir kart delme makinesinde oluşturuldu:

Görüntüler yazara ait orijinal fotoğraflardır

Gösterilen örnek kartta gösterildiği gibi format, ifade etiketleri için ayrılmış ilk beş sütuna sahipti. İlk sütun, yorumları C harfiyle belirtmek için kullanıldı. Altıncı sütun, bir ifadenin devamını belirtmek için kullanıldı (sıfır ‘0’ dışında herhangi bir karakter ekleyerek). Son 8 sütun, kart tanımlama ve sıralama için kullanıldı; bu, kart destenizi yere düşürürseniz oldukça değerliydi! Delikli kartlar için karakter kodlaması yalnızca sınırlı sayıda karakter içeriyordu ve yalnızca büyük harfliydi. Sonuç olarak, Fortran programları şöyle görünüyordu:

       DIMENSION A(10)                                                    00000001
C THIS IS A COMMENT STATEMENT TO EXPLAIN THIS EXAMPLE PROGRAM             00000002
       WRITE (6,100)                                                      00000003
 100   FORMAT(169HTHIS IS A RATHER LONG STRING BEING OUTPUT WHICH GOES OVE00000004
      1R MORE THAN ONE LINE, AND USES THE STATEMENT CONTINUATION MARKER IN00000005
      2COLUMN 6, AND ALSO USES HOLLERITH STRING FORMAT)                   00000006
       STOP                                                               00000007
       END                                                                00000008

Boşluk karakteri de Hollerith karakter sabiti dışında (yukarıda gösterildiği gibi) her yerde yok sayılırdı. Bu, ayrılmış kelimelerin ve sabitlerin içinde boşlukların oluşabileceği veya tamamen gözden kaçırılabileceği anlamına geliyordu. Bu, aşağıdakiler gibi oldukça yanıltıcı ifadelerin yan etkisine sahipti:

       DO 1 I = 1.0

“DO1I” değişkenine bir atamadır, oysa:

       DO1I = 1,0

aslında “I” değişkeni üzerinde bir “DO” döngüsüdür.

Modern Fortran artık bu sabit girdi biçimine ihtiyaç duymaz ve herhangi bir sütun kullanarak serbest biçime izin verir. Yorumlar artık bir ifade satırına da eklenebilen bir ! ile gösterilir. Boşluklara artık hiçbir yerde izin verilmiyor ve diğer birçok dilde olduğu gibi ayırıcı olarak kullanılmaları gerekiyor. Yukarıdaki program modern Fortran’da şu şekilde yazılabilir:

! This is a comment statement to explain this example program
Print *,"THIS IS A RATHER LONG STRING BEING OUTPUT WHICH no longer GOES OVER MORE THAN ONE LINE, AND does not need to USE THE STATEMENT CONTINUATION MARKER IN COLUMN 6, or the HOLLERITH STRING FORMAT"

Eski tarz devam artık kullanılmasa da, yukarıdaki örnek çok uzun ifadelerin yine de oluşacağını göstermektedir. Modern Fortran, devamın sonunda ve başında bir “&” sembolü kullanır. Örneğin, yukarıdakileri daha okunaklı bir biçimde yazabiliriz:

! This is a comment statement to explain this example program
Print *,"THIS IS A RATHER LONG STRING BEING OUTPUT WHICH still &
         &GOES OVER MORE THAN ONE LINE, AND does need to USE THE STATEMENT &
         &CONTINUATION notation"

Ortak Bloklar #

Fortran’ın ilk biçimlerinde, alt programlardan ve işlevlerden görülebilen global değişken deposu yaratmanın tek mekanizması ‘ORTAK’ blok mekanizmasını kullanmaktır. Bu, değişken dizilerinin ad olmasına ve ortak olarak paylaşılmasına izin verdi.

Adlandırılmış ortak bloklara ek olarak, boş (adsız) bir ortak blok da olabilir.

Boş bir ortak blok şöyle bildirilebilir

common i, j

adlandırılmış blok ‘değişkenleri’ şöyle bildirilebilirken

common /variables/ i, j

Tam bir örnek olarak, değerler ekleyip çıkarabilen rutinler tarafından kullanılan bir yığın deposu hayal edebiliriz:

       PROGRAM STACKING
       COMMON /HEAP/ ICOUNT, ISTACK(1023)
       ICOUNT = 0
       READ *, IVAL
       CALL PUSH(IVAL)
       CALL POP(IVAL)
       END

       SUBROUTINE PUSH(IVAL)
       COMMON /HEAP/ ICOUNT, ISTACK(1023)
       ICOUNT = ICOUNT + 1
       ISTACK(ICOUNT) = IVAL
       RETURN
       END

       SUBROUTINE POP(IVAL)
       COMMON /HEAP/ ICOUNT, ISTACK(1023)
       IVAL = ISTACK(ICOUNT)
       ICOUNT = ICOUNT - 1
       RETURN
       END

Bir değişkenin türünü örtük olarak bildirmek ve “boyut” niteliğini belirtmek için ortak ifadeler kullanılabilir. Bu davranış tek başına genellikle yeterli bir kafa karışıklığı kaynağıdır. Ayrıca, program birimleri arasında tekrarlanan tanımlar için ima edilen depolama ilişkisi ve gereksinimleri, ortak blokların kullanımını hataya açık hale getirir.

Son olarak, ortak bloklar içerdikleri nesnelerde çok kısıtlıdır. Örneğin, ortak bir bloktaki bir dizi açık boyutta olmalıdır; tahsis edilebilir nesneler oluşmayabilir; türetilmiş türler varsayılan başlatmaya sahip olmamalıdır.

Modern Fortran’da bu değişken paylaşımı modüller kullanılarak yapılabilir. Yukarıdaki örnek şu şekilde yazılabilir:

module heap
  implicit none
  ! In Fortran 2008 all module variables are implicitly saved
  integer, save :: count = 0
  integer, save :: stack(1023)
end module heap

program stacking
  implicit none
  integer val
  read *, val
  call push(val)
  call pop(val)

contains
  subroutine push(val)
    use heap, only : count, stack
    integer val
    count = count + 1
    stack(count) = val
  end subroutine push

  subroutine pop(val)
    use heap, only : count, stack
    integer val
    val = stack(count)
    count = count - 1
  end subroutine pop
end program stacking

Adlandırılmış ve boş ortak bloklar biraz farklı davranışlara sahiptir. Notun:

  • adlandırılmış ortak bloklardaki nesneler başlangıçta tanımlanabilir; boş ortak nesneler olmayacaktır
  • boş ortak bloklardaki nesneler, ortak blok “kaydet” özelliğine sahipmiş gibi davranır; ‘save’ özniteliği olmayan adlandırılmış ortak bloklardaki nesneler, blok aktif bir program birimi kapsamında olmadığında tanımsız hale gelebilir.

Bu son nokta, modern koddaki modül değişkenlerinin davranışıyla karşılaştırılabilir. Fortran 2008’deki tüm modül değişkenleri örtük olarak kaydedilir ve modül kapsam dışına çıktığında tanımsız hale gelmez. Fortran 2008’den önce modül değişkenleri, adlandırılmış ortak bloklardaki değişkenler gibi, modül kapsam dışına çıktığında tanımsız hale gelirdi.

Bloksuz DO yapıları #

Blok olmayan “do” yapısı şuna benzer:

    integer i
    do 100, i=1, 5
100 print *, i

Yani, etiketli sonlandırma ifadesinin bir “devam” ifadesi olmadığı durumlarda. Sonlandırma ifadesi olarak kullanılabilecek ifadede çeşitli kısıtlamalar vardır ve her şey genellikle çok kafa karıştırıcıdır.

Böyle bir blok olmayan yapı, blok biçiminde şu şekilde yeniden yazılabilir:

    integer i
    do 100 i=1,5
      print *, i
100 continue

veya daha iyisi, bir ’end do’ sonlandırma ifadesi kullanarak,

integer i
do i=1,5
  print *, i
end do

Alternatif dönüş #

Alternatif dönüş, bir alt programdan dönüşte yürütme akışını kontrol etme olanağıdır. Genellikle bir hata işleme biçimi olarak kullanılır:

real x

call sub(x, 1, *100, *200)
print*, "Success:", x
stop

100 print*, "Negative input value"
stop

200 print*, "Input value too large"
stop

end

subroutine sub(x, i, *, *)
  real, intent(out) :: x
  integer, intent(in) :: i
  if (i<0) return 1
  if (i>10) return 2
  x = i
end subroutine

Alternatif dönüş, altyordam kukla argüman listesinde * argümanları ile işaretlenir.

Yukarıdaki “çağrı” ifadesinde “*100” ve “*200” sırasıyla “100” ve “200” etiketli ifadelere atıfta bulunur.

Altyordamın kendisinde, alternatif dönüşe karşılık gelen ‘dönüş’ ifadelerinin bir numarası vardır. Bu sayı bir dönüş değeri değildir, ancak iadede yürütmenin iletildiği sağlanan etiketi belirtir. Bu durumda, “return 1”, yürütmeyi “100” etiketli ifadeye ve “return 2”, yürütmeyi “200” etiketli ifadeye geçirir. Süslenmemiş bir “dönüş” ifadesi veya “dönüş” ifadesi olmadan altyordam yürütmesinin tamamlanması, yürütmeyi çağrı ifadesinden hemen sonraya geçirir.

Alternatif dönüş sözdizimi, diğer argüman ilişkilendirme biçimlerinden çok farklıdır ve tesis, modern zevklerin aksine akış kontrolü sunar. Bir tamsayı “durum” kodunun döndürülmesiyle daha hoş akış kontrolü yönetilebilir.

real x
integer status

call sub(x, 1, status)
select case (status)
case (0)
  print*, "Success:", x
case (1)
  print*, "Negative input value"
case (2)
  print*, "Input value too large"
end select

end

subroutine sub(x, i, status)
  real, intent(out) :: x
  integer, intent(in) :: i
  integer, intent(out) :: status

  status = 0

  if (i<0) then
    status = 1
  else if (i>10)
    status = 2
  else
    x = i
  end if

end subroutine

Atanan GOTO #

Atanan GOTO, ASSIGN ifadesi kullanılarak bir ifade etiketinin atandığı tamsayı değişkenini kullanır.

100 CONTINUE

...

ASSIGN 100 TO ILABEL

...


GOTO ILABEL

Atanan GOTO, Fortran 90’da eskimiş ve Fortran 95 ve sonraki sürümlerde silinmiştir. Modern kodda prosedürler, dahili prosedürler, prosedür işaretçileri ve diğer özellikler kullanılarak bundan kaçınılabilir.

Hesaplanmış GOTO #

Hesaplanan GOTO, programın bir tamsayı ifadesinin değerine göre dallanmasına izin verir.

GOTO (label_1, label_2,... label_n) scalar-integer-expression

Eğer skaler-tamsayı-ifadesi 1’e eşitse program ’label_1’ deyimiyle devam eder, 2’ye eşitse ’label_2’ye gider ve bu şekilde devam eder. ‘1’den küçükse veya ’n’den büyükse program bir sonraki satırda devam eder.

Örnek:

ivar = 2

...

GOTO (10, 20, 30, 40) ivar

ifade etiketi 20’ye atlayacaktır.

Bu ‘goto’ biçimi, Fortran 95 ve sonraki sürümlerde eskimiştir ve yerini ‘seçilmiş durum’ yapısı almıştır.

Atanan biçim belirteçleri #

Fortran 95’ten önce, giriş veya çıkış için atanmış biçimleri kullanmak mümkündü. Düşünmek

integer i, fmt
read *, i

assign 100 to fmt
if (i<100000) assign 200 to fmt

print fmt, i

100 format ("This is a big number", I10)
200 format ("This is a small number", I6)

end

“atama” ifadesi, bir tamsayı değişkenine bir ifade etiketi atar. Bu tamsayı değişkeni daha sonra “print” ifadesinde biçim belirteci olarak kullanılır.

Bu tür biçim belirteci ataması Fortran 95’te silinmişti. Bunun yerine, daha modern kodlar, başka bir yürütme akışı denetimi biçimini kullanabilir.

integer i
read *, i

if (i<100000) then
  print 100, i
else
  print 200, i
end if

100 format ("This is a big number", I10)
200 format ("This is a small number", I6)

end

veya biçim belirteci olarak bir karakter değişkeni kullanılabilir

character(29), target :: big_fmt='("This is a big number", I10)'
character(30), target :: small_fmt='("This is a small number", I6)'
character(:), pointer :: fmt

integer i
read *, i

fmt=>big_fmt
if (i<100000) fmt=>small_fmt

print fmt, i

end

İfade işlevleri #

Programı düşünün

implicit none
integer f, i
f(i)=i

print *, f(1)
end

Burada ‘f’ bir ifade işlevidir. Bir tamsayı kukla argüman alarak tamsayı sonuç türüne sahiptir.1

Böyle bir deyim işlevi, tanımlandığı kapsamda bulunur. Özellikle, bu kapsamda erişilebilen değişkenlere ve adlandırılmış sabitlere erişimi vardır.

Bununla birlikte, deyim işlevleri birçok kısıtlamaya tabidir ve potansiyel olarak kafa karıştırıcıdır (sıradan bir bakışta bir dizi öğesi atama deyimi gibi bakmak). Önemli kısıtlamalar şunlardır:

  • fonksiyon sonucu ve yapay argümanlar skaler olmalıdır
  • sahte argümanlar, işlevle aynı kapsamdadır
  • deyim fonksiyonlarının yerel değişkenleri yoktur
  • deyim işlevleri gerçek argümanlar olarak iletilemez

İfade işlevlerinin temel faydaları, dahili işlevler tarafından tekrarlanır.

implicit none

print *, f(1)

contains

  integer function f(i)
    integer i
    f = i
  end function

end

Dahili fonksiyonlar yukarıda bahsedilen kısıtlamalara tabi değildir, ancak dahili bir alt programın daha fazla dahili alt program içermeyebileceğini (fakat bir ifade fonksiyonu içerebilir) belki de dikkate değerdir.

Dahili işlevlerin kendi kapsamı vardır, ancak aynı zamanda mevcut ana bilgisayar ilişkisine de sahiptir.

1 Gerçek eski kod örneklerinde, sonuç açık bir türe sahip olsa bile, bir ifade işlevinin yapay argümanlarının örtük olarak yazıldığını görmek olağandışı bir durum değildir.
