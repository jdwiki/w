Prosedürler - Fonksiyonlar ve Alt Programlar

Fonksiyonlar ve altyordamlar, modüller ile birlikte bir programı birimlere ayırmaya yarayan araçlardır. Bu, programı daha okunabilir ve yönetilebilir hale getirir. Bu birimlerin her biri, ideal olarak ayrı ayrı derlenip test edilebilecek kodun bir parçası olarak düşünülebilir. Ana program(lar), bir görevi gerçekleştirmek için bu tür alt programları (fonksiyonlar veya alt rutinler) çağırabilir (veya çağırabilir).

Fonksiyonlar ve alt programlar şu anlamda farklıdır:

  • İşlevler tek bir nesne döndürür ve - genellikle - argümanlarının değerlerini değiştirmez (yani, tıpkı bir matematiksel işlev gibi davranırlar!);
  • Alt yordamlar genellikle daha karmaşık bir görevi yerine getirir ve normal olarak argümanlarını (varsa) ve diğer değişkenleri (örneğin alt yordamı içeren modülde bildirilenler) değiştirirler.

Fonksiyonlar ve alt programlar topluca prosedürler adı altında toplanır. (Aşağıda teknik olarak ‘çağrılan’ prosedürler ‘alt program’ olsa da, ‘fonksiyon’ atamanın sağ tarafında veya ifadelerde görünse bile, “çağrı” fiilini “çağırmak” ile eşanlamlı olarak kullanacağız. .)

İşlev sözdizimi #

Fonksiyonlar birkaç tür sözdizimi kullanılarak yazılabilir.

function name()
  integer name
  name = 42
end function

integer function name()
  name = 42
end function

function name() result(res)
  integer res
  res = 42
end function

İşlevler, bir fonksiyon sonucu aracılığıyla değerler döndürür. İşlev ifadesinin bir “sonuç” yan tümcesi olmadıkça, işlevin sonucu işlevle aynı ada sahiptir. ‘sonuç’ ile işlev sonucu, ‘sonuç’ tarafından verilendir. Yukarıdaki ilk iki örneğin her birinde, fonksiyon sonucu ‘isim’ ile verilmiştir; üçüncü ‘res’ ile.

İşlev sonucu, işlevin yürütülmesi sırasında tanımlanmalıdır.

İşlevler, bazı özel öneklerin kullanılmasına izin verir.

Pure işlevi, bu işlevin hiçbir yan etkisinin olmadığı anlamına gelir:

pure real function square(x)
  real, intent(in) :: x
  square = x * x
end function

Elemental işlevi skaler operatör olarak tanımlanır, ancak gerçek argüman olarak dizi ile çağrılabilir, bu durumda işlev eleman bazında uygulanır. “Saf olmayan” öneki (Fortran 2008’de tanıtıldı) belirtilmedikçe, bir elemental işlevi aynı zamanda bir pure işlevidir.

elemental real function square(x)
  real, intent(in) :: x
  square = x * x
end function

İade beyanı #

return ifadesi, fonksiyondan ve alt programdan çıkmak için kullanılabilir. Diğer birçok programlama dilinden farklı olarak, dönüş değerini ayarlamak için kullanılmaz.

real function f(x)
  real, intent(in) :: x
  integer :: i

  f = x

  do i = 1, 10

    f = sqrt(f) - 1.0

    if (f < 0) then
      f = -1000.
      return
    end if

  end do
end function

Bu işlev yinelemeli bir hesaplama gerçekleştirir. f değeri negatif olursa, fonksiyon -1000 değerini döndürür.

Özyinelemeli Prosedürler #

Fortran’da işlevler ve alt programlar, kendilerini doğrudan veya dolaylı olarak yeniden çağıracaklarsa, açıkça recursive olarak bildirilmelidirler. Böylece, Fibonacci serisinin özyinelemeli bir uygulaması şöyle görünebilir:

recursive function fibonacci(term) result(fibo)
  integer, intent(in) :: term
  integer :: fibo

  if (term <= 1) then
    fibo = 1
  else
    fibo = fibonacci(term-1) + fibonacci(term-2)
  end if
  
end function fibonacci

Faktöriyel hesaplamak için başka bir örneğe izin verilir:

recursive function factorial(n)  result(f)
  integer :: f
  integer, intent(in) :: n
  
  if(n == 0) then
    f = 1
  else
    f = n * f(n-1)
  end if
end function factorial

Bir işlevin doğrudan özyinelemeli olarak kendisine başvurması için tanımının “sonuç” son ekini kullanması gerekir. Bir fonksiyonun hem “özyinelemeli” hem de “temel” olması mümkün değildir.

Kukla Argümanların Amacı #

Bir alt yordam veya işlevdeki yapay bir argümanın ‘intent’ özelliği, amaçlanan kullanımını bildirir. Sözdizimi şunlardan biridir:

intent(IN)
intent(OUT)
intent(INOUT)

Örneğin, bu işlevi düşünün:

real function f(x)
  real, intent(IN) :: x

  f = x*x
end function

“intent(IN)”, (işaretçi olmayan) sahte argüman “x"in hiçbir zaman tanımlanamayacağını veya işlev veya başlatma boyunca tanımsız hale gelemeyeceğini belirtir. Bir işaretçi kukla argümanı “intent(IN)” özelliğine sahipse, bu onun ilişkilendirmesi için geçerlidir.

İşaretçi olmayan bir yapay argüman için ‘intent(OUT)’, yapay argümanın alt program çağrıldığında tanımsız hale geldiği (varsayılan başlatmaya sahip türetilmiş bir türün herhangi bir bileşeni hariç) ve yürütme sırasında ayarlanacağı anlamına gelir. Sahte argüman olarak iletilen asıl argüman tanımlanabilir olmalıdır: adlandırılmış veya değişmez bir sabitin veya bir ifadenin iletilmesine izin verilmez.

Öncekine benzer şekilde, bir işaretçi kukla argümanı “intent(OUT)” ise, işaretçinin ilişkilendirme durumu tanımsız hale gelir. Buradaki asıl argüman bir işaretçi değişkeni olmalıdır.

intent(INOUT) gerçek argümanın tanımlanabilir olduğunu ve prosedürden hem veri göndermek hem de veri döndürmek için uygun olduğunu belirtir.

Son olarak, kukla bir argüman “niyet” özniteliği olmadan olabilir. Böyle sahte bir argümanın kullanımı, iletilen gerçek argümanla sınırlıdır.

Örneğin, düşünün

integer :: i = 0
call sub(i, .TRUE.)
call sub(1, .FALSE.)

end

subroutine sub(i, update)
  integer i
  logical, intent(in) :: update
  if (update) i = i+1
end subroutine

“i” argümanı, ana programın her iki altyordam çağrısına izin veren bir “niyet” niteliğine sahip olamaz.

Bir prosedüre atıfta bulunmak #

Bir fonksiyonun veya alt programın faydalı olması için referans alınması gerekir. Bir “call” ifadesinde bir alt programa başvurulur

call sub(...)

ve bir ifade içinde bir işlev. Diğer birçok dilden farklı olarak, bir ifade tam bir ifade oluşturmaz, bu nedenle bir işlev başvurusu genellikle bir atama ifadesinde görülür veya başka bir şekilde kullanılır:

x = func(...)
y = 1 + 2*func(...)

Başvurulan bir prosedürü belirlemenin üç yolu vardır:

  • bir prosedürün veya prosedür işaretçisinin adı olarak
  • türetilmiş bir tür nesnesinin prosedür bileşeni
  • bir türe bağlı prosedür bağlama adı

İlk olarak görülebilir

procedure(), pointer :: sub_ptr=>sub
call sub()   ! With no argument list the parentheses are optional
call sub_ptr()
end

subroutine sub()
end subroutine

ve son ikisi olarak

module mod
  type t
    procedure(sub), pointer, nopass :: sub_ptr=>sub
  contains
    procedure, nopass :: sub
  end type

contains

  subroutine sub()
  end subroutine

end module

use mod
type(t) x
call x%sub_ptr()   ! Procedure component
call x%sub()       ! Binding name

end

Sahte argümanları olan bir prosedür için, isteğe bağlı sahte argümanlar verilememesine rağmen, referans karşılık gelen actual argümanları gerektirir.

Alt programı düşünün

subroutine sub(a, b, c)
  integer a, b
  integer, optional :: c
end subroutine

Bu, aşağıdaki iki şekilde referans alınabilir

call sub(1, 2, 3)   ! Passing to the optional dummy c
call sub(1, 2)      ! Not passing to the optional dummy c

Bu, konumsal referanslama olarak adlandırılır: gerçek argümanlar, argüman listelerindeki pozisyona göre ilişkilendirilir. Burada, “a” kuklası “1” ile, “b” ile “2” ve “c” (belirtildiğinde) “3” ile ilişkilendirilir.

Alternatif olarak, prosedür açık bir arayüze sahip olduğunda keyword referansı kullanılabilir.

call sub(a=1, b=2, c=3)
call sub(a=1, b=2)

ki yukarıdaki ile aynıdır.

Ancak, anahtar kelimelerle gerçek argümanlar herhangi bir sırayla sunulabilir.

call sub(b=2, c=3, a=1)
call sub(b=2, a=1)

Konumsal ve anahtar kelime referansı kullanılabilir

call sub(1, c=3, b=2)

bir anahtar kelimenin ilk görünümünü izleyen her argüman için bir anahtar kelime verildiği sürece

call sub(b=2, 1, 3)  ! Not valid: all keywords must be specified

Anahtar kelime referanslamanın değeri, özellikle birden fazla isteğe bağlı sahte argüman olduğunda telaffuz edilir, eğer yukarıdaki alt program tanımında “b” de isteğe bağlıysa aşağıda görüldüğü gibi

call sub(1, c=3)  ! Optional b is not passed

Türe bağlı yordamlar için bağımsız değişken listeleri veya geçirilen bir bağımsız değişkene sahip bileşen yordam işaretçileri ayrı olarak değerlendirilir.
