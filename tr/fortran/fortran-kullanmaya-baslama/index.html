Selam Dünya #

Herhangi bir Fortran programı, son ifade olarak ’end’i içermelidir. Bu nedenle, en basit Fortran programı şöyle görünür:

end

İşte “merhaba, dünya” programlarından bazı örnekler:

print *, "Hello, world" 
end

‘write’ ifadesi ile:

write(*,*) "Hello, world"
end

Anlaşılır olması açısından, artık bir programı başlatmak ve ona bir isim vermek için ‘program’ ifadesini kullanmak yaygındır. ’end’ ifadesi daha sonra neyi kastettiğini açıklığa kavuşturmak için bu isme başvurabilir ve derleyicinin kodun doğruluğunu kontrol etmesine izin verebilir. Ayrıca, tüm Fortran programları bir “örtük yok” ifadesi içermelidir. Bu nedenle, minimal bir Fortran programı aslında aşağıdaki gibi görünmelidir:

program hello
  implicit none
  write(*,*) 'Hello world!'
end program hello

Bu noktadan sonraki mantıklı adım, merhaba dünya programının sonucunun nasıl görüleceğidir. Bu bölüm, linux benzeri bir ortamda bunu nasıl başaracağınızı gösterir. Bazı temel kabuk komutları kavramlarına sahip olduğunuzu varsayıyoruz, esas olarak kabuk terminaline nasıl ulaşacağınızı biliyorsunuz. Ayrıca zaten [fortran ortamınızı 2 kurmuş olduğunuzu varsayıyoruz. Tercih ettiğiniz metin düzenleyiciyi (notepad, notepad++, vi, vim, emacs, gedit, kate, vb.) kullanarak, yukarıdaki merhaba programını ana dizininizde hello.f90 adlı bir dosyaya kaydedin (kopyalayıp yapıştırın). hello.f90 kaynak dosyanızdır. Ardından komut satırına gidin ve kaynak dosyanızı kaydettiğiniz dizine (ev dizini?) gidin, ardından aşağıdaki komutu yazın:

>gfortran -o hello hello.f90

Merhaba dünya yürütülebilir programınızı az önce yarattınız. Teknik açıdan, programınızı derlediniz. Çalıştırmak için aşağıdaki komutu yazın:

>./hello

Shell terminalinizde aşağıdaki satırın yazdırıldığını görmelisiniz.

> Hello world!

Tebrikler, az önce “Merhaba Dünya” programını yazdınız, derlediniz ve çalıştırdınız.

İkinci dereceden denklem #

Bugün Fortran esas olarak sayısal hesaplama için kullanılmaktadır. Bu çok basit örnek, ikinci dereceden denklemleri çözmek için temel program yapısını göstermektedir:

program quadratic
  !a comment

  !should be present in every separate program unit
  implicit none

  real :: a, b, c
  real :: discriminant
  real :: x1, x2

  print *, "Enter the quadratic equation coefficients a, b and c:"
  read *, a, b, c

  discriminant = b**2 - 4*a*c

  if ( discriminant>0 ) then

    x1 = ( -b + sqrt(discriminant)) / (2 * a)
    x2 = ( -b - sqrt(discriminant)) / (2 * a)
    print *, "Real roots:"
    print *, x1, x2

    ! Comparison of floating point numbers for equality is often not recommended. 
    ! Here, it serves the purpose of illustrating the "else if" construct. 
  else if ( discriminant==0 ) then

    x1 = - b / (2 * a)
    print *, "Real root:"
    print *, x1
  else

    print *, "No real roots."
  end if
end program quadratic

Büyük/küçük harf duyarlılığı #

Alfabedeki büyük ve küçük harfler eşdeğerdir. Fortran karakter seti. Başka bir deyişle, Fortran büyük/küçük harfe duyarlı değildir. Bu davranış, C++ ve diğerleri gibi büyük/küçük harfe duyarlı dillerin aksine.

Sonuç olarak, “a” ve “A” değişkenleri aynı değişkendir. Prensipte bir program şu şekilde yazılabilir:

pROgrAm MYproGRaM
..
enD mYPrOgrAM

Böyle çirkin seçimlerden kaçınmak iyi bir programcıya düşer.

Kurulum veya Kurulum #

Fortran, birçok satıcı tarafından sağlanan derleyiciler kullanılarak derlenebilen bir dildir. Farklı donanım platformları ve işletim sistemleri için farklı derleyiciler mevcuttur. Bazı derleyiciler ücretsiz yazılımdır, bazıları ücretsiz olarak kullanılabilir ve bazıları lisans satın alınmasını gerektirir.

En yaygın ücretsiz Fortran derleyicisi GNU Fortran veya gfortran’dır. Kaynak kodu, GNU derleyici koleksiyonu olan GCC’nin bir parçası olarak GNU’dan edinilebilir. Birçok işletim sistemi için ikili dosyalar https://gcc.gnu.org/wiki/GFortranBinaries adresinde mevcuttur. Linux dağıtımları genellikle paket yöneticilerinde gfortran içerir.

Örneğin başka derleyiciler mevcuttur:

  • PathScale tarafından EKOPath

  • LLVM (DragonEgg aracılığıyla arka uç)

  • Oracle Geliştirici Stüdyosu

  • Absoft Fortran Derleyici

  • Intel Fortran Derleyici

  • NAG Fortran Derleyici

  • PGI Derleyicileri

HPC-Sistemlerinde, örneğin IBM veya Cray derleyiciler gibi sistem sağlayıcısı tarafından sağlanan özel derleyiciler vardır.

Tüm bu derleyiciler Fortran 95 standardını destekler. Fortran 2003 durumu ve Fortran 2008 durumu hakkında çeşitli derleyiciler tarafından bir genel bakış, ACM Fortran Forum tarafından sunulmaktadır ve Fortran Wiki’de mevcuttur.
