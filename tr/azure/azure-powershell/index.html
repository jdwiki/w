Azure’da oturum açın #

Klasik (Hizmet Yönetimi) modu:

Add-AzureAccount

Bu, Azure Active Directory kullanarak kimliğinizi doğrular ve PowerShell, yaklaşık 12 saat sonra süresi dolan bir erişim belirteci alır. Bu nedenle, 12 saat sonra kimlik doğrulamasını tekrarlamanız gerekir.

Bir alternatif, aşağıdaki cmdlet’i çalıştırmaktır:

Get-AzurePublishSettingsFile

Bu, yayınlama ayarları dosyasını indirebileceğiniz bir tarayıcı penceresi açar. Bu dosya, PowerShell’in kimlik doğrulaması yapmasına izin veren bir sertifika içerir. Ardından, aşağıdakileri kullanarak yayınlama ayarları dosyanızı içe aktarabilirsiniz:

Import-AzurePublishSettingsFile

Yayınlama ayarları dosyasının, aboneliğinizde etkin yönetici ayrıcalıklarına sahip bir sertifika içerdiğini unutmayın. Güvenli tutun veya kullandıktan sonra silin.

Kaynak Yöneticisi

Kaynak Yöneticisi tarafında, yalnızca 12 saatlik erişim belirteçleriyle Azure Active Directory kimlik doğrulamasını kullanabiliriz. Şu anda kullanabileceğiniz iki alternatif komut vardır:

Login-AzureRmAccount
Add-AzureRmAccount

Abonelik seçme #

Azure hesabınız altında birden çok aboneliğiniz olduğunda; üzerinde çalışmak istediğinizi seçmeniz (ve bunu varsayılan olarak kullanmanız) önemlidir; Yanlış abonelikte kaynakların başına gelen kazaları önlemek için.

Klasik mod

Set-AzureSubscription
Select-AzureSubscription

Kaynak Yöneticisi

Select-AzureRmSubscription

Abonelik Bilgileri

Yukarıdaki komutlar, geçmek istediğiniz aboneliği tanımlamak için bilgileri (ör. Abonelik Kimliği) belirtmenizi ister. Erişiminiz olan abonelikler için bu bilgileri listelemek için şu komutu çalıştırın:

Get-AzureSubscription

Klasik mod vs ARM modu #

Azure ile PowerShell kullanarak çalışırken bilmeniz gereken 2 farklı yol vardır ve her ikisine de atıfta bulunan birçok Microsoft belgesi göreceksiniz:

“Klasik mod (Hizmet Yönetimi)”

Bu, Azure’u çalıştırmanın ve Azure’u yönetmenin eski yoludur. Gittikçe daha fazla hizmet yeni ARM moduna doğru ilerliyor olsa da, Azure’da yalnızca klasik mod kullanılarak yönetilebilen bazı hizmetler var.

Klasik modda çalışan makinelerinizde kurulu modülleri listelemek için aşağıdakileri yapabilirsiniz:

Get-Module -ListAvailable Azure.*

“Kaynak yöneticisi (ARM)”

Bu, Azure’u yönetmenin yeni yoludur (sağlanan REST API’lerine göre). Azure’da Powershell klasik modundan yönetebileceğiniz hizmetlerin çoğu, bazı istisnalar dışında artık yeni mod kullanılarak yönetilebilir. Henüz ARM modunda desteklenmeyen belirli hizmetleriniz yoksa, Azure kaynaklarınızı yönetmenin tercih edilen yolu bu olmalıdır.

ARM modunda çalışacak komutları içeren makinenizde kurulu modülleri listelemek için aşağıdakileri yapabilirsiniz:

Get-Module -ListAvailable AzureRM*

Mevcut Azure PowerShell Sürümünü Alın #

Yüklemiş olduğunuz Azure PowerShell sürümünü belirlemek için aşağıdakileri çalıştırın:

Get-Module -ListAvailable -Name Azure -Refresh

Bu komut, mevcut PowerShell oturumunuzda Azure PowerShell modülünü yüklememiş olsanız bile yüklü sürümü döndürür.

Azure Varlıklarını Değiştirin #

Azure Cmdlet’leri, C# kodunu veya Azure portalını kullandığınız bazı eylemleri PowerShell aracılığıyla Azure varlıklarında gerçekleştirmenize olanak tanır.

Örneğin, bu adımlar bir Azure blobunun içeriğini yerel bir dizine indirmenize olanak tanır:

New-Item -Path .\myblob -ItemType Directory
$context = New-AzureStorageContext -StorageAccountName MyAccountName -StorageAccountKey {key from the Azure portal}
$blob = Get-AzureStorageBlob -Container MyContainerName -Context $context
$blob | Get-AzureStorageBlobContent -Destination .\myblob\

Trafik Yöneticilerini Yönetme #

Azure PowerShell ile, şu anda Azure Portal‘da kullanılamayan belirli işlevleri elde edebilirsiniz, örneğin:

  • Tüm Traffic Manager’ın uç noktalarını bir kerede yeniden yapılandırın
  • Diğer hizmetleri etki alanı adı yerine Azure ‘ResourceId’ aracılığıyla adresleyin, böylece Azure Endpoints için Konum’u manuel olarak ayarlamanız gerekmez

Önkoşullar #

Başlamak için [oturum açmanız](https://www.wikiod.com/tr/azure/azure-powershell#Azure’da oturum açın) ve RM aboneliğini seçmeniz gerekir belgeler/azure/3961/azure-powershell/13825/seçme-abonelik).

TrafficManager profilini alın #

PowerShell aracılığıyla Trafik Yöneticileri ile işlemler üç adımda gerçekleştirilir:

  1. TM profilini alın:
    $profile = Get-AzureRmTrafficManagerProfile -ResourceGroupName-resource-group -name my-traffic-manager
    Veya [bu makaledeki](https://docs. microsoft.com/en-us/powershell/resourcemanager/azurerm.trafficmanager/v2.2.0/new-azurermtrafficmanagerprofile).
  2. TM profilini keşfedin ve değiştirin
    Her bir uç noktanın yapılandırmasını görmek için ‘$profile’ alanlarını ve ‘$profile.Endpoints’i kontrol edin.
  3. Set-AzureRmTrafficManagerProfile -TrafficManagerProfile $profile aracılığıyla değişiklikleri kaydedin.

Uç noktaları değiştir #

Geçerli tüm uç noktalar $profile.Endpoints listesinde depolanır, böylece onları doğrudan indeks
$profile.Endpoints[0].Weight = 100
veya ada göre
$profile değiştirebilirsiniz. .Uç Noktalar | ?{ $_.Name -eq 'bitiş noktam' } | %{ $_.Weight = 100 }

Tüm uç noktaları temizlemek için
öğesini kullanın. $profile.Endpoints.Clear()

Belirli bir uç noktayı silmek için
`Remove-AzureRmTrafficManagerEndpointConfig -TrafficManagerProfile $profile -EndpointName ‘benim uç noktam’’

Yeni uç nokta eklemek için
Add-AzureRmTrafficManagerEndpointConfig -TrafficManagerProfile $profile -EndpointName "benim uç noktam" -Type AzureEndpoints -TargetResourceId "/subscriptions/00000000-0000-0000-0000-000000000000/resourceGroups/my-resource-put/soft.ClassdoicinName/ my-azure-service" -EndpointStatus Etkin -Weight 100

Gördüğünüz gibi, son durumda azure hizmetimizi alan adı yerine ResourceId aracılığıyla ele aldık.

Aklında tut #

TM ve uç noktalarında yaptığınız değişiklikler, Set-AzureRmTrafficManagerProfile -TrafficManagerProfile $profile çağırana kadar uygulanmaz. Bu, TM’yi tek bir işlemde tamamen yeniden yapılandırmanıza olanak tanır.

Traffic Manager, DNS uygulamasının bir uygulamasıdır ve istemcilere verilen IP adresinin yaşaması için biraz zaman vardır (diğer adıyla TTL, süresini $ içinde saniye cinsinden görebilirsiniz). profile.Ttl alanı). Bu nedenle, TM’yi yeniden yapılandırdıktan sonra bazı istemciler, bu TTL’nin süresi dolana kadar önbelleğe aldıkları eski uç noktaları kullanmaya devam edecektir.
