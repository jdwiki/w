Her iş parçacığının kendi son hata kodu olacaktır. Windows API, çağıran iş parçacığındaki son hata kodunu ayarlayacaktır.

Bir Windows API işlevinin dönüş değerini kontrol ettikten hemen sonra GetLastError işlevini her zaman çağırmalısınız.

Windows API işlevlerinin çoğu, başarısız olduklarında son hata kodunu ayarlar. Bazıları başarılı olduklarında son hata kodunu da ayarlayacaktır. Son hata kodunu ayarlamayan bir dizi işlev vardır. Her zaman Windows API işlevinin belgelerine bakın.

Bir hata kodunun açıklamasını almak için “FormatMessage” işlevini kullanırken “FORMAT_MESSAGE_IGNORE_INSERTS” olmadan “FORMAT_MESSAGE_FROM_SYSTEM” kullanmak güvenli değildir.

Bir hata kodunu mesaj dizisine dönüştürme #

[GetLastError](https://www.wikiod.com/tr/winapi/hata-raporlama-ve-isleme#Arızayla ilgili ek bilgilerle rapor edilen hata) sayısal bir hata kodu döndürür. Açıklayıcı bir hata mesajı almak için (ör., bir kullanıcıya göstermek için), “FormatMessage"ı arayabilirsiniz:

// This functions fills a caller-defined character buffer (pBuffer)
// of max length (cchBufferLength) with the human-readable error message
// for a Win32 error code (dwErrorCode).
// 
// Returns TRUE if successful, or FALSE otherwise.
// If successful, pBuffer is guaranteed to be NUL-terminated.
// On failure, the contents of pBuffer are undefined.
BOOL GetErrorMessage(DWORD dwErrorCode, LPTSTR pBuffer, DWORD cchBufferLength)
{
    if (cchBufferLength == 0)
    {
        return FALSE;
    }

    DWORD cchMsg = FormatMessage(FORMAT_MESSAGE_FROM_SYSTEM | FORMAT_MESSAGE_IGNORE_INSERTS,
                                 NULL,  /* (not used with FORMAT_MESSAGE_FROM_SYSTEM) */
                                 dwErrorCode,
                                 MAKELANGID(LANG_NEUTRAL, SUBLANG_DEFAULT),
                                 pBuffer,
                                 cchBufferLength,
                                 NULL);
    return (cchMsg > 0);
}

C++‘da, ‘std::string’ sınıfını kullanarak arayüzü önemli ölçüde basitleştirebilirsiniz:

#include <Windows.h>
#include <exception>
#include <stdexcept>
#include <memory>
#include <string>
typedef std::basic_string<TCHAR> String;

String GetErrorMessage(DWORD dwErrorCode)
{
    LPTSTR psz = NULL;
    const DWORD cchMsg = FormatMessage(FORMAT_MESSAGE_FROM_SYSTEM
                                         | FORMAT_MESSAGE_IGNORE_INSERTS
                                         | FORMAT_MESSAGE_ALLOCATE_BUFFER,
                                       NULL, // (not used with FORMAT_MESSAGE_FROM_SYSTEM)
                                       dwErrorCode,
                                       MAKELANGID(LANG_NEUTRAL, SUBLANG_DEFAULT),
                                       reinterpret_cast<LPTSTR>(&psz),
                                       0,
                                       NULL);
    if (cchMsg > 0)
    {
        // Assign buffer to smart pointer with custom deleter so that memory gets released
        // in case String's c'tor throws an exception.
        auto deleter = [](void* p) { ::HeapFree(::GetProcessHeap(), 0, p); };
        std::unique_ptr<TCHAR, decltype(deleter)> ptrBuffer(psz, deleter);
        return String(ptrBuffer.get(), cchMsg);
    }
    else
    {
        throw std::runtime_error("Failed to retrieve error message string.");
    }
}

NOT: Bu işlevler ayrıca [HRESULT değerleri](https://www.wikiod.com/tr/winapi/hata-raporlama-ve-isleme#HRESULT değeri olarak bildirilen hata için de çalışır) değer). Sadece “DWORD dwErrorCode” olan ilk parametreyi “HRESULT hResult” olarak değiştirin. Kodun geri kalanı değişmeden kalabilir.

Giriiş #

Windows API, C tarafından çağrılabilir bir arabirim aracılığıyla sağlanır. Bir API çağrısının başarısı veya başarısızlığı, kesinlikle dönüş değerleri aracılığıyla bildirilir. İstisnalar belgelenmiş sözleşmenin bir parçası değildir (bazı API uygulamaları SEH istisnalarını artırabilir, örneğin, salt okunur bir lpCommandLine bağımsız değişkenini CreateProcess) iletirken.

Hata raporlama kabaca dört kategoriden birine girer:

  • [Yalnızca dönüş değeri](https://www.wikiod.com/tr/winapi/hata-raporlama-ve-isleme#Yalnızca dönüş değeri tarafından bildirilen hata)
  • [Arızayla ilgili ek bilgilerle birlikte dönüş değeri](https://www.wikiod.com/tr/winapi/hata-raporlama-ve-isleme#Arızayla ilgili ek bilgilerle rapor edilen hata)
  • [Başarısızlık ve başarı hakkında ek bilgiler içeren dönüş değeri](https://www.wikiod.com/tr/winapi/hata-raporlama-ve-isleme#Başarısızlık ve başarı hakkında ek bilgilerle bildirilen hata -ve-başarı)
  • [HRESULT dönüş değeri](https://www.wikiod.com/tr/winapi/hata-raporlama-ve-isleme#HRESULT değeri olarak bildirilen hatavalue)

Her API çağrısına ilişkin belgeler, hataların nasıl bildirildiğini açıkça belirtir. Her zaman belgelere bakın.

Yalnızca dönüş değeri tarafından bildirilen hata #

Bazı API çağrıları, herhangi bir ek bilgi olmadan tek bir başarısızlık/başarı işareti döndürür (ör. GetObject):

if ( GetObjectW( obj, 0, NULL ) == 0 ) {
    // Failure: no additional information available.
}

Başarısızlıkla ilgili ek bilgilerle birlikte hata bildirildi #

Başarısızlık/başarı dönüş değerine ek olarak, bazı API çağrıları başarısızlık durumunda son hatayı da ayarlar (ör. CreateWindow ). Belgeler genellikle bu durum için aşağıdaki standart ifadeyi içerir:

İşlev başarılı olursa, dönüş değeri <API’ye özgü başarı değeri> olur. İşlev başarısız olursa, dönüş değeri <API’ye özgü hata değeri> olur. Genişletilmiş hata bilgilerini almak için GetLastError numaralı telefonu arayın.

if ( CreateWindowW( ... ) == NULL ) {
    // Failure: get additional information.
    DWORD dwError = GetLastError();
} else {
    // Success: must not call GetLastError.
}

Hemen GetLastError()‘u çağırmanız çok önemlidir. Son hata kodunun üzerine başka herhangi bir işlev yazılabilir, bu nedenle, başarısız olan işlev ile GetLastError() çağrısı arasında fazladan bir işlev çağrısı varsa , GetLastError()dan gelen dönüş artık güvenilir olmayacak. C++ yapıcılarıyla uğraşırken ekstra dikkatli olun.

Bir hata kodu aldığınızda, onu yorumlamanız gerekecektir. MSDN’de, [Sistem Hata Kodları (Windows) sayfasından](https://msdn.microsoft.com/en-us/library/windows/desktop/ms681381(v=vs.ms) kapsamlı bir hata kodları listesine ulaşabilirsiniz. 85).aspx). Alternatif olarak, sistem başlık dosyalarınıza bakabilirsiniz; tüm hata kodu sabitlerini içeren dosya “winerror.h"dir. (Windows 8 veya daha yenisi için Microsoft’un resmi SDK’sına sahipseniz, bu, içerme klasörünün “paylaşılan” alt klasöründedir.)

Diğer programlama dillerinde GetLastError() çağrısına ilişkin notlar #

.net dilleri (C#, VB, vb.) #

.net ile, doğrudan GetLastError()a P/Invoke yapmamalısınız**. Bunun nedeni, .net çalışma zamanının arkanızdan aynı iş parçacığı üzerinde başka Windows API çağrıları yapmasıdır. Örneğin, çöp toplayıcı, artık kullanmadığı yeterli bellek bulursa ‘VirtualFree()‘yi çağırabilir ve bu, amaçlanan işlev çağrınız ile ‘GetLastError()’* çağrınız arasında olabilir.

Bunun yerine, .net, kendi yaptığınız son P/Invoke çağrısındaki son hatayı alacak olan Marshal.GetLastWin32Error() işlevini sağlar. Doğrudan GetLastError() çağırmak yerine bunu kullanın.

(.net zaten GetLastError() dosyasını içe aktarmanızı engellemiyor gibi görünüyor; neden olduğundan emin değilim.)

Gitmek #

Go tarafından DLL işlevlerini çağırmak için sağlanan çeşitli olanaklar (hem “syscall” paketinde hem de “golang.org/x/sys/windows” paketinde bulunur) üç değer döndürür: “r1”, “r2” ve “err”. “r2” asla kullanılmaz; orada boş tanımlayıcıyı kullanabilirsiniz. ‘r1’, işlevin dönüş değeridir. ’err’, ‘GetLastError()’ çağrısının sonucudur, ancak ‘hata’ uygulayan bir türe dönüştürülür, böylece onu işlemek için çağrı yapan işlevlere iletebilirsiniz.

Go, “GetLastError()“u ne zaman çağıracağını ve ne zaman aramayacağını bilmediğinden, her zaman bir “nil” olmayan hata döndürür. Bu nedenle, tipik Go hata işleme deyimi

r1, _, err := syscall.Syscall12(CreateWindowW.Addr(), ...)
if err != nil {
    // handle err
}
// use r1

çalışmayacak. Bunun yerine, tam olarak C’de yaptığınız gibi “r1"i kontrol etmelisiniz ve yalnızca bu işlevin bir hata döndürdüğünü gösteriyorsa “err"i kullanmalısınız:

r1, _, err := syscall.Syscall12(CreateWindowW.Addr(), ...)
if r1 == 0 {
    // handle err
}
// use r1

Başarısızlık ve başarı hakkında ek bilgilerle bildirilen hata #

Bazı API çağrıları birden fazla şekilde başarılı veya başarısız olabilir. API’ler genellikle hem başarılı çağrılar hem de hatalar için ek bilgiler döndürür (ör. CreateMutex).

if ( CreateMutexW( NULL, TRUE, L"Global\\MyNamedMutex" ) == NULL ) {
    // Failure: get additional information.
    DWORD dwError = GetLastError();
} else {
    // Success: Determine which mutex was returned.
    if ( GetLastError() == ERROR_ALREADY_EXISTS ) {
        // Existing mutex object returned.
    } else {
        // Newly created mutex object returned.
    }
}

HRESULT değeri olarak bildirilen hata #

HRESULTs, bitlerin veya bit aralıklarının iyi tanımlanmış bilgileri kodladığı 32 bitlik sayısal değerlerdir. MSB, kalan bitlerin ek bilgi depoladığı bir başarısızlık/başarı bayrağıdır. Başarısızlık veya başarı, BAŞARISIZ veya BAŞARILI kullanılarak belirlenebilir. /en-us/library/windows/desktop/ms687197.aspx) makroları. “HRESULT"ler COM ile yaygın olarak kullanılır, ancak COM dışı uygulamalarda da görünür (ör. StringCchPrintf) .

const size_t cchBuf = 5;
wchar_t buffer[cchBuf] = { 0 };
HRESULT hr = StringCchPrintfW( buffer, cchBuf, L"%s", L"Hello, world!" );
if ( FAILED( hr ) ) {
    // Failure: Determine specific reason.
    switch ( hr ) {
    case STRSAFE_E_INSUFFICIENT_BUFFER:
        // Buffer too small; increase buffer and retry.
        ...
    case STRSAFE_E_INVALID_PARAMETER:
        // Invalid parameter; implement custom error handling (e.g. logging).
        ...
    default:
        // Some other error code; implement custom error handling (e.g. logging).
        ...
    }
}
