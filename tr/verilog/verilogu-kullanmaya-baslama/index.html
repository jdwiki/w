Giriiş #

Verilog, elektronik sistemleri modellemek için kullanılan bir donanım tanımlama dilidir (HDL). En yaygın olarak, soyutlamanın kayıt-aktarma düzeyinde (RTL) bir elektronik sistemi tanımlar. Analog devrelerin ve karışık sinyal devrelerinin doğrulanmasında da kullanılır. Yapısı ve ana ilkeleri (aşağıda açıklandığı gibi), bir elektronik sistemi tanımlamak ve başarılı bir şekilde uygulamak için tasarlanmıştır.

  • Sertlik Elektronik devre, sabit bir yapıya sahip fiziksel bir varlıktır ve Verilog buna uyarlanmıştır. Modüller (modül), Bağlantı Noktaları (giriş/çıkış/giriş), bağlantılar (kablolar), bloklar (@her zaman), kayıtlar (reg) derleme zamanında sabitlenir. Varlıkların ve ara bağlantıların sayısı dinamik olarak değişmez. Her zaman en üst seviyede çip yapısını (sentez için) temsil eden bir “modül” ve doğrulama için sistem seviyesinde bir tane vardır.

  • paralellik Fiziksel çipteki doğal eşzamanlı işlemler, dilde her zaman (en yaygın), ilk ve çatal/birleştirme blokları tarafından taklit edilir.

    module top();
reg r1,r2,r3,r4; // 1-bit registers
  initial
  begin
    r1 <= 0 ;
  end
  initial
  begin
    fork
       r2 <= 0 ;
       r3 <= 0 ;
    join
  end
   always @(r4)
      r4 <= 0 ;

    endmodule

Yukarıdaki ifadelerin tümü aynı zaman birimi içinde paralel olarak yürütülür.

  • Zamanlama ve Senkronizasyon Verilog, devrelerin zamansal yapısını açıklamak için çeşitli yapıları destekler. Devrelerdeki zamanlamalar ve gecikmeler, örneğin #delay yapıları ile Verilog’da uygulanabilir. Benzer şekilde, Verilog ayrıca senkron ve asenkron devreler ve floplar, mandallar ve kombinatoryal mantık gibi bileşenler için çeşitli yapılar, örneğin “always” blokları kullanarak uyum sağlar. Bir dizi blok, ortak bir saat sinyali aracılığıyla da senkronize edilebilir veya belirli bir giriş grubuna dayalı olarak bir blok tetiklenebilir.

     #10   ;                 // delay for 10 time units
 always @(posedge clk )  // synchronous 
 always @(sig1 or sig2 ) // combinatorial logic  
 @(posedge event1)       // wait for post edge transition of event1
 wait (signal  == 1)     // wait for signal to be 1

  • Belirsizlik Verilog, elektronik devrelerde bulunan bazı belirsizliği destekler. “X” devrenin bilinmeyen durumunu temsil etmek için kullanılır. “Z”, devrenin sürülmemiş durumunu temsil etmek için kullanılır.

     reg1 = 1'bx;
 reg2 = 1'bz;

  • Soyutlama Verilog, farklı soyutlama seviyelerinde tasarımı destekler. Bir tasarım için en yüksek soyutlama seviyesi, Direnç Transfer Seviyesidir (RTL), bir sonraki kapı seviyesi ve en düşük hücre seviyesidir (Kullanıcı Tanımlı İlkeller), RTL soyutlaması en yaygın kullanılanıdır. Verilog ayrıca, öncelikle doğrulama için kullanılan, tasarımın yapısal gerçekleştirilmesine bakılmaksızın, davranışsal soyutlama seviyesini de destekler.

    // Example of a D flip flop at RTL abstraction module dff ( clk , // Clock Input reset , // Reset input d , // Data Input q // Q output ); //———–Input Ports————— input d, clk, reset ;

    //———–Output Ports————— output q;

    reg q;

    always @ ( posedge clk) if (~reset) begin q <= 1’b0; end else begin q <= d; end

    endmodule

    // And gate model based at Gate level abstraction module and(input x,input y,output o);

    wire w; // Two instantiations of the module NAND nand U1(w,x, y); nand U2(o, w, w);

    endmodule

    // Gate modeled at Cell-level Abstraction primitive udp_and( a, // declare three ports b, c ); output a; // Outputs input b,c; // Inputs

    // UDP function code here // A = B & C; table // B C : A 1 1 : 1; 0 1 : 0; 1 0 : 0; 0 0 : 0; endtable

    endprimitive

Verilog için üç ana kullanım durumu vardır. Kodun yapısını ve yorumunu belirlerler ve ayrıca kullanılan araç setlerini belirlerler. Her üç uygulama da herhangi bir Verilog tasarımının başarılı bir şekilde uygulanması için gereklidir.

  1. Fiziksel Tasarım / Arka Uç Burada Verilog, tasarımı öncelikle mantıksal tasarımı uygulayan birbirine bağlı kapıların bir matrisi olarak görmek için kullanılır. RTL/mantık/Tasarım, sentez -> yerleştirme -> saat ağacı yapımı -> yönlendirme -> DRC -> LVS ->‘den bant çıkışına kadar çeşitli adımlardan geçer. Kesin adımlar ve sıralar, uygulamanın kesin doğasına göre değişir.
  2. Simülasyon Bu kullanım durumunda birincil amaç, tasarımı spesifikasyona göre doğrulamak için test vektörleri oluşturmaktır. Bu kullanım durumunda yazılan kodun sentezlenebilir olması gerekmez ve doğrulama alanı içinde kalır. Buradaki kod, for/while/do döngüleri vb. gibi genel yazılım yapılarına daha çok benzemektedir.
  3. Tasarım Tasarım, genel olarak RTL soyutlama seviyesinde bir devrenin spesifikasyonunun uygulanmasını içerir. Daha sonra Doğrulama için Verilog kodu verilir ve fiziksel uygulama için tam olarak doğrulanmış kod verilir. Kod, yalnızca Verilog’un sentezlenebilir yapıları kullanılarak yazılır. Belirli RTL kodlama stili, simülasyon ile sentez uyumsuzluğuna neden olabilir ve bunlardan kaçınmak için özen gösterilmelidir.

İki ana uygulama akışı vardır. Ayrıca Verilog kodunun yazılma ve uygulanma şeklini de etkilerler. Belirli kodlama stilleri ve belirli yapılar bir akışta diğerine göre daha uygundur.

  • ASIC Flow (uygulamaya özel entegre devre)
  • FPGA Akışı (Alan programlanabilir kapı dizisi) - FPGA ve CPLD’leri içerir

Selam Dünya #

Bu örnek, icarus verilog derleyicisini kullanır.

Adım 1: merhaba.v adlı bir dosya oluşturun

module myModule();

initial
  begin
    $display("Hello World!");   // This will display a message
    $finish ; // This causes the simulation to end.  Without, it would go on..and on.
  end

endmodule

Adım 2. icarus kullanarak .v dosyasını derliyoruz:

>iverilog -o hello.vvp hello.v

-o anahtarı, çıktı nesne dosyasına bir ad atar. Bu anahtar olmadan çıktı dosyası a.out olarak adlandırılırdı. merhaba.v derlenecek kaynak dosyayı belirtir. Bu kaynak kodu derlerken, hata olmadıkça pratikte hiçbir çıktı olmamalıdır.

Adım 3. Bu Hello World verilog programını simüle etmeye hazırsınız. Bunu yapmak için şu şekilde çağırın:

>vvp hello.vvp 
Hello World!
>

Mac OSX Sierra için Icarus Verilog Derleyicisinin Kurulumu #

  1. App Store’dan Xcode’u yükleyin.

  2. Xcode geliştirici araçlarını yükleyin

    xcode-select –install

Bu, “gcc” ve “make” gibi temel komut satırı araçlarını sağlayacaktır.

  1. Mac Bağlantı Noktalarını Kurun https://www.macports.org/install.php

OSX Sierra kurulum paketi, Mac platformunda ek yazılım paketlerini kurmak ve yükseltmek için açık kaynaklı bir yöntem sağlayacaktır. Mac için “yum” veya “apt-get” düşünün.

  1. icarus’u Mac bağlantı noktalarını kullanarak yükleyin

    sudo port install iverilog

  2. Kurulumu komut satırından doğrulayın

    $ iverilog iverilog: no source files.

    Usage: iverilog [-ESvV] [-B base] [-c cmdfile|-f cmdfile] [-g1995|-g2001|-g2005] [-g] [-D macro[=defn]] [-I includedir] [-M depfile] [-m module] [-N file] [-o filename] [-p flag=value] [-s topmodule] [-t target] [-T min|typ|max] [-W class] [-y dir] [-Y suf] source_file(s)

    See the man page for details. $ Artık Mac’te ilk Verilog dosyanızı derlemeye ve simüle etmeye hazırsınız.

Mac OSx Sierra’da simülasyon verilerinin grafiksel gösterimi için GTKWave’i kurun #

GTKWave, birkaç grafik veri depolama standardını destekleyen tam özellikli bir grafik görüntüleme paketidir, ancak aynı zamanda ‘vvp’nin çıktılayacağı format olan VCD’yi de destekler. Yani, GTKWave’i almak için birkaç seçeneğiniz var

  1. http://gtkwave.sourceforge.net/gtkwave.zip adresine gidin ve indirin. Bu sürüm genellikle en son sürümdür.
  2. MacPorts’u (https://www.macports.org/) yüklediyseniz, sudo port install gtkwave komutunu çalıştırmanız yeterlidir. Bu muhtemelen bağımlılıklara yüklemek isteyecektir. Bu yöntemin genellikle size daha eski bir sürüm getireceğini unutmayın. Eğer MacPorts kurulu değilse, bu sayfada bunu yapmak için bir kurulum kurulum örneği bulunmaktadır. Evet! Tüm xcode geliştirici araçlarına ihtiyacınız olacak, çünkü bu yöntemler size kaynaktan bir GTKWave “oluşturacaktır”.

Kurulum tamamlandığında, bir python sürümü seçmeniz istenebilir. 2.7.10’u zaten yüklemiştim, bu yüzden asla yeni bir tane “seçmedim”.

Bu noktada gtkwave’i komut satırından gtkwave ile başlatabilirsiniz. Başladığında, XQuarts’ı yüklemeniz veya güncellemeniz istenebilir. Böyle yap. Benim durumumda XQuarts 2.7.11 kurulu.

Not: Aslında XQuarts’ı doğru bir şekilde almak için yeniden başlatmam gerekiyordu, sonra tekrar `gtkwave’ yazdım ve uygulama geldi.

Bir sonraki örnekte, test etmek için bir testbench ve modül olmak üzere iki bağımsız dosya oluşturacağım ve tasarımı görüntülemek için gtkwave’i kullanacağız.

Bir tasarımı grafik olarak simüle etmek ve görüntülemek için Icarus Verilog ve GTKWaves’i kullanma #

Bu örnekte Icarus ve GTKWave kullanılmaktadır. OSx’teki bu araçlar için kurulum talimatları bu sayfanın başka bir yerinde verilmiştir.

Modül tasarımı ile başlayalım. Bu modül bir BCD ila 7 segment göstergesidir. Tasarımı, bize kolayca kırılabilen ve grafiksel olarak düzeltmeye zaman harcayabileceğimiz bir şey vermek için geniş bir şekilde kodladım. Yani bir saatimiz, sıfırlamamız, bir BCD değerini temsil eden 4 veri girişimiz ve yedi segmentli ekranı temsil eden 7 bitlik bir çıkışımız var. bcd_to_7seg.v adlı bir dosya oluşturun ve aşağıdaki kaynağı içine yerleştirin.

module bcd_to_7seg (
   input clk,
   input reset,
   input [3:0] bcd,
   output [6:0] seven_seg_display

);
   parameter TP = 1;
   reg seg_a;
   reg seg_b;
   reg seg_c;
   reg seg_d;
   reg seg_e;
   reg seg_f;
   reg seg_g;

   
   always @ (posedge clk or posedge reset)
      begin
      if (reset)
         begin
            seg_a <= #TP 1'b0;
            seg_b <= #TP 1'b0;
            seg_c <= #TP 1'b0;
            seg_d <= #TP 1'b0;
            seg_e <= #TP 1'b0;
            seg_f <= #TP 1'b0;
            seg_g <= #TP 1'b0;
         end
      else
         begin
            seg_a <= #TP  ~(bcd == 4'h1 || bcd == 4'h4);
            seg_b <= #TP  bcd < 4'h5 || bcd > 6;
            seg_c <= #TP   bcd != 2;
            seg_d <= #TP   bcd == 0 || bcd[3:1] == 3'b001 || bcd == 5 || bcd == 6 || bcd == 8;
            seg_e <= #TP  bcd == 0 || bcd == 2 || bcd == 6 || bcd == 8;
            seg_f <= #TP  bcd == 0 || bcd == 4 || bcd == 5 || bcd == 6 || bcd > 7;
            seg_g <= #TP  (bcd > 1 && bcd < 7) || (bcd > 7);
         end
    end
 
    assign seven_seg_display = {seg_g,seg_f,seg_e,seg_d,seg_c,seg_b,seg_a};
endmodule

Ardından, bu modülün doğru çalışıp çalışmadığını kontrol etmek için bir teste ihtiyacımız var. Test tezgahındaki vaka ifadesinin bence okunması daha kolay ve ne yaptığı konusunda daha açık. Ama aynı durum ifadesini tasarıma VE teste koymak istemedim. Bu kötü bir uygulamadır. Aksine, birbirini doğrulamak için iki bağımsız tasarım kullanılmaktadır.

Aşağıdaki kod ile iki satır fark edeceksiniz. $dumpfile("testbench.vcd"); ve $dumpvars(0,testbench);. Bu satırlar, tasarımın grafiksel analizini gerçekleştirmek için kullanılacak VCD çıktı dosyasını oluşturan şeydir. Bunları dışarıda bırakırsanız, oluşturulan bir VCD dosyası alamazsınız. Testbench.v adlı bir dosya oluşturun ve aşağıdaki kaynağı içine yerleştirin.

`timescale 1ns/100ps
module testbench;
reg clk;
reg reset;
reg [31:0] ii;
reg [31:0] error_count;
reg [3:0] bcd;
wire [6:0] seven_seg_display; 
parameter TP = 1;
parameter CLK_HALF_PERIOD = 5;
 
   // assign clk = #CLK_HALF_PERIOD ~clk;  // Create a clock with a period of ten ns
   initial
   begin
     clk = 0;
     #5;
     forever clk = #( CLK_HALF_PERIOD )  ~clk;
   end

   initial
     begin
       $dumpfile("testbench.vcd");
       $dumpvars(0,testbench);
       // clk  = #CLK_HALF_PERIOD ~clk; 
       $display("%0t, Reseting system", $time);
       error_count = 0;
       bcd  = 4'h0;
       reset = #TP 1'b1;
       repeat (30) @ (posedge clk);
       reset  = #TP 1'b0;
       repeat (30) @ (posedge clk);
       $display("%0t, Begin BCD test", $time); // This displays a message


       for (ii = 0; ii < 10; ii = ii + 1)
          begin
          repeat (1) @ (posedge clk);
          bcd  = ii[3:0];
          repeat (1) @ (posedge clk); 
          if (seven_seg_display !== seven_seg_prediction(bcd)) 
             begin
                $display("%0t, ERROR: For BCD %d, module output 0b%07b does not match prediction logic value of 0b%07b.",$time,bcd, seven_seg_display,seven_seg_prediction(bcd));
                error_count = error_count + 1;
             end
          end
       $display("%0t, Test Complete with %d errors", $time, error_count);
       $display("%0t, Test %s", $time, ~|error_count ? "pass." : "fail.");
       $finish ; // This causes the simulation to end.
     end


parameter SEG_A = 7'b0000001;
parameter SEG_B = 7'b0000010;
parameter SEG_C = 7'b0000100;
parameter SEG_D = 7'b0001000;
parameter SEG_E = 7'b0010000;
parameter SEG_F = 7'b0100000;
parameter SEG_G = 7'b1000000;

function [6:0] seven_seg_prediction;
   input [3:0] bcd_in;

   //    +--- A ---+
   //    |         |
   //    F         B
   //    |         |
   //    +--- G ---+
   //    |         |
   //    E         C
   //    |         |
   //    +--- D ---+

   begin
      case (bcd_in)
         4'h0: seven_seg_prediction = SEG_A | SEG_B | SEG_C | SEG_D | SEG_E | SEG_F;
         4'h1: seven_seg_prediction = SEG_B | SEG_C;
         4'h2: seven_seg_prediction = SEG_A | SEG_B | SEG_G | SEG_E | SEG_D;
         4'h3: seven_seg_prediction = SEG_A | SEG_B | SEG_G | SEG_C | SEG_D;
         4'h4: seven_seg_prediction = SEG_F | SEG_G | SEG_B | SEG_C;
         4'h5: seven_seg_prediction = SEG_A | SEG_F | SEG_G | SEG_C | SEG_D;
         4'h6: seven_seg_prediction = SEG_A | SEG_F | SEG_G | SEG_E | SEG_C | SEG_D;
         4'h7: seven_seg_prediction = SEG_A | SEG_B | SEG_C;
         4'h8: seven_seg_prediction = SEG_A | SEG_B | SEG_C | SEG_D | SEG_E | SEG_F | SEG_G;
         4'h9: seven_seg_prediction = SEG_A | SEG_F | SEG_G | SEG_B | SEG_C;
         default: seven_seg_prediction = 7'h0;
      endcase
   end
endfunction


bcd_to_7seg u0_bcd_to_7seg (
.clk               (clk),
.reset             (reset),
.bcd               (bcd),
.seven_seg_display (seven_seg_display)
);


endmodule

Artık iki dosyamız var, bir testbench.v ve bcd_to_7seg.v, Icarus kullanarak derlememiz ve detaylandırmamız gerekiyor. Bunu yapmak için:

$ iverilog -o testbench.vvp testbench.v bcd_to_7seg.v

Sonra simüle etmemiz gerekiyor

$ vvp testbench.vvp 
LXT2 info: dumpfile testbench.vcd opened for output.
0, Reseting system
6000, Begin BCD test
8000, Test Complete with          0 errors
8000, Test pass.

Bu noktada, dosyanın gerçekten test edildiğini doğrulamak istiyorsanız, bcd_2_7seg.v dosyasına gidin ve mantığın bir kısmını hareket ettirin ve bu ilk iki adımı tekrarlayın.

Örnek olarak seg_c <= #TP bcd != 2; satırını seg_c <= #TP bcd != 4; olarak değiştiriyorum. Yeniden derleme ve benzetim şunları yapar:

$ iverilog -o testbench.vvp testbench.v bcd_to_7seg.v
$ vvp testbench.vvp 
LXT2 info: dumpfile testbench.vcd opened for output.
0, Reseting system
6000, Begin BCD test
6600, ERROR: For BCD  2, module output 0b1011111 does not match prediction logic value of 0b1011011.
7000, ERROR: For BCD  4, module output 0b1100010 does not match prediction logic value of 0b1100110.
8000, Test Complete with          2 errors
8000, Test fail.
$

Şimdi simülasyonu GTKWave kullanarak görelim. Komut satırından bir

gtkwave testbench.vcd &

GTKWave penceresi göründüğünde, sol üstteki kutuda modül adı testbench’i göreceksiniz. Tıkla. Bu, o dosyayla ilişkili alt modülleri, görevleri ve işlevleri ortaya çıkaracaktır. Teller ve kayıtlar da sol alt kutuda görünecektir.

Şimdi, clk, bcd, error_count ve seven_seg_display’i dalga formu penceresinin yanındaki sinyal kutusuna sürükleyin. Sinyaller şimdi çizilecektir. Error_count, hangi belirli BCD girişinin yanlış yedi_seg_display çıktısını oluşturduğunu size gösterecektir.

Artık bir Verilog hatasını grafiksel olarak gidermeye hazırsınız.
