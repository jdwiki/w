Maven ile Vaadin projesi oluşturun #

Maven ile vaadin-archetype-application arketipiyle Vaadin projesi oluşturabilirsiniz. IDE ile maven projesi oluşturmak için bu arketipi IDE’ye de ekleyebilirsiniz.

mvn archetype:generate 
   -DarchetypeGroupId=com.vaadin 
   -DarchetypeArtifactId=vaadin-archetype-application 
   -DarchetypeVersion=7.6.8 
   -DgroupId=myvaadin.project 
   -DartifactId=DemoVaadinProject 
   -Dversion=0.1 
   -Dpackaging=war

Yukarıdaki komutu yürüttüğünüzde, aşağıdaki proje yapısına sahip olacaksınız.

DemoVaadinProject 
  |-src
     |-main
         |-java
         |   |-myvaadin
         |         |-project
         |            |-MyUI.java
         |-resource
         |    |-myvaadin
         |         |-project
         |            |-MyAppWidgetset.gwt.xml
         |-webapps
              |- VAADIN
                   |-theme   
                      |- mytheme.scss
                      |- addons.scss
                      |- styles.scss
                      |- favicon.ico

Oluşturulan varsayılan maven projesi doğrudan IDE’ye aktarılabilir. Maven uygulamasını çalıştırmak için vaadin’in varsayılan widget setlerini derlememiz gerekir.

Vaadin uygulamasını paketlemek için aşağıdaki maven komutunu doğrudan kullanabileceğimizi ve varsayılan olarak widget setlerini derleyeceğini unutmayın. Vaadin uygulamasını Jetty’de dağıtmak için maven jetty eklentisini kullanabilirsiniz.

cd path/to/DemoVaadinProject
mvn package jetty:run

Bu, varsayılan uygulamayı dağıtacak ve varsayılan “8080” bağlantı noktasında çalıştırmaya başlayacaktır. Dağıtılan uygulamaya http://localhost:8080 adresinden erişebilirsiniz.

Herhangi bir değişiklik yapılmadan çalışmaya hazırdır. Varsayılan olarak Vaadin arketipi, varsayılan tema, widgetset xml ve vaadin uygulaması için bir giriş noktası olan MyUI sınıfını ekler.

Tarayıcıda aşağıdaki formu göreceğiz.

Eclipse’de Vaadin Projesi Oluştur #

Eclipse için Vaadin eklentisi, Apache Ivy bağımlılık yöneticisi ile vaadin projesi oluşturmanın hızlı bir yolunu sağlar. Vaadin’in belgeleri Eclipse eklentisi yardımıyla vaadin projesinin nasıl oluşturulacağını açıklar.

Eklentiyi kurmak için Eclipse marketplace’e gidin ve vaadin‘i arayın. Eklentinin şu anki sürümü ‘3.0.0’.

Eklentiyi kurduktan sonra aşağıdaki hızlı özelliklere sahip olacaksınız,

  • ‘Vaadin6’ veya ‘Vaadin 7’ projesi oluşturun (Varsayılan bağımlılık yöneticisi Ivy’dir)
  • Widgetset’leri Derleme (İstemci tarafı widget’larını derlemek için)
  • Temayı Derleme (Son CSS’yi oluşturmak için Temayı derlemek için)
  • Widget Oluştur (Özel widget’ınızı oluşturmak için)
  • Vaadin Teması Oluştur

Bu nedenle, eklentiyi kurduktan sonra, minimum konfigürasyonla yeni Vaadin projesi oluşturmanız yeterlidir. Projeyi oluştururken vaadin versiyonunu da belirtebilirsiniz.

  • `Dosya > Yeni > Vaadin7 Projesi’
  • Projede kullanılacak vaadin sürümünü belirtin
  • Kullanmak istediğiniz Hedef Çalışma Zamanını belirtin
  • Bitiş!

Ivy tüm bağımlılıkları çözdükten sonra vaadin için gerekli tüm kavanozları indirmek zaman alacaktır. Projeyi doğrudan sunucuda çalıştırabilirsiniz ve tarayıcı ekranında ‘Click Me’ ile bir ‘Buton’ göreceksiniz. Vaadin7’nin Java 6 ve daha yeni sürümlerle uyumlu olduğunu lütfen unutmayın.

Netbeans için Vaadin Eklentisi #

NetBeans IDE ile Proje Oluşturma

Aşağıda, NetBeans’te bir Vaadin projesinin oluşturulmasında size yol gösteriyor ve nasıl çalıştırılacağını gösteriyoruz.

NetBeans ve Vaadin eklentisinin kurulumu, NetBeans IDE ve Eklentisinin Kurulumu bölümünde ele alınmıştır.

Eklenti olmadan, Vaadin arketipini kullanarak Maven projesi olarak en kolay şekilde bir Vaadin projesi oluşturabilirsiniz. Ayrıca normal bir web uygulaması projesi olarak bir Vaadin projesi oluşturabilirsiniz, ancak tüm Vaadin kitaplıklarını yüklemek, UI sınıfını oluşturmak, sunucu uygulamasını yapılandırmak, tema oluşturmak vb. için birçok manuel adım gerekir.

Proje Oluşturma #

  1. Ana menüden Dosya ▸ Yeni Proje…‘yi seçin veya Ctrl+Shift+N.

  2. Açılan Yeni Proje penceresinde, Vaadin category and one of the Vaadin archetypes from the right.

    The archetypes are described in more detail in Overview of Maven Archetypes.

  3. Ad ve Yer adımında proje parametrelerini girin.

    Project Name

    A project name. The name must be a valid identifier that may only contains alphanumerics, minus, and underscore. It is appended to the group ID to obtain the Java package name for the sources.

    Project Location

    Path to the folder where the project is to be created.

    Group Id

    A Maven group ID for your project. It is normally your organization domain name in reverse order, such as com.example. The group ID is also used as a prefix for the Java source package, so it should be Java-compatible package name.

    Version

    Initial version of your application. The number must obey the Maven version numbering format.

    Package

    The Java package name to put sources in.

    Additional Creation Properties

    The properties control various names. They are specific to the archetype you chose.

    Click Finish.

    Creating the project can take a while as Maven loads all the needed dependencies.

Projeyi Keşfetmek #

Proje sihirbazı sizin için tüm işi yaptı: src dizinine bir UI sınıfı iskeleti yazıldı. Proje Gezgini’nde gösterilen proje hiyerarşisi, NetBeans’te yeni bir Vaadin projesi içinde gösterilir. Şekil 1. NetBeans’te yeni bir Vaadin projesi

benim Temam

Kullanıcı arayüzünün teması. Temalar hakkında bilgi için Temalar’a bakın.

MyUI.java

Uygulamanızın ana giriş noktası olan UI sınıfı. Vaadin uygulamalarının temel yapısı hakkında bilgi için Sunucu Tarafı Uygulamalarına bakın.

Vaadin kitaplıkları ve diğer bağımlılıklar Maven tarafından yönetilir. Java Kaynakları ▸ Kitaplıklar ▸ Maven Bağımlılıkları sanal klasöründe listelenmiş olsalar bile kitaplıkların proje klasörü altında depolanmadığına dikkat edin. Uygulamayı Çalıştırmak

Oluşturulduktan sonra aşağıdaki gibi bir sunucuda çalıştırabilirsiniz.

  1. Projeler sekmesinde, projeyi seçin ve Projeyi Çalıştır’a tıklayın. button in the tool bar (or press F6).

  2. Dağıtım sunucusu seç penceresinde, Sunucu listesinden bir sunucu seçin. GlassFish veya Apache Tomcat’i göstermelidir veya both, depending on what you chose in NetBeans installation.

    Also, select Remember Permanently if you want to use the same server also in future while developing applications.

    Click OK.

Widget seti bu noktada derlenecek ve bu biraz zaman alabilir.

Her şey yolunda giderse, NetBeans sunucuyu 8080 numaralı bağlantı noktasında başlatır ve sistem yapılandırmanıza bağlı olarak web uygulamasını görüntülemek için varsayılan tarayıcıyı başlatır. Değilse, örneğin http://localhost:8080/myproject adresinden manuel olarak açabilirsiniz. Proje adı, varsayılan olarak uygulamanın bağlam yolu olarak kullanılır.

Artık UI sınıfını kaynak düzenleyicide düzenlediğinizde ve kaydettiğinizde, NetBeans uygulamayı otomatik olarak yeniden konuşlandıracaktır. Birkaç saniye sonra tamamlandıktan sonra uygulamayı tarayıcıda yeniden yükleyebilirsiniz.

İlk program - “Merhaba Dünya” #

Bu kodu kopyalayıp yapıştırın ve programınızı başlatın:

@Theme(ValoTheme.THEME_NAME) //[optional] adds Vaadin built in theming 
public class SampleUI extends UI {
    
@Override
protected void init(VaadinRequest request) {
         final VerticalLayout rootLayout = new VerticalLayout();
         Label label = new Label("Hello World"!);
         rootLayout.addComponent(label);
         setContent(rootLayout);
     }
 }

Lauching başarılı olduktan sonra, lütfen localhost:8080/yourApplicationName konumuna gidin veya http://localhost:8080/ uygulamanızın çalışır durumda olduğunu görmek için.
