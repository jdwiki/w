C++‘da özel öğeler oluşturma #

QML, zengin görsel öğeler kümesiyle geldi. Yalnızca QML kullanarak bu öğelerle karmaşık uygulamalar oluşturabiliriz. Ayrıca Dikdörtgen, Düğme, Görüntü vb. gibi standart öğelere dayalı olarak kendi öğenizi oluşturmak çok kolaydır. Ayrıca, özel boyama ile öğe oluşturmak için Canvas gibi öğeleri kullanabiliriz. C++‘ın yeteneklerine dokunmadan yalnızca QML’de çeşitli uygulamalar oluşturabileceğimiz görülüyor. Ve bu aslında doğru ama yine de bazen uygulamamızı daha hızlı hale getirmek istiyoruz veya onu Qt’nin gücüyle genişletmek veya QML’de bulunmayan bazı fırsatlar eklemek istiyoruz. Ve kesinlikle QML‘de böyle bir olasılık var. Temel olarak QtQuick, içeriğini OpenGL tabanlı yüksek performanslı bir işleme motoru boyamak için Scene Graph kullanır. Kendi görsel öğemizi uygulamak için 2 yol kullanabiliriz:

  1. QPainter (QQuickPaintedItem) kullanarak Qt için geleneksel yöntem.
  2. QQuickItem ve OpenGL işlevselliğini kullanan ortak QML yolu.

İlk yöntemin daha kolay görünmesi mümkündür, ancak QtQuick öğenin içeriğini bir yüzeyde boyadığından ve daha sonra ilkinden daha yavaş olduğunu düşünmeye değer. oluşturmanın iki aşamalı bir işlem olması için sahne grafiğine ekleyin. Bu nedenle, sahne grafiği API’sini doğrudan kullanmak her zaman önemli ölçüde daha hızlıdır.

Her iki yöntemi de daha yakından keşfetmek için, QML’de kesinlikle bulunmayan kendi elemanımızı, örneğin bir üçgeni oluşturalım.

Sınıf beyanı

class QQuickCustomItem : public QQuickItem
{
    Q_OBJECT
    Q_PROPERTY(QColor color READ color WRITE setColor NOTIFY colorChanged)
public:
    QQuickCustomItem(QQuickItem *parent = Q_NULLPTR);

protected:
    QSGNode *updatePaintNode(QSGNode *oldNode, UpdatePaintNodeData *updatePaintNodeData);

    QColor color() const;
    void setColor(const QColor &color);

private:
    QColor m_color;
    bool m_needUpdate;

signals:
    void colorChanged();
};

Sinyallerle çalışmak için Q_OBJECT makrosu ekliyoruz. Ayrıca Dikdörtgenimizin rengini belirtmek için özel özellik ekliyoruz. Çalışması için tek ihtiyacımız olan sanal işlevi QQuiclItem::updatePaintNode() yeniden uygulamak.

Sınıf uygulaması.

İlk olarak bir kurucu tanımlıyoruz.

QQuickCustomItem::QQuickCustomItem(QQuickItem *parent) :
    QQuickItem(parent),
    m_color(Qt::red),
    m_needUpdate(true)
{
    setFlag(QQuickItem::ItemHasContents);
}

Lütfen setFlag() işlev çağrısının zorunlu olduğunu unutmayın, aksi takdirde nesneniz sahne grafiğine eklenmeyecektir. Daha sonra, ağrı için bir fonksiyon tanımlarız.

QSGNode *QQuickCustomItem::updatePaintNode(QSGNode *oldNode, QQuickItem::UpdatePaintNodeData *updatePaintNodeData)
{
    Q_UNUSED(updatePaintNodeData)
    QSGGeometryNode *root = static_cast<QSGGeometryNode *>(oldNode);

    if(!root) {
        root = new QSGGeometryNode;
        QSGGeometry *geometry = new QSGGeometry(QSGGeometry::defaultAttributes_Point2D(), 3);
        geometry->setDrawingMode(GL_TRIANGLE_FAN);
        geometry->vertexDataAsPoint2D()[0].set(width() / 2, 0);
        geometry->vertexDataAsPoint2D()[1].set(width(), height());
        geometry->vertexDataAsPoint2D()[2].set(0, height());

        root->setGeometry(geometry);
        root->setFlag(QSGNode::OwnsGeometry);
        root->setFlag(QSGNode::OwnsMaterial);
    }

    if(m_needUpdate) {
        QSGFlatColorMaterial *material = new QSGFlatColorMaterial;
        material->setColor(m_color);
        root->setMaterial(material);
        m_needUpdate = false;
    }

    return root;
}

İşleve yapılan ilk çağrıda düğümümüz henüz oluşturulmadı, bu nedenle ‘oldNode’ NULL olacaktır. Böylece düğümü oluşturuyoruz ve ona geometri ve malzeme atıyoruz. Burada geometrimizin katı dikdörtgeni boyaması için GL_TRIANGLE_FAN kullanıyoruz. Bu nokta OpenGL’deki ile aynıdır. Örneğin üçgen çerçeve çizmek için kodu şu şekilde değiştirebiliriz:

geometry->setDrawingMode(GL_LINE_LOOP);
geometry->setLineWidth(5);

Diğer şekilleri kontrol etmek için OpenGL kılavuzuna başvurabilirsiniz. Yani, geriye kalan tek şey mülkümüz için ayarlayıcı/alıcıyı tanımlamak:

QColor QQuickCustomItem::color() const
{
    return m_color;
}

void QQuickCustomItem::setColor(const QColor &color)
{
    if(m_color != color) {
        m_color = color;
        m_needUpdate = true;
        update();
        colorChanged();
    }
}

Şimdi çalışması için sadece küçük bir detay var. Yeni öğeyi QtQuick’e bildirmemiz gerekiyor. Örneğin, bu kodu main.cpp’nize ekleyebilirsiniz:

qmlRegisterType<QQuickCustomItem>("stackoverflow.qml", 1, 0, "Triangle");

Ve işte QML test dosyamız:

import QtQuick 2.7
import QtQuick.Window 2.0
import stackoverflow.qml 1.0

Window {
    width: 800
    height: 800
    visible: true

    Rectangle {
        width: 200
        height: 200
        anchors.centerIn: parent
        color: "lightgrey"

        Triangle {
            id: rect
            width: 200
            height: 200
            transformOrigin: Item.Top
            color: "green"
            onColorChanged: console.log("color was changed");
            PropertyAnimation on rotation {
                from: 0
                to: 360
                duration: 5000
                loops: Animation.Infinite
            }
        }
    }
    Timer {
        interval: 1000
        repeat: true
        running: true
        onTriggered: rect.color = Qt.rgba(Math.random(),Math.random(),Math.random(),1);
    }
}

Gördüğünüz gibi, öğemiz diğer tüm QML öğeleri gibi davranır. Şimdi aynı öğeyi QPainter kullanarak oluşturalım:

İhtiyacımız olan tek şey değiştirmek

QSGNode *updatePaintNode(QSGNode *oldNode, UpdatePaintNodeData *updatePaintNodeData);

ile birlikte

void paint(QPainter *painter);

ve tabii ki sınıfımızı ‘QQuickItem’ yerine ‘QQuickPaintedItem’den devralır. İşte boyama fonksiyonumuz:

void QQuickCustomItem::paint(QPainter *painter)
{
    QPainterPath path;
    path.moveTo(width() / 2, 0);
    path.lineTo(width(), height());
    path.lineTo(0, height());
    path.lineTo(width() / 2, 0);
    painter->fillPath(path, m_color);
}

Diğer her şey değişmeden kalır.
