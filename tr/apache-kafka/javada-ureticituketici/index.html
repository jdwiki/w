Bu konu, Java’da kayıtların nasıl üretileceğini ve tüketileceğini gösterir.

SimpleConsumer (Kafka >= 0.9.0) #

Kafka’nın 0.9 sürümü, kafka tüketicisinin tamamen yeniden tasarlanmasını sağladı. Eski “SimpleConsumer” (0.8.X) ile ilgileniyorsanız, bu sayfaya bir göz atın. Kafka kurulumunuz 0.8.X’ten daha yeniyse, aşağıdaki kodlar kutudan çıkar çıkmaz çalışmalıdır.

Yapılandırma ve başlatma #

Kafka 0.9 artık Java 6 veya Scala 2.9’u desteklememektedir. Hala Java 6 kullanıyorsanız, desteklenen bir sürüme yükseltmeyi düşünün.

İlk önce bir maven projesi oluşturun ve pom’inize aşağıdaki bağımlılığı ekleyin:

<dependencies>
    <dependency>
        <groupId>org.apache.kafka</groupId>
        <artifactId>kafka-clients</artifactId>
        <version>0.9.0.1</version>
    </dependency>
</dependencies>

Note : en son sürümler (şimdi > 0.10) için sürüm alanını güncellemeyi unutmayın.

Tüketici, bir “Özellikler” nesnesi kullanılarak başlatılır. Tüketici davranışına ince ayar yapmanızı sağlayan birçok özellik vardır. Aşağıda gereken minimum yapılandırma verilmiştir:

Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("group.id", "consumer-tutorial");
props.put("key.deserializer", StringDeserializer.class.getName());
props.put("value.deserializer", StringDeserializer.class.getName());

“Önyükleme-sunucuları”, tüketicinin kümenin geri kalanını keşfedebilmesi için ilk aracılar listesidir. Bunun kümedeki tüm sunucular olması gerekmez: istemci, bu listedeki aracılardan tam canlı aracı kümesini belirleyecektir.

‘Seri hale getirici’ tüketiciye mesaj anahtarlarını ve değerlerini nasıl yorumlayacağını/seri hale getireceğini söyler. Burada yerleşik StringDeserializer kullanıyoruz.

Son olarak, “group.id” bu müşterinin tüketici grubuna karşılık gelir. Unutmayın: bir tüketici grubunun tüm tüketicileri mesajları aralarında böler (kafka bir mesaj kuyruğu gibi davranırken), farklı tüketici gruplarından tüketiciler aynı mesajları alır (kafka bir yayınla-abone ol sistemi gibi davranır).

Diğer faydalı özellikler şunlardır:

  • “auto.offset.reset”: Zookeeper’da saklanan ofset eksik veya aralık dışındaysa ne yapılacağını kontrol eder. Olası değerler “en son” ve “en erken"dir. Başka herhangi bir şey bir istisna atar;

  • “enable.auto.commit”: “true” ise (varsayılan), tüketici ofseti periyodik olarak (bkz. “auto.commit.interval.ms”) arka planda kaydedilir. Bunu “false” olarak ayarlamak ve “auto.offset.reset=earliest” - kullanmak, taahhüt edilen ofset bilgisinin bulunmaması durumunda tüketicinin nereden başlaması gerektiğini belirlemektir. ’en erken’, atanan konu bölümünün başlangıcından itibaren anlamına gelir. “en son”, bölüm için en yüksek kullanılabilir taahhüt edilmiş ofset sayısından anlamına gelir. Ancak, Kafka tüketicisi, geçerli bir denkleştirme kaydı bulunduğu sürece (yani, “auto.offset.reset” yok sayıldığı sürece) her zaman son taahhüt edilen ofsetten devam edecektir. En iyi örnek, yepyeni bir tüketici grubunun bir konuya abone olmasıdır. konunun başından (en erken) veya sonundan (en son) başlayıp başlamayacağını belirlemek için ‘auto.offset.reset’i kullanır.

  • session.timeout.ms: bir oturum zaman aşımı, tüketici çökerse veya bir ağ bölümü tüketiciyi koordinatörden yalıtırsa kilidin serbest bırakılmasını sağlar. Aslında:

    When part of a consumer group, each consumer is assigned a subset of the partitions from topics it has subscribed to. This is basically a group lock on those partitions. As long as the lock is held, no other members in the group will be able to read from them. When your consumer is healthy, this is exactly what you want. It’s the only way that you can avoid duplicate consumption. But if the consumer dies due to a machine or application failure, you need that lock to be released so that the partitions can be assigned to a healthy member. source

Özelliklerin tam listesi buradahttp://kafka.apache.org/090/documentation.html#newconsumerconfigs mevcuttur.

Tüketici oluşturma ve konu aboneliği #

Bu özelliklere sahip olduğumuzda, bir tüketici oluşturmak kolaydır:

KafkaConsumer<String, String> consumer = new KafkaConsumer<>( props );
consumer.subscribe( Collections.singletonList( "topic-example" ) );

Abone olduktan sonra, tüketici, bölüm atamasını almak için grubun geri kalanıyla koordine edebilir. Tüm bunlar, veri tüketmeye başladığınızda otomatik olarak gerçekleştirilir.

Temel anket #

Tüketicinin, birçok aracıya yayılmış olması muhtemel birçok konu için potansiyel olarak birçok bölümden paralel olarak veri alabilmesi gerekir. Neyse ki, veri tüketmeye başladığınızda bunların hepsi otomatik olarak gerçekleştirilir. Bunu yapmak için tek yapmanız gereken bir döngüde ‘anket’ çağırmak ve gerisini tüketici halleder.

“anket”, atanmış bölümlerden (muhtemelen boş) bir dizi mesaj döndürür.

while( true ){
    ConsumerRecords<String, String> records = consumer.poll( 100 );
    if( !records.isEmpty() ){
        StreamSupport.stream( records.spliterator(), false ).forEach( System.out::println );
    }
}

Kod #

Temel örnek #

Bu, bir kafka konusundan mesaj almak için kullanabileceğiniz en temel koddur.

public class ConsumerExample09{

    public static void main( String[] args ){

        Properties props = new Properties();
        props.put( "bootstrap.servers", "localhost:9092" );
        props.put( "key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer" );
        props.put( "value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer" );
        props.put( "auto.offset.reset", "earliest" );
        props.put( "enable.auto.commit", "false" );
        props.put( "group.id", "octopus" );

        try( KafkaConsumer<String, String> consumer = new KafkaConsumer<>( props ) ){
            consumer.subscribe( Collections.singletonList( "test-topic" ) );

            while( true ){
                // poll with a 100 ms timeout
                ConsumerRecords<String, String> records = consumer.poll( 100 );
                if( records.isEmpty() ) continue;
                StreamSupport.stream( records.spliterator(), false ).forEach( System.out::println );
            }
        }
    }
}

Çalıştırılabilir örnek #

Tüketici, kendi iş parçacığında çalıştırılmak üzere tasarlanmıştır. Harici senkronizasyon olmadan çok iş parçacıklı kullanım için güvenli değildir ve muhtemelen denemek iyi bir fikir değildir.

Aşağıda, tüketiciyi başlatan, bir konu listesine abone olan ve harici olarak kapatılana kadar anket döngüsünü süresiz olarak yürüten basit bir Çalıştırılabilir görev bulunmaktadır.

public class ConsumerLoop implements Runnable{
    private final KafkaConsumer<String, String> consumer;
    private final List<String> topics;
    private final int id;

    public ConsumerLoop( int id, String groupId, List<String> topics ){
        this.id = id;
        this.topics = topics;
        Properties props = new Properties();
        props.put( "bootstrap.servers", "localhost:9092");
        props.put( "group.id", groupId );
        props.put( "auto.offset.reset", "earliest" );
        props.put( "key.deserializer", StringDeserializer.class.getName() );
        props.put( "value.deserializer", StringDeserializer.class.getName() );
        this.consumer = new KafkaConsumer<>( props );
    }
    
    @Override
    public void run(){
        try{
            consumer.subscribe( topics );

            while( true ){
                ConsumerRecords<String, String> records = consumer.poll( Long.MAX_VALUE );
                StreamSupport.stream( records.spliterator(), false ).forEach( System.out::println );
            }
        }catch( WakeupException e ){
            // ignore for shutdown 
        }finally{
            consumer.close();
        }
    }


    public void shutdown(){
        consumer.wakeup();
    }
}

Anket sırasında Long.MAX_VALUE zaman aşımı kullandığımızı unutmayın, bu nedenle süresiz olarak yeni bir mesaj bekleyecektir. Tüketiciyi düzgün bir şekilde kapatmak için uygulamayı sonlandırmadan önce shutdown() yöntemini çağırmak önemlidir.

Bir sürücü bunu şu şekilde kullanabilir:

public static void main( String[] args ){

    int numConsumers = 3;
    String groupId = "octopus";
    List<String> topics = Arrays.asList( "test-topic" );

    ExecutorService executor = Executors.newFixedThreadPool( numConsumers );
    final List<ConsumerLoop> consumers = new ArrayList<>();

    for( int i = 0; i < numConsumers; i++ ){
        ConsumerLoop consumer = new ConsumerLoop( i, groupId, topics );
        consumers.add( consumer );
        executor.submit( consumer );
    }

    Runtime.getRuntime().addShutdownHook( new Thread(){
        @Override
        public void run(){
            for( ConsumerLoop consumer : consumers ){
                consumer.shutdown();
            }
            executor.shutdown();
            try{
                executor.awaitTermination( 5000, TimeUnit.MILLISECONDS );
            }catch( InterruptedException e ){
                e.printStackTrace();
            }
        }
    } );
}

SimpleProducer (kafka >= 0.9) #

Yapılandırma ve başlatma #

İlk önce bir maven projesi oluşturun ve pom’inize aşağıdaki bağımlılığı ekleyin:

 <dependencies>
     <dependency>
         <groupId>org.apache.kafka</groupId>
         <artifactId>kafka-clients</artifactId>
         <version>0.9.0.1</version>
     </dependency>
 </dependencies>

Üretici, bir “Özellikler” nesnesi kullanılarak başlatılır. Üretici davranışına ince ayar yapmanıza izin veren birçok özellik vardır. Aşağıda gereken minimum yapılandırma verilmiştir:

Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("client.id", "simple-producer-XX");

“Önyükleme sunucuları”, üreticinin kümenin geri kalanını keşfedebilmesi için bir veya daha fazla aracının ilk listesidir. Serileştirici özellikleri Kafka’ya mesaj anahtarının ve değerinin nasıl kodlanması gerektiğini söyler. Burada string mesajları göndereceğiz. Gerekli olmamasına rağmen, o zamandan beri bir “client.id” ayarlamak her zaman önerilir: bu, aracıdaki istekleri, onu yapan müşteri eşgörünümüyle kolayca ilişkilendirmenize olanak tanır.

Diğer ilginç özellikler şunlardır:

props.put("acks", "all"); 
props.put("retries", 0);  
props.put("batch.size", 16384);
props.put("linger.ms", 1);
props.put("buffer.memory", 33554432);

‘acks’ ayarıyla Kafka’ya yazılan _mesajların dayanıklılığını kontrol edebilirsiniz. Varsayılan “1” değeri, bölüm liderinden yazmanın başarılı olduğuna dair açık bir onay gerektirir. Kafka’nın sağladığı en güçlü garanti, yalnızca bölüm liderinin yazmayı kabul ettiğini değil, aynı zamanda tüm eşitlenmiş kopyalara başarıyla çoğaltıldığını garanti eden ‘acks=all’ iledir. Verimi en üst düzeye çıkarmak için “0” değerini de kullanabilirsiniz, ancak bu durumda aracı bir yanıt bile göndermediğinden iletinin aracının günlüğüne başarıyla yazıldığının garantisi yoktur.

“reries” (varsayılan >0), üreticinin bir hatadan sonra mesajı yeniden göndermeye çalışıp çalışmayacağını belirler. Yeniden deneme > 0 olduğunda, bir sonraki yazma başarılı olduktan sonra yeniden deneme gerçekleşebileceğinden ileti yeniden sıralamasının gerçekleşebileceğini unutmayın.

Kafka üreticileri, verimi artırmak için gönderilen mesajları gruplar halinde toplamaya çalışır. Java istemcisi ile, her ileti grubunun bayt cinsinden maksimum boyutunu kontrol etmek için “batch.size” öğesini kullanabilirsiniz. Grupların doldurulması için daha fazla zaman vermek için, üreticinin göndermeyi geciktirmesi için “linger.ms"yi kullanabilirsiniz. Son olarak, sıkıştırma, “compression.type” ayarıyla etkinleştirilebilir.

Java istemcisinin gönderilmemiş mesajları toplamak için kullanabileceği toplam belleği sınırlamak için “buffer.memory"yi kullanın. Bu sınıra ulaşıldığında, üretici bir istisna oluşturmadan önce “max.block.ms” kadar ek gönderimleri engeller. Ek olarak, kayıtların süresiz olarak sıraya alınmasını önlemek için ‘request.timeout.ms’ kullanarak bir zaman aşımı ayarlayabilirsiniz.

Mülklerin tam listesine buradan ulaşabilirsiniz. Daha fazla ayrıntı için Confluent’ten bu makaleyi okumanızı öneririm.

Mesaj gönderme #

send() yöntemi eşzamansızdır. Çağrıldığında, kaydı bekleyen kayıt gönderme arabelleğine ekler ve hemen geri döner. Bu, üreticinin verimlilik için bireysel kayıtları bir araya getirmesini sağlar.

Göndermenin sonucu, kaydın gönderildiği bölümü ve atandığı ofseti belirten bir ‘RecordMetadata’dır. Gönderme çağrısı asenkron olduğundan, bu kayda atanacak RecordMetadata için bir ‘Gelecek’ döndürür. Meta verilere bakmak için, istek tamamlanana kadar engellenecek olan get()‘i çağırabilir veya bir geri arama kullanabilirsiniz.

// synchronous call with get()
RecordMetadata recordMetadata = producer.send( message ).get();
// callback with a lambda
producer.send( message, ( recordMetadata, error ) -> System.out.println(recordMetadata) );

Kod #

public class SimpleProducer{
    
    public static void main( String[] args ) throws ExecutionException, InterruptedException{
        Properties props = new Properties();

        props.put("bootstrap.servers", "localhost:9092");
        props.put("acks", "all");
        props.put("retries", 0);
        props.put("batch.size", 16384);
        props.put("linger.ms", 1);
        props.put("buffer.memory", 33554432);
        props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        props.put( "client.id", "octopus" );

        String topic = "test-topic";

        Producer<String, String> producer = new KafkaProducer<>( props );

        for( int i = 0; i < 10; i++ ){
            ProducerRecord<String, String> message = new ProducerRecord<>( topic, "this is message " + i );
            producer.send( message );
            System.out.println("message sent.");
        }

        producer.close(); // don't forget this
    }
}
