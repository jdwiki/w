Bu sayfada

  • Nesne adına göre revizyon belirtme
  • Sembolik referans adları: dallar, etiketler, uzaktan izleme dalları
  • Varsayılan revizyon: HEAD
  • Reflog referansları: @{}
  • Reflog referansları: @{}
  • İzlenen / yukarı akış şubesi: @{upstream}
  • Ata zincirini taahhüt et: ^, ~, vb.
  • Dalları ve etiketleri referans alma: ^0, ^{}
  • En genç eşleşen taahhüt: ^{/}, :/

Birçok Git komutu, revizyon parametrelerini argüman olarak alır. Komuta bağlı olarak, belirli bir taahhüdü belirtirler veya revizyon grafiğinde yürüyen komutlar için (git-log(1) gibi), bu taahhütten ulaşılabilen tüm taahhütler . Sözdizimi açıklamasında genellikle “<taahhüt>” veya “” veya “” olarak belirtilirler.

Git revizyonları sözdizimi için başvuru belgeleri gitrevisions(7) kılavuz sayfasıdır.

Bu sayfada hala eksik:

  • [_] “git tanımlama"dan çıktı, ör. v1.7.4.2-679-g3bee7fb
  • [_] @ tek başına HEAD kısayolu olarak
  • [_] @{-<n>}, ör. “@{-1}” ve “-”, “@{-1}” anlamına gelir
  • [_] <branchname>@{push}
  • [_] <rev>^@, tüm <rev> ebeveynleri için

Ayrı belgelere ihtiyaç duyar:

  • [_] Depodaki ve dizindeki bloblara ve ağaçlara atıfta bulunarak: <rev>:<path> ve :<n>:<path> sözdizimi
  • [_] “A..B”, “A…B”, “B ^A”, “A^1” gibi revizyon aralıkları ve “-”, “–since” gibi revizyon sınırlamaları

Nesne adına göre revizyon belirtme #

$ git show dae86e1950b1277e545cee180551750029cfe735
$ git show dae86e19

SHA-1 nesne adını, tam 40 baytlık onaltılık dizeyi veya depoya özgü bir alt dizeyi kullanarak revizyonu (veya gerçekte herhangi bir nesne: etiket, ağaç yani dizin içeriği, blob yani dosya içeriği) belirtebilirsiniz.

Sembolik referans adları: dallar, etiketler, uzaktan izleme dalları #

$ git log master    # specify branch
$ git show v1.0     # specify tag
$ git show HEAD     # specify current branch
$ git show origin   # specify default remote-tracking branch for remote 'origin'

Dallar (örneğin ‘master’, ’next’, ‘maint’), etiketler (örneğin ‘v1.0’, ‘v0.6.3-rc2’), uzak- izleme dalları (örneğin ‘origin’, ‘origin/master’) ve mevcut dal için ‘HEAD’ gibi özel referanslar.

Sembolik referans adı belirsizse, örneğin ‘fix’ adında hem dalınız hem de etiketiniz varsa (dal ve etiketin aynı ada sahip olması önerilmez), kullanmak istediğiniz referans türünü belirtmeniz gerekir:

$ git show heads/fix      # or 'refs/heads/fix', to specify branch
$ git show tags/fix       # or 'refs/tags/fix', to specify tag

Varsayılan revizyon: HEAD #

$ git show         # equivalent to 'git show HEAD'

‘HEAD’, çalışan ağaçtaki değişiklikleri temel aldığınız taahhüdü adlandırır ve genellikle mevcut dalın sembolik adıdır. Eksikse, revizyon parametresi varsayılanlarını ‘HEAD’ olarak alan birçok (ancak tümü değil) komut.

Reflog referansları: @{} #

$ git show @{1}            # uses reflog for current branch
$ git show master@{1}      # uses reflog for branch 'master'
$ git show HEAD@{1}        # uses 'HEAD' reflog

Bir başvuru, genellikle bir dal veya HEAD, ardından bir küme ayracı içine alınmış bir sıra belirtimi olan “@” son eki (ör. “{1}”, “{15}”), bu başvurunun n’inci önceki değerini belirtir * yerel deponuzda*. Son reflog girişlerini git reflog komutu ile veya --walk-reflogs / -g seçeneği ile git log kontrol edebilirsiniz.

$ git reflog
08bb350 HEAD@{0}: reset: moving to HEAD^
4ebf58d HEAD@{1}: commit: gitweb(1): Document query parameters
08bb350 HEAD@{2}: pull: Fast-forward
f34be46 HEAD@{3}: checkout: moving from af40944bda352190f05d22b7cb8fe88beb17f3a7 to master
af40944 HEAD@{4}: checkout: moving from master to v2.6.3

$ git reflog gitweb-docs
4ebf58d gitweb-docs@{0}: branch: Created from master

Not: reflog’ların kullanılması, ‘ORIG_HEAD’ referansını kullanmanın eski mekanizmasının yerini almıştır (kabaca ‘HEAD@{1}’ ile eşdeğerdir).

Reflog referansları: @{} #

$ git show master@{yesterday}
$ git show HEAD@{5 minutes ago}   # or HEAD@{5.minutes.ago}

Bir ayraç çifti (ör. “{dün}”, “{1 ay 2 hafta 3 gün 1 saat 1 saniye önce}”) veya “{1979-02-26 18 :30:00}`), zamanın önceki bir noktasındaki (veya ona en yakın noktadaki) ref değerini belirtir. Bunun, belirli bir zamanda yerel referansınızın durumunu aradığını unutmayın; örneğin, geçen hafta yerel ‘usta’ şubenizde neler vardı.

git reflog‘u, yerel depoda verilen ref’e bir şey yaptığınız tam zamanı aramak için bir tarih belirteciyle birlikte kullanabilirsiniz.

$ git reflog HEAD@{now}
08bb350 HEAD@{Sat Jul 23 19:48:13 2016 +0200}: reset: moving to HEAD^
4ebf58d HEAD@{Sat Jul 23 19:39:20 2016 +0200}: commit: gitweb(1): Document query parameters
08bb350 HEAD@{Sat Jul 23 19:26:43 2016 +0200}: pull: Fast-forward

İzlenen / yukarı akış şubesi: @{upstream} #

$ git log @{upstream}..       # what was done locally and not yet published, current branch
$ git show master@{upstream}  # show upstream of branch 'master'

Bir dal adına eklenen @{upstream} soneki (kısa biçim <branchname>@{u}), branchname ile belirtilen şubenin üzerine inşa etmek üzere ayarlandığı dalı ifade eder (branch.<name ile yapılandırılır). >.remote ve branch.<name>.merge veya git branch --set-upstream-to=<branch> ile). Eksik bir dal adı varsayılan olarak geçerli bir daldır.

Revizyon aralıkları için sözdizimi ile birlikte, şubenizin yukarı yöndeki taahhütleri (yerel deponuzda henüz mevcut olmayan taahhütler) ve geride olduğunuz taahhütleri (yukarı akıştaki taahhütler yerel şubeyle birleştirilmemiştir) veya ikisi birden:

$ git log --oneline @{u}..
$ git log --oneline ..@{u}
$ git log --oneline --left-right @{u}...  # same as ...@{u}

Ata zincirini taahhüt et: ^, ~, vb. #

$ git reset --hard HEAD^             # discard last commit
$ git rebase --interactive HEAD~5    # rebase last 4 commits

Bir revizyon parametresinin ^' soneki, o kesinleştirme nesnesinin ilk ebeveyni anlamına gelir. ^, \<n>-th ebeveyni anlamına gelir (yani ^, ^1` ile eşdeğerdir).

Bir revizyon parametresinin “~” soneki, yalnızca ilk üst öğeyi takip eden, adlandırılmış taahhüt nesnesinin <n>-inci nesil atası olan taahhüt nesnesi anlamına gelir. Bu, örneğin “~3” ifadesinin “^^^” ile eşdeğer olduğu anlamına gelir. Kısayol olarak, “~”, “~1” anlamına gelir ve “^1” veya kısaca “^” ile eşdeğerdir.

Bu sözdizimi birleştirilebilir.

Bu tür sembolik isimleri bulmak için git name-rev komutunu kullanabilirsiniz:

$ git name-rev 33db5f4d9027a10e477ccf054b2c1ab94f74c85a
33db5f4d9027a10e477ccf054b2c1ab94f74c85a tags/v0.99~940

Aşağıdaki örnekte “–pretty=oneline” değil “–oneline” kullanılması gerektiğine dikkat edin.

$ git log --pretty=oneline | git name-rev --stdin --name-only
master Sixth batch of topics for 2.10
master~1 Merge branch 'ls/p4-tmp-refs'
master~2 Merge branch 'js/am-call-theirs-theirs-in-fallback-3way'
[...]
master~14^2 sideband.c: small optimization of strbuf usage
master~16^2 connect: read $GIT_SSH_COMMAND from config file
[...]
master~22^2~1 t7810-grep.sh: fix a whitespace inconsistency
master~22^2~2 t7810-grep.sh: fix duplicated test name

Dalları ve etiketleri referans alma: ^0, ^{} #

Bazı durumlarda bir komutun davranışı, kendisine dal adı, etiket adı veya rastgele bir revizyon verilmesine bağlıdır. İkinciye ihtiyacınız varsa, “referans kaldırma” sözdizimini kullanabilirsiniz.

Ayraç çifti (örneğin ‘v0.99.8^{commit}’) içine alınmış bir nesne türü adının (’etiket’, ‘bağla’, ‘ağaç’, ‘blob’) ardından gelen ‘^’ soneki, nesnenin “”, “” türünde bir nesne bulunana veya nesnenin başvurusu artık kaldırılamayana kadar yinelemeli olarak. “^0”, “^{commit}” ifadesinin kısaltılmış halidir.

$ git checkout HEAD^0    # equivalent to 'git checkout --detach' in modern Git

Bir “^” soneki ardından boş bir ayraç çifti (örneğin “v0.99.8^{}”), etiket olmayan bir nesne bulunana kadar etiketin tekrar tekrar başvurudan çıkarılması anlamına gelir.

Karşılaştırmak

$ git show v1.0
$ git cat-file -p v1.0
$ git replace --edit v1.0

ile birlikte

$ git show v1.0^{}
$ git cat-file -p v1.0^{}
$ git replace --edit v1.0^{}

En genç eşleşen taahhüt: ^{/}, :/ #

$ git show HEAD^{/fix nasty bug}   # find starting from HEAD
$ git show ':/fix nasty bug'       # find starting from any branch

İki nokta üst üste (’:’), ardından bir eğik çizgi (’’/'), ardından bir metin, taahhüt mesajı belirtilen normal ifadeyle eşleşen bir taahhüt adını verir. Bu ad, *any* ref'den ulaşılabilen en genç eşleştirme taahhüdünü döndürür. Normal ifade, taahhüt mesajının herhangi bir bölümüyle eşleşebilir. Bir dize ile başlayan iletileri eşleştirmek için, ör. :/^foo. :/!özel dizisi, eşleşenle ilgili değiştiriciler için ayrılmıştır.:/!-foonegatif bir eşleşme gerçekleştirirken,:/!!foo` değişmez ! karakter, ardından “foo”.

Bir revizyon parametresinin ‘^’ soneki ve ardından bir eğik çizgi ile başlayan bir metin içeren bir küme ayracı çifti, aşağıdaki ‘:/’ sözdizimiyle aynıdır; “^“den önce “”.
