Sözdizimi #

  • git uzak [-v | --ayrıntılı]
  • git remote add [-t <branch>] [-m <master>] [-f] [--[no-]tags] [--mirror=<fetch|push>]<name> <url>
  • git uzaktan yeniden adlandır <eski> <yeni>
  • `git remote ‘yi kaldır
  • git remote set-head <ad> (-a | --auto | -d | --delete | <branch>)
  • git uzak set-dallar [--add] <name> <branch>...
  • git remote set-url [--push] <name> <newurl> [<oldurl>]
  • git remote set-url --add [--push] <name> <newurl>
  • git remote set-url --delete [--push] <name> <url>
  • git uzak [-v | --verbose] göster [-n] <ad>...
  • git uzak kuru erik [-n | --kuru çalışma] <ad>...
  • git uzak [-v | --verbose] güncelleme [-p | --prune] [(<grup> | <uzaktan>)...]
  • git uzak gösteri <ad>

Parametreler #

parametre Ayrıntılar
-v, –ayrıntılı Ayrıntılı olarak çalıştırın.
-m <usta> Başı uzaktan kumandanın <master> şubesine ayarlar
–mirror=getir Referanslar, refs/remotes ad alanında saklanmayacak, bunun yerine yerel depoda yansıtılacak
–mirror=itme git push –mirror geçirilmiş gibi davranacak
– etiketsiz git fetch <name> uzak depodan etiketleri içe aktarmıyor
-t <şube> yalnızca izlenecek uzaktan kumandayı belirtir <branch>
-f git fetch <name>, uzaktan kumanda kurulduktan hemen sonra çalıştırılır
–etiketler git fetch <name> uzak depodan her etiketi içe aktarır
-a, –oto Sembolik referansın HEAD’i, uzaktan kumandanın HEAD
-d, –delete Listelenen tüm referanslar uzak depodan silinir
–ekle Şu anda izlenen dallar (set-dallar) listesine <name> ekler
–ekle Bazı URL’leri değiştirmek yerine yeni URL eklendi (set-url)
–hepsi Tüm dalları itin.
–sil <url> ile eşleşen tüm url’ler silinir. (set-url)
–itme Push URL’leri, getirme URL’leri yerine işlenir
-n Uzak kafalar önce git ls-remote <name> ile sorgulanmaz, bunun yerine önbelleğe alınan bilgiler kullanılır
–kuru çalışma hangi dalların budanacağını bildirin, ancak onları gerçekten budamayın
–kuru erik Yerel karşılığı olmayan uzak dalları kaldırın

Uzak Depoları Görüntüle #

Tüm yapılandırılmış uzak depoları listelemek için git remote kullanın.

Yapılandırdığınız her bir uzak tanıtıcının kısa adını (diğer adları) gösterir.

$ git remote
premium
premiumPro
origin

Daha ayrıntılı bilgi göstermek için --verbose veya -v bayrağı kullanılabilir. Çıktı, URL’yi ve uzaktan kumandanın türünü (“push” veya “pull”) içerecektir:

$ git remote -v
premiumPro    https://github.com/user/CatClickerPro.git (fetch)
premiumPro    https://github.com/user/CatClickerPro.git (push)
premium    https://github.com/user/CatClicker.git (fetch)
premium    https://github.com/user/CatClicker.git (push)
origin    https://github.com/ud/starter.git (fetch)
origin    https://github.com/ud/starter.git (push)

Git deponuzun uzak URL’sini değiştirin #

Uzak depo taşınırsa bunu yapmak isteyebilirsiniz. Uzak URL’yi değiştirme komutu:

git remote set-url

2 bağımsız değişken alır: mevcut bir uzak ad (origin, upstream) ve url.

Mevcut uzak url’nizi kontrol edin:

git remote -v
origin    https://bitbucket.com/develop/myrepo.git (fetch)
origin    https://bitbucket.com/develop/myrepo.git (push)

Uzak URL’nizi değiştirin:

git remote set-url origin https://localserver/develop/myrepo.git

Uzak URL’nizi tekrar kontrol edin:

git remote -v
origin    https://localserver/develop/myrepo.git (fetch)
origin    https://localserver/develop/myrepo.git (push)

Uzak Depoyu Kaldırma #

<name> adlı uzaktan kumandayı kaldırın. Uzaktan kumanda için tüm uzaktan izleme dalları ve yapılandırma ayarları kaldırılır.

Uzak bir depoyu “dev” kaldırmak için:

git remote rm dev

Uzak Depo Ekle #

Uzaktan kumanda eklemek için yerel deponuzun kök dizininde git remote add kullanın.

<url> uzak Git deposunu kolay bir kısa ad olarak eklemek için <name> kullanın

git remote add <name> <url>

“git fetch ” komutu daha sonra “/” uzaktan izleme dallarını oluşturmak ve güncellemek için kullanılabilir.

Uzak depo hakkında daha fazla bilgi göster #

Uzak depo hakkında daha fazla bilgiyi git remote show <remote repository alias> ile görüntüleyebilirsiniz.

git remote show origin

sonuç:

remote origin
Fetch URL:  https://localserver/develop/myrepo.git
Push  URL:  https://localserver/develop/myrepo.git
HEAD branch: master
Remote branches:
  master      tracked
Local branches configured for 'git pull':
  master      merges with remote master
Local refs configured for 'git push':
  master      pushes to master      (up to date)

Uzak Depoyu Yeniden Adlandır #

“” adlı uzaktan kumandayı “” olarak yeniden adlandırın. Uzaktan kumanda için tüm uzaktan izleme dalları ve yapılandırma ayarları güncellenir.

“dev” uzak şube adını “dev1” olarak yeniden adlandırmak için:

git remote rename dev dev1
