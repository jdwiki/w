Sözdizimi #

  • git log [seçenekler] [revizyon aralığı] [[–] yol…]

Parametreler #

Parametre          Açıklama
-q, –sessiz Sessiz, fark çıktısını bastırır
–kaynak Taahhüt kaynağını gösterir
–use-mailmap Posta eşleme dosyasını kullan (kullanıcıyı taahhüt etmek için kullanıcı bilgilerini değiştirir)
–süslemek[=…] Süsleme seçenekleri
–L <n,m:dosya> Bir dosyada 1’den başlayarak belirli satır aralığı için günlüğü göster. n satırından başlar, m satırına gider. Ayrıca fark gösterir.
–göster-imza İmzalanan taahhütlerin imzalarını göster
-i, –regexp-ignore-case Büyük/küçük harfe bakılmaksızın normal ifade sınırlayıcı kalıpları eşleştirin

Referanslar ve güncel belgeler: git-log resmi belgeler

Daha güzel günlük #

Günlüğü daha güzel bir grafik benzeri yapıda görmek için şunu kullanın:

git log --decorate --oneline --graph

örnek çıktı:

* e0c1cea (HEAD -> maint, tag: v2.9.3, origin/maint) Git 2.9.3
*   9b601ea Merge branch 'jk/difftool-in-subdir' into maint
|\  
| * 32b8c58 difftool: use Git::* functions instead of passing around state
| * 98f917e difftool: avoid $GIT_DIR and $GIT_WORK_TREE
| * 9ec26e7 difftool: fix argument handling in subdirs
* |   f4fd627 Merge branch 'jk/reset-ident-time-per-commit' into maint
...

Oldukça büyük bir komut olduğundan, bir takma ad atayabilirsiniz:

git config --global alias.lol "log --decorate --oneline --graph"

Takma ad sürümünü kullanmak için:

# history of current branch :
git lol

# combined history of active branch (HEAD), develop and origin/master branches :
git lol HEAD develop origin/master

# combined history of everything in your repo :
git lol --all

“Düzenli” Git Günlüğü #

git log

tüm taahhütlerinizi yazar ve karma ile gösterecektir. Bu, taahhüt başına birden çok satırda gösterilecektir. (Taahhüt başına tek bir satır göstermek istiyorsanız, onelineing’e bakın). Günlükten çıkmak için q tuşunu kullanın.

Varsayılan olarak, herhangi bir argüman olmadan git log, bu durumda yapılan taahhütleri listeler. ters kronolojik sırada arşiv – yani en son taahhütler önce ortaya çıkar. Gördüğünüz gibi, bu komut her bir taahhüdü listeler. SHA-1 sağlama toplamı, yazarın adı ve e-posta adresi, tarih yazılı ve taahhüt mesajı. - kaynak

git log -2

Çevrimiçi günlük #

git log --oneline

tüm taahhütlerinizi karmanın yalnızca ilk kısmı ve taahhüt mesajı ile gösterecektir. ‘oneline’ bayrağının önerdiği gibi, her bir taahhüt tek bir satırda olacaktır.

Oneline seçeneği, her bir taahhüdü tek bir satıra yazdırır; bu, çok sayıda taahhüd arıyorsanız kullanışlıdır. - kaynak

Komutunuzu son n günlük kaydıyla sınırlamak istiyorsanız, basitçe bir parametre iletebilirsiniz. Örneğin, son 2 taahhüt günlüğünü listelemek istiyorsanız

git log -2 --oneline

Günlükleri Renklendir #

git log --graph --pretty=format:'%C(red)%h%Creset -%C(yellow)%d%Creset %s %C(green)(%cr) %C(yellow)<%an>%Creset'

“Biçim” seçeneği, kendi günlük çıktı biçiminizi belirlemenize olanak tanır:

parametre Detaylar
%C(renk_adı) seçenek, kendisinden sonra gelen çıktıyı renklendirir
%h veya %H sağlama karmasını kısaltır (tam karma için %H kullanın)
%Creset rengi varsayılan terminal rengine sıfırlar
%d referans isimleri
%s konu [mesaj gönder]
%cr geçerli tarihe göre taahhüt tarihi
%an yazar adı

Günlük arama #

git log -S"#define SAMPLES"

Belirli bir dizenin veya sağlanan REGEXP eşleşen dizesinin eklenmesi veya kaldırılması için arama yapar. Bu durumda, #define SAMPLES dizesinin eklenmesini/kaldırılmasını arıyoruz. Örneğin:

+#define SAMPLES  100000

veya

-#define SAMPLES  100000

git log -G"#define SAMPLES"

satırlarda belirli dizeleri içeren veya sağlanan REGEXP eşleşen dizesindeki değişiklikleri arar. Örneğin:

-#define SAMPLES  100000
+#define SAMPLES  100000000

Yazar adına göre gruplandırılmış tüm katkıları listeleyin #

“git shortlog”, “git log"u ve yazara göre grupları özetler

Herhangi bir parametre verilmezse, taahhüt eden başına yapılan tüm taahhütlerin bir listesi kronolojik sırada gösterilecektir.

$ git shortlog
Committer 1 (<number_of_commits>):
    Commit Message 1
    Commit Message 2
    ...
Committer 2 (<number_of_commits>):
    Commit Message 1
    Commit Message 2
    ...

Yalnızca taahhüt sayısını görmek ve taahhüt açıklamasını bastırmak için özet seçeneğini iletin:

-s

`-özet’

$ git shortlog -s
<number_of_commits> Committer 1
<number_of_commits> Committer 2

Çıktıyı taahhüt eden adına göre alfabetik olarak değil de taahhüt sayısına göre sıralamak için numaralı seçeneği iletin:

-n

`–numaralı'

Bir taahhüt edenin e-postasını eklemek için e-posta seçeneğini ekleyin:

-e

“–e-posta”

Taahhüt konusu dışındaki bilgileri görüntülemek istiyorsanız, özel bir biçim seçeneği de sağlanabilir:

“–biçim”

Bu, “git log"un “–format” seçeneği tarafından kabul edilen herhangi bir dize olabilir.

Bununla ilgili daha fazla bilgi için yukarıdaki Renklendirme Günlükleri bölümüne bakın.

1: https://www.wikiod.com/tr/git/tarihe-goz-atmak#Günlükleri Renklendir

Git günlüğünde taahhüt dizesi aranıyor #

Günlükte bir dize kullanarak git günlüğü aranıyor:

git log [options] --grep "search_string"

Örnek:

git log --all --grep "removed file"

all dalları içindeki all günlüklerde “kaldırılan dosya” dizesini arayacaktır.

Git 2.4+‘dan başlayarak, arama --invert-grep seçeneği kullanılarak tersine çevrilebilir.

Örnek:

git log --grep="add file" --invert-grep

‘dosya ekle’ içermeyen tüm taahhütleri gösterecektir.

Günlükleri filtrele #

git log --after '3 days ago'

Belirli tarihler de işe yarar:

git log --after 2016-05-01

Bir tarih parametresini kabul eden diğer komutlar ve bayraklarda olduğu gibi, izin verilen tarih formatı GNU tarihi tarafından desteklendiği gibidir (oldukça esnek).

“–sonra"nın diğer adı “–since"dir.

Konuşma için de bayraklar vardır: `–before’ ve ‘–until’.

Ayrıca günlükleri ‘yazar’a göre filtreleyebilirsiniz. Örneğin.

git log --author=author

Satır içi değişikliklerle günlüğe kaydet #

Satır içi değişiklikleri içeren günlüğü görmek için ‘-p’ veya ‘–patch’ seçeneklerini kullanın.

git log --patch
