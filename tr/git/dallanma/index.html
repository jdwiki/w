Sözdizimi #

  • git şubesi [--set-upstream | --parça | --no-track] [-l] [-f] <branchname> [<start-point>]
  • git şubesi (--set-upstream-to=<upstream> | -u <upstream>) [<branchname>]
  • git branch --unset-upstream [<branchname>]
  • git dalı (-m | -M) [<oldbranch>] <newbranch>
  • git dalı (-d | -D) [-r] <branchname>…​
  • git branch --edit-description [<branchname>]
  • git dalı [--color[=<ne zaman>] | --no-color] [-r | -a] [--liste] [-v [--kısaltma=<uzunluk> | --no-kısaltması]] [--sütun[=<seçenekler>] | --no-column] [(--birleştirilmiş | --no-birleştirilmiş | --içerir) [<taahhüt>]] [--sort=<anahtar>] [--points-at <object>] [<pattern >…​]

Parametreler #

parametre Ayrıntılar
-d, –delete Bir dalı silin. Dal, yukarı akış dalında veya ‘–track’ veya ‘–set-upstream’
-D --delete --force kısayolu
-m, –hareket Bir şubeyi ve ilgili reflog’u taşıyın/yeniden adlandırın
-M --move --force kısayolu
-r, –uzaktan kumandalar Uzaktan izleme dallarını listeleyin veya silin (-d ile kullanılıyorsa)
-a, –hepsi Hem uzaktan izleme şubelerini hem de yerel şubeleri listeleyin
–liste Liste modunu etkinleştirin. git branch <pattern> bir dal oluşturmaya çalışır, eşleşen dalları listelemek için git branch --list <pattern> kullanın
–set-upstream Belirtilen dal henüz mevcut değilse veya ‘–force’ verilmişse, tam olarak ‘-track’ gibi davranır. Aksi takdirde, dalın işaret ettiği yerin değiştirilmemesi dışında, –track dalı oluştururken yapacağı gibi yapılandırmayı ayarlar

Her git deposunda bir veya daha fazla dal vardır. Dal, bir dizi taahhütün ‘HEAD’ine adlandırılmış bir referanstır.

Bir git repo’nun bir current dalı vardır (‘git branch’ komutuyla yazdırılan dal adları listesinde ‘*’ ile gösterilir), ‘git commit’ komutuyla yeni bir taahhüt oluşturduğunuzda, yeni taahhüdünüz ‘olur. HEAD` geçerli dalın ve önceki HEAD, yeni taahhüdün ebeveyni olur.

Yeni şubeye bir şey taahhüt edilene kadar yeni bir şube, oluşturulduğu şubeyle aynı ‘HEAD’e sahip olacaktır.

Yeni şubeler oluşturma ve kontrol etme #

Mevcut dalda kalırken yeni bir dal oluşturmak için şunu kullanın:

git branch <name>

Genel olarak, şube adı boşluk içermemelidir ve burada listelenen diğer spesifikasyonlara tabidir. Mevcut bir şubeye geçmek için:

git checkout <name>

Yeni bir şube oluşturmak ve buna geçmek için:

git checkout -b <name>

Geçerli dalın (HEAD olarak da bilinir) son taahhüdü dışında bir noktada bir dal oluşturmak için şu komutlardan birini kullanın:

git branch <name> [<start-point>]
git checkout -b <name> [<start-point>]

“”, git tarafından bilinen herhangi bir revizyon olabilir (ör. başka bir dal adı, SHA kaydı veya HEAD veya etiket adı):

git checkout -b <name> some_other_branch
git checkout -b <name> af295
git checkout -b <name> HEAD~5
git checkout -b <name> v1.0.5

Bir uzak daldan bir dal oluşturmak için (varsayılan “<remote_name>”, Origin’dir):

git branch <name> <remote_name>/<branch_name>
git checkout -b <name> <remote_name>/<branch_name>

Belirli bir dal adı yalnızca bir uzaktan kumandada bulunursa,

git checkout -b <branch_name>

hangi eşdeğerdir

git checkout -b <branch_name> <remote_name>/<branch_name>

Bazen son taahhütlerinizin birkaçını yeni bir şubeye taşımanız gerekebilir. Bu, şu şekilde dallara ayrılma ve “geri alma” ile başarılabilir:

git branch <new_name>
git reset --hard HEAD~2 # Go back 2 commits, you will lose uncommitted work.
git checkout <new_name>

İşte bu tekniğin açıklayıcı bir açıklaması:

 Initial state       After git branch <new_name>    After git reset --hard HEAD~2
                             newBranch                        newBranch
                                 ↓                                ↓
A-B-C-D-E (HEAD)         A-B-C-D-E (HEAD)                 A-B-C-D-E (HEAD)
        ↑                        ↑                            ↑
      master                   master                       master

Dalları listeleme #

Git, dalları listelemek için birden çok komut sağlar. Tüm komutlar, komut satırına hangi seçeneklerin konulduğuna bağlı olarak belirli dalların bir listesini sağlayan “git branch” işlevini kullanır. Git, mümkünse, şu anda seçili olan dalı yanında bir yıldızla belirtecektir.

Gol Komut
Yerel şubeleri listele git dalı
Yerel şubeleri listele ayrıntılı git dalı -v
Uzak ve yerel şubeleri listeleyin git şubesi -a VEYA git şubesi --all
Uzak ve yerel şubelerin listesi (ayrıntılı) git şubesi -av
Uzak şubeleri listele git dalı -r
En son taahhütle uzak şubeleri listeleyin git şubesi -rv
Birleştirilmiş şubeleri listele git dalı --birleştirilmiş
Birleştirilmemiş dalları listele git dalı --no-birleştirilmiş
Taahhüt içeren dalları listeleyin git dalı --içerir [<taahhüt>]

Notlar:

  • “-v"ye ek bir “v” ekleme, ör. $ git branch -avv veya $ git branch -vv yukarı akış dalının adını da yazdırır.
  • Kırmızı renkle gösterilen dallar uzak dallardır.

Uzak bir dalı sil #

‘Origin’ uzak deposundaki bir dalı silmek için Git sürüm 1.5.0 ve daha yenisini kullanabilirsiniz.

git push origin :<branchName>

ve Git 1.7.0 sürümünden itibaren, kullanarak uzak bir dalı silebilirsiniz.

git push origin --delete <branchName>

Yerel bir uzaktan izleme dalını silmek için:

git branch --delete --remotes <remote>/<branch>
git branch -dr <remote>/<branch> # Shorter

git fetch <remote> --prune # Delete multiple obsolete tracking branches
git fetch <remote> -p      # Shorter

Bir şubeyi yerel olarak silmek için. Birleştirilmemiş değişiklikler varsa bunun dalı silmeyeceğini unutmayın:

git branch -d <branchName>

Birleştirilmemiş değişiklikleri olsa bile bir dalı silmek için:

git branch -D <branchName>

Önceki şubeye hızlı geçiş #

kullanarak hızlı bir şekilde önceki şubeye geçebilirsiniz.

git checkout -

Uzak bir dalı izleyen yeni bir dala göz atın #

Uzak dalın ‘başlangıcı/özelliği’ni izleyen yeni bir dal ‘özelliği’ yaratmanın üç yolu vardır:

  • git checkout --track -b özellik kaynağı/özelliği,
  • git checkout -t kaynak/özellik,
  • “git checkout özelliği” - yerel bir “özellik” dalı olmadığını ve “özellik” dalı olan yalnızca bir uzaktan kumanda olduğunu varsayarsak.

Uzak dalı izlemek üzere yukarı akışı ayarlamak için - şunu yazın:

  • git branch --set-upstream-to=<remote>/<branch> <branch>
  • git branch -u <remote>/<branch> <branch>

nerede:

  • “” şunlar olabilir: “origin”, “develop” veya kullanıcı tarafından oluşturulan,
  • <branch>, kullanıcının uzaktan izlenecek dalıdır.

Yerel şubelerinizin hangi uzak şubeleri izlediğini doğrulamak için:

  • git dalı -vv

Bir şubeyi yerel olarak sil #

$ git branch -d dev

‘dev’ adlı dalı siler eğer değişiklikleri başka bir dalla birleştirilir ve kaybolmaz. “dev” dalı, henüz birleştirilmemiş ve kaybolacak değişiklikler içeriyorsa, “git branch -d” başarısız olur:

$ git branch -d dev
error: The branch 'dev' is not fully merged.
If you are sure you want to delete it, run 'git branch -D dev'.

Uyarı mesajına göre, “-D” bayrağını kullanarak dalı silmeye (ve o daldaki birleştirilmemiş tüm değişiklikleri kaybetmeye) zorlayabilirsiniz:

$ git branch -D dev

Yetim bir şube oluşturun (yani, ebeveyn taahhüdü olmayan şube) #

git checkout --orphan new-orphan-branch

Bu yeni dalda yapılan ilk taahhüdün ebeveyni olmayacak ve diğer tüm dallardan ve taahhütlerden tamamen kopuk yeni bir tarihin kökü olacaktır.

kaynak

Şubeyi yeniden adlandırın #

Kontrol ettiğiniz şubeyi yeniden adlandırın:

git branch -m new_branch_name

Başka bir dalı yeniden adlandırın:

git branch -m branch_you_want_to_rename new_branch_name

Şubeyi uzaktan kumandaya itin #

Yerel şubenizde yapılan taahhütleri uzak bir havuza göndermek için kullanın.

git push komutu iki argüman alır:

  • Uzak bir ad, örneğin “origin”
  • Örneğin bir şube adı, ‘usta’

Örneğin:

git push  <REMOTENAME> <BRANCHNAME>

Örnek olarak, yerel değişikliklerinizi çevrimiçi deponuza göndermek için genellikle “git push Origin master” komutunu çalıştırırsınız.

-u (--set-upstream kısaltması) kullanmak, push sırasında izleme bilgilerini ayarlayacaktır.

git push -u <REMOTENAME> <BRANCHNAME>

Varsayılan olarak, “git” yerel dalı aynı ada sahip uzak bir dala iter. Örneğin, “yeni-özellik” adında bir yereliniz varsa, yerel şubeye basarsanız, uzak bir şube “yeni-özellik” de yaratacaktır. Uzak dal için farklı bir ad kullanmak istiyorsanız, uzak adı yerel dal adından sonra : ile ayırarak ekleyin:

git push <REMOTENAME> <LOCALBRANCHNAME>:<REMOTEBRANCHNAME>

Şubelerde arama #

Belirli bir taahhüt veya etiket içeren yerel şubeleri listelemek için

git branch --contains <commit>

Belirli bir taahhüt veya etiket içeren yerel ve uzak dalları listelemek için

git branch -a --contains <commit>

Geçerli dalı HEAD’i isteğe bağlı bir işleme taşıyın #

Bir dal, yalnızca bir taahhüdün bir göstergesidir, böylece onu özgürce hareket ettirebilirsiniz. Şubenin ‘aabbcc’ taahhüdüne atıfta bulunmasını sağlamak için komutu verin

git reset --hard aabbcc

Lütfen bunun şubenizin mevcut taahhüdünün ve dolayısıyla tüm geçmişinin üzerine yazacağını unutmayın. Bu komutu vererek bazı işlerinizi kaybedebilirsiniz. Bu durumda, kaybedilen taahhütleri kurtarmak için reflog öğesini kullanabilirsiniz. Bu komutu mevcut şubeniz yerine yeni bir şubede gerçekleştirmeniz tavsiye edilebilir.

Bununla birlikte, bu komut, yeniden temellendirme yaparken veya bu tür diğer büyük geçmiş değişiklikleri yaparken özellikle yararlı olabilir.

Geçerli çalışma dizinindeki tek bir dosyanın üzerine başka bir daldan aynı dosyanın üzerine yaz #

Teslim alınan dosya, bu dosyada yaptığınız değişikliklerin üzerine yazılacaktır.

Bu komut, file.example (path/to/ dizininde bulunur) dosyasını kontrol edecek ve bu dosyada yapmış olabileceğiniz tüm değişikliklerin üzerine yazacaktır.

git checkout some-branch path/to/file

some-branch git tarafından bilinen bir ağaç benzeri herhangi bir şey olabilir (bkz. Revizyon Seçimi ve gitrevisions daha fazla ayrıntı için)

Dosyanız bir dosyayla karıştırılabilirse, yolun önüne --' eklemelisiniz (aksi takdirde isteğe bağlı). –` işaretinden sonra başka seçenek sağlanamaz.

git checkout some-branch -- some-file

İkinci “bir dosya” bu örnekte bir dosyadır.
