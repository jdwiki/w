Sözdizimi #

  • git log [<options>] [<revizyon aralığı>] [[–] <yol>]
  • git günlüğü –pretty=short | git kısa günlüğü [<seçenekler>]
  • git shortlog [<options>] [<revizyon aralığı>] [[–] <yol>]

Parametreler #

parametre Ayrıntılar
-n, --numaralı Çıktıyı alfabetik sıra yerine yazar başına taahhüt sayısına göre sıralayın
-s, --özet Yalnızca bir taahhüt sayısı özeti sağlayın
-e, --eposta Her yazarın e-posta adresini göster
--format[=<format>] Taahhüt konusu yerine, her bir taahhüdü açıklamak için başka bilgiler kullanın. <format>, “git log"un “–format” seçeneği tarafından kabul edilen herhangi bir dize olabilir.
-w[<genişlik>[,<indent1>[,<indent2>]]] Her satırı “genişlik"te sararak çıktıyı satır sarın. Her girdinin ilk satırı, “girint1” boşluk sayısıyla girintilenir ve sonraki satırlar “indent2” boşluklarıyla girintilenir.
<revizyon aralığı> Yalnızca belirtilen revizyon aralığındaki taahhütleri göster. Geçerli taahhüde kadar tüm geçmişi varsayılan olarak ayarlayın.
[--] <yol> Yalnızca “yol” ile eşleşen dosyaların nasıl ortaya çıktığını açıklayan taahhütleri göster. Seçeneklerden veya revizyon aralığından ayırmak için yolların önüne “–” eklenmesi gerekebilir.

Geliştirici başına kod satırı #

git ls-tree -r HEAD | sed -Ee 's/^.{53}//' | \
while read filename; do file "$filename"; done | \
grep -E ': .*text' | sed -E -e 's/: .*//' | \
while read filename; do git blame --line-porcelain "$filename"; done | \
sed -n 's/^author //p' | \
sort | uniq -c | sort -rn

Her şubenin ve son revizyon tarihinin listelenmesi #

for k in `git branch -a | sed s/^..//`; do echo -e `git log -1 --pretty=format:"%Cgreen%ci %Cblue%cr%Creset" $k --`\\t"$k";done | sort

Geliştirici başına taahhüt #

Git shortlog, git log çıktılarını özetlemek ve taahhütleri yazara göre gruplamak için kullanılır.

Varsayılan olarak, tüm taahhüt mesajları gösterilir, ancak --summary veya -s argümanları mesajları atlar ve toplam taahhüt sayılarıyla birlikte yazarların bir listesini verir.

“–numaralı” veya “-n”, sıralamayı alfabetikten (yazar tarafından artan) azalan tamamlama sayısına değiştirir.

git shortlog -sn        #Names and Number of commits

git shortlog -sne       #Names along with their email ids and the Number of commits

veya

git log --pretty=format:%ae \
| gawk -- '{ ++c[$0]; } END { for(cc in c) printf "%5d %s\n",c[cc],cc; }'

Not: Aynı kişi tarafından yapılan taahhütler, adlarının ve/veya e-posta adreslerinin farklı yazıldığı durumlarda birlikte gruplandırılamaz. Örneğin, ‘John Doe’ ve ‘Johnny Doe’ listede ayrı ayrı görünecektir. Bunu çözmek için .mailmap özelliğine bakın.

Tarih başına taahhüt #

git log --pretty=format:"%ai" | awk '{print " : "$1}' | sort -r | uniq -c

Bir şubedeki toplam taahhüt sayısı #

git log  --pretty=oneline |wc -l

Tüm taahhütleri güzel biçimde listeleyin #

git log --pretty=format:"%Cgreen%ci %Cblue%cn  %Cgreen%cr%Creset %s"

Bu, tarih, kullanıcı ve taahhüt mesajı ile tüm taahhütlere (satır başına 1) güzel bir genel bakış sağlayacaktır.

“–pretty” seçeneği, her biri “%” ile başlayan birçok yer tutucuya sahiptir. Tüm seçenekler burada bulunabilir

Bilgisayardaki Tüm Yerel Git Depolarını Bulun #

Tüm git deposu konumlarını listelemek için aşağıdakileri çalıştırabilirsiniz.

find $HOME -type d -name ".git"

“Bulunduğunuzu” varsayarsak, bu çok daha hızlı olmalıdır:

locate .git |grep git$

“gnu bul” veya “mlocate” varsa, bu yalnızca git dizinlerini seçecektir:

locate -ber \\.git$

Yazar başına toplam taahhüt sayısını göster #

Her geliştiricinin veya katkıda bulunanın bir depoda yaptığı toplam taahhüt sayısını elde etmek için, basitçe “git kısa günlüğü"nü kullanabilirsiniz:

git shortlog -s

bu, yazar adlarını ve her biri için taahhüt sayısını sağlar.

Ek olarak, sonuçların tüm dallarda hesaplanmasını istiyorsanız, komuta “–all” bayrağını ekleyin:

git shortlog -s --all
