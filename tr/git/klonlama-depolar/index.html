Sığ Klon #

Büyük bir havuzun klonlanması (birden fazla yıllık geçmişe sahip bir proje gibi) uzun zaman alabilir veya aktarılacak veri miktarı nedeniyle başarısız olabilir. Tam geçmişe sahip olmanızın gerekmediği durumlarda, sığ bir klonlama yapabilirsiniz:

git clone [repo_url] --depth 1

Yukarıdaki komut, uzak depodan yalnızca son taahhüdü getirecektir.

Sığ bir depodaki birleştirmeleri çözemeyebileceğinizi unutmayın. Birleştirmeleri çözmek için geri izlemeniz gerekecek en az sayıda taahhüt almak genellikle iyi bir fikirdir. Örneğin, bunun yerine son 50 taahhüdü almak için:

git clone [repo_url] --depth 50

Daha sonra gerekirse deponun geri kalanını getirebilirsiniz:

git fetch --unshallow     # equivalent of git fetch -–depth=2147483647
                          # fetches the rest of the repository
son sürüm 
git fetch --depth=1000    # fetch the last 1000 commits
son sürüm

Normal Klon #

Tam geçmiş ve tüm dallar dahil olmak üzere tüm depoyu indirmek için şunu yazın:

git clone <url>

Yukarıdaki örnek, onu depo adıyla aynı ada sahip bir dizine yerleştirecektir.

Depoyu indirmek ve belirli bir dizine kaydetmek için şunu yazın:

git clone <url> [directory]

Daha fazla ayrıntı için [Depoyu klonla](https://www.wikiod.com/tr/git/giti-kullanmaya-baslama#Bir depoyu klonla) sayfasını ziyaret edin.

Belirli bir dalı klonla #

Bir havuzun belirli bir dalını klonlamak için, depo url’sinden önce “–branch ” yazın:

git clone --branch <branch name> <url> [directory]

“–branch” için kısayol seçeneğini kullanmak için “-b” yazın. Bu komut tüm depoyu indirir ve <branch name> öğesini kontrol eder.

Disk alanından tasarruf etmek için, yalnızca tek bir şubeye giden geçmişi aşağıdakilerle klonlayabilirsiniz:

git clone --branch <branch_name> --single-branch <url> [directory]

Komuta “–single-branch” eklenmezse, tüm dalların geçmişi “[dizin]” içine kopyalanacaktır. Bu, büyük depolarda sorun olabilir.

Daha sonra “–single-branch” bayrağını geri almak ve havuzun geri kalanını kullanmak için komutu almak için:

git config remote.origin.fetch "+refs/heads/*:refs/remotes/origin/*"
git fetch origin

Özyinelemeli klonlama #

git clone <url> --recursive

Depoyu klonlar ve ayrıca tüm alt modülleri klonlar. Alt modüllerin kendileri ek alt modüller içeriyorsa Git bunları da klonlayacaktır.

son sürüm

Proxy kullanarak klonlama #

Git ile dosyaları bir proxy altında indirmeniz gerekiyorsa, proxy sunucusunu sistem genelinde ayarlamak yeterli olamaz. Aşağıdakileri de deneyebilirsiniz:

git config --global http.proxy http://<proxy-server>:<port>/
