Ezmek nedir? #

Squashing, birden fazla taahhüt alma ve bunları ilk taahhütlerdeki tüm değişiklikleri kapsayan tek bir taahhütte birleştirme sürecidir.

Ezme ve Uzak Dallar #

Uzak bir dalı izleyen bir dalda squash işlemi yaparken özellikle dikkat edin; Zaten uzak bir şubeye gönderilmiş bir taahhüdü ezerseniz, iki dal birbirinden ayrılacak ve bu değişiklikleri uzak şubeye zorlamak için git push -f kullanmanız gerekecek. Bunun, o uzaktaki şubeyi izleyen başkaları için sorunlara neden olabileceğini unutmayın, bu nedenle, zorla sıkıştırılmış taahhütleri herkese açık veya paylaşılan depolara zorlarken dikkatli olunmalıdır.

Proje GitHub’da barındırılıyorsa, “Ayarlar” - “Dallar” - “Korumalı Dallar"a ekleyerek “ana” gibi bazı dallarda “zorla itme korumasını” etkinleştirebilirsiniz.

Squash Son İşlemleri Yeniden Temellendirme Olmadan #

Önceki ‘x’ taahhütlerini tek bir taahhütte ezmek istiyorsanız, aşağıdaki komutları kullanabilirsiniz:

git reset --soft HEAD~x
git commit

‘x’ yerine, ezilmiş işleme dahil edilmesini istediğiniz önceki taahhütlerin sayısı.

Bunun, yazarı, mesajı ve tarihi de dahil olmak üzere önceki ‘x’ taahhütleri hakkındaki bilgileri unutarak yeni bir taahhüt oluşturacağını unutmayın. Muhtemelen mevcut bir taahhüt mesajını önce kopyalayıp yapıştırmak istersiniz.

Yeniden Temellendirme Sırasında Taahhüt Etmek #

Taahhütler, bir “git rebase” sırasında ezilebilir. Taahhütleri bu şekilde ezmeye çalışmadan önce yeniden temellendirmeyi anlamanız önerilir.

  1. Hangi taahhütten yeniden temel almak istediğinizi belirleyin ve taahhüt karmasını not edin.
  2. git rebase -i [commit hash] komutunu çalıştırın.

Alternatif olarak, en son taahhüdü ve en sondan önceki 4 taahhüdü görüntülemek için kesinleştirme karması yerine ‘HEAD~4’ yazabilirsiniz. 3. Bu komutu çalıştırırken açılan düzenleyicide, hangi taahhütleri ezmek istediğinizi belirleyin. Bu satırların başındaki “pick"i “squash” ile değiştirerek önceki işleme dahil edin. 4. Hangi taahhütleri ezmek istediğinizi seçtikten sonra, bir taahhüt mesajı yazmanız istenecektir.

Günlük kaydı, nerede yeniden temel alınacağını belirlemek için taahhüt eder

> git log --oneline
612f2f7 This commit should not be squashed
d84b05d This commit should be squashed
ac60234 Yet another commit
36d15de Rebase from here
17692d1 Did some more stuff
e647334 Another Commit
2e30df6 Initial commit

> git rebase -i 36d15de

Bu noktada, taahhütlerle ne yapmak istediğinizi tanımlayabileceğiniz editör seçiminiz açılır. Git, yorumlarda yardım sağlar. Olduğu gibi bırakırsanız, hiçbir şey olmayacak çünkü her taahhüt tutulacak ve siparişleri yeniden başlatmadan öncekiyle aynı olacaktır. Bu örnekte aşağıdaki komutları uyguluyoruz:

pick ac60234 Yet another commit
squash d84b05d This commit should be squashed
pick 612f2f7 This commit should not be squashed

# Rebase 36d15de..612f2f7 onto 36d15de (3 command(s))
#
# Commands:
# p, pick = use commit
# r, reword = use commit, but edit the commit message
# e, edit = use commit, but stop for amending
# s, squash = use commit, but meld into previous commit
# f, fixup = like "squash", but discard this commit's log message
# x, exec = run command (the rest of the line) using shell
#
# These lines can be re-ordered; they are executed from top to bottom.
#
# If you remove a line here THAT COMMIT WILL BE LOST.
#
# However, if you remove everything, the rebase will be aborted.
#
# Note that empty commits are commented out

taahhüt mesajı yazdıktan sonra Git günlüğü

> git log --oneline
77393eb This commit should not be squashed
e090a8c Yet another commit
36d15de Rebase from here
17692d1 Did some more stuff
e647334 Another Commit
2e30df6 Initial commit

Birleştirme Sırasında Taahhüdü Ezme #

Bir dal tarafından yapılan değişiklikleri tek bir taahhütte ezmek için git merge --squash kullanabilirsiniz. Gerçek bir taahhüt oluşturulmayacak.

git merge --squash <branch>
git commit

Bu, aşağı yukarı “git reset” kullanımına eşdeğerdir, ancak dahil edilen değişikliklerin sembolik bir adı olduğunda daha uygundur. Karşılaştırmak:

git checkout <branch>
git reset --soft $(git merge-base master <branch>)
git commit

Autosquash: Yeniden düzenleme sırasında ezmek istediğiniz kodu işlemek #

Aşağıdaki geçmiş göz önüne alındığında, ‘bbb2222 A ikinci taahhüt’ taahhüdüne ezmek istediğiniz bir değişiklik yaptığınızı hayal edin:

$ git log --oneline --decorate
ccc3333 (HEAD -> master) A third commit
bbb2222 A second commit
aaa1111 A first commit
9999999 Initial commit

Değişikliklerinizi yaptıktan sonra, bunları her zamanki gibi dizine ekleyebilir, ardından ‘–fixup’ argümanını kullanarak, ezmek istediğiniz taahhüdü referans alabilirsiniz:

$ git add .
$ git commit --fixup bbb2222
[my-feature-branch ddd4444] fixup! A second commit

Bu, Git’in etkileşimli bir yeniden başlatma sırasında tanıyabileceği bir taahhüt mesajıyla yeni bir taahhüt oluşturacaktır:

$ git log --oneline --decorate
ddd4444 (HEAD -> master) fixup! A second commit
ccc3333 A third commit
bbb2222 A second commit
aaa1111 A first commit
9999999 Initial commit

Ardından, “–autosquash” argümanıyla etkileşimli bir yeniden temel oluşturun:

$ git rebase --autosquash --interactive HEAD~4

Git, ‘commit –fixup’ ile yaptığınız taahhüdü doğru konuma getirmenizi önerecektir:

pick aaa1111 A first commit
pick bbb2222 A second commit
fixup ddd4444 fixup! A second commit
pick ccc3333 A third commit

Her yeniden düzenlemede --autosquash yazmak zorunda kalmamak için bu seçeneği varsayılan olarak etkinleştirebilirsiniz:

$ git config --global rebase.autosquash true

Otomatik ezme ve düzeltmeler #

Değişiklikler yapılırken, taahhüdün gelecekte başka bir taahhüde ezileceğini belirtmek mümkündür ve bu böyle yapılabilir,

git commit --squash=[bu taahhüdün ezileceği taahhüt karmasını taahhüt et]

Düzeltmeye alternatif olarak `–fixup=[karma işlemi yap]’ da kullanılabilir.

Ayrıca, taahhüt karması yerine taahhüt mesajındaki kelimeleri kullanmak da mümkündür, bunun gibi,

git commit --squash :/şeyler

burada ‘şeyler’ kelimesiyle yapılan en son taahhüt kullanılacaktır.

Bu taahhütlerin mesajı ‘düzeltme!’ veya ‘squash!’ ile başlar ve ardından bu taahhütlerin ezileceği taahhüt mesajının geri kalanı gelir.

Yeniden temel alırken, autosquash/düzeltme özelliğini kullanmak için –autosquash bayrağı kullanılmalıdır.
