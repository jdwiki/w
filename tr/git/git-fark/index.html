Sözdizimi #

  • git diff [seçenekler] [<taahhüt>] [--] [<yol>…​]
  • git diff [seçenekler] --cached [<commit>] [--] [<path>…​]
  • git diff [seçenekler] <commit> <commit> [--] [<path>…​]
  • git diff [seçenekler] <blob> <blob>
  • git diff [seçenekler] [--index yok] [--] <yol> <yol>

Parametreler #

parametre Ayrıntılar
-p, -u, –patch Yama oluştur
-s, –no-yama Fark çıktısını bastır. Varsayılan olarak yamayı gösteren ‘git show’ gibi komutlar için veya ‘–patch’
–ham Farkı ham biçimde oluşturun
–diff-algoritma= Bir fark algoritması seçin. Varyantlar şu şekildedir: “myers”, “minimal”, “sabır”, “histogram”
–özet Oluşturmalar, yeniden adlandırmalar ve mod değişiklikleri gibi genişletilmiş başlık bilgilerinin özet bir özetini çıktılayın
–yalnızca isim Yalnızca değiştirilen dosyaların adlarını göster
–isim-durumu Değiştirilen dosyaların adlarını ve durumlarını göster En yaygın durumlar M (Değiştirildi), A (Eklendi) ve D (Silindi)
–kontrol Değişiklikler çakışma işaretleri veya boşluk hataları ortaya çıkarırsa uyar. Boşluk hataları olarak kabul edilenler, “core.whitespace” yapılandırması tarafından kontrol edilir. Varsayılan olarak, sondaki boşluklar (yalnızca boşluklardan oluşan satırlar dahil) ve satırın ilk girintisinin içinde bir sekme karakterinin hemen ardından gelen boşluk karakteri boşluk hataları olarak kabul edilir. Sorun bulunursa sıfır olmayan durumla çıkar. –exit-code
–tam-indeks Yama biçimi çıktısı oluştururken, ilk bir avuç karakter yerine, “indeks” satırında görüntü öncesi ve sonrası blob nesne adlarının tamamını gösterin
–ikili “–full-index"e ek olarak, “git application”
-a, –metin Tüm dosyaları metin olarak ele alın.
–renk Renk modunu ayarlayın; yani, bir farkı daha az değere aktarmak ve git’in rengini korumak istiyorsanız --color=always kullanın

Çalışma dalındaki farklılıkları göster #

git diff

Bu, mevcut daldaki unstaged değişikliklerini ondan önceki taahhütten gösterecektir. Yalnızca dizine göre değişiklikleri gösterecektir, yani bir sonraki işleme ne ekleyebileceğinizi, ancak ne ekleyemeyeceğinizi gösterir. Bu değişiklikleri eklemek (aşama) için git add kullanabilirsiniz.

Bir dosya aşamalıysa, ancak oluşturulduktan sonra değiştirilmişse, “git diff”, geçerli dosya ile aşamalı sürüm arasındaki farkları gösterecektir.

İki taahhüt arasındaki değişiklikleri göster #

git diff 1234abc..6789def    # old   new

Örn: Son 3 taahhütte yapılan değişiklikleri göster:

git diff @~3..@    # HEAD -3   HEAD

Not: iki nokta (..) isteğe bağlıdır, ancak netlik sağlar.

Bu, ağaçta nerede olduklarına bakılmaksızın taahhütler arasındaki metinsel farkı gösterecektir.

Aşamalı dosyalar için farklılıkları göster #

git diff --staged

Bu, önceki taahhüt ile şu anda sahnelenen dosyalar arasındaki değişiklikleri gösterecektir.

NOT: Aynı şeyi gerçekleştirmek için aşağıdaki komutları da kullanabilirsiniz:

git diff --cached

Hangisi ‘–stage’ ile eşanlamlıdır veya

git status -v

Bu, ‘status’ komutunun ayrıntılı ayarlarını tetikleyecektir.

Dalları karşılaştırma #

yeni ucu ile orijinal ucu arasındaki değişiklikleri göster:

git diff original new     # equivalent to original..new

orijinal‘den dallandığından beri yeni üzerindeki tüm değişiklikleri göster:

git diff original...new     # equivalent to $(git merge-base original new)..new

gibi sadece bir parametre kullanarak

git diff orijinal

eşdeğerdir

git diff orijinal..HEAD

Hem aşamalı hem de aşamalı olmayan değişiklikleri göster #

Tüm aşamalı ve aşamalı olmayan değişiklikleri göstermek için şunu kullanın:

git diff HEAD

NOT: Aşağıdaki komutu da kullanabilirsiniz:

git status -vv

Fark, ikincisinin çıktısının aslında size hangi değişikliklerin taahhüt için hazırlandığını ve hangilerinin yapılmadığını söyleyecek olmasıdır.

Belirli bir dosya veya dizin için farklılıkları göster #

git diff myfile.txt

Belirtilen dosyanın (‘myfile.txt’) önceki kaydı ile henüz hazırlanmamış yerel olarak değiştirilmiş sürüm arasındaki değişiklikleri gösterir.

Bu aynı zamanda dizinler için de çalışır:

git diff documentation

Yukarıdakiler, belirtilen dizindeki tüm dosyaların (documentation/) önceki kaydı ile bu dosyaların henüz hazırlanmamış yerel olarak değiştirilmiş sürümleri arasındaki değişiklikleri gösterir.

Belirli bir taahhütte bir dosyanın bazı sürümleri ile yerel ‘HEAD’ sürümü arasındaki farkı göstermek için, karşılaştırmak istediğiniz taahhüdü belirtebilirsiniz:

git diff 27fa75e myfile.txt

Veya iki ayrı taahhüt arasındaki sürümü görmek istiyorsanız:

git diff 27fa75e ada9b57 myfile.txt

“ada9b57” hash tarafından belirtilen sürüm ile “my_branchname” dalındaki en son taahhüt arasındaki farkı sadece “my_changed_directory/” adlı göreli dizin için göstermek için şunu yapabilirsiniz:

git diff ada9b57 my_branchname my_changed_directory/

Uzun satırlar için kelime farkını görüntüleme #

git diff [HEAD|--staged...] --word-diff

Bu, değiştirilen satırları göstermek yerine, satırlar içindeki farklılıkları gösterecektir. Örneğin, yerine:

-Hello world
+Hello world!

Tüm satırın değiştirilmiş olarak işaretlendiği yerde, “word-diff” çıktıyı şu şekilde değiştirir:

Hello [-world-]{+world!+}

“–word-diff=color” veya “–color-words” belirterek “[-”, “-]”, “{+”, “+}” işaretlerini atlayabilirsiniz. Bu, farkı belirtmek için yalnızca renk kodlamasını kullanır:

Çalışma dizinindeki tüm değişiklikleri görmek için meld kullanma #

git difftool -t meld --dir-diff

çalışma dizini değişikliklerini gösterecektir. Alternatif olarak,

git difftool -t meld --dir-diff  [COMMIT_A] [COMMIT_B]

2 belirli taahhüt arasındaki farkları gösterecektir.

Mevcut sürüm ile son sürüm arasındaki farkları göster #

git diff HEAD^ HEAD

Bu, önceki taahhüt ile mevcut taahhüt arasındaki değişiklikleri gösterecektir.

Yama uyumlu bir fark üretin #

Bazen yama kullanarak uygulamak için sadece bir farka ihtiyacınız vardır. Normal “git –diff” çalışmıyor. Bunun yerine şunu deneyin:

git diff --no-prefix > some_file.patch

O zaman başka bir yerde onu tersine çevirebilirsin:

patch -p0 < some_file.patch

iki taahhüt veya dal arasındaki fark #

İki dal arasındaki farkı görmek için

git diff <branch1>..<branch2>

İki dal arasındaki farkı görmek için

git diff <commitId1>..<commitId2>

Mevcut dal ile farkı görüntülemek için

git diff <branch/commitId>

Değişikliklerin özetini görüntülemek için

git diff --stat <branch/commitId>

Belirli bir taahhütten sonra değişen dosyaları görüntülemek için

git diff --name-only <commitId>

Bir daldan farklı dosyaları görüntülemek için

git diff --name-only <branchName>

Belirli bir işlemden sonra bir klasörde değişen dosyaları görüntülemek için

git diff --name-only <commitId> <folder_path>

Diff UTF-16 kodlu metin ve ikili plist dosyaları #

Git’in bu dosyaları nasıl dağıtması gerektiğini belirterek UTF-16 kodlu dosyaları dağıtabilirsiniz (yerelleştirme dizeleri dosyası os iOS ve macOS örneklerdir).

Aşağıdakileri ~/.gitconfig dosyanıza ekleyin.

[diff "utf16"]
textconv = "iconv -f utf-16 -t utf-8"

‘iconv’, [farklı kodlamaları dönüştürmek] için bir programdır (http://linux.die.net/man/1/iconv).

Ardından, kullanmak istediğiniz deponun kökünde bir .gitattributes dosyasını düzenleyin veya oluşturun. Veya sadece ~/.gitattributes dosyasını düzenleyin.

*.strings diff=utf16

Bu, git diffs’den önce “.strings” ile biten tüm dosyaları dönüştürür.

Metne dönüştürülebilen diğer dosyalar için de benzer şeyler yapabilirsiniz.

İkili plist dosyaları için .gitconfig dosyasını düzenlersiniz

[diff "plist"]
textconv = plutil -convert xml1 -o -

ve “.gitattributes”

*.plist diff=plist

Ortak ata dahil olmak üzere üç yönlü bir birleştirmeyi görüntüleme #

git config --global merge.conflictstyle diff3

İki dosyayı gösteren, çakışan bölümlerde normal biçim yerine “diff3” stilini varsayılan olarak ayarlar:

<<<<<<< HEAD
left
=======
right
>>>>>>> master

orijinal metni (ortak atadan gelen) içeren ek bir bölüm içerecektir:

<<<<<<< HEAD
first
second
||||||| 
first
=======
last
>>>>>>> master

Bu biçim, birleştirme-çatışmasını anlamayı kolaylaştırır, yani. bu durumda yerel olarak “ikinci” eklendi, uzaktan kumanda “ilk” olarak “son” olarak değiştirildi ve şu şekilde çözüldü:

last
second

Aynı çözünürlük, varsayılanı kullanarak çok daha zor olurdu:

<<<<<<< HEAD
first
second
=======
last
>>>>>>> master

git diff branch1..branch2
