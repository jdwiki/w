İlk deponuzu oluşturun, ardından dosyaları ekleyin ve kaydedin #

Komut satırında önce Git’in kurulu olduğunu doğrulayın:

Tüm işletim sistemlerinde:

git --version

UNIX benzeri işletim sistemlerinde:

which git

Hiçbir şey döndürülmezse veya komut tanınmazsa, yükleyiciyi indirip çalıştırarak Git’i sisteminize yüklemeniz gerekebilir. Son derece net ve kolay kurulum talimatları için Git ana sayfasına bakın.

Git’i kurduktan sonra, [kullanıcı adınızı ve e-posta adresinizi yapılandırın][2]. Bunu bir taahhütte bulunmadan önce yapın.

Git yüklendikten sonra, sürüm kontrolü altına yerleştirmek istediğiniz dizine gidin ve boş bir Git deposu oluşturun:

git init

Bu, Git’in çalışması için gereken tesisatı içeren gizli bir “.git” klasörü oluşturur.

Ardından Git’in yeni deponuza hangi dosyaları ekleyeceğini kontrol edin; bu adım özel bakıma değer:

git status

Ortaya çıkan dosya listesini gözden geçirin; Git’e hangi dosyaların sürüm kontrolüne yerleştirileceğini söyleyebilirsiniz (parolalar gibi gizli bilgiler içeren veya depoyu karmaşık hale getiren dosyalar eklemekten kaçının):

git add <file/directory name #1> <file/directory name #2> < ... >

Listedeki tüm dosyaların havuza erişimi olan herkesle paylaşılması gerekiyorsa, tek bir komut geçerli dizininize ve alt dizinlerine her şeyi ekleyecektir:

git add .

Bu, sürüm kontrolüne eklenecek tüm dosyaları [“aşamalı”][3] ve ilk taahhüdünüzde işlenmeye hazırlayacaktır.

Asla sürüm kontrolü altında olmasını istemediğiniz dosyalar için, daha önce .gitignore adlı bir dosya oluşturun ve doldurun “ekle” komutunu çalıştırarak.

Eklenen tüm dosyaları bir taahhüt mesajı ile birlikte taahhüt edin:

git commit -m "Initial commit"

Bu, verilen mesajla yeni bir [taahhüt][4] oluşturur. Taahhüt, tüm projenizin bir kaydı veya anlık görüntüsü gibidir. Şimdi onu [push][5] veya uzak bir havuza yükleyebilirsiniz ve daha sonra gerekirse ona geri dönebilirsiniz. -m parametresini atlarsanız, varsayılan düzenleyiciniz açılır ve taahhüt mesajını orada düzenleyebilir ve kaydedebilirsiniz.

Uzaktan kumanda ekleme

Yeni bir uzaktan kumanda eklemek için, deponuzun bulunduğu dizinde terminalde git remote add komutunu kullanın.

git remote add komutu iki argüman alır:

  1. Uzak bir ad, örneğin “origin”

  2. Uzak bir URL, örneğin, “https:///user/repo.git”

     git remote add origin https://<your-git-service-address>/owner/repository.git

NOT: Uzaktan kumandayı eklemeden önce git hizmetinizde gerekli depoyu oluşturmanız gerekir, Uzaktan kumandanızı ekledikten sonra taahhütleri itebilir/çekebilirsiniz.

[2]: https://www.wikiod.com/tr/git/giti-kullanmaya-baslama#Kullanıcı adınızı ve e-postanızı ayarlama [3]: https://www.wikiod.com/tr/git/evreleme [4]: https://www.wikiod.com/tr/git/taahhut [5]: https://www.wikiod.com/tr/git/itmek [6]: https://www.wikiod.com/tr/git

Bir depoyu klonlayın #

Git klonu komutu, mevcut bir Git deposunu bir sunucudan yerel makineye kopyalamak için kullanılır.

Örneğin, bir GitHub projesini klonlamak için:

cd <path where you'd like the clone to create a directory>
git clone https://github.com/username/projectname.git

Bir BitBucket projesini klonlamak için:

cd <path where you'd like the clone to create a directory>
git clone https://yourusername@bitbucket.org/username/projectname.git

Bu, yerel makinede uzak Git deposundaki tüm dosyaları içeren “proje adı” adlı bir dizin oluşturur. Bu, proje için kaynak dosyalarının yanı sıra proje için tüm geçmişi ve yapılandırmayı içeren bir “.git” alt dizinini içerir.

Dizinin farklı bir adını belirtmek için, örn. “Klasörüm”:

git clone https://github.com/username/projectname.git MyFolder

Veya geçerli dizinde klonlamak için:

git clone https://github.com/username/projectname.git .

Not:

  1. Belirli bir dizine klonlanırken, dizin boş olmalı veya mevcut olmamalıdır.

  2. Komutun ssh versiyonunu da kullanabilirsiniz:

    git clone git@github.com:username/projectname.git

‘https’ sürümü ve ‘ssh’ sürümü eşdeğerdir. Ancak, GitHub önerilir gibi bazı barındırma hizmetleri `https’yi kullanır. ‘ssh’ yerine.

Kodu paylaşma #

Kodunuzu paylaşmak için, yerel havuzunuzu kopyalayacağınız uzak bir sunucuda bir havuz oluşturursunuz.

Uzak sunucudaki alan kullanımını en aza indirmek için, yalnızca .git nesnelerine sahip olan ve dosya sisteminde çalışan bir kopya oluşturmayan çıplak bir havuz oluşturursunuz. Bonus olarak, güncellemeleri diğer programcılarla kolayca paylaşmak için bu uzaktan kumandayı bir yukarı akış sunucusu olarak ayarlarsınız.

Uzak sunucuda:

git init --bare /path/to/repo.git

Yerel makinede:

git remote add origin ssh://username@server:/path/to/repo.git

(ssh: uzak depoya erişmenin olası yollarından yalnızca biri olduğunu unutmayın.)

Şimdi yerel deponuzu uzaktan kumandaya kopyalayın:

git push --set-upstream origin master

“–set-upstream” (veya “-u”) eklenmesi, bağımsız değişkensiz Git komutları tarafından kullanılan bir yukarı akış (izleme) referansı oluşturdu, ör. ‘git çekme’.

Kullanıcı adınızı ve e-postanızı ayarlama #

You need to set who you are *before* creating any commit.  That will allow commits to have the right author name and email associated to them.

Uzak bir havuza gönderirken kimlik doğrulama ile ilgisi yoktur (örneğin, GitHub, BitBucket veya GitLab hesabınızı kullanarak uzak bir havuza gönderirken)

tüm depolar için bu kimliği bildirmek için git config --global kullanın Bu, ayarı kullanıcınızın .gitconfig dosyasında saklayacaktır: ör. $HOME/.gitconfig veya Windows için %USERPROFILE%\.gitconfig.

git config --global user.name "Your Name"
git config --global user.email mail@example.com

Tek bir depo için bir kimlik bildirmek için bir depo içinde git config kullanın. Bu, ayarı bireysel depoda, $GIT_DIR/config dosyasında saklayacaktır. Örneğin. /path/to/your/repo/.git/config.

cd /path/to/my/repo
git config user.name "Your Login At Work"
git config user.email mail_at_work@example.com

Bir havuzun yapılandırma dosyasında saklanan ayarlar, o havuzu kullandığınızda genel yapılandırmaya göre öncelikli olacaktır.

İpuçları: Farklı kimlikleriniz varsa (biri açık kaynak projesi için, biri iş yerinde, biri özel depolar için, …) ve üzerinde çalıştığınız her farklı depo için doğru olanı ayarlamayı unutmak istemiyorsanız :

  • Küresel bir kimliği kaldırın

    git config --global --remove-section user.name
git config --global --remove-section user.email

  • Git’i kimliğinizi global yapılandırmada değil, yalnızca bir havuzun ayarlarında aramaya zorlamak için:

      git config --global user.useConfigOnly true
son sürüm

Bu şekilde, belirli bir depo için “user.name” ve “user.email"i ayarlamayı unutursanız ve bir taahhütte bulunmaya çalışırsanız, şunu göreceksiniz:

no name was given and auto-detection is disabled
no email was given and auto-detection is disabled

Yukarı akış uzaktan kumandasını ayarlama #

Bir çatal klonladıysanız (örneğin, Github’daki açık kaynaklı bir proje), yukarı akış deposuna anında erişiminiz olmayabilir, bu nedenle hem çatalınıza ihtiyacınız vardır hem de yukarı akış deposunu getirebilirsiniz.

Önce uzak adları kontrol edin:

$ git remote -v
origin    https://github.com/myusername/repo.git (fetch)
origin    https://github.com/myusername/repo.git (push)
upstream  # this line may or may not be here

Eğer “yukarı akış” zaten varsa (bazı Git sürümlerinde bulunur) URL’yi ayarlamanız gerekir (şu anda boştur):

$ git remote set-url upstream https://github.com/projectusername/repo.git

Yukarı akış orada değilse veya bir arkadaşın/iş arkadaşının çatalını da eklemek istiyorsanız (şu anda mevcut değiller):

$ git remote add upstream https://github.com/projectusername/repo.git
$ git remote add dave https://github.com/dave/repo.git

Bir komut hakkında bilgi edinme #

Herhangi bir git komutu hakkında daha fazla bilgi almak için - yani komutun ne yaptığı, mevcut seçenekler ve diğer belgeler hakkında ayrıntılar - “–help” seçeneğini veya “help” komutunu kullanın.

Örneğin, “git diff” komutuyla ilgili mevcut tüm bilgileri almak için şunu kullanın:

git diff --help
git help diff

Benzer şekilde, “status” komutuyla ilgili mevcut tüm bilgileri almak için şunu kullanın:

git status --help
git help status

Yalnızca en çok kullanılan komut satırı işaretlerinin anlamını gösteren hızlı bir yardım istiyorsanız, -h kullanın:

git checkout -h

Git için SSH’yi kurun #

Windows kullanıyorsanız Git Bash açın. Mac veya Linux kullanıyorsanız Terminalinizi açın.

Bir SSH anahtarı oluşturmadan önce, mevcut herhangi bir SSH anahtarınız olup olmadığını kontrol edebilirsiniz.

~/.ssh dizininizin içeriğini listeleyin:

$ ls -al ~/.ssh 
# Lists all the files in your ~/.ssh directory

Zaten bir genel SSH anahtarınız olup olmadığını görmek için dizin listesini kontrol edin. Varsayılan olarak ortak anahtarların dosya adları aşağıdakilerden biridir:

id_dsa.pub
id_ecdsa.pub
id_ed25519.pub
id_rsa.pub

Bitbucket, GitHub (veya benzeri) hesabınızda kullanmak istediğiniz mevcut bir genel ve özel anahtar çiftinin listelendiğini görürseniz, ‘id_*.pub’ dosyasının içeriğini kopyalayabilirsiniz.

Değilse, aşağıdaki komutla yeni bir genel ve özel anahtar çifti oluşturabilirsiniz:

$ ssh-keygen

Varsayılan konumu kabul etmek için Enter veya Return tuşuna basın. İstendiğinde bir parola girin ve yeniden girin veya boş bırakın.

SSH anahtarınızın ssh aracısına eklendiğinden emin olun. Zaten çalışmıyorsa, ssh-agent’ı arka planda başlatın:

$ eval "$(ssh-agent -s)"

SSH anahtarınızı ssh aracısına ekleyin. Komuttaki ‘id_rsa’yı özel anahtar dosyanızın adıyla değiştirmeniz gerektiğine dikkat edin:

$ ssh-add ~/.ssh/id_rsa

Mevcut bir havuzun yukarı akışını HTTPS’den SSH’ye değiştirmek istiyorsanız, aşağıdaki komutu çalıştırabilirsiniz:

$ git remote set-url origin ssh://git@bitbucket.server.com:7999/projects/your_project.git

SSH üzerinden yeni bir depo klonlamak için aşağıdaki komutu çalıştırabilirsiniz:

$ git clone ssh://git@bitbucket.server.com:7999/projects/your_project.git

Git Kurulumu #

Biraz Git kullanmaya başlayalım. İlk önce ilk şeyler - yüklemeniz gerekir. Bunu birkaç yolla elde edebilirsiniz; başlıca iki tanesi, onu kaynaktan kurmak veya platformunuz için mevcut bir paketi kurmaktır.

Kaynaktan Yükleme

Yapabiliyorsanız, en son sürümü alacağınız için Git’i kaynaktan yüklemek genellikle yararlıdır. Git’in her sürümü, kullanışlı UI geliştirmeleri içerir, bu nedenle yazılımı kaynaktan derleme konusunda rahat hissediyorsanız, en son sürümü almak genellikle en iyi yoldur. Pek çok Linux dağıtımının çok eski paketler içermesi de söz konusudur; bu nedenle, çok güncel bir dağıtımda değilseniz veya backport kullanmıyorsanız, kaynaktan yüklemek en iyi seçenek olabilir.

Git’i kurmak için Git’in bağlı olduğu şu kitaplıklara sahip olmanız gerekir: curl, zlib, openssl, expat ve libiconv. Örneğin, yum (Fedora gibi) veya apt-get (Debian tabanlı bir sistem gibi) olan bir sistemdeyseniz, tüm bağımlılıkları yüklemek için şu komutlardan birini kullanabilirsiniz:

$ yum install curl-devel expat-devel gettext-devel \
  openssl-devel zlib-devel

$ apt-get install libcurl4-gnutls-dev libexpat1-dev gettext \
  libz-dev libssl-dev

Gerekli tüm bağımlılıklara sahip olduğunuzda, devam edip Git web sitesinden en son anlık görüntüyü alabilirsiniz:

http://git-scm.com/download Ardından derleyin ve kurun:

$ tar -zxf git-1.7.2.2.tar.gz
$ cd git-1.7.2.2
$ make prefix=/usr/local all
$ sudo make prefix=/usr/local install

Bu yapıldıktan sonra, güncellemeler için Git’i Git’in kendisi aracılığıyla da alabilirsiniz:

$ git clone git://git.kernel.org/pub/scm/git/git.git

Linux’a yükleme

Git’i bir ikili yükleyici aracılığıyla Linux’a kurmak istiyorsanız, bunu genellikle dağıtımınızla birlikte gelen temel paket yönetimi aracıyla yapabilirsiniz. Fedora’daysanız, yum’u kullanabilirsiniz:

$ yum install git

Veya Ubuntu gibi Debian tabanlı bir dağıtım kullanıyorsanız apt-get’i deneyin:

$ apt-get install git

Mac’te yükleme

Git’i bir Mac’e kurmanın üç kolay yolu vardır. En kolayı, SourceForge sayfasından indirebileceğiniz grafik Git yükleyicisini kullanmaktır.

http://sourceforge.net/projects/git-osx-installer/

Şekil 1-7. Git OS X yükleyici. Diğer önemli yol ise Git’i MacPorts (http://www.macports.org) aracılığıyla kurmaktır. Yüklü MacPorts’unuz varsa, Git’i şuradan yükleyin:

$ sudo port install git +svn +doc +bash_completion +gitweb

Tüm ekstraları eklemek zorunda değilsiniz, ancak Git’i Subversion depoları ile kullanmak zorunda kalmanız durumunda muhtemelen +svn eklemek isteyeceksiniz (bkz. Bölüm 8).

Homebrew (http://brew.sh/), Git’i kurmak için başka bir alternatiftir. Homebrew yüklediyseniz, Git’i şuradan yükleyin:

$ brew install git

Windows’ta yükleme

Git’i Windows’a yüklemek çok kolaydır. msysGit projesi, daha kolay kurulum prosedürlerinden birine sahiptir. GitHub sayfasından yükleyici exe dosyasını indirmeniz ve çalıştırmanız yeterlidir:

http://msysgit.github.io

Kurulduktan sonra, hem komut satırı sürümünüz (daha sonra kullanışlı olacak bir SSH istemcisi dahil) hem de standart GUI’niz olur.

Windows kullanımı ile ilgili not: Git’i sağlanan msysGit kabuğu (Unix stili) ile kullanmalısınız, bu kitapta verilen karmaşık komut satırlarını kullanmanıza izin verir. Herhangi bir nedenle yerel Windows kabuk/komut satırı konsolunu kullanmanız gerekiyorsa, tek tırnak yerine çift tırnak kullanmanız (içlerinde boşluk olan parametreler için) ve inceltme vurgusu (^) ile biten parametreleri alıntılamanız gerekir. ) Windows’ta bir devam sembolü olduğu için satırda sonuncuysalar.
