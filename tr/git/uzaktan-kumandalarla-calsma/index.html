Sözdizimi #

  • git uzak [-v | --ayrıntılı]
  • git remote add [-t <branch>] [-m <master>] [-f] [--[no-]tags] [--mirror=<fetch|push>] <name> <url>
  • git uzaktan yeniden adlandır <eski> <yeni>
  • `git uzaktan ‘ı kaldır
  • git remote set-head <ad> (-a | --auto | -d | --delete | <branch>)
  • git uzak set-dallar [--add] <name> <branch>…​
  • git remote get-url [--push] [--all] <name>
  • git remote set-url [--push] <name> <newurl> [<oldurl>]
  • git remote set-url --add [--push] <name> <newurl>
  • git remote set-url --delete [--push] <name> <url>
  • git uzak [-v | --verbose] göster [-n] <name>…​
  • git uzak kuru erik [-n | --kuru çalışma] <ad>…​
  • git uzak [-v | --verbose] güncelleme [-p | --prune] [(<grup> | <uzaktan>)…​]

Uzak Şubeyi Silme #

Git Uzak URL’sini Değiştirme #

Mevcut uzaktan kumandayı kontrol edin

git remote -v 
# origin https://github.com/username/repo.git (fetch)
# origin https://github.com/usernam/repo.git (push)

Depo URL’sini değiştirme

git remote set-url origin https://github.com/username/repo2.git
# Change the 'origin' remote's URL

Yeni uzak URL’yi doğrulayın

git remote -v
# origin  https://github.com/username/repo2.git (fetch)
# origin  https://github.com/username/repo2.git (push)

Upstream Deposundan Güncelleme #

Yukarı akışı ayarladığınızı varsayarsak (“bir yukarı akış deposu ayarlama” gibi)

git fetch remote-name
git merge remote-name/branch-name

‘pull’ komutu, ‘getir’ ve ‘birleştirme’yi birleştirir.

git pull

“–rebase” bayrak komutuyla “pull”, “birleştirme” yerine “getirme” ve “yeniden tabana alma"yı birleştirir.

git pull --rebase remote-name branch-name

ls-uzaktan kumanda #

Silinen Uzak Dalların Yerel Kopyalarını Kaldırma #

Mevcut Uzaktan Kumandaları Listeleme #

Bu depoyla ilişkili tüm mevcut uzaktan kumandaları listeleyin:

git remote

“Getir” ve “push” URL’leri dahil olmak üzere bu depoyla ilişkili tüm mevcut uzaktan kumandaları ayrıntılı olarak listeleyin:

git remote --verbose

ya da sadece

git remote -v

Yeni Bir Uzak Depo Ekleme #

git remote add upstream git-repository-url

“git-repository-url” ile temsil edilen uzak git deposunu, “upstream” adlı yeni uzaktan kumanda olarak git deposuna ekler

Başlarken #

Yeni Bir Dalda Akış Yukarı Ayarlayın #

Belirli bir Uzaktan Kumandayla ilgili bilgileri göster #

Bilinen bir uzaktan kumandayla ilgili bazı bilgilerin çıktısını alın: ‘origin’

git remote show origin

Yalnızca uzaktan kumandanın URL’sini yazdırın:

git config --get remote.origin.url

2.7+ ile, ‘config’ komutunu kullanan yukarıdakinden tartışmasız daha iyi olan yapmak da mümkündür.

git remote get-url origin

Uzak Depoyu Değiştirme #

Uzaktan kumandanızın işaret etmesini istediğiniz veri havuzunun URL’sini değiştirmek için ‘set-url’ seçeneğini şu şekilde kullanabilirsiniz:

git remote set-url <remote_name> <remote_repository_url>

Örnek:

git remote set-url heroku https://git.heroku.com/fictional-remote-repository.git

Uzaktan Kumandayı Yeniden Adlandırma #

Uzaktan kumandayı yeniden adlandırmak için “git remote rename” komutunu kullanın.

git remote rename komutu iki argüman alır:

  • Mevcut bir uzak ad, örneğin: Origin
  • Uzaktan kumanda için yeni bir ad, örneğin: hedef

Mevcut uzak adı al

git remote
# origin

URL ile mevcut uzaktan kumandayı kontrol edin

git remote -v 
# origin https://github.com/username/repo.git (fetch)
# origin https://github.com/usernam/repo.git (push)

Uzaktan kumandayı yeniden adlandır

 git remote rename origin destination
 # Change remote name from 'origin' to 'destination'

Yeni adı doğrulayın

git remote -v 
# destination https://github.com/username/repo.git (fetch)
# destination https://github.com/usernam/repo.git (push)

=== Olası Hatalar ===

  1. ‘uzak.[eski ad]’ yapılandırma bölümünün adı ‘uzak.[yeni ad]’ olarak yeniden adlandırılamadı

    This error means that the remote you tried the old remote name (origin) doesn’t exist.

  2. Uzak [yeni ad] zaten var.

    Error message is self explanatory.

Belirli Bir Uzaktan Kumandanın URL’sini Ayarlayın #

Komutla mevcut bir uzaktan kumandanın url’sini değiştirebilirsiniz.

git remote set-url remote-name url

Belirli Bir Uzaktan Kumandanın URL’sini Alın #

Komutu kullanarak mevcut bir uzaktan kumandanın url’sini alabilirsiniz.

git uzak get-url <ad>

Varsayılan olarak, bu olacak

`git uzak get-url kaynağı’
