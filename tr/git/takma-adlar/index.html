Mevcut takma adları listeleyin / arayın #

“–get-regexp” kullanarak mevcut git takma adlarını listeleyebilirsiniz:

$ git config --get-regexp '^alias\.'

Takma ad arama #

Takma adları aramak, “[alias” altındaki “.gitconfig” dosyanıza şunu ekleyin ]`:

aliases = !git config --list | grep ^alias\\. | cut -c 7- | grep -Ei --color \"$1\" "#"

O zaman yapabilirsin:

  • git takma adları - TÜM takma adları göster
  • “git aliases commit” - yalnızca “commit” içeren takma adlar

Basit takma adlar #

Git’te takma ad oluşturmanın iki yolu vardır:

  • ~/.gitconfig dosyasıyla:

    [alias] ci = commit st = status co = checkout

  • komut satırı ile:

    git config –global alias.ci “commit” git config –global alias.st “status” git config –global alias.co “checkout”

Takma ad oluşturulduktan sonra - şunu yazın:

  • “git taahhüt” yerine “git ci”,
  • “git durumu” yerine “git st”,
  • “git checkout” yerine “git co”.

Normal git komutlarında olduğu gibi, takma adlar argümanların yanında kullanılabilir. Örneğin:

 git ci -m "Commit message..."
 git co -b feature-42

Gelişmiş Takma Adlar #

Git, takma adlarınızda ! ile ön eklerseniz, git olmayan komutları ve tam sh kabuk sözdizimini kullanmanıza izin verir.

~/.gitconfig dosyanızda:

[alias]
    temp = !git add -A && git commit -m "Temp"

Bu ön ekli takma adlarda tam kabuk sözdiziminin mevcut olması, komut satırı bağımsız değişkenlerini kullananlar gibi daha karmaşık takma adlar oluşturmak için kabuk işlevlerini kullanabileceğiniz anlamına gelir:

[alias]
    ignore = "!f() { echo $1 >> .gitignore; }; f"

Yukarıdaki takma ad, “f” işlevini tanımlar, ardından diğer ada ilettiğiniz herhangi bir bağımsız değişkenle onu çalıştırır. Bu nedenle, “git görmezden .tmp/“yi çalıştırmak, “.gitignore” dosyanıza “.tmp/” ekler.

Aslında bu kalıp o kadar kullanışlıdır ki Git sizin için $1, $2 vb. değişkenleri tanımlar, böylece onun için özel bir fonksiyon tanımlamanıza bile gerek kalmaz. (Ancak Git’in bu değişkenler aracılığıyla erişseniz bile argümanları yine de ekleyeceğini unutmayın, bu nedenle sonunda boş bir komut eklemek isteyebilirsiniz.)

Bu şekilde ön eki ! olan takma adların, geçerli dizininiz ağaçta daha derinde olsa bile, git checkout’unuzun kök dizininden çalıştırıldığını unutmayın. Bu, orada açıkça “cd” yapmak zorunda kalmadan kökten bir komut çalıştırmanın yararlı bir yolu olabilir.

[alias]
    ignore = "! echo $1 >> .gitignore"

İzlenen dosyaları geçici olarak yoksay #

Bir dosyayı geçici olarak yoksayıldı olarak işaretlemek için (dosyayı diğer ada parametre olarak iletin) - şunu yazın:

unwatch = update-index --assume-unchanged

Dosyayı tekrar izlemeye başlamak için - şunu yazın:

watch = update-index --no-assume-unchanged

Geçici olarak yoksayılan tüm dosyaları listelemek için şunu yazın:

unwatched = "!git ls-files -v | grep '^[[:lower:]]'"

İzlenmeyen listeyi temizlemek için - şunu yazın:

watchall = "!git unwatched | xargs -L 1 -I % sh -c 'git watch `echo % | cut -c 2-`'"

Takma adları kullanma örneği:

git unwatch my_file.txt
git watch my_file.txt
git unwatched
git watchall

Dal grafiğiyle güzel günlüğü göster #

[alias]
  logp=log --pretty=format:'%h %ad | %s%d [%an]' --graph --date=short

  lg =  log --graph --date-order --first-parent \
     --pretty=format:'%C(auto)%h%Creset %C(auto)%d%Creset %s %C(green)(%ad) %C(bold cyan)<%an>%Creset'
  lgb = log --graph --date-order --branches --first-parent \                      
     --pretty=format:'%C(auto)%h%Creset %C(auto)%d%Creset %s %C(green)(%ad) %C(bold cyan)<%an>%Creset'
 lga = log --graph --date-order --all \                                          
   --pretty=format:'%C(auto)%h%Creset %C(auto)%d%Creset %s %C(green)(%ad) %C(bold cyan)<%an>%Creset'

Burada ‘–pretty’ biçiminde kullanılan seçeneklerin ve yer tutucunun açıklaması (‘git help log’ ile birlikte kapsamlı liste mevcuttur)

–graph - taahhüt ağacını çizin

–date-order - mümkün olduğunda taahhüt zaman damgası sırasını kullanın

–first-parent - birleştirme düğümünde yalnızca ilk ebeveyni takip edin.

–branches - tüm yerel dalları göster (varsayılan olarak yalnızca geçerli dal gösterilir)

–all - tüm yerel ve uzak şubeleri göster

%h - taahhüt için hash değeri (kısaltılmış)

%ad - Tarih damgası (yazar)

%an - Yazar kullanıcı adı

%an - Kullanıcı adını taahhüt et

%C(auto) - [renk] bölümünde tanımlanan renkleri kullanmak için

%Creset - rengi sıfırlamak için

%d - –decorate (dal ve etiket adları)

%s - mesajı onayla

%ad - yazar tarihi (–tarih yönergesini takip eder) (taahhüt tarihini değil)

%an - yazar adı (taahhüt eden adı için %cn olabilir)

Doğrusal bir geçmiş tutarken kodu güncelleme #

Bazen kod taahhütlerinizin doğrusal (dallanmayan) bir geçmişini tutmanız gerekir. Bir süredir bir dal üzerinde çalışıyorsanız, normal bir “git pull” yapmanız gerekiyorsa bu zor olabilir, çünkü bu, yukarı akışla bir birleştirme kaydedecektir.

[alias]
  up = pull --rebase

Bu, yukarı akış kaynağınızla güncellenir, ardından aşağı çektiğiniz şeyin üstüne itmediğiniz herhangi bir işi yeniden uygular.

Kullanmak:

git up

.gitignore yapılandırmanız tarafından hangi dosyaların yoksayıldığını görün #

[ alias ]

    ignored = ! git ls-files --others --ignored --exclude-standard --directory \
            && git ls-files --others -i --exclude-standard

Dosya başına bir satır gösterir, böylece grep yapabilirsiniz (yalnızca dizinler):

$ git ignored | grep '/$'
.yardoc/
doc/

Veya sayın:

~$ git ignored | wc -l
199811                 # oops, my home directory is getting crowded

Aşamalı olmayan aşamalı dosyalar #

Normalde, ‘git reset’ taahhüdü kullanılarak kaydedilmek üzere hazırlanan dosyaları kaldırmak için, ‘reset’, kendisine sağlanan argümanlara bağlı olarak birçok işleve sahiptir. Aşamalı tüm dosyaları tamamen kaldırmak için, “reset” kullanan yeni bir takma ad oluşturmak için git takma adlarını kullanabiliriz, ancak şimdi “sıfırlama” için doğru argümanları sağlamayı hatırlamamız gerekmiyor.

git config --global alias.unstage "reset --"

Şimdi, dosyaları aşama aşamalarını çevre dışı bırakmak istediğiniz zaman, git unstage yazın ve hazırsınız.
