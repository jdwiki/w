Dojo'yu dojoConfig ile yapılandırma

  • Yükleyici Yapılandırması
  • DojoConfig’i Yükle

Bu sayfada

  • Yükleyici Yapılandırması
  • DojoConfig’i Yükle

dojoConfig nesnesi (önceden djConfig), dojo araç setinin çeşitli yönleri için seçenekleri ve varsayılan davranışı ayarlamanıza olanak tanır. Örnekler, neyin mümkün olduğunu ve kodunuzda kullanmak üzere dojoConfig’i nasıl koyabileceğinizi açıklayacaktır.

dojoConfig’in dojo.js yüklenmeden önce bir komut dosyası bloğunda tanımlandığını unutmayın. Bu son derece önemlidir—eğer tersine çevrilirse, konfigürasyon özellikleri göz ardı edilecektir.

Yükleyici Yapılandırması #

Dojo, araç setinin yeni AMD modül formatına uyum sağlamak için Dojo 1.7’de yeni bir yükleyici aldı. Bu yeni yükleyici, paketleri, haritaları ve daha fazlasını tanımlamak için çok önemli olan birkaç yeni yapılandırma seçeneği ekledi. Yükleyiciyle ilgili ayrıntılar için Gelişmiş AMD Kullanımı öğreticisine bakın. Önemli yükleyici yapılandırma parametreleri şunları içerir:

baseUrl: Aşağıdaki durumlarda bir modül tanımlayıcısının başına eklenen temel URL bir yola veya URL’ye dönüştürme.

baseUrl: "/js"

paketler: Paket adını ve konum:

packages: [{
    name: "myapp",
    location: "/js/myapp"
}]

map: Modül tanımlayıcılarındaki yolları farklı yollara eşlemenizi sağlar:

map: {
    dijit16: {
        dojo: "dojo16"
    }
}

yollar: modül kimliği parçalarının dosya yollarına haritası:

var dojoConfig = {
    packages: [
        "package1",
        "package2"
    ],
    paths: {
        package1: "../lib/package1",
        package2: "/js/package2"
    }
};

    // ...is equivalent to:
var dojoConfig = {
    packages: [
        { name: "package1", location: "../lib/package1" },
        { name: "package2", location: "/js/package2" }
    ]
};

eşzamansız: Dojo çekirdeğinin eşzamansız olarak yüklenmesi gerekip gerekmediğini tanımlar. değerler yükleyiciyi kalıcı olarak yerleştiren true, false veya LegacyAsync olabilir eski etki alanları arası modunda.

async: true

parseOnLoad: true ise, DOM ve tüm ilk bağımlılıklar (dojoConfig.deps dosyasındakiler dahil) dizi) yüklendi.

parseOnLoad: true

It is recommended that parseOnLoad be left at false (it defaults to false, so you can simply omit this property), and that developers explicitly require dojo/parser and call parser.parse().

deps: Bir kez hemen yüklenmesi gereken bir dizi kaynak yolu Dojo şunları yükledi:

deps: ["dojo/parser"]

geri arama: Deps alındıktan sonra yürütülecek geri arama:

callback: function(parser) {
    // Use the resources provided here
}

waitSeconds: Yükleme zaman aşımı sinyali verilmeden önce beklenecek süre bir modül; varsayılan olarak 0’dır (sonsuza kadar bekleyin):

waitSeconds: 5

cacheBust: Doğruysa, zamanı her modüle bir sorgu dizesi olarak ekler Modül önbelleğe almayı önlemek için URL:

cacheBust: true

Şimdi, kullanılacak temel parametreleri koyan basit bir demo oluşturalım. Çok yaygın bir senaryo, yerel modüllerle CDN’den Dojo Toolkit kullanmaktır.

Google CDN’yi /documentation/tutorials/1.10/dojo_config/demo alanında modüllerle kullandığımızı varsayalım:

<!-- Configure Dojo first -->
<script>
    dojoConfig = {
        has: {
            "dojo-firebug": true,
            "dojo-debug-messages": true
        },
        // Don't attempt to parse the page for widgets
        parseOnLoad: false,
        packages: [
            // Any references to a "demo" resource should load modules locally, *not* from CDN
            {
                name: "demo",
                location: "/documentation/tutorials/1.10/dojo_config/demo"
            }
        ],
        // Timeout after 10 seconds
        waitSeconds: 10,
        map: {
            // Instead of having to type "dojo/domReady!", we just want "ready!" instead
            "*": {
                ready: "dojo/domReady"
            }
        },
        // Get "fresh" resources
        cacheBust: true
    };
</script>

<!-- Load Dojo, Dijit, and DojoX resources from Google CDN -->
<script src="//ajax.googleapis.com/ajax/libs/dojo/1.10.4/dojo/dojo.js"></script>

<!-- Load a "demo" module -->

<script>
    require(["demo/AuthoredDialog", "dojo/parser", "ready!"], function(AuthoredDialog, parser) {
        // Parse the page
        parser.parse();

        // Do something with demo/AuthoredDialog...
    });
</script>

Paket konfigürasyonunu kullanarak, yerel /documentation/tutorials/1.10/dojo_config/demo/ dizinimize demo/* noktasına tüm referansları yaptık ve dojo, dijit, ve ‘dojox’ Google CDN’den gelecek. Demo paketi tanımlanmasaydı, “demo/AuthoredDialog” isteği “//ajax.googleapis.com/ajax/libs/dojo/1.10.4/dojo/demo/AuthoredDialog.js” adresine gidecekti. Ayrıca ‘dojo/domReady’ ile hazır ile ilişkilendirerek takma ad kullandık.

DojoConfig’i Yükle #

Aşağıdaki örnekte, tüm konfigürasyon değerlerini içerecek olan bir global javascript nesnesi dojoConfig oluşturuyoruz.

Not: dojoConfig, dojo.js yüklenmeden önce bir komut dosyası bloğunda tanımlanır. Bu son derece önemlidir—eğer tersine çevrilirse, konfigürasyon özellikleri göz ardı edilecektir.

<script>
    dojoConfig= {
        has: {
            "dojo-firebug": true
        },
        parseOnLoad: false,
        foo: "bar",
        async: true
    };
</script>
<script src="//ajax.googleapis.com/ajax/libs/dojo/1.10.4/dojo/dojo.js"></script>

<script>
// Require the registry, parser, Dialog, and wait for domReady
require(["dijit/registry", "dojo/parser", "dojo/json", "dojo/_base/config", "dijit/Dialog", "dojo/domReady!"]
, function(registry, parser, JSON, config) {
    // Explicitly parse the page
    parser.parse();
    // Find the dialog
    var dialog = registry.byId("dialog");
    // Set the content equal to what dojo.config is
    dialog.set("content", "<pre>" + JSON.stringify(config, null, "\t") + "```");
    // Show the dialog
    dialog.show();
});
</script>

<!-- and later in the page -->
<div id="dialog" data-dojo-type="dijit/Dialog" data-dojo-props="title: 'dojoConfig / dojo/_base/config'"></div>
