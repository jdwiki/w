System.Runtime ile mscorlib karşılaştırmasını anlama

  • Popüler Hata: Yanıltıcı NuGet yanlış yoldan
  • Popüler Hata: netstandard / netcoreapp (System.Runtime) için yapılmayan bir NuGet paketi ekleyin
  • Popüler Hata: Sonucu yanlış anlama

Bu sayfada

  • Popüler Hata: Yanıltıcı NuGet yanlış yoldan
  • Popüler Hata: netstandard / netcoreapp (System.Runtime) için yapılmayan bir NuGet paketi ekleyin
  • Popüler Hata: Sonucu yanlış anlama
  • Popüler Hata: Yanlışlıkla bir netstandard/netcoreapp’a bağımlılık olarak bir mscorlib kitaplığı ekleme

Her .NET kitaplığı ve programlama dili, ‘System.Int32’, ‘System.Object’, ‘System.Type’ veya ‘System.Uri’ gibi bir dizi temel veri türünü kullanır. Bu veri türleri, tüm özel yazılmış .NET kitaplıkları dahil olmak üzere diğer tüm yapıların temelini oluşturur. Tüm bu türler, “mscorlib” veya “System.Runtime” olan bir temel kitaplıkta barındırılır.

.NET Core ile kullanılabilen kitaplıklar ‘System.Runtime’ çekirdek kitaplığını temel alırken, .NET Framework (Windows bileşeni) için ‘mscorlib’i temel alırlar. Bu temel farklılık,…

  • bir .NET Core kitaplığı ‘System.Object, System.Runtime’ beklerken bir tür ‘System.Object, mscorlib’ bekledikleri için eski kitaplıkların uyumsuzluğu.
  • .NET Framework içindeki ‘mscorlib’e ‘System.Runtime’ adlı bir tip iletme cephe kitaplığı. Bu kitaplık aksi takdirde (neredeyse) boştur ancak .NET Framework üzerinde ‘System.Runtime’ tabanlı PCL kitaplıklarının kullanımını sağlar.
  • .NET Core‘un gelecekteki bir sürümünde ‘System.Runtime’a ‘mscorlib’ ileten bir tür.
  • taşınabilir sınıf kitaplığı kavramının (PCL) ve ikinci nesil olarak iki çekirdek kitaplık arasında bir birleştirme yöntemi olarak “netstandard"ın tanıtılması.

VE bunun dışında, Yığın Taşması ile ilgili sayısız soru.

Popüler Hata: Yanıltıcı NuGet yanlış yoldan #

.NET Core ‘project.json’, NuGet içe aktarmayı destekler (a.k.a. [bu SO yanıtına] göre yalan söyler(http://stackoverflow.com/a/38688430/2410379)). Bir “import” ifadesi nedeniyle “mscorlib” tabanlı bir kitaplık eklemek mümkün değildir.

{
    "version": "1.0.0-*",

    "dependencies": {
        "Microsoft.AspNet.Identity.EntityFramework": "2.2.1",
        "NETStandard.Library": "1.6.0"
    },

    "frameworks": {
        "netstandard1.6": {
            "imports": [
                "net461"
            ]
        }
    }
}

İçe aktarma işlemleri yalnızca taşınabilir sınıf kitaplıkları (“System.Runtime” tabanlı) veya aynı zamanda “System.Runtime” tabanlı (ör. “dotnet” veya “dnxcore”) kullanımdan kaldırılmış hedef çerçeve takma adlarıyla çalışır.

Popüler Hata: netstandard / netcoreapp (System.Runtime) için yapılmayan bir NuGet paketi ekleyin #

Aşağıdaki “project.json” örneğinde, “mscorlib” tabanlı bir “Microsoft.AspNet.Identity.EntityFramework” derlemesi eklenmiştir.

{
    "version": "1.0.0-*",

    "dependencies": {
        "Microsoft.AspNet.Identity.EntityFramework": "2.2.1",
        "NETStandard.Library": "1.6.0"
    },

    "frameworks": {
        "netstandard1.6": { }
    }
}

“Microsoft.AspNet.Identity.EntityFramework” derlemesinin yazarı, NuGet paketini henüz “netstandard"a taşımadı (aslında yaptılar, paketin adını da “Microsoft.AspNetCore.Identity.EntityFrameworkCore” olarak değiştirdiler;))

İhtiyacınız olan ve henüz netstandard’da olmayan bir paketle karşılaşırsanız, lütfen yazarla iletişime geçin ve mümkünse paketin taşınmasına yardımcı olun.

Popüler Hata: Sonucu yanlış anlama #

“project.json” ile birden çok çerçeveyi hedeflemek basittir. Ancak sonuç iki farklı derlemeler. Aşağıdaki örneği alın:

{
    "version": "1.0.0-*",

    "dependencies": {
        "NETStandard.Library": "1.6.0",
        "System.Collections.Immutable": "1.2.0"
    },

    "frameworks": {
        "netstandard1.3": { },
        "net451": { }
    }
}

‘project.json’ dosyası için derleme işlemi, sonuçta ortaya çıkan iki esere yol açacaktır:

  • .NET Core, .NET Framework (tür ileticileri aracılığıyla) ve Xamarin ürünlerinde (tür ileticileri aracılığıyla) kullanılabilen ‘System.Runtime’ tabanlı ’netstandard’ dünyası için derlenmiş bir dll. Bu dll’nin System.Runtime ve System.Collections.Immutable’a referansları var.
  • Doğrudan mscorlib tabanlı .NET Framework için başka bir derlenmiş dll. Bu dll, “mscorlib” ve “System.Collection.Immutable” için referanslara sahip olacaktır.

Ancak, “netstandard1.0” tabanlı “System.Collections.Immutable"ın, çalışma zamanında her yapı dll için farklı “System.Runtime” uygulamalarını kullanacağını anlamak önemlidir. .NET Core ile birlikte gelen System.Runtime kendi başına herhangi bir derleme bağımlılığına sahip değildir (çünkü çekirdek kitaplığı uygular). .NET Framework ile kullanılan ‘System.Runtime’, .NET Framework derlemelerine ‘mscorlib’, ‘System.Core’, ‘System’ ve ‘System.ComponentModel.Composition’ başvurularına (tür ileticiler için) sahiptir.

Popüler Hata: Yanlışlıkla bir netstandard/netcoreapp’a bağımlılık olarak bir mscorlib kitaplığı ekleme #

Diğer bir popüler hata, sistemdeki tüm çerçeveyi karşılamayan paketlere atıfta bulunulmasıdır. birden çok çerçeve hedeflendiğinde küresel kapsam.

{
    "version": "1.0.0-*",

    "dependencies": {
        "NETStandard.Library": "1.6.0",
        "Microsoft.AspNet.Identity.EntityFramework": "2.2.1"
    },

    "frameworks": {
        "netstandard1.3": { },
        "net451": { }
    }
}

‘NETStandard.Library’ (meta) kitaplığı, hedeflediği için bu örnekte iyi çalışıyor. hem “netstandard1.3” hem de “net451”. Ancak kütüphane “Microsoft.AspNet.Identity.EntityFramework” yalnızca .NET Framework “net” ve “mscorlib"i hedefler ve bu nedenle bir “netstandart” çıktı için kullanılamaz.

Her iki çerçeveyi de kapsayan bir kitaplık (sürüm) arayın veya kitaplığı ekleyin çerçevenin altındaki koşullu bağımlılıklarda.

{
    "version": "1.0.0-*",

    "dependencies": {
        "NETStandard.Library": "1.6.0"
    },

    "frameworks": {
        "netstandard1.3": { },
        "net451": {
            "dependencies": {
                "Microsoft.AspNet.Identity.EntityFramework": "2.2.1",
            }
        }
    }
}

Bu durumda, kitaplık yalnızca “net451” yapısı için koşullu #ifdef bloklarında kullanılabilir.
