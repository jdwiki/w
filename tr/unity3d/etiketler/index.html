Etiket, “GameObject” türlerini işaretlemek için uygulanabilen bir dizedir. Bu şekilde, belirli ‘GameObject’ nesnelerini kod aracılığıyla tanımlamayı kolaylaştırır.

Bir etiket, bir veya daha fazla oyun nesnesine uygulanabilir, ancak bir oyun nesnesinin her zaman yalnızca bir etiketi olacaktır. Varsayılan olarak, “Etiketlenmemiş” etiketi, kasıtlı olarak etiketlenmemiş bir “Oyun Nesnesi"ni temsil etmek için kullanılır.

Etiketleri Oluşturma ve Uygulama #

Etiketler tipik olarak düzenleyici aracılığıyla uygulanır; ancak etiketleri komut dosyası aracılığıyla da uygulayabilirsiniz. Herhangi bir özel etiket, bir oyun nesnesine uygulanmadan önce Etiketler ve Katmanlar penceresi aracılığıyla oluşturulmalıdır.

Editör’de Etiketleri Ayarlama #

Bir veya daha fazla oyun nesnesi seçiliyken denetçiden bir etiket seçebilirsiniz. Oyun nesneleri her zaman tek bir etiket taşır; varsayılan olarak, oyun nesneleri “Etiketlenmemiş” olarak etiketlenecektir. “Etiket Ekle…”‘yi seçerek Etiketler ve Katmanlar penceresine de gidebilirsiniz. ancak bunun sizi yalnızca Etiketler ve Katmanlar penceresine götürdüğünü unutmamak önemlidir. Oluşturduğunuz herhangi bir etiket, oyun nesnesine otomatik olarak uygulanmaz*.

Etiketleri Komut Dosyası ile Ayarlama #

Bir oyun nesneleri etiketini kod aracılığıyla doğrudan değiştirebilirsiniz. Geçerli etiketler listesinden bir etiket sağlamanız gerektiğine dikkat etmek önemlidir; önceden oluşturulmamış bir etiket sağlarsanız, bu bir hataya neden olur.

Diğer örneklerde ayrıntılı olarak açıklandığı gibi, her bir etiketi manuel olarak yazmak yerine bir dizi ‘statik dize’ değişkeni kullanmak tutarlılık ve güvenilirlik sağlayabilir.

Aşağıdaki komut dosyası, tutarlılığı sağlamak için “statik dize” referanslarını kullanarak bir dizi oyun nesnesi etiketini nasıl değiştirebileceğimizi gösterir. Her “statik dizenin”, Etiketler ve Katmanlar penceresinde önceden oluşturulmuş bir etiketi temsil ettiği varsayımına dikkat edin.

using UnityEngine;

public class Tagging : MonoBehaviour
{
    static string tagUntagged = "Untagged";
    static string tagPlayer = "Player";
    static string tagEnemy = "Enemy";

    /// <summary>Represents the player character. This game object should 
    /// be linked up via the inspector.</summary>
    public GameObject player;
    /// <summary>Represents all the enemy characters. All enemies should 
    /// be added to the array via the inspector.</summary>
    public GameObject[] enemy;

    void Start ()
    {
        // We ensure that the game object this script is attached to
        // is left untagged by using the default "Untagged" tag.
        gameObject.tag = tagUntagged;

        // We ensure the player has the player tag.
        player.tag = tagUntagged;

        // We loop through the enemy array to ensure they are all tagged.
        for(int i = 0; i < enemy.Length; i++)
        {
            enemy[i].tag = tagEnemy;
        }
    }
}

Özel Etiketler Oluşturma #

Etiketleri Denetçi aracılığıyla mı yoksa komut dosyası aracılığıyla mı ayarladığınızdan bağımsız olarak, etiketler kullanımdan önce Etiketler ve Katmanlar penceresi aracılığıyla * bildirilmelidir*. Bu pencereye, oyun nesneleri etiketi açılır menüsünden “Etiket Ekle…” öğesini seçerek erişebilirsiniz. Alternatif olarak, pencereyi Düzenle > Proje Ayarları > Etiketler ve Katmanlar altında bulabilirsiniz.

Bir etiket oluşturmak için + düğmesini seçin, istediğiniz adı girin ve Kaydet öğesini seçin. - düğmesinin seçilmesi, o anda vurgulanan etiketi kaldıracaktır. Bu şekilde, etiketin hemen "(Kaldırıldı)” olarak görüntüleneceğini ve proje yeniden yüklendiğinde tamamen kaldırılacağını unutmayın.

Pencerenin sağ üst köşesinden dişli/dişliyi seçmek, tüm özel seçenekleri sıfırlamanıza olanak tanır. Bu, “Katmanları Sıralama” ve “Katmanlar” altında sahip olabileceğiniz herhangi bir özel katmanla birlikte tüm özel etiketleri hemen kaldıracaktır.

GameObjects’i Etikete Göre Bulma: #

Etiketler, belirli oyun nesnelerini bulmayı özellikle kolaylaştırır. Tek bir oyun nesnesini arayabiliriz veya birden fazlasını arayabiliriz.

Tek Bir “GameObject” Bulma #

Bireysel oyun nesnelerini aramak için GameObject.FindGameObjectWithTag(string tag) statik fonksiyonunu kullanabiliriz. Bu şekilde oyun nesnelerinin belirli bir sırayla sorgulanmadığını belirtmek önemlidir. Sahnedeki birden fazla oyun nesnesinde kullanılan bir etiketi ararsanız, bu işlev hangi oyun nesnesinin döndürüleceğini garanti edemez. Bu nedenle, yalnızca bir oyun nesnesinin böyle bir etiketi kullandığını bildiğimizde veya döndürülen ‘GameObject’in tam örneği hakkında endişelenmediğimizde daha uygundur.

///<summary>We create a static string to allow us consistency.</summary>
string playerTag = "Player"

///<summary>We can now use the tag to reference our player GameObject.</summary>
GameObject player = GameObject.FindGameObjectWithTag(playerTag);

“GameObject” örneklerinden oluşan bir Dizi Bulma #

Belirli bir etiketi kullanan tüm oyun nesnelerini aramak için GameObject.FindGameObjectsWithTag(string tag) statik işlevini kullanabiliriz. Bu, bir grup belirli oyun nesnesini yinelemek istediğimizde kullanışlıdır. Bu, tek bir oyun nesnesi bulmak istiyorsak da yararlı olabilir, ancak aynı etiketi kullanan birden çok oyun nesnesi olabilir. “GameObject.FindGameObjectWithTag(string tag)” tarafından döndürülen tam örneği garanti edemediğimiz için, bunun yerine “GameObject.FindGameObjectsWithTag(dize etiketi)” ile tüm potansiyel “GameObject” örneklerinin bir dizisini almalı ve elde edilen diziyi daha fazla analiz etmeliyiz. aradığımız örneği bulmak için.

///<summary>We create a static string to allow us consistency.</summary>
string enemyTag = "Enemy";

///<summary>We can now use the tag to create an array of all enemy GameObjects.</summary>
GameObject[] enemies = GameObject.FindGameObjectsWithTag(enemyTag );

// We can now freely iterate through our array of enemies
foreach(GameObject enemy in enemies)
{
    // Do something to each enemy (link up a reference, check for damage, etc.)
}

Etiketleri Karşılaştırma #

Etiketlere göre iki GameObject’i karşılaştırırken, her seferinde bir dize oluşturulduğundan aşağıdakilerin Garbage Collector ek yüküne neden olacağına dikkat edilmelidir:

if (go.Tag == "myTag")
{ 
    //Stuff 
}

Bu karşılaştırmaları Update() ve diğer normal Unity’nin geri çağrısı (veya bir döngü) içinde gerçekleştirirken, bu yığın ayırmasız yöntemi kullanmalısınız:

if (go.CompareTag("myTag")
{ 
    //Stuff 
}

Ek olarak, etiketlerinizi statik bir sınıfta tutmak daha kolaydır.

public static class Tags
{
    public const string Player = "Player";
    public const string MyCustomTag = "MyCustomTag";
}

O zaman güvenle karşılaştırabilirsiniz

if (go.CompareTag(Tags.MyCustomTag)
{ 
    //Stuff 
}

bu şekilde, etiket dizeleriniz derleme zamanında oluşturulur ve yazım hatalarının sonuçlarını sınırlandırırsınız.

Etiketleri statik bir sınıfta tutmak gibi, onu bir numaralandırmada saklamak da mümkündür:

public enum Tags
{
    Player, Ennemies, MyCustomTag;
}

ve sonra bunu enum toString() yöntemini kullanarak karşılaştırabilirsiniz:

if (go.CompareTag(Tags.MyCustomTag.toString())
{ 
    //Stuff 
}
