SerileştirAlanı #

Unity’nin serileştirme sistemi aşağıdakileri yapmak için kullanılabilir:

  • kamuya açık statik olmayan alanları (serileştirilebilir türlerde) seri hale getirebilir
  • [SerializeField] özniteliği ile işaretlenmiş genel olmayan statik olmayan alanları serileştirebilir
  • Statik alanlar serileştirilemez
  • Statik özellikler serileştirilemiyor

Alanınız, SerializeField özniteliği ile işaretlenmiş olsa bile, yalnızca Unity’nin seri hale getirebileceği bir türdeyse ilişkilendirilecektir:

  • UnityEngine.Object’den devralan tüm sınıflar (ör. GameObject, Component, MonoBehaviour, Texture2D)
  • int, string, float, bool gibi tüm temel veri türleri
  • Vector2/3/4, Quaternion, Matrix4x4, Color, Rect, LayerMask gibi bazı yerleşik türler
  • Serileştirilebilir türden diziler
  • Serileştirilebilir bir türün listesi
  • Numaralandırmalar
  • Yapılar

Ortak denetçi özellikleri #

[Header( "My variables" )]
public string MyString;

[HideInInspector]
public string MyHiddenString;

[Multiline( 5 )]
public string MyMultilineString;

[TextArea( 2, 8 )]
public string MyTextArea;

[Space( 15 )]
public int MyInt;

[Range( 2.5f, 12.5f )]
public float MyFloat;

[Tooltip( "This is a tip for MyDouble" )]
public double MyDouble;

[SerializeField]
private double myHiddenDouble;

Bir alanın etiketinin üzerine geldiğinizde:

[Header( "My variables" )]
public string MyString;

Başlık, atfedilen alanın üstüne metni içeren kalın bir etiket yerleştirir. Bu, genellikle grupları diğer etiketlere karşı öne çıkarmak için etiketlemek için kullanılır.

[HideInInspector]
public string MyHiddenString;

HideInInspector, genel alanların denetçide gösterilmesini engeller. Bu, kodun başka bir şekilde görünür veya değiştirilebilir olmadığı diğer bölümlerinden alanlara erişmek için kullanışlıdır.

[Multiline( 5 )]
public string MyMultilineString;

Çok satırlı, belirtilen sayıda satır içeren bir metin kutusu oluşturur. Bu miktarın aşılması ne kutuyu genişletecek ne de metni saracaktır.

[TextArea( 2, 8 )]
public string MyTextArea;

TextArea, metin ayrılan alanı aşarsa otomatik sözcük kaydırma ve kaydırma çubukları ile çok satırlı metne izin verir.

[Space( 15 )]
public int MyInt;

Boşluk, denetçiyi önceki ve mevcut öğeler arasında fazladan boşluk eklemeye zorlar - grupları ayırt etmede ve ayırmada faydalıdır.

[Range( 2.5f, 12.5f )]
public float MyFloat;

Aralık, minimum ve maksimum arasında sayısal bir değer zorlar. Bu öznitelik, minimum ve maksimum değişkenler olarak belirtilmiş olsa da, tamsayılar ve çiftler üzerinde de çalışır.

[Tooltip( "This is a tip for MyDouble" )]
public double MyDouble;

Araç ipucu, alanın etiketi üzerine gelindiğinde ek bir açıklama gösterir.

[SerializeField]
private double myHiddenDouble;

SerializeField, Unity’yi alanı seri hale getirmeye zorlar - özel alanlar için kullanışlıdır.

[MenuItem] özelliğinin sonucu

[CanEditMultipleObjects]
public class AttributesExample : MonoBehaviour

CanEditMultipleObjects özelliği, bileşeninizdeki değerleri birden çok GameObjects üzerinde düzenlemenize olanak tanır. Bu bileşen olmadan, birden fazla GameObjects seçtiğinizde bileşeninizin normal gibi göründüğünü görmezsiniz, bunun yerine “Çoklu nesne düzenleme desteklenmiyor” mesajını görürsünüz.

Bu özellik, özel editörlerin çoklu düzenlemeyi desteklemesi içindir. Özel olmayan düzenleyiciler otomatik olarak çoklu düzenlemeyi destekler.

[PreferenceItem( "My Prefs" )]
public static void PreferencesGUI()

PreferenceItem özelliği, Unity’nin tercihler menüsünde fazladan bir öğe oluşturmanıza olanak tanır. Alma yönteminin kullanılabilmesi için statik olması gerekir.

[MenuItem( "Attributes/Foo" )]
private static void Foo()
{
    [...]
}

[MenuItem( "Attributes/Foo", true )]
private static bool FooValidate() 
{
    return false;
}

MenuItem niteliği, işlevleri yürütmek için özel menü öğeleri oluşturmanıza olanak tanır. Bu örnek, işlevin yürütülmesini önlemek için (her zaman false döndüren) bir doğrulayıcı işlevi de kullanır.

[CustomEditor( typeof( MyComponent ) )]
public class AttributesExample : Editor

}

CustomEditor özelliği, bileşenleriniz için özel düzenleyiciler oluşturmanıza olanak tanır. Bu editörler, bileşeninizi denetçide çizmek için kullanılacaktır ve Editor sınıfından türetilmesi gerekir.

[CustomPropertyDrawer( typeof( MyClass ) )]
public class AttributesExample : PropertyDrawer 
{
    [...]
}

CustomPropertyDrawer özniteliği, denetçide için özel bir özellik çekmecesi oluşturmanıza olanak tanır. Bu çekmeceleri, özel veri türleriniz için kullanabilirsiniz, böylece denetçide kullanılmış olarak görülebilirler.

[DrawGizmo( GizmoType.Selected )]
private static void DoGizmo( AttributesExample obj, GizmoType type )
{
    [...]
}

DrawGizmo özelliği, bileşenleriniz için özel gizmos çizmenize olanak tanır. Bu gizmoslar, Sahne Görünümünde çizilecektir. DrawGizmo özniteliğindeki GizmoType parametresini kullanarak gizmoyu ne zaman çizeceğinize karar verebilirsiniz.

Alma yöntemi iki parametre gerektirir, birincisi zımbayı çizecek bileşen ve ikincisi zımbırtıya ihtiyaç duyan nesnenin içinde bulunduğu durumdur.

Bileşen özellikleri #

[DisallowMultipleComponent]
[RequireComponent( typeof( Rigidbody ) )]
public class AttributesExample : MonoBehaviour 
{
    [...]
}

[DisallowMultipleComponent]

DisallowMultipleComponent özniteliği, kullanıcıların bu bileşenin birden çok örneğini bir GameObject’e eklemesini engeller.

[RequireComponent( typeof( Rigidbody ) )]

RequireComponent özniteliği, bu bileşen bir GameObject’e eklendiğinde gereksinimler olarak başka bir bileşen (veya daha fazla) belirtmenize izin verir. Bu bileşeni bir GameObject’e eklediğinizde, gerekli bileşenler (zaten mevcut değilse) otomatik olarak eklenir ve bu bileşenler, bunları gerektiren bileşenler kaldırılana kadar kaldırılamaz.

Çalışma zamanı özellikleri #

[ExecuteInEditMode]
public class AttributesExample : MonoBehaviour 
{

    [RuntimeInitializeOnLoadMethod]
    private static void FooBar() 
    {
        [...]
    }

    [RuntimeInitializeOnLoadMethod( RuntimeInitializeLoadType.BeforeSceneLoad )]
    private static void Foo() 
    {
        [...]
    }

    [RuntimeInitializeOnLoadMethod( RuntimeInitializeLoadType.AfterSceneLoad )]
    private static void Bar() 
    {
        [...]
    }

    void Update() 
    {
        if ( Application.isEditor )
        {
            [...]
        } 
        else
        {
            [...]
        }
    }
}

[ExecuteInEditMode]
public class AttributesExample : MonoBehaviour

ExecuteInEditMode özniteliği, Unity’yi oyun oynanmıyorken bile bu komut dosyasının sihirli yöntemlerini yürütmeye zorlar.

İşlevler, oynatma modunda olduğu gibi sürekli çağrılmaz

  • Güncelleme yalnızca sahnede bir şey değiştiğinde çağrılır.
  • Oyun Görünümü bir Etkinlik aldığında OnGUI çağrılır.
  • OnRenderObject ve diğer işleme geri çağırma işlevleri, Sahne Görünümünün veya Oyun Görünümünün her yeniden boyamasında çağrılır.
[RuntimeInitializeOnLoadMethod]
private static void FooBar()

[RuntimeInitializeOnLoadMethod( RuntimeInitializeLoadType.BeforeSceneLoad )]
private static void Foo() 

[RuntimeInitializeOnLoadMethod( RuntimeInitializeLoadType.AfterSceneLoad )]
private static void Bar()

RuntimeInitializeOnLoadMethod özniteliği, oyun çalışma zamanını yüklediğinde, kullanıcıdan herhangi bir etkileşim olmadan bir çalışma zamanı sınıf yönteminin çağrılmasına izin verir.

Yöntemin, sahne yüklemesinden önce mi yoksa sonra mı çağrılmasını istediğinizi belirtebilirsiniz (varsayılandan sonra). Bu özniteliği kullanan yöntemler için yürütme sırası garanti edilmez.
