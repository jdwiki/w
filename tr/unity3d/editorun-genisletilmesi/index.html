Özel Denetçi #

Özel bir denetçi kullanmak, Müfettiş’te bir komut dosyasının çizilme şeklini değiştirmenize olanak tanır. Bazen, komut dosyanız için denetçiye, özel bir özellik çekmecesiyle yapılması mümkün olmayan ek bilgiler eklemek istersiniz.

Aşağıda, özel bir denetçi kullanarak daha yararlı bilgiler gösterebilen özel bir nesnenin basit bir örneği verilmiştir.

using UnityEngine;
#if UNITY_EDITOR
using UnityEditor;
#endif

public class InspectorExample : MonoBehaviour {

    public int Level;
    public float BaseDamage;

    public float DamageBonus {
        get {
            return Level / 100f * 50;
        }
    }

    public float ActualDamage {
        get {
            return BaseDamage + DamageBonus;
        }
    }
}

#if UNITY_EDITOR
[CustomEditor( typeof( InspectorExample ) )]
public class CustomInspector : Editor {

    public override void OnInspectorGUI() {
        base.OnInspectorGUI();

        var ie = (InspectorExample)target;

        EditorGUILayout.LabelField( "Damage Bonus", ie.DamageBonus.ToString() );
        EditorGUILayout.LabelField( "Actual Damage", ie.ActualDamage.ToString() );
    }
}
#endif

İlk önce özel davranışımızı bazı alanlarla tanımlıyoruz

public class InspectorExample : MonoBehaviour {
    public int Level;
    public float BaseDamage;
}

Denetçi penceresinde komut dosyasını görüntülerken, yukarıda gösterilen alanlar otomatik olarak (özel denetçi olmadan) çizilir.

public float DamageBonus {
    get {
        return Level / 100f * 50;
    }
}

public float ActualDamage {
    get {
        return BaseDamage + DamageBonus;
    }
}

Bu özellikler Unity tarafından otomatik olarak çizilmez. Bu özellikleri Denetçi görünümünde göstermek için Özel Denetçimizi kullanmalıyız.

İlk önce özel denetçimizi bu şekilde tanımlamamız gerekiyor.

[CustomEditor( typeof( InspectorExample ) )]
public class CustomInspector : Editor {

Özel denetçinin Editor‘dan türetilmesi gerekir ve CustomEditor özniteliğine ihtiyacı vardır. Özniteliğin parametresi, bu özel denetçinin kullanılması gereken nesnenin türüdür.

Sıradaki OnInspectorGUI yöntemidir. Bu yöntem, komut dosyası denetçi penceresinde her gösterildiğinde çağrılır.

public override void OnInspectorGUI() {
    base.OnInspectorGUI();
}

Unity’nin komut dosyasındaki diğer alanları işlemesine izin vermek için base.OnInspectorGUI()‘ya bir çağrı yapıyoruz. Bunu aramasaydık, kendimiz daha çok iş yapmak zorunda kalırdık.

Sırada göstermek istediğimiz özel özelliklerimiz var

var ie = (InspectorExample)target;

EditorGUILayout.LabelField( "Damage Bonus", ie.DamageBonus.ToString() );
EditorGUILayout.LabelField( "Actual Damage", ie.ActualDamage.ToString() );

Özel türümüze atılan hedefi tutan geçici bir değişken oluşturmalıyız (hedef, Editör’den türediğimiz için kullanılabilir).

Daha sonra özelliklerimizi nasıl çizeceğimize karar verebiliriz, bu durumda sadece değerleri göstermek istediğimiz ve onları düzenleyemediğimiz için iki etiket alanı yeterlidir.

Özel Özellik Çekmecesi #

Bazen veri içeren ancak MonoBehaviour’dan türetilmeyen özel nesneleriniz olur. Bu nesneleri MonoBehaviour olan bir sınıfa alan olarak eklemek, nesnenin türü için kendi özel özellik çekmecenizi yazmadığınız sürece görsel bir etkiye sahip olmayacaktır.

Aşağıda, MonoBehaviour’a eklenen özel bir nesnenin basit bir örneği ve özel nesne için bir özel özellik çekmecesi bulunmaktadır.

public enum Gender {
    Male,
    Female,
    Other
}

// Needs the Serializable attribute otherwise the CustomPropertyDrawer wont be used
[Serializable]
public class UserInfo {
    public string Name;
    public int Age;
    public Gender Gender;
}

// The class that you can attach to a GameObject
public class PropertyDrawerExample : MonoBehaviour {
    public UserInfo UInfo;
}

[CustomPropertyDrawer( typeof( UserInfo ) )]
public class UserInfoDrawer : PropertyDrawer {

    public override float GetPropertyHeight( SerializedProperty property, GUIContent label ) {
        // The 6 comes from extra spacing between the fields (2px each)
        return EditorGUIUtility.singleLineHeight * 4 + 6;
    }

    public override void OnGUI( Rect position, SerializedProperty property, GUIContent label ) {
        EditorGUI.BeginProperty( position, label, property );

        EditorGUI.LabelField( position, label );

        var nameRect = new Rect( position.x, position.y + 18, position.width, 16 );
        var ageRect = new Rect( position.x, position.y + 36, position.width, 16 );
        var genderRect = new Rect( position.x, position.y + 54, position.width, 16 );

        EditorGUI.indentLevel++;

        EditorGUI.PropertyField( nameRect, property.FindPropertyRelative( "Name" ) );
        EditorGUI.PropertyField( ageRect, property.FindPropertyRelative( "Age" ) );
        EditorGUI.PropertyField( genderRect, property.FindPropertyRelative( "Gender" ) );

        EditorGUI.indentLevel--;

        EditorGUI.EndProperty();
    }
}

Öncelikle, özel nesneyi tüm gereksinimleriyle tanımlarız. Sadece bir kullanıcıyı tanımlayan basit bir sınıf. Bu sınıf, bir GameObject’e ekleyebileceğimiz PropertyDrawerExample sınıfımızda kullanılmaktadır.

public enum Gender {
    Male,
    Female,
    Other
}

[Serializable]
public class UserInfo {
    public string Name;
    public int Age;
    public Gender Gender;
}

public class PropertyDrawerExample : MonoBehaviour {
    public UserInfo UInfo;
}

Özel sınıfın Serileştirilebilir özniteliğine ihtiyacı vardır, aksi takdirde CustomPropertyDrawer kullanılmaz

Sıradaki CustomPropertyDrawer

İlk önce PropertyDrawer’dan türetilen bir sınıf tanımlamamız gerekiyor. Sınıf tanımı ayrıca CustomPropertyDrawer özniteliğine ihtiyaç duyar. Geçirilen parametre, bu çekmecenin kullanılmasını istediğiniz nesnenin türüdür.

[CustomPropertyDrawer( typeof( UserInfo ) )]
public class UserInfoDrawer : PropertyDrawer {

Sonra GetPropertyHeight işlevini geçersiz kılıyoruz. Bu, mülkümüz için özel bir yükseklik tanımlamamızı sağlar. Bu durumda, mülkümüzün dört bölümden oluşacağını biliyoruz: etiket, ad, yaş ve cinsiyet. Bu nedenle EditorGUIUtility.singleLineHeight * 4 kullanıyoruz, her alanı aralarında iki piksel olacak şekilde boşluk bırakmak istediğimiz için 6 piksel daha ekliyoruz.

public override float GetPropertyHeight( SerializedProperty property, GUIContent label ) {
    return EditorGUIUtility.singleLineHeight * 4 + 6;
}

Sonraki, gerçek OnGUI yöntemidir. EditorGUI.BeginProperty([…]) ile başlıyoruz ve EditorGUI.EndProperty() ile fonksiyonu sonlandırıyoruz. Bunu, bu özellik bir prefabrikin parçası olacaksa, gerçek prefabrik geçersiz kılma mantığının bu iki yöntem arasındaki her şey için işe yaraması için yapıyoruz.

public override void OnGUI( Rect position, SerializedProperty property, GUIContent label ) {
    EditorGUI.BeginProperty( position, label, property );

Bundan sonra alanın adını içeren bir etiket gösteriyoruz ve alanlarımız için zaten dikdörtgenleri tanımlıyoruz.

EditorGUI.LabelField( position, label );

var nameRect = new Rect( position.x, position.y + 18, position.width, 16 );
var ageRect = new Rect( position.x, position.y + 36, position.width, 16 );
var genderRect = new Rect( position.x, position.y + 54, position.width, 16 );

Her alan 16 + 2 piksel aralıklıdır ve yükseklik 16’dır (ki bu EditorGUIUtility.singleLineHeight ile aynıdır)

Daha sonra, biraz daha güzel bir düzen için kullanıcı arayüzünü bir sekmeyle girintili hale getiriyoruz, özellikleri gösteriyoruz, GUI’nin girintisini kaldırıyoruz ve EditorGUI.EndProperty ile sonlandırıyoruz.

EditorGUI.indentLevel++;

EditorGUI.PropertyField( nameRect, property.FindPropertyRelative( "Name" ) );
EditorGUI.PropertyField( ageRect, property.FindPropertyRelative( "Age" ) );
EditorGUI.PropertyField( genderRect, property.FindPropertyRelative( "Gender" ) );

EditorGUI.indentLevel--;

EditorGUI.EndProperty();

Alanları, konum için bir dikdörtgen ve özelliğin gösterilmesi için bir SerializedProperty gerektiren EditorGUI.PropertyField kullanarak görüntüleriz. OnGUI işlevinde geçirilen özellik üzerinde FindPropertyRelative("…") öğesini çağırarak özelliği elde ederiz. Bunların büyük/küçük harfe duyarlı olduğunu ve halka açık olmayan mülklerin bulunamayacağını unutmayın!

Bu örnek için, property.FindPropertyRelative("…") öğesinden dönen özellikleri kaydetmiyorum. Gereksiz aramaları önlemek için bunları sınıfta özel alanlara kaydetmelisiniz.

Menü Öğeleri #

Menü öğeleri, düzenleyiciye özel eylemler eklemenin harika bir yoludur. Menü çubuğuna menü öğeleri ekleyebilir, belirli bileşenlere bağlam tıklamaları olarak veya hatta komut dosyalarınızdaki alanlara bağlam tıklamaları olarak sahip olabilirsiniz.

Aşağıda menü öğelerini nasıl uygulayabileceğinize ilişkin bir örnek verilmiştir.

public class MenuItemsExample : MonoBehaviour {

    [MenuItem( "Example/DoSomething %#&d" )]
    private static void DoSomething() {
        // Execute some code
    }

    [MenuItem( "Example/DoAnotherThing", true )]
    private static bool DoAnotherThingValidator() {
        return Selection.gameObjects.Length > 0;
    }

    [MenuItem( "Example/DoAnotherThing _PGUP", false )]
    private static void DoAnotherThing() {
        // Execute some code
    }

    [MenuItem( "Example/DoOne %a", false, 1 )]
    private static void DoOne() {
        // Execute some code
    }

    [MenuItem( "Example/DoTwo #b", false, 2 )]
    private static void DoTwo() {
        // Execute some code
    }

    [MenuItem( "Example/DoFurther &c", false, 13 )]
    private static void DoFurther() {
        // Execute some code
    }

    [MenuItem( "CONTEXT/Camera/DoCameraThing" )]
    private static void DoCameraThing( MenuCommand cmd ) {
        // Execute some code
    }

    [ContextMenu( "ContextSomething" )]
    private void ContentSomething() {
        // Execute some code
    }

    [ContextMenuItem( "Reset", "ResetDate" )]
    [ContextMenuItem( "Set to Now", "SetDateToNow" )]
    public string Date = "";

    public void ResetDate() {
        Date = "";
    }

    public void SetDateToNow() {
        Date = DateTime.Now.ToString();
    }
}

Hangisi buna benziyor

Temel menü öğesinin üzerinden geçelim. Aşağıda görebileceğiniz gibi, menü öğesinin başlığı olarak bir dize ilettiğiniz MenuItem özniteliği ile statik bir işlev tanımlamanız gerekir. Adına bir / ekleyerek menü öğenizi birden çok düzey derinliğe yerleştirebilirsiniz.

[MenuItem( "Example/DoSomething %#&d" )]
private static void DoSomething() {
    // Execute some code
}

En üst düzeyde bir menü öğeniz olamaz. Menü öğelerinizin bir alt menüde olması gerekir!

MenuItem adının sonundaki özel karakterler kısayol tuşları içindir, bunlar bir gereklilik değildir.

Kısayol tuşlarınız için kullanabileceğiniz özel karakterler vardır, bunlar:

  • % - Windows’ta Ctrl, OS X’te Cmd
  • # - Vardiya
  • & - Alt

Bu, %#&d kısayolunun Windows’ta ctrl+shift+alt+D ve OS X’te cmd+shift+alt+D anlamına geldiği anlamına gelir.

Herhangi bir özel tuş olmadan bir kısayol kullanmak istiyorsanız, örneğin sadece ‘D’ tuşu, kullanmak istediğiniz kısayol tuşunun başına _ (alt çizgi) karakterini ekleyebilirsiniz.

Desteklenen başka özel anahtarlar da vardır:

  • SOL, SAĞ, YUKARI, AŞAĞI - ok tuşları için
  • F1..F12 - fonksiyon tuşları için
  • HOME, END, PGUP, PGDN - gezinme tuşları için

Kısayol tuşlarının diğer metinlerden boşlukla ayrılması gerekir

Sonraki doğrulayıcı menü öğeleridir. Doğrulayıcı menü öğeleri, koşul karşılanmadığında menü öğelerinin devre dışı bırakılmasına (grileştirilmiş, tıklanamaz) izin verir. Buna bir örnek, menü öğenizin, validator menü öğesinde kontrol edebileceğiniz geçerli GameObjects seçimine etki etmesi olabilir.

[MenuItem( "Example/DoAnotherThing", true )]
private static bool DoAnotherThingValidator() {
    return Selection.gameObjects.Length > 0;
}

[MenuItem( "Example/DoAnotherThing _PGUP", false )]
private static void DoAnotherThing() {
    // Execute some code
}

Doğrulayıcı menü öğesinin çalışması için hem MenuItem özniteliği hem de aynı ada sahip (kısayol tuşu önemli değil) iki statik işlev oluşturmanız gerekir. Aralarındaki fark, bunları bir boole parametresi ileterek doğrulama işlevi olarak işaretlemeniz veya işaretlememenizdir.

Menü öğelerinin sırasını bir öncelik ekleyerek de tanımlayabilirsiniz. Öncelik, üçüncü parametre olarak ilettiğiniz bir tam sayı ile tanımlanır. Rakam ne kadar küçükse listede o kadar üstte, sayı o kadar büyükse listede o kadar altta. Menü öğelerinin önceliği arasında en az 10 hane olduğundan emin olarak iki menü öğesi arasına bir ayırıcı ekleyebilirsiniz.

[MenuItem( "Example/DoOne %a", false, 1 )]
private static void DoOne() {
    // Execute some code
}

[MenuItem( "Example/DoTwo #b", false, 2 )]
private static void DoTwo() {
    // Execute some code
}

[MenuItem( "Example/DoFurther &c", false, 13 )]
private static void DoFurther() {
    // Execute some code
}

Öncelikli ve öncelikli olmayan öğelerin bir kombinasyonuna sahip bir menü listeniz varsa, öncelikli olmayan öğeler öncelikli öğelerden ayrılacaktır.

Sonraki, zaten var olan bir bileşenin bağlam menüsüne bir menü öğesi eklemektir. MenuItem adını CONTEXT (büyük/küçük harf duyarlı) ile başlatmanız ve işlevinizin bir MenuCommand parametresini almasını sağlamanız gerekir.

Aşağıdaki kod parçası, Kamera bileşenine bir bağlam menüsü öğesi ekleyecektir.

[MenuItem( "CONTEXT/Camera/DoCameraThing" )]
private static void DoCameraThing( MenuCommand cmd ) {
    // Execute some code
}

Hangisi buna benziyor

MenuCommand parametresi, bileşen değerine ve onunla birlikte gönderilen tüm kullanıcı verilerine erişmenizi sağlar.

ContextMenu özniteliğini kullanarak kendi bileşenlerinize bir bağlam menüsü öğesi de ekleyebilirsiniz. Bu öznitelik yalnızca bir ad alır, doğrulama veya öncelik yoktur ve statik olmayan bir yöntemin parçası olmalıdır.

[ContextMenu( "ContextSomething" )]
private void ContentSomething() {
    // Execute some code
}

Hangisi buna benziyor

Kendi bileşeninizdeki alanlara bağlam menüsü öğeleri de ekleyebilirsiniz. Bu menü öğeleri, ait oldukları alana bağlam tıkladığınızda görünür ve o bileşende tanımladığınız yöntemleri çalıştırabilir. Bu şekilde, aşağıda gösterildiği gibi örneğin varsayılan değerleri veya geçerli tarihi ekleyebilirsiniz.

[ContextMenuItem( "Reset", "ResetDate" )]
[ContextMenuItem( "Set to Now", "SetDateToNow" )]
public string Date = "";

public void ResetDate() {
    Date = "";
}

public void SetDateToNow() {
    Date = DateTime.Now.ToString();
}

Neden Düzenleyici Penceresi? #

Gördüğünüz gibi, bir özel denetçide birçok şey yapabilirsiniz (özel denetçinin ne olduğunu bilmiyorsanız, buradaki örneğe bakın: https://www.wikiod.com/tr/unity3d/editorun-genisletilmesi -editör). Ancak bir noktada bir yapılandırma paneli veya özelleştirilmiş bir varlık paleti uygulamak isteyebilirsiniz. Bu durumlarda bir EditorWindow kullanacaksınız. Unity UI’nin kendisi Editör Windows’tan oluşur; bunları açabilir (genellikle üst çubuktan), sekmeye vb.

Temel bir EditorWindow oluşturun #

Basit Örnek #

Özel bir düzenleyici penceresi oluşturmak oldukça basittir. Tek yapmanız gereken EditorWindow sınıfını genişletmek ve Init() ve OnGUI() yöntemlerini kullanmak. İşte basit bir örnek :

using UnityEngine;
using UnityEditor;

public class CustomWindow : EditorWindow
{
    // Add menu named "Custom Window" to the Window menu
    [MenuItem("Window/Custom Window")]
    static void Init()
    {
        // Get existing open window or if none, make a new one:
        CustomWindow window = (CustomWindow) EditorWindow.GetWindow(typeof(CustomWindow));
        window.Show();
    }

    void OnGUI()
    {
        GUILayout.Label("This is a custom Editor Window", EditorStyles.boldLabel);
    }
}

3 önemli nokta şunlardır:

  1. EditorWindow’u genişletmeyi unutmayın
  2. Örnekte belirtildiği gibi Init()‘i kullanın. EditorWindow.GetWindow, bir CustomWindow’un zaten oluşturulup oluşturulmadığını kontrol ediyor. Değilse, yeni bir örnek oluşturacaktır. Bunu kullanarak, aynı anda pencerenizin birkaç örneğine sahip olmadığınızdan emin olursunuz.
  3. Pencerenizde bilgileri görüntülemek için her zamanki gibi OnGUI() kullanın

Daha derine inmek #

Elbette bu EditorWindow’u kullanarak bazı varlıkları yönetmek veya değiştirmek isteyeceksiniz. Selection sınıfını (etkin Seçimi almak için) kullanan ve seçilen varlık özelliklerini SerializedObject ve SerializedProperty aracılığıyla değiştiren bir örnek.

    using System.Linq;
    using UnityEngine;
    using UnityEditor;
    
    public class CustomWindow : EditorWindow
    {
        private AnimationClip _animationClip;
        private SerializedObject _serializedClip;
        private SerializedProperty _events;
    
        private string _text = "Hello World";
    
        // Add menu named "Custom Window" to the Window menu
        [MenuItem("Window/Custom Window")]
        static void Init()
        {
            // Get existing open window or if none, make a new one:
            CustomWindow window = (CustomWindow) EditorWindow.GetWindow(typeof(CustomWindow));
            window.Show();
        }
    
        void OnGUI()
        {
            GUILayout.Label("This is a custom Editor Window", EditorStyles.boldLabel);
    
            // You can use EditorGUI, EditorGUILayout and GUILayout classes to display anything you want
            // A TextField example
            _text = EditorGUILayout.TextField("Text Field", _text);
    
            // Note that you can modify an asset or a gameobject using an EditorWindow. Here is a quick example with an AnimationClip asset
            // The _animationClip, _serializedClip and _events are set in OnSelectionChange()
    
            if (_animationClip == null || _serializedClip == null || _events == null) return;
    
            // We can modify our serializedClip like we would do in a Custom Inspector. For example we can grab its events and display their information
    
            GUILayout.Label(_animationClip.name, EditorStyles.boldLabel);
    
            for (var i = 0; i < _events.arraySize; i++)
            {
                EditorGUILayout.BeginVertical();
    
                EditorGUILayout.LabelField(
                    "Event : " + _events.GetArrayElementAtIndex(i).FindPropertyRelative("functionName").stringValue,
                    EditorStyles.boldLabel);
                EditorGUILayout.PropertyField(_events.GetArrayElementAtIndex(i).FindPropertyRelative("time"), true,
                    GUILayout.ExpandWidth(true));
                EditorGUILayout.PropertyField(_events.GetArrayElementAtIndex(i).FindPropertyRelative("functionName"),
                    true, GUILayout.ExpandWidth(true));
                EditorGUILayout.PropertyField(_events.GetArrayElementAtIndex(i).FindPropertyRelative("floatParameter"),
                    true, GUILayout.ExpandWidth(true));
                EditorGUILayout.PropertyField(_events.GetArrayElementAtIndex(i).FindPropertyRelative("intParameter"),
                    true, GUILayout.ExpandWidth(true));
                EditorGUILayout.PropertyField(
                    _events.GetArrayElementAtIndex(i).FindPropertyRelative("objectReferenceParameter"), true,
                    GUILayout.ExpandWidth(true));
    
                EditorGUILayout.Separator();
                EditorGUILayout.EndVertical();
            }
    
            // Of course we need to Apply the modified properties. We don't our changes won't be saved
            _serializedClip.ApplyModifiedProperties();
        }
    
        /// This Message is triggered when the user selection in the editor changes. That's when we should tell our Window to Repaint() if the user selected another AnimationClip
        private void OnSelectionChange()
        {
            _animationClip =
                Selection.GetFiltered(typeof(AnimationClip), SelectionMode.Assets).FirstOrDefault() as AnimationClip;
            if (_animationClip == null) return;
    
            _serializedClip = new SerializedObject(_animationClip);
            _events = _serializedClip.FindProperty("m_Events");
            Repaint();
        }
    }

Gelişmiş konular #

Düzenleyicide gerçekten gelişmiş bazı şeyler yapabilirsiniz ve EditorWindow sınıfı büyük miktarda bilgiyi görüntülemek için mükemmeldir. Unity Asset Store’daki (NodeCanvas veya PlayMaker gibi) en gelişmiş varlıklar, özel görünümleri görüntülemek için EditorWindow’u kullanır.

SceneView’da Çizim #

EditorWindow ile yapılacak ilginç bir şey, bilgileri doğrudan SceneView’da görüntülemektir. Bu yolla, örneğin özel EditorWindow’unuzu bir varlık paleti olarak kullanarak ve yeni nesneleri başlatmak için SceneView’daki tıklamaları dinleyerek, tamamen özelleştirilmiş bir harita/dünya düzenleyicisi oluşturabilirsiniz. İşte bir örnek :

using UnityEngine;
using System;
using UnityEditor;

public class CustomWindow : EditorWindow {

    private enum Mode {
        View = 0,
        Paint = 1,
        Erase = 2
    }

    private Mode CurrentMode = Mode.View;

    [MenuItem ("Window/Custom Window")]
    static void Init () {
        // Get existing open window or if none, make a new one:
        CustomWindow window = (CustomWindow)EditorWindow.GetWindow (typeof (CustomWindow));
        window.Show();
    }

    void OnGUI () {
        GUILayout.Label ("This is a custom Editor Window", EditorStyles.boldLabel);
    }

    void OnEnable() {
        SceneView.onSceneGUIDelegate = SceneViewGUI;
        if (SceneView.lastActiveSceneView) SceneView.lastActiveSceneView.Repaint();
    }

    void SceneViewGUI(SceneView sceneView) {
        Handles.BeginGUI();
        // We define the toolbars' rects here
        var ToolBarRect = new Rect((SceneView.lastActiveSceneView.camera.pixelRect.width / 6), 10, (SceneView.lastActiveSceneView.camera.pixelRect.width * 4 / 6) , SceneView.lastActiveSceneView.camera.pixelRect.height / 5);
        GUILayout.BeginArea(ToolBarRect);
        GUILayout.BeginHorizontal();
        GUILayout.FlexibleSpace();
         CurrentMode = (Mode) GUILayout.Toolbar(
            (int) CurrentMode,
            Enum.GetNames(typeof(Mode)),
            GUILayout.Height(ToolBarRect.height));
        GUILayout.FlexibleSpace();
        GUILayout.EndHorizontal();
        GUILayout.EndArea();
        Handles.EndGUI();
    }
}

Bu, bir araç çubuğunu doğrudan SceneView içinde görüntüler.

Gizmolar #

Gizmos, sahne görünümünde şekil çizmek için kullanılır. Bu şekilleri GameObject’leriniz hakkında, örneğin sahip oldukları kesiklik veya algılama aralığı gibi ek bilgiler çizmek için kullanabilirsiniz.

Aşağıda bunun nasıl yapılacağına dair iki örnek verilmiştir.

Örnek Bir #

Bu örnekte OnDrawGizmos ve OnDrawGizmosSelected (magic) yöntemleri kullanılmaktadır.

public class GizmoExample : MonoBehaviour {

    public float GetDetectionRadius() {
        return 12.5f;
    }

    public float GetFOV() {
        return 25f;
    }

    public float GetMaxRange() {
        return 6.5f;
    }

    public float GetMinRange() {
        return 0;
    }

    public float GetAspect() {
        return 2.5f;
    }

    public void OnDrawGizmos() {
        var gizmoMatrix = Gizmos.matrix;
        var gizmoColor = Gizmos.color;

        Gizmos.matrix = Matrix4x4.TRS( transform.position, transform.rotation, transform.lossyScale );
        Gizmos.color = Color.red;
        Gizmos.DrawFrustum( Vector3.zero, GetFOV(), GetMaxRange(), GetMinRange(), GetAspect() );

        Gizmos.matrix = gizmoMatrix;
        Gizmos.color = gizmoColor;
    }

    public void OnDrawGizmosSelected() {
        Handles.DrawWireDisc( transform.position, Vector3.up, GetDetectionRadius() );
    }
}

Bu örnekte, gizmos çizmek için iki yöntemimiz var, biri nesne aktifken çizen (OnDrawGizmos) ve diğeri nesne hiyerarşide seçildiğinde (OnDrawGizmosSelected).

public void OnDrawGizmos() {
    var gizmoMatrix = Gizmos.matrix;
    var gizmoColor = Gizmos.color;

    Gizmos.matrix = Matrix4x4.TRS( transform.position, transform.rotation, transform.lossyScale );
    Gizmos.color = Color.red;
    Gizmos.DrawFrustum( Vector3.zero, GetFOV(), GetMaxRange(), GetMinRange(), GetAspect() );

    Gizmos.matrix = gizmoMatrix;
    Gizmos.color = gizmoColor;
}

İlk önce gizmo matrisini ve rengini kaydediyoruz çünkü değiştireceğiz ve işimiz bittiğinde başka bir gizmo çizimini etkilememek için geri almak istiyoruz.

Daha sonra, nesnemizin sahip olduğu frustum’u çizmek istiyoruz, ancak Gizmos’ matrisini konum, döndürme ve ölçekle eşleşecek şekilde değiştirmemiz gerekiyor. Frustum’u vurgulamak için Gizmos’un rengini de kırmızıya ayarladık. Bu yapıldığında, sahne görünümünde frustum çizmek için Gizmos.DrawFrustum‘u çağırabiliriz.

Çizmek istediğimizi çizmeyi bitirdiğimizde Gizmos’un matrisini ve rengini sıfırlıyoruz.

public void OnDrawGizmosSelected() {
    Handles.DrawWireDisc( transform.position, Vector3.up, GetDetectionRadius() );
}

Ayrıca GameObject’imizi seçtiğimizde bir algılama aralığı çizmek istiyoruz. Bu, Gizmos sınıfının diskler için herhangi bir yöntemi olmadığından Handles sınıfı aracılığıyla yapılır.

Bu çizim biçimini kullanmak, aşağıda gösterilen çıktıya yol açar.

Örnek iki #

Bu örnek, DrawGizmo özniteliğini kullanır.

public class GizmoDrawerExample {

    [DrawGizmo( GizmoType.Selected | GizmoType.NonSelected, typeof( GizmoExample ) )]
    public static void DrawGizmo( GizmoExample obj, GizmoType type ) {
        var gizmoMatrix = Gizmos.matrix;
        var gizmoColor = Gizmos.color;

        Gizmos.matrix = Matrix4x4.TRS( obj.transform.position, obj.transform.rotation, obj.transform.lossyScale );
        Gizmos.color = Color.red;
        Gizmos.DrawFrustum( Vector3.zero, obj.GetFOV(), obj.GetMaxRange(), obj.GetMinRange(), obj.GetAspect() );

        Gizmos.matrix = gizmoMatrix;
        Gizmos.color = gizmoColor;

        if ( ( type & GizmoType.Selected ) == GizmoType.Selected ) {
            Handles.DrawWireDisc( obj.transform.position, Vector3.up, obj.GetDetectionRadius() );
        }
    }
}

Bu yol, gizmo çağrılarını betiğinizden ayırmanıza izin verir. Bunun çoğu, iki şey dışında diğer örnekle aynı kodu kullanır.

[DrawGizmo( GizmoType.Selected | GizmoType.NonSelected, typeof( GizmoExample ) )]
public static void DrawGizmo( GizmoExample obj, GizmoType type ) {

İlk parametre olarak enum GizmoType’ı ve ikinci parametre olarak bir Type’ı alan DrawGizmo niteliğini kullanmanız gerekir. Type, gizmoyu çizmek için kullanmak istediğiniz tip olmalıdır.

Gizmo çizme yönteminin statik, genel veya genel olmayan olması gerekir ve istediğiniz şekilde adlandırılabilir. İlk parametre, öznitelikte ikinci parametre olarak iletilen türle eşleşmesi gereken türdür ve ikinci parametre, nesnenizin mevcut durumunu tanımlayan enum GizmoType’dır.

if ( ( type & GizmoType.Selected ) == GizmoType.Selected ) {
    Handles.DrawWireDisc( obj.transform.position, Vector3.up, obj.GetDetectionRadius() );
}

Diğer fark, nesnenin GizmoType’ının ne olduğunu kontrol etmek için parametre ve istediğiniz tür üzerinde bir AND kontrolü yapmanız gerektiğidir.
