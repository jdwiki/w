Sözdizimi #

  • genel Coroutine StartCoroutine(IEnumerator rutini);
  • public Coroutine StartCoroutine(string methodName, nesne değeri = null);
  • public void StopCoroutine(string methodName);
  • public void StopCoroutine(IEnumerator rutini);
  • genel geçersiz StopAllCoroutines();

Performans konuları #

Esneklik bir performans maliyeti ile birlikte geldiğinden, eşyordamları ölçülü kullanmak en iyisidir.

  • Çok sayıda eşyordam, CPU’dan standart Güncelleme yöntemlerinden daha fazlasını talep eder.
  • Unity’nin bazı sürümlerinde, Unity’nin MoveNext dönüş değerini kutulaması nedeniyle, coroutinlerin her güncelleme döngüsünde çöp ürettiği bir sorun var. Bu en son 5.4.0b13’te gözlemlendi. (Hata raporu)

YieldInstructions’ı önbelleğe alarak çöpü azaltın #

Eşyordamlarda üretilen çöpü azaltmak için yaygın bir hile, ‘YieldInstruction’ı önbelleğe almaktır.

IEnumerator TickEverySecond()
{
    var wait = new WaitForSeconds(1f); // Cache
    while(true)
    {
        yield return wait; // Reuse
    }
}

’null’ verimi fazladan çöp üretmez.

eşyordamlar #

Öncelikle, oyun motorlarının (Unity gibi) “çerçeve tabanlı” bir paradigma üzerinde çalıştığını anlamak önemlidir.

Kod her çerçeve sırasında yürütülür.

Bu, Unity’nin kendi kodunu ve kodunuzu içerir.

Çerçeveler hakkında düşünürken, çerçevelerin ne zaman olacağına dair kesinlikle hiçbir garanti olmadığını anlamak önemlidir. Normal bir vuruşta *** olmazlar. Çerçeveler arasındaki boşluklar örneğin 0.02632 sonra 0.021167 sonra 0.029778 vb. olabilir. Örnekte hepsi saniyenin 1/50’si kadardır, ancak hepsi farklıdır. Ve istediğiniz zaman, çok daha uzun veya daha kısa süren bir kare elde edebilirsiniz; ve kodunuz çerçeve içinde herhangi bir zamanda yürütülebilir.

Bunu akılda tutarak, şunu sorabilirsiniz: Unity’de kodunuzdaki bu çerçevelere nasıl erişirsiniz?

Oldukça basit, ya Update() çağrısını kullanırsınız ya da bir eşyordam kullanırsınız. (Aslında - bunlar tamamen aynı şeydir: her karede kodun çalıştırılmasına izin verirler.)

Bir eşyordamın amacı şudur:

biraz kod çalıştırabilir ve ardından “dur ve bekle” gelecekteki bir çerçeveye kadar.

Bir sonraki kareye kadar bekleyebilir, birkaç kare bekleyebilir veya gelecekte saniye cinsinden yaklaşık bir süre bekleyebilirsiniz.

Örneğin, “yaklaşık bir saniye” bekleyebilirsiniz, yani yaklaşık bir saniye bekleyecek ve ardından kodunuzu yaklaşık bir saniye sonra bir çerçeveye koyacaksınız. (Ve gerçekten de, bu çerçeve içinde, kod herhangi bir zamanda çalıştırılabilir.) Tekrarlamak gerekirse: tam olarak bir saniye olmayacak. Bir oyun motorunda doğru zamanlama anlamsızdır.

Bir eşyordamın içinde:

Bir kare beklemek için:

// do something
yield return null;  // wait until next frame
// do something

Üç kare beklemek için:

// do something
yield return null;  // wait until three frames from now
yield return null;
yield return null;
// do something

Yaklaşık yarım saniye beklemek için:

// do something
yield return new WaitForSeconds (0.5f); // wait for a frame in about .5 seconds
// do something

Her karede bir şeyler yapın:

while (true)
{
    // do something
    yield return null;  // wait until the next frame
}

Bu örnek, Unity’nin “Güncelleme” çağrısının içine basitçe bir şey koymakla tam anlamıyla aynıdır: “bir şeyler yap"daki kod her karede çalıştırılır.

Örnek #

Bir ‘GameObject’ öğesine Ticker ekleyin. Bu oyun nesnesi aktifken kene çalışır. Oyun nesnesi devre dışı kaldığında komut dosyasının eşyordamı dikkatlice durdurduğunu unutmayın; bu genellikle eşyordam kullanımının doğru mühendisliğinin önemli bir yönüdür.

using UnityEngine;
using System.Collections;

public class Ticker:MonoBehaviour {

    void OnEnable()
    {
        StartCoroutine(TickEverySecond());
    }

    void OnDisable()
    {
        StopAllCoroutines();
    }

    IEnumerator TickEverySecond()
    {
        var wait = new WaitForSeconds(1f); // REMEMBER: IT IS ONLY APPROXIMATE
        while(true)
        {
            Debug.Log("Tick");
            yield return wait;  // wait for a frame, about 1 second from now
        }
    }
}

Zincirleme eşyordamları #

Eşyordamlar kendi içlerinde teslim olabilir ve diğer eşyordamları bekleyebilir.

Böylece dizileri zincirleyebilirsiniz - “birbiri ardına”.

Bu çok kolaydır ve Unity’de temel, çekirdek bir tekniktir.

Oyunlarda bazı şeylerin “sırasıyla” olması kesinlikle doğaldır. Bir oyunun hemen hemen her “turu”, belirli bir zaman aralığında, belirli bir sırayla gerçekleşen belirli bir dizi olayla başlar. Bir araba yarışı oyununa şu şekilde başlayabilirsiniz:

IEnumerator BeginRace()
{
  yield return StartCoroutine(PrepareRace());
  yield return StartCoroutine(Countdown());
  yield return StartCoroutine(StartRace());
}

Yani BeginRace’i aradığınızda…

 StartCoroutine(BeginRace());

“Yarışı hazırla” rutininizi yürütecek. (Belki, bazı ışıkların yanıp sönmesi ve biraz kalabalık gürültüsünün çalıştırılması, puanların sıfırlanması vb.) Bu bittiğinde, kullanıcı arayüzünde belki bir geri sayımı canlandıracağınız “geri sayım” dizinizi çalıştıracaktır. Bu bittiğinde, belki ses efektleri çalıştıracağınız, bazı AI sürücülerini başlatacağınız, kamerayı belirli bir şekilde hareket ettireceğiniz ve benzeri yerlerde yarış başlatma kodunuzu çalıştıracaktır.

Netlik için, üç çağrının

  yield return StartCoroutine(PrepareRace());
  yield return StartCoroutine(Countdown());
  yield return StartCoroutine(StartRace());

kendileri bir eşyordamın içinde olmalıdır. Yani, bunlar ‘IEnumerator’ tipinde bir fonksiyonda olmalıdırlar. Örneğimizde bu IEnumerator BeginRace. Böylece, “normal” koddan, o eşyordamı ‘StartCoroutine’ çağrısıyla başlatırsınız.

 StartCoroutine(BeginRace());

Zincirlemeyi daha iyi anlamak için, işte eşyordamları zincirleyen bir fonksiyon. Bir dizi eşyordamı geçersiniz. İşlev, birbiri ardına sırayla, geçtiğiniz kadar çok eşyordamı çalıştırır.

// run various routines, one after the other
IEnumerator OneAfterTheOther( params IEnumerator[] routines ) 
{
    foreach ( var item in routines ) 
    {
        while ( item.MoveNext() ) yield return item.Current;
    }

    yield break;
}

İşte buna nasıl diyorsunuz… diyelim ki üç fonksiyonunuz var. Hepsinin “IEnumerator” olması gerektiğini hatırlayın:

IEnumerator PrepareRace() 
{
    // codesay, crowd cheering and camera pan around the stadium
    yield break;
}

IEnumerator Countdown() 
{
    // codesay, animate your countdown on UI
    yield break;
}

IEnumerator StartRace() 
{
    // codesay, camera moves and light changes and launch the AIs
    yield break;
}

buna böyle derdin

StartCoroutine( MultipleRoutines( PrepareRace(), Countdown(), StartRace() ) );

ya da belki böyle

IEnumerator[] routines = new IEnumerator[] {
     PrepareRace(),
     Countdown(),
     StartRace() };
StartCoroutine( MultipleRoutines( routines ) );

Tekrarlamak gerekirse, oyunlardaki en temel gereksinimlerden biri, belirli şeylerin zaman içinde “sırasıyla” arka arkaya gerçekleşmesidir. Bunu Unity’de çok basit bir şekilde elde edersiniz,

  yield return StartCoroutine(PrepareRace());
  yield return StartCoroutine(Countdown());
  yield return StartCoroutine(StartRace());

Bir eşyordamı bitirmek #

Çoğu zaman, belirli hedeflere ulaşıldığında doğal olarak sona erecek şekilde eşyordamlar tasarlarsınız.

IEnumerator TickFiveSeconds()
{
    var wait = new WaitForSeconds(1f);
    int counter = 1;
    while(counter < 5)
    {
        Debug.Log("Tick");
        counter++;
        yield return wait;
    }
    Debug.Log("I am done ticking");
}

Bir eşyordamı eşyordamın “içerisinden” durdurmak için, sıradan bir işlevden erken ayrılmak gibi basitçe “geri dönemezsiniz”. Bunun yerine, “verim molası” kullanırsınız.

IEnumerator ShowExplosions()
{
    ... show basic explosions
    if(player.xp < 100) yield break;
    ... show fancy explosions
}

Ayrıca, komut dosyası tarafından başlatılan tüm eşyordamları bitirmeden önce durmaya zorlayabilirsiniz.

void OnDisable()
{
    // Stops all running coroutines
    StopAllCoroutines();
}

Arayandan bir belirli eşyordamı durdurma yöntemi, onu nasıl başlattığınıza bağlı olarak değişir.

Dize adıyla bir eşyordam başlattıysanız:

StartCoroutine("YourAnimation");

daha sonra aynı dize adıyla StopCoroutine numaralı telefonu arayarak durdurabilirsiniz:

StopCoroutine("YourAnimation");

Alternatif olarak, eşyordam yöntemi tarafından döndürülen ‘IEnumerator’a, _veya ‘StartCoroutine’ tarafından döndürülen ‘Coroutine’ nesnesine _ya da bir başvuru tutabilir ve bunlardan herhangi birinde ‘StopCoroutine’i çağırabilirsiniz:

public class SomeComponent : MonoBehaviour 
{
    Coroutine routine;

    void Start () {
        routine = StartCoroutine(YourAnimation());
    }

    void Update () {
        // later, in response to some input...
        StopCoroutine(routine);
    }

    IEnumerator YourAnimation () { /* ... */ }
}

Eşyordamlar olabilecek MonoBehaviour yöntemleri #

Eşyordamlar yapılabilecek üç MonoBehaviour yöntemi vardır.

  1. Başlat()
  2. OnBecameVisible()
  3. DüzeydeYüklendi()

Bu, örneğin yalnızca nesne bir kamera tarafından görünür olduğunda yürütülen komut dosyaları oluşturmak için kullanılabilir.

using UnityEngine;
using System.Collections;

public class RotateObject : MonoBehaviour
{
    IEnumerator OnBecameVisible()
    {
        var tr = GetComponent<Transform>();
        while (true)
        {
            tr.Rotate(new Vector3(0, 180f * Time.deltaTime));
            yield return null;
        }
    }
    
    void OnBecameInvisible()
    {
        StopAllCoroutines();
    }
}

Verim yolları #

Bir sonraki kareye kadar bekleyebilirsiniz.

yield return null; // wait until sometime in the next frame

İstediğiniz kadar çerçeve beklemek için bu çağrılardan birden fazlasını arka arkaya yapabilirsiniz.

//wait for a few frames
yield return null;
yield return null;

yaklaşık n saniye bekleyin. Bunun yalnızca yaklaşık bir süre olduğunu anlamak son derece önemlidir.

yield return new WaitForSeconds(n);

Herhangi bir doğru zamanlama biçimi için “WaitForSeconds” çağrısını kullanmak kesinlikle mümkün değildir.

Genellikle eylemleri zincirlemek istersiniz. Öyleyse bir şey yap ve o bittiğinde başka bir şey yap ve o bittiğinde başka bir şey yap. Bunu başarmak için başka bir eşyordamı bekleyin:

yield return StartCoroutine(coroutine);

Bunu yalnızca bir eşyordam içinden arayabileceğinizi anlayın. Yani:

StartCoroutine(Test());

“Normal” bir kod parçasından bir eşyordamı böyle başlatırsınız.

Ardından, bu çalışan eşyordamın içinde:

Debug.Log("A");
StartCoroutine(LongProcess());
Debug.Log("B");

Bu, A yazdıracak, uzun süreci başlatacak ve hemen B yazdıracaktır. Uzun sürecin bitmesini beklemez. Diğer taraftan:

Debug.Log("A");
yield return StartCoroutine(LongProcess());
Debug.Log("B");

Bu, A yazdıracak, uzun süreci başlatacak, bitene kadar bekle ve ardından B yazdıracaktır.

Eşyordamların herhangi bir şekilde iş parçacığı oluşturma ile kesinlikle hiçbir bağlantısı olmadığını her zaman hatırlamaya değer. Bu kodla:

Debug.Log("A");
StartCoroutine(LongProcess());
Debug.Log("B");

Arka planda başka bir iş parçacığında LongProcess’i başlatmak “gibi” olarak düşünmek kolaydır. Ama bu kesinlikle yanlış. Bu sadece bir koroutindir. Oyun motorları çerçeve tabanlıdır ve Unity’deki “eşyordamlar” çerçevelere erişmenize izin verir.

Bir web isteğinin tamamlanmasını beklemek çok kolaydır.

void Start() {
    string url = "http://google.com";
    WWW www = new WWW(url);
    StartCoroutine(WaitForRequest(www));
}

IEnumerator WaitForRequest(WWW www) {
    yield return www;
    
    if (www.error == null) {
        //use www.data);
    }
    else {
        //use www.error);
    }
}

Tamlık için: Çok nadir durumlarda Unity’de sabit güncelleme kullanırsınız; normalde asla kullanılmayacak bir WaitForFixedUpdate() çağrısı var. Geliştirme sırasında ekran görüntüleri oluşturmakla ilgili olarak belirli durumlarda kullanılan belirli bir çağrı (Unity’nin şu anki sürümünde WaitForEndOfFrame()) vardır. (Unity geliştikçe tam mekanizma biraz değişir, bu nedenle ilgiliyse en son bilgiler için google.)
