Kullanıcılar Python API, App Engine Kimlik Doğrulaması ile hızlı başlangıç

  • Ana Sayfa İşleyici [views.py]
  • Uygulama Yönlendirme [urls.py]
  • Html, Kullanıcılar API’sinin [index.html] nasıl kullanılacağına ilişkin ön uç örneği

Bu sayfada

  • Ana Sayfa İşleyici [views.py]
  • Uygulama Yönlendirme [urls.py]
  • Html, Kullanıcılar API’sinin [index.html] nasıl kullanılacağına ilişkin ön uç örneği

Users API kullanmak, App Engine’de kimlik doğrulamayı çalıştırmanın çok basit ve esnek bir yoludur, ancak lütfen uygulamanızın durumlar, kimlik doğrulama ortamı için daha fazla öğe gerektirmez.

Not: Bir App Engine uygulamasının geleneksel yapısı hakkında daha fazla bilgiye ihtiyacınız varsa lütfen bu bilgiyi inceleyin oluşturma-ziyaretçi defteri).

Kullanıcı API’si şunları sağlar:

  • Mevcut kullanıcının oturum açıp açmadığını tespit edin.
  • Oturum açmak için kullanıcıyı uygun oturum açma sayfasına yönlendirin.
  • Uygulama kullanıcınızdan henüz yoksa yeni bir Google hesabı oluşturmasını isteyin.

Referans ve daha fazla ayrıntı

Görünümdeki önemli unsurlar:

İçe aktarmak:

from google.appengine.api import users

Kullanıcı nesnesi ve yöntemleri:

user = users.get_current_user()

Not: jinja2’nin uygulanması isteğe bağlıdır, ancak makalenin içine tamamen iş akışını açıklamak için kullanılır.

Ana Sayfa İşleyici [views.py] #

Şablonları html’lere yerleştirmek için jinja2’yi kullanan Genel İthalat.

import jinja2
import webapp2

Kullanıcılar API’sini kullanmak için önemli içe aktarma:

from google.appengine.api import users

Jinja ortamının ayarlanması: [örnekte bilgileri ön uçta doldurmak için seçilen teknoloji]

JINJA_ENVIRONMENT = jinja2.Environment(
       loader=jinja2.FileSystemLoader(os.path.dirname(__file__)),
       extensions=['jinja2.ext.autoescape'],
       autoescape=True)

Beton Taşıyıcı:

class MainPage(webapp2.RequestHandler):
   def get(self):

       user = users.get_current_user()
       if user:
           url = users.create_logout_url(self.request.uri)

Kullanıcılar için buraya daha fazla mantık ekleyebilirsiniz

       else:
           url = users.create_login_url(self.request.uri)

Jinja2 kullanarak bilgi iletmek için şablonlar. Bu örnek için, kullanıcı nesnesi ve url dizesi.

       template_values = {
           'user': user,
           'url': url,
       }

       JINJA_ENVIRONMENT.add_extension('jinja2.ext.do')

index.html örneğini kullanma. [geleneksel html sayfası]

       template = JINJA_ENVIRONMENT.get_template('index.html')
       self.response.write(template.render(template_values))

Uygulama Yönlendirme [urls.py] #

Yönlendirmeyi kapsayacak şekilde bu örnekte webapp2 kullandım.

from webapp2_extras.routes import RedirectRoute as Route

Görünümlerden içe aktar:

from views import MainPage

MainPage, “/” köküne ayarlanmış işleyicidir:

urlpatterns = [
   Route('/', MainPage),
]

Html, Kullanıcılar API’sinin [index.html] nasıl kullanılacağına ilişkin ön uç örneği #

index.html’nin basit özü:

    <div class="sign-in">

        {% if user %}

[URL’yi geçerek kullanıcıdan çıkış yapma fırsatımız var]

            <a href="{{ url|safe }}">LOG OUT</a>

[Kimliği doğrulanmış kullanıcı için işlemleri buraya dahil edebilirsiniz]

        {% else %}

[URL’yi geçerek kullanıcıya giriş yapma fırsatımız var]

            <a href="{{ url|safe }}">SIGN IN</a>

        {% endif %}
    </div>

Bu, işlemin index.html sayfasında nasıl kullanıldığına dair basit bir örnektir.
