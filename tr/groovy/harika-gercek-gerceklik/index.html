Groovy, if, while ve for ifadelerindeki koşulları Java’nın standart Java koşulları için yaptığı şekilde değerlendirir: Java’da bir boole ifadesi (bir boole olarak değerlendirilen bir ifade) sağlamanız gerekir ve sonuç değerlendirmenin sonucu.

Groovy’de sonuç, bu koşullar için Java’daki ile aynıdır. (örnek verilmemiştir, bu standart Java’dır).

Örneklerle gösterilen diğer doğruluk değerlendirme mekanizması şu şekilde özetlenebilir:

  • sayılar: sıfır değeri yanlış, sıfır olmayan değer doğru olarak değerlendirilir.
  • nesneler: boş bir nesne başvurusu yanlış olarak değerlendirilir, boş olmayan bir başvuru doğru olarak değerlendirilir.
  • Karakter : sıfır değerine sahip bir karakter yanlış, aksi takdirde doğru olarak değerlendirilir.
  • Dize : bir dize, boş değilse ve boş değilse true, boş veya boşsa false olarak değerlendirilir (GStrings ve CharSequences için de geçerlidir).
  • Koleksiyonlar ve Haritalar (List, Map, Set, HashSet… null ve boş değil, null veya boş ise false.
  • Numaralandırmalar ve Yineleyiciler, null değilse true olarak değerlendirilir ve bunlar daha fazla öğedir (groovy, nesnede hasMoreElements veya hasNext değerini değerlendirir), null veya daha fazla öğe yoksa false.
  • Eşleştirici : bir eşleştirici, en az bir eşleşme varsa doğru, eşleşme yoksa yanlış olarak değerlendirir.
  • Kapanış : Bir kapatma, kapatma tarafından döndürülen sonucun değerlendirilmesini değerlendirir.

asBoolean yöntemi, özel boole değerlendirmesi sağlamak için kullanıcı tanımlı bir sınıfta geçersiz kılınabilir.

Sayılar boole değerlendirmesi #

sayılar için sıfır değeri yanlış, sıfır olmayan değer doğru olarak değerlendirilir

    int i = 0
...
    if (i)
        print "some ${i}"
    else
        print "nothing"

“hiçbir şey” yazdıracak

Dizeler boole değerlendirmesi #

bir dize (GString’ler dahil) boş değilse ve boş değilse true, boş veya boşsa false olarak değerlendirilir

def s = ''
...
if (s) 
    println 's is not empty'
else
    println 's is empty'

yazdıracak: ’s boş'

Koleksiyonlar ve haritalar boole değerlendirmesi #

Koleksiyonlar ve Haritalar, boş değilse doğru, boş değilse doğru, boş veya boşsa yanlış olarak değerlendirilir

/* an empty map example*/
def userInfo = [:]
if (!userInfo)
    userInfo << ['user': 'Groot', 'species' : 'unknown' ]

userInfo haritası boş olduğundan varsayılan userInfo olarak `user: ‘Groot’, türler: ‘unknown’’ ekleyecektir (haritanın burada boş olmadığını unutmayın)

Boş bir nesneyle, kod biraz farklıdır, null olduğu için userInfo’da « öğesini çağıramayız, bir atama yapmamız gerekir (ayrıca Object boolean değerlendirmesine bakın):

/* an example with a null object (def does not implies Map type) */
def userInfo = null
if (!userInfo)
    userInfo = ['user': 'Groot', 'species' : 'unknown' ]

Ve boş bir Harita ile:

/* The same example with a null Map */
Map<String,String> userInfo = null
if (!userInfo)
    userInfo = ['user': 'Groot', 'species' : 'unknown' ]

Nesne boole değerlendirmesi #

boş bir nesne başvurusu yanlış olarak değerlendirilir, boş olmayan bir başvuru doğru olarak değerlendirilir, ancak diziler, koleksiyonlar, yineleyiciler ve numaralandırmalar için boyutu da hesaba katar.

def m = null

if (!m)
    println "empty"
else
    println "${m}"

“boş” yazdıracak

def m = [:]

if (!m)
    println "empty"
else
    println "${m}"

Harita boş değil ama boş, bu kod “boş” yazdıracak

Yaptıktan sonra

m << ['user' : 'Groot' ]

Haritayı yazdıracak:

[user:Groot]

Kullanıcı tanımlı bir sınıfta boole değerlendirmesini geçersiz kılma #

Bazen, bazı nesneler için kendi programınızda belirli bir asBoolean tanımına sahip olmak yararlı olabilir.

/** an oversimplified robot controller */
class RunController {
    
    def complexCondition
    int position = 0
    
    def asBoolean() {
        return complexCondition(this);
    }
    def advanceTo(step) {
        position += step
    }
}
def runController = new RunController(complexCondition : { c -> c.position < 10 } )

assert runController
runController.advanceTo(5)
assert runController
runController.advanceTo(5)
// The limit has been reached : the controller evaluates to false
assert !runController

Bu kod, robotun konumunun 10’u geçmediğini kontrol eden aşırı basitleştirilmiş bir robot kontrolörünü gösterir (durum değerlendirmesi için bir kapatma ile)

Karakter değerlendirmesi #

a Karakter değeri sıfır değilse true, sıfır ise false olarak değerlendirilir

assert ! new Character((char)0)
assert ! new Character('\u0000Hello Zero Char'.charAt(0))
assert  new Character('Hello'.charAt(0))

Eşleştirici değerlendirmesi #

bir Eşleştirici, en az bir eşleşme bulabilirse doğru, eşleşme bulunamazsa yanlış olarak değerlendirir

// a match is found => true
assert 'foo' =~ /[a-z]/
// the regexp does not match fully => no match => false
assert !( 'foo' ==~ /[a-z]/ )
// a match is found => true
assert 'foo' =~ /o/
// no match => false
assert !( 'foo' =~ /[A-Z]/ )

Kapanış değerlendirmesi #

Bir kapanışın değerlendirilmesi, kapanışın sonucunun değerlendirilmesidir.

Tüm kurallar geçerlidir: kapatma işlemi null , sıfır sayı veya boş bir Dize, Koleksiyon, Harita veya Dizi döndürürse false olarak değerlendirilir aksi halde doğru.

// Closure return non zero number => true
assert { 42 }()
// closure returns 0 => false
assert ! ( { 0 }())
// closure returns null => false
assert !( { }())
