Kurulum veya Kurulum #

Groovy’yi kurmanın iki yaygın yolu vardır.

İndirmek

Groovy ikili dosyası, Groovy web sitesinin download sayfasından indirilebilir. Arşivi açabilir ve PATH sistem ortam değişkenine %GROOVY_HOME%/bin/groovy.bat yolunu ekleyebilirsiniz; burada %GROOVY_HOME%, Groovy’nin paketinin açıldığı dizindir.

SDKMAN

Diğer seçenek SDKMAN kullanmaktır. Bu seçeneğin popülaritesi hızla arttı ve Groovy’nin birden çok sürümünü yönetmeyi çok basit hale getirdi. Ayrıca “GR8” ekosferindeki diğer uygulamaları da destekler. Bu seçenek yerel olarak Linux ve Mac’te çok iyi çalışır, ancak Windows’ta Cygwin gerektirir.

Groovy indirme sayfasındaki talimatları izleyerek SDKMAN’i kurmak için aşağıdaki adımları uygulayabilirsiniz.

`$ curl -s get.sdkman.io | bash'

SDKMAN kurulduktan sonra artık sdk komutuna erişiminiz olur. Bu komutla birçok yararlı şey yapabilirsiniz.

Groovy’yi yükleyin

$ sdk groovy yükleyin

Bu, Groovy’nin en son sürümünü yükleyecektir.

Groovy’nin sürümlerini listeleyin

$ sdk ls harika

Bu, Groovy yazılımında mevcut tüm seçenekleri listeleyen bir Linux stili ls komutunu çalıştırmanıza izin verir. Yüklü her sürümün yanında bir * ve mevcut sürümlerinizi belirtmek için bir > bulunur.

Groovy’nin sürümlerini değiştir

`$ SDK harika 2.4.7 kullanın'

Bu, Groovy’nin mevcut sürümünü 2.4.7 olarak değiştirecektir. Yüklü başka sürümleriniz varsa, bunlardan herhangi birine geçebilirsiniz.

groovy’nin güncel sürümünü ‘groovy -version’ komutu ile listeleyebilirsiniz.

posh-gvm

SDKMAN’in ilk adı GVM idi ve posh-gvm, Windows Powershell için bir GVM bağlantı noktasıdır. Yani, bir Windows makinesinde geliştiriyorsanız ve Cygwin’de SDKMAN kullanmak istemiyorsanız, posh-gvm tam size göre. SDKMAN ile aynı şekilde çalışır, ancak “sdk” yerine komut “gmv"dir. Yani

PS C:\Users\You> gmv install groovy

Windows makinenize posh-gvm aracılığıyla groovy yükleyecektir.

Harika Dünya #

Aşağıdaki örnek, komut dosyası kullanarak groovy’deki en basit ‘Merhaba Dünya’yı göstermektedir, aşağıdaki kod parçasını bir dosyaya yerleştirin, ‘helloWorld.groovy’ deyin.

println 'Hello World!'

Nasıl yürütülür: Komut satırında, ‘harika merhabaWorld.groovy’

Çıktı: “Merhaba Dünya!”

Groovy’yi Java projesinde kullanma #

Groovy’nin tüm java sınıflarına erişimi vardır, aslında Groovy sınıfları Java sınıflarıdır ve doğrudan JVM tarafından çalıştırılabilir. Bir Java projesi üzerinde çalışıyorsanız, java kodunuzla etkileşim kurmak için Groovy’yi basit bir betik dili olarak kullanmak hiç akıllıca olmaz.

İşleri daha da iyi hale getirmek için, neredeyse tüm Java sınıfları .groovy olarak yeniden adlandırılabilir ve derlenebilir ve çalıştırılabilir ve tam olarak olduğu gibi çalışır, harika bir Java süper seti olmaya yakındır, bu harika bir hedeftir.

Groovy’nin bir REPL’si var. ‘groovysh’, Groovy ile birlikte gelir ve sınıf yolunuz doğru ayarlanmışsa bir Java sınıfını hızlı bir şekilde başlatmak ve test etmek için kullanılabilir. Örneğin, “sınıf yolunuz” Eclipse “classes/bin” dizininizi işaret ediyorsa, dosyanızı Eclipse’e kaydedebilir, “groovysh” a atlayabilir ve test etmek için sınıfı başlatabilirsiniz.

Bunu yapmak için yalnızca Java yerine Groovy kullanmanın nedenleri: Sınıf yükleyici, derlenirken yeni sınıfları almakta BÜYÜK. Geliştirme sırasında genellikle ‘groovysh’tan çıkmanız/yeniden başlatmanız gerekmez.

Sözdizimi TERSE’dir. Bu, sürdürülebilir kod için harika değil, ancak komut dosyaları ve testler için kodunuzu önemli ölçüde azaltabilir. Yaptığı en büyük şeylerden biri, kontrol edilen istisnaları ortadan kaldırmaktır (veya daha doğrusu, kontrol edilen tüm istisnaları kontrol edilmeyen istisnalara dönüştürmektir). Bu, kodu şöyle döndürür (Bir saniye sonra merhaba yazdır):

class JavaClass {
    public static void main(String[] args) {
        try {
            Thread.sleep(1000);
        } catch(InterruptedException e) {
            // You shouldn't leave an empty catch block, but who cares if this was interrupted???
        }
        System.out.println("Hello!");
    }
}

Groovy’nin içine:

Thread.sleep(1000)
print "Hello!"

Groovy ayrıca çok sıkı başlatma sözdizimine sahiptir. Bu, verileri düşünmeden istediğiniz gibi belirtmenize olanak tanır:

Java’da bir haritayı başlatmak için muhtemelen şöyle bir şey yapmalısınız:

String[] init = { "1:Bill", "2:Doug", "3:Bev" };
// Note the rest of this can be put in a function and reused or maybe found in a library, but I always seem to have to write this function!
Map m = new HashMap<Integer, String>();
for(String pair : int) {
    String[] split = pair.split(":");
    m.put(new Integer(split[0]), split[1])
}

Bu kötü değil, ama bakımı başka bir şey. Groovy’de sadece kullanırsınız:

Map map = { 1 : "Bill", 2 : "Doug", 3 : "Bev" }

Ve bitirdin. Liste sözdizimi de aynı derecede kolaydır.

Diğer gerçekten büyük avantaj, groovy’nin kapatma sözdizimidir. Şaşırtıcı derecede özlü ve eğlenceli, bakımı biraz daha zor, ancak komut dosyaları için bu bir öncelik değil. Örnek olarak, geçerli dizinde “Merhaba” kelimesini içeren tüm “.txt” dosyalarını bulmak için bazı harika kodlar:

println new File('.').files.findAll{ it.name.endsWith('.txt') && it.text.contains('Hello') }.collect{ it.name }

Bu örnekte birkaç “Groovy” hilesi kullanılmaktadır:

  • “.files”, “getFiles()” yöntemini ifade eder - groovy istediği zaman alıcı/ayarlayıcı ve özellik sözdizimi arasında geçiş yapabilir

  • it. bir yinelemenin geçerli öğesini ifade eder. { it }, { it -> it } için bir kısayoldur, ör. :

    [1, 2, 3].collect{ it ^ 2 } == [1, 4, 9]

  • “it.text” (“it"in bir dosya olduğu yerde), dosyanın tüm metnini almak için groovy’nin “Dosya"ya eklediği bir yöntemi kullanır. Bu, komut dosyalarında inanılmaz derecede yararlıdır.

Merhaba dünya Shebang (linux) #

İçeriği olan bir merhaba.groovy dosyası verildi:

#!/usr/bin/env groovy
println "Hello world"

Yürütme izni verilirse komut satırından yürütülebilir

$ ./hello.groovy

CSV Dizisi Oluşturmak İçin Listede inject() Kullanımı #

Groovy’de, inject() yöntemi, inject() yöntemini uygulayan herhangi bir nesneye yeni işlevler eklememize (veya enjekte etmemize) izin veren kümülatif yöntemlerden biridir. Koleksiyon durumunda, bir nesne koleksiyonuna tek tip bir kapatma uygulayabilir ve ardından sonuçları tek bir değerde harmanlayabiliriz. inject() yönteminin ilk parametresi, kümülasyonun başlangıç ​​değeri ve ikinci parametre ise kapanıştır.

Bu örnekte, parametre olarak bir Dize Listesi alacağız ve virgülle ayrılmış bu dizelerin değerlerini çıkaracağız. Bu işlevi, bir REST sorgu dizesine bir değerler listesi eklemek için kullandım ve biraz değiştirirseniz, IN yan tümcesinin bir parçası olarak bir SQL deyimine değerler eklemek için kullandım. İşte bunu yapmak için kod:

public String convertToCSV( List<String> list ) {
    if (list == null) {
        return ""
    }
    return list.inject( '' ) { result, item ->
        result + ( result && item ? ',' : '' ) + ( item ? "${item.trim()}" : '' )
    }
}

assert convertToCSV( null ) == ""
assert convertToCSV( ["aaa", "bbb  ", null, "  ccc  "] ) == "aaa,bbb,ccc"

Bu örnekte, inject() yönteminin ilk parametresi sıfır uzunluklu bir dizedir; bu, listenin ilk öğesini işlerken sonucun da sıfır uzunluklu bir dize olduğu anlamına gelir. Bu, ilk üçlü değerlendirmede false olarak çözülür, bu nedenle dizenin başında virgül alamıyoruz. Listenin öğeleri boyunca her ardışık yinelemede, sonuç, listedeki son öğeye ulaşana kadar bir virgül ve ardından bir sonraki öğenin birleştirilmesi olur.

Bu yaklaşımın avantajı, birleştirilmiş String sonucunu tutmak için bir döngü yapısının dışında bir değişkene ihtiyacınız olmamasıdır. Bunun, kodunuzda yan etkilere yol açabileceği anlamına gelir. inject() yaklaşımıyla, bu davranış enjekte edilir ve koleksiyon, sizin için kapatma çağrılarının sonucunu derler. Bu yaklaşımın dezavantajı okunabilirlik olabilir. Ancak biraz deneyimle, okumak ve anlamak daha kolay hale gelir ve umarım bu örnek bu hedefe ulaşmanıza yardımcı olur.
