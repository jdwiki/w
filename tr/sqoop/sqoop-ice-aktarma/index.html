Sözdizimi #

  • <rdbms-jdbc-url> // RDBMS JDBC URL’si
  • <username> // RDBMS veritabanının kullanıcı adı
  • <password> // RDBMS veritabanının şifresi
  • <table-name> // RDBMS veritabanı tablosu
  • <hdfs-home-dir> // HDFS ana dizini
  • <condition> // WHERE yan tümcesi ile SQL sorgusu biçiminde ifade edilebilen koşul.
  • <sql sorgusu> // SQL Sorgusu
  • <target-dir> // HDFS Hedef Dizini

Sqoop, tabloyu bir RDBMS veri kaynağından HDFS’ye veya tam tersi şekilde içe aktaran bir Hadoop Komut Satırı aracıdır. İçe aktarılan verilerle etkileşime girmemizi sağlayan bir Java sınıfı oluşturur. Bir tablodaki her satır, HDFS’de ayrı bir kayıt olarak kaydedilir. Kayıtlar, metin dosyaları olarak veya Avro veya Sıra Dosyaları olarak ikili gösterimde saklanabilir. sqoop’un 2 sürümü vardır:

Sqoop1 ve Sqoop2

Sqoop1, yaygın olarak kabul edilen bir araçtır ve üretim ortamları için önerilir. Cloudera’nın web sitesinde belirtildiği gibi Sqoop1 ve Sqoop2 arasındaki karşılaştırmayı bulun.

RDBMS Tablosunu HDFS’ye Aktar #

sqoop import \
--connect <rdbms-jdbc-url> \
--username <username> \
--password <password> \
--table <table-name>

Mysql ile örnek:

sqoop import \
--connect jdbc:mysql://mysql.example.com/testdb \
--username root \
--password root \
--table employees

İçe aktarılan verileri içeren CSV dosyası, ana dizin altındaki çalışanlar dizini altında oluşturulacaktır.

Komutu kullanarak inceleyin:

hadoop fs -cat <hdfs-home-dir>/employees/part-m-*

HDFS’de belirli bir dizine içe aktarın #

sqoop import \
--connect jdbc:mysql://mysql.example.com/testdb \
--username root \
--password root \
--table emplyoees \
--target-dir /dev/data/employees

Bu, /dev/data/employees dizini altında CSV dosyası oluşturacaktır.

Sqoop işi için ana HDFS dizinini belirtin #

sqoop import \
--connect jdbc:mysql://mysql.example.com/testdb \
--username root \
--password root \
--table emplyoees \
--warehouse-dir /dev/warehouse/

Yukarıdaki komuttaki --warehouse-dir etiketi, ana dizininizi /dev/warehouse/ olarak değiştirecektir.

RDBMS Tablosunun alt kümesini HDFS’ye içe aktar #

--where etiketini kullanma #

sqoop import \
--connect <rdbms-jdbc-url> \
--username <username> \
--password <password> \
--table <table-name> \
--where "<condition>"

Mysql ile örnek:

sqoop import \
--connect jdbc:mysql://mysql.example.com/testdb \
--username root \
--password root \
--table employees \
--where "country = 'INDIA'"

Herhangi bir özel işlev, dönüştürme ve hatta kullanıcı tanımlı işlevler “–where” yan tümcesinde kullanılabilir.

Serbest Biçimli Sorguyu Kullanma #

sqoop import \
--connect <rdbms-jdbc-url> \
--username <username> \
--password <password> \
--query <sql-query> \
--target-dir <target-dir>

Mysql ile örnek:

sqoop import \
--connect jdbc:mysql://mysql.example.com/testdb \
--username root \
--password root \
--query "SELECT emplyoees.name, \
                address.city \
                FROM emplyoees \
                JOIN address USING(emp_id) \
                WHERE \$CONDITION" \
--taget-dir /dev/data/employees

Tüm Tabloları İçe Aktar #

SQOOP, tüm tabloları içe aktarma olanağı sağlar

sqoop import-all-tables \
--connect <rdbms-jdbc-url> \
--username <username> \
--password <password> \
--hive-import \
--create-hive-table \
--hive-database <dbname> \
--warehouse-dir <warehouse-dir>

Tüm tabloları içe aktarma ve içe aktarma arasındaki farklar hakkında dikkat edilmesi gereken önemli noktalar:

Tüm tabloları bir veritabanına indirmek için –warehouse-dir=//stage.db veritabanı adını giriş parametresi olarak sağlamanız gerekir. Sqoop içe aktarmada, –warehouse-dir’i değil, yalnızca –target-dir’i sağlayacağız

Örnek:

sqoop import-all-tables --connect="jdbc:mysql://serverip:3306/dbname" 
--username=xxx --password=yyy 
-m 1 --hive-import 
--hive-overwrite 
--create-hive-table 
--hive-database dbname 
--hive-home /user/hive/warehouse
--warehouse-dir=/user/hive/warehouse/retail_stage.db
