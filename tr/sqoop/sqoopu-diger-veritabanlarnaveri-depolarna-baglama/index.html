Sqoop'u diğer veritabanlarınaveri depolarına bağlama

  • JDBC Sürücüsünü Yükle
  • Bağlantıyı doğrulayın
  • Tabloyu yeni kataloğa aktar

Çeşitli veri depolarından/veritabanlarından veri almak için bir sqoop betiğinin nasıl kullanılabileceğini gösterir.

JDBC Sürücüsünü Yükle #

MS SQL Server veritabanına erişmek için Sqoop, Microsoft’tan indirilebilen ek bir JDBC sürücüsü gerektirir. Aşağıdaki adımlar MSSQL Server JDBC sürücüsünü Sqoop’a yükleyecektir:

wget 'http://download.microsoft.com/download/0/2/A/02AAE597-3865-456C-AE7F-613F99F850A8/sqljdbc_4.0.2206.100_enu.tar.gz'

tar -xvzf sqljdbc_4

cp sqljdbc_4.0/enu/sqljdbc4.jar /usr/hdp/current/sqoop-server/lib/

Bağlantıyı doğrulayın #

Sunucuya bağlantının geçerli olduğunu kontrol etmek için:

sqoop list-tables --connect "jdbc:sqlserver://<server_ip>:1433;database=<database_name>" 
                  --username <user_name>
                  --password <password>

Bunu yapmadan önce SqlServer’ın yapılandırmasının 1433 bağlantı noktasına uzaktan erişime izin verip vermediğini kontrol etmeniz önerilir.

Configuration Manager => SQL Server Network configuration => Protocols for MSSQLSERVER‘ı açın ve Protokol’ün etkinleştirildiğini ve gerekli IP ve bağlantı noktasının etkinleştirildiğini ve etkin olduğunu kontrol edin.

Tabloyu yeni kataloğa aktar #

SQL Server’dan Hadoop’a veri aktarmak için:

sqoop import --table TestTable
             --connect "jdbc:sqlserver://192.168.1.100:1433;database=Test_db"  
             --username user 
             --password password  
             --split-by id  
             --target-dir /user/test
  • bölme - birincil anahtar yoksa zorunlu olarak kullanılır
  • target-dir – henüz mevcut olmayan yeni katalog

Bir sorgunun sonuçlarını ilişkisel bir veritabanından HDFS’ye aktarın: #

İçe aktarma işleminde tablo yerine sorgu kullanılabilir:

sqoop import --query 'select Id,Message from TestTable where $CONDITIONS' 
             --where 'id>100' 
             --connect "jdbc:sqlserver://192.168.1.100:1433;database=Test_db
             --username user 
             -–password password 
             --split-by id  
             --target-dir /user/test/ 
             --fields-terminated-by '\t'
  • nerede $CONDITIONS - koşul mevcut olmasa bile zorunludur
  • bölme - zorunlu, bölme işlemi için sütunu belirtir. MapReduce işinde görevleri bölmek için kullanılır

Verileri doğrudan Hive Warehouse’a aktarın #

Veriler doğrudan Hive’a aktarılabilir:

sqoop import --hive-import 
             --table EventLog 
             --connect "jdbc:sqlserver://192.168.1.99:1433;database=Test_db" 
             --username user 
             --password password 
             --split-by id 
             --fields-terminated-by '\t'

Verileri RDBMS’den HBase tablosuna aktarın #

Aşağıdaki sqoop komutu, verileri RDBMS tablosundan HBase tablosuna aktarmak için kullanılacaktır, tablo HBase’de mevcut değilse yeni bir tablo oluşturacak ve verileri bu tabloya aktaracaktır.

sqoop import \
        --query 'select emp_id, emp_name, emp_sal from employee where $CONDITIONS' \
        --connect "jdbc:sqlserver://192.168.1.99:1433;database=test_db" \
        --username username \
        -–password password \
        --hbase-create-table \
        --hbase-table employee_table \
        --hbase-row-key emp_id
