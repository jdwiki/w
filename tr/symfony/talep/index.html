Symfony’nin İstek sınıfı, HTTP isteğinin nesne yönelimli bir temsilidir. URL, sorgu dizesi, yüklenen dosyalar, çerezler ve tarayıcıdan gelen diğer başlıklar gibi bilgileri içerir.

Sözdizimi #

  • $request->getPathInfo(); // istenen yolu (URL’nin yerel kısmı) döndürür (ancak sorgu dizesi olmadan). yani https://example.com/foo/bar?key=value adresini ziyaret ederken, bu /foo/bar içerecektir
  • $request->query->get(‘id’); // bir sorgu dizesi parametresi döndürür ($_GET)
  • $request->query->get(‘id’, 1); // varsayılan değere sahip bir sorgu dizesi parametresi döndürür
  • $request->request->get(‘isim’); // bir istek gövdesi değişkeni ($_POST) döndürür
  • $request->files->get(’ek’); // “ek” ile tanımlanan UploadedFile örneğini döndürür
  • $request->cookies->get(‘PHPSESSID’); // bir tanımlama bilgisinin değerini döndürür ($_COOKIE)
  • $request->headers->get(‘content_type’); // bir HTTP istek başlığı döndürür
  • $request->getMethod(); // HTTP istek yöntemini döndürür (GET, POST, PUT, DELETE, vb.)
  • $request->getLanguages(); // istemcinin kabul ettiği bir dizi dil döndürür

Sorgu dizesi parametresi alma #

Sayfa numarasının bir sorgu dizesi parametresi olarak iletildiği, sayfalandırılmış bir ürün listesi oluşturmak istediğimizi varsayalım. Örneğin, 3. sayfayı getirmek için şuraya gidersiniz:

http://example.com/products?page=3

Ham HTTP isteği şuna benzer:

GET /products?page=3 HTTP/1.1
Host: example.com
Accept: text/html
User-Agent: Mozilla/5.0 (Macintosh)

Sayfa numarasını istek nesnesinden almak için query özelliğine erişebilirsiniz:

$page = $request->query->get('page'); // 3

Bir “sayfa” parametresi durumunda, sorgu dizesi parametresinin ayarlanmaması durumunda muhtemelen varsayılan bir değer iletmek isteyeceksiniz:

$page = $request->query->get('page', 1);

Bu, birisi http://example.com/products adresini ziyaret ettiğinde (sorgu dizesinin bulunmadığına dikkat edin), “$page” değişkeninin varsayılan “1” değerini içereceği anlamına gelir.

Global değişkenlerden bir İstek nesnesi oluşturma #

PHP, $_POST, $_GET, [$_FILES][3 gibi HTTP isteği hakkında bilgi içeren bir dizi genel değişken sunar. ], $_SESSION, vb. ‘Request’ sınıfı, bu değişkenlere dayalı bir istek nesnesi başlatmak için statik bir ‘createFromGlobals()’ yöntemini içerir:

use Symfony\Component\HttpFoundation\Request;

$request = Request::createFromGlobals();

Symfony çerçevesini kullanırken, istek nesnesini kendiniz başlatmamalısınız. Bunun yerine, çerçeve “app.php” / “app_dev.php” içinde önyüklendiğinde örneklenen nesneyi kullanmalısınız. Örneğin, [denetleyicinizdeki istek nesnesini ima ederek][5] yazın.

[5]: https://www.wikiod.com/tr/symfony/kontrolorler#İstek nesnesindeki verileri kullanma

Bir POST değişkenine erişme #

method="post" ile gönderilen bir formun içeriğini almak için post özelliğini kullanın:

$name = $request->request->get('name');

Bir çerezin içeriğine erişme #

$id = $request->cookies->get('PHPSESSID');

Bu, tarayıcı tarafından gönderilen ‘PHPSESSID’ çerezinin değerini döndürür.
