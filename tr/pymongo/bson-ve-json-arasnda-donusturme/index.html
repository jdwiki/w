Birçok uygulamada, MongoDB’den gelen kayıtların JSON formatında serileştirilmesi gerekir. Kayıtlarınızda tarih, tarih, saat, nesne kimliği, ikili, kod vb. tür alanları varsa, “json.dumps” kullanırken “TypeError: JSON serileştirilemez” istisnalarıyla karşılaşırsınız. Bu konu, bunun nasıl üstesinden gelineceğini gösterir.

json_util’i kullanma #

json_util, yerel json yöntemlerini saran ve açık BSON dönüşümü sağlayan “dökümler” ve “yükler” olmak üzere iki yardımcı yöntem sağlar. json’a ve json’dan.

Basit kullanım #

from bson.json_util import loads, dumps
record = db.movies.find_one()
json_str = dumps(record)
record2 = loads(json_str)

“kayıt” ise:

{ 
    "_id" : ObjectId("5692a15524de1e0ce2dfcfa3"), 
    "title" : "Toy Story 4", 
    "released" : ISODate("2010-06-18T04:00:00Z") 
}

sonra “json_str” şuna dönüşür:

{
    "_id": {"$oid": "5692a15524de1e0ce2dfcfa3"},
    "title" : "Toy Story 4", 
    "released": {"$date": 1276833600000}
}

JSONSeçenekleri #

Bir “JSONOptions” nesnesi aracılığıyla “dumps” davranışını özelleştirmek mümkündür. İki seçenek grubu zaten mevcuttur: “DEFAULT_JSON_OPTIONS” ve “STRICT_JSON_OPTIONS”.

>>> bson.json_util.DEFAULT_JSON_OPTIONS
    JSONOptions(strict_number_long=False, datetime_representation=0,
     strict_uuid=False, document_class=dict, tz_aware=True, 
     uuid_representation=PYTHON_LEGACY, unicode_decode_error_handler='strict',
     tzinfo=<bson.tz_util.FixedOffset object at 0x7fc168a773d0>)

Farklı seçenekleri kullanmak için şunları yapabilirsiniz:

  1. “DEFAULT_JSON_OPTIONS” nesnesini değiştirin. Bu durumda, seçenekler “dumps"a yapılan sonraki tüm çağrılar için kullanılacaktır:

     from bson.json_util import DEFAULT_JSON_OPTIONS
 DEFAULT_JSON_OPTIONS.datetime_representation = 2
 dumps(record)

  2. “json_options” parametresini kullanarak bir “dumps” çağrısında bir “JSONOptions” belirtin:

     # using strict
 dumps(record, json_options=bson.json_util.STRICT_JSON_OPTIONS)

 # using a custom set of options
 from bson.json_util import JSONOptions
 options = JSONOptions() # options is a copy of DEFAULT_JSON_OPTIONS
 options.datetime_representation=2
 dumps(record, json_options=options)

JSONOptions parametreleri şunlardır:

  • strict_number_long: True ise, Int64 nesneleri MongoDB Extended JSON’un Strict mod türü NumberLong’a kodlanır, yani {"$numberLong": "<number>" }. Aksi takdirde int olarak kodlanırlar. Varsayılan olarak False’dır.
  • datetime_representation: datetime.datetime örneklerini kodlarken kullanılacak temsil. 0 => {"$date": <dateAsMilliseconds>}, 1 => {"$date": {"$numberLong": "<dateAsMilliseconds>"}}, 2 => {"$date" : "<ISO-8601>"}
  • strict_uuid: True ise, uuid.UUID nesnesi MongoDB Extended JSON’un Strict mod türü Binary’ye kodlanır. Aksi takdirde, {"$uuid": "<hex>" } olarak kodlanacaktır. Varsayılan olarak False’dır.
  • document_class: load() tarafından döndürülen BSON belgelerinin kodu bu sınıfın bir örneğine çözülecektir. Collections.MutableMapping’in bir alt sınıfı olmalıdır. Dikte etmek için varsayılanlar.
  • uuid_representation: uuid.UUID örneklerini kodlarken ve kodunu çözerken kullanılacak BSON temsili. Varsayılan olarak PYTHON_LEGACY’dir.
  • tz_aware: Doğruysa, MongoDB Extended JSON’un Strict mod türü Date’in kodu, datetime.datetime’ın saat dilimine duyarlı örneklerine çözülecektir. Aksi takdirde naif olacaklardır. Varsayılan Değeri True’dur.
  • tzinfo: Datetime nesnelerinin kodunun çözülmesi gereken saat dilimini belirten bir “datetime.tzinfo” alt sınıfı. Varsayılan olarak utc’dir.

json modülünü özel işleyicilerle kullanma #

Tek ihtiyacınız olan mongo sonuçlarını json’a serileştirmekse, seri hale getirilemeyen alan türleriyle başa çıkmak için özel işleyiciler tanımlamanız şartıyla ‘json’ modülünü kullanmak mümkündür. Avantajlardan biri, tarih saat gösterimi gibi belirli alanları nasıl kodladığınız konusunda tam yetkiye sahip olmanızdır.

İşte iso gösterimini ve kimliği onaltılık bir dize olarak kullanarak tarihleri ​​kodlayan bir işleyici:

import pymongo
import json 
import datetime
import bson.objectid

def my_handler(x):
    if isinstance(x, datetime.datetime):
        return x.isoformat()
    elif isinstance(x, bson.objectid.ObjectId):
        return str(x)
    else:
        raise TypeError(x)

db = pymongo.MongoClient().samples
record = db.movies.find_one()
# {u'_id': ObjectId('5692a15524de1e0ce2dfcfa3'), u'title': u'Toy Story 4',
#   u'released': datetime.datetime(2010, 6, 18, 4, 0),}

json_record = json.dumps(record, default=my_handler)
# '{"_id": "5692a15524de1e0ce2dfcfa3", "title": "Toy Story 4", 
#    "released": "2010-06-18T04:00:00"}'

python-bsonjs’yi kullanma #

python-bsonjs PyMongo’ya bağlı değildir ve “json_util” üzerinde güzel bir performans iyileştirmesi sunabilir:

bsonjs, BSON’u JSON’a kod çözme ve JSON’u BSON’a kodlamada PyMongo’nun json_util’inden yaklaşık 10-15 kat daha hızlıdır.

Dikkat:

  • bsonjs’yi etkin bir şekilde kullanmak için doğrudan RawBSONDocument ile çalışmanız önerilir.
  • tarihler, LEGACY gösterimi kullanılarak kodlanır, yani {"$date": <dateAsMilliseconds>}. Şu anda bunu değiştirmek için bir seçenek yok.

Kurulum #

pip install python-bsonjs

Kullanım #

Bsonjs’den tam olarak yararlanmak için veritabanını ‘RawBSONDocument’ sınıfını kullanacak şekilde yapılandırın. Ardından, bson ham baytlarını json’a dönüştürmek için “dumps” ve json’u bson ham baytlarına dönüştürmek için “loads” kullanın:

import pymongo
import bsonjs
from pymongo import MongoClient
from bson.raw_bson import RawBSONDocument

# configure mongo to use the RawBSONDocument representation
db = pymongo.MongoClient(document_class=RawBSONDocument).samples
# convert json to a bson record
json_record = '{"_id": "some id", "title": "Awesome Movie"}' 
raw_bson = bsonjs.loads(json_record)
bson_record = RawBSONDocument(raw_bson)
# insert the record
result = db.movies.insert_one(bson_record)
print(result.acknowledged)

# find some record
bson_record2 = db.movies.find_one()
# convert the record to json
json_record2 = bsonjs.dumps(bson_record2.raw)
print(json_record2)
