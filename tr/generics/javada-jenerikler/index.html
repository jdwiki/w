Sözdizimi #

  • class MyClass<T1, T2 CharSequence'ı genişletir>, Comparable<MyClass>'ı uygular //...
  • interface MyListInterface<T Serileştirilebilir'i genişletir> Listeyi genişletir<T> //...
  • public <T1, T2 Extends Instant> T1 ProvideClone(T1 toClone, T2 instant> //...
  • public static List<CharSequence> safe(Collection<? CharSequence> uzatır> l) { return new ArrayList<>(l);}
  • Set<String> strings = Collections.singleton("Merhaba dünya");
  • List<CharSequence> chsList = safe(strings);

Tür silme yansımayı sınırlar, ancak bu JVM’ye özgü değildir, örneğin Ceylon [birleştirilmiş jenerikleri kullanır](http://ceylon-lang.org/documentation/1.2/spec /html/execution.html#reification).

Varoluşsal tür desteği, bu biçimdeki diğer diller tarafından mutlaka desteklenmez: Kotlin [tür projeksiyonları aracılığıyla destekler](https://kotlinlang.org/docs/reference/java-interop. html#java-generics-in-kotlin).

Giriiş #

Jenerikler Java’da (1.)5 sürümünde tanıtıldı. Bunlar derleme sırasında silinir, bu nedenle çalışma zamanı yansıtması onlar için mümkün değildir. Jenerikler, diğer türler tarafından parametrelenen yeni türler oluşturur. Örneğin, ‘String’ler ve ‘Number’ların tür güvenli koleksiyonunu kullanmak için yeni sınıflar yaratmamız gerekmiyor, genel ‘ArrayList’ her durumda kullanılabilir, örneğin: ’new ArrayList ()`.

Örnek:

List<String> variable = new ArrayList<String>();

Java 7’de, inşaatı kolaylaştırmak için bazı sözdizimsel şeker tanıtıldı (’<>’ aka. elmas):

List<String> variable = new ArrayList<>();

İlginç bir şekilde (Java 5’ten), statik bir yöntemin dönüş değeri olduğu zaman tür çıkarımı kullanmak da mümkündü (örneğin, genellikle Google Guava‘da kullanılır):

List<String> singleton = Collections.singletonList();//Note the missing `<>` or `<String>`!

Java’da, türler için polimorfizm sağlamak için varoluşsal türler kullanıldı, çünkü genel türler değişmezdir (örneğin: ‘List’, Java ‘String’de olmasına rağmen ‘List’ öğesinin bir alt türü veya bir üst türü değildir. []’, “CharSequence[]” öğesinin bir alt türüdür; not: “Dize”, “CharSequence” arabirimini uygular). Varoluşsal genel türler şu şekilde ifade edilebilir:

List<? extends CharSequence> list = new ArrayList<String>();
Comparable<? super ChronoLocalDate> ccld = LocalDate.now();
ChronoLocalDate cld = JapaneseDate.now(); //ChronoLocalDate extends Comparable<ChronoLocalDate>
ccld.compareTo(cld);
//cld.compareTo(ccld);//fails to compile because ccld is not a `ChronoLocalDate` (compile time)

Her iki örnek de karşılık gelen “Karşılaştırılabilir” ile parametrelenen bir listede kullanılabilir:

List<Comparable<? super ChronoLocalDate>> list2 = new ArrayList<>();
list2.add(cld);
list2.add(ccld);

Genel Yöntemler #

Genel type parametreleri genellikle sınıf veya arabirim düzeyinde tanımlanır, ancak methods ve (nadiren) constructors, tek bir yöntem çağrısının kapsamına bağlı tür parametrelerinin bildirilmesini de destekler.

class Utility // no generics at the class level
{
    @SafeVarargs
    public static <T> T randomOf(T first, T... rest) {
        int choice = new java.util.Random().nextInt(rest.length + 1);
        return choice == rest.length ? first : rest[choice];
    }

    public static <T extends Comparable<T>> T max(T t1, T t2) {
        return t1.compareTo(t2) < 0 ? t2 : t1;
    }
}

Tür parametresi bildirimlerinin, sırasıyla ‘T’ ve ‘<T, Karşılaştırılabilir>‘yi genişlettiğine, yöntem değiştiricilerinden sonra ve dönüş türünden önce göründüğüne dikkat edin. Bu, “T” tür parametresinin bu tür yöntemler kapsamında kullanılmasına izin verir ve şu şekilde hareket eder:

  • argüman türleri
  • dönüş türü
  • yerel değişken türleri

Yukarıdaki her iki yöntem de aynı tür parametre adı ‘T’yi kullanmasına rağmen, yöntem düzeyinde birbirlerinden tamamen bağımsızdırlar. Derleyici, yöntemi çağıran her arama sitesinde yönteme iletilen bağımsız değişkenlere dayalı olarak gerçek türü infer eder. ‘max’ yöntemi, ‘T’nin Karşılaştırılabilir‘yi genişlettiğini bildirdiği için, derleyici, çıkarılan türlerin, ‘Karşılaştırılabilir’ arabirimin uyumlu uygulamaları olduğunu da zorunlu kılar.

Integer num1 = 1;
Integer num2 = 2;
String str1 = "abc";
String str2 = "xyz";

Integer bigger = Utility.max(num1, num2);
assert bigger == num2;

String later = Utility.max(str2, str1);
assert later == str2;

Utility.max(num1, str1); // compiler error: num1 and str1 are incompatible types

Utility.max(new Object(), new Object()); // compiler error: Object does not implement Comparable

Java 8, derleyicinin çağrı sitelerinde genel türleri doğru şekilde çıkarsama yeteneğini önemli ölçüde geliştirdi. Derleyici uygun türü çıkaramazsa, geliştiriciler türü çağrının bir parçası olarak açıkça belirtebilir:

Object obj = Utility.<Object>randomOf(str1, new Object(), num1);
