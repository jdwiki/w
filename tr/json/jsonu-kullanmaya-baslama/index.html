JSON Dizisi #

JSON Dizisi, sıralı bir değerler topluluğudur. Köşeli parantezler, yani “[]” ile çevrilidir ve değerler virgülle ayrılmıştır:

{ "colors" : [ "red", "green", "blue" ] }

JSON Dizileri, 2 nesneli bir dizi örneğinde olduğu gibi (RFC belgesinden alınmıştır):

[
  {
     "precision": "zip",
     "Latitude":  37.7668,
     "Longitude": -122.3959,
     "Address":   "",
     "City":      "SAN FRANCISCO",
     "State":     "CA",
     "Zip":       "94107",
     "Country":   "US"
  },
  {
     "precision": "zip",
     "Latitude":  37.371991,
     "Longitude": -122.026020,
     "Address":   "",
     "City":      "SUNNYVALE",
     "State":     "CA",
     "Zip":       "94085",
     "Country":   "US"
  }
]

Ayrıca karışık tipte öğeler de içerebilirler, örneğin:

[
    "red",
    51,
    true,
    null,
    {
        "state": "complete"
    }
]

JSON dizileri (ve nesneleri) yazarken yapılan yaygın bir hata, son öğeden sonra virgül bırakmaktır. Bu, birçok dilde yaygın bir kalıptır, ancak ne yazık ki JSON’da geçerli değildir. Örneğin, aşağıdaki dizi geçersiz:

[
    1,
    2,
]

Bunu geçerli kılmak için, son öğeden sonra virgülü kaldırmanız gerekir:

[
    1,
    2
]

JSON Nesnesi #

Bir JSON Nesnesi, küme parantezleri ile çevrilidir ve anahtar/değer çiftlerini içerir.

{ "key1": "value1", "key2": "value2", ... }

Anahtarlar geçerli dizeler olmalıdır, bu nedenle çift tırnak içine alınmalıdır. Değerler JSON nesneleri, sayılar, dizeler, diziler veya şu “değişmez adlardan” biri olabilir: “false”, “null” veya “true”. Anahtar/değer çiftinde anahtar, değerden iki nokta üst üste ile ayrılır. Birden çok anahtar/değer çifti virgülle ayrılır.

Nesnelerde sıra önemli değildir. Aşağıdaki JSON nesnesi bu nedenle yukarıdakine eşdeğerdir:

{ "key2": "value2", "key1": "value1", ... }

Hepsini özetlemek gerekirse, bu geçerli bir JSON Nesnesi örneğidir:

{
  "image": {
    "width":  800,
    "height": 600,
    "title":  "View from 15th Floor",
    "thumbnail": {
      "url": "http://www.example.com/image/481989943",
      "height": 125,
      "width":  100
    },
    "visible": true,
    "ids": [116, 943, 234, 38793]
  }
}

İlgili (Java) nesne karşılıklarıyla birlikte JSON nesnelerinin yaygın örnekleri #

Bu örnek boyunca, JSON’a seri hale getirilen ‘root’ nesnesinin aşağıdaki sınıfın bir örneği olduğu varsayılır:

public class MyJson {
}

 

Örnek 1 : “MyJson” örneğinin bir örneği, olduğu gibi:

{}

yani, sınıfın alanı olmadığı için yalnızca küme parantezleri serileştirilir. Kıvrımlı parantezler, bir nesneyi temsil eden yaygın sınırlayıcılardır. Ayrıca kök nesnenin nasıl olmadığına da dikkat edin. anahtar/değer çifti olarak serileştirilir. Bu aynı zamanda basit türler için de geçerlidir (Dize, sayılar, diziler) bir (dış) nesnenin alanları olmadığında.

 

Örnek 2: MyJson’a bazı alanlar ekleyelim ve bunları bazı değerlerle başlatalım:

// another class, useful to show how objects are serialized when inside other objects
public class MyOtherJson {} 

// an enriched version of our test class
public class MyJson {
  String myString = "my string";
  int myInt = 5;
  double[] myArrayOfDoubles = new double[] { 3.14, 2.72 };
  MyOtherJson objectInObject = new MyOtherJson();    
}

Bu, ilgili JSON temsilidir:

{
  "myString" : "my string",
  "myInt" : 5,
  "myArrayOfDoubles" : [ 3.14, 2.72 ],
  "objectInObject" : {}
}

Tüm alanların bir anahtar/değer yapısında nasıl serileştirildiğine dikkat edin, burada anahtar değeri tutan alanın adı. Diziler için ortak sınırlayıcılar köşeli parantezlerdir. Ayrıca, son çift dışında her bir anahtar/değer çiftinin ardından bir virgül geldiğine dikkat edin.

JSON’u El İle Düzenleme #

JSON çok katı bir formattır (bkz. http://json.org). Bu, makineler için ayrıştırmayı ve yazmayı kolaylaştırır, ancak göze çarpmayan bir hata belgeyi bozduğunda insanları şaşırtıyor.

Genel Sorunlar #

Sondaki Virgül #

Çoğu programlama dilinden farklı olarak, sonuna virgül eklemenize izin verilmez:

{
  a: 1,
  b: 2,
  c: 3
}

“3"ten sonra virgül eklemek, bir sözdizimi hatası üretecektir.

Aynı sorun diziler için de var:

[ 
  1,
  2
]

Öğeleri yeniden sıralamanız gerekiyorsa ekstra özen göstermelisiniz.

Eksik Virgül #

{
  a: 1,
  b: 2,
  c: 3
  d: 4
}

Sondaki virgüllere izin verilmediğinden, yeni bir değer eklemeden önce bir tane eklemeyi unutmak kolaydır (bu durumda ‘3’ten sonra).

Yorumlar #

JSON, veri değişim biçimi olduğu için yorumlara izin vermez. Bu hala sıcak bir konudur, ancak bunları kullanmamaktan başka net cevaplar yoktur.

Birkaç geçici çözüm vardır:

  • C stili yorumları kullanın, ardından ayrıştırıcıya iletmeden önce bunları çıkarın
  • Yorumları verilere gömün
{
  "//": "comment",
  "data": 1
}
  • Yorumları gömün ve verilerin üzerine yazın
{
  "data": "comment",
  "data": 1
}

İkinci ‘veri’ girişi, çoğu ayrıştırıcıda_ yorumun üzerine yazacaktır.

Çözümler #

JSON yazmayı kolaylaştırmak için, sözdizimi hatalarını kontrol eden ve sözdizimi renklendirmesi sağlayan bir IDE kullanın. Eklentiler çoğu editör için mevcuttur.

Uygulamalar ve araçlar geliştirirken JSON’u dahili olarak ve bir protokol olarak kullanın, ancak bir insanın muhtemelen elle düzenlemesi gerekeceği yerlerde (hata ayıklama dışında) açığa çıkarmamaya çalışın.

Bu kullanım için daha uygun olan diğer biçimleri değerlendirin, örneğin:

  • Hjson, JSON’a ve JSON’dan sorunsuz bir şekilde dönüştürülebilir
  • TOML, INI dosyalarına benzer
  • YAML, daha fazla özellik ancak ek karmaşıklık pahasına

JSON Sözdizimi Kuralları #

JSON (JavaScript Object Notation) sözdizimi, JavaScript’in bir alt kümesini temel alır (ayrıca bkz. json.org).

Geçerli bir JSON ifadesi aşağıdaki veri türlerinden biri olabilir

  • basit veri türleri: String, Number, Boolean, Null
  • bileşik veri türleri: Değer, Nesne, Dizi

Basit veri türleri #

Bir JSON dizesi çift tırnak içine alınmalıdır ve sıfır veya daha fazla Unicode karakteri içerebilir; ters eğik çizgi kaçışlarına izin verilir. Kabul edilen JSON numaraları E notasyonunda. Boole “doğru”, “yanlış"lardan biridir. Null, ayrılmış “null” anahtar sözcüğüdür.

Veri türü Geçerli JSON örnekleri
### Dize "elma"
"Elma"
"\u00c4pfel\n"
""
### Sayı ‘3’
‘1.4’
-1.5e3
### Boole “doğru”
“yanlış”
### Boş “boş”

Bileşik veri türleri #

Değer #

Bir JSON Değeri şunlardan biri olabilir: Dize, Sayı, Boolean, Null, Object, Array.

Nesne #

Bir JSON Nesnesi, adın bir Dize olduğu ve bir JSON değerinin değerinin olduğu küme parantezleri içine alınmış ad:değer çiftlerinin virgülle ayrılmış sırasız bir koleksiyonudur.

Dizi #

Bir JSON Dizisi, sıralı bir JSON değerleri koleksiyonudur.

JSON dizisi örneği:

["home", "wooden"]

JSON nesnelerine örnekler:

{
    "id": 1,
    "name": "A wooden door",
    "price": 12.50,
    "tags": ["home", "wooden"]
}

[
  1,
  2,
  [3, 4, 5, 6],
  {
    "id": 1,
    "name": "A wooden door",
    "price": 12.50,
    "tags": ["home", "wooden"]
  }
]

Array vs Object için Rationale (yani ne zaman kullanılacak) #

JSON dizileri, bir nesne koleksiyonunu temsil eder. JS’de, “slice”, “pop”, “push” gibi bir grup toplama işlevi vardır. Nesneler sadece daha fazla ham veriye sahiptir.

Bir JSONArray, değerlerden oluşan sıralı bir dizidir. Harici metin biçimi, değerleri ayıran virgüllerle köşeli parantez içine alınmış bir dizedir.

Bir JSONObject, ad/değer çiftlerinden oluşan sırasız bir koleksiyondur. Dış biçimi, adlar ve değerler arasında iki nokta üst üste ve değerler ve adlar arasında virgül bulunan küme parantezlerine sarılmış bir dizedir.

Nesne - anahtar ve değer, Dizi - sayılar, dizeler, booleanlar. Bunu veya bunu ne zaman kullanıyorsun?

Dizileri “a/an” ve Nesneleri “a” olarak düşünebilirsiniz. Örnek olarak “Meyve” kullanalım. Meyve dizisindeki her öğe bir meyve türüdür.

‘dizi meyvesi: [portakal, mango, muz]’

Diziler nesneler, dizeler, sayılar, diziler içerebilir, ancak yalnızca nesneler ve dizilerle ilgilenilmesine izin verir.

meyve dizisi : [turuncu:[], mango:{}, muz:{}]

. Turuncunun da bir dizi olduğunu görebilirsiniz. Turuncuya dönüşen herhangi bir öğenin bir portakal türü olduğunu ima eder, örneğin: bitter_orange, mandarin, sweet_orange.

meyve nesnesi için, içindeki herhangi bir öğe meyvenin bir özelliğidir. böylece meyve bir

object fruit :{seed:{}, endocarp:{},flesh:{}}

Bu aynı zamanda tohum nesnesindeki herhangi bir şeyin tohumun özelliği olması gerektiği anlamına gelir, örneğin: renk, ..

JSON, öncelikle javascript nesnelerini dizelere serileştirmeye izin veren bir dildir. Bu nedenle, bir JSON dizesini seri durumdan çıkardıktan sonra bir javascript nesne yapısı almalısınız. Json’unuz, nesne1 ila nesne100 olarak adlandırılan 100 nesneyi depolayan bir nesneye seri hale getirirse, bu çok uygunsuz olacaktır. Çoğu seri kaldırıcı, dizeleri kullandığınız dilde gerçek nesne yapısına dönüştürebilmeleri için bilinen nesnelere ve bilinen nesne dizilerine sahip olmanızı bekler. Ayrıca bu, nesne yönelimli tasarım felsefesinin size cevap vereceği bir sorudur.
