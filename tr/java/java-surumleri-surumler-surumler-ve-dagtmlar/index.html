Java SE JRE veya Java SE JDK dağıtımları arasındaki farklar #

Java SE’nin Sun / Oracle sürümleri iki biçimde gelir: JRE ve JDK. Basit bir ifadeyle, JRE’ler Java uygulamalarını çalıştırmayı destekler ve JDK’lar ayrıca Java geliştirmeyi de destekler.

Java Çalışma Zamanı Ortamı #

Java Runtime Environment veya JRE dağıtımları, Java uygulamalarını çalıştırmak ve yönetmek için gereken kitaplıklar ve araçlardan oluşur. Tipik bir modern JRE’deki araçlar şunları içerir:

  • Bir JVM’de (Java Sanal Makinesi) bir Java programını çalıştırmak için java komutu
  • Nashorn Javascript motorunu çalıştırmak için ‘jjs’ komutu.
  • Java anahtar depolarını işlemek için “keytool” komutu.
  • Güvenlik sanal alanı güvenlik ilkelerini düzenlemek için “policytool” komutu.
  • Web dağıtımı için “pack200” dosyasını paketlemek ve açmak için “pack200” ve “unpack200” araçları.
  • Java CORBA ve RMI uygulamalarını destekleyen ‘orbd’, ‘rmid’, ‘rmiregistry’ ve ’tnameserv’ komutları.

“Masaüstü JRE” yükleyicileri, bazı web tarayıcılarına uygun bir Java eklentisi içerir. Bu, kasıtlı olarak “Sunucu JRE” installers.linux sistem çağrısı okuma kıyaslamasının dışında bırakılmıştır.

Java 7 güncelleme 6’dan itibaren, JRE yükleyicileri JavaFX’i (sürüm 2.2 veya üzeri) içermektedir.

Java Geliştirme kiti #

Bir Java Geliştirme Kiti veya JDK dağıtımı, JRE araçlarını ve Java yazılımı geliştirmek için ek araçları içerir. Ek araçlar tipik olarak şunları içerir:

  • Java kaynak kodunu (".java") bayt kodu dosyalarına (".class") derleyen javac komutu.
  • “jar” ve “jarsigner” gibi JAR dosyaları oluşturmaya yönelik araçlar
  • Aşağıdakiler gibi geliştirme araçları:
    • appletviewer for running applets
    • idlj the CORBA IDL to Java compiler
    • javah the JNI stub generator
    • native2ascii for character set conversion of Java source code
    • schemagen the Java to XML schema generator (part of JAXB)
    • serialver generate Java Object Serialization version string.
    • the wsgen and wsimport support tools for JAX-WS
  • Aşağıdakiler gibi teşhis araçları:
    • jdb the basic Java debugger
    • jmap and jhat for dumping and analysing a Java heap.
    • jstack for getting a thread stack dump.
    • javap for examining “.class” files.
  • Aşağıdakiler gibi uygulama yönetimi ve izleme araçları:
    • jconsole a management console,
    • jstat, jstatd, jinfo and jps for application monitoring

Tipik bir Sun / Oracle JDK kurulumu, Java kitaplıklarının kaynak kodunu içeren bir ZIP dosyası da içerir. Java 6’dan önce bu, herkese açık tek Java kaynak koduydu.

Java 6’dan itibaren, OpenJDK için tam kaynak kodu OpenJDK sitesinden indirilebilir. Genellikle (Linux) JDK paketlerine dahil değildir, ancak ayrı bir paket olarak mevcuttur.

Java EE, Java SE, Java ME ve JavaFX arasındaki farklar #

Java teknolojisi hem bir programlama dili hem de bir platformdur. Java programlama dili, belirli bir sözdizimi ve stili olan yüksek seviyeli nesne yönelimli bir dildir. Java platformu, Java programlama dili uygulamalarının çalıştığı belirli bir ortamdır.

Birkaç Java platformu vardır. Birçok geliştirici, hatta uzun süredir Java programlama dili geliştiricileri bile, farklı platformların birbirleriyle nasıl ilişkili olduğunu anlamıyor.

Java Programlama Dili Platformları #

Java programlama dilinin dört platformu vardır:

  • Java Platformu, Standart Sürüm (Java SE)

  • Java Platformu, Enterprise Edition (Java EE)

  • Java Platformu, Mikro Sürüm (Java ME)

  • Java FX

Tüm Java platformları, bir Java Sanal Makinesi (VM) ve bir uygulama programlama arabiriminden (API) oluşur. Java Sanal Makinesi, belirli bir donanım ve yazılım platformu için Java teknolojisi uygulamalarını çalıştıran bir programdır. API, diğer yazılım bileşenleri veya uygulamaları oluşturmak için kullanabileceğiniz bir yazılım bileşenleri topluluğudur. Her Java platformu bir sanal makine ve bir API sağlar ve bu, o platform için yazılmış uygulamaların Java programlama dilinin tüm avantajlarıyla uyumlu herhangi bir sistemde çalışmasına izin verir: platform bağımsızlığı, güç, kararlılık, geliştirme kolaylığı ve güvenlik.

Java SE #

Çoğu insan Java programlama dilini düşündüğünde, Java SE API’sini düşünür. Java SE’nin API’si, Java programlama dilinin temel işlevselliğini sağlar. Java programlama dilinin temel türleri ve nesnelerinden ağ oluşturma, güvenlik, veritabanı erişimi, grafik kullanıcı arabirimi (GUI) geliştirme ve XML ayrıştırma için kullanılan üst düzey sınıflara kadar her şeyi tanımlar.

Çekirdek API’ye ek olarak, Java SE platformu bir sanal makine, geliştirme araçları, dağıtım teknolojileri ve Java teknolojisi uygulamalarında yaygın olarak kullanılan diğer sınıf kitaplıkları ve araç takımlarından oluşur.

Java EE #

Java EE platformu, Java SE platformunun üzerine inşa edilmiştir. Java EE platformu, büyük ölçekli, çok katmanlı, ölçeklenebilir, güvenilir ve güvenli ağ uygulamaları geliştirmek ve çalıştırmak için bir API ve çalışma zamanı ortamı sağlar.

Java ME #

Java ME platformu, Java programlama dili uygulamalarını cep telefonları gibi küçük cihazlarda çalıştırmak için bir API ve az yer kaplayan bir sanal makine sağlar. API, Java SE API’nin bir alt kümesidir ve küçük cihaz uygulama geliştirme için kullanışlı özel sınıf kitaplıklarıdır. Java ME uygulamaları genellikle Java EE platform hizmetlerinin istemcileridir.

Java FX #

Java FX teknolojisi, Java FX ScriptTM ile yazılmış zengin internet uygulamaları oluşturmak için bir platformdur. Java FX Komut Dosyası, Java teknolojisi bayt koduyla derlenen ve daha sonra bir Java VM’sinde çalıştırılabilen, statik olarak yazılmış bir bildirim dilidir. Java FX platformu için yazılan uygulamalar Java programlama dili sınıflarını içerebilir ve bunlara bağlanabilir ve Java EE platform hizmetlerinin istemcileri olabilir.

  • Oracle dokümantasyonundan alınmıştır.

Oracle Hotspot ve OpenJDK arasındaki fark nedir #

JRE’ye karşı JDK dikotomisine dik, yaygın olarak kullanılabilen iki tür Java sürümü vardır:

  • Oracle Hotspot sürümleri, Oracle indirme sitelerinden indirdiğiniz sürümlerdir.
  • OpenJDK sürümleri, OpenJDK kaynak havuzlarından oluşturulan (genellikle üçüncü taraf sağlayıcılar tarafından) sürümlerdir.

İşlevsel açıdan, bir Hotspot sürümü ile bir OpenJDK sürümü arasında çok az fark vardır. Hotspot’ta Oracle (ücretli) Java müşterilerinin etkinleştirebileceği bazı ekstra “kurumsal” özellikler vardır, ancak bunun dışında aynı teknoloji hem Hotspot hem de OpenJDK’da mevcuttur.

Hotspot’un OpenJDK’ya göre bir başka avantajı, Hotspot için yama sürümlerinin biraz daha erken kullanılabilir olma eğiliminde olmasıdır. Bu aynı zamanda OpenJDK sağlayıcınızın ne kadar çevik olduğuna da bağlıdır; Örneğin. bir Linux dağıtımının derleme ekibinin yeni bir OpenJDK derlemesini hazırlaması ve QA’sını yapması ve onu genel depolarına alması ne kadar sürer.

İşin ters tarafı, Hotspot sürümlerinin çoğu Linux dağıtımı için paket havuzlarında mevcut olmamasıdır. Bu, Hotspot kullanıyorsanız Java yazılımınızı bir Linux makinesinde güncel tutmanın genellikle daha fazla iş olduğu anlamına gelir.
