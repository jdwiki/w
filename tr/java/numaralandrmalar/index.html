Java numaralandırmaları (’enum’ anahtar sözcüğü kullanılarak bildirilir), tek bir sınıfın büyük miktardaki sabitleri için kestirme sözdizimidir.

Sözdizimi #

  • [genel/korumalı/özel] enum Numaralandırma_adı { // Yeni bir numaralandırma bildir.
  • ENUM_CONSTANT_1[, ENUM_CONSTANT_2…]; // Enum sabitlerini bildir. Bu, numaralandırmanın içindeki ilk satır olmalı ve sonunda noktalı virgülle virgülle ayrılmalıdır.
  • ENUM_CONSTANT_1(param)[, ENUM_CONSTANT_2(param)…]; // Enum sabitlerini parametrelerle bildirin. Parametre türleri yapıcıyla eşleşmelidir.
  • ENUM_CONSTANT_1 {…}[, ENUM_CONSTANT_2 {…}…]; // Geçersiz kılınan yöntemlerle enum sabitlerini bildirin. Bu, numaralandırma soyut yöntemler içeriyorsa yapılmalıdır; tüm bu yöntemler uygulanmalıdır.
  • ENUM_CONSTANT.name() // Enum sabitinin adını içeren bir Dize döndürür.
  • ENUM_CONSTANT.ordinal() // Bu numaralandırma sabitinin sırasını, ilk sabite sıfır bir sıra atandığı enum bildirimindeki konumunu döndürür.
  • Enum_name.values() // Her çağrıldığında o numaralandırmanın her sabitini içeren yeni bir dizi (Enum_name[] türünden) döndürür.
  • Enum_name.valueOf(“ENUM_CONSTANT”) // ENUM_CONSTANT.name() öğesinin tersi – verilen adla enum sabitini döndürür.
  • Enum.valueOf(Enum_name.class, “ENUM_CONSTANT”) // Bir öncekinin eş anlamlısı: ENUM_CONSTANT.name() öğesinin tersi – verilen adla enum sabitini döndürür.

Kısıtlamalar #

Numaralandırmalar her zaman java.lang.Enum öğesini genişletir, bu nedenle bir numaralandırmanın bir sınıf. Ancak, birçok arabirimi uygulayabilirler.

İpuçları & Hileler #

Özel temsilleri nedeniyle daha verimli haritalar ve kümeler vardır. .oracle.com/javase/7/docs/api/java/util/EnumSet.html) anahtarları olarak numaralandırmalarla kullanılabilir. Bunlar genellikle uzman olmayan meslektaşlarından daha hızlı çalışır.

Temel bir enum bildirme ve kullanma #

Enum, bir dizi sabiti tanımlamak için derleme zamanında yalnızca birkaç kez bilinen bir kapalı sınıf için sözdizimi şekeri olarak kabul edilebilir.

Farklı mevsimleri listelemek için basit bir numaralandırma aşağıdaki gibi ilan edilecektir:

public enum Season {
    WINTER,
    SPRING,
    SUMMER,
    FALL
}

Enum sabitlerinin mutlaka tamamı büyük harflerle olması gerekmese de, sabitlerin adlarının alt çizgi ile ayrılmış sözcüklerle tamamen büyük harf olması Java kuralıdır.

Bir Enum’u kendi dosyasında bildirebilirsiniz:

/**
 * This enum is declared in the Season.java file.
*/
public enum Season {
    WINTER,
    SPRING,
    SUMMER,
    FALL
}

Ancak bunu başka bir sınıfın içinde de bildirebilirsiniz:

 public class Day {

    private Season season;

    public String getSeason() {
        return season.name();
    }

    public void setSeason(String season) {
        this.season = Season.valueOf(season);
    }

    /**
     * This enum is declared inside the Day.java file and 
     * cannot be accessed outside because it's declared as private.
     */
    private enum Season {
        WINTER,
        SPRING,
        SUMMER,
        FALL
    }

}

Son olarak, bir yöntem gövdesi veya yapıcısı içinde bir Enum bildiremezsiniz:

public class Day {

    /**
     * Constructor
    */
    public Day() {
        // Illegal. Compilation error
        enum Season {
            WINTER,
            SPRING,
            SUMMER,
            FALL
        }
    }

    public void aSimpleMethod() {
        // Legal. You can declare a primitive (or an Object) inside a method. Compile!
        int primitiveInt = 42;

        // Illegal. Compilation error.
        enum Season {
            WINTER,
            SPRING,
            SUMMER,
            FALL
        }

        Season season = Season.SPRING;
    }
    
}

Yinelenen numaralandırma sabitlerine izin verilmez:

public enum Season {
    WINTER,
    WINTER, //Compile Time Error : Duplicate Constants
    SPRING,
    SUMMER,
    FALL
}

Her enum sabiti varsayılan olarak public, static ve final şeklindedir. Her sabit “statik” olduğundan, bunlara doğrudan numaralandırma adı kullanılarak erişilebilir.

Enum sabitleri, yöntem parametreleri olarak iletilebilir:

public static void display(Season s) {
    System.out.println(s.name());  // name() is a built-in method that gets the exact name of the enum constant
}

display(Season.WINTER);  // Prints out "WINTER"

values() yöntemini kullanarak enum sabitlerinin bir dizisini alabilirsiniz. Değerlerin, döndürülen dizide bildirim sırasında olması garanti edilir:

Season[] seasons = Season.values();

Not: Bu yöntem her çağrıldığında yeni bir değer dizisi tahsis eder.

Enum sabitlerini yinelemek için:

public static void enumIterate() {
    for (Season s : Season.values()) {
        System.out.println(s.name());
    }
}

Numaralandırmaları bir “switch” ifadesinde kullanabilirsiniz:

public static void enumSwitchExample(Season s) {
    switch(s) {
        case WINTER:
            System.out.println("It's pretty cold");
            break;
        case SPRING:
            System.out.println("It's warming up");
            break;
        case SUMMER:
            System.out.println("It's pretty hot");
            break;
        case FALL:
            System.out.println("It's cooling down");
            break;
    }
}

Enum sabitlerini == kullanarak da karşılaştırabilirsiniz:

Season.FALL == Season.WINTER    // false
Season.SPRING == Season.SPRING  // true

Enum sabitlerini karşılaştırmanın başka bir yolu da aşağıdaki gibi “equals()” kullanmaktır; bu, aşağıdaki gibi kolayca tuzaklara düşebileceğiniz için kötü bir uygulama olarak kabul edilir:

Season.FALL.equals(Season.FALL); // true
Season.FALL.equals(Season.WINTER); // false
Season.FALL.equals("FALL"); // false and no compiler error

Ayrıca, “enum"daki örnekler kümesi çalışma zamanında değiştirilemese de, örneklerin kendileri doğası gereği değişmez değildir çünkü diğer herhangi bir sınıf gibi bir “enum” aşağıda gösterildiği gibi değişken alanlar içerebilir.

public enum MutableExample {
    A,
    B;

    private int count = 0;

    public void increment() {
        count++;
    }

    public void print() {
        System.out.println("The count of " + name() + " is " + count);
    }
}

// Usage:
MutableExample.A.print();       // Outputs 0
MutableExample.A.increment();
MutableExample.A.print();       // Outputs 1 -- we've changed a field   
MutableExample.B.print();       // Outputs 0 -- another instance remains unchanged

Bununla birlikte, iyi bir uygulama, “enum” örneklerini değişmez yapmaktır, yani, ek alanları olmadığında veya bu tür alanların tümü “final” olarak işaretlendiğinde ve kendileri değişmez olduğunda. Bu, uygulamanın ömrü boyunca bir “enum"un herhangi bir bellek sızdırmamasını ve örneklerini tüm iş parçacıklarında kullanmanın güvenli olmasını sağlayacaktır.

Numaralandırmalar dolaylı olarak “Seri hale getirilebilir” ve “Karşılaştırılabilir” öğelerini uygular çünkü “Enum” sınıfı şunları yapar:

public abstract class Enum<E extends Enum<E>>
extends Object
implements Comparable<E>, Serializable

Yapıcılı numaralandırmalar #

Bir “enum"un bir public kurucusu olamaz; ancak, özel kurucular kabul edilebilir (sayılamalar için kurucular varsayılan olarak paket-özel‘dir):

public enum Coin {
    PENNY(1), NICKEL(5), DIME(10), QUARTER(25); // usual names for US coins
    // note that the above parentheses and the constructor arguments match
    private int value;

    Coin(int value) { 
        this.value = value;
    }

    public int getValue() {
        return value;
    }
}

int p = Coin.NICKEL.getValue(); // the int value will be 5

Bir numaralandırma için sınırlı sayıda örnek olduğundan, tüm alanları özel tutmanız ve alıcı yöntemleri sağlamanız önerilir.

Bunun yerine bir “Enum"u “sınıf” olarak uygularsanız, şöyle görünür:

public class Coin<T extends Coin<T>> implements Comparable<T>, Serializable{
    public static final Coin PENNY = new Coin(1);
    public static final Coin NICKEL = new Coin(5);
    public static final Coin DIME = new Coin(10);
    public static final Coin QUARTER = new Coin(25);

    private int value;

    private Coin(int value){
        this.value = value;
    }

    public int getValue() {
        return value;
    }
}

int p = Coin.NICKEL.getValue(); // the int value will be 5

Enum sabitleri teknik olarak değiştirilebilir, bu nedenle bir enum sabitinin iç yapısını değiştirmek için bir ayarlayıcı eklenebilir. Ancak, bu çok kötü bir uygulama olarak kabul edilir ve bundan kaçınılmalıdır.

En iyi uygulama, Enum alanlarını “final” ile değişmez yapmaktır:

public enum Coin {
    PENNY(1), NICKEL(5), DIME(10), QUARTER(25);

    private final int value;

    Coin(int value){ 
        this.value = value;
    }

    ...

}

Aynı numaralandırmada birden çok kurucu tanımlayabilirsiniz. Bunu yaptığınızda, enum bildiriminizde ilettiğiniz argümanlar hangi kurucunun çağrılacağına karar verir:

public enum Coin {
    PENNY(1, true), NICKEL(5, false), DIME(10), QUARTER(25);

    private final int value;
    private final boolean isCopperColored;

    Coin(int value){
        this(value, false);
    }

    Coin(int value, boolean isCopperColored){ 
        this.value = value;
        this.isCopperColored = isCopperColored;
    }

    ...

}

Not: [Enum][3] sınıfı yaptığı için tüm ilkel olmayan numaralandırma alanları [Serializable][2] uygulamalıdır.

1: https://www.wikiod.com/tr/Java/gorunurluk-bir-snfn-uyelerine-erisimi-kontrol-etme#Paket Görünürlüğü [2]: https://docs.Oracle.com/javase/7/docs/api/java/io/Serializable.html [3]: https://docs.Oracle.com/javase/7/docs/api/java/lang/Enum.html

Soyut Yöntemlerle Numaralandırmalar #

Numaralandırmalar, her bir “enum” üyesinin uygulaması gereken soyut yöntemleri tanımlayabilir.

enum Action {
    DODGE {
        public boolean execute(Player player) {
            return player.isAttacking();
        }
    },
    ATTACK {
        public boolean execute(Player player) {
            return player.hasWeapon();
        }
    },
    JUMP {
        public boolean execute(Player player) {
            return player.getCoordinates().equals(new Coordinates(0, 0));
        }
    };

    public abstract boolean execute(Player player);
}

Bu, her numaralandırma üyesinin, üst düzey tanımdaki bir yöntemde türleri açmak zorunda kalmadan belirli bir işlem için kendi davranışını tanımlamasına olanak tanır.

Bu modelin, tipik olarak polimorfizm ve/veya uygulama arabirimleri kullanılarak elde edilenin kısa bir biçimi olduğuna dikkat edin.

Uygulama Arayüzü #

Bu, aynı zamanda, önceden derlenmiş düzenli ifade kalıplarına karşı “Dize” girdilerini test eden çağrılabilir bir işlev olan bir “enum"dur.

import java.util.function.Predicate;
import java.util.regex.Pattern;

enum RegEx implements Predicate<String> {
    UPPER("[A-Z]+"), LOWER("[a-z]+"), NUMERIC("[+-]?[0-9]+");

    private final Pattern pattern;

    private RegEx(final String pattern) {
        this.pattern = Pattern.compile(pattern);
    }

    @Override 
    public boolean test(final String input) {
        return this.pattern.matcher(input).matches();
    }
}

public class Main {
    public static void main(String[] args) {
        System.out.println(RegEx.UPPER.test("ABC"));
        System.out.println(RegEx.LOWER.test("abc"));
        System.out.println(RegEx.NUMERIC.test("+111"));
    }
}

Enum’un her üyesi de yöntemi uygulayabilir:

import java.util.function.Predicate;

enum Acceptor implements Predicate<String> {
    NULL {
        @Override
        public boolean test(String s) { return s == null; }
    },
    EMPTY {
        @Override
        public boolean test(String s) { return s.equals(""); }
    },
    NULL_OR_EMPTY {
        @Override
        public boolean test(String s) { return NULL.test(s) || EMPTY.test(s); }
    };
}

public class Main {
    public static void main(String[] args) {
        System.out.println(Acceptor.NULL.test(null));  // true
        System.out.println(Acceptor.EMPTY.test(""));   // true
        System.out.println(Acceptor.NULL_OR_EMPTY.test(" ")); // false
    }
}

Tek elemanlı bir numaralandırma ile Singleton desenini uygulayın #

Enum sabitleri, bir numaralandırmaya ilk kez başvurulduğunda somutlaştırılır. Bu nedenle, Singleton yazılım tasarım deseninin tek elemanlı bir numaralandırma ile uygulanmasına izin verir.

public enum Attendant {

    INSTANCE;

    private Attendant() {
        // perform some initialization routine
    }

    public void sayHello() {
        System.out.println("Hello!");
    }
}


public class Main {

    public static void main(String... args) {
        Attendant.INSTANCE.sayHello();// instantiated at this point
    }
}

Joshua Bloch’un “Etkili Java” kitabına göre, tek elemanlı bir numaralandırma, bir singleton’u uygulamanın en iyi yoludur. Bu yaklaşımın aşağıdaki avantajları vardır:

  • iplik güvenliği
  • tek örnekleme garantisi
  • kullanıma hazır serileştirme

Ve [arayüz uygular][2] bölümünde gösterildiği gibi, bu singleton ayrıca bir veya daha fazla arabirim uygulayabilir.

[2]: https://www.wikiod.com/tr/Java/numaralandrmalar#Arayüzü uygular “arayüz uygular”

Yöntemleri ve statik blokları kullanma #

Bir numaralandırma, herhangi bir sınıf gibi bir yöntem içerebilir. Bunun nasıl çalıştığını görmek için şöyle bir numaralandırma bildireceğiz:

public enum Direction {
    NORTH, SOUTH, EAST, WEST;
}

Enum’u ters yönde döndüren bir yöntemimiz olsun:

public enum Direction {
    NORTH, SOUTH, EAST, WEST;

    public Direction getOpposite(){
        switch (this){
            case NORTH:
                return SOUTH;               
            case SOUTH:
                return NORTH;                
            case WEST:
                return EAST; 
            case EAST:
                return WEST;  
            default: //This will never happen
                return null;
        }
    }
}

Bu, alanlar ve statik başlatıcı blokları kullanılarak daha da geliştirilebilir:

public enum Direction {
    NORTH, SOUTH, EAST, WEST;
    
    private Direction opposite;
    
    public Direction getOpposite(){
        return opposite;
    }
    
    static {
        NORTH.opposite = SOUTH;
        SOUTH.opposite = NORTH;
        WEST.opposite = EAST;
        EAST.opposite = WEST;
    }
}

Bu örnekte, ters yön, bir “Yön” ilk kez kullanıldığında statik olarak başlatılan özel bir “karşıt” örnek alanında depolanır. Bu özel durumda (“KUZEY”, “GÜNEY"e atıfta bulunduğundan ve bunun tersine), burada Yapıcılar ile Numaralandırmalar kullanamayız (Yapıcılar KUZEY(GÜNEY), GÜNEY(KUZEY), DOĞU(BATI), BATI(Doğu) ) daha zarif olurdu ve ‘karşıt’ın ’nihai’ olarak ilan edilmesine izin verir, ancak kendi kendine referans olur ve bu nedenle izin verilmez).

1: https://www.wikiod.com/tr/Java/numaralandrmalar#Yapıcılarla numaralandırma

Sınırlı tür parametresi olarak Enum #

Java’da jenerik içeren bir sınıf yazarken, type parametresinin bir enum olmasını sağlamak mümkündür. Tüm numaralandırmalar ‘Enum’ sınıfını genişlettiği için aşağıdaki sözdizimi kullanılabilir.

public class Holder<T extends Enum<T>> {
    public final T value;

    public Holder(T init) {
        this.value = init;
    }
}

Bu örnekte, T * türü bir numaralandırma olmalıdır*.

Enum sabiti belirli gövde #

Bir “enum"da, “enum"un varsayılan davranışını geçersiz kılan belirli bir “enum” sabiti için belirli bir davranış tanımlamak mümkündür, bu teknik sabit özel gövde olarak bilinir.

Üç piyano öğrencisinin - John, Ben ve Luke - “PianoClass” adlı bir “enum” içinde aşağıdaki gibi tanımlandığını varsayalım:

    enum PianoClass {
    JOHN, BEN, LUKE;
        public String getSex() {
            return "Male";
        }
        public String getLevel() {
            return "Beginner";
        }
    }

Ve bir gün diğer iki öğrenci - Rita ve Tom - öncekilerle eşleşmeyen bir cinsiyet (Kadın) ve seviye (Orta) ile geliyor:

    enum PianoClass2 {
    JOHN, BEN, LUKE, RITA, TOM;
        public String getSex() {
            return "Male"; // issue, Rita is a female
        }
        public String getLevel() {
            return "Beginner"; // issue, Tom is an intermediate student
        }
    }

böylece yeni öğrencileri sabit bildirime aşağıdaki gibi eklemek doğru değildir:

PianoClass2 tom = PianoClass2.TOM;
PianoClass2 rita = PianoClass2.RITA;
System.out.println(tom.getLevel()); // prints Beginner -> wrong Tom's not a beginner
System.out.println(rita.getSex()); // prints Male -> wrong Rita's not a male

“PianoClass2” varsayılan davranışını geçersiz kılan Rita ve Tom sabitlerinin her biri için aşağıdaki gibi belirli bir davranış tanımlamak mümkündür:

enum PianoClass3 {
    JOHN, BEN, LUKE,
    RITA {
        @Override
        public String getSex() {
            return "Female";
        }
    },
    TOM {
        @Override
        public String getLevel() {
            return "Intermediate";
        }
    };
    public String getSex() {
        return "Male";
    }
    public String getLevel() {
        return "Beginner";
    }
}

ve şimdi Tom’un seviyesi ve Rita’nın cinsiyeti olması gerektiği gibi:

PianoClass3 tom = PianoClass3.TOM;
PianoClass3 rita = PianoClass3.RITA;
System.out.println(tom.getLevel()); // prints Intermediate
System.out.println(rita.getSex()); // prints Female

İçeriğe özel gövdeyi tanımlamanın başka bir yolu da yapıcı kullanmaktır, örneğin:

enum Friend {
    MAT("Male"),
    JOHN("Male"),
    JANE("Female");
    
    private String gender;

    Friend(String gender) {
        this.gender = gender;
    }

    public String getGender() {
        return this.gender;
    }
}

ve kullanım:

Friend mat = Friend.MAT;
Friend john = Friend.JOHN;
Friend jane = Friend.JANE;
System.out.println(mat.getGender());     // Male
System.out.println(john.getGender());    // Male
System.out.println(jane.getGender());    // Female

Sıfır örnek enum #

enum Util {
    /* No instances */;

    public static int clamp(int min, int max, int i) {
        return Math.min(Math.max(i, min), max);
    }

    // other utility methods...
}

Tıpkı “enum” [tek tonlar için kullanılabilir](https://www.wikiod.com/tr/java/numaralandrmalar#Tek elemanlı bir numaralandırma ile Singleton desenini uygulayın) (1 örnek) gibi sınıflar), yardımcı sınıflar için kullanılabilir (0 örnek sınıfları). Enum sabitlerinin (boş) listesini ; ile sonlandırdığınızdan emin olun.

Özel kuruculara kıyasla profesyoneller ve olumsuzlar hakkında bir tartışma için http://stackoverflow.com/questions/25137490/zero-instance-enum-vs-private-constructors-for-preventing-instantiation sorusuna bakın.

Bir numaralandırmanın değerlerini alma #

Her enum sınıfı, ‘values()’ adlı örtük bir statik yöntem içerir. Bu yöntem, o numaralandırmanın tüm değerlerini içeren bir dizi döndürür. Değerler üzerinde yineleme yapmak için bu yöntemi kullanabilirsiniz. Bununla birlikte, bu yöntemin her çağrıldığında bir yeni dizi döndürdüğünü not etmek önemlidir.

public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY;
    
    /**
    * Print out all the values in this enum.
    */
    public static void printAllDays() {
        for(Day day : Day.values()) {
            System.out.println(day.name());
        }
    }
}

Bir ‘Set’e ihtiyacınız varsa, ‘EnumSet.allOf(Day.class)‘ı da kullanabilirsiniz.

Enum Polimorfizm Modeli #

Bir yöntemin “genişletilebilir” bir “enum” değerleri kümesini kabul etmesi gerektiğinde, programcı “enum"ların kullanılacağı her yerde kullanılacak bir arabirim oluşturarak normal bir “sınıf"taki gibi polimorfizmi uygulayabilir:

public interface ExtensibleEnum {
    String name();
}

Bu şekilde, arabirim tarafından etiketlenen (uygulayan) herhangi bir “enum” parametre olarak kullanılabilir ve programcının yöntem tarafından kabul edilecek değişken miktarda “enum” oluşturmasına olanak tanır. Bu, örneğin varsayılan (değiştirilemez) bir “enum"un bulunduğu ve bu API’lerin kullanıcısının “enum"u daha fazla değerle “genişletmek” istediği API’lerde yararlı olabilir.

Bir dizi varsayılan numaralandırma değeri aşağıdaki gibi tanımlanabilir:

public enum DefaultValues implements ExtensibleEnum {
    VALUE_ONE, VALUE_TWO;
}

Ek değerler daha sonra şu şekilde tanımlanabilir:

public enum ExtendedValues implements ExtensibleEnum {
    VALUE_THREE, VALUE_FOUR;
}

Numaralandırmaların nasıl kullanılacağını gösteren örnek - “printEnum()” öğesinin her iki “enum” türünden değerleri nasıl kabul ettiğini not edin:

private void printEnum(ExtensibleEnum val) {
    System.out.println(val.name());
}  

printEnum(DefaultValues.VALUE_ONE);    // VALUE_ONE
printEnum(DefaultValues.VALUE_TWO);    // VALUE_TWO
printEnum(ExtendedValues.VALUE_THREE); // VALUE_THREE
printEnum(ExtendedValues.VALUE_FOUR);  // VALUE_FOUR

Not: Bu model, bir numaralandırmada zaten tanımlanmış olan numaralandırma değerlerini başka bir numaralandırmada yeniden tanımlamanızı engellemez. Bu numaralandırma değerleri o zaman farklı örnekler olacaktır. Ayrıca, sahip olduğumuz tek şey gerçek “enum” değil, arayüz olduğu için anahtarlamalı numaralandırmayı kullanmak mümkün değildir.

Enum değerleri için Karşılaştır ve İçerir #

Numaralandırmalar yalnızca sabitleri içerir ve doğrudan == ile karşılaştırılabilir. Bu nedenle, yalnızca referans kontrolü gereklidir, .equals yöntemini kullanmaya gerek yoktur. Ayrıca, ‘.equals’ yanlış kullanılırsa, ‘==’ kontrolünde durum böyle değilken ‘NullPointerException’ı yükseltebilir.

enum Day {
    GOOD, AVERAGE, WORST;
}

public class Test {

    public static void main(String[] args) {
        Day day = null;

        if (day.equals(Day.GOOD)) {//NullPointerException!
            System.out.println("Good Day!");
        }

        if (day == Day.GOOD) {//Always use == to compare enum
            System.out.println("Good Day!");
        }

    }
}

Gruplamak, tamamlamak, sıralamak için farklı yöntemler içeren EnumSet sınıfımız var enum değerlerini.

  • EnumSet#range : İki uç nokta tarafından tanımlanan aralığa göre numaralandırmanın alt kümesini almak için

  • EnumSet#of : Herhangi bir aralık içermeyen belirli numaralandırmalar kümesi. Birden fazla aşırı yüklenmiş “of” yöntemi var.

  • EnumSet#complementOf : Yöntem parametresinde sağlanan numaralandırma değerlerinin tamamlayıcısı olan numaralandırma seti

    enum Page {
   A1, A2, A3, A4, A5, A6, A7, A8, A9, A10
}

public class Test {

  public static void main(String[] args) {
      EnumSet<Page> range = EnumSet.range(Page.A1, Page.A5);

      if (range.contains(Page.A4)) {
          System.out.println("Range contains A4");
      }

      EnumSet<Page> of = EnumSet.of(Page.A1, Page.A5, Page.A3);

      if (of.contains(Page.A1)) {
          System.out.println("Of contains A1");
      }
  }
}

İsme göre enum sabiti alın #

Diyelim ki bir DayOfWeek numaramız var:

enum DayOfWeek {
    SUNDAY, MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY;
}

Bir numaralandırma, adıyla bir sabit aramak için kullanılabilen yerleşik bir statik “valueOf()” yöntemiyle derlenir:

String dayName = DayOfWeek.SUNDAY.name();
assert dayName.equals("SUNDAY");

DayOfWeek day = DayOfWeek.valueOf(dayName);
assert day == DayOfWeek.SUNDAY;

Bu, dinamik bir numaralandırma türü kullanılarak da mümkündür:

Class<DayOfWeek> enumType = DayOfWeek.class;
DayOfWeek day = Enum.valueOf(enumType, "SUNDAY");
assert day == DayOfWeek.SUNDAY;

Belirtilen numaralandırma eşleşen bir ada sahip bir sabite sahip değilse, bu “valueOf()” yöntemlerinin ikisi de bir “IllegalArgumentException” oluşturur.

Guava kitaplığı, açık istisna işlemeyi ortadan kaldırmak için bir Guava [Opsiyonel][2] döndüren bir yardımcı yöntem Enums.getIfPresent() sağlar:

DayOfWeek defaultDay = DayOfWeek.SUNDAY;
DayOfWeek day = Enums.valueOf(DayOfWeek.class, "INVALID").or(defaultDay);
assert day == DayOfWeek.SUNDAY;

1: https://google.github.io/guava/releases/18.0/api/docs/com/google/common/base/Enums.html#getIfPresent(java.lang.Class,%20java.lang.String ) [2]: https://google.github.io/guava/releases/18.0/api/docs/com/google/common/base/Optional.html

Özellikli numaralandırma (alanlar) #

Eğer ’enum’u sadece sabit değerler olarak değil, daha fazla bilgi ile kullanmak istiyorsak ve iki numaralandırmayı karşılaştırabilmek istiyoruz.

Aşağıdaki örneği göz önünde bulundurun:

public enum Coin {
    PENNY(1), NICKEL(5), DIME(10), QUARTER(25);

    private final int value;

    Coin(int value){
        this.value = value;
    }

    public boolean isGreaterThan(Coin other){
        return this.value > other.value;
    }

}

Burada değerini temsil eden ‘Coin’ adında bir ‘Enum’ tanımladık. “isGreaterThan” yöntemiyle iki “enum"u karşılaştırabiliriz:

Coin penny = Coin.PENNY;
Coin dime = Coin.DIME;

System.out.println(penny.isGreaterThan(dime)); // prints: false
System.out.println(dime.isGreaterThan(penny)); // prints: true

enum’u String’e dönüştür #

Bazen numaranızı bir String’e dönüştürmek istersiniz, bunu yapmanın iki yolu vardır.

Sahip olduğumuzu varsayalım:

public enum Fruit {
    APPLE, ORANGE, STRAWBERRY, BANANA, LEMON, GRAPE_FRUIT;
}

Peki “Fruit.APPLE” gibi bir şeyi “ELMA”‘ya nasıl çevirebiliriz?


name() kullanarak dönüştürün #

“name()”, “enum” içinde, numaralandırmanın “Dize” gösterimini döndüren dahili bir yöntemdir, “Dize” dönüşü, numaralandırma değerinin nasıl tanımlandığını tam olarak temsil eder.

Örneğin:

System.out.println(Fruit.BANANA.name());      // "BANANA"
System.out.println(Fruit.GRAPE_FRUIT.name()); // "GRAPE_FRUIT"

toString() kullanarak dönüştürün #

“toString()”, varsayılan olarak, “name()” ile aynı davranışa sahip olacak şekilde geçersiz kılınmıştır.

Ancak, daha kullanıcı dostu bir “Dize” yazdırmasını sağlamak için “toString()” büyük olasılıkla geliştiriciler tarafından geçersiz kılınır.

Kodunuzu kontrol etmek istiyorsanız toString() kullanmayın, name() bunun için çok daha kararlıdır. Değeri günlüklere veya stdout’a veya başka bir şeye çıkaracağınız zaman toString() kullanın.

Varsayılan olarak: #

System.out.println(Fruit.BANANA.toString());      // "BANANA"
System.out.println(Fruit.GRAPE_FRUIT.toString()); // "GRAPE_FRUIT"

Geçersiz kılınma örneği #

System.out.println(Fruit.BANANA.toString());      // "Banana"
System.out.println(Fruit.GRAPE_FRUIT.toString()); // "Grape Fruit"

Statik alanlara sahip numaralandırmalar #

Enum sınıfınızın statik alanlara sahip olması gerekiyorsa, bunların enum değerlerinden sonra oluşturulduğunu unutmayın. Bu, aşağıdaki kodun bir “NullPointerException” ile sonuçlanacağı anlamına gelir:

enum Example {
    ONE(1), TWO(2);

    static Map<String, Integer> integers = new HashMap<>();

    private Example(int value) {
        integers.put(this.name(), value);
    }
}

Bunu düzeltmenin olası bir yolu:

enum Example {
    ONE(1), TWO(2);

    static Map<String, Integer> integers;

    private Example(int value) {
        putValue(this.name(), value);
    }

    private static void putValue(String name, int value) {
        if (integers == null)
            integers = new HashMap<>();
        integers.put(name, value);
    }
}

Statik alanı başlatmayın:

enum Example {
    ONE(1), TWO(2);

    // after initialisisation integers is null!!
    static Map<String, Integer> integers = null;

    private Example(int value) {
        putValue(this.name(), value);
    }

    private static void putValue(String name, int value) {
        if (integers == null)
            integers = new HashMap<>();
        integers.put(name, value);
    }
    // !!this may lead to null poiner exception!!
    public int getValue(){
        return (Example.integers.get(this.name()));
    }
}

başlatma:

  • enum değerlerini oluştur
  • yan etki olarak putValue() tamsayıları başlatır.
  • statik değerler ayarlanır
  • tamsayılar = boş; // numaralandırmalardan sonra yürütülür, böylece tam sayıların içeriği kaybolur
