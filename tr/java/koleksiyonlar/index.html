‘java.util’ içindeki koleksiyon çerçevesi, normal diziler tarafından sağlanamayan işlevselliğe sahip veri kümeleri için bir dizi genel sınıf sağlar.

Koleksiyonlar çerçevesi, ‘Liste’ ve ‘Set’ ana alt arabirimleri ve eşleme koleksiyonu ‘Harita<K,V>’ ile ‘Koleksiyon’ için arabirimler içerir. Koleksiyonlar, kök arabirimdir ve diğer birçok koleksiyon çerçevesi tarafından uygulanmaktadır.

Koleksiyonlar, içinde başka nesnelerin koleksiyonlarını depolayabilen nesnelerdir. Generics kullanarak bir koleksiyonda depolanan verilerin türünü belirtebilirsiniz.

Koleksiyonlar genellikle “java.util” veya “java.util.concurrent” ad alanlarını kullanır.

Java 1.4.2 ve altı, jenerikleri desteklemez. Bu nedenle, bir koleksiyonun içerdiği tür parametrelerini belirtemezsiniz. Tip güvenliğine sahip olmamanın yanı sıra, bir koleksiyondan doğru tipi geri almak için dökümleri de kullanmalısınız.

son sürüm

“Koleksiyon“ye ek olarak, bazılarının alt türleri olan birden çok ana koleksiyon türü vardır.

  • ‘Liste’ sıralı bir nesne koleksiyonudur. Bir diziye benzer, ancak bir boyut sınırı tanımlamaz. Uygulamalar, genellikle yeni unsurlara uyum sağlamak için dahili olarak boyut olarak büyüyecektir.
  • Set<E>, kopyalara izin vermeyen bir nesneler topluluğudur.
  • ‘SortedSet’, öğe sıralamasını da belirten bir ‘Set‘dir.
  • Map<K,V> bir anahtar/değer çiftleri topluluğudur.
  • ‘SortedMap<K,V>’, öğe sıralamasını da belirten bir ‘Harita<K,V>‘dir.

Java 5, yeni bir koleksiyon türü ekler:

  • “Kuyruk”, belirli bir sırada işlenmesi amaçlanan bir öğeler topluluğudur. Uygulama, bunun FIFO veya LIFO olup olmadığını belirtir. Bu, “Yığın” sınıfını geçersiz kılar.
son sürüm

Java 6, bazı yeni koleksiyon alt türleri ekler.

  • NavigableSet<E>, yerleşik özel gezinme yöntemlerine sahip bir Set<E>dir.
  • NavigableMap<K,V>, yerleşik özel navigasyon yöntemlerine sahip bir Harita<K,V>dir.
  • ‘Deque’, her iki uçtan da okunabilen bir ‘Sıra‘dir.
son sürüm

Yukarıdaki öğelerin hepsinin arayüzler olduğunu unutmayın. Bunları kullanmak için, ‘ArrayList’, ‘HashSet’, ‘HashMap’ veya ‘PriorityQueue’ gibi uygun uygulama sınıflarını bulmanız gerekir.

Her koleksiyon türü, farklı performans ölçütlerine ve kullanım senaryolarına sahip birden çok uygulamaya sahiptir.

[Liskov Değiştirme İlkesi](https://www.wikiod.com/tr/java/miras#Liskov İkame Prensibi) koleksiyon alt türleri için geçerlidir. Yani, bir ‘SortedSet’, ‘Set’ bekleyen bir işleve geçirilebilir. Koleksiyonların nasıl kullanılacağı hakkında daha fazla bilgi için Genel Bilgiler bölümündeki Sınırlı Parametreler hakkında bilgi edinmek de yararlıdır. sınıf mirası ile.

Kendi koleksiyonlarınızı oluşturmak istiyorsanız, arayüzü uygulamak yerine soyut sınıflardan birini (“AbstractList” gibi) devralmak daha kolay olabilir.

1.2’den önce, bunun yerine aşağıdaki sınıfları/arayüzleri kullanmanız gerekiyordu:

  • “ArrayList” yerine “Vektör”
  • “Harita” yerine “Sözlük”. Dictionary’nin de bir arayüzden ziyade soyut bir sınıf olduğunu unutmayın.
  • “HashMap” yerine “Hashtable”

Bu sınıflar eskidir ve modern kodda kullanılmamalıdır.

Bir döngü içindeki bir Listeden öğeleri kaldırma #

Bir döngü içindeyken listeden öğeleri çıkarmak zordur, bunun nedeni listenin indeksinin ve uzunluğunun değişmesidir.

Aşağıdaki liste göz önüne alındığında, beklenmedik bir sonuç verecek bazı örnekler ve doğru sonucu verecek bazı örnekler.

List<String> fruits = new ArrayList<String>();
fruits.add("Apple");
fruits.add("Banana");
fruits.add("Strawberry");

YANLIŞ #

for ifadesinin yinelemesinde kaldırılıyor “Muz” atlanıyor: #

Kod örneği yalnızca “Elma” ve “Çilek” yazdıracaktır. “Elma” silindikten sonra “0” dizinine geçtiği için “Muz” atlanır, ancak aynı zamanda “i”, “1” olarak artar.

for (int i = 0; i < fruits.size(); i++) {
    System.out.println (fruits.get(i)); 
    if ("Apple".equals(fruits.get(i))) {
         fruits.remove(i);
    }     
}

Geliştirilmiş for ifadesinde kaldırma İstisna Atar: #

Koleksiyon üzerinde yineleme ve aynı anda değiştirme nedeniyle.

atar: java.util.ConcurrentModificationException

for (String fruit : fruits) { 
    System.out.println(fruit);
    if ("Apple".equals(fruit)) {
        fruits.remove(fruit);
    }
}

DOĞRU #

Bir “Yineleyici” kullanarak while döngüsünü kaldırma #

Iterator<String> fruitIterator = fruits.iterator();
while(fruitIterator.hasNext()) {     
    String fruit = fruitIterator.next();     
    System.out.println(fruit);
    if ("Apple".equals(fruit)) {
        fruitIterator.remove();
    } 
}

“Yineleyici” arabiriminde, yalnızca bu durum için yerleşik bir “remove()” yöntemi bulunur. Ancak, bu yöntem belgelerde “isteğe bağlı” olarak işaretlenmiştir ve bir ‘UnsupportedOperationException’ atın.

atar: UnsupportedOperationException - if the remove operation is not supported by this iterator

Bu nedenle, bu işlemin desteklendiğinden emin olmak için belgelerin kontrol edilmesi tavsiye edilir (pratikte, koleksiyon bir 3. taraf kitaplığı aracılığıyla elde edilen değişmez bir koleksiyon değilse veya ‘Collections.unmodified…()’ öğelerinden birinin kullanımı değilse. yöntem, işlem neredeyse her zaman desteklenir).

Bir ‘Yineleyici’ kullanılırken, ‘Liste’nin ‘modCount’ değeri ‘Yineleyici’ oluşturulduğunda değiştirildiğinde bir ‘ConcurrentModificationException’ atılır. Bu, aynı iş parçacığında veya aynı listeyi paylaşan çok iş parçacıklı bir uygulamada olmuş olabilir.

“modCount”, bu listenin yapısal olarak değiştirilme sayısını sayan bir “int” değişkenidir. Yapısal bir değişiklik, esasen, “Koleksiyon” nesnesinde bir “add()” veya “remove()” işleminin çağrıldığı anlamına gelir (“Yineleyici” tarafından yapılan değişiklikler sayılmaz). “Yineleyici” oluşturulduğunda, bu “modCount"u depolar ve “Liste"nin her yinelemesinde, geçerli “modCount"un “Yineleyici” oluşturulduğunda ve aynı olup olmadığını kontrol eder. “modCount” değerinde bir değişiklik olursa bir “ConcurrentModificationException” oluşturur.

Bu nedenle, yukarıda belirtilen liste için aşağıdaki gibi bir işlem herhangi bir istisna oluşturmaz:

Iterator<String> fruitIterator = fruits.iterator();
fruits.set(0, "Watermelon");
while(fruitIterator.hasNext()){
    System.out.println(fruitIterator.next());
}

Ancak, bir “Yineleyici” başlatıldıktan sonra “Liste"ye yeni bir öğe eklemek, bir “ConcurrentModificationException” oluşturur:

Iterator<String> fruitIterator = fruits.iterator();
fruits.add("Watermelon");
while(fruitIterator.hasNext()){
    System.out.println(fruitIterator.next());    //ConcurrentModificationException here
}

Geriye doğru yineleme #

for (int i = (fruits.size() - 1); i >=0; i--) {
    System.out.println (fruits.get(i));
    if ("Apple".equals(fruits.get(i))) {
         fruits.remove(i);
    }
}

Bu hiçbir şeyi atlamaz. Bu yaklaşımın dezavantajı, çıktının ters olmasıdır. Ancak, çoğu durumda önemli olmayacak öğeleri kaldırdığınız yer. Bunu asla LinkedList ile yapmamalısınız.

İleriye doğru yineleme, döngü indeksini ayarlama #

for (int i = 0; i < fruits.size(); i++) {
    System.out.println (fruits.get(i)); 
    if ("Apple".equals(fruits.get(i))) {
         fruits.remove(i);
         i--;
    }     
}

Bu hiçbir şeyi atlamaz. “i” öğesi “Liste"den kaldırıldığında, orijinal olarak “i+1” dizininde konumlanan öğe yeni “i” öğesi olur. Bu nedenle, bir sonraki yinelemenin sonraki öğeyi atlamadan işlemesi için döngü ‘i’ değerini azaltabilir.

“Kaldırılması gerekenler” listesi kullanma #

ArrayList shouldBeRemoved = new ArrayList();
for (String str : currentArrayList) {
    if (condition) {
        shouldBeRemoved.add(str);
    }
}
currentArrayList.removeAll(shouldBeRemoved);

Bu çözüm, geliştiricinin doğru öğelerin daha temiz bir şekilde kaldırılıp kaldırılmadığını kontrol etmesini sağlar.

Java 8’de aşağıdaki alternatifler mümkündür. Kaldırmanın bir döngüde gerçekleşmesi gerekmiyorsa, bunlar daha temiz ve daha doğrudandır.

Akışı Filtreleme #

Bir “Liste” yayınlanabilir ve filtrelenebilir. İstenmeyen tüm öğeleri çıkarmak için uygun bir filtre kullanılabilir.

List<String> filteredList = 
    fruits.stream().filter(p -> !"Apple".equals(p)).collect(Collectors.toList());

Buradaki diğer tüm örneklerden farklı olarak, bu örneğin yeni bir “Liste” örneği oluşturduğunu ve orijinal “Liste"yi değiştirmediğini unutmayın.

‘removeIf’ kullanma #

Tüm gereken bir dizi öğeyi kaldırmaksa, bir akış oluşturma ek yükünü kaydeder.

fruits.removeIf(p -> "Apple".equals(p));

Java Koleksiyonları çerçevesi #

Bireysel veri değerlerinden bir “Liste” oluşturmanın basit bir yolu, “java.utils.Arrays” yöntemini “Arrays.asList” kullanmaktır:

List<String> data = Arrays.asList("ab", "bc", "cd", "ab", "bc", "cd");

Tüm standart koleksiyon uygulamaları, yapım sırasında tüm öğeleri yeni koleksiyona ekleyen bir argüman olarak başka bir koleksiyonu alan yapıcılar sağlar:

Set<String> set1 = new HashSet<>(data); // will add data keeping only unique values
SortedSet<String> set2 = new TreeSet<>(data); // will add data keeping unique values and sorting
Set<String> set3 = new LinkedHashSet<>(data); // will add data keeping only unique values and preserving the original order

Değişmez Boş Koleksiyonlar #

Bazen değişmez bir boş koleksiyon kullanmak uygundur. Collections sınıfı, bu tür koleksiyonları verimli bir şekilde elde etmek için yöntemler sağlar:

List<String> anEmptyList = Collections.emptyList();
Map<Integer, Date> anEmptyMap = Collections.emptyMap();
Set<Number> anEmptySet = Collections.emptySet();

Bu yöntemler geneldir ve döndürülen koleksiyonu, atandığı türe otomatik olarak dönüştürür. Yani, örn. “emptyList()” herhangi bir “List” türüne atanabilir ve aynı şekilde “emptySet()” ve “emptyMap()” için de atanabilir.

Bu yöntemler tarafından döndürülen koleksiyonlar değişmezdir, çünkü içeriklerini değiştirecek yöntemleri (“add”, “put” vb.) Bu koleksiyonlar, “null” kullanmak veya “yeni” ile nesneler oluşturmak yerine, öncelikle boş yöntem sonuçlarının veya diğer varsayılan değerlerin yerine geçmek için kullanışlıdır.

Değiştirilemez Koleksiyon #

Değiştirilebilir özelliği nedeniyle kötü amaçlı kod güvenlik açığına yol açabileceğinden, dahili bir koleksiyonu açığa çıkarmak bazen iyi bir uygulama değildir. Java, “salt okunur” koleksiyonlar sağlamak için değiştirilemez sürümlerini sağlar.

Değiştirilemez bir koleksiyon genellikle, koleksiyonun kendisinin değiştirilemeyeceğini garanti eden değiştirilebilir bir koleksiyonun bir kopyasıdır. Bunu değiştirme girişimleri, UnsupportedOperationException istisnasına neden olur.

Koleksiyonun içinde bulunan nesnelerin hala değiştirilebileceğini fark etmek önemlidir.

import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

public class MyPojoClass {
    private List<Integer> intList = new ArrayList<>();

    public void addValueToIntList(Integer value){
        intList.add(value);
    }
    
    public List<Integer> getIntList() {
        return Collections.unmodifiableList(intList);
    }
}

Değiştirilemeyen bir koleksiyonu değiştirmeye yönelik aşağıdaki girişim, bir özel durum oluşturacaktır:

import java.util.List;

public class App {

    public static void main(String[] args) {
        MyPojoClass pojo = new MyPojoClass();
        pojo.addValueToIntList(42);
        
        List<Integer> list = pojo.getIntList();
        list.add(69);
    }
}

çıktı:

Exception in thread "main" java.lang.UnsupportedOperationException
    at java.util.Collections$UnmodifiableCollection.add(Collections.java:1055)
    at App.main(App.java:12)

Tuzak: eşzamanlı değişiklik istisnaları #

Bu istisna, bir koleksiyon üzerinde yineleme nesnesi tarafından sağlananlar dışındaki yöntemler kullanılarak yinelenirken değiştirildiğinde ortaya çıkar. Örneğin, bir şapka listemiz var ve kulak kepçesi olan tüm şapkaları kaldırmak istiyoruz:

List<IHat> hats = new ArrayList<>();
hats.add(new Ushanka()); // that one has ear flaps
hats.add(new Fedora());
hats.add(new Sombrero());
for (IHat hat : hats) {
    if (hat.hasEarFlaps()) {
        hats.remove(hat);
    }
}

Bu kodu çalıştırırsak, kod koleksiyonu yinelerken değiştirdiği için ConcurrentModificationException ortaya çıkar. Aynı özel durum, aynı listeyle çalışan birden çok iş parçacığından biri koleksiyonu değiştirmeye çalışırken diğerleri üzerinde yinelenirse ortaya çıkabilir. Koleksiyonların birden fazla iş parçacığında eşzamanlı olarak değiştirilmesi doğal bir şeydir, ancak eşzamanlı programlama araç kutusundaki senkronizasyon kilitleri, eşzamanlı değişiklik için benimsenen özel koleksiyonlar, klonlanmış koleksiyonu başlangıçtan değiştirme vb. gibi olağan araçlarla ele alınmalıdır.

Alt Koleksiyonlar #

List subList(int fromIndex, int toIndex) #

Burada fromIndex kapsayıcıdır ve toIndex özeldir.

List list = new ArrayList(); 
List list1 = list.subList(fromIndex,toIndex);
  1. Liste, verilen aralıkta mevcut değilse, IndexOutofBoundException’ı atar.
  2. Liste1’de yapılan herhangi bir değişiklik, listedeki aynı değişiklikleri etkileyecektir. Buna yedekli koleksiyonlar denir.
  3. fromnIndex, toIndex’ten (fromIndex > toIndex) büyükse, IllegalArgumentException’ı atar.

Örnek:

List<String> list = new ArrayList<String>();
List<String> list = new ArrayList<String>();
list.add("Hello1"); 
list.add("Hello2"); 
System.out.println("Before Sublist "+list); 
List<String> list2 = list.subList(0, 1);
list2.add("Hello3"); 
System.out.println("After sublist changes "+list);

Çıktı:
Alt Listeden Önce [Merhaba1, Merhaba2]
Alt liste değiştikten sonra [Hello1, Hello3, Hello2]

subSet’i ayarla(fromIndex,toIndex) #

Burada fromIndex kapsayıcıdır ve toIndex özeldir.

Set set = new TreeSet(); 
Set set1 = set.subSet(fromIndex,toIndex);

Döndürülen küme, aralığının dışına bir öğe ekleme girişiminde bir IllegalArgumentException oluşturur.

Map subMap(fromKey,toKey) #

fromKey kapsayıcıdır ve toKey özeldir

Map map = new TreeMap();
Map map1 = map.get(fromKey,toKey);

fromKey toKey’den büyükse veya bu haritanın kendisinin kısıtlı bir aralığı varsa ve fromKey veya toKey aralığın sınırlarının dışındaysa, IllegalArgumentException’ı atar.

Tüm koleksiyonlar destekli koleksiyonları destekler, alt koleksiyonda yapılan değişikliklerin ana koleksiyonda da aynı değişikliğe sahip olacağı anlamına gelir.

Yineleyici kullanarak eşleşen öğeleri Listelerden kaldırma. #

Yukarıda, bir Döngü içindeki bir Listeden öğeleri kaldırmak için bir örnek fark ettim ve bu sefer ‘Yineleyici’ arayüzünü kullanarak kullanışlı olabilecek başka bir örnek düşündüm.
Bu, kurtulmak istediğiniz listelerdeki yinelenen öğelerle uğraşırken kullanışlı olabilecek bir hilenin gösterimidir.

Not: Bu, yalnızca Öğeleri bir döngü içindeki bir Listeden Kaldırma örneğine eklenir:

O halde listelerimizi her zamanki gibi tanımlayalım

    String[] names = {"James","Smith","Sonny","Huckle","Berry","Finn","Allan"};
    List<String> nameList = new ArrayList<>();

    //Create a List from an Array
    nameList.addAll(Arrays.asList(names));
    
    String[] removeNames = {"Sonny","Huckle","Berry"};
    List<String> removeNameList = new ArrayList<>();

    //Create a List from an Array
    removeNameList.addAll(Arrays.asList(removeNames));

Aşağıdaki yöntem, iki Collection nesnesini alır ve “nameList” içindeki öğelerle eşleşen “removeNameList"teki öğeleri kaldırma büyüsünü gerçekleştirir.

private static void removeNames(Collection<String> collection1, Collection<String> collection2) {
      //get Iterator.
    Iterator<String> iterator = collection1.iterator();
    
    //Loop while collection has items
    while(iterator.hasNext()){
        if (collection2.contains(iterator.next()))
            iterator.remove(); //remove the current Name or Item
    }
}

Yöntemi çağırmak ve ’nameList’ ve ‘removeNameList’i şu şekilde geçirmek ‘removeNames(nameList,removeNameList);’
Aşağıdaki çıktıyı üretecektir:

İsimleri kaldırmadan önce Dizi Listesi: James Smith Sonny Huckle Berry Finn Allan
İsimleri çıkardıktan sonra Dizi Listesi: James Smith Finn Allan

Listelerdeki yinelenen öğeleri kaldırmak için kullanışlı olabilecek Koleksiyonlar için basit ve temiz bir kullanım.

Koleksiyonlar ve İlkel Değerler #

Java’daki koleksiyonlar yalnızca nesneler için çalışır. yani Java’da Map<int, int> yoktur. Bunun yerine, “Harita<Tamsayı, Tamsayı>” örneğinde olduğu gibi, ilkel değerlerin nesnelere kutulanması gerekir. Java otomatik kutulama, bu koleksiyonların şeffaf kullanımını sağlayacaktır:

Map<Integer, Integer> map = new HashMap<>();
map.put(1, 17); // Automatic boxing of int to Integer objects
int a = map.get(1); // Automatic unboxing.

Ne yazık ki, bunun ek yükü önemli. Bir “HashMap<Tamsayı, Tamsayı>”, giriş başına yaklaşık 72 bayt gerektirir (örn. Gerçek veriler yalnızca 8 bayt olduğundan, bu çok büyük bir ek yük sağlar. Ayrıca, iki seviyeli dolaylılık gerektirir (Harita -> Giriş -> Değer) gereksiz yere yavaştır.

İlkel veri türleri için optimize edilmiş koleksiyonlara sahip birkaç kitaplık vardır (%50 yükte giriş başına yalnızca ~16 bayt gerektirir, yani 4 kat daha az bellek ve bir düzey daha az dolaylılık gerektirir), büyük ilkel koleksiyonları kullanırken önemli performans avantajları sağlayabilir. Java’daki değerler.

Yineleyici veya her döngü için kullanmak için kendi Yinelenebilir yapınızı oluşturma. #

Koleksiyonumuzun yineleyici veya her döngü için yinelenebilmesini sağlamak için aşağıdaki adımlara dikkat etmeliyiz:

  1. Üzerinde yinelemek istediğimiz şeyler ‘Yinelenebilir’ olmalı ve açığa çıkarılmalıdır. ‘yineleyici ()’.
  2. ‘hasNext()’, ’next()’ ve ‘remove()’ öğelerini geçersiz kılarak bir ‘java.util.Iterator’ tasarlayın.

Aşağıda, bağlantılı listeyi yinelenebilir hale getirmek için yukarıdaki varlıkları kullanan basit bir genel bağlantılı liste uygulaması ekledim.

package org.algorithms.linkedlist;
 
import java.util.Iterator;
import java.util.NoSuchElementException;
 
 
public class LinkedList<T> implements Iterable<T> {
 
    Node<T> head, current;
 
    private static class Node<T> {
        T data;
        Node<T> next;
 
        Node(T data) {
            this.data = data;
        }
    }
 
    public LinkedList(T data) {
        head = new Node<>(data);
    }
 
    public Iterator<T> iterator() {
        return new LinkedListIterator();
    }
 
    private class LinkedListIterator implements Iterator<T> {
 
        Node<T> node = head;
 
        @Override
        public boolean hasNext() {
            return node != null;
        }
 
        @Override
        public T next() {
            if (!hasNext())
                throw new NoSuchElementException();
            Node<T> prevNode = node;
            node = node.next;
            return prevNode.data;
        }
 
        @Override
        public void remove() {
            throw new UnsupportedOperationException("Removal logic not implemented.");
        }
    }
 
    public void add(T data) {
        Node current = head;
        while (current.next != null)
            current = current.next;
        current.next = new Node<>(data);
    }
 
}
 
class App {
    public static void main(String[] args) {
 
        LinkedList<Integer> list = new LinkedList<>(1);
        list.add(2);
        list.add(4);
        list.add(3);
 
        //Test #1
        System.out.println("using Iterator:");
        Iterator<Integer> itr = list.iterator();
        while (itr.hasNext()) {
            Integer i = itr.next();
            System.out.print(i + " ");
        }
 
        //Test #2
        System.out.println("\n\nusing for-each:");
        for (Integer data : list) {
            System.out.print(data + " ");
        }
    }
}

Çıktı

using Iterator:
1 2 4 3
using for-each:
1 2 4 3

Bu, Java 7+ sürümünde çalışacaktır. Aşağıdakileri değiştirerek de Java 5 ve Java 6’da çalışmasını sağlayabilirsiniz:

LinkedList<Integer> list = new LinkedList<>(1);

ile birlikte

LinkedList<Integer> list = new LinkedList<Integer>(1);

veya uyumlu değişiklikleri dahil ederek başka herhangi bir sürüm.
