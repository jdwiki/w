Java kullanarak geliştiriciler, başka bir sınıf içinde bir sınıf tanımlama yeteneğine sahiptir. Böyle bir sınıfa İç İçe Sınıf denir. Statik olmayan olarak bildirilmişlerse İç İçe Sınıflar, İç İçe Sınıflar olarak adlandırılırlar, değilse, yalnızca Statik İç İçe Sınıflar olarak adlandırılırlar. Bu sayfa, Java Nested ve Inner Classes’ın nasıl kullanılacağına ilişkin örneklerle ayrıntıları belgelemek ve sağlamak içindir.

Sözdizimi #

  • genel sınıf DışSınıf { genel sınıf İçSınıf { } } // İç sınıflar özel de olabilir
  • genel sınıf DışSınıf { genel statik sınıf StaticNestedClass { } } // Statik iç içe sınıflar ayrıca özel olabilir
  • public void method() { private class LocalClass { } } // Yerel sınıflar her zaman özeldir
  • SomeClass anonimClassInstance = yeni SomeClass() { }; // Anonim iç sınıflar adlandırılamaz, dolayısıyla erişim tartışmalıdır. ‘SomeClass()’ soyut ise, gövde tüm soyut yöntemleri uygulamalıdır.
  • SomeInterface anonimClassInstance = yeni SomeInterface() { }; // Gövde, tüm arabirim yöntemlerini uygulamalıdır.

Terminoloji ve sınıflandırma #

Java Dil Belirtimi (JLS), farklı Java sınıfı türlerini aşağıdaki gibi sınıflandırır:

üst düzey sınıf, iç içe sınıf olmayan bir sınıftır.

Bir iç içe sınıf, bildirimi başka bir sınıfın veya arabirimin gövdesi içinde gerçekleşen herhangi bir sınıftır.

Bir iç sınıf, açıkça veya dolaylı olarak static olarak bildirilmeyen iç içe geçmiş bir sınıftır.

Bir iç sınıf statik olmayan üye sınıf, yerel sınıf veya anonim sınıf olabilir. Bir arabirimin üye sınıfı, dolaylı olarak statiktir, dolayısıyla hiçbir zaman bir iç sınıf olarak düşünülmez.

Uygulamada programcılar, bir iç sınıf içeren bir üst düzey sınıfa “dış sınıf” olarak atıfta bulunur. Ayrıca, yalnızca (açıkça veya dolaylı olarak) statik iç içe sınıflara atıfta bulunmak için “iç içe sınıf” kullanma eğilimi vardır.

Anonim iç sınıflar ile lambdalar arasında yakın bir ilişki olduğunu unutmayın, ancak lambdalar sınıflardır.

anlamsal farklılıklar #

  • Üst düzey sınıflar “temel durum"dur. Erişim değiştirici anlambilimine dayalı normal görünürlük kurallarına tabi olan bir programın diğer bölümleri tarafından görülebilirler. Soyut değilse, erişim değiştiricilerine dayalı olarak ilgili oluşturucuların görünür olduğu herhangi bir kodla başlatılabilirler.

  • Statik iç içe sınıflar, iki istisna dışında, üst düzey sınıflarla aynı erişim ve örnekleme kurallarını takip eder:

  • Yuvalanmış bir sınıf “private” olarak bildirilebilir, bu da onu çevreleyen üst düzey sınıfın dışında erişilemez hale getirir.

  • Yuvalanmış bir sınıf, çevreleyen üst düzey sınıfın “özel” üyelerine ve test edilen sınıfın tümüne erişebilir.

Bu, statik iç içe sınıfları, sıkı bir soyutlama sınırı içinde birden çok “varlık türünü” temsil etmeniz gerektiğinde kullanışlı hale getirir; Örneğin. iç içe sınıflar “uygulama ayrıntılarını” gizlemek için kullanıldığında.

  • İç sınıflar, çevreleyen kapsamlarda bildirilen statik olmayan değişkenlere erişme yeteneği ekler:
  • Statik olmayan bir üye sınıf, örnek değişkenlere başvurabilir.
  • Bir yerel sınıf (bir yöntem içinde bildirilen), aynı zamanda, “son” olmaları koşuluyla, yöntemin yerel değişkenlerine de başvurabilir. (Java 8 ve sonraki sürümler için etkili bir şekilde nihai olabilirler.)
  • Anonim bir iç sınıf, bir sınıf veya bir yöntem içinde bildirilebilir ve aynı kurallara göre değişkenlere erişebilir.

Bir iç sınıf örneğinin, çevreleyen bir sınıf örneğindeki değişkenlere başvurabilmesi, somutlaştırma için çıkarımlara sahiptir. Spesifik olarak, bir iç sınıfın örneği oluşturulduğunda, örtük veya açık bir şekilde çevreleyen bir örnek sağlanmalıdır.

Yuvalanmış Sınıf Kullanan Basit Bir Yığın #

public class IntStack {

    private IntStackNode head;

    // IntStackNode is the inner class of the class IntStack
    // Each instance of this inner class functions as one link in the
    // Overall stack that it helps to represent
    private static class IntStackNode {

        private int val;
        private IntStackNode next;

        private IntStackNode(int v, IntStackNode n) {
            val = v;
            next = n;
        }
    }

    public IntStack push(int v) {
        head = new IntStackNode(v, head);
        return this;
    }

    public int pop() {
        int x = head.val;
        head = head.next;
        return x;
    }
}

Ve (özellikle) iç içe sınıfın varlığını hiç kabul etmeyen kullanımı.

public class Main {
    public static void main(String[] args) {
 
        IntStack s = new IntStack();
        s.push(4).push(3).push(2).push(1).push(0);

        //prints: 0, 1, 2, 3, 4, 
        for(int i = 0; i < 5; i++) {
            System.out.print(s.pop() + ", ");
        }            
    }
}

Statik ve Statik Olmayan İç İçe Sınıflar #

Yuvalanmış bir sınıf oluştururken, o yuvalanmış sınıfın statik olması seçeneğiyle karşı karşıya kalırsınız:

public class OuterClass1 {

    private static class StaticNestedClass {

    }

}

Veya statik olmayan:

public class OuterClass2 {

    private class NestedClass {

    }

}

Özünde, statik iç içe sınıflar , dış sınıfın çevreleyen bir örneğine sahip değildir, oysa statik olmayan iç içe sınıflar vardır. Bu, hem iç içe bir sınıfı başlatmaya nerede/ne zaman izin verildiğini hem de bu iç içe sınıfların hangi örneklerine erişmesine izin verildiğini etkiler. Yukarıdaki örneğe ek olarak:

public class OuterClass1 {

    private int aField;
    public void aMethod(){}

    private static class StaticNestedClass {
        private int innerField;

        private StaticNestedClass() {
             innerField = aField; //Illegal, can't access aField from static context 
             aMethod();           //Illegal, can't call aMethod from static context 
        }

        private StaticNestedClass(OuterClass1 instance) {
             innerField = instance.aField; //Legal
        }

    }

    public static void aStaticMethod() {
        StaticNestedClass s = new StaticNestedClass(); //Legal, able to construct in static context
        //Do stuff involving s...
    }

}

public class OuterClass2 {

    private int aField;

    public void aMethod() {}

    private class NestedClass {
        private int innerField;

        private NestedClass() {
             innerField = aField; //Legal   
             aMethod(); //Legal
        }
    }

    public void aNonStaticMethod() {
        NestedClass s = new NestedClass(); //Legal
    }

    public static void aStaticMethod() {
        NestedClass s = new NestedClass(); //Illegal. Can't construct without surrounding OuterClass2 instance.
                                         //As this is a static context, there is no surrounding OuterClass2 instance
    }
}

Bu nedenle, statik ve statik olmayan kararınız temel olarak dış sınıfın alanlarına ve yöntemlerine doğrudan erişebilmenizin gerekip gerekmediğine bağlıdır, ancak bunun aynı zamanda iç içe sınıfı ne zaman ve nerede oluşturabileceğiniz konusunda da sonuçları vardır.

Genel bir kural olarak, dış sınıfın alanlarına ve yöntemlerine erişmeniz gerekmedikçe iç içe sınıflarınızı statik yapın. Alanlarınızı herkese açık hale getirmenize benzer şekilde, bu, iç içe sınıf için kullanılabilir görünürlüğü azaltır (dış örneğe erişime izin vermeyerek), hata olasılığını azaltır.

İç Sınıflar için Erişim Değiştiricileri #

Java’daki Erişim Değiştiricilerinin tam açıklaması burada bulunabilir. Ama İç sınıflarla nasıl etkileşime giriyorlar?

‘public’, her zamanki gibi, türe erişebilen herhangi bir kapsama sınırsız erişim sağlar.

public class OuterClass {

    public class InnerClass {

        public int x = 5;

    }

    public InnerClass createInner() {
        return new InnerClass();
    }
}

public class SomeOtherClass {

    public static void main(String[] args) {
        int x = new OuterClass().createInner().x; //Direct field access is legal
    }
}

hem “korumalı” hem de varsayılan değiştirici (hiçbir şeyin olmaması) beklendiği gibi davranır, iç içe olmayan sınıflarda olduğu gibi.

‘private’, ilginçtir ki, ait olduğu sınıfla sınırlı değildir. Bunun yerine, derleme birimi olan .java dosyasıyla sınırlıdır. Bu, Dış sınıfların, “özel” olarak işaretlenmiş olsalar bile, İç sınıf alanlarına ve yöntemlerine tam erişime sahip olduğu anlamına gelir.

public class OuterClass {

    public class InnerClass {

        private int x;
        private void anInnerMethod() {}
    }

    public InnerClass aMethod() {
        InnerClass a = new InnerClass();
        a.x = 5; //Legal
        a.anInnerMethod(); //Legal
        return a;
    }
}

Inner Class’ın kendisi ‘public’ dışında bir görünürlüğe sahip olabilir. ‘özel’ veya başka bir kısıtlı erişim değiştirici olarak işaretlendiğinde, diğer (harici) sınıfların türü içe aktarmasına ve atamasına izin verilmeyecektir. Ancak yine de bu tür nesnelere referanslar alabilirler.

public class OuterClass {

    private class InnerClass{}

    public InnerClass makeInnerClass() {
        return new InnerClass();
    }
}

public class AnotherClass {

    public static void main(String[] args) {
        OuterClass o = new OuterClass();
     
        InnerClass x = o.makeInnerClass(); //Illegal, can't find type
        OuterClass.InnerClass x = o.makeInnerClass(); //Illegal, InnerClass has visibility private
        Object x = o.makeInnerClass(); //Legal
    }
}

Anonim İç Sınıflar #

Anonim bir iç sınıf, tek bir ifadeyle bildirilen ve somutlaştırılan bir iç sınıf biçimidir. Sonuç olarak, programın başka bir yerinde kullanılabilecek bir sınıf adı yoktur; yani anonimdir.

Anonim sınıflar genellikle parametre olarak geçirilecek hafif bir sınıf oluşturabilmeniz gereken durumlarda kullanılır. Bu genellikle bir arayüz ile yapılır. Örneğin:

public static Comparator<String> CASE_INSENSITIVE =
        new Comparator<String>() {
            @Override
            public int compare(String string1, String string2) {
                return string1.toUpperCase().compareTo(string2.toUpperCase());
            }
        };

Bu anonim sınıf, büyük/küçük harf farklarını göz ardı ederek iki dizeyi karşılaştıran bir Comparator<String> nesnesi (CASE_INSENSITIVE) tanımlar.

Anonim sınıflar kullanılarak sıklıkla uygulanan ve somutlaştırılan diğer arabirimler “Runnable” ve “Callable"dır. Örneğin:

// An anonymous Runnable class is used to provide an instance that the Thread
// will run when started.
Thread t = new Thread(new Runnable() {
        @Override 
        public void run() {
              System.out.println("Hello world");
        }
    });
t.start();  // Prints "Hello world"

Anonim iç sınıflar da sınıflara dayalı olabilir. Bu durumda, anonim sınıf, mevcut sınıfı örtük olarak “genişletir”. Genişletilen sınıf soyut ise, anonim sınıf tüm soyut yöntemleri uygulamalıdır. Ayrıca soyut olmayan yöntemleri geçersiz kılabilir.

yapıcılar #

Anonim bir sınıfın açık bir kurucusu olamaz. Bunun yerine, genişletilmekte olan sınıftaki bir kurucuya herhangi bir parametre iletmek için “super(…)” kullanan örtük bir kurucu tanımlanır. Örneğin:

SomeClass anon = new SomeClass(1, "happiness") {
            @Override
            public int someMethod(int arg) {
                // do something
            }
        };

Anonim ‘SomeClass’ alt sınıfımızın örtük yapıcısı, ‘SomeClass(int, String)’ çağrı imzasıyla eşleşen bir ‘SomeClass’ yapıcısını çağıracaktır. Herhangi bir yapıcı yoksa, bir derleme hatası alırsınız. Eşleşen kurucu tarafından oluşturulan istisnalar, örtük kurucu tarafından da atılır.

Doğal olarak, bir arabirim genişletilirken bu çalışmaz. Bir arabirimden anonim bir sınıf oluşturduğunuzda, sınıfların üst sınıfı, yalnızca bağımsız değişken olmayan bir kurucuya sahip olan “java.lang.Object” olur.

Dışarıdan statik olmayan bir iç sınıf örneği oluşturun #

Herhangi bir dış sınıf tarafından görülebilen bir iç sınıf da bu sınıftan oluşturulabilir.

İç sınıf, dış sınıfa bağlıdır ve bir örneğine referans gerektirir. İç sınıfın bir örneğini oluşturmak için, “yeni” operatörünün yalnızca dış sınıfın bir örneğinde çağrılması gerekir.

class OuterClass {

    class InnerClass {
    }
}

class OutsideClass {

    OuterClass outer = new OuterClass();
    
    OuterClass.InnerClass createInner() {
        return outer.new InnerClass();
    }
}

Kullanıma “outer.new” olarak dikkat edin.

Yöntem Yerel İç Sınıflar #

yöntem yerel iç sınıf adlı bir yöntemle yazılmış bir sınıf. Bu durumda, iç sınıfın kapsamı yöntem içinde sınırlandırılır.

Bir yöntem-yerel iç sınıf, yalnızca iç sınıfın tanımlandığı yöntem içinde somutlaştırılabilir.

Yerel iç sınıf yöntemini kullanma örneği:

public class OuterClass {
    private void outerMethod() {
       final int outerInt = 1;
        // Method Local Inner Class
        class MethodLocalInnerClass {
            private void print() {
                System.out.println("Method local inner class " + outerInt);
            }
        }
        // Accessing the inner class
        MethodLocalInnerClass inner = new MethodLocalInnerClass();
        inner.print();
    }

    public static void main(String args[]) {
        OuterClass outer = new OuterClass();
        outer.outerMethod();
    }
}

Yürütme bir çıktı verecektir: ‘Yöntem yerel iç sınıf 1’.

Statik olmayan bir iç sınıftan dış sınıfa erişme #

Dış sınıfa yapılan başvuru, sınıf adını ve “bu"nu kullanır.

public class OuterClass {
    public class InnerClass {
        public void method() {
            System.out.println("I can access my enclosing class: " + OuterClass.this);
        }
    }
}

Dış sınıfın alanlarına ve yöntemlerine doğrudan erişebilirsiniz.

public class OuterClass {
    private int counter;

    public class InnerClass {
        public void method() {
            System.out.println("I can access " + counter);
        }
    }
}

Ancak isim çakışması durumunda dış sınıf referansını kullanabilirsiniz.

public class OuterClass {
    private int counter;

    public class InnerClass {
        private int counter;
        
        public void method() {
            System.out.println("My counter: " + counter);
            System.out.println("Outer counter: " + OuterClass.this.counter);
            
            // updating my counter
            counter = OuterClass.this.counter;
        }
    }
}
