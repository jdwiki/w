Varsayımlar: #

  • Bir Oracle JDK kuruldu.
  • JDK, varsayılan dizine kuruldu.

Kurulum adımları #

  1. Windows Gezgini’ni açın.
  2. Soldaki gezinme bölmesinde Bu PC‘ye (veya daha eski Windows sürümleri için Bilgisayar) sağ tıklayın. Gerçek Windows sürümlerinde gezgini kullanmanın daha kısa bir yolu vardır: Sadece Win+Pause tuşlarına basın
  3. Yeni açılan Denetim Masası penceresinde, sol üst köşede olması gereken Gelişmiş Sistem Ayarları’na sol tıklayın. Bu Sistem Özellikleri penceresini açacaktır.

Alternatif olarak, Çalıştır‘a (Win+R) ‘SystemPropertiesAdvanced’ (büyük/küçük harfe duyarlı değil) yazın ve Enter tuşuna basın.

  1. Sistem Özellikleri’nin Gelişmiş sekmesinde, pencerenin sağ alt köşesindeki Ortam Değişkenleri… düğmesini seçin.
  2. Sistem Değişkenleri içindeki Yeni… düğmesini tıklatarak ‘JAVA_HOME’ adlı ve değeri JDK’nın bulunduğu dizinin yolu olan bir Yeni Sistem Değişkeni ekleyin. Kurulmuş. Bu değerleri girdikten sonra Tamam‘a basın.

  1. Sistem Değişkenleri listesini aşağı kaydırın ve ‘Yol’ değişkenini seçin.

  2. DİKKAT:** Windows, önemli programları bulmak için ‘Yolu’ kullanır. Herhangi biri veya tamamı kaldırılırsa, Windows düzgün çalışmayabilir. Windows’un JDK’yı çalıştırmasına izin vermek için değiştirilmelidir. Bunu akılda tutarak, seçili Yol değişkeni ile “Düzenle…” düğmesini tıklayın. “Yol” değişkeninin başına “%JAVA_HOME%\bin;” ekleyin.

Satırın başına eklemek daha iyidir çünkü Oracle’ın yazılımı kendi Java sürümlerini `Yol’a kaydederdi - Bu, Oracle’ın beyanından sonra ortaya çıkarsa sürümünüzün yok sayılmasına neden olur.

macOS’ta Java JDK yükleme #

Oracle Java 7 ve Java 8

macOS için Java 7 ve Java 8, Oracle’dan edinilebilir. Bu Oracle sayfası, Mac için Java ile ilgili birçok soruyu yanıtlıyor. 7u25’ten önceki Java 7’nin güvenlik nedeniyle Apple tarafından devre dışı bırakıldığını unutmayın.

Genel olarak, Oracle Java (Sürüm 7 ve üstü), macOS 10.7.3 veya sonraki bir sürümünü çalıştıran Intel tabanlı bir Mac gerektirir.

Oracle Java’nın Kurulumu

macOS için Java 7 & 8 JDK ve JRE yükleyicileri Oracle’ın web sitesinden indirilebilir:

  • Java 8 - Java SE İndirmeleri
  • Java 7 - Oracle Java Arşivi.

İlgili paketi indirdikten sonra pakete çift tıklayın ve normal kurulum sürecinden geçin. Bir JDK buraya kurulmalıdır:

/Library/Java/JavaVirtualMachines/<version>.jdk/Contents/Home

kurulu sürüme karşılık gelir.

Komut Satırı Geçişi

Java yüklendiğinde, yüklenen sürüm otomatik olarak varsayılan olarak ayarlanır. Farklı arasında geçiş yapmak için şunu kullanın:

export JAVA_HOME=/usr/libexec/java_home -v 1.6 #Or 1.7 or 1.8

Kullanım kolaylığı için ~/.bash_profile (varsayılan Bash kabuğunu kullanıyorsanız) öğesine aşağıdaki işlevler eklenebilir:

function java_version {
   echo 'java -version';
}

function java_set {
   if [[ $1 == "6" ]]
   then
       export JAVA_HOME='/usr/libexec/java_home -v 1.6';
       echo "Setting Java to version 6..."
       echo "$JAVA_HOME"
   elif [[ $1 == "7" ]]
   then
       export JAVA_HOME='/usr/libexec/java_home -v 1.7';
       echo "Setting Java to version 7..."
       echo "$JAVA_HOME"
   elif [[ $1 == "8" ]]
   then
       export JAVA_HOME='/usr/libexec/java_home -v 1.8';
       echo "Setting Java to version 8..."
       echo "$JAVA_HOME"
   fi            
}

/System/Library/Java/JavaVirtualMachines/1.6.0.jdk/Contents/Home

Java 6’nın kullanım ömrünün çok uzun zaman önce geçtiğini unutmayın, bu nedenle daha yeni bir sürüme yükseltme yapılması önerilir. Oracle web sitesinde Apple Java 6’yı yeniden yükleme hakkında daha fazla bilgi var.

Windows’a Java JDK veya JRE Yükleme #

Windows platformları için yalnızca Oracle JDK’ler ve JRE’ler mevcuttur. Kurulum prosedürü basittir:

  1. Oracle Java İndirilenler sayfasını ziyaret edin:
  2. JDK düğmesine, JRE düğmesine veya Sunucu JRE düğmesine tıklayın. Java kullanarak geliştirmek için JDK’ya ihtiyacınız olduğunu unutmayın. JDK ve JRE arasındaki farkı öğrenmek için buraya bakın.
  3. İndirmek istediğiniz sürüme ilerleyin. (Genel olarak konuşursak, en yenisi önerilir.)
  4. “Lisans Sözleşmesini Kabul Et” radyo düğmesini seçin.
  5. Windows x86 (32 bit) veya Windows x64 (64 bit) yükleyiciyi indirin.
  6. Yükleyiciyi … Windows sürümünüz için normal şekilde çalıştırın.

Komut istemini kullanarak Java’yı Windows’a yüklemenin alternatif bir yolu Chocolately kullanmaktır:

  1. https://chocolatey.org/ adresinden Chocolately’yi yükleyin

  2. Bir cmd örneği açın, örneğin Win+R tuşlarına basın ve ardından “Çalıştır” penceresine “cmd” yazın ve ardından enter yapın.

  3. Cmd örneğinizde, bir Java 8 JDK indirmek ve kurmak için aşağıdaki komutu çalıştırın:

     C:\> choco install jdk8

Taşınabilir sürümlerle çalışmaya başlama

VM gibi sınırlı ayrıcalıklara sahip bir sisteme JDK/JRE yüklemek isteyebileceğiniz veya JDK/JRE’nin birden çok sürümünü veya mimarisini (x64/x86) kurmak ve kullanmak isteyebileceğiniz durumlar vardır. Adımlar, yükleyiciyi (.EXE) indirene kadar aynı kalır. Bundan sonraki adımlar aşağıdaki gibidir (Adımlar JDK/JRE 7 ve üstü için geçerlidir, eski sürümler için klasör ve dosya adlarında biraz farklıdırlar):

  1. Dosyayı, Java ikili dosyalarının kalıcı olarak kalmasını istediğiniz uygun bir konuma taşıyın.

  2. Sınırlı ayrıcalıklarınız varsa 7-Zip’i veya taşınabilir sürümünü yükleyin.

  3. 7-Zip ile, dosyaları Java yükleyici EXE’sinden konuma çıkarın.

  4. Explorer’da klasörde Shift ve Sağ Tık tuşlarına basılı tutarak komut istemini açın veya herhangi bir yerden o konuma gidin.

  5. Yeni oluşturulan klasöre gidin. Diyelim ki klasör adı jdk-7u25-windows-x64. Bu yüzden ‘cd jdk-7u25-windows-x64’ yazın. Ardından sırasıyla aşağıdaki komutları yazın:

    cd .rsrc\JAVA_CAB10

    extrac32 111

  6. Bu, o konumda bir “tools.zip” dosyası oluşturacaktır. ’tools.zip’ dosyasını 7-Zip ile ayıklayın, böylece içindeki dosyalar aynı dizinde ’tools’ altında oluşturulur.

  7. Şimdi bu komutları önceden açılmış komut isteminde yürütün:

    cd tools

    for /r %x in (*.pack) do .\bin\unpack200 -r "%x" "%~dx%~px%~nx.jar"

  8. Komutun tamamlanmasını bekleyin. ’tools’ içeriğini ikili dosyalarınızın olmasını istediğiniz konuma kopyalayın.

Bu şekilde, yüklemeniz gereken herhangi bir JDK/JRE sürümünü aynı anda yükleyebilirsiniz.

Uygun bir Java SE sürümü seçme #

1995’teki orijinal Java 1.0 sürümünden bu yana Java’nın birçok sürümü olmuştur. (Özet için Java sürüm geçmişi’ne bakın.) Ancak çoğu sürüm resmi Kullanım Sonu tarihlerini geçmiştir. Bu, satıcının (genellikle Oracle şimdi) sürüm için yeni geliştirmeyi durdurduğu ve artık herhangi bir hata veya güvenlik sorunu için genel/ücretsiz yamalar sağlamadığı anlamına gelir. (Özel yama sürümleri genellikle destek sözleşmesi olan kişiler/kuruluşlar için mevcuttur; satıcınızın satış ofisi ile iletişime geçin.)

Genel olarak, kullanım için önerilen Java SE sürümü, en son genel sürüm için en son güncelleme olacaktır. Şu anda bu, mevcut en son Java 8 sürümü anlamına gelir. Java 9, 2017’de genel kullanıma sunulacaktır. (Java 7, Kullanım Ömrünün Sonunu geçmiştir ve son genel sürüm Nisan 2015’te olmuştur. Java 7 ve önceki sürümler önerilmez.)

Bu öneri, tüm yeni Java geliştirmeleri ve Java öğrenen herkes için geçerlidir. Ayrıca, yalnızca bir üçüncü tarafça sağlanan Java yazılımını çalıştırmak isteyen kişiler için de geçerlidir. Genel olarak konuşursak, iyi yazılmış Java kodu, Java’nın daha yeni bir sürümünde çalışır. (Ancak yazılımın sürüm notlarını kontrol edin ve şüpheleriniz varsa yazar/tedarikçi/satıcı ile iletişime geçin.)

Daha eski bir Java kod tabanı üzerinde çalışıyorsanız, kodunuzun Java’nın en son sürümünde çalıştığından emin olmanız önerilir. Daha yeni Java sürümlerinin özelliklerini kullanmaya ne zaman başlayacağınıza karar vermek daha zordur, çünkü bu, Java yüklemelerini yapamayan veya istemeyen müşterileri destekleme yeteneğinizi etkileyecektir.

Alternatifleri kullanarak Linux’ta Java sürümlerini yapılandırma ve değiştirme #

Alternatifleri Kullanmak #

Birçok Linux dağıtımı, bir komutun farklı sürümleri arasında geçiş yapmak için “alternatifler” komutunu kullanır. Bunu, bir makinede kurulu farklı Java sürümleri arasında geçiş yapmak için kullanabilirsiniz.

  1. Bir komut kabuğunda, $JDK’yı yeni yüklenen bir JDK’nın yol adına ayarlayın; Örneğin.

     $ JDK=/Data/jdk1.8.0_67

  2. Java SDK’sındaki komutları alternatiflere eklemek için ‘alternatives –install’ı kullanın:

     $ sudo alternatives --install /usr/bin/java java $JDK/bin/java 2
 $ sudo alternatives --install /usr/bin/javac javac $JDK/bin/javac 2
 $ sudo alternatives --install /usr/bin/jar jar $JDK/bin/jar 2

    And so on.

Artık bir Java komutunun farklı sürümleri arasında aşağıdaki gibi geçiş yapabilirsiniz:

$ sudo alternatives --config javac

There is 1 program that provides 'javac'.

  Selection    Command
-----------------------------------------------
*+ 1           /usr/lib/jvm/java-1.8.0-openjdk-1.8.0.101-1.b14.fc23.x86_64/bin/javac
   2           /Data/jdk1.8.0_67/bin/javac

Enter to keep the current selection[+], or type selection number: 2
$

‘Alternatifler’ kullanımı hakkında daha fazla bilgi için alternatifler(8) manuel girişine bakın.

Linux’ta kurulum sonrası kontrol ve konfigürasyon #

Bir Java SDK kurduktan sonra, kullanıma hazır olup olmadığını kontrol etmeniz önerilir. Bunu, normal kullanıcı hesabınızı kullanarak şu iki komutu çalıştırarak yapabilirsiniz:

$ java -version
$ javac -version

Bu komutlar, kabuğunuzun komut arama yolunda bulunan JRE ve JDK (sırasıyla) için sürüm bilgilerini yazdırır. JDK / JRE sürüm dizesini arayın.

  • Yukarıdaki komutlardan herhangi biri “komut bulunamadı” diyerek başarısız olursa, JRE veya JDK arama yolunda hiç değildir; aşağıdaki YOLU doğrudan yapılandırma bölümüne gidin.
  • Yukarıdaki komutlardan herhangi biri beklediğinizden farklı bir sürüm dizesi gösteriyorsa, arama yolunuzun veya “alternatifler” sisteminin ayarlanması gerekir; Alternatifleri Kontrol Etme bölümüne gidin
  • Doğru sürüm dizileri görüntüleniyorsa, işiniz neredeyse tamamlanmış demektir; Java_HOME’u Kontrol Etme bölümüne atlayın

PATH’i doğrudan yapılandırma

Şu anda arama yolunda ‘java’ veya ‘javac’ yoksa, basit çözüm onu ​​arama yolunuza eklemektir.

İlk olarak, Java’yı nereye kurduğunuzu bulun; şüpheniz varsa aşağıdaki Java nereye kuruldu? bölümüne bakın.

Ardından, komut kabuğunuzun “bash” olduğunu varsayarak, aşağıdaki satırları “~/.bash_profile” veya “~/.bashrc"nin sonuna eklemek için bir metin düzenleyici kullanın (Kabuğunuz olarak Bash kullanıyorsanız).

JAVA_HOME=<installation directory>
PATH=$JAVA_HOME/bin:$PATH

export JAVA_HOME
export PATH

… ““ni Java kurulum dizininizin yol adıyla değiştirmek. Yukarıdakilerin kurulum dizininin bir “bin” dizini içerdiğini ve “bin” dizininin kullanmaya çalıştığınız “java” ve “javac” komutlarını içerdiğini varsaydığını unutmayın.

Ardından, mevcut kabuğunuz için ortam değişkenlerinin güncellenmesi için az önce düzenlediğiniz dosyayı kaynaklayın.

$ source ~/.bash_profile

Ardından, “java” ve “javac” sürüm kontrollerini tekrarlayın. Hâlâ sorun varsa, ortam değişkenlerini doğru şekilde güncellediğinizi doğrulamak için “hangi java” ve “hangi javac"ı kullanın.

Son olarak, güncellenen ortam değişkenlerinin tüm mermilerinize geçiş yapması için oturumu kapatın ve tekrar oturum açın. Şimdi bitmiş olmalısın.

Alternatifleri Kontrol Etme

“Java -version” veya “javac -version” çalıştıysa ancak beklenmeyen bir sürüm numarası verdiyse, komutların nereden geldiğini kontrol etmeniz gerekir. Bunu aşağıdaki gibi bulmak için ‘hangi’ ve ’ls -l’ kullanın:

$ ls -l `which java`

Çıktı şöyle görünüyorsa, :

lrwxrwxrwx. 1 root root 22 Jul 30 22:18 /usr/bin/java -> /etc/alternatives/java

o zaman ‘alternatifler’ sürüm değiştirme kullanılıyor. Kullanmaya devam edip etmeyeceğinize veya doğrudan PATH ayarını yaparak geçersiz kılmanıza karar vermeniz gerekir.

  • Alternatifleri kullanarak Linux’ta Java sürümlerini yapılandırma ve değiştirme
  • Yukarıdaki “YOLU doğrudan yapılandırma” bölümüne bakın.

Java nereye kuruldu?

Java, kurulum yöntemine bağlı olarak çeşitli yerlere kurulabilir.

  • Oracle RPM’leri Java kurulumunu “/usr/java” içine yerleştirir.
  • Fedora’da varsayılan konum “/usr/lib/jvm"dir.
  • Java, ZIP veya JAR dosyalarından elle yüklendiyse, yükleme herhangi bir yerde olabilir.

Kurulum dizinini bulmakta zorlanıyorsanız, komutu bulmak için find (veya slocate) kullanmanızı öneririz. Örneğin:

$ find / -name java -type f 2> /dev/null

Bu size sisteminizde ‘java’ adı verilen tüm dosyaların yol adlarını verir. (Standart hatanın “/dev/null” olarak yeniden yönlendirilmesi, erişemediğiniz dosya ve dizinlerle ilgili mesajları gizler.)

Oracle Java’yı en son tar dosyasıyla Linux’a yükleme #

Oracle JDK’yı en son tar dosyasından yüklemek için aşağıdaki adımları izleyin:

  1. En son tar dosyasını [buradan] indirin(http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html) - En son sürüm Java SE Development Kit 8u112.

  2. Sudo ayrıcalıklarına ihtiyacınız var:

    sudo su

  3. jdk kurulumu için bir dizin oluşturun:

    mkdir /opt/jdk

  4. İndirilen tar dosyasını içine çıkarın:

    tar -zxf jdk-8u5-linux-x64.tar.gz -C /opt/jdk

  5. Dosyaların ayıklanıp ayıklanmadığını doğrulayın:

    ls /opt/jdk

  6. Oracle JDK’yı varsayılan JVM olarak ayarlama:

    update-alternatives --install /usr/bin/java java /opt/jdk/jdk1.8.0_05/bin/java 100

ve

   update-alternatives --install /usr/bin/javac javac /opt/jdk/jdk1.8.0_05/bin/javac 100

  1. Java sürümünü kontrol edin:

    java -version
