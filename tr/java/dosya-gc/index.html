Kanalı kullanarak dosya kopyalama #

Dosya içeriğini daha hızlı kopyalamak için “Kanal"ı kullanabiliriz. Bunu yapmak için, ‘FileChannel’ın ’transferTo()’ yöntemini kullanabiliriz.

import java.io.File;
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.IOException;
import java.nio.channels.FileChannel;

public class FileCopier {
    
    public static void main(String[] args) {
        File sourceFile = new File("hello.txt");
        File sinkFile = new File("hello2.txt");
        copy(sourceFile, sinkFile);
    }

    public static void copy(File sourceFile, File destFile) {
        if (!sourceFile.exists() || !destFile.exists()) {
            System.out.println("Source or destination file doesn't exist");
            return;
        }


        try (FileChannel srcChannel = new FileInputStream(sourceFile).getChannel();
             FileChannel sinkChanel = new FileOutputStream(destFile).getChannel()) {

            srcChannel.transferTo(0, srcChannel.size(), sinkChanel);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Akış ve Yazar/Okuyucu API’sı #

Akışlar ikili içeriğe en doğrudan erişimi sağlar, bu nedenle herhangi bir InputStream / [``OutputStream][2] uygulamaları her zaman intvebyte` üzerinde çalışır.

// Read a single byte from the stream
int b = inputStream.read();
if (b >= 0) { // A negative value represents the end of the stream, normal values are in the range 0 - 255
    // Write the byte to another stream
    outputStream.write(b);
}

// Read a chunk
byte[] data = new byte[1024];
int nBytesRead = inputStream.read(data);
if (nBytesRead >= 0) { // A negative value represents end of stream
    // Write the chunk to another stream
    outputStream.write(data, 0, nBytesRead);
}

Bazı istisnalar vardır, muhtemelen en önemlisi, “çeşitli veri değerlerinin temsillerini uygun bir şekilde yazdırma yeteneği” ekleyen PrintStream. Bu, System.out dosyasının hem ikili bir “InputStream” olarak hem de “System.out.println()” gibi yöntemler kullanılarak metin çıktısı olarak kullanılmasına izin verir.

Ayrıca, bazı akış uygulamaları, Java nesneleri (bkz. Serileştirme) veya yerel türler, örn. DataOutputStream / DataInputStream.

Writer ve Reader sınıfları ile Java ayrıca açık karakter akışları için bir API sağlar. Çoğu uygulama bu uygulamaları akışlara dayandıracak olsa da, karakter akışı API’si ikili içerik için herhangi bir yöntem göstermez.

// This example uses the platform's default charset, see below
// for a better implementation.

Writer writer = new OutputStreamWriter(System.out);
writer.write("Hello world!");

Reader reader = new InputStreamReader(System.in);
char singleCharacter = reader.read();

Karakterleri ikili verilere kodlamak gerektiğinde (örneğin, InputStreamWriter /OutputStreamWriter sınıflarını kullanırken), platformun varsayılan karakter kümesine bağlı olmak istemiyorsanız bir karakter kümesi belirtmelisiniz. Şüpheye düştüğünüzde, Unicode uyumlu bir kodlama kullanın, ör. Tüm Java platformlarında desteklenen UTF-8. Bu nedenle, muhtemelen her zaman varsayılan platform karakter setini kullanan ‘FileWriter’ ve ‘FileReader’ gibi sınıflardan uzak durmalısınız. Karakter akışlarını kullanarak dosyalara erişmenin daha iyi bir yolu şudur:

InputStream ve OutputStream kullanarak dosya kopyalama #

Bir döngü kullanarak verileri doğrudan bir kaynaktan veri havuzuna kopyalayabiliriz. Bu örnekte, bir InputStream’den veri okuyoruz ve aynı zamanda bir OutputStream’e yazıyoruz. Okumayı ve yazmayı bitirdiğimizde, kaynağı kapatmamız gerekiyor.

public void copy(InputStream source, OutputStream destination) throws IOException {
    try {
        int c;
        while ((c = source.read()) != -1) {
            destination.write(c);
        }
    } finally {
        if (source != null) {
            source.close();
        }
        if (destination != null) {
            destination.close();
        }
    }
}

Kanal ve Tampon kullanarak dosya okuma #

“Kanal”, verileri okumak/yazmak için bir “Tampon” kullanır. Tampon, bir kerede bir veri bloğu yazabileceğimiz sabit boyutlu bir kaptır. “Kanal”, akış tabanlı G/Ç’den oldukça hızlıdır.

Channel kullanarak bir dosyadan veri okumak için aşağıdaki adımlara ihtiyacımız var-

  1. Bir FileInputStream örneğine ihtiyacımız var. FileInputStream, bir Kanal döndüren getChannel() adında bir metoda sahiptir.

  2. FileInputStream’in getChannel() yöntemini çağırın ve Channel alın.

  3. Bir ByteBuffer oluşturun. ByteBuffer, sabit boyutlu bir bayt kapsayıcıdır.

  4. Kanalın bir okuma yöntemi var ve bu okuma yöntemine bir argüman olarak bir ByteBuffer sağlamamız gerekiyor. ByteBuffer’ın iki modu vardır - salt okunur ruh hali ve salt okunur ruh hali. flip() metod çağrısını kullanarak modu değiştirebiliriz. Tamponun bir konumu, sınırı ve kapasitesi vardır. Sabit bir boyutta bir arabellek oluşturulduğunda, sınırı ve kapasitesi boyutla aynıdır ve konum sıfırdan başlar. Bir arabellek verilerle yazılırken konumu giderek artar. Mod değiştirmek, pozisyonu değiştirmek anlamına gelir. Bir arabelleğin başlangıcından veri okumak için konumu sıfıra ayarlamamız gerekir. flip() yöntemi konumu değiştirir

  5. Kanalın read metodunu çağırdığımızda data kullanarak tamponu dolduruyor.

  6. Eğer ByteBufferdan veri okumamız gerekiyorsa, modunu salt yazma moduna değiştirmek için tamponu çevirmeli ve sonra tampondan veri okumaya devam etmeliyiz.

  7. Okunacak veri kalmadığında, kanalın read() yöntemi 0 veya -1 döndürür.

PrintStream kullanarak dosya yazma #

Dosya yazmak için PrintStream sınıfını kullanabiliriz. Herhangi bir veri türü değerini yazdırmanıza izin veren birkaç yöntemi vardır. println() yöntemi yeni bir satır ekler. Yazdırmayı bitirdiğimizde, `PrintStream’i temizlememiz gerekiyor.

import java.io.FileNotFoundException;
import java.io.PrintStream;
import java.time.LocalDate;

public class FileWritingDemo {
    public static void main(String[] args) {
        String destination = "file1.txt";

        try(PrintStream ps = new PrintStream(destination)){
            ps.println("Stackoverflow documentation seems fun.");
            ps.println();
            ps.println("I love Java!");
            ps.printf("Today is: %1$tm/%1$td/%1$tY", LocalDate.now());

            ps.flush();
        } catch (FileNotFoundException e) {
            e.printStackTrace();
        }

    }
}

Standart çıktıyı/hatayı engelleme veya yeniden yönlendirme #

Bazen kötü tasarlanmış bir üçüncü taraf kitaplığı, istenmeyen tanılamaları “System.out” veya “System.err” akışlarına yazar. Bunun için önerilen çözümler ya daha iyi bir kitaplık bulmak ya da (açık kaynak olması durumunda) sorunu çözmek ve geliştiricilere bir yama eklemek olacaktır.

Yukarıdaki çözümler uygulanabilir değilse, akışları yeniden yönlendirmeyi düşünmelisiniz.

Komut satırında yönlendirme

Bir UNIX’te, Linux veya MacOSX sistemi, > yeniden yönlendirme kullanılarak kabuktan yapılabilir. Örneğin:

$ java -jar app.jar arg1 arg2 > /dev/null 2>&1
$ java -jar app.jar arg1 arg2 > out.log 2> error.log

İlki, standart çıktıyı ve standart hatayı “/dev/null"a yönlendirir ve bu akışlara yazılan her şeyi atar. İkincisi, standart çıktıyı “out.log"a ve standart hatayı “error.log"a yönlendirir.

(Yönlendirme hakkında daha fazla bilgi için kullandığınız komut kabuğunun belgelerine bakın. Benzer tavsiyeler Windows için de geçerlidir.)

Alternatif olarak, yeniden yönlendirmeyi, Java uygulamasını başlatan bir sarmalayıcı komut dosyasında veya toplu iş dosyasında uygulayabilirsiniz.

Java uygulaması içinde yönlendirme

Ayrıca, bir Java uygulaması içindeki* akışları System.setOut() ve System.setErr() kullanarak yeniden yönlendirmek de mümkündür. Örneğin, aşağıdaki kod parçası, standart çıktıyı ve standart hatayı 2 günlük dosyasına yönlendirir:

System.setOut(new PrintStream(new FileOutputStream(new File("out.log"))));
System.setErr(new PrintStream(new FileOutputStream(new File("err.log"))));

Çıktıyı tamamen atmak istiyorsanız, geçersiz bir dosya tanıtıcısına “yazan” bir çıktı akışı oluşturabilirsiniz. Bu, işlevsel olarak UNIX’te “/dev/null” öğesine yazmaya eşdeğerdir.

System.setOut(new PrintStream(new FileOutputStream(new FileDescriptor())));
System.setErr(new PrintStream(new FileOutputStream(new FileDescriptor())));

Dikkat: setOut ve setErr nasıl kullandığınıza dikkat edin:

  1. Yönlendirme, JVM’nin tamamını etkileyecektir.
  2. Bunu yaparak, kullanıcının komut satırından akışları yeniden yönlendirme yeteneğini ortadan kaldırmış olursunuz.

ZIP dosyasının içeriğine erişme #

Java 7’nin FileSystem API’si, Java NIO dosya API’sini kullanarak diğer herhangi bir dosya sisteminde çalıştığı gibi bir Zip dosyasından veya bir Zip dosyasından girişlerin okunmasına ve eklenmesine izin verir.

Dosya Sistemi, kullanımdan sonra uygun şekilde kapatılması gereken bir kaynaktır, bu nedenle kaynaklarla dene bloğu kullanılmalıdır.

Mevcut bir dosyadan okuma #

Path pathToZip = Paths.get("path/to/file.zip");
try(FileSystem zipFs = FileSystems.newFileSystem(pathToZip, null)) {
  Path root = zipFs.getPath("/");
  ... //access the content of the zip file same as ordinary files
} catch(IOException ex) {
  ex.printStackTrace();
}

Yeni bir dosya oluşturma #

Map<String, String> env = new HashMap<>();  
env.put("create", "true"); //required for creating a new zip file
env.put("encoding", "UTF-8"); //optional: default is UTF-8
URI uri = URI.create("jar:file:/path/to/file.zip");
try (FileSystem zipfs = FileSystems.newFileSystem(uri, env)) {
  Path newFile = zipFs.getPath("/newFile.txt");
  //writing to file
  Files.write(newFile, "Hello world".getBytes());
} catch(IOException ex) {
  ex.printStackTrace();
}
