BigDecimal sınıfı, aritmetik (toplama, çıkarma, çarpma, bölme), ölçek işleme, yuvarlama işlemleri sağlar , karşılaştırma, karma ve biçim dönüştürme. BigDecimal, değişmez, keyfi kesinlik işaretli ondalık sayıları temsil eder. Bu sınıf, yüksek hassasiyetli bir hesaplama gerekliliğinde kullanılacaktır.

BigDecimals’ı Karşılaştırma #

“BigDecimals"ı karşılaştırmak için compareTo yöntemi kullanılmalıdır:

BigDecimal a = new BigDecimal(5);
a.compareTo(new BigDecimal(0));    // a is greater, returns 1
a.compareTo(new BigDecimal(5));    // a is equal, returns 0
a.compareTo(new BigDecimal(10));   // a is less, returns -1

Genellikle equals yöntemini kullanmamalısınız çünkü iki BigDecimals yalnızca değer ve **ölçek olarak eşitse eşit sayılır:

BigDecimal a = new BigDecimal(5);
a.equals(new BigDecimal(5));       // value and scale are equal, returns true
a.equals(new BigDecimal(5.00));    // value is equal but scale is not, returns false

BigDecimal ile matematiksel işlemler #

Bu örnek, BigDecimals kullanarak temel matematiksel işlemlerin nasıl gerçekleştirileceğini gösterir.

1.İlave #

BigDecimal a = new BigDecimal("5");
BigDecimal b = new BigDecimal("7");

//Equivalent to result = a + b

3.Çarpma #

İki BigDecimal çarpıldığında, sonuç, işlenenlerin ölçeklerinin toplamına eşit bir ölçeğe sahip olacaktır.

BigDecimal a = new BigDecimal("5.11");
BigDecimal b = new BigDecimal("7.221");

//Equivalent to result = a * b
BigDecimal result = a.multiply(b);
System.out.println(result);

Sonuç : 36.89931

Sonucun ölçeğini değiştirmek için, özellikle sonucun kesinliği ve yuvarlama modu olmak üzere operatörler için kuralları açıklayan bir nesne olan ‘MathContext’in geçmesine izin veren aşırı yüklenmiş çarpma yöntemini kullanın. Kullanılabilir yuvarlama modları hakkında daha fazla bilgi için lütfen Oracle Belgelerine bakın.

BigDecimal a = new BigDecimal("5.11");
BigDecimal b = new BigDecimal("7.221");

MathContext returnRules = new MathContext(4, RoundingMode.HALF_DOWN);

//Equivalent to result = a * b
BigDecimal result = a.multiply(b, returnRules);
System.out.println(result);

Sonuç : 36.90

4.Bölüm #

Bölme, diğer aritmetik işlemlerden biraz daha karmaşıktır, örneğin aşağıdaki örneğe bakalım:

BigDecimal a = new BigDecimal("5");
BigDecimal b = new BigDecimal("7");

BigDecimal result = a.divide(b);
System.out.println(result);

Bunun 0.7142857142857143’e benzer bir şey vermesini beklerdik, ancak şunu elde ederiz:

Sonuç: java.lang.ArithmeticException: Sonlanmayan ondalık genişletme; tam temsil edilebilir ondalık sonuç yok.

Bu, eğer 5’i 2’ye bölmek isteseydim, sonuç sonlandırıcı bir ondalık sayı olduğunda mükemmel şekilde çalışırdı, ancak bölündüklerinde sonlandırıcı olmayan bir sonuç verecek olan sayılar için bir ‘ArithmeticException’ alırdık. Gerçek dünya senaryosunda, bölme sırasında karşılaşılacak değerler tahmin edilemez, bu nedenle BigDecimal bölümü için Ölçek ve Yuvarlama Modu belirtmemiz gerekir. Ölçek ve Yuvarlama Modu hakkında daha fazla bilgi için Oracle Belgelerine bakın.

Örneğin şunları yapabilirim:

BigDecimal a = new BigDecimal("5");
BigDecimal b = new BigDecimal("7");

//Equivalent to result = a / b (Upto 10 Decimal places and Round HALF_UP)
BigDecimal result = a.divide(b,10,RoundingMode.HALF_UP);
System.out.println(result);

Sonuç: 0.7142857143

BigDecimal.valueOf() #

BigDecimal sınıfı, sık kullanılan sayıların dahili bir önbelleğini içerir; 0 ila 10. BigDecimal.valueOf() yöntemleri, benzer tür parametreleri olan yapıcılara tercihli olarak sağlanır, yani aşağıdaki örnekte a, b’ye tercih edilir.

BigDecimal a = BigDecimal.valueOf(10L); //Returns cached Object reference
BigDecimal b = new BigDecimal(10L); //Does not return cached Object reference

BigDecimal a = BigDecimal.valueOf(20L); //Does not return cached Object reference
BigDecimal b = new BigDecimal(20L); //Does not return cached Object reference


BigDecimal a = BigDecimal.valueOf(15.15); //Preferred way to convert a double (or float) into a BigDecimal, as the value returned is equal to that resulting from constructing a BigDecimal from the result of using Double.toString(double)
BigDecimal b = new BigDecimal(15.15); //Return unpredictable result

BigDecimals’ın sıfır, bir veya on değeriyle başlatılması #

BigDecimal, sıfır, bir ve on sayıları için statik özellikler sağlar. Gerçek sayıları kullanmak yerine bunları kullanmak iyi bir uygulamadır:

  • BigDecimal.ZERO
  • BigDecimal.ONE
  • BigDecimal.TEN

Statik özellikleri kullanarak gereksiz bir örneklemeden kaçınırsınız, ayrıca kodunuzda ‘sihirli sayı’ yerine bir hazır bilginiz olur.

//Bad example:
BigDecimal bad0 = new BigDecimal(0);
BigDecimal bad1 = new BigDecimal(1);
BigDecimal bad10 = new BigDecimal(10);

//Good Example:
BigDecimal good0 = BigDecimal.ZERO;
BigDecimal good1 = BigDecimal.ONE;
BigDecimal good10 = BigDecimal.TEN;

BigDecimal nesneleri değişmez #

BigDecimal ile hesaplamak istiyorsanız, BigDecimal nesneleri değişmez olduğu için döndürülen değeri kullanmanız gerekir:

BigDecimal a = new BigDecimal("42.23");
BigDecimal b = new BigDecimal("10.001");

a.add(b); // a will still be 42.23

BigDecimal c = a.add(b); // c will be 52.231
