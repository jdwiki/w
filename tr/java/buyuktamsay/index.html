BigInteger sınıfı, ilkel veri türleri için büyüklükleri çok büyük olan büyük tam sayıları içeren matematiksel işlemler için kullanılır. . Örneğin, 100 faktöriyel 158 basamaktır - bir “uzun"un gösterebileceğinden çok daha büyüktür. “BigInteger”, Java’nın tüm ilkel tamsayı operatörlerine ve diğer birkaç işlemin yanı sıra “java.lang.Math"deki tüm ilgili yöntemlere analoglar sağlar.

Sözdizimi #

  • BigInteger değişken_adı = new BigInteger(“12345678901234567890”); // dize olarak bir ondalık tamsayı
  • BigInteger değişken_adı = new BigInteger(“10101011010101001010100110001100111010110001111000101011010010”, 2) // dize olarak ikili bir tamsayı
  • BigInteger değişken_adı = new BigInteger(“ab54a98ceb1f0800”, 16) // dize olarak onaltılık bir tamsayı
  • BigInteger değişken_adı = new BigInteger(64, new Random()); // bir tamsayı oluşturmak için 64 bit sağlayan bir sözde rasgele sayı üreteci
  • BigInteger değişken_adı = yeni BigInteger(yeni bayt[]{0, -85, 84, -87, -116, -21, 31, 10, -46}); // bir tamsayının ikinin tümleyen temsilini imzaladı (büyük endian)
  • BigInteger değişken_adı = yeni BigInteger(1, yeni bayt[]{-85, 84, -87, -116, -21, 31, 10, -46}); // pozitif bir tamsayının işaretsiz ikinin tümleyen temsili (büyük endian)

“BigInteger” değiştirilemez. Bu nedenle durumunu değiştiremezsiniz. Örneğin, değişmezlik nedeniyle “toplam” güncellenmeyeceğinden aşağıdakiler çalışmayacaktır.

BigInteger sum = BigInteger.ZERO;
for(int i = 1; i < 5000; i++) {
   sum.add(BigInteger.valueOf(i));  
}

Çalışması için sonucu “sum” değişkenine atayın.

sum = sum.add(BigInteger.valueOf(i));

BigIntegerin resmi belgeleri, “BigInteger” uygulamalarının -2< arasındaki tüm tam sayıları desteklemesi gerektiğini belirtir. sup>2147483647 ve 22147483647 (özel). Bu, BigIntegerların 2 milyar bitten daha fazlasına sahip olabileceği anlamına gelir!

Başlatma #

java.math.BigInteger sınıfı, Java’nın tüm ilkel tamsayı operatörlerine ve java.lang.Math içindeki tüm ilgili yöntemlere işlem analogları sağlar. “Java.math” paketi otomatik olarak kullanıma sunulmadığından, basit sınıf adını kullanmadan önce “java.math.BigInteger” dosyasını içe aktarmanız gerekebilir.

“long” veya “int” değerlerini “BigInteger"e dönüştürmek için şunu kullanın:

long longValue = Long.MAX_VALUE;
BigInteger valueFromLong = BigInteger.valueOf(longValue);

veya, tamsayılar için:

int intValue = Integer.MIN_VALUE; // negative
BigInteger valueFromInt = BigInteger.valueOf(intValue);

bu, negatif değerler için işaret bit uzantısını kullanarak ‘intValue’ tamsayısını uzunluğa genişletecek, böylece negatif değerler negatif kalacaktır.

Sayısal bir “Dize"yi “BigInteger"e dönüştürmek için şunu kullanın:

String decimalString = "-1";
BigInteger valueFromDecimalString = new BigInteger(decimalString);

Aşağıdaki kurucu, belirtilen tabandaki bir “BigInteger"in Dize gösterimini bir “BigInteger"e çevirmek için kullanılır.

String binaryString = "10";
int binaryRadix = 2;
BigInteger valueFromBinaryString = new BigInteger(binaryString , binaryRadix);

Java ayrıca baytların bir “BigInteger” örneğine doğrudan dönüştürülmesini de destekler. Şu anda yalnızca imzalı ve imzasız büyük endian kodlaması kullanılabilir:

byte[] bytes = new byte[] { (byte) 0x80 }; 
BigInteger valueFromBytes = new BigInteger(bytes);

Bu, ilk bit işaret biti olarak yorumlandığından, -128 değerine sahip bir “BigInteger” örneği oluşturacaktır.

byte[] unsignedBytes = new byte[] { (byte) 0x80 };
int sign = 1; // positive
BigInteger valueFromUnsignedBytes = new BigInteger(sign, unsignedBytes);

Bu, baytlar işaretsiz sayı olarak yorumlandığından ve işaret açıkça pozitif bir sayı olan 1’e ayarlandığından, 128 değerinde bir “BigInteger” örneği oluşturur.

Ortak değerler için önceden tanımlanmış sabitler vardır:

  • BigInteger.ZERO — “0” değeri.
  • “BigInteger.ONE” — “1” değeri.
  • “BigInteger.TEN” — “10” değeri.

Ayrıca “BigInteger.TWO” (“2” değeri) vardır, ancak “özel” olduğu için kodunuzda kullanamazsınız.

BigInteger Matematiksel İşlem Örnekleri #

BigInteger değişmez bir nesnededir, bu nedenle herhangi bir matematiksel işlemin sonuçlarını yeni bir BigInteger örneğine atamanız gerekir.

İlave: 10 + 10 = 20

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("10");

BigInteger sum = value1.add(value2);
System.out.println(sum);

çıktı: 20

Çıkarma: 10 - 9 = 1

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("9");

BigInteger sub = value1.subtract(value2);
System.out.println(sub);

çıktı: 1

Bölüm: 10 / 5 = 2

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("5");

BigInteger div = value1.divide(value2);
System.out.println(div);

çıktı: 2

Bölüm: 17/4 = 4

BigInteger value1 = new BigInteger("17");
BigInteger value2 = new BigInteger("4");

BigInteger div = value1.divide(value2);
System.out.println(div);

çıkış: 4

Çarpma işlemi: 10 * 5 = 50

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("5");

BigInteger mul = value1.multiply(value2);
System.out.println(mul);

çıktı: 50

Güç: 10 ^ 3 = 1000

BigInteger value1 = new BigInteger("10");
BigInteger power = value1.pow(3);
System.out.println(power);

çıktı: 1000

Kalan: %10 6 = 4

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("6");

BigInteger power = value1.remainder(value2);
System.out.println(power);

çıkış: 4

GCD: “12” ve “18” için En Büyük Ortak Bölen (GCD) “6"dır.

BigInteger value1 = new BigInteger("12");
BigInteger value2 = new BigInteger("18");

System.out.println(value1.gcd(value2));

Çıkış: 6

Çıkış: 11

BigInteger value1 = new BigInteger("10");

BüyükTamsayıları Karşılaştırma #

Java’daki “String” veya diğer nesneleri karşılaştırdığınız gibi “BigIntegers"ı da karşılaştırabilirsiniz.

Örneğin:

BigInteger one = BigInteger.valueOf(1);
BigInteger two = BigInteger.valueOf(2);

if(one.equals(two)){
    System.out.println("Equal");
}
else{
    System.out.println("Not Equal");
}

Çıktı:

Not Equal

Not:

Genel olarak, BigInteger’ları karşılaştırmak için == operatörünü kullanmayın**

  • == operatörü: referansları karşılaştırır; yani, iki değerin aynı nesneye atıfta bulunup bulunmadığı
  • equals() yöntemi: iki BigInteger’ın içeriğini karşılaştırır.

Örneğin, BigIntegers aşağıdaki şekilde karşılaştırılmamalıdır:

if (firstBigInteger == secondBigInteger) {
  // Only checks for reference equality, not content equality!
}

== operatörü yalnızca referans eşitliğini kontrol ettiğinden, bunu yapmak beklenmeyen davranışlara yol açabilir. Her iki BigInteger aynı içeriği içeriyor ancak aynı nesneye atıfta bulunmuyorsa, bu başarısız olur. Bunun yerine, BigInteger’ları yukarıda açıklandığı gibi “eşittir” yöntemlerini kullanarak karşılaştırın.

BigIntegerinizi 0,1,10 gibi sabit değerlerle de karşılaştırabilirsiniz.

örneğin:

BigInteger reallyBig = BigInteger.valueOf(1);
if(BigInteger.ONE.equals(reallyBig)){
    //code when they are equal.
}

Ayrıca, aşağıdaki gibi “compareTo()” yöntemini kullanarak iki BigInteger’ı karşılaştırabilirsiniz: compareTo() 3 değer döndürür.

  • 0: Her ikisi de eşit olduğunda.

  • 1: Birinci, ikinciden büyük olduğunda (parantez içindeki).

  • -1: Birinci ** saniyeden** az olduğunda.

    BigInteger reallyBig = BigInteger.valueOf(10); BigInteger reallyBig1 = BigInteger.valueOf(100);

    if(reallyBig.compareTo(reallyBig1) == 0){ //code when both are equal. } else if(reallyBig.compareTo(reallyBig1) == 1){ //code when reallyBig is greater than reallyBig1. } else if(reallyBig.compareTo(reallyBig1) == -1){ //code when reallyBig is less than reallyBig1. }

BigInteger’da İkili Mantık İşlemleri #

BigInteger, “Number” türlerinde de kullanılabilen ikili mantık işlemlerini destekler. Tüm işlemlerde olduğu gibi, bir yöntem çağrılarak uygulanırlar.

İkili Veya:

BigInteger val1 = new BigInteger("10");
BigInteger val2 = new BigInteger("9");

val1.or(val2);

Çıktı: 11 (10 | 9a eşdeğerdir)

İkili Ve:

BigInteger val1 = new BigInteger("10");
BigInteger val2 = new BigInteger("9");

val1.and(val2);

Çıktı: 8 (“10 ve 9"a eşdeğerdir)

İkili Xor:

BigInteger val1 = new BigInteger("10");
BigInteger val2 = new BigInteger("9");

val1.xor(val2);

Çıktı: 3 (“10 ^ 9"a eşdeğerdir)

Sağa kaydırma:

BigInteger val1 = new BigInteger("10");

val1.shiftRight(1);   // the argument be an Integer

Çıktı: 5 (“10 » 1"e eşdeğer)

Sol shift:

BigInteger val1 = new BigInteger("10");

val1.shiftLeft(1);   // here parameter should be Integer

Çıktı: 20 (‘10 « 1’e eşdeğer)

İkili Ters Çevirme (Değil) :

BigInteger val1 = new BigInteger("10");

val1.not();

Çıkış: 5

NAND (Ve-Değil):*

BigInteger val1 = new BigInteger("10");
BigInteger val2 = new BigInteger("9");

val1.andNot(val2);

Çıkış: 7

Rastgele BigIntegers oluşturma #

“BigInteger” sınıfı, bir “java.util.Random” örneği ve “BigInteger"ın kaç bit olacağını belirten bir “int” verildiğinde, rastgele “BigIntegers” oluşturmaya adanmış bir kurucuya sahiptir. Kullanımı oldukça basittir - yapıcıyı çağırdığınızda [BigInteger(int, Random)](https://docs.Oracle.com/javase/8/docs/api/java/math/BigInteger.html#BigInteger- int-java.util.Random-) şöyle:

BigInteger randomBigInt = new BigInteger(bitCount, sourceOfRandomness);

sonra değeri 0 (dahil) ile 2“bitCount” (hariç) arasında olan bir “BigInteger” elde edersiniz.

Bu aynı zamanda “yeni BigInteger(2147483647, sourceOfRandomness)“ın yeterli süre verildiğinde tüm pozitif “BigInteger"leri döndürebileceği anlamına gelir.

‘SourceOfRandomness’ın ne olacağı size kalmış. Örneğin, bir “new Random()” çoğu durumda yeterince iyidir:

new BigInteger(32, new Random());

Daha yüksek kaliteli rastgele sayılar için hızdan vazgeçmek istiyorsanız, yeni bir SecureRandom kullanabilirsiniz. Bunun yerine SecureRandom.html)():

import java.security.SecureRandom;

// somewhere in the code...
new BigInteger(32, new SecureRandom());

Anonim bir sınıfla anında bir algoritma bile uygulayabilirsiniz! Kendi RNG algoritmanızı oluşturmanın sizi düşük kaliteli rastgelelikle sonuçlayacağını unutmayın; bu nedenle, elde edilen “BigInteger”(ler)in tahmin edilebilir olmasını istemiyorsanız, her zaman iyi olduğu kanıtlanmış bir algoritma kullandığınızdan emin olun.

new BigInteger(32, new Random() {
    int seed = 0;

    @Override
    protected int next(int bits) {
        seed = ((22695477 * seed) + 1) & 2147483647; // Values shamelessly stolen from Wikipedia
        return seed;
    }
});
