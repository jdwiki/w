Çoklu Yayın JAR Dosyaları

  • Çok sürümlü Jar dosyasının içeriği örneği
  • Jar aracını kullanarak çok çıkışlı bir Jar oluşturma
  • Çok sürümlü bir Jar içindeki yüklü bir sınıfın URL’si

Bu sayfada

  • Çok sürümlü Jar dosyasının içeriği örneği
  • Jar aracını kullanarak çok çıkışlı bir Jar oluşturma
  • Çok sürümlü bir Jar içindeki yüklü bir sınıfın URL’si

Java 9’da tanıtılan özelliklerden biri, aynı Jar dosyası içinde birden çok Java sürümünü hedefleyen kodların paketlenmesine izin veren çok sürümlü Jar’dır (MRJAR). Bu özellik JEP 238 içinde belirtilmiştir.

Çok sürümlü Jar dosyasının içeriği örneği #

MANIFEST.MF dosyasında Multi-Release: true ayarını yaparak , Jar dosyası çok sürümlü bir Jar haline gelir ve Java çalışma zamanı (MRJAR biçimini desteklediği sürece) mevcut ana sürüme bağlı olarak uygun sınıf sürümlerini seçer.

Böyle bir Kavanozun yapısı aşağıdaki gibidir:

jar root
  - A.class
  - B.class
  - C.class
  - D.class
  - META-INF
     - versions
        - 9
           - A.class
           - B.class
        - 10
           - A.class
  • JDK’lar < 9’da, Java çalışma zamanında yalnızca kök girişteki sınıflar görünür.
  • Bir JDK 9’da, A ve B sınıfları `root/META-INF/versions/9’ dizininden yüklenecek, C ve D ise temel girişten yüklenecektir.
  • Bir JDK 10’da, A sınıfı “root/META-INF/versions/10” dizininden yüklenir.

Jar aracını kullanarak çok çıkışlı bir Jar oluşturma #

Jar komutu, aynı sınıfın hem Java 8 hem de Java 9 için derlenmiş iki sürümünü içeren çok sürümlü bir Jar oluşturmak için kullanılabilir, ancak sınıfların aynı olduğunu söyleyen bir uyarı ile birlikte:

C:\Users\manouti>jar --create --file MR.jar -C sampleproject-base demo --release 9 -C sampleproject-9 demo
Warning: entry META-INF/versions/9/demo/SampleClass.class contains a class that
is identical to an entry already in the jar

“–release 9” seçeneği, “jar"a, takip eden her şeyi (“sampleproject-9” dizini içindeki “demo” paketi) MRJAR’da, yani “root/META-INF/versions” altındaki sürümlü bir girişin içine dahil etmesini söyler. /9`. Sonuç aşağıdaki içeriklerdir:

jar root
  - demo
     - SampleClass.class
  - META-INF
     - versions
        - 9
           - demo
              - SampleClass.class

Şimdi SampleClassın URL’sini yazdıran Main adında bir sınıf oluşturalım ve onu Java 9 sürümü için ekleyelim:

package demo;

import java.net.URL;

public class Main {

    public static void main(String[] args) throws Exception {
        URL url = Main.class.getClassLoader().getResource("demo/SampleClass.class");
        System.out.println(url);
    }
}

Bu sınıfı derlersek ve jar komutunu yeniden çalıştırırsak bir hata alırız:

C:\Users\manouti>jar --create --file MR.jar -C sampleproject-base demo --release 9 -C sampleproject-9 demoentry: META-INF/versions/9/demo/Main.class, contains a new public class not found in base entries
Warning: entry META-INF/versions/9/demo/Main.java, multiple resources with same name
Warning: entry META-INF/versions/9/demo/SampleClass.class contains a class that
is identical to an entry already in the jar
invalid multi-release jar file MR.jar deleted

Bunun nedeni, ‘jar’ aracının, temel girdilere de eklenmemişlerse, sürümlü girdilere genel sınıflar eklenmesini engellemesidir. Bu, MRJAR’ın farklı Java sürümleri için aynı genel API’yi göstermesi için yapılır. Çalışma zamanında bu kuralın gerekli olmadığını unutmayın. Sadece ‘jar’ gibi araçlarla uygulanabilir. Bu özel durumda, ‘Ana’nın amacı örnek kodu çalıştırmaktır, böylece temel girişe bir kopyasını kolayca ekleyebiliriz. Sınıf, yalnızca Java 9 için ihtiyaç duyduğumuz daha yeni bir uygulamanın parçasıysa, halka açık olmayabilir.

Kök girişe ‘Main’ eklemek için, önce onu Java 9 öncesi bir sürümü hedefleyecek şekilde derlememiz gerekir. Bu, “javac"ın yeni “–release” seçeneği kullanılarak yapılabilir:

C:\Users\manouti\sampleproject-base\demo>javac --release 8 Main.java
C:\Users\manouti\sampleproject-base\demo>cd ../..
C:\Users\manouti>jar --create --file MR.jar -C sampleproject-base demo --release 9 -C sampleproject-9 demo

Main sınıfının çalıştırılması, SampleClass’ın sürümlü dizinden yüklendiğini gösterir:

C:\Users\manouti>java --class-path MR.jar demo.Main
jar:file:/C:/Users/manouti/MR.jar!/META-INF/versions/9/demo/SampleClass.class

Çok sürümlü bir Jar içindeki yüklü bir sınıfın URL’si #

Aşağıdaki çoklu yayın Kavanozu verildiğinde:

jar root
  - demo
     - SampleClass.class
  - META-INF
     - versions
        - 9
           - demo
              - SampleClass.class

Aşağıdaki sınıf, “SampleClass"ın URL’sini yazdırır:

package demo;

import java.net.URL;

public class Main {

    public static void main(String[] args) throws Exception {
        URL url = Main.class.getClassLoader().getResource("demo/SampleClass.class");
        System.out.println(url);
    }
}

Sınıf derlenir ve MRJAR’daki Java 9 için sürümlü girişe eklenirse, çalıştırılması şu sonuçlarla sonuçlanır:

C:\Users\manouti>java --class-path MR.jar demo.Main
jar:file:/C:/Users/manouti/MR.jar!/META-INF/versions/9/demo/SampleClass.class
