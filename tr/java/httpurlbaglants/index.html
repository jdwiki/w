HttpURLbağlantısı

  • Bir URL’den Dize olarak yanıt gövdesi alın
  • POST verileri

  • Kaynak olup olmadığını kontrol edin

Bu sayfada

  • Bir URL’den Dize olarak yanıt gövdesi alın
  • POST verileri
  • Nasıl çalışır
  • Kaynağı sil
  • Kaynak olup olmadığını kontrol edin

  • Android’de HttpUrlConnection kullanmak, uygulamanıza İnternet izni eklemenizi gerektirir (AndroidManifest.xmlde).

  • Ayrıca kullanımı daha kolay olan ve daha iyi performans veya daha fazla özellik sunabilen Square’in OkHttp gibi başka Java HTTP istemcileri ve kitaplıkları da vardır.

Bir URL’den Dize olarak yanıt gövdesi alın #

String getText(String url) throws IOException {
    HttpURLConnection connection = (HttpURLConnection) new URL(url).openConnection();
    //add headers to the connection, or check the status if desired..
    
    // handle error response code it occurs
    int responseCode = conn.getResponseCode();
    InputStream inputStream;
    if (200 <= responseCode && responseCode <= 299) {
        inputStream = connection.getInputStream();
    } else {
        inputStream = connection.getErrorStream();
    }

    BufferedReader in = new BufferedReader(
        new InputStreamReader(
            inputStream));

    StringBuilder response = new StringBuilder();
    String currentLine;

    while ((currentLine = in.readLine()) != null) 
        response.append(currentLine);

    in.close();

    return response.toString();
}

Bu, belirtilen URL’den metin verilerini indirecek ve bir Dize olarak döndürecektir.

Bu nasıl çalışır:

  • İlk olarak, URL’mizden new URL(url).openConnection() ile bir HttpUrlConnection oluşturuyoruz. ‘UrlConnection’ı yayınlıyoruz, bu bir ‘HttpUrlConnection’a dönüyor, böylece başlık ekleme (Kullanıcı Aracısı gibi) veya yanıt kodunu kontrol etme gibi şeylere erişebiliyoruz. (Bu örnek bunu yapmaz, ancak eklenmesi kolaydır.)

  • Ardından, yanıt kodunu temel alarak InputStream oluşturun (hata işleme için)

  • Ardından, bağlantıdan aldığımız ‘InputStream’den metin okumamızı sağlayan bir ‘BufferedReader’ oluşturun.

  • Şimdi metni bir StringBuildera satır satır ekliyoruz.

  • InputStreami kapatın ve şimdi sahip olduğumuz String’i döndürün.

Notlar:

  • Bu yöntem, hata durumunda (ağ hatası veya internet bağlantısı olmaması gibi) bir ‘IoException’ verir ve ayrıca verilen URL geçerli değilse bir unchecked ‘MalformedUrlException’ verir.

  • Web sayfaları (HTML), JSON veya XML döndüren REST API’leri gibi metin döndüren herhangi bir URL’den okumak için kullanılabilir.

  • Ayrıca bakınız: Birkaç satır Java kodunda URL’yi Dize’ye okuyun.

public static void post(String url, byte [] data, String contentType) throws IOException {
    HttpURLConnection connection = null;
    OutputStream out = null;
    InputStream in = null;

    try {
        connection = (HttpURLConnection) new URL(url).openConnection();
        connection.setRequestProperty("Content-Type", contentType);
        connection.setDoOutput(true);

        out = connection.getOutputStream();
        out.write(data);
        out.close();

        in = connection.getInputStream();
        BufferedReader reader = new BufferedReader(new InputStreamReader(in));
        String line = null;
        while ((line = reader.readLine()) != null) {
            System.out.println(line);
        }
        in.close();

    } finally {
        if (connection != null) connection.disconnect();
        if (out != null) out.close();
        if (in != null) in.close();
    }
}

Bu, verileri belirtilen URL’ye POST yapacak ve ardından yanıtı satır satır okuyacaktır.

Kaynak olup olmadığını kontrol edin #

/**
 * Checks if a resource exists by sending a HEAD-Request.
 * @param url The url of a resource which has to be checked.
 * @return true if the response code is 200 OK.
 */
public static final boolean checkIfResourceExists(URL url) throws IOException {
    HttpURLConnection conn = (HttpURLConnection) url.openConnection();
    conn.setRequestMethod("HEAD");
    int code = conn.getResponseCode();
    conn.disconnect();
    return code == 200;
}

Açıklama: #

Yalnızca bir kaynağın olup olmadığını kontrol ediyorsanız, GET yerine HEAD isteği kullanmak daha iyidir. Bu, kaynağı aktarmanın ek yükünü önler.

Yanıt kodu “200” ise yöntemin yalnızca “true” değerini döndürdüğünü unutmayın. Yeniden yönlendirme (yani 3XX) yanıtları bekliyorsanız, bu yanıtları onurlandırmak için yöntemin geliştirilmesi gerekebilir.

Örnek: #

checkIfResourceExists(new URL("http://images.google.com/")); // true
checkIfResourceExists(new URL("http://pictures.google.com/")); // false
