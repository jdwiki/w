iş projesi ‘Statik olmayan’ ‘yerel’ yöntemi çağıran nesne
jsınıfı ‘Statik’ ‘yerel’ yöntemi çağıran sınıf

JNI’yi kurmak, hem bir Java hem de yerel bir derleyici gerektirir. IDE ve işletim sistemine bağlı olarak, gerekli bazı ayarlar vardır. Eclipse için bir kılavuz burada bulunabilir. Tam bir eğitici burada bulunabilir.

Windows’ta Java-C++ bağlantısını kurma adımları şunlardır:

  • Java kaynak dosyalarını (.java) javac kullanarak sınıflar (.class) halinde derleyin.
  • ‘javah’ kullanarak ‘yerel’ yöntemler içeren Java sınıflarından başlık (’.h’) dosyaları oluşturun. Bu dosyalar, yerel koda hangi yöntemlerin uygulanmasından sorumlu olduğunu “öğretir”.
  • Üstbilgi dosyalarını (#include), yerel yöntemleri uygulayan C++ kaynak dosyalarına (.cpp) dahil edin.
  • C++ kaynak dosyalarını derleyin ve bir kitaplık (.dll) oluşturun. Bu kitaplık, yerel kod uygulamasını içerir.
  • Kitaplık yolunu (-Djava.library.path) belirtin ve bunu Java kaynak dosyasına (System.loadLibrary(...)) yükleyin.

Geri aramalar (yerel koddan Java yöntemlerini çağırmak) bir yöntem tanımlayıcı belirtmeyi gerektirir. Tanımlayıcı yanlışsa, bir çalışma zamanı hatası oluşur. Bu nedenle tanımlayıcıların bizim için yapılmış olmasında fayda var, bu javap -s ile yapılabilir.

Java’dan C++ yöntemlerini çağırma #

Java’daki statik ve üye yöntemler, uygulamalarının paylaşılan bir kitaplık dosyasında bulunacağını belirtmek için yerel olarak işaretlenebilir. Yerel bir yöntemin yürütülmesi üzerine, JVM, basit bir ad yönetimi şeması kullanarak, yüklenen kitaplıklarda karşılık gelen bir işlevi arar (bkz. .

Java kodu #

/*** com/example/jni/JNIJava.java **/

package com.example.jni;

public class JNIJava {
    static {
        System.loadLibrary("libJNI_CPP");
    }

    // Obviously, native methods may not have a body defined in Java
    public native void printString(String name);
    public static native double average(int[] nums);

    public static void main(final String[] args) {
        JNIJava jniJava = new JNIJava();
        jniJava.printString("Invoked C++ 'printString' from Java");

        double d = average(new int[]{1, 2, 3, 4, 7});
        System.out.println("Got result from C++ 'average': " + d);
    }
}

C++ kodu #

Yerel işlev bildirimlerini içeren başlık dosyaları, hedef sınıflarda ‘javah’ aracı kullanılarak oluşturulmalıdır. Derleme dizininde aşağıdaki komutu çalıştırın:

javah -o com_example_jni_JNIJava.hpp com.example.jni.JNIJava

… aşağıdaki başlık dosyasını üretir (kısa olması için yorumlardan arındırılmıştır):

// com_example_jni_JNIJava.hpp

/* DO NOT EDIT THIS FILE - it is machine generated */
#include <jni.h> // The JNI API declarations

#ifndef _Included_com_example_jni_JNIJava
#define _Included_com_example_jni_JNIJava
#ifdef __cplusplus
extern "C" { // This is absolutely required if using a C++ compiler
#endif

JNIEXPORT void JNICALL Java_com_example_jni_JNIJava_printString
  (JNIEnv *, jobject, jstring);

JNIEXPORT jdouble JNICALL Java_com_example_jni_JNIJava_average
  (JNIEnv *, jclass, jintArray);

#ifdef __cplusplus
}
#endif
#endif

İşte örnek bir uygulama:

// com_example_jni_JNIJava.cpp

#include <iostream>
#include "com_example_jni_JNIJava.hpp"

using namespace std;

JNIEXPORT void JNICALL Java_com_example_jni_JNIJava_printString(JNIEnv *env, jobject jthis, jstring string) {
    const char *stringInC = env->GetStringUTFChars(string, NULL);
    if (NULL == stringInC)
        return;
    cout << stringInC << endl;
    env->ReleaseStringUTFChars(string, stringInC);
}

JNIEXPORT jdouble JNICALL Java_com_example_jni_JNIJava_average(JNIEnv *env, jclass jthis, jintArray intArray) {
    jint *intArrayInC = env->GetIntArrayElements(intArray, NULL);
    if (NULL == intArrayInC)
        return -1;
    jsize length = env->GetArrayLength(intArray);
    int sum = 0;
    for (int i = 0; i < length; i++) {
        sum += intArrayInC[i];
    }
    env->ReleaseIntArrayElements(intArray, intArrayInC, 0);
    return (double) sum / length;
}

Çıktı #

Yukarıdaki örnek sınıfı çalıştırmak aşağıdaki çıktıyı verir:

Java’dan çağrılan C++ ‘printString’ C++ ‘ortalamasından’ sonuç alındı: 3.4

1: https://www.wikiod.com/tr/java/java-yerel-arayuzu#Yerel kitaplıklar yükleniyor

Java yöntemlerini C++‘dan çağırma (geri arama) #

Yerel koddan bir Java yöntemini çağırmak iki adımlı bir işlemdir:

  1. yöntem adını ve tanımlayıcısını kullanarak ‘GetMethodID’ JNI işleviyle bir yöntem işaretçisi elde edin;
  2. burada listelenen ‘Çağrı*Yöntemi’ işlevlerinden birini çağırın.

Java kodu #

/*** com.example.jni.JNIJavaCallback.java ***/

package com.example.jni;

public class JNIJavaCallback {
    static {
        System.loadLibrary("libJNI_CPP");
    }
    
    public static void main(String[] args) {
        new JNIJavaCallback().callback();
    }

    public native void callback(); 

    public static void printNum(int i) {
        System.out.println("Got int from C++: " + i);
    }
    
    public void printFloat(float i) {
        System.out.println("Got float from C++: " + i);
    }
}

C++ kodu #

// com_example_jni_JNICppCallback.cpp

#include <iostream>
#include "com_example_jni_JNIJavaCallback.h"

using namespace std;

JNIEXPORT void JNICALL Java_com_example_jni_JNIJavaCallback_callback(JNIEnv *env, jobject jthis) {
    jclass thisClass = env->GetObjectClass(jthis);

    jmethodID printFloat = env->GetMethodID(thisClass, "printFloat", "(F)V");
    if (NULL == printFloat)
        return;
    env->CallVoidMethod(jthis, printFloat, 5.221);

    jmethodID staticPrintInt = env->GetStaticMethodID(thisClass, "printNum", "(I)V");
    if (NULL == staticPrintInt)
        return;
    env->CallVoidMethod(jthis, staticPrintInt, 17);
}

Çıktı #

C++‘dan float var: 5.221 C++‘dan int alındı: 17

Tanımlayıcıyı alma #

Tanımlayıcılar (veya dahili tip imzalar), derlenmiş .class dosyasındaki javap programı kullanılarak elde edilir. İşte “javap -p -s com.example.jni.JNIJavaCallback” çıktısı:

Compiled from "JNIJavaCallback.java"
public class com.example.jni.JNIJavaCallback {
  static {};
    descriptor: ()V

  public com.example.jni.JNIJavaCallback();
    descriptor: ()V

  public static void main(java.lang.String[]);
    descriptor: ([Ljava/lang/String;)V

  public native void callback();
    descriptor: ()V

    descriptor: (I)V  // <---- Needed

Yerel kitaplıkları yükleme #

Java’da paylaşılan kitaplık dosyalarını yüklemek için yaygın deyim şudur:

public class ClassWithNativeMethods {
    static {
        System.loadLibrary("Example");
    }

    public native void someNativeMethod(String arg);
    ...

System.loadLibrary öğesine yapılan çağrılar, sınıf yükleme sırasında meydana gelecek şekilde neredeyse her zaman statiktir ve paylaşılan kitaplık yüklenmeden önce hiçbir yerel yöntemin yürütülmemesini sağlar. Ancak şunlar mümkündür:

public class ClassWithNativeMethods {
    // Call this before using any native method
    public static void prepareNativeMethods() {
        System.loadLibrary("Example");
    }

    ...

Bu, gerekli olana kadar paylaşılan kitaplık yüklemesini ertelemeye izin verir, ancak java.lang.UnsatisfiedLinkErrorlardan kaçınmak için ekstra özen gerektirir.

Hedef dosya arama #

Paylaşılan kitaplık dosyaları, çalışma zamanında -Djava.library.path= JVM argümanı kullanılarak geçersiz kılınabilen java.library.path sistem özelliği tarafından tanımlanan yollarda aranır:

java -Djava.library.path=path/to/lib/:path/to/other/lib MainClassWithNativeMethods

Sistem yolu ayırıcılarına dikkat edin: örneğin, Windows : yerine ; kullanır.

“System.loadLibrary"nin kitaplık dosya adlarını platforma bağlı bir şekilde çözdüğünü unutmayın: yukarıdaki kod parçacığı, Linux’ta “libExample.so” ve Windows’ta “Example.dll” adlı bir dosya bekler.

“System.loadLibrary"ye bir alternatif, “java.library.path” aramasını atlayarak paylaşılan bir kitaplık dosyasının tam yolunu alan “System.load(String)”‘dir:

public class ClassWithNativeMethods {
    static {
        System.load("/path/to/lib/libExample.so");
    }

    ...
