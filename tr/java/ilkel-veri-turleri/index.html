İlkel Veri Türleri

  • Sözdizimi
  • Karakter ilkel

Bu sayfada

  • Sözdizimi
  • Karakter ilkel

8 temel veri türü “byte”, “short”, “int”, “long”, “char”, “boolean”, “float” ve “double” Java programlarında en çok ham sayısal veriyi depolayan türlerdir.

Sözdizimi #

  • int aInt = 8; // Bu int bildiriminin tanımlayıcı (sayı) kısmına değişmez değer denir.

  • int hexInt = 0x1a; // = 26; 0x ön ekine sahip onaltılık değerlere sahip değişmezler tanımlayabilirsiniz.

  • int binInt = 0b11010; // = 26; İkili değişmezleri de tanımlayabilirsiniz; 0b ile başlar.

  • uzun iyiUzun = 10000000000L; // Varsayılan olarak, tamsayı değişmezleri int türündedir. Değişmezin sonuna L ekleyerek derleyiciye değişmezin uzun olduğunu söylüyorsunuz. Bu olmadan derleyici bir “Tamsayı çok büyük” hatası verir.

  • çift aDouble = 3.14; // Kayan Nokta Değişmezleri varsayılan olarak double türündedir.

  • yüzer aFloat = 3.14F; // Varsayılan olarak bu değişmez değer bir double olurdu (ve “Uyumsuz Türler” hatasına neden olur), ancak bir F ekleyerek derleyiciye bunun bir kayan nokta olduğunu söyleriz.

Java, “boolean”, “byte”, “short”, “char”, “int”, “long”, “float” ve “double” olmak üzere 8 ilkel veri türüne sahiptir. (Diğer tüm türler referans türleridir. Bu, tüm dizi türlerini ve Java dilinde özel önemi olan yerleşik nesne türlerini/sınıflarını içerir; örneğin, “Dize”, “Sınıf” ve “Atılabilir” ve alt sınıfları.)

İlkel bir türdeki tüm işlemlerin (toplama, çıkarma, çarpma vb.) sonucu en az bir “int"tir, bu nedenle “kısa"ya “kısa” eklemek, “bayt” eklemek gibi bir “int” üretir. “bayt"a veya “karakter"den “karakter"e. Bunun sonucunu aynı türden bir değere geri atamak istiyorsanız, onu yayınlamalısınız. Örneğin.

byte a = 1;
byte b = 2;
byte c = (byte) (a + b);

İşlemi yayınlamamak derleme hatasına neden olur.

Bunun nedeni, [Java Language Spec, §2.11.1](http://docs.Oracle.com/javase/specs/jvms/se7/html/jvms-2.html#jvms-2.11. 1):

[…]. Bu nedenle, “boolean”, “byte”, “char” ve “short” gerçek türlerinin değerleri üzerindeki çoğu işlem, “int” hesaplama türündeki değerler üzerinde çalışan talimatlar tarafından doğru bir şekilde gerçekleştirilir.

Bunun arkasındaki neden de o bölümde belirtilmiştir:

Java Sanal Makinesi’nin bir baytlık işlem kodu boyutu göz önüne alındığında, türleri işlem kodlarına kodlamak, komut kümesinin tasarımı üzerinde baskı oluşturur. Yazılan her komut, Java Sanal Makinesi’nin tüm çalışma zamanı veri türlerini destekliyorsa, bir “bayt"ta gösterilebilecekten daha fazla talimat olacaktır. […] Desteklenmeyen ve desteklenen veri türleri arasında gerektiğinde dönüştürmek için ayrı talimatlar kullanılabilir.

## Şamandıra ilkel
Bir "kayan nokta", tek duyarlıklı bir 32-bit IEEE 754 kayan noktalı sayıdır. Varsayılan olarak, ondalık sayılar çiftler olarak yorumlanır. Bir "kayan nokta" oluşturmak için, ondalık değere bir "f" eklemeniz yeterlidir.

    double doubleExample = 0.5;      // without 'f' after digits = double
    float floatExample = 0.5f;       // with 'f' after digits    = float

    float myFloat = 92.7f;           // this is a float...
    float positiveFloat = 89.3f;     // it can be positive,
    float negativeFloat = -89.3f;    // or negative
    float integerFloat = 43.0f;      // it can be a whole number (not an int)
    float underZeroFloat = 0.0549f;  // it can be a fractional value less than 0

Şamandıralar beş yaygın aritmetik işlemi gerçekleştirir: toplama, çıkarma, çarpma, bölme ve modül.

*Not: Aşağıdakiler, kayan nokta hatalarının bir sonucu olarak biraz değişebilir. Bazı sonuçlar netlik ve okunabilirlik amacıyla yuvarlanmıştır (yani, ek örneğinin yazdırılan sonucu aslında 34.600002 idi).*
    
    // addition
    float result = 37.2f + -2.6f;  // result: 34.6
    
    // subtraction
    float result = 45.1f - 10.3f;    // result: 34.8

    // multiplication
    float result = 26.3f * 1.7f;   // result: 44.71

    // division
    float result = 37.1f / 4.8f;   // result: 7.729166

    // modulus
    float result = 37.1f % 4.8f;   // result: 3.4999971

Kayan noktalı sayıların saklanma şekli nedeniyle (yani ikili biçimde), birçok sayının tam bir temsili yoktur.

    float notExact = 3.1415926f;
    System.out.println(notExact); // 3.1415925

Çoğu uygulama için "kayan" kullanmak iyi olsa da, ondalık sayıların (parasal tutarlar gibi) tam temsillerini veya daha yüksek kesinliğin gerekli olduğu sayıları saklamak için ne "kayan" ne de "çift" kullanılmamalıdır. Bunun yerine `BigDecimal` sınıfı kullanılmalıdır.

Bir "float"ın varsayılan değeri _0.0f_'dir.
    
    float defaultFloat;    // defaultFloat == 0.0f

Bir "yüzdürme" kabaca 10 milyonda 1 hataya eşittir.

**Not:** "Float.POSITIVE_INFINITY", "Float.NEGATIVE_INFINITY", "Float.NaN", "kayan" değerlerdir. "NaN", 2 sonsuz değeri bölmek gibi belirlenemeyen işlemlerin sonuçlarını ifade eder. Ayrıca '0f' ve '-0f' farklıdır, ancak '==' doğru sonuç verir:

    float f1 = 0f;
    float f2 = -0f;
    System.out.println(f1 == f2); // true
    System.out.println(1f / f1); // Infinity
    System.out.println(1f / f2); // -Infinity
    System.out.println(Float.POSITIVE_INFINITY / Float.POSITIVE_INFINITY); // NaN

## int ilkel
'int' gibi ilkel bir veri türü, değerleri doğrudan onu kullanan değişkende tutar, bu arada 'Tamsayı' kullanılarak bildirilen bir değişken, değere bir referans tutar.

[Java API][1]'ye göre: "Tamsayı sınıfı, bir nesnede ilkel int türünün bir değerini sarar. Tamsayı türünde bir nesne, türü int olan tek bir alan içerir."

Varsayılan olarak, "int" 32 bitlik işaretli bir tamsayıdır. Minimum -2<sup>31</sup> değerini ve maksimum 2<sup>31</sup> - 1 değerini saklayabilir.

    int example = -42;
    int myInt = 284;
    int anotherInt = 73;
    
    int addedInts = myInt + anotherInt; // 284 + 73 = 357
    int subtractedInts = myInt - anotherInt; // 284 - 73 = 211

Bu aralığın dışında bir sayı kaydetmeniz gerekiyorsa, bunun yerine "uzun" kullanılmalıdır. 'int' değer aralığının aşılması, bir tamsayı taşmasına yol açar ve aralığı aşan değerin aralığın karşı tarafına eklenmesine neden olur (pozitif, negatif olur ve tam tersi). Değer `((değer - MIN_VALUE) % ARALIK) + MIN_VALUE` veya `((değer + 2147483648) % 4294967296) - 2147483648`

    int demo = 2147483647; //maximum positive integer
    System.out.println(demo); //prints 2147483647
    demo = demo + 1; //leads to an integer overflow
    System.out.println(demo); // prints -2147483648

`int` maksimum ve minimum değerleri şurada bulunabilir:

    int high = Integer.MAX_VALUE;    // high == 2147483647
    int low = Integer.MIN_VALUE;     // low == -2147483648

Bir "int"in varsayılan değeri _0_
    
    int defaultInt;    // defaultInt == 0


[1]: https://docs.oracle.com/javase/7/docs/api/

## İlkellerin bellek tüketimi ve kutulu ilkellerin bellek tüketimi
| İlkel | Kutulu Tip | İlkel / kutulu bellek boyutu |
| --------- | ----------- | ---------- |
| boole | Boole | 1 bayt / 16 bayt |
| bayt | bayt | 1 bayt / 16 bayt |
| kısa | Kısa | 2 bayt / 16 bayt |
| karakter | Karakter | 2 bayt / 16 bayt |
| int | Tamsayı | 4 bayt / 16 bayt |
| uzun | Uzun | 8 bayt / 16 bayt |
| yüzer | Şamandıra | 4 bayt / 16 bayt |
| çift ​​| Çift Kişilik | 8 bayt / 16 bayt |

Kutulu nesneler, tür ve bellek yönetimi için her zaman 8 bayt gerektirir ve nesnelerin boyutu her zaman 8'in katı olduğundan, kutulu türlerin *hepsi toplam 16 bayt gerektirir*. *Ek* olarak, kutulu bir nesnenin her kullanımı, JVM ve JVM seçeneklerine bağlı olarak 4 veya 8 baytlık bir referansın depolanmasını gerektirir.

Veri yoğun işlemlerde, bellek tüketimi performans üzerinde büyük bir etkiye sahip olabilir. Diziler kullanıldığında bellek tüketimi daha da artar: bir "kayan[5]" dizisi yalnızca 32 bayt gerektirir; 5 farklı boş olmayan değeri saklayan bir "Float[5]" için toplam 112 bayt gerekir (sıkıştırılmış işaretçiler olmadan 64 bitte bu 152 bayta çıkar).

Kutulu değer önbellekleri
------------------

Kutulu tiplerin alan genel giderleri kutulu değer önbellekleri tarafından bir dereceye kadar azaltılabilir. Kutulu türlerden bazıları, bir örnek önbelleği uygular. Örneğin, varsayılan olarak "Integer" sınıfı, "-128" ile "+127" aralığındaki sayıları temsil etmek için örnekleri önbelleğe alır. Ancak bu, ek bellek dolaylı aktarımından kaynaklanan ek maliyeti azaltmaz.

Otomatik kutulama yoluyla veya statik "valueOf(primitive)" yöntemini çağırarak kutulu türde bir örnek oluşturursanız, çalışma zamanı sistemi önbelleğe alınmış bir değer kullanmayı dener. Uygulamanız önbelleğe alınan aralıkta çok sayıda değer kullanıyorsa, bu, kutulu türleri kullanmanın bellek cezasını önemli ölçüde azaltabilir. Elbette, "elle" kutulu değer örnekleri oluşturuyorsanız, "yeni" yerine "değerOf" kullanmak daha iyidir. ("yeni" işlemi her zaman yeni bir örnek oluşturur.) Bununla birlikte, değerlerinizin çoğu önbelleğe alınmış aralıkta *değilse*, "yeni"yi çağırmak ve önbellek aramasını kaydetmek daha hızlı olabilir.

## İlkelleri Dönüştürme
Java'da tamsayı değerleri ve kayan nokta değerleri arasında dönüşüm yapabiliriz. Ayrıca, Unicode kodlamasında her karakter bir sayıya karşılık geldiğinden, "char" türleri tamsayı ve kayan nokta türlerine dönüştürülebilir. "boolean", başka herhangi bir ilkel veri türüne dönüştürülemeyen veya bunlardan dönüştürülemeyen tek ilkel veri türüdür.

İki tür dönüşüm vardır: *dönüşüm genişletme* ve *dönüşüm daraltma*.

*Genişleyen dönüştürme*, bir veri türünün değerinin öncekinden daha fazla bit kaplayan başka bir veri türünün değerine dönüştürülmesidir. Bu durumda veri kaybı sorunu yoktur.
 
Buna uygun olarak, *daraltma dönüştürme*, bir veri türünün değerinin öncekinden daha az bit kaplayan başka bir veri türünün değerine dönüştürülmesidir. Bu durumda veri kaybı meydana gelebilir.

Java *genişletme dönüşümlerini* otomatik olarak gerçekleştirir. Ancak bir *daraltma dönüştürme* yapmak istiyorsanız (veri kaybı olmayacağından eminseniz), Java'yı "cast" olarak bilinen bir dil yapısını kullanarak dönüştürmeyi gerçekleştirmeye zorlayabilirsiniz.

**Genişletme Dönüşümü:**

    int a = 1;    
    double d = a;    // valid conversion to double, no cast needed (widening)

**Dönüştürme Daraltma:**

    double d = 18.96
    int b = d;       // invalid conversion to int, will throw a compile-time error
    int b = (int) d; // valid conversion to int, but result is truncated (gets rounded down)
                     // This is type-casting
                     // Now, b = 18

## Uzun ilkel
Varsayılan olarak, "uzun" 64 bit işaretli bir tamsayıdır (Java 8'de imzalı veya imzasız olabilir). İmzalı, minimum -2<sup>63</sup> ve maksimum 2<sup>63</sup> - 1 değerini saklayabilir ve imzasız minimum 0 ve maksimum değeri saklayabilir. 2 değeri<sup>64</sup> - 1

## Negatif değer gösterimi
Java ve diğer dillerin çoğu, negatif tam sayıları *2'nin tamamlayıcısı* gösterimi adı verilen bir gösterimde saklar.

"n" bitlerini kullanan bir veri türünün benzersiz bir ikili gösterimi için değerler şu şekilde kodlanır:

En az anlamlı 'n-1' bitleri, integral gösteriminde pozitif bir 'x' tamsayısını depolar. En önemli değer, `s` değeri ile biraz depolar. Bu bitler tarafından temsil edilen değer

x - s * 2<sup>n-1</sup>

yani en anlamlı bit 1 ise, diğer bitlerle temsil edebileceğiniz sayıdan sadece 1 büyük olan bir değer (<code>2<sup>n-2</sup> + 2<sup>n- 3</sup> + ... + 2<sup>1</sup> + 2<sup>0</sup> = 2<sup>n-1</sup> - 1</code>) çıkarılır - 2<sup>n-1</sup> (s = 1; x = 0) ile 2<sup>n-1</sup> - 1 (s = 0; x) arasındaki her bir değer için benzersiz bir ikili gösterime izin verir = 2<sup>n-1</sup> - 1).

Bunun aynı zamanda, ikili gösterimleri pozitif ikili sayılarmış gibi ekleyebileceğiniz hoş bir yan etkisi vardır:

<pre>v1 = x1 - s1 * 2<sup>n-1</sup>
v2 = x2 - s2 * 2<sup>n-1</sup></pre>

| s1 | s2 | x1 + x2 taşması | ekleme sonucu |
| ------ | ------ | --- | --- |
| 0 | 0 | Hayır | x1 + x2 = v1 + v2 |
| 0 | 0 | Evet | veri türüyle temsil edilemeyecek kadar büyük (taşma) |
| 0 | 1 | Hayır | <pre>x1 + x2 - 2<sup>n-1</sup> = x1 + x2 - s2 * 2<sup>n-1</sup><br> = v1 + v2</pre> |
| 0 | 1 | Evet | <pre>(x1 + x2) <b>mod</b> 2<sup>n-1</sup> = x1 + x2 - 2<sup>n-1</sup><br> = v1 + v2 </pre> |
| 1 | 0 | \* | yukarıya bakın (takas toplamları) |
| 1 | 1 | Hayır | veri türüyle temsil edilemeyecek kadar küçük (x1 + x2 - 2<sup>n</sup> < -2<sup>n-1</sup> ; taşma) |
| 1 | 1 | Evet | <pre>(x1 + x2) <b>mod</b> 2<sup>n-1</sup> - 2<sup>n-1</sup> = (x1 + x2 - 2<sup>n -1</sup>) - 2<sup>n-1</sup><br> = (x1 - s1 * 2<sup>n-1</sup>) + (x2 - s2 * 2<sup> n-1</sup>)<br> = v1 + v2</pre> |

Bu gerçeğin, toplamsal tersinin (yani negatif değerin) ikili gösterimini bulmayı kolaylaştırdığına dikkat edin:

Sayıya bitsel tümleyeni eklemenin tüm bitlerin 1 olmasına neden olduğunu gözlemleyin. Şimdi değer taşması yapmak için 1 ekleyin ve nötr eleman 0'ı elde edin (tüm bitler 0).

Bu nedenle, "i" sayısının negatif değeri kullanılarak hesaplanabilir (burada olası "int" yükseltmesi göz ardı edilerek)

    (~i) + 1

---

**Örnek:** 0'ın (`byte`) negatif değerini almak:

"0"ın olumsuzlanmasının sonucu "11111111" olur. 1 eklenmesi "100000000" (9 bit) değerini verir. Bir "bayt" yalnızca 8 bit depolayabildiğinden, en soldaki değer kesilir ve sonuç "00000000" olur.

| Orijinal | süreç | Sonuç |
| ------ | ------ | ------ |
| 0 (00000000) | olumsuzla | -0 (11111111)
| 11111111 | 1'i ikiliye ekle | 100000000
| 100000000 | 8 bite kısalt | 00000000 (-0 eşittir 0)

 

## Kısa ilkel
"Kısa", 16 bitlik işaretli bir tamsayıdır. Minimum değeri -2<sup>15</sup> (-32.768) ve maksimum değeri 2<sup>15</sup>&nbsp;&#8209;1 (32.767)

    short example = -48;
    short myShort = 987;
    short anotherShort = 17;

    short addedShorts = (short) (myShort + anotherShort); // 1,004
    short subtractedShorts = (short) (myShort - anotherShort); // 970

"Kısa"nın maksimum ve minimum değerleri şurada bulunabilir:

    short high = Short.MAX_VALUE;        // high == 32767
    short low = Short.MIN_VALUE;         // low == -32768

Bir "kısa"nın varsayılan değeri _0_
    
    short defaultShort;    // defaultShort == 0
