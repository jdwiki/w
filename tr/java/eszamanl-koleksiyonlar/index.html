Bir concurrent collection, aynı anda birden fazla iş parçacığı tarafından erişime izin veren bir koleksiyon‘dir. Farklı iş parçacıkları tipik olarak koleksiyonun içeriğini yineleyebilir ve öğeler ekleyebilir veya kaldırabilir. Koleksiyonun bozulmamasını sağlamaktan koleksiyon sorumludur.

İplik Güvenli Koleksiyonlar #

Varsayılan olarak, çeşitli Koleksiyon türleri iş parçacığı için güvenli değildir.

Ancak, bir koleksiyonu iş parçacığı için güvenli hale getirmek oldukça kolaydır.

List<String> threadSafeList = Collections.synchronizedList(new ArrayList<String>());
Set<String> threadSafeSet = Collections.synchronizedSet(new HashSet<String>());
Map<String, String> threadSafeMap = Collections.synchronizedMap(new HashMap<String, String>());

İş parçacığı güvenli bir koleksiyon oluşturduğunuzda, ona asla orijinal koleksiyon üzerinden erişmemelisiniz, yalnızca iş parçacığı güvenli sarmalayıcı aracılığıyla.

Java 5’ten başlayarak, “java.util.collections”, çeşitli “Collections.synchronized” yöntemlerine ihtiyaç duymayan, iş parçacığı için güvenli birkaç yeni koleksiyona sahiptir.

List<String> threadSafeList = new CopyOnWriteArrayList<String>();
Set<String> threadSafeSet = new ConcurrentHashSet<String>();
Map<String, String> threadSafeMap = new ConcurrentHashMap<String, String>();

Eşzamanlı Koleksiyonlar #

Eşzamanlı koleksiyonlar, eşzamanlı bir ortamda daha geniş bir kullanıma izin veren iş parçacığı güvenli koleksiyonların bir genellemesidir.

İş parçacığı güvenli koleksiyonlar, birden çok iş parçacığından güvenli öğe ekleme veya çıkarma işlemine sahip olsa da, aynı bağlamda güvenli yinelemeye sahip olmaları gerekmez (bir iş parçacığında koleksiyonda güvenli bir şekilde yineleme yapamayabilirken, diğeri onu ekleyerek/ekleyerek değiştirir). elemanların çıkarılması).

Eşzamanlı koleksiyonların kullanıldığı yer burasıdır.

Yineleme genellikle koleksiyonlardaki ‘addAll’, ‘removeAll’ veya ayrıca koleksiyon kopyalama (bir kurucu veya başka yollarla), sıralama gibi çeşitli toplu yöntemlerin temel uygulaması olduğundan, eşzamanlı koleksiyonlar için kullanım durumu aslında oldukça büyük.

Örneğin, Java SE 5 java.util.concurrent.CopyOnWriteArrayList iş parçacığı için güvenli ve eşzamanlı bir List uygulamasıdır, onun javadoc /Java/util/concurrent/CopyOnWriteArrayList.html) şunu belirtir:

“Anlık görüntü” stili yineleyici yöntemi, yineleyicinin oluşturulduğu noktada dizinin durumuna bir başvuru kullanır. Bu dizi, yineleyicinin ömrü boyunca hiçbir zaman değişmez, bu nedenle girişim imkansızdır ve yineleyicinin ConcurrentModificationException oluşturmaması garanti edilir.

Bu nedenle, aşağıdaki kod güvenlidir:

public class ThreadSafeAndConcurrent {

public static final List<Integer> LIST = new CopyOnWriteArrayList<>();

public static void main(String[] args) throws InterruptedException {
    Thread modifier = new Thread(new ModifierRunnable());
    Thread iterator = new Thread(new IteratorRunnable());
    modifier.start();
    iterator.start();
    modifier.join();
    iterator.join();
}

public static final class ModifierRunnable implements Runnable {
    @Override
    public void run() {
        try {
            for (int i = 0; i < 50000; i++) {
                LIST.add(i);
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

public static final class IteratorRunnable implements Runnable {
    @Override
    public void run() {
        try {
            for (int i = 0; i < 10000; i++) {
                long total = 0;
                for(Integer inList : LIST) {
                    total += inList;
                }
                System.out.println(total);
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
}

Yineleyiciler, yineleyicinin oluşturulmasından bu yana veya bir noktada sıranın durumunu yansıtan öğeler döndürerek zayıf tutarlıdır. Java.util.ConcurrentModificationException’ı atmazlar ve diğer işlemlerle aynı anda devam edebilirler. Yineleyicinin oluşturulmasından bu yana kuyrukta bulunan öğeler tam olarak bir kez döndürülür.

Bir koleksiyonun eşzamanlı olup olmadığını görmek için javadocs kontrol edilmelidir. iterator() yöntemi tarafından döndürülen yineleyicinin nitelikleri (“hızlı başarısız”, “zayıf tutarlı”, …) aranacak en önemli niteliktir.

İş parçacığı güvenli ancak eşzamanlı olmayan örnekler #

Yukarıdaki kodda, ‘LIST’ bildirimini şu şekilde değiştirmek:

public static final List<Integer> LIST = Collections.synchronizedList(new ArrayList<>());

İstisnalara yol açabilir (ve istatistiksel olarak çoğu modern, çoklu CPU/çekirdek mimarisinde olacaktır).

‘Koleksiyonlar’ yardımcı program yöntemlerinden senkronize edilmiş koleksiyonlar, öğelerin eklenmesi/kaldırılması için iş parçacığı açısından güvenlidir, ancak yineleme değildir (temel koleksiyon zaten kendisine iletilmediği sürece).
