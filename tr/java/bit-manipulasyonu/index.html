Bu sayfada

  • Ayrı bitleri kontrol etme, ayarlama, temizleme ve değiştirme. Bit maskesi kadar uzun kullanma
  • java.util.BitSet sınıfı
  • Bir sayının 2’nin katı olup olmadığını kontrol etme
  • Değerleri bit parçaları olarak paketleme / açma
  • 2’nin gücünü ifade etme
  • İmzalı ve imzasız vardiya

  • C/C++‘dan farklı olarak Java, temel alınan makine donanımına göre tamamen endian-nötrdür. Varsayılan olarak büyük veya küçük endian davranışı almazsınız; hangi davranışı istediğinizi açıkça belirtmelisiniz.

  • “bayt” türü, -128 ila +127 aralığında imzalanmıştır. Bir bayt değerini işaretsiz eşdeğerine dönüştürmek için 0xFF ile şu şekilde maskeleyin: (b & 0xFF).

Ayrı bitleri kontrol etme, ayarlama, temizleme ve değiştirme. Bit maskesi kadar uzun kullanma #

Bir tamsayı primitifinin “n” bitini, “i"yi (bayt, kısa, karakter, int veya uzun) değiştirmek istediğimizi varsayarsak:

(i & 1 << n) != 0 // checks bit 'n'
i |= 1 << n;      // sets bit 'n' to 1
i &= ~(1 << n);   // sets bit 'n' to 0
i ^= 1 << n;      // toggles the value of bit 'n'

Bit maskesi olarak long/int/short/byte kullanmak:

public class BitMaskExample {
    private static final long FIRST_BIT = 1L << 0;
    private static final long SECOND_BIT = 1L << 1;
    private static final long THIRD_BIT = 1L << 2;
    private static final long FOURTH_BIT = 1L << 3;
    private static final long FIFTH_BIT = 1L << 4;
    private static final long BIT_55 = 1L << 54;

    public static void main(String[] args) {
        checkBitMask(FIRST_BIT | THIRD_BIT | FIFTH_BIT | BIT_55);
    }

    private static void checkBitMask(long bitmask) {
        System.out.println("FIRST_BIT: " + ((bitmask & FIRST_BIT) != 0));
        System.out.println("SECOND_BIT: " + ((bitmask & SECOND_BIT) != 0));
        System.out.println("THIRD_BIT: " + ((bitmask & THIRD_BIT) != 0));
        System.out.println("FOURTh_BIT: " + ((bitmask & FOURTH_BIT) != 0));
        System.out.println("FIFTH_BIT: " + ((bitmask & FIFTH_BIT) != 0));
        System.out.println("BIT_55: " + ((bitmask & BIT_55) != 0));
    }
}

Baskılar

FIRST_BIT: true
SECOND_BIT: false
THIRD_BIT: true
FOURTh_BIT: false
FIFTH_BIT: true
BIT_55: true

hangi checkBitMask parametresi olarak ilettiğimiz maskeyle eşleşir: FIRST_BIT | ÜÇÜNCÜ_BIT | BEŞİNCİ_BIT | BIT_55.

java.util.BitSet sınıfı #

1.7’den beri, basit ve kullanıcı dostu bit depolama ve işleme arabirimi sağlayan bir java.util.BitSet sınıfı vardır:

final BitSet bitSet = new BitSet(8); // by default all bits are unset

IntStream.range(0, 8).filter(i -> i % 2 == 0).forEach(bitSet::set); // {0, 2, 4, 6}

bitSet.set(3); // {0, 2, 3, 4, 6}

bitSet.set(3, false); // {0, 2, 4, 6}

final boolean b = bitSet.get(3); // b = false

bitSet.flip(6); // {0, 2, 4}

bitSet.set(100); // {0, 2, 4, 100} - expands automatically

‘BitSet’, ‘Klonlanabilir’ ve ‘Serializable’ı uygular ve başlık altında tüm bit değerleri, otomatik olarak genişleyen ‘uzun[] kelimeler’ alanında saklanır.

Ayrıca, “and”, “veya”, “xor”, “andNot” gibi tam küme mantıksal işlemlerini de destekler:

bitSet.and(new BitSet(8));
bitSet.or(new BitSet(8));
bitSet.xor(new BitSet(8));
bitSet.andNot(new BitSet(8));

Bir sayının 2’nin katı olup olmadığını kontrol etme #

Bir “x” tamsayısı 2’nin bir katıysa, yalnızca bir bit ayarlanır, oysa “x-1” bundan sonra tüm bitleri ayarlar. Örneğin: ‘4’, ‘100’ ve ‘3’, ikili sayı olarak ‘011’dir ve yukarıda belirtilen koşulu sağlar. Sıfır, 2’nin kuvveti değildir ve açıkça kontrol edilmesi gerekir.

boolean isPowerOfTwo(int x)
{
    return (x != 0) && ((x & (x - 1)) == 0);
}

Sol ve Sağ Kaydırma için Kullanım

Diyelim ki READ, WRITE ve EXECUTE olmak üzere üç tür iznimiz var. Her izin 0 ile 7 arasında değişebilir. (4 bitlik sayı sistemini varsayalım)

KAYNAK = OKUMA YAZMA ÇALIŞTIR (12 bitlik sayı)

KAYNAK = 0100 0110 0101 = 4 6 5 (12 bitlik sayı)

Yukarıda (12 bitlik sayı) ayarlanan (12 bitlik sayı) izinlerini nasıl alabiliriz?

0100 0110 0101

0000 0000 0111 (&)

0000 0000 0101 = 5

RESOURCE‘un EXECUTE izinlerini bu şekilde alabiliriz. Şimdi, KAYNAK için OKU izinlerini almak istiyorsak ne olur?

0100 0110 0101

0111 0000 0000 (&)

0100 0000 0000 = 1024

Doğru? Muhtemelen bunu varsayıyorsunuz? Ancak, izinler 1024 ile sonuçlanır. Kaynak için yalnızca OKUMA izinleri almak istiyoruz. Endişelenme, bu yüzden vardiya operatörlerimiz vardı. Görüyorsak, OKUMA izinleri gerçek sonucun 8 bit gerisindedir, bu yüzden bazı kaydırma operatörleri uygularsak, OKUMA izinlerini sonucun en sağına getirecek mi? Ne yaparsak:

0100 0000 0000 » 8 => 0000 0000 0100 (Çünkü pozitif bir sayıdır yani 0’larla değiştirildi, işareti umursamıyorsanız, sadece imzasız kullanın sağa kaydırma operatörü)

Artık aslında 4 olan OKUMA izinlerine sahibiz.

Şimdi, örneğin, bir KAYNAK için bize OKU, YAZ, YÜRÜT izinleri verildi, bu KAYNAK için izinler almak için ne yapabiliriz?

Önce ikili izinler örneğini ele alalım. (Hala 4 bit sayı sistemi varsayılarak)

OKUYUN = 0001

YAZ = 0100

YÜRÜT = 0110

Basitçe yapacağımızı düşünüyorsanız:

OKU | YAZ | EXECUTE, biraz haklısın ama tam olarak değil. Bakın READ yaparsak ne olacak | YAZ | UYGULAMAK

0001 | 0100 | 0110 => 0111

Ancak izinler aslında (bizim örneğimizde) 0001 0100 0110 olarak temsil ediliyor.

Yani bunu yapabilmek için OKU’nun 8 bit, YAZIN‘ın 4 bit ve İZİNLER‘in en sona yerleştirildiğini biliyoruz. RESOURCE izinleri için kullanılan sayı sistemi aslında 12 bittir (bizim örneğimizde). Farklı sistemlerde farklı olabilir(olacaktır).

(OKUYUN « 8) | (YAZ « 4) | (UYGULAMAK)

0000 0000 0001 « 8 (OKU)

0001 0000 0000 (8 bit sola kaydırma)

0000 0000 0100 « 4 (YAZ)

0000 0100 0000 (4 bit sola kaydırma)

0000 0000 0001 (ÇALIŞTIR)

Şimdi yukarıdaki kaydırmanın sonuçlarını eklersek, şöyle bir şey olacak;

0001 0000 0000 (OKU)

0000 0100 0000 (YAZ)

0000 0000 0001 (ÇALIŞTIR)

0001 0100 0001 (İZİNLER)

Değerleri bit parçaları olarak paketleme / açma #

Bellek performansının birden çok değeri tek bir ilkel değere sıkıştırması yaygındır. Bu, çeşitli bilgileri tek bir değişkene aktarmak için faydalı olabilir.

Örneğin, RGB içindeki renk kodu gibi - 3 bayt tek bir int olarak paketlenebilir.

Değerleri paketleme

// Raw bytes as input
byte[] b = {(byte)0x65, (byte)0xFF, (byte)0x31};

// Packed in big endian: x == 0x65FF31
int x = (b[0] & 0xFF) << 16  // Red
      | (b[1] & 0xFF) <<  8  // Green
      | (b[2] & 0xFF) <<  0; // Blue

// Packed in little endian: y == 0x31FF65
int y = (b[0] & 0xFF) <<  0
      | (b[1] & 0xFF) <<  8
      | (b[2] & 0xFF) << 16;

Değerleri açma

// Raw int32 as input
int x = 0x31FF65;

// Unpacked in big endian: {0x65, 0xFF, 0x31}
byte[] c = {
    (byte)(x >> 16),
    (byte)(x >>  8),
    (byte)(x & 0xFF)
};

// Unpacked in little endian: {0x31, 0xFF, 0x65}
byte[] d = {
    (byte)(x & 0xFF),
    (byte)(x >>  8),
    (byte)(x >> 16)
};

2’nin gücünü ifade etme #

2 (2^n) tamsayının gücünü ifade etmek için, “n"yi açıkça belirtmeye izin veren bir bit kaydırma işlemi kullanılabilir.

Sözdizimi temel olarak:

int pow2 = 1<<n;

Örnekler:

int twoExp4 = 1<<4; //2^4
int twoExp5 = 1<<5; //2^5
int twoExp6 = 1<<6; //2^6
...
int twoExp31 = 1<<31; //2^31

Bu, özellikle onaltılık veya ondalık değerler kullanmak yerine 2’nin kuvvetinin kullanıldığını açıkça göstermesi gereken sabit değerleri tanımlarken kullanışlıdır.

int twoExp4 = 0x10; //hexadecimal
int twoExp5 = 0x20; //hexadecimal
int twoExp6 = 64; //decimal
...
int twoExp31 = -2147483648; //is that a power of 2?

2’nin int gücünü hesaplamak için basit bir yöntem

int pow2(int exp){
    return 1<<exp;
}

İmzalı ve imzasız vardiya #

Java’da tüm sayı ilkelleri imzalanmıştır. Örneğin, bir int her zaman [-2^31 - 1, 2^31] arasındaki değerleri temsil eder ve ilk biti - negatif değer için 1, pozitif için 0 değerini imzalayacak şekilde tutar.

Temel kaydırma operatörleri >> ve << imzalı operatörlerdir. Değerin işaretini koruyacaklar.

Ancak programcıların işaretsiz değerleri saklamak için sayıları kullanması yaygındır. Bir int için, işaretli bir int ile iki kat daha fazla değere sahip olmak için aralığı [0, 2^32 - 1]’e kaydırmak anlamına gelir.

Bu uzman kullanıcılar için, işaretin biti anlamsızdır. Bu nedenle Java, bu işaret bitini göz ardı ederek bir sola kaydırma operatörü olan >>> ekledi.

                initial value:               4 (                                100)
     signed left-shift: 4 << 1               8 (                               1000)
    signed right-shift: 4 >> 1               2 (                                 10)
 unsigned right-shift: 4 >>> 1               2 (                                 10)
                initial value:              -4 (   11111111111111111111111111111100)
    signed left-shift: -4 << 1              -8 (   11111111111111111111111111111000)
   signed right-shift: -4 >> 1              -2 (   11111111111111111111111111111110)
unsigned right-shift: -4 >>> 1      2147483646 (    1111111111111111111111111111110)

Neden <<< ? yok

Bu, sağa kaymanın amaçlanan tanımından gelir. Soldaki boş yerleri doldurduğu için işaret biti konusunda alınacak bir karar yok. Sonuç olarak, 2 farklı operatöre gerek yoktur.

Daha ayrıntılı bir yanıt için bu [soruya](https://www.quora.com/ Why-is-there-no-unsigned-left-shift-operator-in-Java) bakın.
