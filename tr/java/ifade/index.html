Java’daki ifadeler, hesaplama yapmak için birincil yapıdır.

İfadelerde kullanılabilecek işleçlere ilişkin bir başvuru için bkz. İşleçler.

Operatör Önceliği #

Bir ifade birden çok operatör içerdiğinde, potansiyel olarak farklı şekillerde okunabilir. Örneğin, “1 + 2 x 3” matematiksel ifadesi iki şekilde okunabilir:

  1. “1” ve “2"yi ekleyin ve sonucu “3” ile çarpın. Bu “9” cevabını verir. Parantez eklersek, bu ( 1 + 2 ) x 3 gibi görünür.
  2. “2” ve “3"ü çarpmanın sonucuna “1” ekleyin. Bu “7” cevabını verir. Parantez eklersek, bu 1 + ( 2 x 3 ) gibi görünür.

Matematikte uzlaşım, ifadeyi ikinci yoldan okumaktır. Genel kural, çarpma ve bölmenin toplama ve çıkarmadan önce yapılmasıdır. Daha ileri matematiksel gösterim kullanıldığında, anlam ya “apaçıktır” (eğitimli bir matematikçiye!) ya da belirsizliği gidermek için parantezler eklenir. Her iki durumda da, notasyonun anlamı aktarmadaki etkinliği, matematikçilerin zekasına ve paylaşılan bilgisine bağlıdır.

Java, kullanılan operatörlerin önceliğine dayalı olarak bir ifadenin nasıl okunacağı konusunda aynı net kurallara sahiptir.

Genel olarak, her operatöre bir öncelik değeri atanır; aşağıdaki tabloya bakın.

Örneğin:

  1 + 2 * 3

‘+’ önceliği, ‘*’ önceliğinden daha düşüktür, bu nedenle ifadenin sonucu 9 değil 7’dir.

İfade değerlendirme sırası #

Java ifadeleri aşağıdaki kurallara göre değerlendirilir:

  • İşlenenler soldan sağa doğru değerlendirilir.
  • Bir operatörün işlenenleri, operatörden önce değerlendirilir.
  • Operatörler, operatör önceliğine göre değerlendirilir
  • Argüman listeleri soldan sağa doğru değerlendirilir.

Basit Örnek #

Aşağıdaki örnekte:

int i = method1() + method2();

değerlendirme sırası:

  1. = operatörünün sol işleneni i adresine değerlendirilir.
  2. + operatörünün (method1()) sol işleneni değerlendirilir.
  3. + operatörünün (method2()) sağ işleneni değerlendirilir.
  4. + işlemi değerlendirilir.
  5. ‘=’ işlemi, ‘i’ye eklemenin sonucu atanarak değerlendirilir.

Çağrıların etkileri gözlemlenebilirse, “method1” çağrısının “method2” çağrısından önce gerçekleştiğini gözlemleyebileceksiniz.

Yan etkisi olan bir operatör ile örnek #

Aşağıdaki örnekte:

int i = 1;
intArray[i] = ++i + 1;

değerlendirme sırası:

  1. = operatörünün sol işleneni değerlendirilir. Bu, ‘intArray1’ adresini verir.
  2. Ön artış değerlendirilir. Bu, “i"ye “1” ekler ve “2” olarak değerlendirilir.
  3. ‘+‘nın sağ işleneni değerlendirilir.
  4. “+” işlemi şu şekilde değerlendirilir: “2 + 1” -> “3”.
  5. ‘=’ işlemi değerlendirilir ve ‘intArray1’ öğesine ‘3’ atanır.

= nin sol işleneni ilk olarak değerlendirildiği için, ++i alt ifadesinin yan etkisinden etkilenmediğine dikkat edin.

Referans:

  • JLS 15.7 - Değerlendirme Siparişi

İfade Temelleri #

Java’daki ifadeler, hesaplama yapmak için birincil yapıdır. İşte bazı örnekler:

1                 // A simple literal is an expression
1 + 2             // A simple expression that adds two numbers
(i + j) / k       // An expression with multiple operations
(flag) ? c : d    // An expression using the "conditional" operator
(String) s        // A type-cast is an expression
obj.test()        // A method call is an expression
new Object()      // Creation of an object is an expression
new int[]         // Creation of an object is an expression

Genel olarak, bir ifade aşağıdaki formlardan oluşur:

  • Aşağıdakilerden oluşan ifade adları:
    • Simple identifiers; e.g. someIdentifier
    • Qualified identifiers; e.g. MyClass.someField
  • Aşağıdakilerden oluşan primerler:
    • Literals; e.g. 1, 1.0, 'X', "hello", false and null
    • Class literal expressions; e.g. MyClass.class
    • this and <TypeName> . this
    • Parenthesized expressions; e.g. ( a + b )
    • Class instance creation expressions; e.g. new MyClass(1, 2, 3)
    • Array instance creation expressions; e.g. new int[3]
    • Field access expressions; e.g. obj.someField or this.someField
    • Array access expressions; e.g. vector[21]
    • Method invocations; e.g. obj.doIt(1, 2, 3)
    • Method references (Java 8 and later); e.g. MyClass::doIt
  • Tekli operatör ifadeleri; Örneğin. !a veya i++
  • İkili operatör ifadeleri; Örneğin. “a + b” veya “obj == boş”
  • Üçlü operatör ifadeleri; Örneğin. (nesne == boş) ? 1: obj.getCount()
  • Lambda ifadeleri (Java 8 ve üstü); Örneğin. obj -> obj.getCount()

Farklı ifade biçimlerinin ayrıntıları diğer Konularda bulunabilir.

  • Operators konusu birli, ikili ve üçlü operatör ifadelerini kapsar.
  • Lambda ifadeleri konusu, lambda ifadeleri ve yöntem referans ifadelerini kapsar.
  • Sınıflar ve Nesneler konusu, sınıf örneği oluşturma ifadelerini kapsar.
  • Diziler konusu, dizi erişim ifadelerini ve dizi örneği oluşturma ifadelerini kapsar.
  • Sabit Değerler konusu, farklı sabit değer ifadeleri türlerini kapsar.

Bir İfadenin Türü #

Çoğu durumda, bir ifade, derleme zamanında ve alt ifadeleri incelenerek belirlenebilen statik bir türe sahiptir. Bunlara bağımsız ifadeler denir.

Ancak (Java 8 ve sonraki sürümlerde) aşağıdaki türde ifadeler çoklu ifadeler: olabilir:

  • Parantez içindeki ifadeler
  • Sınıf örneği oluşturma ifadeleri
  • Yöntem çağırma ifadeleri
  • Yöntem referans ifadeleri
  • Koşullu ifadeler
  • Lambda ifadeleri

Bir ifade bir poli ifadesi olduğunda, türü ifadenin hedef türünden etkilenebilir; yani ne için kullanılıyor.

Bir İfadenin değeri #

Bir ifadenin değeri, türüyle uyumlu atamadır. Bunun istisnası, yığın kirliliğinin meydana gelmesidir; Örneğin. çünkü “güvenli olmayan dönüştürme” uyarıları (uygun olmayan bir şekilde) bastırılmış veya yok sayılmıştır.

İfade İfadeleri #

Diğer birçok dilin aksine, Java genellikle ifadelerin ifade olarak kullanılmasına izin vermez. Örneğin:

public void compute(int i, int j) {
    i + j;   // ERROR
}

Gibi bir ifadenin değerlendirilmesinin sonucu kullanılamayacağından ve programın yürütülmesini başka bir şekilde etkileyemeyeceğinden, Java tasarımcıları bu tür bir kullanımın ya bir hata ya da yanlış yönlendirildiği görüşünü aldılar.

Ancak, bu tüm ifadeler için geçerli değildir. İfadelerin bir alt kümesi (aslında) ifadeler kadar yasaldır. Set şunları içerir:

  • işlem-ve-oluşur atamaları dahil atama ifadesi.
  • Ön ve son artırma ve eksiltme ifadeleri.
  • Yöntem çağrıları (“void” veya “void olmayan”).
  • Sınıf örneği oluşturma ifadeleri.

Sabit İfadeler #

Sabit bir ifade, ilkel bir tür veya bir Dize veren ve değeri derleme zamanında bir hazır bilgi olarak değerlendirilebilen bir ifadedir. İfade, bir istisna atmadan değerlendirmeli ve yalnızca aşağıdakilerden oluşmalıdır:

  • İlkel ve String değişmezleri.

  • İlkel türlere veya ‘Dize’ye yayınları yazın.

  • Şu tekli operatörler: +, -, ~ ve !.

  • Şu ikili operatörler: *, /, %, +, -, <<, >>, >>>, <, <= , >, >=, ==, !=, &, ^, |, && ve ||.

  • Üçlü koşul operatörü ? :.

  • Parantez içindeki sabit ifadeler.

  • Sabit değişkenlere atıfta bulunan basit isimler. (Sabit değişken, başlatıcı ifadesinin kendisinin sabit bir ifade olduğu ‘final’ olarak bildirilen bir değişkendir.)

  • <TypeName> formunun nitelikli adları. Sabit değişkenlere atıfta bulunan <Identifier>.

Yukarıdaki listede *++” ve “–”, atama operatörleri, “sınıf” ve “instanceof”, yöntem çağrıları ve genel değişkenlere veya alanlara yapılan referanslar *hariç tutulur.

“Dize” türündeki sabit ifadeler “iç içe” bir “Dize” ile sonuçlanır ve sabit ifadelerdeki kayan nokta işlemleri FP katı semantiği ile değerlendirilir.

Sabit İfadeler için Kullanımlar #

Sabit ifadeler, normal bir ifadenin kullanılabileceği her yerde (hemen hemen) kullanılabilir. Ancak, aşağıdaki bağlamlarda özel bir öneme sahiptirler.

Switch deyimlerindeki büyük/küçük harf etiketleri için sabit ifadeler gereklidir. Örneğin:

switch (someValue) {
case 1 + 1:            // OK
case Math.min(2, 3):   // Error - not a constant expression
    doSomething();
}

Bir atamanın sağ tarafındaki ifade sabit bir ifade olduğunda, atama ilkel bir daraltma dönüşümü gerçekleştirebilir. Buna, sabit ifadenin değerinin sol taraftaki türün aralığında olması koşuluyla izin verilir. (Bkz. JLS 5.1.3 ve 5.2) Örneğin:

byte b1 = 1 + 1;             // OK - primitive narrowing conversion.
byte b2 = 127 + 1;           // Error - out of range
byte b3 = b1 + 1;            // Error - not a constant expession
byte b4 = (byte) (b1 + 1);   // OK

Bir “do”, “while” veya “for” içinde koşul olarak sabit bir ifade kullanıldığında, okunabilirlik analizini etkiler. Örneğin:

while (false) {
    doSomething();           // Error - statenent not reachable
}
boolean flag = false;
while (flag) {
    doSomething();           // OK
}

(Bunun “if” ifadeleri için geçerli olmadığını unutmayın. Java derleyicisi “if” ifadesinin “then” veya “else” bloğunun erişilemez olmasına izin verir. Bu, C ve C++‘daki koşullu derlemenin Java analoğudur.)

Son olarak, sabit ifade başlatıcıları olan bir sınıf veya arabirimdeki “statik final” alanları hevesle başlatılır. Böylece, sınıf başlatma bağımlılık grafiğinde bir döngü olsa bile, bu sabitlerin başlatılmış durumda gözlemlenmesi garanti edilir.

Daha fazla bilgi için bkz. JLS 15.28. Sabit İfadeler.
