  • Sözdizimi
  • Yakalama gruplarını kullanma
  • Deseni bayraklarla derleyerek normal ifadeyi özel davranışla kullanma
  • Kaçış Karakterleri
  • Belirli bir dizeyle eşleşmiyor
  • Normal ifade değişmeziyle eşleştirme.
  • Ters eğik çizgi eşleştirme

Normal ifade, bir kalıpta tutulan özel bir sözdizimi kullanarak diğer dizeleri veya dize kümelerini eşleştirmeye veya bulmaya yardımcı olan özel bir karakter dizisidir. Java, java.util.regex paketi aracılığıyla düzenli ifade kullanımını destekler. Bu konu, Java’da Normal İfadelerin nasıl kullanılması gerektiğine ilişkin örneklerle geliştiricilerin daha fazlasını anlamalarına yardımcı olmak ve tanıtmaktır.

Sözdizimi #

  • Pattern patternName = Pattern.compile(regex);
  • Eşleştirici eşleştiriciAdı = modelAdı.eşleştirici(textToSearch);
  • matcherName.matches() // textToSearch normal ifadeyle tam olarak eşleşirse true döndürür
  • matcherName.find() // Normal ifadeyle eşleşen bir alt dizenin ilk örneğini textToSearch aracılığıyla arar. Sonraki çağrılar, Dize’nin geri kalanını arayacaktır.
  • matcherName.group(groupNum) //Bir yakalama grubunun içindeki alt dizeyi döndürür
  • matcherName.group(groupName) //Adlandırılmış bir yakalama grubunun içindeki alt dizeyi döndürür (Java 7+)

İthalat #

Regex’i kullanmadan önce aşağıdaki içe aktarmaları eklemeniz gerekecek:

import java.util.regex.Matcher
import java.util.regex.Pattern

Tuzaklar #

Java’da, bir ters eğik çizgi çift ters eğik çizgi ile kaçar, bu nedenle normal ifade dizesindeki bir ters eğik çizgi çift ters eğik çizgi olarak girilmelidir. Çift ters eğik çizgiden kaçmanız gerekiyorsa (tek bir ters eğik çizgiyi normal ifadeyle eşleştirmek için, onu dörtlü ters eğik çizgi olarak girmeniz gerekir.

Yakalama gruplarını kullanma #

Girdi dizesinden dizenin bir bölümünü çıkarmanız gerekirse, normal ifadenin yakalama gruplarını kullanabiliriz.

Bu örnek için basit bir telefon numarası normal ifadesi ile başlayacağız:

\d{3}-\d{3}-\d{4}

Normal ifadeye parantezler eklenirse, her parantez kümesi bir yakalama grubu olarak kabul edilir. Bu durumda, numaralandırılmış yakalama grupları denilen şeyi kullanıyoruz:

(\d{3})-(\d{3})-(\d{4})
^-----^ ^-----^ ^-----^
Group 1 Group 2 Group 3

Java’da kullanmadan önce, ters eğik çizgilerden kaçarak aşağıdaki örüntüyle sonuçlanan Strings kurallarına uymayı unutmamalıyız:

"(\\d{3})-(\\d{3})-(\\d{4})"

İlk önce bir “Pattern” oluşturmak için regex modelini derlememiz gerekiyor ve ardından girdi dizgimizi modelle eşleştirmek için bir “Eşleştirici"ye ihtiyacımız var:

Pattern phonePattern = Pattern.compile("(\\d{3})-(\\d{3})-(\\d{4})");
Matcher phoneMatcher = phonePattern.matcher("abcd800-555-1234wxyz");

Ardından, Eşleştiricinin normal ifadeyle eşleşen ilk alt diziyi bulması gerekir:

phoneMatcher.find();

Şimdi, group yöntemini kullanarak, dizgeden verileri çıkarabiliriz:

String number = phoneMatcher.group(0); //"800-555-1234" (Group 0 is everything the regex matched)
String aCode = phoneMatcher.group(1); //"800"
String threeDigit = phoneMatcher.group(2); //"555"
String fourDigit = phoneMatcher.group(3); //"1234"

Not: ‘Matcher.group()’, ‘Matcher.group(0)’ yerine kullanılabilir.

Java 7, adlandırılmış yakalama gruplarını tanıttı. Adlandırılmış yakalama grupları, küçük sözdizimi değişiklikleri olsa da, numaralandırılmış yakalama gruplarıyla aynı işlevi görür (ancak sayı yerine bir adla). Adlandırılmış yakalama gruplarının kullanılması okunabilirliği artırır.

Adlandırılmış grupları kullanmak için yukarıdaki kodu değiştirebiliriz:

(?<AreaCode>\d{3})-(\d{3})-(\d{4})
^----------------^ ^-----^ ^-----^
AreaCode           Group 2 Group 3

“AreaCode” içeriğini almak için bunun yerine şunları kullanabiliriz:

String aCode = phoneMatcher.group("AreaCode"); //"800"
son sürüm

Deseni bayraklarla derleyerek normal ifadeyi özel davranışla kullanma #

Bir “Pattern”, bayraklarla derlenebilir, regex değişmez bir “Dize” olarak kullanılıyorsa, satır içi değiştiricileri kullanın:

Pattern pattern = Pattern.compile("foo.", Pattern.CASE_INSENSITIVE | Pattern.DOTALL);
pattern.matcher("FOO\n").matches(); // Is true.

/* Had the regex not been compiled case insensitively and singlelined,
 * it would fail because FOO does not match /foo/ and \n (newline)
 * does not match /./.
 */

Pattern anotherPattern = Pattern.compile("(?si)foo");
anotherPattern.matcher("FOO\n").matches(); // Is true.

"foOt".replaceAll("(?si)foo", "ca"); // Returns "cat".

Kaçış Karakterleri #

Genel olarak

Normal ifadeye özgü karakterleri (?+| vb.) gerçek anlamlarında kullanmak için kaçmaları gerekir. Genel normal ifadede bu, ters eğik çizgi \ ile yapılır. Ancak Java Dizelerinde özel bir anlamı olduğu için çift ters eğik çizgi \\ kullanmanız gerekir.

Bu iki örnek çalışmayacaktır:

"???".replaceAll ("?", "!"); //java.util.regex.PatternSyntaxException
"???".replaceAll ("\?", "!"); //Invalid escape sequence

Bu örnek işe yarıyor

"???".replaceAll ("\\?", "!"); //"!!!"

Boru İle Sınırlandırılmış Bir Diziyi Bölme

Bu, beklenen sonucu döndürmez:

"a|b".split ("|"); // [a, |, b]

Bu, beklenen sonucu döndürür:

"a|b".split ("\\|"); // [a, b]

Ters eğik çizgiden kaçan \

Bu bir hata verecektir:

"\\".matches("\\"); // PatternSyntaxException
"\\".matches("\\\"); // Syntax Error

Bu çalışıyor:

"\\".matches("\\\\"); // true

Belirli bir dizeyle eşleşmiyor #

Belirli bir dizeyi içermeyen bir şeyi eşleştirmek için negatif bakış kullanılabilir:

Normal ifade sözdizimi: (?!string-to-not-match)

Örnek:

//not matching "popcorn"
String regexString = "^(?!popcorn).*$";
System.out.println("[popcorn] " + ("popcorn".matches(regexString) ? "matched!" : "nope!"));
System.out.println("[unicorn] " + ("unicorn".matches(regexString) ? "matched!" : "nope!"));

Çıktı:

[popcorn] nope!
[unicorn] matched!

Normal ifade değişmeziyle eşleştirme. #

Normal ifade sözdiziminin parçası olan karakterleri eşleştirmeniz gerekiyorsa, kalıbın tamamını veya bir kısmını normal ifade değişmezi olarak işaretleyebilirsiniz.

`\Q’, normal ifade değişmezinin başlangıcını işaretler. “\E”, normal ifade değişmezinin sonunu belirtir.

// the following throws a PatternSyntaxException because of the un-closed bracket
"[123".matches("[123");

// wrapping the bracket in \Q and \E allows the pattern to match as you would expect.
"[123".matches("\\Q[\\E123"); // returns true

“\Q” ve “\E” kaçış dizilerini hatırlamak zorunda kalmadan bunu yapmanın daha kolay bir yolu, “Pattern.quote()” kullanmaktır.

"[123".matches(Pattern.quote("[") + "123"); // returns true

Ters eğik çizgi eşleştirme #

Normal ifadenizde ters eğik çizgiyle eşleştirmek istiyorsanız, ondan kaçmanız gerekir.

Ters eğik çizgi, normal ifadelerde bir kaçış karakteridir. Normal bir ifadede tek bir ters eğik çizgiye atıfta bulunmak için ‘\\’ kullanabilirsiniz.

Ancak ters eğik çizgi, Java değişmez dizelerinde ayrıca bir kaçış karakteridir. Bir dize değişmezinden normal bir ifade yapmak için, it ters eğik çizgilerinin her birinden kaçmanız gerekir. Bir dize değişmezinde ‘\\\\’, ‘\\’ ile normal bir ifade oluşturmak için kullanılabilir ve bu da ‘\’ ile eşleşebilir.

Örneğin, “C:\dir\myfile.txt” gibi eşleşen dizeleri düşünün. Normal bir ([A-Za-z]):\\(.*) ifadesi eşleşecek ve sürücü harfini bir yakalama grubu olarak sağlayacaktır. Çift ters eğik çizgiye dikkat edin.

Bu kalıbı bir Java dize değişmezinde ifade etmek için, normal ifadedeki ters eğik çizgilerin her birinin kaçılması gerekir.

    String path = "C:\\dir\\myfile.txt";
    System.out.println( "Local path: " + path ); // "C:\dir\myfile.txt"
    
    String regex = "([A-Za-z]):\\\\.*"; // Four to match one
    System.out.println("Regex:      " + regex ); // "([A-Za-z]):\\(.*)"
    
    Pattern pattern = Pattern.compile( regex );
    Matcher matcher = pattern.matcher( path );
    if ( matcher.matches()) {
        System.out.println( "This path is on drive " + matcher.group( 1 ) + ":.");
        // This path is on drive C:.
    }

İki ters eğik çizgiyi eşleştirmek istiyorsanız, normal ifadede dördü temsil etmek, ikiyle eşleştirmek için kendinizi bir değişmez dizede sekizi kullanırken bulacaksınız.

    String path = "\\\\myhost\\share\\myfile.txt";
    System.out.println( "UNC path: " + path ); // \\myhost\share\myfile.txt"
    
    String regex = "\\\\\\\\(.+?)\\\\(.*)"; // Eight to match two
    System.out.println("Regex:    " + regex ); // \\\\(.+?)\\(.*) 
    
    Pattern pattern = Pattern.compile( regex );
    Matcher matcher = pattern.matcher( path );
    
    if ( matcher.matches()) {
        System.out.println( "This path is on host '" + matcher.group( 1 ) + "'.");
        // This path is on host 'myhost'.
    }
