Dizeler (java.lang.String), programınızda depolanan metin parçalarıdır. Dizeler Java’da bir [ilkel veri türü] değildir, ancak bunlar Java programlarında çok yaygındır.

Java’da Dizeler değişmezdir, yani değiştirilemezler. (Değişmezliğin daha kapsamlı bir açıklaması için buraya tıklayın.)

Java dizeleri değişmez olduğundan, bir “Dize"yi işleyen tüm yöntemler yeni bir “Dize” nesnesi döndürür. Orijinal ‘Dize’yi değiştirmezler. Bu, C ve C++ programcılarının hedef ‘String’ nesnesini mutasyona uğratmayı umduğu alt dizgi ve değiştirme yöntemlerini içerir.

Birden fazlasını birleştirmek istiyorsanız “String” yerine bir StringBuilder kullanın. değerleri derleme zamanında belirlenemeyen iki ‘String’ nesnesi. Bu teknik, yeni “String” nesneleri oluşturmaktan ve “StringBuilder” değişken olduğundan bunları birleştirmekten daha performanslıdır.

StringBuffer ‘String’ nesnelerini birleştirmek için de kullanılabilir. Ancak bu sınıf, iş parçacığı için güvenli olacak şekilde tasarlandığı ve her işlemden önce bir muteks aldığı için daha az performanslıdır. Dizeleri birleştirirken iş parçacığı güvenliğine neredeyse hiç ihtiyaç duymadığınız için, StringBuilder kullanmak en iyisidir.

Bir dize birleştirmeyi tek bir ifade olarak ifade edebiliyorsanız, + operatörünü kullanmak daha iyidir. Java derleyicisi, “+” birleştirmelerini içeren bir ifadeyi “String.concat(…)” veya “StringBuilder” kullanarak verimli bir işlem dizisine dönüştürür. ‘StringBuilder’ kullanma tavsiyesi, yalnızca birleştirme birden çok ifade içerdiğinde geçerlidir.

Hassas bilgileri dizelerde saklamayın. Birisi çalışan uygulamanızın bellek dökümünü elde edebilirse, mevcut tüm ‘String’ nesnelerini bulabilir ve içeriklerini okuyabilir. Buna erişilemeyen ve çöp toplamayı bekleyen ‘String’ nesneleri de dahildir. Bu bir endişeyse, işiniz biter bitmez hassas dize verilerini silmeniz gerekir. Değişmez oldukları için bunu ‘String’ nesneleri ile yapamazsınız. Bu nedenle, hassas karakter verilerini tutmak için bir ‘char[]’ nesneleri kullanmanız ve işiniz bittiğinde bunları silmeniz (örneğin, ‘\000’’ karakterlerinin üzerine yazmanız) tavsiye edilir.

Tüm “Dize” örnekleri, dize değişmezlerine karşılık gelen örnekler bile öbek üzerinde oluşturulur. Dize değişmezleriyle ilgili özel olan şey, JVM’nin eşit olan (yani aynı karakterlerden oluşan) tüm değişmezlerin tek bir “String” nesnesi tarafından temsil edilmesini sağlamasıdır (bu davranış JLS’de belirtilir). Bu, JVM sınıfı yükleyiciler tarafından uygulanır. Bir sınıf yükleyici bir sınıf yüklediğinde, sınıf tanımında kullanılan dize değişmezlerini tarar, her birini gördüğünde, bu değişmez için dize havuzunda zaten bir kayıt olup olmadığını kontrol eder (literal’i anahtar olarak kullanarak) . Hazır bilgi için zaten bir giriş varsa, o hazır bilgi için çift olarak saklanan bir ‘String’ örneğine yapılan başvuru kullanılır. Aksi takdirde, yeni bir String örneği oluşturulur ve örneğe bir referans, string havuzundaki değişmez bilgi (anahtar olarak kullanılır) için saklanır. (Ayrıca bkz. string interning).

Dahili Dizelerle Karşılaştırma #

Java Dil Belirtimi (JLS 3.10.6) şunları belirtir:

“Ayrıca, bir dize değişmezi her zaman String sınıfının aynı örneğine atıfta bulunur. Bunun nedeni, dize değişmezlerinin - veya daha genel olarak, sabit ifadelerin değerleri olan dizelerin - benzersiz örnekleri paylaşmak için içerilmiş olmasıdır. String.intern yöntemini kullanarak.”

Bu, == kullanarak referansları iki literal dizesine karşılaştırmanın güvenli olduğu anlamına gelir. Ayrıca, ‘String.intern()’ yöntemi kullanılarak üretilmiş ‘String’ nesnelerine yapılan başvurular için de aynısı geçerlidir.

Örneğin:

String strObj = new String("Hello!");
String str = "Hello!";

// The two string references point two strings that are equal
if (strObj.equals(str)) {
    System.out.println("The strings are equal");
}

// The two string references do not point to the same object
if (strObj != str) {
    System.out.println("The strings are not the same object");
}

// If we intern a string that is equal to a given literal, the result is
// a string that has the same reference as the literal.
String internedStr = strObj.intern();

if (internedStr == str) {
    System.out.println("The interned string and the literal are the same object");
}

Perde arkasında, interning mekanizması, hala erişilebilir olan tüm interne edilmiş dizeleri içeren bir hash tablosunu korur. Bir String’de ‘intern()’ öğesini çağırdığınızda, yöntem karma tablosundaki nesneyi arar:

  • Dize bulunursa, bu değer interned dize olarak döndürülür.
  • Aksi takdirde, karma tablosuna dizenin bir kopyası eklenir ve bu dize, interned dize olarak döndürülür.

Bir String içindeki karakterlerin büyük/küçük harf durumunu değiştirme #

String türü, dizeleri büyük harf ve küçük harf arasında dönüştürmek için iki yöntem sağlar:

  • toUpperCase tüm karakterleri büyük harfe dönüştürmek için
  • toLowerCase tüm karakterleri küçük harfe dönüştürmek için

Bu yöntemlerin her ikisi de dönüştürülmüş dizeleri yeni “Dize” örnekleri olarak döndürür: “Dize” Java’da değişmez olduğundan orijinal “Dize” nesneleri değiştirilmez. Değişmezlik hakkında daha fazla bilgi için buna bakın: Java’da Dizelerin Değişmezliği

String string = "This is a Random String";
String upper = string.toUpperCase();
String lower = string.toLowerCase();

System.out.println(string);   // prints "This is a Random String"
System.out.println(lower);    // prints "this is a random string"
System.out.println(upper);    // prints "THIS IS A RANDOM STRING"

Rakamlar ve noktalama işaretleri gibi alfabetik olmayan karakterler bu yöntemlerden etkilenmez. Bu yöntemlerin belirli koşullar altında belirli Unicode karakterleriyle de hatalı şekilde ilgilenebileceğini unutmayın.

Not: Bu yöntemler yerel ayara duyarlıdır ve yerel ayardan bağımsız olarak yorumlanması amaçlanan dizelerde kullanılırsa beklenmedik sonuçlar verebilir. Örnekler, programlama dili tanımlayıcıları, protokol anahtarları ve ‘HTML’ etiketleridir.

Örneğin, Türkçe bir yerel ayarda "TITLE".toLowerCase(), “title” değerini döndürür, burada ı (\u0131), LATIN KÜÇÜK HARF I karakteridir. Yerel ayara duyarlı olmayan dizeler için doğru sonuçları elde etmek için, ilgili büyük/küçük harf dönüştürme yöntemine bir parametre olarak “Locale.ROOT"u iletin (ör. “toLowerCase(Locale.ROOT)” veya “toUpperCase(Locale.ROOT)”).

‘Locale.ENGLISH’ kullanmak da çoğu durumda doğru olsa da, dil değişmez yolu ‘Locale.ROOT’ şeklindedir.

Özel büyük/küçük harf kullanımı gerektiren Unicode karakterlerinin ayrıntılı bir listesi Unicode Consortium web sitesinde bulunabilir.

ASCII dizesindeki belirli bir karakterin büyük/küçük harf değişimi:

Bir ASCII dizesinin belirli bir karakterinin büyük/küçük harf durumunu değiştirmek için aşağıdaki algoritma kullanılabilir:

Adımlar:

  1. Bir dize bildirin.
  2. Dizeyi girin.
  3. Dizeyi bir karakter dizisine dönüştürün.
  4. Aranacak karakteri girin.
  5. Karakter dizisinde karakteri arayın.
  6. Bulunursa, karakterin küçük mü yoksa büyük mü olduğunu kontrol edin.
    • If Uppercase, add 32 to the ASCII code of the character.
    • If Lowercase, subtract 32 from the ASCII code of the character.
  7. Karakter dizisinden orijinal karakteri değiştirin.
  8. Karakter dizisini tekrar dizgeye dönüştürün.

Voila, karakterin Vakası değişti.

Algoritma için kod örneği:

Scanner scanner = new Scanner(System.in);
System.out.println("Enter the String");
String s = scanner.next();
char[] a = s.toCharArray();
System.out.println("Enter the character you are looking for");
System.out.println(s);
String c = scanner.next();
char d = c.charAt(0);

for (int i = 0; i <= s.length(); i++) {
    if (a[i] == d) {
        if (d >= 'a' && d <= 'z') {
            d -= 32;
        } else if (d >= 'A' && d <= 'Z') {
            d += 32;
        }
        a[i] = d;
        break;
    }
}
s = String.valueOf(a);
System.out.println(s);

Dizeleri Bölme #

Bir String belirli bir sınırlayıcı karaktere veya bir Normal İfade üzerine bölebilirsiniz, aşağıdaki imzaya sahip String.split() yöntemini kullanabilirsiniz:

public String[] split(String regex)

Sınırlayıcı karakterin veya normal ifadenin ortaya çıkan Dize Dizisinden kaldırıldığını unutmayın.

Sınırlayıcı karakter kullanan örnek:

String lineFromCsvFile = "Mickey;Bolton;12345;121216";
String[] dataCells = lineFromCsvFile.split(";");
// Result is dataCells = { "Mickey", "Bolton", "12345", "121216"};

Normal ifade kullanan örnek:

String lineFromInput = "What    do you need    from me?";
String[] words = lineFromInput.split("\\s+"); // one or more space chars
// Result is words = {"What", "do", "you", "need", "from", "me?"};

Hatta bir ‘String’ değişmezini doğrudan bölebilirsiniz:

String[] firstNames = "Mickey, Frank, Alicia, Tom".split(", ");
// Result is firstNames = {"Mickey", "Frank", "Alicia", "Tom"};

Uyarı: Parametrenin her zaman normal bir ifade olarak ele alındığını unutmayın.

"aaa.bbb".split("."); // This returns an empty array

Önceki örnekte ., herhangi bir karakterle eşleşen normal ifade joker karakteri olarak ele alınır ve her karakter bir sınırlayıcı olduğundan, sonuç boş bir dizidir.

Normal ifade meta karakteri olan bir sınırlayıcıya dayalı bölme

Aşağıdaki karakterler normal ifadede özel olarak kabul edilir (diğer bir deyişle meta karakterler)

  < > - = ! ( ) [ ] { } \ ^ $ | ? * + .

Yukarıdaki sınırlayıcılardan birine dayalı olarak bir dizeyi bölmek için, “\” kullanarak onlardan kaçmanız veya “Pattern.quote()” kullanmanız gerekir:

  • Pattern.quote() kullanarak:

     String s = "a|b|c";
 String regex = Pattern.quote("|");
 String[] arr = s.split(regex);

  • Özel karakterlerden kaçış:

     String s = "a|b|c";
 String[] arr = s.split("\\|");

Split boş değerleri kaldırır

split(sınırlayıcı) varsayılan olarak sonuç dizisinden sondaki boş dizeleri kaldırır. Bu mekanizmayı kapatmak için “split(delimiter, limit)“nin aşırı yüklenmiş versiyonunu limiti negatif değere ayarlanmış şekilde kullanmamız gerekiyor.

String[] split = data.split("\\|", -1);

“split(regex)”, dahili olarak “split(regex, 0)” sonucunu döndürür.

Dize birleştirme ve StringBuilders #

Dize birleştirme, + operatörü kullanılarak gerçekleştirilebilir. Örneğin:

String s1 = "a";

Normalde bir derleyici uygulaması, başlık altında bir StringBuilder içeren yöntemleri kullanarak yukarıdaki birleştirme işlemini gerçekleştirir. Derlendiğinde, kod aşağıdakine benzer görünür:

StringBuilder sb = new StringBuilder("a");
String s = sb.append("b").append("c").toString();

“StringBuilder”, farklı türleri eklemek için, örneğin “Dize” yerine “int” eklemek için birkaç aşırı yüklenmiş yönteme sahiptir. Örneğin, bir uygulama şunları dönüştürebilir:

String s1 = "a";
String s2 = "b";    
String s = s1 + s2 + 2; // ab2

aşağıdakilere:

StringBuilder sb = new StringBuilder("a");
String s = sb.append("b").append(2).toString();

Yukarıdaki örnekler, kodda tek bir yerde etkin bir şekilde yapılan basit bir birleştirme işlemini göstermektedir. Birleştirme, “StringBuilder"ın tek bir örneğini içerir. Bazı durumlarda, bir döngüde olduğu gibi kümülatif bir şekilde bir birleştirme gerçekleştirilir:

String result = "";
for(int i = 0; i < array.length; i++) {
    result += extractElement(array[i]);
}
return result;

Bu gibi durumlarda, derleyici optimizasyonu genellikle uygulanmaz ve her yineleme yeni bir ‘StringBuilder’ nesnesi oluşturur. Bu, kodu tek bir StringBuilder kullanacak şekilde açıkça dönüştürerek optimize edilebilir:

StringBuilder result = new StringBuilder();
for(int i = 0; i < array.length; i++) {
    result.append(extractElement(array[i]));
}
return result.toString();

Bir “StringBuilder”, yalnızca 16 karakterlik boş bir alanla başlatılacaktır. Daha büyük dizeler oluşturacağınızı önceden biliyorsanız, dahili arabelleğin yeniden boyutlandırılmasına gerek kalmaması için önceden yeterli boyutta başlatmak faydalı olabilir:

import java.util.regex.Matcher;
import java.util.regex.Pattern;
 
String text = "One fish, two fish, red fish, blue fish";
System.out.println(countStringInString("fish", text)); // prints 4
System.out.println(countStringInString(",", text)); // prints 3
 

public static int countStringInString(String search, String text) {
    Pattern pattern = Pattern.compile(search);
    Matcher matcher = pattern.matcher(text);
    
    int stringOccurrences = 0;
    while (matcher.find()) {
      stringOccurrences++;
    }
    return stringOccurrences;
}
