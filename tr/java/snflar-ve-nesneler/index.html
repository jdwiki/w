Nesnelerin durumları ve davranışları vardır. Örnek: Bir köpeğin durumları - rengi, adı, cinsi ve davranışları - kuyruğunu sallama, havlama, yeme. Bir nesne, bir sınıfın bir örneğidir.

Sınıf – Bir sınıf, türünün nesnesinin desteklediği davranışı/durumu tanımlayan bir şablon/plan olarak tanımlanabilir.

Aşırı Yükleme Yöntemleri #

Bazen farklı girdi türleri için aynı işlevsellik yazılmalıdır. O zaman, aynı yöntem adı farklı bir parametre kümesiyle kullanılabilir. Her farklı parametre kümesi, bir yöntem imzası olarak bilinir. Örnekte görüldüğü gibi, tek bir yöntemin birden fazla imzası olabilir.

public class Displayer {

    public void displayName(String firstName) {
        System.out.println("Name is: " + firstName);
    }

    public void displayName(String firstName, String lastName) {
        System.out.println("Name is: " + firstName + " " + lastName);
    }

    public static void main(String[] args) {
        Displayer displayer = new Displayer();
        displayer.displayName("Ram");          //prints "Name is: Ram"
        displayer.displayName("Jon", "Skeet"); //prints "Name is: Jon Skeet"
    }
}

Avantajı, aynı işlevselliğin iki farklı sayıda girişle çağrılmasıdır. Geçirdiğimiz girişe göre metodu çağırırken, (Bu durumda ya bir string değeri ya da iki string değeri) karşılık gelen metot çalıştırılır.

Yöntemler aşırı yüklenebilir:

  1. Geçilen parametre sayısına göre.

    Example: method(String s) and method(String s1, String s2).

  2. parametrelerin sırasına göre.

    Example: method(int i, float f) and method(float f, int i)).

Not: Dönüş türü sadece değiştirilerek yöntemler aşırı yüklenemez (int method(),String method() ile aynı kabul edilir ve denenirse bir RuntimeException verir). Dönüş türünü değiştirirseniz, aşırı yükleme için parametreleri de değiştirmelisiniz.

Yöntem aşırı yükleme ve geçersiz kılmanın ne olduğunu açıklama. #

Nesne Üyesi ve Statik Üye #

Bu sınıfla:

class ObjectMemberVsStaticMember {

    static int staticCounter = 0;
    int memberCounter = 0;

    void increment() {
        staticCounter ++;
        memberCounter++;
    }
}

aşağıdaki kod parçacığı:

final ObjectMemberVsStaticMember o1 = new ObjectMemberVsStaticMember();
final ObjectMemberVsStaticMember o2 = new ObjectMemberVsStaticMember();

o1.increment();

o2.increment();
o2.increment();

System.out.println("o1 static counter " + o1.staticCounter);
System.out.println("o1 member counter " + o1.memberCounter);
System.out.println();

System.out.println("o2 static counter " + o2.staticCounter);
System.out.println("o2 member counter " + o2.memberCounter);
System.out.println();

System.out.println("ObjectMemberVsStaticMember.staticCounter = " + ObjectMemberVsStaticMember.staticCounter);

// the following line does not compile. You need an object
// to access its members
//System.out.println("ObjectMemberVsStaticMember.staticCounter = " + ObjectMemberVsStaticMember.memberCounter);

bu çıktıyı üretir:

o1 static counter 3
o1 member counter 1

o2 static counter 3
o2 member counter 2

ObjectMemberVsStaticMember.staticCounter = 3

Not: Nesnelerde değil, sınıflarda ‘statik’ üyeler çağırmalısınız. JVM için bir fark yaratmasa da, insan okuyucular bunu takdir edecektir.

‘static’ üyeler sınıfın bir parçasıdır ve sınıf başına yalnızca bir kez bulunur. Örneklerde statik olmayan üyeler bulunur, her örnek için bağımsız bir kopya vardır. Bu aynı zamanda üyelerine erişmek için o sınıfın bir nesnesine erişmeniz gerektiği anlamına gelir.

Temel Nesne Yapısı ve Kullanımı #

Nesneler kendi sınıflarında gelir, bu nedenle basit bir örnek bir araba olabilir (aşağıdaki ayrıntılı açıklamalar):

public class Car {
    
    //Variables describing the characteristics of an individual car, varies per  object
   private int milesPerGallon;
   private String name;
   private String color;
   public int numGallonsInTank; 
    
    public Car(){
        milesPerGallon = 0;
        name = "";
        color = "";
        numGallonsInTank = 0;
    }
    
    //this is where an individual object is created
    public Car(int mpg, int, gallonsInTank, String carName, String carColor){
        milesPerGallon = mpg;
        name = carName;
        color = carColor;
        numGallonsInTank = gallonsInTank;
    }

    //methods to make the object more usable

    //Cars need to drive
    public void drive(int distanceInMiles){
        //get miles left in car
        int miles = numGallonsInTank * milesPerGallon;
        
        //check that car has enough gas to drive distanceInMiles
        if (miles <= distanceInMiles){
            numGallonsInTank = numGallonsInTank - (distanceInMiles / milesPerGallon)
            System.out.println("Drove " + numGallonsInTank + " miles!");
        } else {
            System.out.println("Could not drive!");
        }
    }

    public void paintCar(String newColor){
        color = newColor;
    }
        //set new Miles Per Gallon
    public void setMPG(int newMPG){
        milesPerGallon = newMPG;
    }

       //set new number of Gallon In Tank
    public void setGallonsInTank(int numGallons){
        numGallonsInTank = numGallons;
    }
    
    public void nameCar(String newName){
        name = newName;
    }

    //Get the Car color
    public String getColor(){
        return color;
    }

    //Get the Car name
    public String getName(){
        return name;
    }

    //Get the number of Gallons
    public String getGallons(){
        return numGallonsInTank;
    }
    
}

Nesneler, sınıflarının örnekleridir. Bu nedenle, bir nesne yaratmanızın yolu, ana sınıfınızdaki iki yoldan biriyle Car sınıfını çağırmak olacaktır (Java’da ana yöntem veya Android’de onCreate).

Seçenek 1

`Car newCar = new Car(30, 10, "Ferrari", "Red");

Seçenek 1, nesnenin oluşturulmasından sonra programa Araba hakkında her şeyi anlatacağınız yerdir. Arabanın herhangi bir özelliğini değiştirmek, “repaintCar” yöntemi gibi yöntemlerden birinin çağrılmasını gerektirir. Örnek:

 newCar.repaintCar("Blue");

Not: Yönteme doğru veri türünü ilettiğinizden emin olun. Yukarıdaki örnekte, repaintCar yöntemine veri türü doğru olduğu sürece` bir değişken de iletebilirsiniz.

Bu, bir nesnenin özelliklerini değiştirmenin bir örneğiydi, bir nesnenin özelliklerini almak için Car sınıfından bir dönüş değeri olan bir yöntemin kullanılması gerekirdi (yani “void” olmayan bir yöntem). Örnek:

String myCarName = newCar.getName();  //returns string "Ferrari"

Seçenek 1, oluşturma sırasında nesnenin tüm verilerine sahip olduğunuzda en iyi seçenektir.

Seçenek 2

`Car newCar = new Car();

Seçenek 2 aynı etkiyi alır ancak bir nesneyi doğru bir şekilde oluşturmak için daha fazla çalışma gerektirir. Araba sınıfındaki bu Yapıcıyı hatırlamak istiyorum:

public void Car(){
        milesPerGallon = 0;
        name = "";
        color = "";
        numGallonsInTank = 0;
    }

Bunu oluşturmak için nesneye herhangi bir parametre iletmeniz gerekmediğine dikkat edin. Bu, nesnenin tüm yönlerine sahip olmadığınızda ancak sahip olduğunuz parçaları kullanmanız gerektiğinde çok kullanışlıdır. Bu, nesnenin örnek değişkenlerinin her birine genel verileri ayarlar, böylece var olmayan bir veri parçasını çağırırsanız hiçbir hata oluşmaz.

Not: Nesnenin başlatmadığınız kısımlarını daha sonra ayarlamanız gerektiğini unutmayın. Örneğin,

Car myCar = new Car();
String color = Car.getColor(); //returns empty string

Bu, tüm verileriyle başlatılmayan nesneler arasında yaygın bir hatadır. Hatalardan kaçınıldı çünkü stand-in değişkenlerle (public Car(){} ) boş bir Car nesnesinin oluşturulmasına izin veren bir Oluşturucu var, ancak myCar’ın hiçbir parçası gerçekten özelleştirilmedi. Araba Nesnesi oluşturmanın doğru örneği:

Car myCar = new Car();
myCar.nameCar("Ferrari");
myCar.paintCar("Purple");
myCar.setGallonsInTank(10);
myCar.setMPG(30);

Ve bir hatırlatma olarak, ana sınıfınızdaki bir yöntemi çağırarak bir nesnenin özelliklerini alın. Örnek:

String myCarName = myCar.getName(); //returns string "Ferrari"

Yapıcılar #

Yapıcılar, sınıftan sonra adlandırılan ve dönüş türü olmayan özel yöntemlerdir ve nesneleri oluşturmak için kullanılır. Yapıcılar, yöntemler gibi girdi parametrelerini alabilir. Yapıcılar nesneleri başlatmak için kullanılır. Soyut sınıfların yapıcıları da olabilir.

public class Hello{
    // constructor
    public Hello(String wordToPrint){
        printHello(wordToPrint);
    }
    public void printHello(String word){
        System.out.println(word);
    }
}
// instantiates the object during creating and prints out the content
// of wordToPrint

Yapıcıların yöntemlerden birkaç yönden farklı olduğunu anlamak önemlidir:

  1. Yapıcılar yalnızca “public”, “private” ve “protected” değiştiricilerini alabilir ve “abstract”, “final”, “static” veya “synchronize” olarak bildirilemezler.

  2. Yapıcıların bir dönüş türü yoktur.

  3. Yapıcılar, sınıf adıyla aynı şekilde adlandırılmalıdır ZORUNLU. “Merhaba” örneğinde, “Merhaba” nesnesinin yapıcı adı, sınıf adıyla aynıdır.

  4. this anahtar sözcüğü, yapıcılar içinde ek bir kullanıma sahiptir. ’this.method(…)’ geçerli örnekte bir yöntemi çağırırken, ’this(…)’ geçerli sınıftaki farklı imzalara sahip başka bir kurucuya atıfta bulunur.

Yapıcılar ayrıca ‘super’ anahtar sözcüğü kullanılarak kalıtım yoluyla da çağrılabilir.

public class SuperManClass{

    public SuperManClass(){
        // some implementation
    }
    
    // ... methods
}


public class BatmanClass extends SupermanClass{
    public BatmanClass(){
        super();
    }
    //... methods...
}

Statik bir başlatıcı kullanarak statik son alanları başlatma #

Tek bir ifadeden fazlasının kullanılmasını gerektiren bir “statik final” alanlarını başlatmak için, değeri atamak için bir “statik” başlatıcı kullanılabilir. Aşağıdaki örnek, değiştirilemez bir “Dize” kümesini başlatır:

public class MyClass {

    public static final Set<String> WORDS;
    
    static {
        Set<String> set = new HashSet<>();
        set.add("Hello");
        set.add("World");
        set.add("foo");
        set.add("bar");
        set.add("42");
        WORDS = Collections.unmodifiableSet(set);
    }
}
