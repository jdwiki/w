Sözdizimi #

  • genel sınıf MyClass, Comparable<MyClass> öğesini uygular
  • public class MyComparator, Comparator<SomeOtherClass> öğesini uygular
  • public int CompareTo(MyClass diğer)
  • public int karşılaştırması(SomeOtherClass o1, SomeOtherClass o2)

Bir “double"a bağlı olan bir “compareTo(..)” yöntemini uygularken aşağıdakileri yapmayın:

public int comareTo(MyClass other) {
    return (int)(doubleField - other.doubleField); //THIS IS BAD
}

(int) dökümünün neden olduğu kesme, yöntemin bazen pozitif veya negatif bir sayı yerine yanlış bir şekilde “0” döndürmesine neden olur ve bu nedenle karşılaştırma ve sıralama hatalarına yol açabilir.

Bunun yerine, en basit doğru uygulama Double.compare kullanmaktır, şu şekilde:

public int comareTo(MyClass other) {
    return Double.compare(doubleField,other.doubleField); //THIS IS GOOD
}

‘Karşılaştırılabilir‘nin jenerik olmayan bir sürümü, basitçe ‘Karşılaştırılabilir’, Java 1.2’den beri mevcuttur. Eski kodla arabirim oluşturmanın dışında, karşılaştırma sırasında yayınlama gerektirmediğinden, ‘Karşılaştırılabilir’ genel sürümünü uygulamak her zaman daha iyidir.

Aşağıdaki gibi, bir sınıfın kendisiyle karşılaştırılabilir olması çok standarttır:

public class A implements Comparable<A>

Bu paradigmadan kopmak mümkün olsa da, bunu yaparken dikkatli olun.

Bir ‘Karşılaştırıcı’, o sınıf ‘Karşılaştırılabilir’ uygularsa, sınıfın örneklerinde yine kullanılabilir. Bu durumda Karşılaştırıcı mantığı kullanılacaktır; ‘Karşılaştırılabilir’ uygulama tarafından belirtilen doğal sıralama göz ardı edilecektir.

Karşılaştırılabilir veya Karşılaştırıcı Kullanarak Listeyi Sıralama #

Bir Kişiyi ad ve soyadlarıyla temsil eden bir sınıf üzerinde çalıştığımızı varsayalım. Bunu yapmak için temel bir sınıf oluşturduk ve uygun ’equals’ ve ‘hashCode’ yöntemlerini uyguladık.

public class Person {

    private final String lastName; //invariant - nonnull
    private final String firstName; //invariant - nonnull

    public Person(String firstName, String lastName){
        this.firstName = firstName != null ? firstName : "";
        this.lastName = lastName != null ? lastName : "";
    }

    public String getFirstName() {
        return firstName;
    }

    public String getLastName() {
        return lastName;
    }

    public String toString() {
        return lastName + ", " + firstName;
    }

    @Override
    public boolean equals(Object o) {
        if (! (o instanceof Person)) return false;
        Person p = (Person)o;
        return firstName.equals(p.firstName) && lastName.equals(p.lastName);
    }

    @Override
    public int hashCode() {
        return Objects.hash(firstName, lastName);
    }
}

Şimdi, aşağıdaki senaryoda olduğu gibi, ‘Person’ nesnelerinin bir listesini adlarına göre sıralamak istiyoruz:

public static void main(String[] args) {
    List<Person> people = Arrays.asList(new Person("John", "Doe"),
                                     new Person("Bob", "Dole"),
                                     new Person("Ronald", "McDonald"),
                                     new Person("Alice", "McDonald"),
                                     new Person("Jill", "Doe"));
    Collections.sort(people); //This currently won't work.
}

Ne yazık ki, işaretlendiği gibi, yukarıdakiler şu anda derlenmeyecek. Collections.sort(..) yalnızca o listedeki öğeler karşılaştırılabilirse veya özel bir karşılaştırma yöntemi verilmişse bir listenin nasıl sıralanacağını bilir.

Şu listeyi sıralamanız istendiyse: ‘1,3,5,4,2’, cevabın ‘1,2,3,4,5’ olduğunu söylemekte sorun yaşamazsınız. Bunun nedeni, Tamsayıların (hem Java’da hem de matematiksel olarak) doğal bir sıralamaya, standart, varsayılan bir karşılaştırma temel sıralamasına sahip olmasıdır. Person sınıfımıza doğal bir sıralama vermek için, ‘compareTo(Person p):’ yönteminin uygulanmasını gerektiren ‘Comparable‘u uygularız:

public class Person implements Comparable<Person> {

    private final String lastName; //invariant - nonnull
    private final String firstName; //invariant - nonnull

    public Person(String firstName, String lastName) {
        this.firstName = firstName != null ? firstName : "";
        this.lastName = lastName != null ? lastName : "";
    }

    public String getFirstName() {
        return firstName;
    }

    public String getLastName() {
        return lastName;
    }

    public String toString() {
        return lastName + ", " + firstName;
    }

    @Override
    public boolean equals(Object o) {
        if (! (o instanceof Person)) return false;
        Person p = (Person)o;
        return firstName.equals(p.firstName) && lastName.equals(p.lastName);
    }

    @Override
    public int hashCode() {
        return Objects.hash(firstName, lastName);
    }

    @Override
    public int compareTo(Person other) {
        // If this' lastName and other's lastName are not comparably equivalent,
        // Compare this to other by comparing their last names.
        // Otherwise, compare this to other by comparing their first names
        int lastNameCompare = lastName.compareTo(other.lastName);
        if (lastNameCompare != 0) {
            return lastNameCompare;
        } else {
            return firstName.compareTo(other.firstName);
        }
    }
}

Şimdi, verilen ana yöntem doğru bir şekilde çalışacaktır.

public static void main(String[] args) {
    List<Person> people = Arrays.asList(new Person("John", "Doe"),
                                     new Person("Bob", "Dole"),
                                     new Person("Ronald", "McDonald"),
                                     new Person("Alice", "McDonald"),
                                     new Person("Jill", "Doe"));
    Collections.sort(people); //Now functions correctly

    //people is now sorted by last name, then first name:
    // --> Jill Doe, John Doe, Bob Dole, Alice McDonald, Ronald McDonald
}

Bununla birlikte, ‘Person’ sınıfını istemiyorsanız veya değiştiremiyorsanız, herhangi iki ‘Person’ nesnesinin karşılaştırmasını yöneten özel bir ‘Karşılaştırıcı’ sağlayabilirsiniz. Aşağıdaki listeyi sıralamanız istense: ‘daire, kare, dikdörtgen, üçgen, altıgen’ yapamazdınız, ancak bu listeyi köşe sayısına göre sıralamanız istense, yapabilirsiniz. Aynen öyle, bir karşılaştırıcı sağlamak, Java’ya normalde karşılaştırılabilir olmayan iki nesneyi nasıl karşılaştıracağını bildirir.

public class PersonComparator implements Comparator<Person> {

    public int compare(Person p1, Person p2) {
        // If p1's lastName and p2's lastName are not comparably equivalent,
        // Compare p1 to p2 by comparing their last names.
        // Otherwise, compare p1 to p2 by comparing their first names
        if (p1.getLastName().compareTo(p2.getLastName()) != 0) {
            return p1.getLastName().compareTo(p2.getLastName());
        } else {
            return p1.getFirstName().compareTo(p2.getFirstName());
        }
    }
}

//Assume the first version of Person (that does not implement Comparable) is used here
public static void main(String[] args) {
    List<Person> people = Arrays.asList(new Person("John", "Doe"),
                                     new Person("Bob", "Dole"),
                                     new Person("Ronald", "McDonald"),
                                     new Person("Alice", "McDonald"),
                                     new Person("Jill", "Doe"));
    Collections.sort(people); //Illegal, Person doesn't implement Comparable.
    Collections.sort(people, new PersonComparator()); //Legal

    //people is now sorted by last name, then first name:
    // --> Jill Doe, John Doe, Bob Dole, Alice McDonald, Ronald McDonald
}

Karşılaştırıcılar, anonim bir iç sınıf olarak da oluşturulabilir/kullanılabilir

//Assume the first version of Person (that does not implement Comparable) is used here
public static void main(String[] args) {

Karşılaştırma ve Karşılaştırma Yöntemleri #

‘Karşılaştırılabilir’ arabirimi bir yöntem gerektirir:

public interface Comparable<T> {

    public int compareTo(T other);

}

Ve ‘Karşılaştırıcı’ arabirimi bir yöntem gerektirir:

public interface Comparator<T> {

    public int compare(T t1, T t2);

}

Bu iki yöntem, küçük bir farkla temelde aynı şeyi yapar: “compareTo”, “bu"yu “diğer” ile karşılaştırırken, “karşılaştır”, “bu” ile hiç ilgilenmeden “t1” ile “t2"yi karşılaştırır.

Bu farkın yanı sıra, iki yöntemin de benzer gereksinimleri vardır. Spesifik olarak (karşılaştırma için), [Bu nesneyi sipariş için belirtilen nesneyle karşılaştırır. Bu nesne belirtilen nesneden küçük, ona eşit veya daha büyük olduğundan negatif bir tamsayı, sıfır veya pozitif bir tamsayı döndürür.1 Bu nedenle, “a” ve “b"nin karşılaştırılması için:

  • “a < b”, “a.compareTo(b)” ve “compare(a,b)” negatif bir tamsayı döndürmeli ve “b.compareTo(a)” ve “compare(b,a)” olmalıdır pozitif bir tamsayı döndür
  • “a > b” ise, “a.compareTo(b)” ve “compare(a,b)” pozitif bir tamsayı döndürmeli ve “b.compareTo(a)” ve “compare(b,a)” olmalıdır negatif bir tamsayı döndür
  • Karşılaştırma için “a”, “b"ye eşitse, tüm karşılaştırmalar “0” döndürmelidir.

Doğal (karşılaştırılabilir) ve açık (karşılaştırıcı) sıralama #

İki “Collections.sort()” yöntemi vardır:

  • “T"nin uygulanması gereken bir parametre olarak “List” alan bir Karşılaştırılabilir ve belirleyen ‘compareTo()’ yöntemini geçersiz kılar. sıralama düzeni.
  • Argüman olarak bir Liste ve Karşılaştırıcı alan, burada Karşılaştırıcı, sıralama düzenini belirler.

İlk olarak, Karşılaştırılabilir’i uygulayan bir Kişi sınıfı:

public class Person implements Comparable<Person> {         
    private String name;  
    private int age;

    public String getName() {
        return name;
    }
    public void setName(String name) {
        this.name = name;
    }
    public int getAge() {
        return age;
    }
    public void setAge(int age) {
        this.age = age;
    }         

    @Override
    public int compareTo(Person o) {
        return this.getAge() - o.getAge();
    }
    @Override
    public String toString() {
        return this.getAge()+"-"+this.getName();
    }

}

‘compareTo()’ yöntemi geçersiz kılma tarafından tanımlanan öğelerinin doğal sıralamasına göre bir Listeyi sıralamak için yukarıdaki sınıfı nasıl kullanacağınız aşağıda açıklanmıştır:

//-- usage
List<Person> pList = new ArrayList<Person>();
            Person p = new Person();
            p.setName("A");
            p.setAge(10);
            pList.add(p);
            p = new Person();
            p.setName("Z");
            p.setAge(20);
            pList.add(p);
            p = new Person();
            p.setName("D");
            p.setAge(30);
            pList.add(p);
            
            //-- natural sorting i.e comes with object implementation, by age
            Collections.sort(pList);

            System.out.println(pList);

Karşılaştırılabilir’i uygulamayan bir Listeyi sıralamak veya bu durumda bir Listeyi doğal sıralama dışında bir sıraya göre sıralamak için anonim bir satır içi Karşılaştırıcıyı nasıl kullanacağınız aşağıda açıklanmıştır:

            //-- explicit sorting, define sort on another property here goes with name
            Collections.sort(pList, new Comparator<Person>() {

                @Override
                public int compare(Person o1, Person o2) {
                    return o1.getName().compareTo(o2.getName());
                }
            });            
            System.out.println(pList);

Karşılaştırma yöntemini kullanarak bir Karşılaştırıcı oluşturma #

Comparator.comparing(Person::getName)

Bu, bu kişi adını karşılaştırma kaynağı olarak kullanan ‘Kişi’ sınıfı için bir karşılaştırıcı oluşturur. Ayrıca long, int ve double’ı karşılaştırmak için yöntem sürümünü kullanmak mümkündür. Örneğin:

Comparator.comparingInt(Person::getAge)

Ters sıra

Ters sıralama uygulayan bir karşılaştırıcı oluşturmak için “reversed()” yöntemini kullanın:

Comparator.comparing(Person::getName).reversed()

Karşılaştırıcılar zinciri

Comparator.comparing(Person::getLastName).thenComparing(Person::getFirstName)

Bu, ilk önce soyadıyla, ardından adla karşılaştıran bir karşılaştırıcı oluşturacaktır. İstediğiniz kadar karşılaştırıcı zincirleyebilirsiniz.
