Akışları Kullanma #

Bir Akış, üzerinde sıralı ve paralel toplama işlemlerinin gerçekleştirilebildiği bir dizi öğedir. Herhangi bir “Akış”, potansiyel olarak içinden akan sınırsız miktarda veriye sahip olabilir. Sonuç olarak, bir “Akım"dan alınan veriler, veriler üzerinde toplu işlem gerçekleştirmenin aksine, geldiği anda tek tek işlenir. lambda ifadeleri ile birleştirildiğinde, işlevsel bir yaklaşım kullanarak veri dizileri üzerinde işlemler gerçekleştirmenin kısa bir yolunu sağlarlar.

           .sorted()

Akışları Kapatma #

Bir “Akış"ın genellikle kapatılması gerekmediğini unutmayın. Yalnızca IO kanallarında çalışan akışları kapatmak gerekir. Çoğu “Akış” türü kaynaklar üzerinde çalışmaz ve bu nedenle kapatma gerektirmez.

“Akış” arabirimi, AutoCloseable öğesini genişletir. Akışlar, close yöntemi çağrılarak veya kaynakla dene deyimleri kullanılarak kapatılabilir.

Bir “Akış"ın kapatılması gereken örnek bir kullanım örneği, bir dosyadan bir “Akış” satırı oluşturduğunuzdadır:

try (Stream<String> lines = Files.lines(Paths.get("somePath"))) {
    lines.forEach(System.out::println);
}

‘Akış’ arabirimi, akış kapatıldığında çağrılacak ‘Runnable’ işleyicilerini kaydetmenize izin veren ‘Stream.onClose()’ yöntemini de bildirir. Örnek bir kullanım durumu, bir akış oluşturan kodun bazı temizlemeleri gerçekleştirmek için ne zaman tüketildiğini bilmesi gerektiğidir.

public Stream<String>streamAndDelete(Path path) throws IOException {
    return Files.lines(path).onClose(() -> someClass.deletePath(path));
}

Çalıştırma işleyicisi, yalnızca bir try-with-resources deyimi tarafından açık veya örtük olarak ‘close()’ yöntemi çağrılırsa yürütülür.

Sipariş İşleme #

Bir “Akış” nesnesinin işlenmesi sıralı veya [paralel](https://www.wikiod.com/tr/java/canl-yaynlar#Paralel Akış) olabilir.

Bir sıralı modda, öğeler “Akış” kaynağının sırasına göre işlenir. “Akış” sıralanırsa (örneğin bir SortedMap uygulaması veya bir Liste gibi), işlemenin kaynağın sıralamasıyla eşleşmesi garanti edilir. Ancak diğer durumlarda, sıralamaya bağlı kalmamaya özen gösterilmelidir (bkz: Java HashMap keySet() yineleme sırası tutarlı mı?).

Örnek:

List<Integer> integerList = Arrays.asList(0, 1, 2, 3, 42); 

// sequential 
long howManyOddNumbers = integerList.stream()
                                    .filter(e -> (e % 2) == 1)
                                    .count(); 

System.out.println(howManyOddNumbers); // Output: 2

Ideone’da Canlı

Paralel modu, birden çok çekirdekte birden çok iş parçacığının kullanılmasına izin verir, ancak öğelerin işlenme sırasının garantisi yoktur.

Ardışık bir “Akış” üzerinde birden çok yöntem çağrılırsa, her yöntemin çağrılması gerekmez. Örneğin, bir “Akış” filtrelenirse ve öğe sayısı bire düşürülürse, “sıralama” gibi bir yönteme yapılan sonraki çağrı gerçekleşmez. Bu, paralel bir “Akış” ile mümkün olmayan bir optimizasyon olan sıralı “Akış"ın performansını artırabilir.

Örnek:

// parallel
long howManyOddNumbersParallel = integerList.parallelStream()
                                            .filter(e -> (e % 2) == 1)
                                            .count();

System.out.println(howManyOddNumbersParallel); // Output: 2

Sonsuz Akışlar #

Bitmeyen bir ‘Akış’ oluşturmak mümkündür. Sonsuz bir “Akış” üzerinde bir uçbirim yönteminin çağrılması, “Akış"ın sonsuz bir döngüye girmesine neden olur. Java’nın işlediği “Akış” terimlerinin sayısını sınırlamak için bir “Akış"ın limit yöntemi kullanılabilir.

Bu örnek, 1 ile başlayan tüm doğal sayılardan oluşan bir “Akım” oluşturur. “Akış"ın ardışık her terimi bir öncekinden bir büyüktür. Bu “Akış"ın limit yöntemi çağrılarak, “Akış"ın yalnızca ilk beş terimi dikkate alınır ve yazdırılır.

// Generate infinite stream - 1, 2, 3, 4, 5, 6, 7, ...
IntStream naturalNumbers = IntStream.iterate(1, x -> x + 1);

// Print out only the first 5 terms
naturalNumbers.limit(5).forEach(System.out::println);

toList() ve toSet() ile toplayın #

Bir Stream öğesinden öğeler, Stream.collect işlemi kullanılarak kolayca bir kapsayıcıda toplanabilir:

System.out.println(Arrays
    .asList("apple", "banana", "pear", "kiwi", "orange")
    .stream()
    .filter(s -> s.contains("a"))
    .collect(Collectors.toList())
);
// prints: [apple, banana, pear, orange]

Set gibi diğer koleksiyon örnekleri, diğer yerleşik Collectors yöntemleri kullanılarak yapılabilir. Örneğin, Collectors.toSet(), bir “Akış"ın öğelerini bir “Set” içinde toplar.

Liste veya Set uygulaması üzerinde açık kontrol

Collectors#toList() ve Collectors#toSet() belgelerine göre, öğesinin türü, değişebilirliği, serileştirilebilirliği veya iş parçacığı güvenliği konusunda hiçbir garanti yoktur. List veya Set döndürüldü.

Döndürülecek uygulama üzerinde açık kontrol için, verilen tedarikçinin yeni ve boş bir koleksiyon döndürdüğü yerde Collectors#toCollection(Supplier) kullanılabilir.

// syntax with method reference
System.out.println(strings
        .stream()
        .filter(s -> s != null && s.length() <= 3)
        .collect(Collectors.toCollection(ArrayList::new))
);

// syntax with lambda
System.out.println(strings
        .stream()
        .filter(s -> s != null && s.length() <= 3)
        .collect(Collectors.toCollection(() -> new LinkedHashSet<>()))
);

Collectors.toMap’in başka bir uygulaması Collector<T, ?, Map<K,U>> toMap(Function<? super T, ? extensions K> keyMapper, Function<? super T, ? extends U> valueMapper, BinaryOperator<U> mergeFunction).The mergeFunction is mostly used to select either new value or retain old value if the key is repeated when adding a new member in the Map from a list.

Kendi Kendini Belgeleyen Süreçleri Yazmak için Akışları ve Yöntem Referanslarını Kullanma #

Metot referansları mükemmel kendi kendini belgeleyen kodlar oluşturur ve Streamler ile metot referansları kullanmak karmaşık süreçleri okumayı ve anlamayı kolaylaştırır. Aşağıdaki kodu göz önünde bulundurun:

public interface Ordered {
    default int getOrder(){
        return 0;
    }
}

public interface Valued<V extends Ordered> {
    boolean hasPropertyTwo();
    V getValue();
}

public interface Thing<V extends Ordered> {
    boolean hasPropertyOne();
    Valued<V> getValuedProperty();
}

public <V extends Ordered> List<V> myMethod(List<Thing<V>> things) {
    List<V> results = new ArrayList<V>();
    for (Thing<V> thing : things) {
        if (thing.hasPropertyOne()) {
            Valued<V> valued = thing.getValuedProperty();
            if (valued != null && valued.hasPropertyTwo()){
                V value = valued.getValue();
                if (value != null){
                    results.add(value);
                }
            }
        }
    }
    results.sort((a, b)->{
        return Integer.compare(a.getOrder(), b.getOrder());
    });
    return results;
}

‘Akış’ ve yöntem referansları kullanılarak yeniden yazılan bu son yöntem çok daha okunaklı ve sürecin her adımı hızlı ve kolay anlaşılır - sadece daha kısa değil, aynı zamanda her birindeki koddan hangi arayüzlerin ve sınıfların sorumlu olduğunu bir bakışta gösterir. adım:

public <V extends Ordered> List<V> myMethod(List<Thing<V>> things) {
    return things.stream()
        .filter(Thing::hasPropertyOne)
        .map(Thing::getValuedProperty)
        .filter(Objects::nonNull)
        .filter(Valued::hasPropertyTwo)
        .map(Valued::getValue)
        .filter(Objects::nonNull)
        .sorted(Comparator.comparing(Ordered::getOrder))
        .collect(Collectors.toList());
}

Azaltma, bir değer elde etmek için bir akışın her öğesine bir ikili işleç uygulama işlemidir.

Bir “IntStream"in “sum()” yöntemi, bir indirgeme örneğidir; Akışın her terimine ek olarak uygulanır ve bu da bir nihai değerle sonuçlanır:

Bir Akışın “azaltma” yöntemi, kişinin özel bir azaltma oluşturmasına olanak tanır. ‘sum()’ yöntemini uygulamak için ‘reduce’ yöntemini kullanmak mümkündür:

Başka bir indirgeme örneği, bir ‘Stream<LinkedList>’ öğesini tek bir ‘LinkedList‘de birleştirmektir:

    LinkedList<T> retList = new LinkedList<T>();

            .collect(Collectors.toMap(element -> element.hashCode(), element -> element));

İşleri daha açıklayıcı hale getirmek için, Function arayüzünde - Function.identity() statik yöntemini kullanabiliriz. Bu lambda element -> element öğesini Function.identity() ile değiştirebiliriz.

Yinelenen anahtarların olabileceği durum

“Collectors.toMap” için javadoc şunu belirtir:

Eşlenen anahtarlar kopyalar içeriyorsa (Object.equals(Object)e göre), toplama işlemi gerçekleştirildiğinde bir IllegalStateException atılır. Eşlenen anahtarların kopyaları varsa, bunun yerine `toMap(Function, Function, BinaryOperator)’ kullanın.

Stream<String> characters = Stream.of("A", "B", "B", "C");

Map<Integer, String> map = characters
            .collect(Collectors.toMap(
                element -> element.hashCode(),
                element -> element,
                (existingVal, newVal) -> (existingVal + newVal)));

// map = {65=A, 66=BB, 67=C}

‘Collectors.toMap(…)’ öğesine iletilen ‘BinaryOperator’, bir çarpışma durumunda depolanacak değeri oluşturur. Bu olabilir:

  • akıştaki ilk değerin öncelikli olması için eski değeri döndürür,
  • akıştaki son değerin öncelikli olması için yeni değeri döndürün veya
  • eski ve yeni değerleri birleştirin

Argümanı olarak bir karşılaştırıcı alan aşırı yüklenmiş bir sıralanmış sürümü olduğundan farklı karşılaştırma mekanizmaları kullanmak da mümkündür.

Ayrıca, sıralama için bir lambda ifadesi kullanabilirsiniz:

List<String> sortedData2 = data.stream().sorted((s1,s2) -> s2.compareTo(s1)).collect(Collectors.toList());

Bu, [Sydney, New York, Mumbai, London, California, Amsterdam] sonucunu verir.

Doğal sıralamanın ’tersini’ uygulayan bir karşılaştırıcıya sahip olmak için ‘Comparator.reverseOrder()’ kullanabilirsiniz.

List<String> reverseSortedData = data.stream().sorted(Comparator.reverseOrder()).collect(Collectors.toList());

İlkel Akışlar #

Java, “IntStream” (“int"ler için), “LongStream” (“long"lar için) ve “DoubleStream” (“double"lar için) olmak üzere üç tür ilkel için özel “Akış” sağlar. İlgili ilkelleri için optimize edilmiş uygulamalar olmanın yanı sıra, tipik olarak matematiksel işlemler için birkaç özel terminal yöntemi de sağlarlar. Örneğin.:

IntStream is = IntStream.of(10, 20, 30);
double average = is.average().getAsDouble(); // average is 20.0

Akış işlemleri kategorileri #

Akış işlemleri, ara işlemler ve uç işlemler olmak üzere iki ana kategoriye ve durumsuz ve durum bilgili olmak üzere iki alt kategoriye ayrılır.

Ara İşlemler: #

Basit bir ‘Stream.map’ gibi bir ara işlem her zaman tembeldir. Akış gerçekten tüketilene kadar çağrılmaz. Bu kolayca doğrulanabilir:

Arrays.asList(1, 2 ,3).stream().map(i -> {
    throw new RuntimeException("not gonna happen");
    return i;
});

Ara işlemler, kaynaktan sonra zincirlenen ve genellikle akış zincirini tetikleyen bir terminal işlemi tarafından takip edilen bir akışın ortak yapı taşlarıdır.

Terminal İşlemleri #

Terminal işlemleri, bir akışın tüketimini tetikleyen şeydir. Daha yaygın olanlardan bazıları ‘Stream.forEach’ veya ‘Stream.collect’tir. Genellikle bir ara işlemler zincirinden sonra yerleştirilirler ve neredeyse her zaman hevesli olurlar.

Vatansız İşlemler #

Vatansızlık, her bir öğenin diğer öğelerin bağlamı olmadan işlenmesi anlamına gelir. Durum bilgisi olmayan işlemler, akışların bellek açısından verimli işlenmesine olanak tanır. Akışın diğer öğeleri hakkında bilgi gerektirmeyen ‘Stream.map’ ve ‘Stream.filter’ gibi işlemler durumsuz olarak kabul edilir.

Durum bilgisi olan işlemler #

Durumsallık, her bir öğe üzerindeki işlemin akışın (bazı) diğer öğelerine bağlı olduğu anlamına gelir. Bu, bir devletin korunmasını gerektirir. Durum bilgisi işlemleri, uzun veya sonsuz akışlarla kesilebilir. ‘Stream.sorted’ gibi işlemler, yeterince uzun bir öğe akışında kırılacak herhangi bir öğe yayınlanmadan önce akışın tamamının işlenmesini gerektirir. Bu, uzun bir akışla gösterilebilir (kendi sorumluluğunuzdadır):

// works - stateless stream
long BIG_ENOUGH_NUMBER = 999999999;
IntStream.iterate(0, i -> i + 1).limit(BIG_ENOUGH_NUMBER).forEach(System.out::println);

Bu, ‘Stream.sorted’ durumunun durumu nedeniyle yetersiz bellek oluşmasına neden olur:

// Out of memory - stateful stream
IntStream.iterate(0, i -> i + 1).limit(BIG_ENOUGH_NUMBER).sorted().forEach(System.out::println);

Bir akışa tek bir Dizeye katılma #

Sıklıkla karşılaşılan bir kullanım durumu, akış öğelerinin belirli bir karakterle ayrıldığı bir akıştan bir “Dize” oluşturmaktır. Aşağıdaki örnekte olduğu gibi, bunun için Collectors.joining() yöntemi kullanılabilir:

Stream<String> fruitStream = Stream.of("apple", "banana", "pear", "kiwi", "orange");

String result = fruitStream.filter(s -> s.contains("a"))
           .map(String::toUpperCase)
           .sorted()
           .collect(Collectors.joining(", "));
           
System.out.println(result);

Çıktı:

ELMA, MUZ, PORTAKAL, ARMUT

‘Collectors.joining()’ yöntemi ayrıca ön ve son düzeltmeleri de karşılayabilir:

String result = fruitStream.filter(s -> s.contains("e"))
           .map(String::toUpperCase)
           .sorted()
           .collect(Collectors.joining(", ", "Fruits: ", "."));
           
System.out.println(result);

Çıktı:

Meyveler: ELMA, PORTAKAL, ARMUT.

Ideone’da Canlı

Bir Akışın Sonuçlarını Bir Dizide Toplayın #

Bir ‘Stream’ için ‘collect()’ tarafından bir koleksiyon elde etmek için analog, ‘Stream.toArray()’ yöntemiyle bir dizi elde edilebilir:

List<String> fruits = Arrays.asList("apple", "banana", "pear", "kiwi", "orange");

String[] filteredFruits = fruits.stream()
    .filter(s -> s.contains("a"))
    .toArray(String[]::new);     

// prints: [apple, banana, pear, orange]
System.out.println(Arrays.toString(filteredFruits));

‘String[]::new’ özel bir tür yöntem referansıdır: bir yapıcı referansı.

Akışları Kullanarak Rastgele Dizeler Oluşturma #

Bazen bir web hizmeti için Oturum Kimliği olarak veya bir uygulamaya kaydolduktan sonra bir başlangıç ​​parolası olarak rastgele ‘Dizeler’ oluşturmak yararlıdır. Bu, ‘Stream’ler kullanılarak kolayca başarılabilir.

İlk önce rastgele bir sayı üreteci başlatmamız gerekiyor. Oluşturulan “Dize"lerin güvenliğini artırmak için “SecureRandom” kullanmak iyi bir fikirdir.

Not: Bir “SecureRandom” oluşturmak oldukça pahalıdır, bu nedenle bunu yalnızca bir kez yapmak ve yeniden tohumlamak için zaman zaman “setSeed()” yöntemlerinden birini çağırmak en iyi uygulamadır.

private static final SecureRandom rng = new SecureRandom(SecureRandom.generateSeed(20)); 
//20 Bytes as a seed is rather arbitrary, it is the number used in the JavaDoc example

Rastgele “Dizeler” oluştururken, genellikle yalnızca belirli karakterleri (örneğin yalnızca harfler ve rakamlar) kullanmalarını isteriz. Bu nedenle, daha sonra “Akımı” filtrelemek için kullanılabilecek bir “boolean” döndüren bir yöntem oluşturabiliriz.

//returns true for all chars in 0-9, a-z and A-Z
boolean useThisCharacter(char c){
    //check for range to avoid using all unicode Letter (e.g. some chinese symbols)
    return c >= '0' && c <= 'z' && Character.isLetterOrDigit(c);
}

Daha sonra, useThisCharacter kontrolümüzü geçen karakter kümesini içeren belirli uzunlukta rastgele bir String oluşturmak için RNG’yi kullanabiliriz.

public String generateRandomString(long length){
    //Since there is no native CharStream, we use an IntStream instead
    //and convert it to a Stream<Character> using mapToObj.
    //We need to specify the boundaries for the int values to ensure they can safely be cast to char
    Stream<Character> randomCharStream = rng.ints(Character.MIN_CODE_POINT, Character.MAX_CODE_POINT).mapToObj(i -> (char)i).filter(c -> this::useThisCharacter).limit(length);

    //now we can use this Stream to build a String utilizing the collect method.
    String randomString = randomCharStream.collect(StringBuilder::new, StringBuilder::append, StringBuilder::append).toString();
    return randomString;
}
