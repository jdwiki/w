Kayan nokta sayıları, kesirli kısımlara sahip sayılardır (genellikle ondalık nokta ile ifade edilir). Java’da kayan noktalı sayılar için “kayan” (4 bayt kullanır) ve “double” (8 bayt kullanır) olmak üzere iki temel tür vardır. Bu dokümantasyon sayfası, Java’da kayan noktalar üzerinde yapılabilecek örnek işlemlerle detaylandırmak içindir.

Kayan nokta değerlerini karşılaştırma #

İlişkisel operatörleri kullanarak kayan nokta değerlerini (“kayan” veya “double”) karşılaştırırken dikkatli olmalısınız: ‘==’, ‘!=’, ‘<’ vb. Bu operatörler, kayan nokta değerlerinin ikili gösterimlerine göre sonuçlar verir. Örneğin:

public class CompareTest {
    public static void main(String[] args) {
        double oneThird = 1.0 / 3.0;
        double one = oneThird * 3;
        System.out.println(one == 1.0);      // prints "false"
    }
}

“OneThird” hesaplaması küçük bir yuvarlama hatası getirdi ve “oneThird"i “3” ile çarptığımızda “1.0"dan biraz farklı bir sonuç elde ediyoruz.

Bu kesin olmayan temsiller sorunu, hesaplamalarda ‘double’ ve ‘float’ı karıştırmaya çalıştığımızda daha belirgindir. Örneğin:

public class CompareTest2 {
    public static void main(String[] args) {
        float floatVal = 0.1f;
        double doubleVal = 0.1;
        double doubleValCopy = floatVal;

        System.out.println(floatVal);      // 0.1
        System.out.println(doubleVal);     // 0.1
        System.out.println(doubleValCopy); // 0.10000000149011612
        
        System.out.println(floatVal == doubleVal); // false
        System.out.println(doubleVal == doubleValCopy); // false
    }
}

Java’da “float” ve “double” türleri için kullanılan kayan nokta gösterimleri, sınırlı sayıda kesinliğe sahiptir. “Float” türü için, kesinlik 23 ikili basamak veya yaklaşık 8 ondalık basamaktır. “Çift” tip için 52 bit veya yaklaşık 15 ondalık basamaktır. Bunun da ötesinde, bazı aritmetik işlemler yuvarlama hatalarına neden olacaktır. Bu nedenle, bir program kayan nokta değerlerini karşılaştırdığında, karşılaştırma için kabul edilebilir bir delta tanımlamak standart uygulamadır. İki sayı arasındaki fark deltadan küçükse eşit sayılırlar. Örneğin

if (Math.abs(v1 - v2) < delta)

Üçgen karşılaştırma örneği:

public class DeltaCompareExample {

    private static boolean deltaCompare(double v1, double v2, double delta) {
        // return true iff the difference between v1 and v2 is less than delta
        return Math.abs(v1 - v2) < delta;
    }
    
    public static void main(String[] args) {
        double[] doubles = {1.0, 1.0001, 1.0000001, 1.000000001, 1.0000000000001};
        double[] deltas = {0.01, 0.00001, 0.0000001, 0.0000000001, 0};

        // loop through all of deltas initialized above
        for (int j = 0; j < deltas.length; j++) {
            double delta = deltas[j];
            System.out.println("delta: " + delta);

            // loop through all of the doubles initialized above
            for (int i = 0; i < doubles.length - 1; i++) {
                double d1 = doubles[i];
                double d2 = doubles[i + 1];
                boolean result = deltaCompare(d1, d2, delta);

                System.out.println("" + d1 + " == " + d2 + " ? " + result);
                
            }

            System.out.println();
        }
    }
}

Ayrıca ‘double’ ve ‘float’ ilkel tiplerinin karşılaştırılması için ilgili kutu tipinin statik ‘karşılaştırma’ yöntemi kullanılabilir. Örneğin:

double a = 1.0;
double b = 1.0001;

System.out.println(Double.compare(a, b));//-1

Son olarak, bir karşılaştırma için hangi deltaların en uygun olduğunu belirlemek zor olabilir. Yaygın olarak kullanılan bir yaklaşım, sezgilerimizin doğru olduğunu söylediği delta değerlerini seçmektir. Ancak, giriş değerlerinin ve yapılan hesaplamaların ölçeğini ve (gerçek) doğruluğunu biliyorsanız, sonuçların ve dolayısıyla deltaların doğruluğu konusunda matematiksel olarak sağlam sınırlar bulmak mümkün olabilir. IEEE Spesifikasyonuna Kesin Uyum #

Varsayılan olarak, “float” ve “double” üzerindeki kayan nokta işlemleri IEEE 754 belirtiminin kurallarına kesinlikle uymaz. Bir ifadenin, bu değerlerin aralığı için uygulamaya özel uzantıları kullanmasına izin verilir; temel olarak, gerekenden * daha * doğru olmalarına izin verir.

‘strictfp’ bu davranışı devre dışı bırakır. Bir sınıfa, arabirime veya yönteme uygulanır ve sınıflar, arabirimler, yöntemler, yapıcılar, değişken başlatıcılar vb. gibi içinde bulunan her şeye uygulanır. ‘strictfp’ ile, bir kayan nokta ifadesinin ara değerleri * * kayan değer seti veya çift değer seti içinde olmalıdır. Bu, bu tür ifadelerin sonuçlarının tam olarak IEEE 754 belirtiminin öngördüğü sonuçlar olmasına neden olur.

Tüm sabit ifadeler, bir “strictfp” kapsamında olmasalar bile, örtük olarak katıdır.

Bu nedenle, ‘strictfp’, bazen belirli köşe durumu hesaplamalarını daha az doğru yapma gibi net bir etkiye sahiptir ve ayrıca kayan nokta işlemlerini *yavaşlaştırabilir (çünkü CPU artık herhangi bir yerel ekstra hassasiyetin sonuç). Ancak sonuçların tüm platformlarda birebir aynı olmasına da neden oluyor. Bu nedenle, tekrarlanabilirliğin hızdan daha önemli olduğu bilimsel programlar gibi şeylerde yararlıdır.

public class StrictFP { // No strictfp -> default lenient
    public strictfp float strict(float input) {
        return input * input / 3.4f; // Strictly adheres to the spec.
                                     // May be less accurate and may be slower.
    }

    public float lenient(float input) {
        return input * input / 3.4f; // Can sometimes be more accurate and faster,
                                     // but results may not be reproducable.
    }

    public static final strictfp class Ops { // strictfp affects all enclosed entities
        private StrictOps() {}

        public static div(double dividend, double divisor) { // implicitly strictfp
            return dividend / divisor;
        }
    }
}
