Diziler, isteğe bağlı miktarda değerin depolanmasına ve alınmasına izin verir. Matematikteki vektörlere benzerler. Dizi dizileri matrislere benzer ve çok boyutlu diziler gibi davranır. Diziler her türden veriyi depolayabilir: “int” gibi temel öğeler veya “Nesne” gibi başvuru türleri.

Sözdizimi #

  • ArrayType[] myArray; // Dizileri bildirme
  • ArrayType myArray[]; // Başka bir geçerli sözdizimi (daha az kullanılan ve önerilmeyen)
  • ArrayType[][][] myArray; // Çok boyutlu pürüzlü dizileri bildirme (tekrar []s)
  • ArrayType myVar = myArray[index]; // Dizindeki öğeye erişme (okuma)
  • myArray[index] = value; // Dizinin `index’ konumuna değer ata
  • ArrayType[] myArray = new ArrayType[arrayLength]; // Dizi başlatma sözdizimi
  • int[] ints = {1, 2, 3}; // Sağlanan değerlerle dizi başlatma sözdizimi, uzunluk sağlanan değerlerin sayısından çıkarılır: {[değer1[, değer2]*]}
  • `new int[]{4, -5, 6} // Yerel değişken olmadan bağımsız değişken olarak kullanılabilir'
  • int[] ints = yeni int[3]; // {0, 0, 0} ile aynı
  • int[][] ints = {{1, 2}, {3}, null}; // Çok boyutlu dizi başlatma. int[], Object öğesini genişletir (ve anyType[] da öyle) bu nedenle null geçerli bir değerdir.

Giriş #

Bir dizi, sabit sayıda ilkel değeri veya nesne örneklerine yapılan başvuruları tutan bir veri yapısıdır.

Bir dizideki her öğeye öğe denir ve her öğeye sayısal dizini ile erişilir. Bir dizinin uzunluğu, dizi oluşturulduğunda belirlenir:

int size = 42;
int[] array = new int[size];

Bir dizinin boyutu, başlatıldığında çalışma zamanında sabitlenir. Başlatıldıktan sonra değiştirilemez. Çalışma zamanında boyutun değişken olması gerekiyorsa, bunun yerine ‘ArrayList’ gibi bir ‘Collection’ sınıfı kullanılmalıdır. “ArrayList”, öğeleri bir dizide saklar ve [yeni bir dizi ayırarak yeniden boyutlandırmayı] destekler(https://www.wikiod.com/tr/java/diziler#Bir dizinin boyutunu nasıl değiştirirsiniz? ) ve eski dizideki öğeleri kopyalama.

Dizi ilkel türdeyse, yani.

int[] array1 = { 1,2,3 };
int[] array2 = new int[10];

değerler dizinin kendisinde saklanır. Başlatıcının yokluğunda (yukarıdaki “dizi2"de olduğu gibi), her öğeye atanan varsayılan değer “0"dır (sıfır).

Dizi türü, aşağıdaki gibi bir nesne başvurusuysa

SomeClassOrInterface[] array = new SomeClassOrInterface[10];

daha sonra dizi, ‘SomeClassOrInterface’ türündeki nesnelere references içerir. Bu referanslar, bir “SomeClassOrInterface” örneğine veya “SomeClassOrInterface"in herhangi bir alt sınıfına (sınıflar için) veya uygulama sınıfına (arayüzler için) başvurabilir. Dizi bildiriminde başlatıcı yoksa, her öğeye varsayılan “null” değeri atanır.

Tüm diziler “int” dizinli olduğundan, dizinin boyutu bir “int” ile belirtilmelidir. Dizinin boyutu “uzun” olarak belirtilemez:

long size = 23L;
int[] array = new int[size]; // Compile-time error:
                             // incompatible types: possible lossy conversion from
                             // long to int

Diziler, sıfır tabanlı bir dizin sistemi kullanır; bu, dizin oluşturmanın “0” ile başlayıp “uzunluk - 1” ile bittiği anlamına gelir.

Örneğin, aşağıdaki görüntü “10” boyutunda bir diziyi temsil etmektedir. Burada, ilk eleman “1” dizininde ve son eleman “10” dizininde olmak yerine, ilk eleman “0” dizininde ve son eleman “9” dizinindedir (aşağıdaki şekle bakın).

Dizilerin öğelerine erişim sabit zamanda yapılır. Bu, dizinin ilk öğesine erişmenin, ikinci öğeye, üçüncü öğeye vb. erişme maliyetinin (zaman açısından) aynı olduğu anlamına gelir.

Java, literal ve constructor gösterimleri dahil olmak üzere dizileri tanımlamanın ve başlatmanın çeşitli yollarını sunar. ’new Type[length]’ yapıcısını kullanarak dizileri bildirirken, her öğe aşağıdaki varsayılan değerlerle başlatılacaktır:

  • [ilkel sayısal türler][belgeler-ilkel-türler] için “0”: “byte”, “short”, “int”, “long”, “float” ve “double”.
  • ‘char’ türü için ‘\u0000’’ (boş karakter).
  • “boolean” türü için “false”.
  • referans türleri için “null”.

İlkel tür dizileri oluşturma ve başlatma #

int[] array1 = new int[] { 1, 2, 3 }; // Create an array with new operator and 
                                      // array initializer.
int[] array2 = { 1, 2, 3 };           // Shortcut syntax with array initializer.
int[] array3 = new int[3];            // Equivalent to { 0, 0, 0 }
int[] array4 = null;                  // The array itself is an object, so it
                                      // can be set as null.

Bir dizi bildirirken, “[]”, bildirimin başlangıcında türün bir parçası olarak (tür adından sonra) veya belirli bir değişken için (değişken adından sonra) bildiricinin bir parçası olarak veya her ikisinde görünür:

int array5[];       /* equivalent to */  int[] array5;
int a, b[], c[][];  /* equivalent to */  int a; int[] b; int[][] c;
int[] a, b[];       /* equivalent to */  int[] a; int[][] b;
int a, []b, c[][];  /* Compilation Error, because [] is not part of the type at beginning
                       of the declaration, rather it is before 'b'. */    
// The same rules apply when declaring a method that returns an array:
int foo()[] { ... } /* equivalent to */  int[] foo() { ... }

Aşağıdaki örnekte, her iki bildirim de doğrudur ve sorunsuz bir şekilde derlenip çalıştırılabilir. Ancak, hem Java Kodlama Kuralı hem de [Google Java Stil Kılavuzu][google-docs-styleguide], formun değişken adından sonra köşeli parantez kullanmasını önermemektedir—parantezler dizi türünü tanımlar ve tip tanımı. Aynısı yöntem dönüş imzaları için kullanılmalıdır.

float array[]; /* and */ int foo()[] { ... } /* are discouraged */
float[] array; /* and */ int[] foo() { ... } /* are encouraged */

Cesaretini kıran tür, değişken adından sonra köşeli parantezler içeren C sözdizimine aşina olan [geçiş yapan C kullanıcılarını [so-qa-arraysyntax] barındırmak içindir.

Java’da “0” boyutunda dizilere sahip olmak mümkündür:

int[] array = new int[0]; // Compiles and runs fine.
int[] array2 = {};        // Equivalent syntax.

Ancak, boş bir dizi olduğundan, ondan hiçbir öğe okunamaz veya atanamaz:

array[0] = 1;     // Throws java.lang.ArrayIndexOutOfBoundsException.
int i = array2[0]; // Also throws ArrayIndexOutOfBoundsException.

Bu tür boş diziler tipik olarak dönüş değerleri olarak kullanışlıdır, böylece çağıran kodun bir [NullPointerException][so-docs-npe]‘ye yol açabilecek potansiyel bir “boş” değerden ziyade yalnızca bir diziyle uğraşma konusunda endişelenmesi gerekir. .

Bir dizinin uzunluğu, negatif olmayan bir tam sayı olmalıdır:

int[] array = new int[-1]; // Throws java.lang.NegativeArraySizeException

Dizi boyutu, “uzunluk” adı verilen genel bir son alan kullanılarak belirlenebilir:

System.out.println(array.length); // Prints 0 in this case.

Not: “array.length”, döndürülen [ArrayList.size()][arraylist-size-javadocs]‘un aksine, bir değer atanan dizi öğelerinin sayısını değil dizinin gerçek boyutunu döndürür. bir değer atanan dizi öğelerinin sayısı.

Çok boyutlu diziler oluşturma ve başlatma #

Çok boyutlu bir dizi oluşturmanın en basit yolu şudur:

int[][] a = new int[2][3];

İki üç uzunluklu “int” dizisi oluşturacaktır: “a[0]” ve “a1”. Bu, dikdörtgen çok boyutlu dizilerin klasik, C-tarzı başlatmasına çok benzer.

Aynı anda oluşturabilir ve başlatabilirsiniz:

int[][] a = { {1, 2}, {3, 4}, {5, 6} };

Yalnızca dikdörtgen çok boyutlu dizilerin desteklendiği C‘den farklı olarak, iç dizilerin aynı uzunlukta olması veya hatta tanımlanmış olması gerekmez:

int[][] a = { {1}, {2, 3}, null };

Burada “a[0]” tek uzunluklu bir “int” dizisidir, “a1” ise iki uzunluklu bir “int” dizisidir ve “a2” “boş"tur. Bunun gibi dizilere [tırtıklı diziler veya düzensiz diziler][so-docs-jaggedarrays] denir, yani bunlar dizi dizileridir. Java’da çok boyutlu diziler, dizi dizileri olarak uygulanır, yani “dizi[i][j][k]’, ‘((dizi[i])[j])[k]’ ile eşdeğerdir. [C#][so-qa-jaggeddifference]‘dan farklı olarak, “dizi[i,j]” sözdizimi Java’da desteklenmez.

Object[] array10 = { new Object(), new Object() };

Diziler kovaryant olduğundan, bir öğeyi ‘Dize’ dışında bir şeye ayarlamaya çalışırsanız bir ‘ArrayStoreException’ atılmasına rağmen, bir başvuru türü dizisi bir alt sınıfın dizisi olarak başlatılabilir:

Object[] array11 = new String[] { "foo", "bar", "baz" };
array11[1] = "qux"; // fine
array11[1] = new StringBuilder(); // throws ArrayStoreException

Kısayol sözdizimi bunun için kullanılamaz çünkü kısayol sözdizimi örtük bir ‘Nesne[]’ türüne sahip olacaktır.

Bir dizi, String[] emptyArray = new String[0] kullanılarak sıfır elemanla başlatılabilir. Örneğin, bunun gibi sıfır uzunlukta bir dizi [“Koleksiyondan” Bir “Dizi” Yaratmak için kullanılır](https://www.wikiod.com/tr/Java/diziler#Koleksiyondan Dizi Oluşturma from-a-collection) yöntemin bir nesnenin çalışma zamanı türüne ihtiyacı olduğunda.

generic type dizileri oluşturma ve başlatma #

Genel sınıflarda, genel tür dizileri [tür silme][belgeler-türü silme] nedeniyle şu şekilde başlatılamaz:

public class MyGenericClass<T> {
    private T[] a;

    public MyGenericClass() {
        a = new T[5]; // Compile time error: generic array creation
    }
}

Bunun yerine, aşağıdaki yöntemlerden biri kullanılarak oluşturulabilirler: (bunların denetlenmeyen uyarılar oluşturacağını unutmayın)

  1. Bir “Nesne” dizisi oluşturarak ve onu genel türe çevirerek:

    a = (T[]) new Object[5];

    This is the simplest method, but since the underlying array is still of type Object[], this method does not provide type safety. Therefore, this method of creating an array is best used only within the generic class - not exposed publicly.

  2. Bir sınıf parametresiyle Array.newInstance kullanarak:

    public MyGenericClass(Class<T> clazz) {
    a = (T[]) Array.newInstance(clazz, 5);
}

    Here the class of T has to be explicitly passed to the constructor. The return type of Array.newInstance is always Object. However, this method is safer because the newly created array is always of type T[], and therefore can be safely externalized.

Dizilerin ayrı bildirimi ve başlatılması #

Dizi öğesi için bir dizinin değeri bir tam sayı (0, 1, 2, 3, 4, …) olmalı ve dizinin uzunluğundan küçük olmalıdır (dizinler sıfır tabanlıdır). Aksi takdirde, bir [ArrayIndexOutOfBoundsException][docs-aioobe] atılır:

int[] array9;             // Array declaration - uninitialized
array9 = new int[3];      // Initialize array  - { 0, 0, 0 }
array9[0] = 10;           // Set index 0 value - { 10, 0, 0 }
array9[1] = 20;           // Set index 1 value - { 10, 20, 0 }
array9[2] = 30;           // Set index 2 value - { 10, 20, 30 }

Diziler, dizi başlatıcı kısayol sözdizimi ile yeniden başlatılamayabilir #

Bir dizi başlatıcı yalnızca bir alan bildiriminde veya yerel değişken bildiriminde veya bir dizi oluşturma ifadesi.

Ancak, yeni bir dizi oluşturmak ve onu eski diziye başvurmak için kullanılan değişkene atamak mümkündür. Bu, o değişken tarafından başvurulan dizinin yeniden başlatılmasıyla sonuçlanırken, değişken içerikleri tamamen yeni bir dizidir. Bunu yapmak için, “yeni” operatörü bir dizi başlatıcı ile kullanılabilir ve dizi değişkenine atanabilir:

// First initialization of array
int[] array = new int[] { 1, 2, 3 };

// Prints "1 2 3 ".
for (int i : array) {
    System.out.print(i + " ");
}

// Re-initializes array to a new int[] array.
array = new int[] { 4, 5, 6 };

// Prints "4 5 6 ".
for (int i : array) {
    System.out.print(i + " ");
}

array = { 1, 2, 3, 4 }; // Compile-time error! Can't re-initialize an array via shortcut 
                        // syntax with array initializer.

Diziden Liste Oluşturma #

Arrays.asList() yöntemi, verilen dizinin öğelerini içeren sabit boyutlu bir “Liste” döndürmek için kullanılabilir. Ortaya çıkan ‘Liste’, dizinin temel türüyle aynı parametre türünde olacaktır.

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = Arrays.asList(stringArray);

Not: Bu liste, orijinal dizinin (bir görünümü) tarafından desteklenir; bu, listede yapılacak herhangi bir değişikliğin diziyi değiştireceği ve bunun tersi anlamına gelir. Ancak, boyutunu (ve dolayısıyla dizi uzunluğunu) değiştirecek olan listede yapılan değişiklikler bir istisna oluşturacaktır.

ArrayIndexOutOfBoundsException #

ArrayIndexOutOfBoundsException, bir dizinin var olmayan bir dizinine erişildiğinde atılır.

Diziler sıfır tabanlı indekslenir, bu nedenle ilk elemanın indeksi “0” ve son elemanın indeksi dizi kapasitesi eksi “1"dir (yani “dizi.uzunluk - 1”).

Bu nedenle, ‘i’ dizini tarafından bir dizi öğesi için yapılan herhangi bir istek, ‘0 <= i < dizi.length’ koşulunu sağlamalıdır, aksi takdirde ‘ArrayIndexOutOfBoundsException’ atılır.

Aşağıdaki kod, bir ArrayIndexOutOfBoundsException atıldığı basit bir örnektir.

String[] people = new String[] { "Carol", "Andy" };

// An array will be created:
// people[0]: "Carol"
// people[1]: "Andy"

// Notice: no item on index 2. Trying to access it triggers the exception:
System.out.println(people[2]);  // throws an ArrayIndexOutOfBoundsException.

Çıktı:

Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: 2
    at your.package.path.method(YourClass.java:15)

Erişilmekte olan geçersiz dizinin de istisnaya dahil edildiğini unutmayın (örnekte ‘2’); bu bilgi, istisnanın nedenini bulmak için faydalı olabilir.

Bunu önlemek için, dizinin dizinin sınırları içinde olup olmadığını kontrol edin:

int index = 2;
if (index >= 0 && index < people.length) {
    System.out.println(people[index]);
}

Bir Dizinin Uzunluğunu Alma #

Diziler, belirtilen türdeki öğelerin boyutuna kadar depolamak için alan sağlayan nesnelerdir. Dizi oluşturulduktan sonra dizinin boyutu değiştirilemez.

int[] arr1 = new int[0];
int[] arr2 = new int[2];
int[] arr3 = new int[]{1, 2, 3, 4};
int[] arr4 = {1, 2, 3, 4, 5, 6, 7};

int len1 = arr1.length; // 0
int len2 = arr2.length; // 2
int len3 = arr3.length; // 4
int len4 = arr4.length; // 7

Bir dizideki “uzunluk” alanı, bir dizinin boyutunu saklar. Bu bir “son” alandır ve değiştirilemez.

Bu kod, bir dizinin “uzunluğu” ile bir dizinin depoladığı nesne miktarı arasındaki farkı gösterir.

public static void main(String[] args) {
    Integer arr[] = new Integer[] {1,2,3,null,5,null,7,null,null,null,11,null,13};

    int arrayLength = arr.length;
    int nonEmptyElementsCount = 0;

    for (int i=0; i<arrayLength; i++) {
        Integer arrElt = arr[i];
        if (arrElt != null) {
            nonEmptyElementsCount++;
        }
    }

    System.out.println("Array 'arr' has a length of "+arrayLength+"\n"
                            + "and it contains "+nonEmptyElementsCount+" non-empty values");
}

Diziden bir öğeyi kaldırın #

Java, bir diziden bir öğeyi kaldırmak için java.util.Arrays içinde doğrudan bir yöntem sağlamaz. Bunu gerçekleştirmek için, kaldırılacak öğe olmadan orijinal diziyi yeni bir diziye kopyalayabilir veya dizinizi kaldırmaya izin veren başka bir yapıya dönüştürebilirsiniz.

ArrayList’i Kullanma #

Diziyi bir java.util.List‘ye dönüştürebilir, öğeyi kaldırabilir ve listeyi aşağıdaki gibi bir diziye dönüştürebilirsiniz:

String[] array = new String[]{"foo", "bar", "baz"};

List<String> list = new ArrayList<>(Arrays.asList(array));
list.remove("foo");

// Creates a new array with the same size as the list and copies the list
// elements to it.
array = list.toArray(new String[list.size()]);

System.out.println(Arrays.toString(array)); //[bar, baz]

System.arraycopy’yi kullanma #

System.arraycopy() orijinal dizinin bir kopyasını oluşturmak ve istediğiniz öğeyi kaldırmak için kullanılabilir. Bir örneğin altında:

int[] array = new int[] { 1, 2, 3, 4 }; // Original array.
int[] result = new int[array.length - 1]; // Array which will contain the result.
int index = 1; // Remove the value "2".

// Copy the elements at the left of the index.
System.arraycopy(array, 0, result, 0, index);
// Copy the elements at the right of the index.
System.arraycopy(array, index + 1, result, index, array.length - index - 1);

System.out.println(Arrays.toString(result)); //[1, 3, 4]

Apache Commons Lang’i Kullanma #

Bir öğeyi kolayca kaldırmak için Apache Commons Lang kitaplığını ve özellikle ArrayUtils sınıfının removeElement() statik yöntemini kullanabilirsiniz. Bir örneğin altında:

int[] array = new int[]{1,2,3,4};
array = ArrayUtils.removeElement(array, 2); //remove first occurrence of 2
System.out.println(Arrays.toString(array)); //[1, 3, 4]

Bir dizinin boyutunu nasıl değiştirirsiniz? #

Basit cevap, bunu yapamayacağınızdır. Bir dizi oluşturulduktan sonra boyutu değiştirilemez. Bunun yerine, bir dizi yalnızca uygun boyutta yeni bir dizi oluşturularak ve mevcut dizideki öğeler yeni diziye kopyalanarak “yeniden boyutlandırılabilir”.

String[] listOfCities = new String[3];   // array created with size 3.
listOfCities[0] = "New York";
listOfCities[1] = "London";
listOfCities[2] = "Berlin";

(Örneğin) yukarıda tanımlandığı gibi ’listOfCities’ dizisine yeni bir öğenin eklenmesi gerektiğini varsayalım. Bunu yapmak için yapmanız gerekenler:

  1. boyutu 4 olan yeni bir dizi oluşturun,
  2. eski dizinin mevcut 3 öğesini yeni diziye 0, 1 ve 2 uzaklıklarında kopyalayın ve
  3. yeni öğeyi yeni diziye ofset 3’te ekleyin.

Yukarıdakileri yapmanın çeşitli yolları vardır. Java 6’dan önce en özlü yol şuydu:

String[] newArray = new String[listOfCities.length + 1];
System.arraycopy(listOfCities, 0, newArray, 0, listOfCities.length);
newArray[listOfCities.length] = "Sydney";

Java 6’dan itibaren, “Arrays.copyOf” ve “Arrays.copyOfRange” yöntemleri bunu daha basit bir şekilde yapabilir:

String[] newArray = Arrays.copyOf(listOfCities, listOfCities.length + 1);
newArray[listOfCities.length] = "Sydney";

Bir diziyi kopyalamanın diğer yolları için aşağıdaki örneğe bakın. Yeniden boyutlandırırken orijinalinden farklı uzunlukta bir dizi kopyasına ihtiyacınız olduğunu unutmayın.

Dizi yeniden boyutlandırma için daha iyi bir alternatif #

Bir diziyi yukarıda açıklandığı gibi yeniden boyutlandırmanın iki büyük dezavantajı vardır:

  • Verimsiz. Bir diziyi büyütmek (veya küçültmek), mevcut dizi öğelerinin çoğunu veya tamamını kopyalamayı ve yeni bir dizi nesnesi tahsis etmeyi içerir. Dizi ne kadar büyük olursa, o kadar pahalı olur.
  • Eski diziye referanslar içeren herhangi bir “canlı” değişkeni güncelleyebilmeniz gerekir.

Bir alternatif, başlamak için yeterince büyük bir dizi oluşturmaktır. Bu, yalnızca bu boyutu diziyi ayırmadan önce doğru bir şekilde belirleyebilirseniz geçerlidir. Bunu yapamazsanız, diziyi yeniden boyutlandırma sorunu tekrar ortaya çıkar.

Diğer alternatif, Java SE sınıf kitaplığı veya üçüncü taraf kitaplığı tarafından sağlanan bir veri yapısı sınıfını kullanmaktır. Örneğin, Java SE “koleksiyonları” çerçevesi, farklı çalışma zamanı özelliklerine sahip “Liste”, “Set” ve “Harita” API’lerinin bir dizi uygulamasını sağlar. ‘ArrayList’ sınıfı, düz bir dizinin performans özelliklerine en yakın olanıdır (örneğin, O(N) araması, O(1) alma ve ayarlama, O(N) rastgele ekleme ve silme) ve referans güncelleme sorunu olmadan daha verimli yeniden boyutlandırma sağlar.

(“ArrayList” için yeniden boyutlandırma verimliliği, her yeniden boyutlandırmada destek dizisinin boyutunu iki katına çıkarma stratejisinden gelir. Tipik bir kullanım durumu için bu, yalnızca ara sıra yeniden boyutlandırmanız anlamına gelir. Listenin ömrü boyunca amorti ettiğinizde, ekleme başına yeniden boyutlandırma maliyeti ‘O(1)‘dir. Düz bir diziyi yeniden boyutlandırırken aynı stratejiyi kullanmak mümkün olabilir.)

Dizileri eşitlik için karşılaştırma #

Dizi türleri [equals() (ve hashCode()) uygulamalarını Java.lang.Object](https://www.wikiod.com/tr/Java/nesne-snf-yontemleri-ve-olusturucu#eşittir() yöntemi)‘dan devralır -method#t=201607191720385793182), bu nedenle “eşittir()” yalnızca tam olarak aynı dizi nesnesiyle karşılaştırırken true değerini döndürür. Dizileri değerlerine göre eşitlik açısından karşılaştırmak için java.util.Arrays.equals kullanın -Java.lang.Object:A-Java.lang.Object:A-), tüm dizi türleri için aşırı yüklenmiştir.

int[] a = new int[]{1, 2, 3};
int[] b = new int[]{1, 2, 3};
System.out.println(a.equals(b)); //prints "false" because a and b refer to different objects
System.out.println(Arrays.equals(a, b)); //prints "true" because the elements of a and b have the same values

Öğe türü bir başvuru türü olduğunda, “Arrays.equals()” eşitliği belirlemek için dizi öğelerinde “equals()” öğesini çağırır. Özellikle, eleman tipinin kendisi bir dizi tipi ise, kimlik karşılaştırması kullanılacaktır. Çok boyutlu dizileri eşitlik açısından karşılaştırmak için Arrays.deepEquals() kullanın. Object:A-Java.lang.Object:A-) yerine aşağıdaki gibi:

int a[] = { 1, 2, 3 };
int b[] = { 1, 2, 3 };

Object[] aObject = { a }; // aObject contains one element
Object[] bObject = { b }; // bObject contains one element

System.out.println(Arrays.equals(aObject, bObject)); // false
System.out.println(Arrays.deepEquals(aObject, bObject));// true

Kümeler ve haritalar “equals()” ve “hashCode()” kullandığından, diziler genellikle set öğeleri veya harita anahtarları olarak kullanışlı değildir. Bunları, dizi öğeleri açısından “equals()” ve “hashCode()” uygulayan bir yardımcı sınıfa sarın veya bunları “List” örneklerine dönüştürün ve listeleri depolayın.

Döküm Dizileri #

Diziler nesnelerdir, ancak türleri içerilen nesnelerin türüne göre tanımlanır. Bu nedenle, bir kişi sadece “A[]“yı “T[]“ye dönüştüremez, ancak belirli “A[]“nın her A üyesi bir “T” nesnesine dönüştürülmelidir. Genel örnek:

public static <T, A> T[] castArray(T[] target, A[] array) {
    for (int i = 0; i < array.length; i++) {
        target[i] = (T) array[i];
    }
    return target;
}

Böylece, bir “A[]” dizisi verilir:

T[] target = new T[array.Length];
target = castArray(target, array);

Java SE, bu amaç için Arrays.copyOf(original, newLength, newType) yöntemini sağlar:

Double[] doubles = { 1.0, 2.0, 3.0 };
Number[] numbers = Arrays.copyOf(doubles, doubles.length, Number[].class);
