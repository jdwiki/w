  • Sözdizimi

  • Özellikler-DB’de oturum açmak için dosya

Bu sayfada

  • Sözdizimi
  • Log4j nasıl edinilir
  • Özellik dosyasını ayarlama
  • Özellikler-DB’de oturum açmak için dosya
  • Java kodunda Log4j nasıl kullanılır
  • Temel log4j2.xml yapılandırma dosyası
  • log4j 1.x’ten 2.x’e geçiş
  • Seviyeye göre çıkış çıkışını filtrele (log4j 1.x)

Apache Log4j Java tabanlı bir günlük kaydı aracıdır, birkaç Java günlükleme çerçevesinden biridir. Bu konu, Log4j’nin Java’da nasıl kurulacağını ve yapılandırılacağını, tüm olası kullanım yönleriyle ilgili ayrıntılı örneklerle göstermektir.

Sözdizimi #

  • Logger.debug(“günlüğe yazılacak metin”); // Hata ayıklama bilgilerini günlüğe kaydetme
  • Logger.info(“günlüğe yazılacak metin”); // Ortak bilgileri günlüğe kaydetme
  • Logger.error(“günlüğe yazılacak metin”); // Günlüğe kaydetme hatası bilgisi
  • Logger.warn(“günlüğe yazılacak metin”); // Günlüğe kaydetme uyarıları
  • Logger.trace(“günlüğe yazılacak metin”); // İzleme bilgilerini günlüğe kaydetme
  • Logger.fatal(“günlüğe yazılacak metin”); // Önemli hataları günlüğe kaydetme
  • Parametre günlüğü ile Log4j2 kullanımı:
  • Logger.debug(“Parametrelerde hata ayıklama {} {} {}”, param1, param2, param3); // Hata ayıklamayı parametrelerle günlüğe kaydetme
  • Logger.info(“Bilgi parametreleri {} {} {}”, param1, param2, param3); // Parametrelerle loglama bilgisi
  • Logger.error(“Hata parametreleri {} {} {}”, param1, param2, param3); // Parametrelerle loglama hatası
  • Logger.warn(“Parametreleri uyar {} {} {}”, param1, param2, param3); // Uyarıları parametrelerle günlüğe kaydetme
  • Logger.trace(“Parametreleri izle {} {} {}”, param1, param2, param3); // İzlemeyi parametrelerle günlüğe kaydetme
  • Logger.fatal(“Önemli parametreler {} {} {}”, param1, param2, param3); // Parametrelerle ölümcül günlüğe kaydetme
  • Logger.error(“Yakalanan İstisna: “, örn ); // Mesaj ve stacktrace ile günlük istisnası (otomatik olarak eklenecektir)

Log4j 1 için Kullanım Ömrünün Sonuna Ulaştı

5 Ağustos 2015’te Kayıt Hizmetleri Proje Yönetim Komitesi, Log4j 1.x’in kullanım ömrünün sonuna geldiğini duyurdu. Duyurunun tam metni için lütfen Apache Blog’una bakın. Log4j 1 kullanıcılarının Apache Log4j 2’ye yükseltme yapmaları önerilir.

Kimden: http://logging.apache.org/log4j/1.2/

Özellik dosyasını ayarlama #

Log4j, verileri aynı anda konsola ve dosyaya kaydetme imkanı sunar. Bir “log4j.properties” dosyası oluşturun ve bu temel yapılandırmanın içine koyun:

# Root logger option
log4j.rootLogger=DEBUG, stdout, file

# Redirect log messages to console
log4j.appender.stdout=org.apache.log4j.ConsoleAppender
log4j.appender.stdout.Target=System.out
log4j.appender.stdout.layout=org.apache.log4j.PatternLayout
log4j.appender.stdout.layout.ConversionPattern=%d{yyyy-MM-dd HH:mm:ss} %-5p %c{1}:%L - %m%n

# Redirect log messages to a log file, support file rolling.
log4j.appender.file=org.apache.log4j.RollingFileAppender
log4j.appender.file.File=C:\\log4j-application.log
log4j.appender.file.MaxFileSize=5MB
log4j.appender.file.MaxBackupIndex=10
log4j.appender.file.layout=org.apache.log4j.PatternLayout
log4j.appender.file.layout.ConversionPattern=%d{yyyy-MM-dd HH:mm:ss} %-5p %c{1}:%L - %m%n

Maven kullanıyorsanız, bu özellik dosyasını yola koyun:

/ProjectFolder/src/java/resources

Özellikler-DB’de oturum açmak için dosya #

Bu örneğin çalışması için veritabanının çalıştığı sistemle uyumlu bir JDBC sürücüsüne ihtiyacınız olacak. Bir IBM System i üzerindeki DB2 veritabanlarına bağlanmanıza izin veren bir açık kaynak kodu burada bulunabilir: JT400

Bu örnek DB2’ye özel olsa da, sürücüyü değiştirirseniz ve JDBC URL’sini uyarlarsanız, hemen hemen tüm diğer sistemler için çalışır.

# Root logger option
log4j.rootLogger= ERROR, DB

# Redirect log messages to a DB2
# Define the DB appender   
log4j.appender.DB=org.apache.log4j.jdbc.JDBCAppender

# Set JDBC URL (!!! adapt to your target system !!!)
log4j.appender.DB.URL=jdbc:as400://10.10.10.1:446/DATABASENAME;naming=system;errors=full;

# Set Database Driver (!!! adapt to your target system !!!)
log4j.appender.DB.driver=com.ibm.as400.access.AS400JDBCDriver

# Set database user name and password
log4j.appender.DB.user=USER
log4j.appender.DB.password=PASSWORD

# Set the SQL statement to be executed.
log4j.appender.DB.sql=INSERT INTO DB.TABLENAME VALUES('%d{yyyy-MM-dd}','%d{HH:mm:ss}','%C','%p','%m')

# Define the layout for file appender
log4j.appender.DB.layout=org.apache.log4j.PatternLayout

Java kodunda Log4j nasıl kullanılır #

İlk önce bir “son statik günlükçü” nesnesi oluşturmanız gerekir:

final static Logger logger = Logger.getLogger(classname.class);

Ardından, günlüğe kaydetme yöntemlerini arayın:

//logs an error message
logger.info("Information about some param: " + parameter); // Note that this line could throw a NullPointerException!

//in order to improve performance, it is advised to use the `isXXXEnabled()` Methods
if( logger.isInfoEnabled() ){
    logger.info("Information about some param: " + parameter);
}

// In log4j2 parameter substitution is preferable due to readability and performance
// The parameter substitution only takes place if info level is active which obsoletes the use of isXXXEnabled().
logger.info("Information about some param: {}" , parameter);

//logs an exception
logger.error("Information about some error: ", exception);

Temel log4j2.xml yapılandırma dosyası #

<?xml version="1.0" encoding="UTF-8"?>
<Configuration>
  <Appenders>
    <Console name="STDOUT" target="SYSTEM_OUT">
      <PatternLayout pattern="%d %-5p [%t] %C{2} %m%n"/>
    </Console>
  </Appenders>
  <Loggers>
    <Root level="debug">
      <AppenderRef ref="STDOUT"/>
    </Root>
  </Loggers>
</Configuration>

Bu, konsol ekleyicisi ve kök kaydedicisi olan temel bir log4j2.xml yapılandırmasıdır. Kalıp düzeni, ifadeleri günlüğe kaydetmek için hangi kalıbın kullanılması gerektiğini belirtir.
log4j2.xml yüklemesinde hata ayıklamak için status = <WARN | hata ayıklama | HATA | ÖNEMLİ | İZ | INFO> log4j2.xml dosyanızın yapılandırma etiketinde.
Ayrıca, belirtilen aralık süresinden sonra yapılandırmayı yeniden yüklemesi için bir izleme aralığı ekleyebilirsiniz. İzleme aralığı, yapılandırma etiketine şu şekilde eklenebilir: monitorInterval = 30. Bu, yapılandırmanın her 30 saniyede bir yükleneceği anlamına gelir.

Seviyeye göre çıkış çıkışını filtrele (log4j 1.x) #

Yalnızca “düşük” iletileri günlüğe kaydetmek için bir filtre kullanabilirsiniz; “HATA” seviyesi. Ancak filtre PropertyConfigurator tarafından desteklenmez. Bu yüzden kullanmak için XML yapılandırmasına geçmelisiniz. filtreler hakkında log4j-Wiki konusuna bakın.

Örnek “belirli seviye”

<appender name="info-out" class="org.apache.log4j.FileAppender"> 
            <param name="File" value="info.log"/> 
            <layout class="org.apache.log4j.PatternLayout"> 
                    <param name="ConversionPattern" value="%m%n"/> 
            </layout> 
            <filter class="org.apache.log4j.varia.LevelMatchFilter">
                    <param name="LevelToMatch" value="info" />
                    <param name="AcceptOnMatch" value="true"/>
            </filter>
            <filter class="org.apache.log4j.varia.DenyAllFilter" />
</appender>

Veya “Seviye aralığı”

<appender name="info-out" class="org.apache.log4j.FileAppender"> 
            <param name="File" value="info.log"/> 
            <layout class="org.apache.log4j.PatternLayout"> 
                    <param name="ConversionPattern" value="%m%n"/> 
            </layout> 
            <filter class="org.apache.log4j.varia.LevelRangeFilter">
                    <param name="LevelMax" value="info"/>
                    <param name="LevelMin" value="info"/>
                    <param name="AcceptOnMatch" value="true"/>
            </filter>
</appender>
