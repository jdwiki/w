Java.util.Map arabirimi, anahtarlar ve değerleri arasında bir eşlemeyi temsil eder. Bir harita yinelenen anahtarlar içeremez; ve her anahtar en fazla bir değere eşlenebilir.

“Harita” bir arayüz olduğundan, kullanmak için o arayüzün somut bir uygulamasını başlatmanız gerekir; birkaç “Harita” uygulaması vardır ve en çok kullanılanlar “java.util.HashMap” ve “java.util.TreeMap"tir.

map, her bir anahtar için ilişkili bir value ile keys depolayan bir nesnedir. Bir anahtar ve değeri bazen bir anahtar/değer çifti veya bir giriş olarak adlandırılır. Haritalar genellikle şu özellikleri sağlar:

  • Veriler, anahtar/değer çiftleri halinde haritaya kaydedilir.
  • Harita, belirli bir anahtar için yalnızca bir giriş içerebilir. Bir harita belirli bir anahtara sahip bir giriş içeriyorsa ve aynı anahtarla ikinci bir giriş kaydetmeye çalışırsanız, ikinci giriş birincinin yerini alacaktır. Başka bir deyişle, bu, anahtarla ilişkili değeri değiştirecektir.
  • Haritalar, haritada bir anahtarın olup olmadığını test etmek, bir anahtarla ilişkili değeri getirmek ve bir anahtar/değer çiftini kaldırmak için hızlı işlemler sağlar.

En sık kullanılan harita uygulaması HashMap‘dir. Dizeler veya sayılar olan tuşlarla iyi çalışır.

HashMap gibi düz haritalar sırasızdır. Anahtar/değer çiftleri üzerinden yineleme, herhangi bir sırayla bireysel girişleri döndürebilir. Harita girişlerini kontrollü bir şekilde yinelemeniz gerekiyorsa, aşağıdakilere bakmalısınız:

  • TreeMap gibi Sıralanmış haritalar, anahtarlar arasında doğal sıralarında (veya bir Karşılaştırıcı sağlayarak sizin belirtebileceğiniz bir sırada) yinelenir. Örneğin, sayıları anahtar olarak kullanan sıralanmış bir haritanın, girişlerini sayısal sırayla yinelemesi beklenir.

  • LinkedHashMap, girişlerin haritaya eklendikleri sırayla veya en son erişilen sırayla yinelenmesine izin verir.

Java 8’den Varsayılan Harita Yöntemlerini Kullanma #

Harita arayüzünde Java 8’de tanıtılan Varsayılan Yöntemleri kullanma örnekleri

  1. getOrDefault‘u kullanma

Anahtarla eşlenen değeri döndürür veya anahtar yoksa varsayılan değeri döndürür

Map<Integer, String> map = new HashMap<>();
map.put(1, "First element");
map.get(1);                                 // => First element
map.get(2);                                 // => null
map.getOrDefault(2, "Default element");     // => Default element
  1. forEach kullanımı

Her Harita Girişinde ’eylemde’ belirtilen işlemi gerçekleştirmeye izin verir

  Map<Integer, String> map = new HashMap<Integer, String>();
  map.put(1, "one");
  map.put(2, "two");
  map.put(3, "three");
  map.forEach((key, value) -> System.out.println("Key: "+key+ " :: Value: "+value));

   // Key: 1 :: Value: one
   // Key: 2 :: Value: two
   // Key: 3 :: Value: three
  1. replaceAll‘ı kullanma

Yalnızca anahtar varsa yeni değerle değiştirilir

 Map<String, Integer> map = new HashMap<String, Integer>();
 map.put("john", 20);
 map.put("paul", 30);
 map.put("peter", 40);
 map.replaceAll((key,value)->value+10);   //{john=30, paul=40, peter=50}
  1. putIfAbsent kullanma

Anahtar mevcut değilse veya boş olarak eşlenmişse, anahtar-değer çifti haritaya eklenir

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.putIfAbsent("kelly", 50);     //{john=20, paul=30, peter=40, kelly=50}
  1. kaldır‘ı kullanma

Anahtarı yalnızca verilen değerle ilişkiliyse kaldırır

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.remove("peter",40); //{john=30, paul=40}

  1. değiştir‘i kullanma

    If the key is present then the value is replaced by new-value. If the key is not present, does nothing.

    Map<String, Integer> map = new HashMap<String, Integer>(); map.put(“john”, 20); map.put(“paul”, 30); map.put(“peter”, 40); map.replace(“peter”,50); //{john=20, paul=30, peter=50} map.replace(“jack”,60); //{john=20, paul=30, peter=50}

  2. computeIfAbsent‘ı kullanma

Bu yöntem, Haritaya bir giriş ekler. anahtar, işlevde belirtilir ve değer, eşleme işlevinin uygulamasının sonucudur.

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.computeIfAbsent("kelly", k->map.get("john")+10); //{john=20, paul=30, peter=40, kelly=30}
    map.computeIfAbsent("peter", k->map.get("john")+10); //{john=20, paul=30, peter=40, kelly=30} //peter already present
  1. computeIfPresent‘i kullanma

Bu yöntem, Haritada bir girdi ekler veya mevcut bir girdiyi değiştirir. Bu anahtara sahip bir giriş yoksa hiçbir şey yapmaz

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.computeIfPresent("kelly", (k,v)->v+10); //{john=20, paul=30, peter=40} //kelly not present
    map.computeIfPresent("peter", (k,v)->v+10); //{john=20, paul=30, peter=50} // peter present, so increase the value
  1. hesaplama kullanımı

Bu yöntem, bir anahtarın değerini yeni hesaplanan değerle değiştirir.

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.compute("peter", (k,v)->v+50); //{john=20, paul=30, peter=90} //Increase the value
  1. birleştirme kullanma

Anahtar yoksa veya anahtarın değeri boşsa anahtar/değer çiftini haritaya ekler Anahtar varsa, değeri yeni hesaplanan değerle değiştirir Hesaplanan yeni değer null ise anahtar haritadan kaldırılır

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    
    //Adds the key-value pair to the map, if key is not present or value for the key is null
    map.merge("kelly", 50 , (k,v)->map.get("john")+10); // {john=20, paul=30, peter=40, kelly=50}
    
    //Replaces the value with the newly computed value, if the key is present
    map.merge("peter", 50 , (k,v)->map.get("john")+10); //{john=20, paul=30, peter=30, kelly=50}
   
    //Key is removed from the map , if new value computed is null
    map.merge("peter", 30 , (k,v)->map.get("nancy")); //{john=20, paul=30, kelly=50}

HashMap Kullanımı #

HashMap, verileri Anahtar-Değer çiftlerinde depolamak için bir Veri Yapısı sağlayan Harita arayüzünün bir uygulamasıdır.

1. HashMap’i Bildirme

Map<KeyType, ValueType> myMap = new HashMap<KeyType, ValueType>();

KeyType ve ValueType, Java’da - String, Integer, Float gibi geçerli türler veya Çalışan, Öğrenci vb. gibi herhangi bir özel sınıf olmalıdır.

Örneğin: Map<String,Integer> myMap = new HashMap<String,Integer>();

2. Değerleri HashMap’e yerleştirme.

HashMap’e bir değer koymak için, Key ve Value’yu parametre olarak geçirerek HashMap nesnesine `put’ yöntemini çağırmalıyız.

myMap.put("key1", 1);
myMap.put("key2", 2);

Put yöntemini Haritada zaten var olan Anahtar ile çağırırsanız, yöntem değerini geçersiz kılar ve eski değeri döndürür.

3. HashMap’ten değerler alma.

Değeri bir HashMap’ten almak için Anahtarı parametre olarak ileterek ‘get’ yöntemini çağırmanız gerekir.

myMap.get("key1");    //return 1 (class Integer)

HashMap’te olmayan bir anahtarı iletirseniz, bu yöntem ’null’ değerini döndürür.

4. Anahtarın Haritada olup olmadığını kontrol edin.

myMap.containsKey(varKey);

5. Değerin Haritada olup olmadığını kontrol edin.

myMap.containsValue(varValue);

Yukarıdaki yöntemler, Haritada anahtar, değer varsa veya yoksa bir “boolean” değeri true veya false döndürür.

Bir Haritanın içeriğini yineleme #

Haritalar, haritanın anahtarlarına, değerlerine veya anahtar/değer çiftlerine koleksiyonlar halinde erişmenizi sağlayan yöntemler sağlar. Bu koleksiyonları yineleyebilirsiniz. Örneğin aşağıdaki harita verildiğinde:

Map<String, Integer> repMap = new HashMap<>();
repMap.put("Jon Skeet", 927_654);
repMap.put("BalusC", 708_826);
repMap.put("Darin Dimitrov", 715_567);

Harita tuşlarıyla yineleme:

for (String key : repMap.keySet()) {
    System.out.println(key);
}

Baskılar:

Darin Dimitrov
Jon Skeet
BalusC

keySet(), haritanın anahtarlarını bir Set olarak sağlar. Anahtarlar yinelenen değerler içeremeyeceğinden “Set” kullanılır. Küme boyunca yineleme, sırayla her anahtarı verir. HashMaps sipariş edilmemiştir, bu nedenle bu örnekte anahtarlar herhangi bir sırayla iade edilebilir.

Harita değerleri üzerinden yineleme:

for (Integer value : repMap.values()) {
    System.out.println(value);
}

Baskılar:

715567
927654
708826

values(), haritanın değerlerini bir Koleksiyon olarak döndürür. Koleksiyon boyunca yineleme, sırayla her değeri verir. Yine, değerler herhangi bir sırayla döndürülebilir.

Anahtarlar ve değerler arasında birlikte yineleme

for (Map.Entry<String, Integer> entry : repMap.entrySet()) {
    System.out.printf("%s = %d\n", entry.getKey(), entry.getValue());
}

Baskılar:

Darin Dimitrov = 715567
Jon Skeet = 927654
BalusC = 708826

entrySet(), Map.Entry nesnelerinin bir koleksiyonunu döndürür. Map.Entry, her giriş için anahtara ve değere erişim sağlar.

Çıktı:

‘3’

Birçok öğe eklemek için bunun gibi bir iç sınıf kullanabilirsiniz:

Map<Integer, String> map = new HashMap<>() {{
    // This is now an anonymous inner class with an unnamed instance constructor
    put(5, "high");
    put(4, "low");
    put(1, "too slow");
}};

Anonim bir iç sınıf oluşturmanın her zaman verimli olmadığını ve bellek sızıntılarına yol açabileceğini unutmayın, bu nedenle mümkün olduğunda bunun yerine bir başlatıcı bloğu kullanın:

static Map<Integer, String> map = new HashMap<>();

static {
    // Now no inner classes are created so we can avoid memory leaks
    put(5, "high");
    put(4, "low");
    put(1, "too slow");
}

Yukarıdaki örnek haritayı statik hale getirir. Tüm ‘statik’ oluşumları kaldırılarak statik olmayan bir bağlamda da kullanılabilir.

Buna ek olarak çoğu uygulama, bir haritadaki tüm girdileri diğerine şu şekilde ekleyebilen ‘putAll’ı destekler:

another.putAll(one);

Haritaları birleştirme, birleştirme ve oluşturma #

Bir haritanın her üyesini diğerine yerleştirmek için ‘putAll’ı kullanın. Haritada zaten mevcut olan anahtarların üzerine karşılık gelen değerlerin üzerine yazılacaktır.

Map<String, Integer> numbers = new HashMap<>();
numbers.put("One", 1)
numbers.put("Three", 3)
Map<String, Integer> other_numbers = new HashMap<>();
other_numbers.put("Two", 2)
other_numbers.put("Three", 4)

numbers.putAll(other_numbers)

Bu, “sayılarda” aşağıdaki eşlemeyi verir:

"One" -> 1
"Two" -> 2
"Three" -> 4 //old value 3 was overwritten by new value 4

Değerlerin üzerine yazmak yerine birleştirmek istiyorsanız, Map.merge kullanabilirsiniz. K-V-java.util.function.BiFunction-), yinelenen anahtarlar için değerleri birleştirmek için kullanıcı tarafından sağlanan bir “BiFunction” kullanan Java 8’e eklendi. “birleştirme”, bireysel anahtarlar ve değerler üzerinde çalışır, bu nedenle bir döngü veya “Map.forEach” kullanmanız gerekir. Burada yinelenen anahtarlar için dizeleri birleştiriyoruz:

for (Map.Entry<String, Integer> e : other_numbers.entrySet())
    numbers.merge(e.getKey(), e.getValue(), Integer::sum);
//or instead of the above loop
other_numbers.forEach((k, v) -> numbers.merge(k, v, Integer::sum));

Kısıtlamayı zorlamak istiyorsanız, yinelenen anahtar yoktur, bir “AssertionError” oluşturan bir birleştirme işlevi kullanabilirsiniz:

mapA.forEach((k, v) ->
    mapB.merge(k, v, (v1, v2) ->
        {throw new AssertionError("duplicate values for key: "+k);}));

Harita<X,Z> almak için Harita<X,Y> ve Harita<Y,Z> Oluşturma #

İki eşleme oluşturmak istiyorsanız aşağıdaki gibi yapabilirsiniz.

    Map<String, Integer> map1 = new HashMap<String, Integer>();
    map1.put("key1", 1);
    map1.put("key2", 2);
    map1.put("key3", 3);

    map2.put(2, 2.0);

    "key1" -> 1.0
    "key2" -> 2.0
    "key3" -> 3.0

Haritalar boş değerler içerebilir #

Haritalar için “anahtar içeren” ile “değeri olan"ı birbirine karıştırmamak için dikkatli olunmalıdır. Örneğin, HashMap’ler null içerebilir, bu da aşağıdakilerin tamamen normal davranış olduğu anlamına gelir:

Map<String, String> map = new HashMap<>();
map.put("one", null);
if (map.containsKey("one")) {
    System.out.println("This prints !"); // This line is reached 
}
if (map.get("one") != null) {
    System.out.println("This is never reached !"); // This line is never reached 
}

Daha resmi olarak, map.contains(key) <=> map.get(key)!=null garantisi yoktur.

Özel nesneyi anahtar olarak kullan #

Kendi nesnenizi anahtar olarak kullanmadan önce, nesnenizin hashCode() ve equals() yöntemini geçersiz kılmalısınız.

Basit bir durumda şöyle bir şeye sahip olursunuz:

class MyKey {
    private String name;
    MyKey(String name) {
        this.name = name;
    }

    @Override
    public boolean equals(Object obj) {            
        if(obj instanceof MyKey) {
            return this.name.equals(((MyKey)obj).name);
        }
        return false;
    }
    
    @Override
    public int hashCode() {
        return this.name.hashCode();
    }
}

“hashCode” anahtarın hangi karma kovaya ait olduğuna karar verecek ve “equals” o karma kovanın içindeki hangi nesneye karar verecek.

Bu yöntem olmadan, her zaman aynı nesne referansını kullanmadığınız sürece, yukarıdaki karşılaştırma için nesnenizin referansı kullanılacaktır.

Haritalar Oluşturma ve Başlatma #

Giriiş #

“Haritalar”, her bir anahtarın ilişkili bir değere sahip olduğu anahtar/değer çiftlerini saklar. Belirli bir anahtar verildiğinde, harita ilgili değeri çok hızlı bir şekilde arayabilir.

İlişkili dizi olarak da bilinen “Haritalar”, verileri anahtarlar ve değerler biçiminde depolayan bir nesnedir. Java’da haritalar, koleksiyon arabiriminin bir uzantısı olmayan Harita arabirimi kullanılarak temsil edilir.
