  • Bir uygulama ve bağımlılıkları için UberJAR oluşturma

Java uygulamalarını, web uygulamalarını vb. üzerinde çalışacakları platforma yerleştirmek için “paketlemek” için çeşitli teknolojiler vardır. Basit kitaplık veya yürütülebilir ‘JAR’ dosyalarından, ‘WAR’ ve ‘EAR’ dosyalarından yükleyicilere ve bağımsız yürütülebilir dosyalara kadar çeşitlilik gösterirler.

En temel düzeyde, bir Java programı, derlenmiş bir sınıfı (yani bir “.class” dosyası) veya derlenmiş sınıfları içeren bir dizin ağacını kopyalayarak konuşlandırılabilir. Ancak Java normalde aşağıdaki yollardan biriyle dağıtılır:

  • Bir JAR dosyasını veya JAR dosyaları koleksiyonunu yürütülecekleri sisteme kopyalayarak; Örneğin. ‘javac’ kullanarak.

  • Bir WAR, EAR veya benzeri bir dosyayı bir “sunucu uygulaması kapsayıcısına” veya “uygulama sunucusuna” kopyalayarak veya yükleyerek.

  • Yukarıdakileri otomatikleştiren bir tür uygulama yükleyici çalıştırarak. Yükleyici, yerleşik bir JRE de kurabilir.

  • Java WebStart kullanılarak başlatılmasına izin vermek için uygulama için JAR dosyalarını bir web sunucusuna koyarak.

JAR, WAR ve EAR dosyaları oluşturma örneği, bu dosyaları oluşturmanın farklı yollarını özetler.

Çok sayıda açık kaynak ve ticari “kurucu oluşturucu” vardır ve Java için “EXE oluşturucu” araçları. Benzer şekilde, Java sınıfı dosyalarını karartmak (tersine mühendisliği zorlaştırmak için) ve çalışma zamanı lisans denetimi eklemek için araçlar vardır. Bunların hepsi “Java Programlama Dili” belgelerinin kapsamı dışındadır.

Komut satırından yürütülebilir bir JAR oluşturma #

Bir kavanoz yapmak için bir veya daha fazla sınıf dosyasına ihtiyacınız var. Çift tıklama ile çalıştırılacaksa, bunun bir ana yöntemi olmalıdır.

Bu örnek için kullanacağız:

import javax.swing.*;
import java.awt.Container;

public class HelloWorld {

    public static void main(String[] args) {
        JFrame f = new JFrame("Hello, World"); 
        JLabel label = new JLabel("Hello, World");
        Container cont = f.getContentPane();
        cont.add(label);
        f.setSize(400,100); 
        f.setVisible(true);
        f.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
    }

}

HelloWorld.java olarak adlandırıldı.

Ardından, bu programı derlemek istiyoruz.

Bunu yapmak için istediğiniz programı kullanabilirsiniz. Komut satırından çalıştırmak için, ilk Java programınızı derlemek ve çalıştırmakla ilgili belgelere bakın.

HelloWorld.class’a sahip olduğunuzda, yeni bir klasör oluşturun ve onu istediğiniz gibi adlandırın.

manifest.txt adlı başka bir dosya oluşturun ve içine yapıştırın

Main-Class: HelloWorld
Class-Path: HelloWorld.jar

HelloWorld.class \ ile aynı klasöre koyun Geçerli dizininizi (pencerelerde cd C:\Your\Klasör\Yol\Burada) klasörünüz yapmak için komut satırını kullanın.

Terminal’i kullanın ve dizini (Mac’te cd /Users/user/Documents/Java/jarfolder) klasörünüze değiştirin

Bu yapıldığında, `jar -cvfm HelloWorld.jar manifest.txt HelloWorld.class’ yazın ve enter tuşuna basın. Bu, belirtilen ve HelloWorld.jar adlı .class dosyalarını kullanarak (bildiriminizin ve HelloWorld.class’ınızın bulunduğu klasörde) bir jar dosyası oluşturur. Seçenekler hakkında bilgi için Sözdizimi bölümüne bakın (-m ve -v gibi).
Bu adımlardan sonra manifest dosyası ile dizininize gidin ve HelloWorld.jar
Üzerine tıklamak, bir metin kutusunda ‘Merhaba, Dünya’yı göstermelidir.

JAR, WAR ve EAR dosyaları oluşturma #

JAR, WAR ve EAR dosya türleri temelde bir “manifest” dosyasına ve (WAR ve EAR dosyaları için) belirli bir dahili dizin/dosya yapısına sahip ZIP dosyalarıdır.

Bu dosyaları oluşturmanın önerilen yolu, ilgili dosya türlerinin gereksinimlerini “anlayan” Java’ya özgü bir oluşturma aracı kullanmaktır. Bir derleme aracı kullanmıyorsanız, denemek için bir sonraki seçenek IDE “dışa aktarma"dır.

(Yazı notu: Bu dosyaların nasıl oluşturulacağına ilişkin açıklamalar en iyi ilgili araçların belgelerinde yer alır. Bunları oraya koyun. Lütfen biraz kendinize hakim olun ve bu örneğe OYNAMAYIN!)

Maven kullanarak JAR ve WAR dosyaları oluşturma #

Maven kullanarak bir JAR veya WAR oluşturmak, yalnızca doğru <packaging> öğesini POM dosyasına yerleştirme meselesidir; Örneğin,

<packaging>jar</packaging>

veya

<packaging>war</packaging>

Daha fazla ayrıntı için. Maven, giriş noktası sınıfı ve harici bağımlılıklar hakkında gerekli bilgileri maven jar eklentisi için eklenti özellikleri olarak ekleyerek “yürütülebilir” JAR dosyaları oluşturacak şekilde yapılandırılabilir. Bir uygulamayı ve bağımlılıklarını tek bir JAR dosyasında birleştiren “uberJAR” dosyaları oluşturmak için bir eklenti bile var.

Daha fazla bilgi için lütfen Maven belgelerine ( https://www.wikiod.com/tr/maven ) bakın.

Ant oluşturma aracının JAR, WAR ve EAR oluşturmak için ayrı “görevleri” vardır. Daha fazla bilgi için lütfen Ant belgelerine ( https://www.wikiod.com/tr/ant ) bakın.

Jar komutunu kullanarak JAR, WAR ve EAR dosyaları oluşturma.

Bu dosyaları jar komutunu kullanarak “elle” oluşturmak da mümkündür. Bu, doğru bileşen dosyalarıyla doğru yerde bir dosya ağacını bir araya getirmek, bir bildirim dosyası oluşturmak ve JAR dosyasını oluşturmak için ‘jar’ı çalıştırmaktan ibarettir.

Önkoşullar #

Yüksek düzeyde Web Start, uzak bir web sunucusundan JAR dosyaları olarak paketlenmiş Java uygulamalarını dağıtarak çalışır. Önkoşullar:

  • Uygulamanın çalıştırılacağı hedef makinede önceden var olan bir Java kurulumu (JRE veya JDK). Java 1.2.2 veya üstü gereklidir:

    • From Java 5.0 onwards, Web Start support is included in the JRE / JDK.
    • For earlier releases, Web Start support is installed separately.
    • The Web Start infrastructure includes some Javascript that can be included in a web page to assist the user to install the necessary software.

  • Yazılımı barındıran web sunucusu, hedef makine tarafından erişilebilir olmalıdır.

  • Kullanıcı bir web sayfasındaki bir bağlantıyı kullanarak bir Web Start uygulamasını başlatacaksa, o zaman: - they need a compatible web browser, and - for modern (secure) browsers, they need to be told how to tell the browser to allow Java to run … without compromising web browser security.

Örnek bir JNLP dosyası #

Aşağıdaki örnek, JNLP’nin temel işlevselliğini göstermeyi amaçlamaktadır.

<?xml version="1.0" encoding="UTF-8"?>
<jnlp spec="1.0+" codebase="https://www.example.com/demo" 
    href="demo_webstart.jnlp">
    <information>
        <title>Demo</title>
        <vendor>The Example.com Team</vendor>
    </information>
    <resources>
        <!-- Application Resources -->
        <j2se version="1.7+" href="http://java.sun.com/products/autodl/j2se"/>
        <jar href="Demo.jar" main="true"/>
    </resources>
    <application-desc
         name="Demo Application"
         main-class="com.example.jwsdemo.Main"
         width="300"
         height="300">
     </application-desc>
     <update check="background"/>
</jnlp>

Gördüğünüz gibi, XML tabanlı bir JNLP dosyasıdır ve bilgilerin tümü <jnlp> öğesinde bulunur.

  • “spec” özelliği, bu dosyanın uyumlu olduğu JNPL spesifikasyonunun sürümünü verir.
  • “codebase” özelliği, dosyanın geri kalanındaki göreli “href” URL’lerini çözümlemek için temel URL’yi verir.
  • “href” özelliği, bu JNLP dosyasının kesin URL’sini verir.
  • <information> öğesi, başlık, yazarlar, açıklama ve yardım web sitesi dahil olmak üzere uygulamanın meta verilerini içerir.
  • “” öğesi, gerekli Java sürümü, işletim sistemi platformu ve JAR dosyaları dahil olmak üzere uygulamanın bağımlılıklarını açıklar.
  • “” (veya “”) öğesi, uygulamayı başlatmak için gereken bilgileri sağlar.

Web sunucusunu ayarlama #

Web sunucusu, ‘.jnlp’ dosyaları için MIMEtype olarak ‘application/x-java-jnlp-file’ kullanmak üzere yapılandırılmalıdır.

JNLP dosyası ve uygulamanın JAR dosyaları, JNLP dosyası tarafından belirtilen URL’ler kullanılarak erişilebilir olmaları için web sunucusuna yüklenmelidir.

Bir web sayfası aracılığıyla başlatmayı etkinleştirme #

Uygulama bir web bağlantısı üzerinden başlatılacaksa, bağlantının bulunduğu sayfa web sunucusunda oluşturulmalıdır.

  • Java Web Start’ın kullanıcının makinesinde zaten yüklü olduğunu varsayarsanız, web sayfasının uygulamayı başlatmak için bir bağlantı içermesi yeterlidir. Örneğin.

     <a href="https://www.example.com/demo_webstart.jnlp">Launch the application</a>

  • Aksi takdirde, sayfa, kullanıcının kullandığı tarayıcının türünü tespit etmek ve gerekli Java sürümünü indirmek ve yüklemek için istekte bulunmak için bazı komut dosyaları da içermelidir.

NOT: Kullanıcıları Java’yı bu şekilde yüklemeye teşvik etmek, hatta JNLP web sayfası başlatmanın çalışabilmesi için web tarayıcılarında Java’yı etkinleştirmeye teşvik etmek kötü bir fikirdir.

Web Start uygulamalarını komut satırından başlatma

Komut satırından bir Web Başlatma uygulamasını başlatma talimatları basittir. Kullanıcının bir Java 5.0 JRE veya JDK yüklü olduğunu varsayarsak, şunu çalıştırmanız yeterlidir:

$ javaws <url>

“” uzak sunucudaki JNLP dosyasının URL’sidir.
