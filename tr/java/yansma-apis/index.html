  • (tam nitelikli) adı verilen Sınıfı Alın
  • Yansıma kullanarak aşırı yüklenmiş kurucuları çağırın
  • İç içe sınıfın çağrı yapıcısı

Yansıma, JVM’de çalışan uygulamaların çalışma zamanı davranışını inceleme veya değiştirme yeteneği gerektiren programlar tarafından yaygın olarak kullanılır. Java Reflection API, derleme zamanında adlarını bilmeden çalışma zamanında sınıfları, arabirimleri, alanları ve yöntemleri incelemeyi mümkün kılan bu amaçla kullanılır. Ayrıca yeni nesneleri somutlaştırmayı ve yansıma kullanan yöntemleri çağırmayı mümkün kılar.

Verim #

Yansımanın performansı düşürebileceğini unutmayın, yalnızca göreviniz yansıma olmadan tamamlanamadığında kullanın.

Java öğreticisinden The Reflection API :

Yansıma, dinamik olarak çözümlenen türleri içerdiğinden, belirli Java sanal makine optimizasyonları gerçekleştirilemez. Sonuç olarak, yansıtıcı işlemler, diğerlerine göre daha yavaş performansa sahiptir. yansıtıcı olmayan muadiller ve kod bölümlerinde kaçınılmalıdır performansa duyarlı uygulamalarda sıklıkla çağrılanlar.

Giriiş #

Temel bilgiler

Reflection API, bir kişinin kodun sınıf yapısını çalışma zamanında kontrol etmesine ve kodu dinamik olarak çağırmasına izin verir. Bu çok güçlüdür, ancak derleyici dinamik çağrıların geçerli olup olmadığını statik olarak belirleyemediğinden tehlikelidir.

Basit bir örnek, belirli bir sınıfın genel oluşturucularını ve yöntemlerini elde etmek olabilir:

import java.lang.reflect.Constructor;
import java.lang.reflect.Method;

// This is a object representing the String class (not an instance of String!)
Class<String> clazz = String.class;

Constructor<?>[] constructors = clazz.getConstructors(); // returns all public constructors of String
Method[] methods = clazz.getMethods(); // returns all public methods from String and parents

Bu bilgi ile nesneyi örneklemek ve dinamik olarak farklı metotları çağırmak mümkündür.

Yansıma ve Genel Türler

Genel tür bilgileri şunlar için mevcuttur:

  • “getGenericParameterTypes()” kullanılarak yöntem parametreleri.
  • getGenericReturnType() kullanarak yöntem dönüş türleri.
  • genel alanlar, “getGenericType” kullanılarak.

Aşağıdaki örnek, her üç durumda da genel tür bilgilerinin nasıl çıkarılacağını gösterir:

import java.lang.reflect.Field;
import java.lang.reflect.Method;
import java.lang.reflect.ParameterizedType;
import java.lang.reflect.Type;
import java.util.List;
import java.util.Map;

public class GenericTest {

    public static void main(final String[] args) throws Exception {
        final Method method = GenericTest.class.getMethod("testMethod", Map.class);
        final Field field = GenericTest.class.getField("testField");

        System.out.println("Method parameter:");
        final Type parameterType = method.getGenericParameterTypes()[0];
        displayGenericType(parameterType, "\t");

        System.out.println("Method return type:");
        final Type returnType = method.getGenericReturnType();
        displayGenericType(returnType, "\t");

        System.out.println("Field type:");
        final Type fieldType = field.getGenericType();
        displayGenericType(fieldType, "\t");

    }

    private static void displayGenericType(final Type type, final String prefix) {
        System.out.println(prefix + type.getTypeName());
        if (type instanceof ParameterizedType) {
            for (final Type subtype : ((ParameterizedType) type).getActualTypeArguments()) {
                displayGenericType(subtype, prefix + "\t");
            }
        }

    }

    public Map<String, Map<Integer, List<String>>> testField;

    public List<Number> testMethod(final Map<String, Double> arg) {
        return null;
    }

}

Bu, aşağıdaki çıktıyla sonuçlanır:

Method parameter:
    java.util.Map<java.lang.String, java.lang.Double>
        java.lang.String
        java.lang.Double
Method return type:
    java.util.List<java.lang.Number>
        java.lang.Number
Field type:
    java.util.Map<java.lang.String, java.util.Map<java.lang.Integer, java.util.List<java.lang.String>>>
        java.lang.String
        java.util.Map<java.lang.Integer, java.util.List<java.lang.String>>
            java.lang.Integer
            java.util.List<java.lang.String>
                java.lang.String

Dinamik Proxy’ler #

Dinamik Proxy’lerin Reflection ile pek ilgisi yoktur, ancak API’nin bir parçasıdırlar. Temel olarak, bir arayüzün dinamik bir uygulamasını yaratmanın bir yoludur. Bu, maket hizmetleri oluştururken yardımcı olabilir.
Dinamik Proxy, tüm yöntem çağrılarını engelleyen ve çağrılarının manuel olarak işlenmesine izin veren bir çağrı işleyicisi ile oluşturulmuş bir arabirim örneğidir.

public class DynamicProxyTest {

    public interface MyInterface1{
        public void someMethod1();
        public int someMethod2(String s);
    }

    public interface MyInterface2{
        public void anotherMethod();
    }
   
    public static void main(String args[]) throws Exception {
        // the dynamic proxy class 
        Class<?> proxyClass = Proxy.getProxyClass(
                ClassLoader.getSystemClassLoader(),
                new Class[] {MyInterface1.class, MyInterface2.class});
        // the dynamic proxy class constructor
        Constructor<?> proxyConstructor = 
            proxyClass.getConstructor(InvocationHandler.class);

        // the invocation handler
        InvocationHandler handler = new InvocationHandler(){
            // this method is invoked for every proxy method call
            // method is the invoked method, args holds the method parameters
            // it must return the method result
            @Override
            public Object invoke(Object proxy, Method method, Object[] args) throws Throwable { 
                String methodName = method.getName();

                if(methodName.equals("someMethod1")){
                    System.out.println("someMethod1 was invoked!");
                    return null;
                }
                if(methodName.equals("someMethod2")){
                    System.out.println("someMethod2 was invoked!");
                    System.out.println("Parameter: " + args[0]);
                    return 42;
                }
                if(methodName.equals("anotherMethod")){
                    System.out.println("anotherMethod was invoked!");
                    return null;
                }
                System.out.println("Unkown method!");
                return null;
            }
        };

        // create the dynamic proxy instances
        MyInterface1 i1 = (MyInterface1) proxyConstructor.newInstance(handler);
        MyInterface2 i2 = (MyInterface2) proxyConstructor.newInstance(handler);

        // and invoke some methods
        i1.someMethod1();
        i1.someMethod2("stackoverflow");
        i2.anotherMethod();
    }
}

Bu kodun sonucu şudur:

someMethod1 was invoked!
someMethod2 was invoked!
Parameter: stackoverflow
anotherMethod was invoked!

Alma ve Ayarlama alanları #

Reflection API’sini kullanarak, çalışma zamanında bir alanın değerini değiştirmek veya almak mümkündür. Örneğin, işletim sistemi gibi bir faktöre dayalı olarak farklı alanları almak için bir API’de kullanabilirsiniz. Son olan alanların değiştirilmesine izin vermek için “final” gibi değiştiricileri de kaldırabilirsiniz.

Bunu yapmak için, Class#getField() yöntemini aşağıda gösterildiği gibi kullanmanız gerekir:

// Get the field in class SomeClass "NAME".
Field nameField = SomeClass.class.getDeclaredField("NAME");

// Get the field in class Field "modifiers". Note that it does not 
// need to be static
Field modifiersField = Field.class.getDeclaredField("modifiers");

// Allow access from anyone even if it's declared private
modifiersField.setAccessible(true);

// Get the modifiers on the "NAME" field as an int.
int existingModifiersOnNameField = nameField.getModifiers();

// Bitwise AND NOT Modifier.FINAL (16) on the existing modifiers
// Readup here https://en.wikipedia.org/wiki/Bitwise_operations_in_C
// if you're unsure what bitwise operations are.
int newModifiersOnNameField = existingModifiersOnNameField & ~Modifier.FINAL;

// Set the value of the modifiers field under an object for non-static fields
modifiersField.setInt(nameField, newModifiersOnNameField);

// Set it to be accessible. This overrides normal Java 
// private/protected/package/etc access control checks.
nameField.setAccessible(true);

 // Set the value of "NAME" here. Note the null argument. 
 // Pass null when modifying static fields, as there is no instance object
nameField.set(null, "Hacked by reflection...");

// Here I can directly access it. If needed, use reflection to get it. (Below)
System.out.println(SomeClass.NAME);

Alan almak çok daha kolay. Değerini elde etmek için Field#get() ve türevlerini kullanabiliriz:

// Get the field in class SomeClass "NAME".
Field nameField = SomeClass.class.getDeclaredField("NAME");

// Set accessible for private fields
nameField.setAccessible(true);

// Pass null as there is no instance, remember?
String name = (String) nameField.get(null);

Bunu not edin:

Class#getDeclaredField kullanırken, sınıfın kendisinde bir alan almak için kullanın:

class HackMe extends Hacked {
    public String iAmDeclared;
}

class Hacked {
    public String someState;
}

Burada HackMe#iAmDeclared alanı ilan edilir. Ancak, ‘HackMe#someState’, üst sınıfı Hacked’den miras alındığından beyan edilmiş bir alan değildir.

Özel ve nihai değişkenleri değiştirmek için Reflection API’sinin yanlış kullanımı #

Yansıma, doğru amaç için uygun şekilde kullanıldığında yararlıdır. Yansımayı kullanarak özel değişkenlere erişebilir ve son değişkenleri yeniden başlatabilirsiniz.

Aşağıda önerilmeyen kod parçacığı verilmiştir.

import java.lang.reflect.*;

public class ReflectionDemo{
    public static void main(String args[]){
        try{
            Field[] fields = A.class.getDeclaredFields();
            A a = new A();
            for ( Field field:fields ) {
                if(field.getName().equalsIgnoreCase("name")){
                    field.setAccessible(true);
                    field.set(a, "StackOverFlow");
                    System.out.println("A.name="+field.get(a));
                }
                if(field.getName().equalsIgnoreCase("age")){
                    field.set(a, 20);
                    System.out.println("A.age="+field.get(a));
                }
                if(field.getName().equalsIgnoreCase("rep")){
                    field.setAccessible(true);
                    field.set(a,"New Reputation");
                    System.out.println("A.rep="+field.get(a));
                }
                if(field.getName().equalsIgnoreCase("count")){
                    field.set(a,25);
                    System.out.println("A.count="+field.get(a));
                }
            }                
        }catch(Exception err){
            err.printStackTrace();
        }
    }
}

class A {
    private String name;
    public int age;
    public final String rep;
    public static int count=0;
    
    public A(){
        name = "Unset";
        age = 0;
        rep = "Reputation";
        count++;
    }
}

Çıktı:

A.name=StackOverFlow
A.age=20
A.rep=New Reputation
A.count=25

Açıklama:

Normal senaryoda, ‘private’ değişkenlere bildirilen sınıfın dışında erişilemez ( getter ve setter yöntemleri olmadan). “final” değişkenleri, başlatmadan sonra yeniden atanamaz.

‘Yansıma’, yukarıda açıklandığı gibi hem özel hem de nihai değişkenleri değiştirmek için her iki engeli de ihlal edebilir.

field.setAccessible(true) istenen işlevselliği elde etmenin anahtarıdır.

Reflection ile Kötü Java hileleri #

Reflection API, JDK varsayılan kitaplığında bile özel ve son alanların değerlerini değiştirmek için kullanılabilir. Bu, göreceğimiz gibi, iyi bilinen bazı sınıfların davranışını değiştirmek için kullanılabilir.

Ne mümkün değil

İlk önce tek sınırlama ile başlayalım, Reflection ile değiştiremeyeceğimiz tek alan anlamına gelir. Java SecurityManager budur. Java.lang.System içinde şu şekilde bildirildi:

private static volatile SecurityManager security = null;

Ancak bu kodu çalıştırırsak System sınıfında listelenmeyecektir.

for(Field f : System.class.getDeclaredFields())
    System.out.println(f);

Bunun nedeni, haritanın kendisini ve güvenlik alanını tutan sun.reflect.Reflection içindeki fieldFilterMap System.class içinde ve onları Reflection ile herhangi bir erişime karşı korur. Bu yüzden ‘SecurityManager’ı devre dışı bırakamadık.

Çılgın Dizeler

Her Java Dizisi, JVM tarafından “String” sınıfının bir örneği olarak temsil edilir. Ancak, bazı durumlarda JVM, Dizeler için aynı örneği kullanarak yığın alanından tasarruf sağlar. Bu, dize değişmezleri için ve ayrıca String.intern() çağrılarak “interne edilmiş” dizeler için olur. Bu nedenle, kodunuzda birden çok kez “merhaba” varsa, bu her zaman aynı nesne örneğidir.

Dizelerin değişmez olması gerekir, ancak onları değiştirmek için “kötü” yansımayı kullanmak mümkündür. Aşağıdaki örnek, ‘değer’ alanını değiştirerek bir String’deki karakterleri nasıl değiştirebileceğimizi göstermektedir.

public class CrazyStrings {
    static {
        try {
            Field f = String.class.getDeclaredField("value");
            f.setAccessible(true);
            f.set("hello", "you stink!".toCharArray());
        } catch (Exception e) {
        }
    }
    public static void main(String args[])  {
        System.out.println("hello");
    }
}

Yani bu kod “Sen kokuyorsun!” Yazacak.

1 = 42

Aynı fikir Tamsayı Sınıfı ile de kullanılabilir

public class CrazyMath {
    static {
        try {
            Field value = Integer.class.getDeclaredField("value");    
            value.setAccessible(true);          
            value.setInt(Integer.valueOf(1), 42);      
        } catch (Exception e) {
        }
    }
    public static void main(String args[])  {
        System.out.println(Integer.valueOf(1));
    }
}

Her şey doğru

Ve bu yığın akışı gönderisine göre, gerçekten kötü bir şey yapmak için yansımayı kullanabiliriz.

public class Evil {    
    static {
        try {
            Field field = Boolean.class.getField("FALSE");
            field.setAccessible(true);
            Field modifiersField = Field.class.getDeclaredField("modifiers");
            modifiersField.setAccessible(true);
            modifiersField.setInt(field, field.getModifiers() & ~Modifier.FINAL);
            field.set(null, true);
        } catch (Exception e) {
        }
    }
    public static void main(String args[]){
        System.out.format("Everything is %s", false);
    }
}

Burada yaptığımız şeyin JVM’nin anlaşılmaz şekillerde davranmasına neden olacağına dikkat edin. Bu çok tehlikeli.

Çağrı yapıcı #

Yapıcı Nesnesini Alma #

‘Sınıf’ nesnesinden ‘Yapıcı’ sınıfını şu şekilde elde edebilirsiniz:

Class myClass = ... // get a class object
Constructor[] constructors = myClass.getConstructors();

‘constructors’ değişkeninin, sınıfta bildirilen her bir public kurucu için bir ‘Constructor’ örneğine sahip olacağı yer.

Erişmek istediğiniz yapıcının kesin parametre türlerini biliyorsanız, belirli yapıcıyı filtreleyebilirsiniz. Sonraki örnek, parametre olarak bir “Tamsayı” alan, verilen sınıfın genel oluşturucusunu döndürür:

Class myClass = ... // get a class object
Constructor constructor = myClass.getConstructor(new Class[]{Integer.class});

Hiçbir yapıcı, verilen yapıcı bağımsız değişkenleriyle eşleşmezse, bir `NoSuchMethodException’ atılır.

Yapıcı Nesnesini Kullanan Yeni Örnek #

Class myClass = MyObj.class // get a class object
Constructor constructor = myClass.getConstructor(Integer.class);
MyObj myObj = (MyObj) constructor.newInstance(Integer.valueOf(123));

Bir yöntemi çağırmak #

Yansıma kullanılarak, çalışma zamanı sırasında bir nesnenin yöntemi çağrılabilir.

Örnek, bir ‘String’ nesnesinin yöntemlerinin nasıl çağrılacağını gösterir.

import java.lang.reflect.InvocationTargetException;
import java.lang.reflect.Method;

String s = "Hello World!";

// method without parameters
// invoke s.length()
Method method1 = String.class.getMethod("length");
int length = (int) method1.invoke(s); // variable length contains "12"

// method with parameters
// invoke s.substring(6)
Method method2 = String.class.getMethod("substring", int.class);
String substring = (String) method2.invoke(s, 6); // variable substring contains "World!"

Bir Numaralandırmanın Sabitlerini Alma #

Bu numaralandırmayı Örnek olarak vermek:

enum Compass {
    NORTH(0),
    EAST(90),
    SOUTH(180),
    WEST(270);
    private int degree;
    Compass(int deg){
        degree = deg;
    }
    public int getDegree(){
        return degree;
    }
}

Java’da bir enum sınıfı diğer sınıflar gibidir ancak enum değerleri için tanımlanmış bazı sabitlere sahiptir. Ek olarak, tüm değerleri tutan bir dizi olan bir alana ve ‘values()’ ve ‘valueOf(String)’ adlı iki statik yönteme sahiptir.
Bu sınıftaki tüm alanları yazdırmak için Reflection kullanırsak bunu görebiliriz.

for(Field f : Compass.class.getDeclaredFields())
    System.out.println(f.getName());

çıktı olacaktır:

KUZEY
Doğu
GÜNEY
BATI
derece
ENUM$VALUES

Böylece enum sınıflarını diğer sınıflar gibi Reflection ile inceleyebiliriz. Ancak Reflection API, numaralandırmaya özgü üç yöntem sunar.

sayı kontrolü

Compass.class.isEnum();

Bir numaralandırma türünü temsil eden sınıflar için true değerini döndürür.

değerleri alma

Object[] values = Compass.class.getEnumConstants();

Compass.values() gibi tüm enum değerlerinin bir dizisini döndürür, ancak bir örneğe ihtiyaç duymaz.

enum sabit kontrolü

for(Field f : Compass.class.getDeclaredFields()){
    if(f.isEnumConstant())
        System.out.println(f.getName());
}

Enum değerleri olan tüm sınıf alanlarını listeler.

(tam nitelikli) adı verilen Sınıfı Alın #

Bir sınıfın adını içeren bir “Dize” verildiğinde, “Class” nesnesine “Class.forName” kullanılarak erişilebilir:

Class clazz = null;
try {
    clazz = Class.forName("java.lang.Integer");
} catch (ClassNotFoundException ex) {
    throw new IllegalStateException(ex);
}

Sınıfın başlatılıp başlatılmayacağı (‘forName’in ikinci parametresi) ve hangi ‘ClassLoader’ın kullanılacağı (üçüncü parametre) belirtilebilir:

ClassLoader classLoader = ...
boolean initialize = ...
Class clazz = null;
try {
    clazz = Class.forName("java.lang.Integer", initialize, classLoader);
} catch (ClassNotFoundException ex) {
    throw new IllegalStateException(ex);
}
son sürüm

Yansıma kullanarak aşırı yüklenmiş kurucuları çağırın #

Örnek: İlgili parametreleri ileterek farklı kurucuları çağırın

import java.lang.reflect.*;

class NewInstanceWithReflection{
    public NewInstanceWithReflection(){
        System.out.println("Default constructor");
    }
    public NewInstanceWithReflection( String a){
        System.out.println("Constructor :String => "+a);
    }
    public static void main(String args[]) throws Exception {
        
        NewInstanceWithReflection object = (NewInstanceWithReflection)Class.forName("NewInstanceWithReflection").newInstance();
        Constructor constructor = NewInstanceWithReflection.class.getDeclaredConstructor( new Class[] {String.class});
        NewInstanceWithReflection object1 = (NewInstanceWithReflection)constructor.newInstance(new Object[]{"StackOverFlow"});
        
    }
}

çıktı:

Default constructor
Constructor :String => StackOverFlow

Açıklama:

  1. Class.forName kullanarak sınıf örneği oluşturun: Varsayılan kurucuyu çağırır
  2. Parametre türlerini ‘Sınıf dizisi’ olarak ileterek sınıfın ‘getDeclaredConstructor’ını çağırın
  3. Yapıcıyı aldıktan sonra, parametre değerini ‘Object array’ olarak ileterek ’newInstance’ oluşturun.

İç içe sınıfın çağrı yapıcısı #

İç içe geçmiş bir sınıfın örneğini oluşturmak istiyorsanız, Class#getDeclaredConstructor ile ek bir parametre olarak çevreleyen sınıfın bir sınıf nesnesini sağlamanız gerekir.

public class Enclosing{
    public class Nested{
    public Nested(String a){
            System.out.println("Constructor :String => "+a);
        }
    }       
    public static void main(String args[]) throws Exception {
        Class<?> clazzEnclosing = Class.forName("Enclosing");            
        Class<?> clazzNested = Class.forName("Enclosing$Nested");
        Enclosing objEnclosing = (Enclosing)clazzEnclosing.newInstance();
        Constructor<?> constructor = clazzNested.getDeclaredConstructor(new Class[]{Enclosing.class, String.class});
        Nested objInner = (Nested)constructor.newInstance(new Object[]{objEnclosing, "StackOverFlow"});
    }
}

İç içe sınıf statik ise, bu çevreleyen örneğe ihtiyacınız olmaz.
