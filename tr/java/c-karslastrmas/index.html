Geçersiz Kılma vs Aşırı Yükleme #

Aşağıdaki Geçersiz Kılma ve Aşırı Yükleme noktaları hem C++ hem de Java için geçerlidir:

  • Geçersiz kılınan bir yöntem, tabanıyla aynı ada ve argümanlara sahiptir yöntem.
  • Aşırı yüklenmiş bir yöntem aynı ada sahip ancak farklı argümanlar ve mirasa dayanmaz.
  • Aynı ada ve bağımsız değişkenlere sahip ancak farklı dönüş tipine sahip iki yöntem geçersizdir. “Java’da farklı dönüş türleriyle aşırı yükleme” ile ilgili Stackoverflow sorularına bakın - [Soru 1](https://stackoverflow.com/questions/2439782/overload-with- Different-return-type-in-java); [Soru 2](https://stackoverflow.com/questions/5561436/can-two-java-methods-have-same-name-with- Different-return-types)

Polimorfizm #

Polimorfizm, kalıtımla ilişkili farklı sınıflardaki nesnelerin aynı yöntem çağrısına farklı yanıt verme yeteneğidir. İşte bir örnek:

  • temel sınıf Soyut bir yöntem olarak alanlı şekil
  • türetilmiş iki sınıf, Square ve Circle, alan yöntemlerini uygular
  • Kare ve alan referans noktalarının şekli çağrılır

C++‘da polimorfizm sanal yöntemlerle etkinleştirilir. Java’da yöntemler varsayılan olarak sanaldır.

Yapım/Yıkım Sırası #

Nesne Temizleme #

C++‘da, temel sınıf işaretçisi silinirse alt sınıfın yıkıcısının çağrılmasını sağlamak için bir yıkıcıyı sanal olarak bildirmek iyi bir fikirdir.

Java’da, bir sonlandırma yöntemi, C++‘daki bir yıkıcıya benzer; ancak kesinleştiriciler tahmin edilemez (GC’ye güvenirler). En iyi uygulama - açıkça temizlemek için “kapat” yöntemini kullanın.

protected void close() {
    try {
       // do subclass cleanup
    }
    finally {
       isClosed = true;
       super.close();
    }
}

protected void finalize() {
    try {
       if(!isClosed) close();
    }
    finally {
       super.finalize();
    }
}

Korkunç Elmas Problemi #

Elmas sorunu, iki sınıf B ve C, A’dan miras alır ve D sınıfı hem B’den hem de C’den miras alır. A’da B ve C’nin geçersiz kıldığı ve D’nin geçersiz kılmadığı bir yöntem o, o zaman D yöntemin hangi sürümünü miras alır: B’ninki veya C? (Wikipedia’dan)

C++ elmas sorununa her zaman duyarlıyken, Java Java 8’e kadar hassastı. Başlangıçta Java çoklu kalıtımı desteklemiyordu, ancak varsayılan arabirim yöntemlerinin ortaya çıkmasıyla Java sınıfları birden fazla sınıftan “uygulamayı” miras alamaz. .

java.lang.Object Sınıfı #

Java’da tüm sınıflar, örtük veya açık olarak Object sınıfından miras alır. Herhangi bir Java referansı, Nesne türüne dönüştürülebilir.

C++, karşılaştırılabilir bir “Object” sınıfına sahip değildir.

Statik Sınıf Üyeleri #

Statik üyeler, nesne kapsamının aksine sınıf kapsamına sahiptir

C++ Örnek #

// define in header
class Singleton {
   public:
      static Singleton *getInstance();

   private:
      Singleton() {}
      static Singleton *instance;
};

// initialize in .cpp
Singleton* Singleton::instance = 0;

Java Örnek #

public class Singleton {
    private static Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if(instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

Değere göre ve referansa göre geçiş #

Birçoğu, Java’nın SADECE değere göre olduğunu iddia ediyor, ancak bundan daha nüanslı. Değere göre geçiş (diğer adıyla kopya) ve başvuruya göre geçişin (diğer adı) birçok çeşidini görmek için aşağıdaki C++ ve Java örneklerini karşılaştırın.

C++ Örneği (kodun tamamı) #

  // passes a COPY of the object
  static void passByCopy(PassIt obj) {
     obj.i = 22;  // only a "local" change
  }

  // passes a pointer
  static void passByPointer(PassIt* ptr) {
     ptr->i = 33;
     ptr = 0; // better to use nullptr instead if '0'
  }

  // passes an alias (aka reference)
  static void passByAlias(PassIt& ref) {
     ref.i = 44;
  }

  // This is an old-school way of doing it.
  // Check out std::swap for the best way to do this
  static void swap(PassIt** pptr1, PassIt** pptr2) {
     PassIt* tmp = *pptr1;
     *pptr1 = *pptr2;
     *pptr2 = tmp;
  }

Java Örneği (kodun tamamı) #

   // passes a copy of the variable
   // NOTE: in java only primitives are pass-by-copy
   public static void passByCopy(int copy) {
      copy = 33;  // only a "local" change
   }

   // No such thing as pointers in Java
   /*
   public static void passByPointer(PassIt *ptr) {
      ptr->i = 33;
      ptr = 0; // better to use nullptr instead if '0'
   }
   */

   // passes an alias (aka reference)
   public static void passByAlias(PassIt ref) {
      ref.i = 44;
   }

   // passes aliases (aka references),
   // but need to do "manual", potentially expensive copies
   public static void swap(PassIt ref1, PassIt ref2) {
      PassIt tmp = new PassIt(ref1);
      ref1.copy(ref2);
      ref2.copy(tmp);
   }

Kalıtım ve Kompozisyon #

C++ ve Java her ikisi de nesne yönelimli dillerdir, bu nedenle aşağıdaki diyagram her ikisi için de geçerlidir.

Outcast Downcasting #

“Aşağıya yayın” kullanmaya dikkat edin - Downcasting, kalıtım hiyerarşisini bir temel sınıftan bir alt sınıfa (yani polimorfizmin tersi) düşürmektir. Genel olarak, instanceof & downcasting yerine polimorfizm & geçersiz kılma kullanın.

C++ Örneği #

// explicit type case required
Child *pChild =  (Child *) &parent;

Java Örneği #

if(mySubClass instanceof SubClass) {
   SubClass mySubClass = (SubClass)someBaseClass;
   mySubClass.nonInheritedMethod();
}
