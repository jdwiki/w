polimorfizm

Polimorfizm, ana OOP (nesne yönelimli programlama) kavramlarından biridir. Polimorfizm kelimesi Yunanca “poly” ve “morphs” kelimelerinden türetilmiştir. Poly “birçok” anlamına gelir ve morphs “formlar” (birçok form) anlamına gelir.

Polimorfizmi gerçekleştirmenin iki yolu vardır. Yöntem Aşırı Yükleme ve Yöntem Geçersiz Kılma.

Arayüzler, sınıf tabanlı kalıtım dışında Java’da polimorfizm elde etmenin başka bir yoludur. Arayüzler, programın API’sini oluşturan yöntemlerin bir listesini tanımlar. Sınıflar, tüm yöntemlerini geçersiz kılarak bir “arayüz” ‘uygulamalı’.

Yöntemi Geçersiz Kılma #

Yöntem geçersiz kılma, alt tiplerin üst tiplerinin davranışını yeniden tanımlama (geçersiz kılma) yeteneğidir.

Java’da bu, süper sınıfta tanımlanan yöntemleri geçersiz kılan alt sınıflara çevrilir. Java’da, tüm ilkel olmayan değişkenler aslında, bellekteki gerçek nesnenin konumuna işaretçilere benzeyen “referanslardır”. “Referanslar” yalnızca bir türe sahiptir, bu da bildirildikleri türdür. Ancak, beyan ettikleri türdeki veya alt türlerinden herhangi bir nesneye işaret edebilirler.

Bir “başvuru” üzerinde bir yöntem çağrıldığında, karşılık gelen gösterilen gerçek nesnenin yöntemi çağrılır.

class SuperType {
    public void sayHello(){
        System.out.println("Hello from SuperType");
    }

    public void sayBye(){
        System.out.println("Bye from SuperType");
    }
}

class SubType extends SuperType {
    // override the superclass method
    public void sayHello(){
        System.out.println("Hello from SubType");
    }
}

class Test {
    public static void main(String... args){
        SuperType superType = new SuperType();
        superType.sayHello(); // -> Hello from SuperType

        // make the reference point to an object of the subclass
        superType = new SubType();
        // behaviour is governed by the object, not by the reference
        superType.sayHello(); // -> Hello from SubType

        // non-overridden method is simply inherited
        superType.sayBye(); // -> Bye from SuperType
    }
}

Akılda tutulması gereken kurallar

Alt sınıftaki bir yöntemi geçersiz kılmak için, geçersiz kılma yöntemi (yani alt sınıftaki yöntem) ** MUTLAKA OLMALIDIR**:

  • aynı isim
  • ilkellerde aynı dönüş türü (sınıflar için bir alt sınıfa izin verilir, bu aynı zamanda kovaryant dönüş türleri olarak da bilinir).
  • aynı tip ve parametre sırası
  • yalnızca üst sınıfın yönteminin throws yan tümcesinde bildirilen istisnaları veya beyan edilen istisnaların alt sınıfları olan istisnaları atabilir. Ayrıca herhangi bir istisna atmamayı da seçebilir. The names of the parameter types do not matter. For example, void methodX(int i) is same as void methodX(int k)
  • Nihai veya Statik yöntemleri geçersiz kılamıyoruz. Yapabileceğimiz tek şey, yalnızca yöntem gövdesini değiştirmek.

Yöntem Aşırı Yüklemesi #

İşlev aşırı yüklemesi olarak da bilinen yöntem aşırı yükleme, bir sınıfın aynı ada sahip birden çok yönteme sahip olma yeteneğidir ve bunların bağımsız değişken sayısı veya türü bakımından farklılık gösterir.

Derleyici, yöntem aşırı yüklemesi için yöntem imzasını kontrol eder.

Yöntem imzası üç şeyden oluşur -

  1. Yöntem adı
  2. Parametre sayısı
  3. Parametre türleri

Bu üçü bir sınıftaki herhangi iki yöntem için aynıysa, derleyici yinelenen yöntem hatası verir.

Bu tür polimorfizm statik veya derleme zamanı polimorfizmi olarak adlandırılır, çünkü çağrılacak uygun yöntem, derleme zamanı sırasında argüman listesine dayalı olarak derleyici tarafından kararlaştırılır.

class Polymorph {

    public int add(int a, int b){
        return a + b;
    }
    
    public int add(int a, int b, int c){
        return a + b + c;
    }

    public float add(float a, float b){
        return a + b;
    }

    public static void main(String... args){
        Polymorph poly = new Polymorph();
        int a = 1, b = 2, c = 3;
        float d = 1.5, e = 2.5;

        System.out.println(poly.add(a, b));
        System.out.println(poly.add(a, b, c));
        System.out.println(poly.add(d, e));
    }

}

Bu, aşağıdakilerle sonuçlanacaktır:

2
6
4.000000

Aşırı yüklenmiş yöntemler statik veya statik olmayabilir. Bu aynı zamanda yöntem aşırı yüklemesini de etkilemez.

public class Polymorph {

    private static void methodOverloaded()
    {
        //No argument, private static method
    }
 
    private int methodOverloaded(int i)
    {
        //One argument private non-static method

    }
 
    static int methodOverloaded(double d)
    {
        //static Method
        return 0;
    }
 
    public void methodOverloaded(int i, double d)

Polimorfizm ve farklı geçersiz kılma türleri #

Java’dan eğitici

Polimorfizmin sözlük tanımı, bir organizmanın veya türün birçok farklı form veya aşamaya sahip olabileceği biyolojideki bir ilkeye atıfta bulunur. Bu ilke, nesne yönelimli programlamaya ve Java dili gibi dillere de uygulanabilir. Bir sınıfın alt sınıfları kendi benzersiz davranışlarını tanımlayabilir ve yine de üst sınıfın aynı işlevselliklerinden bazılarını paylaşabilir.

Farklı geçersiz kılma türlerini anlamak için bu örneğe bir göz atın.

  1. Temel sınıf uygulama sağlamaz ve alt sınıfın tam yöntemi geçersiz kılması gerekir - (soyut)
  2. Temel sınıf varsayılan uygulama sağlar ve alt sınıf davranışı değiştirebilir
  3. Alt sınıf, ilk ifade olarak super.methodName() öğesini çağırarak temel sınıf uygulamasına uzantı ekler
  4. Temel sınıf, algoritmanın yapısını tanımlar (Şablon yöntemi) ve alt sınıf, algoritmanın bir bölümünü geçersiz kılar

kod parçacığı:

import java.util.HashMap;

abstract class Game implements Runnable{

    protected boolean runGame = true;
    protected Player player1 = null;
    protected Player player2 = null;
    protected Player currentPlayer = null;
    
    public Game(){
        player1 = new Player("Player 1");
        player2 = new Player("Player 2");
        currentPlayer = player1;
        initializeGame();
    }

    /* Type 1: Let subclass define own implementation. Base class defines abstract method to force
        sub-classes to define implementation    
    */
    
    protected abstract void initializeGame();
    
    /* Type 2: Sub-class can change the behaviour. If not, base class behaviour is applicable */
    protected void logTimeBetweenMoves(Player player){
        System.out.println("Base class: Move Duration: player.PlayerActTime - player.MoveShownTime");
    }
    
    /* Type 3: Base class provides implementation. Sub-class can enhance base class implementation by calling
        super.methodName() in first line of the child class method and specific implementation later */
    protected void logGameStatistics(){
        System.out.println("Base class: logGameStatistics:");
    }
    /* Type 4: Template method: Structure of base class can't be changed but sub-class can some part of behaviour */
    protected void runGame() throws Exception{
        System.out.println("Base class: Defining the flow for Game:");    
        while (runGame) {
            /*
            1. Set current player
            2. Get Player Move
            */
            validatePlayerMove(currentPlayer);    
            logTimeBetweenMoves(currentPlayer);
            Thread.sleep(500);
            setNextPlayer();
        }
        logGameStatistics();
    }
    /* sub-part of the template method, which define child class behaviour */
    protected abstract void validatePlayerMove(Player p);
    
    protected void setRunGame(boolean status){
        this.runGame = status;
    }
    public void setCurrentPlayer(Player p){
        this.currentPlayer = p;
    }
    public void setNextPlayer(){
        if (currentPlayer == player1) {
            currentPlayer = player2;
        }else{
            currentPlayer = player1;
        }
    }
    public void run(){
        try{
            runGame();
        }catch(Exception err){
            err.printStackTrace();
        }
    }
}

class Player{
    String name;
    Player(String name){
        this.name = name;
    }
    public String getName(){
        return name;
    }
}

/* Concrete Game implementation  */
class Chess extends Game{
    public Chess(){
        super();
    }
    public void initializeGame(){
        System.out.println("Child class: Initialized Chess game");
    }
    protected void validatePlayerMove(Player p){
        System.out.println("Child class: Validate Chess move:" + p.getName());
    }
    protected void logGameStatistics(){
        super.logGameStatistics();
        System.out.println("Child class: Add Chess specific logGameStatistics:");
    }
}
class TicTacToe extends Game{
    public TicTacToe(){
        super();
    }
    public void initializeGame(){
        System.out.println("Child class: Initialized TicTacToe game");
    }
    protected void validatePlayerMove(Player p){
        System.out.println("Child class: Validate TicTacToe move:" + p.getName());
    }
}

public class Polymorphism{
    public static void main(String args[]){
        try{
        
            Game game = new Chess();
            Thread t1 = new Thread(game);
            t1.start();
            Thread.sleep(1000);
            game.setRunGame(false);
            Thread.sleep(1000);
                        
            game = new TicTacToe();
            Thread t2 = new Thread(game);
            t2.start();
            Thread.sleep(1000);
            game.setRunGame(false);
        
        }catch(Exception err){
            err.printStackTrace();
        }        
    }
}

Mevcut koda dokunmadan sınıf ekleyerek davranış ekleme #

import java.util.ArrayList;
import java.util.List;

import static java.lang.System.out;

public class PolymorphismDemo {

    public static void main(String[] args) {
        List<FlyingMachine> machines = new ArrayList<FlyingMachine>();
        machines.add(new FlyingMachine());
        machines.add(new Jet());
        machines.add(new Helicopter());
        machines.add(new Jet());

        new MakeThingsFly().letTheMachinesFly(machines);
    }
}

class MakeThingsFly {
    public void letTheMachinesFly(List<FlyingMachine> flyingMachines) {
        for (FlyingMachine flyingMachine : flyingMachines) {
            flyingMachine.fly();
        }
    }
}

class FlyingMachine {
    public void fly() {
        out.println("No implementation");
    }
}

class Jet extends FlyingMachine {
    @Override
    public void fly() {
        out.println("Start, taxi, fly");
    }

    public void bombardment() {
        out.println("Fire missile");
    }
}

class Helicopter extends FlyingMachine {
    @Override
    public void fly() {
        out.println("Start vertically, hover, fly");
    }
}

Sanal işlevler #

Sanal Yöntemler, Java’da statik olmayan ve başında Final anahtar sözcüğü olmayan yöntemlerdir. Java’da varsayılan olarak tüm yöntemler sanaldır. Sanal Yöntemler Polimorfizmde önemli roller oynar çünkü Java’daki alt sınıflar, geçersiz kılınan işlev statik değilse ve aynı yöntem imzasına sahipse üst sınıfların yöntemlerini geçersiz kılabilir.

Bununla birlikte, sanal olmayan bazı yöntemler vardır. Örneğin, yöntem özel olarak veya final anahtar sözcüğüyle bildirilmişse, yöntem Sanal değildir.

Bu StackOverflow gönderisinden http://stackoverflow.com/questions/460446/how-do-virtual-functions-work-in-c-sharp-and-java) Sanal Yöntemlerle aşağıdaki değiştirilmiş kalıtım örneğini göz önünde bulundurun:

public class A{
    public void hello(){
        System.out.println("Hello");
    }
    
    public void boo(){
        System.out.println("Say boo");

    }
}

public class B extends A{
     public void hello(){
        System.out.println("No");
     }
    
    public void boo(){
        System.out.println("Say haha");

    }
}

B sınıfını çağırır ve merhaba() ve boo()‘yu çağırırsak, sonuç çıktısı olarak “Hayır” ve “Hahaha Deyin” alırız çünkü B, A’dan gelen aynı yöntemleri geçersiz kılar. Yukarıdaki örnek neredeyse tamamen aynı olsa da yöntemi geçersiz kılmak için, A sınıfındaki yöntemlerin hepsinin varsayılan olarak Sanal olduğunu anlamak önemlidir.

Ek olarak, soyut anahtar kelimeyi kullanarak Sanal yöntemleri uygulayabiliriz. “abstract” anahtar sözcüğüyle bildirilen yöntemlerin bir yöntem tanımı yoktur, yani yöntemin gövdesi henüz uygulanmamıştır. boo() yönteminin soyut olarak bildirilmesi dışında, yukarıdaki örneği tekrar düşünün:

public class A{
   public void hello(){
        System.out.println("Hello");
    }
    
    abstract void boo();
}

public class B extends A{
     public void hello(){
        System.out.println("No");
     }
    
    public void boo(){
        System.out.println("Say haha");

    }
}

B’den boo() işlevini çağırırsak, B, soyut boo() yöntemini devraldığı ve boo() çıktısını “Hahaha Deyin” yaptığı için çıktı yine “Hahaha Deyin” olacaktır.
