Java kaynak dosyalarını bytecode dosyalarına derlemek için ‘javac’ komutu kullanılır. Bayt kodu dosyaları platformdan bağımsızdır. Bu, kodunuzu bir tür donanım ve işletim sisteminde derleyebileceğiniz ve ardından kodu Java’yı destekleyen herhangi bir başka platformda çalıştırabileceğiniz anlamına gelir.

“javac” komutu, Java Development Kit (JDK) dağıtımlarında bulunur.

Java derleyicisi ve standart Java araç zincirinin geri kalanı, koda aşağıdaki kısıtlamaları getirir:

  • Kaynak kodu “.java” sonekine sahip dosyalarda tutulur
  • Bayt kodları, “.class” sonekine sahip dosyalarda tutulur
  • Dosya sistemindeki kaynak ve bayt kodu dosyaları için dosya yol adları, paket ve sınıf adını yansıtmalıdır.

Not: “javac” derleyicisi, bayt kodlarını yerel koda derleyen https://www.wikiod.com/tr/java/tam-zamannda-jit-derleyicisi ile karıştırılmamalıdır.

‘javac’ komutu - başlarken #

Basit örnek #

“HelloWorld.java"nın aşağıdaki Java kaynağını içerdiğini varsayarsak:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello world!");
    }
}

(Yukarıdaki kodun açıklaması için lütfen https://www.wikiod.com/tr/java/java-dilini-kullanmaya-baslama adresine bakın.)

Bu komutu kullanarak yukarıdaki dosyayı derleyebiliriz:

$ javac HelloWorld.java

Bu, “HelloWorld.class” adında bir dosya üretir ve daha sonra aşağıdaki gibi çalıştırabiliriz:

$ java HelloWorld
Hello world!

Bu örnekte dikkat edilmesi gereken önemli noktalar şunlardır:

  1. “HelloWorld.java” kaynak dosya adı, kaynak dosyadaki … sınıf adıyla “HelloWorld” ile eşleşmelidir. Eşleşmezlerse derleme hatası alırsınız.
  2. “HelloWorld.class” bayt kodu dosya adı, sınıf adına karşılık gelir. “HelloWorld.class” dosyasını yeniden adlandırırsanız, çalıştırmayı denediğinizde bir hata alırsınız.
  3. Java uygulamasını ‘java’ kullanarak çalıştırırken, bayt kodu dosya adını DEĞİL sınıf adını sağlarsınız.

Paketlerle örnek #

En pratik Java kodu, sınıflar için ad alanını düzenlemek ve yanlışlıkla sınıf adı çakışması riskini azaltmak için paketleri kullanır.

“Com.example” adlı bir pakette “HelloWorld” sınıfını bildirmek isteseydik, “HelloWorld.java” aşağıdaki Java kaynağını içerecektir:

package com.example;

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello world!");
    }
}

Bu kaynak kod dosyasının, yapısı paket adlandırmasına karşılık gelen bir dizin ağacında saklanması gerekir.

.    # the current directory (for this example)
|
 ----com
     |
      ----example
          |
           ----HelloWorld.java

Bu komutu kullanarak yukarıdaki dosyayı derleyebiliriz:

$ javac com/example/HelloWorld.java

Bu, “com/example/HelloWorld.class” adında bir dosya üretir; yani derlemeden sonra dosya yapısı şöyle görünmelidir:

.    # the current directory (for this example)
|
 ----com
     |
      ----example
          |
           ----HelloWorld.java
           ----HelloWorld.class

Daha sonra uygulamayı şu şekilde çalıştırabiliriz:

$ java com.example.HelloWorld
Hello world!

Bu örnekte dikkat edilmesi gereken ek noktalar şunlardır:

  1. Dizin yapısı, paket adı yapısıyla eşleşmelidir.
  2. Sınıfı çalıştırdığınızda, tam sınıf adı sağlanmalıdır; yani “MerhabaDünya” değil “com.example.HelloWorld”.
  3. Java kodunu geçerli dizinden derlemeniz ve çalıştırmanız gerekmez. Burada sadece örnek olsun diye yapıyoruz.

‘javac’ ile birden fazla dosyayı aynı anda derlemek. #

Uygulamanız birden çok kaynak kod dosyasından oluşuyorsa (ve çoğu bunu yapıyor!), bunları birer birer derleyebilirsiniz. Alternatif olarak, yol adlarını listeleyerek birden çok dosyayı aynı anda derleyebilirsiniz:

$ javac Foo.java Bar.java

veya komut kabuğunuzun dosya adı joker işlevini kullanarak ….

$ javac *.java
$ javac com/example/*.java
$ javac */**/*.java #Only works on Zsh or with globstar enabled on your shell

Bu, geçerli dizindeki, “com/example” dizinindeki ve alt dizinlerdeki özyinelemeli olarak tüm Java kaynak dosyalarını derleyecektir. Üçüncü bir alternatif, bir dosya olarak kaynak dosya adlarının (ve derleyici seçeneklerinin) bir listesini sağlamaktır. Örneğin:

$ javac @sourcefiles

‘sourcefiles’ dosyası şunları içerir:

Foo.java
Bar.java
com/example/HelloWorld.java

Not: Bunun gibi bir kod derlemek, küçük tek kişilik projeler ve bir defaya mahsus programlar için uygundur. Bunun ötesinde, bir Java oluşturma aracı seçmeniz ve kullanmanız önerilir. Alternatif olarak, çoğu programcı gömülü bir derleyici ve artımlı “projeler” inşası sunan bir Java IDE (örneğin NetBeans, Eclipse, IntelliJ IDEA) kullanır.

Yaygın olarak kullanılan ‘javac’ seçenekleri #

İşte size faydalı olabilecek ‘javac’ komutu için birkaç seçenek

  • -d seçeneği, “.class” dosyalarını yazmak için bir hedef dizin belirler.
  • “-sourcepath” seçeneği, bir kaynak kodu arama yolu ayarlar.
  • ‘-cp’ veya ‘-classpath’ seçeneği, harici ve önceden derlenmiş sınıfları bulmak için arama yolunu ayarlar. Sınıf yolu ve bunun nasıl belirleneceği hakkında daha fazla bilgi için https://www.wikiod.com/tr/Java/snf-yolu Konusuna bakın.
  • -version seçeneği derleyicinin sürüm bilgisini yazdırır.

Derleyici seçeneklerinin daha eksiksiz bir listesi ayrı bir örnekte açıklanacaktır.

Referanslar #

‘javac’ komutu için kesin referans, ‘javac’ için Oracle kılavuz sayfası.

Farklı bir Java sürümü için derleme #

Java programlama dili (ve çalışma zamanı), ilk halka açık sürümünden bu yana piyasaya sürülmesinden bu yana çok sayıda değişiklik geçirdi. Bu değişiklikler şunları içerir:

  • Java programlama dili söz dizimi ve anlambilimindeki değişiklikler
  • Java standart sınıf kitaplıkları tarafından sağlanan API’lerdeki değişiklikler.
  • Java (bayt kodu) komut seti ve sınıf dosyası biçimindeki değişiklikler.

Çok az istisna dışında (örneğin “enum” anahtar sözcüğü, bazı “dahili” sınıflarda yapılan değişiklikler vb.), bu değişiklikler geriye dönük olarak uyumludur.

  • Java araç zincirinin eski bir sürümü kullanılarak derlenmiş bir Java programı, yeniden derleme olmaksızın daha yeni bir Java platformu sürümünde çalışacaktır.
  • Java’nın eski bir sürümünde yazılmış bir Java programı, yeni bir Java derleyicisi ile başarılı bir şekilde derlenecektir.

Eski Java’yı daha yeni bir derleyici ile derleme #

Daha yeni platformda çalıştırmak için eski Java kodunu daha yeni bir Java platformunda (yeniden) derlemeniz gerekiyorsa, genellikle herhangi bir özel derleme bayrağı vermeniz gerekmez. Birkaç durumda (örneğin, tanımlayıcı olarak “enum” kullandıysanız) yeni sözdizimini devre dışı bırakmak için “-source” seçeneğini kullanabilirsiniz. Örneğin, aşağıdaki sınıf verildi:

public class OldSyntax {
    private static int enum;  // invalid in Java 5 or later
}

Java 5 derleyicisi (veya üstü) kullanarak sınıfı derlemek için aşağıdakiler gereklidir:

$ javac -source 1.4 OldSyntax.java

Daha eski bir yürütme platformu için derleme #

Java’yı daha eski bir Java platformlarında çalışacak şekilde derlemeniz gerekiyorsa, en basit yaklaşım, desteklemeniz gereken en eski sürüm için bir JDK yüklemek ve derlemelerinizde bu JDK’nın derleyicisini kullanmaktır.

Ayrıca daha yeni bir Java derleyicisiyle de derleyebilirsiniz, ancak karmaşık olanlar var. Her şeyden önce, yerine getirilmesi gereken bazı önemli ön koşullar vardır:

  • Derlediğiniz kod, hedeflediğiniz Java sürümünde bulunmayan Java dili yapılarını kullanmamalıdır.
  • Kod, eski platformlarda bulunmayan standart Java sınıflarına, alanlarına, yöntemlerine vb. bağlı olmamalıdır.
  • Kodun bağlı olduğu üçüncü taraf kitaplıkları da eski platform için oluşturulmalı ve derleme ve çalışma zamanında kullanılabilir olmalıdır.

Ön koşullar karşılandığında, -target seçeneğini kullanarak daha eski bir platform için kodu yeniden derleyebilirsiniz. Örneğin,

$ javac -target 1.4 SomeClass.java

Java 1.4 veya sonraki JVM ile uyumlu bayt kodları üretmek için yukarıdaki sınıfı derleyecektir. (Aslında, “-source” seçeneği, uyumlu bir “-target” anlamına gelir, bu nedenle “javac -source 1.4 …” aynı etkiye sahip olacaktır. “-source” ve “-target” arasındaki ilişki aşağıdaki bölümde açıklanmıştır. Oracle belgeleri.)

Bunu söyledikten sonra, sadece ‘-target’ veya ‘-source’ kullanırsanız, derleyicinin JDK’sı tarafından sağlanan standart sınıf kitaplıklarına karşı derleme yapmaya devam edeceksiniz. Dikkatli olmazsanız, doğru bayt kodu sürümüne sahip ancak API’lere bağımlı olmayan sınıflara sahip olabilirsiniz. Çözüm, -bootclasspath seçeneğini kullanmaktır. Örneğin:

$ javac -target 1.4 --bootclasspath path/to/java1.4/rt.jar SomeClass.java

alternatif bir çalışma zamanı kitaplığı kümesine karşı derlenir. Derlenen sınıfın daha yeni kitaplıklara (yanlışlıkla) bağımlılıkları varsa, bu size derleme hataları verecektir.
