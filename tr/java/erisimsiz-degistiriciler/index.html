Erişimsiz Değiştiriciler ***değişkenlerin ve yöntemlerin erişilebilirliğini değiştirmezler, ancak onlara özel özellikler sağlarlar.

son #

Java’daki ‘final’ değişkenlere, yöntemlere ve sınıflara atıfta bulunabilir. Üç basit kural vardır:

  • son değişken yeniden atanamaz
  • son yöntem geçersiz kılınamaz
  • son sınıf uzatılamaz

Kullanımlar

İyi Programlama Uygulaması

Bazı geliştiriciler, mümkün olduğunda bir değişken finalini işaretlemenin iyi bir uygulama olduğunu düşünür. Değiştirilmemesi gereken bir değişkeniniz varsa, onu final olarak işaretlemelisiniz.

Yöntem parametreleri için “final” anahtar sözcüğünün önemli bir kullanımı. Bir yöntemin giriş parametrelerini değiştirmediğini vurgulamak istiyorsanız, özellikleri son olarak işaretleyin.

public int sumup(final List<Integer> ints);

Bu, ‘sumup’ yönteminin ‘int’leri değiştirmeyeceğini vurgular.

İç sınıf Erişimi

Anonim iç sınıfınız bir değişkene erişmek istiyorsa, değişken “final” olarak işaretlenmelidir.

  public IPrintName printName(){
    String name;
    return new IPrintName(){
        @Override
        public void printName(){
            System.out.println(name);
        }
    };
}

’name’ değişkeni nihai olmadığı için bu sınıf derlenmiyor.

Etkili bir şekilde son değişkenler bir istisnadır. Bunlar, yalnızca bir kez yazılan ve bu nedenle nihai hale getirilebilen yerel değişkenlerdir. Son değişkenlere anonim sınıflardan da etkin bir şekilde erişilebilir.

son statik değişken

Aşağıdaki kod, ‘final’ değişkeni ‘foo’ ‘statik’ olmadığında tamamen yasal olsa da, ‘statik’ olması durumunda derlenmeyecektir:

class TestFinal {
    private final static List foo;

    public Test() {
        foo = new ArrayList();
    }
}

Nedeni, tekrar edelim, son değişken yeniden atanamaz. “foo” statik olduğundan, “TestFinal” sınıfının tüm örnekleri arasında paylaşılır. TestFinal sınıfının yeni bir örneği oluşturulduğunda, yapıcısı çağrılır ve bu nedenle derleyicinin izin vermediği foo yeniden atanır. Bu durumda “foo” değişkenini başlatmanın doğru bir yolu şudur:

class TestFinal {
    private static final List foo = new ArrayList();
    //..
}

veya statik bir başlatıcı kullanarak:

class TestFinal {
    private static final List foo;
    static {
        foo = new ArrayList();
    }
    //..
}

‘final’ yöntemleri, temel sınıf, türetilmiş sınıfın değiştirmemesi gereken bazı önemli işlevleri uyguladığında faydalıdır. Ayrıca, nihai olmayan yöntemlerden daha hızlıdırlar, çünkü ilgili sanal tablo kavramı yoktur.

Java’daki tüm sarmalayıcı sınıflar, ‘Integer’, ‘Long’ vb. gibi nihaidir. Bu sınıfların yaratıcıları, örneğin; Integer’ı kendi sınıfına genişletin ve Integer sınıfının temel davranışını değiştirin. Bir sınıfı değişmez kılmak için gereksinimlerden biri, alt sınıfların yöntemleri geçersiz kılmamasıdır. Bunu yapmanın en basit yolu, sınıfı ‘final’ olarak ilan etmektir.

statik #

‘static’ anahtar sözcüğü, sınıfın herhangi bir örneğinden bağımsız olarak çalışmalarını sağlamak için bir sınıf, yöntem veya alan üzerinde kullanılır.

  • Statik alanlar, bir sınıfın tüm örnekleri için ortaktır. Onlara erişmek için bir örneğe ihtiyaçları yoktur.

  • Statik metotlar içinde bulundukları sınıfın bir örneği olmadan çalıştırılabilirler. Ancak sadece o sınıfın statik alanlarına erişebilirler.

  • Statik sınıflar, diğer sınıfların içinde bildirilebilir. Örneklenebilmek için içinde bulundukları sınıfın bir örneğine ihtiyaçları yoktur.

    public class TestStatic { static int staticVariable;

      static {
      // This block of code is run when the class first loads
      staticVariable = 11;
  }

  int nonStaticVariable = 5;

  static void doSomething() {
      // We can access static variables from static methods
      staticVariable = 10;
  }

  void add() {
      // We can access both static and non-static variables from non-static methods
      nonStaticVariable += staticVariable;
  }

  static class StaticInnerClass {
      int number;
      public StaticInnerClass(int _number) {
          number = _number;
      }

      void doSomething() {
          // We can access number and staticVariable, but not nonStaticVariable
          number += staticVariable;
      }

      int getNumber() {
          return number;
      }
  }

    }

    // Static fields and methods TestStatic object1 = new TestStatic();

    System.out.println(object1.staticVariable); // 11 System.out.println(TestStatic.staticVariable); // 11

    TestStatic.doSomething();

    TestStatic object2 = new TestStatic();

    System.out.println(object1.staticVariable); // 10 System.out.println(object2.staticVariable); // 10 System.out.println(TestStatic.staticVariable); // 10

    object1.add();

    System.out.println(object1.nonStaticVariable); // 15 System.out.println(object2.nonStaticVariable); // 10

    // Static inner classes StaticInnerClass object3 = new TestStatic.StaticInnerClass(100); StaticInnerClass object4 = new TestStatic.StaticInnerClass(200);

    System.out.println(object3.getNumber()); // 100 System.out.println(object4.getNumber()); // 200

    object3.doSomething();

    System.out.println(object3.getNumber()); // 110 System.out.println(object4.getNumber()); // 200

Öz #

Soyutlama, uygulama ayrıntılarını gizleme ve kullanıcıya yalnızca işlevselliği gösterme işlemidir. Soyut bir sınıf asla somutlaştırılamaz. Bir sınıf soyut olarak bildirilirse, tek amaç sınıfın genişletilmesidir.

abstract class Car
{
    abstract void tagLine();
}
 
class Honda extends Car
{
    void tagLine()
    {
        System.out.println("Start Something Special");
    }
}
 
class Toyota extends Car
{
    void tagLine()
    {
        System.out.println("Drive Your Dreams");
    }
}

katı fp #

kayan nokta hesaplamaları için strictfp değiştiricisi kullanılır. Bu değiştirici, kayan nokta değişkenini birden çok platformda daha tutarlı hale getirir ve hem 32 bit hem de 64 bit mimaride hesaplama hatalarını (yuvarlama hataları), taşmaları ve eksikleri önlemek için tüm kayan nokta hesaplamalarının IEEE 754 standartlarına göre yapılmasını sağlar. Bu, soyut yöntemlere, değişkenlere veya yapıcılara uygulanamaz.

// strictfp keyword can be applied on methods, classes and interfaces.

strictfp class A{}

strictfp interface M{}

class A{  
    strictfp void m(){}
}

uçucu #

‘uçucu’ değiştirici, çok iş parçacıklı programlamada kullanılır. Bir alanı “geçici” olarak bildirirseniz, bu, iş parçacıklarına yerel olarak önbelleğe alınmış bir değeri değil, en son değeri okumaları gerektiğinin bir işaretidir. Ayrıca, “geçici” okuma ve yazma işlemlerinin atomik olması garanti edilir (“uçucu olmayan” “uzun” veya “çift” erişim atomik değildir), böylece birden çok iş parçacığı arasında belirli okuma/yazma hatalarından kaçınılır.

public class MyRunnable implements Runnable
{
    private volatile boolean active;
 
    public void run(){ // run is called in one thread 
        active = true;
        while (active){
            // some code here
        }
    }
    
    public void stop(){ // stop() is called from another thread
        active = false;
    }
}

senkronize #

Senkronize değiştirici, belirli bir yöntemin veya bir bloğun birden çok iş parçacığı tarafından erişimini kontrol etmek için kullanılır. Senkronize olarak bildirilen bir yönteme veya bloğa yalnızca bir iş parçacığı girebilir. synchronized anahtar sözcüğü bir nesnenin içsel kilidi üzerinde çalışır, senkronize bir yöntem olması durumunda mevcut nesneler kilidi ve statik yöntem sınıf nesnesini kullanır. Senkronize bir blok yürütmeye çalışan herhangi bir iş parçacığı önce nesne kilidini almalıdır.

class Shared
{
    int i;
 
    synchronized void SharedMethod()
    {
        Thread t = Thread.currentThread();
 
        for(int i = 0; i <= 1000; i++)
        {
            System.out.println(t.getName()+" : "+i);
        }
    }

    void SharedMethod2()
    {
        synchronized (this)
        {
            System.out.println("Thais access to currect object is synchronize "+this);
        }
    }
}
 
public class ThreadsInJava
{
    public static void main(String[] args)
    {
        final Shared s1 = new Shared();
 
        Thread t1 = new Thread("Thread - 1")
        {
            @Override
            public void run()
            {
                s1.SharedMethod();
            }
        };
 
        Thread t2 = new Thread("Thread - 2")
        {
            @Override
            public void run()
            {
                s1.SharedMethod();
            }
        };
 
        t1.start();
 
        t2.start();
    }
}
