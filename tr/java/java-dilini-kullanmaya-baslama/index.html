İlk Java Programınızı Oluşturma #

Metin düzenleyicinizde veya IDE‘de HelloWorld.java adında yeni bir dosya oluşturun. Ardından bu kod bloğunu dosyaya yapıştırın ve kaydedin:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Ideone’da canlı çalıştırın

Not: Java’nın bunu bir “genel sınıf” olarak tanıması (ve bir [derleme zamanı hatası][3] atmaması), dosya adının sınıf adıyla aynı olması gerekir (burada “MerhabaDünya”) örnek) .java uzantısıyla. Ayrıca ondan önce bir ‘public’ erişim değiştiricisi olmalıdır.

[Adlandırma kuralları][4] Java sınıflarının büyük harfle başlamasını ve camel case biçiminde (ilk harfin olduğu) olmasını önerir. Her kelimenin büyük harfle yazılır). Kurallar, alt çizgilere (_) ve dolar işaretlerine ($) karşı tavsiyede bulunur.

Derlemek için bir terminal penceresi açın ve HelloWorld.java dizinine gidin:

cd /path/to/containing/folder/

Not: cd, dizini değiştirmek için kullanılan terminal komutudur.

“javac” yazıp ardından dosya adını ve uzantısını aşağıdaki gibi girin:

$ javac HelloWorld.java

“Javac”, dahili veya harici bir komut, çalıştırılabilir program veya toplu iş dosyası olarak tanınmıyor hatası almak oldukça yaygındır. “JDK"yı yüklediğinizde ve programı “IDE"den çalıştırabilseniz bile. eclipse vb. Yol varsayılan olarak ortama eklenmediğinden.

Bunu Windows’ta alırsanız, çözmek için önce ‘javac.exe’ yolunuza göz atmayı deneyin, büyük olasılıkla ‘C:\Program Files\Java\jdk(sürüm numarası)\bin’ dizinindedir. Ardından aşağıdaki ile çalıştırmayı deneyin.

$ C:\Program Files\Java\jdk(version number)\bin\javac HelloWorld.java

Önceden javac’ı çağırırken yukarıdaki komutla aynıydı. Yalnızca bu durumda ‘işletim sisteminiz’ ‘javac’ın nerede olduğunu biliyordu. Şimdi söyleyelim, bu şekilde her seferinde tüm yolu yazmak zorunda kalmazsınız. Bunu ‘YOL’umuza eklememiz gerekecek

Windows XP/Vista/7/8/10’da “PATH” ortam değişkenini düzenlemek için:

  • Kontrol Paneli ⇒ Sistem ⇒ Gelişmiş sistem ayarları
  • “Gelişmiş” sekmesine geçin ⇒ Ortam Değişkenleri
  • “Sistem Değişkenleri"nde, “YOL” ⇒ Düzenle’yi seçmek için aşağı kaydırın

Bunu geri alamazsınız bu yüzden dikkatli olun. Önce mevcut yolunuzu not defterine kopyalayın. Ardından ‘javac’ınızın tam PATH’ini almak için ‘javac’ın bulunduğu klasöre manuel olarak göz atın ve adres çubuğuna tıklayın ve ardından kopyalayın. c:\Program Files\Java\jdk1.8.0_xx\bin gibi görünmelidir.

“Değişken değer” alanına, mevcut tüm dizinlerin bunu ÖNÜNE ve ardından noktalı virgül (;) yapıştırın. SİLMEYİN mevcut girişleri.

Variable name  : PATH
Variable value : c:\Program Files\Java\jdk1.8.0_xx\bin;[Existing Entries...]

Şimdi bu çözülmeli.

Linux Tabanlı sistemler için [burayı deneyin][10].

Not: [javac][5] komutu Java derleyicisini çağırır.

Derleyici daha sonra [Java Sanal Makinesinde (JVM)] çalıştırılabilen ‘HelloWorld.class’ adında bir bytecode dosyası oluşturacaktır. Java programlama dili derleyicisi “javac”, Java programlama dilinde yazılmış kaynak dosyaları okur ve bunları “bytecode” sınıf dosyalarında derler. İsteğe bağlı olarak, derleyici, “Pluggable” Annotation Processing API’sini kullanarak kaynak ve sınıf dosyalarında bulunan açıklamaları da işleyebilir. Derleyici bir komut satırı aracıdır ancak Java Derleyici API’si kullanılarak da çağrılabilir.

Programınızı çalıştırmak için ‘java’ ve ardından ‘main’ yöntemini içeren sınıfın adını girin (örneğimizde ‘HelloWorld’). “.class” öğesinin nasıl atlandığına dikkat edin:

$ java HelloWorld

Not: [java][7] komutu bir Java uygulamasını çalıştırır.

Bu, konsolunuza çıktı verecektir:

Merhaba, Dünya!

İlk Java programınızı başarıyla kodladınız ve oluşturdunuz!

Not: Java komutlarının (java, javac vb.) tanınması için şunlardan emin olmanız gerekir:

  • Bir JDK kurulur (ör. [Oracle][8], [OpenJDK][9] ve diğer kaynaklar)
  • Ortam değişkenleriniz düzgün [kurulu][10]

JVM’niz tarafından sağlanan bir derleyici (‘javac’) ve bir yürütücü (‘java’) kullanmanız gerekecektir. Hangi sürümleri yüklediğinizi öğrenmek için komut satırına java -version ve javac -version girin. Programınızın sürüm numarası terminalde yazdırılacaktır (örn. 1.8.0_73).

Merhaba Dünya programına daha yakından bir bakış #

“Hello World” programı, “HelloWorld” sınıf tanımı, “main” yöntemi ve “main” yönteminin içindeki bir ifadeden oluşan tek bir dosya içerir.

public class HelloWorld {

‘class’ anahtar sözcüğü, ‘HelloWorld’ adlı bir sınıfın sınıf tanımını başlatır. Her Java uygulaması en az bir sınıf tanımı içerir ([Sınıflar hakkında daha fazla bilgi][11]).

    public static void main(String[] args)  {

Bu, “JVM"nin programınızı çalıştırabileceği bir giriş noktası yöntemidir (adıyla ve “public static void main(String[])” imzasıyla tanımlanır). Her Java programında bir tane olmalıdır. Bu:

  • ‘public’: yöntemin herhangi bir yerden çağrılabileceği anlamına gelir, yani programın dışından da. Bununla ilgili daha fazla bilgi için [Görünürlük][12] bölümüne bakın.
  • ‘statik’: var olduğu ve kendi başına çalıştırılabileceği anlamına gelir (sınıf düzeyinde bir nesne oluşturmadan).
  • void: değer döndürmediği anlamına gelir. Not: Bu, “int” gibi bir dönüş kodunun beklendiği C ve C++‘dan farklıdır (Java’nın yolu [System.exit()][13]).

Bu ana yöntem şunları kabul eder:

  • “Dize"nin bir [dizi][14] (tipik olarak ‘args’ olarak adlandırılır) ana işleve bağımsız değişken olarak iletilir (ör. [komut satırı bağımsız değişkenlerinden][15]).

Bunların hemen hepsi bir Java giriş noktası yöntemi için gereklidir.

Gerekli olmayan parçalar:

  • ‘args’ adı bir değişken adıdır, bu nedenle genellikle ‘args’ olarak adlandırılsa da istediğiniz herhangi bir şey olarak adlandırılabilir.
  • Parametre türünün bir dizi (String[] args) veya Varargs (String... args) olup olmadığı önemli değildir, çünkü diziler varargs’a geçirilebilir.

Not: Tek bir uygulama, bir giriş noktası (`ana’) yöntemi içeren birden çok sınıfa sahip olabilir. Uygulamanın giriş noktası, ‘java’ komutuna argüman olarak iletilen sınıf adına göre belirlenir.

Ana yöntemin içinde aşağıdaki ifadeyi görüyoruz:

        System.out.println("Hello, World!");

Bu ifadeyi tek tek parçalayalım:

eleman Amaç
‘Sistem’ bu, sonraki ifadenin ‘java.lang’ paketinden ‘System’ sınıfını çağıracağını belirtir.
. bu bir “nokta operatörü"dür. Nokta operatörleri, bir sınıf üyesine erişmenizi sağlar1; yani alanları (değişkenleri) ve yöntemleri. Bu durumda, bu nokta operatörü, ‘System’ sınıfı içindeki ‘out’ statik alanına başvurmanıza izin verir.
‘dışarı’ bu, standart çıktı işlevselliğini içeren “Sistem” sınıfı içindeki “PrintStream” türündeki statik alanın adıdır.
. bu başka bir nokta operatörüdür. Bu nokta operatörü, ‘out’ değişkeni içindeki ‘println’ yöntemine erişim sağlar.
println bu, PrintStream sınıfındaki bir yöntemin adıdır. Bu yöntem özellikle parametrelerin içeriğini konsola yazdırır ve arkasına yeni bir satır ekler.
( bu parantez, bir yönteme erişildiğini (bir alana değil) belirtir ve println yöntemine geçirilen parametreleri başlatır.
`“Merhaba Dünya!”’ bu, “println” yöntemine parametre olarak geçirilen [String][16] değişmez değeridir. Her iki uçtaki çift tırnak işaretleri, metni bir Dize olarak sınırlar.
) bu parantez, ‘println’ yöntemine geçirilen parametrelerin kapanışını belirtir.
; bu noktalı virgül, ifadenin sonunu gösterir.
Not: Java’daki her ifade noktalı virgül (;) ile bitmelidir.

Yöntem gövdesi ve sınıf gövdesi daha sonra kapatılır.

    }  // end of main function scope
}      // end of class HelloWorld scope

İşte OO paradigmasını gösteren başka bir örnek. Bir (evet, bir!) üyesi olan bir futbol takımını modelleyelim. Daha fazlası olabilir, ancak dizilere geldiğimizde bunu tartışacağız.

İlk önce Team sınıfımızı tanımlayalım:

public class Team {
    Member member;
    public Team(Member member) {  // who is in this Team?
        this.member = member;  //  one 'member' is in this Team!
    }
}

Şimdi Member sınıfımızı tanımlayalım:

class Member {
    private String name;
    private String type;
    private int level; // note the data type here
    private int rank; // note the data type here as well

    public Member(String name, String type, int level, int rank) {
        this.name = name; 
        this.type = type;
        this.level = level;
        this.rank = rank;
    }
}

Neden burada ‘özel’ kullanıyoruz? Pekala, eğer biri adınızı bilmek isterse, cebinize uzanıp Sosyal Güvenlik kartınızı çıkarmak yerine doğrudan size sormalıdır. Bu “özel” şöyle bir şey yapar: dış varlıkların değişkenlerinize erişmesini engeller. ‘private’ üyeleri yalnızca alıcı işlevleri aracılığıyla döndürebilirsiniz (aşağıda gösterilmiştir).

Hepsini bir araya getirdikten ve daha önce tartışıldığı gibi alıcıları ve ana yöntemi ekledikten sonra, elimizde:

public class Team {
    Member member;
    public Team(Member member) {  
        this.member = member;  
    }

    // here's our main method
    public static void main(String[] args) {
       Member myMember = new Member("Aurieel", "light", 10, 1);
