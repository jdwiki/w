Nashorn JavaScript motoru

Bu sayfada

  • Sözdizimi
  • JavaScript dosyasını yürütün
  • Komut dosyası çıktısını engelleme
  • Merhaba Nashorn
  • Aritmetik Dizeleri Değerlendirin
  • Genel değişkenleri ayarla
  • Global değişkenleri ayarlayın ve alın
  • Nashorn’da JavaScript’te Java nesnelerinin kullanımı
  • Komut dosyasından bir arayüz uygulamak

Nashorn Oracle tarafından Java’da geliştirilen bir JavaScript motorudur ve Java 8 ile piyasaya sürülmüştür. Nashorn, Javascript’in JSR-223 aracılığıyla Java uygulamalarına gömülmesine izin verir ve bağımsız Javascript uygulamalarının geliştirilmesine olanak tanır ve daha iyi sağlar çalışma zamanı performansı ve ECMA normalleştirilmiş Javascript spesifikasyonu ile daha iyi uyumluluk.

Sözdizimi #

  • ScriptEngineManager // ScriptEngine sınıfları için bir keşif ve kurulum mekanizması sağlar; bir SPI (Servis Sağlayıcı Arayüzü) kullanır
  • ScriptEngineManager.ScriptEngineManager() // Önerilen kurucu
  • ScriptEngine // Komut dosyası diline arayüz sağlar
  • ScriptEngine ScriptEngineManager.getEngineByName(String shortName) // Verilen uygulama için fabrika yöntemi
  • Object ScriptEngine.eval(String script) // Belirtilen scripti çalıştırır
  • Object ScriptEngine.eval(Reader okuyucu) // Belirtilen kaynaktan bir komut dosyası yükler ve ardından çalıştırır
  • ScriptContext ScriptEngine.getContext() // Varsayılan bağlamaları, okuyucuları ve yazar sağlayıcısını döndürür
  • void ScriptContext.setWriter(Yazar yazar) // Komut dosyası çıktısının gönderileceği hedefi ayarlar

Nashorn, Java ile yazılmış ve Java 8’e dahil edilmiş bir JavaScript motorudur. İhtiyacınız olan her şey javax.script paketinde toplanmıştır.

‘ScriptEngineManager’ın, çeşitli komut dosyası dilleri için komut dosyası motorları elde etmenize olanak tanıyan genel bir API sağladığını unutmayın (yani yalnızca Nashorn değil, yalnızca JavaScript değil).

JavaScript dosyasını yürütün #

// Required imports
import javax.script.ScriptEngineManager;
import javax.script.ScriptEngine;
import javax.script.ScriptException;
import java.io.FileReader;
import java.io.FileNotFoundException;

// Obtain an instance of the JavaScript engine
ScriptEngineManager manager = new ScriptEngineManager();
ScriptEngine engine = manager.getEngineByName("nashorn");

// Load and execute a script from the file 'demo.js'
try {
    engine.eval(new FileReader("demo.js"));
} catch (FileNotFoundException ex) {
    ex.printStackTrace();
} catch (ScriptException ex) {
    // This is the generic Exception subclass for the Scripting API
    ex.printStackTrace();
}

// Outcome:
// 'Script from file!' printed on standard output

demo.js:

print('Script from file!');

Komut dosyası çıktısını engelleme #

// Obtain an instance of JavaScript engine
ScriptEngineManager manager = new ScriptEngineManager();
ScriptEngine engine = manager.getEngineByName("nashorn");

// Setup a custom writer
StringWriter stringWriter = new StringWriter();
// Modify the engine context so that the custom writer is now the default
// output writer of the engine
engine.getContext().setWriter(stringWriter);

// Execute some script
try {
    engine.eval("print('Redirected text!');");
} catch (ScriptException ex) {
    ex.printStackTrace();
}

// Outcome:
// Nothing printed on standard output, but
// stringWriter.toString() contains 'Redirected text!'

Nashorn’da JavaScript’te Java nesnelerinin kullanımı #

Java nesnelerini Java kodunda işlenmek üzere Nashorn motoruna geçirmek mümkündür. Aynı zamanda, bazı JavaScript’e (ve Nashorn’a) özel yapılar vardır ve bunların Java nesneleriyle nasıl çalıştıkları her zaman açık değildir.

Aşağıda, JavaScript yapıları içindeki yerel Java nesnelerinin davranışını açıklayan bir tablo bulunmaktadır.

Test edilmiş yapılar:

  1. If cümlesinde ifade. JS ifadesinde if yan tümcesinin Java’dan farklı olarak boole olması gerekmez. Sahte değerler (boş, tanımsız, 0, boş dizeler vb.) için false olarak değerlendirilir.
  2. her ifade için Nashorn’un her biri için farklı JS ve Java nesnesi üzerinde yinelenebilen özel bir döngü türü vardır.
  3. Nesne boyutunu alma. JS’de nesnelerin, bir dizinin veya bir dizenin boyutunu döndüren bir özellik uzunluğu vardır.

Sonuçlar:

Tip Eğer her biri için .uzunluk
Java boş yanlış Yineleme yok İstisna
Java boş dize yanlış Yineleme yok 0
Java dizisi doğru Dize karakterleri üzerinde yinelenir dizenin uzunluğu
Java Tamsayı/Uzun değer != 0 Yineleme yok tanımsız
Java Dizi Listesi doğru Öğeler üzerinde yinelenir Listenin uzunluğu
Java HashMap doğru Değerler üzerinde yinelenir boş
Java HashSet doğru Öğeler üzerinde yinelenir tanımsız

Öneriler:

  • Bir dizgenin boş ve boş olmadığını kontrol etmek için if (some_string) kullanılması tavsiye edilir
  • “for her” herhangi bir koleksiyon üzerinde yineleme yapmak için güvenle kullanılabilir ve koleksiyon yinelenebilir, boş veya tanımsız değilse istisna oluşturmaz
  • Bir nesnenin uzunluğunu almadan önce, boş veya tanımsız olup olmadığı kontrol edilmelidir (aynısı, herhangi bir yöntemi çağırma veya Java nesnesinin bir özelliğini alma girişimi için de geçerlidir)

import java.io.FileReader;
import java.io.IOException;

import javax.script.ScriptEngine;
import javax.script.ScriptEngineManager;
import javax.script.ScriptException;

public class InterfaceImplementationExample {
    public static interface Pet {
        public void eat();
    }

    public static void main(String[] args) throws IOException {
        // Obtain an instance of JavaScript engine
        ScriptEngineManager manager = new ScriptEngineManager();
        ScriptEngine engine = manager.getEngineByName("nashorn");

        try {
            //evaluate a script
            /* pet.js */
            /*
                var Pet = Java.type("InterfaceImplementationExample.Pet");
                
                new Pet() {
                    eat: function() { print("eat"); }
                }            
            */
            
            Pet pet = (Pet) engine.eval(new FileReader("pet.js"));
            
            pet.eat();
        } catch (ScriptException ex) {
            ex.printStackTrace();
        }

        // Outcome:
        // 'eat' printed on standard output
    }
}
