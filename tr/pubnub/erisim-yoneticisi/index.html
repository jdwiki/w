sunucu tarafında pubnub erişim yöneticisi #

PubNub Erişim Yöneticisi (PAM), geliştiricilerin PubNub Gerçek Zamanlı Ağı boyunca kanallara güvenli erişim oluşturmasına ve bunları uygulamasına olanak tanıyarak PubNub’ın mevcut güvenlik çerçevesini genişletir.

Access Manager, gerçek zamanlı uygulamalarınız ve veri akışlarınız için ayrıntılı izinleri yönetmenize, birden çok izin düzeyi oluşturmanıza, erişim izni vermenize ve erişimi iptal etmenize ve kullanıcı erişimini denetlemenize olanak tanır.

Access Manager’ı kullanmak için Yönetici Panosu’nda Access Manager’ı etkinleştirmeniz gerekir. Access Manager’ı etkinleştirdikten sonra, herhangi bir veri gönderilmeden veya alınmadan önce izinler vermeniz gerekir.

PAM Server side Configuration

İstemci tarafının düzgün çalışması için, sunucu tarafında önce belirli bir PAM kanalı veya kanal grubu ve auth token kombinasyonu için uygun izinler verilmelidir.

bu izinleri vermek için en azından abone ve gizli anahtarlarınızla pubnub örneğini başlatmalısınız.

Örnek :

Adım 1. Pubnub Yapılandırmasını Yapın : -

PNConfiguration pnConfiguration = new PNConfiguration();
        pnConfiguration.setSubscribeKey(SUBSCRIBE_KEY);
        pnConfiguration.setPublishKey(PUBLISH_KEY);;
        pnConfiguration.setSecretKey(SECRET_KEY);
        pnConfiguration.setSecure(true);
        pnConfiguration.setLogVerbosity(PNLogVerbosity.BODY);

Adım 2. PubNub’ı pnConfiguration ile başlatın

PubNub pubnub = new PubNub(pnConfiguration);

PAM İşlemi üç düzeyde gerçekleşir

1. A global level (no auth key, and no channel/channel group is defined) 

2. A channel/channel group level (only a channel/channel group is defined) 

3. A channel/channel group and key level (where both the channel/channel     group and key are defined)

Tüm bu seviyelerde izinler verebilir, iptal edebilir ve denetleyebiliriz. Burada aynısını kanal/kanal grubu ve auth key seviyesinde yapıyoruz.

PAM  Grant

belirli kanallarda veya kanal gruplarında auth_key için okuma/yazma izni verebiliriz

Örnek:

eşzamanlı:

try {

    pubnub.grant().authKeys(Arrays.asList("auth1,auth2"))
    .channels(Arrays.asList("channel1,channel2")).read(true).write(true    ).ttl(0).sync();

} catch (PubNubException e) {
    e.printStackTrace();
}

Eşzamansız olarak:

pubNub.grant().channels(channels).authKeys(Arrays.asList(authKey)).read(true).write(true).manage(false).ttl(0) .async(new PNCallback() {

@Override
public void onResponse(PNAccessManagerGrantResult result,
     PNStatus status) {
}});

PAM REVOKE: belirli bir kanaldan veya kanal gruplarından auth_key iznini iptal edebiliriz.

Verme ile aynı izni iptal etmek için sözdizimi. Sadece izni true olarak false olarak değiştirmemiz gerekiyor.

try {

    pubnub.grant().authKeys(Arrays.asList("auth1,auth2"))
    .channels(Arrays.asList("channel1,channel2")).read(false).write( false    ).ttl(0).sync();

} catch (PubNubException e) {
    e.printStackTrace();
}

PAM Denetimi: belirli bir kanal/kanal grubuna veya belirli bir kanal veya kanal grubundaki belirli bir auth_key için verilen bir izni denetleyebiliriz.

Örnek:

pubnub.audit().channel("mycha").authKeys(Arrays.asList("a1")).async(new PNCallback<PNAccessManagerAuditResult>(){

    @Override
    public void onResponse(PNAccessManagerAuditResult result,
            PNStatus status) {
                
            }
    });

PAM Kanalları gruplara ekle: kanalları da kanal gruplarına ekleyebiliriz

Örnek:

pubnub.addChannelsToChannelGroup().channelGroup("my_channel").channels(Arrays.asList("my_channel5"))
    .async(new PNCallback<PNChannelGroupsAddChannelResult>() {

            @Override
            public void onResponse(PNChannelGroupsAddChannelResult                                                                                                                                             result,PNStatus status) {
                
                
            }
});

İstemci Tarafında Kimlik Doğrulama Sorunu (403 Yasak):

PAM işlemlerini gerçekleştirirken bir hata varsa 403 hatası alabilirsiniz. Bunu yaparsanız, doğru secret_key’i ayarladığınızdan ve kartı veren bilgisayarın saatinin NTP ile eşitlendiğinden emin olun.

NTP  Setup

Ağ Zaman Protokolü (NTP), bir bilgisayar ağındaki bilgisayar saat zamanlarını senkronize etmek için kullanılan bir protokoldür. NTP, bilgisayar saat sürelerini bir milisaniyeye ve bazen de bir milisaniyenin bir kısmına eşitlemek için Eşgüdümlü Evrensel Zaman (UTC) kullanır.

Burada sunucu zamanını pubnub ile taramamız gerekiyor. Bunu yapmak için adımı izleyin

Adım 1 Kurulum NTP

$ sudo apt-get update
$ sudo apt-get install ntp

Adım 2 ntp.conf’u düzenleyin

Replace these four with pubnub server 

server 0.ubuntu.pool.ntp.org 
server 1.ubuntu.pool.ntp.org 
server 2.ubuntu.pool.ntp.org 
server 3.ubuntu.pool.ntp.org

ile

server 0.pubsub.pubnub.com
server 1.pubsub.pubnub.com
server 2.pubsub.pubnub.com
server 3.pubsub.pubnub.com

Adım 3 NTP Hizmetini Yeniden Başlatın

$ sudo hizmeti ntp yeniden başlatma

Wildcard Kanal Grubu Hibe Yönetimi - Java SDK v4 #

Kanal gruplarınıza kanal ekleme/çıkarma söz konusu olduğunda, bu kanal grupları için ‘yönetme’ iznine sahip olmanız gerekir. Ancak müşterilere abone olacakları kanal gruplarını “yönetme” iznini asla vermemelisiniz. Yaparlarsa, kanal gruplarına istedikleri kanalı ekleyebilir ve anında o kanala okuma erişimi elde edebilirler.

Bu yüzden sunucunuz ‘yönetme’ iznine sahip tek varlık olmalıdır. Ancak sunucunuzun, tüm istemciler adına kanal gruplarına/kanal gruplarına kanal ekleyebilmesi/kaldırabilmesi için her bir kanal grubu için ‘yönetme’ iznine sahip olması gerekir.

Ancak her kanal grubuna “yönet” yetkisi vermek biraz sıkıcı olabilir. Bunun yerine, tüm kanal gruplarına (mevcut ve oluşturulacak) tek bir joker karakter yetkisiyle ‘yönet’ verebilirsiniz.

// init PubNub instance using PNConfiguration with the secret-key
PNConfiguration pnConfiguration = new PNConfiguration();
pnConfiguration.setSubscribeKey("my_subkey")
pnConfiguration.setPublishKey("my_pubkey");
// secret key allows server to `grant` permissions
pnConfiguration.setSecretKey("my_secretkey");
pnConfiguration.setSecure(true);
// set the the server's auth key
pnConfiguration.setAuthKey("server_authkey");
PubNub pubnub = new PubNub(pnConfiguration);

// grant read and manage using the channel group wildcard - ":" 
// with forever ttl (0) 
pubNub.grant()
    .channelGroups(Arrays.asList(":")) // colon (:) is channel group wildcard
    .manage(true) // add/remove channels to/from channel groups
    .read(true) // in case server needs to subscribe or do here-now on channel groups
    .ttl(0) // 0 = forever grant
    .async(new PNCallback<PNAccessManagerGrantResult>() {
        @Override
        public void onResponse(PNAccessManagerGrantResult result, PNStatus status) {
            // check status for success or failure of grant
        }
    });

Bundan sonra sunucunuz, uygulamanızın oluşturduğu herhangi bir kanal grubuna kanal ekleyebilecek/kaldırabilecektir.
