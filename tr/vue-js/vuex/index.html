Vuex, Vue.js uygulamaları için bir durum yönetimi modeli + kitaplığıdır. Durumun yalnızca öngörülebilir bir şekilde mutasyona uğramasını sağlayan kurallarla, bir uygulamadaki tüm bileşenler için merkezi bir depo görevi görür. Ayrıca, sıfır yapılandırmalı zaman yolculuğu hata ayıklama ve durum anlık görüntüsünü dışa/içe aktarma gibi gelişmiş özellikler sağlamak için Vue’nun resmi geliştirme araçları uzantısıyla bütünleşir.

Vuex nedir? #

Vuex, uygulamanızda kullanım için merkezi bir veri deposu sunan Vue.js için resmi bir eklentidir. Daha basit uygulama tasarımı ve muhakemesine yol açan tek yönlü bir veri akışına sahip Flux uygulama mimarisinden büyük ölçüde etkilenir.

Bir Vuex uygulamasında veri deposu tüm paylaşılan uygulama durumunu tutar. Bu durum, gönderici aracılığıyla bir mutasyon olayı başlatan bir eylemeye yanıt olarak gerçekleştirilen mutasyonlar tarafından değiştirilir.

Bir Vuex uygulamasındaki veri akışının bir örneği aşağıdaki şemada özetlenmiştir. MIT lisansı altında kullanılan diyagram, orijinal olarak Resmi Vuex GitHub deposundan.

Bireysel Vue.js uygulama bileşenleri, istenen verilerin salt okunur bir kopyasını döndüren saf işlevler olan alıcılar aracılığıyla verileri almak için mağaza nesnesine erişebilir.

Bileşenler, bileşenin kendi veri kopyasında değişiklikler gerçekleştiren işlevler olan eylemlere sahip olabilir ve ardından bir mutasyon olayı göndermek için göndericiyi kullanabilir. Bu olay daha sonra durumu gerektiği gibi güncelleyen veri deposu tarafından işlenir.

Tüm bileşenler alıcıları aracılığıyla mağazaya reaktif olarak bağlı olduğundan, değişiklikler uygulama boyunca otomatik olarak yansıtılır.

Bir vue projesinde vuex kullanımını gösteren bir örnek.

const state = {
    lastClickTime: null
}

const mutations = {
    updateLastClickTime: (state, payload) => {
      state.lastClickTime = payload
  }
}

const getters = {
  getLastClickTime: state => {
    return new Date(state.lastClickTime)
  }
}

const actions = {
    syncUpdateTime: ({ commit }, payload) => {
    commit("updateLastClickTime", payload)
  },
  asyncUpdateTime: ({ commit }, payload) => {
    setTimeout(() => {
      commit("updateLastClickTime", payload)
    }, Math.random() * 5000)
  }
}

const store = new Vuex.Store({
  state,
  getters,
  mutations,
  actions
})

const { mapActions, mapGetters } = Vuex;

// Vue 
const vm = new Vue({
    el: '#container',
  store,
  computed: {
      ...mapGetters([
        'getLastClickTime'
    ])
  },
  methods: {
      ...mapActions([
        'syncUpdateTime',
      'asyncUpdateTime'
    ]),
    updateTimeSyncTest () {
        this.syncUpdateTime(Date.now())
    },
    updateTimeAsyncTest () {
        this.asyncUpdateTime(Date.now())
    }
  }
})

Ve bunun için HTML şablonu:

<div id="container">
  <p>{{ getLastClickTime || "No time selected yet" }}</p>
  <button @click="updateTimeSyncTest">Sync Action test</button>
  <button @click="updateTimeAsyncTest">Async Action test</button>
</div>

  1. Burada durum, null olarak başlatılan lastClickTime özelliğini içerir. Varsayılan değerlerin bu ayarı, özellikleri reaktif tutmak için önemlidir. Durumda belirtilmeyen özellikler mevcut olacak ancak bundan sonra yapılan değişiklikler alıcılar kullanılarak erişilemez.

  2. Kullanılan alıcı, bir mutasyon durum özelliğinin değerini her güncellediğinde güncellenecek olan hesaplanmış bir özellik sağlar.

  3. Yalnızca mutasyonların durumu ve özelliklerini değiştirmesine izin verilir, yani bunu yalnızca eşzamanlı olarak yapar.

  4. Eylemde API çağrısının (burada rasgele zamanlanmış setTimeout ile alay edilir) yapılabildiği asenkron güncellemeler durumunda bir Eylem kullanılabilir ve yanıt alındıktan sonra değişikliği yapmak için bir mutasyon taahhüt edilebilir. eyalet.

Neden Vuex’i kullanmalısınız? #

Genellikle yeniden kullanılabilir birçok bileşenden oluşan Tek Sayfalı Uygulamalar (SPA’lar) gibi büyük uygulamalar oluştururken, bunların oluşturulması ve bakımı hızla zorlaşabilir. Bu bileşenler arasında veri ve durum paylaşımı da hızla bozulabilir ve hata ayıklaması ve bakımı zorlaşabilir.

Merkezileştirilmiş bir uygulama veri deposu kullanarak, tüm uygulama durumu, uygulamayı daha organize hale getirerek tek bir yerde temsil edilebilir. Tek yönlü bir veri akışının, mutasyonların kullanılması ve bileşen veri erişiminin yalnızca gerekli verilere kapsamının belirlenmesi yoluyla, bileşen rolü ve uygulama durumunu nasıl etkilemesi gerektiği hakkında akıl yürütmek çok daha kolay hale gelir.

VueJS bileşenleri ayrı varlıklardır ve birbirleri arasında kolayca veri paylaşamazlar. Verileri vuex olmadan paylaşmak için data ile olayı ’emit’ ve ardından ‘on’ ile bu olayı dinlememiz ve yakalamamız gerekiyor.

bileşen 1

this.$emit('eventWithDataObject', dataObject)

bileşen 2

this.$on('eventWithDataObject', function (dataObject) {
    console.log(dataObject)
})

vuex yüklendiğinde, olayları dinlemeye gerek kalmadan herhangi bir bileşenden verilerine kolayca erişebiliriz.

this.$store.state.myData

Ayrıca mutators ile senkron olarak veri değiştirebilir, asenkron actions kullanabilir ve getter fonksiyonları ile veri alabiliriz.

Getter işlevleri, genel hesaplanmış işlevler olarak çalışabilir. Onlara bileşenlerden erişebiliriz:

this.$store.getters.myGetter

Eylemler küresel yöntemlerdir. Bunları bileşenlerden gönderebiliriz:

this.$store.dispatch('myAction', myDataObject)

Ve mutasyonlar, vuex’te verileri değiştirmenin tek yoludur. Değişiklikleri taahhüt edebiliriz:

this.$store.commit('myMutation', myDataObject)

Vuex kodu şöyle görünür

state: {
  myData: {
    key: 'val'
  }
},
getters: {
  myGetter: state => {
     return state.myData.key.length
  }
},
actions: {
  myAction ({ commit }, myDataObject) {
    setTimeout(() => {
      commit('myMutation', myDataObject)
    }, 2000)
  }
},
mutations: {
  myMutation (state, myDataObject) {
    state.myData = myDataObject
  }
}

Vuex nasıl kurulur? #

Vuex’i kullanacağınız çoğu zaman, tek dosyalar kullanıyorsanız, Vueify ile birlikte Webpack veya Browserify gibi bir modül paketleyici kullandığınız daha büyük bileşen tabanlı uygulamalarda olacaktır.

Bu durumda Vuex’i almanın en kolay yolu NPM’den. Vuex’i yüklemek ve uygulama bağımlılıklarınıza kaydetmek için aşağıdaki komutu çalıştırın.

 npm install --save vuex

Aşağıdaki satırı ‘require(‘vue’)’ ifadenizden sonra yerleştirerek Vuex kurulumunuzla bağlantı Vuex’i yüklediğinizden emin olun.

Vue.use(require('vuex'))

Vuex ayrıca CDN’de de mevcuttur; en son sürümü cdnjs’den buradan edinebilirsiniz.

Otomatik kapatılabilir bildirimler #

Bu örnek, otomatik olarak kapatılabilen özel bildirimleri depolamak için dinamik olarak bir vuex modülünü kaydeder.

bildirimler.js

vuex deposunu çözün ve bazı sabitleri tanımlayın

//Vuex store previously configured on other side
import _store from 'path/to/store';

//Notification default duration in milliseconds
const defaultDuration = 8000;

//Valid mutation names
const NOTIFICATION_ADDED = 'NOTIFICATION_ADDED';
const NOTIFICATION_DISMISSED = 'NOTIFICATION_DISMISSED';

modülümüzün başlangıç ​​durumunu ayarla

const state = {
  Notifications: []
}

modül alıcılarımızı ayarla

const getters = {
  //All notifications, we are returning only the raw notification objects
  Notifications: state => state.Notifications.map(n => n.Raw)
}

modülümüzü ayarlayın Eylemler

const actions = {
  //On actions we receive a context object which exposes the 
  //same set of methods/properties on the store instance
  //{commit} is a shorthand for context.commit, this is an 
  //ES2015 feature called argument destructuring
  Add({ commit }, notification) {
    //Get notification duration or use default duration
    let duration = notification.duration || defaultDuration

    //Create a timeout to dismiss notification
    var timeOut = setTimeout(function () {
      //On timeout mutate state to dismiss notification
      commit(NOTIFICATION_DISMISSED, notification);
    }, duration);

    //Mutate state to add new notification, we create a new object 
    //for save original raw notification object and timeout reference
    commit(NOTIFICATION_ADDED, {
      Raw: notification,
      TimeOut: timeOut
    })
  },
  //Here we are using context object directly
  Dismiss(context, notification) {
    //Just pass payload
    context.commit(NOTIFICATION_DISMISSED, notification);
  }
}

modül mutasyonlarımızı ayarla

const mutations = {
  //On mutations we receive current state and a payload
  [NOTIFICATION_ADDED](state, notification) {
    state.Notifications.push(notification);
  },
  //remember, current state and payload
  [NOTIFICATION_DISMISSED](state, rawNotification) {
    var i = state.Notifications.map(n => n.Raw).indexOf(rawNotification);
    if (i == -1) {
      return;
    }

    clearTimeout(state.Notifications[i].TimeOut);
    state.Notifications.splice(i, 1);
  }
}

Modülümüzü tanımlanmış durum, alıcılar, eylemler ve mutasyon ile kaydedin

_store.registerModule('notifications', {
  state,
  getters,
  actions,
  mutations
});

Kullanım

bileşenA.vue

Bu bileşen, tüm bildirimleri ekranın sağ üst köşesinde önyükleme uyarıları olarak görüntüler, ayrıca her bir bildirimin manuel olarak kapatılmasına izin verir.

<template>
<transition-group name="notification-list" tag="div" class="top-right">
  <div v-for="alert in alerts" v-bind:key="alert" class="notification alert alert-dismissible" v-bind:class="'alert-'+alert.type">
    <button v-on:click="dismiss(alert)" type="button" class="close" aria-label="Close"><span aria-hidden="true">&times;</span></button>
    <div>
      <div>
        <strong>{{alert.title}}</strong>
      </div>
      <div>
        {{alert.text}}
      </div>
    </div>
  </div>
</transition-group>
</template>

<script>
export default {
  name: 'arc-notifications',
  computed: {
    alerts() {
      //Get all notifications from store
      return this.$store.getters.Notifications;
    }
  },
  methods: {
    //Manually dismiss a notification
    dismiss(alert) {
      this.$store.dispatch('Dismiss', alert);
    }
  }
}
</script>
