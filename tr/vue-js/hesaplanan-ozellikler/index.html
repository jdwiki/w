Veri ve Hesaplanan Özellikler #

Bir “Vue” örneğinin “veri” ve “hesaplanmış” özellikleri için temel kullanım durumu farkı, verilerin potansiyel durumuna veya değişme olasılığına bağlıdır. Belirli bir nesnenin hangi kategoride olması gerektiğine karar verirken şu sorular yardımcı olabilir:

  • Bu sabit bir değer mi? (veri)
  • Bunun değişme olasılığı var mı? (hesaplanmış veya veri)
  • Değeri diğer verilerin değerine bağlı mı? (hesaplanmış)
  • Kullanılmadan önce tamamlanması için ek verilere veya hesaplamalara ihtiyaç var mı? (hesaplanmış)
  • Değer sadece belirli koşullar altında mı değişecek? (veri)

Temel Örnek #

Şablon

<div id="example">
  a={{ a }}, b={{ b }}
</div>

JavaScript

var vm = new Vue({
  el: '#example',
  data: {
    a: 1
  },
  computed: {
    // a computed getter
    b: function () {
      // `this` points to the vm instance
      return this.a + 1
    }
  }
})

Sonuç

a=1, b=2

Burada ‘b’ hesaplanmış bir özellik ilan ettik. Sağladığımız işlev, “vm.b” özelliği için alıcı işlevi olarak kullanılacaktır:

console.log(vm.b) // -> 2
vm.a = 2
console.log(vm.b) // -> 3

“vm.b"nin değeri her zaman “vm.a"nın değerine bağlıdır.

Tıpkı normal bir özellik gibi, şablonlardaki hesaplanmış özelliklere veri bağlayabilirsiniz. Vue, “vm.b"nin “vm.a"ya bağlı olduğunun farkındadır, bu nedenle “vm.a” değiştiğinde “vm.b"ye bağlı tüm bağlamaları günceller.

Hesaplanan özellikler vs izle #

şablon

<div id="demo">{{fullName}}</div>

örnek izle

var vm = new Vue({
  el: '#demo',
  data: {
    firstName: 'Foo',
    lastName: 'Bar',
    fullName: 'Foo Bar'
  }
})

vm.$watch('firstName', function (val) {
  this.fullName = val + ' ' + this.lastName
})

vm.$watch('lastName', function (val) {
  this.fullName = this.firstName + ' ' + val
})

Bilgisayarlı örnek

var vm = new Vue({
  el: '#demo',
  data: {
    firstName: 'Foo',
    lastName: 'Bar'
  },
  computed: {
    fullName: function () {
      return this.firstName + ' ' + this.lastName
    }
  }
})

Hesaplanmış Ayarlayıcılar #

Hesaplanan özellikler, hesaplamanın bağlı olduğu herhangi bir veri değiştiğinde otomatik olarak yeniden hesaplanacaktır. Ancak, hesaplanan bir özelliği manuel olarak değiştirmeniz gerekiyorsa, Vue bunu yapmak için bir ayarlayıcı yöntemi oluşturmanıza olanak tanır:

Şablon (yukarıdaki temel örnekten):

<div id="example">
  a={{ a }}, b={{ b }}
</div>

Javascript:

var vm = new Vue({
  el: '#example',
  data: {
    a: 1
  },
  computed: {
    b: {
      // getter
      get: function () {
        return this.a + 1
      },
      // setter
      set: function (newValue) {
        this.a = newValue - 1 
      }
    }
  }

Artık alıcıyı veya ayarlayıcıyı çağırabilirsiniz:

console.log(vm.b)       // -> 2
vm.b = 4                // (setter) 
console.log(vm.b)       // -> 4
console.log(vm.a)       // -> 3

‘vm.b = 4’ ayarlayıcıyı çağırır ve this.a’yı 3’e ayarlar; uzantı olarak, vm.b 4 olarak değerlendirecektir.

v-modeli için hesaplanan ayarlayıcıları kullanma #

Hesaplanmış bir özellik üzerinde bir “v-modeli"ne ihtiyacınız olabilir. Normalde v-modeli hesaplanan özellik değerini güncellemeyecektir.

Şablon:

<div id="demo">
  <div class='inline-block card'>
    <div :class='{onlineMarker: true, online: status, offline: !status}'></div>  
    <p class='user-state'>User is {{ (status) ? 'online' : 'offline' }}</p>
  </div>

  <div class='margin-5'>
    <input type='checkbox' v-model='status'>Toggle status (This will show you as offline to others)
  </div>
</div>

stil:

#demo {
  font-family: Helvetica;
  font-size: 12px;
}
.inline-block > * {
  display: inline-block;
}
.card {
  background: #ddd;
  padding:2px 10px;
  border-radius: 3px;
}
.onlineMarker {
  width: 10px;
  height: 10px;
  border-radius: 50%;
  transition: all 0.5s ease-out;
}

.online {
  background-color: #3C3;
}

.offline {
  background-color: #aaa;
}

.user-state {
  text-transform: uppercase;
  letter-spacing: 1px;
}

.margin-5 {
  margin: 5px;
}

Bileşen:

var demo = new Vue({
    el: '#demo',
    data: {
        statusProxy: null
    },
    computed: {
        status: {
            get () {
                return (this.statusProxy === null) ? true : this.statusProxy     
            }
        }
    }
})

keman Burada, radyo düğmesine tıklamanın hiçbir faydası olmadığını göreceksiniz, durumunuz hala çevrimiçi.

var demo = new Vue({
    el: '#demo',
    data: {
        statusProxy: null
    },
    computed: {
        status: {
            get () {
                return (this.statusProxy === null) ? true : this.statusProxy     
            },
            set (val) {
                this.statusProxy = val            
            }
        }
    }
})
