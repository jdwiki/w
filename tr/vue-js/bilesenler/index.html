Bileşen(ler)de:

props, üst bileşenden veri iletmek için kullanılan bir dize değişmezleri veya nesne referansları dizisidir. Varsayılan değerlerin belirtilmesi, kabul edilen veri türü, gerekli veya isteğe bağlı olup olmadığı gibi daha ince taneli kontrollere sahip olmak istendiğinde nesne formunda da olabilir.

veri, düz bir nesne yerine bir nesne döndüren bir işlev olmalıdır. Bunun nedeni, yeniden kullanılabilirlik amacıyla bileşenin her örneğinin kendi verilerine sahip olmasını gerektirmemizdir.

olaylar, bileşenin davranış değişikliği ile yanıt verebileceği olaylar için dinleyicileri içeren bir nesnedir.

yöntemler bileşenle ilişkili davranışı tanımlayan işlevleri içeren nesne

hesaplanan özellikler tıpkı izleyiciler veya gözlemlenebilirler gibidir, herhangi bir bağımlılık değiştiğinde, özellikler otomatik olarak yeniden hesaplanır ve DOM herhangi bir hesaplanmış özellik kullanıyorsa değişiklikler anında DOM’a yansıtılır

hazır bir Vue örneğinin yaşam döngüsü kancasıdır

Kapsamlı bileşen (genel değil) #

Demo

HTML #

<script type="x-template" id="form-template">
    <label>{{inputLabel}} :</label>
  <input type="text" v-model="name" />
</script>

<div id="app">
  <h2>{{appName}}</h2>
  <form-component title="This is a form" v-bind:name="userName"></form-component>
</div>

JS #

// Describe the form component
// Note: props is an array of attribute your component can take in entry.
// Note: With props you can pass static data('title') or dynamic data('userName').
// Note: When modifying 'name' property, you won't modify the parent variable, it is only descendent.
// Note: On a component, 'data' has to be a function that returns the data.
var formComponent = {
  template: '#form-template',
  props: ['title', 'name'],
  data: function() {
    return {
      inputLabel: 'Name'
    }
  }
};

// This vue has a private component, it is only available on its scope.
// Note: It would work the same if the vue was a component.
// Note: You can build a tree of components, but you have to start the root with a 'new Vue()'.
var vue = new Vue({
  el: '#app',
  data: {
    appName: 'Component Demo',
    userName: 'John Doe'
  },
  components: {
    'form-component': formComponent
  }
});

Bileşenler nedir ve bileşenler nasıl tanımlanır? #

Vue’daki bileşenler, widget’lar gibidir. İstenen davranışla yeniden kullanılabilir özel öğeler yazmamıza izin veriyorlar.

Bunlar, oluşturulacak bir HTML şablonu da dahil olmak üzere, kök veya herhangi bir Vue örneğinin içerebileceği seçeneklerin herhangi birini/tümünü içerebilen nesnelerden başka bir şey değildir.

Bileşenler şunlardan oluşur:

  • HTML işaretlemesi: bileşenin şablonu
  • CSS stilleri: HTML işaretlemesinin nasıl görüntüleneceği
  • JavaScript kodu: veri ve davranış

Bunların her biri ayrı bir dosyaya veya ‘.vue’ uzantılı tek bir dosyaya yazılabilir. Aşağıda her iki yolu da gösteren örnekler verilmiştir:

.VUE - bileşen için tek bir dosya olarak

<style>  
    .hello-world-compoment{  
        color:#eeeeee;  
        background-color:#555555; 
    }  
</style>  

<template>
    <div class="hello-world-component">
        <p>{{message}}</p>
        <input @keyup.enter="changeName($event)"/>
    </div>
</template>

<script>
    export default{            
        props:[ /* to pass any data from the parent here... */ ],   
        events:{ /* event listeners go here */},
        ready(){
            this.name= "John";
        },
        data(){
           return{
              name:''
           }
        },
        computed:{
            message(){
                return "Hello from " + this.name;
            }
        },
        methods:{
            // this could be easily achieved by using v-model on the <input> field, but just to show a method doing it this way.
            changeName(e){
                this.name = e.target.value;
            }
        }  
    }
</script>

Ayrı Dosyalar

hello-world.js - bileşen nesnesi için JS dosyası

export default{
    template:require('./hello-world.template.html'),
    props:[ /* to pass any data from the parent here... */ ],  
    events:{ /* event listeners go here */ },
    ready(){
        this.name="John";
    },
    data(){
        return{
            name:''
        }
    },
    computed:{
        message(){
            return "Hello World! from " + this.name;
        }
    },
    methods:{
        changeName(e){
            let name = e.target.value;
            this.name = name;
        }
    }
}

merhaba-dünya.template.html

<div class="hello-world-component">
    <p>{{message}}</p>
    <input class="form-control input-sm" @keyup.enter="changeName($event)">
</div>

merhaba-dünya.css

 .hello-world-compoment{  
    color:#eeeeee;  
    background-color:#555555; 
}

Bu örnekler es2015 sözdizimini kullanır, dolayısıyla eski tarayıcılar için bunları es5’e derlemek için Babel’e ihtiyaç duyulacaktır. Babel, Browserify + vueify veya Webpack + vue-loader ile birlikte “hello-world.vue” dosyasını derlemek için gerekli olacaktır.

Artık ‘hello-world’ bileşenini tanımladığımıza göre, onu Vue.js’ye kaydettirmeliyiz.

Bu iki şekilde yapılabilir:

Küresel bir bileşen olarak kaydolun “main.js” dosyasında (uygulamanın giriş noktası) herhangi bir bileşeni global olarak “Vue.component” ile kaydedebiliriz:

import Vue from 'vue'; // Note that 'vue' in this case is a Node module installed with 'npm install Vue'
Vue.component('hello-world', require('./hello-world');  // global registeration

new Vue({
    el:'body',

    // Templates can be defined as inline strings, like so:
    template:'<div class="app-container"><hello-world></hello-world></div>'
});

Veya bir üst bileşen veya kök bileşen içinde yerel olarak kaydedin

import Vue from 'vue'; // Note that 'vue' in this case is a Node module installed with 'npm install Vue'
import HelloWorld from './hello-world.js';  

new Vue({
    el:'body',
    template:'<div class="app-container"><hello-world></hello-world></div>",

    components:{HelloWorld}  // local registeration
});

Global Components, Vue uygulamasının herhangi bir yerinde kullanılabilir.

Yerel Bileşenler yalnızca kayıtlı oldukları ana bileşende kullanılabilir.

Parça bileşeni
Sizinki bir fragment bileşeni olduğu için bir şeyi yapamayacağınızı söyleyen bir konsol hatası alabilirsiniz. Bu tür bir sorunu çözmek için bileşen şablonunuzu “

” gibi tek bir etiketin içine sarmanız yeterlidir.

Bileşenlerin yerel kaydı #

Bir bileşen, global veya yerel olarak kaydedilebilir (başka bir belirli bileşene bağlanabilir).

var Child = Vue.extend({
    // ...
})

var Parent = Vue.extend({
    template: '...',
    components: {
         'my-component': Child
    }
})

Bu yeni bileşen () yalnızca Ana bileşenin kapsamı (şablonu) içinde kullanılabilir.

Satır içi kayıt #

Bir bileşeni tek adımda genişletebilir ve kaydedebilirsiniz:

Vue.component('custom-component', {
    template: '<div>A custom component!</div>'
})

Ayrıca bileşen yerel olarak kaydedildiğinde:

var Parent = Vue.extend({
    components: {
        'custom-component': {
            template: '<div>A custom component!</div>'
        }
    }
})

Bileşenlerde veri kaydı #

Bir bileşeni kaydederken “data” özelliğine bir nesne iletmek, bileşenin tüm örneklerinin aynı verileri göstermesine neden olur. Bunu çözmek için bir fonksiyondan veri döndürmemiz gerekiyor.

var CustomComponent = Vue.extend({
    data: function () {
        return { a: 1 }
    }
})

Olaylar #

Bileşenlerin ataları/torunları ile iletişim kurma yollarından biri özel iletişim olaylarıdır. Tüm Vue örnekleri de yayıcıdır ve bir bileşen ağacı içinde iletişimi kolaylaştıran özel bir olay arabirimi uygular. Aşağıdakileri kullanabiliriz:

  • $on: Bu bileşenlerin ataları veya torunları tarafından yayılan olayları dinleyin.
  • $yayın: Tüm alt nesillere aşağı doğru yayılan bir olay yayar.
  • $dispatch: İlk önce bileşenin kendisinde tetiklenen ve ardından tüm atalara yukarıya doğru yayılan bir olay yayar.
  • $emit: Kendi kendine bir olayı tetikler.

Örneğin, form gönderildiğinde bir form bileşeninin içindeki belirli bir düğme bileşenini gizlemek istiyoruz. Ana öğede:

var FormComponent = Vue.extend({
  // ...
  components: {
    ButtonComponent
  },
  methods: {
    onSubmit () {
        this.$broadcast('submit-form')
    }
  }
})

Alt öğede:

var FormComponent = Vue.extend({
  // ...
  events: {
    'submit-form': function () {
        console.log('I should be hiding');
    }
  }
})

Akılda tutulması gereken bazı şeyler:

  • Bir olay kendisini dinleyen bir bileşen bulduğunda ve tetiklendiğinde, bu bileşendeki işlev geri çağırma işlevi ’true’ döndürmediği sürece yayılmayı durduracaktır.
  • $dispatch() her zaman ilk onu yayan bileşende tetiklenir.
  • Olay işleyicisine istediğimiz sayıda argüman iletebiliriz. this.$yayın('gönder-form', this.formData, this.formStatus) yapmak, ``gönder-form’ gibi bu argümanlara erişmemizi sağlar: function (formData, formStatus) {}`
