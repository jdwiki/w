camelCase <=> kebap çantası #

“props"larınızın adlarını tanımlarken, HTML öznitelik adlarının büyük/küçük harfe duyarlı olmadığını her zaman unutmayın. Bunun anlamı, bileşen tanımınızda deve durumunda bir ‘prop’ tanımlarsanız…

Vue.component('child', {
    props: ['myProp'],
    ...
});

…HTML bileşeninizde onu benim-prop olarak çağırmalısınız.

Sahne ile ebeveynden çocuğa Veri Aktarma #

Vue.js’de her bileşen örneğinin kendi yalıtılmış kapsamı vardır; bu, bir üst bileşenin bir alt bileşeni varsa, alt bileşenin kendi yalıtılmış kapsamı ve üst bileşenin kendi yalıtılmış kapsamı olduğu anlamına gelir.

Orta ila büyük boyutlu herhangi bir uygulama için, en iyi uygulama kurallarına uymak, geliştirme aşamasında ve sonrasında bakım sırasında çok fazla baş ağrısını önler. İzlenecek bu tür şeylerden biri, *** doğrudan alt bileşenden ana verilere atıfta bulunmaktan/mutasyona uğratmaktan kaçınmaktır***. Öyleyse, bir alt bileşen içindeki ana verilere nasıl başvuracağız?

Bir alt bileşende gerekli olan üst veri ne olursa olsun, alt öğeye üst öğeden “sahne” olarak iletilmelidir.

Kullanım Örneği: Aşağıdaki alanlara sahip iki tablo ‘users’ ve ‘addresses’ içeren bir Kullanıcı veritabanımız olduğunu varsayalım: ‘kullanıcılar’ Tablosu

ad telefon e-posta
John Mclane (1) 234 5678 9012 john@dirhard.com
James Bond (44) 777 0007 0077 bond@mi6.com

‘adresler’ Tablosu

blok sokak şehir
Nakatomi Kuleleri Broadway New York
Mi6 Evi Buckingham Yolu Londra

ve ilgili kullanıcı bilgilerini uygulamamızın herhangi bir yerinde görüntülemek için üç bileşene sahip olmak istiyoruz

user-component.js

export default{
    template:`<div class="user-component">
                <label for="name" class="form-control">Name: </label>  
                <input class="form-control input-sm" name="name" v-model="name">
                <contact-details :phone="phone" :email="email"></contact-details>
              </div>`,
    data(){
        return{
            name:'',
            phone:'',
            email:''
        }
    },
}

contact-details.js

import Address from './address';
export default{
    template:`<div class="contact-details-component>
                <h4>Contact Details:</h4>
                <label for="phone" class="form-control">Phone: </label>  
                <input class="form-control input-sm" name="phone" v-model="phone">
                <label for="email" class="form-control">Email: </label>  
                <input class="form-control input-sm" name="email" v-model="email"> 
        
                <h4>Address:</h4>
                <address :address-type="addressType"></address>
                //see camelCase vs kebab-case explanation below
            </div>`,
    props:['phone', 'email'],
    data:(){
        return:{
            addressType:'Office'
        }
    },
    components:{Address}  
}

adres.js

export default{
    template:`<div class="address-component">
                <h6>{{addressType}}</h6>
                <label for="block" class="form-control">Block: </label>  
                <input class="form-control input-sm" name="block" v-model="block">
                <label for="street" class="form-control">Street: </label>  
                <input class="form-control input-sm" name="street" v-model="street">
                <label for="city" class="form-control">City: </label>  
                <input class="form-control input-sm" name="city" v-model="city">
             </div>`,
    props:{
        addressType:{
            required:true,
            type:String,
            default:'Office'
        },
    data(){
        return{
            block:'',
            street:'',
            city:''
        }
    }
}

import Vue from 'vue';  
  
Vue.component('user-component', require'./user-component');  
Vue.component('contact-details', require'./contact-details');  

new Vue({
    el:'body'  
});

index.html

</body>

Telefon veya e-posta verileri olmayan “iletişim bilgileri"nde “kullanıcı bileşeni"nin özellikleri olan “telefon” ve “e-posta” verilerini görüntülüyoruz.

Sahne olarak veri aktarma

Bu nedenle, “” bileşenini eklediğimiz template özelliğindeki “user-component.js” içinde, phone ve ** öğelerini geçiyoruz. ‘’(ana bileşen)‘den ‘’(alt bileşen)’e e-posta** verisini dinamik olarak props - ‘:phone=“phone”öğesine bağlayarak ve:email=“email, v-bind:phone=“phone”vev-bind:email=“email”` ile aynı

Sahne - Dinamik Bağlama

Destekleri dinamik olarak bağladığımız için, ana bileşendeki telefon veya e-posta‘daki herhangi bir değişiklik, yani <user-bileşen>, hemen alt bileşene, yani <contact-'a yansıtılacaktır. ayrıntılar>.

Sahneler - Değişmez Olarak

Ancak, phone ve email değerlerini phone="(44) 777 0007 0077" email="bond@mi6.com" gibi dize değişmez değerleri olarak iletmiş olsaydık o zaman ana bileşende meydana gelen herhangi bir veri değişikliğini yansıtmaz.

Tek Yönlü ciltleme

Varsayılan olarak, değişikliklerin yönü yukarıdan aşağıya doğrudur, yani ana bileşendeki dinamik olarak bağlı aksesuarlarda yapılan herhangi bir değişiklik alt bileşene yayılır, ancak bir alt bileşendeki pervane değerlerinde yapılan herhangi bir değişiklik üst bileşene yayılmaz.

Örneğin: “” içinden “bond@mi6.com” olan e-postayı “jamesbond@mi6.com” olarak değiştirirsek, ana veri, yani “” içindeki telefon verileri özelliği yine de “bond@mi6.com” değerini içerecektir.

Ancak, ana bileşende (bizim kullanım durumumuzda “”) e-postanın değerini “bond@mi6.com"dan “jamesbond@mi6.co"ya değiştirirsek, alt öğedeki e-postanın değeri bileşeni (kullanım durumumuzda <contact-details>) otomatik olarak jamesbond@mi6.com olarak değişecektir - ebeveyndeki değişiklik anında çocuğa yayılır.

İki Yönlü Bağlama

İki yönlü bağlama istiyorsak, iki yönlü bağlamayı :email="email" yerine açıkça :email.sync="email" olarak belirtmeliyiz. Şimdi, alt bileşendeki prop değerini değiştirirsek, değişiklik ana bileşene de yansıyacaktır.

Orta ila büyük bir uygulamada, alt durumdan ana durumu değiştirmek, özellikle hata ayıklama sırasında tespit etmek ve takip etmek çok zor olacaktır - Dikkatli olun .

Vue.js 2.0’da herhangi bir .sync seçeneği olmayacak. Sahne öğeleri için iki yönlü bağlama, Vue.js 2.0’da kullanımdan kaldırılmıştır.

Tek seferlik Bağlama

Açık tek seferlik bağlamayı :email.once="email olarak tanımlamak da mümkündür, bir hazır bilgi iletmeye az çok benzer, çünkü üst özellik değerindeki sonraki değişiklikler yayılmayacaktır. çocuk.

UYARI Nesne veya Dizi prop olarak iletildiğinde, bunlar HER ZAMAN REFERANS TARAFINDAN GEÇİLİR, yani açıkça tanımlanan bağlama türünden bağımsız olarak :email.sync="email" veya :email="email " veya :email.once="email", e-posta üst öğedeki bir Nesne veya Dizi ise, bağlama türünden bağımsız olarak, alt bileşen içindeki prop değerindeki herhangi bir değişiklik, ebeveyndeki değeri şu şekilde etkiler: kuyu.

Dizi olarak sahne

‘contact-details.js’ dosyasında ‘props:[‘phone’, ’email’]’ öğesini bir dizi olarak tanımladık, bu, props ile ince taneli kontrol istemiyorsak sorun değil.

** Nesne Olarak Sahne **

Sahne öğeleri üzerinde daha ince taneli kontrol istiyorsak,

  • prop olarak ne tür değerlerin kabul edilebilir olduğunu tanımlamak istiyorsak
  • pervane için varsayılan değer ne olmalıdır
  • prop için bir değerin iletilmesi ZORUNLU (gerekli) mi yoksa isteğe bağlı mı

daha sonra, `address.js’de yaptığımız gibi, props’u tanımlamak için nesne notasyonu kullanmamız gerekiyor.

Ekipteki diğer geliştiriciler tarafından da kullanılabilecek yeniden kullanılabilir bileşenler yazıyorsak, bileşeni kullanan herkesin veri türünün ne olması gerektiği ve olup olmadığı konusunda net bir fikre sahip olması için sahne öğelerini nesneler olarak tanımlamak iyi bir uygulamadır. zorunlu veya isteğe bağlıdır.

Ayrıca sahne doğrulama olarak da adlandırılır. type aşağıdaki yerel kuruculardan herhangi biri olabilir:

  • Sicim
  • Sayı
  • Boole
  • Dizi
  • Nesne
  • İşlev
  • veya bir Özel Yapıcı

http://vuejs.org/guide/components.html#Props adresinden alınan bazı prop doğrulama örnekleri

Vue.component('example', {
   props: {
       // basic type check (`null` means accept any type)
       propA: Number,
       // multiple possible types (1.0.21+)
       propM: [String, Number],
       // a required string
       propB: {
         type: String,
         required: true
       },
       // a number with default value
       propC: {
          type: Number,
          default: 100
       },
       // object/array defaults should be returned from a
       // factory function
       propD: {
          type: Object,
          default: function () {
             return { msg: 'hello' }
         }
       },
       // indicate this prop expects a two-way binding. will
       // raise a warning if binding type does not match.
       propE: {
          twoWay: true
       },
       // custom validator function
       propF: {
          validator: function (value) {
             return value > 10
          }
       },
       // coerce function (new in 1.0.12)
       // cast the value before setting it on the component
       propG: {
          coerce: function (val) {
            return val + '' // cast the value to string
          }
       },
       propH: {
          coerce: function (val) {
            return JSON.parse(val) // cast the value to Object
          }
       }
    }
});

camelCase vs kebap-case

HTML öznitelikleri büyük/küçük harfe duyarlı değildir, yani “addresstype” ve “addressType” arasında ayrım yapamaz, bu nedenle öznitelik olarak camelCase prop adlarını kullanırken onların kebab-case (tireyle ayrılmış) eşdeğerlerini kullanmamız gerekir: “addressType”, HTML özniteliğinde “address-type” olarak yazılmalıdır.

Vue JSX Kullanırken Propları Geçmek #

Bir ana bileşenimiz var: İçindeki bir alt bileşeni içe aktarırken, bir öznitelik aracılığıyla sahne öğelerini ileteceğiz. Burada öznitelik ‘src’ ve biz de ‘src’yi geçiyoruz.

ParentComponent.js #

import ChildComponent from './ChildComponent';
export default {
    render(h, {props}) {
        const src = 'https://cdn-images-1.medium.com/max/800/1*AxRXW2j8qmGJixIYg7n6uw.jpeg';
        return (
           <ChildComponent src={src} />   
        );
    }
};

Ve sahne malzemelerini geçmemiz gereken bir alt bileşen. Hangi sahneleri geçtiğimizi belirtmemiz gerekiyor.

ChildComponent.js: #

export default {
    props: ['src'],
    render(h, {props}) {
        return ( 
            <a href = {props.src} download = "myimage" >
                Click this link
            </a>
        );
    }
};

Dinamik Sahne #

Bir görünümden modele veri bağlayabildiğiniz gibi, üst bileşenlerden alt bileşenlere bilgi aktarmak için aynı v-bind yönergesini kullanarak sahne öğelerini de bağlayabilirsiniz.

JS #

new Vue({
    el: '#example',
    data: {
        msg: 'hello world'
    }
});

Vue.component('child', {
    props: ['myMessage'],
    template: '<span>{{ myMessage }}</span>
});

HTML #

<div id="example">
    <input v-model="msg" />
    <child v-bind:my-message="msg"></child>
    <!-- Shorthand ... <child :my-message="msg"></child> -->
</div>
