SugarCRM’in RESTful API’si, SugarCRM 7.x sürümlerinde sürüm 10’a güncellendi.

7.x sürümleri tek sayfalık bir uygulama olarak oluşturulduğundan, API, verileri almak için sunucuyla ana iletişim aracıdır. Üçüncü taraf entegrasyonları da verilere bağlanabilir ve bunlara erişebilir. API, istenen işlevselliği uygulayan özel uç noktalar sağlamak için de genişletilebilir.

Resmi belgeler

Giriş #

Bu örneği koddan bir çağrı olarak veya Postman gibi bir REST istemcisi aracılığıyla kullanabilirsiniz:

“POST https://YOURSITE.com/rest/v10/oauth2/token”

{
"grant_type":"password",
"client_id":"sugar",
"client_secret": "",
"username":"your_username",
"password":"your_P@$sw0rd",
"platform":"api"
}

Yukarıdaki ayrıntıları detaylandırmak için, bunlar şu anlama gelir:

“grant_type”:“password” - Bu, oturum açma yöntemidir. Bu durumda, bir kullanıcı adı/şifre kombinasyonu kullanıyoruz, bu yüzden “parola” giriyoruz.

“client_id”:“sugar” - Bu, standart “şeker” istemci adı/parola aracılığıyla kimliğimizi doğruladığımızı gösterir. “support_portal” gibi diğer istemciler. Bu ayrıca uygulama içindeki özel bir OAuth anahtarına başvurabilir.

“client_secret”: “” - Bu, “sugar"ın istemci kimliğini kullandığımız için boştur, ancak özel bir istemci kullanıyorsanız, istemciyle ilişkili sır budur.

“username”:“your_username” - Kullanıcı adınız

“password”:“your_P@$sw0rd” - Şifreniz

“platform”:“api” - Şeker içinde kullanılan tipik platformlar “baz”, “mobil” ve “portal"dır. Güvenlik nedeniyle, aynı kullanıcı aynı platformda aynı anda oturum açarsa, kullanıcının önceki oturumdan çıkış yapmasına neden olur. Ben “api” kullandım, ama isterseniz bu “myawesomeşeker entegrasyonu” olabilir.

Başarılı bir istek üzerine, sunucu bir “access_token” içeren bir nesne döndürür, yani:

“access_token”:”abcdef01-2345-6789-0abc-def012345678”

API’ye yapılan sonraki her istek için, kendinizi doğrulamak için bu belirteci eklemelisiniz.

Bu, diğer isteklere access_token değeriyle “OAuth-Token”` başlığını ekleyerek başarılabilir.

Temel CRUD işlemleri #

Öncelikle oturum açtığınızdan emin olun ve access_token’ı istek başlığınıza ekleyin.

Bu örnekte, kayıtlara erişmek için bazı temel işlemler yapacağız. Bunlar, Örnek olarak Hesaplar modülünü kullanır, ancak diğer standart ve özel modüller (yani Müşteri Adayları, İlgili Kişiler, Fırsatlar) aynı şekilde davranır.

Oluşturmak

Bir Hesap oluşturmak için, eklemek istediğimiz ayrıntılarla birlikte Hesaplar uç noktasına bir gönderi talebi yapmamız gerekiyor. Başarılı olduğunda, bu, yeni kaydın kimliğini ve mevcut verileri içeren bir nesne döndürür.

POST https://YOURSITE.com/rest/v10/Accounts
{
"name":"My New Account"
}

Okumak

İlk olarak, az önce oluşturduğumuz kaydı alacağız, benim için kimlik “9174c58c-409c-11e7-bfdf-00163ef1f82f” idi, bu nedenle kayıt için tüm bilgileri almak için aşağıdakileri yapıyoruz:

GET https://YOURSITE.com/rest/v10/Accounts/9174c58c-409c-11e7-bfdf-00163ef1f82f

Bu kesinlikle büyük bir nesne! Sadece ismi ve onu yarattığım tarihi görmeye ne dersin?

GET https://YOURSITE.com/rest/v10/Accounts/9174c58c-409c-11e7-bfdf-00163ef1f82f?fields=name,date_entered

Çok daha iyi. Ama ya sistemde zaten binlerce Hesabım varsa ve bunun için GUID’i hatırlamayı başaramadıysam?

GET https://YOURSITE.com/rest/v10/Accounts?fields=name,date_entered&filter[0][name]=My New Account

Güncelleme

Peki ya uygulamada bir şeyi değiştirmek istediğime karar verirsem? Adı değiştirmeye ve bir açıklama eklemeye ne dersiniz?

PUT https://YOURSITE.com/rest/v10/Accounts/9174c58c-409c-11e7-bfdf-00163ef1f82f
{
    "name":"My Updated Account",
    "description":"Updated via REST API"
}

Silmek

Tamam, sorun değil, ama gerçek verileri koymadan önce bunu açıklığa kavuşturalım:

DELETE https://YOURSITE.com/rest/v10/Accounts/9174c58c-409c-11e7-bfdf-00163ef1f82f

Özel Uç Noktalar oluşturma #

SugarCRM 7.x’teki özelliklerden biri, ihtiyaç duyduğunuz şeyi gerçekleştirmek için özel uç noktaları kolayca ekleyip genişletebilmektir.

Bu örnekte, istekle ilgili bazı verileri döndürmek için birkaç özel uç nokta oluşturacağız.

Bu özel dosya ‘custom/clients/base/api/DescriptionAPI.php’ içine yerleştiriliyor.

<?php

if(!defined('sugarEntry') || !sugarEntry) die('Not A Valid Entry Point');

//You need to inherit the SugarApi Class to
class DescriptionApi extends SugarApi
{
    public function registerApiRest()
    {
        return array(

            //Define a key for the array
            'DescribeRequest' => array(

                //Array of the acceptable kinds of requests for this method
                'reqType' => array('GET','POST','PUT','DELETE'),

                //If true, anyone can access. If false, only authenticated users.
                'noLoginRequired' => true,

                //Here is the path to access the endpoint, in this case: Describe/Request
                'path' => array('Describe', 'Request'),

                //Specify an empty string for the path variables
                'pathVars' => array('', ''),

                //method to call
                'method' => 'DescribeMyRequest',

                //A small description, displayed in rest/v10/help page
                'shortHelp' => 'Describes your Request Method',

                //Further help, displayed when drilling down into the help page
                'longHelp' => 'custom/clients/base/api/help/DescribeRequestHelp.html',
            ),
            //Here's another entry with some more in depth information
            'DescribeIncludingArgs' => array(
                'reqType' => array('GET','POST','PUT','DELETE'),
                'noLoginRequired' => true,

                //This time, we'll include a third element with a ?
                //So now the path is Describe/Request/{dataFromURL}
                'path' => array('Describe', 'Request', '?'),

                //Here, we specify the key for accessing that data within the function
                'pathVars' => array('', '','dataFromURL'),
                
                'method' => 'DescribeMyRequestIncludingArguments',
                'shortHelp' => 'Describes the request you sent, including method, URL parameters, and request body',
                'longHelp' => 'custom/clients/base/api/help/DescribeIncludingArgsHelp.html',
            ),
        );

    }

    /**
     * Your custom logic goes in here.
     */
    public function DescribeMyRequest($api, $args)
    {
        //Find out the request method sent
        $requestType = $_SERVER['REQUEST_METHOD'];

        return "You sent a $requestType request.";
    }

    /**
     * Here is the second function
     */
    public function DescribeMyRequestIncludingArguments($api, $args)
    {
        //Find out the request method sent
        $requestType = $_SERVER['REQUEST_METHOD'];

        //Get the data included in the URL parameter
        $data = $args['dataFromURL'];

        //Read from the request body
        $body = file_get_contents('php://input');

        return "You sent a $requestType request including the header argument: `$data` and the body: `$body`";
    }

}

Bu dosyayı ekledikten sonra Sugar’ın uç noktanızı doğru bir şekilde kaydetmesi için bir Onarım ve Yeniden Oluşturma yapmanız gerekir.

Ardından, tarayıcınızda dinlenme/v10/Açıklama/Talep’e giderseniz şunları görmelisiniz:

“Bir GET isteği gönderdiniz.”

Şimdi, bazı verilerle bir POST isteği göndermek için bir REST istemcisi kullanıyorsanız, örneğin: {"key":"value"} gövdesiyle POST rest/v10/Describe/Request/Stuff almalısınız. :

“Başlık bağımsız değişkenini içeren bir POST isteği gönderdiniz: ‘Stuff’ ve gövde: {\"anahtar\":\"değer\"}”
