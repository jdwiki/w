Şifreli Sorgu Dili #

Bu, Neo4j’nin sorgulama dili olan Cypher’dır. Cypher birçok yönden SQL’e benzer, eğer aşinaysanız SQL’in bir tabloda depolanan öğeleri, Cypher ise bir grafikte depolanan öğeleri ifade etmesi dışında.

İlk olarak, bir grafiğin nasıl oluşturulacağını ve ilişkiler ekleneceğini öğrenerek başlamalıyız, çünkü esasen Neo4j’nin konusu budur.

CREATE (ab:Object { age: 30, destination: "England", weight: 99 })
  • Veri oluşturmak için CREATE kullanırsınız
  • Bir düğümü belirtmek için parantez kullanırsınız: ()
  • ab:Object kısmı şu şekilde ayrılabilir: yeni düğüm için bir ‘ab’ değişkeni ve ‘Object’ etiketi. Değişkenin herhangi bir şey olabileceğini unutmayın, ancak bir Cypher Query satırında tutarlı olmanız gerekir.
  • Düğüme özellikler eklemek için köşeli ayraçları kullanın: {} köşeli ayraçlar

Daha sonra, MAÇLARI bulmayı öğreneceğiz

MATCH (abc:Object) WHERE abc.destination = "England" RETURN abc;

KAÇINCI, belirli bir düğüm/ilişki modelini aramak istediğinizi belirtir. (abc:Object), eşleşmeleri abc değişkeninde saklayan (Object etiketli) bir düğüm Modeline atıfta bulunur. Bu satırın tamamını aşağıdaki gibi düşünebilirsiniz.

abc =  find the matches that is an Object WHERE the destination is England.

Bu durumda WHERE, hedefin İngiltere olması gerektiği şeklinde bir kısıtlama ekler. Tüm MATCH sorgularının sonuna bir dönüş eklemelisiniz (neo4j yalnızca bir Eşleşmeyi kabul etmeyecektir…sorgunuz her zaman bir değer döndürmelidir [bu ayrıca ne tür bir sorgu yazdığınıza da bağlıdır…hakkında daha fazla konuşacağız… Bunu daha sonra, yapabileceğiniz diğer sorgu türlerini tanıtacağız].

Sonraki satır gelecekte, Cypher Query Language’in bazı öğelerini gözden geçirdikten sonra açıklanacaktır. Bu, bu dille neler yapabileceğimize dair size bir fikir vermek için! Aşağıda, başlığı ‘T’ ile başlayan filmlerin kadrosunu alan bir örnek bulacaksınız.

MATCH (actor:Person)-[:ACTED_IN]->(movie:Movie)
WHERE movie.title STARTS WITH "T"
RETURN movie.title AS title, collect(actor.name) AS cast
ORDER BY title ASC LIMIT 10;

Komutların ve söz dizimlerinin tam listesi resmi Neo4j Cypher Reference Card burada bulunabilir.

Kurulum veya Kurulum #

Web tarayıcınız aracılığıyla OS platformunu algılaması gereken Install Neo4j’e gidin, OS’niz için normal kurulum talimatlarını indirin ve izleyin.

Neo4j, Java ile oluşturulmuştur, bu nedenle Java’nın kurulu olduğu herhangi bir platformda çalışır, ancak Neo4j ekibi, popüler platform için kolay kurulum paketleri (örneğin, Mac için bir .dmg, Debian ve Ubuntu için bir .deb, bir .exe) sağlayarak kurulumu basitleştirmiştir. Windows 64 ve 32 bit platformlar için…).

Mevcut diğer sürümleri ve platformları incelemek için Diğer Neo4j Sürümleri Sayfası’na bakın

Neo4j’yi Docker kapsayıcısı olarak kurun:

## Required : Docker machine, docker cli

# Pull neo4j image from the docker hub
docker pull neo4j

# create the docker container
docker run \
    --publish=7474:7474 --publish=7687:7687 \
    --volume=$HOME/neo4j/data:/data \
    neo4j

# If you are running docker directly on the host (e.g ubuntu, RHEL, CentOs etc)
#     Access the neo4j console at http://localhost:7474
# If you are on OSX/ Windows
#     Access the neo4j console at http://<docker-machine-ip>:7474

Neo4j sunucusunu yükleme ve başlatma #

Ön koşul adımları:

  • Java’yı makinenize yükleyin
  • neo4j web sitesini ziyaret edin ve “Topluluk Sürümünü İndir” bağlantısını tıklayın veya doğrudan indirme bağlantısını ziyaret edin.
  • .tar indirilen dosyayı ana dizininizde açın

Neo4j’yi konsoldan başlatın (başsız, web sunucusu olmadan) #

  • Çıkarılan klasörün /bin alt dizinini ziyaret edin ve terminalde çalıştırın `./neo4j konsolu’
  • Artık terminalde neo4j sorguları çalıştırabilirsiniz

Neo4j web sunucusunu başlatın #

  • Çıkarılan klasörün /bin alt dizinini ziyaret edin ve terminalde çalıştırın ./neo4j başlat
  • http://localhost:7474/ adresini ziyaret edin
  • Yalnızca ilk kez, varsayılan hesapla oturum açmanız ve varsayılan şifreyi değiştirmeniz gerekecektir. Topluluk sürümü 3.0.3’ten itibaren, varsayılan kullanıcı adı ve parola neo4j ve neo4j’dir.
  • Artık web tarayıcınızda sağlanan konsolda Neo4j sorguları ekleyebilir ve her sorgunun sonuçlarını görsel olarak inceleyebilirsiniz.

Neo4j web sunucusunu başlatın #

Şu anda her Neo4j sunucusu (topluluk sürümünde) tek bir Neo4j veritabanını barındırabilir, bu nedenle yeni bir veritabanı kurmak için:

  • /bin alt dizinini ziyaret edin ve sunucuyu durdurmak için ./neo4j stop komutunu çalıştırın
  • /conf alt dizinini ziyaret edin ve dbms.active_database parametresinin değerini oluşturmak istediğiniz yeni veritabanının adıyla değiştirerek neo4j.conf dosyasını düzenleyin.
  • /bin alt dizinini tekrar ziyaret edin ve ./neo4j start komutunu çalıştırın
  • Web sunucusu yeni boş veritabanıyla yeniden başladı. Yeni veritabanı ile çalışmak için http://localhost:7474/ adresini tekrar ziyaret edebilirsiniz.
  • Oluşturulan veritabanı /data/databases alt dizininde, dbms.active_database parametresinde belirtilen ada sahip bir klasör altında bulunur.

Veritabanlarından birini silin #

  • Neo4j sunucusunun çalışmadığından emin olun; /bin alt dizinine gidin ve ./neo4j status komutunu çalıştırın. Çıktı mesajı sunucunun çalıştığını gösteriyorsa, ./neo4j stop komutunu da yürütün.
  • Ardından /data/databases alt dizinine gidin ve kaldırmak istediğiniz veritabanının klasörünü silin.
