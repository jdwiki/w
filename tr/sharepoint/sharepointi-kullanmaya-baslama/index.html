Tek Sunucu Grubu için SharePoint 2016 Kurulumu #

giriiş #

SharePoint 2016, SharePoint ürün ailesinin sürüm 16 sürümüdür. 4 Mayıs 2016’da yayımlanmıştır. Bu örnek, Tek Sunucu Grubu yapılandırması kullanılarak SharePoint 2016’nın yüklenmesini kapsar. Bu yapılandırma, birden çok sunucuya ihtiyaç duymadan bir SharePoint grubu kurmanın temellerini kapsar. Tek Sunucu Çiftliği tarafından kapsanan senaryoların genellikle geliştirme ve çok küçük üretim senaryolarıyla sınırlı olduğunu unutmayın.

Gereksinimler #

SharePoint’i yüklemeden önce temel ortamın ayarlanması gerekir. SharePoint, belgeleri, meta verileri, günlükleri, özel uygulamaları, özelleştirmeleri ve çok daha fazlasını depolar. Temel gereksinimlerin üzerinde yeterli disk alanınız ve RAM’iniz olduğundan emin olun.

  • 64 bit uyumlu işlemcilerde 4 Çekirdek
  • 12 - 24 GB RAM (test veya ürün dağıtımına bağlı olarak)
  • Sistem için 80GB sabit disk
  • İkinci sürücü olarak 100 GB sabit sürücü
  • 64-bit Windows Server 2012 R2 veya Teknik Önizleme “Eşik” içeren sunucu
  • SQL Server 2014 veya SQL Server 2016
  • .NET Framework 4.5.2 veya .NET Framework 4.6
  • Etki alanına katılmış bilgisayar ve temsilci çiftlik hizmeti hesapları

Diğer tüm önkoşullar manuel olarak yüklenebilir veya SharePoint kurulumunda bulunan SharePoint Önkoşul yükleyicisi kullanılarak yapılabilir.

Kurulum #

  • Önkoşul yükleyicisini çalıştırın; devam etmeden önce sunucuyu yeniden başlatmayı isteyebilir
  • SharePoint kurulumundan Setup.exe’yi çalıştırın
  • Lisans anahtarını girin
  • Lisans sözleşmesini kabul edin
  • Sunucu Türü Sekmesinde “Tamamla” yı seçin
  • Kurulum başarıyla tamamlanmalıdır
  • Tam sayfada, Ürün Yapılandırma Sihirbazını Çalıştır’ın yanındaki onay kutusunu işaretli bırakın ve Kapat’ı tıklayın.

Yapılandırma #

Önceki adımdan devam ediyorsanız, SharePoint 2016 Ürün Yapılandırma Sihirbazı otomatik olarak açılmalıdır. kutu görünmüyorsa veya yapılandırmayı daha sonra çalıştırıyorsanız, Başlat -> SharePoint 2016 Ürünleri -> SharePoint 2016 Ürün Yapılandırma Sihirbazı’na giderek yapılandırma sihirbazını açın.

  • Karşılama sayfasında ileriyi tıklayın
  • Yapılandırma sırasında bazı hizmetlerin yeniden başlatılacağını söyleyen kalıcı bir iletişim kutusu açılır; henüz hiçbir şey yüklenmedi, bu yüzden evet’i tıklayın
  • Çiftlik için veritabanı sunucusunu ekleyin
    • Enter the name of the machine running SQL Server; in this case, it is the local machine
    • Enter the name of the Configuration database or keep the default name SharePoint_Config
    • Enter the username of the domain service user who will be accessing the database (in the form of DOMAIN\user) *Enter the password for the domain user
    • Click next when done
  • Çiftlik şifresini girin; bu, yeni çiftliğe ek sunucular katılırken kullanılacaktır.
  • Tek Sunucu Grubu rolünü seçin
  • Central Admin Web Uygulamasını yapılandırın (SharePoint’in çiftlik yöneticileri tarafından yönetileceği yer) bağlantı noktası numarasını seçin ve kimlik doğrulama federasyonu türünü seçin (NTLM veya Negotate (Kerberose))
  • Son sayfalardaki ayarları gözden geçirin ve gerekli değişiklikleri yapın
  • Hazır olduğunuzda, birkaç dakika sürebilecek yapılandırmayı çalıştırın
  • Tamamlandığında, Sihirbazı açacaksınız, Central Admin sitesini açmanıza izin verecek
  • Başarısızlık durumunda, %COMMONPROGRAMFILES%\Microsoft Shared\Web Server Extensions\16\LOG klasöründeki günlükleri inceleyebilirsiniz.

Çiftlik Yapılandırması #

Merkezi web uygulaması, yapılandırma veritabanı ve merkezi yönetici ayarlandıktan sonra, kullanıcılar veya geliştirme için çiftliği yapılandırmaya hazır olacaksınız. Central Admin sitesinin konumunu işaretleyebilir veya Ürün Yapılandırma Sihirbazı ile aynı konumdaki bir kısayol aracılığıyla siteye erişebilirsiniz.

  • Yapılandırmayı daha sonra başlatıyorsanız, Hızlı Başlat -> Yapılandırma Sihirbazları -> Çiftlik Yapılandırma Sihirbazı’na tıklayın.
  • Sihirbazı kurulum adımından başlatıyorsanız, Sihirbazı Başlat’a tıklayın.
  • Evet veya Hayır’a tıklayarak müşteri iyileştirme programının bir parçası olmak isteyip istemediğinizi seçin.
  • Çiftlik yapılandırma sayfasında, çiftlikte arka plan hizmetlerini çalıştıracak etki alanı hesabını seçin
    • While this account may be the same as the database account, they may also be different for separation of roles and privileges
    • Enter the account as DOMAIN\user
  • Hizmetler sayfasında çiftlikte olmasını istediğiniz hizmetleri doğrulayın
  • Çiftlikte ilk site koleksiyonunu oluşturun (bu adım atlanabilir ve daha sonra yapılabilir)
    • Enter the site collection’s title, description, web address (usually the first site is at the server root), and the template
    • Most things can be changed (title, description) can be changed easily, but others like the web URL may take much more work to change; the template can also not be easily rolled back, but SharePoint allows a large amount of customizations that allows you to take any base template and convert the style and layout of the site
  • Yapılandırmayı tamamladığınızda, bitir’i tıklayın.

Çiftlik ve ilk site koleksiyonu artık kullanım için yapılandırıldı.

SharePoint Framework ile bir web bölümü oluşturun #

dev.office.com/sharepoint, SharePoint Framework’ü kullanmaya başlamak için harika bir yerdir.

SharePoint Framework, başlangıçta Office 365’te SharePoint Online’ı hedefleyen SharePoint Geliştirmeye yönelik modern, istemci tarafı bir yaklaşımdır. SharePoint Framework ile oluşturulan web bölümleri yeni bir web bölümü türüdür ve hem mevcut SharePoint sayfalarına hem de yeni SharePoint sayfaları.

İlk SharePoint istemci tarafı web bölümünüzü oluşturun (Merhaba Dünya bölüm 1)‘de barındırılan bu işlem için harika bir merhaba dünya örneği var. dev.office.com’daki tüm örnekler, github aracılığıyla topluluk katkıları için kullanılabilir.

Merhaba Dünya’nın SharePoint Çerçevesindeki temel adımları şunlardır:

  1. Yeoman SharePoint Generator ile projenin iskeletini oluşturun.

    yo @microsoft/SharePoint

  2. Oluşturulan kodu istediğiniz düzenleyicide düzenleyin. Visual Studio Code desteği, platformlar arasında güçlüdür.

  3. Gulp ve yerel SharePoint Workbench’i kullanarak web bölümünü önizleyin

    gulp serve

  4. SharePoint Online ortamınızda önizleme yapın

Şu URL’ye gidin: ‘https://sharepoint-site/_layouts/workbench.aspx’

SharePoint ULS Günlükleri ve Günlük Kaydı #

SharePoint Unified Logging Service (ULS), hem operasyonlar hem de geliştiriciler için destek ve hata ayıklama yetenekleri sağlar. Günlüklerin nasıl okunacağını anlamak, sorunları çözmek için önemli bir ilk adımdır.

Takım #

Microsoft, çiftlik çalışırken o anda yazılmakta olan eski günlüklerin ve günlüklerin okunmasına yardımcı olmak için ULS Görüntüleyici sağlar. Ayrıca, bir sorunu daraltmaya yardımcı olmak için günlükleri filtreleyebilir ve biçimlendirmeyi uygulayabilir.

Korelasyon Tanımlayıcı #

Bir sorunu izole etmek için yalnızca belirli bir korelasyon kimliğine bakmak yararlıdır. Her korelasyon kimliği, sistemin bir isteğiyle veya uçtan uca eylemiyle (zaman çalışanı gibi) ilişkilendirilir. Oluşturulan bir web sayfasıyla ilgili bir sorun varsa, isteği ULS günlüklerinde bulmak ve belirli korelasyon kimliğine yalıtmak, diğer günlüklerdeki tüm gürültüyü ortadan kaldırarak sorunun tam olarak belirlenmesine yardımcı olur.

Koduma SPMonitoredScope Ekleme #

Günlüğe kaydetme ve bazı performans izleme eklemeyi anlamanın bir yolu, kodunuza SPMonitoredScope eklemektir.

using (new SPMonitoredScope("Feature Monitor"))
{
    // My code here 
}

Bu kod, isteklerinizin başlangıcını ve sonunu ve ayrıca bazı performans verilerini günlüğe kaydeder. ISPScopedPerformanceMonitor uygulayan kendi özel monitörünüzü oluşturmak, bir dizi kod için izleme düzeyini veya maksimum yürütme süresini ayarlamanıza olanak tanır.
