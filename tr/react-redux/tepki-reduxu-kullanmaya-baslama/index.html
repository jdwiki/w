React Redux kullanarak Merhaba Dünya #

Bu kılavuz, ‘react’, ‘redux’, ‘react-router’ ve ‘react-redux’‘u zaten yüklediğinizi ve ‘react’, ‘redux’ ve ‘react-router’ı yapılandırdığınızı varsayar. yapmadım, lütfen yapın.

Not: ‘react-router’, ‘react-redux’ bağımlılığına bağlı olmasa da, yönlendirme için tepki uygulamamızda kullanmamız çok muhtemeldir ve bu, tepki-redux’u kullanmamızı gerçekten kolaylaştırır .

DOSYA ADI: “app.js”

'use strict';

import React from 'react';
import { render } from 'react-dom';
import { Router, Route, Link, browserHistory, IndexRoute } from 'react-router';
import { Provider } from 'react-redux';
import store from './stores';

render(
  (
    <Provider store={ store }>
      <Router history={ browserHistory }>
        {/* all the routes here */}
      </Router>
    </Provider>
  ),
  document.getElementById('app')
);

Bu dosya çoğunuz için mantıklı olacaktır, burada yaptığımız şey ./stores dizininden mağazayı almak ve tarafından sağlanan Yüksek Dereceli Bileşen Sağlayıcı kullanarak tüm rotalara geçirmektir. “tepki-redux”.

Bu, mağazayı uygulamamız boyunca kullanılabilir hale getirir.

Şimdi bu senaryoyu ele alalım. “Kullanıcı” redüktöründen veri alan ve tıklandığında mağazadaki verileri güncelleyen bir butona sahip olan bir “UserComponent” bileşenimiz var.

Uygulama Yapısı

‘rootReducer’ımızda ‘kullanıcı’ redüktörü var

const rootReducer = combineReducers({
    user: userReducer,
})
export default rootReducer;

‘userReducer’ımız şuna benziyor

const default_state = {
  users: [],
  current_user: {
    name: 'John Doe',
    email: 'john.doe@gmail.com',
    gender: 'Male'
  },
  etc: {}
};

function userReducer( state=default_state, action ) {

  if ( action.type === "UPDATE_CURRENT_USER_DATA" ) {
    return Object.assign( {}, state, { current_user: Object.assign( {}, state.current_user, { [action.payload.field]: action.payload.value } ) } );
  }
  else {
    return state;
  }

}

export default userReducer;

Ve bizim ’eylemler’ dosyamız şuna benziyor

export function updateCurrentUserData( data ) {
  return {
    type: "UPDATE_CURRENT_USER_DATA",
    payload: data
  }
}

Son olarak, bileşenimiz üzerinde çalışalım

DOSYA ADI: “UserComponent.js”

'use strict';

import React from 'react';
import { connect } from 'react-redux';
import * as Action from './actions';

let UserComponent = (props) => {

  let changeUserDetails = (field, value) => {
    // do nothing
  }

  return(
    <div>
      <h1>Hello { props.current_user.name }</h1>
      <p>Your email address is { props.current_user.email }</p>
      <div style={{ marginTop: 30 }}>
        <button onClick={ () => { changeUserDetails('name', 'Jame Smith') } }>Change Name</button>
        <button onClick={ () => { changeUserDetails('email', 'jane@gmail.com') } }>Change Email Address</button>
      </div>
    </div>
  )

}

export default UserComponent;

Tabii ki bu işe yaramaz, Henüz mağazaya bağlamadığımız için.

Merak ediyorsanız, bu durumsuz bir işlevsel bileşendir, çünkü `redux’ kullanıyoruz ve bileşenimiz için gerçekten bir dahili duruma ihtiyacımız yok, onu kullanmak için doğru zaman.

“react-redux” tarafından sağlanan “connect” yöntemi üç parametre alır

mapStateToProps, mapDispatchToProps ve Bileşen‘in kendisi.

connect( mapStateToProps, mapDispatchToProps )(Bileşen)

UserComponent bileşenimize mapStateToProps ve mapDispatchToProps ile birlikte connect ekleyelim

Ayrıca changeUserDetails işlevimizi de güncelleyelim, bu nedenle çağrıldığında, ‘redüktörlerimize’ bir ’eylem’ ‘gönderecek’ ve eylemin türüne bağlı olarak redüktörümüz devreye girecek ve mağazada değişiklikler yapacak, ve mağaza güncellendiğinde ‘react-redux’ bileşenimizi yeni verilerle yeniden oluşturacaktır.

Karmaşık geliyor mu? Gerçekten değil.

UserComponent.js dosyamız şöyle görünecek

'use strict';

import React from 'react';
import { connect } from 'react-redux';
import * as Action from './actions';

const mapStateToProps = ( state, ownProps ) => {
  return {
    current_user: state.user.current_user,
  }
}

const mapDispatchToProps = ( dispatch, ownProps ) => {
  return {
    updateCurrentUserData: (payload) => dispatch( Action.updateCurrentUserData(payload) ),
  }
}


let UserComponent = (props) => {

  let changeUserDetails = (field, value) => {
    props.updateCurrentUserData({ field: field, value: value });
  }

  return(
    <div>
      <h1>Hello { props.current_user.name }</h1>
      <p>Your email address is { props.current_user.email }</p>
      <div style={{ marginTop: 30 }}>
        <button onClick={ () => { changeUserDetails('name', 'Jame Smith') } }>Change Name</button>
        <button onClick={ () => { changeUserDetails('email', 'jane@gmail.com') } }>Change Email Address</button>
      </div>
    </div>
  )

}

const ConnectedUserComponent = connect(
  mapStateToProps,
  mapDispatchToProps
)(UserComponent)

export default ConnectedUserComponent;

Burada yaptıklarımız eklendi

  • mapStateToProps: Bu, verileri mağazadan almamızı sağlar ve bu veriler değiştiğinde, bileşenimiz yeni verilerle yeniden oluşturulacaktır.

Bileşenimiz, yalnızca bileşenimizin verileri mağazada değişiklik talep ediyorsa ve mağazada başka herhangi bir veri değiştiğinde değil, yeniden oluşturulacaktır.

  • mapDispatchToProps: Bu, bileşenimizden tüm redüktörlere ’eylemleri göndermemizi’ sağlar.. (herhangi bir bileşen olabilir), Ve eylemin ’türüne’ bağlı olarak, userReducer devreye girecek ve yeni bir güncellenmiş verilerle durum.

  • ConnectedUserComponent: Son olarak tüm parametreleri geçirerek ‘connect’ yöntemini kullanarak bileşenimizi mağazaya bağladık ve bağlı bileşeni ’export’ ettik.

  • Ayrıca changeUserDetails fonksiyonumuzu props üzerinde ‘method’u çağıracak ve verileri de aktaracak şekilde güncelledik.

NOT:

  • Redüktörden yeni bir durum döndürmezsek, ‘react-redux’ bileşenimizi yeniden oluşturmaz.

Kurulum veya Kurulum #

‘redux’u doğrudan ‘react’ ile kullanmak biraz zor görünebilir, Mağaza değiştiğinde güncellemek istediğiniz her ‘bileşen’ için, o bileşeni ‘redux mağazasına’ abone olmanız gerekir.

React Redux tüm bunlarla ilgilenir ve ihtiyaç duyduğu verileri `redux deposundan’ talep edebilen ve Sadece bu veriler değiştiğinde haberdar olabilen bileşenlerin yazılmasını gerçekten kolaylaştırır., Bu gerçekten etkili bileşenler yazmamızı sağlar.

‘react-redux’u kurmak için tek yapmanız gereken bu ’npm’ komutunu çalıştırmak.

npm install --save react-redux

Ve işin bitti.

Not: React Redux şunlara bağlıdır:

  • React (Sürüm 0.14 veya üzeri) ve
  • Redux

Tam örnek #

Bir “Müşteriler” aptal bileşenini Redux mağazasına bağlayan “CustomersContainer” kapsayıcımız olduğunu varsayalım.

index.js’de:

import { Component }, React from 'react';
import { render } from 'react-dom';
import { Provider } from 'react-redux';
import { createStore } from 'redux';
import rootReducer from './redux/rootReducer';
import CustomersContainer from './containers/CustomersContainer';

let store = createStore(rootReducer);

render(
  <Provider store={store}>
    <CustomersContainer />
  </Provider>,
  document.getElementById('root')
);

MüşterilerContainer’da:

import React, { Component } from 'react';
import { connect } from 'react-redux';

// Import action creators
import { fetchCustomers } from '../redux/actions';

// Import dumb component
import Customers from '../components/Customers';

// ES6 class declaration
class CustomersContainer extends Component {
  componentWillMount() {
    // Action fetchCustomers mapped to prop fetchCustomers
    this.props.fetchCustomers();
  }

  render() {
    return <Customers customers={this.props.customers} />;
  }
}

function mapStateToProps(state) {
  return {
    customers: state.customers
  };
}

// Here we use the shorthand notation for mapDispatchToProps
// it can be used when the props and action creators have the same name 
const CustomersContainer = connect(mapStateToProps, { fetchCustomers })(CustomersContainer);

export default CustomersContainer;
