Düğüm Ekspres Sunucu Örneği #

Bu örnekte, PayPal Düğüm SDK’sını kullanarak PayPal ile bir ödemenin nasıl işleneceğini görüntülemek için bir Ekspres sunucu entegrasyonu kuracağız. Kısalık olması açısından ödeme ayrıntıları için statik bir JSON yapısı kullanacağız.

PayPal ödemesini işlemek için işlevleri oluştururken izleyeceğimiz üç genel adım vardır:

  1. PayPal aracılığıyla işlemeyi planladığımız ödemeyi içeren bir JSON nesnesi oluşturuyoruz. Ardından, ödemeyi onaylamak için kullanıcıyı yönlendirecek bir bağlantı elde etmek için bunu PayPal’a göndeririz.
  2. Ardından, ödemeyi onaylamak için kullanıcıyı PayPal’a yönlendiririz. Onaylandıktan sonra PayPal, kullanıcıyı tekrar uygulamamıza yönlendirir.
  3. Uygulamaya geri döndüğümüzde, ödemeyi kullanıcı adına tamamlarız.

Bunu basit bir Düğüm uygulaması olarak parçalara ayırarak, NPM’den PayPal Düğüm SDK’sını alarak başlıyoruz:

npm install paypal-rest-sdk

Ardından, uygulama yapılandırmasını ve paketlerini kurduk.

var http = require('http'),
    paypal = require('paypal-rest-sdk'),
    bodyParser = require('body-parser'),
    app = require('express')();

var client_id = 'YOUR APPLICATION CLIENT ID';
var secret = 'YOUR APPLICATION SECRET';

//allow parsing of JSON bodies
app.use(bodyParser.json());

//configure for sandbox environment
paypal.configure({
    'mode': 'sandbox', //sandbox or live
    'client_id': client_id,
    'client_secret': secret
});

Bu uygulama için dört gereksinime ihtiyacımız var:

  1. Sunucumuz için HTTP paketi.
  2. PayPal Düğüm SDK paketi.
  3. JSON kodlu gövdelerle çalışmak için bodyParser paketi.
  4. Sunucumuz için Express çerçevesi.

Sonraki birkaç satır, bir uygulama oluştururken elde edilen istemci kimliği ve sır için değişkenleri ayarlar. Ardından, JSON kodlu gövdelere izin vermek için “bodyParser"ı kurduk, ardından uygulama ayrıntılarını kullanarak uygulamamızı yapılandırdık ve içinde çalıştığımız ortamı belirledik (üretim için canlı veya test için sanal alan).

Şimdi PayPal ile ödeme talebi oluşturmak için rota oluşturalım.

app.get('/create', function(req, res){
    //build PayPal payment request
    var payReq = JSON.stringify({
        'intent':'sale',
        'redirect_urls':{
            'return_url':'http://localhost:3000/process',
            'cancel_url':'http://localhost:3000/cancel'
        },
        'payer':{
            'payment_method':'paypal'
        },
        'transactions':[{
            'amount':{
                'total':'7.47',
                'currency':'USD'
            },
            'description':'This is the payment transaction description.'
        }]
    });

    paypal.payment.create(payReq, function(error, payment){
        if(error){
            console.error(error);
        } else {
            //capture HATEOAS links
            var links = {};
            payment.links.forEach(function(linkObj){
                links[linkObj.rel] = {
                    'href': linkObj.href,
                    'method': linkObj.method
                };
            })
        
            //if redirect url present, redirect user
            if (links.hasOwnProperty('approval_url')){
                res.redirect(links['approval_url'].href);
            } else {
                console.error('no redirect URI present');
            }
        }
    });
});

Yaptığımız ilk şey, ödemeyi oluşturmak için PayPal’a sağlamamız gereken bilgileri içeren ödeme talebi JSON nesnesini ayarlamaktır. “Niyet"i “satış” olarak belirledik, yönlendirme URL’lerini belirledik (PayPal’ın ödemeyi onayladıktan / iptal ettikten sonra kullanıcıyı yönlendirmesi gerekir), PayPal yapacağımızı belirtmek için “paypal” bir “ödeme_yöntemi” ekledik ödeme, ardından ödemeyi yapanın onaylaması için işlem bilgilerini belirtin.

Daha sonra ‘payReq’ nesnemizi ileterek ‘payment.create(…)‘i çağırırız. Bu, ödeme talebini PayPal’a gönderecektir. Bu geri döndüğünde ve başarılı olduğunda, kullanıcıyı yeniden yönlendirmemiz gereken, “onay_url” altında etiketlenen URL’yi çıkarmak için dönüş nesnesindeki sağlanan HATEOAS bağlantıları arasında dolaşabiliriz.

HATEOAS bağlantılarının biçimi, doğrudan kullanılırsa hassas referans koduna neden olabilir, bu nedenle, sağlanan tüm bağlantılar arasında geçiş yapar ve gelecekteki değişikliklere karşı kanıt sağlamak için bunları daha iyi bir referans nesnesine koyarız. Eğer o nesnede approval_url bulunursa, kullanıcıyı yeniden yönlendiririz.

Bu noktada kullanıcı, ödemeyi onaylamak için PayPal’a yönlendirilir. Bunu yaptıklarında, ‘createPayment(…)’ işlevinde belirttiğimiz ‘return_url’ye geri yönlendirilirler.

Şimdi, ödemeyi tamamlamak için bu iadeyi işlemek için bir yol sağlamamız gerekiyor.

app.get('/process', function(req, res){
    var paymentId = req.query.paymentId;
    var payerId = { 'payer_id': req.query.PayerID };

    paypal.payment.execute(paymentId, payerId, function(error, payment){
        if(error){
            console.error(error);
        } else {
            if (payment.state == 'approved'){ 
                res.send('payment completed successfully');
            } else {
                res.send('payment not successful');
            }
        }
    });
});

Kullanıcı uygulamanıza geri döndüğünde, “ödeme kimliği”, “PayerID” ve “belirteç” olmak üzere üç sorgu dizesi parametresi de gönderilir. Sadece ilk ikisiyle ilgilenmemiz gerekiyor.

Parametreleri çıkarıyoruz ve ödeme yürütme adımının ihtiyacı için PayerID’yi basit bir nesneye yerleştiriyoruz. Ardından, ödemeyi tamamlamak için bu iki parametreyi geçerek payment.execute(...) adresine bir çağrı yapılır.

Bu istek yapıldıktan sonra, “ödeme.durumu"nun “onaylandı” olarak ayarlanıp ayarlanmadığını kontrol ederek ödemenin başarıyla tamamlanıp tamamlanmadığını görürüz. Eğer öyleyse, ihtiyacımız olanı iade edilen ödeme nesnesinden saklayabiliriz.

Son adımımız sunucumuzu başlatıp belirlediğimiz rotalara gelen trafiği dinlemek.

//create server
http.createServer(app).listen(3000, function () {
   console.log('Server started: Listening on port 3000');
});

Sunucu başlatıldığında, http://localhost:3000/create adresine gitmek ödeme sürecini başlatır.
