Sözdizimi #

  • dbstop, ifade durumunda dosyadaki dosyada

Parametreler #

parametre Ayrıntılar
dosya “.m” dosyasının adı (uzantısız), ör. “uygun”. Bu parametre, ‘dbstop if error’ veya ‘dbstop if naninf’ gibi özel koşullu kesme noktası türleri ayarlanmadıkça (Gerekli)‘dir.
konum Kesme noktasının yerleştirileceği satır numarası. Belirtilen satır çalıştırılabilir kod içermiyorsa, kesme noktası belirtilenden sonra ilk geçerli satıra yerleştirilecektir.
ifade Bir boole değeri olarak değerlendirilen herhangi bir ifade veya bunların kombinasyonu. Örnekler: ind == 1, nargin < 4 && isdir('Q:\').

Kesme Noktalarıyla Çalışma #

Tanım #

Yazılım geliştirmede, bir kesme noktası, bir programda hata ayıklama amacıyla yerleştirilen kasıtlı bir durdurma veya duraklatma yeridir.

Daha genel olarak, bir kesme noktası, yürütülmesi sırasında bir program hakkında bilgi edinmenin bir yoludur. Kesinti sırasında programcı, programın beklendiği gibi çalışıp çalışmadığını öğrenmek için test ortamını (genel amaçlı kayıtlar, bellek, günlükler, dosyalar vb.) inceler. Pratikte, bir kesme noktası, bir programın yürütülmesinin ne zaman kesintiye uğraması gerektiğini belirleyen bir veya daha fazla koşuldan oluşur.

-Vikipedi

MATLAB’de kesme noktaları #

Motivasyon #

MATLAB’de, yürütme bir kesme noktasında durakladığında, mevcut çalışma alanında (a.k.a. scope) veya çağıran çalışma alanlarından herhangi birinde var olan değişkenler incelenebilir (ve ayrıca genellikle değiştirilebilir).

Kesme Noktası Türleri #

MATLAB, kullanıcıların .m dosyalarına iki tür kesme noktası yerleştirmesine izin verir:

  • Standart (veya “unrestricted”) kesme noktaları (kırmızı ile gösterilmiştir) - işaretli satıra ulaşıldığında yürütmeyi duraklatın.
  • “Koşullu” kesme noktaları (sarı renkle gösterilmiştir) - işaretli satıra ulaşıldığında yürütmeyi duraklatın VE kesme noktasında tanımlanan koşul “doğru” olarak değerlendirilir.

Kesme Noktaları Yerleştirme #

Her iki kesme noktası türü de çeşitli şekillerde oluşturulabilir:

  • MATLAB Editor GUI’sini kullanarak, satır numarasının yanındaki yatay çizgiye sağ tıklayarak.

  • ‘dbstop’ komutunu kullanarak:

    % Create an unrestricted breakpoint:
dbstop in file at location
% Create a conditional breakpoint:
dbstop in file at location if expression

% Examples and special cases: 
dbstop in fit at 99 % Standard unrestricted breakpoint.

dbstop in fit at 99 if nargin==3 % Standard conditional breakpoint.

dbstop if error % This special type of breakpoint is not limited to a specific file, and
                % will trigger *whenever* an error is encountered in "debuggable" code.

dbstop in file % This will create an unrestricted breakpoint on the first executable line
               % of "file".

dbstop if naninf % This special breakpoint will trigger whenever a computation result 
                 % contains either a NaN (indicates a division by 0) or an Inf

  • Klavye kısayollarını kullanma: Windows’ta standart bir kesme noktası oluşturmak için varsayılan anahtar F12‘dir; koşullu kesme noktaları için varsayılan anahtar unset’tir.

Kesme Noktalarını Devre Dışı Bırakma ve Yeniden Etkinleştirme #

Geçici olarak yoksaymak için bir kesme noktasını devre dışı bırakın: devre dışı bırakılan kesme noktaları, yürütmeyi duraklatmaz. Bir kesme noktasını devre dışı bırakmak birkaç yolla yapılabilir:

  • Kırmızı/sarı kesme noktası dairesine sağ tıklayın > Kesme Noktasını Devre Dışı Bırak.
  • Koşullu (sarı) bir kesme noktasına sol tıklayın.
  • Düzenleyici sekmesi > Kesme Noktaları > Etkinleştir\Devre Dışı Bırak’ta.

Kesme Noktalarını Kaldırma #

Tüm kesme noktaları, manuel veya otomatik olarak kaldırılana kadar bir dosyada kalır. MATLAB oturumu sonlandırılırken (yani programı sonlandırırken) kesme noktaları _otomatik olarak temizlenir. Kesme noktalarının manuel olarak temizlenmesi aşağıdaki yollardan biriyle yapılır:

  • dbclear komutunu kullanarak:

    dbclear all
dbclear in file   
dbclear in file at location    
dbclear if condition

  • Standart bir kesme noktası simgesine veya devre dışı bırakılmış bir koşullu kesme noktası simgesine sol tıklama.

  • Herhangi bir kesme noktasına sağ tıklayarak > Kesme Noktasını Temizle.

  • Düzenleyici sekmesi > Kesme Noktaları > Tümünü Temizle’de.

  • MATLAB’ın R2015b öncesi sürümlerinde clear komutunu kullanarak.

Yürütmeye Devam Etme #

Yürütme bir kesme noktasında duraklatıldığında, programı yürütmeye devam etmenin iki yolu vardır:

  • Geçerli satırı yürütün ve bir sonraki satırdan önce tekrar duraklatın.

Editör’de F101, Komut Penceresinde `dbstep’, Şerit > Editör > DEBUG’da “Adım”.

  • Bir sonraki kesme noktasına kadar yürütün (eğer daha fazla kesme noktası yoksa, yürütme programın sonuna kadar devam eder).

    F121 in the Editor, dbcont in the Command Window, “Continue” in Ribbon > Editor > DEBUG.

1 - Windows’ta varsayılan.

MATLAB tarafından çağrılan Java kodunda hata ayıklama #

Genel Bakış #

MATLAB yürütmesi sırasında çağrılan Java sınıflarının hatalarını ayıklamak için iki adımın gerçekleştirilmesi gerekir:

  1. MATLAB’ı JVM hata ayıklama modunda çalıştırın.
  2. MATLAB işlemine bir Java hata ayıklayıcı ekleyin.

MATLAB, JVM hata ayıklama modunda başlatıldığında, komut penceresinde aşağıdaki mesaj görünür:

JVM is being started with debugging enabled.
Use "jdb -connect com.sun.jdi.SocketAttach:port=4444" to attach debugger.

MATLAB sonu #

Pencereler: #

MATLAB yürütülebilir dosyasına (matlab.exe) bir kısayol oluşturun ve aşağıda gösterildiği gibi sonuna -jdb bayrağını ekleyin:

MATLAB bu kısayolu kullanarak çalıştırıldığında, JVM hata ayıklama etkinleştirilecektir.

Alternatif olarak java.opts dosyası oluşturulabilir/güncellenebilir. Bu dosya, “matlab-root\bin\arch” içinde depolanır, burada “matlab-root”, MATLAB kurulum dizinidir ve “arch”, mimaridir (örneğin, “win32”).

Dosyaya aşağıdakiler eklenmelidir:

-Xdebug
-Xrunjdwp:transport=dt_socket,address=1044,server=y,suspend=n
