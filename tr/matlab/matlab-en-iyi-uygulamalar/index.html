Bu konu, topluluğun zaman içinde öğrendiği en iyi uygulamaları gösterir.

Kodu düzgün girin #

Doğru girinti, yalnızca estetik bir görünüm kazandırmakla kalmaz, aynı zamanda kodun okunabilirliğini de artırır.

Örneğin, aşağıdaki kodu göz önünde bulundurun:

%no need to understand the code, just give it a look
n = 2;
bf = false;
while n>1
for ii = 1:n
for jj = 1:n
if ii+jj>30
bf = true;
break
end
end
if bf
break 
end
end
if bf
break 
end
n = n + 1;
end

Gördüğünüz gibi, hangi döngünün ve ‘if’ ifadelerinin nerede bittiğini görmek için dikkatli bir şekilde bakmanız gerekiyor.
Akıllı girinti ile şu görünümü elde edeceksiniz:

n = 2;
bf = false;
while n>1
    for ii = 1:n
        for jj = 1:n
            if ii+jj>30
                bf = true;
                break
            end
        end
        if bf
            break
        end
    end
    if bf
        break
    end
    n = n + 1;
end

Bu, döngülerin/‘if’ ifadesinin başlangıcını ve bitişini açıkça gösterir.

Akıllı girintiyi şu şekilde yapabilirsiniz:
• tüm kodunuzu seçin (Ctrl+A)
• ve ardından Ctrl+I tuşlarına basın veya düzenleme çubuğundan öğesini tıklayın.

Satırları kısa tutun #

Uzun ifadeye devam etmek için devam karakterini (üç nokta) ... kullanın.

Örnek:

MyFunc( parameter1,parameter2,parameter3,parameter4, parameter5, parameter6,parameter7, parameter8, parameter9)

ile değiştirilebilir:

MyFunc( parameter1, ...
        parameter2, ...
        parameter3, ...
        parameter4, ...
        parameter5, ...
        parameter6, ...
        parameter7, ...
        parameter8, ...
        parameter9)

onaylamayı kullan #

Matlab, bazı çok önemsiz hataların sessizce geçmesine izin verir, bu da çalıştırmada çok daha sonra bir hatanın ortaya çıkmasına neden olabilir - hata ayıklamayı zorlaştırır. Değişkenlerinizle ilgili bir varsayım varsa, onu doğrulayın.

function out1 = get_cell_value_at_index(scalar1,cell2)
assert(isscalar(scalar1),'1st input must be a scalar')
assert(iscell(cell2),'2nd input must be a cell array')

assert(numel(cell2) >= scalar1),'2nd input must have more elements than the value of the 1st input')
assert(~isempty(cell2{scalar1}),'2nd input at location is empty')

out1 = cell2{scalar1};

Döngülerden kaçının #

Çoğu zaman, döngüler Matlab ile hesaplama açısından pahalıdır. Vektörleştirme kullanıyorsanız kodunuz daha hızlı olacaktır. Ayrıca genellikle kodunuzu daha modüler, kolayca değiştirilebilir ve hata ayıklamayı daha kolay hale getirir. En büyük dezavantajı, veri yapılarını planlamak için zaman ayırmanız gerektiğidir ve boyut hatalarının gelmesi daha kolaydır.

Örnekler #

yazma

for t=0:0.1:2*pi
    R(end+1)=cos(t);
end

ancak

t=0:0.1:2*pi;
R=cos(t)

yazma

for i=1:n
    for j=1:m
        c(i,j)=a(i)+2*b(j);
    end
end

Ama buna benzer bir şey

c=repmat(a.',1,m)+2*repmat(b,n,1)

Daha fazla ayrıntı için bkz. vektörleştirme

Blok Yorum Operatörü #

Kodu açıklayan yorumlar eklemek iyi bir uygulamadır. Başkaları ve daha sonra iade edildiğinde kodlayıcı için bile yararlıdır. Tek bir satır, “%” sembolü veya “Ctrl+R” kısayolu kullanılarak yorumlanabilir. Önceden yorumlanmış bir satırın yorumunu kaldırmak için % sembolünü kaldırın veya Crtl+T kısayolunu kullanın.

Bir kod bloğunu yorumlamak, her satırın başına bir % sembolü eklenerek yapılabilirken, MATLAB’ın daha yeni sürümleri (2015a’dan sonra) Blok Yorum Operatörü %{ code %} kullanmanıza izin verir. Bu operatör kodun okunabilirliğini arttırır. Hem kod yorumu hem de işlev yardım belgeleri için kullanılabilir. Kodun okunabilirliğini artırmak için Blok katlanabilir ve açılabilir.

Görüldüğü gibi %{ ve %} operatörleri satırlarda tek başına görünmelidir. Bu satırlara başka metin eklemeyin.

function y = myFunction(x)
%{
myFunction  Binary Singleton Expansion Function
y = myFunction(x) applies the element-by-element binary operation
specified by the function handle FUNC to arrays A and B, with implicit
expansion enabled.
%}

%%  Compute z(x, y) = x.*sin(y) on a grid:
% x = 1:10;
y = x.';

%{
z = zeros(numel(x),numel(y));
for ii=1:numel(x)
    for jj=1:numel(y)
        z(ii,jj) = x(ii)*sin(y(jj));
    end
end
%}

z = bsxfun(@(x, y) x.*sin(y), x, y);
y = y + z;

end

Geçici Dosya için Benzersiz Ad Oluştur #

Bir script veya fonksiyon kodlanırken, bir veya birden fazla örneğin, bazı verileri depolamak için geçici dosyaya ihtiyaç vardır.

Mevcut bir dosyanın üzerine yazılmasını önlemek veya bir MATLAB fonksiyonunu gölgelemek için tempname işlevi sistem geçici klasöründeki geçici bir dosya için benzersiz bir ad oluşturmak için kullanılabilir.

my_temp_file=tempname

Dosya adı, uzantı olmadan oluşturulur; birleştirerek eklenebilir ’tempname’ tarafından oluşturulan ismin istenen uzantısı

my_temp_file_with_ext=[tempname '.txt']

Sistem geçici klasörünün konumu, tempdir işlevi.

İşlev / komut dosyasının yürütülmesi sırasında, geçici dosya artık gerektiğinde sil işlevi kullanılarak silinebilir.

“Sil” onay istemeyeceğinden, “açık” olarak ayarlamak faydalı olabilir. silinecek dosyayı recycle klasörüne taşıma seçeneği.

Bu, geri dönüşüm işlevi kullanılarak yapılabilir. yol:

recycle('on')

Aşağıdaki örnekte, ’tempname’, ‘delete’ ve ‘delete’ işlevlerinin olası bir kullanımı ‘geri dönüşüm’ önerildi.

%
% Create some example data
%
theta=0:.1:2*pi;
x=cos(theta);
y=sin(theta);
%
% Generate the temporary filename
%
my_temp_file=[tempname '.mat'];
%
% Split the filename (path, name, extension) and display them in a message box
[tmp_file_path,tmp_file_name, tmp_file_ext]=fileparts(my_temp_file)
uiwait(msgbox(sprintf('Path= %s\nName= %s\nExt= %s', ...
              tmp_file_path,tmp_file_name,tmp_file_ext),'TEMPORARY FILE'))
%
% Save the varaibles in a temporary file
%
save(my_temp_file,'x','y','theta')
%
% Load the varaibles from the temporary file
%
load(my_temp_file)
%
% Set the reclycle option on
%
recycle('on')
%
% Delete the temporary file
%
delete(my_temp_file)

Uyarı

Geçici dosya adı, “java.util.UUID.randomUUID” yöntemi kullanılarak oluşturulur. (randomUUID).

MATLAB, JVM olmadan çalıştırılırsa, geçici dosya adı kullanılarak oluşturulur. CPU sayacına ve zamana dayalı “matlab.internal.timing.timing”. Bu durumda geçici dosya adının benzersiz olması garanti edilmez.

validateattributes kullanın #

validateattributes işlevi bir diziyi bir dizi spesifikasyona göre doğrulamak için kullanılabilir

Bu nedenle, bir işleve sağlanan girişi doğrulamak için kullanılabilir.

Aşağıdaki örnekte, “test_validateattributes” işlevi üç giriş

function test_validateattributes(input_1,input_2,input_3)

Giriş özellikleri şunlardır:

  • dizi_1:

  • sınıf: çift

  • boyut: [3,2]

  • değerler: öğeler NaN olmamalıdır

  • char_array:

  • sınıf: karakter

  • değer: dize boş olmamalıdır

  • dizi_3

  • sınıf: çift

  • boyut: [5 1]

  • değerler: öğeler gerçek olmalıdır

Üç girişi doğrulamak için “validateattributes” işlevi çağrılabilir. aşağıdaki sözdizimi ile:

validateattributes(A,classes,attributes,funcName,varName,argIndex)

nerede:

  • “A” doğrulanacak dizidir
  • “sınıflar”: dizinin “türü"dür (ör. “tek”, “çift”, “mantıksal”)
  • “nitelikler”: giriş dizisinin eşleşmesi gereken niteliklerdir (ör. “[3,2], boyut” dizinin boyutunu belirtmek için, “nonnan” dizinin NaN değerlerine sahip olmayacağını belirtmek için)
  • “funcName”: doğrulamanın gerçekleştiği işlevin adıdır. Bu argüman (varsa) hata mesajının oluşturulmasında kullanılır.
  • “varName”: doğrulama altındaki dizinin adıdır. Bu argüman (varsa) hata mesajının oluşturulmasında kullanılır.
  • ‘argIndex’: giriş listesindeki giriş dizisinin konumudur. Bu argüman (varsa) hata mesajının oluşturulmasında kullanılır.

Bir veya birden fazla girişin spesifikasyona uymaması durumunda, bir hata mesaj oluşturulur.

Birden fazla geçersiz giriş olması durumunda, ilk uyumsuzluk bulunduğunda doğrulama durur.

Bu, giriş doğrulamasının yapıldığı “işlev test_validateattributes” işlevidir. uygulandı.

Fonksiyon üç giriş gerektirdiğinden, ilk giriş sayısı kontrol edilir. sağlanan işlev nargin işlevi kullanılarak gerçekleştirilir.

function test_validateattributes(array_1,char_array_1,array_3)
%
% Check for the number of expected input: if the number of input is less
% than the require, the function exits with an error message
%
if(nargin ~= 3)
   error('Error: TEST_VALIDATEATTRIBUTES requires 3 input, found %d',nargin)
end
%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
% Definition of the expected characteristics of the first input %
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%
% INPUT #1 name (only used in the generation of the error message)
%
input_1_name='array_1';
%
% INPUT #1 position (only used in the generation of the error message)
%
input_1_position=1;
%
% Expected CLASS of the first input MUST BE "double"
%
input_1_class={'double'};
%
% Expected ATTRIBUTES of the first input
%   SIZE: MUST BE [3,2]
%
input_1_size_attribute='size';
input_1_size=[3,2];
%
%   VALUE CHECK: the element MUST BE NOT NaN
%
input_1_value_type='nonnan';
%
% Build the INPUT 1 attributes
%
input_1_attributes={input_1_size_attribute,input_1_size, ...
                    input_1_value_type};
%
% CHECK THE VALIDITY OF THE FIRST INPUT
%
validateattributes(array_1, ...
                   input_1_class,input_1_attributes,'', ...
                   input_1_name,input_1_position);

%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
% Definition of the expected characteristics of the second input %
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%
% INPUT #1 name (only used in the generation of the error message)
%
input_2_name='char_array_1';
%
% INPUT #2 position (only used in the generation of the error message)
%
input_2_position=2;
%
% Expected CLASS of the first input MUST BE "string"
%
input_2_class={'char'};
%
%   VALUE CHECK: the element must be not NaN
%
input_2_size_attribute='nonempty';
%
% Build the INPUT 2 attributes
%
input_2_attributes={input_2_size_attribute};
%
% CHECK THE VALIDITY OF THE SECOND INPUT
%
validateattributes(char_array_1, ...
                   input_2_class,input_2_attributes,'', ...
                   input_2_name,input_2_position);
%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
% Definition of the expected characteristics of the third input %
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%
% INPUT #3 name (only used in the generation of the error message)
%
input_3_name='array_3';
%
% INPUT #3 position (only used in the generation of the error message)
%
input_3_position=3;
%
% Expected CLASS of the first input MUST BE "double"
%
input_3_class={'double'};
%
% Expected ATTRIBUTES of the first input
%   SIZE: must be [5]
input_3_size_attribute='size';
input_3_size=[5 1];
%   VALUE CHECK: the elements must be real
input_3_value_type='real';
%
% Build the INPUT 3 attributes
%
input_3_attributes={input_3_size_attribute,input_3_size, ...
                    input_3_value_type};
%
% CHECK THE VALIDITY OF THE FIRST INPUT
%
validateattributes(array_3, ...
                   input_3_class,input_3_attributes,'', ...
                   input_3_name,input_3_position);

disp('All the three input are OK')

Doğrulama prosedürünün uygulanmasını test etmek için aşağıdaki komut dosyası kullanılabilir.

Gerekli üç girdiyi üretir ve rastgele onları geçersiz kılar.

%
% Generate the first input
%
n_rows=randi([2 3],1);
n_cols=2;
input_1=randi([20 30],n_rows,n_cols);
%
% Generate the second input
%
if(rand > 0.5)
   input_2='This is a string';
else
   input_2='';
end
%
% Generate the third input
%
input_3=acos(rand(5,1)*1.2);
%
% Call the test_validateattributes function with the above generated input
%
input_1
input_2
input_3
%
test_validateattributes(input_1,input_2,input_3)
