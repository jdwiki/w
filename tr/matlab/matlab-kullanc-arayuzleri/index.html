Kullanıcı Arayüzü Çevresinde Veri Aktarma #

Çoğu gelişmiş kullanıcı arabirimi, kullanıcının bir kullanıcı arabirimini oluşturan çeşitli işlevler arasında bilgi aktarabilmesini gerektirir. MATLAB bunu yapmak için bir dizi farklı yönteme sahiptir.

[yönlendirilmiş] 1 #

MATLAB’ın kendi GUI Geliştirme Ortamı (GUIDE), geri aramalar arasında veri iletmek için “handles” adlı bir “yapı” kullanmayı tercih eder. Bu “yapı”, çeşitli UI bileşenlerinin tüm grafik tutamaçlarını ve ayrıca kullanıcı tarafından belirlenen verileri içerir. Otomatik olarak “tutamaçları” geçen GUIDE tarafından oluşturulmuş bir geri arama kullanmıyorsanız, geçerli değeri guidata kullanarak alabilirsiniz.

% hObject is a graphics handle to any UI component in your GUI
handles = guidata(hObject);

Bu veri yapısında saklanan bir değeri değiştirmek istiyorsanız, değiştirebilirsiniz ancak daha sonra değişikliklerin diğer geri aramalar tarafından görülebilmesi için onu ‘hObject’ içinde saklamanız gerekir. guidata için ikinci bir giriş argümanı belirleyerek bunu saklayabilirsiniz.

% Update the value
handles.myValue = 2;

% Save changes
guidata(hObject, handles)

Aynı “şekil”* içinde bir UI bileşeni olduğu sürece “hObject"in değeri önemli değildir, çünkü sonuçta veriler “hObject” içeren şekil içinde depolanır.

İçin en iyisi:

  • Tüm öğeleri saklayabileceğiniz ’tutamak’ yapısının saklanması GUI bileşenlerinizin tutamaçları.
  • Çoğu geri arama tarafından erişilmesi gereken “küçük” diğer değişkenleri saklamak.

** Aşağıdakiler için önerilmez:

  • Herkes tarafından erişilmesi gerekmeyen büyük değişkenlerin saklanması geri aramalar ve alt işlevler (için setappdata/getappdata kullanın bunlar).

setappdata/getappdata #

“guidata” yaklaşımına benzer şekilde, bir grafik tutamacı içindeki değerleri depolamak ve almak için setappdata ve getappdata kullanabilirsiniz. Bu yöntemleri kullanmanın avantajı, saklanan tüm verileri içeren bir “yapı"nın tamamı yerine yalnızca istediğiniz değeri alabilmenizdir. Anahtar/değer deposuna benzer.

Verileri bir grafik nesnesi içinde depolamak için

% Create some data you would like to store
myvalue = 2

% Store it using the key 'mykey'
setappdata(hObject, 'mykey', myvalue)

Ve aynı değeri farklı bir geri arama içinden almak için

value = getappdata(hObject, 'mykey');

Not: “getappdata” çağrılmadan önce hiçbir değer saklanmadıysa, boş bir dizi (”[]”) döndürür.

“guidata"ya benzer şekilde, veriler “hObject” içeren şekilde saklanır.

İçin en iyisi:

  • Herkes tarafından erişilmesi gerekmeyen büyük değişkenlerin saklanması geri aramalar ve alt işlevler.

Kullanıcı Verileri #

Her grafik tutamacının, istediğiniz herhangi bir veriyi içerebilen ‘UserData’ adlı özel bir özelliği vardır. Bir hücre dizisi, bir “yapı” veya hatta bir skaler içerebilir. Bu özellikten faydalanabilir ve belirli bir grafik tutamacı ile ilişkilendirilmesini istediğiniz herhangi bir veriyi bu alanda saklayabilirsiniz. R2014b veya daha yenisini kullanıyorsanız, grafik nesneleri veya nokta gösterimi için standart get/set yöntemlerini kullanarak değeri kaydedebilir ve alabilirsiniz.

% Create some data to store
mydata = {1, 2, 3};

% Store it within the UserData property
set(hObject, 'UserData', mydata)

% Of if you're using R2014b or newer:
% hObject.UserData = mydata;

Ardından başka bir geri arama içinden bu verileri alabilirsiniz:

their_data = get(hObject, 'UserData');

% Or if you're using R2014b or newer:
% their_data = hObject.UserData;

İçin en iyisi:

  • Sınırlı bir kapsamda değişkenlerin saklanması (yalnızca saklandıkları nesne tarafından kullanılması muhtemel değişkenler veya onunla doğrudan ilişkisi olan nesneler).

İç İçe İşlevler #

MATLAB’de iç içe bir işlev, üst işlevde tanımlanan herhangi bir değişkeni okuyabilir ve değiştirebilir. Bu şekilde, bir geri aramayı yuvalanmış bir işlev olarak belirtirseniz, ana işlevde depolanan tüm verileri alabilir ve değiştirebilir.

function mygui()    
    hButton = uicontrol('String', 'Click Me', 'Callback', @callback);

    % Create a counter to keep track of the number of times the button is clicked
    nClicks = 0;

    % Callback function is nested and can therefore read and modify nClicks
    function callback(source, event)
        % Increment the number of clicks
        nClicks = nClicks + 1;

        % Print the number of clicks so far
        fprintf('Number of clicks: %d\n', nClicks);
    end
end

İçin en iyisi:

  • Küçük, basit GUI’ler. (hızlı prototipleme için, “guidata” ve/veya “set/getappdata” yöntemlerini uygulamak zorunda kalmamak için).

** Aşağıdakiler için önerilmez:

  • Orta, büyük veya karmaşık GUI’ler.

  • GUI, GUIDE ile oluşturuldu.

Açık girdi argümanları #

Bir geri arama işlevine veri göndermeniz gerekiyorsa ve geri arama içindeki verileri değiştirmeniz gerekmiyorsa, özenle hazırlanmış bir geri arama tanımı kullanarak verileri her zaman geri aramaya aktarmayı düşünebilirsiniz.

Girdi ekleyen anonim bir işlev kullanabilirsiniz.

% Create some data to send to mycallback
data = [1, 2, 3];

% Pass data as a third input to mycallback
set(hObject, 'Callback', @(source, event)mycallback(source, event, data))

Veya bir geri arama belirtmek için hücre dizisi sözdizimini kullanabilir ve yine ek girdiler belirtebilirsiniz.

set(hObject, 'Callback', {@mycallback, data})

İçin en iyisi:

  • Geri arama, bazı işlemleri gerçekleştirmek için “veri"ye ihtiyaç duyduğunda, ancak “veri” değişkeninin değiştirilip yeni bir durumda kaydedilmesi gerekmediğinde.

Arayüzünüzde geri arama yürütmesini duraklatan bir düğme oluşturma #

Bazen uygulamanın durumunu incelemek için kod yürütmeyi duraklatmak isteriz (bkz. Hata Ayıklama). MATLAB düzenleyicisi aracılığıyla kod çalıştırırken, bu, kullanıcı arayüzündeki “Duraklat” düğmesi kullanılarak veya Ctrl+c (Windows’ta) tuşlarına basılarak yapılabilir. Ancak, bir GUI’den bir hesaplama başlatıldığında (bazı ‘uicontrol’lerin geri çağrısı yoluyla), bu yöntem artık çalışmaz ve geri çağrının başka bir geri çağrı yoluyla _kesilmesi gerekir. Aşağıda bu ilkenin bir gösterimi bulunmaktadır:

function interruptibleUI
dbclear in interruptibleUI % reset breakpoints in this file
figure('Position',[400,500,329,160]); 

uicontrol('Style', 'pushbutton',...
          'String', 'Compute',...
          'Position', [24 55 131 63],...
          'Callback', @longComputation,...
          'Interruptible','on'); % 'on' by default anyway
      
uicontrol('Style', 'pushbutton',...
          'String', 'Pause #1',...
          'Position', [180 87 131 63],...
          'Callback', @interrupt1);       

uicontrol('Style', 'pushbutton',...
          'String', 'Pause #2',...
          'Position', [180 12 131 63],...
          'Callback', @interrupt2);    

end

function longComputation(src,event)
  superSecretVar = rand(1);
  pause(15);
  print('done!'); % we'll use this to determine if code kept running "in the background".
end

function interrupt1(src,event) % depending on where you want to stop
  dbstop in interruptibleUI at 27 % will stop after print('done!');
  dbstop in interruptibleUI at 32 % will stop after **this** line.
end

function interrupt2(src,event) % method 2
  keyboard;
  dbup; % this will need to be executed manually once the code stops on the previous line.
end

Bu örneği anladığınızdan emin olmak için aşağıdakileri yapın:

  1. Yukarıdaki kodu adı verilen yeni bir dosyaya yapıştırın ve kod dosyanın ilk satırından başlayacak şekilde ‘interruptibleUI.m’ olarak kaydedin (bu, 1. yöntemin çalışması için önemlidir).
  2. Komut dosyasını çalıştırın.
  3. Hesapla‘ı tıklayın ve kısa bir süre sonra 1. Duraklat‘ı veya 2. Duraklat‘ı tıklayın.
  4. superSecretVar değerini bulabildiğinizden emin olun.

“Tutamak” yapısını kullanarak veri iletme #

Bu, GUI’nin “tutamaçları” yapısında depolanan bir değeri değiştiren iki düğmeli temel bir GUI örneğidir.

function gui_passing_data()
    % A basic GUI with two buttons to show a simple use of the 'handles'
    % structure in GUI building

    %  Create a new figure.
    f = figure();

    % Retrieve the handles structure
    handles = guidata(f);

    % Store the figure handle
    handles.figure = f;

    % Create an edit box and two buttons (plus and minus), 
    % and store their handles for future use
    handles.hedit  = uicontrol('Style','edit','Position',[10,200,60,20] , 'Enable', 'Inactive');

    handles.hbutton_plus  = uicontrol('Style','pushbutton','String','+',...
               'Position',[80,200,60,20] , 'Callback' , @ButtonPress);

    handles.hbutton_minus  = uicontrol('Style','pushbutton','String','-',...
               'Position',[150,200,60,20] , 'Callback' , @ButtonPress);

    % Define an initial value, store it in the handles structure and show
    % it in the Edit box
    handles.value = 1;
    set(handles.hedit , 'String' , num2str(handles.value))

    % Store handles
    guidata(f, handles);



function  ButtonPress(hObject, eventdata)
    % A button was pressed
    % Retrieve the handles
    handles = guidata(hObject);

    % Determine which button was pressed; hObject is the calling object
    switch(get(hObject , 'String'))
        case '+'
            % Add 1 to the value
            handles.value = handles.value + 1;
            set(handles.hedit , 'String', num2str(handles.value))
        case '-'
            % Substract 1 from the value
            handles.value = handles.value - 1;
    end

    % Display the new value
    set(handles.hedit , 'String', num2str(handles.value))

    % Store handles
    guidata(hObject, handles);

Örneği test etmek için, onu gui_passing_data.m adlı bir dosyaya kaydedin ve F5 ile başlatın. Lütfen bu kadar basit bir durumda, düzenleme kutusunun String özelliğinden doğrudan erişebildiğiniz için değeri tutamaç yapısında saklamanıza bile gerek olmayacağını unutmayın.

Kullanıcı Arayüzü Çevresinde Veri Aktarırken Performans Sorunları #

İki ana teknik, GUI işlevleri ve Geri Aramalar arasında veri aktarımına izin verir: setappdata/getappdata ve guidata (bunun hakkında daha fazla bilgi edinin). İlki, daha fazla zaman verimli olduğu için daha büyük değişkenler için kullanılmalıdır. Aşağıdaki örnek, iki yöntemin verimliliğini test eder.

Basit bir tuşa sahip bir GUI oluşturulur ve büyük bir değişken (10000x10000 double) hem guidata hem de setappdata ile saklanır. Düğme, yürütmelerini zamanlarken iki yöntemi kullanarak değişkeni yeniden yükler ve geri depolar. Setappdata kullanılarak çalışma süresi ve yüzde iyileştirme, komut penceresinde görüntülenir.

function gui_passing_data_performance()
    % A basic GUI with a button to show performance difference between
    % guidata and setappdata

    %  Create a new figure.
    f = figure('Units' , 'normalized');

    % Retrieve the handles structure
    handles = guidata(f);

    % Store the figure handle
    handles.figure = f;

    handles.hbutton = uicontrol('Style','pushbutton','String','Calculate','units','normalized',...
               'Position',[0.4 , 0.45 , 0.2 , 0.1] , 'Callback' , @ButtonPress);

    % Create an uninteresting large array
    data = zeros(10000);

    % Store it in appdata
    setappdata(handles.figure , 'data' , data);

    % Store it in handles
    handles.data = data;

    % Save handles
    guidata(f, handles);



function  ButtonPress(hObject, eventdata)

    % Calculate the time difference when using guidata and appdata
    t_handles = timeit(@use_handles);
    t_appdata = timeit(@use_appdata);

    % Absolute and percentage difference
    t_diff = t_handles - t_appdata;
    t_perc = round(t_diff / t_handles * 100);

    disp(['Difference: ' num2str(t_diff) ' ms / ' num2str(t_perc) ' %'])




function  use_appdata()  

    % Retrieve the data from appdata
    data = getappdata(gcf , 'data');

    % Do something with data %

    % Store the value again
    setappdata(gcf , 'data' , data);


function use_handles()

    % Retrieve the data from handles
    handles = guidata(gcf);
    data = handles.data;

    % Do something with data %

    % Store it back in the handles
    handles.data = data;
    guidata(gcf, handles);

Xeon W3530@2.80 GHz cihazımda ‘Fark: 0.00018957 ms / %73’ alıyorum, böylece getappdata/setappdata kullanarak %73 performans artışı elde ediyorum! 10x10 çift değişken kullanılırsa sonucun değişmediğini, ancak “tutamaçlar” büyük veri içeren birçok alan içeriyorsa sonucun değişeceğini unutmayın.
