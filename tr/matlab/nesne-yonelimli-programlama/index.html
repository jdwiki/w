Value vs Handle sınıfları #

MATLAB’daki sınıflar iki ana kategoriye ayrılır: değer sınıfları ve tutamaç sınıfları. En büyük fark, bir değer sınıfının bir örneğini kopyalarken, temel alınan verilerin yeni örneğe kopyalanmasıdır, ancak tutamaç sınıfları için yeni örneğin orijinal verilere işaret etmesi ve yeni örnekteki değerlerin değiştirilmesi onları orijinalde değiştirmesidir. Bir sınıf, “handle” sınıfından miras alınarak bir tanıtıcı olarak tanımlanabilir.

classdef valueClass
    properties
        data
    end
end

ve

classdef handleClass < handle
    properties
        data
    end
end

sonra

>> v1 = valueClass;
>> v1.data = 5;
>> v2 = v1;
>> v2.data = 7;
>> v1.data
ans =
     5

>> h1 = handleClass;
>> h1.data = 5;
>> h2 = h1;
>> h2.data = 7;
>> h1.data
ans =
     7

Bir sınıf tanımlama #

Bir sınıf, sınıfla aynı ada sahip bir “.m” dosyasında “classdef” kullanılarak tanımlanabilir. Dosya, sınıf yöntemlerinde kullanım için ‘classdef’…’end’ bloğunu ve yerel işlevleri içerebilir.

En genel MATLAB sınıf tanımı aşağıdaki yapıya sahiptir:

classdef (ClassAttribute = expression, ...) ClassName < ParentClass1 & ParentClass2 & ...

   properties (PropertyAttributes) 
      PropertyName
   end 

   methods (MethodAttributes) 
      function obj = methodName(obj,arg2,...)
         ...
      end
   end

   events (EventAttributes) 
      EventName
   end

   enumeration
      EnumName
   end

end

MATLAB Belgeleri: Sınıf öznitelikleri, [Özellik öznitelikleri](http://www.mathworks.com/help/matlab/ matlab_oop/property-attributes.html), Yöntem öznitelikleri, [Olay öznitelikleri](http://www.mathworks. com/help/matlab/matlab_oop/event-attributes.html), Sayılandırma sınıfı kısıtlamaları.

Örnek sınıf: #

‘Car’ adlı bir sınıf, ‘Car.m’ dosyasında şu şekilde tanımlanabilir:

classdef Car < handle % handle class so properties persist
    properties
        make
        model
        mileage = 0;
    end

    methods
        function obj = Car(make, model)
            obj.make = make;
            obj.model = model;
        end
        function drive(obj, milesDriven)
            obj.mileage = obj.mileage + milesDriven;
        end
    end
end

Yapıcının, sınıfla aynı ada sahip bir yöntem olduğuna dikkat edin. <Yapıcı, nesne yönelimli programlamada o türde bir nesneyi başlatan bir sınıf veya yapının özel bir yöntemidir. Yapıcı, genellikle sınıfla aynı ada sahip olan ve bir nesnenin üyelerinin değerlerini varsayılan veya kullanıcı tanımlı değerlere ayarlamak için kullanılabilen bir örnek yöntemidir.>

Bu sınıfın bir örneği, yapıcı çağrılarak oluşturulabilir;

>> myCar = Car('Ford', 'Mustang'); //creating an instance of car class

“Sürücü” yöntemini çağırmak, kilometreyi artıracaktır

>> myCar.mileage 
    
    ans = 
            0

>> myCar.drive(450);

>> myCar.mileage
    
   ans = 
            450

Yapıcılar #

Bir yapıcı, bir nesnenin örneği oluşturulduğunda çağrılan bir sınıftaki özel bir yöntemdir. Girdi parametrelerini kabul eden normal bir MATLAB işlevidir, ancak aynı zamanda belirli kurallara uyması gerekir.

MATLAB varsayılan bir tane oluşturduğu için yapıcılar gerekli değildir. Ancak pratikte burası bir nesnenin durumunu tanımlamak için kullanılan bir yerdir. Örneğin, özellikler öznitelikler belirtilerek kısıtlanabilir. Ardından, bir kurucu bu tür özellikleri varsayılan olarak başlatabilir veya aslında bir kurucunun giriş parametreleri tarafından gönderilebilen kullanıcı tanımlı değerler.

Basit bir sınıfın yapıcısını çağırma

Bu basit bir ‘Kişi’ sınıfıdır.

classdef Person
    properties
        name
        surname
        address
    end
    
    methods
        function obj = Person(name,surname,address)
            obj.name = name;
            obj.surname = surname;
            obj.address = address;
        end
    end
end

Bir kurucunun adı, bir sınıfın adıyla aynıdır. Sonuç olarak, yapıcılar sınıfının adıyla çağrılır. Bir “Kişi” sınıfı aşağıdaki gibi oluşturulabilir:

>> p = Person('John','Smith','London')
p = 
  Person with properties:

       name: 'John'
    surname: 'Smith'
    address: 'London'

Bir alt sınıfın yapıcısını çağırma

Ortak özellikler veya yöntemler paylaşılıyorsa, sınıflar üst sınıflardan devralınabilir. Bir sınıf diğerinden miras alındığında, muhtemelen bir üst sınıfın kurucusunun çağrılması gerekir.

“Üye” sınıfı, “Kişi” sınıfından miras alır, çünkü “Üye”, Kişi sınıfıyla aynı özellikleri kullanır, ancak tanımına “ödeme"yi de ekler.

classdef Member < Person
    properties
        payment
    end

    methods
        function obj = Member(name,surname,address,payment)
            obj = obj@Person(name,surname,address);
            obj.payment = payment;
        end
    end
end

“Kişi” sınıfına benzer şekilde, “Üye”, yapıcısı çağrılarak oluşturulur:

>> m = Member('Adam','Woodcock','Manchester',20)
m = 
  Member with properties:

    payment: 20
       name: 'Adam'
    surname: 'Woodcock'
    address: 'Manchester'

‘Kişi’ yapıcısı üç giriş parametresi gerektirir. ‘Üye’ bu gerçeğe saygı göstermeli ve bu nedenle, ‘Person’ sınıfının bir yapıcısını üç parametre ile çağırmalıdır. Satır tarafından yerine getirilir:

obj = obj@Person(name,surname,address);

Yukarıdaki örnek, bir alt sınıfın üst sınıfı için bilgiye ihtiyaç duyduğu durumu gösterir. Bu nedenle, bir “Member” yapıcısının dört parametreye ihtiyacı vardır: üçü ana sınıfı için ve bir tanesi kendisi için.

Sınıflardan ve soyut sınıflardan miras alma #

Sorumluluk Reddi: Burada sunulan örnekler yalnızca soyut sınıfların ve kalıtımın kullanımını gösterme amaçlıdır ve pratik bir kullanım olmayabilir. Ayrıca MATLAB’de polimorfik diye bir şey yoktur ve bu nedenle soyut sınıfların kullanımı sınırlıdır. Bu örnek, bir sınıfı kimin oluşturacağını, başka bir sınıftan miras alacağını ve ortak bir arayüz tanımlamak için soyut bir sınıf uygulayacağını göstermek içindir.

MATLAB’da soyut sınıfların kullanımı oldukça sınırlıdır, ancak yine de birkaç durumda faydalı olabilir.

Diyelim ki bir mesaj kaydedici istiyoruz. Aşağıdakine benzer bir sınıf oluşturabiliriz:

classdef ScreenLogger
    properties(Access=protected)
        scrh;
    end
    
    methods
        function obj = ScreenLogger(screenhandler)
            obj.scrh = screenhandler;
        end
        
        function LogMessage(obj, varargin)
            if ~isempty(varargin)
                varargin{1} = num2str(varargin{1});
                fprintf(obj.scrh, '%s\n', sprintf(varargin{:}));
            end
        end
    end
end

Özellikler ve yöntemler

Kısacası, özellikler bir nesnenin durumunu tutarken, yöntemler arayüz gibidir ve nesneler üzerindeki eylemleri tanımlar.

“scrh” özelliği korunur. Bu yüzden bir kurucuda başlatılmalıdır. Bu özelliğe erişmek için başka yöntemler (alıcılar) vardır, ancak bu örneğin üstesinden gelinemez. Özellikler ve yöntemlere, nokta gösterimi ve ardından bir yöntem veya özellik adı kullanılarak bir nesneye referans tutan bir değişken aracılığıyla erişilebilir:

mylogger = ScreenLogger(1);                         % OK
mylogger.LogMessage('My %s %d message', 'very', 1); % OK
mylogger.scrh = 2;                                  % ERROR!!! Access denied

Özellikler ve yöntemler genel, özel veya korumalı olabilir. Bu durumda, korumalı, ‘scrh’ ye kalıtsal bir sınıftan erişebileceğim, ancak dışarıdan erişemeyeceğim anlamına gelir. Varsayılan olarak tüm özellikler ve yöntemler herkese açıktır. Bu nedenle LogMessage() sınıf tanımının dışında serbestçe kullanılabilir. Ayrıca LogMessage bir arayüz tanımlar, yani bir nesnenin özel mesajlarımızı günlüğe kaydetmesini istediğimizde aramamız gereken şey budur.

Başvuru

Diyelim ki logger’ımı kullandığım bir komut dosyam var:

clc;
% ... a code
logger = ScreenLogger(1);
% ... a code
logger.LogMessage('something');
% ... a code
logger.LogMessage('something');
% ... a code
logger.LogMessage('something');
% ... a code
logger.LogMessage('something');

Aynı kaydediciyi kullandığım birden fazla yerim varsa ve bunu bir dosyaya mesaj yazmak gibi daha karmaşık bir şeyle değiştirmek istersem, başka bir nesne oluşturmam gerekir:

classdef DeepLogger
    properties(SetAccess=protected)
        FileName
    end
    methods
        function obj = DeepLogger(filename)
            obj.FileName = filename;
        end
        
        function LogMessage(obj, varargin)
            if ~isempty(varargin)
                varargin{1} = num2str(varargin{1});
                fid = fopen(obj.fullfname, 'a+t');
                fprintf(fid, '%s\n', sprintf(varargin{:}));
                fclose(fid);
            end
        end
    end 
end

ve bir kodun bir satırını şununla değiştirin:

clc;
% ... a code
logger = DeepLogger('mymessages.log');

Yukarıdaki yöntem basitçe bir dosya açar, dosyanın sonuna bir mesaj ekler ve dosyayı kapatır. Şu anda, arayüzümle tutarlı olmak için, bir yöntemin adının ‘LogMessage()’ olduğunu hatırlamam gerekiyor, ancak aynı şekilde başka bir şey de olabilir. MATLAB, soyut sınıfları kullanarak geliştiriciyi aynı ada bağlı kalmaya zorlayabilir. Diyelim ki herhangi bir günlükçü için ortak bir arayüz tanımladık:

classdef MessageLogger
    methods(Abstract=true)
        LogMessage(obj, varargin);
    end
end

Şimdi, hem ‘ScreenLogger’ hem de ‘DeepLogger’ bu sınıftan miras alırsa, ‘LogMessage()’ tanımlı değilse MATLAB bir hata üretecektir. Soyut sınıflar, aynı arabirimi kullanabilen benzer sınıflar oluşturmaya yardımcı olur.

Bu örnek uğruna, biraz farklı bir değişiklik yapacağım. DeepLogger’ın aynı anda hem ekranda hem de dosyada günlük mesajı yapacağını varsayacağım. “ScreenLogger” zaten ekranda mesajları günlüğe kaydettiği için, tekrarı önlemek için “DeepLogger"ı “ScreenLogger"dan devralacağım. “ScreenLogger”, ilk satır dışında hiç değişmiyor:

classdef ScreenLogger < MessageLogger
// the rest of previous code

Ancak, “DeepLogger”, “LogMessage” yönteminde daha fazla değişikliğe ihtiyaç duyuyor:

classdef DeepLogger < MessageLogger & ScreenLogger
    properties(SetAccess=protected)
        FileName
        Path
    end
    methods
        function obj = DeepLogger(screenhandler, filename)
            [path,filen,ext] = fileparts(filename);
            obj.FileName = [filen ext];
            pbj.Path     = pathn;
            obj = obj@ScreenLogger(screenhandler);
        end
        function LogMessage(obj, varargin)
            if ~isempty(varargin)
                varargin{1} = num2str(varargin{1});
                LogMessage@ScreenLogger(obj, varargin{:});
                fid = fopen(obj.fullfname, 'a+t');
                fprintf(fid, '%s\n', sprintf(varargin{:}));
                fclose(fid);
            end
        end
    end
end

İlk olarak, yapıcıdaki özellikleri başlatıyorum. İkincisi, bu sınıf ScreenLoggerdan miras aldığı için bu ana nesneyi de başlatmam gerekiyor. Bu satır daha da önemlidir çünkü ‘ScreenLogger’ yapıcısı kendi nesnesini başlatmak için bir parametreye ihtiyaç duyar. Bu hat:

obj = obj@ScreenLogger(screenhandler);

basitçe “ScreenLogger’ın kurucusunu arayın ve bir ekran işleyici ile başlatın” diyor. Burada “scrh"yi korumalı olarak tanımladığımı belirtmekte fayda var. Bu nedenle, bu özelliğe ‘DeepLogger’dan eşit olarak erişebildim. Özellik özel olarak tanımlanmışsa. Başlatmanın tek yolu consuctor kullanmaktır.

Başka bir değişiklik “yöntemler” bölümündedir. Tekrardan kaçınmak için bir üst sınıftan bir mesajı ekrana kaydetmek için LogMessage() çağırıyorum. Ekran kaydında iyileştirmeler yapmak için herhangi bir şeyi değiştirmem gerekirse, şimdi bunu tek bir yerde yapmam gerekiyor. Geri kalan kod, ‘DeepLogger’ın bir parçası olduğu gibi aynıdır.

Bu sınıf aynı zamanda soyut bir ‘MessageLogger’ sınıfından miras aldığı için, ‘DeepLogger’ içindeki ‘LogMessage()’ öğesinin de tanımlandığından emin olmak zorundaydım. ‘MessageLogger’dan miras almak burada biraz zor. Sanırım ‘LogMessage’ın yeniden tanımlanması zorunlu - benim tahminim.

logger’ın uygulandığı kod açısından, sınıflarda ortak bir arayüz sayesinde, tüm kodda bu tek satırın değiştirilmesinin herhangi bir sorun yaratmayacağından emin olabilirim. Aynı mesajlar daha önce olduğu gibi ekranda oturum açacak, ancak ek olarak kod bu tür mesajları bir dosyaya yazacak.

clc;
% ... a code
logger = DeepLogger(1, 'mylogfile.log');
% ... a code
logger.LogMessage('something');
% ... a code
logger.LogMessage('something');
% ... a code
logger.LogMessage('something');
% ... a code
logger.LogMessage('something');

Umarım bu örnekler sınıfların kullanımını, kalıtımın kullanımını ve soyut sınıfların kullanımını açıklamıştır.

not. Yukarıdaki sorunun çözümü, birçoğundan biridir. Daha az karmaşık olan başka bir çözüm, ‘ScreenLoger’ı ‘FileLogger’ vb. gibi başka bir kaydedicinin bileşeni yapmak olacaktır. ‘ScreenLogger’ özelliklerden birinde tutulacaktır. “LogMessage”, “ScreenLogger"ın “LogMessage"ını çağırır ve bir ekranda metin gösterir. MATLAB’da sınıfların nasıl çalıştığını göstermek için daha karmaşık bir yaklaşım seçtim. Aşağıdaki örnek kod:

classdef DeepLogger < MessageLogger
    properties(SetAccess=protected)
        FileName
        Path
        ScrLogger
    end
    methods
        function obj = DeepLogger(screenhandler, filename)
            [path,filen,ext] = fileparts(filename);
            obj.FileName     = [filen ext];
            obj.Path         = pathn;
            obj.ScrLogger    = ScreenLogger(screenhandler);
        end
        function LogMessage(obj, varargin)
            if ~isempty(varargin)
                varargin{1} = num2str(varargin{1});
                obj.LogMessage(obj.ScrLogger, varargin{:}); % <-------- thechange here
                fid = fopen(obj.fullfname, 'a+t');
                fprintf(fid, '%s\n', sprintf(varargin{:}));
                fclose(fid);
            end
        end
    end
end
