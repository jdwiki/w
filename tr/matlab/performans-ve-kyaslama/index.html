Profiler’ı kullanarak performans darboğazlarını belirleme #

MATLAB Profiler, [yazılım profili oluşturma](https://en.wikipedia.org/wiki/_(computer_programming) için bir araçtır )) MATLAB kodu. Profiler’ı kullanarak, hem yürütme süresinin hem de bellek tüketiminin görsel bir temsilini elde etmek mümkündür.

Profiler’ı çalıştırmak iki şekilde yapılabilir:

  • Editörde bazı .m dosyası açıkken MATLAB GUI’de “Çalıştır ve Zaman” düğmesine tıklamak (R2012b‘ye eklendi).

  • Programlı olarak, şunu kullanarak:

    profile on
<some code we want to test>
profile off

Aşağıda bazı örnek kodlar ve profil oluşturmanın sonucu verilmiştir:

function docTest

for ind1 = 1:100
  [~] = var(...
            sum(...
                randn(1000)));
end

spy

Yukarıdakilerden ‘spy’ fonksiyonunun toplam yürütme süresinin yaklaşık %25’ini aldığını öğreniyoruz. “Gerçek kod” durumunda, bu kadar büyük bir yürütme süresi yüzdesi alan bir işlev, optimizasyonundan kaçınılması gereken “var” ve “cla"ya benzer işlevlerin aksine, optimizasyon için iyi bir aday olacaktır.

Ayrıca, o giriş için yürütme süresinin ayrıntılı bir dökümünü görmek için Function Name sütunundaki girişlere tıklamak mümkündür. İşte “casus"a tıklama örneği:

Profiler’ı çalıştırmadan önce profile('-memory') çalıştırılarak bellek tüketiminin profilini çıkarmak da mümkündür.

Birden çok işlevin yürütme zamanını karşılaştırma #

Yaygın olarak kullanılan tic ve [toc](http:// de.mathworks.com/help/matlab/ref/toc.html), bir işlevin veya kod parçacıklarının yürütme süresi hakkında kabaca bir fikir sağlayabilir.

Birkaç işlevi karşılaştırmak için kullanılmamalıdır. Neden? Yukarıdaki çözümü kullanarak tüm kod parçacıklarının bir komut dosyası içinde karşılaştırması için eşit koşullar sağlamak neredeyse imkansızdır. Belki işlevler aynı işlev alanını ve ortak değişkenleri paylaşır, bu nedenle daha sonra işlevler ve kod parçacıkları olarak adlandırılanlar zaten önceden başlatılmış değişkenlerden ve işlevlerden yararlanır. Ayrıca, JIT derleyicisinin daha sonra adlandırılan parçacıkları eşit olarak ele alıp almayacağına dair bir fikir yoktur.

Karşılaştırmalar için özel işlev timeit. Aşağıdaki örnek, kullanımını göstermektedir.

“A” dizisi ve “B” matrisi vardır. Farklı eleman sayıları sayılarak ‘B’nin hangi satırının ‘A’ya en çok benzediği belirlenmelidir.

function t = bench()
    A = [0 1 1 1 0 0];
    B = perms(A);

    % functions to compare
    fcns = {
        @() compare1(A,B);
        @() compare2(A,B);
        @() compare3(A,B);
        @() compare4(A,B);
    };

    % timeit
    t = cellfun(@timeit, fcns);
end

function Z = compare1(A,B)  
    Z = sum(  bsxfun(@eq,  A,B) , 2);
end
function Z = compare2(A,B)  
    Z = sum(bsxfun(@xor, A, B),2);
end
function Z = compare3(A,B)  
    A = logical(A);
    Z = sum(B(:,~A),2) + sum(~B(:,A),2);
end
function Z = compare4(A,B)  
     Z = pdist2( A, B, 'hamming', 'Smallest', 1 );
end

Bu karşılaştırma yöntemi ilk olarak bu yanıtta görüldü.

‘Bekar’ olmak sorun değil! #

Genel Bakış: #

MATLAB’deki sayısal diziler için varsayılan veri türü “double"dır. “double”, sayıların kayan nokta gösterimi‘dir ve bu biçim değer başına 8 bayt (veya 64 bit) alır. bazı durumlarda, ör. yalnızca tamsayılarla ilgilenirken veya sayısal kararsızlık yakın bir sorun olmadığında, bu kadar yüksek bit derinliği gerekli olmayabilir. Bu nedenle, “tek” kesinliğin (veya diğer uygun [türlerin2) faydalarını göz önünde bulundurmanız önerilir:

  • Daha hızlı yürütme süresi (özellikle GPU’larda fark edilir).
  • Bellek tüketiminin yarısı: yetersiz bellek hatası nedeniyle ‘double’ başarısız olduğunda başarılı olabilir; dosya olarak saklarken daha kompakt.

Bir değişkeni desteklenen herhangi bir veri türünden “tek"e dönüştürmek aşağıdakiler kullanılarak yapılır:

sing_var = single(var);

Yaygın olarak kullanılan bazı işlevler (örneğin: sıfırlar, [eye](http://www .mathworks.com/help/matlab/ref/eye.html#btpkgsh-1), birler, vb.) varsayılan olarak “çift” değerler verir, çıktının tipini/sınıfını belirlemeye izin verir.

Bir komut dosyasındaki değişkenleri varsayılan olmayan bir kesinlik/tür/sınıfa dönüştürme: #

Temmuz 2016 itibariyle, default MATLAB veri türünü “double"dan değiştirmenin belgelenmiş bir yolu yoktur.

MATLAB’de, yeni değişkenler genellikle onları oluştururken kullanılan değişkenlerin veri türlerini taklit eder. Bunu göstermek için aşağıdaki örneği göz önünde bulundurun:

A = magic(3);
B = diag(A);
C = 20*B;
>> whos C
  Name      Size            Bytes  Class     Attributes
  C         3x1                24  double 


A = single(magic(3)); % A is converted to "single"
B = diag(A);
C = B*double(20);     % The stricter type, which in this case is "single", prevails
D = single(size(C));  % It is generally advised to cast to the desired type explicitly.
>> whos C
  Name      Size            Bytes  Class     Attributes
  C         3x1                12  single

Bu nedenle, değişikliğin kod boyunca nüfuz etmesi için birkaç başlangıç ​​değişkenini oluşturmak/dönüştürmek yeterli görünebilir - ancak bu önerilir (aşağıdaki Uyarılar ve Tuzaklar bölümüne bakın).

Uyarılar ve Tuzaklar: #

  1. Sayısal parazit (“tekli"den “çift"e yayın yaparken) veya bilgi kaybı (“çift"ten “tekli"ye yayın yaparken veya belirli tamsayılar arasında yayın yaparken) nedeniyle tekrarlanan dönüştürmeler önerilir. türler), ör. :

     double(single(1.2)) == double(1.2)   
 ans =
      0

    This can be mitigated somewhat using typecast. See also https://www.wikiod.com/tr/matlab/yaygn-hatalar-ve-hatalar#Kayan nokta yanlışlığının farkında olun

  2. Yalnızca örtük veri yazmaya (yani, MATLAB’ın bir hesaplamanın çıktısının türünü tahmin ettiği gibi) güvenmek, ortaya çıkabilecek birkaç istenmeyen etki nedeniyle önerilir:

    • Loss of information: when a double result is expected, but a careless combination of single and double operands yields single precision.

    • Unexpectedly high memory consumption: when a single result is expected but a careless computation results in a double output.

    • Unnecessary overhead when working with GPUs: when mixing gpuArray types (i.e. variables stored in VRAM) with non-gpuArray variables (i.e. those usually stored in RAM) the data will have to be transferred one way or the other before the computation can be performed. This operation takes time, and can be very noticeable in repetative computations.

    • Errors when mixing floating-point types with integer types: functions like mtimes (*) are not defined for mixed inputs of integer and floating point types - and will error. Functions like times (.*) are not defined at all for integer-type inputs - and will again error.

      >> ones(3,3,'int32')*ones(3,3,'int32')
Error using  * 
MTIMES is not fully supported for integer classes. At least one input must be scalar.

>> ones(3,3,'int32').*ones(3,3,'double')
Error using  .* 
Integers can only be combined with integers of the same class, or scalar doubles.

    For better code readability and reduced risk of unwanted types, a defensive approach is advised, where variables are explicitly cast to the desired type.

Ön tahsisin önemi #

MATLAB’daki diziler bellekte sürekli bloklar olarak tutulur, MATLAB tarafından otomatik olarak tahsis edilir ve serbest bırakılır. MATLAB, bir dizinin yeniden boyutlandırılması gibi bellek yönetimi işlemlerini kullanımı kolay sözdiziminin arkasına gizler:

a = 1:4

a =

     1     2     3     4

a(5) = 10  % or alternatively a = [a, 10]

a =

     1     2     3     4    10

Yukarıdakilerin önemsiz bir işlem olmadığını anlamak önemlidir, a(5) = 10, MATLAB’ın 5 boyutunda yeni bir bellek bloğu tahsis etmesine, ilk 4 sayıyı kopyalamasına ve 5’inci sayıya ayarlamasına neden olacaktır. 10. Bu bir ‘O(numel(a))’ işlemidir ve ‘O(1)’ değil.

Aşağıdakileri göz önünde bulundur:

clear all
n=12345678;
a=0;
tic
for i = 2:n
    a(i) = sqrt(a(i-1)) + i;
end
toc

Elapsed time is 3.004213 seconds.

‘a’ bu döngüde ’n’ kez yeniden tahsis edilir (MATLAB tarafından üstlenilen bazı optimizasyonlar hariç)! MATLAB’ın bize bir uyarı verdiğini unutmayın:

“‘a’ değişkeni, her döngü yinelemesinde boyutunu değiştiriyor gibi görünüyor. Hız için önceden ayırmayı düşünün.”

Önceden tahsis ettiğimizde ne olur?

a=zeros(1,n);
tic
for i = 2:n
    a(i) = sqrt(a(i-1)) + i;
end
toc

Elapsed time is 0.410531 seconds.

Çalışma zamanının bir büyüklük sırasına göre azaldığını görebiliriz.

Ön tahsis yöntemleri:

MATLAB, kullanıcının özel gereksinimlerine bağlı olarak vektörlerin ve matrislerin tahsisi için çeşitli işlevler sağlar. Bunlar: sıfırlar, birler, nan, göz, doğru vb.

a = zeros(3)       % Allocates a 3-by-3 matrix initialized to 0
a =

     0     0     0
     0     0     0
     0     0     0

a = zeros(3, 2)     % Allocates a 3-by-2 matrix initialized to 0
a =

     0     0
     0     0
     0     0

a = ones(2, 3, 2)      % Allocates a 3 dimensional array (2-by-3-by-2) initialized to 1
a(:,:,1) =

     1     1     1
     1     1     1


a(:,:,2) =

     1     1     1
     1     1     1

a = ones(1, 3) * 7  % Allocates a row vector of length 3 initialized to 7
a =

     7     7     7

Bir veri türü de belirtilebilir:

a = zeros(2, 1, 'uint8');  % allocates an array of type uint8

Mevcut bir dizinin boyutunu klonlamak da kolaydır:

a = ones(3, 4);       % a is a 3-by-4 matrix of 1's
b = zeros(size(a));  % b is a 3-by-4 matrix of 0's

Ve türü klonlayın:

a = ones(3, 4, 'single');       % a is a 3-by-4 matrix of type single
b = zeros(2, 'like', a);        % b is a 2-by-2 matrix of type single

‘beğen’in aynı zamanda complexity ve sparsity‘yi de klonladığını unutmayın.

Ön tahsisat, rand, gallery, kron, bsxfun gibi son gerekli boyutta bir dizi döndüren herhangi bir işlev kullanılarak örtük olarak elde edilir. , kolon ve diğerleri. Örneğin, doğrusal olarak değişen öğelere sahip vektörleri ayırmanın yaygın bir yolu, iki nokta üst üste operatörünü kullanmaktır (2 veya 3 işlenen değişkeni1 ile):

a = 1:3 
a =

     1     2     3

a = 2:-3:-4
a =

     2    -1    -4

Hücre dizileri, “sıfır()” işlevine çok benzer şekilde “hücre()” işlevi kullanılarak atanabilir.

a = cell(2,3)
a = 

    []    []    []
    []    []    []

Hücre dizilerinin, hücre içeriğinin belleğindeki konumlara işaretçiler tutarak çalıştığını unutmayın. Bu nedenle, tüm ön tahsis ipuçları, tek tek hücre dizisi öğeleri için de geçerlidir.

bir ND dizisini yeniden düzenlemek genel performansı iyileştirebilir #

Bazı durumlarda işlevleri bir dizi ND dizisine uygulamamız gerekir. Bu basit örneğe bakalım.

A(:,:,1) = [1 2; 4 5];
A(:,:,2) = [11 22; 44 55];
B(:,:,1) = [7 8; 1 2];
B(:,:,2) = [77 88; 11 22];

A =

ans(:,:,1) =

   1   2 
   4   5 

ans(:,:,2) =

   11   22
   44   55

>> B
B =

ans(:,:,1) =

   7   8
   1   2

ans(:,:,2) =

   77   88
   11   22

Her iki matris de 3B, diyelim ki aşağıdakileri hesaplamamız gerekiyor:

result= zeros(2,2);
...
for k = 1:2 
   result(i,j) = result(i,j) + abs( A(i,j,k) - B(i,j,k) );
...

if k is very large, this for-loop can be a bottleneck since MATLAB order the data in a column major fashion. So a better way to compute "result" could be:

% trying to exploit the column major ordering
Aprime = reshape(permute(A,[3,1,2]), [2,4]);
Bprime = reshape(permute(B,[3,1,2]), [2,4]);


>> Aprime
Aprime =

    1    4    2    5
   11   44   22   55

>> Bprime
Bprime =

    7    1    8    2
   77   11   88   22

Şimdi yukarıdaki döngüyü aşağıdaki gibi değiştiriyoruz:

result= zeros(2,2);
....
temp = abs(Aprime - Bprime);
for k = 1:2
    result(i,j) = result(i,j) + temp(k, i+2*(j-1));
...

Verileri yeniden düzenledik, böylece önbelleği kullanabiliriz. Permütasyon ve yeniden şekillendirme maliyetli olabilir, ancak büyük ND dizileriyle çalışırken bu işlemlerle ilgili hesaplama maliyeti, düzenlenmemiş dizilerle çalışmaktan çok daha düşüktür.
