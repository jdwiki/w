Hizmetleri Kullanma

  • Farklı sayfalar arasında bilgi paylaşın

Bu sayfada

  • Farklı sayfalar arasında bilgi paylaşın

Paylaşılan hizmetleri kullanmayla ilgili çok önemli bir şey, paylaşılmaları gereken en üstteki bileşenin “sağlayıcılar” dizisine dahil edilmeleri gerektiğidir.

Nedenmiş? Diyelim ki, her bir ‘Bileşen’den ‘sağlayıcılar’ dizisine ‘MyService’ referansını dahil ettiğimizi varsayalım. Gibi bir şey:

@Component({
  templateUrl:"page1.html",
  providers: [MyService]
})

Ve

@Component({
  templateUrl:"page2.html",
  providers: [MyService]
})

Bu şekilde her bileşen için hizmetin yeni bir örneği oluşturulacaktır, böylece bir sayfanın verileri kaydedeceği örnek, verileri almak için kullanılan örnekten farklı olacaktır. Yani bu işe yaramayacak.

Tüm uygulamanın aynı örneği kullanmasını sağlamak için (hizmetin singleton hizmeti olarak çalışmasını sağlamak) referansını şu şekilde ‘Uygulama Bileşeni’ne ekleyebiliriz:

@Component({
  template: '<ion-nav [root]="rootPage"></ion-nav>',
  providers: [MyService]
})

Ayrıca ‘ionicBootstrap(MyApp, [MyService]);’ içine ‘MyService’ referansını da ekleyebilirsiniz, ancak Angular2 stil kılavuzlarına göre

En üstte Angular 2 enjektöre hizmet verin paylaşılacakları bileşen.

Neden? Angular 2 enjektörü hiyerarşiktir.

Neden? Bir üst düzey bileşene hizmet sağlarken, bu örnek paylaşılır ve o üst öğenin tüm alt bileşenleri tarafından kullanılabilir seviye bileşeni.

Neden? Bu, bir hizmet yöntemleri veya durumu paylaştığında idealdir.

Neden? Bu, iki farklı bileşenin farklı olması gerektiğinde ideal değildir. bir hizmetin örnekleri. Bu senaryoda sağlamak daha iyi olurdu yeni ve ayrı olması gereken bileşen düzeyinde hizmet örnek.

Ve

Çalışacaktır. Bu sadece en iyi uygulama değil. Önyükleme sağlayıcısı seçenek, Angular’ın kendi yapılandırmasını yapılandırmak ve geçersiz kılmak için tasarlanmıştır yönlendirme desteği gibi önceden kayıtlı hizmetler.

… “Uygulama Bileşeni” en iyi seçim olacaktır.

Farklı sayfalar arasında bilgi paylaşın #

Paylaşılan hizmetleri kullanmanın en kolay örneklerinden biri, uygulamamızın belirli bir sayfasından bazı verileri depolamak ve ardından bu verileri başka bir sayfadan tekrar almak istediğimiz zamandır.

Bir seçenek, bu verileri bir parametre olarak göndermek olabilir (örneğin, bir sayfa diğerini çağırırsa), ancak bu verileri uygulamanın tamamen farklı bir bölümünden kullanmak istiyorsak, bu en iyi yol değil gibi görünüyor. BT. İşte o zaman shared services devreye girer.

Bu örnekte, yalnızca iki basit yöntemi olan ‘MyService’ adlı basit bir hizmet kullanacağız: bir dizeyi saklamak için ‘saveMessage()’ ve onu tekrar almak için ‘getMessage()’. Bu kod, çalışırken görebileceğiniz bu çalışan planlayıcının parçasıdır.

import {Injectable} from '@angular/core';

@Injectable()
export class MyService { 
  
  private message: string;
  
  constructor(){ }
  
  public saveMessage(theMessage: string): void {
    this.message = theMessage;
  }
  
  public getMessage(): string {
    return this.message;
  }
}

Ardından, yeni bir mesajı saklamak istediğimizde, MyService örneğinden (sadece service olarak adlandırılır) saveMessage(theMessageWeWantToSave); yöntemini kullanabiliriz.

import { Component } from "@angular/core";
import { MyService } from 'service.ts';

@Component({
  templateUrl:"page1.html"
})
export class Page1 {
  
  message: string;
  
  // ... 
   
  public saveSecretMessage(): void {
    this.service.saveMessage(this.message);
  }      
}

Aynı şekilde, bu verileri almak istediğimizde servis örneğinden getMessage() yöntemini şu şekilde kullanabiliriz:

import { Component } from "@angular/core";
import { MyService } from 'service.ts';

@Component({
  templateUrl:"page2.html"
})
export class Page2 {
  
  enteredMessage: string;
  
  constructor(private service: MyService) {
    this.enteredMessage = this.service.getMessage();
  }
    
  // ...
}

Lütfen “MyService” hizmetinin referansının nereye dahil edilmesi gerektiğini ve nedenini görmek için Açıklamalar bölümünü kontrol etmeyi unutmayın.
