Genel olarak Birim Testi, özellikleri değiştirirken/eklerken sorunları önlemek için bir ürüne ek güvenlik sağlar. “HER ŞEY HALA ÇALIŞIYOR” yazan bir güvenlik ağı. Birim Testleri, uygun bir QA’nın yapabileceği gerçek kullanıcı testlerinin hiçbir şekilde yerini almaz.

Bu belgede örnekleri bu depoya dayandıracağız: https://github.com/driftyco/ionic-unit-testing-example

Karma/Yasemin ile Birim Testleri #

İyonikte birim testi, herhangi bir açısal uygulamadakiyle aynıdır.

Bunu yapmak için birkaç çerçeve kullanacağız.

Karma - testleri çalıştırmak için bir çerçeve

Yasemin - testler yazmak için bir çerçeve

PhantomJS - javascript’i tarayıcı olmadan çalıştıran bir uygulama

Her şeyden önce, her şeyi yüklememize izin verin, bu nedenle package.json’ınızın bu satırları geliştirici bağımlılıklarında içerdiğinden emin olun. Geliştirici bağımlılıklarının uygulamanızı hiçbir şekilde etkilemediğini ve yalnızca geliştiriciye yardımcı olmak için orada olduklarını not etmenin önemli olduğunu düşünüyorum.

"@ionic/app-scripts": "1.1.4",
"@ionic/cli-build-ionic-angular": "0.0.3",
"@ionic/cli-plugin-cordova": "0.0.9",

Paketlerin üzerinden biraz geçmek için

"angular2-template-loader": "^0.6.2", - will load and compile the angular2 html files.

"ts-loader": "^2.0.3", - will compile the actual typescript files

"null-loader": "^0.1.1", - will not load the assets that will be missing, such as fonts and images. We are testing, not image lurking.

Bu betiği ayrıca package.json betiklerimize eklemeliyiz:

"test": "karma start ./test-config/karma.conf.js"

Ayrıca tsconfig dosyasında, spec.ts dosyalarını derlemeden hariç tuttuğunuzu da not edin:

 "exclude": [
    "node_modules",
    "src/**/*.spec.ts"
  ],

Tamam, şimdi gerçek test yapılandırmasını alalım. Proje klasörünüzde bir test-config klasörü oluşturun. (Paket.json betiğinde bahsedildiği gibi) Klasörün içinde 3 dosya oluşturun:

“webpack.test.js” - web paketine test süreci için hangi dosyaların yükleneceğini söyler

var webpack = require('webpack');
var path = require('path');

module.exports = {
  devtool: 'inline-source-map',

  resolve: {
    extensions: ['.ts', '.js']
  },

  module: {
    rules: [
      {
        test: /\.ts$/,
        loaders: [
          {
            loader: 'ts-loader'
          } , 'angular2-template-loader'
        ]
      },
      {
        test: /\.html$/,
        loader: 'html-loader'
      },
      {
        test: /\.(png|jpe?g|gif|svg|woff|woff2|ttf|eot|ico)$/,
        loader: 'null-loader'
      }
    ]
  },

  plugins: [
    new webpack.ContextReplacementPlugin(
      // The (\\|\/) piece accounts for path separators in *nix and Windows
      /angular(\\|\/)core(\\|\/)(esm(\\|\/)src|src)(\\|\/)linker/,
      root('./src'), // location of your src
      {} // a map of your routes
    )
  ]
};

function root(localPath) {
  return path.resolve(__dirname, localPath);
}

karma-test-shim.js - bölge ve test kitaplıkları gibi açısal ilgili kitaplıkları yükleyecek ve modülü test için yapılandıracak.

“karma.conf.js” - karma ile nasıl test edileceğinin yapılandırmasını tanımlar. Burada, bu işlemi diğer şeylerin yanı sıra görünmez ve daha hızlı hale getirmek için Chrome’dan PhantomJS’ye geçebilirsiniz.

var webpackConfig = require('./webpack.test.js');

module.exports = function (config) {
  var _config = {
    basePath: '',

    frameworks: ['jasmine'],

    files: [
      {pattern: './karma-test-shim.js', watched: true}
    ],

    preprocessors: {
      './karma-test-shim.js': ['webpack', 'sourcemap']
    },

    webpack: webpackConfig,

    webpackMiddleware: {
      stats: 'errors-only'
    },

    webpackServer: {
      noInfo: true
    },

    browserConsoleLogOptions: {
      level: 'log',
      format: '%b %T: %m',
      terminal: true
    },

    reporters: ['kjhtml', 'dots'],
    port: 9876,
    colors: true,
    logLevel: config.LOG_INFO,
    autoWatch: true,
    browsers: ['Chrome'],
    singleRun: false
  };

  config.set(_config);
};

Artık her şeyi yapılandırdığımıza göre, gerçek bir test yazalım. Bu örnek için bir app.component spec dosyası yazacağız. Ana bileşen yerine bir sayfa için testler görmek istiyorsanız buraya bakabilirsiniz: https://github.com/driftyco/ionic-unit-testing-example/blob/master/src/pages/page1/page1. özellikler

İlk yapmamız gereken, yapıcımızı test etmek. Bu, app.component’imizin yapıcısını oluşturacak ve çalıştıracaktır.

  beforeEach(async(() => {
    TestBed.configureTestingModule({
      declarations: [MyApp],
      imports: [
        IonicModule.forRoot(MyApp)
      ],
      providers: [
        StatusBar,
        SplashScreen
      ]
    })
  }));

Bildiri, ana iyonik uygulamamızı içerecektir. İthalatlar, bu test için gereken ithalatları yapacaktır. Her şey değil.

Sağlayıcılar, yapıcıya enjekte edilen ancak içe aktarmanın parçası olmayan şeyleri içerecektir. Örneğin, app.component Platform hizmetini enjekte eder, ancak IonicModule’un bir parçası olduğu için sağlayıcılarda bundan bahsetmeye gerek yoktur.

Sonraki testler için bileşenimizin bir örneğini almamız gerekecek:

  beforeEach(() => {
    fixture = TestBed.createComponent(MyApp);
    component = fixture.componentInstance;
  });

Ardından, her şeyin yolunda olduğunu görmek için birkaç test yapın:

  it ('should be created', () => {
    expect(component instanceof MyApp).toBe(true);
  });

  it ('should have two pages', () => {
    expect(component.pages.length).toBe(2);
  });

Yani sonunda şöyle bir şeye sahip olacağız:

import { async, TestBed } from '@angular/core/testing';
import { IonicModule } from 'ionic-angular';

import { StatusBar } from '@ionic-native/status-bar';
import { SplashScreen } from '@ionic-native/splash-screen';

import { MyApp } from './app.component';

describe('MyApp Component', () => {
  let fixture;
  let component;

  beforeEach(async(() => {
    TestBed.configureTestingModule({
      declarations: [MyApp],
      imports: [
        IonicModule.forRoot(MyApp)
      ],
      providers: [
        StatusBar,
        SplashScreen
      ]
    })
  }));

  beforeEach(() => {
    fixture = TestBed.createComponent(MyApp);
    component = fixture.componentInstance;
  });

  it ('should be created', () => {
    expect(component instanceof MyApp).toBe(true);
  });

  it ('should have two pages', () => {
    expect(component.pages.length).toBe(2);
  });

});

Testleri şuna göre çalıştırın:

npm run test

Ve temel testler için bu kadar. Kendi TestBed’inizi yazmak ve uzun vadede size yardımcı olabilecek testlerde miras almak gibi test yazısını kısaltmanın birkaç yolu vardır.
