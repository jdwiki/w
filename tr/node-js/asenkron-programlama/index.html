Düğüm, her şeyin eşzamansız bir şekilde çalışabileceği bir programlama dilidir. Aşağıda bazı örnekleri ve asenkron çalışmanın tipik özelliklerini bulabilirsiniz.

Sözdizimi #

  • doSomething([args], function([argsCB]) { /* bittiğinde bir şeyler yap */});
  • doSomething([args], ([argsCB]) => { /* bittiğinde bir şeyler yap */ });

Geri arama işlevleri #

JavaScript’te geri arama işlevleri #

JavaScript’te geri arama işlevleri yaygındır. JavaScript’te geri arama işlevleri mümkündür, çünkü işlevler birinci sınıf vatandaşlardır.

Senkronize geri aramalar. #

Geri arama işlevleri senkron veya asenkron olabilir. Eşzamansız geri arama işlevleri daha karmaşık olabileceğinden, burada eşzamanlı bir geri arama işlevine basit bir örnek verilmiştir.

// a function that uses a callback named `cb` as a parameter
function getSyncMessage(cb) {
    cb("Hello World!");
}

console.log("Before getSyncMessage call");
// calling a function and sending in a callback function as an argument.
getSyncMessage(function(message) {
    console.log(message);
});
console.log("After getSyncMessage call");

Yukarıdaki kodun çıktısı:

> Before getSyncMessage call
> Hello World!
> After getSyncMessage call

İlk önce yukarıdaki kodun nasıl yürütüldüğünü anlatacağız. Bu, geri arama kavramını henüz anlamayanlar için daha fazla, zaten anladıysanız bu paragrafı atlamaktan çekinmeyin. İlk önce kod ayrıştırılır ve ardından gerçekleşen ilk ilginç şey, konsola getSyncMessage çağrısından önce çıktısı veren 6. satırın çalıştırılmasıdır. Ardından, ‘getSyncMessage’ işlevinde ‘cb’ adlı parametre için bir argüman olarak anonim bir işlevde gönderen ‘getSyncMessage’ işlevini çağıran satır 8 yürütülür. Yürütme şimdi 3. satırda az önce iletilen cb işlevini yürüten getSyncMessage işlevinin içinde yapılır, bu çağrı anonim işlevde iletilen ileti adlı param için bir “Merhaba Dünya” bağımsız değişken dizesi gönderir. Yürütme daha sonra konsola ‘Merhaba Dünya!’ yazan satır 9’a gider. Ardından yürütme, callstack (ayrıca bkz) satırından 10. satıra, ardından 4. satıra ve son olarak 11. satıra geri dönme sürecinden geçer.

Genel olarak geri aramalar hakkında bilinmesi gereken bazı bilgiler:

  • Bir işleve geri arama olarak gönderdiğiniz işlev, sıfır kez, bir kez veya birden çok kez çağrılabilir. Her şey uygulamaya bağlıdır.
  • Geri arama işlevi, eşzamanlı veya eşzamansız olarak ve muhtemelen hem eşzamanlı hem de eşzamansız olarak çağrılabilir.
  • Tıpkı normal fonksiyonlarda olduğu gibi, fonksiyonunuza parametre verdiğiniz isimler önemli değil, sıra önemlidir. Örneğin 8. satırda “message” parametresi “ifade”, “msg” olarak adlandırılmış olabilir veya saçmalık yapıyorsanız “jellybean” gibi bir şey olabilir. Bu nedenle, geri aramanıza hangi parametrelerin gönderildiğini bilmelisiniz, böylece bunları uygun adlarla doğru sırada alabilirsiniz.

Eşzamansız geri aramalar. #

JavaScript hakkında unutulmaması gereken bir şey, varsayılan olarak eşzamanlı olmasıdır, ancak ortamda onu eşzamansız hale getirebilecek API’ler (tarayıcı, Node.js, vb.) vardır (bununla ilgili daha fazlası burada).

Geri aramaları kabul eden JavaScript ortamlarında zaman uyumsuz olan bazı yaygın şeyler:

  • Olaylar
  • setZaman aşımı
  • setInterval
  • getirme API’si
  • vaatler

Ayrıca, yukarıdaki işlevlerden birini kullanan herhangi bir işlev, bir geri arama alan bir işlevle sarılabilir ve bu durumda geri arama, asenkron bir geri arama olur (gerçi, bir sözü geri arama alan bir işlevle sarmak muhtemelen bir anti-kalıp olarak kabul edilir). vaatleri yerine getirmenin daha çok tercih edilen yolları vardır).

Bu bilgiler ışığında, yukarıdaki senkronize olana benzer bir asenkron fonksiyon oluşturabiliriz.

// a function that uses a callback named `cb` as a parameter
function getAsyncMessage(cb) {
    setTimeout(function () { cb("Hello World!") }, 1000);
}

console.log("Before getSyncMessage call");
// calling a function and sending in a callback function as an argument.
getAsyncMessage(function(message) {
    console.log(message);
});
console.log("After getSyncMessage call");

Aşağıdakileri konsola yazdıran:

> Before getSyncMessage call
> After getSyncMessage call
// pauses for 1000 ms with no output
> Hello World!

Satır yürütme, “getSyncMessage çağrısından önce” satır 6 günlüklerine gider. Ardından yürütme, “cb” parametresi için bir geri arama ile getAsyncMessage’ı çağıran 8. satıra gider. Daha sonra, ilk argüman olarak bir geri arama ve ikinci argüman olarak 300 sayısı ile setTimeout’u çağıran 3. satır yürütülür. ‘setTimeout’ ne yaparsa yapar ve 1000 milisaniyede daha sonra çağırabilmesi için bu geri aramayı tutar, ancak zaman aşımını ayarladıktan sonra ve 1000 milisaniyeyi duraklatmadan önce yürütmeyi kaldığı yere geri verir, böylece devam eder. satır 4, sonra satır 11 ve ardından 1 saniye duraklar ve setTimeout daha sonra yürütmeyi geri çağıran geri çağırma işlevini çağırır ve burada ‘getAsyncMessages’ geri çağrısı, ‘message’ parametresi için ‘Merhaba Dünya’ değeriyle çağrılır ve daha sonra günlüğe kaydedilir 9. satırdaki konsola.

Node.js’deki geri arama işlevleri #

NodeJS, zaman uyumsuz geri aramalara sahiptir ve genellikle işlevlerinize bazen geleneksel olarak “err” ve “veri” olarak adlandırılan iki parametre sağlar. Bir dosya metnini okumakla ilgili bir örnek.

const fs = require("fs");

fs.readFile("./test.txt", "utf8", function(err, data) {
    if(err) {
        // handle the error 
    } else {
        // process the file text given with data
    }
});

Bu, tek seferlik olarak adlandırılan bir geri arama örneğidir.

Sadece günlüğe kaydetmiş veya atmış olsanız bile hatayı bir şekilde ele almak iyi bir uygulamadır. Eğer fırlatırsanız veya geri dönerseniz, else gerekli değildir ve eğer atma veya döndürme gibi bir şey yaparak mevcut işlevin yürütülmesini durdurduğunuz sürece girintiyi azaltmak için kaldırılabilir.

‘Err’, ‘veri’ görmek yaygın olsa da, geri aramalarınızın bu kalıbı kullanması her zaman böyle olmayabilir, belgelere bakmak en iyisidir.

Başka bir örnek geri arama ekspres kitaplığından gelir (express 4.x):

// this code snippet was on http://expressjs.com/en/4x/api.html
const express = require('express');
const app = express();

// this app.get method takes a url route to watch for and a callback
// to call whenever that route is requested by a user.
app.get('/', function(req, res){
  res.send('hello world');
});

app.listen(3000);

Bu örnek, birden çok kez çağrılan bir geri aramayı gösterir. Geri arama, burada paramlar olarak ‘req’ ve ‘res’ olarak adlandırılan iki nesne ile sağlanır, bu adlar sırasıyla request ve cevaba karşılık gelir ve gelen talebi görüntülemenin ve kullanıcıya gönderilecek yanıtı kurmanın yollarını sağlar. .

Gördüğünüz gibi, JavaScript’te eşitleme ve eşzamansız kod yürütmek için bir geri aramanın kullanılabilecek çeşitli yolları vardır ve geri aramalar JavaScript’in her yerinde çok yaygındır.

Geri arama cehennemi #

Geri arama cehennemi (aynı zamanda bir kıyamet veya bumerang etkisi piramidi), bir geri arama işlevinin içine çok fazla geri arama işlevi yerleştirdiğinizde ortaya çıkar. İşte bir dosyayı okumak için bir örnek (ES6’da).

const fs = require('fs');
let filename = `${__dirname}/myfile.txt`;

fs.exists(filename, exists => {
    if (exists) {
        fs.stat(filename, (err, stats) => {
            if (err) {
                throw err;
            }
            if (stats.isFile()) {
                fs.readFile(filename, null, (err, data) => {
                    if (err) {
                        throw err;
                    }
                    console.log(data);
                });
            }
            else {
                throw new Error("This location contains not a file");
            }
        });
    }
    else {
        throw new Error("404: file not found");
    }
});

“Geri Arama Cehennemi” nasıl önlenir

En fazla 2 geri arama işlevinin iç içe yerleştirilmesi önerilir. Bu, kodun okunabilirliğini korumanıza yardımcı olacak ve gelecekte bakımı çok daha kolay hale getirecek. 2’den fazla geri aramayı iç içe geçirmeniz gerekiyorsa, bunun yerine dağıtılmış olaylar kullanmayı deneyin.

Ayrıca, npm’de kullanılabilen geri aramaları ve bunların yürütülmesini yönetmeye yardımcı olan async adlı bir kitaplık da bulunmaktadır. Geri arama kodunun okunabilirliğini artırır ve paralel veya seri olarak çalıştırmanıza izin vermek de dahil olmak üzere geri arama kodu akışınız üzerinde size daha fazla kontrol sağlar.

Yerel Vaatler #

Sözler, zaman uyumsuz programlama için bir araçtır. JavaScript’te vaatler, “o zaman” yöntemleriyle bilinir. Sözlerin ‘beklemede’ ve ‘yerleşmiş’ olmak üzere iki ana durumu vardır. Bir söz ‘yerleştikten’ sonra ‘beklemede’ geri dönemez. Bu, vaatlerin çoğunlukla yalnızca bir kez gerçekleşen olaylar için iyi olduğu anlamına gelir. ‘Oturmuş’ durum, ‘çözümlenmiş’ ve ‘reddedilmiş’ olmak üzere iki duruma sahiptir. ’new’ anahtar sözcüğünü kullanarak ve ’new Promise(function (çözme, reddet) {})’ yapıcısına bir işlev ileterek yeni bir söz oluşturabilirsiniz.

Promise yapıcısına iletilen işlev, her zaman sırasıyla “çöz” ve “reddet” olarak adlandırılan birinci ve ikinci parametreyi alır. Bu iki parametrenin adlandırılması gelenekseldir, ancak vaadi ya ‘çözüldü’ ya da ‘reddedildi’ durumuna getireceklerdir. Bunlardan herhangi biri çağrıldığında, söz ‘beklemede’ olmaktan ‘yerleşmiş’ hale gelir. Genellikle eşzamansız olan istenen eylem gerçekleştirildiğinde “çözme” çağrılır ve eylemde hata varsa “reddetme” kullanılır.

Aşağıdaki zaman aşımında bir Promise döndüren bir işlev vardır.

function timeout (ms) {
  return new Promise(function (resolve, reject) {
    setTimeout(function () {
      resolve("It was resolved!");
    }, ms)
  });
}

timeout(1000).then(function (dataFromPromise) {
  // logs "It was resolved!"
  console.log(dataFromPromise);
})

console.log("waiting...");

konsol çıkışı

waiting...
// << pauses for one second>>
It was resolved!

Zaman aşımı çağrıldığında, Promise yapıcısına iletilen işlev gecikmeden yürütülür. Ardından setTimeout yöntemi yürütülür ve geri çağrısı sonraki “ms” milisaniyesinde, bu durumda “ms=1000"de tetiklenmeye ayarlanır. setTimeout’a yapılan geri arama henüz başlatılmadığından, zaman aşımı işlevi, kontrolü çağıran kapsama döndürür. “Sonra” yöntemleri zinciri daha sonra Söz çözüldüğünde/eğer daha sonra çağrılmak üzere saklanır. Burada ‘yakalama’ yöntemleri olsaydı, bunlar da saklanırdı, ancak söz ‘reddedildiğinde’/eğer kovulurdu.

Komut dosyası daha sonra ‘bekliyor…’ yazdırır. Bir saniye sonra setTimeout, “Çözüldü!” dizesiyle çözüm işlevini çağıran geri çağrısını çağırır. Bu dize daha sonra ’then’ yönteminin geri çağrısına iletilir ve ardından kullanıcıya kaydedilir.

Aynı anlamda, bir geri çağırma gerektiren asenkron setTimeout işlevini de sarabilirsiniz, herhangi bir tekil asenkron eylemi bir söz ile sarabilirsiniz.

Kod örneği #

Soru: Aşağıdaki kodun çıktısı nedir ve neden?

setTimeout(function() {
    console.log("A");
}, 1000);

setTimeout(function() {
    console.log("B");
}, 0);

getDataFromDatabase(function(err, data) {
    console.log("C");
    setTimeout(function() {
        console.log("D");
    }, 1000);
});

console.log("E");

Çıktı: Bu kesin olarak bilinmektedir: EBAD. “C"nin ne zaman kaydedileceği bilinmiyor.

Açıklama: Derleyici ‘setTimeout’ ve ‘getDataFromDatabase’ yöntemlerinde durmayacaktır. Yani giriş yapacağı ilk satır ‘E’. Geri çağırma işlevleri (setTimeoutun ilk argümanı) ayarlanan zaman aşımından sonra asenkron bir şekilde çalışacaktır!

Daha fazla detay:

  1. “E"de “setTimeout” yok
  2. “B”, 0 milisaniyelik bir zaman aşımına sahiptir
  3. “A”, 1000 milisaniyelik bir zaman aşımına sahiptir
  4. D bir veritabanı talep etmelidir, D 1000 milisaniye beklemelidir, böylece Adan sonra gelir.
  5. C bilinmiyor çünkü veritabanı verilerinin ne zaman istendiği bilinmiyor. “A"dan önce veya sonra olabilir.

Zaman uyumsuz hata işleme #

Yakalamayı dene #

Hatalar her zaman ele alınmalıdır. Eşzamanlı programlama kullanıyorsanız, bir “try catch” kullanabilirsiniz. Ancak asenkron çalışıyorsanız bu çalışmaz! Örnek:

try {
    setTimeout(function() {
        throw new Error("I'm an uncaught error and will stop the server!");
    }, 100); 
}
catch (ex) {
    console.error("This error will not be work in an asynchronous situation: " + ex);
}

Zaman uyumsuz hatalar yalnızca geri arama işlevi içinde ele alınacaktır!

process.on("UncaughtException", function(err, data) { 
    if (err) {
        // error handling
    }

Alan Adları #

Bir etki alanı içinde, hatalar olay yayıcılar aracılığıyla serbest bırakılır. Bunu kullanarak, tüm hatalar, zamanlayıcılar, geri arama yöntemleri örtük olarak yalnızca etki alanı içinde kayıtlıdır. Bir hatayla, bir hata olayı gönderilir ve uygulamayı çökmez.

var domain = require("domain");
var d1 = domain.create();
var d2 = domain.create();

d1.run(function() {
    d2.add(setTimeout(function() {
        throw new Error("error on the timer of domain 2");
    }, 0));
});

d1.on("error", function(err) {
    console.log("error at domain 1: " + err);
});

d2.on("error", function(err) {
    console.log("error at domain 2: " + err);
});
