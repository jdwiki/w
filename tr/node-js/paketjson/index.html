Kodlar #

Başka bir komut dosyasından önce veya sonra çalıştırılabilen veya tetiklenen komut dosyaları tanımlayabilirsiniz.

{
  "scripts": {
    "pretest": "scripts/pretest.js",
    "test": "scripts/test.js",
    "posttest": "scripts/posttest.js"
  }
}

Bu durumda, aşağıdaki komutlardan birini çalıştırarak komut dosyasını çalıştırabilirsiniz:

$ npm run-script test
$ npm run test
$ npm test
$ npm t

Kullanıcı tanımlı komut dosyaları #

Önceden tanımlanmış komut dosyalarında yaptığınız gibi kendi komut dosyalarınızı da tanımlayabilirsiniz:

{
  "scripts": {
    "preci": "scripts/preci.js",
    "ci": "scripts/ci.js",
    "postci": "scripts/postci.js"
  }
}

Bu durumda, aşağıdaki komutlardan birini çalıştırarak komut dosyasını çalıştırabilirsiniz:

$ npm run-script ci
$ npm run ci

Kullanıcı tanımlı komut dosyaları, yukarıdaki örnekte gösterildiği gibi pre ve post komut dosyalarını da destekler.

package.json’ı keşfetme #

Genellikle proje kökünde bulunan bir “package.json” dosyası, uygulamanız veya modülünüzle ilgili meta verilerin yanı sıra “npm install” çalıştırılırken npm’den yüklenecek bağımlılıkların listesini içerir.

Bir “package.json” başlatmak için komut isteminize “npm init” yazın.

Varsayılan değerlerle bir “package.json” oluşturmak için şunu kullanın:

npm init --yes
# or
npm init -y

Bir paketi kurmak ve onu package.json dosyasına kaydetmek için şunu kullanın:

npm install {package name} --save

Kısa gösterimi de kullanabilirsiniz:

 npm i -S {package name}

NPM, sırasıyla üretim veya geliştirme bağımlılıklarınızdan tasarruf etmek için “-S” ile “–save” ve “-D” ile “–save-dev” arasındaki diğer adları kullanır.

Paket, bağımlılıklarınızda görünecektir; Eğer kullanırsan ‘–save’ yerine ‘–save-dev’, paket devBağımlılıklarınızda görünecektir.

“package.json” un önemli özellikleri:

{
  "name": "module-name",
  "version": "10.3.1",
  "description": "An example module to illustrate the usage of a package.json",
  "author": "Your Name <your.name@example.org>",
  "contributors": [{
    "name": "Foo Bar",
    "email": "foo.bar@example.com"
  }],
  "bin": {
    "module-name": "./bin/module-name"
  },
  "scripts": {
    "test": "vows --spec --isolate",
    "start": "node index.js",
    "predeploy": "echo About to deploy",
    "postdeploy": "echo Deployed",
    "prepublish": "coffee --bare --compile --output lib/foo src/foo/*.coffee"
  },
  "main": "lib/foo.js",
  "repository": {
    "type": "git",
    "url": "https://github.com/username/repo"
  },
  "bugs": {
    "url": "https://github.com/username/issues"
  },
  "keywords": [
    "example"
  ],
  "dependencies": {
    "express": "4.2.x"
  },
  "devDependencies": {
    "assume": "<1.0.0 || >=2.3.1 <2.4.5 || >=2.5.2 <3.0.0"

Bazı önemli özellikler hakkında bilgi:

name

Paketinizin benzersiz adı ve küçük harfle yazılmalıdır. Bu özellik gereklidir ve paketiniz bu özellik olmadan kurulmaz.

  1. Ad, 214 karakterden küçük veya ona eşit olmalıdır.
  2. Ad nokta veya alt çizgi ile başlayamaz.
  3. Yeni paketlerin adında büyük harf olmamalıdır.
version

Paketin sürümü Semantik Sürüm Oluşturma (semver) tarafından belirlenir. Bu, bir sürüm numarasının MAJOR.MINOR.PATCH olarak yazıldığını varsayar ve şunları artırırsınız:

  1. Uyumsuz API değişiklikleri yaptığınızda MAJOR sürümü

  2. KÜÇÜK sürüm when you add functionality in a backwards-compatible manner

  3. YAMA version when you make backwards-compatible bug fixes

    description

Projenin açıklaması. Kısa ve öz tutmaya çalışın.

author

Bu paketin yazarı.

bin

Paketinizden ikili komut dosyalarını ortaya çıkarmak için kullanılan bir nesne. Nesne, anahtarın ikili komut dosyasının adı ve değerin komut dosyasına göreli bir yol olduğunu varsayar.

Bu özellik, bir CLI (komut satırı arayüzü) içeren paketler tarafından kullanılır.

script

Ek npm komutlarını ortaya çıkaran bir nesne. Nesne, anahtarın npm komutu olduğunu ve değerin de komut dosyası yolu olduğunu varsayar. Bu komut dosyaları, “npm run {command name}” veya “npm run-script {command name}” komutunu çalıştırdığınızda çalıştırılabilir.

Bir komut satırı arabirimi içeren ve yerel olarak kurulan paketler, göreli bir yol olmadan çağrılabilir. Böylece ./node-modules/.bin/mocha yerine doğrudan mochayı arayabilirsiniz.

main

Paketinizin ana giriş noktası. Düğümde require('{module name}') çağrılırken, bu gerekli olan gerçek dosya olacaktır.

Ana dosyaya ihtiyaç duymanın herhangi bir yan etki yaratmaması şiddetle tavsiye edilir. Örneğin, ana dosyanın istenmesi, bir HTTP sunucusunu başlatmamalı veya bir veritabanına bağlanmamalıdır. Bunun yerine, ana betiğinizde exports.init = function () {...} gibi bir şey oluşturmalısınız.

keywords

Paketinizi tanımlayan bir dizi anahtar kelime. Bunlar, insanların paketinizi bulmasına yardımcı olacaktır.

devDependencies

Bunlar, yalnızca modülünüzün geliştirilmesi ve test edilmesi için amaçlanan bağımlılıklardır. NODE_ENV=üretim ortam değişkeni ayarlanmadıkça, bağımlılıklar otomatik olarak kurulacaktır. Bu durumda, bu paketleri npm install --dev kullanarak yine de yapabilirsiniz.

peerDependencies

Bu modülü kullanıyorsanız, peerDependencies bunun yanında kurmanız gereken modülleri listeler. Örneğin, “moment-timezone”, “moment” ile birlikte kurulmalıdır, çünkü doğrudan “require(“moment”)” gerektirmese bile, bir an için bir eklentidir.

preferGlobal

Bu sayfanın global olarak npm install -g {module-name} kullanılarak yüklenmeyi tercih ettiğini belirten bir özellik. Bu özellik, bir CLI (komut satırı arayüzü) içeren paketler tarafından kullanılır.

Diğer tüm durumlarda bu özelliği KULLANMAMALISINIZ.

publishConfig

PublishConfig, modülleri yayınlamak için kullanılacak yapılandırma değerlerine sahip bir nesnedir. Ayarlanan yapılandırma değerleri, varsayılan npm yapılandırmanızı geçersiz kılar.

‘publishConfig’in en yaygın kullanımı, paketinizi özel bir npm kayıt defterinde yayınlamaktır, böylece npm’nin avantajlarına ancak özel paketler için sahip olmaya devam edersiniz. Bu, özel npm’nizin URL’sini kayıt defteri anahtarının değeri olarak ayarlayarak yapılır.

files

Bu, yayınlanan pakete dahil edilecek tüm dosyaların bir dizisidir. Bir dosya yolu veya klasör yolu kullanılabilir. Bir klasör yolunun tüm içeriği dahil edilecektir. Bu, yalnızca dağıtılacak doğru dosyaları dahil ederek paketinizin toplam boyutunu azaltır. Bu alan bir “.npmignore” kural dosyasıyla birlikte çalışır.

Bağımlılıklar #

“bağımlılıklar”: { “module-name”: “0.1.0” }

  • tam: “0.1.0”, modülün belirli bir sürümünü yükleyecektir.
  • en yeni alt sürüm: ^0.1.0, örneğin 0.2.0 gibi en yeni alt sürümü yükler, ancak daha yüksek bir ana sürüme sahip bir modül yüklemez örn. ‘1.0.0’
  • en yeni yama: ‘0.1.x’ veya ‘~0.1.0’, mevcut en yeni yama sürümünü yükler, örneğin ‘0.1.4’, ancak daha yüksek ana veya alt sürüme sahip bir modül yüklemez, Örneğin. “0.2.0” veya “1.0.0”.
  • joker karakter: * modülün en son sürümünü yükleyecektir.
  • git deposu: aşağıdaki, bir git deposunun ana dalından bir tarball yükleyecektir. Bir “#sha”, “#tag” veya “#branch” da sağlanabilir:
  • GitHub: “kullanıcı/proje” veya “kullanıcı/proje#v1.0.0”
  • url: “git://gitlab.com/user/project.git” veya “git://gitlab.com/user/project.git#develop”
  • yerel yol: dosya:../lib/project

Bunları package.json’ınıza ekledikten sonra, terminaldeki proje dizininizde npm install komutunu kullanın.

devBağımlılıkları #

"devDependencies": {
    "module-name": "0.1.0"
}

Yalnızca geliştirme için gerekli olan bağımlılıklar için, örneğin styling proxy’leri ext’yi test etmek gibi. Bu geliştirme bağımlılıkları, üretim modunda “npm install” çalıştırıldığında kurulmayacaktır.
