Express, web uygulamaları oluşturmak için sağlam bir dizi özellik sağlayan, minimal ve esnek bir Node.js web uygulaması çerçevesidir.

Express’in resmi web sitesi expressjs.com‘dir. Kaynak GitHub adresinde bulunabilir.

Sözdizimi #

  • app.get(yol [, ara katman yazılımı], geri arama[, geri arama…])
  • app.put(yol [, ara katman yazılımı], geri arama[, geri arama…])
  • app.post(yol [, ara katman], geri arama[, geri arama…])
  • app[‘delete’](yol [, ara katman yazılımı], geri arama[, geri arama…])
  • app.use(yol [, ara katman yazılımı], geri arama[, geri arama…])
  • app.use(geri arama)

Parametreler #

parametre Ayrıntılar
‘yol’ Verilen geri aramanın işleyeceği yol bölümünü veya URL’yi belirtir.
‘ara yazılım’ Geri aramadan önce çağrılacak bir veya daha fazla işlev. Esasen birden çok “geri arama” işlevinin zincirlenmesi. Yetkilendirme veya hata işleme gibi daha spesifik işlemler için kullanışlıdır.
“geri arama” Belirtilen “yol"a yönelik istekleri işlemek için kullanılacak bir işlev. ‘request’, ‘response’ ve ’next’ aşağıda açıklandığında, ‘callback(request, response, next)’ gibi çağrılacaktır.
geri arama “istek” Geri çağırmanın işlemek için çağrıldığı HTTP isteğiyle ilgili ayrıntıları kapsayan bir nesne.
‘yanıt’ Sunucunun isteğe nasıl yanıt vereceğini belirtmek için kullanılan bir nesne.
‘sonraki’ Kontrolü bir sonraki eşleşen rotaya aktaran bir geri arama. İsteğe bağlı bir hata nesnesini kabul eder.

Temel yönlendirme #

Önce bir ekspres uygulama oluşturun:

const express = require('express');
const app = express();

Ardından şu şekilde rotaları tanımlayabilirsiniz:

app.get('/someUri', function (req, res, next) {})

Bu yapı, tüm HTTP yöntemleri için çalışır ve ilk argüman olarak bir yol ve bu yol için istek ve yanıt nesnelerini alan bir işleyici bekler. Yani, temel HTTP yöntemleri için bunlar rotalardır.

// GET www.domain.com/myPath
app.get('/myPath', function (req, res, next) {})

// POST www.domain.com/myPath
app.post('/myPath', function (req, res, next) {})

// PUT www.domain.com/myPath
app.put('/myPath', function (req, res, next) {})

// DELETE www.domain.com/myPath
app.delete('/myPath', function (req, res, next) {})

Desteklenen fiillerin tam listesini buradan kontrol edebilirsiniz. Bir rota ve tüm HTTP yöntemleri için aynı davranışı tanımlamak istiyorsanız, şunları kullanabilirsiniz:

app.all('/myPath', function (req, res, next) {})

veya

app.use('/myPath', function (req, res, next) {})

veya

app.use('*', function (req, res, next) {})

// * wildcard will route for all paths

Rota tanımlarınızı tek bir yol için zincirleyebilirsiniz

app.route('/myPath')
  .get(function (req, res, next) {})
  .post(function (req, res, next) {})
  .put(function (req, res, next) {})

Ayrıca herhangi bir HTTP yöntemine işlevler ekleyebilirsiniz. Son geri aramadan önce çalışacaklar ve parametreleri (req, res, next) argüman olarak alacaklar.

// GET www.domain.com/myPath
app.get('/myPath', myFunction, function (req, res, next) {})

Bir dosyaya çok fazla kod koymaktan kaçınmak için son geri aramalarınız harici bir dosyada saklanabilir:

// other.js
exports.doSomething = function(req, res, next) {/* do some stuff */};

Ve sonra rotalarınızı içeren dosyada:

const other = require('./other.js');
app.get('/someUri', myFunction, other.doSomething);

Bu, kodunuzu çok daha temiz hale getirecektir.

Başlarken #

Önce bir dizin oluşturmanız, kabuğunuzdan erişmeniz ve npm install express --save komutunu çalıştırarak npm kullanarak Express’i kurmanız gerekir.

Bir dosya oluşturun ve onu ‘app.js’ olarak adlandırın ve yeni bir Express sunucusu oluşturan ve ‘app.get’ yöntemiyle ona bir uç nokta (’/ping’) ekleyen aşağıdaki kodu ekleyin:

const express = require('express');

const app = express();

app.get('/ping', (request, response) => {
    response.send('pong');
});

app.listen(8080, 'localhost');

Komut dosyanızı çalıştırmak için kabuğunuzda aşağıdaki komutu kullanın:

> node app.js

Uygulamanız, 8080 numaralı yerel ana bilgisayar bağlantı noktasındaki bağlantıları kabul edecektir. “app.listen” için ana bilgisayar adı bağımsız değişkeni atlanırsa, sunucu, yerel ana makinenin yanı sıra makinenin IP adresindeki bağlantıları da kabul edecektir. Bağlantı noktası değeri 0 ise, işletim sistemi kullanılabilir bir bağlantı noktası atayacaktır.

Komut dosyanız çalıştığında, sunucudan beklenen “pong” yanıtını aldığınızı doğrulamak için bir kabukta test edebilirsiniz:

> curl http://localhost:8080/ping
pong

Ayrıca bir web tarayıcısı açabilir, çıktıyı görüntülemek için http://localhost:8080/ping url’sine gidebilirsiniz.

Statik dosyalar sunma #

Express ile bir web sunucusu oluştururken, genellikle dinamik içerik ve statik dosyaların bir kombinasyonunu sunmak gerekir.

Örneğin, dosya sisteminde tutulan statik dosyalar olan index.html ve script.js’ye sahip olabilirsiniz.

Statik dosyalara sahip olmak için ‘public’ adlı klasörün kullanılması yaygındır. Bu durumda klasör yapısı şöyle görünebilir:

project root
├── server.js
├── package.json 
└── public
    ├── index.html
    └── script.js

Statik dosyalar sunmak için Express’i şu şekilde yapılandırabilirsiniz:

const express = require('express');
const app = express();

app.use(express.static('public'));

Not: Klasör yapılandırıldıktan sonra, index.html, script.js ve “public” klasöründeki tüm dosyalar kök yolunda kullanılabilir olacaktır (url’de /public/ belirtmemelisiniz). Bunun nedeni, express’in yapılandırılan statik klasöre göre dosyaları aramasıdır. sanal yol önekini aşağıda gösterildiği gibi belirtebilirsiniz:

app.use('/static', express.static('public'));

kaynakları /static/ öneki altında kullanılabilir hale getirecektir.

Birden çok klasör #

Aynı anda birden fazla klasör tanımlamak mümkündür:

app.use(express.static('public'));
app.use(express.static('images'));
app.use(express.static('files'));

Kaynakları sunarken Express, klasörü tanım sırasına göre inceler. Aynı ada sahip dosyalar olması durumunda, ilk eşleşen klasördeki dosya sunulacaktır.

Hata yönetimi #

Temel Hata İşleme

Varsayılan olarak, Express, oluşturmak için /views dizininde bir ‘hata’ görünümü arayacaktır. Basitçe ‘hata’ görünümünü oluşturun ve hataları işlemek için onu görünümler dizinine yerleştirin. Hatalar, hata mesajı, durum ve yığın izleme ile yazılır, örneğin:

görünümler/error.pug

html
  body
      h1= message
      h2= error.status
      p= error.stack

Gelişmiş Hata İşleme

Ara yazılım işlev yığınının en sonunda hata işleme ara yazılım işlevlerinizi tanımlayın. Bunların üç “(err, req, res, next)” yerine dört argümanı vardır, örneğin:

app.js

// catch 404 and forward to error handler
app.use(function(req, res, next) {
    var err = new Error('Not Found');
    err.status = 404;

    //pass error to the next matching route.
    next(err);
});

// handle error, print stacktrace
app.use(function(err, req, res, next) {
    res.status(err.status || 500);

    res.render('error', {
        message: err.message,
        error: err
    });
});

Normal ara katman işlevleriyle yaptığınız gibi, birkaç hata işleme ara katman işlevi tanımlayabilirsiniz.

Ara Katman Ekleme #

Ara yazılım işlevleri, uygulamanın istek-yanıt döngüsündeki istek nesnesine (req), yanıt nesnesine (res) ve sonraki ara katman yazılımı işlevine erişimi olan işlevlerdir.

Ara yazılım işlevleri herhangi bir kodu çalıştırabilir, “res” ve “req” nesnelerinde değişiklik yapabilir, yanıt döngüsünü sonlandırabilir ve sonraki ara katman yazılımını çağırabilir.

Ara katman yazılımının çok yaygın bir örneği ‘cors’ modülüdür. CORS desteği eklemek için basitçe kurun, talep edin ve şu satırı yazın:

app.use(cors());

herhangi bir yönlendirici veya yönlendirme işlevinden önce.

Kanca: Herhangi bir istekten önce ve herhangi bir çözümden sonra kod nasıl çalıştırılır #

app.use() ve ara katman yazılımı “before” için kullanılabilir ve “after” için close ve finish olaylarının bir kombinasyonu kullanılabilir.

app.use(function (req, res, next) {
    function afterResponse() {
        res.removeListener('finish', afterResponse);
        res.removeListener('close', afterResponse);

        // actions after response
    }
    res.on('finish', afterResponse);
    res.on('close', afterResponse);

    // action before request
    // eventually calling `next()`
    next();
});
...
app.use(app.router);

Buna bir örnek, varsayılan olarak yanıttan sonra günlüğe eklenecek olan logger ara yazılımıdır.

Sıra önemli olduğundan, bu “ara yazılımın” “app.router"dan önce kullanıldığından emin olun.

Ara yazılımı ve sonraki geri aramayı kullanma #

Express, birden çok işleyicide tek bir yol için mantığı kırmak için kullanılabilecek her yol işleyicisine ve ara katman yazılımı işlevine bir “sonraki” geri çağrı gönderir. Argümansız “next()” çağrısı, express’e bir sonraki eşleşen ara katman yazılımına veya rota işleyicisine devam etmesini söyler. Bir hatayla “next(err)” çağrısı yapmak, herhangi bir hata işleyici ara yazılımını tetikleyecektir. next('route') çağrısı, mevcut rotadaki sonraki ara katman yazılımlarını atlayacak ve bir sonraki eşleşen rotaya atlayacaktır. Bu, etki alanı mantığının bağımsız, test edilmesi daha basit ve bakımı ve değiştirilmesi daha kolay olan yeniden kullanılabilir bileşenlere ayrıştırılmasına olanak tanır.

Birden fazla eşleşen rota

/api/foo’ya veya ‘/api/bar’a yapılan istekler, üyeyi aramak için ilk işleyiciyi çalıştıracak ve ardından kontrolü her bir yol için gerçek işleyiciye iletecektir.

app.get('/api', function(req, res, next) {
  // Both /api/foo and /api/bar will run this
  lookupMember(function(err, member) {
    if (err) return next(err);
    req.member = member;
    next();
  });
});

app.get('/api/foo', function(req, res, next) {
  // Only /api/foo will run this
  doSomethingWithMember(req.member);
});

app.get('/api/bar', function(req, res, next) {
  // Only /api/bar will run this
  doSomethingDifferentWithMember(req.member);
});

Hata işleyici

Hata işleyiciler, “function(err, req, res, next)” imzasına sahip ara katman yazılımıdır. Bunlar rota başına ayarlanabilir (ör. app.get('/foo', function(err, req, res, next)), ancak genellikle bir hata sayfası oluşturan tek bir hata işleyicisi yeterlidir.

app.get('/foo', function(req, res, next) {
  doSomethingAsync(function(err, data) {
    if (err) return next(err);
    renderPage(data);
  });
});

// In the case that doSomethingAsync return an error, this special
// error handler middleware will be called with the error as the 
// first parameter.
app.use(function(err, req, res, next) {
  renderErrorPage(err);
});

Ara katman

Yukarıdaki işlevlerin her biri aslında bir istek tanımlanan rotayla eşleştiğinde çalıştırılan bir ara katman işlevidir, ancak herhangi bir sayıda ara katman işlevi tek bir rota üzerinde tanımlanabilir. Bu, ara katman yazılımının ayrı dosyalarda tanımlanmasına ve ortak mantığın birden çok rotada yeniden kullanılmasına olanak tanır.

app.get('/bananas', function(req, res, next) {
  getMember(function(err, member) {
    if (err) return next(err);
    // If there's no member, don't try to look
    // up data. Just go render the page now.
    if (!member) return next('route');
    // Otherwise, call the next middleware and fetch
    // the member's data.
    req.member = member;
    next();
  });
}, function(req, res, next) {
  getMemberData(req.member, function(err, data) {
    if (err) return next(err);
    // If this member has no data, don't bother
    // parsing it. Just go render the page now.
    if (!data) return next('route');
    // Otherwise, call the next middleware and parse
    // the member's data. THEN render the page.
    req.member.data = data;
    next();
  });
}, function(req, res, next) {
  req.member.parsedData = parseMemberData(req.member.data);
  next();
});

app.get('/bananas', function(req, res, next) {
  renderBananas(req.member);
});

Bu örnekte, her bir ara katman işlevi ya kendi dosyasında ya da dosyanın başka bir yerindeki bir değişkende olacak ve böylece başka rotalarda yeniden kullanılabilecektir.

POST İsteklerini İşleme #

App.get yöntemiyle Express’te alma isteklerini ele aldığınız gibi, gönderi isteklerini işlemek için app.post yöntemini kullanabilirsiniz.

Ancak POST isteklerini işlemeden önce, ‘body-parser’ ara katman yazılımını kullanmanız gerekecek. Sadece “POST”, “PUT”, “DELETE” ve diğer isteklerin gövdesini ayrıştırır.

“Body-Parser” ara yazılımı, isteğin gövdesini ayrıştırır ve onu “req.body” içinde bulunan bir nesneye dönüştürür.

var bodyParser = require('body-parser');

const express = require('express');

const app = express();

// Parses the body for POST, PUT, DELETE, etc.
app.use(bodyParser.json());

app.use(bodyParser.urlencoded({ extended: true }));

app.post('/post-data-here', function(req, res, next){

    console.log(req.body); // req.body contains the parsed body of the request.

});

app.listen(8080, 'localhost');
