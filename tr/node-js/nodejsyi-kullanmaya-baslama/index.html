Merhaba Dünya HTTP sunucusu #

İlk olarak, platformunuz için Node.js’yi yükleyin.

Bu örnekte, tarayıcıya Merhaba Dünya! gönderen 1337 numaralı bağlantı noktasını dinleyen bir HTTP sunucusu oluşturacağız. 1337 numaralı bağlantı noktasını kullanmak yerine, şu anda başka bir hizmet tarafından kullanılmayan istediğiniz herhangi bir bağlantı noktası numarasını kullanabileceğinizi unutmayın.

“http” modülü bir Node.js’dir [core module](https://www.wikiod.com/tr/node-js/nodejsyi-kullanmaya-baslama#Çekirdek modüller ) (Ek kaynakların yüklenmesini gerektirmeyen, Node.js’nin kaynağında bulunan bir modül). http modülü, http.createServer() yöntemini kullanarak bir HTTP sunucusu oluşturma işlevselliği sağlar. To create the application, create a file containing the JavaScript kodunu izleyerek.

const http = require('http'); // Loads the http module

http.createServer((request, response) => {

    // 1. Tell the browser everything is OK (Status code 200), and the data is in plain text
    response.writeHead(200, {
        'Content-Type': 'text/plain'
    });

    // 2. Write the announced text to the body of the page
    response.write('Hello, World!\n');

    // 3. Tell the server that all of the response headers and body have been sent
    response.end();

}).listen(1337); // 4. Tells the server what port to be on

Dosyayı herhangi bir dosya adıyla kaydedin. Bu durumda merhaba.js adını verirsek, dosyanın bulunduğu dizine gidip aşağıdaki komutu kullanarak uygulamayı çalıştırabiliriz:

node hello.js

Oluşturulan sunucuya daha sonra http://localhost:1337 URL’si ile erişilebilir. veya tarayıcıda http://127.0.0.1:1337.

“Merhaba, Dünya!” ile basit bir web sayfası görünecektir. metin, aşağıdaki ekran görüntüsünde gösterildiği gibi.

Merhaba Dünya komut satırı #

Node.js ayrıca komut satırı yardımcı programları oluşturmak için de kullanılabilir. Aşağıdaki örnek, komut satırından ilk argümanı okur ve bir Merhaba mesajı yazdırır.

Bu kodu bir Unix Sisteminde çalıştırmak için:

  1. Yeni bir dosya oluşturun ve aşağıdaki kodu yapıştırın. Dosya adı alakasız.
  2. Bu dosyayı chmod 700 FILE_NAME ile çalıştırılabilir yapın
  3. Uygulamayı ./APP_NAME David ile çalıştırın

Windows’ta 1. adımı yapın ve düğüm APP_NAME David ile çalıştırın

#!/usr/bin/env node

'use strict';

/*
    The command line arguments are stored in the `process.argv` array, 
    which has the following structure:
    [0] The path of the executable that started the Node.js process
    [1] The path to this application
    [2-n] the command line arguments

    Example: [ '/bin/node', '/path/to/yourscript', 'arg1', 'arg2', ... ]
    src: https://nodejs.org/api/process.html#process_process_argv
 */

// Store the first argument as username.
var username = process.argv[2];

// Check if the username hasn't been provided.
if (!username) {

    // Extract the filename
    var appName = process.argv[1].split(require('path').sep).pop();

    //  Give the user an example on how to use the app.
    console.error('Missing argument! Example: %s YOUR_NAME', appName);

    // Exit the app (success: 0, error: 1). 
    // An error will stop the execution chain. For example:
    //   ./app.js && ls       -> won't execute ls
    //   ./app.js David && ls -> will execute ls
    process.exit(1);
}

// Print the message to the console.
console.log('Hello %s!', username);

Express ile Merhaba Dünya #

Aşağıdaki örnek, 3000 numaralı bağlantı noktasını dinleyen ve “Hello, World!” ile yanıt veren bir HTTP sunucusu oluşturmak için Express’i kullanır. Express, HTTP API’leri oluşturmak için yararlı olan, yaygın olarak kullanılan bir web çerçevesidir.

İlk önce yeni bir klasör oluşturun, ör. ‘uygulamam’. Uygulamam’a gidin ve aşağıdaki kodu içeren yeni bir JavaScript dosyası oluşturun (örneğin, onu ‘hello.js’ olarak adlandıralım). Ardından komut satırından npm install --save express komutunu kullanarak ekspres modülü kurun. Paketlerin nasıl kurulacağı hakkında daha fazla bilgi için bu belgelere bakın.

// Import the top-level function of express
const express = require('express');

// Creates an Express application using the top-level function
const app = express();

// Define port number as 3000
const port = 3000;

// Routes HTTP GET requests to the specified path "/" with the specified callback function
app.get('/', function(request, response) {
  response.send('Hello, World!');
});

// Make the app listen on port 3000
app.listen(port, function() {
  console.log('Server listening on http://localhost:' + port);
});

Komut satırından aşağıdaki komutu çalıştırın:

node hello.js

Tarayıcınızı açın ve yanıtı görmek için “http://localhost:3000” veya “http://127.0.0.1:3000” adresine gidin.

Dosya ayrıca yürütülebilir olarak ayarlanmalıdır, bu ‘chmod’ kullanılarak yapılabilir. Artık komut dosyası doğrudan komut satırından çalıştırılabilir.

Merhaba Dünya temel yönlendirme #

Düğümlü bir HTTP Sunucusu nasıl oluşturulacağını anladıktan sonra, bir kullanıcının gittiği yola dayalı olarak onun işleri nasıl “yapacağını” anlamak önemlidir. Bu fenomene “yönlendirme” denir.

Bunun en temel örneği, if (request.url === 'some/path/here') öğesini kontrol etmek ve ardından yeni bir dosyayla yanıt veren bir işlevi çağırmak olacaktır.

Bunun bir örneği burada görülebilir:

const http = require('http');

function index (request, response) {
    response.writeHead(200);
    response.end('Hello, World!');
}

http.createServer(function (request, response) {
    
    if (request.url === '/') {
        return index(request, response);
    }

    response.writeHead(404);
    response.end(http.STATUS_CODES[404]);

}).listen(1337);

Yine de “rotalarınızı” bu şekilde tanımlamaya devam ederseniz, büyük bir geri arama işleviyle karşılaşacaksınız ve böyle devasa bir karmaşa istemiyoruz, bakalım bunu temizleyebilir miyiz.

İlk olarak, tüm rotalarımızı bir nesnede saklayalım:

var routes = {
    '/': function index (request, response) {
        response.writeHead(200);
        response.end('Hello, World!');
    },
    '/foo': function foo (request, response) {
        response.writeHead(200);
        response.end('You are now viewing "foo"');
    }
}

Artık bir nesnede 2 rota depoladığımıza göre, artık onları ana geri aramamızda kontrol edebiliriz:

http.createServer(function (request, response) {
    
    if (request.url in routes) {
        return routes[request.url](request, response);
    }

    response.writeHead(404);
    response.end(http.STATUS_CODES[404]);

}).listen(1337);

Artık web sitenizde gezinmeyi her denediğinizde, rotalarınızda o yolun varlığını kontrol edecek ve ilgili işlevi çağıracaktır. Rota bulunamazsa, sunucu bir 404 (Bulunamadı) ile yanıt verecektir.

Ve işte karşınızda – HTTP Sunucusu API’si ile yönlendirme çok basittir.

Uygulamanızı çevrimiçi olarak dağıtma #

Uygulamanızı (Node.js’ye özgü) barındırılan bir ortama dağıttığınızda, bu ortam genellikle sunucunuzu çalıştırmak için kullanabileceğiniz bir “PORT” ortamı değişkeni sunar. Port numarasını process.env.PORT olarak değiştirmek uygulamaya erişmenizi sağlar.

Örneğin,

http.createServer(function(request, response) {
   // your server code
}).listen(process.env.PORT);

Ayrıca, hata ayıklama sırasında buna çevrimdışı erişmek isterseniz, şunu kullanabilirsiniz:

http.createServer(function(request, response) {
  // your server code
}).listen(process.env.PORT || 3000);

burada “3000” çevrimdışı bağlantı noktası numarasıdır.

TLS Soketi: sunucu ve istemci #

Bu ve normal bir TCP bağlantısı arasındaki tek büyük fark, özel Anahtar ve bir seçenek nesnesine ayarlamanız gereken genel sertifikadır.

Anahtar ve Sertifika Nasıl Oluşturulur #

Bu güvenlik sürecindeki ilk adım, özel bir Anahtarın oluşturulmasıdır. Ve bu özel anahtar nedir? Temel olarak, bilgileri şifrelemek için kullanılan bir dizi rastgele gürültüdür. Teoride, bir anahtar oluşturabilir ve onu istediğiniz her şeyi şifrelemek için kullanabilirsiniz. Ancak belirli şeyler için farklı anahtarlara sahip olmak en iyi uygulamadır. Çünkü birisi özel anahtarınızı çalarsa, birinin ev anahtarlarınızı çalmasına benzer. Arabanızı, garajınızı, ofisinizi vb. kilitlemek için aynı anahtarı kullandığınızı hayal edin.

openssl genrsa -out private-key.pem 1024

Özel anahtarımızı aldıktan sonra, özel anahtarın süslü bir otorite tarafından imzalanmasını talep eden bir CSR (sertifika imzalama talebi) oluşturabiliriz. Bu nedenle firmanız ile ilgili bilgileri girmeniz gerekmektedir. Bu bilgiler imza yetkilisi tarafından görülecek ve sizi doğrulamak için kullanılacaktır. Bizim durumumuzda, ne yazdığınızın bir önemi yok, çünkü bir sonraki adımda sertifikamızı kendimiz imzalayacağız.

openssl req -new -key private-key.pem -out csr.pem

Artık evrak işlerimizi doldurduğumuza göre, havalı bir imza yetkilisiymişiz gibi davranmanın zamanı geldi.

openssl x509 -req -in csr.pem -signkey private-key.pem -out public-cert.pem

Artık özel anahtara ve genel sertifikaya sahip olduğunuza göre, iki NodeJS uygulaması arasında güvenli bir bağlantı kurabilirsiniz. Ve örnek kodda da görebileceğiniz gibi, çok basit bir işlemdir.

Önemli! #

Kamusal sertifikayı kendimiz yarattığımız için, dürüst olmak gerekirse, sertifikamız değersizdir, çünkü biz birer hiç kimseyiz. NodeJS sunucusu varsayılan olarak böyle bir sertifikaya güvenmez ve bu nedenle, aşağıdaki seçeneği reddetme Yetkisiz: false ile sertifikamıza gerçekten güvenmesini söylememiz gerekir. Çok önemli: bir üretim ortamında bu değişkeni asla true olarak ayarlamayın.

TLS Soket Sunucusu #

'use strict';

var tls = require('tls');
var fs = require('fs');

const PORT = 1337;
const HOST = '127.0.0.1'

var options = {
    key: fs.readFileSync('private-key.pem'),
    cert: fs.readFileSync('public-cert.pem')
};

var server = tls.createServer(options, function(socket) {

    // Send a friendly message
    socket.write("I am the server sending you a message.");

    // Print the data that we received
    socket.on('data', function(data) {

        console.log('Received: %s [it is %d bytes long]',
            data.toString().replace(/(\n)/gm,""),
            data.length);

    });

    // Let us know when the transmission is over
    socket.on('end', function() {

        console.log('EOT (End Of Transmission)');

    });

});

// Start listening on a specific port and address
server.listen(PORT, HOST, function() {

    console.log("I'm listening at %s, on port %s", HOST, PORT);

});

// When an error occurs, show it.
server.on('error', function(error) {

    console.error(error);

    // Close the connection after the error occurred.
    server.destroy();

});

TLS Soket İstemcisi #

'use strict';

var tls = require('tls');
var fs = require('fs');

const PORT = 1337;
const HOST = '127.0.0.1'

// Pass the certs to the server and let it know to process even unauthorized certs.
var options = {
    key: fs.readFileSync('private-key.pem'),
    cert: fs.readFileSync('public-cert.pem'),
    rejectUnauthorized: false
};

var client = tls.connect(PORT, HOST, options, function() {

    // Check if the authorization worked
    if (client.authorized) {
        console.log("Connection authorized by a Certificate Authority.");
    } else {
        console.log("Connection not authorized: " + client.authorizationError)
    }

    // Send a friendly message
    client.write("I am the client sending you a message.");

});

client.on("data", function(data) {

    console.log('Received: %s [it is %d bytes long]',
        data.toString().replace(/(\n)/gm,""),
        data.length);

    // Close the connection after receiving the message
    client.end();

});

client.on('close', function() {

    console.log("Connection closed");

});

// When an error ocoures, show it.
client.on('error', function(error) {

    console.error(error);

    // Close the connection after the error occurred.
    client.destroy();

});

Temel bir HTTPS web sunucusu nasıl kurulur ve çalıştırılır! #

Node.js’yi sisteminize yükledikten sonra, hem HTTP hem de HTTPS desteğiyle çalışan temel bir web sunucusu elde etmek için aşağıdaki prosedürü izlemeniz yeterlidir!

Adım 1: Bir Sertifika Yetkilisi Oluşturun #

  1. Anahtarınızı ve sertifikanızı saklamak istediğiniz klasörü oluşturun:

    mkdir conf

  1. o dizine gidin:

    cd conf

  1. Yapılandırma kısayolu olarak kullanmak için bu “ca.cnf” dosyasını alın:

    wget https://raw.githubusercontent.com/anders94/https-authorized-clients/master/keys/ca.cnf

  1. Bu yapılandırmayı kullanarak yeni bir sertifika yetkilisi oluşturun:

    openssl req -new -x509 -days 9999 -config ca.cnf -keyout ca-key.pem -out ca-cert.pem

  1. Artık ca-key.pem ve ca-cert.pem içinde sertifika yetkimiz olduğuna göre, sunucu için özel bir anahtar üretelim:

    openssl genrsa -out key.pem 4096

  1. Yapılandırma kısayolu olarak kullanmak için bu server.cnf dosyasını alın:

    wget https://raw.githubusercontent.com/anders94/https-authorized-clients/master/keys/server.cnf

  1. Bu yapılandırmayı kullanarak sertifika imzalama isteğini oluşturun:

    openssl req -new -config server.cnf -key key.pem -out csr.pem

  1. isteği imzalayın:

    openssl x509 -req -extfile server.cnf -days 999 -passin "pass:password" -in csr.pem -CA ca-cert.pem -CAkey ca-key.pem -CAcreateserial -out cert.pem

2. Adım : Sertifikanızı kök sertifika olarak yükleyin #

  1. sertifikanızı kök sertifikalarınızın klasörüne kopyalayın:

    sudo cp ca-crt.pem /usr/local/share/ca-certificates/ca-crt.pem

  1. CA mağazasını güncelleyin:

    sudo update-ca-certificates

3. Adım : Düğüm sunucunuzu başlatma #

İlk olarak, gerçek sunucu kodunuzu içeren bir “server.js” dosyası oluşturmak istiyorsunuz.

Node.js’deki bir HTTPS sunucusunun minimum kurulumu şuna benzer:

var https = require('https');
var fs = require('fs');

var httpsOptions = {
    key: fs.readFileSync('path/to/server-key.pem'),
    cert: fs.readFileSync('path/to/server-crt.pem')
};

var app = function (req, res) {
  res.writeHead(200);
  res.end("hello world\n");
}

https.createServer(httpsOptions, app).listen(4433);

Ayrıca http isteklerini desteklemek istiyorsanız, sadece şu küçük değişikliği yapmanız gerekir:

var http = require('http');
var https = require('https');
var fs = require('fs');

var httpsOptions = {
    key: fs.readFileSync('path/to/server-key.pem'),
    cert: fs.readFileSync('path/to/server-crt.pem')
};

var app = function (req, res) {
  res.writeHead(200);
  res.end("hello world\n");
}

http.createServer(app).listen(8888);
https.createServer(httpsOptions, app).listen(4433);

  1. server.js dosyanızın bulunduğu dizine gidin:

    cd /path/to

  1. “server.js"yi çalıştırın:

    node server.js
