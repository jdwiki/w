ES6 ile Node.JS

  • Düğüm ES6 Destek ve Babel ile proje oluşturma

Bu sayfada

  • Düğüm ES6 Destek ve Babel ile proje oluşturma
  • NodeJS uygulamanızda JS es6 kullanın

ES6, ECMAScript 6 veya ES2015, JavaScript için dile biraz sözdizimsel şeker getiren en son spesifikasyon‘dir. Bu, dilde yapılan büyük bir güncellemedir ve birçok yeni özellik sunar.

Düğüm ve ES6 hakkında daha fazla ayrıntı kendi sitelerinde bulunabilir https://nodejs.org/en/docs/es6/

Düğüm ES6 Destek ve Babel ile proje oluşturma #

ES6 spesifikasyonunun tamamı henüz bütünüyle uygulanmadı, bu nedenle yalnızca bazı yeni özellikleri kullanabileceksiniz. Şu anda desteklenen ES6 özelliklerinin bir listesini http://node.green/ adresinde görebilirsiniz.

NodeJS v6’dan beri oldukça iyi bir destek var. Yani NodeJS v6 veya üstünü kullanıyorsanız ES6’yı kullanmanın keyfini çıkarabilirsiniz. Ancak, bazı yayınlanmamış özellikleri ve bazılarını da ötesinde kullanmak isteyebilirsiniz. Bunun için bir aktarıcı kullanmanız gerekecek

Tüm ES6 özelliklerini ve daha fazlasını kullanmak için çalışma zamanında ve derlemede bir aktarıcı çalıştırmak mümkündür. JavaScript için en popüler aktarıcının adı Babel

Babel, ES6 spesifikasyonundaki tüm özellikleri ve ‘var şey = require(’thing’)’ yerine ‘şeyden bir şeyi ‘şeyden içe aktar’ gibi ‘stage-0’ ile bazı ek spesifikasyon dışı özellikleri kullanmanıza izin verir.

Import gibi ‘stage-0’ özelliklerini kullandığımız bir proje yaratmak isteseydik Babel’i aktarıcı olarak eklememiz gerekecekti. Tepki kullanan projelerin ve Vue ve diğer ortak JS tabanlı kalıpların oldukça sık aşama-0 uyguladığını göreceksiniz.

yeni bir düğüm projesi oluştur

mkdir my-es6-app
cd my-es6-app
npm init

babil’i ES6 ön ayarını ve aşama-0’ı yükleyin

npm install --save-dev babel-preset-es2015 babel-preset-stage-2 babel-cli babel-register

Create a new file called server.js and add a basic HTTP server.

import http from 'http'

http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'})
  res.end('Hello World\n')
}).listen(3000, '127.0.0.1')

console.log('Server running at http://127.0.0.1:3000/')

Note that we use an import http from 'http' this is a stage-0 feature and if it works it means we’ve got the transpiler working correctly.

If you run node server.js it will fail not knowing how to handle the import.

Creating a .babelrc file in the root of your directory and add the following settings

{
  "presets": ["es2015", "stage-2"],
  "plugins": []
}

you can now run the server with node src/index.js --exec babel-node

Finishing off it is not a good idea to run a transpiler at runtime on a production app. We can however implement some scripts in our package.json to make it easier to work with.

"scripts": {
    "start": "node dist/index.js",
    "dev": "babel-node src/index.js",
    "build": "babel src -d dist",
    "postinstall": "npm run build"
  },

The above will on npm install build the transpiled code to the dist directory allow npm start to use the transpiled code for our production app.

npm run dev will boot the server and babel runtime which is fine and preferred when working on a project locally.

Going one further you could then install nodemon npm install nodemon --save-dev to watch for changes and then reboot the node app.

This really speeds up working with babel and NodeJS. In you package.json just update the “dev” script to use nodemon

"dev": "nodemon src/index.js --exec babel-node",

NodeJS uygulamanızda JS es6 kullanın #

JS es6 (es2015 olarak da bilinir), OOP kullanırken veya modern geliştirme görevleriyle karşı karşıya kalırken onu daha sezgisel hale getirmeyi amaçlayan JS diline yönelik bir dizi yeni özelliktir.

Önkoşullar: #

  1. http://es6-features.org adresindeki yeni es6 özelliklerine göz atın - bir sonraki NodeJS uygulamanızda gerçekten kullanmayı düşünüyorsanız, bu size netlik kazandırabilir.

  2. http://node.green adresinden düğüm sürümünüzün uyumluluk düzeyini kontrol edin.

  3. Her şey yolundaysa - hadi kod yazalım!

İşte JS es6 ile basit bir “merhaba dünya” uygulamasının çok kısa bir örneği

'use strict'

class Program
{
    constructor()
    {
        this.message = 'hello es6 :)';
    }

    print()
    {
        setTimeout(() =>
        {
            console.log(this.message);
            
            this.print();

        }, Math.random() * 1000);
    }
}

new Program().print();

Bu programı çalıştırabilir ve aynı mesajı tekrar tekrar nasıl yazdırdığını gözlemleyebilirsiniz.

Şimdi.. satır satır ayıralım:

'use strict'

Bu satır aslında js es6 kullanmayı düşünüyorsanız gereklidir. “katı” modun kasıtlı olarak normal koddan farklı bir anlamı vardır (lütfen bu konuda daha fazla bilgiyi MDN’de okuyun - https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Strict_mode)

class Program

İnanılmaz - bir “sınıf” anahtar kelimesi! Sadece hızlı bir referans için - es6’dan önce js’de bir sınıf tanımlamanın tek yolu… function anahtar kelimesiydi!

function MyClass() // class definition
{

}

var myClassObject = new MyClass(); // generating a new object with a type of MyClass

OOP kullanırken, bir sınıf, geliştiricinin bir sistemin belirli bir bölümünü temsil etmesine yardımcı olan çok temel bir yetenektir (kod büyüdüğünde kodu parçalamak çok önemlidir.. örneğin: sunucu tarafı kodu yazarken)

constructor()
{
    this.message = 'hello es6 :)';
}

Kabul etmelisiniz - bu oldukça sezgisel! Bu, sınıfımın c’tor’udur - bu benzersiz “işlev”, bu belirli sınıftan her nesne oluşturulduğunda ortaya çıkacaktır (programımızda - yalnızca bir kez)

print()
{
    setTimeout(() => // this is an 'arrow' function
    {
        console.log(this.message);
        
        this.print(); // here we call the 'print' method from the class template itself (a recursion in this particular case)

    }, Math.random() * 1000);
}

Print sınıf kapsamında tanımlandığı için - bu aslında bir yöntemdir - sınıfın nesnesinden veya sınıfın kendisinden çağrılabilir!

Yani.. şimdiye kadar sınıfımızı tanımladık.. onu kullanma zamanı:

new Program().print();

Hangisi gerçekten eşittir:

var prog = new Program(); // define a new object of type 'Program'

prog.print(); // use the program to print itself

Sonuç olarak: JS es6 kodunuzu basitleştirebilir - daha sezgisel ve anlaşılır hale getirebilir (önceki JS sürümüyle karşılaştırıldığında).. mevcut bir kodunuzu yeniden yazmayı deneyebilir ve aradaki farkı görebilirsiniz. kendin
