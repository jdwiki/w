node.js’de basit bir modül kullanma #

node.js modülü nedir (makaleye bağlantı):

Bir modül, ilgili kodu tek bir kod biriminde kapsüller. Bir modül oluştururken bu, ilgili tüm fonksiyonları bir dosyaya taşımak olarak yorumlanabilir.

Şimdi bir örnek görelim. Tüm dosyaların aynı dizinde olduğunu hayal edin:

Dosya: “printer.js”

"use strict";

exports.printHelloWorld = function (){
    console.log("Hello World!!!");
}

Modülleri kullanmanın başka bir yolu:

“animals.js” dosyası

"use strict";

module.exports = {
    lion: function() {
        console.log("ROAARR!!!");
    }

};

Dosya: “app.js”

Dizininize gidip şunu yazarak bu dosyayı çalıştırın: node app.js

"use strict";

//require('./path/to/module.js') node which module to load
var printer = require('./printer');
var animals = require('./animals');

printer.printHelloWorld(); //prints "Hello World!!!"
animals.lion(); //prints "ROAARR!!!"

ES6 sözdizimi ile dışa aktarma #

Bu, [diğer örneğin](https://www.wikiod.com/tr/node-js/nodejsde-modulu-dsa-ve-ice-aktarma#node.js’de basit bir modül kullanma) eşdeğeridir -node-js) ama bunun yerine ES6 kullanıyor.

export function printHelloWorld() {
  console.log("Hello World!!!");
}

ES6’da İçe Aktarmayı Kullanma #

Node.js, V8’in modern sürümlerine karşı oluşturulmuştur. Bu motorun en son sürümlerini güncel tutarak, JavaScript ECMA-262 spesifikasyonundaki yeni özelliklerin Node.js geliştiricilerine zamanında sunulmasını ve ayrıca sürekli performans ve kararlılık iyileştirmelerini sağlıyoruz.

Tüm ECMAScript 2015 (ES6) özellikleri, gönderim, aşamalı ve devam eden özellikler için üç gruba ayrılır:

V8’in kararlı olarak kabul ettiği tüm gönderim özellikleri, Node.js’de varsayılan olarak açıktır ve herhangi bir çalışma zamanı bayrağı GEREKTİRMEZ. V8 ekibi tarafından kararlı kabul edilmeyen, neredeyse tamamlanmış özellikler olan aşamalı özellikler, bir çalışma zamanı bayrağı gerektirir: –harmony. Devam eden özellikler, ilgili uyum bayrağıyla ayrı ayrı etkinleştirilebilir, ancak bu, test amaçlı olmadığı sürece kesinlikle önerilmez. Not: Bu bayraklar V8 tarafından kullanıma sunulur ve herhangi bir kullanımdan kaldırma bildirimi olmaksızın potansiyel olarak değişebilir.

Şu anda ES6, içe aktarma ifadelerini yerel olarak desteklemektedir Buraya bakın

Yani fun.js adında bir dosyamız varsa…

export default function say(what){
  console.log(what);
}

export function sayLoud(whoot) {
  say(whoot.toUpperCase());
}

…ve daha önce tanımladığımız işlevleri kullanmak istediğimiz app.js adında başka bir dosya varsa, bunları içe aktarmanın üç yolu vardır.

Varsayılanı içe aktar

import say from './fun';
say('Hello Stack Overflow!!');  // Output: Hello Stack Overflow!!

Kaynak dosyada varsayılan dışa aktarma olarak işaretlendiğinden say() işlevini içe aktarır (export default …)

Adlandırılmış ithalat

import { sayLoud } from './fun';
sayLoud('JS modules are awesome.'); // Output: JS MODULES ARE AWESOME.

Adlandırılmış ithalatlar, tam olarak ihtiyacımız olan bir modülün parçalarını içe aktarmamızı sağlar. Bunu açıkça isimlendirerek yapıyoruz. Bizim durumumuzda, import deyiminde ‘sayLoud’u süslü parantez içinde adlandırarak.

Birlikte içe aktarma

import * as i from './fun';
i.say('What?'); // Output: What?
i.sayLoud('Whoot!'); // Output: WHOOT!

Her şeye sahip olmak istiyorsak, gidilecek yol budur. * i sözdizimini kullanarak, import ifadesine sahibiz, bize eğlence modülümüzün tüm ihracatlarını uygun şekilde adlandırılmış özellikler olarak tutan bir i nesnesi sağlarız.

Yollar

İçe aktarılacak dosya ./ kullanarak içe aktardığınız dosyayla aynı dizinde bulunsa bile, içe aktarma yollarınızı göreli yollar olarak açıkça işaretlemeniz gerektiğini unutmayın. gibi öneksiz yollardan içe aktarma

import express from 'express';

yerel ve global ’node_modules’ klasörlerinde aranacak ve eşleşen modül bulunmazsa hata verecektir.
