Daha fazla okuma:

Installing grunt guide, Grunt ve grunt-cli’nin belirli, üretim veya geliştirme aşamasındaki sürümlerinin yüklenmesi hakkında ayrıntılı bilgi içerir.

Yapılandırma Görevleri kılavuzu, Gruntfile içindeki görevlerin, hedeflerin, seçeneklerin ve dosyaların nasıl yapılandırılacağına ilişkin ayrıntılı bir açıklamanın yanı sıra şablonların, küresel kalıpların ve harici verilerin içe aktarılmasının açıklamasına sahiptir.

Görev Oluşturma kılavuzu, Grunt görev türleri arasındaki farkları listeler ve bir dizi örnek görev ve yapılandırmayı gösterir.

GruntJ’lere Giriş #

Grunt, küçültme, derleme, birim testi, linting vb. gibi tekrarlayan görevlerin otomasyonu için kullanılan bir JavaScript Görev Çalıştırıcısıdır.

Başlamak için Grunt’un komut satırı arayüzünü (CLI) global olarak kurmak isteyeceksiniz.

npm install -g grunt-cli

Yeni bir Grunt projesi hazırlama: Tipik bir kurulum, projenize iki dosya eklemeyi içerir: package.json ve Gruntfile.

package.json: Bu dosya npm tarafından npm modülleri olarak yayınlanan projeler için meta verileri depolamak için kullanılır. Projenizin ihtiyaç duyduğu grunt ve Grunt eklentilerini bu dosyada devDependencies olarak listeleyeceksiniz.

Gruntfile: Bu dosya Gruntfile.js olarak adlandırılır ve görevleri yapılandırmak veya tanımlamak ve Grunt eklentilerini yüklemek için kullanılır.

Example package.json:

{
  "name": "my-project-name",
  "version": "0.1.0",
  "devDependencies": {
    "grunt": "~0.4.5",
    "grunt-contrib-jshint": "~0.10.0",
    "grunt-contrib-nodeunit": "~0.4.1",
    "grunt-contrib-uglify": "~0.5.0"
  }
}

Örnek gruntfile:

module.exports = function(grunt) {

  // Project configuration.
  grunt.initConfig({
    pkg: grunt.file.readJSON('package.json'),
    uglify: {
      options: {
        banner: '/*! <%= pkg.name %> <%= grunt.template.today("yyyy-mm-dd") %> */\n'
      },
      build: {
        src: 'src/<%= pkg.name %>.js',
        dest: 'build/<%= pkg.name %>.min.js'
      }
    }
  });

  // Load the plugin that provides the "uglify" task.
  grunt.loadNpmTasks('grunt-contrib-uglify');

  // Default task(s).
  grunt.registerTask('default', ['uglify']);

};

Grunt eklentilerini yükleme #

bağımlılık ekleme

Grunt eklentisi kullanmak için önce onu projenize bir bağımlılık olarak eklemeniz gerekir. Örnek olarak jshint eklentisini kullanalım.

npm install grunt-contrib-jshint --save-dev

“–save-dev” seçeneği, eklentiyi “package.json"a eklemek için kullanılır, bu şekilde eklenti her zaman “npm kurulumu"ndan sonra kurulur.

Eklentiyi yükleme

Eklentinizi gruntfile dosyasına loadNpmTasks kullanarak yükleyebilirsiniz.

grunt.loadNpmTasks('grunt-contrib-jshint');

Görevi yapılandırma

Grunt.initConfig’e iletilen nesneye ‘jshint’ adlı bir özellik ekleyerek gruntfile’deki görevi yapılandırırsınız.

grunt.initConfig({
  jshint: {
    all: ['Gruntfile.js', 'lib/**/*.js', 'test/**/*.js']
  }
});

Kullanmakta olduğunuz diğer eklentiler için başka özelliklere sahip olabileceğinizi unutmayın.

Görevi çalıştırma

Görevi eklentiyle çalıştırmak için komut satırını kullanabilirsiniz.

grunt jshint

Veya başka bir göreve “jshint” ekleyebilirsiniz.

grunt.registerTask('default', ['jshint']);

Varsayılan görev, herhangi bir seçenek olmadan terminalde grunt komutuyla çalışır.
