Mongoose Kullanarak MongoDB’ye Bağlanma #

İlk önce, Mongoose’u aşağıdakilerle yükleyin:

npm install mongoose

Ardından, onu bağımlılıklar olarak “server.js"ye ekleyin:

var mongoose = require('mongoose');
var Schema = mongoose.Schema;

Ardından, veritabanı şemasını ve koleksiyonun adını oluşturun:

var schemaName = new Schema({
    request: String,
    time: Number
}, {
    collection: 'collectionName'
});

Bir model oluşturun ve veritabanına bağlanın:

var Model = mongoose.model('Model', schemaName);
mongoose.connect('mongodb://localhost:27017/dbName');

Ardından, MongoDB’yi başlatın ve “node server.js"yi kullanarak “server.js"yi çalıştırın.

Veritabanına başarılı bir şekilde bağlanıp bağlanmadığımızı kontrol etmek için, ‘mongoose.connection’ nesnesinden ‘open’, ’error’ olaylarını kullanabiliriz.

var db = mongoose.connection;
db.on('error', console.error.bind(console, 'connection error:'));
db.once('open', function() {
  // we're connected!
});

Mongoose, Express.js Routes ve $text Operator Kullanarak MongoDB’de Veri Bulun #

Kurmak #

İlk olarak, gerekli paketleri aşağıdakilerle kurun:

npm install express cors mongoose

Kod #

Ardından, “server.js"ye bağımlılıklar ekleyin, veritabanı şemasını ve koleksiyonun adını oluşturun, bir Express.js sunucusu oluşturun ve MongoDB’ye bağlanın:

var express = require('express');
var cors = require('cors'); // We will use CORS to enable cross origin domain requests.
var mongoose = require('mongoose');
var Schema = mongoose.Schema;

var app = express();

var schemaName = new Schema({
    request: String,
    time: Number
}, {
    collection: 'collectionName'
});

var Model = mongoose.model('Model', schemaName);
mongoose.connect('mongodb://localhost:27017/dbName');

var port = process.env.PORT || 8080;
app.listen(port, function() {
    console.log('Node.js listening on port ' + port);
});

Şimdi verileri sorgulamak için kullanacağımız Express.js rotalarını ekleyin:

app.get('/find/:query', cors(), function(req, res) {
    var query = req.params.query;

    Model.find({
        'request': query
    }, function(err, result) {
        if (err) throw err;
        if (result) {
            res.json(result)
        } else {
            res.send(JSON.stringify({
                error : 'Error'
            }))
        }
    })
})

Aşağıdaki belgelerin modeldeki koleksiyonda olduğunu varsayalım:

{
        "_id" : ObjectId("578abe97522ad414b8eeb55a"),
        "request" : "JavaScript is Awesome",
        "time" : 1468710551
}
{
        "_id" : ObjectId("578abe9b522ad414b8eeb55b"),
        "request" : "JavaScript is Awesome",
        "time" : 1468710555
}
{
        "_id" : ObjectId("578abea0522ad414b8eeb55c"),
        "request" : "JavaScript is Awesome",
        "time" : 1468710560
}

Ve amaç sadece JavaScript kelimesini içeren tüm dokümanları "request" tuşu altında bulmak ve görüntülemektir.

Bunu yapmak için önce koleksiyonda "request" için bir metin dizini oluşturun. Bunun için server.js dosyasına aşağıdaki kodu ekleyin:

schemaName.index({ request: 'text' });

Ve değiştirin:

    Model.find({
        'request': query
    }, function(err, result) {

İle birlikte:

Model.find({
    $text: {
        $search: query
    }
}, function(err, result) {

Burada, belirtilen bulma sorgusundan en az bir kelime içeren collectionName koleksiyonundaki tüm belgeleri bulmak için $text ve $search MongoDB operatörlerini kullanıyoruz.

Kullanım #

Bunu veri bulmak amacıyla kullanmak için bir tarayıcıda aşağıdaki URL’ye gidin:

http://localhost:8080/find/<query>

“” arama sorgusudur.

Örnek:

http://localhost:8080/find/JavaScript

Çıktı:

[{
    _id: "578abe97522ad414b8eeb55a",
    request: "JavaScript is Awesome",
    time: 1468710551,
    __v: 0
},
{
    _id: "578abe9b522ad414b8eeb55b",
    request: "JavaScript is Awesome",
    time: 1468710555,
    __v: 0
},
{
    _id: "578abea0522ad414b8eeb55c",
    request: "JavaScript is Awesome",
    time: 1468710560,
    __v: 0
}]

Mongoose ve Express.js Routes kullanarak Verileri MongoDB’ye Kaydetme #

Kurmak #

İlk olarak, gerekli paketleri aşağıdakilerle kurun:

npm install express cors mongoose

Kod #

Ardından, server.js dosyanıza bağımlılıklar ekleyin, veritabanı şemasını ve koleksiyonun adını oluşturun, bir Express.js sunucusu oluşturun ve MongoDB’ye bağlanın:

var express = require('express');
var cors = require('cors'); // We will use CORS to enable cross origin domain requests.
var mongoose = require('mongoose');
var Schema = mongoose.Schema;

var app = express();

var schemaName = new Schema({
    request: String,
    time: Number
}, {
    collection: 'collectionName'
});

var Model = mongoose.model('Model', schemaName);
mongoose.connect('mongodb://localhost:27017/dbName');

var port = process.env.PORT || 8080;
app.listen(port, function() {
    console.log('Node.js listening on port ' + port);
});

Şimdi verileri yazmak için kullanacağımız Express.js rotalarını ekleyin:

app.get('/save/:query', cors(), function(req, res) {
    var query = req.params.query;

    var savedata = new Model({
        'request': query,
        'time': Math.floor(Date.now() / 1000) // Time of save the data in unix timestamp format
    }).save(function(err, result) {
        if (err) throw err;

        if(result) {
            res.json(result)
        }
    })
})

Burada “query” değişkeni, MongoDB’ye kaydedilecek olan gelen HTTP isteğinden “” parametresi olacaktır:

var savedata = new Model({
    'request': query,
    //...

MongoDB’ye yazmaya çalışırken bir hata oluşursa konsolda bir hata mesajı alırsınız. Her şey başarılı olursa, kaydedilen verileri sayfada JSON formatında göreceksiniz.

//...
}).save(function(err, result) {
    if (err) throw err;

    if(result) {
        res.json(result)
    }
})
//...

Şimdi MongoDB’yi başlatmanız ve ‘server.js’ dosyanızı ’node server.js’ kullanarak çalıştırmanız gerekiyor.

Kullanım #

Bunu verileri kaydetmek için kullanmak için tarayıcınızda aşağıdaki URL’ye gidin:

Mongoose ve Express.js Yollarını Kullanarak MongoDB’de Veri Bulma #

Kurmak #

İlk olarak, gerekli paketleri aşağıdakilerle kurun:

npm install express cors mongoose

Kod #

Ardından, “server.js"ye bağımlılıklar ekleyin, veritabanı şemasını ve koleksiyonun adını oluşturun, bir Express.js sunucusu oluşturun ve MongoDB’ye bağlanın:

var express = require('express');
var cors = require('cors'); // We will use CORS to enable cross origin domain requests.
var mongoose = require('mongoose');
var Schema = mongoose.Schema;

var app = express();

var schemaName = new Schema({
    request: String,
    time: Number
}, {
    collection: 'collectionName'
});

var Model = mongoose.model('Model', schemaName);
mongoose.connect('mongodb://localhost:27017/dbName');

var port = process.env.PORT || 8080;
app.listen(port, function() {
    console.log('Node.js listening on port ' + port);
});

Şimdi verileri sorgulamak için kullanacağımız Express.js rotalarını ekleyin:

app.get('/find/:query', cors(), function(req, res) {
    var query = req.params.query;

    Model.find({
        'request': query
    }, function(err, result) {
        if (err) throw err;
        if (result) {
            res.json(result)
        } else {
            res.send(JSON.stringify({
                error : 'Error'
            }))
        }
    })
})

Modellerdeki dizinler. #

MongoDB ikincil dizinleri destekler. Mongoose’da bu indeksleri şemamız içerisinde tanımlıyoruz. Bileşik dizinler oluşturmamız gerektiğinde dizinleri şema düzeyinde tanımlamak gereklidir.

Firavun Faresi Bağlantısı

var strConnection = 'mongodb://localhost:27017/dbName';
var db = mongoose.createConnection(strConnection)

Temel bir şema oluşturma

var Schema = require('mongoose').Schema;
var usersSchema = new Schema({
    username: {
        type: String,
        required: true,
        unique: true
    },
    email: {
        type: String,
        required: true
    },
    password: {
        type: String,
        required: true
    },
    created: {
        type: Date,
        default: Date.now
    }
});

var usersModel = db.model('users', usersSchema);
module.exports = usersModel;

Varsayılan olarak mongoose, modelde tanımlanmamış olsalar bile modelimize iki yeni alan ekler. Bu alanlar şunlardır:

_İD

Mongoose, Schema yapıcısına aktarılmamışsa, şemalarınızın her birine varsayılan olarak bir _id alanı atar. Atanan tür, MongoDB’nin varsayılan davranışıyla çakışacak bir ObjectId’dir. Şemanıza bir _id eklenmesini hiç istemiyorsanız, bu seçeneği kullanarak devre dışı bırakabilirsiniz.

var usersSchema = new Schema({
    username: {
        type: String,
        required: true,
        unique: true
    }, {
        _id: false 
});

__v veya versionKey

versionKey, Mongoose tarafından ilk kez oluşturulduğunda her belgede ayarlanan bir özelliktir. Bu anahtar değeri, belgenin dahili revizyonunu içerir. Bu belge özelliğinin adı yapılandırılabilir.

Bu alanı model yapılandırmasında kolayca devre dışı bırakabilirsiniz:

var usersSchema = new Schema({
    username: {
        type: String,
        required: true,
        unique: true
    }, {
    versionKey: false 
});

Bileşik dizinler

Mongoose’un yarattıklarının yanında başka dizinler de oluşturabiliriz.

usersSchema.index({username: 1 });
usersSchema.index({email: 1 });

Bu durumda modelimizde, biri alan kullanıcı adı ve diğeri e-posta alanı için olmak üzere iki dizin daha vardır. Ancak bileşik dizinler oluşturabiliriz.

usersSchema.index({username: 1, email: 1 });

Dizin performans etkisi

Varsayılan olarak, mongoose her dizin için her zaman sırayla emineIndex’i çağırır ve tüm garantiIndex çağrıları başarılı olduğunda veya bir hata olduğunda modelde bir ‘index’ olayı yayar.

MongoDB’de, sureIndex 3.0.0 sürümünden beri kullanımdan kaldırılmıştır, şimdi createIndex için bir takma addır.

Şemanızın autoIndex seçeneğini false olarak ayarlayarak davranışı devre dışı bırakmanız veya config.autoIndex seçeneğini false olarak ayarlayarak bağlantıda genel olarak devre dışı bırakmanız önerilir.

usersSchema.set('autoIndex', false);

    Model.find({

        if (result) {
            res.json(result)
        } else {
            next() //pass to 404 handler

    })
    .catch(next) //pass to error handler
})
