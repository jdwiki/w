Sözdizimi #

  • add_action(etiket, function_to_call, öncelik, num_of_args);
  • add_filter(etiket, function_to_call, öncelik, num_of_args);

Parametreler #

parametre Açıklama
$etiket (string) (Zorunlu) $fonksiyonunun bağlı olduğu eylemin adı.
$fonksiyon (çağrılabilir) (Gerekli) İşlev adını veya anonim işlevi içeren bir dize gerektirir. Sınıflar içinde işlev ekleme örneklerine bakın.
$öncelik (int) varsayılan = 10. Kancalara/filtrelere eklenen işlevler, atanan öncelikte çalışır. Diğer işlemlerden önce kodla çalışmak istediğiniz bir durum olabilir, öncelik =1 veya diğer tüm ek fonksiyonlardan sonra öncelik = 100 vb. ayarlayın. Tüm php fonksiyonlarında olduğu gibi, fonksiyonu bir değişken için bir değer iletmeden kullanabilirsiniz. bir varsayılan değerin ayarlandığı yerde, ancak döndürülen parametre sayısını değiştirmek istiyorsanız, belirtmeniz gerekir!
$parametreler (int) varsayılan = 1. Ekli işlevinize döndürülen parametre sayısı. Döndürülen parametreler, kancanın oluşturulduğu yere eklenen sayıya bağlı olacaktır. Daha fazla ayrıntı için ‘apply_filters()’ ve ‘do_action()‘a bakın.

Wordpress Kancaları

WordPress ile çalışmaya başlarken geliştiricilerin kafasını karıştıran bir şey, ‘apply_filters()’ ve ‘add_action()’ kullanımıdır. Bunları kodda kullanan eklentileri/temaları sık sık göreceksiniz ve konsepti anlamadıysanız, onlarla çalışmakta zorlanacaksınız.

Kısaca (çok kısa, süreç için ayrıntılı olarak WordPress yük akış şemasına bakın), WordPress şu şekilde yüklenir:

  1. wp-load.php - fonksiyonlar vb.
  2. mu-plugins - mu-plugins klasöründe bulunan herhangi bir dosya - genellikle önbelleğe alınmış nesneleri sunmak için kullanılır
  3. Eklentiler - belirli bir sıra yok, kurulu ve etkinleştirilmiş eklentiler yüklenecek
  4. Aktif alt tema / ana tema
  5. init - verilerin geri kalanı
  6. şablon

Bir geliştiriciyseniz ve bir işlev dosyasıyla çalışıyorsanız, her ikisinin de işlemde birlikte çalıştığınız dosyalardan daha önce yüklendiğini görebilirsiniz. Yani süreçleri (işlevlerin üzerine yazamayacağınızı unutmayın) veya daha sonra çalışan veya henüz tanımlanmamış değişkenleri değiştiremezsiniz. Ayrıca tema geliştiricileri, eklentilerin bağlanmasına izin vermek için kodlarına kancalar yerleştirebilir veya eklentiler, diğer eklentilerin değişkenlerinin üzerine yazmasına izin verebilir. Şimdi bu şimdiye kadar kafa karıştırıcı olabilir, ama orada durun.

add_filter() ve add_action() öğelerini anlamak için ilk olarak kancaların nasıl oluşturulduğuna bakmamız gerekir.

$arga= 'hello';
do_action('im_a_hook', $arga );

WordPress’te yukarıdakilerle karşılaştığınızda, ‘im_a_hook’ kancasına bağlı tüm işlevleri çağıracaktır (işlem hakkında bilgi için ‘$wp_filter’a bakın). Ekli işlevinizde, ekli işlevin çalışması için `$arga’ mevcut olacaktır.

add_action('im_a_hook', 'attached_function');

function attached_function($arga){
     echo $arga;
}

Bu, yükleme işleminin belirli noktalarında değişkenleri değiştirmek için güçlü yeni fırsatlar açar. Daha önce şablonların eklentilerden/temalardan sonra yüklendiğini söylediğimizi hatırlıyor musunuz? Yaygın bir eklenti, sürecin ilerleyen aşamalarında ekranlar oluşturan WooCommerce’dir, nasıl olduğunu belgelemeyeceğim, ancak eklentide bir do_action örneği bulunabilir.

do_action( 'woocommerce_after_add_to_cart_button' );

Burada hiçbir değişkeni geri iletmeyen bir kanca oluşturduk, ancak yine de onunla eğlenebiliriz:

add_action( 'woocommerce_after_add_to_cart_button', 'special_offer');

function special_offer(){
    echo '<h1>Special Offer!</h1>;
}

Yukarıdaki “add_action”, bir WooCommerce ekranı oluştururken “do_action(‘woocommerce_after_add_to_cart_button’)” öğesinin bulunduğu özel teklifin bir başlığını “yankılayacaktır”. Böylece html eklemek için bu kancayı kullanabiliriz. Diğer kullanımlar, tamamen farklı bir ekrana yeniden yönlendirmeyi vb. içerebilir.

Ayrıca fonksiyona birden fazla değişken aktarılabilir. Bunu tema işlevlerinizde deneyin. 3 olarak ayarladığımız son parametreye dikkat edin, çünkü mevcut 3 parametre ile çalışmak istiyoruz. Bunu 2 olarak değiştirirsek sadece 2 tanesi döndürülür ve tanımsız bir hata alırız.

add_action('custom_hook', 'attached_function', 10, 3);

function attached_function($a,$b,$c){
    
    var_dump($a);
    var_dump($b);
    var_dump($c);
    
}
    

$arga = 1;
$argb = 2;
$argc = 3;

do_action('custom_hook', $arga, $argb, $argc);
exit;

Filtre adı verilen başka bir WP kanca türü vardır. Bir filtre kullanımındaki bir eylemden farklıdır, bir eylem yalnızca değişkenleri alabilir, açıkçası bu değişkenler işlev kapsamındadır (google değilse php kapsamının ne olduğunu bilmelisiniz). Filtreler, döndürülen verileri geri iletir, böylece değişkenleri değiştirmek için kullanabilirsiniz.

$filter_me= apply_filters('im_a_filter', $variable_to_filter);

Yukarıdakileri gördüğünüz yerde, döndürdüğünüz herhangi bir veri değişkende depolanan değer olacağından $filter_me değerini değiştirebilirsiniz. Örneğin (örnekte “$variable_to_filter” öğesini “$filter_me” olarak değiştirdiğimizi unutmayın):

add_filter('im_a_filter', 'attached_function', 100);

function attached_function($filter_me){
    
    $filter_me= 'ray';
    
    return $filter_me;
    
}


$filter_me = 'bob';
$filter_me= apply_filters('im_a_filter', $filter_me);

$filter_me değişkeni artık ‘bob’ yerine ‘ray’ içerecek, 100’lük bir öncelik belirledik, bu nedenle kullanımdan sonra kimsenin değeri değiştirmediğinden eminiz (birden fazla filtre çalışıyor olabilir). Aynı kanca) Böylece, eğer apply_filters() mevcutsa, işlemde daha sonra kullanılan değişkenleri değiştirebiliriz.

Ayrıca birden çok parametre iletebilirsiniz, ancak yalnızca birinin değerini değiştirebilirsiniz. Ayrıca bir değer döndürmelisiniz, yoksa değişkeniniz hiçbir şey içermeyecektir. Değişkenlere değerler/diziler/nesneler atamak için php’yi nasıl kullandığınızı anlarsanız, bu sizin için açık olacaktır, örneğin:

add_filter('im_a_filter', 'attached_function', 100, 3);

function attached_function($filter_me, $arga, $argb){
    
    $filter_me= 'ray'.$arga.$argb;

    $arga= 'you fool';
    
    return $filter_me;
    
}

$filter_me = 'bob';

$arga = ' middlename';
$argb = ' surname';

$filter_me= apply_filters('im_a_filter', $filter_me, $arga, $argb);

$filter_me değişkeni artık ‘ray orta ad soyadı’ içeriyor. Peki ya “$arga”? Bu hala ‘middlename’ içeriyor, fonksiyonumuz içinde bir ‘$arga’yı ‘aptal’ olarak değiştirmenin, kapsamı dışında tanımlanan değer üzerinde hiçbir etkisi yoktur (yollar, google globalleri vb. vardır)

add_action($hook_name, $işlev, $öncelik, $parametreler)

add_filter($hook_name, $işlev, $öncelik, $parametreler);

add_action - başlat #

add_action('init', 'process_post');

function process_post(){
   if($_POST)
     var_dump($_POST);
}

add_action - init - anonim işlev #

add_action('init' , function(){
    echo 'i did something';
});

add_action - init - sınıf nesnesi içinde #

 class sample{

     public function __construct(){
         add_action('init', array($this, 'samp') );
     }


     public function samp(){ // must be public!!
         echo 'i did something';
     }
 }

 new sample();

add_action - init - statik sınıf içinde #

 class sample{

     public static function add_action_func(){
         //note __CLASS__ will also include any namespacing
         add_action('init', array(__CLASS__, 'samp') );
     }

     public static function samp(){
         echo 'i did something';
     }
     
 }

 sample::add_action_func();
