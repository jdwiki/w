Bu sayfada

  • 1. Başlarken
  • 2. HelloWorldLeftAndMain.php’yi Yapılandırma
  • 3. Şablon (HelloWorldLeftAndMain_Content.ss)
  • Bu kılavuz için dikkate değer 3 bölüm vardır:

“LeftAndMain” daha çok alt düzey bir API’dir ve “ModelAdmin"in varlığından dolayı genellikle gerekli değildir. Ancak, modülünüzün yönetim panelindeki ‘ModelAdmin’ işlevselliğini zorunlu olarak gerektirmeyen özel bir kullanıcı arabirimi oluşturmak istiyorsanız, başlamak istediğiniz yer ‘LeftAndMain’ olacaktır.

1. Başlarken #

Bu kılavuz, ‘LeftAndMain’ sınıfını alt sınıflara ayırarak kendi Kullanıcı Arayüzünüzü oluşturmaya başlamanız için hazırlanmıştır.

Bu kılavuzun sonunda, Yönetim Panelinde ilk ‘Merhaba Dünya’ arayüzünüzü oluşturmuş olacaksınız.

Gereksinimler #

Bu kılavuz, çerçeve VE CMS’nin en az “3.” sürümüne sahip olmanızı, ancak “4.” sürümünden daha düşük bir sürüme sahip olmanızı gerektirir.

Bu kılavuzu kullanmak istiyorsanız, tüm sınıf referanslarını SS4 yükseltme kılavuzunda tanımlandığı gibi Tam Kalite Sınıf Adı (FQCN) ile değiştirmeniz gerekecektir.

Hazırlık #

tl;dr Aşağıdaki adımları göz ardı edin ve bunların altındaki yapıyı oluşturun

  1. SilverStripe projeniz için kök dizinde seçtiğiniz herhangi bir isimle bir klasör oluşturun, bu örnek için /helloworld/ kullanacağız ve bu klasörde _config.php adında boş bir dosya oluşturacağız. SilverStripe’in varlığını algılaması için her modül dizininde en azından bir `_config.php’ gereklidir.

  2. Yeni klasörünüzün içinde, tam olarak /code/ adında bir alt klasör oluşturun ve bu klasör içinde organizasyon amacıyla; /admin/ adında başka bir klasör oluşturun

  3. /helloworld/code/admin/HelloWorldLeftAndMain.php dosyasını oluşturun ve şimdilik aşağıdaki kodu içine yerleştirin.

     class HelloWorldLeftAndMain extends LeftAndMain {

 }

  4. /helloworld/templates/Includes/HelloWorldLeftAndMain.ss adlı bu sınıfla kullanılacak şablon dosyasını oluşturun.

Yapı #

/framework/
/cms/
/helloworld/
    + _config.php
    + /code/
        + /admin/
            + /HelloWorldLeftAndMain.php
    + /templates/
        + /Includes/
            + /HelloWorldLeftAndMain_Content.ss

2. HelloWorldLeftAndMain.php’yi Yapılandırma #

Henüz yapmadıysanız, bu dosyayı şununla başlatmanıza izin verin:

class HelloWorldLeftAndMain extends LeftAndMain {

}

Yapılandır #

Yapmanız gereken ilk şey, arayüze erişmek için kullanılacak $url_segment ve yönetim panelinin yan gezinme menüsünde görünecek başlığı ($menu_title) tanımlamaktır:

private static $url_segment = 'helloworld';
private static $menu_title  = 'Hello World';

Aşağıdaki yapılandırma değişken(ler)i isteğe bağlıdır ve bu kılavuzda kullanılmamaktadır:

private static $menu_icon   = 'helloworld/path/to/my/icon.png';
private static $url_rule    = '/$Action/$ID/$OtherID';

Stil Sayfaları ve Javascript Ekleme #

‘LeftAndMain’, ebeveynindeki ‘init’ yöntemini geçersiz kılmanıza izin verir, bunu arayüzümüz için belirli dosyalar istemek için kullanabiliriz. Kuşkusuz, her zaman kullanıcı arayüzünüz için öğeleri stillendirecek bir CSS stil sayfasına ihtiyaç duymalısınız.

Bir ipucu olarak, CMS tarafından sağlanan CSS sınıflarına asla güvenmemeniz önerilir, çünkü bunlar önceden bildirilmeksizin değiştirilebilir ve daha sonra kullanıcı arayüzünüzün Görünüm ve Hissini bozar (örneğin, “3.” ila “4.” , arayüzün tamamen yenilendiğini gördü, bu nedenle 3.içinde güvendiğiniz tüm CSS sınıflarının4.` biçimine dönüştürülmesi için yeniden biçimlendirilmesi gerekir)

Şimdi helloworld/css/styles.css dosyamızı ekleyelim:

public function init() {
    parent::init();

    Requirements::css('helloworld/css/styles.css');
    //Requirements::javascript('helloworld/javascript/script.min.js');
}

Bu örnek için herhangi bir Javascript işlevine ihtiyacımız yok ama yukarıda, Requirements sınıfını kullanarak bir dosyaya Javascript eklemenin nasıl başarılacağını ekledim.

Bundan sonra, ‘$allowed_actions’ vb. gibi ‘Page_Controller’ ile uğraşırken alışık olduğunuz şeyi, ancak dikkate değer bir farkla benimseyebilirsiniz.

“index()“i geçersiz kılamazsınız**.

Bunun yerine ‘index()’, ‘HelloWorldLeftAndMain_Content.ss’ olarak kabul edilir ve buradan, şablon işlevleri aracılığıyla dizinlerin görüntülenmesiyle ilgilenmek gerekir (aşağıdaki örneğe bakın)

Kodu Tamamla #

class HelloWorldLeftAndMain extends LeftAndMain {
    private static $url_segment = 'helloworld';
    private static $menu_title  = 'Hello World';
    private static $allowed_actions = array(
        'some_action'
    );

    public function init() {
        parent::init();

        Requirements::css('helloworld/css/styles.css');
        //Requirements::javascript('helloworld/javascript/script.min.js');
    }

    public function Hello($who=null) {
        if (!$who) {
            $who = 'World';
        }

        return "Hello " . htmlentities($who);
    } 
}

3. Şablon (HelloWorldLeftAndMain_Content.ss) #

Bu şablonun beklenen yapısı biraz dolambaçlı olabilir, ancak hepsi şu şekilde özetlenebilir:

  1. Bu kılavuz için dikkate değer 3 bölüm vardır: #

    • .north
    • .center
    • .south

  2. Tamamen data-pjax-fragment="Content" niteliğine sahip bir öğenin içine sarılmalıdır. Bu, yan menüden oluşturulan AJAX çağrılarının “İçeriğin” nerede olduğunu bilmesi için uygun şekilde görüntüleyebilmesi içindir:

<div class="cms-content center $BaseCSSClasses" data-layout-type="border" data-pjax-fragment="Content">

</div>

Şablon işlevselliği hakkında ayrıntılara girmeyeceğim, ilgili yerlere yorumlar ekledim ancak SilverStripe için şablon sözdizimini anlamıyorsanız bu kılavuzu okumamalısınız.

Kodu Tamamla #

Aşağıdan gelen tek şey; <% include CMSBreadcrumbs %> daha önceden dahil edilmiş olan CSS dosyasında kendiniz için sağlamanız gereken her şey.

<div class="cms-content center $BaseCSSClasses" data-layout-type="border" data-pjax-fragment="Content">
    <%-- This will add the breadcrumb that you see on every other menu item --%>
    <div class="cms-content-header north">
        <div class="cms-content-header-info">
            <% include CMSBreadcrumbs %>
        </div>
    </div>
    
    <div class="center">
        <%-- Our function in HelloWorldLeftAndMain.php --%>
        $Hello('USER');
        <%-- ^ outputs "Hello USER" --%>
    </div>

    <div class='south'>
        Some footer-worthy content
    </div>
</div>

Şimdi tek yapmanız gereken /dev/build ve ?flush=1 yapmak, o zaman Yönetim Panelindeki işe yaramaz küçük modülümüze göz atabilirsiniz!
