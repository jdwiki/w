Kurulum veya Kurulum #

Ravendb’nin kurulması veya kurulmasıyla ilgili ayrıntılı talimatlar.

Basit bir RavenDB konsol uygulaması #

Bu örnek için Live Test RavenDB örneğini kullanacağız.

Burada en temel işlemleri gösteren basit bir konsol uygulaması oluşturacağız:

  • Yaratılış
  • Kimliğe göre alma

Yeni bir Visual Studio çözümü oluşturarak başlayın ve buna bir Konsol Uygulaması projesi ekleyin. Buna RavenDBDemoConsole diyelim. Reader, VS Code veya favori düzenleyicisini kullanıyorsa, istemci kitaplığının kullanımı benzer olmalıdır.

Ardından, gerekli referansları eklememiz gerekiyor. Çözüm Gezgini bölmesinde Başvurular düğümüne sağ tıklayın ve NuGet paketlerini Yönet’i seçin. ‘RavenDb.Client’ için çevrimiçi göz atın. Bu yazı itibariyle - 3.5.2 olan en son kararlı sürümü kullanacağım.

Biraz kod yazalım, olur mu? Aşağıdaki kullanarak ifadeleri ekleyerek başlayın:

using Raven.Client;
using Raven.Client.Document;

Bunlar, RavenDB örneğine bağlanmak için bir arayüz ve onun kullanıma hazır uygulaması olan RavenDB’nin ‘IDocumentStore’ ve ‘DocumentStore’u kullanmamıza izin verir. Bu, bir sunucuya bağlanmak için kullanmamız gereken en üst düzey nesnedir ve [RavenDB belgeleri2, uygulamada tekil olarak kullanılmasını önerir.

Bu yüzden devam edeceğiz ve bir tane oluşturacağız, ancak basitlik için etrafına singleton sarmalayıcı uygulamayacağız - sadece program çıktığında onu atacağız, böylece bağlantı temiz bir şekilde kapatılacaktır. Ana yönteminize aşağıdaki kodu ekleyin:

using (IDocumentStore store = new DocumentStore
{
    Url = "http://live-test.ravendb.net",
    DefaultDatabase = "Pets"
})
{
    store.Initialize();
}

Başta da söylediğimiz gibi Live Test RavenDB Instance’ını kullanıyoruz, adresini ‘DocumentStore’un ‘Url’ özelliği olarak kullanıyoruz. Ayrıca varsayılan bir veritabanı adı da belirtiyoruz, bu durumda “Evcil Hayvanlar”. Veritabanı henüz mevcut değilse, RavenDB ona erişmeye çalışırken onu oluşturur. Varsa, müşteri mevcut olanı kullanabilir. Üzerinde çalışmaya başlayabilmemiz için Initialize() yöntemini çağırmamız gerekiyor.

Bu basit uygulamada, sahiplerini ve evcil hayvanlarını koruyacağız. Bağlantılarını düşünüyoruz, çünkü bir sahibin keyfi sayıda evcil hayvanı olabilir, ancak bir evcil hayvanın yalnızca bir sahibi olabilir. Gerçek dünyada, bir evcil hayvanın, örneğin bir karı koca gibi keyfi sayıda sahibi olsa da, bir belge veritabanındaki çoktan çoğa ilişkiler biraz farklı şekilde ele alındığından bu varsayımı tercih edeceğiz. ] bundan ilişkisel bir veritabanında bulunur ve kendi başına bir konuyu hak eder. Bu alanı seçtim çünkü kavranacak kadar yaygın.

Şimdi etki alanı nesne türlerimizi tanımlamalıyız:

public class Owner
{
    public Owner()
    {
        Pets = new List<Pet>();
    }

    public string Id { get; set; }
    public string Name { get; set; }
    public List<Pet> Pets { get; set; }

    public override string ToString()
    {
        return
            "Owner's Id: " + Id + "\n" +
            "Owner's name: " + Name + "\n" +
            "Pets:\n\t" +
            string.Join("\n\t", Pets.Select(p => p.ToString()));
    }
}

public class Pet
{
    public string Color { get; set; }
    public string Name { get; set; }
    public string Race { get; set; }

    public override string ToString()
    {
        return string.Format("{0}, a {1} {2}", Name, Color, Race);
    }
}

Burada dikkat edilmesi gereken bazı şeyler var:

İlk olarak, “Sahip"lerimiz sıfır veya daha fazla “Evcil Hayvan” içerebilir. “Owner” sınıfının “Id” adında bir özelliği varken “Pet” sınıfının olmadığını unutmayın. Bunun nedeni, ‘Pet’ nesnelerinin, bu tür bir ilişkinin ilişkisel bir veritabanında nasıl uygulanacağından oldukça farklı olan ‘Sahip’ nesnelerinin içinde saklanmasıdır.

Bunun böyle uygulanmaması gerektiği ileri sürülebilir - ve bu doğru olabilir, gerçekten gereksinimlere bağlıdır. Genel bir kural olarak, eğer bir “Evcil Hayvan” bir “Sahip” olmadan var olmak mantıklıysa, o zaman gömülü olmamalı, kendi tanımlayıcısıyla kendi başına var olmalıdır. Uygulamamızda, bir ‘Pet’in yalnızca sahibi varsa evcil hayvan olarak kabul edildiğini, aksi takdirde bir yaratık veya canavar olacağını varsayıyoruz. Bu nedenle ‘Pet’ sınıfına ‘Id’ özelliği eklemiyoruz.

İkinci olarak, genellikle RavenDB belgelerindeki örneklerde gösterildiği gibi, sahibi sınıfın tanımlayıcısının bir dize olduğuna dikkat edin. İlişkisel veritabanlarına alışkın birçok geliştirici, bunun genellikle ilişkisel dünyada mantıklı olan kötü bir uygulama olduğunu düşünebilir. Ancak RavenDB, görevlerini yerine getirmek için Lucene.Net’i kullandığından ve Lucene.Net, dizelerle çalışma konusunda uzmanlaştığından, burada tamamen kabul edilebilir - ayrıca, JSON’u depolayan bir belge veritabanıyla uğraşıyoruz ve sonuçta temelde her şey bir dize olarak temsil ediliyor. ‘Id’ özelliği hakkında not edilmesi gereken bir şey daha, zorunlu olmamasıdır. Aslında, RavenDB kaydettiğimiz herhangi bir belgeye kendi meta verilerini ekler, bu yüzden onu tanımlamamış olsak bile RavenDB’nin nesnelerimizle hiçbir sorunu olmazdı. Bununla birlikte, genellikle daha kolay erişim için tanımlanmıştır.

Kodumuzdan RavenDB’yi nasıl kullanabileceğimizi görmeden önce, birkaç yaygın yardımcı yöntemi tanımlayalım. Bunlar açıklayıcı olmalıdır.

// Returns the entered string if it is not empty, otherwise, keeps asking for it.
private static string ReadNotEmptyString(string message)
{
    Console.WriteLine(message);
    string res;
    do
    {
        res = Console.ReadLine().Trim();
        if (res == string.Empty)
        {
            Console.WriteLine("Entered value cannot be empty.");
        }
    } while (res == string.Empty);

    return res;
}

// Will use this to prevent text from being cleared before we've read it.
private static void PressAnyKeyToContinue()
{
    Console.WriteLine();
    Console.WriteLine("Press any key to continue.");
    Console.ReadKey();
}

// Prepends the 'owners/' prefix to the id if it is not present (more on it later)
private static string NormalizeOwnerId(string id)
{
    if (!id.ToLower().StartsWith("owners/"))
    {
        id = "owners/" + id;
    }

    return id;
}

// Displays the menu
private static void DisplayMenu()
{
    Console.WriteLine("Select a command");
    Console.WriteLine("C - Create an owner with pets");
    Console.WriteLine("G - Get an owner with its pets by Owner Id");
    Console.WriteLine("N - Query owners whose name starts with...");
    Console.WriteLine("P - Query owners who have a pet whose name starts with...");
    Console.WriteLine("R - Rename an owner by Id");
    Console.WriteLine("D - Delete an owner by Id");
    Console.WriteLine();
}

Ve ana yöntemimiz:

private static void Main(string[] args)
{
    using (IDocumentStore store = new DocumentStore
    {
        Url = "http://live-test.ravendb.net", 
        DefaultDatabase = "Pets"
    })
    {
        store.Initialize();

        string command;
        do
        {
            Console.Clear();
            DisplayMenu();

            command = Console.ReadLine().ToUpper();
            switch (command)
            {
                case "C":
                    Creation(store);
                    break;
                case "G":
                    GetOwnerById(store);
                    break;
                case "N":
                    QueryOwnersByName(store);
                    break;
                case "P":
                    QueryOwnersByPetsName(store);
                    break;
                case "R":
                    RenameOwnerById(store);
                    break;
                case "D":
                    DeleteOwnerById(store);
                    break;
                case "Q":
                    break;
                default:
                    Console.WriteLine("Unknown command.");
                    break;
            }
        } while (command != "Q");
    }
}

Yaratılış #

Şimdi bazı nesneleri RavenDB’ye nasıl kaydedebileceğimizi görelim. Aşağıdaki yaygın yöntemleri tanımlayalım:

private static Owner CreateOwner()
{
    string name = ReadNotEmptyString("Enter the owner's name.");

    return new Owner { Name = name };
}

private static Pet CreatePet()
{
    string name = ReadNotEmptyString("Enter the name of the pet.");
    string race = ReadNotEmptyString("Enter the race of the pet.");
    string color = ReadNotEmptyString("Enter the color of the pet.");

    return new Pet
    {
        Color = color,
        Race = race,
        Name = name
    };
}

private static void Creation(IDocumentStore store)
{
    Owner owner = CreateOwner();
    Console.WriteLine(
        "Do you want to create a pet and assign it to {0}? (Y/y: yes, anything else: no)", 
        owner.Name);

    bool createPets = Console.ReadLine().ToLower() == "y";
    do
    {
        owner.Pets.Add(CreatePet());

        Console.WriteLine("Do you want to create a pet and assign it to {0}?", owner.Name);
        createPets = Console.ReadLine().ToLower() == "y";
    } while (createPets);

    using (IDocumentSession session = store.OpenSession())
    {
        session.Store(owner);
        session.SaveChanges();
    }
}

Şimdi nasıl çalıştığını görelim. ‘Sahip’ nesneleri oluşturmak ve kullanıcı istediği kadar ‘Pet’ nesneleri oluşturmaya ve bunlara atamaya devam etmek için bazı basit C# mantığı tanımladık. RavenDB’nin söz konusu olduğu ve dolayısıyla bu makalenin odak noktası olan kısım, nesneleri nasıl kaydettiğimizdir.

Yeni oluşturulan ‘Owner’ı ‘Pet’leri ile birlikte kaydetmek için öncelikle ‘IDocumentSession’ı uygulayan bir oturum açmamız gerekiyor. Belge deposu nesnesinde ‘OpenSession’ı çağırarak bir tane oluşturabiliriz.

Bu nedenle, farka dikkat edin, belge deposu genellikle uygulamanın tüm ömrü boyunca var olan kalıcı bir nesne iken, “IDocumentSession” kısa ömürlü, hafif bir nesnedir. Tek seferde (veya en azından veritabanına yapılan birkaç çağrıda) gerçekleştirmek istediğimiz bir dizi işlemi temsil eder.

RavenDB, sunucuya aşırı sayıda gidiş dönüşlerden kaçınmanızı vurgular (ve bir şekilde zorlar), RavenDB web sitesinde ‘İstemci-Sunucu sohbet koruması’ olarak adlandırdıkları bir şey. Bu nedenle, bir oturumun kaç tane veritabanı çağrısına tahammül edeceği konusunda varsayılan bir sınırı vardır ve bu nedenle, bir oturumun ne zaman açılıp atıldığına dikkat edilmelidir. Bu örnekte, bir ‘Sahip’ ve ‘Pet’lerinin oluşturulmasını, kendi başına yürütülmesi gereken bir işlem olarak ele aldığımız için, bunu tek bir oturumda yapıyoruz ve sonra elden çıkarıyoruz.

Bizi ilgilendiren iki yöntem çağrısı daha görebiliriz:

  • nesneyi kaydedilmek üzere kaydeden ve ayrıca henüz ayarlanmamışsa nesnenin ‘Id’ özelliğini ayarlayan session.Store(sahip). Tanımlayıcı özelliğin ‘Id’ olarak adlandırılması bu nedenle bir gelenektir.
  • session.Savehanges(), yürütülecek asıl işlemleri RavenDB sunucusuna gönderir ve bekleyen tüm işlemleri gerçekleştirir.

Kimliğe göre alma #

Diğer bir yaygın işlem, bir nesneyi tanımlayıcısına göre almaktır. İlişkisel dünyada, bunu normalde tanımlayıcıyı belirten bir ‘Where’ ifadesi kullanarak yaparız. Ancak RavenDB’de her sorgu eski olabilen dizinler kullanılarak yapıldığından, alınacak yaklaşım bu değildir - aslında, id ile sorgulamaya çalışırsak RavenDB bir istisna atar. Bunun yerine, kimliği belirterek Load<T> yöntemini kullanmalıyız. Menü mantığımız zaten yerindeyken, istenen verileri gerçekten yükleyen ve ayrıntılarını görüntüleyen yöntemi tanımlamamız yeterli:

private static void GetOwnerById(IDocumentStore store)
{
    Owner owner;
    string id = NormalizeOwnerId(ReadNotEmptyString("Enter the Id of the owner to display."));

    using (IDocumentSession session = store.OpenSession())
    {
        owner = session.Load<Owner>(id);
    }

    if (owner == null)
    {
        Console.WriteLine("Owner not found.");
    }
    else
    {
        Console.WriteLine(owner);
    }

    PressAnyKeyToContinue();
}

Burada RavenDB ile ilgili olan tek şey, bir kez daha, bir oturumun başlatılması ve ardından ‘Load’ yönteminin kullanılmasıdır. RavenDB istemci kitaplığı, seri durumdan çıkarılmış nesneyi type parametresi olarak belirttiğimiz tür olarak döndürür. RavenDB’nin burada herhangi bir uyumluluğu zorlamadığını bilmek önemlidir - eşlenebilir tüm özellikler eşlenir ve eşlenemez olanlar yapmaz.

RavenDB, ‘Id’ başına eklenen belge türü önekine ihtiyaç duyar - ‘NormalizeOwnerId’ çağrısının nedeni budur. Belirtilen kimliğe sahip bir belge yoksa, “null” döndürülür.

Sorgulama #

Burada iki tür sorgu göreceğiz: Biri “Sahip” belgelerinin kendi özelliklerini sorguladığımız, diğeri ise gömülü “Pet” nesneleri üzerinde sorguladığımız.

‘Ad’ özelliği belirtilen dize ile başlayan ‘Sahip’ belgelerini sorguladığımız daha basit olanla başlayalım.

private static void QueryOwnersByName(IDocumentStore store)
{
    string namePart = ReadNotEmptyString("Enter a name to filter by.");

    List<Owner> result;
    using (IDocumentSession session = store.OpenSession())
    {
        result = session.Query<Owner>()
           .Where(ow => ow.Name.StartsWith(namePart))
           .Take(10)
           .ToList();
    }

    if (result.Count > 0)
    {
        result.ForEach(ow => Console.WriteLine(ow));
    }
    else
    {
        Console.WriteLine("No matches.");
    }
    PressAnyKeyToContinue();
}

Bir kez daha sorguyu bağımsız bir çalışma olarak gerçekleştirmek istediğimiz için bir session açıyoruz. Oturum nesnesinde Query<TDocumentType> çağırarak bir belge koleksiyonunu sorgulayabiliriz. Üzerinde olağan LINQ yöntemlerini ve ayrıca bazı RavenDB’ye özgü uzantıları arayabileceğimiz bir “IRavenQueryable” nesnesi döndürür. Burada basit bir filtreleme yapıyoruz ve koşul, Name özelliğinin değerinin girilen dize ile başlamasıdır. Sonuç kümesinin ilk 10 maddesini alıp bir listesini oluşturuyoruz. Sonuç kümesi boyutunun doğru bir şekilde belirtilmesine dikkat edilmelidir - burada RavenDB tarafından gerçekleştirilen, Sınırsız sonuç kümesi koruması adı verilen oyunda başka bir savunma zorlaması. Bu, (varsayılan olarak) yalnızca ilk 128 öğenin döndürüldüğü anlamına gelir.

İkinci sorgumuz aşağıdaki gibi görünüyor:

private static void QueryOwnersByPetsName(IDocumentStore store)
{
    string namePart = ReadNotEmptyString("Enter a name to filter by.");

    List<Owner> result;
    using (IDocumentSession session = store.OpenSession())
    {
       result = session.Query<Owner>()
           .Where(ow => ow.Pets.Any(p => p.Name.StartsWith(namePart)))
           .Take(10)
           .ToList();
    }

    if (result.Count > 0)
    {
        result.ForEach(ow => Console.WriteLine(ow));
    }
    else
    {
        Console.WriteLine("No matches.");
    }
    PressAnyKeyToContinue();
}

Bu çok daha karmaşık değil, gömülü nesne özelliklerini sorgulamanın ne kadar doğal olduğunu göstermek için yazdım. Bu sorgu, adı girilen değerle başlayan en az bir “Evcil Hayvan"ı olan ilk 10 “Sahip"i döndürür.

Silme #

Silme işlemini gerçekleştirmek için iki seçeneğimiz var. Birincisi, bellekte nesnenin kendisi yoksa ancak tanımlayıcımız varsa ve veritabanına aksi takdirde önlenebilir bir gidiş dönüşünü önlemek istiyorsak yararlı olan belge tanımlayıcısını iletmektir. Diğer yol, açıkçası o zaman, RavenDB’ye kaydedilen gerçek bir nesneyi iletmektir. Burada ilk seçeneğe bakacağız, diğeri sadece başka bir aşırı yükleme kullanıyor ve uygun bir nesneyi geçiyor:

private static void DeleteOwnerById(IDocumentStore store)
{
    string id = NormalizeOwnerId(ReadNotEmptyString("Enter the Id of the owner to delete."));

    using (IDocumentSession session = store.OpenSession())
    {
        session.Delete(id);
        session.SaveChanges();
    }
}

Bir kez daha, işimizi gerçekleştirmek için bir oturum açmamız gerekiyor. Daha önce de belirtildiği gibi, burada tanımlayıcısını ‘Delete’ yöntemine geçirerek istenen nesneyi siliyoruz. Tanımlayıcı öneki burada da “Load” yönteminde olduğu gibi yerinde olmalıdır. Sil komutunu veritabanına gerçekten göndermek için, aynı oturumda kayıtlı diğer bekleyen işlemlerle birlikte tam da bunu yapacak olan ‘SaveChanges’ yöntemini çağırmamız gerekir.

Güncelleniyor #

Son olarak, belgelerin nasıl güncelleneceğine bir göz atacağız. Temel olarak, bunu yapmanın iki yolu var. İlki basittir, bir belgeyi yükleriz, özelliklerini gerektiği gibi güncelleriz ve ardından onu ‘Mağaza’ yöntemine geçiririz. Bu, yükleme ve kaydetme gösterimine göre basit olmalıdır, ancak dikkate alınması gereken birkaç şey var.

İlk olarak, RavenDB istemci kitaplığı, belgeyi yükleyen oturum hala açık olduğu sürece, herhangi bir belgeyi ‘Mağaza’ya geçirmeden güncellemeyi mümkün kılan bir değişiklik izleyici kullanır. Bu durumda, güncellemenin gerçekleşmesi için oturumda Değişiklikleri Kaydeti ​​çağırmak yeterlidir.

İkinci olarak, bunun çalışması için, RavenDB’nin neyin güncelleneceğini bulabilmesi için nesnenin tanımlayıcısının ayarlanması gerekir.

Bunları söyledikten sonra, yalnızca güncellemenin diğer yoluna bir göz atacağız. Belgeleri güncellemek için kullanılabilecek yama adı verilen bir kavram vardır. Silmede olduğu gibi kendi kullanım senaryoları da vardır. Nesne zaten bellekteyse ve/veya tür güvenliğini kullanmak istiyorsak, güncelleme gerçekleştirmek için önceki yöntemi kullanmak iyi bir yoldur. Nesne zaten bellekte yoksa, veritabanına aksi takdirde gereksiz bir gidiş dönüşten kaçınmak istiyorsak, yama kullanmak bir seçenektir. Dezavantajı, düz dizeler kullanarak güncellenecek özellikleri belirtmemiz gerektiğinden, tür güvenliğinin bir kısmını kaybetmemizdir (bazı LINQ-magic’in çözemeyeceği hiçbir şey). Kodu görelim:

private static void RenameOwnerById(IDocumentStore store)
{
    string id = NormalizeOwnerId(ReadNotEmptyString("Enter the Id of the owner to rename."));
    string newName = ReadNotEmptyString("Enter the new name.");

    store.DatabaseCommands.Patch(id, new Raven.Abstractions.Data.PatchRequest[]{
        new Raven.Abstractions.Data.PatchRequest
        {
            Name = "Name",
            Value = newName
        }
    });
}

Bu onu tamamlar. Kod parçalarını bir konsol uygulamasına yapıştırarak örneğin çalıştığını görebilmeniz gerekir.
