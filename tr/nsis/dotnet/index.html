DotNET

  • Örnek Örnek
  • Makrolu İşlev

Bu sayfada

  • Örnek Örnek
  • Makrolu İşlev

.NET Framework bir bilgisayara kurulurken, kurulum başarılı olduğunda .NET yükleyicisi kayıt defteri anahtarlarını yazar. .NET Framework 4.5 veya sonraki sürümünün yüklü olup olmadığını, ‘HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full’ kayıt defteri anahtarını Release adlı bir ‘DWORD’ değeri için kontrol ederek test edebilirsiniz. Bu anahtarın varlığı, o bilgisayarda .NET Framework 4.5 veya sonraki sürümünün yüklü olduğunu gösterir.

.NET Değerleri #

Aşağıda, Release anahtarındaki ‘DWORD’ değerine karşılık gelen sürüm numarasını gösteren bir grafik bulunmaktadır. Bunun eylemde nasıl kullanılacağına ilişkin bir örnek için bu tablonun altına bakın. Bu sayfayı en son değerlerle güncel tutmak için elimden geleni yapacağım ancak [bu sayfayı](https://msdn.microsoft.com/en-us/library/hh925568(v=vs) ziyaret edebilirsiniz. .110).aspx) ihtiyacınız olan sürüm burada listelenmemişse, güncellenmiş bir liste için.

“4.7”, “4.6.2”, “4.6.1” ve “4.6” için iki satır olduğuna özellikle dikkat edin. Bu sürümler, işletim sistemleri değişiklik gösterdiğinden her iki değeri de kontrol etmenizi gerektirir.

Sürüm İşletim Sistemi Salım Değeri
4.7 Windows 10 Oluşturucu Güncellemesi 460798
4.7 Windows 10 Oluşturucu Güncellemesi hariç tümü 460805
4.6.2 Windows 10 Yıldönümü Güncellemesi 394802
4.6.2 Windows 10 Yıldönümü Güncellemesi hariç tümü 394806
4.6.1 Windows 10 Kasım Güncellemesi 394254
4.6.1 Windows 10 Kasım Güncelleştirmesi hariç tümü 394271
4.6 Windows 10 393295
4.6 Windows 10 393297
4.5.2 Hepsi 379893
4.5.1 Windows 8.1 veya Windows Server 2012 R2 378675
4.5.1 Windows 8 ve Windows 7 378758
4.5 Hepsi 378389

Çözüm #

Bu işlevleri tam olarak test etmedim, bu nedenle bunlardan biriyle sorun yaşarsanız bana bildirin, yardımcı olmak için elimden geleni yapacağım. Bu işlevin SharpDevelop Portable ile çalışırken çalıştığını görebilirsiniz.

Yukarıda söylediğim gibi, bunları güncel tutmak için elimden gelenin en iyisini yapacağım, ancak bu işlevlerin kontrol etmediği bir sürüme ihtiyacınız varsa, bunları gözden geçirip bu sayfayı güncelleyebilmem için bana bildirin.

Örnek Örnek #

Burada, dotNET 4.5 veya üstünü kontrol etmek için kayıt defterindeki değerlerin nasıl kullanılacağına dair bir örnek verilmiştir. Bu kod parçacığını .OnInit gibi bir yere koymanızı tavsiye ederim çünkü bu, başka bir şey olmadan önce yürütülecektir; bu şekilde, herhangi bir dosya kopyalanmadan veya kayıt defteri değişiklikleri yapılmadan önce .NET’i kontrol eder.

;=#
;= Define the registry key we're looking for.
!define DOTNET  `SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full`

Section "Main"
    ClearErrors ;= Clear any errors we may have encountered before hand.
    ReadRegDWORD $0 HKLM `${DOTNET}` `Release` ;= Read the value of Release.
    IfErrors +3 ;= If there is an error than there is no .NET installed so we jump 3 lines down which lands on MessageBox.
    IntCmp $0 394806 +5 0 +5    ;= Compares the value for v4.6.2 if it matches then we jump 5 lines and avoids the MessageBox
    IntCmp $0 394802 +4 0 +4    ;= Remember to check for Windows 10's value aswell as the above line won't.
    MessageBox MB_ICONSTOP|MB_TOPMOST `You must have v4.6.2 or greater of the .NET Framework installed. Launcher aborting!` ;= If the check failed then we alert the user the required version wasn't found.
    Call Unload ;= We call the Unload function here because we failed the .NET check.
    Quit ;= Closes the Launcher
SectionEnd

Makrolu İşlev #

dotNETCheck işlevi #

Alternatif olarak aşağıda yazdığım fonksiyonu kullanabilirsiniz. Bu, LogicLib.nsh‘yi kullanır. Kayıt defterindeki Release anahtarından .NET sürümlerinin değerini bilmek zorunda kalmadan kutudan çıktığı gibi çalışmalıdır. Şu anda yazıldığı gibi, yalnızca “4.5”–“4.7” arasındaki sürümleri kontrol eder.

##
# This one requires the use of LogicLib.nsh
# Copy and paste this code somewhere like .OnInit
Function dotNETCheck
    !define CheckDOTNET "!insertmacro _CheckDOTNET"
    !macro _CheckDOTNET _RESULT _VALUE
        Push `${_VALUE}`
        Call dotNETCheck
        Pop ${_RESULT}
    !macroend
    Exch $1
    Push $0
    Push $1
    
    ${If} $1 == "4.7"
        StrCpy $R1 460798
    ${ElseIf} $1 == "4.6.2"
        StrCpy $R1 394802
    ${ElseIf} $1 == "4.6.1"
        StrCpy $R1 394254
    ${ElseIf} $1 == "4.6"
        StrCpy $R1 393295
    ${ElseIf} $1 == "4.5.2"
        StrCpy $R1 379893
    ${ElseIf} $1 == "4.5.1"
        StrCpy $R1 378675
    ${ElseIf} $1 == "4.5"
        StrCpy $R1 378389
    ${Else}
        Goto dotNET_FALSE
    ${EndIf}
    
    ReadRegDWORD $R0 HKLM `SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full` `Release`
    IfErrors dotNET_FALSE
    
    IntCmp $R0 $R1 dotNET_TRUE dotNET_FALSE
    
    dotNET_TRUE:
    StrCpy $0 true
    Goto dotNET_END
    
    dotNET_FALSE:
    StrCpy $0 false
    SetErrors
    
    dotNET_END:    
    Pop $1
    Exch $0
FunctionEnd

##
# USAGE
# ${CheckDOTNET} $0 "Version Number"
##
# $0 Will hold the version number of the installed .NET
# If $0 is empty ($0 == "") then the error flag is set.
${CheckDOTNET} $0 "4.5"
IfErrors 0 +4
MessageBox MB_ICONSTOP|MB_TOPMOST `You must have v4.5 or greater of the .NET Framework installed. Launcher aborting!`
Call Unload
Quit
StrCmpS $0 true 0 -3
