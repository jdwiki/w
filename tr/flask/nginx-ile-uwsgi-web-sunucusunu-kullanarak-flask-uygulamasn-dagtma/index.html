Şişeden akışı etkinleştir #

Flask, jeneratörleri kullanarak bir görünümden veri akışı yapmanızı sağlayan bir özelliğe sahiptir.

app.py dosyasını değiştirelim

  • “şişe içe aktarma Yanıtından” ekleyin

  • ’tarih saatinden içe aktarma tarihinden itibaren’ ekleyin

  • ‘zamandan içe aktarma uykusu’ ekleyin

  • yeni bir görünüm oluşturun:

    @app.route("/time/") def time(): def streamer(): while True: yield “

    {}

    ".format(datetime.now()) sleep(1)

      return Response(streamer())

Şimdi tarayıcınızı localhost/time/ konumunda açın. Site sonsuza kadar yüklenecek çünkü nginx yanıt tamamlanana kadar bekliyor. Bu durumda yanıt hiçbir zaman tamamlanmayacaktır çünkü geçerli tarih ve saati sonsuza kadar gönderecektir.

Nginx’in beklemesini önlemek için konfigürasyona yeni bir satır eklememiz gerekiyor.

/etc/nginx/sites-available/flaskconfig dosyasını düzenleyin

server {
    listen 80;
    server_name localhost;

    location / {
        include uwsgi_params;
        uwsgi_pass unix:///tmp/flask.sock;
        uwsgi_buffering off;  # <-- this line is new
    }
}

uwsgi_buffering off; satırı, nginx’e bir yanıt tamamlanana kadar beklememesini söyler.

Nginx’i yeniden başlatın: “sudo service nginx restart” ve tekrar “localhost/time/” öğesine bakın.

Şimdi her saniye yeni bir satırın açıldığını göreceksiniz.

** Şişeyi ve uWSGI ağ geçidini kurun**

Şişeyi ve uSWGI ağ geçidini kurun:

pip install uwsgi flask

myexample.py’deki şişe uygulaması örneği:

from flask import Flask
application = Flask(__name__)

@application.route("/")
def hello():
    return "<h1>Hello World</h1>"

if __name__ == "__main__":
    application.run(host='0.0.0.0')

Web uygulamanız ve web sunucusu arasında iletişim kurmak için dosya oluşturun: ağ geçidi arayüzü [https://en.wikipedia.org/wiki/Web_Server_Gateway_Interface]

nano wsgi.py

ardından webapp modülünüzü içe aktarın ve ağ geçidi giriş noktasından çalışmasını sağlayın.

from myexample import application

if __name__ == "__main__":
    application.run()

uWSGI’yi test etmek için:

uwsgi --socket 0.0.0.0:8000 --protocol=http -w wsgi

uWSGI’yi yapılandırmak için:

  1. Bir yapılandırma dosyası .ini oluşturun

    nano myexample.ini

  2. uWSGI ağ geçidi için temel yapılandırma

    include header for using uwsgi #

    [uwsgi]

    point it to your python module wsgi.py #

    module = wsgi

    tell uWSGI to start a master node to serve requests #

    master = true

    spawn number of processes handling requests #

    processes = 5

    use a Unix socket to communicate with Nginx. Nginx will pass connections to uWSGI through a socket, instead of using ports. This is preferable because Nginx and uWSGI stays on the same machine. #

    socket = myexample.sock

    ensure file permission on socket to be readable and writable #

    chmod-socket = 660

    clean the socket when processes stop #

    vacuum = true

    use die-on-term to communicate with Ubuntu versions using Upstart initialisations: see: #

    http://uwsgi-docs.readthedocs.io/en/latest/Upstart.html?highlight=die%20on%20term #

    die-on-term = true

sanal env kullanıyorsanız isteğe bağlıdır Sanal ortamınızı ‘devre dışı bırakabilirsiniz’.

Nginx yapılandırması Nginx’i şu şekilde kullanacağız:

  1. uwsgi protokolünü kullanarak isteği sokete iletmek için varsayılan sunucu
  2. varsayılan sunucunun önünde proxy sunucusu
  3. Başarılı istekleri önbelleğe almak için önbellek sunucusu (örneğin, web uygulamanız varsa GET isteklerini önbelleğe almak isteyebilirsiniz)

“Siteler için uygun” dizininizi bulun ve uygulamanız için bir yapılandırma dosyası oluşturun:

sudo nano /etc/nginx/sites-available/myexample

Ne yaptığını yorumlarda aşağıdaki bloğu ekleyin:

server {
   

    # setting up default server listening to port 80
    listen 8000 default_server;
    server_name myexample.com; #you can also use your IP 
    
    # specify charset encoding, optional
    charset utf-8;

    # specify root of your folder directory
    root /var/www/myexample;

    # specify locations for your web apps.
    # here using /api endpoint as example
    location /api {
        # include parameters of wsgi.py and pass them to socket
        include uwsgi_params;
        uwsgi_pass unix:/var/www/myexample/myexample.sock;
    }

}

# Here you will specify caching zones that will be used by your virtual server
# Cache will be stored in /tmp/nginx folder
# ensure nginx have permissions to write and read there!
# See also:
# http://nginx.org/en/docs/http/ngx_http_proxy_module.html

proxy_cache_path /tmp/nginx levels=1:2 keys_zone=my_zone:10m inactive=60m;
proxy_cache_key "$scheme$request_method$host$request_uri";

# set up the virtual host!
server {
    listen   80  default_server;
    
    # Now www.example.com will listen to port 80 and pass request to http://example.com
    server_name www.example.com;

    # Why not caching responses

    location /api {
        # set up headers for caching
        add_header X-Proxy-Cache $upstream_cache_status;

        # use zone specified above
        proxy_cache my_zone;
        proxy_cache_use_stale updating;
        proxy_cache_lock on;
        
        # cache all responses ?
        # proxy_cache_valid 30d;

        # better cache only 200 responses :)
        proxy_cache_valid 200 30d;

        # ignore headers to make cache expire
        proxy_ignore_headers X-Accel-Expires Expires Cache-Control;

        # pass requests to default server on port 8000
        proxy_pass http://example.com:8000/api;
    }
}

Son olarak, dosyayı “siteler etkin” dizine bağlayın. Kullanılabilir ve etkin sitelerin açıklaması için cevaba bakın: [http://serverfault.com/a/527644]

sudo ln -s /etc/nginx/sites-available/myproject /etc/nginx/sites-enabled

Artık nginx ile işiniz bitti. Ancak, bu çok değerli kazan şablonuna göz atmak isteyebilirsiniz: [https://github.com/h5bp/server-configs-nginx]

İnce ayar için çok kullanışlıdır.

Şimdi Nginx’i test edin:

sudo nginx -t

Nginx’i başlatın:

sudo service nginx restart

UWSGI’yi başlatmak için Ubuntu’yu otomatikleştirin Son şey, Ubuntu’nun uygulamanızla iletişim kuran wsgi ağ geçidini başlatmasını sağlamaktır, aksi takdirde bunu manuel olarak yapmalısınız.

  1. Ubuntu’da başlatma komut dosyaları için dizini bulun ve yeni bir komut dosyası oluşturun:

sudo nano /etc/init/myexample.conf

  1. Ne yaptığını açıklamak için aşağıdaki bloğu, yorumları satıra ekleyin

    # description for the purpose of this script
description "uWSGI server instance configured to serve myproject"

# Tell to start on system runtime 2, 3, 4, 5. Stop at any other level (0,1,6). 
# Linux run levels: [http://www.debianadmin.com/debian-and-ubuntu-linux-run-levels.html]
start on runlevel [2345]
stop on runlevel [!2345]

# Set up permissions! "User" will be the username of your user account on ubuntu.
setuid user
# Allow www-data group to read and write from the socket file. 
# www-data is normally the group Nginx and your web applications belong to.
# you may have all web application projects under /var/www/ that belongs to www-data group
setgid www-data

# tell Ubunutu which environment to use.
# This is the path of your virtual environment: python will be in this path if you installed virtualenv. Otherwise, use path of your python installation
env PATH=/var/www/myexample/myexample/bin
# then tell to Ubuntu to change and locate your web application directory
chdir /var/www/myexample
# finally execute initialisation script, that load your web app myexample.py
exec uwsgi --ini myexample.ini

Artık betiğinizi etkinleştirebilirsiniz: sudo örneğimi başlat

Bir şişe uygulamasını çalıştırmak için uWSGI kullanma #

Yerleşik ‘werkzeug’ sunucusu, üretim sunucularını çalıştırmak için kesinlikle uygun değildir. Bunun en belirgin nedeni, ‘werkzeug’ sunucusunun tek iş parçacıklı olması ve bu nedenle bir seferde yalnızca bir isteği işleyebilmesidir.

Bu nedenle, uygulamamıza hizmet etmek için uWSGI Sunucusunu kullanmak istiyoruz. Bu örnekte uWSGI’yi kuracağız ve onunla basit bir test uygulaması çalıştıracağız.

uWSGI’yi yükleme:

pip install uwsgi

Bu kadar basit. Pip’inizin kullandığı python sürümünden emin değilseniz, bunu açıkça belirtin:

python3 -m pip install uwsgi  # for python3
python2 -m pip install uwsgi  # for python2

Şimdi basit bir test uygulaması oluşturalım:

app.py

from flask import Flask
from sys import version

app = Flask(__name__)

@app.route("/")
def index():
    return "Hello uWSGI from python version: <br>" + version

application = app

Şişede, uygulamanın geleneksel adı ‘app’ şeklindedir, ancak uWSGI varsayılan olarak ‘uygulama’yı arar. Bu yüzden son satırda uygulamamız için bir takma ad oluşturuyoruz.

Şimdi uygulamayı çalıştırma zamanı:

uwsgi --wsgi-file app.py --http :5000

Tarayıcınızı localhost:5000 konumuna getirerek “Merhaba uWSGI …” mesajını görmelisiniz.

Her seferinde tam komutu yazmamak için bu konfigürasyonu saklamak için bir ‘uwsgi.ini’ dosyası oluşturacağız:

uwsgi.ini

[uwsgi]
http = :9090
wsgi-file = app.py
single-interpreter = true
enable-threads = true
master = true

“http” ve “wsgi-file” seçenekleri manuel komuttakiyle aynıdır. Ancak üç seçenek daha var:

  • `tek tercüman’: Bir sonraki seçeneğe müdahale edebileceğinden bunu açmanız önerilir.

  • enable-threads: Uygulamanızda ek iş parçacıkları kullanıyorsanız bu özelliğin açılması gerekir. Şu anda kullanmıyoruz ama şimdi bunun için endişelenmemize gerek yok.

  • “master”: Ana mod, [çeşitli nedenlerle] etkinleştirilmelidir(http://stackoverflow.com/questions/20197259/what-is-uwsgi-master-mode)

Şimdi uygulamayı şu komutla çalıştırabiliriz:

uwsgi --ini uwsgi.ini

nginx’i yükleme ve uWSGI için ayarlama #

Şimdi uygulamamıza hizmet etmek için nginx’i kurmak istiyoruz.

sudo apt-get install nginx  # on debian/ubuntu

Ardından web sitemiz için bir konfigürasyon oluşturuyoruz

cd /etc/nginx/site-available  # go to the configuration for available sites
# create a file flaskconfig with your favourite editor

flaskconfig

server {
    listen 80;
    server_name localhost;

    location / {
        include uwsgi_params;
        uwsgi_pass unix:///tmp/flask.sock;
    }
}

Bu, nginx’e 80 numaralı bağlantı noktasını (http için varsayılan) dinlemesini ve kök yolunda (/) bir şey sunmasını söyler. Orada nginx’e sadece bir proxy gibi davranmasını ve her isteği /tmp/ içinde bulunan flask.sock adlı bir sokete iletmesini söylüyoruz.

Siteyi etkinleştirelim:

cd /etc/nginx/sites-enabled
sudo ln -s ../sites-available/flaskconfig .

Etkinleştirilmişse varsayılan yapılandırmayı kaldırmak isteyebilirsiniz:

# inside /etc/sites-enabled
sudo rm default

Ardından nginx’i yeniden başlatın:

sudo service nginx restart

Tarayıcınızı ’localhost’a yönlendirin ve bir hata göreceksiniz: ‘502 Bad Gateway’.

Bu, nginx’in çalıştığı ve çalıştığı ancak soketin olmadığı anlamına gelir. Öyleyse bunu oluşturalım.

‘uwsgi.ini’ dosyanıza geri dönün ve açın. Ardından şu satırları ekleyin:

socket = /tmp/flask.sock
chmod-socket = 666

İlk satır, uwsgi’ye verilen konumda bir soket oluşturmasını söyler. Soket, istekleri almak ve yanıtları geri göndermek için kullanılacaktır. Son satırda, diğer kullanıcıların (nginx dahil) o soketten okuma ve yazmalarına izin veriyoruz.

uwsgi’yi “uwsgi –ini uwsgi.ini” ile yeniden başlatın. Şimdi tarayıcınızı tekrar localhosta çevirin ve tekrar “Merhaba uWSGI” selamını göreceksiniz.

uWSGI artık uygulamaya http __ ve__ soket üzerinden hizmet verdiğinden, yanıtı ’localhost:5000’ üzerinde görebileceğinizi unutmayın. Öyleyse ini dosyasındaki http seçeneğini devre dışı bırakalım

http = :5000  # <-- remove this line and restart uwsgi

Artık uygulamaya yalnızca nginx’ten erişilebilir (veya doğrudan o soketi okuyarak :)).
