Flask-login uzantısını kullanma #

Bir yetkilendirme sistemini uygulamanın daha basit yollarından biri, flask-login uzantısını kullanmaktır. Projenin web sitesi, bu örnekte daha kısa bir versiyonu bulunan ayrıntılı ve iyi yazılmış bir hızlı başlangıç ​​içermektedir.

Genel fikir #

Uzantı, aşağıdakiler için kullanılan bir dizi işlevi ortaya çıkarır: *kullanıcıların oturum açması

  • kullanıcıların oturumu kapatması
  • bir kullanıcının oturum açıp açmadığını kontrol etme ve bu kullanıcının hangi kullanıcı olduğunu bulma

Ne yapmaz ve kendi başınıza yapmanız gerekenler:

  • kullanıcıları örneğin veritabanında saklamanın bir yolunu sağlamaz
  • örneğin kullanıcı adı ve şifre gibi kullanıcının kimlik bilgilerini kontrol etmenin bir yolunu sağlamaz

Aşağıda, her şeyin çalışmasını sağlamak için gereken minimum bir dizi adım vardır.

Auth ile ilgili tüm kodları ayrı bir modüle veya pakete yerleştirmenizi tavsiye ederim, örneğin “auth.py”. Bu şekilde gerekli sınıfları, nesneleri veya özel işlevleri ayrı ayrı oluşturabilirsiniz.

Bir ‘LoginManager’ oluşturun #

Uzantı, Flask uygulama nesnenize kaydedilmesi gereken bir LoginManager sınıfını kullanır.

from flask_login import LoginManager
login_manager = LoginManager()
login_manager.init_app(app) # app is a Flask object

Daha önce bahsedildiği gibi, ‘LoginManager’ örneğin ayrı bir dosya veya pakette global bir değişken olabilir. Ardından, Flask ​​nesnesinin oluşturulduğu dosyaya veya uygulama fabrika işlevinize alınabilir ve başlatılabilir.

Kullanıcıları yüklemek için kullanılan bir geri arama belirtin #

Bir kullanıcı normalde bir veritabanından yüklenir. Geri arama, sağlanan kimliğe karşılık gelen bir kullanıcıyı temsil eden bir nesne döndürmelidir. Kimlik geçerli değilse “Yok” döndürmelidir.

@login_manager.user_loader
def load_user(user_id):
    return User.get(user_id) # Fetch the user from the database

Bu, ‘LoginManager’ınızı oluşturmanın hemen altında yapılabilir.

Kullanıcınızı temsil eden bir sınıf #

Belirtildiği gibi, “user_loader” geri çağrısının bir kullanıcıyı temsil eden bir nesne döndürmesi gerekir. Bu tam olarak ne anlama geliyor? Bu nesne, örneğin, veritabanınızda depolanan kullanıcı nesnelerinin etrafındaki bir sarmalayıcı veya doğrudan veritabanınızdan bir model olabilir. Bu nesnenin aşağıdaki yöntemleri ve özellikleri uygulaması gerekir. Bu, geri arama, veritabanı modelinizi döndürürse, belirtilen özelliklerin ve yöntemlerin modelinize eklendiğinden emin olmanız gerektiği anlamına gelir.

  • “kimliği doğrulandı”

    This property should return True if the user is authenticated, i.e. they have provided valid credentials. You will want to ensure that the objects which represent your users returned by the user_loader callback return True for that method.

  • “et_aktif”

    This property should return True if this is an active user - in addition to being authenticated, they also have activated their account, not been suspended, or any condition your application has for rejecting an account. Inactive accounts may not log in. If you don’t have such a mechanism present return True from this method.

  • “anonim değil”

    This property should return True if this is an anonymous user. That means that your user object returned by the user_loader callback should return True.

  • get_id()

    This method must return a unicode that uniquely identifies this user, and can be used to load the user from the user_loader callback. Note that this must be a unicode - if the ID is natively an int or some other type, you will need to convert it to unicode. If the user_loader callback returns objects from the database this method will most likely return the database ID of this particular user. The same ID should of course cause the user_loader callback to return the same user later on.

Kendiniz için işleri kolaylaştırmak istiyorsanız (**aslında tavsiye edilir), “user_loader” geri çağrısı tarafından döndürülen nesnedeki “UserMixin” öğesinden (muhtemelen bir veritabanı modeli) devralabilirsiniz. Bu yöntem ve özelliklerin varsayılan olarak nasıl uygulandığını bu karışımda burada görebilirsiniz.

Kullanıcıların oturum açması #

Uzantı, kullanıcı tarafından girilen kullanıcı adı ve şifrenin doğrulanmasını size bırakır. Aslında, bir kullanıcı adı ve şifre kombinasyonu veya başka bir mekanizma kullanmanız, uzantının umrunda değil. Bu, kullanıcıların kullanıcı adı ve parola kullanarak oturum açmasına bir örnektir.

@app.route('/login', methods=['GET', 'POST'])
def login():
    # Here we use a class of some kind to represent and validate our
    # client-side form data. For example, WTForms is a library that will
    # handle this for us, and we use a custom LoginForm to validate.
    form = LoginForm()
    if form.validate_on_submit():
        # Login and validate the user.
        # user should be an instance of your `User` class
        login_user(user)

        flask.flash('Logged in successfully.')

        next = flask.request.args.get('next')
        # is_safe_url should check if the url is safe for redirects.
        # See http://flask.pocoo.org/snippets/62/ for an example.
        if not is_safe_url(next):
            return flask.abort(400)

        return flask.redirect(next or flask.url_for('index'))
    return flask.render_template('login.html', form=form)

Genel olarak, kullanıcıların oturum açması, login_user öğesini çağırarak ve daha önce bahsedilen kullanıcınızı temsil eden bir nesnenin örneğini ona ileterek gerçekleştirilir. Gösterildiği gibi, bu genellikle kullanıcıyı veritabanından aldıktan ve kimlik bilgilerini doğruladıktan sonra gerçekleşir, ancak kullanıcı nesnesi bu örnekte sihirli bir şekilde görünür.

Bir kullanıcıya giriş yaptım, şimdi ne olacak? #

“user_loader” geri çağrısı tarafından döndürülen nesneye birden çok yoldan erişilebilir.

  • Şablonlarda:

    The extension automatically injects it under the name current_user using a template context processor. To disable that behaviour and use your custom processor set add_context_processor=False in your LoginManager constructor.

      {% if current_user.is_authenticated %}
    Hi {{ current_user.name }}!
  {% endif %}

  • Python kodunda:

    The extension provides a request-bound object called current_user.

      from flask_login import current_user    

  @app.route("/hello")
  def hello():
      # Assuming that there is a name property on your user object
      # returned by the callback
      if current_user.is_authenticated:
          return 'Hello %s!' % current_user.name 
      else:
          return 'You are not logged in!'

  • Bir dekoratör kullanarak erişimi hızlı bir şekilde sınırlama Erişimi hızlı bir şekilde sınırlamak için bir login_required dekoratör kullanılabilir.

      from flask_login import login_required

  @app.route("/settings")
  @login_required
  def settings():
      pass

Kullanıcıların oturumunu kapatma #

Kullanıcılar logout_user() çağrılarak oturumu kapatılabilir. Kullanıcı oturum açmamış olsa bile, “@login_required” dekoratörünün büyük olasılıkla atlanabilmesi için bunu yapmanın güvenli olduğu görülüyor.

@app.route("/logout")
@login_required
def logout():
    logout_user()
    return redirect(somewhere)

Bir kullanıcı oturum açmadıysa ve ‘current_user’ nesnesine erişirsem ne olur? #

Varsayılan olarak bir AnonymousUserMixin döndürülür:

  • “is_active” ve “is_authenticated”, “Yanlış”
  • “anonim değil”, “Doğru"dur
  • “get_id()”, “Yok” değerini döndürür

Anonim kullanıcılar için farklı bir nesne kullanmak için, “LoginManager"ınıza anonim kullanıcılar oluşturan bir çağrılabilir (sınıf veya fabrika işlevi) sağlayın:

login_manager.anonymous_user = MyAnonymousUser

Sıradaki ne? #

Bu, uzantının temel girişini tamamlar. Yapılandırma ve ek seçenekler hakkında daha fazla bilgi edinmek için resmi kılavuzu okuyun şiddetle tavsiye edilir.

Oturum açma oturumunun zaman aşımına uğraması #

Belirli bir süre sonra oturum açma zaman aşımına uğramak için iyi bir uygulamadır, bunu Flask-Login ile başarabilirsiniz.

from flask import Flask, session
from datetime import timedelta
from flask_login import LoginManager, login_require, login_user, logout_user

# Create Flask application

app = Flask(__name__) 

# Define Flask-login configuration 

login_mgr = LoginManager(app)
login_mgr.login_view = 'login'
login_mgr.refresh_view = 'relogin'
login_mgr.needs_refresh_message = (u"Session timedout, please re-login")
login_mgr.needs_refresh_message_category = "info"


@app.before_request
def before_request():
    session.permanent = True
    app.permanent_session_lifetime = timedelta(minutes=5)

Varsayılan oturum ömrü 31 gündür, kullanıcının zaman aşımı durumunda oturum açma yenileme görünümünü belirtmesi gerekir.

app.permanent_session_lifetime = timedelta(minutes=5)

Yukarıdaki satır, kullanıcıyı her 5 dakikada bir yeniden giriş yapmaya zorlayacaktır.
