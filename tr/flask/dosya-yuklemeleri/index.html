Sözdizimi #

  • request.files[’name’] # tek gerekli dosya
  • request.files.get(’name’) # Gönderilmediyse yok
  • request.files.getlist(’name’) # gönderilen sıfır veya daha fazla dosyanın listesi
  • CombinedMultiDict((request.files, request.form)) # form ve dosya verilerini birleştir

Dosyaları Yükleme #

HTML Formu #

  • Bir dosya tipi giriş kullanın ve tarayıcı, kullanıcının yüklenecek bir dosya seçmesine izin veren bir alan sağlayacaktır.
  • Yalnızca ‘post’ yöntemine sahip formlar dosya verilerini gönderebilir.
  • Formun enctype=multipart/form-data niteliğini ayarladığınızdan emin olun. Aksi takdirde dosyanın adı gönderilir ancak dosyanın verileri gönderilmez.
  • Tek alan için birden fazla dosya seçilmesine izin vermek için girişteki ‘multiple’ özniteliğini kullanın.
<form method=post enctype=multipart/form-data>
    <!-- single file for the "profile" field -->
    <input type=file name=profile>
    <!-- multiple files for the "charts" field -->
    <input type=file multiple name=charts>
    <input type=submit>
</form>

Python İstekleri #

Requests, HTTP istekleri yapmak için güçlü bir Python kitaplığıdır. Tarayıcı olmadan dosyaları göndermek için onu (veya diğer araçları) kullanabilirsiniz.

  • İkili modda okumak için dosyaları açın.
  • Dosyaların aldığı birden çok veri yapısı vardır. Bu, yukarıdaki form gibi birden çok dosyaya izin veren (name, data) demetlerinin bir listesini gösterir.
import requests

with open('profile.txt', 'rb') as f1, open('chart1.csv', 'rb') as f2, open('chart2.csv', 'rb') as f3:
    files = [
        ('profile', f1),
        ('charts', f2),
        ('charts', f3)
    ]
    requests.post('http://localhost:5000/upload', files=files)

Bu, kapsamlı bir liste değildir. Favori aracınızı kullanan örnekler veya daha karmaşık senaryolar için o araca ilişkin belgelere bakın.

WTForms ve Flask-WTF’ye veri aktarma #

WTForms, bir dosya türü girdisi oluşturmak için bir ‘FileField’ sağlar. Yüklenen verilerle özel bir şey yapmaz. Ancak Flask, form verilerini (‘request.form’) ve dosya verilerini (‘request.files’) böldüğü için, formu oluştururken doğru verileri ilettiğinizden emin olmanız gerekir. İkisini WTForms’un anlayacağı tek bir yapıda birleştirmek için bir “CombinedMultiDict” kullanabilirsiniz.

form = ProfileForm(CombinedMultiDict((request.files, request.form)))

Flask-WTF kullanıyorsanız, Flask ve WTForms’u entegre etmek için bir uzantı, doğru verileri iletmek sizin için otomatik olarak işlenecektir.

WTForms’daki bir hata nedeniyle, birden fazla yüklenmiş olsa bile her alan için yalnızca bir dosya bulunacaktır. Daha fazla ayrıntı için bu sorun konusuna bakın. 3.0’da düzeltilecek.

Yüklemeleri sunucuya kaydet #

Yüklenen dosyalar, alan adlarını dosya nesnelerine eşleyen bir ‘MultiDict’ olan ‘request.files’ içinde bulunur. Aynı alan adıyla birden fazla dosya yüklenmişse, “[]” veya “get” yerine “getlist” kullanın.

request.files['profile']  # single file (even if multiple were sent)
request.files.getlist('charts')  # list of files (even if one was sent)

“request.files” içindeki nesneler, dosyayı yerel olarak kaydeden bir “kaydetme” yöntemine sahiptir. Dosyaları kaydetmek için ortak bir dizin oluşturun.

“filename” özelliği, dosyanın yüklendiği addır. Bu, istemci tarafından isteğe bağlı olarak ayarlanabilir, bu nedenle, farklı kaydetmek üzere geçerli ve güvenli bir ad oluşturmak için onu `secure_filename’ yönteminden geçirin. Bu, adın benzersiz olmasını sağlamaz, bu nedenle, bunu tespit etmek için fazladan çalışma yapmadığınız sürece mevcut dosyaların üzerine yazılacaktır.

import os
from flask import render_template, request, redirect, url_for
from werkzeug import secure_filename

# Create a directory in a known location to save files to.
uploads_dir = os.path.join(app.instance_path, 'uploads')
os.makedirs(uploads_dir, exists_ok=True)

@app.route('/upload', methods=['GET', 'POST'])
def upload():
    if request.method == 'POST':
        # save the single "profile" file
        profile = request.files['profile']
        profile.save(os.path.join(uploads_dir, secure_filename(profile.filename)))

        # save each "charts" file
        for file in request.files.getlist('charts'):
            file.save(os.path.join(uploads_dir, secure_filename(file.name)))

        return redirect(url_for('upload'))

    return render_template('upload.html')

PARSE CSV DOSYASINI KAYDETMEDEN FLASK İÇİNDE SÖZLÜK LİSTESİ OLARAK YÜKLEYİN #

Geliştiricilerin genellikle kullanıcıların bir CSV dosyası yüklemesine izin veren web siteleri tasarlaması gerekir. Genellikle gerçek CSV dosyasını kaydetmek için hiçbir neden yoktur, çünkü veriler yüklendikten sonra işlenecek ve/veya bir veritabanında saklanacaktır. Ancak, çoğu olmasa da çoğu, PYTHON CSV verilerini ayrıştırma yöntemleri, verilerin bir dosya olarak okunmasını gerektirir. Web geliştirme için FLASK kullanıyorsanız, bu biraz başınızı ağrıtabilir.

CSV’mizin bir başlık satırı olduğunu ve aşağıdaki gibi göründüğünü varsayalım:

h1,h2,h3
'yellow','orange','blue'
'green','white','black'
'orange','pink','purple'

Şimdi, bir dosya yüklemek için html formunun aşağıdaki gibi olduğunu varsayalım:

<form action="upload.html" method="post" enctype="multipart/form-data">
    <input type="file" name="fileupload" id="fileToUpload">
    <input type="submit" value="Upload File" name="submit">
</form>

Hiç kimse tekerleği yeniden icat etmek istemediğinden, FLASK betiğinize **csv’yi IMPORT AL‘a karar verdiniz. Kişilerin sütunları doğru sırada olan csv dosyasını karşıya yükleyeceğinin garantisi yoktur. CSV dosyasının bir başlık satırı varsa, csv.DictReader yönteminin yardımıyla, CSV dosyasını başlık satırındaki girişler tarafından anahtarlanmış bir sözlük listesi olarak okuyabilirsiniz. Ancak, csv.DictReader bir dosyaya ihtiyaç duyar ve doğrudan dizeleri kabul etmez. Yüklenen dosyayı önce kaydetmek, yeni dosya adını ve konumunu almak, csv.DictReader kullanarak açmak ve ardından dosyayı silmek için FLASK yöntemlerini kullanmanız gerektiğini düşünebilirsiniz. Biraz israf gibi görünüyor.

Neyse ki, dosya içeriğini bir dize olarak alabilir ve ardından dizeyi sonlandırılmış satırlara bölebiliriz. csv yöntemi csv.DictReader bunu bir dosyanın yerine kabul eder. Aşağıdaki kod, dosyayı geçici olarak kaydetmeden bunun nasıl gerçekleştirilebileceğini gösterir.

@application.route('upload.html',methods = ['POST'])
def upload_route_summary():
    if request.method == 'POST':

        # Create variable for uploaded file
        f = request.files['fileupload']  

        #store the file contents as a string
        fstring = f.read()

        csv_dicts = [{k: v for k, v in row.items()} for row in csv.DictReader(fstring.splitlines(), skipinitialspace=True)]

        #do something list of dictionaries
    return "success"

csv_dicts değişkeni artık aşağıdaki sözlük listesidir:

   csv_dicts = 
    [
        {'h1':'yellow','h2':'orange','h3':'blue'},
        {'h1':'green','h2':'white','h3':'black'},
        {'h1':'orange','h2':'pink','h3':'purple'}
    ]

PYTHON’da yeniyseniz, aşağıdaki gibi verilere erişebilirsiniz:

csv_dicts[1]['h2'] = 'white'
csv_dicts[0]['h3'] = 'blue'

Diğer çözümler, io modülünü içe aktarmayı ve io.Stream yöntemini kullanmayı içerir. Bunun daha basit bir yaklaşım olduğunu hissediyorum. Kodun izlenmesinin io yöntemini kullanmaktan biraz daha kolay olduğuna inanıyorum. Bu yaklaşım, yüklenen bir CSV dosyasını ayrıştırma örneğine özgüdür.
