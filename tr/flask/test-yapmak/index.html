Flask’ta uygulanan bir JSON API’sini test etme #

Bu örnek, [pytest kullanarak bir Flask uygulamasının nasıl test edileceğini](https://www.wikiod.com/tr/flask/test-yapmak#Merhaba Dünya uygulamamızı test etme) bildiğinizi varsayar

Aşağıda, “a” ve “b” tamsayı değerlerine sahip bir JSON girişi alan bir API bulunmaktadır. {"a": 1, "b": 2}, bunları toplar ve bir JSON yanıtında “a + b” toplamını döndürür; ör. {"toplam": 3}.

# hello_add.py
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/add', methods=['POST'])
def add():
    data = request.get_json()
    return jsonify({'sum': data['a'] + data['b']})

Bu API’yi “pytest” ile test etme #

‘pytest’ ile test edebiliriz

# test_hello_add.py
from hello_add import app
from flask import json

def test_add():        
    response = app.test_client().post(
        '/add',
        data=json.dumps({'a': 1, 'b': 2}),
        content_type='application/json',
    )

    data = json.loads(response.get_data(as_text=True))

    assert response.status_code == 200
    assert data['sum'] == 3

Şimdi testi py.test komutu ile çalıştırın.

Hello World uygulamamızı test etme #

giriiş #

Bu minimalist örnekte, pytest kullanarak Hello World uygulamamızın gerçekten de “Hello, World!” döndürdüğünü test edeceğiz. URL / üzerinde bir GET isteği ile vurulduğunda 200’lük bir HTTP OK durum koduyla

Önce sanal ortamımıza pytest’i yükleyelim

pip install pytest

Ve sadece referans olarak, bu merhaba dünya uygulamamız:

# hello.py
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello, World!'

Testi tanımlama #

“hello.py” ile birlikte “py.test” tarafından keşfedilecek olan “test_hello.py” adında bir test modülü tanımlıyoruz.

# test_hello.py
from hello import app

def test_hello():
    response = app.test_client().get('/')

    assert response.status_code == 200
    assert response.data == b'Hello, World!'

Sadece gözden geçirmek gerekirse, bu noktada ağaç komutu ile elde edilen proje yapımız şu şekildedir:

.
├── hello.py
└── test_hello.py

Testi çalıştırma #

Şimdi bu testi, test_hello.py ve içindeki test fonksiyonunu otomatik olarak keşfedecek olan py.test komutu ile çalıştırabiliriz.

$ py.test

Bazı çıktılar ve 1 testin geçtiğine dair bir gösterge görmelisiniz, örn.

=== test session starts ===
collected 1 items 
test_hello.py .
=== 1 passed in 0.13 seconds ===

Flask-Testing kullanarak testlerinizde oturum değişkenlerine erişme ve bunları değiştirme #

Web uygulamalarının çoğu, bazı önemli bilgileri depolamak için oturum nesnesini kullanır. Bu örnekler, Flask-Testing kullanarak böyle bir uygulamayı nasıl test edebileceğinizi gösterir. Tam çalışma örneği github adresinde de mevcuttur.

Bu yüzden önce sanal ortamınıza Flask-Testing’i kurun

pip install flask_testing

Oturum nesnesini kullanabilmek için gizli anahtarı ayarlamanız gerekir.

app.secret_key = 'my-seCret_KEy'

Bunun gibi oturum değişkenlerinde bazı verileri depolamanız gereken uygulama işlevinizde olduğunu düşünelim.

@app.route('/getSessionVar', methods=['GET', 'POST'])
def getSessionVariable():
  if 'GET' == request.method:
    session['sessionVar'] = 'hello'
  elif 'POST' == request.method:
    session['sessionVar'] = 'hi'
  else:
    session['sessionVar'] = 'error'

  return 'ok'

Bu işlevi test etmek için flask_testing’i içe aktarabilir ve test sınıfınızın flask_testing.TestCase’i devralmasına izin verebilirsiniz. Ayrıca gerekli tüm kütüphaneleri içe aktarın

import flask
import unittest
import flask_testing
from myapp.run import app
    
class TestMyApp(flask_testing.TestCase):

Teste başlamadan önce create_app işlevini uygulamak çok önemlidir, aksi takdirde istisna olacaktır.

  def create_app(self):
    return app

Uygulamanızın istediğiniz gibi çalıştığını test etmek için birkaç olanağınız var. Eğer işlevinizin bir oturum değişkenine belirli değerler ayarladığından emin olmak istiyorsanız, bağlamı etrafta tutabilir ve flask.session‘a erişebilirsiniz.

def testSession1(self):
    with app.test_client() as lTestClient:
      lResp= lTestClient.get('/getSessionVar')
      self.assertEqual(lResp.status_code, 200)
      self.assertEqual(flask.session['sessionVar'], 'hello')

Bir diğer yararlı hile, bir sonraki test işlevinde olduğu gibi GET ve POST yöntemleri arasında ayrım yapmaktır.

def testSession2(self):
    with app.test_client() as lTestClient:
      lResp= lTestClient.post('/getSessionVar')
      self.assertEqual(lResp.status_code, 200)
      self.assertEqual(flask.session['sessionVar'], 'hi')

Şimdi, işlevinizin bir oturum değişkeninin ayarlanmasını beklediğini ve bunun gibi belirli değerlere farklı tepki verdiğini hayal edin.

@app.route('/changeSessionVar')
def changeSessionVariable():
  if session['existingSessionVar'] != 'hello':
    raise Exception('unexpected session value of existingSessionVar!')

  session['existingSessionVar'] = 'hello world'
  return 'ok'

Bu işlevi test etmek için oturum işlemi denen şeyi kullanmanız ve oturumu test istemcisi bağlamında açmanız gerekir. Bu fonksiyon Flask 0.8‘den beri mevcuttur.

def testSession3(self):
    with app.test_client() as lTestClient:
      #keep the session
      with lTestClient.session_transaction() as lSess:
        lSess['existingSessionVar'] = 'hello'

      #here the session is stored
      lResp = lTestClient.get('/changeSessionVar')
      self.assertEqual(lResp.status_code, 200)
      self.assertEqual(flask.session['existingSessionVar'], 'hello world')

Unittest için testleri çalıştırmak her zamanki gibi

if __name__ == "__main__":
    unittest.main()

Ve komut satırında

python tests/test_myapp.py

Testlerinizi çalıştırmanın bir başka güzel yolu da şu şekilde Unittest Discovery kullanmaktır:

python -m unittest discover -s tests
