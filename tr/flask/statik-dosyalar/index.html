Statik Dosyaları Kullanma #

Web uygulamaları genellikle CSS veya JavaScript dosyaları gibi statik dosyalar gerektirir. Bir Flask uygulamasında statik dosyaları kullanmak için paketinizde veya modülünüzün yanında ‘static’ adında bir klasör oluşturun ve bu klasör uygulamada ‘/static’ konumunda bulunacaktır.

Şablonları kullanmak için örnek bir proje yapısı aşağıdaki gibidir:

MyApplication/
    /static/
        /style.css
        /script.js
    /templates/
        /index.html
    /app.py

app.py, şablon oluşturma özelliğine sahip temel bir Flask örneğidir.

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def index():
    return render_template('index.html')

Statik CSS ve JavaScript dosyasını index.html şablonunda kullanmak için özel ‘statik’ uç nokta adını kullanmamız gerekir:

{{url_for('static', filename = 'style.css')}}

Dolayısıyla, index.html şunları içerebilir:

<html>
    <head>
        <title>Static File</title>
        <link href="{{url_for('static', filename = 'style.css')}}" rel="stylesheet">
        <script src="{{url_for('static', filename = 'script.js')}}"></script>
    </head>
    <body>
        <h3>Hello World!</h3>
    </body>
</html>

app.py’yi çalıştırdıktan sonra web sayfasını http://localhost:5000/ içinde göreceğiz.

Üretimdeki Statik Dosyalar (ön uç web sunucusu tarafından sunulur) #

Flask’ın yerleşik web sunucusu, statik varlıklara hizmet edebilir ve bu, geliştirme için iyi çalışır. Ancak, Flask uygulamasına hizmet etmek için uWSGI veya Gunicorn gibi bir şey kullanan üretim dağıtımları için, statik dosyaları sunma görevi, genellikle ön uç web sunucusuna (Nginx, Apache, vb.) yüklenen bir görevdir. Bu, özellikle tüm statik varlıklar tek bir klasörde olduğunda, daha küçük uygulamalarla küçük/kolay bir iştir; ancak daha büyük uygulamalar ve/veya statik varlıklar sağlayan Flask eklenti(ler)ini kullananlar için, o zaman tüm bu dosyaların konumlarını hatırlamak ve bunları manuel olarak tek bir dizine kopyalamak/toplamak zor olabilir. Bu belge, bu görevi basitleştirmek için Flask-Collect eklentisinin nasıl kullanılacağını gösterir.

Bu belgelerin odak noktasının statik varlıkların toplanması olduğunu unutmayın. Bu işlevi göstermek için bu örnek, statik varlıklar sağlayan Flask-Bootstrap eklentisini kullanır. Ayrıca komut satırı görevleri oluşturma sürecini basitleştirmek için kullanılan Flask-Script eklentisini kullanır. Bu eklentilerin hiçbiri bu belge için kritik değildir, sadece işlevselliği göstermek için burada kullanılmaktadırlar. Flask-Script kullanmamayı seçerseniz, ’toplama’ komutunu çağırmanın alternatif yolları için Flask-Collect belgelerini incelemek isteyeceksiniz.

Ayrıca, bu statik varlıklara hizmet etmek için ön uç web sunucunuzun yapılandırmasının bu belgenin kapsamı dışında olduğunu unutmayın, Nginx kullanarak bazı örneklere göz atmak isteyeceksiniz. nasıl-deploy-flask-web-applications-using-uwsgi-behind-nginx-on-centos-6-4) ve [Apache](http://stackoverflow.com/questions/24739925/serving-static-files -through-apache) daha fazla bilgi için. Bu örnekte Flask-Collect’in sizin için oluşturacağı merkezi dizine “/static” ile başlayan URL’leri takma ad vereceğinizi söylemek yeterlidir.

Uygulama şu şekilde yapılandırılmıştır:

/manage.py - The app management script, used to run the app, and to collect static assets
/app/ - this folder contains the files that are specific to our app
    | - __init__.py - Contains the create_app function
    | - static/ - this folder contains the static files for our app.
        | css/styles.css - custom styles for our app (we will leave this file empty)
        | js/main.js - custom js for our app (we will leave this file empty)
    | - templates/index.html - a simple page that extends the Flask-Bootstrap template

  1. Öncelikle sanal ortamınızı oluşturun ve gerekli paketleri kurun: (sizin-sanalenv) $ pip flask flask-script flask-bootstrap flask-collect yükleyin

  2. Yukarıda açıklanan dosya yapısını oluşturun:

    $ touch manage.py; mkdir -p app/{static/{css,js},templates}; touch app/{init.py,static/{css/styles.css,js/main.js}}

  3. “manage.py”, “app/init.py” ve “app/templates/index.html” dosyalarının içeriğini belirleyin:

    manage.py #

    #!/usr/bin/env python import os from flask_script import Manager, Server from flask import current_app from flask_collect import Collect from app import create_app

    class Config(object): # CRITICAL CONFIG VALUE: This tells Flask-Collect where to put our static files! # Standard practice is to use a folder named “static” that resides in the top-level of the project directory. # You are not bound to this location, however; you may use basically any directory that you wish. COLLECT_STATIC_ROOT = os.path.dirname(file) + ‘/static’ COLLECT_STORAGE = ‘flask_collect.storage.file’

    app = create_app(Config)

    manager = Manager(app) manager.add_command(‘runserver’, Server(host=‘127.0.0.1’, port=5000))

    collect = Collect() collect.init_app(app)

    @manager.command def collect(): “““Collect static from blueprints. Workaround for issue: https://github.com/klen/Flask-Collect/issues/22"”” return current_app.extensions[‘collect’].collect()

    if name == “main”: manager.run()

# app/__init__.py
from flask import Flask, render_template
from flask_collect import Collect
from flask_bootstrap import Bootstrap

def create_app(config):
  app = Flask(__name__)
  app.config.from_object(config)

  Bootstrap(app)
  Collect(app)

  @app.route('/')
  def home():
    return render_template('index.html')

  return app

# app/templates/index.html
{% extends "bootstrap/base.html" %}
{% block title %}This is an example page{% endblock %}

{% block navbar %}
<div class="navbar navbar-fixed-top">
  <!-- ... -->
</div>
{% endblock %}

{% block content %}
  <h1>Hello, Bootstrap</h1>
{% endblock %}

  1. Bu dosyalar yerindeyken, artık uygulamayı çalıştırmak için yönetim komut dosyasını kullanabilirsiniz:

    $ ./manage.py runserver # visit http://localhost:5000 to verify that the app works correctly.

  2. Şimdi, statik varlıklarınızı ilk kez toplamak için. Bunu yapmadan önce, uygulamanızın en üst seviyesinde bir ‘static/’ klasörü OLMAMALIDIR; Burası, Flask-Collect’in uygulamanızdan toplayacağı tüm statik dosyaları ve kullanıyor olabileceğiniz çeşitli eklentileri yerleştireceği yerdir. Uygulamanızın en üst düzeyinde bir “statik/” klasörünüz * varsa*, devam etmeden önce onu tamamen silmelisiniz, çünkü temiz bir sayfa ile başlamak Flask-Collect’in ne yaptığına tanık olmanın/anlamanın kritik bir parçasıdır. Bu talimatın günlük kullanım için geçerli olmadığını unutmayın, sadece Flask-Collect’in bu dizini sizin için oluşturacağını ve ardından oraya bir sürü dosya yerleştireceğini göstermek içindir.

Bununla birlikte, statik varlıklarınızı toplamak için aşağıdaki komutu çalıştırabilirsiniz:

$ ./manage.py collect

Bunu yaptıktan sonra, Flask-Collect’in bu üst düzey static/ klasörünü oluşturduğunu ve aşağıdaki dosyaları içerdiğini görmelisiniz:

İşte bu kadar: uygulamanızın CSS’sinde veya JavaScript’inde her düzenleme yaptığınızda veya statik varlıklar sağlayan bir Flask eklentisini güncellediğinizde (bu örnekteki Flask-Bootstrap gibi) “collect” komutunu kullanın.
