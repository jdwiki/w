Verileri Çöz #

Bu örnek, uygulamanızın görünümünü oluşturmadan önce bir hizmetten alınan verileri nasıl çözebileceğinizi gösterecektir.

Yazma sırasında açısal/yönlendirici 3.0.0-beta.2 kullanır

users.service.ts

...
import { Http, Response } from '@angular/http';
import { Observable } from 'rxjs/Rx';
import { User } from './user.ts';



@Injectable()
export class UsersService {

    constructor(public http:Http) {}

    /**
     * Returns all users
     * @returns {Observable<User[]>}
     */
    index():Observable<User[]> {

        return this.http.get('http://mywebsite.com/api/v1/users')
            .map((res:Response) => res.json());
    }

    /**
     * Returns a user by ID
     * @param id
     * @returns {Observable<User>}
     */
    get(id:number|string):Observable<User> {

        return this.http.get('http://mywebsite.com/api/v1/users/' + id)
            .map((res:Response) => res.json());
    }
}

users.resolver.ts

...
import { UsersService } from './users.service.ts';
import { Observable } from 'rxjs/Rx';
import {
    Resolve,
    ActivatedRouteSnapshot,
    RouterStateSnapshot
} from "@angular/router";




@Injectable()
export class UsersResolver implements Resolve<User[] | User> {

    // Inject UsersService into the resolver
    constructor(private service:UsersService) {}

    resolve(route:ActivatedRouteSnapshot, state:RouterStateSnapshot):Observable<User[] | User> {
        // If userId param exists in current URL, return a single user, else return all users
        // Uses brackets notation to access `id` to suppress editor warning, may use dot notation if you create an interface extending ActivatedRoute with an optional id? attribute
        if (route.params['id']) return this.service.get(route.params['id']);
        return this.service.index();
    }
}

users.component.ts

Bu, tüm kullanıcıların listesini içeren bir sayfa bileşenidir. Kullanıcı ayrıntı sayfası bileşeni için benzer şekilde çalışır, “data.users” yerine “data.user” veya app.routes.ts‘de tanımlanan herhangi bir anahtarla değiştirin (aşağıya bakın)

...
import { ActivatedRoute} from "@angular/router";



@Component(...)
export class UsersComponent {

    users:User[];


    constructor(route: ActivatedRoute) {
        route.data.subscribe(data => {
            // data['Match key defined in RouterConfig, see below']
            this.users = data.users;
        });
    }

    /**
     * It is not required to unsubscribe from the resolver as Angular's HTTP
     * automatically completes the subscription when data is received from the server
     */
}

app.routes.ts

...
import { UsersResolver } from './resolvers/users.resolver';



export const routes:RouterConfig = <RouterConfig>[
    ...
    {
        path: 'user/:id',
        component: UserComponent,
        resolve: {
            // hence data.user in UserComponent
            user: UsersResolver
        }
    },
    {
        path: 'users',
        component: UsersComponent,
        resolve: {
            // hence data.users in UsersComponent, note the pluralisation
            users: UsersResolver
        }
    },
    ...
]
...

app.resolver.ts

İsteğe bağlı olarak birden çok çözümleyiciyi bir araya toplayın.

Bu hizmetlerin küresel olarak sunulacağını ve artık bunları herhangi bir bileşenin “sağlayıcılarında” bildirmeniz gerekmeyeceğini unutmayın. Bellek sızıntısını önlemek için herhangi bir abonelikten çıktığınızdan emin olun

import { UsersService } from './users.service';
import { UsersResolver } from './users.resolver';

export const ROUTE_RESOLVERS = [
    ...,
    UsersService,
    UsersResolver
]

Çocuklarla Yönlendirme #

Orijinal belgelerin aksine, bunun alt yolları app.routing.ts veya app.module.ts dosyası (tercihinize bağlı olarak) içine düzgün şekilde yerleştirmenin yolu olduğunu buldum. Bu yaklaşım, WebPack veya SystemJS kullanıldığında işe yarar.

Aşağıdaki örnek ev, ev/sayaç ve ev/sayaç/getirme verileri için rotaları göstermektedir. İlk ve son rotalar yönlendirme örnekleridir. Son olarak, örneğin sonunda, içe aktarılacak Rotayı ayrı bir dosyada dışa aktarmanın uygun bir yolu vardır. Örn. app.module.ts

Daha fazla açıklamak için, Angular, ana yolu temsil etmek için, ana bileşeni içeren alt dizisinde yolsuz bir rotanız olmasını gerektirir. Biraz kafa karıştırıcı ama bir alt rota için boş bir URL düşünürseniz, esasen ana rota ile aynı URL’ye eşit olacaktır.

import { NgModule } from "@angular/core";
import { RouterModule, Routes } from "@angular/router";

import { HomeComponent } from "./components/home/home.component";
import { FetchDataComponent } from "./components/fetchdata/fetchdata.component";
import { CounterComponent } from "./components/counter/counter.component";

const appRoutes: Routes = [
    {
        path: "",
        redirectTo: "home",
        pathMatch: "full"
    },
    {
        path: "home",            
        children: [
            {
                path: "",
                component: HomeComponent
            },
            {
                path: "counter",                    
                children: [
                    {
                        path: "",
                        component: CounterComponent
                    },
                    {
                        path: "fetch-data",
                        component: FetchDataComponent                            
                    }
                ]
            }
        ]
    },
    {
        path: "**",
        redirectTo: "home"
    }
];

@NgModule({
    imports: [
        RouterModule.forRoot(appRoutes)
    ],
    exports: [
        RouterModule
    ]
})
export class AppRoutingModule { }

Siraj ile Harika Örnek ve Açıklama

Temel Yönlendirme #

Yönlendirici, uygulama ile kullanıcı etkileşimlerine dayalı olarak bir görünümden diğerine gezinmeyi sağlar.

Angular 2’de temel yönlendirmeyi uygulama adımları aşağıdadır -

Temel önlem: Etikete sahip olduğunuzdan emin olun

    <base href='/'>

index.html dosyanızdaki head etiketinizin altındaki ilk çocuk olarak. Bu etiket, uygulama klasörünüzün uygulama kökü olduğunu söyler. Angular 2 daha sonra bağlantılarınızı düzenlemeyi bilirdi.

İlk adım, package.json’daki doğru/en son yönlendirme bağımlılıklarını işaret edip etmediğinizi kontrol etmektir -

"dependencies": {
  ......
  "@angular/router": "3.0.0-beta.1",
  ......
}

İkinci adım rotayı sınıf tanımına göre tanımlamaktır -

class Route {
  path : string
  pathMatch : 'full'|'prefix'
  component : Type|string
  .........
}

Bir rota dosyasında (‘route/routes.ts’), farklı yönlendirme yolları için yapılandırmanız gereken tüm bileşenleri içe aktarın. Boş yol, görünümün varsayılan olarak yüklendiği anlamına gelir. Yoldaki “:”, yüklenen bileşene geçirilen dinamik parametreyi gösterir.

Yollar, bağımlılık enjeksiyonu yoluyla uygulamaya sunulur. ProviderRouter yöntemi, yönlendirici belirli görevleri çağırmak için bileşenlere enjekte edilebilmesi için parametre olarak RouterConfig ile çağrılır.

import { provideRouter, RouterConfig } from '@angular/router';
import { BarDetailComponent } from '../components/bar-detail.component';
import { DashboardComponent } from '../components/dashboard.component';
import { LoginComponent } from '../components/login.component';
import { SignupComponent } from '../components/signup.component';

export const appRoutes: RouterConfig = [
  { path: '', pathMatch: 'full', redirectTo: 'login' },
  { path: 'dashboard', component: DashboardComponent },
  { path: 'bars/:id', component: BarDetailComponent },
  { path: 'login', component: LoginComponent },
  { path: 'signup',   component: SignupComponent }
];

export const APP_ROUTER_PROVIDER = [provideRouter(appRoutes)];

Üçüncü adım, rota sağlayıcısının ön yüklemesini yapmaktır.

main.ts dosyanızda (Herhangi bir ad olabilir. Temel olarak, ana dosyanız systemjs.config içinde tanımlanmalıdır)

import { bootstrap } from '@angular/platform-browser-dynamic';
import { AppComponent } from './components/app.component';
import { APP_ROUTER_PROVIDER } from "./routes/routes";

bootstrap(AppComponent, [ APP_ROUTER_PROVIDER ]).catch(err => console.error(err));

Dördüncü adım, erişilen yola göre yönlendirici bileşenlerini yüklemek/göstermektir. yönergesi, bileşenin nereye yükleneceğini açısal olarak belirtmek için kullanılır. kullanmak için ROUTER_DIRECTIVES dosyasını içe aktarın.

import { ROUTER_DIRECTIVES } from '@angular/router';

@Component({
  selector: 'demo-app',
  template: `
    ....................................
    <div>
      <router-outlet></router-outlet>
    </div>
    ....................................
  `,
  // Add our router directives we will be using
  directives: [ROUTER_DIRECTIVES]
})

Beşinci adım, diğer yolları birbirine bağlamaktır. Varsayılan olarak, RouterOutlet, RouterConfig’de boş yolun belirtildiği bileşeni yükleyecektir. RouterLink yönergesi, rotalara bağlı bileşenleri yüklemek için html bağlantı etiketi ile birlikte kullanılır. RouterLink, bağlantılar oluşturmak için kullanılan href niteliğini oluşturur. Örn:

import { Component } from '@angular/core';
import { ROUTER_DIRECTIVES } from '@angular/router';

@Component({
  selector: 'demo-app',
  template: `
    <a [routerLink]="['/login']">Login</a>
    <a [routerLink]="['/signup']">Signup</a>      
    <a [routerLink]="['/dashboard']">Dashboard</a>
    <div>
      <router-outlet></router-outlet>
    </div>
  `,
  // Add our router directives we will be using
  directives: [ROUTER_DIRECTIVES]
})
export class AppComponent { }

Şimdi, statik yola yönlendirme konusunda iyiyiz. RouterLink, yol ile birlikte ekstra parametreler ileterek dinamik yolu da destekler.

{ Bileşen }‘i ‘@angular/core’dan içe aktarın; { ROUTER_DIRECTIVES } ‘@angular/router’dan içe aktarın;

@Component({
  selector: 'demo-app',
  template: `
        <ul>
          <li *ngFor="let bar of bars | async">
            <a [routerLink]="['/bars', bar.id]">
              {{bar.name}}
            </a>
          </li>
        </ul>
    <div>
      <router-outlet></router-outlet>
    </div>
  `,
  // Add our router directives we will be using
  directives: [ROUTER_DIRECTIVES]
})
export class AppComponent { }

RouterLink, ilk öğenin yönlendirme yolu ve sonraki öğelerin dinamik yönlendirme parametreleri için olduğu bir dizi alır.

Çocuk Rotaları #

Bazen görünümleri veya rotaları iç içe yerleştirmek mantıklıdır. Örneğin, panoda, kullanıcıların projelerini, kişilerini, mesajlarını göstermek için, sekmelere benzer, ancak yönlendirme sistemi aracılığıyla uygulanan birkaç alt görünüm istiyorsunuz. Bu tür senaryoları desteklemek için yönlendirici, alt rotaları tanımlamamıza izin verir.

İlk önce RouterConfigimizi yukarıdan ayarlıyoruz ve alt yolları ekliyoruz:

import { ProjectsComponent } from '../components/projects.component';
import { MessagesComponent} from '../components/messages.component';

export const appRoutes: RouterConfig = [
  { path: '', pathMatch: 'full', redirectTo: 'login' },
  { path: 'dashboard', component: DashboardComponent,
    children: [
      { path: '', redirectTo: 'projects', pathMatch: 'full' },
      { path: 'projects', component: 'ProjectsComponent' },
      { path: 'messages', component: 'MessagesComponent' }
    ] },
  { path: 'bars/:id', component: BarDetailComponent },
  { path: 'login', component: LoginComponent },
  { path: 'signup',   component: SignupComponent }
];

Artık alt rotalarımızı tanımladığımıza göre, bu alt rotaların DashboardComponent içinde görüntülenebildiğinden emin olmalıyız, çünkü bu, çocukları eklediğimiz yer. Daha önce bileşenlerin bir <router-outlet></router-outlet> etiketinde görüntülendiğini öğrenmiştik. Benzer şekilde, “DashboardComponent” içinde başka bir “RouterOutlet” ilan ediyoruz:

import { Component } from '@angular/core';

@Component({
  selector: 'dashboard',
  template: `
    <a [routerLink]="['projects']">Projects</a>
    <a [routerLink]="['messages']">Messages</a>   
    <div>
      <router-outlet></router-outlet>
    </div>
  `
})
export class DashboardComponent { }

Gördüğünüz gibi, alt rotaların görüntüleneceği başka bir ‘RouterOutlet’ ekledik. Genellikle yolu boş olan rota gösterilir, ancak ‘projeler’ rotasına bir yönlendirme ayarladık, çünkü bunun ‘dashboard’ rotası yüklendiğinde hemen gösterilmesini istiyoruz. Olduğu söyleniyor, boş bir rotaya ihtiyacımız var, aksi takdirde şöyle bir hata alırsınız:

Cannot match any routes: 'dashboard'

Böylece boş rotayı, yani boş yolu olan bir rotayı ekleyerek, yönlendirici için bir giriş noktası tanımladık.
