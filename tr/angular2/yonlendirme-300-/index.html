Yönlendirici ile yapabileceğimiz birkaç püf noktası daha var (erişim kısıtlaması gibi), ancak bunlar ayrı bir eğitimde ele alınabilir.

Yeni bir rotaya ihtiyacınız varsa, sadece app.routes.ts dosyasını değiştirin ve aşağıdaki adımları izleyin:

  1. Bileşeni İçe Aktarın
  2. “routes” dizisine ekleyin. Yeni bir “yol” ve “bileşen” eklediğinizden emin olun.

Bir Rotaya veya Rotadan Erişimi Kontrol Etme #

Varsayılan Açısal yönlendirici, koşulsuz olarak herhangi bir rotaya ve rotadan navigasyona izin verir. Bu her zaman istenen davranış değildir.

Bir kullanıcının koşullu olarak bir rotaya gitmesine veya rotadan gitmesine izin verildiği bir senaryoda, bu davranışı kısıtlamak için bir Rota Koruması kullanılabilir.

Senaryonuz aşağıdakilerden birine uyuyorsa, bir Rota Koruması kullanmayı düşünün,

  • Hedef bileşene gitmek için kullanıcının kimliğinin doğrulanması gerekir.
  • Kullanıcının hedef bileşene gitmesi için yetkilendirilmesi gerekir.
  • Bileşen, başlatmadan önce eşzamansız istek gerektiriyor.
  • Bileşen, başka bir yere gitmeden önce kullanıcı girişi gerektirir.

Rota Muhafızları nasıl çalışır? #

Rota Korumaları, yönlendirici gezinme davranışını kontrol etmek için bir boole değeri döndürerek çalışır. true döndürülürse, yönlendirici hedef bileşene navigasyona devam eder. false döndürülürse, yönlendirici hedef bileşene navigasyonu reddeder.

Rota Koruma Arayüzleri #

Yönlendirici birden çok koruma arabirimini destekler:

  • CanActivate: rota navigasyonu arasında gerçekleşir.
  • CanActivateChild: bir alt rotaya rota navigasyonu arasında gerçekleşir.
  • Deaktive Edebilir: mevcut rotadan uzaklaşırken gerçekleşir.
  • CanLoad: zaman uyumsuz olarak yüklenen bir özellik modülüne rota navigasyonu arasında gerçekleşir.
  • Çözüm: rota aktivasyonundan önce veri alımını gerçekleştirmek için kullanılır.

Bu arayüzler, belirli navigasyon işlemlerine erişim vermek veya erişimi kaldırmak için korumanıza uygulanabilir.

Eşzamanlı ve Eşzamansız Rota Korumaları #

Rota Korumaları, eşzamanlı ve eşzamansız işlemlerin navigasyonu koşullu olarak kontrol etmesine izin verir.

Senkronize Rota Koruması #

Eşzamanlı bir rota koruması, navigasyonu koşullu olarak kontrol etmek için, bir anlık sonucu hesaplayarak olduğu gibi bir boole döndürür.

import { Injectable }     from '@angular/core';
import { CanActivate }    from '@angular/router';

@Injectable()
export class SynchronousGuard implements CanActivate {
  canActivate() {
    console.log('SynchronousGuard#canActivate called');
    return true;
  }
}

Eşzamansız Rota Koruması #

Daha karmaşık davranış için, bir rota koruması, navigasyonu eşzamansız olarak engelleyebilir. Eşzamansız bir rota koruması, bir Observable veya Promise döndürebilir.

Bu, bir soruyu yanıtlamak için kullanıcı girişinin beklenmesi, sunucudaki değişikliklerin başarıyla kaydedilmesini beklemek veya uzak bir sunucudan alınan verilerin alınmasını beklemek gibi durumlar için kullanışlıdır.

import { Injectable }     from '@angular/core';
import { CanActivate, Router, ActivatedRouteSnapshot, RouterStateSnapshot } from '@angular/router';
import { Observable }     from 'rxjs/Rx';
import { MockAuthenticationService } from './authentication/authentication.service';

@Injectable()
export class AsynchronousGuard implements CanActivate {
    constructor(private router: Router, private auth: MockAuthenticationService) {}

    canActivate(route:ActivatedRouteSnapshot, state:RouterStateSnapshot):Observable<boolean>|boolean {
        this.auth.subscribe((authenticated) => {
            if (authenticated) {
                return true;
            }
            this.router.navigateByUrl('/login');
            return false;
        });
    }
}

Rota yapılandırmasına koruma ekleyin #

Çözümleyicileri ve Korumaları Kullanma #

İlk sayfa yüklemesinde geçerli kullanıcıyı yakalamak için rota yapılandırmamızda üst düzey bir koruma ve arka uçtan kimliği doğrulanmış kullanıcımız olan ‘currentUser’ değerini depolamak için bir çözümleyici kullanıyoruz.

Uygulamamızın basitleştirilmiş bir versiyonu aşağıdaki gibidir:

İşte en üst seviye rotamız:

export const routes = [
{
  path: 'Dash',
  pathMatch : 'prefix',
  component: DashCmp,
  canActivate: [AuthGuard],
  resolve: {
      currentUser: CurrentUserResolver
  },
  children: [...[
    path: '',
    component: ProfileCmp,
    resolve: {
        currentUser: currentUser
    }
  ]]
  }
];

İşte bizim ‘AuthService’imiz

import { Injectable } from '@angular/core';
import { Http, Headers, RequestOptions } from '@angular/http';
import { Observable } from 'rxjs/Rx';
import 'rxjs/add/operator/do';

@Injectable()
export class AuthService {
  constructor(http: Http) {
    this.http = http;

    let headers = new Headers({ 'Content-Type': 'application/json' });
    this.options = new RequestOptions({ headers: headers });
  }
  fetchCurrentUser() {
    return this.http.get('/api/users/me')
      .map(res => res.json())
      .do(val => this.currentUser = val);
 }
}

İşte bizim ‘AuthGuard’ımız:

import { Injectable } from '@angular/core';
import { CanActivate } from "@angular/router";
import { Observable } from 'rxjs/Rx';

import { AuthService } from '../services/AuthService';

@Injectable()
export class AuthGuard implements CanActivate {
  constructor(auth: AuthService) {
    this.auth = auth;
  }
  canActivate(route, state) {
    return Observable
      .merge(this.auth.fetchCurrentUser(), Observable.of(true))
      .filter(x => x == true);
  }
}

İşte bizim ‘CurrentUserResolver’ımız:

import { Injectable } from '@angular/core';
import { Resolve } from "@angular/router";
import { Observable } from 'rxjs/Rx';

import { AuthService } from '../services/AuthService';

@Injectable()
export class CurrentUserResolver implements Resolve {
  constructor(auth: AuthService) {
    this.auth = auth;
  }
  resolve(route, state) {
    return this.auth.currentUser;
  }
}

Uygulama önyüklemesinde Guard kullanın #

Önyükleme #

Artık rotalar tanımlandığına göre uygulamamıza rotalar hakkında bilgi vermemiz gerekiyor. Bunu yapmak için önceki örnekte dışa aktardığımız sağlayıcıyı önyükleyin.

Önyükleme yapılandırmanızı bulun (“main.ts” içinde olmalıdır, ancak mesafeniz değişebilir).

//main.ts

import {bootstrap} from '@angular/platform-browser-dynamic';

//Import the App component (root component)
import { App } from './app/app';

//Also import the app routes
import { APP_ROUTES_PROVIDER } from './app/app.routes';

bootstrap(App, [
  APP_ROUTES_PROVIDER,
])
.catch(err => console.error(err));

Yönlendirici çıkışını yapılandırma #

Yönlendirici yapılandırıldığına ve uygulamamız rotaları nasıl ele alacağını bildiğine göre, yapılandırdığımız gerçek bileşenleri göstermemiz gerekiyor.

Bunu yapmak için, üst düzey (uygulama) bileşeniniz için HTML şablonunuzu/dosyanızı şu şekilde yapılandırın:

//app.ts

import {Component} from '@angular/core';
import {Router, ROUTER_DIRECTIVES} from '@angular/router';

@Component({
    selector: 'app',
    templateUrl: 'app.html',
    styleUrls: ['app.css'],
    directives: [
        ROUTER_DIRECTIVES,
    ]
})
export class App {
    constructor() {
    }
}

<!-- app.html -->

<!-- All of your 'views' will go here -->
<router-outlet></router-outlet>

<router-outlet></router-outlet> öğesi, rota verilen içeriği değiştirir. Bu öğeyle ilgili bir başka iyi yön de, HTML’nizdeki tek öğe olması *gerekmemesidir.

Örneğin: Her sayfada, Stack Overflow’un görünümüne benzer şekilde, rotalar arasında sabit kalan bir araç çubuğu istediğinizi varsayalım. “” öğesini öğelerin altına yuvalayarak sayfanın yalnızca belirli bölümlerinin değişmesini sağlayabilirsiniz.

Rotaları değiştirme (şablonları ve yönergeleri kullanarak) #

Artık rotalar ayarlandığına göre, rotaları gerçekten değiştirmenin bir yoluna ihtiyacımız var.

Bu örnek, şablonu kullanarak rotaların nasıl değiştirileceğini gösterecektir, ancak TypeScript’te rotaları değiştirmek mümkündür.

İşte bir örnek (bağlayıcı olmadan):

<a routerLink="/home">Home</a>

Kullanıcı bu bağlantıya tıklarsa, /home adresine yönlendirilecektir. Yönlendirici /home ile nasıl başa çıkacağını bilir, bu nedenle Home Bileşenini görüntüler.

İşte veri bağlama ile bir örnek:

<a *ngFor="let link of links" [routerLink]="link">{{link}}</a>

Bu, “bağlantılar” adlı bir dizinin var olmasını gerektirir, bu nedenle bunu “app.ts"ye ekleyin:

public links[] = [
    'home',
    'login'
]

Bu, dizi içinde döngü yapacak ve ‘routerLink’ yönergesi ile bir ‘’ öğesi ekleyecek = dizideki geçerli öğenin değeri, şunu yaratacak:

 <a routerLink="home">home</a>
 <a routerLink="login">login</a>

Bu, özellikle çok sayıda bağlantınız varsa veya bağlantıların sürekli olarak değiştirilmesi gerekiyorsa yararlıdır. Angular’ın sadece ihtiyaç duyduğu bilgiyi besleyerek yoğun bağlantı ekleme işini halletmesine izin verdik.

Şu anda ‘bağlantılar[]’ statiktir, ancak başka bir kaynaktan veri beslemek mümkündür.

Rotaları Ayarlama #

NOT: Bu örnek, @angular/router’ın 3.0.0.-beta.2 sürümüne dayanmaktadır. Yazma sırasında bu, yönlendiricinin en son sürümüdür.

Yönlendiriciyi kullanmak için, aşağıdaki gibi yeni bir TypeScript dosyasında rotaları tanımlayın.

//app.routes.ts

import {provideRouter} from '@angular/router';

import {Home} from './routes/home/home';
import {Profile} from './routes/profile/profile';

export const routes = [
    {path: '', redirectTo: 'home'},
    {path: 'home', component: Home},
    {path: 'login', component: Login},
];

export const APP_ROUTES_PROVIDER = provideRouter(routes);

İlk satırda, önyükleme aşamasında rotaların ne olduğunu uygulamamıza bildirebilmemiz için ‘provideRouter’ı içe aktarıyoruz.

“Ev” ve “Profil”, örnek olarak yalnızca iki bileşendir. İhtiyacınız olan her ‘Bileşen’i bir rota olarak içe aktarmanız gerekecektir.

Ardından, rota dizisini dışa aktarın.

path: Bileşenin yolu. ’/……..’ KULLANMANIZA GEREK YOKTUR Angular bunu otomatik olarak yapacaktır

‘bileşen’: Rotaya erişildiğinde yüklenecek bileşen

redirectTo: İsteğe bağlı. Bir kullanıcıyı belirli bir rotaya eriştiğinde otomatik olarak yeniden yönlendirmeniz gerekiyorsa, bunu sağlayın.

Son olarak, yapılandırılmış yönlendiriciyi dışa aktarıyoruz. ‘provideRouter’, uygulamamızın her bir rotayı nasıl ele alacağını bilmesi için artırabileceğimiz bir sağlayıcı döndürür.
