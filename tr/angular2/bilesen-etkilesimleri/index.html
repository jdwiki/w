Giriş bağlama ile verileri ebeveynden çocuğa iletin #

HeroChildComponent, tipik olarak @Input süslemeleriyle süslenmiş iki giriş özelliğine sahiptir.

import { Component, Input } from '@angular/core';
import { Hero } from './hero';
@Component({
  selector: 'hero-child',
  template: `
    <h3>{{hero.name}} says:</h3>
    <p>I, {{hero.name}}, am at your service, {{masterName}}.</p>
  `
})
export class HeroChildComponent {
  @Input() hero: Hero;
  @Input('master') masterName: string;
}

Bir ayarlayıcı ile giriş özelliği değişikliklerini durdurun

Üst öğeden bir değere müdahale etmek ve bu değere göre hareket etmek için bir girdi özelliği ayarlayıcı kullanın.

NameChildComponent alt öğesindeki name input özelliğinin ayarlayıcısı, bir addan boşlukları keser ve boş bir değeri varsayılan metinle değiştirir.

import { Component, Input } from '@angular/core';
@Component({
  selector: 'name-child',
  template: '<h3>"{{name}}"</h3>'
})
export class NameChildComponent {
  private _name = '';
  @Input()
  set name(name: string) {
    this._name = (name && name.trim()) || '<no name set>';
  }
  get name(): string { return this._name; }
}

Tüm boşluklarla birlikte bir ad da dahil olmak üzere ad varyasyonlarını gösteren NameParentComponent:

import { Component } from '@angular/core';
@Component({
  selector: 'name-parent',
  template: `
  <h2>Master controls {{names.length}} names</h2>
  <name-child *ngFor="let name of names" [name]="name"></name-child>
  `
})
export class NameParentComponent {
  // Displays 'Mr. IQ', '<no name set>', 'Bombasto'
  names = ['Mr. IQ', '   ', '  Bombasto  '];
}

Ebeveyn çocuk olayını dinler

Alt bileşen, bir şey olduğunda olayları yayınladığı bir EventEmitter özelliği sunar. Ebeveyn, bu olay özelliğine bağlanır ve bu olaylara tepki verir.

Çocuğun EventEmitter özelliği, bu VoterComponent’te görüldüğü gibi genellikle @Output dekorasyonuyla süslenmiş bir çıktı özelliğidir:

import { Component, EventEmitter, Input, Output } from '@angular/core';
@Component({
  selector: 'my-voter',
  template: `
    <h4>{{name}}</h4>
    <button (click)="vote(true)"  [disabled]="voted">Agree</button>
    <button (click)="vote(false)" [disabled]="voted">Disagree</button>
  `
})
export class VoterComponent {
  @Input()  name: string;
  @Output() onVoted = new EventEmitter<boolean>();
  voted = false;
  vote(agreed: boolean) {
    this.onVoted.emit(agreed);
    this.voted = true;
  }
}

Bir düğmeye tıklamak, doğru veya yanlış (boole yükü) emisyonunu tetikler.

Ana VoteTakerComponent, alt olay yüküne ($event) yanıt veren ve bir sayacı güncelleyen bir olay işleyicisini (onVoted) bağlar.

import { Component }      from '@angular/core';
@Component({
  selector: 'vote-taker',
  template: `
    <h2>Should mankind colonize the Universe?</h2>
    <h3>Agree: {{agreed}}, Disagree: {{disagreed}}</h3>
    <my-voter *ngFor="let voter of voters"
      [name]="voter"
      (onVoted)="onVoted($event)">
    </my-voter>
  `
})
export class VoteTakerComponent {
  agreed = 0;
  disagreed = 0;
  voters = ['Mr. IQ', 'Ms. Universe', 'Bombasto'];
  onVoted(agreed: boolean) {
    agreed ? this.agreed++ : this.disagreed++;
  }
}

Ebeveyn, çocukla yerel değişken aracılığıyla etkileşime girer

Bir üst bileşen, alt özellikleri okumak veya alt yöntemleri çağırmak için veri bağlamayı kullanamaz. Hem alt öğe için bir şablon referans değişkeni oluşturarak hem de aşağıdaki örnekte görüldüğü gibi üst şablonda bu değişkene başvurarak yapabiliriz.

Tekrar tekrar sıfıra kadar sayan ve bir roket fırlatan bir çocuk CountdownTimerComponent’imiz var. Saati kontrol eden başlatma ve durdurma yöntemlerine sahiptir ve kendi şablonunda bir geri sayım durum mesajı görüntüler.

import { Component, OnDestroy, OnInit } from '@angular/core';
@Component({
  selector: 'countdown-timer',
  template: '<p>{{message}}</p>'
})
export class CountdownTimerComponent implements OnInit, OnDestroy {
  intervalId = 0;
  message = '';
  seconds = 11;
  clearTimer() { clearInterval(this.intervalId); }
  ngOnInit()    { this.start(); }
  ngOnDestroy() { this.clearTimer(); }
  start() { this.countDown(); }
  stop()  {
    this.clearTimer();
    this.message = `Holding at T-${this.seconds} seconds`;
  }
  private countDown() {
    this.clearTimer();
    this.intervalId = window.setInterval(() => {
      this.seconds -= 1;
      if (this.seconds === 0) {
        this.message = 'Blast off!';
      } else {
        if (this.seconds < 0) { this.seconds = 10; } // reset
        this.message = `T-${this.seconds} seconds and counting`;
      }
    }, 1000);
  }
}

Timer bileşenini barındıran CountdownLocalVarParentComponent’i görelim.

import { Component }                from '@angular/core';
import { CountdownTimerComponent }  from './countdown-timer.component';
@Component({
  selector: 'countdown-parent-lv',
  template: `
  <h3>Countdown to Liftoff (via local variable)</h3>
  <button (click)="timer.start()">Start</button>
  <button (click)="timer.stop()">Stop</button>
  <div class="seconds">{{timer.seconds}}</div>
  <countdown-timer #timer></countdown-timer>
  `,
  styleUrls: ['demo.css']
})
export class CountdownLocalVarParentComponent { }

Üst bileşen, alt öğenin başlatma ve durdurma yöntemlerine veya saniye özelliğine veri bağlayamaz.

Alt bileşeni temsil eden etikete () yerel bir değişken (#timer) yerleştirebiliriz. Bu bize alt bileşenin kendisine bir referans verir ve ana şablondan herhangi bir özelliğine veya yöntemine erişme yeteneği verir.

Bu örnekte, çocuğun başlangıç ​​ve bitiş noktalarına ebeveyn düğmeleri bağlarız ve çocuğun saniye özelliğini görüntülemek için enterpolasyon kullanırız.

Burada ebeveyn ve çocuğun birlikte çalıştığını görüyoruz.

Ebeveyn bir ViewChild’i arar

Yerel değişken yaklaşımı basit ve kolaydır. Ancak ebeveyn-çocuk kablolamasının tamamen ebeveyn şablonu içinde yapılması gerektiğinden sınırlıdır. Ana bileşenin kendisinin alt öğeye erişimi yoktur.

Ana bileşen sınıfının bir örneğinin alt bileşen değerlerini okuması veya yazması veya alt bileşen yöntemlerini çağırması gerekiyorsa yerel değişken tekniğini kullanamayız.

Üst bileşen sınıfı bu tür bir erişim gerektirdiğinde, alt bileşeni üst öğeye bir ViewChild olarak enjekte ederiz.

Bu tekniği aynı Geri Sayım Sayacı örneğinde göstereceğiz. Görünümünü veya davranışını değiştirmeyeceğiz. CountdownTimerComponent alt öğesi de aynıdır.

Yalnızca gösterim amacıyla yerel değişkenden ViewChild tekniğine geçiyoruz. İşte ebeveyn, CountdownViewChildParentComponent:

import { AfterViewInit, ViewChild } from '@angular/core';
import { Component }                from '@angular/core';
import { CountdownTimerComponent }  from './countdown-timer.component';
@Component({
  selector: 'countdown-parent-vc',
  template: `
  <h3>Countdown to Liftoff (via ViewChild)</h3>
  <button (click)="start()">Start</button>
  <button (click)="stop()">Stop</button>
  <div class="seconds">{{ seconds() }}</div>
  <countdown-timer></countdown-timer>
  `,
  styleUrls: ['demo.css']
})
export class CountdownViewChildParentComponent implements AfterViewInit {
  @ViewChild(CountdownTimerComponent)
  private timerComponent: CountdownTimerComponent;
  seconds() { return 0; }
  ngAfterViewInit() {
    // Redefine `seconds()` to get from the `CountdownTimerComponent.seconds` ...
    // but wait a tick first to avoid one-time devMode
    // unidirectional-data-flow-violation error
    setTimeout(() => this.seconds = () => this.timerComponent.seconds, 0);
  }
  start() { this.timerComponent.start(); }
  stop() { this.timerComponent.stop(); }
}

Alt öğeyi ana bileşen sınıfına sokmak biraz daha fazla çalışma gerektirir.

Referansları ViewChild dekoratörüne ve AfterViewInit yaşam döngüsü kancasına aktarırız.

CountdownTimerComponent alt öğesini @ViewChild özellik dekorasyonu aracılığıyla private timerComponent özelliğine enjekte ediyoruz.

#timer yerel değişkeni, bileşen meta verilerinden kayboldu. Bunun yerine, düğmeleri ana bileşenin kendi başlatma ve durdurma yöntemlerine bağlarız ve işaretleme saniyelerini, ana bileşenin saniye yöntemi etrafında bir enterpolasyon içinde sunarız.

Bu yöntemler, enjekte edilen zamanlayıcı bileşenine doğrudan erişir.

ngAfterViewInit yaşam döngüsü kancası önemli bir kırışıklıktır. Zamanlayıcı bileşeni, Angular üst görünümü görüntüleyene kadar kullanılamaz. Yani başlangıçta 0 saniye gösteriyoruz.

Ardından Angular, ngAfterViewInit yaşam döngüsü kancasını çağırır ve bu sırada ana görünümün geri sayım saniyelerinin görüntüsünü güncellemek için çok geç olur. Angular’ın tek yönlü veri akışı kuralı, üst görünümün aynı döngüde güncellenmesini engeller. Saniyeleri gösterebilmek için bir tur beklememiz gerekiyor.

Bir onay beklemek için setTimeout’u kullanırız ve ardından saniye yöntemini zamanlayıcı bileşeninden gelecek değerleri alacak şekilde revize ederiz.

Ebeveyn ve çocuklar bir hizmet aracılığıyla iletişim kurar

Bir üst bileşen ve onun çocukları, arabirimi aile içinde çift yönlü iletişime olanak tanıyan bir hizmeti paylaşır.

Hizmet örneğinin kapsamı, ana bileşen ve onun çocuklarıdır. Bu bileşen alt ağacının dışındaki bileşenlerin hizmete veya iletişimlerine erişimi yoktur.

Bu MissionService, MissionControlComponent’i birden çok AstronautComponent alt öğesine bağlar.

import { Injectable } from '@angular/core';
import { Subject }    from 'rxjs/Subject';
@Injectable()
export class MissionService {
  // Observable string sources
  private missionAnnouncedSource = new Subject<string>();
  private missionConfirmedSource = new Subject<string>();
  // Observable string streams
  missionAnnounced$ = this.missionAnnouncedSource.asObservable();
  missionConfirmed$ = this.missionConfirmedSource.asObservable();
  // Service message commands
  announceMission(mission: string) {
    this.missionAnnouncedSource.next(mission);
  }
  confirmMission(astronaut: string) {
    this.missionConfirmedSource.next(astronaut);
  }
}

MissionControlComponent, hem çocuklarıyla paylaştığı hizmetin örneğini sağlar (sağlayıcı meta veri dizisi aracılığıyla) hem de bu örneği yapıcısı aracılığıyla kendisine enjekte eder:

import { Component }          from '@angular/core';
import { MissionService }     from './mission.service';
@Component({
  selector: 'mission-control',
  template: `
    <h2>Mission Control</h2>
    <button (click)="announce()">Announce mission</button>
    <my-astronaut *ngFor="let astronaut of astronauts"
      [astronaut]="astronaut">
    </my-astronaut>
    <h3>History</h3>
    <ul>
      <li *ngFor="let event of history">{{event}}</li>
    </ul>
  `,
  providers: [MissionService]
})
export class MissionControlComponent {
  astronauts = ['Lovell', 'Swigert', 'Haise'];
  history: string[] = [];
  missions = ['Fly to the moon!',
              'Fly to mars!',
              'Fly to Vegas!'];
  nextMission = 0;
  constructor(private missionService: MissionService) {
    missionService.missionConfirmed$.subscribe(
      astronaut => {
        this.history.push(`${astronaut} confirmed the mission`);
      });
  }
  announce() {
    let mission = this.missions[this.nextMission++];
    this.missionService.announceMission(mission);
    this.history.push(`Mission "${mission}" announced`);
    if (this.nextMission >= this.missions.length) { this.nextMission = 0; }
  }
}

AstronautComponent, hizmeti yapıcısına da enjekte eder. Her AstronautComponent, MissionControlComponent’in bir alt öğesidir ve bu nedenle üst öğesinin hizmet örneğini alır:

import { Component, Input, OnDestroy } from '@angular/core';
import { MissionService } from './mission.service';
import { Subscription }   from 'rxjs/Subscription';
@Component({
  selector: 'my-astronaut',
  template: `
    <p>
      {{astronaut}}: <strong>{{mission}}</strong>
      <button
        (click)="confirm()"
        [disabled]="!announced || confirmed">
        Confirm
      </button>
    </p>
  `
})
export class AstronautComponent implements OnDestroy {
  @Input() astronaut: string;
  mission = '<no mission announced>';
  confirmed = false;
  announced = false;
  subscription: Subscription;
  constructor(private missionService: MissionService) {
    this.subscription = missionService.missionAnnounced$.subscribe(
      mission => {
        this.mission = mission;
        this.announced = true;
        this.confirmed = false;
    });
  }
  confirm() {
    this.confirmed = true;
    this.missionService.confirmMission(this.astronaut);
  }
  ngOnDestroy() {
    // prevent memory leak when component destroyed
    this.subscription.unsubscribe();
  }
}

AstronautComponent yok edildiğinde aboneliği yakaladığımıza ve abonelikten çıktığımıza dikkat edin. Bu bir bellek sızıntısı koruma adımıdır. Bu uygulamada gerçek bir risk yoktur çünkü bir AstronautComponent’in ömrü, uygulamanın kendi ömrüyle aynıdır. Daha karmaşık bir uygulamada bu her zaman doğru olmaz.
