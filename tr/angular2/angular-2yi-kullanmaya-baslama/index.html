angular2’yi angular-cli ile kurun #

Bu örnek, Angular 2’nin hızlı kurulumu ve hızlı bir örnek projenin nasıl oluşturulacağıdır.

Önkoşullar: #

  • Node.js v4 veya üstü.
  • npm v3 veya üstü veya iplik.

Bir terminal açın ve komutları tek tek çalıştırın:

npm install -g @angular/cli

veya

yarn global add @angular/cli

paket yöneticisi seçiminize bağlı olarak.

Önceki komut, PATH’e yürütülebilir ng ekleyerek global olarak @angular/cli yükler.

Yeni bir proje kurmak için #

Terminal ile yeni projeyi kurmak istediğiniz klasöre gidin.

Komutları çalıştırın:

ng new PROJECT_NAME
cd PROJECT_NAME
ng serve

İşte bu kadar, şimdi Angular 2 ile yapılmış basit bir örnek projeniz var. Artık terminalde görüntülenen bağlantıya gidebilir ve ne çalıştığını görebilirsiniz.

Mevcut bir projeye eklemek için #

Mevcut projenizin köküne gidin.

Komutu çalıştırın:

ng init

Bu, projenize gerekli iskeleyi ekleyecektir. Dosyalar geçerli dizinde oluşturulacak, bu yüzden bunu boş bir dizinde çalıştırdığınızdan emin olun.

Projeyi Yerel Olarak Çalıştırmak #

Uygulamanızı tarayıcıda çalışırken görmek ve onunla etkileşimde bulunmak için, projenizin dosyalarını barındıran yerel bir geliştirme sunucusu başlatmanız gerekir.

ng serve

Sunucu başarılı bir şekilde başlatılırsa, sunucunun çalıştığı bir adres göstermelidir. Genellikle bu:

http://localhost:4200

Kutunun dışında bu yerel geliştirme sunucusu Hot Module Reloading ile bağlantılıdır, bu nedenle html, typescript veya css’de yapılan herhangi bir değişiklik, tarayıcının otomatik olarak yeniden yüklenmesini tetikler (ancak istenirse devre dışı bırakılabilir).

Bileşenler, Yönergeler, Borular ve Hizmetler Oluşturma #

ng create <scaffold-type> <name> (veya sadece ng g <scaffold-type> <name>) komutu, otomatik olarak Angular bileşenleri oluşturmanıza olanak tanır:

# The command below will generate a component in the folder you are currently at
ng generate component my-generated-component
# Using the alias (same outcome as above)
ng g component my-generated-component

Angular-cli’nin oluşturabileceği birkaç olası iskele türü vardır:

İskele Tipi kullanım
Modül `ng g modülü benim-yeni-modülüm'
Bileşen ng g bileşeni benim-yeni-bileşenim
Direktif `ng g yönergesi benim-yeni yönergem'
boru ng g boru benim-yeni borum
Hizmet ng g service benim-yeni-hizmetim
sınıf `ng g sınıfı benim-yeni-sınıfım'
arayüz `ng g arayüzü benim-yeni-arayüzüm'
Numara ng g enum my-new-enum

Ayrıca tür adını ilk harfiyle değiştirebilirsiniz. Örneğin:

Yeni bir modül oluşturmak için ‘g m my-new-module’ veya bir bileşen oluşturmak için ‘g c my-new-bileşenim’.

Bina/Donatılacak

Angular 2 web uygulamanızı oluşturmayı tamamladığınızda ve bunu Apache Tomcat gibi bir web sunucusuna kurmak istediğinizde, tek yapmanız gereken, üretim bayrağı seti olsun ya da olmasın build komutunu çalıştırmaktır. Üretim, kodu en aza indirecek ve bir üretim ayarı için optimize edecektir.

ng build

veya

ng build --prod

Ardından, derlemeyi içeren bir /dist klasörü için projelerin kök dizinine bakın.

Daha küçük bir üretim paketinin avantajlarından yararlanmak istiyorsanız, şablon derleyicisini son derlemeden kaldıran Zaman Öncesi şablon derlemesini de kullanabilirsiniz:

ng build --prod --aot

Birim Testi

Angular 2, yerleşik birim testi sağlar ve angular-cli tarafından oluşturulan her öğe, genişletilebilen temel bir birim testi oluşturur. Birim testleri yasemin kullanılarak yazılır ve Karma aracılığıyla yürütülür. Testi başlatmak için aşağıdaki komutu yürütün:

ng test

Bu komut, projedeki tüm testleri yürütecek ve ister bir test ister uygulamadan bir kod olsun, bir kaynak dosya her değiştiğinde bunları yeniden yürütecektir.

Daha fazla bilgi için ayrıca şu adresi ziyaret edin: angular-cli github sayfası

Angular-cli olmadan Angular 2’ye başlarken. #

Açısal 2.0.0-rc.4

Bu örnekte bir “Merhaba Dünya!” oluşturacağız. basitlik adına yalnızca bir kök bileşenli (AppComponent) app.

Ön koşullar:

  • Node.js v5 veya üzeri
  • npm v3 veya üstü

Not: Konsolda/terminalde “node -v” ve “npm -v” komutlarını çalıştırarak sürümleri kontrol edebilirsiniz.

Aşama 1 #

Projeniz için yeni bir klasör oluşturun ve girin. Buna ‘açısal2-örnek’ diyelim.

mkdir angular2-example
cd angular2-example

Adım 2 #

Uygulama kodumuzu yazmaya başlamadan önce, aşağıda verilen 4 dosyayı ekleyeceğiz: package.json, tsconfig.json, typings.json ve systemjs.config.js.

Sorumluluk reddi: Aynı dosyalar Resmi 5 Dakikada Hızlı Başlangıç içinde bulunabilir.

systemjs.config.js - SystemJS yapılandırır (webpack da kullanabilirsiniz).

/**
 * System configuration for Angular 2 samples
 * Adjust as necessary for your application's needs.
 */
(function(global) {
  // map tells the System loader where to look for things
  var map = {
    'app':                        'app', // 'dist',
    '@angular':                   'node_modules/@angular',
    'angular2-in-memory-web-api': 'node_modules/angular2-in-memory-web-api',
    'rxjs':                       'node_modules/rxjs'
  };
  // packages tells the System loader how to load when no filename and/or no extension
  var packages = {
    'app':                        { main: 'main.js',  defaultExtension: 'js' },
    'rxjs':                       { defaultExtension: 'js' },
    'angular2-in-memory-web-api': { main: 'index.js', defaultExtension: 'js' },
  };
  var ngPackageNames = [
    'common',
    'compiler',
    'core',
    'forms',
    'http',
    'platform-browser',
    'platform-browser-dynamic',
    'router',
    'router-deprecated',
    'upgrade',
  ];
  // Individual files (~300 requests):
  function packIndex(pkgName) {
    packages['@angular/'+pkgName] = { main: 'index.js', defaultExtension: 'js' };
  }
  // Bundled (~40 requests):
  function packUmd(pkgName) {
    packages['@angular/'+pkgName] = { main: '/bundles/' + pkgName + '.umd.js', defaultExtension: 'js' };
  }
  // Most environments should use UMD; some (Karma) need the individual index files
  var setPackageConfig = System.packageWithIndex ? packIndex : packUmd;
  // Add package entries for angular packages
  ngPackageNames.forEach(setPackageConfig);
  var config = {
    map: map,
    packages: packages
  };
  System.config(config);
})(this);

Aşama 3 #

Bağımlılıkları yazarak yükleyelim

npm install

konsolda/terminalde.

Adım 4

‘angular2-example’ klasörünün içinde ‘index.html’ oluşturun.

<html>
  <head>
    <title>Angular2 example</title>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <!-- 1. Load libraries -->
    <!-- Polyfill(s) for older browsers -->
    <script src="node_modules/core-js/client/shim.min.js"></script>
    <script src="node_modules/zone.js/dist/zone.js"></script>
    <script src="node_modules/reflect-metadata/Reflect.js"></script>
    <script src="node_modules/systemjs/dist/system.src.js"></script>
    <!-- 2. Configure SystemJS -->
    <script src="systemjs.config.js"></script>
    <script>
      System.import('app').catch(function(err){ console.error(err); });
    </script>
  </head>
  <!-- 3. Display the application -->
  <body>
    <my-app></my-app>
  </body>
</html>

Uygulamanız “my-app” etiketleri arasında işlenecektir.

Ancak, Angular hala * ne * oluşturacağını bilmiyor. Bunu söylemek için ‘AppComponent’i tanımlayacağız.

Adım 5 #

Uygulamamızı oluşturan bileşenleri ve [hizmetleri2 tanımlayabileceğimiz “app” adlı bir alt klasör oluşturun. (Bu durumda, yalnızca “AppComponent” kodunu ve “main.ts"yi içerecektir.)

mkdir app

Adım 6 #

app/app.component.ts dosyasını oluşturun

import { Component } from '@angular/core';

@Component({
  selector: 'my-app',
  template: `
    <h1>{{title}}</h1>
    <ul>
        <li *ngFor="let message of messages">
            {{message}}
        </li>
    </ul>
  `
})
export class AppComponent { 
    title = "Angular2 example";
    messages = [
        "Hello World!",
        "Another string",
        "Another one"
    ];
}

Ne oluyor? İlk olarak, Angular’a bu bileşen için HTML etiketi ve şablonu vermek için kullandığımız @Component dekoratörünü içe aktarıyoruz. Ardından, şablonda kullanabileceğimiz ’title’ ve ‘messages’ değişkenleriyle ‘AppComponent’ sınıfını oluşturuyoruz.

Şimdi o şablona bakalım:

<h1>{{title}}</h1>
<ul>
    <li *ngFor="let message of messages">
        {{message}}
    </li>
</ul>

Bir “h1” etiketinde “title” değişkenini görüntülüyoruz ve ardından “*ngFor” yönergesini kullanarak “messages” dizisinin her öğesini gösteren bir liste yapıyoruz. Dizideki her öğe için, ‘*ngFor’, ’li’ öğesi içinde kullandığımız bir ‘mesaj’ değişkeni oluşturur. Sonuç:

<h1>Angular 2 example</h1>
<ul>
    <li>Hello World!</li>
    <li>Another string</li>
    <li>Another one</li>
</ul>

Adım 7 #

Şimdi, Angular’ın baktığı ilk dosya olacak bir main.ts dosyası oluşturuyoruz.

app/main.ts dosyasını oluşturun.

import { bootstrap }    from '@angular/platform-browser-dynamic';
import { AppComponent } from './app.component';

bootstrap(AppComponent);

“bootstrap” işlevini ve “AppComponent” sınıfını içe aktarıyoruz, ardından Angular’a kök olarak hangi bileşeni kullanacağını söylemek için “bootstrap” kullanıyoruz.

İlk uygulamanızı başlatmanın zamanı geldi. Tip

konsolunuzda/terminalinizde. Bu, lite-server’ı başlatan, uygulamanızı bir tarayıcı penceresinde açan ve TypeScript aktarıcısını izleme modunda çalıştıran “package.json"dan hazırlanmış bir komut dosyasını çalıştıracaktır (böylece “.ts” dosyaları aktarılacak ve tarayıcı yenilendiğinde değişiklikleri kaydedersiniz).

O sinir bozucu şirket proxy’sinden geçmek #

XYZ MegaCorp’taki Windows iş bilgisayarınızda çalışan bir Angular2 sitesi almaya çalışıyorsanız, şirket proxy’sinden geçmekte sorun yaşıyor olabilirsiniz.

Proxy’den geçmesi gereken (en az) iki paket yöneticisi vardır:

  1. NPM
  2. Tiplemeler

NPM için .npmrc dosyasına aşağıdaki satırları eklemeniz gerekir:

proxy=http://[DOMAIN]%5C[USER]:[PASS]@[PROXY]:[PROXYPORT]/
https-proxy=http://[DOMAIN]%5C[USER]:[PASS]@[PROXY]:[PROXYPORT]/

Tiplemeler için .typingsrc dosyasına aşağıdaki satırları eklemeniz gerekir:

proxy=http://[DOMAIN]%5C[USER]:[PASS]@[PROXY]:[PROXYPORT]/
https-proxy=http://[DOMAIN]%5C[USER]:[PASS]@[PROXY]:[PROXYPORT]/
rejectUnauthorized=false

Bu dosyalar muhtemelen henüz mevcut değildir, bu nedenle onları boş metin dosyaları olarak oluşturabilirsiniz. Proje köküne eklenebilirler (package.json ile aynı yerde veya onları %HOMEPATH% içine koyabilirsiniz ve tüm projelerinizde kullanılabilir olacaklardır.

Açık olmayan ve insanların proxy ayarlarının çalışmadığını düşünmesinin ana nedeni, etki alanı ve kullanıcı adlarını ayırmak için “"nin URL kodlaması olan “%5C"dir. Bunun için Steve Roberts’a teşekkürler: http://stackoverflow.com/questions/25660936/using-npm-behind-corporate-proxy-pac
