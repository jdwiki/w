WebAPI'de Öznitelik Yönlendirme

  • Sözdizimi
  • Parametreler

  • Rota Öneki Özelliği

Bu sayfada

  • Sözdizimi

  • Temel Özellik Yönlendirme
  • Rota Öneki Özelliği

Adından da anlaşılacağı gibi, bu, yönlendirmek için öznitelikleri kullanır. Bu, kullanıcıya WebAPI’deki URI’ler üzerinde daha fazla kontrol sağlar. Örneğin, kaynağın hiyerarşilerini tanımlayabilirsiniz. Ancak, daha önceki ‘Geleneksel Yönlendirme’ tam olarak desteklenmektedir. Kullanıcılar da her ikisinin bir karışımına sahip olabilir.

Sözdizimi #

  • [RoutePrefix(“api/books”)] - denetleyici sınıfı için
  • [Route(“getById”)] - eylemler için
  • [Route("~/api/authors/{authorId:int}/books")] - rota önekini geçersiz kılmak için

Parametreler #

Parametre Adı Ayrıntılar
RotaÖnek denetleyici sınıfına öznitelik. eylemlerdeki tüm yaygın url önekleri burada toplanmıştır. girdi olarak string alır
Rota denetleyici eylemlerine öznitelik. her eylemin (mutlaka değil) ile ilişkili rotası olacaktır
Rota("~/api/") bu, Rota Önekini geçersiz kılar

Şu anda, Attribute Routes ‘Denetleyiciye özel İleti İşleyicileri’ne sahip değil. Bildirme sırasında hangi yol için hangi işleyicinin yürütüleceğini belirtmenin bir yolu olmadığından. Bu, ‘Geleneksel Yönlendirme’de mümkündür.

Temel Özellik Yönlendirme #

Denetleyici eylemine bir öznitelik eklemeniz yeterlidir

[Route("product/{productId}/customer")]
public IQueryable<Product> GetProductsByCustomer(int productId) 
{ 
    //action code goes here 
}

bu, /product/1/customer olarak sorgulanacak ve kontrolör eylemine productId=1 gönderilecektir.

‘{ }’ içindeki ile action parametresinin aynı olduğundan emin olun. bu durumda “productId”.

bunu kullanmadan önce, Attribute Routing’i şu şekilde kullandığınızı belirtmelisiniz:

public static class WebApiConfig
{
    public static void Register(HttpConfiguration config)
    {
        config.MapHttpAttributeRoutes();
    }
}

Rota Öneki Özelliği #

Bir denetleyici içindeki tüm rotalar için rotanın ortak bir bölümüne ihtiyaç duyduğunuz durumlarda, RoutePrefix özniteliği kullanılır.

Aşağıdaki örnekte, kodun api/students kısmı ortaktır ve bu nedenle `RoutePrefix’ tanımlayabilir ve tekrar tekrar kullanmaktan kaçınabiliriz.

[RoutePrefix("api/students")]
public class StudentController : ApiController
{
[Route("")]
public IEnumerable<Student> Get() 
{
    //action code goes here 
}

[Route("{id:int}")]
public Student Get(int id) 
{
    //action code goes here 
}

[Route("")]
public HttpResponseMessage Post(Student student) 
{
    //action code goes here 
}

}
