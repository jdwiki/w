Google Cloud’a yükleyin #

Kubernetes, ilk olarak Google tarafından Container Engine için geliştirilmiştir. Bu nedenle, Kubernetes kümeleri Google’da birinci sınıf bir vatandaştır.

Kapsayıcı motorunda bir Kubernetes kümesi oluşturmak için Google Cloud SDK‘dan “gcloud” komutu gerekir. Bu komutu yerel olarak yüklemek için aşağıdaki seçeneklerden birini kullanın:

  • etkileşimli yükleyiciyi kullanın (yeni başlayanlar için en kolay yol):
curl https://sdk.cloud.google.com | bash
exec -l $SHELL
gcloud init

  • https://cloud.google.com/sdk/ adresinden SDK’yı indirin ve uygun yükleme dosyasını çalıştırın.

    For example, to install in Linux (x86_64):

curl -Lo gcloud-sdk.tar.gz https://dl.google.com/dl/cloudsdk/channels/rapid/downloads/google-cloud-sdk-142.0.0-linux-x86_64.tar.gz
tar xvf ./gcloud-sdk.tar.gz
./google-cloud-sdk/install.sh
gcloud init

“gcloud” yüklendikten sonra, aşağıdakileri içeren bir Kubernetes kümesi oluşturun:

# Give our cluster a name
CLUSTER_NAME=example-cluster

# Number of machines in the cluster.
NUM_NODES=3
    
gcloud container clusters create $CLUSTER_NAME --num_nodes=$NUM_VMS

Komut satırında Kubectl #

Çalışan bir kümeniz olduktan sonra onu kubectl komutu ile yönetebilirsiniz. Komutların çoğunu ‘kubectl –help’ komutuyla alabilirsiniz, ancak küme, düğümler, bölmeler, hizmetler ve etiketler hakkında bilgi almak ve yönetmek için size en yaygın komutları gösteriyorum.

Küme hakkında bilgi almak için aşağıdaki komutu kullanabilirsiniz.

kubectl cluster-info

Size çalışan adresi ve bağlantı noktasını gösterecektir.

Kümede yer alan düğümler, podlar, hizmetler vb. veya herhangi bir kaynak hakkında kısa bilgi almak için aşağıdaki komutu kullanabilirsiniz.

kubectl get {nodes, pods, services, ...}

Çıktı çoğunlukla kaynak başına bir satır.

Kaynaklar hakkında ayrıntılı açıklama almak için ‘kubectl’ için ‘describe’ bayrağını kullanabilirsiniz.

kubectl describe {nodes, pods, ...}

Dağıtılan uygulamalar yalnızca kümenin içinde görünür, bu nedenle çıktıyı kümenin dışından almak istiyorsanız, terminal ve kubernetes kümesi arasında bir yol oluşturmalısınız.

kubectl proxy

Kümeden her şeyi alabileceğimiz bir API açacak. Hakkında bilgi almak için podların adını almak istiyorsanız aşağıdaki komutu kullanmalısınız:

kubectl get pods -o go-template --template '{{range .items}}{{.metadata.name}}{{"\n"}}{{end}}'

Daha sonra kullanmak üzere bölmeleri listeleyecektir.

curl http://localhost:8001/api/v1/proxy/namespaces/default/pods/{pod_name}/

Diğer iki yaygın komut, günlükleri almak ve kapsayıcılı uygulamadan/içinde bir komut yürütmektir.

kubectl logs {pod_name}
kubectl exec {pod_name} {command}

Kabuğunuz için sekme tamamlama yapılandırması şu şekilde yapılabilir:

source <(kubectl completion zsh)   # if you're using zsh
source <(kubectl completion bash)  # if you're using bash

veya daha programlı olarak:

source <(kubectl completion "${0/-/}")

Minikube’u Kurma #

Minikube, Kubernetes’i çalıştırmak için yerel bir sanal makine kümesi oluşturur. Yerel makinenizde Kubernetes ile ellerinizi kirletmenin en basit yöntemidir.

Minikube için belgeler şu adreste bulunabilir: http://kubernetes.io/docs/getting-started-guides/minikube/

Gereksinimler #

  • macOS, xhyve sürücüsü, VirtualBox veya VMware Fusion hipervizörlerinde
  • Linux’ta, VirtualBox veya KVM hipervizörleri
  • Windows VirtualBox veya Hyper-V hipervizörlerinde
  • VT-x/AMD-v sanallaştırma etkin

Sanallaştırma desteğinin etkin olup olmadığını kontrol etmek için aşağıdan uygun komutu çalıştırın. Sanallaştırma etkinleştirilirse komut bir şey verir.

# On Linux
cat /proc/cpuinfo | grep 'vmx\|svm'
# On OSX
sysctl -a | grep machdep.cpu.features | grep VMX

Kurulum #

Minikube tek bir ikili dosyadır. Böylece kurulum, ikili dosyayı indirip yolunuza yerleştirmek kadar kolaydır.

# Specify the version of minikube to download.
# Latest version can be retrieved from 
# https://github.com/kubernetes/minikube/releases
VERSION=v0.16.0

# If on Linux
OS=linux  
# If on OSX
# OS=darwin

# URL to download minikube binary from
URL=https://storage.googleapis.com/minikube/releases/$VERSION/minikube-$OS-amd64

# Download binary and place in path.
curl -Lo minikube $URL 
chmod +x minikube 
sudo mv minikube /usr/local/bin/

Kullanım #

Yeni bir küme başlatmak için:

minikube start

Bu, Kubernetes’in önceden yüklenmiş ve yapılandırılmış olduğu yeni bir yerel sanal makine kümesi oluşturacaktır.

Kubernetes panosuna aşağıdakilerle erişebilirsiniz:

minikube dashboard

Minikube, “kubectl” için aşağıdakilerle kullanılabilen ilgili bir bağlam oluşturur:

kubectl config use-context minikube

Hazır olduğunda yerel Kubernet’ler kullanılabilir:

kubectl run hello-minikube --image=gcr.io/google_containers/echoserver:1.4 --port=8080
kubectl expose deployment hello-minikube --type=NodePort
curl $(minikube service hello-minikube --url)

Yerel kümeyi durdurmak için:

minikube stop

Yerel kümeyi silmek için, oluşturulduktan sonra yeni IP’nin tahsis edileceğini unutmayın:

minikube delete

kubectl’yi yapılandır #

Bir Kubernetes kümesi, “kubectl” komutu kullanılarak kontrol edilir. “kubectl"i yapılandırma yöntemi, Kubernetes’in kurulduğu yere bağlıdır.

Google Bulut (Konteyner Motoru) #

Google Cloud SDK’yı kullanarak kubectl’i yüklemek için:

gcloud components install kubectl

Kubectl’yi Container Engine’de mevcut bir Kubernetes kümesini kontrol edecek şekilde yapılandırmak için:

gcloud container clusters get-credentials $CLUSTER_NAME

Minikube #

minikube kullanırken, kubectl ikili dosyasının manuel olarak indirilmesi ve yola yerleştirilmesi gerekir.

# Version of Kubernetes.
K8S_VERSION=$(curl -sS https://storage.googleapis.com/kubernetes-release/release/stable.txt)
# Operating System. Can be one of {linux, darwin}
GOOS=linux
# Architecture. Can be one of {386, amd64, arm64, ppc641e}
GOARCH=amd64

# Download and place in path.
curl -Lo kubectl http://storage.googleapis.com/kubernetes-release/release/${K8S_VERSION}/bin/${GOOS}/${GOARCH}/kubectl
chmod +x kubectl
sudo mv kubectl /usr/local/bin/

minikube ikili, bir küme başlatırken kubectl’yi otomatik olarak yapılandırır.

minikube start
# kubectl is now ready to use!

Selam Dünya #

Kubernetes kümeniz çalıştığında ve “kubectl” yapılandırıldığında, birkaç adımda ilk uygulamanızı çalıştırabilirsiniz. Bu, yapılandırma dosyalarına ihtiyaç duymayan zorunlu komutlar kullanılarak yapılabilir.

Bir uygulamayı çalıştırmak için bir dağıtım adı (‘bootcamp’), kapsayıcı görüntü konumu (‘docker.io/jocatalin/kubernetes-bootcamp:v1’) ve bağlantı noktası (‘8080’) sağlamanız gerekir.

$ kubectl run bootcamp --image=docker.io/jocatalin/kubernetes-bootcamp:v1 --port=8080

Şununla çalıştığını onaylayın:

$ kubectl get deployments
NAME                  DESIRED   CURRENT   UP-TO-DATE   AVAILABLE   AGE
bootcamp               1         1         1            1         6s

Uygulamanızı ortaya çıkarmak ve dışarıdan çalıştırmadan erişilebilir kılmak için:

$ kubectl expose deployment/bootcamp --type="LoadBalancer" --port 8080

Şununla çalıştığını onaylayın:

$ kubectl get services
NAME                  CLUSTER-IP   EXTERNAL-IP         PORT(S)    AGE
kubernetes            10.0.0.1     <none>              443/TCP    3m
bootcamp              10.3.245.61  104.155.111.170     8080:32452/TCP   2m

Hizmetlere erişmek için harici IP’yi ve uygulama bağlantı noktasını kullanın; bunun gibi:

$ export EXTERNAL_IP=$(kubectl get service bootcamp --output=jsonpath='{.status.loadBalancer.ingress[0].ip}')
$ export PORT=$(kubectl get services --output=jsonpath='{.items[0].spec.ports[0].port}')
$ curl "$EXTERNAL_IP:$PORT"
Hello Kubernetes bootcamp! | Running on: bootcamp-390780338-2fhnk | v=1

Aynısı, sağlanan verilerle manuel olarak da yapılabilir:

$ kubectl describe service bootcamp
Name:                   bootcamp
Namespace:              default
Labels:                 run=bootcamp
Selector:               run=bootcamp
Type:                   LoadBalancer
IP:                     10.3.245.61
LoadBalancer Ingress:   104.155.111.170
Port:                   <unset> 8080/TCP
NodePort:               <unset> 32452/TCP
Endpoints:              10.0.0.3:8080
... events and details left out ....

$ export NODE=104.155.111.170
$ export PORT=8080

Bu işe yaradığında, uygulamanızı aşağıdakilerle büyütebilirsiniz:

$ kubectl scale deployments/bootcamp --replicas=4

Ve sonucu şununla kontrol edin:

$ kubectl get deployments
NAME                  DESIRED   CURRENT   UP-TO-DATE   AVAILABLE   AGE
bootcamp               4         4         4            4         30s

$ curl "$EXTERNAL_IP:$PORT"
Hello Kubernetes bootcamp! | Running on: bootcamp-390780338-2fhnk | v=1
$ curl "$EXTERNAL_IP:$PORT"
Hello Kubernetes bootcamp! | Running on: bootcamp-390780338-gmtv5 | v=1

Değişen pod kimliğine dikkat edin.

Yeni bir uygulama sürümünü çalıştırmak için şunu çalıştırın:

kubectl set image deployments/bootcamp bootcamp=jocatalin/kubernetes-bootcamp:v2

Ve şununla onaylayın:

$ curl "$EXTERNAL_IP:$PORT"
Hello Kubernetes bootcamp! | Running on: bootcamp-284539476-gafwev3 | v=2

Temizlik sonunda şu şekilde yapılır:

$ kubectl delete deployment bootcamp
$ kubectl delete service bootcamp
