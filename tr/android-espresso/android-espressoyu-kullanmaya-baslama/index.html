Seçenekler Menüsü öğelerini kontrol etme (ekran görüntüsü almak için Kaşık kullanma) #

/**
 * @author piotrek1543
 *
 * This example provides a specific UI testing problem and how it is already solved 
 * with Google's Espresso. Notice that I used also Spoon framework, as Espresso 
 * lacks of taking screenshots functionality.
 */

@RunWith(AndroidJUnit4.class)
public class MainActivityAndroidTest {
    @Rule
    public ActivityTestRule<MainActivity> mRule = new ActivityTestRule<>(MainActivity.class);

    @Test
    public void checkIfSettingsMenuItemsAreVisible() throws InterruptedException {
        //open OptionsMenu to see available items
        openActionBarOverflowOrOptionsMenu(mRule.getActivity());
        //create a screenshot with 'options_menu' TAG
        Spoon.screenshot(mRule.getActivity(), "options_menu");
        //check if Settings item is Visible
        onView(withText(R.string.action_settings)).check(matches(isDisplayed()));
        //check if `Sort` item is Visible
        onView(withText(R.string.action_sort)).check(matches(isDisplayed()));
        //perform click on `Sort` OptionsMenu item
        onView(withText(R.string.action_sort)).perform(click());
        //create a screenshot with 'options_menu_sort' TAG
        Spoon.screenshot(mRule.getActivity(), "options_menu_sort");
        //check if `Sort -> By Value id` item is Visible
        onView(withText(R.string.menu_sort_length)).check(matches(isDisplayed()));
        //check if `Sort -> By Joke length` item is Visible
        onView(withText(R.string.menu_sort_a_z)).check(matches(isDisplayed()));
    }
}

onView(withId(R.id.greet_button)) // withId(R.id.my_view) is a ViewMatcher
.perform(click())                 // click() is a ViewAction

Merhaba Dünya Espresso Örneği #

Bu, merhaba dünya örneği oluşturmak için bir öğreticidir: Bu örnek için kullanıldı: Android Studio 2.3;

Başlamak için Boş bir aktiviteye sahip yeni bir Proje oluşturmak için Android Studio’yu kullanın. Ardından, Uygulamaya test edebileceğimiz bazı basit işlevler ekliyoruz: Tıklandığında metin görünümünde “Merhaba Dünya” görüntüleyen bir düğme ekliyoruz.

Etkinlik kodu şöyle görünür:

package com.example.testing.helloworld;

import android.os.Bundle;
import android.support.v7.app.AppCompatActivity;
import android.view.View;
import android.widget.TextView;

public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        final TextView textView = (TextView) findViewById(R.id.textView);

        findViewById(R.id.button).setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                textView.setText("Hello World!");
            }
        });
    }
}

Ve bu aktivitenin activity_main düzeni şuna benzer:

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <TextView
        android:id="@+id/textView"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="" />

    <Button
        android:id="@+id/button"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Say Hello" />
</LinearLayout>

Şimdi, espresso kullanarak bu yeni oluşturulan uygulamanın davranışını test etmek istiyoruz. Genellikle uygulamanızın kodu “ana” paket içindedir, Birim Testleri “test” içindedir ve espresso enstrümantasyon testleri “androidTest” paketinin içindedir. Android Studio ile yeni bir boş aktivite projesi oluşturursanız, bu paketleri ve sınıfları zaten oluşturmuş olmalı ve şöyle görünmelidir:

Espresso ile başlamak için, “espresso-core” bağımlılığının “build.gradle” dosyasına dahil edildiğinden emin olmalıyız (bunun “compile” anahtar kelimesiyle değil, “androidTestCompile” ile açıklandığına dikkat edin). Android stüdyosu tarafından oluşturulan build.gradle dosyasındaki bağımlılıklar şöyle görünmelidir:

dependencies {
    compile fileTree(dir: 'libs', include: ['*.jar'])
    androidTestCompile('com.android.support.test.espresso:espresso-core:2.2.2', {
        exclude group: 'com.android.support', module: 'support-annotations'
    })
    compile 'com.android.support:appcompat-v7:25.2.0'
    compile 'com.android.support.constraint:constraint-layout:1.0.2'
    testCompile 'junit:junit:4.12'
}

Artık her şey hazır olduğuna göre asıl teste başlayabiliriz: ExampleInstrumentationTest dosyasını açın ve içinde zaten oluşturulmuş bir useAppContext testi olduğunu göreceksiniz. Bu test sınıfını değiştireceğiz ve uygulama davranışımızı kontrol etmek için bir test oluşturacağız:

@RunWith(AndroidJUnit4.class)
public class ExampleInstrumentedTest {

    @Rule
    public ActivityTestRule<MainActivity> mActivityRule = new ActivityTestRule<>(
        MainActivity.class, false, true);

    @Test
    public void checkHelloWorld() throws Exception {
        onView(withId(R.id.textView)).check(matches(withText("")));
        onView(withId(R.id.button)).perform(click());
        onView(withId(R.id.textView)).check(matches(withText("Hello World!")));
    }
}

ExampleInstrumentedTest sınıfını çalıştırarak testi başlatın. Bu test daha sonra üç şey yapar:

  1. Metin görünümünün boş bir dize ("") içerip içermediğini kontrol eder
  2. Düzenimizdeki düğmeyi tıklar
  3. Metin görünümünün metnini “Merhaba Dünya!” içeriyorsa tekrar kontrol eder.

Üstteki ActivityTestRule, hangi etkinliğin test edildiğini tanımlar ve testin başında başlatır. (Ayrıca bir etkinliğin otomatik başlatılmasını kapatabilir ve bunun yerine her testin içinde manuel olarak başlatabilirsiniz)

Test kuralları oldukça basittir:

  • onView(withId(R.id.textView)), activity_main düzen dosyamızdaki görünümün kimliğine göre geçerli ekranın içindeki bir görünümü arar.
  • .check(matches(withText("")); daha sonra bu görünümde bir test durumu gerçekleştirir.
  • .perform(click()) bir görünüm üzerinde bir eylem gerçekleştirir: Bu eylemler tıklamalar, uzun tıklamalar veya kaydırmalar veya daha fazlası olabilir.

Bu, android espresso Enstrümantasyon testleri ile başlamak için bir eğitimdi, umarım size bazı bilgiler vermiştir!

Test ortamınızı kurun

Kesintiyi önlemek için, test için kullanılan sanal veya fiziksel cihazlarda sistem animasyonlarını kapatmanızı önemle tavsiye ederiz.

Cihazınızda, Ayarlar->Geliştirici seçenekleri altında aşağıdaki 3 ayarı devre dışı bırakın:

  • Pencere animasyon ölçeği
  • Geçiş animasyonu ölçeği
  • Animatör süre ölçeği

Espresso’yu indirin

  • Ekstralar altında en son Android Destek Deposunu yüklediğinizden emin olun (talimatlara bakın).

  • Uygulamanızın build.gradle dosyasını açın. Bu genellikle en üst düzey build.gradle dosyası değil, app/build.gradle dosyasıdır.

  • Bağımlılıkların içine aşağıdaki satırları ekleyin:

    androidTestCompile ‘com.android.support.test.espresso:espresso-core:2.2.2’ androidTestCompile ‘com.android.support.test🏃‍♂️0.5’

  • Daha fazla eser için indirmeler bölümüne bakın (espresso-katkı, espresso-web, vb.)

  • Enstrümantasyon koşucusunu ayarlayın

Aynı build.gradle dosyasına android.defaultConfig’de aşağıdaki satırı ekleyin: testInstrumentationRunner "android.support.test.runner.AndroidJUnitRunner" Örnek build.gradle dosyası

apply plugin: 'com.android.application'

    android {
        compileSdkVersion 22
        buildToolsVersion "22"
    
        defaultConfig {
            applicationId "com.my.awesome.app"
            minSdkVersion 10
            targetSdkVersion 22.0.1
            versionCode 1
            versionName "1.0"
    
            testInstrumentationRunner "android.support.test.runner.AndroidJUnitRunner"
        }
    }

dependencies {
    // App's dependencies, including test
    compile 'com.android.support:support-annotations:22.2.0'

    // Testing-only dependencies
    androidTestCompile 'com.android.support.test🏃‍♂️0.5'
    androidTestCompile 'com.android.support.test.espresso:espresso-core:2.2.2'
}

Analiz

Her yeni sürümde doğru yolda olduğumuzdan emin olmak için test yürütücüsü analizleri toplar. Daha spesifik olarak, her bir çağrı için test edilen uygulamanın paket adının bir karmasını yükler. Bu, hem Espresso kullanan benzersiz paketlerin sayısını hem de kullanım hacmini ölçmemizi sağlar.

Bu verileri yüklemek istemiyorsanız, aşağıdaki bağımsız değişkeni test çalıştırıcısına ileterek devre dışı bırakabilirsiniz: disableAnalytics “true” (özel bağımsız değişkenlerin nasıl iletileceğine bakın).

Çalışma testleri

Android Studio’da

Bir test yapılandırması oluşturun

Android Studio’da:

  • Çalıştır menüsünü açın -> Konfigürasyonları Düzenle

  • Yeni bir Android Testleri yapılandırması ekleyin

  • Bir modül seçin

  • Belirli bir enstrümantasyon koşucusu ekleyin:

    android.support.test.runner.AndroidJUnitRunner

Yeni oluşturulan yapılandırmayı çalıştırın.

Gradle aracılığıyla komut satırından

Uygulamak

./gradlew connectedAndroidTest

Espresso’nun temel olarak üç bileşeni vardır:

  1. ViewMatchers - mevcut görünüm hiyerarşisinde görünümü bulmaya izin verir

  2. ViewActions - görünümler üzerinde eylemler gerçekleştirmeye izin verir

  3. ViewAssertions - bir görünümün durumunu belirtmeye izin verir

Temel Espresso Testi

onView(ViewMatcher)       -- 1     
 .perform(ViewAction)     -- 2
   .check(ViewAssertion); -- 3
  1. Görünümü bulur
  2. Görünüm üzerinde bir eylem gerçekleştirir
  3. Bir iddiayı doğrular
