Tatlar için bağımlılıkları bildirin #

Bağımlılıklar, belirli ürün tatları için yapı yapılandırmaları ile benzer bir şekilde eklenebilir.

android {
    ...   
    productFlavors {
        flavor1 {
            //...
        }
        flavor2 {
            //...
        }
    }
}

dependencies {
    flavor1Compile 'com.android.support:appcompat-v7:24.1.1'
    flavor1Compile 'com.google.firebase:firebase-crash:9.4.0'
        
    flavor2Compile 'com.android.support:appcompat-v7:24.1.1'
}

1: https://www.wikiod.com/tr/android-gradle/bagmllklar-bildir#Konfigürasyonlar için Bağımlılıkları Bildirin

Bağımlılıklar nasıl eklenir #

Aşağıdaki örnek, uygulamanın/modülünün “build.gradle” dosyasında üç farklı türde doğrudan bağımlılığın nasıl bildirileceğini açıklar:

   android {...}
    ...
    dependencies {
        // The 'compile' configuration tells Gradle to add the dependency to the
        // compilation classpath and include it in the final package.
    
        // Dependency on the "mylibrary" module from this project
        compile project(":mylibrary")
    
        // Remote binary dependency
        compile 'com.android.support:appcompat-v7:24.1.0'
    
        // Local binary dependency
        compile fileTree(dir: 'libs', include: ['*.jar'])
    }

Depo nasıl eklenir #

Bağımlılıkları indirmek için, Gradle’ın bunları bulabilmesi için depoyu bildirin. Bunu yapmak için, üst düzey dosyadaki uygulama/modülünün ‘build.gradle’ dosyasına bir ‘depolar { … }’ ekleyin.

repositories {
  // Gradle's Java plugin allows the addition of these two repositories via method calls:
  jcenter()
  mavenCentral()

  maven { url "http://repository.of/dependency" }

  maven { 
      credentials {
          username 'xxx'
          password 'xxx'
      }

  url 'http://my.maven
  }
}

Modül bağımlılıkları #

Çok projeli bir ‘gradle build’de, yapınızdaki başka bir modüle bağımlı olabilirsiniz.

Örnek:

  dependencies {
        // Dependency on the "mylibrary" module from this project
        compile project(":mylibrary")
  }

‘compile project(’:mylibrary’)’ satırı, “mylibrary” adlı yerel bir Android kitaplık modülünü bir bağımlılık olarak bildirir ve derleme sisteminin uygulamanızı oluştururken yerel modülü derlemesini ve içermesini gerektirir.

Yerel ikili bağımlılıklar #

Tek bir jar veya birden çok jar dosyasıyla bir bağımlılığınız olabilir.

Tek bir jar dosyasıyla şunları ekleyebilirsiniz:

dependencies {
    compile files('libs/local_dependency.jar')
}

Derlemek için bir kavanoz dizini eklemek mümkündür.

dependencies {
        compile fileTree(dir: 'libs', include: ['*.jar'])
}

Derleme fileTree(dir: 'libs', include: ['*.jar']) satırı, derleme sistemine herhangi bir JAR dosyasını derleme sınıf yolunda ve son pakette app/libs/ dizinine dahil etmesini söyler. uygulamanızın.

Yerel ikili bağımlılıklar gerektiren modülleriniz varsa, bu bağımlılıklar için JAR dosyalarını projenizin içindeki <moduleName>/libs içine kopyalayın.

Bir aar dosyaları eklemeniz gerekiyorsa daha fazla ayrıntıyı buradan okuyabilirsiniz.

Uzak ikili bağımlılıklar #

Bu yapıyı kullanarak Gradle’a uzak bağımlılıklar ekleyebilirsiniz:

compile 'group:name:version'

veya bu alternatif sözdizimi:

compile group: 'xxx', name: 'xxxxx', version: 'xxxx'

Örneğin:

compile 'com.android.support:appcompat-v7:24.1.0'

compile 'com.android.support:appcompat-v7:24.1.0’ satırı, Android Destek Kitaplığının 24.1.0 sürümüne bağımlılık olduğunu bildirir.

Konfigürasyonlar için Bağımlılıkları Bildirin #

Test/AndroidTest gibi belirli yapılandırmalar için bağımlılıklar eklenebilir

androidTestCompile 'com.android.support.test.espresso:espresso-core:2.2.1'
testCompile 'junit:junit:3.8.1'

Alternatif olarak kendi konfigürasyonunuzu oluşturun

configurations {
    myconfig
}

Ve sonra bu yapılandırma için bağımlılığı indirin

myconfig group: 'com.mycompany', name: 'my_artifact', version: '1.0.0'

Yapı türleri için bağımlılıkları bildirin #

Belirli Derleme türleri için bağımlılıklar eklenebilir:

android {
    ...   
    buildTypes {
        release {
            //...
        }

        debug {
            //....
        }
    }
}

dependencies {
    debugCompile 'com.android.support:appcompat-v7:24.1.1'
    releaseCompile 'com.google.firebase:firebase-crash:9.4.0'            
}
