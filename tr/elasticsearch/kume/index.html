Küme Durumu, küme hakkında, ayrılan (“etkin”) parça sayısı ve kaç tanesinin atanmadığı ve yeniden konumlandırıldığı gibi birçok bilgi sağlar. Ek olarak, eksik düğümler için yoklama yapmanıza olanak tanıyan kümedeki mevcut düğüm ve veri düğümü sayısını sağlar (örneğin, “15” olmasını bekliyorsanız, ancak yalnızca “14” gösteriyorsa, o zaman bir düğüm eksik).

Elasticsearch hakkında bilgisi olan biri için “atanmış” ve “atanmamış” parçalar sorunları izlemelerine yardımcı olabilir.

Küme Sağlığından kontrol edilen en yaygın alan, üç durumdan birinde olabilen “durum"dur:

  • kırmızı
  • Sarı
  • Yeşil

Renklerin her biri bir - ve sadece bir - çok basit bir şey ifade eder:

  1. Kırmızı, en az bir birincil parçanızın eksik olduğunu gösterir.
    • A missing primary shard means that an index cannot be used to write (index) new data in most cases.
      • Technically, you can still index to any primary shards that are available in that index, but practically it means that you cannot because you do not generally control what shard receives any given document.
      • Searching is still possible against a red cluster, but it means that you will get partial results if any index you search is missing shards.
    • In normal circumstances, it just means that the primary shard is being allocated (initializing_shards).
    • If a node just left the cluster (e.g., because the machine running it lost power), then it makes sense that you will be missing some primary shards temporarily.
      • Any replica shard for that primary shard will be promoted to be the primary shard in this scenario.
  2. Sarı, tüm birincil parçaların etkin olduğunu, ancak en az bir kopya parçasının eksik olduğunu gösterir.
    • A missing replica only impacts indexing if consistency settings require it to impact indexing.
      • By default, there is only one replica for any primary and indexing can happen with a single missing replica.
    • In normal circumstances, it just means that the replica shard is being allocated (initializing_shards).
    • A one node cluster with replicas enabled will always be yellow at best. It can be red if a primary shard is not yet assigned.
      • If you only have a single node, then it makes sense to disable replicas because you are not expecting any. Then it can be green.
  3. Yeşil, tüm parçaların etkin olduğunu gösterir.
    • The only shard activity allowed for a green cluster is relocating_shards.
    • New indices, and therefore new shards, will cause the cluster to go from red to yellow to green, as each shard is allocated (primary first, making it yellow, then replicas if possible, making it green).
      • In Elasticsearch 5.x and later, new indices will not make your cluster red unless it takes them too long to allocate.

Seçili başlıklarla insan tarafından okunabilir, tablo halinde Küme Sağlığı #

Örnek, temel HTTP sözdizimini kullanır. Örnekteki herhangi bir “<#>” kopyalanırken kaldırılmalıdır.

Elasticsearch’teki çoğu “_cat” API’si gibi, API de varsayılan bir dizi alanla seçici olarak yanıt verir. Ancak, isterseniz API’de başka alanlar da mevcuttur:

GET /_cat/health?help <1>
  1. ?help, API’nin alanları (ve kısa adları) ve ayrıca kısa bir açıklama döndürmesine neden olur.

“_cat/health” Elasticsearch 1.x’ten beri var, ancak Elasticsearch 5.x’ten çıktısının bir örneği:

Bu örneğin oluşturulma tarihi itibariyle mevcut olan alanlar:

epoch                 | t,time                                   | seconds since 1970-01-01 00:00:00  
timestamp             | ts,hms,hhmmss                            | time in HH:MM:SS                   
cluster               | cl                                       | cluster name                       
status                | st                                       | health status                      
node.total            | nt,nodeTotal                             | total number of nodes              
node.data             | nd,nodeData                              | number of nodes that can store data
shards                | t,sh,shards.total,shardsTotal            | total number of shards             
pri                   | p,shards.primary,shardsPrimary           | number of primary shards           
relo                  | r,shards.relocating,shardsRelocating     | number of relocating nodes         
init                  | i,shards.initializing,shardsInitializing | number of initializing nodes       
unassign              | u,shards.unassigned,shardsUnassigned     | number of unassigned shards        
pending_tasks         | pt,pendingTasks                          | number of pending tasks            
max_task_wait_time    | mtwt,maxTaskWaitTime                     | wait time of longest task pending  
active_shards_percent | asp,activeShardsPercent                  | active number of shards in percent

Daha sonra bunu yalnızca şu alanları yazdırmak için kullanabilirsiniz:

GET /_cat/health?h=timestamp,cl,status&v <1>
  1. h=... döndürülmesini istediğiniz alanların listesini tanımlar.
  2. v (ayrıntılı) onun başlıkları yazdırmasını istediğinizi tanımlar.

Elasticsearch 5.x örneğinin çıktısı:

timestamp cl            status
15:38:00  elasticsearch yellow

JSON tabanlı Küme Sağlığı #

Örnek, temel HTTP sözdizimini kullanır. Örnekteki herhangi bir “<#>” kopyalanırken kaldırılmalıdır.

“_cat” API’leri, insanların küme hakkında bir bakışta ayrıntıları alması için genellikle uygundur. Ancak sık sık, yazılımla kullanmak için tutarlı bir şekilde ayrıştırılabilir çıktı istersiniz. Genel olarak, JSON API’leri bu amaca yöneliktir.

GET /_cluster/health

“_cluster/health” Elasticsearch 1.x’ten beri var, ancak Elasticsearch 5.x’ten çıktısının bir örneği:

{
  "cluster_name": "elasticsearch",
  "status": "yellow",
  "timed_out": false,
  "number_of_nodes": 1,
  "number_of_data_nodes": 1,
  "active_primary_shards": 45,
  "active_shards": 45,
  "relocating_shards": 0,
  "initializing_shards": 0,
  "unassigned_shards": 44,
  "delayed_unassigned_shards": 0,
  "number_of_pending_tasks": 0,
  "number_of_in_flight_fetch": 0,
  "task_max_waiting_in_queue_millis": 0,
  "active_shards_percent_as_number": 50.56179775280899
}

İnsan tarafından okunabilir, başlıklarla tablo halinde Küme Sağlığı #

Örnek, temel HTTP sözdizimini kullanır. Örnekteki herhangi bir “<#>” kopyalanırken kaldırılmalıdır.

Çeşitli nedenlerle insan tarafından okunabilir, tablo şeklinde bir çıktı elde etmek için ‘_cat’ API’lerini kullanabilirsiniz.

GET /_cat/health?v <1>
  1. ?v isteğe bağlıdır, ancak “ayrıntılı” çıktı istediğinizi ima eder.

“_cat/health” Elasticsearch 1.x’ten beri var, ancak Elasticsearch 5.x’ten çıktısının bir örneği:

Ayrıntılı çıktı ile:

epoch      timestamp cluster       status node.total node.data shards pri relo init unassign pending_tasks max_task_wait_time active_shards_percent
1469302011 15:26:51  elasticsearch yellow          1         1     45  45    0    0       44             0                  -                 50.6%

Örnek, temel HTTP sözdizimini kullanır. Örnekteki herhangi bir “<#>” kopyalanırken kaldırılmalıdır.
