İlişkisel Veritabanları ve Elasticsearch Arasındaki Fark

  • Terminoloji Farkı
  • İlişkisel Veritabanlarının uygun olmadığı durumlar

Bu sayfada

  • Terminoloji Farkı
  • İlişkisel Veritabanlarının uygun olmadığı durumlar

Bu, ilişkisel altyapıdan gelen ve elasticsearch’ü öğrenmek isteyen okuyucular içindir. Bu konu, İlişkisel veritabanlarının uygun bir seçenek olmadığı kullanım örneklerini gösterir.

Terminoloji Farkı #

İlişkisel Veritabanı Elasticsearch
Veritabanı dizin
Tablo Tip
Satır/Kayıt Belge
Sütun Adı alan

Yukarıdaki tablo, ilişkisel veritabanının temel öğeleri ile elastik aramanın temel öğeleri arasında kabaca bir analoji çizmektedir.

Kurmak

İlişkisel bir veritabanında aşağıdaki yapıyı göz önünde bulundurarak:

create databse test;

use test;

create table product;

create table product (name varchar, id int PRIMARY KEY);

insert into product  (id,name) VALUES (1,'Shirt');

insert into product  (id,name) VALUES (2,'Red Shirt');

select * from product;

name      | id 
----------+----
Shirt     |  1
Red Shirt |  2

Elasticsearch Eşdeğeri:

POST test/product
{
    "id" : 1,
    "name" : "Shirt"
}

POST test/product
{
    "id" : 2,
    "name" : "Red Shirt"
}

GET test/product/_search


"hits": [
     {                                      ==============
        "_index": "test",                   ===> index    |
        "_type": "product",                 ===>type      |
        "_id": "AVzglFomaus3G2tXc6sB",                    |
        "_score": 1,                                      |
        "_source": {                                      |===> document
           "id": 2,                        ===>field      | 
           "name": "Red Shirt"             ===>field      |
        }                                                 |
     },                                     ==============
     {
        "_index": "test",
        "_type": "product",
        "_id": "AVzglD12aus3G2tXc6sA",
        "_score": 1,
        "_source": {
           "id": 1,                 
           "name": "Shirt"           
        }
     }
  ]

İlişkisel Veritabanlarının uygun olmadığı durumlar #

  • Aramanın özü, düzeninde yatmaktadır. Herkes, arama sonuçlarının en uygun sonuçların en üstte gösterileceği şekilde gösterilmesini ister. İlişkisel veritabanının böyle bir yeteneği yoktur. Elasticsearch ise sonuçları varsayılan olarak alaka düzeyine göre gösterir.

Kurmak

Önceki örnekte kullanılanla aynı.

Sorun bildirimi

Diyelim ki kullanıcı ‘gömlek’ aramak istiyor ama ‘kırmızı’ renkli gömleklerle ilgileniyor. Bu durumda, “red” ve “shirts” anahtar kelimelerini içeren sonuçlar en üstte gelmelidir. Ardından diğer gömleklerin sonuçları onlardan sonra gösterilmelidir.

İlişkisel Veritabanı Sorgusu Kullanan Çözüm

`’%Kırmızı%’ veya ‘%Shirt%’ gibi bir ismin olduğu üründen * seçin’;

**Output**

   name       | id 
   -----------+----
   Shirt      |  1
   Red Shirt  |  2

**Elasticsearch Solution**

   POST test/product/_search
   {
        "query": {
             "match": {
               "name": "Red Shirt"
            }
        }
   }

**Output**

   "hits": [
     {
        "_index": "test",
        "_type": "product",
        "_id": "AVzglFomaus3G2tXc6sB",
        "_score": 1.2422675,              ===> Notice this
        "_source": {
           "id": 2,
           "name": "Red Shirt"
        }
     },
     {
        "_index": "test",
        "_type": "product",
        "_id": "AVzglD12aus3G2tXc6sA",
        "_score": 0.25427115,             ===> Notice this
        "_source": {
           "id": 1,
           "name": "Shirt"
        }
     }
    ]

 **Conclusion**

Yukarıda gördüğümüz gibi, İlişkisel Veritabanı bazı rastgele sırayla sonuçları döndürürken, Elasticsearch alaka düzeyine göre hesaplanan ‘_score’un azalan sırasını döndürür.

  • Arama dizisine girerken hata yapmaya meyilliyiz. Kullanıcının yanlış bir arama parametresi girdiği durumlar vardır. İlişkisel Veritabanları bu tür durumları işlemez. Kurtarmaya Elasticsearch.

Kurmak

Önceki örnekte kullanılanla aynı.

Sorun bildirimi

Kullanıcının “gömlek” aramak istediğini, ancak yanlışlıkla “shrt” kelimesini yanlış girdiğini varsayalım. Kullanıcı hala gömlek sonuçlarını görmeyi umuyor.

İlişkisel Veritabanı Sorgusu Kullanan Çözüm

`’%shrt%’ gibi ismin olduğu üründen * seçin’;

**Output**

   No results found

**Elasticsearch Solution** 
  
   POST /test/product/_search

    {
       "query": {
         "match": {
           "name": {
             "query": "shrt",
             "fuzziness": 2,
             "prefix_length": 0
            }
         }
       }
    }

Çıktı

     {

           "id": 2,
           "name": "Red Shirt"
        }
     }
   ]

 **Conclusion**

Yukarıda gördüğümüz gibi, ilişkisel veritabanı, aranan yanlış bir kelime için hiçbir sonuç döndürmezken, Elasticsearch özel fuzzy sorgusunu kullanarak sonuç döndürür.
