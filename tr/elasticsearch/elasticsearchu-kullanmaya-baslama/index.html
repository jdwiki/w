Elasticsearch’ü Windows’a Yükleme #

Önkoşullar #

Elasticsearch’ün Windows sürümü şu bağlantıdan edinilebilir: https://www.elastic.co/downloads/elasticsearch. En son kararlı sürüm her zaman en üsttedir.

Windows’a kurulum yaparken .ZIP arşivine ihtiyacımız var. İndirilenler: bölümündeki bağlantıya tıklayın ve dosyayı bilgisayarınıza kaydedin.

Elastikin bu sürümü “taşınabilir"dir, yani programı kullanmak için bir yükleyici çalıştırmanız gerekmez. Dosyanın içeriğini kolayca hatırlayabileceğiniz bir konuma açın. Gösteri için, her şeyi C:\elasticsearch klasörüne açtığınızı varsayacağız.

Arşivin varsayılan olarak “elasticsearch-” adlı bir klasör içerdiğini unutmayın, bu klasörü “C:" klasörüne çıkarabilir ve “elasticsearch” olarak yeniden adlandırabilir veya kendiniz “C:\elasticsearch” oluşturabilir, ardından yalnızca sıkıştırmayı açabilirsiniz. arşivdeki klasörün içeriği oraya.

Elasticsearch, Java ile yazıldığından, çalışması için Java Runtime Environment’a ihtiyaç duyar. Bu nedenle, sunucuyu çalıştırmadan önce, bir komut istemi açıp şunu yazarak Java’nın kullanılabilir olup olmadığını kontrol edin:

java -version

Şuna benzeyen bir yanıt almalısınız:

java version "1.8.0_91"
Java(TM) SE Runtime Environment (build 1.8.0_91-b14)
Java HotSpot(TM) Client VM (build 25.91-b14, mixed mode)

Bunun yerine aşağıdakileri görürseniz

‘java’ dahili veya harici bir komut, çalıştırılabilir program veya toplu iş dosyası olarak tanınmıyor.

Java, sisteminizde yüklü değil veya düzgün yapılandırılmamış. Java’yı (yeniden) yüklemek için bu öğreticiyi takip edebilirsiniz. Ayrıca, bu ortam değişkenlerinin benzer değerlere ayarlandığından emin olun:

Değişken Değer
JAVA_HOME C:\Program Dosyaları\Java\jre
YOL …;C:\Program Dosyaları\Java\jre

Bu değişkenleri nasıl inceleyeceğinizi henüz bilmiyorsanız bu eğiticiye bakın.

Toplu iş dosyasından çalıştır #

Java kuruluyken bin klasörünü açın. Her şeyi açtığınız klasörde doğrudan bulunabilir, bu nedenle c:\elasticsearch\bin altında olmalıdır. Bu klasörde, bir komut penceresinde Elasticsearch’ü başlatmak için kullanılabilecek elasticsearch.bat adlı bir dosya bulunur. Bu, işlem tarafından günlüğe kaydedilen bilgilerin komut istemi penceresinde görüneceği anlamına gelir. Sunucuyu durdurmak için CTRLC‘ye basın veya pencereyi kapatın.

Windows hizmeti olarak çalıştır #

İdeal olarak, geliştirme sırasında kurtulamayacağınız fazladan bir pencereye sahip olmak istemezsiniz ve bu nedenle Elasticsearch bir hizmet olarak çalışacak şekilde yapılandırılabilir.

Elasticsearch’ü hizmet olarak kurabilmemiz için C:\elasticsearch\config\jvm.options dosyasına bir satır eklememiz gerekiyor:

Servis yükleyici, servisi kurmadan önce iş parçacığı yığını boyutu ayarının jvm.options içinde yapılandırılmasını gerektirir. 32-bit Windows’ta, -Xss320k […] ve 64-bit Windows’ta jvm.options dosyasına -Xss1m eklemelisiniz. [kaynak]

Bu değişikliği yaptıktan sonra, bir komut istemi açın ve aşağıdaki komutu çalıştırarak bin dizinine gidin:

C:\Users\user> cd c:\elasticsearch\bin

Hizmet yönetimi “elasticsearch-service.bat” tarafından gerçekleştirilir. Eski sürümlerde bu dosyanın adı sadece ‘service.bat’ olabilir. Mevcut tüm argümanları görmek için, herhangi bir argüman olmadan çalıştırın:

C:\elasticsearch\bin> elasticsearch-service.bat

Usage: elasticsearch-service.bat install|remove|start|stop|manager [SERVICE_ID]

Çıktı ayrıca bize isteğe bağlı bir “SERVICE_ID” argümanı olduğunu söylüyor, ancak şimdilik onu görmezden gelebiliriz. Hizmeti yüklemek için şunu çalıştırın:

C:\elasticsearch\bin> elasticsearch-service.bat install

Hizmeti kurduktan sonra, ilgili argümanlarla başlatabilir ve durdurabilirsiniz. Hizmeti başlatmak için çalıştırın

C:\elasticsearch\bin> elasticsearch-service.bat start

ve onu durdurmak için koş

C:\elasticsearch\bin> elasticsearch-service.bat stop

Bunun yerine hizmeti yönetmek için bir GUI’yi tercih ederseniz, aşağıdaki komutu kullanabilirsiniz:

C:\elasticsearch\bin> elasticsearch-service.bat manager

Bu, hizmetle ilgili bazı ayarları özelleştirmenize ve ilk sekmenin altında bulunan düğmeleri kullanarak hizmeti durdurmanıza/başlatmanıza izin veren Elastik Hizmet Yöneticisini açacaktır.

Elasticsearch’ü Ubuntu’ya Yükleme 14.04 #


Önkoşullar #

Elasticsearch’ü çalıştırmak için makinede bir Java Runtime Environment (JRE) gereklidir. Elasticsearch, Java 7 veya üstünü gerektirir ve ‘Oracle JDK sürüm 1.8.0_73’ü önerir.

Oracle Java 8’i yükleyin

sudo add-apt-repository -y ppa:webupd8team/java
sudo apt-get update
echo "oracle-java8-installer shared/accepted-oracle-license-v1-1 select true" | sudo debconf-set-selections
sudo apt-get install -y oracle-java8-installer

Java Sürümünü Kontrol Edin

java -version

Paketi indirin ve yükleyin #

İkili Dosyaları Kullanma

  1. Elasticsearch’ün en son kararlı sürümünü [buradan] indirin(https://www.elastic.co/downloads/elasticsearch).
  2. Dosyayı açın ve Çalıştırın

Linux:

$ bin/elasticsearch

apt-get’i kullanma

Elasticsearch’ü web sitesinden indirmenin bir alternatifi, ‘apt-get’ kullanarak yüklemektir.

wget -qO - https://packages.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -
echo "deb https://packages.elastic.co/elasticsearch/2.x/debian stable main" | sudo tee -a /etc/apt/sources.list.d/elasticsearch-2.x.list
sudo apt-get update && sudo apt-get install elasticsearch
sudo /etc/init.d/elasticsearch start

Elasticsearch sürüm 5.x’i yükleme

wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -
sudo apt-get install apt-transport-https
echo "deb https://artifacts.elastic.co/packages/5.x/apt stable main" | sudo tee -a /etc/apt/sources.list.d/elastic-5.x.list
sudo apt-get update && sudo apt-get install elasticsearch

Linux’ta hizmet olarak çalıştırma: #

Yukarıdakileri kurduktan sonra kendi kendine başlamıyor. bu yüzden bir hizmet olarak başlatmamız gerekiyor. Elasticsearch’ün nasıl başlatılacağı veya durdurulacağı, sisteminizin SysV init mi yoksa systemd mi kullandığına bağlıdır. aşağıdaki komutla kontrol edebilirsiniz.

ps -p 1

Dağıtımınız SysV init kullanıyorsa, şunları çalıştırmanız gerekir:

sudo update-rc.d elasticsearch defaults 95 10
sudo /etc/init.d/elasticsearch start

Aksi takdirde, dağıtımınız systemd kullanıyorsa:

sudo /bin/systemctl daemon-reload
sudo /bin/systemctl enable elasticsearch.service

Elasticsearch’ün doğru yüklenip yüklenmediğini kontrol etmek için tarayıcınızdan veya bir REST istemcisinden ‘CURL’ komutunu çalıştırın.

curl -X GET http://localhost:9200/

Bir belgeyi indeksleme ve alma #

Elasticsearch’e, genellikle cURL kitaplığı kullanılarak bir HTTP REST API aracılığıyla erişilir. Arama sunucusu ve istemci (sizin veya uygulamanız) arasındaki mesajlar JSON dizeleri biçiminde gönderilir. Elasticsearch varsayılan olarak 9200 numaralı bağlantı noktasında çalışır.

Aşağıdaki örneklerde, Elasticsearch’e JSON yanıtını güzelleştirmesini söylemek için “?pretty” eklenmiştir. Bu uç noktaları bir uygulama içinde kullanırken bu sorgu parametresini eklemeniz gerekmez.

Elasticsearch ve Kibana’yı CentOS 7’ye Yükleme #

Elasticsearch’ü çalıştırmak için makinede bir Java Runtime Environment (JRE) gereklidir. Elasticsearch, Java 7 veya üstünü gerektirir ve ‘Oracle JDK sürüm 1.8.0_73’ü önerir.

Bu nedenle, sisteminizde Java olup olmadığından emin olun. Değilse, prosedürü izleyin:

# Install wget with yum 
yum -y install wget

# Download the rpm jre-8u60-linux-x64.rpm for 64 bit 
wget --no-cookies --no-check-certificate --header "Cookie: gpw_e24=http%3A%2F%2Fwww.oracle.com%2F; oraclelicense=accept-securebackup-cookie" "http://download.oracle.com/otn-pub/java/jdk/8u60-b27/jre-8u60-linux-x64.rpm"

# Download the rpm jre-8u101-linux-i586.rpm for 32 bit
wget --no-cookies --no-check-certificate --header "Cookie: gpw_e24=http%3A%2F%2Fwww.oracle.com%2F; oraclelicense=accept-securebackup-cookie" "http://download.oracle.com/otn-pub/java/jdk/8u101-b13/jre-8u101-linux-i586.rpm"

# Install jre-.*.rpm
rpm -ivh jre-.*.rpm

Java şimdiye kadar centOS sisteminize yüklenmiş olmalıdır. Şununla kontrol edebilirsiniz:

java -version

Elasticsearch’ü indirin ve kurun

# Download elasticsearch-2.3.5.rpm 
wget https://download.elastic.co/elasticsearch/release/org/elasticsearch/distribution/rpm/elasticsearch/2.3.5/elasticsearch-2.3.5.rpm

# Install elasticsearch-.*.rpm
rpm -ivh elasticsearch-.*.rpm

Elasticsearch’ü başlangıçta sistemd hizmeti olarak çalıştırma

sudo systemctl daemon-reload
sudo systemctl enable elasticsearch
sudo systemctl start elasticsearch

# check the current status to ensure everything is okay.
systemctl status elasticsearch

Kibana’yı yükleme

İlk önce GPG anahtarını rpm’de içe aktarın

sudo rpm --import http://packages.elastic.co/GPG-KEY-elasticsearch

Ardından yerel bir kibana.repo deposu oluşturun

sudo vi /etc/yum.repos.d/kibana.repo

Ve aşağıdaki içeriği ekleyin:

[kibana-4.4]
name=Kibana repository for 4.4.x packages
baseurl=http://packages.elastic.co/kibana/4.4/centos
gpgcheck=1
gpgkey=http://packages.elastic.co/GPG-KEY-elasticsearch
enabled=1

Şimdi aşağıdaki komutu kullanarak kibana’yı kurun:

yum -y install kibana

Şununla başlayın:

systemctl start kibana

Durumu şununla kontrol edin:

systemctl status kibana

Bir başlangıç ​​hizmeti olarak çalıştırabilirsiniz.

systemctl enable kibana

Örneklerle birlikte Temel Arama Parametreleri: #

Varsayılan olarak, tam dizine alınmış belge, tüm aramaların bir parçası olarak döndürülür. Bu, kaynak (arama isabetlerinde “_source” alanı) olarak adlandırılır. Kaynak belgenin tamamının döndürülmesini istemiyorsak, kaynak içinden yalnızca birkaç alanın döndürülmesini talep etme olanağına sahibiz veya alanı tamamen atlamak için “_kaynak"ı false olarak ayarlayabiliriz.

Bu örnek, aramadan “hesap_numarası” ve “denge” ("_kaynak"ın içinde) olmak üzere iki alanın nasıl döndürüleceğini gösterir:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match_all": {} },
  "_source": ["account_number", "balance"]
}'

Yukarıdaki örneğin yalnızca “_source” alanında döndürülen bilgileri azalttığına dikkat edin. Yine de ‘_source’ adlı yalnızca bir alan döndürecek, ancak yalnızca ‘hesap_numarası’ ve ‘bakiye’ alanları dahil edilecek.

Bir SQL arka planından geliyorsanız, yukarıdakiler kavram olarak SQL sorgusuna biraz benzer.

SELECT account_number, balance FROM bank;

Şimdi sorgulama kısmına geçelim. Daha önce, tüm belgeleri eşleştirmek için “match_all” sorgusunun nasıl kullanıldığını görmüştük. Şimdi, temel bir alan arama sorgusu (yani belirli bir alan veya alan kümesine karşı yapılan bir arama) olarak düşünülebilecek eşleşme sorgusu adlı yeni bir sorguyu tanıtalım.

Bu örnek, “hesap_numarası"nın “20” olarak ayarlandığı hesabı döndürür:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match": { "account_number": 20 } }
}'

Bu örnek, “adres"te “değirmen” terimini içeren tüm hesapları döndürür:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match": { "address": "mill" } }
}'

Bu örnek, “adres"te “değirmen” veya “şerit” terimini içeren tüm hesapları döndürür:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match": { "address": "mill lane" } }
}'

Bu örnek, sorguyu terimlere bölen ve yalnızca “adres"teki tüm terimleri birbirine göre aynı konumlarda içeren belgeleri döndüren bir “eşleşme"nin (“eşleşme_deyimi”) bir türevidir[1].

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match_phrase": { "address": "mill lane" } }
}'

Şimdi bool(ean) sorgusunu tanıtalım. Bool sorgusu, boole mantığını kullanarak daha küçük sorguları daha büyük sorgulara dönüştürmemize olanak tanır.

Bu örnek, iki eşleşme sorgusu oluşturur ve adreste “değirmen” ve “şerit” içeren tüm hesapları döndürür:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": {
    "bool": {
      "must": [
        { "match": { "address": "mill" } },
        { "match": { "address": "lane" } }
      ]
    }
  }
}'

Yukarıdaki örnekte, bool “zorunlu” yan tümcesi, bir belgenin eşleşme olarak kabul edilmesi için doğru olması gereken tüm sorguları belirtir.

Buna karşılık, bu örnek iki eşleşme sorgusu oluşturur ve “adres"te “değirmen” veya “şerit” içeren tüm hesapları döndürür:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": {
    "bool": {
      "should": [
        { "match": { "address": "mill" } },
        { "match": { "address": "lane" } }
      ]
    }
  }
}'

Yukarıdaki örnekte, bool ‘should’ yan tümcesi, bir belgenin eşleşme olarak kabul edilmesi için her ikisinin de doğru olması gereken bir sorgu listesi belirtir.

Bu örnek, iki eşleşme sorgusu oluşturur ve “adres"te ne “değirmen” ne de “şerit” içermeyen tüm hesapları döndürür:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": {
    "bool": {
      "must_not": [
        { "match": { "address": "mill" } },
        { "match": { "address": "lane" } }
      ]
    }
  }
}'

Yukarıdaki örnekte, bool must_not yan tümcesi, bir belgenin eşleşme olarak kabul edilmesi için hiçbirinin doğru olmaması gereken bir sorgu listesi belirtir.

Bir bool sorgusu içinde must, must ve must_not cümleciklerini aynı anda birleştirebiliriz. Ayrıca, herhangi bir karmaşık çok seviyeli boole mantığını taklit etmek için bu bool yan tümcelerinden herhangi birinin içinde bool sorguları oluşturabiliriz.

Bu örnek, tam olarak 40 yaşında olan ve Washington’da yaşamayan kişilere (kısaca “WA”) ait tüm hesapları döndürür:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": {
    "bool": {
      "must": [
        { "match": { "age": "40" } }
      ],
      "must_not": [
        { "match": { "state": "WA" } }
      ]
    }
  }
}'
