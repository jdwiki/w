Sözdizimi #

  • [ yerel bölüm “@” ] etki alanı bölümü [ “/” kaynak bölümü ]

Parametreler #

Bölüm Genel kullanım
Yerel Bölüm Bir XMPP varlığını tanımlar (isteğe bağlı)
Alan Bölümü XMPP hizmetini tanımlar
Kaynak Bölümü Bir XMPP varlığının oturumunu tanımlar (isteğe bağlı)

Daha yaygın olarak JID’ler (Jabber Tanımlayıcılar) olarak bilinen XMPP adresleri, RFC 7622 içinde tanımlanır ve XMPP ağında adresler olarak işlev görür. Bir e-posta adresi gibi görünürler, ancak bazen sonunda, adresin geri kalanı tarafından temsil edilen hesap olarak oturum açan belirli bir müşteriyi tanımlayan isteğe bağlı bir “kaynak bölümü” bulunur (çünkü XMPP’nin hesap başına birden fazla müşterisi olabilir). Kaynak bölümü (bir istemci) “xyz” olan bir XMPP adresi örneği:

romeo@example.net/xyz

JID’yi bölme (genel) #

Bir JID’yi bileşen bölümlerine (yerel bölüm, etki alanı bölümü ve kaynak bölümü) bölmek için aşağıdaki algoritma kullanılmalıdır (burada yerel bölüm “lp”, kaynak bölümü “rp” ve etki alanı bölümü “dp” ile temsil edilir ve ∈, verilen karakterin dizgeye dahil olup olmadığını kontrol etmek için kullanılır):

Yerel bölüm ve kaynak bölümünün isteğe bağlı olduğunu ve boş dizelerle sonuçlanabileceğini unutmayın (sadece bir etki alanı bölümü olan bir jid’iniz olabilir).

JID Türleri #

Bir JID üç bölümden oluşur: localpart@domainpart/resourcepart.

Tam JID’ler (her zaman bir kaynak bölümü vardır)

romeo@example.org/orchard
example.org/da863ab

Çıplak JID’ler (her zaman kaynak kısmı olmadan)

 romeo@example.org
 example.org

JID’yi doğrulama (genel) #

E-postalardan farklı olarak, JID’ler, aşağıdakiler kullanılarak Uluslararasılaştırma (i18n) göz önünde bulundurularak tanımlandı: Internationalized Strings (PRECIS) çerçevesinin hazırlanması, uygulanması ve karşılaştırılması. PRECIS (RFC 7564 içinde tanımlanmıştır), çeşitli bağlamlarda dizeleri güvenli bir şekilde karşılaştırmak için bir çerçevedir. Örneğin, bir grup sohbetinde “Richard IV” (Latin büyük harfleri I, Vee) takma adını kaydettiğinizi düşünün: PRECIS kullanarak sohbet uygulaması başka kimsenin gelmemesini ve “Richard Ⅳ” (Unicode Roman) takma adını kaydetmesini sağlayabilir. Rakam 4) ve sizi taklit etmek için kullanır.

Halihazırda yerel bölümü, etki alanı bölümü ve kaynak bölümü olarak ayrılmış bir JID’yi doğrulamak için kullanılan algoritma (Bkz. https://www.wikiod.com/tr/xmpp/xmpp-adresleri-aka-jidler-jabber-tanmlayclar bir JID’nin parçalarını bir dizgeden çıkarma hakkında bilgi için split-a-jid) aşağıdaki gibidir:

“Doğrulamalar” adımı aşağıdakileri gerçekleştirmelidir:

  • Yerel bölümün 1024 bayttan az olduğunu kontrol edin (bayt, glif değil)
  • Yerel bölümün "&'/:<>@ içermediğini kontrol edin
  • Kaynak bölümünün 1024 bayttan az olduğunu kontrol edin
  • Etki alanı bölümünün sıfır bayttan büyük ve 1024 bayttan küçük olduğunu kontrol edin (ve muhtemelen etki alanının tek tek bölümlerinin DNS gereksinimlerine uyduğunu doğrulayın)
  • Etki alanı geçerli bir IPv6 adresiyse, parantez içine alınmış notasyon kullandığından emin olun (ör. “::1” yerine “[::1]”)

JID’yi bölme (Git) #

mellium.im/xmpp/jid paketi, JID’ler üzerinde işlemler uygular. Bir JID dizesini bileşen parçalarına bölmek için “SplitString” işlevi kullanılabilir:

lp, dp, rp, err := SplitString("romeo@example.net")

İşlev tarafından doğrulama yapılmaz ve parçaların geçerliliği garanti edilmez.

Bir dizgiyi “jid” paketine bağlı olmadan manuel olarak bölmek için, alttaki kod şöyle görünür:

// SplitString splits out the localpart, domainpart, and resourcepart from a
// string representation of a JID. The parts are not guaranteed to be valid, and
// each part must be 1023 bytes or less.
func SplitString(s string) (localpart, domainpart, resourcepart string, err error) {

    // RFC 7622 §3.1.  Fundamentals:
    //
    //    Implementation Note: When dividing a JID into its component parts,
    //    an implementation needs to match the separator characters '@' and
    //    '/' before applying any transformation algorithms, which might
    //    decompose certain Unicode code points to the separator characters.
    //
    // so let's do that now. First we'll parse the domainpart using the rules
    // defined in §3.2:
    //
    //    The domainpart of a JID is the portion that remains once the
    //    following parsing steps are taken:
    //
    //    1.  Remove any portion from the first '/' character to the end of the
    //        string (if there is a '/' character present).
    sep := strings.Index(s, "/")

    if sep == -1 {
        sep = len(s)
        resourcepart = ""
    } else {
        // If the resource part exists, make sure it isn't empty.
        if sep == len(s)-1 {
            err = errors.New("The resourcepart must be larger than 0 bytes")
            return
        }
        resourcepart = s[sep+1:]
        s = s[:sep]
    }

    //    2.  Remove any portion from the beginning of the string to the first
    //        '@' character (if there is an '@' character present).

    sep = strings.Index(s, "@")

    switch sep {
    case -1:
        // There is no @ sign, and therefore no localpart.
        localpart = ""
        domainpart = s
    case 0:
        // The JID starts with an @ sign (invalid empty localpart)
        err = errors.New("The localpart must be larger than 0 bytes")
        return
    default:
        domainpart = s[sep+1:]
        localpart = s[:sep]
    }

    // We'll throw out any trailing dots on domainparts, since they're ignored:
    //
    //    If the domainpart includes a final character considered to be a label
    //    separator (dot) by [RFC1034], this character MUST be stripped from
    //    the domainpart before the JID of which it is a part is used for the
    //    purpose of routing an XML stanza, comparing against another JID, or
    //    constructing an XMPP URI or IRI [RFC5122].  In particular, such a
    //    character MUST be stripped before any other canonicalization steps
    //    are taken.

    domainpart = strings.TrimSuffix(domainpart, ".")

    return
}

JID’yi (Pas) bölme #

Rust’ta xmpp-addr ([docs]) kasası JID’leri işlemek için kullanılabilir. Bir JID’yi bileşen parçalarına bölmek için (bu parçaların geçerli olduğunu doğrulamadan), Jid::split işlevi kullanılabilir:

let (lp, dp, rp) = Jid::split("feste@example.net")?;
assert_eq!(lp, Some("feste"));
assert_eq!(dp, "example.net");
assert_eq!(rp, None);
