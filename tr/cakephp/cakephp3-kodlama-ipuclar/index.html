    public function initialize(){

    }
    public function view($id) {
       //Your code here

$_POST’a eşdeğer gönderi verileri alınıyor #

Gönderi verilerini Array olarak alabilirsiniz.

$post_data= $this->request->data;

Belirli bir anahtar için gönderi verilerini alabilirsiniz.

$this->request->data['field'];

Belirli anahtar değerini al

$this->request->data('key_name');

İç içe dizinin belirli anahtar değerini alın

$this->request->data('data.subfield');

dizi gösterimi ile “data()” yöntemi arasındaki fark, “data()“nın hataya karşı güvenli olması ve anahtar dizide yoksa “null” döndürmesidir.

yapmak yerine

if(isset($this->request->data['field']) && $this->request->data['field']) { ...}

yapabilirsin

if($this->request->data('field')) { ...}

CakePHP 3.4.x için +

tüm verileri al:

$this->request->getData();

belirli anahtarı al:

$this->request->getData('key');

getData işlevi için mevcut verileri ayarlamak için şöyle bir şey yapmanız gerekir:

$this->request = $this->request->withData('some_key_on_the_fly', 'value');
$some_key_on_the_fly = $this->request->getData('some_key_on_the_fly');

denetleyicideki modelleri statik verilerle güncellemek için kullanışlıdır

Controller’da BeforeFilter() yöntemini ekleyin #

BeforeFilter() yöntemi, denetleyicide herhangi bir başka yöntem yürütülmeden önce yürütülür.

Denetleyici dosyanızda sınıfı tanımlamadan önce ilk olarak Event ad alanını kullanın.

use Cake\Event\Event;

Kontrol cihazınızda, aşağıda gösterildiği gibi BeforeFilter() yöntemini ekleyin.

public function beforeFilter(Event $event) {
    parent::beforeFilter($event);
}

Veya initialize() yöntemini kullanabilirsiniz.

public function initialize(){
    parent::initialize();
}

Değişkenleri Görünüme Geçmek #

Bir seferde görüntülemek için her değişkeni iletin

$this->set('color', 'pink');
$this->set('color', $color);

Compact() işlevi aracılığıyla birlikte görüntülemek için birden çok değişken iletin

$color1 = 'pink';
$color2 = 'red';
$this->set(compact('color1', 'color2'));

Yönlendirme ile URL’den Değişkeni Eyleme Geçirme #

URL’de Değişkeni yöntem parametresi olarak geçirme

return $this->redirect([
    'controller' => 'users',
    'action' => 'profile',
    $id
]);

URL şu şekilde görünmelidir: http://app_url/users/profile/{id}

UsersController.php dosyasında profile() yönteminde

class UsersController extends Controller {
    public function profile($id=null) {
        $userData=$this->Users->get($id);
    }
}

URL’de değişkeni Sorgu Dizesi olarak geçirme

return $this->redirect([
    'controller' => 'users',
    'action' => 'profile',
    '?'=>['id'=>$id]
]);

URL şu şekilde görünmelidir http://app_url/users/profile/?id={id}

UsersController.php dosyasında profile() yönteminde

class UsersController extends Controller {
    public function profile() {
        $userData=$this->Users->get($this->request->query('id'));
    }
}

Denetleyiciden başka bir sayfaya yönlendirme #

Uygulama içine yönlendir (belirli denetleyicinin başka bir eylemi).

return $this->redirect([
    'controller' => 'myController',
    'action' => 'myAction'
]);

Yönlendiren sayfasına yönlendir

return $this->redirect($this->referer());

Uygulamanın dışına veya belirli bir URL’ye yönlendirme

return $this->redirect("http://stackoverflow.com/users/1793428/haresh-vidja");

class AppsController extends Controller {

        parent::beforeFilter($event);
        $this->viewBuilder()->layout('admin'); // For Version >= 3.1 or
        $this->layout = 'admin'; // for version < 3.1
        
        // your other code should be here
    }
}

Uygulamadaki belirli eylem için varsayılan düzeni ayarlayın. yani, uygulamanın /src/Template/Layout/login.ctp içindeki oturum açma sayfasında farklı bir düzeni var

class UsersController extends Controller {

    public function login() {

        $this->viewBuilder()->layout('login'); // For Version >= 3.1 or
        $this->layout = 'login'; // for version < 3.1
        
        //your other code should be here

Ajax İstek Düzenini Ayarla #

Genellikle AJAX isteğinde CSS, JS yüklemeye gerek yoktur. Ayrıca diğer HTML kodunu atlayarak.

ajax.ctp dosyasını /src/Template/Layout içinde yapın ve kod şöyle olmalıdır:

<?php 
    $this->fetch('content');

AppsController.php içinde, tüm uygulama için AJAX tabanlı düzeni ayarlayın

class AppsController, Controller’ı genişletir {

public function beforeFilter(Event $event) {
    parent::beforeFilter($event);
    if($this->request->isAjax())
    {
        $this->viewBuilder()->layout('ajax'); // For Version >= 3.1 or
        $this->layout = 'ajax'; // for version < 3.1

    }
    else
    {
        $this->viewBuilder()->layout('admin'); // For Version >= 3.1 or
        $this->layout = 'admin'; // for version < 3.1
    }
    
    // your other code should be here
}

}

CakePHP’de Bileşenleri Yükle #

Bileşenleri iki şekilde yükleyebiliriz.

  1. Controller’da $components özelliğini başlatarak veya geçersiz kılarak
  2. Controller’ın initialize() yönteminde loadComponent() yöntemini kullanarak.

Yol-1 AppsController.php tarafından yükleme bileşenini geçersiz kılmalıdır bir veya daha fazla bileşen yükle

class UsersController extends AppController {
    public $components = ['RequestHandler','Auth','Flash']; 
}

Yol-2 Belirli bir denetleyici için bileşeni dinamik olarak yüklemeniz gerektiğinde bu yolu kullanın. Bir bileşen yükle

class UsersController extends AppController {
    public function initialize() {
        parent::initialize();
        $this->loadComponent("RequestHandler"); // load specific component
        $this->loadComponent(["RequestHandler","Auth","Flash"]); // load specific component
    }
}

initilaize() yöntemi nedir? #

initialize() CakePHP sürüm > 3.0’da tanıtıldı

Kod yapısı olarak, BeforeFilter() yöntemiyle aynı görünüyor. ancak BeforeFilter() ve initialize() arasında birçok fark vardır.

  1. initialize() her zaman kurucu çağrıldıktan sonra çağrılır. ancak, eylem yönteminin belirli denetleyicide bulunmaması durumunda BeforeFilter() çağrılmıyor.
  2. initialize() yöntemi genellikle yeni bileşenler ve yardımcılar eklemek gibi bir şeyi başlatmak için kullanılır. ancak önceFilter()‘de genellikle bazı global mantık kısımlarını yürütmek için kullanılır.

Denetleyiciye başka bir Model yükleyin #

Varsayılan olarak CakePHP, ilgili modeli denetleyiciye yükler. Denetleyiciye başka bir model yüklemek için loadModel() yöntemini kullanın:

$this->loadModel('Articles');

veya anında yük

$table = TableRegistry::get('Articles');
$table->find();

Tablo(Model) Sınıfı Oluşturma #

Kullanıcı Modeli Sınıfı nasıl oluşturulur

namespace App\Model\Table;
use Cake\ORM\Table;

class UsersTable extends Table {
    public function initialize(array $config) {
        $this->table('users'); //define table name
        $this->displayField('username'); // unique or other special field of users table
        $this->primaryKey('id'); // primary key of users table
        $this->tablePrefix('prefix_'); // if prefix set tablename should be prefix_users

        // your other code here
    }

    // your other methods here
}

CakePHP’de Model İlişkilendirmeleri #

CakePHP’de tanımlayabileceğimiz 4 tip ilişki(ilişki) vardır.

class PostsTable extends Table {
    public function initialize(array $config) {

        // table initialization code should be here

        $this->belongsTo('Authors', [
            'className' => 'Authors',
            'foreignKey' => 'author_id',
            'joinType' => 'INNER',
        ]);
        $this->hasMany('Tags');
        $this->hasOne('Categories');
        $this->hasAndBelongsToMany('Topics');
    }
}

Yukarıdaki örnekte 4 tür ilişki görebilirsiniz.

Yazı **Yazara (Bire Bir) aittir, bu, “yazılar” tablosunda “yazarlar” tablosunun “kimliği” ile ilişkili bir yabancı anahtar “yetkili_kimliği” olduğu anlamına gelir.

Gönderinin çok Etiketi var (Birden Çoka), bunun anlamı, ’etiketler’ tablosunda, ‘gönderiler’ tablosunun ‘id’ ile ilişkilendirilmiş bir yabancı ‘post_id’ anahtarı olduğu anlamına gelir.

Gönderinin bir Etiketi var (Birden Bire veya Bire Bir), bu, ‘gönderiler’ tablosunda, ‘kategoriler’ tablosunun ‘id’ ile ilişkilendirilmiş bir yabancı anahtar ‘category_id’ olduğu anlamına gelir.

Gönderi **Konu (Çoktan Çoka)‘ya aittir ve buna aittir, bu ‘gönderiler’ ve ‘konular’ tablosu arasındaki çoktan çoğa ilişkidir. çoktan çoğa ilişkiyi sürdürmek için üçüncü tablo, tablo oluşturmanız gerekir, tablo adı ‘posts_categories’ olmalıdır. Bu tablonun alanları aşağıda belirtildiği gibi

  1. id (tablonun birincil anahtarı)
  2. post_id (posts tablosunun yabancı anahtarı)
  3. topic_id (“konular” tablosunun yabancı anahtarı)
