Apache’nin “VirtualHost"u için ana giriş noktası Apache Virtual Host belgelerinde. Oradan, sanal ana bilgisayar yapılandırması hakkında genel belgelere ve ‘VirtualHost’ ve ilgili yönergelere ilişkin başvuru belgelerine sahip olursunuz.

1) IP tabanlı sankonlar 2) Aynı Port ile birden fazla sankon 3) Makro kullanarak sankon tanımlama (Apache2.4) #

1) IP tabanlı hayaletler

 <VirtualHost 192.168.13.37>
     ServerName example.com
     DocumentRoot /var/www/domains/example.com/html
     ErrorLog /var/log/example.com/error.log
     CustomLog /var/log/example.com/access.log common
 </VirtualHost>

 <VirtualHost 192.168.47.11>
     ServerName otherurl.com
     DocumentRoot /srv/www/htdocs/otherurl.com/html
     ErrorLog /var/log/otherurl.com/error.log
     CustomLog /var/log/otherurl.com/access.log common
 </VirtualHost>

Portu verilen IP(ler) ile değiştirmeniz yeterlidir. Hangi vhost’un seçileceğine karar vermek için bağlantı noktası önemsizdir.

2) Aynı Bağlantı Noktasına sahip birden fazla sankon

NameVirtualHost artık gerekli olmadığından, aynı port ile birden fazla sankon yazabilirsiniz.

<VirtualHost *:80>
     DocumentRoot /srv/www/htdocs/otherurl.com/html
     ErrorLog /var/log/otherurl.com/error.log
     CustomLog /var/log/otherurl.com/access.log common
</VirtualHost>

<VirtualHost *:80>
     ServerName example.com
     ServerAlias ex1.com ex2.com
     DocumentRoot /var/www/domains/example.com/html
     ErrorLog /var/log/example.com/error.log
     CustomLog /var/log/example.com/access.log common
 </VirtualHost>

Burada tam tersi geçerlidir: IP önemsizdir, ancak istek 80 numaralı bağlantı noktasından alınırsa girdiğiniz ad değerlendirilir. Ex1.com’u aradın mı, 2. hayalet seçildi. Ve eğer başka bir url’yi aradıysanız (diğer url.com ve ayrıca example3.com gibi) ilki seçilecektir. İsterseniz bu hayaleti bir ‘yedek’ olarak kullanabilirsiniz.

3) Makro kullanarak sankon tanımlama (Apache2.4)

<Macro VHost $port $host>
    <VirtualHost *:$port>
        Servername $host
        DocumentRoot /srv/www/htdocs/$host
        ErrorLog /var/log/$host/error.log
    </VirtualHost>
</Macro>

Use VHost 80 example.com
Use VHost 443 secure_example.com

Biri bağlantı noktası 80, diğeri 443 için olmak üzere iki sankon oluşturur ve kullanılan değişkenleri buna göre ayarlar.

Ad tabanlı sanal ana bilgisayar yapılandırması #

Apache’de isme dayalı sanal barındırma, Apache web sitesinde şu şekilde açıklanmıştır:

Ad tabanlı sanal barındırma ile sunucu, ana bilgisayar adını HTTP başlıklarının bir parçası olarak bildirin. Bu tekniği kullanarak, birçok farklı ana bilgisayar aynı IP adresini paylaşabilir.

Dolayısıyla bu yöntemle birden fazla web sitesi bir sunucuda barındırılabilir. Ubuntu’da, yapılandırma dosyaları /etc/apache2/sites-available içindedir. Bu dizinde 000-default.conf dosyasını bulacaksınız. Bu varsayılan yapılandırmadır, diğerleri ayarlanana kadar tüm istekler bu yapılandırma dosyasına gönderilecektir.

Bir sanal ana makine kurmak için burada example.com kullanılacaktır, ancak bunu domain.com ile değiştirmelisiniz. Varsayılan dosyayı kopyalayın:

 cp 000-default.conf example.com.conf

Yapılandırma dosyası aşağıdaki yönergelere sahip olabilir:

 <VirtualHost *:80>
        ServerAdmin admin@example.com
        ServerName example.com
        ServerAlias www.example.com

        DocumentRoot /var/www/example.com/html
        
        ErrorLog /var/log/apache/logs/error.log
        # Possible values include: debug, info, notice, warn, error, crit,
        # alert, emerg.
        LogLevel warn

        CustomLog /var/log/apache/logs/access.log combined    
</VirtualHost>
  • İlk satır, 80 numaralı bağlantı noktasındaki (varsayılan http bağlantı noktası) tüm isteklerin eşleşmesi gerektiğini belirtir. Ayrıca sunucunun IP’si olan * yerine bir IP adresiniz olabilir.
  • ServerAdmin, http hata mesajlarını görüntülemek için kullanılan web sitesi yöneticisinin iletişim bilgileridir.
  • SunucuAdı web sitesinin alan adıdır.
  • ServerAlias, web sitesinin ikincil adıdır ve genellikle www.domain.com olacaktır.
  • DocumentRoot, bir web sitesine göz attığımız zaman yüklenen kök klasördür.
  • ErrorLog, hataların yönlendirildiği dosyadır
  • “Günlük Seviyesi”. günlüğe gönderilecek hataların seviyesidir
  • CustomLog erişim bilgilerinin yönlendirildiği dosyadır

example.com’u web sitenizin alan adı ve web sitesi dosyaları için uygun dizin ile değiştirerek dosyayı düzenleyin.

Dosyayı kaydedin ve aşağıdaki Apache komutuyla siteyi etkinleştirin:

sudo a2ensite example.com.conf

apache’yi yeniden yükle

sudo service apache2 reload

Kontrol edilmesi gereken birkaç şey daha:

  • Alan adınız için DNS’inizin doğru IP için ayarlandığından emin olun (bunun yayılması zaman alabilir)
  • Güvenlik duvarında 80 numaralı bağlantı noktanızın açık olduğundan emin olun
  • Sunucu dosyalarında dosya izinlerinizin doğru ayarlandığından emin olun - sahiplik www-data:www-data ve dizin izinleri 750 ve dosya izinleri 640 olmalıdır.

Sanal sunucunuz çalışır durumda olmalı! Bunu, farklı bir yapılandırma dosyasıyla (aynı adlandırma kuralını kullanarak) ve /var/www altındaki farklı dizinlerle aynı sunucudaki diğer web siteleri için tekrarlayabilirsiniz.

WAMP’ta Sanal Ana Bilgisayar #

Sanal ana bilgisayar kullanarak HTTPS’yi zorla #

Kullanıcıları güvenli URL’ye bağlanmaya zorlamak için Yönlendir‘i kullanın.

<VirtualHost *:80>
    ServerName example.com
    SSLProxyEngine on
    Redirect permanent / https://secure_example.com/
</VirtualHost>

Yapılandırmanın geri kalanı, her şey yeniden yönlendirildiğinden ssl sanal ana bilgisayarına (bağlantı noktası 443) yerleştirilebilir.

<VirtualHost _default_:443>
    ServerName secure_example.com
    ServerAdmin webmaster@example.com 
    DocumentRoot /var/www/domains/secure_example.com/html
    ErrorLog /var/log/secure_example.com/error.log
    CustomLog /var/log/secure_example.com/access.log common
    SSLEngine On
    ...
</VirtualHost>

PHP Geliştirme Sanal Ana Bilgisayarı #

Bu, geliştirme ve hata ayıklama için bir sanal ana bilgisayar sitesinde PHP hata günlüğünün nasıl kontrol edileceğine dair bir örnektir. varsayımlar

  • PHP modülü kuruldu.

  • Geliştirme ortamı üretim için değildir.

    <VirtualHost *:80> ServerName example.com DocumentRoot /var/www/domains/example.com/html ErrorLog /var/www/domains/example.com/apache.error.log CustomLog /var/www/domains/example.com/apache.access.log common php_flag log_errors on php_flag display_errors on php_value error_reporting 2147483647 php_value error_log /var/www/domains/example.com/php.error.log

Not: Sanal Ana Bilgisayar yapılandırması yalnızca geliştirme içindir çünkü display_errors etkindir ve bunu üretimde istemezsiniz.
