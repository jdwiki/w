Scolector Süreç ve Hizmet İzleme

Bu sayfada

  • Linux süreci ve sistemd hizmeti izleme

  • Windows işlem ve hizmet izleme

Scollector, Windows ve Linux’ta süreçleri ve hizmetleri izlemek için kullanılabilir. IIS uygulama havuzları gibi bazı işlemler otomatik olarak izlenir, ancak genellikle hangi işlemleri ve hizmetleri izlemek istediğinizi belirtmeniz gerekir.

Linux süreci ve sistemd hizmeti izleme #

Scollector, yapılandırma dosyasında belirtilen Linux işlemlerini izleyecektir.

[[Process]]
  Command = "/opt/bosun/bosun"
  Name = "bosun"

[[Process]]
  Command = "ruby"
  Name = "puppet-agent"
  Args = "puppet"

[[Process]]
  Command = "/haproxy$"
  Name = "haproxy-t1"
  Args = "/etc/haproxy-t1/haproxy-t1.cfg"

[[Process]]
  Command = '/usr/bin/redis-server \*:16389'
  Name = "redis-bosun-dev"
  IncludeCount = true

Scollector, systemd tarafından yönetilen hizmetlerin ve yapılandırma dosyasında belirtilen durumunu belirlemek için D-Bus API’sini de kullanabilir.

[[SystemdService]]
  Name = "^(puppet|redis-.*|keepalived|haproxy-t.*)$"
  WatchProc = false

[[SystemdService]]
  Name = "^(scollector|memcached)$"
  WatchProc = true

Windows .NET işlem izleme #

Scollector ayrıca herhangi bir Windows’u [.NET çerçevesini kullanan işlemleri] de izleyebilir (http://bosun.org/scollector/process-monitoring#net-processes). ProcessDotNet ayarları belirtilmezse, varsayılan olarak yalnızca IIS için w3wp çalışan işlemlerini izlemek için kullanılır. Yapılandırma dosyasında hangi uygulamaların izleneceğini belirtebilirsiniz.

[[ProcessDotNet]]
  Name = "^w3wp"

[[ProcessDotNet]]
  Name = "LINQPad"

Eşleşme süreci dotnet.* metrikleri altında izlenecek ve eğer birden fazla eşleştirme işlemi varsa, bunlara 1’den başlayarak artan id etiketi değerleri atanacaktır. süreç izleme için.

Docker Konteynerlerini İzleme #

Scollector, bir ana bilgisayardaki her Docker kapsayıcısı için Bosun’da container.* metrikleri oluşturmak üzere cAdvisor kullanımına yönelik yerleşik destek sunar. Başlamak için her liman işçisi ana bilgisayarında yeni bir kapsayıcı başlatmanız gerekir:

docker run --name cadvisor --restart=always -d -p 8080:8080 google/cadvisor

Ardından, Cadvisor yapılandırma seçeneğiyle scollector kullanarak metrikler için harici bir kaynak anketinden. Kapsayıcıları yönetmek için Kubernetes kullanıyorsanız, “docker_id” etiketlerini geçersiz kılmak (12 karaktere kısaltılmış) için TagOverride seçeneğini kullanmak, bir “container_name” ve “pod_name” etiketi eklemek ve “docker_name” ve “docker_name” etiketlerini kaldırmak isteyebilirsiniz. isim etiketi:

[[Cadvisor]]
  URL = "http://mydockerhost01:8080"

[[Cadvisor]]
  URL = "http://mydockerhost02:8080"

#Override tags for Kubernetes containers
[[TagOverride]]
  CollectorExpr = "cadvisor"
  [TagOverride.MatchedTags]
    docker_name = 'k8s_(?P<container_name>[^\.]+)\.[0-9a-z]+_(?P<pod_name>[^-]+)'
    docker_id = '^(?P<docker_id>.{12})'
  [TagOverride.Tags]
    docker_name = ''
    name = ''

Ayrıca, bir üretim Bosun örneğine göndermeden önce metriklerin doğru göründüğünü doğrulamak için (belki [Bosun Docker Container[2] kullanarak) bir Bosun test örneğine metrikleri göndermek isteyebilirsiniz (bundan sonra verileri temizlemek zordur). gönderilmiş).

[2]: https://www.wikiod.com/tr/bosun/bosunu-kullanmaya-baslama#Docker Hızlı Başlangıç

Windows işlem ve hizmet izleme #

Scollector, yapılandırma dosyasında belirtilen tüm Windows işlemlerini veya hizmetlerini izleyecektir.

[[Process]]
  Name = "^scollector"

[[Process]]
  Name = "^chrome"

[[Process]]
  Name = "^(MSSQLSERVER|SQLSERVERAGENT)$"
