Bosun Şablonları, bir bildirim gönderirken daha fazla bilgi sağlamak için grafikler içerebilir. Grafikler, uyarı örneği için etiket kümesindeki uyarı ve filtre tabanındaki değişkenleri kullanabilir veya tüm serilerin grafiğini oluşturmak için GraphAll işlevini kullanabilir. Gösterge Tablosunda veya bir e-postada görüntülendiğinde, İfade sayfasına yüklemek için grafiğe tıklayabilirsiniz.

Ayrıca, birden çok uyarı arasında paylaşılabilen bir İsteğe bağlı Grafiklere sahip Genel Şablon oluşturabilirsiniz.

2: https://www.wikiod.com/tr/bosun/sablonlara-genel-baks#İsteğe bağlı Grafiklerle Genel Şablon

Uyarı Değişkenini Kullanan Grafik #

.Graph’ı kullanmak, sonuçları yalnızca uyarı için etiket kümesiyle eşleşenleri içerecek şekilde filtreleyecektir. Örneğin, os.low.memory{host=ny-web01} için bir uyarı yalnızca host=ny-web01 etiketlerine sahip dizileri içerir. Birden fazla seri eşleşirse, yalnızca ilk eşleşen sonuç kullanılır.

template graph.template {
    subject = ...

    body = `{{template "header" .}}

    <strong>Graph</strong>
    <div>{{.Graph .Alert.Vars.graph}}</div>

    <strong>Graph With Y Axis Label Literal</strong>
    <div>{{.Graph .Alert.Vars.graph "Free Memory in GB"}}</div>

    <strong>Graph With Y Axis Label From Variable</strong>
    <div>{{.Graph .Alert.Vars.graph .Alert.Vars.graph_unit}}</div>

    `
}

alert os.low.memory {
    template = graph.template
    ...
    $graph = q("avg:300s-avg:os.mem.percent_free{host=$host}", "1d", "")
    $graph_unit = Percent Free Memory (Including Buffers and Cache)
    ...
}

Uyarı Değişkenini Kullanan GraphAll #

.GraphAll kullanımı, grafikteki tüm sonuçları içerecektir.

template graph.template {
    subject = ...

    body = `{{template "header" .}}

    <strong>GraphAll</strong>
    <div>{{.GraphAll .Alert.Vars.graph}}</div>

    <strong>GraphAll With Y Axis Label Literal</strong>
    <div>{{.GraphAll .Alert.Vars.graph "All Systems Free Memory in GB"}}</div>

    <strong>GraphAll With Y Axis Label From Variable</strong>
    <div>{{.GraphAll .Alert.Vars.graph .Alert.Vars.graph_unit}}</div>

    `
}

alert os.low.memory {
    template = graph.template
    ...
    $graph = q("avg:300s-avg:os.mem.percent_free{host=$host}", "1d", "")
    $graph_unit = All Systems Percent Free Memory (Including Buffers and Cache)
    ...
}

Satır içi veya dinamik sorgu kullanarak Graph veya GraphAll #

Bir Alert değişkeni kullanmak istemiyorsanız, grafik sorguları satır içi olarak tanımlanabilir.

template graph.template {
    subject = ...

    body = `{{template "header" .}}

    <strong>Graph With Inline Query</strong>
    <div>{{.Graph "q(\"avg:300s-avg:os.mem.percent_free{host=specifichost}\", \"1d\", \"\")" "Free Memory in GB"}}</div>

    <strong>GraphAll with Inline Query</strong>
    <div>{{.GraphAll "q(\"avg:300s-avg:os.mem.percent_free{host=host1|host2|host3}\", \"1d\", \"\")" "All Systems Free Memory in GB"}}</div>

    `
}

Bazen bir veya daha fazla değişkeni birleştirerek şablonun kendisinde dinamik olarak bir grafik için sorgu oluşturmak isteyebilirsiniz. Örneğin, bir ana bilgisayar kapalı uyarısı, dst_host etiketini kullanarak Bosun bilinen ana bilgisayarların ping metriğini dahil etmek isteyebilir.

template host.down {
    subject = ...

    body = `{{template "header" .}}

    <strong>Graph from one variable</strong>
    <div>{{printf "q(\"sum:bosun.ping.timeout{dst_host=%s}\", \"8h\", \"\")" (.Group.host) | .Graph}}</div>

    <strong>Graph from multiple variables</strong>
    <div>{{printf "q(\"sum:%s{host=%s,anothertag=%s}\", \"8h\", \"\")" "some.metric.name" .Group.host "anothervalue" | .Graph}}</div>
    `
}

Ana bilgisayar bir uyarıyı tetiklediğinde printf ifadesi q("sum:bosun.ping.timeout{dst_host=alerthostname}", "8h", "") üretecek ve ardından bunu bildirimde grafiği oluşturmak için kullanacaktır.

Filtre, Sıralama, Sınırlama ve Grafik #

GraphAll kullanırken yine de sonuçları filtrelemek isteyebilirsiniz, bu durumda Filter, Sort ve Limit işlevleriyle bir Uyarı değişkeni kullanabilirsiniz.

template graph.template {
    subject = ...

    body = `{{template "header" .}}

    <strong>Graph Filtered Variable</strong>
    <div>{{.Graph .Alert.Vars.graph_below_5 .Alert.Vars.graph_unit}}</div>

    <strong>Graph Filter+Sort+Limit Variable (Maximum of 10 series)</strong>
    <div>{{.Graph .Alert.Vars.graph_lowest_10 .Alert.Vars.graph_unit2}}</div>

    `
}

alert os.low.memory {
    template = graph.template
    ...
    $graph_all = q("avg:300s-avg:os.mem.percent_free{host=ny-*}", "1d", "")
    $graph_unit = All Systems with Less than 5 Percent Free Memory
    $graph_below_5 = filter($graph_all, min($graph_all) < 5)

    $graph_unit2 = Ten Systems with lowest Percent Free Memory
    $graph_lowest_10 = filter($graph_all, limit(sort(min($graph_min_5),"asc"),10))        
    ...
}

Serileri Birleştirmek için Birleştirmeyi Kullanma #

İki seriyi tek bir grafik üzerinde grafiğe dökmek istiyorsanız Birleştir fonksiyonunu kullanabilirsiniz. Bu, Y eksenini işlemek için Seri işleviyle de birleştirilebilir (sıfırdan başlamaya zorlamak gibi).

template graph.template {
    subject = ...

    body = `{{template "header" .}}

    <strong>Graph With Merge+Series so Y Axis Starts At Zero</strong>
    <div>{{.Graph .Alert.Vars.graph_merged .Alert.Vars.graph_unit}}</div>
    `
}

alert os.low.memory {
    template = graph.template
    ...
    $graph_time = "1d"
    $graph_host = q("avg:300s-avg:os.mem.percent_free{host=myhost}", $graph_time, "")
    $graph_unit = Notice the Y axis always starts at zero now
    $graph_series = series("value=zero", epoch()-d($graph_time), 0, epoch(),0)
    $graph_merged = merge($graph_host,$graph_series)    
    ...
}
