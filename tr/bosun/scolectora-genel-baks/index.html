Scolector'a Genel Bakış

  • Örnek scollector.toml dosyasıyla kurulum
  • Scolector’ı hizmet olarak çalıştırma

Bu sayfada

  • Örnek scollector.toml dosyasıyla kurulum
  • Scolector’ı hizmet olarak çalıştırma

Scollector, metrikleri Bosun’a veya OpenTSDB tarzı metrikleri kabul eden herhangi bir sisteme göndermek için kullanılabilen bir izleme aracıdır. OpenTSDB’nin tcollector veri toplama çerçevesinden sonra modellenmiştir ancak Go’da yazılmıştır ve tek bir ikili dosyada derlenmiştir. Tasarım hedeflerinden biri, metriklerin minimum yapılandırmayla veya hiç yapılandırma gerektirmeden gönderilmesi için hizmetleri otomatik olarak algılamaktır. Ayrıca, bir komut dosyası veya yürütülebilir dosya kullanarak metrikler oluşturan harici toplayıcılar oluşturabilir ve metrikleri sıraya almak ve sunucuya göndermek için Scollector’ı kullanabilirsiniz.

Ayrıca, metrikleri doğrudan /api/put yoluna gönderebileceğiniz, başka bir izleme aracısı kullanabileceğiniz veya Graphite, InfluxDB veya ElasticSearch gibi farklı bir arka uç kullanabileceğiniz için Bosun’u kullanırken Scollector kullanmanıza GEREK YOKTUR.

Örnek scollector.toml dosyasıyla kurulum #

Windows, Mac ve Linux için Scollektör ikili dosyaları Bosun sürüm sayfasından edinilebilir ve /opt/scollector/ veya C:\Program Dosyaları\scollector\. Scollector yapılandırma dosyası TOML v0.2.0 kullanır /en/toml-v0.2.0.md) çeşitli ayarları ve varsayılanları ikili dosyayla aynı klasörde scollector.toml olarak adlandırmak için belirtin. Yapılandırma dosyası isteğe bağlıdır ve yalnızca belirli bir toplayıcıyı etkinleştirmek için varsayılan bir değeri geçersiz kılmanız veya ayarları dahil etmeniz gerekiyorsa gereklidir.

#Where to send metrics. If omitted the default is bosun:80. 
#Config file setting can also be overridden using -h bosunhostname on command line 
Host = "mybosunserver.example.com:8080"

#Optional folder where to find external collector scripts/binaries
ColDir  = 'C:\Program Files\scollector\collectors'

#Number of data points to include in each batch. Default is 500, should be set higher if you are sending a lot of metrics.
BatchSize = 5000

Daha sonra Scollector’u bir hizmet olarak yükleyebilir veya şu yollarla manuel olarak çalıştırabilirsiniz:

#Override default configuration file location
scollector -conf /path/to/myconfig.toml

#List all built-in collectors
scollector -l

#-p will print metrics to the screen instead of sending to Bosun. 
#-f "..." will only run specific collectors. Add DisableSelf = true to toml file to exclude scollector.* self metrics
scollector -p -f "c_cpu_windows,c_network_"

Scolector’ı hizmet olarak çalıştırma #

Windows’ta Scollector’ı -winsvc=“install” bayrağını kullanarak bir hizmet olarak yükleyebilirsiniz. Mac ve Linux’ta manuel olarak bir hizmet veya init betiği oluşturmalısınız. Örneğin, burada temel bir systemd birim dosyası var:

#Scollector unit file saved to /etc/systemd/system/scollector.service
[Unit]
Description=Scollector Service
After=network.target

[Service]
Type=simple
User=root
ExecStart=/opt/scollector/scollector -h mybosunserver.example.com
Restart=on-abort

[Install]
WantedBy=multi-user.target
