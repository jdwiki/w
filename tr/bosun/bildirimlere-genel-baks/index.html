Sözdizimi #

  • bildirim isim {
  • e-posta = dev-alerts@example.com, prod-alerts@example.com, …
  • gönderi = http://example.com
  • get = http://example.com
  • sonraki = başka bir bildirim tanımı
  • zaman aşımı = 30m
  • runOnActions = yanlış
  • gövde = {“metin”: {{.|json}}}
  • contentType = uygulama/json
  • yazdır = doğru -}

Bosun’da bildirimler hem yeni uyarı olayları için hem de bir uyarı onaylandığında/kapatıldığında/vb. için kullanılır. Diğer olayların bir bildirim tetiklemesini istemiyorsanız, bildirim tanımına “runOnActions = false” ifadesini ekleyin.

Ayrıca bakınız:

  • Bildirimler: Sohbet Sistemleri

HTTP GET/POST Bildirimleri #

Uyarı olayları, HTTP GET veya HTTP POST istekleri kullanılarak başka bir sisteme gönderilebilir. Oluşturulan uyarıyı doğrudan gönderebilir (belki de şablondaki işaretlemeyi kullanarak) veya uyarı verilerini bir JSON nesnesinin parçası olarak göndermek için body = ... {{.|json}} ... ve contentType’ı kullanabilirsiniz. Diğer bir yaklaşım, yalnızca temel olay bilgilerini göndermek ve ardından alıcı sistemin bosun API’sinden ek ayrıntılar çekmesini sağlamaktır.

notification postjson {
    post = ${env.POSTURL}
    body = {"text": {{.|json}}, apiKey=${env.APIKEY}}
    contentType = application/json
}

HTTP GET/POST istekleri için contentType, varsayılan olarak application/x-www-form-urlencoded şeklindedir.

Plivo ile SMS Bildirimleri #

plivo hesabınızdan ihtiyacınız olacak iki kimlik var. Bu snippet’teki “authid” ve “authtoken” değerlerini bu değerlerle değiştirin. src değeri de hesabınıza atanmış geçerli bir numara olmalıdır. ‘dst’, istediğiniz herhangi bir sayı olabilir veya ‘<’ ile ayrılmış birden fazla olabilir.

notification sms {
  post = https://authid:authtoken@api.plivo.com/v1/Account/authid/Message/
  body = {"text": {{.|json}}, "dst":"15551234567","src":"15559876543"}
  contentType = application/json
  runOnActions = false
}

Bu, uyarı konusunu ‘dst’ içindeki tüm numaralara mesaj olarak gönderir.

E-posta Bildirimleri #

E-posta bildirimleri göndermek için yapılandırma dosyanıza aşağıdaki ayarları eklemeniz gerekir:

#Using a company SMTP server (note only one can be define)
smtpHost = mail.example.com:25
emailFrom = bosun@example.com

#Using Gmail with TLS and username/password
smtpHost = smtp.gmail.com:587
emailFrom = myemail@gmail.com
smtpUsername = myemail@gmail.com
smtpPassword = ${env.EMAILPASSWORD}

#Chained notifications will escalate if an incident is not acked before the timeout
notification it {
    email = it-alerts@example.com
    next = oncall
    timeout = 30m
}

#Could set additional escalations here using any notification type (email/get/post)
#or set next = oncall to send another email after the timeout if alert is still not acked
notification oncall {
    email = escalated-alerts@example.com
}

#Multiple email addresses can be specified in one notification definition
#Use runOnActions = false to exclude updates on actions (ack/closed/forget)
notification engineering {
    email = core-alerts@example.com,qa-alerts@example.com,prod-alerts@example.com
    runOnActions = false
}

Genel Bakış #

Bosun bildirimleri, warnNotification ve critNotification kullanılarak uyarı tanımlarına atanır ve yeni bir olay meydana geldiğinde oluşturulan uyarı şablonunun nereye gönderileceğini belirtir. Bildirimler e-posta yoluyla gönderilebilir veya HTTP GET/POST istekleri kullanılabilir. Ayrıca sadece Bosun günlük dosyasına bilgi ekleyen bir Yazdır bildirimi vardır.

Bir URL, Parola veya API Anahtarının düz metin olmasını engellemek istiyorsanız, değeri bir Çevresel Değişkenden (genellikle Bosun başlatma komut dosyasından dışa aktarılır) yüklemek için “${env.VARIABLENAME}” öğesini kullanabilirsiniz. Lütfen, değişkenlere kimlerin erişebileceğine ilişkin hiçbir koruma olmadığını (bir şablonda kolayca görüntülenebilirler) ancak bunların doğrudan Kural Düzenleyici sayfasında veya .conf dosyasında görüntülenmelerini engellediğini unutmayın.

notification logfile {
    print = true
}

#print can be added to any notification type to help with debugging
notification email {
    email = sysadmins@example.com
    print = true
}

Twilio ile SMS Bildirimleri #

Kimlik bilgileriniz/amaçlanan alıcılar için ‘AccountSid’, ‘AuthToken’, ‘ToPhoneNumber’ ve ‘FromPhoneNumber’ öğelerini değiştirin. ToPhoneNumber ve FromPhoneNumber içlerinde + varsa, bunların urlencoding (yani: %2B olarak) olduğundan emin olmanız gerekir.

notification sms {
    post = https://{AccountSid}:{Authtoken}@api.twilio.com/2010-04-01/Accounts/{AccountSid}/Messages.json
    body = Body={{.}}&To={ToPhoneNumber}&From={FromPhoneNumber}
}

Özden: https://gist.github.com/aodj/58535c4c152b6073eaf5

Çağrı Görevi Bildirimleri #

#Post to pagerduty.com
notification pagerduty {
    post = https://events.pagerduty.com/generic/2010-04-15/create_event.json
    contentType = application/json
    runOnActions = false
    body = `{
     "service_key": "myservicekey",
     "incident_key": {{.|json}},
     "event_type": "trigger",
     "description": {{.|json}},
     "client": "Bosun",
     "client_url": "http://bosun.example.com/"
    }`
}

Aramayı Kullanarak Bildirimi Değiştirme #

Bazı durumlarda, Uyarı anahtarlarındaki bir etikete göre hangi bildirimi kullanacağınızı değiştirmek isteyebilirsiniz. Bunu Arama özelliğini kullanarak yapabilirsiniz. Not: Arama, yalnızca OpenTSDB kullanıyorsanız ve dizine eklenecek Bosun’a veri gönderiyorsanız çalışır. Diğer arka uçlar veya dizine eklenmemiş veriler için bunun yerine lookupSeries kullanmanız gerekir.

notification default {
    email = team@example.com
}

notification JSmith{
    email = JSmith@example.com
}

#This will use the JSmith lookup for any alerts where the host tag starts with ny-jsmith
lookup host_base_contact {
    entry host=ny-jsmith* {
        main_contact = JSmith
    }
    entry host=* {
        main_contact = default
    }
}

alert blah {
    ...
    warn = q(...)
    warnNotification = lookup("host_base_contact", "main_contact")
    critNotification = lookup("host_base_contact", "main_contact")
}

Bu, Makrolar kullanılarak birden çok uyarıya da uygulanabilir:

macro host.based.contacts {
    warnNotification = lookup("host_base_contact", "main_contact")
    critNotification = lookup("host_base_contact", "main_contact")
}
