Bukkit’in içinde bir şey olduğunda, bir Olay çağrılır, böylece her eklenti bir şey olduğunda ne yapacağına karar verebilir.

Bir oyuncu bir blok oynamaya çalıştığında, bir varlık umutsuzluğa düştüğünde, biri oturum açtığında bir Etkinlik çağrılır… Eklentiler belirli olayları dinleyebilir ve bununla birçok farklı şekilde ilgilenebilir, örneğin, Oyuncu, PlayerLoginEvent aracılığıyla oturum açar.

Sözdizimi #

  • Bukkit.getPluginManager().registerEvents(Dinleyici l, Eklenti p);

Bir etkinliği kaydederken, iki kez kaydetmediyseniz bir göz atın! Veya eklentiniz kayıtlı etkinlik için iki kez hareket edecektir.

Belirli olayların nasıl ele alınacağı konusunda fazladan bir göz atın:

  • Oyuncu Etkinlikleri
  • Varlık Olayları

Özel Etkinlikler Oluşturma #

Bazen diğer eklentilerin dinleyebileceği (diğer eklentilerin yanı sıra Apps Kasası bunu yapar) ve hatta iptal edebileceği kendi Etkinliğinizi oluşturmanız gerekir. Bukkit’in Event API’si bunun mümkün olmasını sağlar. Tek yapmanız gereken yeni bir sınıf oluşturmak, ‘Etkinliği’ genişletmesini sağlamak, işleyicileri ve etkinliğinizin ihtiyaç duyduğu nitelikleri (Oyuncu veya mesaj gibi) eklemek.

import org.bukkit.event.Event;
import org.bukkit.event.HandlerList;

public final class CustomEvent extends Event {
    private static final HandlerList handlers = new HandlerList();
    private String message;

    public CustomEvent(String example) {
        message = example;
    }

    public String getMessage() {
        return message;
    }

    public HandlerList getHandlers() {
        return handlers;
    }

    public static HandlerList getHandlerList() {
        return handlers;
    }
}

Özel Etkinliğinizi Çağırma #

Etkinliklerinizi oluşturmak ve çağırmak sizin elinizde, nerede çağıracağınız tamamen size kalmış. İşte bir örnek

// Create the event here
CustomEvent event = new CustomEvent("Sample Message");
// Call the event
Bukkit.getServer().getPluginManager().callEvent(event);
Bukkit.getServer().broadcastMessage(event.getMessage());

Unutmayın: Olaylarınızın kontrolü sizde. Onu çağırmaz ve ona göre hareket etmezsen, olmaz!

Özel Bir Etkinliği Dinleme #

Özel bir olayı dinlemek, normal bir olayı dinlemekle aynıdır.

import org.bukkit.event.Listener;
import org.bukkit.event.EventHandler;

public final class CustomListener implements Listener {
    
    @EventHandler
    public void onCustomEvent(CustomEvent event) {
    // Some code here
    }
}

CustomEvent’inizi İptal Edilebilir Hale Getirme #

Etkinliğinizi iptal edilebilir hale getirmek isterseniz, sadece ‘implements Cancellable’, ‘boolean iptal edildi’ ve bir alıcı ve ayarlayıcı ekleyin:

import org.bukkit.event.Event;
import org.bukkit.event.HandlerList;
import org.bukkit.event.Cancellable;

public final class CustomEvent extends Event implements Cancellable {
    private static final HandlerList handlers = new HandlerList();
    private String message;
    private boolean cancelled;

    public CustomEvent(String example) {
        message = example;
    }

    public String getMessage() {
        return message;
    }

    public boolean isCancelled() {
        return cancelled;
    }

    public void setCancelled(boolean cancel) {
        cancelled = cancel;
    }

    public HandlerList getHandlers() {
        return handlers;
    }

    public static HandlerList getHandlerList() {
        return handlers;
    }
}

Ardından, bir eklentinin özel etkinliği iptal edip etmediğini kontrol edersiniz.

// Create the event here
CustomEvent event = new CustomEvent("Sample Message");
// Call the event
Bukkit.getServer().getPluginManager().callEvent(event);
// Check if the event is not cancelled
if (!event.isCancelled()) {
    Bukkit.getServer().broadcastMessage(event.getMessage());
}

Temel Olay İşleme #

Bukkit, eklenti geliştiricilerinin sunucuyla ve dünyada meydana gelen belirli eylemlerle etkileşime girmesine ve bunları değiştirmesine izin veren olay tabanlı bir sistem kullanır.


Olay İşleyici Oluşturma #

Olay işleyicileri, olayları gerçekleştiğinde çağrılan yöntemlerdir. Genellikle herkese açık ve geçersizdirler ve ayrıca geleneksel olarak “on{EventNameStem}” olarak adlandırılırlar. Ancak tüm işleyiciler, @EventHandler notuna sahip olmalı ve olayını SADECE parametre olarak içermelidir. İşte ‘PlayerJoinEvent’ için bir olay işleyici örneği

@EventHandler
public void onPlayerJoin(PlayerJoinEvent event){
    //Run when a player joins
}

Not: Bukkit etkinliklerinin adlandırma biçimi {Source}{Action}({Target})Event şeklindedir. Bu olay adlarına bazı örnekler: “PlayerInteractEvent” veya “BlockBreakEvent” veya “PlayerJoinEvent”. Tüm etkinliklerin bir listesi Spigot Javadocs adresinde bulunabilir.


Olayları Kaydetme #

Bukkit’in yönteminize olay çağrıları göndermeye başlamasına izin vermek için yalnızca bir olay işleyicisi oluşturmak yeterli değildir. Ayrıca PluginManager arabirimi aracılığıyla da kaydetmeniz gerekir.

Olayları kaydetmenin en yaygın yolu, Listener arabirimini uygulayan bir sınıf oluşturmak ve bunu olay işleyicilerinizi sarmaktır.

public class EventListener implements Listener { //Implements the Listener interface

    @EventHandler
    public void onPlayerJoin(PlayerJoinEvent event){
        //Run when a player joins
    }

}

Bu dinleyici sınıfı ve tüm olayları daha sonra ana eklenti sınıfınıza şu şekilde kaydedilebilir:

@Override
public void onEnable(){
    Bukkit.getPluginManager().registerEvents(new EventListener(), this); //Register your listener and its event handlers
}

Listener sınıfındaki Olayları Kaydedin #

import org.bukkit.event.Listener;
import org.bukkit.event.EventHandler;
import org.bukkit.event.EventPriority;
import org.bukkit.event.player.PlayerLoginEvent;
import org.bukkit.event.player.PlayerQuitEvent;

public class MyEventListener implements Listener {

    /**
     * Constructor
     */
    public MyEventListener(Main plugin){
        //register Events of this class
        //with methode: registerEvents(Listener, Plugin);
        plugin.getServer().getPluginManager().registerEvents(this, plugin);
    }      

    /**
     * A Event with HIGH priority
     */
    @EventHandler(priority = EventPriority.HIGH) //An EventHandler annotation
    public void onPlayerLogin(PlayerLoginEvent event){  //A bukkit event
        event.getPlayer().sendMessage("Welcome.");
    }
    /**
     * A Event with NORMAL (default) priority
     */
    @EventHandler    
    public void onPlayerQuit(PlayerQuitEvent event){ 
        Bukkit.broadcastMessage(event.getPlayer().getName() + " left the Server.");
    }
        

}

/**
 * Main class
 */
public class Main extends JavaPlugin {
    public void onEnable(){
        //Register Events
        new MyEventListener(this);
    }
}

Etkinlik Öncelikleri #

Bukkit, eklentilerin olayları doğru eski sürümde işlemesine yardımcı olmak için Etkinlik Öncelikleri adlı bir sisteme sahiptir. Yedi öncelik şunlardır (ilk yürütülenden sonuncuya kadar daha eski):

  • En düşük
  • Düşük
  • Normal (varsayılan)
  • Yüksek
  • En yüksek
  • Monitör

Çok sayıda etkinliği (ör. koruma eklentisi) iptal etmeyi planlıyorsanız, sorunları önlemek için daha düşük öncelik (veya en düşük) kullanmak iyi bir fikir olacaktır.

MONITOR’da bir olayın sonucunu asla değiştirmemelisiniz.

@EventHandler //same as @EventHandler(priority = EventPriority.NORMAL)
public void onLogin(PlayerLoginEvent event) {
    // normal login
}

@EventHandler(priority = EventPriority.HIGH)
public void onLogin(PlayerLoginEvent event) {
    // high login
}
