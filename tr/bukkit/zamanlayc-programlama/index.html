Sözdizimi #

  • Bukkit.getScheduler().scheduleSyncRepeatingTask(Eklenti eklentisi, Çalıştırılabilir görev, int initialDelay, int tekrarlamaGecikmesi)
  • Bukkit.getScheduler().scheduleSyncDelayedTask(Eklenti eklentisi, Çalıştırılabilir görev, int initialDelay)
  • Bukkit.getScheduler().runTaskAsynchronously(Eklenti eklentisi, Çalıştırılabilir görev)
  • Bukkit.getScheduler().runTask(Eklenti eklentisi, Çalıştırılabilir görev)
  • new BukkitRunnable() { @Override public void run() { /* CODE */ } }.runTaskLater(Eklenti eklentisi, uzun gecikme);
  • new BukkitRunnable() { @Override public void run() { /* CODE */ } }.runTaskTimer(Eklenti eklentisi, uzun başlangıçDelay, uzun tekrarlamaGecikmesi);

Birkaç Bukkit API yöntemi iş parçacığı için güvenlidir ve zaman uyumsuz olarak çağrılabilir. Bu nedenle, Bukkit API yöntemleri, birkaç istisna dışında yalnızca ana iş parçacığında çalıştırılmalıdır.

‘scheduleSync’ yöntemlerinin içinde çalıştırılan kod, ana iş parçacığında ‘runTask’ yönteminin yanı sıra çalıştırılacaktır.

runTaskAsynchronously içinde çalıştırılan kod, ana iş parçacığından asenkron olarak çalıştırılacaktır. Eşzamansız yöntemler, sunucuyu geciktirmeden büyük matematik veya veritabanı işlemleri yapmak için çok kullanışlıdır, ancak Bukkit API yöntemlerini çağırmak için kullanılırsa tanımsız davranışa neden olur. Bu nedenle asenkron koddan sonra çalıştırılması gereken Bukkit API metodları her zaman bir runTask metoduna konulmalıdır.

Zamanlayıcı Yinelenen görev #

Zamanlayıcı Görevlerinin süresi Keneler cinsinden ölçülür. Normal şartlar altında saniyede 20 tik vardır.

.scheduleSyncRepeatingTask ile zamanlanan görevler Ana Konuda çalıştırılacaktır.

Bukkit.getScheduler().scheduleSyncRepeatingTask(eklenti, yeni Runnable() {
    @Override
    public void run() {
        Bukkit.broadcastMessage("This message is shown immediately and then repeated every second");
    }
}, 0L, 20L); //0 İlk gecikmeyi işaretleyin, tekrarlar arasında 20 Tick (1 Saniye)

Zamanlayıcı Gecikmeli Görev #

Zamanlayıcı Görevlerinin süresi Keneler cinsinden ölçülür. Normal şartlar altında saniyede 20 tik vardır.

.scheduleSyncDelayedTask ile zamanlanan görevler Ana Konuda çalıştırılacaktır.

Bukkit.getScheduler().scheduleSyncDelayedTask(eklenti, yeni Runnable() {
    @Override
    public void run() {
        Bukkit.broadcastMessage("This message is shown after one second");
    }
}, 20L); //20 Run() çağrılmadan önce (1 Saniye) gecikmeyi işaretleyin

Görevleri Eşzamansız Olarak Çalıştırma #

runTaskAsynchronously kullanarak kodun ana iş parçacığından asenkron olarak çalışmasını sağlayabilirsiniz. Bu, ana iş parçacığının donmasını (ve sunucunun gecikmesini) önleyeceğinden, yoğun matematik veya veritabanı işlemleri yapmak için kullanışlıdır.

Birkaç Bukkit API yöntemi iş parçacığı için güvenlidir, bu nedenle çoğu ana iş parçacığından eşzamansız olarak çağrılırsa tanımsız davranışa neden olur.

Bukkit.getScheduler().runTaskAsynchronously(eklenti, yeni Runnable() {
    @Override
    public void run() {
        Bukkit.getLogger().info("This message was printed to the console asynchronously");
        //Bukkit.broadcastMessage is not thread-safe
    }
});

Ana Konuda Görevleri Çalıştırma #

Ayrıca runTask kullanarak kodun ana iş parçacığı ile senkronize çalışmasını sağlayabilirsiniz. Bu, kodu ana iş parçacığından eşzamansız olarak çalıştırdıktan sonra Bukkit API yöntemlerini çağırmak istediğinizde kullanışlıdır.

Bu Runnable’ın içinde çağrılan kod, ana iş parçacığında yürütülecek ve Bukkit API yöntemlerini çağırmayı güvenli hale getirecektir.

Bukkit.getScheduler().runTask(eklenti, yeni Runnable() {
    @Override
    public void run() {
        Bukkit.broadcastMessage("This message is displayed to the server on the main thread");
        //Bukkit.broadcastMessage is thread-safe
    }
});

Bir BukkitRunnable Çalıştırma #

BukkitRunnable, Bukkit’te bulunan bir Runnable’dır. Bir görevi doğrudan bir BukkitRunnable’dan zamanlamak ve ayrıca kendi içinden iptal etmek mümkündür.

Önemli: Görevlerin süresi Keneler cinsinden ölçülür. Saniyede 20 tik vardır.

Tekrarlanmayan Görev:

JavaPlugin plugin;    //Your plugin instance    
Long timeInSeconds = 10;
Long timeInTicks = 20 * timeInSeconds;
new BukkitRunnable() {
        
    @Override
    public void run() {
        //The code inside will be executed in {timeInTicks} ticks.
        
    }
}.runTaskLater(plugin, timeInTicks);   // Your plugin instance, the time to be delayed.

Yinelenen Görev:

JavaPlugin plugin;    //Your plugin instance    
Long timeInSeconds = 10;
Long timeInTicks = 20 * timeInSeconds;
new BukkitRunnable() {
        
    @Override
    public void run() {
        //The code inside will be executed in {timeInTicks} ticks.
       //After that, it'll be re-executed every {timeInTicks} ticks;
      //Task can also cancel itself from running, if you want to.

       if (boolean) {
           this.cancel();
       }
        
    }
}.runTaskTimer(plugin, timeInTicks, timeInTicks);   //Your plugin instance, 
                                                   //the time to wait until first execution,
                                                  //the time inbetween executions.

İş Parçacığı Güvenli Kodunu Eşzamansız Bir Görevden Çalıştırma #

Bazen senkronize olmayan bir görevin içinden senkronize kod çalıştırmanız gerekebilir. Bunu yapmak için, asenkron blok içinden senkronize bir görev zamanlamanız yeterlidir.

Bukkit.getScheduler().runTaskTimerAsynchronously(VoidFlame.getPlugin(), () -> {

    Bukkit.getScheduler().runTask(VoidFlame.getPlugin(), () -> {
        World world = Bukkit.getWorld("world");
        world.spawnEntity(new Location(world, 0, 100, 0), EntityType.PRIMED_TNT);
    });

}, 0L, 20L);
