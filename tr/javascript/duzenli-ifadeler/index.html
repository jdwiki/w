Sözdizimi #

  • normal ifadeye izin ver = /pattern/[flags]
  • let normal ifade = new RegExp(’pattern’, [flags])
  • eşleşmeye izin ver = regex.test(’metin')
  • sonuçlara izin ver = regex.exec(’metin')

Parametreler #

Bayraklar Ayrıntılar
g global. Tüm eşleşmeler (ilk maçta geri dönme).
m mçok satırlı. ^ ve $ ’nin her satırın başlangıcı/sonu ile eşleşmesine neden olur (yalnızca dizenin başlangıcı/sonu değil).
ben **duyarsız. Büyük/küçük harfe duyarlı olmayan eşleşme ([a-zA-Z] büyük/küçük harf durumunu yok sayar).
sen unicode : Kalıp dizeleri UTF-16 olarak değerlendirilir. Ayrıca kaçış dizilerinin Unicode karakterleriyle eşleşmesine neden olur.
y sticky: yalnızca hedef dizedeki bu normal ifadenin lastIndex özelliği tarafından belirtilen dizinden eşleşir (ve sonraki dizinlerden eşleşmeye çalışmaz).

RegExp nesnesi, yalnızca Normal İfadeler hakkındaki bilginiz ne kadar güçlüyse o kadar yararlıdır. Başlangıç ​​bilgisi için buraya bakın veya bkz. [MDN](https://developer.mozilla.org/en-US/docs/ Web/JavaScript/Reference/Global_Objects/RegExp) daha ayrıntılı bir açıklama için.

Bir RegExp Nesnesi Oluşturma #

Standart Oluşturma #

Bu formu yalnızca dinamik değişkenlerden normal ifade oluştururken kullanmanız önerilir.

İfadenin değişebileceği veya ifadenin kullanıcı tarafından oluşturulduğu durumlarda kullanın.

var re = new RegExp(".*");

bayraklar ile:

var re = new RegExp(".*", "gmi");

Ters eğik çizgi ile: (normal ifade bir dize ile belirtildiğinden bu, kaçılmalıdır)

var re = new RegExp("\\w*");

Statik başlatma #

Normal ifadenin değişmeyeceğini bildiğinizde ve çalışma zamanından önce ifadenin ne olduğunu bildiğinizde kullanın.

var re = /.*/;

bayraklar ile:

var re = /.*/gmi;

Ters eğik çizgi ile: (normal ifade bir değişmez olarak belirtildiğinden bu kaçmamalıdır)

var re = /\w*/;

Normal İfade İşaretleri #

RegEx davranışını değiştirmek için belirtebileceğiniz birkaç bayrak vardır. Bayraklar, ‘/test/gi’ içinde ‘gi’ belirtmek gibi bir regex değişmezinin sonuna eklenebilir veya ’new RegExp(’test’te olduğu gibi ‘RegExp’ yapıcısının ikinci argümanı olarak belirtilebilirler. ‘, ‘gi’)`.

“g” - Küresel. İlkinden sonra durmak yerine tüm eşleşmeleri bulur.

i - Büyük/küçük harf yoksay. “/[a-z]/i”, “/[a-zA-Z]/” ile eşdeğerdir.

m - Çok satırlı. ‘^’ ve ‘$’, tüm dizenin yalnızca başlangıcı ve sonu yerine, sırasıyla ‘\n’ ve ‘\r’yi sınırlayıcı olarak ele alan her satırın başlangıcı ve sonuyla eşleşir.

“u” - Unicode. Bu bayrak desteklenmiyorsa, belirli Unicode karakterlerini “\uXXXX” ile eşleştirmeniz gerekir; burada “XXXX”, karakterin onaltılık değeridir.

y - Tüm ardışık/bitişik eşleşmeleri bulur.

son sürüm

.test() kullanarak dizenin kalıp içerip içermediğini kontrol edin #

var re = /[a-z]+/;
if (re.test("foo")) {
    console.log("Match exists.");
}

“test” yöntemi, normal ifadenin bir dizeyle eşleşip eşleşmediğini görmek için bir arama gerçekleştirir. ‘[a-z]+’ normal ifadesi bir veya daha fazla küçük harf arayacaktır. Kalıp dizeyle eşleştiğinden, “eşleşme var” konsolda günlüğe kaydedilir.

.exec() ile Eşleştirme #

.exec() Kullanarak Eşleştirme #

RegExp.prototype.exec(string) bir dizi yakalama veya eşleşme yoksa null döndürür.

var re = /([0-9]+)[a-z]+/;
var match = re.exec("foo123bar");

“match.index”, eşleşmenin (sıfır tabanlı) konumu olan 3’tür.

“match[0]” tam eşleşme dizesidir.

‘match[1]’, yakalanan ilk gruba karşılık gelen metindir. “match[n]”, yakalanan n. grubun değeri olacaktır.

.exec() Kullanarak Eşleşmeler Arasında Döngü #

var re = /a/g;
var result;
while ((result = re.exec('barbatbaz')) !== null) {
    console.log("found '" + result[0] + "', next exec starts at index '" + re.lastIndex + "'");
}

Beklenen çıktı

‘a’ bulundu, sonraki yürütme ‘2’ dizininde başlıyor ‘a’ bulundu, sonraki yürütme ‘5’ dizininde başlıyor ‘a’ bulundu, sonraki yürütme ‘8’ dizininde başlıyor

RegExp’i Dizelerle Kullanma #

String nesnesi, düzenli ifadeleri bağımsız değişken olarak kabul eden aşağıdaki yöntemlere sahiptir.

  • "string".match(...
  • "string".replace(...
  • "dize".split(...
  • "dize".arama(...

RegExp ile eşleştir #

console.log("string".match(/[i-n]+/));
console.log("string".match(/(r)[i-n]+/));

Beklenen çıktı

Dizi [“in”] Dizi [“rin”, “r”]

RegExp ile değiştir #

console.log("string".replace(/[i-n]+/, "foo"));

Beklenen çıktı

ip göz

RegExp ile Böl #

console.log("stringstring".split(/[i-n]+/));

Beklenen çıktı

Dizi [“str”, “gstr”, “g”]

RegExp ile arama yapın #

.search(), bir eşleşmenin bulunduğu dizini veya -1’i döndürür.

console.log("string".search(/[i-n]+/));
console.log("string".search(/[o-q]+/));

Beklenen çıktı

3 -1

Normal İfade Grupları #

JavaScript, Normal İfadeleri, yakalama grupları, yakalamayan gruplar ve ileriye bakmak’ta çeşitli grup türlerini destekler. Şu anda, _arkasına bakma desteği yoktur.

Ele geçirmek #

Bazen istenen eşleşme, bağlamına bağlıdır. Bu, basit bir RegExp’nin ilgilenilen String parçasını aşırı bulacağı anlamına gelir, bu nedenle çözüm bir yakalama grubu ‘(pattern)’ yazmaktır. Yakalanan verilere daha sonra şu şekilde atıfta bulunulabilir:

  • Dize değiştirme "$n" burada n, n th yakalama grubudur (“1"den başlayarak)
  • Geri arama işlevindeki n th bağımsız değişkeni
  • RegExp ‘g’ ile işaretlenmemişse, döndürülen bir ‘str.match’ Array içindeki n+1 th öğesi
  • RegExp ‘g’ olarak işaretlenmişse, ‘str.match’ yakalamaları atar, bunun yerine ’re.exec’ kullanın

Diyelim ki tüm “+” işaretlerinin bir boşlukla değiştirilmesi gereken bir Dize var, ancak yalnızca bir harf karakteri izliyorlarsa. Bu, basit bir eşleşmenin o harf karakterini içereceği ve aynı zamanda kaldırılacağı anlamına gelir. Bunu yakalamak çözümdür, çünkü eşleşen harf korunabilir.

let str = "aa+b+cc+1+2",
    re = /([a-z])\+/g;

// String replacement
str.replace(re, '$1 '); // "aa b cc 1+2"
// Function replacement
str.replace(re, (m, $1) => $1 + ' '); // "aa b cc 1+2"

Yakalamama #

(?:pattern) formunu kullanarak, bunlar, eşleşmeden sonra grubun içeriğini saklamamaları dışında, grupları yakalamaya benzer şekilde çalışırlar.

Endekslerini taşımak istemediğiniz, ancak OR gibi bazı gelişmiş desen eşleştirmeleri yapmanız gereken başka veriler yakalanıyorsa özellikle yararlı olabilirler.

let str = "aa+b+cc+1+2",
    re = /(?:\b|c)([a-z])\+/g;

str.replace(re, '$1 '); // "aa+b c 1+2"

İleriye Bakmak #

İstenen eşleşme, onu eşleştirmek ve onu yakalamak yerine onu takip eden bir şeye dayanıyorsa, onu test etmek için ileriye dönük bir bakış kullanmak ancak maça dahil etmemek mümkündür. Olumlu bir ileriye bakış ‘(?=pattern)’ biçimine sahiptir, olumsuz bir ileriye bakış (burada ifade eşleşmesi yalnızca ileriye dönük model eşleşmediğinde gerçekleşir) ‘(?!pattern)’ biçimine sahiptir.

let str = "aa+b+cc+1+2",
    re = /\+(?=[a-z])/g;

str.replace(re, ' '); // "aa b cc+1+2"

Dize eşleşmesini bir geri arama işleviyle değiştirme #

String#replace ikinci argümanı olarak bir fonksiyona sahip olabilir, böylece bir mantığa dayalı olarak bir değiştirme sağlayabilirsiniz.

"Some string Some".replace(/Some/g, (match, startIndex, wholeString) => {
  if(startIndex == 0){
    return 'Start';
  } else {
    return 'End';
  }
});
// will return Start string End

Tek satırlık şablon kitaplığı

let data = {name: 'John', surname: 'Doe'}
"My name is {surname}, {name} {surname}".replace(/(?:{(.+?)})/g, x => data[x.slice(1,-1)]);

// "My name is Doe, John Doe"

Bir dizenin eşleşmelerini çıkarmak için Regex.exec()‘i parantez içinde normal ifade kullanma #

Bazen dizeyi basitçe değiştirmek veya kaldırmak istemezsiniz. Bazen eşleşmeleri çıkarmak ve işlemek istersiniz. Burada eşleşmeleri nasıl manipüle ettiğinize dair bir örnek.

Maç nedir? Dizedeki tüm normal ifade için uyumlu bir alt dize bulunduğunda, exec komutu bir eşleşme üretir. Bir eşleşme, öncelikle eşleşen tüm alt dize ve eşleşmedeki tüm parantezlerden oluşan bir dizidir.

Bir html dizesi düşünün:

<html>
<head></head>
<body>
  <h1>Example</h1>
  <p>Look a this great link : <a href="https://stackoverflow.com">Stackoverflow</a> http://anotherlinkoutsideatag</p>
  Copyright <a href="https://stackoverflow.com">Stackoverflow</a>
</body>

Bir “a” etiketi içindeki tüm bağlantıları çıkarmak ve almak istiyorsunuz. İlk başta, burada yazdığınız regex:

var re = /<a[^>]*href="https?:\/\/.*"[^>]*>[^<]*<\/a>/g;

Ama şimdi, her bağlantının ‘href’ini ve ‘çıpasını’ istediğinizi hayal edin. Ve birlikte istiyorsun. Her eşleşme için yeni bir normal ifade ekleyebilirsiniz VEYA parantez kullanabilirsiniz:

var re = /<a[^>]*href="(https?:\/\/.*)"[^>]*>([^<]*)<\/a>/g; 
var str = '<html>\n    <head></head>\n    <body>\n        <h1>Example</h1>\n        <p>Look a this great link : <a href="https://stackoverflow.com">Stackoverflow</a> http://anotherlinkoutsideatag</p>\n\n        Copyright <a href="https://stackoverflow.com">Stackoverflow</a>\n    </body>\';\n';
var m;
var links = [];

while ((m = re.exec(str)) !== null) {
    if (m.index === re.lastIndex) {
        re.lastIndex++;
    }
    console.log(m[0]); // The all substring
    console.log(m[1]); // The href subpart
    console.log(m[2]); // The anchor subpart

    links.push({
      match : m[0],   // the entire match
      href : m[1],    // the first parenthesis => (https?:\/\/.*)
      anchor : m[2],  // the second one => ([^<]*)
    });
}

Döngünün sonunda, ‘anchor’ ve ‘href’ içeren bir bağlantı diziniz var ve bunu, örneğin markdown yazmak için kullanabilirsiniz:

links.forEach(function(link) {
  console.log('[%s](%s)', link.anchor, link.href);
});
