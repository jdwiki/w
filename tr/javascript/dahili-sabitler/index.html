hükümsüz #

’null’, bir nesne değerinin kasıtlı olarak yokluğunu temsil etmek için kullanılır ve ilkel bir değerdir. “Undefined"den farklı olarak, global nesnenin bir özelliği değildir.

‘undefined’ değerine eşittir, ancak onunla aynı değildir.

null == undefined; // true
null === undefined; // false

DİKKAT: “typeof” “null”, “nesne"dir.

typeof null; // 'object';

Bir değerin “null” olup olmadığını doğru şekilde kontrol etmek için, katı eşitlik operatörü ile karşılaştırın.

var a = null;

a === null; // true

isNaN() kullanarak NaN testi #

NaN #

NaN “Sayı Değil” anlamına gelir. JavaScript’teki bir matematiksel işlev veya işlem belirli bir sayı döndüremediğinde, bunun yerine “NaN” değerini döndürür.

Bu, global nesnenin bir özelliğidir ve Number.NaN öğesine bir başvurudur.

window.hasOwnProperty('NaN'); // true
NaN; // NaN

Belki de kafa karıştırıcı bir şekilde, ‘NaN’ hala bir sayı olarak kabul ediliyor.

typeof NaN; // 'number'

Eşitlik operatörünü kullanarak NaN olup olmadığını kontrol etmeyin. Bunun yerine [isNaN](https://www.wikiod.com/tr/javascript/dahili-sabitler#isNaN() kullanarak NaN testi) konusuna bakın.

NaN == NaN  // false
NaN === NaN // false

tanımsız ve boş #

İlk bakışta “boş” ve “tanımsız” temelde aynı görünebilir, ancak ince fakat önemli farklılıklar vardır.

‘undefined’ derleyicide bir değerin olmamasıdır, çünkü değer olması gereken yerde, atanmamış bir değişken durumunda olduğu gibi bir değer konulmamıştır.

  • “tanımsız”, atanmış bir değerin yokluğunu temsil eden genel bir değerdir.
  • `typeof undefined === ’tanımsız’’
  • “null”, bir değişkene açıkça “değer yok” atandığını gösteren bir nesnedir.
  • `typeof null === ’nesne’'

Bir değişkeni “tanımsız” olarak ayarlamak, değişkenin fiilen var olmadığı anlamına gelir. JSON serileştirme gibi bazı işlemler, nesnelerden “tanımsız” özellikleri çıkarabilir. Buna karşılık, “boş” bir özellik kavramını açıkça iletebilmeniz için “null” özellikleri korunacaktır.

Aşağıdakiler “tanımsız” olarak değerlendirilir:

  • Bir değişken bildirildiğinde ancak bir değer atanmadığında (yani tanımlı)
  •     let foo;
    console.log('is undefined?', foo === undefined);
    // is undefined? true
    ```
  • Var olmayan bir mülkün değerine erişme
  •     let foo = { a: 'a' };
    console.log('is undefined?', foo.b === undefined);
    // is undefined? true
    ```
  • Değer döndürmeyen bir fonksiyonun dönüş değeri
  •     function foo() { return; }
    console.log('is undefined?', foo() === undefined);
    // is undefined? true
    ```
  • Bildirilen ancak işlev çağrısından çıkarılmış bir işlev argümanının değeri
  •     function foo(param) { 
      console.log('is undefined?', param === undefined);
    }
    foo('a');
    foo();
    // is undefined? false
    // is undefined? true
    ```

“undefined” aynı zamanda global “window” nesnesinin bir özelliğidir.

// Only in browsers
console.log(window.undefined); // undefined
window.hasOwnProperty('undefined'); // true

ECMAScript 5’ten önce, “window.undefined” özelliğinin değerini, potansiyel olarak her şeyi bozabilecek herhangi bir başka değerle değiştirebiliyordunuz.

Sonsuzluk ve -Sonsuzluk #

1 / 0; // Infinity
// Wait! WHAAAT?

“Sonsuz”, matematiksel sonsuzu temsil eden global nesnenin (dolayısıyla global bir değişkenin) bir özelliğidir. “Number.POSITIVE_INFINITY” referansıdır

Diğer tüm değerlerden daha büyüktür ve 0’a bölerek veya taşacak kadar büyük bir sayının ifadesini değerlendirerek elde edebilirsiniz. Bu aslında JavaScript’te 0 hataya bölünme olmadığı anlamına gelir, Infinity vardır!

Ayrıca matematiksel negatif sonsuzluk olan `-Sonsuz’ vardır ve diğer tüm değerlerden daha düşüktür.

“-Sonsuz"u elde etmek için “Sonsuz"u reddedersiniz veya ona “Number.NEGATIVE_INFINITY” içinde bir referans alırsınız.

- (Infinity); // -Infinity

Şimdi örneklerle biraz eğlenelim:

Infinity > 123192310293; // true
-Infinity < -123192310293; // true
1 / 0; // Infinity
Math.pow(123123123, 9123192391023); // Infinity
Number.MAX_VALUE * 2; // Infinity
23 / Infinity; // 0
-Infinity; // -Infinity
-Infinity === Number.NEGATIVE_INFINITY; // true
-0; // -0 , yes there is a negative 0 in the language
0 === -0; // true
1 / -0; // -Infinity
1 / 0 === 1 / -0; // false
Infinity + Infinity; // Infinity

var a = 0, b = -0;

a === b; // true
1 / a === 1 / b; // false

// Try your own!

Sayı sabitleri #

“Number” yapıcısı, yararlı olabilecek bazı yerleşik sabitlere sahiptir.

Number.MAX_VALUE;          // 1.7976931348623157e+308
Number.MAX_SAFE_INTEGER;   // 9007199254740991

Number.MIN_VALUE;          // 5e-324
Number.MIN_SAFE_INTEGER;   // -9007199254740991

Number.EPSILON;            // 0.0000000000000002220446049250313

Number.POSITIVE_INFINITY;  // Infinity
Number.NEGATIVE_INFINITY;  // -Infinity

Number.NaN;                // NaN

Çoğu durumda Javascript’teki çeşitli operatörler, (Number.MIN_SAFE_INTEGER, Number.MAX_SAFE_INTEGER) aralığının dışındaki değerlerle kırılacaktır.

‘Number.EPSILON’ öğesinin bir ile birden büyük olan en küçük ‘Sayı’ arasındaki farkı ve dolayısıyla iki farklı ‘Sayı’ değeri arasındaki olası en küçük farkı temsil ettiğini unutmayın. Bunu kullanmanın bir nedeni, sayıların JavaScript tarafından nasıl saklandığının doğasıdır, bkz. [İki sayının eşitliğini kontrol edin](https://www.wikiod.com/tr/javascript -of-iki-sayı#t=201607291343405411584)
