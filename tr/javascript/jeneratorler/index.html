Üreteç işlevleri (‘function*’ anahtar sözcüğüyle tanımlanır), eşyordamlar olarak çalışır ve bir yineleyici aracılığıyla istendikleri gibi bir dizi değer üretir.

Sözdizimi #

  • fonksiyon* isim(parametreler) { verim değeri; geri dönüş değeri }
  • üreteç = isim(argümanlar)
  • { değer, tamamlandı } = jeneratör.next(değer)
  • { değer, tamamlandı } = üreteç.dönüş(değer)
  • generator.throw(hata)

Jeneratör işlevleri, ES 2015 spesifikasyonunun bir parçası olarak sunulan bir özelliktir ve tüm tarayıcılarda mevcut değildir. Ayrıca, “v6.0"dan itibaren Node.js’de tam olarak desteklenirler. Ayrıntılı bir tarayıcı uyumluluk listesi için MDN Belgelerine bakın ve Düğüm için bkz. node.green web sitesi.

Jeneratör Fonksiyonları #

Bir generator function, bir “function*” bildirimiyle oluşturulur. Çağrıldığında, gövdesi hemen yürütülmez. Bunun yerine, işlevin yürütülmesinde “adım adım ilerlemek” için kullanılabilen bir generator nesnesi döndürür.

İşlev gövdesi içindeki bir “verim” ifadesi, yürütmenin askıya alınabileceği ve devam ettirilebileceği bir noktayı tanımlar.

function* nums() {
    console.log('starting');  // A
    yield 1;                  // B
    console.log('yielded 1'); // C
    yield 2;                  // D
    console.log('yielded 2'); // E
    yield 3;                  // F
    console.log('yielded 3'); // G
}
var generator = nums(); // Returns the iterator. No code in nums is executed

generator.next();  // Executes lines A,B returning { value: 1, done: false }
// console: "starting"
generator.next();  // Executes lines C,D returning { value: 2, done: false }
// console: "yielded 1"
generator.next();  // Executes lines E,F returning { value: 3, done: false }
// console: "yielded 2"
generator.next();  // Executes line G returning { value: undefined, done: true }
// console: "yielded 3"

Erken yineleme çıkışı #

generator = nums();
generator.next(); // Executes lines A,B returning { value: 1, done: false }
generator.next(); // Executes lines C,D returning { value: 2, done: false }
generator.return(3); // no code is executed returns { value: 3, done: true }
// any further calls will return done = true 
generator.next(); // no code executed returns { value: undefined, done: true }

Jeneratör işlevine hata atma #

function* nums() {
    try {
        yield 1;                  // A
        yield 2;                  // B
        yield 3;                  // C
    } catch (e) {
        console.log(e.message);    // D
    }
}

var generator = nums();

generator.next();  // Executes line A returning { value: 1, done: false }
generator.next();  // Executes line B returning { value: 2, done: false }
generator.throw(new Error("Error!!"));  // Executes line D returning { value: undefined, done: true}
// console: "Error!!"
generator.next();  // no code executed. returns { value: undefined, done: true }

Jeneratöre Değer Gönderme #

Bir değeri next() yöntemine geçirerek jeneratöre _göndermek mümkündür.

function* summer() {
    let sum = 0, value;
    while (true) {
        // receive sent value
        value = yield;
        if (value === null) break;

        // aggregate values
        sum += value;
    }
    return sum;
}
let generator = summer();

// proceed until the first "yield" expression, ignoring the "value" argument
generator.next();

// from this point on, the generator aggregates values until we send "null"
generator.next(1);
generator.next(10);
generator.next(100);

// close the generator and collect the result
let sum = generator.next(null).value;  // 111

Başka Üreticiye Yetkilendirme #

Bir üreteç işlevi içinden, kontrol, “verim*” kullanılarak başka bir oluşturucu işlevine devredilebilir.

function* g1() {
  yield 2;
  yield 3;
  yield 4;
}

function* g2() {
  yield 1;
  yield* g1();
  yield 5;
}

var it = g2();

console.log(it.next()); // 1
console.log(it.next()); // 2
console.log(it.next()); // 3
console.log(it.next()); // 4
console.log(it.next()); // 5
console.log(it.next()); // undefined

Yineleme #

Bir üreteç _yinelenebilir_dir. Bir “for…of” ifadesi ile döngüye alınabilir ve yineleme protokolüne bağlı olan diğer yapılarda kullanılabilir.

function* range(n) {
    for (let i = 0; i < n; ++i) {
        yield i;
    }
}

// looping
for (let n of range(10)) {
    // n takes on the values 0, 1, ... 9
}

// spread operator
let nums = [...range(3)];  // [0, 1, 2]
let max = Math.max(...range(100));  // 99

Burada, ES6’da özel yinelenebilir nesne için oluşturucu kullanımına başka bir örnek verilmiştir. Burada anonim üreteç işlevi fonksiyonu * kullanılır.

let user = {
  name: "sam", totalReplies: 17, isBlocked: false
};

user[Symbol.iterator] = function *(){

  let properties = Object.keys(this);
  let count = 0;
  let isDone = false;

  for(let p of properties){
    yield this[p];
  }
};

for(let p of user){
  console.log( p );
}

Jeneratörlerle zaman uyumsuz akış #

Jeneratörler, yürütmeyi duraklatabilen ve ardından devam ettirebilen işlevlerdir. Bu, başta q veya co olmak üzere harici kitaplıkları kullanarak zaman uyumsuz işlevlerin öykünmesine olanak tanır. Temel olarak, devam etmek için zaman uyumsuz sonuçları bekleyen işlevlerin yazılmasına izin verir:

function someAsyncResult() {
    return Promise.resolve('newValue')
}

q.spawn(function * () {
    var result = yield someAsyncResult()
    console.log(result) // 'newValue'
})

Bu, zaman uyumsuz kod sanki senkronmuş gibi yazmaya izin verir. Ayrıca, birkaç eşzamansız blok üzerinde çalışmayı deneyin ve yakalayın. Söz reddedilirse, hata bir sonraki yakalama tarafından yakalanır:

function asyncError() {
    return new Promise(function (resolve, reject) {
        setTimeout(function () {
            reject(new Error('Something went wrong'))
        }, 100)
    })
}

q.spawn(function * () {
    try {
        var result = yield asyncError()
    } catch (e) {
        console.error(e) // Something went wrong
    }
})

co kullanmak tamamen aynı şekilde çalışır ancak ‘q.spawn’ yerine ‘co(function * (){…})’ ile çalışır.

Yineleyici-Gözlemci arayüzü #

Bir üreteç, iki şeyin birleşimidir - bir “Yineleyici” ve bir “Gözlemci”.

Yineleyici #

Yineleyici, çağrıldığında bir “yinelenebilir” döndüren bir şeydir. “Yinelenebilir”, üzerinde yineleyebileceğiniz bir şeydir. ES6/ES2015’ten itibaren tüm koleksiyonlar (Array, Map, Set, WeakMap, WeakSet) Yinelenebilir sözleşmeye uygundur.

Bir üreteç(yineleyici) bir üreticidir. Yinelemede tüketici PULL üreticinin değeridir.

Örnek:

function *gen() { yield 5; yield 6; }
let a = gen();

“a.next()“i her çağırdığınızda, aslında yineleyiciden değer “çekiyorsunuz” ve yürütmeyi “verim"de “duraklatıyorsunuz”. “a.next()“i bir sonraki çağırışınızda, yürütme önceden duraklatılmış durumdan devam eder.

Gözlemci #

Bir jeneratör aynı zamanda bazı değerleri jeneratöre geri gönderebileceğiniz bir gözlemcidir.

function *gen() {
  document.write('<br>observer:', yield 1);
}
var a = gen();
var i = a.next();
while(!i.done) {
  document.write('<br>iterator:', i.value);
  i = a.next(100);
}

Burada ‘verim 1’in bir değer olarak değerlendirilen bir ifade gibi kullanıldığını görebilirsiniz. Değerlendirdiği değer, “a.next” işlev çağrısına argüman olarak gönderilen değerdir.

Bu nedenle, ilk kez ‘i.value’, elde edilen ilk değer (‘1’) olacaktır ve yinelemeyi bir sonraki duruma devam ederken, ‘a.next(100)’ kullanarak üreticiye bir değer göndeririz. .

Jeneratörlerle zaman uyumsuz yapmak #

Jeneratörler, fonksiyonun bir jeneratörü aldığı ve senkronize bir şekilde asenkron kod yazmamıza izin verdiği ‘spawn’ (görev JS veya co’dan) işleviyle yaygın olarak kullanılır. Bu, zaman uyumsuz kodun eşitleme koduna dönüştürüleceği/eşzamanlı olarak yürütüleceği anlamına DEĞİLDİR. Bu, ‘sync’ gibi görünen ancak dahili olarak hala ‘async’ olan kod yazabileceğimiz anlamına gelir.

Senkronizasyon ENGELLENİYOR; Zaman uyumsuz BEKLİYOR. Engelleyen kod yazmak kolaydır. PULLing yaparken, atama konumunda değer belirir. PUSHing sırasında, geri aramanın argüman konumunda değer görünür.

Yineleyicileri kullandığınızda, değeri üreticiden ‘ÇEKİRSİNİZ’. Geri aramaları kullandığınızda, üretici değeri geri aramanın bağımsız değişken konumuna PUSH gönderir.

var i = a.next() // PULL
dosomething(..., v => {...}) // PUSH

Burada, değeri “a.next()“den çekersiniz ve ikincisinde, “v => {…}” geri aramadır ve bir değer, geri aramanın “v” bağımsız değişken konumuna “PUSH” gönderilir. işlev.

Bu çekme-itme mekanizmasını kullanarak, aşağıdaki gibi zaman uyumsuz programlama yazabiliriz,

let delay = t => new Promise(r => setTimeout(r, t));
spawn(function*() {
  // wait for 100 ms and send 1
  let x = yield delay(100).then(() => 1);
  console.log(x); // 1

   // wait for 100 ms and send 2
  let y = yield delay(100).then(() => 2);
  console.log(y); // 2
});

Bu nedenle, yukarıdaki koda bakarak, ’engelliyor’ gibi görünen zaman uyumsuz kod yazıyoruz (verim ifadeleri 100ms bekler ve ardından yürütmeye devam eder), ancak aslında ‘bekliyor’. Jeneratörün “duraklat” ve “devam ettir” özelliği bu harika numarayı yapmamızı sağlar.

O nasıl çalışır ? #

Oluşturma işlevi, söz verme durumunu jeneratörden ÇEKMEK için ‘verim vaadi’ kullanır, söz verilene kadar bekler ve çözümlenen değeri, tüketebilmesi için jeneratöre geri iter.

Şimdi kullan #

Böylece, üreteçler ve yumurtlama işleviyle, eşzamanlı gibi görünmek ve hissetmek için NodeJS’deki tüm zaman uyumsuz kodunuzu temizleyebilirsiniz. Bu, hata ayıklamayı kolaylaştıracaktır. Ayrıca kod düzgün görünecek.

Bu özellik, JavaScript’in gelecekteki sürümlerine “async…await” olarak geliyor. Ancak bunları bugün ES2015/ES6’da kitaplıklarda tanımlanan spawn işlevini kullanarak kullanabilirsiniz - taskjs, co veya bluebird
