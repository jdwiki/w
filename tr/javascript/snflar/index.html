Sözdizimi #

  • sınıf Foo {}
  • sınıf Foo, Bar’ı {} genişletir
  • sınıf Foo { yapıcı() {} }
  • class Foo { myMethod() {} }
  • class Foo { myProperty() {} }
  • sınıf Foo { myProperty(newValue) {} } ayarla
  • sınıf Foo { statik myStaticMethod() {} }
  • sınıf Foo { statik get myStaticProperty() {} }
  • const Foo = sınıf Foo {};
  • const Foo = sınıf {};

“sınıf” desteği, yalnızca 2015 [tag:es6] standardının bir parçası olarak JavaScript’e eklendi.

Javascript sınıfları, JavaScript’in zaten var olan prototip tabanlı mirasına göre sözdizimsel şekerdir. Bu yeni sözdizimi, JavaScript’e yeni bir nesne yönelimli kalıtım modeli sunmaz, yalnızca nesneler ve kalıtımla başa çıkmanın daha basit bir yoludur. Bir “sınıf” bildirimi, esasen, bir yapıcı “işlevini” manuel olarak tanımlamak ve yapıcının prototipine özellikler eklemek için kullanılan bir kısayoldur. Önemli bir fark, işlevlerin doğrudan çağrılabilmesidir (’new’ anahtar sözcüğü olmadan), oysa doğrudan çağrılan bir sınıf bir istisna atar.

class someClass {
    constructor () {}
    someMethod () {}
}
 
console.log(typeof someClass);               
console.log(someClass);
console.log(someClass === someClass.prototype.constructor);                         
console.log(someClass.prototype.someMethod);
 
// Output:
// function
// function someClass() { "use strict"; }
// true
// function () { "use strict"; }

JavaScript’in daha eski bir sürümünü kullanıyorsanız, kodu hedef platformun anlayabileceği bir sürümde derlemek için [tag:babel] veya [tag:google-closure-compiler] gibi bir aktarıcıya ihtiyacınız olacaktır.

Sınıf Oluşturucu #

Çoğu sınıfın temel parçası, her örneğin başlangıç ​​durumunu ayarlayan ve “yeni” çağrılırken geçirilen tüm parametreleri işleyen yapıcısıdır.

Bir “sınıf” bloğunda, “yapıcı” adında bir yöntem tanımlıyormuşsunuz gibi tanımlanır, ancak aslında özel bir durum olarak ele alınır.

class MyClass {
    constructor(option) {
        console.log(`Creating instance using ${option} option.`);
        this.option = option;
    }
}

Örnek kullanım:

const foo = new MyClass('speedy'); // logs: "Creating instance using speedy option"

Unutulmaması gereken küçük bir nokta, diğer yöntemler için aşağıda açıklandığı gibi, bir sınıf yapıcısının ‘static’ anahtar sözcüğü aracılığıyla statik hale getirilemeyeceğidir.

Sınıf Mirası #

Kalıtım, diğer nesne yönelimli dillerde olduğu gibi çalışır: üst sınıfta tanımlanan yöntemlere, genişleyen alt sınıftan erişilebilir.

Alt sınıf kendi kurucusunu bildirirse, o zaman ‘buna’ erişebilmeden önce üst yapıcıyı ‘super()’ aracılığıyla çağırmalıdır.

class SuperClass {

    constructor() {
        this.logger = console.log;
    }

    log() {
        this.logger(`Hello ${this.name}`);
    }

}

class SubClass extends SuperClass {

    constructor() {
        super();
        this.name = 'subclass';
    }

}

const subClass = new SubClass();

subClass.log(); // logs: "Hello subclass"

Statik Yöntemler #

Statik yöntemler ve özellikler, örnek nesnelerde değil, sınıfın/yapıcının kendisinde tanımlanır. Bunlar, ‘static’ anahtar sözcüğü kullanılarak bir sınıf tanımında belirtilir.

class MyClass {
    static myStaticMethod() {
        return 'Hello';
    }

    static get myStaticProperty() {
        return 'Goodbye';
    }
}

console.log(MyClass.myStaticMethod()); // logs: "Hello"
console.log(MyClass.myStaticProperty); // logs: "Goodbye"

Nesne örneklerinde statik özelliklerin tanımlanmadığını görebiliriz:

const myClassInstance = new MyClass();

console.log(myClassInstance.myStaticProperty); // logs: undefined

Ancak bunlar *alt sınıflarda tanımlanırlar:

class MySubClass extends MyClass {};

console.log(MySubClass.myStaticMethod()); // logs: "Hello"
console.log(MySubClass.myStaticProperty); // logs: "Goodbye"

Alıcılar ve Ayarlayıcılar #

Alıcılar ve ayarlayıcılar, sınıfınızda belirli bir özelliği okumak ve yazmak için özel davranış tanımlamanıza olanak tanır. Kullanıcıya, herhangi bir tipik mülkle aynı görünürler. Ancak, dahili olarak sağladığınız özel bir işlev, özelliğe erişildiğinde (alıcı) değeri belirlemek ve özellik atandığında (ayarlayıcı) gerekli değişiklikleri önceden yapmak için kullanılır.

Bir ‘sınıf’ tanımında, bir alıcı, ‘get’ anahtar sözcüğü ile öneki argümansız bir yöntem gibi yazılır. Bir ayarlayıcı, bir argümanı kabul etmesi (atanan yeni değer) ve bunun yerine “set” anahtar sözcüğünün kullanılması dışında benzerdir.

İşte .name özelliği için bir alıcı ve ayarlayıcı sağlayan örnek bir sınıf. Her atandığında, yeni adı dahili bir .names_ dizisine kaydedeceğiz. Her erişildiğinde, en son adı döndürürüz.

class MyClass {
    constructor() {
        this.names_ = [];
    }

    set name(value) {
        this.names_.push(value);
    }

    get name() {
        return this.names_[this.names_.length - 1];
    }
}

const myClassInstance = new MyClass();
myClassInstance.name = 'Joe';
myClassInstance.name = 'Bob';

console.log(myClassInstance.name); // logs: "Bob"
console.log(myClassInstance.names_); // logs: ["Joe", "Bob"]

Yalnızca bir ayarlayıcı tanımlarsanız, özelliğe erişmeye çalışmak her zaman ’tanımsız’ döndürür.

const classInstance = new class {
    set prop(value) {
        console.log('setting', value);
    }
};

classInstance.prop = 10; // logs: "setting", 10

console.log(classInstance.prop); // logs: undefined

Yalnızca bir alıcı tanımlarsanız, özelliği atamaya çalışmanın hiçbir etkisi olmaz.

const classInstance = new class {
    get prop() {
        return 5;
    }
};

classInstance.prop = 10;

console.log(classInstance.prop); // logs: 5

Özel Üyeler #

JavaScript, bir dil özelliği olarak özel üyeleri teknik olarak desteklemez. Gizlilik - [Douglas Crockford1 tarafından açıklanmıştır] - bunun yerine kapatmalar yoluyla öykünür (korunmuş işlev kapsamı) her örnekleme çağrısında üretilecek bir yapıcı işlevin.

“Kuyruk” örneği, yapıcı işlevlerle yerel durumun nasıl olduğunu gösterir. ayrıcalıklı yöntemlerle de korunabilir ve erişilebilir hale getirilebilir.

class Queue {

  constructor () {                    // - does generate a closure with each instantiation.

    const list = [];                  // - local state ("private member").

    this.enqueue = function (type) {  // - privileged public method
                                      //   accessing the local state
      list.push(type);                //   "writing" alike.
      return type;
    };
    this.dequeue = function () {      // - privileged public method
                                      //   accessing the local state
      return list.shift();            //   "reading / writing" alike.
    };
  }
}


var q = new Queue;            //
                              //
q.enqueue(9);                 // ... first in ...
q.enqueue(8);                 //
q.enqueue(7);                 //
                              //
console.log(q.dequeue());     // 9 ... first out.
console.log(q.dequeue());     // 8
console.log(q.dequeue());     // 7
console.log(q);               // {}
console.log(Object.keys(q));  // ["enqueue","dequeue"]

Bir “Queue” türünün her örneğinde, yapıcı bir kapatma oluşturur.

Bu nedenle, bir ‘Queue’ türünün her ikisi de ’enqueue’ ve ‘dequeue’ yöntemlerinin her ikisi de (bkz. ‘Object.keys(q)’) hala kendi kapsamı içinde yaşamaya devam eden “listeye” erişiminiz var, yapım aşamasında korunmuştur.

Bu kalıptan faydalanmak - ayrıcalıklı kamu aracılığıyla özel üyeleri taklit etmek yöntemler - her örnekte ek belleğin olacağı akılda tutulmalıdır. her kendi mülkü yöntemi için tüketilir (çünkü bu, paylaşılamayan/yeniden kullanılamayan koddur). Aynısı, böyle bir kapatma içinde saklanacak olan durumun miktarı/boyutu için de geçerlidir.

Dinamik Yöntem Adları #

Bir nesnenin özelliklerine [] ile nasıl erişebileceğinize benzer şekilde, yöntemleri adlandırırken ifadeleri değerlendirme yeteneği de vardır. Bu, dinamik özellik adlarına sahip olmak için yararlı olabilir, ancak genellikle Semboller ile birlikte kullanılır.

let METADATA = Symbol('metadata');

class Car {
  constructor(make, model) {
    this.make = make;
    this.model = model;
  }
  
  // example using symbols
  [METADATA]() {
    return {
      make: this.make,
      model: this.model
    };
  }

  // you can also use any javascript expression

  // this one is just a string, and could also be defined with simply add()
  ["add"](a, b) {
    return a + b;
  }

  // this one is dynamically evaluated
  [1 + 2]() {
    return "three";
  }
}

let MazdaMPV = new Car("Mazda", "MPV");
MazdaMPV.add(4, 5); // 9
MazdaMPV[3](); // "three"
MazdaMPV[METADATA](); // { make: "Mazda", model: "MPV" }

Yöntemler #

Yöntemler, bir işlevi gerçekleştirmek ve isteğe bağlı olarak bir sonuç döndürmek için sınıflarda tanımlanabilir. Arayandan argüman alabilirler.

class Something {
    constructor(data) {
        this.data = data
    }

    doSomething(text) {
        return {
            data: this.data,
            text
        }
    }
}

var s = new Something({})
s.doSomething("hi") // returns: { data: {}, text: "hi" }

Sınıflarla Özel Verileri Yönetme #

Sınıfları kullanmanın en yaygın engellerinden biri, özel durumları ele almak için uygun yaklaşımı bulmaktır. Özel durumları ele almak için 4 ortak çözüm vardır:

Sembolleri Kullanma #

Semboller, ES2015’te tanıtılan yeni ilkel türdür. MDN‘de tanımlandığı gibi

Sembol, nesne özellikleri için tanımlayıcı olarak kullanılabilen benzersiz ve değişmez bir veri türüdür.

Bir özellik anahtarı olarak sembol kullanıldığında, numaralandırılamaz.

Bu nedenle, “for var in” veya “Object.keys” kullanılarak gösterilmeyeceklerdir.

Böylece özel verileri saklamak için sembolleri kullanabiliriz.

const topSecret = Symbol('topSecret'); // our private key; will only be accessible on the scope of the module file
export class SecretAgent{
    constructor(secret){
        this[topSecret] = secret; // we have access to the symbol key (closure)
        this.coverStory = 'just a simple gardner';
        this.doMission = () => {
            figureWhatToDo(topSecret[topSecret]); // we have access to topSecret
        };
    }
}

“Semboller” benzersiz olduğundan, özel mülke erişmek için orijinal simgeye başvuruda bulunmamız gerekir.

import {SecretAgent} from 'SecretAgent.js'
const agent = new SecretAgent('steal all the ice cream');
// ok lets try to get the secret out of him!
Object.keys(agent); // ['coverStory'] only cover story is public, our secret is kept.
agent[Symbol('topSecret')]; // undefined, as we said, symbols are always unique, so only the original symbol will help us to get the data.

Ama %100 özel değil; hadi o ajanı kıralım! Nesne sembollerini almak için Object.getOwnPropertySymbols yöntemini kullanabiliriz.

const secretKeys = Object.getOwnPropertySymbols(agent);
agent[secretKeys[0]] // 'steal all the ice cream' , we got the secret.

WeakMaps’i Kullanma #

“WeakMap”, es6 için eklenen yeni bir nesne türüdür.

MDN‘de tanımlandığı gibi

WeakMap nesnesi, anahtarlara zayıf referans verilen bir anahtar/değer çiftleri topluluğudur. Anahtarlar nesneler olmalıdır ve değerler isteğe bağlı değerler olabilir.

‘WeakMap’in bir diğer önemli özelliği de MDN‘de tanımlandığı gibidir.

Bir WeakMap’teki anahtar zayıf tutulur. Bunun anlamı, anahtara başka güçlü referanslar yoksa, tüm girişin çöp toplayıcı tarafından WeakMap’ten kaldırılacağıdır.

Buradaki fikir, her örneği anahtar olarak tutmak ve özel verileri o örnek anahtarı için bir değer olarak tutmak için tüm sınıf için statik bir harita olarak WeakMap’i kullanmaktır.

Böylece WeakMap koleksiyonuna sadece sınıfın içinde erişebileceğiz.

Temsilcimizi WeakMap ile deneyelim:

const topSecret = new WeakMap(); // will hold all private data of all instances.
export class SecretAgent{
    constructor(secret){
        topSecret.set(this,secret); // we use this, as the key, to set it on our instance private data
        this.coverStory = 'just a simple gardner';
        this.doMission = () => {
            figureWhatToDo(topSecret.get(this)); // we have access to topSecret
        };
    }
}

const ’topSecret’ modül kapatmamızın içinde tanımlandığından ve onu örnek özelliklerimize bağlamadığımızdan, bu yaklaşım tamamen özeldir ve ’topSecret’ aracısına ulaşamıyoruz.

Yapıcı içindeki tüm yöntemleri tanımlayın #

Buradaki fikir, yapıcı içindeki tüm yöntemlerimizi ve üyelerimizi tanımlamak ve özel üyelere “buna” atamadan erişmek için kapatmayı kullanmaktır.

   export class SecretAgent{
        constructor(secret){
            const topSecret = secret;
            this.coverStory = 'just a simple gardner';
            this.doMission = () => {
                figureWhatToDo(topSecret); // we have access to topSecret
            };
        }
    }

Bu örnekte de veriler %100 özeldir ve sınıf dışından erişilemez, dolayısıyla aracımız güvendedir.

Adlandırma kurallarını kullanma #

Özel olan herhangi bir mülkün önüne ‘_’ getirileceğine karar vereceğiz.

Bu yaklaşım için verilerin gerçekten özel olmadığını unutmayın.

export class SecretAgent{
    constructor(secret){
        this._topSecret = secret; // it private by convention
        this.coverStory = 'just a simple gardner';
        this.doMission = () => {
            figureWhatToDo(this_topSecret); 
        };
    }
}

Sınıf Adı bağlama #

ClassDeclaration’ın Adı, farklı kapsamlarda farklı şekillerde bağlıdır -

  1. Sınıfın tanımlandığı kapsam - let bağlama
  2. Sınıfın kendisinin kapsamı - “sınıf {}” içindeki “{” ve “}” içinde - “const” bağlama
class Foo {
  // Foo inside this block is a const binding
}
// Foo here is a let binding

Örneğin,

class A {
  foo() {
    A = null; // will throw at runtime as A inside the class is a `const` binding
  }
}
A = null; // will NOT throw as A here is a `let` binding

Bu, bir İşlev için aynı değildir -

function A() {
  A = null; // works
}
A.prototype.foo = function foo() {
  A = null; // works
}
A = null; // works
