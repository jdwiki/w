Sözdizimi #

  • nesne = { }
  • nesne = yeni Nesne()
  • nesne = Object.create(prototip[, özelliklerObject])
  • nesne.anahtar = değer
  • nesne[“anahtar”] = değer
  • nesne[Symbol()] = değer
  • nesne = { anahtar1: değer1, “anahtar2”: değer2, ‘anahtar3’: değer3 }
  • nesne = { özlüYöntem() { … } }
  • nesne = { [hesaplanmış () + “anahtar”]: değer }
  • Object.defineProperty(obj, propertyName, propertyDescriptor)
  • property_desc = Object.getOwnPropertyDescriptor(obj, propertyName)
  • Object.freeze(obj)
  • Object.seal(obj)

Parametreler #

Mülk Açıklama
“değer” Özelliğe atanacak değer.
‘yazılabilir’ Özelliğin değerinin değiştirilip değiştirilemeyeceği.
“numaralandırılabilir” Özelliğin for in döngülerinde numaralandırılıp numaralandırılmayacağı.
‘yapılandırılabilir’ Özellik tanımlayıcısını yeniden tanımlamanın mümkün olup olmayacağı.
al Özelliğin değerini döndürecek çağrılacak bir işlev.
“ayarla” Özelliğe bir değer atandığında çağrılacak bir fonksiyon.

Nesneler, anahtar/değer çiftlerinin veya özelliklerin koleksiyonlarıdır. Anahtarlar “Dize” veya “Symbol” olabilir ve değerler ya temel öğelerdir (sayılar, dizeler, semboller) ya da diğer nesnelere referanslardır.

JavaScript’te önemli miktarda değerler, nesneler (örneğin işlevler, diziler) veya değişmez nesneler (sayılar, dizeler, booleanlar) gibi davranan ilkellerdir. Özelliklerine veya ‘prototip’ özelliklerine nokta (‘obj.prop’) veya parantez (‘obj[‘prop’]’) gösterimi kullanılarak erişilebilir. Kayda değer istisnalar, “undefined” ve “null” özel değerleridir.

Nesneler, değere göre değil, JavaScript’te referansa göre tutulur. Bu, kopyalandığında veya işlevlere argüman olarak iletildiğinde, “kopya” ve orijinalin aynı nesneye referanslar olduğu ve birinin özelliklerinde yapılan bir değişikliğin diğerinin aynı özelliğini değiştireceği anlamına gelir. Bu, değişmez olan ve değere göre iletilen ilkel öğeler için geçerli değildir.

Sığ klonlama #

ES6’nın ‘Object.assign()’ işlevi, mevcut bir ‘Object’ örneğinden tüm enumerable özelliklerini yenisine kopyalamak için kullanılabilir.

const existing = { a: 1, b: 2, c: 3 };

const clone = Object.assign({}, existing);

Bu, “Dize” özelliklerine ek olarak “Sembol” özelliklerini de içerir.

son sürüm

Şu anda bir 3. aşama önerisi olan Nesne dinlenme/yayılma imhası, Nesne örneklerinin sığ klonlarını oluşturmak için daha da basit bir yol sağlar:

const existing = { a: 1, b: 2, c: 3 };

const { ...clone } = existing;

JavaScript’in eski sürümlerini desteklemeniz gerekiyorsa, bir Nesneyi klonlamanın en uyumlu yolu, özellikleri üzerinde manuel olarak yineleme yapmak ve .hasOwnProperty() kullanarak devralınanları filtrelemektir.

var existing = { a: 1, b: 2, c: 3 };

var clone = {};
for (var prop in existing) {
  if (existing.hasOwnProperty(prop)) {
    clone[prop] = existing[prop];
  }
}

Nesne.dondur #

‘Object.freeze’, yeni özelliklerin eklenmesini, mevcut özelliklerin kaldırılmasını ve mevcut özelliklerin numaralandırılabilirliğini, yapılandırılabilirliğini ve yazılabilirliğini değiştirerek bir nesneyi değişmez kılar. Ayrıca mevcut özelliklerin değerinin değiştirilmesini de engeller. Ancak, özyinelemeli olarak çalışmaz; bu, alt nesnelerin otomatik olarak dondurulmadığı ve değişikliğe tabi olduğu anlamına gelir.

son sürüm

Dondurmayı izleyen işlemler, kod katı modda çalışmadığı sürece sessizce başarısız olur. Kod katı moddaysa, bir ‘TypeError’ atılacaktır.

var obj = {
  foo: 'foo',
  bar: [1, 2, 3],
  baz: {
    foo: 'nested-foo'
  }
};

Object.freeze(obj);

// Cannot add new properties
obj.newProperty = true;

// Cannot modify existing values or their descriptors
obj.foo = 'not foo';
Object.defineProperty(obj, 'foo', {
    writable: true
});

// Cannot delete existing properties
delete obj.foo;

// Nested objects are not frozen
obj.bar.push(4);
obj.baz.foo = 'new foo';

Nesne klonlama #

Bir nesnenin tam bir kopyasını istediğinizde (yani nesne özellikleri ve bu özelliklerin içindeki değerler vb…), buna derin klonlama denir.

Bir nesne JSON’a serileştirilebiliyorsa, “JSON.parse” ve “JSON.stringify” kombinasyonuyla onun derin bir klonunu oluşturabilirsiniz:

var existing = { a: 1, b: { c: 2 } };
var copy = JSON.parse(JSON.stringify(existing));
existing.b.c = 3; // copy.b.c will not change

“JSON.stringify” öğesinin “Date” nesnelerini ISO biçiminde dize temsillerine dönüştüreceğini, ancak “JSON.parse” öğesinin dizeyi yeniden “Tarih"e dönüştürmeyeceğini unutmayın.

son sürüm

JavaScript’te derin klonlar oluşturmak için yerleşik bir işlev yoktur ve genel olarak birçok nedenden dolayı her nesne için derin klonlar oluşturmak mümkün değildir. Örneğin,

  • nesneler, tespit edilemeyen numaralandırılamayan ve gizli özelliklere sahip olabilir.
  • nesne alıcıları ve ayarlayıcıları kopyalanamaz.
  • nesneler döngüsel bir yapıya sahip olabilir.
  • işlev özellikleri, gizli bir kapsamda duruma bağlı olabilir.

Özellikleri yalnızca ilkel değerler, tarihler, diziler veya diğer “güzel” nesneler içeren bir “güzel” nesneniz olduğunu varsayarsak, derin klonlar yapmak için aşağıdaki işlev kullanılabilir. Döngüsel yapıya sahip nesneleri algılayabilen ve bu gibi durumlarda hata verecek özyinelemeli bir fonksiyondur.

function deepClone(obj) {
    function clone(obj, traversedObjects) {
        var copy;
        // primitive types
        if(obj === null || typeof obj !== "object") {
            return obj;
        }

        // detect cycles
        for(var i = 0; i < traversedObjects.length; i++) {
            if(traversedObjects[i] === obj) {
                throw new Error("Cannot clone circular object.");
            }
        }

        // dates
        if(obj instanceof Date) {
            copy = new Date();
            copy.setTime(obj.getTime());
            return copy;
        }
        // arrays
        if(obj instanceof Array) {
            copy = [];
            for(var i = 0; i < obj.length; i++) {
                copy.push(clone(obj[i], traversedObjects.concat(obj)));
            }
            return copy;
        }
        // simple objects
        if(obj instanceof Object) {
            copy = {};
            for(var key in obj) {
                if(obj.hasOwnProperty(key)) {
                    copy[key] = clone(obj[key], traversedObjects.concat(obj));
                }
            }
            return copy;
        }
        throw new Error("Not a cloneable object.");
    }

    return clone(obj, []);
}

Nesne.atama #

Object.assign() yöntemi, numaralandırılabilir tüm kendi özelliklerinin değerlerini bir veya daha fazla kaynak nesneden bir hedef nesneye kopyalamak için kullanılır. Hedef nesneyi döndürür.

Mevcut bir nesneye değer atamak için kullanın:

var user = {
    firstName: "John"
};

Object.assign(user, {lastName: "Doe", age:39});
console.log(user); // Logs: {firstName: "John", lastName: "Doe", age: 39}

Veya bir nesnenin sığ bir kopyasını oluşturmak için:

var obj = Object.assign({}, user);

console.log(obj); // Logs: {firstName: "John", lastName: "Doe", age: 39}

Veya birden çok nesneden birçok özelliği bire birleştirin:

var obj1 = {
    a: 1
};
var obj2 = {
    b: 2
};
var obj3 = {
    c: 3
};
var obj = Object.assign(obj1, obj2, obj3);

console.log(obj); // Logs: { a: 1, b: 2, c: 3 }
console.log(obj1); // Logs: { a: 1, b: 2, c: 3 }, target object itself is changed

İlkeller sarılacak, null ve undefined yoksayılacak:

var var_1 = 'abc';
var var_2 = true;
var var_3 = 10;
var var_4 = Symbol('foo');

var obj = Object.assign({}, var_1, null, var_2, undefined, var_3, var_4);
console.log(obj); // Logs: { "0": "a", "1": "b", "2": "c" }

Not, yalnızca dize sarmalayıcıların kendi numaralandırılabilir özellikleri olabilir

Redüktör olarak kullan: (bir diziyi bir nesneyle birleştirir)

return users.reduce((result, user) => Object.assign({}, {[user.id]: user})

Bir özelliği, biri ondan değeri almak için diğeri içindeki değeri ayarlamak için olmak üzere iki işlevin birleşimi olarak ele alın.

Nesne.mühür #

‘Object.seal’, bir nesneden özelliklerin eklenmesini veya kaldırılmasını engeller. Bir nesne mühürlendikten sonra, özellik tanımlayıcıları başka bir türe dönüştürülemez. Object.freeze‘den farklı olarak, özelliklerin düzenlenmesine izin verir.

Bu işlemleri kapalı bir nesne üzerinde yapma girişimleri sessizce başarısız olur

var obj = { foo: 'foo', bar: function () { return 'bar'; } };

Object.seal(obj)

obj.newFoo = 'newFoo';
obj.bar = function () { return 'foo' };

obj.newFoo; // undefined
obj.bar(); // 'foo'

// Can't make foo an accessor property
Object.defineProperty(obj, 'foo', { 
    get: function () { return 'newFoo'; }
}); // TypeError

// But you can make it read only
Object.defineProperty(obj, 'foo', { 
    writable: false
}); // TypeError

obj.foo = 'newFoo';
obj.foo; // 'foo';

Katı modda bu işlemler bir ‘TypeError’ verir

(function () {
    'use strict';

    var obj = { foo: 'foo' };

    Object.seal(obj);

    obj.newFoo = 'newFoo'; // TypeError
}());

Özelliklerin özellikleri : #

Bir nesneden alınabilen özellikler aşağıdaki özelliklere sahip olabilir,

  • Sayılabilir
  • Numaralandırılamaz
  • sahip olmak

Object.defineProperty(ies) kullanarak özellikleri oluştururken, “own” dışındaki özelliklerini ayarlayabiliriz. Bir nesnenin prototip düzeyinde (__proto__) olmayan, doğrudan seviyede bulunan özelliklere kendi özellikleri denir.

Ve bir nesneye ‘Object.defindProperty(ies)’ kullanılmadan eklenen özellikler, onun numaralandırılabilir karakteristiğine sahip olmayacaktır. Bu, doğru olarak kabul edilmesi anlamına gelir.

Sayılandırılabilirliğin amacı : #

Bir özelliğe numaralandırılabilir özellikler ayarlamanın temel amacı, farklı programatik yöntemler kullanarak nesnesinden alınırken belirli özelliğin kullanılabilirliğini sağlamaktır. Bu farklı yöntemler aşağıda derinlemesine tartışılacaktır.

Özellikleri alma yöntemleri : #

Bir nesneden özellikler aşağıdaki yöntemlerle alınabilir,

  1. for..in döngüsü

    This loop is very useful in retrieving enumerable properties from an object. Additionally this loop will retrieve enumerable own properties as well as it will do the same retrieval by traversing through the prototype chain until it sees the prototype as null.

     //Ex 1 : Simple data
 var x = { a : 10 , b : 3} , props = [];

 for(prop in x){
   props.push(prop);
 }

 console.log(props); //["a","b"]

 //Ex 2 : Data with enumerable properties in prototype chain
 var x = { a : 10 , __proto__ : { b : 10 }} , props = [];

 for(prop in x){
   props.push(prop);
 }

 console.log(props); //["a","b"]

 //Ex 3 : Data with non enumerable properties
 var x = { a : 10 } , props = [];
 Object.defineProperty(x, "b", {value : 5, enumerable : false});

 for(prop in x){
   props.push(prop);
 }

 console.log(props); //["a"]

  2. Object.keys() işlevi

    This function was unveiled as a part of EcmaScript 5. It is used to retrieve enumerable own properties from an object. Prior to its release people used to retrieve own properties from an object by combining for..in loop and Object.prototype.hasOwnProperty() function.

     //Ex 1 : Simple data
 var x = { a : 10 , b : 3} , props;

 props = Object.keys(x);

 console.log(props); //["a","b"]

 //Ex 2 : Data with enumerable properties in prototype chain
 var x = { a : 10 , __proto__ : { b : 10 }} , props;

 props = Object.keys(x);

 console.log(props); //["a"]

 //Ex 3 : Data with non enumerable properties
 var x = { a : 10 } , props;
 Object.defineProperty(x, "b", {value : 5, enumerable : false});

 props = Object.keys(x);

 console.log(props); //["a"]

  3. Object.getOwnProperties() işlevi

    This function will retrieve both enumerable and non enumerable, own properties from an object. It was also released as a part of EcmaScript 5.

     //Ex 1 : Simple data
 var x = { a : 10 , b : 3} , props;

 props = Object.getOwnPropertyNames(x);

 console.log(props); //["a","b"]

 //Ex 2 : Data with enumerable properties in prototype chain
 var x = { a : 10 , __proto__ : { b : 10 }} , props;

 props = Object.getOwnPropertyNames(x);

 console.log(props); //["a"]

 //Ex 3 : Data with non enumerable properties
 var x = { a : 10 } , props;
 Object.defineProperty(x, "b", {value : 5, enumerable : false});

 props = Object.getOwnPropertyNames(x);

 console.log(props); //["a", "b"]

Çeşitli : #

Bir nesneden tüm (kendi, numaralandırılabilir, numaralandırılamaz, tüm prototip düzeyinde) özellikleri almak için bir teknik aşağıda verilmiştir,

function getAllProperties(obj, props = []){
  return obj == null ? props :
           getAllProperties(Object.getPrototypeOf(obj),
               props.concat(Object.getOwnPropertyNames(obj)));
}

var x = {a:10, __proto__ : { b : 5, c : 15 }};

//adding a non enumerable property to first level prototype
Object.defineProperty(x.__proto__, "d", {value : 20, enumerable : false});

console.log(getAllProperties(x)); ["a", "b", "c", "d", "...other default core props..."]

Bu da EcmaScript 5’i destekleyen tarayıcılar tarafından desteklenecektir.

Tanımlayıcılar ve Adlandırılmış Özellikler #

Özellikler bir nesnenin üyeleridir. Adlandırılmış her özellik bir çifttir (ad, tanımlayıcı). Ad, erişime izin veren bir dizedir (“nesne.özellikAdı” nokta gösterimini veya köşeli parantez “nesne[“özellikAdı”]` gösterimini kullanarak). Tanımlayıcı, erişildiğinde özelliğin davranışını tanımlayan alanların bir kaydıdır (özelliğe ne olur ve ona erişimden döndürülen değer nedir). Genel olarak, bir özellik bir ismi bir davranışla ilişkilendirir (davranışı bir kara kutu olarak düşünebiliriz).

İki tür adlandırılmış özellik vardır:

  1. veri özelliği: özelliğin adı bir değerle ilişkilendirilir.
  2. erişimci özelliği: özelliğin adı bir veya iki erişimci işleviyle ilişkilendirilir.

Gösteri:

obj.propertyName1 = 5; //translates behind the scenes into
                       //either assigning 5 to the value field* if it is a data property
                //or calling the set function with the parameter 5 if accessor property


//*actually whether an assignment would take place in the case of a data property
//also depends on the presence and value of the writable field - on that later on

Özelliğin türü, tanımlayıcısının alanları tarafından belirlenir ve bir özellik her iki türden de olamaz.

Veri tanımlayıcıları -

  • Gerekli alanlar: “değer” veya “yazılabilir” veya her ikisi
  • İsteğe bağlı alanlar: “yapılandırılabilir”, “numaralandırılabilir”

Örneklem:

{
   value: 10,
   writable: true;
}

Erişim tanımlayıcıları -

  • Gerekli alanlar: “get” veya “set” veya her ikisi
  • İsteğe bağlı alanlar: “yapılandırılabilir”, “numaralandırılabilir”

Örneklem:

{
    get: function () {
        return 10;
    },
    enumerable: true
}

alanların anlamı ve varsayılanları

"yapılandırılabilir", "numaralandırılabilir" ve "yazılabilir":
  • Bu anahtarların tümü varsayılan olarak “false"dır.
  • “yapılandırılabilir”, ancak ve ancak bu özellik tanımlayıcısının türü değiştirilebiliyorsa ve özellik ilgili nesneden silinebiliyorsa “doğru” olur.
  • “numaralandırılabilir”, ancak ve ancak bu özellik, karşılık gelen nesnedeki özelliklerin numaralandırılması sırasında ortaya çıkarsa “doğru” olur.
  • “yazılabilir”, ancak ve ancak özellik ile ilişkili değer bir atama operatörü ile değiştirilebiliyorsa “doğru” olur.

“get” ve “set”:

  • Bu anahtarlar varsayılan olarak “tanımsız"dır.
  • ‘get’, özellik için alıcı işlevi gören bir işlevdir veya alıcı yoksa ’tanımsız’dır. İşlev dönüşü, özelliğin değeri olarak kullanılacaktır.
  • ‘set’, özellik için ayarlayıcı görevi gören bir fonksiyondur veya ayarlayıcı yoksa ’tanımsız’dır. İşlev, özelliğe atanan yeni değeri yalnızca argüman olarak alır.

“değer”:

  • Bu anahtar varsayılan olarak “tanımsız"dır.
  • Özellik ile ilişkili değer. Herhangi bir geçerli JavaScript değeri olabilir (sayı, nesne, işlev vb.).

Örnek:

    var obj = {propertyName1: 1}; //the pair is actually ('propertyName1', {value:1,
                                                                    // writable:true,
                                                                    // enumerable:true,
                                                                    // configurable:true})
    Object.defineProperty(obj, 'propertyName2', {get: function() {
                                                    console.log('this will be logged ' + 
                                 'every time propertyName2 is accessed to get its value');
                                                },
                                            set: function() {
                                                    console.log('and this will be logged ' + 
                                'every time propertyName2\'s value is tried to be set')
                      //will be treated like it has enumerable:false, configurable:false
                                                }});
//propertyName1 is the name of obj's data property 
//and propertyName2 is the name of its accessor property





obj.propertyName1 = 3;
console.log(obj.propertyName1); //3

obj.propertyName2 = 3; //and this will be logged every time propertyName2's value is tried to be set
console.log(obj.propertyName2); //this will be logged every time propertyName2 is accessed to get its value

Object.getOwnPropertyDescriptor #

Bir nesnedeki belirli bir özelliğin açıklamasını alın.

var sampleObject = {
    hello: 'world'
};

Object.getOwnPropertyDescriptor(sampleObject, 'hello');
// Object {value: "world", writable: true, enumerable: true, configurable: true}

Özel karakterler veya ayrılmış kelimeler içeren özellikler #

Nesne özelliği gösterimi genellikle ‘myObject.property’ olarak yazılırken, bu yalnızca normalde [JavaScript değişken adlarında] bulunan karakterlere izin verir(http://stackoverflow.com/questions/1661197/what-characters-are-valid- esas olarak harfler, sayılar ve alt çizgi (_) olan javascript-değişken-adları için).

Boşluk, ☺ veya kullanıcı tarafından sağlanan içerik gibi özel karakterlere ihtiyacınız varsa, bu [] köşeli ayraç gösterimi kullanılarak mümkündür.

myObject['special property ☺'] = 'it works!'
console.log(myObject['special property ☺'])

Tüm basamaklı özellikler: #

Özel karakterlere ek olarak, tamamı basamaklı özellik adları için parantez gösterimi gerekir. Ancak, bu durumda özelliğin bir dize olarak yazılmasına gerek yoktur.

myObject[123] = 'hi!' // number 123 is automatically converted to a string
console.log(myObject['123']) // notice how using string 123 produced the same result
console.log(myObject['12' + '3']) // string concatenation
console.log(myObject[120 + 3]) // arithmetic, still resulting in 123 and producing the same result
console.log(myObject[123.0]) // this works too because 123.0 evaluates to 123
console.log(myObject['123.0']) // this does NOT work, because '123' != '123.0'

Ancak, sekizli gösterim olarak yorumlandığı için baştaki sıfırlar önerilmez. (TODO, sekizli, onaltılı ve üslü gösterimi açıklayan bir örnek üretmeli ve bağlantı vermeliyiz)

Yinelenebilir bir nesne oluşturma #

var myIterableObject = {};
// An Iterable object must define a method located at the Symbol.iterator key:
myIterableObject[Symbol.iterator] = function () {
  // The iterator should return an Iterator object
  return {
    // The Iterator object must implement a method, next()
    next: function () {
      // next must itself return an IteratorResult object 
      if (!this.iterated) {
        this.iterated = true;
        // The IteratorResult object has two properties
        return {
          // whether the iteration is complete, and
          done: false,
          // the value of the current iteration
          value: 'One'
        };
      }
      return {
        // When iteration is complete, just the done property is needed
        done: true
      };
    },
    iterated: false
  };
};

for (var c of myIterableObject) {
  console.log(c); 
}

Nesne girişleri üzerinde yineleme - Object.entries() #

Önerilen Object.entries() yöntemi, verilen nesne için bir dizi anahtar/değer çifti döndürür. [Array.prototype.entries()](https://www.wikiod.com/tr/javascript/diziler#input() yöntemi) gibi bir yineleyici döndürmez, ancak ‘Object.entries()’ tarafından döndürülen dizi, ne olursa olsun yinelenebilir.

const obj = {
    one: 1,
    two: 2,
    three: 3
};

Object.entries(obj);

Sonuçlar:

[
    ["one", 1],
    ["two", 2],
    ["three", 3]
]

Bir nesnenin anahtar/değer çiftleri üzerinde yineleme yapmanın kullanışlı bir yoludur:

for(const [key, value] of Object.entries(obj)) {
    console.log(key); // "one", "two" and "three"
    console.log(value); // 1, 2 and 3
}

Nesne.değerler() #

‘Object.values()’ yöntemi, belirli bir nesnenin kendi numaralandırılabilir özellik değerlerinden oluşan bir diziyi, for…in döngüsü tarafından sağlananla aynı sırada döndürür (aradaki fark, bir for-in döngüsünün, prototip zinciri de).

var obj = { 0: 'a', 1: 'b', 2: 'c' };
console.log(Object.values(obj)); // ['a', 'b', 'c']
