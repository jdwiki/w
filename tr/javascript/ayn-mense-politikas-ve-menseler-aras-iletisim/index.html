Aynı Menşe Politikası ve Menşeler Arası İletişim

  • Mesajlarla güvenli çapraz kaynak iletişimi
  • Bir alt çerçeve ile iletişim kuran Pencere örneği
  • Aynı Köken Politikasını Atlatmanın Yolları

Bu sayfada

  • Mesajlarla güvenli çapraz kaynak iletişimi
  • Bir alt çerçeve ile iletişim kuran Pencere örneği
  • Aynı Köken Politikasını Atlatmanın Yolları

Same-Origin politikası, uzak adres komut dosyasının Origin‘iyle aynı değilse, komut dosyalarının uzak içeriğe erişmesini önlemek için web tarayıcıları tarafından kullanılır. Bu, kötü amaçlı komut dosyalarının hassas verileri elde etmek için diğer web sitelerine istekte bulunmasını önler.

Her iki URL de aynı protokol, hostname ve port‘a sahipse, iki adresin kökeni aynı kabul edilir.

Mesajlarla güvenli çapraz kaynak iletişimi #

‘window.postMessage()’ yöntemi, ilgili olay işleyicisi ‘window.onmessage’ ile birlikte, kaynaklar arası iletişimi etkinleştirmek için güvenle kullanılabilir.

Hedef “pencere"nin “postMessage()” yöntemi, başka bir “pencereye” mesaj göndermek için çağrılabilir; bu, “onmessage” olay işleyicisi ile onu durdurabilecek, detaylandırabilecek ve gerekirse gönderebilecektir. tekrar postMessage() kullanarak gönderen penceresine bir yanıt.

Bir alt çerçeve ile iletişim kuran Pencere örneği #

  • “http://main-site.com/index.html” içeriği:

      <!-- ... -->
  <iframe id="frame-id" src="http://other-site.com/index.html"></iframe>
  <script src="main_site_script.js"></script>
  <!-- ... -->

  • “http://other-site.com/index.html” içeriği:

      <!-- ... -->
  <script src="other_site_script.js"></src>
  <!-- ... -->

  • “main_site_script.js"nin içeriği:

      // Get the <iframe>'s window
  var frameWindow = document.getElementById('frame-id').contentWindow;

  // Add a listener for a response
  window.addEventListener('message', function(evt) {

      // IMPORTANT: Check the origin of the data! 
      if (event.origin.indexOf('http://other-site.com') == 0) {

          // Check the response
          console.log(evt.data);
          /* ... */
      }
  });        

  // Send a message to the frame's window
  frameWindow.postMessage(/* any obj or var */, '*');

  • “other_site_script.js"nin içeriği:

      window.addEventListener('message', function(evt) { 

      // IMPORTANT: Check the origin of the data! 
      if (event.origin.indexOf('http://main-site.com') == 0) {

          // Read and elaborate the received data
          console.log(evt.data);
          /* ... */

          // Send a response back to the main window
          window.parent.postMessage(/* any obj or var */, '*');
      }
  });

Aynı Köken Politikasını Atlatmanın Yolları #

İstemci tarafı JavaScript motorları (tarayıcı içinde çalışan) söz konusu olduğunda, mevcut alan dışındaki kaynaklardan içerik istemek için basit bir çözüm yoktur. (Bu arada, Node JS gibi JavaScript sunucu araçlarında bu sınırlama yoktur.)

Ancak, (bazı durumlarda) aşağıdaki yöntemleri kullanarak diğer kaynaklardan veri almak gerçekten de mümkündür. Lütfen, bazılarının üretim sisteminin dayanması gereken çözümler yerine hack’ler veya geçici çözümler sunabileceğini unutmayın.

Yöntem 1: CORS #

Günümüzde çoğu genel API, geliştiricilerin CORS (Çapraz Kaynaklı Kaynak Paylaşımı) adlı bir özelliği etkinleştirerek istemci ve sunucu arasında iki yönlü veri göndermesine olanak tanır. Tarayıcı, belirli bir HTTP başlığının (‘Erişim-Kontrol-İzin Ver-Köken’) ayarlanıp ayarlanmadığını ve istekte bulunan sitenin etki alanının başlık değerinde listelenip listelenmediğini kontrol edecektir. Eğer öyleyse, tarayıcı AJAX bağlantılarının kurulmasına izin verecektir.

Ancak, geliştiriciler diğer sunucuların yanıt başlıklarını değiştiremediğinden, bu yönteme her zaman güvenilemez.

Yöntem 2: JSONP #

P eklenmiş JSON genellikle bir geçici çözüm olarak suçlanır. En basit yöntem değil, ama yine de işi hallediyor. Bu yöntem, komut dosyalarının herhangi bir etki alanından yüklenebilmesi gerçeğinden yararlanır. Yine de, harici kaynaklardan JavaScript kodu istemenin her zaman potansiyel bir güvenlik riski olduğunu ve daha iyi bir çözüm varsa bu genellikle kaçınılması gerektiğini belirtmek çok önemlidir.

JSONP kullanılarak istenen veriler tipik olarak JSON şeklindedir; bu, JavaScript’te nesne tanımı için kullanılan sözdizimine uygundur ve bu taşıma yöntemini yapar. Çok basit. Web sitelerinin JSONP aracılığıyla elde edilen harici verileri kullanmasına izin vermenin yaygın bir yolu, bunları URL’deki bir “GET” parametresi aracılığıyla ayarlanan bir geri arama işlevinin içine yerleştirmektir. Harici komut dosyası yüklendiğinde, işlev ilk parametresi olarak verilerle birlikte çağrılır.

<script>
function myfunc(obj){
    console.log(obj.example_field);
}
</script>
<script src="http://example.com/api/endpoint.js?callback=myfunc"></script>

“http://example.com/api/endpoint.js?callback=myfunc” içeriği şöyle görünebilir:

myfunc({"example_field":true})

İşlev her zaman önce tanımlanmalıdır, aksi takdirde harici komut dosyası yüklendiğinde tanımlanmaz.
