Bazı diller, ne tür bir değişken bildirdiğinizi önceden tanımlamanızı gerektirir. JavaScript bunu yapmaz; bunu kendi başına çözmeye çalışacaktır. Bazen bu beklenmedik davranışlara neden olabilir.

Aşağıdaki HTML’yi kullanırsak

<span id="freezing-point">0</span>

Ve içeriğini JS aracılığıyla alın, beklense bile, onu bir sayıya dönüştürmez. Aşağıdaki parçacığı kullanırsak, “boilingPoint"in “100” olması beklenebilir. Ancak, JavaScript ‘moreHeat’i bir dizgeye dönüştürür ve iki dizgiyi birleştirir; sonuç “0100” olacaktır.

var el = document.getElementById('freezing-point');
var freezingPoint = el.textContent || el.innerText;
var moreHeat = 100;
var boilingPoint = freezingPoint + moreHeat;

Bunu, “freezingPoint"i açıkça bir sayıya çevirerek düzeltebiliriz.

var el = document.getElementById('freezing-point');
var freezingPoint = Number(el.textContent || el.innerText);
var boilingPoint = freezingPoint + moreHeat;

İlk satırda, kaydetmeden önce “0”‘ı (dize) ‘0’a (sayı) dönüştürüyoruz. Eklemeyi yaptıktan sonra beklenen sonucu (100) alırsınız.

Çift Olumsuzlama (!!x) #

Çift olumsuzlama `!!’, ayrı bir JavaScript operatörü veya özel bir sözdizimi değil, yalnızca iki olumsuzlama dizisidir. Herhangi bir türün değerini, truthy veya falsy olmasına bağlı olarak uygun ’true’ veya ‘false’ Boole değerine dönüştürmek için kullanılır.

!!1            // true
!!0            // false
!!undefined    // false
!!{}           // true
!![]           // true

İlk olumsuzlama, truthy ise herhangi bir değeri ‘false’a, falsy ise ’true’ya dönüştürür. İkinci olumsuzlama daha sonra normal bir Boole değeri üzerinde çalışır. Birlikte herhangi bir truthy değerini ’true’ya ve herhangi bir falsy değerini ‘false’a dönüştürürler.

Bununla birlikte, birçok profesyonel, bu tür sözdizimi kullanma pratiğini kabul edilemez olarak değerlendirir ve yazması daha uzun olsalar bile, okunması daha basit alternatifler önerir:

x !== 0        // instead of !!x in case x is a number
x != null      // instead of !!x in case x is an object, a string, or an undefined

‘!!x’ kullanımı, aşağıdaki nedenlerden dolayı kötü bir uygulama olarak kabul edilir:

  1. Biçimsel olarak farklı bir özel sözdizimi gibi görünebilir, oysa aslında örtük tip dönüşümü ile iki ardışık olumsuzlamadan başka bir şey yapmıyor.
  2. Değişkenlerde ve özelliklerde saklanan değer türleri hakkında kod aracılığıyla bilgi vermek daha iyidir. Örneğin, ‘x !== 0’, ‘x’in muhtemelen bir sayı olduğunu söylerken, ‘!!x’ kodun okuyucularına böyle bir avantaj sağlamaz.
  3. ‘Boolean(x)’ kullanımı benzer işlevselliğe izin verir ve daha açık bir tür dönüşümüdür.

Örtülü dönüştürme #

JavaScript, kullanım sırasında değişkenleri otomatik olarak daha uygun türlere dönüştürmeye çalışacaktır. Genellikle dönüşümleri açıkça yapmanız önerilir (diğer örneklere bakın), ancak yine de hangi dönüşümlerin dolaylı olarak gerçekleştiğini bilmeye değer.

"1" + 5 === "15" // 5 got converted to string.
1 + "5" === "15" // 1 got converted to string.
1 - "5" === -4 // "5" got converted to a number.
alert({}) // alerts "[object Object]", {} got converted to string.
!0 === true // 0 got converted to boolean
if ("hello") {} // runs, "hello" got converted to boolean.
new Array(3) === ",,"; // Return true. The array is converted to string - Array.toString();

Daha zor kısımlardan bazıları:

!"0" === false // "0" got converted to true, then reversed.
!"false" === false // "false" converted to true, then reversed.

Boole değerine dönüştürme #

“Boolean(…)” herhangi bir veri türünü “doğru” veya “yanlış"a dönüştürür.

Boolean("true") === true
Boolean("false") === true
Boolean(-1) === true
Boolean(1) === true
Boolean(0) === false
Boolean("") === false
Boolean("1") === true
Boolean("0") === true
Boolean({}) === true
Boolean([]) === true

Boş dizeler ve 0 sayısı false’a dönüştürülür ve diğerleri true’ya dönüştürülür.

Daha kısa, ancak daha az net bir form:

!!"true" === true
!!"false" === true
!!-1 === true
!!1 === true
!!0 === false
!!"" === false
!!"1" === true
!!"0" === true
!!{} === true
!![] === true

Bu daha kısa biçim, https://www.wikiod.com/tr/javascript/karslastrma-islemleri#Çift Olumsuzlama (!!x) içinde açıklandığı gibi mantıksal NOT operatörünü iki kez kullanarak örtük tür dönüştürmeden yararlanır.

ECMAScript özelliğinden boole dönüşümlerinin tam listesi burada

  • “myArg” türü “undefined” veya “null” ise ‘Boolean(myArg) === yanlış’
  • “myArg”, “boolean” türündeyse, “Boolean(myArg) === myArg’ olur
  • “myArg”, “number” türündeyse, “myArg” “+0”, “‑0” veya “NaN” ise “Boolean(myArg) === false”; aksi halde ‘doğru’
  • “myArg”, “string” türündeyse, “myArg” boş Dize ise “Boolean(myArg) === false” olur (uzunluğu sıfırdır); aksi halde ‘doğru’
  • “myArg”, “symbol” veya “object” türündeyse, “Boolean(myArg) === true”

Boolean olarak “yanlış"a dönüştürülen değerlere falsy (ve diğerlerinin tümüne truthy adı verilir) denir. https://www.wikiod.com/tr/javascript/karslastrma-islemleri bakın.

Bir dizeyi sayıya dönüştürme #

Number('0') === 0

Number('0'), ('0') dizesini bir sayıya (0) dönüştürür

Daha kısa, ancak daha az net bir form:

+'0' === 0

Tekli ‘+’ operatörü sayılara hiçbir şey yapmaz, ancak başka bir şeyi sayıya dönüştürür. İlginç bir şekilde, +(-12) === -12.

parseInt('0', 10) === 0

‘parseInt(‘0’, 10)’ dizgiyi (‘0’’) bir sayıya (‘0’) dönüştürür, ikinci argüman olan radix’i unutmayın. Verilmezse, parseInt dizeyi yanlış sayıya dönüştürebilir.

Sayıyı dizeye dönüştürme #

String(0) === '0'

‘String(0)’, sayıyı (‘0’) bir dizgeye (‘‘0’’) dönüştürür.

Daha kısa, ancak daha az net bir form:

'' + 0 === '0'

Bir sayıyı boole dönüştürme #

Boolean(0) === false

“Boolean(0)”, “0” sayısını bir boolean “yanlış"a dönüştürür.

Daha kısa, ancak daha az net bir form:

!!0 === false

Bir dizeyi boolea dönüştürme #

Bir dizeyi boole kullanımına dönüştürmek için

Boolean(myString)

veya daha kısa ama daha az net form

!!myString

Boş dize (sıfır uzunlukta) dışındaki tüm dizeler, boolean olarak “true” olarak değerlendirilir.

Boolean('') === false   // is true
Boolean("") === false   // is true
Boolean('0') === false  // is false
Boolean('any_nonempty_string') === true // is true

Kayan Tamsayı #

Tamsayıya Kaydır #

Bir kayan noktayı bir tamsayıya dönüştürmek için JavaScript birden çok yöntem sağlar.

‘floor’ işlevi, kayan nokta değerinden küçük veya ona eşit olan ilk tamsayıyı döndürür.

Math.floor(5.7); // 5

‘ceil’ işlevi, kayan nokta değerinden büyük veya ona eşit olan ilk tamsayıyı döndürür.

Math.ceil(5.3); // 6

‘yuvarlak’ işlevi, şamandırayı yuvarlar.

Math.round(3.2); // 3
Math.round(3.6); // 4

Kesme (trunc), kayan noktadaki ondalık sayıları kaldırır.

Math.trunc(3.7); // 3

Kesme (“trunc”) ve “zemin” arasındaki farka dikkat edin:

Math.floor(-3.1); // -4
Math.trunc(-3.1); // -3
son sürüm

Dizeyi float’a dönüştür #

“parseFloat”, bir dizeyi bir değişkene dönüştürdüğü bir argüman olarak kabul eder/

parseFloat("10.01") // = 10.01

Bir diziyi bir dizgeye dönüştürün #

‘Array.join(separator)’, yapılandırılabilir bir ayırıcıyla bir diziyi dize olarak çıkarmak için kullanılabilir.

Varsayılan (ayırıcı = “,”):

["a", "b", "c"].join() === "a,b,c"

Bir dize ayırıcı ile:

[1, 2, 3, 4].join(" + ") === "1 + 2 + 3 + 4"

Boş bir ayırıcı ile:

["B", "o", "b"].join("") === "Bob"

Dizi yöntemlerini kullanarak diziden dizeye #

var  arr = ['a', 'á', 'b', 'c']

function upper_lower (a, b, i) {
  //...do something here
  b = i & 1 ? b.toUpperCase() : b.toLowerCase();
  return a + ',' + b
}
arr = arr.reduce(upper_lower); // "a,Á,b,C"
