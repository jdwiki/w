Kapsam, değişkenlerin yaşadığı ve aynı kapsamdaki diğer kodlar tarafından erişilebildiği bağlamdır. JavaScript büyük ölçüde işlevsel bir programlama dili olarak kullanılabildiğinden, çalışma zamanında hataları ve beklenmeyen davranışları önlemeye yardımcı olduğundan, değişkenlerin ve işlevlerin kapsamını bilmek önemlidir.

Kapanışlar #

Bir işlev bildirildiğinde, bildirimi bağlamındaki değişkenler kapsamında yakalanır. Örneğin, aşağıdaki kodda, “x” değişkeni dış kapsamdaki bir değere bağlıdır ve ardından “x"e yapılan başvuru “bar” bağlamında yakalanır:

var x = 4; // declaration in outer scope

function bar() {
    console.log(x); // outer scope is captured on declaration
}

bar(); // prints 4 to console

Örnek çıktı: “4”

Bu kapsam “yakalama” kavramı ilginçtir çünkü dış kapsam çıktıktan sonra bile bir dış kapsamdaki değişkenleri kullanabilir ve değiştirebiliriz. Örneğin, aşağıdakileri göz önünde bulundurun:

function foo() {
    var x = 4; // declaration in outer scope

    function bar() {
        console.log(x); // outer scope is captured on declaration
    }

    return bar;
    
    // x goes out of scope after foo returns
}

var barWithX = foo();
barWithX(); // we can still access x

Örnek çıktı: “4”

Yukarıdaki örnekte, “foo” çağrıldığında, bağlamı “bar” işlevinde yakalanır. Böylece, geri döndükten sonra bile, “bar”, “x” değişkenine erişmeye ve onu değiştirmeye devam edebilir. Bağlamı başka bir işlevde yakalanan ‘foo’ işlevine bir kapatma olduğu söylenir.

Özel veriler #

Bu, yalnızca belirli bir işlev veya işlevler kümesi tarafından görülebilen “özel” değişkenleri tanımlamak gibi bazı ilginç şeyler yapmamızı sağlar. Yapmacık (ama popüler) bir örnek:

function makeCounter() {
    var counter = 0;

    return {
        value: function () {
            return counter;
        },
        increment: function () {
            counter++;
        }
    };
}

var a = makeCounter();
var b = makeCounter();

a.increment();

console.log(a.value());
console.log(b.value());

Örnek çıktı:

1
0

‘makeCounter()’ çağrıldığında, o işlevin bağlamının bir anlık görüntüsü kaydedilir. ‘makeCounter()’ içindeki tüm kodlar, yürütmelerinde bu anlık görüntüyü kullanır. İki “makeCounter()” çağrısı, kendi “counter” kopyalarıyla iki farklı anlık görüntü oluşturacaktır.

Hemen çağrılan işlev ifadeleri (IIFE) #

Kapanışlar ayrıca, genellikle hemen çağrılan işlev ifadelerinin kullanılması yoluyla genel ad alanı kirliliğini önlemek için de kullanılır.

Hemen çağrılan işlev ifadeleri (veya belki daha sezgisel olarak, kendi kendini yürüten anonim işlevler) esasen bildirimden hemen sonra çağrılan kapanışlardır. IIFE’lerle ilgili genel fikir, yalnızca IIFE içindeki kod tarafından erişilebilen ayrı bir bağlam oluşturmanın yan etkisini çağırmaktır.

“$” ile “jQuery"ye başvurabilmek istediğimizi varsayalım. IIFE kullanmadan saf yöntemi düşünün:

var $ = jQuery;
// we've just polluted the global namespace by assigning window.$ to jQuery

Aşağıdaki örnekte, “$” öğesinin yalnızca kapatma tarafından oluşturulan bağlamda “jQuery"ye bağlanmasını sağlamak için bir IIFE kullanılır:

(function ($) {
    // $ is assigned to jQuery here
})(jQuery);
// but window.$ binding doesn't exist, so no pollution

Kapanışlarla ilgili daha fazla bilgi için Stackoverflow’daki standart yanıta bakın.

Kaldırma #

Kaldırma nedir? #

Kaldırma, tüm değişken ve işlev bildirimlerini kapsamlarının en üstüne taşıyan bir mekanizmadır. Ancak, değişken atamaları hala orijinal oldukları yerde gerçekleşir.

Örneğin, aşağıdaki kodu göz önünde bulundurun:

console.log(foo);  // → undefined
var foo = 42;
console.log(foo);  // → 42

Yukarıdaki kod şununla aynıdır:

var foo;             // → Hoisted variable declaration
console.log(foo);    // → undefined
foo = 42;            // → variable assignment remains in the same place
console.log(foo);    // → 42

Kaldırma nedeniyle yukarıdaki “tanımsız"ın, çalıştırmadan kaynaklanan “tanımlanmamış” ile aynı olmadığına dikkat edin:

console.log(foo);    // → foo is not defined

Benzer bir ilke fonksiyonlar için de geçerlidir. İşlevler bir değişkene atandığında (yani bir işlev ifadesi), atama aynı yerde kalırken değişken bildirimi kaldırılır. Aşağıdaki iki kod parçacığı eşdeğerdir.

console.log(foo(2, 3));     // → foo is not a function

var foo = function(a, b) {
    return a * b;
}

var foo;
console.log(foo(2, 3));     // → foo is not a function
foo = function(a, b) {
    return a * b;
}

İşlev deyimleri bildirilirken farklı bir senaryo oluşur. İşlev deyimlerinden farklı olarak, işlev bildirimleri kapsamlarının en üstüne çıkarılır. Aşağıdaki kodu göz önünde bulundurun:

console.log(foo(2, 3));  // → 6
function foo(a, b) {
    return a * b;
}

Yukarıdaki kod, kaldırma nedeniyle bir sonraki kod parçacığıyla aynıdır:

function foo(a, b) {
    return a * b;
}

console.log(foo(2, 3));  // → 6

İşte kaldırmanın ne olduğu ve ne olmadığına dair bazı örnekler:

// Valid code:
foo();

function foo() {}

// Invalid code:
bar();                     // → TypeError: bar is not a function
var bar = function () {};


// Valid code:
foo();
function foo() {
    bar();
}
function bar() {}


// Invalid code:
foo();
function foo() {
    bar();                // → TypeError: bar is not a function
}
var bar = function () {};


// (E) valid:
function foo() {
    bar();
}
var bar = function(){};
foo();

Kaldırma Sınırlamaları #

Bir değişkenin başlatılması kaldırılamaz veya basit JavaScript Kaldırma bildirimlerinde başlatılamaz.

Örneğin: Aşağıdaki betikler farklı çıktılar verecektir.

var x = 2; 
var y = 4; 
alert(x + y);

Bu size 6 çıktı verecektir. Ama bu…

var x = 2; 
alert(x + y);
var y = 4;

Bu size bir NaN çıktısı verecektir. y değerini başlattığımız için JavaScript Kaldırma gerçekleşmiyor, bu nedenle y değeri tanımsız olacak. JavaScript, y’nin henüz bildirilmediğini dikkate alacaktır.

Yani ikinci örnek aşağıdaki ile aynıdır.

var x = 2; 
var y;
alert(x + y);
y = 4;

Bu size bir NaN çıktısı verecektir.

var ve let arasındaki fark #

(Not: ’let’ kullanan tüm örnekler aynı zamanda ‘const’ için de geçerlidir)

“var” JavaScript’in tüm sürümlerinde bulunurken “let” ve “const” ECMAScript 6’nın bir parçasıdır ve yalnızca bazı yeni tarayıcılarda mevcuttur.

“var”, ne zaman bildirildiğine bağlı olarak, içeren işlevin veya genel alanın kapsamına girer:

var x = 4; // global scope

function DoThings() {
    var x = 7; // function scope
    console.log(x);
}

console.log(x); // >> 4
DoThings();     // >> 7
console.log(x); // >> 4

Bu, “if” ifadelerinden ve tüm benzer blok yapılarından “kaçtığı” anlamına gelir:

var x = 4;
if (true) {
    var x = 7;
}
console.log(x); // >> 7

for (var i = 0; i < 4; i++) {
    var j = 10;
}
console.log(i); // >> 4
console.log(j); // >> 10

Karşılaştırıldığında, “let” blok kapsamlıdır:

let x = 4;

if (true) {
    let x = 7;
    console.log(x); // >> 7
}

console.log(x); // >> 4

for (let i = 0; i < 4; i++) {
    let j = 10;
}
console.log(i); // >> "ReferenceError: i is not defined"
console.log(j); // >> "ReferenceError: j is not defined"

“i” ve “j"nin yalnızca “for” döngüsünde bildirildiğini ve bu nedenle bunun dışında bildirilmediğini unutmayın.

Birkaç önemli farklılık daha var:

Genel değişken bildirimi #

Üst kapsamda (herhangi bir fonksiyon ve bloğun dışında), ‘var’ bildirimleri global nesneye bir öğe koyar. ’let’ şunları yapmaz:

var x = 4;
let y = 7;

console.log(this.x); // >> 4
console.log(this.y); // >> undefined

Yeniden beyan #

Bir değişkeni “var” kullanarak iki kez bildirmek bir hata üretmez (bir kez bildirmeye eşdeğer olsa da):

var x = 4;
var x = 7;

’let’ ile bu bir hata üretir:

let x = 4;
let x = 7;

TypeError: “x” tanımlayıcısı zaten bildirildi

Aynısı, “y”, “var” ile bildirildiğinde de geçerlidir:

var y = 4;
let y = 7;

TypeError: “y” tanımlayıcısı zaten bildirildi

Ancak let ile bildirilen değişkenler, iç içe geçmiş bir blokta yeniden kullanılabilir (yeniden bildirilemez).

let i = 5;    
{
   let i = 6;
   console.log(i); // >> 6
}
console.log(i); // >> 5

Blok içinde dış “i"ye erişilebilir, ancak iç blok “i” için bir “let” bildirimine sahipse, dış “i"ye erişilemez ve ikinciden önce kullanılırsa bir “ReferenceError” verir. beyan.

let i = 5;
{
    i = 6;  // outer i is unavailable within the Temporal Dead Zone
    let i;
}

ReferenceError: i tanımlı değil

Kaldırma #

Hem ‘var’ hem de ’let’ ile bildirilen değişkenler kaldırılır. Aradaki fark, “var” ile bildirilen bir değişkene, otomatik olarak atandığından (değeri “undefined” olan) kendi atamasından önce başvurulabilmesidir, ancak “let” yapılamaz, özellikle değişkenin olmadan önce bildirilmesini gerektirir. çağrılır:

console.log(x); // >> undefined
console.log(y); // >> "ReferenceError: `y` is not defined"
//OR >> "ReferenceError: can't access lexical declaration `y` before initialization"
var x = 4;
let y = 7;

Bir bloğun başlangıcı ile “let” veya “const” bildirimi arasındaki alan [Geçici Ölü Bölge](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/ olarak bilinir) İfadeler/let#Temporal_dead_zone_and_errors_with_let) ve bu alandaki değişkene yapılan herhangi bir başvuru bir ‘ReferenceError’a neden olacaktır. Bu, değişken bildirilmeden önce atanmış olsa bile gerçekleşir:

y=7; // >> "ReferenceError: `y` is not defined"
let y;

Kesin olmayan modda, herhangi bir bildirim olmadan bir değişkene değer atamak, değişkeni global kapsamda otomatik olarak bildirir. Bu durumda, ‘y’nin global kapsamda otomatik olarak bildirilmesi yerine, ’let’ değişkenin adını (‘y’) saklar ve bildirildiği/başlatıldığı satırdan önce herhangi bir erişime veya atamaya izin vermez.

3: https://www.wikiod.com/tr/javascript/kat-mod#Genel özelliklerde yapılan değişiklikler

Uygula ve Çağrı sözdizimi ve çağırma. #

Her işlevdeki “apply” ve “call” yöntemleri, “bu” için özel bir değer sağlamasına olanak tanır.

function print() {
    console.log(this.toPrint);
}

print.apply({ toPrint: "Foo" }); // >> "Foo"
print.call({ toPrint: "Foo" }); // >> "Foo"

Yukarıda kullanılan her iki çağrının sözdiziminin aynı olduğunu fark edebilirsiniz. yani imza benzer görünüyor.

Ancak kullanımlarında küçük bir fark vardır, çünkü işlevlerle uğraştığımız ve kapsamlarını değiştirdiğimiz için, yine de işleve iletilen orijinal argümanları korumamız gerekir. Hem “apply” hem de “call”, hedef işleve argümanları aşağıdaki gibi iletmeyi destekler:

function speak() {
    var sentences = Array.prototype.slice.call(arguments);
    console.log(this.name+": "+sentences);
}
var person = { name: "Sunny" };
speak.apply(person, ["I", "Code", "Startups"]); // >> "Sunny: I Code Startups"
speak.call(person, "I", "<3", "Javascript"); // >> "Sunny: I <3 Javascript"

“Uygula"nın, argüman listesi olarak bir “Array” veya “arguments” nesnesini (dizi benzeri) iletmenize izin verdiğine dikkat edin, oysa “call”, her bir argümanı ayrı ayrı iletmenizi gerektirir.

Bu iki yöntem, her zaman orijinal bir işlevden bir nesne yöntemi olarak çağrılacak bir işlev oluşturmak için ECMAScript’in yerel ‘bağlama’sının zayıf bir sürümünü uygulamak gibi istediğiniz kadar süslü olma özgürlüğü verir.

function bind (func, obj) { 
    return function () {
        return func.apply(obj, Array.prototype.slice.call(arguments, 1));
    }
}

var obj = { name: "Foo" };

function print() {
    console.log(this.name);
}

printObj = bind(print, obj);

printObj();

Bu günlüğe kaydeder

“Foo”

‘bağlama’ işlevinde çok şey oluyor

  1. “bu"nun değeri olarak “obj” kullanılacaktır.
  2. argümanları fonksiyona iletin
  3. ve ardından değeri döndürün

Ok işlevi çağırma #

Ok işlevlerini kullanırken “this”, ekteki yürütme bağlamının “this"inden değeri alır (yani, ok işlevlerinde “bu”, olağan dinamik kapsamdan ziyade sözcüksel kapsama sahiptir). Global kodda (herhangi bir fonksiyona ait olmayan kod) global nesne olacaktır. Ve burada açıklanan diğer yöntemlerden herhangi birinden ok gösterimi ile bildirilen işlevi çağırsanız bile bu şekilde kalır.

var globalThis = this; //"window" in a browser, or "global" in Node.js

var foo = (() => this);           

console.log(foo() === globalThis);          //true

var obj = { name: "Foo" };
console.log(foo.call(obj) === globalThis);  //true

Yöntemin çağrıldığı nesneye atıfta bulunmak yerine ‘bunun’ bağlamı nasıl devraldığını görün.

var globalThis = this;

var obj = {
    withoutArrow: function() {
        return this;
    },
    withArrow: () => this
};

console.log(obj.withoutArrow() === obj);      //true
console.log(obj.withArrow() === globalThis);  //true

var fn = obj.withoutArrow; //no longer calling withoutArrow as a method
var fn2 = obj.withArrow;
console.log(fn() === globalThis);             //true
console.log(fn2() === globalThis);            //true
son sürüm

Bağlı çağrı #

Her işlevin ‘bağlama’ yöntemi, bağlam belirli bir nesneye sıkı sıkıya bağlı olarak bu işlevin yeni bir sürümünü oluşturmanıza olanak tanır. Bir işlevi bir nesnenin yöntemi olarak çağrılmaya zorlamak özellikle yararlıdır.

var obj = { foo: 'bar' };

function foo() {
    return this.foo;
}

fooObj = foo.bind(obj);

fooObj();

Bu günlüğe kaydeder:

çubuk

Yöntem çağırma #

Bir nesnenin yöntemi olarak bir işlevi çağırmak, “bu"nun değeri o nesne olacaktır.

var obj = {
    name: "Foo",
    print: function () {
        console.log(this.name)
    }
}

Artık bir obj yöntemi olarak print’i çağırabiliriz. bu obj olacak

obj.print();

Bu, böylece günlüğe kaydeder:

Foo

Anonim çağrı #

Anonim bir işlev olarak bir işlev çağrıldığında, “bu” global nesne olacaktır (tarayıcıda “self”).

function func() {
    return this;
}

func() === window; // true

ECMAScript 5’in katı kipinde, işlev anonim olarak çağrılırsa “bu” “tanımsız” olacaktır.

(function () {
    "use strict";
    func();
}())

Bu çıktı

tanımsız

son sürüm

Oluşturucu çağırma #

’new’ anahtar kelimesiyle yapıcı olarak bir işlev çağrıldığında, ’this’, oluşturulan nesnenin değerini alır.

function Obj(name) {
    this.name = name;
}

var obj = new Obj("Foo");

console.log(obj);

Bu günlüğe kaydeder

{isim: “Foo” }

var yerine izin verme döngülerini kullanma (tıklama işleyicileri örneği) #

Diyelim ki ’loadedData’ dizisinin her bir parçası için bir düğme eklememiz gerekiyor (örneğin, her düğme verileri gösteren bir kaydırıcı olmalıdır; basitlik adına, sadece bir mesajı uyaracağız). Biri şöyle bir şey deneyebilir:

for(var i = 0; i < loadedData.length; i++)
    jQuery("#container").append("<a class='button'>"+loadedData[i].label+"</a>")
        .children().last() // now let's attach a handler to the button which is a child
        .on("click",function() { alert(loadedData[i].content); });

Ancak uyarmak yerine, her düğme,

TypeError: loadData[i] tanımsız

hata. Bunun nedeni, “i” kapsamının genel kapsam (veya bir işlev kapsamı) olması ve döngüden sonra “i == 3” olmasıdır. İhtiyacımız olan “i"nin durumunu hatırlamak değil. Bu, ’let’ kullanılarak yapılabilir:

for(let i = 0; i < loadedData.length; i++)
    jQuery("#container").append("<a class='button'>"+loadedData[i].label+"</a>")
        .children().last() // now let's attach a handler to the button which is a child
        .on("click",function() { alert(loadedData[i].content); });

Bu kodla test edilecek bir “loadedData” örneği:

    var loadedData = [
        { label:"apple",      content:"green and round" },
        { label:"blackberry", content:"small black or blue" },
        { label:"pineapple",  content:"weird stuff.. difficult to explain the shape" }
    ];
