Ok işlevleri, [ECMAScript 2015 (ES6)] içinde sözlüksel olarak kapsamlı işlevler olan [anonim](https://www.wikiod.com/tr/javascript/fonksiyonlar#Anonim İşlev) yazmanın kısa bir yoludur. (https://developer.mozilla.org/en-US/docs/Web/JavaScript/New_in_JavaScript/ECMAScript_2015_support_in_Mozilla).

Sözdizimi #

  • x => y // Örtük dönüş
  • x => { dönüş y } // Açık dönüş
  • (x, y, z) => { … } // Birden çok argüman
  • zaman uyumsuz () => { … } // Zaman uyumsuz ok işlevleri
  • (() => { … })() // Hemen çağrılan fonksiyon ifadesi
  • const myFunc = x

=> x*2 // Okun önündeki satır sonu ‘Beklenmeyen belirteç’ hatası verir

  • const myFunc = x =>

x*2 // Oktan sonraki satır sonu geçerli bir sözdizimidir

JavaScript’teki işlevler hakkında daha fazla bilgi için lütfen İşlevler belgelerine bakın.

Ok işlevleri, ECMAScript 6 spesifikasyonunun bir parçasıdır, bu nedenle tarayıcı desteği sınırlı olabilir. Aşağıdaki tablo, ok işlevlerini destekleyen en eski tarayıcı sürümlerini göstermektedir.

Giriiş #

JavaScript’te işlevler, Common Lisp benzerlikleri nedeniyle bazen lambda ifadesi olarak adlandırılan “ok” (=>) sözdizimi kullanılarak anonim olarak tanımlanabilir.

Bir ok işlevinin en basit biçiminin argümanları => öğesinin sol tarafında ve dönüş değeri sağ tarafında bulunur:

item => item + 1 // -> function(item){return item + 1}

Bu işlev, ifadeye bir argüman sağlanarak hemen çağrılabilir:

(item => item + 1)(41) // -> 42

Bir ok işlevi tek bir parametre alıyorsa, bu parametrenin etrafındaki parantezler isteğe bağlıdır. Örneğin, aşağıdaki ifadeler [sabit değişkenler][3]’e aynı türde işlev atar:

const foo = bar => bar + 1;
const bar = (baz) => baz + 1;

Ancak, ok işlevi parametre almıyorsa veya birden fazla parametre alıyorsa, yeni bir parantez kümesi gerekirse tüm bağımsız değişkenleri kapsar:

(() => "foo")() // -> "foo"

((bow, arrow) => bow + arrow)('I took an arrow ', 'to the knee...')
// -> "I took an arrow to the knee..."

İşlev gövdesi tek bir ifadeden oluşmuyorsa, parantez içine alınmalı ve sonuç sağlamak için açık bir “dönüş” ifadesi kullanılmalıdır:

(bar => {
  const baz = 41;
  return bar + baz;
})(1); // -> 42

Ok işlevinin gövdesi yalnızca bir nesne değişmezinden oluşuyorsa, bu nesne değişmezi parantez içine alınmalıdır:

(bar => ({ baz: 1 }))(); // -> Object {baz: 1}

Fazladan parantezler, açma ve kapama parantezlerinin nesne değişmezinin parçası olduğunu, yani işlev gövdesinin sınırlayıcıları olmadığını belirtir.

Sözlüksel Kapsam Belirleme ve Bağlama (“bu"nun Değeri) #

Ok işlevleri sözcüksel olarak kapsamlandırılmıştır; bu, onların “bu” Bağlamalarının çevreleyen kapsamın bağlamına bağlı olduğu anlamına gelir. Başka bir deyişle, ‘bu’ ne ifade ediyorsa, bir ok işlevi kullanılarak korunabilir.

Aşağıdaki örneğe bir göz atın. ‘İnek’ sınıfının, 1 saniye sonra çıkardığı sesi yazdırmasına izin veren bir yöntemi vardır.

class Cow {

  constructor() {
    this.sound = "moo";
  }

  makeSoundLater() {
    setTimeout(() => console.log(this.sound), 1000);
  }
}

const betsy = new Cow();

betsy.makeSoundLater();

“makeSoundLater()” yönteminde, “bu” bağlamı, “İnek” nesnesinin geçerli örneğini ifade eder, bu nedenle “betsy.makeSoundLater()” adını verdiğim durumda, “bu” bağlamı, “betsy” anlamına gelir. `.

Ok işlevini kullanarak, ‘bu’ bağlamını koruuyorum, böylece yazdırma zamanı geldiğinde ’this.sound’a başvurabilirim, bu da düzgün bir şekilde “moo” yazdıracaktır.

Ok işlevi yerine normal bir işlev kullanmış olsaydınız, sınıf içinde olma bağlamını kaybedersiniz ve ‘ses’ özelliğine doğrudan erişemezsiniz.

Bağımsız Değişken Nesnesi #

Ok işlevleri bir argüman nesnesini göstermez; bu nedenle, “argümanlar” yalnızca geçerli kapsamdaki bir değişkene atıfta bulunur.

const arguments = [true];
const foo = x => console.log(arguments[0]);

foo(false); // -> true

Bu nedenle, ok işlevleri de arayanın/arayanın farkında değildir.

Bir argüman nesnesinin olmaması bazı uç durumlarda bir sınırlama olabilirken, dinlenme parametreleri genellikle uygun bir alternatiftir.

const arguments = [true];
const foo = (...arguments) => console.log(arguments[0]);

foo(false); // -> false

Örtülü Dönüş #

Ok işlevleri, gövdeleri yalnızca tek bir ifade içeriyorsa, bir işlevin gövdesini geleneksel olarak saran küme parantezlerini atlayarak örtük olarak değerler döndürebilir.

const foo = x => x + 1;
foo(1); // -> 2

Örtük dönüşler kullanılırken, küme parantezlerinin işlevin gövdesinin açılmasıyla karıştırılmaması için nesne değişmezleri parantez içine alınmalıdır.

const foo = () => { bar: 1 } // foo() returns undefined
const foo = () => ({ bar: 1 }) // foo() returns {bar: 1}

Ok yapıcı olarak işlev görür #

Ok işlevleri, ’new’ anahtar sözcüğüyle kullanıldığında bir ‘TypeError’ verir.

const foo = function () {
  return 'foo';
}

const a = new foo();

const bar = () => {
  return 'bar';
}

const b = new bar(); // -> Uncaught TypeError: bar is not a constructor...

Açık İade #

Ok işlevleri, ‘return’ anahtar sözcüğünü kullanarak onlardan açıkça bir değer döndürebilmeniz açısından klasik işlevler işlevine çok benzer şekilde davranabilir; işlevinizin gövdesini küme parantezlerine sarın ve bir değer döndürün:

const foo = x => {
  return x + 1;
}

foo(1); // -> 2
