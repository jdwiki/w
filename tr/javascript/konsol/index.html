Bir tarayıcının hata ayıklama konsolu veya web konsolu genellikle geliştiriciler tarafından hataları belirlemek, yürütme akışını anlamak, verileri günlüğe kaydetmek ve çalışma zamanında birçok başka amaç için kullanılır. Bu bilgilere console nesnesi aracılığıyla erişilir.

Sözdizimi #

  • void console.log(obj1 [, obj2, …, objN]);
  • void console.log(msg [, sub1, …, subN]);

Parametreler #

parametre Açıklama
obj1 … objN Dizi gösterimleri konsolda gösterilen JavaScript nesnelerinin listesi
mesaj Sıfır veya daha fazla ikame dizesi içeren bir JavaScript dizesi.
alt1 … altN msg içindeki ikame dizelerinin değiştirileceği JavaScript nesneleri.

Bir hata ayıklama/web konsolu tarafından görüntülenen bilgiler, ‘console.dir(console)’ aracılığıyla başvurulabilen birden çok ‘konsol’ Javascript nesnesinin yöntemi aracılığıyla kullanılabilir hale getirilir. ‘console.memory’ özelliğinin yanı sıra, görüntülenen yöntemler genellikle aşağıdaki gibidir (Chromium’un çıktısından alınmıştır):

Konsolu Açmak #

Güncel tarayıcıların çoğunda JavaScript Konsolu, Geliştirici Araçları’na bir sekme olarak entegre edilmiştir. Aşağıda listelenen kısayol tuşları Geliştirici Araçları’nı açacaktır, bundan sonra sağ sekmeye geçmek gerekebilir.

Krom #

Chrome’un DevTools uygulamasının “Konsol” panelini açma:

  • Windows / Linux: aşağıdaki seçeneklerden herhangi biri.

    • Ctrl + Shift + J
    • Ctrl + Shift + I, then click on the “Web Console” tab or press ESC to toggle the console on and off
    • F12, then click on the “Console” tab or press ESC to toggle the console on and off

  • Mac OS: Cmd + Opt + J

Firefox #

Firefox’un Geliştirici Araçları’nda “Konsol” panelini açma:

  • Windows / Linux: aşağıdaki seçeneklerden herhangi biri.

    • Ctrl + Shift + K
    • Ctrl + Shift + I, then click on the “Web Console” tab or press ESC to toggle the console on and off
    • F12, then click on the “Web Console” tab or press ESC to toggle the console on and off

  • Mac OS: Cmd + Opt + K

Edge ve Internet Explorer #

F12 Developer’da “Konsol” panelini açma Aletler:

  • F12, ardından “Konsol” sekmesine tıklayın

Safari #

Safari’nin Web Denetçisinde “Konsol” panelini açarak, önce Safari’nin Tercihlerinde geliştirme menüsünü etkinleştirmelisiniz.

Ardından menülerden “Geliştir->Hata Konsolunu Göster"i seçebilir veya ⌘ + Option + C tuşlarına basabilirsiniz.

Opera #

Operada “Konsol”u Açmak:

  • Ctrl + Shift + I, ardından “Konsol” sekmesine tıklayın

Uyumluluk #

Internet Explorer 8 veya önceki sürümleri kullanırken veya öykünürken (örn. Uyumluluk Görünümü / Kurumsal Mod aracılığıyla) konsol yalnızca Geliştirici Araçları etkin olduğunda tanımlanır, bu nedenle ‘console.log()’ ifadeleri bir istisnaya neden olabilir ve kodun yürütülmesini engelle. Bunu azaltmak için, oturum açmadan önce konsolun kullanılabilir olup olmadığını kontrol edebilirsiniz:

if (typeof window.console !== 'undefined')
{
   console.log("Hello World");
}

Veya betiğinizin başlangıcında, konsolun kullanılabilir olup olmadığını belirleyebilirsiniz ve değilse, tüm referanslarınızı yakalamak ve istisnaları önlemek için boş bir işlev tanımlayabilirsiniz.

if (!window.console)
{ 
    console = {log: function() {}}; 
}

Bu ikinci örneğin, geliştirici penceresi açılmış olsa bile tüm konsol günlüklerini durduracağını unutmayın.

Bu ikinci örneğin kullanılması, özellikle eklenmedikçe console.dir(obj) gibi diğer işlevlerin kullanılmasını engelleyecektir.

Zamanı ölçme - console.time() #

“console.time()”, kodunuzdaki bir görevin ne kadar sürede çalıştığını ölçmek için kullanılabilir.

console.time([label]) çağrıldığında yeni bir zamanlayıcı başlatılır. console.timeEnd([label]) çağrıldığında, geçen süre milisaniye cinsinden, orijinal .time() çağrısı hesaplanır ve günlüğe kaydedilir. Bu davranış nedeniyle, orijinal ‘.time()’ çağrısının yapılmasından bu yana geçen süreyi kaydetmek için ‘.timeEnd()’ öğesini aynı etiketle birden çok kez arayabilirsiniz.

Örnek 1:

console.time('response in');

alert('Click to continue');
console.timeEnd('response in');

alert('One more time');
console.timeEnd('response in');

çıktı alacak:

response in: 774.967ms
response in: 1402.199ms

Örnek 2:

var elms = document.getElementsByTagName('*'); //select all elements on the page

console.time('Loop time');

for (var i = 0; i < 5000; i++) {
    for (var j = 0, length = elms.length; j < length; j++) {
        // nothing to do ...
    }
}

console.timeEnd('Loop time');

çıktı alacak:

Loop time: 40.716ms

Konsol çıktısını biçimlendirme #

[Konsolun yazdırma yöntemlerinin1 birçoğu, “%” belirteçlerini kullanarak C benzeri dize biçimlendirmesini de işleyebilir:

console.log('%s has %d points', 'Sam', 100);

“Sam’in 100 puanı var” mesajını görüntüler.

Javascript’teki format belirteçlerinin tam listesi:

Tanımlayıcı Çıktı
%s Değeri bir dize olarak biçimlendirir
%i veya %d Değeri tamsayı olarak biçimlendirir
%f Değeri kayan nokta değeri olarak biçimlendirir
%o Değeri genişletilebilir bir DOM öğesi olarak biçimlendirir
%O Değeri genişletilebilir bir JavaScript nesnesi olarak biçimlendirir
%c İkinci parametre tarafından belirtildiği gibi çıktı dizesine CSS stil kurallarını uygular

Gelişmiş stil #

CSS biçim belirteci ("%c”) dizenin sol tarafına yerleştirildiğinde, yazdırma yöntemi, o dizenin biçimlendirmesi üzerinde ayrıntılı kontrole izin veren CSS kurallarına sahip ikinci bir parametreyi kabul eder:

console.log('%cHello world!', 'color: blue; font-size: xx-large');

görüntüler:

Birden çok “%c” biçim belirteci kullanmak mümkündür:

  • bir “%c"nin sağındaki herhangi bir alt dize, yazdırma yönteminde karşılık gelen bir parametreye sahiptir;

  • aynı alt dizeye CSS kurallarının uygulanmasına gerek yoksa bu parametre boş bir dize olabilir;

  • iki “%c” biçim belirteci bulunursa, 1st ("%c” içine alınmış) ve 2nd alt dizesinin kuralları 2<‘de tanımlanır print yönteminin sırasıyla sup>nd ve 3rd parametresi.

  • üç %c biçim belirteci bulunursa, o zaman 1st, 2nd ve 3rd alt dizelerinin kuralları şurada tanımlanır: sırasıyla 2nd , 3rd ve 4th parametresi vb…

    console.log("%cHello %cWorld%c!!", // string to be printed “color: blue;”, // applies color formatting to the 1st substring “font-size: xx-large;”, // applies font formatting to the 2nd substring “/* no CSS rule*/” // does not apply any rule to the remaing substring );

görüntüler:

Çıktıyı girintilemek için grupları kullanma #

Çıktı, aşağıdaki yöntemlerle hata ayıklama konsolunda tanımlanabilir ve daraltılabilir bir grup içine alınabilir:

  • console.groupCollapsed(): Bu yöntem çağrıldıktan sonra gerçekleştirilen tüm girişleri ortaya çıkarmak için açıklama düğmesi aracılığıyla genişletilebilen daraltılmış bir giriş grubu oluşturur;
  • console.group(): Bu yöntem çağrıldıktan sonra girişleri gizlemek için daraltılabilen genişletilmiş bir giriş grubu oluşturur.

Son girişler için kimlik, aşağıdaki yöntem kullanılarak kaldırılabilir:

  • console.groupEnd(): geçerli gruptan çıkar ve bu yöntem çağrıldıktan sonra üst grupta daha yeni girişlerin yazdırılmasına izin verir.

Gruplar, birbiri içinde birden çok tanımlanmış çıktıya veya daraltılabilir katmana izin vermek için basamaklandırılabilir:

Bir tarayıcının hata ayıklama konsoluna yazdırma #

Basit mesajları yazdırmak için bir tarayıcının hata ayıklama konsolu kullanılabilir. Bu hata ayıklama veya web konsolu doğrudan tarayıcıda açılabilir (çoğu tarayıcıda F12 tuşu – daha fazla bilgi için aşağıdaki Açıklamalar‘a bakın) ve ‘console’ Javascript nesnesinin ’log’ yöntemi, aşağıdakiler yazılarak çağrılabilir:

console.log('My message');

Ardından, Enter‘e basıldığında, hata ayıklama konsolunda ‘Mesajım’ görüntülenir.

console.log() geçerli kapsamda bulunan herhangi bir sayıda argüman ve değişken ile çağrılabilir. Birden çok argüman, aralarında küçük bir boşluk olacak şekilde bir satırda yazdırılacaktır.

var obj = { test: 1 };
console.log(['string'], 1, obj, window);

“log” yöntemi, hata ayıklama konsolunda aşağıdakileri görüntüler:

['string']  1  Object { test: 1 }  Window { /* truncated */ }

Düz dizelerin yanı sıra, “console.log()” diziler, nesneler, tarihler, işlevler vb. gibi diğer türleri de işleyebilir:

console.log([0, 3, 32, 'a string']);
console.log({ key1: 'value', key2: 'another value'});

görüntüler:

Array [0, 3, 32, 'a string']
Object { key1: 'value', key2: 'another value'}

İç içe nesneler daraltılabilir:

console.log({ key1: 'val', key2: ['one', 'two'], key3: { a: 1, b: 2 } });

görüntüler:

Object { key1: 'val', key2: Array[2], key3: Object }

“Tarih” nesneleri ve “işlev” gibi belirli türler farklı şekilde görüntülenebilir:

console.log(new Date(0));
console.log(function test(a, b) { return c; });

görüntüler:

Wed Dec 31 1969 19:00:00 GMT-0500 (Eastern Standard Time)
function test(a, b) { return c; }

Diğer yazdırma yöntemleri #

“log” yöntemine ek olarak, modern tarayıcılar da benzer yöntemleri destekler:

  • console.info – yazdırılan dize(ler)in veya nesne(ler)in sol tarafında küçük bilgilendirici simge (ⓘ) görünür.

  • console.warn – sol tarafta küçük uyarı simgesi (!) belirir. Bazı tarayıcılarda günlüğün arka planı sarıdır.

  • console.error – sol tarafta küçük zamanlar simgesi (⊗) görünür. Bazı tarayıcılarda günlüğün arka planı kırmızıdır.

  • console.timeStamp – geçerli saati ve belirtilen bir dizeyi verir, ancak standart değildir:

    console.timeStamp('msg');

görüntüler:

  00:00:00.001 msg

  • console.trace – geçerli yığın izinin çıktısını verir veya global kapsamda çağrıldıysa “log” yöntemiyle aynı çıktıyı görüntüler.

    function sec() {
   first();
}
function first() {
   console.trace();
}
sec();

görüntüler:

  first
  sec
  (anonymous function)

Yukarıdaki resim, Chrome 56 sürümündeki “timeStamp” dışında tüm işlevleri gösterir.

Bu yöntemler “log” yöntemine benzer şekilde davranır ve farklı hata ayıklama konsollarında farklı renk veya formatlarda görüntülenebilir.

Belirli hata ayıklayıcılarda, tek tek nesne bilgileri, ilgili nesne özelliklerine atıfta bulunan basılı metne veya küçük bir üçgene (►) tıklanarak daha da genişletilebilir. Bu çöken nesne özellikleri, günlükte açık veya kapalı olabilir. Bununla ilgili ek bilgi için console.dir sayfasına bakın.

Sayma - console.count() #

console.count([obj]) nesnenin argüman olarak sağlanan değerine bir sayaç yerleştirir. Bu yöntem her çağrıldığında, sayaç artar (boş '' hariç). Hata ayıklama konsolunda bir sayı ile birlikte aşağıdaki biçime göre bir etiket görüntülenir:

[label]: X

“etiket”, argüman olarak iletilen nesnenin değerini ve “X”, sayacın değerini temsil eder.

Değişkenler argüman olarak sağlanmış olsa bile, bir nesnenin değeri her zaman dikkate alınır:

var o1 = 1, o2 = '2', o3 = "";
console.count(o1);
console.count(o2);
console.count(o3);

console.count(1);
console.count('2');
console.count('');

görüntüler:

1: 1
2: 1
: 1
1: 2
2: 2
: 1

Sayı içeren dizeler “Number” nesnelerine dönüştürülür:

console.count(42.3);
console.count(Number('42.3'));
console.count('42.3');

görüntüler:

42.3: 1
42.3: 2
42.3: 3

Fonksiyonlar her zaman global “Function” nesnesine işaret eder:

console.count(console.constructor);
console.count(function(){});
console.count(Object);
var fn1 = function myfn(){};
console.count(fn1);
console.count(Number);

görüntüler:

[object Function]: 1
[object Function]: 2
[object Function]: 3
[object Function]: 4
[object Function]: 5

Belirli nesneler, başvurdukları nesne türüyle ilişkili belirli sayaçlar alır:

console.count(document);
console.count(console);
console.count(console.__proto__);
console.count(console.constructor.prototype);
console.count(console.__proto__.constructor.prototype);
console.count(Object.getPrototypeOf(console));
console.count(null);

görüntüler:

undefined: 1
undefined: 2

NaN: 3

Boş dize veya argüman yokluğu #

Hata ayıklama konsolunda **count yöntemi sırayla girilirken hiçbir bağımsız değişken sağlanmazsa, parametre olarak boş bir dize varsayılır, yani:

> console.count();
  : 1
> console.count('');
  : 2
> console.count("");
  : 3

İddialarla hata ayıklama - console.assert() #

Onay “false” ise konsola bir hata mesajı yazar. Aksi takdirde, iddia ‘doğru’ ise, bu hiçbir şey yapmaz.

console.assert('one' === 1);

İddiadan sonra birden fazla argüman sağlanabilir–bunlar sadece iddia “false” ise yazdırılacak olan dizeler veya diğer nesneler olabilir:

[

console.assert bir “AssertionError” (Node.js dışında) atmaz, bu, bu yöntemin çoğu test çerçevesiyle uyumlu olmadığı ve kod yürütmenin kesintiye uğramadığı anlamına gelir başarısız bir iddia.

Konsolu temizleme - console.clear() #

console.clear() yöntemini kullanarak konsol penceresini temizleyebilirsiniz. Bu, konsolda önceden yazdırılan tüm mesajları kaldırır ve bazı ortamlarda “Konsol temizlendi” gibi bir mesaj yazdırabilir.

Nesneleri ve XML’i etkileşimli olarak görüntüleme - console.dir(), console.dirxml() #

‘console.dir(object)’, belirtilen JavaScript nesnesinin özelliklerinin etkileşimli bir listesini görüntüler. Çıktı, alt nesnelerin içeriğini görmenizi sağlayan açıklama üçgenleriyle birlikte hiyerarşik bir liste olarak sunulur.

var myObject = {
    "foo":{
        "bar":"data"
    }
};

console.dir(myObject);

console.dirxml(document)

console.log(document)
