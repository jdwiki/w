Sözdizimi #

  • boşluk ifadesi; // İfadeyi değerlendirir ve dönüş değerini atar
  • +ifade; // İfadeyi sayıya dönüştürmeye çalış

  • !ifade; // İfade üzerinde mantıksal olumsuzlama gerçekleştir

operatörün typeof #

’typeof’ operatörü, değerlendirilmemiş işlenenin veri türünü bir dizge olarak döndürür.

Sözdizimi: #

typeof operand

İadeler: #

Bunlar, “typeof"tan olası dönüş değerleridir:

Tür Dönüş değeri
Tanımsız "tanımsız"
Null "nesne"
Boolean "boolean"
Sayı "sayı"
Dize "dize"
Sembol (ES6) "sembol"
İşlev nesnesi "işlev"
document.all "tanımsız"
Ana bilgisayar nesnesi (JS ortamı tarafından sağlanır) Uygulamaya bağlı
Başka herhangi bir nesne "nesne"

‘document.all’ın ’typeof’ operatörüyle olan olağandışı davranışı, eski tarayıcıları algılamak için eski kullanımından kaynaklanmaktadır. Daha fazla bilgi için bkz. http://stackoverflow.com/q/40643613/6388243

Örnekler: #

// returns 'number'
typeof 3.14;
typeof Infinity;
typeof NaN;               // "Not-a-Number" is a "number"

// returns 'string'
typeof "";
typeof "bla";
typeof (typeof 1);        // typeof always returns a string

// returns 'boolean'
typeof true;
typeof false;

// returns 'undefined'
typeof undefined;
typeof declaredButUndefinedVariable;
typeof undeclaredVariable;
typeof void 0;
typeof document.all       // see above

// returns 'function'
typeof function(){};
typeof class C {};
typeof Math.sin;

// returns 'object'
typeof { /*<...>*/ };
typeof null;
typeof /regex/;           // This is also considered an object
typeof [1, 2, 4];         // use Array.isArray or Object.prototype.toString.call.
typeof new Date();
typeof new RegExp();
typeof new Boolean(true); // Don't use!
typeof new Number(1);     // Don't use!
typeof new String("abc"); // Don't use!

// returns 'symbol'
typeof Symbol();
typeof Symbol.iterator;

Genel Bakış #

Birli operatörler, yalnızca bir işlenene sahip operatörlerdir. Tekli operatörler, standart JavaScript işlev çağrılarından daha verimlidir. Ek olarak, tekli operatörler geçersiz kılınamaz ve bu nedenle işlevleri garanti edilir.

Aşağıdaki tekli operatörler mevcuttur:

Operatör Operasyon Örnek
‘sil’ Silme operatörü, bir nesneden bir özelliği siler. örnek
void void operatörü, bir ifadenin dönüş değerini atar. [örnek][2]
typeof typeof operatörü, belirli bir nesnenin türünü belirler. [örnek][3]
+ Birli artı operatörü, işlenenini Sayı türüne dönüştürür. [örnek][4]
- Birli olumsuzlama operatörü, işlenenini Sayıya çevirir, sonra onu olumsuzlar. [örnek][5]
~ Bitsel DEĞİL işleci. [örnek][6]
! Mantıksal DEĞİL operatörü. [örnek][7]

1: https://www.wikiod.com/tr/javascript/tekli-operatorler#silme operatörü [2]: https://www.wikiod.com/tr/javascript/tekli-operatorler#boşluk operatörü [3]: https://www.wikiod.com/tr/javascript/tekli-operatorler#typeof operatörü [4]: https://www.wikiod.com/tr/javascript/tekli-operatorler#tekli artı operatörü (+) [5]: https://www.wikiod.com/tr/javascript/tekli-operatorler#Tekli olumsuzlama operatörü (-) [6]: https://www.wikiod.com/tr/javascript/tekli-operatorler#Bit düzeyinde DEĞİL operatörü (~) [7]: https://www.wikiod.com/tr/javascript/tekli-operatorler#Mantıksal DEĞİL operatörü (!)

Silme operatörü #

‘delete’ operatörü, bir nesneden bir özelliği siler.

Sözdizimi: #

delete object.property

delete object['property']

İadeler: #

Silme başarılıysa veya özellik mevcut değilse:

  • “doğru”

Silinecek özellik, kendi yapılandırılamayan bir özelliğiyse (silinemez):

  • katı olmayan modda “yanlış”.
  • Katı modda bir hata atar

Tanım #

‘Sil’ operatörü doğrudan hafızayı boşaltmaz. İşlem, özelliğe yapılan tüm referansların kaybolduğu anlamına gelirse, dolaylı olarak belleği boşaltabilir.

‘delete’ bir nesnenin özellikleri üzerinde çalışır. Nesnenin prototip zincirinde aynı ada sahip bir özellik varsa, özellik prototipten devralınır. “sil”, değişkenler veya işlev adları üzerinde çalışmaz.

Örnekler: #

// Deleting a property
foo = 1;              // a global variable is a property of `window`: `window.foo`
delete foo;           // true
console.log(foo);     // Uncaught ReferenceError: foo is not defined

// Deleting a variable
var foo = 1;
delete foo;           // false
console.log(foo);     // 1 (Not deleted)

// Deleting a function
function foo(){ };
delete foo;           // false
console.log(foo);     // function foo(){ } (Not deleted)

// Deleting a property
var foo = { bar: "42" };
delete foo.bar;       // true
console.log(foo);     // Object { } (Deleted bar)

// Deleting a property that does not exist
var foo = { };
delete foo.bar;       // true
console.log(foo);     // Object { } (No errors, nothing deleted)

// Deleting a non-configurable property of a predefined object
delete Math.PI;       // false  ()
console.log(Math.PI); // 3.141592653589793 (Not deleted)

tekli artı operatörü (+) #

Tekli artı (+) işleneninden önce gelir ve işleneni değerlendirir. Henüz değilse, işleneni bir sayıya dönüştürmeye çalışır.

Sözdizimi: #

+expression

İadeler: #

  • bir “Sayı”.

Tanım #

Tekli artı (’+’) operatörü, bir şeyi sayıya dönüştürmenin en hızlı (ve tercih edilen) yöntemidir.

Şunları dönüştürebilir:

  • tam sayıların (ondalık veya onaltılık) ve kayan sayıların dize temsilleri.
  • boole değerleri: “doğru”, “yanlış”.
  • “boş”

Dönüştürülemeyen değerler “NaN” olarak değerlendirilir.

Örnekler: #

+42           // 42
+"42"         // 42
+true         // 1
+false        // 0
+null         // 0
+undefined    // NaN
+NaN          // NaN
+"foo"        // NaN
+{}           // NaN
+function(){} // NaN

Bir diziyi dönüştürmeye çalışmanın beklenmedik dönüş değerleriyle sonuçlanabileceğini unutmayın. Arka planda, diziler önce dize temsillerine dönüştürülür:

[].toString() === '';
[1].toString() === '1';
[1, 2].toString() === '1,2';

Operatör daha sonra bu dizeleri sayılara dönüştürmeye çalışır:

+[]           // 0   ( === +'' )
+[1]          // 1   ( === +'1' )
+[1, 2]       // NaN ( === +'1,2' )

boşluk operatörü #

‘void’ operatörü verilen ifadeyi değerlendirir ve ardından ’tanımsız’ değerini döndürür.

Sözdizimi: #

void expression

İadeler: #

  • “tanımsız”

Tanım #

‘void’ operatörü genellikle ‘void 0’ veya ‘void(0)’ yazarak ‘undefined’ ilkel değerini elde etmek için kullanılır. “void"in bir işlev değil bir operatör olduğuna dikkat edin, bu nedenle “()” gerekli değildir.

Genellikle bir “void” ifadesinin sonucu ve “tanımsız” birbirinin yerine kullanılabilir. Ancak, ECMAScript’in eski sürümlerinde, ‘window.undefined’ herhangi bir değere atanabilirdi ve fonksiyonlar içindeki fonksiyon parametreleri değişkenleri için ‘undefined’ kullanmak hala mümkündür, böylece ‘undefined’ değerine dayanan diğer kodu bozar. `. “void” her zaman true “undefined” değerini verir.

“void 0”, “undefined” yazmanın daha kısa bir yolu olarak kod küçültmede de yaygın olarak kullanılır. Ayrıca, başka bir kod “window.undefined” ile kurcalanmış olabileceğinden, muhtemelen daha güvenlidir.

Örnekler: #

“tanımsız” döndürme:

function foo(){
    return void 0;
}
console.log(foo()); // undefined

Belirli bir kapsam içinde “tanımsız” değerini değiştirmek:

(function(undefined){
    var str = 'foo';
    console.log(str === undefined); // true
})('foo');

Birli olumsuzlama operatörü (-) #

Birli olumsuzlama (-) işleneninden önce gelir ve onu sayıya dönüştürmeye çalıştıktan sonra onu olumsuzlar.

Sözdizimi: #

-expression

İadeler: #

  • bir “Sayı”.

Tanım #

Tekli olumsuzlama (-), tekli artı (+) operatörünün yapabildiği aynı türleri/değerleri dönüştürebilir.

Dönüştürülemeyen değerler “NaN” olarak değerlendirilir ("-NaN” yoktur).

Örnekler: #

-42           // -42
-"42"         // -42
-true         // -1
-false        // -0
-null         // -0
-undefined    // NaN
-NaN          // NaN
-"foo"        // NaN
-{}           // NaN
-function(){} // NaN

Bir diziyi dönüştürmeye çalışmanın beklenmedik dönüş değerleriyle sonuçlanabileceğini unutmayın. Arka planda, diziler önce dize temsillerine dönüştürülür:

[].toString() === '';
[1].toString() === '1';
[1, 2].toString() === '1,2';

Operatör daha sonra bu dizeleri sayılara dönüştürmeye çalışır:

-[]           // -0  ( === -'' )
-[1]          // -1  ( === -'1' )
-[1, 2]       // NaN ( === -'1,2' )

Bit düzeyinde DEĞİL operatörü (~) #

Bitsel NOT (~), bir değerdeki her bit üzerinde bir NOT işlemi gerçekleştirir.

Sözdizimi: #

~expression

İadeler: #

  • bir “Sayı”.

Tanım #

DEĞİL işlemi için doğruluk tablosu:

a bir değil

Mantıksal NOT operatörü (!) #

Mantıksal NOT (!) operatörü, bir ifade üzerinde mantıksal olumsuzlama gerçekleştirir.

Sözdizimi: #

!expression

İadeler: #

  • bir “Boole”.

Tanım #

Mantıksal NOT (!) operatörü, bir ifade üzerinde mantıksal olumsuzlama gerçekleştirir.

Boole değerleri basitçe tersine çevrilir: !true === false ve !false === true. Boole olmayan değerler önce boole değerlerine dönüştürülür, ardından olumsuzlanır.

Bu, bir boole değerine herhangi bir değer atamak için çift mantıksal bir NOT (!!) kullanılabileceği anlamına gelir:

!!"FooBar" === true
!!1 === true
!!0 === false

Bunların hepsi !true değerine eşittir:

!'true' === !new Boolean('true');
!'false' === !new Boolean('false');
!'FooBar' === !new Boolean('FooBar');
![] === !new Boolean([]);
!{} === !new Boolean({});

Bunların hepsi !false değerine eşittir:

!0 === !new Boolean(0);
!'' === !new Boolean('');
!NaN === !new Boolean(NaN);
!null === !new Boolean(null);
!undefined === !new Boolean(undefined);

Örnekler: #

!true         // false
!-1           // false
!"-1"         // false
!42           // false
!"42"         // false
!"foo"        // false
!"true"       // false
!"false"      // false
!{}           // false
![]           // false
!function(){} // false

!false        // true
!null         // true
!undefined    // true
!NaN          // true
!0            // true
!""           // true
