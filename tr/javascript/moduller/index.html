Sözdizimi #

  • defaultMember’ı ‘modülden’ içe aktarın;
  • ‘modülden’ { üyeA, üyeB, … } içe aktarın;
    • ‘modülden’ modül olarak içe aktarın;
  • ‘modülden’ { üyeA as a, üyeB, … } içe aktarın;
  • defaultMember, * ‘modülden’ modül olarak içe aktarın;
  • defaultMember, { moduleA, … } ‘modülden’ içe aktarın;
  • ‘modülü’ içe aktarın;

MDN‘den (vurgu eklendi):

Bu özellik şu anda hiçbir tarayıcıda yerel olarak uygulanmamaktadır. [Traceur Compiler][traceur], Babel veya [Rollup] gibi birçok aktarıcıda uygulanmaktadır.

Birçok aktarıcı, ES6 modül sözdizimini Node ekosisteminde kullanım için CommonJS‘ye veya tarayıcıda kullanım için RequireJS veya System.js‘ye dönüştürebilir.

Birbirine bağımlı CommonJS modüllerini tarayıcıya yüklenebilecek tek bir dosyada birleştirmek için [Browserify] gibi bir modül paketleyici kullanmak da mümkündür.

Modül tanımlama #

ECMAScript 6’da, modül sözdizimini (“içe aktarma”/“dışa aktarma”) kullanırken, her dosya özel bir ad alanına sahip kendi modülü olur. Üst düzey işlevler ve değişkenler, genel ad alanını kirletmez. Diğer modüllerin içe aktarılacak işlevleri, sınıfları ve değişkenleri ortaya çıkarmak için ’export’ anahtar sözcüğünü kullanabilirsiniz.

// not exported
function somethingPrivate() {
    console.log('TOP SECRET')
}


export const PI = 3.14;

export function doSomething() {
    console.log('Hello from a module!')
}

function doSomethingElse(){ 
    console.log("Something else")
}

export {doSomethingElse}

export class MyClass {
    test() {}
}
