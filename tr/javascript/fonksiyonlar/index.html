JavaScript’teki işlevler, bir dizi eylemi gerçekleştirmek için düzenli, yeniden kullanılabilir kod sağlar. Fonksiyonlar kodlama sürecini basitleştirir, gereksiz mantığı önler ve kodun takip edilmesini kolaylaştırır. Bu konu, JavaScript’te işlevlerin, bağımsız değişkenlerin, parametrelerin, dönüş ifadelerinin ve kapsamın bildirilmesini ve kullanımını açıklar.

Sözdizimi #

  • fonksiyon örneği(x) { dönüş x }

  • var örnek = işlev (x) { dönüş x }

  • (işlev() { … })(); // Hemen Çağrılan İşlev İfadesi (IIFE)

  • var örnek = yeni Örnek(x);

  • Yöntemler

  • fn.apply(valueForThis[, arrayOfArgs])

  • fn.bind(değerForBu[, arg1[, arg2, …]])

  • fn.call(değerForBu[, arg1[, arg2, …]])

  • ES2015+ (ES6+):

  • sabit örnek = x => { dönüş x }; // Ok işlevi kesin dönüş

  • sabit örnek = x => x; // Ok işlevi örtük dönüş

  • const örnek = (x, y, z) => { … } // Ok fonksiyonu çoklu argümanlar

  • (() => { … })(); // bir ok işlevi kullanarak IIFE

Ok işlevleri hakkında bilgi için lütfen Ok İşlevleri belgelerine bakın.

İşlev Kapsamı #

Bir işlev tanımladığınızda, bir kapsam oluşturur.

İşlev içinde tanımlanan her şeye, işlev dışındaki kodla erişilemez. işlev. Yalnızca bu kapsamdaki kod, içinde tanımlanan varlıkları görebilir. dürbün.

function foo() {
  var a = 'hello';
  console.log(a);  // => 'hello'
}

console.log(a);  // reference error

JavaScript’te iç içe işlevler mümkündür ve aynı kurallar geçerlidir.

function foo() {
  var a = 'hello';
  
  function bar() {
    var b = 'world';
    console.log(a);  // => 'hello'
    console.log(b);  // => 'world'
  }

  console.log(a);  // => 'hello'
  console.log(b);  // reference error
}

console.log(a);  // reference error
console.log(b);  // reference error

JavaScript bir referansı veya değişkeni çözmeye çalıştığında, aramaya başlar. mevcut kapsamda. Bu beyanı mevcut kapsamda bulamazsa, onu aramak için bir dürbünün yukarısına tırmanır. Bu işlem beyanname bulunana kadar tekrarlanır. JavaScript ayrıştırıcısı genel kapsama ulaşırsa ve yine de referans, bir referans hatası atılacaktır.

var a = 'hello';

function foo() {
  var b = 'world';

  function bar() {
    var c = '!!';

    console.log(a);  // => 'hello'
    console.log(b);  // => 'world'
    console.log(c);  // => '!!'
    console.log(d);  // reference error
  }
}

Bu tırmanma davranışı aynı zamanda bir referansın bir referansın üzerinde “gölge” olabileceği anlamına da gelebilir. ilk görüldüğü için dış kapsamda benzer şekilde adlandırılmış referans.

var a = 'hello';

function foo() {
  var a = 'world';

  function bar() {
    console.log(a);  // => 'world'
  }
}

JavaScript’in kapsam belirlemeyi çözme şekli, “const” anahtar sözcüğü için de geçerlidir. ‘const’ anahtar sözcüğüyle bir değişken bildirmek, değeri yeniden atamanıza izin verilmediği anlamına gelir, ancak bunu bir işlevde bildirmek yeni bir kapsam ve bununla birlikte yeni bir değişken oluşturacaktır.

function foo() {
  const a = true;

  function bar() {
    const a = false;  // different variable
    console.log(a);   // false
  }

  const a = false;    // SyntaxError
  a = false;          // TypeError
  console.log(a);     // true
}

Ancak, kapsam oluşturan bloklar yalnızca işlevler değildir (eğer “let” veya “const” kullanıyorsanız). “let” ve “const” bildirimleri, en yakın blok ifadesinin kapsamına sahiptir. Daha ayrıntılı bir açıklama için buraya bakın.

Köri #

Currying, “n” arity veya argümanların bir fonksiyonunun sadece bir argüman alan bir “n” fonksiyon dizisine dönüştürülmesidir.

Kullanım durumları: Bazı argümanların değerleri diğerlerinden önce mevcut olduğunda, bir fonksiyonu, her bir değer geldiğinde işi aşamalar halinde tamamlayan bir dizi fonksiyona ayırmak için körlemeyi kullanabilirsiniz. Bu yararlı olabilir:

  • Bir argümanın değeri neredeyse hiç değişmediğinde (örneğin, bir dönüştürme faktörü), ancak bu değeri ayarlama esnekliğini korumanız gerektiğinde (sabit olarak kodlamak yerine).
  • Döndürülen bir işlevin sonucu, diğer seçilen işlevler çalıştırılmadan önce yararlı olduğunda.
  • İşlevlerin belirli bir sırayla gelişini doğrulamak için.

Örneğin, bir dikdörtgen prizmanın hacmi üç faktörün fonksiyonu ile açıklanabilir: uzunluk (’l’), genişlik (‘w’) ve yükseklik (‘h’):

var prism = function(l, w, h) {
    return l * w * h;
}

Bu işlevin curried versiyonu şöyle görünür:

function prism(l) {
    return function(w) {
        return function(h) {
            return l * w * h;
        }
    }
}

// alternatively, with concise ECMAScript 6+ syntax:
var prism = l => w => h => l * w * h;

Hemen Çağrılan İşlev İfadeleri #

Bazen işlevinizin bir değişken olarak erişilebilir/saklanmış olmasını istemezsiniz. Hemen Çağrılan İşlev İfadesi (kısaca IIFE) oluşturabilirsiniz. Bunlar esasen kendi kendini yürüten anonim işlevlerdir. Çevreleyen kapsama erişimleri vardır, ancak işlevin kendisine ve herhangi bir dahili değişkene dışarıdan erişilemez. IIFE hakkında dikkat edilmesi gereken önemli bir nokta, işlevinizi adlandırsanız bile, IIFE’nin standart işlevler gibi kaldırılmaması ve bildirildikleri işlev adıyla çağrılamamasıdır.

(function() {
   alert("I've run - but can't be run again because I'm immediately invoked at runtime,
          leaving behind only the result I generate");
}());

Bu, IIFE yazmanın başka bir yoludur. Noktalı virgülden önceki kapanış parantezinin hareket ettirildiğine ve kapanış küme parantezinden hemen sonra yerleştirildiğine dikkat edin:

(function() {
   alert("This is IIFE too.");
})();

Parametreleri bir IIFE’ye kolayca iletebilirsiniz:

(function(message) {
   alert(message);
}("Hello World!"));

Ek olarak, değerleri çevreleyen kapsama döndürebilirsiniz:

var example = (function() {
   return 42;
}());
console.log(example); // => 42

Gerekirse, bir IIFE adlandırmak mümkündür. Daha az sıklıkta görülse de, bu modelin bir özyineleme için kullanılabilecek bir referans sağlaması ve ad çağrı yığınına dahil edildiğinden hata ayıklamayı kolaylaştırması gibi çeşitli avantajları vardır.

(function namedIIFE() { 
    throw error; // We can now see the error thrown in 'namedIIFE()'
}());

Bir işlevi parantez içine sarmak Javascript ayrıştırıcısına bir ifade beklemesini belirtmenin en yaygın yolu olsa da, bir ifadenin zaten beklendiği yerlerde gösterim daha kısa yapılabilir:

var a = function() { return 42 }();
console.log(a)  // => 42

Hemen çağrılan işlevin ok versiyonu:

(() => console.log("Hello!"))(); // => Hello!

‘bu’ ve bağımsız değişkenleri bağlama #

JavaScript’te bir yönteme (bir işlev olan bir özellik) başvuru yaptığınızda, genellikle başlangıçta eklendiği nesneyi hatırlamaz. Yöntemin bu nesneye ‘bu’ olarak başvurması gerekiyorsa, bunu yapamaz ve onu çağırmak muhtemelen bir çökmeye neden olur.

“this” değerini ve herhangi bir sayıda önde gelen bağımsız değişkeni içeren bir sarmalayıcı oluşturmak için bir işlevde “.bind()” yöntemini kullanabilirsiniz.

var monitor = {
  threshold: 5,
  check: function(value) {
    if (value > this.threshold) {
      this.display("Value is too high!");
    }
  },
  display(message) {
    alert(message);
  }
};

monitor.check(7); // The value of `this` is implied by the method call syntax.


var badCheck = monitor.check;
badCheck(15); // The value of `this` is window object and this.threshold is undefined, so value > this.threshold is false

var check = monitor.check.bind(monitor);
check(15); // This value of `this` was explicitly bound, the function works.

var check8 = monitor.check.bind(monitor, 8);
check8(); // We also bound the argument to `8` here. It can't be re-specified.

Katı modda olmadığında, bir işlev genel nesneyi (tarayıcıdaki “pencere”) “bu” olarak kullanır, işlev bir yöntem olarak çağrılmadığı, bağlı olmadığı veya yöntem “.call” sözdizimi ile çağrılmadığı sürece.

window.x = 12; 

function example() {
  return this.x;
}

console.log(example()); // 12

Katı modda “bu” varsayılan olarak “tanımsızdır”

window.x = 12; 
    
function example() {
  "use strict";
  return this.x;
}

console.log(example()); // Uncaught TypeError: Cannot read property 'x' of undefined(…)
son sürüm

Bu sözdizimi, “bunu” her yere bağlama konusunda endişelenmeden normal şekilde yazmanıza olanak tanır.

Adlandırılmış işlevler, birden çok senaryoda anonim işlevlerden farklıdır:

  • Hata ayıklarken, fonksiyonun adı hata/yığın izinde görünecektir.

Bağımsız Değişken Sayısı Bilinmeyen İşlevler (değişken işlevler) #

Belirsiz sayıda argüman kabul eden bir fonksiyon yaratmak için ortamınıza bağlı olarak iki yöntem vardır.

Bir işlev çağrıldığında, kapsamında işleve iletilen tüm bağımsız değişkenleri içeren Array benzeri bir arguments nesnesi bulunur. Buna indeksleme veya bunun üzerinde yineleme yapmak, örneğin argümanlara erişim sağlayacaktır.

function logSomeThings() {
    for (var i = 0; i < arguments.length; ++i) {
        console.log(arguments[i]);
    }
}

logSomeThings('hello', 'world');
// logs "hello"
// logs "world"

Gerekirse “argümanları” gerçek bir Diziye dönüştürebileceğinizi unutmayın; bakınız: https://www.wikiod.com/tr/javascript/diziler#Dizi Benzeri Nesneleri Dizilere Dönüştürme

son sürüm

ES6’dan, işlev rest operatörü (...) kullanılarak son parametresiyle bildirilebilir. Bu, o noktadan itibaren argümanları tutan bir Dizi oluşturur.

function personLogsSomeThings(person, ...msg) {
   msg.forEach(arg => {
       console.log(person, 'says', arg);
   });
}

personLogsSomeThings('John', 'hello', 'world');
// logs "John says hello"
// logs "John says world"

İşlevler de benzer şekilde çağrılabilir, spread syntax

const logArguments = (...args) => console.log(args)
const list = [1, 2, 3]

logArguments('a', 'b', 'c', ...list)
// output: Array [ "a", "b", "c", 1, 2, 3 ]

Bu sözdizimi, herhangi bir konuma rastgele sayıda argüman eklemek için kullanılabilir ve herhangi bir yinelenebilir (“apply” yalnızca dizi benzeri nesneleri kabul eder) ile kullanılabilir.

const logArguments = (...args) => console.log(args)
function* generateNumbers() {
  yield 6
  yield 5
  yield 4
}

logArguments('a', ...generateNumbers(), ...'pqr', 'b')
// output: Array [ "a", 6, 5, 4, "p", "q", "r", "b" ]

Anonim İşlev #

Anonim Bir Fonksiyon Tanımlama #

Bir işlev tanımlandığında, genellikle ona bir ad verirsiniz ve ardından bu adı kullanarak onu çağırırsınız, şöyle:

foo();

function foo(){
    // ...
}

Bir işlevi bu şekilde tanımladığınızda, Javascript çalışma zamanı işlevinizi bellekte saklar ve atadığınız adı kullanarak bu işleve bir başvuru oluşturur. Bu ada daha sonra geçerli kapsamda erişilebilir. Bu, bir işlev oluşturmak için çok uygun bir yol olabilir, ancak Javascript, bir işleve bir ad atamanızı gerektirmez. Aşağıdakiler de tamamen yasaldır:

function() {
    // ...
}

Bir fonksiyon isimsiz tanımlandığında, isimsiz fonksiyon olarak bilinir. İşlev bellekte saklanır, ancak çalışma zamanı sizin için buna otomatik olarak bir başvuru oluşturmaz. İlk bakışta, böyle bir şeyin hiçbir faydası yokmuş gibi görünebilir, ancak anonim işlevlerin çok uygun olduğu birkaç senaryo vardır.

Değişkene Anonim Fonksiyon Atama #

Anonim işlevlerin çok yaygın bir kullanımı, onları bir değişkene atamaktır:

var foo = function(){ /*...*/ };

foo();

Anonim işlevlerin bu kullanımı https://www.wikiod.com/tr/javascript/fonksiyonlar#Değişken olarak işlevler adresinde daha ayrıntılı olarak ele alınmaktadır.

Anonim Bir Fonksiyonu Başka Bir Fonksiyona Parametre Olarak Vermek #

Bazı işlevler, bir işleve yapılan başvuruyu parametre olarak kabul edebilir. Bunlara bazen “bağımlılık enjeksiyonları” veya “geri aramalar” denir, çünkü çağrınızın kodunuza “geri arama” yapmasına izin verir ve size çağrılan işlevin çalışma şeklini değiştirme fırsatı verir. Örneğin, Array nesnesinin harita işlevi, bir dizinin her bir öğesi üzerinde yineleme yapmanıza ve ardından her öğeye bir dönüştürme işlevi uygulayarak yeni bir dizi oluşturmanıza olanak tanır.

var nums = [0,1,2];
var doubledNums = nums.map( function(element){ return element * 2; } ); // [0,2,4]

Kapsamınızı yalnızca bu tek yerde ihtiyaç duyulan bir işlevle karıştıracak ve kodunuzun doğal akışını ve okunmasını bozacak adlandırılmış bir işlev oluşturmak sıkıcı, özensiz ve gereksiz olacaktır (bir meslektaşınızın kodunuzu bulması için bu kodu bırakması gerekirdi). neler olduğunu anlama işlevi).

Başka Bir Fonksiyondan Anonim Bir Fonksiyonu Döndürmek #

Bazen bir işlevi başka bir işlevin sonucu olarak döndürmek yararlıdır. Örneğin:

var hash = getHashFunction( 'sha1' );
var hashValue = hash( 'Secret Value' );

function getHashFunction( algorithm ){

    if ( algorithm === 'sha1' ) return function( value ){ /*...*/ };
    else if ( algorithm === 'md5' ) return function( value ){ /*...*/ };

}
