Sözdizimi #

  • deneyin { … } yakalama (hata) { … }
  • sonunda { … } deneyin { … }
  • { … } yakalamayı (hata) { … } sonunda { … } deneyin
  • yeni Hata ([mesaj]);
  • Hata ([mesaj]);

try, yürütülürken hatalara karşı test edilecek bir kod bloğu tanımlamanıza izin verir.

‘catch’, ’try’ bloğunda bir hata meydana gelirse, yürütülecek bir kod bloğu tanımlamanıza izin verir.

’nihayet’, sonuçtan bağımsız olarak kodu çalıştırmanıza izin verir. Yine de dikkat edin, try ve catch bloklarının kontrol akışı deyimleri, son bloğun yürütülmesi bitene kadar askıya alınacaktır.

Hata nesneleri #

JavaScript’teki çalışma zamanı hataları, “Error” nesnesinin örnekleridir. ‘Error’ nesnesi olduğu gibi veya kullanıcı tanımlı istisnalar için temel olarak da kullanılabilir. Herhangi bir değer türü - örneğin, dizeler - atmak mümkündür, ancak yığın izleri gibi hata ayıklama bilgilerinin doğru şekilde korunmasını sağlamak için “Hata"yı veya türevlerinden birini kullanmanız şiddetle tavsiye edilir.

‘Error’ yapıcısının ilk parametresi, insan tarafından okunabilen hata mesajıdır. Başka bir yerde ek bilgiler bulunsa bile, her zaman neyin yanlış gittiğine dair yararlı bir hata mesajı belirtmeye çalışmalısınız.

try {
   throw new Error('Useful message');
} catch (error) {
   console.log('Something went wrong! ' + error.message);
}

Sözlerle Etkileşim #

İstisnalar, eşzamanlı kod için, reddetmelerin söz vermek tabanlı eşzamansız koddur. Bir söz işleyicide bir istisna atılırsa, hatası otomatik olarak yakalanacak ve bunun yerine sözü reddetmek için kullanılacaktır.

Promise.resolve(5)
    .then(result => {
        throw new Error("I don't like five");
    })
    .then(result => {
        console.info("Promise resolved: " + result);
    })
    .catch(error => {
        console.error("Promise rejected: " + error);
    });

 

Promise rejected: Error: I don't like five
son sürüm

ECMAScript 2017’nin bir parçası olması beklenen zaman uyumsuz işlevler önerisi bunu ters yönde genişletiyor. Reddedilen bir sözü beklerseniz, istisna olarak hatası ortaya çıkar:

async function main() {
  try {
    await Promise.reject(new Error("Invalid something"));
  } catch (error) {
    console.log("Caught error: " + error);
  }
}
main();

 

Caught error: Invalid something
son sürüm

İşlem sırası artı gelişmiş düşünceler #

Bir try catch bloğu olmadan, tanımsız işlevler hata verir ve yürütmeyi durdurur:

undefinedFunction("This will not get executed");
console.log("I will never run because of the uncaught error!");

Bir hata verir ve ikinci satırı çalıştırmaz:

// Uncaught ReferenceError: undefinedFunction is not defined

Kodun çalışmaya devam edebilmesi için bu hatayı yakaladığınızdan emin olmak için diğer dillere benzer bir try catch bloğuna ihtiyacınız vardır:

try {
    undefinedFunction("This will not get executed");
} catch(error) {
    console.log("An error occured!", error);
} finally {
    console.log("The code-block has finished");
}
console.log("I will run because we caught the error!");

Şimdi hatayı yakaladık ve kodumuzun yürütüleceğinden emin olabiliriz.

// An error occured! ReferenceError: undefinedFunction is not defined(…)
// The code-block has finished
// I will run because we caught the error!

Ya yakalama bloğumuzda bir hata oluşursa!?

try {
    undefinedFunction("This will not get executed");
} catch(error) {
    otherUndefinedFunction("Uh oh... ");
    console.log("An error occured!", error);
} finally {
    console.log("The code-block has finished");
}
console.log("I won't run because of the uncaught error in the catch block!");

Catch bloğumuzun geri kalanını işlemeyeceğiz ve final bloğu dışında yürütme duracaktır.

// The code-block has finished
// Uncaught ReferenceError: otherUndefinedFunction is not defined(…)

Deneme yakalama bloklarınızı her zaman iç içe geçirebilirsiniz .. ama yapmamalısınız çünkü bu aşırı derecede dağınık olacaktır ..

try {
    undefinedFunction("This will not get executed");
} catch(error) {
    try {
        otherUndefinedFunction("Uh oh... ");
    } catch(error2) {
        console.log("Too much nesting is bad for my heart and soul...");
    }
    console.log("An error occured!", error);
} finally {
    console.log("The code-block has finished");
}
console.log("I will run because we caught the error!");

Önceki örnekteki tüm hataları yakalayacak ve aşağıdakileri günlüğe kaydedecektir:

//Too much nesting is bad for my heart and soul...
//An error occured! ReferenceError: undefinedFunction is not defined(…)
//The code-block has finished
//I will run because we caught the error!

Peki, tüm hataları nasıl yakalayabiliriz!? Tanımsız değişkenler ve işlevler için: yapamazsınız.

Ayrıca, her değişkeni ve işlevi bir dene/yakala bloğuna sarmamalısınız, çünkü bunlar siz onları düzeltene kadar yalnızca bir kez gerçekleşecek basit örneklerdir. Ancak, var olduğunu bildiğiniz, ancak özelliklerinin veya alt işlemlerinin veya yan etkilerinin olup olmayacağını bilmediğiniz veya bazı durumlarda bazı hata durumları beklediğiniz nesneler, işlevler ve diğer değişkenler için, hata işlemenizi soyutlamalısınız. bir şekilde. İşte çok temel bir örnek ve uygulama.

Güvenilmeyen veya istisna atma yöntemlerini çağırmanın korumalı bir yolu olmadan:

function foo(a, b, c) {
    console.log(a, b, c);
    throw new Error("custom error!");
}
try {
    foo(1, 2, 3);
} catch(e) { 
    try {
        foo(4, 5, 6); 
    } catch(e2) {
        console.log("We had to nest because there's currently no other way...");
    }
    console.log(e);
}
// 1 2 3
// 4 5 6
// We had to nest because there's currently no other way...
// Error: custom error!(…)

Ve koruma ile:

function foo(a, b, c) {
    console.log(a, b, c);
    throw new Error("custom error!");
}
function protectedFunction(fn, ...args) {
    try {
        fn.apply(this, args);
    } catch (e) {
        console.log("caught error: " + e.name + " -> " + e.message);
    }
}

protectedFunction(foo, 1, 2, 3);
protectedFunction(foo, 4, 5, 6);

// 1 2 3
// caught error: Error -> custom error!
// 4 5 6
// caught error: Error -> custom error!

Hataları yakalarız ve yine de biraz farklı bir sözdizimi ile beklenen tüm kodu işliyoruz. Her iki şekilde de işe yarayacaktır, ancak daha gelişmiş uygulamalar oluştururken, hata işlemenizi soyutlamanın yollarını düşünmeye başlamak isteyeceksiniz.

Hata türleri #

JavaScript’te altı belirli temel hata oluşturucu vardır:

  • EvalError - eval() genel işleviyle ilgili olarak oluşan bir hatayı temsil eden bir örnek oluşturur.

  • InternalError - JavaScript motorunda bir dahili hata atıldığında oluşan bir hatayı temsil eden bir örnek oluşturur. Örneğin. “çok fazla özyineleme”. (Yalnızca Mozilla Firefox tarafından desteklenir)

  • RangeError - sayısal bir değişken veya parametre geçerli değerinin dışında olduğunda oluşan bir hatayı temsil eden bir örnek oluşturur Aralık.

  • ReferenceError - geçersiz bir referansın referansı kaldırılırken oluşan bir hatayı temsil eden bir örnek oluşturur.

  • SyntaxError - eval() içinde kod ayrıştırılırken oluşan sözdizimi hatasını temsil eden bir örnek oluşturur.

  • TypeError - bir değişken veya parametre geçerli bir tür olmadığında oluşan bir hatayı temsil eden bir örnek oluşturur.

  • URIError - “encodeURI()” veya “decodeURI()” geçersiz parametreler geçirildiğinde oluşan bir hatayı temsil eden bir örnek oluşturur.

Hata işleme mekanizması uyguluyorsanız, koddan ne tür bir hata yakaladığınızı kontrol edebilirsiniz.

try {
    throw new TypeError();
}
catch (e){
    if(e instanceof Error){
        console.log('instance of general Error constructor');
    }

    if(e instanceof TypeError) {
        console.log('type error');
    }
}

Böyle bir durumda “e”, “TypeError” örneğinin bir örneği olacaktır. Tüm hata türleri, temel oluşturucu ‘Hata’yı genişletir, bu nedenle aynı zamanda bir ‘Hata’ örneğidir.

Bunu akılda tutmak, çoğu durumda ’e’yi ‘Hata’ örneği olarak işaretlemenin yararsız olduğunu gösterir.
