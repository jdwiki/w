Soyut eşitlik / eşitsizlik ve tür dönüştürme #

Sorun #

Soyut eşitlik ve eşitsizlik operatörleri (== ve !=), işlenen türleri eşleşmezse işlenenlerini dönüştürür. Bu tür zorlama, bu operatörlerin sonuçları hakkında yaygın bir kafa karışıklığı kaynağıdır, özellikle bu operatörler beklendiği gibi her zaman geçişli değildir.

"" ==  0;     // true A
 0 == "0";    // true A
"" == "0";    // false B
false == 0;   // true
false == "0"; // true

"" !=  0;     // false A
 0 != "0";    // false A
"" != "0";    // true B
false != 0;   // false
false != "0"; // false

JavaScript’in boş dizeleri sayılara nasıl dönüştürdüğünü düşünürseniz, sonuçlar anlamlı olmaya başlar.

Number("");    // 0
Number("0");   // 0
Number(false); // 0

Çözüm #

false B ifadesinde, her iki işlenen de dizgedir ("" ve "0"), bu nedenle tür dönüşümü olmayacaktır ve "" ve "0" olduğundan aynı değer değil, "" == "0" beklendiği gibi “yanlış”.

Burada beklenmeyen davranışları ortadan kaldırmanın bir yolu, her zaman aynı türdeki işlenenleri karşılaştırdığınızdan emin olmaktır. Örneğin, sayısal karşılaştırmanın sonuçlarını istiyorsanız, açık dönüştürmeyi kullanın:

var test = (a,b) => Number(a) == Number(b); 
test("", 0);        // true;
test("0", 0);       // true
test("", "0");      // true;
test("abc", "abc"); // false as operands are not numbers

Veya dize karşılaştırması istiyorsanız:

var test = (a,b) => String(a) == String(b);
test("", 0);   // false;
test("0", 0);  // true
test("", "0"); // false;

Yan not: Number("0") ve new Number("0") aynı şey değildir! İlki bir tür dönüştürme gerçekleştirirken, ikincisi yeni bir nesne yaratacaktır. Nesneler, aşağıdaki sonuçları açıklayan değere göre değil, referansa göre karşılaştırılır.

Number("0") == Number("0");         // true;
new Number("0") == new Number("0"); // false

Son olarak, herhangi bir örtük tür dönüştürme gerçekleştirmeyecek katı eşitlik ve eşitsizlik işleçleri kullanma seçeneğiniz vardır.

"" ===  0;  // false
 0 === "0"; // false
"" === "0"; // false

Bu konuya daha fazla referans burada bulunabilir:

JavaScript karşılaştırmalarında hangi eşittir operatörü (== vs ===) kullanılmalıdır?.

Global Nesnenin NaN Özelliği #

NaN ("Not a Number"), IEEE Kayan Nokta Aritmetiği Standardı tarafından tanımlanan özel bir değerdir ve şu durumlarda kullanılır: sayısal olmayan bir değer sağlanır, ancak bir sayı beklenir (1 * "iki") veya bir hesaplamanın geçerli bir “sayı” sonucu olmadığında (Math.sqrt(-1)).

‘NaN’ ile herhangi bir eşitlik veya ilişkisel karşılaştırma, kendisiyle karşılaştırsa bile ‘yanlış’ döndürür. Çünkü ‘NaN’ anlamsız bir hesaplamanın sonucunu ifade etmesi gerekiyor ve bu nedenle diğer anlamsız hesaplamaların sonucuna eşit değil.

(1 * "two") === NaN  //false

NaN === 0;          // false
NaN === NaN;        // false
Number.NaN === NaN; // false

NaN < 0;            // false
NaN > 0;            // false
NaN > 0;            // false
NaN >= NaN;         // false
NaN >= 'two';       // false

Eşit olmayan karşılaştırmalar her zaman “true” değerini döndürür:

NaN !== 0;          // true
NaN !== NaN;        // true

Bir değerin NaN olup olmadığı kontrol ediliyor #

Number.isNaN() işlevini kullanarak “NaN” için bir değeri veya ifadeyi test edebilirsiniz:

Number.isNaN(NaN);         // true
Number.isNaN(0 / 0);       // true
Number.isNaN('str' - 12);  // true

Number.isNaN(24);          // false
Number.isNaN('24');        // false
Number.isNaN(1 / 0);       // false
Number.isNaN(Infinity);    // false

Number.isNaN('str');       // false
Number.isNaN(undefined);   // false
Number.isNaN({});          // false
son sürüm

Bir değerin “NaN” olup olmadığını kendisiyle karşılaştırarak kontrol edebilirsiniz:

value !== value;    // true for NaN, false for any other value

Number.isNaN() için aşağıdaki çoklu dolguyu kullanabilirsiniz:

Number.isNaN = Number.isNaN || function(value) {     
    return value !== value;
}
son sürüm

Buna karşılık, “isNaN()” global işlevi yalnızca “NaN” için değil, aynı zamanda bir sayıya zorlanamayan herhangi bir değer veya ifade için de “true” değerini döndürür:

isNaN(NaN);         // true
isNaN(0 / 0);       // true
isNaN('str' - 12);  // true

isNaN(24);          // false
isNaN('24');        // false
isNaN(Infinity);    // false

isNaN('str');       // true
isNaN(undefined);   // true
isNaN({});          // true

ECMAScript, ECMAScript 6’dan beri ‘Object.is’ ile çağrılabilen ‘SameValue’ adlı bir “aynılık” algoritması tanımlar. == ve === karşılaştırmasından farklı olarak, Object.is() kullanmak, NaN öğesini kendisiyle aynı olarak ele alır (ve -0, +0 ile aynı değildir):

Object.is(NaN, NaN)      // true
Object.is(+0, 0)         // false

NaN === NaN              // false
+0 === 0                 // true

‘Object.is()’ için aşağıdaki çoklu dolguyu kullanabilirsiniz ([MDN](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/is#Polyfill_for_non- adresinden) ES6_browsers)):

if (!Object.is) {
  Object.is = function(x, y) {
    // SameValue algorithm
    if (x === y) { // Steps 1-5, 7-10
      // Steps 6.b-6.e: +0 != -0
      return x !== 0 || 1 / x === 1 / y;
    } else {
      // Step 6.a: NaN == NaN
      return x !== x && y !== y;
    }
  };
}
son sürüm

Dikkat edilmesi gereken noktalar #

NaN’nin kendisi bir sayıdır, yani “NaN” dizesine eşit değildir ve en önemlisi (belki de sezgisel olmasa da):

typeof(NaN) === "number"; //true

Boole operatörlerinde kısa devre #

and-operatörü (&&) ve or-operatörü (||), işlemin sonucu fazladan çalışma ile değişmiyorsa gereksiz çalışmayı önlemek için kısa devre kullanır.

“x && y"de, “x” “yanlış” olarak değerlendirilirse “y” değerlendirilmez, çünkü tüm ifadenin “yanlış” olacağı garanti edilir.

`x || y’, ‘x’, ’true’ olarak değerlendirilirse, ‘y’ değerlendirilmez, çünkü tüm ifadenin ’true’ olacağı garanti edilir.

İşlevlerle örnek

Aşağıdaki iki işlevi alın:

function T() { // True
  console.log("T");
  return true;
}

function F() { // False
  console.log("F");
  return false;
}

Örnek 1

T() && F(); // false

Çıktı:

‘T’
‘F’

Örnek 2

F() && T(); // false

Çıktı:

‘F’

Örnek 3

T() || F(); // true

Çıktı:

‘T’

Örnek 4

F() || T(); // true

Çıktı:

‘F’
‘T’

Hataları önlemek için kısa devre

var obj; // object has value of undefined
if(obj.property){ }// TypeError: Cannot read property 'property' of undefined
if(obj.property && obj !== undefined){}// Line A TypeError: Cannot read property 'property' of undefined

Satır A: Sıralamayı tersine çevirirseniz, ilk koşullu ifade, hatayı atarsa, yürütmeyerek ikincideki hatayı önleyecektir.

if(obj !== undefined && obj.property){}; // no error thrown

Ancak yalnızca “tanımsız” olmasını bekliyorsanız kullanılmalıdır.

if(typeof obj === "object" && obj.property){}; // safe option but slower

Varsayılan bir değer sağlamak için kısa devre

|| operatörü “gerçek” bir değer veya varsayılan değer seçmek için kullanılabilir.

Örneğin, bu, null yapılabilir bir değerin null olamaz bir değere dönüştürülmesini sağlamak için kullanılabilir:

var nullableObj = null;
var obj = nullableObj || {};  // this selects {}

var nullableObj2 = {x: 5};
var obj2 = nullableObj2 || {} // this selects {x: 5}

Veya ilk doğruluk değerini döndürmek için

var truthyValue = {x: 10};
return truthyValue || {}; // will return {x: 10}

Aynısı, birden çok kez geri dönmek için kullanılabilir:

envVariable || configValue || defaultConstValue // select the first "truthy" of these

Opsiyonel bir işlevi çağırmak için kısa devre

&& operatörü bir geri aramayı değerlendirmek için kullanılabilir, ancak şu durumda iletilir:

function myMethod(cb) {
    // This can be simplified
    if (cb) {
       cb();
    }

    // To this
    cb && cb();
}

Elbette yukarıdaki test, “cb"nin aslında bir “işlev” olduğunu ve yalnızca bir “Nesne”/“Dizi”/“Dize”/“Sayı” olmadığını doğrulamaz.

Soyut Eşitlik (==) #

Soyut eşitlik operatörünün işlenenleri, ortak bir türe dönüştürüldükten sonra _karşılaştırılır. Bu dönüşümün nasıl gerçekleştiği, operatörün belirtimine bağlıdır:

== operatörü için spesifikasyon:

7.2.13 Özet Eşitlik Karşılaştırması

"x" ve "y"nin değerler olduğu "x == y" karşılaştırması "doğru" veya "yanlış" sonucunu verir. Böyle bir karşılaştırma şu şekilde yapılır: 1. "Tür(x)", "Tür(y)" ile aynıysa, o zaman: > **a.** Kesin Eşitlik Karşılaştırması `x === y` gerçekleştirmenin sonucunu döndür. 2. "x" "boş" ve "y" "tanımsız" ise, "true" değerini döndürün. 3. "x" "tanımsız" ve "y" "boş" ise, "true" değerini döndürün. 4. "Tür(x)" "Sayı" ve "Tür(y)" "Dize" ise, "x == ToNumber(y)" karşılaştırmasının sonucunu döndürün. 5. "Tür(x)" "Dize" ve "Tür(y)" "Sayı" ise, "ToNumber(x) == y" karşılaştırmasının sonucunu döndürün. 6. "Tür(x)" "Boolean" ise, "ToNumber(x) == y" karşılaştırmasının sonucunu döndürün. 7. "Tür(y)" "Boolean" ise, "karşılaştırma x == ToNumber(y)" sonucunu döndürün. 8. "Tip(x)", "Dize", "Sayı" veya "Symbol" ve "Tip(y)", "Nesne" ise, "x == ToPrimitive(y)" karşılaştırmasının sonucunu döndürün. 9. "Type(x)" Nesne ise ve "Type(y)" "Dize", "Sayı" veya "Symbol" ise, "ToPrimitive(x) == y" karşılaştırmasının sonucunu döndürün. 10. 'yanlış' döndür.

Örnekler: #

1 == 1;                     // true
1 == true;                  // true  (operand converted to number: true => 1)
1 == '1';                   // true  (operand converted to number: '1' => 1 )
1 == '1.00';                // true
1 == '1.00000000001';       // false
1 == '1.00000000000000001'; // true  (true due to precision loss)
null == undefined;          // true  (spec #2)
1 == 2;                     // false
0 == false;                 // true
0 == undefined;             // false
0 == "";                    // true

Boş ve Tanımsız #

“null” ve “undefined” arasındaki farklar #

’null’ ve ‘undefined’ soyut eşitliği paylaşır ‘==’ ama kesin eşitliği değil ‘===’,

null == undefined   // true
null === undefined  // false

Biraz farklı şeyleri temsil ediyorlar:

  • “tanımsız”, örneğin bir tanımlayıcı/Nesne özelliği oluşturulmadan önce veya tanımlayıcı/Fonksiyon parametresi oluşturma ile varsa ilk seti arasındaki süre gibi bir değerin yokluğunu temsil eder.
  • “null”, önceden oluşturulmuş bir tanımlayıcı veya özellik için kasıtlı bir değerin yokluğunu temsil eder.

Bunlar farklı sözdizimi türleridir:

  • “tanımsız”, genellikle global kapsamda değişmez olan global Nesnenin bir _özelliğidir. Bu, global ad alanı dışında bir tanımlayıcı tanımlayabileceğiniz herhangi bir yerde bu kapsamdan “tanımsız” ifadesini gizleyebileceği anlamına gelir (yine de her şey tanımsız olabilir)
  • ’null’ bir kelime literal‘dir, bu nedenle anlamı asla değiştirilemez ve bunu yapmaya çalışmak bir Hata verir.

’null’ ve ‘undefined’ arasındaki benzerlikler #

“boş” ve “tanımsız” her ikisi de yanlıştır.

if (null) console.log("won't be logged");
if (undefined) console.log("won't be logged");

Ne “boş” ne de “tanımsız” eşittir “yanlış” (bkz. bu soru).

false == undefined   // false
false == null        // false
false === undefined  // false
false === null       // false

’tanımsız’ kullanma #

  • Geçerli kapsam güvenilir değilse, undefined olarak değerlendirilen bir şey kullanın, örneğin void 0;.
  • “undefined” başka bir değer tarafından gölgeleniyorsa, “Array” veya “Number"ın gölgelenmesi kadar kötüdür.
  • Bir şeyi “tanımsız” olarak ayarlamaktan kaçının. Bir bar özelliğini bir Object “foo"dan kaldırmak istiyorsanız, bunun yerine “delete foo.bar;”.
  • ‘undefined’ yerine ‘foo’ varlığı testi tanımlayıcısı bir Referans Hatası verebilir, bunun yerine ‘undefined’’ yerine ’typeof foo’ kullanın.

Booleanlı Mantık Operatörleri #

var x = true,
    y = false;

VE #

Her iki ifade de doğru olarak değerlendirilirse bu operatör true değerini döndürür. Bu boole operatörü kısa devre yapacak ve “x” “yanlış” olarak değerlendirilirse “y"yi değerlendirmeyecektir.

x && y;

“y” yanlış olduğu için bu false döndürür.

VEYA #

Bu operatör, iki ifadeden biri doğru olarak değerlendirilirse true değerini döndürür. Bu boole operatörü kısa devre yapacak ve “x” “doğru” olarak değerlendirilirse “y” değerlendirilmeyecektir.

x || y;

Bu doğru dönecektir, çünkü “x” doğrudur.

OLUMSUZLUK #

Bu operatör, sağdaki ifade doğru olarak değerlendirilirse false döndürür ve sağdaki ifade false olarak değerlendirilirse true döndürür.

!x;

“x” doğru olduğu için bu false döndürür.

Boole olmayan değerlere sahip Mantık Operatörleri (boole zorlaması) #

Mantıksal VEYA (||), soldan sağa okuma, ilk truthy değerini değerlendirecektir. truthy değeri bulunamazsa, son değer döndürülür.

var a = 'hello' || '';             // a = 'hello'
var b = '' || [];                  // b = []
var c = '' || undefined;           // c = undefined
var d = 1 || 5;                    // d = 1
var e = 0 || {};                   // e = {}
var f = 0 || '' || 5;              // f = 5
var g = '' || 'yay' || 'boo';      // g = 'yay'

Soldan sağa doğru okunan mantıksal AND (&&), ilk falsy değerini değerlendirecektir. falsey değeri bulunamazsa, son değer döndürülür.

var a = 'hello' && '';                  // a = ''
var b = '' && [];                       // b = ''
var c = undefined && 0;                 // c = undefined
var d = 1 && 5;                         // d = 5
var e = 0 && {};                        // e = 0
var f = 'hi' && [] && 'done';           // f = 'done'
var g = 'bye' && undefined && 'adios';  // g = undefined

Bu numara, örneğin, bir işlev argümanına (ES6’dan önce) varsayılan bir değer ayarlamak için kullanılabilir.

var foo = function(val) {
    // if val evaluates to falsey, 'default' will be returned instead.
    return val || 'default';
}

console.log( foo('burger') );  // burger
console.log( foo(100) );       // 100
console.log( foo([]) );        // []
console.log( foo(0) );         // default
console.log( foo(undefined) ); // default

Yalnızca argümanlar için “0” ve (daha az ölçüde) boş dizenin de genellikle açıkça iletilebilmesi ve bu kalıpla geçmeyecek olan bir varsayılanı geçersiz kılması gereken geçerli değerler olduğunu unutmayın ( çünkü onlar falsy).

Boş Dizi #

/* ToNumber(ToPrimitive([])) == ToNumber(false) */
[] == false; // true

[].toString() yürütüldüğünde, varsa [].join(), aksi takdirde Object.prototype.toString() çağırır. Bu karşılaştırma ’true’ döndürüyor çünkü ‘[].join()’, ‘0’a zorlandığında false ToNumber değerine eşit olan ’’’’ değerini döndürüyor -262/5.1/#sec-9.3).

Ancak dikkatli olun, tüm nesneler doğrudur ve “Array”, “Object"in bir örneğidir:

// Internally this is evaluated as ToBoolean([]) === true ? 'truthy' : 'falsy'
[] ? 'truthy' : 'falsy'; // 'truthy'

Otomatik Tür Dönüşümleri #

Sayıların yanlışlıkla dizelere veya NaN’ye (Sayı Değil) dönüştürülebileceğine dikkat edin.

JavaScript gevşek yazılmış. Bir değişken, farklı veri türleri içerebilir ve bir değişken, veri türünü değiştirebilir:

var x = "Hello";     // typeof x is a string
x = 5;               // changes typeof x to a number

JavaScript, matematiksel işlemler yaparken sayıları dizelere dönüştürebilir:

var x = 5 + 7;       // x.valueOf() is 12,  typeof x is a number
var x = 5 + "7";     // x.valueOf() is 57,  typeof x is a string
var x = "5" + 7;     // x.valueOf() is 57,  typeof x is a string
var x = 5 - 7;       // x.valueOf() is -2,  typeof x is a number
var x = 5 - "7";     // x.valueOf() is -2,  typeof x is a number
var x = "5" - 7;     // x.valueOf() is -2,  typeof x is a number
var x = 5 - "x";     // x.valueOf() is NaN, typeof x is a number

Bir dizeden bir dize çıkarmak bir hata oluşturmaz, ancak NaN (Sayı Değil) döndürür:

"Hello" - "Dolly"    // returns NaN

Eşitlik karşılaştırma işlemleri #

JavaScript’in dört farklı eşitlik karşılaştırma işlemi vardır.

SameValue #

Her iki işlenen de aynı Türe aitse ve aynı değerdeyse ’true’ değerini döndürür.

Not: Bir nesnenin değeri bir referanstır.

Bu karşılaştırma algoritmasını ‘Object.is’ (ECMAScript 6) aracılığıyla kullanabilirsiniz.

Örnekler:

Object.is(1, 1);            // true
Object.is(+0, -0);          // false
Object.is(NaN, NaN);        // true
Object.is(true, "true");    // false
Object.is(false, 0);        // false
Object.is(null, undefined); // false
Object.is(1, "1");          // false
Object.is([], []);          // false

Bu algoritma bir eşdeğerlik ilişkisi özelliklerine sahiptir:

  • Dönüştürme: “Object.is(x, x)”, “x” herhangi bir değeri için “true” olur
  • Simetri: “Object.is(x, y)”, yalnızca ve yalnızca “Object.is(y, x)” “true” ise, “x” ve “x” değerleri için “true” olur. “y”.
  • Geçişlilik: “Object.is(x, y)” ve “Object.is(y, z)” “true” ise, “Object.is(x, z)” de “true” olur , x, yvez` değerleri için.

SameValueSıfır #

SameValue gibi davranır, ancak “+0” ve “-0"ın eşit olduğunu düşünür.

Bu karşılaştırma algoritmasını Array.prototype.includes (ECMAScript 7) üzerinden kullanabilirsiniz.

Örnekler:

[1].includes(1);            // true
[+0].includes(-0);          // true
[NaN].includes(NaN);        // true
[true].includes("true");    // false
[false].includes(0);        // false
[1].includes("1");          // false
[null].includes(undefined); // false
[[]].includes([]);          // false

Bu algoritma hala bir eşdeğerlik ilişkisi özelliklerine sahiptir:

  • Yansımacılık: “ .includes(x)”, “x” değerleri için “doğru"dur
  • Simetri: “ .includes(y)”, yalnızca ve yalnızca “[y].includes(x)” “doğru” ise, “x” ve tüm değerler için “doğru” olur. “y”.
  • Geçişlilik: “ .includes(y)” ve “[y].includes(z)” “true” ise, “ .includes(z)” de “true” olur , x, yvez` değerleri için.

Kesin Eşitlik Karşılaştırması #

SameValue gibi davranır, ancak

  • ‘+0’ ve ‘-0’ı eşit kabul eder.
  • NaNı kendisi de dahil olmak üzere herhangi bir değerden farklı kabul eder.

Bu karşılaştırma algoritmasını === operatörü (ECMAScript 3) aracılığıyla kullanabilirsiniz.

Ayrıca, === sonucunu olumsuzlayan !== operatörü (ECMAScript 3) vardır.

Örnekler:

1 === 1;            // true
+0 === -0;          // true
NaN === NaN;        // false
true === "true";    // false
false === 0;        // false
1 === "1";          // false
null === undefined; // false
[] === [];          // false

Bu algoritma aşağıdaki özelliklere sahiptir:

  • Simetri: “x === y”, “x” ve “y” değerleri için yalnızca ve yalnızca y === x’ “doğru” ise “doğru” olur.
  • Geçişlilik: “x === y” ve “y === z” “doğru” ise, “x” değerleri için “x === z” de “doğru” olur, “y” ve “z”.

Ama bir denklik bağıntısı değildir çünkü

  • NaN [yansıma] değildir3: NaN !== NaN

Soyut Eşitlik Karşılaştırması #

Her iki işlenen de aynı Türe aitse, Kesin Eşitlik Karşılaştırması gibi davranır.

Aksi takdirde, onları şu şekilde zorlar:

  • “tanımsız” ve “boş” eşit kabul edilir
  • Bir sayıyı bir dizeyle karşılaştırırken, dize bir sayıya zorlanır
  • Bir boole değerini başka bir şeyle karşılaştırırken, boole değeri bir sayıya zorlanır
  • Bir nesneyi bir sayı, dize veya sembolle karşılaştırırken, nesne bir ilkel olmaya zorlanır.

Bir zorlama varsa, zorlanan değerler özyinelemeli olarak karşılaştırılır. Aksi takdirde algoritma “false” döndürür.

Bu karşılaştırma algoritmasını == operatörü (ECMAScript 1) aracılığıyla kullanabilirsiniz.

Ayrıca, ‘==’ sonucunu olumsuzlayan ‘!=’ operatörü (ECMAScript 1) vardır.

Örnekler:

1 == 1;            // true
+0 == -0;          // true
NaN == NaN;        // false
true == "true";    // false
false == 0;        // true
1 == "1";          // true
null == undefined; // true
[] == [];          // false

Bu algoritma aşağıdaki özelliğe sahiptir:

  • Simetri: “x == y”, “x” ve “y” değerleri için yalnızca “y == x” “doğru” ise “doğru” olur.

Ama bir denklik bağıntısı değildir çünkü

  • NaN [yansıma] değildir3: NaN != NaN
  • Geçişlilik tutmaz, ör. 0 == '' ve 0 == '0', ancak '' != '0'

İlişkisel operatörler (<, <=, >, >=) #

Her iki işlenen de sayısal olduğunda normal olarak karşılaştırılır:

1 < 2        // true
2 <= 2       // true
3 >= 5       // false
true < false // false (implicitly converted to numbers, 1 > 0)

Her iki işlenen de dize olduğunda, sözlükbilimsel olarak karşılaştırılır (alfabetik sıraya göre):

'a' < 'b'    // true
'1' < '2'    // true
'100' > '12' // false ('100' is less than '12' lexicographically!)

Bir işlenen bir dize ve diğeri bir sayı olduğunda, karşılaştırmadan önce dize bir sayıya dönüştürülür:

'1' < 2      // true
'3' > 2      // true
true > '2'   // false (true implicitly converted to number, 1 < 2)

Dize sayısal olmadığında, sayısal dönüştürme “NaN” (sayı değil) döndürür. “NaN” ile karşılaştırmak her zaman “yanlış” döndürür:

1 < 'abc'    // false
1 > 'abc'    // false

Ancak sayısal bir değeri “null”, “undefined” veya boş dizelerle karşılaştırırken dikkatli olun:

1 > ''        // true
1 < ''        // false
1 > null      // true
1 < null      // false
1 > undefined // false
1 < undefined // false

Bir işlenen bir nesne ve diğeri bir sayı olduğunda, nesne karşılaştırmadan önce bir sayıya dönüştürülür. Bu nedenle “null” özel bir durumdur çünkü “Sayı(null);//0”

new Date(2015)< 1479480185280            // true
null > -1                                //true
({toString:function(){return 123}}) > 122  //true

Eşitsizlik #

Operatör !=, == operatörünün tersidir. İşlenenler eşit değilse ’true’ değerini döndürür. Javascript motoru, aynı türde değillerse, her iki işleneni de eşleşen türlere dönüştürmeye çalışacaktır. Not: İki işlenenin bellekte farklı dahili referansları varsa, ‘false’ döndürülür.

Örneklem:

1 != '1'     // false
1 != 2       // true

Yukarıdaki örnekte, ‘1 != ‘1’ ‘yanlış’tır, çünkü ilkel bir sayı türü bir ‘char’ değeriyle karşılaştırılır. Bu nedenle, Javascript motoru, R.H.S değerinin veri tipini umursamaz.

Operatör: !==, === operatörünün tersidir. İşlenenler eşit değilse veya türleri eşleşmiyorsa true değerini döndürür.

Örnek:

1 !== '1'    // true
1 !== 2      // true
1 !== 1      // false

Birden çok mantıksal ifadeyi gruplama #

Özellikle if ifadelerinde faydalı olan daha karmaşık bir mantık değerlendirmesi oluşturmak için birden fazla boole mantıksal ifadesini parantez içinde gruplayabilirsiniz.

if ((age >= 18 && height >= 5.11) || (status === 'royalty' && hasInvitation)) {
  console.log('You can enter our club');
}

İfadeyi biraz daha kısa ve açıklayıcı hale getirmek için gruplandırılmış mantığı değişkenlere de taşıyabiliriz:

var isLegal = age >= 18;
var tall = height >= 5.11;
var suitable = isLegal && tall;
var isRoyalty = status === 'royalty';
var specialCase = isRoyalty && hasInvitation;
var canEnterOurBar = suitable || specialCase;

if (canEnterOurBar) console.log('You can enter our club');

Bu özel örnekte (ve diğer birçok örnekte), ifadeleri parantez içinde gruplamanın, onları kaldırmışız gibi çalıştığına dikkat edin, sadece doğrusal bir mantık değerlendirmesini takip edin ve kendinizi aynı sonuçla bulacaksınız. Ne amaçladığımı daha net anlamamı sağladığı ve mantık hatalarını önleyebileceği için parantez kullanmayı tercih ediyorum.

Çoklu durum verilerinin karşılaştırmasını optimize etmek için bit alanları #

Bir bit alanı, çeşitli boole durumlarını ayrı bitler olarak tutan bir değişkendir. Biraz açık doğruyu temsil eder ve kapalı yanlış olur. Geçmişte bit alanları, bellekten tasarruf sağladığı ve işlem yükünü azalttığı için rutin olarak kullanılıyordu. Bit alanını kullanma ihtiyacı artık o kadar önemli olmasa da, birçok işleme görevini basitleştirebilecek bazı avantajlar sunuyorlar.

Örneğin kullanıcı girişi. Bir klavyenin yukarı, aşağı, sol, sağ yön tuşlarından girdi alırken, çeşitli tuşları, her yöne bir bit atanmış olarak tek bir değişkene kodlayabilirsiniz.

Bitfield aracılığıyla örnek okuma klavyesi

var bitField = 0;  // the value to hold the bits
const KEY_BITS = [4,1,8,2]; // left up right down
const KEY_MASKS = [0b1011,0b1110,0b0111,0b1101]; // left up right down
window.onkeydown = window.onkeyup = function (e) {
    if(e.keyCode >= 37 && e.keyCode <41){
        if(e.type === "keydown"){
            bitField |= KEY_BITS[e.keyCode - 37];
        }else{
            bitField &= KEY_MASKS[e.keyCode - 37];
        }
    }    
}
