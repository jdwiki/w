Zaman uyumsuz işlevler (zaman uyumsuzbeklemede)

“async” ve “await”, satır içi eşzamansız eylemleri ifade etmek için vaatlerin ve oluşturucuların üzerine kuruludur. Bu, eşzamansız veya geri arama kodunun bakımını çok daha kolay hale getirir.

“async” anahtar sözcüğüne sahip işlevler bir “Söz” döndürür ve bu sözdizimi ile çağrılabilir.

Bir ‘async işlevi’ içinde ‘await’ anahtar sözcüğü herhangi bir ‘Söz’e uygulanabilir ve söz çözüldükten sonra ‘bekleme’den sonra tüm işlev gövdesinin yürütülmesine neden olur.

Sözdizimi #

  • zaman uyumsuz işlev foo() {    …
       await asyncCall()
    }
  • zaman uyumsuz işlev() { … }
  • zaman uyumsuz() => { … }
  • (zaman uyumsuz () => {    const data = await asyncCall()
       console.log(data) })()

Zaman uyumsuz işlevler, vaatler ve oluşturucular üzerinde sözdizimsel bir şekerdir. Kodunuzu daha okunabilir, bakımı yapılabilir, hataları yakalaması daha kolay ve daha az girinti düzeyiyle yapmanıza yardımcı olurlar.

Giriiş #

“async” olarak tanımlanan bir fonksiyon, asenkron eylemler gerçekleştirebilen ancak yine de senkronize görünen bir fonksiyondur. Bunun yapılma şekli, bir Söz‘in çözümlenmesini veya reddetmesini beklerken işlevi ertelemek için ‘await’ anahtar sözcüğünü kullanmaktır.

Not: Zaman uyumsuz işlevler 4. Aşama (“Bitmiş”) önerisidir ECMAScript 2017 standardı.

Örneğin, söze dayalı Getirme API’si kullanarak:

async function getJSON(url) {
    try {
        const response = await fetch(url);
        return await response.json();
    }
    catch (err) {
        // Rejections in the promise will get thrown here
        console.error(err.message);
    }
}

Zaman uyumsuz bir işlev her zaman bir Sözün kendisini döndürür, böylece onu diğer zaman uyumsuz işlevlerde kullanabilirsiniz.

Ok işlev stili #

const getJSON = async url => {
    const response = await fetch(url);
    return await response.json();
}
son sürüm

Await ve operatör önceliği #

‘await’ anahtar sözcüğünü kullanırken operatör önceliğini göz önünde bulundurmalısınız.

Başka bir eşzamansız işlevi çağıran eşzamansız bir işlevimiz olduğunu hayal edin, ‘Unicorn’ sınıfının bir örneğine çözümlenen bir Söz döndüren ‘getUnicorn()’. Şimdi o sınıfın getSize() yöntemini kullanarak tek boynuzlu atın boyutunu elde etmek istiyoruz.

Aşağıdaki koda bakın:

async function myAsyncFunction() {
    await getUnicorn().getSize();
}

İlk bakışta, geçerli gibi görünüyor, ama değil. Operatör önceliği nedeniyle, aşağıdakine eşdeğerdir:

async function myAsyncFunction() {
    await (getUnicorn().getSize());
}

Burada Promise nesnesinin getSize() yöntemini çağırmaya çalışıyoruz, bu bizim istediğimiz şey değil.

Bunun yerine, önce tek boynuzlu atı beklemek istediğimizi belirtmek için parantez kullanmalı ve ardından sonucun getSize() yöntemini çağırmalıyız:

async function asyncFunction() {
    (await getUnicorn()).getSize();
}

Tabii ki. örneğin, “getUnicorn()” işlevi eşzamanlıysa, ancak “getSize()” yöntemi eşzamansızsa, önceki sürüm bazı durumlarda geçerli olabilir.

Vaatlerle karşılaştırıldığında zaman uyumsuz işlevler #

“async” işlevleri “Promise” türünün yerini almaz; vaatlerin çağrılmasını kolaylaştıran dil anahtar sözcükleri eklerler. Değiştirilebilirler:

async function doAsyncThing() { ... }

function doPromiseThing(input) { return new Promise((r, x) => ...); }

// Call with promise syntax
doAsyncThing()
    .then(a => doPromiseThing(a))
    .then(b => ...)
    .catch(ex => ...);

// Call with await syntax
try {
    const a = await doAsyncThing();
    const b = await doPromiseThing(a);
    ...
}
catch(ex) { ... }

Söz zincirlerini kullanan herhangi bir işlev, ‘bekliyor’ kullanılarak yeniden yazılabilir:

function newUnicorn() {
  return fetch('unicorn.json')                     // fetch unicorn.json from server
  .then(responseCurrent => responseCurrent.json()) // parse the response as JSON
  .then(unicorn =>
    fetch('new/unicorn', {                         // send a request to 'new/unicorn' 
        method: 'post',                            // using the POST method
        body: JSON.stringify({unicorn})            // pass the unicorn to the request body
    })
  )
  .then(responseNew => responseNew.json())
  .then(json => json.success)                      // return success property of response
  .catch(err => console.log('Error creating unicorn:', err));
 }

İşlev, “async” / “await” kullanılarak aşağıdaki gibi yeniden yazılabilir:

async function newUnicorn() {
  try {
    const responseCurrent = await fetch('unicorn.json'); // fetch unicorn.json from server
    const unicorn = await responseCurrent.json();        // parse the response as JSON
    const responseNew = await fetch('new/unicorn', {     // send a request to 'new/unicorn'
      method: 'post',                                    // using the POST method
      body: JSON.stringify({unicorn})                    // pass the unicorn to the request body
    });
    const json = await responseNew.json();
    return json.success                                  // return success property of response
  } catch (err) {
    console.log('Error creating unicorn:', err);
  }
}

“newUnicorn()“un bu “eşzamansız” varyantı bir “Söz” veriyor gibi görünüyor, ancak gerçekten birden çok “bekleme” anahtar kelimesi vardı. Her biri bir ‘Söz’ verdi, yani gerçekten bir zincirden ziyade bir vaatler koleksiyonumuz vardı.

Aslında bunu, her bir ‘bekleme’nin bir ‘yeni bir getiri vaadi’ olduğu bir ‘fonksiyon*’ üreteci olarak düşünebiliriz. Bununla birlikte, işlevi devam ettirmek için her sözün sonuçlarına bir sonraki tarafından ihtiyaç duyulur. Bu nedenle, Javascript’e bu yineleme için otomatik olarak bir gözlemci oluşturmasını söylediğinden, işlevde ek olarak ‘async’ anahtar kelimesine (ve ayrıca vaatleri çağırırken ‘await’ anahtar kelimesine) ihtiyaç duyulur. “async function newUnicorn()” tarafından döndürülen “Promise”, bu yineleme tamamlandığında çözülür.

Pratik olarak, bunu düşünmenize gerek yok; ‘await’ vaadi gizler ve ‘async’ jeneratör yinelemesini gizler.

“async” işlevlerini sanki vaatlermiş gibi çağırabilir ve herhangi bir sözü ya da herhangi bir “async” işlevini “bekleyebilirsiniz”. Bir ‘.then()’ olmadan bir sözü yerine getirebildiğiniz gibi, bir zaman uyumsuz işlevi ‘beklemeniz’ gerekmez.

Bu kodu hemen yürütmek istiyorsanız, bir “async” IIFE de kullanabilirsiniz:

(async () => {
  await makeCoffee()
  console.log('coffee is ready!')
})()

Zaman uyumsuz bekleme ile döngü #

Döngülerde zaman uyumsuz bekleme kullanırken, bu sorunlardan bazılarıyla karşılaşabilirsiniz.

‘ForEach’ içinde beklemeyi kullanmayı denerseniz, bu bir ‘Beklenmeyen belirteç’ hatası verecektir.

(eşzamansız() => {
veri = [1, 2, 3, 4, 5];
data.forEach(e => {
   const i = await somePromiseFn(e);
   console.log(i);
 });
})();

Bu, ok işlevini yanlışlıkla bir blok olarak görmüş olmanızdan kaynaklanmaktadır. “bekliyor”, “async” olmayan geri arama işlevi bağlamında olacaktır. Yorumlayıcı bizi yukarıdaki hatayı yapmaktan korur, ancak ‘forEach’ geri çağrısına ‘async’ eklerseniz hiçbir hata oluşmaz. Bunun sorunu çözdüğünü düşünebilirsiniz, ancak beklendiği gibi çalışmayacaktır.

Örnek:

(eşzamansız() => {
veri = [1, 2, 3, 4, 5];
data.forEach(async(e) => {
    const i = await somePromiseFn(e);
    console.log(i);
  });
konsol.log('önce bu yazdırılacak');
})();

Bunun nedeni, geri arama eşzamansız işlevinin üst eşzamansız işlevi değil, yalnızca kendisini duraklatabilmesidir.

Bir söz veren bir asyncForEach işlevi yazabilir ve ardından şöyle bir şey yapabilirsiniz: await asyncForEach(async (e) => wait somePromiseFn(e), data ) Temel olarak, tüm geri aramalar beklendiğinde ve yapıldığında çözülen bir söz verirsiniz. Ama bunu yapmanın daha iyi yolları var ve bu sadece bir döngü kullanmak.

Bir “for-of” döngüsü veya bir “for/while” döngüsü kullanabilirsiniz, hangisini seçtiğiniz gerçekten önemli değil.

(eşzamansız() => {
veri = [1, 2, 3, 4, 5];
for (veri e olsun) {
    const i = await somePromiseFn(e);
    console.log(i);
  }
konsol.log('bu en son yazdırılacak');
})();

Ama başka bir yakalama var. Bu çözüm, bir sonrakini yinelemeden önce “somePromiseFn"ye yapılan her çağrının tamamlanmasını bekleyecektir. Bu, ‘somePromiseFn’ çağrılarınızın sırayla yürütülmesini istiyorsanız harikadır, ancak aynı anda çalışmasını istiyorsanız, ‘Promise.all’ üzerinde ‘beklemeniz’ gerekir.

(eşzamansız() => {
veri = [1, 2, 3, 4, 5];
const p = bekle Promise.all(data.map(async(e) => bekle bazıPromiseFn(e)));
konsol.log(...p);
})();

“Promise.all”, tek parametresi olarak bir dizi söz alır ve bir söz döndürür. Dizideki tüm vaatler çözüldüğünde, döndürülen söz de çözülür. Bu sözü ‘bekliyoruz’ ve çözüldüğünde tüm değerlerimiz mevcut.

Yukarıdaki örnekler tamamen çalıştırılabilir. “somePromiseFn” işlevi, zaman aşımına sahip bir zaman uyumsuz yankı işlevi olarak yapılabilir. babel-repl içindeki örnekleri en azından stage-3 ön ayarıyla deneyebilir ve çıktıya bakabilirsiniz.

function somePromiseFn(n) {
yeni Söz ver((res, red) => {
   setTimeout(() => res(n), 250);
 });
}

Daha az girinti #

vaatlerle:

function doTheThing() {
    return doOneThing()
        .then(doAnother)
        .then(doSomeMore)
        .catch(handleErrors)
}

Zaman uyumsuz işlevlerle:

async function doTheThing() {
    try {
        const one = await doOneThing();
        const another = await doAnother(one);
        return await doSomeMore(another);
    } catch (err) {
        handleErrors(err);
    }
}

Geri dönüşün nasıl üstte değil de altta olduğuna dikkat edin ve dilin yerel hata işleme mekaniğini (‘dene/yakala’) kullanın.

Eşzamanlı zaman uyumsuz (paralel) işlemler #

Genellikle eşzamansız işlemleri paralel olarak gerçekleştirmek isteyeceksiniz. “async”/“await” teklifinde bunu destekleyen doğrudan bir sözdizimi vardır, ancak “await” bir söz için bekleyeceğinden, birden çok sözü “Promise.all” içinde onları beklemek için bir araya getirebilirsiniz:

// Not in parallel

async function getFriendPosts(user) {
    friendIds = await db.get("friends", {user}, {id: 1});
    friendPosts = [];
    for (let id in friendIds) {
        friendPosts = friendPosts.concat( await db.get("posts", {user: id}) );
    }
    // etc.
}

Bu, her bir arkadaşın gönderilerini seri olarak almak için her sorguyu yapacaktır, ancak bunlar aynı anda yapılabilir:

// In parallel

async function getFriendPosts(user) {
    friendIds = await.db.get("friends", {user}, {id: 1});
    friendPosts = await Promise.all( friendIds.map(id => 
      db.get("posts", {user: id})
    );
    // etc.
}

Bu, bir dizi vaat oluşturmak için kimlikler listesinin üzerinden geçecektir. ‘await’, all vaatlerinin tamamlanmasını bekleyecektir. “Promise.all” bunları tek bir sözde birleştirir, ancak bunlar paralel olarak yapılır.
