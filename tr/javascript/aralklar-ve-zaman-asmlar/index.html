Sözdizimi #

  • timeoutID = setTimeout(function() {}, milisaniye)
  • intervalID = setInterval(function() {}, milisaniye)
  • timeoutID = setTimeout(function() {}, milisaniye, parametre, parametre, …)
  • intervalID = setInterval(function() {}, milisaniye, parametre, parametre, …)
  • clearTimeout (zaman aşımı kimliği)
  • clearInterval(intervalID)

Gecikme belirtilmemişse, varsayılan olarak 0 milisaniyedir. Ancak asıl gecikme bundan daha uzun olacak; örneğin, HTML5 özelliği minimum 4 milisaniyelik bir gecikme belirtir.

‘setTimeout’ sıfır gecikme ile çağrıldığında bile, ‘setTimeout’ tarafından çağrılan işlev eşzamansız olarak yürütülecektir.

İşlemi yapıp bir sonraki kod cümlesine geçseniz bile DOM manipülasyonu gibi birçok işlemin mutlaka tamamlanmayacağını unutmayın, bu nedenle bunların eşzamanlı olarak çalışacaklarını varsaymamalısınız.

“setTimeout(someFunc, 0)” kullanımı, geçerli JavaScript motorunun çağrı yığınının sonunda “someFunc” işlevinin yürütülmesini sıraya koyar, bu nedenle işlev, bu işlemler tamamlandıktan sonra çağrılır.

İşlev (setTimeout(function(){some..code}, 1000)) yerine JavaScript kodunu (setTimeout("some..code", 1000)) içeren bir dize iletmek mümkündür. . Kod bir dizgeye yerleştirilirse, daha sonra eval() kullanılarak ayrıştırılacaktır. Dize tarzı zaman aşımları performans, netlik ve bazen güvenlik nedenleriyle önerilmez, ancak bu stili kullanan daha eski kodlar görebilirsiniz. Geçiş işlevleri, Netscape Navigator 4.0 ve Internet Explorer 5.0’dan beri desteklenmektedir.

Özyinelemeli setTimeout #

Bir işlevi süresiz olarak tekrarlamak için setTimeout yinelemeli olarak çağrılabilir:

function repeatingFunc() {
    console.log("It's been 5 seconds. Execute the function again.");
    setTimeout(repeatingFunc, 5000);
}

setTimeout(repeatingFunc, 5000);

‘setInterval’den farklı olarak, bu, işlevin çalışma süresi belirtilen gecikmeden daha uzun olsa bile işlevin yürütülmesini sağlar. Ancak, işlev yürütmeleri arasında düzenli bir aralık garanti etmez. Bu davranış, aynı zamanda, ‘setTimeout’a özyinelemeli çağrıdan önceki bir istisna, bunun yeniden tekrarlanmasını önleyeceğinden, ‘setInterval’, istisnalardan bağımsız olarak süresiz olarak tekrar edeceğinden de değişir.

Aralıklar #

function waitFunc(){
    console.log("This will be logged every 5 seconds");
}

window.setInterval(waitFunc,5000);

Aralıkları kaldırma #

“window.setInterval()”, o aralığın çalışmaya devam etmesini durdurmak için kullanılabilen bir “IntervalID” döndürür. Bunu yapmak için, “window.setInterval()” dönüş değerini bir değişkende saklayın ve tek argüman olarak bu değişkenle “clearInterval()” öğesini çağırın:

function waitFunc(){
    console.log("This will be logged every 5 seconds");
}    

var interval = window.setInterval(waitFunc,5000);

window.setTimeout(function(){
    clearInterval(interval);
},32000);

Bu, her 5 saniyede bir Bu, her 5 saniyede bir günlüğe kaydedilecek şeklinde günlüğe kaydeder, ancak 32 saniye sonra durdurur. Böylece mesajı 6 kez günlüğe kaydeder.

Zaman aşımlarını kaldırma #

“window.setTimout()”, bu zaman aşımının çalışmasını durdurmak için kullanılabilecek bir “TimeoutID” döndürür. Bunu yapmak için, “window.setTimeout()“un dönüş değerini bir değişkende saklayın ve tek argüman olarak bu değişkenle “clearTimeout()“u çağırın:

function waitFunc(){
    console.log("This will not be logged after 5 seconds");
}
function stopFunc(){
    clearTimeout(timeout);
}

var timeout = window.setTimeout(waitFunc,5000);
window.setTimeout(stopFunc,3000);

Bu, zamanlayıcı 3 saniye sonra durdurulduğu için mesajı günlüğe kaydetmez.

Aralıklar #

Standart

Değişkeni oluşturmanız gerekmez, ancak o anda çalışan aralığı durdurmak için bu değişkeni clearInterval ile kullanabileceğiniz için bu iyi bir uygulamadır.

var int = setInterval("doSomething()", 5000 ); /* 5 seconds */
var int = setInterval(doSomething, 5000 ); /* same thing, no quotes, no parens */

doSomething işlevine parametreler iletmeniz gerekiyorsa, bunları setInterval öğesine ilk ikisinin ötesinde ek parametreler olarak iletebilirsiniz.

Üst üste binmeden

setInterval, yukarıdaki gibi, ne olursa olsun her 5 saniyede bir (veya ne ayarladıysanız) çalışacaktır. doSomething işlevinin çalışması 5 saniyeden uzun sürse bile. Bu sorun yaratabilir. Sadece doSomething’in çalışmaları arasında bir duraklama olduğundan emin olmak istiyorsanız, şunu yapabilirsiniz:

(function(){

   doSomething();

   setTimeout(arguments.callee, 5000);

})()

setTimeout, işlem sırası, clearTimeout #

setZaman aşımı #

  • Belirtilen sayıda milisaniye bekledikten sonra bir işlevi yürütür.
  • bir işlevin yürütülmesini geciktirmek için kullanılır.

Sözdizimi : setTimeout(işlev, milisaniye) veya window.setTimeout(işlev, milisaniye)

Örnek : Bu örnek, 1 saniye sonra konsola “merhaba” verir. İkinci parametre milisaniye cinsindendir, yani 1000 = 1 sn, 250 = 0.25 sn, vb.

setTimeout(function() {
    console.log('hello');
}, 1000);

setTimeout ile ilgili sorunlar #

for döngüsünde “setTimeout” yöntemini kullanıyorsanız :

for (i = 0; i < 3; ++i) {
  setTimeout(function(){
    console.log(i);
  }, 500);
}

Bu, doğru olmayan “3 üç” değerini verir.

Bu sorunun geçici çözümü:

for (i = 0; i < 3; ++i) {
  setTimeout(function(j){
    console.log(i);
  }(i), 1000);
}

‘0’, ‘1’, ‘2’ değerini verir. Burada, ‘i’yi fonksiyona parametre(‘j’) olarak geçiriyoruz.

Operasyonların sırası #

Buna ek olarak, Javascript’in tek iş parçacıklı olması ve global bir olay döngüsü kullanması nedeniyle, setTimeout sıfır gecikmeyle ‘setTimeout’u çağırarak yürütme kuyruğunun sonuna bir öğe eklemek için ‘setTimeout’ kullanılabilir. Örneğin:

setTimeout(function() {
    console.log('world');
}, 0);

console.log('hello');

Aslında çıktı verecek:

hello
world

Ayrıca, burada sıfır milisaniye, setTimeout içindeki işlevin hemen yürütüleceği anlamına gelmez. Yürütme kuyruğunda kalan yürütülecek öğelere bağlı olarak bundan biraz daha fazla zaman alacaktır. Bu sadece kuyruğun sonuna itilir.

Zaman aşımını iptal etme #

clearTimeout() : setTimeout() içinde belirtilen işlevin yürütülmesini durdurur

Sözdizimi: clearTimeout(timeoutVariable) veya window.clearTimeout(timeoutVariable)

Örnek :

var timeout = setTimeout(function() {
    console.log('hello');
}, 1000);

clearTimeout(timeout); // The timeout will no longer be executed
