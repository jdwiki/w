JSON (JavaScript Object Notation), hafif bir veri değişim biçimidir. İnsanların okuması ve yazması, makinelerin ayrıştırması ve üretmesi kolaydır. JavaScript’te JSON’un bir nesne değil bir dize olduğunu anlamak önemlidir.

Birçok farklı programlama dilinde standardın uygulamalarına bağlantılar da içeren json.org web sitesinde temel bir genel bakış bulunabilir.

Sözdizimi #

  • JSON.parse(girdi[, canlandırma])
  • JSON.stringify(değer[, replacer[, boşluk]])

Parametreler #

parametre Ayrıntılar
JSON.parse JSON dizesini ayrıştırma
giriş(dize) Ayrıştırılacak JSON dizesi.
‘canlandırıcı(işlev)’ Giriş JSON dizesi için bir dönüşüm öngörür.
JSON.stringify Seri hale getirilebilir bir değeri seri hale getirin
değer(dize) JSON belirtimine göre seri hale getirilecek değer.
replacer(function veya String[] veya Number[]) “değer” nesnesinin belirli özelliklerini seçici olarak içerir.
space(String veya Number) Bir “sayı” sağlanırsa, okunabilirlik için “boşluk” sayısı eklenecektir. Bir “dize” sağlanmışsa, dize (ilk 10 karakter) boşluk olarak kullanılacaktır.

JSON yardımcı program yöntemleri ilk olarak ECMAScript 5.1 §15.12‘de standartlaştırıldı.

Biçim, daha sonra JSON Veri Değişim Biçimi’nde (RFC 7158 Mart 2013, [ECMA-404)] güncellenen JSON için uygulama/json Ortam Türü’nde (RFC 4627 Temmuz 2006) tanımlanmıştır. 1] Ekim 2013 ve RFC 7159 Mart 2014).

Bu yöntemleri Internet Explorer 8 gibi eski tarayıcılarda kullanılabilir hale getirmek için Douglas Crockford’un json2.js dosyasını kullanın.

Bir canlandırma işleviyle ayrıştırma #

Ayrıştırılan değeri filtrelemek veya dönüştürmek için bir reviver işlevi kullanılabilir.

var jsonString = '[{"name":"John","score":51},{"name":"Jack","score":17}]';

var data = JSON.parse(jsonString, function reviver(key, value) {
  return key === 'name' ? value.toUpperCase() : value;
});
son sürüm 
const jsonString = '[{"name":"John","score":51},{"name":"Jack","score":17}]';

const data = JSON.parse(jsonString, (key, value) =>
  key === 'name' ? value.toUpperCase() : value
);
son sürüm

Bu, aşağıdaki sonucu üretir:

[
  {
    'name': 'JOHN',
    'score': 51
  },
  {
    'name': 'JACK',
    'score': 17
  }
]

Bu, JSON ile iletilirken serileştirilmesi/kodlanması gereken verilerin gönderilmesi gerektiğinde özellikle yararlıdır, ancak seri hale getirilmiş/kodu çözülmüş verilere erişmek istenir. Aşağıdaki örnekte, ISO 8601 gösterimine bir tarih kodlanmıştır. Bunu bir JavaScript ‘Date’ içinde ayrıştırmak için reviver işlevini kullanıyoruz.

var jsonString = '{"date":"2016-01-04T23:00:00.000Z"}';

var data = JSON.parse(jsonString, function (key, value) {
  return (key === 'date') ? new Date(value) : value;
});
son sürüm 
const jsonString = '{"date":"2016-01-04T23:00:00.000Z"}';

const data = JSON.parse(jsonString, (key, value) =>
  key === 'date' ? new Date(value) : value
);
son sürüm

Reviver işlevinin her yinelemenin sonunda yararlı bir değer döndürdüğünden emin olmak önemlidir. Reviver işlevi ‘undefined’ değerini döndürürse, değer yoksa veya işlevin sonuna doğru yürütme düşerse, özellik nesneden silinir. Aksi takdirde, özellik dönüş değeri olarak yeniden tanımlanır.

Bir değeri seri hale getirme #

Bir JavaScript değeri, “JSON.stringify” işlevi kullanılarak bir JSON dizesine dönüştürülebilir.

JSON.stringify(value[, replacer[, space]])
  1. “değer” JSON dizesine dönüştürülecek değer.
/* Boolean */  JSON.stringify(true)             // 'true'
/* Number  */  JSON.stringify(12)               // '12'
/* String  */  JSON.stringify('foo')            // '"foo"'
/* Object  */  JSON.stringify({})               // '{}'
               JSON.stringify({foo: 'baz'})     // '{"foo": "baz"}'
/* Array   */  JSON.stringify([1, true, 'foo']) // '[1, true, "foo"]'
/* Date    */  JSON.stringify(new Date())       // '"2016-08-06T17:25:23.588Z"'
/* Symbol  */  JSON.stringify({x:Symbol()})     // '{}'
  1. ‘replacer’ JSON dizgisine dahil edilecek değer nesnesinin özelliklerini filtrelemek için beyaz liste görevi gören String ve Number nesnelerinden oluşan bir diziyi veya stringleştirme işleminin davranışını değiştiren bir işlev. Bu değer boşsa veya sağlanmazsa, nesnenin tüm özellikleri elde edilen JSON dizesine dahil edilir.
// replacer as a function
function replacer (key, value) {
    // Filtering out properties
    if (typeof value === "string") {
        return
    }    
    return value
}

var foo = { foundation: "Mozilla", model: "box", week: 45, transport: "car", month: 7 }
JSON.stringify(foo, replacer)
// -> '{"week": 45, "month": 7}'

// replacer as an array
JSON.stringify(foo, ['foundation', 'week', 'month'])
// -> '{"foundation": "Mozilla", "week": 45, "month": 7}'
// only the `foundation`, `week`, and `month` properties are kept
  1. boşluk Okunabilirlik için girinti için kullanılan boşluk sayısı üçüncü parametre olarak belirtilebilir.
JSON.stringify({x: 1, y: 1}, null, 2)  // 2 space characters will be used for indentation
/* output:
    {
      'x': 1,
      'y': 1
    }
*/

Alternatif olarak, girinti için kullanılacak bir dize değeri sağlanabilir. Örneğin, \t' geçmek, sekme karakterinin girinti için kullanılmasına neden olur.

JSON.stringify({x: 1, y: 1}, null, '\t')
/* output:
    {
        'x': 1,
        'y': 1
    }
*/

Sınıf örneklerini seri hale getirme ve geri yükleme #

JSON’da kendi sınıfınızın örneklerini iletmek için özel bir “toJSON” yöntemi ve canlandırma işlevi kullanabilirsiniz. Bir nesnenin “toJSON” yöntemi varsa, nesnenin kendisi yerine sonucu serileştirilir.

function Car(color, speed) {
  this.color = color;
  this.speed = speed;
}

Car.prototype.toJSON = function() {
  return {
    $type: 'com.example.Car',
    color: this.color,
    speed: this.speed
  };
};

Car.fromJSON = function(data) {
  return new Car(data.color, data.speed);
};
son sürüm 
class Car {
  constructor(color, speed) {
    this.color = color;
    this.speed = speed;
    this.id_ = Math.random();
  }

  toJSON() {
    return {
      $type: 'com.example.Car',
      color: this.color,
      speed: this.speed
    };
  }

  static fromJSON(data) {
    return new Car(data.color, data.speed);
  }
}
son sürüm 
var userJson = JSON.stringify({
  name: "John",
  car: new Car('red', 'fast')
});

Bu, aşağıdaki içeriğe sahip bir dize üretir:

{"name":"John","car":{"$type":"com.example.Car","color":"red","speed":"fast"}}

var userObject = JSON.parse(userJson, function reviver(key, value) {
  return (value && value.$type === 'com.example.Car') ? Car.fromJSON(value) : value;
});

Bu, aşağıdaki nesneyi üretir:

{
  name: "John",
  car: Car {
    color: "red",
    speed: "fast",
    id_: 0.19349242527065402
  }
}
son sürüm

JavaScript değişmezlerine karşı JSON #

JSON, “JavaScript Object Notation” anlamına gelir, ancak JavaScript değildir. Bunu yalnızca bir JavaScript değişmezi olarak olabilen bir veri serileştirme biçimi olarak düşünün. Ancak, harici bir kaynaktan alınan JSON’un doğrudan çalıştırılması (yani “eval()” aracılığıyla) önerilmez. İşlevsel olarak, JSON, XML veya YAML’den çok farklı değildir - JSON, JavaScript’e çok benzeyen bir serileştirme biçimi olarak hayal edilirse, bazı karışıklıklar önlenebilir.

Ad yalnızca nesneleri ifade etse de ve bir tür API aracılığıyla kullanım durumlarının çoğu her zaman nesneler ve diziler olsa da, JSON yalnızca nesneler veya diziler için değildir. Aşağıdaki ilkel türler desteklenir:

  • Dize (ör. "Merhaba Dünya!")
  • Sayı (ör. “42”)
  • Boole (ör. “true”)
  • “boş” değeri

“tanımsız”, serileştirme üzerine JSON’dan tanımsız bir özelliğin çıkarılacağı anlamında desteklenmez. Bu nedenle, JSON’u seri durumdan çıkarmanın ve değeri ‘undefined’ olan bir özellik elde etmenin bir yolu yoktur.

"42" dizesi geçerli JSON’dur. JSON’un her zaman "{...}" veya "[...]" dış zarfına sahip olması gerekmez.

Nome JSON da geçerli JavaScript ve bazı JavaScript’ler de geçerli JSON olsa da, her iki dil arasında bazı ince farklar vardır ve her iki dil de diğerinin alt kümesi değildir.

Aşağıdaki JSON dizesini örnek olarak alın:

{"color": "blue"}

Bu doğrudan JavaScript’e eklenebilir. Sözdizimsel olarak geçerli olacak ve doğru değeri verecektir:

const skin = {"color": "blue"};

Ancak, “renk"in geçerli bir tanımlayıcı ad olduğunu biliyoruz ve özellik adının etrafındaki tırnak işaretleri ihmal edilebilir:

const skin = {color: "blue"};

Çift tırnak yerine tek tırnak kullanabileceğimizi de biliyoruz:

const skin = {'color': 'blue'};

Ancak, bu değişmez değerlerin ikisini de alıp JSON olarak ele alırsak, hiçbiri sözdizimsel olarak geçerli olmayacaktır JSON:

{color: "blue"}
{'color': 'blue'}

JSON, kesinlikle tüm özellik adlarının çift tırnak içine alınmasını ve dize değerlerinin de çift tırnak içine alınmasını gerektirir.

JSON’a yeni başlayanların, JavaScript değişmezleri içeren kod alıntılarını JSON olarak kullanmaya çalışması ve JSON ayrıştırıcısından aldıkları sözdizimi hataları konusunda kafalarını karıştırmaları yaygındır.

Kodda veya konuşmada yanlış terminoloji uygulandığında daha fazla kafa karışıklığı ortaya çıkmaya başlar.

Yaygın bir anti-desen, JSON olmayan değerleri tutan değişkenleri “json” olarak adlandırmaktır:

fetch(url).then(function (response) {
  const json = JSON.parse(response.data); // Confusion ensues!

  // We're done with the notion of "JSON" at this point,
  // but the concept stuck with the variable name.
});

Yukarıdaki örnekte, “response.data”, bazı API’ler tarafından döndürülen bir JSON dizesidir. JSON, HTTP yanıt etki alanında durur. “Json” yanlış adlandırmasına sahip değişken yalnızca bir JavaScript değerine sahiptir (bir nesne, dizi veya hatta basit bir sayı olabilir!)

Yukarıdakileri yazmanın daha az kafa karıştırıcı bir yolu:

fetch(url).then(function (response) {
  const value = JSON.parse(response.data);

  // We're done with the notion of "JSON" at this point.
  // You don't talk about JSON after parsing JSON.
});

Geliştiriciler ayrıca “JSON nesnesi” ifadesini çok fazla kullanma eğilimindedir. Bu da kafa karışıklığına yol açar. Çünkü yukarıda belirtildiği gibi, bir JSON dizesinin bir nesneyi değer olarak tutması gerekmez. “JSON dizesi” daha iyi bir terimdir. Tıpkı “XML dizisi” veya “YAML dizisi” gibi. Bir dize alırsınız, onu ayrıştırırsınız ve sonunda bir değer elde edersiniz.

Değiştirici işleviyle seri hale getirme #

Serileştirilen değerleri filtrelemek veya dönüştürmek için bir “değiştirici” işlevi kullanılabilir.

const userRecords = [
  {name: "Joe", points: 14.9, level: 31.5},
  {name: "Jane", points: 35.5, level: 74.4},
  {name: "Jacob", points: 18.5, level: 41.2},
  {name: "Jessie", points: 15.1, level: 28.1},
];

// Remove names and round numbers to integers to anonymize records before sharing
const anonymousReport = JSON.stringify(userRecords, (key, value) =>
  key === 'name'
    ? undefined
    : (typeof value === 'number' ? Math.floor(value) : value)
);

Bu, aşağıdaki dizeyi üretir:

'[{"points":14,"level":31},{"points":35,"level":74},{"points":18,"level":41},{"points":15,"level":28}]'

Basit bir JSON dizesini ayrıştırma #

“JSON.parse()” yöntemi, bir dizeyi JSON olarak ayrıştırır ve bir JavaScript temel öğesi, dizi veya nesne döndürür:

const array = JSON.parse('[1, 2, "c", "d", {"e": false}]');
console.log(array); // logs: [1, 2, "c", "d", {e: false}]

Döngüsel nesne değerleri #

Tüm nesneler bir JSON dizesine dönüştürülemez. Bir nesnenin döngüsel öz referansları olduğunda, dönüştürme başarısız olur.

Bu genellikle, ebeveyn ve çocuğun her ikisinin de birbirine başvurduğu hiyerarşik veri yapıları için geçerlidir:

const world = {
  name: 'World',
  regions: []
};

world.regions.push({
  name: 'North America',
  parent: 'America'
});
console.log(JSON.stringify(world));
// {"name":"World","regions":[{"name":"North America","parent":"America"}]}

world.regions.push({
  name: 'Asia',
  parent: world
});

console.log(JSON.stringify(world));
// Uncaught TypeError: Converting circular structure to JSON

İşlem bir döngü algılar algılamaz, özel durum ortaya çıkar. Döngü algılama olmasaydı, dizi sonsuz uzunlukta olurdu.
