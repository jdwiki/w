Sözdizimi #

  • ‘katı kullanın’;
  • “katı kullan”;
  • `katı kullan`;

Katı mod, geriye dönük uyumsuz birkaç geliştirmeyi etkinleştirmek için ECMAScript 5’te eklenen bir seçenektir. “Sıkı mod” kodundaki davranış değişiklikleri şunları içerir:

  • Tanımsız değişkenlere atama, yeni global değişkenler tanımlamak yerine bir hataya neden olur;
  • Yazılamaz özelliklere (“window.undefined” gibi) atama veya silme, sessizce yürütmek yerine bir hataya neden olur;
  • Eski sekizli sözdizimi (ör. “0777”) desteklenmiyor;
  • ‘with’ ifadesi desteklenmiyor;
  • “eval”, çevreleyen kapsamda değişkenler oluşturamaz;
  • “.caller” ve “.arguments” işlevlerinin özellikleri desteklenmez;
  • Bir fonksiyonun parametre listesinde kopya olamaz;
  • “pencere” artık “bu"nun değeri olarak otomatik olarak kullanılmamaktadır.

NOT:- ‘katı’ modu, bir sayfa katı olmayan modun özelliklerine bağlı olan JavaScript kullanıyormuş gibi varsayılan olarak ETKİN DEĞİLDİR, o zaman bu kod bozulur. Bu nedenle programcının kendisi tarafından açılması gerekir.

Tüm komut dosyaları için #

Katı modu, diğer tüm ifadelerin önüne "use strict"; ifadesi yerleştirilerek tüm komut dosyalarına uygulanabilir.

"use strict";
// strict mode now applies for the rest of the script

Katı mod, yalnızca "kesin kullan" tanımladığınız komut dosyalarında etkinleştirilir. Kesin durum farklı komut dosyaları arasında paylaşılmadığından, komut dosyalarını katı modla ve katı mod olmadan birleştirebilirsiniz.

Not: ES2015+ modüller ve sınıflar içinde yazılan tüm kodlar varsayılan olarak katıdır.

son sürüm

Fonksiyonlar için #

Katı mod, işlev bildiriminin başına "use strict"; ifadesinin başına eklenerek tek işlevlere de uygulanabilir.

function strict() {
   "use strict";
   // strict mode now applies to the rest of this function
   var innerFunction = function () {
     // strict mode also applies here
   };
}

function notStrict() {
  // but not here
}

Katı mod ayrıca herhangi bir iç kapsamlı işlev için de geçerli olacaktır.

Özelliklerde yapılan değişiklikler #

Katı mod ayrıca silinemeyen özellikleri silmenizi de engeller.

"use strict";
delete Object.prototype; // throws a TypeError

Sıkı modu kullanmazsanız yukarıdaki ifade basitçe yok sayılır, ancak şimdi neden beklendiği gibi çalışmadığını biliyorsunuz.

Ayrıca, genişletilemeyen bir özelliği genişletmenizi de engeller.

var myObject = {name: "My Name"}
Object.preventExtensions(myObject);

function setAge() {
    myObject.age = 25;   // No errors
}

function setAge() {
    "use strict";
    myObject.age = 25;  // TypeError: can't define property "age": Object is not extensible
}

Global özelliklerde yapılan değişiklikler #

Kesin olmayan mod kapsamında, bir değişken “var”, “const” veya “let” anahtar sözcüğü ile başlatılmadan atandığında, global kapsamda otomatik olarak bildirilir:

a = 12;
console.log(a); // 12

Ancak katı modda, bildirilmemiş bir değişkene herhangi bir erişim bir referans hatası verir:

"use strict";
a = 12; // ReferenceError: a is not defined
console.log(a);

JavaScript’te bazen beklenmeyen bir dizi olası olay bulunduğundan bu yararlıdır. Katı olmayan modda, bu olaylar genellikle geliştiricilerin hata veya beklenmeyen davranış olduğuna inanmalarına neden olur, bu nedenle katı modu etkinleştirerek, atılan herhangi bir hata, onları tam olarak ne yapıldığını bilmeye zorlar.

"use strict";
                       // Assuming a global variable mistypedVariable exists
mistypedVaraible = 17; // this line throws a ReferenceError due to the 
                       // misspelling of variable

Katı moddaki bu kod olası bir senaryoyu görüntüler: atamanın satır numarasını gösteren bir referans hatası vererek geliştiricinin değişken adındaki yanlış türü hemen algılamasını sağlar.

Kesin olmayan modda, hata atılmaması ve atamanın başarıyla yapılmasının yanı sıra, ‘mistypedVaraible’ global kapsamda otomatik olarak global bir değişken olarak bildirilecektir. Bu, geliştiricinin koddaki bu özel atamayı manuel olarak araması gerektiği anlamına gelir.

Ayrıca, değişkenlerin bildirimini zorlayarak, geliştirici yanlışlıkla global değişkenleri işlevlerin içinde bildiremez. Katı olmayan modda:

function foo() { 
   a = "bar"; // variable is automatically declared in the global scope
}
foo();
console.log(a); // >> bar

Katı modda, değişkeni açıkça bildirmek gerekir:

function strict_scope() { 
   "use strict";
   var a = "bar"; // variable is local
}
strict_scope();
console.log(a); // >> "ReferenceError: a is not defined"

Değişken ayrıca, örneğin global kapsamda kullanılmasına izin vererek, bir fonksiyonun dışında ve sonrasında bildirilebilir:

function strict_scope() { 
   "use strict";
   a = "bar"; // variable is global
}
var a;
strict_scope();
console.log(a); // >> bar

Yinelenen Parametreler #

Katı mod, yinelenen işlev parametre adlarını kullanmanıza izin vermez.

function foo(bar, bar) {}  // No error. bar is set to the final argument when called

"use strict";
function foo(bar, bar) {}; // SyntaxError: duplicate formal argument bar

Katı modda işlev kapsamı #

Katı Modda, yerel bir blokta bildirilen işlevlere bloğun dışında erişilemez.

"use strict";
{
  f(); // 'hi'
  function f() {console.log('hi');}
}
f(); // ReferenceError: f is not defined

Kapsam bazında, Strict Mode’daki işlev bildirimleri, “let” veya “const” ile aynı türde bağlamaya sahiptir.

Bir işlevin argüman listesinin davranışı #

“argümanlar” nesnesi katı ve katı olmayan kiplerde farklı davranır. katı olmayan modda, “argüman” nesnesi mevcut olan parametrelerin değerindeki değişiklikleri yansıtacaktır, ancak katı modda parametre değerindeki herhangi bir değişiklik “argüman"a yansıtılmayacaktır. nesne.

function add(a, b){
    console.log(arguments[0], arguments[1]); // Prints : 1,2

    a = 5, b = 10;

    console.log(arguments[0], arguments[1]); // Prints : 5,10
}

add(1, 2);

Yukarıdaki kod için, parametrelerin değerini değiştirdiğimizde ‘arguments’ nesnesi değiştirilir. Ancak, katı mod için aynı şey yansıtılmayacaktır.

function add(a, b) {
    'use strict';

    console.log(arguments[0], arguments[1]); // Prints : 1,2

    a = 5, b = 10;

    console.log(arguments[0], arguments[1]); // Prints : 1,2
}

Parametrelerden herhangi biri “tanımsız” ise ve parametrenin değerini hem katı mod hem de * katı olmayan* modda değiştirmeye çalışırsak, “argümanlar” nesnesinin değişmeden kaldığını belirtmekte fayda var.

Sıkı mod

function add(a, b) {
    'use strict';

    console.log(arguments[0], arguments[1]); // undefined,undefined 
                                             // 1,undefined
    a = 5, b = 10;

    console.log(arguments[0], arguments[1]); // undefined,undefined
                                             // 1, undefined
}
add();
// undefined,undefined 
// undefined,undefined

add(1)
// 1, undefined
// 1, undefined

Sıkı Olmayan Mod

function add(a,b) {

    console.log(arguments[0],arguments[1]);

    a = 5, b = 10;

    console.log(arguments[0],arguments[1]);
}
add();
// undefined,undefined 
// undefined,undefined

add(1);
// 1, undefined
// 5, undefined

Basit Olmayan parametre listeleri #

function a(x = 5) {
  "use strict";
}

geçersiz JavaScript ve bir SyntaxError atacak çünkü yukarıdaki gibi Basit Olmayan Parametre listesi olan bir işlevde "use strict" yönergesini kullanamazsınız - varsayılan atama x = 5

Basit olmayan parametreler şunları içerir:

  • Varsayılan atama
function a(x = 1) {
  "use strict";
}
  • Yıkım
function a({ x }) {
  "use strict";
}
  • Dinlenme parametreleri
function a(...args) {
  "use strict";
}
