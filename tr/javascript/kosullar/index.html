if ve else gibi anahtar sözcükleri içeren koşullu ifadeler, JavaScript programlarına bir Boole koşuluna bağlı olarak farklı eylemler gerçekleştirme yeteneği sağlar: true veya false. Bu bölüm JavaScript koşul ifadelerinin, Boole mantığının ve üçlü ifadelerin kullanımını kapsar.

Sözdizimi #

  • if (koşul) ifade;
  • if (koşul) ifade_1, ifade_2, …, ifade_n;
  • if (koşul) {   ifade }
  • if (koşul) {   ifade_1;   ifade_2;   …   ifade_n; }
  • if (koşul) {   ifade } başka {   ifade }
  • if (koşul) {   ifade } else if (koşul) {   ifade } başka {   ifade }
  • geçiş (ifade) { durum değer1:   ifade   [kes;] durum değer2:   ifade   [kes;] vaka değerN:   ifade   [kes;] varsayılan:   ifade   [kes;] }
  • şart ? değer_için_doğru : yanlış_için_değer;

Koşullar, bir ifadenin değerine dayalı olarak kod yürüterek normal program akışını bozabilir. JavaScript’te bu, “if”, “else if” ve “else” ifadelerinin ve üçlü operatörlerin kullanılması anlamına gelir.

Üçlü operatörler #

if/else işlemlerini kısaltmak için kullanılabilir. Bu, bir değeri hızlı bir şekilde döndürmek için kullanışlıdır (yani, onu başka bir değişkene atamak için).

Örneğin:

var animal = 'kitty';
var result = (animal === 'kitty') ? 'cute' : 'still nice';

Bu durumda, ‘sonuç’ ‘şirin’ değerini alır, çünkü hayvanın değeri ‘kitty’dir. Eğer hayvanın başka bir değeri olsaydı, sonuç ‘hala güzel’ değerini alırdı.

Bunu, kodun “if/else” koşullarıyla istediğiyle karşılaştırın.

var animal = 'kitty';
var result = '';
if (animal === 'kitty') {
    result = 'cute';
} else {
    result = 'still nice';
}

“if” veya “else” koşullarının birkaç işlemi olabilir. Bu durumda operatör, son ifadenin sonucunu döndürür.

var a = 0;
var str = 'not a';
var b = '';
b = a === 0 ? (a = 1, str += ' test') : (a = 2);

‘a’ 0’a eşit olduğundan, ‘1’ olur ve ‘str’ ’test değil’ olur. ‘str’ içeren işlem sonuncuydu, bu nedenle ‘b’, ‘str’ içindeki değer olan işlemin sonucunu alır, yani ‘bir test değil’.

Üçlü operatörler her zaman başka koşullar bekler, aksi takdirde bir sözdizimi hatası alırsınız. Geçici bir çözüm olarak, else dalında sıfıra benzer bir şey döndürebilirsiniz - bu, dönüş değerini kullanmıyor olmanız, ancak işlemi kısaltmanız (veya kısaltmaya çalışmanız) önemli değildir.

var a = 1;
a === 1 ? alert('Hey, it is 1!') : 0;

Gördüğünüz gibi, if (a === 1) alert('Hey, 1!'); aynı şeyi yapardı. Zorunlu bir ’else’ koşulu gerektirmediğinden, yalnızca bir karakter daha uzun olacaktır. Eğer bir “else” koşulu söz konusu olsaydı, üçlü yöntem çok daha temiz olurdu.

a === 1 ? alert('Hey, it is 1!') : alert('Weird, what could it be?');
if (a === 1) alert('Hey, it is 1!') else alert('Weird, what could it be?');

Üçlüler, ek mantığı kapsüllemek için iç içe yerleştirilebilir. Örneğin

foo ? bar ? 1 : 2 : 3

// To be clear, this is evaluated left to right 
// and can be more explicitly expressed as:

foo ? (bar ? 1 : 2) : 3

Bu, aşağıdaki “if/else” ile aynıdır

if (foo) {
  if (bar) {
    1
  } else {
    2
  }
} else {
  3
}

Çok satırlı üçlüler okunabilirliği önemli ölçüde azaltabileceğinden, biçimsel olarak bu yalnızca kısa değişken adlarıyla kullanılmalıdır.

Üçlü olarak kullanılamayan tek deyimler kontrol deyimleridir. Örneğin, üçlüler ile dönüş veya break kullanamazsınız. Aşağıdaki ifade geçersiz olacaktır.

var animal = 'kitty';
for (var i = 0; i < 5; ++i) {
    (animal === 'kitty') ? break:console.log(i);
}

İade ifadeleri için aşağıdakiler de geçersiz olacaktır:

var animal = 'kitty';
(animal === 'kitty') ? return 'meow' : return 'woof';

Yukarıdakileri doğru şekilde yapmak için üçlüyü aşağıdaki gibi döndürürsünüz:

var animal = 'kitty';
return (animal === 'kitty') ? 'meow' : 'woof';

Anahtar deyimi #

Switch deyimleri, bir ifadenin değerini 1 veya daha fazla değerle karşılaştırır ve bu karşılaştırmaya dayalı olarak farklı kod bölümlerini yürütür.

var value = 1;
switch (value) {
  case 1:
    console.log('I will always run');
    break;
  case 2:
    console.log('I will never run');
    break;
}

“break” ifadesi, switch ifadesinden “kırılır” ve switch ifadesi içinde daha fazla kodun çalıştırılmamasını sağlar. Bölümler bu şekilde tanımlanır ve kullanıcının “düşme” durumları yapmasına izin verir.

Uyarı: her vaka için bir “break” veya “return” ifadesinin olmaması, vaka kriterleri karşılanmasa bile programın bir sonraki vakayı değerlendirmeye devam edeceği anlamına gelir!

switch (value) {
  case 1:
    console.log('I will only run if value === 1');
    // Here, the code "falls through" and will run the code under case 2
  case 2:
    console.log('I will run if value === 1 or value === 2');
    break;
  case 3:
    console.log('I will only run if value === 3');
    break;
}

Son durum “varsayılan” durumdur. Bu, başka bir eşleşme yapılmadıysa çalışacaktır.

var animal = 'Lion';
switch (animal) {
  case 'Dog':
    console.log('I will not run since animal !== "Dog"');
    break;
  case 'Cat':
    console.log('I will not run since animal !== "Cat"');
    break;
  default:
    console.log('I will run since animal does not match any other case');
}

Unutulmamalıdır ki bir vaka ifadesi her türlü ifade olabilir. Bu, durum değerleri olarak karşılaştırmaları, işlev çağrılarını vb. kullanabileceğiniz anlamına gelir.

function john() {
  return 'John';
}

function jacob() {
  return 'Jacob';
}

switch (name) {
  case john(): // Compare name with the return value of john() (name == "John")
    console.log('I will run if name === "John"');
    break;
  case 'Ja' + 'ne': // Concatenate the strings together then compare (name == "Jane")
    console.log('I will run if name === "Jane"');
    break;
  case john() + ' ' + jacob() + ' Jingleheimer Schmidt':
    console.log('His name is equal to name too!');
    break;
}

Vakalar için Çoklu Kapsayıcı Kriterler #

Vakalar “break” veya “return” ifadesi olmadan “geçerli” olduğundan, bunu birden çok kapsayıcı ölçüt oluşturmak için kullanabilirsiniz:

var x = "c"
switch (x) {
   case "a":
   case "b":
   case "c":
      console.log("Either a, b, or c was selected.");
      break;
   case "d":
      console.log("Only d was selected.");
      break;
   default:
      console.log("No case was matched.");
      break;  // precautionary break if case order changes
}

Eğer / Else If / Else Kontrolü #

En basit haliyle, bir “if” koşulu şu şekilde kullanılabilir:

var i = 0;

if (i < 1) {
    console.log("i is smaller than 1");
}

‘i < 1’ koşulu değerlendirilir ve ’true’ olarak değerlendirilirse takip eden blok yürütülür. ‘false’ olarak değerlendirilirse blok atlanır.

Bir “if” koşulu, bir “else” bloğu ile genişletilebilir. Koşul, yukarıdaki gibi bir kez kontrol edilir ve “yanlış” olarak değerlendirilirse ikincil bir blok yürütülür (koşul “doğru” olsaydı atlanır). Bir örnek:

if (i < 1) {
    console.log("i is smaller than 1");
} else {
    console.log("i was not smaller than 1");
}

“else” bloğunun başka bir “if” bloğundan (isteğe bağlı olarak bir “else” bloğuyla birlikte) başka bir şey içermediğini varsayalım:

if (i < 1) {
    console.log("i is smaller than 1");
} else {
    if (i < 2) {
        console.log("i is smaller than 2");
    } else {
        console.log("none of the previous conditions was true");
    }
}

O zaman bunu yazmanın yuvalamayı azaltan farklı bir yolu da var:

if (i < 1) {
    console.log("i is smaller than 1");
} else if (i < 2) {
    console.log("i is smaller than 2");
} else {
    console.log("none of the previous conditions was true");
}

Yukarıdaki örneklerle ilgili bazı önemli dipnotlar:

  • Herhangi bir koşul “doğru” olarak değerlendirilirse, o blok zincirindeki başka hiçbir koşul değerlendirilmez ve ilgili tüm bloklar (“else” bloğu dahil) yürütülmez.

  • “else if” bölümlerinin sayısı pratikte sınırsızdır. Yukarıdaki son örnek yalnızca bir tanesini içermektedir, ancak istediğiniz kadarını alabilirsiniz.

  • Bir “if” ifadesinin içindeki koşul, bir boole değerine zorlanabilecek herhangi bir şey olabilir, boolean logic konusuna bakın. daha fazla ayrıntı için;

  • “if-else-if” merdiveni ilk başarıda çıkar. Yani yukarıdaki örnekte ‘i’ değeri 0,5 ise ilk dal yürütülür. Koşullar örtüşürse, yürütme akışında meydana gelen ilk kriter yürütülür. Aynı zamanda doğru olabilecek diğer koşul göz ardı edilir.

  • Yalnızca bir ifadeniz varsa, bu ifadenin etrafındaki parantezler teknik olarak isteğe bağlıdır, örneğin bu iyidir:

    if (i < 1) console.log("i is smaller than 1");

Ve bu da işe yarayacak:

  if (i < 1) 
      console.log("i is smaller than 1");

Bir ‘if’ bloğu içinde birden çok ifade yürütmek istiyorsanız, bunların etrafındaki kaşlı ayraçlar zorunludur. Yalnızca girinti kullanmak yeterli değildir. Örneğin, aşağıdaki kod:

  if (i < 1) 
      console.log("i is smaller than 1");
      console.log("this will run REGARDLESS of the condition"); // Warning, see text!

şuna eşittir:

  if (i < 1) {
      console.log("i is smaller than 1");
  }
  console.log("this will run REGARDLESS of the condition");

Strateji #

Javascript’te birçok durumda bir switch ifadesini değiştirmek için bir strateji modeli kullanılabilir. Koşul sayısı dinamik veya çok büyük olduğunda özellikle yararlıdır. Her koşul için kodun bağımsız ve ayrı ayrı test edilebilir olmasını sağlar.

Strateji nesnesi, her bir ayrı koşulu temsil eden, birden çok işlevi olan basit bir nesnedir. Örnek:

const AnimalSays = {
    dog () {
        return 'woof';
    },

    cat () {
        return 'meow';
    },

    lion () {
        return 'roar';
    },

    // ... other animals

    default () {
        return 'moo';
    }
};

Yukarıdaki nesne aşağıdaki gibi kullanılabilir:

function makeAnimalSpeak (animal) {
    // Match the animal by type
    const speak = AnimalSays[animal] || AnimalSays.default;
    console.log(animal + ' says ' + speak());
}

Sonuçlar:

makeAnimalSpeak('dog') // => 'dog says woof'
makeAnimalSpeak('cat') // => 'cat says meow'
makeAnimalSpeak('lion') // => 'lion says roar'
makeAnimalSpeak('snake') // => 'snake says moo'

Son durumda, varsayılan işlevimiz tüm eksik hayvanları ele alır.

Kullanma || ve && kısa devre #

Boole işleçleri || ve && “kısa devre” yapacak ve ilk parametrenin doğru veya yanlış olması durumunda ikinci parametreyi değerlendirmeyecektir. Bu, aşağıdaki gibi kısa koşullar yazmak için kullanılabilir:

var x = 10

x == 10 && alert("x is 10")
x == 10 || alert("x is not 10")
