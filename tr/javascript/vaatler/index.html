Sözdizimi #

  • yeni Söz ( /* yürütücü işlevi: */ işlev(çöz, reddet) { })
  • söz.then(onFulfilled[, onRejected])
  • söz.catch(Reddedildi)
  • Promise.resolve(çözünürlük)
  • Söz vermek.reddetmek(sebep)
  • Promise.all(yinelenebilir)
  • Promise.race (yinelenebilir)

Sözler, ECMAScript 2015 spesifikasyonunun bir parçasıdır ve tarayıcı desteği sınırlıdır, Temmuz 2017 itibariyle dünya çapındaki tarayıcıların %88’i bunu desteklemektedir. Aşağıdaki tablo, vaatler için destek sağlayan en eski tarayıcı sürümlerine genel bir bakış sunmaktadır.

| krom | Kenar | Firefox | Internet Explorer | Opera | Opera Mini | Safari | iOS Safari | | :—-: | :–: | :—-: | :————-:: | :—: | :——-: | :—-: | :——-: | | 32 | 12 | 27 | x | 19 | x | 7.1 | 8 |

Bunları desteklemeyen ortamlarda, ‘Promise’ çoklu doldurulabilir. Üçüncü taraf kitaplıkları, geri arama işlevlerinin otomatik olarak “vaad edilmesi” veya “ilerleme” gibi ek yöntemler gibi genişletilmiş işlevler de sağlayabilir - “bildir” olarak da bilinir.

Promises/A+ standart web sitesi bir 1.0 ve 1.1 uyumlu uygulamaların listesi sağlar. A+ standardını temel alan söz verilen geri aramalar her zaman olay döngüsündeki mikro görevler olarak eşzamansız olarak yürütülür.

Giriiş #

Bir Promise nesnesi bir değeri üreten veya sonunda üretecek bir işlemi temsil eder. Sözler, zaman uyumsuz çalışmanın (muhtemelen beklemede olan) sonucunu sarmak için sağlam bir yol sağlar ve derinlemesine iç içe geçmiş geri aramalar ("[geri arama cehennemi][geri arama_hell]" olarak bilinir) sorununu azaltır.

Durumlar ve kontrol akışı #

Bir söz üç durumdan birinde olabilir:

  • pending — Temel işlem henüz tamamlanmadı ve taahhüt beklemede.
  • yerine getirildi — İşlem tamamlandı ve söz değer ile * yerine getirildi*. Bu, senkronize bir işlevden bir değer döndürmeye benzer.
  • reddedildi — İşlem sırasında bir hata oluştu ve söz sebebi ile reddedildi. Bu, senkronize bir işlevde bir hata oluşturmaya benzer.

Bir sözün yerine getirildiği veya reddedildiği zaman yerleştirildi (veya çözümlendi) olduğu söylenir. Bir söz bir kez yerine getirildikten sonra değişmez hale gelir ve durumu değişemez. Bir sözün then ve catch yöntemleri, yerine getirildiğinde yürütülen geri aramaları eklemek için kullanılabilir. Bu geri aramalar, sırasıyla yerine getirme değeri ve reddetme nedeni ile çağrılır.

Örnek #

const promise = new Promise((resolve, reject) => {
    // Perform some work (possibly asynchronous)
    // ...

    if (/* Work has successfully finished and produced "value" */) {
        resolve(value);
    } else {
        // Something went wrong because of "reason"
        // The reason is traditionally an Error object, although
        // this is not required or enforced.
        let reason = new Error(message);
        reject(reason);

        // Throwing an error also rejects the promise.
        throw reason;
    }
});

then ve catch yöntemleri, yerine getirme ve reddetme geri aramalarını eklemek için kullanılabilir:

promise.then(value => {
    // Work has completed successfully,
    // promise has been fulfilled with "value"
}).catch(reason => {
    // Something went wrong,
    // promise has been rejected with "reason"
});

Not: Aynı söz üzerinde “promise.then(…)” ve “promise.catch(…)” çağrıları, yürütme sırasında bir hata oluşursa “Sözde Yakalanmayan istisna” ile sonuçlanabilir söz veya geri aramalardan birinin içinde, bu nedenle tercih edilen yol, bir sonraki dinleyiciyi önceki ’then’ / ‘catch’ tarafından döndürülen söze eklemek olacaktır.

Alternatif olarak, her iki geri arama da then öğesine yapılan tek bir çağrıya eklenebilir:

promise.then(onFulfilled, onRejected);

Önceden yapılmış bir söze geri aramalar eklemek, onları hemen [mikro görev kuyruğuna][mikro görev_kuyruğa] yerleştirir ve “mümkün olan en kısa sürede” (yani, şu anda yürütülmekte olan komut dosyasından hemen sonra) çağrılır. Diğer birçok olay yayan uygulamadan farklı olarak, geri aramaları eklemeden önce sözün durumunu kontrol etmek gerekli değildir.

Söz zincirleme #

Bir sözün then yöntemi yeni bir söz döndürür.

const promise = new Promise(resolve => setTimeout(resolve, 5000));

promise
    // 5 seconds later
    .then(() => 2)
    // returning a value from a then callback will cause
    // the new promise to resolve with this value
    .then(value => { /* value === 2 */ });

then geri çağrısından bir Promise döndürmek, onu söz zincirine ekleyecektir.

function wait(millis) {
    return new Promise(resolve => setTimeout(resolve, millis));
}

const p = wait(5000).then(() => wait(4000)).then(() => wait(1000));
p.then(() => { /* 10 seconds have passed */ });

Bir catch, bir try/catch ifadesindeki catch işlevinin çalışmasına benzer şekilde, reddedilen bir sözün kurtarılmasına izin verir. Bir catch‘dan sonra zincirlenen herhangi bir then, catch öğesinden çözülen değeri kullanarak çözümleme işleyicisini yürütür.

const p = new Promise(resolve => {throw 'oh no'});
p.catch(() => 'oh yes').then(console.log.bind(console));  // outputs "oh yes"

Zincirin ortasında ‘catch’ veya ‘reject’ işleyicisi yoksa, sonundaki bir ‘catch’ zincirdeki tüm reddetmeleri yakalayacaktır:

p.catch(() => Promise.reject('oh yes'))
  .then(console.log.bind(console))      // won't be called
  .catch(console.error.bind(console));  // outputs "oh yes"

Belirli durumlarda, işlevlerin yürütülmesini “dallandırmak” isteyebilirsiniz. Koşullara bağlı olarak bir fonksiyondan farklı sözler vererek bunu yapabilirsiniz. Kodun ilerleyen bölümlerinde, üzerlerindeki diğer işlevleri çağırmak ve/veya tüm hataları tek bir yerde işlemek için tüm bu dalları bir araya getirebilirsiniz.

promise
    .then(result => {          
        if (result.condition) {
            return handlerFn1() 
                .then(handlerFn2);
        } else if (result.condition2) {
            return handlerFn3()
                .then(handlerFn4);
        } else {
            throw new Error("Invalid result");
        }
    })
    .then(handlerFn5)
    .catch(err => {
        console.error(err);
    });

Böylece, işlevlerin yürütme sırası şöyle görünür:

promise --> handlerFn1 -> handlerFn2 --> handlerFn5 ~~> .catch()
         |                            ^
         V                            |
         -> handlerFn3 -> handlerFn4 -^

Tek ‘yakalama’, hangi dalda meydana gelirse gelsin hatayı alır.

Birden fazla eşzamanlı vaat bekleniyor #

Promise.all() statik yöntemi, yinelenebilir bir vaat (örneğin bir Dizi) kabul eder ve yinelenebilirdeki tüm vaatler çözüldüğünde veya reddedildiğinde çözülen yeni bir söz döndürür yinelenebilirdeki vaatlerden en az biri reddedilmişse.

// wait "millis" ms, then resolve with "value"
function resolve(value, milliseconds) {
    return new Promise(resolve => setTimeout(() => resolve(value), milliseconds));
}

// wait "millis" ms, then reject with "reason"
function reject(reason, milliseconds) {
    return new Promise((_, reject) => setTimeout(() => reject(reason), milliseconds));
}

Promise.all([
    resolve(1, 5000),
    resolve(2, 6000),
    resolve(3, 7000)    
]).then(values => console.log(values)); // outputs "[1, 2, 3]" after 7 seconds.

Promise.all([
    resolve(1, 5000),
    reject('Error!', 6000),
    resolve(2, 7000)
]).then(values => console.log(values)) // does not output anything
.catch(reason => console.log(reason)); // outputs "Error!" after 6 seconds.

Yinelenebilirdeki söz dışı değerler [“vaat edilen”][vaat veren]‘dir.

Promise.all([
    resolve(1, 5000),
    resolve(2, 6000),
    { hello: 3 }
])
.then(values => console.log(values)); // outputs "[1, 2, { hello: 3 }]" after 6 seconds

Yıkıcı atama, birden çok vaatten sonuç alınmasına yardımcı olabilir.

Promise.all([
    resolve(1, 5000),
    resolve(2, 6000),
    resolve(3, 7000)
])
.then(([result1, result2, result3]) => {
    console.log(result1);
    console.log(result2);
    console.log(result3);
});

[umut verici]: https://www.wikiod.com/tr/javascript/vaatler#“Umut veren” değerler

Bir diziyi zincirleme vaatlere indirgeme #

Bu tasarım deseni, bir öğe listesinden bir dizi eşzamansız eylem oluşturmak için kullanışlıdır.

İki çeşidi vardır:

  • zincir başarılı olduğu sürece devam eden bir zincir oluşturan “o zaman” indirgemesi.
  • zincir hata yaptığı sürece devam eden bir zincir oluşturan “yakalama” azaltma.

“o zaman” indirimi

Kalıbın bu varyantı bir .then() zinciri oluşturur ve animasyonları zincirlemek veya bir dizi bağımlı HTTP isteği yapmak için kullanılabilir.

[1, 3, 5, 7, 9].reduce((seq, n) => {
    return seq.then(() => {
        console.log(n);
        return new Promise(res => setTimeout(res, 1000));
    });
}, Promise.resolve()).then(
    () => console.log('done'),
    (e) => console.log(e)
);
// will log 1, 3, 5, 7, 9, 'done' in 1s intervals

Açıklama:

  1. Bir kaynak dizide .reduce() öğesini çağırır ve başlangıç ​​değeri olarak Promise.resolve() sağlarız.
  2. Küçültülmüş her öğe, başlangıç ​​değerine bir .then() ekler.
  3. reduce()‘un ürünü Promise.resolve().then(…).then(…) olacaktır.
  4. Önceki tüm adımlar çözümlendikten sonra ‘successHandler’ı çalıştırmak için, azaltma işleminden sonra manuel olarak bir .then(successHandler, errorHandler) ekleriz. Herhangi bir adım başarısız olursa, “errorHandler” yürütülür.

Not: “Sonra” indirgemesi, Promise.all() öğesinin sıralı karşılığıdır.

“Yakalama” azaltma

Modelin bu varyantı bir .catch() zinciri oluşturur ve çalışan bir sunucu bulunana kadar bazı yansıtılmış kaynaklar için bir dizi web sunucusunu sırayla araştırmak için kullanılabilir.

var working_resource = 5; // one of the values from the source array
[1, 3, 5, 7, 9].reduce((seq, n) => {
    return seq.catch(() => {
        console.log(n);
        if(n === working_resource) { // 5 is working
            return new Promise((resolve, reject) => setTimeout(() => resolve(n), 1000));
        } else { // all other values are not working
            return new Promise((resolve, reject) => setTimeout(reject, 1000));
        }
    });
}, Promise.reject()).then(
    (n) => console.log('success at: ' + n),
    () => console.log('total failure')
);
// will log 1, 3, 5, 'success at 5' at 1s intervals

Açıklama:

  1. Bir kaynak dizide .reduce() öğesini çağırırız ve başlangıç ​​değeri olarak Promise.reject() sağlarız.
  2. Küçültülmüş her öğe, başlangıç ​​değerine bir .catch() ekleyecektir.
  3. reduce()‘un ürünü Promise.reject().catch(...).catch(...) olacaktır.
  4. Önceki adımlardan herhangi biri çözüldükten sonra ‘successHandler’ı çalıştırmak için, azaltma işleminden sonra .then(successHandler, errorHandler)‘u manuel olarak ekleriz. Tüm adımlar başarısız olursa, “errorHandler” yürütülür.

Not: “Yakalama” azaltması, “Promise.any()“nin ardışık karşılığıdır ([bluebird.js]((http://bluebirdjs.com/docs/api/promise.any.html) içinde uygulandığı gibi) ), ancak şu anda yerel ECMAScript’te değil).

Birden çok eşzamanlı sözün ilkini bekliyorum #

Promise.race() statik yöntemi, yinelenen bir Sözü kabul eder ve yinelenebilirdeki vaatlerin ilki çözülür veya reddedilir çözülmez çözen veya reddeden yeni bir Söz döndürür.

// wait "milliseconds" milliseconds, then resolve with "value"
function resolve(value, milliseconds) {
    return new Promise(resolve => setTimeout(() => resolve(value), milliseconds));
}

// wait "milliseconds" milliseconds, then reject with "reason"
function reject(reason, milliseconds) {
    return new Promise((_, reject) => setTimeout(() => reject(reason), milliseconds));
}

Promise.race([
    resolve(1, 5000),
    resolve(2, 3000),
    resolve(3, 1000)
])
.then(value => console.log(value)); // outputs "3" after 1 second.


Promise.race([
    reject(new Error('bad things!'), 1000),
    resolve(2, 2000)
])
.then(value => console.log(value)) // does not output anything
.catch(error => console.log(error.message)); // outputs "bad things!" after 1 second

Geri aramalarla “umut verici” işlevler #

Düğüm tarzı bir geri aramayı kabul eden bir işlev verildiğinde,

fooFn(options, function callback(err, result) { ... });

bunu vaat edebilirsiniz (söze dayalı bir işleve dönüştürün) şöyle:

function promiseFooFn(options) {
    return new Promise((resolve, reject) =>
        fooFn(options, (err, result) =>
            // If there's an error, reject; otherwise resolve
            err ? reject(err) : resolve(result)
        )
    );
}

Bu işlev daha sonra aşağıdaki gibi kullanılabilir:

promiseFooFn(options).then(result => {
    // success!
}).catch(err => {
    // error!
});

Daha genel bir şekilde, verilen herhangi bir geri arama stili işlevine nasıl söz verileceği aşağıda açıklanmıştır:

function promisify(func) {
    return function(...args) {
        return new Promise((resolve, reject) => {
            func(...args, (err, result) => err ? reject(err) : resolve(result));
        });
    }
}

Bu şu şekilde kullanılabilir:

const fs = require('fs');
const promisedStat = promisify(fs.stat.bind(fs));

promisedStat('/foo/bar')
    .then(stat => console.log('STATE', stat))
    .catch(err => console.log('ERROR', err));

Hata yönetimi #

Sözlerden kaynaklanan hatalar, “then” öğesine iletilen ikinci parametre (“reddetme”) veya “catch” öğesine iletilen işleyici tarafından işlenir:

throwErrorAsync()
  .then(null, error => { /* handle error here */ });
// or
throwErrorAsync()
  .catch(error => { /* handle error here */ });

Zincirleme #

Bir söz zinciriniz varsa, bir hata “çözüm” işleyicilerinin atlanmasına neden olur:

throwErrorAsync()
  .then(() => { /* never called */ })
  .catch(error => { /* handle error here */ });

Aynısı ’then’ işlevleriniz için de geçerlidir. Bir “çözüm” işleyicisi bir istisna atarsa, bir sonraki “reddetme” işleyicisi çağrılır:

doSomethingAsync()
  .then(result => { throwErrorSync(); })
  .then(() => { /* never called */ })
  .catch(error => { /* handle error from throwErrorSync() */ });

Bir hata işleyicisi, bir söz zincirine devam etmenize olanak tanıyan yeni bir söz verir. Hata işleyici tarafından döndürülen söz, işleyici tarafından döndürülen değerle çözülür:

throwErrorAsync()
  .catch(error => { /* handle error here */; return result; })
  .then(result => { /* handle result here */ });

Hatayı yeniden atarak bir hatanın bir söz zincirini basamaklandırmasına izin verebilirsiniz:

throwErrorAsync()
  .catch(error => {
      /* handle error from throwErrorAsync() */
      throw error;
  })
  .then(() => { /* will not be called if there's an error */ })
  .catch(error => { /* will get called with the same error */ });

“throw” ifadesini bir “setTimeout” geri çağrısı içine sararak, söz tarafından işlenmeyen bir istisna atmak mümkündür:

new Promise((resolve, reject) => {
  setTimeout(() => { throw new Error(); });
});

Bu işe yarar çünkü sözler eşzamansız olarak atılan istisnaları işleyemez.

İşlenmeyen reddetmeler #

Bir sözün bir “catch” bloğu veya “reddet” işleyicisi yoksa, bir hata sessizce yok sayılır:

throwErrorAsync()
  .then(() => { /* will not be called */ });
// error silently ignored

Bunu önlemek için her zaman bir “catch” bloğu kullanın:

throwErrorAsync()
  .then(() => { /* will not be called */ })
  .catch(error => { /* handle error*/ });
// or
throwErrorAsync()
  .then(() => { /* will not be called */ }, error => { /* handle error*/ });

Alternatif olarak, işlenmeyen reddedilen sözleri yakalamak için unhandledrejection olayına abone olun:

window.addEventListener('unhandledrejection', event => {});

Bazı sözler, reddedilmelerini oluşturma zamanından sonra halledebilir. rejectionhandled olayı, böyle bir söz işlendiğinde tetiklenir:

window.addEventListener('unhandledrejection', event => console.log('unhandled'));
window.addEventListener('rejectionhandled', event => console.log('handled'));
var p = Promise.reject('test');

setTimeout(() => p.catch(console.log), 1000);

// Will print 'unhandled', and after one second 'test' and 'handled'

’event’ argümanı ret hakkında bilgi içerir. “event.reason” hata nesnesidir ve “event.promise” olaya neden olan vaat nesnesidir.

Nodejs’de ‘rejectionhandled’ ve ‘unhandledrejection’ olayları, ‘process’ üzerinde sırasıyla ‘rejectionHandled’ ve ‘unhandledRejection’ olarak adlandırılır ve farklı bir imzaya sahiptir:

process.on('rejectionHandled', (reason, promise) => {});
process.on('unhandledRejection', (reason, promise) => {});

“sebep” argümanı hata nesnesidir ve “söz” argümanı, olayın tetiklenmesine neden olan vaat nesnesine bir referanstır.

Bu “işlenmeyen reddetme” ve “reddetme işleme” olaylarının kullanımı yalnızca hata ayıklama amacıyla düşünülmelidir. Tipik olarak, tüm sözler reddedilmelerini ele almalıdır.

Not: Şu ​​anda yalnızca Chrome 49+ ve Node.js “işlenmeyen reddetme” ve “reddetme işleme” olaylarını desteklemektedir.

Uyarılar #

‘yerine getirme’ ve ‘reddetme’ ile zincirleme #

’then(fulfill, red)’ işlevi (her iki parametre de ’null’ değil) benzersiz ve karmaşık bir davranışa sahiptir ve tam olarak nasıl çalıştığını bilmiyorsanız kullanılmamalıdır.

Girişlerden biri için “null” verilirse, işlev beklendiği gibi çalışır:

// the following calls are equivalent
promise.then(fulfill, null) 
promise.then(fulfill)

// the following calls are also equivalent
promise.then(null, reject) 
promise.catch(reject)

Ancak, her iki girdi de verildiğinde benzersiz davranışı benimser:

// the following calls are not equivalent!
promise.then(fulfill, reject)
promise.then(fulfill).catch(reject)

// the following calls are not equivalent!
promise.then(fulfill, reject)
promise.catch(reject).then(fulfill)

’then( yerine getir, reddet)’ işlevi, ’then(fulfill).catch(red)’ için bir kısayol gibi görünür, ancak değildir ve birbirinin yerine kullanılırsa sorunlara neden olur. Böyle bir sorun, “reddetme” işleyicisinin “yerine getirme” işleyicisinden gelen hataları işlememesidir. İşte olacaklar:

Promise.resolve() // previous promise is fulfilled
    .then(() => { throw new Error(); }, // error in the fulfill handler
        error => { /* this is not called! */ });

Yukarıdaki kod, hata yayıldığı için reddedilen bir sözle sonuçlanacaktır. Bunu, yerine getirilmiş bir sözle sonuçlanan aşağıdaki kodla karşılaştırın:

Promise.resolve() // previous promise is fulfilled
    .then(() => { throw new Error(); }) // error in the fulfill handler
    .catch(error => { /* handle error */ });

Benzer bir sorun, reddedilen vaatler yerine yerine getirilen vaatleri yaymak dışında, “sonra(yerine getir, reddet)” ile “yakala(reddet).then(yerine getir)” ile dönüşümlü olarak kullanıldığında da ortaya çıkar.

Bir söz döndürmesi gereken işlevden eşzamanlı olarak atma #

Bunun gibi bir işlev hayal edin:

function foo(arg) {
  if (arg === 'unexepectedValue') {
    throw new Error('UnexpectedValue')
  }

  return new Promise(resolve => 
    setTimeout(() => resolve(arg), 1000)
  )
}

Böyle bir işlev bir söz zincirinin ortasında kullanılıyorsa, görünüşe göre sorun yok:

makeSomethingAsync().
  .then(() => foo('unexpectedValue'))
  .catch(err => console.log(err)) // <-- Error: UnexpectedValue will be caught here

Ancak, aynı işlev bir söz zincirinin dışında çağrılırsa, hata bu işlev tarafından işlenmez ve uygulamaya atılır:

foo('unexpectedValue') // <-- error will be thrown, so the application will crash
  .then(makeSomethingAsync) // <-- will not run
  .catch(err => console.log(err)) // <-- will not catch

Hatayla birlikte reddedilen bir söz verin #

Fırlatmak yerine aşağıdakileri yapın:

function foo(arg) {
  if (arg === 'unexepectedValue') {
    return Promise.reject(new Error('UnexpectedValue'))
  }

  return new Promise(resolve => 
    setTimeout(() => resolve(arg), 1000)
  )
}

İşlevinizi bir söz zincirine sarın #

‘Throw’ ifadeniz, zaten bir söz zincirinin içindeyken uygun şekilde yakalanacaktır:

function foo(arg) {
  return Promise.resolve()
    .then(() => {
      if (arg === 'unexepectedValue') {
        throw new Error('UnexpectedValue')
      }

      return new Promise(resolve => 
        setTimeout(() => resolve(arg), 1000)
      )
    })
}

Senkron ve asenkron işlemleri uzlaştırma #

Bazı durumlarda, kod dallarında tekrarlamayı önlemek için senkronize bir işlemi bir söz içine sarmak isteyebilirsiniz. Bu örneği alın:

if (result) { // if we already have a result
  processResult(result); // process it
} else {
  fetchResult().then(processResult);
}

Yukarıdaki kodun senkron ve asenkron dalları, senkronize işlemi bir söz içine yedekli olarak sarmak suretiyle uzlaştırılabilir:

var fetch = result
  ? Promise.resolve(result)
  : fetchResult();

fetch.then(processResult);

Eşzamansız bir çağrının sonucunu önbelleğe alırken, sonuçtan ziyade sözü önbelleğe almak tercih edilir. kendisi. Bu, birden çok paralel isteği çözümlemek için yalnızca bir eşzamansız işlemin gerekli olmasını sağlar.

Hata koşullarıyla karşılaşıldığında önbelleğe alınan değerleri geçersiz kılmak için özen gösterilmelidir.

// A resource that is not expected to change frequently
var planets = 'http://swapi.co/api/planets/';
// The cached promise, or null
var cachedPromise;

function fetchResult() {
    if (!cachedPromise) {
        cachedPromise = fetch(planets)
            .catch(function (e) {
                // Invalidate the current result to retry on the next fetch
                cachedPromise = null;
                // re-raise the error to propagate it to callers
                throw e;
            });
    }
    return cachedPromise;
}

Gecikme işlevi çağrısı #

setTimeout() yöntemi, belirli bir milisaniye sayısından sonra bir işlevi çağırır veya bir ifadeyi değerlendirir. Aynı zamanda asenkron bir işlem gerçekleştirmenin önemsiz bir yoludur.

Bu örnekte, ‘wait’ işlevinin çağrılması, ilk argüman olarak belirtilen süreden sonra sözü çözer:

function wait(ms) {
    return new Promise(resolve => setTimeout(resolve, ms));  
}

wait(5000).then(() => { 
    console.log('5 seconds have passed...');
});

“Umut veren” değerler #

Promise.resolve statik yöntemi, değerleri vaatlere sarmak için kullanılabilir.

let resolved = Promise.resolve(2);
resolved.then(value => {
    // immediately invoked
    // value === 2
});

“Değer” zaten bir sözse, Promise.resolve bunu basitçe yeniden düzenler.

let one = new Promise(resolve => setTimeout(() => resolve(2), 1000));
let two = Promise.resolve(one);
two.then(value => {
    // 1 second has passed
    // value === 2
});

Aslında, “değer” herhangi bir “sonra” olabilir (belirli bir şekilde uyumlu bir söz gibi çalışan bir “sonra” yöntemini tanımlayan nesne). Bu, Promise.resolve‘un güvenilmeyen 3. taraf nesnelerini güvenilir 1. taraf Sözlerine dönüştürmesine olanak tanır.

let resolved = Promise.resolve({
    then(onResolved) {
        onResolved(2);
    }
});
resolved.then(value => {
    // immediately invoked
    // value === 2
});

Promise.reject statik yöntemi, verilen “sebep” ile hemen reddeden bir söz verir.

let rejected = Promise.reject("Oops!");
rejected.catch(reason => {
    // immediately invoked
    // reason === "Oops!"
});

ES2017 async/await’i kullanma #

Yukarıdaki aynı örnek, [Görüntü yükleniyor][görüntü_yükleniyor], eşzamansız işlevler kullanılarak yazılabilir. Bu aynı zamanda istisna işleme için ortak ‘dene/yakala’ yönteminin kullanılmasına da izin verir.

Not: [Nisan 2017 itibariyle, tüm tarayıcıların güncel sürümleri, ancak Internet Explorer eşzamansız işlevleri destekler2.

function loadImage(url) {
    return new Promise((resolve, reject) => {
        const img = new Image();
        img.addEventListener('load', () => resolve(img));
        img.addEventListener('error', () => {
            reject(new Error(`Failed to load ${url}`));
        });
        img.src = url;
    });
}

(async () => {

    // load /image.png and append to #image-holder, otherwise throw error
    try {
        let img = await loadImage('http://example.com/image.png');
        document.getElementById('image-holder').appendChild(img);
    }
    catch (error) {
        console.error(error);
    }

})();

vaatlerle her biri için #

Bir dizinin her bir elemanına bir söz veren bir işlevi (‘cb’) etkin bir şekilde uygulamak mümkündür, her eleman bir önceki eleman işlenene kadar işlenmeyi bekler.

function promiseForEach(arr, cb) {
  var i = 0;

  var nextPromise = function () {
    if (i >= arr.length) {
      // Processing finished.
      return;
    }

    // Process next function. Wrap in `Promise.resolve` in case
    // the function does not return a promise
    var newPromise = Promise.resolve(cb(arr[i], i));
    i++;
    // Chain to finish processing.
    return newPromise.then(nextPromise);
  };

  // Kick off the chain.
  return Promise.resolve().then(nextPromise);
};

Bu, binlerce öğeyi birer birer verimli bir şekilde işlemeniz gerekiyorsa yardımcı olabilir. Sözleri oluşturmak için normal bir “for” döngüsü kullanmak, hepsini bir kerede oluşturacak ve önemli miktarda RAM alacaktır.

nihayet() ile temizleme gerçekleştirme #

Şu anda, sözün yerine getirilip getirilmediğine veya reddedilmesine bakılmaksızın yürütülecek sözlere “nihayet” bir geri arama eklemek için bir teklif (henüz ECMAScript standardının bir parçası değil) bulunmaktadır. Anlamsal olarak, bu, ’try’ bloğunun ‘finally’ yan tümcesi ile benzerdir.

Temizleme için genellikle bu işlevi kullanırsınız:

var loadingData = true;

fetch('/data')
    .then(result => processData(result.data))
    .catch(error => console.error(error))
    .finally(() => {
        loadingData = false;
    });

’nihayet’ geri aramanın, sözün durumunu etkilemediğini not etmek önemlidir. Hangi değeri döndürdüğü önemli değil, vaat daha önce olduğu gibi yerine getirilmiş/reddedilmiş durumda kalır. Yani sözün üstündeki örnekte “nihai” geri çağırma “tanımsız” döndürse bile, “processData(result.data)” dönüş değeriyle çözülecektir.

Standardizasyon süreci devam ederken, vaatlerin uygulanması büyük olasılıkla kutudan çıkan “nihayet” geri aramalarını desteklemeyecektir. Eşzamanlı geri aramalar için bu işlevi bir çoklu doldurma ile ekleyebilirsiniz:

if (!Promise.prototype.finally) {
    Promise.prototype.finally = function(callback) {
        return this.then(result => {
            callback();
            return result;
        }, error => {
            callback();
            throw error;
        });
    };
}

Eşzamansız API isteği #

Bu, eşzamansız işlevselliğinden yararlanma sözüne sarılmış basit bir “GET” API çağrısı örneğidir.

var get = function(path) {
  return new Promise(function(resolve, reject) {
    let request = new XMLHttpRequest();
    request.open('GET', path);
    request.onload = resolve;
    request.onerror = reject;
    request.send();
  });
};

Aşağıdaki [onload][onload] ve onerror işlevleri kullanılarak daha sağlam hata işleme yapılabilir.

request.onload = function() {
  if (this.status >= 200 && this.status < 300) {
    if(request.response) {
      // Assuming a successful call returns JSON
      resolve(JSON.parse(request.response));
    } else {
      resolve();
  } else {
    reject({
      'status': this.status,
      'message': request.statusText
    });
  }
};

request.onerror = function() {
  reject({
    'status': this.status,
    'message': request.statusText
  });
};
