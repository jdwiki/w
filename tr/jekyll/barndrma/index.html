GitHub Sayfaları #

GitHub, kullanıcılar veya kuruluşlar ve proje sitesi için sınırsız barındırma hizmeti sunar. Hem Jekyll hem de statik dosyalar mevcuttur.

Jekyll blogunuzu Github’da barındırmanın adımları.

Kurmak #

Kullanıcılar veya kuruluşlar sitesi #

  1. username.github.io adında bir havuz oluşturun; burada kullanıcı adı GitHub’da kullanıcı adınız (veya kuruluş adınız) olur.
  2. Depoyu bilgisayarınıza klonlayın: 
    $ git clone https://github.com/username/username.github.io
  3. Proje klasörünü girin, bootstrap, sitenizi tasarlayın ve hata ayıklayın: 
    $ cd username.github.io
$ bundle install
$ bundle exec jekyll serve
  4. Depoyu taahhüt edin ve itin: 
    $ git add --all
$ git commit -m "Initial commit"
$ git push -u origin master

Artık blogunuzu görmek için username.github.io adresine gidebilmelisiniz.

Proje sitesi #

Proje sitesi, özel havuzlar dahil her havuzda etkinleştirilebilir.

  1. Proje sitesini etkinleştirin.

    Go to Settings-GitHub Pages-Sources, choose a source to switch on GitHub Pages for the repository.

  2. Site oluşturun

    You may build a Jekyll site from scratch or use Theme Chooser to find a theme for your project site.

  3. İçeriği düzenleyin

  4. Taahhüt

Artık proje sitenizi görmek için username.github.io/your-project adresine gidebilmelisiniz.

Özel Alan Adları #

  1. Ayarlar->GitHub Sayfaları->Özel alan adını açın, özel alan adınızı ekleyin.
  2. Bir “CNAME” dosyası oluşturun: 
    $ cd username.github.io
$ echo "example.com" > CNAME
  3. Taahhüt edin ve itin 
    $ git commit -m "Add CNAME" CNAME
$ git push -u origin master

Kısıtlamalar #

Eklentiler #

Jekyll, sitenize özgü özel olarak oluşturulmuş içerik oluşturmanıza olanak tanıyan kancalı bir eklenti sistemine sahiptir. Ancak GitHub Pages, güvenlik nedenleriyle yalnızca beyaz bir eklenti listesine izin verir.

İşte beyaz liste:

  • Jekyll Site Haritası
  • Jekyll SEO Etiketi
  • github-meta verisi
  • Jekyll Yemi
  • Jekyll Yönlendirmesi
  • Yemojiler
  • Jekyll Mansiyonları

GitHub Pages ile tutarsızlığı önlemek için yerel olarak sunmak için --safe kullanabilirsiniz.

Oluşturduğunuz siteyi GitHub Sayfalarında yayınlayarak, siteyi yerel olarak dönüştürerek ve oluşturulan statik dosyaları Jekyll kaynak dosyaları yerine GitHub deponuza iterek tüm eklentileri kullanmaya devam edebilirsiniz.

İşaretleme Motoru #

01/05/2016 tarihinden bu yana GitHub Pages, Markdown motoru olarak yalnızca kramdown’ı desteklemektedir.

Daha fazla ayrıntı için https://github.com/blog/2100-github-pages-now-faster-and-simpler-with-jekyll-3-0 adresine bakın.

Kaynaklar #

GitHub, Jekyll kaynaklarını “master” dalında, “gh-pages” dalı veya “/docs” klasörüne ayarlamanıza izin verir.

Tam bir eğitim https://pages.github.com/ adresinde mevcuttur.

Yerel makine #

Test amacıyla blogunuzu yerel makinenizde barındırabilirsiniz. Ayarladıktan ve herhangi bir değişiklik yaptıktan sonra, Jekyll blogu http://localhost:4000 olarak sunabilir. Projenin kök dizinindeki komut satırında şunu çalıştırın:

$ bundle exec jekyll serve

‘Bundle exec’ kısmı isteğe bağlıdır, ancak Bundler kullanırsanız, gem bağımlılıklarının güncel olmasını sağlar. Daha hızlı bir Edit-Build-Test döngüsü, ‘_drafts’ dizininden makaleler oluşturmak için ‘–draft’ seçeneğini kullanın:

$ bundle exec jekyll serve --draft --detach

--detach kullanımı, işlemi arka plana alır, böylece komut istemi başka bir şey için kullanılabilir.

(2.4 sürümünden itibaren, “–watch” seçeneği varsayılan olarak etkindir. Şans eseri elinizde daha eski bir sürüm varsa, eklemeniz gerekir Bu seçenek, değişikliklerin izlenmesi için.)

Hedef dizini Apache, nginx veya Lighttpd gibi farklı bir web sunucusundaki bir dizine de ayarlayabilirsiniz:

$ jekyll build --destination /path/to/web_server/root

CloudCannon barındırma (ve CMS) #

CloudCannon, Jekyll uygulamaları için barındırma ve bir CMS sunar. Jekyll uygulamanızı CloudCannon’da (http://cloudcannon.com) barındırmaya ilişkin adımlar aşağıda verilmiştir.

Sürüm kontrolü olmadan:

  • Bazı yerel dosyaları ve “jekyll serve” kullanarak blogunuzu yerel olarak oluşturun
  • Bir CloudCannon hesabı oluşturun ve yeni bir site oluşturun
  • Sitenizi CloudCannon’daki ‘Dosya Tarayıcısına’ sürükleyin

Sürüm kontrolü ile:

  • Github veya Bitbucket’te bir depo oluşturun
  • Bazı yerel dosyaları ve “jekyll serve” kullanarak blogunuzu yerel olarak oluşturun
  • Bir CloudCannon hesabı oluşturun ve yeni bir site oluşturun
  • Github veya Bitbucket deponuzu bağlayın

CloudCannon’un tamamen ücretsiz olmadığını unutmayın. Ücretsiz planda CloudCannon’u grafiksel bir CMS olarak kullanabilirsiniz, ancak harici barındırmaya ihtiyacınız olacaktır. Ücretli plana hosting de dahildir.
