Bu konu, kendi derleyicinizi oluşturmaya başlayabilmeniz için bilmeniz gereken derleyici yapımındaki tüm temel bilgileri içerecektir. Bu dokümantasyon konusu, derleyici yapılarındaki ‘4’ bölümlerinden ilk ‘2’sini içerecek ve geri kalanı farklı bir konuda olacaktır.

Ele alınacak konular şunlardır:

Sözcük Analizi

Ayrıştırma

Sözdizimi #

  • Sözcük Analizi kaynak metin, tür ve değer belirteçlerine dönüştürülür.

  • Ayrıştırma kaynak belirteçleri soyut bir sözdizimi ağacına (AST) dönüştürülür.

Basit Sözcük Analizcisi #

Bu örnekte size ‘python’da bir tamsayı değişken bildirimi için belirteçleri oluşturacak temel bir sözlük oluşturmayı göstereceğim.

Sözlüksel çözümleyici ne yapar? #

Bir sözlükçünün (sözcüksel çözümleyicinin) amacı, kaynak kodunu taramak ve her sözcüğü bir liste öğesine bölmektir. Bir kez yapıldığında, bu kelimeleri alır ve bir belirteç oluşturmak için bu `[‘INTEGER’, ‘178’]’ gibi görünen bir tür ve değer çifti oluşturur.

Bu belirteçler, dilinizin sözdizimini tanımlamak için oluşturulur, bu nedenle sözlüğün tüm amacı, farklı öğeleri nasıl tanımlamak ve yorumlamak istediğinize bağlı olduğundan, dilinizin sözdizimini oluşturmaktır.

Bu sözlük için örnek kaynak kodu:

int result = 100;

‘python’da lexer kodu:

import re                                 # for performing regex expressions

tokens = []                               # for string tokens
source_code = 'int result = 100;'.split() # turning source code into list of words

# Loop through each source code word
for word in source_code:
    
    # This will check if a token has datatype decleration
    if word in ['str', 'int', 'bool']: 
        tokens.append(['DATATYPE', word])
    
    # This will look for an identifier which would be just a word
    elif re.match("[a-z]", word) or re.match("[A-Z]", word):
        tokens.append(['IDENTIFIER', word])
    
    # This will look for an operator
    elif word in '*-/+%=':
        tokens.append(['OPERATOR', word])
    
    # This will look for integer items and cast them as a number
    elif re.match(".[0-9]", word):
        if word[len(word) - 1] == ';': 
            tokens.append(["INTEGER", word[:-1]])
            tokens.append(['END_STATEMENT', ';'])
        else: 
            tokens.append(["INTEGER", word])

print(tokens) # Outputs the token array

Bu kod parçacığını çalıştırırken çıktı aşağıdaki gibi olmalıdır:

[['DATATYPE', 'int'], ['IDENTIFIER', 'result'], ['OPERATOR', '='], ['INTEGER', '100'], ['END_STATEMENT', ';']]

Gördüğünüz gibi, tamsayı değişken bildirimi gibi bir kaynak kodu parçasını tür ve değer çifti belirteçlerinden oluşan bir belirteç akışına dönüştürdük.

Hadi parçalayalım #

  1. Belirli kelimelerin belirli bir normal ifade kalıbıyla eşleşip eşleşmediğini kontrol ederken gerekli olacağından, normal ifade kitaplığını içe aktararak başlıyoruz.

  1. tokens adında boş bir liste oluşturuyoruz. Bu, oluşturduğumuz tüm belirteçleri depolamak için kullanılacaktır.

  2. Bir dize olan kaynak kodumuzu, dizedeki her kelimenin bir boşlukla ayrılmış olduğu bir kelime listesine böldük. Daha sonra bunları ‘source_code’ adlı bir değişkende saklarız.

  3. “Kaynak_kodu” listemizde kelime kelime dolaşmaya başlıyoruz.

  4. Şimdi ilk kontrolümüzü yapıyoruz:

    if word in ['str', 'int', 'bool']: 
   tokens.append(['DATATYPE', word])

    What we check for here is a datatype which will tell us what type our variable will be.

  5. Bundan sonra, kaynak kodumuzdaki her bir kelimeyi tanımlayarak ve onun için bir jeton oluşturarak yukarıdaki gibi daha fazla kontrol yapıyoruz. Bu belirteçler daha sonra bir Soyut Sözdizimi Ağacı (AST) oluşturmak için ayrıştırıcıya iletilecektir.

Bu kodla etkileşimde bulunmak ve onunla oynamak istiyorsanız, burada çevrimiçi bir derleyicideki kodun bağlantısı https://repl.it/J9Hj/latest

Basit Ayrıştırıcı #

Bu, önceki örnek Simple Lexical Analyzer’da oluşturduğumuz bir tamsayı değişkeni bildirim belirteci akışını ayrıştıracak basit bir ayrıştırıcıdır. Bu ayrıştırıcı ayrıca python ile kodlanacaktır.

Ayrıştırıcı nedir? #

Ayrıştırıcı, kaynak metnin soyut sözdizimi ağacına (AST) dönüştürüldüğü süreçtir. Ayrıca, hiçbir anlam ifade etmeyen sözdizimsel olarak doğru ifadeleri ayıklayan anlamsal doğrulama yapmaktan da sorumludur, örn. erişilemeyen kod veya yinelenen bildirimler.

Örnek jetonlar:

[['DATATYPE', 'int'], ['IDENTIFIER', 'result'], ['OPERATOR', '='], ['INTEGER', '100'], ['END_STATEMENT', ';']]

‘python3’ içindeki ayrıştırıcı kodu:

ast = { 'VariableDecleration': [] }

tokens = [ ['DATATYPE', 'int'], ['IDENTIFIER', 'result'], ['OPERATOR', '='],  
           ['INTEGER', '100'], ['END_STATEMENT', ';'] ]
           
# Loop through the tokens and form ast
for x in range(0, len(tokens)):
    
    # Create variable for type and value for readability
    token_type  = tokens[x][0]
    token_value = tokens[x][1]
    
    # This will check for the end statement which means the end of var decl
    if token_type == 'END_STATEMENT': break
    
    # This will check for the datatype which should be at the first token
    if x == 0 and token_type == 'DATATYPE':
        ast['VariableDecleration'].append( {'type': token_value} )
    
    # This will check for the name which should be at the second token
    if x == 1 and token_type == 'IDENTIFIER':
        ast['VariableDecleration'].append( {'name': token_value} )
        
    # This will check to make sure the equals operator is there
    if x == 2 and token_value == '=': pass
    
    # This will check for the value which should be at the third token    
    if x == 3 and token_type == 'INTEGER' or token_type == 'STRING':
        ast['VariableDecleration'].append( {'value': token_value} )
        
print(ast)

Sonuç olarak aşağıdaki kod parçası bunu çıkarmalıdır:

{'VariableDecleration': [{'type': 'int'}, {'name': 'result'}, {'value': '100'}]}

Gördüğünüz gibi, ayrıştırıcının yaptığı her şey kaynak koddandır belirteçler değişken bildirimi için bir kalıp bulur (bu durumda) ve onunla “tür”, “ad” ve “değer” gibi özelliklerini tutan bir nesne oluşturur.

Hadi parçalayalım #

  1. AST’nin tamamını tutacak ‘ast’ değişkenini yarattık.

  2. Şimdi ayrıştırılması gereken lexer’ımız tarafından oluşturulan belirteçleri tutan örnekler ’token’ değişkenini oluşturduk.

  3. Ardından, her bir belirteç arasında dolaşıyoruz ve belirli belirteçleri bulmak için bazı kontroller yapıyoruz ve onlarla AST’mizi oluşturuyoruz.

  4. Tip için değişken, okunabilirlik için değer oluşturuyoruz

  5. Şimdi bunun gibi kontroller yapıyoruz:

    if x == 0 and token_type == 'DATATYPE':
     ast['VariableDecleration'].append( {'type': token_value} )

    which looks for a datatype and adds it to the AST. We keep doing this for the value and name which will then result in a full VariableDecleration AST.
