EAV (Varlık Özellik Değeri)

  • öznitelik ön uç, kaynak ve arka uç modellerinin arayüzünü uygula

Bu sayfada

  • öznitelik ön uç, kaynak ve arka uç modellerinin arayüzünü uygula

Varlık

Depolanan verilerin türü hakkında bilgi depolar. Magento durumunda bu müşteri, ürün, kategori vb.

Bağlanmak

Varlıkların her birinin bireysel özellikleri, ör. isim, ağırlık, e-posta adresi vb.

Değer

Belirli bir varlığın ve özniteliğin değeri. Örneğin, müşteri varlığını ve email özniteliğini belirtebilir ve ardından ona merhaba@example.com değerini verebiliriz.

Veritabanı Şeması

eav_entity

Varlık tablosu.

eav_entity_attribute

Öznitelik tablosu.

eav_entity_{type}

Değer tabloları. Türler tarih saat, ondalık sayılar, int, metin ve varchar’dır.

Tarih veya tamsayı değere daha iyi uysa bile, eav_entity_varchar tablosunun değerler için varchar türüne sahip olduğuna dikkat etmek önemlidir.

Modeller ve Kaynak Modelleri

Mage/Eav/Model/Resource içindeki tüm modeller Mysql4’tür ve kaynak modellerdir.

Ek olarak Entity/Abstract.php ve Entity/Setup.php.

Düz Karşı EAV

EAV modelleri daha karmaşıktır ve birden çok tablodan kaydetme ve yükleme mantığı sağlarken, düz veya standart modeller nispeten basittir (geleneksel).

Standart modeller, özelliklerini çoğunlukla tek bir tablo ile çalışan veri belirleyiciler ve alıcılar ile yönetir. EAV modelleri esas olarak öznitelik modellerini yönetir. Standart modeller yalnızca verilerini bir tabloya kaydeder ve ondan yüklenir. EAV modelleri, temel verileri yükledikten sonra tüm (veya belirli bir küme) öznitelikleri yükler ve verileri kaydettikten sonra öznitelikleri kaydeder (öznitelikleri ekleme, güncelleme ve silme dahil).

EAV Kaynak Modeli Örnekleri

EAV depolama şemasını kullanarak varlıkları bulmak için, uzantıları arayan Mage_Eav_Model_Entity_Abstract kullanılabilir. Bu, EAV yapısına dayalı tüm kaynak modellerini ortaya çıkarmalıdır. Ancak, ortaya çıkan liste Mage_Sales modülünden artık kullanılmayan birçok eski sınıfı içerecektir.

EAV depolama şemasını kullanan varlıkları içeren tek modüller Mage_Catalog (kategoriler ve ürünler) ve Mage_Customer’dır (müşteriler ve adresler).

Müşteri Grupları, düz masa depolama şemasını kullanır. Magento 1.4’ün piyasaya sürülmesiyle düz tablo varlıklarına dönüştürülen tüm satış varlıkları.

EAV’nin kullanılmasının nedeni, varlıkların belirsiz sayıda özelliğe sahip olabilmeleri ve dolayısıyla esnek kalabilmeleridir. Örneğin, bir Müşteri varlığına (bir EAV varlığı olan) yeni bir öznitelik eklediğinizde, bu yeni özniteliğin eklenmesi için veritabanı tablosunun değiştirilmesi gerekmez.

Avantajlar

Esneklik Veritabanı şemasının modele göre değişmesi gerekmez Uygulaması hızlı

Dezavantajları

  • Yetersiz

Normalde 20 sütun döndüren bir sorgu, EAV’de 20 kendi kendine birleştirmeden oluşur. Ancak, Mage_Eav modülü, öznitelik değeri verilerini yüklemek için genellikle birleşimleri kullanmaz. Bunun yerine birleşim seçimleri kullanılır. Birleştirmeler yalnızca EAV koleksiyonlarını filtrelemek için kullanılır.

  • Alt türler arasındaki ilişkiler için mekanizma yok
  • Varlık alt türlerinin gruplandırılması yok

Web Sitesi ve Mağaza Kapsamları

EAV içinde web sitesi ve mağaza kapsamı öznitelik değerlerini işlemek için, core_store’a geri bağlanan kapsamı göstermek için katalog varlığında bir store_id değeri bulunur. Normal mağazaların (mağaza görünümleri) yanı sıra global değer olan bir mağaza ‘0’ da vardır. Belirli bir mağazadayken, sistem önce mevcut mağazadaki bir varlık değerini kontrol edecek ve ardından global varlığa geri dönecektir. Mage_Customer EAV varlıklarının bir store_id kapsam sütunu yoktur.

Ekle, Güncelle ve Sil

Bir öznitelik üzerinde bir ekleme, güncelleme veya silme yapılması gerekip gerekmediğini belirlemek için orijinal nesneyle bir karşılaştırma yapılır. Orijinal nesne, varlık veritabanından alındığında temel olarak veri nesnesinin bir kopyasıdır.

  • Öznitelik orijinal olarak mevcutsa ve yeni değeri boş değilse; günceller.
  • Öznitelik orijinal olarak mevcutsa ancak yeni değeri boş olarak ayarlanmışsa; siler. - Öznitelik orijinal olarak mevcut değilse ve yeni değeri boş değilse; ekler.

Özellik Yönetimi

Özellik Modelleri

Özellik Modeli Veritabanı formundaki özniteliği temsil eder, mantığı tüm öznitelikler arasında standarttır ve değiştirilmesi zordur.

Ön Uç Modeli

Özniteliğin ön uçla olan arayüzü ve özniteliğin ön uçta gerektirdiği herhangi bir mantığı sağlar, ör. görüntülerde getUrl() yöntemi.

Arka Uç Modeli

Bunlar, veritabanına kaydedilmeden önce öznitelik üzerinde doğrulama gerçekleştirir. Örneğin, parola arka uç modeli, parolayı kaydedilmeden önce bir karmaya dönüştürür. Ayrıca kaydetmeden önce parola ve parola onayının eşleşip eşleşmediğini kontrol eder.

Kaynak Modeller

Bir öznitelik için mevcut seçenekleri doldurmak için kullanılır, ör. katalog/ürün_ durumu etkinleştirildi ve devre dışı bırakıldı.

Gerekli Yöntemler

Bir kaynak model şunları gerektirir:

<?php
    public function getAllOptions();
    public function getOptionText($value);
?>

Mage_Eav_Model_Entity_Attribute_Source_Abstract soyut kaynak modelinde getOptionText() için bir uygulama zaten mevcut olduğundan, genellikle yalnızca getAllOptions() öğesinin uygulanması gerekir.

Bir ön uç modeli getValue() yöntemini gerektirmez.

Bir arka uç modeli şunları gerektirir:

<?php
    public function getTable();
    public function isStatic();
    public function getType();
    public function getEntityIdField();
    public function setValueId($valueId);
    public function getValueId();
    public function afterLoad($object);
    public function beforeSave($object);
    public function afterSave($object);
    public function beforeDelete($object);
    public function afterDelete($object);
    public function getEntityValueId($entity);
    public function setEntityValidId($entity, $valueId);
?>

Tüm bu yöntemler, Mage_Eav_Model_Entity_Attribute_Backend_Abstract soyut arka uç modelinde uygulanır. Özel arka uç modelleri için yalnızca özelleştirme gerektiren yöntemlerin geçersiz kılınması gerekir.

Sistem Konfigürasyonu Kaynak Modelleri

EAV öznitelikleri için kullanılamaz. EAV kaynak modelleri getAllOptions yöntemini uygularken adminhtml kaynak modelleri toOptionArray() yöntemini uygular.

Varsayılan sistem yapılandırma kaynak modelleri Mage/Adminhtml/Model/System/Config/Source/ içinde bulunabilir.

Özellik Kaynak Modelleri

Öznitelik Kaynak Modellerinin amacı, seçme ve çoklu seçim öznitelikleri için seçenekler ve değerler listesi sağlamaktır. Ayrıca gerekirse sütun bilgilerini katalog düz tablo indeksleyicisine sağlarlar.

Bir özniteliğe ilişkin tüm seçeneklerin listesini almak için aşağıdakileri gerçekleştirin:

<?php
    $options = $attribute->getSource()->getAllOptions(false);

    // or for admin
    $options = $_attribute->getSource()->getAllOptions(true, true);
?>

Varsayılan Özellik Modelleri

Ön uç, arka uç veya - seçme veya çoklu seçim öznitelikleri için - kaynak modeller olarak herhangi bir sınıf belirtilmemişse, varsayılan bir sınıf kullanılır.

Varsayılan öznitelik ön uç modeli Mage_Eav_Model_Entity_Attribute_Frontend_Default’dur.

Varsayılan öznitelik arka uç modeli, öznitelik koduna bağlıdır ve Mage_Eav_Model_Entity_Attribute::_getDefaultBackendModel() yönteminde belirlenir.

<?php
    protected function _getDefaultBackendModel()
    {
        switch ($this->getAttributeCode()) {
            case 'created_at':
                return 'eav/entity_attribute_backend_time_created';

            case 'updated_at':
                return 'eav/entity_attribute_backend_time_updated';

            case 'store_id':
                return 'eav/entity_attribute_backend_store';

            case 'increment_id':
                return 'eav/entity_attribute_backend_increment';
        }

        return parent::_getDefaultBackendModel();
    }
?>

Yöntem son satıra düşerse Mage_Eav_Model_Entity_Attribute_Backend_Default kullanılır.

Varsayılan kaynak modeli, Mage_Eav_Model_Entity_Attribute_Source_Table’da ayarlanır. Bu, katalog modülleri öznitelik modelinde ayarlanır. eav modülünde belirtilen varsayılan yapılandırma kaynağı modeli hiçbir zaman kullanılmaz.

Özellik Ekle

EAV öznitelikleri eklemek için kurulum sınıfında genişleterek Mage_Eav_Model_Entity_Setup kullanın.

addAttribute() Nitelikleri oluşturur, gruplara ve kümelere ekler (varsayılan dahil) veya zaten varsa günceller güncelleme Özniteliği() Yalnızca öznitelik verilerini günceller. Bu yöntemleri genişletmek, ek veriler eklemek veya gereken bağımsız değişkenleri basitleştirmek için özel kurulum sınıfları kullanılabilir.

Düz Masalar

Düz katalog öznitelikleri, dizin oluşturucular tarafından yönetilir:

Mage_Catalog_Model_Resource_Product_Flat_Indexer::updateAttribute() Mage_Catalog_Model_Resource_Category_Flat::synchroise() Ürün öznitelikleri, düz tabloya eklenirlerse (bkz. Mage_Catalog_Model_Resource_Product_Flat_Indexer::getAttributeCodes()):

Statik (arka uç türü) Filtrelenebilir Ürün listelemesinde kullanılır Promosyon kuralları için kullanılır Sıralama ölçütü için kullanılır Sistem Özellikleri Her mağaza için farklı bir düz tablo vardır, her biri farklı bir mağaza kapsamlı varlık öznitelik değeri içerir. Çok dilli değerler, her dil için farklı depolara sahip olarak yönetilir.

Kaynak Arayüzü

/**
* Entity attribute select source interface
*
* Source is providing the selection options for user interface
*
*/
interface Mage_Eav_Model_Entity_Attribute_Source_Interface
{
/**
* Retrieve All options
*
* @return array
*/
public function getAllOptions();

/**
* Retrieve Option value text
*
* @param string $value
* @return mixed
*/
public function getOptionText($value);
}

Arka Uç Arayüzü

/**
* Entity attribute backend interface
*
* Backend is responsible for saving the values of the attribute
* and performing pre and post actions
*
*/
interface Mage_Eav_Model_Entity_Attribute_Backend_Interface
{
public function getTable();
public function isStatic();
public function getType();
public function getEntityIdField();
public function setValueId($valueId);
public function getValueId();
public function afterLoad($object);
public function beforeSave($object);
public function afterSave($object);
public function beforeDelete($object);
public function afterDelete($object);

/**
* Get entity value id
*
* @param Varien_Object $entity
*/
public function getEntityValueId($entity);

/**
* Set entity value id
*
* @param Varien_Object $entity
* @param int $valueId
*/
public function setEntityValueId($entity, $valueId);
}
