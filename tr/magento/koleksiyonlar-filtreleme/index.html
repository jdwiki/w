Parametreler #

Parametre Ayrıntılar
$addFieldToFilter($field, $condition = null) { string } Filtreye eklediğimiz alan.
$addFieldToFilter($field, $condition = null) { karışık } Kullanacağımız filtrenin tanımı.
addAttributeToFilter($attr, $condition = null, $join = ‘inner’) { string } Filtreye eklediğimiz alan.
addAttributeToFilter($attr, $condition = null, $join = ‘inner’) { karışık } Kullanacağımız filtrenin tanımı.
addAttributeToFilter($attr, $condition = null, $join = ‘inner’) { (‘inner’,’left’) } EAV tablosuna katılırken kullanılacak sql birleştirme türü.

Filtre Karşılaştırma Argümanları

Magento ayrıca karşılaştırma operatörlerini kullanarak esnek bir filtreleme yöntemi sunar. Geçerli operatörlerin ve sözdizimlerinin bir listesi:

Tüm karşılaştırma bağımsız değişkenleri, ‘addFieldToFielter()’ veya ‘addAttributeToFilter()’ yöntemlerinin ikinci parametresine iletilebilir.

$collection_of_products->addAttributeToFilter('visible',array("eq"=>1));
Karşılaştırma Argüman Dizisi Sonuç SQL Snippet’i
Eşittir array(“eq”=>$var) NEREDE (`my_field` = $var)
Eşit Değil array(“neq”=>$var) NEREDE (`my_field` != $var)
Beğen array(“like”=>$var) NEREDE (`my_field` GİBİ $var)
Beğenmedim array(“nlike”=>$var) NEREDE (`my_field` $var GİBİ DEĞİL)
Is array(“is”=>$var) NEREDE (`my_field` IS $var)
In array(“in”=>$var) WHERE (`my_field` IN($var))
İçinde Değil array(“nin”=>$var) WHERE (`my_field` NOT IN($var))
Null array(“null”=>true) NEREDE (`my_field` NULL)
Boş Değil array(“notnull”=>true) NEREDE (`my_field` NULL DEĞİL)
Büyüktür array(“gt”=>$var) WHERE (`my_field` > $var)
Less Than array(“lt”=>$var) WHERE (`my_field` < $var)
Büyüktür veya Eşit array(“gteq”=>$var) WHERE (`my_field` >= $var)

Koleksiyonları Filtreleme #

Magento, koleksiyonları filtrelemek için güçlü bir dizi yönteme sahiptir. Koleksiyonlarda içerilebilecek iki tür Nesne olduğundan, filtrelemeden önce önce hangi tür verilerle çalıştığımızı belirlememiz gerekir. Magento, ürünler ve kategoriler gibi varlıklar için bir EAV veri modeli uygular. Bir EAV Nesneleri koleksiyonunu filtreliyorsak, kullanılacak farklı bir dizi yöntem vardır.

Magento’da Siparişler EAV Nesneleri olarak saklanmaz. Bu, sipariş koleksiyonunu temel bir koleksiyonu filtrelemek için iyi bir örnek yapar.

$collection_of_orders = Mage::getModel('sales/order')->getCollection();
$collection_of_orders->addFieldToFilter('status','processing');

Ürün koleksiyonuna bakarsak ürünlerin bir EAV veri modelinde saklandığını görebiliriz. EAV özelliklerine göre de kolayca filtreleyebiliriz.

$collection_of_products = Mage::getModel('catalog/product')->getCollection();
$collection_of_products->addAttributeToFilter('visible',1);

Filtrelerdeki AND’leri ve OR’leri İşleme #

Verilerimizi sorguladığımızda, tam olarak aradığımız veri setini elde etmek için genellikle birden fazla filtreye ihtiyaç duyarız. SQL’de bunu AND ve OR yan tümceleri ile hallederiz. Aynı şeyi koleksiyonlarla da başarabiliriz.

Sorgunuza bir AND yan tümcesi eklemek için başka bir yöntem çağrısı eklemeniz yeterlidir. Bu, ikinci filtreyi bir AND ile birleştiren orijinal WHERE ifadesine ekleyecektir.

Mage::getModel('catalog/product')->getCollection()
        ->addFieldToFilter('sku',array('like'=>'a%'))
        ->addFieldToFilter('sku',array('like'=>'%b'));

Ortaya çıkan WHERE yan tümcesi şöyle görünecektir:

WHERE (e.sku like 'a%') AND (e.sku like '%b')

Şimdi diyelim ki ‘a’ ile başlayan VEYA ‘b’ ile biten tüm sku’lar istiyoruz. OR yan tümcesini nasıl ekleyeceğiz? Magento’nun koleksiyonları sayesinde oldukça yalındır. Filtreyi filtre dizisine ikinci bir eleman olarak ekliyoruz.

Mage::getModel('catalog/product')->getCollection()
        ->addFieldToFilter('sku', array(
            array('like'=>'a%'), 
            array('like'=>'%b')
        ));

Şimdi, ortaya çıkan WHERE yan tümcesi şöyle görünecektir:

WHERE (((e.sku like 'a%') or (e.sku like '%b')))
