Tastypie, Django için bir web hizmeti API çerçevesidir. REST tarzı arayüzler oluşturmak için kullanışlı, ancak güçlü ve son derece özelleştirilebilir bir soyutlama sağlar. Tastypie, modellerinizi açığa çıkarmayı kolaylaştırır, ancak ortaya çıkardığınız şey üzerinde tam kontrol sağlayarak veritabanını gerektiği kadar soyutlamanıza izin verir. Tastypie, ORM dışı veri kaynaklarıyla entegrasyonu da kolaylaştırır. Dolayısıyla Tastypie, ORM ve ORM Olmayan veritabanları ile kullanılabilir.

Hızlı başlangıç #

  1. INSTALLED_APPS’ye lezzetli pasta ekleyin.

  2. Uygulamanızda çıplak bir init.py ile bir api dizini oluşturun.

  3. Bir <my_app>/api/resources.py dosyası oluşturun ve içine aşağıdakileri yerleştirin:

    from tastypie.resources import ModelResource from my_app.models import MyModel

    class MyModelResource(ModelResource): class Meta: queryset = MyModel.objects.all() allowed_methods = [‘get’]

Kök URLconf’unuza aşağıdaki kodu ekleyin (yönetici kodunun olabileceği yere):

from django.conf.urls import url, include
from tastypie.api import Api
from my_app.api.resources import MyModelResource

v1_api = Api(api_name='v1')
v1_api.register(MyModelResource())

urlpatterns = [
  # ...more URLconf bits here...
  # Then add:
  url(r'^api/', include(v1_api.urls)),
]

Lezzet Kurulumu #

Tastypie, python paket yönetimi, yani pip ile kurulabilir veya kodu doğrudan Github’dan kontrol edebiliriz.

  1. pip kurulumu Django-tastypie
  2. Github‘dan ödeme

Neden Lezzetli Pasta? #

Django için başka API çerçeveleri de var. Mevcut seçenekleri değerlendirmeniz ve kendiniz karar vermeniz gerekir. Bununla birlikte, lezzetli yemek için bazı yaygın nedenler var.

  • RESTful olan ve HTTP’yi iyi kullanan bir API’ye ihtiyacınız var.
  • Derin ilişkileri desteklemek istiyorsunuz.
  • Bunu yapmak için kendi serileştiricinizi yazmak zorunda kalmak İSTEMEZSİNİZ. çıkış hakkı.
  • Çok az sihire sahip, çok esnek ve sorun alanıyla iyi eşleşir.
  • JSON’a eşit olarak muamele edilen (ve YAML da var).

Django ile API geliştirmek için birçok seçeneğimiz var. Ana seçenekler Tastypie ve Django Rest Framework(DRF).

Uygun bir API Çerçevesi yapan nedir?

Bu özellikler:

  1. sayfalandırma
  2. doğrulama ile verilerin gönderilmesi
  3. Sorgu kümeleriyle birlikte meta verilerin yayınlanması
  4. API keşfi
  5. uygun HTTP yanıtı işleme
  6. önbelleğe alma
  7. serileştirme
  8. kısma
  9. izinler
  10. kimlik doğrulama

Uygun API çerçeveleri ayrıca şunlara ihtiyaç duyar:

  1. Kodlarının gerçekten iyi bir test kapsamı

  2. İyi performans

  3. Belgeler

  4. Çerçeveyi ilerletmek ve desteklemek için aktif bir topluluk

Bu faktörleri dikkate alırsanız, şu anda kullanılmaya değer yalnızca iki API çerçevesi vardır (kullanıcı görünümlerine bağlıdır), Django-tastypie ve Django-rest-framework.

Hangisi daha iyi? django-tastypie veya django-dinlenme çerçevesi?

  • Eşit olduklarını söylüyorum. Her ikisiyle de yanlış gidemezsiniz. Her ikisinin de arkasındaki yazarlar ve topluluklar aktif, kod sağlam ve test edilmiş. İşte her ikisi hakkında da özel düşüncelerim:*

lezzetli pasta:

Avantajlar:

  • Başlamak ve temel işlevleri sağlamak kolay OOB (çıkış kutunun)
  • Çoğu zaman Advanced Django kavramlarıyla uğraşmayacaksınız CBV’ler, Formlar vb.
  • Daha okunabilir kod ve daha az sihir!
  • Modelleriniz ORM OLMAYAN ise, devam edin.

Dezavantajları:

  • Deyimsel Django’yu kesinlikle takip etmez (python’a iyi bakın ve django’nun felsefeleri oldukça farklıdır)
  • Büyüdükten sonra API’leri özelleştirmek muhtemelen biraz zor
  • O-Auth Yok

DRF:

Avantajlar:

  • Deyimsel Django’yu takip edin. (Django’nun içini dışını biliyorsanız ve çok CBV, Formlar vb. ile rahat, şüphesiz bunun için gidin)

  • ModelViewSets kullanarak kullanıma hazır REST işlevselliği sağlar. saat aynı zamanda, kullanarak özelleştirme için daha fazla kontrol sağlar CustomSerializer, APIView, GenericViews vb.

  • Daha iyi kimlik doğrulama.

  • Özel izin sınıfları yazmak daha kolay.

  • Çok iyi çalışın ve en önemlisi 3. ile çalışmasını sağlamak çok kolay parti kütüphaneleri ve OAuth.

  • DJANGO-REST-AUTH için KÜTÜPHANE’den bahsetmeye değer Yetkilendirme/Sosyal Kimlik Doğrulama/Kayıt.

Dezavantajları:

  • Django’yu çok iyi bilmiyorsanız, bunun için gitmeyin.
  • Büyü! Bazı zamanlar sihri anlamak çok zor.
  • Çünkü sırayla Django’nun CBV’sinin üstüne yazılmıştır. doğada oldukça karmaşık.
  • Dik bir öğrenme eğrisine sahiptir.

Tastypie’ye Başlarken #

Tastypie yeniden kullanılabilir bir uygulamadır (yani yalnızca kendi koduna dayanır ve yalnızca REST tarzı bir API sağlamaya odaklanır) ve herhangi bir uygulamaya o uygulamanın kaynaklarını değiştirmek zorunda kalmadan bir API sağlamak için uygundur.

Herkesin ihtiyaçları aynı değildir, bu nedenle Tastypie, nasıl çalıştığını geçersiz kılmak veya genişletmek için çok sayıda kanca sağlamak için kendi yolundan çıkıyor.

Örneğin, basit bir bloga bir API ekleyeceğiz uygulama.

Zorunlu olan tek yapılandırma, INSTALLED_APPS‘ınıza ‘tastypie’ eklemektir. Tastypie’nin yalnızca iki gerekli olmayan modeli olduğundan, bu kesinlikle gerekli değildir, ancak kullanımı kolaylaştırabilir.

İşte myapp/models.py:

from tastypie.utils.timezone import now
from django.contrib.auth.models import User
from django.db import models
from django.utils.text import slugify


class Entry(models.Model):
    user = models.ForeignKey(User)
    pub_date = models.DateTimeField(default=now)
    title = models.CharField(max_length=200)
    slug = models.SlugField(null=True, blank=True)
    body = models.TextField()

Kaynak Oluşturma

REST tarzı mimari kaynaklardan bahseder, bu nedenle Tastypie ile şaşırtıcı olmayan bir şekilde entegrasyon Kaynak sınıfları oluşturmayı içerir. Basit uygulamamız için, bunlar için myapp/api.py içinde bir dosya oluşturacağız, ancak bunlar uygulamanızın herhangi bir yerinde bulunabilir:

uygulamam/api.py #

from tastypie.resources import ModelResource
from myapp.models import Entry


class EntryResource(ModelResource):
    class Meta:
        queryset = Entry.objects.all()
        resource_name = 'entry'

REST’te her şey bir kaynaktır, nesnelerin kaynak olarak tanımlandığı anlamına gelir. Bu nedenle, bir Tastypie REST API oluşturmak için bir kaynak sınıfı oluşturmayı içerir. Bir kaynak oluşturmak için ModelResource sınıfını alt sınıfa almamız gerekir. Bu EntryResource sınıfı, Entry modelindeki tüm ilişkisel olmayan alanları kontrol edecek ve kendi Api alanlarını yaratacaktır. Bu, Django Forms’un ModelForm’u gibi çalışır.

Kaynağı Bağlama: Kaynağı oluşturduktan sonra, Django’ya EntryResource’umuzu URL’ye bağlayarak bir kaynağın oluşturulduğunu söylemeliyiz.

urls.py #

from django.conf.urls import url, include
from myapp.api import EntryResource

entry_resource = EntryResource()

urlpatterns = [
    url(r'^blog/', include('myapp.urls')),
    url(r'^api/', include(entry_resource.urls)),
]

Şimdi localhost:8000/api/entry/‘yi kontrol edersek API yanıtını alacağız.

(Meta sınıfındaki resource_name isteğe bağlıdır. Sağlanmazsa, sınıf adından otomatik olarak oluşturulur, tüm Resource örnekleri kaldırılır ve dize küçük harfe dönüştürülür. Bu nedenle EntryResource yalnızca giriş olur. Bu nedenle URL’de belirttiğimiz ../entry/ )

Tastypie Yetkilendirmesi #

Şimdiye kadar istek almaya çalıştık. (Başka bir HTTP isteğini denemeniz gerekiyorsa, curl veya postacı gibi bazı API test araçlarını kullanın)

Kaynağa POST/PUT/DELETE göndermeyi denerseniz, kendinizi “401 Yetkisiz” hatası alıyor buluyorsunuz. Güvenlik için Tastypie, ReadOnlyAuthorization olarak ayarlanmış yetki sınıfıyla birlikte gönderilir. Bu, web’de açığa çıkmayı güvenli hale getirir, ancak POST/PUT/DELETE yapmamızı engeller. Bunları etkinleştirelim.

uygulamam/api.py #

from tastypie.authorization import Authorization
from tastypie.resources import ModelResource
from myapp.models import Entry


class EntryResource(ModelResource):
    class Meta:
        queryset = Entry.objects.all()
        resource_name = 'entry'
        authorization = Authorization()
