Sözdizimi #

  • varName ?değer? ayarla – ayarlanmadı mı?-nocomplain? , ?varName varName varName?

  • $varName koyar

  • [set varName] koyar

  • değişken varName

  • global varName ?varName varName?

  • ?…? içinde ?varName? gibi parametreler, bir Tcl komutunun isteğe bağlı argümanlarını temsil eder.

  • Belgeler: global, upvar

Değişkenlere değer atama #

Tcl’de değerler atamak için set komutu kullanılır. Aşağıdaki şekilde iki argümanla çağrıldığında,

% set tempVar "This is a string."
This is a string.

ikinci bağımsız değişkeni (“Bu bir dizedir.”) ilk bağımsız değişkenin (tempVar) başvurduğu bellek alanına yerleştirir. ‘set’ her zaman ilk argümanda belirtilen değişkenin içeriğini döndürür. Yukarıdaki örnekte, “set”, “Bu bir dizedir” sonucunu döndürür. alıntılar olmadan.

  • değer belirtilirse, varName değişkeninin içeriği değer’e eşit olarak ayarlanır.
  • varName yalnızca şunlardan oluşuyorsa alfasayısal karakterler ve parantez yok, bu bir skalerdir değişken.
  • varName varName(index) biçimindeyse, ilişkisel bir dizinin üyesi.

Değişkenin adının Latin alfabesiyle sınırlı olmadığını, herhangi bir unicode karakter kombinasyonundan (örneğin Ermenice) oluşabileceğini unutmayın:

% set տուն house
house
% puts ${տուն}
house

Kapsam belirleme #

set alpha 1

proc myproc {} {
    puts $alpha
}

myproc

Bu kod çalışmaz çünkü iki alfa farklı kapsamlardadır.

“set alpha 1” komutu, global kapsamda bir değişken oluşturur (bu, onu global bir değişken yapar).

“puts $alpha” komutu, “myproc” komutu yürütüldüğünde oluşturulan kapsamda yürütülür.

İki kapsam birbirinden farklıdır. Bu, “puts $alpha”, “alpha” adını aramaya çalıştığında, böyle bir değişken bulamadığı anlamına gelir.

Ancak bunu düzeltebiliriz:

proc myproc {} {
    global alpha beta
    puts $alpha
}

Bu durumda iki global değişken, “alfa” ve “beta”, prosedürün kapsamındaki diğer ad değişkenlerine (aynı ada sahip) bağlanır. Takma ad değişkenlerinden okumak, global değişkenlerdeki değeri alır ve bunlara yazmak, globallerdeki değerleri değiştirir.

Daha genel olarak, “upvar” komutu, önceki kapsamlardan herhangi birinden değişkenlere takma adlar oluşturur. Global kapsam (#0) ile kullanılabilir:

proc myproc {} {
    upvar #0 alpha alpha beta b
    puts $alpha
}

Takma adlara, (‘alfa’) ile bağlantılı değişkenle aynı ad veya başka bir ad (‘beta’ / ‘b’) verilebilir.

Global kapsamdan “myproc"u çağırırsak, bu değişken de çalışır:

proc myproc {} {
    upvar 1 alpha alpha beta b
    puts $alpha
}

Kapsam numarası “1”, “önceki kapsam” veya “arayanın kapsamı” anlamına gelir.

Ne yaptığınızı gerçekten bilmiyorsanız, “#0”, “0” ve “1”, “upvar” ile kullanılması mantıklı olan kapsamlardır. (“upvar 0”, kesinlikle bir kapsam belirleme işlemi değil, yerel bir değişken için yerel bir takma ad oluşturur.)

Diğer bazı diller, kapsamı küme parantezleri ile tanımlar ve her kapsamda çalışan kodun, çevreleyen kapsamlardaki tüm adları görmesine izin verir. Tcl’de, bir prosedür çağrıldığında tek bir kapsam oluşturulur ve yalnızca kendi adları görünür. Bir yordam başka bir yordamı çağırırsa, kapsamı önceki kapsamın üzerine yığılır ve bu böyle devam eder. Bu, yalnızca global kapsamı ve yerel kapsamı (alt kapsamları olan) olan C tarzı dillerin aksine, her kapsamın, açtığı herhangi bir kapsamı çevreleyen (hemen görünür olmasa da) bir kapsam işlevi gördüğü anlamına gelir. Bir prosedür döndüğünde, kapsamı yok edilir.

Belgeler: global, upvar

Bir bağımsız değişkenle küme çağırma #

‘set’ sadece bir argümanla da çağrılabilir. Tek bir argümanla çağrıldığında, o argümanın içeriğini döndürür.

% set x 235
235
% set x
235

Değişkenleri silme #

‘unset’ komutu, bir veya daha fazla değişkeni kaldırmak için kullanılır.

unset ?-nocomplain? ?--? ?name name name name?

  • Her isim, ‘set’ komutu için kabul edilebilir yollardan herhangi birinde belirtilen bir değişken adıdır.

  • Bir isim bir dizinin bir elemanına atıfta bulunuyorsa, o eleman dizinin geri kalanını etkilemeden kaldırılır.

  • Bir ad, parantez içinde indeksi olmayan bir dizi adından oluşuyorsa, dizinin tamamı silinir.

  • İlk argüman olarak -nocomplain verilirse, komutun çıktısından olası tüm hatalar bastırılır.

  • Seçenek – seçeneklerin sonunu belirtir ve seçeneklerden herhangi biri ile aynı ada sahip bir değişkeni kaldırmak istiyorsanız kullanılmalıdır.

    % set x 235 235 % set x 235 % unset x % set x can’t read “x”: no such variable

Bir değişkenin değerini yazdırma #

Gibi bir değişkenin değerini yazdırmak için,

set tempVar "This is a string."

Puts ifadesindeki argümanın önünde, Tcl’ye değişkenin değerini kullanmasını söyleyen bir $ işareti gelir.

% set tempVar "This is a string."
This is a string.
% puts $tempVar
This is a string.

Ad alanı değişkenleri #

“değişken” komutu, belirli bir ad alanı değişkeninin oluşturulmasını sağlar. Kendisine bir değer atanana kadar değişkenin değeri tanımsızdır:

namespace eval mynamespace {
    variable alpha
    set alpha 0
}

Değişkene ad alanının dışından (aslında herhangi bir yerden) ad alanının adını ekleyerek erişilebilir:

set ::mynamespace::alpha

Erişim, bir prosedür içinde tekrar “değişken” komutu kullanılarak basitleştirilebilir:

proc ::mynamespace::myproc {} {
    variable alpha
    set alpha
}

Bu, ad alanı değişkeni için yerel bir takma ad oluşturur.

Başka bir ad alanında tanımlanan bir prosedür için, değişken adı, “değişken” çağrısında ad alanını içermelidir:

proc myproc {} {
    variable ::mynamespace::alpha
    set alpha
}
