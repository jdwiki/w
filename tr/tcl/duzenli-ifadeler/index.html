Sözdizimi #

  • normal ifade ?anahtarlar? exp dize ?matchVar? ?subMatchVar subMatchVar …?
  • regsub ?anahtarlar? exp string subSpec ?varName?

Bu konu, normal ifadelerin kendilerini tartışmak için tasarlanmamıştır. İnternette düzenli ifadeler ve düzenli ifadeler oluşturmaya yardımcı olacak araçları açıklayan birçok kaynak vardır.

Bu konu, Tcl’de normal ifadeleri kullanmanın yaygın anahtarlarını ve yöntemlerini ve Tcl ile diğer normal ifade motorları arasındaki bazı farkları ele almaya çalışacaktır.

Normal ifadeler genellikle yavaştır. Sormanız gereken ilk soru “Gerçekten düzenli bir ifadeye ihtiyacım var mı?”. Sadece istediğinizi eşleştirin. Diğer verilere ihtiyacınız yoksa, eşleştirmeyin.

Bu normal ifade örneklerinin amaçları doğrultusunda, normal ifadeyi yorumlayabilmek ve açıklayabilmek için -genişletilmiş anahtarı kullanılacaktır.

Eşleştirme #

‘regexp’ komutu, normal bir ifadeyi bir dizeyle eşleştirmek için kullanılır.

# This is a very simplistic e-mail matcher.
# e-mail addresses are extremely complicated to match properly.
# there is no guarantee that this regex will properly match e-mail addresses.
set mydata "send mail to john.doe.the.23rd@no.such.domain.com please"
regexp -expanded {
    \y           # word boundary
    [^@\s]+      # characters that are not an @ or a space character
    @            # a single @ sign
    [\w.-]+      # normal characters and dots and dash
    \.           # a dot character
    \w+          # normal characters.
    \y           # word boundary
    } $mydata emailaddr
puts $emailaddr
john.doe.the.23rd@no.such.domain.com

‘regexp’ komutu, bir eşleşme yapılmışsa 1 (doğru) değerini, değilse 0 (yanlış) değerini döndürür.

set mydata "hello wrld, this is Tcl"
# faster would be to use: [string match *world* $mydata] 
if { [regexp {world} $mydata] } {
   puts "spelling correct"
} else {
   puts "typographical error"
}

Bazı verilerdeki tüm ifadeleri eşleştirmek için verileri döndürmek için -all anahtarını ve -inline anahtarını kullanın. Varsayılanın yeni satırları diğer veriler gibi ele almak olduğunu unutmayın.

# simplistic english ordinal word matcher.
set mydata {
    This is the first line.
    This is the second line.
    This is the third line.
    This is the fourth line.
    }
set mymatches [regexp -all -inline -expanded {
    \y                  # word boundary
    \w+                 # standard characters
    (?:st|nd|rd|th)     # ending in st, nd, rd or th
                        # The ?: operator is used here as we don't
                        # want to return the match specified inside
                        # the grouping () operator.
    \y                  # word boundary
    } $mydata]
puts $mymatches
first second third fourth
# if the ?: operator was not used, the data returned would be:
first st second nd third rd fourth th

Yeni satır işleme

# find real numbers at the end of a line (fake data).
set mydata {
    White 0.87 percent saturation.
    Specular reflection: 0.995
    Blue 0.56 percent saturation.
    Specular reflection: 0.421
    }
# the -line switch will enable newline matching.
# without -line, the $ would match the end of the data.
set mymatches [regexp -line -all -inline -expanded {
    \y                  # word boundary
    \d\.\d+             # a real number
    $                   # at the end of a line.
    } $mydata]
puts $mymatches
0.995 0.421

Açgözlü ve Açgözlü Olmayan Niceleyicileri Karıştırma #

İlk niceleyici olarak açgözlü bir eşleşmeniz varsa, RE’nin tamamı açgözlü olacaktır,

İlk niceleyici olarak açgözlü olmayan eşleşmeniz varsa, RE’nin tamamı açgözlü olmayacaktır.

set mydata {
    Device widget1: port: 156 alias: input2
    Device widget2: alias: input1 
    Device widget3: port: 238 alias: processor2
    Device widget4: alias: output2
    }
regexp {Device\s(\w+):\s(.*?)alias} $mydata alldata devname devdata
puts "$devname $devdata"
widget1 port: 156 alias: input2
regexp {Device\s(.*?):\s(.*?)alias} $mydata alldata devname devdata
puts "$devname $devdata" 
widget1 port: 156

İlk durumda, ilk \w+ açgözlüdür, bu nedenle tüm niceleyiciler açgözlü olarak işaretlenir ve .*? beklenenden daha fazla maç.

İkinci durumda, ilk .*? açgözlü değildir ve tüm niceleyiciler açgözlü olmayan olarak işaretlenir.

Diğer düzenli ifade motorlarının açgözlü/açgözlü olmayan niceleyicilerle ilgili bir sorunu olmayabilir, ancak bunlar çok daha yavaştır.

İkame #

‘regsub’ komutu, düzenli ifade eşleştirme ve değiştirme için kullanılır.

set mydata {The yellow dog has the blues.}
# create a new string; only the first match is replaced.
set newdata [regsub {(yellow|blue)} $mydata green]
puts $newdata
The green dog has the blues.
# replace the data in the same string; all matches are replaced
regsub -all {(yellow|blue)} $mydata red mydata
puts $mydata
The red dog has the reds.
# another way to create a new string
regsub {(yellow|blue)} $mydata red mynewdata
puts $mynewdata
The red dog has the blues.

Eşleşen verilere başvurmak için geri referansları kullanma.

set mydata {The yellow dog has the blues.}
regsub {(yellow)} $mydata {"\1"} mydata
puts $mydata
The "yellow" dog has the blues.

Tcl’nin RE motoru ile diğer RE motorları arasındaki farklar. #

  • \m : Bir kelimenin başlangıcı.
  • \M : Bir kelimenin sonu.
  • \y : Kelime sınırı.
  • \Y : kelime sınırı olmayan bir nokta.
  • \Z : verilerin sonuyla eşleşir.

Belgeler: re_syntax

Bir hazır bilgi dizesini normal bir ifadeyle eşleştirme #

Bazen, RE meta karakterlerini içeren alt dizeye rağmen, bir değişmez (alt) dizeyi normal bir ifadeyle eşleştirmeniz gerekir. Evet olsa da, bu işi yapmak için uygun ters eğik çizgiler eklemek için kod yazmak mümkündür (“dize haritası” kullanarak), en kolayı desenin önüne “***=” koymaktır, bu da RE motorunun dizenin geri kalanını işlemesini sağlar sadece gerçek karakterler olarak, all diğer meta karakterleri devre dışı bırakır.

set sampleText "This is some text with \[brackets\] in it."
set searchFor {[brackets]}

if {[ regexp ***=$searchFor $sampleText ]} {
    # This message will be printed
    puts "Found it!"
}

Bunun aynı zamanda çapalardan hiçbirini kullanamayacağınız anlamına geldiğini unutmayın.
