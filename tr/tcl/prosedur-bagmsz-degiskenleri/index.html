Referanslar: proc Argüman Genişletme (bölüm 5)

Argümanları kabul etmeyen bir prosedür #

proc myproc {} {
    puts "hi"
}
myproc
# => hi

Boş bir argüman listesi (prosedür adından sonraki ikinci argüman, “myproc”), prosedürün argümanları kabul etmeyeceği anlamına gelir.

Değişken sayıda argüman kabul eden bir prosedür #

### Definition
proc myproc {alpha {beta {}} {gamma green}} {
    puts [list $alpha $beta $gamma]
}

### Use
myproc A
# => A {} green
myproc A B
# => A B green
myproc A B C
# => A B C

Bu prosedür bir, iki veya üç bağımsız değişkeni kabul eder: adları iki öğeli bir listedeki ilk öğe olan parametreler isteğe bağlıdır. Parametre değişkenleri (“alfa”, “beta”, “gama”), soldan sağa atanan mümkün olduğu kadar çok argüman değeri alır. Herhangi bir bağımsız değişken değeri almayan parametre değişkenleri, bunun yerine değerlerini, parçası oldukları listedeki ikinci öğeden alır.

İsteğe bağlı bağımsız değişkenlerin bağımsız değişken listesinin sonunda gelmesi gerektiğini unutmayın. N-1 bağımsız değişkeni isteğe bağlıysa, N bağımsız değişkeni de isteğe bağlı olmalıdır. KullanıcınınN argümanına sahip olduğu ancakN-1 argümanının olmadığı bir durumda, argümanın varsayılan değeriN-1 açıkça belirtilmelidir. N argümanından önce, prosedürü çağırırken.

myproc A B C D
# (ERROR) wrong # args: should be "myproc alpha ?beta? ?gamma?"

Prosedür üçten fazla bağımsız değişkeni kabul etmez: bağımsız değişken sözdizimini açıklayan yardımcı bir hata mesajının otomatik olarak oluşturulduğuna dikkat edin.

Bir değişkene bir ad/referans kabul eden bir prosedür #

proc myproc {varName alpha beta} {
    upvar 1 $varName var
    set var [expr {$var * $alpha + $beta}]
}
set foo 1
myproc foo 10 5
puts $foo
# => 15

Bu özel durumda, prosedüre mevcut kapsamdaki bir değişkenin adı verilir. Bir Tcl prosedürü içinde, bu tür değişkenler otomatik olarak görünmez, ancak ‘upvar’ komutu başka bir yığın seviyesinden bir değişken için bir takma ad oluşturabilir: 1, arayanın yığın seviyesi, #0 ise genel seviye anlamına gelir, vb. Bu durumda , yığın düzeyi 1 ve “foo” adı (“varName” parametre değişkeninden) “upvar"ın bu değişkeni bulmasını ve “var” adında bir takma ad oluşturmasını sağlar. “var” üzerindeki her okuma veya yazma işlemi, arayan kişinin yığın düzeyinde “foo"ya da olur.

İki bağımsız değişkeni kabul eden bir prosedür #

proc myproc {alpha beta} {
    ...
    set foo $alpha
    set beta $bar     ;# note: possibly useless invocation
}

myproc 12 34          ;# alpha will be 12, beta will be 34

Argüman listesi kelimelerden oluşuyorsa, bunlar prosedürdeki yerel değişkenlerin isimleri olacak ve başlangıç ​​değerleri komut satırındaki argüman değerlerine eşit olacaktır. Argümanlar değere göre iletilir ve prosedür içindeki değişken değerlerine ne olursa olsun, prosedürün dışındaki verilerin durumunu etkilemez.

Herhangi bir sayıda argümanı kabul eden bir prosedür #

proc myproc args { ... }
proc myproc {args} { ... } ;# equivalent

Özel parametre adı “args” argüman listesindeki son öğeyse, komut satırında o noktada tüm argümanların bir listesini alır. Hiçbiri yoksa, liste boştur.

“args"dan önce isteğe bağlı olanlar da dahil olmak üzere argümanlar olabilir:

proc myproc {alpha {beta {}} args} { ... }

Bu prosedür bir veya daha fazla argümanı kabul edecektir. Varsa ilk ikisi “alfa” ve “beta” tarafından tüketilecektir: geri kalan argümanların listesi “args"a atanacaktır.

{*} sözdizimi #

Bazen sahip olduğunuz şey bir listedir, ancak listedeki öğeleri iletmek istediğiniz komut, her bir öğeyi ayrı bir argüman olarak almak için talep eder. Örneğin: “winfo children” komutu bir pencere listesi döndürür, ancak “destroy” komutu yalnızca bir dizi pencere adı argümanı alır.

set alpha [winfo children .]
# => .a .b .c
destroy $alpha
# (no response, no windows are destroyed)

Çözüm, “{*}” sözdizimini kullanmaktır:

destroy {*}[winfo children .]

veya

destroy {*}$alpha

{*} sözdiziminin yaptığı şey, aşağıdaki değeri almak (arada boşluk yok!) ve bu değerdeki öğeleri ayrı argümanlarmış gibi komut satırına eklemek.

Aşağıdaki değer boş bir listeyse, hiçbir şey eklenmez:

puts [list a b {*}{} c d]
# => a b c d

Bir veya daha fazla öğe varsa, bunlar eklenir:

puts [list a b {*}{1 2 3} c d]
# => a b 1 2 3 c d
