Bir DLL’yi Sarma: C++‘dan Cython’dan Python’a #

Bu, bir C++ dll’yi Cython ile sarmanın önemsiz bir örneğini gösterir. Aşağıdaki ana adımları kapsayacaktır:

  • Visual Studio kullanarak C++ ile örnek bir DLL oluşturun.
  • Python’da çağrılabilmesi için DLL’yi Cython ile sarın.

Cython’un kurulu olduğu ve Python’a başarıyla aktarabileceğiniz varsayılmaktadır.

DLL adımı için, Visual Studio’da bir DLL oluşturmaya aşina olduğunuz da varsayılır.

Tam örnek, aşağıdaki dosyaların oluşturulmasını içerir:

  1. complexFunLib.h: C++ DLL kaynağı için başlık dosyası
  2. complexFunLib.cpp: C++ DLL kaynağı için CPP dosyası
  3. ccomplexFunLib.pxd: Cython “başlık” dosyası
  4. complexFunLib.pyx: Cython “sarmalayıcı” dosyası
  5. setup.py: Cython ile complexFunLib.pyd oluşturmak için Python kurulum dosyası
  6. run.py: Derlenmiş, Cython sarılı DLL dosyasını içe aktaran örnek Python dosyası

C++ DLL Kaynak: “complexFunLib.h” ve “complexFunLib.cpp” #

Zaten bir DLL ve başlık kaynak dosyanız varsa bunu atlayın. İlk olarak, Visual Studio kullanarak DLL’nin derleneceği C++ kaynağını oluşturuyoruz. Bu durumda karmaşık üstel fonksiyon ile hızlı dizi hesaplamaları yapmak istiyoruz. Aşağıdaki iki işlev, sonuçların “res” içinde depolandığı “k” ve “ee” dizilerinde “k*exp(ee)” hesaplamasını gerçekleştirir. Hem tekli hem de çift hassasiyeti barındırmak için iki işlev vardır. Bu örnek işlevlerin OpenMP kullandığını unutmayın, bu nedenle proje için Visual Studio seçeneklerinde OpenMP’nin etkinleştirildiğinden emin olun.

H Dosyası

// Avoids C++ name mangling with extern "C"
#define EXTERN_DLL_EXPORT extern "C" __declspec(dllexport)  
#include <complex>
#include <stdlib.h>

// Handles 64 bit complex numbers, i.e. two 32 bit (4 byte) floating point numbers
EXTERN_DLL_EXPORT void mp_mlt_exp_c4(std::complex<float>* k, 
                                     std::complex<float>* ee,
                                     int sz, 
                                     std::complex<float>* res, 
                                     int threads);

// Handles 128 bit complex numbers, i.e. two 64 bit (8 byte) floating point numbers
EXTERN_DLL_EXPORT void mp_mlt_exp_c8(std::complex<double>* k,                                       std::complex<double>* ee,
                                     int sz, 
                                     std::complex<double>* res, 
                                     int threads);

CPP Dosyası

#include "stdafx.h"
#include <stdio.h>
#include <omp.h>
#include "complexFunLib.h"

void mp_mlt_exp_c4(std::complex<float>* k,
                   std::complex<float>* ee,
                   int sz,
                   std::complex<float>* res,
                   int threads)
{
    // Use Open MP parallel directive for multiprocessing
    #pragma omp parallel num_threads(threads)
    {
        #pragma omp for
        for (int i = 0; i < sz; i++) res[i] = k[i] * exp(ee[i]);
    }
}

void mp_mlt_exp_c8(std::complex<double>* k,
                   std::complex<double>* ee,
                   int sz, std::complex<double>* res,
                   int threads)
{
    // Use Open MP parallel directive for multiprocessing
    #pragma omp parallel num_threads(threads)
    {
        #pragma omp for
        for (int i = 0; i < sz; i++) res[i] = k[i] * exp(ee[i]);
    }
}

Cython Kaynağı: “ccomplexFunLib.pxd” ve “complexFunLib.pyx” #

Ardından, C++ DLL dosyasını sarmak için gerekli Cython kaynak dosyalarını oluşturuyoruz. Bu adımda, aşağıdaki varsayımları yaparız:

  • Cython’u kurdunuz
  • Çalışan bir DLL’niz var, örn. yukarıda anlatılan

Nihai hedef, bu Cython kaynak dosyalarını orijinal DLL ile birlikte kullanarak Python modülü olarak içe aktarılabilen ve C++ ile yazılmış işlevleri ortaya çıkaran bir `.pyd’ dosyasını derlemek için kullanmaktır.

PXD Dosyası

Bu dosya, C++ başlık dosyasına karşılık gelir. Çoğu durumda, Cython’a özgü küçük değişikliklerle başlığı bu dosyaya kopyalayıp yapıştırabilirsiniz. Bu durumda, spesifik Cython kompleksi türleri kullanıldı. “ccomplexFunLib.pxd” dosyasının başına “c” eklenmesine dikkat edin. Bu gerekli değildir, ancak böyle bir adlandırma kuralının organizasyonun korunmasına yardımcı olduğunu gördük.

cdef extern from "complexFunLib.h":
    void mp_mlt_exp_c4(float complex* k, float complex* ee, int sz,
                       float complex* res, int threads);
    void mp_mlt_exp_c8(double complex* k, double complex* ee, int sz,
                       double complex* res, int threads);

PYX Dosyası

Bu dosya, C++ cpp kaynak dosyasına karşılık gelir. Bu örnekte, import DLL işlevlerine Numpy ndarray nesnelerine işaretçiler ileteceğiz. Diziler için yerleşik Cython “memoryview” nesnesini kullanmak da mümkündür, ancak performansı “ndarray” nesneleri kadar iyi olmayabilir (ancak sözdizimi önemli ölçüde daha temizdir).

cimport ccomplexFunLib  # Import the pxd "header"
# Note for Numpy imports, the C import most come AFTER the Python import
import numpy as np  # Import the Python Numpy
cimport numpy as np  # Import the C Numpy

# Import some functionality from Python and the C stdlib
from cpython.pycapsule cimport *

# Python wrapper functions.
# Note that types can be delcared in the signature

def mp_exp_c4(np.ndarray[np.complex64_t, ndim=1] k,
              np.ndarray[np.complex64_t, ndim=1] ee,
              int sz,
              np.ndarray[np.complex64_t, ndim=1] res,
              int threads):
    '''
    TODO: Python docstring
    '''
    # Call the imported DLL functions on the parameters.
    # Notice that we are passing a pointer to the first element in each array
    ccomplexFunLib.mp_mlt_exp_c4(&k[0], &ee[0], sz, &res[0], threads)
    
def mp_exp_c8(np.ndarray[np.complex128_t, ndim=1] k,
              np.ndarray[np.complex128_t, ndim=1] ee,
              int sz,
              np.ndarray[np.complex128_t, ndim=1] res,
              int threads):
    '''
    TODO: Python docstring
    '''
    ccomplexFunLib.mp_mlt_exp_c8(&k[0], &ee[0], sz, &res[0], threads)

Python Kaynağı: “setup.py” ve “run.py” #

setup.py

Bu dosya, Cython derlemesini yürüten bir Python dosyasıdır. Amacı, daha sonra Python modülleri tarafından içe aktarılabilecek derlenmiş .pyd dosyasını oluşturmaktır. Bu örnekte, gerekli tüm dosyaları (yani ‘complexFunLib.h’, ‘complexFunLib.dll’, ‘ccomplexFunLib.pxd’ ve ‘complexFunLib.pyx’) ‘setup.py’ ile aynı dizinde tuttuk.

Bu dosya oluşturulduktan sonra, komut satırından şu parametrelerle çalıştırılmalıdır: build_ext --inplace

Bu dosya yürütüldüğünde, herhangi bir hata oluşturmadan bir .pyd dosyası üretmelidir. Bazı durumlarda bir hata olması durumunda .pyd oluşturulabileceğini ancak geçersiz olduğunu unutmayın. Oluşturulan “.pyd"yi kullanmadan önce “setup.py"nin yürütülmesinde herhangi bir hata oluşmadığından emin olun.

from distutils.core import setup
from distutils.extension import Extension
from Cython.Distutils import build_ext
import numpy as np

ext_modules = [
    Extension('complexFunLib',
              ['complexFunLib.pyx'],
              # Note here that the C++ language was specified
              # The default language is C
              language="c++",  
              libraries=['complexFunLib'],
              library_dirs=['.'])
    ]

setup(
    name = 'complexFunLib',
    cmdclass = {'build_ext': build_ext},
    ext_modules = ext_modules,
    include_dirs=[np.get_include()]  # This gets all the required Numpy core files
)

run.py

Şimdi ‘complexFunLib’ doğrudan bir Python modülüne aktarılabilir ve sarılmış DLL işlevleri çağrılır.

import complexFunLib
import numpy as np

# Create arrays of non-trivial complex numbers to be exponentiated,
# i.e. res = k*exp(ee)
k = np.ones(int(2.5e5), dtype='complex64')*1.1234 + np.complex64(1.1234j)
ee = np.ones(int(2.5e5), dtype='complex64')*1.1234 + np.complex64(1.1234j) 
sz = k.size  # Get size integer
res = np.zeros(int(2.5e5), dtype='complex64')  # Create array for results

# Call function
complexFunLib.mp_exp_c4(k, ee, sz, res, 8)  

# Print results
print(res)
