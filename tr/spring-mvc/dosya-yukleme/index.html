Sözdizimi #

  • @RequestPart(String, String, boolean)

Parametreler #

parametre Ayrıntılar
@RequestPart Bu açıklama, bir parametrenin belirli bir istek parçasına eşlenmesi gerektiğini belirtir. Parça adı, bunu @RequestPart argümanı olarak sağlamayı seçmediğiniz sürece *yöntem parametresinin adıyla eşleşmelidir. Parça adı bir Java adı olarak ifade edilemezse (ör. “123”), gerçek adı belirtmek için “@RequestPart"ın “değer” niteliğini kullanabilirsiniz. Örneğin. @RequestPart("123") Dize _123.

Java’nın daha eski bir sürümünde (1.7 öncesi) çalışıyorsanız veya hata ayıklama bilgisi olmadan derliyorsanız, Java parametrenin adını ‘arg0’, ‘arg1’ vb. ile değiştirir, bu da Spring’in kullanılmasını engelleyecektir. bunları parça adlarıyla eşleştirebilir. Bu durumda, Parametreler’de belgelendiği gibi @RequestPart ek açıklamasında parçanın adını ayarlamanız gerekecektir.

Tek bir dosya yükleme #

HTTP Gönderisi aracılığıyla yüklenen bir dosyayı almak için aşağıdakileri yapmanız gerekir:

@RequestMapping(
    value = "...",
    method = RequestMethod.POST,
    consumes = MediaType.MULTIPART_FORM_DATA_VALUE
)
public Object uploadFile(
    @RequestPart MultipartFile file
) {
    String fileName = file.getOriginalFilename();
    InputStream inputStream = file.getInputStream();
    String contentType = file.getContentType();
    .
    .
    .
}

@RequestPart parametresinin adının, istekteki parçanın adıyla eşleşmesi gerektiğini unutmayın.

HTML olarak:

 <form action="/..." enctype="multipart/form-data" method="post">
        <input type="file" name="file">
    </form>

HTML olarak (Spring TagLibs):

<form action="/..." enctype="multipart/form-data" method="post">
    <form:input type="file" path="file">
</form>

Ham HTTP isteği olarak:

POST /... HTTP/1.1
Host: ...
Content-Type: multipart/form-data; boundary=----------287032381131322

------------287032381131322
Content-Disposition: form-data; name="file"; filename="r.gif"
Content-Type: image/gif

GIF87a.............,...........D..;
------------287032381131322--

Bu istek şu anlama gelir:

fileName == "r.gif"
contentType == "image/gif"

İlkbaharda MVC

Çok parçalı işlevselliğe erişmek için belirtilen fasulyeyi eklemeniz gerekiyor 

    <!-- max size of file in memory (in bytes) -->
    <property name="maxInMemorySize" value="1048576" /> <!-- 1MB -->

</bean>

Bir parçayı bir nesneye sıralama #

Bir parçanın içeriğini bir etki alanı nesnesine dönüştürmek istiyorsanız (örneğin, bir “Kullanıcı” veya “Hesap” veya “Adres”), işlem çok basittir:

Her biri farklı bir ada sahip birden fazla parça yüklemek mümkündür. Her parça adı için, adı parça adıyla eşleşen “@RequestPart” ile açıklamalı bir parametreye ihtiyacınız olacaktır.

HTTP Gönderisi aracılığıyla yüklenen bir dosyayı almak için aşağıdakileri yapmanız gerekir:

@RequestMapping(
    value = "...",
    method = RequestMethod.POST,
    consumes = MediaType.MULTIPART_FORM_DATA_VALUE
)
public Object uploadFile(
    @RequestPart Address address,
) {
    .
    .
    .
}

Ham HTTP isteği olarak:

POST /... HTTP/1.1
Host: ...
Content-Type: multipart/form-data; boundary=----------287032381131322

------------287032381131322
Content-Disposition: form-data; name="address"; filename="address.json"
Content-Type: application/json

{"houseNumber": "10/A", "streetName": "Dumbldore Road", "town": "Hogsmede"}
------------287032381131322--

En önemli şeyler şunlardır:

  • Parçanın adı, değişkenin adıyla eşleşmelidir.
  • Parçanın ‘İçerik Türü’, onu normal bir istek olarak göndermiş olsaydınız, Spring’in işleyebileceği türden olmalıdır. Yani, bir uç noktaya ‘Foo/bar’ ‘Content-Type’ ile bir ‘POST’ gerçekleştirebilirseniz ve Spring bunu bir nesneye dönüştürebilirse, o zaman aynı zamanda bir parçayı da sıraya koyabilecektir. bir obje.
  • Parçanın ‘İçerik-Türünü’ belirleyebilmelisiniz. Yapamazsanız, bu yaklaşım işe yaramaz - Spring, parçanın “İçerik Türü"nü tahmin etmeye çalışmaz.

Birden fazla dosya yükleme #

Tek bir HTTP Gönderisi aracılığıyla yüklenen birden fazla dosyayı almak için aşağıdakileri yapmanız gerekir:

@RequestMapping(
    value = "...",
    method = RequestMethod.POST,
    consumes = MediaType.MULTIPART_FORM_DATA_VALUE
)
public Object uploadFile(
    @RequestPart MultipartFile[] files
) {
    for (file : files) {
        String fileName = file.getOriginalFilename();
        InputStream inputStream = file.getInputStream();
        String contentType = file.getContentType();
        .
        .
        .
    }
}

@RequestPart parametresinin adının, istekteki parçanın adıyla eşleşmesi gerektiğini unutmayın.

HTML olarak:

<form action="/..." enctype="multipart/form-data" method="post">
    <input type="file" name="files">
    <input type="file" name="files">
</form>

Ham HTTP isteği olarak:

POST /... HTTP/1.1
Host: ...
Content-Type: multipart/form-data; boundary=----------287032381131322

------------287032381131322
Content-Disposition: form-data; name="files"; filename="r.gif"
Content-Type: image/gif

GIF87a.............,...........D..;
------------287032381131322
Content-Disposition: form-data; name="files"; filename="banana.jpeg"
Content-Type: image/jpeg

GIF87a.............,...........D..;
------------287032381131322--

Bu istek şu anlama gelir:

files[0].getOriginalFilename() == "r.gif"
files[0].getContentType() == "image/gif"
files[1].getOriginalFilename() == "r.jpeg"
files[1].getContentType() == "image/jpeg"
