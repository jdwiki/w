Bir monad, birleştirilebilir eylemlerin bir veri türüdür. “Monad”, değerleri bu tür eylemleri temsil eden tür oluşturucular sınıfıdır. Belki de ‘IO’ en çok tanınanıdır: ‘IO a’ değeri, “gerçek dünyadan bir ‘a’ değeri almak için bir reçetedir”.

Eylemlerin bileşimiyle ilgili belirli yasaları karşılayan bir “Monad m örneği” varsa, bir tür oluşturucu “m” ("[]" veya “Belki” gibi) bir monad oluşturur deriz. O zaman m a hakkında “sonucu a tipinde bir eylem” olarak akıl yürütebiliriz.

Monadik bir hesaplamadan değer çıkarmanın genel bir yolu yok #

Değerleri eylemlere sarabilir ve bir hesaplamanın sonucunu diğerine aktarabilirsiniz:

return :: Monad m => a -> m a
(>>=)  :: Monad m => m a -> (a -> m b) -> m b

Ancak, bir Monad’ın tanımı, “Monad m => m a -> a” türünde bir fonksiyonun varlığını garanti etmez.

Bu, genel olarak, bir hesaplamadan bir değer çıkarmanın hiçbir yolu olmadığı anlamına gelir (yani, onu “açın”). Bu, birçok örnek için geçerlidir:

extract :: Maybe a -> a
extract (Just x) = x          -- Sure, this works, but...
extract Nothing  = undefined  -- We can’t extract a value from failure.

Spesifik olarak, genellikle yeni başlayanların kafasını karıştıran ‘IO a -> a’ işlevi yoktur; [bu örneğe](https://www.wikiod.com/tr/haskell/monadlar#GÇ monad) bakın.

Uygulamanın Alt Sınıfı Olarak Monad #

GHC 7.10’dan itibaren, “Uygulamalı”, “Monad"ın bir üst sınıfıdır (yani, “Monad” olan her tür aynı zamanda “Uygulamalı” olmalıdır). ‘Uygulamalı’ (‘pure’, ‘<*>’) tüm yöntemleri, ‘Monad’ (‘return’, ‘»=’) yöntemleri açısından uygulanabilir.

‘Saf’ ve ‘dönüş’ün eşdeğer amaçlara hizmet ettiği açıktır, bu nedenle ‘saf = dönüş’. “<*>” tanımı çok açık:

mf <*> mx = do { f <- mf; x <- mx; return (f x) }                 
       -- = mf >>= (\f -> mx >>= (\x -> return (f x)))
       -- = [r   | f <- mf, x <- mx, r <- return (f x)]   -- with MonadComprehensions
       -- = [f x | f <- mf, x <- mx]

Bu işlev, standart kitaplıklarda ‘ap’ olarak tanımlanır.

Bu nedenle, bir tür için zaten bir ‘Monad’ örneği tanımladıysanız, tanımlayarak bunun için bir ‘Uygulamalı’ örneğini “ücretsiz” olarak etkili bir şekilde alabilirsiniz.

instance Applicative < type > where
    pure  = return
    (<*>) = ap

Monad yasalarında olduğu gibi, bu eşdeğerlikler uygulanmaz, ancak geliştiriciler bunların her zaman desteklendiğinden emin olmalıdır.

Monad’ın Tanımı #

class Monad m where
    return :: a -> m a
    (>>=) :: m a -> (a -> m b) -> m b

Monad’larla uğraşmanın en önemli işlevi bağlama operatörü >>=‘dır:

(>>=) :: m a -> (a -> m b) -> m b
  • m a kelimesini "a sonucu olan bir eylem” olarak düşünün.
  • ‘a -> m b’yi *“(bir ‘a’ parametresine bağlı olarak) ‘b’ sonucu olan bir eylem” olarak düşünün.

>>= sonucu ilk işlemden ikinciye aktararak iki işlemi birlikte sıralar.

‘Monad’ tarafından tanımlanan diğer işlev:

return :: a -> m a

Adı talihsiz: Bu “dönüş"ün, zorunlu programlama dillerinde bulunan “dönüş” anahtar sözcüğüyle hiçbir ilgisi yok.

‘return x’ sonucu olarak ‘x’ veren önemsiz eylemdir. ([Aşağıdaki anlamda önemsizdir](https://www.wikiod.com/tr/haskell/monadlar#belki monad -yasalar-ve-belki-monad):)

return x >>= f       ≡  f x     --  “left identity” monad law
       x >>= return  ≡  x       -- “right identity” monad law

Belki monad #

“Belki”, muhtemelen boş değerleri temsil etmek için kullanılır - diğer dillerdeki “boş"a benzer. Genellikle, bir şekilde başarısız olabilecek işlevlerin çıktı türü olarak kullanılır.

Aşağıdaki işlevi göz önünde bulundurun:

halve :: Int -> Maybe Int
halve x
  | even x = Just (x `div` 2)
  | odd x  = Nothing

‘Yarıya’yı, tamsayıyı yarıya indirmeye çalışan, tek ise başarısız olan bir ‘Int’e bağlı olarak bir eylem olarak düşünün.

Bir tamsayıyı üç kez nasıl “yarıya böleriz”?

takeOneEighth :: Int -> Maybe Int            -- (after you read the 'do' sub-section:)
takeOneEighth x =                
  case halve x of                               --  do {
    Nothing -> Nothing
    Just oneHalf ->                             --     oneHalf    <- halve x
      case halve oneHalf of
        Nothing -> Nothing
        Just oneQuarter ->                      --     oneQuarter <- halve oneHalf
          case halve oneQuarter of
            Nothing -> Nothing                  --     oneEighth  <- halve oneQuarter
            Just oneEighth ->                         
              Just oneEighth                    --     return oneEighth }
  • “takeOneEighth”, birbirine zincirlenmiş üç “yarım” adımdan oluşan bir dizidir.
  • Bir “halve” adımı başarısız olursa, “takeOneEighth” kompozisyonunun tamamının başarısız olmasını isteriz.
  • Bir “yarım” adım başarılı olursa, sonucunu ileriye aktarmak isteriz.

instance Monad Maybe where
  -- (>>=) :: Maybe a -> (a -> Maybe b) -> Maybe b
  Nothing >>= f  = Nothing                            -- infixl 1 >>=
  Just x  >>= f  = Just (f x)                         -- also, f =<< m = m >>= f
  
  -- return :: a -> Maybe a
  return x       = Just x

ve şimdi yazabiliriz:

takeOneEighth :: Int -> Maybe Int
takeOneEighth x = halve x >>= halve >>= halve             -- or,
    -- return x >>= halve >>= halve >>= halve             -- which is parsed as
    -- (((return x) >>= halve) >>= halve) >>= halve       -- which can also be written as
    -- (halve =<<) . (halve =<<) . (halve =<<) $ return x    -- or, equivalently, as
    --  halve <=<     halve <=<     halve      $        x

Kleisli kompozisyon <=<, (g <=< f) x = g =<< f x veya eşdeğer olarak (f >=> g) x = f x >>= g olarak tanımlanır. Bununla birlikte yukarıdaki tanım adil olur

takeOneEighth :: Int -> Maybe Int
takeOneEighth = halve <=< halve <=< halve               -- infixr 1 <=<
        -- or, equivalently,                    
        --      halve >=> halve >=> halve               -- infixr 1 >=>

Her monad, yani “Monad” tip sınıfının bir örneği olan her tip tarafından uyulması gereken üç monad yasası vardır:

1.  return x >>= f  =  f x
2.    m >>= return  =  m
3. (m >>= g) >>= h  =  m >>= (\y -> g y >>= h)

“m” bir monad olduğunda, “f”, “a -> m b” türünde ve “g”, “b -> m c” türündedir.

Veya eşdeğer olarak, yukarıda tanımlanan >=> Kleisli kompozisyon operatörünü kullanarak:

1.    return >=> g  =  g                    -- do { y <- return x ; g y } == g x
2.    f >=> return  =  f                    -- do { y <- f x ; return y } == f x
3. (f >=> g) >=> h  =  f >=> (g >=> h)      -- do { z <- do { y <- f x; g y } ; h z }
                                            --  == do { y <- f x ; do { z <- g y; h z } }

Bu yasalara uymak, monad hakkında akıl yürütmeyi çok daha kolaylaştırır, çünkü monadik işlevleri kullanmanın ve bunları oluşturmanın diğer monadlara benzer şekilde makul bir şekilde davranmasını garanti eder.

“Belki” monadının üç monad yasasına uyup uymadığını kontrol edelim.

  1. Sol özdeşlik yasası - dönüş x >>= f = f x

    return z »= f = (Just z) »= f = f z

  2. Doğru kimlik yasası - m >>= dönüş = m

  • “Sadece” veri oluşturucu

    Just z »= return = return z = Just z

  • “Hiçbir şey” veri oluşturucusu

    Nothing »= return = Nothing

  1. Birleşme yasası - (m >>= f) >>= g = m >>= (\x -> f x >>= g)

  • “Sadece” veri oluşturucu

    – Left-hand side ((Just z) »= f) »= g = f z »= g

    – Right-hand side (Just z) »= (\x -> f x »= g) (\x -> f x »= g) z = f z »= g

  • “Hiçbir şey” veri oluşturucusu

    – Left-hand side (Nothing »= f) »= g = Nothing »= g = Nothing

    – Right-hand side Nothing »= (\x -> f x »= g) = Nothing

ben monad #

‘IO a’ türündeki bir ifadeden ‘a’ türünde bir değer elde etmenin bir yolu yoktur ve olmamalıdır. Bu aslında monadların “IO"yu modellemek için neden kullanıldığının büyük bir kısmıdır.

“IO a” türünde bir ifadenin, gerçek dünyayla etkileşime girebilen ve yürütülürse “a” türünden bir şeyle sonuçlanabilecek bir eylemi temsil ettiği düşünülebilir. Örneğin, başlangıç ​​bölümündeki “getLine :: IO String” işlevi, “getLine"ın altında çıkarabileceğim belirli bir dize olduğu anlamına gelmez - bu, “getLine"ın standart girdiden bir satır alma eylemini temsil ettiği anlamına gelir. .

Şaşırtıcı olmayan bir şekilde, bir Haskell programı gerçek dünya ile etkileşime giren bir hesaplamayı/eylemi temsil ettiğinden, “ana :: IO ()”.

“IO” bir monad olduğu için “IO a” türündeki ifadelere _yapabileceğiniz şeyler:

  • İlk eylemi gerçekleştiren, ürettiği değeri atan ve ardından ikinci eylemi yürüten yeni bir eylem üretmek için (>>) kullanarak iki eylemi sıralayın.

       -- print the lines "Hello" then "World" to stdout
   putStrLn "Hello" >> putStrLn "World"

  • Bazen ilk eylemde üretilen değeri atmak istemezsiniz - aslında ikinci bir eyleme beslenmesini istersiniz. Bunun için >>= var. ‘IO’ için, ‘(»=) :: IO a -> (a -> IO b) -> IO b’ tipine sahiptir.

      -- get a line from stdin and print it back out
  getLine >>= putStrLn

  • Normal bir değer alın ve verdiğiniz değeri hemen döndüren bir eyleme dönüştürün. Bu işlev, siz ‘do’ notasyonunu kullanmaya başlayana kadar daha az kullanışlıdır.

      -- make an action that just returns 5
  return 5

IO monad’daki Haskell Wiki’den daha fazlası burada.

Monad’ı Listele #

Listeler bir monad oluşturur. Buna eşdeğer bir monad başlatmaları var:

instance Monad [] where 
  return x = [x]
  xs >>= f = concat (map f xs)

Bunları hesaplamalarımızda determinizm olmayanı taklit etmek için kullanabiliriz. ‘xs »= f’ kullandığımızda, ‘f :: a -> [b]’ işlevi, ‘xs’ listesi üzerinde eşlenir ve her öğe üzerindeki her bir ‘f’ uygulamasının sonuçlarının bir listesini alır. ‘xs’ ve tüm sonuç listeleri daha sonra tüm sonuçların tek bir listesinde birleştirilir. Örnek olarak, do-notation‘ı kullanarak deterministik olmayan iki sayının toplamını hesaplıyoruz; toplam, iki listedeki tüm tam sayı çiftlerinin toplamlarının listesiyle temsil ediliyor, her liste bir olmayanın olası tüm değerlerini temsil ediyor. deterministik sayı:

sumnd xs ys = do
  x <- xs
  y <- ys
  return (x + y)

Veya eşdeğer olarak, ‘Control.Monad’ içinde ’liftM2’ kullanarak:

sumnd = liftM2 (+)

elde ederiz:

> sumnd [1,2,3] [0,10]
[1,11,2,12,3,13]

notasyon #

“do” gösterimi, monadlar için sözdizimsel şekerdir. İşte kurallar:

do x <- mx do x <- mx
y <- my, y <- my ile eşdeğerdir
... ...

a = b'ye izin verin, a = b'ye izin verin
... yapmak eşdeğerdir ...

yap >> (
e, e'ye eşittir)

do x <- m m >>= (\x ->
e, e'ye eşittir)

do m, m'ye eşittir

Örneğin, bu tanımlar eşdeğerdir:

example :: IO Integer
example =
  putStrLn "What's your name?" >> (
    getLine >>= (\name ->
      putStrLn ("Hello, " ++ name ++ ".") >> (
        putStrLn "What should we return?" >> (
          getLine >>= (\line ->
            let n = (read line :: Integer) in
              return (n + n))))))

example :: IO Integer
example = do
  putStrLn "What's your name?"
  name <- getLine
  putStrLn ("Hello, " ++ name ++ ".")
  putStrLn "What should we return?"
  line <- getLine
  let n = (read line :: Integer)
  return (n + n)
