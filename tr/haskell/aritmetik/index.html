Haskell’de, tüm ifadeler (sayısal sabitleri ve bunlar üzerinde çalışan işlevleri içerir) karar verilebilir bir türe sahiptir. Derleme zamanında, tür denetleyici, onu oluşturan temel işlevlerin türlerinden bir ifadenin türünü çıkarır. Veriler varsayılan olarak değişmez olduğu için, “tür döküm” işlemleri yoktur, ancak verileri kopyalayan ve türleri mantık dahilinde genelleştiren veya özelleştiren işlevler vardır.

Sayısal typeclass hiyerarşisi #

“Num”, sayısal tür sınıfı hiyerarşisinin kökünde yer alır. Karakteristik işlemleri ve bazı genel örnekleri aşağıda gösterilmiştir (varsayılan olarak Prelude ile yüklenenler artı Data.Complexinkiler):

λ> :i Num
class Num a where
  (+) :: a -> a -> a
  (-) :: a -> a -> a
  (*) :: a -> a -> a
  negate :: a -> a
  abs :: a -> a
  signum :: a -> a
  fromInteger :: Integer -> a
  {-# MINIMAL (+), (*), abs, signum, fromInteger, (negate | (-)) #-}
      -- Defined in ‘GHC.Num’
instance RealFloat a => Num (Complex a) -- Defined in ‘Data.Complex’
instance Num Word -- Defined in ‘GHC.Num’
instance Num Integer -- Defined in ‘GHC.Num’
instance Num Int -- Defined in ‘GHC.Num’
instance Num Float -- Defined in ‘GHC.Float’
instance Num Double -- Defined in ‘GHC.Float’

“Sayı” gerektiren ve “bölme” “(/)” ve bir sayının tersi kavramlarını tanıtan “Kesirli” sınıfını daha önce görmüştük:

λ> :i Fractional
class Num a => Fractional a where
  (/) :: a -> a -> a
  recip :: a -> a
  fromRational :: Rational -> a
  {-# MINIMAL fromRational, (recip | (/)) #-}
      -- Defined in ‘GHC.Real’
instance RealFloat a => Fractional (Complex a) -- Defined in ‘Data.Complex’
instance Fractional Float -- Defined in ‘GHC.Float’
instance Fractional Double -- Defined in ‘GHC.Float’

‘Gerçek’ sınıf modelleri .. gerçek sayılar. ‘Num’ ve ‘Ord’ gerektirir, bu nedenle sıralı bir sayısal alan modeller. Bir karşı örnek olarak, Karmaşık sayılar sıralı bir alan değildir (yani, doğal bir sıralama ilişkisine sahip değildirler):

λ> :i Real
class (Num a, Ord a) => Real a where
  toRational :: a -> Rational
  {-# MINIMAL toRational #-}
      -- Defined in ‘GHC.Real’
instance Real Word -- Defined in ‘GHC.Real’
instance Real Integer -- Defined in ‘GHC.Real’
instance Real Int -- Defined in ‘GHC.Real’
instance Real Float -- Defined in ‘GHC.Float’
instance Real Double -- Defined in ‘GHC.Float’

“RealFrac” yuvarlanabilecek sayıları temsil eder

λ> :i RealFrac
class (Real a, Fractional a) => RealFrac a where
  properFraction :: Integral b => a -> (b, a)
  truncate :: Integral b => a -> b
  round :: Integral b => a -> b
  ceiling :: Integral b => a -> b
  floor :: Integral b => a -> b
  {-# MINIMAL properFraction #-}
      -- Defined in ‘GHC.Real’
instance RealFrac Float -- Defined in ‘GHC.Float’
instance RealFrac Double -- Defined in ‘GHC.Float’

“Kayan” (“Kesirli” anlamına gelir), sonlu bir ondalık genişlemeye sahip olmayabilecek sabitleri ve işlemleri temsil eder.

λ> :i Floating
class Fractional a => Floating a where
  pi :: a
  exp :: a -> a
  log :: a -> a
  sqrt :: a -> a
  (**) :: a -> a -> a
  logBase :: a -> a -> a
  sin :: a -> a
  cos :: a -> a
  tan :: a -> a
  asin :: a -> a
  acos :: a -> a
  atan :: a -> a
  sinh :: a -> a
  cosh :: a -> a
  tanh :: a -> a
  asinh :: a -> a
  acosh :: a -> a
  atanh :: a -> a
  GHC.Float.log1p :: a -> a
  GHC.Float.expm1 :: a -> a
  GHC.Float.log1pexp :: a -> a
  GHC.Float.log1mexp :: a -> a
  {-# MINIMAL pi, exp, log, sin, cos, asin, acos, atan, sinh, cosh,
              asinh, acosh, atanh #-}
      -- Defined in ‘GHC.Float’
instance RealFloat a => Floating (Complex a) -- Defined in ‘Data.Complex’
instance Floating Float -- Defined in ‘GHC.Float’
instance Floating Double -- Defined in ‘GHC.Float’

Dikkat: while sqrt gibi ifadeler. olumsuzlama :: Kayan a => a -> a tamamen geçerlidir, amaçlanan bir davranış olmayabilecek NaN (“sayı değil”) döndürebilirler. Bu gibi durumlarda, Karmaşık alanı (daha sonra gösterilecektir) üzerinde çalışmak isteyebiliriz.

Temel örnekler #

λ> :t 1
1 :: Num t => t

λ> :t pi
pi :: Floating a => a

Yukarıdaki örneklerde, tür denetleyicisi, iki sabit için somut bir tür yerine bir tür-sınıf çıkarımı yapar. Haskell’de ‘Num’ sınıfı en genel sayısal sınıftır (tam sayıları ve gerçekleri içerdiğinden), ancak ‘pi’ sıfırdan farklı bir kesirli kısma sahip olduğundan daha özel bir sınıfa ait olmalıdır.

list0 :: [Integer]
list0 = [1, 2, 3]

list1 :: [Double]
list1 = [1, 2, pi]

Yukarıdaki beton türleri GHC tarafından çıkarılmıştır. “list0 :: Num a => [a]” gibi daha genel türler işe yarayabilirdi, ancak korunması da daha zor olurdu (örneğin, bir “Sayı” listesine “Double” atanırsa), yukarıda gösterilen uyarılar.

(Kesirli İnt) ... #

Başlıktaki hata mesajı, yaygın bir başlangıç ​​hatasıdır. Nasıl ortaya çıktığını ve nasıl düzeltileceğini görelim.

Bir sayı listesinin ortalama değerini hesaplamamız gerektiğini varsayalım; aşağıdaki bildirim bunu yapıyor gibi görünebilir, ancak derlenmez:

averageOfList ll = sum ll / length ll

Sorun ‘(/)’ bölümü işlevindedir: imzası ‘(/) :: Kesirli a => a -> a -> a’dır, ancak paydanın üzerindeki durumda (uzunluk :: Katlanabilir olarak verilir) t => t a -> Int’) ‘Int’ türündedir (ve ‘Int’, ‘Kesirli’ sınıfına ait değildir), bu nedenle hata mesajı.

Hata mesajını fromIntegral :: (Num b, Integral a) => a -> b ile düzeltebiliriz. Bu işlevin herhangi bir Integral türündeki değerleri kabul ettiği ve ‘Num’ sınıfında karşılık gelen değerleri döndürdüğü görülebilir:

averageOfList' :: (Foldable t, Fractional a) => t a -> a
averageOfList' ll = sum ll / fromIntegral (length ll)

sqrt . fromIntegral :: (Integral a, Floating c) => a -> c
