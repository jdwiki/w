Bu dil uzantıları, onaylanmış Haskell 2010 dil Raporu kapsamında olmadıkları için Glasgow Haskell Derleyicisi (GHC) kullanılırken genellikle mevcuttur. Bu uzantıları kullanmak için, ya bir flag kullanarak derleyiciyi bilgilendirmeli ya da bir dosyadaki ‘module’ anahtar sözcüğünün önüne bir ‘LANGUAGE’ programı yerleştirmelisiniz. Resmi belgeler, GCH kullanıcı kılavuzunun bölüm 7 bölümünde bulunabilir.

‘LANGUAGE’ programının formatı ‘{-# LANGUAGE ExtensionOne, ExtensionTwo … #-}’ şeklindedir. Bu, “{-#” harfinin ardından “LANGUAGE”, ardından virgülle ayrılmış bir uzantı listesi ve son olarak “#-}” kapanışıdır. Bir dosyaya birden fazla ‘DİL’ programı yerleştirilebilir.

SıraNTipleri #

Aşağıdaki durumu hayal edin:

foo :: Show a => (a -> String) -> String -> Int -> IO ()
foo show' string int = do
   putStrLn (show' string)
   putStrLn (show' int)

Burada, bir değeri String’e çeviren bir fonksiyona geçmek, o fonksiyonu hem string parametresine hem de int parametresine uygulamak ve ikisini de yazdırmak istiyoruz. Bana göre, bunun başarısız olması için hiçbir neden yok! Geçirdiğimiz parametrelerin her iki türü üzerinde de çalışan bir fonksiyonumuz var.

Ne yazık ki, bu kontrol yazmayacak! GHC, işlev gövdesindeki ilk oluşumuna dayalı olarak “a” türünü çıkarır. Yani, vurduğumuz anda:

putStrLn (show' string)

GHC, “dizi” bir “Dize” olduğundan, “göster” :: Dize -> Dize" sonucunu çıkarır. İnt’yi ‘göstermeye’ çalışırken patlamaya devam edecek.

‘RankNTypes’, bunun yerine, ‘show’’ türünü karşılayan tüm işlevleri nicelendirerek, tür imzasını aşağıdaki gibi yazmanıza olanak tanır:

foo :: (forall a. Show a => (a -> String)) -> String -> Int -> IO ()

Bu, 2. derece polimorfizmdir: ‘show’’ fonksiyonunun, fonksiyonumuzdaki tüm ‘a’lar için çalışması gerektiğini ve önceki uygulamanın şimdi çalıştığını iddia ediyoruz.

‘RankNTypes’ uzantısı, tür imzalarında ‘forall …’ bloklarının rastgele iç içe yerleştirilmesine izin verir. Başka bir deyişle, rank N polimorfizmine izin verir.

Aşırı Yüklenmiş Dizeler #

Normalde, Haskell’deki dize değişmezleri bir tür “Dize"ye sahiptir (bu, “[Char]” için bir tür diğer adıdır). Bu, daha küçük eğitim programları için bir sorun olmasa da, gerçek dünya uygulamaları genellikle “Metin” veya “ByteString” gibi daha verimli depolama gerektirir.

‘OverloadedStrings’ sadece hazır bilgi türünü şu şekilde değiştirir:

"test" :: Data.String.IsString a => a

Böyle bir tür bekleyen işlevlere doğrudan iletilmesine izin vermek. Birçok kitaplık, bu arabirimi, Data.Text ve Data.ByteString dahil olmak üzere, “[Char]“a göre belirli zaman ve alan avantajları sağlayan dize benzeri türleri için uygular.

Ayrıca, SQL sorgularının normal bir dize gibi çift tırnak içinde yazılmasına izin veren, ancak kötü şöhretli bir SQL kaynağı olan uygunsuz birleştirmeye karşı koruma sağlayan Postgresql-simple kitaplığındakiler gibi ‘OverloadedStrings’in bazı benzersiz kullanımları da vardır. enjeksiyon saldırıları

“IsString” sınıfının bir örneğini oluşturmak için “fromString” işlevini uygulamanız gerekir. Örnek†:

data Foo = A | B | Other String deriving Show

instance IsString Foo where
  fromString "A" = A
  fromString "B" = B
  fromString xs  = Other xs

tests :: [ Foo ]
tests = [ "A", "B", "Testing" ]

† Bu örnek Lyndon Maydwell’in izniyle (GitHub’da sordina) burada bulundu.

İkili Değişmezler #

Standart Haskell, tamsayı değişmezlerini ondalık (herhangi bir önek olmadan), onaltılık (“0x” veya “0X"den önce gelir) ve sekizlik (“0o” veya “0O"dan önce gelir) olarak yazmanıza olanak tanır. “BinaryLiterals” uzantısı, ikili seçeneği ekler (“0b” veya “0B"den önce gelir).

0b1111 == 15     -- evaluates to: True

Varoluşsal Niceleme #

Bu, varoluşsal olarak nicelenen türlere veya başka bir deyişle, yalnızca çalışma zamanında† somutlaştırılan tür değişkenlerine sahip türlere izin veren bir tür sistem uzantısıdır.

Varoluşsal türün bir değeri, OO dillerindeki soyut temel sınıf başvurusuna benzer: içerme içindeki tam türü bilmiyorsunuz, ancak türlerin _sınıfını sınırlayabilirsiniz.

data S = forall a. Show a => S a

veya eşdeğer olarak, GADT sözdizimi ile:

{-# LANGUAGE GADTs #-}
data S where
   S :: Show a => a -> S

Varoluşsal tipler, neredeyse heterojen kaplar gibi şeylere kapı açar: Yukarıda belirtildiği gibi, aslında bir ‘S’ değerinde çeşitli tipler olabilir, ancak hepsi ‘show’n olabilir, dolayısıyla siz de yapabilirsiniz.

instance Show S where
    show (S a) = show a   -- we rely on (Show a) from the above

Şimdi bu tür nesnelerin bir koleksiyonunu oluşturabiliriz:

ss = [S 5, S "test", S 3.0]

Bu da polimorfik davranışı kullanmamıza izin verir:

mapM_ print ss

Varoluşçular çok güçlü olabilir, ancak Haskell’de aslında çok sık gerekli olmadıklarına dikkat edin. Yukarıdaki örnekte, ‘Show’ örneğiyle yapabileceğiniz tek şey değerleri göstermek (hah!), yani bir dize temsili oluşturmaktır. Bu nedenle, “S” türünün tamamı, tam olarak, onu gösterirken aldığınız dize kadar bilgi içerir. Bu nedenle, özellikle Haskell tembel olduğundan ve bu nedenle dize ilk başta yalnızca değerlendirilmemiş bir thunk olacağından, bu dizeyi hemen saklamak genellikle daha iyidir.

Öte yandan, varoluşsal bazı benzersiz sorunlara neden olur. Örneğin, tür bilgisinin varoluşsal bir durumda “gizli” olma şekli. Bir ‘S’ değeri üzerinde kalıp-eşleştirirseniz, kapsam içinde kapsanan türe sahip olursunuz (daha doğrusu, onun ‘Göster’ örneği), ancak bu bilgi kapsamından asla kaçamaz, bu nedenle biraz “gizli bir toplum” haline gelir. ”: derleyici, türü zaten dışarıdan bilinen değerler dışında anything‘in kapsamdan çıkmasına izin vermez. Bu, [‘a0’ türü ‘()’ ile eşleştirilemedi ‘a0’ dokunulmazdır’ gibi garip hatalara yol açabilir](http://stackoverflow.com/questions/28582210/type-inference-with-gadts- a0-dokunulmazdır).

†Bunu, genellikle derleme zamanında çözülen (tam tip silmeye izin veren) sıradan parametrik polimorfizm ile karşılaştırın.

Varoluşsal türler, Sıra-N türlerinden farklıdır - bu uzantılar, kabaca konuşursak, birbirine ikilidir: gerçekten varoluşsal bir türün değerlerini kullanmak için, örnekteki “göster” gibi (muhtemelen kısıtlı) bir polimorfik işleve ihtiyacınız vardır. Bir polimorfik fonksiyon evrensel olarak nicelendirilir, yani belirli bir sınıftaki herhangi bir tür için işe yarar, buna karşın varoluşsal niceleme, önceden bilinmeyen belirli bir tür için çalıştığı anlamına gelir. Polimorfik bir fonksiyonunuz varsa, bu yeterlidir, ancak argümanlar gibi polimorfik fonksiyonları iletmek için {-# LANGUAGE Rank2Types #-} gerekir:

genShowSs :: (∀ x . Show x => x -> String) -> [S] -> [String]
genShowSs f = map (\(S a) -> f a)

LambdaKılıf #

“\arg -> case arg of” yerine “\case” yazmanıza izin veren sözdizimsel bir uzantı.

Aşağıdaki işlev tanımını göz önünde bulundurun:

dayOfTheWeek :: Int -> String
dayOfTheWeek 0 = "Sunday"
dayOfTheWeek 1 = "Monday"
dayOfTheWeek 2 = "Tuesday"
dayOfTheWeek 3 = "Wednesday"
dayOfTheWeek 4 = "Thursday"
dayOfTheWeek 5 = "Friday"
dayOfTheWeek 6 = "Saturday"

İşlev adını tekrarlamaktan kaçınmak istiyorsanız, şöyle bir şey yazabilirsiniz:

dayOfTheWeek :: Int -> String
dayOfTheWeek i = case i of
    0 -> "Sunday"
    1 -> "Monday"
    2 -> "Tuesday"
    3 -> "Wednesday"
    4 -> "Thursday"
    5 -> "Friday"
    6 -> "Saturday"

LambdaCase uzantısını kullanarak, argümanı adlandırmak zorunda kalmadan bunu bir işlev ifadesi olarak yazabilirsiniz:

{-# LANGUAGE LambdaCase #-}

dayOfTheWeek :: Int -> String
dayOfTheWeek = \case
    0 -> "Sunday"
    1 -> "Monday"
    2 -> "Tuesday"
    3 -> "Wednesday"
    4 -> "Thursday"
    5 -> "Friday"
    6 -> "Saturday"

İşlevsel Bağımlılıklar #

a, b, c ve x bağımsız değişkenlerine sahip çok parametreli bir tür sınıfınız varsa, bu uzantı, x türünün a, b ve c’den benzersiz olarak tanımlanabileceğini ifade etmenizi sağlar:

class SomeClass a b c x | a b c -> x where ...

Böyle bir sınıfın bir örneğini bildirirken, işlevsel bağımlılığın geçerli olduğundan emin olmak için diğer tüm örneklere karşı kontrol edilecektir, yani aynı “a b c"ye sahip ancak farklı “x"e sahip başka bir örnek mevcut değildir.

Virgülle ayrılmış bir listede birden çok bağımlılık belirtebilirsiniz:

class OtherClass a b c d | a b -> c d, a d -> b where ...

Örneğin MTL’de şunları görebiliriz:

class MonadReader r m| m -> r where ...
instance MonadReader r ((->) r) where ...

Şimdi, eğer MonadReader a ((->) Foo) => a türünde bir değeriniz varsa, derleyici, ikinci argüman tamamen birinciyi belirlediğinden ve türü buna göre basitleştireceğinden, a ~ Foo sonucunu çıkarabilir. .

“SomeClass” sınıfı, “x” ile sonuçlanan “a b c” argümanlarının bir fonksiyonu olarak düşünülebilir. Bu tür sınıflar, tip sisteminde hesaplamalar yapmak için kullanılabilir.

Esnek Örnekler #

Normal örnekler şunları gerektirir:

All instance types must be of the form (T a1 ... an)
where a1 ... an are *distinct type variables*,
and each type variable appears at most once in the instance head.

Bu, örneğin, ‘[a]’ için bir örnek oluşturabildiğiniz halde, özellikle ‘[Int]’ için bir örnek oluşturamayacağınız anlamına gelir.; “FlexibleInstances” şunları rahatlatır:

class C a where

-- works out of the box
instance C [a] where

-- requires FlexibleInstances
instance C [Int] where

GADT’ler #

Geleneksel cebirsel veri türleri, tür değişkenlerinde parametriktir. Örneğin, şöyle bir ADT tanımlarsak

data Expr a = IntLit Int 
            | BoolLit Bool 
            | If (Expr Bool) (Expr a) (Expr a)

bunun statik olarak iyi yazılmamış koşulları ekarte edeceği umuduyla, IntLit :: Int -> Expr a türü evrensel olarak nicelleştirildiğinden bu beklendiği gibi davranmaz: “a"nın herhangi bir seçimi için, “İfade a” türünde bir değer üretir. Özellikle, ‘a ~ Bool’ için, ‘If (IntLit 1) e1 e2’ gibi bir şey oluşturmamıza izin veren ‘IntLit :: Int -> Expr Bool’a sahibiz ve bu, ‘If’ yapıcısının türüydü. dışlamaya çalışıyor.

Genelleştirilmiş Cebirsel Veri Türleri, yalnızca parametrik olmamaları için elde edilen bir veri oluşturucu türünü kontrol etmemizi sağlar. ‘Expr’ türümüzü GADT olarak şu şekilde yeniden yazabiliriz:

data Expr a where
  IntLit :: Int -> Expr Int
  BoolLit :: Bool -> Expr Bool
  If :: Expr Bool -> Expr a -> Expr a -> Expr a

Burada, IntLit yapıcısının tipi Int -> Expr Int şeklindedir ve bu nedenle IntLit 1 :: Expr Bool tip kontrolü yapmayacaktır.

Bir GADT değerindeki kalıp eşleştirme, döndürülen terim türünün iyileştirilmesine neden olur. Örneğin, Expr a için şu şekilde bir değerlendirici yazmak mümkündür:

crazyEval :: Expr a -> a
crazyEval (IntLit x) = 
   -- Here we can use `(+)` because x :: Int
   x + 1 
crazyEval (BoolLit b) = 
   -- Here we can use `not` because b :: Bool
   not b
crazyEval (If b thn els) = 
  -- Because b :: Expr Bool, we can use `crazyEval b :: Bool`.
  -- Also, because thn :: Expr a and els :: Expr a, we can pass either to 
  -- the recursive call to `crazyEval` and get an a back
  crazyEval $ if crazyEval b then thn else els

Yukarıdaki tanımlarda “(+)” kullanabileceğimizi unutmayın, çünkü ör. ‘IntLit x’ kalıp eşleşir, ayrıca ‘a ~ Int’ olduğunu da öğreniriz (ve aynı şekilde ‘a ~ Bool’ olduğunda ’not’ ve ‘if_then_else_’ için).

TupleBölümleri #

Tuple yapıcısının (bir operatör olan) bir bölüm şeklinde uygulanmasına izin veren sözdizimsel bir uzantı:

(a,b) == (,) a b

-- With TupleSections
(a,b) == (,) a b == (a,) b == (,b) a

N-tuple’lar #

Ayrıca, ikiden büyük arity’ye sahip demetler için de çalışır.

(,2,) 1 3 == (1,2,3)

Haritalama #

Bu, bölümlerin kullanıldığı diğer yerlerde yararlı olabilir:

map (,"tag") [1,2,3] == [(1,"tag"), (2, "tag"), (3, "tag")]

Bu uzantı olmadan yukarıdaki örnek şöyle görünür:

map (\a -> (a, "tag")) [1,2,3]

Aşırı Yüklenmiş Listeler #

GHC 7.8‘de eklendi.

[OverloadedStrings](https://www.wikiod.com/tr/haskell/ortak-ghc-dil-uzantlar#Aşırı Yüklenmiş Dizeler) benzeri OverloadedLists, liste değişmezlerinin şu şekilde şekerinden arındırılmasına izin verir:

[]          -- fromListN 0 []
[x]         -- fromListN 1 (x : [])
[x .. ]     -- fromList (enumFrom x)

Bu, “Set”, “Vektör” ve “Harita” gibi türlerle uğraşırken kullanışlıdır.

['0' .. '9']             :: Set Char
[1 .. 10]                :: Vector Int
[("default",0), (k1,v1)] :: Map String Int
['a' .. 'z']             :: Text

“GHC.Exts” içindeki IsList sınıfı bununla kullanılmak üzere tasarlanmıştır eklenti.

‘IsList’, bir tür işlevle, ‘Öğe’ ve ‘fromList :: [Item l] -> l’, ’toList :: l -> [Item l]’ ve ‘fromListN :: Int - olmak üzere üç işlevle donatılmıştır. > [Öğe l] -> l` burada “fromListN” isteğe bağlıdır. Tipik uygulamalar şunlardır:

instance IsList [a] where
  type Item [a] = a
  fromList = id
  toList   = id

instance (Ord a) => IsList (Set a) where
  type Item (Set a) = a
  fromList = Set.fromList
  toList   = Set.toList

OverloadedLists – GHC‘den alınan örnekler.

MultiParamTypeSınıflar #

Birden çok tür parametresine sahip tür sınıflarına izin veren çok yaygın bir uzantıdır. MPTC’yi türler arası bir ilişki olarak düşünebilirsiniz.

{-# LANGUAGE MultiParamTypeClasses #-}

class Convertable a b where
    convert :: a -> b

instance Convertable Int Float where
    convert i = fromIntegral i

Parametrelerin sırası önemlidir.

MPTC’ler bazen tip aileleri ile değiştirilebilir.

UnicodeSözdizimi #

Belirli yerleşik operatörler ve adlar yerine Unicode karakterleri kullanmanıza izin veren bir uzantı.

ASCII tek kod Kullanım(lar)
:: ∷ tipi var
-> → fonksiyon türleri, lambdalar, ‘case’ dalları, vs.
=> ⇒ sınıf kısıtlamaları
‘forall’ ∀ açık polimorfizm
<- ← do gösterimi
* ★ türlerin türü (veya türü) (örneğin, Int :: ★)
>- ⤚ Oklar için [proc notasyonu]
-< ⤙ Oklar için [proc notasyonu]
>>- ⤜ Oklar için [proc notasyonu]
-<< ⤛ Oklar için [proc notasyonu]

Örneğin:

runST :: (forall s. ST s a) -> a

olacaktı

runST ∷ (∀ s. ST s a) → a

* ve ★ örneğinin biraz farklı olduğuna dikkat edin: * rezerve edilmediğinden, ★ da çarpma için * veya (*) adlı başka bir işlevle aynı şekilde çalışır. , ve tersi. Örneğin:

ghci> 2 ★ 3
6
ghci> let (*) = (+) in 2 ★ 3
5
ghci> let (★) = (-) in 2 * 3
-1

ModelEş anlamlıları #

Desen eşanlamlıları, işlevlerin ifadelerin soyutlamaları olmasına benzer kalıpların soyutlamalarıdır.

Bu örnek için, Data.Sequence arayüzüne bakalım ve nasıl olduğunu görelim. desen eşanlamlıları ile geliştirilebilir. Seq türü, dahili olarak karmaşık bir temsili çeşitli işlemler için iyi asimptotik karmaşıklık elde etmek için, özellikle de hem O(1) (un)consing hem de (un)snocing.

Ancak bu temsil hantaldır ve bazı değişmezleri Haskell’in tip sisteminde ifade edilemez. Bu nedenle, “Seq” türü, kullanıcılara, aralarında değişmeyen koruma sağlayan erişimci ve yapıcı işlevlerin yanı sıra soyut bir tür olarak sunulur:

empty :: Seq a

(<|) :: a -> Seq a -> Seq a
data ViewL a = EmptyL | a :< (Seq a)
viewl :: Seq a -> ViewL a

(|>) :: Seq a -> a -> Seq a 
data ViewR a = EmptyR | (Seq a) :> a 
viewr :: Seq a -> ViewR a

Ancak bu arayüzü kullanmak biraz zahmetli olabilir:

uncons :: Seq a -> Maybe (a, Seq a)
uncons xs = case viewl xs of
    x :< xs' -> Just (x, xs')
    EmptyL -> Nothing

Bir şekilde temizlemek için modelleri görüntüle kullanabiliriz:

{-# LANGUAGE ViewPatterns #-}

uncons :: Seq a -> Maybe (a, Seq a)
uncons (viewl -> x :< xs) = Just (x, xs)
uncons _ = Nothing

“PatternSynonyms” dil uzantısını kullanarak, bir eksi veya snoc listemiz varmış gibi yapmak için kalıp eşleştirmeye izin vererek daha da güzel bir arayüz verebiliriz:

{-# LANGUAGE PatternSynonyms #-}
import Data.Sequence (Seq)
import qualified Data.Sequence as Seq

pattern Empty :: Seq a
pattern Empty <- (Seq.viewl -> Seq.EmptyL)

pattern (:<) :: a -> Seq a -> Seq a
pattern x :< xs <- (Seq.viewl -> x Seq.:< xs)

pattern (:>) :: Seq a -> a -> Seq a
pattern xs :> x <- (Seq.viewr -> xs Seq.:> x)

Bu, çok doğal bir tarzda “uncons” yazmamıza izin verir:

uncons :: Seq a -> Maybe (a, Seq a)
uncons (x :< xs) = Just (x, xs)
uncons _ = Nothing

ScopedTypeDeğişkenler #

‘ScopedTypeVariables’, bir bildirimin içindeki evrensel olarak nicelenmiş türlere başvurmanıza izin verir. Daha açık olmak gerekirse:

import Data.Monoid

foo :: forall a b c. (Monoid b, Monoid c) => (a, b, c) -> (b, c) -> (a, b, c)
foo (a, b, c) (b', c') = (a :: a, b'', c'')
    where (b'', c'') = (b <> b', c <> c') :: (b, c)

Önemli olan, derleyiciye bildirimin alt ifadelerinde talimat vermek için ‘a’, ‘b’ ve ‘c’ kullanabilmemizdir (‘where’ yan tümcesindeki tuple ve nihai sonuçtaki ilk ‘a’). Uygulamada, ‘ScopedTypeVariables’ karmaşık işlevleri parçaların toplamı olarak yazmaya yardımcı olarak programcının somut türleri olmayan ara değerlere tür imzaları eklemesine olanak tanır.
