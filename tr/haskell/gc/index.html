‘IO a’dan ‘a’ “çıkarma” #

Yaygın bir soru şudur: “‘IO a’ değerine sahibim, ancak bu ‘a’ değerine bir şeyler yapmak istiyorum: buna nasıl erişebilirim?” Dış dünyadan gelen veriler üzerinde nasıl işlem yapılabilir (örneğin, kullanıcı tarafından yazılan bir sayıyı artırmak)?

Mesele şu ki, saf olmayan bir şekilde elde edilen veriler üzerinde saf bir işlev kullanırsanız, sonuç hala saf değildir. Kullanıcının ne yaptığına bağlı! “IO a” türünde bir değer, (a) onu “ana” olarak birleştirerek ve (b) derleyerek ve yürüterek yalnızca çalıştırılabilen “a” türünde bir değerle sonuçlanan “yan etkili hesaplama” anlamına gelir. senin programın. Bu nedenle, saf Haskell dünyasında “a’yı çıkarmanın” bir yolu yoktur.

Bunun yerine, çalışma zamanında* “a” değerini kullanan yeni bir “IO” değeri olan yeni bir hesaplama oluşturmak istiyoruz. Bu, “IO” değerlerini oluşturmanın başka bir yoludur ve böylece yine “do” gösterimini kullanabiliriz:

-- assuming
myComputation :: IO Int

getMessage :: Int -> String
getMessage int = "My computation resulted in: " ++ show int
 
newComputation :: IO ()
newComputation = do
  int <- myComputation       -- we "bind" the result of myComputation to a name, 'int'
  putStrLn $ getMessage int   -- 'int' holds a value of type Int

Burada, bir “Int"i “Dize"ye dönüştürmek için saf bir işlev (“getMessage”) kullanıyoruz, ancak bir “IO” hesaplamasının “myComputation” sonucuna uygulanmasını sağlamak için “do” notasyonunu kullanıyoruz. ` * ne zaman* (sonra) bu hesaplama çalışır. Sonuç, daha büyük bir “IO” hesaplamasıdır, “newComputation”. Saf işlevleri saf olmayan bir bağlamda kullanma tekniğine kaldırma denir.

IO, programınızın “ana” eylemini tanımlar #

Bir Haskell programını çalıştırılabilir yapmak için, ‘IO ()’ türünde bir ‘ana’ işlevi olan bir dosya sağlamalısınız.

main :: IO ()
main = putStrLn "Hello world!"

Haskell derlendiğinde buradaki ‘IO’ verilerini inceler ve çalıştırılabilir hale getirir. Bu programı çalıştırdığımızda Merhaba dünya! yazacaktır.

“Ana” dışında “IO a” türünde değerleriniz varsa, hiçbir şey yapmazlar.

other :: IO ()
other = putStrLn "I won't get printed"

main :: IO ()
main = putStrLn "Hello world!"

Bu programı derlemek ve çalıştırmak son örnektekiyle aynı etkiye sahip olacaktır. “Diğer"deki kod yok sayılır.

‘other’ içindeki kodun çalışma zamanı etkilerine sahip olması için onu ‘main’ olarak oluşturmanız gerekir. Sonunda “ana” olarak oluşturulmayan hiçbir “IO” değeri, herhangi bir çalışma zamanı etkisine sahip olmayacaktır. İki “IO” değerini sırayla oluşturmak için “do” gösterimini kullanabilirsiniz:

other :: IO ()
other = putStrLn "I will get printed... but only at the point where I'm composed into main"

main :: IO ()
main = do 
  putStrLn "Hello world!"
  other

bu programı derleyip çalıştırdığınızda, çıktıları

Hello world!
I will get printed... but only at the point where I'm composed into main

İşlem sırasının, tanım sırasına göre değil, “diğer"in “ana” olarak nasıl oluşturulduğuna göre açıklandığına dikkat edin.

Dosya sonu koşullarını kontrol etme #

Diğer birçok dilin standart I/O kitaplıklarının bunu yapma biçimine biraz ters düşen Haskell’in ‘isEOF’u, bir EOF koşulunu kontrol etmeden önce bir okuma işlemi gerçekleştirmenizi gerektirmez; çalışma zamanı sizin için yapacak.

import System.IO( isEOF )


eofTest :: Int -> IO ()
eofTest line = do
    end <- isEOF
    if end then
        putStrLn $ "End-of-file reached at line " ++ show line ++ "."
    else do
        getLine
        eofTest $ line + 1


main :: IO ()
main =
    eofTest 1

Giriş:

Line #1.
Line #2.
Line #3.

Çıktı:

End-of-file reached at line 4.

Standart girdinin tüm içeriğini bir dizgeye okuma #

main = do
    input <- getContents
    putStr input

Giriş:

This is an example sentence.
And this one is, too!

Çıktı:

This is an example sentence.
And this one is, too!

Not: Bu program aslında çıktının bazı kısımlarını tüm girdiler tamamen okunmadan önce yazdıracaktır. Bu, örneğin, bir 50MiB dosyası üzerinde “getContents” kullanırsanız, Haskell’in tembel değerlendirmesi ve çöp toplayıcısının bunu sağlayacağı anlamına gelir. dosyanın yalnızca o anda ihtiyaç duyulan kısımları (okuma: daha fazla yürütme için vazgeçilmez) belleğe yüklenecektir. Böylece 50MiB dosyası bir kerede belleğe yüklenmeyecektir.

IO’nun Rolü ve Amacı #

Haskell saf bir dildir, yani ifadelerin yan etkileri olamaz. Yan etki, ifadenin veya işlevin bir değer üretmek dışında yaptığı herhangi bir şeydir, örneğin genel bir sayacı değiştirmek veya standart çıktıya yazdırmak.

Haskell’de yan etkili hesaplamalar (özellikle gerçek dünya üzerinde etkisi olabilecek olanlar) ‘IO’ kullanılarak modellenir. Kesin olarak konuşursak, ‘IO’ bir tür oluşturucudur, bir tür alır ve bir tür üretir. Örneğin, “IO Int”, “Int” değeri üreten bir G/Ç hesaplamasının türüdür. ‘IO’ türü soyuttur ve ‘IO’ için sağlanan arabirim, IO gerçekleştiren tüm yerleşik işlevlerin bir ‘IO’ içine alınmış dönüş türü.

Bir Haskell programı çalıştırıldığında, türü herhangi bir “x” türü için “IO x” olabilen “main” adlı Haskell değeriyle temsil edilen hesaplama yürütülür.

IO değerlerini manipüle etme #

Standart kitaplıkta, dosya tanıtıcılarına okuma ve yazma gibi genel amaçlı bir programlama dilinin gerçekleştirmesi gereken tipik “IO” eylemlerini sağlayan birçok işlev vardır. Genel “IO” eylemleri oluşturulur ve öncelikle iki işlevle birleştirilir:

 (>>=) :: IO a -> (a -> IO b) -> IO b

Bu işlev (tipik olarak bağlama olarak adlandırılır) bir “IO” eylemi ve bir “IO” eylemi döndüren bir işlev alır ve işlevin ilk “IO” tarafından üretilen değere uygulanmasının sonucu olan “IO” eylemini üretir. `eylem.

 return :: a -> IO a

Bu işlev herhangi bir değeri (yani saf bir değer) alır ve GÇ yapmayan ve verilen değeri üreten GÇ hesaplamasını döndürür. Başka bir deyişle, işlem gerektirmeyen bir G/Ç eylemidir.

Sıklıkla kullanılan ek genel işlevler vardır, ancak tümü yukarıdaki iki terimle yazılabilir. Örneğin, (>>) :: IO a -> IO b -> IO b, (>>=) ile benzerdir ancak ilk işlemin sonucu yok sayılır.

Bu işlevleri kullanarak kullanıcıyı karşılayan basit bir program:

 main :: IO ()
 main =
   putStrLn "What is your name?" >>
   getLine >>= \name ->
   putStrLn ("Hello " ++ name ++ "!")

Bu program ayrıca putStrLn :: String -> IO () ve getLine :: IO String kullanır.

Not: Yukarıdaki belirli işlevlerin türleri aslında verilen türlerden daha geneldir (yani >>=, >> ve return).

IO semantiği #

Haskell’deki ‘IO’ türü, zorunlu programlama dillerininkine çok benzer anlamlara sahiptir. Örneğin, `s1 yazıldığında; s2’, ‘s1’ ifadesinin, ardından ‘s2’ ifadesinin yürütüldüğünü belirtmek için zorunlu bir dilde, aynı şeyi Haskell’de modellemek için ‘s1 » s2’ yazılabilir.

Bununla birlikte, “IO"nun semantiği, zorunlu bir arka plandan gelmesi beklenenden biraz farklıdır. ‘dönüş’ işlevi kontrol akışını kesmez - sırayla başka bir ‘IO’ eylemi çalıştırılırsa program üzerinde hiçbir etkisi yoktur. Örneğin, return () >> putStrLn "boom" standart çıktıya “boom"u doğru şekilde yazdırır.

‘IO’nun biçimsel semantiği, önceki bölümdeki işlevleri içeren basit eşitlikler cinsinden verilebilir:

 return x >>= f ≡ f x, ∀ f x
 y >>= return ≡ return y, ∀ y
 (m >>= f) >>= g ≡ m >>= (\x -> (f x >>= g)), ∀ m f g

Bu yasalar tipik olarak sırasıyla sol özdeşlik, sağ özdeşlik ve bileşim olarak adlandırılır. İşlev açısından daha doğal olarak ifade edilebilirler.

 (>=>) :: (a -> IO b) -> (b -> IO c) -> a -> IO c
 (f >=> g) x = (f x) >>= g

aşağıdaki gibi:

 return >=> f ≡ f, ∀ f
 f >=> return ≡ f, ∀ f
 (f >=> g) >=> h ≡ f >=> (g >=> h), ∀ f g h

Tembel GÇ #

G/Ç hesaplamalarını gerçekleştiren işlevler tipik olarak katıdır, yani bir dizi eylemdeki önceki tüm eylemlerin bir sonraki eylem başlamadan önce tamamlanması gerekir. Tipik olarak bu yararlı ve beklenen davranıştır - putStrLn "X" >> putStrLn "Y" “XY” yazmalıdır. Ancak, belirli kitaplık işlevleri G/Ç’yi tembelce gerçekleştirir, yani değeri üretmek için gereken G/Ç eylemleri yalnızca değer gerçekten tüketildiğinde gerçekleştirilir. Bu tür işlevlere örnek olarak “getContents” ve “readFile” verilebilir. Tembel G/Ç, bir Haskell programının performansını büyük ölçüde azaltabilir, bu nedenle kitaplık işlevlerini kullanırken hangi işlevlerin tembel olduğuna dikkat edilmelidir.

IO ve ‘do’ notasyonu #

Haskell, farklı IO değerlerini daha büyük IO değerleriyle birleştirmek için daha basit bir yöntem sağlar. Bu özel sözdizimi ‘do’ notasyonu* olarak bilinir ve ‘»=’, ‘»’ ve ‘dönüş’ işlevlerinin kullanımları için basitçe sözdizimsel şekerdir.

Bir önceki bölümdeki program, do notasyonu kullanılarak iki farklı şekilde yazılabilir; birincisi yerleşime duyarlı ve ikincisi yerleşime duyarsızdır:

 main = do
   putStrLn "What is your name?"
   name <- getLine
   putStrLn ("Hello " ++ name ++ "!")


 main = do {
   putStrLn "What is your name?" ;
   name <- getLine ;
   putStrLn ("Hello " ++ name ++ "!")
   }

Her üç program da tam olarak eşdeğerdir.

*‘do’ notasyonunun monads adı verilen daha geniş bir tür oluşturucu sınıfı için de geçerli olduğunu unutmayın.

Standart girdiden satır okuma #

Tüm dosyadan kelimeleri okuma #

Haskell’de, genellikle dosya tutamaçlarıyla hiç uğraşmamak mantıklıdır, ancak tüm bir dosyayı doğrudan diskten belleğe okuma veya yazma† ve metnin tüm bölümleme/işleme işlemlerini saf dize veri yapısı. Bu, hataların önlenmesine büyük ölçüde yardımcı olabilecek IO ve program mantığının karıştırılmasını önler.

-- | The interesting part of the program, which actually processes data
--   but doesn't do any IO!
reverseWords :: String -> [String]
reverseWords = reverse . words

-- | A simple wrapper that only fetches the data from disk, lets
--   'reverseWords' do its job, and puts the result to stdout.
main :: IO ()
main = do
   content <- readFile "loremipsum.txt"
   mapM_ putStrLn $ reverseWords content

“loremipsum.txt” içeriyorsa

Lorem ipsum dolor sit amet,
consectetur adipiscing elit

sonra program çıkacak

elit
adipiscing
consectetur
amet,
sit
dolor
ipsum
Lorem

Burada, mapM_ dosyadaki tüm kelimelerin listesini gözden geçirdi ve her birini putStrLn ile ayrı bir satıra yazdırdı.

†Bunun hafızayı boşa harcadığını düşünüyorsanız, haklısınız. Aslında, Haskell’in tembelliği, tüm dosyanın aynı anda bellekte kalması gerektiğinden genellikle kaçınabilir… ancak dikkat edin, bu tür tembel IO kendi sorunlarına neden olur. Performans açısından kritik uygulamalar için, tüm dosyanın bir kerede kesinlikle okunmasını zorunlu kılmak genellikle mantıklıdır; bunu [readFile`````Data.Text sürümüyle] yapabilirsiniz.](http://hackage.haskell.org/package/text-1.2.2.1/docs/Data-Text-IO.html#v: readFile)

stdout’a yazma #

Haskell 2010 Dil Spesifikasyonu uyarınca, aşağıdakiler Prelude’da bulunan standart IO işlevleridir, dolayısıyla bunları kullanmak için içe aktarma gerekmez.

‘putChar :: Char -> IO ()’ - ‘stdout’a bir ‘char’ yazar #

Prelude> putChar 'a'
aPrelude>  -- Note, no new line

putStr :: String -> IO () - stdouta bir String yazar #

Prelude> putStr "This is a string!"
This is a string!Prelude>  -- Note, no new line

putStrLn :: String -> IO () - stdouta bir String yazar ve yeni bir satır ekler #

Prelude> putStrLn "Hi there, this is another String!"
Hi there, this is another String!

print :: Show a => a -> IO () - “a"yı “Show” örneğini “stdout"a yazar #

Prelude> print "hi"
"hi"
Prelude> print 1
1
Prelude> print 'a'
'a'
Prelude> print (Just 'a')  -- Maybe is an instance of the `Show` type class
Just 'a'
Prelude> print Nothing
Nothing

Türetmeyi kullanarak kendi türleriniz için Show’u başlatabileceğinizi hatırlayın:

-- In ex.hs
data Person = Person { name :: String } deriving Show
main = print (Person "Alex")  -- Person is an instance of `Show`, thanks to `deriving`

GHCi’de yükleme ve çalıştırma:

Prelude> :load ex.hs
[1 of 1] Compiling ex             ( ex.hs, interpreted )
Ok, modules loaded: ex.
*Main> main  -- from ex.hs
Person {name = "Alex"}
*Main>

readFloat :: IO Float

    fmap read getLine


main :: IO ()
main = do
    putStr "Type the first number: "
    first <- readFloat

    putStr "Type the second number: "

Dosya tanıtıcılarından okuma #

G/Ç kitaplığının diğer bazı bölümlerinde olduğu gibi, örtük olarak standart bir akış kullanan işlevlerin, aynı işi gerçekleştiren “System.IO"da bir karşılığı vardır, ancak solda “Handle” türünde fazladan bir parametre vardır. işlenen akışı temsil eder. Örneğin, “getLine :: IO Dizesi"nin bir karşılığı “hGetLine :: Handle -> IO String” vardır.

import System.IO( Handle, FilePath, IOMode( ReadMode ), 
                  openFile, hGetLine, hPutStr, hClose, hIsEOF, stderr )

import Control.Monad( when )


dumpFile :: Handle -> FilePath -> Integer -> IO ()

dumpFile handle filename lineNumber = do      -- show file contents line by line
    end <- hIsEOF handle
    when ( not end ) $ do
        line <- hGetLine handle
        putStrLn $ filename ++ ":" ++ show lineNumber ++ ": " ++ line
        dumpFile handle filename $ lineNumber + 1


main :: IO ()

main = do
    hPutStr stderr "Type a filename: "
    filename <- getLine
    handle <- openFile filename ReadMode
    dumpFile handle filename 1
    hClose handle

“example.txt” dosyasının içeriği:

This is an example.
Hello, world!
This is another example.

Giriş:

Type a filename: example.txt

Çıktı:

example.txt:1: This is an example.
example.txt:2: Hello, world!
example.txt:3: This is another example

stdinden okuma #

Haskell 2010 Dil Spesifikasyonu uyarınca, aşağıdakiler Prelude’da bulunan standart IO işlevleridir, dolayısıyla bunları kullanmak için içe aktarma gerekmez.

‘getChar :: IO Karakteri’ - ‘stdin’den bir ‘Karakter’ okuyun #

-- MyChar.hs
main = do
  myChar <- getChar
  print myChar

-- In your shell

runhaskell MyChar.hs
a -- you enter a and press enter
'a'  -- the program prints 'a'

‘getLine :: IO String’ - ‘stdin’den bir ‘Dize’ okuyun, yeni satır karakteri yok #

Prelude> getLine
Hello there!  -- user enters some text and presses enter
"Hello there!"

read :: Read a => String -> a - bir String’i bir değere dönüştürün #

Prelude> read "1" :: Int
1
Prelude> read "1" :: Float
1.0
Prelude> read "True" :: Bool
True

Diğer, daha az yaygın işlevler şunlardır:

  • getContents :: IO String - tüm kullanıcı girdilerini gerektiğinde tembelce okunan tek bir dize olarak döndürür
  • interact :: (String -> String) -> IO () - argümanı olarak String->String türünde bir fonksiyon alır. Standart giriş cihazından gelen tüm giriş, argümanı olarak bu fonksiyona iletilir.
