Yığınla Profil Oluşturma #

Bir proje için profil oluşturmayı “yığın” aracılığıyla yapılandırın. Önce projeyi --profile bayrağıyla oluşturun:

stack build --profile

Bunun çalışması için cabal dosyasında GHC bayrakları gerekli değildir (-prof gibi). “stack”, projedeki hem kitaplık hem de yürütülebilir dosyalar için profil oluşturmayı otomatik olarak açar. Projede bir daha yürütülebilir dosya çalıştırıldığında, olağan “+RTS” bayrakları kullanılabilir:

stack exec -- my-bin +RTS -p

Yapı #

Dosya yapısı #

Basit bir projede aşağıdaki dosyalar bulunur:

➜  helloworld ls 
LICENSE          Setup.hs         helloworld.cabal src              stack.yaml

‘src’ klasöründe ‘Main.hs’ adında bir dosya var. Bu, “helloworld” projesinin “başlangıç ​​noktası"dır. Varsayılan olarak Main.hs basit bir “Merhaba, Dünya!” programı.

Ana.hs

module Main where

main :: IO ()
main = do
  putStrLn "hello world"

Programın çalıştırılması #

‘helloworld’ dizininde olduğunuzdan emin olun ve şunu çalıştırın:

stack build # Compile the program
stack exec helloworld # Run the program
# prints "hello world"

Bir Yığın Projesi Oluşturun ve Çalıştırın #

Bu örnekte proje adımız “helloworld” olup, stack new helloworld simple ile oluşturulmuştur.

İlk önce projeyi `stack build’ ile inşa etmeliyiz ve sonra onu çalıştırabiliriz.

stack exec helloworld-exe

Bağımlılıkları görüntüleme #

Projenizin doğrudan hangi paketlere bağlı olduğunu öğrenmek için bu komutu kullanabilirsiniz:

stack list-dependencies

Bu şekilde, bağımlılıklarınızın hangi sürümünün gerçekte yığın tarafından aşağı çekildiğini öğrenebilirsiniz.

Haskell projeleri sıklıkla kendilerini birçok kütüphaneyi dolaylı olarak çekerken bulurlar ve bazen bu dış bağımlılıklar, izlemeniz gereken sorunlara neden olur. Kendinizi, tanımlamak istediğiniz sahte bir dış bağımlılıkla bulursanız, tüm bağımlılık grafiğini gözden geçirebilir ve bağımlılıklarınızdan hangisinin istenmeyen paketi nihayetinde çektiğini belirleyebilirsiniz:

stack dot --external | grep template-haskell

‘stack dot’, aranabilecek metin biçiminde bir bağımlılık grafiği yazdırır. Ayrıca görüntülenebilir:

stack dot --external | dot -Tpng -o my-project.png

İsterseniz bağımlılık grafiğinin derinliğini de ayarlayabilirsiniz:

stack dot --external --depth 3 | dot -Tpng -o my-project.png

Yığını Yükleme #

Mac OS X

Homebrew kullanarak:

brew install haskell-stack

Basit bir proje oluşturma #

helloworld adlı bir proje oluşturmak için şunu çalıştırın:

stack new helloworld simple

Bu, bir Stack projesi için gerekli dosyalarla “helloworld” adlı bir dizin oluşturacaktır.

Yığın yükleme #

Komutu çalıştırarak

stack install

Stack, yürütülebilir bir dosyayı klasöre kopyalayacak

/Users/<yourusername>/.local/bin/

Yığın Paketleri ve LTS (çözücü) sürümünün değiştirilmesi #

Stackage, Haskell paketleri için bir depodur. Bu paketleri bir yığın projesine ekleyebiliriz.

Bir projeye lens ekleme. #

Bir yığın projesinde stack.yaml adında bir dosya vardır. stack.yaml içinde şuna benzeyen bir segment var:

resolver: lts-6.8

Stackage, lts’nin her revizyonu için bir paket listesi tutar. Bizim durumumuzda lts-6.8 paketlerinin listesini istiyoruz Bu paketleri bulmak için şu adresi ziyaret edin:

https://www.stackage.org/lts-6.8 # if a different version is used, change 6.8 to the correct resolver number.

Paketlere bakıldığında bir Lens-4.13 var.

Şimdi helloworld.cabal bölümünü değiştirerek dil paketini ekleyebiliriz:

  build-depends: base >= 4.7 && < 5

ile:

  build-depends: base >= 4.7 && 5,
                 lens == 4.13

Açıkçası, daha yeni bir LTS’yi değiştirmek istiyorsak (yayınlandıktan sonra), yalnızca çözümleyici numarasını değiştiririz, örneğin:

resolver: lts-6.9

Bir sonraki “yığın oluşturma” ile Stack, LTS 6.9 sürümünü kullanacak ve bu nedenle bazı yeni bağımlılıklar indirecektir.
