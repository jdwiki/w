Parametreler #

Tür/Fonksiyon Ayrıntı
data Eval a Değerlendirme, paralel stratejileri tanımlamayı kolaylaştıran bir Monad’dır
type Strategy a = a -> Eval a paralel bir değerlendirme stratejisi içeren bir işlev. İşlev, argümanının (parçalarının) üzerinden geçerek, alt ifadeleri paralel veya sırayla değerlendirir
rpar :: Strateji a argümanını ateşler (paralel olarak değerlendirme için)
rseq :: Strateji a argümanını zayıf kafa normal formuna göre değerlendirir
force :: NFData a => a -> a argümanın kendisini döndürmeden önce argümanının tüm yapısını değerlendirerek normal forma indirger. Control.DeepSeq modülü tarafından sağlanır

[Simon Marlow’un kitabı1, Haskell’de Eşzamanlı ve Paralel Programlama, olağanüstüdür ve çok sayıda kavramı kapsar. Ayrıca en yeni Haskell programcısı için bile çok erişilebilir. Şiddetle tavsiye edilir ve PDF olarak veya çevrimiçi olarak ücretsiz olarak kullanılabilir.

Paralel ve Eşzamanlı

Simon Marlow en iyi şekilde ifade ediyor:

Paralel program, bir hesaplamayı daha hızlı gerçekleştirmek için çok sayıda hesaplama donanımı (örneğin birkaç işlemci çekirdeği) kullanan programdır. Amaç, hesaplamanın farklı bölümlerini aynı anda çalışan farklı işlemcilere devrederek cevaba daha erken ulaşmaktır.

Buna karşılık, eşzamanlılık, birden çok kontrol iş parçacığının olduğu bir program yapılandırma tekniğidir. Kavramsal olarak, kontrol dizileri “aynı anda” yürütülür; yani, kullanıcı etkilerini serpiştirilmiş olarak görür. Gerçekte aynı anda çalışıp çalışmadıkları bir uygulama detayıdır; eşzamanlı bir program, serpiştirilmiş yürütme yoluyla tek bir işlemcide veya birden çok fiziksel işlemcide yürütülebilir.

Zayıf Baş Normal Form

Tembel değerlendirmenin nasıl çalıştığının farkında olmak önemlidir. Bu bölümün ilk bölümü, WHNF’ye ve bunun paralel ve eşzamanlı programlama ile nasıl ilişkili olduğuna güçlü bir giriş yapacaktır.

Değerlendirme Monad #

Haskell’deki paralellik, ‘Control.Parallel.Strategies’‘den ‘Değerlendirme’ Monad’ı, ‘rpar’ ve ‘rseq’ işlevleri (diğerlerinin yanı sıra) kullanılarak ifade edilebilir.

f1 :: [Int]
f1 = [1..100000000]

f2 :: [Int]
f2 = [1..200000000]

main = runEval $ do
  a <- rpar (f1) -- this'll take a while...
  b <- rpar (f2) -- this'll take a while and then some...
  return (a,b)

Yukarıda “main” çalıştırıldığında, “a” ve “b” değerleri arka planda “rpar” aracılığıyla hesaplanırken, yürütülür ve hemen “döndürülür”.

Not: Paralel yürütmenin gerçekleşmesi için -threaded ile derlediğinizden emin olun.

rpar #

rpar :: Strateji a verilen stratejiyi paralel olarak yürütür (hatırlayın: type Strategy a = a -> Eval a) paralel olarak:

import Control.Concurrent
import Control.DeepSeq
import Control.Parallel.Strategies
import Data.List.Ordered

main = loop
  where 
    loop = do
      putStrLn "Enter a number"
      n <- getLine

      let lim = read n :: Int
          hf  = quot lim 2
          result = runEval $ do
            -- we split the computation in half, so we can concurrently calculate primes
            as <- rpar (force (primesBtwn 2 hf))
            bs <- rpar (force (primesBtwn (hf + 1) lim))
            return (as ++ bs)

      forkIO $ putStrLn ("\nPrimes are: " ++ (show result) ++ " for " ++ n ++ "\n")
      loop

-- Compute primes between two integers
-- Deliberately inefficient for demonstration purposes
primesBtwn n m = eratos [n..m]
  where
    eratos []     = []
    eratos (p:xs) = p : eratos (xs `minus` [p, p+p..])

sıra #

Bir argümanı Zayıf Kafa Normal Formuna zorlamak için rseq :: Strateji ayı kullanabiliriz:

f1 :: [Int]
f1 = [1..100000000]

f2 :: [Int]
f2 = [1..200000000]

main = runEval $ do
  a <- rpar (f1) -- this'll take a while...
  b <- rpar (f2) -- this'll take a while and then some...
  rseq a
  return (a,b)

Bu, ‘rpar’ örneğinin anlamını ustaca değiştirir; ikincisi, arka planda değerleri hesaplarken hemen dönerken, bu örnek ‘a’ WHNF olarak değerlendirilene kadar bekleyecektir.
