Temel Sözdizimi #

Kayıtlar, alanların adlandırılmasına izin veren toplam cebirsel “veri” türünün bir uzantısıdır:

data StandardType = StandardType String Int Bool --standard way to create a sum type

data RecordType = RecordType { -- the same sum type with record syntax
    aString :: String
  , aNumber :: Int
  , isTrue  :: Bool
  }

Alan adları daha sonra adlandırılmış alanı kayıttan çıkarmak için kullanılabilir.

> let r = RecordType {aString = "Foobar", aNumber= 42, isTrue = True}
> :t r
  r :: RecordType
> :t aString
  aString :: RecordType -> String
> aString r
  "Foobar"

Kayıtlar, desenle eşleştirilebilir

case r of
  RecordType{aNumber = x, aString=str} -> ... -- x = 42, str = "Foobar"

Tüm alanların adlandırılması gerekmediğine dikkat edin

Kayıtlar, alanları adlandırılarak oluşturulur, ancak normal toplam türleri olarak da oluşturulabilir (genellikle alan sayısı az olduğunda ve değişme olasılığı olmadığında yararlıdır)

r  = RecordType {aString = "Foobar", aNumber= 42, isTrue = True}
r' = RecordType  "Foobar" 42 True

Adlandırılmış alan olmadan bir kayıt oluşturulursa, derleyici bir uyarı verir ve sonuç değeri ’tanımsız’ olur.

> let r = RecordType {aString = "Foobar", aNumber= 42}
  <interactive>:1:9: Warning:
     Fields of RecordType not initialized: isTrue
> isTrue r
  Error 'undefined'

Bir kaydın alanı, değeri ayarlanarak güncellenebilir. Belirtilmeyen alanlar değişmez.

> let r = RecordType {aString = "Foobar", aNumber= 42, isTrue = True}
> let r' = r{aNumber=117}
    -- r'{aString = "Foobar", aNumber= 117, isTrue = True}

Karmaşık kayıt türleri için [lensler][1] oluşturmak genellikle yararlıdır.

[1]: https://www.wikiod.com/tr/haskell/lens#Kayıtlar için lensler

Alan etiketleriyle bir veri türü tanımlama #

Alan Değerlerini Değiştirirken Kayıtları Kopyalama #

Bu türe sahip olduğunuzu varsayalım:

data Person = Person { name :: String, age:: Int } deriving (Show, Eq)

ve iki değer:

alex = Person { name = "Alex", age = 21 }
jenny = Person { name = "Jenny", age = 36 }

“Kişi” türünde yeni bir değer, “alex"ten kopyalanarak ve hangi değerlerin değiştirileceğini belirterek oluşturulabilir:

anotherAlex = alex { age = 31 }

“alex” ve “anotherAlex” değerleri şimdi şöyle olacaktır:

Person {name = "Alex", age = 21}

Person {name = "Alex", age = 31}

newtype ile kayıtlar #

Kayıt sözdizimi, tam olarak bir alana sahip tam olarak bir kurucu olduğu kısıtlamasıyla ’newtype’ ile kullanılabilir. Buradaki fayda, yeni türün paketini açmak için bir işlevin otomatik olarak oluşturulmasıdır. Bu alanlar genellikle monadlar için “run”, monoidler için “get” ve diğer türler için “un” ile başlayarak adlandırılır.

newtype State s a = State { runState :: s -> (s, a) }

newtype Product a = Product { getProduct :: a }

newtype Fancy = Fancy { unfancy :: String } 
  -- a fancy string that wants to avoid concatenation with ordinary strings

Kayıt sözdiziminin genellikle değerleri oluşturmak için asla kullanılmadığını ve alan adının kesinlikle sarmayı açmak için kullanıldığını belirtmek önemlidir.

getProduct $ mconcat [Product 7, Product 9, Product 12]
-- > 756

RecordWildCard’lar #

{-# LANGUAGE RecordWildCards #-}

data Client = Client { firstName     :: String
                     , lastName      :: String
                     , clientID      :: String 
                     } deriving (Show)

printClientName :: Client -> IO ()
printClientName Client{..} = do
    putStrLn firstName
    putStrLn lastName
    putStrLn clientID

‘Client{..}’ kalıbı, ‘Client’ yapıcısının tüm alanlarını kapsam içine alır ve kalıba eşdeğerdir.

Client{ firstName = firstName, lastName = lastName, clientID = clientID }

Ayrıca aşağıdaki gibi diğer alan eşleştiricilerle birleştirilebilir:

Client { firstName = "Joe", .. }

Bu eşdeğerdir

Client{ firstName = "Joe", lastName = lastName, clientID = clientID }
