Uygulamalı Functor

  • Tanım
  • Alternatif tanım
  • Yaygın Uygulama örnekleri

Bu sayfada

  • Tanım
  • Alternatif tanım
  • Yaygın Uygulama örnekleri

‘Uygulamalı’, işlevin de bu yapıya gömülü olduğu bir yapı üzerinde kaldırılmış işlev uygulamasına izin veren ‘f :: * -> *’ türlerinin sınıfıdır.

Tanım #

class Functor f => Applicative f where
    pure  :: a -> f a
    (<*>) :: f (a -> b) -> f a -> f b

‘f’ üzerindeki ‘Functor’ kısıtlamasına dikkat edin. ‘pure’ işlevi, ‘Uygulamalı’ yapıya gömülü argümanını döndürür. “<*>” infix işlevi (“apply” olarak telaffuz edilir), “Uygulamalı” yapıya gömülü işlev dışında “fmap” işlevine çok benzer.

Doğru bir “Uygulamalı” örneği, derleyici tarafından uygulanmasa da uygulama yasalarını karşılamalıdır:

pure id <*> a = a                              -- identity
pure (.) <*> a <*> b <*> c = a <*> (b <*> c)   -- composition
pure f <*> pure a = pure (f a)                 -- homomorphism
a <*> pure b = pure ($ b) <*> a                -- interchange

Alternatif tanım #

Her Uygulamalı Functor bir Functor olduğundan, üzerinde “fmap” her zaman kullanılabilir; Bu nedenle, Applicative’in özü, taşınan içeriklerin eşleştirilmesi ve onu oluşturma yeteneğidir:

class Functor f => PairingFunctor f where
  funit :: f ()                  -- create a context, carrying nothing of import
  fpair :: (f a,f b) -> f (a,b)  -- collapse a pair of contexts into a pair-carrying context

Bu sınıf, “Uygulamalı” ile izomorfiktir.

pure a = const a <$> funit = a <$ funit  

fa <*> fb = (\(a,b) -> a b) <$> fpair (fa, fb) = uncurry ($) <$> fpair (fa, fb)

Tersine,

funit = pure ()

fpair (fa, fb) = (,) <$> fa <*> fb

Yaygın Uygulama örnekleri #

Belki #

“Belki”, muhtemelen mevcut olmayan bir değer içeren uygulamalı bir işlevdir.

instance Applicative Maybe where
    pure = Just
    
    Just f <*> Just x = Just $ f x
    _ <*> _ = Nothing

“saf”, verilen değeri “Sadece” uygulayarak “Belki” değerine yükseltir. “(<*>)” işlevi, “Belki” içine sarılmış bir işlevi “Belki” içindeki bir değere uygular. Hem işlev hem de değer mevcutsa (‘Just’ ile oluşturulmuş), işlev değere uygulanır ve sarılmış sonuç döndürülür. Herhangi biri eksikse, hesaplama devam edemez ve bunun yerine “Hiçbir şey” döndürülür.

Listeler #

Listelerin <*> :: [a -> b] -> [a] -> [b] tür imzasına uymasının bir yolu, ilk listenin her bir öğesini eşleştirerek iki listenin Kartezyen çarpımını almaktır. ikincisinin her bir elemanı ile:

fs <*> xs = [f x | f <- fs, x <- xs]
         -- = do { f <- fs; x <- xs; return (f x) }

pure x = [x]

Bu genellikle, olası değerleri bu listenin üzerinde olan, belirlenimsiz bir değeri temsil eden bir değerler listesiyle, belirlenimsizliğe öykünme olarak yorumlanır; bu nedenle, iki deterministik olmayan değerin bir kombinasyonu, iki listedeki değerlerin tüm olası kombinasyonları arasında değişir:

ghci> [(+1),(+2)] <*> [3,30,300]
[4,31,301,5,32,302]

Sonsuz akışlar ve posta listeleri #

İki girdisini birlikte “zipleyen” bir ‘Uygulamalı’ sınıfı var. Basit bir örnek sonsuz akışlardır:

data Stream a = Stream { headS :: a, tailS :: Stream a }

‘Akış’ın ‘Uygulamalı’ örneği, iki akıştaki değerleri konuma göre eşleştirerek nokta bazında bir argüman akışına bir işlev akışı uygular. “pure”, sabit bir akış döndürür – tek bir sabit değerin sonsuz listesi:

instance Applicative Stream where
    pure x = let s = Stream x s in s
    Stream f fs <*> Stream x xs = Stream (f x) (fs <*> xs)

Listeler de “ZipList” yeni türünün bulunduğu “Zip” bir “Uygulamalı” örneğini kabul eder:

newtype ZipList a = ZipList { getZipList :: [a] }

instance Applicative ZipList where
    ZipList xs <*> ZipList ys = ZipList $ zipWith ($) xs ys

“Zip”, sonucunu en kısa girdiye göre kırptığından, “Uygulama” yasalarını karşılayan “saf"ın tek uygulaması sonsuz bir liste döndürendir:

    pure a = ZipList (repeat a)   -- ZipList (fix (a:)) = ZipList [a,a,a,a,...

Örneğin:

ghci> getZipList $ ZipList [(+1),(+2)] <*> ZipList [3,30,300]
[4,32]

İlk durumda 1 sütunlu (n x 1) bir matrisi 1 satırlı (“1 x m”) bir matrisle çarpmaya benzer şekilde, iki olasılık bize dış ve iç çarpımı hatırlatır, n x m matrisini alır. sonuç olarak (ancak düzleştirilmiş); veya ikinci durumda 1 satır ve 1 sütunlu matrisleri (ancak toplama olmadan) çarparak.

Fonksiyonlar #

‘(->) r’ işlevlerine özelleştirildiğinde, ‘pure’ ve ‘<*>’ tip imzaları sırasıyla ‘K’ ve ‘S’ birleştiricilerininkiyle eşleşir:

pure :: a -> (r -> a)
<*> :: (r -> (a -> b)) -> (r -> a) -> (r -> b)

“pure”, “const” olmalıdır ve “<*>” bir çift işlevi alır ve her birini sabit bir bağımsız değişkene uygulayarak iki sonucu uygular:

instance Applicative ((->) r) where
    pure = const
    f <*> g = \x -> f x (g x)

İşlevler prototipik “zippy” uygulamasıdır. Örneğin, sonsuz akışlar (->) Nat ile eşbiçimli olduğundan, …

-- | Index into a stream
to :: Stream a -> (Nat -> a)
to (Stream x xs) Zero = x
to (Stream x xs) (Suc n) = to xs n

-- | List all the return values of the function in order
from :: (Nat -> a) -> Stream a
from f = from' Zero
    where from' n = Stream (f n) (from' (Suc n))

… akışları daha yüksek sıralı bir şekilde temsil etmek, zippy ‘Uygulamalı’ örneğini otomatik olarak üretir.
