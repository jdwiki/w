İşlev birleştirme operatörü (.) şu şekilde tanımlanır

(.) :: (b -> c) -> (a -> b) ->  (a -> c)
(.)       f           g          x =  f (g x)     -- or, equivalently,  

(.)       f           g     =   \x -> f (g x)     
(.)       f     =    \g     ->  \x -> f (g x)      
(.) =    \f     ->   \g     ->  \x -> f (g x)      
(.) =    \f     ->  (\g     -> (\x -> f (g x) ) )

(b -> c) -> (a -> b) -> (a -> c) türü (b -> c) -> (a -> b) -> a -> şeklinde yazılabilir. c çünkü -> türündeki imzalar sağda “ilişkilendirilir”, solla ilişkilendirilen işlev uygulamasına karşılık gelir,

 f g x y z ...    ==    (((f g) x) y) z ...

Yani “veri akışı” sağdan soladır: “x” “g"ye “gider”, sonucu “f"ye gider ve nihai sonucu üretir:

(.)       f           g          x =  r
                                      where r = f (g x)  
-- g :: a -> b
-- f ::      b -> c
-- x :: a      
-- r ::           c   

(.)       f           g     =    q
                                 where q = \x -> f (g x) 
-- g :: a -> b
-- f ::      b -> c
-- q :: a      -> c

....

Sözdizimsel olarak, aşağıdakilerin hepsi aynıdır:

(.) f g x  =  (f . g) x  =  (f .) g x  =  (. g) f x

“operatör bölümlerinin üç kuralı” olarak anlaşılması kolaydır, burada “eksik argüman” operatörün yanındaki boş yuvaya girer:

(.) f g    =  (f . g)    =  (f .) g    =  (. g) f   
--         1             2             3

Denklemin her iki tarafında da bulunan ‘x’ ihmal edilebilir. Bu eta-kasılma olarak bilinir. Bu nedenle, fonksiyon kompozisyonunun tanımını yazmanın basit yolu sadece

(f . g) x   =   f (g x)

Bu, elbette, “x” argümanına atıfta bulunur; ne zaman ‘x’ olmadan sadece ‘(f . g)’ yazsak, bu noktasız stil olarak bilinir.

Sağdan sola kompozisyon #

(.), birinin çıktısını diğerine girdi olarak besleyerek iki fonksiyon oluşturmamıza izin verir:

(f . g) x = f (g x)

Örneğin, bir giriş numarasının ardılının karesini almak istersek, yazabiliriz.

((^2) . succ) 1        --    4

Ayrıca ‘(.)’ için bir takma ad olan ‘(«<)’ vardır. Yani,

(+ 1) <<< sqrt $ 25    --    6

İkili işlevli kompozisyon #

Normal kompozisyon, tekli işlevler için çalışır. İkili durumda, tanımlayabiliriz

(f .: g) x y = f (g x y)          -- which is also
             = f ((g x) y)
             = (f . g x) y        -- by definition of (.)
             = (f .) (g x) y
             = ((f .) . g) x y

Böylece (f .:g) = ((f .) .g) eta-daralma ile ve ayrıca,

(.:) f g    = ((f .) . g)
            = (.) (f .) g
            = (.) ((.) f) g
            = ((.) . (.)) f g

yani (.:) = ((.) . (.)), yarı ünlü bir tanım.

Örnekler:

(map (+1) .: filter) even [1..5]      --  [3,5]
(length   .: filter) even [1..5]      --  2

Soldan sağa kompozisyon #

“Control.Category”, [(>>>)] tanımlar(http://hackage.haskell.org/package/base-4.9.0.0/docs/Control-Category.html#v:-62–62- -62-), işlevlere özelleştiğinde,

-- (>>>) :: Category cat => cat a b -> cat b c -> cat a c  
-- (>>>) :: (->) a b -> (->) b c -> (->) a c 
-- (>>>) :: (a -> b) -> (b -> c) -> (a -> c) 
( f >>> g ) x = g (f x)

Örnek:

sqrt >>> (+ 1) $ 25    --    6.0
