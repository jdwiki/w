“Functor”, üzerinde kovaryant olarak eşlenen olabilen “f :: * -> *” türlerinin sınıfıdır. Bir işlevi bir veri yapısı üzerinde eşlemek, işlevi yapının kendisini değiştirmeden yapının tüm öğelerine uygular.

Bir Functor, bir değer için bir kapsayıcı veya bir hesaplama bağlamı olarak düşünülebilir. Örnekler “Belki a” veya “[a]“dır. Typeclassopedia makalesinde, Functors’ın arkasındaki kavramlar hakkında iyi bir yazı bulunmaktadır.

Gerçek bir Functor olarak kabul edilmek için, bir örneğin aşağıdaki 2 yasaya uyması gerekir:

Kimlik #

fmap id == id

Kompozisyon #

fmap (f . g) = (fmap f) . (fmap g)

Functor ve Kanunların Sınıf Tanımı #

class Functor f where
    fmap :: (a -> b) -> f a -> f b

Buna bakmanın bir yolu, “fmap"in bir değerler fonksiyonunu bir “f” bağlamında bir değerler fonksiyonuna kaldırmasıdır.

Doğru bir “Functor” örneği, derleyici tarafından uygulanmasa da, functor yasalarını karşılamalıdır:

fmap id = id                    -- identity
fmap f . fmap g = fmap (f . g)  -- composition

“fmap” için “<$>” adında yaygın olarak kullanılan bir ek takma adı vardır.

infixl 4 <$>
(<$>) :: Functor f => (a -> b) -> f a -> f b
(<$>) = fmap

Yaygın Functor örnekleri #

Belki #

“Belki”, muhtemelen mevcut olmayan bir değer içeren bir “İşlevseldir”:

instance Functor Maybe where
    fmap f Nothing = Nothing
    fmap f (Just x) = Just (f x)

“Belki"nin “Functor” örneği, “Just” içine alınmış bir değere bir işlev uygular. Hesaplama daha önce başarısız olduysa (“Belki” değeri “Hiçbir şey” ise), işlevi uygulayacak bir değer yoktur, bu nedenle “fmap” işlemsizdir.

> fmap (+ 3) (Just 3)
Just 6
> fmap length (Just "mousetrap")
Just 9
> fmap sqrt Nothing
Nothing

Denklemsel akıl yürütmeyi kullanarak bu örnek için işlev yasalarını kontrol edebiliriz. Kimlik kanunu için,

fmap id Nothing
Nothing  -- definition of fmap
id Nothing  -- definition of id

fmap id (Just x)
Just (id x)  -- definition of fmap
Just x  -- definition of id
id (Just x)  -- definition of id

Kompozisyon kanunu için,

(fmap f . fmap g) Nothing
fmap f (fmap g Nothing)  -- definition of (.)
fmap f Nothing  -- definition of fmap
Nothing  -- definition of fmap
fmap (f . g) Nothing  -- because Nothing = fmap f Nothing, for all f

(fmap f . fmap g) (Just x)
fmap f (fmap g (Just x))  -- definition of (.)
fmap f (Just (g x))  -- definition of fmap
Just (f (g x))  -- definition of fmap
Just ((f . g) x)  -- definition of (.)
fmap (f . g) (Just x)  -- definition of fmap

Listeler #

Lists’in ‘Functor’ örneği, işlevi yerinde listedeki her değere uygular.

instance Functor [] where
    fmap f [] = []
    fmap f (x:xs) = f x : fmap f xs

Bu, alternatif olarak bir liste kavrayışı olarak da yazılabilir: fmap f xs = [f x | x <- xs].

Bu örnek, “fmap"in “harita"yı genelleştirdiğini gösterir. “map” yalnızca listelerde çalışır, oysa “fmap” keyfi bir “İşlev” üzerinde çalışır.

Kimlik yasasının tümevarım yoluyla geçerli olduğu gösterilebilir:

-- base case
fmap id []
[]  -- definition of fmap
id []  -- definition of id

-- inductive step
fmap id (x:xs)
id x : fmap id xs  -- definition of fmap
x : fmap id xs  -- definition of id
x : id xs  -- by the inductive hypothesis
x : xs  -- definition of id
id (x : xs)  -- definition of id

ve benzer şekilde, bileşim yasası:

-- base case
(fmap f . fmap g) []
fmap f (fmap g [])  -- definition of (.)
fmap f []  -- definition of fmap
[]  -- definition of fmap
fmap (f . g) []  -- because [] = fmap f [], for all f

-- inductive step
(fmap f . fmap g) (x:xs)
fmap f (fmap g (x:xs))  -- definition of (.)
fmap f (g x : fmap g xs)  -- definition of fmap
f (g x) : fmap f (fmap g xs)  -- definition of fmap
(f . g) x : fmap f (fmap g xs)  -- definition of (.)
(f . g) x : fmap (f . g) xs  -- by the inductive hypothesis
fmap (f . g) xs  -- definition of fmap

Fonksiyonlar #

Her “Functor” bir kapsayıcıya benzemez. Functions’ın ‘Functor’ örneği, bir işlevi başka bir işlevin dönüş değerine uygular.

instance Functor ((->) r) where
    fmap f g = \x -> f (g x)

Bu tanımın “fmap = (.)” ile eşdeğer olduğunu unutmayın. Yani ‘fmap’ fonksiyon kompozisyonunu genelleştirir.

Kimlik yasasını bir kez daha kontrol edin:

fmap id g
\x -> id (g x)  -- definition of fmap
\x -> g x  -- definition of id
g  -- eta-reduction
id g  -- definition of id

ve kompozisyon yasası:

(fmap f . fmap g) h
fmap f (fmap g h)  -- definition of (.)
fmap f (\x -> g (h x))  -- definition of fmap
\y -> f ((\x -> g (h x)) y)  -- definition of fmap
\y -> f (g (h y))  -- beta-reduction
\y -> (f . g) (h y)  -- definition of (.)
fmap (f . g) h  -- definition of fmap

Bir Functor’ın tüm öğelerini tek bir değerle değiştirme #

“Data.Functor” modülü, bir işlevde bulunan tüm değerleri yok sayan ve hepsini tek bir sabit değerle değiştiren “<$” ve “$>” adlı iki birleştirici içerir.

infixl 4 <$, $>

<$ :: Functor f => a -> f b -> f a
(<$) = fmap . const

$> :: Functor f => f a -> b -> f b
($>) = flip (<$)

“void”, bir hesaplamanın dönüş değerini yok sayar.

void :: Functor f => f a -> f ()
void = (() <$)

Polinom fonksiyonları #

Küçük olanlardan büyük ‘Functor’lar oluşturmak için kullanışlı bir dizi tip birleştirici var. Bunlar, ‘Functor’un örnek örnekleri olarak öğreticidir ve ayrıca genel programlama için bir teknik olarak yararlıdırlar, çünkü geniş bir ortak işlev sınıfını temsil etmek için kullanılabilirler.

kimlik functor #

Kimlik functor, argümanını basitçe tamamlar. SKI hesabındaki “I” birleştiricisinin tür düzeyinde bir uygulamasıdır.

newtype I a = I a

instance Functor I where
    fmap f (I x) = I (f x)

“Ben”, “Kimlik” adı altında Data.Functor.Identity modülünde bulunabilir. Functor-Identity.html).

sabit işlev #

Sabit işlev, yalnızca sabit bir değer içeren ikinci argümanını yok sayar. Bu, SKI hesabındaki “K” birleştiricisi olan “const"un tür düzeyinde bir analogudur.

newtype K c a = K c

“K c a"nın herhangi bir “a” değeri içermediğine dikkat edin; “K ()”, “Proxy” ile eş biçimlidir. Bu, Knin fmap uygulamasının hiçbir eşleme yapmadığı anlamına gelir!

instance Functor (K c) where
    fmap _ (K c) = K c

“K”, Data.Functor.Const kaynağından “Const” olarak da bilinir. .

Bu örnekte kalan işlevler, daha küçük işlevlerle daha büyük olanları birleştirir.

Functor ürünleri #

Functor ürünü bir çift functor alır ve bunları paketler. (,) :: * -> * -> * types üzerinde çalışırken *, (:*:) :: (* -> *) -> (*) dışında bir demete benzer. -> *) -> (* -> *), işlevler``* -> * üzerinde çalışır.

infixl 7 :*:
data (f :*: g) a = f a :*: g a

instance (Functor f, Functor g) => Functor (f :*: g) where
    fmap f (fx :*: gy) = fmap f fx :*: fmap f gy

Bu tür, “Ürün” adı altında Data.Functor.Product modülünde bulunabilir. Ürün.html).

fonksiyonel yan ürünler #

Tıpkı ‘:*:’ ‘(,)’ ile benzer olduğu gibi, ‘:+:’ ‘Either’ın işlev düzeyindeki analogudur.

infixl 6 :+:
data (f :+: g) a = InL (f a) | InR (g a)

instance (Functor f, Functor g) => Functor (f :+: g) where
    fmap f (InL fx) = InL (fmap f fx)
    fmap f (InR gy) = InR (fmap f gy)

:+:, [Data.Functor.Sum modülünde]Sum adı altında bulunabilir(https://hackage.haskell.org/package/base-4.9.0.0/docs/Data- Functor-Sum.html).

işlev kompozisyonu #

Son olarak, :.: bir tür seviyesi (.) gibi çalışır, bir işlevcinin çıktısını alır ve onu diğerinin girişine bağlar.

infixr 9 :.:
newtype (f :.: g) a = Cmp (f (g a))

instance (Functor f, Functor g) => Functor (f :.: g) where
    fmap f (Cmp fgx) = Cmp (fmap (fmap f) fgx)

“Oluştur” türü Data.Functor.Compose içinde bulunabilir.

Genel programlama için polinom fonksiyonları #

‘I’, ‘K’, ‘:*:’, ‘:+:’ ve ‘:.:’ belirli bir basit veri türü sınıfı için bir yapı taşları seti olarak düşünülebilir. Kit, [sabit noktalar](https://www.wikiod.com/tr/haskell/ozyineleme-semalar#Sabit noktalar) ile birleştirdiğinizde özellikle güçlü hale gelir çünkü bu birleştiricilerle oluşturulan veri türleri otomatik olarak Functor örnekleri. Kiti bir şablon türü oluşturmak için kullanırsınız, “I"yi kullanarak özyinelemeli noktaları işaretlersiniz ve ardından standart özyineleme şemaları hayvanat bahçesiyle kullanılabilecek bir tür elde etmek için onu “Düzelt"e eklersiniz.

| İsim | Veri türü olarak | İşlev setini kullanma | |————————–|—————————— ———————————–|————– ———————————–| | Değer çiftleri | veri Çifti a = Çift a a | tür Çifti = Ben :*: Ben | | İkiye iki ızgara | türü Izgara a = Çift (Çift a) | type Grid = Çift :.: Çift | | Doğal sayılar | veri Nat = Sıfır | Succ Nat | type Nat = Düzelt (K () :+: I) | | Listeler | veri Listesi a = Nil | Eksileri a (Liste a) | tür Liste a = Düzelt (K () :+: Ka :*: I) | | İkili ağaçlar | veri Ağacı a = Yaprak | Düğüm (Ağaç a) a (Ağaç a) | türü Ağaç a = Düzelt (K () :+: I :*: K a :*: I) | | Gül ağaçları | data Rose a = Rose a (Liste (Rose a)) | type Rose a = Fix (K a :*: Liste :.: I) |

Veri türlerini tasarlamaya yönelik bu “kit” yaklaşımı, generics-sop gibi genel programlama kitaplıklarının arkasındaki fikirdir. Buradaki fikir, yukarıda sunulana benzer bir kit kullanarak genel işlemler yazmak ve ardından rastgele veri türlerini genel temsillerine dönüştürmek için bir tür sınıfı kullanmaktır:

class Generic a where
    type Rep a  -- a generic representation built using a kit
    to :: a -> Rep a
    from :: Rep a -> a

Kategori Teorisinde Fonksiyonlar #

Bir Functor, kategori teorisinde kategoriler arasında yapıyı koruyan bir harita (bir ‘homomorfizm’) olarak tanımlanır. Spesifik olarak, (tüm) nesneler nesnelere eşlenir ve (tümü) oklar, kategori yasaları korunacak şekilde oklara eşlenir.

Nesnelerin Haskell türleri ve morfizmlerin Haskell işlevleri olduğu kategoriye Hask denir. Bu nedenle, Hask’tan Hask‘a bir işlev, türlerin türlere eşlenmesinden ve işlevlerden işlevlere eşlenmesinden oluşur.

Bu kategori teorik kavramının Haskell programlama yapısı “Functor” ile olan ilişkisi oldukça doğrudandır. Türlerden türlere eşleme f :: * -> * tipinin biçimini alır ve işlevlerden işlevlere eşleme bir işlev biçimini alır fmap :: (a -> b) -> (f a - > f b). Bunları bir sınıfta bir araya getirmek,

class Functor (f :: * -> *) where
    fmap :: (a -> b) -> f a -> f b

“fmap”, bir işlevi (bir tür morfizm), :: a -> b alan ve onu başka bir işlev olan :: f a -> f b ile eşleyen bir işlemdir. Hask‘ın kategorik yapısını koruyarak, “Functor” örneklerinin gerçekten matematiksel işlevler olduğu varsayılır (ancak bunu sağlamak programcıya bırakılır):

fmap (id {- :: a -> a -})  ==  id {- :: f a -> f a -}
fmap (h . g)               ==  fmap h . fmap g

‘fmap’, ‘:: a -> b’ işlevini, hem kimlik oklarının varlığını hem de bileşimin ilişkilendirilebilirliğini koruyacak şekilde Hask alt kategorisine kaldırır.

“Functor” sınıfı yalnızca Hask üzerindeki endoişlevlerini kodlar. Ancak matematikte, işlevler keyfi kategoriler arasında eşlenebilir. Bu kavramın daha sadık bir kodlaması şöyle görünür:

class Category c where
    id  :: c i i
    (.) :: c j k -> c i j -> c i k

class (Category c1, Category c2) => CFunctor c1 c2 f where
    cfmap :: c1 a b -> c2 (f a) (f b)

Standart Functor sınıfı, kaynak ve hedef kategorilerinin her ikisinin de Hask olduğu bu sınıfın özel bir durumudur. Örneğin,

instance Category (->) where        -- Hask
    id    = \x -> x
    f . g = \x -> f (g x)

instance CFunctor (->) (->) [] where
    cfmap = fmap

Functor Türetme #

{-# LANGUAGE DeriveFunctor #-}

data List a = Nil | Cons a (List a) deriving Functor

map :: (a -> b) -> List a -> List b
