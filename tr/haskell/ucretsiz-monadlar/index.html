Ücretsiz monad’lar, monadik hesaplamaları veri yapılarına ve yorumlayıcılara böler #

Örneğin, komut isteminden okuma ve yazma komutlarını içeren bir hesaplama:

İlk önce hesaplamamızın “komutlarını” bir Functor veri türü olarak tanımlarız.

{-# LANGUAGE DeriveFunctor #-}

data TeletypeF next
    = PrintLine String next
    | ReadLine (String -> next)
    deriving Functor

Daha sonra “TeletypeF” üzerinden Serbest Monad oluşturmak için “Ücretsiz"i kullanırız ve bazı temel işlemler oluştururuz.

import Control.Monad.Free (Free, liftF, iterM)

type Teletype = Free TeletypeF

printLine :: String -> Teletype ()
printLine str = liftF (PrintLine str ())

readLine :: Teletype String
readLine = liftF (ReadLine id)

“F” bir “Functor” olduğunda “Free f” bir “Monad” olduğundan, “Teletype” hesaplamaları oluşturmak için standart “Monad” birleştiricilerini (“do” notasyonu dahil) kullanabiliriz.

import Control.Monad -- we can use the standard combinators

echo :: Teletype ()
echo = readLine >>= printLine

mockingbird :: Teletype a
mockingbird = forever echo

Son olarak, “Teletype a” değerlerini “IO a” gibi nasıl çalışılacağını bildiğimiz bir şeye dönüştüren bir “yorumlayıcı” yazarız.

interpretTeletype :: Teletype a -> IO a
interpretTeletype = foldFree run where
  run :: TeletypeF a -> IO a
  run (PrintLine str x) = putStrLn *> return x
  run (ReadLine f) = fmap f getLine

Bunu, “IO"da “Teletype a” hesaplamasını “çalıştırmak” için kullanabiliriz.

> interpretTeletype mockingbird
hello
hello
goodbye
goodbye
this will go on forever
this will go on forever

Daha Özgür monad #

Serbest monadın Freer (veya Prompt veya Operational) monad olarak adlandırılan alternatif bir formülasyonu vardır. Freer monad, temel komut seti için bir Functor örneğine ihtiyaç duymaz ve standart ücretsiz monaddan daha tanınabilir şekilde liste benzeri bir yapıya sahiptir.

Freer monad, programları i :: * -> * komut kümesine ait bir atomik talimatlar dizisi olarak temsil eder. Her komut, dönüş türünü bildirmek için parametresini kullanır. Örneğin, “Devlet” monad için temel komutlar seti aşağıdaki gibidir:

data StateI s a where
    Get :: StateI s s  -- the Get instruction returns a value of type 's'
    Put :: s -> StateI s ()  -- the Put instruction contains an 's' as an argument and returns ()

Bu talimatların sıralanması :>>= yapıcısı ile gerçekleşir. :>>=, bir a döndüren tek bir talimat alır ve onu programın geri kalanına ekler, dönüş değerini devama aktarır. Başka bir deyişle, bir “a” döndüren bir talimat ve bir “a"yı “b” döndüren bir programa dönüştüren bir işlev verildiğinde, “:»=”, bir “b” döndüren bir program üretecektir.

data Freer i a where
    Return :: a -> Freer i a
    (:>>=) :: i a -> (a -> Freer i b) -> Freer i b

“a” öğesinin “:»=’ yapıcısında varoluşsal olarak nicelleştirildiğine dikkat edin. Bir tercümanın “a"nın ne olduğunu öğrenmesinin tek yolu, GADT “i” üzerindeki kalıp eşleştirmesidir.

Bir kenara: Ortak Yoneda lemması bize “Özgür"ün “Özgür” ile eşbiçimli olduğunu söyler. ‘CoYoneda’ functor tanımını hatırlayın:

veri CoYoneda i b nerede CoYoneda :: i a -> (a -> b) -> CoYoneda i b

“Freer i”, “Free (CoYoneda i)” ile eşdeğerdir. [Freenin yapıcılarını](https://www.wikiod.com/tr/haskell/ucretsiz-monadlar#Serbest Monadlar sabit noktalar gibidir) alır ve ayarlarsanız f ~ CoYoneda i, şunları elde edersiniz:

Saf :: a -> Ücretsiz (CoYoneda i) a Ücretsiz :: CoYoneda i (Ücretsiz (CoYoneda i) b) -> Ücretsiz (CoYonda i) b ~ i a -> (a -> Ücretsiz (CoYoneda i) b) -> Ücretsiz (CoYoneda i) b

Freer i kurucularını sadece ayarlayarak kurtarabileceğimiz Daha özgür i ~ Ücretsiz (CoYoneda i).

“CoYoneda i” herhangi bir “i” için bir “İşlev” olduğundan, “Serbest”, “i” bir “İşlev” olmasa bile, herhangi bir “i” için bir “Monad"dır.

instance Monad (Freer i) where
    return = Return
    Return x >>= f = f x
    (i :>>= g) >>= f = i :>>= fmap (>>= f) g  -- using `(->) r`'s instance of Functor, so fmap = (.)

Talimatları bazı işleyici monadlarına eşleyerek ‘Freer’ için yorumlayıcılar oluşturulabilir.

foldFreer :: Monad m => (forall x. i x -> m x) -> Freer i a -> m a
foldFreer eta (Return x) = return x
foldFreer eta (i :>>= f) = eta i >>= (foldFreer eta . f)

Örneğin, bir işleyici olarak normal ‘State s’ monadını kullanarak ‘Freer (StateI s)’ monadını yorumlayabiliriz:

runFreerState :: Freer (StateI s) a -> s -> (a, s)
runFreerState = State.runState . foldFreer toState
    where toState :: StateI s a -> State s a
          toState Get = State.get
          toState (Put x) = State.put x

Ücretsiz Monad’lar sabit noktalar gibidir #

“Ücretsiz” tanımını [“Düzelt”](https://www.wikiod.com/tr/haskell/ozyineleme-semalar#Sabit noktalar) ile karşılaştırın:

data Free f a = Return a
              | Free (f (Free f a))

newtype Fix f = Fix { unFix :: f (Fix f) }

Özellikle, “Serbest” kurucunun tipini “Fix” kurucu tipi ile karşılaştırın. “Serbest”, “Fix” gibi bir işlev üstlenir, ancak “Free"nin ek bir “Return a” durumu olması dışında.

FoldFree ve iterM nasıl çalışır? #

‘Serbest’ hesaplamaları başka bir monad ’m’: ‘iterM :: (Functor f, Monad m) => (f (m a) -> m a) -> (Serbest f a - > m a)vefoldFree :: Monad m => (tüm x. f x -> m x) -> (Serbest f a -> m a)`. Onlar ne yapıyor?

İlk önce, bir ‘Teletype a’ işlevini manuel olarak ‘IO’ya yorumlamak için ne gerektiğini görelim. Free f ayı tanımlı olarak görebiliriz

data Free f a 
    = Pure a 
    | Free (f (Free f a))

“Saf” durum kolaydır:

interpretTeletype :: Teletype a -> IO a
interpretTeletype (Pure x) = return x
interpretTeletype (Free teletypeF) = _

Şimdi, ‘Free’ yapıcısı ile oluşturulmuş bir ‘Teletype’ hesaplaması nasıl yorumlanır? “TeletypeF :: TeletypeF (Teletype a)“yı inceleyerek “IO a” türünde bir değere ulaşmak istiyoruz. Başlangıç ​​olarak, ücretsiz monadın tek bir katmanını bir “IO” eylemine eşleyen “runIO :: TeletypeF a -> IO a” işlevini yazacağız:

runIO :: TeletypeF a -> IO a
runIO (PrintLine msg x) = putStrLn msg *> return x
runIO (ReadLine k) = fmap k getLine

Şimdi ‘interpretTeletype’ın geri kalanını doldurmak için ‘runIO’ kullanabiliriz. ’teletypeF :: TeletypeF (Teletype a)’ öğesinin, ‘Serbest’ hesaplamanın geri kalanını içeren ‘TeletypeF’ işlevinin bir katmanı olduğunu hatırlayın. En dış katmanı yorumlamak için ‘runIO’ kullanacağız (böylece ‘runIO teletypeF :: IO (Teletype a)’ var) ve sonra döndürülen ‘Teletype a’yı yorumlamak için ‘IO’ monad’ın ‘»=’ birleştiricisini kullanacağız. .

interpretTeletype :: Teletype a -> IO a
interpretTeletype (Pure x) = return x
interpretTeletype (Free teletypeF) = runIO teletypeF >>= interpretTeletype

“foldFree"nin tanımı, “runIO” fonksiyonunun dışlanmış olması dışında, sadece “interpretTeletype” tanımıdır. Sonuç olarak, ‘foldFree’ herhangi bir belirli temel işlevden ve herhangi bir hedef monaddan bağımsız olarak çalışır.

foldFree :: Monad m => (forall x. f x -> m x) -> Free f a -> m a
foldFree eta (Pure x) = return x
foldFree eta (Free fa) = eta fa >>= foldFree eta

‘foldFree’ bir rank-2 tipine sahiptir: ’eta’ doğal bir dönüşümdür. ‘foldFree’ye bir tür ‘Monad m => (f (Serbest f a) -> m (Serbest f a)) -> Serbest f a -> m a’ verebilirdik, ancak bu, ’eta’ya ` “f” katmanının içindeki ücretsiz hesaplama. Bu daha kısıtlayıcı türü “foldFree” vermek, “eta"nın bir seferde yalnızca tek bir katmanı işleyebilmesini sağlar.

“iterM”, katlama işlevine alt hesaplamayı inceleme yeteneği verir. Önceki yinelemenin (monadik) sonucu, “f” parametresinin içinde bir sonraki için kullanılabilir. “iterM”, [paramorfizm](https://www.wikiod.com/tr/haskell/ozyineleme-semalar#ilkel özyineleme) ile benzerdir, oysa “foldFree” [catamorfizm]( https://www.wikiod.com/tr/haskell/ozyineleme-semalar#Bir yapıyı her seferinde bir katman katlamak).

iterM :: (Monad m, Functor f) => (f (m a) -> m a) -> Free f a -> m a
iterM phi (Pure x) = return x
iterM phi (Free fa) = phi (fmap (iterM phi) fa)
