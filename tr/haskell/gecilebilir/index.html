‘Traversable’ sınıfı, daha önce ‘mapM :: Monad m => (a -> m b) -> [a] -> m [b]’ olarak bilinen işlevi, listeler dışındaki yapılar üzerinde ‘Uygulamalı’ etkilerle çalışmak üzere genelleştirir.

Bir yapıyı tersten dolaşmak #

Backwards uygulamalı işlev yardımıyla bir geçiş ters yönde çalıştırılabilir. , mevcut bir uygulamayı ters çevirerek oluşan etkilerin ters sırada gerçekleşmesini sağlar.

newtype Backwards f a = Backwards { forwards :: f a }

instance Applicative f => Applicative (Backwards f) where
    pure = Backwards . pure
    Backwards ff <*> Backwards fx = Backwards ((\x f -> f x) <$> fx <*> ff)

“Geri”, “tersine çevrilmiş bir “hareket” içinde kullanıma konabilir. Bir “geçiş” çağrısının temeldeki uygulaması “Geri” ile çevrildiğinde, ortaya çıkan etki ters sırada gerçekleşir.

newtype Reverse t a = Reverse { getReverse :: t a }

instance Traversable t => Traversable (Reverse t) where
    traverse f = fmap Reverse . forwards . traverse (Backwards . f) . getReverse

ghci> traverse print (Reverse "abc")
'c'
'b'
'a'

‘Reverse’ yeni tipi Data.Functor.Reverse altında bulunur.

Traversable’ın Tanımı #

class (Functor t, Foldable t) => Traversable t where
    {-# MINIMAL traverse | sequenceA #-}
    
    traverse :: Applicative f => (a -> f b) -> t a -> f (t b)
    traverse f = sequenceA . fmap f
    
    sequenceA :: Applicative f => t (f a) -> f (t a)
    sequenceA = traverse id
    
    mapM :: Monad m => (a -> m b) -> t a -> m (t b)
    mapM = traverse
    
    sequence :: Monad m => t (m a) -> m (t a)
    sequence = sequenceA

“Geçilebilir” yapılar “t”, üzerinde etkili bir “ziyaretçi” işlemiyle çalıştırılabilen “a” öğelerinin sonlu kapsayıcılarıdır. Ziyaretçi işlevi ‘f :: a -> f b’ yapının her bir öğesi üzerinde bir yan etki gerçekleştirir ve ‘geçiş’ bu yan etkileri ‘Uygulamalı’ kullanarak oluşturur. Buna bakmanın başka bir yolu, “diziA"nın, “Geçitilebilir” yapıların “Uygulamalı” yapılarla gidip geldiğini söylemesidir.

İkili ağaç için Traversable örneği #

“Geçiş” uygulamaları genellikle “Uygulama” bağlamına kaldırılmış bir “fmap” uygulaması gibi görünür.

data Tree a = Leaf
            | Node (Tree a) a (Tree a)

instance Traversable Tree where
    traverse f Leaf = pure Leaf
    traverse f (Node l x r) = Node <$> traverse f l <*> f x <*> traverse f r

Bu uygulama, ağacın sıralı geçişini gerçekleştirir.

ghci> let myTree = Node (Node Leaf 'a' Leaf) 'b' (Node Leaf 'c' Leaf)

--    +--'b'--+
--    |       |
-- +-'a'-+ +-'c'-+
-- |     | |     |
-- *     * *     *

ghci> traverse print myTree
'a'
'b'
'c'

“DeriveTraversable” uzantısı, GHC’nin türün yapısına göre “Gezilebilir” örnekler oluşturmasına olanak tanır. “Düğüm” yapıcısının düzenini ayarlayarak makine tarafından yazılan geçişin sırasını değiştirebiliriz.

data Inorder a = ILeaf
               | INode (Inorder a) a (Inorder a)  -- as before
               deriving (Functor, Foldable, Traversable)  -- also using DeriveFunctor and DeriveFoldable

data Preorder a = PrLeaf
                | PrNode a (Preorder a) (Preorder a)
                deriving (Functor, Foldable, Traversable)

data Postorder a = PoLeaf
                 | PoNode (Postorder a) (Postorder a) a
                 deriving (Functor, Foldable, Traversable)

-- injections from the earlier Tree type
inorder :: Tree a -> Inorder a
inorder Leaf = ILeaf
inorder (Node l x r) = INode (inorder l) x (inorder r)

preorder :: Tree a -> Preorder a
preorder Leaf = PrLeaf
preorder (Node l x r) = PrNode x (preorder l) (preorder r)

postorder :: Tree a -> Postorder a
postorder Leaf = PoLeaf
postorder (Node l x r) = PoNode (postorder l) (postorder r) x

ghci> traverse print (inorder myTree)
'a'
'b'
'c'
ghci> traverse print (preorder myTree)
'b'
'a'
'c'
ghci> traverse print (postorder myTree)
'a'
'c'
'b'

Geçilebilir bir yapı için Functor ve Katlanabilir Örnekleme #

import Data.Traversable as Traversable

data MyType a =  -- ...
instance Traversable MyType where
    traverse = -- ...

Her ‘Gezilebilir’ yapı, ‘fmapDefault’ kullanılarak bir ‘Katlanabilir’ ‘İşlevsel’ yapılabilir. :fmapDefault) ve foldMapDefault “Data.Traversable” içinde bulunan işlevler.

instance Functor MyType where
    fmap = Traversable.fmapDefault

instance Foldable MyType where
    foldMap = Traversable.foldMapDefault

“fmapDefault”, “Identity” uygulama işlevinde “traverse” çalıştırılarak tanımlanır.

newtype Identity a = Identity { runIdentity :: a }

instance Applicative Identity where
    pure = Identity
    Identity f <*> Identity x = Identity (f x)

fmapDefault :: Traversable t => (a -> b) -> t a -> t b
fmapDefault f = runIdentity . traverse (Identity . f)

“foldMapDefault”, biriktirirken parametresini yok sayan Const uygulamalı işlevi kullanılarak tanımlanır. monoid bir değer.

newtype Const c a = Const { getConst :: c }

instance Monoid m => Applicative (Const m) where
    pure _ = Const mempty
    Const x <*> Const y = Const (x `mappend` y)

foldMapDefault :: (Traversable t, Monoid m) => (a -> m) -> t a -> m
foldMapDefault f = getConst . traverse (Const . f)

İçeriğe sahip şekiller olarak geçilebilir yapılar #

Bir t’ türü ‘Gezilebilir’ ise, t a’nın değerleri iki parçaya ayrılabilir: “şekil” ve “içerik”:

data Traversed t a = Traversed { shape :: t (), contents :: [a] }

burada “içerik”, “Katlanabilir” bir örnek kullanarak “ziyaret ettiğiniz” ile aynıdır.

’t a’dan ‘Traversed t a’ya tek bir yöne gitmek, ‘Functor’ ve ‘Foldable’ dışında hiçbir şey gerektirmez

break :: (Functor t, Foldable t) => t a -> Traversed t a 
break ta = Traversed (fmap (const ()) ta) (toList ta)

ancak geri gitmek, ‘geçiş’ işlevini çok önemli bir şekilde kullanır

import Control.Monad.State

-- invariant: state is non-empty
pop :: State [a] a
pop = state $ \(a:as) -> (a, as)

recombine :: Traversable t => Traversed t a -> t a
recombine (Traversed s c) = evalState (traverse (const pop) s) c

“Geçilebilir” yasalar bu “kopmayı” gerektirir. rekombine’ ve ‘rekombine. break` her ikisi de kimliktir. Dikkat çekici bir şekilde, bu, “şekli” hiçbir artık bırakmadan tamamen doldurmak için “içeriklerde” tam olarak doğru sayı öğelerinin olduğu anlamına gelir.

“Geçilen t"nin kendisi “Geçitilebilir"dir. ‘Traverse’ uygulaması, listenin ‘Traversable’ örneğini kullanarak öğeleri ziyaret ederek ve ardından eylemsiz şekli sonuca yeniden ekleyerek çalışır.

instance Traversable (Traversed t) where
    traverse f (Traversed s c) = fmap (Traversed s) (traverse f c)

Biriken bir parametre yardımıyla Traversable bir yapının dönüştürülmesi #

Liste listesini aktarma #

‘zip’in bir dizi listeyi bir demet listesine aktardığına dikkat ederek,

ghci> uncurry zip ([1,2],[3,4])
[(1,3), (2,4)]

ve “devrik” ve “diziA” türleri arasındaki benzerlik,

-- transpose exchanges the inner list with the outer list
--           +---+-->--+-+
--           |   |     | |
transpose :: [[a]] -> [[a]]
--            | |     |   |
--            +-+-->--+---+

-- sequenceA exchanges the inner Applicative with the outer Traversable
--                                             +------>------+
--                                             |             |
sequenceA :: (Traversable t, Applicative f) => t (f a) -> f (t a)
--                                                |       |
--                                                +--->---+

Buradaki fikir, “diziA"yı bir tür n-ary “zip” olarak dağıtmak için “[]“nin “Gezilebilir” ve “Uygulamalı” yapısını kullanmak ve tüm iç listeleri noktasal olarak birlikte sıkıştırmaktır.

[] varsayılan “öncelikli seçim” “Uygulamalı” örneği bizim kullanımımız için uygun değil - “zippy” bir “Uygulamalı"ya ihtiyacımız var. Bunun için ‘Control.Aplicative’ içinde bulunan ‘ZipList’ yeni türünü kullanıyoruz.

newtype ZipList a = ZipList { getZipList :: [a] }

instance Applicative ZipList where
    pure x = ZipList (repeat x)
    ZipList fs <*> ZipList xs = ZipList (zipWith ($) fs xs)

Şimdi ‘ZipList’ ‘Uygulamalı’ içinde gezinerek ücretsiz olarak ’transpoze’ alıyoruz.

transpose :: [[a]] -> [[a]]
transpose = getZipList . traverse ZipList

ghci> let myMatrix = [[1,2,3],[4,5,6],[7,8,9]]
ghci> transpose myMatrix
[[1,4,7],[2,5,8],[3,6,9]]
