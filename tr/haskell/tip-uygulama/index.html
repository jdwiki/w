‘TypeApplications’, derleyici belirli bir ifade için türler çıkarmaya çalışırken annotations türüne bir alternatiftir.

Bu örnek dizisi, ‘TypeApplications’ uzantısının amacını ve nasıl kullanılacağını açıklayacaktır.

Kaynak dosyanızın en üstüne {-# LANGUAGE TypeApplications #-} koyarak uzantıyı etkinleştirmeyi unutmayın.

Tür ek açıklamalarından kaçınma #

Belirsizliği önlemek için tür açıklamaları kullanıyoruz. Tip uygulamaları aynı amaç için kullanılabilir. Örneğin

x :: Num a => a
x = 5

main :: IO ()
main = print x

Bu kodda belirsizlik hatası var. “a"nın bir “Num” örneğine sahip olduğunu biliyoruz ve onu yazdırmak için bir “Show” örneğine ihtiyacı olduğunu biliyoruz. Bu, örneğin “a” bir “Int” ise işe yarayabilir, bu nedenle hatayı düzeltmek için bir tür ek açıklaması ekleyebiliriz

main = print (x :: Int)

Tip uygulamalarını kullanan başka bir çözüm şöyle görünür

main = print @Int x

Bunun ne anlama geldiğini anlamak için “print” tipi imzasına bakmamız gerekiyor.

print :: Show a => a -> IO ()

İşlev, “a” türünden bir parametre alır, ancak ona bakmanın başka bir yolu, aslında iki parametre almasıdır. Birincisi bir type parametresidir, ikincisi ise tipi ilk parametre olan bir değerdir.

Değer parametreleri ile tür parametreleri arasındaki temel fark, sonuncuların, onları çağırdığımızda işlevlere örtük olarak sağlanmasıdır. Onları kim sağlıyor? Tür çıkarım algoritması! ‘TypeApplications’ bize bu tip parametreleri açıkça vermektir. Bu, özellikle tür çıkarımı doğru türü belirleyemediğinde kullanışlıdır.

Yani yukarıdaki örneği yıkmak için

print :: Show a => a -> IO ()
print @Int :: Int -> IO ()
print @Int x :: IO ()

Diğer dillerdeki uygulamaları yazın #

Java, C# veya C++ gibi dillere ve jenerikler/şablonlar kavramına aşinaysanız, bu karşılaştırma sizin için yararlı olabilir.

Diyelim ki C#‘da genel bir fonksiyonumuz var.

public static T DoNothing<T>(T in) { return in; }

Bu işlevi bir ‘float’ ile çağırmak için ‘DoNothing(5.0f)’ yapabiliriz veya açık olmak istiyorsak ‘DoNothing(5.0f)’ diyebiliriz. Köşeli parantezlerin içindeki o kısım tip uygulamasıdır.

Haskell’de, tür parametrelerinin yalnızca çağrı sitelerinde değil, aynı zamanda tanım sitelerinde de örtük olması dışında aynıdır.

doNothing :: a -> a
doNothing x = x

Bu ayrıca, bunun gibi ‘ScopedTypeVariables’, ‘Rank2Types’ veya ‘RankNTypes’ uzantıları kullanılarak da açık hale getirilebilir.

doNothing :: forall a. a -> a
doNothing x = x

Ardından çağrı yerine tekrar doNothing 5.0 veya doNothing @Float 5.0 yazabiliriz.

Parametrelerin sırası #

Birden fazla argümana sahip olduğumuzda, tür argümanlarının örtük olmasıyla ilgili sorun bariz hale gelir. Hangi sırayla geliyorlar?

const :: a -> b -> a

“const @Int” yazmak, “a"nın “Int"e eşit olduğu anlamına mı geliyor, yoksa “b” mi? Tür parametrelerini, “const :: forall a b” gibi bir “forall” kullanarak açıkça belirtmemiz durumunda. a -> b -> asonra sıra yazıldığı gibidir:a, sonra b`.

Bunu yapmazsak, değişkenlerin sırası soldan sağadır. Bahsedilecek ilk değişken birinci tip parametre, ikincisi ikinci tip parametre vb.

Ya ikinci tür değişkeni belirtmek istiyorsak, ancak ilkini değil? Bunun gibi ilk değişken için bir joker karakter kullanabiliriz

const @_ @Int

Bu ifadenin türü

const @_ @Int :: a -> Int -> a

Belirsiz türlerle etkileşim #

Bayt cinsinden boyutu olan bir tür sınıfını tanıttığınızı varsayalım.

class SizeOf a where
    sizeOf :: a -> Int

Sorun, boyutun bu türün her değeri için sabit olması gerektiğidir. Biz aslında ‘sizeOf’ işlevinin ‘a’ya bağlı olmasını istemiyoruz, sadece türüne bağlı olmasını istiyoruz.

Tip uygulamaları olmadan elimizdeki en iyi çözüm bu şekilde tanımlanan Proxy tipiydi.

data Proxy a = Proxy

Bu türün amacı, tür bilgisini taşımaktır, ancak değer bilgisi içermez. O zaman sınıfımız şöyle görünebilir

class SizeOf a where
    sizeOf :: Proxy a -> Int

Şimdi merak ediyor olabilirsiniz, neden ilk argümanı tamamen bırakmıyorsunuz? O zaman fonksiyonumuzun türü sadece ‘sizeOf :: Int’ veya bir sınıfın yöntemi olduğu için daha kesin olmak gerekirse ‘sizeOf :: SizeOf a => Int’ veya daha açık olmak gerekirse ‘sizeOf :: olacaktır. forall a. SizeOf a => Int`.

Sorun tür çıkarımıdır. Bir yere “sizeOf” yazarsam, çıkarım algoritması yalnızca bir “Int” beklediğimi bilir. “a” yerine hangi türü koymak istediğime dair hiçbir fikri yok. Bu nedenle, {-# LANGUAGE AllowAmbigulousTypes #-} uzantısı etkinleştirilmediği sürece, tanım derleyici tarafından reddedilir. Bu durumda tanım derlenir, belirsizlik hatası olmadan hiçbir yerde kullanılamaz.

Neyse ki, tip uygulamalarının tanıtılması günü kurtarıyor! Şimdi ‘a’nın ‘Int’ olduğunu açıkça söyleyerek ‘sizeOf @Int’ yazabiliriz. Tür uygulamaları, işlevin _gerçek parametrelerinde görünmese bile bir tür parametresi sağlamamıza izin verir!
