SQL Server’da prosedür, içine parametreleri aktarabileceğiniz depolanmış bir programdır. Fonksiyonun yaptığı gibi bir değer döndürmez. Ancak, kendisini çağıran prosedüre bir başarı/başarısızlık durumu döndürebilir.

Sözdizimi #

  • OLUŞTUR { PROSEDÜR | PROC } [şema_adı.]prosedür_adı
  • [ @parametre [type_schema_name.] veri türü
  • [ DEĞİŞEN ] [ = varsayılan ] [ ÇIKIŞ | ÇIKTI | SADECE OKU ]
  • , @parametre [type_schema_name.] veri türü
  • [ DEĞİŞEN ] [ = varsayılan ] [ ÇIKIŞ | ÇIKTI | SADECE OKU ] ]
  • [ { ŞİFRELEME İLE | YENİDEN DERLEME | EXECUTE AS Maddesi } ]
  • [ ÇOĞALTMA İÇİN ]
  • OLARAK
  • BAŞLAMAK
  • [bildirim_bölümü]
  • yürütülebilir_bölüm
  • SON;

Temel bir saklı yordam oluşturma ve yürütme #

Kitaplık Veritabanında Yazarlar tablosunu kullanma

CREATE PROCEDURE GetName
(
    @input_id INT = NULL,      --Input parameter,  id of the person, NULL default
    @name VARCHAR(128) = NULL  --Input parameter, name of the person, NULL default
) 
AS
BEGIN 
    SELECT Name + ' is from ' + Country 
    FROM Authors 
    WHERE Id = @input_id OR Name = @name
END 
GO

Birkaç farklı sözdizimi ile bir prosedür yürütebilirsiniz. İlk olarak, ‘EXECUTE’ veya ‘EXEC’ kullanabilirsiniz.

EXECUTE GetName @id = 1
EXEC Getname @name = 'Ernest Hemingway'

Ek olarak, EXEC komutunu atlayabilirsiniz. Ayrıca tüm parametreleri ilettiğiniz için hangi parametreyi ilettiğinizi belirtmek zorunda da değilsiniz.

GetName NULL, 'Ernest Hemingway'

Giriş parametrelerini prosedürde bildirildiğinden farklı bir sırada belirtmek istediğinizde, parametre adını belirtebilir ve değerler atayabilirsiniz. Örneğin

 CREATE PROCEDURE dbo.sProcTemp 
 (
    @Param1 INT,
    @Param2 INT
)
AS
BEGIN

    SELECT
        Param1 = @Param1,
        Param2 = @Param2

END

Bu prosedürü yürütmek için normal sıra, önce @Param1 ve sonra @Param2 saniye için değer belirtmektir. Yani böyle bir şey görünecek

  EXEC dbo.sProcTemp @Param1 = 0,@Param2=1

Ancak aşağıdakileri kullanmanız da mümkündür

  EXEC dbo.sProcTemp @Param2 = 0,@Param1=1

bunda, önce @param2 ve @Param1 saniye için değer belirtiyorsunuz. Yani prosedürde belirtildiği gibi aynı sırayı tutmak zorunda değilsiniz, ancak istediğiniz gibi sipariş verebilirsiniz. ancak değeri hangi parametreye ayarladığınızı belirtmeniz gerekecektir.

Herhangi bir veritabanından saklı yordama erişin

Ayrıca ‘sp_’ öneki ile bir prosedür oluşturabilirsiniz, bu procuedres, tüm sistem saklı prosedürleri gibi, SQL Server’ın varsayılan davranışı nedeniyle veritabanını belirtmeden yürütülebilir. “sp_” ile başlayan bir saklı yordamı yürüttüğünüzde, SQL Server yordamı önce ana veritabanında arar. Prosedür master’da bulunamazsa, aktif veritabanına bakar. Tüm veritabanlarınızdan erişmek istediğiniz saklı bir prosedürünüz varsa, onu master’da oluşturun ve “sp_” önekini içeren bir ad kullanın.

Use Master

CREATE PROCEDURE sp_GetName
(
    @input_id INT = NULL,      --Input parameter,  id of the person, NULL default
    @name VARCHAR(128) = NULL  --Input parameter, name of the person, NULL default
) 
AS
BEGIN 
    SELECT Name + ' is from ' + Country 
    FROM Authors 
    WHERE Id = @input_id OR Name = @name
END 
GO

If…Else ve Insert Into ile Saklı Yordam #

Örnek tablo Çalışan oluşturun:

CREATE TABLE Employee
(
    Id INT,
    EmpName VARCHAR(25),
    EmpGender VARCHAR(6),
    EmpDeptId INT
)

Saklı yordamda iletilen değerlerin boş olup olmadığını kontrol eden ve Çalışan tablosunda ekleme işlemi gerçekleştiren saklı yordam oluşturur.

CREATE PROCEDURE spSetEmployeeDetails
(
    @ID int,
    @Name VARCHAR(25),
    @Gender VARCHAR(6),
    @DeptId INT
)
AS
BEGIN
    IF (
        (@ID IS NOT NULL AND LEN(@ID) !=0) 
        AND (@Name IS NOT NULL AND LEN(@Name) !=0)
        AND (@Gender IS NOT NULL AND LEN(@Gender) !=0)
        AND (@DeptId IS NOT NULL AND LEN(@DeptId) !=0)
    )
    BEGIN
        INSERT INTO Employee
        (
            Id,
            EmpName,
            EmpGender,
            EmpDeptId
        )
        VALUES
        (
            @ID,
            @Name,
            @Gender,
            @DeptId
        )
    END
ELSE
    PRINT 'Incorrect Parameters'
END
GO

Saklı yordamı yürütün

DECLARE @ID INT,
    @Name VARCHAR(25),
    @Gender VARCHAR(6),
    @DeptId INT

EXECUTE spSetEmployeeDetails
    @ID = 1,
    @Name = 'Subin Nepal',
    @Gender = 'Male',
    @DeptId = 182666

Saklı yordamda dinamik SQL #

Dinamik SQL, çalışma zamanında SQL ifadeleri oluşturmamızı ve çalıştırmamızı sağlar. SQL ifadelerimiz farklı derleme zamanlarında değişebilen tanımlayıcı içerdiğinde Dinamik SQL gereklidir.

Dinamik SQL’in Basit Örneği:

CREATE PROC sp_dynamicSQL
@table_name      NVARCHAR(20),
@col_name        NVARCHAR(20), 
@col_value       NVARCHAR(20) 
AS
BEGIN
DECLARE  @Query  NVARCHAR(max)
SET      @Query = 'SELECT * FROM ' + @table_name
SET      @Query = @Query + ' WHERE ' + @col_name + ' = ' + ''''+@col_value+''''
EXEC     (@Query)
END

Yukarıdaki sql sorgusunda, çalışma anında @table_name, @col_name ve @col_value içindeki değerleri tanımlayarak yukarıdaki sorguyu kullanabileceğimizi görebiliriz. Sorgu, çalışma zamanında oluşturulur ve yürütülür. Bu, tüm komut dosyalarını bir değişkende dize olarak oluşturabileceğimiz ve çalıştırabildiğimiz tekniktir. Dinamik SQL ve birleştirme konseptini kullanarak daha karmaşık sorgular oluşturabiliriz. Bu konsept, birkaç koşul altında kullanılabilecek bir komut dosyası oluşturmak istediğinizde çok güçlüdür.

Saklı yordam yürütme

DECLARE @table_name     NVARCHAR(20) = 'ITCompanyInNepal',
        @col_name       NVARCHAR(20) = 'Headquarter',
        @col_value      NVARCHAR(20) = 'USA'
    
EXEC    sp_dynamicSQL   @table_name,    
                        @col_name,
                        @col_value

OUT parametreleriyle DEPOLANMIŞ PROSEDÜR #

Saklı prosedürler, parametre listesinde OUTPUT anahtar sözcüğünü kullanarak değerleri döndürebilir.

Tek bir çıkış parametresi ile saklı yordam oluşturma #

CREATE PROCEDURE SprocWithOutParams
(
    @InParam VARCHAR(30),
    @OutParam VARCHAR(30) OUTPUT
)
AS
BEGIN
    SELECT @OutParam = @InParam + ' must come out'   
    RETURN
END   
GO

Saklı yordamın yürütülmesi #

DECLARE @OutParam VARCHAR(30)    
EXECUTE SprocWithOutParams 'what goes in', @OutParam OUTPUT   
PRINT @OutParam

Birden çok çıkış parametresi ile saklı yordam oluşturma #

CREATE PROCEDURE SprocWithOutParams2
(
    @InParam VARCHAR(30),
    @OutParam VARCHAR(30) OUTPUT,
    @OutParam2 VARCHAR(30) OUTPUT
)
AS
BEGIN
    SELECT @OutParam = @InParam +' must come out'   
    SELECT @OutParam2 = @InParam +' must come out'
    RETURN
END    
GO

Saklı yordamın yürütülmesi #

DECLARE @OutParam VARCHAR(30)    
DECLARE @OutParam2 VARCHAR(30)  
EXECUTE SprocWithOutParams2 'what goes in', @OutParam OUTPUT, @OutParam2 OUTPUT   
PRINT @OutParam 
PRINT @OutParam2

Basit Döngü #

Önce bazı verileri “#systables” adlı geçici bir tabloya alalım ve her seferinde bir kaydı sorgulayabilmemiz için artan bir satır numarası verelim

select
    o.name,
    row_number() over (order by o.name) as rn
into
    #systables
from
    sys.objects as o
where
    o.type = 'S'

Daha sonra bu örnekte döngüyü kontrol etmek ve tablo adını saklamak için bazı değişkenler tanımlıyoruz.

declare
    @rn int = 1,
    @maxRn int = (
                    select
                        max(rn)
                    from
                        #systables as s
                    )
declare    @tablename sys name

Şimdi basit bir while kullanarak döngü yapabiliriz. “select” ifadesinde “@rn"yi artırdık, ancak bu, eski “set @rn = @rn + 1” için ayrı bir ifade de olabilirdi, gereksinimlerinize bağlı olacaktır. Ayrıca, “#systables"dan tek bir kayıt seçmek için artırılmadan önce “@rn” değerini kullanırız. Son olarak tablo adını yazdırıyoruz.

while @rn <= @maxRn
    begin

        select
            @tablename = name,
            @rn = @rn + 1
        from
            #systables as s
        where
            s.rn = @rn

        print @tablename
    end

CREATE PROCEDURE SprocWithSimpleLoop
(
    @SayThis VARCHAR(30),
    @ThisManyTimes INT
)
AS
BEGIN
    WHILE @ThisManyTimes > 0
    BEGIN
        PRINT @SayThis;
        SET @ThisManyTimes = @ThisManyTimes - 1;
    END
    
    RETURN;
END    
GO
