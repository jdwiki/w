FORMAT kullanarak Tarih ve Saat Biçimlendirme #

Yeni işlevi kullanabilirsiniz: FORMAT().

Bunu kullanarak “DATETIME” alanlarınızı kendi özel “VARCHAR” biçiminize dönüştürebilirsiniz.

Örnek

DECLARE @Date DATETIME = '2016-09-05 00:01:02.333'

SELECT FORMAT(@Date, N'dddd, MMMM dd, yyyy hh:mm:ss tt')

05 Eylül 2016 Pazartesi 12:01:02

Argümanlar

“DATETIME” biçiminin “2016-09-05 00:01:02.333” olduğu göz önüne alındığında, aşağıdaki grafik, sağlanan bağımsız değişken için çıktılarının ne olacağını gösterir.

HH 00

Ayrıca önceden biçimlendirilmiş bir çıktı oluşturmak için “FORMAT()” işlevine tek bir bağımsız değişken de sağlayabilirsiniz:

DECLARE @Date DATETIME = '2016-09-05 00:01:02.333'

SELECT FORMAT(@Date, N'U')

05 Eylül 2016 Pazartesi 04:01:02

Not: Yukarıdaki listede “en-US” kültürü kullanılmaktadır. Üçüncü parametre aracılığıyla ‘FORMAT()’ için farklı bir kültür belirtilebilir:

DECLARE @Date DATETIME = '2016-09-05 00:01:02.333'

SELECT FORMAT(@Date, N'U', 'zh-cn')

5 Eylül 2016 4:01:02

DATEADD zaman periyotlarını eklemek ve çıkarmak için #

Genel sözdizimi:

DATEADD (datepart , number , datetime_expr)

Bir zaman ölçüsü eklemek için “sayı” pozitif olmalıdır. Bir zaman ölçüsünü çıkarmak için “sayı” negatif olmalıdır.

Örnekler

DECLARE @now DATETIME2 = GETDATE();
SELECT @now;                        --2016-07-21 14:39:46.4170000
SELECT DATEADD(YEAR, 1, @now)       --2017-07-21 14:39:46.4170000

Geçerli DateTime’ı al #

Yerleşik işlevlerin GETDATE ve GETUTCDATE her biri, bir saat dilimi farkı olmaksızın geçerli tarih ve saati döndürür.

Her iki işlevin de dönüş değeri, SQL Server örneğinin çalıştığı bilgisayarın işletim sistemine bağlıdır.

GETDATE’in dönüş değeri, işletim sistemiyle aynı saat diliminde geçerli saati temsil eder. GETUTCDATE dönüş değeri, geçerli UTC zamanını temsil eder.

Her iki işlev de bir sorgunun ‘SELECT’ yan tümcesine veya ‘WHERE’ yan tümcesindeki boolean ifadesinin bir parçası olarak dahil edilebilir.

Örnekler:

-- example query that selects the current time in both the server time zone and UTC
SELECT GETDATE() as SystemDateTime, GETUTCDATE() as UTCDateTime

-- example query records with EventDate in the past.
SELECT * FROM MyEvents WHERE EventDate < GETDATE()

Geçerli tarih-saatin farklı varyasyonlarını döndüren birkaç yerleşik işlev daha vardır:

SELECT 
    GETDATE(),          --2016-07-21 14:27:37.447
    GETUTCDATE(),       --2016-07-21 18:27:37.447
    CURRENT_TIMESTAMP,  --2016-07-21 14:27:37.447
    SYSDATETIME(),      --2016-07-21 14:27:37.4485768
    SYSDATETIMEOFFSET(),--2016-07-21 14:27:37.4485768 -04:00
    SYSUTCDATETIME()    --2016-07-21 18:27:37.4485768

Bir ayın son gününü almak #

DATEADD ve DATEDIFF fonksiyonlarını kullanarak bir ayın son tarihini döndürmek mümkündür.

SELECT DATEADD(d, -1, DATEADD(m, DATEDIFF(m, 0, '2016-09-23') + 1, 0))
-- 2016-09-30 00:00:00.000

‘EOMONTH’ işlevi, bir ayın son tarihini döndürmek için daha kısa bir yol sağlar ve ayı dengelemek için isteğe bağlı bir parametreye sahiptir.

SELECT EOMONTH('2016-07-21')        --2016-07-31
SELECT EOMONTH('2016-07-21', 4)     --2016-11-30
SELECT EOMONTH('2016-07-21', -5)    --2016-02-29

Belirli bir tarihte bir kişinin yaşını hesaplamak için işlev oluşturun #

Bu işlev, 2 tarih saat parametresi, DOB ve yaşı kontrol etmek için bir tarih alacaktır.

  CREATE FUNCTION [dbo].[Calc_Age]
    (
    @DOB datetime , @calcDate datetime 
    )
    RETURNS int
    AS
   BEGIN
declare @age int

IF (@calcDate < @DOB  )
RETURN -1

-- If a DOB is supplied after the comparison date, then return -1
SELECT @age = YEAR(@calcDate) - YEAR(@DOB) +
  CASE WHEN DATEADD(year,YEAR(@calcDate) - YEAR(@DOB)
  ,@DOB) > @calcDate THEN -1 ELSE 0 END
    

RETURN @age
    

END

örneğin 1/1/2000 tarihinde doğan birinin bugünkü yaşını kontrol etmek için

SELECT  dbo.Calc_Age('2000-01-01',Getdate())

DATEDIFF zaman aralığı farklarını hesaplamak için #

Genel sözdizimi:

DATEDIFF (datepart, datetime_expr1, datetime_expr2)

“datetime_expr”, “datetime_expr2"ye göre geçmişteyse pozitif bir sayı, aksi takdirde negatif bir sayı döndürür.

Örnekler

DECLARE @now DATETIME2 = GETDATE();
DECLARE @oneYearAgo DATETIME2 = DATEADD(YEAR, -1, @now);
SELECT @now                                    --2016-07-21 14:49:50.9800000
SELECT @oneYearAgo                             --2015-07-21 14:49:50.9800000
SELECT DATEDIFF(YEAR, @oneYearAgo, @now)       --1
SELECT DATEDIFF(QUARTER, @oneYearAgo, @now)    --4
SELECT DATEDIFF(WEEK, @oneYearAgo, @now)       --52
SELECT DATEDIFF(DAY, @oneYearAgo, @now)        --366
SELECT DATEDIFF(HOUR, @oneYearAgo, @now)       --8784
SELECT DATEDIFF(MINUTE, @oneYearAgo, @now)     --527040
SELECT DATEDIFF(SECOND, @oneYearAgo, @now)     --31622400

NOT: “DATEDIFF”, “datepart” parametresindeki kısaltmaları da kabul eder. Bu kısaltmaların kullanılması genellikle kafa karıştırıcı olabileceğinden önerilmez (“m” ile “mi”, “ww” ile “w” vb.).

“DATEDIFF”, UTC ile SQL Server’ın yerel saati arasındaki farkı belirlemek için de kullanılabilir. Aşağıdaki ifade, UTC ile yerel saat (saat dilimi dahil) arasındaki farkı hesaplamak için kullanılabilir.

select  DATEDIFF(hh, getutcdate(), getdate()) as 'CentralTimeOffset'

DATEPART & DATENAME #

“DATEPART”, belirtilen tarihsaat ifadesinin belirtilen “tarih bölümü"nü sayısal bir değer olarak döndürür.

“DATENAME”, belirtilen tarihin belirtilen “tarih bölümünü” temsil eden bir karakter dizesi döndürür. Uygulamada DATENAME çoğunlukla ayın veya haftanın gününün adını almak için kullanışlıdır.

Ayrıca, ilgili “datepart” birimleriyle “DATEPART” gibi davranan bir datetime ifadesinin yılını, ayını veya gününü almak için bazı kısayol işlevleri vardır.

Sözdizimi:

DATEPART ( datepart , datetime_expr )
DATENAME ( datepart , datetime_expr )
DAY ( datetime_expr )
MONTH ( datetime_expr )
YEAR ( datetime_expr )

Örnekler:

DECLARE @now DATETIME2 = GETDATE();
SELECT @now                       --2016-07-21 15:05:33.8370000
SELECT DATEPART(YEAR, @now)       --2016
SELECT DATEPART(QUARTER, @now)    --3
SELECT DATEPART(WEEK, @now)       --30
SELECT DATEPART(HOUR, @now)       --15
SELECT DATEPART(MINUTE, @now)     --5
SELECT DATEPART(SECOND, @now)     --33
-- Differences between DATEPART and DATENAME:
SELECT DATEPART(MONTH, @now)      --7
SELECT DATENAME(MONTH, @now)      --July
SELECT DATEPART(WEEKDAY, @now)    --5
SELECT DATENAME(WEEKDAY, @now)    --Thursday
--shorthand functions
SELECT DAY(@now)    --21
SELECT MONTH(@now)  --7
SELECT YEAR(@now)   --2016

NOT: “DATEPART” ve “DATENAME”, “datepart” parametresindeki kısaltmaları da kabul eder. Bu kısaltmaların kullanılması genellikle kafa karıştırıcı olabileceğinden önerilmez (“m” ile “mi”, “ww” ile “w” vb.).

DateTime’dan sadece Date döndür #

DateTime nesnesinden bir Date döndürmenin birçok yolu vardır.

  1. SEÇ DÖNÜŞTÜRME(Tarih, GETDATE())
  2. DATEADD(gg, 0, DATEDIFF(dd, 0, GETDATE())), 2016-07-21 00:00:00.000 değerini döndürür
  3. CAST(GETDATE() TARİH OLARAK) SEÇİN
  4. ÇEVİRİ SEÇ(CHAR(10),GETDATE(),111)
  5. FORMATI SEÇİN(GERİ TARİHİ(), 'yyyy-AA-gg')
  1. ve 5. seçeneklerin bir tarih değil bir dize döndürdüğünü unutmayın.

ÇAPRAZ PLATFORM TARİH OBJESİ #

Transact SQL’de, aşağıdaki gibi ‘DATEFROMPARTS’ (veya ‘DATETIMEFROMPARTS’) işlevini kullanarak bir nesneyi ‘Date’ (veya ‘DateTime’) olarak tanımlayabilirsiniz:

 DECLARE @myDate DATE=DATEFROMPARTS(1988,11,28) 
 DECLARE @someMoment DATETIME=DATEFROMPARTS(1988,11,28,10,30,50,123)

Sağladığınız parametreler, ‘DATEFROMPARTS’ işlevi için Yıl, Ay, Gün’dür ve ‘DATETIMEFROMPARTS’ işlevi için yıl, ay, gün, saat, dakika, saniye ve milisaniye sağlamanız gerekir.

Bu yöntemler kullanışlıdır ve kullanılmaya değerdir çünkü bir tarih (veya tarih saat) oluşturmak için düz dize kullanmak, ana makine bölgesi, konum veya tarih biçimi ayarlarına bağlı olarak başarısız olabilir.
