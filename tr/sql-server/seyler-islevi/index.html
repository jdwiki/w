Birden Çok Satırdan Değerleri Birleştirmek için FOR XML Kullanma #

‘FOR XML’ işlevinin yaygın bir kullanımı, birden çok satırın değerlerini birleştirmektir.

İşte Müşteriler tablosu kullanan bir örnek:

SELECT 
    STUFF( (SELECT ';' + Email
        FROM Customers 
        where (Email is not null and Email <> '')
        ORDER BY Email ASC 
        FOR XML PATH('')), 
    1, 1, '')

Yukarıdaki örnekte, e-posta adreslerini birleştirmek için FOR XML PATH('')), sınırlayıcı karakter olarak ; kullanılarak kullanılmaktadır. Ayrıca, “STUFF"un amacı, birleştirilmiş dizeden baştaki “;” işaretini kaldırmaktır. “STUFF” ayrıca, birleştirilmiş dizeyi XML’den varchar’a örtük olarak yayınlıyor.

Not: Yukarıdaki örneğin sonucu XML kodlu olacaktır, yani < karakterlerini &lt; vb. ile değiştirecektir. Bunu istemiyorsanız, FOR XML PATH('')) öğesini olarak değiştirin. XML YOLU İÇİN, TYPE).value('.[1]','varchar(MAX)'), ör.:

SELECT 
    STUFF( (SELECT ';' + Email
        FROM Customers 
        where (Email is not null and Email <> '')
        ORDER BY Email ASC 
        FOR XML PATH, TYPE).value('.[1]','varchar(900)'),
    1, 1, '')

Bu, GROUP BY kümeleri yerine alt sorgular kullanmamıza rağmen, MySQL’deki “GROUP_CONCAT” veya PostgreSQL 9.0+‘daki “string_agg” ile benzer bir sonuç elde etmek için kullanılabilir. (Alternatif olarak, “GROUP_CONCAT” işlevine daha yakın bir işlev arıyorsanız bu gibi kullanıcı tanımlı bir küme yükleyebilirsiniz.

STUFF() ile Temel Karakter Değiştirme #

STUFF() işlevi, önce belirtilen sayıda karakteri silerek bir dizeyi başka bir dizeye ekler. Aşağıdaki örnek, “Svr"yi siler ve onu “Sunucu” ile değiştirir. Bu, değiştirmenin “başlangıç_konumu” ve “uzunluğu” belirtilerek gerçekleşir.

SELECT STUFF('SQL Svr Documentation', 5, 3, 'Server')

Bu örneğin yürütülmesi, ‘SQL Svr Dokümantasyonu’ yerine ‘SQL Server Dokümantasyonu’ döndürülmesine neden olacaktır.

STUFF() işlevinin Temel Örneği. #

STUFF(Orijinal_İfade, Başlangıç, Uzunluk, Değiştirme_ifadesi)

STUFF() işlevi, Uzunluk parametresi kullanılarak belirtilen karakterleri kaldırmanın yanı sıra belirtilen başlangıç ​​konumuna Değiştirme_ifadesini ekler.

 Select FirstName, LastName,Email, STUFF(Email, 2, 3, '*****') as StuffedEmail From Employee

Bu örneğin yürütülmesi, verilen tablonun döndürülmesine neden olacaktır

, Shyam ratod Shyam@hotmail.com S*****m@hotmail.com
Koç shinde Ram@hotmail.com *****hotmail.com

Virgülle ayrılmış sütun adlarını alın (liste değil) #

/*
The result can be use for fast way to use columns on Insertion/Updates.
Works with tables and views.

Example: eTableColumns  'Customers'
ColumnNames
------------------------------------------------------
Id, FName, LName, Email, PhoneNumber, PreferredContact

INSERT INTO Customers (Id, FName, LName, Email, PhoneNumber, PreferredContact)
    VALUES (5, 'Ringo', 'Star', 'two@beatles.now', NULL, 'EMAIL')
*/
CREATE PROCEDURE eTableColumns (@Table VARCHAR(100))
AS
SELECT ColumnNames = 
   STUFF( (SELECT ', ' +  c.name
FROM    
    sys.columns c
INNER JOIN 
    sys.types t ON c.user_type_id = t.user_type_id
WHERE
    c.object_id = OBJECT_ID( @Table)
        FOR XML PATH, TYPE).value('.[1]','varchar(2000)'),
    1, 1, '')
GO

sql sunucusunda virgülle ayrılmış öğeler #

Birden çok satırı tek bir satırda birleştirmek için “FOR XML PATH” ve “STUFF”:

  select distinct t1.id,
      STUFF(
             (SELECT ', ' + convert(varchar(10), t2.date, 120)
              FROM yourtable t2
              where t1.id = t2.id
              FOR XML PATH (''))
              , 1, 1, '')  AS date
    from yourtable t1;
