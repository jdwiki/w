Diğer ilişkisel veritabanı sistemlerinde olduğu gibi, SQL Server da veritabanlarınız hakkında meta verileri ortaya çıkarır.

Bu, ISO Standardı “INFORMATION_SCHEMA” şeması veya SQL Server’a özgü “sys” katalog görünümleri aracılığıyla sağlanır.

Tüm Saklı Prosedürlerin Listesini Alın #

Aşağıdaki sorgular, her Saklı Yordam hakkında temel bilgilerle birlikte veritabanındaki tüm Saklı Yordamların bir listesini döndürür:

SELECT *
FROM INFORMATION_SCHEMA.ROUTINES
WHERE ROUTINE_TYPE = 'PROCEDURE'

“ROUTINE_NAME”, “ROUTINE_SCHEMA” ve “ROUTINE_DEFINITION” sütunları genellikle en kullanışlı olanlardır.

son sürüm 
SELECT *
FROM sys.objects
WHERE type = 'P'
son sürüm 
SELECT *
FROM sys.procedures

Bu sürümün, ‘is_auto_executed’, ‘is_execution_replicated’, ‘is_repl_serializable’ ve ‘skips_repl_constraints’ ek sütunlarını içerdiğinden, sys.objects’ten seçmeye göre bir avantajı olduğunu unutmayın.

son sürüm 
SELECT *
FROM sysobjects
WHERE type = 'P'

Çıktının hiçbir zaman saklı bir yordamla ilgili olmayacak birçok sütun içerdiğini unutmayın.

son sürüm

Sonraki sorgu grubu, veritabanındaki ‘SearchTerm’ dizesini içeren tüm Saklı Yordamları döndürür:

SELECT o.name
FROM syscomments c
INNER JOIN sysobjects o
    ON c.id=o.id
WHERE o.xtype = 'P'
    AND c.TEXT LIKE '%SearchTerm%'
son sürüm 
SELECT p.name
FROM sys.sql_modules AS m
INNER JOIN sys.procedures AS p
    ON m.object_id = p.object_id
WHERE definition LIKE '%SearchTerm%'
son sürüm

Bir sunucudaki tüm veritabanlarının listesini alın #

Yöntem 1: Aşağıdaki sorgu SQL Server 2000+ sürümü için geçerli olacaktır (12 sütun içerir)

SELECT * FROM dbo.sysdatabases

Yöntem 2: Aşağıdaki sorgu, daha fazla bilgi içeren veritabanları hakkındaki bilgileri (ör: Durum, İzolasyon, kurtarma modeli vb.)

Not: Bu bir katalog görünümüdür ve SQL SERVER 2005+ sürümleri mevcut olacaktır.

SELECT * FROM sys.databases

Yöntem 3: Yalnızca veritabanı adlarını görmek için belgelenmemiş sp_MSForEachDB kullanabilirsiniz

EXEC sp_MSForEachDB 'SELECT ''?'' AS DatabaseName'

Yöntem 4: SP’nin altında, sunucudaki veritabanlarının adı, sahibi, durumu vb. ile birlikte veritabanı boyutunu sağlamanıza yardımcı olacaktır.

EXEC sp_helpdb

Yöntem 5 Benzer şekilde, aşağıdaki saklı yordam veritabanı adını, veritabanı boyutunu ve Açıklamaları verecektir.

EXEC sp_databases

Veritabanındaki Tablo Sayısını Sayma #

Bu sorgu, belirtilen veritabanındaki tablo sayısını döndürür.

USE YourDatabaseName
SELECT COUNT(*) from INFORMATION_SCHEMA.TABLES
WHERE TABLE_TYPE = 'BASE TABLE'

SQL Server 2008+ ile tüm kullanıcı tabloları için bunu yapmanın başka bir yolu aşağıdadır. Referans burada şeklindedir.

SELECT COUNT(*) FROM sys.tables

Veritabanı Dosyaları #

Tüm veritabanları için tüm veri dosyalarını boyut ve büyüme bilgileriyle görüntüleyin

SELECT  d.name AS 'Database',
        d.database_id,  
        SF.fileid,
        SF.name AS 'LogicalFileName',   
        CASE SF.status & 0x100000  
            WHEN 1048576 THEN 'Percentage' 
            WHEN 0 THEN 'MB' 
        END AS 'FileGrowthOption',
        Growth AS GrowthUnit,
        ROUND(((CAST(Size AS FLOAT)*8)/1024)/1024,2) [SizeGB], -- Convert 8k pages to GB
        Maxsize,        
        filename AS PhysicalFileName

FROM    Master.SYS.SYSALTFILES SF
Join    Master.SYS.Databases d on sf.fileid = d.database_id

Order by d.name

Windows Oturum Açma İzin Yolunu Belirleyin #

Bu, kullanıcı türünü ve izin yolunu (kullanıcının izinlerini hangi pencere grubundan aldığını) gösterecektir.

xp_logininfo 'DOMAIN\user'

Enterprise’a özgü özelliklerin kullanılıp kullanılmadığını görün #

Geliştirici sürümündeki çalışmanızın Enterprise sürümüyle sınırlı herhangi bir özelliğe bağımlılık getirmediğini doğrulamak bazen yararlıdır.

Bunu, “sys.dm_db_persisted_sku_features” sistem görünümünü kullanarak şu şekilde yapabilirsiniz:

SELECT * FROM sys.dm_db_persisted_sku_features

Veritabanının kendisine karşı.

Bu, varsa, kullanılan özellikleri listeleyecektir.

Veritabanı Seçeneklerini Al #

Aşağıdaki sorgu, veritabanı seçeneklerini ve meta verileri döndürür:

select * from sys.databases WHERE name = 'MyDatabaseName';

Mevcut Veritabanındaki Tüm Tabloların Boyutunu Göster #

SELECT
    s.name + '.' + t.NAME AS TableName,
    SUM(a.used_pages)*8 AS 'TableSizeKB'  --a page in SQL Server is 8kb
FROM sys.tables t
    JOIN sys.schemas s on t.schema_id = s.schema_id
    LEFT JOIN sys.indexes i ON t.OBJECT_ID = i.object_id
    LEFT JOIN sys.partitions p ON i.object_id = p.OBJECT_ID AND i.index_id = p.index_id
    LEFT JOIN sys.allocation_units a ON p.partition_id = a.container_id
GROUP BY
    s.name, t.name
ORDER BY
    --Either sort by name:
    s.name + '.' + t.NAME
    --Or sort largest to smallest:
    --SUM(a.used_pages) desc

Bilinen Sütunu İçeren Tabloları Al #

Bu sorgu, belirli bir sütun adı için tüm “SÜTUNLAR"ı ve bunlarla ilişkili “TABLOLAR"ı döndürür. Hangi tabloların (bilinmeyen) belirli bir sütun içerdiğini (bilinen) göstermek için tasarlanmıştır.

SELECT 
    c.name AS ColName, 
    t.name AS TableName
FROM

Belirtilen Sütun Değerini İçeren Tüm Tabloları ve Sütunları Ara ve Döndür #

buradan ve burada‘den gelen bu komut dosyası, belirtilen bir değerin bulunduğu tüm Tabloları ve Sütunları döndürür. Bu, belirli bir değerin bir veritabanında nerede olduğunu bulmada güçlüdür. Vergilendirilebilir, bu nedenle önce bir yedekleme/test ortamında yürütülmesi önerilir.

DECLARE @SearchStr nvarchar(100)
SET @SearchStr = '## YOUR STRING HERE ##'
 
 
    -- Copyright © 2002 Narayana Vyas Kondreddi. All rights reserved.
    -- Purpose: To search all columns of all tables for a given search string
    -- Written by: Narayana Vyas Kondreddi
    -- Site: http://vyaskn.tripod.com
    -- Updated and tested by Tim Gaunt
    -- http://www.thesitedoctor.co.uk
    -- http://blogs.thesitedoctor.co.uk/tim/2010/02/19/Search+Every+Table+And+Field+In+A+SQL+Server+Database+Updated.aspx
    -- Tested on: SQL Server 7.0, SQL Server 2000, SQL Server 2005 and SQL Server 2010
    -- Date modified: 03rd March 2011 19:00 GMT
    CREATE TABLE #Results (ColumnName nvarchar(370), ColumnValue nvarchar(3630))
 
    SET NOCOUNT ON
 
    DECLARE @TableName nvarchar(256), @ColumnName nvarchar(128), @SearchStr2 nvarchar(110)
    SET  @TableName = ''
    SET @SearchStr2 = QUOTENAME('%' + @SearchStr + '%','''')
 
    WHILE @TableName IS NOT NULL
     
    BEGIN
        SET @ColumnName = ''
        SET @TableName =
        (
            SELECT MIN(QUOTENAME(TABLE_SCHEMA) + '.' + QUOTENAME(TABLE_NAME))
            FROM     INFORMATION_SCHEMA.TABLES
            WHERE         TABLE_TYPE = 'BASE TABLE'
                AND    QUOTENAME(TABLE_SCHEMA) + '.' + QUOTENAME(TABLE_NAME) > @TableName
                AND    OBJECTPROPERTY(
                        OBJECT_ID(
                            QUOTENAME(TABLE_SCHEMA) + '.' + QUOTENAME(TABLE_NAME)
                             ), 'IsMSShipped'
                               ) = 0
        )
 
        WHILE (@TableName IS NOT NULL) AND (@ColumnName IS NOT NULL)
             
        BEGIN
            SET @ColumnName =
            (
                SELECT MIN(QUOTENAME(COLUMN_NAME))
                FROM     INFORMATION_SCHEMA.COLUMNS
                WHERE         TABLE_SCHEMA    = PARSENAME(@TableName, 2)
                    AND    TABLE_NAME    = PARSENAME(@TableName, 1)
                    AND    DATA_TYPE IN ('char', 'varchar', 'nchar', 'nvarchar', 'int', 'decimal')
                    AND    QUOTENAME(COLUMN_NAME) > @ColumnName
            )
     
            IF @ColumnName IS NOT NULL
             
            BEGIN
                INSERT INTO #Results
                EXEC
                (
                    'SELECT ''' + @TableName + '.' + @ColumnName + ''', LEFT(' + @ColumnName + ', 3630) FROM ' + @TableName + ' (NOLOCK) ' +
                    ' WHERE ' + @ColumnName + ' LIKE ' + @SearchStr2
                )
            END
        END   
    END
 
    SELECT ColumnName, ColumnValue FROM #Results
 
DROP TABLE #Results

Tüm şemaları, tabloları, sütunları ve dizinleri alın #

SELECT 
    s.name AS [schema],
    t.object_id AS [table_object_id], 
    t.name AS [table_name],
    c.column_id,
    c.name AS [column_name],
    i.name AS [index_name],
    i.type_desc AS [index_type]
FROM sys.schemas AS s
INNER JOIN sys.tables AS t 
    ON s.schema_id = t.schema_id
INNER JOIN sys.columns AS c 
    ON t.object_id = c.object_id
LEFT JOIN sys.index_columns AS ic 
    ON c.object_id = ic.object_id and c.column_id = ic.column_id
LEFT JOIN sys.indexes AS i 
    ON ic.object_id = i.object_id and ic.index_id = i.index_id
ORDER BY [schema], [table_name], c.column_id;

Yedekleme ve geri yükleme işlemleriyle ilgili bilgileri alın #

Geçerli veritabanı örneğinde gerçekleştirilen tüm yedekleme işlemlerinin listesini almak için:

SELECT sdb.Name AS DatabaseName,
    COALESCE(CONVERT(VARCHAR(50), bus.backup_finish_date, 120),'-') AS LastBackUpDateTime
FROM sys.sysdatabases sdb
    LEFT OUTER JOIN msdb.dbo.backupset bus ON bus.database_name = sdb.name
ORDER BY sdb.name, bus.backup_finish_date DESC

Geçerli veritabanı örneğinde gerçekleştirilen tüm geri yükleme işlemlerinin listesini almak için:

SELECT 
    [d].[name] AS database_name, 
    [r].restore_date AS last_restore_date, 
    [r].[user_name], 
    [bs].[backup_finish_date] AS backup_creation_date, 
    [bmf].[physical_device_name] AS [backup_file_used_for_restore] 
FROM master.sys.databases [d] 
    LEFT OUTER JOIN msdb.dbo.[restorehistory] r ON r.[destination_database_name] = d.Name 
    INNER JOIN msdb.dbo.backupset [bs] ON [r].[backup_set_id] = [bs].[backup_set_id] 
    INNER JOIN msdb.dbo.backupmediafamily bmf ON [bs].[media_set_id] = [bmf].[media_set_id] 
ORDER BY [d].[name], [r].restore_date DESC
