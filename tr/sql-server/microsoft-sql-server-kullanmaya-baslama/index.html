EKLE / SEÇ / GÜNCELLEME / SİL: Veri İşleme Dilinin temelleri #

Data Manipülasyon Language (kısaca DML) “INSERT”, “UPDATE” ve “DELETE” gibi işlemleri içerir:

-- Create a table HelloWorld

CREATE TABLE HelloWorld (
    Id INT IDENTITY,
    Description VARCHAR(1000)
)


-- DML Operation INSERT, inserting a row into the table
INSERT INTO HelloWorld (Description) VALUES ('Hello World')


-- DML Operation SELECT, displaying the table 
SELECT * FROM HelloWorld  


-- Select a specific column from table
SELECT Description FROM HelloWorld


-- Display number of records in the table
SELECT Count(*) FROM HelloWorld


-- DML Operation UPDATE, updating a specific row in the table
UPDATE HelloWorld SET Description = 'Hello, World!' WHERE Id = 1


-- Selecting rows from the table (see how the Description has changed after the update?)
SELECT * FROM HelloWorld


-- DML Operation - DELETE, deleting a row from the table
DELETE FROM HelloWorld WHERE Id = 1


-- Selecting the table. See table content after DELETE operation 
SELECT * FROM HelloWorld

Bu komut dosyasında, bazı temel sorguları göstermek için bir tablo oluşturuyoruz.

Aşağıdaki örnekler, tabloların nasıl sorgulanacağını gösterir:

USE Northwind;
GO
SELECT TOP 10 * FROM Customers 
ORDER BY CompanyName

“Müşteri” tablosunun, “Northwind” veritabanından “CompanyName” sütununa göre sıralanan ilk 10 kaydını seçecektir (Microsoft’un örnek veritabanlarından biridir, [buradan](https:// technet.microsoft.com/en-us/library/ms143221(v=sql.105).aspx)):

Use Northwind’in sonraki tüm sorgular için varsayılan veritabanını değiştirdiğine dikkat edin. Yine de [Veritabanı].[Şema].[Tablo] biçimindeki tam sözdizimini kullanarak veritabanına başvurabilirsiniz:

SELECT TOP 10 * FROM Northwind.dbo.Customers 
ORDER BY CompanyName

SELECT TOP 10 * FROM Pubs.dbo.Authors
ORDER BY City

Bu, farklı veritabanlarından veri sorguluyorsanız kullanışlıdır. “Arada” belirtilen “dbo"nun şema olarak adlandırıldığını ve tam sözdizimi kullanılırken belirtilmesi gerektiğini unutmayın. Bunu veritabanınızdaki bir klasör olarak düşünebilirsiniz. “dbo” varsayılan şemadır. Varsayılan şema atlanabilir. Diğer tüm kullanıcı tanımlı şemaların belirtilmesi gerekir.

Veritabanı tablosu, ayrılmış kelimeler gibi adlandırılan sütunlar içeriyorsa, ör. “Tarih”, sütun adını aşağıdaki gibi parantez içine almanız gerekir:

-- descending order
SELECT TOP 10 [Date] FROM dbo.MyLogTable
ORDER BY [Date] DESC

Aynı durum, sütun adı adında boşluklar içeriyorsa da geçerlidir (bu önerilmez). Alternatif bir sözdizimi, köşeli parantezler yerine çift tırnak kullanmaktır, örneğin:

-- descending order
SELECT top 10 "Date" from dbo.MyLogTable
order by "Date" desc

eşdeğerdir, ancak çok yaygın olarak kullanılmaz. Çift tırnak ve tek tırnak arasındaki farka dikkat edin: Dizeler için tek tırnak kullanılır, yani.

-- descending order
SELECT top 10 "Date" from dbo.MyLogTable
where UserId='johndoe'
order by "Date" desc

geçerli bir sözdizimidir. T-SQL’in NChar ve NVarchar veri türleri için bir “N” ön ekine sahip olduğuna dikkat edin, ör.

SELECT TOP 10 * FROM Northwind.dbo.Customers 
WHERE CompanyName LIKE N'AL%'
ORDER BY CompanyName

“AL” ile başlayan bir şirket adına sahip tüm şirketleri döndürür (”%" bir joker karakterdir, bir DOS komut satırında yıldız işaretini kullanacağınız gibi kullanın, örneğin “DIR AL*”). “LIKE” için birkaç joker karakter mevcuttur, daha fazla ayrıntı için buraya bakın.

Katılır #

Birleştirmeler, tek bir tabloda olmayan, ancak birden çok tabloda bulunan alanları sorgulamak istiyorsanız kullanışlıdır. Örneğin: ‘Northwind’ veritabanındaki ‘Region’ tablosundaki tüm sütunları sorgulamak istiyorsunuz. Ancak, farklı bir tablo olan ‘Bölge’de saklanan ‘Bölge Tanımı’na da ihtiyacınız olduğunu fark ettiniz. Bununla birlikte, bu bilgiyi aşağıdaki gibi tek bir sorguda birleştirmek için kullanabileceğiniz ortak bir “RgionID” anahtarı vardır (“İlk 5” yalnızca ilk 5 satırı döndürür, tüm satırları almak için atlayın):

SELECT TOP 5 Territories.*, 
    Regions.RegionDescription 
FROM Territories 
INNER JOIN Region 
    ON Territories.RegionID=Region.RegionID
ORDER BY TerritoryDescription

“Bölgeler"deki tüm sütunları ve “Bölge"deki “Bölge Tanımı” sütununu gösterir. Sonuç, ‘TerritoryDescription’ tarafından sıralanır.

Tablo Takma Adları #

Sorgunuz iki veya daha fazla tabloya başvuru gerektirdiğinde, Tablo Takma Adı kullanmayı yararlı bulabilirsiniz. Tablo takma adları, tam tablo adı yerine kullanılabilen ve yazmayı ve düzenlemeyi azaltabilen tablolara kısa referanslardır. Takma ad kullanmak için sözdizimi şöyledir:

<TableName> [as] <alias>

Burada “as” isteğe bağlı bir anahtar kelimedir. Örneğin, önceki sorgu şu şekilde yeniden yazılabilir:

SELECT TOP 5 t.*, 
    r.RegionDescription 
FROM Territories t
INNER JOIN Region r 
    ON t.RegionID = r.RegionID
ORDER BY TerritoryDescription

Aynı tabloyu iki kez kullansanız bile, sorgudaki tüm tablolar için takma adlar benzersiz olmalıdır. Örneğin, Çalışan tablonuz bir SupervisorId alanı içeriyorsa, bir çalışanı ve onun amirinin adını döndürmek için bu sorguyu kullanabilirsiniz:

SELECT e.*, 
    s.Name as SupervisorName -- Rename the field for output
FROM Employee e
INNER JOIN Employee s
    ON e.SupervisorId = s.EmployeeId
WHERE e.EmployeeId = 111

Birlikler #

Daha önce gördüğümüz gibi, Join, farklı tablo kaynaklarından sütunlar ekler. Peki ya farklı kaynaklardan gelen satırları birleştirmek isterseniz? Bu durumda bir UNION kullanabilirsiniz. Bir parti planladığınızı ve sadece çalışanları değil müşterileri de davet etmek istediğinizi varsayalım. Sonra bunu yapmak için bu sorguyu çalıştırabilirsiniz:

SELECT FirstName+' '+LastName as ContactName, Address, City FROM Employees
UNION
SELECT ContactName, Address, City FROM Customers

Çalışan ve müşterilerden isim, adres ve şehirleri tek bir tabloda döndürecektir. Yinelenen satırların (varsa) otomatik olarak ortadan kaldırıldığını unutmayın (bunu istemiyorsanız, bunun yerine bir ‘UNION ALL’ kullanın). Sütun numarası, sütun adları, sıra ve veri türü, birliğin parçası olan tüm seçme ifadeleri arasında eşleşmelidir - bu nedenle ilk SELECT, Çalışandan ‘FirstName’ ve ‘LastName’i ‘ContactName’ olarak birleştirir.

Tablo Değişkenleri #

Geçici verilerle (özellikle saklı bir prosedürde) uğraşmanız gerekiyorsa, tablo değişkenlerini kullanmak faydalı olabilir: “Gerçek” bir tablo ile bir tablo değişkeni arasındaki fark, yalnızca geçici işlem için bellekte bulunmasıdır.

Örnek:

DECLARE @Region TABLE
(
  RegionID int, 
  RegionDescription NChar(50)
)

bellekte bir tablo oluşturur. Bu durumda, bir değişken olduğu için @ öneki zorunludur. Satır eklemek, silmek ve seçmek için yukarıda bahsedilen tüm DML işlemlerini gerçekleştirebilirsiniz, örn.

INSERT INTO @Region values(3,'Northern')
INSERT INTO @Region values(4,'Southern')

Ama normalde, onu aşağıdaki gibi gerçek bir tabloya göre doldurursunuz:

INSERT INTO @Region
SELECT * FROM dbo.Region WHERE RegionID>2;

bu, ‘dbo.Region’ gerçek tablosundan filtrelenmiş değerleri okur ve daha sonraki işlemler için kullanılabileceği ‘@Region’ bellek tablosuna ekler. Örneğin, bir birleştirme gibi kullanabilirsiniz

SELECT * FROM Territories t
JOIN @Region r on t.RegionID=r.RegionID

ki bu durumda tüm ‘Kuzey’ ve ‘Güney’ bölgeleri geri dönecektir. Daha ayrıntılı bilgi buradan bulunabilir. Bu konu hakkında daha fazla bilgi edinmek istiyorsanız, geçici tablolar burada tartışılmaktadır. .

NOT: Microsoft, yalnızca tablo değişkenindeki veri satırlarının sayısı 100’den az olduğunda tablo değişkenlerinin kullanılmasını önerir. Daha fazla miktarda veriyle çalışacaksanız, bir geçici tablo kullanın, veya bunun yerine geçici tablo.

Bir tablodaki tüm satırları ve sütunları SEÇ #

Sözdizimi:

SELECT *
FROM table_name

Yıldız operatörünün * kullanılması, tablodaki tüm sütunları seçmek için bir kısayol görevi görür. Bu ‘SELECT’ deyiminde herhangi bir filtreleme kriteri belirtmek için ‘WHERE’ yan tümcesi bulunmadığından tüm satırlar da seçilecektir.

Bu, tabloya bir takma ad eklerseniz de aynı şekilde çalışır, örneğin bu durumda “e”:

SELECT *
FROM Employees AS e

Veya belirli bir tablodan tümünü seçmek isterseniz, + " .* " takma adını kullanabilirsiniz:

SELECT e.*, d.DepartmentName
FROM Employees AS e
    INNER JOIN Department AS d 
        ON e.DepartmentID = d.DepartmentID

Veritabanı nesnelerine tam nitelikli adlar kullanılarak da erişilebilir:

SELECT * FROM [server_name].[database_name].[schema_name].[table_name]

Sunucu ve/veya veritabanı adlarının değiştirilmesi, geçersiz nesne adları nedeniyle tam nitelikli adlar kullanan sorguların artık yürütülmemesine neden olacağından, bu mutlaka önerilmez.

Sorgular sırasıyla tek bir sunucuda, veritabanında ve şemada yürütülüyorsa, çoğu durumda “tablo_adı"ndan önceki alanların atlanabileceğini unutmayın. Ancak, bir veritabanının birden çok şemaya sahip olması yaygındır ve bu durumlarda, mümkün olduğunda şema adı atlanmamalıdır.

Uyarı: Üretim kodunda veya saklı yordamlarda ‘SELECT *’ kullanılması daha sonra (tabloya yeni sütunlar eklendikçe veya tabloda sütunlar yeniden düzenlenirse) sorunlara yol açabilir, özellikle kodunuz basit varsayımlarda bulunursa sütunların sırası veya döndürülen sütunların sayısı hakkında. Bu nedenle, üretim kodu için SELECT ifadelerinde sütun adlarını her zaman açıkça belirtmek daha güvenlidir.

SELECT col1, col2, col3
FROM table_name

GÜNCELLEME Belirli Satır #

UPDATE HelloWorlds
SET HelloWorld = 'HELLO WORLD!!!'
WHERE Id = 5

Yukarıdaki kod, “HelloWorld” alanının değerini “HELLO WORLD!!!” ile günceller. HelloWorlds tablosunda “Id = 5” olan kayıt için.

Not: Bir güncelleme bildiriminde, gereksiniminiz farklı olmadıkça ve tüm tabloyu güncellemekten kaçınmak için bir “nerede” yan tümcesi kullanmanız önerilir.

Koddaki yorumlar #

Transact-SQL, iki yorum yazma biçimini destekler. Yorumlar veritabanı motoru tarafından yok sayılır ve insanların okuması içindir.

Yorumlardan önce “–” gelir ve yeni bir satırla karşılaşılıncaya kadar yok sayılır:

-- This is a comment
SELECT *
FROM MyTable -- This is another comment
WHERE Id = 1;

Slash star yorumları /* ile başlar ve */ ile biter. Bu sınırlayıcılar arasındaki tüm metinler bir yorum bloğu olarak kabul edilir.

/* This is
a multi-line
comment block. */
SELECT Id = 1, [Message] = 'First row'
UNION ALL
SELECT 2, 'Second row'
/* This is a one liner */
SELECT 'More';

Eğik çizgi yıldız yorumları, SQL Deyimi yeni satır karakterlerini kaybederse yorumu kullanılabilir tutma avantajına sahiptir. Bu, sorun giderme sırasında SQL yakalandığında gerçekleşebilir.

Eğik çizgi yorumları iç içe yerleştirilebilir ve bir eğik çizgi yorumu içinde başlayan bir /* geçerli olması için bir */ ile sona ermelidir. Aşağıdaki kod bir hataya neden olacaktır

/*
SELECT *
FROM CommentTable
WHERE Comment = '/*'
*/

Alıntının içinde olmasına rağmen eğik çizgi yıldızı bir yorumun başlangıcı olarak kabul edilir. Bu nedenle, başka bir kapanış yıldız eğik çizgi ile sona ermesi gerekiyor. Doğru yol olurdu

/*
SELECT *
FROM CommentTable
WHERE Comment = '/*'
*/  */

Tüm Satırları SİL #

DELETE
FROM Helloworlds

Bu, tablodaki tüm verileri siler. Bu kodu çalıştırdıktan sonra tablo satır içermeyecektir. ‘DROP TABLE’dan farklı olarak bu, tablonun kendisini ve yapısını korur ve bu tabloya yeni satırlar eklemeye devam edebilirsiniz.

Tablodaki tüm satırları silmenin başka bir yolu da aşağıdaki gibi kısaltmaktır:

TRUNCATE TABLE HelloWords

DELETE işleminin birkaç farkı vardır:

  1. Kesme işlemi işlem günlüğü dosyasında saklanmaz
  2. “KİMLİK” alanı varsa, bu sıfırlanacaktır.
  3. TRUNCATE tüm tabloya uygulanabilir ve bir kısmına hayır uygulanabilir (DELETE komutu yerine bir WHERE yan tümcesini ilişkilendirebilirsiniz)

TRUNCATE Kısıtlamaları

  1. ‘YABANCI ANAHTAR’ referansı varsa tablo kesilemez
  2. Tablo bir DİZİNLENDİRİLMİŞ GÖRÜNÜMe katılmışsa
  3. Tablo, ‘İŞLEMSEL ÇOĞALTMA’ veya ‘MERGE ÇOĞALTMA’ kullanılarak yayınlanmışsa
  4. Tabloda tanımlanan hiçbir TRIGGER’ı tetiklemez.

[sic]

YAZDIR #

Çıkış konsoluna bir mesaj görüntüleyin. SQL Server Management Studio kullanıldığında bu, sonuçlar sekmesi yerine mesajlar sekmesinde görüntülenecektir:

PRINT 'Hello World!';

Bir koşulla eşleşen satırları seçin #

Genel olarak, sözdizimi şöyledir:

SELECT <column names>
FROM <table name>
WHERE <condition>

Örneğin:

SELECT FirstName, Age
FROM Users
WHERE LastName = 'Smith'

Koşullar karmaşık olabilir:

SELECT FirstName, Age
FROM Users
WHERE LastName = 'Smith' AND (City = 'New York' OR City = 'Los Angeles')

Tüm Satırları GÜNCELLE #

Basit bir güncelleme şekli, tablonun belirli bir alanındaki tüm değerleri arttırmaktır. Bunun için alanı ve artım değerini tanımlamamız gerekiyor.

Aşağıda, “Puan” alanını 1 (tüm satırlarda) artıran bir örnek verilmiştir:

UPDATE Scores
SET score = score + 1

Tablodaki Tüm satırlar için bir GÜNCELLEME ile belirli bir Satır için yanlışlıkla GÜNCELLEME yaparsanız verilerinizi bozabileceğiniz için bu tehlikeli olabilir.

Örnekte çalışan MS SQL Server sürümünü döndürür.

SELECT @@SERVERNAME

MS SQL Server örneğinin adını döndürür.

SELECT @@SERVICENAME

MS SQL Server’ın şu şekilde çalıştığı Windows hizmetinin adını döndürür.

SQL Server’ın çalıştığı makinenin fiziksel adını döndürür. Yük devretme kümesindeki düğümü tanımlamak için kullanışlıdır.

Yük devretme kümesinde, SQL Server’ın çalışabileceği her düğümü döndürür. Küme değilse hiçbir şey döndürmez.

KESME TABLOSU #

TRUNCATE TABLE Helloworlds

Bu kod, Helloworlds tablosundaki tüm verileri siler. Truncate tablosu, ‘Tablodan Sil’ koduna neredeyse benzer. Aradaki fark, Truncate ile where yan tümcelerini kullanamamanızdır. Truncate tablosu, daha az işlem günlüğü alanı kullandığından, silmekten daha iyi kabul edilir.

Bir kimlik sütunu varsa, ilk çekirdek değerine sıfırlandığını unutmayın (örneğin, otomatik artan kimlik 1’den yeniden başlar). Kimlik sütunları başka bir tabloda yabancı anahtar olarak kullanılıyorsa, bu tutarsızlığa neden olabilir.

Yeni tablo oluşturun ve eski tablodan kayıt ekleyin #

SELECT * INTO NewTable FROM OldTable

Eski tablo yapısıyla yeni bir tablo oluşturur ve tüm satırları yeni tabloya ekler.

Bazı Kısıtlamalar

  1. Yeni tablo olarak bir tablo değişkeni veya tablo değerli parametre belirtemezsiniz.
  2. Kaynak tablo bölümlenmiş olsa bile bölümlenmiş bir tablo oluşturmak için SELECT…INTO’yu kullanamazsınız. SELECT…INTO, kaynak tablonun bölüm şeması; bunun yerine, yeni tablo varsayılan dosya grubunda oluşturuldu. Bölümlenmiş bir satıra satır eklemek için tablo, önce bölümlenmiş tabloyu oluşturmalı ve ardından INSERT INTO…SELECT FROM deyimi.
  3. Kaynak tabloda tanımlanan dizinler, kısıtlamalar ve tetikleyiciler yeni tabloya aktarılmaz ve SELECT…INTO deyimi. Bu nesneler gerekliyse, oluşturabilirsiniz SELECT…INTO deyimini yürüttükten sonra.
  4. ORDER BY yan tümcesinin belirtilmesi, satırların belirtilen sırada eklendiğini garanti etmez. Seyrek bir sütun dahil edildiğinde seçim listesinde, seyrek sütun özelliği, yeni tablodaki sütun. Bu özellik yeni durumda gerekliyse tablo, SELECT…INTO’yu yürüttükten sonra sütun tanımını değiştirin bu özelliği dahil etmek için ifade.
  5. Hesaplanan bir sütun seçim listesine dahil edildiğinde, yeni tablodaki ilgili sütun hesaplanan bir sütun değildir. bu yeni sütundaki değerler, o sırada hesaplanan değerlerdir SELECT…INTO yürütüldü.

[sic]

Verileri güvenli bir şekilde değiştirmek için İşlemleri kullanma #

Verileri her değiştirdiğinizde, bir Veri Manipülasyon Dili (DML) komutunda, değişikliklerinizi bir işleme sarabilirsiniz. DML, ‘GÜNCELLEME’, ‘TRUNCATE’, ‘INSERT’ ve ‘DELETE’ içerir. Doğru verileri değiştirdiğinizden emin olmanın yollarından biri, bir işlem kullanmaktır.

DML değişiklikleri, etkilenen satırlarda kilitlenir. Bir işleme başladığınızda, işlemi sonlandırmalısınız, aksi takdirde DML’de değiştirilen tüm nesneler, işlemi başlatan kişi tarafından kilitli kalır. ‘ROLLBACK’ veya ‘COMMIT’ ile işleminizi sonlandırabilirsiniz. ‘ROLLBACK’, işlem içindeki her şeyi orijinal durumuna döndürür. “COMMIT”, verileri başka bir DML ifadesi olmadan değişikliklerinizi geri alamayacağınız son bir duruma yerleştirir.

Örnek:

--Create a test table

USE [your database]
GO
CREATE TABLE test_transaction (column_1 varchar(10))
GO

INSERT INTO 
 dbo.test_transaction
        ( column_1 )
VALUES
        ( 'a' )

BEGIN TRANSACTION --This is the beginning of your transaction

UPDATE dbo.test_transaction
SET column_1 = 'B'
OUTPUT INSERTED.*
WHERE column_1 = 'A'
  

ROLLBACK TRANSACTION  --Rollback will undo your changes
           --Alternatively, use COMMIT to save your results

SELECT * FROM dbo.test_transaction   --View the table after your changes have been run

DROP TABLE dbo.test_transaction

Notlar:

  • Bu, hata işlemeyi içermeyen basitleştirilmiş bir örnektir. Ancak herhangi bir veritabanı işlemi başarısız olabilir ve bu nedenle bir istisna oluşturabilir. İşte bir örnek Böyle bir gerekli hata işlemenin nasıl görünebileceği. İşlemleri asla bir hata işleyicisi olmayan** kullanmamalısınız, aksi takdirde işlemi bilinmeyen bir durumda bırakabilirsiniz.
  • izolasyon düzeyine bağlı olarak, işlemler sorgulanan verilere kilitleniyor veya değişti. İşlemlerin uzun süre çalışmadığından emin olmanız gerekir, çünkü bunlar kayıtları bir veritabanında kilitler ve kilitlenmelere yol açabilir. örnekle sunucu kilitlenmeleri/) diğer paralel çalışan işlemlerle. İşlemlerde özetlenen işlemleri mümkün olduğunca kısa tutun ve kilitlediğiniz veri miktarıyla etkiyi en aza indirin.

Tablo Satır Sayısını Alma #

Aşağıdaki örnek, bir veritabanındaki belirli bir tablonun toplam satır sayısını bulmak için kullanılabilir, eğer “tablo_adı” sorgulamak istediğiniz tablo ile değiştirilirse:

SELECT COUNT(*) AS [TotalRowCount] FROM table_name;

Aşağıdaki komut dosyasını kullanarak tabloların HEAP (index_id = 0) veya küme kümelenmiş dizinine (index_id = 1) dayalı olarak tablonun bölümüne geri katılarak tüm tablolar için satır sayısını elde etmek de mümkündür:

SELECT  [Tables].name                AS [TableName],
        SUM( [Partitions].[rows] )    AS [TotalRowCount]
FROM    sys.tables AS [Tables]
JOIN    sys.partitions AS [Partitions]
    ON  [Tables].[object_id]    =    [Partitions].[object_id]
    AND [Partitions].index_id IN ( 0, 1 )
--WHERE    [Tables].name = N'table name' /* uncomment to look for a specific table */
GROUP BY    [Tables].name;

Ek bölümler eklenmedikçe, her tablo esasen tek bir bölüm tablosu olduğundan bu mümkündür. Bu komut dosyası ayrıca, tablo satırlarına okuma/yazma işlemlerine müdahale etmeme avantajına da sahiptir.
