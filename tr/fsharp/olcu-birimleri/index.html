Çalışma Zamanında Birimler #

Ölçü Birimleri yalnızca derleyici tarafından statik kontrol için kullanılır ve çalışma zamanında kullanılamaz. Yansımada veya ‘ToString’ gibi yöntemlerde kullanılamazlar.

Örneğin, C#, bir F# kitaplığı tarafından tanımlanan ve bu kitaplıktan gösterilen “float” türünde bir alan için birimi olmayan bir “double” verir.

Hesaplamalarda Tutarlı Birimlerin Sağlanması #

Ölçü birimleri, kayan noktalara veya tam sayılara eklenebilen ek tür açıklamalarıdır. Derleme zamanında hesaplamaların birimleri tutarlı bir şekilde kullandığını doğrulamak için kullanılabilirler.

Ek açıklamaları tanımlamak için:

[<Measure>] type m // meters
[<Measure>] type s // seconds
[<Measure>] type accel = m/s^2 // acceleration defined as meters per second squared

Tanımlandıktan sonra, bir ifadenin beklenen türle sonuçlandığını doğrulamak için ek açıklamalar kullanılabilir.

// Compile-time checking that this function will return meters, since (m/s^2) * (s^2) -> m
// Therefore we know units were used properly in the calculation.
let freeFallDistance (time:float<s>) : float<m> = 
    0.5 * 9.8<accel> * (time*time)    

// It is also made explicit at the call site, so we know that the parameter passed should be in seconds
let dist:float<m> = freeFallDistance 3.0<s>
printfn "%f" dist

Birimler arasındaki dönüşümler #

[<Measure>] type m // meters
[<Measure>] type cm // centimeters

// Conversion factor
let cmInM = 100<cm/m>

let distanceInM = 1<m>
let distanceInCM = distanceInM * cmInM // 100<cm>

// Conversion function
let cmToM (x : int<cm>) = x / 100<cm/m>
let mToCm (x : int<m>) = x * 100<cm/m>

cmToM 100<cm> // 1<m>
mToCm 1<m> // 100<cm>

F# derleyicisinin “1“nin “100“ye eşit olduğunu bilmediğine dikkat edin. Umurunda olduğu sürece, birimler ayrı tiplerdir. Metreden kilograma dönüştürmek için benzer fonksiyonlar yazabilirsiniz ve derleyici umursamaz.

[<Measure>] type kg

// Valid code, invalid physics
let kgToM x = x / 100<kg/m>

Ölçü birimlerini, aşağıdaki gibi diğer birimlerin katları olarak tanımlamak mümkün değildir.

// Invalid code
[<Measure>] type m = 100<cm>

Bununla birlikte, “bir şey başına” birimleri tanımlamak, örneğin Hertz, frekansı ölçmek basitçe “saniyede” dir, oldukça basittir.

// Valid code
[<Measure>] type s
[<Measure>] type Hz = /s

1 / 1<s> = 1 <Hz> // Evaluates to true

[<Measure>] type N = kg m/s // Newtons, measuring force. Note lack of multiplication sign.

// Usage
let mass = 1<kg>
let distance = 1<m>
let time = 1<s>

let force = mass * distance / time // Evaluates to 1<kg m/s>
force = 1<N> // Evaluates to true

Ölçü birimi türü parametreleri #

“[]” özelliği, ölçü birimlerine göre genel olan türleri bildirmek için tür parametrelerinde kullanılabilir:

type CylinderSize<[<Measure>] 'u> =
    { Radius : float<'u>
      Height : float<'u> }

Test kullanımı:

open Microsoft.FSharp.Data.UnitSystems.SI.UnitSymbols

/// This has type CylinderSize<m>.
let testCylinder =
    { Radius = 14.<m>
      Height =  1.<m> }

Birimleri korumak veya ayarlamak için LanguagePrimitives’i kullanma #

Bir işlev, alt düzey işlemler nedeniyle birimleri otomatik olarak korumadığında, onları destekleyen temel öğelerdeki birimleri ayarlamak için “LanguagePrimitives” modülü kullanılabilir:

/// This cast preserves units, while changing the underlying type
let inline castDoubleToSingle (x : float<'u>) : float32<'u> =
    LanguagePrimitives.Float32WithMeasure (float32 x)

Çift duyarlıklı kayan noktalı bir değere ölçü birimleri atamak için, doğru birimlerle bir ile çarpmanız yeterlidir:

[<Measure>]
type USD

let toMoneyImprecise (amount : float) =
   amount * 1.<USD>

System.Double olmayan, birimsiz bir değere ölçü birimleri atamak için, örneğin başka bir dilde yazılmış bir kitaplıktan geliyorsanız, bir dönüştürme kullanın:

open LanguagePrimitives

let toMoney amount =
   amount |> DecimalWithMeasure<'u>

F# interaktif tarafından bildirilen fonksiyon türleri şunlardır:

val toMoney : amount:decimal -> decimal<'u>
val toMoneyImprecise : amount:float -> float<USD>

Uyumluluğu korumak için standartlaştırılmış birim türlerini kullanın #

Örneğin, SI birimlerinin türleri, F# çekirdek kitaplığında, “Microsoft.FSharp.Data.UnitSystems.SI” içinde standartlaştırılmıştır. Bunları kullanmak için uygun alt ad alanını, ‘UnitNames’ veya ‘UnitSymbols’u açın. Veya yalnızca birkaç SI birimi gerekiyorsa, tür takma adlarıyla içe aktarılabilirler:

/// Seconds, the SI unit of time. Type abbreviation for the Microsoft standardized type.
type [<Measure>] s = Microsoft.FSharp.Data.UnitSystems.SI.UnitSymbols.s

Bazı kullanıcılar, bir tanım zaten mevcut olduğunda yapılmaması gereken aşağıdakileri yapma eğilimindedir:

/// Seconds, the SI unit of time
type [<Measure>] s // DO NOT DO THIS! THIS IS AN EXAMPLE TO EXPLAIN A PROBLEM.

Standart SI türlerine atıfta bulunan diğer kodlarla arabirim oluşturulduğunda fark belirginleşir. Standart birimlere atıfta bulunan kod uyumluyken, kendi türünü tanımlayan kod, belirli tanımını kullanmayan herhangi bir kodla uyumsuzdur.

Bu nedenle, her zaman SI birimleri için standart türleri kullanın. ‘UnitNames’ veya ‘UnitSymbols’a atıfta bulunmanızın bir önemi yoktur, çünkü bu ikisi içindeki eşdeğer adlar aynı türe atıfta bulunur:

open Microsoft.FSharp.Data.UnitSystems.SI

/// This is valid, since both versions refer to the same authoritative type.
let validSubtraction = 1.<UnitSymbols.s> - 0.5<UnitNames.second>
