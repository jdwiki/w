Türlere Giriş #

Türler çeşitli şeyleri temsil edebilir. Tek bir veri, bir veri seti veya bir fonksiyon olabilir.

F#‘da türleri iki kategoriye ayırabiliriz:

  • F# türleri:

      // Functions
  let a = fun c -> c

  // Tuples
  let b = (0, "Foo")

  // Unit type
  let c = ignore

  // Records
  type r = { Name : string; Age : int }
  let d = { Name = "Foo"; Age = 10 }

  // Discriminated Unions
  type du = | Foo | Bar
  let e = Bar

  // List and seq
  let f = [ 0..10 ]
  let g = seq { 0..10 }

  // Aliases
  type MyAlias = string

  • .NET türleri

    • Built-in type (int, bool, string,…)
    • Classes, Structs & Interfaces
    • Delegates
    • Arrays

Tür Kısaltmaları #

Tür kısaltmaları, onlara daha anlamlı bir anlam kazandırmak için mevcut türler üzerinde takma adlar oluşturmanıza olanak tanır.

// Name is an alias for a string
type Name = string

// PhoneNumber is an alias for a string
type PhoneNumber = string

Ardından, diğer adı herhangi bir türde olduğu gibi kullanabilirsiniz:

// Create a record type with the alias
type Contact = {
    Name : Name
    Phone : PhoneNumber }

// Create a record instance
// We can assign a string since Name and PhoneNumber are just aliases on string type
let c = {
    Name = "Foo"
    Phone = "00 000 000" }

printfn "%A" c

// Output
// {Name = "Foo";
// Phone = "00 000 000";}

Dikkatli olun, takma adlar tür tutarlılığını kontrol etmez. Bu, aynı türü hedefleyen iki takma adın birbirine atanabileceği anlamına gelir:

let c = {
    Name = "Foo"
    Phone = "00 000 000" }
let d = {
    Name = c.Phone
    Phone = c.Name }

printfn "%A" d

// Output
// {Name = "00 000 000";
// Phone = "Foo";}

Türler, type anahtar sözcüğü kullanılarak F#‘da oluşturulur #

“F#”, farklı türler oluşturmak için “type” anahtar sözcüğünü kullanır.

  1. Takma ad yazın
  2. Ayrımcı sendika türleri
  3. Kayıt türleri
  4. Arayüz türleri
  5. Sınıf türleri
  6. Yapı türleri

Mümkün olduğunda eşdeğer “C#” koduna sahip örnekler:

// Equivalent C#:
//  using IntAliasType = System.Int32;
type IntAliasType = int // As in C# this doesn't create a new type, merely an alias

type DiscriminatedUnionType =
  | FirstCase
  | SecondCase  of int*string

  member x.SomeProperty = // We can add members to DU:s
    match x with
    | FirstCase         -> 0
    | SecondCase (i, _) -> i

type RecordType =
  {
    Id    : int
    Name  : string
  }
  static member New id name : RecordType = // We can add members to records
    { Id = id; Name = name } // { ... } syntax used to create records

// Equivalent C#:
//  interface InterfaceType
//  {
//    int     Id    { get; }
//    string  Name  { get; }
//    int Increment (int i);
//  }  
type InterfaceType =
  interface // In order to create an interface type, can also use [<Interface>] attribute
    abstract member Id        : int
    abstract member Name      : string
    abstract member Increment : int -> int
  end

// Equivalent C#:
//  class ClassType : InterfaceType
//  {
//    static int increment (int i)
//    {
//      return i + 1;
//    }
//  
//    public ClassType (int id, string name)
//    {
//      Id    = id    ;
//      Name  = name  ;
//    }
//  
//    public int    Id    { get; private set; }
//    public string Name  { get; private set; }
//    public int   Increment (int i)
//    {
//      return increment (i);
//    }
//  }
type ClassType (id : int, name : string) = // a class type requires a primary constructor
  let increment i = i + 1 // Private helper functions

  interface InterfaceType with // Implements InterfaceType
    member x.Id           = id
    member x.Name         = name
    member x.Increment i  = increment i


// Equivalent C#:
//  class SubClassType : ClassType
//  {
//    public SubClassType (int id, string name) : base(id, name)
//    {
//    }
//  }
type SubClassType (id : int, name : string) =
  inherit ClassType (id, name) // Inherits ClassType

// Equivalent C#:
//  struct StructType
//  {
//    public StructType (int id)
//    {
//      Id = id;
//    }
//  
//    public int Id { get; private set; }
//  }
type StructType (id : int) =
  struct  // In order create a struct type, can also use [<Struct>] attribute
    member x.Id = id
  end

Tür Çıkarımı #

Teşekkür

Bu örnek, tür çıkarımı hakkındaki bu makaleden uyarlanmıştır.

Tür Çıkarım nedir?

Tür Çıkarımı, derleyicinin hangi türlerin nerede kullanıldığını çıkarmasına izin veren mekanizmadır. Bu mekanizma, genellikle “Hindley-Milner” veya “HM” olarak adlandırılan bir algoritmaya dayanmaktadır. Basit ve işlev değerlerinin türlerini belirlemek için bazı kurallara bakın:

  • Harflere bakın
  • Bir şeyin etkileşime girdiği işlevlere ve diğer değerlere bakın
  • Herhangi bir açık tür kısıtlamasına bakın
  • Herhangi bir yerde kısıtlama yoksa, otomatik olarak genel türlere genelleme yapın

Yazılara bakın

Derleyici, değişmezlere bakarak türleri çıkarabilir. Eğer değişmez değer bir int ise ve ona “x” ekliyorsanız, “x” de bir int olmalıdır. Ancak, değişmez değer bir kayan nokta ise ve ona “x” ekliyorsanız, o zaman “x” de bir kayan nokta olmalıdır.

İşte bazı örnekler:

let inferInt x = x + 1
let inferFloat x = x + 1.0
let inferDecimal x = x + 1m     // m suffix means decimal
let inferSByte x = x + 1y       // y suffix means signed byte
let inferChar x = x + 'a'       // a char
let inferString x = x + "my string"

Etkileşime girdiği işlevlere ve diğer değerlere bakın

Herhangi bir yerde değişmez değer yoksa, derleyici, etkileşimde bulundukları işlevleri ve diğer değerleri analiz ederek türleri çözmeye çalışır.

let inferInt x = x + 1
let inferIndirectInt x = inferInt x       //deduce that x is an int

let inferFloat x = x + 1.0
let inferIndirectFloat x = inferFloat x   //deduce that x is a float

let x = 1
let y = x     //deduce that y is also an int

Herhangi bir açık tür kısıtlamasına veya ek açıklamaya bakın

Belirtilen herhangi bir açık tür kısıtlaması veya ek açıklama varsa, derleyici bunları kullanır.

let inferInt2 (x:int) = x                // Take int as parameter
let inferIndirectInt2 x = inferInt2 x    // Deduce from previous that x is int

let inferFloat2 (x:float) = x                // Take float as parameter
let inferIndirectFloat2 x = inferFloat2 x    // Deduce from previous that x is float

Otomatik genelleme

Tüm bunlardan sonra, herhangi bir kısıtlama bulunmadıysa, derleyici türleri yalnızca genel yapar.

let inferGeneric x = x 
let inferIndirectGeneric x = inferGeneric x 
let inferIndirectGenericAgain x = (inferIndirectGeneric x).ToString()

Tür çıkarımıyla ilgili yanlış gidebilecek şeyler

Tür çıkarımı mükemmel değil, ne yazık ki. Bazen derleyici ne yapacağını bilemez. Yine, neler olduğunu anlamak, derleyiciyi öldürmek yerine sakin kalmanıza gerçekten yardımcı olacaktır. Tür hatalarının ana nedenlerinden bazıları şunlardır:

  • Düzensiz beyanlar
  • Yeterli bilgi yok
  • Aşırı yüklenmiş yöntemler

Sıra dışı beyanlar

Temel bir kural, işlevleri kullanılmadan önce bildirmeniz gerektiğidir.

Bu kod başarısız:

let square2 x = square x   // fails: square not defined
let square x = x * x

Ama bu tamam:

let square x = x * x       
let square2 x = square x   // square already defined earlier

Yinelemeli veya eşzamanlı bildirimler

Birbirine atıfta bulunması gereken özyinelemeli işlevler veya tanımlarla, “düzensiz” sorununun bir çeşidi ortaya çıkar. Bu durumda hiçbir yeniden sıralama yardımcı olmaz - derleyiciye yardımcı olmak için ek anahtar kelimeler kullanmamız gerekir.

Bir işlev derlenirken, işlev tanımlayıcısı gövde tarafından kullanılamaz. Yani basit bir özyinelemeli fonksiyon tanımlarsanız derleyici hatası alırsınız. Düzeltme, işlev tanımının bir parçası olarak “rec” anahtar sözcüğünü eklemektir. Örneğin:

// the compiler does not know what "fib" means
let fib n =
   if n <= 2 then 1
   else fib (n - 1) + fib (n - 2)
   // error FS0039: The value or constructor 'fib' is not defined

İşte özyinelemeli olduğunu belirtmek için “rec fib” eklenmiş sabit sürüm:

let rec fib n =              // LET REC rather than LET
   if n <= 2 then 1
   else fib (n - 1) + fib (n - 2)

Yeterli bilgi yok

Bazen derleyici bir tür belirlemek için yeterli bilgiye sahip değildir. Aşağıdaki örnekte, derleyici Uzunluk yönteminin ne tür üzerinde çalışması gerektiğini bilmiyor. Ama onu da jenerik yapamıyor, bu yüzden şikayet ediyor.

let stringLength s = s.Length
  // error FS0072: Lookup on object of indeterminate type
  // based on information prior to this program point.
  // A type annotation may be needed ...

Bu tür hatalar, açık açıklamalarla düzeltilebilir.

let stringLength (s:string) = s.Length

Aşırı yüklenmiş yöntemler

.NET’te harici bir sınıfı veya yöntemi çağırırken, aşırı yükleme nedeniyle sıklıkla hatalar alırsınız.

Aşağıdaki concat örneği gibi birçok durumda, derleyicinin hangi aşırı yüklenmiş yöntemi çağıracağını bilmesi için harici işlevin parametrelerine açıkça açıklama eklemeniz gerekir.

let concat x = System.String.Concat(x)           //fails
let concat (x:string) = System.String.Concat(x)  //works
let concat x = System.String.Concat(x:string)    //works

Bazen aşırı yüklenmiş yöntemlerin farklı argüman adları vardır, bu durumda, argümanları adlandırarak derleyiciye bir ipucu da verebilirsiniz. İşte StreamReader yapıcısı için bir örnek.

let makeStreamReader x = new System.IO.StreamReader(x)        //fails
let makeStreamReader x = new System.IO.StreamReader(path=x)   //works
