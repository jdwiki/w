Basit Aktif Modeller #

Etkin kalıplar, verilerinizin içine girebileceği adlandırılmış kategoriler belirleyebileceğiniz ve ardından bu kategorileri “eşleşme” ifadelerinde kullanabileceğiniz özel bir kalıp eşleştirme türüdür.

Sayıları pozitif, negatif veya sıfır olarak sınıflandıran etkin bir kalıp tanımlamak için:

let (|Positive|Negative|Zero|) num = 
    if num > 0 then Positive 
    elif num < 0 then Negative
    else Zero

Bu daha sonra bir kalıp eşleştirme ifadesinde kullanılabilir:

let Sign value = 
    match value with
    | Positive -> printf "%d is positive" value
    | Negative -> printf "%d is negative" value
    | Zero -> printf "The value is zero"

Sign -19 // -19 is negative
Sign 2 // 2 is positive
Sign 0 // The value is zero

Aktif Modeller, fonksiyon argümanlarını doğrulamak ve dönüştürmek için kullanılabilir #

F# içinde Aktif Modellerin ilginç fakat oldukça bilinmeyen bir kullanımı, fonksiyon argümanlarını doğrulamak ve dönüştürmek için kullanılabilmeleridir.

Argüman doğrulaması yapmanın klasik yolunu düşünün:

// val f : string option -> string option -> string
let f v u =
  let v = defaultArg v "Hello"
  let u = defaultArg u "There"
  v + " " + u

// val g : 'T -> 'T (requires 'T null)
let g v =
  match v with
  | null  -> raise (System.NullReferenceException ())
  | _     -> v.ToString ()

Tipik olarak, argümanların doğru olduğunu doğrulamak için yönteme kod ekleriz. F# içinde Aktif Kalıpları kullanarak bunu genelleştirebilir ve argüman bildiriminde amacı beyan edebiliriz.

Aşağıdaki kod, yukarıdaki koda eşdeğerdir:

let inline (|DefaultArg|) dv ov = defaultArg ov dv

let inline (|NotNull|) v =
  match v with
  | null  -> raise (System.NullReferenceException ())
  | _     -> v

// val f : string option -> string option -> string
let f (DefaultArg "Hello" v) (DefaultArg "There" u) = v + " " + u

// val g : 'T -> string (requires 'T null)
let g (NotNull v) = v.ToString ()

‘f’ ve ‘g’ işlevlerinin kullanıcısı için iki farklı sürüm arasında hiçbir fark yoktur.

printfn "%A" <| f (Some "Test") None  // Prints "Test There"
printfn "%A" <| g "Test"              // Prints "Test"
printfn "%A" <| g null                // Will throw

Bir endişe, Aktif Kalıpların performans yükü eklemesidir. Durumun böyle olup olmadığını görmek için ‘f’ ve ‘g’yi geri derlemek için ‘ILSpy’ kullanalım.

public static string f(FSharpOption<string> _arg2, FSharpOption<string> _arg1)
{
  return Operators.DefaultArg<string>(_arg2, "Hello") + " " + Operators.DefaultArg<string>(_arg1, "There");
}

public static string g<a>(a _arg1) where a : class
{
  if (_arg1 != null)
  {
    a a = _arg1;
    return a.ToString();
  }
  throw new NullReferenceException();
}

“Satır içi” sayesinde, Aktif Modeller, argüman doğrulama yapmanın klasik yöntemine kıyasla fazladan bir ek yük oluşturmaz.

.NET API sarmalayıcıları olarak Aktif Kalıplar #

Aktif Modeller, bazı .NET API’lerini çağırmayı daha doğal hissettirmek için kullanılabilir, özellikle de sadece fonksiyon dönüş değerinden daha fazlasını döndürmek için bir çıkış parametresi kullananlar.

Örneğin, normalde “System.Int32.TryParse” yöntemini şu şekilde çağırırsınız:

let couldParse, parsedInt = System.Int32.TryParse("1")
if couldParse then printfn "Successfully parsed int: %i" parsedInt
else printfn "Could not parse int"

Desen eşleştirmeyi kullanarak bunu biraz geliştirebilirsiniz:

match System.Int32.TryParse("1") with 
| (true, parsedInt) -> printfn "Successfully parsed int: %i" parsedInt
| (false, _) -> printfn "Could not parse int"

Ancak, System.Int32.TryParse fonksiyonunu saran aşağıdaki Aktif Modeli de tanımlayabiliriz:

let (|Int|_|) str =
    match System.Int32.TryParse(str) with
    | (true, parsedInt) -> Some parsedInt
    | _ -> None

Şimdi şunları yapabiliriz:

match "1" with
| Int parsedInt -> printfn "Successfully parsed int: %i" parsedInt
| _ -> printfn "Could not parse int"

Aktif Modellere sarılmak için bir başka iyi aday, normal ifadeler API’leridir:

let (|MatchRegex|_|) pattern input =
    let m = System.Text.RegularExpressions.Regex.Match(input, pattern) 
    if m.Success then Some m.Groups.[1].Value 
    else None  

match "bad" with
| MatchRegex "(good|great)" mood -> 
    printfn "Wow, it's a %s day!" mood
| MatchRegex "(bad|terrible)" mood -> 
    printfn "Unfortunately, it's a %s day." mood
| _ -> 
    printfn "Just a normal day"

Parametreli Aktif Modeller #

Aktif desenler sadece basit işlevlerdir.

Fonksiyonlar gibi ek parametreler tanımlayabilirsiniz:

let (|HasExtension|_|) expected (uri : string) = 
    let result = uri.EndsWith (expected, StringComparison.CurrentCultureIgnoreCase)
    match result with
    | true -> Some true
    | _ -> None

Bu, şu şekilde eşleşen bir desende kullanılabilir:

    let isXMLFile uri =
        match uri with
        | HasExtension ".xml" _ -> true
        | _ -> false

Tam ve Kısmi Aktif Modeller #

Kullanımda biraz farklılık gösteren iki tür Aktif Model vardır - Tam ve Kısmi.

Tam Aktif Modeller, “sayı tek mi çift mi?” gibi tüm sonuçları sıralayabildiğiniz zaman kullanılabilir.

let (|Odd|Even|) number = 
  if number % 2 = 0
  then Even
  else Odd

Aktif Model tanımının hem olası durumları listelediğine hem de başka hiçbir şey listelemediğine ve gövdenin listelenen durumlardan birini döndürdüğüne dikkat edin. Bir eşleşme ifadesinde kullandığınızda, bu tam bir eşleşmedir:

let n = 13
match n with
| Odd -> printf "%i is odd" n
| Even -> printf "%i is even" n

Bu, girdi alanını, onu tamamen kapsayan bilinen kategorilere ayırmak istediğinizde kullanışlıdır.

Kısmi Aktif Modeller ise bir ‘seçenek’ döndürerek bazı olası sonuçları açıkça görmezden gelmenize izin verir. Tanımları, eşleşmeyen durum için özel bir “_” durumu kullanır.

let (|Integer|_|) str =
  match Int32.TryParse(str) with
  | (true, i) -> Some i
  | _ -> None

Bu şekilde, bazı durumlar ayrıştırma fonksiyonumuz tarafından ele alınamadığında bile eşleştirebiliriz.

let s = "13"
match s with
| Integer i -> "%i was successfully parsed!" i
| _ -> "%s is not an int" s

Kısmi Aktif Modeller, girdinin diğer seçenekleri göz ardı ederken girdi alanında belirli bir kategoriye girip girmediğini test etme biçimi olarak kullanılabilir.
