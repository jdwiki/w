Desen Eşleştirme

Bu sayfada

  • Kalıp eşleştirme, tüm alanın kapsandığını kontrol eder
  • Korumalar isteğe bağlı koşul eklemenize izin verdiğinde
  • Vakalar yukarıdan aşağıya değerlendirilir ve ilk eşleşme kullanılır
  • Eşleştirme Seçenekleri

Kalıp Eşleştirme, birden çok “if”/“else if”/“else” tarzı deyim kullanmaya kıyasla dallanmanın çok kısa ve öz bir şekilde ele alınmasına izin verdiği için birçok işlevsel dilin güçlü bir özelliğidir. Bununla birlikte, yeterli seçenek ve [“ne zaman” korumaları1 verilirse, Model Eşleştirme de ayrıntılı olabilir ve bir bakışta anlaşılması zor olabilir.

Bu olduğunda, F#‘ın Aktif Kalıpları, eşleşen mantığa anlamlı adlar vermenin harika bir yolu olabilir, bu da kodu basitleştirir ve ayrıca yeniden kullanıma olanak tanır.

Kalıp eşleştirme, tüm alanın kapsandığını kontrol eder #

let x = true
match x with
| true -> printfn "x is true"

bir uyarı verir #

C:\Program Files (x86)\Microsoft VS Code\Untitled-1(2,7): uyarı FS0025: Bu ifadede eksik kalıp eşleşmeleri. Örneğin, ‘yanlış’ değeri kalıp(lar)ın kapsamadığı bir durumu belirtebilir.

Bunun nedeni, olası tüm bool değerlerinin kapsanmamış olmasıdır.

bools açıkça listelenebilir, ancak int’leri listelemek daha zordur #

let x = 5
match x with
| 1 -> printfn "x is 1"
| 2 -> printfn "x is 2"
| _ -> printfn "x is something else"

burada özel _ karakterini kullanıyoruz. “_” diğer tüm olası durumlarla eşleşir.

‘_’ başını belaya sokabilir #

kendimiz yarattığımız bir tür düşünün, buna benziyor

type Sobriety = 
    | Sober
    | Tipsy
    | Drunk

Buna benzeyen bir expession ile bir eşleşme yazabiliriz

match sobriety with
| Sober -> printfn "drive home"
| _ -> printfn "call an uber"

Yukarıdaki kod mantıklı. Ayık değilseniz bir uber çağırmanız gerektiğini varsayıyoruz, bu yüzden bunu belirtmek için “_” kullanıyoruz

Daha sonra kodumuzu buna göre yeniden düzenleriz

type Sobriety = 
    | Sober
    | Tipsy
    | Drunk
    | Unconscious

F# derleyicisi bize bir uyarı vermeli ve kişinin tıbbi yardım alması için eşleşme ifademizi yeniden düzenlememizi istemelidir. Bunun yerine kibrit ifadesi, bilinçsiz kişiye sanki sadece sarhoşmuş gibi davranır. Mesele şu ki, mantık hatalarından kaçınmak için mümkün olduğunda durumları açıkça listelemeyi seçmelisiniz.

Korumalar isteğe bağlı koşul eklemenize izin verdiğinde #

type Person = {
    Age : int
    PassedDriversTest : bool }

let someone = { Age = 19; PassedDriversTest = true }

match someone.PassedDriversTest with
| true when someone.Age >= 16 -> printfn "congrats"
| true -> printfn "wait until you are 16"
| false -> printfn "you need to pass the test"

Vakalar yukarıdan aşağıya değerlendirilir ve ilk eşleşme kullanılır #

Yanlış kullanım:

Aşağıdaki snippet’te son eşleşme asla kullanılmayacaktır:

let x = 4
match x with
| 1 -> printfn "x is 1"
| _ -> printfn "x is anything that wasn't listed above"
| 4 -> printfn "x is 4"

baskılar

x, yukarıda listelenmeyen herhangi bir şeydir

Doğru kullanım:

Burada, hem “x = 1” hem de “x = 4” kendi özel durumlarına ulaşırken, diğer her şey varsayılan durum olan “_” durumuna düşer:

let x = 4
match x with
| 1 -> printfn "x is 1"
| 4 -> printfn "x is 4"
| _ -> printfn "x is anything that wasn't listed above"

baskılar

x 4’tür

Eşleştirme Seçenekleri #

Desen eşleştirme, Seçenekleri işlemek için yararlı olabilir:

let result = Some("Hello World")
match result with
| Some(message) -> printfn message
| None -> printfn "Not feeling talkative huh?"
