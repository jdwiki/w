Ayrımcı birlikler içindeki demetlerin öğelerini adlandırma #

Ayrımlı birleşimleri tanımlarken, tanımlama grubu türlerinin öğelerini adlandırabilir ve bu adları kalıp eşleştirme sırasında kullanabilirsiniz.

type Shape = 
    | Circle of diameter:int
    | Rectangle of width:int * height:int

let shapeIsTenWide = function
    | Circle(diameter=10) 
    | Rectangle(width=10) -> true
    | _ -> false

Ek olarak, ayrımcı birleşimlerin öğelerini adlandırmak, kodun okunabilirliğini ve C# ile birlikte çalışabilirliği geliştirir - özelliklerin adları ve kurucuların parametreleri için sağlanan adlar kullanılacaktır. Birlikte çalışma kodunda varsayılan olarak oluşturulan adlar “Öğe”, “Öğe1”, “Öğe2”…

Tek Vaka Ayrımcılığa Olan Sendikaları Kayıt Olarak Kullanma #

Bazen, kayıt benzeri türleri uygulamak için yalnızca bir vaka ile birleşim türleri oluşturmak yararlıdır:

type Point = Point of float * float

let point1 = Point(0.0, 3.0)

let point2 = Point(-2.5, -4.0)

Bunlar çok kullanışlıdır, çünkü bunlar, tuple argümanlarının yapabildiği gibi, desen eşleştirme yoluyla ayrıştırılabilir:

let (Point(x1, y1)) = point1
// val x1 : float = 0.0
// val y1 : float = 3.0

let distance (Point(x1,y1)) (Point(x2,y2)) =
    pown (x2-x1) 2 + pown (y2-y1) 2 |> sqrt
// val distance : Point -> Point -> float

distance point1 point2
// val it : float = 7.433034374

Temel Ayrımcı Birlik Kullanımı #

F#‘daki ayrımcı birleşimler, herhangi bir sayıda farklı veri türünü tutabilen türleri tanımlamanın bir yolunu sunar. İşlevleri, C++ birleşimlerine veya VB türevlerine benzer, ancak ek olarak güvenli olma avantajına sahiptir.

// define a discriminated union that can hold either a float or a string
type numOrString = 
    | F of float
    | S of string

let str = S "hi" // use the S constructor to create a string
let fl = F 3.5 // use the F constructor to create a float

// you can use pattern matching to deconstruct each type
let whatType x = 
    match x with
        | F f -> printfn "%f is a float" f
        | S s -> printfn "%s is a string" s

whatType str // hi is a string
whatType fl // 3.500000 is a float

Gerekli Nitelikli Erişim #

‘RequireQualifiedAccess’ özniteliği ile, sendika vakalarına sadece ‘MyCase’ yerine ‘MyUnion.MyCase’ olarak atıfta bulunulmalıdır. Bu, çevreleyen ad alanında veya modülde ad çakışmalarını önler:

type [<RequireQualifiedAccess>] Requirements =
    None | Single | All

// Uses the DU with qualified access
let noRequirements = Requirements.None

// Here, None still refers to the standard F# option case
let getNothing () = None

// Compiler error unless All has been defined elsewhere
let invalid = All

Örneğin, “Sistem” açıldıysa, “Single”, “System.Single” anlamına gelir. ‘Requirements.Single’ sendika durumuyla çakışma yok.

Enum tarzı birleşimler #

Ayrımcı bir birliktelik durumlarında tür bilgilerinin dahil edilmesine gerek yoktur. Tür bilgilerini atlayarak, bir numaralandırmaya benzer şekilde bir dizi seçeneği temsil eden bir birleşim oluşturabilirsiniz.

// This union can represent any one day of the week but none of 
// them are tied to a specific underlying F# type
type DayOfWeek = Monday | Tuesday | Wednesday | Thursday | Friday | Saturday | Sunday

Yansıma ile dizelere ve dizelerden dönüştürme #

Bazen Ayrımcı Bir Birliği bir dizgeye ve dizgeden dönüştürmek gerekir:

module UnionConversion 
    open Microsoft.FSharp.Reflection
    
    let toString (x: 'a) = 
        match FSharpValue.GetUnionFields(x, typeof<'a>) with
        | case, _ -> case.Name

    let fromString<'a> (s : string) =
        match FSharpType.GetUnionCases typeof<'a> |> Array.filter (fun case -> case.Name = s) with 
        | [|case|] -> Some(FSharpValue.MakeUnion(case, [||])) :?> 'a)
        | _ -> None

Tek vaka ayrımcılığa maruz kalan sendika #

Tek vaka ayrımcılığı yapılan bir sendika, yalnızca bir davaya sahip olması dışında, ayrımcılığa uğrayan diğer sendikalar gibidir.

// Define single-case discriminated union type.
type OrderId = OrderId of int
// Construct OrderId type.
let order = OrderId 123
// Deconstruct using pattern matching. 
// Parentheses used so compiler doesn't think it is a function definition.   
let (OrderId id) = order

Tür güvenliğini zorlamak için kullanışlıdır ve yeni tür oluşturmanın daha fazla ek yük getirdiği C# ve Java’nın aksine F#‘da yaygın olarak kullanılır.

Aşağıdaki iki alternatif tip tanımı, aynı tek durumlu ayrımcı birliğin bildirilmesine neden olur:

type OrderId = | OrderId of int

type OrderId =
      | OrderId of int

Özyinelemeli ayrımcı birlikler #

Özyinelemeli tip #

Ayrımcı sendikalar özyinelemeli olabilir, yani tanımlarında kendilerine atıfta bulunabilirler. Buradaki en önemli örnek bir ağaçtır:

type Tree =
    | Branch of int * Tree list
    | Leaf of int

Örnek olarak aşağıdaki ağacı tanımlayalım:

    1
  2   5
3   4

Bu ağacı özyinelemeli ayrımlı birliğimizi kullanarak aşağıdaki gibi tanımlayabiliriz:

let leaf1 = Leaf 3
let leaf2 = Leaf 4
let leaf3 = Leaf 5

let branch1 = Branch (2, [leaf1; leaf2])
let tip = Branch (1, [branch1; leaf3])

Ağaç üzerinde yineleme yapmak, yalnızca bir desen eşleştirme meselesidir:

let rec toList tree =
    match tree with
    | Leaf x -> [x]
    | Branch (x, xs) -> x :: (List.collect toList xs)

let treeAsList = toList tip // [1; 2; 3; 4; 5]

Karşılıklı bağımlı özyinelemeli türler #

Özyinelemeyi başarmanın bir yolu, iç içe karşılıklı bağımlı türlere sahip olmaktır.

// BAD
type Arithmetic = {left: Expression; op:string; right: Expression}
// illegal because until this point, Expression is undefined
type Expression = 
| LiteralExpr of obj
| ArithmeticExpr of Arithmetic

Doğrudan ayrımcılığa maruz kalmış bir birliğin içinde bir kayıt türü tanımlamak kullanımdan kaldırılmıştır:

// BAD
type Expression = 
| LiteralExpr of obj
| ArithmeticExpr of {left: Expression; op:string; right: Expression}
// illegal in recent F# versions

Karşılıklı bağımlı tanımları zincirlemek için “ve” anahtar sözcüğünü kullanabilirsiniz:

// GOOD
type Arithmetic = {left: Expression; op:string; right: Expression}
and Expression = 
| LiteralExpr of obj
| ArithmeticExpr of Arithmetic
