Boş değerler üzerinde Option<‘T> kullanın #

F# gibi fonksiyonel programlama dillerinde null değerleri dikkate alınır. potansiyel olarak zararlı ve zayıf stil (deyimsel olmayan).

Bu “C#” kodunu göz önünde bulundurun:

string x = SomeFunction ();
int    l = x.Length;

x.Length atacak eğer x null ise koruma ekleyelim:

string x = SomeFunction ();
int    l = x != null ? x.Length : 0;

Veya:

string x = SomeFunction () ?? "";
int    l = x.Length;

Veya:

string x = SomeFunction ();
int    l = x?.Length;

Deyimsel F# null değerleri kullanılmadığından kodumuz şöyle görünür:

let x = SomeFunction ()
let l = x.Length

Ancak bazen boş veya geçersiz değerleri temsil etmeye ihtiyaç duyulur. O zamanlar Option<'T> kullanabiliriz:

let SomeFunction () : string option = ...

“SomeFunction”, “Some” “dize” değerini veya “Yok” değerini döndürür. çıkarıyoruz kalıp eşleştirmeyi kullanan “dize” değeri

let v =
  match SomeFunction () with
  | Some x  -> x.Length
  | None    -> 0

Bu kodun daha az kırılgan olmasının nedeni:

string x = SomeFunction ();
int    l = x.Length;

Çünkü bir ‘string seçeneği’ üzerinde ‘Length’ arayamayız. çıkarmamız gerekiyor ‘string’ değeri, desen eşleştirmeyi kullanarak ve bunu yaparak, “string” değerinin kullanımı güvenlidir.

Opsiyon Modülü Demiryolu Odaklı Programlamaya olanak tanır #

Hata işleme önemlidir ancak zarif bir algoritmayı bir karmaşaya dönüştürebilir. Demiryoluna Yönelik Programlama (ROP) hata işlemeyi zarif ve birleştirilebilir hale getirmek için kullanılır.

Basit f fonksiyonunu düşünün:

let tryParse s =
  let b, v = System.Int32.TryParse s
  if b then Some v else None

let f (g : string option) : float option =
  match g with
  | None    -> None
  | Some s  ->
    match tryParse s with           // Parses string to int
    | None              -> None
    | Some v when v < 0 -> None     // Checks that int is greater than 0
    | Some v -> v |> float |> Some  // Maps int to float

“f"nin amacı, girdi “dize” değerini ayrıştırmaktır (“Bazı” varsa) bir ‘int’ içine. “int”, “0"dan büyükse, onu bir “kayan nokta"ya dönüştürürüz. Tümünde Diğer durumlarda ‘Yok’ ile kurtarıyoruz.

Son derece basit bir işlev olmasına rağmen, iç içe “eşleşme” okunabilirliği azaltır önemli ölçüde.

‘ROP’, programımızda iki tür yürütme yolu olduğunu gözlemliyor

  1. Mutlu yol - Sonunda “Some” değerini hesaplar
  2. Hata yolu - Diğer tüm yollar “Yok” üretir

Hata yolları daha sık olduğu için kodu devralma eğilimindedirler. Mutlu yol kodunun en görünür kod yolu olmasını isteriz.

“ROP” kullanan eşdeğer bir “g” işlevi şöyle görünebilir:

let g (v : string option) : float option =
  v
  |> Option.bind    tryParse  // Parses string to int
  |> Option.filter  ((<) 0)   // Checks that int is greater than 0
  |> Option.map     float     // Maps int to float

Listeleri ve dizileri “F#” ile işleme eğilimimize çok benziyor.

Yalnızca ‘0’ veya ‘1’ öğesi içerebilen ‘Liste<‘T>’ gibi bir ‘Seçenek<‘T>’ görülebilir Burada ‘Option.bind’, ‘List.pick’ gibi davranır (kavramsal olarak ‘Option.bind’ haritaları) ‘List.collect’ için daha iyidir, ancak ‘List.pick’in anlaşılması daha kolay olabilir).

‘bind’, ‘filter’ ve ‘map’ hata yollarını işler ve ‘g’ sadece mutlu yolları içerir. yol kodu.

Doğrudan Seçenek<_> kabul eden ve Seçenek<_> döndüren tüm işlevler |> ve >> ile doğrudan birleştirilebilir.

‘ROP’ bu nedenle okunabilirliği ve birleştirilebilirliği artırır.

Seçeneğin Tanımı #

“Seçenek”, “Hiçbiri” veya “Bazıları” olmak üzere iki durum içeren ayrımcı bir birlikteliktir.

type Option<'T> = Some of 'T | None

C#‘dan Seçenek türlerini kullanma #

C#‘ın bunları işlemenin bir yolu olmadığından, Option türlerini C# koduna maruz bırakmak iyi bir fikir değildir. Seçenekler, C# projenizde bir bağımlılık olarak “FSharp.Core"u tanıtmaktır (bu, C# ile birlikte çalışmak için * tasarlanmayan* bir F# kitaplığı kullanıyorsanız yapmanız gereken şeydir) veya " None` değerleri “null” olur.

Ön F# 4.0 #

Bunu yapmanın yolu, kendinize ait bir dönüştürme işlevi oluşturmaktır:

let OptionToObject opt =
    match opt with
    | Some x -> x 
    | None -> null

Değer türleri için, onları kutulamaya veya `System.Nullable’ kullanmaya başvurmanız gerekir.

let OptionToNullable x = 
    match x with 
    | Some i -> System.Nullable i
    | None -> System.Nullable ()

F# 4.0 #

F# 4.0’da, “ofObj”, “toObj”, “ofNullable” ve “toNullable” işlevleri, “Option” modülünde tanıtılmıştır. F# interaktif’te aşağıdaki gibi kullanılabilirler:

let l1 = [ Some 1 ; None ; Some 2]
let l2 = l1 |> List.map Option.toNullable;;

// val l1 : int option list = [Some 1; null; Some 2]
// val l2 : System.Nullable<int> list = [1; null; 2]

let l3 = l2 |> List.map Option.ofNullable;;
// val l3 : int option list = [Some 1; null; Some 2]

// Equality
l1 = l2 // fails to compile: different types
l1 = l3 // true

“None” öğesinin dahili olarak “null” olarak derlendiğini unutmayın. Ancak F# söz konusu olduğunda bu bir ‘Yok’tur.

let lA = [Some "a"; None; Some "b"]
let lB = lA |> List.map Option.toObj

// val lA : string option list = [Some "a"; null; Some "b"]
// val lB : string list = ["a"; null; "b"]

let lC = lB |> List.map Option.ofObj
// val lC : string option list = [Some "a"; null; Some "b"]

// Equality
lA = lB // fails to compile: different types
lA = lC // true
