İleri ve Geri Boru #

Boru operatörleri, parametreleri bir fonksiyona basit ve zarif bir şekilde geçirmek için kullanılır. Ara değerlerin elimine edilmesini ve fonksiyon çağrılarının daha kolay okunmasını sağlar.

F#‘da iki boru operatörü vardır:

  • İleri (|>): Parametreleri soldan sağa geçirme

      let print message =
      printf "%s" message

  // "Hello World" will be passed as a parameter to the print function
  "Hello World" |> print

  • Geri (<|): Parametreleri sağdan sola geçirme

      let print message =
      printf "%s" message

  // "Hello World" will be passed as a parameter to the print function
  print <| "Hello World"

Boru operatörleri olmayan bir örnek:

// We want to create a sequence from 0 to 10 then:
// 1 Keep only even numbers
// 2 Multiply them by 2
// 3 Print the number

let mySeq = seq { 0..10 }
let filteredSeq = Seq.filter (fun c -> (c % 2) = 0) mySeq
let mappedSeq = Seq.map ((*) 2) filteredSeq
let finalSeq = Seq.map (sprintf "The value is %i.") mappedSeq

printfn "%A" finalSeq

Önceki örneği kısaltabilir ve ileri boru operatörü ile daha temiz hale getirebiliriz:

// Using forward pipe, we can eliminate intermediate let binding
let finalSeq = 
    seq { 0..10 }
    |> Seq.filter (fun c -> (c % 2) = 0)
    |> Seq.map ((*) 2)
    |> Seq.map (sprintf "The value is %i.")

printfn "%A" finalSeq

Her fonksiyon sonucu bir sonraki fonksiyona parametre olarak iletilir.

Boru operatörüne birden fazla parametre iletmek istiyorsanız, her ek parametre için bir | eklemeniz ve parametrelerle bir Tuple oluşturmanız gerekir. Yerel F# boru operatörü, üç adede kadar parametreyi destekler (|||> veya <|||).

let printPerson name age =
    printf "My name is %s, I'm %i years old" name age

("Foo", 20) ||> printPerson

Birden fazla parametrenin işlevleri #

F#‘da tüm işlevler tam olarak bir parametre alır. Bir işlev bildiriminde birden fazla parametre bildirmek çok kolay olduğundan, bu garip bir ifade gibi görünüyor:

let add x y = x + y

Ancak bu işlev bildirimini F# etkileşimli yorumlayıcıya yazarsanız, tür imzasının şu şekilde olduğunu görürsünüz:

val add : x:int -> y:int -> int

Parametre adları olmadan, bu imza int -> int -> int olur. -> operatörü sağ ilişkiseldir, yani bu imza int -> (int -> int) ile eşdeğerdir. Başka bir deyişle, “add”, bir “int” parametresi alan ve bir “int” alan ve “int” döndüren bir işlev döndüren bir işlevdir. Dene:

let addTwo = add 2
// val addTwo : (int -> int)
let five = addTwo 3
// val five : int = 5

Ancak, “add” gibi bir işlevi daha “geleneksel” bir şekilde doğrudan iki parametre ileterek de çağırabilirsiniz ve beklediğiniz gibi çalışacaktır:

let three = add 1 2
// val three : int = 3

Bu, istediğiniz kadar parametreli işlevler için geçerlidir:

let addFiveNumbers a b c d e = a + b + c + d + e
// val addFiveNumbers : a:int -> b:int -> c:int -> d:int -> e:int -> int
let addFourNumbers = addFiveNumbers 0
// val addFourNumbers : (int -> int -> int -> int -> int)
let addThreeNumbers = addFourNumbers 0
// val addThreeNumbers : (int -> int -> int -> int)
let addTwoNumbers = addThreeNumbers 0
// val addTwoNumbers : (int -> int -> int)
let six = addThreeNumbers 1 2 3  // This will calculate 0 + 0 + 1 + 2 + 3
// val six : int = 6

Çok parametreli işlevleri bir parametre alan ve yeni işlevler döndüren işlevler olarak düşünmenin bu yöntemi (son parametreyi alan ve sonunda bir sonuç döndüren son işleve ulaşana kadar bir parametre alıp yeni işlevler döndürebilir) kavramı geliştirmesiyle ünlü matematikçi Haskell Curry’nin onuruna currying olarak adlandırılır. (Bir başkası tarafından icat edildi, ancak Curry haklı olarak bunun için kredinin çoğunu alıyor.)

Bu kavram F# boyunca kullanılır ve buna aşina olmak isteyeceksiniz.

Fonksiyonların temelleri #

F#‘daki çoğu işlev “let” sözdizimi ile oluşturulur:

let timesTwo x = x * 2

Bu, tek bir “x” parametresi alan “timesTwo” adlı bir işlevi tanımlar. Etkileşimli bir F# oturumu çalıştırırsanız (OS X ve Linux’ta ‘fsharpi’, Windows’ta ‘fsi.exe’) ve bu işlevi içine yapıştırın (ve ‘fsharpi’ye az önce yazdığınız kodu değerlendirmesini söyleyen ‘;;’ ekleyin) ), yanıt verdiğini göreceksiniz:

val timesTwo : x:int -> int

Bu, “timesTwo"nun “int” türünde tek bir “x” parametresi alan ve bir “int” döndüren bir işlev olduğu anlamına gelir. İşlev imzaları genellikle parametre adları olmadan yazılır, bu nedenle bu işlev türünün genellikle int -> int olarak yazıldığını görürsünüz.

Fakat bekle! F#, türünü hiç belirtmediğinize göre, “x"in bir tamsayı parametresi olduğunu nasıl bildi? Bunun nedeni tür çıkarımı. İşlev gövdesinde “x"i “2” ile çarptığınız için, “x” ve “2” türleri aynı olmalıdır. (Genel bir kural olarak, F# değerleri farklı türlere örtük olarak atamaz; istediğiniz herhangi bir tür tahminini açıkça belirtmelisiniz).

Herhangi bir parametre almayan bir işlev oluşturmak istiyorsanız, bunu yapmanın yanlış yolu şudur:

let hello =  // This is a value, not a function
    printfn "Hello world"

Bunu yapmanın doğru yolu:

let hello () =
    printfn "Hello world"

Bu “merhaba” işlevi, https://www.wikiod.com/tr/docs/f%23/2513/the-unit-type’da açıklanan “unit -> unit” tipine sahiptir.

Curried ve Tupled Fonksiyonlar #

F#‘da birden çok parametreli işlevleri tanımlamanın iki yolu vardır, Curried işlevler ve Tupled işlevler.

let curriedAdd x y = x + y // Signature: x:int -> y:int -> int
let tupledAdd (x, y) = x + y // Signature:  x:int * y:int -> int

F# dışından tanımlanan tüm işlevler (.NET çerçevesi gibi), F#’ta Tupled formuyla kullanılır. F# çekirdek modüllerindeki çoğu işlev Curried form ile kullanılır.

Kısmi uygulamaya izin verdiği için Curried formu deyimsel F# olarak kabul edilir. Aşağıdaki iki örnekten hiçbiri Tupled formuyla mümkün değildir.

let addTen = curriedAdd 10 // Signature: int -> int

// Or with the Piping operator
3 |> curriedAdd 7 // Evaluates to 10

Bunun nedeni, Curried fonksiyonunun bir parametre ile çağrıldığında bir fonksiyon döndürmesidir. Fonksiyonel programlamaya hoş geldiniz !!

let explicitCurriedAdd x = (fun y -> x + y) // Signature: x:int -> y:int -> int
let veryExplicitCurriedAdd = (fun x -> (fun y -> x + y)) // Same signature

Tam olarak aynı imza olduğunu görebilirsiniz.

Ancak, kitaplık yazarken olduğu gibi, diğer .NET koduyla arabirim oluştururken, Tupled formunu kullanarak işlevleri tanımlamak önemlidir.

Satır içi #

Satır içi, bir işleve yapılan çağrıyı işlevin gövdesiyle değiştirmenize olanak tanır.

Bu, bazen kodun kritik kısmında performans nedeni ile kullanışlıdır. Ancak bunun karşılığı, işlevin gövdesi bir çağrının gerçekleştiği her yerde çoğaltıldığından, derlemenizin çok fazla yer kaplamasıdır. Bir işlevi satır içi yapıp yapmamaya karar verirken dikkatli olmalısınız.

Bir işlev, “inline” anahtar sözcüğüyle satır içine alınabilir:

// inline indicate that we want to replace each call to sayHello with the body 
// of the function
let inline sayHello s1 =
    sprintf "Hello %s" s1

// Call to sayHello will be replaced with the body of the function
let s = sayHello "Foo"
// After inlining -> 
// let s = sprintf "Hello %s" "Foo"

printfn "%s" s
// Output
// "Hello Foo"

Yerel değere sahip başka bir örnek:

let inline addAndMulti num1 num2 =
    let add = num1 + num2
    add * 2

let i = addAndMulti 2 2
// After inlining ->
// let add = 2 + 2
// let i = add * 2

printfn "%i" i
// Output
// 8
