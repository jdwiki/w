CSV dosyalarından veri yükleme #

DOM öğelerine bağlayacağınız verileri almanın birkaç yolu vardır. Daha basit olanı, verilerinizi komut dosyanızda bir dizi olarak bulundurmaktır …

var data = [ ... ];

Ancak D3.js, harici bir dosyadan veri yüklememize izin verir. Bu örnekte, bir CSV dosyasındaki verilerin nasıl düzgün şekilde yükleneceğini ve bunlarla nasıl ilgilenileceğini göreceğiz.

CSV dosyaları virgülle ayrılmış değerlerdir. Bu tür dosyalarda her satır bir veri kaydıdır, her kayıt virgülle ayrılmış bir veya daha fazla alandan oluşur. Kullanmak üzere olduğumuz ‘d3.csv’ işlevinin, CSV’nin ilk satırını başlık olarak, yani alanların adlarını içeren satırı kullandığını bilmek önemlidir.

Bu nedenle, “data.csv” adlı bu CSV’yi düşünün:

city,population,area
New York,3400,210
Melbourne,1200,350
Tokyo,5200,125
Paris,800,70

“data.csv” dosyasını yüklemek için ‘d3.csv’ işlevini kullanırız. Bunu kolaylaştırmak için, “data.csv"nin betiğimizle aynı dizinde olduğunu ve onun göreli yolunun basitçe “data.csv” olduğunu varsayalım. Yani, yazıyoruz:

d3.csv("data.csv", function(data){
    //code dealing with data here
});

Geri aramada argüman olarak “data” kullandığımıza dikkat edin. Bu, D3’te yaygın bir uygulamadır, ancak başka bir ad kullanabilirsiniz.

CSV’mizde “d3.csv” ne yapar? CSV’yi bir dizi nesneye dönüştürür. Örneğin, verilerimizi “console.log” edersek:

d3.csv("data.csv", function(data){
    console.log(data)
});

Göreceğimiz şey şu:

[
    {
        "city": "New York",
        "population": "3400",
        "area": "210"
    },{
        "city": "Melbourne",
        "population": "1200",
        "area": "350"
    },{
        "city": "Tokyo",
        "population": "5200",
        "area": "125"
    },{
        "city": "Paris",
        "population": "800",
        "area": "70"
    }
]

Artık bu verileri DOM öğelerimize bağlayabiliriz.

Bu örnekte “population” ve “area"nın dizeler olduğuna dikkat edin. Ancak, muhtemelen, onlarla sayı olarak ilgilenmek istersiniz. Bunları geri çağırma içindeki bir işlevde değiştirebilirsiniz (“forEach” olarak), ancak “d3.csv"de bir “erişimci” işlevi kullanabilirsiniz:

d3.csv("data.csv", conversor, function(data){
    //code here
});

function conversor(d){
    d.population = +d.population;
    d.area = +d.area;
    return d;
}

Erişimcileri “d3.tsv” içinde de kullanabilirsiniz, ancak “d3.json"da kullanamazsınız.

Not: ‘d3.csv’ eşzamansız bir işlevdir, yani ondan sonraki kodun CSV dosyası yüklenmeden önce bile hemen yürütüleceği anlamına gelir. Bu nedenle, geri arama içinde ‘verilerinizi’ kullanmaya özellikle dikkat edin.

Geri aramada bir veya iki parametre—d3.request()’te hata işleme #

d3.request() veya kolaylık kurucularından (d3.json, d3.csv, d3.tsv, [d3] birini kullanırken .html]5 ve d3.xml) hata için birçok kaynak vardır. Ağ hataları nedeniyle verilen istekte veya yanıtında sorunlar olabilir veya içerik iyi biçimlendirilmemişse ayrıştırma başarısız olabilir.

Bu nedenle, yukarıda belirtilen yöntemlerden herhangi birine geçirilen geri aramalar içinde, bazı hata işlemelerinin uygulanması arzu edilir. Bu amaçla, geri aramalar iki argümanı kabul edebilir, birincisi hata, eğer varsa, ikincisi veridir. Yükleme veya ayrıştırma sırasında herhangi bir hata oluşursa, hatayla ilgili bilgiler, “veriler” “boş” olacak şekilde ilk “hata” argümanı olarak iletilecektir.

d3.json{"some_file.json", function(error, data) {
  if (error) throw error;   // Exceptions handling
  // Further processing if successful
});

Her halükarda, iki parametre sağlamak zorunda değilsiniz. Yalnızca bir parametre içeren bir geri arama ile istek yöntemlerini kullanmak gayet iyi. Bu tür geri aramaları işlemek için request.js’de bilginin yöntemin tek argümanına iletilme şeklini ayarlayan özel bir işlev fixCallback() vardır.

function fixCallback(callback) {
  return function(error, xhr) {
    callback(error == null ? xhr : null);
  };
}

Bu, tanım gereği veri olan yalnızca bir parametreye sahip tüm geri aramalar için D3 tarafından çağrılır.

İstek yönteminin geri çağrısına kaç parametre sağlanmış olursa olsun, “data” parametresinin kuralı şudur:

  • istek başarısız olursa, “veri” “boş” olur
  • istek başarılı olursa, “veri” yüklenen (ve ayrıştırılan) içeriği içerecektir

Tek parametreli sürüm ile iki parametreli sürüm arasındaki tek fark, Oluşabilecek hata hakkında yol bilgisi verilir. “hata” parametresi ise atlanırsa, yöntem sessizce “veri"yi “boş” olarak bırakarak başarısız olur. Öte yandan, eğer geri arama, iki argümana sahip olacak şekilde tanımlanır, yükleme veya ayrıştırma sırasındaki bir hata hakkındaki bilgiler, ilk parametreye iletilerek uygun şekilde işlemenizi sağlar.

Aşağıdaki dört “d3.json” çağrısı, bir parametre/iki parametre geri çağrısına karşı mevcut/var olmayan dosyalar için olası senaryoları gösterir:

// FAIL: resource not available or content not parsable
// error contains information about the error
// data will be null because of the error
d3.json("non_existing_file.json", function(error, data) {
  console.log("Fail, 2 parameters: ", error, data);
});

// FAIL: resource not available or content not parsable
// no information about the error
// data will be null because of the error
d3.csv("non_existing_file.json", function(data) {
  console.log("Fail, 1 parameter: ", data);
});

// OK: resource loaded successfully
// error is null
// data contains the JSON loaded from the resource
d3.json("existing_file.json", function(error, data) {
  console.log("OK, 2 parameters: ", error, data);
});

// OK: resource loaded successfully
// no information about the error; this fails silently on error
// data contains the JSON loaded from the resource
d3.json("existing_file.json", function(data) {
  console.log("OK, 1 parameter: ", data);
});
