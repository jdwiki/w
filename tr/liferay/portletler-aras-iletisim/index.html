Genel oluşturma parametresini kullanma #

Bu yaklaşım JSR 286’da tanıtıldı.

JSR 168’de, bir portletin processAction‘ında ayarlanan işleme parametreleri yalnızca o portlette kullanılabilirdi. Genel İşleme Parametreleri özelliğiyle, bir portletin processAction‘ında ayarlanan oluşturma parametreleri, diğer portletlerin oluşturulmasında da kullanılabilir. Bunu yapılandırmak için, bunu destekleyen tüm portletler için:

“” etiketini, portlet etiketi “portlet.xml” ile bitmeden hemen önce ekleyin

<security-role-ref>
    <role-name>user</role-name>
</security-role-ref>
<supported-public-render-parameter>{param-name}</supported-public-render-parameter>
</portlet>

“” etiketi sona ermeden hemen önce “” etiketini ekleyin

  <public-render-parameter>
    <identifier>{param-name}</identifier>
    <qname xmlns:x="localhost">x:{param-name}</qname>
  </public-render-parameter>
</portlet-app>

“processAction” yönteminde, yanıtta param değerinin ayarlanması gerekir.

res.setRenderParameter({param-name},{param-value});

İlgili portletin eylem aşamasını yürüttükten sonra, gerekli tüm portlet için bunu yapılandırmayı bitirdikten sonra, aynı veya farklı uygulamanın (savaş) parçası olmalarına bakılmaksızın, paragraf sayfadaki tüm destekleyici portletler için oluşturma aşamasında mevcut olmalıdır. ).

Portlet oturumunu kullanma #

Bu, JSR 168’den beri var olan bir yaklaşımdır. Portlet oturumunu kullanarak nitelikleri paylaşmamıza olanak tanır. Bir portlet oturumunun farklı türde kapsamları olabilir:

  1. Portlet kapsamı (yalnızca portlet içinde kullanılabilen nitelikler)
  2. Uygulama kapsamı (tüm uygulama[savaş] içinde mevcut nitelikler)

Bu yaklaşımı kullanmak için portlet yapılandırmasında herhangi bir giriş yapmamıza gerek yoktur, çünkü portlet oturumu portlet isteğinde hazırdır:

PortletSession session = renderRequest.getPortletSession();
session.setAttribute("attribute-name","attribute-value", PortletSession.APPLICATION_SCOPE);

veya

PortletSession session = renderRequest.getPortletSession();
session.setAttribute("attribute-name","attribute-value", PortletSession.PORTLET_SCOPE);

Öznitelik yalnızca ilgili kapsamdan alınabilir. Portlet kapsamında ayarlanan öznitelik için olduğu gibi, onu kullanarak getirmemiz gerekir.

PortletSession session = renderRequest.getPortletSession();
String attributeValue = (String) session.getAttribute("attribute-name", PortletSession.PORTLET_SCOPE);

Bu yaklaşımın en büyük sınırlaması, uygulama kapsamı dışında diğer portletler arasında paylaşımın olmamasıdır. Bunun üstesinden gelmek için, ’liferay-portlet.xml’ dosyasına ‘<private-session-attributes’> eklemek için liferay’e özel bir yaklaşım vardır.

    <private-session-attributes>false</private-session-attributes>
    <header-portlet-css>/css/main.css</header-portlet-css>
    <footer-portlet-javascript>/js/main.js</footer-portlet-javascript>
    <css-class-wrapper>{portlet-name}</css-class-wrapper>
</portlet>

özniteliklerin ayarlandığı ve alındığı tüm portletler için.

Olay oluşturma özelliğini kullanma #

Olay oluşturma mekanizması, özel nesneleri diğer portletlere geçirmek için ek özelliğe sahip, ancak olay aşamasının bir ek yükü olan, genel oluşturma parametresinin genişletilmiş bir sürümüdür.

Bunu başarmak için, bu mekanizma oluşur

  1. Yayıncı portleti
  2. Her ikisinin de farklı portlet uygulamalarının parçası olabileceği İşlemci (tüketici) portleti.

Başlamak için,

“portlet.xml” içindeki yayıncı portletine “” etiketini ekleyin

    <security-role-ref>
        <role-name>user</role-name>
    </security-role-ref>
    <supported-publishing-event>
         <qname xmlns:x="http:sun.com/events">x:Employee</qname>
    </supported-publishing-event>
  </portlet>

“portlet.xml” içindeki işlemci portletine “” etiketini ekleyin

<security-role-ref>
        <role-name>user</role-name>
    </security-role-ref>
    <supported-processing-event>
        <qname xmlns:x="http:sun.com/events">x:Employee</qname>
     </supported-processing-event>
</portlet>

Her iki portlete de “” etiketini ekleyin, olay adını tanımlayın ve “portlet.xml” yazın

<event-definition>   
  <qname xmlns:x="http:sun.com/events">x:Employee</qname>
  <value-type>com.sun.portal.portlet.users.Employee</value-type>
</event-definition>
   </portlet-app>

Ardından, olay türü için sınıf oluşturmamız gerekiyor (özel tür olması durumunda)

public class Employee implements Serializable {
  public Employee() {
  }
  private String name;
  private int userId; 

  public String getName() {
    return name;
  }

  public void setName(String name) {
    this.name = name;
  }

  public int getUserId() {
    return userId;
  }
  public void setUserId(int id)
  {
    this.userId = id;
  }

}

Şimdi, yayıncı portletinde, etkinliğin eylem aşamasında yayınlanması gerekiyor

    QName qname = new QName("http:sun.com/events" , "Employee");
    Employee emp = new Employee();
    emp.setName("Rahul");
    emp.setUserId(4567);
    res.setEvent(qname, emp);

Etkinliği yayınladık, etkinlik aşamasında yayıncı portleti tarafından işlenmesi gerekiyor.

Olay aşaması JSR 286’da tanıtıldı ve uygun olduğunda portletin oluşturma aşamasından önce yürütülür.

@ProcessEvent(qname = "{http:sun.com/events}Employee")
public void processEvent(EventRequest request, EventResponse response) {

    Event event = request.getEvent();
    if(event.getName().equals("Employee")){
      Employee payload = (Employee)event.getValue();
      response.setRenderParameter("EmpName",
      payload.getName());
    }

}

bu daha sonra oluşturma isteği aracılığıyla oluşturma parametresinden alınabilir.
