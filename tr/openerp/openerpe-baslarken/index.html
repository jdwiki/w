Debian/Ubuntu sisteminde kaynaktan yükleme #

Debian/Ubuntu’da openerp kurulumu veya kurulumu hakkında ayrıntılı talimatlar.

Kaynak koddan kurulum yapmak için Python 2.7, Git ve bir PostgreSQL veritabanına ihtiyacımız var:

$ sudo apt-get install git python-pip python2.7-dev -y
$ sudo apt-get install postgresql -y
$ sudo su -c "createuser -s $(whoami)" postgres

Ardından, Odoo tarafından kullanılan Python paketlerinin gerektirdiği bazı sistem bağımlılıklarına ihtiyacımız var:

$ sudo apt-get install libxml2-dev libxslt1-dev \
    libevent-dev libsasl2-dev libpq-dev \
    libpng12-dev libjpeg-dev \
    node-less node-clean-css -y

Python bağımlılıkları “requirements.txt” dosyasında bildirilir:

$ wget https://raw.githubusercontent.com/odoo/odoo/master/requirements.txt
$ sudo -H pip install --upgrade pip
$ sudo -H pip install -r requirements.txt

Artık GitHub’dan kaynak kodunu alabilir ve ilk Odoo örneğinizi oluşturmaya başlayabilirsiniz:

$ git clone https://github.com/odoo/odoo.git -b 9.0 --depth=1
$ ./odoo/odoo.py -d myodoo

AWS’de Odoo #

Not: #

Odoo’yu bir AWS EC2 Kapsayıcısına yüklemek için bir AWS hesabı gerekir #

Odoo’yu bir AWS EC2 Eşgörünümüne yüklemek tek tıklamayla yapılabilir, sadece buradan veya Google’da “Odoo AWS"yi arayın.

Bu biraz zaman alabilir, ancak hazır olduğunda iki şey yapmanız gerekir:

  1. Parolanızı alın
  2. Kullanıcı adınızı bulun

Şifrenizi Alma #

Amazonaws EC2 kontrol panelinizde yeni Odoo bulut sunucunuzu seçin ve en üstteki ‘Eylemler’ düğmesini tıklayın. Şunu seçin: Eylemler > Örnek Ayarları > Sistem Günlüğünü Al. Size Bitnami uygulama şifresini xxxxxxxxx olarak ayarlama yazan kısmı bulana kadar sistem günlüğünde ilerleyin. Buna benzer görünecektir:

Uygulama portalına giriş yaptıktan sonra şifrenizi değiştirdiğinizden emin olun. #

Kullanıcı Adınızı Alma #

Kullanıcı adınız, Odoo Makinesi Görüntüsünü indirdiğiniz sayfanın Kullanım Talimatları başlıklı bölümünde bulunabilir. Ancak, varsayılan e-posta genellikle kullanıcı@example.com veya admin@example.com şeklindedir.

Yüklemeden sonra #

Odoo yüklendikten sonra, kendi güvenlik grubunu da hesabınıza ekleyecek ve bu güvenlik grubunu kendisine uygulayacaktır. Varsayılan olarak, bu güvenlik grubu aşağıdaki gelen trafiğe izin verir

Limanlar Protokol Kaynak
80 tcp 0.0.0.0/0
22 tcp 0.0.0.0/0
443 tcp 0.0.0.0/0

Bu, temel HTTP ve HTTPS’nin yanı sıra SSH ve SFTP bağlantılarına izin verecektir. Bu ayarları değiştirmek isterseniz, güvenlik grubuna tıklayabilir ve inboud etiketindeki kuralları düzenleyebilirsiniz VEYA EC2’de Eylemler > Ağ Oluşturma > Güvenlik Gruplarını Değiştir’e tıklayarak yeni bir güvenlik grubu ekleyebilirsiniz. Sunucu Örneklerini Yönetin #

Odoo’yu kurduktan sonra bir sunucu örneği oluşturmamız gerekiyor. Sunucu örneği, varsayılan olarak 8060 olan belirli bir bağlantı noktasını dinleyen ve verileri depolamak için bir veritabanı kullanan bir Odoo hizmetidir.

“mydb” veritabanını kullanarak bir Odoo örneğini başlatmak için minimum komut:

$ ./odoo.py -d mydb

Veritabanı henüz mevcut değilse, Odoo otomatik olarak onu oluşturmaya çalışır ve ardından ihtiyaç duyduğu tüm veritabanı tablolarını otomatik olarak oluşturur. Hizmeti başlatan kullanıcının veritabanları oluşturma ayrıcalıkları yoksa (en iyi güvenlik uygulaması), onu oluşturmak için PostgreSQL ‘createdb’ komutunu kullanmalıyız. önceden.

Aşağıda Odoo ile çalışırken kullanılan en yaygın seçeneklere bir örnek verilmiştir:

$ ./odoo.py -d mydb --db-filter=^mydb$ --addons-path=./addons,../myaddons --xmlrpc-port=8080

Odoo’nun Nightly depoları #

Odoo, Nightly deposu’nda çeşitli işletim sistemi türleri ve Openerp/Odoo sürümleri için resmi yükleyiciler sağlar.

Seçenek 1: önceden oluşturulmuş yükleyici paketinden (Windows, Linux) manuel yükleme: #

Sisteminiz için seçtiğiniz yükleyiciyi/sürüm paketini ( Windows (.exe), Debian/Ubuntu (.deb), Redhat/CentOS(.rpm)) Odoo’s Nightly Repository‘den indirin: http://nightly .odoo.com/ adresine gidin ve yükleyin.

Seçenek 2: Standart işletim sistemi paket yönetim sistemini (yalnızca Linux) kullanarak daha fazla güncelleme almak için depo girişini ayarlayın: #

  • Debian gecelik deposunun kurulumu detaylar
    wget -O - https://nightly.odoo.com/odoo.key | apt-key add -
    echo "deb http://nightly.odoo.com/8.0/nightly/deb/ ./" >> /etc/apt/sources.list
    apt-get update && apt-get install odoo
  • RedHat gecelik deposunun kurulumu ayrıntılar
    yum-config-manager --add-repo=https://nightly.odoo.com/8.0/nightly/rpm/odoo.repo
    yum update && yum install odoo

Not: #

Örnekler, Odoo sürüm 8.0 veri havuzu bağlantıları gösterilmektedir, yukarıdaki veri havuzu URL’lerinde ‘/8.0/’ değerini Openerp v7 için ‘/7.0/’ veya Odoo v9 için ‘/9.0/’ olarak değiştirerek, sürümünüze göre bu bağlantıları farklı sürüm yüklemek üzere uyarlayın seçim
