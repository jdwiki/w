Temel Örnek #

Tensorflow, yalnızca bir derin öğrenme çerçevesinden daha fazlasıdır. Genel matematiksel işlemleri paralel ve dağıtılmış bir şekilde gerçekleştirmek için genel bir hesaplama çerçevesidir. Bunun bir örneği aşağıda açıklanmıştır.

Doğrusal Regresyon #

Yaygın olarak kullanılan ve hesaplanması oldukça basit olan temel bir istatistiksel örnek, bir veri kümesine bir satır sığdırmaktır. Tensorflow’ta bunu yapmanın yöntemi, aşağıda kod ve yorumlarda açıklanmıştır.

(TensorFlow) komut dosyasının ana adımları şunlardır:

  1. yer tutucular (x_ph, y_ph) ve değişkenler (W, b) bildirin
  2. Başlatma operatörünü (‘init’) tanımlayın
  3. Yer tutucular ve değişkenler üzerindeki işlemleri bildirin (y_pred, loss, train_op)
  4. Bir oturum oluşturun (sess)
  5. Başlatma operatörünü çalıştırın (sess.run(init))
  6. Bazı grafik işlemlerini çalıştırın (örn. sess.run([train_op, loss], feed_dict={x_ph: x, y_ph: y}))

Grafik yapısı Python TensorFlow API kullanılarak yapılır (C++ TensorFlow API kullanılarak da yapılabilir). Grafiğin çalıştırılması, düşük seviyeli C++ rutinlerini çağıracaktır.

'''
function: create a linear model which try to fit the line 
          y = x + 2 using SGD optimizer to minimize 
          root-mean-square(RMS) loss function

'''
import tensorflow as tf
import numpy as np

# number of epoch
num_epoch = 100

# training data x and label y
x = np.array([0., 1., 2., 3.], dtype=np.float32)
y = np.array([2., 3., 4., 5.], dtype=np.float32)

# convert x and y to 4x1 matrix
x = np.reshape(x, [4, 1])
y = np.reshape(y, [4, 1])

# test set(using a little trick)
x_test = x + 0.5
y_test = y + 0.5

# This part of the script builds the TensorFlow graph using the Python API

# First declare placeholders for input x and label y
# Placeholders are TensorFlow variables requiring to be explicitly fed by some 
# input data
x_ph = tf.placeholder(tf.float32, shape=[None, 1])
y_ph = tf.placeholder(tf.float32, shape=[None, 1])

# Variables (if not specified) will be learnt as the GradientDescentOptimizer
# is run
# Declare weight variable initialized using a truncated_normal law
W = tf.Variable(tf.truncated_normal([1, 1], stddev=0.1))
# Declare bias variable initialized to a constant 0.1
b = tf.Variable(tf.constant(0.1, shape=[1]))

# Initialize variables just declared 
init = tf.initialize_all_variables()

# In this part of the script, we build operators storing operations
# on the previous variables and placeholders.
# model: y = w * x + b
y_pred = x_ph * W + b

# loss function
loss = tf.mul(tf.reduce_mean(tf.square(tf.sub(y_pred, y_ph))), 1. / 2)
# create training graph
train_op = tf.train.GradientDescentOptimizer(0.1).minimize(loss)

# This part of the script runs the TensorFlow graph (variables and operations
# operators) just built.
with tf.Session() as sess:
    # initialize all the variables by running the initializer operator
    sess.run(init)
    for epoch in xrange(num_epoch):
        # Run sequentially the train_op and loss operators with
        # x_ph and y_ph placeholders fed by variables x and y
        _, loss_val = sess.run([train_op, loss], feed_dict={x_ph: x, y_ph: y})
        print('epoch %d: loss is %.4f' % (epoch, loss_val))

    # see what model do in the test set
    # by evaluating the y_pred operator using the x_test data
    test_val = sess.run(y_pred, feed_dict={x_ph: x_test})
    print('ground truth y is: %s' % y_test.flatten())
    print('predict y is     : %s' % test_val.flatten())

10’a kadar sayma #

Bu örnekte 10’a kadar saymak için Tensorflow’u kullanıyoruz. Evet bu tamamen abartı, ancak Tensorflow’u kullanmak için gereken mutlak minimum kurulumu göstermek için güzel bir örnek

import tensorflow as tf

# create a variable, refer to it as 'state' and set it to 0
state = tf.Variable(0)

# set one to a constant set to 1
one = tf.constant(1)

# update phase adds state and one and then assigns to state
addition = tf.add(state, one)
update = tf.assign(state, addition )

# create a session
with tf.Session() as sess:
  # initialize session variables
  sess.run( tf.global_variables_initializer() )

  print "The starting state is",sess.run(state)

  print "Run the update 10 times..."
  for count in range(10):
    # execute the update
    sess.run(update)

  print "The end state is",sess.run(state)

Burada fark edilmesi gereken en önemli şey, durum, bir, toplama ve güncelleme’nin aslında değer içermemesidir. Bunun yerine, Tensorflow nesnelerine referanslardır. Nihai sonuç durum değil, bunun yerine sess.run(durum) kullanılarak değerlendirmek için bir Tensorflow kullanılarak alınır.

Bu örnek https://github.com/panchishin/learn-to-tensorflow adresinden alınmıştır. Orada birkaç başka örnek ve python’da Tensorflow grafiğini manipüle etmeyi öğrenmek için güzel bir kademeli öğrenme planı var.

Kurulum veya Kurulum #

Tensorflow sürüm 1.0’dan itibaren kurulumun gerçekleştirilmesi çok daha kolay hale geldi. En azından TensorFlow’u kurmak için, makinelerine en az 2.7 veya 3.3+ python sürümüyle kurulu pip gerekir.

pip install --upgrade tensorflow      # for Python 2.7
pip3 install --upgrade tensorflow     # for Python 3.n

Bir GPU makinesinde tensorflow için (1.0’dan itibaren CUDA 8.0 ve cudnn 5.1 gerektirir, AMD GPU desteklenmez)

pip install --upgrade tensorflow-gpu  # for Python 2.7 and GPU
pip3 install --upgrade tensorflow-gpu # for Python 3.n and GPU

Çalışıp çalışmadığını test etmek için python 2 veya 3’ün doğru sürümünü açın ve çalıştırın

import tensorflow

Bu hatasız başarılı olduysa, makinenizde tensorflow yüklüdür.

Tensorflow Temelleri #

Tensorflow, veri akışı grafikleri ilkesine göre çalışır. Bazı hesaplamaları gerçekleştirmek için iki adım vardır:

  1. Hesaplamayı bir grafik olarak gösterin.
  2. Grafiği yürütün.

Temsil: Herhangi bir yönlendirilmiş grafik gibi, bir Tensorflow grafiği de düğümlerden ve yönlü kenarlardan oluşur.

Düğüm: Bir Düğüm ayrıca Op (operasyon anlamına gelir) olarak da adlandırılır. Bir düğümün birden çok gelen kenarı olabilir, ancak tek bir giden kenarı olabilir.

Kenar: Bir Düğümden gelen veya giden verileri belirtin. Bu durumda bazı Node(Op) giriş(ler)i ve çıkışı.

Veri dediğimizde, Tensör olarak bilinen n-boyutlu bir vektörü kastediyoruz. Bir Tensörün üç özelliği vardır: Rank, Shape ve Type

  • Rütbe, Tensörün boyut sayısı anlamına gelir (bir küp veya kutunun sıralaması 3’tür).
  • Şekil, bu boyutların değerleri anlamına gelir (kutu 1x1x1 veya 2x5x7 şeklinde olabilir).
  • Tür, Tensörün her koordinatındaki veri türü anlamına gelir.

Uygulamak: Bir grafik oluşturulmuş olsa bile, yine de soyut bir varlıktır. Biz çalıştırana kadar hiçbir hesaplama gerçekleşmez. Bir grafiği çalıştırmak için, grafik içindeki Ops’a CPU kaynağı ayırmamız gerekiyor. Bu, Tensorflow Oturumları kullanılarak yapılır. Adımlar:

  1. Yeni bir oturum oluşturun.
  2. Grafiğin içindeki herhangi bir Op’u çalıştırın. Genellikle, hesaplamamızın çıktısını beklediğimiz son Operasyonu çalıştırırız.

Bir Operasyona gelen uç, başka bir Operasyondaki verilere bağımlılık gibidir. Böylece herhangi bir Op çalıştırdığımızda, üzerine gelen tüm kenarlar izlenir ve diğer taraftaki oplar da çalıştırılır.

Not: Veri kaynağı veya havuz rolü oynayan özel düğümler de mümkündür. Örneğin, sabit bir değer veren ve dolayısıyla hiçbir gelen kenar olmayan (aşağıdaki örnekte ‘matris1’ değerine bakın) ve benzer şekilde, sonuçların toplandığı giden kenarları olmayan bir Op’a sahip olabilirsiniz (aşağıdaki örnekte ‘ürün’ değerine bakın).

Örnek:

import tensorflow as tf

# Create a Constant op that produces a 1x2 matrix.  The op is
# added as a node to the default graph.
#
# The value returned by the constructor represents the output
# of the Constant op.
matrix1 = tf.constant([[3., 3.]])

# Create another Constant that produces a 2x1 matrix.
matrix2 = tf.constant([[2.],[2.]])

# Create a Matmul op that takes 'matrix1' and 'matrix2' as inputs.
# The returned value, 'product', represents the result of the matrix
# multiplication.
product = tf.matmul(matrix1, matrix2)

# Launch the default graph.
sess = tf.Session()

# To run the matmul op we call the session 'run()' method, passing 'product'
# which represents the output of the matmul op.  This indicates to the call
# that we want to get the output of the matmul op back.
#
# All inputs needed by the op are run automatically by the session.  They
# typically are run in parallel.
#
# The call 'run(product)' thus causes the execution of three ops in the
# graph: the two constants and matmul.
#
# The output of the op is returned in 'result' as a numpy `ndarray` object.
result = sess.run(product)
print(result)
# ==> [[ 12.]]

# Close the Session when we're done.
sess.close()
