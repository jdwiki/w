Angular-cli aracı, bir açısal uygulamanın farklı bölümlerini (bileşenler, yönergeler, borular, hizmetler, sınıflar, korumalar, arabirimler, numaralandırmalar ve modüller) iskele etmenize yardımcı olabilir.

Sözdizimi #

  • oluşturma [bileşen | direktif | hizmet | boru | sınıf | numaralandırma | arayüz | koruma | modül] [ad] [bayraklar…]
  • ng g [c | gün | s | p | cl | e | ben | g | m] [isim] [bayraklar…]

Parametreler #

parametre Tanım
“bileşen” veya “c” Bileşen oluşturmak için kullanılır
“yönerge” veya “d” Yönerge oluşturmak için kullanılır
‘hizmet’ veya ’s’ Hizmet üretmek için kullanılır
‘boru’ veya ‘p’ Boru oluşturmak için kullanılır
‘sınıf’ veya ‘cl’ Sınıfları oluşturmak için kullanılır
“enum” veya “e” Numaralandırma oluşturmak için kullanılır
‘arayüzler’ veya ‘i’ Arayüz oluşturmak için kullanılır
‘koruyucu’ veya ‘g’ Muhafızlar oluşturmak için kullanılır
“modül” veya “m” Modül oluşturmak için kullanılır
‘–flat’ veya ‘-f’ Dizin oluşturmayı etkinleştirmek/devre dışı bırakmak için kullanılır
‘–inline-template’ veya ‘-it’ Bileşenlerde satır içi html şablonlarını etkinleştirmek/devre dışı bırakmak için kullanılır
‘–inline-style’ veya ‘-is’ Bileşenlerde satır içi stilleri etkinleştirmek/devre dışı bırakmak için kullanılır
--prefix veya -p Öneki devre dışı bırakmak veya değiştirmek için kullanılır
‘–spec’ veya ‘-s’ “.spec” dosyalarının oluşturulmasını etkinleştirmek/devre dışı bırakmak için kullanılır
`–atla-içe aktar’ Modül içe aktarımını atlamak için kullanılır
--app veya -a Kullanılacak uygulama adını belirtmek için kullanılır
“–modül” veya “-m” Bildirme modülünü belirtmek için kullanılır
--view-encapsulation veya -ve Bileşenlerde görünüm kapsülleme stratejisini belirtmek için kullanılır
`–değişiklik-algılama’ veya ‘-cd’ Bileşenlerde değişiklik algılama stratejisini belirtmek için kullanılır
--yönlendirme veya -r Yönlendirme modülü dosyasının oluşturulup oluşturulmayacağını belirtmek için kullanılır

“Komut oluştur” kullanımı #

Açısal yapı taşları (bileşenler, hizmetler, borular, vb.) oluşturmak için ’ng üret’ veya ’ng g’ komutunu kullanabilirsiniz.

Tüm olası planları aşağıdaki tabloda bulabilirsiniz:

İskele Kullanım kısaltılmış
Bileşen ‘bileşen bileşeni adı oluşturma’ “g c bileşen-adı”
Direktif `yönetim yönergesi-adı oluştur’ `ng g d yönerge adı’
boru `boru-adı üret’ `ng g p boru adı’
Hizmet hizmet servis adı oluştur ng g s hizmet adı
sınıf `sınıf sınıf-adı oluştur’ ng g cl sınıf adı
Muhafız koruyucu-adı oluştur `ng g g koruyucu-adı’
arayüz arayüz arayüzü-adı oluştur ng g i arayüz-adı
Numara enum numaralandırma-adı oluştur ng g e enum-adı
Modül `modül modül adı oluşturma’ “g m modül-adı”

Bu nedenle, örneğin, “bileşen kullanıcı listesi oluştur” komutunu çalıştırırsanız - angular-cli:

  • komutu çalıştırdığınız src/app klasöründe veya klasöründe user-list dizini oluşturun.
  • bu dizinin içinde 4 dosya (‘user-list.component.ts’, ‘user-list.component.html’, ‘user-list.component.css’ ve ‘user-list.component.spec.ts’) oluşturun
  • en yakın modülün ‘@NgModule’ dekoratörüne bildirim olarak ‘user-list’ ekleyin.

Hizmet oluşturma #

“UserService” adlı bir hizmet eklemek için şunu çalıştırın:

  $ ng g s user

  installing service
    create src/app/user.service.spec.ts
    create src/app/user.service.ts
  • “.spec” dosyalarının oluşturulmasını önlemek için “–spec false” veya “-sp false” bayrağı ekleyin
  $ ng g s user --spec false
  
  installing service
    create src/app/user.service.ts
  • Klasör oluşturmayı etkinleştirmek için “–flat false” veya “-f false” bayrağı ekleyin
  $ ng g s user --flat false

  installing service
    create src/app/user/user.service.spec.ts
    create src/app/user/user.service.ts

Yukarıda listelenen bayrakları da birleştirebilirsiniz. Örneğin, “.spec” dosyası olmadan “user” klasörü içinde yalnızca “user.service.ts” dosyasını oluşturmak için aşağıdaki komutu kullanın.

  $ ng g s user -f false -sp false

  installing service
    create src/app/user/user.service.ts

Tüm “hizmet oluştur” işaretleri:

Açıklama bayrak Kısaltılmış Varsayılan değer
Klasör oluşturmayı etkinleştir `–düz yanlış’ `-f yanlış’ ‘doğru’
.spec dosyalarının oluşturulmasını engelle `–spec yanlış’ `-sp yanlış’ ‘doğru’

Modül oluşturma #

“GuestModule” adlı bir modül eklemek için şunu çalıştırın:

  $ ng g m guest

  installing module
    create src/app/guest/guest.module.ts
  • ‘.spec’ dosyaları oluşturmayı etkinleştirmek için ‘–spec’ veya ‘-sp’ bayrağı ekleyin
  $ ng g m guest --spec

  installing module
    create src/app/guest/guest.module.spec.ts
    create src/app/guest/guest.module.ts
  • Yönlendirmeyi etkinleştirmek için “–routing” veya “-r” bayrağı ekleyin
  $ ng g m guest --routing

  installing module
    create src/app/guest/guest-routing.module.ts
    create src/app/guest/guest.module.ts

Yukarıda listelenen bayrakları da birleştirebilirsiniz. Örneğin, yönlendirme ve özelliklere sahip modül oluşturmak için aşağıdaki komutu kullanın.

  $ ng g m guest -sp -r
  
  installing module
    create src/app/guest/guest-routing.module.ts
    create src/app/guest/guest.module.spec.ts
    create src/app/guest/guest.module.ts

Tüm “modül oluştur” işaretleri:

Açıklama bayrak Kısaltılmış Varsayılan değer
.spec dosyalarının oluşturulmasını etkinleştir --spec -sp “yanlış”
Yönlendirmeyi etkinleştir --yönlendirme -r “yanlış”

Bileşen oluşturma #

‘[prefix]-user-list’ seçicili bir bileşen eklemek için şunu çalıştırın:

  $ ng g c user-list
  
  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts

önek, diğer uygulamalardaki bileşenlerle ve yerel HTML öğeleriyle öğe adı çakışmalarını önler. Bu nedenle, örneğin, önek “app” ise - oluşturulan bileşende “app-user-list” seçicisi olacaktır.

  • Önek kullanımını önlemek için “–prefix false” veya “-p false” bayrağı ekleyin
  $ ng g c user-list --prefix false

import { Component } from '@angular/core';

@Component({
  selector: 'user-list',
  templateUrl: './user-list.component.html',
  styleUrls: ['./user-list.component.css']
})
export class UserListComponent {}
  • “.spec” dosyalarının oluşturulmasını önlemek için “–spec false” veya “-sp false” bayrağı ekleyin
  $ ng g c user-list --spec false

  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Harici şablonlar yerine satır içi html şablonlarını kullanmak için “–inline-template” veya “-it” bayrağı ekleyin
  $ ng g c user-list --inline-template

  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Harici stiller yerine satır içi stilleri kullanmak için “–inline-style” veya “-is” bayrağı ekleyin
  $ ng g c user-list --inline-style

  installing component
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Klasör oluşturmayı önlemek için “–flat” veya “-f” bayrağı ekleyin
  $ ng g c user-list --flat
  
  installing component
    create src/app/user-list.component.css
    create src/app/user-list.component.html
    create src/app/user-list.component.spec.ts
    create src/app/user-list.component.ts
    update src/app/app.module.ts

Yukarıda listelenen bayrakları da birleştirebilirsiniz. Örneğin, ‘.css’, ‘.html’, ‘.spec’ dosyaları ve klasörü olmadan yalnızca ‘.component.ts’ dosyası oluşturmak için aşağıdaki komutu kullanın.

  $ ng g c user-list -f -it -is -sp false
  
  installing component
    create src/app/user-list.component.ts
    update src/app/app.module.ts

Tüm “bileşen oluştur” bayrakları:

Açıklama bayrak Kısaltılmış Varsayılan değer
Klasör oluşturmayı engelle `–düz’ -f “yanlış”
Ön ek kullanımını engelle `–ön ek yanlış’ `-p yanlış’ ‘doğru’
.spec dosyalarının oluşturulmasını engelle `–spec yanlış’ `-sp yanlış’ ‘doğru’
Satır içi html şablonlarını etkinleştir `–inline-şablon’ -it “yanlış”
Satır içi stilleri etkinleştir `–inline-style’ -is “yanlış”

Yönerge oluşturma #

‘[prefix]Vurgulama’ seçicili bir yönerge eklemek için şunu çalıştırın:

  $ ng g d highlight

  installing directive
    create src/app/highlight.directive.spec.ts
    create src/app/highlight.directive.ts
    update src/app/app.module.ts
  • Önek kullanımını önlemek için “–prefix false” veya “-p false” bayrağı ekleyin
  $ ng g d highlight --prefix false

import { Directive } from '@angular/core';

@Directive({
  selector: '[highlight]'
})
export class HighlightDirective {}
  • “.spec” dosyalarının oluşturulmasını önlemek için “–spec false” veya “-sp false” bayrağı ekleyin
  $ ng g d highlight --spec false

  installing directive
    create src/app/highlight.directive.ts
    update src/app/app.module.ts
  • Klasör oluşturmayı etkinleştirmek için “–flat false” veya “-f false” bayrağı ekleyin
  $ ng g d highlight --flat false
  
  installing directive
    create src/app/highlight/highlight.directive.spec.ts
    create src/app/highlight/highlight.directive.ts
    update src/app/app.module.ts

Yukarıda listelenen bayrakları da birleştirebilirsiniz. Örneğin, ‘.spec’ dosyası olmadan ‘highlight’ klasörü içinde sadece ‘highlight.directive.ts’ dosyası oluşturmak için aşağıdaki komutu kullanın.

  $ ng g d highlight -f false -sp false

  installing directive
    create src/app/highlight/highlight.directive.ts
    update src/app/app.module.ts

Tüm “yönerge oluştur” işaretleri:

Açıklama bayrak Kısaltılmış Varsayılan değer
Klasör oluşturmayı etkinleştir `–düz yanlış’ `-f yanlış’ ‘doğru’
Ön ek kullanımını engelle `–ön ek yanlış’ `-p yanlış’ ‘doğru’
.spec dosyalarının oluşturulmasını engelle `–spec yanlış’ `-sp yanlış’ ‘doğru’

Boru üretme #

“searchByName” adlı bir boru eklemek için şunu çalıştırın:

  $ ng g p search-by-name

  installing pipe
    create src/app/search-by-name.pipe.spec.ts
    create src/app/search-by-name.pipe.ts
    update src/app/app.module.ts
  • “.spec” dosyalarının oluşturulmasını önlemek için “–spec false” veya “-sp false” bayrağı ekleyin
  $ ng g p search-by-name --spec false
  
  installing pipe
    create src/app/search-by-name.pipe.ts
    update src/app/app.module.ts
  • Klasör oluşturmayı etkinleştirmek için “–flat false” veya “-f false” bayrağı ekleyin
  $ ng g p search-by-name --flat false
  
  installing pipe
    create src/app/search-by-name/search-by-name.pipe.spec.ts
    create src/app/search-by-name/search-by-name.pipe.ts
    update src/app/app.module.ts

Yukarıda listelenen bayrakları da birleştirebilirsiniz. Örneğin, ‘.spec’ dosyası olmadan sadece ‘ada göre arama.pipe.ts’ dosyası oluşturmak için, ‘ada göre arama’ klasöründe aşağıdaki komutu kullanın.

  $ ng g p search-by-name -f false -sp false

  installing pipe
    create src/app/search-by-name/search-by-name.pipe.ts
    update src/app/app.module.ts

Tüm “boru oluştur” bayrakları:
