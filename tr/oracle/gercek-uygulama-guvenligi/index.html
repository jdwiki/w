Gerçek Uygulama Güvenliği

  • Başvuru

Bu sayfada

  • Başvuru

Oracle Real Application Security, Oracle 12c’de tanıtıldı. Kullanıcı Rol Modeli, Erişim Kontrolü, Uygulamaya Karşı Veritabanı, Son Kullanıcı Güvenliği veya Satır ve Sütun Düzeyinde Güvenlik gibi birçok Güvenlik Konusunu özetler.

Başvuru #

Bir Uygulamayı Veritabanındaki bir şeyle ilişkilendirmek için üç ana bölüm vardır:

Uygulama Ayrıcalığı: Bir Uygulama Ayrıcalığı, ‘SELECT’, ‘INSERT’, ‘UPDATE’, ‘DELETE’, … gibi Ayrıcalıkları tanımlar. Uygulama Ayrıcalıkları, Toplam Ayrıcalık olarak özetlenebilir.

XS$PRIVILEGE(
    name=>'privilege_name'
    [, implied_priv_list=>XS$NAME_LIST('"SELECT"', '"INSERT"', '"UPDATE"', '"DELETE"')]
)

XS$PRIVILEGE_LIST(
    XS$PRIVILEGE(...),
    XS$PRIVILEGE(...),
    ...
);

Uygulama Kullanıcısı:

Basit Uygulama Kullanıcısı:

BEGIN 
    SYS.XS_PRINCIPAL.CREATE_USER('user_name'); 
END;

Doğrudan Giriş Uygulaması Kullanıcısı:

BEGIN 
    SYS.XS_PRINCIPAL.CREATE_USER(name => 'user_name', schema => 'schema_name');
END;

BEGIN 
    SYS.XS_PRINCIPAL.SET_PASSWORD('user_name', 'password'); 
END;
CREATE PROFILE prof LIMIT 
    PASSWORD_REUSE_TIME 1/4440 
    PASSWORD_REUSE_MAX 3 
    PASSWORD_VERIFY_FUNCTION Verify_Pass;

BEGIN 
    SYS.XS_PRINCIPAL.SET_PROFILE('user_name', 'prof'); 
END;

BEGIN 
    SYS.XS_PRINCIPAL.GRANT_ROLES('user_name', 'XSONNCENT');
END;

(isteğe bağlı:)

BEGIN 
    SYS.XS_PRINCIPAL.SET_VERIFIER('user_name', '6DFF060084ECE67F', XS_PRINCIPAL.XS_SHA512“); 
END;

Uygulama Rolü:

Düzenli Uygulama Rolü:

DECLARE
    st_date TIMESTAMP WITH TIME ZONE;
    ed_date TIMESTAMP WITH TIME ZONE;
BEGIN
    st_date := SYSTIMESTAMP;
    ed_date := TO_TIMESTAMP_TZ('2013-06-18 11:00:00 -5:00','YYYY-MM-DD HH:MI:SS');
    SYS.XS_PRINCIPAL.CREATE_ROLE
        (name => 'app_regular_role',
        enabled => TRUE,
        start_date => st_date,
        end_date => ed_date);
END;

Dinamik Uygulama Rolü: (kimlik doğrulama durumuna göre dinamik olarak etkinleştirilir)

BEGIN
    SYS.XS_PRINCIPAL.CREATE_DYNAMIC_ROLE
        (name => 'app_dynamic_role',
        duration => 40,
        scope => XS_PRINCIPAL.SESSION_SCOPE);
END;

Önceden Tanımlanmış Uygulama Rolleri:

Düzenli:

  • “XSPUBLIC”
  • “XSBYPASS”
  • XSSESSIONADMIN
  • XSNAMESPACEADMIN
  • “XS KOMİSYONLARI”
  • “XSCACHEADMIN”
  • “XSDISPATCHER”

Dinamik: (uygulama kullanıcısının kimlik doğrulama durumuna bağlıdır)

  • DBMS_AUTH: (doğrudan oturum açma veya diğer veritabanı kimlik doğrulama yöntemi)
  • EXTERNAL_DBMS_AUTH: (doğrudan oturum açma veya başka bir veritabanı doğrulama yöntemi ve kullanıcı haricidir)
  • DBMS_PASSWD: (parola ile doğrudan oturum açma)
  • MIDTIER_AUTH: (orta seviye uygulama aracılığıyla kimlik doğrulama)
  • XSAUTHENTICATED: (doğrudan veya orta seviye uygulama)
  • XSSWITCH: (kullanıcı proxy kullanıcısından uygulama kullanıcısına geçti)
