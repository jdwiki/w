Tarih Aritmetiği - Gün, Saat, Dakika ve/veya Saniye Olarak Tarihler Arasındaki Fark #

Oracle’da, iki “TARİH” arasındaki fark (gün ve/veya kesir olarak) çıkarma işlemi kullanılarak bulunabilir:

SELECT DATE '2016-03-23' - DATE '2015-12-25' AS difference FROM DUAL;

İki tarih arasındaki gün sayısını verir:

DIFFERENCE
----------
        89

Ve:

SELECT TO_DATE( '2016-01-02 01:01:12', 'YYYY-MM-DD HH24:MI:SS' )
         - TO_DATE( '2016-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS' )
         AS difference
FROM   DUAL

İki tarih arasındaki günlerin kesirini verir:

DIFFERENCE
----------
    1.0425

Saat, dakika veya saniye cinsinden fark, bu sayı sırasıyla ‘24’, ‘2460’ veya ‘2460*60’ ile çarpılarak bulunabilir.

Önceki örnek, aşağıdakiler kullanılarak iki tarih arasındaki günleri, saatleri, dakikaları ve saniyeleri alacak şekilde değiştirilebilir:

SELECT TRUNC( difference                       ) AS days,
       TRUNC( MOD( difference * 24,       24 ) ) AS hours,
       TRUNC( MOD( difference * 24*60,    60 ) ) AS minutes,
       TRUNC( MOD( difference * 24*60*60, 60 ) ) AS seconds
FROM   (
  SELECT TO_DATE( '2016-01-02 01:01:12', 'YYYY-MM-DD HH24:MI:SS' )
         - TO_DATE( '2016-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS' )
         AS difference
  FROM   DUAL

);

(Not: TRUNC() [FLOOR()] yerine kullanılır](https: //docs.oracle.com/cd/B19306_01/server.102/b14200/functions058.htm) negatif farkları doğru şekilde işlemek için.)

Çıktılar:

DAYS HOURS MINUTES SECONDS
---- ----- ------- -------
   1     1       1      12

Önceki örnek, sayısal farkı bir aralığa dönüştürerek de NUMTODSINTERVAL()() kullanılarak çözülebilir. :

SELECT EXTRACT( DAY    FROM difference ) AS days,
       EXTRACT( HOUR   FROM difference ) AS hours,
       EXTRACT( MINUTE FROM difference ) AS minutes,
       EXTRACT( SECOND FROM difference ) AS seconds
FROM   (
  SELECT NUMTODSINTERVAL(
           TO_DATE( '2016-01-02 01:01:12', 'YYYY-MM-DD HH24:MI:SS' )
             - TO_DATE( '2016-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS' ),
           'DAY'
         ) AS difference
  FROM   DUAL
);

Varsayılan Tarih Formatı Modelini Ayarlama #

Oracle örtük olarak bir “DATE"den bir dizgeye veya tam tersine dönüştürdüğünde (veya TO_CHAR() ) veya TO_DATE() bir biçim modeli olmadan açıkça çağrılır) NLS_DATE_FORMAT oturum parametresi dönüştürmede biçim modeli olarak kullanılır. Hazır bilgi, biçim modeliyle eşleşmiyorsa, bir istisna oluşturulacaktır.

Bu parametreyi aşağıdakileri kullanarak inceleyebilirsiniz:

SELECT VALUE FROM NLS_SESSION_PARAMETERS WHERE PARAMETER = 'NLS_DATE_FORMAT';

Bu değeri mevcut oturumunuzda aşağıdakileri kullanarak ayarlayabilirsiniz:

ALTER SESSION SET NLS_DATE_FORMAT = 'YYYY-MM-DD HH24:MI:SS';

(Not: Bu, diğer kullanıcılar için değeri değiştirmez.)

‘TO_DATE()’ veya ‘TO_CHAR()’ içinde biçim maskesi sağlamak için ‘NLS_DATE_FORMAT’a güveniyorsanız, bu değer hiç değiştirilirse sorgularınız bozulduğunda şaşırmamalısınız.

Tarih Aritmetiği - Aylar veya Yıllar Olarak Tarihler Arasındaki Fark #

İki tarih arasındaki ay farkı, MONTHS_BETWEEN( date1, date2 ) kullanılarak bulunabilir:

SELECT MONTHS_BETWEEN( DATE '2016-03-10', DATE '2015-03-10' ) AS difference FROM DUAL;

Çıktılar:

DIFFERENCE
----------
        12

Aradaki fark ayları içeriyorsa, her ay 31 gün olmasına bağlı olarak ayın kesirini döndürür:

SELECT MONTHS_BETWEEN( DATE '2015-02-15', DATE '2015-01-01' ) AS difference FROM DUAL;

Çıktılar:

DIFFERENCE
----------
 1.4516129

MONTHS_BETWEEN, ayda daha az gün olabileceği ayda 31 gün varsayıldığından, bu, aylar arasındaki sınırları kapsayan farklar için farklı değerlere neden olabilir.

Örnek:

SELECT MONTHS_BETWEEN( DATE'2016-02-01', DATE'2016-02-01' - INTERVAL '1' DAY ) AS "JAN-FEB",
       MONTHS_BETWEEN( DATE'2016-03-01', DATE'2016-03-01' - INTERVAL '1' DAY ) AS "FEB-MAR",
       MONTHS_BETWEEN( DATE'2016-04-01', DATE'2016-04-01' - INTERVAL '1' DAY ) AS "MAR-APR",
       MONTHS_BETWEEN( DATE'2016-05-01', DATE'2016-05-01' - INTERVAL '1' DAY ) AS "APR-MAY"
FROM   DUAL;

Çıktı:

JAN-FEB FEB-MAR MAR-APR APR-MAY
------- ------- ------- -------
0.03226 0.09677 0.03226 0.06452

Yıl farkı, ay farkını 12’ye bölerek bulunabilir.

Zaman Bileşeni Olmadan Tarih Oluşturma #

Tüm “DATE"lerin bir zaman bileşeni vardır; bununla birlikte, saat/dakika/saniye sıfıra ayarlanmış (yani gece yarısı) zaman bilgisi içermesi gerekmeyen tarihlerin saklanması gelenekseldir.

Bir ANSI “DATE” değişmezi kullanın (ISO 8601 Tarih biçimini .wikipedia.org/wiki/ISO_8601#Calendar_dates)):

SELECT DATE '2000-01-01' FROM DUAL;

TO_DATE() kullanarak bir dize değişmezinden dönüştürün:

SELECT TO_DATE( '2001-01-01', 'YYYY-MM-DD' ) FROM DUAL;

(tarih formatı modelleri hakkında daha fazla bilgiyi Oracle belgelerinde bulabilirsiniz.)

veya:

SELECT TO_DATE(
         'January 1, 2000, 00:00 A.M.',
         'Month dd, YYYY, HH12:MI A.M.',
         'NLS_DATE_LANGUAGE = American'
       )
FROM   DUAL;

(Ay isimleri gibi dile özgü terimleri dönüştürüyorsanız, o zaman 3. nlsparam parametresini TO_DATE() işlevine dahil etmek ve beklenen dili belirtmek iyi bir uygulamadır.)

Zaman Bileşeniyle Tarih Oluşturma #

TO_DATE() kullanarak bir dize değişmezinden dönüştürün:

SELECT TO_DATE( '2000-01-01 12:00:00', 'YYYY-MM-DD HH24:MI:SS' ) FROM DUAL;

Veya bir TIMESTAMP değişmezi kullanın:

CREATE TABLE date_table(
  date_value DATE
);

INSERT INTO date_table ( date_value ) VALUES ( TIMESTAMP '2000-01-01 12:00:00' );

Oracle, bir tablonun bir “DATE” sütununda saklarken, örtük olarak bir “TARİH"e bir “TIMESTAMP” atar; ancak değeri açıkça bir “DATE” olarak CAST() yapabilirsiniz:

SELECT CAST( TIMESTAMP '2000-01-01 12:00:00' AS DATE ) FROM DUAL;

Tarih Biçimi #

Oracle’da bir ‘TARİH’ veri türünün bir formatı yoktur; Oracle, istemci programına (SQL/Plus, SQL/Developer, Toad, Java, Python, vb.) bir `TARİH’ gönderdiğinde, tarihi temsil eden 7- veya 8- bayt gönderir.

Bir tabloda depolanmayan bir “DATE” (yani “SYSDATE” tarafından oluşturulan ve “DUMP()” komutu kullanılırken “type 13” olan) 8 bayta sahiptir ve yapıya sahiptir (sağdaki sayılar “2012-11-26 16:41:09” dahili temsili):

BYTE VALUE                           EXAMPLE
---- ------------------------------- --------------------------------------
1    Year modulo 256                 220
2    Year multiples of 256           7   (7 * 256 + 220 = 2012)
3    Month                           11
4    Day                             26
5    Hours                           16
6    Minutes                         41
7    Seconds                         9
8    Unused                          0

Bir tabloda saklanan bir “TARİH” (“DUMP()” komutunu kullanırken “tip 12”) 7 bayta sahiptir ve yapıya sahiptir (sağdaki sayılar “2012-11-26"nın dahili temsilidir 16:41:09`):

BYTE VALUE                           EXAMPLE
---- ------------------------------- --------------------------------------
1    ( Year multiples of 100 ) + 100 120
2    ( Year modulo 100 ) + 100       112 ((120-100)*100 + (112-100) = 2012)
3    Month                           11
4    Day                             26
5    Hours + 1                       17
6    Minutes + 1                     42
7    Seconds + 1                     10

Tarihin belirli bir biçime sahip olmasını istiyorsanız, onu biçimi olan bir şeye (yani bir dizeye) dönüştürmeniz gerekir. SQL istemcisi bunu dolaylı olarak yapabilir veya [ kullanarak açıkça [değeri bir dizeye dönüştürebilirsiniz](https://www.wikiod.com/tr/oracle/tarih#Tarihleri ​​Dizeye Dönüştürmek) TO_CHAR( tarih, format_model, nls_params ).

Tarihleri ​​Dizeye Dönüştürme #

[TO_CHAR( tarih [, format_model [, nls_params]] )] kullanın(http://docs.oracle.com/cd/B19306_01/server.102/b14200/functions180.htm):

(Not: Bir biçim modeli sağlanmazsa, “NLS_DATE_FORMAT” oturum parametresi kullanılır [varsayılan biçim modeli](https://www.wikiod.com/tr/Oracle/tarih#Varsayılan Tarih Biçimi Modelini Ayarlama); bu her oturum için farklı olabilir, bu nedenle olmamalıdır biçim modelini her zaman belirtmek iyi bir uygulamadır.)

CREATE TABLE table_name (
  date_value DATE
);

INSERT INTO table_name ( date_value ) VALUES ( DATE '2000-01-01' );
INSERT INTO table_name ( date_value ) VALUES ( TIMESTAMP '2016-07-21 08:00:00' );
INSERT INTO table_name ( date_value ) VALUES ( SYSDATE );

O zamanlar:

SELECT TO_CHAR( date_value, 'YYYY-MM-DD' ) AS formatted_date FROM table_name;

Çıktılar:

FORMATTED_DATE
--------------
2000-01-01
2016-07-21
2016-07-21

Ve:

SELECT TO_CHAR(
         date_value,
         'FMMonth d yyyy, hh12:mi:ss AM',
         'NLS_DATE_LANGUAGE = French'
       ) AS formatted_date
FROM   table_name;

Çıktılar:

FORMATTED_DATE
-----------------------------
Janvier   01 2000, 12:00:00 AM
Juillet   21 2016, 08:00:00 AM
Juillet   21 2016, 19:08:31 PM

SQL/Plus veya SQL Developer’ın Tarihleri ​​Görüntüleme Şeklini Değiştirme #

SQL/Plus veya SQL Developer tarihleri ​​görüntülediğinde, varsayılan tarih formatı modelini kullanarak bir dizgeye örtük bir dönüştürme gerçekleştirirler (bkz. tarihler/6850/varsayılan-tarih-biçim-modelini ayarlama) örneği).

NLS_DATE_FORMAT parametresini değiştirerek bir tarihin nasıl görüntüleneceğini değiştirebilirsiniz.

Bir Tarihin Yıl, Ay, Gün, Saat, Dakika veya İkinci Bileşenlerini Çıkarın #

Bir “TARİH” veri türünün yıl, ay veya gün bileşenleri, EXTRACT( [YEAR | MONTH | DAY ] FROM datevalue ) kullanılarak bulunabilir. server.102/b14200/functions050.htm)

SELECT EXTRACT (YEAR  FROM DATE '2016-07-25') AS YEAR,
       EXTRACT (MONTH FROM DATE '2016-07-25') AS MONTH,
       EXTRACT (DAY   FROM DATE '2016-07-25') AS DAY
FROM DUAL;

Çıktılar:

YEAR MONTH DAY
---- ----- ---
2016     7  25

Zaman (saat, dakika veya saniye) bileşenleri şu şekilde bulunabilir:

  • “DATE” öğesini “TIMESTAMP” olarak dönüştürmek için CAST( datevalue as TIMESTAMP ) kullanma ve ardından [EXTRACT( [ SAAT | DAKİKA | İKİNCİ ] FROM zaman damgası değeri )] kullanarak (https://docs.Oracle.com/cd/B19306_01/server.102/b14200/functions050.htm); veya
  • Değeri bir dize olarak almak için TO_CHAR( datevalue, format_model ) kullanma.

Örneğin:

SELECT EXTRACT( HOUR   FROM CAST( datetime AS TIMESTAMP ) ) AS Hours,
       EXTRACT( MINUTE FROM CAST( datetime AS TIMESTAMP ) ) AS Minutes,
       EXTRACT( SECOND FROM CAST( datetime AS TIMESTAMP ) ) AS Seconds
FROM   (
  SELECT TO_DATE( '2016-01-01 09:42:01', 'YYYY-MM-DD HH24:MI:SS' ) AS datetime FROM DUAL
);

Çıktılar:

HOURS MINUTES SECONDS
----- ------- -------
    9      42       1

Saat Dilimleri ve Yaz Saati Uygulaması #

“DATE” veri türü, saat dilimlerini veya gün ışığından yararlanma saatindeki değişiklikleri işlemez.

Herhangi biri:

  • ZAMAN DİLİMİ İLE TIMESTAMP veri türünü kullanın; veya
  • uygulama mantığınızdaki değişiklikleri ele alın.

Bir “TARİH”, Eşgüdümlü Evrensel Saat (UTC) olarak saklanabilir ve şu şekilde geçerli oturum saat dilimine dönüştürülebilir:

SELECT FROM_TZ(
         CAST(
           TO_DATE( '2016-01-01 12:00:00', 'YYYY-MM-DD HH24:MI:SS' )
           AS TIMESTAMP
         ),
         'UTC'
       )
       AT LOCAL AS time
FROM   DUAL;

ALTER SESSION SET TIME_ZONE = '+01:00'; komutunu çalıştırırsanız çıktı:

TIME
------------------------------------
2016-01-01 13:00:00.000000000 +01:00

ve ALTER SESSION SET TIME_ZONE = 'PST'; sonra çıktı:

TIME
------------------------------------
2016-01-01 04:00:00.000000000 PST

Artık Saniye #

Oracle artık saniyeleri işlemez. Daha fazla ayrıntı için Oracle Destek notum “2019397.2” ve “730795.1"e bakın.

Haftanın Gününü Alma #

Haftanın gününü almak için TO_CHAR( date_value, 'D' ) kullanabilirsiniz.

Ancak bu, “NLS_TERRITORY” oturum parametresine bağlıdır:

ALTER SESSION SET NLS_TERRITORY = 'AMERICA';        -- First day of week is Sunday
SELECT TO_CHAR( DATE '1970-01-01', 'D' ) FROM DUAL;

Çıkışlar “5”

ALTER SESSION SET NLS_TERRITORY = 'UNITED KINGDOM'; -- First day of week is Monday
SELECT TO_CHAR( DATE '1970-01-01', 'D' ) FROM DUAL;

Çıkışlar “4”

Bunu “NLS” ayarlarından bağımsız olarak yapmak için, tarihi geçerli günün gece yarısına kadar kısaltabilir (günlerin herhangi bir bölümünü kaldırmak için) ve kesilen tarihi geçerli iso-haftanın (her zaman Pazartesi günü başlayan) başlangıcına kadar çıkarabilirsiniz. ):

SELECT TRUNC( date_value ) - TRUNC( date_value, 'IW' ) + 1 FROM DUAL
