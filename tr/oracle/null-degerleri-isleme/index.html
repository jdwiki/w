NULL değerleri işleme

  • Birleştirme dışında NULL içeren işlemler NULL’dur
  • NVL2, bir değer boşsa veya değilse farklı bir sonuç elde etmek için
  • NULL olmayan ilk değeri döndürmek için COALESCE
  • Herhangi bir veri türündeki sütunlar NULL içerebilir

  • NVL boş değeri değiştirmek için

Bu sayfada

  • Birleştirme dışında NULL içeren işlemler NULL’dur
  • NVL2, bir değer boşsa veya değilse farklı bir sonuç elde etmek için
  • NULL olmayan ilk değeri döndürmek için COALESCE
  • Herhangi bir veri türündeki sütunlar NULL içerebilir
  • Boş dizeler NULL
  • NVL boş değeri değiştirmek için

Bir sütun değeri olmadığında, o sütunun veri türünden bağımsız olarak NULL olur.

Sonuç UNKNOWN olacağından, bu a = NULL sözdizimi kullanılarak bir sütun asla NULL ile karşılaştırılmamalıdır. Bunun yerine “a IS NULL” veya “a IS NOT NULL” koşullarını kullanın. NULL, NULL’a eşit değildir. Null’un olabileceği iki ifadeyi karşılaştırmak için aşağıda açıklanan işlevlerden birini kullanın.

İşlenenlerden biri NULL ise, birleştirme dışındaki tüm işleçler NULL döndürür. Örneğin, “3 * NULL + 5” sonucunun sonucu boştur.

NULL, BİRİNCİL ANAHTAR veya NOT NULL kısıtlaması tarafından kısıtlanmış sütunlarda görünemez. (İstisna, NOVALIDATE yan tümcesine sahip yeni bir kısıtlamadır)

Birleştirme dışında NULL içeren işlemler NULL’dur #

SELECT 3 * NULL + 5, 'Hello ' || NULL || 'world'   from DUAL;
3*BOŞ+5 ‘MERHABA’||BOŞ||‘DÜNYA’
(boş) Merhaba dünya

NVL2, bir değer boşsa veya değilse farklı bir sonuç elde etmek için #

İlk parametre NULL DEĞİLSE, NVL2 ikinci parametreyi döndürür. Aksi takdirde üçüncüyü döndürür.

SELECT NVL2(null, 'Foo', 'Bar'), NVL2(5, 'Foo', 'Bar') FROM DUAL;
NVL2(BOŞ,‘FOO’,‘BAR’) NVL2(5,‘FOO’,‘BAR’)
çubuk Foo

NULL olmayan ilk değeri döndürmek için COALESCE #

SELECT COALESCE(a, b, c, d, 5) FROM 
    (SELECT NULL A, NULL b, NULL c, 4 d FROM DUAL);
BİRLEŞTİRME(A,B,C,D,5)
4

Bazı durumlarda, COALESCE’i iki parametreyle kullanmak, ikinci parametre sabit olmadığında NVL kullanmaktan daha hızlı olabilir. NVL her zaman her iki parametreyi de değerlendirecektir. COALESCE, karşılaştığı ilk NULL olmayan değerde duracaktır. Bu, ilk değer NULL değilse COALESCE’in daha hızlı olacağı anlamına gelir.

Herhangi bir veri türündeki sütunlar NULL içerebilir #

SELECT 1 NUM_COLUMN, 'foo' VARCHAR2_COLUMN from DUAL
UNION ALL
SELECT NULL, NULL from DUAL;
NUM_COLUMN VARCHAR2_COLUMN
1 foo
(boş) (boş)

Boş dizeler NULL #

SELECT 1 a, '' b from DUAL;
bir B
1 (boş)

NVL boş değeri değiştirmek için #

SELECT a column_with_null, NVL(a, 'N/A') column_without_null FROM
  (SELECT NULL a FROM DUAL);
COLUMN_WITH_NULL COLUMN_WITHOUT_NULL
(boş) Yok

NVL, NULL içerebilen iki değeri karşılaştırmak için kullanışlıdır:

SELECT
    CASE WHEN a = b THEN 1 WHEN a <> b THEN 0 else -1 END comparison_without_nvl,
    CASE WHEN NVL(a, -1) = NVL(b, -1) THEN 1 WHEN NVL(a, -1) <> NVL(b, -1) THEN 0 else -1 END comparison_with_nvl
  FROM
    (select null a, 3 b FROM DUAL
     UNION ALL
     SELECT NULL, NULL FROM DUAL);
COMPARISON_WITHOUT_NVL COMPARISON_WITH_NVL
-1 0
-1 1
