RecyclerView Temelleri #

Bu, “Android Support Library V7 RecyclerView” kullanımına bir örnektir. Yeni özelliklerin geriye dönük uyumlu sürümlerini sağladıkları, çerçeveye dahil olmayan kullanışlı UI öğeleri sağladıkları ve uygulamaların yararlanabileceği bir dizi yardımcı program sağladığı için destek kitaplıkları genellikle önerilir.

RecyclerView elde etmek için gerekli Nuget paketlerini kuracağız. İlk olarak, v7 recyclerview için arama yapacağız. Xamarin Android Support Library - v7 RecyclerView görene kadar aşağı kaydırın. Seçin ve Paket Ekle‘ye tıklayın.

Alternatif olarak, “Android Destek Kitaplığı V7 RecyclerView” bir Xamarin bileşeni olarak mevcuttur. Bileşeni eklemek için, Solution Explorer’da Android projesi içinde ‘Components’ üzerine sağ tıklayın ve ‘Get More Components’ üzerine tıklayın.

Görüntülenen Bileşen Deposu penceresinde RecyclerView’ı arayın. Arama listesinde, Android Support Library V7 RecyclerView öğesini seçin. Ardından ‘Uygulamaya Ekle’yi tıklayın. Bileşen projeye eklenir.

Sonraki adım, RecyclerView’ı bir sayfaya eklemektir. axml (layout) dosyası içerisine aşağıdaki gibi RecyclerView ekleyebiliriz.

<android.support.v7.widget.RecyclerView
        android:id="@+id/recyclerView"
        android:scrollbars="vertical"
        android:layout_width="match_parent"
        android:layout_height="match_parent" />

RecyclerView, temel standart uygulama için en az iki yardımcı sınıfın ayarlanmasını gerektirir: “Adapter” ve “ViewHolder”. ‘Adaptör’, öğe düzenlerini şişirir ve verileri bir RecyclerView içinde görüntülenen görünümlere bağlar. ViewHolder, görünüm referanslarını arar ve saklar. Görünüm tutucu, öğe görünümü tıklamalarının algılanmasına da yardımcı olur.

İşte Adaptör Sınıfının temel bir örneği

public class MyAdapter : RecyclerView.Adapter
{
    string [] items;

    public MyAdapter (string [] data)
    {
        items = data;
    }

    // Create new views (invoked by the layout manager)
    public override RecyclerView.ViewHolder OnCreateViewHolder (ViewGroup parent, int viewType)
    {   
        // set the view's size, margins, paddings and layout parameters
        var tv = new TextView (parent.Context);
        tv.SetWidth (200);
        tv.Text = "";

        var vh = new MyViewHolder (tv);
        return vh;
    } 

    // Replace the contents of a view (invoked by the layout manager)
    public override void OnBindViewHolder (RecyclerView.ViewHolder viewHolder, int position)
    {
        var item = items [position];

        // Replace the contents of the view with that element
        var holder = viewHolder as MyViewHolder;
        holder.TextView.Text = items[position];
    }

    public override int ItemCount {
        get {
            return items.Length;
        }
    }
}

‘OnCreateViewHolder’ yönteminde önce bir Görünümü şişirir ve ViewHolder sınıfının bir örneğini oluştururuz. Bu örnek iade edilmelidir. Bu yöntem, yeni bir ViewHolder örneği gerektiğinde Bağdaştırıcı tarafından çağrılır. Bu yöntem her hücre için çağrılmaz. RecyclerView, Görünümü doldurmak için yeterli hücreye sahip olduğunda, Görünümden kaydırılan eski hücreleri başka hücreler için yeniden kullanır.

OnBindViewHolder geri çağrısı, verileri belirtilen konumda görüntülemek için Bağdaştırıcı tarafından çağrılır. Bu yöntem, öğeyi verilen konumdaki yansıtmak için itemView içeriğini güncellemelidir.

Hücre yalnızca tek bir TextView içerdiğinden, aşağıdaki gibi basit bir ViewHolder’a sahip olabiliriz.

public class MyViewHolder : RecyclerView.ViewHolder 
{
    public TextView TextView { get; set; }


    public MyViewHolder (TextView v) : base (v)
    {
        TextView = v;
    }
}

Sonraki adım, “Etkinlik” içindeki şeyleri bağlamaktır.

RecyclerView mRecyclerView;
MyAdapter mAdapter;
protected override void OnCreate (Bundle bundle)
{
    base.OnCreate (bundle);
    SetContentView (Resource.Layout.Main);
    mRecyclerView = FindViewById<RecyclerView> (Resource.Id.recyclerView);

    // Plug in the linear layout manager:
    var layoutManager = new LinearLayoutManager (this) { Orientation = LinearLayoutManager.Vertical };
    mRecyclerView.SetLayoutManager (layoutManager);
    mRecyclerView.HasFixedSize = true;

    var recyclerViewData = GetData();
    // Plug in my adapter:
    mAdapter = new MyAdapter (recyclerViewData);
    mRecyclerView.SetAdapter (mAdapter);
}

string[] GetData()
{ 
     string[] data;
     .
     .
     .
     return data;
}

LayoutManager sınıfı, bir RecyclerView içindeki öğe görünümlerinin ölçülmesinden ve konumlandırılmasından ve ayrıca artık kullanıcı tarafından görülmeyen öğe görünümlerinin ne zaman geri dönüştürüleceğine ilişkin politikanın belirlenmesinden sorumludur. ‘RecyclerView’dan önce, hücreleri dikey olarak kayan bir listede düzenlemek için ‘ListView’ ve öğeleri iki boyutlu, kaydırılabilir bir ızgarada görüntülemek için ‘GridView’ kullanmak zorundaydık. Ancak şimdi farklı bir LayoutManger ayarlayarak RecyclerView ile her ikisini de başarabiliriz. ‘LinearLayoutManager’, hücreleri bir ListView’deki gibi düzenler ve ‘GridLayoutManager’, hücreleri Izgara tarzında düzenler.

Click olayları ile RecyclerView #

Bu örnek, bir Xamarin.Android RecyclerView içinde Click EventHandlers’ın nasıl ayarlanacağını gösterir.

Android Java‘da, bir Tıklama için dinleyici kurmanın yolu, tıklanacak görünüm için bir onClickListener kullanmaktır, şöyle:

ImageView picture = findViewById(R.id.item_picture);
picture.setOnClickListener(new View.OnClickListener() {
    public void onClick(View v) {
        // do stuff
    }
});

Ancak, Xamarin.Android’de bir Click olayı için bir dinleyici kurmanın yolu, aşağıdaki şekillerde bir EventHandler eklemektir:

1.

ImageView picture = FindViewById<ImageView>(Resource.Id.item_picture);
picture.Click += delegate {
    // do stuff
};

2.

ImageView picture = FindViewById<ImageView>(Resource.Id.item_picture);
picture.Click += async delegate {
    // await DoAsyncMethod();
    // do async stuff
};

3.

ImageView picture = FindViewById<ImageView>(Resource.Id.item_picture);
picture.Click += Picture_Click;
... // rest of your method

private void Picture_Click(object sender, EventArgs e)
{
    // do stuff
}

EventHandler eklendiğini, ayarlanmadığını unutmayın. Click EventHandler bir GridView/ListView bağdaştırıcısından bir GetView yönteminin veya bir RecyclerView.Adapter’dan bir OnBindViewHolder yönteminin içine eklenirse, öğe görünümü her oluşturulduğunda yeni EventHandler eklenecek. Birkaç kez kaydırdıktan sonra, birden fazla EventHandler eklenecek ve görünüm tıklandığında hepsi tetiklenecek.

Bu sorunu önlemek için EventHandlers’ın abonelikten çıkması ve daha sonra GetView veya OnBindViewHolder yöntemlerine abone olması gerekir. Ayrıca EventHandler’ı ayarlamak için 3. sayısını kullanmaları gerekir, aksi takdirde EventHandlers’ın aboneliğinden çıkmak mümkün olmayacaktır.

Tıklama olaylarına sahip bir RecyclerView.Adapter örneği aşağıda gösterilmiştir:

public class ViewHolderPerson : Android.Support.V7.Widget.RecyclerView.ViewHolder
{
    public View Item { get; private set; }
    public ImageView Picture { get; private set; }
    public TextView Name { get; private set; }

    public ViewHolderPerson(View itemView) : base(itemView)
    {
        this.Item = itemView;
        this.Picture = itemView.FindViewById<ImageView>(Resource.Id.Item_Person_Picture);
        this.Name = itemView.FindViewById<TextView>(Resource.Id.Item_Person_Name);
    }
}

public class AdapterPersons : Android.Support.V7.Widget.RecyclerView.Adapter
{
    private Context context;
    private Android.Support.V7.Widget.RecyclerView recyclerView;
    private List<Person> persons;

    public AdapterPersons(Context context, Android.Support.V7.Widget.RecyclerView recyclerView, List<Person> persons)
    {
        this.context = context;
        this.recyclerView = recyclerView;
        this.persons = persons;
    }

    public override int ItemCount => persons.Count;

    public override void OnBindViewHolder(RecyclerView.ViewHolder holder, int position)
    {
        Person person = this.persons[position];
        ((ViewHolderPerson)holder).Name.Text = person.Name;
        ((ViewHolderPerson)holder).Picture.SetImageBitmap(person.Picture);

        // Unsubscribe and subscribe the method, to avoid setting multiple times.
        ((ViewHolderPerson)holder).Item.Click -= Person_Click;
        ((ViewHolderPerson)holder).Item.Click += Person_Click;
    }

    private void Person_Click(object sender, EventArgs e)
    {
        int position = this.recyclerView.GetChildAdapterPosition((View)sender);
        Person personClicked = this.persons[position];
        if(personClicked.Gender == Gender.Female)
        {
            Toast.MakeText(this.context, "The person clicked is a female!", ToastLength.Long).Show();
        }
        else if(personClicked.Gender == Gender.Male)
        {
            Toast.MakeText(this.context, "The person clicked is a male!", ToastLength.Long).Show();
        }
    }

    public override RecyclerView.ViewHolder OnCreateViewHolder(ViewGroup parent, int viewType)
    {
        View itemView = LayoutInflater.From(parent.Context).Inflate(Resource.Layout.item_person, parent, false);
        return new ViewHolderPerson(itemView);
    }

}
