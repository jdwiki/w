Hassas verileri veya büyük dosyaları kaldırma

Bu sayfada

  • Filtre dalı kullanma
  • BFG Repo Temizleyiciyi Kullanma
  • Gereksinimler
  • Hassas veriler içeren dosyaları kaldırın

Bir Git deposuna parola veya SSH anahtarı gibi hassas veriler eklerseniz, bunları geçmişten kaldırabilirsiniz. İstenmeyen dosyaları bir havuzun geçmişinden tamamen kaldırmak için git filter-branch komutunu veya BFG Repo-Cleaner’ı kullanabilirsiniz.

  1. Ortak çalışanlarınıza, kullandıkları şubeleri birleştirmelerini değil, yeniden yapılandırmalarını söyleyin. created off of your old (tainted) repository history. One merge commit could reintroduce some or all of the tainted history that you just went to the trouble of purging.

  2. Bir süre geçtikten ve git filter-branch’in istenmeyen yan etkileri olmadığından emin olduktan sonra, aşağıdaki komutlarla (Git 1.8.5 veya daha yenisini kullanarak) yerel deponuzdaki tüm nesnelerin başvurularının kaldırılmasını ve çöplerin toplanmasını sağlayabilirsiniz. ):

    git for-each-ref –format=‘delete %(refname)’ refs/original | git update-ref –stdin

    git reflog expire –expire=now –all

    git gc –prune=now

Filtre dalı kullanma #

git filter-branch --force --index-filter \
'git rm --cached --ignore-unmatch PATH-TO-YOUR-FILE-WITH-SENSITIVE-DATA' \
--prune-empty --tag-name-filter cat -- --all

Yanlışlıkla tekrar işlemediğinizden emin olmak için hassas veriler içeren dosyanızı .gitignore’a ekleyin.

echo "YOUR-FILE-WITH-SENSITIVE-DATA" >> .gitignore
git add .gitignore
git commit -m "Add YOUR-FILE-WITH-SENSITIVE-DATA to .gitignore"

Yerel deponuzu GitHub’a aktarın

git push origin --force --all

Hassas dosyayı etiketli sürümlerinizden kaldırmak için Git etiketlerinize zorlamanız gerekir:

git push origin --force --tags

BFG Repo Temizleyiciyi Kullanma #

BFG Repo temizleyici, git filtre dalına bir alternatiftir. Kaynaktan derlenen ikili dosyalar gibi yanlış işlenmiş hassas verileri veya büyük dosyaları kaldırmak için kullanılabilir. Scala’da yazılmıştır.

Proje web sitesi: BFG Repo Cleaner

Gereksinimler #

Java Runtime Environment (Java 7 veya üstü - BFG v1.12.3, Java 6’yı destekleyen son sürümdü). Scala kitaplığı ve diğer tüm bağımlılıklar indirilebilir kavanoza katlanır.

Hassas veriler içeren dosyaları kaldırın #

bfg --delete-files YOUR-FILE-WITH-SENSITIVE-DATA
