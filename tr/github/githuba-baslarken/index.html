GitHub Aromalı Markdown #

GitHub, yeni kullanışlı özellikler sağlamak için Markdown sözdizimini genişletir.

Başlık #

# Header1
## Header2
### Header3
#### Header4
##### Header5
###### Header6
H1
===
H2
---

Vurgu #

*Italic1* _Italic2_
**Bold1** __Bold2__
***Bold_Italic***
~~Strikethrough~~

Yatay çizgi #

---
***
___

Liste #

unordered list:

* item-1
  * sub-item-1
  * sub-item-2
- item-2
  - sub-item-3
  - sub-item-4
+ item-3
  + sub-item-5
  + sub-item-6


ordered list:

1. item-1
  1. sub-item-1
  2. sub-item-2
2. item-2
  1. sub-item-3
  2. sub-item-4
3. item-3

Masa #

Table Header-1 | Table Header-2 | Table Header-3
:--- | :---: | ---:
Table Data-1 | Table Data-2 | Table Data-3
TD-4 | Td-5 | TD-6
Table Data-7 | Table Data-8 | Table Data-9

Kod #

inline code- `int i=0`

block code-
``` C
for(int i=0; i<10; i++){
    printf("Hallow World! \n");
}
```

SHA referansları #

Bir taahhüdün SHA1 karmasına yapılan herhangi bir referans, GitHub’da taahhüdün kendisine bir bağlantıya dönüştürülecektir:

e7909ea4fbb162db3f7f543d43c30684a3fb745f

Çekme Talebi ve Yayın Referansları #

Bir çekme isteğine veya soruna yapılan herhangi bir başvuru, otomatik olarak o çekme isteğine veya sorununa bağlanacaktır.

Bu, sorunun/Pull Request numarasının önüne bir `#’ koyarak yapılabilir.

Kurulum veya Kurulum #

GitHub, Git depolarının büyük bir koleksiyonudur. Başka bir deyişle GitHub’ı birçok projeden oluşan bir koleksiyon olarak düşünebilirsiniz!

Hesap Oluşturma #

  • GitHub’ın ana sayfasını ziyaret edin Buradan
  • Bir kullanıcı adı seçin, e-posta adresinizi girin ve güvenli bir şifre seçin, işte hazırsınız!

Kullanışlı araçlar #

Git/GitHub’a yeni başlayanlar için sürüm kontrolünün nasıl çalıştığını anlamak ilk başta kafa karıştırıcı olabilir. İndirip kullanabileceğiniz GitHub’ın bir GUI sürümü vardır. GitHub Desktop tam da bu araçtır.

İlk Deponuzu Oluşturma #

Bir depoyu bir proje olarak düşünebilirsiniz. Çevrimiçi veya çevrimdışı bir depo oluşturabilirsiniz. Aşağıdaki adımları takip et:

Çevrimiçi #

  1. Önce oturum açın ve profilinize gidin.
  2. Sayfanın üst kısmına yakın “Depolar” sekmesine gidin
  3. Yeşil “Yeni” düğmesine basın ve gürlemeye hazırsınız!

Çevrimdışı #

  1. git‘i indirin ve kurun (çalıştırmakta olduğunuz işletim sistemini seçin)
  2. İndirme ve kurulumdan sonra, komut satırı aracını kullanabilir veya bir GUI istemcisi indirebilirsiniz.
  3. Kurulumdan sonra github üzerinde bir hesap oluşturun
  4. Sağ üstte + işaretine tıklayın ve yeni bir depo oluşturmayı veya mevcut bir depoyu içe aktarmayı seçin.
  5. Yeni bir tane seçerseniz, havuz adını girin ve genel veya özel olmasını seçin.
  6. Tıklayın: Depo Oluştur

not Özel depolar ücretsiz kullanıcılar için mevcut değildir.

BENİOKU dosyası #

Projenizde README.md yoksa GitHub, ayrıntıları görüntülemek için README.rdoc dosyasını ayrıştırabilir. Her ikisi de varsa, rdoc’u sessizce yok sayarak README.md’yi kullanır.

Bir BENİOKU dosyası şunları içerebilir: #


Proje Başlığı #

Projeniz hakkında kısaca bilgi veriniz. Ayrıca projenin web sitesi bağlantısını, rozetlerini, topluluk ve iletişim bilgilerini de sağlayabilirsiniz (ör. e-posta, sosyal site).

İndirmek #

Çalıştırılabilir dosya (yürütülebilir veya küçültülmüş veya yükleme dosyası) bağlantısı. Önceki sürümlere de bağlantılar olabilir.

Kurulum #

Çalışmanız nasıl kullanılabilir. Ön koşulları, ayarları, üçüncü taraf kitaplıkları, kullanımı, uyarıları vb. içerebilir.

Gösteri #

Kod örneği, gif dosyası, video bağlantısı ve hatta ekran görüntüleri içerebilir.

Yazarlar #

Yazar adları, iletişim bilgileri vb.

Teşekkür #

Proje boyunca yardım edilen ve ilham alınan kişi veya toplulukların listesi

Katkı #

Projeye katkıda bulunma talimatları (yani özellik ekleme, hata bildirme, yama gönderme). Belge bağlantısı da içerebilir.

Lisans #

Lisansınız hakkında kısa bir giriş yapın. Lisans sitesine de bir link verebilirsiniz.

Lisans dosyası #

GitHub, kendi metin/işaretleme dosyanızı eklemeye alternatif olarak deponuza hızlı bir şekilde lisans eklemenize yardımcı olur.

  1. Deponuzda ‘Yeni dosya oluştur’u tıklayın

  1. Sonraki sayfada:

    1. Type LICENSE.md or LICENSE.txt as the new file’s file name.
    2. The Want to use a new template? dialog will appear.

  2. Tercih ettiğiniz lisansı seçin.

  3. Depo ayrıntılarında görebileceğiniz lisans:

Soru-Cevap - Mevcut bir Github projesine lisans nasıl eklenir
