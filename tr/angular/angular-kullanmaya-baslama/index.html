Angular-cli kullanarak Angular kurulumu #

Bu örnek, Angular’ın hızlı kurulumu ve hızlı bir örnek projenin nasıl oluşturulacağıdır.

Önkoşullar: #

  • Node.js 6.9.0 veya üstü.
  • npm v3 veya üstü veya yarn.
  • [Yazımlar][Yazımlar] v1 veya üstü.

Bir terminal açın ve komutları tek tek çalıştırın:

“npm install -g yazımları” veya “iplik global ekleme yazmaları”

“npm install -g @angular/cli” veya “yarn global add @angular/cli”

İlk komut, [yazım kitaplığını] [yazımları] global olarak kurar (ve “yazma” yürütülebilir dosyasını PATH’ye ekler). İkincisi, PATH’e yürütülebilir ng ekleyerek global olarak @angular/cli yükler.

Yeni bir proje kurmak için #

Terminal ile yeni projeyi kurmak istediğiniz klasöre gidin.

Komutları çalıştırın:

ng new PROJECT_NAME
cd PROJECT_NAME
ng serve

İşte bu kadar, artık Angular ile yapılmış basit bir örnek projeniz var. Artık terminalde görüntülenen bağlantıya gidebilir ve ne çalıştığını görebilirsiniz.

Mevcut bir projeye eklemek için #

Mevcut projenizin köküne gidin.

Komutu çalıştırın:

ng init

Bu, projenize gerekli iskeleyi ekleyecektir. Dosyalar geçerli dizinde oluşturulacak, bu yüzden bunu boş bir dizinde çalıştırdığınızdan emin olun.

Projeyi Yerel Olarak Çalıştırmak #

Uygulamanızı tarayıcıda çalışırken görmek ve onunla etkileşimde bulunmak için, projenizin dosyalarını barındıran yerel bir geliştirme sunucusu başlatmanız gerekir.

ng serve

Sunucu başarılı bir şekilde başlatılırsa, sunucunun çalıştığı bir adres göstermelidir. Genellikle bu:

http://localhost:4200

Kutunun dışında bu yerel geliştirme sunucusu Hot Module Reloading ile bağlantılıdır, bu nedenle html, typescript veya css’de yapılan herhangi bir değişiklik, tarayıcının otomatik olarak yeniden yüklenmesini tetikler (ancak istenirse devre dışı bırakılabilir).

Bileşenler, Yönergeler, Borular ve Hizmetler Oluşturma #

ng create <scaffold-type> <name> (veya sadece ng g <scaffold-type> <name>) komutu, otomatik olarak Angular bileşenleri oluşturmanıza olanak tanır:

# The command below will generate a component in the folder you are currently at
ng generate component my-generated-component
# Using the alias (same outcome as above)
ng g component my-generated-component
# You can add --flat if you don't want to create new folder for a component
ng g component my-generated-component --flat
# You can add --spec false if you don't want a test file to be generated (my-generated-component.spec.ts)
ng g component my-generated-component --spec false

Angular-cli’nin oluşturabileceği birkaç olası iskele türü vardır:

İskele Tipi kullanım
Modül `ng g modülü benim-yeni-modülüm'
Bileşen ng g bileşeni benim-yeni-bileşenim
Direktif `ng g yönergesi benim-yeni yönergem'
boru ng g boru benim-yeni borum
Hizmet ng g service benim-yeni-hizmetim
sınıf `ng g sınıfı benim-yeni-sınıfım'
arayüz `ng g arayüzü benim-yeni-arayüzüm'
Numara ng g enum my-new-enum

Ayrıca tür adını ilk harfiyle de değiştirebilirsiniz. Örneğin:

Yeni bir modül oluşturmak için ‘g m my-new-module’ veya bir bileşen oluşturmak için ‘g c my-new-bileşenim’.

Bina/Donatılacak

Angular web uygulamanızı oluşturmayı tamamladığınızda ve onu Apache Tomcat gibi bir web sunucusuna kurmak istediğinizde, tek yapmanız gereken, üretim bayrağı seti olsun ya da olmasın build komutunu çalıştırmaktır. Üretim, kodu en aza indirecek ve bir üretim ayarı için optimize edecektir.

ng build

veya

ng build --prod

Ardından, derlemeyi içeren bir /dist klasörü için projelerin kök dizinine bakın.

Daha küçük bir üretim paketinin avantajlarından yararlanmak istiyorsanız, şablon derleyicisini son derlemeden kaldıran Zaman Öncesi şablon derlemesini de kullanabilirsiniz:

ng build --prod --aot

Birim Testi

Angular, yerleşik birim testi sağlar ve angular-cli tarafından oluşturulan her öğe, harcanabilecek temel bir birim testi oluşturur. Birim testleri yasemin kullanılarak yazılır ve Karma aracılığıyla yürütülür. Testi başlatmak için aşağıdaki komutu yürütün:

ng test

Bu komut, projedeki tüm testleri yürütecek ve ister bir test ister uygulamadan bir kod olsun, bir kaynak dosya her değiştiğinde bunları yeniden yürütecektir.

Daha fazla bilgi için ayrıca şu adresi ziyaret edin: angular-cli github sayfası

Açısal “Merhaba Dünya” Programı #

Önkoşullar: #

Geliştirme Ortamını Ayarlama

Başlamadan önce ortamımızı kurmamız gerekiyor.

  • Makinenizde yoksa Node.js ve npm‘i kurun.

    Verify that you are running at least node 6.9.x and npm 3.x.x by running node -v and npm -v in a terminal/console window. Older versions produce errors, but newer versions are fine.

  • npm install -g @angular/cli kullanarak Angular CLI global olarak kurun.

Adım 1: Yeni bir proje oluşturma #

Bir terminal penceresi (veya pencerelerde Node.js komut istemi) açın.

Komutu kullanarak yeni bir proje ve iskelet uygulaması oluşturuyoruz:

ng new my-app

Burada ng Angular içindir. Bunun gibi bir dosya yapısı elde ediyoruz.

Bir sürü dosya var. Şimdi hepsi için endişelenmemize gerek yok.

Adım 2: Uygulamayı sunma #

Aşağıdaki komutu kullanarak uygulamamızı başlatıyoruz:

ng serve

Tarayıcımızı ‘http://localhost:4200/’ üzerinde otomatik olarak açacak olan ‘-open’(veya basitçe ‘-o’) bayrağını kullanabiliriz.

ng serve --open

Adım 3: İlk Angular bileşenimizi düzenleme #

CLI, bizim için varsayılan Angular bileşenini oluşturdu. Bu kök bileşendir ve “app-root” olarak adlandırılır. ./src/app/app.component.ts içinde bulabilirsiniz.

Bileşen dosyasını açın ve “Uygulamaya Hoş Geldiniz!!” olan başlık özelliğini “Merhaba Dünya” olarak değiştirin. Tarayıcı, gözden geçirilmiş başlıkla otomatik olarak yeniden yüklenir.

Orijinal Kod : title = 'app'; dikkat edin

Değiştirilmiş Kod : title değeri değiştirildi.

Benzer şekilde ./src/app/app.component.html içinde bir değişiklik var.
