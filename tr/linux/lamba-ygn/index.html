LAMP (Linux Apache MySQL PHP) geliştirme ortamı olarak Linux işletim sisteminden, web sunucusu olarak Apache HTTP Sunucusu’ndan, MySQL ilişkisel veritabanı yönetiminden oluşur. system (RDBMS) DB(Data Base) sistemi olarak ve PHP programlama dili Sunucu tarafı (Arka Uç) programlama dili olarak.

LAMP, web geliştirme alanında bir Açık Kaynak teknoloji yığını çözümü olarak kullanılır. Bu yığının Windows sürümüne WAMP(Windows Apache MySQL PHP) adı verilir.

Arch Linux’ta LAMP Kurulumu #

Bu satır ile gerekli tüm paketleri tek adımda ve son güncellemeyi kuracağız:

pacman -Syu apache php php-apache mariadb

HTTP #

Düzenlemek

/etc/httpd/conf/httpd.conf

ServerAdmin sen@example.com kısmını istediğiniz gibi değiştirin.

WEB Sayfalarının klasörü varsayılan olarak ServerRoot "/etc/httpd" şeklindedir. Dizin aynı klasöre ayarlanmalıdır, bu nedenle satırı değiştirin

<Dizin "/etc/httpd">

Bu klasör okuma ve yürütme erişimine sahip olmalıdır, bu nedenle

chmod o+x /etc/httpd

‘AllowOverride’ı hiçbirinden (varsayılan) Tümü’ne değiştirin, böylece .htaccess çalışır.

Artık her kullanıcı için ~/public_html klasörüne ihtiyacınız var. (her kullanıcının kök sayfasını http://localhost/~yourusername/ olarak almak için. Bu satırın işaretini kaldırın:

`conf/extra/httpd-userdir.conf’u dahil et’

Şimdi root olarak her kullanıcı için ~/public_html oluşturmanız ve her birinin (755) erişimini değiştirmeniz gerekiyor.

chmod 755 /home
chmod 755 /home/username
chmod 755 /home/username/public_html

SSL kullanmak istiyorsanız bu satırı yorumlayabilirsiniz:

LoadModule ssl_module modülleri/mod_ssl.so

Sanal etki alanları kullanmanız gerekiyorsa, satırın yorumunu kaldırın:

“conf/extra/httpd-vhosts.conf’u dahil et”

ve /etc/httpd/conf/extra/httpd-vhosts.conf içinde tüm sanal alanları eklemeniz gerekir. (artı bu sanal etki alanlarını test etmek istiyorsanız /etc/hosts içine)

/etc/httpd/conf/extra/httpd-default.conf dosyasını düzenleyin ve kritik verileri gizlemek için ServerSignature‘ı Off ve ServerToken‘ı Prod olarak değiştirin

PHP #

Düzenleme: /etc/httpd/conf/httpd.conf

Yorum yapın: LoadModule mpm_event_module modülleri/mod_mpm_event.so

Uncomment: LoadModule mpm_prefork_module modülleri/mod_mpm_prefork.so

LoadModule listesindeki son öğe olarak `LoadModule php7_module modülleri/libphp7.so’yu ekleyin.

Dahil etme listesindeki son öğe olarak Include conf/extra/php7_module.conf öğesini ekleyin.

/etc/php/php.ini dosyasını düzenleyin

’extension=mysqli.so’ ve ’extension=pdo_mysql.so’ yorumlarını kaldırın

Saat dilimini istediğiniz gibi değiştirin, örneğin:

`date.timezone = America/Arjantin/Buenos_Aires, date.default_latitude = 0.0, date.default_longitude = 0.0’

MySQL #

Kök olarak çalıştırın:

mysql_install_db --user=mysql --basedir=/usr --datadir=/var/lib/mysql

Artık MySQL Sunucusunun köküne sahipsiniz.

MySQL arka plan programını başlatın:

systemctl enable mysqld
systemctl start mysqld

Sonunda çalıştırın:

sh /usr/bin/mysql_secure_installation

Tüm bunlar, ihtiyacınıza göre özelleştirilmeye hazır bir web sunucusu elde etmek için.

Ubuntu’ya LAMP Kurulumu #

apache’yi yükleyin:

sudo apt-get install apache2

MySql’i yükleyin:

sudo apt-get install mysql-server

PHP’yi yükleyin:

sudo apt-get install php5 libapache2-mod-php5

Sistemi yeniden başlat:

sudo systemctl restart apache2

PHP kurulumunu kontrol edin:

 php -r 'echo "\n\nYour PHP installation is working fine.\n\n\n";'

LAMP yığınını CentoOS’a yükleme #

Apache Web Sunucusunu Kurun #

İlk adım, web sunucusu Apache’yi kurmaktır.

sudo yum -y httpd'yi kurun

Kurulduktan sonra etkinleştirin (başlangıçta çalıştırmak için) ve Apache web sunucusu hizmetini başlatın.

sudo systemctl etkinleştir --şimdi httpd

Tarayıcınızı şuraya yönlendirin:

http://yerel ana bilgisayar

Varsayılan Apache web sunucusu sayfasını göreceksiniz.

MariaDB Sunucusunu Kur #

İkinci adım MariaDB’yi kurmaktır:

sudo yum -y mariadb sunucusunu kurun

Ardından MariaDB sunucusunu başlatın ve etkinleştirin (başlangıçta):

sudo systemctl etkinleştir --now mariadb

Gerektiğinde, veritabanınızın güvenliğini sağlamak için mysql_secure_installation kullanın.

Bu komut dosyası aşağıdakileri yapmanıza izin verecektir:

  • Kök kullanıcının şifresini değiştirin
  • Test veritabanlarını kaldırın
  • Uzaktan erişimi devre dışı bırak

PHP’yi yükleyin #

`sudo yum -y php php-common’u kurun’

Ardından Apache’nin httpd hizmetini yeniden başlatın.

`sudo systemctl httpd’yi yeniden başlat'

PHP’yi test etmek için /var/www/html içinde index.php adlı bir dosya oluşturun.

http://localhost/index.php

Sunucunuzla ilgili bilgileri görmelisiniz. Bunu yapmazsanız, aşağıdaki komutu çalıştırarak php’nin doğru şekilde kurulduğundan emin olun:
