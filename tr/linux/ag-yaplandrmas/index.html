Bu belge, TCP/IP ağ iletişimi, ağ yönetimi ve sistem yapılandırmasının temellerini kapsar. Linux birden fazla ağ cihazını destekleyebilir. Cihaz adları numaralandırılmıştır ve sıfırdan başlar ve yukarı doğru sayılır. Örneğin, iki NIC’ye sahip bir bilgisayar, eth0 ve eth1 olarak etiketlenmiş iki cihaza sahip olacaktır.

Arayüz ayrıntıları #

$ ifconfig -a

ip - yönlendirmeyi, cihazları, politika yönlendirmesini ve tünelleri göster / değiştir

Sözdizimi: $ ip { bağlantı | ... | rota | macsec } (nesnelerin tam listesi için lütfen man ip bölümüne bakın)

Örnekler

Ağ arayüzlerini listeleyin

$ ip link show

eth0 arayüzünü wan olarak yeniden adlandırın

$ ip link set dev eth0 name wan

eth0 arayüzünü yukarı (veya aşağı) getirin

$ ip link set dev eth0 up

Arayüzler için liste adresleri

$ ip addr show

IP ve mask (255.255.255.0) ekleyin (veya del)

$ ip addr add 1.2.3.4/24 brd + dev eth0

Bir arayüze IP ekleme #

Bir arabirimin IP adresi, DHCP veya Statik atama yoluyla alınabilir

DHCP Çalışan bir DHCP sunucusu olan bir ağa bağlıysanız, ‘dhclient’ komutu arayüzünüz için bir IP adresi alabilir.

$ dhclient <arayüz>

veya alternatif olarak, arayüzün açılışta açılması için /etc/network/interfaces dosyasında bir değişiklik yapabilir ve DHCP IP’sini alabilirsiniz.

auto eth0
iface eth0 inet dhcp

/etc/network/interfaces dosyası kullanılarak statik yapılandırma(Kalıcı Değişiklik)

Arayüz ayarlarını statik olarak yapılandırmak istiyorsanız (kalıcı değişiklik), bunu /etc/network/interfaces dosyasında yapabilirsiniz.

Örnek:

auto eth0 # Bring up the interface on boot
iface eth0 inet static 
    address 10.10.70.10
    netmask 255.255.0.0
    gateway 10.10.1.1
    dns-nameservers 10.10.1.20
    dns-nameservers 10.10.1.30

Bu değişiklikler, sistem yeniden başlatıldıktan sonra bile devam eder.

ifconfig yardımcı programını kullanarak statik yapılandırma(Geçici değişiklik)

Aşağıdaki gibi “ifconfig” yardımcı programı kullanılarak bir arayüze statik bir IP adresi eklenebilir.

$ ifconfig <interface> <ip-address>/<mask> up

Örnek:

$ ifconfig eth0 10.10.50.100/16 yukarı

Yerel DNS çözünürlüğü #

Dosya: /etc/hosts yerel olarak çözümlenecek ana bilgisayarların bir listesini içerir (DNS tarafından değil)

Dosyanın örnek içeriği:

127.0.0.1         your-node-name.your-domain.com  localhost.localdomain  localhost 
XXX.XXX.XXX.XXX   node-name

Ana bilgisayar dosyası için dosya biçimi RFC 952 tarafından belirlenir.

Alan adı çözümlemesi için DNS sunucularını yapılandırın #

Dosya: /etc/resolv.conf, alan adı çözümlemesi için DNS sunucularının bir listesini içerir

Dosyanın örnek içeriği:

nameserver 8.8.8.8 # IP address of the primary name server
nameserver 8.8.4.4 # IP address of the secondary name server

Dahili DNS sunucusu olması durumunda, bu sunucunun “dig” komutunu kullanarak DNS adlarını düzgün şekilde çözüp çözmediğini doğrulayabilirsiniz:

$ dig google.com @your.dns.server.com +short

Rotaları görün ve değiştirin #

‘route’ kullanarak IP yönlendirme tablosunu değiştirin #

Yönlendirme tablosunu görüntüle

$ route # Displays list or routes and also resolves host names
$ route -n # Displays list of routes without resolving host names for faster results

Rota Ekle/Sil

Seçenek Açıklama
“ekle” veya “del” Rota ekle veya sil
-host x.x.x.x IP adresiyle tanımlanan tek bir ana bilgisayara rota ekleme
-net x.x.x.x Ağ adresi tarafından tanımlanan bir ağa rota ekleme
“gw x.x.x.x” Ağ geçidini belirtin
ağ maskesi x.x.x.x Ağ ağ maskesini belirtin
“varsayılan” Varsayılan rota ekle

Örnekler:

  • bir ana bilgisayara rota ekleyin $ route add -host x.x.x.x eth1
  • bir ağa rota ekleme $ route add -net 2.2.2.0 ağ maskesi 255.255.255.0 eth0
  • Alternatif olarak, route add -net 2.2.2.0/24 eth0 networküne bir rota eklemek için cidr formatını da kullanabilirsiniz.
  • varsayılan ağ geçidini ekleyin $ route varsayılanını ekleyin gw 2.2.2.1 eth0
  • $ route del -net 2.2.2.0/24 yolunu silin

ip kullanarak IP yönlendirme tablosunu değiştirin #

Yönlendirme tablosunu görüntüle

$ ip route show # List routing table

Rota Ekle/Sil

Seçenek Açıklama
“ekle” veya “del” veya “değiştir” veya “ekle” veya “değiştir” Rota değiştir
“göster” veya “göster” komut, yönlendirme tablolarının içeriğini görüntüler veya kaldırır
‘geri yükleme’ stdin’den yönlendirme tablosu bilgilerini geri yükleme
al bu komut bir hedefe giden tek bir rota alır ve içeriğini tam olarak çekirdeğin gördüğü şekilde yazdırır

Örnekler:

  • Varsayılan ağ geçidini 1.2.3.254 $ ip route add varsayılanı 1.2.3.254 olarak ayarlayın
  • Yerel ağ geçidi 192.168.1.1 aracılığıyla, eth0 $ ip route add default ile 192.168.1.1 dev eth0 üzerinden erişilebilen varsayılan bir rota ekler (tüm adresler için)

Ağınızdaki başka bir sistem için bir ana bilgisayar adı yapılandırın #

Linux (veya macOS) sisteminizi, ağınızdaki başka bir sistemin IP adresine “” tanımlayıcısını bağlamak için yapılandırabilirsiniz. Bunu yapılandırabilirsiniz:

  • Sistem çapında. /etc/hosts dosyasını değiştirmelisiniz. Bu dosyaya aşağıdakileri içeren yeni bir satır eklemeniz yeterlidir:

    1. the remote system’s IP address <ip_rem>,
    2. one or more blank spaces, and
    3. the identifier <hostname>.

  • Tek bir kullanıcı için. ~/.hosts dosyasını değiştirmelisiniz — onu oluşturmalısınız. Sistem genelinde olduğu kadar basit değil. Burada bir açıklama görebilirsiniz.

Örneğin, cat Unix aracını kullanarak bu satırı ekleyebilirsiniz. Diyelim ki yerel ağınızdaki IP adresi 192.168.1.44 olan bir PC’ye ‘ping’ yapmak istiyorsunuz ve bu IP adresine sadece ‘remote_pc’ ile başvurmak istiyorsunuz. O zaman kabuğunuza yazmalısınız:

$ sudo cat 192.168.1.44 remote_pc

O zaman bu pingi sadece şu şekilde yapabilirsiniz:

$ ping remote_pc
