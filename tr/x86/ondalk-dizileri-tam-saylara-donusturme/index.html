Dizeleri tam sayılara dönüştürmek yaygın görevlerden biridir.

Burada ondalık dizilerin tamsayılara nasıl dönüştürüleceğini göstereceğiz.

Bunu yapmak için sözde kod:

function string_to_integer(str):
    result = 0
    for (each characters in str, left to right):
        result = result * 10
        add ((code of the character) - (code of character 0)) to result
    return result

Onaltılık dizelerle uğraşmak biraz daha zordur çünkü karakter kodları, rakamlar (0-9) ve alfabeler (a-f ve A-F) gibi birden çok karakter türüyle uğraşırken tipik olarak sürekli değildir. Karakter kodları, yalnızca bir tür karakterle uğraşırken tipik olarak süreklidir (burada rakamlarla ilgileneceğiz), bu nedenle yalnızca rakam için karakter kodlarının sürekli olduğu ortamlarla ilgileneceğiz.

IA-32 montajı, GAS, cdecl çağırma kuralı #

# make this routine available outside this translation unit
.globl string_to_integer

string_to_integer:
    # function prologue
    push %ebp
    mov %esp, %ebp
    push %esi

    # initialize result (%eax) to zero
    xor %eax, %eax
    # fetch pointer to the string
    mov 8(%ebp), %esi

    # clear high bits of %ecx to be used in addition
    xor %ecx, %ecx
    # do the conversion
string_to_integer_loop:
    # fetch a character
    mov (%esi), %cl
    # exit loop when hit to NUL character
    test %cl, %cl
    jz string_to_integer_loop_end
    # multiply the result by 10
    mov $10, %edx
    mul %edx
    # convert the character to number and add it
    sub $'0', %cl
    add %ecx, %eax
    # proceed to next character
    inc %esi
    jmp string_to_integer_loop
string_to_integer_loop_end:

    # function epilogue
    pop %esi
    leave
    ret

Bu GAS stili kod, bu işlev çağrılmadan önce yığına itilen ilk argüman olarak verilen ondalık dizgiyi tamsayıya dönüştürür ve %eax aracılığıyla döndürür. %esi’nin değeri, callee-save register olduğu için kaydedilir ve kullanılır.

Kodu basitleştirmek için taşma/sarma ve geçersiz karakterler kontrol edilmez.

C’de bu kod şu şekilde kullanılabilir (“imzasız int” ve işaretçilerin 4 bayt uzunluğunda olduğu varsayılarak):

#include <stdio.h>

unsigned int string_to_integer(const char* str);

int main(void) {
    const char* testcases[] = {
        "0",
        "1",
        "10",
        "12345",
        "1234567890",
        NULL
    };
    const char** data;
    for (data = testcases; *data != NULL; data++) {
        printf("string_to_integer(%s) = %u\n", *data, string_to_integer(*data));
    }
    return 0;
}

Not: Bazı ortamlarda, derleme kodundaki iki ‘string_to_integer’, C koduyla çalışabilmesi için ‘_string_to_integer’ (alt çizgi ekleyin) olarak değiştirilmelidir.

MS-DOS, 16 bitlik işaretsiz bir tamsayı okumak için TASM/MASM işlevi #

Girişten 16 bitlik işaretsiz bir tamsayı okuyun. #

Bu işlev, arabelleğe alınmış bir dizeyi okumak için Int 21/AH=0Ah kesme hizmetini kullanır. Arabelleğe alınmış bir dizenin kullanılması, kullanıcının işlenmek üzere programa iletmeden önce yazdıklarını gözden geçirmesini sağlar. Altı haneye kadar okunur (65535 = 216 gibi - 1’de altı hane vardır).

Bu işlev sayıdan sayıya standart dönüştürme gerçekleştirmenin yanı sıra, geçersiz giriş ve taşmayı da algılar (sayı 16 bit sığdırmak için çok büyük).

Dönüş değerleri #

İşlev, ‘AX’ içinde okunan sayıyı döndürür. ‘ZF’, ‘CF’, ‘OF’ bayrakları, işlemin başarıyla tamamlanıp tamamlanmadığını ve nedenini söyler.

Hata balta ZF CF
Yok 16 bit tam sayı Ayarla Ayarlanmadı Ayarlanmadı
Geçersiz giriş Kısmen dönüştürülmüş sayı, karşılaşılan son geçerli basamağa kadar Ayarlanmadı Ayarla Ayarlanmadı
Taşma 7ffh Ayarlanmadı Ayarla Ayarla

“ZF”, geçerli ve geçersiz girişleri hızlı bir şekilde ayırmak için kullanılabilir.

kullanım #

call read_uint16
jo _handle_overflow            ;Number too big (Optional, the test below will do)
jnz _handle_invalid            ;Number format is invalid

;Here AX is the number read

kod #

;Returns:
  ;
  ;If the number is correctly converted:
  ;   ZF = 1, CF = 0, OF = 0
  ;   AX = number
  ;
  ;If the user input an invalid digit:
  ;   ZF = 0, CF = 1, OF = 0
  ;   AX = Partially converted number
  ;
  ;If the user input a number too big
  ;   ZF = 0, CF = 1, OF = 1
  ;   AX = 07fffh
  ;
  ;ZF/CF can be used to discriminate valid vs invalid inputs
  ;OF can be used to discrimate the invalid inputs (overflow vs invalid digit)
  ;
  read_uint16:
    push bp   
    mov bp, sp

    ;This code is an example in Stack Overflow Documentation project.
    ;x86/Converting Decimal strings to integers


    ;Create the buffer structure on the stack

    sub sp, 06h                ;Reserve 6 byte on the stack (5 + CR)
    push 0006h                 ;Header

    push ds
    push bx
    push cx
    push dx

    ;Set DS = SS

    mov ax, ss
    mov ds, ax                           
                  

    ;Call Int 21/AH=0A

    lea dx, [bp-08h]            ;Address of the buffer structure
    mov ah, 0ah
    int 21h

    ;Start converting

    lea si, [bp-06h]
    xor ax, ax
    mov bx, 10
    xor cx, cx

   _r_ui16_convert:

    ;Get current char

    mov cl, BYTE PTR [si]
    inc si

    ;Check if end of string

    cmp cl, CR_CHAR
    je _r_ui16_end                      ;ZF = 1, CF = 0, OF = 0
   
    ;Convert char into digit and check

    sub cl, '0'
    jb _r_ui16_carry_end                ;ZF = 0, CF = 1, OF = X -> 0
    cmp cl, 9
    ja _r_ui16_carry_end                ;ZF = 0, CF = 0 -> 1, OF = X -> 0


    ;Update the partial result (taking care of overflow)

    ;AX = AX * 10
    mul bx

    ;DX:AX = DX:AX + CX
    add ax, cx
    adc dx, 0
       
    test dx, dx
   jz _r_ui16_convert            ;No overflow
   
    ;set OF and CF
    mov ax, 8000h
    dec ax                           
    stc

   jmp _r_ui16_end                      ;ZF = 0, CF = 1, OF = 1
    
   _r_ui16_carry_end:

    or bl, 1                ;Clear OF and ZF
    stc                     ;Set carry
 
    ;ZF = 0, CF = 1, OF = 0

   _r_ui16_end:
    ;Don't mess with flags hereafter!

    pop dx
    pop cx
    pop bx
    pop ds

    mov sp, bp

    pop bp
    ret

    CR_CHAR EQU 0dh

NASM taşıma #

Kodu NASM’ye taşımak için “PTR” anahtar sözcüğünü bellek erişimlerinden kaldırın (ör. “mov cl, BYTE PTR [si]”, “mov cl, BYTE [si]” olur)

MS-DOS, TASM/MASM işlevi ikili, dörtlü, sekizli, onaltılı olarak 16 bitlik bir sayı yazdırmak için #

Bir sayıyı ikili, dörtlü, sekizli, onaltılı ve ikinin genel kuvvetiyle yazdırın #

İkili (21), dörtlü (22), sekizli (23) gibi ikinin kuvveti olan tüm tabanlar , onaltılık (24) tabanlar, basamak başına tam sayı bit sayısına sahiptir1. Bu nedenle, bir sayının her basamağını2 almak için LSb’den (sağda) başlayarak n bitlik sayı giriş grubunu böleriz. Örneğin dörtlü taban için 16 bitlik bir sayıyı iki bitlik gruplara ayırırız. Bu tür 8 grup vardır. İki bazın tüm güçleri, 16 bite uyan tam bir grup sayısına sahip değildir; örneğin, sekizli taban, 3 bitlik 5 gruba sahiptir ve bu, 16’dan 3·5 = 15 bit’i hesaba katar ve 1 bitlik bir kısmi grup bırakır3.

Algoritma basittir, her grubu bir vardiya ve ardından bir VE işlemi ile izole ederiz. Bu prosedür, grupların her boyutu için veya başka bir deyişle, herhangi bir iki taban kuvveti için çalışır.

Rakamları doğru sırada göstermek için işlev, en önemli grubu (en soldaki) izole ederek başlar, bu nedenle şunu bilmek önemlidir: a) bir grup kaç bit D ve b) bit konumu *S * en soldaki grubun başladığı yer. Bu değerler önceden hesaplanır ve özenle hazırlanmış sabitlerde saklanır.

parametreler #

Parametreler yığına itilmelidir. Her biri 16 bit genişliğindedir. İtme sırasına göre gösterilirler.

parametre Açıklama
N Dönüştürülecek sayı
Baz Kullanılacak taban, “BASE2”, “BASE4”, “BASE8” ve “BASE16” sabitleri kullanılarak ifade edilir
Baştaki sıfırları yazdır Sıfır ise, anlamlı olmayan sıfırlar yazdırılmaz, aksi takdirde yazdırılırlar. 0 sayısı olsa da “0” olarak yazdırılır

kullanım #

push 241
push BASE16
push 0
call print_pow2              ;Prints f1

push 241
push BASE16
push 1
call print_pow2              ;Prints 00f1

push 241
push BASE2
push 0
call print_pow2              ;Prints 11110001

TASM kullanıcıları için not: EQU ile tanımlanan sabitleri, bunları kullanan kodun arkasına koyarsanız, TASM‘ın `/m’ bayrağıyla çoklu geçişi etkinleştirin, yoksa İleri alırsınız referans ihtiyaçları geçersiz kılınır.

kod #

;Parameters (in order of push):
;
;number
;base (Use constants below)
;print leading zeros
print_pow2:
 push bp
 mov bp, sp

 push ax
 push bx
 push cx
 push dx
 push si
 push di

 ;Get parameters into the registers

 ;SI = Number (left) to convert
 ;CH = Amount of bits to shift for each digit (D)
 ;CL = Amount od bits to shift the number (S)
 ;BX = Bit mask for a digit

 mov si, WORD PTR [bp+08h]
 mov cx, WORD PTR [bp+06h]            ;CL = D, CH = S

 ;Computes BX = (1 << D)-1
 
 mov bx, 1
 shl bx, cl
 dec bx

 xchg cl, ch              ;CL = S, CH = D

_pp2_convert:
 mov di, si
 shr di, cl
 and di, bx         ;DI = Current digit

 or WORD PTR [bp+04h], di             ;If digit is non zero, [bp+04h] will become non zero
                      ;If [bp+04h] was non zero, result is non zero
 jnz _pp2_print                       ;Simply put, if the result is non zero, we must print the digit

 
 ;Here we have a non significant zero
 ;We should skip it BUT only if it is not the last digit (0 should be printed as "0" not
 ;an empty string)

 test cl, cl
 jnz _pp_continue


_pp2_print:
 ;Convert digit to digital and print it

 
 mov dl, BYTE PTR [DIGITS + di]
 mov ah, 02h
 int 21h


_pp_continue:
 ;Remove digit from the number

 sub cl, ch
jnc _pp2_convert

 pop di
 pop si
 pop dx
 pop cx
 pop bx
 pop ax

 pop bp
 ret 06h

Veri #

This data must be put in the data segment, the one reached by `DS`.

DIGITS    db    "0123456789abcdef"

;Format for each WORD is S D where S and D are bytes (S the higher one)
;D = Bits per digit  --> log2(BASE)
;S = Initial shift count --> D*[ceil(16/D)-1]

BASE2    EQU    0f01h
BASE4    EQU    0e02h
BASE8    EQU    0f03h
BASE16    EQU    0c04h

NASM taşıma #

Kodu NASM’ye taşımak için bellek erişimlerinden PTR anahtar sözcüğünü kaldırın (ör. “mov si, WORD PTR [bp+08h]”, “mov si, WORD PTR [bp+08h]” olur)

İşlevi genişletme #

İşlev, 2255‘a kadar herhangi bir tabana kolayca genişletilebilir, ancak 216‘ın üzerindeki her taban, sayı yalnızca 16 bit olduğundan aynı sayıyı yazdıracaktır.

Baz eklemek için:

  1. x‘in 2n olduğu yeni bir ‘BASEx’ sabiti tanımlayın. D adlı alt bayt, D = n‘dir. S adlı üst bayt, üst grubun bit cinsinden konumudur. S = n · (⌈16/n⌉ - 1) şeklinde hesaplanabilir.
  2. ‘DIGITS’ dizisine gerekli rakamları ekleyin.

Örnek: taban 32 ekleme

D = 5 ve S = 15’e sahibiz, bu yüzden BASE32 EQU 0f05h tanımlıyoruz. Ardından on altı basamak daha ekleriz: DIGITS db "0123456789abcdefghijklmnopqrstuv".

Açık olması gerektiği gibi, rakamlar ‘DIGITS’ dizesini düzenleyerek değiştirilebilir.

1 B bir taban ise, tanım başına B basamaklıdır. Böylece basamak başına bit sayısı log2(B) olur. İki tabanın gücü için bu, tanım gereği bir tamsayı olan log2(2n) = n‘yi basitleştirir.

2 Bu bağlamda, söz konusu tabanın iki taban 2n kuvveti olduğu örtük olarak varsayılır.

3 B = 2n tabanının tam sayıda bit grubuna sahip olması için, n | 16 (n 16’yı böler). 16’nın tek çarpanı 2 olduğundan, n’nin kendisinin ikinin kuvveti olması gerekir. Yani B, 22k veya eşdeğeri olarak log2(log2 biçimindedir) >(B)) bir tam sayı olmalıdır.

MS-DOS, TASM/MASM, 16 bitlik bir sayıyı ondalık olarak yazdırma işlevi #

16 bitlik işaretsiz bir sayıyı ondalık olarak yazdırın #

Rakamları yazdırmak için kesme hizmeti Int 21/AH=02h kullanılır. sayı‘dan sayı‘ya standart dönüştürme ‘div’ komutuyla gerçekleştirilir, temettü başlangıçta on uygun 16 bitin (104) en yüksek gücüdür ve daha düşük güçlere indirgenir her yinelemede.

parametreler #

Parametreler itme sırasına göre gösterilir. Her biri 16 bittir.

parametre Açıklama
sayı Ondalık olarak yazdırılacak 16 bitlik işaretsiz sayı
baştaki sıfırları göster 0 ise, anlamlı olmayan sıfırlar yazdırılmaz, aksi takdirde yazdırılırlar. 0 sayısı her zaman “0” olarak yazdırılır

kullanım #

push 241
push 0
call print_dec          ;prints 241

push 56
push 1
call print_dec          ;prints 00056

push 0
push 0
call print_dec          ;prints 0

kod #

;Parameters (in order of push):
;
;number
;Show leading zeros
print_dec:
 push bp
 mov bp, sp

 push ax
 push bx
 push cx
 push dx

 ;Set up registers:
 ;AX = Number left to print
 ;BX = Power of ten to extract the current digit
 ;DX = Scratch/Needed for DIV
 ;CX = Scratch

 mov ax, WORD PTR [bp+06h]
 mov bx, 10000d
 xor dx, dx

_pd_convert:     
 div bx                           ;DX = Number without highmost digit, AX = Highmost digit
 mov cx, dx                       ;Number left to print

 ;If digit is non zero or param for leading zeros is non zero
 ;print the digit
 or WORD PTR [bp+04h], ax
 jnz _pd_print

 ;If both are zeros, make sure to show at least one digit so that 0 prints as "0"
 cmp bx, 1
 jne _pd_continue

_pd_print:

 ;Print digit in AL

 mov dl, al
 add dl, '0'
 mov ah, 02h
 int 21h

_pd_continue:
 ;BX = BX/10
 ;DX = 0

 mov ax, bx
 xor dx, dx
 mov bx, 10d
 div bx
 mov bx, ax

 ;Put what's left of the number in AX again and repeat...
 mov ax, cx

 ;...Until the divisor is zero
 test bx, bx
jnz _pd_convert

 pop dx
 pop cx
 pop bx
 pop ax

 pop bp
 ret 04h

NASM taşıma #

Kodu NASM’ye taşımak için “PTR” anahtar sözcüğünü bellek erişimlerinden kaldırın (ör. “mov ax, WORD PTR [bp+06h]”, “mov ax, WORD [bp+06h]” olur)
