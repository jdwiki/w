  • SystemV x86-64 ABI (resmi standart). Windows dışındaki tüm işletim sistemleri tarafından kullanılır. (Bu github wiki sayfası, H.J. Lu tarafından güncel tutulmuştur, 32bit, 64bit ve x32 bağlantıları vardır. Ayrıca ABI yöneticileri/katkıda bulunanlar için resmi foruma bağlantılar.) Ayrıca [clang/gcc işaretinin /zero, ABI’nin yazıldığı gibi gerektirmemesine rağmen, dar argümanları 32bit’e[5] genişletir. Clang tarafından oluşturulan kod buna bağlıdır.

  • SystemV 32bit (i386) ABI (resmi standart), Linux ve Unix tarafından kullanılır. (eski sürüm).

  • diğerlerine bağlantılar içeren OS X 32bit x86 çağrı kuralı. 64bit arama kuralı System V’dir. Apple’ın sitesi bunun için sadece bir FreeBSD pdf’sine bağlantı verir.

  • Windows x86-64 __fastcall çağrı kuralı

  • Windows __vectorcall: 32bit ve 64bit sürümlerini belgeler

  • Windows 32bit __stdcall: Win32 API işlevlerini çağırmak için kullanılır. Bu sayfa, diğer çağrı kuralı belgelerine bağlantı verir (ör. “__cdecl”).

  • [Windows64 neden x86-64’teki diğer tüm işletim sistemlerinden farklı bir çağrı kuralı kullanıyor?](http://stackoverflow.com/questions/4429398/why-does-windows64-use-a- Different-calling-convention-from -all-other-oses-on-x86): bazı ilginç tarihler, özellikle. posta listesi arşivlerinin herkese açık olduğu ve AMD’nin ilk silikonunu piyasaya sürmesinden önceye giden SysV ABI için.

32 bit cdecl #

cdecl, birçok POSIX işletim sisteminde kullanılan çağrı kuralına çok benzeyen bir Windows 32-bit işlev çağırma kuralıdır (i386 System V ABI‘de belgelenmiştir). Farklılıklardan biri, küçük yapıların döndürülmesidir.

Parametreler #

Parametreler, çağrı sırasında yığındaki en düşük adreste ilk argüman olacak şekilde yığına iletilir (son olarak gönderilir, bu nedenle işleve girişte dönüş adresinin hemen üzerindedir). Arayan, aramadan sonra parametreleri yığından geri almaktan sorumludur.

Geri dönüş değeri #

Skaler dönüş türleri için, dönüş değeri EAX’e veya 64 bit tamsayılar için EDX:EAX’e yerleştirilir. Kayan nokta türleri st0 (x87) olarak döndürülür. Yapılar gibi daha büyük türlerin döndürülmesi, örtük bir ilk parametre olarak geçirilen bir işaretçi ile başvuru yoluyla yapılır. (Bu işaretçi EAX’te döndürülür, bu nedenle arayanın ne geçtiğini hatırlaması gerekmez).

Kaydedilmiş ve Kapatılmış Kayıtlar #

EBX, EDI, ESI, EBP ve ESP (ve FP / SSE yuvarlama modu ayarları) arayan tarafından korunmalıdır, böylece arayan kişi bir çağrı tarafından değiştirilmemiş olan bu kayıtlara güvenebilir.

Diğer tüm kayıtlar (EAX, ECX, EDX, FLAGS (DF dışında), x87 ve vektör kayıtları) aranan tarafından serbestçe değiştirilebilir; Çağıran, işlev çağrısından önce ve sonra bir değeri korumak isterse, değeri başka bir yere (kaydedilen kayıtlardan birinde veya yığında olduğu gibi) kaydetmesi gerekir.

32 bit standart çağrı #

stdcall, 32 bit Windows API çağrıları için kullanılır.

Parametreler #

Parametreler, ilk parametre yığının en üstüne en yakın olacak şekilde yığında iletilir. Arayan, geri dönmeden önce bu değerleri yığından çıkaracaktır.

Geri dönüş değeri #

Skaler dönüş değerleri EAX’e yerleştirilir.

Kaydedilmiş ve Kapatılmış Kayıtlar #

EAX, ECX ve EDX aranan tarafından serbestçe değiştirilebilir ve istenirse arayan tarafından kaydedilmelidir. EBX, ESI, EDI ve EBP, değiştirilirse ve dönüşte orijinal değerlerine geri yüklenirse, aranan tarafından kaydedilmelidir.

32 bit, cdecl — Kayan Noktayla Başa Çıkma #

Parametre olarak (float, double) #

Float’lar 32 bit boyutundadır, yığın üzerinde doğal olarak geçirilirler. Çiftlerin boyutu 64 bittir, Little Endian kuralına uygun olarak yığında geçirilir1, önce üstteki 32 biti sonra da alttakileri itin.

//C prototype of callee
double foo(double a, float b);

foo(3.1457, 0.241);

;Assembly call

;3.1457 is 0x40092A64C2F837B5ULL
;0.241 is 0x3e76c8b4

push DWORD  3e76c8b4h        ;b, is 32 bits, nothing special here
push DWORD 0c2f837b5h        ;a, is 64 bits, Higher part of 3.1457
push DWORD  40092a64h        ;a, is 64 bits, Lower part of 3.1457
call foo
add esp, 0ch

;Call, using the FPU
;ST(0) = a, ST(1) = b
sub esp, 0ch
fstp QWORD PTR [esp]        ;Storing a as a QWORD on the stack
fstp DWORD PTR [esp+08h]    ;Storing b as a DWORD on the stack
call foo
add esp, 0ch

Parametre olarak (uzun çift) #

Uzun çiftler 80 bit2 genişliğindedir, yığında ise bir TBYTE ile depolanabilir. yığını aynı hizada tutmak için iki adet 32 ​​bitlik itme ve bir adet 16 bitlik itme (4 + 4 + 2 = 10 için) 4 bayt, 12 bayt işgal ederek sona erer, böylece üç 32 bitlik itme kullanır. Little Endian kuralına uyarak, önce 79-64 bitleri 3, sonra 63-32 bitleri itilir 31-0 bitleri takip eder.

//C prototype of the callee
void  __attribute__((cdecl)) foo(long double a);

foo(3.1457);

;Call to foo in assembly
;3.1457 is 0x4000c9532617c1bda800

push DWORD 4000h        ;Bits 79-64, as 32 bits push
push DWORD 0c9532617h        ;Bits 63-32
push DWORD 0c1bda800h        ;Bits 31-0
call foo
add esp, 0ch

;Call to foo, using the FPU
;ST(0) = a

sub esp, 0ch
fstp TBYTE PTR [esp]        ;Store a as ten byte on the stack
call foo
add esp, 0ch

Dönüş değeri olarak #

Boyutu ne olursa olsun bir kayan nokta değeri, ST(0)4 içinde döndürülür.

//C
float one() { return 1; }

;Assembly
fld1            ;ST(0) = 1
ret

//C
double zero() { return 0; }

;Assembly
fldz            ;ST(0) = 0
ret

//C
long double pi() { return PI; }

;Assembly
fldpi            ;ST(0) = PI
ret

1 Alt adreste DWORD’u düşürün.

2 On Bayt’tan TBYTE olarak bilinir.

3 Herhangi bir uzantıyla tam genişlikte bir itme kullanıldığında, daha yüksek WORD kullanılmaz.

4 Hangisi TBYE genişliğindedir, tam sayıların aksine FP’nin her zaman gerekenden daha fazla hassasiyetle döndürülür.

32 bit, cdecl — Tamsayılarla İşlem Yapma #

Parametre olarak (8, 16, 32 bit) #

8, 16, 32 bit tamsayılar her zaman yığında tam genişlikte 32 bitlik değerler olarak geçirilir1. İmzalı veya sıfırlanmış uzatmaya gerek yoktur. Arayan sadece alt kısmını kullanacak tam genişlik değerleri.

//C prototype of the callee
void  __attribute__((cdecl)) foo(char a, short b, int c, long d);

foo(-1, 2, -3, 4);


;Call to foo in assembly

push DWORD 4             ;d, long is 32 bits, nothing special here
push DWORD 0fffffffdh    ;c, int is 32 bits, nothing special here
push DWORD 0badb0002h    ;b, short is 16 bits, higher WORD can be any value
push DWORD 0badbadffh    ;a, char is 8 bits, higher three bytes can be any value
call foo
add esp, 10h             ;Clean up the stack

Parametre olarak (64 bit) #

64 bitlik değerler, küçük endian dikkate alınarak iki itme kullanılarak yığında iletilir kural2, önce yüksek 32 biti, sonra alt bitleri itin.

//C prototype of the callee
void  __attribute__((cdecl)) foo(char a, short b, int c, long d);

foo(0x0123456789abcdefLL);

;Call to foo in assembly

push DWORD 89abcdefh        ;Higher DWORD of 0123456789abcdef
push DWORD 01234567h        ;Lower DWORD of 0123456789abcdef
call foo
add esp, 08h

Dönüş değeri olarak #

8 bitlik tamsayılar “AL” içinde döndürülür ve sonunda tüm “eax"ı sekteye uğratır. 16 bitlik tamsayılar “AX” içinde döndürülür ve sonunda “eax"ın tamamı engellenir. 32 bit tam sayılar “EAX” içinde döndürülür. 64 bit tam sayılar “EDX:EAX” içinde döndürülür, burada “EAX” alt 32 biti ve “EDX” üsttekileri tutar.

//C
char foo() { return -1; }

;Assembly
mov al, 0ffh
ret

//C
unsigned short foo() { return 2; }

;Assembly
mov ax, 2
ret

//C
int foo() { return -3; }

;Assembly
mov eax, 0fffffffdh
ret

//C
int foo() { return 4; }

;Assembly
xor edx, edx            ;EDX = 0
mov eax, 4            ;EAX = 4
ret

1 Bu, yığının doğal kelime boyutu olan 4 bayt üzerinde hizalı kalmasını sağlar. Ayrıca bir x86 CPU, uzun modda olmadığında yalnızca 2 veya 4 bayta basabilir.

2 Alt adreste alt DWORD

64 bit Windows #

Parametreler #

İlk 4 parametre (sırasıyla) RCX, RDX, R8 ve R9’da geçirilir. XMM0 ila XMM3, kayan nokta parametrelerini iletmek için kullanılır.

Diğer parametreler yığına iletilir.

64 bit’ten büyük parametreler adrese göre iletilir.

Dökülme Alanı

Fonksiyon 4’ten az parametre kullansa bile, çağıran her zaman yığında 4 QWORD boyutlu parametre için alan sağlar. Arayan kişi bunları herhangi bir amaç için kullanmakta özgürdür, başka bir arama tarafından dökülecekleri takdirde parametreleri oraya kopyalamak yaygındır.

Geri dönüş değeri #

Skaler dönüş türleri için dönüş değeri RAX’e yerleştirilir. Dönüş türü 64 bitten büyükse (örneğin yapılar için) RAX bunun bir göstergesidir.

Kaydedilmiş ve Kapatılmış Kayıtlar #

Parametre geçişinde (RCX, RDX, R8, R9 ve XMM0’dan XMM3’e), RAX, R10, R11, XMM4 ve XMM5’te kullanılan tüm kayıtlar, aranan tarafından dökülebilir. Diğer tüm kayıtların arayan tarafından korunması gerekir (örneğin yığında).

Yığın hizalama #

Yığın 16 bayt hizalı tutulmalıdır. “call” komutu 8 baytlık bir dönüş adresini ittiğinden, bu, 16 baytlık hizalamayı geri yüklemek için her yaprak olmayan işlevin yığını 16n+8 biçiminde bir değerle ayarlayacağı anlamına gelir. Bir aramadan sonra yığını temizlemek arayanların işidir.

Kaynak: Çağrı kurallarının geçmişi, bölüm 5: amd64 Raymond Chen

32-bit, cdecl — Yapılarla Başa Çıkma #

Dolgu malzemesi #

Unutmayın, bir yapının üyeleri, doğal sınırlarında hizalanmalarını sağlamak için genellikle dolguludur:

struct t
{
    int a, b, c, d;    // a is at offset 0, b at 4, c at 8, d at 0ch
    char e;            // e is at 10h
    short f;           // f is at 12h (naturally aligned)
    long g;            // g is at 14h
    char h;            // h is at 18h
    long i;            // i is at 1ch (naturally aligned)
};

Parametre olarak (referans olarak iletin) #

Başvuru yoluyla iletildiğinde, yığındaki ilk argüman olarak bellekteki yapıya bir işaretçi iletilir. Bu, doğal boyutlu (32 bit) bir tamsayı değeri iletmeye eşdeğerdir; ayrıntılar için https://www.wikiod.com/tr/x86/cagr-kurallar#32 bit cdecl adresine bakın.

Parametre olarak (değere göre iletin) #

Değere göre iletildiğinde, yapılar orijinal bellek düzenine uygun olarak yığına tamamen kopyalanır (yani, ilk üye alt adreste olacaktır).

int __attribute__((cdecl)) foo(struct t a);

struct t s = {0, -1, 2, -3, -4, 5, -6, 7, -8};
foo(s);

; Assembly call

push DWORD 0fffffff8h    ; i (-8)
push DWORD 0badbad07h    ; h (7), pushed as DWORD to naturally align i, upper bytes can be garbage
push DWORD 0fffffffah    ; g (-6)
push WORD 5              ; f (5)
push WORD 033fch         ; e (-4), pushed as WORD to naturally align f, upper byte can be garbage
push DWORD 0fffffffdh    ; d (-3)
push DWORD 2             ; c (2)
push DWORD 0ffffffffh    ; b (-1)
push DWORD 0             ; a (0)
call foo
add esp, 20h

Dönüş değeri olarak #

Önemsiz olmadıkça1, yapılar geri dönmeden önce arayan tarafından sağlanan bir arabelleğe kopyalanır. Bu, gizli bir ilk parametre olan “struct S *retval"e sahip olmakla eşdeğerdir (burada “struct S”, yapının türüdür).

İşlev, bu işaretçiyle ’eax’ içindeki dönüş değerine dönmelidir; Arayanın, işaretçiyi “çağrı"dan hemen önce ittiği dönüş değerine tutan “eax"a bağlı olmasına izin verilir.

struct S
{
    unsigned char a, b, c;
};

struct S foo();         // compiled as struct S* foo(struct S* _out)

Gizli parametre, aranan tarafından işlenmesi gerektiğinden yığın temizleme amacıyla parametre sayımına eklenmez.

sub esp, 04h        ; allocate space for the struct

; call to foo
push esp            ; pointer to the output buffer
call foo
add esp, 00h        ; still as no parameters have been passed

Yukarıdaki örnekte, yapı yığının en üstüne kaydedilecektir.

struct S foo()
{
    struct S s;
    s.a = 1; s.b = -2; s.c = 3;
    return s;
}

; Assembly code
push ebx
mov eax, DWORD PTR [esp+08h]   ; access hidden parameter, it is a pointer to a buffer
mov ebx, 03fe01h               ; struct value, can be held in a register
mov DWORD [eax], ebx           ; copy the structure into the output buffer 
pop ebx
ret 04h                        ; remove the hidden parameter from the stack
                               ; EAX = pointer to the output buffer

1 Bir “önemsiz” yapı, yapı olmayan, dizi olmayan türden (en fazla 32 bit) yalnızca bir üye içeren yapıdır. Bu tür yapılar için, o üyenin değeri basitçe ’eax’ kaydında döndürülür. (Bu davranış, Linux’u hedefleyen GCC ile gözlemlenmiştir)

cdecl’nin Windows sürümü, System V ABI’nin çağrı kuralından farklıdır: “Önemsiz” bir yapının, yapı olmayan, dizi olmayan türden (en fazla 32 bit) iki üye içermesine izin verilir. Bu değerler, tıpkı 64 bitlik bir tamsayı gibi “eax” ve “edx” olarak döndürülür. (Bu davranış, Win32’yi hedefleyen MSVC ve Clang için gözlemlenmiştir.)

64-bit Sistem V #

Bu, birçok POSIX işletim sisteminde 64 bit uygulamalar için varsayılan çağrı kuralıdır.

Parametreler #

İlk sekiz skaler parametre (sırasıyla) RDI, RSI, RDX, RCX, R8, R9, R10, R11’de geçirilir. İlk sekizi geçen parametreler, daha önceki parametreler yığının tepesine daha yakın olacak şekilde yığına yerleştirilir. Arayan, artık gerekmediğinde, aramadan sonra bu değerleri yığından çıkarmaktan sorumludur.

Geri dönüş değeri #

Skaler dönüş türleri için dönüş değeri RAX’e yerleştirilir. Yapılar gibi daha büyük türlerin döndürülmesi, dönüş değerinin yerleştirileceği bir konuma işaretçi olan parametre listesinin başına bir parametre eklemek için işlevin imzasını kavramsal olarak değiştirerek yapılır.

Kaydedilmiş ve Kapatılmış Kayıtlar #

RBP, RBX ve R12–R15, aranan kişi tarafından korunur. Diğer tüm kayıtlar, aranan tarafından değiştirilebilir ve arayan, daha sonra bu değeri kullanmak isterse, bir kaydın değerini (örneğin yığında) korumalıdır.
