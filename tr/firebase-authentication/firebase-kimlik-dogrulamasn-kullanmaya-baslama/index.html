Parola Tabanlı Kullanıcı Nasıl Oluşturulur #

Kullanıcılarınızın e-posta adreslerini ve şifrelerini kullanarak Firebase ile kimlik doğrulaması yapmasına izin vermek ve uygulamanızın şifre tabanlı hesaplarını yönetmek için Firebase Authentication’ı kullanabilirsiniz.

Bu örnekte, JavaScript’e dayalı Android projemiz için kurmak için bu adımları kullanıyoruz.

Ancak bunu yapmadan önce, yapılması gereken şudur:

  1. JavaScript projenize Firebase’i ekleyin.
  2. Uygulamanızı henüz Firebase projenize bağlamadıysanız, bunu Firebase konsolundan yapın.
  3. E-posta/Parola ile oturum açmayı etkinleştirin: Firebase konsolunda Auth bölümünü açın. Oturum açma yöntemi sekmesinde, E-posta/parola ile oturum açma yöntemini etkinleştirin ve Kaydet’e tıklayın.

displayName ile parola tabanlı bir kullanıcı oluşturmak için gereken 2 kimlik doğrulama yöntemi vardır, bunlar .createUserWithEmailAndPassword ve .updateProfile. İkincisini iç içe geçirdim ve kullanım kolaylığı için bu yöntemlerin her ikisini de ateşleyen tek bir işlev oluşturdum.

  function registerPasswordUser(email,displayName,password,photoURL){
    var user = null;
    //NULLIFY EMPTY ARGUMENTS
    for (var i = 0; i < arguments.length; i++) {
      arguments[i] = arguments[i] ? arguments[i] : null;
    }
    auth.createUserWithEmailAndPassword(email, password)
    .then(function () {
      user = auth.currentUser;
      user.sendEmailVerification();
    })
    .then(function () {
      user.updateProfile({
        displayName: displayName,
        photoURL: photoURL
      });
    })
    .catch(function(error) {
      console.log(error.message,7000);
    });
    console.log('Validation link was sent to ' + email + '.');
  }

Özet #

Firebase v3 Web kimlik doğrulamasının tam işlevsel demosu buradan görüntülenebilir. Facebook, Github, Google, Twitter, şifre tabanlı ve anonim hesaplarla oturum açın. Github’da mevcut kodu, okunması ve takip edilmesi kolaydır ve iyi belgelenmiştir. Odak noktası, tamamen işlevsel kimlik doğrulama sistemidir.

Parola tabanlı kullanıcılara bir doğrulama bağlantısı gönderilir. Ayrıca e-posta adreslerini ve şifrelerini de değiştirebilirler - bu olayların her ikisi de ek bir güvenlik önlemi olarak bir doğrulama e-postası gönderir.

Son olarak, kimlik doğrulama, istemci tarafı yetkilendirme ve Firebase Gerçek Zamanlı Veritabanı Güvenlik Kuralları ile güvence altına alınan sunucu tarafı yetkilendirme arasındaki fark gösterilmektedir.

  1. Ön koşullar

    1. A Firebase Web project. FREE!
    2. An IDE. What’s an IDE? Try Cloud9. FREE!
    3. A Github, Google, Facebook, and Twitter account. FREE!
    4. Two email accounts. FREE!

  2. IDE’nizi Yapılandırın 2. Create an HTML5 project. 3. Install Firebase Tools. npm install -g firebase-tools 4. Using Firebase Tools command line, login to your Firebase project. firebase login --no-localhost 5. Using Firebase Tools command line, setup a Firebase project in the current directory. firebase init 6. Clone this set of files and folders to your IDE. git clone https://github.com/rhroyston/firebase-auth.git 7. Using Firebase Tools command line, push your IDE project to your Firebase project. firebase deploy 8. View Firebase project in your browser. Any broken images or errors? Easy fix below. 8. You may need to update href, src, and background: url in all JS, CSS, and all HTML files depending on your Web hosting folder structure . 1. Use Find feature to search for both href and src and update as necessary. 2. Browser Console will display any remaining incorrect paths errors. 3. Note script.js line 781 privateLink.href = "../firebase-auth/private" the .. seems to be required. 4. Once all pages render properly from Firebase Hosting (no broken images or console errors), continue.

  3. Firebase’i yapılandırın

    1. Enable all 6 forms of authentication. Follow the instructions on configuring social media site settings.
    2. Customize the Email Action Handler URL to point to your Firebase Web app URL + ‘/ack’, e.g. https://my-app-1234/ack.

  4. Web uygulamasına giriş yapın

    1. Login using an oAuth provider.
    2. From the browser command line, use the exposed demo.update('mynode','myKey','myValue') method to add secure markup to your Realtime Database.
      1. A success message will show up in your browser console.
      2. You may need to update the href path to match your folder structure.

Firebase v3 Kimlik Doğrulama ve Yetkilendirme Demo İncelemesi #

  1. Her bir oAuth sağlayıcısını kullanarak giriş yapın

    1. Notice that updating email address or password options are not present in your Account page.
    2. Notice any extra links in the side menu drawer?
    3. Try Deleting your account. What Happens?

  2. Parola tabanlı bir kullanıcı olarak kaydolun

    1. Did you get a verification email?
    2. Can you view private data until you clicked the verification link?
    3. Can you change your password?
    4. Can you change your email address?
    5. Can you undo the email change by clicking the email change notification email revokation link?

  3. Çıkış

    1. What links are present in the side menu drawer?
    2. Can you access private data?
    3. Can you view private data?
