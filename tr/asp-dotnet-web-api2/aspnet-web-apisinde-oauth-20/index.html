Bir Android Uygulamasına Kaydolma

Bir Android uygulaması kullanarak giriş yapmak/kaydolmak için attığım adımlar şunlardır:

  • ExternalLogins yolunu sorgulayan bir oturum açma etkinliğine sahip olun, mevcut sağlayıcılar. Bu aktivite NoHistory bayrağına sahip olmalıdır etkinleştirin ve tek bir örnek olarak başlatın.
  • Bir kullanıcının düğmesine basıldığında, Özel Chrome Sekmesi’ni başlatın. sağlayıcının URL’si. Kullanıcı oturum açmalı ve geri yönlendirilmeli verilen dönüş URL’sinde yayınlanan siteniz. WebView kullanmayın.
  • Özel bir URI şeması kullanarak bu sayfanın kullanıcıyı yeniden yönlendirmesini sağlayın. erişim ile uygulamanız içinde bir oturum açma sonrası etkinliği başlatın ek veri olarak eklenen belirteç, son kullanma tarihi ve kullanıcı hesabı ayrıntıları. Bunun web sayfasında JavaScript’te yapılması gerekecek, çünkü sunucunun denetleyicileri URL parametrelerine erişemez.
  • Kullanıcının ayrıntılarını ve belirtecini yerel bir MySQL veritabanında saklayın. Her birinde giriş yapın, belirtecin hala güncel olup olmadığını kontrol edin.
  • API’ye yapılan tüm çağrılar artık Yetkilendirme kullanılarak yetkilendirilebilir HTTP başlığı, depolanmış jetonunuz şu şekilde eklendi: “Bearer {token}”

Bir OAuth Sağlayıcısını Yapılandırma #

Seçtiğiniz OAuth sağlayıcınızdan bazı ayrıntıları almanız gerekir. Google’a bakacağız, ancak ASP.NET ayrıca Twitter, Facebook ve Microsoft’un (tabii ki) kullanıma hazır kullanımına izin verecek şekilde ayarlanmıştır.

Google geliştirici konsoluna (https://console.developers.google.com/) gidip bir proje oluşturmak, Google+ API’sini etkinleştirmek (kullanıcının adı ve avatarı gibi profil bilgilerini almak için) ve “Kimlik Bilgileri” bölümünde yeni bir OAuth 2 İstemci Kimliği oluşturun. Yetkili JavaScript kaynakları, projenizin kök URL’si (ör. https://yourapi.azurewebsites.net) olmalıdır ve yönlendirme URI’leri, ASP’nin yerleşik Google geri arama uç noktasını (https://yourapi.azurewebsites.net/signin-google) içermelidir. ) yanı sıra seçtiğiniz geri arama rotanız (https://yourapi.azurewebsites.net/callback). Bunları yanlış yapmak, Google’ın tıslama uyumuna neden olur.

Visual Studio projenize geri dönün, App_Start > Startup.Auth.cs’yi açın. Google Developers Console’dan ID ve Secret ekleyerek alttaki yorum yapılan Google bölümünü aşağıdaki kodla değiştirin:

var googleAuthOptions = new GoogleOAuth2AuthenticationOptions()
{
      ClientId = "YOUR ID",
      ClientSecret = "YOUR SECRET",
            Provider = new GoogleOAuth2AuthenticationProvider()
            {
                  OnAuthenticated = (context) =>
                  {
                        context.Identity.AddClaim(new Claim("urn:google:name", context.Identity.FindFirstValue(ClaimTypes.Name)));
                        context.Identity.AddClaim(new Claim("urn:google:email", context.Identity.FindFirstValue(ClaimTypes.Email)));
                        //This following line is need to retrieve the profile image
                        context.Identity.AddClaim(new Claim("urn:google:accesstoken", context.AccessToken, ClaimValueTypes.String, "Google"));
                        return System.Threading.Tasks.Task.FromResult(0);
                  }
            }
      };
app.UseGoogleAuthentication(googleAuthOptions);

Bu ek hak talepleri, kullanıcının adı ve avatar URL’si gibi profil bilgilerini Google’da sorgulamanıza olanak tanır.

OAuth Kullanıcı Profillerini Saklama #

Birisi uygulamanıza kaydolduğunda, veritabanında yeni bir ApplicationUser nesnesi depolanacaktır. Varsayılan olarak sınıf çok barebone’dur, ancak özelleştirilebilir - onu Modeller > IdentityModels.cs içinde bulabilirsiniz. Bu benim:

public class ApplicationUser : IdentityUser
{
      public string ImageUrl { get; set; }
      public DateTime DateCreated { get; set; }
      public string FirstName { get; set; }
      public string AuthProvider { get; set; }
      public string Surname { get; set; }

      public async Task<ClaimsIdentity> GenerateUserIdentityAsync(UserManager<ApplicationUser> manager, string authenticationType)
      {
            // Note the authenticationType must match the one defined in CookieAuthenticationOptions.AuthenticationType
            var userIdentity = await manager.CreateIdentityAsync(this, authenticationType);
            // Add custom user claims here
            return userIdentity;
      }
}

Başvuru için, Google+ API etkinleştirilmiş olarak Google’dan döndürülen kullanıcı profili aşağıdaki JSON yapısını alır:

{{
    "id": "1****************6",
    email": "dan********@gmail.com",
    "verified_email": true,
    "name": "Dan Richardson",
    "given_name": "Dan",
    "family_name": "Richardson",
    "link": "https://plus.google.com/+DanRichardson",
    "picture": "https://lh4.googleusercontent.com/photo.jpg",
    "gender": "male",
    "locale": "en"
}}

Site Kökü Dışındaki Yönlendirme URL’lerine İzin Verme #

Sağlayıcılar > ApplicationOAuthProvider.cs’e gidin ve ValidateClientRedirectUri işlevini düzenleyin. Bu benim için büyük bir sorundu, sanki bunu yapmazsan fevkalade yararsız bir hata mesajı olacak. Varsayılan olarak, bu kod sitenizin köküne yapılmadığı sürece sitenize yapılan tüm geri aramaları geçersiz kılar. Büyük olasılıkla bir denetleyicideki geri aramaları işleyebilmek istersiniz, bu nedenle bunu şuna benzer bir şeyle değiştirmeniz gerekir:

public override Task ValidateClientRedirectUri(OAuthValidateClientRedirectUriContext context)
{
      if (context.ClientId == _publicClientId)
      {
            Uri expectedRootUri = new Uri(context.Request.Uri, "/");
            Uri expectedCallbackUri = new Uri(context.Request.Uri, "/callback");

            if (expectedRootUri.AbsoluteUri == context.RedirectUri ||
                  expectedCallbackUri.AbsoluteUri == context.RedirectUri)
            {
                  context.Validated();
            }
      }
      return Task.FromResult<object>(null);
}

Kayıt Akışı #

Bir kullanıcıyı Web API’sine kaydetmenin varsayılan akışı buradadır. Bu yolların tümü AccountController’da bulunabilir:

  • Kullanıcı, aşağıdakileri kullanarak oturum açma sağlayıcılarının bir listesini ister: GetExternalLogins yolu, bir dönüş URL’sini parametre olarak iletir. Bu sağlayıcının adını içeren bir dizi sağlayıcı nesnesi döndürür ve onunla giriş yapmak için talep edilmesi gereken rota (her biri verilen dönüş URL’sini kullanacak şekilde yapılandırılmıştır).

    e.g. GET: /api/Hesap/ExternalLogins?returnUrl=/callback&generateState=true, istenen dönüş URL’sinin /geri arama olduğu yer

  • Kullanıcı, bu döndürülen URL’lerden birini bir tarayıcıda arar. sağlayıcının oturum açma sayfasına yönlendirilir. Giriş yaptıktan sonra, sağlayıcı bir tanımlama bilgisinin oluşturulmasını yöneten ASP’ye geri iletir. harici kullanıcı hesabı.

  • Kullanıcı, girişte ilettikleri dönüş URL’sine yönlendirilecektir. ilk adım. Kullanıcıya bir harici erişim belirteci geri gönderilir, URL’ye # param ile eklenir. Bu belirteç yalnızca seçimde kullanılabilir RegisterExternal gibi yollar.

  • Kullanıcı şimdi yeni kullanarak RegisterExternal’a bir POST isteği gönderir. Taşıyıcı anahtarı olarak erişim belirteci. ASP daha sonra yeni bir ApplicationUser oluşturur ve herhangi bir rotada kullanılabilecek uygun bir erişim belirteci döndürür.

OAuth Profil Bilgilerini Saklama #

Web API şablonunun bozuk olduğunu buldum - varsayılan uygulama, muhtemelen bir Rest API’de kullanmak istemediğiniz son adımda çerezlere dayanır. Tanımlama bilgisi olmadan, RegisterExternal’daki GetExternalLoginInfoAsync her zaman null değerini döndürür. RegisterExternal’ı tamamen kaldırdım, bunun yerine GetExternalLogin’de son kullanıcı hesabını oluşturdum - OAuth sağlayıcısından (bu durumda Google) geri döndüğünde çağrılır:

[OverrideAuthentication]
[HostAuthentication(DefaultAuthenticationTypes.ExternalCookie)]
[AllowAnonymous]
[Route("ExternalLogin", Name = "ExternalLogin")]
public async Task<IHttpActionResult> GetExternalLogin(string provider, string error = null)
{
      if (error != null)
      {
            return Redirect(Url.Content("~/") + "#error=" + Uri.EscapeDataString(error));
      }

      if (!User.Identity.IsAuthenticated)
      {
            return new ChallengeResult(provider, this);
      }

      ExternalLoginData externalLogin = ExternalLoginData.FromIdentity(User.Identity as ClaimsIdentity);

      if (externalLogin == null)
      {
            return InternalServerError();
      }

      if (externalLogin.LoginProvider != provider)
      {
            Authentication.SignOut(DefaultAuthenticationTypes.ExternalCookie);
            return new ChallengeResult(provider, this);
      }

      ApplicationUser user = await UserManager.FindAsync(new UserLoginInfo(externalLogin.LoginProvider,
                externalLogin.ProviderKey));

      bool hasRegistered = user != null;

      if (hasRegistered)
      {
            Authentication.SignOut(DefaultAuthenticationTypes.ExternalCookie);
                
            ClaimsIdentity oAuthIdentity = await user.GenerateUserIdentityAsync(UserManager,
                    OAuthDefaults.AuthenticationType);
            ClaimsIdentity cookieIdentity = await user.GenerateUserIdentityAsync(UserManager,
            CookieAuthenticationDefaults.AuthenticationType);

            AuthenticationProperties properties = ApplicationOAuthProvider.CreateProperties(user.UserName);
                Authentication.SignIn(properties, oAuthIdentity, cookieIdentity);
      }
      else
      {
            var accessToken = Authentication.User.Claims.Where(c => c.Type.Equals("urn:google:accesstoken")).Select(c => c.Value).FirstOrDefault();
            Uri apiRequestUri = new Uri("https://www.googleapis.com/oauth2/v2/userinfo?access_token=" + accessToken);

            RegisterExternalBindingModel model = new RegisterExternalBindingModel();

            using (var webClient = new System.Net.WebClient())
            {
                  var json = webClient.DownloadString(apiRequestUri);
                  dynamic jsonResult = JsonConvert.DeserializeObject(json);
                  model.Email = jsonResult.email;
                  model.Picture = jsonResult.picture;
                  model.Family_name = jsonResult.family_name;
                  model.Given_name = jsonResult.given_name;
            }

            var appUser = new ApplicationUser
            {
                  UserName = model.Email,
                  Email = model.Email,
                  ImageUrl = model.Picture,
                  FirstName = model.Given_name,
                  Surname = model.Family_name,
                  DateCreated = DateTime.Now
            };

            var loginInfo = await Authentication.GetExternalLoginInfoAsync();

            IdentityResult result = await UserManager.CreateAsync(appUser);
            if (!result.Succeeded)
            {
                  return GetErrorResult(result);
            }

            result = await UserManager.AddLoginAsync(appUser.Id, loginInfo.Login);
            if (!result.Succeeded)
            {
                  return GetErrorResult(result);
            }

            ClaimsIdentity oAuthIdentity = await appUser.GenerateUserIdentityAsync(UserManager,
                   OAuthDefaults.AuthenticationType);
            ClaimsIdentity cookieIdentity = await appUser.GenerateUserIdentityAsync(UserManager,
                    CookieAuthenticationDefaults.AuthenticationType);

            AuthenticationProperties properties = ApplicationOAuthProvider.CreateProperties(appUser.UserName);
            Authentication.SignIn(properties, oAuthIdentity, cookieIdentity);
      }

      return Ok();
}

Bu, istemci uygulamasının harici bir erişim belirteci aldıktan sonra gereksiz bir POST isteği gerçekleştirme ihtiyacını ortadan kaldırır.
