Spring Data projesi, uygulama programcılarının “Depo” adlı bir soyutlamayı kullanan tutarlı bir arabirim kullanarak veri depolarıyla çalışmasına olanak tanır. Bir Spring Data “Repository”, domain-driven design tarafından popüler hale getirilen Depo kalıbından sonra modellenmiştir. /Etki alanına dayalı_tasarım). Spring Data, alt projelerin veri depolarına özgü özellikler sağlamak için genişletebileceği ‘Depo’ adlı merkezi bir Java arabirimi sağlar.

‘Depo’ arayüzüne ek olarak, Spring Data ayrıca iki çekirdek arayüz daha sağlar - temel CRUD (create, read, update ve delete) işlevselliği için sözleşmeyi tanımlayan ‘CrudRepository’; ve sayfalandırma ve sıralama için bir sözleşme tanımlayarak ‘CrudRepository’yi genişleten ‘PagingAndSortingRepository’.

Bu üç temel arayüz (“Repository”, “CrudRepository” ve “PagingAndSortingRepository”) şunları sağlar:

  1. Uygulama programcıları, veri depolarına (ilişkisel veritabanları, belge tabanlı NoSQL veritabanları, grafik veritabanları vb. gibi) tutarlı bir şekilde erişebilir.
  2. Bir alan varlığı için temel depolamayı değiştirmek kolaydır (bkz. uygulamanın veri deposuyla etkileşime girdiği.
  3. Belirli uygulamalar, veri depolarına özgü özellikler sağlayabilir.

JPA tarafından yönetilen bir varlık için bir havuz oluşturma #

Varlık sınıfı

@Entity
@Table(name = "USER")
public class User {
    @Id
    @Column(name = "ID")
    private Long id;
    
    @Column(name = "USERNAME")
    private String username;
    
    @ManyToOne
    @JoinColumn("ORGANIZATION_ID")
    private Organization organization;
}

Depo arayüzü

@Repository
public interface UserRepository extends CrudRepository<User, Long> {
    public User findByUsername(String username);
}

Arayüzdeki yöntem bildirimi aşağıdaki jpql sorgusunu üretecektir:

select u from User u where u.username = :username

alternatif olarak özel bir sorgu tanımlayabiliriz:

@Query("select u from User u where u.username = :username")
public User findByUsername(@Param("username") String username)

yöntem bildirimine kolayca sıralama ekleyebiliriz:

public interface UserRepository extends PagingAndSortingRepository<User, Long> {
    public User findByUsernameOrderByUsernameAsc(String username);
}

yerleşik sayfalama desteğini de kullanabiliriz:

public Page<User> findByOrganizationPaged(Organization organization, Pageable pageable);

hizmet katmanı (veya bu yöntemi kim çağırırsa) daha sonra yönteme bir PageRequest iletir:

public Page<User> getByOrganizationPagedOrderByUsername(Organization organization, int page, int size, String direction){
    return userRepository.findByOrganizationPaged(organization, new PageRequest(page, size, Direction.valueOf(direction),
                "username")
}

Bir varlık sınıfının tüm örneklerini bulma #

Bir varlık sınıfının tüm örnekleri (nesneleri), temel alınan veritabanı tablosundan aşağıdaki gibi yüklenebilir (tablodan tüm satırları almaya benzer):

Iterable<Foo> foos = fooRepository.findAll();

‘findAll’ yöntemi, ‘CrudRepository’ arabirimi tarafından sağlanır. “Liste” veya “Set” gibi daha somut bir tür yerine “Yinelenebilir” döndürür, çünkü [arayüzün bazı uygulamaları bir “Koleksiyon” türü döndüremeyebilir](https://jira.spring.io/browse /DATACMNS-21) ve bu nedenle döndürülen değer için bir ‘Koleksiyon’ türü kullanmak, onlar için işlevsellik kaybına neden olur.

‘findAll’ yönteminin çağrılması, temel alınan veritabanında yürütülen JPA sorgusu ‘select foo from Foo foo’ ile sonuçlanır.

Tanımlayıcı tarafından bir varlık sınıfının belirli bir örneğini bulma #

Bir varlık sınıfının belirli bir örneği aşağıdaki gibi yüklenebilir:

Foo foo = fooRepository.findOne(id);

‘findOne’ yöntemi, ‘CrudRepository’ arabirimi tarafından sağlanır. Bir varlık örneğini benzersiz şekilde tanımlayan bir tanımlayıcı bekler (örneğin, bir veritabanı tablosundaki birincil anahtar). “id” parametresinin Java türü, JPA “@Id” notuyla açıklamalı varlık özniteliğine atanan türle eşleşmelidir.

‘findOne’ yönteminin çağrılması, JPA sorgusu foo foo'dan foo'yu seçin, burada foo.[birincil-anahtar- sütun] = :id temel alınan veritabanında yürütülüyor.

Belirtilen bir değerle eşleşen bir özniteliğe sahip bir varlık sınıfının tüm örneklerini bulma #

Belirtilen bir değerle eşleşen sınıf özniteliklerinden birine sahip bir varlık sınıfının tüm örnekleri aşağıdaki gibi alınabilir:

public interface FooRepository extends CrudRepository<Foo, Long> {
  List<Foo> findAllByName(String name);
}

“findAllByName” yöntemini çağırmak, JPA sorgusu foo foo'dan foo seçin, burada foo.name = :name ile sonuçlanır temel alınan veritabanında yürütülecektir.

Dikkat edilecek noktalar

  1. “name”, “Foo” varlık sınıfında bir öznitelik olmalıdır.
  2. Yöntem adı ‘find’, ‘get’ veya ‘read’ ile başlamalıdır. “Seç” gibi diğer anahtar kelimeler çalışmayacaktır.
  3. Sonuçların hangi sırayla iade edileceğine dair bir garanti yoktur.
