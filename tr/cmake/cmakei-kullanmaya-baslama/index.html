Kurulumu Yap #

CMake indirme sayfasına gidin ve işletim sisteminiz için bir ikili dosya edinin, ör. Windows, Linux veya Mac OS X. Windows’ta kurmak için ikili dosyaya çift tıklayın. Linux’ta ikili dosyayı bir terminalden çalıştırın.

Linux’ta paketleri dağıtımın paket yöneticisinden de yükleyebilirsiniz. Ubuntu 16.04’te komut satırını ve grafik uygulamasını aşağıdakilerle yükleyebilirsiniz:

sudo apt-get install cmake
sudo apt-get install cmake-gui

FreeBSD’de komut satırını ve Qt tabanlı grafik uygulamasını aşağıdakilerle kurabilirsiniz:

pkg install cmake
pkg install cmake-gui

Mac OSX’te, yazılımınızı kurmak için mevcut olan paket yöneticilerinden birini kullanırsanız, en dikkat çekici olanı MacPorts (MacPorts) ve Homebrew (Homebrew), CMake’i aşağıdakilerden biri aracılığıyla da kurabilirsiniz. onlara. Örneğin, MacPorts olması durumunda aşağıdakini yazarak

sudo port install cmake

CMake’i kuracak, Homebrew paket yöneticisini kullanmanız durumunda yazacaksınız

brew install cmake

CMake’i kurduktan sonra aşağıdakileri yaparak kolayca kontrol edebilirsiniz.

cmake --version

Aşağıdakine benzer bir şey görmelisiniz

cmake version 3.5.1

CMake suite maintained and supported by Kitware (kitware.com/cmake).

Basit “Merhaba Dünya” Projesi #

Bir “main()” işlevini tanımlayan bir “main.cpp” C++ kaynak dosyası verildiğinde, eşlik eden bir “CMakeLists.txt” dosyası (aşağıdaki içerikle birlikte) CMake’e mevcut sistem ve varsayılan C++ derleyicisi için uygun oluşturma talimatlarını oluşturma talimatını verecektir. .

main.cpp (C++ Merhaba Dünya Örneği)

#include <iostream>

int main()
{
    std::cout << "Hello World!\n";
    return 0;
}

CMakeLists.txt

cmake_minimum_required(VERSION 2.4)

project(hello_world)

add_executable(app main.cpp)

Coliru’da canlı izleyin

  1. cmake_minimum_required(VERSION 2.4) mevcut komut dosyasını değerlendirmek için gereken minimum CMake sürümünü ayarlar.

  2. project(hello_world) yeni bir CMake projesi başlatır. Bu, birçok dahili CMake mantığını, özellikle de varsayılan C ve C++ derleyicisinin algılanmasını tetikleyecektir.

  3. add_executable(app main.cpp) ile, yapılandırılmış derleyiciyi çağıracak bir yapı hedefi ‘app’ oluşturulur Verilen kaynak dosya ‘main.cpp’den yürütülebilir bir ‘uygulama’ derlemek için geçerli ayar için bazı varsayılan bayraklarla.

Komut Satırı (Kaynak İçi-Derleme, önerilmez)

> cmake .
...
> cmake --build .

cmake . derleyici tespitini yapar, verilen . dizinindeki CMakeLists.txt dosyasını değerlendirir ve oluşturur geçerli çalışma dizinindeki yapı ortamı.

cmake --build . komutu, gerekli inşa/yap çağrısı için bir soyutlamadır.

Komut Satırı (Kaynak Dışı, önerilir)

Kaynak kodunuzu herhangi bir derleme yapıtından temiz tutmak için “kaynak dışı” derlemeler yapmalısınız.

> mkdir build
> cd build
> cmake ..
> cmake --build .

Veya CMake, yukarıdaki örnekten platform kabuğunuzun temel komutlarını da soyutlayabilir:

> cmake -E make_directory build
> cmake -E chdir build cmake .. 
> cmake --build build

Kitaplık olarak “Merhaba Dünya” #

Bu örnek, “Merhaba Dünya” programının bir kitaplık olarak nasıl dağıtılacağını ve bunun diğer hedeflerle nasıl ilişkilendirileceğini gösterir.

Diyelim ki https://www.wikiod.com/tr/docs/cmake/862/getting-started-with-cmake/22391/hello-world-with-multiple-source-files ile aynı kaynak/başlık dosyalarına sahibiz. #t=201610310659039267444 örnek. Birden çok kaynak dosyadan derlemek yerine, önce add_library() kullanarak ve ardından [target_link_libraries()][2 ile ana programa bağlayarak foo.cpp dosyasını bir kitaplık olarak dağıtabiliriz. ].

CMakeLists.txt dosyasını şu şekilde değiştiririz:

cmake_minimum_required(VERSION 2.4)

project(hello_world)

include_directories(${PROJECT_SOURCE_DIR})
add_library(applib foo.cpp)
add_executable(app main.cpp)
target_link_libraries(app applib)

ve aynı adımları izleyerek aynı sonucu alacağız.

Yapı türleri arasında geçiş yapma, ör. hata ayıklama ve serbest bırakma #

CMake, genellikle varsayılan derleyici ve bağlayıcı parametrelerini (oluşturulan hata ayıklama bilgileri gibi) veya alternatif kod yollarını etkileyen birkaç yapı türünü bilir.

Varsayılan olarak, CMake aşağıdaki yapı türlerini işleyebilir:

  • Hata Ayıklama: Genellikle hata ayıklama bilgileri içeren, optimizasyon vb. içermeyen klasik bir hata ayıklama yapısı.
  • Sürüm: Hata ayıklama bilgisi ve tam optimizasyon içermeyen tipik sürüm derlemeniz.
  • RelWithDebInfo:: Release ile aynı, ancak hata ayıklama bilgileri var.
  • MinSizeRel: Boyut için optimize edilmiş özel bir Release yapısı.

Konfigürasyonların nasıl işlendiği, kullanılan jeneratöre bağlıdır.

Bazı oluşturucular (Visual Studio gibi) birden çok yapılandırmayı destekler. CMake tüm konfigürasyonları bir kerede oluşturacaktır ve IDE’den veya oluşturmak istediğiniz konfigürasyonu --config CONFIG (cmake --build ile) kullanarak seçebilirsiniz. Bu oluşturucular için CMake, farklı konfigürasyonlardaki dosyaların birbirinin üzerine basmayacağı bir yapı dizini yapısı oluşturmak için elinden gelenin en iyisini yapacaktır.

Yalnızca tek bir yapılandırmayı (Unix Makefiles gibi) destekleyen jeneratörler farklı şekilde çalışır. Burada şu anda etkin olan yapılandırma, CMake değişkeninin “CMAKE_BUILD_TYPE” değeriyle belirlenir.

Örneğin, farklı bir yapı türü seçmek için aşağıdaki komut satırı komutları verilebilir:

cmake -DCMAKE_BUILD_TYPE=Debug path/to/source
cmake -DCMAKE_BUILD_TYPE=Release path/to/source

Bir CMake betiği, “CMAKE_BUILD_TYPE"nin kendisini ayarlamaktan kaçınmalıdır, çünkü bunu yapmak genellikle kullanıcıların sorumluluğundadır.

Tek yapılandırmalı oluşturucular için yapılandırmanın değiştirilmesi CMake’in yeniden çalıştırılmasını gerektirir. Sonraki bir derlemenin, önceki yapılandırma tarafından üretilen nesne dosyalarının üzerine yazması muhtemeldir.

Birden çok kaynak dosyayla “Merhaba Dünya” #

Önce include_directories() ile başlık dosyalarının dizinlerini belirleyebiliriz, ardından add_executable() ile hedef yürütülebilir dosyanın karşılık gelen kaynak dosyalarını belirtmemiz ve orada olduğundan emin olmamız gerekir. kaynak dosyalarda tam olarak bir “main()” işlevi.

Aşağıda basit bir örnek verilmiştir, tüm dosyaların PROJECT_SOURCE_DIR dizinine yerleştirildiği varsayılmaktadır.

main.cpp

#include "foo.h"

int main()
{
    foo();
    return 0;
}

foo.h

void foo();

foo.cpp

#include <iostream>
#include "foo.h"

void foo()
{
    std::cout << "Hello World!\n";
}

CMakeLists.txt

cmake_minimum_required(VERSION 2.4)

project(hello_world)

include_directories(${PROJECT_SOURCE_DIR})
add_executable(app main.cpp foo.cpp)  # be sure there's exactly one main() function in the source files

Projemizi oluşturmak için yukarıdaki örnekte aynı prosedürü takip edebiliriz. Ardından ‘app’ yürütüldüğünde yazdırılacaktır

>./app
Hello World!
