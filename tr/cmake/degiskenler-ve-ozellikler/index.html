Temel CMake değişkenlerinin basitliği, tam değişken sözdiziminin karmaşıklığını gizler. Bu sayfa çeşitli değişken durumları örneklerle belgeliyor ve kaçınılması gereken tuzaklara işaret ediyor.

Sözdizimi #

  • set(değişken_adı değeri [CACHE türü açıklaması [Kuvvet]])

Değişken isimleri büyük/küçük harf duyarlıdır. Değerleri string türündedir. Bir değişkenin değerine şu şekilde başvurulur:

${variable_name}

ve alıntılanan bir argüman içinde değerlendirilir

"${variable_name}/directory"

Değişkenler ve Global Değişkenler Önbelleği #

Çoğunlukla “normal değişkenler” kullanacaksınız:

set(VAR TRUE)
set(VAR "main.cpp")
set(VAR1 ${VAR2})

Ancak CMake, global “önbelleğe alınmış değişkenler” da biliyor (“CMakeCache.txt” içinde devam ediyor). Geçerli kapsamda aynı ada sahip normal ve önbelleğe alınmış değişkenler varsa, normal değişkenler önbelleğe alınmış olanları gizler:

cmake_minimum_required(VERSION 2.4)
project(VariablesTest)

set(VAR "CACHED-init" CACHE STRING "A test")
message("VAR = ${VAR}")

set(VAR "NORMAL")
message("VAR = ${VAR}")

set(VAR "CACHED" CACHE STRING "A test" FORCE)
message("VAR = ${VAR}")

İlk Çalıştırmanın Çıktısı

VAR = CACHED-init
VAR = NORMAL
VAR = CACHED

İkinci Çalıştırmanın Çıktısı

VAR = CACHED
VAR = NORMAL
VAR = CACHED

Not: ‘FORCE’ seçeneği aynı zamanda normal değişkeni geçerli kapsamdan kaldırır/kaldırır.

Önbelleğe Alınmış Değişkenler İçin Kullanım Örnekleri #

Tipik olarak iki kullanım durumu vardır (lütfen bunları global değişkenler için kötüye kullanmayın):

  1. Kodunuzdaki bir değer, projenizin kullanıcısı tarafından değiştirilebilir olmalıdır; cmakegui, ccmake ile /latest/manual/ccmake.1.html) veya cmake -D ... seçeneğiyle:

CMakeLists.txt / MyToolchain.cmake

   set(LIB_A_PATH "/some/default/path" CACHE PATH "Path to lib A")

Komut satırı

   $ cmake -D LIB_A_PATH:PATH="/some/other/path" ..

Bu, bu değeri önbellekte önceden ayarlar ve yukarıdaki satır onu değiştirmez.

CMUI yap

GUI’de kullanıcı önce yapılandırma sürecini başlatır, ardından önbelleğe alınmış herhangi bir değeri değiştirebilir ve yapı ortamı oluşturma işlemini başlatarak bitirir.

  1. Ek olarak, CMake arama/test/derleyici tanımlama sonuçlarını önbelleğe alır (böylece yapılandırma/oluşturma adımlarını yeniden çalıştırırken bunu tekrar yapması gerekmez)

    find_path(LIB_A_PATH libA.a PATHS "/some/default/path")

Burada LIB_A_PATH önbelleğe alınmış bir değişken olarak oluşturulur.

Dizeler ve Listeler #

CMake’in listeleri ve düz dizeleri nasıl ayırt ettiğini bilmek önemlidir. Yazdığınızda:

set(VAR "a b c")

"a b c" değerine sahip bir dize oluşturursunuz. Ancak bu satırı tırnak işaretleri olmadan yazdığınızda:

set(VAR a b c)

Bunun yerine üç öğeden oluşan bir liste oluşturursunuz: "a", "b" ve "c".

Liste dışı değişkenler de aslında listelerdir (tek bir öğenin).

Listeler, listeleri birleştirmeye, onları aramaya, isteğe bağlı öğelere erişmeye vb. izin veren list() komutuyla çalıştırılabilir (documentation of list() /komut/list.html)).

Biraz kafa karıştırıcı, bir liste aynı zamanda bir dizedir. Çizgi

set(VAR a b c)

eşdeğerdir

set(VAR "a;b;c")

Bu nedenle, listeleri birleştirmek için set() komutu da kullanılabilir:

set(NEW_LIST "${OLD_LIST1};${OLD_LIST2})"

Önbelleğe Alınmış (Global) Değişken #

set(my_global_string "a string value"
    CACHE STRING "a description about the string variable")
set(my_global_bool TRUE
    CACHE BOOL "a description on the boolean cache entry")

CMake ilgili satırı işlerken (örneğin, CMake yeniden çalıştırıldığında) önbellekte önbelleğe alınmış bir değişken zaten tanımlanmışsa, değiştirilmez. Varsayılanın üzerine yazmak için son argüman olarak “FORCE” ifadesini ekleyin:

set(my_global_overwritten_string "foo"
    CACHE STRING "this is overwritten each time CMake is run" FORCE)

gprof kullanmak için CMake’e profil oluşturma bayrakları ekleme #

Buradaki olaylar dizisinin aşağıdaki gibi çalışması gerekiyor:

  1. Kodu -pg seçeneğiyle derleyin
  2. -pg seçeneğiyle bağlantı kodu
  3. Programı çalıştırın
  4. Program gmon.out dosyasını oluşturur
  5. gprof programını çalıştırın

Profil oluşturma bayrakları eklemek için CMakeLists.txt dosyanıza şunları eklemelisiniz:

SET(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -pg")
SET(CMAKE_EXE_LINKER_FLAGS "${CMAKE_EXE_LINKER_FLAGS} -pg")
SET(CMAKE_SHARED_LINKER_FLAGS "${CMAKE_SHARED_LINKER_FLAGS} -pg")

Derlemek ve bağlamak için bayraklar eklemeli ve programı yürüttükten sonra kullanmalıdır:

gprof ./my_exe

Aşağıdaki gibi bir hata alırsanız:

gmon.out: No such file or directory

Bu, derlemenin profil oluşturma bilgilerini doğru şekilde eklemediği anlamına gelir.

Yerel değişken #

set(my_variable "the value is a string")

Varsayılan olarak, bir yerel değişken yalnızca geçerli dizinde ve ‘add_subdirectory’ komutuyla eklenen tüm alt dizinlerde tanımlanır.

Bir değişkenin kapsamını genişletmek için iki olasılık vardır:

  1. ‘Önbelleğe Alın’, bu da onu küresel olarak kullanılabilir hale getirecek

  2. ‘PARENT_SCOPE’ kullanın, bu onu ana kapsamda kullanılabilir hale getirir. Üst kapsam, üst dizindeki “CMakeLists.txt” dosyası veya geçerli işlevin çağırıcısıdır.

    Technically the parent directory will be the CMakeLists.txt file that included the current file via the add_subdirectory command.
