Firebase Realtime Database ile Veritabanı kurallarınız, sunucu tarafı güvenliğinizdir. Veritabanınıza kimlerin eriştiği konusunda çok dikkatli olmanız ve farkında olmanız gerekir. Verilerinize erişmemesi gereken hiç kimsenin erişmemesi önemlidir.

Varsayılan olarak, Firebase Gerçek Zamanlı Veritabanı kuralları, kimliği doğrulanmış herhangi bir kullanıcının tüm verileri okumasına ve yazmasına izin verir, muhtemelen uygulamanızın yapmasını istediğiniz şey bu değildir.

Farklı senaryolar için aşağıdaki örneklere bir göz atın.

Firebase Gerçek Zamanlı Veritabanı, verilerinizin nasıl yapılandırılması gerektiğini, nasıl dizine eklenmesi gerektiğini ve verilerinizin ne zaman okunup yazılabileceğini kolayca tanımlamak için JavaScript benzeri sözdizimine sahip esnek, ifade tabanlı bir kural dili sağlar. Kimlik doğrulama hizmetlerimizle birlikte, kimin hangi verilere erişimi olduğunu tanımlayabilir ve kullanıcılarınızın kişisel bilgilerini yetkisiz erişime karşı koruyabilirsiniz.

Varsayılan olarak, veritabanı kurallarınız Firebase Kimlik Doğrulaması gerektirir ve yalnızca kimliği doğrulanmış kullanıcılara tam okuma ve yazma izinleri verir. Varsayılan kurallar, siz i’yi yapılandırma şansınız olmadan önce veritabanınızın herhangi biri tarafından erişilebilir olmamasını sağlar.

Resmi Belgeler #

https://firebase.google.com/docs/database/security/quickstart

Yalnızca kimliği doğrulanmış kullanıcılara erişim nasıl verilir #

Burada, kimliği doğrulanmış her kullanıcıya /users/$user_id konumunda kişisel bir düğüm veren bir kural örneği verilmiştir; burada $user_id, Kimlik Doğrulama yoluyla elde edilen kullanıcının kimliğidir.

// These rules grant access to a node matching the authenticated
// user's ID from the Firebase auth token
{
  "rules": {
    "users": {
      "$user_id": {
        ".read": "$user_id === auth.uid",
        ".write": "$user_id === auth.uid"
      }
    }
  }
}

Gruptan belirli bir öğenin okunmasına nasıl izin verilir, ancak grup üyelerinin listelenmesi engellenir #

Anahtar olarak öğe kimliğine sahip basit değer düğümleri oluşturarak öğe grupları oluşturmak yaygın bir uygulamadır. Örneğin, /administrators/$user_id konumunda true değerinde bir düğüm oluşturarak “yöneticiler” grubuna bir kullanıcı ekleyebiliriz. Güvenlik nedeniyle kimsenin yöneticilerin kim olduğunu bilmesini istemiyoruz, ancak yine de Kimliği Doğrulanmış bir kullanıcının yönetici olup olmadığını kontrol etmek istiyoruz. Bu kurallarla tam olarak şunu yapabiliriz:

{
  "rules": {
    "administrators": {
      // No one can list administrators
      ".read": "false",
      "$uid": {
        // Authenticated user can check if they are in this group
        ".read": "$uid === auth.uid",
        // Administrators can write
        ".write": "data.parent().child(auth.uid).val() === true",
        // Allow only add or delete, no duplicates
        ".validate": "!data.exists() || !newData.exists() || newData.isBoolean()",
      }
    }
  }
}

Varsayılan kurallar #

Varsayılan kurallar, Kimlik Doğrulama gerektirir. Uygulamanızın kimliği doğrulanmış kullanıcılarına tam okuma ve yazma erişimi sağlarlar. Verilerin uygulamanızın tüm kullanıcılarına açık olmasını istiyor ancak dünyaya açılmasını istemiyorsanız kullanışlıdırlar.

// These rules require authentication
{
  "rules": {
    ".read": "auth != null",
    ".write": "auth != null"
  }
}

Dosyalarınızı herkes tarafından okunabilir ve yazılabilir hale getirme #

Sadece tanımlayın:

// These rules give anyone, even people who are not users of your app,
// read and write access to your database
{
  "rules": {
    ".read": true,
    ".write": true
  }
}

Geliştirme sırasında faydalı olabilir, ancak dikkat edin çünkü Bu erişim düzeyi, herkesin veritabanınızı okuyabileceği veya yazabileceği anlamına gelir.

Okuma ve yazma erişimi nasıl devre dışı bırakılır #

Kullanıcıların veritabanınıza okuma ve yazma erişimini devre dışı bırakmak için özel kurallar tanımlayabilirsiniz. Bu kurallarla, veritabanına yalnızca yönetici ayrıcalıklarına sahip olduğunuzda erişebilirsiniz (bu, veritabanına Firebase konsolu üzerinden erişerek veya bir sunucudan oturum açarak belgeler/veritabanı/sunucu/başlangıç)).

// These rules don't allow anyone read or write access to your database
{
  "rules": {
    ".read": false,
    ".write": false
  }
}

Kurallar nasıl yapılandırılır #

  1. Firebase konsoluna gidin.
  2. Projenizi seçin
  3. Soldaki Veritabanı bölümüne tıklayın ve ardından Kurallar sekmesini seçin.

Güvenlik kurallarınızı üretime geçirmeden önce test etmek isterseniz, kural düzenleyicinin sağ üst kısmındaki Simulate düğmesini kullanarak konsoldaki işlemleri simüle edebilirsiniz.
