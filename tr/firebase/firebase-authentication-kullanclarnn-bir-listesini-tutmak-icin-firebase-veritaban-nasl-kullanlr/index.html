Kullanıcı profili verileri nasıl kaydedilir #

Kimliği doğrulanmış her kullanıcının, tüm sağlayıcılar arasında benzersiz olan ve her kimlik doğrulama yönteminin sonucunda döndürülen bir Firebase “uid"i vardır.

Kullanıcınızın verilerini depolamanın iyi bir yolu, tüm kullanıcıların verilerini tutmak ve güvenlik kurallarınızı kullanarak korumak için bir düğüm oluşturmaktır.

- Veri tabanı

{
   "users": {
      "uid1" : {
         "name": "Steve",
         "surname": "Jobs"
      },
      "uid2" : {
         "name": "Bill",
         "surname": "Gates"
      }
   }
}

- Güvenlik

{
    "rules": {
        "users": {
            "$uid": {
                // If node's key matches the id of the auth user
                ".write": "$uid == auth.uid"
            }
        }
    }
}

Yukarıdaki kurallardaki “$uid”, altındaki kuralların “kullanıcıların” tüm alt düğümlerine uygulanmasını sağlayan “dolar değişkeni” olarak adlandırılır. Daha fazla bilgi için Yol Segmentlerini Yakalamak için $ Değişkenlerini Kullanma hakkındaki belgelere bakın.

Kullanıcı verilerini neden veritabanına kaydetmelisiniz? #

Firebase Kimlik Doğrulaması, uygulamanızın kullanıcılarının sosyal sağlayıcılarla veya e-posta+şifreleriyle oturum açmasına olanak tanır. Ancak bir kullanıcı hakkında, Firebase Authentication’ın belirlemenize izin verdiğinin ötesinde ek bilgiler depolamak isterseniz ne olur?

Ya da uygulamanızdaki kullanıcıların bir listesini görüntülemek isterseniz? Firebase Authentication bunun için bir API’ye sahip değildir.

Çoğu geliştirici, ek bilgileri ayrı bir veritabanında depolayarak bu sorunu çözer. Bu konu, bu tür bilgilerin Firebase Gerçek Zamanlı Veritabanında nasıl saklanacağını kapsar.

Gerçek Zamanlı Veritabanında Kullanıcı Hesabı Verilerini İşleme #

Firebase yetkilendirme sistemi, bir kullanıcının “uid”, “displayName”, “photoURL” ve belki “email” kaynağıdır. Parola tabanlı hesaplar, bu persistent değerlerini auth sisteminde .updateProfile yöntemi ile ayarlar. Bu değerleri Realtime Database, rDB, users düğümünde saklamak, eski veri sorununu ortaya çıkarır. Örneğin görünen adlar değişebilir. Bu değerleri senkronize halde tutmak için “.onAuthStateChange” ile uyum içinde yerel depolamayı kullanın.

her .onAuthStateChange üzerinde

  • getItem('displayName') ve getItem('photoURL')
  • “user.displayName” ve “user.photoURL” ile karşılaştırın
  • farklı ise
    • setItem('displayName') and setItem('photoURL')
    • db.ref.child('users').update the values of displayName and/or photoURL

“.onAuthStateChange” her sayfa yükleme veya yeniden yüklemede ve ayrıca her auth durumu değişikliğinde tetiklenir. Potansiyel olarak sık sık ateşlenir, örn. çok sayfalı uygulamalar Ancak yerel depolamaya okuma ve yazma eşzamanlı ve çok hızlıdır, bu nedenle uygulama performansı üzerinde gözle görülür bir etki olmayacaktır.
