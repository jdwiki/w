Mac için Kurulum #

Paket yöneticisi Homebrew ‘brew’ kurulumu

Bunu bir Terminal istemine yapıştırın.

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Xcode IDE’yi yükleme

Aşağıdaki bağlantıyı kullanarak indirin veya Mac App Store’da bulun

https://developer.apple.com/download/

NOT: Xcode.app üretim sürümüyle birlikte Xcode-beta.app yüklüyse, “xcodebuild” aracının üretim sürümünü kullandığınızdan emin olun. Şunlarla ayarlayabilirsiniz:

sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer/

Android ortamını yükleme

  • Git git

*XCode’u kurduysanız, Git zaten kuruludur, aksi takdirde aşağıdakileri çalıştırın

   brew install git

  • En son JDK

  • Android Stüdyosu

    Choose a Custom installation

    Choose both Performance and Android Virtual Device

    After installation, choose Configure -> SDK Manager from the Android Studio welcome window.

    In the SDK Platforms window, choose Show Package Details and under Android 6.0 (Marshmallow), make sure that Google APIs, Intel x86 Atom System Image, Intel x86 Atom_64 System Image, and Google APIs Intel x86 Atom_64 System Image are checked.

    In the SDK Tools window, choose Show Package Details and under Android SDK Build Tools, make sure that Android SDK Build-Tools 23.0.1 is selected.

  • Ortam Değişkeni “ANDROID_HOME”

    Ensure the ANDROID_HOME environment variable points to your existing Android SDK. To do that, add this to your ~/.bashrc, ~/.bash_profile (or whatever your shell uses) and re-open your terminal:

    If you installed the SDK without Android Studio, then it may be something like: /usr/local/opt/android-sdk

    export ANDROID_HOME=~/Library/Android/sdk

Mac için bağımlılıklar

iOS için Xcode’a ve android için Android Studio’ya, node.js’ye, React Native komut satırı araçlarına ve Watchman’a ihtiyacınız olacak.

Homebrew aracılığıyla düğüm ve bekçi kurmanızı öneririz.

brew install node
brew install watchman

Watchman, Facebook’un dosya sistemindeki değişiklikleri izlemek için kullandığı bir araçtır. Daha iyi performans için yüklemeniz şiddetle tavsiye edilir. İsteğe bağlı.

Node, React Native komut satırı arayüzünü kurmanıza izin veren npm ile birlikte gelir.

npm install -g react-native-cli

Bir izin hatası alırsanız, sudo ile deneyin:

sudo npm install -g react-native-cli.

iOS için Xcode’u kurmanın en kolay yolu Mac App Store’dur. Ve android için Android Studio’yu indirin ve yükleyin.

Java kodunda değişiklik yapmayı planlıyorsanız, derlemeyi hızlandıran Gradle Daemon’u öneririz.

React Native Kurulumunuzu test etme

“AwesomeProject” adlı yeni bir React Native projesi oluşturmak için React Native komut satırı araçlarını kullanın, ardından yeni oluşturulan klasörün içinde react-native run-ios’u çalıştırın.

react-native init AwesomeProject
cd AwesomeProject
react-native run-ios

Yeni uygulamanızın kısa süre içinde iOS Simulator’da çalıştığını görmelisiniz. tepki-yerel run-ios, uygulamanızı çalıştırmanın yalnızca bir yoludur - ayrıca doğrudan Xcode veya Nuclide içinden de çalıştırabilirsiniz.

Uygulamanızı değiştirme

Uygulamayı başarıyla çalıştırdığınıza göre şimdi değiştirelim.

  • İstediğiniz metin düzenleyicide index.ios.js veya index.android.js dosyasını açın ve bazı satırları düzenleyin.
  • Uygulamayı yeniden yüklemek için iOS Simülatörünüzde Command⌘ + R tuşlarına basın ve değiştirmek! Bu kadar!

Tebrikler! İlk React Native uygulamanızı başarıyla çalıştırdınız ve değiştirdiniz.

Linux Kurulumu (Ubuntu) #

1) Kurulum Node.JS

Terminali başlatın ve nodeJS’yi kurmak için aşağıdaki komutları çalıştırın: #

curl -sL https://deb.nodesource.com/setup_5.x | sudo -E bash -

sudo apt-get install nodejs

Düğüm komutu kullanılamıyorsa #

sudo ln -s /usr/bin/nodejs /usr/bin/node

Alternatifler NodeJS kurulumları: #

curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -
sudo apt-get install -y nodejs

veya

curl -sL https://deb.nodesource.com/setup_7.x | sudo -E bash -
sudo apt-get install -y nodejs

mevcut sürüme sahip olup olmadığınızı kontrol edin #

node -v

React-native’i kurmak için npm’yi çalıştırın #

sudo npm install -g react-native-cli

2) Java’yı Kur

sudo apt-get install lib32stdc++6 lib32z1 openjdk-7-jdk

3) Android Studio’yu kurun:

Android SDK veya Android Studio #

http://developer.android.com/sdk/index.html

Android SDK ve ENV #

export ANDROID_HOME=/YOUR/LOCAL/ANDROID/SDK
export PATH=$PATH:$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools

4) Kurulum emülatörü:

Terminalde komutu çalıştırın

android

SDK Yöneticisinden “SDK Platformları"nı seçin ve “Android 7.0 (Nougat)“ın yanında mavi bir onay işareti görmelisiniz. Değilse, onay kutusunu ve ardından “Uygula"yı tıklayın.

5) Bir proje başlatın

Örnek uygulama başlatma #

react-native init ReactNativeDemo && cd ReactNativeDemo

Gözlem: Her zaman “android/app/build.gradle” üzerindeki sürümün android SDK’nıza indirilen Derleme Araçları ile aynı olup olmadığını kontrol edin #

android {
    compileSdkVersion XX
    buildToolsVersion "XX.X.X"
...

6) Projeyi çalıştırın

Sanal bir android kurmak için Android AVD’yi açın. Komut satırını yürütün: #

android avd

Sanal bir cihaz oluşturmak ve başlatmak için talimatları izleyin

Başka bir terminal açın ve komut satırlarını çalıştırın:

react-native run-android
react-native start

Windows için Kurulum #

Not: Windows’ta iOS için yerel tepki uygulamaları geliştiremezsiniz, yalnızca tepki yerel android uygulamaları geliştirebilirsiniz.

Windows’ta yerel tepki için resmi kurulum belgeleri [burada 1 bulunabilir]. Daha fazla ayrıntıya ihtiyacınız varsa, burada ayrıntılı bir kılavuz bulabilirsiniz.

Araçlar/Ortam

  • Windows 10
  • komut satırı aracı (örneğin, Powershell veya windows komut satırı)
  • Çikolata (PowerShell aracılığıyla kurulum adımları)
  • JDK (sürüm 8)
  • Android Stüdyosu
  • HAXM için etkinleştirilen Sanallaştırma teknolojisine sahip bir Intel makinesi (isteğe bağlı, yalnızca bir öykünücü kullanmak istiyorsanız gereklidir)

1) Makinenizi yerel tepki geliştirme için kurun

Komut satırını yönetici olarak başlatın, aşağıdaki komutları çalıştırın:

choco install nodejs.install
choco install python2

Npm’yi çalıştırabilmeniz için komut satırını yönetici olarak yeniden başlatın

npm install -g react-native-cli

Son komutu çalıştırdıktan sonra, react-native’in kurulu olduğu dizini kopyalayın. 4. Adım için buna ihtiyacınız olacak. Bunu iki bilgisayarda denedim, bir durumda: C:\Program Files (x86)\Nodist\v- x64\6.2.2. Diğerinde şuydu: C:\Users\admin\AppData\Roaming\npm

2) Ortam Değişkenlerinizi ayarlayın

Bu bölüm için resimli Adım Adım kılavuz burada bulunabilir.

Şuraya giderek Ortam Değişkenleri penceresini açın:

[Sağ tıklayın] “Başlat” menüsü -> Sistem -> Gelişmiş Sistem Ayarları -> Ortam Değişkenleri

Alt bölümde “Yol” Sistem Değişkenini bulun ve 1. adımda tepki-yerel’in yüklendiği konumu ekleyin.

ANDROID_HOME ortam değişkeni eklemediyseniz, bunu burada da yapmanız gerekecektir. Hala “Ortam Değişkenleri” penceresindeyken, “ANDROID_HOME” adında yeni bir Sistem Değişkeni ekleyin ve android sdk’nizin yolu olarak değer.

Ardından komut satırını yönetici olarak yeniden başlatın, böylece içinde yerel tepki komutları çalıştırabilirsiniz.

3) Projenizi oluşturun Komut satırında, projenizi yerleştirmek istediğiniz klasöre gidin ve aşağıdaki komutu çalıştırın:

react-native init ProjectName

4) Projenizi çalıştırın Android stüdyosundan bir emülatör başlatın Komut satırında projenizin kök dizinine gidin ve çalıştırın:

cd ProjectName
react-native run-android

Bağımlılık sorunları yaşayabilirsiniz. Örneğin, doğru derleme araçları sürümüne sahip olmadığınız bir hata olabilir. Bunu düzeltmek için Android Studio’da sdk yöneticisini açmanız ve buradan derleme araçlarını indirmeniz gerekir.

Tebrikler!

Kullanıcı arayüzünü yenilemek için öykünücüdeyken ve uygulamayı çalıştırırken r tuşuna iki kez basabilirsiniz. Geliştirici seçeneklerini görmek için ctrl + m tuşlarına basabilirsiniz.
