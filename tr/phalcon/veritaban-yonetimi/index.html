Phalcon Modeli kullanarak veritabanı yönetimi #

Terminal kök konumundan aşağıdaki komut çalıştırılarak yeni bir tablo için bir model oluşturulabilir:

phalcon model <table-name>

Model Kullanıcılarını alalım.

SEÇME

Phalcon’da seçme işlemi yapmak için iki varsayılan işlev vardır, find() ve findFirst()

findFirst(), geçmekte olduğumuz koşulları sağlayan ilk satırı almak için kullanılır. İlk satırdaki verilerle tek bir nesne döndürür.

Örnek:

$user = Users::findFirst("active=1 AND verified=1 AND email='a@a.com'");

Bu, kullanıcıyı verilen e-postayla döndürür ve doğrulanmış ve etkin sütunun değeri 1’dir.

find(), geçmekte olduğumuz koşulları sağlayan tüm satırları almak için kullanılır.

Örnek:

$users = Users::find("active=1 AND verified=1");

Bu, doğrulanmış ve etkin sütunun değeri 1 olan kullanıcıları döndürür.

SOKMAK

Ekleme aşağıdaki kod kullanılarak yapılabilir:

$user = new Users();

$user->name = "Arun";
$user->email = "abc@gmail.com";
$user->verified = 1;
$user->active = 1;

$user->save();

Bu değerleri içeren yeni bir satır eklenecektir.

GÜNCELLEME

Güncelleme aşağıdaki kod kullanılarak yapılabilir:

Öncelikle findFirst() kullanarak güncellememiz gereken satırı seçmeliyiz.

$user = Users::findFirst("email='a@a.com'");

$user->verified = 0;
$user->active = 0;

$user->save();

Bu, verilen e-posta ile satır için doğrulanmış ve etkin olan sütunun değerlerini değiştirecektir.

SİLMEK Silme işlemi findFirst() kullanılarak da yapılabilir.

Örnek:

Users::findFirst("email='a@a.com'")->delete();

Bu, verilen e-posta ile satırı siler.

Aşağıdaki kodu kullanarak özel sql komutlarını modellerle de çalıştırabilirsiniz:

$query = $this->modelsManager->createQuery("SELECT * FROM Users WHERE email='a@a.com'");

$user = $query->execute();

Standart SQL’i doğrudan modellerle kullanma #

SQL sözdizimini, sonucu uygun örneklere aktaracak modelle kullanmak için, doğrudan Phalcon\Mvc\Model\Resultset sınıflarından birini kullanmalısınız:

$users = new \Application\Models\Users();

// bitwise operation on `flag` field
$sql = 'SELECT * FROM phorum.users WHERE
        (15 & (1 << (flag - 1))) > 0  ORDER BY login DESC';

// as a result you will have a Resultset\Simple with Models\Users instances.
$result = new \Phalcon\Mvc\Model\Resultset\Simple(
    null,

    // what model to use for data returned from SQL
    $users,

    // setting result via "read connection" proper for this model.
    $users->getReadConnection()->query($sql) 
);

Varsayılan bağlantı hizmetini ayarlama #

Phalcon, veritabanlarına bağlantı sağlamak için varsayılan olarak ‘db’ hizmetini kullanır.

“Veritabanı” alanı ayarlanmış bir yapılandırma dosyanız olduğunu varsayarsak, projeniz için veritabanı ile bağlantı elde etmek için aşağıdaki kodu ekleyebilir veya otomatik olarak yükleyebilirsiniz:

$di->set('db', function () use ($config) {
    $dbconf = $config->database;
    switch(strtolower($dbconf->adapter)) {

        case 'mysql':
            return new \Phalcon\Db\Adapter\Pdo\Mysql(array(
                'host' => $dbconf->host,
                'username' => $dbconf->username,
                'password' => $dbconf->password,
                // default database to work with
                'dbname' => $dbconf->dbname,
                // default character set
                'charset' => $dbconf->charset,
                // connection warm-up commands for PDO
                'options' => array(
                    PDO::MYSQL_ATTR_INIT_COMMAND => 'SET NAMES "' . $dbconf->charset . '"',
                    PDO::ATTR_CASE => PDO::CASE_LOWER
                )
            ));

        case 'postgresql':
            return new \Phalcon\Db\Adapter\Pdo\Postgresql(array(
                'host' => $dbconf->host,
                'username' => $dbconf->username,
                'password' => $dbconf->password,
                'dbname' => $dbconf->dbname,
                'options' => array(
                )
            ));
        
        default: 
            throw new \Exception('Unimplemented database::adapter in config.ini');
    }
});

Modellerin Meta Verilerini Önbelleğe Alma. #

Phalcon, kullandığı tablolar hakkında bazı bilgiler oluşturur, böylece her şeyi elle uygulamadan bunlara eklenen verileri doğrulamak mümkündür. Bunlar modeller için meta verilerdir. Sayfa her yenilendiğinde Phalcon’un Meta Veri oluşturmasını hızlandırmak ve önlemek için, bunları önbelleğe almak mümkündür. Bunu yapmak için, kullanması için “metaData” hizmetini uygulamanız gerekir:

$di->set('modelsMetadata', function() use ($config)
{
    // assuming that you have a $config var with
    // models.metadata.adapter field declared
    switch (strtolower($config->models->metadata->adapter)) {
        case 'apc':
            $metaData = new MetaDataApcAdapter([
                'lifetime' => $config->models->metadata->lifetime,
                'suffix' => $config->models->metadata->suffix,
            ]);
            break;
        case 'xcache':
            $metaData = new MetaDataXCacheAdapter([
                'lifetime' => $config->models->metadata->lifetime,
                'prefix' => $config->models->metadata->suffix,
            ]);
            break;
        case 'memory':
            $metaData = new MetaDataMemoryAdapter();
            break;
        default:
            throw new \Exception('Unimplemented models::metadata.adapter in config.ini');
    }

    return $metaData;
});

Phalcons’un özel sayfasından daha fazla belge alınabilir.
