Temel Hareket #

Notlar #

  • Her hareket, bir operatör komutundan sonra kullanılabilir, bu nedenle komut, hareketin erişiminden oluşan metin üzerinde çalışır.
  • Operatör komutları gibi, hareketler de bir sayı içerebilir, bu nedenle örneğin 2word hareket edebilirsiniz.

Oklar #

Vim’de normal ok/imleç tuşları (←↓↑→) beklendiği gibi çalışır. Ancak, dokunmatik olarak yazanlar için hjkl alternatif tuşlarını kullanmak daha kolaydır. Tipik bir klavyede, aynı sırada yan yana bulunurlar ve sağ elle kolayca erişilebilirler. Hangisinin hangisi olduğunu hatırlamak için anımsatıcı teknik şu şekildedir:

  • h/l — bunlar klavyedeki dört harf arasında “en solda/sağda” bulunur, yani sırasıyla “sola/sağa gitmeye” eşdeğerdir;
  • j — küçük “j” harfinin kuyruğu, küçük bir ok gibi tipik harflerin altında “aşağıya” gider - yani “aşağıya” eşdeğerdir;
  • k — tersine, küçük “k” harfinin “yükselticisi”, küçük bir işaretçi gibi tipik harflerin üzerinde “yukarı” çıkar - yani “yukarı çıkmak” ile eşdeğerdir.

Temel hareketler #

Aşağıdaki tüm komutlar normal modda yapılmalıdır.

Komut Açıklama
h veya sol sola [say] karakter git
j veya aşağı karakterin altına [say] git
k veya yukarı [say] karakter yukarı git
l veya sağ sağa [say] karakter git
gg ilk satıra gidin veya verilmişse [say]‘inci satıra gidin
H görünür ekranda ilk satıra git
M görünür ekranda orta çizgiye gidin
L görünür ekranda son satıra git
G son satıra gidin veya verilmişse [say]‘inci satıra gidin
Ana Sayfa veya 0 satırın ilk karakterine git
^ satırın ilk boş olmayan karakterine git
+ bir satır aşağı ilk boş olmayan karaktere git
- ilk boş olmayan karaktere bir satır yukarı git
$ veya Son satırın sonuna git ([count] verilmişse, [count - 1] satır aşağı git)
| [count]‘inci karaktere gidin veya ‘count’ belirtilmemişse satırın başına gidin
f{char} {char} öğesinin [sayım]‘inci yerine git dahil
F{char} soldaki [sayım]’{char}‘ye git dahil
t{char} sağda [count]’ {char}‘nin geçtiği yere git exclusive
T{char} soldaki [sayım]’’ {char}‘ye git exclusive
; en son f, t, F veya T [count] kez tekrarlayın
, en son f, t, F veya T‘yi ters yönde tekrarlayın, [say]< /kbd> kez
w sonraki kelimenin başına git
b önceki kelimenin başına git
e sonraki kelimenin sonuna git
ge önceki kelimenin sonuna git
% eşleşen çiftlere gidin, örneğin (), [], {}, /* */ veya #if, #ifdef, #else, #elif, #endif
{ } önceki/sonraki paragraf
[{]} bloğun başlangıcı/bitişi
’{char} İşarete git (m{char} ile işaretle)
<C-B><C-F> önceki/sonraki sayfa
<C-O><C-I> “Atlama listesinde” geri veya ileri gidin (jumplist özelliği gerektirir, bkz.:help jumps)

Not: b, e ve w bir sözcüğü varsayılan olarak harf, sayı ve alt çizgi olarak kabul eder (bu, anahtar kelime ayarı). Bunların her biri de büyük harfle yazılabilir, bu da boşluk olmayan herhangi bir şeyi atlamalarına neden olur.

Not: Vim iki tür hareketi tanır: operatör hareketi (:yardım hareketi) ve atlamalar (:yardım atlama listesi). ‘g’ (‘gg’, ‘G’, ‘g,’) ile yürütülenler gibi hareketler, değişiklikler gibi sıçrama olarak sayılır. Değişiklikler, yukarıda ‘g,’ ve ‘g;’ aracılığıyla gezinilebilen kendi atlama listelerini alır (bkz. ‘:help changelist’). Atlamalar, Vim tarafından hareket komutları olarak ele alınmaz

Satırlar arasında yukarı veya aşağı hareket ederken, imleç beklendiği gibi sütununu korur. Yeni satır çok kısaysa, imleç yeni satırın sonuna gider. Sütun satırın sonundaysa, imleç satırın sonunda görüntülenir. İlk sütun numarası, onu değiştirmek için bir işlem yapılana kadar (metni düzenlemek veya sütunu açıkça taşımak gibi) korunur.

Bir satırın uzunluğu ekranın genişliğini aşarsa, metin kaydırılır (varsayılan ayarlar altında bu davranış yapılandırılabilir). Dosya içindeki satırlar yerine ekranda görüntülendiği gibi satırlar arasında hareket etmek için normal komutun önüne g ekleyin. Örneğin, gj, dosyanın aynı satırında olsa bile, imleci geçerli konumunun bir satır altında görüntülenen konuma taşır.

Aranıyor #

Karakterlere atlama #

f{char} - aynı satırda imlecin sağındaki bir sonraki {char} oluşumuna git

F{char} - aynı satırda imlecin solundaki bir sonraki {char} oluşumuna git

t{char} - aynı satırda imlecin sağındaki bir sonraki {char} oluşumunun soluna git

T{char} - aynı satırda imlecin solundaki {char} öğesinin bir sonraki oluşumunun sağına git

; ve , aracılığıyla ‘sonuçlar’ arasında ileri / geri atlayın.

Ayrıca, /<searchterm>Enter aracılığıyla tam kelimeleri arayabilirsiniz.

Dize aranıyor #

* - imlecin altındaki kelimenin bir sonraki geçtiği yere git

# - imlecin altındaki kelimenin önceki geçtiği yere git

/searchtermEnter sizi bir sonraki eşleşmeye getirir (ileriye doğru arama). / yerine ? kullanırsanız, arama geriye doğru gider.

n (sonraki) ve N (önceki) üzerinden maçlar arasında geçiş yapın.

Önceki aramalarınızı görüntülemek/düzenlemek için / yazın ve yukarı ok tuşuna basın.

Şu ayarlar da yararlıdır: (:senin :sete eşit olduğuna dikkat edin)

  • :se hls HighLightSearch, tüm arama eşleşmelerini vurgular; arama/işaret vurgulamayı geçici olarak kapatmak için :noh kullanın (:set noh veya :set nohls kapanır.)
  • :se is veya :set incs Artımlı Aramayı açar, imleç otomatik olarak bir sonraki eşleşmeye atlar. (:se nois kapanır.)
  • :se ic IgnoreCase, büyük/küçük harf duyarlılığını kapatır. (:se noic tekrar açılır.)
  • :se scs SmartCaSe, IgnoreCase ayarlandığında kullanılabilir; büyük/küçük harf duyarlılığını akıllı yapar! Örneğin. /the the, The, TheE vb. için arama yaparken /The yalnızca The kelimesini arayacaktır.

Belirli bir kelimenin başına gitme #

Metni düzenlerken, yaygın bir görev, ekrandaki belirli bir kelimeye gitmektir. Bu örneklerde “güncellendi” kelimesine nasıl gidebileceğimizi araştırıyoruz. Örnekler arasında tutarlılık olması için kelimenin ilk harfine inmeyi amaçlıyoruz.

Orta ekran atlama M$B

Bu yaklaşım, yalnızca 3 tuş vuruşunu kullanarak hızlıdır. Ancak dezavantajı, hedef çizgimizin ekranın tam ortasında olması yaygın olmadığı için çok genel olmamasıdır. Yine de daha az tanecikli hareketler yaparken faydalı bir harekettir.

Sayı kullanma 3jfu;;

İlk bakışta, bu, tuş vuruşlarının sayısı nedeniyle ilk yaklaşımdan bir adım geri gibi görünebilir. Ancak burada M yerine bir sayım kullandığımız için daha esnektir. [relativenumber][3] etkinleştirilmişse kullanılacak doğru sayımı hızlı bir şekilde belirleyebiliriz. Hedef kelimeye geçmek için f‘yi ; ile birlikte kullanmak şaşırtıcı derecede etkili olabilir - ve kesinlikle w‘ye tekrar tekrar basmaktan daha iyidir. ; ile hedefinizi aşarsanız, , ile geriye gidebilirsiniz.

Açık arama /yukarıEnternn

[![buraya resim açıklamasını girin][4]][4]

/ üzerinden gezinmek çok güçlü olabilir. Sıklıkla hedef kelimemize doğrudan yazarak atlayabiliriz. Burada, sözümüze benzersiz bir şekilde uyması umuduyla yalnızca ilk iki karakteri yazıyoruz. Ne yazık ki, birden fazla eşleşme var, ancak n ile hızlı bir şekilde bir sonraki maça geçebiliriz.

Örtülü arama /yBoşlukGirinw

[![buraya resim açıklamasını girin][5]][5]

Bazı durumlarda, doğrudan hedefimize gitmek yerine hedefimizin yakına atlamak daha verimli olabilir. Burada, hedefin hemen yanında nadiren ortaya çıkan bir ‘y’ harfi olduğunu gözlemliyoruz. Yol boyunca başka bir “y” karakterine çarpma şansımızı azaltmak için arama terimimize bir Boşluk ekleyebiliriz. Bu aynı zamanda, örneğin Sayı kullanma örneğinde olduğu gibi, f{char} ile büyük etki için kullanılabilir.

[3]: https://www.wikiod.com/tr/vim/verimliligi-artrmak-icin-ipuclar-ve-puf-noktalar#Göreli Satır Numaralarını Aç [4]: http://i.stack.imgur.com/Qq1Xa.gif [5]: http://i.stack.imgur.com/EarRy.gif

Desen Arama #

Vim, bir dosyada arama yaparken normal ifadelerin kullanımını destekler.

Arama yapmak istediğinizi belirten karakter /.

Yapabileceğiniz en basit arama şudur:

/if

Bu, tüm “if” örnekleri için tüm dosyayı arayacaktır. Ancak, ‘if’ aramamız aslında, diğer kelimelerin içindekiler de dahil olmak üzere ‘if’ kelimesinin herhangi bir oluşumuyla eşleşen normal bir ifadedir.

Örneğin, aramamız şu kelimelerin tümünün aramamızla eşleştiğini söyler: “if”, “spiffy”, “endif”, vb.

Daha karmaşık düzenli ifadeler kullanarak daha karmaşık aramalar yapabiliriz.

Aramamız şöyle olsaydı:

/\<if\>

o zaman aramamız yalnızca tam ‘if’ kelimesiyle tam eşleşmeleri döndürür. Yukarıdaki “spiffy” ve “endif” arama tarafından döndürülmez, yalnızca “if” döndürülür.

Aralıkları da kullanabiliriz. Bir dosya verildi:

hello1
hello2
hello3
hello4

“Merhaba” ve ardından 1 ile 3 arasında bir rakam içeren satırları aramak istiyorsak şunu söyleriz:

/hello[1-3]

Başka bir örnek:

/(?:\d*\.)?\d+

dosyadaki tüm tamsayı ve ondalık sayıları bulur.

Hareket Etmek İçin İşaretleri Kullanma #

İşaretler yer imleri gibidir; daha önce bulunduğunuz yerleri bulmanıza yardımcı olurlar.

TLDR #

Bunları m{a-zA-Z} ile normal modda ayarlayın ve '{a-zA-Z} (tek tırnak) veya `{a- ile normal veya görsel modda bunlara atlayın. zA-Z} (ters tik). Küçük harfler bir arabellek içindeki işaretler içindir ve büyük harfler ve rakamlar geneldir. Şu anda ayarlanmış işaretlerinizi :marks ile görün ve daha fazla bilgi için :help mark bölümüne bakın.

Bir işaret koy #

Vim’in yerleşik yardımı şunları söylüyor:

m{a-zA-Z}               Set mark {a-zA-Z} at cursor position (does not move
                        the cursor, this is not a motion command).

İşaret, hangi satıra ve sütuna yerleştirildiğini takip edecektir. Bir işaretin ayarlandığına veya bir işaretin önceki bir değeri olup olmadığına ve üzerine yazıldığına dair görsel bir onay yoktur.

Bir işarete atla #

Vim’in yerleşik yardımı şunları söylüyor:

Jumping to a mark can be done in two ways:  
1. With ` (backtick):     The cursor is positioned at the specified location
                          and the motion is exclusive.
2. With ' (single quote): The cursor is positioned on the first non-blank
                          character in the line of the specified location and
                          the motion is linewise.

Backtick, sütun konumunu kullanır, Tek tırnak ise kullanmaz. Aradaki fark, isterseniz işaretinizin sütun konumunu görmezden gelmenizi sağlar.

İşaretlere dayalı metin seçimine izin vermek için normal moda ek olarak görsel modda küresel olmayan işaretler arasında geçiş yapabilirsiniz.

Küresel İşaretler #

Genel işaretler (büyük harfler) dosyalar arasında geçişe izin verir. Bunun anlamı, örneğin, “A” işareti “foo.txt” içinde ayarlanmışsa, ardından “bar.txt"den (dosya sistemimin herhangi bir yerinde) “A” işaretine atlarsam, mevcut arabelleğim “foo.txt” ile değiştirildi. Vim, değişiklikleri kaydetmenizi isteyecektir.

Başka bir dosyadaki bir işarete atlamak bir hareket olarak kabul edilmez ve görsel seçimler (diğer şeylerin yanı sıra) bir arabellek içindeki işaretlere atlamak gibi çalışmayacaktır.

Önceki dosyaya (bu durumda bar.txt) geri dönmek için :b[uffer] # (yani, :b# veya :buffer#) kullanın.

Not:

Özel işaretler #

Vim’in otomatik olarak ayarladığı belirli işaretler vardır (bunları kendiniz üzerine yazabilirsiniz, ancak muhtemelen buna gerek duymazsınız).

Örneğin (Vim’in yardımından alıntılanmıştır):

`[` and `]`: jump to the first or last character of the previously changed or 
             yanked text.  {not in Vi}

`<` and `>`: jump to the first or last line (with `'`) or character (with 
             <code>`</code>) of the last selected Visual area in the current 
             buffer.  For block mode it may also be the last character in the 
             first line (to be able to define the block).  {not in Vi}.

Vim’in yerleşik yardımından daha fazlası:

''  ``              To the position before the latest jump, or where the
                    last "m'" or "m`" command was given.  Not set when the
                    :keepjumps command modifier was used.
                    Also see restore-position.


'"  `"              To the cursor position when last exiting the current
                    buffer.  Defaults to the first character of the first
                    line.  See last-position-jump for how to use this
                    for each opened file.
                    Only one position is remembered per buffer, not one
                    for each window.  As long as the buffer is visible in
                    a window the position won't be changed.
                    {not in Vi}.

'.  `.              To the position where the last change was made.  The
                    position is at or near where the change started.
                    Sometimes a command is executed as several changes,
                    then the position can be near the end of what the
                    command changed.  For example when inserting a word,
                    the position will be on the last character.
                    {not in Vi}

'"  `"              To the cursor position when last exiting the current
                    buffer.  Defaults to the first character of the first
                    line.  See last-position-jump for how to use this
                    for each opened file.
                    Only one position is remembered per buffer, not one
                    for each window.  As long as the buffer is visible in
                    a window the position won't be changed.
                    {not in Vi}.

'^  `^              To the position where the cursor was the last time
                    when Insert mode was stopped.  This is used by the
                    gi command.  Not set when the :keepjumps command
                    modifier was used.  {not in Vi}

Ek olarak, (,), { ve } karakterleri, normal mod komutlarıyla aynı konuma atlayan işaretlerdir - yani, '} normal modda aynı şeyi yapar } olarak.
