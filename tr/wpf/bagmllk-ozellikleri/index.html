Bağımlılık Özellikleri, bir CLR özelliğini genişleten bir özellik türüdür. Bir CLR özelliği doğrudan sınıfınızın bir üyesinden okunurken, nesnenizin temel DependencyObject sınıfından devralma yoluyla kazandığı GetValue() yöntemi çağrılırken bir Bağımlılık Özelliği dinamik olarak çözümlenir.

Bu bölüm, Bağımlılık Özelliklerini kıracak ve kullanımlarını hem kavramsal olarak hem de kod örnekleriyle açıklayacaktır.

Sözdizimi #

  • DependencyProperty.Register(string name, Type propertyType, Type OwnerType)
  • DependencyProperty.Register(dize adı, Type propertyType, Type OwnerType, PropertyMetadata typeMetadata)
  • DependencyProperty.Register(dize adı, Type propertyType, Type OwnerType, PropertyMetadata typeMetadata, ValidateValueCallback validateValueCallback)
  • DependencyProperty.RegisterAttached(dize adı, Type propertyType, Type OwnerType)
  • DependencyProperty.RegisterAttached(dize adı, Type propertyType, Type OwnerType, PropertyMetadata typeMetadata)
  • DependencyProperty.RegisterAttached(dize adı, Type propertyType, Type OwnerType, PropertyMetadata typeMetadata, ValidateValueCallback validateValueCallback)
  • DependencyProperty.RegisterReadOnly(dize adı, Type propertyType, Type OwnerType, PropertyMetadata typeMetadata)
  • DependencyProperty.RegisterReadOnly(dize adı, Type propertyType, Type OwnerType, PropertyMetadata typeMetadata, ValidateValueCallback validateValueCallback)
  • DependencyProperty.RegisterAttachedReadOnly(dize adı, Type propertyType, Type OwnerType, PropertyMetadata typeMetadata)
  • DependencyProperty.RegisterAttachedReadOnly(dize adı, Type propertyType, Type OwnerType, PropertyMetadata typeMetadata, ValidateValueCallback validateValueCallback)

Parametreler #

parametre Ayrıntılar
isim Özellik adının ‘Dize’ temsili
mülkTürü Özelliğin “Türü”, ör. typeof(int)
sahipTürü Özelliğin tanımlandığı sınıfın “Türü”, ör. “typeof(MyControl)” veya “typeof(MyAttachedProperties)”.
tipMeta veriler Varsayılan değerleri, özellik değiştirilen geri aramaları tanımlayan System.Windows.PropertyMetadata’nın (veya alt sınıflarından birinin) örneği, ‘FrameworkPropertyMetadata’, ‘System.Windows.Data.BindingMode.TwoWay’ gibi bağlama seçeneklerinin tanımlanmasına izin verir.
validateValueCallback Özelliğin yeni değeri geçerliyse true, aksi takdirde false döndüren özel geri arama.

Standart bağımlılık özellikleri #

Ne zaman kullanılır #

Hemen hemen tüm WPF denetimleri, bağımlılık özelliklerini yoğun şekilde kullanır. Bağımlılık özelliği, stiller, animasyonlar, veri bağlama, değer devralma ve değişiklik bildirimleri dahil ancak bunlarla sınırlı olmamak üzere, yalnızca standart CLR özellikleriyle mümkün olmayan birçok WPF özelliğinin kullanılmasına izin verir.

TextBox.Text özelliği, standart bir bağımlılık özelliğinin gerekli olduğu basit bir örnektir. Burada, “Metin” standart bir CLR özelliği olsaydı, veri bağlama mümkün olmazdı.

<TextBox Text="{Binding FirstName}" />

Nasıl tanımlanır #

Bağımlılık özellikleri yalnızca “FrameworkElement”, “Control” vb. gibi “DependencyObject"den türetilen sınıflarda tanımlanabilir.

Sözdizimini hatırlamak zorunda kalmadan standart bir bağımlılık özelliği oluşturmanın en hızlı yollarından biri, “propdp” snippet’ini “propdp” yazıp ardından Tab tuşuna basarak kullanmaktır. Daha sonra ihtiyaçlarınıza göre değiştirilebilecek bir kod parçacığı eklenecektir:

public class MyControl : Control
{
    public int MyProperty
    {
        get { return (int)GetValue(MyPropertyProperty); }
        set { SetValue(MyPropertyProperty, value); }
    }

    // Using a DependencyProperty as the backing store for MyProperty.
    // This enables animation, styling, binding, etc...
    public static readonly DependencyProperty MyPropertyProperty =
        DependencyProperty.Register("MyProperty", typeof(int), typeof(MyControl),
            new PropertyMetadata(0));
}

Özellik adı, özellik türü, içeren sınıf türü ve varsayılan değer dahil olmak üzere gerekli değişiklikleri yapmak için kod parçacığının farklı bölümlerinde Sekme yapmalısınız.

Önemli sözleşmeler #

Burada izlenecek birkaç önemli sözleşme/kural vardır:

  1. Bağımlılık özelliği için bir CLR özelliği oluşturun. Bu özellik, nesnenizin arka plan kodunda veya diğer tüketiciler tarafından kullanılır. Tüketicilerin bunu yapmasına gerek kalmaması için “GetValue” ve “SetValue"yi çağırmalıdır.

  2. Bağımlılık özelliğini doğru bir şekilde adlandırın. ‘DependencyProperty’ alanı ‘public static salt okunur’ olmalıdır. CLR özellik adına karşılık gelen ve “Özellik” ile biten bir ada sahip olmalıdır, ör. “Metin” ve “Metin Özelliği”.

  3. CLR özelliğinin ayarlayıcısına ek mantık eklemeyin. Bağımlılık özelliği sistemi (ve özellikle XAML), CLR özelliğini kullanmaz. Özelliğin değeri değiştiğinde bir işlem yapmak istiyorsanız, PropertyMetadata üzerinden bir geri arama sağlamalısınız:

     public static readonly DependencyProperty MyPropertyProperty =
     DependencyProperty.Register("MyProperty", typeof(int), typeof(MyControl),
         new PropertyMetadata(0, MyPropertyChangedHandler));

 private static void MyPropertyChangedHandler(DependencyObject sender, DependencyPropertyChangedEventArgs args)
 {
     // Use args.OldValue and args.NewValue here as needed.
     // sender is the object whose property changed.
     // Some unboxing required.
 }

Bağlama modu #

Bağlamalarda “Mode=TwoWay” belirtme ihtiyacını ortadan kaldırmak için (“TextBox.Text” davranışına benzer şekilde) kodu “PropertyMetadata” yerine “FrameworkPropertyMetadata” kullanacak şekilde güncelleyin ve uygun bayrağı belirtin:

public static readonly DependencyProperty MyPropertyProperty =
    DependencyProperty.Register("MyProperty", typeof(int), typeof(MyControl), 
        new FrameworkPropertyMetadata(0, FrameworkPropertyMetadataOptions.BindsTwoWayByDefault));

Ekli bağımlılık özellikleri #

Ne zaman kullanılır #

Ekli bir özellik, özelliğin varlığından haberdar olan çeşitli denetimlerin veya hizmetlerin davranışını geliştirmek için herhangi bir “DependencyObject” öğesine uygulanabilen bir bağımlılık özelliğidir.

Ekli mülkler için bazı kullanım örnekleri şunları içerir:

  1. Bir ebeveyn öğeye sahip olmak, çocukları arasında yinelenir ve çocuklar üzerinde belirli bir şekilde hareket eder. Örneğin, “Grid” denetimi, öğeleri satırlar ve sütunlar halinde düzenlemek için “Grid.Row”, “Grid.Column”, “Grid.RowSpan” ve “Grid.ColumnSpan” ekli özellikleri kullanır.
  2. Özel şablonlarla mevcut kontrollere görseller ekleme, ör. “TextBox” alt sınıfına gerek kalmadan uygulama genelindeki boş metin kutularına filigran ekleme.
  3. Mevcut kontrollerin bazılarına veya tümüne genel bir hizmet veya özellik sağlanması, örn. “ToolTipService” veya “FocusManager”. Bunlara genellikle bağlı davranışlar denir.
  4. Görsel ağacın aşağısında kalıtım gerektiğinde, örn. “DataContext” davranışına benzer.

Bu ayrıca “Grid” kullanım durumunda neler olduğunu gösterir:

<Grid>
    <Grid.ColumnDefinitions>
        <ColumnDefinition />
        <ColumnDefinition />
    </Grid.ColumnDefinitions>

    <Label Grid.Column="0" Content="Your Name:" />
    <TextBox Grid.Column="1" Text="{Binding FirstName}" />
</Grid>

“Grid.Column”, “Label” veya “TextBox” üzerinde bulunan bir özellik değildir. Bunun yerine, “Grid” denetimi alt öğelerine bakar ve bunları ekli özelliklerin değerlerine göre düzenler.

Nasıl tanımlanır #

Bu örnek için Grid kullanmaya devam edeceğiz. ‘Grid.Column’ tanımı aşağıda gösterilmiştir, ancak ‘DependencyPropertyChangedEventHandler’ kısa olması için hariç tutulmuştur.

public static readonly DependencyProperty RowProperty =
    DependencyProperty.RegisterAttached("Row", typeof(int), typeof(Grid),
        new FrameworkPropertyMetadata(0, ...));

public static void SetRow(UIElement element, int value)
{
    if (element == null)
        throw new ArgumentNullException("element");

    element.SetValue(RowProperty, value);
}

public static int GetRow(UIElement element)
{
    if (element == null)
        throw new ArgumentNullException("element");

    return ((int)element.GetValue(RowProperty));
}

Ekli özellikler çok çeşitli öğelere eklenebildiğinden, CLR özellikleri olarak uygulanamazlar. Bunun yerine bir çift statik yöntem tanıtılır.

Bu nedenle, standart bağımlılık özelliklerinin aksine, ekli özellikler de ‘DependencyObject’den türetilmemiş sınıflarda tanımlanabilir.

Normal bağımlılık özelliklerine uygulanan aynı adlandırma kuralları burada da geçerlidir: “RowProperty” bağımlılık özelliği, karşılık gelen “GetRow” ve “SetRow” yöntemlerine sahiptir.

Uyarılar #

MSDN’de belgelendiği gibi:

Özellik değeri devralma, eklenmemiş bağımlılık özellikleri için çalışıyor gibi görünse de, çalışma zamanı ağacındaki belirli öğe sınırları yoluyla eklenmemiş bir özelliğin devralma davranışı tanımsızdır. Meta verilerde Devralanlar belirttiğiniz özellikleri kaydetmek için her zaman RegisterAttached’ı kullanın.

Salt okunur bağımlılık özellikleri #

Ne zaman kullanılır #

Salt okunur bir bağımlılık özelliği, normal bir bağımlılık özelliğine benzer, ancak değerinin denetim dışından ayarlanmasına izin vermeyecek şekilde yapılandırılmıştır. Bu, yalnızca tüketiciler için bilgi veren bir mülkünüz varsa işe yarar, ör. “IsMouseOver” veya “IsKeyboardFocusWithin”.

Nasıl tanımlanır #

Tıpkı standart bağımlılık özellikleri gibi, ‘DependencyObject’den türetilen bir sınıf üzerinde salt okunur bir bağımlılık özelliği tanımlanmalıdır.

public class MyControl : Control
{
    private static readonly DependencyPropertyKey MyPropertyPropertyKey = 
        DependencyProperty.RegisterReadOnly("MyProperty", typeof(int), typeof(MyControl),
            new FrameworkPropertyMetadata(0));

    public static readonly DependencyProperty MyPropertyProperty = MyPropertyPropertyKey.DependencyProperty;

    public int MyProperty
    {
        get { return (int)GetValue(MyPropertyProperty); }
        private set { SetValue(MyPropertyPropertyKey, value); }
    }
}

Normal bağımlılık özellikleri için geçerli olan aynı kurallar burada da geçerlidir, ancak iki önemli fark vardır:

  1. “DependencyProperty”, bir “private” “DependencyPropertyKey"den kaynaklanır.
  2. CLR özellik ayarlayıcı, “genel” yerine “korumalı” veya “özel"dir.

Ayarlayıcının, “SetValue” yöntemine “MyPropertyProperty"yi değil “MyPropertyPropertyKey"i ilettiğini unutmayın. Özellik salt okunur olarak tanımlandığından, özellikte “SetValue” kullanma girişimi, “DependencyPropertyKey” alan aşırı yüklemeyle kullanılmalıdır; aksi takdirde, bir ‘InvalidOperationException’ atılır.
