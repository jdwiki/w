Modeller ve Görünüm-Modeller

Bir modelin tanımı genellikle hararetle tartışılır ve bir model ile bir görünüm modeli arasındaki çizgi bulanık olabilir. Bazıları, modellerini “INotifyPropertyChanged” arabirimiyle “kirletmeyi” ve bunun yerine, bu arabirimi uygulayan* görünüm modelindeki model özelliklerini çoğaltmayı tercih eder. Yazılım geliştirmedeki birçok şey gibi, doğru ya da yanlış cevap yoktur. Pragmatik olun ve size doğru gelen her şeyi yapın.

Ayrılığı Görüntüle

MVVM’nin amacı, bu üç farklı alanı ayırmaktır - Model, görünüm-model ve Görünüm. Görünümün görünüm modeline (VM) ve (dolaylı olarak) modele erişmesi kabul edilebilir olsa da, MVVM ile ilgili en önemli kural, VM’nin görünüme veya kontrollerine erişimi olmaması gerektiğidir. VM, görünümün ihtiyaç duyduğu her şeyi genel mülkler aracılığıyla ortaya çıkarmalıdır. VM, ‘TextBox’, ‘Button’ vb. gibi UI kontrollerini doğrudan göstermemeli veya değiştirmemelidir.

Bazı durumlarda, özellikle bazı karmaşık UI işlevlerini çalıştırmanız gerekiyorsa, bu katı ayrımla çalışmak zor olabilir. Burada, görünümün “arka plan kodu” dosyasında olayları ve olay işleyicilerini kullanmaya başvurmak tamamen kabul edilebilir. Tamamen UI işlevselliği ise, elbette görünümdeki olayları kullanın. Bu olay işleyicilerin sanal makine örneğinde genel yöntemleri çağırması da kabul edilebilir - yalnızca kullanıcı arabirimi denetimlerine veya bunun gibi bir şeye referanslar iletmeyin.

Röle Komutu

Ne yazık ki bu örnekte kullanılan ‘RelayCommand’ sınıfı WPF çerçevesinin bir parçası değil (olması gerekirdi!), ama onu hemen hemen her WPF geliştiricisinin araç kutusunda bulacaksınız. Çevrimiçi hızlı bir arama, kendinizinkini oluşturmak için kaldırabileceğiniz birçok kod parçacığını ortaya çıkaracaktır.

“RelayCommand"a yararlı bir alternatif, karşılaştırılabilir işlevsellik sağlayan “Microsoft.Expression.Interactivity.Core"un bir parçası olarak sağlanan “ActionCommand"dır.

Model #

Model, MVVM’deki ilk “M"dir. Model genellikle bir tür kullanıcı arabirimi aracılığıyla ortaya çıkarmak istediğiniz verileri içeren bir sınıftır.

İşte birkaç özelliği ortaya çıkaran çok basit bir model sınıfı: -

public class Customer : INotifyPropertyChanged
{
    private string _forename;
    private string _surname;
    private bool _isValid;

    public event PropertyChangedEventHandler PropertyChanged;

    /// <summary>
    /// Customer forename.
    /// </summary>
    public string Forename
    {
        get
        {
            return _forename;
        }
        set
        {
            if (_forename != value)
            {
                _forename = value;
                OnPropertyChanged();
                SetIsValid();
            }
        }
    }

    /// <summary>
    /// Customer surname.
    /// </summary>
    public string Surname
    {
        get
        {
            return _surname;
        }
        set
        {
            if (_surname != value)
            {
                _surname = value;
                OnPropertyChanged();
                SetIsValid();
            }
        }
    }

    /// <summary>
    /// Indicates whether the model is in a valid state or not.
    /// </summary>
    public bool IsValid
    {
        get
        {
            return _isValid;
        }
        set
        {
            if (_isValid != value)
            {
                _isValid = value;
                OnPropertyChanged();
            }
        }
    }

    /// <summary>
    /// Sets the value of the IsValid property.
    /// </summary>
    private void SetIsValid()
    {
        IsValid = !string.IsNullOrEmpty(Forename) && !string.IsNullOrEmpty(Surname);
    }

    /// <summary>
    /// Raises the PropertyChanged event.
    /// </summary>
    /// <param name="propertyName">Name of the property.</param>
    private void OnPropertyChanged([CallerMemberName] string propertyName = "")
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}

Bu sınıf, bir “PropertyChanged” olayını ortaya çıkaran “INotifyPropertyChanged” arabirimini uygular. Bu olay, özellik değerlerinden biri değiştiğinde ortaya çıkarılmalıdır - bunu yukarıdaki kodda çalışırken görebilirsiniz. “PropertyChanged” olayı, WPF veri bağlama mekanizmalarında önemli bir parçadır, çünkü o olmadan kullanıcı arayüzü bir mülkün değerinde yapılan değişiklikleri yansıtamaz.

Model ayrıca, özellik ayarlayıcılardan çağrılan çok basit bir doğrulama rutini içerir. Modelin geçerli bir durumda olup olmadığını gösteren bir kamu mülkü belirler. Bu işlevi, birazdan göreceğiniz WPF komutlarının “özel” bir özelliğini göstermek için ekledim. WPF çerçevesi, doğrulama için bir dizi daha karmaşık yaklaşım sağlar, ancak bunlar bu makalenin kapsamı dışındadır.

Görünüm #

Görünüm, MVVM’deki “V” dir. Bu sizin kullanıcı arayüzünüzdür. Visual Studio sürükle ve bırak tasarımcısını kullanabilirsiniz, ancak çoğu geliştirici sonunda ham XAML’yi kodlar - HTML yazmaya benzer bir deneyim.

İşte bir “Müşteri” modelinin düzenlenmesine izin veren basit bir görünümün XAML’si. Yeni bir görünüm oluşturmak yerine, bu sadece bir WPF projesinin “MainWindow.xaml” dosyasına “<Window …>” ve “” etiketleri arasına yapıştırılabilir:-

<StackPanel Orientation="Vertical"
            VerticalAlignment="Top"
            Margin="20">
    <Label Content="Forename"/>
    <TextBox Text="{Binding CustomerToEdit.Forename}"/>

    <Label Content="Surname"/>
    <TextBox Text="{Binding CustomerToEdit.Surname}"/>

    <Button Content="Apply Changes"
            Command="{Binding ApplyChangesCommand}" />
</StackPanel>

Bu kod, biri müşterinin adı, diğeri soyadı için olmak üzere iki TextBoxtan oluşan basit bir veri giriş formu oluşturur. Her bir “Metin Kutusu"nun üzerinde bir “Etiket” ve formun altında “Uygula” “Düğmesi” vardır.

İlk “TextBox"ı bulun ve “Text” özelliğine bakın:

Text="{Binding CustomerToEdit.Forename}"

Bu özel küme ayracı sözdizimi, “Metin Kutusu"nun metnini sabit bir değere ayarlamak yerine, metni “yolun” “CustomerToEdit.Forename"ye bağlar. Bu yol neye göre? Bu, görünümün “veri bağlamı” - bu durumda, görünüm modelimiz. Bağlama yolu, anlayabileceğiniz gibi, görünüm modelinin “CustomerToEdit” özelliğidir; bu özellik, “Customer” türündedir ve bu, “Forename” adlı bir özelliği ortaya çıkarır - dolayısıyla “noktalı” yol gösterimi.

Benzer şekilde, “Button"un XAML’sine bakarsanız, görünüm modelinin “ApplyChangesCommand” özelliğine bağlı bir “Command"a sahiptir. VM’nin komutuna bir düğme bağlamak için gereken tek şey bu.

Veri Bağlamı

Peki, görünüm modelini görünümün veri bağlamı olacak şekilde nasıl ayarlarsınız? Bir yol, onu görünümün “arka plan koduna” ayarlamaktır. Bu kod dosyasını görmek için F7’ye basın ve görünüm modelinin bir örneğini oluşturmak ve onu pencerenin ‘DataContext’ özelliğine atamak için mevcut kurucuya bir satır ekleyin. Sonu şöyle görünmelidir:

    public MainWindow()
    {
        InitializeComponent();

        // Our new line:-
        DataContext = new CustomerEditViewModel();
    }

Gerçek dünya sistemlerinde, görünüm modelini oluşturmak için genellikle bağımlılık ekleme veya MVVM çerçeveleri gibi başka yaklaşımlar kullanılır.

Görünüm Modeli #

Görünüm modeli, MVVM içindeki “VM” dir. Bu, arabulucu görevi gören, model(ler)i kullanıcı arabirimine (görünüm) sunan ve düğme tıklamalarıyla oluşturulan komutlar gibi görünümden gelen istekleri işleyen bir sınıftır. İşte temel bir görünüm modeli:

public class CustomerEditViewModel
{
    /// <summary>
    /// The customer to edit.
    /// </summary>
    public Customer CustomerToEdit { get; set; }

    /// <summary>
    /// The "apply changes" command
    /// </summary>
    public ICommand ApplyChangesCommand { get; private set; }

    /// <summary>
    /// Constructor
    /// </summary>
    public CustomerEditViewModel()
    {
        CustomerToEdit = new Customer
                         {
                             Forename = "John",
                             Surname = "Smith"
                         };

        ApplyChangesCommand = new RelayCommand(
            o => ExecuteApplyChangesCommand(), 
            o => CustomerToEdit.IsValid);
    }

    /// <summary>
    /// Executes the "apply changes" command.
    /// </summary>
    private void ExecuteApplyChangesCommand()
    {
        // E.g. save your customer to database
    }
}

Yapıcı bir “Customer” model nesnesi oluşturur ve onu “CustomerToEdit” özelliğine atar, böylece görünüm tarafından görülebilir.

Yapıcı ayrıca bir ‘RelayCommand’ nesnesi yaratır ve onu ‘ApplyChangesCommand’ özelliğine atar ve tekrar görünümde görünür hale getirir. WPF komutları, düğme veya menü öğesi tıklamaları gibi görünümden gelen istekleri işlemek için kullanılır.

‘RelayCommand’ iki parametre alır - ilki, komut yürütüldüğünde çağrılan temsilcidir (örneğin, bir düğme tıklamasına yanıt olarak). İkinci parametre, komutun yürütülüp yürütülemeyeceğini gösteren bir boole değeri döndüren bir temsilcidir; bu örnekte müşteri nesnesinin “IsValid” özelliğine bağlıdır. Bu false değerini döndürdüğünde, bu komuta bağlı olan düğmeyi veya menü öğesini devre dışı bırakır (diğer kontroller farklı davranabilir). Bu, farklı koşullara göre kontrolleri etkinleştirmek veya devre dışı bırakmak için kod yazma ihtiyacını ortadan kaldıran basit ama etkili bir özelliktir.

Bu örneği çalışır duruma getirirseniz, “Metin Kutuları"ndan birini boşaltmayı deneyin (“Müşteri” modelini geçersiz bir duruma getirmek için). TextBoxtan uzaklaştığınızda, “Uygula” düğmesinin devre dışı kaldığını görmelisiniz.

Müşteri Oluşturma Üzerine Açıklama

Görünüm modeli “INotifyPropertyChanged” (INPC) uygulamıyor. Bu, “CustomerToEdit” özelliğine farklı bir “Customer” nesnesi atanacaksa, görünümün kontrollerinin yeni nesneyi yansıtacak şekilde değişmeyeceği anlamına gelir - “TextBox"lar hala önceki müşterinin adını ve soyadını içerecektir. .

Örnek kod, görünümün “Veri Bağlamı"na (bağlamaların bağlandığı noktada) atanmadan önce görünüm modelinin yapıcısında “Müşteri” oluşturulduğu için çalışır. Gerçek dünyadaki bir uygulamada, kurucu dışındaki yöntemlerle bir veritabanından müşterileri alıyor olabilirsiniz. Bunu desteklemek için VM, INPC’yi uygulamalı ve “CustomerToEdit” özelliği, örnek Model kodunda gördüğünüz “genişletilmiş” alıcı ve ayarlayıcı modelini kullanacak şekilde değiştirilmelidir, ayarlayıcıda “PropertyChanged” olayını yükseltir.

Görünüm modelinin ‘ApplyChangesCommand’ komutu, komutun değişmesi pek olası olmadığından INPC’yi uygulamaya ihtiyaç duymaz. Komutu yapıcıdan başka bir yerde, örneğin bir tür Initialize() yöntemi oluşturuyorsanız, bu kalıbı uygulamanız gerekir.

Genel kural şudur: Özellik herhangi bir görünüm kontrolüne bağlıysa INPC’yi uygulayın ve özelliğin değeri yapıcı dışında herhangi bir yerde değişebilir. Özellik değeri yalnızca kurucuda atanmışsa INPC’yi uygulamanız gerekmez (ve bu süreçte kendinize biraz yazmaktan tasarruf edersiniz).

WPF ve C# kullanan temel MVVM örneği #

Bu, WPF ve C# kullanarak bir Windows masaüstü uygulamasında MVVM modelini kullanmak için temel bir örnektir. Örnek kod, basit bir “kullanıcı bilgisi” iletişim kutusu uygular.

Görünüm

XAML

<Grid>
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="Auto"/>
        <ColumnDefinition Width="*"/>
    </Grid.ColumnDefinitions>
    <Grid.RowDefinitions>
        <RowDefinition Height="Auto"/>
        <RowDefinition Height="Auto"/>
         <RowDefinition Height="Auto"/>
         <RowDefinition Height="Auto"/>
     </Grid.RowDefinitions>

    <Label Grid.Column="0" Grid.Row="1" Margin="4" Content="First Name:" HorizontalAlignment="Right"/>
    <!-- UpdateSourceTrigger=PropertyChanged makes sure that changes in the TextBoxes are immediately applied to the model. -->
    <TextBox Grid.Column="1" Grid.Row="1" Margin="4" Text="{Binding FirstName, UpdateSourceTrigger=PropertyChanged}" HorizontalAlignment="Left" Width="200"/>

    <Label Grid.Column="0" Grid.Row="2" Margin="4" Content="Last Name:" HorizontalAlignment="Right"/>
    <TextBox Grid.Column="1" Grid.Row="2" Margin="4" Text="{Binding LastName, UpdateSourceTrigger=PropertyChanged}" HorizontalAlignment="Left"  Width="200"/>

    <Label Grid.Column="0" Grid.Row="3" Margin="4" Content="Age:" HorizontalAlignment="Right"/>
    <TextBlock Grid.Column="1" Grid.Row="3" Margin="4" Text="{Binding Age}" HorizontalAlignment="Left"/>

</Grid>

public partial class MainWindow : Window
{
    private readonly MyViewModel _viewModel;

    public MainWindow() {
        InitializeComponent();
        _viewModel = new MyViewModel();
        // The DataContext serves as the starting point of Binding Paths
        DataContext = _viewModel;
    }
}

Görünüm Modeli

// INotifyPropertyChanged notifies the View of property changes, so that Bindings are updated.
sealed class MyViewModel : INotifyPropertyChanged
{
    private User user;
    
    public string FirstName { 
        get {return user.FirstName;} 
        set {
            if(user.FirstName != value) {
                user.FirstName = value;
                OnPropertyChange("FirstName");
                // If the first name has changed, the FullName property needs to be udpated as well.
                OnPropertyChange("FullName");
            }
        }
    }

    public string LastName {
        get { return user.LastName; }
        set {
            if (user.LastName != value) {
                user.LastName = value;
                OnPropertyChange("LastName");
                // If the first name has changed, the FullName property needs to be udpated as well.
                OnPropertyChange("FullName");
            }
        }
    }
    
    // This property is an example of how model properties can be presented differently to the View.
    // In this case, we transform the birth date to the user's age, which is read only.
    public int Age { 
        get {
            DateTime today = DateTime.Today;
            int age = today.Year - user.BirthDate.Year;
            if (user.BirthDate > today.AddYears(-age)) age--;
            return age;
        }
    }

    // This property is just for display purposes and is a composition of existing data.
    public string FullName {
        get { return FirstName + " " + LastName; }
    }

    public MyViewModel() {
        user = new User {
            FirstName = "John",
            LastName = "Doe",
            BirthDate = DateTime.Now.AddYears(-30)
        };
    }

    public event PropertyChangedEventHandler PropertyChanged;

    protected void OnPropertyChange(string propertyName) {
        if(PropertyChanged != null) {
            PropertyChanged(this, new PropertyChangedEventArgs(propertyName));
        }
    }
}

Model

sealed class User
{
    public string FirstName { get; set; }

    public string LastName { get; set; }

    public DateTime BirthDate { get; set; }
}

MVVM’de Komuta Etme #

Komutlar, MVVM-Pattern’e saygı gösterirken WPF’de ‘Olaylar’ı işlemek için kullanılır.

Normal bir ‘EventHandler’ şöyle görünür (‘Arkasındaki Kod’da bulunur):

public MainWindow()
{
    _dataGrid.CollectionChanged += DataGrid_CollectionChanged;
}

private void DataGrid_CollectionChanged(object sender, System.Collections.Specialized.NotifyCollectionChangedEventArgs e)
{
        //Do what ever
}

MVVM’de aynı şeyi yapmak için Hayır, “Komutlar” kullanıyoruz:

 <Button Command="{Binding Path=CmdStartExecution}" Content="Start" />

Komut özellikleriniz için bir tür önek (Cmd) kullanmanızı tavsiye ederim, çünkü bunlara esas olarak xaml’de ihtiyacınız olacak - bu şekilde tanınmaları daha kolay olacaktır.

MVVM olduğundan, ViewModel’inizde bu Komutu (Button “eq” Button_Click için) işlemek istiyorsunuz.

Bunun için temelde iki şeye ihtiyacımız var:

  1. System.Windows.Input.ICommand
  2. RelayCommand (örneğin buradan alınmıştır.

Basit bir örnek şöyle görünebilir:

private RelayCommand _commandStart;
public ICommand CmdStartExecution
{
   get
   {
      if(_commandStart == null)
      {
          _commandStart = new RelayCommand(param => Start(), param => CanStart());
      }
      return _commandStart;
   }
}

public void Start()
{
   //Do what ever
}

public bool CanStart()
{
    return (DateTime.Now.DayOfWeek == DayOfWeek.Monday); //Can only click that button on mondays.
}

Peki bu ayrıntılı olarak ne yapıyor:

“ICommand”, xaml’deki “Kontrol"ün bağlandığı şeydir. ‘RelayCommand’, komutunuzu bir ‘Eylem’e yönlendirecektir (yani, bir ‘Yöntem’ çağrısı). Null-Check, her “Komut” un yalnızca bir kez başlatılmasını sağlar (performans sorunları nedeniyle). Yukarıdaki ‘RelayCommand’ bağlantısını okuduysanız, ‘RelayCommand’ın yapıcısı için iki aşırı yüklemeye sahip olduğunu fark etmiş olabilirsiniz. (Eylem<nesne> yürütür) ve (Eylem<nesne> yürütür, Yüklem<nesne> yürütebilir).

Bu, (ilave olarak) “Kontrol"ün “Etkinliğin” tetiklenip ateşlenmediğini söylemek için bir “bool” döndüren ikinci bir “Yöntem” ekleyebileceğiniz anlamına gelir.

Bunun için iyi bir şey, örneğin, “Yöntem” “yanlış” döndürürse, “Düğme"nin “Etkin = “yanlış” olacaktır.

Komut Parametreleri

<DataGrid x:Name="TicketsDataGrid">
    <DataGrid.InputBindings>
        <MouseBinding Gesture="LeftDoubleClick"
