WPF uygulaması başlatıldığında, geçerli bir dil çalışma zamanının (CLR) .NET Framework’ü başlatması biraz zaman alabilir. Sonuç olarak, uygulamanın karmaşıklığına bağlı olarak, uygulama başlatıldıktan bir süre sonra ilk uygulama penceresi görünebilir.

WPF’deki açılış ekranı, uygulamanın ilk pencere görünmeden önce başlatma sırasında statik görüntü veya özel dinamik içerik göstermesine izin verir.

Basit Açılış Ekranı Ekleme #

Visual Studio’da WPF uygulamasına açılış ekranı eklemek için şu adımları izleyin:

  1. Herhangi bir resim oluşturun veya alın ve projenize ekleyin (örneğin, Images klasörünün içinde):

  1. Bu görüntü için özellikler penceresini açın (Görünüm → Özellikler Penceresi) ve Yapı Eylemi ayarını SplashScreen değerine değiştirin:

  1. Uygulamayı çalıştırın. Uygulama penceresi görünmeden önce ekranın ortasında açılış ekranı resminizi göreceksiniz (pencere göründükten sonra, açılış ekranı görüntüsü yaklaşık 300 milisaniye içinde kaybolacaktır).

Açılış Ekranını Test Etme #

Uygulamanız hafif ve basitse, çok hızlı başlayacak ve benzer hızda açılış ekranı görünecek ve kaybolacaktır.

“Application.Startup” yöntemi tamamlandıktan sonra açılış ekranı kaybolur kaybolmaz, aşağıdaki adımları izleyerek uygulama başlatma gecikmesini simüle edebilirsiniz:

  1. App.xaml.cs dosyasını açın
  2. using ad alanını using System.Threading; ekleyin
  3. ‘OnStartup’ yöntemini geçersiz kılın ve içine ‘Thread.Sleep(3000);’ ekleyin:

Kod şöyle görünmelidir:

using System;
using System.Collections.Generic;
using System.Configuration;
using System.Data;
using System.Linq;
using System.Threading;
using System.Windows;

namespace WpfApplication1
{
    /// <summary>
    /// Interaction logic for App.xaml
    /// </summary>
    public partial class App : Application
    {
        protected override void OnStartup(StartupEventArgs e)
        {
            base.OnStartup(e);

            Thread.Sleep(3000);
        }
    }
}
  1. Uygulamayı çalıştırın. Şimdi yaklaşık 3 saniye daha uzun süre başlatılacak, böylece açılış ekranınızı test etmek için daha fazla zamanınız olacak.

İlerleme raporlaması içeren açılış ekranı penceresi oluşturma #

WPF, kullanıma hazır bir açılış ekranı olarak bir görüntüden başka bir şeyin görüntülenmesini desteklemez, bu nedenle açılış ekranı olarak hizmet edecek bir “Pencere” oluşturmamız gerekecek. Uygulama ana penceresi olacak olan MainWindow sınıfını içeren bir proje oluşturduğumuzu varsayıyoruz.

Öncelikle projemize bir SplashScreenWindow penceresi ekliyoruz:

<Window x:Class="SplashScreenExample.SplashScreenWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        WindowStartupLocation="CenterScreen"
        WindowStyle="None"
        AllowsTransparency="True"
        Height="30"
        Width="200">
    <Grid>
        <ProgressBar x:Name="progressBar" />
        <TextBlock HorizontalAlignment="Center"
                   VerticalAlignment="Center">Loading...</TextBlock>
    </Grid>
</Window>

Ardından, mevcut ilerleme değerini (SplashScreenWindow.xaml.cs) kolayca güncelleyebilmemiz için SplashScreenWindow sınıfında bir özellik gösteririz:

public partial class SplashScreenWindow : Window
{
    public SplashScreenWindow()
    {
        InitializeComponent();
    }

    public double Progress
    {
        get { return progressBar.Value; }
        set { progressBar.Value = value; }
    }
}

Ardından, açılış ekranını göstermek için Application.OnStartup yöntemini geçersiz kılıyoruz, biraz iş yapıyoruz ve son olarak ana pencereyi gösteriyoruz (App.xaml.cs):

public partial class App : Application
{
    protected override void OnStartup(StartupEventArgs e)
    {
        base.OnStartup(e);

        //initialize the splash screen and set it as the application main window
        var splashScreen = new SplashScreenWindow();
        this.MainWindow = splashScreen;
        splashScreen.Show();

        //in order to ensure the UI stays responsive, we need to
        //do the work on a different thread
        Task.Factory.StartNew(() =>
        {
            //we need to do the work in batches so that we can report progress
            for (int i = 1; i <= 100; i++)
            {
                //simulate a part of work being done
                System.Threading.Thread.Sleep(30);

                //because we're not on the UI thread, we need to use the Dispatcher
                //associated with the splash screen to update the progress bar
                splashScreen.Dispatcher.Invoke(() => splashScreen.Progress = i);
            }

            //once we're done we need to use the Dispatcher
            //to create and show the main window
            this.Dispatcher.Invoke(() =>
            {
                //initialize the main window, set it as the application main window
                //and close the splash screen
                var mainWindow = new MainWindow();
                this.MainWindow = mainWindow;
                mainWindow.Show();
                splashScreen.Close();
            });
        });
    }
}

Son olarak, uygulama başlangıcında ‘MainWindow’u gösteren varsayılan mekanizmaya dikkat etmemiz gerekiyor. Tek yapmamız gereken, App.xaml dosyasındaki kök ‘Application’ etiketinden StartupUri="MainWindow.xaml" niteliğini kaldırmak.

Özel açılış ekranı penceresi oluşturma #

WPF, kullanıma hazır bir açılış ekranı olarak bir görüntüden başka bir şeyin görüntülenmesini desteklemez, bu nedenle açılış ekranı olarak hizmet edecek bir “Pencere” oluşturmamız gerekecek. Uygulama ana penceresi olacak olan MainWindow sınıfını içeren bir proje oluşturduğumuzu varsayıyoruz.

Öncelikle projemize bir SplashScreenWindow penceresi ekliyoruz:

<Window x:Class="SplashScreenExample.SplashScreenWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        WindowStartupLocation="CenterScreen"
        WindowStyle="None"
        AllowsTransparency="True"
        Height="30"
        Width="200">
    <Grid>
        <ProgressBar IsIndeterminate="True" />
        <TextBlock HorizontalAlignment="Center"
                   VerticalAlignment="Center">Loading...</TextBlock>
    </Grid>
</Window>

Ardından, açılış ekranını göstermek, biraz çalışmak ve son olarak ana pencereyi (App.xaml.cs) göstermek için Application.OnStartup yöntemini geçersiz kılıyoruz:

public partial class App
{
    protected override void OnStartup(StartupEventArgs e)
    {
        base.OnStartup(e);

        //initialize the splash screen and set it as the application main window
        var splashScreen = new SplashScreenWindow();
        this.MainWindow = splashScreen;
        splashScreen.Show();

        //in order to ensure the UI stays responsive, we need to
        //do the work on a different thread
        Task.Factory.StartNew(() =>
        {
            //simulate some work being done
            System.Threading.Thread.Sleep(3000);

            //since we're not on the UI thread
            //once we're done we need to use the Dispatcher
            //to create and show the main window
            this.Dispatcher.Invoke(() =>
            {
                //initialize the main window, set it as the application main window
                //and close the splash screen
                var mainWindow = new MainWindow();
                this.MainWindow = mainWindow;
                mainWindow.Show();
                splashScreen.Close();
            });
        });
    }
}

Son olarak, uygulama başlangıcında ‘MainWindow’u gösteren varsayılan mekanizmaya dikkat etmemiz gerekiyor. Tek yapmamız gereken, App.xaml dosyasındaki kök ‘Application’ etiketinden StartupUri="MainWindow.xaml" niteliğini kaldırmak.
