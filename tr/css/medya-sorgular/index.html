Sözdizimi #

  • @media [değil|yalnızca] mediatype ve (medya özelliği) { /* Uygulanacak CSS kuralları */ }

Parametreler #

parametre Detaylar
“ortam türü” (İsteğe bağlı) Bu, ortam türüdür. “tümü” ile “ekran” aralığında herhangi bir şey olabilir.
“değil” (İsteğe bağlı) Bu belirli ortam türü için CSS’yi uygulamaz ve diğer her şey için geçerlidir.
medya özelliği CSS için kullanım durumunu belirleme mantığı. Aşağıda belirtilen seçenekler.
Medya Özelliği Detaylar
’en-boy oranı' Çıkış cihazının hedeflenen görüntüleme alanının en boy oranını tanımlar.
“renk” Çıkış aygıtının renk bileşeni başına bit sayısını gösterir. Cihaz renkli bir cihaz değilse bu değer sıfırdır.
‘renk indeksi’ Çıkış aygıtı için renk arama tablosundaki girişlerin sayısını gösterir.
“ızgara” Çıkış aygıtının bir ızgara aygıtı mı yoksa bir bit eşlem aygıtı mı olduğunu belirler.
‘yükseklik’ Yükseklik ortamı özelliği, çıktı aygıtının işleme yüzeyinin yüksekliğini tanımlar.
maks-genişlik CSS, belirtilenden daha geniş bir ekran genişliğine uygulanmaz.
min-genişlik CSS, belirtilenden daha dar bir ekran genişliğine uygulanmaz.
‘maks-yükseklik’ CSS, belirtilenden daha uzun bir ekran yüksekliğine uygulanmaz.
min-yükseklik CSS, belirtilenden daha kısa bir ekran yüksekliğine uygulanmaz.
“tek renkli” Tek renkli (gri tonlamalı) bir aygıtta piksel başına bit sayısını gösterir.
“yönlendirme” CSS, yalnızca cihaz belirtilen yönlendirmeyi kullanıyorsa görüntülenecektir. Daha fazla ayrıntı için açıklamalara bakın.
“çözünürlük” Çıkış cihazının çözünürlüğünü (piksel yoğunluğu) gösterir.
’tarama' Televizyon çıkış cihazlarının tarama işlemini açıklar.
“genişlik” Genişlik ortamı özelliği, çıktı aygıtının işleme yüzeyinin genişliğini (belge penceresinin genişliği veya bir yazıcıdaki sayfa kutusunun genişliği gibi) tanımlar.
Kullanımdan Kaldırılan Özellikler Detaylar
‘aygıt-en-boy oranı’ Kullanımdan kaldırıldı CSS, yalnızca yükseklik/genişlik oranı belirtilen oranla eşleşen cihazlarda görüntülenecektir. Bu, “kullanımdan kaldırılmış” bir özelliktir ve çalışması garanti edilmez.
‘maks-cihaz genişliği’ Kullanımdan kaldırıldı max-width ile aynıdır ancak tarayıcının görüntü genişliği yerine fiziksel ekran genişliğini ölçer.
min-cihaz genişliği Kullanımdan kaldırıldı min-width ile aynıdır ancak tarayıcının görüntü genişliği yerine fiziksel ekran genişliğini ölçer.
‘maks-cihaz yüksekliği’ Kullanımdan kaldırıldı max-height ile aynıdır ancak tarayıcının görüntü genişliği yerine fiziksel ekran genişliğini ölçer.
min-cihaz yüksekliği “Kullanımdan kaldırıldı” “min-height” ile aynıdır ancak tarayıcının görüntü genişliği yerine fiziksel ekran genişliğini ölçer.

Medya sorguları, Chrome, Firefox, Opera ve Internet Explorer 9 ve sonraki sürümleri dahil tüm modern tarayıcılarda desteklenir.

“Oryantasyon” medya özelliğinin mobil cihazlarla sınırlı olmadığını belirtmek önemlidir. Görünüm alanının genişliğine ve yüksekliğine bağlıdır (pencere veya cihazlar değil).

Manzara Modu, görüntü alanı genişliğinin görüntü alanı yüksekliğinden büyük olduğu zamandır.

Portre Modu, görüntü alanı yüksekliğinin görüntü alanı genişliğinden büyük olduğu zamandır.

Bu genellikle masaüstü monitörünün yatay modda olduğu anlamına gelir, ancak bazen portre olabilir.

Çoğu durumda mobil cihazlar, piksel yoğunluğu nedeniyle değişebilen gerçek piksel boyutlarını değil, çözünürlüklerini bildirir. Onları gerçek piksel boyutlarını bildirmeye zorlamak için “head” etiketinizin içine aşağıdakini ekleyin:

<meta name="viewport" content="width=cihaz genişliği, ilk ölçek=1">

Temel Örnek #

@media screen and (min-width: 720px) {
    body {
        background-color: skyblue;
    }
}

Yukarıdaki medya sorgusu iki koşulu belirtir:

  1. Sayfa normal bir ekranda görüntülenmelidir (basılı sayfa, projektör vb. değil).
  2. Kullanıcının görüntüleme portunun genişliği en az 720 piksel olmalıdır.

Bu koşullar sağlanırsa, medya sorgusu içindeki stiller aktif olacak ve sayfanın arka plan rengi gök mavisi olacaktır.

Medya sorguları dinamik olarak uygulanır. Sayfa yüklemesinde ortam sorgusunda belirtilen koşullar karşılanırsa, CSS uygulanır, ancak koşulların yerine getirilmesinin sona ermesi durumunda hemen devre dışı bırakılır. Tersine, koşullar başlangıçta karşılanmazsa, belirtilen koşullar karşılanana kadar CSS uygulanmayacaktır.

Örneğimizde, kullanıcının görüntüleme bağlantı noktası genişliği başlangıçta 720 pikselden büyükse, ancak kullanıcı tarayıcının genişliğini küçültüyorsa, kullanıcı görüntüleme bağlantı noktasını 720 pikselden daha az olacak şekilde yeniden boyutlandırdığı anda arka plan rengi gök mavisi olmaktan çıkacaktır. Genişlik.

ortam türü #

Medya sorgularının isteğe bağlı bir “mediatype” parametresi vardır. Bu parametre, @media bildiriminden (@media mediatype) hemen sonra yer alır, örneğin:

@media print {
    html {
        background-color: white;
    }
}

Yukarıdaki CSS kodu, yazdırılırken DOM HTML öğesine beyaz bir arka plan rengi verecektir.

“mediatype” parametresinin, stilleri belirtilen medya türü veya yalnızca belirtilen medya türü dışındaki her şeye uygulayacak isteğe bağlı bir “değil” veya “yalnızca” öneki vardır. Örneğin, aşağıdaki kod örneği, stili “yazdır” dışında her ortam türüne uygulayacaktır.

@media not print {
    html {
        background-color: green;
    }
}

Ve aynı şekilde, sadece ekranda göstermek için bu kullanılabilir:

@media only screen {
    .fadeInEffects {
        display: block;
    }
}

‘mediatype’ listesi aşağıdaki tablo ile daha iyi anlaşılabilir:

Medya Türü Açıklama
“hepsi” Tüm cihazlara uygula
ekran Varsayılan bilgisayarlar
‘baskı’ Genel olarak yazıcılar. Web sitelerinin baskı sürümlerini biçimlendirmek için kullanılır
’el tipi’ Küçük ekranlı PDA’lar, cep telefonları ve el cihazları
“projeksiyon” Projeksiyonlu sunum için, örneğin projektörler
‘işitsel’ Konuşma Sistemleri
‘braille’ Braille dokunsal cihazlar
“kabartmalı” Sayfalı braille yazıcılar
tv Televizyon tipi cihazlar
’ti' Sabit aralıklı karakter ızgarasına sahip cihazlar. Terminaller, taşınabilirler.

Terminoloji ve Yapı #

Medya sorguları, medya türü olarak adlandırılan cihaz / ortam türüne (ör. ekran, baskı veya el tipi) dayalı CSS kurallarının uygulanmasına izin verir, cihazın ek özellikleri medya özellikleri ile tanımlanır renk veya görüntü alanı boyutlarının kullanılabilirliği gibi.

Medya Sorgusunun Genel Yapısı #

@media [...] {
    /* One or more CSS rules to apply when the query is satisfied */
}

Medya Türü içeren Medya Sorgusu #

@media print {
    /* One or more CSS rules to apply when the query is satisfied */
}

Medya Türü ve Medya Özelliği içeren Medya Sorgusu #

@media screen and (max-width: 600px) {
    /* One or more CSS rules to apply when the query is satisfied */
}

Bir Medya Özelliği (ve “tümü” şeklinde örtük bir Medya Türü) içeren bir Medya Sorgusu #

@media (orientation: portrait) {        
    /* One or more CSS rules to apply when the query is satisfied */
}

Genişlik ve Görünüm Alanı #

Medya sorgularıyla “width” kullandığımızda meta etiketi doğru ayarlamak önemlidir. Temel meta etiketi şöyle görünür ve <head> etiketinin içine yerleştirilmesi gerekir.

<meta name="viewport" content="width=device-width,initial-scale=1">

Bu neden önemli?

MDN’nin “genişlik” tanımına göre

Genişlik medya özelliği, işleme yüzeyinin genişliğini tanımlar çıkış aygıtının (belge penceresinin genişliği veya bir yazıcıdaki sayfa kutusunun genişliği).

Bu ne anlama geliyor?

Görünüm portu, cihazın kendisinin genişliğidir. Ekran çözünürlüğünüz çözünürlüğün 1280 x 720 olduğunu söylüyorsa, görünüm bağlantı noktası genişliğiniz “1280 piksel"dir.

Daha sık olarak, birçok cihaz bir pikseli görüntülemek için farklı piksel miktarları tahsis eder. Örneğin iPhone 6 Plus 1242 x 2208 çözünürlüğe sahip. Ancak gerçek görüntü alanı genişliği ve görüntü alanı yüksekliği 414 x 736’dır. Bu, 1 piksel oluşturmak için 3 pikselin kullanıldığı anlamına gelir.

Ancak “meta” etiketini doğru ayarlamadıysanız, web sayfanızı yerel çözünürlüğüyle göstermeye çalışır ve bu da yakınlaştırılmış bir görünümle sonuçlanır (daha küçük metinler ve resimler).

Retina ve Retina Olmayan Ekranlar için Medya Sorguları #

Bu yalnızca WebKit tabanlı tarayıcılar için işe yarasa da, bu yararlıdır:

/* ----------- Non-Retina Screens ----------- */
@media screen 
  and (min-width: 1200px) 
  and (max-width: 1600px) 
  and (-webkit-min-device-pixel-ratio: 1) { 
}

/* ----------- Retina Screens ----------- */
@media screen 
  and (min-width: 1200px) 
  and (max-width: 1600px) 
  and (-webkit-min-device-pixel-ratio: 2)
  and (min-resolution: 192dpi) { 
}

Arkaplan bilgisi

Ekranda iki tür piksel vardır. Biri mantıksal pikseller, diğeri ise fiziksel piksellerdir. Çoğunlukla, fiziksel pikseller her zaman aynı kalır, çünkü tüm görüntüleme aygıtları için aynıdır. Mantıksal pikseller, daha yüksek kaliteli pikseller görüntülemek için cihazların çözünürlüğüne göre değişir. Cihaz piksel oranı, fiziksel pikseller ile mantıksal pikseller arasındaki orandır. Örneğin, MacBook Pro Retina, iPhone 4 ve üstü, fiziksel doğrusal çözünürlük mantıksal çözünürlüğün iki katı olduğundan, aygıt piksel oranını 2 bildirir.

Bunun yalnızca WebKit tabanlı tarayıcılarda çalışmasının nedeni şudur:

  • Kuraldan önceki satıcı öneki -webkit-.
  • Bu, WebKit ve Blink dışındaki motorlarda uygulanmadı.

Farklı Ekran Boyutlarını Hedeflemek için Medya Sorgularını Kullanma #

Çoğu zaman, duyarlı web tasarımı, yalnızca bir koşul yerine getirildiğinde yürütülen CSS blokları olan medya sorgularını içerir. Bu, duyarlı web tasarımı için kullanışlıdır, çünkü web sitenizin mobil sürümü için masaüstü sürümüne karşı farklı CSS stilleri belirlemek üzere medya sorgularını kullanabilirsiniz.

@media only screen and (min-width: 300px) and (max-width: 767px) {
    .site-title {
        font-size: 80%;
    }

    /* Styles in this block are only applied if the screen size is atleast 300px wide, but no more than 767px */
}

@media only screen and (min-width: 768px) and (max-width: 1023px) {
    .site-title {
        font-size: 90%;
    }

    /* Styles in this block are only applied if the screen size is atleast 768px wide, but no more than 1023px */
}

@media only screen and (min-width: 1024px) {
    .site-title {
        font-size: 120%;
    }

    /* Styles in this block are only applied if the screen size is over 1024px wide. */
}

Bağlantı etiketinde kullanın #

<link rel="stylesheet" media="min-width: 600px" href="example.css" />

Bu stil sayfası hala indirilmektedir ancak yalnızca ekran genişliği 600 pikselden büyük olan cihazlarda uygulanmaktadır.

Medya sorguları ve IE8 #

Medya sorguları, IE8 ve daha düşük sürümlerde hiç desteklenmez.

Javascript tabanlı bir geçici çözüm #

IE8 desteği eklemek için çeşitli JS çözümlerinden birini kullanabilirsiniz. Örneğin, IE8 için medya sorgusu desteği eklemek için Respond yalnızca aşağıdaki kodla eklenebilir:

<!--[if lt IE 9]>
<script 
   src="respond.min.js">
</script>
<![endif]-->

CSS Mediaqueries aynı şeyi yapan başka bir kitaplıktır. Bu kitaplığı HTML’nize ekleme kodu aynı olacaktır:

<!--[if lt IE 9]>
<script 
   src="css3-mediaqueries.js">
</script>
<![endif]-->

Alternatif #

JS tabanlı bir çözümden hoşlanmıyorsanız, stilinizi IE<9’a özel olarak ayarladığınız yalnızca IE<9 stil sayfası eklemeyi de düşünmelisiniz. Bunun için kodunuza aşağıdaki HTML’yi eklemelisiniz:

<!--[if lt IE 9]>
<link rel="stylesheet" type="text/css" media="all" href="style-ielt9.css"/>
<![endif]-->

Not :

Teknik olarak bir alternatif daha: IE<9’u hedeflemek için CSS hacks kullanmak. Yalnızca IE<9 stil sayfasıyla aynı etkiye sahiptir, ancak bunun için ayrı bir stil sayfasına ihtiyacınız yoktur. Yine de, geçersiz CSS kodu ürettikleri için bu seçeneği önermiyorum (bu, günümüzde CSS korsanlığının kullanılmasının genellikle hoş karşılanmamasının birkaç nedeninden biridir).
