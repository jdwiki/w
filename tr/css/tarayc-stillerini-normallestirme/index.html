Tarayıcı Stillerini Normalleştirme

  • normalize.css
  • Bu ne işe yarıyor
  • reset.css farkı
  • Yaklaşımlar ve Örnekler

Bu sayfada

  • normalize.css
  • Bu ne işe yarıyor
  • reset.css farkı
  • Yaklaşımlar ve Örnekler

Her tarayıcının, öğeleri oluşturmak için kullandığı varsayılan bir CSS stili kümesi vardır. Bu varsayılan stiller tarayıcılar arasında tutarlı olmayabilir çünkü: dil özellikleri belirsizdir, bu nedenle temel stiller yorumlanmaya hazırdır, tarayıcılar verilen özelliklere uymayabilir veya tarayıcılar daha yeni HTML öğeleri için varsayılan stillere sahip olmayabilir. Sonuç olarak, insanlar mümkün olduğu kadar çok tarayıcıda varsayılan stilleri normalleştirmek isteyebilir.

Meyer’s Reset, etkili olmakla birlikte, yaygın olarak kullanılan hemen hemen her öğede aynı değişiklikleri yapar. Bu, web tarayıcısı denetçi pencerelerinin aynı uygulanmış stiller ile tekrar tekrar kirlenmesine neden olur ve tarayıcı için daha fazla iş yaratır (daha fazla öğeye uygulanacak daha fazla kural). Normalize tekniği ise çok daha odaklı ve daha az geniş fırça tekniğidir. Bu, tarayıcının işini basitleştirir ve tarayıcı inceleme araçlarında daha az karışıklığa neden olur.

normalize.css #

Tarayıcıların, öğeleri oluşturmak için kullandıkları varsayılan bir CSS stilleri kümesi vardır. Bu stillerden bazıları, örneğin varsayılan yazı tipi yüzü ve boyutu tanımlarını değiştirmek için tarayıcının ayarları kullanılarak özelleştirilebilir. Stiller, diğer şeylerin yanı sıra, hangi öğelerin blok düzeyinde veya satır içi olması gerektiğinin tanımını içerir.

Bu varsayılan stiller, dil özellikleri tarafından bir miktar boşluk bırakıldığından ve tarayıcılar, özellikleri doğru şekilde takip etmeyebileceğinden, tarayıcıdan tarayıcıya farklılık gösterebilir.

İşte burada normalize.css devreye giriyor. En yaygın tutarsızlıkları geçersiz kılar ve bilinen hataları düzeltir.

Bu ne işe yarıyor #

  • Birçok CSS sıfırlamasının aksine yararlı varsayılanları korur.
  • Çok çeşitli öğeler için stilleri normalleştirir.
  • Hataları ve yaygın tarayıcı tutarsızlıklarını düzeltir.
  • Küçük değişikliklerle kullanılabilirliği artırır.
  • Ayrıntılı yorumları kullanarak kodun ne yaptığını açıklar.

Böylece, projenize normalize.css’yi dahil ederek tasarımınız farklı tarayıcılarda daha benzer ve tutarlı görünecektir.

reset.css farkı #

“reset.css"yi duymuş olabilirsiniz. İkisi arasındaki fark nedir?

Normalize.css, farklı özellikleri birleştirilmiş varsayılanlara ayarlayarak tutarlılık sağlarken, reset.css, bir tarayıcının uygulayabileceği tüm temel stilleri kaldırarak tutarlılık sağlar. Bu ilk başta iyi bir fikir gibi görünse de, aslında bu, katı bir standarda aykırı olan tüm kuralları kendiniz yazmanız gerektiği anlamına gelir.

Yaklaşımlar ve Örnekler #

CSS sıfırlamaları, tarayıcı varsayılanlarına ayrı yaklaşımlar alır. Eric Meyer’in Sıfırlama CSS’si bir süredir buralarda. Yaklaşımı, arkadan sorunlara neden olduğu bilinen tarayıcı öğelerinin çoğunu geçersiz kılıyor. Aşağıdakiler onun sürümünden (v2.0 | 20110126) CSS Sıfırlama.

html, body, div, span, applet, object, iframe,
h1, h2, h3, h4, h5, h6, p, blockquote, pre,
a, abbr, acronym, address, big, cite, code,
del, dfn, em, img, ins, kbd, q, s, samp,
small, strike, strong, sub, sup, tt, var,
b, u, i, center,
dl, dt, dd, ol, ul, li,
fieldset, form, label, legend,
table, caption, tbody, tfoot, thead, tr, th, td,
article, aside, canvas, details, embed, 
figure, figcaption, footer, header, hgroup, 
menu, nav, output, ruby, section, summary,
time, mark, audio, video {
    margin: 0;
    padding: 0;
    border: 0;
    font-size: 100%;
    font: inherit;
    vertical-align: baseline;
}

Eric Meyer’in Sıfırlama CSS’si

Öte yandan CSS’yi normalleştirin ve bunların çoğuyla ayrı ayrı ilgilenin. Aşağıdaki, kodun (v4.2.0) sürümünden bir örnektir.

/**
 * 1. Change the default font family in all browsers (opinionated).
 * 2. Correct the line height in all browsers.
 * 3. Prevent adjustments of font size after orientation changes in IE and iOS.
 */

/* Document
   ========================================================================== */

html {
  font-family: sans-serif; /* 1 */
  line-height: 1.15; /* 2 */
  -ms-text-size-adjust: 100%; /* 3 */
  -webkit-text-size-adjust: 100%; /* 3 */
}

/* Sections
   ========================================================================== */

/**
 * Remove the margin in all browsers (opinionated).
 */

body {
  margin: 0;
}

/**
 * Add the correct display in IE 9-.
 */

article,
aside,
footer,
header,
nav,
section {
  display: block;
}

/**
 * Correct the font size and margin on `h1` elements within `section` and
 * `article` contexts in Chrome, Firefox, and Safari.
 */

h1 {
  font-size: 2em;
  margin: 0.67em 0;
}
