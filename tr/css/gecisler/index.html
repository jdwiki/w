Sözdizimi #

  • geçiş: [geçiş-özelliği] [geçiş-süresi] [geçiş-zamanlama-fonksiyonu] [geçiş-gecikmesi];

Parametreler #

parametre Ayrıntılar
geçiş mülkiyeti Tüm geçiş yapılabilir özelliklerin olması gerekiyorsa, değer değişikliğinin geçişi (veya) “tümü” olması gereken belirli CSS özelliği geçiş yaptı.
geçiş süresi Geçişin gerçekleşmesi gereken saniye (s) veya milisaniye (ms) cinsinden süre (veya dönem).
geçiş-zamanlama-fonksiyonu Geçiş sırasında ara değerlerin nasıl hesaplandığını açıklayan bir fonksiyon. Yaygın olarak kullanılan değerler ’ease’, ’ease-in’, ’ease-out’, ’ease-in-out’, ’linear’, ‘cubic-bezier()’, ‘adımlar()‘dır. Çeşitli zamanlama işlevleri hakkında daha fazla bilgiyi W3C özelliklerinde bulabilirsiniz.
geçiş gecikmesi Geçişin başlayabilmesi için geçmesi gereken süre. Saniye (s) veya milisaniye (ms) olarak belirtilebilir

Bazı eski tarayıcılar yalnızca vendor-prefixed “geçiş” özelliklerini destekler:

  • -webkit: Chrome 25-, Safari 6-, iOS 6.1- için Safari ve Chrome, Android 4.3- Tarayıcı, Blackberry Tarayıcı 7-, Android için UC Tarayıcı 9.9-.
  • -moz: Firefox 15-.
  • -o: Opera 11.5-, Opera Mobile 12-.

Örnek:

-webkit-transition: all 1s;
-moz-transition: all 1s;
-o-transition: all 1s;
transition: all 1s;

Geçiş kısayolu #

CSS

div{
    width: 150px;
    height:150px;
    background-color: red;
    transition: background-color 1s;
}
div:hover{
    background-color: green;
}

HTML

<div></div>

Bu örnek, div üzerine gelindiğinde arka plan rengini değiştirecek, arka plan rengi değişikliği 1 saniye sürecektir.

kübik-bezier #

“kübik-bezier” işlevi, genellikle özel ve yumuşak geçişler için kullanılan bir geçiş zamanlama işlevidir.

geçiş-zamanlama-fonksiyonu: kübik-bezier(0.1, 0.7, 1.0, 0.1);

İşlev dört parametre alır:

kübik-bezier(P1_x, P1_y, P2_x, P2_y)

Bu parametreler, bir Bézier eğrisinin parçası olan noktalara eşlenecektir:

CSS Bézier Eğrileri için P0 ve P3 her zaman aynı noktadadır. P0 (0,0) konumunda ve P3 (1,1) konumunda, bu da kübik-bezier işlevine iletilen parametrelerin yalnızca 0 ile 1 arasında olabileceği anlamına geliyor.

Bu aralıkta olmayan parametreleri iletirseniz, işlev varsayılan olarak “doğrusal” bir geçişe sahip olacaktır.

Kübik-bezier, CSS’deki en esnek geçiş olduğundan, diğer tüm geçiş zamanlama işlevlerini kübik-bezier işlevlerine çevirebilirsiniz:

doğrusal: kübik-bezier(0,0,1,1)

ease-in: kübik-bezier(0.42, 0.0, 1.0, 1.0)

ease-out: kübik-bezier(0.0, 0.0, 0.58, 1.0)

ease-in-out: kübik-bezier(0.42, 0.0, 0.58, 1.0)

Geçiş (uzun el) #

CSS #

div {
  height: 100px;
  width: 100px;
  border: 1px solid;
  transition-property: height, width;
  transition-duration: 1s, 500ms;
  transition-timing-function: linear;
  transition-delay: 0s, 1s;
}
div:hover {
  height: 200px;
  width: 200px;
}

HTML #

<div></div>
  • geçiş-özelliği: Geçiş efektinin ait olduğu CSS özelliklerini belirtir. Bu durumda, div, üzerine gelindiğinde hem yatay hem de dikey olarak genişleyecektir.
  • geçiş süresi: Bir geçişin tamamlanması için gereken süreyi belirtir. Yukarıdaki örnekte yükseklik ve genişlik geçişleri sırasıyla 1 saniye ve 500 milisaniye sürecektir.
  • geçiş-zamanlama-fonksiyonu: Geçiş etkisinin hız eğrisini belirtir. Doğrusal bir değer, geçişin baştan sona aynı hızda olacağını gösterir.
  • geçiş gecikmesi: Geçiş efekti başlamadan önce beklemek için gereken süreyi belirtir. Bu durumda yükseklik hemen geçiş yapmaya başlayacak, genişlik ise 1 saniye bekleyecektir.
