Sözdizimi #

  • geçiş: <özellik> <süre> <zamanlama işlevi> <gecikme>;
  • @keyframes <tanımlayıcı>
  • [ [ dan | için | <yüzde> ] [, başlangıç ​​| için | <yüzde> ]* blok ]*

Parametreler #

Geçiş  
parametre Ayrıntılar
mülk Ya geçiş yapılacak CSS özelliği ya da tüm geçiş yapılabilen özellikleri belirten “tümü”.
süre Geçiş süresi, saniye veya milisaniye cinsinden.
zamanlama-fonksiyonu Özellikler için ara değerlerin nasıl hesaplanacağını tanımlayan bir işlev belirtir. Ortak değerler “kolaylık”, “doğrusal” ve “adım sonu"dur. Daha fazlası için yavaşlatma işlevi hile sayfasına göz atın.
gecikme Animasyonu oynatmadan önce beklenecek saniye veya milisaniye cinsinden süre.
   
@keyframes  
[ gelen | için | <yüzde> ] Anahtar karenin ayarlanan özniteliklerinin ayarlandığı bir süre için yüzde değeriyle ayarlı bir süre veya iki yüzde değeri, yani “%10, %20” belirtebilirsiniz.
“blok” Animasyon karesi için herhangi bir miktarda CSS özelliği.

Animasyon kareleri içeren animasyonlar #

Çok aşamalı CSS animasyonları için CSS @keyframes oluşturabilirsiniz. Ana kareler, daha karmaşık animasyonlar tanımlamak için ana kare adı verilen birden çok animasyon noktası tanımlamanıza olanak tanır.

Temel Örnek #

Bu örnekte, tüm renkler arasında geçiş yapan temel bir arka plan animasyonu yapacağız.

@keyframes rainbow-background {
         0%     { background-color: #ff0000; }
     8.333%     { background-color: #ff8000; }
    16.667%     { background-color: #ffff00; }
    25.000%     { background-color: #80ff00; }
    33.333%     { background-color: #00ff00; }
    41.667%     { background-color: #00ff80; }
    50.000%     { background-color: #00ffff; }
    58.333%     { background-color: #0080ff; }
    66.667%     { background-color: #0000ff; }
    75.000%     { background-color: #8000ff; }
    83.333%     { background-color: #ff00ff; }
    91.667%     { background-color: #ff0080; }
    100.00%     { background-color: #ff0000; }
}

.RainbowBackground {
    animation: rainbow-background 5s infinite;
}

Sonucu Görüntüle

Burada dikkat edilmesi gereken birkaç farklı şey var. İlk olarak, gerçek “@keyframes” sözdizimi.

@keyframes rainbow-background{

Bu, animasyonun adını ‘gökkuşağı arkaplanı’ olarak ayarlar.

0%     { background-color: #ff0000; }

Bu, animasyon içindeki bir ana karenin tanımıdır. İlk bölüm, durumda “%0”, animasyon sırasında ana karenin nerede olduğunu tanımlar. “0%”, baştan itibaren toplam animasyon süresinin %0’ı olduğu anlamına gelir.

Animasyon, ana kareler arasında otomatik olarak geçiş yapacaktır. Bu nedenle, sonraki arka plan rengini “%8.333” olarak ayarlayarak, animasyonun bu ana kareler arasında geçiş yapması zamanın %8.333’ünü sorunsuzca alacaktır.

.RainbowBackground {
    animation: rainbow-background 5s infinite;
}

Bu kod, animasyonumuzu .RainbowBackground sınıfına sahip tüm öğelere ekler.

Gerçek animasyon özelliği aşağıdaki argümanları alır.

  • animasyon-adı: Animasyonumuzun adı. Bu durumda, ‘gökkuşağı arka planı’
  • animasyon süresi: Animasyonun ne kadar süreceği, bu durumda 5 saniye.
  • animasyon-yineleme-sayısı (İsteğe bağlı): Animasyonun döngü sayısı. Bu durumda animasyon süresiz olarak devam edecektir. Varsayılan olarak, animasyon bir kez oynatılacaktır.
  • animasyon gecikmesi (İsteğe bağlı): Animasyon başlamadan önce ne kadar bekleneceğini belirtir. Varsayılan olarak 0 saniyedir ve negatif değerler alabilir. Örneğin, ‘-2s’, animasyonu 2 saniye içinde döngüsüne başlatır.
  • animasyon-zamanlama-fonksiyonu (Opsiyonel): Animasyonun hız eğrisini belirtir. Varsayılan olarak, animasyonun yavaş başladığı, hızlandığı ve yavaş bittiği yerde “ease” değerine ayarlanır.

Bu özel örnekte, hem “%” hem de “%100” anahtar kareleri “{ background-color: #ff0000; }`. İki veya daha fazla ana karenin bir durumu paylaştığı her yerde, bunlar tek bir ifadede belirtilebilir. Bu durumda, iki “0%” ve “%100” satırı, bu tek satırla değiştirilebilir:

0%, 100%     { background-color: #ff0000; }

Tarayıcılar arası uyumluluk #

Daha eski WebKit tabanlı tarayıcılar için, hem “@keyframes” bildiriminde hem de “animation” özelliğinde satıcı önekini kullanmanız gerekir, örneğin:

@-webkit-keyframes{}

-webkit-animation: ...

Geçiş özelliğine sahip animasyonlar #

Basit animasyonlar için kullanışlı olan CSS ‘geçiş’ özelliği, sayılara dayalı CSS özelliklerinin durumlar arasında canlandırılmasına izin verir.

Örnek #

.Example{
    height: 100px;
    background: #fff;
}

.Example:hover{
    height: 120px;
    background: #ff0000;
}

Sonucu Görüntüle

Varsayılan olarak, “.Example” sınıfına sahip bir öğenin üzerine gelindiğinde, öğenin yüksekliği hemen “120 piksel"e ve arka plan renginin kırmızıya ("#ff0000”) yükselmesine neden olur.

‘geçiş’ özelliğini ekleyerek, bu değişikliklerin zaman içinde gerçekleşmesine neden olabiliriz:

.Example{
    ...
    transition: all 400ms ease;
}

Sonucu Görüntüle

“all” değeri, geçişi tüm uyumlu (sayılara dayalı) özelliklere uygular. Bu anahtar sözcük için herhangi bir uyumlu özellik adı (“yükseklik” veya “üst” gibi) kullanılabilir.

“400ms” geçişin ne kadar süreceğini belirtir. Bu durumda, elemanın yüksekliğindeki değişikliğin tamamlanması 400 milisaniye sürer.

Son olarak, “ease” değeri, animasyonun nasıl oynatılacağını belirleyen animasyon işlevidir. “kolaylık”, yavaş başlamak, hızlandırmak ve sonra tekrar yavaş bitirmek anlamına gelir. Diğer değerler “doğrusal”, “kolaylık” ve “kolaylıktır”.

Çapraz Tarayıcı Uyumluluğu #

‘geçiş’ özelliği genellikle IE 9 hariç tüm büyük tarayıcılarda iyi desteklenir. Firefox’un önceki sürümleri ve Webkit tabanlı tarayıcılar için aşağıdaki gibi satıcı öneklerini kullanın:

.Example{
    transition:         all 400ms ease;
    -moz-transition:    all 400ms ease;
    -webkit-transition: all 400ms ease;
}

Not: ‘geçiş’ özelliği, birimden bağımsız olarak herhangi iki sayısal değer arasındaki değişiklikleri canlandırabilir. “100px” ile “50vh” gibi birimler arasında da geçiş yapabilir. Ancak, bir öğenin yüksekliğini “100 piksel"den “otomatik"e geçirmek gibi bir sayı ile varsayılan veya otomatik değer arasında geçiş yapamaz.

Sözdizimi Örnekleri #

İlk sözdizimi örneğimiz, mevcut tüm özellikleri/parametreleri kullanarak animasyon kestirme özelliğini gösterir:

  animation: 3s         ease-in           1s      2                 reverse     both        paused       slidein;
  /*         duration | timing-function | delay | iteration-count | direction | fill-mode | play-state | name   */

İkinci örneğimiz biraz daha basit ve bazı özelliklerin ihmal edilebileceğini gösteriyor:

  animation: 3s         linear            1s      slidein;
  /*         duration | timing-function | delay | name   */

Üçüncü örneğimiz en minimal beyanı göstermektedir. Animasyon adının ve animasyon süresinin bildirilmesi gerektiğini unutmayın:

  animation: 3s         slidein;
  /*         duration | name */

Ayrıca animasyon stenografisini kullanırken özelliklerin sırasının bir fark yarattığını belirtmekte fayda var. Açıkçası, tarayıcı sürenizi gecikmenizle karıştırabilir.

Kısalık sizin işiniz değilse, steno özelliğini atlayabilir ve her özelliği ayrı ayrı yazabilirsiniz:

animation-duration: 3s;
animation-timing-function: ease-in;
animation-delay: 1s;
animation-iteration-count: 2;
animation-direction: reverse;
animation-fill-mode: both;
animation-play-state: paused;
animation-name: slidein;

“will-change” Özelliğini Kullanarak Animasyon Performansını Artırma #

Animasyonlar ve diğer GPU ağırlıklı eylemler oluştururken, “will-change” özniteliğini anlamak önemlidir.

Hem CSS ana kareleri hem de “geçiş” özelliği GPU hızlandırmasını kullanır. Hesaplamalar cihazın GPU’suna yüklenerek performans artırılır. Bu, hesaplanmak üzere GPU’ya yüklenen boya katmanları (sayfanın ayrı ayrı oluşturulan bölümleri) oluşturularak yapılır. ‘will-change’ özelliği, tarayıcıya neyin canlandırılacağını söyler ve tarayıcının daha küçük boyama alanları oluşturmasına izin vererek performansı artırır.

‘will-change’ özelliği, canlandırılacak özelliklerin virgülle ayrılmış bir listesini kabul eder. Örneğin, bir nesneyi dönüştürmeyi ve opaklığını değiştirmeyi planlıyorsanız, şunları belirtirsiniz:

.Example{
    ...
    will-change: transform, opacity;
}

Not: ‘will-change’i dikkatli kullanın. Bir sayfadaki her öğe için “değişecek” ayarı, performans sorunlarına neden olabilir, çünkü tarayıcı her öğe için boya katmanları oluşturmaya çalışabilir ve bu da GPU tarafından yapılan işlem miktarını önemli ölçüde artırır.
