Parametreler #

Değer Etki
“hiç” Öğeyi gizleyin ve yer kaplamasını önleyin.
“blok” Blok elemanı, mevcut genişliğin %100’ünü kaplar, elemandan sonra bölünür.
“satır içi” Satır içi eleman, genişlik içermez, elemandan sonra kesinti olmaz.
“satır içi blok” Hem satır içi hem de blok elemanlardan özel özellikler alarak, kırılmaz, ancak genişliğe sahip olabilir.
“satır içi esnek” Bir öğeyi satır içi düzeyde esnek kapsayıcı olarak görüntüler.
‘satır içi tablo’ Öğe, satır içi düzey bir tablo olarak görüntülenir.
“ızgara” Bir blok eleman gibi davranır ve içeriğini ızgara modeline göre düzenler.
“esnek” Bir blok eleman gibi davranır ve içeriğini flexbox modeline göre düzenler.
‘miras’ Değeri üst öğeden devralır.
“ilk” Değeri, HTML belirtimlerinde açıklanan davranışlardan veya tarayıcı/kullanıcı varsayılan stil sayfasından alınan varsayılan değere sıfırlayın.
“masa” HTML “table” öğesi gibi davranır.
’tablo hücresi' Öğenin bir “” öğesi gibi davranmasına izin verin
table-column Öğenin bir <col> öğesi gibi davranmasına izin ver
table-row Öğenin bir <tr> öğesi gibi davranmasına izin verin
’liste öğesi' Öğenin bir “
  • ” öğesi gibi davranmasına izin verin.

    Görüntü özelliği #

    ‘display’ CSS özelliği, bir HTML belgesinin düzenini ve akışını kontrol etmek için esastır. Çoğu öğe, “blok” veya “satır içi” varsayılan bir “görüntüleme” değerine sahiptir (ancak bazı öğelerin başka varsayılan değerleri vardır).

    Çizgide #

    Bir “satır içi” öğe yalnızca gerektiği kadar genişlik kaplar. Aynı türdeki diğer öğelerle yatay olarak istiflenir ve diğer satır içi olmayan öğeler içermeyebilir.

    <span>This is some <b>bolded</b> text!</span>

    Yukarıda gösterildiği gibi, iki “satır içi” öğe, “” ve “” satır içidir (adından dolayı) ve metnin akışını bozmaz.

    Engellemek #

    Bir “blok” öğesi, ana öğesinin maksimum kullanılabilir genişliğini kaplar. Yeni bir satırla başlar ve ‘inline’ elemanlarının aksine içerebileceği elemanların türünü kısıtlamaz.

    <div>Hello world!</div><div>This is an example!</div>

    “div” öğesi varsayılan olarak blok düzeyindedir ve yukarıda gösterildiği gibi, iki “blok” öğesi dikey olarak istiflenir ve “satır içi” öğelerin aksine metnin akışı kesilir.

    Satır İçi Blok #

    “inline-block” değeri bize her iki dünyanın da en iyisini verir: öğeyi metnin akışıyla harmanlarken, “dolgu”, “kenar boşluğu”, “yükseklik” ve görünür etkisi olmayan benzer özellikleri kullanmamıza izin verir. “satır içi” öğeler üzerinde.

    Bu görüntüleme değerine sahip öğeler, normal metinlermiş gibi davranır ve sonuç olarak, “metin hizalama” gibi metin akışını kontrol eden kurallardan etkilenir. Varsayılan olarak, içeriklerini barındırmak için mümkün olan en küçük boyuta da küçültülürler.

    <!--Inline: unordered list-->
<style>
li {
    display : inline;
    background : lightblue;
    padding:10px;

    border-width:2px;
    border-color:black;
    border-style:solid;
    }
</style>    

<ul>
<li>First Element </li>
<li>Second Element </li>
<li>Third Element </li>
</ul>


    <!--block: unordered list-->
<style>
li {
    display : block;
    background : lightblue;
    padding:10px;

    border-width:2px;
    border-color:black;
    border-style:solid;
    }
</style>    

<ul>
<li>First Element </li>
<li>Second Element </li>
<li>Third Element </li>
</ul>


    <!--Inline-block: unordered list-->
<style>
li {
    display : inline-block;
    background : lightblue;
    padding:10px; 

    border-width:2px;
    border-color:black;
    border-style:solid;
    }

</style>    

<ul>
<li>First Element </li>
<li>Second Element </li>
<li>Third Element </li>
</ul>

    Yok #

    Display özelliğine none değeri verilen bir öğe hiç görüntülenmez.

    Örneğin, kimliği “myDiv” olan bir div öğesi oluşturalım:

    <div id="myDiv"></div>

    Bu, artık aşağıdaki CSS kuralı tarafından görüntülenmiyor olarak işaretlenebilir:

    #myDiv {
    display: none;
}

    Bir öğe ‘display:none;’ olarak ayarlandığında, tarayıcı o belirli öğe için diğer tüm düzen özelliklerini yok sayar (hem ‘konum’ hem de ‘kayan’). Bu öğe için hiçbir kutu oluşturulmaz ve html’deki varlığı, aşağıdaki öğelerin konumunu etkilemez.

    Bunun, “görünürlük” özelliğini “gizli” olarak ayarlamaktan farklı olduğunu unutmayın. Bir öğe için visibility: hidden; ayarı, öğeyi sayfada görüntülemez, ancak öğe, görünür olacakmış gibi oluşturma sürecinde yine de yer kaplar. Bu nedenle, aşağıdaki öğelerin sayfada nasıl görüntülendiğini etkiler.

    display özelliği için “none” değeri, öğeleri istendiğinde göstermek veya gizlemek için JavaScript ile birlikte yaygın olarak kullanılır, bu da onları gerçekten silme ve yeniden oluşturma ihtiyacını ortadan kaldırır.

    div kullanarak eski tablo yapısını elde etmek için #

    Bu normal HTML tablo yapısıdır

    <style>
    table {
        width: 100%;
    }
</style>

<table>
  <tr>
    <td>
        I'm a table
    </td>
  </tr>
</table>

    Aynı uygulamayı bunun gibi yapabilirsiniz

    <style>
    .table-div {
        display: table;
    }
    .table-row-div {
        display: table-row;
    }
    .table-cell-div {
        display: table-cell;
    }
</style>

<div class="table-div>
  <div class="table-row-div>
    <div class="table-cell-div>
      I behave like a table now
    </div>
  </div>
</div>
