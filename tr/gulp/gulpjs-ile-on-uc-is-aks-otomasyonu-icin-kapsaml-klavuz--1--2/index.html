Tüm Eklentileri Package.JSON’dan Yükleme #

Gulp’un nasıl kurulacağını anladığınızı varsayarak, projenizin kök klasörünün altında package.json’dan tüm gulp- bağımlılıklarını istememize izin verin.

Henüz bir gulpfileniz yoksa, lütfen isimle boş bir dosya oluşturun.

gulpfile.js

İlk olarak, yutmaya ihtiyacımız var. şöyle:

var $ = require('gulp');

Sırada, tüm eklentilerimizi aşağıdaki kod parçası ile gulpfile’ımıza yükleyelim.

Not #

Lütfen bu okumaya dahil edeceğimiz tüm snippet’lerdeki yorumları okuyun, bunlar her işlevsellik hakkında daha fazla bilgi sağlar.

/*
require gulp-load-plugins, and call it
gulp-load-plugins will require individually,
all the plugins installed in package.json at the root directory
these required plugins, are lazy loaded
that is, gulp-load-plugins will only require them when they are referenced in this file
plugins are available for perusal, via camelcased names, minus gulp
eg: gulp-clean-css is accessed by $$.cleanCss
*/
var $$ = require('gulp-load-plugins')();

NOT #

Makalede sahip olacağımız birçok örnek boyunca, biz takma ad

  1. $ olarak yut ve
  2. $$ olarak gulp-load-plugins

tamamen yazma kolaylığı için.

Duyarlı görüntüler için Eklentiler Yükleme|Css Minification|Js minification #

Amacımız,

  1. Çeşitli genişliklerde bir görüntü pili oluşturarak ve bunları küçülterek, Görüntülerinizi genişliklere ve uygun şekilde ölçeklendirin
  2. Javascript’inizi Lint
  3. Varlıklarınızı en aza indirin - JS/CSS/HTML , böylece en hafif koddan daha hafif bir kod barındırmanıza olanak tanır
  4. Değişiklik için css/js/image dosyalarını izleyin ve optimizasyonları yeniden oluşturun
  5. Geliştirme sırasında yaptığınız değişiklikleri sitenize hizmet eden bir tarayıcıda senkronize etmek.

Bir dizi eklentiye ihtiyacımız var, bu yüzden hepsini yükleyelim. Lütfen tüm bunları projenin kök klasöründe çalıştırın.

Not #

pipe, glob girişiyle eşleşen tüm dosyaları eklentilerimize (bu durumda minifyhtml) aktaran bir yöntemdir.

Bunu şu şekilde hayal etmek basittir:

$.src, akışı oluşturan şeydir ve boruları her bir bireyden çıkarır küreyi boru hattındaki her bir eklentiyle aşağı doğru eşleştiren dosya.Her dosyanın iletildiği eklenti, içeriğini yalnızca bellekte değiştirir akan dosyaları güncelleyen/oluşturan $.dest’e ulaşılana kadar $.src tarafından

Varlık Optimizasyonu ve Minyatür #

Bu nedenle, optimize edici işlevleri yazmadan önce, birkaç önbellek eklentisi yüklememiz gerekiyor.

bash $ npm install --save-dev gulp-cached
bash $ npm install --save-dev gulp-remember

Neden iki önbellek eh! diye merak ediyor olabilirsiniz. gulp-cached , yalnızca değiştirilmiş veya yeni içeriği boru hattından diğer eklentilere iletir. Bu nedenle, değiştirilmemiş dosyaların da varlık başına ( css | js ) tek bir dosyada birleştirmek için kullanılmasını istediğimizden, gulp-cached’e ek olarak gulp-remember‘a ihtiyacımız var.

İlk önce, değişen dosyaların bir listesini oluşturmak için gulp-cached kullanıyoruz

İkinci olarak, o listeden geçen tüm dosyaların kaydını bellekte tutmak için gulp-remember‘a ihtiyacımız var.

İlk Çalıştırma : Hiçbir dosya önbelleğe alınmaz , bu nedenle gulp-cached hepsini gulp-remember’a iletir

Sonraki çalıştırmalar : Yalnızca değiştirilmiş veya yeni dosyalar, yudum önbelleğe alınarak aktarılır. Geçerli dosyanın içeriği değiştiğinden, gulp-remember önbelleğini günceller.

Harika, ilk optimize edicimizi yazalım

Stil Optimize Edici #


// Goal

/*

1. cache existing files on the first run
2. each file , 
       a. Is autoprefixed with cross browser compatible prefixes for any css property ( justify-content for e.g)
       b. Is concatenated into app.css
3. app.css is minified
4. On subsequent runs , the above process is implemented on file modifications/additions only

*/

/*
*@src - input a glob pattern - a string eg 'images/**/*' or '**/*.css' or, an array eg ['glob1','glob2']
*/
var optimizeStyles = function(src) {

return $.src(src).
pipe($$.cached('styles')).
pipe($$.autoprefixer({
browsers: ['last 2 versions']
})).
pipe($$.remember('auto-prefixed-stylesheets')).
pipe($$.concat('app.css')).
pipe($.dest('build/css')).
pipe($$.cleanCss()).
pipe($$.rename({
suffix: '.min'
})).
pipe($.dest('build/css'))
}

Not #

$ - > yudum

$$ - > gulp-load-eklentiler

$.src - > src olarak geçirilen glob ile eşleşen dosya akışları oluşturur

$.dest - > değiştirilen dosyayı belirtilen yola kaydeder

Komut Dosyası Optimize Edici #


// Goal

/*

1. cache existing files on the first run
2. each file , 
       a. Is linted with jshint 
       b. Is concatenated into app.js
3. app.js is minified
4. On subsequent runs , the above process is implemented on file modifications/additions only

*/

/*
*@src - input a glob pattern - a string eg 'js/**/*' or '**/*.js' or, an array eg ['glob1','glob2']
*/

var optimizeScripts = function(src) {

    return $.src(src).
    pipe($$.cached('scripts')).
    pipe($$.jshint()).
    pipe($$.jshint.reporter('default')).
    pipe($$.jshint.reporter('fail')).
    pipe($$.remember('linted-scripts')).
    pipe($$.concat('app.js')).
    pipe($.dest('build/js')).
    pipe($$.uglify()).
    pipe($$.rename({
        suffix: '.min'
    })).
    pipe($.dest('build/js'))


}

$ - > yudum

$$ - > gulp-load-eklentiler

$.src - > src olarak geçirilen glob ile eşleşen dosya akışları oluşturur

Duyarlı Görüntüler Oluşturun #

Şimdi görüntü işlemeye geçelim. Yani amaç, sunacağınız her görüntü için bir dizi boyuta sahip olmaktır.

Neden? Niye?

Peki, neden bir dizi genişliğe sahip bir dizi görüntüye ihtiyacımız olduğunu anlamak için, muhtemelen çeşitli çözünürlüklere sahip zilyonlarca cihaz. Çok fazla pikselleşme olmadan ölçeklendirmek için bir görüntüye ihtiyacımız var. Aynı zamanda, sadece aşağıdakileri indirerek sayfa yükleme sürelerini iyileştirmemiz gerekiyor. içerdiği genişliğe uyan ve aynı zamanda bunu yapmak için mümkün olan en küçük boyut. bilimsel bloglar var etkisizliği vurgulayan Eric Portis’in yazdığı gibi yalnızca medya sorgularından oluşur ve kapsamlı bir kılavuz görevi görür. srcset’ler ve boyutlar gibi kavramları anlama.

Eric Portis’in epik yazısına buradan bakabilirsiniz

Şimdi, fonksiyonumuzun girdi olarak bir glob ve bir genişlik alması ve sihrini yapması ve her çalıştırmanın oluşturduğu dosyayı bir hedefe itmesi ve yanıtlanan görüntüyü küçültmesi gerekiyor.

İki görüntü sıkıştırma eklentisi kurduk ilk örneğimizde

Bu eklentiler YAPMA ön eki olan bir “gulp-” ile BAŞLAMADIĞINDAN, onları gulpfile’ımıza manuel olarak yüklememiz gerekiyor.

SO Gulpfile dosyamızın üst kısmındaki gulp-load-plugins bildiriminden sonra bunları manuel olarak talep edelim.

şöyle:

var compressJpg = require('imagemin-jpeg-recompress');
var pngquant = require('imagemin-pngquant');

Şunu belirtmekte fayda var ki, yutkunma tepkisi keskin FAAAAR TARAFINDAN imagemagick’ten daha iyi olan görüntü işlemcisi !. Keskin resimlerinizi istediğiniz şekilde kırpmak için gulp-responsive tarafından kullanılan şeydir genişlikler.

için gulp-responsive-configuration-options’a bakabilirsiniz. kapsamlı yapılandırma parametreleri listesi. sadece kullandım

  • genişlik - resimlerimizi bir parametre olarak geçirilen w genişliğine kırpmak için
  • yeniden adlandır - benzersiz kalması için görüntü adına bir sonek eklemek için

aşağıdaki yapılandırma işlevimde. yani fonksiyonumuz, görüntüyü girdi olarak geçirilen genişliğe göre kırpacak, glob girişi yoluyla deşifre edilen tüm eşleşen görüntüler için. ardından, her görüntü jpeg-recompress veya pngquant kullanılarak sıkıştırılır ve yapı/görüntüler içine kaydedildi.

Bunu akılda tutarak, fonksiyonumuz şöyle olurdu:

/*
@generateResponsiveImages
*@Description:takes in a src of globs, to stream matching image files , a width,
*to resize the matching image to, and a dest to write the resized and minified files to
*@src - input a glob pattern - a string eg 'images/** /*' or 'images/*' or, an array
eg ['glob1','glob2']
*@return returns a stream
*/
var generateResponsiveImages = function(src, width, dest) {

    //declare a default destination
    if (!dest)
        dest = 'build/images';
    //case 1: src glob -  images/**/*
    // the base is the directory immediately preceeding the glob - images/ in this case
    //case 2: images/fixed/flourish.png : the base here is images/fixed/ - we are overriding
    // that by setting base to images.This is done so that, the path beginning after images/
    // - is the path under our destination - without us setting the base, dest would be,
    //build/images/flourish.png.But with us setting the base, the destination would be
    // build/images/fixed/flourish.png
    return $.src(src, {
        base: 'images'
    })

    //generate resized images according to the width passed
    .pipe($$.responsive({
            //match all pngs within the src stream
            '**/*.png': [{
                width: width,
                rename: {
                    suffix: '-' + width
                },
                withoutEnlargement: false,
            }],
            //match all jpgs within the src stream
            '**/*.jpg': [{
                width: width,
                rename: {
                    suffix: '-' + width
                },
                progressive: true,
                withoutEnlargement: false,
            }]
        }, {

            errorOnEnlargement: false,
            errorOnUnusedConfig: false,
            errorOnUnusedImage: false

        }))
        //once the file is resized to width, minify it using the plugins available per format
        .pipe($$.if('*.jpg', compressJpg({
            min: 30,
            max: 90,
            target: 0.5
        })()))
        //use file based cache gulp-cache and it will minify only changed or new files
        //if it is not a new file and if the contents havent changed, the file is served from cache
        .pipe($$.cache($$.imagemin({
            verbose: true
        })))


    //write to destination - dest + path from base
    .pipe($.dest(dest));
}

$.src - > src olarak geçirilen glob ile eşleşen dosya akışları oluşturur

HTML Küçültücü #

*
 *@minifyHtml
 *Description:takes in a glob src, and minifies all '.html' files matched by the glob
 *@src - input a glob pattern - a string eg '/**/*.html /*' or '*.html' or, an array eg ['glob1','glob2']
 *@dest=file.base means, the modified html file will be in the same directory as the src file being minified
 *often means, the process is just a modification on the existing src file
 *@return returns a stream
 */
var minifyHtml = function(src) {
    return $.src(src)
        .pipe($$.minifyHtml())
        .pipe($.dest(function(file) {
            //file is provided to a dest callback -
            // Refer here https://github.com/gulpjs/gulp/blob/master/docs/API.md#path
            return file.base;
        }));
}

Bir yudum görevinin anatomisi #

Bir görev tanımının anatomisi şöyledir:

$.task(<taskname> , [dependencies] , <body>);

dependencies , is an array of tasks that HAVE to finish before the current task you are defining , runs. More like forcing a synchronous execution instead of the default Asynchronous functionality.

Adding Gulp Tasks #

So, we now have

  • A function defined Above to optimise Styles
  • A function defined Above to optimise scripts
  • A function defined Above to optimise HTML
  • A function to generate multiple images per image Above

All we need to do now, is to invoke them when needed.

Let us write our tasks according to the syntax we defined earlier

/*
* $.task('isim', 'bağımlılık dizisi',işlev)
aşağıdaki gibi bir görev nesnesi oluşturmayla sonuçlanır
görev:{
'isim': isim,
'dep':[bağımlılık dizisi],
'fn':işlev
}
*/


//*@return, görev tamamlandığında bildirimde bulunmak için bir akış döndürür
$.task('optimizeHtml', function() {
    var src = ['**/*.html', '!{node_modules}/**/*.html'];
    return minifyHtml(src);
});

//*@return, görev tamamlandığında bildirimde bulunmak için bir akış döndürür
$.task('optimizeScripts', function() {
    var src = ['js/**/*.js'];
    return optimizeScripts(src);
});

//*@return, görev tamamlandığında bildirimde bulunmak için bir akış döndürür
$.task('optimizeStyles', function() {
    var src = ['css/**/*.css', 'fonts/google/**/*.css'];
    return optimizeStyles(src);
});

//Gülüp motoruna görevin tamamlandığını bildirmek için bir geri arama yapın
// çünkü bu görevde bir akış döndürmediğiniz için gerekli
$.task('generateResponsiveImages', function(geri arama) {
    var src = ['images/**/*.{jpg,png}'];
    for (var i = widths.length - 1; i >= 0; i--) {
        generateResponsiveImages(src, widths[i]);
    }
    callback();

});
