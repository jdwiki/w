Önce ‘Merhaba Dünya!’ CakePHP 3.x ile uygulama (Veritabanı tabloları geçişleri. Bölüm 2) #

Veritabanınız için kolayca “oluşturabilirsiniz” veya isterseniz bunları “bırakabilirsiniz”. Bunu yapmak istiyorsanız, istediğiniz veritabanı için Migrations yazmayı öğrenmelisiniz.

Migrations dosyaları config/Migrations klasöründe bulunmalıdır. Dosya adları sonraki biçimlerde olabilir:

  • YYYYMMDDHHIISS_(Oluştur|Değiştir|Sil)AdministratorsTable.php

  • (1-9){1,}_(Oluştur|Değiştir|Sil)AdministratorsTable.php

    _tableName = '"Administrators"'; $this->_tablePrefix = 'administrators'; } public function up() { Log::info("Trying to create {$this->_tableName} table"); $administratorsTable = $this->table($this->_tablePrefix); if ($administratorsTable->exists()) { return Log::warning("Table {$this->_tableName} already exists"); } $administratorsTable ->addPrimaryKey('id') ->addColumn('username', 'char', [ 'length' => 25, 'null' => false ]) ->addColumn('password', 'char', [ 'length' => 255, 'null' => false ]) ->addColumn('email', 'char', [ 'length' => 50, 'null' => false ]) ->addColumn('first_name', 'char', [ 'length' => 50, 'null' => false ]) ->addColumn('last_name', 'char', [ 'length' => 50, 'null' => false ]) ->addColumn('avatar', 'char', [ 'length' => 255, 'default' => '/img/no-avatar.png' ]) ->addColumn('active', 'boolean', [ 'default' => 0 ]) ->addTimestamps() ->create(); return Log::notice("Table {$this->_tableName} has been created"); } public function down() { if ($this->table($this->_tablePrefix)->exists()) { $this->table($this->_tablePrefix)->drop(); return Log::info("Table {$this->_tableName} has been dropped"); } return Log::warning("Table {$this->_tableName} does not exists"); } }

Taşımayı çalıştırmak istiyorsanız, sonraki komutu çalıştırmanız gerekir:

Tablo(-lar) oluşturmak için “bin/kek geçişleri taşınır”.

Geri almak istiyorsanız:

“bin/cake taşımaları geri alma” - “drop()” işlevinin mevcut olduğu son taşımayı geri döndürür

bin/cake göçleri (-t|--target) all - drop() fonksiyonunun mevcut olduğu tüm geçişleri geri alır

Cakephp 3.X Kurulumu #

Gereksinimler:

  • PHP 5.6.0 veya üzeri mbstring PHP uzantısı (varsayılan WAMP/XAMPP, değilse etkinleştirin)
  • intl PHP uzantısı (WAMP/XAMPP’de mevcuttur, etkinleştirmeniz gerekir php.ini’den)
  • CakePHP, nginx gibi çeşitli web sunucularında çalışacak,
  • LightHTTPD veya Microsoft IIS.

Başlamadan önce PHP sürümünüzün güncel olduğundan emin olmalısınız:

php -v

Cakephp 3 Framework’ü kurmak için Composer’ı kullanın,

Besteci, Kurulum için resmi olarak desteklenen bir yöntemdir, bu yüzden besteciyi Composer (Windows/Linux/Mac) adresinden indirin

Cakephp’yi yüklemek için bunu çalıştırın,

php composer.phar create-project --prefer-dist cakephp/app my_app_name

Composer, uygulama iskeletini ve temel CakePHP kitaplığını indirmeyi bitirdiğinde, Composer aracılığıyla kurulu çalışan bir CakePHP uygulamasına sahip olmalısınız. Composer.json ve composer.lock dosyalarını kaynak kodunuzun geri kalanıyla birlikte sakladığınızdan emin olun.

Veya cakephp’yi kurmak için bu en kolay yolu izleyin

Aşağıdaki adımları izleyin,

  1. bu Git Deposu adresine gidin
  2. Kolay kurulum için cakeDC/fırını indirin
  3. Zip dosyasını LOCALHOST içine çıkarın
  4. Bu klasöre 777 izni verin
  5. Fırın.php dosyasını çalıştırın (klasörün içinde bulunur)
  6. Çok seçenekli bir sayfa açılacaktır, tercihinize göre seçeneği seçin
  7. Resme tıklayın ve arkanıza yaslanın. Projeniz birkaç dakika içinde kurulacak

uygulamak:

bin/cake server

Varsayılan olarak, herhangi bir argüman sağlanmadan bu, uygulamanıza http://localhost:8765/ adresinde hizmet verecektir.

bunu tarayıcınızda çalıştırın, http://example.com/ veya http://localhost:8765/. Bu noktada, CakePHP’nin varsayılan ana sayfası ve mevcut veritabanı bağlantınızın durumunu bildiren bir mesaj ile karşılaşacaksınız ve ilk uygulamanız için hazırsınız.

Kurulum ve Kurulum hakkında daha fazla ayrıntı için, Cakephp 3.X Kurulumu sayfasını takip edin.

Proje Kurulumu #

İlk olarak, Veritabanı ile çalışmak için mysql, phpMyAdmin, HeidiSQL veya başka araçlarla veritabanı oluşturmalı ve kullanıcının yeni bir tane oluşturmasına izin vermelisiniz.

Bu işlemden sonra proje için veritabanına erişim sağlamalısınız.

/path/to/your/project/config/app.php dosyasına girmeniz ve ardından Datasources varsayılan aramanız gerekir. Bu dizide, ’localhost’ (isteğe bağlı), ‘user’, ‘password’ ve ‘database’i değiştirmeniz gerekiyor.

o Tarayıcınıza gidin ve sayfayı yenileyin. DB sorunu gitmeli ve sol tarafta ‘Yeşil Kene’ göstermelidir.

Önce ‘Merhaba Dünya!’ CakePHP 3.x ile uygulama (Giriş. Bölüm 1) #

CakePHP 3.x, “kontrolörler”, “modeller”, “görünümler” ve diğer çerçeve tanımlı nesneleri “pişirme” yeteneğine sahiptir.

Not : Eğer ‘Laravel’ çerçevesi ile biraz tecrübeniz varsa, ‘artisan’ bileşeni ‘bake’ ile benzerdir.

‘bake’ uygulaması ‘bin’ klasöründe bulunur; kullanılabilir komutlardan bazıları şunlardır:

  • bin/kek kabuk %shellName% - ShellClass pişirmek için
  • bin/kek pişirme kontrolörü %controllerName% - Kontrolör Sınıfını pişirmek için
  • bin/kek fırınlama modeli %modelName% - Model + Varlık Sınıfını pişirmek için
  • bin/kek bake görünümü %viewName% - pişirmek için Şablonu görüntüle
  • bin/cake tüm %className%'yi pişir - Denetleyici, Model + Varlık, Geliştirici için Görünüm’ü pişirmek için.

Not : Veritabanınızda hiç tablonuz yoksa ‘model pişiremezsiniz’

Not : Tüm bileşenleri “pişirirseniz”, önceden tanımlanmış “CRUD” eylemlerine sahip “Kontrolörler” alırsınız.

Önce ‘Merhaba Dünya!’ CakePHP 3.x ile uygulama (Kontrolörler, Yanıt, Görünüm. Bölüm 3) #

Bir denetleyici oluşturmak ister misiniz? Oluşturmanın 2 yolu vardır:

  • Manuel olarak (src/Controller içinde Kontrolör dosyasını manuel olarak oluşturmak zorunda kalacaksınız)
  • Fırında (CLI’den bin/kek pişirme kontrolörü %controllerName% komutunu çalıştırıyor)

Manuel olarak oluşturmak istiyorsanız, src/Controller klasörüne gidin ve aşağıdaki kalıbı takip eden dosyayı oluşturun:

([A-Z]{1}[a-z]{1,})Controller.php

Bu denetleyicide, kullanılacak olan ’namspace’i tanımlamanız gerekir:

<?php
    namespace App\Controller;

O zaman onu dosya adı olarak adlandırmalısın, ör. Yönetici Denetleyici:

use App\Controller\AppController;

class AdminController extends AppController{}

Bu sınıfın içinde, ilk yönteminizi oluşturmalısınız, örn. “giriş”:

public function login(){}

Tarayıcınıza http://{{proje-adı}}/admin/login yazarsanız, şablon eksik hatası verir. Bu sorun nasıl çözülür?

src/Template/Admin/```login.ctp dosyası altında oluşturmanız gerekiyor.

Not : *.ctp joker karakter - Denetleyici aracılığıyla ayarladığınız verileri geçirmek/işlemek için kullanılan Cake Template dosyasıdır.

Bu dosyaya sadece ‘Merhaba Dünya!’ yazın. İstediğiniz yerde, şablon hatasıyla sayfayı yenileyin ve sizi karşılayan `Dünya’nızı alacaksınız!

Not : Varsayılan olarak, src/Template/Layout/default.ctp, eğer siz bir tane tanımlamadıysanız, düzen olarak işleniyor.

CakePHP 3.4’ü CentOS 7’ye PHP 7 ve SELinux etkinleştirilmiş olarak yükleme #

CakePHP’yi yeni kurulmuş bir minimal CentOS 7’ye kurmak için yaptığım şey buydu.

  • VirtualBox’a bir CentOS-7-x86_64-Minimal-1611.iso yüklendi, iki ağ arabirimi: birinci NAT, ikinci Yalnızca Ana Bilgisayar

  • /etc/sysconfig/network-scripts/ifcfg-enp0s3 içinde ONBOOT=yes olarak ayarlayın

  • yeniden başlat

  • yum güncellemesi

  • yum net-tools yükleyin (ifconfig ve netstat almak için)

  • yum wget’i yükleyin

  • yum yum-utils yükleyin

  • wget -q http://rpms.remirepo.net/enterprise/remi-release-7.rpm

  • wget -q https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm

  • rpm -Uvh remi-release-7.rpm epel-release-son-7.noarch.rpm

  • yum-config-manager –remi-php71’i etkinleştir

  • yum php yükleyin

  • systemctl httpd’yi etkinleştir

  • systemctl httpd’yi başlat

  • güvenlik duvarı-cmd –kalıcı –zone=genel –add-service=http

  • güvenlik duvarı-cmd –kalıcı –zone=genel –add-service=https

  • güvenlik duvarı-cmd –yeniden yükle

  • yum httpd mariadb-server mariadb php phpmyadmin’i kurun

  • systemctl mariadb’yi başlat

  • systemctl mariadb’yi etkinleştirir

  • systemctl httpd’yi yeniden başlat

  • yum php-mbstring php-intl’yi kurun

  • mysql_secure_installation

  • curl -s https://getcomposer.org/installer | php

  • cd /var/www/html/

  • php composer.phar create-project –prefer-dist cakephp/app MyApp

  • chown apache: -R Uygulamam/

  • Veritabanı yarat:

    mysql -u root -p #

    Enter password:

    mysql> CREATE DATABASE mydb; mysql> GRANT ALL ON mydb.* to ‘myuser’@’localhost’ IDENTIFIED BY ‘password’; mysql> FLUSH PRIVILEGES; mysql> quit

  • İçeriği olan /etc/httpd/conf.d/my_app.conf dosyasını oluşturun:

    <VirtualHost *:80> ServerAdmin root@localhost ServerName cakephp.myapp.net DocumentRoot /var/www/html/MyApp <Directory /var/www/html/MyApp> Allowoverride All

  • cd /var/www/html/secure_logging; chcon -Rv –type=httpd_user_content_rw_t tmp

  • /.autorelabel’e dokunun; yeniden başlat

  • Ana makinemde /etc/hosts dosyasını düzenler ve satırı girerim (192.168.56.101, sanal makinemin yalnızca ana bilgisayar ip adresidir) 192.168.56.101 cakephp.myapp.net

  • http://cakephp.myapp.net/‘de gezinin

  • Yapılacaklar: Veritabanı bağlantı dosyasını düzenleyin.
