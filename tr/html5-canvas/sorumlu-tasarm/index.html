Duyarlı bir tam sayfa tuval oluşturma #

Javascript aracılığıyla olayları yeniden boyutlandırmaya yanıt veren tam sayfa bir tuval oluşturmak ve kaldırmak için başlangıç ​​kodu.

var canvas;    // Global canvas reference
var ctx;       // Global 2D context reference
// Creates a canvas
function createCanvas () {                
    const canvas = document.createElement("canvas"); 
    canvas.style.position = "absolute"; // Set the style 
    canvas.style.left     = "0px";      // Position in top left
    canvas.style.top      = "0px";
    canvas.style.zIndex   = 1;        
    document.body.appendChild(canvas);  // Add to document
    return canvas;
}
// Resizes canvas. Will create a canvas if it does not exist
function sizeCanvas () {                
    if (canvas === undefined) {         // Check for global canvas reference
        canvas = createCanvas();        // Create a new canvas element
        ctx = canvas.getContext("2d");  // Get the 2D context
    }
    canvas.width  = innerWidth;         // Set the canvas resolution to fill the page
    canvas.height = innerHeight;        
}
// Removes the canvas
function removeCanvas () {
    if (canvas !== undefined) {              // Make sure there is something to remove
        removeEventListener("resize", sizeCanvas); // Remove resize event
        document.body.removeChild(canvas);   // Remove the canvas from the DOM
        ctx = undefined;                     // Dereference the context
        canvas = undefined;                  // Dereference the canvas
     }
}

// Add the resize listener
addEventListener("resize", sizeCanvas); 
// Call sizeCanvas to create and set the canvas resolution
sizeCanvas();
// ctx and canvas are now available for use.

Tuvale artık ihtiyacınız yoksa, removeCanvas() diyerek onu kaldırabilirsiniz.

Yeniden boyutlandırma (veya kaydırma) sonrasında fare koordinatları #

Tuval uygulamaları genellikle fare ile kullanıcı etkileşimine dayanır, ancak pencere yeniden boyutlandırıldığında, tuvalin dayandığı fare olay koordinatları büyük olasılıkla değişir, çünkü yeniden boyutlandırma tuvalin pencereye göre farklı bir konumda kaymasına neden olur. Bu nedenle, duyarlı tasarım, pencere yeniden boyutlandırıldığında tuval ofset konumunun yeniden hesaplanmasını ve ayrıca pencere kaydırıldığında yeniden hesaplanmasını gerektirir.

Bu kod, pencere yeniden boyutlandırma olaylarını dinler ve fare olay işleyicilerinde kullanılan ofsetleri yeniden hesaplar:

// variables holding the current canvas offset position
//    relative to the window
var offsetX,offsetY;

// a function to recalculate the canvas offsets
function reOffset(){
    var BB=canvas.getBoundingClientRect();
    offsetX=BB.left;
    offsetY=BB.top;        
}

// listen for window resizing (and scrolling) events
//     and then recalculate the canvas offsets
window.onscroll=function(e){ reOffset(); }
window.onresize=function(e){ reOffset(); }

// example usage of the offsets in a mouse handler
function handleMouseUp(e){
    // use offsetX & offsetY to get the correct mouse position
    mouseX=parseInt(e.clientX-offsetX);
    mouseY=parseInt(e.clientY-offsetY);
    // ...
}

Yeniden boyutlandırma olayları olmadan duyarlı tuval animasyonları. #

Pencere yeniden boyutlandırma olayları, kullanıcının giriş cihazının hareketine yanıt olarak tetiklenebilir. Bir tuvali yeniden boyutlandırdığınızda, otomatik olarak temizlenir ve içeriği yeniden oluşturmanız gerekir. Animasyonlar için bunu her karede, işlemeyi ekran donanımıyla senkronize tutmak için elinden gelenin en iyisini yapan “requestAnimationFrame” tarafından çağrılan ana döngü işlevi aracılığıyla yaparsınız.

Yeniden boyutlandırma olayıyla ilgili sorun, pencereyi yeniden boyutlandırmak için fare kullanıldığında olayların, tarayıcının standart 60 fps hızından çok daha hızlı tetiklenebilmesidir. Yeniden boyutlandırma olayı çıktığında, tuval geri arabelleği, DOM’a görüntüleme aygıtıyla senkronize değil olarak sunulur ve bu da kesmeye ve diğer olumsuz etkilere neden olabilir. Ayrıca, bir süre sonra GC temizlendiğinde animasyonu daha fazla etkileyebilecek çok sayıda gereksiz bellek ayırma ve serbest bırakma vardır.

Geri dönen yeniden boyutlandırma etkinliği #

Yeniden boyutlandırma olayının yüksek tetikleme oranlarıyla başa çıkmanın yaygın bir yolu, yeniden boyutlandırma olayının geri dönmesidir.

 // Assume canvas is in scope
 addEventListener.("resize", debouncedResize );

 // debounce timeout handle
 var debounceTimeoutHandle;

 // The debounce time in ms (1/1000th second)
 const DEBOUNCE_TIME = 100; 

 // Resize function 
 function debouncedResize () { 
     clearTimeout(debounceTimeoutHandle);  // Clears any pending debounce events

     // Schedule a canvas resize 
     debounceTimeoutHandle = setTimeout(resizeCanvas, DEBOUNCE_TIME);
 }

 // canvas resize function
 function resizeCanvas () { ... resize and redraw ... }

Yukarıdaki örnek, yeniden boyutlandırma olayından 100 ms sonra tuvalin yeniden boyutlandırılmasını geciktirir. Bu süre içinde daha fazla yeniden boyutlandırma olayı tetiklenirse, mevcut yeniden boyutlandırma zaman aşımı iptal edilir ve yenisi planlanır. Bu, yeniden boyutlandırma olaylarının çoğunu etkili bir şekilde tüketir.

Hala bazı sorunları var, en dikkate değer olanı yeniden boyutlandırma ve yeniden boyutlandırılmış tuvali görme arasındaki gecikme. Geri dönme süresinin azaltılması bunu iyileştirir, ancak yeniden boyutlandırma hala görüntüleme cihazıyla senkronize değildir. Ayrıca, uygun olmayan bir tuvale animasyon ana döngüsü oluşturma işlemine de sahipsiniz.

Basit ve en iyi yeniden boyutlandırma #

Tuvalin yeniden boyutlandırılmasını yumuşatmak için çok farklı yollar denedikten sonra, saçma sapan karmaşıklıktan sorunu görmezden gelmeye kadar (kimin umurunda ki?) Güvenilir bir arkadaşıma geri döndüm.

K.I.S.S. çoğu programcının bilmesi gereken bir şeydir ((Keep It Sbasit Saptal) Aptal, düşünmediğim için beni kastediyor yıllar önce. ) ve en iyi çözümün hepsinin en basiti olduğu ortaya çıktı.

Tuvali ana animasyon döngüsünden yeniden boyutlandırmanız yeterlidir. Görüntüleme cihazıyla senkronize kalır, gereksiz işleme yoktur ve tam kare hızını korurken kaynak yönetimi mümkün olan en düşük seviyededir. Ayrıca pencereye bir yeniden boyutlandırma olayı veya herhangi bir ek yeniden boyutlandırma işlevi eklemeniz gerekmez.

Yeniden boyutlandırmayı, tuval boyutunun pencere boyutuyla eşleşip eşleşmediğini kontrol ederek normalde tuvali temizleyeceğiniz yere eklersiniz. Yeniden boyutlandırılmazsa.

// Assumes canvas element is in scope as canvas

// Standard main loop function callback from requestAnimationFrame
function mainLoop(time) {

    // Check if the canvas size matches the window size
    if (canvas.width !== innerWidth || canvas.height !== innerHeight) {
        canvas.width = innerWidth;    // resize canvas
        canvas.height = innerHeight;  // also clears the canvas
    } else {
        ctx.clearRect(0, 0, canvas.width, canvas.height); // clear if not resized
    }

    // Animation code as normal.



    requestAnimationFrame(mainLoop);
}
