Çevrilmiş, ölçeklenmiş ve döndürülmüş birçok görüntüyü hızla çizme #

Döndürülmüş, ölçeklenmiş ve çevrilmiş bir görüntü çizmek istediğiniz birçok durum vardır. Döndürme görüntünün merkezi etrafında gerçekleşmelidir. 2B tuvalde bunu yapmanın en hızlı yolu budur. Bu işlevler, saniyenin her 60’ında bir, birkaç yüz hatta 1000’den fazla görüntü oluşturma beklentisinin olduğu 2D oyunlara çok uygundur. (donanım bağlıdır)

// assumes that the canvas context is in ctx and in scope
function drawImageRST(image, x, y, scale, rotation){
    ctx.setTransform(scale, 0, 0, scale, x, y); // set the scale and translation
    ctx.rotate(rotation);                       // add the rotation
    ctx.drawImage(image, -image.width / 2, -image.height / 2); // draw the image offset by half its width and height
}

Bir varyant, parçacık sistemleri için yararlı olan alfa değerini de içerebilir.

function drawImageRST_Alpha(image, x, y, scale, rotation, alpha){
    ctx.setTransform(scale, 0, 0, scale, x, y); // set the scale and translation
    ctx.rotate(rotation);                       // add the rotation
    ctx.globalAlpha = alpha;
    ctx.drawImage(image, -image.width / 2, -image.height / 2); // draw the image offset by half its width and height
}

Her iki işlevin de tuval bağlamını rastgele bir durumda bıraktığına dikkat etmek önemlidir. Olsa da diğer render fonksiyonlarım etkilenmeyecek. Görüntüleri oluşturmayı tamamladığınızda, varsayılan dönüşümü geri yüklemeniz gerekebilir.

ctx.setTransform(1, 0, 0, 1, 0, 0); // set the context transform back to the default

Alfa sürümünü (ikinci örnek) ve ardından standart sürümü kullanırsanız, genel alfa durumunun geri yüklendiğinden emin olmanız gerekir.

ctx.globalAlpha = 1;

Bazı parçacıkları ve birkaç görüntüyü oluşturmak için yukarıdaki işlevlerin kullanılmasına bir örnek

// assume particles to contain an array of particles
for(var i = 0; i < particles.length; i++){
    var p = particles[i];
    drawImageRST_Alpha(p.image, p.x, p.y, p.scale, p.rot, p.alpha);
    // no need to rest the alpha in the loop
}
// you need to reset the alpha as it can be any value 
ctx.globalAlpha = 1;

drawImageRST(myImage, 100, 100, 1, 0.5);  // draw an image at 100,100
// no need to reset the transform 
drawImageRST(myImage, 200, 200, 1, -0.5); // draw an image at 200,200 
ctx.setTransform(1,0,0,1,0,0);            // reset the transform

Bir Resmi veya Yolu merkez noktasının etrafında döndürün #

Aşağıdaki 1-5. adımlar, herhangi bir görüntünün veya yol şeklinin, görüntünün/yol şeklinin orijinal nokta koordinatlarını değiştirmeden tuval üzerinde herhangi bir yere taşınmasına ve herhangi bir açıya döndürülmesine izin verir.

  1. Kanvasın [0,0] orijinini şeklin merkez noktasına taşıyın

    context.translate( shapeCenterX, shapeCenterY );

  2. Tuvali istediğiniz açıda döndürün (radyan cinsinden)

    context.rotate( radianAngle );

  3. Tuval orijini sol üst köşeye geri taşıyın

     context.translate( -shapeCenterX, -shapeCenterY );

  4. Orijinal koordinatlarını kullanarak görüntüyü veya yol şeklini çizin.

     context.fillRect( shapeX, shapeY, shapeWidth, shapeHeight );

  5. Daima temizleyin! Dönüşüm durumunu #1’den önceki haline geri getirin

  • Adım#5, Seçenek#1: Her dönüşümü ters sırada geri alın

          // undo #3
      context.translate( shapeCenterX, shapeCenterY );
      // undo #2
      context.rotate( -radianAngle );
      // undo #1
      context.translate( -shapeCenterX, shapeCenterY );

    • Step#5, Option#2: If the canvas was in an untransformed state (the default) before beginning step#1, you can undo the effects of steps#1-3 by resetting all transformations to their default state

         // set transformation to the default state (==no transformation applied)
   context.setTransform(1,0,0,1,0,0)

Örnek kod demosu:

// canvas references & canvas styling
var canvas=document.createElement("canvas");
canvas.style.border='1px solid red';
document.body.appendChild(canvas);
canvas.width=378;
canvas.height=256;
var ctx=canvas.getContext("2d");
ctx.fillStyle='green';
ctx.globalAlpha=0.35;        

// define a rectangle to rotate
var rect={ x:100, y:100, width:175, height:50 };

// draw the rectangle unrotated
ctx.fillRect( rect.x, rect.y, rect.width, rect.height );

// draw the rectangle rotated by 45 degrees (==PI/4 radians)
ctx.translate( rect.x+rect.width/2, rect.y+rect.height/2 );
ctx.rotate( Math.PI/4 );
ctx.translate( -rect.x-rect.width/2, -rect.y-rect.height/2 );
ctx.fillRect( rect.x, rect.y, rect.width, rect.height );

Dönüşümlere Giriş #

Dönüşümler, belirli bir noktanın başlangıç ​​konumunu, o noktayı hareket ettirerek, döndürerek ve ölçeklendirerek değiştirir.

  • Çeviri: Bir noktayı “distanceX” ve “distanceY” ile hareket ettirir.
  • Döndürme: Bir noktayı kendi dönüş noktası etrafında “radyan açısı” ile döndürür. Html Canvas’taki varsayılan döndürme noktası, Canvas’ın sol üst başlangıç ​​noktasıdır [x=0,y=0]. Ancak çevirileri kullanarak dönüş noktasını yeniden konumlandırabilirsiniz.
  • Scaling: Bir noktanın konumunu, ölçekleme noktasından bir “scalingFactorX” ve “scalingFactorY” ile ölçekler. Html Canvas’taki varsayılan ölçekleme noktası, Canvas’ın sol üst başlangıç ​​noktasıdır [x=0,y=0]. Ancak, çevirileri kullanarak ölçekleme noktasını yeniden konumlandırabilirsiniz.

Ayrıca, “context.transform” kullanarak tuvalin dönüşüm matrisini doğrudan ayarlayarak, kesme (eğriltme) gibi daha az yaygın olan dönüşümleri de yapabilirsiniz.

Bir noktayı context.translate(75,25) ile çevir (==move)

context.rotate(Math.PI/8) ile bir noktayı döndür

Bir noktayı context.scale(2,2) ile ölçeklendir

Kanvas, aslında tuvalin tüm koordinat sistemini değiştirerek dönüşümler gerçekleştirir.

  • “context.translate”, kanvasın [0,0] başlangıç ​​noktasını sol üst köşeden yeni bir konuma taşır.
  • “context.rotate”, tüm tuval koordinat sistemini orijin etrafında döndürür.
  • “bağlam.ölçek”, tüm tuval koordinat sistemini başlangıç ​​noktası etrafında ölçeklendirir. Bunu tuvaldeki her x,y’nin boyutunu artırmak olarak düşünün: “her x*=ölçekX” ve “her y*=ölçekY”.

Tuval dönüşümleri kalıcıdır. Tüm Yeni çizimler, siz tuvalin dönüşümünü varsayılan durumuna (==tamamen dönüştürülmemiş) sıfırlayana kadar dönüştürülmeye devam edecektir. Aşağıdakilerle varsayılana sıfırlayabilirsiniz:

// reset context transformations to the default (untransformed) state
context.setTransform(1,0,0,1,0,0);

Çevrilmiş, döndürülmüş ve ölçeklenmiş şekil(ler)i izlemek için bir Dönüşüm Matrisi #

Kanvas, şeklinizi istediğiniz konum ve boyutta çizmek için “bağlam.çevir”, “bağlam.döndür” ve “bağlam.ölçek” yapmanıza olanak tanır.

Canvas, dönüşümleri verimli bir şekilde izlemek için bir dönüşüm matrisi kullanır.

  • Canvas’ın matrisini context.transform ile değiştirebilirsiniz.
  • Bireysel çevir, döndür ve ölçeklendir komutlarıyla Canvas matrisini değiştirebilirsiniz.
  • context.setTransform ile Canvas matrisinin üzerine tamamen yazabilirsiniz,
  • Ancak Canvas’ın dahili dönüşüm matrisini okuyamazsınız – bu salt yazılır.

Neden bir dönüşüm matrisi kullanalım? #

Bir dönüşüm matrisi, birçok bireysel ötelemeyi, döndürmeyi ve ölçeklendirmeyi tek, kolayca yeniden uygulanabilen bir matriste toplamanıza olanak tanır.

Karmaşık animasyonlar sırasında bir şekle düzinelerce (veya yüzlerce) dönüşüm uygulayabilirsiniz. Bir dönüşüm matrisi kullanarak (neredeyse) bu düzinelerce dönüşümü tek bir kod satırıyla anında yeniden uygulayabilirsiniz.

Bazı Örnek kullanımları:

  • Farenin çevirdiğiniz, döndürdüğünüz ve ölçeklediğiniz bir şeklin içinde olup olmadığını test edin

    There is a built-in context.isPointInPath that tests if a point (eg the mouse) is inside a path-shape, but this built-in test is very slow compared to testing using a matrix.

Farenin bir şeklin içinde olup olmadığının verimli bir şekilde test edilmesi, tarayıcı tarafından bildirilen fare konumunu almayı ve onu şeklin dönüştürüldüğü şekilde dönüştürmeyi içerir. Ardından, şekil dönüştürülmemiş gibi isabet testi uygulayabilirsiniz.

  • ** Kapsamlı bir şekilde çevrilmiş, döndürülmüş ve ölçeklenmiş bir şekli yeniden çizin.**

    Instead of reapplying individual transformations with multiple .translate, .rotate, .scale you can apply all the aggregated transformations in a single line of code.

  • Çevrilmiş, döndürülmüş ve ölçeklendirilmiş çarpışma testi şekilleri

    You can use geometry & trigonometry to calculate the points that make up transformed shapes, but it’s faster to use a transformation matrix to calculate those points.

Bir Dönüşüm Matrisi “Sınıf” #

Bu kod, yerel “context.translate”, “context.rotate”, “context.scale” dönüştürme komutlarını yansıtır. Yerel tuval matrisinin aksine, bu matris okunabilir ve yeniden kullanılabilir.

Yöntemler:

  • ‘çevir’, ‘döndür’, ‘ölçek’ bağlam dönüştürme komutlarını yansıtır ve dönüşümleri matrise beslemenize izin verir. Matris, toplu dönüşümleri verimli bir şekilde tutar.

  • setContextTransform bir bağlam alır ve bu bağlamın matrisini bu dönüşüm matrisine eşit olarak ayarlar. Bu, bu matriste depolanan tüm dönüşümleri bağlama verimli bir şekilde yeniden uygular.

  • ‘resetContextTransform’, bağlamın dönüşümünü varsayılan durumuna (==dönüştürülmemiş) sıfırlar.

  • getTransformedPoint dönüştürülmemiş bir koordinat noktasını alır ve onu dönüştürülmüş bir noktaya dönüştürür.

  • “getScreenPoint” dönüştürülmüş bir koordinat noktasını alır ve onu dönüştürülmemiş bir noktaya dönüştürür.

  • “getMatrix”, toplu dönüşümleri bir matris dizisi biçiminde döndürür.

Kod:

var TransformationMatrix=( function(){
    // private
    var self;
    var m=[1,0,0,1,0,0];
    var reset=function(){ var m=[1,0,0,1,0,0]; }
    var multiply=function(mat){
        var m0=m[0]*mat[0]+m[2]*mat[1];
        var m1=m[1]*mat[0]+m[3]*mat[1];
        var m2=m[0]*mat[2]+m[2]*mat[3];
        var m3=m[1]*mat[2]+m[3]*mat[3];
        var m4=m[0]*mat[4]+m[2]*mat[5]+m[4];
        var m5=m[1]*mat[4]+m[3]*mat[5]+m[5];
        m=[m0,m1,m2,m3,m4,m5];
    }
    var screenPoint=function(transformedX,transformedY){
        // invert
        var d =1/(m[0]*m[3]-m[1]*m[2]);
        im=[ m[3]*d, -m[1]*d, -m[2]*d, m[0]*d, d*(m[2]*m[5]-m[3]*m[4]), d*(m[1]*m[4]-m[0]*m[5]) ];
        // point
        return({
            x:transformedX*im[0]+transformedY*im[2]+im[4],
            y:transformedX*im[1]+transformedY*im[3]+im[5]
        });
    }
    var transformedPoint=function(screenX,screenY){
        return({
            x:screenX*m[0] + screenY*m[2] + m[4],
            y:screenX*m[1] + screenY*m[3] + m[5]
        });    
    }
    // public
    function TransformationMatrix(){
        self=this;
    }
    // shared methods
    TransformationMatrix.prototype.translate=function(x,y){
        var mat=[ 1, 0, 0, 1, x, y ];
        multiply(mat);
    };
    TransformationMatrix.prototype.rotate=function(rAngle){
        var c = Math.cos(rAngle);
        var s = Math.sin(rAngle);
        var mat=[ c, s, -s, c, 0, 0 ];    
        multiply(mat);
    };
    TransformationMatrix.prototype.scale=function(x,y){
        var mat=[ x, 0, 0, y, 0, 0 ];        
        multiply(mat);
    };
    TransformationMatrix.prototype.skew=function(radianX,radianY){
        var mat=[ 1, Math.tan(radianY), Math.tan(radianX), 1, 0, 0 ];
        multiply(mat);
    };
    TransformationMatrix.prototype.reset=function(){
        reset();
    }
    TransformationMatrix.prototype.setContextTransform=function(ctx){
        ctx.setTransform(m[0],m[1],m[2],m[3],m[4],m[5]);
    }
    TransformationMatrix.prototype.resetContextTransform=function(ctx){
        ctx.setTransform(1,0,0,1,0,0);
    }
    TransformationMatrix.prototype.getTransformedPoint=function(screenX,screenY){
        return(transformedPoint(screenX,screenY));
    }
    TransformationMatrix.prototype.getScreenPoint=function(transformedX,transformedY){
        return(screenPoint(transformedX,transformedY));
    }
    TransformationMatrix.prototype.getMatrix=function(){
        var clone=[m[0],m[1],m[2],m[3],m[4],m[5]];
        return(clone);
    }
    // return public
    return(TransformationMatrix);
})();

    var m=rect.matrix;
    m.translate(100,0);
    m.scale(2,2);
    m.rotate(Math.PI/12);

    // use the rect's saved transformation matrix to reposition, 
    //     resize & redraw the rect
    ctx.strokeStyle='blue';
    drawTransformedRect(rect);

    // Demo: instructions
    ctx.font='14px arial';
    ctx.fillText('Demo: click inside the blue rect',30,200);

    // redraw a rect based on it's saved transformation matrix
    function drawTransformedRect(r){
        // set the context transformation matrix using the rect's saved matrix
        m.setContextTransform(ctx);
        // draw the rect (no position or size changes needed!)
        ctx.strokeRect( r.x, r.y, r.w, r.h );
        // reset the context transformation to default (==untransformed);
        m.resetContextTransform(ctx);
    }

    // is the point in the transformed rectangle?
    function isPointInTransformedRect(r,transformedX,transformedY){
        var p=r.matrix.getScreenPoint(transformedX,transformedY);
        var x=p.x;
        var y=p.y;
        return(x>r.x && x<r.x+r.w && y>r.y && y<r.y+r.h);
    } 

    // listen for mousedown events
    canvas.onmousedown=handleMouseDown;
    function handleMouseDown(e){
        // tell the browser we're handling this event
        e.preventDefault();
        e.stopPropagation();
        // get mouse position
        mouseX=parseInt(e.clientX-offsetX);
        mouseY=parseInt(e.clientY-offsetY);
        // is the mouse inside the transformed rect?
        if(isPointInTransformedRect(rect,mouseX,mouseY)){
            alert('You clicked in the transformed Rect');
        }
    }

    // Demo: redraw transformed rect without using
    //       context transformation commands
    function drawTransformedRect(r,color){
        var m=r.matrix;
        var tl=m.getTransformedPoint(r.x,r.y);
        var tr=m.getTransformedPoint(r.x+r.w,r.y);
        var br=m.getTransformedPoint(r.x+r.w,r.y+r.h);
        var bl=m.getTransformedPoint(r.x,r.y+r.h);
        ctx.beginPath();
        ctx.moveTo(tl.x,tl.y);
        ctx.lineTo(tr.x,tr.y);
        ctx.lineTo(br.x,br.y);
        ctx.lineTo(bl.x,bl.y);
        ctx.closePath();
        ctx.strokeStyle=color;
        ctx.stroke();
    }

}); // end window.onload
</script>
