Görüntüler

Tarayıcılar tarafından desteklenen çok çeşitli görüntü biçimleri vardır, ancak hiçbir tarayıcı hepsini desteklemez. Kullanmak istediğiniz belirli görüntü biçimleriniz varsa Wiki Tarayıcıları ve desteklenen görüntü biçimleri iyi bir genel bakış sağlar.

En iyi destek, tüm büyük tarayıcıların destek sağladığı “jpeg”, “png” ve “gif” olmak üzere 3 ana biçim içindir.

JPEG

JPEG görüntüler, fotoğraflara ve fotoğraf benzeri görüntülere en uygun olanıdır. Çizelgelere, diyagramlara ve metne pek uymazlar. JPEG görüntüleri saydamlığı desteklemez.

Canvas, “canvas.toDataURL” ve “canvas.toBlob” aracılığıyla JPEG görüntülerin çıktısını alabilir ve bir kalite ayarı sağlar. JPEG saydamlığı desteklemediğinden, son çıktı JPG için saydam pikseller siyahla karıştırılacaktır. Ortaya çıkan görüntü, tuvalin mükemmel bir kopyası olmayacaktır.

wikipedia’da JPEG

PNG

PNG Görüntüsü en yüksek kaliteli görüntülerdir ve saydam pikseller için bir alfa kanalı da içerebilir. Görüntü verileri sıkıştırılır ancak JPG görüntüleri gibi yapaylıklar oluşturmaz.

Kayıpsız sıkıştırma ve alfa kanalı desteği nedeniyle PNG’ler oyunlar, ui bileşen görüntüleri, çizelgeler, diyagramlar, metin için kullanılır. Bunları fotoğraf ve fotoğraf benzeri görüntüler için kullanırken dosya boyutları JPEG’lerden çok daha büyük olabilir. .

PNG formatı ayrıca tarayıcı desteği sınırlı olsa da animasyon desteği sağlar ve tuvalde kullanım için animasyona erişim yalnızca Javascript API’leri ve kitaplıkları aracılığıyla yapılabilir.

Çıktı formatı sıkıştırılmış 32Bit RGBA ile sınırlı olsa da tuval, PNG görüntülerini ‘canvas.toDataURL’ ve ‘canvas.toBlob’ aracılığıyla kodlamak için kullanılabilir. PNG, tuvalin piksel olarak mükemmel bir kopyasını sağlayacaktır.

wikipedia’da PNG

GIF

GIF’ler kısa animasyonlar için kullanılır, ancak aynı zamanda yüksek kaliteli çizelgeler, diyagramlar ve metin benzeri görüntüler sağlamak için de kullanılabilir. GIF’ler, çerçeve başına maksimum 256 renkle çok sınırlı renk desteğine sahiptir. Cleaver görüntü işleme ile gif görüntüleri, özellikle canlandırıldığında şaşırtıcı derecede iyi sonuçlar verebilir. Açık veya kapalı ile sınırlı olmasına rağmen, GIF’ler de şeffaflık sağlar

PNG’de olduğu gibi, GIF animasyonlarına tuval üzerinde kullanım için doğrudan erişilemez ve erişim elde etmek için bir Javascript API’sine veya kitaplığına ihtiyacınız olacaktır. GIF, tuval aracılığıyla kaydedilemez ve bunu yapmak için API veya kitaplık gerektirir.

Tuval üzerine temel video yükleme ve oynatma. #

Tuval, çeşitli kaynaklardan gelen videoyu görüntülemek için kullanılabilir. Bu örnek, bir videonun dosya kaynağı olarak nasıl yükleneceğini, görüntüleneceğini ve ekranda oynat/duraklat geçişine basit bir tıklamanın nasıl ekleneceğini gösterir.

Bu stackoverflow kendi kendine yanıtlanan soru HTML5 canvas etiketi kullanarak bir videoyu nasıl görüntülerim aşağıdaki örnek kodu çalışırken gösterir.

Sadece bir görüntü #

Bir video, tuval söz konusu olduğunda sadece bir görüntüdür. Herhangi bir resim gibi çizebilirsiniz. Fark, videonun oynatılabilmesi ve sesin olması.

Görüntü öğelerinin aksine, videoların tuvalde görüntülenmesi için tam olarak yüklenmesi gerekmez. Videolar ayrıca, videonun durumunu izlemek için kullanılabilecek bir dizi ekstra olay sağlar.

Bu durumda, videonun ne zaman oynatılmaya hazır olduğunu bilmek isteriz. “oncanplay”, videonun bir kısmını oynatmaya yetecek kadar yüklendiği, ancak sonuna kadar oynatılmaya yetmeyebileceği anlamına gelir.

video.oncanplay = readyToPlayVideo; // set the event to the play function that 
                                  // can be found below

Alternatif olarak, sonuna kadar oynatılabilmesi için yeterli video yüklendiğinde patlayacak olan “oncanplaythrough"u kullanabilirsiniz.

video.oncanplaythrough = readyToPlayVideo; // set the event to the play function that
                                         // can be found below

Görüntüleniyor #

Video, tuval üzerinde kendi kendine oynatılmayacak. Her yeni çerçeve için çizmeniz gerekir. Tam kare hızını ve ne zaman oluştuğunu bilmek zor olduğu için en iyi yaklaşım videoyu 60 fps’de çalışıyormuş gibi göstermektir. Kare hızı daha düşükse, aynı kareyi iki kez oluşturmanız yeterlidir. Kare hızı daha yüksekse, ekstra kareleri görmek için yapılabilecek hiçbir şey yoktur, bu yüzden onları görmezden geliriz.

Video öğesi yalnızca bir görüntü öğesidir ve herhangi bir görüntü gibi çizilebilir, videoyu ölçekleyebilir, döndürebilir, kaydırabilir, yansıtabilir, soldurabilir, kırpabilir ve yalnızca parçaları görüntüleyebilir, global bir bileşik modla ikinci kez iki kez çizebilirsiniz. aydınlatmak, ekran vb. gibi FX eklemek için.

function updateCanvas(){
    ctx.clearRect(0,0,canvas.width,canvas.height); // Though not always needed 
                                                     // you may get bad pixels from 
                                                     // previous videos so clear to be
                                                     // safe
    // only draw if loaded and ready
    if(videoContainer !== undefined && videoContainer.ready){ 
        // find the top left of the video on the canvas
        var scale = videoContainer.scale;
        var vidH = videoContainer.video.videoHeight;
        var vidW = videoContainer.video.videoWidth;
        var top = canvas.height / 2 - (vidH /2 ) * scale;
        var left = canvas.width / 2 - (vidW /2 ) * scale;
        // now just draw the video the correct size
        ctx.drawImage(videoContainer.video, left, top, vidW * scale, vidH * scale);
        if(videoContainer.video.paused){ // if not playing show the paused screen 
            drawPayIcon();
        }
    }
    // all done for display 
    // request the next frame in 1/60th of a second
    requestAnimationFrame(updateCanvas);
}

Temel oynatma duraklatma kontrolü #

Şimdi videoyu yükledik ve görüntüledik, tek ihtiyacımız olan oynatma kontrolü. Ekranda bir tıklama geçiş oyunu olarak yapacağız. Video oynatılırken ve kullanıcı tıkladığında video duraklatılır. Duraklatıldığında, tıklama oynatmaya devam eder. Videoyu karartmak ve bir oynatma simgesi (üçgen) çizmek için bir işlev ekleyeceğiz

function drawPayIcon(){
     ctx.fillStyle = "black";  // darken display
     ctx.globalAlpha = 0.5;
     ctx.fillRect(0,0,canvas.width,canvas.height);
     ctx.fillStyle = "#DDD"; // colour of play icon
     ctx.globalAlpha = 0.75; // partly transparent
     ctx.beginPath(); // create the path for the icon
     var size = (canvas.height / 2) * 0.5;  // the size of the icon
     ctx.moveTo(canvas.width/2 + size/2, canvas.height / 2); // start at the pointy end
     ctx.lineTo(canvas.width/2 - size/2, canvas.height / 2 + size);
     ctx.lineTo(canvas.width/2 - size/2, canvas.height / 2 - size);
     ctx.closePath();
     ctx.fill();
     ctx.globalAlpha = 1; // restore alpha
}

Şimdi oynatma duraklatma olayı #

function playPauseClick(){
     if(videoContainer !== undefined && videoContainer.ready){
          if(videoContainer.video.paused){                                 
                videoContainer.video.play();
          }else{
                videoContainer.video.pause();
          }
     }
}
// register the event
canvas.addEventListener("click",playPauseClick);

Özet #

Tuvali kullanarak video oynatmak çok kolaydır, gerçek zamanlı efekt eklemek de kolaydır. Ancak formatlar, nasıl oynayabileceğiniz ve arayabileceğiniz konusunda bazı sınırlamalar vardır. MDN HTMLMediaElement, video nesnesine tam başvurunun alınacağı yerdir.

Resim tuval üzerine çizildikten sonra içerdiği piksellere erişmek için ctx.getImageData’yı kullanabilirsiniz. Ya da bir fotoğraf çekip indirmek için canvas.toDataURL kullanabilirsiniz. (Yalnızca video güvenilir bir kaynaktan geliyorsa ve tuvali lekelemiyorsa).

Videonun sesi varsa, oynatmanın sesi de oynayacağını unutmayın.

Resim bir HTMLImageElement

Image.src özelliği

“src” resmi, herhangi bir geçerli resim URL’si veya kodlanmış dataURL olabilir. Görüntü biçimleri ve destek hakkında daha fazla bilgi için bu konunun Açıklamalar bölümüne bakın.

  • image.src = "http://my.domain.com/images/myImage.jpg"
  • image.src = "data:image/gif;base64,R0lGODlhAQABAIAAAAUEBAAAACwAAAAAAQABAAAACAkQBADs=" *

*DataURL, siyah içeren 1’e 1 piksellik bir gif resmidir

Yükleme ve hatalarla ilgili açıklamalar

src özelliği ayarlandığında resim yüklenmeye başlayacaktır. Yükleme eşzamanlıdır, ancak işlev veya koddan çıkılana/geri dönene kadar ‘onload’ olayı çağrılmayacaktır.

Sayfadan bir resim alırsanız (örneğin document.getElementById("myImage")) ve onun src değeri ayarlanmışsa, yüklenmiş veya yüklenmemiş olabilir. Tamamlandığında ’true’ olacak olan ‘HTMLImageElement.complete’ ile görüntünün durumunu kontrol edebilirsiniz. Bu, görüntünün yüklendiği anlamına gelmez, ikisinden birine sahip olduğu anlamına gelir.

  • yüklendi
  • Bir hata oluştu
  • src özelliği ayarlanmadı ve boş "" Dizesine eşit

Görüntü güvenilir olmayan bir kaynaktan geliyorsa ve çeşitli nedenlerle erişilemiyorsa, bir hata olayı oluşturacaktır. Bu olduğunda, görüntü bozuk bir durumda olacaktır. Daha sonra tuval üzerine çizmeye çalışırsanız, aşağıdaki hatayı verir.

Uncaught DOMException: Failed to execute 'drawImage' on 'CanvasRenderingContext2D': The HTMLImageElement provided is in the 'broken' state.

image.onerror = myImgErrorHandler olayını sağlayarak hataları önlemek için uygun eylemi gerçekleştirebilirsiniz.

                return new Uint8Array([num>>24, num>>16, num>>8, num])
            }            
        }
    }
    const ids = { // header names and values
        videoData          : 0x1a45dfa3, 
        Version            : 0x4286,
        ReadVersion        : 0x42f7,

        MaxSizeLength      : 0x42f3,

    function getHeader(){
        header[3][2][3] = name;
        header[3][2][5] = name;
        header[3][2][7] =  dataTypes.double2Str(frameDelay);
        header[3][3][1][15][1] =  width;
        header[3][3][1][15][3] =  height;
        function create(dat){
            var i,kv,data;
            data = [];
            for(i = 0; i < dat.length; i += 2){
                kv = {i : dat[i]};
                if(Array.isArray(dat[i + 1])){
                    kv.d = create(dat[i + 1]);
                }else{
                    kv.d = dat[i + 1];
                }
                data.push(kv);
            }
            return data;
        }
        return create(header);
    }
    function addCluster(){
        webmData[videoDataPos].d.push({ i: ids.Cluster,d: [{ i: ids.Timecode, d: Math.round(clusterTimecode)}]}); // Fixed bug with Round
        clusterCounter = 0;
    }
    function addFrame(frame){
        var VP8, kfS,riff;
        riff = getWebPChunks(atob(frame.toDataURL(frameMimeType, quality).slice(23)));
        VP8 = riff.RIFF[0].WEBP[0];
        kfS = VP8.indexOf(keyframeD64Header) + 3;
        frame = {
            width: ((VP8.charCodeAt(kfS + 1) << 8) | VP8.charCodeAt(kfS)) & 0x3FFF,
            height: ((VP8.charCodeAt(kfS + 3) << 8) | VP8.charCodeAt(kfS + 2)) & 0x3FFF,
            data: VP8,
            riff: riff
        };
        if(clusterCounter > CLUSTER_MAX_DURATION){
            addCluster();            
        }
        webmData[videoDataPos].d[webmData[videoDataPos].d.length-1].d.push({
            i: ids.Frame, 
            d: S(ids.FrameBlock) + S( Math.round(clusterCounter) >> 8) +  S( Math.round(clusterCounter) & 0xff) + S(128) + frame.data.slice(4),
        });
        clusterCounter += frameDelay;        
        clusterTimecode += frameDelay;
        webmData[videoDataPos].d[0].d[3].d = dataTypes.double2Str(clusterTimecode);
    }
    function startEncoding(){
        frameNumber = clusterCounter = clusterTimecode = 0;
        webmData  = getHeader();
        addCluster();
    }    
    function toBlob(vidData){
        var data,i,vData, len;
        vData = [];
        for(i = 0; i < vidData.length; i++){
            data = dataTypes[typeof vidData[i].d](vidData[i].d);
            len  = data.size || data.byteLength || data.length;
            vData.push(stream.num(vidData[i].i));
            vData.push(stream.compInt(len));
            vData.push(data)
        }
        return new Blob(vData, {type: videoMimeType});
    }
    function getWebPChunks(str){
        var offset, chunks, id, len, data;
        offset = 0;
        chunks = {};
        while (offset < str.length) {
            id = str.substr(offset, 4);
            // value will have top bit on (bit 32) so not simply a bitwise operation
            // Warning little endian (Will not work on big endian systems)
            len = new Uint32Array(
                new Uint8Array([
                    str.charCodeAt(offset + 7),
                    str.charCodeAt(offset + 6),
                    str.charCodeAt(offset + 5),
                    str.charCodeAt(offset + 4)
                ]).buffer)[0];
            id = str.substr(offset, 4);
            chunks[id] = chunks[id] === undefined ? [] : chunks[id];
            if (id === 'RIFF' || id === 'LIST') {
                chunks[id].push(getWebPChunks(str.substr(offset + 8, len)));
                offset += 8 + len;
            } else if (id === 'WEBP') {
                chunks[id].push(str.substr(offset + 8));
                break;
            } else {
                chunks[id].push(str.substr(offset + 4));
                break;
            }
        }
        return chunks;
    }
    function Encoder(fps, _quality = 0.8, _name = "Groover"){ 
        this.fps = fps;
        this.quality = quality = _quality;
        this.frameDelay = frameDelay = 1000 / fps;
        this.frame = 0;
        this.width = width = null;
        this.timecode = 0;
        this.name = name = _name;
    }
    Encoder.prototype = {
        addFrame : function(frame){
            if('canvas' in frame){
                frame = frame.canvas;    
            }
            if(width === null){
                this.width = width = frame.width,
                this.height = height = frame.height
                startEncoding();
            }else
            if(width !== frame.width || height !== frame.height){
                throw RangeError("Frame size error. Frames must be the same size.");
            }            
            addFrame(frame);   
            this.frame += 1;
            this.timecode = clusterTimecode;
        },        
        toBlob : function(){
            return toBlob(webmData);
        }
    }
    return {
        Video: Encoder,
    }
