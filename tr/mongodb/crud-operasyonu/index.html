Sözdizimi #

  • insert(belge veya belge dizisi)
  • insertBir( ‘belge’, { writeConcern: ‘document’ } )
  • insertMany( { [ belge 1 , belge 2, … ] }, { writeConcern: document, ordered: boolean } )
  • bul(sorgu, projeksiyon)
  • findOne(sorgu, projeksiyon)
  • güncelleme(sorgu, güncelleme)
  • güncellemeBir( sorgu, Güncelleme, { upsert: boolean, writeConcern: document } )
  • güncellemeMany( sorgu, Güncelleme, { upsert: boolean, writeConcern: document } )
  • birini değiştir( sorgu, yenisiyle değiştirme, { upsert: boolean, writeConcern: document } )
  • kaldır(sorgu, justOne)
  • findAndModify(sorgu, sort, güncelleme, seçenekler[isteğe bağlı])

Bir belgenin güncellenmesi ve silinmesi dikkatli bir şekilde yapılmalıdır. İşlem birden fazla belgeyi etkileyebileceğinden.

Oluşturmak #

db.people.insert({name: 'Tom', age: 28});

Veya

db.people.save({name: 'Tom', age: 28});

save ile olan fark, geçirilen belge bir _id alanı içeriyorsa, bu _id ile zaten bir belge mevcutsa, yeni olarak eklenmek yerine güncellenecektir.

MongoDB 3.2.x’te bir koleksiyona belge eklemek için iki yeni yöntem: -

Yalnızca bir kayıt eklemek için ‘insertOne’ı kullanın: -

db.people.insertOne({name: 'Tom', age: 28});

Birden çok kayıt eklemek için “insertMany"yi kullanın: -

db.people.insertMany([{name: 'Tom', age: 28},{name: 'John', age: 25}, {name: 'Kathy', age: 23}])

Sürüm 3.0’dan bu yana her resmi dil sürücüsünde “insert” öğesinin kullanımdan kaldırıldığına dikkat edin. Tam ayrım, kabuk yöntemlerinin, yöntemin uygulanmasında diğer sürücülerin gerisinde kalmasıdır. Aynı şey diğer tüm CRUD yöntemleri için de geçerlidir.

Güncelleme #

tüm nesneyi güncelleyin:

db.people.update({name: 'Tom'}, {age: 29, name: 'Tom'})

// New in MongoDB 3.2
db.people.updateOne({name: 'Tom'},{age: 29, name: 'Tom'}) //Will replace only first matching document.

db.people.updateMany({name: 'Tom'},{age: 29, name: 'Tom'}) //Will replace all matching documents.

Veya bir belgenin tek bir alanını güncelleyin. Bu durumda “yaş”:

db.people.update({name: 'Tom'}, {$set: {age: 29}})

Ayrıca üçüncü bir parametre ekleyerek birden çok belgeyi aynı anda güncelleyebilirsiniz. Bu sorgu, adın “Tom"a eşit olduğu tüm belgeleri güncelleyecektir:

db.people.update({name: 'Tom'}, {$set: {age: 29}}, {multi: true})

// New in MongoDB 3.2
db.people.updateOne({name: 'Tom'},{$set:{age: 30}) //Will update only first matching document.

db.people.updateMany({name: 'Tom'},{$set:{age: 30}}) //Will update all matching documents.

Güncelleme için yeni bir alan geliyorsa, o alan belgeye eklenecektir.

db.people.updateMany({name: 'Tom'},{$set:{age: 30, salary:50000}})// Document will have `salary` field as well.

Belgenin değiştirilmesi gerekiyorsa,

db.collection.replaceOne({name:'Tom'}, {name:'Lakmal',age:25,address:'Sri Lanka'})

kullanılabilir.

Not: Nesneyi tanımlamak için kullandığınız alanlar güncellenen belgeye kaydedilecektir. Güncelleme bölümünde tanımlı olmayan alanlar belgeden kaldırılacaktır.

Silmek #

Sorgu parametresiyle eşleşen tüm belgeleri siler:

// New in MongoDB 3.2
db.people.deleteMany({name: 'Tom'})

// All versions
db.people.remove({name: 'Tom'})

Ya da sadece bir

// New in MongoDB 3.2
db.people.deleteOne({name: 'Tom'})

// All versions
db.people.remove({name: 'Tom'}, true)

MongoDB’nin remove() yöntemi. Bu komutu herhangi bir argüman olmadan veya boş argüman olmadan çalıştırırsanız, tüm belgeleri koleksiyondan kaldıracaktır.

db.people.remove();

veya

db.people.remove({});

Okumak #

“Tom” değerine sahip bir “ad” alanına sahip “kişiler” koleksiyonundaki tüm belgeleri sorgulayın:

db.people.find({name: 'Tom'})

Veya sadece ilki:

db.people.findOne({name: 'Tom'})

Bir alan seçimi parametresi ileterek hangi alanların döndürüleceğini de belirtebilirsiniz. Aşağıdakiler “_id” alanını hariç tutar ve yalnızca “yaş” alanını içerir:

db.people.find({name: 'Tom'}, {_id: 0, age: 1})

Not: Varsayılan olarak, siz istemeseniz bile “_id” alanı döndürülecektir. ‘_id’yi geri almak istemiyorsanız, önceki örneği takip edebilir ve ‘_id:0’ (veya ‘_id: false’) belirterek ‘_id’nin hariç tutulmasını isteyebilirsiniz. adres nesnesinin ülke, şehir vb. içerdiği gibi alt kaydı bulun.

db.people.find({'address.country': 'US'})

& gerekirse alanı da belirtin

db.people.find({'address.country': 'US'}, {'name': true, 'address.city': true})Remember that the result has a `.pretty()` method that pretty-prints resulting JSON:

db.people.find().pretty()

Daha fazla güncelleme operatörü #

Bir belgeyi güncellerken $set dışında başka operatörleri de kullanabilirsiniz. $push operatörü, bir diziye bir değer göndermenize izin verir, bu durumda nicknames dizisine yeni bir takma ad ekleyeceğiz.

db.people.update({name: 'Tom'}, {$push: {nicknames: 'Tommy'}})
// This adds the string 'Tommy' into the nicknames array in Tom's document.

‘$pull’ operatörü ‘$push’un tersidir, dizilerden belirli öğeleri çekebilirsiniz.

db.people.update({name: 'Tom'}, {$pull: {nicknames: 'Tommy'}})
// This removes the string 'Tommy' from the nicknames array in Tom's document.

$pop operatörü, bir diziden ilk veya son değeri kaldırmanıza izin verir. Diyelim ki Tom’un belgesinde ‘[‘Marie’, ‘Bob’, ‘Kevin’, ‘Alex’]’ değerine sahip kardeşler adı verilen bir özellik var.

db.people.update({name: 'Tom'}, {$pop: {siblings: -1}})
// This will remove the first value from the siblings array, which is 'Marie' in this case.


db.people.update({name: 'Tom'}, {$pop: {siblings: 1}})
// This will remove the last value from the siblings array, which is 'Alex' in this case.

Birden çok belgeyi güncellerken “çoklu” Parametre #

Bir koleksiyondaki birden çok belgeyi güncellemek için çoklu seçeneği true olarak ayarlayın.

db.collection.update(
   query,
   update,
   {
     upsert: boolean,
     multi: boolean,
     writeConcern: document
   }
)

çoklu isteğe bağlıdır. Doğru olarak ayarlanırsa, sorgu ölçütlerini karşılayan birden çok belgeyi günceller. Yanlış olarak ayarlanırsa bir belgeyi günceller. Varsayılan değer yanlıştır.

db.mycol.find() { “_id”: ObjectId(598354878df45ec5), “title”:“MongoDB’ye Genel Bakış”} { “_id”: ObjectId(59835487adf45ec6), “title”:“NoSQL’e Genel Bakış”} { “_id” : ObjectId(59835487adf45ec7), “title”:“Öğreticilere Genel Bakış”}

db.mycol.update({’title’:‘MongoDB’ye Genel Bakış’}, {$set:{’title’:‘Yeni MongoDB Eğitimi’}},{multi:true})

Gömülü belgelerin güncellenmesi. #

Aşağıdaki şema için:

{name: 'Tom', age: 28, marks: [50, 60, 70]}

İşaretlerin 50 olduğu yerde Tom’un işaretlerini 55’e güncelleyin (Konum operatörünü $ kullanın):

db.people.update({name: "Tom", marks: 50}, {"$set": {"marks.$": 55}})

Aşağıdaki şema için:

{name: 'Tom', age: 28, marks: [{subject: "English", marks: 90},{subject: "Maths", marks: 100}, {subject: "Computes", marks: 20}]}

Tom’un İngilizce notlarını 85 olarak güncelleyin:

db.people.update({name: "Tom", "marks.subject": "English"},{"$set":{"marks.$.marks": 85}})

Yukarıdaki örneği açıklamak:

{name: “Tom”, “marks.subject”: “English”} kullanarak nesnenin konumunu, konunun İngilizce olduğu işaretler dizisinde alırsınız. “marks.$.marks” içinde, mark dizisinin bu konumunda güncellemek için $ kullanılır

Bir Dizideki Değerleri Güncelleme

Konumsal $ operatörü, dizideki öğenin konumunu açıkça belirtmeden güncellenecek bir dizideki bir öğeyi tanımlar.

Aşağıdaki belgelere sahip bir koleksiyon öğrencisi düşünün:

{ "_id" : 1, "grades" : [ 80, 85, 90 ] }
{ "_id" : 2, "grades" : [ 88, 90, 92 ] }
{ "_id" : 3, "grades" : [ 85, 100, 90 ] }

İlk belgedeki notlar dizisini 80’den 82’ye güncellemek için, öğenin dizideki konumunu bilmiyorsanız konumsal $ operatörünü kullanın:

db.students.update(
   { _id: 1, grades: 80 },
   { $set: { "grades.$" : 82 } }
)
