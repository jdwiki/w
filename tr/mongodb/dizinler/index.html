Sözdizimi #

  • db.collection.createIndex({ <string field> : <1|-1 order> [, <string field> : <1|-1 order>] });

Performans Etkisi: Dizinlerin okuma performanslarını iyileştirdiğini, ancak bir belge eklemek için tüm dizinlerin güncellenmesini gerektirdiğinden yazma performansı üzerinde kötü bir etkisi olabileceğini unutmayın.

Dizin Bırakma/Silme #

Dizin adı biliniyorsa,

db.collection.dropIndex('name_of_index');

Dizin adı bilinmiyorsa,

db.collection.dropIndex( { 'name_of_field' : -1 } );

Seyrek dizinler ve Kısmi dizinler #

** Seyrek dizinler:**

Bunlar, isteğe bağlı olan ancak aynı zamanda benzersiz olması gereken alanlar için özellikle yararlı olabilir.

{ "_id" : "john@example.com", "nickname" : "Johnnie" }
{ "_id" : "jane@example.com" }
{ "_id" : "julia@example.com", "nickname" : "Jules"}
{ "_id" : "jack@example.com" }

İki girişin belirtilen bir “takma adı” olmadığından ve dizin oluşturma belirtilmemiş alanları boş olarak değerlendireceğinden, dizin oluşturma işlemi ‘boş’ olan 2 belgeyle başarısız olur, bu nedenle:

db.scores.createIndex( { nickname: 1 } , { unique: true, sparse: true } )

hala ‘boş’ takma adlara sahip olmanıza izin verecek.

Seyrek dizinler, o alanı belirtmeyen belgeleri atladıkları/yok saydıkları için daha kompakttır. Bu nedenle, belgelerin yalnızca %10’undan daha azının bu alanı belirttiği bir koleksiyonunuz varsa, çok daha küçük dizinler oluşturabilirsiniz - aşağıdaki gibi sorgular yapmak istiyorsanız sınırlı belleği daha iyi kullanarak:

db.scores.find({'nickname': 'Johnnie'})

Kısmi dizinler:

Kısmi dizinler, tarafından sunulan işlevselliğin bir üst kümesini temsil eder. seyrek dizinler ve seyrek dizinler yerine tercih edilmelidir. (3.2 sürümünde yeni)

Kısmi dizinler, belirtilen filtreye dayalı olarak dizin girişlerini belirler.

db.restaurants.createIndex(
  { cuisine: 1 },
  { partialFilterExpression: { rating: { $gt: 5 } } }
)

“Derecelendirme” 5’ten büyükse, “mutfak” dizine eklenir. Evet, diğer özelliklerin değerine göre de indekslenecek bir özellik belirtebiliriz.

** Seyrek ve Kısmi dizinler arasındaki fark:**

Seyrek indeksler, indekslenecek belgeleri yalnızca indekslenen alanın varlığına veya bileşik indeksler için indekslenen alanların varlığına göre seçer.

Kısmi dizinler, belirtilen filtreye dayalı olarak dizin girişlerini belirler. Filtre, dizin anahtarları dışındaki alanları içerebilir ve yalnızca bir mevcudiyet kontrolünden başka koşullar belirtebilir.

Yine de kısmi bir dizin, seyrek bir dizinle aynı davranışı uygulayabilir.

Örneğin:

db.contacts.createIndex(
   { name: 1 },
   { partialFilterExpression: { name: { $exists: true } } }
)

Not: Hem partialFilterExpression seçeneği hem de sparse seçeneği aynı anda belirtilemez.

Birleştirmek #

db.people.createIndex({name: 1, age: -1})

Bu, birden çok alanda, bu durumda “ad” ve “yaş” alanlarında bir dizin oluşturur. “Ad"da artan ve “yaş"ta azalan olacaktır.

Bu indeks türünde, sıralama düzeni önemlidir, çünkü indeksin bir sıralama işlemini destekleyip destekleyemeyeceğini belirleyecektir. Sıralama, sıralamadaki anahtarların tümü için ters sıralama yönünde olduğu sürece, bir bileşik dizinin herhangi bir önekinde ters sıralama desteklenir. Aksi takdirde, bileşik dizinler için sıralamanın dizinin sırasına uygun olması gerekir.

Alan sırası da önemlidir, bu durumda dizin önce “ad"a göre ve her ad değeri içinde “yaş” alanının değerlerine göre sıralanır. Bu, dizinin “ad” alanındaki veya “ad” ve “yaş” üzerindeki sorgular tarafından kullanılmasına izin verir, ancak yalnızca “yaş” üzerinde kullanılamaz.

Benzersiz Dizin #

db.collection.createIndex( { "user_id": 1 }, { unique: true } )

tanımlanan dizinde (tek veya bileşik) benzersizliği zorunlu kılar. Koleksiyon zaten yinelenen değerler içeriyorsa, dizin oluşturma başarısız olur; “sparse: true” belirtilmediği sürece (hepsi “null” değeriyle dizine ekleneceğinden) birden fazla giriş alanı eksik olduğunda da dizin oluşturma başarısız olur.

Dizin Oluşturma Temelleri #

Aşağıdaki işlem koleksiyonuna bakın.

> db.transactions.insert({ cr_dr : "D", amount : 100, fee : 2});
> db.transactions.insert({ cr_dr : "C", amount : 100, fee : 2});
> db.transactions.insert({ cr_dr : "C", amount : 10,  fee : 2});
> db.transactions.insert({ cr_dr : "D", amount : 100, fee : 4});
> db.transactions.insert({ cr_dr : "D", amount : 10,  fee : 2});
> db.transactions.insert({ cr_dr : "C", amount : 10,  fee : 4});
> db.transactions.insert({ cr_dr : "D", amount : 100, fee : 2});

getIndexes() işlevleri, bir koleksiyon için mevcut olan tüm indeksleri gösterecektir.

db.transactions.getIndexes();

Yukarıdaki ifadenin çıktısını görelim.

[
    {
        "v" : 1,
        "key" : {
            "_id" : 1
        },
        "name" : "_id_",
        "ns" : "documentation_db.transactions"
    }
]

İşlem toplama için zaten bir dizin var. Bunun nedeni, MongoDB’nin bir koleksiyonun oluşturulması sırasında ‘_id’ alanında benzersiz bir dizin oluşturmasıdır. “_id” dizini, istemcilerin “_id” alanı için aynı değere sahip iki belge eklemesini engeller. Bu dizini “_id” alanına bırakamazsınız.

Şimdi cr_dr alanı için bir indeks ekleyelim;

db.transactions.createIndex({ cr_dr : 1 });

İndeks yürütmenin sonucu aşağıdaki gibidir.

{
    "createdCollectionAutomatically" : false,
    "numIndexesBefore" : 1,
    "numIndexesAfter" : 2,
    "ok" : 1
}

CreateCollectionOtomatik olarak işlemin oluşturulup oluşturulmadığını gösterir. bir koleksiyon. Bir koleksiyon yoksa, MongoDB onu oluşturur. indeksleme işleminin bir parçası olarak toplama.

Tekrar db.transactions.getIndexes(); komutunu çalıştıralım.

[
    {
        "v" : 1,
        "key" : {
            "_id" : 1
        },
        "name" : "_id_",
        "ns" : "documentation_db.transactions"
    },
    {
        "v" : 1,
        "key" : {
            "cr_dr" : 1
        },
        "name" : "cr_dr_1",
        "ns" : "documentation_db.transactions"
    }
]

Şimdi işlem koleksiyonunun iki indeksi olduğunu görüyorsunuz. Varsayılan ‘_id’ dizini ve oluşturduğumuz ‘cr_dr_1’. İsim MongoDB tarafından atanır. Aşağıdaki gibi kendi adınızı belirleyebilirsiniz.

db.transactions.createIndex({ cr_dr : -1 },{name : "index on cr_dr desc"})

Şimdi db.transactions.getIndexes(); size üç indeks verecektir.

[
    {
        "v" : 1,
        "key" : {
            "_id" : 1
        },
        "name" : "_id_",
        "ns" : "documentation_db.transactions"
    },
    {
        "v" : 1,
        "key" : {
            "cr_dr" : 1
        },
        "name" : "cr_dr_1",
        "ns" : "documentation_db.transactions"
    },
    {
        "v" : 1,
        "key" : {
            "cr_dr" : -1
        },
        "name" : "index on cr_dr desc",
        "ns" : "documentation_db.transactions"
    }
]

{ cr_dr : -1 } indeksini oluştururken 1, indeksin artan sırada ve -1 ise azalan sırada olacağı anlamına gelir.

Karma dizinler #

İndeksler ayrıca karma olarak da tanımlanabilir. Bu, eşitlik sorgularında daha performanslıdır, ancak aralık sorguları için verimli değildir; ancak aynı alanda hem karma hem de artan/azalan dizinleri tanımlayabilirsiniz.

> db.transactions.createIndex({ cr_dr : "hashed" });    

> db.transactions.getIndexes(
[
    {
        "v" : 1,
        "key" : {
            "_id" : 1
        },
        "name" : "_id_",
        "ns" : "documentation_db.transactions"
    },
    {
        "v" : 1,
        "key" : {
            "cr_dr" : "hashed"
        },
        "name" : "cr_dr_hashed",
        "ns" : "documentation_db.transactions"
    }
]
