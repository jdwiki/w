Farklı Cihazlarda Sayfa Düzeni - CSS #

Uygulamanız farklı cihazlarda çalışacaksa, cihaz boyutuna bağlı olarak farklı ViewPort’lara işlenmesi gerekecektir. Bununla iki şekilde başa çıkabilirsiniz: javascript kuralları veya CSS medya stilleri ile. Angular veya Ember (veya bu konuda Blaze) gibi bir MVC veya MVVM kitaplığı kullanıyorsanız ve yalnızca tek bir cihazı veya donanım platformunu hedefliyorsanız, MVC modelinizi farklı donanım ViewPort’ları olarak yeniden düşünmeniz gerekebilir. uygulamanıza tanıtıldı.

// desktop 
@media only screen and (min-width: 960px) {
}

// landscape orientation
@media only screen and (min-width: 768px) {
}

// portrait orientation
@media only screen and (min-width: 480px) {
}

Stilleri 768 pikselde mi (portre modu) yoksa 1024 pikselde mi (manzara) kırmak isteyip istemediğinize karar vermeniz gerekecek. Bu, hedef mobil cihazınızın 3:4 oranını kullanan iPad olduğunu varsayar. Aksi takdirde, hedeflemek istediğiniz cihazların en boy oranlarını hesaplamanız ve oradan eşik seviyelerini bulmanız gerekir.

Sabit Boyutlu Pencereler #

Farklı mobil cihazlar için sabit boyutlu ekranlara sahip düzenler tasarlayacaksanız, uygulamanızı bir masaüstünde çalıştırırken bu tasarımı yansıtmak isteyebilirsiniz. Aşağıdaki yöntem, PhoneGap’in DIŞINDA penceresinin boyutunu düzelterek masaüstünde sabit boyutlu bir pencere verir. Bazen seçenekleri sınırlayarak kullanıcının beklentilerini ve UI tasarımını yönetmek en kolayıdır!

// create a window of a specific size
var w=window.open('','', 'width=100,height=100');
w.resizeTo(500,500);

// prevent window resize
var size = [window.width,window.height];  //public variable
$(window).resize(function(){
    window.resizeTo(size[0],size[1]);
});

Çevrimdışı Önbelleğe Alma #

Tüm bunların işe yaraması için, muhtemelen uygulama verilerini ve kullanıcı verilerini önbelleğe almak anlamına gelen çevrimdışı desteğe ihtiyacınız olacak.

meteor add appcache
meteor add grounddb

Kaydırma-Sıçramayı Devre Dışı Bırak #

Masaüstü uygulamalarında, uygulamanıza daha yerel bir his vermek için kaydırmayı devre dışı bırakmak isteyebilirsiniz. Bunu, tarayıcının DOM’yi nasıl kontrol ettiğini devre dışı bırakarak javascript ile yapabilirsiniz:

// prevent scrolling on the whole page
// this is not meteorish; TODO: translate to meteor-centric code
document.ontouchmove = function(e) {e.preventDefault()};

// prevent scrolling on specific elements
// this is not meteorish; TODO: translate to meteor-centric code
scrollableDiv.ontouchmove = function(e) {e.stopPropagation()};

Alternatif olarak, css ile taşma ve kaydırma stillerini kullanabilirsiniz.

#appBody {
  overflow: hidden;
}

#contentContainer {
  .content-scrollable {
    overflow-y: auto;
    -webkit-overflow-scrolling: touch;
  }
}

Yukarıdakilerin çalışması için gereken nesne modeli şuna benzer:

<div id="appBody">
  <div id="contentContainer">
    <div class="content-scrollable">
      <!-- content -->
    </div>
  </div>
</div>

Çoklu Dokunma ve Hareketler #

Mobil cihazlarda genellikle klavye yoktur, bu nedenle uygulamanıza bazı dokunsal denetleyiciler eklemeniz gerekir. İnsanların kullandığı görünen iki popüler paket FastClick ve Hammer. Kurulum kolaydır.

meteor add fastclick
meteor add hammer:hammer

FastClick neredeyse hiç yapılandırma gerektirmezken, Hammer’ın kablo bağlantısı biraz çalışma gerektirir. Todos uygulamasındaki kononik örnek şöyle görünür:

Template.appBody.onRendered(function() {
  if (Meteor.isCordova) {
    // set up a swipe left / right handler
    this.hammer = new Hammer(this.find('#appBody'));
    this.hammer.on('swipeleft swiperight', function(event) {
      if (event.gesture.direction === 'right') {
        Session.set(MENU_KEY, true);
      } else if (event.gesture.direction === 'left') {
        Session.set(MENU_KEY, false);
      }
    });
  }
});

Meteor Cordova Mimari Boru Hattı #

Şimdi Meteor Cordova Phonegap Entegrasyonu belgelerini gözden geçirme zamanı.

Bu belgeler yazıldığından beri, kurulumda bazı aksaklıklara neden olan XCode ve Yosemite yayınlandı. Meteor’un bir iOS cihazına derlenmesini sağlamak için geçmemiz gereken adımlar.

  • Yosemite’e yükseltin.
  • XCode’u silin (Uygulamalar klasöründen Çöp Kutusu’na sürükleyin)
  • Uygulama mağazasından XCode 6.1’i yükleyin.
  • Çeşitli hüküm ve koşulları kabul edin.
# 5.  clone and rebuild the ios-sim locally
#     (this step will not be needed in future releases)
git clone https://github.com/phonegap/ios-sim.git
cd ios-sim
rake build

# 6.  make sure we can update the .meteor/packages locations
#     (this step will not be needed in future releases)
sudo chmod -R 777 ~/.meteor/packages

# 7.  copy the new build into Meteor locations
#     (this step will not be needed in future releases)
for i in `find ~/.meteor/packages/meteor-tool/ -name ios-sim -type f`; do
  cp -R ./build/Release/ios-sim "$i"
done

# 8.  install the ios platform to your app
cd myapp
meteor list-platforms
meteor add-platform ios
meteor list-platforms

# 9.  and that there aren't dead processes
ps -ax 
kill -9 <pid>
# /Users/abigailwatson/.meteor/packages/meteor-tool/.1.0.35.wql4jh++os.osx.x86_64+web.browser+web.cordova/meteor-tool-os.osx.x86_64/dev_bundle/mongodb/bin/mongod
# tail -f /Users/abigailwatson/Code/Medstar/dart/webapp/.meteor/local/cordova-build/platforms/ios/cordova/console.log

# 10.  make sure there are correct permissions on the application (important!)
sudo chmod -R 777 .meteor/local/

# 11.  run app
meteor run ios

# 12.  if that doesn't work, clear the directory
sudo rm -rf .meteor/local

# 13a.  run meteor again to create the default browser build
meteor

# 13b.  run it a second time so bootstrap and other packages get downloaded into the browser build
ctrl-x
meteor

# 14.  then run the ios version
ctrl-x
meteor run ios

İşlem sırasında XCode başlatılmalıdır. Simülatörünüzü seçin ve ‘Oynat’ düğmesine basın.

IOS Geliştirme #

  • Apple Geliştirici Hesabınızı kaydedin
  • Uygulamanız için bir Uygulama Kimliği kaydedin
  • Test cihazlarınızın UUID’sini kaydedin
  • Bir iOS Uygulama Geliştirme ön hazırlık profili oluşturun
  • KeychainAccess’ten bir CertificateSigningRequest oluşturun
  • CertificateSigningRequest’i https://developer.apple.com/account/ios/profile/profileCreate.action adresine gönderin
  • Anahtarlık’a aktarmak için sertifikayı indirin ve çift tıklayın
  • XCode > Tercihler > Hesaplar’a gidin ve Apple Geliştirici Hesabınızı kaydedin

IOS Cihaz Testi #

  • Geliştirme iş istasyonunuzun ve iPhone’unuzun aynı WiFi ağına bağlı olduğundan emin olun. İnternet paylaşımı, etkin noktalar ve diğer geçici ağlar çalışmaz.
  • Sudo meteor run ios-device komutunu çalıştırın
  • Cihazınıza dağıtın!

Cordova projenizi yapılandırın (config.xml) #

Meteor, derleme sırasında uygulama dizininizin kökündeki bir “mobile-config.js” dosyasını okur ve Cordova’nın “config.xml” dosyasını oluşturmak için burada belirtilen ayarları kullanır.

Project_folder
 ├── /.meteor
 └── mobile-config.js

Çoğu yapılandırma, “mobile-config.js” (uygulama meta verileri, tercihler, simgeler ve başlatma ekranları ile Cordova eklentileri yükleme parametreleri) ile gerçekleştirilebilir.

App.info({
  id: 'com.example.matt.uber',
  name: 'über',
  description: 'Get über power in one button click',
  author: 'Matt Development Group',
  email: 'contact@example.com',
  website: 'http://example.com'
});

// Set up resources such as icons and launch screens.
App.icons({
  'iphone': 'icons/icon-60.png',
  'iphone_2x': 'icons/icon-60@2x.png',
  // ... more screen sizes and platforms ...
});

App.launchScreens({
  'iphone': 'splash/Default~iphone.png',
  'iphone_2x': 'splash/Default@2x~iphone.png',
  // ... more screen sizes and platforms ...
});

// Set PhoneGap/Cordova preferences
App.setPreference('BackgroundColor', '0xff0000ff');
App.setPreference('HideKeyboardFormAccessoryBar', true);
App.setPreference('Orientation', 'default');
App.setPreference('Orientation', 'all', 'ios');

// Pass preferences for a particular PhoneGap/Cordova plugin
App.configurePlugin('com.phonegap.plugins.facebookconnect', {
  APP_ID: '1234567890',
  API_KEY: 'supersecretapikey'
});

_’/.meteor/local/cordova-build/config.xml’ dosyasını manuel olarak düzenlemeyin, çünkü bu dosya her ‘meteor run ios/android’ veya ‘meteor build’de yeniden oluşturulacaktır, dolayısıyla tüm değişikliklerinizi kaybedersiniz.

Referans sayfası: Meteor Rehberi > İnşa Et > Mobil > Uygulamanızı Yapılandırma

deviceready olayını algılama #

Tabii ki, cep telefonunu tespit etmenin en iyi yolu, donanımın sizi doğrudan bilgilendirmesidir. Cordova PhoneGap, bir olay dinleyicisi ekleyebileceğiniz ‘cihaza hazır’ bir olay sunar.

 document.addEventListener('deviceready', function(){
  Session.set('deviceready', true);
 }, false);
