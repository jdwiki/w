Başlarken #

Meteor’u yükle #

OS X ve Linux’ta #

Terminalinizden en son resmi Meteor sürümünü yükleyin:

$ curl https://install.meteor.com/ | sh

Windows’ta #

Resmi Meteor yükleyicisini buradan indirin.

Uygulamanızı oluşturun #

Meteor’u yükledikten sonra bir proje oluşturun:

$ meteor create myapp

Çalıştır #

Yerel olarak çalıştırın:

$ cd myapp
$ meteor npm install
$ meteor

Not: Şu ​​anda çalışan Meteor sunucusu: http://localhost:3000/

Ardından yeni Meteor uygulamanızı görmek için http://localhost:3000 adresine gidin.

  • Meteor’u kullanmaya başlama hakkında daha fazla bilgiyi [Meteor Rehberi]‘den okuyun.
  • Meteor Paketlerini atmosphere‘de keşfedin - modern, hızlı, iyi tasarlanmış bir paket yöneticisi.

Örnek uygulamalar #

Meteor’un yerleşik birkaç örnek uygulaması vardır. Bunlardan biriyle bir proje oluşturabilir ve nasıl inşa edildiğini öğrenebilirsiniz. Örnek bir uygulama oluşturmak için Meteor’u yükleyin (bkz. Başlarken) ve ardından şunu yazın:

meteor create --example <app name>

Örneğin, örnek bir “todos” uygulaması oluşturmak için şunu yazın:

meteor create --example todos

Tüm örnek uygulamaların bir listesini almak için şunu yazın:

meteor create --list

Derleme ilerlemesini anlama #

Bazen yapılar beklenenden daha uzun sürer. Oluşturma işlemi sırasında neler olduğunu daha iyi anlamak için ayarlayabileceğiniz birkaç ortam değişkeni vardır.

METEOR_DEBUG_BUILD=1       (logs progress)
METEOR_PROFILE=<n>         (logs time spent)
METEOR_DEBUG_SPRINGBOARD=1 (?)
METEOR_DEBUG_SQL=1         (logs SQLITE calls)
METEOR_PROGRESS_DEBUG=1    (? looks like it might be useful, but seems confusing)

Burada “” bir ms sayısıdır. Bundan daha uzun süren herhangi bir işlem günlüğe kaydedilir.

Linux/OSX Örneği #

export METEOR_DEBUG_BUILD=1
export METEOR_PROFILE=100
meteor

Windows Örneği #

set METEOR_DEBUG_BUILD=1
set METEOR_PROFILE=100
meteor

Meteor Projelerini ve Kurulu Paketleri Güncelleme #

Meteor Aracı, daha yeni bir sürüm mevcut olduğunda sizi bilgilendirecektir.

Meteor projelerini en son sürüme güncellemek için bir Meteor projesi içinde aşağıdaki komutu yürütün:

meteor update

Meteor projenizi belirli bir Meteor sürümüne güncellemek istiyorsanız, proje içinde aşağıdaki komutu çalıştırın:

meteor update --release <release>

Çekirdek olmayan tüm paketleri güncellemek istiyorsanız, şunu çalıştırın:

meteor update --packages-only

Belirli paketleri, adlarını komut satırı argümanı olarak “meteor güncellemesi"ne ileterek de güncelleyebilirsiniz, örneğin:

meteor update [packageName packageName2 ...]

Mobil Uygulamalar Oluşturun #

Meteor, uygulamanızı bir hibrit Mobil Uygulamada paketlemek için Cordova kullanır. Paketlendikten sonra, Uygulama yerel Uygulamalar gibi dağıtılabilir (Apple App Store, Google Play Store vb. aracılığıyla)

  1. Meteor projenize hedef platform(lar)ı ekleyin:
meteor add-platform android
meteor add-platform ios # Only available with Mac OS

  1. Yükleyin Android SDK ve/veya Xcode (iOS için Mac OS gerektirir).

  2. Projenizi Çalıştır (geliştirme moduyla başlayın):

meteor run android # You may need to configure a default Android emulator first

iOS için (yalnızca Mac OS ile kullanılabilir):

meteor run ios # This will auto start an iOS simulator
  1. Dağıtım için Uygulama paketinizi oluşturun:
meteor build <output_folder> --server <url_app_should_connect_to>

Bu, sunucu paketinizin yanında “android” ve/veya “ios” klasör(ler)i oluşturacaktır.

  • “android” klasörü, imzalayıp sıkıştırmanız gereken “release-unsigned.apk” dosyasını içerir.
  • ‘ios’ klasörü, imzalamanız gereken Xcode projesini içerir.

Ayrıca Meteor Mobil Uygulamalar konusuna bakın. Referans sayfası: Meteor Kılavuzu > İnşa > Mobil

Paketleri Yönetme #

Meteor’un atmospherejs.com üzerinde kendi paket deposu var

Aşağıdakileri çalıştırarak atmosferden yeni paketler ekleyebilirsiniz:

meteor add [package-author-name:package-name]

Örneğin:

meteor add kadira:flow-router

Benzer şekilde, aynı paketi şu şekilde kaldırabilirsiniz:

meteor remove kadira:flow-router

Projenizdeki mevcut paketleri görmek için şunu yazın:

meteor list

Paketlerin listesi ./meteor/packages dosyasında da bulunabilir. Paket eklemek için paket adını bu dosyaya ekleyin ve kaldırmak için silin.

Yerel olarak bir paket eklemek için (örneğin yayınlanmamış paketler veya yayınlanmış paketlerin düzenlenmiş versiyonu), paketi kökteki ‘paketler’ klasörüne kaydedin.

1.3 sürümünden itibaren Meteor npm paketleri için destek eklendi.

’npm’ komutunu Meteor projesinin dizini içinde, normalde Meteor olmadan yapacağınız gibi veya npm’nin paketlenmiş sürümünü kullanacak olan ‘meteor npm’ komutuyla kullanabilirsiniz.

Meteor Aracı ve Meteor Projelerinin Versiyonunu Kontrol Etme #

Meteor Aracı #

Meteor aracının kurulu sürümünü kontrol etmek için, herhangi bir Meteor projesinin dışında aşağıdaki komutu çalıştırmanız yeterlidir:

meteor --version

Tüm resmi (önerilen) Meteor sürümlerinin bir listesini almak için şunu çalıştırın:

meteor show METEOR

Meteor Projeleri #

Meteor’un proje sürümünü kontrol etmek isterseniz, bir proje içinde aşağıdaki komutu da uygulayabilirsiniz:

meteor --version

veya sadece .meteor/release dosyasının içeriğini yazdırın:

cat .meteor/release

Meteor projenizde kurulu olan paketlerin versiyonunu kontrol etmek isterseniz, .meteor/versions dosyasının içeriğini yazdırın:

cat .meteor/versions

Meteor Web Sitesi #

Meteor tabanlı bir web sitesinin hangi Meteor sürümünün çalıştığını görmek için, web sitesini ziyaret ederken ‘Meteor.release’ içeriğini tarayıcınızın konsoluna dökün:

Meteor.release
