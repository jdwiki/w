AWS Hizmetine Kaydolun #

Yeni başlayanların çoğu bulut barındırma konusunda kafası karıştığından, bu kılavuzu ubuntu işletim sistemi ile aws’de meteor ayarlamayı öğrenmek için yazıyorum. Bulut sunucunuz zaten çalışıyorsa, bu adımı atlamaktan çekinmeyin ve doğrudan aws’ye meteor yüklemeye gidin.

AWS Konsolunda oturum açın.EC2’yi seçin. EC2 Dashboard’a gidin. Örnek Oluştur altında, örneği başlat’a tıklayın.

Sonraki adımda ubuntu örneğini seçin

Anahtar çifti oluşturun ve yerel makinenize özel anahtarı indirin.

Kabuk üzerinden aws’ye giriş yapın (özel anahtarı kullanarak, özel anahtarın yolunuzda olduğundan emin olun veya özel anahtarı içeren dizinden komutu çalıştırın)

ssh -i "myprivatekey.pem" ubuntu@ec2-xx-xx-xx-xx.ap-south-1.compute.amazonaws.com

ec2-xx-xx-xx-xx.ap-south-1.compute.amazonaws.com amazon konsolundaki genel dns örneği adıdır. ubuntu kullanıcı adıdır. Ayrıca genel ip adresini de kullanabilirsiniz.

METEOR’U AWS INSTANCE’E YÜKLEME ADIMLARI (mupx kullanarak)

  1. özel anahtarı yerel makineden aws sunucusu ssh klasörüne kopyalayın

örnek /home/ubuntu/.ssh/myprivatekey.pem

  1. paketleyiciyi en son sürüme güncelleyin

    sudo apt-get update

  2. python yazılım özelliklerini yükleyin

    sudo apt-get install python-software-properties

  3. npm ve node’u kurun (isteğe bağlı olarak nvm’yi de kurun)

    sudo apt-get install npm

nvm’yi yükle

curl https://raw.githubusercontent.com/creationix/nvm/v0.11.1/install.sh | bash

Düğüm yükle

nvm install 4.4.7

nvm use 4.4.7

  1. aws cli’yi yükleyin

    sudo apt-get install awscli

  2. Meteor yükleyin

    sudo npm install -g mupx

    sudo npm install -g mupx-letsencrypt
    (meteor 1.4 şu anda yalnızca mpux-letsencrypt tarafından kullanılabilir)

  3. Proje dizininize giderek mupx’i başlatın veya yoksa yeni dizin oluşturun

    mupx-letsencrypt init

Aşağıdaki gibi bir hata alırsanız, eski düğüm olabilir, bağlantı oluşturmanız gerekir.

/usr/bin/env: node: No such file or directory



sudo ln -s /usr/bin/nodejs /usr/bin/node

  1. Meteor yükleyin

    curl https://install.meteor.com | /bin/sh

  2. mup.json’u düzenleyin (1. adımdaki kullanıcı adı:ubuntu’yu ve özel anahtarın konumunu doğru girdiğinizden emin olun)

    use nano file editor (to edit on files on ubuntu, also can use vi)

    nano mup.json

Örnek mup.json

    {
  // Server authentication info
  "servers": [
    {
      "host": "ec2-xx-xx-xx-xx.ap-south-1.compute.amazonaws.com",
      "username": "ubuntu",
      //"password": "password",
      // or pem file (ssh based authentication)
      "pem": "~/.ssh/myprivatekey.pem",
      // Also, for non-standard ssh port use this
      //"sshOptions": { "port" : 49154 },
      // server specific environment variables
      "env": {}
    }
  ],

  // Install MongoDB on the server. Does not destroy the local MongoDB on future setups
  "setupMongo": true,

  // WARNING: Node.js is required! Only skip if you already have Node.js installed on server.
  "setupNode": false,

  // WARNING: nodeVersion defaults to 0.10.36 if omitted. Do not use v, just the version number.
  //"nodeVersion": "4.4.7",

  // Install PhantomJS on the server
  "setupPhantom": true,

  // Show a progress bar during the upload of the bundle to the server.
  // Might cause an error in some rare cases if set to true, for instance in Shippable CI
  "enableUploadProgressBar": true,

  // Application name (no spaces).
  "appName": "my-app",

  // Location of app (local directory). This can reference '~' as the users home directory.
  // i.e., "app": "/Users/ubuntu/my-app",
  // This is the same as the line below.
  "app": "/Users/ubuntu/my-app",

  // Configure environment
  // ROOT_URL must be set to https://YOURDOMAIN.com when using the spiderable package & force SSL
  // your NGINX proxy or Cloudflare. When using just Meteor on SSL without spiderable this is not necessary
  "env": {
    "PORT": 80,
    "ROOT_URL": "http://myapp.com",
     // only needed if mongodb is on separate server
    "MONGO_URL": "mongodb://url:port/MyApp",
    "MAIL_URL":   "smtp://postmaster%40myapp.mailgun.org:adj87sjhd7s@smtp.mailgun.org:587/"
  },

  // Meteor Up checks if the app comes online just after the deployment.
  // Before mup checks that, it will wait for the number of seconds configured below.
  "deployCheckWaitTime": 60
}

  1. Proje dizininde aşağıdaki komutu çalıştıran mongo dahil Meteor’u kurun.

    mupx-letsencrypt setup

  2. projeyi mupx kullanarak dağıtın

    mupx-letsencrypt deploy

Bazı yardımcı komutlar

Mupx günlüklerini kontrol etmek için

mupx logs -f

Docker’ı kontrol etmek için

 docker -D info

Ağ durumunu kontrol etmek için

netstat -a

İşlemci ve bellek kullanımı dahil olmak üzere mevcut çalışan süreci kontrol etmek için

 top

Aws’de mongo kabuğuna erişmek için mongo istemcisini kurun

sudo apt-get install mongodb-clients

mongodb sorgularını çalıştırmak için

mongo projectName

Bir kez İçinde mongo kabuğu koşusu

 db.version()
 db.users.find()

Harika bir araç sağladığın için teşekkürler arunoda https://github.com/arunoda/meteor-up

İyi çalışmalar için teşekkürler mupx-letsencrypt ekibi. https://www.npmjs.com/package/mupx-letsencrypt
