Meteor + React + ReactRouter

  • Projeyi oluşturun

Bu sayfada

  • Projeyi oluşturun
  • React + ReactRouter ekleyin
  • Adım 3- Hesap Ekle
  • Rol ekle

Bu belge, ReactRouter’ın Meteor ve React ile nasıl kullanılacağını gösterecektir. Roller ve kimlik doğrulama dahil olmak üzere sıfırdan çalışan bir uygulamaya.

Her adımı bir örnekle göstereceğim

1- Projeyi oluşturun

2- React + ReactRouter ekleyin

3- Hesap Ekle

4- Rol paketleri ekleyin

Projeyi oluşturun #

1- Öncelikle https://www.meteor.com/install kurun

2- Bir proje oluşturun. (--bare boş bir proje oluşturmaktır)

meteor oluştur --bare MyAwesomeProject

3- Minimum dosya yapısını (-p ara dizinleri oluşturmak için) oluşturun:

`cd MyAwesomeProject'

mkdir -p client server imports/api imports/ui/{bileşenler,layouts,pages} import/startup/{client,server}

4- Şimdi client/main.html’de bir HTML dosyası oluşturun

<head>
   <meta charset="utf-8">
   <title>My Awesome Meteor_React_ReactRouter_Roles App</title>
</head>
 
<body>
  Welcome to my Meteor_React_ReactRouter_Roles app
</body>

5- Çalıştığından emin olun: (3000 varsayılan bağlantı noktasıdır, böylece ‘-p 3000’i gerçekten atlayabilirsiniz)

meteor koşusu -p 3000

ve tarayıcınızı ’localhost:3000’ üzerinde açın

Not: #

  • İşleri kısaltmak için oluşturmanız gereken diğer dosyaları atlıyorum. Spesifik olarak, client , imports/startup/{client,server} ve server dizinlerinde bazı index.js dosyaları oluşturmanız gerekecektir.

  • Tam bir örneği https://github.com/rafa-lft/Meteor_React_Base adresinde görüntüleyebilirsiniz. Step1_CreateProject etiketini arayın

React + ReactRouter ekleyin #

Gerekirse, cd MyAwesomeProject proje dizininizi değiştirin

1- Tepki ve tepki-yönlendirici ekleyin

meteor npm kurulumu --save tepki-yönlendirici@3.0.0 tepki@15.5.4 tepki-dom@15.5.4

2- client/main.html dosyasını düzenleyin ve içeriğini değiştirin:

 <body>
    <div id="react-root"></div>
 </body>

ReactRouter neyi göstermeye karar verirse versin, bunu ‘#react-root’ öğesinde gösterecektir.

3- Layouts dosyasını imports/ui/layouts/App.jsx içinde oluşturun

import React, { Component } from 'react';
import PropTypes from 'prop-types';


class App extends Component {
  constructor(props) {
    super(props);
  }

  render() {
    return (
      <div>
        {this.props.children}
      </div>
    );
  }
}

App.propTypes = {
  children: PropTypes.node
};

export default App;

4- Imports/startup/client/Routes.jsx içinde Routes dosyasını oluşturun

import ReactDOM from 'react-dom';
import React, { Component } from 'react';
import { Router, Route, IndexRoute, browserHistory } from 'react-router';

import App from '../../ui/layouts/App.jsx';

import NotFound from '../../ui/pages/NotFound.jsx';
import Index from '../../ui/pages/Index.jsx';


class Routes extends Component {
  constructor(props) {
    super(props);
  }

  render() {
    return (
      <Router history={ browserHistory }>
        <Route path="/" component={ App }>
          <IndexRoute name="index" component={ Index }/>
          <Route path="*" component={ NotFound }/>
        </Route>
      </Router>
    );
  }
}

Routes.propTypes = {};


Meteor.startup(() =>{
  ReactDOM.render(
    <Routes/>,
    document.getElementById('react-root')
  );
});

Not: #

  • İşleri kısaltmak için oluşturmanız gereken diğer dosyaları atlıyorum. Özellikle, import/ui/pages{Index.jsx,NotFound.jsx} olup olmadığını kontrol edin.

  • Tam bir örneği https://github.com/rafa-lft/Meteor_React_Base adresinde görüntüleyebilirsiniz. Step2_ReactRouter etiketini arayın

Adım 3- Hesap Ekle #

Gerekirse, cd MyAwesomeProject proje dizininizi değiştirin

1- Hesap paketleri ekleyin: meteor hesap ekle-taban hesapları-şifre tepki-meteor-verileri

2- Imports/startup/Routes.jsx içindeki login ve signup sayfalarına rotaları ekleyin render() yöntemi aşağıdaki gibi olacaktır:

  render() {
    return (
      <Router history={ browserHistory }>
        <Route path="/" component={ App }>
          <IndexRoute name="index" component={ Index }/>
          <Route name="login" path="/login" component={ Login }/>
          <Route name="signup" path="/signup" component={ Signup }/>
          <Route name="users" path="/users" component={ Users }/>
          <Route name="editUser" path="/users/:userId" component={ EditUser }/>
          <Route path="*" component={ NotFound }/>
        </Route>
      </Router>
    );
  }

Not: #

  • İşleri kısaltmak için ihtiyaç duyacağınız diğer bazı dosyaları atlıyorum. Özellikle, imports/startup/server/index.js imports/ui/layouts/{App,NavBar}.jsx ve import/ui/pages/{Login,Signup,Users,EditUser}.jsx’i kontrol edin

  • Tam bir örneği https://github.com/rafa-lft/Meteor_React_Base adresinde görüntüleyebilirsiniz. Step3_Accounts etiketini arayın

Rol ekle #

1- Rol paketi ekleyin ( https://github.com/alanning/meteor-roles)

meteor alanning ekleyin:roller

2- Bazı rol sabitleri oluşturun. Imports/api/accounts/roles.js dosyasında

const ROLES = {
  ROLE1: 'ROLE1',
  ROLE2: 'ROLE2',
  ADMIN: 'ADMIN'
};

export default ROLES;

3- Bir kullanıcıya rollerin nasıl ekleneceğini/güncelleneceğini göstermeyeceğim, sadece sunucu tarafında, kullanıcı rollerini Roles.setUserRoles(user.id, roller); ile ayarlayabileceğinizi belirteceğim. https’de daha fazla bilgi için kontrol edin. ://github.com/alanning/meteor-roles ve http://alanning.github.io/meteor-roles/classes/Roles.html

4- Tüm hesapları ve rol dosyalarını zaten kurduğunuzu varsayarsak (https://github.com/rafa-lft/Meteor_React_Base adresindeki tam örneğe bakın. Step4_roles etiketini arayın) artık sorumlu olacak bir yöntem oluşturabiliriz. farklı rotalara erişime izin vermek (veya vermemek). imports/startup/client/Routes.jsx’te


class Routes extends Component {
  constructor(props) {
    super(props);
  }

  authenticate(roles, nextState, replace) {
    if (!Meteor.loggingIn() && !Meteor.userId()) {
      replace({
        pathname: '/login',
        state: {nextPathname: nextState.location.pathname}
      });
      return;
    }
    if ('*' === roles) { // allow any logged user
      return;
    }
    let rolesArr = roles;
    if (!_.isArray(roles)) {
      rolesArr = [roles];
    }
    // rolesArr = _.union(rolesArr, [ROLES.ADMIN]);// so ADMIN has access to everything
    if (!Roles.userIsInRole(Meteor.userId(), rolesArr)) {
      replace({
        pathname: '/forbidden',
        state: {nextPathname: nextState.location.pathname}
      });
    }
  }

  render() {
    return (
      <Router history={ browserHistory }>
        <Route path="/" component={ App }>
          <IndexRoute name="index" component={ Index }/>
          <Route name="login" path="/login" component={ Login }/>
          <Route name="signup" path="/signup" component={ Signup }/>

          <Route name="users" path="/users" component={ Users }/>

          <Route name="editUser" path="/users/:userId" component={ EditUser }
                 onEnter={_.partial(this.authenticate, ROLES.ADMIN)} />


          {/* ********************
           Below links are there to show Roles authentication usage.
           Note that you can NOT hide them by
           { Meteor.user() && Roles.userIsInRole(Meteor.user(), ROLES.ROLE1) &&
           <Route name=.....
           }
           as doing so will change the Router component on render(), and ReactRouter will complain with:
           Warning: [react-router] You cannot change <Router routes>; it will be ignored

           Instead, you can/should hide them on the NavBar.jsx component... don't worry: if someone tries to access
           them, they will receive the Forbidden.jsx component
           *************/ }
          <Route name="forAnyOne" path="/for_any_one" component={ ForAnyone }/>

          <Route name="forLoggedOnes" path="/for_logged_ones" component={ ForLoggedOnes }
                 onEnter={_.partial(this.authenticate, '*')} />

          <Route name="forAnyRole" path="/for_any_role" component={ ForAnyRole }
                 onEnter={_.partial(this.authenticate, _.keys(ROLES))}/>

          <Route name="forRole1or2" path="/for_role_1_or_2" component={ ForRole1or2 }
                 onEnter={_.partial(this.authenticate, [ROLES.ROLE1, ROLES.ROLE2])} />

          <Route name="forRole1" path="/for_role1" component={ ForRole1 }
                 onEnter={_.partial(this.authenticate, ROLES.ROLE1)}/>

          <Route name="forRole2" path="/for_role2" component={ ForRole2 }
                 onEnter={_.partial(this.authenticate, ROLES.ROLE2)} />


          <Route name="forbidden" path="/forbidden" component={ Forbidden }/>

          <Route path="*" component={ NotFound }/>
        </Route>
      </Router>
    );
  }
}

Bazı rotalara onEnter tetikleyicisi ekledik. Bu rotalar için, hangi Roller’in girmesine izin verildiğini de geçiyoruz. onEnter geri aramasının orijinal olarak 2 param aldığını unutmayın. Başka bir tane (roller) eklemek için alt çizginin kısmi (http://underscorejs.org/#partial) kullanıyoruz authenticate yöntemi (onEnter tarafından çağrılır) rolleri alır ve:

  • Kullanıcının hiç oturum açıp açmadığını kontrol edin. Değilse, ‘/login’e yönlendirir.
  • Eğer roller === ‘*’ ise, oturum açmış herhangi bir kullanıcının girebileceğini varsayıyoruz, bu yüzden izin veriyoruz
  • Aksi takdirde, kullanıcıya izin verilip verilmediğini (Roles.userIsInRole) doğrularız ve değilse, yasaklıya yönlendiririz.
  • İsteğe bağlı olarak, bir satırın yorumunu kaldırabilirsiniz, böylece YÖNETİCİ her şeye erişebilir.

Kod, herkes için (onEnter geri çağrısı yok), oturum açmış herhangi bir kullanıcı için, en az 1 rolü olan oturum açmış herhangi bir kullanıcı için ve belirli roller için izin verilen birkaç farklı rota örneğine sahiptir.

Ayrıca, ReactRouter’ın (en azından sürüm 3’te), Render’daki rotaları değiştirmeye izin vermediğini unutmayın. Yani Routes.jsx içindeki rotaları gizleyemezsiniz. Bu nedenle, kimlik doğrulama yönteminde /forbidden’a yönlendiririz.

5- ReactRouter ve Meteor ile ilgili yaygın bir hata, kullanıcı durumu güncellemelerinin gösterilmemesiyle ilgilidir. Örneğin, kullanıcı oturumu kapattı, ancak adını hala gezinme çubuğunda gösteriyoruz. Bunun nedeni Meteor.user()‘in değişmesidir, ancak biz yeniden oluşturmuyoruz.

Bu hata, createContainer’da Meteor.user() çağrılarak çözülebilir. İşte imports/ui/layouts/NavBar.jsx’te kullanılan bir örnek:

export default createContainer((/* {params}*/) =>{
  Meteor.user(); // so we render again in logout or if any change on our User (ie: new roles)
  const loading = !subscription.ready();
  return {subscriptions: [subscription], loading};
}, NavBar);

Not: #

  • İşleri kısaltmak için ihtiyaç duyacağınız diğer bazı dosyaları atlıyorum. Özellikle, imports/startup/server/index.js imports/ui/layouts/{App,NavBar}.jsx ve import/ui/pages/{Login,Signup,Users,EditUser}.jsx’i kontrol edin

  • Tam bir örneği https://github.com/rafa-lft/Meteor_React_Base adresinde görüntüleyebilirsiniz. Step4_roles etiketini arayın
