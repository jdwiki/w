Meteor’un veri alt sistemi içinde, bir sunucu yayını ve buna karşılık gelen istemci abonelikleri, temel alınan verilerin sunucu ve istemci arasında sürekli olarak senkronize edildiği reaktif, canlı veri aktarımının ana mekanizmalarıdır.

Küresel yayınlar #

Global yayının bir adı yoktur ve bağlı istemciden bir abonelik gerektirmez ve bu nedenle istemci sunucuya bağlanır bağlanmaz bağlı istemci tarafından kullanılabilir.

Bunu başarmak için, yayını şöyle ‘boş’ olarak adlandırmak yeterlidir.

Meteor.publish(null, function() {
  return SomeCollection.find();
})

Bir yayında hata oluşturma ve yanıt verme. #

Sunucuda bunun gibi bir yayın oluşturabilirsiniz. this.userId, şu anda oturum açmış olan kullanıcının kimliğidir. Herhangi bir kullanıcı oturum açmadıysa, bir hata atmak ve buna yanıt vermek isteyebilirsiniz.

import Secrets from '/imports/collections/Secrets';

Meteor.publish('protected_data', function () {
  if (!this.userId) {
    this.error(new Meteor.Error(403, "Not Logged In."));
    this.ready();
  } else {
    return Secrets.find();
  }
});

İstemcide, aşağıdakilerle yanıt verebilirsiniz.

Meteor.subscribe('protected_data', {
  onError(err) {
    if (err.error === 403) {
      alert("Looks like you're not logged in");
    }
  },
});

File /imports/collections/Secrets, gizli dizi koleksiyonuna aşağıdaki gibi referans oluşturur:

const Secrets = new Mongo.Collection('secrets');

Şablon kapsamlı abonelikler #

Meteor’un varsayılan şablonlama sistemi Boşluk çubukları ve onun temel oluşturma alt sistemi Blaze, basit bir şablon kodu parçasının kendi verilerine abone olabilmesi, şablonun yıkılması sırasında kendi izlerini durdurabilmesi ve temizleyebilmesi gibi yayın yaşam döngüsü yöntemleriyle görünüşte bütünleşir.

Bundan faydalanmak için, aşağıdaki gibi ‘Meteor’ sembolü yerine şablon örneğine abone olmanız gerekir:

İlk önce şablonu ayarlayın

<template name="myTemplate">
  We will use some data from a publication here
</template>

Ardından ilgili yaşam döngüsü geri aramasına dokunun

Template.myTemplate.onCreated(function() {
  const templateInstance = this;
  templateInstance.subscribe('somePublication')
})

Şimdi bu şablon yok edildiğinde yayın da otomatik olarak durdurulacak.

Not: Abone olunan veriler tüm şablonlar için geçerli olacaktır.

Adlandırılmış yayınlar #

Adlandırılmış yayın, bir ada sahip olan ve istemciden açıkça abone olması gereken yayındır.

Bu sunucu tarafı kodunu göz önünde bulundurun:

Meteor.publish('somePublication', function() {
  return SomeCollection.find()
})

Müşterinin bunu şu şekilde istemesi gerekir:

Meteor.subscribe('somePublication')

Yayınlandığında Kullanıcı Hesabını Doğrulama #

Bazen, bir kullanıcı girişi gerektirerek yayınlarınızın güvenliğini daha da artırmak iyi bir fikirdir. İşte bunu Meteor aracılığıyla nasıl başaracağınız.

import { Recipes } from '../imports/api/recipes.js';
import { Meteor } from 'meteor/meteor';

Meteor.publish('recipes', function() {
  if(this.userId) {
    return Recipe.find({});
  } else {
    this.ready();  // or: return [];
  }
});

Temel Abonelik ve Yayın #

İlk olarak, “otomatik yayınlama"yı kaldırın. “autopublish” tüm veritabanını istemci tarafında otomatik olarak yayınlar ve bu nedenle yayınların ve aboneliklerin etkileri görülmez.

“otomatik yayınlama"yı kaldırmak için:

$ meteor remove autopublish

Ardından yayınlar oluşturabilirsiniz. Aşağıda tam bir örnek bulunmaktadır.

import { Mongo } from 'meteor/mongo';
import { Meteor } from 'meteor/meteor';

const Todos = new Mongo.Collection('todos');

const TODOS = [
  { title: 'Create documentation' },
  { title: 'Submit to Stack Overflow' }
];

if (Meteor.isServer) {
  Meteor.startup(function () {
    TODOS.forEach(todo => {
      Todos.upsert(
        { title: todo.title },
        { $setOnInsert: todo }
      );
    });
  });

  // first parameter is a name.
  Meteor.publish('todos', function () {
    return Todos.find();
  });
}

if (Meteor.isClient) {
  // subscribe by name to the publication.
  Meteor.startup(function () {
    Meteor.subscribe('todos');
  })
}

Geçici bir istemci tarafı adlı koleksiyonda yayınlayın. #

Çünkü yayınlananlara ince ayar yapmanız gerekiyorsa.

import { Mongo } from 'meteor/mongo';
import { Meteor } from 'meteor/meteor';
import { Random } from 'meteor/random';

if (Meteor.isClient) {
  // established this collection on the client only.
  // a name is required (first parameter) and this is not persisted on the server.
  const Messages = new Mongo.Collection('messages');
  Meteor.startup(function () {
    Meteor.subscribe('messages');
    Messages.find().observe({
      added: function (message) {
        console.log('Received a new message at ' + message.timestamp);
      }
    });
  })
}

if (Meteor.isServer) {
  // this will add a new message every 5 seconds.
  Meteor.publish('messages', function () {
    const interval = Meteor.setInterval(() => {
      this.added('messages', Random.id(), {
        message: '5 seconds have passed',
        timestamp: new Date()
      })
    }, 5000);
    this.added('messages', Random.id(), {
      message: 'First message',
      timestamp: new Date()
    });
    this.onStop(() => Meteor.clearInterval(interval));
  });
}

Bir yayına reaktif olarak yeniden abone olma #

Bir yayına (yeniden) abone olmak için bir şablon otomatik çalıştırma kullanılabilir. Herhangi bir reaktif veri değiştiğinde yeniden çalıştırılan bir reaktif bağlam oluşturur. Ayrıca, otomatik çalıştırma her zaman bir kez çalışır (ilk kez yürütüldüğünde).

Şablon otomatik çalıştırmaları normalde bir “onCreated” yöntemine yerleştirilir.

Template.myTemplate.onCreated(function() {
  this.parameter = new ReactiveVar();
  this.autorun(() => {
    this.subscribe('myPublication', this.parameter.get());
  });
});

Bu bir kez (ilk kez) çalışacak ve bir abonelik oluşturacaktır. Ardından, “parametre” reaktif değişkeni her değiştiğinde yeniden çalışacaktır.

Yayınlanan veriler alınırken Blaze görünümünde bekleyin #

Şablon JS kodu

Template.templateName.onCreated(function(){
    this.subscribe('subsription1');
    this.subscribe('subscription2');
});

Şablon HTML kodu

<template name="templateName">
    {{#if Template.subscriptionsReady }}
        //your actual view with data. it can be plain HTML or another template
    {{else}}
        //you can use any loader or a simple header
        <h2> Please wait ... </h2>
    {{/if}}
</template>

Yayınlardaki gecikmeyi simüle edin #

Gerçek dünyada, bağlantı ve sunucu gecikmeleri meydana gelebilir, geliştirme ortamındaki gecikmeleri simüle etmek için Meteor._sleepForMs(ms); kullanılabilir.

Meteor.publish('USER_DATA', function() {
    Meteor._sleepForMs(3000); // Simulate 3 seconds delay
    return Meteor.users.find({});
});

Birden çok imleç yayınlayın #

Bir dizi imleç döndürülerek aynı yayın yönteminden birden çok veritabanı imleci yayınlanabilir.

“Alt” imleçler birleşim olarak kabul edilecek ve reaktif olmayacaktır.

Meteor.publish('USER_THREAD', function(postId) {
  let userId   = this.userId;

  let comments = Comments.find({ userId, postId });
  let replies  = Replies.find({ userId, postId });

  return [comments, replies];
});

Yayınları Birleştirme #

İstemcide yayınlar birleştirilebilir, bu da tek bir imleç içinde farklı şekilli belgelerle sonuçlanır. Aşağıdaki örnek, bir kullanıcı dizininin bir uygulamanın kullanıcıları için minimum miktarda genel veriyi nasıl yayınlayabileceğini ve oturum açmış kullanıcı için daha ayrıntılı bir profil sunabileceğini gösterir.

// client/subscriptions.js  
Meteor.subscribe('usersDirectory');
Meteor.subscribe('userProfile', Meteor.userId());

// server/publications.js  
// Publish users directory and user profile

Meteor.publish("usersDirectory", function (userId) {
    return Meteor.users.find({}, {fields: {
        '_id': true,
        'username': true,
        'emails': true,
        'emails[0].address': true,

        // available to everybody
        'profile': true,
        'profile.name': true,
        'profile.avatar': true,
        'profile.role': true
    }});
});
Meteor.publish('userProfile', function (userId) {
    return Meteor.users.find({_id: this.userId}, {fields: {
        '_id': true,
        'username': true,
        'emails': true,
        'emails[0].address': true,
    
        'profile': true,
        'profile.name': true,
        'profile.avatar': true,
        'profile.role': true,
    
        // privately accessible items, only availble to the user logged in
        'profile.visibility': true,
        'profile.socialsecurity': true,
        'profile.age': true,
        'profile.dateofbirth': true,
        'profile.zip': true,
        'profile.workphone': true,
        'profile.homephone': true,
        'profile.mobilephone': true,
        'profile.applicantType': true
    }});
});
