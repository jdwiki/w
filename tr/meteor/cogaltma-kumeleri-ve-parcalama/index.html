Çoğaltma Kümeleri ve Parçalama

  • Çoğaltma Ayarı Hızlı Başlangıç
  • Replika Seti Yapılandırması

Bu sayfada

  • Çoğaltma Ayarı Hızlı Başlangıç
  • Replika Seti Yapılandırması

Tanıdık olmayanlar için, bir Replika Seti, üç sunucunun yedekli bir konfigürasyonu olarak tanımlanır. Sharded Database, her Shard’ın bir Replika Seti olarak tanımlandığı, yatay olarak çağrılan bir veritabanı olarak tanımlanır. Bu nedenle, parçalanmış bir Mongo kümesi, 2 parçalı bir küme için en az 11 sunucu içerir ve her ek parça için üç sunucu artar. Bu nedenle, parçalanmış bir kümede her zaman 11, 14, 17, 20, 23 vb. sunucu örnekleri bulunur. Yani, her biri 3 sunucudan oluşan 2 parça, 3 yapılandırma denetleyicisi ve 2 yönlendirici daha var. 2 parçalı bir küme için toplam 11 sunucu.

Çoğaltma Ayarı Hızlı Başlangıç #

İstediğiniz fiziksel veya sanal donanımı kullanarak kendinize üç sunucu oluşturun. (Bu öğretici, işletim sisteminiz olarak Ubuntu kullandığınızı varsayar.) Ardından, aşağıdaki talimatları üç kez tekrarlayın… her sunucu için bir kez.

# add the names of each server to the host file of each server
sudo nano /etc/hosts
  10.123.10.101 mongo-a
  10.123.10.102 mongo-b
  10.123.10.103 mongo-c

# install mongodb on the server
sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv 7F0CEB10
echo 'deb http://downloads-distro.mongodb.org/repo/ubuntu-upstart dist 10gen' | sudo tee /etc/apt/sources.list.d/mongodb.list
sudo apt-get update
sudo apt-get install mongodb-10gen


# create the /data/ directories
sudo mkdir /data
sudo mkdir /data/logs
sudo mkdir /data/db

# make sure the mongodb user and group have access to our custom directories
sudo chown -R mongodb:mongodb /data

# edit the mongo upstart file in /etc/init/mongodb.conf
sudo nano /etc/init/mongodb.conf
  start on started mountall
  stop on shutdown
  respawn
  respawn limit 99 5
  setuid mongodb
  setgid mongodb
  script
    exec /usr/bin/mongod --config /etc/mongodb.conf >> /data/logs/mongo-a.log 2>&1
  end script


# edit mongodb configuration file
sudo nano /etc/mongodb.conf
    dbpath=/data/db
    logpath=/data/logs/mongod.log
    logappend=true
    port=27017
    noauth=true
    replSet=meteor
    fork=true

# add a mongo log-rotation file
sudo nano /etc/logrotate.d/mongod
  /data/logs/*.log {
    daily
    rotate 30
    compress
    dateext
    missingok
    notifempty
    sharedscripts
    copytruncate
    postrotate
        /bin/kill -SIGUSR1 `cat /data/db/mongod.lock 2> /dev/null` 2> /dev/null || true
    endscript
  }

# make sure mongod service is started and running
sudo service mongodb start
sudo reboot

Replika Seti Yapılandırması #

Ardından mongo kabuğuna gidin ve kopya setini şu şekilde başlatın:

meteor mongo

  > rs.initiate()
  PRIMARY> rs.add("mongo-a")
  PRIMARY> rs.add("mongo-b")
  PRIMARY> rs.add("mongo-c")
  PRIMARY> rs.setReadPref('secondaryPreferred')
