Temel Sunucu Tarafı Günlüğü #

Günlüğe kaydetmenin ilk adımı basitçe Meteor’u kabuktan çalıştırmaktır ve sunucu günlüklerini komut konsolunda alırsınız.

meteor

Sonraki adım, std_out ve std_err içeriklerini aşağıdaki gibi bir günlük dosyasına aktarmaktır:

meteor > my_app_log.log 2> my_app_err.log

İstemci Tarafı Günlük Araçları #

Sunucu tarafınızın girişini yaptıktan sonra, istemci tarafına geçmenin zamanı geldi. Konsol API’sini keşfetmediyseniz, bir ikram için hazır olun. Aslında, her Chrome ve Safari kurulumuna özgü yerleşik Konsol API’si ile yapabileceğiniz her türlü şey vardır. Hatta o kadar ki, Winston’a veya diğer kayıt çerçevelerine ihtiyaç duymayabilirsiniz.

Yapmak isteyeceğiniz ilk şey, istemci tarafı günlük kaydı ve geliştirici araçlarını yüklemek. Hem Chrome hem de Safari onlarla birlikte gelir, ancak Firefox, Firebug uzantısını gerektirir.

Firebug Uzantısı

Ardından, Konsol API belgelerine göz atmak isteyeceksiniz. Aşağıdaki iki belge, öğrenme konsolu günlük kaydı için paha biçilmez kaynaklardır.

Gelişmiş Sunucu Günlüğü Araçları #

Hem sunucu tarafı günlük kaydınızı hem de istemci tarafı geliştirme araçlarınızı çalıştırdıktan sonra, Meteor Chrome DevTools Uzantısı gibi Meteor’a özgü uzantılara bakmaya başlayabilirsiniz. Bu, istemcide sunucu günlüğünü gerçekten gözlemlemenizi sağlar! Çünkü veritabanı her yerdedir. Günlüğe kaydetme gibi.

Chrome DevTools Uzantısı (Sunucu)

Veritabanı kapağında günlüğe kaydetme hatası #

Aşağıdaki örnek 0,5 - 0,7 gündür ve veritabanı henüz istemci tarafı imlecini doldurmadığında bir hatanın nasıl günlüğe kaydedileceğini gösterir.

Template.landingPage.postsList = function(){
  try{
    return Posts.find();
  }catch(error){
    //color code the error (red)
    console.error(error);
  }
}

Bir şablon yardımcısında veri bağlamıyla ilgili bilgileri günlüğe kaydetme #

Aşağıdakiler, Chrome Logging API’sini kullanır. .group() sözdizimi birden fazla şablonda kullanılıyorsa, farklı şablonlardan gelen konsol günlüklerini hiyerarşik bir ağaçta grafiksel olarak düzenler.

Geçerli veri bağlamının nasıl inceleneceğini ve verilerin nasıl dizileceğini de görebilirsiniz.

Template.landingPage.getId = function(){
  // using a group block to illustrate function scoping
  console.group('coolFunction');

  // inspect the current data object that landingPage is using
  console.log(this);

  // inspect a specific field of the locally scoped data object
  console.log(JSON.stringify(this._id);

  // close the function scope
  console.groupEnd();
  return this._id;
}

Olayları ve kullanıcı etkileşimlerini günlüğe kaydetme #

Chrome Logging API’sini kullanmaya ilişkin basit bir örnek.

Template.landingPage.events({
  'click .selectItemButton':function(){
    // color code and count the user interaction (blue)
    console.count('click .selectItemButton');
  }
});

Günlük düzeyi değişkenleriyle günlük kaydı #

Günlüğe kaydetme genellikle konsolu karmaşık hale getirebilir, bu nedenle verilerin hangi ayrıntılarının günlüğe kaydedildiğini kontrol etmek için günlük düzeylerinin tanımlanması yaygındır. Yaygın bir kalıp, bir günlük düzeyi değişkenleri belirtmektir.

var DEBUG = false;
var TRACE = false;
Template.landingPage.events({
  'click .selectItemButton':function(){
    TRACE && console.count('click .selectItemButton');

    Meteor.call('niftyAction', function(errorMessage, result){
        if(errorMessage){
            DEBUG && console.error(errorMessage);    
        }
    });
  }
});

Üretimde Oturum Açmayı Devre Dışı Bırak #

Bazı ekipler, kodlarında konsol günlüğü ifadeleri bırakmak istediklerini ancak üretimde görüntülenmesini istemediğini fark eder. Bir değişken ayarlanmazsa (muhtemelen bir ortam değişkeni) günlük işlevlerini geçersiz kılarlar. Ek olarak, bu bazı durumlarda bir güvenlik özelliği olarak nitelendirilebilir.

if (!DEBUG_MODE_ON) {
    console = console || {};
    console.log = function(){};

    console.log = function(){};
    console.error = function(){};
    console.count = function(){};
    console.info = function(){};
}

Winston #

Varsayılan günlük kaydı seçeneklerinden daha güçlü bir şeye ihtiyacınız varsa, Winston gibi bir araca bakmak isteyebilirsiniz. Atmosfer’e gidin ve mevcut birçok Winston paketinden birini arayın.

https://atmospherejs.com/?q=winston

Ancak dikkatli olun - Winston sofistike bir üründür ve birçok işlevsellik sunarken, uygulamanıza bir karmaşıklık katmanı da ekleyecektir.

Günlük düzeyi #

Topluluk tarafından geliştirilen LogLevel paketi için özel bir söz yapılmalıdır. Hafif ve kullanımı basit olmakla birlikte, Meteor’un paket boru hattıyla iyi çalışırken ve satır numaralarını ve dosya adlarını korurken bir denge kuruyor gibi görünüyor.

https://atmospherejs.com/practicalmeteor/loglevel
