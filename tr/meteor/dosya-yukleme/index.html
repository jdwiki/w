CollectionFS paketi, yazarı tarafından rafa kaldırıldı ve kullanımdan kaldırıldı; ancak, Atmosfer’de veya Meteor ekosisteminde Mongo’nun GridFS işlevini kullanmak için alternatif bir paket olmadığından ve kod hala mükemmel bir şekilde çalışıyor; Örneği başka bir GridFS çözümünün değiştirildiği belgelenene kadar StackOverflow Belgelerinden kaldırmamanızı öneririz.

Sunucu/İstemci #

Dosyaları yüklemek, ne yapmak istediğinize bağlı olarak kolay veya gerçekten karmaşık olabilir. Genel olarak, bir dosyanın kendisini transfer etmek o kadar da zor değildir. Ancak ekler, ikili dosyalar ve benzerleriyle ilgili çok sayıda uç durum vardır. Ve asıl sorun yatay ölçeklendirme ve sunucu ikinci, üçüncü ve n’inci kez klonlandığında işe yarayan bir çözüm yaratmaktır.

Temel bir sunucu/istemci yükleme modeliyle başlayalım. Belge nesne modeline bir dosya giriş öğesi ekleyerek başlıyoruz.

<template name="example">
  <input type=file />
</template>

Ardından, denetleyicinizdeki giriş öğesine bir olay ekleyin ve aktarımı başlatmak için yerel bir Meteor yöntemini ``startFileTransfer’’ çağırın.

// client/example.js
Template.example.events({
  'change input': function(ev) {  
    _.each(ev.srcElement.files, function(file) {
      Meteor.startFileTransfer(file, file.name);
    });
  }
});

// client/save.js
/**
 * @blob (https://developer.mozilla.org/en-US/docs/DOM/Blob)
 * @name the file's name
 * @type the file's type: binary, text (https://developer.mozilla.org/en-US/docs/DOM/FileReader#Methods) 
 *
 * TODO Support other encodings: https://developer.mozilla.org/en-US/docs/DOM/FileReader#Methods
 * ArrayBuffer / DataURL (base64)
 */
Meteor.startFileTransfer = function(blob, name, path, type, callback) {
  var fileReader = new FileReader(),
    method, encoding = 'binary', type = type || 'binary';
  switch (type) {
    case 'text':
      // TODO Is this needed? If we're uploading content from file, yes, but if it's from an input/textarea I think not...
      method = 'readAsText';
      encoding = 'utf8';
      break;
    case 'binary': 
      method = 'readAsBinaryString';
      encoding = 'binary';
      break;
    default:
      method = 'readAsBinaryString';
      encoding = 'binary';
      break;
  }
  fileReader.onload = function(file) {
    Meteor.call('saveFileToDisk', file.srcElement.result, name, path, encoding, callback);
  }
  fileReader[method](blob);
}

İstemci daha sonra gerçek aktarımı yapan ve her şeyi diske koyan saveFileToDisk sunucu yöntemini arayacaktır.

// 
/**
 * TODO support other encodings:
 * http://stackoverflow.com/questions/7329128/how-to-write-binary-data-to-a-file-using-node-js
 */
Meteor.methods({
  saveFileToDisk: function(blob, name, path, encoding) {
    var path = cleanPath(path), fs = __meteor_bootstrap__.require('fs'),
      name = cleanName(name || 'file'), encoding = encoding || 'binary',
      chroot = Meteor.chroot || 'public';
    // Clean up the path. Remove any initial and final '/' -we prefix them-,
    // any sort of attempt to go to the parent directory '..' and any empty directories in
    // between '/////' - which may happen after removing '..'
    path = chroot + (path ? '/' + path + '/' : '/');

    // TODO Add file existance checks, etc...
    fs.writeFile(path + name, blob, encoding, function(err) {
      if (err) {
        throw (new Meteor.Error(500, 'Failed to save file.', err));
      } else {
        console.log('The file ' + name + ' (' + encoding + ') was saved to ' + path);
      }
    }); 

    function cleanPath(str) {
      if (str) {
        return str.replace(/\.\./g,'').replace(/\/+/g,'').
          replace(/^\/+/,'').replace(/\/+$/,'');
      }
    }
    function cleanName(str) {
      return str.replace(/\.\./g,'').replace(/\//g,'');
    }
  }
});

Bu bir tür çıplak kemik yaklaşımı ve arzulanan çok şey bırakıyor. Bir CSV dosyası veya başka bir şey yüklemek için iyi olabilir, ancak bu kadar.

Dropzone (ütü ile:yönlendirici) #

Entegre bir Dropzone UI ve bir REST uç noktası ile biraz daha cilalı bir şey istiyorsak, UI yardımcıları ile özel REST yolları ve paketleri eklemeye başlamamız gerekecek.

Iron Router ve Dropzone’u içe aktararak başlayalım.

 meteor add iron:router
 meteor add awatson1978:dropzone

Dropzone yardımcısında belirtilen yükleme url yolunu yapılandırın.

Router.map(function () {
    this.route('uploads', {
      where: 'server',
      action: function () {
        var fs = Npm.require('fs');
        var path = Npm.require('path');
        var self = this;

        ROOT_APP_PATH = fs.realpathSync('.');

        // dropzone.js stores the uploaded file in the /tmp directory, which we access
        fs.readFile(self.request.files.file.path, function (err, data) {

          // and then write the file to the uploads directory
          fs.writeFile(ROOT_APP_PATH + "/assets/app/uploads/" +self.request.files.file.name, data, 'binary', function (error, result) {
            if(error){
              console.error(error);
            }
            if(result){
              console.log('Success! ', result);
            }
          });
        });
      }
    });
  });

Serin! Şık kullanıcı arayüzüne ve programlanabilir bir REST uç noktasına sahip bir dosya yükleyicimiz var. Ne yazık ki, bu özellikle iyi ölçeklenmiyor.

Filepicker.io #

İşleri ölçeklendirmek için sunucumuzda yerel depolamayı kullanmayı bırakmalı ve özel bir dosya depolama hizmeti kullanmaya başlamalı veya yatay bir depolama katmanı uygulamalıyız. Ölçeklenebilir dosya depolamaya başlamanın en kolay yolu, S3, Azure, Rackspace ve Dropbox’ı destekleyen Filepicker.io gibi bir çözüm kullanmaktır. loadpicker, bir süredir popüler bir Filerpicker unipackage olmuştur.

meteor add mrt:filepicker

Filepicker modeli diğer çözümlerden oldukça farklıdır, çünkü bu gerçekten 3. parti entegrasyonu ile ilgilidir. Meteor uygulamalarında oldukça nadir görülen bir model olan data-* özniteliklerine büyük ölçüde dayandığını göreceğiniz bir dosya seçici girişi ekleyerek başlayın.

<input type="filepicker"
  id="filepickerAttachment"
  data-fp-button-class="btn filepickerAttachment"
  data-fp-button-text="Add image" 
  data-fp-mimetypes="image/*"
  data-fp-container="modal"
  data-fp-maxsize="5000000" 
  data-fp-services="COMPUTER,IMAGE_SEARCH,URL,DROPBOX,GITHUB,GOOGLE_DRIVE,GMAIL">

Ayrıca bir API anahtarı ayarlamak, dosya seçici widget’ını oluşturmak, tetiklemek ve çıktılarını gözlemlemek isteyeceksiniz.

if(Meteor.isClient){
  Meteor.startup(function() {
    filepicker.setKey("YourFilepickerApiKey");
  });
  Template.yourTemplate.rendered = function(){
    filepicker.constructWidget($("#filepickerAttachment"));
  }
  Template.yourTemplate.events({
  'change #filepickerAttachment': function (evt) {
    console.log("Event: ", evt, evt.fpfile, "Generated image url:", evt.fpfile.url);
  });
});

Sunucu Yüklemeleri #

Aşağıdaki komut dosyaları, sunucu dosya sisteminden sunucuya bir dosya yüklemek içindir. Çoğunlukla yapılandırma dosyaları ve dosya gözlemcileri için.

//https://forums.meteor.com/t/read-file-from-the-public-folder/4910/5

// Asynchronous Method.
Meteor.startup(function () {
    console.log('starting up');

    var fs = Npm.require('fs');
    // file originally saved as public/data/taxa.csv
    fs.readFile(process.cwd() + '/../web.browser/app/data/taxa.csv', 'utf8', function (err, data) {
        if (err) {
            console.log('Error: ' + err);
            return;
        }

        data = JSON.parse(data);
        console.log(data);
    });
});


// Synchronous Method.
Meteor.startup(function () {
    var fs = Npm.require('fs');
    // file originally saved as public/data/taxa.csv
    var data = fs.readFileSync(process.cwd() + '/../web.browser/app/data/taxa.csv', 'utf8');

    if (Icd10.find().count() === 0) {
        Icd10.insert({
            date:  new Date(),
            data:  JSON.parse(data)
        });
    }
});


Meteor.methods({
  parseCsvFile:function (){
    console.log('parseCsvFile');

    var fs = Npm.require('fs');
    // file originally saved as public/data/taxa.csv
    var data = fs.readFileSync(process.cwd() + '/../web.browser/app/data/taxa.csv', 'utf8');
    console.log('data', data);
  }
});

KoleksiyonFS #

Ancak, depolama konusunda gerçekten ciddiyseniz ve milyonlarca görüntü depolamak istiyorsanız, Mongo’nun GridFS altyapısından yararlanmanız ve kendinize bir depolama katmanı oluşturmanız gerekecek. Bunun için mükemmel CollectionFS alt sistemine ihtiyacınız olacak.

Gerekli paketleri ekleyerek başlayın.

meteor add cfs:standard-packages
meteor add cfs:filesystem

Ve nesne modelinize bir dosya yükleme öğesi ekleme.

<template name="yourTemplate">
    <input class="your-upload-class" type="file">
</template>

Ardından istemciye bir olay denetleyicisi ekleyin.

Template.yourTemplate.events({
    'change .your-upload-class': function(event, template) {
        FS.Utility.eachFile(event, function(file) {
            var yourFile = new FS.File(file);
            yourFile.creatorId = Meteor.userId(); // add custom data
            YourFileCollection.insert(yourFile, function (err, fileObj) {
                if (!err) {
                   // do callback stuff
                }
            });
        });
    }
});

Ve koleksiyonlarınızı sunucunuzda tanımlayın:

YourFileCollection = new FS.Collection("yourFileCollection", {
    stores: [new FS.Store.FileSystem("yourFileCollection", {path: "~/meteor_uploads"})]
});
YourFileCollection.allow({
    insert: function (userId, doc) {
        return !!userId;
    },
    update: function (userId, doc) {
        return doc.creatorId == userId
    },
    download: function (userId, doc) {
        return doc.creatorId == userId
    }
});

Bu mükemmel örnek için Raz’a teşekkürler. Tüm CollectionFS’nin neler yapabileceği hakkında daha fazla ayrıntı için eksiksiz CollectionFS Belgelerine göz atmak isteyeceksiniz.
