Redis komut satırı arayüzü #

‘redis-cli’, komutların Redis’e gönderilmesine ve sunucu tarafından gönderilen yanıtların doğrudan terminalden okunmasına izin veren Redis komut satırı arabirim programıdır. Temel komut satırı kullanımı aşağıdadır:

Redis’e erişim:

$ redis-cli
127.0.0.1:6379>

Kimlik doğrulama ile redis’e erişim:

$ redis-cli -a myPassword
127.0.0.1:6379>

Veritabanını seçin ve veritabanı boyutunu gösterin (varsayılan veritabanı numarası 0’dır):

127.0.0.1:6379> dbsize
(integer) 2
127.0.0.1:6379> select 1
OK
127.0.0.1:6379[1]> dbsize
(integer) 20

Sunucu hakkında bilgi ve istatistik alın:

Docker kullanarak Redis’i yükleyin #

Docker kullanarak Redis’i kullanmaya başlamak kolaydır:

docker pull redis
docker run -p 6379:6379 --rm --name redis redis

Artık ‘6397’ bağlantı noktasında çalışan örneğiniz var

Dikkat: Redis durdurulduğunda tüm veriler silinecektir.

Redis-cli’yi bağlamak için başka bir liman işçisi başlatın:

docker run -it --link redis:redis --rm redis redis-cli -h redis -p 6379

Artık redis docker’ınızla oynayabilirsiniz.

Genel Bakış #

Redis, sorunları çözmek için bir platform oluşturmak için yüksek performans, çoğaltma ve benzersiz bir veri modeli sunan bir bellek içi uzak veritabanıdır. Redis, veritabanı, önbellek ve ileti aracısı olarak kullanılan açık kaynak kodlu (BSD lisanslı), bellek içi veri yapısıdır. NoSQL anahtar/değer deposu olarak kategorize edilir. Dizeler, karmalar, listeler, kümeler, aralık sorguları ile sıralanmış kümeler, bitmapler, hiperloglar ve yarıçap sorguları ile coğrafi uzamsal dizinler gibi veri yapılarını destekler. Beş farklı veri yapısını destekleyen,

  1. STRING (Tüm dizi, parçalar, tamsayılar ve kayanlar üzerinde çalış)
  2. LIST (Öğeleri her iki uçtan itin veya çıkarın)
  3. SET (Ekle, getir, kaldır, kontrol et, kesiştir, birleşim, fark vb.)
  4. HASH (depolayın, yağlayın, hash olarak çıkarın)
  5. ZSET (set ile aynı fakat sıralı şekilde)
  6. GEO (Enlem ve boylam ekle, güncelle, sil, verilen redius içine gir)

Redis, yerleşik çoğaltmaya, Lua komut dosyası oluşturmaya, LRU tahliyesine, işlemlere ve farklı disk üzerinde kalıcılık düzeylerine (eşitleme/zaman uyumsuz) sahiptir.

Sürüm 3’ten önce Redis, master-slave modunda çalışır ve yüksek kullanılabilirlik sağlamak için Redis-Sentinel’e ihtiyaç duyar.

Sürüm 3’ten itibaren Redis, yük devretme, parçalama/bölümleme, yeniden parçalama özelliklerinin yerleşik olduğu çoklu ana modda çalışır ve önerir. Redis-Sentinel, sürüm-3’ten gerekli değildir. Redis kümesinin çalışması için en az 3 ana düğüm/süreç gerekir.

Ek özellikler, çoğaltma, kalıcılık ve istemci tarafı parçalamadır. Redis, doğal olarak Redis’in sunduklarıyla eşleştirilebilen çok çeşitli sorunları barındırır ve diğer veritabanlarının gerektirdiği kavramsal çalışmayı gerçekleştirmek zorunda kalmadan sorunlarınızı çözmenize olanak tanır.

Redis “Merhaba Dünya” #

Öncelikle Redis sunucunuzu kurmanız ve başlatmanız gerekiyor, sunucunuza veya yerel makinenize redis yüklemenize yardımcı olabilecek aşağıdaki bağlantıyı kontrol edin.

https://www.wikiod.com/tr/redis/kurulum-ve-kurulum

Şimdi komut isteminizi açın ve redis-cli komutunu çalıştırın:

İlk seti kaydetmek için > SET ‘anahtar adı’ ve ardından ‘değer’

127.0.0.1:6379> SET hkey "Hello World!"

Enter’a basın, görmelisiniz

OK

Ardından şunu girin:

GET hkey

görmelisin:

"Hello World!"

Windows’ta Redis kurulumu, Node.js örneği ile #

Redis, ‘Microsoft Open Technologies’ tarafından sağlanan bir Windows bağlantı noktasına sahiptir. Şurada bulunan msi yükleyiciyi kullanabilirsiniz: https://github.com/MSOpenTech/redis/releases

Yükleme tamamlandıktan sonra ‘Redis’in bir Windows hizmeti olduğunu görebilirsiniz (ve durumu “Başlatıldı” olmalıdır)

Node.js’de (pencerelerde de) Redis kullanan bir ‘Merhaba dünya’ örneği yazmak için aşağıdaki npm modülünü kullanabilirsiniz: https://www.npmjs.com/package/redis

kod örneği:

var redis = require('redis'),
    client = redis.createClient();

client.set('mykey', 'Hello World');
client.get('mykey', function(err,res){
    console.log(res);
});
