Belge dizilerine programlı olarak erişme #

Belge dizileri - normal yorumların aksine - belgeledikleri işlevin bir özelliği olarak saklanır, yani bunlara programlı olarak erişebilirsiniz.

Örnek bir işlev #

def func():
    """This is a function that does nothing at all"""
    return

Belge dizisine __doc__ özelliği kullanılarak erişilebilir:

print(func.__doc__)

Bu, hiçbir şey yapmayan bir işlevdir

help(func)

‘main’ modülündeki ‘func’ işleviyle ilgili yardım:

işlev()

     Bu, hiçbir şey yapmayan bir işlevdir

Başka bir örnek işlev #

‘function.doc’, bir dize olarak yalnızca gerçek belge dizesidir, “yardım” işlevi, belge dizesi de dahil olmak üzere bir işlev hakkında genel bilgiler sağlar. İşte daha yararlı bir örnek:

def greet(name, greeting="Hello"):
    """Print a greeting to the user `name`

    Optional parameter `greeting` can change what they're greeted with."""
    
    print("{} {}".format(greeting, name))

help(greet)

‘main’ modülündeki ‘greet’ işleviyle ilgili yardım:

selamla(isim, selamlama='Merhaba')

    Kullanıcının adı'na bir karşılama mesajı yazdırın     İsteğe bağlı “tebrik” parametresi, karşılandıkları şeyi değiştirebilir.

Belge dizilerinin normal yorumlara göre avantajları #

Sadece bir fonksiyona hiçbir dokümantasyon veya normal bir yorum koymak, onu çok daha az faydalı hale getirir.

def greet(name, greeting="Hello"):
    # Print a greeting to the user `name`
    # Optional parameter `greeting` can change what they're greeted with.
    
    print("{} {}".format(greeting, name))

print(greet.__doc__)

Yok

help(greet)

main modülünde karşılama işleviyle ilgili yardım:

selamla(isim, selamlama='Merhaba')

Tek satır, satır içi ve çok satırlı yorumlar #

Yorumlar, temel kodun kendisi net olmadığında kodu açıklamak için kullanılır.

Python yorumları yok sayar ve bu nedenle orada kod yürütmez veya düz ingilizce cümleler için sözdizimi hataları oluşturmaz.

Tek satırlı yorumlar, hash karakteri (`#’) ile başlar ve ile sonlandırılır. satırın sonu.

  • Tek satırlık yorum:

    This is a single line comment in Python #

  • Satır içi yorum:

    print(“Hello World”) # This line prints “Hello World”

  • Birden çok satıra yayılan yorumların her iki ucunda da """ veya ''' bulunur. Bu, çok satırlı bir dize ile aynıdır, ancak yorum olarak kullanılabilirler:

    """ This type of comment spans multiple lines. These are mostly used for documentation of functions, classes and modules. """

Belge dizilerini kullanarak belgeler yazın #

docstring, modülleri, sınıfları, işlevleri ve yöntemleri belgelemek için kullanılan bir [çok satırlı yorum][2]‘dur. Tanımladığı bileşenin ilk ifadesi olmalıdır.

def hello(name):
    """Greet someone.

    Print a greeting ("Hello") for the person with the given name.
    """

    print("Hello "+name)

class Greeter:
    """An object used to greet people.

    It contains multiple greeting functions for several languages
    and times of the  day.
    """

Belge dizisinin değeri [programdan erişilebilir][3] ve - örneğin - ‘help’ komutu tarafından kullanılır.

Sözdizimi kuralları #

PEP 257 #

PEP 257, belge dizisi yorumları için bir sözdizimi standardı tanımlar. Temel olarak iki türe izin verir:

  • Tek satırlık Belge Dizileri:

PEP 257’ye göre kısa ve basit işlevlerle kullanılmalıdır. Her şey tek bir satıra yerleştirilir, örneğin:

def hello():
    """Say hello to your friends."""
    print("Hello my friends!")

Belge dizisi nokta ile bitmeli, fiil emir kipinde olmalıdır.

  • Çok Satırlı Belge Dizileri:

Daha uzun, daha karmaşık işlevler, modüller veya sınıflar için çok satırlı belge dizisi kullanılmalıdır.

def hello(name, language="en"):
    """Say hello to a person.

    Arguments:
    name: the name of the person
    language: the language in which the person should be greeted
    """

    print(greeting[language]+" "+name)

Tırnak işaretleri ile aynı satırda veya bir sonraki satırda olabilen kısa bir özet (tek satırlık bir belge dizisinin içeriğine eşdeğer) ile başlarlar, ek ayrıntı verir ve parametreleri listeler ve değerleri döndürürler.

Not PEP 257, bir doküman dizisi içinde [hangi bilgilerin verilmesi gerektiğini][4] tanımlar, hangi biçimde verilmesi gerektiğini tanımlamaz. Bu, diğer tarafların ve belge ayrıştırma araçlarının, bazıları aşağıda ve [bu soru][9]‘da listelenen belgeler için kendi standartlarını belirlemelerinin nedeniydi.

Sfenks #

[Sphinx][5] Python projeleri için doküman dizilerine dayalı HTML tabanlı dokümantasyon oluşturmak için bir araçtır. Kullanılan biçimlendirme dili [reStructuredText][6]‘dır. Dokümantasyon için kendi standartlarını tanımlarlar, pythonhosted.org [onların çok iyi bir tanımını][7] barındırır. Sfenks formatı örneğin [pyCharm IDE][11] tarafından kullanılır.

Sphinx/reStructuredText formatı kullanılarak bir fonksiyon şu şekilde belgelenecektir:

def hello(name, language="en"):
    """Say hello to a person.

    :param name: the name of the person
    :type name: str
    :param language: the language in which the person should be greeted
    :type language: str
    :return: a number
    :rtype: int
    """

    print(greeting[language]+" "+name)
    return 4

Google Python Stil Kılavuzu #

Google, dokümantasyon yorumları da dahil olmak üzere Python için kodlama kurallarını tanımlayan [Google Python Stil Kılavuzu][8] yayınladı. Sphinx/reST ile karşılaştırıldığında birçok kişi, Google’ın yönergelerine göre dokümantasyonun insan tarafından daha iyi okunabilir olduğunu söylüyor.

Yukarıda bahsedilen [pythonhosted.org sayfası][7], Google Stil Kılavuzuna göre iyi dokümantasyon için bazı örnekler de sağlar.

[Napolyon][10] eklentisini kullanarak Sphinx, Google Stil Rehberi ile uyumlu biçimdeki belgeleri de ayrıştırabilir.

Google Stil Kılavuzu biçimi kullanılarak bir işlev şu şekilde belgelenir:

def hello(name, language="en"):
    """Say hello to a person.

    Args:
        name: the name of the person as string
        language: the language code string

    Returns:
        A number.
    """

    print(greeting[language]+" "+name)
    return 4
