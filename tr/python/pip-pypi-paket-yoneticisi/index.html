pip, Python’un son sürümleriyle varsayılan olarak yüklenen Python Paket Dizini için en yaygın kullanılan paket yöneticisidir.

Sözdizimi #

  • pip <komut> [seçenekler] burada <komut> şunlardan biridir:
  • Yüklemek
    • Install packages
  • kaldır
    • Uninstall packages
  • donmak
    • Output installed packages in requirements format
  • liste
    • List installed packages
  • göstermek
    • Show information about installed packages
  • arama
    • Search PyPI for packages
  • teker
    • Build wheels from your requirements
  • fermuar
    • Zip individual packages (deprecated)
  • fermuarını aç
    • Unzip individual packages (deprecated)
  • paket
    • Create pybundles (deprecated)
  • Yardım Edin
    • Show help for commands

Bazen “pip”, yerel kodun manuel olarak derlenmesini gerçekleştirir. Linux’ta python, sisteminizde mevcut bir C derleyicisini otomatik olarak seçecektir. Windows’ta gerekli Visual Studio/Visual C++ sürümü için aşağıdaki tabloya bakın (daha yeni sürümler çalışmayacaktır).

Python Sürümü Visual Studio Sürümü Görsel C++ Sürümü
2.6 - 3.2 Visual Studio 2008 Görsel C++ 9.0
3.3 - 3.4 Visual Studio 2010 Görsel C++ 10.0
3.5 Visual Studio 2015 Görsel C++ 14.0

Kaynak: wiki.python.org

Paketleri Yükle #

‘SomePackage’ adlı bir paketin en son sürümünü yüklemek için:

$ pip install SomePackage

Bir paketin belirli bir sürümünü yüklemek için:

$ pip install SomePackage==1.0.4

Bir paket için yüklenecek minimum sürümü belirtmek için:

$ pip install SomePackage>=1.0.4

Linux/Unix’te komutlar izin reddedildi hatası gösteriyorsa, komutlarla birlikte “sudo” kullanın.

Gereksinim dosyalarından yükleyin #

$ pip install -r requirements.txt

Gereksinim dosyasının her satırı, kurulacak bir şeyi belirtir ve pip kurulumuna ilişkin argümanlar gibi, Dosyaların biçimiyle ilgili ayrıntılar burada: Gereksinim Dosyası Biçimi.

Paketi kurduktan sonra freeze komutunu kullanarak kontrol edebilirsiniz:

$ pip freeze

pip kullanılarak kurulan tüm paketleri listelemek için #

Kurulu paketleri listelemek için:

$ pip list
# example output
docutils (0.9.1)
Jinja2 (2.6)
Pygments (1.5)
Sphinx (1.1.2)

Eski paketleri listelemek ve mevcut en son sürümü göstermek için:

$ pip list --outdated
# example output
docutils (Current: 0.9.1 Latest: 0.10)
Sphinx (Current: 1.1.2 Latest: 1.1.3)

Yükseltme Paketleri #

Koşma

$ pip install --upgrade SomePackage

“SomePackage” paketini ve tüm bağımlılıklarını yükseltecektir. Ayrıca, pip, yükseltmeden önce paketin eski sürümünü otomatik olarak kaldırır.

Pip’in kendisini yükseltmek için şunu yapın:

$ pip install --upgrade pip

Unix’te veya

$ python -m pip install --upgrade pip

Windows makinelerinde.

Paketleri Kaldır #

Bir paketi kaldırmak için:

$ pip uninstall SomePackage

Linux’ta tüm güncel olmayan paketleri güncelleme #

pip, kullanıcının tüm güncel olmayan paketleri tek seferde güncellemesine izin verecek bir bayrak içermiyor. Ancak bu, komutları bir Linux ortamında birbirine bağlayarak gerçekleştirilebilir:

pip list --outdated --local | grep -v '^\-e' | cut -d = -f 1  | xargs -n1 pip install -U

Bu komut yerel sanal ortamdaki tüm paketleri alır ve eski olup olmadıklarını kontrol eder. Bu listeden paket adını alır ve ardından bunu bir pip install -U komutuna yönlendirir. Bu işlemin sonunda tüm yerel paketler güncellenmelidir.

Windows’ta tüm güncel olmayan paketleri güncelleme #

pip, kullanıcının tüm güncel olmayan paketleri tek seferde güncellemesine izin verecek bir bayrak içermiyor. Ancak bu, komutları bir Windows ortamında birbirine bağlayarak gerçekleştirilebilir:

for /F "delims= " %i in ('pip list --outdated --local') do pip install -U %i

Bu komut yerel sanal ortamdaki tüm paketleri alır ve eski olup olmadıklarını kontrol eder. Bu listeden paket adını alır ve ardından bunu bir pip install -U komutuna yönlendirir. Bu işlemin sonunda tüm yerel paketler güncellenmelidir.

Sistemdeki tüm paketlerin bir gereksinimi.txt dosyası oluşturun #

‘pip’, ‘dondur’ seçeneğini sağlayarak ‘requirements.txt’ dosyalarının oluşturulmasına yardımcı olur.

pip freeze > requirements.txt

Bu, sistemde yüklü olan tüm paketlerin ve sürümlerinin bir listesini mevcut klasördeki requirements.txt adlı bir dosyaya kaydeder.

Yalnızca geçerli sanalenv’de paketlerin bir gereksinimi.txt dosyası oluşturun #

‘pip’, ‘dondur’ seçeneğini sağlayarak ‘requirements.txt’ dosyalarının oluşturulmasına yardımcı olur.

pip freeze --local > requirements.txt

--local parametresi yalnızca bir sanal ortama yerel olarak yüklenen paketlerin ve sürümlerin bir listesini verir. Global paketler listelenmeyecektir.

Belirli bir Python sürümünü pip ile kullanma #

Hem Python 3 hem de Python 2 kuruluysa, pip’in hangi Python sürümünü kullanmasını istediğinizi belirtebilirsiniz. Bu, paketler yalnızca Python 2 veya 3’ü desteklediğinde veya her ikisiyle de test etmek istediğinizde kullanışlıdır.

Python 2 için paketler kurmak istiyorsanız, aşağıdakilerden birini çalıştırın:

pip install [package]

veya:

pip2 install [package]

Python 3 için paketler kurmak istiyorsanız şunları yapın:

pip3 install [package]

Ayrıca, aşağıdakilerle belirli bir python kurulumuna bir paketin kurulumunu çağırabilirsiniz:

\path\to\that\python.exe -m pip install some_package # on Windows OR
/usr/bin/python25 -m pip install some_package # on OS-X/Linux

OS-X/Linux/Unix platformlarında, python’un sistem sürümü (ki yükseltme, sisteminizi çalışamaz hale getirir) ile python’un kullanıcı sürümleri arasındaki ayrımın farkında olmak önemlidir. Hangisini yükseltmeye çalıştığınıza bağlı olarak, bu komutların önüne sudo eklemeniz ve bir şifre girmeniz gerekebilir.

Aynı şekilde Windows’ta bazı python kurulumları, özellikle başka bir paketin parçası olanlar, sistem dizinlerine kurulabilir - Yönetici modunda çalışan bir komut penceresinden yükseltmeniz gerekecek olanlar - ihtiyacınız gibi görünüyorsa bunu yapın, hangi python kurulumunu yükseltmeye çalıştığınızı python -c"import sys;print(sys.path);" veya py -3.5 gibi bir komutla kontrol etmek **çok** iyi bir fikirdir - c"import sys;print(sys.path);" hangi pip’i çalıştırmaya çalıştığınızı pip --version ile de kontrol edebilirsiniz.

Windows’ta, hem python 2 hem de python 3 yüklüyse ve yolunuz üzerinde ve python 3’ünüz 3.4’ten büyükse, muhtemelen sistem yolunuzda python başlatıcısı ‘py’ olacaktır. Daha sonra aşağıdaki gibi hileler yapabilirsiniz:

py -3 -m pip install -U some_package # Install/Upgrade some_package to the latest python 3
py -3.3 -m pip install -U some_package # Install/Upgrade some_package to python 3.3 if present
py -2 -m pip install -U some_package # Install/Upgrade some_package to the latest python 2 - 64 bit if present
py -2.7-32 -m pip install -U some_package # Install/Upgrade some_package to python 2.7 - 32 bit if present

Birden fazla python sürümünü çalıştırıyorsanız ve bakımını yapıyorsanız, hem python sürümünü hem de hangi paketlerin mevcut olduğunu yalıtmanıza izin veren python ‘virtualenv’ veya ‘venv’ sanal ortamlar hakkında bilgi edinmenizi şiddetle tavsiye ederim.

Henüz pip üzerinde olmayan paketleri tekerlek olarak yükleme #

Pek çok saf python paketi henüz Python Paket Dizini’nde tekerlek olarak mevcut değildir, ancak yine de iyi bir şekilde kurulur. Ancak, Windows’taki bazı paketler korkunç vcvarsall.bat bulunamadı hatası veriyor.

Sorun, yüklemeye çalıştığınız paketin bir C veya C++ uzantısı içermesi ve şu anda python paket dizininden, pypi önceden oluşturulmuş bir tekerlek olarak mevcut olmaması ve pencerelerde bu araca sahip olmamasıdır. Bu tür öğeleri oluşturmak için gerekli zincir.

En basit yanıt, Christoph Gohlke’nin mükemmel sitesine gitmek ve ihtiyacınız olan kitaplıkların uygun sürümünü bulmaktır. Paket adına uygun olarak, bir -cpNN**-** python sürümünüzle eşleşmelidir, yani Windows 32 bit python * kullanıyorsanız, win64* üzerinde bile ad -win32- içerir ve 64 bit python kullanılıyorsa -win_amd64- içermelidir ve ardından python sürümü eşleşmelidir, yani Python 34 için dosya adı -cp içermelidir* *34- vb. Bu temelde pip’in pypi sitesinde sizin için yaptığı sihirdir.

Alternatif olarak, kullandığınız python sürümü için uygun Windows geliştirme kitini, arabirim oluşturmaya çalıştığınız paketin herhangi bir kitaplığının başlıklarını, muhtemelen python sürümünün python başlıklarını vb. almanız gerekir.

Python 2.7, Visual Studio 2008’i kullandı, Python 3.3 ve 3.4, Visual Studio 2010’u kullandı ve Python 3.5+, Visual Studio 2015’i kullandı.

  • Microsoft’un web sitesinden veya edinilebilen “Visual C++ Compiler Package for Python 2.7”yi kurun
  • Microsoft’un web sitesinden veya edinilebilen “Windows 7 ve .NET Framework 4 için Windows SDK” (v7.1) yükleyin
  • Visual Studio 2015 Community Edition, (ya da bunlar yayınlandığında daha sonraki herhangi bir sürüm) yükleyin, C & C++ desteğini yükleme seçeneklerini belirlediğinizden emin olun artık varsayılan değil -* Bunun indirilip yüklenmesinin* 8 saat kadar sürebileceği söylendi, bu yüzden bu seçeneklerin ilk denemede ayarlandığından emin olun.

O zaman, istediğiniz paketin bağlantı verdiği tüm kitaplıklar için eşleşen revizyonda başlık dosyalarını bulmanız gerekebilir ve bunları uygun konumlara indirin.

Son olarak yapınızı pip’e bırakabilirsiniz - tabii ki pakette henüz sahip olmadığınız bağımlılıklar varsa, onlar için başlık dosyalarını da bulmanız gerekebilir.

Alternatifler: Ayrıca, hem pypi’de hem de Christop’un sitesinde, aradığınız paketin saf python veya platformunuz ve python sürümünüz için önceden oluşturulmuş biraz daha eski sürümlerine göz atmaya değer. ve muhtemelen bulunursa, paketiniz hazır olana kadar bunları kullanın. Aynı şekilde, python’un en son sürümünü kullanıyorsanız, paket bakımcılarının yetişmesinin biraz zaman aldığını görebilirsiniz, bu nedenle gerçekten belirli bir pakete ihtiyacı olan projeler için biraz daha eski bir python kullanmanız gerekebilir. an. Ayrıca önceden oluşturulmuş veya saf python olarak kullanılabilen çatallı bir sürüm olup olmadığını görmek için paketlerin kaynak sitesini kontrol edebilir ve ihtiyacınız olan ancak mevcut olan işlevselliği sağlayan alternatif paketleri arayabilirsiniz - akla gelen bir örnek şudur: Yastık, aktif olarak korunur, PIL için değişim düşüşü şu anda 6 yıldır güncellenmiyor ve python 3 için mevcut değil.

Sonsöz, bu sorunu yaşayan herkesi paket için hata izleyiciye gitmeye ve bir bilet eklemeye ya da zaten yoksa yükseltmeye teşvik ediyorum kibarca paket sahiplerinden talepte bulunun. özel platform ve python kombinasyonunuz için pypi’de bir tekerlek sağlayın, bu yapılırsa normalde işler zamanla daha iyi hale gelir, bazı paket sahipleri, insanların kullanıyor olabileceği belirli bir kombinasyonu kaçırdıklarını fark etmezler.

Ön Sürümleri Yüklemeye İlişkin Not #

Pip, Semantik Sürüm Oluşturma kurallarına uyar ve varsayılan olarak yayınlanmış paketleri ön yayınlara tercih eder. Bu nedenle, belirli bir paket ‘V0.98’ olarak yayınlandıysa ve ayrıca ‘V1.0-rc1’ sürüm adayı varsa, ‘pip kurulumu’nun varsayılan davranışı ‘V0.98’ yüklemek olacaktır - isterseniz sürüm adayını kurun, önce sanal bir ortamda test etmeniz önerilir, bunu --pip install --pre paket-adı veya --pip install --pre --upgrade ile etkinleştirebilirsiniz paket adı. Çoğu durumda, ön sürümler veya sürüm adayları, tüm platform ve sürüm kombinasyonları için oluşturulmuş tekerleklere sahip olmayabilir, bu nedenle yukarıdaki sorunlarla karşılaşmanız daha olasıdır.

Geliştirme Sürümlerini Yüklemeyle İlgili Not #

Ayrıca, github ve diğer konumlardan paketlerin geliştirme sürümlerini yüklemek için pip kullanabilirsiniz, çünkü bu tür kodlar akış halinde olduğundan, bunun için tekerleklerin inşa edilmesi pek olası değildir, bu nedenle herhangi bir saf olmayan paket, oluşturma araçlarının varlığını gerektirecektir ve bunlar olabilir. herhangi bir zamanda bozulabilir, bu nedenle kullanıcının bu tür paketleri yalnızca sanal bir ortama kurması şiddetle teşvik edilir.

Bu manuel olarak indirilebilir ve ardından pip install path/to/downloaded/file ile kurulabilir, unutmayın çoğu sıkıştırma formatı için pip, bir önbellek alanına vb.

Depoyu ‘git’, ‘mercurial’ veya diğer kabul edilebilir araçlar, tercihen bir DVCS aracı kullanarak klonlayın ve ‘pip install’ path/to/cloned/repo kullanın - bu her ikisi de herhangi bir şeyi işler require.text dosyasını açın ve derleme ve kurulum adımlarını gerçekleştirin, dizini klonlanmış deponuza manuel olarak değiştirebilir ve aynı efekti elde etmek için pip install -r require.txt ve ardından python setup.py install komutunu çalıştırabilirsiniz. bu yaklaşımın büyük avantajları, ilk klonlama işleminin anlık görüntü indirme işleminden daha uzun sürebilmesine rağmen, git: git pull Origin master durumunda en son güncellemeye güncelleyebilirsiniz ve mevcut sürümde hatalar varsa pip uninstall paket-adı, ardından depo geçmişinde önceki sürüm(ler)e geri dönmek için git checkout komutlarını kullanın ve yeniden deneyin.
