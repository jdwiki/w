Python’daki işlevler, bir dizi belirli eylemi gerçekleştirmek için organize, yeniden kullanılabilir ve modüler kod sağlar. Fonksiyonlar kodlama sürecini basitleştirir, gereksiz mantığı önler ve kodun takip edilmesini kolaylaştırır. Bu konu, Python’da işlevlerin bildirilmesini ve kullanımını açıklar.

Python’da print(), input(), len() gibi birçok yerleşik işlev vardır. Yerleşiklerin yanı sıra daha özel işler yapmak için kendi işlevlerinizi de oluşturabilirsiniz; bunlara kullanıcı tanımlı işlevler denir.

Sözdizimi #

  • def işlev_adı(arg1, … argN, *args, kw1, kw2=varsayılan, …, **kwargs): ifadeler
  • lambda arg1, … argN, *args, kw1, kw2=varsayılan, …, **kwargs: ifade

Parametreler #

parametre Ayrıntılar
arg1, …, argN Normal argümanlar
*arg Adsız konumsal bağımsız değişkenler
kw1, …, kwN Yalnızca anahtar kelime bağımsız değişkenleri
**kwargs Anahtar kelime argümanlarının geri kalanı

İşlevlerle yapabileceğiniz 5 temel şey:

  • Değişkenlere işlev atama

    def f():
  print(20)
y = f
y()
# Output: 20

  • İşlevleri diğer işlevler içinde tanımlayın (https://www.wikiod.com/tr/python/fonksiyonlar#iç içe işlevler )

    def f(a, b, y):
    def inner_add(a, b):      # inner_add is hidden from outer code
        return a + b
    return inner_add(a, b)**y

  • Fonksiyonlar diğer fonksiyonları döndürebilir

    def f(y):
    def nth_power(x):
        return x ** y
    return nth_power    # returns a function

squareOf = f(2)         # function that returns the square of a number           
cubeOf = f(3)           # function that returns the cube of a number
squareOf(3)             # Output: 9
cubeOf(2)               # Output: 8

  • Fonksiyonlar diğer fonksiyonlara parametre olarak aktarılabilir

    def a(x, y):
    print(x, y)
def b(fun, str):        # b has two arguments: a function and a string 
    fun('Hello', str)
b(a, 'Sophia')           # Output: Hello Sophia

  • İç işlevlerin ek kapsama erişimi vardır (https://www.wikiod.com/tr/python/fonksiyonlar#kapatma )

    def outer_fun(name):
    def inner_fun():     # the variable name is available to the inner function
        return "Hello "+ name + "!"
    return inner_fun
greet = outer_fun("Sophia")
print(greet())            # Output: Hello Sophia!

İsteğe bağlı sayıda bağımsız değişken içeren bir işlev tanımlama #

İsteğe bağlı konumsal argüman sayısı: #

İsteğe bağlı sayıda argüman alabilen bir fonksiyon tanımlama, argümanlardan birinin önüne * koyarak yapılabilir.

def func(*args):
    # args will be a tuple containing all values that are passed in
    for i in args:
        print(i)

func(1, 2, 3)  # Calling it with 3 arguments
# Out: 1
#      2
#      3

list_of_arg_values = [1, 2, 3]
func(*list_of_arg_values)  # Calling it with list of values, * expands the list
# Out: 1
#      2
#      3 

func()  # Calling it without arguments
# No Output

‘args’ için bir varsayılan sağlayamazsınız, örneğin ‘func(*args=[1, 2, 3])’ bir sözdizimi hatasına neden olur (derleme bile yapmaz).

İşlevi çağırırken bunları ada göre sağlayamazsınız, örneğin “func(*args=[1, 2, 3])” bir “TypeError” oluşturur.

Ancak argümanlarınız zaten bir dizide (veya başka herhangi bir ‘Yinelenebilir’) varsa, işlevinizi şu şekilde çarpabilirsiniz: ‘func(*my_stuff)’.

Bu argümanlara (*args) dizin ile erişilebilir, örneğin args[0] ilk argümanı döndürür

Keyfi sayıda anahtar kelime argümanı #

Önünde iki * ile tanımda bir argüman tanımlayarak, bir isimle rasgele sayıda argüman alabilirsiniz:

def func(**kwargs):
    # kwargs will be a dictionary containing the names as keys and the values as values
    for name, value in kwargs.items():
        print(name, value)

func(value1=1, value2=2, value3=3)   # Calling it with 3 arguments
# Out: value1 1
#      value2 2
#      value3 3

func()                               # Calling it without arguments
# No Out put

my_dict = {'foo': 1, 'bar': 2}
func(**my_dict)                      # Calling it with a dictionary
# Out: foo 1
#      bar 2

Bu adsız adları sağlayamazsınız, örneğin “func(1, 2, 3)” bir “TypeError” oluşturacaktır.

kwargs sade bir yerel python sözlüğüdür. Örneğin, ‘args[‘değer1’]’, ‘değer1’ bağımsız değişkeni için değer verecektir. Böyle bir argüman olup olmadığını önceden kontrol ettiğinizden emin olun, aksi takdirde bir ‘KeyError’ ortaya çıkacaktır.

Uyarı #

Bunları diğer isteğe bağlı ve gerekli argümanlarla karıştırabilirsiniz, ancak tanımın içindeki sıra önemlidir.

konumsal/anahtar kelime bağımsız değişkenleri önce gelir. (Gerekli argümanlar). Ardından keyfi ‘*arg’ argümanları gelir. (İsteğe bağlı). Ardından yalnızca anahtar kelime bağımsız değişkenleri gelir. (Gerekli). Sonunda keyfi anahtar kelime **kwargs gelir. (İsteğe bağlı).

#       |-positional-|-optional-|---keyword-only--|-optional-|
def func(arg1, arg2=10 , *args, kwarg1, kwarg2=2, **kwargs):
     pass
  • ‘arg1’ verilmelidir, aksi takdirde bir ‘TypeError’ ortaya çıkar. Konumsal (func(10)) veya anahtar kelime argümanı (func(arg1=10)) olarak verilebilir.
  • kwarg1 de verilmelidir, ancak sadece anahtar kelime-argümanı olarak sağlanabilir: func(kwarg1=10).
  • “arg2” ve “kwarg2” isteğe bağlıdır. Değer değiştirilecekse, “arg1” (konumsal veya anahtar sözcük) ve “kwarg1” (yalnızca anahtar sözcük) için geçerli olan aynı kurallar geçerlidir.
  • *args ek konumsal parametreleri yakalar. Ancak, argümanları “*args”: “func(1, 1, 1, 1)” öğesine iletmek için “arg1” ve “arg2"nin konumsal bağımsız değişkenler olarak sağlanması gerektiğini unutmayın.
  • **kwargs tüm ek anahtar kelime parametrelerini yakalar. Bu durumda “arg1”, “arg2”, “kwarg1” veya “kwarg2” olmayan herhangi bir parametre. Örneğin: “fonk(kwarg3=10)”.
  • Python 3’te, sonraki tüm argümanların anahtar kelimeler olarak belirtilmesi gerektiğini belirtmek için tek başına * kullanabilirsiniz. Örneğin Python 3.5 ve sonraki sürümlerinde ‘math.isclose’ işlevi ‘def math.isclose (a, b, *, rel_tol=1e-09, abs_tol=0.0)’ kullanılarak tanımlanır, bu da ilk iki argümanın sağlanabileceği anlamına gelir. konumsal olarak, ancak isteğe bağlı üçüncü ve dördüncü parametreler yalnızca anahtar sözcük bağımsız değişkenleri olarak sağlanabilir.

Python 2.x, yalnızca anahtar sözcük parametrelerini desteklemez. Bu davranış “kwargs” ile taklit edilebilir:

def func(arg1, arg2=10, **kwargs):
    try:
        kwarg1 = kwargs.pop("kwarg1")
    except KeyError:
        raise TypeError("missing required keyword-only argument: 'kwarg1'")

    kwarg2 = kwargs.pop("kwarg2", 2)
    # function body ...

Adlandırmaya İlişkin Not #

İsteğe bağlı konumsal bağımsız değişkenleri ‘args’ ve isteğe bağlı anahtar sözcük bağımsız değişkenlerini ‘kwargs’ olarak adlandırma kuralı, yalnızca sizin istediğiniz herhangi bir adı kullanabileceğiniz bir kuraldır ancak diğerlerinin sizin ne istediğinizi bilmesi için bu kuralı takip etmek yararlıdır. yapıyor, hatta daha sonra kendiniz bile lütfen yapın.

Benzersizlik Üzerine Not #

Herhangi bir işlev none veya one *args ve none veya one **kwargs ile tanımlanabilir, ancak her birinin birden fazlası ile tanımlanamaz. Ayrıca *args son konumsal argüman olmalı ve **kwargs son parametre olmalıdır. **Bu ikisinden birden fazlasını kullanmaya çalışmak, Sözdizimi Hatası istisnasına neden olur.

İsteğe Bağlı Bağımsız Değişkenlerle İç İçe Yerleştirme İşlevleri Hakkında Not #

Bu tür işlevleri iç içe yerleştirmek mümkündür ve genel kural, kodun zaten işlediği öğeleri kaldırmaktır ancak parametreleri aktarıyorsanız, bir * öneki ve isteğe bağlı anahtar kelime ile isteğe bağlı konumsal argümanları iletmeniz gerekir. ‘**’ öneki ile argümanlar, aksi takdirde bir liste veya demet olarak iletilecek olan argümanlar ve tek bir sözlük olarak kwargs. Örneğin.:

def fn(**kwargs):
    print(kwargs)
    f1(**kwargs)

def f1(**kwargs):
    print(len(kwargs))

fn(a=1, b=2)
# Out:
# {'a': 1, 'b': 2}
# 2

Basit işlevleri tanımlama ve çağırma #

Python’da bir işlevi tanımlamanın en yaygın yolu ‘def’ ifadesini kullanmaktır. Bu deyim, aşağıdaki sözdizimine sahip tek yan tümceli bileşik deyim olarak adlandırılır:

def function_name(parameters):
    statement(s)

işlev_adı, işlevin tanımlayıcısı olarak bilinir. Bir işlev tanımı yürütülebilir bir ifade olduğundan, yürütülmesi işlev adını, daha sonra tanımlayıcı kullanılarak çağrılabilecek olan işlev nesnesine bağlar.

parameters, işlev çağrıldığında argüman olarak sağlanan değerlere bağlanan isteğe bağlı bir tanımlayıcı listesidir. Bir işlev, virgülle ayrılmış rastgele sayıda bağımsız değişkene sahip olabilir.

ifade(ler) – işlev gövdesi olarak da bilinir – fonksiyon her çağrıldığında yürütülen boş olmayan bir ifade dizisidir. Bu, herhangi bir [girintili blok](https://www.wikiod.com/tr/python/python-dili-ile-baslarken#Girintiyi Engelle) gibi bir işlev gövdesinin boş olamayacağı anlamına gelir.

Her çağrıldığında “Merhaba"yı yazdırmayı amaçlayan basit bir işlev tanımı örneği aşağıda verilmiştir:

def greet():
    print("Hello")

Şimdi tanımlı greet() fonksiyonunu çağıralım:

greet()
# Out: Hello

Bu, tek bir argüman alan ve işlev her çağrıldığında iletilen değeri görüntüleyen bir işlev tanımının başka bir örneğidir:

def greet_two(greeting):
    print(greeting)

Bundan sonra ‘greet_two()’ işlevi bir argümanla çağrılmalıdır:

greet_two("Howdy")
# Out: Howdy

Ayrıca bu işlev argümanına varsayılan bir değer verebilirsiniz:

def greet_two(greeting="Howdy"):
    print(greeting)

Artık bir değer vermeden fonksiyonu çağırabilirsiniz:

greet_two()
# Out: Howdy

Diğer birçok dilden farklı olarak, işlevin dönüş türünü açıkça belirtmeniz gerekmediğini fark edeceksiniz. Python işlevleri, “return” anahtar sözcüğü aracılığıyla herhangi bir türden değer döndürebilir. Bir işlev, herhangi bir sayıda farklı tür döndürebilir!

def many_types(x):
    if x < 0:
        return "Hello!"
    else:
        return 0

print(many_types(1))
print(many_types(-1))

# Output:
0
Hello!

Bu, arayan tarafından doğru bir şekilde ele alındığı sürece, bu tamamen geçerli Python kodudur.

Bir return ifadesi olmadan yürütmenin sonuna ulaşan bir işlev her zaman ‘Yok’ döndürür:

def do_nothing():
    pass

print(do_nothing())
# Out: None

Daha önce bahsedildiği gibi, bir fonksiyon tanımı bir fonksiyon gövdesine, boş olmayan bir ifade dizisine sahip olmalıdır. Bu nedenle, ‘pass’ ifadesi, boş bir işlem olan işlev gövdesi olarak kullanılır - yürütüldüğünde hiçbir şey olmaz. Demek istediğini yapar, atlar. Sözdizimsel olarak bir ifade gerektiğinde, ancak hiçbir kodun yürütülmesi gerekmediğinde, yer tutucu olarak kullanışlıdır.

Lambda (Satır İçi/Anonim) İşlevleri #

“lambda” anahtar sözcüğü, tek bir ifade içeren satır içi bir işlev oluşturur. Bu ifadenin değeri, çağrıldığında işlevin döndürdüğü değerdir.

İşlevi düşünün:

def greeting():
    return "Hello"

hangi, olarak çağrıldığında:

print(greeting())

baskılar:

Hello

Bu, aşağıdaki gibi bir lambda işlevi olarak yazılabilir:

greet_me = lambda: "Hello"

Lambdaların değişkenlere atanmasıyla ilgili olarak bu bölümün altındaki nota bakın. Genel olarak, yapmayın.

Bu, “Merhaba” döndüren “greet_me” adında bir satır içi işlev oluşturur. Lambda ile fonksiyon oluştururken ‘return’ yazmadığınızı unutmayın. : sonrasındaki değer otomatik olarak döndürülür.

Bir değişkene atandığında, normal bir işlev gibi kullanılabilir:

print(greet_me())

baskılar:

Hello

lambdalar da argüman alabilir:

strip_and_upper_case = lambda s: s.strip().upper()

strip_and_upper_case("  Hello   ")

dizeyi döndürür:

HELLO

Ayrıca normal işlevler gibi isteğe bağlı sayıda argüman / anahtar kelime argümanı alabilirler.

greeting = lambda x, *args, **kwargs: print(x, args, kwargs)
greeting('hello', 'world', world='world')

baskılar:

hello ('world',) {'world': 'world'}

“lambda"lar, çağrıldıkları noktada (genellikle “sıralı”, “filtre” ve “harita” ile) tanımlanması kolay olan kısa işlevler için yaygın olarak kullanılır.

Örneğin, bu satır, büyük/küçük harflerini yok sayan ve başlangıçtaki ve sonundaki boşlukları yok sayan bir dizeler listesini sıralar:

sorted( [" foo ", "    bAR", "BaZ    "], key=lambda s: s.strip().upper())
# Out:
# ['    bAR', 'BaZ    ', ' foo ']

Listeyi yalnızca boşlukları yok sayarak sıralayın:

sorted( [" foo ", "    bAR", "BaZ    "], key=lambda s: s.strip())
# Out:
# ['BaZ    ', '    bAR', ' foo ']

“harita” içeren örnekler:

sorted( map( lambda s: s.strip().upper(), [" foo ", "    bAR", "BaZ    "]))
# Out:
# ['BAR', 'BAZ', 'FOO']

sorted( map( lambda s: s.strip(), [" foo ", "    bAR", "BaZ    "]))
# Out:
# ['BaZ', 'bAR', 'foo']

Sayısal listelerle örnekler:

my_list = [3, -4, -2, 5, 1, 7]
sorted( my_list, key=lambda x: abs(x))
# Out:
# [1, -2, 3, -4, 5, 7]

list( filter( lambda x: x>0, my_list))
# Out:
# [3, 5, 1, 7]

list( map( lambda x: abs(x), my_list))
# Out:
[3, 4, 2, 5, 1, 7]

Bir lambda işlevinin içinden başka işlevler (argümanlı/argümansız) çağrılabilir.

def foo(msg):
    print(msg)

greet = lambda x = "hello world": foo(x)
greet()

baskılar:

hello world

Bu yararlıdır çünkü ’lambda’ yalnızca bir ifade içerebilir ve bir yardımcı işlev kullanılarak birden çok ifade çalıştırılabilir.

NOT

PEP-8‘in (resmi Python stil kılavuzu) değişkenlere lambda atamayı önermediğini unutmayın (ilk iki örnekte yaptığımız gibi):

Bir lambda ifadesini doğrudan bir tanımlayıcıya bağlayan atama ifadesi yerine her zaman bir def ifadesi kullanın.

Evet:

def f(x): 2*x döndür Hayır:

f = lambda x: 2*x İlk biçim, sonuçtaki işlev nesnesinin adının genel “” yerine özel olarak “f” olduğu anlamına gelir. Bu, genel olarak geri izlemeler ve dize temsilleri için daha kullanışlıdır. Atama ifadesinin kullanılması, bir lambda ifadesinin açık bir “def” ifadesi üzerinden sunabileceği tek yararı ortadan kaldırır (yani, daha büyük bir ifadenin içine gömülebilir).

İsteğe bağlı bağımsız değişkenlerle bir işlev tanımlama #

İsteğe bağlı bağımsız değişkenler, bağımsız değişken adına varsayılan bir değer atanarak (= kullanılarak) tanımlanabilir:

def make(action='nothing'):
    return action

Bu işlevi çağırmak 3 farklı şekilde mümkündür:

make("fun")
# Out: fun

make(action="sleep")
# Out: sleep

# The argument is optional so the function will use the default value if the argument is 
# not passed in.
make()   
# Out: nothing

Uyarı #

İsteğe bağlı değişken argümanlarla bir işlev tanımlama #

Değişken varsayılan tür ile isteğe bağlı bağımsız değişkenler kullanılırken bir sorun var ([İsteğe bağlı bağımsız değişkenlerle işlev tanımlama](https://www.wikiod.com/tr/python/fonksiyonlar bölümünde açıklanmıştır) /defining-a-function-with-opsiyonel-arguments)), potansiyel olarak beklenmedik davranışlara yol açabilir.

Açıklama #

Bu sorun, bir işlevin varsayılan argümanlarının, işlev tanımlı olduğunda bir kez ve işlev çağrıldığında değil (diğer birçok dilde olduğu gibi) başlatılmış olması nedeniyle ortaya çıkar. Varsayılan değerler, işlev nesnesinin __defaults__ üye değişkeninde saklanır.

def f(a, b=42, c=[]):
    pass

print(f.__defaults__)
# Out: (42, [])

değişmez türleri için (bkz. Argüman geçişi ve değişebilirlik) bu bir sorun değildir çünkü değişkeni değiştirmenin bir yolu yoktur; orijinal değeri değişmeden bırakılarak yalnızca yeniden atanabilir. Bu nedenle, müteakiplerin aynı varsayılan değere sahip olması garanti edilir. Ancak, değişken bir tür için, orijinal değer, çeşitli üye işlevlerine çağrılar yaparak değişebilir. Bu nedenle, işleve yapılan ardışık çağrıların ilk varsayılan değere sahip olması garanti edilmez.

def append(elem, to=[]):
    to.append(elem)      # This call to append() mutates the default variable "to"
    return to

append(1)
# Out: [1]

append(2)  # Appends it to the internally stored list
# Out: [1, 2]

append(3, [])  # Using a new created list gives the expected result
# Out: [3]

# Calling it again without argument will append to the internally stored list again
append(4)   
# Out: [1, 2, 4]

Not: PyCharm gibi bazı IDE’ler, varsayılan öznitelik olarak değişken bir tür belirtildiğinde bir uyarı verir.

Çözüm #

Varsayılan argümanın her zaman fonksiyon tanımında belirttiğiniz argüman olduğundan emin olmak istiyorsanız, o zaman çözüm, varsayılan argümanınız olarak daima değişmez bir tür kullanmaktır.

Varsayılan olarak değiştirilebilir bir tür gerektiğinde bunu başarmak için yaygın bir deyim, varsayılan argüman olarak ‘Yok’u (değişmez) kullanmak ve ardından ‘Yok’a eşitse, gerçek varsayılan değeri bağımsız değişken değişkenine atamaktır.

def append(elem, to=None):
    if to is None:
        to = []

    to.append(elem)
    return to

Argüman geçişi ve değişkenlik #

İlk olarak, bazı terminoloji:

  • argüman (gerçek parametre): bir işleve geçirilen gerçek değişken;
  • parametre (formal parametre): bir fonksiyonda kullanılan alıcı değişken.

Python’da argümanlar assignment tarafından iletilir (argümanların değer/referans/işaretçi ile iletilebildiği diğer dillerin aksine).

  • Bir parametreyi değiştirmek, argümanı değiştirir (eğer argümanın türü değişebilir ise).

    def foo(x):        # here x is the parameter
    x[0] = 9       # This mutates the list labelled by both x and y
    print(x)

y = [4, 5, 6]
foo(y)             # call foo with y as argument
# Out: [9, 5, 6]   # list labelled by x has been mutated
print(y)           
# Out: [9, 5, 6]   # list labelled by y has been mutated too

  • Parametreyi yeniden atamak, bağımsız değişkeni yeniden atamaz.

    def foo(x):        # here x is the parameter, when we call foo(y) we assign y to x
    x[0] = 9       # This mutates the list labelled by both x and y
    x = [1, 2, 3]  # x is now labeling a different list (y is unaffected)
    x[2] = 8       # This mutates x's list, not y's list

y = [4, 5, 6]      # y is the argument, x is the parameter
foo(y)             # Pretend that we wrote "x = y", then go to line 1
y
# Out: [9, 5, 6]

Python’da değişkenlere gerçekten değer atamayız, bunun yerine nesnelere değişkenleri (names olarak kabul edilir) bind (yani atama, ekleme) yaparız.

  • Değişmez: Tamsayılar, dizeler, demetler vb. Tüm işlemler kopya oluşturur.

  • Değişken: Listeler, sözlükler, kümeler vb. İşlemler değişebilir veya değişmeyebilir.

    x = [3, 1, 9] y = x x.append(5) # Mutates the list labelled by x and y, both x and y are bound to [3, 1, 9] x.sort() # Mutates the list labelled by x and y (in-place sorting) x = x + [4] # Does not mutate the list (makes a copy for x only, not y) z = x # z is x ([1, 3, 9, 4]) x += [6] # Mutates the list labelled by both x and z (uses the extend function). x = sorted(x) # Does not mutate the list (makes a copy for x only). x

    Out: [1, 3, 4, 5, 6, 9] #

    y

    Out: [1, 3, 5, 9] #

    z

    Out: [1, 3, 5, 9, 4, 6] #

Fonksiyonlardan değer döndürme #

İşlevler, doğrudan kullanabileceğiniz bir değer “döndürebilir”:

def give_me_five():
    return 5

print(give_me_five())  # Print the returned value
# Out: 5

veya değeri daha sonra kullanmak üzere kaydedin:

num = give_me_five()
print(num)             # Print the saved returned value
# Out: 5

veya herhangi bir işlem için değeri kullanın:

print(give_me_five() + 10)
# Out: 15

Eğer fonksiyonda “return” ile karşılaşılırsa fonksiyondan hemen çıkılır ve sonraki işlemler değerlendirilmez:

def give_me_another_five():
    return 5
    print('This statement will not be printed. Ever.')

print(give_me_another_five())
# Out: 5

Ayrıca birden çok değeri (bir demet biçiminde) “döndürebilirsiniz”:

def give_me_two_fives():
    return 5, 5  # Returns two 5

first, second = give_me_two_fives()
print(first)
# Out: 5
print(second)
# Out: 5

no “return” ifadesine sahip bir işlev, örtük olarak “Yok” döndürür. Benzer şekilde, ‘return’ ifadesi olan, ancak dönüş değeri veya değişkeni olmayan bir işlev ‘Yok’ döndürür.

Kapatma #

Python’daki kapanışlar, işlev çağrıları tarafından oluşturulur. Burada, “makeInc” çağrısı, “inc” işlevi içinde başvurulan “x” için bir bağlama oluşturur. “makeInc"e yapılan her çağrı, bu işlevin yeni bir örneğini oluşturur, ancak her örneğin farklı bir “x” bağlamasına bir bağlantısı vardır.

def makeInc(x):
  def inc(y):
     # x is "attached" in the definition of inc
     return y + x

  return inc

incOne = makeInc(1)
incFive = makeInc(5)

incOne(5) # returns 6
incFive(5) # returns 10

Düzenli bir kapanışta, ekteki işlevin tüm değişkenleri çevreleyen ortamından tamamen devraldığına dikkat edin, bu yapıda, ekteki işlevin yalnızca devralınan değişkenlere okuma erişimi olduğuna, ancak bunlara atama yapamayacağına dikkat edin.

def makeInc(x):
  def inc(y):
     # incrementing x is not allowed
     x += y  
     return x

  return inc

incOne = makeInc(1)
incOne(5) # UnboundLocalError: local variable 'x' referenced before assignment

Python 3, yuvalanmış işlevlerle tam bir kapatma gerçekleştirmek için “yerel olmayan” ifadesini (https://www.wikiod.com/tr/python/degisken-kapsam-ve-baglama#Yerel Olmayan Değişkenler) sunar.

def makeInc(x):
  def inc(y):
     nonlocal x
     # now assigning a value to x is allowed
     x += y  
     return x

  return inc

incOne = makeInc(1)
incOne(5) # returns 6
son sürüm

Adlandırılmış parametrelerin kullanımını zorlama #

İşlev imzasındaki ilk yıldızdan sonra belirtilen tüm parametreler yalnızca anahtar sözcüktür.

def f(*a, b):
    pass

f(1, 2, 3)
# TypeError: f() missing 1 required keyword-only argument: 'b'

Python 3’te, kalan argümanların yalnızca anahtar kelime argümanları kullanılarak iletilebildiğinden emin olmak için fonksiyon imzasına tek bir yıldız işareti koymak mümkündür.

def f(a, b, *, c):
    pass

f(1, 2, 3)
# TypeError: f() takes 2 positional arguments but 3 were given
f(1, 2, c=3)
# No error

Argümanlarla bir fonksiyon tanımlama #

Bağımsız değişkenler, işlev adından sonra parantez içinde tanımlanır:

def divide(dividend, divisor):  # The names of the function and its arguments
    # The arguments are available by name in the body of the function
    print(dividend / divisor)

İşlev adı ve argüman listesi, işlevin imzası olarak adlandırılır. Adlandırılmış her bağımsız değişken, etkin bir şekilde işlevin yerel bir değişkenidir.

Fonksiyonu çağırırken, argümanları sırayla listeleyerek değerleri verin.

divide(10, 2)
# output: 5

veya işlev tanımındaki adları kullanarak bunları herhangi bir sırayla belirtin:

divide(divisor=2, dividend=10)
# output: 5

Özyinelemeli işlevler #

Özyinelemeli bir işlev, tanımında kendisini çağıran bir işlevdir. Örneğin, “faktöriyel(n) = n*(n-1)(n-2)…32*1” ile tanımlanan faktöriyel matematiksel fonksiyon. olarak programlanabilir

def factorial(n):
    #n here should be an integer
    if n == 0:
        return 1
    else:
        return n*factorial(n-1)

buradaki çıktılar:

factorial(0)
#out 1
factorial(1)
#out 1
factorial(2)
#out 2
factorial(3)
#out 6

beklenildiği gibi. Bu işlevin özyinelemeli olduğuna dikkat edin, çünkü işlevin tanımında kendisini çağırdığı ikinci “dönüş faktörü(n-1)”.

Bazı özyinelemeli işlevler lambda kullanılarak gerçekleştirilebilir, lambda kullanan faktör işlevi şuna benzer:

factorial = lambda n: 1 if n == 0 else n*factorial(n-1)

İşlev, yukarıdakiyle aynı çıktıyı verir.

Yineleme sınırı #

Python uygulamasına bağlı olan olası özyineleme derinliğinin bir sınırı vardır. Sınıra ulaşıldığında, bir RuntimeError istisnası ortaya çıkar:

def cursing(depth):
  try:
    cursing(depth + 1) # actually, re-cursing
  except RuntimeError as RE:
    print('I recursed {} times!'.format(depth))

cursing(0)
# Out: I recursed 1083 times!

sys.setrecursionlimit(limit) kullanarak özyineleme derinlik limitini değiştirmek ve sys.getrecursionlimit() ile bu limiti kontrol etmek mümkündür.

sys.setrecursionlimit(2000)
cursing(0)
# Out: I recursed 1997 times!

Python 3.5’ten istisna, “RuntimeError"dan türetilen bir “RecursionError"dur.

İç içe işlevler #

Python’daki işlevler birinci sınıf nesnelerdir. Herhangi bir kapsamda tanımlanabilirler

def fibonacci(n):
    def step(a,b):
        return b, a+b
    a, b = 0, 1
    for i in range(n):
        a, b = step(a, b)
    return a

İşlevler, çevreleyen kapsamlarını yakalar, diğer herhangi bir nesne türü gibi iletilebilir

def make_adder(n):
    def adder(x):
        return n + x
    return adder
add5 = make_adder(5)
add6 = make_adder(6)
add5(10)
#Out: 15
add6(10)
#Out: 16

def repeatedly_apply(func, n, x):
    for i in range(n):
        x = func(x)
    return x

repeatedly_apply(add5, 5, 1)
#Out: 26

Birden fazla argüman içeren bir fonksiyon tanımlama #

Bir fonksiyona istenildiği kadar argüman verilebilir, sabit olan tek kural, her argüman adının benzersiz olması ve isteğe bağlı argümanların isteğe bağlı olmayanlardan sonra olması gerektiğidir:

def func(value1, value2, optionalvalue=10):
    return '{0} {1} {2}'.format(value1, value2, optionalvalue1)

İşlevi çağırırken, her bir anahtar kelimeyi isimsiz olarak verebilirsiniz, ancak sıra önemlidir:

print(func(1, 'a', 100))
# Out: 1 a 100

print(func('abc', 14))
# abc 14 10

Veya argümanları adla ve adsız vererek birleştirin. O halde ismi olanlar olmayanları takip etmelidir ama ismi olanların sırası önemli değil:

print(func('This', optionalvalue='StackOverflow Documentation', value2='is'))
# Out: This is StackOverflow Documentation
