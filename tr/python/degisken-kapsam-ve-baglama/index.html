Sözdizimi #

  • küresel a, b, c
  • yerel olmayan a, b
  • x = bir şey # x’i bağlar
  • (x, y) = bir şey # x ve y’yi bağlar
  • x += bir şey # x’i bağlar. Diğer tüm “op=” için benzer şekilde
  • del x # x’i bağlar
  • bir şeydeki x için: # x’i bağlar
  • x gibi bir şeyle: # x’i bağlar
  • örneğin İstisna dışında: # eski bloğun içine bağlar

Yerel Olmayan Değişkenler #

Python 3, yerel olmayan adlı yeni bir anahtar kelime ekledi. Yerel olmayan anahtar sözcük, iç kapsama bir kapsam geçersiz kılma ekler. Bununla ilgili her şeyi PEP 3104‘de okuyabilirsiniz. Bu en iyi birkaç kod örneği ile gösterilmiştir. En yaygın örneklerden biri, artırabilen işlev oluşturmaktır:

def counter():
    num = 0
    def incrementer():
        num += 1
        return num
    return incrementer

Bu kodu çalıştırmayı denerseniz, bir UnboundLocalError alırsınız, çünkü num değişkenine en içteki işlevde atanmadan önce başvurulur. Karışıma yerel olmayanı ekleyelim:

def counter():
    num = 0
    def incrementer():
        nonlocal num
        num += 1
        return num
    return incrementer

c = counter()
c() # = 1
c() # = 2
c() # = 3

Temel olarak ‘yerel olmayan’, global bir kapsamda değil, bir dış kapsamdaki değişkenlere atamanıza izin verir. Dolayısıyla, ‘sayaç’ işlevimizde ‘yerel olmayan’ı kullanamazsınız, çünkü o zaman global bir kapsama atamaya çalışır. Bir deneyin ve hemen bir SyntaxError alacaksınız. Bunun yerine yuvalanmış bir işlevde “yerel olmayan"ı kullanmalısınız.

(Burada sunulan işlevselliğin, jeneratörler kullanılarak daha iyi uygulandığına dikkat edin.)

Genel Değişkenler #

Python’da, işlevlerin içindeki değişkenler, yalnızca bir atama ifadesinin sol tarafında veya başka bir bağlama oluşumunda görünüyorlarsa yerel olarak kabul edilir; aksi takdirde böyle bir bağlama, genel kapsama kadar, çevreleyen işlevlerde aranır. Bu, atama ifadesi hiçbir zaman yürütülmese bile geçerlidir.

x = 'Hi'

def read_x():
    print(x)   # x is just referenced, therefore assumed global

read_x()       # prints Hi

def read_y():
    print(y)   # here y is just referenced, therefore assumed global

read_y()       # NameError: global name 'y' is not defined

def read_y():
    y = 'Hey'  # y appears in an assignment, therefore it's local
    print(y)   # will find the local y

read_y()       # prints Hey

def read_x_local_fail():
    if False:
        x = 'Hey'  # x appears in an assignment, therefore it's local
    print(x)   # will look for the _local_ z, which is not assigned, and will not be found

read_x_local_fail()   # UnboundLocalError: local variable 'x' referenced before assignment

Normalde, bir kapsam içindeki bir atama, kapsamın herhangi bir dış değişkenini gölgeleyecektir. aynı isim:

x = 'Hi'

def change_local_x():
    x = 'Bye'
    print(x)
change_local_x()  # prints Bye
print(x)  # prints Hi

Bir adın “genel” olarak bildirilmesi, kapsamın geri kalanı için, ada yapılan tüm atamaların modülün en üst düzeyinde gerçekleşeceği anlamına gelir:

x = 'Hi'

def change_global_x():
    global x
    x = 'Bye'
    print(x)

change_global_x()  # prints Bye
print(x)  # prints Bye

“Global” anahtar sözcüğü, atamaların programın en üst düzeyinde değil, modülün en üst düzeyinde gerçekleştirileceği anlamına gelir. Diğer modüller, modül içindeki değişkenlere normal noktalı erişime ihtiyaç duyacaktır.

Özetlemek gerekirse: bir ‘x’ değişkeninin bir fonksiyon için yerel olup olmadığını bilmek için entire fonksiyonunu okumalısınız:

  1. “global x"i bulduysanız, “x” bir global değişkendir
  2. ‘yerel olmayan x’i bulduysanız, o zaman ‘x’ çevreleyen bir işleve aittir ve ne yerel ne de küreseldir.
  3. “x = 5” veya “aralık(3) içindeki x için” veya başka bir bağlama bulduysanız, “x” bir yerel değişkendir
  4. Aksi takdirde, ‘x’ bazı çevreleyen kapsama aittir (işlev kapsamı, genel kapsam veya yerleşikler)

Yerel Değişkenler #

Bir işlev içinde bir ad bound ise, varsayılan olarak yalnızca işlev içinde erişilebilir:

def foo():
    a = 5
    print(a) # ok

print(a) #  NameError: name 'a' is not defined

Kontrol akışı yapılarının kapsam üzerinde hiçbir etkisi yoktur (“hariç” dışında), ancak henüz atanmamış değişkene erişmek bir hatadır:

def foo():
    if True: 
        a = 5
    print(a) # ok

b = 3
def bar():
    if False:
        b = 5
    print(b) # UnboundLocalError: local variable 'b' referenced before assignment

Yaygın bağlama işlemleri, atamalar, “for” döngüleri ve “a += 5” gibi artırılmış atamalardır.

Bağlanma Oluşumu #

x = 5
x += 7
for x in iterable: pass

Yukarıdaki ifadelerin her biri bir _bağlayıcı oluşumdur - ‘x’, ‘5’ ile gösterilen nesneye bağlı hale gelir. Bu ifade bir fonksiyonun içinde görünüyorsa, o zaman ‘x’ varsayılan olarak fonksiyon-yerel olacaktır. Bağlayıcı deyimlerin listesi için “Sözdizimi” bölümüne bakın.

Adları ararken işlevler sınıf kapsamını atlar #

Tanımlama sırasında sınıfların yerel bir kapsamı vardır, ancak sınıf içindeki işlevler, adları ararken bu kapsamı kullanmaz. Lambdalar işlevler olduğundan ve anlamalar işlev kapsamı kullanılarak uygulandığından, bu bazı şaşırtıcı davranışlara yol açabilir.

a = 'global'

class Fred:
    a = 'class'  # class scope
    b = (a for i in range(10))  # function scope
    c = [a for i in range(10)]  # function scope
    d = a  # class scope
    e = lambda: a  # function scope
    f = lambda a=a: a  # default argument uses class scope
    
    @staticmethod  # or @classmethod, or regular instance method
    def g():  # function scope
        return a

print(Fred.a)  # class
print(next(Fred.b))  # global
print(Fred.c[0])  # class in Python 2, global in Python 3
print(Fred.d)  # class
print(Fred.e())  # global
print(Fred.f())  # class
print(Fred.g()) # global

Bu kapsamın nasıl çalıştığını bilmeyen kullanıcılar, “b”, “c” ve “e"nin “sınıf” yazdırmasını bekleyebilirler.

PEP 227 adresinden:

Sınıf kapsamındaki adlara erişilemez. Adlar, en içteki işlev kapsamında çözümlenir. Bir iç içe kapsamlar zincirinde bir sınıf tanımı meydana gelirse, çözümleme süreci sınıf tanımlarını atlar.

Python’un [adlandırma ve bağlama] ile ilgili belgelerinden(http://docs.python.org/3/reference/executionmodel.html#naming):

Bir sınıf bloğunda tanımlanan isimlerin kapsamı, sınıf bloğu ile sınırlıdır; yöntemlerin kod bloklarını kapsamaz - bu, bir işlev kapsamı kullanılarak uygulandıkları için anlamaları ve oluşturucu ifadeleri içerir. Bu, aşağıdakilerin başarısız olacağı anlamına gelir:

A sınıfı: bir = 42 b = liste(10 aralığındaki i için a + i)

Bu örnek, Martijn’in bu yanıttan referansları kullanır. Bu davranışın daha derinlemesine analizini içeren Pieters.

del komutu #

Bu komutun birbiriyle ilişkili ancak farklı birkaç formu vardır.

del v #

“v” bir değişkense, “del v” komutu değişkeni kapsamından kaldırır. Örneğin:

x = 5
print(x) # out: 5
del x
print(x) # NameError: name 'f' is not defined

“del"in bir bağlayıcı oluşum olduğuna dikkat edin; bu, aksi açıkça belirtilmedikçe (“yerel olmayan” veya “küresel” kullanılarak), “del v"nin “v"yi geçerli kapsamda yerel yapacağı anlamına gelir. Bir dış kapsamda “v"yi silmek istiyorsanız, “del v” ifadesinin aynı kapsamında “yerel olmayan v” veya “global v” kullanın.

Aşağıdakilerin hepsinde, bir komutun amacı varsayılan bir davranıştır, ancak dil tarafından uygulanmaz. Bir sınıf bu niyeti geçersiz kılacak şekilde yazılabilir.

del v.name #

Bu komut, v.__delattr__(name) aramasını tetikler.

Amaç, “name” özniteliğini kullanılamaz hale getirmektir. Örneğin:

class A:
    pass

a = A()
a.x = 7
print(a.x) # out: 7
del a.x
print(a.x) # error: AttributeError: 'A' object has no attribute 'x'

del v[item] #

Bu komut, v.__delitem__(item) için bir çağrıyı tetikler.

Amaç, “öğe"nin “v” nesnesi tarafından uygulanan eşlemeye ait olmamasıdır. Örneğin:

x = {'a': 1, 'b': 2}
del x['a']
print(x) #  out: {'b': 2}
print(x['a']) # error: KeyError: 'a'

del v[a:b] #

Bu aslında v.__delslice__(a, b) çağırır.

Amaç, yukarıda açıklanana benzer, ancak dilimlerle - tek bir öğe yerine öğe aralıkları. Örneğin:

x = [0, 1, 2, 3, 4]
del x[1:3]
print(x) #  out: [0, 3, 4]

Ayrıca bkz. [Çöp Toplama#del komutu](https://www.wikiod.com/tr/python/cop-toplama#del komutunun etkileri).

Yerel ve Küresel Kapsam #

Yerel ve küresel kapsam nedir? #

Kodun bir noktasında erişilebilir olan tüm Python değişkenleri ya yerel kapsamda ya da küresel kapsamda bulunur.

Açıklama, yerel kapsamın mevcut fonksiyonda tanımlanan tüm değişkenleri ve global kapsamın mevcut fonksiyonun dışında tanımlanan değişkenleri içermesidir.

foo = 1  # global

def func():
    bar = 2  # local
    print(foo)  # prints variable foo from global scope
    print(bar)  # prints variable bar from local scope

Hangi değişkenlerin hangi kapsamda olduğu incelenebilir. Yerleşik işlevler ’locals()’ ve ‘globals()’ tüm kapsamları sözlük olarak döndürür.

foo = 1

def func():
    bar = 2
    print(globals().keys())  # prints all variable names in global scope
    print(locals().keys())  # prints all variable names in local scope

İsim çakışmaları ne olur? #

foo = 1

def func():
    foo = 2  # creates a new variable foo in local scope, global foo is not affected

    print(foo)  # prints 2

    # global variable foo still exists, unchanged:
    print(globals()['foo'])  # prints 1
    print(locals()['foo'])  # prints 2

Global bir değişkeni değiştirmek için “global” anahtar sözcüğünü kullanın:

foo = 1

def func():
    global foo
    foo = 2  # this modifies the global foo, rather than creating a local variable

Kapsam, işlevin tüm gövdesi için tanımlanmıştır!

Bunun anlamı, bir değişkenin hiçbir zaman fonksiyonun yarısı için global olmayacağı ve sonrasında yerel olmayacağı veya tam tersi olacağıdır.

foo = 1

def func():
    # This function has a local variable foo, because it is defined down below.
    # So, foo is local from this point. Global foo is hidden.

    print(foo) # raises UnboundLocalError, because local foo is not yet initialized
    foo = 7
    print(foo)

Aynı şekilde, tersi:

foo = 1

def func():
    # In this function, foo is a global variable from the begining

    foo = 7  # global foo is modified

    print(foo)  # 7
    print(globals()['foo'])  # 7

    global foo  # this could be anywhere within the function
    print(foo)  # 7

Fonksiyonlar içindeki fonksiyonlar #

İşlevler içinde iç içe geçmiş birçok işlev düzeyi olabilir, ancak herhangi bir işlevde, o işlev ve genel kapsam için yalnızca bir yerel kapsam vardır. Ara kapsamlar yoktur.

foo = 1

def f1():
    bar = 1

    def f2():
        baz = 2
        # here, foo is a global variable, baz is a local variable
        # bar is not in either scope
        print(locals().keys())  # ['baz']
        print('bar' in locals())  # False
        print('bar' in globals())  # False

    def f3():
        baz = 3
        print(bar)  # bar from f1 is referenced so it enters local scope of f3 (closure)
        print(locals().keys())  # ['bar', 'baz']
        print('bar' in locals())  # True
        print('bar' in globals())  # False

    def f4():
        bar = 4  # a new local bar which hides bar from local scope of f1
        baz = 4
        print(bar)
        print(locals().keys())  # ['bar', 'baz']
        print('bar' in locals())  # True
        print('bar' in globals())  # False

“global” ve “yerel olmayan” karşılaştırması (yalnızca Python 3) #

Bu anahtar sözcüklerin her ikisi de, geçerli işlevlerde yerel olmayan değişkenlere yazma erişimi elde etmek için kullanılır.

“Global” anahtar sözcüğü, bir adın global bir değişken olarak ele alınması gerektiğini bildirir.

foo = 0  # global foo

def f1():
    foo = 1  # a new foo local in f1
    
    def f2():
        foo = 2  # a new foo local in f2
        
        def f3():
            foo = 3  # a new foo local in f3
            print(foo)  # 3
            foo = 30  # modifies local foo in f3 only
        
        def f4():
            global foo
            print(foo)  # 0
            foo = 100  # modifies global foo

Öte yandan, Python 3’te bulunan “yerel olmayan” (bkz. https://www.wikiod.com/tr/python/degisken-kapsam-ve-baglama#Yerel Olmayan Değişkenler ), yerel değişkeni çevreleyen kapsamdan geçerli işlevin yerel kapsamına.

‘yerel olmayan’ ile ilgili Python belgelerinde:

Yerel olmayan ifade, listelenen tanımlayıcıların hariç tutulan en yakın kapsamdaki önceden bağlanmış değişkenler küreseller.

def f1():
    
    def f2():
        foo = 2  # a new foo local in f2

        def f3():
            nonlocal foo  # foo from f2, which is the nearest enclosing scope
            print(foo)  # 2
            foo = 20  # modifies foo from f2!
