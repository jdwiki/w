Python, ’re’ modülü aracılığıyla düzenli ifadeleri kullanılabilir hale getirir.

Normal ifadeler, alt dizeleri eşleştirmek için kurallar olarak yorumlanan karakter kombinasyonlarıdır. Örneğin, “tutar\D+\d+” ifadesi, “tutar” sözcüğünden oluşan herhangi bir dizeyle ve bir veya daha fazla basamaksız sayıyla ayrılmış bir tam sayıyla eşleşir, örneğin: “tutar=100”, “tutar” 3, tutar şuna eşittir: 33` vb.

Sözdizimi #

  • Doğrudan Normal İfadeler

  • re.match(pattern, string, flag=0) # Out: stringin başlangıcındaki eşleşme kalıbı veya Yok

  • re.search(pattern, string, flag=0) # Out: string içindeki kalıbı eşleştir veya Yok

  • re.findall(pattern, string, flag=0) # Out: string veya [] içindeki tüm kalıp eşleşmelerinin listesi

  • re.finditer(pattern, string, flag=0) # Out: re.findall ile aynı, ancak yineleyici nesnesini döndürür

  • re.sub(desen, değiştirme, dize, bayrak=0) # Out: model yerine değiştirme (dize veya işlev) içeren dize

  • Önceden Derlenmiş Normal İfadeler

  • precompiled_pattern = yeniden derleme(desen, bayrak=0)

  • precompiled_pattern.match(string) # Out: dizgenin başındaki eşleşme veya Yok

  • precompiled_pattern.search(string) # Out: string veya None içinde herhangi bir yeri eşleştirin

  • precompiled_pattern.findall(string) # Out: eşleşen tüm alt dizelerin listesi

  • precompiled_pattern.sub(dize/desen/işlev, dize) # Çıkış: değiştirilen dize

Bir dizenin başlangıcını eşleştirme #

“re.match()” öğesinin ilk argümanı normal ifadedir, ikincisi ise eşleşecek dizedir:

import re

pattern = r"123"
string = "123zzb"

re.match(pattern, string)
# Out: <_sre.SRE_Match object; span=(0, 3), match='123'>

match = re.match(pattern, string)

match.group()
# Out: '123'

Model değişkeninin, dizenin ham dize değişmezi olduğunu gösteren ‘r’ ön eki olan bir dize olduğunu fark edebilirsiniz.

Bir ham dize değişmezi, bir dize değişmezinden biraz farklı bir sözdizimine sahiptir, yani bir ham dize değişmezinde ters eğik çizgi \ “sadece bir ters eğik çizgi” anlamına gelir ve yeni satırlar ( \n), sekmeler (\t), geri almalar (\), form beslemeleri (\r) vb. Normal dize değişmezlerinde, bir kaçış dizisinin başlangıcı olarak alınmamak için her ters eğik çizgi iki katına çıkarılmalıdır.

Bu nedenle, “r”\n"2 karakterden oluşan bir dizedir:` ve n. Normal ifade kalıpları ayrıca ters eğik çizgi kullanır, ör. \d' herhangi bir rakam karakterini ifade eder. Ham dizeleri (r"\d") kullanarak dizelerimizden ("\d"`) iki kez kaçmaktan kaçınabiliriz.

Örneğin:

string = "\\t123zzb" # here the backslash is escaped, so there's no tab, just '\' and 't'
pattern = "\\t123"   # this will match \t (escaping the backslash) followed by 123
re.match(pattern, string).group()   # no match
re.match(pattern, "\t123zzb").group()  # matches '\t123'

pattern = r"\\t123"  
re.match(pattern, string).group()   # matches '\\t123'

Eşleştirme yalnızca dizenin başından itibaren yapılır. Herhangi bir yerde eşleştirmek istiyorsanız, bunun yerine re.search kullanın:

match = re.match(r"(123)", "a123zzb")

match is None
# Out: True

match = re.search(r"(123)", "a123zzb")

match.group()
# Out: '123'

Aranıyor #

pattern = r"(your base)"
sentence = "All your base are belong to us."

match = re.search(pattern, sentence)
match.group(1)
# Out: 'your base'

match = re.search(r"(belong.*)", sentence)
match.group(1)
# Out: 'belong to us.'

Arama, “re.match"den farklı olarak dizgenin herhangi bir yerinde yapılır. “re.findall"ı da kullanabilirsiniz.

Ayrıca dizenin başında arama yapabilirsiniz (^ kullanın),

match = re.search(r"^123", "123zzb")
match.group(0)
# Out: '123'

match = re.search(r"^123", "a123zzb")
match is None
# Out: True

dizenin sonunda ($ kullanın),

match = re.search(r"123$", "zzb123")
match.group(0)
# Out: '123'

match = re.search(r"123$", "123zzb")
match is None
# Out: True

veya her ikisini de kullanın (hem ^ hem de $ kullanın):

match = re.search(r"^123$", "123")
match.group(0)
# Out: '123'

Önceden derlenmiş desenler #

import re

precompiled_pattern = re.compile(r"(\d+)")
matches = precompiled_pattern.search("The answer is 41!")
matches.group(1)
# Out: 41

matches = precompiled_pattern.search("Or was it 42?")
matches.group(1)
# Out: 42

Bir kalıbı derlemek, daha sonra bir programda yeniden kullanılmasına izin verir. Ancak, Python’un son kullanılan ifadeleri (docs, SO answer) önbelleğe aldığını unutmayın, bu nedenle “yalnızca birkaç normal ifade kullanan programlar normal ifadeleri derleme konusunda endişelenmenize gerek yok”.

import re

precompiled_pattern = re.compile(r"(.*\d+)")
matches = precompiled_pattern.match("The answer is 41!")
print(matches.group(1))
# Out: The answer is 41

matches = precompiled_pattern.match("Or was it 42?")
print(matches.group(1))
# Out: Or was it 42

re.match() ile kullanılabilir.

Bayraklar #

Bazı özel durumlar için Normal İfadenin davranışını değiştirmemiz gerekir, bu bayraklar kullanılarak yapılır. Bayraklar, ‘flags’ anahtar sözcüğü aracılığıyla veya doğrudan ifadede olmak üzere iki şekilde ayarlanabilir.

Anahtar kelimeyi işaretler #

Aşağıda ’re.search’ için bir örnek var, ancak ’re.search’ modülündeki çoğu fonksiyon için çalışıyor.

m = re.search("b", "ABC")  
m is None
# Out: True

m = re.search("b", "ABC", flags=re.IGNORECASE)
m.group()
# Out: 'B'

m = re.search("a.b", "A\nBC", flags=re.IGNORECASE) 
m is None
# Out: True

m = re.search("a.b", "A\nBC", flags=re.IGNORECASE|re.DOTALL) 
m.group()
# Out: 'A\nB'

Ortak İşaretler

bayrak Kısa Açıklama
’re.IGNORECASE’, ’re.I’ Desenin durumu yok saymasını sağlar
’re.DOTALL’, ’re.S’ Yeni satırlar dahil her şeyle eşleşir . yapar
’re.MULTILINE’, ’re.M' ^ ile satırın başlangıcını ve $ sonunu eşleştirir
re.DEBUG Hata ayıklama bilgilerini açar

Mevcut tüm bayrakların tam listesi için belgeler’e bakın.

Satır içi bayraklar #

belgeler‘den:

(?iLmsux) (‘i’, ‘L’, ’m’, ’s’, ‘u’, ‘x’ kümesinden bir veya daha fazla harf.)

Grup boş dizeyle eşleşir; harfler karşılık gelen işaretleri ayarlar: re.I (büyük/küçük harf yoksay), re.L (yerel ayara bağlıdır), re.M (çok satırlı), re.S (nokta hepsiyle eşleşir), re.U (Unicode’a bağlıdır) ve re.X (ayrıntılı), tüm normal ifade için. Bu, re.compile() işlevine bir bayrak argümanı iletmek yerine, normal ifadenin bir parçası olarak bayrakları dahil etmek istiyorsanız kullanışlıdır.

(?x) bayrağının, ifadenin ayrıştırılma şeklini değiştirdiğine dikkat edin. İfade dizesinde önce veya bir veya daha fazla boşluk karakterinden sonra kullanılmalıdır. Bayraktan önce boşluk olmayan karakterler varsa, sonuçlar tanımsızdır.

Değiştirme #

re.sub kullanılarak dizelerde değişiklik yapılabilir.

Dizelerin değiştirilmesi #

re.sub(r"t[0-9][0-9]", "foo", "my name t13 is t44 what t99 ever t44")
# Out: 'my name foo is foo what foo ever foo'

Grup referanslarını kullanma #

Az sayıda grupla değiştirmeler aşağıdaki gibi yapılabilir:

re.sub(r"t([0-9])([0-9])", r"t\2\1", "t13 t19 t81 t25")
# Out: 't31 t91 t18 t52'

Ancak, ‘10’ gibi bir grup kimliği yaparsanız bu çalışmaz: ‘\10’ okunur ‘Kimlik numarası 1 ve ardından 0’ olarak. Bu nedenle daha spesifik olmanız ve \g<i> gösterimini kullanmanız gerekir:

re.sub(r"t([0-9])([0-9])", r"t\g<2>\g<1>", "t13 t19 t81 t25")
# Out: 't31 t91 t18 t52'

Bir değiştirme işlevi kullanma #

items = ["zero", "one", "two"]
re.sub(r"a\[([0-3])\]", lambda match: items[int(match.group(1))], "Items: a[0], a[1], something, a[2]")
# Out: 'Items: zero, one, something, two'

Çakışmayan Tüm Eşleşmeleri Bul #

re.findall(r"[0-9]{2,3}", "some 1 text 12 is 945 here 4445588899")
# Out: ['12', '945', '444', '558', '889']

“[0-9]{2,3}"‘den önceki “r"nin python’a dizeyi olduğu gibi yorumlamasını söylediğine dikkat edin; “ham” bir dize olarak.

Ayrıca, “re.findall()” ile aynı şekilde çalışan ancak bir dize listesi yerine “SRE_Match” nesneleri içeren bir yineleyici döndüren “re.finditer()” öğesini de kullanabilirsiniz:

results = re.finditer(r"([0-9]{2,3})", "some 1 text 12 is 945 here 4445588899")
print(results)
# Out: <callable-iterator object at 0x105245890>
for result in results:
     print(result.group(0))
''' Out:
12
945
444
558
889
'''

İzin verilen karakterler kontrol ediliyor #

Bir dizenin yalnızca belirli bir karakter kümesi içerdiğini kontrol etmek istiyorsanız, bu durumda a-z, A-Z ve 0-9, bunu şöyle yapabilirsiniz,

import re

def is_allowed(string):
    characherRegex = re.compile(r'[^a-zA-Z0-9.]')
    string = characherRegex.search(string)
    return not bool(string)
    
print (is_allowed("abyzABYZ0099")) 
# Out: 'True'

print (is_allowed("#*@#$%^")) 
# Out: 'False'

Örneğin büyük harflere izin vermemek için ifade satırını “[^a-zA-Z0-9.]” ile “[^a-z0-9.]” arasında da uyarlayabilirsiniz.

Kısmi kredi : http://stackoverflow.com/a/1325265/2697955

Normal ifadeler kullanarak bir dizeyi bölme #

Bir dizeyi bölmek için normal ifadeleri de kullanabilirsiniz. Örneğin,

import re
data = re.split(r'\s+', 'James 94 Samantha 417 Scarlett 74')
print( data )
# Output: ['James', '94', 'Samantha', '417', 'Scarlett', '74']

Gruplandırma #

Gruplandırma parantez içinde yapılır. “group()“u çağırmak, eşleşen parantez içindeki alt gruplardan oluşan bir dize döndürür.

match.group() # Group without argument returns the entire match found
# Out: '123'
match.group(0) # Specifying 0 gives the same result as specifying no argument
# Out: '123'

Belirli bir alt grubu getirmek için “group()“a argümanlar da sağlanabilir.

belgeler‘den:

Tek bir argüman varsa, sonuç tek bir dizgedir; birden fazla argüman varsa, sonuç argüman başına bir öğe içeren bir demet olur.

Öte yandan groups() çağrısı, alt grupları içeren bir demet listesi döndürür.

sentence = "This is a phone number 672-123-456-9910"
pattern = r".*(phone).*?([\d-]+)"

match = re.match(pattern, sentence)

match.groups()   # The entire match as a list of tuples of the paranthesized subgroups
# Out: ('phone', '672-123-456-9910')

m.group()        # The entire match as a string
# Out: 'This is a phone number 672-123-456-9910'

m.group(0)       # The entire match as a string
# Out: 'This is a phone number 672-123-456-9910'

m.group(1)       # The first parenthesized subgroup.
# Out: 'phone'

m.group(2)       # The second parenthesized subgroup.
# Out: '672-123-456-9910'

m.group(1, 2)    # Multiple arguments give us a tuple.
# Out: ('phone', '672-123-456-9910')

Adlandırılmış gruplar #

match = re.search(r'My name is (?P<name>[A-Za-z ]+)', 'My name is John Smith')
match.group('name')
# Out: 'John Smith'

match.group(1)
# Out: 'John Smith'

Dizine göre olduğu kadar ada göre de başvurulabilecek bir yakalama grubu oluşturur.

Yakalamayan gruplar #

``(?:)``` kullanmak bir grup oluşturur, ancak grup yakalanmaz. Bu, onu bir grup olarak kullanabileceğiniz anlamına gelir, ancak “grup alanınızı” kirletmez.

re.match(r'(\d+)(\+(\d+))?', '11+22').groups()
# Out: ('11', '+22', '22')

re.match(r'(\d+)(?:\+(\d+))?', '11+22').groups()
# Out: ('11', '22')

Bu örnek “11+22” veya “11” ile eşleşir, ancak “11+”” ile eşleşmez. Bunun nedeni ``+işareti ve ikinci terim gruplandırılmış olmasıdır. Öte yandan,+``` işareti yakalanmadı.

Özel Karakterlerden Kaçış #

Özel karakterler (aşağıdaki [ ve ] karakter sınıfı parantezleri gibi) tam anlamıyla eşleşmez:

match = re.search(r'[b]', 'a[b]c')
match.group()
# Out: 'b'

Özel karakterlerden kaçarak tam anlamıyla eşleştirilebilirler:

match = re.search(r'\[b\]', 'a[b]c')
match.group()
# Out: '[b]'

re.escape() işlevi sizin için bunu yapmak için kullanılabilir:

re.escape('a[b]c')
# Out: 'a\\[b\\]c'
match = re.search(re.escape('a[b]c'), 'a[b]c')
match.group()
# Out: 'a[b]c'

re.escape() işlevi tüm özel karakterlerden kaçar, bu nedenle kullanıcı girdisine dayalı bir normal ifade oluşturuyorsanız kullanışlıdır:

username = 'A.C.'  # suppose this came from the user
re.findall(r'Hi {}!'.format(username), 'Hi A.C.! Hi ABCD!')
# Out: ['Hi A.C.!', 'Hi ABCD!']
re.findall(r'Hi {}!'.format(re.escape(username)), 'Hi A.C.! Hi ABCD!')
# Out: ['Hi A.C.!']

“re.finditer” kullanarak eşleşmeler üzerinde yineleme #

Bir dizgedeki tüm eşleşmeleri yinelemek için ’re.finditer’ı kullanabilirsiniz. Bu size (dizedeki (dizinler) eşleşme konumuyla ilgili bilgiler gibi “re.findall” ek bilgileriyle karşılaştırıldığında) verir:

import re
text = 'You can try to find an ant in this string'
pattern = 'an?\w' # find 'an' either with or without a following word character

for match in re.finditer(pattern, text):
    # Start index of match (integer)
    sStart = match.start()

    # Final index of match (integer)
    sEnd = match.end()

    # Complete match (string)
    sGroup = match.group()

    # Print match
    print('Match "{}" found at: [{},{}]'.format(sGroup, sStart,sEnd))

Sonuç:

Match "an" found at: [5,7]
Match "an" found at: [20,22]
Match "ant" found at: [23,26]

Bir ifadeyi yalnızca belirli konumlarda eşleştirin #

Çoğu zaman bir ifadeyi yalnızca belirli yerlerde eşleştirmek istersiniz (diğerlerinde onlara dokunmadan bırakırsınız). Aşağıdaki cümleyi düşünün:

An apple a day keeps the doctor away (I eat an apple everyday).

Burada “elma” iki kez meydana gelir ve bu, daha yeni regex modülü tarafından desteklenen sözde geri izleme kontrol fiilleri ile çözülebilir . Fikir şudur:

forget_this | or this | and this as well | (but keep this)

Elma örneğimizle, bu şöyle olur:

import regex as re
string = "An apple a day keeps the doctor away (I eat an apple everyday)."
rx = re.compile(r'''
    \([^()]*\) (*SKIP)(*FAIL)  # match anything in parentheses and "throw it away"
    |                          # or
    apple                      # match an apple
    ''', re.VERBOSE)
apples = rx.findall(string)
print(apples)
# only one

Bu, “elma” ile yalnızca parantezlerin dışında bulunduğunda eşleşir. İşte nasıl çalıştığı:

  • Soldan sağa bakarken, normal ifade motoru soldaki her şeyi tüketir, (*SKIP) bir “her zaman doğru-iddia” görevi görür. Daha sonra, (*FAIL) üzerinde doğru şekilde başarısız olur ve geri döner.
  • Şimdi, daha fazla sola gitmenin yasak olduğu (*SKIP) sağdan sola (aka geri izleme sırasında) noktasına geliyor. Bunun yerine, motora herhangi bir şeyi sola atması ve “(*SKIP)” öğesinin çağrıldığı noktaya atlaması söylenir.
