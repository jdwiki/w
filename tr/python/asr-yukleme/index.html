Operatör aşırı yüklemesi #

Aşağıda, sınıflarda aşırı yüklenebilen işleçler, gerekli yöntem tanımları ve bir ifade içinde kullanılan işleç örneği yer almaktadır.

N.B. Değişken adı olarak ‘diğer’ kullanılması zorunlu değildir, ancak norm olarak kabul edilir.

Operatör Yöntem ifade
+ Ekleme __add__(kendi, diğer) “a1 + a2”
- Çıkarma __sub__(kendi, diğer) “a1 - a2”
* Çarpma __mul__(kendi, diğer) a1 * a2
@ Matris Çarpımı __matmul__(kendi, diğer) a1 @ a2 (Python 3.5)
/ Bölümü __div__(kendi, diğer) a1 / a2 (Yalnızca Python 2)
/ Bölümü __truediv__(kendi, diğer) a1 / a2 (Python 3)
// Kat Bölümü __floordiv__(kendi, diğer) a1 // a2
% Modulo/Kalan __mod__(kendi, diğer) a1 % a2
** Güç __pow__(kendi, diğer[, modulo]) a1 ** a2
<< Bitsel Sola Kaydırma __lshift__(kendi, diğer) ‘a1 « a2’
>> Bitsel Sağa Kaydırma __rshift__(kendi, diğer) a1 >> a2
& Bitsel VE __ve__(kendi, diğer) ‘a1 ve a2’
^ Bitsel XOR __xor__(kendi, diğer) ‘a1 ^ a2’
| (Bitsel VEYA) __veya__(kendi, diğer) a1 | a2
- Olumsuzlama (Aritmetik) __neg__(kendi) -a1
+ Olumlu __pos__(kendi) +a1
~ Bitsel DEĞİL __invert__(self) ~a1
< Küçük __lt__(kendi, diğer) ‘a1 < a2’
<= Küçüktür veya Eşittir __le__(kendi, diğer) ‘a1 <= a2’
== Eşittir __eq__(kendi, diğer) ‘a1 == a2’
!= Eşit Değil __ne__(kendi, diğer) a1 != a2
> Büyüktür __gt__(kendi, diğer) ‘a1 > a2’
>= Büyüktür veya Eşittir __ge__(kendi, diğer) a1 >= a2
“[indeks]” Dizin operatörü __getitem__(kendi, dizin) ‘a1[indeks]’
in Operatörde __contains__(self, other) ‘a2 içinde a1’
(*args, ...) Arama __call__(self, *args, **kwargs) a1(*args, **kwargs)

"pow" için isteğe bağlı “modulo” parametresi yalnızca “pow” yerleşik işlevi tarafından kullanılır.

Bir binary işlecine karşılık gelen yöntemlerin her birinin, “__r” ile başlayan karşılık gelen “doğru” bir yöntemi vardır, örneğin “radd”:

class A:
    def __init__(self, a):
        self.a = a
    def __add__(self, other):
        return self.a + other
    def __radd__(self, other):
        print("radd")
        return other + self.a

A(1) + 2  # Out:  3
2 + A(1)  # prints radd. Out: 3

“__i” ile başlayan karşılık gelen bir yerinde sürümün yanı sıra:

class B:
    def __init__(self, b):
        self.b = b
    def __iadd__(self, other):
        self.b += other
        print("iadd")
        return self

b = B(2)
b.b       # Out: 2
b += 1    # prints iadd
b.b       # Out: 3

Bu yöntemlerde özel bir şey olmadığından, dilin diğer birçok bölümü, standart kitaplığın bölümleri ve hatta üçüncü taraf modüller, bir nesneyi bir türe dönüştürme yöntemleri veya nesnenin özelliklerini kontrol etme yöntemleri gibi kendi başlarına sihirli yöntemler ekler. Örneğin, yerleşik str() işlevi, varsa, nesnenin __str__ yöntemini çağırır. Bu kullanımlardan bazıları aşağıda listelenmiştir.

fonksiyon Yöntem ifade
int için Döküm __int__(kendi) int(a1)
Mutlak fonksiyon __abs__(kendi) abs(a1)
str için Döküm __str__(kendi) str(a1)
‘unicode’a Döküm __unicode__(self) unicode(a1) (yalnızca Python 2)
Dizi gösterimi __repr__(kendi) repr(a1)
‘bool’a Döküm __nonzero__(self) bool(a1)
Dize biçimlendirme __format__(self, formatstr) "Merhaba {:abc}".format(a1)
Hashleme __hash__(kendi) hash(a1)
uzunluk __len__(kendi) len(a1)
Ters __ters__(kendi) ters(a1)
Kat __floor__(self) math.floor(a1)
Tavan __ceil__(self) math.ceil(a1)

Ayrıca bağlam yöneticileri için __enter__ ve __exit__ gibi özel yöntemler ve daha fazlası vardır.

Büyü/Dunder Yöntemleri #

Python’daki Magic (çift alt çizginin kısaltması olarak dunder olarak da adlandırılır) yöntemleri, diğer dillerdeki operatör aşırı yüklemesine benzer bir amaca hizmet eder. Birli veya ikili operatör ifadelerinde işlenen olarak kullanıldığında bir sınıfın davranışını tanımlamasına izin verirler. Ayrıca bazı yerleşik işlevler tarafından çağrılan uygulamalar olarak da hizmet ederler.

İki boyutlu vektörlerin bu uygulamasını düşünün.

import math

class Vector(object):
    # instantiation
    def __init__(self, x, y):
        self.x = x
        self.y = y

    # unary negation (-v)
    def __neg__(self):
        return Vector(-self.x, -self.y)

    # addition (v + u)
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    # subtraction (v - u)
    def __sub__(self, other):
        return self + (-other)

    # equality (v == u)
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

    # abs(v)
    def __abs__(self):
        return math.hypot(self.x, self.y)

    # str(v)
    def __str__(self):
        return '<{0.x}, {0.y}>'.format(self)

    # repr(v)
    def __repr__(self):
        return 'Vector({0.x}, {0.y})'.format(self)

Artık çeşitli ifadelerde ‘Vector’ sınıfının örneklerini doğal olarak kullanmak mümkün.

v = Vector(1, 4)
u = Vector(2, 0)

u + v           # Vector(3, 4)
print(u + v)    # "<3, 4>" (implicit string conversion)
u - v           # Vector(1, -4)
u == v          # False
u + v == v + u  # True
abs(u + v)      # 5.0

Uygulanmamış davranışları ele alma #

Sınıfınız, sağlanan bağımsız değişken türleri için belirli bir aşırı yüklenmiş işleç uygulamıyorsa, “NotImplemented” döndürmelidir (**bunun bir özel sabit olduğunu, “NotImplementedError” ile aynı olmadığını unutmayın). Bu, Python’un işlemin çalışması için diğer yöntemleri denemeye geri dönmesine izin verecektir:

NotImplemented döndürüldüğünde, tercüman daha sonra diğer türde yansıyan işlem veya başka bir geri dönüş, operatöre bağlı olarak. Tüm denenen işlemler geri dönerse ‘Uygulanmadı’, yorumlayıcı uygun bir istisna oluşturacaktır.

Örneğin, “x + y” verildiğinde, “x.add(y)” uygulanmamışsa, bunun yerine “y.radd(x)” denenir.

class NotAddable(object):

    def __init__(self, value):
        self.value = value

    def __add__(self, other):
        return NotImplemented


class Addable(NotAddable):

    def __add__(self, other):
        return Addable(self.value + other.value)

    __radd__ = __add__

Bu yansıtılmış yöntem olduğundan, her durumda beklenen davranışı elde etmek için __add__ ve __radd__ uygulamak zorundayız; Neyse ki, bu basit örnekte ikisi de aynı şeyi yaptığı için kısa yoldan gidebiliriz.

Kullanımda:

>>> x = NotAddable(1)
>>> y = Addable(2)
>>> x + x
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: unsupported operand type(s) for +: 'NotAddable' and 'NotAddable'
>>> y + y
<so.Addable object at 0x1095974d0>
>>> z = x + y
>>> z
<so.Addable object at 0x109597510>
>>> z.value
3
