Python, herhangi bir belirsizlik ve beklenmedik davranışlar olmadan açık ve okunabilir olması amaçlanan bir dildir. Ne yazık ki, bu hedefler her durumda elde edilemez ve bu nedenle Python’un beklediğinizden farklı bir şey yapabileceği birkaç köşe durumu vardır.

Bu bölüm size Python kodu yazarken karşılaşabileceğiniz bazı sorunları gösterecektir.

Liste çarpma ve ortak referanslar #

Aşağıdakileri çarparak iç içe bir liste yapısı oluşturma durumunu düşünün:

li = [[]] * 3
print(li)
# Out: [[], [], []]

İlk bakışta 3 farklı iç içe liste içeren bir listemiz olduğunu düşünebiliriz. İlkine ‘1’ eklemeye çalışalım:

li[0].append(1)
print(li)
# Out: [[1], [1], [1]]

’li’ içindeki tüm listelere ‘1’ eklendi.

Bunun nedeni, [[]] * 3 öğesinin 3 farklı listeden oluşan bir liste oluşturmamasıdır. Bunun yerine, aynı “liste” nesnesine 3 referans içeren bir “liste” oluşturur. Bu nedenle, li[0]‘ye eklediğimizde, değişiklik ’li’nin tüm alt öğelerinde görünür. Bu şuna eşdeğerdir:

li = []
element = [[]]
li = element + element + element
print(li)
# Out: [[], [], []]
element.append(1)
print(li)
# Out: [[1], [1], [1]]

İçerdiği “liste"nin bellek adreslerini “id” kullanarak yazdırırsak, bu daha da doğrulanabilir:

li = [[]] * 3
print([id(inner_list) for inner_list in li])
# Out: [6830760, 6830760, 6830760]

Çözüm, iç listeleri bir döngü ile oluşturmaktır:

li = [[] for _ in range(3)]

Tek bir liste oluşturup ardından ona 3 referans vermek yerine artık 3 farklı liste oluşturuyoruz. Bu, yine ‘id’ işlevi kullanılarak doğrulanabilir:

print([id(inner_list) for inner_list in li])
# Out: [6331048, 6331528, 6331488]

Bunu da yapabilirsiniz. Her birinde yeni bir boş liste oluşturulmasına neden olur. ’ekle’ çağrısı.

>>> li = []
>>> li.append([])
>>> li.append([])
>>> li.append([])
>>> for k in li: print(id(k))
... 
4315469256
4315564552
4315564808

Bir dizi üzerinde döngü yapmak için dizin kullanmayın.

Yapma:

for i in range(len(tab)):
    print(tab[i])

Yapmak:

for elem in tab:
    print(elem)

‘for’ sizin için çoğu yineleme işlemini otomatikleştirecektir.

Hem dizine hem de öğeye gerçekten ihtiyacınız varsa numaralandırmayı kullanın.

for i, elem in enumerate(tab):
     print((i, elem))

Doğru veya Yanlış olup olmadığını kontrol etmek için “==” kullanırken dikkatli olun

if (var == True):
    # this will execute if var is True or 1, 1.0, 1L

if (var != True):
    # this will execute if var is neither True nor 1

if (var == False):
    # this will execute if var is False or 0 (or 0.0, 0L, 0j)

if (var == None):
    # only execute if var is None

if var:
    # execute if var is a non-empty string/list/dictionary/tuple, non-0, etc

if not var:
    # execute if var is "", {}, [], (), 0, None, etc.

if var is True:
    # only execute if var is boolean True, not 1

if var is False:
    # only execute if var is boolean False, not 0

if var is None:
    # same as var == None

Pythonistalar genellikle “Af dilemek izin almaktan daha kolaydır” derler.

Yapma:

if os.path.isfile(file_path):
    file = open(file_path)
else:
    # do something

Yapmak:

try:
    file = open(file_path)
except OSError as e:
    # do something

Veya “Python 2.6+” ile daha da iyisi:

with open(file_path) as file:

Çok daha iyi çünkü çok daha genel. Hemen hemen her şeye ‘dene/hariç’ uygulayabilirsiniz. Bunu önlemek için ne yapacağınızı umursamanıza gerek yok, sadece risk aldığınız hatayı önemseyin.

Tipe göre kontrol etmeyin

Python dinamik olarak yazılmıştır, bu nedenle türü kontrol etmek esnekliğinizi kaybetmenize neden olur. Bunun yerine, davranışı kontrol ederek ördek yazarak kullanın. Bir işlevde bir dize bekliyorsanız, herhangi bir nesneyi bir dizeye dönüştürmek için str() kullanın. Bir liste bekliyorsanız, yinelenebilir herhangi bir öğeyi listeye dönüştürmek için list() kullanın.

Yapma:

def foo(name):
    if isinstance(name, str):
        print(name.lower())

def bar(listing):
    if isinstance(listing, list):
        listing.extend((1, 2, 3))
        return ", ".join(listing)

Yapmak:

def foo(name) :
    print(str(name).lower())

def bar(listing) :
    l = list(listing)
    l.extend((1, 2, 3))
    return ", ".join(l)

Son yolu kullanarak, ‘foo’ herhangi bir nesneyi kabul edecektir. bar stringleri, tupleleri, setleri, listeleri ve çok daha fazlasını kabul eder. Ucuz KURU.

Bazı özelliklerden yoksundurlar ve kalıtımla garip davranışlar sergileyebilirler. Kullanılabilir olması için, sınıfınızdan herhangi birinin “yeni stil” olması gerekir. Bunu yapmak için, onu “nesne"den devralın.

Yapma:

class Father:
    pass

class Child(Father):
    pass

Yapmak:

class Father(object):
    pass


class Child(Father):
    pass

Python 3.xde tüm sınıflar yeni stildir, dolayısıyla bunu yapmanıza gerek yoktur.

Sınıf niteliklerini init yönteminin dışında başlatma

Diğer dillerden gelen insanlar bunu cazip buluyor çünkü Java veya PHP’de yaptığınız şey bu. Sınıf adını yazarsınız, ardından niteliklerinizi listeler ve onlara varsayılan bir değer verirsiniz. Python’da çalışıyor gibi görünüyor, ancak bu düşündüğünüz gibi çalışmıyor. Bunu yapmak sınıf niteliklerini (statik nitelikler) kuracak, daha sonra nesne niteliğini almaya çalıştığınızda, boş olmadığı sürece size değerini verecektir. Bu durumda sınıf özniteliklerini döndürür. İki büyük tehlikeyi ifade eder:

  • Sınıf özniteliği değiştirilirse, başlangıç ​​değeri değiştirilir.

  • Değişken bir nesneyi varsayılan değer olarak ayarlarsanız, aynı değeri elde edersiniz. örnekler arasında paylaşılan nesne.

Yapma (statik istemiyorsanız):

class Car(object):
    color = "red"
    wheels = [Wheel(), Wheel(), Wheel(), Wheel()]

Yapmak :

class Car(object):
    def __init__(self):
        self.color = "red"
        self.wheels = [Wheel(), Wheel(), Wheel(), Wheel()]

Değişken varsayılan argüman #

def foo(li=[]):
    li.append(1)
    print(li)

foo([2])
# Out: [2, 1]
foo([3])
# Out: [3, 1]

Bu kod beklendiği gibi davranır, ancak ya bir argüman iletmezsek?

foo()
# Out: [1] As expected...

foo()
# Out: [1, 1]  Not as expected...

Bunun nedeni, işlevlerin ve yöntemlerin varsayılan argümanlarının çalışma zamanından ziyade tanım zamanında değerlendirilmesidir. Yani ’li’ listesinin yalnızca tek bir örneğine sahibiz.

Bunu aşmanın yolu, varsayılan argümanlar için yalnızca değişmez türleri kullanmaktır:

def foo(li=None):
    if not li:
        li = []
    li.append(1)
    print(li)

foo()
# Out: [1]

foo()
# Out: [1]

Bir iyileştirme ve ‘if değilse li’ doğru olarak ‘Yanlış’ olarak değerlendirilmesine rağmen, sıfır uzunluklu diziler gibi diğer birçok nesne de aynı şeyi yapar. Aşağıdaki örnek argümanlar istenmeyen sonuçlara neden olabilir:

x = []
foo(li=x)
# Out: [1]

foo(li="")
# Out: [1]

foo(li=0) 
# Out: [1]

Deyimsel yaklaşım, argümanı ‘Yok’ nesnesine karşı doğrudan kontrol etmektir:

def foo(li=None):
    if li is None:
        li = []
    li.append(1)
    print(li)

foo()
# Out: [1]

Yinelediğiniz sırayı değiştirme #

Bir “for” döngüsü bir dizi üzerinde yinelenir, bu nedenle döngü içinde bu diziyi değiştirmek beklenmedik sonuçlara yol açabilir (özellikle öğe eklerken veya çıkarırken):

alist = [0, 1, 2]
for index, value in enumerate(alist):
    alist.pop(index)
print(alist)
# Out: [1]

Not: Listeden öğeleri kaldırmak için list.pop() kullanılıyor.

İkinci öğe silinmedi çünkü yineleme dizinlerden sırayla geçiyor. Yukarıdaki döngü, aşağıdaki sonuçlarla iki kez yinelenir:

# Iteration #1
index = 0
alist = [0, 1, 2]
alist.pop(0) # removes '0'

# Iteration #2
index = 1
alist = [1, 2]
alist.pop(1) # removes '2'

# loop terminates, but alist is not empty:
alist = [1]

Bu sorun, endekslerin artan endeks yönünde yinelenirken değişmesinden kaynaklanmaktadır. Bu sorunu önlemek için döngüyü geriye doğru yineleyebilirsiniz:

alist = [1,2,3,4,5,6,7]
for index, item in reversed(list(enumerate(alist))):
    # delete all even items
    if item % 2 == 0:
        alist.pop(index)
print(alist)
# Out: [1, 3, 5, 7]

Öğeler kaldırılırken (veya eklenirken) sondan başlayarak döngü boyunca yineleme yaparak, listede daha önce yer alan öğelerin dizinlerini etkilemez. Dolayısıyla bu örnek, ‘alist’ten bile olan tüm öğeleri düzgün bir şekilde kaldıracaktır.

Benzer bir sorun, üzerinde yinelediğiniz bir listeye öğe eklerken veya eklerken ortaya çıkar ve bu, sonsuz bir döngüye neden olabilir:

alist = [0, 1, 2]
for index, value in enumerate(alist):
    # break to avoid infinite loop:
    if index == 20:     
        break           
    alist.insert(index, 'a')
print(alist)
# Out (abbreviated): ['a', 'a', ..., 'a', 'a',  0,   1,   2]

‘break’ koşulu olmadan, bilgisayarın belleği tükenmediği ve programın devam etmesine izin verildiği sürece döngü ‘a’’ ekleyecektir. Böyle bir durumda, genellikle yeni bir liste oluşturmak ve orijinal listede dolaşırken yeni listeye öğeler eklemek tercih edilir.

Bir “for” döngüsü kullanırken, liste öğelerini yer tutucu değişkenle değiştiremezsiniz:

alist = [1,2,3,4]
for item in alist:
    if item % 2 == 0:
        item = 'even'
print(alist)
# Out: [1,2,3,4]

Yukarıdaki örnekte, “öğeyi” değiştirmek aslında orijinal listedeki hiçbir şeyi değiştirmez. Liste indeksini (alist[2]) kullanmanız gerekir ve enumerate() bunun için iyi çalışır:

alist = [1,2,3,4]
for index, item in enumerate(alist):
    if item % 2 == 0:
        alist[index] = 'even'
print(alist)
# Out: [1, 'even', 3, 'even']

while döngüsü bazı durumlarda daha iyi bir seçim olabilir:

Listedeki tüm öğeleri silecekseniz:

zlist = [0, 1, 2]
while zlist:
    print(zlist[0])
    zlist.pop(0)
print('After: zlist =', zlist)

# Out: 0
#      1
#      2
# After: zlist = []

Her ne kadar sadece zlist’i sıfırlamak aynı sonucu elde edecekse de;

zlist = []

Yukarıdaki örnek, belirli bir noktadan sonra durmak veya listedeki “x” dışındaki tüm öğeleri silmek için “len()” ile de birleştirilebilir:

zlist = [0, 1, 2]
x = 1
while len(zlist) > x:
    print(zlist[0])
    zlist.pop(0)
print('After: zlist =', zlist)

# Out: 0
#      1
# After: zlist = [2]

Veya belirli bir koşulu karşılayan öğeleri silerken bir listede dolaşmak (bu durumda tüm çift öğeleri silmek):

zlist = [1,2,3,4,5]
i = 0
while i < len(zlist):
    if zlist[i] % 2 == 0:
        zlist.pop(i)
    else:
        i += 1
print(zlist)
# Out: [1, 3, 5]

Bir öğeyi sildikten sonra “i"yi artırmadığınıza dikkat edin. “zlist[i]” öğesindeki öğeyi silerek, sonraki öğenin dizini bir azaldı, bu nedenle bir sonraki yinelemede “i” için aynı değerle “zlist[i]“yi işaretleyerek, doğru bir şekilde kontrol edeceksiniz. listedeki bir sonraki öğe.

İstenmeyen öğeleri bir listeden çıkarmayı düşünmenin tersi bir yolu, istenen öğeleri yeni bir listeye eklemektir. Aşağıdaki örnek, ikinci “while” döngüsü örneğine bir alternatiftir:

zlist = [1,2,3,4,5]

z_temp = []
for item in zlist:
    if item % 2 != 0:
        z_temp.append(item)
zlist = z_temp
print(zlist)
# Out: [1, 3, 5]

Burada istenen sonuçları yeni bir listeye akıtıyoruz. Daha sonra isteğe bağlı olarak geçici listeyi orijinal değişkene yeniden atayabiliriz.

Bu düşünce eğilimiyle, Python’un en zarif ve güçlü özelliklerinden birini, geçici listeleri ortadan kaldıran ve daha önce tartışılan yerinde liste/dizin mutasyon ideolojisinden ayrılan liste kavrayışlarını çağırabilirsiniz.

zlist = [1,2,3,4,5]
[item for item in zlist if item % 2 != 0]
# Out: [1, 3, 5]

Tamsayı ve Dize kimliği #

Python, tekrarlanan oluşturma işlemlerinden kaynaklanan gereksiz ek yükü azaltmak için bir dizi tamsayı için dahili önbelleğe alma kullanır.

Aslında bu, tamsayı kimliklerini karşılaştırırken kafa karıştırıcı davranışlara yol açabilir:

>>> -8 is (-7 - 1)
False
>>> -3 is (-2 - 1)
True

ve başka bir örnek kullanarak:

>>> (255 + 1) is (255 + 1)
True
>>> (256 + 1) is (256 + 1)
False

Bir dakika ne?

Kimlik işleminin, bazı tamsayılar (’-3’, ‘256’) için ‘Doğru’ verdiğini ancak diğerleri için (’-8’, ‘257’) vermediğini görebiliriz.

Daha açık olmak gerekirse, “[-5, 256]” aralığındaki tamsayılar, yorumlayıcı başlatılırken dahili olarak önbelleğe alınır ve yalnızca bir kez oluşturulur. Bu nedenle, özdeştirler ve kimliklerini ‘is’ ile karşılaştırmak ‘True’ sonucunu verir; bu aralığın dışındaki tamsayılar (genellikle) anında oluşturulur ve kimlikleri “Yanlış” ile karşılaştırılır.

Bu yaygın bir tuzaktır, çünkü bu test için yaygın bir aralıktır, ancak genellikle yeterince, kod geliştirme aşamasında mükemmel bir şekilde çalıştıktan sonra belirgin bir neden olmaksızın sonraki aşamalandırma sürecinde (veya daha kötüsü - üretimde) başarısız olur.

Çözüm, değerleri her zaman eşitlik** (==) operatörünü kullanarak ve **kimlik (is) operatörünü değil kullanarak karşılaştırmaktır.

Python ayrıca yaygın olarak kullanılan dizelere referanslar tutar ve dizelerin kimliklerini (yani “is” kullanarak) karşılaştırırken benzer şekilde kafa karıştırıcı davranışlara neden olabilir.

>>> 'python' is 'py' + 'thon'
True

‘‘python’’ dizgisi yaygın olarak kullanılır, bu nedenle Python’un ‘‘python’’ dizgisine yapılan tüm referansların kullandığı bir nesnesi vardır.

Yaygın olmayan dizeler için, dizeler eşit olduğunda bile kimlik karşılaştırması başarısız olur.

>>> 'this is not a common string' is 'this is not' + ' a common string'
False
>>> 'this is not a common string' == 'this is not' + ' a common string'
True

Bu nedenle, Tamsayılar kuralı gibi, daima dize değerlerini eşitlik (==) operatörünü kullanarak karşılaştırın ve **kimlik (is) operatörünü değil.

Sözlükler sırasız #

Bir Python sözlüğünün örneğin bir C++ `std::map’ gibi anahtarlara göre sıralanmasını bekleyebilirsiniz, ancak durum böyle değil:

myDict = {'first': 1, 'second': 2, 'third': 3}
print(myDict)
# Out: {'first': 1, 'second': 2, 'third': 3}

print([k for k in myDict])
# Out: ['second', 'third', 'first']

Python, öğelerini anahtara göre otomatik olarak sıralayan yerleşik bir sınıfa sahip değildir.

Ancak, sıralama zorunlu değilse ve sözlüğünüzün yalnızca anahtar/değer çiftlerinin ekleme sırasını hatırlamasını istiyorsanız, ‘collections.OrderedDict’ kullanabilirsiniz:

from collections import OrderedDict

oDict = OrderedDict([('first', 1), ('second', 2), ('third', 3)])

print([k for k in oDict])
# Out: ['first', 'second', 'third']

Standart bir sözlükle bir ‘OrderedDict’ başlatmanın sözlüğü sizin için hiçbir şekilde sıralamayacağını unutmayın. Bu yapının yaptığı tek şey, anahtar yerleştirme sırasını korumaktır.

Sözlüklerin uygulanması, bellek tüketimini iyileştirmek için [Python 3.6]‘da değiştirildi(https://docs.python.org/3.6/whatsnew/3.6.html#new-dict-implementation). Bu yeni uygulamanın bir yan etkisi, bir işleve iletilen anahtar kelime argümanlarının sırasını da korumasıdır:

def func(**kw): print(kw.keys())

func(a=1, b=2, c=3, d=4, e=5) 
dict_keys(['a', 'b', 'c', 'd', 'e']) # expected order
son sürüm

Uyarı: “bu yeni uygulamanın düzeni koruyan yönü, bir uygulama detayı olarak kabul edilir ve buna güvenilmemelidir” dikkat edin 3.6.html#new-dict-implementation), gelecekte değişebileceğinden.

Liste kavramalarında ve döngüler için değişken sızıntısı #

Aşağıdaki liste kavrayışını göz önünde bulundurun

i = 0
a = [i for i in range(3)]
print(i) # Outputs 2
son sürüm

Bu, liste kavrayışının döngü kontrol değişkenini çevreleyen kapsama “sızdırması” nedeniyle yalnızca Python 2’de gerçekleşir ([kaynak](http://python-history.blogspot.com/2010/06/from-list- anlamalar-to-generator.html)). Bu davranış, bulunması zor hatalara yol açabilir ve Python 3’te düzeltilmiştir.

i = 0
a = [i for i in range(3)]
print(i) # Outputs 0
son sürüm

Benzer şekilde, for döngülerinin yineleme değişkenleri için özel bir kapsamı yoktur.

i = 0
for i in range(3):
    pass
print(i) # Outputs 2

Bu tür davranış hem Python 2’de hem de Python 3’te ortaya çıkar.

Değişkenlerin sızdırılmasıyla ilgili sorunları önlemek için, uygun olduğu şekilde liste kavramalarında ve döngüler için yeni değişkenler kullanın.

veya operatörün zincirlenmesi #

Birkaç eşitlik karşılaştırmasından herhangi birini test ederken:

if a == 3 or b == 3 or c == 3:

bunu kısaltmak cazip geliyor

if a or b or c == 3: # Wrong

Bu yanlış; ‘veya’ operatörü, ‘==’ yerine daha düşük önceliğe sahiptir, bu nedenle ifade ‘if (a) veya (b) veya (c == 3):’ olarak değerlendirilir. Doğru yol, tüm koşulları açıkça kontrol etmektir:

if a == 3 or b == 3 or c == 3:  # Right Way

Alternatif olarak, zincirleme “veya” operatörleri yerine yerleşik “any()” işlevi kullanılabilir:

if any([a == 3, b == 3, c == 3]): # Right

Veya daha verimli hale getirmek için:

if any(x == 3 for x in (a, b, c)): # Right

Veya kısaltmak için:

if 3 in (a, b, c): # Right

Burada, karşılaştırmak istediğimiz değerleri içeren bir tanımlama grubunda değerin mevcut olup olmadığını test etmek için ‘in’ operatörünü kullanırız.

Aynı şekilde yazmak yanlış

if a == 1 or 2 or 3:

olarak yazılması gereken

if a in (1, 2, 3):

int değişmezlerinin niteliklerine erişme #

Python’daki her şeyin bir nesne olduğunu duymuş olabilirsiniz, hatta değişmezler bile. Bu, örneğin, ‘7’nin de bir nesne olduğu anlamına gelir, yani öznitelikleri vardır. Örneğin, bu özniteliklerden biri “bit_uzunluğu"dur. Çağrıldığı değeri temsil etmek için gereken bit miktarını döndürür.

x = 7
x.bit_length()
# Out: 3

Yukarıdaki kodun işe yaradığını görünce, sezgisel olarak ‘7.bit_length()‘in de işe yarayacağını düşünebilirsiniz, sadece bir ‘SyntaxError’ ortaya çıkardığını bulmak için. Neden? Niye? yorumlayıcının bir öznitelik erişimi ile bir kayan sayı arasında ayrım yapması gerektiğinden (örneğin, “7.2” veya “7.bit_length()”). Yapamaz ve bu yüzden bir istisna ortaya çıkar.

Bir “int” değişmezi özniteliğine erişmenin birkaç yolu vardır:

# parenthesis
(7).bit_length()
# a space
7 .bit_length()

İki nokta kullanmak (bu “7..bit_length()” gibi) bu durumda işe yaramaz, çünkü bu bir “float” değişmez değeri oluşturur ve kayan noktalar “bit_length()” yöntemine sahip değildir.

Çevirmen, bir “kayan” değişmezinin iki “.” içeremeyeceğini bilecek kadar “akıllı” olduğundan, “kayan” değişmez değerlerine erişilirken bu sorun mevcut değildir, örneğin:

7.2.as_integer_ratio()
# Out: (8106479329266893, 1125899906842624)

Global Tercüman Kilidi (GIL) ve dizileri engelleme #

Python’un hakkında hakkında pek çok yazılı olmuştur. .org/3/glossary.html#term-global-interpreter-lock) GIL. Çok iş parçacıklı (çok işlemli ile karıştırılmaması gereken) uygulamalarla uğraşırken bazen kafa karışıklığına neden olabilir.

İşte bir örnek:

import math
from threading import Thread

def calc_fact(num):
    math.factorial(num)

num = 600000
t = Thread(target=calc_fact, daemon=True, args=[num])
print("About to calculate: {}!".format(num))
t.start()
print("Calculating...")
t.join()
print("Calculated")

İş parçacığı başlatıldıktan hemen sonra Hesaplanıyor... ifadesinin yazdırılmasını beklersiniz, sonuçta hesaplamanın yeni bir iş parçacığında yapılmasını istedik! Ama gerçekte, hesaplama tamamlandıktan sonra yazdırıldığını görüyorsunuz. Bunun nedeni, yeni iş parçacığının, yürütülürken GIL’i kilitleyecek bir C işlevine (‘math.factorial’) dayanmasıdır.

Bunun etrafında birkaç yol var. Birincisi, faktöriyel işlevinizi yerel Python’da uygulamaktır. Bu, döngünüzün içindeyken ana iş parçacığının kontrolü ele geçirmesine izin verecektir. Dezavantajı, artık C işlevini kullanmadığımız için bu çözümün çok daha yavaş olmasıdır.

def calc_fact(num):
    """ A slow version of factorial in native Python """
    res = 1
    while num >= 1:
        res = res * num
        num -= 1
    return res

Ayrıca, infazınıza başlamadan önce bir süre ‘uyuyabilirsiniz’. Not: Bu aslında programınızın C işlevi içinde gerçekleşen hesaplamayı kesmesine izin vermez, ancak ana iş parçacığınızın yumurtlamadan sonra devam etmesine izin verir, ki bu beklediğiniz şeydir.

def calc_fact(num):
    sleep(0.001)
    math.factorial(num)

Çoklu dönüş #

xyz işlevi iki a ve b değeri döndürür:

def xyz():
  return a, b

Kod çağırma xyz, xyz’nin yalnızca bir değer döndürdüğünü varsayarak sonucu tek bir değişkende saklar:

t = xyz()

“t"nin değeri aslında bir tanımlama grubudur (a, b) bu ​​nedenle, tanımlama grubu olmadığını varsayarak “t” üzerindeki herhangi bir eylem, tanımlama grupları hakkında beklenmeyen bir hata ile kodda derin başarısız olabilir.

TypeError: type tuple … yöntemini tanımlamıyor

Düzeltme yapmak olacaktır:

a, b = xyz()

Yeni başlayanlar, yalnızca demet hata mesajını okuyarak bu mesajın nedenini bulmakta zorlanacaklar!

Pythonic JSON anahtarları #

my_var = 'bla';
api_key = 'key';
...lots of code here...
params = {"language": "en", my_var: api_key}

JavaScript’e alışkınsanız, Python sözlüklerinde değişken değerlendirmesi beklediğiniz gibi olmayacaktır. JavaScript’teki bu ifade, aşağıdaki gibi “params” nesnesiyle sonuçlanır:

{
    "language": "en",
    "my_var": "key"
}

Ancak Python’da aşağıdaki sözlükle sonuçlanır:

{
    "language": "en",
    "bla": "key"
}

my_var değerlendirilir ve değeri anahtar olarak kullanılır.
