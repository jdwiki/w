Tasarım deseni, yazılım geliştirmede yaygın olarak ortaya çıkan bir soruna genel bir çözümdür. Bu belgeleme konusu, özellikle Python’daki yaygın tasarım modellerinin örneklerini sağlamayı amaçlamaktadır.

Tasarım desenlerine ve Singleton Pattern’e giriş #

Tasarım Kalıpları, yazılım tasarımında ‘sık karşılaşılan sorunlara’ çözümler sunar. Tasarım kalıpları ilk olarak ‘GoF(Dörtlü Çete)’ tarafından tanıtıldı, burada ortak kalıpları tekrar tekrar ortaya çıkan problemler ve bu problemlere çözümler olarak tanımladılar.

Tasarım kalıplarının dört temel öğesi vardır:

  1. “Desen adı”, bir tasarım problemini, çözümlerini ve sonuçlarını bir veya iki kelimeyle tanımlamak için kullanabileceğimiz bir tutamaçtır.
  2. “Sorun”, kalıbın ne zaman uygulanacağını açıklar.
  3. ‘Çözüm’ tasarımı oluşturan öğeleri, bunların ilişkilerini, sorumluluklarını ve işbirliklerini tanımlar.
  4. “Sonuçlar”, kalıbı uygulamanın sonuçları ve takaslarıdır.

Tasarım desenlerinin avantajları:

  1. Birden çok projede yeniden kullanılabilirler.
  2. Mimari düzeydeki problemler çözülebilir
  3. Geliştiricilerin ve mimarların deneyimi olan, zaman içinde test edilmiş ve kanıtlanmışlardır.
  4. Güvenilirlik ve bağımlılıkları vardır

Tasarım kalıpları üç kategoriye ayrılabilir:

  1. Yaratılış Modeli
  2. Yapısal Model
  3. Davranışsal Model

Creational Pattern - Nesnenin nasıl oluşturulabileceği ile ilgilenirler ve nesne yaratmanın ayrıntılarını izole ederler.

Structural Pattern - Daha büyük sonuçlar elde etmek için oluşturabilmeleri için sınıfların ve nesnelerin yapısını tasarlarlar.

‘Davranış Modeli’ - Nesneler arasındaki etkileşim ve nesnelerin sorumluluğu ile ilgilenirler.

Singleton Desen:

Belirli bir türde yalnızca bir ve bir nesneye sahip olmak için bir mekanizma sağlayan ve küresel bir erişim noktası sağlayan bir “yaratıcı model” türüdür.

Örneğin. Singleton, veritabanı nesnesinin veri tutarlılığını korumasını istediğimiz veritabanı işlemlerinde kullanılabilir.

Uygulama

Singleton sınıfının yalnızca bir örneğini oluşturarak ve aynı nesneyi tekrar sunarak Python’da Singleton Pattern uygulayabiliriz.

class Singleton(object):
    def __new__(cls):
        # hasattr method checks if the class object an instance property or not.
        if not hasattr(cls, 'instance'):
            cls.instance = super(Singleton, cls).__new__(cls)
        return cls.instance

s = Singleton()
print ("Object created", s)

s1 = Singleton()
print ("Object2 created", s1)

Çıktı:

('Object created', <__main__.Singleton object at 0x10a7cc310>)
('Object2 created', <__main__.Singleton object at 0x10a7cc310>)

C++ veya Java gibi dillerde bu kalıbın, yapıcıyı özel hale getirerek ve nesne başlatma işlemini yapan statik bir yöntem oluşturarak uygulandığını unutmayın. Bu şekilde, ilk çağrıda bir nesne oluşturulur ve sınıf daha sonra aynı nesneyi döndürür. Ancak Python’da özel kurucular yaratmanın herhangi bir yolu yok.

Fabrika Modeli

Fabrika kalıbı aynı zamanda bir ‘Yaratılış kalıbı’dır. “Fabrika” terimi, bir sınıfın diğer türlerdeki nesneleri oluşturmaktan sorumlu olduğu anlamına gelir. Kendisiyle ilişkili nesneleri ve yöntemleri olan bir fabrika gibi davranan bir sınıf vardır. İstemci, belirli parametrelerle yöntemleri çağırarak bir nesne oluşturur ve fabrika, istenen türde nesneyi oluşturur ve istemciye geri gönderir.

from abc import ABCMeta, abstractmethod

class Music():
    __metaclass__ = ABCMeta
    @abstractmethod
    def do_play(self):
        pass

class Mp3(Music):
    def do_play(self):
        print ("Playing .mp3 music!")
    
class Ogg(Music):
    def do_play(self):
        print ("Playing .ogg music!")
    
class MusicFactory(object):
    def play_sound(self, object_type):
        return eval(object_type)().do_play()
    
if __name__ == "__main__":
    mf = MusicFactory()
    music = input("Which music you want to play Mp3 or Ogg")
    mf.play_sound(music)

Çıktı:

Which music you want to play Mp3 or Ogg"Ogg"
Playing .ogg music!

“MusicFactory”, kullanıcının sağladığı seçime bağlı olarak “Mp3” veya “Ogg” türünde bir nesne oluşturan fabrika sınıfıdır.

Strateji Modeli #

Bu tasarım kalıbına Strateji Kalıbı denir. Bir algoritma ailesini tanımlamak için kullanılır, her birini kapsüller. bir ve onları değiştirilebilir hale getirin. strateji tasarım deseni, bir algoritmanın bağımsız olarak değişmesine izin verir onu kullanan müşterilerden.

Örneğin, hayvanlar birçok farklı şekilde “yürüyebilir”. Yürümek, farklı hayvan türleri tarafından uygulanan bir strateji olarak düşünülebilir:

from types import MethodType


class Animal(object):
    
    def __init__(self, *args, **kwargs):
        self.name = kwargs.pop('name', None) or 'Animal'
        if kwargs.get('walk', None):
            self.walk = MethodType(kwargs.pop('walk'), self)

    def walk(self):
        """
        Cause animal instance to walk
        
        Walking funcionallity is a strategy, and is intended to
        be implemented separately by different types of animals.
        """
        message = '{} should implement a walk method'.format(
            self.__class__.__name__)
        raise NotImplementedError(message)


# Here are some different walking algorithms that can be used with Animal
def snake_walk(self):
    print('I am slithering side to side because I am a {}.'.format(self.name))

def four_legged_animal_walk(self):
    print('I am using all four of my legs to walk because I am a(n) {}.'.format(
        self.name))

def two_legged_animal_walk(self):
    print('I am standing up on my two legs to walk because I am a {}.'.format(
        self.name))

Bu örneği çalıştırmak aşağıdaki çıktıyı üretecektir:

generic_animal = Animal()
king_cobra = Animal(name='King Cobra', walk=snake_walk)
elephant = Animal(name='Elephant', walk=four_legged_animal_walk)
kangaroo = Animal(name='Kangaroo', walk=two_legged_animal_walk)

kangaroo.walk()
elephant.walk()
king_cobra.walk()
# This one will Raise a NotImplementedError to let the programmer
# know that the walk method is intended to be used as a strategy.
generic_animal.walk()

    # OUTPUT:
    #
    # I am standing up on my two legs to walk because I am a Kangaroo.
    # I am using all four of my legs to walk because I am a(n) Elephant.
    # I am slithering side to side because I am a King Cobra.
    # Traceback (most recent call last):
    #   File "./strategy.py", line 56, in <module>
    #     generic_animal.walk()
    #   File "./strategy.py", line 30, in walk
    #     raise NotImplementedError(message)
    # NotImplementedError: Animal should implement a walk method

C++ veya Java gibi dillerde, bu kalıbın bir strateji tanımlamak için soyut bir sınıf veya bir arayüz kullanılarak uygulandığına dikkat edin. Python’da, “types.MethodType” kullanılarak bir sınıfa dinamik olarak eklenebilecek bazı işlevleri harici olarak tanımlamak daha mantıklıdır.

Proxy #

Proxy nesnesi genellikle, dahili iş mantığının güvenlik gereksinimleriyle kirletmesini istemediğimiz başka bir nesneye korumalı erişim sağlamak için kullanılır.

Yalnızca belirli izinlere sahip kullanıcının kaynağa erişebileceğini garanti etmek istediğimizi varsayalım.

Proxy tanımı: (yalnızca rezervasyonları gerçekten görebilen kullanıcıların tüketici rezervasyon hizmetini kullanabilmesini sağlar)

from datetime import date
from operator import attrgetter

class Proxy:
    def __init__(self, current_user, reservation_service):
        self.current_user = current_user
        self.reservation_service = reservation_service

    def highest_total_price_reservations(self, date_from, date_to, reservations_count):
        if self.current_user.can_see_reservations:
            return self.reservation_service.highest_total_price_reservations(
                date_from,
                date_to,
                reservations_count
              )
        else:
            return []

#Models and ReservationService:

class Reservation:
    def __init__(self, date, total_price):
        self.date = date
        self.total_price = total_price

class ReservationService:
    def highest_total_price_reservations(self, date_from, date_to, reservations_count):
        # normally it would be read from database/external service
        reservations = [
            Reservation(date(2014, 5, 15), 100),
            Reservation(date(2017, 5, 15), 10),
            Reservation(date(2017, 1, 15), 50)
        ]

        filtered_reservations = [r for r in reservations if (date_from <= r.date <= date_to)]

        sorted_reservations = sorted(filtered_reservations, key=attrgetter('total_price'), reverse=True)

        return sorted_reservations[0:reservations_count]


class User:
    def __init__(self, can_see_reservations, name):
        self.can_see_reservations = can_see_reservations
        self.name = name

#Consumer service:

class StatsService:
    def __init__(self, reservation_service):
        self.reservation_service = reservation_service

    def year_top_100_reservations_average_total_price(self, year):
        reservations = self.reservation_service.highest_total_price_reservations(
            date(year, 1, 1),
            date(year, 12, 31),
            1
        )

        if len(reservations) > 0:
            total = sum(r.total_price for r in reservations)

            return total / len(reservations)
        else:
            return 0

#Test:
def test(user, year):
    reservations_service = Proxy(user, ReservationService())
    stats_service = StatsService(reservations_service)
    average_price = stats_service.year_top_100_reservations_average_total_price(year)
    print("{0} will see: {1}".format(user.name, average_price))

test(User(True, "John the Admin"), 2017)
test(User(False, "Guest"),         2017)

FAYDALAR #

  • erişim kısıtlamaları değiştiğinde ReservationServicede herhangi bir değişiklikten kaçınıyoruz.
  • işle ilgili verileri (“date_from”, “date_to”, “reservations_count”) hizmetteki alanla ilgisi olmayan kavramlarla (kullanıcı izinleri) karıştırmıyoruz.
  • Tüketici (StatsService) mantıkla ilgili izinlerden de muaftır

ÖNLEMLER #

  • Proxy arabirimi her zaman gizlediği nesneyle tamamen aynıdır, böylece proxy tarafından sarılmış hizmeti tüketen kullanıcı proxy varlığından bile haberdar değildir.
