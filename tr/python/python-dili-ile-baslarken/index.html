Başlarken #

Python, Guido van Rossum tarafından yaratılan ve ilk olarak 1991’de piyasaya sürülen genel amaçlı programlama için yaygın olarak kullanılan üst düzey bir programlama dilidir. Python, dinamik bir tip sistemi ve otomatik bellek yönetimine sahiptir ve nesne yönelimli, zorunlu, zorunlu, fonksiyonel programlama ve prosedürel stiller. Geniş ve kapsamlı bir standart kütüphaneye sahiptir.

Python’un iki ana sürümü şu anda aktif olarak kullanılıyor:

  • Python 3.x güncel sürümdür ve aktif geliştirme aşamasındadır.
  • Python 2.x eski sürümdür ve 2020’ye kadar yalnızca güvenlik güncellemelerini alacaktır. Hiçbir yeni özellik uygulanmayacaktır. Python 3’e geçiş kolaylaşıyor olsa da, birçok projenin hala Python 2 kullandığını unutmayın.

Python’un herhangi bir sürümünü buradan indirip yükleyebilirsiniz. Aralarında bir karşılaştırma için Python 3 vs. Python 2 bölümüne bakın. Ayrıca, bazı üçüncü taraflar, matematik, veri analizi veya bilimsel kullanım gibi yaygın kullanım durumları için kurulumu kolaylaştırmak için yaygın olarak kullanılan kitaplıkları ve diğer özellikleri ekleyen Python’un yeniden paketlenmiş sürümlerini sunar. Resmi sitedeki listeye bakın.

Python’un kurulu olup olmadığını doğrulayın #

Python’un doğru şekilde kurulduğunu doğrulamak için, favori terminalinizde aşağıdaki komutu çalıştırarak bunu doğrulayabilirsiniz (Windows işletim sistemi kullanıyorsanız, komut isteminde kullanmadan önce ortam değişkenine python yolunu eklemeniz gerekir):

$ python --version

Python 3 kuruluysa ve bu sizin varsayılan sürümünüzse (daha fazla ayrıntı için Sorun Giderme bölümüne bakın) şöyle bir şey görmelisiniz:

$ python --version
Python 3.6.0
son sürüm

Python 2 kuruluysa ve bu sizin varsayılan sürümünüzse (daha fazla ayrıntı için Sorun Giderme bölümüne bakın) şöyle bir şey görmelisiniz:

$ python --version
Python 2.7.13
son sürüm

Python 3’ü yüklediyseniz, ancak “$ python –version” bir Python 2 sürümü çıkarıyorsa, Python 2’yi de yüklemişsinizdir. Bu genellikle MacOS’ta ve birçok Linux dağıtımında geçerlidir. Python 3 yorumlayıcısını açıkça kullanmak için bunun yerine `$ python3’ kullanın.

Değişkenler oluşturma ve değer atama #

Python’da bir değişken oluşturmak için tek yapmanız gereken değişken adını belirlemek ve ardından ona bir değer atamak.

<variable name> = <value>

Python, değişkenlere değer atamak için = kullanır. Önceden bir değişken tanımlamaya (veya ona bir veri tipi atamaya) gerek yoktur, bir değişkene bir değer atamak, değişkeni o değerle bildirir ve başlatır. Bir değişkene bir başlangıç ​​değeri atamadan tanımlamanın bir yolu yoktur.

# Integer
a = 2
print(a)
# Output: 2

# Integer    
b = 9223372036854775807
print(b)
# Output: 9223372036854775807

# Floating point
pi = 3.14
print(pi)
# Output: 3.14

# String
c = 'A'
print(c)
# Output: A

# String    
name = 'John Doe'
print(name)
# Output: John Doe

# Boolean    
q = True
print(q)
# Output: True

# Empty value or null data type
x = None
print(x)
# Output: None

Değişken ataması soldan sağa çalışır. Yani aşağıdaki size bir sözdizimi hatası verecektir.

0 = x
=> Output: SyntaxError: can't assign to literal

Python’un anahtar sözcüklerini geçerli bir değişken adı olarak kullanamazsınız. Anahtar kelime listesini şu şekilde görebilirsiniz:

import keyword
print(keyword.kwlist)

Değişken adlandırma kuralları:

  1. Değişken isimleri bir harf veya alt çizgi ile başlamalıdır.

    x = True # valid _y = True # valid

    9x = False # starts with numeral => SyntaxError: invalid syntax

    $y = False # starts with symbol => SyntaxError: invalid syntax

  2. Değişken adınızın geri kalanı harflerden, rakamlardan ve alt çizgilerden oluşabilir.

    has_0_in_it = “Still Valid”

  3. İsimler büyük/küçük harf duyarlıdır.

    x = 9
    y = X*5
    =>NameError: name ‘X’ is not defined

Python’da bir değişken bildirirken bir veri türü belirtmeye gerek olmamasına rağmen, değişken için bellekte gerekli alanı ayırırken, Python yorumlayıcısı otomatik olarak bunun için en uygun yerleşik türü seçer:

a = 2

b = 9223372036854775807

# Output: <type 'int'>

pi = 3.14
print(type(pi))

# Output: <type 'bool'>

x = None

# Output: <type 'NoneType'>

Artık atamanın temellerini biliyorsunuz, hadi python’da atama hakkındaki bu inceliği aradan çıkaralım.

Bir atama işlemi yapmak için = kullandığınızda, = solundaki şey sağdaki nesne için bir ad‘dır. Son olarak, = sağdaki nesnenin referansını soldaki isme atamaktır.

Yani:

a_name = an_object  # "a_name" is now a name for the reference to the object "an_object"

Dolayısıyla, yukarıdaki birçok atama örneğinde, eğer pi = 3.14’ü seçersek, o zaman ‘pi’, ‘3.14’ nesnesi için a addır (bir nesnenin birden fazla adı olabileceğinden ad değil). `. Aşağıdaki bir şeyi anlamadıysanız, bu noktaya geri dönün ve bunu tekrar okuyun! Ayrıca, daha iyi anlamak için [bu2]‘ya da bakabilirsiniz.

Bir satırda birden çok değişkene birden çok değer atayabilirsiniz. = operatörünün sağ ve sol taraflarında aynı sayıda argüman olması gerektiğini unutmayın:

a, b, c = 1, 2, 3
print(a, b, c)
# Output: 1 2 3

a, b, c = 1, 2
=> Traceback (most recent call last):
=>   File "name.py", line N, in <module>
=>     a, b, c = 1, 2
=> ValueError: need more than 2 values to unpack

a, b = 1, 2, 3
=> Traceback (most recent call last):
=>   File "name.py", line N, in <module>
=>     a, b = 1, 2, 3
=> ValueError: too many values to unpack

Son örnekteki hata, kalan değerleri eşit sayıda rastgele değişkene atayarak önlenebilir. Bu kukla değişken herhangi bir ada sahip olabilir, ancak istenmeyen değerleri atamak için alt çizgi (_) kullanmak gelenekseldir:

a, b, _ = 1, 2, 3
print(a, b)
# Output: 1, 2

_ sayısının ve kalan değerlerin sayısının eşit olması gerektiğini unutmayın. Aksi takdirde, ‘paketi açmak için çok fazla değer’ yukarıdaki gibi atılır:

a, b, _ = 1,2,3,4
=>Traceback (most recent call last):
=>File "name.py", line N, in <module>
=>a, b, _ = 1,2,3,4
=>ValueError: too many values to unpack (expected 3)

Aynı anda birkaç değişkene tek bir değer de atayabilirsiniz.

a = b = c = 1
print(a, b, c)
# Output: 1 1 1

Bu tür basamaklı atamayı kullanırken, “a”, “b” ve “c” değişkenlerinin üçünün de bellekteki aynı nesneye, 1 değerine sahip bir “int” nesnesine atıfta bulunduğuna dikkat etmek önemlidir. , “a”, “b” ve “c” aynı int nesnesine verilen üç farklı addır. Daha sonra bunlardan birine farklı bir nesne atamak, beklendiği gibi diğerlerini değiştirmez:

a = b = c = 1    # all three names a, b and c refer to same int object with value 1
print(a, b, c)
# Output: 1 1 1
b = 2            # b now refers to another int object, one with a value of 2
print(a, b, c)
# Output: 1 2 1  # so output is as expected.

Yukarıdakiler, değişmez türler için olduğu gibi (‘int’, ‘string’, ’tuple’ vb. gibi) değiştirilebilir türler için de (’list’, ‘dict’ vb.) geçerlidir:

x = y = [7, 8, 9]   # x and y refer to the same list object just created, [7, 8, 9]
x = [13, 8, 9]      # x now refers to a different list object just created, [13, 8, 9]
print(y)            # y still refers to the list it was first assigned
# Output: [7, 8, 9]

Çok uzak çok iyi. Değişken türler için basamaklı atama kullanıldığında, nesneyi değiştirmek söz konusu olduğunda (yukarıda yaptığımız gibi adı farklı bir nesneye * atamanın* aksine) işler biraz farklıdır. Aşağıya bir göz atın ve ilk elden göreceksiniz:

x = y = [7, 8, 9]     # x and y are two different names for the same list object just created, [7, 8, 9]
x[0] = 13             # we are updating the value of the list [7, 8, 9] through one of its names, x in this case
print(y)              # printing the value of the list using its other name
# Output: [13, 8, 9]  # hence, naturally the change is reflected

İç içe listeler python’da da geçerlidir. Bu, bir listenin öğe olarak başka bir liste içerebileceği anlamına gelir.

x = [1, 2, [3, 4, 5], 6, 7] # this is nested list
print x[2]
# Output: [3, 4, 5]
print x[2][1]
# Output: 4

Son olarak, Python’daki değişkenlerin ilk tanımlandıkları türde kalması gerekmez - bir değişkene yeni bir değer atamak için, bu değer farklı türde olsa bile `=’ kullanabilirsiniz.

a = 2 
print(a)
# Output: 2

a = "New value"
print(a)
# Output: New value

Bu sizi rahatsız ediyorsa, = işaretinin solundakinin sadece bir nesnenin adı olduğunu düşünün. Önce 2 ‘a’ değerine sahip ‘int’ nesnesini çağırırsınız, sonra fikrinizi değiştirir ve ‘Yeni değer’ değerine sahip bir ‘string’ nesnesine ‘a’ adını vermeye karar verirsiniz. Basit, değil mi?

Girintiyi Engelle #

Python, kontrol ve döngü yapılarını tanımlamak için girinti kullanır. Bu Python’un okunabilirliğine katkıda bulunur, ancak programcının boşluk kullanımına çok dikkat etmesini gerektirir. Bu nedenle, düzenleyici yanlış kalibrasyonu, beklenmedik şekillerde davranan kodla sonuçlanabilir.

Python, kod bloklarının nerede başlayıp nerede bittiğini göstermek için iki nokta üst üste sembolünü (:) ve girintiyi kullanır (Başka bir dilden geliyorsanız, bunu bir şekilde üçlü operatör ile ilgili olmakla karıştırmayın). Diğer bir deyişle, Python’daki işlevler, döngüler, ‘if’ yan tümceleri ve diğer yapılar gibi blokların bitiş tanımlayıcıları yoktur. Tüm bloklar iki nokta üst üste ile başlar ve ardından altındaki girintili satırları içerir.

Örneğin:

def my_function():    # This is a function definition. Note the colon (:)
    a = 2             # This line belongs to the function because it's indented
    return a          # This line also belongs to the same function
print(my_function())  # This line is OUTSIDE the function block

veya

if a > b:             # If block starts here
    print(a)          # This is part of the if block
else:                 # else must be at the same level as if
    print(b)          # This line is part of the else block

Tam olarak bir tek satırlık ifade içeren bloklar aynı satıra yerleştirilebilir, ancak bu form genellikle iyi bir stil olarak kabul edilmez:

if a > b: print(a)
else: print(b)

Bir “küme”, tekrarı olmayan ve ekleme sırası olmayan ancak sıralanmış bir öğeler topluluğudur. Bir “küme” tanımlamak, bir “sözlük” tanımlamaya çok benzer:

varsayılan hüküm

Bir “defaultdict”, anahtarlar için varsayılan bir değere sahip bir sözlüktür, böylece hiçbir değeri açıkça tanımlanmamış anahtarlara hatasız olarak erişilebilir. ‘defaultdict’, özellikle sözlükteki değerler koleksiyonlar (listeler, dikteler, vb.) olduğunda, her yeni anahtar kullanıldığında başlatılması gerekmediğinden kullanışlıdır.

Bir “defaultdict” asla bir KeyError oluşturmaz. Var olmayan herhangi bir anahtar, döndürülen varsayılan değeri alır.

Örneğin, aşağıdaki sözlüğü düşünün

>>> state_capitals = {
    'Arkansas': 'Little Rock',
    'Colorado': 'Denver',
    'California': 'Sacramento', 
    'Georgia': 'Atlanta'
}

Var olmayan bir anahtara erişmeye çalışırsak python bize aşağıdaki gibi bir hata verir.

>>> state_capitals['Alabama']
Traceback (most recent call last):

  File "<ipython-input-61-236329695e6f>", line 1, in <module>
    state_capitals['Alabama']

KeyError: 'Alabama'

Bir ‘defaultdict’ ile deneyelim. Koleksiyon modülünde bulunabilir.

>>> from collections import defaultdict
>>> state_capitals = defaultdict(lambda: 'Boston')

Burada yaptığımız, verme anahtarının olmaması durumunda varsayılan bir değer (Boston) belirlemektir. Şimdi dict’i daha önce olduğu gibi doldurun:

>>> state_capitals['Arkansas'] = 'Little Rock'
>>> state_capitals['California'] = 'Sacramento'
>>> state_capitals['Colorado'] = 'Denver'
>>> state_capitals['Georgia'] = 'Atlanta'

Var olmayan bir anahtarla dict’e erişmeye çalışırsak, python bize varsayılan değeri, yani Boston’ı döndürür.

>>> state_capitals['Alabama']
'Boston'

ve normal bir “sözlük” gibi mevcut anahtar için oluşturulan değerleri döndürür

>>> state_capitals['Arkansas']
'Little Rock'

Modül oluşturma #

Modül, tanımları ve ifadeleri içeren içe aktarılabilir bir dosyadır.

Bir .py dosyası oluşturularak bir modül oluşturulabilir.

# hello.py
def say_hello():
    print("Hello!")

Bir modüldeki işlevler, modül içe aktarılarak kullanılabilir.

Yaptığınız modüller için, onları içe aktardığınız dosyayla aynı dizinde olmaları gerekir. (Ancak, bunları önceden dahil edilmiş modüllerle Python lib dizinine de koyabilirsiniz, ancak mümkünse kaçınılmalıdır.)

$ python
>>> import hello
>>> hello.say_hello()
=> "Hello!"

Modüller diğer modüller tarafından içe aktarılabilir.

# greet.py
import hello
hello.say_hello()

Bir modülün belirli işlevleri içe aktarılabilir.

# greet.py
from hello import say_hello
say_hello()

Modüllere takma ad verilebilir.

# greet.py
import hello as ai
ai.say_hello()

Bir modül tek başına çalıştırılabilir komut dosyası olabilir.

# run_hello.py
if __name__ == '__main__':
    from hello import say_hello
    say_hello()

Çalıştır!

$ python run_hello.py
=> "Hello!"

Modül bir dizinin içindeyse ve python tarafından algılanması gerekiyorsa, dizin ‘init.py’ adlı bir dosya içermelidir.

Dize işlevi - str() ve repr() #

Bir nesnenin okunabilir bir temsilini elde etmek için kullanılabilecek iki işlev vardır.

“repr(x)”, “x.repr()” öğesini çağırır: “x"in bir temsili. ’eval’ genellikle bu işlevin sonucunu orijinal nesneye geri döndürür.

str(x), x.__str__() çağırır: nesneyi tanımlayan insan tarafından okunabilir bir dize. Bu, bazı teknik ayrıntıları atlayabilir.

tekrar() #

Birçok tür için bu işlev, ’eval()’ öğesine iletildiğinde aynı değere sahip bir nesne verecek bir dize döndürmeye çalışır. Aksi takdirde, gösterim, ek bilgilerle birlikte nesnenin türünün adını içeren köşeli parantez içine alınmış bir dizedir. Bu genellikle nesnenin adını ve adresini içerir.

str() #

Dizeler için bu, dizenin kendisini döndürür. Bununla “repr(nesne)” arasındaki fark, “str(nesne)“nin her zaman “eval()” için kabul edilebilir bir dize döndürmeye çalışmamasıdır. Bunun yerine amacı, yazdırılabilir veya ‘insan tarafından okunabilir’ bir dize döndürmektir. Argüman verilmezse, bu boş dizeyi döndürür ''.

Örnek 1:

s = """w'o"w"""
repr(s) # Output: '\'w\\\'o"w\''  
str(s)  # Output: 'w\'o"w'
eval(str(s)) == s  # Gives a SyntaxError 
eval(repr(s)) == s # Output: True

Örnek 2:

import datetime
today = datetime.datetime.now()
str(today)  # Output: '2016-09-15 06:58:46.915000'
repr(today) # Output: 'datetime.datetime(2016, 9, 15, 6, 58, 46, 915000)'

Bir sınıf yazarken, istediğinizi yapmak için bu yöntemleri geçersiz kılabilirsiniz:

class Represent(object):

    def __init__(self, x, y):
        self.x, self.y = x, y

    def __repr__(self):
        return "Represent(x={},y=\"{}\")".format(self.x, self.y)

    def __str__(self):
        return "Representing x as {} and y as {}".format(self.x, self.y)

Yukarıdaki sınıfı kullanarak sonuçları görebiliriz:

r = Represent(1, "Hopper")
print(r)  # prints __str__
print(r.__repr__)  # prints __repr__: '<bound method Represent.__repr__ of Represent(x=1,y="Hopper")>'
rep = r.__repr__()  # sets the execution of __repr__ to a new variable
print(rep)  # prints 'Represent(x=1,y="Hopper")'
r2 = eval(rep) # evaluates rep
print(r2)  # prints __str__ from new object
print(r2 == r)  # prints 'False' because they are different objects

pip kullanarak harici modüller yükleme #

‘pip’, python paket dizininde (PyPI) bulunan çok sayıda seçenek arasından herhangi bir paketi yüklemeniz gerektiğinde arkadaşınızdır. Python 2 >= 2.7.9 veya python.org’dan indirilen Python 3 >= 3.4 kullanıyorsanız “pip” zaten kuruludur. Yerel bir paket yöneticisine sahip Linux veya başka bir *nix çalıştıran bilgisayarlar için, ‘pip’ genellikle manuel olarak yüklenmelidir.

Hem Python 2 hem de Python 3’ün kurulu olduğu durumlarda, “pip” genellikle Python 2’yi ve “pip3”, Python 3’ü ifade eder. “pip” kullanılması yalnızca Python 2 için paketleri kurar ve “pip3” yalnızca Python 3 için paketleri kurar.

Paket bulma / yükleme #

Paket aramak, yazmak kadar basittir

$ pip search <query>
# Searches for packages whose name or summary contains <query>

Bir paketi kurmak (Python yorumlayıcısında değil, bir terminalde / komut isteminde) yazmak kadar basittir

$ pip install [package_name]           # latest version of the package

$ pip install [package_name]==x.x.x    # specific version of the package

$ pip install '[package_name]>=x.x.x'  # minimum version of the package

burada “x.x.x”, yüklemek istediğiniz paketin sürüm numarasıdır.

Sunucunuz proxy arkasındayken, aşağıdaki komutu kullanarak paketi yükleyebilirsiniz:

$ pip --proxy http://<server address>:<port> install

Kurulu paketleri yükseltme #

Kurulu paketlerin yeni sürümleri göründüğünde, sisteminize otomatik olarak yüklenmezler. Yüklü paketlerinizden hangilerinin güncelliğini yitirdiğine dair bir genel bakış elde etmek için şunu çalıştırın:

$ pip list --outdated

Belirli bir paketi yükseltmek için kullanın

$ pip install [package_name] --upgrade

Tüm güncel olmayan paketleri güncellemek standart bir “pip” işlevi değildir.

Aşağıdaki komutları kullanarak mevcut pip kurulumunuzu yükseltebilirsiniz.

  • Linux veya macOS X’te:

    $ pip install -U pip

    You may need to use sudo with pip on some Linux Systems

  • Windows’ta:

    py -m pip install -U pip

    or

    python -m pip install -U pip
