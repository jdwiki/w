Neden bir GIL var?

GIL, 1992'de Python iş parçacıklarının başlangıcından bu yana CPython'da olmuştur. Python kodunu çalıştırmanın iş parçacığı güvenliğini sağlamak için tasarlanmıştır. Bir GIL ile yazılmış Python yorumlayıcıları, birden çok yerel iş parçacığının Python bayt kodlarını aynı anda yürütmesini engeller. Bu, eklentilerin kodlarının iş parçacığı açısından güvenli olduğundan emin olmalarını kolaylaştırır: GIL'i kilitleyin ve yalnızca etkin iş parçacığınız çalışabilir, böylece kodunuz otomatik olarak iş parçacığı için güvenli olur.

Kısa versiyon: GIL, kaç tane işlemciniz ve iş parçacığınız olursa olsun, bir python yorumlayıcısının yalnızca bir iş parçacığının aynı anda çalışmasını sağlar.

Bunun birçok kullanım kolaylığı avantajı vardır, ancak aynı zamanda birçok olumsuz faydası da vardır.

GIL’in Python dilinin bir gereksinimi olmadığını unutmayın. Sonuç olarak, GIL’e doğrudan standart python kodundan erişemezsiniz. Python’un tüm uygulamaları bir GIL kullanmaz.

GIL’e sahip tercümanlar: CPython, PyPy, Cython (ancak GIL’i nogil ile devre dışı bırakabilirsiniz)

GIL’i olmayan tercümanlar: Jython, IronPython

GIL'in nasıl çalıştığına ilişkin ayrıntılar:

Bir iş parçacığı çalışırken, GIL'i kilitler. Bir iş parçacığı çalıştırmak istediğinde GIL'i ister ve kullanılabilir olana kadar bekler. CPython'da, sürüm 3.2'den önce, çalışan iş parçacığı, başka bir kodun kilidi isteyip istemediğini görmek için belirli sayıda python talimatından sonra kontrol ederdi (yani, kilidi serbest bıraktı ve sonra tekrar istedi). Bu yöntem iş parçacığı açlığına neden olma eğilimindeydi, çünkü büyük ölçüde kilidi serbest bırakan iş parçacığı, bekleyen iş parçacıklarının uyanma şansı bulamadan onu tekrar ele geçirecekti. 3.2'den beri, GIL'i isteyen iş parçacıkları bir süre kilidi bekler ve bu süreden sonra, çalışan iş parçacığını vermeye zorlayan paylaşılan bir değişken ayarlarlar. Bu yine de büyük ölçüde daha uzun yürütme sürelerine neden olabilir. Daha fazla ayrıntı için dabeaz.com'dan (referanslar bölümünde) aşağıdaki bağlantılara bakın.

CPython, bir iş parçacığı bir G/Ç işlemi gerçekleştirdiğinde GIL’yi otomatik olarak serbest bırakır. Görüntü işleme kitaplıkları ve numpy sayı kırma işlemleri, işlemlerini yapmadan önce GIL’yi serbest bırakır.

GIL'in Faydaları

GIL kullanan tercümanlar için GIL sistemiktir. Uygulamanın durumunu korumak için kullanılır. Avantajlar şunları içerir:
  • Çöp toplama - GIL kilitliyken iş parçacığı güvenli referans sayıları değiştirilmelidir. CPython’da çöp toplamanın tamamı GIL’e bağlıdır. Bu büyük bir şey; GIL’in kaldırılması istendiğinde hala neyin işlevsel olması gerektiğine ilişkin ayrıntılar için GIL hakkındaki python.org wiki makalesine bakın (aşağıdaki Referanslar’da listelenmiştir).
  • GIL ile uğraşan programcılar için kolaylık - her şeyi kilitlemek basittir, ancak kodlaması kolaydır
  • Modüllerin diğer dillerden içe aktarılmasını kolaylaştırır

GIL'in Sonuçları

GIL, python yorumlayıcısı içinde aynı anda yalnızca bir iş parçacığının python kodunu çalıştırmasına izin verir. Bu, katı python kodu çalıştıran süreçlerin çoklu okumasının basitçe çalışmadığı anlamına gelir. GIL'e karşı iş parçacıkları kullanırken, tek bir iş parçacığında koştuğunuzdan daha kötü iş parçacıkları performansına sahip olacaksınız.

Çoklu İşlem.Havuz #

Python’da iş parçacıklarının nasıl kullanılacağı sorulduğunda verilecek basit yanıt şudur: “Yapma. Bunun yerine süreçleri kullanın.” Çoklu işlem modülü, iş parçacığı oluşturmaya benzer sözdizimine sahip işlemler oluşturmanıza izin verir, ancak ben onların uygun Pool nesnesini kullanmayı tercih ederim.

David Beazley’nin GIL’e karşı iş parçacıklarının tehlikelerini göstermek için ilk kullandığı kodu kullanarak, multiprocessing.Pool kullanarak yeniden yazacağız:

GIL iş parçacığı oluşturma sorunlarını gösteren David Beazley kodu

from threading import Thread
import time
def countdown(n):
    while n > 0:
        n -= 1

COUNT = 10000000

t1 = Thread(target=countdown,args=(COUNT/2,))
t2 = Thread(target=countdown,args=(COUNT/2,))
start = time.time()
t1.start();t2.start()
t1.join();t2.join()
end = time.time()
print end-start

Multiprocessing.Pool kullanılarak yeniden yazıldı: 
import multiprocessing
import time
def countdown(n):
    while n > 0:
        n -= 1

COUNT = 10000000

start = time.time()
with multiprocessing.Pool as pool:
    pool.map(countdown, [COUNT/2, COUNT/2])

    pool.close()
    pool.join()

end = time.time()
print(end-start)

Bu, iş parçacıkları oluşturmak yerine yeni süreçler yaratır. Her işlem kendi yorumlayıcısı olduğundan, GIL çakışması yoktur. multiprocessing.Pool, makinedeki çekirdek sayısı kadar işlem açacaktır, ancak yukarıdaki örnekte yalnızca iki tane gerekir. Gerçek dünya senaryosunda, listenizi en az makinenizdeki işlemciler kadar uzun olacak şekilde tasarlamak istersiniz. Havuz, her bağımsız değişkenle birlikte çalıştırmasını söylediğiniz işlevi, oluşturduğu işlem sayısına kadar çalıştırır. İşlev bittiğinde, listede kalan tüm işlevler bu işlemde çalıştırılacaktır.

with deyimini kullanarak bile, havuzu kapatıp katılmazsanız, süreçlerin var olmaya devam ettiğini buldum. Kaynakları temizlemek için her zaman havuzlarımı kapatıp katılırım.

Cython nogil: #

Cython, alternatif bir python yorumlayıcısıdır. GIL’i kullanır, ancak devre dışı bırakmanıza izin verir. Belgelerine bakın

Örnek olarak, [David Beazley’nin GIL’e karşı iş parçacıklarının tehlikelerini göstermek için ilk kez kullandığı kodu 2 kullanarak, onu nogil kullanarak yeniden yazacağız:

GIL iş parçacığı oluşturma sorunlarını gösteren David Beazley kodu

from threading import Thread
import time
def countdown(n):
    while n > 0:
        n -= 1

COUNT = 10000000

t1 = Thread(target=countdown,args=(COUNT/2,))
t2 = Thread(target=countdown,args=(COUNT/2,))
start = time.time()
t1.start();t2.start()
t1.join();t2.join()
end = time.time()
print end-start

nogil kullanılarak yeniden yazıldı (YALNIZCA CYTHON'DA ÇALIŞIR):

from threading import Thread
import time
def countdown(n):
    while n > 0:
        n -= 1

COUNT = 10000000

with nogil:
    t1 = Thread(target=countdown,args=(COUNT/2,))
    t2 = Thread(target=countdown,args=(COUNT/2,))
    start = time.time()
    t1.start();t2.start()
    t1.join();t2.join()
    
end = time.time()
print end-start

Cython kullandığınız sürece bu kadar basit. Belgelerin, herhangi bir python nesnesini değiştirmediğinizden emin olmanız gerektiğini söylediğini unutmayın:

İfadenin gövdesindeki kod Python nesnelerini değiştirmemelidir hiçbir şekilde ve Python nesnelerini manipüle eden hiçbir şeyi çağırmamalıdır önce GIL’i yeniden edinmeden. Cython şu anda kontrol etmiyor bu.
