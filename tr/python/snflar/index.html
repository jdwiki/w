Python kendisini yalnızca popüler bir betik dili olarak sunmakla kalmaz, aynı zamanda nesne yönelimli programlama paradigmasını da destekler. Sınıflar, verileri tanımlar ve bu verileri işlemek için yöntemler sağlar, tümü tek bir nesne altında toplanır. Ayrıca sınıflar, somut uygulama ayrıntılarını verilerin soyut temsillerinden ayırarak soyutlamaya izin verir.

Sınıfları kullanan kodun okunması, anlaşılması ve bakımı genellikle daha kolaydır.

Sınıflara giriş #

Bir sınıf, belirli bir nesnenin temel özelliklerini tanımlayan bir şablon işlevi görür. İşte bir örnek:

class Person(object):
     """A simple class."""                            # docstring
     species = "Homo Sapiens"                         # class attribute

     def __init__(self, name):                        # special method
         """This is the initializer. It's a special
         method (see below).
         """
         self.name = name                             # instance attribute

     def __str__(self):                               # special method
         """This method is run when Python tries 
         to cast the object to a string. Return 
         this string when using print(), etc.
         """
         return self.name

     def rename(self, renamed):                       # regular method
         """Reassign and print the name attribute."""
         self.name = renamed
         print("Now my name is {}".format(self.name))

Yukarıdaki örneğe bakarken dikkat edilmesi gereken birkaç şey var.

  1. Sınıf, nitelikler (veriler) ve yöntemler (işlevlerden) oluşur.
  2. Nitelikler ve yöntemler, basitçe normal değişkenler ve işlevler olarak tanımlanır.
  3. İlgili belge dizisinde belirtildiği gibi, __init__() yöntemine initializer adı verilir. Diğer nesne yönelimli dillerdeki yapıcıya eşdeğerdir ve yeni bir nesne veya sınıfın yeni bir örneğini oluşturduğunuzda ilk çalıştırılan yöntemdir.
  4. Tüm sınıf için geçerli olan nitelikler önce tanımlanır ve bunlara sınıf nitelikleri adı verilir.
  5. Bir sınıfın (bir nesnenin) belirli bir örneğine uygulanan öznitelikler, örnek öznitelikleri olarak adlandırılır. Genellikle __init__() içinde tanımlanırlar; bu gerekli değildir, ancak önerilir (çünkü __init__() dışında tanımlanan nitelikler, tanımlanmadan önce erişilme riskini taşır).
  6. Sınıf tanımına dahil edilen her yöntem, söz konusu nesneyi ilk parametresi olarak iletir. Bu parametre için “self” kelimesi kullanılır (“self” kelimesinin Python’da doğal bir anlamı olmadığı için “self” kullanımı aslında gelenekseldir, ancak bu Python’un en saygın kurallarından biridir ve her zaman izlemelisiniz. ).
  7. Diğer dillerde nesne yönelimli programlamaya alışmış olanlar birkaç şeye şaşırabilirler. Birincisi, Python’un gerçek bir “özel” öğe kavramına sahip olmamasıdır, bu nedenle her şey varsayılan olarak C++/Java “public” anahtar sözcüğünün davranışını taklit eder. Daha fazla bilgi için bu sayfadaki “Özel Sınıf Üyeleri” örneğine bakın.
  8. Sınıfın metotlarından bazıları şu biçimdedir: __functionname__(self, other_stuff). Tüm bu yöntemlere “sihirli yöntemler” denir ve Python’daki sınıfların önemli bir parçasıdır. Örneğin Python’da operatör aşırı yüklemesi sihirli yöntemlerle uygulanmaktadır. Daha fazla bilgi için ilgili belgelere bakın.

Şimdi ‘Person’ sınıfımızın birkaç örneğini yapalım!

>>> # Instances
>>> kelly = Person("Kelly")
>>> joseph = Person("Joseph")
>>> john_doe = Person("John Doe")

Şu anda üç “Person” nesnemiz var, “kelly”, “joseph” ve “john_doe”.

Nokta operatörünü . kullanarak her bir örnekten sınıfın niteliklerine erişebiliriz. Sınıf ve örnek nitelikleri arasındaki farkı tekrar not edin:

>>> # Attributes
>>> kelly.species
'Homo Sapiens'
>>> john_doe.species
'Homo Sapiens'
>>> joseph.species
'Homo Sapiens'
>>> kelly.name
'Kelly'
>>> joseph.name
'Joseph'

Aynı nokta operatörünü . kullanarak sınıfın yöntemlerini çalıştırabiliriz:

>>> # Methods
>>> john_doe.__str__()
'John Doe'
>>>  print(john_doe)
'John Doe'
>>>  john_doe.rename("John")
'Now my name is John'

Bağlı, ilişkisiz ve statik yöntemler #

Bağlı ve ilişkisiz yöntemler fikri Python 3’te kaldırıldı. Python 3’te bir sınıf içinde bir metot tanımladığınızda, bir ‘def’ anahtar sözcüğünü kullanırsınız, böylece bir fonksiyon nesnesi yaratırsınız. Bu normal bir işlevdir ve çevresindeki sınıf, onun ad alanı olarak çalışır. Aşağıdaki örnekte, “A” sınıfı içinde “f” yöntemini ilan ediyoruz ve bu, bir “A.f” işlevi haline geliyor:

class A(object):
    def f(self, x):
        return 2 * x
A.f
# <function A.f at ...>  (in Python 3.x)
son sürüm

Python 2’de davranış farklıydı: sınıf içindeki işlev nesneleri, herhangi bir belirli sınıf örneğine bağlı olmadıkları için bağsız yöntemler olarak adlandırılan “instancemethod” türündeki nesnelerle örtük olarak değiştirildi. Temel işleve .__func__ özelliğini kullanarak erişmek mümkündü.

A.f
# <unbound method A.f>   (in Python 2.x)
A.f.__class__
# <type 'instancemethod'>
A.f.__func__
# <function f at ...>
son sürüm

Sonraki davranışlar inceleme ile onaylanır - yöntemler Python 3’te işlevler olarak tanınırken, Python 2’de ayrım onaylanır.

import inspect

inspect.isfunction(A.f)
# True
inspect.ismethod(A.f)
# False
son sürüm 
import inspect

inspect.isfunction(A.f)
# False
inspect.ismethod(A.f)
# True
son sürüm

Python işlev/yönteminin her iki sürümünde de, ilk argüman olarak bir “A” sınıfı örneğini iletmeniz koşuluyla, “A.f” doğrudan çağrılabilir.

A.f(1, 7)
# Python 2: TypeError: unbound method f() must be called with
#                      A instance as first argument (got int instance instead) 
# Python 3: 14   
a = A()
A.f(a, 20)
# Python 2 & 3: 40

Şimdi “a"nın “A” sınıfının bir örneği olduğunu varsayalım, o zaman “a.f” nedir? Eh, sezgisel olarak bu, “A” sınıfının “f” yöntemiyle aynı olmalıdır, yalnızca bir şekilde “a” nesnesine uygulandığını “bilmelidir” - Python’da buna “a” yöntemine bağlı denir.

Özgün ayrıntılar aşağıdaki gibidir: “a.f” yazmak, “a"nın sihirli “getattribute” yöntemini çağırır; bu, önce “a"nın “f” adında bir özniteliğe sahip olup olmadığını kontrol eder (yoktur), ardından sınıfı kontrol eder. ‘A’, böyle bir ada sahip bir yöntem içerip içermediğini (vardır) ve ’m.func’ içindeki orijinal ‘A.f’ referansına ve bir referansa sahip olan ‘yöntem’ türünde yeni bir ’m’ nesnesi oluşturur. ’m.self’ içindeki ‘a’ nesnesine. Bu nesne bir fonksiyon olarak çağrıldığında, basitçe şunları yapar: m(...) => m.__func__(m.__self__, ...). Bu nedenle bu nesneye bağlı yöntem denir, çünkü çağrıldığında ilk argüman olarak bağlı olduğu nesneyi sağlamayı bilir. (Bu şeyler Python 2 ve 3’te aynı şekilde çalışır).

a = A()
a.f
# <bound method A.f of <__main__.A object at ...>>
a.f(2)
# 4

# Note: the bound method object a.f is recreated *every time* you call it:
a.f is a.f  # False
# As a performance optimization you can store the bound method in the object's
# __dict__, in which case the method object will remain fixed:
a.f = a.f
a.f is a.f  # True

Son olarak, Python’da sınıf yöntemleri ve statik yöntemler vardır - özel türde yöntemler. Sınıf yöntemleri, bir nesne üzerinde çağrıldıklarında nesne yerine nesnenin sınıfına bağlanmaları dışında normal yöntemlerle aynı şekilde çalışır. Böylece m.__self__ = type(a). Böyle bir bağlı yöntemi çağırdığınızda, ilk argüman olarak ‘a’ sınıfını iletir. Statik yöntemler daha da basittir: hiçbir şeyi bağlamazlar ve herhangi bir dönüşüm olmadan temel işlevi geri döndürürler.

class D(object):
    multiplier = 2

    @classmethod
    def f(cls, x):
        return cls.multiplier * x

    @staticmethod
    def g(name):
        print("Hello, %s" % name)

D.f
# <bound method type.f of <class '__main__.D'>>
D.f(12)
# 24
D.g
# <function D.g at ...>
D.g("world")
# Hello, world

Örnekte erişildiğinde bile sınıf yöntemlerinin sınıfa bağlı olduğunu unutmayın:

d = D()
d.multiplier = 1337
(D.multiplier, d.multiplier)
# (2, 1337)
d.f
# <bound method D.f of <class '__main__.D'>>
d.f(10)
# 20

En düşük düzeyde, işlevlerin, yöntemlerin, statik yöntemlerin vb.’nin aslında [tanımlayıcılar] olduğunu belirtmekte fayda var (https://www.wikiod.com/tr/python/tanmlayc#Basit tanımlayıcı) __get__, __set__ ve isteğe bağlı olarak __del__ özel yöntemlerini çağıran. Temel kalıtım #

Python’da kalıtım, Java, C++ vb. gibi diğer nesne yönelimli dillerde kullanılan benzer fikirlere dayanır. Mevcut bir sınıftan yeni bir sınıf aşağıdaki gibi türetilebilir.

class BaseClass(object):
    pass

class DerivedClass(BaseClass):
    pass

“TemelSınıf” zaten var olan (üst) sınıftır ve “DerivedClass”, “TemelSınıf"tan öznitelikleri devralan (veya alt sınıflar) yeni (alt) sınıftır. Not: Python 2.2’den itibaren, tüm sınıflar örtük olarak “nesne” sınıfından miras alırlar, hangi tüm yerleşik türler için temel sınıftır.

Aşağıdaki örnekte, “nesne"den dolaylı olarak miras alan bir ana “Rectangle” sınıfı tanımlıyoruz:

class Rectangle():
    def __init__(self, w, h):
        self.w = w
        self.h = h
    
    def area(self):
        return self.w * self.h
        
    def perimeter(self):
        return 2 * (self.w + self.h)
son sürüm

Kare, dikdörtgenin özel bir durumu olduğundan, “Rectangle” sınıfı bir “Square” sınıfını tanımlamak için temel sınıf olarak kullanılabilir.

class Square(Rectangle):
    def __init__(self, s):
        # call parent constructor, w and h are both s
        super(Square, self).__init__(s, s)
        self.s = s

“Square” sınıfı, nesne sınıfının yanı sıra “Rectangle” sınıfının tüm niteliklerini otomatik olarak devralır. “super()”, “Rectangle” sınıfının “init()” yöntemini çağırmak için kullanılır ve esasen temel sınıfın geçersiz kılınan herhangi bir yöntemini çağırır. Not: Python 3’te “super()” argüman gerektirmez.

Türetilmiş sınıf nesneleri, temel sınıflarının niteliklerine erişebilir ve bunları değiştirebilir:

r.area()
# Output: 12
r.perimeter()  
# Output: 14

s.area()
# Output: 4
s.perimeter()
# Output: 8

Kalıtımla çalışan yerleşik işlevler #

“issubclass(DerivedClass, BaseClass)”: “DerivedClass”, “BaseClass"ın bir alt sınıfıysa “True” değerini döndürür

‘isinstance(s, Class)’: s ‘Sınıf’ın bir örneğiyse veya ‘Sınıf’ın türetilmiş sınıflarından herhangi biriyse ‘True’ değerini döndürür

# subclass check        
issubclass(Square, Rectangle)
# Output: True

# instantiate
r = Rectangle(3, 4)
s = Square(2)

isinstance(r, Rectangle)  
# Output: True
isinstance(r, Square)
# Output: False
# A rectangle is not a square

isinstance(s, Rectangle)
# Output: True
# A square is a rectangle
isinstance(s, Square)
# Output: True

Maymun Yama #

Bu durumda, “maymun yaması”, bir sınıfa tanımlandıktan sonra yeni bir değişken veya yöntem eklemek anlamına gelir. Örneğin, “A” sınıfını şöyle tanımladığımızı varsayalım.

class A(object):
    def __init__(self, num):
        self.num = num

    def __add__(self, other):
        return A(self.num + other.num)

Ama şimdi koda daha sonra başka bir fonksiyon eklemek istiyoruz. Bu fonksiyonun aşağıdaki gibi olduğunu varsayalım.

def get_num(self):
    return self.num

Ama bunu “A"ya bir metod olarak nasıl ekleyeceğiz? Bu kadar basit, bu işlevi bir atama ifadesiyle ‘A’ya yerleştiriyoruz.

A.get_num = get_num

Bu neden işe yarıyor? Çünkü işlevler, diğer nesneler gibi nesnelerdir ve yöntemler, sınıfa ait işlevlerdir.

‘get_num’ işlevi, tüm mevcut (zaten oluşturulmuş) ve yeni ‘A’ örnekleri için kullanılabilir olacaktır.

Bu eklemeler, o sınıfın (veya alt sınıflarının) tüm örneklerinde otomatik olarak kullanılabilir. Örneğin:

foo = A(42)

A.get_num = get_num

bar = A(6);

foo.get_num() # 42

bar.get_num() # 6

Diğer bazı dillerden farklı olarak, bu tekniğin belirli yerleşik türler için çalışmadığını ve iyi bir stil olarak kabul edilmediğini unutmayın.

Yeni tarz ve eski tarz sınıflar #

classes ve types‘ı birleştirmek için Python 2.2’de Yeni stil sınıfları tanıtıldı. Üst düzey “nesne” türünden miras alırlar. Yeni stil sınıfı, kullanıcı tanımlı bir türdür ve yerleşik türlere çok benzer.

# new-style class
class New(object):
    pass

# new-style instance
new = New()

new.__class__
# <class '__main__.New'>
type(new)
# <class '__main__.New'>
issubclass(New, object)
# True

Eski tarz sınıflar ’nesne’den not miras alır. Eski tarz örnekler her zaman yerleşik bir “örnek” türüyle uygulanır.

# old-style class
class Old:
    pass

# old-style instance
old = Old()

old.__class__
# <class __main__.Old at ...>
type(old)
# <type 'instance'>
issubclass(Old, object)
# False
son sürüm

Python 3’te eski tarz sınıflar kaldırıldı.

Python 3’teki yeni tarz sınıflar örtük olarak “nesne"den miras alırlar, bu nedenle artık “Sınıfım(nesne)“yi belirtmeye gerek yoktur.

class MyClass:
    pass

my_inst = MyClass()

type(my_inst)
# <class '__main__.MyClass'>
my_inst.__class__
# <class '__main__.MyClass'>
issubclass(MyClass, object)
# True
son sürüm

Sınıf yöntemleri: alternatif başlatıcılar #

Sınıf yöntemleri, sınıf örnekleri oluşturmak için alternatif yollar sunar. Açıklamak için, bir örneğe bakalım.

Diyelim ki nispeten basit bir ‘Person’ sınıfımız var:

class Person(object):

    def __init__(self, first_name, last_name, age):
        self.first_name = first_name
        self.last_name = last_name
        self.age = age
        self.full_name = first_name + " " + last_name
    
    def greet(self):
        print("Hello, my name is " + self.full_name + ".")

Ad ve soyad ayrı ayrı yerine tam ad belirterek bu sınıfın örneklerini oluşturmanın bir yolunun olması kullanışlı olabilir. Bunu yapmanın bir yolu, “last_name"nin isteğe bağlı bir parametre olması ve eğer verilmemişse tam adı şu şekilde ilettiğimizi varsayarsak:

class Person(object):

    def __init__(self, first_name, age, last_name=None):
        if last_name is None:
            self.first_name, self.last_name = first_name.split(" ", 2)
        else:
            self.first_name = first_name
            self.last_name = last_name
        
        self.full_name = self.first_name + " " + self.last_name
        self.age = age

    def greet(self):
        print("Hello, my name is " + self.full_name + ".")

Ancak, bu kod parçasıyla ilgili iki ana sorun vardır:

  1. ‘first_name’ için tam bir ad girebildiğiniz için ‘ad_name’ ve ’last_name’ parametreleri artık yanıltıcıdır. Ayrıca, bu tür esnekliğe sahip daha fazla durum ve/veya daha fazla parametre varsa, if/elif/else dallanması can sıkıcı hale gelebilir.

  2. O kadar önemli değil ama yine de belirtmekte fayda var: Peki ya ‘soy_ad’ ‘Yok’ ise ve ‘ad_ad’ boşluklar yoluyla iki veya daha fazla şeye bölünmüyorsa? Başka bir girdi doğrulama ve/veya istisna işleme katmanımız daha var…

Sınıf yöntemlerini girin. Tek bir başlatıcıya sahip olmak yerine, “from_full_name” adında ayrı bir başlatıcı oluşturacağız ve onu (yerleşik) “sınıf yöntemi” dekoratörüyle süsleyeceğiz.

class Person(object):

    def __init__(self, first_name, last_name, age):
        self.first_name = first_name
        self.last_name = last_name
        self.age = age
        self.full_name = first_name + " " + last_name
    
    @classmethod
    def from_full_name(cls, name, age):
        if " " not in name:
            raise ValueError
        first_name, last_name = name.split(" ", 2)
        return cls(first_name, last_name, age)
    
    def greet(self):
        print("Hello, my name is " + self.full_name + ".")

“from_full_name"nin ilk argümanı olarak “self” yerine “cls"ye dikkat edin. Sınıf yöntemleri genel sınıfa uygulanır, belirli bir sınıfın bir örneği *değildir (ki bu genellikle “self"in ifade ettiği şeydir). Dolayısıyla, “cls” bizim “Person” sınıfımızsa, “from_full_name” sınıf yönteminden döndürülen değer, bir örnek oluşturmak için “Person"ın “init“sini kullanan “Person(ad_name, last_name, age)” olur. ‘Kişi’ sınıfının. Özellikle, ‘Person’ın ‘Employee’ alt sınıfını yapacak olsaydık, ‘from_full_name’, ‘Employee’ sınıfında da çalışırdı.

Bunun beklendiği gibi çalıştığını göstermek için, ‘init’ içinde dallanma olmadan birden fazla şekilde ‘Person’ örneklerini oluşturalım:

In [2]: bob = Person("Bob", "Bobberson", 42)

In [3]: alice = Person.from_full_name("Alice Henderson", 31)

In [4]: bob.greet()
Hello, my name is Bob Bobberson.

In [5]: alice.greet()
Hello, my name is Alice Henderson.

Çoklu Kalıtım #

Python, yöntemler de dahil olmak üzere sınıf niteliklerinin çözümlenme sırasını belirlemek için C3 doğrusallaştırma algoritmasını kullanır. Bu, Yöntem Çözüm Sırası (MRO) olarak bilinir.

İşte basit bir örnek:

class Foo(object):
    foo = 'attr foo of Foo'
    

class Bar(object):
    foo = 'attr foo of Bar' # we won't see this.
    bar = 'attr bar of Bar'

class FooBar(Foo, Bar):
    foobar = 'attr foobar of FooBar'

Şimdi FooBar’ı başlatırsak, foo özniteliğine bakarsak, önce Foo’nun özniteliğinin bulunduğunu görürüz.

fb = FooBar()

ve

>>> fb.foo
'attr foo of Foo'

İşte FooBar’ın MRO’su:

>>> FooBar.mro()
[<class '__main__.FooBar'>, <class '__main__.Foo'>, <class '__main__.Bar'>, <type 'object'>]

Basitçe, Python’un MRO algoritmasının şu şekilde olduğu söylenebilir:

  1. Önce derinlik (ör. “FooBar” sonra “Foo”)
  2. paylaşılan bir ebeveyn (“nesne”) bir çocuk (“Bar”) tarafından engellenir ve
  3. Döngüsel ilişkilere izin verilmez.

Diğer bir deyişle, örneğin, Bar, FooBar’dan devralırken FooBar, Bar’dan devralır.

Python’da kapsamlı bir örnek için wikipedia girdisine bakın.

Kalıtımdaki bir başka güçlü özellik de “süper"dir. super, ebeveyn sınıflarının özelliklerini getirebilir.

class Foo(object):
    def foo_method(self):
        print "foo Method"

class Bar(object):
    def bar_method(self):
        print "bar Method"

class FooBar(Foo, Bar):
    def foo_method(self):
        super(FooBar, self).foo_method()

Sınıfın init yöntemiyle çoklu kalıtım, her sınıfın kendi init yöntemi olduğunda, o zaman birden çok ineritance deneriz, sonra sadece init yöntemi, önce kalıtsal olan sınıftan çağrılır.

aşağıdaki örnek için yalnızca Foo sınıfı init yöntemi çağrılıyor Bar sınıfı init çağrılmıyor

    class Foo(object):
        def __init__(self):
            print "foo init"

    class Bar(object):
        def __init__(self):
            print "bar init"

    class FooBar(Foo, Bar):
        def __init__(self):
            print "foobar init"
            super(FooBar, self).__init__()

    a = FooBar()

Özellikleri #

Python sınıfları, normal nesne değişkenleri gibi görünen ancak özel davranış ve belgeler ekleme olasılığı olan özellikleri destekler.

class MyClass(object):

    def __init__(self):
       self._my_string = ""
    
    @property
    def string(self):
        """A profoundly important string."""
        return self._my_string

    @string.setter
    def string(self, new_value):
        assert isinstance(new_value, str), \
               "Give me a string, not a %r!" % type(new_value)
        self._my_string = new_value

    @string.deleter
    def x(self):
        self._my_string = None

‘Sınıfım’ sınıfının nesnesi, ‘.string’ özelliğine sahip olarak görünecektir, ancak davranışı artık sıkı bir şekilde kontrol edilmektedir:

mc = MyClass()

Yukarıdaki gibi faydalı sözdiziminin yanı sıra, özellik sözdizimi, doğrulamaya veya bu niteliklere eklenecek diğer artırmalara izin verir. Bu, özellikle kullanıcıya bir düzeyde yardım verilmesi gereken genel API’ler için yararlı olabilir.

Özelliklerin bir başka yaygın kullanımı, sınıfın ‘sanal öznitelikleri’ - gerçekte depolanmayan ancak yalnızca istendiğinde hesaplanan öznitelikler - sunmasını sağlamaktır.

class Character(object):
    def __init__(name, max_hp):
        self._name = name
        self._hp = max_hp
        self._max_hp = max_hp

    # Make hp read only by not providing a set method
    @property
    def hp(self):
        return self._hp

    # Make name read only by not providing a set method
    @property
    def name(self):
        return self.name

    def take_damage(self, damage):
        self.hp -= damage
        self.hp = 0 if self.hp <0 else self.hp

    @property
    def is_alive(self):
        return self.hp != 0

    @property
    def is_wounded(self):
        return self.hp < self.max_hp if self.hp > 0 else False

    @property
    def is_dead(self):
        return not self.is_alive

Singleton sınıfı #

Singleton, bir sınıfın somutlaştırılmasını bir örnek/nesne ile sınırlayan bir kalıptır. Python singleton tasarım desenleri hakkında daha fazla bilgi için buraya bakın.

class Singleton:
    def __new__(cls):
        try:
            it = cls.__it__
        except AttributeError:
            it = cls.__it__ = object.__new__(cls)
        return it

    def __repr__(self):
        return '<{}>'.format(self.__class__.__name__.upper())

         return other is self

Başka bir yöntem de sınıfınızı dekore etmektir. Bu cevap örneğini izleyerek bir Singleton sınıfı oluşturun:

class Singleton:
    """
    A non-thread-safe helper class to ease implementing singletons.
    This should be used as a decorator -- not a metaclass -- to the
    class that should be a singleton.

    The decorated class can define one `__init__` function that
    takes only the `self` argument. Other than that, there are
    no restrictions that apply to the decorated class.

    To get the singleton instance, use the `Instance` method. Trying
    to use `__call__` will result in a `TypeError` being raised.

    Limitations: The decorated class cannot be inherited from.

    """

    def __init__(self, decorated):
        self._decorated = decorated

    def Instance(self):
        """
        Returns the singleton instance. Upon its first call, it creates a
        new instance of the decorated class and calls its `__init__` method.
        On all subsequent calls, the already created instance is returned.

        """
        try:
            return self._instance
        except AttributeError:
            self._instance = self._decorated()
            return self._instance

    def __call__(self):
        raise TypeError('Singletons must be accessed through `Instance()`.')

    def __instancecheck__(self, inst):
        return isinstance(inst, self._decorated)

Tanımlayıcılar ve Noktalı Aramalar #

Tanımlayıcılar (genellikle) sınıfların öznitelikleri olan ve ‘get’, ‘set’ veya ‘delete’ özel yöntemlerinden herhangi birine sahip olan nesnelerdir.

Veri Tanımlayıcılar ‘set’ veya ‘delete‘den herhangi birine sahiptir

Bunlar, bir örnekteki noktalı aramayı kontrol edebilir ve “staticmethod”, “classmethod” ve “property” işlevlerini uygulamak için kullanılır. Noktalı bir arama (örneğin, “bar” özniteliğini arayan “Foo” sınıfının “foo” örneği - yani “foo.bar” aşağıdaki algoritmayı kullanır:

  1. ‘bar’, ‘Foo’ sınıfında aranır. Varsa ve bir Veri Tanımlayıcı ise, veri tanımlayıcı kullanılır. “Özellik” bu şekilde bir örnekteki verilere erişimi kontrol edebilir ve örnekler bunu geçersiz kılamaz. Bir Veri Tanımlayıcı orada değilse, o zaman

  2. ‘bar’, ‘dict’ örneğinde aranır. Bu nedenle, noktalı bir arama ile bir örnekten çağrılan yöntemleri geçersiz kılabilir veya engelleyebiliriz. Örnekte “bar” varsa, kullanılır. Değilse, biz o zaman

  3. ‘bar’ için ‘Foo’ sınıfına bakın. Tanımlayıcı ise, tanımlayıcı protokolü kullanılır. İşlevler (bu bağlamda, ilişkisiz yöntemler), “sınıf yöntemi” ve “statik yöntem” bu şekilde uygulanır. Aksi takdirde, oradaki nesneyi döndürür veya bir ‘AttributeError’ vardır.
