Yuvarlama: yuvarlak, zemin, tavan, gövde #

Yerleşik “yuvarlak” işlevine ek olarak, “matematik” modülü “zemin”, “tavan” ve “trunc” işlevlerini sağlar.

x = 1.55
y = -1.55

# round to the nearest integer
round(x)       #  2
round(y)       # -2

# the second argument gives how many decimal places to round to (defaults to 0)
round(x, 1)    #  1.6
round(y, 1)    # -1.6

# math is a module so import it first, then use it.
import math

# get the largest integer less than x
math.floor(x)  #  1
math.floor(y)  # -2

# get the smallest integer greater than x
math.ceil(x)   #  2
math.ceil(y)   # -1

# drop fractional part of x
math.trunc(x)  #  1, equivalent to math.floor for positive numbers
math.trunc(y)  # -1, equivalent to math.ceil for negative numbers

“zemin”, “ceil”, “trunc” ve “yuvarlak” her zaman bir “float” döndürür.

round(1.3)  # 1.0

‘yuvarlak’ her zaman bağları sıfırdan koparır.

round(0.5)  # 1.0
round(1.5)  # 2.0
son sürüm

“zemin”, “ceil” ve “trunc” her zaman bir “İntegral” değeri döndürürken “yuvarlak” bir bağımsız değişkenle çağrıldığında bir “İntegral” değeri döndürür.

round(1.3)      # 1
round(1.33, 1)  # 1.3

‘yuvarlak’ en yakın çift sayıya olan bağları koparır. Bu, çok sayıda hesaplama gerçekleştirirken daha büyük sayılara yönelik önyargıyı düzeltir.

round(0.5)  # 0
round(1.5)  # 2
son sürüm

Uyarı! #

Herhangi bir kayan nokta gösteriminde olduğu gibi, bazı kesirler tam olarak temsil edilemez. Bu, bazı beklenmedik yuvarlama davranışlarına yol açabilir.

round(2.675, 2)  # 2.67, not 2.68!

Negatif sayıların taban, kesme ve tamsayı bölümü hakkında uyarı #

Python (ve C++ ve Java) negatif sayılar için sıfırdan uzaklaşır. Düşünmek:

>>> math.floor(-1.7)
-2.0
>>> -5 // 2
-3

trigonometri #

Hipotenüsün uzunluğunu hesaplama #

math.hypot(2, 4) # Just a shorthand for SquareRoot(2**2 + 4**2)
# Out: 4.47213595499958

Dereceleri radyana/radyana dönüştürme #

Tüm “matematik” işlevleri radyan bekler, bu nedenle dereceleri radyana dönüştürmeniz gerekir:

math.radians(45)              # Convert 45 degrees to radians
# Out: 0.7853981633974483

Ters trigonometrik işlevlerin tüm sonuçları, sonucu radyan cinsinden döndürür, bu nedenle onu tekrar dereceye dönüştürmeniz gerekebilir:

math.degrees(math.asin(1))    # Convert the result of asin to degrees
# Out: 90.0

Sinüs, kosinüs, tanjant ve ters fonksiyonlar #

# Sine and arc sine
math.sin(math.pi / 2)
# Out: 1.0
math.sin(math.radians(90))   # Sine of 90 degrees
# Out: 1.0

math.asin(1)
# Out: 1.5707963267948966    # "= pi / 2"
math.asin(1) / math.pi
# Out: 0.5

# Cosine and arc cosine:
math.cos(math.pi / 2)
# Out: 6.123233995736766e-17 
# Almost zero but not exactly because "pi" is a float with limited precision!

math.acos(1)
# Out: 0.0

# Tangent and arc tangent:
math.tan(math.pi/2)
# Out: 1.633123935319537e+16 
# Very large but not exactly "Inf" because "pi" is a float with limited precision

math.atan(math.inf)
# Out: 1.5707963267948966 # This is just "pi / 2"
son sürüm 
math.atan(float('inf'))
# Out: 1.5707963267948966 # This is just "pi / 2"

‘math.atan’ dışında, doğru kadranı hesaplayan ve sıfıra bölmenin tuzaklarından kaçınan iki bağımsız değişkenli bir ‘math.atan2’ işlevi de vardır:

math.atan2(1, 2)   # Equivalent to "math.atan(1/2)"
# Out: 0.4636476090008061 # ≈ 26.57 degrees, 1st quadrant

math.atan2(-1, -2) # Not equal to "math.atan(-1/-2)" == "math.atan(1/2)"
# Out: -2.677945044588987 # ≈ -153.43 degrees (or 206.57 degrees), 3rd quadrant

math.atan2(1, 0)   # math.atan(1/0) would raise ZeroDivisionError
# Out: 1.5707963267948966 # This is just "pi / 2"

Hiperbolik sinüs, kosinüs ve tanjant #

# Hyperbolic sine function
math.sinh(math.pi) # = 11.548739357257746
math.asinh(1)      # = 0.8813735870195429

# Hyperbolic cosine function
math.cosh(math.pi) # = 11.591953275521519
math.acosh(1)      # = 0.0

# Hyperbolic tangent function
math.tanh(math.pi) # = 0.99627207622075
math.atanh(0.5)    # = 0.5493061443340549

Logaritmalar #

“math.log(x)”, “x"in doğal (temel “e”) logaritmasını verir.

math.log(math.e)  # 1.0
math.log(1)       # 0.0
math.log(100)     # 4.605170185988092

“math.log”, kayan noktalı sayıların sınırlamaları nedeniyle 1’e yakın sayılarla kesinliği kaybedebilir. 1’e yakın günlükleri doğru bir şekilde hesaplamak için, 1’in doğal logaritmasını artı argümanı değerlendiren “math.log1p"yi kullanın:

math.log(1 + 1e-20)  # 0.0
math.log1p(1e-20)    # 1e-20

“math.log10”, 10 tabanlı günlükler için kullanılabilir:

math.log10(10)  # 1.0

İki bağımsız değişkenle birlikte kullanıldığında, “math.log(x, base)”, verilen “taban"daki “x"in logaritmasını verir (yani, “log(x) / log(taban)”).

math.log(100, 10) # 2.0
math.log(27, 3)   # 3.0
math.log(1, 10)   # 0.0
son sürüm

Sabitler #

“matematik” modülleri, yaygın olarak kullanılan iki matematiksel sabiti içerir.

  • math.pi - Matematik sabiti pi

  • math.e - Matematik sabiti e (doğal logaritmanın tabanı)

    from math import pi, e pi 3.141592653589793 e 2.718281828459045

Python 3.5 ve üstü, sonsuzluk ve NaN (“sayı değil”) için sabitlere sahiptir. Bir dizgeyi “float()“a geçirmenin eski sözdizimi hala çalışıyor.

math.inf == float('inf')
# Out: True

-math.inf == float('-inf')
# Out: True

# NaN never compares equal to anything, even itself
math.nan == float('nan')
# Out: False
son sürüm

Infinity ve NaN (“sayı değil”) #

Python’un tüm sürümlerinde, sonsuzluğu ve NaN’yi (“sayı değil”) şu şekilde temsil edebiliriz:

pos_inf = float('inf')     # positive infinity
neg_inf = float('-inf')    # negative infinity
not_a_num = float('nan')   # NaN ("not a number")

Python 3.5 ve üzeri sürümlerde, “math.inf” ve “math.nan” olarak tanımlanan sabitleri de kullanabiliriz:

pos_inf = math.inf
neg_inf = -math.inf
not_a_num = math.nan
son sürüm

Dize gösterimleri “inf” ve “-inf” ve “nan” olarak görüntülenir:

pos_inf, neg_inf, not_a_num
# Out: (inf, -inf, nan)

“isinf” yöntemiyle pozitif veya negatif sonsuz olup olmadığını test edebiliriz:

math.isinf(pos_inf)
# Out: True

math.isinf(neg_inf)
# Out: True

Doğrudan karşılaştırma ile özellikle pozitif sonsuz veya negatif sonsuz için test edebiliriz:

pos_inf == float('inf')    # or  == math.inf in Python 3.5+
# Out: True

neg_inf == float('-inf')   # or  == -math.inf in Python 3.5+
# Out: True

neg_inf == pos_inf
# Out: False

Python 3.2 ve üstü ayrıca sonluluğun kontrol edilmesini sağlar:

math.isfinite(pos_inf)
# Out: False

math.isfinite(0.0)
# Out: True
son sürüm

Karşılaştırma operatörleri pozitif ve negatif sonsuz için beklendiği gibi çalışır:

import sys

sys.float_info.max
# Out: 1.7976931348623157e+308  (this is system-dependent)

pos_inf > sys.float_info.max
# Out: True

neg_inf < -sys.float_info.max
# Out: True

Ancak bir aritmetik ifade, “float” olarak gösterilebilecek maksimum değerden daha büyük bir değer üretirse, sonsuz olur:

pos_inf == sys.float_info.max * 1.0000001
# Out: True

neg_inf == -sys.float_info.max * 1.0000001
# Out: True

Ancak sıfıra bölme, sonsuz (veya uygun olduğunda negatif sonsuz) sonucunu vermez, bunun yerine bir ‘ZeroDivisionError’ istisnası oluşturur.

try:
    x = 1.0 / 0.0
    print(x)
except ZeroDivisionError:
    print("Division by zero")

# Out: Division by zero

Sonsuzluktaki aritmetik işlemler sadece sonsuz sonuçlar verir veya bazen NaN:

-5.0 * pos_inf == neg_inf
# Out: True

-5.0 * neg_inf == pos_inf
# Out: True

pos_inf * neg_inf == neg_inf
# Out: True

0.0 * pos_inf
# Out: nan

0.0 * neg_inf
# Out: nan

pos_inf / pos_inf
# Out: nan

NaN asla hiçbir şeye eşit değildir, kendisine bile. isnan yöntemiyle olup olmadığını test edebiliriz:

not_a_num == not_a_num
# Out: False

math.isnan(not_a_num)
Out: True

NaN her zaman “eşit değil” olarak karşılaştırır, ancak asla şundan küçük veya büyük değildir:

not_a_num != 5.0   # or any random value
# Out: True

not_a_num > 5.0   or   not_a_num < 5.0   or   not_a_num == 5.0
# Out: False

NaN üzerindeki aritmetik işlemler her zaman NaN verir. Bu, -1 ile çarpmayı içerir: “negatif NaN” yoktur.

5.0 * not_a_num
# Out: nan

float('-nan')
# Out: nan

-math.nan
# Out: nan
son sürüm

NaN ve infinity’nin eski “float” sürümleri ile Python 3.5+ “matematik” kitaplığı sabitleri arasında ince bir fark vardır:

math.inf is math.inf, math.nan is math.nan
# Out: (True, True)

float('inf') is float('inf'), float('nan') is float('nan')
# Out: (False, False)
son sürüm

Daha hızlı üs alma için Pow #

Komut satırından timeit modülünü kullanarak:

> python -m timeit 'for x in xrange(50000): b = x**3'
10 loops, best of 3: 51.2 msec per loop
> python -m timeit 'from math import pow' 'for x in xrange(50000): b = pow(x,3)' 
100 loops, best of 3: 9.15 msec per loop

Yerleşik ** operatörü genellikle işe yarar, ancak performans önemliyse, math.pow kullanın. Bununla birlikte, argümanlar tamsayı olsa bile, pow dönüşlerinin yüzer olduğunu unutmayın:

> from math import pow
> pow(5,5)
3125.0

İşaretleri kopyalama #

Python 2.6 ve sonraki sürümlerde, “math.copysign(x, y)”, “y” işaretiyle “x” değerini döndürür. Döndürülen değer her zaman bir “kayan nokta"dır.

math.copysign(-2, 3)    # 2.0
math.copysign(3, -3)    # -3.0
math.copysign(4, 14.2)  # 4.0
math.copysign(1, -0.0)  # -1.0, on a platform which supports signed zero
son sürüm

Hayali Sayılar #

Python’daki hayali sayılar, hedef sayının sonundaki bir “j” veya “J” ile temsil edilir.

1j         # Equivalent to the square root of -1.
1j * 1j    # = (-1+0j)

Karmaşık sayılar ve cmath modülü #

“cmath” modülü “matematik” modülüne benzer, ancak işlevleri karmaşık düzlem için uygun şekilde tanımlar.

Her şeyden önce, karmaşık sayılar bir kütüphane sınıfı tarafından sağlanmak yerine Python dilinin bir parçası olan sayısal bir türdür. Bu nedenle, sıradan aritmetik ifadeler için cmath almamız gerekmez.

“i” yerine “j” (veya “J”) kullandığımızı unutmayın.

z = 1 + 3j

‘j’ sayısal bir değişmezden ziyade bir değişkenin adı olacağından ‘1j’ kullanmalıyız.

1j * 1j
Out: (-1+0j)

1j ** 1j
# Out: (0.20787957635076193+0j)     # "i to the i"  ==  math.e ** -(math.pi/2)

Karmaşık “eşlenik"in yanı sıra “gerçek” kısım ve “imag” (hayali) kısıma sahibiz:

# real part and imaginary part are both float type
z.real, z.imag
# Out: (1.0, 3.0)

z.conjugate()
# Out: (1-3j)    # z.conjugate() == z.real - z.imag * 1j

“abs” ve “complex” yerleşik işlevleri de dilin bir parçasıdır ve herhangi bir içe aktarma gerektirmez:

abs(1 + 1j)
# Out: 1.4142135623730951     # square root of 2

complex(1)
# Out: (1+0j)

complex(imag=1)
# Out: (1j)

complex(1, 1)
# Out: (1+1j)

“Karmaşık” işlevi bir dize alabilir, ancak boşluk içeremez:

complex('1+1j')
# Out: (1+1j)

complex('1 + 1j')
# Exception: ValueError: complex() arg is a malformed string

Ancak çoğu işlev için modüle ihtiyacımız var, örneğin sqrt:

import cmath

cmath.sqrt(-1)
# Out: 1j

Doğal olarak ‘sqrt’ davranışı karmaşık sayılar ve gerçek sayılar için farklıdır. Karmaşık olmayan “matematik"te, negatif bir sayının karekökü bir istisna oluşturur:

import math

math.sqrt(-1)
# Exception: ValueError: math domain error

Kutupsal koordinatlara ve kutupsal koordinatlardan dönüştürmek için işlevler sağlanmıştır:

cmath.polar(1 + 1j)
# Out: (1.4142135623730951, 0.7853981633974483)    # == (sqrt(1 + 1), atan2(1, 1))

abs(1 + 1j), cmath.phase(1 + 1j)
# Out: (1.4142135623730951, 0.7853981633974483)    # same as previous calculation

cmath.rect(math.sqrt(2), math.atan(1))
# Out: (1.0000000000000002+1.0000000000000002j)

Karmaşık analizin matematiksel alanı bu örneğin kapsamı dışındadır, ancak karmaşık düzlemdeki birçok fonksiyonda, genellikle gerçek eksen veya hayali eksen boyunca bir “dal kesimi” vardır. Modern platformların çoğu, dal kesiminin her iki tarafında bu işlevlerin sürekliliğini sağlayan IEEE 754’te belirtildiği gibi “sıfır işaretli"yi destekler. Aşağıdaki örnek Python belgelerinden alınmıştır:

cmath.phase(complex(-1.0, 0.0))
# Out: 3.141592653589793

cmath.phase(complex(-1.0, -0.0))
# Out: -3.141592653589793

‘cmath’ modülü aynı zamanda ‘matematik’ modülünün doğrudan karşılığı olan birçok işlevi de sağlar.

‘sqrt’ye ek olarak, ’exp’, ’log’, ’log10’, trigonometrik fonksiyonlar ve bunların terslerinin (‘sin’, ‘cos’, ’tan’, ‘asin’, ‘acos’) karmaşık versiyonları vardır. , ‘atan’) ve hiperbolik fonksiyonlar ve bunların tersleri (‘sinh’, ‘cosh’, ’tanh’, ‘asinh’, ‘acosh’, ‘atanh’). Bununla birlikte, arktanjantın iki argümanlı formu olan ‘math.atan2’nin karmaşık bir karşılığı olmadığına dikkat edin.

cmath.log(1+1j)
# Out: (0.34657359027997264+0.7853981633974483j)

cmath.exp(1j * cmath.pi)
# Out: (-1+1.2246467991473532e-16j)   # e to the i pi == -1, within rounding error

‘pi’ ve ’e’ sabitleri sağlanır. Bunların “yüzer” olduğuna ve “karmaşık” olmadığına dikkat edin.

type(cmath.pi)
# Out: <class 'float'>

“cmath” modülü ayrıca “isinf” ve (Python 3.2+ için) “isfinite"nin karmaşık sürümlerini sağlar. “[Sonsuz ve NaN][1]” bölümüne bakın. Karmaşık bir sayı, gerçek kısmı veya sanal kısmı sonsuz ise sonsuz olarak kabul edilir.

cmath.isinf(complex(float('inf'), 0.0))
# Out: True

Benzer şekilde, “cmath” modülü, “isnan"ın karmaşık bir sürümünü sağlar. “[Sonsuz ve NaN][1]” bölümüne bakın. Karmaşık bir sayı, gerçek kısmı veya sanal kısmı “sayı değil” ise “sayı değil” olarak kabul edilir.

cmath.isnan(0.0, float('nan'))
# Out: True

“math.inf” ve “math.nan” sabitlerinin (Python 3.5 ve sonraki sürümlerinden) “cmath” karşılığı olmadığına dikkat edin.

cmath.isinf(complex(0.0, math.inf))
# Out: True

cmath.isnan(complex(math.nan, 0.0))
# Out: True

cmath.inf
# Exception: AttributeError: module 'cmath' has no attribute 'inf'
son sürüm

Python 3.5 ve üzeri sürümlerde, hem cmath hem de matematik modüllerinde bir “isclose” yöntemi vardır.

z = cmath.rect(*cmath.polar(1+1j))

z
# Out: (1.0000000000000002+1.0000000000000002j)

cmath.isclose(z, 1+1j)
# True
son sürüm
