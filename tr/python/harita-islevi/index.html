Sözdizimi #

  • harita(işlev, yinelenebilir[, *ek_iterables])
  • future_builtins.map(işlev, yinelenebilir[, *ek_iterables])
  • itertools.imap(işlev, yinelenebilir[, *ek_iterables])

Parametreler #

parametre Ayrıntılar
fonksiyon eşleme işlevi (yinelenebilirler olduğu kadar çok parametre almalıdır) (positional-only)
yinelenebilir işlev yinelenebilir öğenin her bir öğesine uygulanır (positional-only)
*ek_iterables yinelenebilir, ancak istediğiniz kadar görün (isteğe bağlı, yalnızca konumsal)

“Harita” ile yapılabilecek her şey, “kavramalar” ile de yapılabilir:

list(map(abs, [-1,-2,-3]))    # [1, 2, 3]
[abs(i) for i in [-1,-2,-3]]  # [1, 2, 3]

Birden fazla yinelenebilir öğeniz varsa ‘zip’e ihtiyacınız olsa da:

import operator
alist = [1,2,3]
list(map(operator.add, alist, alist))  # [2, 4, 6]
[i + j for i, j in zip(alist, alist)]  # [2, 4, 6]

Liste kavrayışları verimlidir ve çoğu durumda “harita"dan daha hızlı olabilir, bu nedenle hız sizin için önemliyse her iki yaklaşımın zamanlarını test edin.

Harita, itertools.imap ve future_builtins.map’in temel kullanımı #

Harita işlevi, işlevsel programlama için kullanılan Python yerleşikleri arasında en basit olanıdır. [map()][3] yinelenebilir bir öğedeki her öğeye belirtilen bir işlevi uygular:

names = ['Fred', 'Wilma', 'Barney']

map(len, names)  # map in Python 3.x is a class; its instances are iterable
# Out: <map object at 0x00000198B32E2CF8>
son sürüm

“future_builtins” modülünde Python 3 uyumlu bir “harita” bulunur:

from future_builtins import map  # contains a Python 3.x compatible map()
map(len, names)                  # see below
# Out: <itertools.imap instance at 0x3eb0a20>
son sürüm

Alternatif olarak, Python 2’de bir jeneratör almak için ‘itertools’tan ‘imap’ kullanılabilir

map(len, names)   # map() returns a list
# Out: [4, 5, 6]

from itertools import imap
imap(len, names)  # itertools.imap() returns a generator
# Out: <itertools.imap at 0x405ea20>
son sürüm

Sonuç, Python 2 ve 3 arasındaki farkları ortadan kaldırmak için açıkça bir “liste"ye dönüştürülebilir:

list(map(len, names))
# Out: [4, 5, 6]

map(), eşdeğer bir [liste anlama][1] veya [oluşturucu ifadesi][2]: ile değiştirilebilir

[len(item) for item in names] # equivalent to Python 2.x map()
# Out: [4, 5, 6]

(len(item) for item in names) # equivalent to Python 3.x map()
# Out: <generator object <genexpr> at 0x00000195888D5FC0>

[1]: https://www.wikiod.com/tr/python/liste-anlama#Anlamaları Listeleme [2]: https://www.wikiod.com/tr/python/liste-anlama#Jeneratör İfadeleri [3]: https://docs.python.org/3/library/functions.html#map

Her değeri yinelenebilir bir şekilde eşleme #

Örneğin, her öğenin mutlak değerini alabilirsiniz:

list(map(abs, (1, -1, 2, -2, 3, -3))) # the call to `list` is unnecessary in 2.x
# Out: [1, 1, 2, 2, 3, 3]

Anonim işlev ayrıca bir listeyi eşlemeyi de destekler:

map(lambda x:x*2, [1, 2, 3, 4, 5])
# Out: [2, 4, 6, 8, 10]

veya ondalık değerleri yüzdelere dönüştürmek:

def to_percent(num):
    return num * 100

list(map(to_percent, [0.95, 0.75, 1.01, 0.1]))
# Out: [95.0, 75.0, 101.0, 10.0]

veya doları avroya çevirmek (bir döviz kuru veriliyse):

from functools import partial
from operator import mul

rate = 0.9  # fictitious exchange rate, 1 dollar = 0.9 euros
dollars = {'under_my_bed': 1000,
           'jeans': 45,
           'bank': 5000}

sum(map(partial(mul, rate), dollars.values()))
# Out: 5440.5

“functools.partial”, “lambda” kullanmak veya özelleştirilmiş işlevler oluşturmak yerine “harita” ile kullanılabilmesi için işlev parametrelerini düzeltmenin uygun bir yoludur.

Farklı yinelenebilirlerin eşleme değerleri #

Örneğin, birden çok yinelenebilir öğenin her bir “i”-inci öğesinin ortalamasını hesaplamak için:

def average(*args):
    return float(sum(args)) / len(args)  # cast to float - only mandatory for python 2.x

measurement1 = [100, 111, 99, 97]
measurement2 = [102, 117, 91, 102]
measurement3 = [104, 102, 95, 101]

list(map(average, measurement1, measurement2, measurement3))
# Out: [102.0, 110.0, 95.0, 100.0]

Python’un sürümüne bağlı olarak birden fazla yinelenebilir “haritaya” geçirilirse farklı gereksinimler vardır:

  • İşlev, yinelenebilir olduğu kadar çok parametre almalıdır:

    def median_of_three(a, b, c):
    return sorted((a, b, c))[1]

list(map(median_of_three, measurement1, measurement2))

TypeError: median_of_ three() gerekli 1 konum bağımsız değişkeni eksik: ‘c’

  list(map(median_of_three, measurement1, measurement2, measurement3, measurement3))

TypeError: median_of_ Three() 3 konumsal argüman alır ancak 4 tanesi verildi

  • map: Eşleme, bir yinelenebilir öğe hala tam olarak tüketilmediği, ancak tamamen tüketilen yinelenebilir öğelerden ‘Yok’ olduğu varsayıldığı sürece yinelenir:

    import operator

measurement1 = [100, 111, 99, 97]
measurement2 = [102, 117]

# Calculate difference between elements
list(map(operator.sub, measurement1, measurement2))

TypeError: -: ‘int’ ve ‘NoneType’ için desteklenmeyen işlenen türleri

  • ‘itertools.imap’ ve ‘future_builtins.map’: Eşleme, yinelenebilir bir şey durur durmaz durur:

    import operator
from itertools import imap

measurement1 = [100, 111, 99, 97]
measurement2 = [102, 117]

# Calculate difference between elements
list(imap(operator.sub, measurement1, measurement2))
# Out: [-2, -6]
list(imap(operator.sub, measurement2, measurement1))
# Out: [2, 6]
son sürüm

  • Eşleme, bir yinelenebilir durur durmaz durur:

    import operator

measurement1 = [100, 111, 99, 97]
measurement2 = [102, 117]

# Calculate difference between elements
list(map(operator.sub, measurement1, measurement2))
# Out: [-2, -6]
list(map(operator.sub, measurement2, measurement1))
# Out: [2, 6]
son sürüm

Harita ile Aktarma: İşlev bağımsız değişkeni olarak “Yok” kullanma (yalnızca python 2.x) #

from itertools import imap
from future_builtins import map as fmap # Different name to highlight differences

image = [[1, 2, 3],
         [4, 5, 6],
         [7, 8, 9]]

list(map(None, *image))
# Out: [(1, 4, 7), (2, 5, 8), (3, 6, 9)]
list(fmap(None, *image))
# Out: [(1, 4, 7), (2, 5, 8), (3, 6, 9)]
list(imap(None, *image))
# Out: [(1, 4, 7), (2, 5, 8), (3, 6, 9)]

image2 = [[1, 2, 3],
          [4, 5],
          [7, 8, 9]]
list(map(None, *image2))
# Out: [(1, 4, 7), (2, 5, 8), (3, None, 9)]  # Fill missing values with None
list(fmap(None, *image2))
# Out: [(1, 4, 7), (2, 5, 8)]                # ignore columns with missing values
list(imap(None, *image2))
# Out: [(1, 4, 7), (2, 5, 8)]                # dito

list(map(None, *image))

TypeError: ‘NoneType’ nesnesi çağrılabilir değil

Ancak benzer sonuçlara sahip olmak için bir geçici çözüm var:

def conv_to_list(*args):
    return list(args)

list(map(conv_to_list, *image))
# Out: [[1, 4, 7], [2, 5, 8], [3, 6, 9]]
son sürüm

Seriler ve Paralel Haritalama #

map() yerleşik bir işlevdir, yani bir ‘import’ ifadesi kullanmaya gerek kalmadan her yerde kullanılabilir. print() gibi her yerde kullanılabilir Örnek 5’e bakarsanız, Pretty print (import pprint) kullanmadan önce bir import ifadesi kullanmam gerektiğini göreceksiniz. Böylece pprint yerleşik bir işlev değildir

Seri haritalama

Bu durumda, yinelenebilirin her argümanı, artan sırada eşleme işlevine argüman olarak sağlanır. Bu, yalnızca bir yinelenebilir haritamız olduğunda ve haritalama işlevi tek bir argüman gerektirdiğinde ortaya çıkar.

örnek 1

insects = ['fly', 'ant', 'beetle', 'cankerworm']
f = lambda x: x + ' is an insect'
print(list(map(f, insects))) # the function defined by f is executed on each item of the iterable insects

sonuçlanır

['fly is an insect', 'ant is an insect', 'beetle is an insect', 'cankerworm is an insect']

Örnek 2

print(list(map(len, insects))) # the len function is executed each item in the insect list

sonuçlanır

[3, 3, 6, 10]

paralel haritalama

Bu durumda, eşleme işlevinin her argümanı, tüm yinelenebilirlerden (her yinelenebilirden bir tane) paralel olarak çekilir. Bu nedenle, sağlanan yinelenebilirlerin sayısı, işlevin gerektirdiği argümanların sayısıyla eşleşmelidir.

carnivores = ['lion', 'tiger', 'leopard', 'arctic fox']
herbivores = ['african buffalo', 'moose', 'okapi', 'parakeet']
omnivores = ['chicken', 'dove', 'mouse', 'pig']

def animals(w, x, y, z):
    return '{0}, {1}, {2}, and {3} ARE ALL ANIMALS'.format(w.title(), x, y, z)

Örnek 3

# Too many arguments
# observe here that map is trying to pass one item each from each of the four iterables to len. This leads len to complain that
# it is being fed too many arguments
print(list(map(len, insects, carnivores, herbivores, omnivores)))

sonuçlanır

TypeError: len() takes exactly one argument (4 given)

Örnek 4

# Too few arguments
# observe here that map is suppose to execute animal on individual elements of insects one-by-one. But animals complain when
# it only gets one argument, whereas it was expecting four.
print(list(map(animals, insects)))

sonuçlanır

TypeError: animals() missing 3 required positional arguments: 'x', 'y', and 'z'

Örnek 5

# here map supplies w, x, y, z with one value from across the list
import pprint
pprint.pprint(list(map(animals, insects, carnivores, herbivores, omnivores)))

sonuçlanır

 ['Fly, lion, african buffalo, and chicken ARE ALL ANIMALS',
 'Ant, tiger, moose, and dove ARE ALL ANIMALS',
 'Beetle, leopard, okapi, and mouse ARE ALL ANIMALS',
 'Cankerworm, arctic fox, parakeet, and pig ARE ALL ANIMALS']
