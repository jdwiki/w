Python, tarih ve saatleri oluşturmak, değiştirmek, ayrıştırmak ve işlemek için hem yerleşik yöntemler hem de harici kitaplıklar sağlar.

Bir dizeyi saat dilimine duyarlı bir tarih saat nesnesine ayrıştırma #

Python 3.2+, bir dize ayrıştırılırken bir “datetime” nesnesine dönüştürülürken “%z” biçimini destekler .

“+HHMM” veya “-HHMM” biçiminde UTC ofseti (nesne saf ise boş dize).

import datetime
dt = datetime.datetime.strptime("2016-04-15T08:27:18-0500", "%Y-%m-%dT%H:%M:%S%z")
son sürüm

Python’un diğer sürümleri için, dateutil gibi bir harici kitaplık kullanabilirsiniz; bu, zaman dilimli bir dizeyi bir “datetime” nesnesine ayrıştırmayı hızlı hale getirir.

import dateutil.parser
dt = dateutil.parser.parse("2016-04-15T08:27:18-0500")

“dt” değişkeni artık aşağıdaki değere sahip bir “datetime” nesnesidir:

datetime.datetime(2016, 4, 15, 8, 27, 18, tzinfo=tzoffset(None, -18000))

Saat dilimine duyarlı tarih saatleri oluşturma #

Varsayılan olarak tüm “datetime” nesneleri saftır. Onları saat dilimine duyarlı hale getirmek için, UTC ofsetini ve saat dilimi kısaltmasını tarih ve saatin bir işlevi olarak sağlayan bir “tzinfo” nesnesi eklemelisiniz.

Sabit Ofset Zaman Dilimleri

UTC’den sabit bir uzaklık olan zaman dilimleri için Python 3.2+ sürümünde, “datetime” modülü “timezone” sınıfını, “tzinfo"nun somut bir uygulamasını sağlar ve bir “timedelta” ve bir (isteğe bağlı) ad parametresi alır:

from datetime import datetime, timedelta, timezone
JST = timezone(timedelta(hours=+9))

dt = datetime(2015, 1, 1, 12, 0, 0, tzinfo=JST)
print(dt)
# 2015-01-01 12:00:00+09:00

print(dt.tzname())
# UTC+09:00

dt = datetime(2015, 1, 1, 12, 0, 0, tzinfo=timezone(timedelta(hours=9), 'JST'))
print(dt.tzname)
# 'JST'
son sürüm

3.2’den önceki Python sürümleri için, dateutil gibi bir üçüncü taraf kitaplığı kullanmak gerekir. “dateutil”, (2.5.3 sürümünden itibaren) “dateutil.tz.tzoffset(tzname, offset)” biçimindeki argümanları alan “tzoffset” adlı eşdeğer bir sınıf sağlar; burada “offset” saniye cinsinden belirtilir:

from datetime import datetime, timedelta
from dateutil import tz

JST = tz.tzoffset('JST', 9 * 3600) # 3600 seconds per hour
dt = datetime(2015, 1, 1, 12, 0, tzinfo=JST)
print(dt)
# 2015-01-01 12:00:00+09:00
print(dt.tzname)
# 'JST'
son sürüm son sürüm

Gün ışığından yararlanma saati olan bölgeler

Yaz saati uygulaması olan bölgeler için python standart kitaplıkları standart bir sınıf sağlamaz, bu nedenle üçüncü taraf kitaplığı kullanmak gerekir. pytz ve dateutil, saat dilimi sınıfları sağlayan popüler kütüphanelerdir.

Statik saat dilimlerine ek olarak, “dateutil”, gün ışığından yararlanma saatini kullanan saat dilimi sınıfları sağlar (tz modülünün belgelerine bakın ). Bir saat dilimi nesnesi almak için ’tz.gettz()’ yöntemini kullanabilirsiniz, bu nesne daha sonra doğrudan ‘datetime’ yapıcısına iletilebilir:

from datetime import datetime
from dateutil import tz
local = tz.gettz() # Local time
PT = tz.gettz('US/Pacific') # Pacific time

dt_l = datetime(2015, 1, 1, 12, tzinfo=local) # I am in EST
dt_pst = datetime(2015, 1, 1, 12, tzinfo=PT)
dt_pdt = datetime(2015, 7, 1, 12, tzinfo=PT) # DST is handled automatically
print(dt_l)
# 2015-01-01 12:00:00-05:00
print(dt_pst)
# 2015-01-01 12:00:00-08:00
print(dt_pdt)
# 2015-07-01 12:00:00-07:00

DİKKAT: 2.5.3 sürümünden itibaren, “dateutil” belirsiz tarih saatlerini doğru şekilde işlemez ve her zaman varsayılan olarak sonraki tarihi kullanır. Örneğin, “2015-11-01 1:30 EDT-4"ü temsil eden bir “dateutil” saat dilimine sahip bir nesne oluşturmanın bir yolu yoktur, çünkü bu bir gün ışığından yararlanma saati geçişi *sırasındadır.

‘pytz’ kullanılırken tüm uç durumlar düzgün bir şekilde işlenir, ancak ‘pytz’ zaman dilimleri yapıcı aracılığıyla doğrudan zaman dilimlerine eklenmemelidir*. Bunun yerine, saat diliminin “localize” yöntemi kullanılarak bir “pytz” saat dilimi eklenmelidir:

from datetime import datetime, timedelta
import pytz

PT = pytz.timezone('US/Pacific')
dt_pst = PT.localize(datetime(2015, 1, 1, 12))
dt_pdt = PT.localize(datetime(2015, 11, 1, 0, 30))
print(dt_pst)
# 2015-01-01 12:00:00-08:00
print(dt_pdt)
# 2015-11-01 00:30:00-07:00

Bir “pytz” farkında saat diliminde tarih saat aritmetiği yapıyorsanız, hesaplamaları UTC’de (mutlak geçen zaman istiyorsanız) yapmanız veya sonucu “normalize()” olarak çağırmanız gerektiğini unutmayın:

dt_new = dt_pdt + timedelta(hours=3) # This should be 2:30 AM PST
print(dt_new)
# 2015-11-01 03:30:00-07:00
dt_corrected = PT.normalize(dt_new)
print(dt_corrected)
# 2015-11-01 02:30:00-08:00

Temel tarih saat nesneleri kullanımı #

Datetime modülü üç ana nesne türü içerir - tarih, saat ve tarih saat.

import datetime

# Date object
today = datetime.date.today()
new_year = datetime.date(2017, 01, 01) #datetime.date(2017, 1, 1)

# Time object
noon = datetime.time(12, 0, 0) #datetime.time(12, 0)

# Current datetime
now = datetime.datetime.now()

# Datetime object
millenium_turn = datetime.datetime(2000, 1, 1, 0, 0, 0) #datetime.datetime(2000, 1, 1, 0, 0)

Bu nesneler için aritmetik işlemler yalnızca aynı veri türü içinde desteklenir ve farklı türlerdeki örneklerle basit aritmetik gerçekleştirmek TypeError ile sonuçlanır.

# subtraction of noon from today
noon-today
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: unsupported operand type(s) for -: 'datetime.time' and 'datetime.date'
However, it is straightforward to convert between types.

# Do this instead
print('Time since the millenium at midnight: ',
      datetime.datetime(today.year, today.month, today.day) - millenium_turn)

# Or this
print('Time since the millenium at noon: ',
      datetime.datetime.combine(today, noon) - millenium_turn)

Zaman farklarını hesaplama #

“timedelta” modülü, zamanlar arasındaki farkları hesaplamak için kullanışlıdır:

from datetime import datetime, timedelta
now = datetime.now()
then = datetime(2016, 5, 23)     # datetime.datetime(2016, 05, 23, 0, 0, 0)

Yeni bir “datetime” nesnesi oluştururken saati belirtmek isteğe bağlıdır

delta = now-then

“delta”, “timedelta” türündedir

print(delta.days)
# 60
print(delta.seconds)
# 40826

n gün sonrası ve n gün öncesi tarihi almak için şunları kullanabiliriz:

n gün sonra:

def get_n_days_after_date(date_format="%d %B %Y", add_days=120):

    date_n_days_after = datetime.datetime.now() + timedelta(days=add_days)
    return date_n_days_after.strftime(date_format)

tarihten önceki gün:

 def get_n_days_before_date(self, date_format="%d %B %Y", days_before=120):

        date_n_days_ago = datetime.datetime.now() - timedelta(days=days_before)
        return date_n_days_ago.strftime(date_format)

Basit tarih aritmetiği #

Tarihler ayrı ayrı mevcut değildir. Tarihler arasındaki süreyi bulmanız veya yarın tarihin ne olacağını belirlemeniz yaygın bir durumdur. Bu, timedelta nesneleri kullanılarak gerçekleştirilebilir

import datetime

today = datetime.date.today()
print('Today:', today)

yesterday = today - datetime.timedelta(days=1)
print('Yesterday:', yesterday)

tomorrow = today + datetime.timedelta(days=1)
print('Tomorrow:', tomorrow)

print('Time between tomorrow and yesterday:', tomorrow - yesterday)

Bu, şuna benzer sonuçlar üretecektir:

Today: 2016-04-15
Yesterday: 2016-04-14
Tomorrow: 2016-04-16
Difference between tomorrow and yesterday: 2 days, 0:00:00

Saat dilimleri arasında geçiş yapma #

Saat dilimleri arasında geçiş yapmak için saat dilimini tanıyan datetime nesnelerine ihtiyacınız vardır.

from datetime import datetime
from dateutil import tz

utc = tz.tzutc()
local = tz.tzlocal()

utc_now = datetime.utcnow()
utc_now # Not timezone-aware.

utc_now = utc_now.replace(tzinfo=utc)
utc_now # Timezone-aware.

local_now = utc_now.astimezone(local)
local_now # Converted to local time.

Bir tarihten ayları doğru şekilde çıkarma #

“takvim” modülünü kullanma

import calendar
from datetime import date

def monthdelta(date, delta):
    m, y = (date.month+delta) % 12, date.year + ((date.month)+delta-1) // 12
    if not m: m = 12
    d = min(date.day, calendar.monthrange(y, m)[1])
    return date.replace(day=d,month=m, year=y)

next_month = monthdelta(date.today(), 1) #datetime.date(2016, 10, 23)

“dateutils” modülünü kullanma

import datetime
import dateutil.relativedelta

d = datetime.datetime.strptime("2013-03-31", "%Y-%m-%d")
d2 = d - dateutil.relativedelta.relativedelta(months=1)  #datetime.datetime(2013, 2, 28, 0, 0)

Tarihleri ​​yineleyin #

Bazen bir başlangıç ​​tarihinden bitiş tarihine kadar bir tarih aralığını yinelemek istersiniz. Bunu “datetime” kitaplığı ve “timedelta” nesnesini kullanarak yapabilirsiniz:

import datetime

# The size of each step in days
day_delta = datetime.timedelta(days=1)

start_date = datetime.date.today()
end_date = start_date + 7*day_delta

for i in range((end_date - start_date).days):
    print(start_date + i*day_delta)

Hangi üretir:

2016-07-21
2016-07-22
2016-07-23
2016-07-24
2016-07-25
2016-07-26
2016-07-27

Kısa bir saat dilimi adına sahip bir dizeyi saat dilimine duyarlı bir tarih saat nesnesine ayrıştırma #

‘dateutil’ kitaplığını saat dilimine duyarlı zaman damgalarını ayrıştırma ile ilgili önceki örnekte olduğu gibi kullanma -time/1592/parsing-a-string-into-a-timezone-aware-datetime-object), belirtilen bir “kısa” zaman dilimi adıyla zaman damgalarını ayrıştırmak da mümkündür.

Genellikle belirsiz olan kısa saat dilimi adları veya kısaltmalarla biçimlendirilmiş tarihler için (ör. CST, Merkezi Standart Saati, Çin Standart Saati, Küba Standart Saati vb. olabilir) - daha fazlasını [burada](https://www. timeanddate.com/time/zones/)) veya standart bir veritabanında bulunması zorunlu değilse, saat dilimi kısaltması ile ’tzinfo’ nesnesi arasında bir eşleme belirtmek gerekir.

from dateutil import tz
from dateutil.parser import parse

ET = tz.gettz('US/Eastern')
CT = tz.gettz('US/Central')
MT = tz.gettz('US/Mountain')
PT = tz.gettz('US/Pacific')

us_tzinfos = {'CST': CT, 'CDT': CT,
              'EST': ET, 'EDT': ET,
              'MST': MT, 'MDT': MT,
              'PST': PT, 'PDT': PT}

dt_est = parse('2014-01-02 04:00:00 EST', tzinfos=us_tzinfos)
dt_pst = parse('2016-03-11 16:00:00 PST', tzinfos=us_tzinfos)

Bunu çalıştırdıktan sonra:

dt_est
# datetime.datetime(2014, 1, 2, 4, 0, tzinfo=tzfile('/usr/share/zoneinfo/US/Eastern'))
dt_pst
# datetime.datetime(2016, 3, 11, 16, 0, tzinfo=tzfile('/usr/share/zoneinfo/US/Pacific'))

Bu yöntemle bir “pytz” zaman dilimi kullanılıyorsa, doğru şekilde yerelleştirilmeyeceğini * belirtmekte fayda var:

from dateutil.parser import parse
import pytz

EST = pytz.timezone('America/New_York')
dt = parse('2014-02-03 09:17:00 EST', tzinfos={'EST': EST})

Bu, yalnızca ‘pytz’ saat dilimini tarih saatine ekler:

dt.tzinfo # Will be in Local Mean Time!
# <DstTzInfo 'America/New_York' LMT-1 day, 19:04:00 STD>

Bu yöntemi kullanıyorsanız, ayrıştırdıktan sonra tarih saatinin saf kısmını muhtemelen yeniden “yerelleştirmelisiniz”:

dt_fixed = dt.tzinfo.localize(dt.replace(tzinfo=None))
dt_fixed.tzinfo # Now it's EST.
# <DstTzInfo 'America/New_York' EST-1 day, 19:00:00 STD>)

Bulanık tarih saat ayrıştırma (bir metinden tarih saati çıkarma) #

Bir tarihin parçası olarak tanınmayan dize bileşenlerinin yok sayıldığı “bulanık” bir modda dateutil ayrıştırıcısı kullanılarak metinden bir tarih çıkarmak mümkündür.

from dateutil.parser import parse

dt = parse("Today is January 1, 2047 at 8:21:00AM", fuzzy=True)
print(dt)

“dt” artık bir “datetime” nesnesidir ve “datetime.datetime(2047, 1, 1, 8, 21)” ifadesinin yazdırıldığını görürsünüz.

Minimum kitaplıklarla rastgele bir ISO 8601 zaman damgasını ayrıştırma #

Python, ISO 8601 zaman damgalarını ayrıştırmak için yalnızca sınırlı desteğe sahiptir. “strptime” için tam olarak hangi formatta olduğunu bilmeniz gerekir. Bir komplikasyon olarak, bir “datetime"ın dizgeleştirilmesi, ayırıcı olarak boşluk ve 6 basamaklı kesir içeren bir ISO 8601 zaman damgasıdır:

str(datetime.datetime(2016, 7, 22, 9, 25, 59, 555555))
# '2016-07-22 09:25:59.555555'

ancak kesir 0 ise, kesirli kısım çıkmaz

str(datetime.datetime(2016, 7, 22, 9, 25, 59, 0))
# '2016-07-22 09:25:59'

Ancak bu 2 formun “strptime” için farklı bir formata ihtiyacı var. Ayrıca, “strptime”, içinde “:” bulunan dakika zaman dilimlerinin ayrıştırılmasını hiçbir şekilde desteklemez, bu nedenle “2016-07-22 09:25:59+0300” ayrıştırılabilir, ancak standart biçim “2016-07-” 22 09:25:59+03:00` olamaz.

tek dosya``https](iso) adlı bir kitaplık var ://pypi.python.org/pypi/iso8601) ISO 8601 zaman damgalarını ve yalnızca bunları uygun şekilde ayrıştırır.

Kesirleri ve saat dilimlerini ve “T” ayırıcısını tek bir işlevle destekler:

import iso8601
iso8601.parse_date('2016-07-22 09:25:59')
# datetime.datetime(2016, 7, 22, 9, 25, 59, tzinfo=<iso8601.Utc>)
iso8601.parse_date('2016-07-22 09:25:59+03:00')
# datetime.datetime(2016, 7, 22, 9, 25, 59, tzinfo=<FixedOffset '+03:00' ...>)
iso8601.parse_date('2016-07-22 09:25:59Z')
# datetime.datetime(2016, 7, 22, 9, 25, 59, tzinfo=<iso8601.Utc>)
iso8601.parse_date('2016-07-22T09:25:59.000111+03:00')
# datetime.datetime(2016, 7, 22, 9, 25, 59, 111, tzinfo=<FixedOffset '+03:00' ...>)

Herhangi bir saat dilimi ayarlanmamışsa, “iso8601.parse_date” varsayılanı UTC’dir. Varsayılan bölge, ‘default_zone’ anahtar kelime argümanı ile değiştirilebilir. Özellikle, bu varsayılan yerine “Yok” ise, açık bir saat dilimine sahip olmayan zaman damgaları bunun yerine saf tarih saatleri olarak döndürülür:

iso8601.parse_date('2016-07-22T09:25:59', default_timezone=None)
# datetime.datetime(2016, 7, 22, 9, 25, 59)
iso8601.parse_date('2016-07-22T09:25:59Z', default_timezone=None)
# datetime.datetime(2016, 7, 22, 9, 25, 59, tzinfo=<iso8601.Utc>)

ISO 8601 zaman damgası alın #

Saat dilimi olmadan, mikrosaniye ile #

from datetime import datetime

datetime.now().isoformat()
# Out: '2016-07-31T23:08:20.886783'

Saat dilimiyle, mikrosaniyeyle #

from datetime import datetime
from dateutil.tz import tzlocal

datetime.now(tzlocal()).isoformat()
# Out: '2016-07-31T23:09:43.535074-07:00'

Saat dilimi ile, mikrosaniye olmadan #

from datetime import datetime
from dateutil.tz import tzlocal

datetime.now(tzlocal()).replace(microsecond=0).isoformat()
# Out: '2016-07-31T23:10:30-07:00'
