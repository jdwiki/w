Liste kavrama, 1’den 10’a kadar sayıların karelerinin bir listesini yapmak için aşağıdaki kodda gösterildiği gibi, doğal ve özlü bir şekilde listeler oluşturmak için sözdizimsel bir araçtır:

[i ** 2 için i aralığında(1,11)]

Varolan bir “aralık” listesindeki kukla “i”, yeni bir eleman deseni oluşturmak için kullanılır. Daha az anlamlı dillerde bir for döngüsünün gerekli olduğu durumlarda kullanılır.

Sözdizimi #

  • [i for i in range(10)] # temel liste anlama
  • [i for i in xrange(10)] # python 2.x’te oluşturucu nesne ile temel liste kavrama
  • [i for i in range(20) if i % 2 == 0] # filtreli
  • [x + y [1, 2, 3] içindeki x için [3, 4, 5]] içindeki y için # iç içe döngüler
  • [i if i > 6 ise 0 for i in range(10)] # üçlü ifade
  • [i if i > 4 değilse 0 için aralıkta (20) if i % 2 == 0] # filtre ve üçlü ifade ile
  • [[x + y için [1, 2, 3]] için y için [3, 4, 5]] # iç içe liste kavrama

Liste kavrayışları PEP 202 içinde özetlendi ve Python 2.0’da tanıtıldı.

Koşullu Liste Kavramaları #

Bir liste anlama verildiğinde, değerleri filtrelemek için bir veya daha fazla “if” koşulu ekleyebilirsiniz.

[<expression> for <element> in <iterable> if <condition>]

“” içindeki her bir “” için; “<Koşul>”, “Doğru” olarak değerlendirilirse, döndürülen listeye “” (genellikle “” işlevinin bir işlevi) ekleyin.

Örneğin, bu, bir tamsayı dizisinden yalnızca çift sayıları çıkarmak için kullanılabilir:

[x for x in range(10) if x % 2 == 0]
# Out: [0, 2, 4, 6, 8]

Canlı demo

Yukarıdaki kod şuna eşdeğerdir:

even_numbers = [] 
for x in range(10):
    if x % 2 == 0:
        even_numbers.append(x)

print(even_numbers)
# Out: [0, 2, 4, 6, 8]

Ayrıca, “[e for x in y if c]” formunun koşullu bir liste kavrayışı (burada “e” ve “c”, “x” cinsinden ifadelerdir) “list(filter(lambda x: c) ile eşdeğerdir. , harita(lambda x: e, y)))`.

Aynı sonucu vermesine rağmen, ilk örneğin ikincisinden neredeyse 2 kat daha hızlı olmasına dikkat edin. Merak edenler için bu bunun güzel bir açıklamasıdır. .

Bunun “… if … else …” koşullu ifadesinden (bazen üçlü ifade olarak da bilinir) oldukça farklı olduğuna dikkat edin. liste anlama. Aşağıdaki örneği göz önünde bulundurun:

[x if x % 2 == 0 else None for x in range(10)]
# Out: [0, None, 2, None, 4, None, 6, None, 8, None]

Canlı demo

Burada koşullu ifade bir filtre değil, liste öğeleri için kullanılacak değeri belirleyen bir operatördür:

<value-if-condition-is-true> if <condition> else <value-if-condition-is-false>

Diğer operatörlerle birleştirirseniz bu daha belirgin hale gelir:

[2 * (x if x % 2 == 0 else -1) + 1 for x in range(10)]
# Out: [1, -1, 5, -1, 9, -1, 13, -1, 17, -1]

Canlı demo

Python 2.7 kullanıyorsanız, “xrange”, “xrange” belgelerinde açıklandığı gibi çeşitli nedenlerle “aralık"tan daha iyi olabilir.

[2 * (x if x % 2 == 0 else -1) + 1 for x in xrange(10)]
# Out: [1, -1, 5, -1, 9, -1, 13, -1, 17, -1]

Yukarıdaki kod şuna eşdeğerdir:

numbers = []
for x in range(10):
    if x % 2 == 0:
        temp = x
    else:
        temp = -1
    numbers.append(2 * temp + 1)
print(numbers)
# Out: [1, -1, 5, -1, 9, -1, 13, -1, 17, -1]

Üçlü ifadeler ve ‘if’ koşulları birleştirilebilir. Üçlü operatör, filtrelenmiş sonuç üzerinde çalışır:

[x if x > 2 else '*' for x in range(10) if x % 2 == 0]
# Out: ['*', '*', 4, 6, 8]

Aynısı yalnızca üçlü operatör tarafından elde edilemezdi:

[x if (x > 2 and x % 2 == 0) else '*' for x in range(10)]
# Out:['*', '*', '*', '*', 4, '*', 6, '*', 8, '*']

Ayrıca bkz: Filtreler, bunlar genellikle koşullu liste kavramalarına yeterli bir alternatif sağlar.

2: https://www.wikiod.com/tr/python/sartllar#Koşullu İfade (veya “Üçlü Operatör”) 3: https://docs.python.org/2/library/functions.html#xrange

İç İçe Döngülerle Anlamaları Listeleme #

Liste Anlamaları iç içe “for” döngülerini kullanabilir. Bir liste kavrayışı içinde istediğiniz sayıda iç içe for döngüsü kodlayabilirsiniz ve her bir “for” döngüsü, isteğe bağlı bir ilişkili “if” testine sahip olabilir. Bunu yaparken, ‘for’ yapılarının sırası, bir dizi iç içe ‘for’ ifadesi yazarkenki sıra ile aynıdır. Liste kavrayışlarının genel yapısı şöyle görünür:

[ expression for target1 in iterable1 [if condition1]
             for target2 in iterable2 [if condition2]...
             for targetN in iterableN [if conditionN] ]

Örneğin, birden çok “for” ifadesi kullanarak bir liste listesini düzleştiren aşağıdaki kod:

data = [[1, 2], [3, 4], [5, 6]]
output = []
for each_list in data:
    for element in each_list:
        output.append(element)
print(output)
# Out: [1, 2, 3, 4, 5, 6]

birden çok “for” yapısıyla eşdeğer bir liste kavrayışı olarak yazılabilir:

data = [[1, 2], [3, 4], [5, 6]]
output = [element for each_list in data for element in each_list]
print(output)
# Out: [1, 2, 3, 4, 5, 6]

Canlı Demo

Hem genişletilmiş formda hem de liste kavrayışında, dış döngü (ilk ifade için) önce gelir.

Daha kompakt olmanın yanı sıra, iç içe kavrama da önemli ölçüde daha hızlıdır.

In [1]: data = [[1,2],[3,4],[5,6]]
In [2]: def f():
   ...:     output=[]
   ...:     for each_list in data:
   ...:         for element in each_list:
   ...:             output.append(element)
   ...:     return output
In [3]: timeit f()
1000000 loops, best of 3: 1.37 µs per loop
In [4]: timeit [inner for outer in data for inner in outer]
1000000 loops, best of 3: 632 ns per loop

Yukarıdaki işlev çağrısının ek yükü yaklaşık 140ns‘dir.

Satır içi “if"ler benzer şekilde iç içedir ve ilk “for"dan sonra herhangi bir konumda olabilir:

data = [[1], [2, 3], [4, 5]]
output = [element for each_list in data
                if len(each_list) == 2
                for element in each_list
                if element != 5]
print(output)
# Out: [2, 3, 4]

Canlı Demo

Ancak okunabilirlik adına geleneksel for-loop kullanmayı düşünmelisiniz. Bu, özellikle yuvalama 2 seviyeden daha derin olduğunda ve/veya anlama mantığının çok karmaşık olduğu durumlarda geçerlidir. birden çok iç içe döngü listesi kavrayışı hataya açık olabilir veya beklenmeyen sonuçlar verir.

Yeniden düzenleme filtresi ve kavramaları listelemek için harita #

“filtre” veya “harita” işlevleri genellikle liste kavramaları ile değiştirilmelidir. Guido Van Rossum bunu bir [2005’teki açık mektupta2] çok iyi anlatıyor:

“filter(P, S)” neredeyse her zaman “[x for x in S if P(x)]” şeklinde daha açık yazılır ve bu, en yaygın kullanımların karşılaştırma olan yüklemleri içermesi gibi büyük bir avantaja sahiptir, ör. x==42 ve bunun için bir lambda tanımlamak okuyucu için çok daha fazla çaba gerektirir (artı lambda liste kavrayışından daha yavaştır). Daha da fazlası, ‘[F(x) for x in S]’ olan ‘harita(F, S)’ için. Elbette, çoğu durumda bunun yerine oluşturucu ifadeleri kullanabilirsiniz.

Aşağıdaki kod satırları “* Pythonic değil*” olarak kabul edilir ve birçok python linter’de hatalara neden olur.

filter(lambda x: x % 2 == 0, range(10)) # even numbers < 10
map(lambda x: 2*x, range(10)) # multiply each number by two
reduce(lambda x,y: x+y, range(10)) # sum of all elements in list

Önceki alıntıdan öğrendiklerimizi alarak, bu ‘filtre’ ve ‘harita’ ifadelerini eşdeğer liste kavrayışlarına ayırabiliriz; ayrıca lambda işlevlerinin her birinden kaldırılması - bu süreçte kodun daha okunabilir olmasını sağlar.

# Filter:
# P(x) = x % 2 == 0
# S = range(10)
[x for x in range(10) if x % 2 == 0]

# Map
# F(x) = 2*x
# S = range(10)
[2*x for x in range(10)]

Zincirleme işlevleriyle uğraşırken okunabilirlik daha da belirgin hale gelir. Okunabilirlik nedeniyle, bir haritanın veya filtre işlevinin sonuçlarının bir sonrakine sonuç olarak iletilmesi gerektiğinde; basit durumlarda, bunlar tek bir liste anlama ile değiştirilebilir. Ayrıca, zincirleme Harita ve Filtre süreci hakkında akıl yürütürken daha fazla bilişsel yükün olduğu, sürecimizin sonucunun ne olduğunu liste kavrayışından kolayca söyleyebiliriz.

# Map & Filter
filtered = filter(lambda x: x % 2 == 0, range(10))
results = map(lambda x: 2*x, filtered)

# List comprehension
results = [2*x for x in range(10) if x % 2 == 0]

Yeniden Düzenleme - Hızlı Başvuru #

  • Harita

    map(F, S) == [F(x) for x in S]

  • Filtre

    filter(P, S) == [x for x in S if P(x)]

burada “F” ve “P” sırasıyla giriş değerlerini dönüştüren ve bir “bool” döndüren işlevlerdir

İç İçe Liste Kavramaları #

İç içe liste kavrayışları, iç içe döngülere sahip liste kavrayışlarından farklı olarak, bir liste kavrayışı içindeki Liste kavrayışlarıdır. İlk ifade, başka bir liste kavrayışı da dahil olmak üzere herhangi bir rastgele ifade olabilir.

#List Comprehension with nested loop
[x + y for x in [1, 2, 3] for y in [3, 4, 5]]
#Out: [4, 5, 6, 5, 6, 7, 6, 7, 8]

#Nested List Comprehension
[[x + y for x in [1, 2, 3]] for y in [3, 4, 5]]
#Out: [[4, 5, 6], [5, 6, 7], [6, 7, 8]]

İç içe örnek eşdeğerdir

l = []
for y in [3, 4, 5]:
    temp = []
    for x in [1, 2, 3]:
        temp.append(x + y)
    l.append(temp)

İç içe kavramanın bir matrisi dönüştürmek için kullanılabileceği bir örnek.

matrix = [[1,2,3],
          [4,5,6],
          [7,8,9]] 

[[row[i] for row in matrix] for i in range(len(matrix))]
# [[1, 4, 7], [2, 5, 8], [3, 6, 9]]

İç içe ‘for’ döngüleri gibi, derin kavrayışların ne kadar iç içe olabileceğinin bir sınırı yoktur.

[[[i + j + k for k in 'cd'] for j in 'ab'] for i in '12']
# Out: [[['1ac', '1ad'], ['1bc', '1bd']], [['2ac', '2ad'], ['2bc', '2bd']]]

Liste kavrayışında iki veya daha fazla listeyi aynı anda yineleyin #

liste anlama içinde aynı anda ikiden fazla listeyi yinelemek için, zip() şu şekilde kullanılabilir:

>>> list_1 = [1, 2, 3 , 4]
>>> list_2 = ['a', 'b', 'c', 'd']
>>> list_3 = ['6', '7', '8', '9']

# Two lists
>>> [(i, j) for i, j in zip(list_1, list_2)]
[(1, 'a'), (2, 'b'), (3, 'c'), (4, 'd')]

# Three lists
>>> [(i, j, k) for i, j, k in zip(list_1, list_2, list_3)]
[(1, 'a', '6'), (2, 'b', '7'), (3, 'c', '8'), (4, 'd', '9')]

# so on ...
