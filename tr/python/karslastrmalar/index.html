Sözdizimi #

  • != - Eşit değildir

  • == - Şuna eşittir:

  • > - şundan büyük

  • < - daha az

  • >= - büyük veya eşit

  • <= - küçük veya eşit

  • is - nesnelerin tamamen aynı nesne olup olmadığını test edin

  • değildir = nesnelerin tam olarak aynı nesne olup olmadığını test edin

Parametreler #

parametre Ayrıntılar
x Karşılaştırılacak ilk öğe
y Karşılaştırılacak ikinci öğe

Zincir Karşılaştırmaları #

Zincir karşılaştırma ile birden çok öğeyi birden çok karşılaştırma operatörüyle karşılaştırabilirsiniz. Örneğin

x > y > z

sadece kısa bir şeklidir:

x > y and y > z

Bu, yalnızca her iki karşılaştırmanın da “Doğru” olması durumunda “Doğru” olarak değerlendirilir.

genel formu

a OP b OP c OP d ...

“OP”, kullanabileceğiniz çoklu karşılaştırma işlemlerinden birini temsil eder ve harfler rastgele geçerli ifadeleri temsil eder.

“0 != 0”, “Yanlış” olsa bile “0 != 1 != 0"ın “Doğru” olarak değerlendirildiğini unutmayın. “x != y != z"nin “x”, “y” ve “z"nin farklı değerlere sahip olduğu anlamına geldiği yaygın matematiksel gösterimin aksine. Eşitlik genellikle geçişli olduğundan zincirleme == işlemleri çoğu durumda doğal bir anlama sahiptir.

Stil #

Doğru sözdizimine sahip olduğunuz sürece, kaç tane öğe ve karşılaştırma işlemi kullanacağınız konusunda teorik bir sınır yoktur:

1 > -1 < 2 > 0.5 < 100 != 24

Her karşılaştırma “True” değerini döndürürse, yukarıdaki “True” değerini döndürür. Ancak, kıvrımlı zincirleme kullanmak iyi bir stil değildir. İyi bir zincirleme “yönlü” olacaktır, bundan daha karmaşık değildir.

1 > x > -4 > y != 8

Yan etkiler #

Bir karşılaştırma “Yanlış” değerini döndürür döndürmez, ifade kalan tüm karşılaştırmaları atlayarak hemen “Yanlış” olarak değerlendirilir.

‘a > exp > b’ içindeki ’exp’ ifadesinin yalnızca bir kez değerlendirileceğini, oysa

a > exp and exp > b

“a > exp” doğruysa “exp” iki kez hesaplanır.

“is” ile “==” karşılaştırması #

Yaygın bir tuzak, eşitlik karşılaştırma işleçlerinin “is” ve “==” karıştırılmasıdır.

“a == b”, “a” ve “b"nin değerini karşılaştırır.

“a is b”, “a” ve “b"nin _kimliklerini karşılaştırır.

Örneklemek için:

a = 'Python is fun!'
b = 'Python is fun!'
a == b # returns True
a is b # returns False

a = [1, 2, 3, 4, 5]
b = a      # b references a
a == b     # True
a is b     # True
b = a[:]   # b now references a copy of a
a == b     # True
a is b     # False [!!]

Temel olarak, “is”, “id(a) == id(b)“nin kısaltması olarak düşünülebilir.

Bunun ötesinde, çalışma zamanı ortamının işleri daha da karmaşık hale getiren tuhaflıkları vardır. Python makinesinin aynı nesneler için daha az bellek kullanmaya çalışması nedeniyle, kısa dizeler ve küçük tamsayılar, “is” ile karşılaştırıldığında “True” değerini döndürür.

a = 'short'
b = 'short'
c = 5
d = 5
a is b # True
c is d # True

Ancak daha uzun dizeler ve daha büyük tamsayılar ayrı ayrı saklanacaktır.

a = 'not so short'
b = 'not so short'
c = 1000
d = 1000
a is b # False
c is d # False

‘Yok’ olup olmadığını test etmek için ‘is’ kullanmalısınız:

if myvar is not None:
    # not None
    pass
if myvar is None:
    # None
    pass

‘is’ kullanımı, bir “nöbetçi” (yani benzersiz bir nesne) olup olmadığını test etmektir.

sentinel = object()
def myfunc(var=sentinel):
    if var is sentinel:
        # value wasn’t provided
        pass
    else:
        # value was provided
        pass

Büyüktür veya küçüktür #

x > y
x < y

Bu operatörler iki tür değeri karşılaştırır, bunlar operatörlerden küçük ve büyüktür. Sayılar için bu, hangisinin daha büyük olduğunu görmek için sayısal değerleri karşılaştırır:

12 > 4
# True
12 < 4
# False
1 < 4
# True

Dizeler için, alfabetik sıraya benzer ancak tamamen aynı olmayan sözlükbilimsel olarak karşılaştıracaklar.

"alpha" < "beta"
# True
"gamma" > "beta"
# True
"gamma" < "OMEGA"
# False

Bu karşılaştırmalarda, küçük harfler büyük harften “büyük” olarak kabul edilir, bu nedenle “gama” < “OMEGA” yanlıştır. Hepsi büyük harf olsaydı, beklenen alfabetik sıralama sonucunu döndürürdü:

"GAMMA" < "OMEGA"
# True

Her tür, hesaplamasını < ve > operatörleriyle farklı şekilde tanımlar, bu nedenle, kullanmadan önce operatörlerin belirli bir türle ne anlama geldiğini araştırmalısınız.

Eşit değil #

x != y

Bu, “x” ve “y” eşit değilse “Doğru”, aksi takdirde “Yanlış” değerini döndürür.

12 != 1
# True
12 != '12'
# True
'12' != '12'
# False

Eşittir #

x == y

Bu ifade, “x” ve “y"nin aynı değer olup olmadığını değerlendirir ve sonucu bir boole değeri olarak döndürür. Genellikle hem tür hem de değerin eşleşmesi gerekir, bu nedenle int “12”, “12” dizesiyle aynı değildir.

12 == 12
# True
12 == 1
# False
'12' == '12'
# True
'spam' == 'spam'
# True
'spam' == 'spam '
# False
'12' == 12
# False

Her türün, iki değerin aynı olup olmadığını değerlendirmek için kullanılacak bir işlev tanımlaması gerektiğini unutmayın. Yerleşik türler için bu işlevler beklediğiniz gibi davranır ve şeyleri aynı değere göre değerlendirir. Ancak özel türler, eşitlik testini, her zaman “Doğru” veya her zaman “Yanlış” döndürme dahil, istedikleri gibi tanımlayabilir.

Nesneleri Karşılaştırma #

Özel sınıfların eşitliğini karşılaştırmak için __eq__ ve __ne__ yöntemlerini tanımlayarak == ve != geçersiz kılabilirsiniz. Ayrıca “lt” (<), __le__ (<=), __gt__ (>) ve __ge__ (>) geçersiz kılabilirsiniz. Yalnızca iki karşılaştırma yöntemini geçersiz kılmanız gerektiğini ve Python’un gerisini halledebileceğini unutmayın (’==’, ‘değil <’ ve ‘değil >’ ile aynıdır, vb.)

class Foo(object):
    def __init__(self, item):
        self.my_item = item
    def __eq__(self, other):
        return self.my_item == other.my_item
    
a = Foo(5)
b = Foo(5)
a == b     # True
a != b     # False
a is b     # False

Bu basit karşılaştırmanın, “diğer"in (karşılaştırılan nesne) aynı nesne türü olduğunu varsaydığına dikkat edin. Başka bir türle karşılaştırmak bir hata verecektir:

class Bar(object):
    def __init__(self, item):
        self.other_item = item
    def __eq__(self, other):
        return self.other_item == other.other_item
    def __ne__(self, other):
        return self.other_item != other.other_item
    
c = Bar(5)
a == c    # throws AttributeError: 'Foo' object has no attribute 'other_item'

‘isinstance()’ veya benzerini kontrol etmek (istenirse) bunu önlemeye yardımcı olacaktır.

Common Gotcha: Python yazmayı zorunlu kılmaz #

Diğer birçok dilde, aşağıdakileri çalıştırırsanız (Java örneği)

if("asgdsrf" == 0) {
    //do stuff
}

… bir hata alırsınız. Dizeleri böyle tam sayılarla karşılaştırmaya gidemezsiniz. Python’da bu tamamen yasal bir ifadedir - sadece ‘Yanlış’ olarak çözülecektir.

Ortak bir yakalama şudur

myVariable = "1"
if 1 == myVariable:
    #do stuff

Bu karşılaştırma, her seferinde bir hatayı gizleyerek veya bir koşulu bozarak, hatasız olarak “Yanlış” olarak değerlendirilir.
