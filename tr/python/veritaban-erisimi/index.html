Python, birçok farklı veritabanı türünü işleyebilir. Bu türlerin her biri için farklı bir API mevcuttur. Bu farklı API’ler arasındaki benzerliği teşvik etmek için PEP 249 tanıtıldı.

Bu API, veritabanlarına erişmek için kullanılan Python modülleri arasındaki benzerliği teşvik etmek için tanımlanmıştır. Bunu yaparak, daha kolay anlaşılan modüller, genellikle veritabanları arasında daha taşınabilir olan kod ve Python’dan daha geniş bir veritabanı bağlantısı erişimi sağlayan bir tutarlılık elde etmeyi umuyoruz. PEP-249

SQLite #

SQLite hafif, disk tabanlı bir veritabanıdır. Ayrı bir veritabanı sunucusu gerektirmediğinden, genellikle tek bir kullanıcı tarafından veya belirli bir zamanda bir kullanıcı tarafından sıklıkla kullanılan prototipleme veya küçük uygulamalar için kullanılır.

import sqlite3

conn = sqlite3.connect("users.db")
c = conn.cursor()

c.execute("CREATE TABLE user (name text, age integer)")

c.execute("INSERT INTO user VALUES ('User A', 42)")
c.execute("INSERT INTO user VALUES ('User B', 43)")

conn.commit()

c.execute("SELECT * FROM user")
print(c.fetchall())

conn.close()

Yukarıdaki kod, users.db adlı dosyada depolanan veritabanına bağlanır ve dosya zaten mevcut değilse önce dosyayı oluşturur. SQL ifadeleri aracılığıyla veritabanı ile etkileşime girebilirsiniz.

Bu örneğin sonucu şöyle olmalıdır:

[(u'User A', 42), (u'User B', 43)]

SQLite Sözdizimi: Derinlemesine bir analiz #

Başlarken #

  1. sqlite modülünü kullanarak içe aktarın

     >>> import sqlite3

  2. Modülü kullanmak için öncelikle veritabanını temsil eden bir Connection nesnesi oluşturmalısınız. Burada veriler example.db dosyasında saklanacaktır:

    >>> conn = sqlite3.connect('users.db')

    Alternatively, you can also supply the special name :memory: to create a temporary database in RAM, as follows:

    >>> conn = sqlite3.connect(':memory:')

  3. Bir “Bağlantı"ya sahip olduğunuzda, bir “İmleç” nesnesi oluşturabilir ve SQL komutlarını gerçekleştirmek için onun “execute()” yöntemini çağırabilirsiniz:

    #

      c = conn.cursor()

  # Create table
  c.execute('''CREATE TABLE stocks
              (date text, trans text, symbol text, qty real, price real)''')

  # Insert a row of data
  c.execute("INSERT INTO stocks VALUES ('2006-01-05','BUY','RHAT',100,35.14)")

  # Save (commit) the changes
  conn.commit()

  # We can also close the connection if we are done with it.
  # Just be sure any changes have been committed or they will be lost.
  conn.close()

“Bağlantı"nın Önemli Nitelikleri ve İşlevleri #

  1. “izolasyon_seviyesi”

    It is an attribute used to get or set the current isolation level. None for autocommit mode or one of DEFERRED, IMMEDIATE or EXCLUSIVE.

  2. “imleç”

    The cursor object is used to execute SQL commands and queries.

  3. taahhüt()

    Commits the current transaction.

  4. geri alma()

    Rolls back any changes made since the previous call to commit()

  5. kapat()

    Closes the database connection. It does not call commit() automatically. If close() is called without first calling commit() (assuming you are not in autocommit mode) then all changes made will be lost.

  6. “toplam_değişiklikler”

    An attribute that logs the total number of rows modified, deleted or inserted since the database was opened.

  7. “execute”, “executemany” ve “executescript”

    These functions perform the same way as those of the cursor object. This is a shortcut since calling these functions through the connection object results in the creation of an intermediate cursor object and calls the corresponding method of the cursor object

  8. “sıra_fabrika”

    You can change this attribute to a callable that accepts the cursor and the original row as a tuple and will return the real result row.

    def dict_factory(cursor, row):
    d = {}
    for i, col in enumerate(cursor.description):
        d[col[0]] = row[i]
    return d

conn = sqlite3.connect(":memory:")
conn.row_factory = dict_factory

“İmleç"in Önemli İşlevleri #

  1. execute(sql[, parametreler])

    Executes a single SQL statement. The SQL statement may be parametrized (i. e. placeholders instead of SQL literals). The sqlite3 module supports two kinds of placeholders: question marks ? (“qmark style”) and named placeholders :name (“named style”).

    import sqlite3
conn = sqlite3.connect(":memory:")
cur = conn.cursor()
cur.execute("create table people (name, age)")

who = "Sophia"
age = 37
# This is the qmark style:
cur.execute("insert into people values (?, ?)",
            (who, age))

# And this is the named style:
cur.execute("select * from people where name=:who and age=:age",
            {"who": who, "age": age})  # the keys correspond to the placeholders in SQL

print(cur.fetchone())

Dikkat: SQL komutlarına dize eklemek için %s kullanmayın. programınızı bir SQL enjeksiyon saldırısına karşı savunmasız hale getirebilir (bkz. https://www.wikiod.com/tr/sql/sql-enjeksiyonu).

  1. executemany(sql, seq_of_parameters)

    Executes an SQL command against all parameter sequences or mappings found in the sequence sql. The sqlite3 module also allows using an iterator yielding parameters instead of a sequence.

    L = [(1, 'abcd', 'dfj', 300),    # A list of tuples to be inserted into the database
     (2, 'cfgd', 'dyfj', 400),
     (3, 'sdd', 'dfjh', 300.50)]                           

conn = sqlite3.connect("test1.db")
conn.execute("create table if not exists book (id int, name text, author text, price real)")
conn.executemany("insert into book values (?, ?, ?, ?)", L)

for row in conn.execute("select * from book"):
    print(row)

    You can also pass iterator objects as a parameter to executemany, and the function will iterate over the each tuple of values that the iterator returns. The iterator must return a tuple of values.

    import sqlite3

class IterChars:
    def __init__(self):
        self.count = ord('a')

    def __iter__(self):
        return self

    def __next__(self):            # (use next(self) for Python 2)
        if self.count > ord('z'):
            raise StopIteration
        self.count += 1
        return (chr(self.count - 1),) 

conn = sqlite3.connect("abc.db")
cur = conn.cursor()
cur.execute("create table characters(c)")

theIter = IterChars()
cur.executemany("insert into characters(c) values (?)", theIter)

rows = cur.execute("select c from characters")
for row in rows:
    print(row[0]),

  2. executescript(sql_script)

    This is a nonstandard convenience method for executing multiple SQL statements at once. It issues a COMMIT statement first, then executes the SQL script it gets as a parameter.

    sql_script can be an instance of str or bytes.

    import sqlite3
conn = sqlite3.connect(":memory:")
cur = conn.cursor()
cur.executescript("""
     create table person(
         firstname,
         lastname,
         age
     );

     create table book(
         title,
         author,
         published
     );

     insert into book(title, author, published)
     values (
         'Dirk Gently''s Holistic Detective Agency',
         'Douglas Adams',
         1987
     );
     """)

    The next set of functions are used in conjunction with SELECT statements in SQL. To retrieve data after executing a SELECT statement, you can either treat the cursor as an iterator, call the cursor’s fetchone() method to retrieve a single matching row, or call fetchall() to get a list of the matching rows.

    Example of the iterator form:

    import sqlite3
stocks = [('2006-01-05', 'BUY', 'RHAT', 100, 35.14),
          ('2006-03-28', 'BUY', 'IBM', 1000, 45.0),
          ('2006-04-06', 'SELL', 'IBM', 500, 53.0),
          ('2006-04-05', 'BUY', 'MSFT', 1000, 72.0)]
conn = sqlite3.connect(":memory:")
conn.execute("create table stocks (date text, buysell text, symb text, amount int, price real)")
conn.executemany("insert into stocks values (?, ?, ?, ?, ?)", stocks)    
cur = conn.cursor()

for row in cur.execute('SELECT * FROM stocks ORDER BY price'):
    print(row)

# Output:
# ('2006-01-05', 'BUY', 'RHAT', 100, 35.14)
# ('2006-03-28', 'BUY', 'IBM', 1000, 45.0)
# ('2006-04-06', 'SELL', 'IBM', 500, 53.0)
# ('2006-04-05', 'BUY', 'MSFT', 1000, 72.0)

  3. getirme()

    Fetches the next row of a query result set, returning a single sequence, or None when no more data is available.

    cur.execute('SELECT * FROM stocks ORDER BY price')
i = cur.fetchone()
while(i): 
    print(i)
    i = cur.fetchone()

# Output:
# ('2006-01-05', 'BUY', 'RHAT', 100, 35.14)
# ('2006-03-28', 'BUY', 'IBM', 1000, 45.0)
# ('2006-04-06', 'SELL', 'IBM', 500, 53.0)
# ('2006-04-05', 'BUY', 'MSFT', 1000, 72.0)

  4. fetchmany(size=cursor.arraysize)

    Fetches the next set of rows of a query result (specified by size), returning a list. If size is omitted, fetchmany returns a single row. An empty list is returned when no more rows are available.

      cur.execute('SELECT * FROM stocks ORDER BY price')
  print(cur.fetchmany(2))

  # Output:    
  # [('2006-01-05', 'BUY', 'RHAT', 100, 35.14), ('2006-03-28', 'BUY', 'IBM', 1000, 45.0)]

  5. fetchall()

    Fetches all (remaining) rows of a query result, returning a list.

    cur.execute('SELECT * FROM stocks ORDER BY price')
print(cur.fetchall())

# Output:
# [('2006-01-05', 'BUY', 'RHAT', 100, 35.14), ('2006-03-28', 'BUY', 'IBM', 1000, 45.0), ('2006-04-06', 'SELL', 'IBM', 500, 53.0), ('2006-04-05', 'BUY', 'MSFT', 1000, 72.0)]

SQLite ve Python veri türleri #

SQLite yerel olarak şu türleri destekler: NULL, INTEGER, REAL, TEXT, BLOB.

SQL’den Python’a veya tam tersine geçerken veri türleri bu şekilde dönüştürülür.

                None     <->     NULL
                int      <->     INTEGER/INT
                float    <->     REAL/FLOAT
                str      <->     TEXT/VARCHAR(n)
                bytes    <->     BLOB

MySQLdb kullanarak MySQL veritabanına erişme #

Yapmanız gereken ilk şey, connect yöntemini kullanarak veritabanına bir bağlantı oluşturmaktır. Bundan sonra, bu bağlantıyla çalışacak bir imlece ihtiyacınız olacak.

Veritabanıyla etkileşim kurmak için imlecin execute yöntemini kullanın ve arada bir, bağlantı nesnesinin taahhüt yöntemini kullanarak değişiklikleri gerçekleştirin.

Her şey bittiğinde, imleci ve bağlantıyı kapatmayı unutmayın.

İşte ihtiyacınız olan her şeyi içeren bir Dbconnect sınıfı.

import MySQLdb

class Dbconnect(object):

    def __init__(self):

        self.dbconection = MySQLdb.connect(host='host_example',
                                           port=int('port_example'),
                                           user='user_example',
                                           passwd='pass_example',
                                           db='schema_example')
        self.dbcursor = self.dbconection.cursor()

    def commit_db(self):
        self.dbconection.commit()

    def close_db(self):
        self.dbcursor.close()
        self.dbconection.close()

Veritabanı ile etkileşim basittir. Nesneyi oluşturduktan sonra, yürütme yöntemini kullanmanız yeterlidir.

db = Dbconnect()
db.dbcursor.execute('SELECT * FROM %s' % 'table_example')

Bir saklı yordamı çağırmak istiyorsanız, aşağıdaki sözdizimini kullanın. Parametre listesinin isteğe bağlı olduğunu unutmayın.

db = Dbconnect()
db.callproc('stored_procedure_name', [parameters] )

Sorgu yapıldıktan sonra sonuçlara birden çok yoldan ulaşabilirsiniz. İmleç nesnesi, tüm sonuçları getirebilen veya döngülenebilen bir oluşturucudur.

results = db.dbcursor.fetchall()
for individual_row in results:
    first_field = individual_row[0]

Doğrudan jeneratörü kullanarak bir döngü istiyorsanız:

for individual_row in db.dbcursor:
    first_field = individual_row[0]

Veritabanında değişiklik yapmak istiyorsanız:

db.commit_db()

İmleci ve bağlantıyı kapatmak istiyorsanız:

db.close_db()

Bağ #

Bağlantı oluşturma

PEP 249’a göre, bir veritabanına bağlantı, bir “Connection” nesnesi döndüren bir “connect()” yapıcısı kullanılarak kurulmalıdır. Bu kurucunun argümanları veritabanına bağlıdır. İlgili argümanlar için veritabanına özel konulara bakın.

import MyDBAPI

con = MyDBAPI.connect(*database_dependent_args)

Bu bağlantı nesnesinin dört yöntemi vardır:

1: kapat

con.close()

Bağlantıyı anında kapatır. Connection.__del___ yöntemi çağrıldığında bağlantının otomatik olarak kapatılacağını unutmayın. Bekleyen tüm işlemler dolaylı olarak geri alınır.

2: taahhüt

con.commit()

Bekleyen herhangi bir işlemi veritabanına taahhüt eder.

3: geri alma

con.rollback()

Bekleyen herhangi bir işlemin başlangıcına geri döner. Başka bir deyişle: bu, veritabanına taahhüt edilmemiş herhangi bir işlemi iptal eder.

4: imleç

cur = con.cursor()

Bir “İmleç” nesnesi döndürür. Veritabanında işlem yapmak için kullanılır.

psycopg2 kullanarak PostgreSQL Veritabanı erişimi #

psycopg2, hem hafif hem de verimli olan en popüler PostgreSQL veritabanı bağdaştırıcısıdır. PostgreSQL bağdaştırıcısının mevcut uygulamasıdır.

Ana özellikleri, Python DB API’sinin eksiksiz uygulanmasıdır. 2.0 spesifikasyonu ve diş güvenliği (birkaç diş aynı bağlantıyı paylaşabilir)

Veritabanına bağlantı kurma ve tablo oluşturma #

import psycopg2

# Establish a connection to the database.
# Replace parameter values with database credentials.
conn = psycopg2.connect(database="testpython", 
                        user="postgres",
                        host="localhost",
                        password="abc123",
                        port="5432") 

# Create a cursor. The cursor allows you to execute database queries. 
cur = conn.cursor()

# Create a table. Initialise the table name, the column names and data type. 
cur.execute("""CREATE TABLE FRUITS (
                    id          INT ,
                    fruit_name  TEXT,
                    color       TEXT,
                    price       REAL
            )""")
conn.commit()
conn.close()

Tabloya veri ekleme: #

# After creating the table as shown above, insert values into it.
cur.execute("""INSERT INTO FRUITS (id, fruit_name, color, price)
               VALUES (1, 'Apples', 'green', 1.00)""")

cur.execute("""INSERT INTO FRUITS (id, fruit_name, color, price)
               VALUES (1, 'Bananas', 'yellow', 0.80)""")

Tablo verilerini alma: #

# Set up a query and execute it 
cur.execute("""SELECT id, fruit_name, color, price 
             FROM fruits""")

# Fetch the data 
rows = cur.fetchall()

# Do stuff with the data
for row in rows:
    print "ID = {} ".format(row[0])
    print "FRUIT NAME = {}".format(row[1])
    print("COLOR = {}".format(row[2]))
    print("PRICE = {}".format(row[3]))

Yukarıdakilerin çıktısı şöyle olacaktır:

ID = 1 
NAME = Apples
COLOR = green
PRICE = 1.0

ID = 2 
NAME = Bananas
COLOR = yellow
PRICE = 0.8

Oracle veritabanı #

Ön koşullar:

  • cx_Oracle paketi - Tüm sürümler için buraya bakın
  • Oracle anlık istemci - Windows x64, Linux x64 için

sudo rpm -i <YOUR_PACKAGE_FILENAME>

    ORACLE_HOME=<PATH_TO_INSTANTCLIENT> PATH=$ORACLE_HOME:$PATH LD_LIBRARY_PATH=<PATH_TO_INSTANTCLIENT>:$LD_LIBRARY_PATH

import cx_Oracle

class OraExec(object):

Bir Oracle Veritabanında ekleme/güncelleme/silme işlemlerini gerçekleştirdiğinizde, değişiklikler yalnızca “taahhüt” verilene kadar oturumunuz içinde kullanılabilir. Güncellenen veriler veritabanına bağlandığında, diğer kullanıcılar ve oturumlar tarafından kullanılabilir hale gelir.

Sorguyu yürütün: Bind değişkenlerini kullanarak INSERT

Referans

Bağlama değişkenleri, deyimi yeniden ayrıştırma ek yükü olmadan deyimleri yeni değerlerle yeniden yürütmenizi sağlar. Bağlama değişkenleri, kodun yeniden kullanılabilirliğini iyileştirir ve SQL Enjeksiyon saldırıları riskini azaltabilir.

rows = [ (1, "First" ),
     (2, "Second" ),
     (3, "Third" ) ]
_db_cur.bindarraysize = 3
_db_cur.setinputsizes(int, 10)
_db_cur.executemany("insert into mytab(id, data) values (:1, :2)", rows)
_db_connection.commit()

Yakın bağlantı:

_db_connection.close()

close() yöntemi bağlantıyı kapatır. Açıkça kapatılmayan tüm bağlantılar, komut dosyası sona erdiğinde otomatik olarak serbest bırakılır.

sqlalchemy’yi kullanma #

Veritabanı için sqlalchemy kullanmak için:

from sqlalchemy import create_engine
from sqlalchemy.engine.url import URL


url = URL(drivername='mysql',
          username='user',
          password='passwd',
          host='host',
          database='db')

engine = create_engine(url)  # sqlalchemy engine

Şimdi bu motor kullanılabilir: Örneğin. veri çerçevelerini doğrudan mysql’den almak için pandalarla

import pandas as pd

con = engine.connect()
dataframe = pd.read_sql(sql=query, con=con)
