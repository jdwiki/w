Karekök: math.sqrt() ve cmath.sqrt #

“Math” modülü, herhangi bir sayının karekökünü hesaplayabilen (bir “kayan"a dönüştürülebilen) “math.sqrt()” işlevini içerir ve sonuç her zaman bir “kayan nokta” olur:

import math

math.sqrt(9)                # 3.0
math.sqrt(11.11)            # 3.3331666624997918
math.sqrt(Decimal('6.25'))  # 2.5

“math.sqrt()” işlevi, sonuç “karmaşık” olursa bir “ValueError” değerini yükseltir:

math.sqrt(-10)

ValueError: matematik etki alanı hatası

“math.sqrt(x)”, “math.pow(x, 0.5)” veya “x ** 0,5"den _daha hızlıdır, ancak sonuçların kesinliği aynıdır. ‘cmath’ modülü, karmaşık sayıları hesaplayabilmesi ve tüm sonuçlarının a + bi biçiminde olması dışında, ‘matematik’ modülüne son derece benzer. Ayrıca .sqrt() kullanabilir:

import cmath

cmath.sqrt(4)  # 2+0j
cmath.sqrt(-4) # 2j

“j” ile ne alakası var? “j”, -1’in kareköküne eşdeğerdir. Tüm sayılar a + bi veya bu durumda a + bj şeklinde yazılabilir. ‘a’, ‘2+0j’deki 2 gibi sayının gerçek kısmıdır. Hayali kısmı olmadığı için, ‘b’ 0’dır. ‘b’, ‘2j’deki 2 gibi, sayının sanal kısmının bir kısmını temsil eder. Bunun reel kısmı olmadığı için 2j 0 + 2j şeklinde de yazılabilir.

Yerleşikleri kullanarak üs alma: ** ve pow() #

[Üsleme] (https://en.wikipedia.org/wiki/Exponentiation) yerleşik “pow” işlevi veya “**” operatörü kullanılarak kullanılabilir:

2 ** 3    # 8
pow(2, 3) # 8

Çoğu (tümü Python 2.x’te) aritmetik işlemler için sonucun türü daha geniş işlenenin türü olacaktır. Bu, ** için doğru değildir; aşağıdaki durumlar bu kuralın istisnalarıdır:

  • Temel: “int”, üs: “int < 0”:

      2 ** -3
  # Out: 0.125 (result is a float)

  • Bu Python 3.x için de geçerlidir.

  • Python 2.2.0’dan önce, bu bir ‘ValueError’ ortaya çıkardı.

  • Temel: ‘int < 0’ veya ‘kayan < 0’, üs: ‘kayan != int’

      (-2) ** (0.5)  # also (-2.) ** (0.5)    
  # Out: (8.659560562354934e-17+1.4142135623730951j) (result is complex)

  • python 3.0.0’dan önce bu bir ‘ValueError’ ortaya çıkardı.

“Operatör” modülü, “**” operatörüne eşdeğer iki işlevi içerir:

import operator
operator.pow(4, 2)      # 16
operator.__pow__(4, 3)  # 64

veya doğrudan ‘pow’ yöntemini çağırabilir:

val1, val2 = 4, 2
val1.__pow__(val2)      # 16
val2.__rpow__(val1)     # 16
# in-place power operation isn't supported by immutable classes like int, float, complex:
# val1.__ipow__(val2)

Modüler üs alma: pow() 3 bağımsız değişkenle #

“pow()“u 3 bağımsız değişkenle “pow(a, b, c)” sağlamak, modüler üstelleştirmeyi ab değerlendirir mod c:

pow(3, 4, 17)   # 13

# equivalent unoptimized expression:
3 ** 4 % 17     # 13

# steps:
3 ** 4          # 81
81 % 17         # 13

Modüler üs kullanan yerleşik türler için yalnızca şu durumlarda mümkündür:

  • İlk argüman bir ‘int’
  • İkinci argüman bir int >= 0
  • Üçüncü argüman bir int != 0

Bu kısıtlamalar python 3.x’te de mevcuttur.

Örneğin, bir modüler ters işlevi tanımlamak için “pow"un 3 bağımsız değişken biçimi kullanılabilir:

def modular_inverse(x, p):
    """Find a such as  a·x ≡ 1 (mod p), assuming p is prime."""
    return pow(x, p-2, p)

[modular_inverse(x, 13) for x in range(1,13)]
# Out: [1, 7, 9, 10, 8, 11, 2, 5, 3, 4, 6, 12]

Matematik modülünü kullanarak üs alma: math.pow() #

“math” modülü başka bir “math.pow()” işlevi içeriyor. Yerleşik “pow()” işlevi veya “**” operatöründen farkı, sonucun her zaman bir “kayan” olmasıdır:

import math
math.pow(2, 2)    # 4.0
math.pow(-2., 2)  # 4.0

Karmaşık girdilere sahip hesaplamaları hariç tutan:

math.pow(2, 2+0j)

TypeError: kompleksi şamandıraya dönüştüremez

ve karmaşık sonuçlara yol açacak hesaplamalar:

math.pow(-2, 0.5)

ValueError: matematik etki alanı hatası

Üstel işlev: math.exp() ve cmath.exp() #

Hem “math” hem de “cmath” modülü Euler numarası: e içerir ve bunu yerleşik “pow()” ile kullanır- function veya ** operatörü çoğunlukla math.exp() gibi çalışır:

import math

math.e ** 2  # 7.3890560989306495
math.exp(2)  # 7.38905609893065

import cmath
cmath.e ** 2 # 7.3890560989306495
cmath.exp(2) # (7.38905609893065+0j)

Ancak sonuç farklıdır ve üstel işlevi doğrudan kullanmak, temel “math.e” ile yerleşik üstelleştirmeden daha güvenilirdir:

print(math.e ** 10)       # 22026.465794806703
print(math.exp(10))       # 22026.465794806718
print(cmath.exp(10).real) # 22026.465794806718
#     difference starts here ---------------^

Üstel işlev eksi 1: math.expm1() #

“math” modülü, “math.e ** x - 1” ifadesini çok küçük “x” için “math.exp(x)” veya “cmath"den daha yüksek bir kesinlikle hesaplayabilen “expm1()” işlevini içerir. .exp(x)` aşağıdakilere izin verir:

import math

print(math.e ** 1e-3 - 1)  # 0.0010005001667083846
print(math.exp(1e-3) - 1)  # 0.0010005001667083846
print(math.expm1(1e-3))    # 0.0010005001667083417
#                            ------------------^

Çok küçük x için fark büyür:

print(math.e ** 1e-15 - 1) # 1.1102230246251565e-15
print(math.exp(1e-15) - 1) # 1.1102230246251565e-15
print(math.expm1(1e-15))   # 1.0000000000000007e-15
#                              ^-------------------

İyileştirme, bilimsel hesaplamada önemlidir. Örneğin, Planck yasası üstel bir işlev eksi 1 içerir:

def planks_law(lambda_, T):
    from scipy.constants import h, k, c  # If no scipy installed hardcode these!
    return 2 * h * c ** 2 / (lambda_ ** 5 * math.expm1(h * c / (lambda_ * k * T)))

def planks_law_naive(lambda_, T):
    from scipy.constants import h, k, c  # If no scipy installed hardcode these!
    return 2 * h * c ** 2 / (lambda_ ** 5 * (math.e ** (h * c / (lambda_ * k * T)) - 1))

planks_law(100, 5000)        # 4.139080074896474e-19
planks_law_naive(100, 5000)  # 4.139080073488451e-19
#                                        ^---------- 

planks_law(1000, 5000)       # 4.139080128493406e-23
planks_law_naive(1000, 5000) # 4.139080233183142e-23
#                                      ^------------

Sihirli yöntemler ve üs: yerleşik, matematik ve cmath #

Tamamen tamsayı değerleri depolayan bir sınıfınız olduğunu varsayalım:

class Integer(object):
    def __init__(self, value):
        self.value = int(value) # Cast to an integer
        
    def __repr__(self):
        return '{cls}({val})'.format(cls=self.__class__.__name__,
                                     val=self.value)
    
    def __pow__(self, other, modulo=None):
        if modulo is None:
            print('Using __pow__')
            return self.__class__(self.value ** other)
        else:
            print('Using __pow__ with modulo')
            return self.__class__(pow(self.value, other, modulo))
    
    def __float__(self):
        print('Using __float__')
        return float(self.value)
    
    def __complex__(self):
        print('Using __complex__')
        return complex(self.value, 0)

Yerleşik “pow” işlevini veya “**” operatörünü kullanmak her zaman “pow” öğesini çağırır:

Integer(2) ** 2                 # Integer(4)
# Prints: Using __pow__
Integer(2) ** 2.5               # Integer(5)
# Prints: Using __pow__
pow(Integer(2), 0.5)            # Integer(1)
# Prints: Using __pow__  
operator.pow(Integer(2), 3)     # Integer(8)
# Prints: Using __pow__
operator.__pow__(Integer(3), 3) # Integer(27)
# Prints: Using __pow__

__pow__() yönteminin ikinci argümanı yalnızca yerleşik-pow() kullanılarak veya doğrudan yöntemi çağırarak sağlanabilir:

pow(Integer(2), 3, 4)           # Integer(0)
# Prints: Using __pow__ with modulo
Integer(2).__pow__(3, 4)        # Integer(0) 
# Prints: Using __pow__ with modulo

“Matematik” işlevleri onu her zaman bir “kayan nokta"ya dönüştürür ve kayan nokta hesaplamasını kullanır:

import math

math.pow(Integer(2), 0.5) # 1.4142135623730951
# Prints: Using __float__

“cmath”-işlevleri onu “karmaşık"a dönüştürmeye çalışır, ancak “karmaşık"a açık bir dönüşüm yoksa “kayan” işlevine de geri dönebilir:

import cmath

cmath.exp(Integer(2))     # (7.38905609893065+0j)
# Prints: Using __complex__

del Integer.__complex__   # Deleting __complex__ method - instances cannot be cast to complex

cmath.exp(Integer(2))     # (7.38905609893065+0j)
# Prints: Using __float__

“float()” yöntemi de eksikse ne “matematik” ne de “cmath” çalışmaz:

del Integer.__float__  # Deleting __complex__ method

math.sqrt(Integer(2))  # also cmath.exp(Integer(2))

TypeError: bir kayan nokta gereklidir

Kökler: kesirli üslü nth-kök #

“math.sqrt” işlevi belirli karekök durumu için sağlanmış olsa da, küp kökleri gibi n’inci kök işlemlerini gerçekleştirmek için kesirli üslerle üs alma operatörünü (**) kullanmak genellikle uygundur.

Bir üs almanın tersi, üssün tersiyle üs almadır. Yani bir sayıyı 3’ün üssüne koyarak küp alıyorsanız, 1/3 üssüne koyarak bir sayının küp kökünü bulabilirsiniz.

>>> x = 3
>>> y = x ** 3
>>> y
27
>>> z = y ** (1.0 / 3)
>>> z
3.0
>>> z == x
True

Büyük tamsayı köklerini hesaplama #

Python doğal olarak büyük tamsayıları desteklese de, Python’da çok büyük sayıların n’inci kökünü almak başarısız olabilir.

x = 2 ** 100
cube = x ** 3
root = cube ** (1.0 / 3)

OverflowError: long int, float’a dönüştürülemeyecek kadar büyük

Bu kadar büyük tamsayılarla uğraşırken, bir sayının n’inci kökünü hesaplamak için özel bir işlev kullanmanız gerekecektir.

def nth_root(x, n):
    # Start with some reasonable bounds around the nth root.
    upper_bound = 1
    while upper_bound ** n <= x:
        upper_bound *= 2
    lower_bound = upper_bound // 2
    # Keep searching for a better result as long as the bounds make sense.
    while lower_bound < upper_bound:
        mid = (lower_bound + upper_bound) // 2
        mid_nth = mid ** n
        if lower_bound < mid and mid_nth < x:
            lower_bound = mid
        elif upper_bound > mid and mid_nth > x:
            upper_bound = mid
        else:
            # Found perfect nth root.
            return mid
    return mid + 1

x = 2 ** 100
cube = x ** 3
root = nth_root(cube, 3)
x == root
# True
