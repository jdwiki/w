Burada Stack Overflow’ta aynı hatalardan bahseden yinelemeler görüyoruz: "ImportError: '??????', SyntaxError: geçersiz sözdizimi' veya NameError: name ‘???’ adlı modül yok tanımlanmadı`.Bu, onları azaltmak ve bağlantı kurulacak bazı belgelere sahip olmak için bir çabadır.

##TürHataları Bu istisnalar, bazı nesnelerin türünün farklı olması gerektiğinde ortaya çıkar.

TypeError: [tanım/yöntem] ? konumsal argümanlar ama ? verilmişti #

Bir işlev veya yöntem, kabul edebileceğinden daha fazla (veya daha az) argümanla çağrıldı.

Örnek #

Daha fazla argüman verilirse:

def foo(a): return a
foo(a,b,c,d) #And a,b,c,d are defined

Daha az argüman verilirse:

def foo(a,b,c,d): return a += b + c + d
foo(a) #And a is defined

Not: Bilinmeyen sayıda argüman kullanmak istiyorsanız, *args veya **kwargs kullanabilirsiniz. Bkz. *args ve **kwargs

TypeError: [işlenen] için desteklenmeyen işlenen türleri: ‘???’ ve ‘???’ #

İşlenene bağlı olarak bazı türler birlikte çalıştırılamaz.

Örnek #

Örneğin: + bitiştirmek ve eklemek için kullanılır, ancak bunların hiçbirini her iki tür için de kullanamazsınız. Örneğin, ‘set1’’ ve ’tuple1’‘i birleştirerek (’+‘ing) bir ‘set’ oluşturmaya çalışmak hata veriyor. Kod:

set1, tuple1 = {1,2}, (3,4)
a = set1 + tuple1

Bazı türler (örneğin: “int” ve “string”) her iki “+“yı da farklı şeyler için kullanır:

b = 400 + 'foo'

Veya hiçbir şey için bile kullanılmayabilirler:

c = ["a","b"] - [1,2]

Ancak örneğin bir “int"e bir “float” ekleyebilirsiniz:

d = 1 + 1.0

TypeError: ‘???’ nesne yinelenebilir/abone olunamaz: #

Bir nesnenin yinelenebilir olması için sıfırdan başlayarak dizinler artık geçerli olmayana ve bir “IndexError” ortaya çıkana kadar sıralı dizinler alabilir (Daha teknik olarak: “iter” döndüren bir “iter” yöntemine sahip olmalıdır veya daha önce bahsedilenleri yapan bir __getitem__ yöntemini tanımlar).

Örnek #

Burada ‘bar’ın 1.’nin sıfırıncı maddesi olduğunu söylüyoruz. Saçmalık:

foo = 1
bar = foo[0]

Bu daha ayrık bir versiyondur: Bu örnekte ‘for’, ‘x’i ’tutar[0]’ olarak ayarlamaya çalışır, yinelenebilirdeki ilk öğedir, ancak olamaz çünkü miktar bir int:

amount = 10
for x in amount: print(x)

TypeError: ‘???’ nesne çağrılabilir değil #

Bir değişken tanımlıyorsunuz ve onu daha sonra çağırıyorsunuz (bir fonksiyon veya yöntemle yaptığınız gibi)

Örnek #

foo = "notAFunction"
foo()

NameError: isim ‘???’ Tanımlanmadı #

Başlatılmamış (en azından daha önce değil) bir değişken, yöntem veya işlev kullanmaya çalıştığınızda ortaya çıkar. Başka bir deyişle, istenen bir yerel veya genel ad bulunamadığında ortaya çıkar. Nesnenin adını yanlış yazmış veya bir şeyi “içe aktarmayı” unutmuş olabilirsiniz. Ayrıca belki başka bir kapsamdadır. Bunları ayrı örneklerle ele alacağız.

Kodun hiçbir yerinde tanımlanmamış #

Özellikle sabitse, başlatmayı unutmuş olabilirsiniz.

foo   # This variable is not defined
bar() # This function is not defined

Belki daha sonra tanımlanmıştır: #

baz()

def baz():
    pass

Ya da “içe aktarılmadı”: #

#needs import math

def sqrt():
    x = float(input("Value: "))
    return math.sqrt(x)

Python kapsamları ve LEGB Kuralı: #

Sözde LEGB Kuralı, Python kapsamlarından bahseder. Adı, muhabir önceliklerine göre sıralanan farklı kapsamlara dayanmaktadır:

Local → Enclosed → Global → Built-in.
  • Local: Global olarak bildirilmeyen veya bir fonksiyona atanmayan değişkenler.
  • Enclosing: Bir fonksiyonda tanımlanan ve başka bir fonksiyonun içine sarılmış değişkenler.
  • Global: Genel olarak bildirilen veya bir dosyanın en üst düzeyinde atanan değişkenler.
  • Built-in: Yerleşik adlar modülünde önceden atanan değişkenler.

Örnek olarak:

for i in range(4):
    d = i * 2
print(d)

’d’ erişilebilir çünkü ‘for’ döngüsü yeni bir kapsamı işaretlemez, ancak öyle olsaydı bir hata alırdık ve davranışı şuna benzer olurdu:

def noaccess():
    for i in range(4):
        d = i * 2
noaccess()
print(d)

Python ‘NameError: ’d’ adı tanımlı değil’ diyor

IndentationErrors (veya girinti SyntaxErrors) #

Diğer dillerin çoğunda girinti zorunlu değildir, ancak Python’da (ve diğer diller: FORTRAN’ın ilk sürümleri, Makefiles, Whitespace (ezoterik dil), vb.) durum böyle değildir, başka bir dilden geliyorsanız ne kafa karıştırıcı olabilir, bir örnekten kendinize kod kopyalıyorsanız veya sadece yeniyseniz.

IndentationError/SyntaxError: beklenmeyen girinti #

Bu istisna, girinti düzeyi sebepsiz olarak arttığında ortaya çıkar.

Örnek #

Burada seviyeyi artırmak için hiçbir neden yok:

 print "This line is ok"
     print "This line isn't ok"
son sürüm 
 print("This line is ok")
     print("This line isn't ok")
son sürüm

Burada iki hata var: sonuncusu ve girintinin herhangi bir girinti düzeyiyle eşleşmemesi. Ancak sadece bir tanesi gösterilir:

 print "This line is ok"
  print "This line isn't ok"
son sürüm 
 print("This line is ok")
  print("This line isn't ok")
son sürüm

IndentationError/SyntaxError: unindent herhangi bir dış girinti düzeyiyle eşleşmiyor #

Görünüşe göre girintiyi tamamen kaldırmamışsınız.

Örnek #

def foo():
    print "This should be part of foo()"
   print "ERROR!"
print "This is not a part of foo()"
son sürüm 
 print("This line is ok")
  print("This line isn't ok")
son sürüm

Girinti Hatası: girintili bir blok bekleniyor #

İki nokta üst üste işaretinden (ve ardından yeni bir satırdan) sonra girinti seviyesinin artması gerekir. Bu hata, bu olmadığında ortaya çıkar.

Örnek #

if ok:
doStuff()

Not: Yalnızca bir “if”, “else”, “except”, “class”, “method” veya “definition” koymak için “pass” (kesinlikle hiçbir şey yapmaz) anahtar kelimesini kullanın, ancak ne olacağını söylemeyin çağrılan/koşul doğruysa gerçekleşir (ancak bunu daha sonra yapın veya “hariç” olması durumunda: hiçbir şey yapmayın):

def foo():
    pass

IndentationError: girintide sekmelerin ve boşlukların tutarsız kullanımı #

Örnek #

def foo():
    if ok:
      return "Two != Four != Tab"
        return "i dont care i do whatever i want"

Bu hata nasıl önlenir #

Sekmeleri kullanmayın. Python için stil kılavuzu olan ‘PEP8’ tarafından önerilmez.

  1. Düzenleyicinizi girinti için 4 boşluk kullanacak şekilde ayarlayın.
  2. Bir arama yapın ve tüm sekmeleri 4 boşlukla değiştirmek için değiştirin.
  3. Düzenleyicinizin, bu hatayı kolayca fark edip düzeltebilmeniz için sekmeleri 8 boşluk olarak görüntülemeye ayarlandığından emin olun.

Daha fazla bilgi edinmek istiyorsanız bu soruya bakın.

Diğer Hatalar #

AssertError #

“Assert” ifadesi hemen hemen her programlama dilinde bulunur. Bunu yaptığınızda:

assert condition

veya:

assert condition, message

Buna eşdeğerdir:

if __debug__:
    if not condition: raise AssertionError(message)

İddialar isteğe bağlı bir mesaj içerebilir ve hata ayıklamayı tamamladığınızda bunları devre dışı bırakabilirsiniz.

Not: debug yerleşik değişkeni normal koşullar altında True, optimizasyon istendiğinde False (komut satırı seçeneği -O). debug atamaları yasa dışıdır. Yerleşik değişkenin değeri, yorumlayıcı başladığında belirlenir.

Klavye Kesintisi #

Kullanıcı kesme tuşuna bastığında ortaya çıkan hata, normalde Ctrl + C veya del.

ZeroDivisionError #

Tanımsız olan ‘1/0’ı hesaplamaya çalıştınız. Bir sayının bölenlerini bulmak için bu örneğe bakın:

div = float(raw_input("Divisors of: "))
for x in xrange(div+1): #includes the number itself and zero
    if div/x == div//x:
        print x, "is a divisor of", div
son sürüm 
div = int(input("Divisors of: "))
for x in range(div+1): #includes the number itself and zero
    if div/x == div//x:
        print(x, "is a divisor of", div)
son sürüm

“For” döngüsü bu değeri “x"e atadığı için “ZeroDivisionError” değerini yükseltir. Bunun yerine şöyle olmalıdır:

div = float(raw_input("Divisors of: "))
for x in xrange(1,div+1): #includes the number itself but not zero
    if div/x == div//x:
        print x, "is a divisor of", div
son sürüm 
div = int(input("Divisors of: "))
for x in range(1,div+1): #includes the number itself but not zero
    if div/x == div//x:
        print(x, "is a divisor of", div)
son sürüm

İyi kodda Sözdizimi Hatası #

İlginç olmayan bir satıra işaret eden bir SyntaxError’ın büyük çoğunluğu, ondan önceki satırda bir sorun olduğu anlamına gelir (bu örnekte, eksik bir parantezdir):

def my_print():
    x = (1 + 1
    print(x)

İadeler

  File "<input>", line 3
    print(x)
        ^
SyntaxError: invalid syntax

Bu sorunun en yaygın nedeni, örnekte gösterildiği gibi eşleşmeyen parantezler/parantezlerdir.

Python 3’te print ifadeleri için önemli bir uyarı vardır:

 >>> print "hello world"
  File "<stdin>", line 1
    print "hello world"
                      ^
SyntaxError: invalid syntax

print ifadesi print() işleviyle değiştirildiği için, yani:

print("hello world")  # Note this is valid for both Py2 & Py3
