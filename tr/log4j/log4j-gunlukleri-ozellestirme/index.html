Yapılandırma dosyası #

Yapılandırma #

Log4j yapılandırma dosyası şu formatlardan herhangi birinde olabilir:

-JSON -YAML

  • özellikler (metin dosyası)
  • XML

Yapılandırma keşfi #

  1. Log4j, “log4j.configurationFile” sistem özelliğini inceleyecek ve ayarlanmışsa, yapılandırmayı yüklemeye çalışacaktır.
  2. Herhangi bir sistem özelliği ayarlanmamışsa, log4j sınıf yolunda `log4j2-test.properties’ arayacaktır.
  3. Böyle bir dosya bulunamazsa, log4j sınıf yolunda ’log4j2-test.yaml’ veya ’log4j2-test.yml’ arar.
  4. Böyle bir dosya bulunamazsa, log4j sınıf yolunda ’log4j2-test.json’ veya ’log4j2-test.jsn’ arar.
  5. Böyle bir dosya bulunamazsa, logj4, sınıf yolunda log4j2-test.xml dosyasını arayacaktır.
  6. Bir test dosyası bulunamazsa, log4j sınıf yolunda ’log4j2.properties’i arar.
  7. Bir özellikler dosyası bulunamazsa, log4j sınıf yolunda ’log4j2.yaml’ veya ’log4j2.yml’ arar.
  8. Bir YAML dosyası bulunamazsa, log4j sınıf yolunda ’log4j2.json’ veya ’log4j2.jsn’ arar.
  9. Bir JSON dosyası bulunamazsa, log4j sınıf yolunda ’log4j2.xml’ dosyasını bulmaya çalışır.
  10. Herhangi bir konfigürasyon dosyası bulunamazsa, DefaultConfiguration kullanılacaktır. Bu, günlük çıktısının konsola gitmesine neden olur.

XML Yapılandırması #

XML Örneği #

Aşağıdaki yapılandırma, iki ekleyiciyi yapılandırır (günlük çıkışı). İlki standart sistem çıktısına (konsol) ve diğer günlükler dosyaya kaydedilir. Bu örnekte, dosyanın konumu yapılandırmada statik olarak (ekleyici dosyası) veya maven filtreleme özelliği (<Özellik adı="APPENDER">) aracılığıyla dinamik olarak ayarlanabilir. Dosyadaki günlükler gün geçtikçe paketlenecektir. Günlük satırı biçimi ‘Dönüşüm Modeli’ bir değişken olarak ayarlanır.

<?xml version="1.0" encoding="UTF-8"?>
<!-- 'status' sets log level for parsing configuration file itself -->
<Configuration status="INFO">
<Properties>
    <!-- Sets variable PID, if it's not present in log4j context will take value as below: -->
    <Property name="PID">????</Property>
    <!-- Sets variable 'LOG_PATTERN', defining how log line will look like -->
    <Property name="LOG_PATTERN">%clr{%d{yyyy-MM-dd HH:mm:ss.SSS}}{faint} %clr{%5p} %clr{${sys:PID}}{magenta} %clr{%X{usr}}{green} %clr{---}{faint}%clr{[%15.15t]}{faint} %clr{%-40.40c{1.}:%l}{cyan} %clr{:}{faint} %m%n%wEx</Property>
    <!-- LOG_DIR may be set by maven filtering feature: -->
    <Property name="LOG_DIR">@logging.path@</Property>
    <!-- APPENDER may be set by maven filtering feature, to set 'console' or 'file' appender: -->
    <Property name="APPENDER">@logging.default.appender@</Property>
</Properties>
<Appenders>
    <!-- Sets console output: -->
    <Console name="console" target="SYSTEM_OUT">
        <PatternLayout pattern="${LOG_PATTERN}"/>
    </Console>
    <!-- Sets output to file, and names it 'file' -->
    <RollingRandomAccessFile append="true" fileName="${LOG_DIR}/log4j2.log"
        filePattern = "${LOG_DIR}/log4j2.%d{yyyy-MM-dd}.nr%i.log.gz" name="file">
        <PatternLayout>
            <Pattern>${LOG_PATTERN}</Pattern>
        </PatternLayout>
        <Policies>
            <TimeBasedTriggeringPolicy />
        </Policies>
    </RollingRandomAccessFile>
</Appenders>
<Loggers>
    <!-- Sets debug for Class 'com.example.package.Clazz', and attaches to file 'file' -->
    <Logger name="com.example.package.Clazz" level="debug">
        <AppenderRef ref="file"/>
    </Logger>
    <!-- Sets 'info' for package 'org.springframework' -->
    <Logger name="org.springframework" level="info" />
    <Root level="WARN">
        <AppenderRef ref="${APPENDER}"/>
    </Root>
</Loggers>
</Configuration>
