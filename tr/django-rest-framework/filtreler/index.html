Basitten Karmaşık olanlara Filtreleme Örnekleri #

Sade Vanilya Filtreleme #

Herhangi bir görünümü filtrelemek için, filtrelenmiş bir sorgu kümesi döndürmek için “get_queryset” yöntemini geçersiz kılın

class HREmployees(generics.ListAPIView):
    def get_queryset(self):
        return Employee.objects.filter(department="Human Resources")

Tüm API işlevleri, işlemler için filtrelenmiş sorgu kümesini kullanır. Örneğin, yukarıdaki görünümün listesi yalnızca departmanı İnsan Kaynakları olan çalışanları içerecektir.

get_queryset’te sorgu parametrelerine erişme #

İstek parametrelerine göre filtreleme yapmak kolaydır.

“self.request”, “self.args” ve “self.kwargs” mevcuttur ve mevcut isteğe ve filtreleme için parametrelerine işaret eder

def DepartmentEmployees(generics.ListAPIView):
    def get_queryset(self):
        return Employee.objects.filter(department=self.kwargs["department"])

Neyin filtreleneceğine API parametrelerinin karar vermesine izin verme #

Daha fazla esneklik istiyorsanız ve API çağrısının görünümü filtrelemek için parametrelerde geçmesine izin vermek istiyorsanız, Django Request Framework (pip ile yüklenir) gibi filtre arka uçlarını ekleyebilirsiniz.

from rest_framework import filters  

class Employees(generics.ListAPIView):
    queryset=Employee.objects.all()
    filter_backends = (filters.DjangoFilterBackend,)
    filter_fields = ("department", "role",)

Artık /api/employees?department=İnsan Kaynakları API araması yapabilirsiniz ve yalnızca İK departmanına ait çalışanların bir listesini alırsınız veya /api/employees?role=manager&department=İnsan Kaynakları için sadece İK departmanındaki yöneticileri alın.

Sorgu kümesi filtrelemesini Django Filter Backend ile sorunsuz bir şekilde birleştirebilirsiniz. Filtreler, “get_queryset” tarafından döndürülen filtrelenmiş sorgu kümesi üzerinde çalışır.

from rest_framework import filters  

class HREmployees(generics.ListAPIView):
    filter_backends = (filters.DjangoFilterBackend,)
    filter_fields = ("department", "role",)

    def get_queryset(self):
        return Employee.objects.filter(department="Human Resources")

Filtre Kümeleri #

Şimdiye kadar, yukarıdaki durumlarda basit tip eşleşmeleri ile idare edebilirsiniz.

Peki ya yaşları 25 ile 32 arasında olan İK çalışanlarının listesi gibi daha karmaşık bir şey istiyorsanız?

Sorunun cevabı: Filtre kümeleri

Filtre kümeleri, modelin çeşitli alanlarının nasıl filtreleneceğini tanımlayan sınıflardır.

Onları böyle tanımla

class EmployeeFilter(django_filters.rest_framework.FilterSet):
    min_age = filters.django_filters.NumberFilter(name="age", lookup_expr='gte')
    max_age = filters.django_filters.NumberFilter(name="price", lookup_expr='lte')     

    class Meta:
        model = Employee
        fields = ['age', 'department']

’name’ filtrelemek istediğiniz alanı gösterir

“lookup_expr” temel olarak sorgu kümelerini filtrelerken kullandığınız adların aynısını ifade eder, örneğin “Employee.objects.filter(department__startswith=“Human) ile eşdeğer olan “lookup_expr=“startswith”” öğesini kullanarak bir “ile başlar” eşleşmesi yapabilirsiniz. “)`

Ardından bunları ‘filter_fields’ yerine ‘filter_class’ kullanarak görünüm sınıflarınızda kullanın.

class Employees(generics.ListAPIView):
    queryset=Employee.objects.all()
    filter_backends = (filters.DjangoFilterBackend,)
    filter_class = EmployeeFilter

Artık /api/employees?department=İnsan Kaynakları&min_age=25&max_age=32 yapabilirsiniz.

Kesin olmayan eşleşmeler ve ilişkiler #

Filtre sınıfları ve ifadeleri, sorgu kümelerinde filtrelemeyi nasıl belirttiğinize çok benzer

İlişkilerdeki alanları filtrelemek için “__” gösterimini kullanabilirsiniz, Örneğin, departman çalışandan gelen bir yabancı anahtar ise, ekleyebilirsiniz.

filter_fields=("department__name",)

ve sonra /api/employees?department__name=İnsan Kaynakları yapabilirsiniz.

Veya daha zarif bir şekilde, bir filtre seti oluşturabilir, dept adında bir filtre değişkeni ekleyebilir ve adını department__name olarak ayarlayarak /api/employees?dept=İnsan Kaynakları yapabilirsiniz.
