Global olarak kimlik doğrulamayı ayarlama #

Django REST Framework’te kimlik doğrulama, varsayılan çerçeve yapılandırmalarının geri kalanı gibi, genel olarak settings.py’deki ‘REST_FRAMEWORK’ değişkeninin bir alt anahtarı olarak yapılandırılabilir.

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework.authentication.BasicAuthentication',
        'rest_framework.authentication.SessionAuthentication',
    )
}

Belirli bir APIView için kimlik doğrulamayı ayarlama #

Kimlik doğrulama, belirli bir APIView uç noktası için “authentication_classes” değişkeni kullanılarak ayarlanabilir:

from rest_framework.authentication import SessionAuthentication, BasicAuthentication
from rest_framework.permissions import IsAuthenticated
from rest_framework.response import Response
from rest_framework.views import APIView

class ExampleView(APIView):
    authentication_classes = (SessionAuthentication, BasicAuthentication)
    permission_classes = (IsAuthenticated,)

    def get(self, request):
        content = {
            'user': unicode(request.user),  # `django.contrib.auth.User` instance.
            'auth': unicode(request.auth),  # None
        }
        return Response(content)

Temel belirteç tabanlı kimlik doğrulamayı kullanma #

Django REST Framework, kullanılabilir olmadan önce Django’da bir uygulama olarak yapılandırılması gereken temel bir belirteç tabanlı kimlik doğrulama mekanizması sağlar, böylece belirteçler veritabanında oluşturulur ve yaşam döngüleri işlenir.

settings.py’ye Token tabanlı kimlik doğrulaması ekleyin #

INSTALLED_APPS = (
    ...
    'rest_framework.authtoken'
)

Veritabanı geçişini çalıştırın #

./manage.py migrate

Kullanıcılarınız için belirteçler oluşturun #

Her nasılsa, bir jeton oluşturmanız ve onu iade etmeniz gerekecek:

def some_api(request):
    token = Token.objects.create(user=request.user)
    return Response({'token': token.key})

Belirteç uygulamasında zaten bir API uç noktası var, böylece urls.py’nize aşağıdakileri kolayca ekleyebilirsiniz:

urls.py #

from rest_framework.authtoken import views
urlpatterns += [
    url(r'^auth-token/', views.obtain_auth_token)
]

İstemciler artık kimlik doğrulaması yapabilir #

Aşağıdaki gibi bir “Yetkilendirme” başlığı kullanma:

Authorization: Token 123456789

Hazır bir “Belirteç” ön eki ve boşluktan sonra belirtecin kendisi.

Değişmez, ‘TokenAuthentication’ alt sınıfları oluşturularak ve ‘anahtar kelime’ sınıf değişkeni değiştirilerek değiştirilebilir.

Kimliği doğrulanırsa, “request.auth”, “rest_framework.authtoken.models.BasicToken” örneğini içerecektir.

OAuth2 kimlik doğrulamasını ayarlama #

OAuth, Django REST Framework tarafından işlenmez, ancak bir OAuth istemcisini uygulayan birkaç pip modülü vardır. REST Framework belgeleri aşağıdaki modüllerden birini önerir:

  • Django OAuth Araç Takımı
  • Django REST Framework OAuth

Django OAuth Araç Seti #

pip install django-oauth-toolkit

settings.py #

INSTALLED_APPS = (
    ...
    'oauth2_provider',
)

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'oauth2_provider.ext.rest_framework.OAuth2Authentication',
    )
}

urls.py #

urlpatterns = patterns(
    ...
    url(r'^o/', include('oauth2_provider.urls', namespace='oauth2_provider')),
)

Django REST Çerçevesi OAuth #

pip install djangorestframework-oauth django-oauth2-provider

settings.py #

INSTALLED_APPS = (
    ...
    'provider',
    'provider.oauth2',
)

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework.authentication.OAuth2Authentication',
    )
}

urls.py #

urlpatterns = patterns(
    ...
    url(r'^oauth2/', include('provider.oauth2.urls', namespace='oauth2')),
)

Yönetici #

Yönetici paneline gidin ve bir “client_id” ve bir “client_secret” sahibi olmak için yeni bir “Provider.Client” oluşturun.

Birkaç istemci belirteci ile daha iyi Belirteç tabanlı yetkilendirme kullanma #

Gerçek belirteçleri yönetmek için en ilginç paket, kullanıcı başına birden çok belirteci destekleyen (ve her bir belirteci bağımsız olarak iptal eden) django-rest-knox paketidir. belirteç süresinin dolması ve diğer birkaç güvenlik mekanizması için desteğe sahip olmak.

“Django-rest-knox”, “kriptografiye” bağlıdır. Nasıl kurulacağı hakkında daha fazla bilgiyi şu adreste bulabilirsiniz: http://james1345.github.io/django-rest-knox/installation/

knox’u yükleme #

pip install django-rest-knox

settings.py #

INSTALLED_APPS = (
  ...
  'rest_framework',
  'knox',
  ...
)

Taşımaları uygula:

./manage.py migrate
