Sayfalandırma, büyük veri kümelerinin ayrılmış ve özerk sayfalara bölünmesidir.

Django dinlenme çerçevesinde, sayfalandırma, kullanıcının her sayfadaki veri miktarını ve bölmenin uygulandığı stili değiştirmesine olanak tanır.

[Giriş] Sayfalandırma Stilini Global Olarak Ayarlayın #

Tüm proje için sayfalama stilini ayarlamak için proje ayarlarında DEFAULT_PAGINATION_CLASS ve PAGE_SIZE ayarlamanız gerekir.

Bunu yapmak için “settings.py"ye gidin ve “REST_FRAMEWORK” değişkenine aşağıdakini ekleyin:

REST_FRAMEWORK = {
    'DEFAULT_PAGINATION_CLASS': 
        'rest_framework.pagination.DESIRED_PAGINATION_STYLE',
    'PAGE_SIZE': 100
}

“DESIRED_PAGINATION_STYLE” yerine aşağıdakilerden biri yerleştirilmelidir:

  • PageNumberPagination: İstek sorgu parametrelerinde tek bir sayfa numarasını kabul eder.

     http://your_api_url/a_table/?page=2

  • LimitOffsetPagination: Döndürülecek maksimum öğe sayısını belirten bir limit parametresini ve veri kümesine göre sorgunun başlangıç ​​konumunu belirten bir offset parametresini kabul eder. Bu stil için “PAGE_SIZE” ayarlanması gerekmez.

     http://your_api_url/a_table/?limit=50&offset=100

  • CursorPagination: İmleç tabanlı sayfalandırma, yukarıdaki stillerden daha karmaşıktır. Veri kümesinin sabit bir sıralama sunmasını gerektirir ve istemcinin veri kümesinin keyfi konumlarına gitmesine izin vermez.

  • Yukarıdakilerin yerine özel sayfalandırma stilleri tanımlanabilir.

[Gelişmiş] Genel Olmayan Görünümlerde/Görünüm Kümelerinde Sayfalandırma #

Bu ileri düzey bir konudur, bu sayfadaki diğer örnekleri anlamadan denemeye kalkışmayın.

Resmi Django Rest Framework on pagination‘de belirtildiği gibi:

Sayfalandırma, yalnızca genel görünümleri veya görünüm kümelerini kullanıyorsanız otomatik olarak gerçekleştirilir. Normal bir APIView kullanıyorsanız, sayfalandırılmış bir yanıt döndürdüğünüzden emin olmak için sayfalandırma API’sini kendiniz aramanız gerekir. Örnek için mixins.ListModelMixin ve generals.GenericAPIView sınıflarının kaynak koduna bakın.

Ancak, genel olmayan bir görünüm/görünüm kümesinde sayfalandırma kullanmak istersek ne olur?

Haydi tavşan deliğinden aşağı inelim:

  1. İlk durak, resmi Django Rest Framework deposu ve özellikle Django-rest-framework/rest_framework/generics.py. Bu bağlantının işaret ettiği belirli satır, bize çerçeve geliştiricilerinin kendi jeneriklerinde sayfalandırma ile nasıl başa çıktıklarını gösterir.
    Bizim görüşümüz için de tam olarak bunu kullanacağız!

  2. [Bu sayfanın giriş örneğinde][3] gösterilen gibi global bir sayfalandırma kurulumumuz olduğunu varsayalım ve sayfalandırma uygulamak istediğimiz bir ‘APIView’ımız olduğunu da varsayalım.

  3. Ardından “views.py” üzerinde:

     from django.conf import settings
 from rest_framework.views import APIView

 class MyView(APIView):
     queryset = OurModel.objects.all()
     serializer_class = OurModelSerializer
     pagination_class = settings.DEFAULT_PAGINATION_CLASS # cool trick right? :)

     # We need to override get method to achieve pagination
     def get(self, request):
         ...
         page = self.paginate_queryset(self.queryset)
         if page is not None:
             serializer = self.serializer_class(page, many=True)
             return self.get_paginated_response(serializer.data)

         ... Do other stuff needed (out of scope of pagination)

     # Now add the pagination handlers taken from 
     #  django-rest-framework/rest_framework/generics.py

     @property
     def paginator(self):
         """
         The paginator instance associated with the view, or `None`.
         """
          if not hasattr(self, '_paginator'):
              if self.pagination_class is None:
                  self._paginator = None
              else:
                  self._paginator = self.pagination_class()
          return self._paginator

      def paginate_queryset(self, queryset):
          """
          Return a single page of results, or `None` if pagination is disabled.
          """
          if self.paginator is None:
              return None
          return self.paginator.paginate_queryset(queryset, self.request, view=self)

      def get_paginated_response(self, data):
          """
          Return a paginated style `Response` object for the given output data.
          """
          assert self.paginator is not None
          return self.paginator.get_paginated_response(data)

Artık sayfalandırmayı yöneten bir ‘APIView’umuz var!

[Orta] Sayfalandırma stilini geçersiz kıl ve sınıf başına Sayfalandırmayı ayarla #

Sayfalandırma Stilini Geçersiz Kıl:

Kullanılabilir her sayfalandırma stili, mevcut stillerden birinden miras alan ve ardından parametrelerini değiştiren yeni bir sınıf oluşturularak geçersiz kılınabilir:

class MyPagination(PageNumberPagination):
    page_size = 20
    page_size_query_param = 'page_size' 
    max_page_size = 200
    last_page_strings = ('the_end',)

Bu parametreler (sayfalandırma resmi belgelerinde listelendiği gibi) şunlardır:

Sayfa NumarasıSayfalandırma

  • page_size: Sayfa boyutunu belirten sayısal bir değer. Ayarlanırsa, bu “PAGE_SIZE” ayarını geçersiz kılar. Varsayılan olarak “PAGE_SIZE” ayar anahtarıyla aynı değere ayarlanır.
  • page_query_param: Sayfalandırma denetimi için kullanılacak sorgu parametresinin adını gösteren bir dize değeri.
  • page_size_query_param: Ayarlanırsa, bu, istemcinin istek başına sayfa boyutunu ayarlamasına izin veren bir sorgu parametresinin adını gösteren bir dize değeridir. Varsayılan olarak, istemcinin istenen sayfa boyutunu kontrol edemeyebileceğini belirten “Yok” olur.
  • max_page_size: Ayarlanırsa, bu, izin verilen maksimum istenen sayfa boyutunu gösteren sayısal bir değerdir. Bu öznitelik yalnızca “page_size_query_param” da ayarlanmışsa geçerlidir.
  • ’last_page_strings’: Kümedeki son sayfayı istemek için ‘page_query_param’ ile kullanılabilecek değerleri gösteren dize değerleri listesi veya demeti. Varsayılan olarak ('son',)
  • “şablon”: Göz atılabilir API’de sayfalandırma kontrollerini işlerken kullanılacak bir şablonun adı. Oluşturma stilini değiştirmek için geçersiz kılınabilir veya HTML sayfalandırma denetimlerini tamamen devre dışı bırakmak için “Yok” olarak ayarlanabilir. Varsayılan olarak "rest_framework/pagination/numbers.html" şeklindedir.

LimitOffsetPagination

  • default_limit: İstemci tarafından bir sorgu parametresinde sağlanmadığında kullanılacak sınırı gösteren sayısal bir değer. Varsayılan olarak “PAGE_SIZE” ayar anahtarıyla aynı değere ayarlanır.
  • limit_query_param: “limit” sorgu parametresinin adını gösteren bir dize değeri. Varsayılan “sınır”‘dır.
  • offset_query_param: “Offset” sorgu parametresinin adını gösteren bir dize değeri. Varsayılan “ofset”’tir.
  • max_limit: Eğer ayarlanırsa, bu, müşteri tarafından talep edilebilecek izin verilen maksimum limiti gösteren sayısal bir değerdir. Varsayılan olarak “Hiçbiri"dir.
  • şablon: PageNumberPagination ile aynı.

İmleç Sayfalandırma

  • page_size: PageNumberPagination ile aynı.
  • cursor_query_param: “imleç” sorgu parametresinin adını gösteren bir dize değeri. Varsayılan “imleç” şeklindedir.
  • “sıralama”: Bu, imleç tabanlı sayfalandırmanın uygulanacağı alanı belirten bir dize veya dize listesi olmalıdır. Örneğin: `sipariş = ‘sümüklü böcek’’. Varsayılan olarak “-created” şeklindedir. Bu değer, görünümde ‘OrderingFilter’ kullanılarak da geçersiz kılınabilir.
  • şablon: PageNumberPagination ile aynı.

Sınıf Başına Sayfalandırma Kurulumu:

Sayfalandırma stilini global olarak ayarlama yeteneğine ek olarak, sınıf başına bir kurulum da mevcuttur:

class MyViewSet(viewsets.GenericViewSet):
    pagination_class = LimitOffsetPagination

Artık yalnızca “MyViewSet” bir “LimitOffsetPagination” sayfalamasına sahip.

Özel bir sayfalandırma stili aynı şekilde kullanılabilir:

class MyViewSet(viewsets.GenericViewSet):
    pagination_class = MyPagination

[Orta] Karmaşık Kullanım Örneği #

Pek çok genel görünüm ve bazı genel görünüm kümeleri içeren karmaşık bir API’miz olduğunu varsayalım. Özelleştirilmiş bir “LimitOffsetPagination” durumu istediğimiz biri (genel görünüm veya görünüm kümesi, fark yaratmaz) dışında her görünümde “PageNumberPagination"ı etkinleştirmek istiyoruz.

Bunu başarmak için yapmamız gerekenler:

  1. Her genel görünüm/görünüm kümesi için etkinleştirmek üzere varsayılan sayfalandırmamızı settings.py üzerine yerleştireceğiz ve PAGE_SIZE değerini 50 öğeye ayarlayacağız:

     REST_FRAMEWORK = {
     'DEFAULT_PAGINATION_CLASS': 
         'rest_framework.pagination.PageNumberPagination',
     'PAGE_SIZE': 50
 }

  2. Şimdi “views.py” dosyamızda (veya başka bir “.py” örneğinde: “paginations.py”), “LimitOffsetPagination"ı geçersiz kılmamız gerekiyor:

     from rest_framework.pagination import LimitOffsetPagination

 class MyOffsetPagination(LimitOffsetPagination):
     default_limit = 20
     max_limit = 1000

    A custom LimitOffsetPagination with PAGE_ZISE of 20 items and maximum limit of 1000 items.

  3. views.py içinde, özel görünümümüzün `pagination_class’ını tanımlamamız gerekiyor:

     imports ...

 # ===================================
 #    PageNumberPagination classes
 # ===================================

 class FirstView(generics.ListAPIView):
     ...

 class FirstViewSet(viewsets.GenericViewSet):
     ...

 ...

 # ===================================
 #     Our custom Pagination class
 # ===================================

 class IAmSpecialView(generics.ListAPIView):
     pagination_class = MyOffsetPagination
     ...

Artık uygulamadaki her genel görünüm/görünüm kümesi, gerçekten özel olan ve kendi özelleştirilmiş “LimitOffsetPagination"a sahip “IAmSpecial” sınıfı dışında “PageNumberPagination"a sahiptir.

[Orta] İşlev tabanlı görünümde sayfalandırma #

Bu örneklerde (ex_1, ex_2) herhangi bir genel sınıf temel görünümünde sayfalandırma sınıflarının nasıl kullanılacağını ve geçersiz kılınacağını gördük.
İşlev tabanlı bir görünümde sayfalamayı kullanmak istediğimizde ne olur?

Ayrıca, “Model” için “PageNumberPagination” ile yalnızca bir “GET” isteğine yanıt veren işlev tabanlı bir görünüm oluşturmak istediğimizi varsayalım. O zamanlar:

from rest_framework.pagination import PageNumberPagination


@api_view(['GET',])
def my_function_based_list_view(request):
    paginator = PageNumberPagination()
    query_set = MyModel.objects.all()
    context = paginator.paginate_queryset(query_set, request)
    serializer = MyModelSerializer(context, many=True)
    return paginator.get_paginated_response(serializer.data)
Bu satırı değiştirerek yukarıdakileri özel bir sayfalandırma için de yapabiliriz: 
paginator = PageNumberPagination()

buna

paginator = MyCustomPagination()

MyCustomPagination bazı varsayılan sayfalandırmayı geçersiz kılmak için tanımlamamız şartıyla
