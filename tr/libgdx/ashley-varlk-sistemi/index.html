Ashley Entity System, LibGDX organizasyonu altında yönetilen ve oyun geliştirme için çok uygun olan bir Entity System kitaplığıdır. LibGDX yardımcı program sınıflarına bağlıdır, ancak bazı çalışmalarla LibGDX’e dayalı olmayan diğer Java oyun çerçeveleriyle birlikte kullanılabilir.

Varlık sistemleri, nesne sınıflarını kalıtım açısından zengin hale getirmek zorunda kalmadan büyük nesne kümelerine yönelik verileri ve işlevleri yönetmek için farklı bir yol sağlar.

Ashley’i kullanmak, Unity’nin sağladığı gibi, ancak oyun motoru yerine bir çerçeve kapsamında bir nesne modelleme yaklaşımı arayanlar için yararlı bir yaklaşım olabilir.

Bileşen Oluşturma #

Bileşenler, yalnızca Ashley bileşen sınıfını uygulayan örneklerdir.

import com.badlogic.ashley.core.Component;
import com.badlogic.ashley.core.ComponentMapper;

public class Position implements Component {
    public static final ComponentMapper<Position> Map = 
                                 ComponentMapper.getFor(Position.class);

    public float x = 0f, 
                 y = 0f;
}

Bileşen haritaları, varlıklar üzerindeki bileşenlere erişmenin hızlı bir yolunu sağlar. Bileşen haritalarınızı yönetmenin iki yaygın yolu, bileşeninizin sınıfında statik bir örnek tutmak veya tüm bileşenleriniz için tüm eşleyicileri içeren bir sınıf/enum’a sahip olmaktır.

Uygulamanızda bir bileşen türü için birden fazla eşleyici bildirmenize gerek yoktur.

Varlık Sistemi Oluşturma #

Varlık sistemleri, varlık kümeleri üzerinde işlevsel işlemleri nasıl gerçekleştirdiğinizdir. Bileşenler, genellikle, bir varlık sisteminin işi olduğu için, veri farkındalığını veya diğer bileşen örneklerinin durumunu içeren kendileriyle ilişkili bir mantığa sahip olmamalıdır. Bir varlık sistemi aynı anda birden fazla motora kaydedilemez.

Varlık sistemleri birden fazla türde işlev gerçekleştirmemelidir. Bir Hareket Sistemi yalnızca konumlandırma ve benzerlerini işlemeli, RenderSystem gibi bir şey ise varlıkların çizimini yapmalıdır.

Bazen EntityListeners’da bulunana benzer ek işlevlerle EntitySystems’inizi genişletmek yararlı olur, böylece her döngüde motordaki tüm varlıklar üzerinde yineleme yapmak yerine yalnızca üzerinde çalışmak istediğiniz varlık türlerini takip edersiniz. EntityListeners, sistemin kayıtlı olduğu aynı motora bir varlık eklendiğinde tetiklenir.

import com.badlogic.ashley.core.EntityListener;
import com.badlogic.gdx.utils.Array;

public class MovementSystem extends EntitySystem implements EntityListener {
    Array<Entity> moveables = new Array<>();
    ...

    @Override
    public void entityAdded(Entity entity) {
        if (FAMILY.matches(entity)) {
            moveables.add(entity);
        }
    }

    @Override
    public void entityRemoved(Entity entity) {
        if (FAMILY.matches(entity)) {
            moveables.removeValue(entity, true);
        }
    }

    public void update (float deltaTime) {
        for (Entity e : this.moveables) {
            Position pos = Position.Map.get(e);
            
            // do entity movement logic on component
            ...
        }
    }
}

Sistemin her zaman işlediği bir varlık alt kümesini takip etmek de optimal olabilir, çünkü belirli bir varlığı, ilişkili bileşeni kaldırmak zorunda kalmadan veya istenirse bunları bir bütün olarak motordan kaldırmadan o sistemin işlemesinden kaldırabilirsiniz.

Sıralanmış Varlık Sistemi Oluşturma #

Belirli bir ailenin her bir varlığını bir ‘karşılaştırıcı’ tarafından belirtilen sırada işleyen ve ‘EntitySystem’ her güncellendiğinde her varlık için ‘processEntity()‘yi çağıran basit bir ‘EntitySystem’. Bu gerçekten sadece bir kolaylık sınıfıdır, çünkü işleme sistemleri bir varlık listesi üzerinde sıralı bir şekilde yineleme eğilimindedir. Varlık eklemek, varlık listesinin başvurulmasına neden olur. Sıralama kriterlerinizi değiştirdiyseniz, forceSort() çağırın. Daha fazla bilgi için lütfen SortedIteratingSystem bölümüne bakın.

Aşağıdaki kod örneğinde, bunun için en iyi kullanım, sprite’larınızın zindex’e göre sıralanmış bir sırada oluşturulmasıdır.

public class SpriteComponent implements Component {
     public TextureRegion region;
     public int z = 0;
}

public class Mapper {
     public static ComponentMapper<SpriteComponent> sprite = ComponentMapper.getFor(SpriteComponent.class);
}

public class RenderingSystem extends SortedIteratingSystem {

    public RenderingSystem () {
         super(Familly.all(SpriteComponent.class).get(), new ZComparator())
    }

    public void processEntity(Entity entity, float deltaTime) {
         if(checkZIndexHasChangeValue()) {
              forceSort();
         }
    }

    private static class ZComparator implements Comparator<Entity> {
        @Override
        public int compare(Entity entityA, Entity entityB) {
            return (int)Math.signum(Mapper.sprite.get(entityA).z - Mapper.sprite.get(entityB).z);
        }
    }

}

Aralıklı Yineleme Sistemi Oluşturma #

Varlıklar Ailesini kare başına bir kez değil, belirli bir aralıktan sonra işleyen basit bir ‘EntitySystem’. Varlık işleme mantığı processEntity(Entity) içine yerleştirilmelidir. Daha fazla bilgi için lütfen IntervalIteratingSystem bölümüne bakın.

Aşağıdaki kod örneğinde bunun için en iyi kullanım fizik world adımıdır.

public class Constants {
     public final static float TIME_STEP = 1 / 60.0f; // 60 fps
     public final static int VELOCITY_ITERATIONS = 6;
     public final static int POSITION_ITERATIONS = 2;
}

public class PhysicsSystem extends IntervalIteratingSystem {
    public PhysicsSystem () {
         super(Family.all(PhysicsComponent.class), Constants.TIME_STEP);
    }

    @Override
    protected void processEntity(Entity entity) {
         // process the physics component here with an interval of 60fps
    }

    @Override
    protected void updateInterval() {
        WorldManager.world.step(Constants.TIME_STEP, Constants.VELOCITY_ITERATIONS, Constants.POSITION_ITERATIONS);
        super.updateInterval();
    }

}
