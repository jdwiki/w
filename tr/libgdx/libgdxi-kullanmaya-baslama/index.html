Kurulum veya Kurulum #

LibGDX, basit bir Java programı yardımıyla oldukça basit bir kuruluma sahiptir. İndirmeyi burada bulabilirsiniz. Uygulamayı başlattığınızda, şöyle görünecektir:
Not: Bu ekran görüntüsü Linux’ta alınmıştır ve Windows kurulumundan farklı olan yolu göstermektedir. Ancak form, bu kurulum uygulamasıyla uyumlu tüm işletim sistemlerinde aynıdır

Giriş kutuları #

“Ad” giriş kutusunda, projeniz için oyunun adıdır. “Paket” giriş kutusu, ana sınıfınızın paketidir. “Oyun Sınıfı” giriş kutusu, oyununuz çalıştırıldığında çağrılan ana sınıftır. “Hedef” giriş kutusu, projenizin oluşturulacağı hedeftir. “Andriod SDK” giriş kutusu, android sdk’nizin bulunduğu yol. Bu giriş kutusu tamamen isteğe bağlıdır, bu nedenle uygulamanızı Android’e dağıtmak istemiyorsanız, bunun için endişelenmenize gerek yok.

Alt Projeler #

Alt projeler yalnızca dağıtılacak platformlardır. Bu kısım oldukça açıklayıcı. Örneğin, HTML’ye dağıtmak istemiyorsanız, onay kutusunun işaretini kaldırmanız yeterlidir.

Nesil #

Tüm ayarlarınızı doğru yaptıktan sonra “Oluştur” düğmesine basabilirsiniz. Bu birkaç saniye sürebilir, ancak projeniz için gerekli olan temel dosyaları ve Gradle’ı oluşturacaktır. Bununla işiniz bittiğinde, projeyi IDE’nize aktarmanın zamanı geldi.

LibGDX Merhaba Dünya #

Temel bilgiler

Oluşturulan projeler, halihazırda uygulanmış temel bir Hello World benzeri uygulama içerir.

Ana proje, platformdan bağımsız tüm kodları içeren çekirdek projedir. Bu zorunludur, ancak oluşturma ayarlarınıza bağlı olarak, seçtiğiniz her platform için birkaç projeniz daha olabilir.

Örnek

“Çekirdek” projesinde “com.mygdx.game.MyGdxGame.java” dosyasını açın. Aşağıdaki kodu göreceksiniz:

public class MyGdxGame extends ApplicationAdapter {
    SpriteBatch batch;
    Texture img;
    
    @Override
    public void create () {
        batch = new SpriteBatch();
        img = new Texture("badlogic.jpg");
    }

    @Override
    public void render () {
        Gdx.gl.glClearColor(1, 0, 0, 1);
        Gdx.gl.glClear(GL20.GL_COLOR_BUFFER_BIT);
        batch.begin();
        batch.draw(img, 0, 0);
        batch.end();
    }
    
    @Override
    public void dispose () {
        batch.dispose();
        img.dispose();
    }
}

Bu sizin ana projeniz olmasına rağmen, onu doğrudan çalıştırmayacaksınız, her zaman platforma özel Launcher’ı çalıştırmanız gerekiyor, masaüstü için ‘desktop’ projesinde ‘com.mygdx.game.desktop.DesktopLauncher.java’ olarak adlandırılıyor.

public class DesktopLauncher {
    public static void main (String[] arg) {
        LwjglApplicationConfiguration config = new LwjglApplicationConfiguration();
        new LwjglApplication(new MyGdxGame(), config);
    }
}

Bu sınıfta platforma özel ayarlar yapabilirsiniz.

MyGdxGame sınıfında neler oluyor? #

İlk olarak, ekrana çizilen grubu başlatacak olan ‘create’ yöntemi çağrılır. Ardından yöntem, badlogic.jpg dosyasını belleğe yükler.

Bundan sonra, uygulama durdurulana kadar ‘render’ yöntemi tekrar tekrar çağrılır. Bu yöntem arka plan rengini kırmızıya döndürür ve görüntüyü ekrana çizer. Gördüğünüz gibi toplu çizime her zaman başlamanız ve bitirmeniz gerekir. Son olarak, uygulama durmak üzereyken, doku ve toplu iş tarafından kullanılan bellek alanını boşaltacak olan ‘dispose’ yöntemi çağrılır.

(Atmanın çalışma zamanında ve Android gibi diğer platformlarda da yapılabileceğini bilmek güzel, çünkü uygulamalar arka plandayken Android bellek alanını boşaltabilir, ancak bu daha gelişmiş bir konudur)

LibGDX’e Genel Bakış #

LibGDX, Java’da geliştirilmiş ücretsiz, açık kaynaklı bir oyun geliştirme kitaplığıdır. Hedefleri, kullanıcıların masaüstü, Android, iOS ve web tarayıcılarında çalışan platformlar arası oyunlar geliştirmelerine izin vermektir. Kodu bir kez yazın, büyük platformlardan herhangi birine dağıtın.

Platforma özel kod desteği ekleme #

LibGDX, aynı kodu yazıp birkaç farklı platforma dağıtabileceğiniz şekilde tasarlanmıştır. Yine de, platforma özel koda erişmek istediğiniz zamanlar vardır. Örneğin, oyununuzda skor tablolarınız ve başarılarınız varsa, bunları yerel olarak depolamanın yanı sıra platforma özel araçlar (Google Play Oyunlar gibi) kullanmak isteyebilirsiniz. Veya bir veritabanı veya tamamen farklı bir şey kullanmak istiyorsunuz.

Bu tür bir kodu çekirdek modüle ekleyemezsiniz. Yani ilk adım bir Arayüz oluşturmaktır. Çekirdek modülde oluşturun. Bu ilki, diğer arayüzleri yönetmek için bir yardımcı programdır:

public interface PlatformWrapper{
    //here you can also add other things you need that are platform specific.
    //If you want to create your own file saver for an instance, a JSON based database,
    //achievements, leaderboards, in app purchases and anything else you need platform specific code for.
    SQLWrapper getSQLSaver();//This one will be used in this example
    AchievementWrapper getAchievementHandler();//this line is here as an example
}

Ardından, SQLWrapper adlı ikinci bir arayüz oluşturmamız gerekiyor. Bu da çekirdek modüle girer.

public interface SQLWrapper{
    void init(String DATABASE);
    void saveSerializable(int id, Object o);
    Object loadSerializable(int id, Object o);
    void saveString(int id, String s);
    //.... and other methods you need here. This is something that varies
    //with usage. You may not need the serializable methods, but really need a custom method for saving 
    //an entire game world. This part is entirely up to you to find what methods you need
   
    //With these three methods, always assume it is the active database in question. Unless 
    //otherwise specified, it most likely is
    String getActiveDatabase();
    void deleteDatabase();
    void deleteTable(String table);//delete the active table
}

Şimdi, her projeye girmeniz ve PlatformWrapper’ı uygulamak için bir sınıf ve SQLWrapper’ı uygulamak için bir sınıf oluşturmanız gerekir. Her projede, örnekler, kurucular vb. gibi gerekli kodu eklersiniz.

Yaptığınız tüm arabirimleri geçersiz kıldıktan sonra, hepsinin sınıfta PlatformWrapper’ı uygulayan bir örneği olduğundan (ve bir alıcının olduğundan) emin olun. Son olarak, ana sınıfta yapıcıyı değiştirirsiniz. Ana sınıf, başlatıcıdan referans aldığınız sınıftır. ApplicationAdapter’ı genişletir, ApplicationListener’ı uygular veya Game’i genişletir. Yapıcıyı düzenleyin ve bir argüman olarak PlatformWrapper’ı ekleyin. Platform sarmalayıcının içinde, diğer tüm sarmalayıcılara (sql, başarımlar veya eklediğiniz herhangi bir şey) ek olarak (eğer eklediyseniz) bazı yardımcı programlarınız vardır.

Şimdi, her şey doğru ayarlanmışsa, PlatformWrapper’ı arayabilir ve platformlar arası arayüzlerden herhangi birini alabilirsiniz. Herhangi bir yöntemi çağırın ve (yürütülen kodun doğru olduğunu varsayarak) herhangi bir platformda göreceksiniz, platforma özel kodla yapması gerekeni yapar

Varlık yöneticisi #

AssetManager, varlıklarınızı yönetmenize yardımcı olan bir sınıftır.

Öncelikle, bir örnek oluşturmanız gerekir:

AssetManager am = new AssetManager();

Bu başlatıldıktan sonra ve herhangi bir şey oluşturmadan önce kaynakları almak istersiniz:

am.load("badlogic.jpg", Texture.class);//Texture.class is the class this asset is of. If it is a 
//sound asset, it doesn't go under Texture. if it is a 3D model, it doesn't go under Texture.class
//Which class added depends on the asset you load

//... other loading here ...//

//when finished, call finishLoading:
am.finishLoading();

Şimdi, “badlogic.jpg” almak istediğiniz her yerde:

Texture texture = am.get("badlogic.jpg");
//Ready to render! The rendering itself is in the normal way

AssetManager’ı kullanmak, bunları AssetManager’ın belleğine bir kez yüklemenize ve ardından istediğiniz kadar almanıza olanak tanır.

Neden AssetManager’ı kullanmalısınız? (wiki’den):

AssetManager (kod), varlıklarınızı yüklemenize ve yönetmenize yardımcı olur. Aşağıdaki güzel davranışlar nedeniyle varlıklarınızı yüklemenin önerilen yolu budur:

  • Çoğu kaynağın yüklenmesi eşzamansız olarak yapılır, böylece işler yüklenirken reaktif bir yükleme ekranı görüntüleyebilirsiniz.
  • Varlıklar referans sayılır. A ve B varlıklarının ikisi de başka bir C varlığına bağlıysa, A ve B elden çıkarılana kadar C elden çıkarılmaz. Bu aynı zamanda bir varlığı birden çok kez yüklerseniz, aslında paylaşılacağı ve yalnızca bir kez bellekte yer kaplayacağı anlamına gelir!
  • Tüm varlıklarınızı saklamak için tek bir yer.
  • Önbellek gibi şeyleri şeffaf bir şekilde uygulamaya izin verir (aşağıdaki FileHandleResolver’a bakın)

2D sahne grafiği #

Java veya Android kullanmaya başladığınızda, sol üst köşede (0,0) olduğunu hemen öğrenirsiniz. Ancak LibGDX’te (0,0) varsayılan olarak sol alt köşededir.

Bir Ortografik kamera kullanarak, (0, 0) sol üst köşede olabilirsiniz. Varsayılan olarak, (0, 0) sol alt köşededir. Bu, dokuların hangi köşelerinin X ve Y koordinatlarına sahip olduğunu değiştirdiği için, bilinmesi önemli bir şeydir.
