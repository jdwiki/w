Tasarım kalıpları, yazılım geliştirmede sıklıkla ortaya çıkan sorunlara genel çözümlerdir. Aşağıda, kodun yapılandırılması ve tasarlanmasında standartlaştırılmış en iyi uygulamaların şablonları ve bu tasarım modellerinin uygun olacağı yaygın bağlam örnekleri yer almaktadır.

Yaratıcı tasarım desenleri, bir sistemi yaratma, kompozisyon ve temsil sürecinden daha bağımsız hale getirmek için nesnelerin somutlaştırılmasını soyutlar.

Tekil #

Singletons, bir program boyunca paylaşılan bir sınıfın tek bir örneğinden oluşan, sık kullanılan bir tasarım desenidir.

Aşağıdaki örnekte, “Foo” sınıfının bir örneğini tutan bir “statik” özellik oluşturuyoruz. ‘Statik’ bir özelliğin bir sınıfın tüm nesneleri arasında paylaşıldığını ve alt sınıflama ile üzerine yazılamayacağını unutmayın.

public class Foo
{
    static let shared = Foo()
    
    // Used for preventing the class from being instantiated directly
    private init() {}
    
    func doSomething()
    {
        print("Do something")
    }
}

Kullanım:

Foo.shared.doSomething()

“Özel” başlatıcıyı hatırladığınızdan emin olun:

Bu, tekillerinizin gerçekten benzersiz olmasını sağlar ve erişim kontrolü sayesinde dış nesnelerin kendi sınıfınızın örneklerini oluşturmasını engeller. Swift’de tüm nesneler varsayılan bir genel başlatıcı ile geldiğinden, init’inizi geçersiz kılmanız ve onu özel yapmanız gerekir. KrakenDev

Oluşturucu Modeli #

Oluşturucu kalıbı bir nesne oluşturma yazılımı tasarım kalıbıdır. Soyut fabrika modeli ve fabrika yöntemi modelinin aksine amacı polimorfizmi sağlamak olan, oluşturucu modeli, teleskopik kurucuya bir çözüm bulmaktır desen karşıtı. Teleskop yapıcı anti-desen, aşağıdaki durumlarda oluşur: nesne oluşturucu parametre kombinasyonunun artması, yapıcıların üstel listesi. Çok sayıda kullanmak yerine yapıcılar, oluşturucu kalıbı başka bir nesne, bir oluşturucu kullanır, her başlatma parametresini adım adım alır ve ardından geri döner bir kerede ortaya çıkan inşa edilmiş nesne.

-Wikipedia

Oluşturucu kalıbının ana amacı, bir nesne için oluşturulduğu andan itibaren varsayılan bir yapılandırma ayarlamaktır. İnşa edilecek nesne ile onu inşa etmekle ilgili diğer tüm nesneler arasında bir aracıdır.

Örnek: #

Daha açık hale getirmek için, bir Car Builder örneğine bakalım.

Bir Car sınıfının bir nesne oluşturmak için birçok seçenek içerdiğini düşünün, örneğin:

  • Renk.

  • Koltukların sayısı.

  • Tekerlek sayısı.

  • Tip.

  • Dişli tipi.

  • Motor.

  • Hava yastığı kullanılabilirliği.

Bir araba nesnesi oluşturma:

*/

Bir araba nesnesi oluştururken ortaya çıkan sorun, arabanın oluşturulması için birçok yapılandırma verisi gerektirmesidir.

Oluşturucu Modelini uygulamak için, başlatıcı parametrelerinin, gerektiğinde değiştirilebilen* varsayılan değerleri olmalıdır.

CarBuilder sınıfı:

class CarBuilder {
    var color: UIColor = UIColor.black
    var numberOfSeats: UInt8 = 5
    var numberOfWheels: UInt8 = 4
    var type: CarType = .saloon
    var gearType: GearType = .automatic
    var motor: Motor = Motor(id: "111", name: "Default Motor",
                             model: "T9", numberOfCylinders: 4)
    var shouldHasAirbags: Bool = false
    
    func buildCar() -> Car {
        return Car(color: color, numberOfSeats: numberOfSeats, numberOfWheels: numberOfWheels, type: type, gearType: gearType, motor: motor, shouldHasAirbags: shouldHasAirbags)
    }
}

“CarBuilder” sınıfı, oluşturulan araba nesnesinin değerlerini düzenlemek için değiştirilebilecek özellikleri tanımlar.

‘CarBuilder’ı kullanarak yeni arabalar yapalım:

var builder = CarBuilder()
// currently, the builder creates cars with default configuration.

let defaultCar = builder.buildCar()
//print(defaultCar.description)
/* prints
 color: UIExtendedGrayColorSpace 0 1
 Number of seats: 5
 Number of Wheels: 4
 Type: automatic
 Motor: Motor(id: "111", name: "Default Motor", model: "T9", numberOfCylinders: 4)
 Airbag Availability: false
*/

builder.shouldHasAirbags = true
// now, the builder creates cars with default configuration,
// but with a small edit on making the airbags available

let safeCar = builder.buildCar()
print(safeCar.description)
/* prints
 color: UIExtendedGrayColorSpace 0 1
 Number of seats: 5
 Number of Wheels: 4
 Type: automatic
 Motor: Motor(id: "111", name: "Default Motor", model: "T9", numberOfCylinders: 4)
 Airbag Availability: true
 */

builder.color = UIColor.purple
// now, the builder creates cars with default configuration
// with some extra features: the airbags are available and the color is purple

let femaleCar = builder.buildCar()
print(femaleCar)
/* prints
 color: UIExtendedSRGBColorSpace 0.5 0 0.5 1
 Number of seats: 5
 Number of Wheels: 4
 Type: automatic
 Motor: Motor(id: "111", name: "Default Motor", model: "T9", numberOfCylinders: 4)
 Airbag Availability: true
*/

Oluşturucu Modelini uygulamanın faydası, varsayılan değerleri ayarlayarak birçok konfigürasyon içermesi gereken nesneleri yaratmanın ve ayrıca bu varsayılan değerleri değiştirmenin kolaylığıdır.

Daha İleriye Gidin: #

İyi bir uygulama olarak, varsayılan değerlere ihtiyaç duyan tüm özellikler, sınıfın kendisi ve oluşturucusu tarafından uygulanması gereken ayrı bir protokolde olmalıdır.

Örneğimize dönerek, CarBluePrint adında yeni bir protokol oluşturalım:

import UIKit

enum CarType {
    case
    
    sportage,
    saloon
}

enum GearType {
    case
    
    manual,
    automatic
}

struct Motor {
    var id: String
    var name: String
    var model: String
    var numberOfCylinders: UInt8
}

protocol CarBluePrint {
    var color: UIColor { get set }
    var numberOfSeats: UInt8 { get set }
    var numberOfWheels: UInt8 { get set }
    var type: CarType { get set }
    var gearType: GearType { get set }
    var motor: Motor { get set }
    var shouldHasAirbags: Bool { get set }
}

class Car: CustomStringConvertible, CarBluePrint {
    var color: UIColor
    var numberOfSeats: UInt8
    var numberOfWheels: UInt8
    var type: CarType
    var gearType: GearType
    var motor: Motor
    var shouldHasAirbags: Bool
    
    var description: String {
        return "color: \(color)\nNumber of seats: \(numberOfSeats)\nNumber of Wheels: \(numberOfWheels)\n Type: \(gearType)\nMotor: \(motor)\nAirbag Availability: \(shouldHasAirbags)"
    }
    
    init(color: UIColor, numberOfSeats: UInt8, numberOfWheels: UInt8, type: CarType, gearType: GearType, motor: Motor, shouldHasAirbags: Bool) {
        
        self.color = color
        self.numberOfSeats = numberOfSeats
        self.numberOfWheels = numberOfWheels
        self.type = type
        self.gearType = gearType
        self.motor = motor
        self.shouldHasAirbags = shouldHasAirbags
        
    }
}

class CarBuilder: CarBluePrint {
    var color: UIColor = UIColor.black
    var numberOfSeats: UInt8 = 5
    var numberOfWheels: UInt8 = 4
    var type: CarType = .saloon
    var gearType: GearType = .automatic
    var motor: Motor = Motor(id: "111", name: "Default Motor",
                             model: "T9", numberOfCylinders: 4)
    var shouldHasAirbags: Bool = false
    
    func buildCar() -> Car {
        return Car(color: color, numberOfSeats: numberOfSeats, numberOfWheels: numberOfWheels, type: type, gearType: gearType, motor: motor, shouldHasAirbags: shouldHasAirbags)
    }
}

Varsayılan değere ihtiyaç duyan özellikleri bir protokole bildirmenin yararı, eklenen herhangi bir yeni özelliği uygulamaya zorlamaktır; Bir sınıf bir protokole uyduğunda, tüm özelliklerini/yöntemlerini bildirmek zorundadır.

“Pil adı” adlı bir araba oluşturma planına eklenmesi gereken yeni bir özellik olduğunu düşünün:

protocol CarBluePrint {
    var color: UIColor { get set }
    var numberOfSeats: UInt8 { get set }
    var numberOfWheels: UInt8 { get set }
    var type: CarType { get set }
    var gearType: GearType { get set }
    var motor: Motor { get set }
    var shouldHasAirbags: Bool { get set }
    
    // adding the new property
    var batteryName: String { get set }
}

Yeni özelliği ekledikten sonra, ‘CarBluePrint’ protokolüne uymanın ‘batteryName’ özelliğini bildirmeyi gerektirdiğini bildiren iki derleme zamanı hatasının ortaya çıkacağını unutmayın. Bu, “CarBuilder"ın “batteryName” özelliği için varsayılan bir değer bildireceğini ve ayarlayacağını garanti eder.

“CarBluePrint” protokolüne “batteryName” yeni özelliği eklendikten sonra, hem “Car” hem de “CarBuilder” sınıflarının uygulanması şöyle olmalıdır:

class Car: CustomStringConvertible, CarBluePrint {
    var color: UIColor
    var numberOfSeats: UInt8
    var numberOfWheels: UInt8
    var type: CarType
    var gearType: GearType
    var motor: Motor
    var shouldHasAirbags: Bool
    var batteryName: String
    
    var description: String {
        return "color: \(color)\nNumber of seats: \(numberOfSeats)\nNumber of Wheels: \(numberOfWheels)\nType: \(gearType)\nMotor: \(motor)\nAirbag Availability: \(shouldHasAirbags)\nBattery Name: \(batteryName)"
    }
    
    init(color: UIColor, numberOfSeats: UInt8, numberOfWheels: UInt8, type: CarType, gearType: GearType, motor: Motor, shouldHasAirbags: Bool, batteryName: String) {
        
        self.color = color
        self.numberOfSeats = numberOfSeats
        self.numberOfWheels = numberOfWheels
        self.type = type
        self.gearType = gearType
        self.motor = motor
        self.shouldHasAirbags = shouldHasAirbags
        self.batteryName = batteryName
    }
}

class CarBuilder: CarBluePrint {
    var color: UIColor = UIColor.red
    var numberOfSeats: UInt8 = 5
    var numberOfWheels: UInt8 = 4
    var type: CarType = .saloon
    var gearType: GearType = .automatic
    var motor: Motor = Motor(id: "111", name: "Default Motor",
                             model: "T9", numberOfCylinders: 4)
    var shouldHasAirbags: Bool = false
    var batteryName: String = "Default Battery Name"
    
    func buildCar() -> Car {
        return Car(color: color, numberOfSeats: numberOfSeats, numberOfWheels: numberOfWheels, type: type, gearType: gearType, motor: motor, shouldHasAirbags: shouldHasAirbags, batteryName: batteryName)
    }
}

Yine, CarBuilder kullanarak yeni arabalar yapalım:

print(defaultCar)
/* prints

*/

let editedBatteryCar = builder.buildCar()
print(editedBatteryCar)
/*

 Airbag Availability: false
 Battery Name: New Battery Name
 */

Fabrika Yöntemi #

Sınıf tabanlı programlamada, fabrika yöntemi kalıbı, oluşturulacak nesnenin tam sınıfını belirtmek zorunda kalmadan nesneler oluşturma sorunuyla başa çıkmak için fabrika yöntemlerini kullanan yaratıcı bir kalıptır. Vikipedi referansı

protocol SenderProtocol
{
    func send(package: AnyObject)
}

class Fedex: SenderProtocol
{
    func send(package: AnyObject)
    {
        print("Fedex deliver")
    }
}

class RegularPriorityMail: SenderProtocol
{
    func send(package: AnyObject)
    {
        print("Regular Priority Mail deliver")
    }
}

// This is our Factory
class DeliverFactory
{
    // It will be responsable for returning the proper instance that will handle the task
    static func makeSender(isLate isLate: Bool) -> SenderProtocol
    {
        return isLate ? Fedex() : RegularPriorityMail()
    }
}

// Usage:
let package = ["Item 1", "Item 2"]

// Fedex class will handle the delivery
DeliverFactory.makeSender(isLate:true).send(package)

// Regular Priority Mail class will handle the delivery
DeliverFactory.makeSender(isLate:false).send(package)

Bunu yaparak, sınıfın gerçek uygulamasına bağlı kalmayız, sender()ı onu kimin tükettiğine karşı tamamen şeffaf hale getiririz.

Bu durumda tek bilmemiz gereken, göndericinin teslimi gerçekleştireceği ve send() adlı bir yöntemi ortaya çıkaracağıdır. Birkaç başka avantajı daha vardır: sınıf birleştirmesini azaltmak, test etmesi daha kolay, onu tüketen kişiyi değiştirmek zorunda kalmadan yeni davranışlar eklemek daha kolay.

Nesne yönelimli tasarımda arayüzler, kodun kavramsal açıklamasını kolaylaştıran ve bağımlılıkları önleyen bir bariyer oluşturan soyutlama katmanları sağlar.Wikipedia referansı

Gözlemci #

Gözlemci modeli, özne olarak adlandırılan bir nesnenin, gözlemciler olarak adlandırılan bağımlılarının bir listesini tuttuğu ve genellikle yöntemlerinden birini çağırarak herhangi bir durum değişikliğini otomatik olarak bildirdiği yerdir. Esas olarak dağıtılmış olay işleme sistemlerini uygulamak için kullanılır. Gözlemci modeli aynı zamanda tanıdık model-görünüm-kontrolör (MVC) mimari modelinde de önemli bir parçadır.Wikipedia referansı

Temel olarak gözlemci kalıbı, gözlemcileri belirli davranışlar veya durum değişiklikleri hakkında bilgilendirebilen bir nesneniz olduğunda kullanılır.

Önce Bildirim Merkezi için global bir referans (sınıfın dışında) oluşturalım:

let notifCentre = NotificationCenter.default

Harika şimdi bunu her yerden arayabiliriz. O zaman bir sınıfı gözlemci olarak kaydetmek isteriz…

notifCentre.addObserver(self, selector: #selector(self.myFunc), name: "myNotification", object: nil)

Bu, sınıfı “readForMyFunc” için bir gözlemci olarak ekler. Ayrıca, bu bildirim alındığında myFunc işlevinin çağrılması gerektiğini belirtir. Bu fonksiyon aynı sınıfta yazılmalıdır:

func myFunc(){
    print("The notification has been received")
}

Bu kalıbın avantajlarından biri, birçok sınıf ekleyebilmenizdir. gözlemci olarak ve böylece bir bildirimden sonra birçok eylemi gerçekleştirin.

Bildirim artık satırın bulunduğu kodun hemen hemen her yerinden kolayca gönderilebilir (veya isterseniz yayınlanabilir):

notifCentre.post(name: "myNotification", object: nil)

Bildirim ile bilgileri Sözlük olarak da iletebilirsiniz.

let myInfo = "pass this on"
notifCentre.post(name: "myNotification", object: ["moreInfo":myInfo])

Ancak, işlevinize bir bildirim eklemeniz gerekir:

func myFunc(_ notification: Notification){
    let userInfo = (notification as NSNotification).userInfo as! [String: AnyObject]
    let passedInfo = userInfo["moreInfo"]
    print("The notification \(moreInfo) has been received")
    //prints - The notification pass this on has been received
}

Sorumluluk zinciri #

Nesne yönelimli tasarımda, sorumluluk zinciri modeli, bir “komut” nesnesi kaynağından ve bir dizi “işleme” nesnesinden oluşan bir tasarım kalıbıdır. Her bir “işleme” nesnesi, işleyebileceği komut nesnesi türlerini tanımlayan mantığı içerir; geri kalanı zincirdeki bir sonraki ‘işleme’ nesnesine iletilir. Bu zincirin sonuna yeni ‘işleme’ nesneleri eklemek için bir mekanizma da mevcuttur. Wikipedia

Sorumluluk zincirini oluşturan sınıfları kurmak.

İlk önce tüm processing nesneleri için bir arayüz oluşturuyoruz.

protocol PurchasePower {  
var allowable : Float { get }
  var role : String { get }
  var successor : PurchasePower? { get set }
}

extension PurchasePower {
  func process(request : PurchaseRequest){
    if request.amount < self.allowable {
      print(self.role + " will approve $ \(request.amount) for \(request.purpose)")
    } else if successor != nil {
      successor?.process(request: request)
    }
  }
}

Ardından ‘command’ nesnesini oluşturuyoruz.

struct PurchaseRequest {
  var amount : Float
  var purpose : String
}

Son olarak, sorumluluk zincirini oluşturan nesneler yaratmak.

class ManagerPower : PurchasePower {
  var allowable: Float = 20
  var role : String = "Manager"
  var successor: PurchasePower?
}

class DirectorPower : PurchasePower {
  var allowable: Float = 100
  var role = "Director"
  var successor: PurchasePower?
}

class PresidentPower : PurchasePower {
  var allowable: Float = 5000
  var role = "President"
  var successor: PurchasePower?
}

Başlatın ve birlikte zincirleyin:

let manager = ManagerPower()
let director = DirectorPower()
let president = PresidentPower()

manager.successor = director
director.successor = president

Burada nesneleri zincirleme mekanizma mülk erişimidir

Çalıştırmak için istek oluşturma:

manager.process(request: PurchaseRequest(amount: 2, purpose: "buying a pen"))  // Manager will approve $ 2.0 for buying a pen
manager.process(request: PurchaseRequest(amount: 90, purpose: "buying a printer")) // Director will approve $ 90.0 for buying a printer

manager.process(request: PurchaseRequest(amount: 2000, purpose: "invest in stock")) // President will approve $ 2000.0 for invest in stock

Yineleyici #

Bilgisayar programlamasında bir yineleyici, bir programcı, özellikle listeler olmak üzere bir kapsayıcıyı geçmek için. Vikipedi

struct Turtle {
  let name: String
}

struct Turtles {
  let turtles: [Turtle]
}

struct TurtlesIterator: IteratorProtocol {
  private var current = 0
  private let turtles: [Turtle]

  init(turtles: [Turtle]) {
    self.turtles = turtles
  }

  mutating func next() -> Turtle? {
    defer { current += 1 }
    return turtles.count > current ? turtles[current] : nil
  }
}

extension Turtles: Sequence {
  func makeIterator() -> TurtlesIterator {
    return TurtlesIterator(turtles: turtles)
  }
}

Ve kullanım örneği

let ninjaTurtles = Turtles(turtles: [Turtle(name: "Leo"),
                                     Turtle(name: "Mickey"),
                                     Turtle(name: "Raph"),
                                     Turtle(name: "Doney")])
print("Splinter and")
for turtle in ninjaTurtles {
  print("The great: \(turtle)")
}
