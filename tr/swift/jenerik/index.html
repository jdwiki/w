Genel kod, tanımladığınız gereksinimlere bağlı olarak her türle çalışabilen esnek, yeniden kullanılabilir işlevler ve türler yazmanıza olanak tanır. Yinelemeyi önleyen ve amacını açık, soyut bir şekilde ifade eden kod yazabilirsiniz.

Genel özellikler, Swift’in en güçlü özelliklerinden biridir ve Swift standart kitaplığının çoğu genel kodla oluşturulmuştur. Örneğin, Swift’in “Array” ve “Sözlük” türlerinin ikisi de genel koleksiyonlardır. ‘Int’ değerlerini tutan bir dizi veya ‘String’ değerlerini tutan bir dizi veya aslında Swift’de oluşturulabilecek herhangi bir başka tür için bir dizi oluşturabilirsiniz. Benzer şekilde, belirtilen herhangi bir türdeki değerleri depolamak için bir sözlük oluşturabilirsiniz ve bu türün ne olabileceği konusunda herhangi bir sınırlama yoktur.

Kaynak: Apple’ın Swift Programlama Dili

Jeneriklerin Temelleri #

Generics, birden çok türe uygulanabilen esnek kod yazmanıza olanak tanıyan türler için yer tutuculardır. Any yerine jenerik kullanmanın avantajı, derleyicinin güçlü tip güvenliği uygulamasına hala izin vermeleridir.

Genel bir yer tutucu, “<>” açılı ayraçlar içinde tanımlanır.

Genel İşlevler #

İşlevler için bu yer tutucu, işlev adından sonra yerleştirilir:

/// Girdilerden birini rastgele seçer ve geri döndürür
func pickRandom<T>(_ a:T, _ b:T) -> T b> {
    return arc4random_uniform(2) == 0 ? a : b
}

Bu durumda, genel yer tutucu “T"dir. Fonksiyonu çağırmaya geldiğinizde, Swift sizin için ‘T’ tipini çıkarabilir (çünkü gerçek bir tip için yer tutucu görevi görür).

let randomOutput = pickRandom(5, 7) // returns an Int (that's either 5 or 7)

Burada fonksiyona iki tamsayı gönderiyoruz. Bu nedenle Swift, T == Int sonucunu çıkarıyor – bu nedenle işlev imzasının (Int, Int) -> Int olduğu anlaşılıyor.

Jeneriklerin sunduğu güçlü tür güvenliği nedeniyle - işlevin hem argümanları hem de dönüşü aynı tür olmalıdır. Bu nedenle aşağıdakiler derlenmeyecektir:

struct Foo {}

let foo = Foo()

let randomOutput = pickRandom(foo, 5) // error: cannot convert value of type 'Int' to expected argument type 'Foo'

Genel Türler #

Jenerikleri classes, structs veya enums ile kullanmak için tür adından sonra genel yer tutucuyu tanımlayabilirsiniz.

sınıf Çubuğu<T> {
    var baz : T
    
    init(baz:T) {
        self.baz = baz
    }
}

Bu genel yer tutucu, ‘Bar’ sınıfını kullanmaya başladığınızda bir tür gerektirecektir. Bu durumda, init(baz:T) başlatıcısından çıkarılabilir.

let bar = Bar(baz: "a string") // bar's type is Bar<String>

Burada ‘T’ genel yer tutucusunun ‘String’ türünde olduğu, dolayısıyla bir ‘Bar’ örneği yaratıldığı sonucuna varılır. Ayrıca türü açıkça belirtebilirsiniz:

let bar = Bar<String>(baz: "bir dize")

Bir türle birlikte kullanıldığında, verilen genel yer tutucu, verilen örneğin tüm ömrü boyunca türünü korur ve başlatmadan sonra değiştirilemez. Bu nedenle, baz özelliğine eriştiğinizde, bu verili örnek için daima String türünde olacaktır.

let str = bar.baz // of type String

Genel Türleri Geçmek #

Genel türler arasında geçiş yapmaya geldiğinizde, çoğu durumda beklediğiniz genel yer tutucu türü hakkında açık olmanız gerekir. Örneğin, bir işlev girişi olarak:

func takeABarInt(bar:Bar<Int>) {
    ...
}

Bu işlev yalnızca bir Bar<Int> kabul eder. Genel yer tutucu türünün “Int” olmadığı bir “Bar” örneğini geçirmeye çalışmak, derleyici hatasına neden olur.

Genel Yer Tutucu Adlandırma #

Genel yer tutucu adları yalnızca tek harflerle sınırlı değildir. Belirli bir yer tutucu anlamlı bir kavramı temsil ediyorsa, ona açıklayıcı bir ad vermelisiniz. Örneğin, Swift’in Array, belirli bir Array örneğinin eleman tipini tanımlayan Element adında genel bir yer tutucuya sahiptir.

public struct Array<Element> : RandomAccessCollection, MutableCollection {
    ...
}

Genel Yer Tutucu Türlerini Kısıtlama #

Genel bir sınıfın tür parametrelerini bir protokol uygulamaya zorlamak mümkündür, örneğin, [Equatable](https://developer .apple.com/library/ios/documentation/Swift/Reference/Swift_Equatable_Protocol/index.html)

class MyGenericClass<Type: Equatable>{
    
    var value: Type
    init(value: Type){
        self.value = value
    }
    
    func getValue() -> Type{
        return self.value
    }

    func valueEquals(anotherValue: Type) -> Bool{
        return self.value == anotherValue
    }
}

Yeni bir “MyGenericClass” oluşturduğumuzda, type parametresinin “Equatable” protokolünü uygulaması gerekir (type parametresinin “==” kullanılarak aynı türden başka bir değişkenle karşılaştırılabilmesini sağlamak)

let myFloatGeneric = MyGenericClass<Double>(değer: 2.71828) // geçerli
let myStringGeneric = MyGenericClass<String>(değer: "My String") // geçerli

// "[Int] tipi 'Equatable' protokolüne uymuyor"
let myInvalidGeneric = MyGenericClass<[Int]>(değer: [2])

let myIntGeneric = MyGenericClass<Int>(değer: 72)
print(myIntGeneric.valueEquals(72)) // doğru
print(myIntGeneric.valueEquals(-274)) // yanlış

// "'Dize' türünün değeri beklenen 'Int' bağımsız değişken türüne dönüştürülemiyor"
print(myIntGeneric.valueEquals("Dizem"))

Genel Sınıf Örnekleri #

‘Type’ tür parametresine sahip genel bir sınıf

class MyGenericClass<Type>{
 
    var value: Type
    init(value: Type){
        self.value = value
    }
 
    func getValue() -> Type{
        return self.value
    }
 
    func setValue(value: Type){
        self.value = value
    }
}

Artık type parametresini kullanarak yeni nesneler oluşturabiliriz.

let myStringGeneric = MyGenericClass<String>(değer: "My String Value")
let myIntGeneric = MyGenericClass<Int>(değer: 42)
 
print(myStringGeneric.getValue()) // "Dize Değerim"
print(myIntGeneric.getValue()) // 42
 
myStringGeneric.setValue("Başka Bir Dize")
myIntGeneric.setValue(1024)
 
print(myStringGeneric.getValue()) // "Başka Bir Dize"
print(myIntGeneric.getValue()) // 1024

Jenerikler, birden fazla tür parametresiyle de oluşturulabilir

class AnotherGenericClass<TypeOne, TypeTwo, TypeThree>{
 
    var value1: TypeOne
    var value2: TypeTwo
    var value3: TypeThree
    init(value1: TypeOne, value2: TypeTwo, value3: TypeThree){
        self.value1 = value1
        self.value2 = value2
        self.value3 = value3
    }
 
    func getValueOne() -> TypeOne{return self.value1}
    func getValueTwo() -> TypeTwo{return self.value2}
    func getValueThree() -> TypeThree{return self.value3}
}

Ve aynı şekilde kullanılır

let myGeneric = AnotherGenericClass<String, Int, Double>(değer1: "pi'nin değeri", değer2: 3, değer3: 3.14159)
 
print(myGeneric.getValueOne() String'dir) // true
print(myGeneric.getValueTwo() Int'dir) // true
print(myGeneric.getValueThree() Çifttir) // true
print(myGeneric.getValueTwo() String'dir) // false
 
print(myGeneric.getValueOne()) // "pi'nin değeri"
print(myGeneric.getValueTwo()) // 3
print(myGeneric.getValueThree()) // 3.14159

Dizi İşlevlerini Basitleştirmek için Jenerikleri Kullanma #

Nesne yönelimli bir kaldırma işlevi oluşturarak dizinin işlevselliğini genişleten bir işlev.

// Need to restrict the extension to elements that can be compared.
// The `Element` is the generics name defined by Array for its item types.
// This restriction also gives us access to `index(of:_)` which is also
// defined in an Array extension with `where Element: Equatable`.
public extension Array where Element: Equatable {
    /// Removes the given object from the array.
    mutating func remove(_ element: Element) {
        if let index = self.index(of: element ) {
            self.remove(at: index)
        } else {
            fatalError("Removal error, no such element:\"\(element)\" in array.\n")
        }
    }
}

Kullanım

var myArray = [1,2,3]
print(myArray)

// Prints [1,2,3]

Dizine ihtiyaç duymadan bir öğeyi kaldırmak için işlevi kullanın. Kaldırılacak nesneyi iletmeniz yeterlidir.

myArray.remove(2)
print(myArray)

// Prints [1,3]

Genel Sınıf Kalıtımı #

Genel sınıflar miras alınabilir:

// Models
class MyFirstModel {
}

class MySecondModel: MyFirstModel {
}

// Generic classes
class MyFirstGenericClass<T: MyFirstModel> {
    
    func doSomethingWithModel(model: T) {
        // Do something here
    }
    
}

class MySecondGenericClass<T: MySecondModel>: MyFirstGenericClass<T> {
    
    override func doSomethingWithModel(model: T) {
        super.doSomethingWithModel(model)
        
        // Do more things here
    }
    
}

Tür güvenliğini artırmak için jenerik kullanın #

Jenerik kullanmadan bu örneği ele alalım

protocol JSONDecodable {
    static func from(_ json: [String: Any]) -> Any?
}

Protokol bildirimi, gerçekten kullanmadığınız sürece iyi görünüyor.

let myTestObject = TestObject.from(myJson) as? TestObject

Sonucu neden ‘TestObject’e atmanız gerekiyor? Protokol bildirimindeki ‘Any’ dönüş türü nedeniyle.

Jenerik ilaçları kullanarak, çalışma zamanı hatalarına neden olabilecek bu sorunu önleyebilirsiniz (ve bunlara sahip olmak istemiyoruz!)

protocol JSONDecodable {
    associatedtype Element 
    static func from(_ json: [String: Any]) -> Element?
}

struct TestObject: JSONDecodable {
    static func from(_ json: [String: Any]) -> TestObject? {
    }
}

let testObject = TestObject.from(myJson) // testObject is now automatically `TestObject?`

Gelişmiş Tür Kısıtlamaları #

‘where’ yan tümcesini kullanarak jenerikler için birkaç tür kısıtlaması belirtmek mümkündür:

func doSomething<T where T: Comparable, T: Hashable>(first: T, second: T) {
    // Access hashable function
    guard first.hashValue == second.hashValue else {
        return
    }
    // Access comparable function
    if first == second {
        print("\(first) and \(second) are equal.")
    }
}

Argüman listesinden sonra ‘where’ yan tümcesini yazmak da geçerlidir:

func doSomething<T>(first: T, second: T) where T: Comparable, T: Hashable {
    // Access hashable function
    guard first.hashValue == second.hashValue else {
        return
    }
    // Access comparable function
    if first == second {
        print("\(first) and \(second) are equal.")
    }
}

Uzantılar, koşulları karşılayan türlerle sınırlandırılabilir. İşlev, yalnızca tür koşullarını karşılayan örnekler için kullanılabilir:

// "Element" is the generics type defined by "Array". For this example, we
// want to add a function that requires that "Element" can be compared, that
// is: it needs to adhere to the Equatable protocol.
public extension Array where Element: Equatable {
    /// Removes the given object from the array.
    mutating func remove(_ element: Element) {
        // We could also use "self.index(of: element)" here, as "index(of:_)"
        // is also defined in an extension with "where Element: Equatable".
        // For the sake of this example, explicitly make use of the Equatable.
        if let index = self.index(where: { $0 == element }) {
            self.remove(at: index)
        } else {
            fatalError("Removal error, no such element:\"\(element)\" in array.\n")
        }
    }
}
