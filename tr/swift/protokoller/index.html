Protokoller, bir nesnenin nasıl kullanılacağını belirtmenin bir yoludur. Protokoller uygulamada herhangi bir kısıtlama getirmese de, bir sınıfın, yapının veya enum’un sağlaması gereken bir dizi özellik ve yöntemi tanımlarlar.

Swift protokolü, uygun türlerin uygulaması gereken bir gereksinimler topluluğudur. Protokol daha sonra, örneğin Diziler ve genel gereksinimler gibi bir türün beklendiği çoğu yerde kullanılabilir.

Protokol üyeleri her zaman tüm protokolle aynı erişim niteleyicisini paylaşır ve ayrı olarak belirtilemez. Her ne kadar bir protokol, yukarıdaki örneklere göre alıcı veya ayarlayıcı gereksinimleriyle erişimi kısıtlayabilir.

Protokoller hakkında daha fazla bilgi için bkz. Swift Programlama Dili.

Objective-C protokolleri Swift protokollerine benzer.

Protokoller ayrıca Java arayüzleriyle karşılaştırılabilir.

Protokol Temelleri #

Protokoller Hakkında #

Bir Protokol, protokole uygun bir Swift nesne türünden (sınıf, yapı veya enum) gereken başlatıcıları, özellikleri, işlevleri, alt simgeleri ve ilişkili türleri belirtir. Bazı dillerde, sonraki nesnelerin gereksinim belirtimleri için benzer fikirler “arayüzler” olarak bilinir.

Bildirilmiş ve tanımlanmış bir Protokol, kendi başına, belirtilen gereksinimlerinin imzasına sahip bir Türdür ve Swift İşlevlerinin parametre ve dönüş imzalarına dayalı bir Tür olma biçimine biraz benzer.

Swift Protokolü özellikleri isteğe bağlı olabilir, açıkça gerekli olabilir ve/veya Protokol Uzantıları olarak bilinen bir tesis aracılığıyla varsayılan uygulamalar verilebilir. Tüm belirtilen gereksinimleri için Uzantılarla donatılmış bir Protokole uymayı arzu eden bir Swift Nesne Türü (sınıf, yapı veya enum), yalnızca tam uygunluk içinde olma arzusunu belirtmelidir. Protokol Uzantılarının varsayılan uygulama olanağı, bir Protokole uymanın tüm yükümlülüklerini yerine getirmek için yeterli olabilir.

Protokoller diğer Protokoller tarafından devralınabilir. Bu, Protokol Uzantıları ile birlikte, Protokollerin Swift’in önemli bir özelliği olarak düşünülebileceği ve düşünülmesi gerektiği anlamına gelir.

Protokoller ve Uzantılar, Swift’in program tasarımı esnekliği ve geliştirme süreçlerine yönelik daha geniş amaçlarını ve yaklaşımlarını gerçekleştirmek için önemlidir. Swift’in Protokol ve Genişletme yeteneğinin belirtilen birincil amacı, program mimarisinde ve geliştirmede kompozisyon tasarımının kolaylaştırılmasıdır. Buna Protokol Yönelimli Programlama denir. Huysuz eski zamanlayıcılar, bunun OOP tasarımına odaklanmaktan daha üstün olduğunu düşünüyor.

Protokoller, herhangi bir struct, class veya enum tarafından uygulanabilen arabirimleri tanımlar:

protocol MyProtocol {
    init(value: Int)                      // required initializer
    func doSomething() -> Bool            // instance method
    var message: String { get }           // instance read-only property
    var value: Int { get set }            // read-write instance property
    subscript(index: Int) -> Int { get }  // instance subscript
    static func instructions() -> String  // static method
    static var max: Int { get }           // static read-only property
    static var total: Int { get set }     // read-write static property
}

Protokollerde tanımlanan özellikler ya { get } veya { get set } olarak açıklama eklenmelidir. { get }, özelliğin alınabilir olması gerektiği anlamına gelir ve bu nedenle herhangi bir tür özellik olarak uygulanabilir. { get set }, özelliğin gettable olduğu kadar ayarlanabilir olması gerektiği anlamına gelir.

Bir yapı, sınıf veya numaralandırma bir protokole uyumlu olabilir:

struct MyStruct : MyProtocol {
    // Implement the protocol's requirements here
}
class MyClass : MyProtocol {
    // Implement the protocol's requirements here
}
enum MyEnum : MyProtocol {
    case caseA, caseB, caseC
    // Implement the protocol's requirements here
}

Bir protokol, gereksinimlerinin herhangi biri için bir uzantı aracılığıyla bir varsayılan uygulama da tanımlayabilir:

extension MyProtocol {
    
    // default implementation of doSomething() -> Bool
    // conforming types will use this implementation if they don't define their own
    func doSomething() -> Bool {
        print("do something!")
        return true
    }
}

Bir protokol, [`associatedtype`` gereksinimlerine[7] sahip olmadığı sürece tür olarak kullanılabilir:

func doStuff(nesne: Protokolüm) {
    // All of MyProtocol's requirements are available on the object
    print(object.message)
    print(object.doSomething())
}

öğelere izin verin: [MyProtocol] = [MyStruct(), MyClass(), MyEnum.caseA]

Ayrıca birden çok protokole uyan bir soyut tip tanımlayabilirsiniz:

Swift 3 veya daha iyisiyle, bu, protokol listesini bir ve işareti (&) ile ayırarak yapılır:

func doStuff(nesne: Protokolüm ve Başka bir Protokol) {
    // ...
}

öğelere izin verin: [MyProtocol & AnotherProtocol] = [MyStruct(), MyClass(), MyEnum.caseA]
son sürüm

Eski sürümlerde, protokollerin ‘<>’ açılı ayraçlar arasında virgülle ayrılmış bir liste olduğu sözdizimi ‘protokol<…>’ vardır.

protokol BaşkaProtokol {
    func doSomethingElse()
}

func doStuff(nesne: protokol<MyProtocol, AnotherProtocol>) {
    
    // All of MyProtocol & AnotherProtocol's requirements are available on the object
    print(object.message)
    object.doSomethingElse()
}

// MyStruct, MyClass ve MyEnum artık hem MyProtocol hem de AnotherProtocol ile uyumlu olmalıdır
öğelere izin verin : [protocol<MyProtocol, AnotherProtocol>] = [MyStruct(), MyClass(), MyEnum.caseA]
son sürüm

Mevcut türler, bir protokole uyması için genişletilebilir:

extension String : MyProtocol {
    // Implement any requirements which String doesn't already satisfy
}

[7]: https://www.wikiod.com/tr/swift/protokoller#İlişkili tür gereksinimleri

Delege kalıbı #

Bir temsilci, Cocoa ve CocoaTouch çerçevelerinde kullanılan, bir sınıfın bazı işlevlerin uygulanması için sorumluluğu diğerine devrettiği yaygın bir tasarım modelidir. Bu, çerçeve sınıfının genel işlevselliği uygularken ayrı bir temsilci örneğinin belirli kullanım durumunu uyguladığı endişelerin ayrılması ilkesini takip eder.

Temsilci modeline bakmanın başka bir yolu da nesne iletişimidir. “Nesneler” genellikle birbirleriyle konuşmaya ihtiyaç duyar ve bunu yapmak için bir nesnenin başka bir Nesnenin temsilcisi olabilmesi için bir “protokole” uyması gerekir. Bu kurulum yapıldıktan sonra, ilginç şeyler olduğunda diğer nesne delegelerine geri döner.

Örneğin, bir veri listesini görüntülemek için kullanıcı arayüzündeki bir görünüm, hangi verilerin görüntülenmesi gerektiğine karar vermekten değil, yalnızca verilerin nasıl görüntülendiğinin mantığından sorumlu olmalıdır.

Daha somut bir örneğe geçelim. iki sınıfınız varsa, bir ebeveyn ve bir çocuk:

class Parent { }
class Child { }

Ve çocuktan gelen bir değişikliği ebeveyne bildirmek istiyorsunuz.

Swift’de, delegeler bir ‘protokol’ bildirimi kullanılarak uygulanır ve bu nedenle, “temsilci"nin uygulayacağı bir “protokol” ilan edeceğiz. Burada delege ‘ana’ nesnedir.

protocol ChildDelegate: class {
    func childDidSomething()
}

Çocuğun, temsilciye yapılan başvuruyu depolamak için bir özellik bildirmesi gerekir:

class Child {
    weak var delegate: ChildDelegate?
}

“Delegate” değişkeninin isteğe bağlı olduğuna ve “ChildDelegate” protokolünün yalnızca sınıf tipine göre uygulanacak şekilde işaretlendiğine dikkat edin (bu olmadan “delegate” değişkeni herhangi bir tutma döngüsünden kaçınarak “zayıf” bir referans olarak bildirilemez. “temsilci” değişkeni artık başka hiçbir yerde referans gösterilmiyorsa, serbest bırakılacaktır). Bu, ebeveyn sınıfının yalnızca gerektiğinde ve uygun olduğunda temsilciyi kaydetmesidir.

Ayrıca, temsilcimizi ‘zayıf’ olarak işaretlemek için, protokol bildiriminde ‘class’ anahtar sözcüğünü ekleyerek ChildDelegate protokolümüzü referans türleriyle sınırlandırmalıyız.

Bu örnekte, çocuk bir şey yaptığında ve ebeveynine haber vermesi gerektiğinde, çocuk şunları arayacaktır:

delegate?.childDidSomething()

Temsilci tanımlanmışsa, temsilciye çocuğun bir şey yaptığı bildirilir.

Ebeveyn sınıfının, eylemlerine yanıt verebilmesi için ‘ChildDelegate’ protokolünü genişletmesi gerekecektir. Bu doğrudan ana sınıfta yapılabilir:

class Parent: ChildDelegate {
    ...

    func childDidSomething() {
        print("Yay!")
    }
}

Veya bir uzantı kullanarak:

extension Parent: ChildDelegate {
    func childDidSomething() {
        print("Yay!")
    }
}

Ebeveynin ayrıca çocuğa kendisinin delegesi olduğunu söylemesi gerekir:

// In the parent
let child = Child()
child.delegate = self

Varsayılan olarak bir Swift “protokolü” isteğe bağlı bir işlevin uygulanmasına izin vermez. Bunlar ancak protokolünüz @objc özniteliği ve opsiyonel değiştiricisi ile işaretlenmişse belirtilebilir.

Örneğin, “UITableView”, iOS’ta bir tablo görünümünün genel davranışını uygular, ancak kullanıcı, belirli hücrelerin nasıl göründüğünü ve davrandığını uygulayan “UITableViewDelegate” ve “UITableViewDataSource” adlı iki temsilci sınıfını uygulamalıdır.

@objc public protocol UITableViewDelegate : NSObjectProtocol, UIScrollViewDelegate {
        
        // Display customization
        optional public func tableView(tableView: UITableView, willDisplayCell cell: UITableViewCell, forRowAtIndexPath indexPath: NSIndexPath)
        optional public func tableView(tableView: UITableView, willDisplayHeaderView view: UIView, forSection section: Int)
        optional public func tableView(tableView: UITableView, willDisplayFooterView view: UIView, forSection section: Int)
        optional public func tableView(tableView: UITableView, didEndDisplayingCell cell: UITableViewCell, forRowAtIndexPath indexPath: NSIndexPath)
        ...
}

uygulayabilirsiniz örneğin, sınıf tanımınızı değiştirerek bu protokolü:

class MyViewController : UIViewController, UITableViewDelegate

Protokol tanımında “isteğe bağlı” olarak işaretlenmemiş herhangi bir yöntem (bu durumda “UITableViewDelegate”) uygulanmalıdır.

İlişkili tür gereksinimleri #

Protokoller, “associatedtype” anahtar sözcüğünü kullanarak ilişkili tür gereksinimleri tanımlayabilir:

protokol Kapsayıcı {
    associatedtype Element
    var count: Int { get }
    subscript(index: Int) -> Element { get set }
}

İlişkili tür gereksinimlerine ** sahip protokoller yalnızca genel kısıtlamalar** olarak kullanılabilir:

// Container'ın ilişkili tür gereksinimleri olduğundan bunlara izin verilmez:
func displayValues(container: Container) { ... }
class MyClass { let kapsayıcı: Kapsayıcı }
// > hata: 'Kapsayıcı' protokolü yalnızca genel bir kısıtlama olarak kullanılabilir
// > Self veya ilişkili tür gereksinimleri olduğundan

// Bunlara izin verilir:
func displayValues<T: Container>(container: T) { ... }
class MyClass<T: Container> { let container: T }

Protokole uyan bir tür, protokolün “ilişkili tür"ün görünmesini beklediği yerde belirli bir tür sağlayarak dolaylı olarak bir “ilişkili tür” gereksinimini karşılayabilir:

struct ContainerOfOne<T>: Container {
    let count = 1          // satisfy the count requirement
    var value: T
    
    // satisfy the subscript associatedtype requirement implicitly,
    // by defining the subscript assignment/return type as T
    // therefore Swift will infer that T == Element
    subscript(index: Int) -> T {
        get {
            precondition(index == 0)
            return value
        }
        set {
            precondition(index == 0)
            value = newValue
        }
    }
}

let container = ContainerOfOne(değer: "Merhaba")

(Bu örneğe açıklık getirmek için, genel yer tutucu türünün ‘T’ olarak adlandırıldığını unutmayın - daha uygun bir ad, protokolün ‘ilişkili tür Öğesini’ gölgeleyen ‘Öğe’ olacaktır. Derleyici yine de, genel yer tutucu “Element”, “ilişkili türdeki Öğe” gereksinimini karşılamak için kullanılır.)

Bir “ilişkili tip”, “typealias” kullanımıyla da açıkça karşılanabilir:

struct ContainerOfOne<T>: Container {

    typealias Element = T
    subscript(index: Int) -> Element { ... }

    // ...
}

Aynı şey uzantılar için de geçerlidir:

// Expose an 8-bit integer as a collection of boolean values (one for each bit).
extension UInt8: Container {

    // as noted above, this typealias can be inferred
    typealias Element = Bool

    var count: Int { return 8 }
    subscript(index: Int) -> Bool {
        get {
            precondition(0 <= index && index < 8)
            return self & 1 << UInt8(index) != 0
        }
        set {
            precondition(0 <= index && index < 8)
            if newValue {
                self |= 1 << UInt8(index)
            } else {
                self &= ~(1 << UInt8(index))
            }
        }
    }
}

Uygun tip zaten gereksinimi karşılıyorsa, uygulamaya gerek yoktur:

extension Array: Container {}  // Array satisfies all requirements, including Element

Yalnızca Sınıf Protokolleri #

Bir protokol, kalıtım listesindeki “sınıf” anahtar sözcüğünü kullanarak yalnızca bir sınıf‘in uygulayabileceğini belirtebilir. Bu anahtar sözcük, bu listedeki diğer devralınan protokollerden önce görünmelidir.

protokol ClassOnlyProtocol: class, SomeOtherProtocol {
    // Protocol requirements 
}

Sınıf dışı bir tür “ClassOnlyProtocol"u uygulamaya çalışırsa, bir derleyici hatası oluşturulur.

struct MyStruct: ClassOnlyProtocol { 
    // error: Non-class type 'MyStruct' cannot conform to class protocol 'ClassOnlyProtocol'  
}

Diğer protokoller ‘ClassOnlyProtocol’dan miras alabilir, ancak aynı yalnızca sınıf gereksinimine sahip olacaklardır.

protocol MyProtocol: ClassOnlyProtocol {
    // ClassOnlyProtocol Requirements
    // MyProtocol Requirements
}

class MySecondClass: MyProtocol {
    // ClassOnlyProtocol Requirements
    // MyProtocol Requirements
}

Yalnızca sınıf protokollerinin referans semantiği #

Yalnızca sınıf protokolünün kullanılması, uygun tür bilinmediğinde referans semantiği için izin verir.

protocol Foo : class {
    var bar : String { get set }
}

func takesAFoo(foo:Foo) {

    // this assignment requires reference semantics,
    // as foo is a let constant in this scope.
    foo.bar = "new value"
}

Bu örnekte, ‘Foo’ yalnızca sınıf protokolü olduğundan, derleyici ‘foo’nun bir sınıf türü olduğunu ve dolayısıyla referans anlambilimine sahip olduğunu bildiğinden, ‘bar’a atama geçerlidir.

“Foo” yalnızca sınıf protokolü değilse, uyumlu tür bir değer türü olabileceğinden bir derleyici hatası verilirdi, bu da değişken olması için bir “var” ek açıklamasına ihtiyaç duyardı.

protocol Foo {
    var bar : String { get set }
}

func takesAFoo(foo:Foo) {
    foo.bar = "new value" // error: Cannot assign to property: 'foo' is a 'let' constant
}


func alırAFoo(foo:Foo) {
    var foo = foo // mutable copy of foo
    foo.bar = "new value" // no error – satisfies both reference and value semantics
}

Protokol türünün zayıf değişkenleri #

‘weak’ değiştiricisini bir protokol tipi değişkenine uygularken, ‘weak’ sadece referans tiplerine uygulanabileceğinden, bu protokol tipi sadece sınıf olmalıdır.

weak var weakReference : ClassOnlyProtocol?

Belirli bir uyumlu sınıf için protokol uzantısı #

Belirli bir sınıf için varsayılan protokol uygulamasını yazabilirsiniz.

protocol MyProtocol {
    func doSomething()
}

extension MyProtocol where Self: UIViewController {
    func doSomething() {
        print("UIViewController default protocol implementation")
    }
}

class MyViewController: UIViewController, MyProtocol { }

let vc = MyViewController()
vc.doSomething() // Prints "UIViewController default protocol implementation"

RawRepresentable protokolünü kullanma (Genişletilebilir Enum) #

// RawRepresentable has an associatedType RawValue.
// For this struct, we will make the compiler infer the type
// by implementing the rawValue variable with a type of String
//
// Compiler infers RawValue = String without needing typealias
//
struct NotificationName: RawRepresentable {
    let rawValue: String

    static let dataFinished = NotificationNames(rawValue: "DataFinishedNotification")
}

Bu yapı, vaka eklemek için başka bir yere genişletilebilir

extension NotificationName {
    static let documentationLaunched = NotificationNames(rawValue: "DocumentationLaunchedNotification")
}

Ve bir arayüz, herhangi bir RawRepresentable türü veya özellikle enum yapınız etrafında tasarlayabilir.

func post(notification notification: NotificationName) -> Void {
    // use notification.rawValue
}

Çağrı sitesinde, typesafe NotificationName için nokta sözdizimi kısayolunu kullanabilirsiniz.

post(notification: .dataFinished)

Genel RawRepresentable işlevini kullanma

// RawRepresentable has an associate type, so the 
// method that wants to accept any type conforming to
// RawRepresentable needs to be generic
func observe<T: RawRepresentable>(object: T) -> Void {
    // object.rawValue
}

Hashable protokolünü uygulama #

“Kümeler” ve “Sözlükler(anahtar)“da kullanılan türler, Equatable protokolünden miras alan Hashable protokolüne uygun olmalıdır.

‘Hashable’ protokolüne uygun özel tip uygulanmalıdır

  • Hesaplanmış bir özellik “hashValue”
  • Eşitlik operatörlerinden birini tanımlayın, yani == veya !=.

Aşağıdaki örnek, özel bir “yapı” için “Hashable” protokolünü uygular:

struct Cell {
    var row: Int
    var col: Int
    
    init(_ row: Int, _ col: Int) {
        self.row = row
        self.col = col
    }
}

extension Cell: Hashable {
    
    // Satisfy Hashable requirement
    var hashValue: Int {
        get {
            return row.hashValue^col.hashValue
        }
    }

    // Satisfy Equatable requirement
    static func ==(lhs: Cell, rhs: Cell) -> Bool {
        return lhs.col == rhs.col && lhs.row == rhs.row
    }
    
}

// Now we can make Cell as key of dictonary
var dict = [Cell : String]()

dict[Cell(0, 0)] = "0, 0"
dict[Cell(1, 0)] = "1, 0"
dict[Cell(0, 1)] = "0, 1"

// Also we can create Set of Cells
var set = Set<Cell>()

set.insert(Cell(0, 0))
set.insert(Cell(1, 0))

Not: Özel tipteki farklı değerlerin farklı hash değerlerine sahip olması gerekli değildir, çakışmalar kabul edilebilir. Hash değerleri eşitse, gerçek eşitliği belirlemek için eşitlik operatörü kullanılacaktır.
