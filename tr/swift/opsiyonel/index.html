" İsteğe bağlı bir değer ya bir değer içerir ya da bir değerin eksik olduğunu belirtmek için sıfır içerir"

Alıntı: Apple Inc. “Swift Programlama Dili (Swift 3.1 Sürümü).” iBooks. https://itun.es/us/k5SW7.l

Temel isteğe bağlı kullanım durumları şunları içerir: bir sabit (let) için, bir döngü içinde isteğe bağlı kullanımı (if-let), isteğe bağlı bir değeri bir yöntem içinde güvenli bir şekilde açma (koruyucu) ve anahtar döngülerinin bir parçası olarak (case-let) ), birleştirme operatörünü kullanarak (??)

Sözdizimi #

  • var isteğe bağlıAdı: isteğe bağlıType? // isteğe bağlı bir tür tanımlayın, varsayılan değer sıfırdır
  • var isteğe bağlıAdı: isteğe bağlıType? = değer // isteğe bağlı bir değer bildir
  • var isteğe bağlıAdı: isteğe bağlıType! // örtük olarak açılmamış bir isteğe bağlı bildir
  • isteğe bağlı! // isteğe bağlı bir paketi açmaya zorla

İsteğe bağlı seçenekler hakkında daha fazla bilgi için, bkz. Swift Programlama Dili.

Opsiyonel Tipler #

İsteğe bağlı seçenekler, sarmalayıcı görevi gören genel bir numaralandırma türüdür. Bu sarmalayıcı, bir değişkenin iki durumdan birine sahip olmasına izin verir: kullanıcı tanımlı türün değeri veya bir değerin yokluğunu temsil eden “nil”.

Bu yetenek Swift’de özellikle önemlidir, çünkü dilin belirtilen tasarım hedeflerinden biri Apple’ın çerçeveleriyle iyi çalışmaktır. Apple’ın çerçevelerinin çoğu (çoğu), kullanım kolaylığı ve Objective-C içindeki programlama kalıpları ve API tasarımı açısından önemi nedeniyle “nil” kullanır.

Swift’de bir değişkenin “nil” değerine sahip olması için isteğe bağlı olması gerekir. Değişken türüne “!” veya “?” eklenerek seçenekler oluşturulabilir. Örneğin, bir “Int” öğesini isteğe bağlı yapmak için şunu kullanabilirsiniz:

var numberOne: Int! = nil
var numberTwo: Int? = nil

? seçenekleri açıkça açılmalı ve değişkene eriştiğinizde bir değeri olup olmayacağından emin değilseniz kullanılmalıdır. Örneğin, bir dizgiyi Inte dönüştürürken sonuç isteğe bağlı bir Int? olur, çünkü dize geçerli bir sayı değilse nil döndürülür.

let str1 = "42"
let num1: Int? = Int(str1) // 42

let str2 = "Hello, World!"
let num2: Int? = Int(str2) // nil

! seçenekleri otomatik olarak açılır ve yalnızca değişkene eriştiğinizde bir değere sahip olacağından emin olduğunuzda kullanılmalıdır. Örneğin, ‘viewDidLoad()’ içinde başlatılan global bir ‘UIButton!’ değişkeni

//myButton will not be accessed until viewDidLoad is called,
//so a ! optional can be used here
var myButton: UIButton!

override func viewDidLoad(){
    self.myButton = UIButton(frame: self.view.frame)
    self.myButton.backgroundColor = UIColor.redColor()
    self.view.addSubview(self.myButton)
}

İsteğe Bağlı Bir Paketi Açma #

Bir İsteğe bağlı değerine erişmek için paketin açılması gerekir.

İsteğe bağlı bağlamayı kullanarak bir İsteğe bağlı paketi koşullu olarak açabilir ve bir İsteğe bağlı paketi `!’ operatörünü kullanarak zorla açabilirsiniz.

Koşullu olarak açma, etkin bir şekilde “Bu değişkenin bir değeri var mı?” sorusunu sorar. açmaya zorla “Bu değişkenin bir değeri var!” yazıyor.

’nil’ olan bir değişkeni açmaya zorlarsanız, programınız isteğe bağlı bir istisnayı açarken *beklenmedik bir şekilde bulunan nil’i atar ve kilitlenir, bu nedenle ‘!’ kullanmanın uygun olup olmadığını dikkatlice düşünmeniz gerekir.

var text: String? = nil
var unwrapped: String = text! //crashes with "unexpectedly found nil while unwrapping an Optional value"

Güvenli açma için, sarılmış değer “nil” ise istisna oluşturmayacak veya kilitlenmeyecek bir “if-let” ifadesi kullanabilirsiniz:

var number: Int?
if let unwrappedNumber = number {       // Has `number` been assigned a value?
    print("number: \(unwrappedNumber)") // Will not enter this line
} else {
    print("number was not assigned a value")
}

Veya, bir koruma ifadesi:

var number: Int?
guard let unwrappedNumber = number else {
    return
}
print("number: \(unwrappedNumber)")

“unwrappedNumber” değişkeninin kapsamının “if-let” ifadesinin içinde ve “koruyucu” bloğunun dışında olduğuna dikkat edin.

Birçok seçeneğin paketten çıkarılmasını zincirleme yapabilirsiniz, bu özellikle kodunuzun doğru çalışması için değişkenden daha fazlasını gerektirdiği durumlarda kullanışlıdır:

var firstName:String?
var lastName:String?

if let fn = firstName, let ln = lastName {
    print("\(fn) + \(ln)")//pay attention that the condition will be true only if both optionals are not nil.
}

Testi başarılı bir şekilde geçmek için tüm değişkenlerin paketinin açılması gerektiğini unutmayın, aksi takdirde hangi değişkenlerin açılıp hangilerinin açılmadığını belirlemenin hiçbir yolu olmazdı.

Paketi açıldıktan hemen sonra isteğe bağlı seçeneklerinizi kullanarak koşullu ifadeleri zincirleyebilirsiniz. Bu, iç içe if - else ifadeleri olmadığı anlamına gelir!

var firstName:String? = "Bob"
var myBool:Bool? = false

if let fn = firstName, fn == "Bob", let bool = myBool, !bool {
    print("firstName is bob and myBool was false!")
}

1: https://www.wikiod.com/tr/Swift/sartllar#Korumayı Kullanma

Nil Birleştirme Operatörü #

Bir değeri sıfır değilse açmak için nil birleştirme operatörünü kullanabilirsiniz, aksi takdirde farklı bir değer sağlayın:

func fallbackIfNil(str: String?) -> String {
    return str ?? "Fallback String"
}
print(fallbackIfNil("Hi")) // Prints "Hi"
print(fallbackIfNil(nil)) // Prints "Fallback String"

Bu operatör kısa devre yapabilir, yani sol işlenen sıfır değilse, sağ işlenen değerlendirilmeyecektir:

func someExpensiveComputation() -> String { ... }

var foo : String? = "a string"
let str = foo ?? someExpensiveComputation()

Bu örnekte, “foo” sıfır olmadığı için “someExpensiveComputation()” çağrılmayacak.

Ayrıca birden fazla nil birleştirme ifadesini birlikte zincirleyebilirsiniz:

var foo : String?
var bar : String?

let baz = foo ?? bar ?? "fallback string"

Bu örnekte ‘baz’a, sıfır değilse, sarmalanmamış ‘foo’ değeri atanacaktır, aksi takdirde sıfır değilse, sarmalanmamış ‘bar’ değeri atanacaktır, aksi takdirde geri dönüş değeri atanacaktır. .

İsteğe Bağlı Zincirleme #

Bir yöntem çağırmak, bir özelliğe veya abonelik isteğe bağlı olarak erişmek için Opsiyonel Zincirleme kullanabilirsiniz. Bu, verilen isteğe bağlı değişken ile verilen üye (yöntem, özellik veya alt simge) arasına bir “?” koyarak yapılır.

struct Foo {
    func doSomething() {
        print("Hello World!")
    }
}

var foo : Foo? = Foo()

foo?.doSomething() // prints "Hello World!" as foo is non-nil

“foo” bir değer içeriyorsa, üzerinde “doSomething()” çağrılır. “foo” “nil” ise, kötü bir şey olmaz – kod sessizce başarısız olur ve çalışmaya devam eder.

var foo : Foo? = nil

foo?.doSomething() // will not be called as foo is nil

(Bu, Objective-C’de “nil"e mesaj göndermeye benzer bir davranıştır)

İsteğe Bağlı Zincirlemenin bu şekilde adlandırılmasının nedeni, aradığınız/eriştiğiniz üyeler aracılığıyla ‘opsiyonelliğin’ yayılacak olmasıdır. Bunun anlamı, isteğe bağlı zincirleme ile kullanılan herhangi bir üyenin dönüş değerlerinin, isteğe bağlı olarak yazılıp yazılmadığına bakılmaksızın isteğe bağlı olacağıdır.

struct Foo {
    var bar : Int
    func doSomething() { ... }
}

let foo : Foo? = Foo(bar: 5)
print(foo?.bar) // Optional(5)

Burada ‘foo?.bar’, ‘foo’nun kendisi isteğe bağlı olduğundan, ‘bar’ isteğe bağlı olmasa da bir ‘Int?’ döndürüyor.

Opsiyonellik yayıldıkça, isteğe bağlı zincirleme ile çağrıldığında ‘Void’ döndüren yöntemler ‘Void?’ döndürecektir. Bu, yöntemin çağrılıp çağrılmadığını (ve dolayısıyla isteğe bağlı bir değere sahip olup olmadığını) belirlemek için yararlı olabilir.

let foo : Foo? = Foo()

if foo?.doSomething() != nil {
    print("foo is non-nil, and doSomething() was called")
} else {
    print("foo is nil, therefore doSomething() wasn't called")
}

Burada, yöntemin çağrılıp çağrılmadığını (ve dolayısıyla ‘foo’nun sıfır olup olmadığını) belirlemek için ‘Void?’ dönüş değerini ’nil’ ile karşılaştırıyoruz.

Genel Bakış - Neden Opsiyonel? #

Çoğu zaman programlama yaparken, değeri olan ve olmayan bir değişken arasında bir ayrım yapmak gerekir. C İşaretçileri gibi referans türleri için, değişkenin değeri olmadığını belirtmek için “null” gibi özel bir değer kullanılabilir. Bir tamsayı gibi içsel türler için daha zordur. -1 gibi bir aday değer kullanılabilir, ancak bu, değerin yorumlanmasına bağlıdır. Ayrıca normal kullanımdaki “özel” değeri de ortadan kaldırır.

Bunu ele almak için Swift, herhangi bir değişkenin isteğe bağlı olarak bildirilmesine izin verir. Bu, bir ? veya ! türden sonra (Bkz. Seçenek türleri)

Örneğin,

var possiblyInt: Int?

bir tamsayı değeri içerebilecek veya içermeyebilecek bir değişken bildirir.

“nil” özel değeri, şu anda bu değişkene atanmış bir değer olmadığını gösterir.

possiblyInt = 5      // PossiblyInt is now 5
possiblyInt = nil    // PossiblyInt is now unassigned

“nil”, atanmış bir değeri test etmek için de kullanılabilir:

if possiblyInt != nil {
    print("possiblyInt has the value \(possiblyInt!)")
}

İsteğe bağlı değeri unwrap için print ifadesinde ! kullanımına dikkat edin.

İsteğe bağlı seçeneklerin yaygın kullanımına bir örnek olarak, rakamlar içeren bir dizeden tamsayı döndüren bir işlev düşünün; Dizenin rakam olmayan karakterler içermesi veya hatta boş olması mümkündür.

Basit bir ‘Int’ döndüren bir işlev başarısızlığı nasıl gösterebilir? Bunu, herhangi bir belirli değeri döndürerek yapamaz, çünkü bu, bu değerin dizeden ayrıştırılmasını engeller.

var someInt
someInt = parseInt("not an integer") // How would this function indicate failure?

Ancak Swift’de bu işlev yalnızca isteğe bağlı bir Int döndürebilir. Daha sonra başarısızlık, ’nil’ dönüş değeri ile gösterilir.

var someInt?
someInt = parseInt("not an integer")  // This function returns nil if parsing fails
if someInt == nil {
    print("That isn't a valid integer")
}
