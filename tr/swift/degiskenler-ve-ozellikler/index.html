Değişkenler: Bir türle ilişkili değil

Daha fazla bilgi için Swift Programlama Dili iBook’e bakın.

Değişken Oluşturma #

“var” ve ardından bir ad, tür ve değer ile yeni bir değişken bildirin:

var num: Int = 10

Değişkenlerin değerleri değişebilir:

num = 20 // num now equals 20

“let” ile tanımlanmadıkları sürece:

let num: Int = 10 // num cannot change

Swift, değişken türünü çıkarır, bu nedenle her zaman değişken türünü bildirmeniz gerekmez:

let ten = 10 // num is an Int
let pi = 3.14 // pi is a Double
let floatPi: Float = 3.14 // floatPi is a Float

Değişken adları harfler ve sayılarla sınırlı değildir - bazı kısıtlamalar olsa da diğer unicode karakterlerin çoğunu da içerebilirler.

Sabit ve değişken adları boşluk karakterleri, matematiksel semboller, oklar, özel kullanım (veya geçersiz) Unicode kod noktaları veya çizgi ve kutu çizim karakterleri içeremez. Ne de bir sayı ile başlayabilirler

Kaynak developer.apple.com

var π: Double = 3.14159
var 🍎🍏: String = "Apples"

Mülk Gözlemcileri #

Mülk gözlemcileri, bir mülkün değerindeki değişikliklere yanıt verir.

var myProperty = 5 {
    willSet {
        print("Will set to \(newValue). It was previously \(myProperty)")
    }
    didSet {
        print("Did set to \(myProperty). It was previously \(oldValue)")
    }
}
myProperty = 6
// prints: Will set to 6, It was previously 5
// prints: Did set to 6. It was previously 5
  • “willSet”, “myProperty” ayarlanmadan önce olarak adlandırılır. Yeni değer ’newValue’ olarak mevcuttur ve eski değer hala ‘myProperty’ olarak mevcuttur.
  • didSet, myProperty ayarlandıktan sonra olarak adlandırılır. Eski değer “oldValue” olarak, yeni değer ise “myProperty” olarak mevcuttur.

Not:** didSet ve willSet aşağıdaki durumlarda çağrılmaz:

  • Bir başlangıç ​​değeri atama
  • Değişkeni kendi “didSet” veya “willSet” içinde değiştirme

  • Okunabilirliği artırmak için “didSet” ve “willSet"in “oldValue” ve “newValue” için parametre adları da bildirilebilir:

    var myFontSize = 10 { willSet(newFontSize) { print(“Will set font to (newFontSize), it was (myFontSize)”) } didSet(oldFontSize) { print(“Did set font to (myFontSize), it was (oldFontSize)”) } }

Dikkat: Ayarlayıcı parametre adlarının bildirilmesi desteklense de, adları karıştırmamak için dikkatli olunmalıdır:

  • willSet(oldValue) ve didSet(newValue) tamamen yasaldır, ancak kodunuzu okuyanların kafasını önemli ölçüde karıştıracaktır.

Tembel Depolanan Özellikler #

Tembel saklanan özellikler, ilk erişilene kadar hesaplanmayan değerlere sahiptir. Bu, değişkenin hesaplanması hesaplama açısından pahalı olduğunda bellek tasarrufu için kullanışlıdır. ’lazy’ ile tembel bir özellik ilan ediyorsunuz:

lazy var veryExpensiveVariable = expensiveMethod()

Genellikle bir kapanışın dönüş değerine atanır:

lazy var veryExpensiveString = { () -> String in
    var str = expensiveStrFetch()
    str.expensiveManipulation(integer: arc4random_uniform(5))
    return str
}()

Tembel saklanan özellikler ‘var’ ile bildirilmelidir.

Mülkiyet Temelleri #

Özellikler bir class veya struct‘ye eklenebilir (teknik olarak enums da, “Bilgisayarlı Özellikler” örneğine bakın). Bunlar, sınıf/yapı örnekleriyle ilişkilendirilen değerler ekler:

class Dog {
    var name = ""
}

Yukarıdaki durumda, “Köpek” örnekleri, “Dize” türünde “ad” adlı bir özelliğe sahiptir. Özelliğe “Köpek” örneklerinde erişilebilir ve değiştirilebilir:

let myDog = Dog()
myDog.name = "Doggy" // myDog's name is now "Doggy"

Bu tür özellikler, bir nesne üzerinde bir şeyler depoladıkları ve belleğini etkiledikleri için depolanmış özellikler olarak kabul edilir.

Hesaplanan Özellikler #

Depolanan özelliklerden farklı olarak, hesaplanan özellikler bir alıcı ve ayarlayıcı ile oluşturulur, erişildiğinde ve ayarlandığında gerekli kodu gerçekleştirir. Hesaplanan özellikler bir tür tanımlamalıdır:

var pi = 3.14

class Circle {
    var radius = 0.0
    var circumference: Double {
        get {
            return pi * radius * 2
        }
        set {
            radius = newValue / pi / 2
        }
    }
}

let circle = Circle()
circle.radius = 1
print(circle.circumference) // Prints "6.28"
circle.circumference = 14
print(circle.radius) // Prints "2.229..."

Salt okunur bir hesaplanmış özellik hala bir “var” ile bildirilir:

var circumference: Double {
    get {
        return pi * radius * 2
    }
}

Salt okunur hesaplanan özellikler, “get"i hariç tutmak için kısaltılabilir:

var circumference: Double {
    return pi * radius * 2
}

Yerel ve Global Değişkenler #

Yerel değişkenler bir işlev, yöntem veya kapatma içinde tanımlanır:

func printSomething() {
    let localString = "I'm local!"
    print(localString)
}

func printSomethingAgain() {
    print(localString) // error
}

Genel değişkenler bir işlev, yöntem veya kapatmanın dışında tanımlanır ve bir tür içinde tanımlanmaz (tüm parantezlerin dışında düşünün). Her yerde kullanılabilirler:

let globalString = "I'm global!"
print(globalString)

func useGlobalString() {
    print(globalString) // works!
}

for i in 0..<2 {
    print(globalString) // works!
}

class GlobalStringUser {
    var computeGlobalString {
        return globalString // works!
    }
}

Global değişkenler tembelce tanımlanır (bkz. “Tembel Özellikler” örneği).

Tür Özellikleri #

Tür özellikleri, örnekte değil, türün kendisindeki özelliklerdir. Hem depolanmış hem de hesaplanmış özellikler olabilir. “Statik” olan bir tür özelliği bildirirsiniz:

struct Dog {
    static var noise = "Bark!"
}

print(Dog.noise) // Prints "Bark!"

Bir sınıfta, onu geçersiz kılmak için ‘static’ yerine ‘class’ anahtar sözcüğünü kullanabilirsiniz. Ancak, bunu yalnızca hesaplanan özelliklere uygulayabilirsiniz:

class Animal {
    class var noise: String {
        return "Animal noise!"
    }
}
class Pig: Animal {
    override class var noise: String {
        return "Oink oink!"
    }
}

Bu genellikle singleton pattern ile kullanılır.
