Sözdizimi #

  • NSJSONSerialization.JSONObjectWithData(jsonData, options: NSJSONReadingOptions) // jsonData’dan bir Nesne döndürür. Bu yöntem başarısızlık üzerine atar.
  • NSJSONSerialization.dataWithJSONObject(jsonObject, options: NSJSONWritingOptions) // Bir JSON nesnesinden NSData döndürür. Daha okunabilir bir çıktı için seçeneklerde NSJSONWritingOptions.PrettyPrinted iletin.

Apple Foundation ve Swift Standard Library ile JSON Serileştirme, Kodlama ve Kod Çözme #

JSONSerialization sınıfı, Apple’ın Foundation çerçevesine yerleştirilmiştir.

JSON’u okuyun #

‘JSONObjectWithData’ işlevi ‘NSData’yı alır ve ‘AnyObject’i döndürür. Sonucu beklediğiniz türe dönüştürmek için as? kullanabilirsiniz.

do {
    guard let jsonData = "[\"Hello\", \"JSON\"]".dataUsingEncoding(NSUTF8StringEncoding) else {
        fatalError("couldn't encode string as UTF-8")
    }

    // Convert JSON from NSData to AnyObject
    let jsonObject = try NSJSONSerialization.JSONObjectWithData(jsonData, options: [])
    
    // Try to convert AnyObject to array of strings
    if let stringArray = jsonObject as? [String] {
        print("Got array of strings: \(stringArray.joinWithSeparator(", "))")
    }
} catch {
    print("error reading JSON: \(error)")
}

Üst düzey nesne bir dizi veya sözlük olmadığında JSON’un okunmasına izin vermek için ‘options: []’ yerine ‘options: .AllowFragments’ iletebilirsiniz.

JSON yaz #

“dataWithJSONObject” çağrısı, JSON uyumlu bir nesneyi (dizeler, sayılar ve “NSNull” içeren iç içe diziler veya sözlükler) UTF-8 olarak kodlanmış ham “NSData"ya dönüştürür.

do {
    // Convert object to JSON as NSData
    let jsonData = try NSJSONSerialization.dataWithJSONObject(jsonObject, options: [])
    print("JSON data: \(jsonData)")
    
    // Convert NSData to String
    let jsonString = String(data: jsonData, encoding: NSUTF8StringEncoding)!
    print("JSON string: \(jsonString)")
} catch {
    print("error writing JSON: \(error)")
}

Güzel baskı için ‘seçenekler: []’ yerine ‘options: .PrettyPrinted’ iletebilirsiniz.

son sürüm

Swift 3’te aynı davranış, ancak farklı bir sözdizimi ile.

do {
    guard let jsonData = "[\"Hello\", \"JSON\"]".data(using: String.Encoding.utf8) else {
        fatalError("couldn't encode string as UTF-8")
    }
    
    // Convert JSON from NSData to AnyObject
    let jsonObject = try JSONSerialization.jsonObject(with: jsonData, options: [])
    
    // Try to convert AnyObject to array of strings
    if let stringArray = jsonObject as? [String] {
        print("Got array of strings: \(stringArray.joined(separator: ", "))")
    }
} catch {
    print("error reading JSON: \(error)")
}

do {
    // Convert object to JSON as NSData
    let jsonData = try JSONSerialization.data(withJSONObject: jsonObject, options: [])
    print("JSON data: \(jsonData)")

    // Convert NSData to String
    let jsonString = String(data: jsonData, encoding: .utf8)!
    print("JSON string: \(jsonString)")
} catch {
    print("error writing JSON: \(error)")
}
son sürüm

Not: Aşağıdakiler şu anda yalnızca Swift 4.0 ve sonraki sürümlerde mevcuttur.

Swift 4.0’dan itibaren Swift standart kitaplığı, Encodeable ve Decodeable protokollerini içerir. /documentation/swift/decodeable) veri kodlama ve kod çözme için standart bir yaklaşım tanımlamak için. Bu protokollerin benimsenmesi, Encoder ve Decoder uygulamalarına izin verecektir. /decoder) protokolleri, verilerinizi alır ve JSON gibi harici bir temsile kodlar veya kodunu çözer. ‘Codable’ protokolüne uygunluk, hem ‘Encodeable’ hem de ‘Decodeable’ protokollerini birleştirir. Bu, programınızda JSON’u işlemek için önerilen yöntemdir.

Otomatik Olarak Kodlayın ve Kodunu Çözün #

Bir türü kodlanabilir yapmanın en kolay yolu, özelliklerini zaten ‘Kodlanabilir’ türler olarak bildirmektir. Bu türler, ‘String’, ‘Int’ ve ‘Double’ gibi standart kitaplık türlerini içerir; ve “Tarih”, “Veri” ve “URL” gibi Temel türleri. Bir türün özellikleri kodlanabilirse, yalnızca uygunluğu bildirerek türün kendisi otomatik olarak “Kodlanabilir” ile uyumlu olacaktır.

“Kitap” yapısının “Kodlanabilir” ile uyumlu olduğu aşağıdaki örneği düşünün.

struct Book: Codable {
    let title: String
    let authors: [String]
    let publicationDate: Date
}

“Array” ve “Dictionary” gibi standart koleksiyonların, kodlanabilir türler içermeleri durumunda “Codable” ile uyumlu olduğunu unutmayın.

“Codable” benimsenerek “Kitap” yapısı artık Apple Foundation sınıfları “JSONEncoder” ve “JSONDecoder” kullanılarak JSON’a kodlanabilir ve kodu çözülebilir, ancak “Kitap"ın kendisi JSON’u özel olarak işlemek için hiçbir kod içermese bile. Özel kodlayıcılar ve kod çözücüler de sırasıyla ‘Encoder’ ve ‘Decoder’ protokollerine uyarlanarak yazılabilir.

JSON verilerine kodlayın #

// Create an instance of Book called book
let encoder = JSONEncoder()
let data = try! encoder.encode(book) // Do not use try! in production code
print(data)

Daha kolay okuma için encoder.outputFormatting = .prettyPrinted değerini ayarlayın. ## JSON verilerinden kod çözme

JSON verilerinden kod çözme #

// Retrieve JSON string from some source
let jsonData = jsonString.data(encoding: .utf8)!
let decoder = JSONDecoder()
let book = try! decoder.decode(Book.self, for: jsonData) // Do not use try! in production code
print(book)

Yukarıdaki örnekte, “Book.self”, kod çözücüye JSON’un kodunun çözülmesi gereken türü bildirir.

Özel Olarak Kodlama veya Kod Çözme #

Bazen, örneğin bir API’den yalnızca JSON verilerini okumanız gerektiğinde veya programınız yalnızca JSON verilerini bir API’ye gönderdiğinde olduğu gibi, hem kodlanabilir hem de kodu çözülebilir verilere ihtiyacınız olmayabilir.

Yalnızca JSON verilerini yazmayı düşünüyorsanız, türünüzü ‘Encodeable’ olarak ayarlayın.

struct Book: Encodable {
    let title: String
    let authors: [String]
    let publicationDate: Date
}

Yalnızca JSON verilerini okumayı planlıyorsanız, türünüzü “Decodeable” olarak ayarlayın.

struct Book: Decodable {
    let title: String
    let authors: [String]
    let publicationDate: Date
}

Özel Anahtar Adlarını Kullanma #

API’ler, yılan durumu gibi Swift standardı deve durumu dışındaki adlandırma kurallarını sıklıkla kullanır. JSON anahtarlarının varsayılan olarak türünüzün özellik adlarıyla tam olarak hizalanması gerektiğinden, JSON kodunun çözülmesi söz konusu olduğunda bu bir sorun haline gelebilir. Bu senaryoları işlemek için ‘CodingKey’ protokolünü kullanarak türünüze özel anahtarlar oluşturabilirsiniz.

struct Book: Codable {
    // ...
    enum CodingKeys: String, CodingKey { 
        case title
        case authors
        case publicationDate = "publication_date"
    }
}

‘Kodlama Anahtarları’, ‘Kodlanabilir’ protokolü benimseyen türler için otomatik olarak oluşturulur, ancak yukarıdaki örnekte kendi uygulamamızı oluşturarak, kod çözücümüzün yerel deve durumu ‘yayınTarihi’ ile yılan durumu ‘yayın_tarihi’ni olduğu gibi eşleştirmesine izin veriyoruz. API tarafından teslim edilir.

SwiftyJSON #

SwiftyJSON, normal JSON serileştirmesinde isteğe bağlı zincirleme ihtiyacını ortadan kaldırmak için oluşturulmuş bir Swift çerçevesidir.

Buradan indirebilirsiniz: https://github.com/SwiftyJSON/SwiftyJSON

SwiftyJSON olmadan, bir JSON nesnesindeki ilk kitabın adını bulmak için kodunuz şöyle görünür:

if let jsonObject = try NSJSONSerialization.JSONObjectWithData(data, options: .AllowFragments) as? [[String: AnyObject]],
let bookName = (jsonObject[0]["book"] as? [String: AnyObject])?["name"] as? String {
    //We can now use the book name
}

SwiftyJSON’da bu oldukça basitleştirilmiştir:

let json = JSON(data: data)
if let bookName = json[0]["book"]["name"].string {
    //We can now use the book name
}

Herhangi biri geçersizse nil döndüreceğinden, her alanı kontrol etme ihtiyacını ortadan kaldırır.

SwiftyJSON’u kullanmak için Git deposundan doğru sürümü indirin - Swift 3 için bir şube var. “SwiftyJSON.swift” dosyasını projenize sürükleyin ve sınıfınıza aktarın:

import SwiftyJSON

Aşağıdaki iki başlatıcıyı kullanarak JSON nesnenizi oluşturabilirsiniz:

let jsonObject = JSON(data: dataObject)

veya

let jsonObject = JSON(jsonObject) //This could be a string in a JSON format for example

Verilerinize erişmek için abonelikleri kullanın:

let firstObjectInAnArray = jsonObject[0]
let nameOfFirstObject = jsonObject[0]["name"]

Ardından değerinizi, isteğe bağlı bir değer döndürecek olan belirli bir veri türüne ayrıştırabilirsiniz:

let nameOfFirstObject = jsonObject[0]["name"].string //This will return the name as a string

let nameOfFirstObject = jsonObject[0]["name"].double //This will return null

Ayrıca yollarınızı hızlı bir Dizide derleyebilirsiniz:

let convolutedPath = jsonObject[0]["name"][2]["lastName"]["firstLetter"].string

Aynıdır:

let convolutedPath = jsonObject[0, "name", 2, "lastName", "firstLetter"].string

SwiftyJSON ayrıca kendi hatalarını yazdırma işlevine de sahiptir:

if let name = json[1337].string {
    //You can use the value - it is valid
} else {
    print(json[1337].error) // "Array[1337] is out of bounds" - You cant use the value
}

JSON nesnenize yazmanız gerekirse, abonelikleri tekrar kullanabilirsiniz:

var originalJSON:JSON = ["name": "Jack", "age": 18]
originalJSON["age"] = 25 //This changes the age to 25
originalJSON["surname"] = "Smith" //This creates a new field called "surname" and adds the value to it

JSON için orijinal Dize’ye ihtiyacınız varsa, örneğin onu bir dosyaya yazmanız gerekiyorsa, ham değeri alabilirsiniz:

if let string = json.rawString() { //This is a String object
    //Write the string to a file if you like
}

if let data = json.rawData() { //This is an NSData object
    //Send the data to your server if you like
}

Freddy #

Freddy, Big Nerd Ranch tarafından sağlanan bir JSON ayrıştırma kitaplığıdır. Üç temel faydası vardır:

  1. Tip Güvenliği: Çalışma zamanı çökmelerini önleyecek şekilde JSON gönderip alma konusunda çalışmanıza yardımcı olur.

  2. Deyimsel: Karmaşık belgeler veya büyülü özel operatörler olmadan Swift’in jeneriklerinden, numaralandırmalarından ve işlevsel özelliklerinden yararlanır.

  3. Hata İşleme: Yaygın olarak oluşan JSON hataları için bilgilendirici hata bilgileri sağlar.

Bu örneklerle kullanmak için bazı örnek JSON verilerini tanımlayalım.

{
  "success": true,
  "people": [
    {
      "name": "Matt Mathias",
      "age": 32,
      "spouse": true
    },
    {
      "name": "Sergeant Pepper",

    "teacher",

  ],

    "Georgia": [
      30301,
      30302,
      30303
    ],

      53000,

let jsonString = "{\"success\": true, \"people\": [{\"name\": \"Matt Mathias\",\"age\": 32,\"spouse\": true},{\"name\": \"Sergeant Pepper\",\"age\": 25,\"spouse\": false}],\"jobs\": [\"teacher\",\"judge\"],\"states\": {\"Georgia\": [30301,30302,30303],\"Wisconsin\": [53000,53001]}}"
let jsonData = jsonString.dataUsingEncoding(NSUTF8StringEncoding)!

Ham Verileri Seri durumdan Çıkarma #

Verileri seri durumdan çıkarmak için bir “JSON” nesnesini başlatırız ve ardından belirli bir anahtara erişiriz.

do {
    let json = try JSON(data: jsonData)
    let success = try json.bool("success")
} catch {
    // do something with the error
}

Burada “deniyoruz”, çünkü “başarı” anahtarı için “json"a erişim başarısız olabilir - bu olmayabilir veya değer bir boole olmayabilir.

JSON yapısında yuvalanmış öğelere erişmek için bir yol da belirtebiliriz. Yol, ilgilenilen bir değere giden yolu tanımlayan virgülle ayrılmış bir anahtar ve dizin listesidir.

do {
    let json = try JSON(data: jsonData)
    let georgiaZipCodes = try json.array("states", "Georgia")
    let firstPersonName = try json.string("people", 0, "name")
} catch {
    // do something with the error
}

Modelleri Doğrudan Seriden Çıkarma #

JSON, “JSONDecodeable” protokolünü uygulayan bir model sınıfına doğrudan ayrıştırılabilir.

public struct Person {
    public let name: String
    public let age: Int
    public let spouse: Bool
}

extension Person: JSONDecodable {
    public init(json: JSON) throws {
        name = try json.string("name")
        age = try json.int("age")
        spouse = try json.bool("spouse")
    }
}

do {
    let json = try JSON(data: jsonData)
    let people = try json.arrayOf("people", type: Person.self)
} catch {
    // do something with the error
}

Ham Verileri Serileştirme #

Herhangi bir “JSON” değeri doğrudan “NSData"ya serileştirilebilir.

let success = JSON.Bool(false)
let data: NSData = try success.serialize()

Modelleri Doğrudan Serileştirme #

“JSONEncodable” protokolünü uygulayan herhangi bir model sınıfı, doğrudan “NSData"ya serileştirilebilir.

extension Person: JSONEncodable {
    public func toJSON() -> JSON {
        return .Dictionary([
            "name": .String(name),
            "age": .Int(age),
            "spouse": .Bool(spouse)
        ])
    }
}

let newPerson = Person(name: "Glenn", age: 23, spouse: true)
let data: NSData = try newPerson.toJSON().serialize()

Özel nesnelere basit JSON ayrıştırma #

Üçüncü taraf kitaplıkları iyi olsa bile, JSON’u ayrıştırmanın basit bir yolu protokoller tarafından sağlanır. ‘Yapılacak’ bir nesneniz olduğunu hayal edebilirsiniz.

struct Todo {
    let comment: String
}

JSON’u her aldığınızda, diğer örnekte gösterildiği gibi ‘NSJSONSerialization’ nesnesini kullanarak düz ‘NSData’yı işleyebilirsiniz.

Bundan sonra, basit bir protokol olan JSONDecodeable kullanarak

typealias JSONDictionary = [String:AnyObject]
protocol JSONDecodable {
    associatedtype Element
    static func from(json json: JSONDictionary) -> Element?
}

Ve ‘Yapılacak’ yapınızı ‘JSONDecodable’a uygun hale getirmek hile yapar

extension Todo: JSONDecodable {
    static func from(json json: JSONDictionary) -> Todo? {
        guard let comment = json["comment"] as? String else { return nil }
        return Todo(comment: comment)
    }
}

Bu json koduyla deneyebilirsiniz:

{
    "todos": [
        {
            "comment" : "The todo comment"
        }
    ]
}

API’den aldığınızda, bir “AnyObject” örneğinde gösterilen önceki örnekler gibi seri hale getirebilirsiniz. Bundan sonra, örneğin bir “JSONDictionary” örneği olup olmadığını kontrol edebilirsiniz.

guard let jsonDictionary = dictionary as? JSONDictionary else { return }

JSON’da bir ‘Yapılacaklar’ dizisine sahip olduğunuz için bu duruma özel olarak kontrol edilecek diğer şey, ’todos’ sözlüğüdür.

guard let todosDictionary = jsonDictionary["todos"] as? [JSONDictionary] else { return }

Artık sözlük dizisine sahip olduğunuza göre, her birini bir “Yapılacaklar” nesnesine “flatMap” kullanarak dönüştürebilirsiniz (diziden “nil” değerlerini otomatik olarak siler)

let todos: [Todo] = todosDictionary.flatMap { Todo.from(json: $0) }

JSON Ayrıştırma Swift 3 #

Ok #

Ok Swift’de zarif bir JSON ayrıştırma kitaplığıdır.

JSON’un ayrıştırılmasına ve bir “<–” operatörü yardımıyla özel model sınıflarına eşlenmesine izin verir:

identifier <-- json["id"]
name <-- json["name"]
stats <-- json["stats"]

Örnek:

hızlı model

struct Profile {
    var identifier = 0
    var name = ""
    var link: NSURL?
    var weekday: WeekDay = .Monday
    var stats = Stats()
    var phoneNumbers = [PhoneNumber]()
}

JSON dosyası

{
    "id": 15678,
    "name": "John Doe",
    "link": "https://apple.com/steve",
    "weekdayInt" : 3,
    "stats": {
        "numberOfFriends": 163,
        "numberOfFans": 10987
    },
    "phoneNumbers": [{
                     "label": "house",
                     "number": "9809876545"
                     }, {
                     "label": "cell",
                     "number": "0908070656"
                     }, {
                     "label": "work",
                     "number": "0916570656"
    }]
}

haritalama

extension Profile: ArrowParsable {
    mutating func deserialize(json: JSON) {
        identifier <-- json["id"]
        link <-- json["link"]
        name <-- json["name"]
        weekday <-- json["weekdayInt"]
        stats <- json["stats"]
        phoneNumbers <-- json["phoneNumbers"]
    }
}

kullanım

let profile = Profile()
profile.deserialize(json)
