Swift 2.0’da kaldırılan “println” ve “debugPrintln”.

Kaynaklar:

https://developer.apple.com/library/content/technotes/tn2347/_index.html http://ericasadun.com/2015/05/22/swift-logging/

http://www.dotnetperls.com/print-swift

çöplük #

‘dump’, bir nesnenin içeriğini yansıma (yansıtma) yoluyla yazdırır.

Bir dizinin ayrıntılı görünümü:

let names = ["Joe", "Jane", "Jim", "Joyce"]
dump(names)

Baskılar:

▿ 4 element - [0]: Joe - [1]: Jane - [2]: Cem - [3]: Joyce

bir sözlük için:

let attributes = ["foo": 10, "bar": 33, "baz": 42]
dump(attributes)

Baskılar:

▿ 3 anahtar/değer çifti ▿ [0]: (2 eleman)

  • .0: çubuk
  • .1: 33 ▿ [1]: (2 eleman)
  • .0: baz
  • .1: 42 ▿ [2]: (2 eleman)
  • .0: foo
  • .1: 10

‘dump’, ‘dump(_:name:indent:maxDepth:maxItems:)’ olarak bildirildi.

İlk parametrenin etiketi yok.

İncelenmekte olan nesne için bir etiket ayarlamak için “ad” gibi başka parametreler de mevcuttur:

dump(attributes, name: "mirroring")

Baskılar:

▿ yansıtma: 3 anahtar/değer çifti ▿ [0]: (2 eleman)

  • .0: çubuk
  • .1: 33 ▿ [1]: (2 eleman)
  • .0: baz
  • .1: 42 ▿ [2]: (2 eleman)
  • .0: foo
  • .1: 10

Ayrıca ‘maxItems:’ ile yalnızca belirli sayıda öğeyi yazdırmayı, ‘maxDepth:’ ile nesneyi belirli bir derinliğe kadar ayrıştırmayı ve ‘indent:’ ile yazdırılan nesnelerin girintisini değiştirmeyi seçebilirsiniz.

Hata Ayıklama Yazdırma #

Hata Ayıklama Yazdırma, hata ayıklama için en uygun örnek temsilini gösterir.

print("Hello")
debugPrint("Hello")

let dict = ["foo": 1, "bar": 2]

print(dict)
debugPrint(dict)

verim

>>> Hello
>>> "Hello"
>>> [foo: 1, bar: 2]
>>> ["foo": 1, "bar": 2]

Bu ekstra bilgi çok önemli olabilir, örneğin:

let wordArray = ["foo", "bar", "food, bars"]

print(wordArray)
debugPrint(wordArray)

verim

>>> [foo, bar, food, bars]
>>> ["foo", "bar", "food, bars"]

İlk çıktıda dizide 3 yerine 4 eleman olduğuna dikkat edin. Bunun gibi nedenlerle, hata ayıklama sırasında ‘debugPrint’ kullanılması tercih edilir.

Sınıf hata ayıklama ve yazdırma değerlerinin güncellenmesi #

struct Foo: Printable, DebugPrintable {
    var description: String {return "Clear description of the object"}
    var debugDescription: String {return "Helpful message for debugging"}
}

var foo = Foo()

print(foo)
debugPrint(foo)

>>> Clear description of the object
>>> Helpful message for debugging

print() vs dökümü() #

Çoğumuz hata ayıklamaya basit print() ile başlarız. Diyelim ki böyle bir sınıfımız var:

class Abc {
    let a = "aa"
    let b = "bb"
}

ve şöyle bir ‘Abc’ örneğine sahibiz:

let abc = Abc()

Değişken üzerinde print() çalıştırdığımızda çıktı

App.Abc

dump() çıktıları iken

App.Abc #0
- a: "aa"
- b: "bb"

Görüldüğü gibi, “dump()” tüm sınıf hiyerarşisini verir, “print()” ise sadece sınıf adını verir.

Bu nedenle, “dump()” özellikle UI hata ayıklaması için kullanışlıdır.

let view = UIView(frame: CGRect(x: 0, y: 0, width: 100, height: 100))

dump(view) ile şunu elde ederiz:

 - <UIView: 0x108a0cde0; frame = (0 0; 100 100); layer = <CALayer: 0x159340cb0>> #0
    - super: UIResponder
      - NSObject

print(view) iken şunu elde ederiz:

<UIView: 0x108a0cde0; frame = (0 0; 100 100); layer = <CALayer: 0x159340cb0>>

dump() ile sınıf hakkında daha fazla bilgi vardır ve bu nedenle sınıfın hatalarını ayıklamak için daha kullanışlıdır.

print vs NSLog #

Swift’de Xcode konsolunda bir şey yazdırmak için hem print() hem de NSLog() işlevlerini kullanabiliriz.

Ancak print() ve NSLog() işlevlerinde pek çok farklılık vardır, örneğin:

1 Zaman Damgası: ‘NSLog()’, kendisine ilettiğimiz dizeyle birlikte zaman damgasını yazdırır, ancak ‘yazdır()’ zaman damgasını yazdırmaz. Örneğin.

let array = [1, 2, 3, 4, 5]
print(array)
NSLog(array.description)

Çıktı:

[1, 2, 3, 4, 5] 2017-05-31 13:14:38.582 ProjeAdı[2286:7473287] [1, 2, 3, 4, 5]

Ayrıca zaman damgasıyla birlikte ProjeAdı da yazdırılır.

2 Only String: ‘NSLog()’ yalnızca String’i girdi olarak alır, ancak ‘print()’ kendisine iletilen her tür girdiyi yazdırabilir. Örneğin.

let array = [1, 2, 3, 4, 5]
print(array) //prints [1, 2, 3, 4, 5]
NSLog(array) //error: Cannot convert value of type [Int] to expected argument type 'String'

3 Performans: NSLog() işlevi, print() işlevine kıyasla çok yavaş.

4 Senkronizasyon: NSLog(), çoklu iş parçacığı ortamından eşzamanlı kullanımı yönetir ve çıktıyı çakıştırmadan yazdırır. Ancak print() çıktıyı değerlendirirken bu tür durumları ve karışıklıkları işlemez.

5 Cihaz Konsolu: Cihaz konsolunda NSLog() çıkışları da vardır, bu çıkışı cihazımızı Xcode’a bağlayarak görebiliriz. print() çıktıyı aygıtın konsoluna yazdırmaz.
