C başlıklarını içe aktarmak için bir modül haritası kullanın #

Bir modül haritası, onu C başlık dosyalarını okuyacak ve onları Swift olarak gösterecek şekilde yapılandırarak basitçe “modülümü içe aktarabilir” fonksiyonlar.

“module.modulemap” adlı bir dosyayı “mymodule” adlı bir dizine yerleştirin:

Modül harita dosyasının içinde:

// mymodule/module.modulemap
module mymodule {
    header "defs.h"
}

Ardından modülü “içe aktarın”:

// demo.swift
import mymodule
print("Empty color: \(Color())")

swiftc modülüne modülü nerede bulacağını söylemek için -I dizin işaretini kullanın:

swiftc -I . demo.swift   # "-I ." means "search for modules in the current directory"

Modül eşleme sözdizimi hakkında daha fazla bilgi için Modül eşlemeleri hakkında Clang dokümantasyonu bölümüne bakın.

Swift kodundan Objective-C sınıflarını kullanma #

MyFramework, genel başlıklarında (ve şemsiye başlığında) Objective-C sınıfları içeriyorsa, bunları Swift’den kullanmak için gereken tek şey MyFramework'ü içe aktar.

Başlıkları köprüleme #

Bir köprü başlığı, Swift koduna ek Objective-C ve C bildirimleri görünür hale getirir. Proje dosyalarını eklerken, Xcode otomatik olarak bir köprüleme başlığı oluşturmayı önerebilir:

Manuel olarak bir tane oluşturmak için Objective-C Bridging Header oluşturma ayarını değiştirin:

Köprüleme başlığının içinde, koddan kullanmak için gerekli olan dosyaları içe aktarın:

// MyApp-Bridging-Header.h
#import "MyClass.h"  // allows code in this module to use MyClass

Oluşturulan Arayüz #

İlgili Öğeler düğmesini tıklayın (veya ⌃1’e basın), ardından bir Objective-C başlığından oluşturulacak Swift arayüzünü görmek için Oluşturulan Arayüz‘ü seçin.

    

swiftc için bir köprüleme başlığı belirtin #

“-import-objc-header” bayrağı, “swiftc"nin içe aktarılacağı bir başlık belirtir:

// defs.h
struct Color {
    int red, green, blue;
};

#define MAX_VALUE 255

// demo.swift
extension Color: CustomStringConvertible {  // extension on a C struct
    public var description: String {
        return "Color(red: \(red), green: \(green), blue: \(blue))"
    }
}
print("MAX_VALUE is: \(MAX_VALUE)")  // C macro becomes a constant
let color = Color(red: 0xCA, green: 0xCA, blue: 0xD0)  // C struct initializer
print("The color is \(color)")

$ swiftc demo.swift -import-objc-header defs.h && ./demo
MAX_VALUE: 255

Objective-C ve Swift arasında ayrıntılı birlikte çalışma #

Bir API ‘NS_REFINED_FOR_SWIFT’ ile işaretlendiğinde, Swift’e aktarıldığında önüne iki alt çizgi (’__’) eklenir:

@interface Sınıfım: NSObject
- (NSInteger)indexOfObject:(id)obj NS_REFINED_FOR_SWIFT;
@end

[Oluşturulan arayüz](https://www.wikiod.com/tr/swift/c-ve-objective-c-ile-calsma#Swift kodundan Objective-C sınıflarını kullanma 201605140416386495969) şöyle görünür:

genel sınıf Sınıfım : NSObject {
    public func __indexOfObject(obj: AnyObject) -> Int
}

Artık API’yi daha “Swifty” bir uzantıyla değiştirebilirsiniz. Bu durumda, NSNotFound filtreleyerek bir isteğe bağlı dönüş değeri kullanabiliriz /library/mac/documentation/Cocoa/Reference/Foundation/Miscellaneous/Foundation_Constants/#//apple_ref/doc/constant_group/NSNotFound):

uzantısı Sınıfım {
    // Rather than returning NSNotFound if the object doesn't exist,
    // this "refined" API returns nil.
    func indexOfObject(obj: AnyObject) -> Int? {
        let idx = __indexOfObject(obj)
        if idx == NSNotFound { return nil }
        return idx
    }
}

// Swift kodu, olması gerektiği gibi "if let" kullanarak:
izin ver myobj = MyClass()
izin ver idx = myobj.indexOfObject(bir şey) {
    // do something with idx
}

Çoğu durumda, bir Objective-C işlevine yönelik bir argümanın “nil” olup olmayacağını kısıtlamak isteyebilirsiniz. Bu, herhangi bir işaretçi veya blok referansını niteleyen ‘_Nonnull’ anahtar sözcüğü kullanılarak yapılır:

geçersiz
doStuff(const void *const _Nonnull veri, void (^_Nonnull tamamlama)())
{
    // complex asynchronous code
}

Bu yazıldığında, Swift kodumuzdan o fonksiyona ’nil’ iletmeye çalıştığımızda derleyici bir hata yayacaktır:

doStuff(
    nil,  // error: nil is not compatible with expected argument type 'UnsafeRawPointer'
    nil)  // error: nil is not compatible with expected argument type '() -> Void'

‘_Nonnull’un tersi ‘_Nullable’dır; bu, bu argümanda ’nil’ iletmenin kabul edilebilir olduğu anlamına gelir. ‘_Nullable’ da varsayılandır; ancak, açıkça belirtilmesi, daha fazla kendi kendini belgeleyen ve geleceğe yönelik kodlara izin verir.

Derleyicinin kodunuzu optimize etmesine daha fazla yardımcı olmak için, bloğun kaçıp kaçmadığını da belirtmek isteyebilirsiniz:

geçersiz
callNow(__attribute__((noescape)) geçersiz (^_Nonnull f)())
{
    // f is not stored anywhere
}

Bu öznitelikle, blok referansını kaydetmeyeceğimize ve fonksiyon yürütmeyi bitirdikten sonra bloğu çağırmayacağımıza söz veriyoruz.

C standart kitaplığını kullanın #

Swift’in C birlikte çalışabilirliği, C standart kitaplığındaki işlevleri ve türleri kullanmanıza olanak tanır.

Linux’ta, C standart kitaplığı ‘Glibc’ modülü aracılığıyla sunulur; Apple platformlarında buna “Darwin” denir.

#if os(macOS) || os(iOS) || os(tvOS) || os(watchOS)
import Darwin
#elseif os(Linux)
import Glibc
#endif

// use open(), read(), and other libc features

Objective-C kodundan Swift sınıflarını kullanma #

Aynı modülde #

“MyModule” adlı bir modülün içinde, Xcode, genel Swift sınıflarını Objective-C’ye sunan MyModule-Swift.h adlı bir başlık oluşturur. Swift sınıflarını kullanmak için bu başlığı içe aktarın:

// MyApp'de MySwiftClass.swift
ithalat Vakfı

// Bu hedefte ayrıca bir köprüleme başlığı yoksa, sınıfın görünür olması için "genel" olması gerekir
genel sınıfı MySwiftClass: NSObject {
    // ...
}
// MyApp'te MyViewController.m

#import "MyViewController.h"
#import "MyApp-Swift.h" // oluşturulan arayüzü içe aktarın
#import <MyFramework/MyFramework-Swift.h> // veya bir çerçeve hedefi için açılı ayraçlar kullanın

@implementation MyViewController
- (geçersiz)demo {
    [[MySwiftClass alloc] init];  // use the Swift class
}
@end

İlgili yapı ayarları:

  • Objective-C Oluşturulan Arabirim Başlık Adı: oluşturulan Obj-C başlığının adını kontrol eder.
  • Hedef-C Uyumluluk Başlığını Yükle: -Swift.h üstbilgisinin genel bir üstbilgi olup olmayacağı (çerçeve hedefleri için).

Başka bir modülde #

@import MyFramework; kullanımı, Obj-C arabirimleri dahil olmak üzere tüm modülü Swift sınıflarına aktarır (yukarıda belirtilen derleme ayarı etkinse).
