Hata işlemeyle ilgili temel bilgiler #

Swift’deki işlevler değerler, atma hatalar veya her ikisini birden döndürebilir:

func reticulateSplines() // dönüş değeri yok ve hata yok
func reticulateSplines() -> Int // her zaman bir değer döndürür
func reticulateSplines() atar // dönüş değeri yok, ancak hata verebilir
func reticulateSplines() throws -> Int // ya bir değer döndürebilir ya da bir hata verebilir

ErrorType protokolüne uyan herhangi bir değer (NSError nesneleri dahil) bir hata olarak atılabilir. Numaralandırmalar özel hataları tanımlamanın uygun bir yolunu sağlar:

enum NetworkError: ErrorType {
    case Offline
    case ServerError(String)
}
son sürüm 
enum NetworkError: Error {
    // Swift 3 dictates that enum cases should be `lowerCamelCase`
    case offline
    case serverError(String)
}
son sürüm

Bir hata, program yürütme sırasında önemli olmayan bir başarısızlığı belirtir ve “do”/“catch”, “throw” ve “try” özel denetim akışı yapıları ile işlenir.

func fetchResource(kaynak: NSURL) atmalar -> Dize {
    if let (statusCode, responseString) = /* ...from elsewhere...*/ {
        if case 500..<600 = statusCode {
            throw NetworkError.serverError(responseString)
        } else {
            return responseString
        }
    } else {
        throw NetworkError.offline
    }
}

Hatalar do/catch ile yakalanabilir:

yapmak {
    let response = try fetchResource(resURL)
    // If fetchResource() didn't throw an error, execution continues here:
    print("Got response: \(response)")
    ...
} yakalamak {
    // If an error is thrown, we can handle it here.
    print("Whoops, couldn't fetch resource: \(error)")
}

Hata verebilecek herhangi bir işlev ****** ’try’, ’try?’ veya ’try!’ kullanılarak çağrılmalıdır:

// hata: çağrı atılabilir ancak 'dene' ile işaretlenmemiş
cevap ver = fetchResource(resURL)

// "try", do/catch içinde veya başka bir fırlatma işlevi içinde çalışır:
yap {
    let response = try fetchResource(resURL)
} yakalamak {
    // Handle the error
}

func foo() atar {
    // If an error is thrown, continue passing it up to the caller.
    let response = try fetchResource(resURL)
}

// "denemek?" işlevin dönüş değerini İsteğe bağlı olarak sarar (bir hata oluştuysa sıfır).
izin ver yanıt = denesin mi? fetchResource(resURL) {
    // no error was thrown
}

// "denemek!" bir hata oluşursa programı çalışma zamanında çöker.
let yanıtı = deneyin! fetchResource(resURL)

Farklı hata türlerini yakalama #

Bu örnek için kendi hata türümüzü oluşturalım.

enum CustomError: ErrorType {
    case SomeError
    case AnotherError
}

func throwing() throws {
    throw CustomError.SomeError
}
son sürüm 
enum CustomError: Error {
    case someError
    case anotherError
}

func throwing() throws {
    throw CustomError.someError
}
son sürüm

Do-Catch sözdizimi, atılan bir hatayı yakalamaya izin verir ve otomatik olarak, “catch” bloğunda bulunan “hata” adlı bir sabit oluşturur:

do {
    try throwing()
} catch {
    print(error)
}

Bir değişkeni kendiniz de bildirebilirsiniz:

do {
    try throwing()
} catch let oops {
    print(oops)
}

Farklı “catch” ifadelerini zincirlemek de mümkündür. Bu, Do bloğuna birkaç tür hata atılabiliyorsa kullanışlıdır.

Burada Do-Catch önce hatayı bir ‘CustomError’ olarak, ardından özel tip eşleşmediyse ‘NSError’ olarak yayınlamaya çalışacaktır.

do {
    try somethingMayThrow()
} catch let custom as CustomError {
    print(custom)
} catch let error as NSError {
    print(error)
}
son sürüm

Swift 3’te, açıkça NSError’a indirmeye gerek yoktur.

do {
    try somethingMayThrow()
} catch let custom as CustomError {
    print(custom)
} catch {
    print(error)
}
son sürüm

Açık Hata İşleme için Modeli Yakala ve Değiştir #

class Plane {
    
    enum Emergency: ErrorType {
        case NoFuel
        case EngineFailure(reason: String)
        case DamagedWing
    }

    var fuelInKilograms: Int

    //... init and other methods not shown

    func fly() throws {
        // ...
        if fuelInKilograms <= 0 {
            // uh oh...
            throw Emergency.NoFuel
        }
    }

}

İstemci sınıfında:

let airforceOne = Plane()
do {
    try airforceOne.fly()
} catch let emergency as Plane.Emergency {
    switch emergency {
    case .NoFuel:
        // call nearest airport for emergency landing
    case .EngineFailure(let reason):
        print(reason) // let the mechanic know the reason
    case .DamagedWing:
        // Assess the damage and determine if the president can make it
    }
}

Hata Yayılımını Devre Dışı Bırakma #

Swift’in yaratıcıları, dili anlamlı hale getirmeye çok dikkat ettiler ve hata işleme tam olarak bu, etkileyici. Hata verebilecek bir işlevi çağırmaya çalışırsanız, işlev çağrısından önce try anahtar sözcüğü gelmelidir. try anahtar sözcüğü sihirli değildir. Tek yaptığı, geliştiriciyi işlevin fırlatma yeteneğinden haberdar etmektir.

Örneğin, aşağıdaki kod, görüntü kaynağını belirli bir yola yükleyen veya görüntü yüklenemezse bir hata veren loadImage(atPath:) işlevini kullanır. Bu durumda imaj uygulama ile birlikte gönderildiği için çalışma zamanında herhangi bir hata atılmayacağı için hata yayılımını devre dışı bırakmak uygundur.

let photo = try! loadImage(atPath: "./Resources/John Appleseed.jpg")

Yerelleştirilmiş açıklama ile özel Hata oluşturun #

Özel hataların “sayısını” oluşturun

enum RegistrationError: Error {
    case invalidEmail
    case invalidPassword
    case invalidPhoneNumber
}

Yerelleştirilmiş açıklamayı işlemek için “RegistrationError"un “uzantısını” oluşturun.

extension RegistrationError: LocalizedError {
    public var errorDescription: String? {
        switch self {
        case .invalidEmail:
            return NSLocalizedString("Description of invalid email address", comment: "Invalid Email")
        case .invalidPassword:
            return NSLocalizedString("Description of invalid password", comment: "Invalid Password")
        case .invalidPhoneNumber:
            return NSLocalizedString("Description of invalid phoneNumber", comment: "Invalid Phone Number")
        }
    }
}

İşlem hatası:

let error: Error = RegistrationError.invalidEmail
print(error.localizedDescription)
