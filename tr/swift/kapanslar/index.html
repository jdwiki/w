Sözdizimi #

  • var closureVar: (<parameters>) -> (<returnType>) // Değişken veya özellik türü olarak
  • typealias ClosureType = (<parameters>) -> (<returnType>)
  • { [<captureList>] (<parameters>) <throws-ness> -> <returnType> in <ifadeler> } // Kapatma sözdizimini tamamlayın

Swift kapatma işlemleri hakkında daha fazla bilgi için Apple’ın belgelerine bakın.

Kapatmayla ilgili temel bilgiler #

Kapanışlar (bloklar veya lambdalar olarak da bilinir) programınızın içinde saklanabilen ve dolaşabilen kod parçalarıdır.

let sayHi = { print("Hello") }
// The type of sayHi is "() -> ()", aka "() -> Void"

sayHi()  // prints "Hello"

Diğer işlevler gibi, kapanışlar da argümanları kabul edebilir ve sonuçları döndürebilir veya [hatalar] atabilir(https://www.wikiod.com/tr/swift/hata-yonetimi):

let addInts = { (x: Int, y: Int) -> Int in
    return x + y
}
// The type of addInts is "(Int, Int) -> Int"

let result = addInts(1, 2)  // result is 3

let divideInts = { (x: Int, y: Int) throws -> Int in
    if y == 0 {
        throw MyErrors.DivisionByZero
    }
    return x / y
}
// The type of divideInts is "(Int, Int) throws -> Int"

Kapanışlar, kapsamlarından değerleri yakalayabilir:

// This function returns another function which returns an integer
func makeProducer(x: Int) -> (() -> Int) {
    let closure = { x }  // x is captured by the closure
    return closure
}

// These two function calls use the exact same code,
// but each closure has captured different values.
let three = makeProducer(3)
let four = makeProducer(4)
three()  // returns 3
four()  // returns 4

Kapanışlar doğrudan işlevlere geçirilebilir:

let squares = (1...10).map({ $0 * $0 })  // returns [1, 4, 9, 16, 25, 36, 49, 64, 81, 100]
let squares = (1...10).map { $0 * $0 }

NSURLSession.sharedSession().dataTaskWithURL(myURL,
    completionHandler: { (data: NSData?, response: NSURLResponse?, error: NSError?) in
        if let data = data {
            print("Request succeeded, data: \(data)")
        } else {
            print("Request failed: \(error)")
        }
    }).resume()

Sözdizimi varyasyonları #

Temel kapatma sözdizimi

{ [yakalama listesi] (parametreler) atmalar -> dönüş türü in gövde }.

Bu parçaların çoğu atlanabilir, bu nedenle basit kapanışları yazmanın birkaç eşdeğer yolu vardır:

let addOne = { [] (x: Int) -> Int in return x + 1 }
let addOne = { [] (x: Int) -> Int in x + 1 }
let addOne = { (x: Int) -> Int in x + 1 }
let addOne = { x -> Int in x + 1 }
let addOne = { x in x + 1 }
let addOne = { $0 + 1 }

let addOneOrThrow = { [] (x: Int) throws -> Int in return x + 1 }
let addOneOrThrow = { [] (x: Int) throws -> Int in x + 1 }
let addOneOrThrow = { (x: Int) throws -> Int in x + 1 }
let addOneOrThrow = { x throws -> Int in x + 1 }
let addOneOrThrow = { x throws in x + 1 }

  • Boşsa yakalama listesi atlanabilir.

  • Türleri çıkarılabiliyorsa, parametrelerin tür açıklamalarına ihtiyacı yoktur.

  • Çıkarılabilirse, dönüş türünün belirtilmesine gerek yoktur.

  • Parametrelerin adlandırılması gerekmez; bunun yerine “$0”, “$1”, “$2” vb. ile anılabilirler.

  • Kapatma, değeri döndürülecek tek bir ifade içeriyorsa, ‘return’ anahtar sözcüğü atlanabilir.

  • Kapatmanın bir hata oluşturduğu anlaşılırsa, bir atma kapanışı bekleyen veya bir hata atmayan bir bağlamda yazılırsa, “atmalar” atlanabilir.

    // The closure’s type is unknown, so we have to specify the type of x and y. // The output type is inferred to be Int, because the + operator for Ints returns Int. let addInts = { (x: Int, y: Int) in x + y }

    // The closure’s type is specified, so we can omit the parameters’ type annotations. let addInts: (Int, Int) -> Int = { x, y in x + y } let addInts: (Int, Int) -> Int = { $0 + $1 }

Kapanışları işlevlere geçirme #

İşlevler, kapanışları (veya diğer işlevleri) parametre olarak kabul edebilir:

func foo(value: Double, block: () -> Void) { ... }
func foo(value: Double, block: Int -> Int) { ... }
func foo(value: Double, block: (Int, Int) -> String) { ... }

Sondaki kapatma sözdizimi #

Bir işlevin son parametresi bir kapatma ise, kapatma parantezleri {/} işlev çağrısından sonra yazılabilir:

foo(3.5, block: { print("Hello") })

foo(3.5) { print("Hello") }

dispatch_async(dispatch_get_main_queue(), {
    print("Hello from the main queue")
})

dispatch_async(dispatch_get_main_queue()) {
    print("Hello from the main queue")
}

Bir işlevin tek argümanı bir kapatma ise, onu sondaki kapatma sözdizimi ile çağırırken () parantez çiftini de atlayabilirsiniz:

func bar(block: () -> Void) { ... }


bar() { print("Hello") }

bar { print("Hello") }

@noescape parametreleri #

“@noescape” olarak işaretlenen kapatma parametrelerinin, işlev çağrısı geri dönmeden önce yürütülmesi garanti edilir, bu nedenle, kapatma gövdesi içinde “self” kullanılması gerekmez:

func executeNow(@noescape bloğu: () -> Void) {
    // Since `block` is @noescape, it's illegal to store it to an external variable.
    // We can only call it right here.
    block()
}

func executeLater(block: () -> Void) {
    dispatch_async(dispatch_get_main_queue()) {
        // Some time in the future...
        block()
    }
}
class MyClass {
    var x = 0
    func showExamples() {
        // error: reference to property 'x' in closure requires explicit 'self.' to make capture semantics explicit
        executeLater { x = 1 }

        executeLater { self.x = 2 }  // ok, the closure explicitly captures self

        // Here "self." is not required, because executeNow() takes a @noescape block.
        executeNow { x = 3 }

        // Again, self. is not required, because map() uses @noescape.
        [1, 2, 3].map { $0 + x }
    }
}

Swift 3 notu: #

Swift 3’te artık blokları @noescape olarak işaretlemediğinizi unutmayın. Bloklar artık varsayılan olarak kaçmıyor**. Swift 3’te, bir kapanışı kaçmayan olarak işaretlemek yerine, “@escaping” anahtar sözcüğünü kullanarak kaçan bir kapanış olan bir işlev parametresini çıkış olarak işaretlersiniz.

“atar” ve “tekrar atar” #

Kapanışlar, diğer işlevler gibi, hatalar verebilir:

func executeNowOrIgnoreError(block: () atmalar -> Void) {
    do {
        try block()
    } catch {
        print("error: \(error)")
    }
}

İşlev, elbette, hatayı arayana iletebilir:

func executeNowOrThrow(blok: () atmalar -> Void) atmalar {
    try block()
}

Ancak, iletilen blok atmaz ise, arayan kişi yine de bir fırlatma işlevine takılıp kalır:

// It's annoying that this requires "try", because "print()" can't throw!
try executeNowOrThrow { print("Just printing, no errors here!") }

Çözüm rethrows şeklindedir; bu, işlevin yalnızca eğer kapatma parametresi atarsa atabileceğini belirtir:

func executeNowOrRethrow(block: () atlar -> Void) yeniden çeker {
    try block()
}

// "try" burada gerekli değildir, çünkü blok hata atamaz.
executeNowOrRethrow { print("Bu kapatmadan hiçbir hata atılmaz") }

// Bu blok bir hata yapabilir, bu nedenle "denemek" gereklidir.
executeNowOrRethrow { MyError.Example fırlatmayı }
deneyin

Birçok standart kitaplık işlevi, ‘map()’, ‘filter()’ ve ‘indexOf()’ dahil olmak üzere ‘rethrows’ kullanır.

Yakalar, güçlü/zayıf referanslar ve döngüleri korur #

class MyClass {
    func sayHi() { print("Hello") }
    deinit { print("Goodbye") }
}

Bir kapatma bir referans türünü (bir sınıf örneği) yakaladığında, varsayılan olarak güçlü bir referansı tutar:

let closure: () -> Void
do {
    let obj = MyClass()
    // Captures a strong reference to `obj`: the object will be kept alive
    // as long as the closure itself is alive.
    closure = { obj.sayHi() }
    closure()  // The object is still alive; prints "Hello"
} // obj goes out of scope
closure()  // The object is still alive; prints "Hello"

Kapatmanın yakalama listesi, zayıf veya sahipsiz bir referans belirtmek için kullanılabilir:

kapatalım: () -> Geçersiz
yapmak {
    let obj = MyClass()
    // Captures a weak reference to `obj`: the closure will not keep the object alive;
    // the object becomes optional inside the closure.
    closure = { [weak obj] in obj?.sayHi() }
    closure()  // The object is still alive; prints "Hello"
} // obj kapsam dışına çıkıyor ve serbest bırakılıyor; "Hoşçakal" yazdırır
closure() // `obj` kapatmanın içinden sıfırdır; bu hiçbir şey yazdırmaz.
kapatalım: () -> Geçersiz
yapmak {
    let obj = MyClass()
    // Captures an unowned reference to `obj`: the closure will not keep the object alive;
    // the object is always assumed to be accessible while the closure is alive.
    closure = { [unowned obj] in obj.sayHi() }
    closure()  // The object is still alive; prints "Hello"
} // obj kapsam dışına çıkıyor ve serbest bırakılıyor; "Hoşçakal" yazdırır
closure() // crash! nesnesi, serbest bırakıldıktan sonra erişiliyor.

Daha fazla bilgi için Bellek Yönetimi konusuna ve Otomatik Referans Sayımı konusuna bakın. Swift Programlama Dilinin .com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/AutomaticReferenceCounting.html) bölümü.

Döngüleri koru #

Bir nesne, aynı zamanda nesneye güçlü bir referansı olan bir kapağı tutuyorsa, bu bir saklama döngüsü referans-döngüler-ve-zayıf-referanslar#t=201607211838216466797). Döngü bozulmadıkça, nesneyi ve kapatmayı depolayan bellek sızdırılır (asla geri kazanılmaz).

sınıf Oyun {
    var score = 0
    let controller: GCController
    init(controller: GCController) {
        self.controller = controller

        // BAD: the block captures self strongly, but self holds the controller
        // (and thus the block) strongly, which is a cycle.
        self.controller.controllerPausedHandler = {
            let curScore = self.score
            print("Pause button pressed; current score: \(curScore)")
        }

        // SOLUTION: use `weak self` to break the cycle.
        self.controller.controllerPausedHandler = { [weak self] in
            guard let strongSelf = self else { return }
            let curScore = strongSelf.score
            print("Pause button pressed; current score: \(curScore)")
        }
    }
}

Asenkron kodlama için kapakları kullanma #

Kapatmalar genellikle eşzamansız görevler için, örneğin bir web sitesinden veri alırken kullanılır.

func getData(urlString: String, callback: (result: NSData?) -> Void) {

    // Turn the URL string into an NSURLRequest.
    guard let url = NSURL(string: urlString) else { return }
    let request = NSURLRequest(URL: url)
    
    // Asynchronously fetch data from the given URL.
    let task = NSURLSession.sharedSession().dataTaskWithRequest(request) {(data: NSData?, response: NSURLResponse?, error: NSError?) in

        // We now have the NSData response from the website.
        // We can get it "out" of the function by using the callback 
        // that was passed to this function as a parameter.

        callback(result: data)
    }
        
    task.resume()
}
son sürüm

Bu işlev eşzamansızdır, bu nedenle çağrıldığı iş parçacığını engellemez (GUI uygulamanızın ana iş parçacığında çağrılırsa arabirimi dondurmaz).

print("1. Going to call getData")

getData("http://www.example.com") {(result: NSData?) -> Void in

    // Called when the data from http://www.example.com has been fetched.
    print("2. Fetched data")
}

print("3. Called getData")
son sürüm

Görev eşzamansız olduğundan, çıktı genellikle şöyle görünür:

"1. Going to call getData"
"3. Called getData"
"2. Fetched data"

Kapatmanın içindeki kod, print("2. Getirilen veriler"), URL’den gelen veriler getirilinceye kadar çağrılmayacak.

Kapanışlar ve Tür Takma Adı #

Bir kapatma, bir “typealias” ile tanımlanabilir. Bu, aynı kapatma imzası birden çok yerde kullanılıyorsa, uygun tipte bir yer tutucu sağlar. Örneğin, yaygın ağ isteği geri aramaları veya kullanıcı arabirimi olay işleyicileri, bir tür takma adıyla “adlandırılmak” için harika adaylar oluşturur.

public typealias ClosureType = (x: Int, y: Int) -> Int

Daha sonra typealias’ı kullanarak bir işlev tanımlayabilirsiniz:

public func closureFunction(kapatma: ClosureType) {
    let z = closure(1, 2)
}
    
closureFunction() { (x: Int, y: Int) -> Dönüşte Int x + y }
