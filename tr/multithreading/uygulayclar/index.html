Farklı ThreadPools türleri #

SingleThreadExecutor: Sınırsız bir kuyrukta çalışan tek bir çalışan iş parçacığı kullanan ve gerektiğinde yeni bir iş parçacığı oluşturmak için sağlanan ThreadFactory’yi kullanan yürütücü. Aksi takdirde eşdeğer olan newFixedThreadPool(1, threadFactory)‘den farklı olarak, döndürülen yürütücünün ek iş parçacıkları kullanmak üzere yeniden yapılandırılamayacağı garanti edilir.

FixedThreadPool:, gerektiğinde yeni iş parçacıkları oluşturmak için sağlanan ThreadFactory’yi kullanarak, paylaşılan bir sınırsız kuyrukta çalışan sabit sayıda iş parçacığını yeniden kullanan iş parçacığı havuzu. Herhangi bir noktada, çoğu nThreads iş parçacığı etkin işleme görevleri olacaktır. Tüm iş parçacıkları aktifken ek görevler gönderilirse, bir iş parçacığı uygun olana kadar kuyrukta beklerler. Kapatmadan önce yürütme sırasındaki bir hata nedeniyle herhangi bir iş parçacığı sona ererse, sonraki görevleri yürütmek için gerekirse yenisi onun yerini alır. Havuzdaki iş parçacıkları, açıkça kapatılana kadar var olacaktır.

CachedThreadPool: Gerektiğinde yeni iş parçacıkları oluşturan, ancak kullanılabilir olduklarında önceden oluşturulmuş iş parçacıklarını yeniden kullanan ve gerektiğinde yeni iş parçacıkları oluşturmak için sağlanan ThreadFactory’yi kullanan iş parçacığı havuzu.

SingleThreadScheduledExecutor: Komutları belirli bir gecikmeden sonra çalışacak veya periyodik olarak yürütülecek şekilde planlayabilen tek iş parçacıklı yürütücü. (Ancak, bu tek iş parçacığı, kapatmadan önce yürütme sırasındaki bir hata nedeniyle sona ererse, sonraki görevleri yürütmek için gerekirse yenisinin yerini alacağını unutmayın.) Görevlerin sırayla yürütülmesi garanti edilir ve birden fazla görev etkin olmaz. Herhangi bir zamanda. Aksi takdirde eşdeğer olan newScheduledThreadPool(1, threadFactory)‘den farklı olarak, döndürülen yürütücünün ek iş parçacıkları kullanmak üzere yeniden yapılandırılamayacağı garanti edilir.

ScheduledThreadPool: Komutları belirli bir gecikmeden sonra çalışacak veya periyodik olarak çalışacak şekilde programlayabilen iş parçacığı havuzu. Farklı İş Kuyrukları türleri

Callable’lar yerine Özel Runnable’lar #

İş parçacıklarımızın, Callable’ımızdan Future nesnesini kurtarmayı bekleyen iş parçacığını engellemeden ne zaman bittiğini kontrol etmek için başka bir iyi uygulama, Runnable’lar için ’execute()’ yöntemiyle birlikte kullanarak kendi uygulamamızı oluşturmaktır.

Bir sonraki örnekte, Runnable’ı dahili bir geri çağırma ile uygulayan, çalıştırılabilirlerin ne zaman bittiğini bilmemize ve daha sonra ThreadPool’da kullanmamıza izin veren özel bir sınıf gösteriyorum:

public class CallbackTask implements Runnable {
    private final Runnable mTask;
    private final RunnableCallback mCallback;

    public CallbackTask(Runnable task, RunnableCallback runnableCallback) {
        this.mTask = task;
        this.mCallback = runnableCallback;
    }

    public void run() {
        long startRunnable = System.currentTimeMillis();
        mTask.run();
        mCallback.onRunnableComplete(startRunnable);
    }

    public interface RunnableCallback {
        void onRunnableComplete(long runnableStartTime);
    }
}

Ve işte ThreadExecutor Uygulamamız:

public class ThreadExecutorExample implements ThreadExecutor {

    private static String TAG = "ThreadExecutorExample";
    public static final int THREADPOOL_SIZE = 4;
    private long mSubmittedTasks;
    private long mCompletedTasks;
    private long mNotCompletedTasks;

    private ThreadPoolExecutor mThreadPoolExecutor;

    public ThreadExecutorExample() {
        Log.i(TAG, "[ThreadExecutorImpl] Initializing ThreadExecutorImpl");
        Log.i(TAG, "[ThreadExecutorImpl] current cores: " + Runtime.getRuntime().availableProcessors());
        this.mThreadPoolExecutor =
            (ThreadPoolExecutor) Executors.newFixedThreadPool(THREADPOOL_SIZE);

    }

    @Override
    public void execute(Runnable runnable) {
        try {
            if (runnable == null) {
                Log.e(TAG, "[execute] Runnable to execute cannot be null");
                return;
            }
            Log.i(TAG, "[execute] Executing new Thread");

            this.mThreadPoolExecutor.execute(new CallbackTask(runnable, new CallbackTask.RunnableCallback() {

                @Override
                public void onRunnableComplete(long RunnableStartTime) {
                    mSubmittedTasks = mThreadPoolExecutor.getTaskCount();
                    mCompletedTasks = mThreadPoolExecutor.getCompletedTaskCount();
                    mNotCompletedTasks = mSubmittedTasks - mCompletedTasks; // approximate

                    Log.i(TAG, "[execute] [onRunnableComplete] Runnable complete in " + (System.currentTimeMillis() - RunnableStartTime) + "ms");
                    Log.i(TAG, "[execute] [onRunnableComplete] Current threads working " + mNotCompletedTasks);
                }
            }));
        }
        catch (Exception e) {
            e.printStackTrace();
            Log.e(TAG, "[execute] Error, shutDown the Executor");
            this.mThreadPoolExecutor.shutdown();
        }
    }
}


 /**
 * Executor thread abstraction created to change the execution context from any thread from out ThreadExecutor.
 */
interface ThreadExecutor  extends Executor {

    void execute(Runnable runnable);

}

Bu örneği, Future kullanmadan, yürütüldüklerinde iş parçacıklarımın hızını milisaniye cinsinden kontrol etmek için yaptım. Bu örneği alabilir ve eşzamanlı görev çalışmasını ve tamamlanmış/tamamlanmış görevleri kontrol etmek için uygulamanıza ekleyebilirsiniz. Tüm anı kontrol etmek, o iş parçacıklarını yürütmek için ihtiyaç duyduğunuz süre.

Yeni bir ThreadPool tanımlama #

Bir ‘ThreadPool’, normalde Executors fabrika yöntemleri kullanılarak yapılandırılan, muhtemelen birkaç havuzlanmış iş parçacığından birini kullanarak gönderilen her görevi yürüten bir ‘ExecutorService’dir.

Uygulamanızda kullanmak üzere yeni bir ThreadPool’u bir singleton olarak başlatmak için temel bir kod:

public final class ThreadPool {

    private static final String TAG = "ThreadPool";
    private static final int CORE_POOL_SIZE = 4;
    private static final int MAX_POOL_SIZE = 8;
    private static final int KEEP_ALIVE_TIME = 10; // 10 seconds
    private final Executor mExecutor;

    private static ThreadPool sThreadPoolInstance;

    private ThreadPool() {
        mExecutor = new ThreadPoolExecutor(
            CORE_POOL_SIZE, MAX_POOL_SIZE, KEEP_ALIVE_TIME,
            TimeUnit.SECONDS, new LinkedBlockingQueue<Runnable>());
    }

    public void execute(Runnable runnable) {
        mExecutor.execute(runnable);
    }

    public synchronized static ThreadPool getThreadPoolInstance() {
        if (sThreadPoolInstance == null) {
            Log.i(TAG, "[getThreadManagerInstance] New Instance");
            sThreadPoolInstance = new ThreadPool();
        }
        return sThreadPoolInstance;
    }
}

Çalıştırılabilir yönteminizi çağırmanın iki yolu vardır, ’execute()’ veya ‘submit()’ kullanın. aralarındaki fark, “gönder ()” öğesinin, “Çağrılabilir” geri çağırmadan T nesnesi döndürüldüğünde, çalışan iş parçacığını programlı olarak iptal etmenize olanak tanıyan bir “Gelecek” nesnesi döndürmesidir. ‘Gelecek’ hakkında daha fazla bilgiyi buradan okuyabilirsiniz

Gelecek ve Çağrılabilirler #

Threadpools ile kullanabileceğimiz özelliklerden biri, thread’in işini ne zaman bitirdiğini bilmemizi sağlayan submit() metodudur. Bunu, Callable’dan kendi hedeflerimize kullanabileceğimiz bir nesne döndüren ‘Future’ nesnesi sayesinde yapabiliriz.

Callable örneğinin nasıl kullanılacağına dair bir örnek:

public class CallablesExample{

//Create MyCustomCallable instance
List<Future<String>> mFutureList = new ArrayList<Future<String>>();

//Create a list to save the Futures from the Callable
Callable<String> mCallable = new MyCustomCallable();

public void main(String args[]){
    //Get ExecutorService from Executors utility class, Creating a 5 threads pool.
    ExecutorService executor = Executors.newFixedThreadPool(5);
   
    
   
    for (int i = 0; i < 100; i++) {
        //submit Callable tasks to be executed by thread pool
        Future<String> future = executor.submit(mCallable);
        //add Future to the list, we can get return value using Future
        mFutureList.add(future);
    }
    for (Future<String> fut : mFutureList) {
        try {
            //Print the return value of Future, Notice the output delay in console
            //because Future.get() stop the thread till the task have been completed
            System.out.println(new Date() + "::" + fut.get());
        } catch (InterruptedException | ExecutionException e) {
            e.printStackTrace();
        }
    }
    //Shut down the service
    executor.shutdown();
}

 class MyCustomCallable implements Callable<String> {

    @Override
    public String call() throws Exception {
        Thread.sleep(1000);
        //return the thread name executing this callable task
        return Thread.currentThread().getName();
    }
}
}

Gördüğünüz gibi 5 Thread ile bir Threadpool oluşturuyoruz, bu da 5 callable’ı paralel atabileceğimiz anlamına geliyor. Threadler bittiğinde, callable’dan Future nesnesini, bu durumda thread’in adını alacağız.

UYARI

Bu örnekte, kaç tane iş parçacığı yürüttüğümüzü bilmek için Futures’ı dizi içinde bir nesne olarak kullanıyoruz ve istediğimiz verilerle bir oturum açma konsolunda bu kadar çok kez yazdırıyoruz. Ancak, çağrılabilirde daha önce kaydettiğimiz verileri bize döndürmek için Future.get() yöntemini kullanmak istiyorsak, görev tamamlanana kadar iş parçacığını engelleyeceğiz. Bunu olabildiğince hızlı yapmak istediğinizde bu tür aramalara dikkat edin.

Yürütücüye ThreadFactory Ekleme #

ExecutorService’i, iç iş parçacığı havuzundan iş parçacıkları atamak veya görevleri gerçekleştirmek için isteğe bağlı olarak oluşturmak için kullanıyoruz. Her ExecutorService’in bir ThreadFactory’si vardır, ancak ExecutorService, özel bir ayar yapmazsak her zaman varsayılanı kullanır. Bunu neden yapmalıyız?

  • Daha açıklayıcı bir iş parçacığı adı ayarlamak için. Varsayılan ThreadFactory, pool-1-thread-1, pool-2-thread-1, pool-3-thread-1 vb. havuz-m-thread-n biçiminde iş parçacığı adları verir. hata ayıklama veya bir şeyi izleme, bu iş parçacıklarının ne yaptığını bilmek zor

  • Özel bir Daemon durumu ayarlayın, varsayılan ThreadFactory arka plan programı olmayan sonuçlar üretir.

  • Önceliği iş parçacıklarımıza ayarlayın, varsayılan ThreadFactory, tüm iş parçacıklarına orta bir öncelik atadı.

  • Thread nesnesinde setUncaughtExceptionHandler() kullanarak iş parçacığımız için UncaughtExceptionHandler belirleyebilirsiniz. Bu, Thread’in çalıştırma yöntemi yakalanmamış bir istisna oluşturduğunda geri çağrılır.

İşte bir ThreadPool üzerinden bir ThreadFactory’nin kolay bir uygulaması.

public class ThreadExecutorExample implements ThreadExecutor {
private static String TAG = "ThreadExecutorExample";
private static final int INITIAL_POOL_SIZE = 3;
private static final int MAX_POOL_SIZE = 5;

// Sets the amount of time an idle thread waits before terminating
private static final int KEEP_ALIVE_TIME = 10;

// Sets the Time Unit to seconds
private static final TimeUnit KEEP_ALIVE_TIME_UNIT = TimeUnit.SECONDS;

private final BlockingQueue<Runnable> workQueue;

private final ThreadPoolExecutor threadPoolExecutor;

private final ThreadFactory threadFactory;
private ThreadPoolExecutor mThreadPoolExecutor;

public ThreadExecutorExample() {
    this.workQueue = new LinkedBlockingQueue<>();
    this.threadFactory = new CustomThreadFactory();
    this.threadPoolExecutor = new ThreadPoolExecutor(INITIAL_POOL_SIZE, MAX_POOL_SIZE,
            KEEP_ALIVE_TIME, KEEP_ALIVE_TIME_UNIT, this.workQueue, this.threadFactory);
}

public void execute(Runnable runnable) {
    if (runnable == null) {
        return;
    }
    this.threadPoolExecutor.execute(runnable);
}

private static class CustomThreadFactory implements ThreadFactory {
    private static final String THREAD_NAME = "thread_";
    private int counter = 0;

    @Override public Thread newThread(Runnable runnable) {
        return new Thread(runnable, THREAD_NAME + counter++);
    }
}
}

/**
 * Executor thread abstraction created to change the execution context from any thread from out ThreadExecutor.
 */
interface ThreadExecutor extends Executor {
    
    void execute(Runnable runnable);
    
}

Bu örnek sadece Thread’in adını bir sayaçla değiştiriyor, ancak istediğimiz kadar değiştirebiliriz.
