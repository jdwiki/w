Efficient R programlama bölümünde konuyla ilgili güzel bir bölüm var.

.Rprofile - yürütülen ilk kod parçası #

.Rprofile, .Rprofile dosyasını içeren dizinden R’yi başlattığınızda yürütülen R kodunu içeren bir dosyadır. R’nin ana dizininde bulunan benzer şekilde adlandırılan ‘Rprofile.site’, herhangi bir dizinden R’yi her yüklediğinizde varsayılan olarak yürütülür. ‘Rprofile.site’ ve daha geniş kapsamlı olarak ‘.Rprofile’, kişisel tercihler ve tanımladığınız çeşitli yardımcı işlevlerle bir R oturumunu başlatmak için kullanılabilir.

Önemli not: RStudio kullanıyorsanız, her RStudio proje dizininde ayrı bir `.Rprofile’niz olabilir.

Bir .Rprofile dosyasına ekleyebileceğiniz bazı kod örnekleri aşağıda verilmiştir.

R ana dizininizi ayarlama #

# set R_home
Sys.setenv(R_USER="c:/R_home") # just an example directory
# but don't confuse this with the $R_HOME environment variable.

Sayfa boyutu seçeneklerini ayarlama #

options(papersize="a4")
options(editor="notepad")
options(pager="internal")

varsayılan yardım türünü ayarlayın #

options(help_type="html")

site kitaplığı ayarla #

.Library.site <- file.path(chartr("\\", "/", R.home()), "site-library")

Bir CRAN aynası ayarla #

local({r <- getOption("repos")
    r["CRAN"] <- "http://my.local.cran"
    options(repos=r)})

Kitaplığınızın konumunu ayarlama #

Bu, her R sürüm güncellemesiyle tüm paketleri yeniden yüklemenize gerek kalmamasını sağlar.

# library location
.libPaths("c:/R_home/Rpackages/win")

Özel kısayollar veya işlevler #

Bazen uzun bir R ifadesi için bir kısayola sahip olmak yararlıdır. Bu ayarın yaygın bir örneği, “.Last.value” yazmak zorunda kalmadan son üst düzey ifade sonucuna erişmek için etkin bir bağlama:

makeActiveBinding(".", function(){.Last.value}, .GlobalEnv)

.Rprofile yalnızca bir R dosyası olduğundan, herhangi bir rastgele R kodu içerebilir.

En kullanışlı paketleri önceden yükleme #

Bu kötü bir uygulamadır ve paket yükleme kodunu, bu paketlerin gerçekte kullanıldığı betiklerden ayırdığından genellikle kaçınılmalıdır.

Ayrıca bakınız #

Tüm farklı başlangıç ​​komut dosyaları ve diğer hususlar için “help(Startup)” bölümüne bakın. Özellikle, sistem çapında iki “Profil” dosyası da yüklenebilir. Birincisi, ‘Rprofile’, genel ayarları içerebilir, diğer ‘Profile.site’ dosyası, sistem yöneticisinin tüm kullanıcılar için yapabileceği yerel seçimleri içerebilir. Her iki dosya da R kurulumunun ${RHOME}/etc dizininde bulunur. Bu dizin ayrıca, her ikisi de kullanıcının ana dizininde yerel bir dosya ‘~/.Renviron’ ile tamamlanabilen ‘Renviron’ ve ‘Renviron.site’ global dosyalarını içerir.

.Rprofil örneği #

Başlatmak #

# Load library setwidth on start - to set the width automatically.
.First <- function() {
  library(setwidth)
  # If 256 color terminal - use library colorout.
  if (Sys.getenv("TERM") %in% c("xterm-256color", "screen-256color")) {
    library("colorout")
  }
}

Seçenekler #

# Select default CRAN mirror for package installation.
options(repos=c(CRAN="https://cran.gis-lab.info/"))

# Print maximum 1000 elements.
options(max.print=1000)

# No scientific notation.
options(scipen=10)

# No graphics in menus.
options(menu.graphics=FALSE)

# Auto-completion for package names.
utils::rc.settings(ipck=TRUE)

Özel İşlevler #

# Invisible environment to mask defined functions
.env = new.env()

# Quit R without asking to save.
.env$q <- function (save="no", ...) {
  quit(save=save, ...)
}

# Attach the environment to enable functions.
attach(.env, warn.conflicts=FALSE)
