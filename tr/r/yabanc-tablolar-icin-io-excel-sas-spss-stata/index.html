rio ile verileri içe aktarma #

Birçok yaygın dosya biçiminden veri içe aktarmanın çok basit bir yolu rio. Bu paket, yaygın olarak kullanılan birçok veri içe aktarma işlevini saran ve böylece standart bir arabirim sağlayan bir “import()” işlevi sağlar. Yalnızca bir dosya adını veya URL’yi “import()” öğesine ileterek çalışır:

import("example.csv")       # comma-separated values
import("example.tsv")       # tab-separated values
import("example.dta")       # Stata
import("example.sav")       # SPSS
import("example.sas7bdat")  # SAS
import("example.xlsx")      # Excel

import() ayrıca sıkıştırılmış dizinlerden, URL’lerden (HTTP veya HTTPS) ve panodan da okuyabilir. Desteklenen tüm dosya biçimlerinin kapsamlı bir listesi rio paketi github deposunda mevcuttur.

Okumaya çalıştığınız belirli dosya biçimiyle ilgili bazı başka parametreleri doğrudan import() işlevine geçirerek belirtmek bile mümkündür:

import("example.csv", format = ",") #for csv file where comma is used as separator
import("example.csv", format = ";") #for csv file where semicolon is used as separator

Stata, SPSS ve SAS dosyalarını okuyun ve yazın #

‘Yabancı’ ve ‘haven’ paketleri, Stata, SPSS ve SAS ve ilgili yazılımlar gibi çeşitli diğer istatistiksel paketlerden dosyaları içe ve dışa aktarmak için kullanılabilir. Dosyaları içe aktarmak için desteklenen veri türlerinin her biri için bir ‘okuma’ işlevi vardır.

# loading the packages
library(foreign)
library(haven)
library(readstata13)
library(Hmisc)

En yaygın veri türleri için bazı örnekler:

# reading Stata files with `foreign`
read.dta("path\to\your\data")
# reading Stata files with `haven`
read_dta("path\to\your\data")

‘yabancı’ paket, Stata 7-12 sürümleri için stata (.dta) dosyalarını okuyabilir. Geliştirme sayfasına göre, ‘read.dta’ aşağı yukarı donmuş durumda ve 13+ sürümlerinde okumak için güncellenmeyecek. Stata’nın daha yeni sürümleri için, ‘readstata13’ paketini veya ‘haven’ paketini kullanabilirsiniz. ‘readstata13’ için dosyalar

# reading recent Stata (13+) files with `readstata13`
read.dta13("path\to\your\data")

SPSS ve SAS dosyalarında okumak için

# reading SPSS files with `foreign`
read.spss("path\to\your\data.sav", to.data.frame = TRUE)
# reading SPSS files with `haven`
read_spss("path\to\your\data.sav")
read_sav("path\to\your\data.sav")
read_por("path\to\your\data.por")

# reading SAS files with `foreign`
read.ssd("path\to\your\data")
# reading SAS files with `haven`
read_sas("path\to\your\data")
# reading native SAS files with `Hmisc`
sas.get("path\to\your\data")   #requires access to saslib 
# Reading SA XPORT format ( *.XPT ) files
sasxport.get("path\to\your\data.xpt")  # does not require access to SAS executable

‘SAScii’ paketi, SAS SET içe aktarma kodunu kabul edecek ve ‘read.fwf’ ile işlenebilecek bir metin dosyası oluşturacak işlevler sağlar. Kamuya açık büyük veri kümelerinin içe aktarılması için çok sağlam olduğu kanıtlanmıştır. Destek https://github.com/ajdamico/SAScii adresindedir.

Veri çerçevelerini diğer istatistiksel paketlere aktarmak için write.foreign() yazma fonksiyonlarını kullanabilirsiniz. Bu, biri verileri içeren ve diğeri diğer paketin verileri okumak için ihtiyaç duyduğu talimatları içeren 2 dosya yazacaktır.

# writing to Stata, SPSS or SAS files with `foreign`
write.foreign(dataframe, datafile, codefile,
              package = c("SPSS", "Stata", "SAS"), ...)
write.foreign(dataframe, "path\to\data\file", "path\to\instruction\file", package = "Stata")

# writing to Stata files with `foreign`
write.dta(dataframe, "file", version = 7L,
          convert.dates = TRUE, tz = "GMT",
          convert.factors = c("labels", "string", "numeric", "codes"))

# writing to Stata files with `haven`
write_dta(dataframe, "path\to\your\data")

# writing to Stata files with `readstata13`
save.dta13(dataframe, file, data.label = NULL, time.stamp = TRUE,
  convert.factors = TRUE, convert.dates = TRUE, tz = "GMT",
  add.rownames = FALSE, compress = FALSE, version = 117,
  convert.underscore = FALSE)

# writing to SPSS files with `haven`
write_sav(dataframe, "path\to\your\data")

SPSS tarafından saklanan dosya ayrıca read.spss ile şu şekilde okunabilir:

 foreign::read.spss('data.sav', to.data.frame=TRUE, use.value.labels=FALSE, 
                     use.missings=TRUE, reencode='UTF-8')
# to.data.frame if TRUE: return a data frame
# use.value.labels if TRUE: convert variables with value labels into R factors with those levels
# use.missings if TRUE: information on user-defined missing values will used to set the corresponding values to NA.
# reencode character strings will be re-encoded to the current locale. The default, NA, means to do so in a UTF-8 locale, only.

Excel dosyalarını içe aktarma #

Aşağıdaki tabloda özetlendiği gibi, her biri farklı diller veya kaynaklar kullanan excel dosyalarını okumak için birkaç R paketi vardır:

R paketi kullanır
xlsx Java
XLbağlantı Java
açıkxlsx C++
okumaxl C++
RODBC ODBC
gveri Perl

Java veya ODBC kullanan paketler için sisteminizle ilgili ayrıntıları bilmek önemlidir, çünkü R sürümünüze ve işletim sisteminize bağlı olarak uyumluluk sorunlarınız olabilir. Örneğin, R 64 bit kullanıyorsanız, “xlsx” veya “XLconnect” kullanmak için Java 64 bit’e de sahip olmanız gerekir.

Her pakette excel dosyalarının okunmasına ilişkin bazı örnekler aşağıda verilmiştir. Paketlerin çoğunun aynı veya çok benzer işlev adlarına sahip olduğunu unutmayın. Bu nedenle, paketi ‘package::function’ gibi açıkça belirtmek yararlıdır. ‘openxlsx’ paketi, RTools’un önceden yüklenmesini gerektirir.

xlsx paketi ile excel dosyalarını okuma #

library(xlsx)

İçe aktarmak için sayfanın dizini veya adı gereklidir.

xlsx::read.xlsx("Book1.xlsx", sheetIndex=1)

xlsx::read.xlsx("Book1.xlsx", sheetName="Sheet1")

XLconnect paketi ile Excel dosyalarını okuma #

library(XLConnect)
wb <- XLConnect::loadWorkbook("Book1.xlsx")

# Either, if Book1.xlsx has a sheet called "Sheet1":
sheet1 <- XLConnect::readWorksheet(wb, "Sheet1")
# Or, more generally, just get the first sheet in Book1.xlsx:
sheet1 <- XLConnect::readWorksheet(wb, getSheets(wb)[1])

‘XLConnect’, ‘Book1.xlsx’ içine yerleştirilmiş önceden tanımlanmış Excel hücre stillerini otomatik olarak içe aktarır. Bu, çalışma kitabı nesnenizi biçimlendirmek ve mükemmel biçimde biçimlendirilmiş bir Excel belgesini dışa aktarmak istediğinizde kullanışlıdır. Öncelikle, ‘Book1.xlsx’ içinde istediğiniz hücre biçimlerini oluşturmanız ve bunları örneğin ‘myHeader’, ‘myBody’ ve ‘myPcts’ olarak kaydetmeniz gerekecektir. Ardından, çalışma kitabını R içine yükledikten sonra (yukarıya bakın):

Headerstyle <- XLConnect::getCellStyle(wb, "myHeader")
Bodystyle <- XLConnect::getCellStyle(wb, "myBody")
Pctsstyle <- XLConnect::getCellStyle(wb, "myPcts")

Hücre stilleri artık ‘R’ ortamınıza kaydedilir. Verilerinizin belirli aralıklarına hücre stilleri atamak için, aralığı tanımlamanız ve ardından stili atamanız gerekir:

Headerrange <- expand.grid(row = 1, col = 1:8)
Bodyrange <- expand.grid(row = 2:6, col = c(1:5, 8))
Pctrange <- expand.grid(row = 2:6, col = c(6, 7))

XLConnect::setCellStyle(wb, sheet = "sheet1", row = Headerrange$row,
             col = Headerrange$col, cellstyle = Headerstyle)
XLConnect::setCellStyle(wb, sheet = "sheet1", row = Bodyrange$row,
             col = Bodyrange$col, cellstyle = Bodystyle)
XLConnect::setCellStyle(wb, sheet = "sheet1", row = Pctrange$row,
             col = Pctrange$col, cellstyle = Pctsstyle)

‘XLConnect’in kolay olduğunu, ancak biçimlendirmede aşırı derecede yavaşlayabileceğini unutmayın. ‘openxlsx’ tarafından çok daha hızlı, ancak daha hantal bir biçimlendirme seçeneği sunulmaktadır.

openxlsx paketi ile excel dosyalarını okuma #

Excel dosyaları “openxlsx” paketiyle içe aktarılabilir

library(openxlsx)

openxlsx::read.xlsx("spreadsheet1.xlsx", colNames=TRUE, rowNames=TRUE)

#colNames: If TRUE, the first row of data will be used as column names.
#rowNames: If TRUE, first column of data will be used as row names.

R’de okunması gereken sayfa, “sayfa” argümanındaki konumu sağlanarak seçilebilir:

openxlsx::read.xlsx("spreadsheet1.xlsx", sheet = 1)

veya adını bildirerek:

openxlsx::read.xlsx("spreadsheet1.xlsx", sheet = "Sheet1")

Ek olarak, “openxlsx” bir okuma sayfasındaki tarih sütunlarını algılayabilir. Tarihlerin otomatik olarak algılanmasına izin vermek için, “tespitTarihleri” bağımsız değişkeni “DOĞRU” olarak ayarlanmalıdır:

openxlsx::read.xlsx("spreadsheet1.xlsx", sheet = "Sheet1", detectDates= TRUE)

readxl paketi ile excel dosyalarını okuma #

Excel dosyaları, ‘readxl’ paketi kullanılarak bir veri çerçevesi olarak ‘R’ içine aktarılabilir.

library(readxl)

Hem “.xls” hem de “.xlsx” dosyalarını okuyabilir.

readxl::read_excel("spreadsheet1.xls")
readxl::read_excel("spreadsheet2.xlsx")

İçe aktarılacak sayfa, numara veya adla belirtilebilir.

readxl::read_excel("spreadsheet.xls", sheet = 1)
readxl::read_excel("spreadsheet.xls", sheet = "summary")

“col_names = TRUE” bağımsız değişkeni, ilk satırı sütun adları olarak ayarlar.

 readxl::read_excel("spreadsheet.xls", sheet = 1, col_names = TRUE)

‘col_types’ bağımsız değişkeni, verilerdeki sütun türlerini vektör olarak belirtmek için kullanılabilir.

readxl::read_excel("spreadsheet.xls", sheet = 1, col_names = TRUE,
                   col_types = c("text", "date", "numeric", "numeric"))

RODBC paketi ile excel dosyalarını okuma #

Excel dosyaları, daha önce JET olan Windows Access Database Engine (ACE) ile arayüz oluşturan ODBC Excel Sürücüsü kullanılarak okunabilir. RODBC paketi ile R, bu sürücüye bağlanabilir ve çalışma kitaplarını doğrudan sorgulayabilir. Çalışma sayfalarının, benzer türdeki organize sütunlardaki verilerle birlikte ilk satırdaki sütun başlıklarını koruduğu varsayılır. NOT: JET/ACE yüklü .dll dosyaları olduğundan ve diğer işletim sistemlerinde kullanılamadığından, bu yaklaşım yalnızca Windows/PC makineleriyle sınırlıdır.

library(RODBC)

xlconn <- odbcDriverConnect('Driver={Microsoft Excel Driver (*.xls, *.xlsx, *.xlsm, *.xlsb)};
                             DBQ=C:\\Path\\To\\Workbook.xlsx')

df <- sqlQuery(xlconn, "SELECT * FROM [SheetName$]")
close(xlconn)

Bu yaklaşımda bir SQL motoruna bağlanarak, Excel çalışma sayfaları, ‘JOIN’ ve ‘UNION’ işlemlerini içeren veritabanı tablolarına benzer şekilde sorgulanabilir. Sözdizimi, JET/ACE SQL lehçesini takip eder. NOT: Çalışma kitaplarında yalnızca veri erişim DML ifadeleri, özellikle “SELECT” çalıştırılabilir, bu sorgular güncellenemez olarak kabul edilir.

joindf <-  sqlQuery(xlconn, "SELECT t1.*, t2.* FROM [Sheet1$] t1
                             INNER JOIN [Sheet2$] t2
                             ON t1.[ID] = t2.[ID]")

uniondf <-  sqlQuery(xlconn, "SELECT * FROM [Sheet1$]
                              UNION  
                              SELECT * FROM [Sheet2$]")

Aynı ODBC kanalından geçerli bir çalışma kitabına işaret eden diğer çalışma kitapları bile sorgulanabilir:

otherwkbkdf <- sqlQuery(xlconn, "SELECT * FROM 
                                 [Excel 12.0 Xml;HDR=Yes;
                                 Database=C:\\Path\\To\\Other\\Workbook.xlsx].[Sheet1$];")
