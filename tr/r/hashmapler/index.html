Karma haritalar olarak ortamlar #

Not: sonraki bölümlerde, karma haritası ve karma tablosu terimleri birbirinin yerine kullanılır ve aynı kavrama atıfta bulunur, yani, dahili bir karma işlevi kullanarak verimli anahtar araması sağlayan bir veri yapısı.

giriiş #

R yerel bir karma tablo yapısı sağlamasa da, “new.env"den döndürülen “environment” nesnesinin (varsayılan olarak) karma anahtar aramaları sağladığı gerçeğinden yararlanılarak benzer işlevsellik elde edilebilir. Aşağıdaki iki ifade, “hash” parametresi varsayılan olarak “TRUE” olarak ayarlandığından eşdeğerdir:

H <- new.env(hash = TRUE)
H <- new.env()

Ek olarak, varsayılan değeri 29 olan ‘size’ parametresi aracılığıyla dahili hash tablosunun belirli bir boyutla önceden tahsis edildiği belirtilebilir. Diğer tüm R nesneleri gibi, ‘ortam’ da kendi hafızasını yönetir ve büyüyecektir. bu nedenle, “size” için varsayılan olmayan bir değer talep etmek gerekli olmasa da, nesne (nihayetinde) çok sayıda elementler. “size” aracılığıyla fazladan alan ayırmanın, kendi başına, daha büyük bellek ayak izine sahip bir nesneyle sonuçlanmadığını belirtmekte fayda var:

object.size(new.env())
# 56 bytes

object.size(new.env(size = 10e4))
# 56 bytes

sokma #

Öğelerin eklenmesi, “environment” sınıfı için sağlanan “[[<-” veya “$<-” yöntemlerinden biri kullanılarak yapılabilir, ancak değil “tek ayraç” ataması (’[<- `):

H <- new.env()

H[["key"]] <- rnorm(1)

key2 <- "xyz"
H[[key2]] <- data.frame(x = 1:3, y = letters[1:3])

H$another_key <- matrix(rbinom(9, 1, 0.5) > 0, nrow = 3)

H["error"] <- 42
#Error in H["error"] <- 42 : 
#  object of type 'environment' is not subsettable

R’nin diğer yönleri gibi, ilk yöntem (’nesne[[anahtar]] <-değer’) genellikle ikinci yönteme (’nesne$anahtar <- değer’) tercih edilir, çünkü ilk durumda, bunun yerine bir değişken kullanılabilir bir hazır değer (örneğin, yukarıdaki örnekte “key2”).

Genelde karma harita uygulamalarında olduğu gibi, “environment” nesnesi yinelenen anahtarları depolamaz. Mevcut bir anahtar için bir anahtar/değer çifti eklemeye çalışmak, önceden depolanan değerin yerini alacaktır:

H[["key3"]] <- "original value"

H[["key3"]] <- "new value"

H[["key3"]]
#[1] "new value"

Anahtar Arama #

Benzer şekilde, öğelere “[[” veya “$” ile erişilebilir, ancak “[” ile erişilemez:

H[["key"]]
#[1] 1.630631
 
H[[key2]]   ## assuming key2 <- "xyz"
#   x y
# 1 1 a
# 2 2 b
# 3 3 c

H$another_key
#       [,1]  [,2]  [,3]
# [1,]  TRUE  TRUE  TRUE
# [2,] FALSE FALSE FALSE
# [3,]  TRUE  TRUE  TRUE

H[1]
#Error in H[1] : object of type 'environment' is not subsettable

Hash Haritasının İncelenmesi #

Sıradan bir “ortam” olan hash haritası tipik yöntemlerle incelenebilir:

names(H)
#[1] "another_key" "xyz"         "key"         "key3"       

ls(H)
#[1] "another_key" "key"         "key3"        "xyz"        
 
str(H)
#<environment: 0x7828228> 
 
ls.str(H)
# another_key :  logi [1:3, 1:3] TRUE FALSE TRUE TRUE FALSE TRUE ...
# key :  num 1.63
# key3 :  chr "new value"
# xyz : 'data.frame':    3 obs. of  2 variables:
#  $ x: int  1 2 3
#  $ y: chr  "a" "b" "c"

Öğeler “rm” kullanılarak kaldırılabilir:

rm(list = c("key", "key3"), envir = H)

ls.str(H)
# another_key :  logi [1:3, 1:3] TRUE FALSE TRUE TRUE FALSE TRUE ...
# xyz : 'data.frame':    3 obs. of  2 variables:
#  $ x: int  1 2 3
#  $ y: chr  "a" "b" "c"

Esneklik #

“Ortam” nesnelerini karma tablo olarak kullanmanın en büyük faydalarından biri, neredeyse her tür nesneyi, * hatta diğer “ortamları”* bir değer olarak saklama yetenekleridir:

H2 <- new.env()

H2[["a"]] <- LETTERS
H2[["b"]] <- as.list(x = 1:5, y = matrix(rnorm(10), 2))
H2[["c"]] <- head(mtcars, 3)
H2[["d"]] <- Sys.Date()
H2[["e"]] <- Sys.time()
H2[["f"]] <- (function() {
    H3 <- new.env()
    for (i in seq_along(names(H2))) {
        H3[[names(H2)[i]]] <- H2[[names(H2)[i]]]
    }
    H3
})()

ls.str(H2)
# a :  chr [1:26] "A" "B" "C" "D" "E" "F" "G" "H" "I" "J" "K" ...
# b : List of 5
#  $ : int 1
#  $ : int 2
#  $ : int 3
#  $ : int 4
#  $ : int 5
# c : 'data.frame':    3 obs. of  11 variables:
#  $ mpg : num  21 21 22.8
#  $ cyl : num  6 6 4
#  $ disp: num  160 160 108
#  $ hp  : num  110 110 93
#  $ drat: num  3.9 3.9 3.85

sınırlamalar #

“Ortam” nesnelerini karma haritalar olarak kullanmanın en büyük sınırlamalarından biri, R’nin birçok özelliğinden farklı olarak, öğe arama/ekleme için vektörleştirmenin desteklenmemesidir:

names(H2)
#[1] "a" "b" "c" "d" "e" "f"

H2[[c("a", "b")]]
#Error in H2[[c("a", "b")]] : 
#  wrong arguments for subsetting an environment
 
Keys <- c("a", "b")
H2[[Keys]]
#Error in H2[[Keys]] : wrong arguments for subsetting an environment

Nesnede saklanan verilerin doğasına bağlı olarak, aynı anda birçok öğe atamak için “vapply” veya “list2env” kullanmak mümkün olabilir:

E1 <- new.env()
invisible({
    vapply(letters, function(x) {
        E1[[x]] <- rnorm(1)
        logical(0)
    }, FUN.VALUE = logical(0))
})

all.equal(sort(names(E1)), letters)
#[1] TRUE

Keys <- letters
E2 <- list2env(
    setNames(
        as.list(rnorm(26)),
        nm = Keys), 
    envir = NULL,
    hash = TRUE
)

all.equal(sort(names(E2)), letters)
#[1] TRUE

Yukarıdakilerin hiçbiri özellikle kısa değildir, ancak anahtar/değer çiftlerinin sayısı büyük olduğunda bir “for” döngüsü vb. kullanmak için tercih edilebilir.

paket:karma #

karma paketi, R’de bir karma yapı sunar. Ancak, hem eklemeler hem de okumalar için zamanlama açısından, kullanıma karşı olumsuz bir şekilde karşılaştırır. ortamlar bir karma olarak. Bu belge, yalnızca varlığını kabul eder ve yukarıda belirtilen nedenlerle aşağıda örnek zamanlama kodunu sağlar. Bugün R kodunda hash’in uygun bir çözüm olduğu tanımlanmış bir durum yoktur.

Düşünmek:

# Generic unique string generator
unique_strings <- function(n){
    string_i <- 1
    string_len <- 1
    ans <- character(n)
    chars <- c(letters,LETTERS)
    new_strings <- function(len,pfx){
    for(i in 1:length(chars)){
        if (len == 1){
        ans[string_i] <<- paste(pfx,chars[i],sep='')
        string_i <<- string_i + 1
        } else {
        new_strings(len-1,pfx=paste(pfx,chars[i],sep=''))
        }
        if (string_i > n) return ()
    }
    }
    while(string_i <= n){
    new_strings(string_len,'')
    string_len <- string_len + 1
    }
    sample(ans)
}

# Generate timings using an enviornment
timingsEnv <- plyr::adply(2^(10:15),.mar=1,.fun=function(i){
    strings <- unique_strings(i)
    ht1 <- new.env(hash=TRUE)
    lapply(strings, function(s){ ht1[[s]] <<- 0L})
    data.frame(
    size=c(i,i),
    seconds=c(
        system.time(for (j in 1:i) ht1[[strings[j]]]==0L)[3]),
    type = c('1_hashedEnv')
    )
})

timingsHash <- plyr::adply(2^(10:15),.mar=1,.fun=function(i){
    strings <- unique_strings(i)
    ht <- hash::hash()
    lapply(strings, function(s) ht[[s]] <<- 0L)
    data.frame(
    size=c(i,i),
    seconds=c(
        system.time(for (j in 1:i) ht[[strings[j]]]==0L)[3]),
    type = c('3_stringHash')
    )
})

paket:dinle #

package:listenv ortamlara liste benzeri bir arayüz uygulasa da, karma benzeri amaçlar için ortamlara göre performansı karma alma konusunda zayıftır. . Ancak indeksler sayısal ise, alma işlemi oldukça hızlı olabilir. Ancak, başka avantajları da vardır, örn. “paket:gelecek” ile uyumluluk. Bu paketi bu amaçla ele almak, mevcut konunun kapsamını aşmaktadır. Ancak, burada sağlanan zamanlama kodu, yazma zamanlamaları için package:hash örneğiyle birlikte kullanılabilir.

timingsListEnv <- plyr::adply(2^(10:15),.mar=1,.fun=function(i){
    strings <- unique_strings(i)
    le <- listenv::listenv()
    lapply(strings, function(s) le[[s]] <<- 0L)
    data.frame(
    size=c(i,i),
    seconds=c(
        system.time(for (k in 1:i) le[[k]]==0L)[3]),
    type = c('2_numericListEnv')
    )
})
