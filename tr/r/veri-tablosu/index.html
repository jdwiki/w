Data.table, R tabanından veri çerçevelerinin işlevselliğini genişleten, özellikle performanslarını ve sözdizimlerini iyileştiren bir pakettir. Ayrıntılar için paketin https://www.wikiod.com/tr/data-table/datatable-kullanmaya-baslama adresindeki Dokümanlar alanına bakın.

Sözdizimi #

  • DT[i, j, by] # DT[where, select|update|do, by]
  • DT[...][...] # chaining
  • ################# DT içindeki kısayollar, özel işlevler ve özel semboller[...]
  • .()
    # in several arguments, replaces list()
  • J() # in i, replaces list()
  • :=
    # in j, a function used to add or modify columns
  • .N # in i, the total number of rows
    # in j, the number of rows in a group
  • .BEN # in j, the vector of row numbers in the table (filtered by i)
  • .SD # in j, the current subset of the data
    # selected by the .SDcols argument
  • .GRP # in j, the current index of the subset of the data
  • .İLE # in j, the list of by values for the current subset of data
  • V1, V2, … # default names for unnamed columns created in j
  • ################# DT içine katılır[...]
  • DT1[DT2, açık, j] # join two tables
  • i.* # special prefix on DT2’s columns after the join
  • tarafından=.EACHI # special option available only with a join
  • DT1[!DT2, açık, j] # anti-join two tables
  • DT1[DT2, açık, yuvarlanma, j] # join two tables, rolling on the last column in on=
  • ################ Yeniden şekillendirme, istifleme ve bölme
  • melt(DT, id.vars, ölçü.vars) # transform to long format
    # for multiple columns, use measure.vars = patterns(…)
  • dcast(DT, formül) # transform to wide format
  • rbind(DT1, DT2, …) # stack enumerated data.tables
  • rbindlist(DT_list, idcol) # stack a list of data.tables
  • bölünmüş(DT, göre) # split a data.table into a list
  • ################ Data.tables için özelleşmiş diğer bazı işlevler
  • üst üste binme # overlap joins
  • birleştirmek # another way of joining two tables
  • Ayarlamak # another way of adding or modifying columns
  • fintersect, fseddiff, funion, fsetequal, benzersiz, yinelenen, anyDuplicated # set-theory operations with rows as elements
  • benzersizN # the number of distinct rows
  • satıridv(DT, sütunlar) # row ID (1 to .N) within each group determined by cols
  • rleidv(DT, sütunlar) # group ID (1 to .GRP) within each group determined by runs of cols
  • shift(DT, n, type=c(“gecikme”, “öncü”)) # apply a shift operator to every column
  • setorder, setcolorder, setnames, setkey, setindex, setattr # modify attributes and order by reference

Kurulum ve destek #

1 data.table paketini kurmak için:

# install from CRAN
install.packages("data.table")       

# or install development version 
install.packages("data.table", type = "source", repos = "http://Rdatatable.github.io/data.table")

# and to revert from devel to CRAN, the current version must first be removed
remove.packages("data.table")
install.packages("data.table")

Paketin resmi sitesi, başlama konusunda yardım sağlayan wiki sayfalarına ve web’deki sunumların ve makalelerin listelerine sahiptir. Bir soru sormadan önce – burada StackOverflow’ta veya başka bir yerde – lütfen destek sayfasını okuyun.

Paket yükleniyor #

Yukarıdaki örneklerdeki işlevlerin çoğu data.table ad alanında bulunur. Bunları kullanmak için önce ’library(data.table)’ gibi bir satır eklemeniz veya sadece ‘fread’ yerine ‘data.table::fread’ gibi tam yolunu kullanmanız gerekir. Bireysel işlevlerle ilgili yardım için sözdizimi help("fread") veya ?fread şeklindedir. Yine, paket yüklenmemişse, tam adı ?data.table::fread gibi kullanın.

data.table oluşturma #

data.table, base R’den data.frame sınıfının geliştirilmiş bir versiyonudur. Bu nedenle, “class()” niteliği, “data.table” “data.frame” vektörüdür ve bir veri üzerinde çalışan fonksiyonlardır. .frame ayrıca bir data.table ile de çalışacaktır. Bir data.table oluşturmanın, yüklemenin veya bu tabloya zorlamanın birçok yolu vardır.

İnşa etmek #

data.table paketini kurmayı ve etkinleştirmeyi unutmayın.

library(data.table)

Aynı isimde bir kurucu var:

DT <- data.table(
  x = letters[1:5], 
  y = 1:5, 
  z = (1:5) > 3
)
#    x y     z
# 1: a 1 FALSE
# 2: b 2 FALSE
# 3: c 3 FALSE
# 4: d 4  TRUE
# 5: e 5  TRUE

“data.frame"den farklı olarak “data.table”, dizeleri faktörlere zorlamaz:

sapply(DT, class)
#               x           y           z 
#     "character"   "integer"   "logical"

Oku #

Bir metin dosyasından okuyabiliriz:

dt <- fread("my_file.csv")

“read.csv"den farklı olarak “fread”, dizeleri faktör olarak değil, dizeler olarak okuyacaktır.

Bir data.frame’i değiştirin #

Verimlilik için data.table, yerinde bir data.table yapmak için bir data.frame veya listeyi değiştirmenin bir yolunu sunar (kopyalamadan veya hafıza konumunu değiştirmeden):

# example data.frame
DF <- data.frame(x = letters[1:5], y = 1:5, z = (1:5) > 3)
# modification
setDT(DF)

“DF” nesnesi yerinde değiştirildiği için sonucu “<-” atamadığımıza dikkat edin. data.frame’in sınıf nitelikleri korunacaktır:

sapply(DF, class)
#         x         y         z 
#  "factor" "integer" "logical"

Nesneyi data.table’a zorla #

**Bir “list”, “data.frame” veya “data.table” varsa, bir “data.table"a dönüştürmek için “setDT” işlevini kullanmalısınız, çünkü bunun yerine dönüştürmeyi referansa göre yapar (‘as.data.table’ın yaptığı) bir kopya oluşturma. Büyük veri kümeleriyle çalışıyorsanız bu önemlidir.

Başka bir R nesneniz varsa (matriks gibi), onu bir “data.table"a zorlamak için “as.data.table” kullanmalısınız.

mat <- matrix(0, ncol = 10, nrow = 10)

DT <- as.data.table(mat)
# or
DT <- data.table(mat)

data.table’daki özel semboller #

.SD #

“.SD”, “by” içinde kullanılan tüm sütunlar hariç olmak üzere, her grup için “veri.tablosu"nun alt kümesini ifade eder.

“.SD” ve “lapply”, herhangi bir işlevi bir “data.table” içindeki gruplara göre birden çok sütuna uygulamak için kullanılabilir.

Aynı yerleşik veri kümesini, “mtcars"ı kullanmaya devam edeceğiz:

mtcars = data.table(mtcars) # Let's not include rownames to keep things simpler

silindir sayısı, “silindir” ile veri kümesindeki tüm sütunların ortalaması:

mtcars[ , lapply(.SD, mean), by = cyl]

#   cyl      mpg     disp        hp     drat       wt     qsec        vs        am     gear     carb
#1:   6 19.74286 183.3143 122.28571 3.585714 3.117143 17.97714 0.5714286 0.4285714 3.857143 3.428571
#2:   4 26.66364 105.1364  82.63636 4.070909 2.285727 19.13727 0.9090909 0.7272727 4.090909 1.545455
#3:   8 15.10000 353.1000 209.21429 3.229286 3.999214 16.77214 0.0000000 0.1428571 3.285714 3.500000

Veri setinde ‘cyl’ dışında ‘vs’, ‘am’, ‘gear’ ve ‘carb’ gibi başka kategorik sütunlar da vardır. Bu sütunların ‘ortalamalarını’ almak gerçekten mantıklı değil. Öyleyse bu sütunları hariç tutalım. İşte burada “.SDcols” devreye giriyor.

.SDcol’ler #

“.SDcols”, “.SD” içinde bulunan “data.table” sütunlarını belirtir.

dişli sayısı “dişli” ve silindir sayısı, “silindir” ile veri kümesindeki tüm sütunların (sürekli sütunlar) ortalaması, “dişli” ve “silindir” ile düzenlenir:

# All the continuous variables in the dataset
cols_chosen <- c("mpg", "disp", "hp", "drat", "wt", "qsec")

mtcars[order(gear, cyl), lapply(.SD, mean), by = .(gear, cyl), .SDcols = cols_chosen]

Belki de “ortalama"yı gruplara göre hesaplamak istemiyoruz. Veri kümesindeki tüm arabaların ortalamasını hesaplamak için “by” değişkenini belirtmiyoruz.

mtcars[ , lapply(.SD, mean), .SDcols = cols_chosen] 

#        mpg     disp       hp     drat      wt     qsec

iris[, .(count=.N), by=Species]

Sütun ekleme ve değiştirme #

DT[nerede, seç|güncelleme|yap, by] sözdizimi bir data.table sütunlarıyla çalışmak için kullanılır.

  • “nerede” kısmı “i” argümanıdır
  • “select|update|do” kısmı j argümanıdır

Bu iki bağımsız değişken genellikle ad yerine konuma göre iletilir.

Aşağıdaki örnek verilerimiz

mtcars = data.table(mtcars, keep.rownames = TRUE)

Tüm sütunları düzenleme #

Yeni sütunlar atamak için ‘j’ içindeki ‘:=’ operatörünü kullanın:

mtcars[, mpg_sq := mpg^2]

‘NULL’ olarak ayarlayarak sütunları kaldırın:

mtcars[, mpg_sq := NULL]

:= operatörünün çok değişkenli biçimini kullanarak birden çok sütun ekleyin:

mtcars[, `:=`(mpg_sq = mpg^2, wt_sqrt = sqrt(wt))]
# or 
mtcars[, c("mpg_sq", "wt_sqrt") := .(mpg^2, sqrt(wt))]

Sütunlar bağımlıysa ve sırayla tanımlanmaları gerekiyorsa, bir yol şudur:

mtcars[, c("mpg_sq", "mpg2_hp") := .(temp1 <- mpg^2, temp1/hp)]

.() sözdizimi, LHS := RHS öğesinin sağ tarafında bir sütun listesi olduğunda kullanılır.

Dinamik olarak belirlenen sütun adları için parantez kullanın:

vn = "mpg_sq"
mtcars[, (vn) := mpg^2]

Sütunlar “set” ile de değiştirilebilir, ancak bu nadiren gerekli olur:

set(mtcars, j = "hp_over_wt", v = mtcars$hp/mtcars$wt)

Sütunların alt kümelerini düzenleme #

“i” bağımsız değişkenini, “düzenlemelerin” yapılması gereken satırlara alt kümelemek için kullanın:

mtcars[1:3, newvar := "Hello"]
# or
set(mtcars, j = "newvar", i = 1:3, v = "Hello")

Bir data.frame’de olduğu gibi, satır numaralarını veya mantıksal testleri kullanarak alt kümeler yapabiliriz. ‘i’ içinde bir “join” kullanmak da mümkündür, ancak bu daha karmaşık görev başka bir örnekte ele alınmıştır.

Sütun niteliklerini düzenleme #

’levels<-’ veya ’names<-’ gibi nitelikleri düzenleyen işlevler, aslında bir nesneyi değiştirilmiş bir kopyayla değiştirir. Bir data.table’da yalnızca bir sütunda kullanılsa bile, tüm nesne kopyalanır ve değiştirilir.

Bir nesneyi kopyalar olmadan değiştirmek için, bir data.table veya data.frame’in sütun adlarını değiştirmek için “setnames"i ve herhangi bir nesnenin özniteliğini değiştirmek için “setattr"yi kullanın.

# Print a message to the console whenever the data.table is copied
tracemem(mtcars)
mtcars[, cyl2 := factor(cyl)]

# Neither of these statements copy the data.table
setnames(mtcars, old = "cyl2", new = "cyl_fac")
setattr(mtcars$cyl_fac, "levels", c("four", "six", "eight"))

# Each of these statements copies the data.table
names(mtcars)[names(mtcars) == "cyl_fac"] <- "cf"
levels(mtcars$cf) <- c("IV", "VI", "VIII")

Bu değişikliklerin referans olarak yapıldığını, dolayısıyla küresel olduklarını unutmayın. Bunları bir ortamda değiştirmek, nesneyi tüm ortamlarda etkiler.

# This function also changes the levels in the global environment
edit_levels <- function(x) setattr(x, "levels", c("low", "med", "high"))
edit_levels(mtcars$cyl_factor)

Hem data.frame hem de data.table ile uyumlu kod yazma #

Alt küme sözdizimindeki farklılıklar #

Bir “veri.tablosu”, “veri.çerçevesi”, “matris” ve (2B) “dizi"nin yanı sıra R’de bulunan birkaç iki boyutlu veri yapısından biridir. Bu sınıfların tümü, “A[satırlar, sütunlar]” şemasını alt kümeleme için çok benzer ancak aynı olmayan bir sözdizimi kullanır.

Bir “matris”, bir “veri.çerçevesi” ve bir “veri.tablosu” içinde depolanan aşağıdaki verileri göz önünde bulundurun:

ma <- matrix(rnorm(12), nrow=4, dimnames=list(letters[1:4], c('X', 'Y', 'Z')))
df <- as.data.frame(ma)
dt <- as.data.table(ma)

ma[2:3]  #---> returns the 2nd and 3rd items, as if 'ma' were a vector (because it is!)
df[2:3]  #---> returns the 2nd and 3rd columns
dt[2:3]  #---> returns the 2nd and 3rd rows!

Neyin iade edileceğinden emin olmak istiyorsanız, açık olmak daha iyidir.

Belirli satırları almak için aralıktan sonra bir virgül eklemeniz yeterlidir:

ma[2:3, ]  # \
df[2:3, ]  #  }---> returns the 2nd and 3rd rows
dt[2:3, ]  # /

Ancak, sütunları alt kümeye eklemek isterseniz, bazı durumlar farklı yorumlanır. Herşey üçü, bir dizinde saklanan not tamsayı veya karakter dizinleriyle aynı şekilde alt küme olabilir. değişken.

ma[, 2:3]          #  \
df[, 2:3]          #   \
dt[, 2:3]          #    }---> returns the 2nd and 3rd columns
ma[, c("Y", "Z")]  #   /
df[, c("Y", "Z")]  #  /
dt[, c("Y", "Z")]  # /

Ancak, alıntılanmamış değişken adları için farklılık gösterirler.

mycols <- 2:3
ma[, mycols]                # \
df[, mycols]                #  }---> returns the 2nd and 3rd columns
dt[, mycols, with = FALSE]  # /

dt[, mycols]                # ---> Raises an error

Son durumda, “mycols” bir sütunun adı olarak değerlendirilir. Çünkü ‘dt’ bulamıyor “mycols” adlı bir sütun, bir hata oluşur.

Not: “data.table” paketinin 1.9.8’den önceki sürümleri için bu davranış biraz farklı. Sütun dizinindeki herhangi bir şey “dt” olarak değerlendirilebilirdi. çevre. Böylece hem “dt[, 2:3]” hem de “dt[, mycols]”, “2:3” vektörünü döndürür. Numara ikinci durumda hata ortaya çıkar, çünkü “mycols” değişkeni ebeveyn ortamı.

data.frame ve data.table ile uyumluluğu koruma stratejileri #

“data.frame” ve “data.table” ile çalışması garanti edilen kod yazmanın birçok nedeni vardır. Belki `data.frame’ kullanmak zorunda kalıyorsunuz ya da nasıl kullanılacağını bilmediğiniz bazı kodları paylaşmanız gerekebilir. Dolayısıyla, kolaylık sırasına göre bunu başarmak için bazı ana stratejiler vardır:

  1. Her iki sınıf için de aynı şekilde davranan sözdizimi kullanın.
  2. En kısa sözdizimi ile aynı şeyi yapan ortak bir işlev kullanın.
  3. “data.table” öğesini “data.frame” gibi davranmaya zorlayın (ör.: “print.data.frame” özel yöntemini çağırın).
  4. Onlara, sonuçta oldukları gibi ’liste’ olarak davranın.
  5. Herhangi bir şey yapmadan önce tabloyu bir ‘data.frame’e dönüştürün (büyük bir tabloysa kötü bir fikir).
  6. Bağımlılıklar önemli değilse, tabloyu ‘data.table’a dönüştürün.

Satırları alt kümeler. Çok basit, sadece [, ] seçicisini kullanın, virgülle _olun:

A[1:10, ]
A[A$var > 17, ]  # A[var > 17, ] just works for data.table

Alt küme sütunları. Tek bir sütun istiyorsanız, “$” veya “[[ ]]” seçicisini kullanın:

A$var
colname <- 'var'
A[[colname]]
A[[1]]

Birden fazla sütunu almak için tek tip bir yol istiyorsanız, biraz itiraz etmeniz gerekir:

B <- `[.data.frame`(A, 2:4)

# We can give it a better name
select <- `[.data.frame`
B <- select(A, 2:4)
C <- select(A, c('foo', 'bar'))

Alt küme ‘dizine eklenmiş’ satırlar. ‘data.frame’, ‘satır.adları’na sahipken, ‘data.table’ın kendine özgü ‘anahtar’ özelliği vardır. En iyi şey, ‘row.names’den tamamen kaçınmak ve mümkün olduğunda ‘data.table’ durumunda mevcut optimizasyonlardan yararlanmaktır.

B <- A[A$var != 0, ]
# or...
B <- with(A, A[var != 0, ])  # data.table will silently index A by var before subsetting

stuff <- c('a', 'c', 'f')
C <- A[match(stuff, A$name), ]  # really worse than: setkey(A); A[stuff, ]

1 sütunlu bir tablo alın, vektör olarak bir satır alın. Şimdiye kadar gördüklerimizle bunlar kolay:

B <- select(A, 2)    #---> a table with just the second column
C <- unlist(A[1, ])  #---> the first row as a vector (coerced if necessary)

data.table’da anahtarları ayarlama #

Evet, 1.9.6 öncesi SETKEY yapmanız gerekir

Geçmişte (1.9.6 öncesi), ‘data.table’ınız, özellikle büyük tablolar için, tablonun anahtarları olarak sütunlar ayarlanarak hızlandırıldı. [Arama hızının 544 kat daha iyi olduğu Eylül 2015 sürümünün intro skeç sayfası 5‘sine bakın.] masayı kurmak.

library(data.table)
DT <- data.table(
  x = letters[1:5], 
  y = 5:1, 
  z = (1:5) > 3
)

#> DT
#   x y     z
#1: a 5 FALSE
#2: b 4 FALSE
#3: c 3 FALSE
#4: d 2  TRUE
#5: e 1  TRUE

Anahtarınızı setkey komutuyla ayarlayın. Birden çok sütunlu bir anahtarınız olabilir.

setkey(DT, y)

Tablolarınızın anahtarını tablolarda kontrol edin()

tables()

> tables()
     NAME NROW NCOL MB COLS  KEY
[1,] DT      5    3  1 x,y,z y  
Total: 1MB

Bunun verilerinizi yeniden sıralayacağını unutmayın.

#> DT
#   x y     z
#1: e 1  TRUE
#2: d 2  TRUE
#3: c 3 FALSE
#4: b 4 FALSE
#5: a 5 FALSE

Artık gereksiz

v1.9.6’dan önce, özellikle tabloları birleştirmek için belirli işlemler için bir anahtar ayarlamanız gerekiyordu. data.table geliştiricileri hızlandı ve anahtarlara olan bağımlılığı değiştirebilecek bir “on=”” özelliğini tanıttı. Ayrıntılı bir tartışma için [SO yanıtı burada] bölümüne bakın.2

Ocak 2017’de geliştiriciler, “açık” sözdizimini açıklayan ve hızlı indeksleme için diğer sütunların tanımlanmasına izin veren bir ikincil indeksler etrafında vinyet yazdılar.

İkincil endeksler mi oluşturuyorsunuz?

Anahtara benzer bir şekilde, setindex(DT, key.col) veya setindexv(DT, "key.col.string") yapabilirsiniz, burada DT sizin data.table’ınızdır. setindex(DT, NULL) ile tüm indeksleri kaldırın.

İkincil endekslerinizi ’endeksler(DT)’ ile görün.

Neden ikincil endeksler?

Bu, tabloyu sıralamaz (anahtarın aksine), ancak “on” sözdizimini kullanarak hızlı indekslemeye izin verir. Yalnızca bir anahtar olabileceğini unutmayın, ancak birden çok ikincil dizin kullanabilirsiniz; bu, tabloyu yeniden anahtarlamak ve yeniden ayarlamak zorunda kalmaktan kurtarır. Bu, alt kümelemek istediğiniz sütunları değiştirirken alt kümenizi hızlandıracaktır.

Hatırlayın, yukarıdaki örnekte y, DT tablosunun anahtarıydı:

DT
# x y     z
# 1: e 1  TRUE
# 2: d 2  TRUE
# 3: c 3 FALSE
# 4: b 4 FALSE
# 5: a 5 FALSE

# Let us set x as index 
setindex(DT, x)

# Use indices to see what has been set
indices(DT)
# [1] "x"

# fast subset using index and not keyed column
DT["c", on ="x"]
#x y     z
#1: c 3 FALSE

# old way would have been rekeying DT from y to x, doing subset and 
# perhaps keying back to y (now we save two sorts)
# This is a toy example above but would have been more valuable with big data sets
