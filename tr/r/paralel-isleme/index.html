Uzak makinelerde paralelleştirme, her makinede kitaplıkların indirilmesini gerektirir. package::function() çağrılarını tercih edin. Birkaç pakette “caret”, “pls” ve “plyr” dahil olmak üzere yerel olarak yerleşik paralelleştirme vardır.

Microsoft R Open (Revolution R) ayrıca birçok ortak işlevi özünde paralel hale getiren çok iş parçacıklı BLAS/LAPACK kitaplıkları kullanır.

Paralel paket ile paralel işleme #

Temel paket ‘paralel’, çatallama, soketler ve rasgele sayı üretimi yoluyla paralel hesaplamaya izin verir.

Yerel ana bilgisayarda bulunan çekirdek sayısını tespit edin:

parallel::detectCores(all.tests = FALSE, logical = TRUE)

Yerel ana bilgisayarda bir çekirdek kümesi oluşturun:

parallelCluster <- parallel::makeCluster(parallel::detectCores())

Öncelikle paralelleştirmeye uygun bir fonksiyon oluşturulmalıdır. “mtcars” veri kümesini düşünün. “mpg” üzerindeki bir regresyon, her “cyl” seviyesi için ayrı bir regresyon modeli yaratılarak geliştirilebilir.

data <- mtcars
yfactor <- 'cyl'
zlevels <- sort(unique(data[[yfactor]]))
datay <- data[,1]
dataz <- data[,2]
datax <- data[,3:11]


fitmodel <- function(zlevel, datax, datay, dataz) {
  glm.fit(x = datax[dataz == zlevel,], y = datay[dataz == zlevel])
}

“zlevels"in tüm olası yinelemeleri arasında dolaşabilen bir işlev oluşturun. Bu hala seri haldedir, ancak paralel hale getirilecek kesin süreci belirlediği için önemli bir adımdır.

fitmodel <- function(zlevel, datax, datay, dataz) {
  glm.fit(x = datax[dataz == zlevel,], y = datay[dataz == zlevel])
}


for (zlevel in zlevels) {
  print("*****")
  print(zlevel)
  print(fitmodel(zlevel, datax, datay, dataz))
}

Bu işlevi körleyin:

worker <- function(zlevel) {
    fitmodel(zlevel,datax, datay, dataz)
  }

‘Paralel’ kullanan paralel bilgi işlem, küresel ortama erişemez. Neyse ki, her işlev ‘paralel’ erişilebilen bir yerel ortam oluşturur. Bir sarmalayıcı işlevinin oluşturulması, paralelleştirmeye izin verir. Uygulanacak işlevin de ortam içine yerleştirilmesi gerekir.

wrapper <- function(datax, datay, dataz) {
  # force evaluation of all paramters not supplied by parallelization apply
  force(datax)
  force(datay)
  force(dataz)
  # these variables are now in an enviroment accessible by parallel function
  
  # function to be applied also in the environment
  fitmodel <- function(zlevel, datax, datay, dataz) {
    glm.fit(x = datax[dataz == zlevel,], y = datay[dataz == zlevel])
  }
  
  # calling in this environment iterating over single parameter zlevel
  worker <- function(zlevel) {
    fitmodel(zlevel,datax, datay, dataz)
  }
  return(worker) 
}

Şimdi bir küme oluşturun ve sarmalayıcı işlevini çalıştırın.

parallelcluster <- parallel::makeCluster(parallel::detectCores())
models <- parallel::parLapply(parallelcluster,zlevels,
                              wrapper(datax, datay, dataz))

Bittiğinde her zaman kümeyi durdurun.

parallel::stopCluster(parallelcluster)

“Paralel” paket, “par” ön eki ile birlikte tüm “apply()” ailesini içerir.

foreach paketi ile paralel işleme #

‘foreach’ paketi R’ye paralel işlemenin gücünü getiriyor. Ancak çok çekirdekli CPU’ları kullanmadan önce çok çekirdekli bir küme atamanız gerekiyor. ‘doSNOW’ paketi bir olasılıktır.

Foreach döngüsünün basit bir kullanımı, 1’den 100000’e kadar olan tüm sayıların karekökünün ve karesinin toplamını hesaplamaktır.

library(foreach)
library(doSNOW)

cl <- makeCluster(5, type = "SOCK")
registerDoSNOW(cl)

f <- foreach(i = 1:100000, .combine = c, .inorder = F) %dopar% {
    k <- i ** 2 + sqrt(i)
    k
}

“foreach” çıktısının yapısı “.combine” argümanı tarafından kontrol edilir. Varsayılan çıktı yapısı bir listedir. Yukarıdaki kodda, bunun yerine bir vektör döndürmek için “c” kullanılır. “+”’ gibi bir hesaplama fonksiyonunun (veya operatörünün) ayrıca bir hesaplama yapmak ve daha fazla işlenmiş nesneyi döndürmek için kullanılabileceğini unutmayın.

Her foreach döngüsünün sonucunun son çağrı olduğunu belirtmek önemlidir. Böylece, bu örnekte sonuca k eklenecektir.

Parametre Ayrıntılar
.birleştirmek Fonksiyonu birleştirin. Döngü sonuçlarının nasıl birleştirileceğini belirler. Olası değerler c, cbind, rbind, "+", "*"…
.sıralama “TRUE” ise, sonuç yineleme değişkeninin (burada “i”) sırasına göre sıralanır. ‘YANLIŞ’ ise sonuç sıralanmaz. Bunun hesaplama süresi üzerinde olumlu etkileri olabilir.
.paketler ‘base’ dışında herhangi bir paket tarafından sağlanan işlevler için, ör. mass, randomForest veya başka, bu paketleri c("mass", "randomForest") ile sağlamanız gerekir.

Rastgele Sayı Üretimi #

Paralelleştirme ile ilgili büyük bir sorun, tohum olarak RNG’nin kullanılmasıdır. Sayıya göre rastgele sayılar, ya oturumun başlangıcından ya da en son “set.seed()“den itibaren işlem sayısına göre yinelenir. Paralel işlemler aynı işlevden kaynaklandığından, aynı tohumu kullanabilir ve muhtemelen aynı sonuçlara neden olabilir! Aramalar farklı çekirdeklerde seri olarak çalışacak, hiçbir avantaj sağlamayacaktır.

Bir dizi tohum üretilmeli ve her paralel işleme gönderilmelidir. Bu, bazı paketlerde otomatik olarak yapılır (“paralel”, “kar” vb.), ancak diğerlerinde açıkça ele alınmalıdır.

s <- seed
for (i in 1:numofcores) {
    s <- nextRNGStream(s)
    # send s to worker i as .Random.seed
}

Tohumlar ayrıca tekrarlanabilirlik için ayarlanabilir.

clusterSetRNGStream(cl = parallelcluster, iseed)

mcparallelDo #

mcparallelDo paketi, Unix benzeri (örn. Linux ve MacOSX) işletim sistemlerinde R kodunun eşzamansız olarak değerlendirilmesine izin verir. Paketin altında yatan felsefe, kodlamadan ziyade keşifsel veri analizi ihtiyaçları ile uyumludur. Eşzamansızlığı kodlamak için future paketini göz önünde bulundurun.

Örnek #

Veri oluştur

data(ToothGrowth)

Bir çatal üzerinde analiz yapmak için mcparallelDo’yu tetikleyin

mcparallelDo({glm(len ~ supp * dose, data=ToothGrowth)},"interactionPredictorModel")

Başka şeyler yapın, örn.

binaryPredictorModel <- glm(len ~ supp, data=ToothGrowth)
gaussianPredictorModel <- glm(len ~ dose, data=ToothGrowth)

mcparallelDo’nun sonucu, targetEnvironment’ınızda, ör. .GlobalEnv, bir mesajla tamamlandığında (varsayılan olarak)

summary(interactionPredictorModel)

Diğer Örnekler #

# Example of not returning a value until we return to the top level
for (i in 1:10) {
  if (i == 1) {
    mcparallelDo({2+2}, targetValue = "output")
  }
  if (exists("output")) print(i)
}

# Example of getting a value without returning to the top level
for (i in 1:10) {
  if (i == 1) {
    mcparallelDo({2+2}, targetValue = "output")
  }
  mcparallelDoCheck()
  if (exists("output")) print(i)
}
