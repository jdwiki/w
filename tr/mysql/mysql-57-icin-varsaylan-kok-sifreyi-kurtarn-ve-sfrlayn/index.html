MySQL 5.7+ için varsayılan kök şifreyi kurtarın ve sıfırlayın

  • Sunucunun ilk açılışında ne olur?
  • Varsayılan şifreyi kullanarak kök şifre nasıl değiştirilir
  • UNIX soket dosyası için " /var/run/mysqld" olmadığında kök parolayı sıfırla

Bu sayfada

  • Sunucunun ilk açılışında ne olur?
  • Varsayılan şifreyi kullanarak kök şifre nasıl değiştirilir
  • UNIX soket dosyası için " /var/run/mysqld" olmadığında kök parolayı sıfırla

MySQL 5.7’den sonra, MySQL’i kurduğumuzda bazen bir root hesabı oluşturmamıza veya bir root şifresi vermemize gerek kalmıyor. Varsayılan olarak sunucuyu başlattığımızda, varsayılan şifre mysqld.log dosyasında saklanır. Bu şifreyi kullanarak sisteme giriş yapmamız ve değiştirmemiz gerekiyor.

Bu yöntemi kullanarak varsayılan kök parolayı kurtarmak ve sıfırlamak yalnızca MySQL 5.7+ için geçerlidir.

Sunucunun ilk açılışında ne olur? #

Sunucunun veri dizini boş olduğu göz önüne alındığında:

  • Sunucu başlatıldı.
  • Veri dizininde SSL sertifikası ve anahtar dosyaları oluşturulur.
  • validate_password eklentisi yüklendi ve etkinleştirildi.
  • ‘root’@’localhost’ süper kullanıcı hesabı oluşturulur. Süper kullanıcı için parola ayarlanır ve hata günlüğü dosyasında saklanır.

Varsayılan şifreyi kullanarak kök şifre nasıl değiştirilir #

Varsayılan “kök” parolayı ortaya çıkarmak için:

shell> sudo grep 'temporary password' /var/log/mysqld.log

Oluşturulan geçici şifre ile giriş yaparak kök şifreyi en kısa sürede değiştirin ve süper kullanıcı hesabı için özel bir şifre belirleyin:

shell> mysql -uroot -p 

mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY 'MyNewPass5!';

Not: MySQL’in validate_password eklentisi varsayılan olarak yüklenir. Bu, şifrelerin en az bir büyük harf, bir küçük harf, bir rakam ve bir özel karakter içermesini ve toplam şifre uzunluğunun en az 8 karakter olmasını gerektirir.

UNIX soket dosyası için " /var/run/mysqld" olmadığında kök parolayı sıfırla #

Şifreyi unutursam hata alırım.

$ mysql -u root -p

Şifre girin:

HATA 1045 (28000): ‘root’@’localhost’ kullanıcısı için erişim reddedildi (şifre kullanılarak: EVET)

Önce durumu bilerek sorunu çözmeye çalıştım:

$ systemctl status mysql.service

mysql.service - MySQL Topluluk Sunucusu Yüklendi: yüklendi (/lib/systemd/system/mysql.service; etkin; satıcı ön ayarı: tr Aktif: aktif (devam ediyor) Per 2017-06-08 14:31:33 IST’den beri; 38 saniye önce

Sonra mysqld_safe --skip-grant-tables & kodunu kullandım ama şu hatayı alıyorum:

UNIX soket dosyası için mysqld_safe Dizini ‘/var/run/mysqld’ mevcut değil.

$ systemctl stop  mysql.service
$ ps -eaf|grep mysql
$ mysqld_safe --skip-grant-tables &

çözdüm:

$ mkdir -p /var/run/mysqld
$ chown mysql:mysql /var/run/mysqld

Şimdi aynı kodu mysqld_safe --skip-grant-tables & kullanıyorum ve

mysqld_safe /var/lib/mysql’den veritabanlarıyla mysqld arka plan programı başlatma

$ mysql -u root kullanırsam şunu elde ederim:

Sunucu sürümü: 5.7.18-0ubuntu0.16.04.1 (Ücretsiz)

Telif hakkı (c) 2000, 2017, Oracle ve/veya bağlı kuruluşları. Tüm hakları Saklıdır.

mysql>

Şimdi şifre değiştirme zamanı:

mysql> use mysql
mysql> describe user;

Tablo ve sütun adlarının tamamlanması için tablo bilgilerini okuma -A ile daha hızlı bir başlangıç ​​elde etmek için bu özelliği kapatabilirsiniz.

Veritabanı değişti

mysql> FLUSH PRIVILEGES;
mysql> SET PASSWORD FOR root@'localhost' = PASSWORD('newpwd');

veya Her yerden bağlanabilen bir mysql root hesabınız varsa şunları da yapmalısınız:

UPDATE mysql.user SET Password=PASSWORD('newpwd') WHERE User='root';

Alternatif Yöntem:

   USE mysql
   UPDATE user SET Password = PASSWORD('newpwd')
   WHERE Host = 'localhost' AND User = 'root';

Ve her yerden erişebilen bir kök hesabınız varsa:

 USE mysql
 UPDATE user SET Password = PASSWORD('newpwd')

FLUSH PRIVILEGES;
sudo /etc/init.d/mysql stop
sudo /etc/init.d/mysql start

mysql>
