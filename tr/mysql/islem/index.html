İşlemi Başlat #

İşlem, tek bir iş birimi olarak gerçekleştirilen, seçme, ekleme, güncelleme veya silme gibi sıralı bir SQL ifadesi grubudur.

Başka bir deyişle, grup içindeki her bir işlem başarılı olmadıkça bir işlem asla tamamlanmayacaktır. İşlem içindeki herhangi bir işlem başarısız olursa, işlemin tamamı başarısız olur.

Banka işlemi bunu açıklamak için en iyi örnek olacaktır. İki hesap arasında bir transfer düşünün. Bunu başarmak için aşağıdakileri yapan SQL ifadeleri yazmanız gerekir.

  1. İlk hesapta istenen miktarın uygunluğunu kontrol edin
  2. İstenen tutarı ilk hesaptan düş
  3. İkinci hesaba yatırın

Bu süreçten herhangi biri başarısız olursa, bütün önceki durumuna döndürülmelidir.

ASİT : İşlemlerin Özellikleri

İşlemler aşağıdaki dört standart özelliğe sahiptir

  • Atomisite: iş birimi içindeki tüm işlemlerin başarıyla tamamlandı; aksi takdirde işlem iptal edilir başarısızlık noktası ve önceki işlemler geri alınır eski devlet.
  • Tutarlılık: veritabanının belirli bir süre sonra durumları düzgün bir şekilde değiştirmesini sağlar. başarıyla gerçekleştirilen işlem.
  • İzolasyon: işlemlerin bağımsız olarak çalışmasını sağlar ve birbirine şeffaf.
  • Dayanıklılık: taahhüt edilen bir işlemin sonucunun veya etkisinin sistem arızası durumunda işlem devam eder.

İşlemler “İŞLEM BAŞLAT” veya “İŞE BAŞLA” ifadesi ile başlar ve “TAMAM” veya “GERİ DÖNÜŞ” ifadesi ile biter. Başlangıç ​​ve bitiş deyimleri arasındaki SQL komutları, işlemin büyük kısmını oluşturur.

START TRANSACTION;
SET @transAmt = '500';
SELECT @availableAmt:=ledgerAmt FROM accTable WHERE customerId=1 FOR UPDATE;
UPDATE accTable SET ledgerAmt=ledgerAmt-@transAmt WHERE customerId=1;
UPDATE accTable SET ledgerAmt=ledgerAmt+@transAmt WHERE customerId=2;
COMMIT;

‘İŞLEMİ BAŞLAT’ ile, işlemi ‘COMMIT’ veya ‘ROLLBACK’ ile sonlandırana kadar otomatik taahhüt devre dışı kalır. Otomatik taahhüt modu daha sonra önceki durumuna geri döner.

“GÜNCELLEME İÇİN” işlemin süresi için satırları belirtir (ve kilitler).

İşlem taahhütsüz kalırken, bu işlem diğer kullanıcılar için geçerli olmayacaktır.

İşlemle İlgili Genel Prosedürler

  • ‘BEGIN WORK’ veya ‘START’ SQL komutunu vererek işleme başlayın İŞLEM’.
  • Tüm SQL ifadelerinizi çalıştırın.
  • Her şeyin ihtiyacınıza göre yürütülüp yürütülmediğini kontrol edin.
  • Evet ise, ‘COMMIT’ komutunu verin, aksi takdirde ‘ROLLBACK’ komutunu verin her şeyi önceki durumuna döndürme komutu.
  • Galera/PXC kullanıyorsanız veya sonunda kullanabilecekseniz, “COMMIT” işleminden sonra bile hataları kontrol edin.

KOMIT, GERİ AL ve OTOMATİK KOMİTE #

OTOMATİK KOMİTE

MySQL, bir işlemin parçası olmayan ifadeleri otomatik olarak taahhüt eder. Başında bir “BEGIN” veya “START TRANSACTION” olmayan herhangi bir “UPDATE”, “DELETE” veya “INSERT” ifadesinin sonuçları tüm bağlantılar tarafından hemen görünür olacaktır.

“AUTOCOMMIT” değişkeni varsayılan olarak true değerine ayarlanmıştır. Bu, aşağıdaki şekilde değiştirilebilir,

--->To make autcommit false
SET AUTOCOMMIT=false;
--or
SET AUTOCOMMIT=0;

--->To make autcommit true
SET AUTOCOMMIT=true;
--or
SET AUTOCOMMIT=1;

‘AUTOCOMMIT’ durumunu görüntülemek için

SELECT @@autocommit;

İŞLEMEK

“AUTOCOMMIT” false olarak ayarlanırsa ve işlem gerçekleştirilmezse, değişiklikler yalnızca mevcut bağlantı için görünür olacaktır.

COMMIT deyimi tablodaki değişiklikleri onayladıktan sonra, sonuç tüm bağlantılar için görünür olacaktır.

Bunu açıklamak için iki bağlantı düşünüyoruz.

Bağlantı 1

--->Before making autocommit false one row added in a new table
mysql> INSERT INTO testTable VALUES (1);

--->Making autocommit = false
mysql> SET autocommit=0;

mysql> INSERT INTO testTable VALUES (2), (3);    
mysql> SELECT * FROM testTable;
+-----+
| tId |
+-----+
|   1 |
|   2 |
|   3 |
+-----+

Bağlantı 2

mysql> SELECT * FROM testTable;
+-----+
| tId |
+-----+
|   1 |
+-----+
---> Row inserted before autocommit=false only visible here

Bağlantı 1

mysql> COMMIT;
--->Now COMMIT is executed in connection 1
mysql> SELECT * FROM testTable;
    +-----+
    | tId |
    +-----+
    |   1 |
    |   2 |
    |   3 |
    +-----+

Bağlantı 2

mysql> SELECT * FROM testTable;
    +-----+
    | tId |
    +-----+
    |   1 |
    |   2 |
    |   3 |
    +-----+
--->Now all the three rows are visible here

GERİ DÖNÜŞ

Sorgu yürütmenizde bir şeyler ters giderse, değişiklikleri geri almak için “ROLLBACK” kullanılır. Aşağıdaki açıklamaya bakın

--->Before making autocommit false one row added in a new table
mysql> INSERT INTO testTable VALUES (1);

--->Making autocommit = false
mysql> SET autocommit=0;

mysql> INSERT INTO testTable VALUES (2), (3);    
mysql> SELECT * FROM testTable;
+-----+
| tId |
+-----+
|   1 |
|   2 |
|   3 |
+-----+

Şimdi ‘ROLLBACK’ yürütüyoruz

--->Rollback executed now
mysql> ROLLBACk;

mysql> SELECT * FROM testTable;
+-----+
| tId |
+-----+
|   1 |
+-----+
--->Rollback removed all rows which all are not committed

“COMMIT” yürütüldüğünde, “ROLLBACK” hiçbir şeye neden olmaz

mysql> INSERT INTO testTable VALUES (2), (3);    
mysql> SELECT * FROM testTable;
mysql> COMMIT;
+-----+
| tId |
+-----+
|   1 |
|   2 |
|   3 |
+-----+

--->Rollback executed now
mysql> ROLLBACk;

mysql> SELECT * FROM testTable;
+-----+
| tId |
+-----+
|   1 |
|   2 |
|   3 |
+-----+
--->Rollback not removed any rows

“AUTOCOMMIT” true olarak ayarlanmışsa, “COMMIT” ve “ROLLBACK” işe yaramaz

JDBC Sürücüsünü kullanarak işlem #

JDBC sürücüsünü kullanan işlem, bir işlemin nasıl ve ne zaman gerçekleştirileceğini ve geri alınacağını kontrol etmek için kullanılır. MySQL sunucusuna bağlantı, JDBC sürücüsü kullanılarak oluşturulur

MySQL için JDBC sürücüsü buradan indirilebilir

JDBC sürücüsünü kullanarak veritabanına bağlantı kurmaya başlayalım

Class.forName("com.mysql.jdbc.Driver");  
Connection con = DriverManager.getConnection(DB_CONNECTION_URL,DB_USER,USER_PASSWORD);
--->Example for connection url "jdbc:mysql://localhost:3306/testDB");

Karakter Kümeleri : Bu, istemcinin sunucuya SQL ifadeleri göndermek için hangi karakter kümesini kullanacağını gösterir. Ayrıca sunucunun sonuçları istemciye geri göndermek için kullanması gereken karakter setini de belirtir.

Bu, sunucuya bağlantı oluşturulurken belirtilmelidir. Yani bağlantı dizesi şöyle olmalı,

jdbc:mysql://localhost:3306/testDB?useUnicode=true&amp;characterEncoding=utf8

Karakter Kümeleri ve Harmanlamalar hakkında daha fazla ayrıntı için buna bakın

Bağlantıyı açtığınızda, ‘AUTOCOMMIT’ modu varsayılan olarak true olarak ayarlanır, işlemi başlatmak için bu false olarak değiştirilmelidir.

con.setAutoCommit(false);

Bağlantıyı açtıktan hemen sonra her zaman setAutoCommit() yöntemini çağırmalısınız.

Aksi takdirde, yeni bir işlem başlatmak için ‘İŞLEM BAŞLAT’ veya ‘İŞE BAŞLA’yı kullanın. ‘İŞLEMİ BAŞLAT’ veya ‘İŞE BAŞLA’ kullanarak, ‘AUTOCOMMIT’ false‘ı değiştirmeye gerek yoktur. Bu otomatik olarak devre dışı bırakılacaktır.

Şimdi işleme başlayabilirsiniz. Aşağıdaki eksiksiz bir JDBC işlem örneğine bakın.

package jdbcTest;
 
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.SQLException;


public class accTrans {

    public static void doTransfer(double transAmount,int customerIdFrom,int customerIdTo) {

        Connection con = null;
        PreparedStatement pstmt = null;
        ResultSet rs = null;
 
        try {
            String DB_CONNECTION_URL = "jdbc:mysql://localhost:3306/testDB?useUnicode=true&amp;characterEncoding=utf8";

            Class.forName("com.mysql.jdbc.Driver");  
            con = DriverManager.getConnection(DB_CONNECTION_URL,DB_USER,USER_PASSWORD);

            --->set auto commit to false
            con.setAutoCommit(false);
            ---> or use con.START TRANSACTION / con.BEGIN WORK

            --->Start SQL Statements for transaction
            --->Checking availability of amount
            double availableAmt    = 0;
            pstmt = con.prepareStatement("SELECT ledgerAmt FROM accTable WHERE customerId=? FOR UPDATE");
            pstmt.setInt(1, customerIdFrom);
            rs = pstmt.executeQuery();
            if(rs.next())
                availableAmt    = rs.getDouble(1);

            if(availableAmt >= transAmount)
            {
                ---> Do Transfer
                ---> taking amount from cutomerIdFrom
                pstmt = con.prepareStatement("UPDATE accTable SET ledgerAmt=ledgerAmt-? WHERE customerId=?");                        
                pstmt.setDouble(1, transAmount);
                pstmt.setInt(2, customerIdFrom);
                pstmt.executeUpdate();

                ---> depositing amount in cutomerIdTo
                pstmt = con.prepareStatement("UPDATE accTable SET ledgerAmt=ledgerAmt+? WHERE customerId=?");                        
                pstmt.setDouble(1, transAmount);
                pstmt.setInt(2, customerIdTo);
                pstmt.executeUpdate();

                con.commit();
            }
            --->If you performed any insert,update or delete operations before 
            ----> this availability check, then include this else part
            /*else { --->Rollback the transaction if availability is less than required
                con.rollback();
            }*/

        } catch (SQLException ex) {
            ---> Rollback the transaction in case of any error
            con.rollback();
        } finally {
            try {
                if(rs != null)  rs.close();
                if(pstmt != null) pstmt.close();
                if(con != null) con.close();
            }
        }
    }
 
    public static void main(String[] args) {
        doTransfer(500, 1020, 1021);
        -->doTransfer(transAmount, customerIdFrom, customerIdTo);
    }
}

JDBC işlemi, bir işlem bloğu içindeki tüm SQL ifadelerinin başarılı bir şekilde yürütüldüğünden emin olun, işlem bloğu içindeki SQL ifadelerinden herhangi biri başarısız olursa, işlem bloğundaki her şeyi iptal edin ve geri alın.
