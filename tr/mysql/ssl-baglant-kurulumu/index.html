Debian tabanlı sistemler için kurulum #

(Bu, MySQL’in kurulduğunu ve “sudo"nun kullanıldığını varsayar.)

CA ve SSL anahtarları oluşturma #

OpenSSL ve kitaplıkların kurulu olduğundan emin olun:

apt-get -y install openssl
apt-get -y install libssl-dev

Ardından, SSL dosyaları için bir dizin oluşturun ve girin:

mkdir /home/ubuntu/mysqlcerts
cd /home/ubuntu/mysqlcerts

Anahtarları oluşturmak için, anahtarları imzalamak üzere bir sertifika yetkilisi (CA) oluşturun (kendinden imzalı):

openssl genrsa 2048 > ca-key.pem
openssl req -new -x509 -nodes -days 3600 -key ca-key.pem -out ca.pem

Her istemde girilen değerler yapılandırmayı etkilemez. Ardından sunucu için bir anahtar oluşturun ve daha önce CA’yı kullanarak imzalayın:

openssl req -newkey rsa:2048 -days 3600 -nodes -keyout server-key.pem -out server-req.pem
openssl rsa -in server-key.pem -out server-key.pem

openssl x509 -req -in server-req.pem -days 3600 -CA ca.pem -CAkey ca-key.pem -set_serial 01 -out server-cert.pem

Ardından bir müşteri için bir anahtar oluşturun:

openssl req -newkey rsa:2048 -days 3600 -nodes -keyout client-key.pem -out client-req.pem
openssl rsa -in client-key.pem -out client-key.pem
openssl x509 -req -in client-req.pem -days 3600 -CA ca.pem -CAkey ca-key.pem -set_serial 01 -out client-cert.pem

Her şeyin doğru ayarlandığından emin olmak için anahtarları doğrulayın:

openssl verify -CAfile ca.pem server-cert.pem client-cert.pem

MySQL’e anahtar ekleme #

[MySQL yapılandırma dosyasını][1] açın. Örneğin:

vim /etc/mysql/mysql.conf.d/mysqld.cnf

“[mysqld]” bölümünün altına aşağıdaki seçenekleri ekleyin:

ssl-ca = /home/ubuntu/mysqlcerts/ca.pem
ssl-cert = /home/ubuntu/mysqlcerts/server-cert.pem
ssl-key = /home/ubuntu/mysqlcerts/server-key.pem

MySQL’i yeniden başlatın. Örneğin:

service mysql restart

SSL bağlantısını test edin #

Oluşturulan istemci anahtarını kullanarak ‘ssl-ca’, ‘ssl-cert’ ve ‘ssl-key’ ekstra seçeneklerini geçerek aynı şekilde bağlanın. Örneğin, “cd /home/ubuntu/mysqlcerts” varsayarsak:

mysql --ssl-ca=ca.pem --ssl-cert=client-cert.pem --ssl-key=client-key.pem -h 127.0.0.1 -u superman -p

Giriş yaptıktan sonra bağlantının gerçekten güvenli olduğunu doğrulayın:

superman@127.0.0.1 [None]> SHOW VARIABLES LIKE '%ssl%';
+---------------+-----------------------------------------+
| Variable_name | Value                                   |
+---------------+-----------------------------------------+
| have_openssl  | YES                                     |
| have_ssl      | YES                                     |
| ssl_ca        | /home/ubuntu/mysqlcerts/ca.pem          |
| ssl_capath    |                                         |
| ssl_cert      | /home/ubuntu/mysqlcerts/server-cert.pem |
| ssl_cipher    |                                         |
| ssl_crl       |                                         |
| ssl_crlpath   |                                         |
| ssl_key       | /home/ubuntu/mysqlcerts/server-key.pem  |
+---------------+-----------------------------------------+

Ayrıca şunları da kontrol edebilirsiniz:

superman@127.0.0.1 [None]> STATUS;
...
SSL:                    Cipher in use is DHE-RSA-AES256-SHA
...

SSL’yi zorunlu kılma #

Bu, “GRANT” aracılığıyla, “SSL REQUIRE” kullanılarak yapılır:

GRANT ALL PRIVILEGES ON *.* TO 'superman'@'127.0.0.1' IDENTIFIED BY 'pass' REQUIRE SSL;
FLUSH PRIVILEGES;

Şimdi, “süpermen” SSL aracılığıyla bağlanmalıdır.

İstemci anahtarlarını yönetmek istemiyorsanız, önceki istemci anahtarını kullanın ve bunu tüm istemciler için otomatik olarak kullanın. [MySQL yapılandırma dosyasını][1] açın, örneğin:

vim /etc/mysql/mysql.conf.d/mysqld.cnf

“[client]” bölümünün altına aşağıdaki seçenekleri ekleyin:

ssl-ca = /home/ubuntu/mysqlcerts/ca.pem
ssl-cert = /home/ubuntu/mysqlcerts/client-cert.pem
ssl-key = /home/ubuntu/mysqlcerts/client-key.pem

Şimdi “süpermen”, SSL ile giriş yapmak için yalnızca aşağıdakileri yazmalı:

mysql -h 127.0.0.1 -u superman -p

Başka bir programdan, örneğin Python’dan bağlanmak, genellikle bağlantı işlevine yalnızca ek bir parametre gerektirir. Bir Python örneği:

import MySQLdb
ssl = {'cert': '/home/ubuntu/mysqlcerts/client-cert.pem', 'key': '/home/ubuntu/mysqlcerts/client-key.pem'}
conn = MySQLdb.connect(host='127.0.0.1', user='superman', passwd='imsoawesome', ssl=ssl)

CentOS7 / RHEL7 için Kurulum #

Bu örnekte iki sunucu olduğu varsayılmaktadır:

  1. dbserver (veritabanımızın bulunduğu yer)
  2. başvuru sahibi (uygulamalarımızın bulunduğu yer)

FWIW, her iki sunucu da SELinux uygulamalıdır.

Önce dbserver’a giriş yapın #

Sertifikaları oluşturmak için geçici bir dizin oluşturun.

mkdir /root/certs/mysql/ && cd /root/certs/mysql/

Sunucu sertifikalarını oluşturun

openssl genrsa 2048 > ca-key.pem
openssl req -sha1 -new -x509 -nodes -days 3650 -key ca-key.pem > ca-cert.pem
openssl req -sha1 -newkey rsa:2048 -days 730 -nodes -keyout server-key.pem > server-req.pem
openssl rsa -in server-key.pem -out server-key.pem
openssl x509 -sha1 -req -in server-req.pem -days 730  -CA ca-cert.pem -CAkey ca-key.pem -set_serial 01 > server-cert.pem

Sunucu sertifikalarını /etc/pki/tls/certs/mysql/ dizinine taşıyın

Dizin yolu, CentOS veya RHEL’i varsayar (diğer dağıtımlar için gerektiği gibi ayarlayın):

mkdir /etc/pki/tls/certs/mysql/

Klasör ve dosyalar üzerinde izinleri ayarladığınızdan emin olun. mysql’nin tam sahiplik ve erişime ihtiyacı var.

chown -R mysql:mysql /etc/pki/tls/certs/mysql

Şimdi MySQL/MariaDB’yi yapılandırın

# vi /etc/my.cnf
# i
[mysqld]
bind-address=*
ssl-ca=/etc/pki/tls/certs/ca-cert.pem
ssl-cert=/etc/pki/tls/certs/server-cert.pem
ssl-key=/etc/pki/tls/certs/server-key.pem
# :wq

O zamanlar

systemctl restart mariadb

Uygulama istemcisinden gelen bağlantılara izin vermek için güvenlik duvarınızı açmayı unutmayın (IP 1.2.3.4 kullanarak)

firewall-cmd --zone=drop --permanent --add-rich-rule 'rule family="ipv4" source address="1.2.3.4" service name="mysql" accept'
# I force everything to the drop zone.  Season the above command to taste.

Şimdi firewalld’yi yeniden başlatın

service firewalld restart

Ardından, dbserver’ın mysql sunucusunda oturum açın:

mysql -uroot -p

İstemci için bir kullanıcı oluşturmak için aşağıdakileri yayınlayın. note GRANT deyiminde REQUIRE SSL.

GRANT ALL PRIVILEGES ON *.* TO ‘iamsecure’@’appclient’ IDENTIFIED BY ‘dingdingding’ REQUIRE SSL;
FLUSH PRIVILEGES; 
# quit mysql

İlk adımdan itibaren hala /root/certs/mysql içinde olmalısınız. Değilse, aşağıdaki komutlardan biri için cd’ye geri dönün.

İstemci sertifikalarını oluşturun

openssl req -sha1 -newkey rsa:2048 -days 730 -nodes -keyout client-key.pem > client-req.pem
openssl rsa -in client-key.pem -out client-key.pem
openssl x509 -sha1 -req -in client-req.pem -days 730 -CA ca-cert.pem -CAkey ca-key.pem -set_serial 01 > client-cert.pem

Not: Hem sunucu hem de istemci sertifikaları için aynı ortak adı kullandım. YMMV.

Bu sonraki komut için hala /root/certs/mysql/ olduğunuzdan emin olun

Sunucu ve istemci CA sertifikasını tek bir dosyada birleştirin:

cat server-cert.pem client-cert.pem > ca.pem

İki sertifika gördüğünüzden emin olun:

cat ca.pem

ŞİMDİLİK SUNUCU YANINDAKİ ÇALIŞMALARIN SONU. #

Başka bir terminal açın ve

ssh appclient

Daha önce olduğu gibi, istemci sertifikaları için kalıcı bir yuva oluşturun

mkdir /etc/pki/tls/certs/mysql/

Şimdi, istemci sertifikalarını (dbserver’da oluşturulan) appclient’e yerleştirin. Bunları gözden geçirebilir veya dosyaları tek tek kopyalayıp yapıştırabilirsiniz.

scp dbserver
# copy files from dbserver to appclient
# exit scp

Yine, klasör ve dosyalar üzerinde izinleri ayarladığınızdan emin olun. mysql’nin tam sahiplik ve erişime ihtiyacı var.

chown -R mysql:mysql /etc/pki/tls/certs/mysql

Her biri mysql kullanıcısına ait olan üç dosyanız olmalıdır:

/etc/pki/tls/certs/mysql/ca.pem
/etc/pki/tls/certs/mysql/client-cert.pem
/etc/pki/tls/certs/mysql/client-key.pem

Şimdi “[client]” bölümünde appclient’in MariaDB/MySQL yapılandırmasını düzenleyin.

vi /etc/my.cnf
# i
[client]
ssl-ca=/etc/pki/tls/certs/mysql/ca.pem
ssl-cert=/etc/pki/tls/certs/mysql/client-cert.pem
ssl-key=/etc/pki/tls/certs/mysql/client-key.pem
# :wq

Appclient’in mariadb hizmetini yeniden başlatın:

systemctl restart mariadb

hala burada müşteride #

Bu şunu döndürmelidir: ssl TRUE

mysql --ssl --help

Şimdi, appclient’in mysql örneğine giriş yapın

mysql -uroot -p

Aşağıdaki her iki değişken için de EVET görmeli

show variables LIKE '%ssl';
    have_openssl    YES
    have_ssl              YES

Başlangıçta gördüm

 have_openssl NO

mariadb.log’a hızlı bir bakış şunu ortaya çıkardı:

SSL hatası: Sertifika alınamıyor ‘/etc/pki/tls/certs/mysql/client-cert.pem’

Sorun, kökün client-cert.pem’e ve içerdiği klasöre ait olmasıydı. Çözüm, /etc/pki/tls/certs/mysql/ dosyasının sahipliğini mysql olarak ayarlamaktı.

chown -R mysql:mysql /etc/pki/tls/certs/mysql

Hemen yukarıdaki adımdan gerekirse mariadb’yi yeniden başlatın

ŞİMDİ GÜVENLİ BAĞLANTIYI TEST ETMEYE HAZIRIZ #

Burada hâlâ başvuru sahibiyiz #

Yukarıda oluşturulan hesabı kullanarak dbserver’ın mysql örneğine bağlanmayı deneyin.

mysql -h dbserver -u iamsecure -p
# enter password dingdingding (hopefully you changed that to something else)

Biraz şansla hatasız giriş yapmalısınız.

SSL etkinleştirilmiş olarak bağlı olduğunuzu doğrulamak için MariaDB/MySQL isteminden aşağıdaki komutu verin:

\s

Bu bir ters eğik çizgi, diğer adıyla durum

Bu, aşağıdaki gibi görünmesi gereken bağlantınızın durumunu gösterecektir:

Connection id:        4
Current database:    
Current user:        iamsecure@appclient
SSL:            Cipher in use is DHE-RSA-AES256-GCM-SHA384
Current pager:        stdout
Using outfile:        ''
Using delimiter:    ;
Server:            MariaDB
Server version:        5.X.X-MariaDB MariaDB Server
Protocol version:    10
Connection:        dbserver via TCP/IP
Server characterset:    latin1
Db     characterset:    latin1
Client characterset:    utf8
Conn.  characterset:    utf8
TCP port:        3306
Uptime:            42 min 13 sec

Bağlantı denemenizde izin reddedildi hataları alırsanız, herhangi bir başıboş karakter veya ’ işareti olmadığından emin olmak için yukarıdaki GRANT ifadenizi kontrol edin.

SSL hatalarınız varsa, adımların düzenli olduğundan emin olmak için bu kılavuzu tekrar gözden geçirin.

Bu, RHEL7’de çalıştı ve muhtemelen CentOS7’de de çalışacak. Bu kesin adımların başka bir yerde işe yarayıp yaramayacağı doğrulanamıyor.

Umarım bu, bir başkasını biraz zaman ve ağırlaştırmadan kurtarır.
