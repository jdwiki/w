Sözdizimi #

  • TABLO OLUŞTUR tablo_adı ( sütun_adı1 veri_türü(boyut), sütun_adı2 veri_türü(boyut), column_name3 data_type(boyut), …. ); // Temel tablo oluşturma

  • CREATE TABLE table_name [VAR DEĞİLSE] ( sütun_adı1 veri_türü(boyut), sütun_adı2 veri_türü(boyut), column_name3 data_type(boyut), …. ); // Mevcut tablo oluşturma kontrolü

  • CREATE [GEÇİCİ] TABLO tablo_adı [MEVCUT DEĞİLSE] ( sütun_adı1 veri_türü(boyut), sütun_adı2 veri_türü(boyut), column_name3 data_type(boyut), …. ); // Geçici tablo oluşturma

  • TABLO OLUŞTUR new_tbl [AS] SELECT * FROM orig_tbl; // SELECT’ten tablo oluşturma

“CREATE TABLE” ifadesi bir “ENGINE” belirtimi ile bitmelidir:

CREATE TABLE table_name ( column_definitions ) ENGINE=engine;

Bazı seçenekler şunlardır:

  • InnoDB: (Sürüm 5.5.5’ten beri varsayılandır) Bu, geçiş açısından güvenli (ACID uyumlu) bir motordur. İşlem tamamlama ve geri alma, kilitlenme kurtarma yetenekleri ve satır düzeyinde kilitleme vardır.
  • MyISAM: (Sürüm 5.5.5’ten önceki varsayılan) Bu çok hızlı bir motordur. İşlemleri veya yabancı anahtarları desteklemez, ancak veri ambarı için kullanışlıdır.
  • Memory: Son derece hızlı işlemler için tüm verileri RAM’de saklar, ancak veritabanı yeniden başlatıldığında tablo tarihi kaybolur.

Daha fazla motor seçeneği burada.

Birincil Anahtar ile tablo oluşturma #

CREATE TABLE Person (
    PersonID     INT UNSIGNED NOT NULL,
    LastName     VARCHAR(66) NOT NULL,
    FirstName    VARCHAR(66),
    Address      VARCHAR(255),
    City         VARCHAR(66),
    PRIMARY KEY (PersonID)
);

Bir birincil anahtar, bir tablo satırını benzersiz olarak tanımlayan bir “BOŞ DEĞİL” tek veya çok sütunlu bir tanımlayıcıdır. Bir index oluşturulur ve açıkça ‘NOT NULL’ olarak bildirilmezse, MySQL bunları çok sessiz ve örtük olarak bildirir.

Bir tabloda yalnızca bir ‘PRIMARY KEY’ olabilir ve her tablonun bir tane olması önerilir. InnoDB, yokluğunda otomatik olarak bir tane oluşturacaktır ([MySQL belgelerinde2 görüldüğü gibi), ancak bu daha az arzu edilir.

Genellikle, “vekil anahtar” olarak da bilinen bir “AUTO_INCREMENT” “INT”, ince dizin optimizasyonu ve diğer tablolarla ilişkiler için kullanılır. Bu değer (normalde) yeni bir kayıt eklendiğinde, varsayılan 1 değerinden başlayarak 1 artar.

Ancak, adına rağmen, amacı değerlerin artımlı olduğunu garanti etmek değil, yalnızca sıralı ve benzersiz olduklarını garanti etmek değildir.

Otomatik artışlı bir “INT” değeri, tablodaki tüm satırlar silinirse, tablo TRUNCATE TABLE ifadesi kullanılarak kesilmediği sürece varsayılan başlangıç ​​değerine sıfırlanmaz.

Bir sütunu Birincil Anahtar olarak tanımlama (satır içi tanım) #

Birincil anahtar tek bir sütundan oluşuyorsa, “PRIMARY KEY” yan tümcesi sütun tanımıyla aynı hizada yerleştirilebilir:

CREATE TABLE Person (
    PersonID     INT UNSIGNED NOT NULL PRIMARY KEY,
    LastName     VARCHAR(66) NOT NULL,
    FirstName    VARCHAR(66),
    Address      VARCHAR(255),
    City         VARCHAR(66)
);

Komutun bu formu daha kısadır ve okunması daha kolaydır.

Çok sütunlu Birincil Anahtar Tanımlama #

Birden fazla sütun içeren bir birincil anahtar tanımlamak da mümkündür. Bu, örn. yabancı anahtar ilişkisinin alt tablosunda. Çok sütunlu bir birincil anahtar, katılan sütunların ayrı bir “PRIMARY KEY” yan tümcesinde listelenmesiyle tanımlanır. Burada satır içi sözdizimine izin verilmez, çünkü yalnızca bir sütun satır içi ‘PRIMARY KEY’ olarak bildirilebilir. Örneğin:

CREATE TABLE invoice_line_items (
    LineNum      SMALLINT UNSIGNED NOT NULL,
    InvoiceNum   INT UNSIGNED NOT NULL,
    -- Other columns go here
    PRIMARY KEY (InvoiceNum, LineNum),
    FOREIGN KEY (InvoiceNum) REFERENCES -- references to an attribute of a table
);

Birincil anahtarın * sütunlarının, yukarıdaki örnekte olduğu gibi sütunların tanımlanma sırasından * farklı olabilecek * mantıksal sıralama düzeninde belirtilmesi gerektiğini * unutmayın.

Daha büyük dizinler daha fazla disk alanı, bellek ve G/Ç gerektirir. Bu nedenle tuşlar mümkün olduğu kadar küçük olmalıdır (özellikle bestelenmiş tuşlarla ilgili olarak). InnoDB’de her ‘ikincil dizin’, ‘PRIMARY KEY’ sütunlarının bir kopyasını içerir.

[4]: https://www.wikiod.com/tr/mysql/tablo-olusturma#Yabancı Anahtar ile tablo oluşturma

Temel tablo oluşturma #

“CREATE TABLE” ifadesi, MySQL veritabanında bir tablo oluşturmak için kullanılır.

CREATE TABLE Person (
    `PersonID`      INTEGER NOT NULL PRIMARY KEY,
    `LastName`      VARCHAR(80),
    `FirstName`     VARCHAR(80),
    `Address`       TEXT,
    `City`          VARCHAR(100)
) Engine=InnoDB;

Her alan tanımında şunlar bulunmalıdır:

  1. Alan adı: Geçerli bir alan Adı. Adları `-chars içine aldığınızdan emin olun. Bu, alan adında örneğin boşluk karakterlerini kullanabilmenizi sağlar.
  2. Veri türü [Uzunluk]: Alan ‘CHAR’ veya ‘VARCHAR’ ise, alan uzunluğunun belirtilmesi zorunludur.
  3. Nitelikler “NULL” | ‘NOT NULL’: ‘NOT NULL’ belirtilirse, o alanda herhangi bir NULL değeri depolama girişimi başarısız olur.
  4. Veri türleri ve özellikleri hakkında daha fazla bilgi edinin buradan.

Engine=..., tablonun depolama motorunu belirtmek için kullanılan isteğe bağlı bir parametredir. Herhangi bir depolama motoru belirtilmezse, tablo sunucunun varsayılan tablo depolama motoru (genellikle InnoDB veya MyISAM) kullanılarak oluşturulur.

Varsayılanları ayarlama #

Ek olarak, mantıklı olduğu durumlarda, “VARSAYILAN"ı kullanarak her alan için varsayılan bir değer ayarlayabilirsiniz:

CREATE TABLE Address (
    `AddressID`   INTEGER NOT NULL PRIMARY KEY,
    `Street`      VARCHAR(80),
    `City`        VARCHAR(80),
    `Country`     VARCHAR(80) DEFAULT "United States",
    `Active`      BOOLEAN DEFAULT 1,
) Engine=InnoDB;

Eklemeler sırasında ‘Street’ belirtilmemişse, bu alan alındığında ‘NULL’ olacaktır. Ekleme sırasında “Ülke” belirtilmediğinde, varsayılan olarak “Amerika Birleşik Devletleri” olacaktır.

“BLOB”, “TEXT”, “GEOMETRY” ve “JSON” alanları için hariç tüm sütun türleri için varsayılan değerler ayarlayabilirsiniz.

Yabancı Anahtar ile tablo oluşturma #

CREATE TABLE Account (
    AccountID     INT UNSIGNED NOT NULL,
    AccountNo     INT UNSIGNED NOT NULL,
    PersonID    INT UNSIGNED,
    PRIMARY KEY (AccountID),
    FOREIGN KEY (PersonID) REFERENCES Person (PersonID)
) ENGINE=InnoDB;

Yabancı anahtar: Yabancı Anahtar (‘FK’), bir referans tablosundaki tek bir sütun veya çok sütunlu sütun bileşimidir. Bu “FK"nin başvurulan tabloda var olduğu onaylandı. “FK"yi onaylayan başvurulan tablo anahtarının Birincil Anahtar olması şiddetle önerilir, ancak bu zorunlu değildir. Benzersiz olması gerekmeyen başvurulan dizine hızlı bir arama olarak kullanılır ve aslında orada en soldaki dizin olabilir.

Yabancı anahtar ilişkileri, merkezi veri değerlerini tutan bir üst tabloyu ve aynı değerlere sahip bir alt tabloyu üst öğesine işaret eder. FOREIGN KEY yan tümcesi alt tabloda belirtilmiştir. Üst ve alt tablolar aynı depolama motorunu kullanmalıdır. TEMPORARY tabloları olmamalıdır.

Yabancı anahtardaki karşılık gelen sütunlar ve başvurulan anahtar benzer veri türlerine sahip olmalıdır. Tamsayı türlerinin boyutu ve işareti aynı olmalıdır. Dize türlerinin uzunluklarının aynı olması gerekmez. İkili olmayan (karakter) dize sütunları için karakter kümesi ve harmanlama aynı olmalıdır.

Not: yabancı anahtar kısıtlamaları InnoDB depolama motoru altında desteklenir (MyISAM veya MEMORY değil). Diğer motorları kullanan DB kurulumları bu “CREATE TABLE” ifadesini kabul edecek, ancak yabancı anahtar kısıtlamalarına uymayacaktır. (Daha yeni MySQL sürümleri varsayılan olarak InnoDB olsa da, ancak açık olmak iyi bir uygulamadır.)

Tablo Yapısını Göster #

Tablonuzun şema bilgilerini görmek istiyorsanız aşağıdakilerden birini kullanabilirsiniz:

SHOW CREATE TABLE child; -- Option 1

CREATE TABLE `child` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `fullName` varchar(100) NOT NULL,
  `myParent` int(11) NOT NULL,
  PRIMARY KEY (`id`),
  KEY `mommy_daddy` (`myParent`),
  CONSTRAINT `mommy_daddy` FOREIGN KEY (`myParent`) REFERENCES `parent` (`id`) 
      ON DELETE CASCADE ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

MySQL komut satırı aracından kullanılıyorsa, bu daha az ayrıntılıdır:

SHOW CREATE TABLE child \G

Tablo yapısını göstermenin daha az açıklayıcı bir yolu:

mysql> CREATE TABLE Tab1(id int, name varchar(30));
Query OK, 0 rows affected (0.03 sec)

mysql> DESCRIBE Tab1; -- Option 2 

+-------+-------------+------+-----+---------+-------+ 
| Field | Type        | Null | Key | Default | Extra |  
+-------+-------------+------+-----+---------+-------+ 
| id    | int(11)     | YES  |     | NULL    |       | 
| name  | varchar(30) | YES  |     | NULL    |       |  
+-------+-------------+------+-----+---------+-------+

Hem DESCRIBE hem de DESC aynı sonucu verir.

Bir veritabanındaki tüm tablolarda aynı anda gerçekleştirilen ‘DESCRIBE’ı görmek için bu Örnek’e bakın.

Mevcut bir tabloyu klonlama #

Bir tablo aşağıdaki gibi çoğaltılabilir:

CREATE TABLE ClonedPersons LIKE Persons;

Yeni tablo, dizinler ve sütun nitelikleri dahil olmak üzere orijinal tabloyla tamamen aynı yapıya sahip olacaktır.

Manuel olarak tablo oluşturmanın yanı sıra başka bir tablodan veri seçerek tablo oluşturmak da mümkündür:

CREATE TABLE ClonedPersons SELECT * FROM Persons;

Devam ederken verileri değiştirmek için bir ‘SELECT’ ifadesinin normal özelliklerinden herhangi birini kullanabilirsiniz:

CREATE TABLE ModifiedPersons
SELECT PersonID, FirstName + LastName AS FullName FROM Persons
WHERE LastName IS NOT NULL;

‘SEÇ’ten tablolar oluştururken birincil anahtarlar ve dizinler korunmaz. Bunları yeniden beyan etmelisiniz:

CREATE TABLE ModifiedPersons (PRIMARY KEY (PersonID))
SELECT PersonID, FirstName + LastName AS FullName FROM Persons
WHERE LastName IS NOT NULL;

SEÇİMDEN TABLO OLUŞTUR #

“CREATE TABLE” ifadesinin sonuna bir “SELECT” ifadesi ekleyerek bir tablodan diğerinden bir tablo oluşturabilirsiniz:

CREATE TABLE stack (
    id_user INT,
    username VARCHAR(30),
    password VARCHAR(30)
);

Aynı veritabanında bir tablo oluşturun:

-- create a table from another table in the same database with all attributes
CREATE TABLE stack2 AS SELECT * FROM stack;

-- create a table from another table in the same database with some attributes
CREATE TABLE stack3 AS SELECT username, password FROM stack;

Farklı veritabanlarından tablolar oluşturun:

-- create a table from another table from another database with all attributes
CREATE TABLE stack2 AS SELECT * FROM second_db.stack;

-- create a table from another table from another database with some attributes
CREATE TABLE stack3 AS SELECT username, password FROM second_db.stack;

Not

Başka bir veritabanında bulunan başka bir tablonun aynısını oluşturmak için veritabanının adını şu şekilde belirtmeniz gerekir:

FROM NAME_DATABASE.name_table

Son Güncellemeyi Göstermek İçin Zaman Damgası Sütunuyla Tablo Oluşturma #

TIMESTAMP sütunu, satırın en son ne zaman güncellendiğini gösterecektir.

CREATE TABLE `TestLastUpdate` (
    `ID` INT NULL,
    `Name` VARCHAR(50) NULL,
    `Address` VARCHAR(50) NULL,
    `LastUpdate` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
)
COMMENT='Last Update'
;
