MySQL, çoğu makinede, hesaplamaları için 64-bit IEEE 754 kayan nokta aritmetiği kullanır.

Tamsayı bağlamlarında tamsayı aritmetiğini kullanır.

  • RAND() mükemmel bir rastgele sayı üreteci değildir. Esas olarak hızlı bir şekilde sahte rasgele sayılar oluşturmak için kullanılır.

Aritmetik operatörler #

MySQL aşağıdaki aritmetik operatörleri sağlar

Operatör İsim Örnek
+ Ekleme SELECT 3+5; -> 8
SELECT 3.5+2.5; -> 6.0
SELECT 3.5+2; -> 5.5
- çıkarma SEÇ 3-5; -> -2
* çarpma SEÇ 3 * 5; -> 15
/ Bölüm SELECT 20 / 4; -> 5
SELECT 355 / 113; -> 3.1416
SELECT 10.0 / 0; -> NULL
“BÖL” Tamsayı Bölümü SELECT 5 DIV 2; -> 2
% veya MOD Modül SELECT 7 % 3; -> 1
SELECT 15 MOD 4 -> 3
SELECT 15 MOD -4 -> 3
SELECT -15 MOD 4 -> -3
SELECT -15 MOD -4 -> -3
SELECT 3 MOD 2.5 -> 0,5

BÜYÜK #

Aritmetiğinizdeki sayıların tümü tamsayıysa, MySQL işini yapmak için BÜYÜK (işaretli 64-bit) tamsayı veri türünü kullanır. Örneğin:

seç (1024 * 1024 * 1024 * 1024 *1024 * 1024) + 1 -> 1.152.921.504.606.846.977

ve

seç (1024 * 1024 * 1024 * 1024 *1024 * 1024 * 1024 -> BÜYÜK aralık dışı hatası

ÇİFT #

Aritmetiğinizdeki herhangi bir sayı kesirliyse, MySQL 64-bit IEEE 754 kayan nokta aritmetiği kullanır. Kayan nokta aritmetiği kullanırken dikkatli olmalısınız, çünkü birçok kayan noktalı sayılar, doğası gereği, kesin değerlerden ziyade yaklaşık değerlerdir.

Matematiksel Sabitler #

Pi #

Aşağıdaki, 6 ondalık basamak olarak biçimlendirilmiş ‘PI’ değerini döndürür. Gerçek değer “ÇİFT” kadar iyidir;

SELECT PI();    -> 3.141593

Trigonometri (SIN, COS) #

Açılar Derece değil Radyan cinsindendir. Tüm hesaplamalar IEEE 754 64-bit kayan nokta ile yapılır. Tüm kayan nokta hesaplamaları, [makine ε (epsilon) hataları2, bu nedenle bunları eşitlik için karşılaştırmaya çalışmaktan kaçının. Kayan nokta kullanırken bu hatalardan kaçınmanın bir yolu yoktur; teknolojinin içine inşa edilmişlerdir.

Trigonometrik hesaplamalarda “DECIMAL” değerleri kullanırsanız, bunlar örtük olarak kayan noktaya ve ardından tekrar ondalık sayıya dönüştürülür.

Sinüs #

Radyan cinsinden ifade edilen bir X sayısının sinüsünü döndürür

SELECT SIN(PI()); -> 1.2246063538224e-16

Kosinüs #

X radyan olarak verildiğinde X’in kosinüsünü döndürür

SELECT COS(PI()); -> -1

teğet #

Radyan cinsinden ifade edilen bir X sayısının tanjantını döndürür. Sonucun sıfıra çok yakın olduğuna, ancak tam olarak sıfır olmadığına dikkat edin. Bu bir makine ε örneğidir.

SELECT TAN(PI());   -> -1.2246063538224e-16

Ark Kosinüs (ters kosinüs) #

X “-1 ila 1” aralığındaysa X’in ark kosinüsünü döndürür

SELECT ACOS(1);    -> 0
SELECT ACOS(1.01); -> NULL

Ark Sinüs (ters sinüs) #

X “-1 ila 1” aralığındaysa X’in ark sinüsünü döndürür

SELECT ASIN(0.2); -> 0.20135792079033

Ark Tanjantı (ters teğet) #

“ATAN(x)”, tek bir sayının yay tanjantını döndürür.

SELECT ATAN(2); -> 1.1071487177941

ATAN2(X, Y), X ve Y değişkenlerinin yay tanjantını döndürür. Y / X’in yay tanjantını hesaplamaya benzer. Ancak sayısal olarak daha sağlamdır: X sıfıra yakın olduğunda t düzgün çalışır ve her iki argümanın işaretleri sonucun çeyreğini belirlemek için kullanılır.

En iyi uygulama, formülleri mümkün olan her yerde ‘ATAN()’ yerine ‘ATAN2()’ kullanacak şekilde yazmayı önerir.

 ATAN2(1,1);    -> 0.7853981633974483 (45 degrees)
 ATAN2(1,-1);   -> 2.356194490192345  (135 degrees)
 ATAN2(0, -1);  -> PI  (180 degrees)  don't try ATAN(-1 / 0)... it won't work

Kotanjant #

X’in kotanjantını döndürür

SELECT COT(12); -> -1.5726734063977

Dönüştürmek #

SELECT RADIANS(90) -> 1.5707963267948966
SELECT SIN(RADIANS(90)) -> 1
SELECT DEGREES(1), DEGREES(PI()) -> 57.29577951308232, 180

Yuvarlama (YUVARLAK, ZEMİN, TAVAN) #

Bir ondalık sayıyı bir tamsayı değerine yuvarlama #

Kesin sayısal değerler için (ör. “DECIMAL”): Bir sayının ilk ondalık basamağı 5 veya daha büyükse, bu işlev bir sayıyı sıfırdan uzaktaki bir sonraki tam sayıya yuvarlar. Bu ondalık basamak 4 veya daha düşükse, bu işlev bir sonraki sıfıra en yakın tamsayı değerine yuvarlar.

SELECT ROUND(4.51) -> 5
SELECT ROUND(4.49) -> 4
SELECT ROUND(-4.51) -> -5

Yaklaşık sayısal değerler için (ör. “DOUBLE”): “ROUND()” işlevinin sonucu C kitaplığına bağlıdır; birçok sistemde bu, “ROUND()“un en yakın çifte yuvarlama kuralını kullandığı anlamına gelir:

SELECT ROUND(45e-1) -> 4  -- The nearest even value is 4
SELECT ROUND(55e-1) -> 6  -- The nearest even value is 6

Bir sayıyı yuvarla #

Bir sayıyı yuvarlamak için ‘CEIL()’ veya ‘CEILING()’ işlevini kullanın

SELECT CEIL(1.23)    -> 2
SELECT CEILING(4.83) -> 5

Bir sayıyı aşağı yuvarla #

Bir sayıyı aşağı yuvarlamak için ‘FLOOR()’ işlevini kullanın

SELECT FLOOR(1.99) -> 1

FLOOR ve CEIL -infinity’e doğru / sonsuzdan uzağa gider:

SELECT FLOOR(-1.01), CEIL(-1.01) -> -2 and -1
SELECT FLOOR(-1.99), CEIL(-1.99) -> -2 and -1

Bir ondalık sayıyı belirtilen sayıda ondalık basamağa yuvarlayın. #

SELECT ROUND(1234.987, 2) -> 1234.99
SELECT ROUND(1234.987, -2) -> 1200

Yukarıya karşı aşağı ve “5” tartışması da geçerlidir.

Bir sayıyı bir güce yükseltin (POW) #

‘x’ sayısını bir ‘y’ kuvvetine yükseltmek için, ‘POW()’ veya ‘POWER()’ işlevlerini kullanın

SELECT POW(2,2); => 4
SELECT POW(4,2); => 16

Kare Kök (SQRT) #

SQRT() işlevini kullanın. Sayı negatifse, ‘NULL’ döndürülür

SELECT SQRT(16); -> 4
SELECT SQRT(-3); -> NULL

Rastgele Sayılar (RAND) #

Rastgele bir sayı üret #

‘0’ ve ‘1’ arasında sahte rasgele kayan nokta sayısı oluşturmak için ‘RAND()’ işlevini kullanın

Diyelim ki aşağıdaki sorgunuz var

SELECT i, RAND() FROM t;

Bu böyle bir şey döndürür

ben RAND()
1 0.6191438870682
2 0.93845168309142
3 0,83482678498591

Bir aralıktaki Rastgele Sayı #

a <= n <= b aralığında rastgele bir sayı oluşturmak için aşağıdaki formülü kullanabilirsiniz.

FLOOR(a + RAND() * (b - a + 1))

Örneğin, bu 7 ile 12 arasında rastgele bir sayı üretecektir.

SELECT FLOOR(7 + (RAND() * 6));

Bir tablodaki satırları rastgele döndürmenin basit bir yolu:

SELECT * FROM tbl ORDER BY RAND();

Bunlar sözde sayılardır.

MySQL’deki sözde rasgele sayı üreteci kriptografik olarak güvenli değildir. Yani, sır olarak kullanılacak rastgele sayılar üretmek için MySQL kullanırsanız, MySQL kullandığınızı bilen kararlı bir düşman, sırlarınızı inandığınızdan daha kolay tahmin edebilecektir.

Mutlak Değer ve İşaret (ABS, İŞARET) #

Bir sayının mutlak değerini döndür

SELECT ABS(2);   -> 2
SELECT ABS(-46); -> 46

Bir sayının ‘işareti’ onu 0 ile karşılaştırır.

İşaret Sonuç Örnek
-1 n < 0 İŞARETİ SEÇ(42); -> 1
0 n = 0 SEÇ İŞARETİ (0); -> 0
1 n > 0 İŞARETİ SEÇ(-3); -> -1 
SELECT SIGN(-423421); -> -1
