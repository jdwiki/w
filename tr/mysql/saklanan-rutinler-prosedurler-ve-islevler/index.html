Parametreler #

parametre Ayrıntılar
İADE Bir işlevden döndürülebilecek veri türünü belirtir.
DÖNÜŞ ‘RETURN’ sözdizimini izleyen gerçek değişken veya değer, işlevin çağrıldığı yere döndürülen değerdir.

Depolanmış bir rutin ya bir prosedür ya da bir fonksiyondur.

Bir CALL deyimi kullanılarak bir prosedür çağrılır ve yalnızca geri dönebilir çıkış değişkenlerini kullanan değerler.

Bir fonksiyon, tıpkı diğerleri gibi, bir deyimin içinden çağrılabilir. işlev ve skaler bir değer döndürebilir.

IN, OUT, INOUT parametreleriyle saklı prosedür #

DELIMITER $$
 
DROP PROCEDURE IF EXISTS sp_nested_loop$$
CREATE PROCEDURE sp_nested_loop(IN i INT, IN j INT, OUT x INT, OUT y INT, INOUT z INT)
BEGIN
    DECLARE a INTEGER DEFAULT 0;
    DECLARE b INTEGER DEFAULT 0;
    DECLARE c INTEGER DEFAULT 0;
    WHILE a < i DO
        WHILE b < j DO
            SET c = c + 1;
            SET b = b + 1;
        END WHILE;    
        SET a = a + 1;
        SET b = 0;        
    END WHILE;
    SET x = a, y = c;
    SET z = x + y + z;
END $$
DELIMITER ;

Saklı yordamı çağırır (CALL):

SET @z = 30;
call sp_nested_loop(10, 20, @x, @y, @z);
SELECT @x, @y, @z;

Sonuç:

+------+------+------+
|  @x  |  @y  |  @z  |
+------+------+------+
|  10  |  200 |  240 |
+------+------+------+

Bir “IN” parametresi, bir değeri bir prosedüre iletir. Yordam değeri değiştirebilir, ancak yordam döndüğünde değişiklik arayan tarafından görülmez.

Bir “OUT” parametresi, prosedürden arayana bir değeri geri iletir. İlk değeri yordam içinde NULL’dur ve değeri yordam döndüğünde arayan tarafından görülebilir.

Bir “INOUT” parametresi arayan tarafından başlatılır, prosedür tarafından değiştirilebilir ve prosedür tarafından yapılan herhangi bir değişiklik, prosedür geri döndüğünde arayan tarafından görülebilir.

Referans: http://dev.mysql.com/doc/refman/5.7/en/create-procedure.html

Bir İşlev Oluşturun #

Aşağıdaki (önemsiz) örnek işlev, yalnızca “INT” sabit değerini “12” döndürür.

DELIMITER ||
CREATE FUNCTION functionname()
RETURNS INT
BEGIN
    RETURN 12;
END;
||
DELIMITER ;

İlk satır, sınırlayıcı karakterin (DELIMITER ||) ne olarak değiştirileceğini tanımlar, bunun bir işlev oluşturulmadan önce ayarlanması gerekir, aksi takdirde varsayılan ; ve ardından ilk ; yani ilk ; işlev gövdesinde bulunan, genellikle istenen şey olmayan ‘CREATE’ ifadesinin sonu olarak alınacaktır.

CREATE FUNCTION çalıştırıldıktan sonra, yukarıdaki örnekte (DELIMITER ;) fonksiyon kodundan sonra görüldüğü gibi, sınırlayıcıyı varsayılan ; değerine ayarlamalısınız.

Yürütme bu işlevi aşağıdaki gibidir:

SELECT functionname();
+----------------+
| functionname() |
+----------------+
|             12 |
+----------------+

Biraz daha karmaşık (ama yine de önemsiz) bir örnek, bir parametre alır ve ona bir sabit ekler:

DELIMITER $$
CREATE FUNCTION add_2 ( my_arg INT )
  RETURNS INT
BEGIN
  RETURN (my_arg + 2);
END;
$$
DELIMITER ;

SELECT add_2(12);
+-----------+
| add_2(12) |
+-----------+
|        14 |
+-----------+

“DELIMITER” yönergesinde farklı bir bağımsız değişkenin kullanıldığına dikkat edin. Aslında ‘CREATE’ ifade gövdesinde görünmeyen herhangi bir karakter dizisini kullanabilirsiniz, ancak genel uygulama, ‘\’, ‘||’ veya ‘$$’ gibi iki kat alfasayısal olmayan karakter kullanmaktır.

Bazı GUI’ler sınırlayıcının değişmesini gerektirmediğinden, komut satırı aracılığıyla sorguları çalıştırmak her zaman sınırlayıcının ayarlanmasını gerektirdiğinden, parametreyi bir işlev, prosedür veya tetik oluşturma veya güncellemeden önce ve sonra her zaman değiştirmek iyi bir uygulamadır.

Yapılandırılmış Hazırlama ile Prosedür Oluşturma #

DROP PROCEDURE if exists displayNext100WithName;
DELIMITER $$
CREATE PROCEDURE displayNext100WithName
(    nStart int,
    tblName varchar(100)
)
BEGIN
    DECLARE thesql varchar(500); -- holds the constructed sql string to execute

    -- expands the sizing of the output buffer to accomodate the output (Max value is at least 4GB)
    SET session group_concat_max_len = 4096; -- prevents group_concat from barfing with error 1160 or whatever it is

    SET @thesql=CONCAT("select group_concat(qid order by qid SEPARATOR '%3B') as nums ","from (    select qid from ");
    SET @thesql=CONCAT(@thesql,tblName," where qid>? order by qid limit 100 )xDerived");
    PREPARE stmt1 FROM @thesql; -- create a statement object from the construct sql string to execute
    SET @p1 = nStart; -- transfers parameter passed into a User Variable compatible with the below EXECUTE
    EXECUTE stmt1 USING @p1;
     
    DEALLOCATE PREPARE stmt1; -- deallocate the statement object when finished
END$$
DELIMITER ;

Saklı yordamın oluşturulması, birçok istemci aracında gerekli olan bir DELIMITER ile sarmayı gösterir.

Çağrı örneği:

call displayNext100WithName(1,"questions_mysql");

İmleçler #

İmleçler, sorgu sonuçlarını satır satır yinelemenizi sağlar. ‘BAYRAM’ komutu, imleci başlatmak ve belirli bir SQL sorgusu ile ilişkilendirmek için kullanılır:

DECLARE student CURSOR FOR SELECT name FROM studend;

Diyelim ki bazı türlerde ürünler satıyoruz. Her türden kaç tane ürün olduğunu saymak istiyoruz.

Verilerimiz:

CREATE TABLE product
(
  id   INT(10) UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY,
  type VARCHAR(50)      NOT NULL,
  name VARCHAR(255)     NOT NULL

);
CREATE TABLE product_type
(
  name VARCHAR(50) NOT NULL PRIMARY KEY
);
CREATE TABLE product_type_count
(
  type  VARCHAR(50)      NOT NULL PRIMARY KEY,
  count INT(10) UNSIGNED NOT NULL DEFAULT 0
);

INSERT INTO product_type (name) VALUES
  ('dress'),
  ('food');

INSERT INTO product (type, name) VALUES
  ('dress', 'T-shirt'),
  ('dress', 'Trousers'),
  ('food', 'Apple'),
  ('food', 'Tomatoes'),
  ('food', 'Meat');

İmleç kullanarak saklı yordamı kullanarak hedefe ulaşabiliriz:

DELIMITER //
DROP PROCEDURE IF EXISTS product_count;
CREATE PROCEDURE product_count()
  BEGIN
    DECLARE p_type VARCHAR(255);
    DECLARE p_count INT(10) UNSIGNED;
    DECLARE done INT DEFAULT 0;
    DECLARE product CURSOR FOR
      SELECT
        type,
        COUNT(*)
      FROM product
      GROUP BY type;
    DECLARE CONTINUE HANDLER FOR SQLSTATE '02000' SET done = 1;

    TRUNCATE product_type;

    OPEN product;
    
    REPEAT
      FETCH product
      INTO p_type, p_count;
      IF NOT done
      THEN
        INSERT INTO product_type_count
        SET
          type  = p_type,
          count = p_count;
      END IF;
    UNTIL done
    END REPEAT;
    
    CLOSE product;
  END //
DELIMITER ;

Ne zaman prosedürü arayabilirsiniz:

CALL product_count();

Sonuç “product_type_count” tablosunda olacaktır:

type   | count
----------------
dress  |   2
food   |   3

Bu, bir ‘CURSOR’ için iyi bir örnek olsa da, prosedürün tüm gövdesinin nasıl değiştirilebileceğine dikkat edin.

INSERT INTO product_type_count
        (type, count)
    SELECT type, COUNT(*)
        FROM product
        GROUP BY type;

Bu çok daha hızlı çalışacaktır.

Birden Çok Sonuç Kümesi #

Bir “SEÇ” ifadesinin aksine, bir “Saklı Yordam”, birden çok sonuç kümesi döndürür. Perl, PHP, vb.‘de bir “ÇAĞRI"nın sonuçlarını toplamak için kullanılacak farklı kod gerektirir.

(Burada veya başka bir yerde belirli bir koda ihtiyacınız var!)

Bir işlev oluşturun #

DELIMITER $$
CREATE
    DEFINER=`db_username`@`hostname_or_IP`
    FUNCTION `function_name`(optional_param data_type(length_if_applicable))
    RETURNS data_type
BEGIN
    /*
    SQL Statements goes here
    */
END$$
DELIMITER ;

RETURNS veri türü, herhangi bir MySQL veri türüdür.
