Depolama motorunu değiştirme; tabloyu yeniden oluştur; file_per_table’ı değiştir #

Örneğin, “t1” şu anda bir InnoDB tablosu değilse, bu ifade depolama motorunu InnoDB olarak değiştirir:

ALTER TABLE t1 ENGINE = InnoDB;

Tablo zaten InnoDB ise, bu, tabloyu ve dizinlerini yeniden oluşturacak ve “TABLOYU OPTİMİZE ET"e benzer bir etkiye sahip olacaktır. Bir miktar disk alanı iyileştirmesi elde edebilirsiniz.

“innodb_file_per_table” değeri, “t1” oluşturulduğunda geçerli olan değerden şu anda farklıysa, bu, file_per_table’a (veya ondan) dönüştürülür.

TABLO SÜTUNU DEĞİŞTİR #

CREATE DATABASE stackoverflow;

USE stackoverflow;

Create table stack(
    id_user int NOT NULL,
    username varchar(30) NOT NULL,
    password varchar(30) NOT NULL
);

ALTER TABLE stack ADD COLUMN submit date NOT NULL; -- add new column
ALTER TABLE stack DROP COLUMN submit; -- drop column
ALTER TABLE stack MODIFY submit DATETIME NOT NULL; -- modify type column
ALTER TABLE stack CHANGE submit submit_date DATETIME NOT NULL; -- change type and name of column
ALTER TABLE stack ADD COLUMN mod_id INT NOT NULL AFTER id_user; -- add new column after existing column

Otomatik artış değerini değiştir #

Büyük bir silme işleminden sonra AUTO_INCREMENT sütununda boşluk istemiyorsanız, otomatik artış değerini değiştirmek yararlıdır.

Örneğin, tablonuzda birçok istenmeyen (reklam) satırınız var, onları sildiniz ve otomatik artış değerlerindeki boşluğu düzeltmek istiyorsunuz. AUTO_INCREMENT sütununun MAX değerinin şimdi 100 olduğunu varsayın. Otomatik artış değerini düzeltmek için aşağıdakileri kullanabilirsiniz.

ALTER TABLE your_table_name AUTO_INCREMENT = 101;

MySQL tablosunu yeniden adlandırma #

Bir tabloyu yeniden adlandırmak tek bir komutla yapılabilir:

RENAME TABLE `<old name>` TO `<new name>`;

Aşağıdaki sözdizimi tam olarak aynı şeyi yapar:

ALTER TABLE `<old name>` RENAME TO `<new name>`;

Geçici bir tablo yeniden adlandırılıyorsa, sözdiziminin ‘ALTER TABLE’ sürümü kullanılmalıdır.

Adımlar:

  1. Yukarıdaki satırda <eski isim> ve <yeni isim>i ilgili değerlerle değiştirin. Not: Tablo farklı bir veritabanına taşınıyorsa, ‘’ ve/veya ‘’ için ‘dbname’.’tabloadı’ sözdizimi kullanılabilir.
  2. MySQL komut satırındaki ilgili veritabanında veya MySQL Workbench gibi bir istemcide çalıştırın. Not: Kullanıcının eski tabloda ALTER ve DROP, yenisinde ise CREATE ve INSERT yetkilerine sahip olması gerekir.

ALTER tablosu INDEX ekle #

Performansı artırmak için sütunlara dizin eklemek isteyebilirsiniz.

ALTER TABLE TABLE_NAME ADD INDEX `index_name` (`column_name`)

bileşik (birden çok sütun) dizinleri eklemek için değiştirme

ALTER TABLE TABLE_NAME ADD INDEX `index_name` (`col1`,`col2`)

Birincil anahtar sütununun türünü değiştirme #

ALTER TABLE fish_data.fish DROP PRIMARY KEY;
ALTER TABLE fish_data.fish MODIFY COLUMN fish_id DECIMAL(20,0) NOT NULL PRIMARY KEY;

Önce birincil anahtarı bırakmadan bu sütunun türünü değiştirme girişimi bir hataya neden olur.

Sütun tanımını değiştir #

Bir db sütununun tanımını değiştirin, örneğin bu db şemasına sahipsek aşağıdaki sorgu kullanılabilir.

users (
    firstname varchar(20),
    lastname varchar(20),
    age char(2)
)

“yaş” sütununun türünü “char"dan “int"e değiştirmek için aşağıdaki sorguyu kullanırız:

ALTER TABLE users CHANGE age age tinyint UNSIGNED NOT NULL;

Genel biçimi:

ALTER TABLE table_name CHANGE column_name new_column_definition

MySQL veritabanını yeniden adlandırma #

MySQL veritabanını yeniden adlandırmak için tek bir komut yoktur, ancak yedekleme ve geri yükleme yoluyla bunu başarmak için basit bir geçici çözüm kullanılabilir:

mysqladmin -uroot -p<password> create <new name>
mysqldump -uroot -p<password> --routines <old name> | mysql -uroot -pmypassword <new name>
mysqladmin -uroot -p<password> drop <old name>

Adımlar:

  1. Yukarıdaki satırları bir metin düzenleyiciye kopyalayın.
  2. ‘’, ‘’ ve ‘’ (+ isteğe bağlı olarak farklı bir kullanıcı kullanmak için ‘kök’) ile ilgili tüm referansları ilgili değerlerle değiştirin.
  3. Komut satırında birer birer yürütün (MySQL “bin” klasörünün yolda olduğunu ve istendiğinde “y” girdiğini varsayarak).

Alternatif Adımlar:

Her tabloyu bir db’den diğerine yeniden adlandırın (taşıyın). Bunu her tablo için yapın:

RENAME TABLE `<old db>`.`<name>` TO  `<new db>`.`<name>`;

Gibi bir şey yaparak bu ifadeleri oluşturabilirsiniz.

SELECT CONCAT('RENAME TABLE old_db.', table_name, ' TO ',
                           'new_db.', table_name)
    FROM information_schema.TABLES
    WHERE table_schema = 'old_db';

Uyarı. Dosyaları dosya sistemi üzerinde hareket ettirerek herhangi bir tür tablo veya veritabanı yapmaya çalışmayın. Bu, sadece MyISAM’ın eski günlerinde iyi çalıştı, ancak InnoDB ve tablo alanlarının yeni günlerinde çalışmayacak. Özellikle “Veri Sözlüğü” dosya sisteminden sistem InnoDB tablolarına taşındığında, muhtemelen bir sonraki ana sürümde. Bir InnoDB tablosunun bir “PARTITION"ını taşımak (sadece “DROPping” yerine) “taşınabilir tablo alanları” kullanılmasını gerektirir. Yakın gelecekte, ulaşılacak bir dosya bile kalmayacak.
