Kurulum veya Kurulum #

WebGL bir tarayıcı teknolojisidir, bu nedenle bir tarayıcıya sahip olmaktan başka kurulacak pek bir şey yoktur. WebGL’yi JSFiddle veya Codepen veya JSBIn veya herhangi bir sayıda başka bir yerde kullanmaya başlayabilirsiniz. Birkaç sınırlama olmasına rağmen HTML, CSS ve JavaScript’i çevrimiçi düzenlemenize izin veren siteler (aşağıya bakın). Açık kaynak dosyalarını github sayfalarında veya benzer hizmetlerde de barındırabilirsiniz.

Öte yandan, bir noktada muhtemelen yerel olarak çalışacaksınız. Bunu yapmak için basit bir web sunucusu çalıştırmanız önerilir. Aralarından seçim yapabileceğiniz, kullanımı basit ve çok az kurulum gerektiren çok şey var.

node.js’yi sunucu olarak kullanma

  1. [node.js]‘yi kurun(http://nodejs.org)
  2. Bir terminal veya düğüm komut istemi açın ve ’npm install -g http-server’ yazın (OSX’te önüne ‘sudo’ yazın.
  3. Geçerli klasördeki dosyaları sunmaya başlamak için ‘http-server’ yazın VEYA farklı bir klasöre sunucuya ‘http-server path-to-folder’
  4. WebGL web sayfanızı görüntülemek için tarayıcınızı http://localhost:8080/name-of-file konumuna getirin.

devd’yi sunucu olarak kullanma

  1. [devd]‘yi indirin (https://github.com/cortesi/devd)
  2. Bir terminal açın ve devd’yi geçerli klasördeki sunucu dosyalarına ‘devd.’ veya farklı bir klasör sunmak için ‘devd path-to-folder’ ile çalıştırın.
  3. WebGL web sayfanızı görüntülemek için tarayıcınızı http://localhost:8000/name-of-file konumuna getirin.

Servez’i sunucu olarak kullanma

  1. Serve dosyasını indirin
  2. Kurun, Çalıştırın
  3. Sunulacak klasörü seçin
  4. “Başlat"ı seçin
  5. http://localhost:8080 adresine gidin veya “Tarayıcıyı Başlat"ı seçin

“Chrome için Web Sunucusu” Chrome Uzantısını Kullanma

  1. Chrome’dan Web Sunucusu yükleyin

  2. Yeni bir sekme sayfasındaki Uygulamalar simgesinden başlatın.

  1. Dosyalarınızın bulunduğu klasörü ayarlayın ve ardından http://127.0.0.1:8787 bağlantısını tıklayın.

Çevrimiçi Hizmetlerde WebGL’nin Sınırlandırılması #

WebGL’de görüntüleri yüklemek çok yaygındır. WebGL’de görüntülerin nasıl kullanılabileceği konusunda kısıtlamalar vardır. Özellikle WebGL, görüntüleri barındıran sunucunun izni olmadan diğer etki alanlarından görüntüleri kullanamaz. Şu anda görüntüleri kullanma izni veren hizmetler arasında imgur ve flickr bulunmaktadır. Bkz. Etki Alanları Arası Görüntüleri Yükleme. Aksi takdirde, resimleri webgl sayfanızla aynı sunucuda bulundurmanız veya tuval etiketiyle resim oluşturma gibi diğer yaratıcı çözümleri kullanmanız gerekir.

Selam Dünya #

Açıklamalar bölümünde bahsettiği gibi iki fonksiyon sağlamamız gerekiyor. Bir köşe gölgelendiricisi ve bir parça gölgelendiricisi

Bir köşe gölgelendiricisi ile başlayalım

  gl_Position = position;

Sonra bir parça gölgelendiriciye ihtiyacımız var

// fragment shaders don't have a default precision so we need
// to pick one. mediump, short for medium precision, is a good default.
precision mediump float;

void main() {
  // gl_FragColor is a special variable a fragment shader
  // is responsible for setting
  gl_FragColor = vec4(1, 0, 0.5, 1); // return redish-purple 
}

Yukarıda, gl_FragColorı 1, 0, 0,5, 1 olarak ayarlıyoruz; bu, kırmızı için 1, yeşil için 0, mavi için 0,5, alfa için 1’dir. WebGL’deki renkler 0’dan 1’e gider.

Şimdi 2 işlevi yazdığımıza göre WebGL ile başlayalım

İlk önce bir HTML tuval öğesine ihtiyacımız var

Sonra JavaScript’te buna bakabiliriz

 var canvas = document.getElementById("c");

Artık bir WebGLRenderingContext oluşturabiliriz.

 var gl = canvas.getContext("webgl");
 if (!gl) {

Şimdi bu gölgelendiricileri GPU’ya yerleştirmek için derlememiz gerekiyor, bu yüzden önce onları dizelere almamız gerekiyor. Dizelerinizi, normalde dizeleri aldığınız herhangi bir şekilde alabilirsiniz. Birleştirerek, AJAX kullanarak, bunları JavaScript olmayan komut dosyası etiketlerine koyarak veya bu durumda çok satırlı şablon değişmezlerini kullanarak

var vertexShaderSource = `
// an attribute will receive data from a buffer
attribute vec4 position;

// all shaders have a main function
void main() {

  // gl_Position is a special variable a vertex shader 
  // is responsible for setting
  gl_Position = position;
}
`;

var fragmentShaderSource = `
// fragment shaders don't have a default precision so we need
// to pick one. mediump is a good default
precision mediump float;

void main() {
  // gl_FragColor is a special variable a fragment shader
  // is responsible for setting
  gl_FragColor = vec4(1, 0, 0.5, 1); // return redish-purple 
}
`;

Sonra bir shader oluşturacak, kaynağı yükleyecek ve shader’ı derleyecek bir fonksiyona ihtiyacımız var.

function createShader(gl, type, source) {
  var shader = gl.createShader(type);
  gl.shaderSource(shader, source);  
  gl.compileShader(shader);
  var success = gl.getShaderParameter(shader, gl.COMPILE_STATUS);
  if (success) {
    return shader;
  }

  console.log(gl.getShaderInfoLog(shader));
  gl.deleteShader(shader);
}

Şimdi 2 gölgelendirici oluşturmak için bu işlevi çağırabiliriz.

var vertexShader = createShader(gl, gl.VERTEX_SHADER, vertexShaderSource);
var fragmentShader = createShader(gl, gl.FRAGMENT_SHADER, fragmentShaderSource);

Daha sonra bu 2 gölgelendiriciyi bir programa bağlamamız gerekir*

function createProgram(gl, vertexShader, fragmentShader) {
  var program = gl.createProgram();
  gl.attachShader(program, vertexShader);
  gl.attachShader(program, fragmentShader);
  gl.linkProgram(program);
  var sucesss = gl.getProgramParameter(program, gl.LINK_STATUS);
  if (success) {
    return program;
  }

  console.log(gl.getProgramInfoLog(program));
  gl.deleteProgram(program);
}

Ve ara

var program = createProgram(gl, vertexShader, fragmentShader);

Artık GPU üzerinde bir GLSL programı oluşturduğumuza göre, ona veri sağlamamız gerekiyor. WebGL API’sinin çoğu, GLSL programlarımıza veri sağlamak için durumu ayarlamakla ilgilidir. Bu durumda GLSL programımıza tek girdimiz bir öznitelik olan ‘pozisyon’dur. Yapmamız gereken ilk şey, az önce oluşturduğumuz programın özniteliğinin konumuna bakmaktır.

var positionAttributeLocation = gl.getAttribLocation(program, "position");

Nitelikler verilerini arabelleklerden alır, bu nedenle bir arabellek oluşturmamız gerekir.

var positionBuffer = gl.createBuffer();

WebGL, küresel bağlantı noktalarında birçok WebGL kaynağını değiştirmemize izin verir. Bağlama noktalarını WebGL içindeki dahili global değişkenler olarak düşünebilirsiniz. İlk önce kaynağınıza bağlama noktasını ayarlarsınız. Ardından, diğer tüm işlevler, bağlama noktası aracılığıyla kaynağa başvurur. Öyleyse, konum arabelleğini bağlayalım.

gl.bindBuffer(gl.ARRAY_BUFFER, positionBuffer);

Şimdi, verileri bu arabelleğe bağlama noktasından referans alarak koyabiliriz.

// three 2d points
var positions = [
  0, 0, 
  0, 0.5,
  0.7, 0,
];
gl.bufferData(gl.ARRAY_BUFFER, new Float32Array(positions), gl.STATIC_DRAW);

Burada çok şey oluyor. İlk olarak, bir JavaScript dizisi olan “pozisyonlar"a sahibiz. Öte yandan WebGL, kesin olarak yazılan verilere ihtiyaç duyar, bu nedenle ’new Float32Array(positions)’ satırı yeni bir 32 bit kayan nokta sayısı dizisi oluşturur ve değerleri ‘positions’dan kopyalar. “gl.bufferData” daha sonra bu verileri GPU’daki “positionBuffer"a kopyalar. Konum arabelleğini kullanıyor çünkü onu yukarıdaki “ARRAY_BUFFER” bağlama noktasına bağladık.

Son argüman olan gl.STATIC_DRAW, verileri nasıl kullanacağımız konusunda WebGL’ye bir ipucudur. Belirli şeyleri optimize etmek için bu bilgiyi kullanmayı deneyebilir. “gl.STATIC_DRAW”, WebGL’ye bu verileri pek değiştirmeyeceğimizi söyler.

Artık verileri bir arabelleğe koyduğumuza göre, özniteliğe ondan nasıl veri çıkarılacağını söylememiz gerekiyor. İlk önce özelliği açmamız gerekiyor

gl.enableVertexAttribArray(positionAttributeLocation);

Daha sonra verilerin nasıl çekileceğini belirtmemiz gerekiyor.

var size = 2;          // 2 components per iteration
var type = gl.FLOAT;   // the data is 32bit floats
var normalize = false; // use the data as is
var stride = 0;        // 0 = move size * sizeof(type) each iteration
var offset = 0;        // start at the beginning of the buffer
gl.vertexAttribPointer(
   positionAttributeLocation, size, type, normalize, stride, offset)

“gl.vertexAttribPointer” öğesinin gizli bir parçası, geçerli “ARRAY_BUFFER” öğesini özniteliğe bağlamasıdır. Başka bir deyişle, artık bu öznitelik “positionBuffer"a bağlı olduğundan, “ARRAY_BUFFER” bağlama noktasına başka bir şey bağlamakta özgürüz.

GLSL köşe gölgelendiricimizin bakış açısından konum özniteliğinin bir “vec4” olduğuna dikkat edin.

attribute vec4 position;

‘vec4’ 4 kayan noktalı bir değerdir. JavaScript’te bunu position = {x: 0, y: 0, z: 0, w: 0} gibi bir şey düşünebilirsiniz. Yukarıda size = 2 olarak ayarladık. Öznitelikler varsayılan olarak “0, 0, 0, 1” şeklindedir, bu nedenle bu öznitelik ilk 2 değerini (x ve y) arabelleğimizden alacaktır. z ve w sırasıyla varsayılan 0 ve 1 olacaktır.

Tüm bunlardan sonra nihayet WebGL’den yürütmesini isteyebileceğimiz GLSL programıdır.

var primitiveType = gl.TRIANGLES;
var offset = 0;
var count = 3;
gl.drawArrays(primitiveType, offset, count);

Bu, köşe gölgelendiricimizi 3 kez çalıştıracaktır. Köşe gölgelendiricimizde ilk kez ‘position.x’ ve ‘position.y’, positionBuffer’dan ilk 2 değere ayarlanacaktır. 2. kez position.xy 2. 2 değerlerine ayarlanacaktır. En son son 2 değere ayarlanacaktır.

primitiveTypeı gl.TRIANGLES olarak ayarladığımız için, köşe gölgelendiricimiz her 3 kez çalıştırıldığında WebGL, gl_Position olarak ayarladığımız 3 değere dayalı olarak bir üçgen çizecektir. Tuvalimizin boyutu ne olursa olsun, bu değerler her yönde -1’den 1’e giden klip uzayı koordinatlarındadır.

Köşe gölgelendiricimiz, positionBuffer değerlerimizi gl_Positiona kopyaladığı için üçgen, klip alanı koordinatlarında çizilecektir.

  0, 0, 
  0, 0.5,
  0.7, 0,

Bu değerlerin piksellere nasıl çevrileceği “gl.viewport” ayarına bağlıdır. gl.viewport varsayılan olarak tuvalin başlangıç ​​boyutudur. Tuvalimiz için bir boyut belirlemediğimiz için varsayılan boyut 300x150’dir. Klip alanından piksellere dönüştürme (genellikle WebGL ve OpenGL literatüründe ekran alanı olarak adlandırılır) WebGL, bir üçgen çizecek

 clip space      screen space
   0, 0       ->   150, 75
   0, 0.5     ->   150, 112.5
 0.7, 0       ->   255, 75

WebGL şimdi bu üçgeni oluşturacaktır. Çizmek üzere olduğu her piksel için bizim parça gölgelendiricimizi arayacaktır. Parça gölgelendiricimiz gl_FragColor değerini 1, 0, 0.5, 1 olarak ayarlar. Kanvas, kanal başına 8 bitlik bir kanvas olduğundan, bu, WebGL’nin “[255, 0, 127, 255]” değerlerini kanvas içine yazacağı anlamına gelir.
