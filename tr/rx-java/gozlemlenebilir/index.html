Bir Gözlenebilir Oluştur #

RxJava’da bir Gözlenebilir oluşturmanın birkaç yolu vardır. En güçlü yol, ‘Observable.create’ yöntemini kullanmaktır. Ama aynı zamanda en karmaşık yoldur. Bu nedenle, mümkün olduğu kadar kullanmaktan kaçınmalısınız.

Çıkan bir değer yayan #

Zaten bir değeriniz varsa, değerinizi yaymak için ‘Observable.just’ kullanabilirsiniz.

 Observable.just("Hello World").subscribe(System.out::println);

Hesaplanması gereken bir değer yayan #

Henüz hesaplanmamış veya hesaplanması uzun sürebilecek bir değer yaymak istiyorsanız, bir sonraki değerinizi göndermek için ‘Observable.fromCallable’ı kullanabilirsiniz.

Observable.fromCallable(() -> longComputation()).subscribe(System.out::println);

“longComputation()” yalnızca “Gözlenebilir” öğenize abone olduğunuzda çağrılır. Bu şekilde, hesaplama * tembel * olacaktır.

Hesaplanması gereken bir değeri yaymanın alternatif yolu #

“Observable.defer”, “Observable.fromCallable” gibi bir “Gözlenebilir” oluşturur, ancak bir değer yerine bir “Gözlenebilir” döndürmeniz gerektiğinde kullanılır. Çağrınızdaki hataları yönetmek istediğinizde kullanışlıdır.

Observable.defer(() -> {
          try {
                return Observable.just(longComputation());
          } catch(SpecificException e) {
                return Observable.error(e);
          }).subscribe(System.out::println);

Sıcak ve Soğuk Gözlemlenebilirler #

Gözlemlenebilirler, emisyon davranışlarına bağlı olarak genel olarak “Sıcak” veya “Soğuk” olarak sınıflandırılır.
Bir “Soğuk Gözlemlenebilir” istek (abonelik) üzerine yayılmaya başlayandır, “Sıcak Gözlemlenebilir” ise aboneliklerden bağımsız olarak yayın yapandır.

Soğuk Gözlemlenebilir

/* Demonstration of a Cold Observable */
Observable<Long> cold = Observable.interval(500, TimeUnit.MILLISECONDS); // emits a long every 500 milli seconds
cold.subscribe(l -> System.out.println("sub1, " + l)); // subscriber1
Thread.sleep(1000); // interval between the two subscribes
cold.subscribe(l -> System.out.println("sub2, " + l)); // subscriber2

Yukarıdaki kodun çıktısı şuna benzer (değişebilir):

sub1, 0    -> subscriber1 starts
sub1, 1
sub1, 2
sub2, 0    -> subscriber2 starts
sub1, 3
sub2, 1
sub1, 4
sub2, 2

‘sub2’ geç başlasa da değerleri baştan aldığına dikkat edin. Sonuç olarak, bir “Soğuk Gözlemlenebilir” yalnızca istendiğinde öğeleri yayar. Birden çok istek, birden çok işlem hattını başlatır.

Sıcak Gözlemlenebilir

Not: Sıcak gözlemlenebilirler, bireysel aboneliklerden bağımsız değerler yayar. Kendi zaman çizelgelerine sahiptirler ve birileri dinlese de dinlemese de olaylar meydana gelir.

Bir “Soğuk Gözlem”, basit bir “yayınlama” ile bir “Sıcak Gözlemlenebilir"e dönüştürülebilir.

Observable.interval(500, TimeUnit.MILLISECONDS)
    .publish(); // publish converts cold to hot

‘yayınla’, gözlemlenebilirden bağlanmak ve bağlantıyı kesmek için işlevler ekleyen bir ‘ConnectableObservable’ döndürür.

ConnectableObservable<Long> hot = Observable
                                    .interval(500, TimeUnit.MILLISECONDS)
                                    .publish(); // returns ConnectableObservable
hot.connect(); // connect to subscribe

hot.subscribe(l -> System.out.println("sub1, " + l));
Thread.sleep(1000);
hot.subscribe(l -> System.out.println("sub2, " + l));

Yukarıdaki verimler:

sub1, 0  -> subscriber1 starts
sub1, 1
sub1, 2
sub2, 2  -> subscriber2 starts
sub1, 3
sub2, 3

“sub2"nin gözlemlemeye geç başlamasına rağmen “sub1” ile senkronize olduğuna dikkat edin.
Bağlantıyı kesmek biraz daha karmaşıktır! Bağlantı kesilmesi, “Gözlenebilir"de değil “Abonelik"te gerçekleşir.

ConnectableObservable<Long> hot = Observable
                                    .interval(500, TimeUnit.MILLISECONDS)
                                    .publish(); // same as above
Subscription subscription = hot.connect(); // connect returns a subscription object, which we store for further use

hot.subscribe(l -> System.out.println("sub1, " + l));
Thread.sleep(1000);
hot.subscribe(l -> System.out.println("sub2, " + l));
Thread.sleep(1000);
subscription.unsubscribe(); // disconnect, or unsubscribe from subscription

System.out.println("reconnecting");
/* reconnect and redo */
subscription = hot.connect();
hot.subscribe(l -> System.out.println("sub1, " + l));
Thread.sleep(1000);
hot.subscribe(l -> System.out.println("sub2, " + l));
Thread.sleep(1000);
subscription.unsubscribe();

Yukarıdaki üretir:

sub1, 0   -> subscriber1 starts
sub1, 1
sub1, 2
sub2, 2   -> subscriber2 starts
sub1, 3
sub2, 3
reconnecting  -> reconnect after unsubscribe
sub1, 0
...

Bağlantı kesildiğinde, “Gözlenebilir” esasen “sona erer” ve yeni bir abonelik eklendiğinde yeniden başlar.

‘Hot Observable’, ‘EventBus’ oluşturmak için kullanılabilir. Bu tür EventBus’lar genellikle hafif ve süper hızlıdır. Bir RxBus’un tek dezavantajı, tüm olayların manuel olarak uygulanması ve veri yoluna iletilmesi gerektiğidir.
