JUnit ve Maven Surefire eklentisi ile bir Java sınıfını test etme #

Maven Surefire eklentisi, Maven oluşturma sürecinin test aşamasında veya Maven hedefi olarak “test” belirtildiğinde çalışır. Aşağıdaki dizin yapısı ve minimum pom.xml dosyası Maven’i bir test çalıştıracak şekilde yapılandıracaktır.

Projenin kök dizini içindeki dizin yapısı:

─ project_root 
  ├─ pom.xml
  ├─ src
  │  ├─ main
  │  │  └─ java
  │  └─ test
  │     └─ java
  └─ target
     └─ ...

“pom.xml” içeriği:

<project>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.example</groupId>
  <artifactId>company-app</artifactId>
  <version>0.0.1</version>
  <dependencies>
    <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>4.11</version>
      <scope>test</scope>
    </dependency>
  </dependencies>
</project>

Projenin src/test/java/com/example/app dizininde aşağıdaki içeriklerle PlusTenTest.java adında bir dosya oluşturun:

package com.example.app;

import static org.junit.Assert.assertEquals;
import org.junit.Test;

public class PlusTenTest {
    
    @Test
    public void incrementTest() {
        int result = PlusTen.increment(10);
        assertEquals("PlusTen.increment(10) result", 20, result);
    }
}

@Test ek açıklaması, JUnit’e Maven oluşturma işleminin “test” aşaması sırasında bir test olarak “incrementTest()” çalıştırması gerektiğini söyler. Şimdi ‘src/main/java/com/example/app’ içinde ‘PlusTen.java’ oluşturun:

package com.example.app;

public class PlusTen {
    public static int increment(int value) {
        return value;
    }
}

Bir komut istemi açarak, projenin kök dizinine giderek ve aşağıdaki komutu çağırarak testi çalıştırın:

mvn -Dtest=com.example.app.PlusTenTest test

Maven programı derleyecek ve “PlusTenTest” içinde “incrementTest()” test yöntemini çalıştıracaktır. Test aşağıdaki hatayla başarısız olur:

Maven Surefire eklentisi, projenizin dizininde “com.example.app.PlusTenTest.txt” ve “TEST-com.example.app.PlusTenTest.xml” dosyalarını içeren bir “/target/surefire-reports/” dizini oluşturur. yukarıdaki çıktının başlangıcındaki hata ayrıntıları.

Test odaklı geliştirme modelini izleyerek “PlusTen.java"yı “increments()” yönteminin doğru çalışması için değiştirin:

package com.example.app;

public class PlusTen {
    public static int increment(int value) {
        return value + 10;
    }
}

Komutu tekrar çağırın:

mvn -Dtest=com.example.app.PlusTenTest test
