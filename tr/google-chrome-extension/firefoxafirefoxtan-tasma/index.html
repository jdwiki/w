Firefox'aFirefox'tan taşıma

  • WebExtensions aracılığıyla taşıma

Bu sayfada

  • WebExtensions aracılığıyla taşıma

48’den önceki bir Firefox sürümünü kullanıyorsanız, ayrıca bir “manifest.json” adlı uygulamalarda ek anahtar:

"applications": {
  "gecko": {
    "id": "borderify@example.com",
    "strict_min_version": "42.0",
    "strict_max_version": "50.*",
    "update_url": "https://example.com/updates.json"
  }
}

Uzantı İmzalama:

Firefox 48’in piyasaya sürülmesiyle, uzantı imzalama artık bir tercih kullanarak sürüm ve beta kanalı derlemelerinde devre dışı bırakıldı. Uzatma imzası duyurulduğunda belirtildiği gibi, yayınlıyoruz geliştiricilerin kullanabilmesi için bu tercihi destekleyen özel yapılar beta ve sürüm derlemelerinin geçerli olduğu koda karşı test etmeye devam edin kaynaklı.

“WebExtensions” durumu:

WebExtensions şu anda deneysel bir alfa durumundadır. İtibaren Firefox 46, WebExtensions’ı Firefox kullanıcılarına yayınlayabilirsiniz. başka herhangi bir eklenti. Firefox 48’de ilk kararlı sürümü hedefliyoruz.

UPD: Firefox 48, 02.08.2016 tarihinde yayınlandı.

WebExtensions aracılığıyla taşıma #

Firefox uzantılarının taşınmasından bahsetmeden önce, ‘WebExtensions’ın ne olduğunu bilmek gerekir.

“WebExtensions” - Firefox uzantıları oluşturmak için bir API’yi temsil eden bir platformdur.

Chromium ile aynı uzantı mimarisini kullanır, sonuç olarak bu API, Google Chrome ve Opera‘daki (Chromium tabanlı Opera) API ile birçok yönden uyumludur. Çoğu durumda, bu tarayıcılar için geliştirilen uzantılar, Firefox’ta birkaç değişiklikle veya hatta bunlar olmadan çalışacaktır.

MDN, yeni uzantılar için “WebExtension” kullanılmasını öneriyor:

Gelecekte, WebExtensions geliştirmenin önerilen yolu olacaktır. Firefox eklentileri ve diğer sistemler kullanımdan kaldırılacaktır.

Yukarıdakileri göz önünde bulundurarak, uzantıları Firefox‘a taşımak istiyorsanız, uzantının nasıl yazıldığını bilmelisiniz.

Firefox için uzantılar “WebExtension”, “Add-on SDK” veya “XUL"a dayalı olabilir.

WebExtension’a dayalı uyumlu uzantılar #

‘WebExtension’ kullanırken, bazı işlevler tamamen veya kısmen desteklendiğinden, uyumsuzluklar listesine bakılmalıdır, yani bir başka deyişle, kişinin ‘manifest.json’ dosyasını kontrol etmesi gerekir.

Aynı ad alanını kullanmayı da sağlar:

Şu anda, tüm API’lere chrome.* ad alanı aracılığıyla erişilebilir. Kendi API’lerimizi eklemeye başladığımızda, onları tarayıcı.* ad alanı. Geliştiriciler özellik algılamayı kullanabilecek tarayıcıda bir API olup olmadığını belirlemek için.*.

Firefox ve Google Chrome’da çalışabilen basit bir uzantı

“manifest.json”:

{
  "manifest_version": 2,
  
  "name": "StackMirror",
  
  "version": "1.0",
  
  "description": "Mirror reflection of StackOverflow sites",
  
  "icons": {
    "48": "icon/myIcon-48.png"
  },
  
  "page_action": {
    "default_icon": "icon/myIcon-48.png"
  },
  
  "background": {
    "scripts"   : ["js/background/script.js"],
    "persistent": false
  },
  
  "permissions": ["tabs", "*://*.stackoverflow.com/*"]
}

“arka plan” komut dosyası:

function startScript(tabId, changeInfo, tab) {

    if (tab.url.indexOf("stackoverflow.com") > -1) {

        chrome.tabs.executeScript(tabId, 
            
            {code: 'document.body.style.transform = "scaleX(-1)";'}, function () {

            if (!chrome.runtime.lastError) {

                chrome.pageAction.show(tabId);
            }
        });
    }
}

chrome.tabs.onUpdated.addListener(startScript);

Projeyi standart “zip” dosyası olarak, ancak “.xpi” uzantılarıyla paketleyin.
