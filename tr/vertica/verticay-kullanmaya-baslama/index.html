Kurulum veya Kurulum #

İşletim Sistemi Kurulumu (Linux-CentOS x64 veya RedHat x64)

1-Takas alanını minimum 2 Gb’ye yükseltin

2-Alan ve CPU gereksinimleri:

-Vertica, CPU başına en az 1 GB gerektirir.

  • düğüm başına disk kullanımı yüzde altmıştan (%60) fazla olmamalıdır. Diske dökülmeleri gerektiğinde, karma birleştirmeler ve sıralamalar gibi belirli sorgu yürütme operatörleri tarafından geçici olarak disk alanı gerekir.

  • TEMP SPACE’i veri disk alanından ayrı olarak yapılandırın.

3-Vertica Cluster için ön koşulu yükleyin

yum install  rsync  python*  telnet ruby*  java* sudo  openssh-server openssh-clients
chkconfig sshd on
service sshd start

4- /etc/pam.d/su dosyasını düzenleyin

vi  /etc/pam.d/su
#add the line
session required pam_limits.so

5-NTP Arka Plan Programının Çalıştığını Doğrulayın

chkconfig --list ntpd
#if is not on use the commands
chkconfig ntpd on
#start ntp service
/etc/init.d/ntpd start

6-Gereksiz Uygulamaları Kaldır

For optimal performance, Vertica is designed to use all available resources on each host machine. Vertica recommends that you remove or disable all non-essential applications from cluster hosts.

7-Ağı Yapılandırma

/etc/hosts dosyasını değiştirin. Dosyanın var olduğundan ve 127.0.0.1 geri döngü adresini içerdiğinden emin olun.

7.1-Küme Ana Bilgisayarlarını Kurma

/etc/hosts dosyasının, kümenin parçası haline gelen tüm ana bilgisayarları içerdiğinden emin olun. Örneğin, ana bilgisayarlar host01, host02, host03 ve host04 olarak adlandırılmışsa, her bir ana bilgisayardaki /etc/hosts dosyası şöyle görünür:

# Do not remove the following line, or various programs
# that require network functionality will fail.
127.0.0.1               localhost.localdomain localhost
192.xxx.13.128          host01
192.xxx.13.129          host02
192.xxx.13.130          host03
192.xxx.13.131          host04

Bu, tüm ana bilgisayarlarda (düğümlerde) yapılmalıdır.

7.2- /etc/sysconfig/network dosyasını düzenleyin :

vim /etc/ sysconfig/network
Alter the hostname and set it to the desired name :
HOSTNAME=host01

7.3 HOSTNAME Ortam Değişkenini Ayarlama

vim /etc/profile or /etc/bashrc
Add the line
export HOSTNAME=hostname

7.4-Ana bilgisayar adı çözümlemesinin doğru çalıştığını doğrulayın Bunu komutla doğrulayın

/bin/hostname
Hostname

Ana bilgisayarları (düğümleri) yeniden başlatın Tüm bu adımları tüm Ana Bilgisayarlarda (düğümlerde) kök kullanıcı olarak yaptığınızdan emin olun.

7.5 Güvenlik duvarını devre dışı bırakın

Veritabanı ana bilgisayarları için önerilmeyen güvenlik duvarları SELinux (Güvenlik-Geliştirilmiş Linux) iptables

8 -Vertica yazılımını Ana Düğüm üzerine indirin ve kurun(kümenin İdari işlerini burada çalıştıracaksınız)

Kök olarak:

#rpm -ihv  vertica-<version>.x86_64.RHEL5.rpm
After entering the command, a progress indicator appears:
Preparing...   ##################################### [100%]
1:vertica      ##################################### [100%]
Vertica 6.0.xx successfully installed on host hostname.

Normalde varsayılan olarak vertica, /opt/vertica dizinine yüklenir.

8.1- Yükleme Komut Dosyasını Çalıştırın

Ana düğümde aşağıdaki komutu çalıştırın (Ana düğüm, yönetici çalışması için en çok eriştiğiniz düğüm olacaktır)

/opt/vertica/sbin/install_vertica -s host_list -r rpm_package -u dba_username

Seçenekler nerede : -s host_list kümeye dahil edilecek ana bilgisayar adlarının veya IP adreslerinin virgülle ayrılmış listesi; listeye boşluk karakterleri eklemeyin. Örnek :

-s host01,host02,host03

veya

-s 192.xxx.233.101,192.xxx.233.102,192.xxx.233.103

-r rpm_package Vertica RPM paketinin yol adı.

Örnek:

-r “vertica_6.0.x.x86_64.RHEL5.rpm” -u dba_kullanıcı adı -bu, admintools’u nasıl çalıştıracak kullanıcının adı olacaktır(yalnızca)

  • Parametreyi atlarsanız, varsayılan veritabanı yöneticisi hesap adı dbadmin’dir.

3 düğüm kümesi için tam komut örneği:

# /opt/vertica/sbin/install_vertica -s 10.xxx.1.216,10.xxx.1.224,10.xxx.1.225 -r
 /home/user/Downloads/vertica-6.0.0-3.x86_64.RHEL5.rpm -u dbadmin

9 – Her düğümde Vertica Veri ve Katalog dizinlerini oluşturun:

Directory names are totally up to you. Remember that database user must have owner rights over them.

Not: Bu, kurulumdan sonra eklenen ekstra dizinleri ifade eder

mkdir /vertica_db/data
mkdir /vertica_db/data
chown  dbadmin:dbadmin  vertica_db/

10- /opt/vertica/bin’i dbadmin kullanıcı yolunuza ekleyin, böylece tam yol olmadan vertica araçlarına erişebilirsiniz.

11- adminTool’a erişin ve Vertica tarafından sağlanan licence.dat dosyasını tanıtmanız istenecektir.(bu sadece Enterprise Edition için geçerlidir).

12- Veritabanını oluşturun ve hangi düğümlerde bulunacağını seçin !! Vertica Cluster’da bulunacak bir veritabanının nasıl oluşturulacağını görmek için sonraki adımları izleyin.
