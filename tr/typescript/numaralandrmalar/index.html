Tüm enum değerleri nasıl alınır #

enum SomeEnum { A, B }

let enumValues:Array<string>= [];

for(let value in SomeEnum) {
    if(typeof SomeEnum[value] === 'number') {
        enumValues.push(value);
    }
}

enumValues.forEach(v=> console.log(v))
//A
//B

Açık değerlere sahip numaralandırmalar #

Varsayılan olarak tüm “enum” değerleri sayılara çözümlenir. Diyelim ki sizin gibi bir şeyiniz varsa

enum MimeType {
  JPEG,
  PNG,
  PDF
}

arkasındaki gerçek değer örn. “MimeType.PDF”, “2” olacaktır.

Ancak bazen numaralandırmanın farklı bir türe çözülmesi önemlidir. Örneğin. değeri kesinlikle bir dize olan arka uç / ön uç / başka bir sistemden alırsınız. Bu bir acı olabilir, ama neyse ki şu yöntem var:

enum MimeType {
  JPEG = <any>'image/jpeg',
  PNG = <any>'image/png',
  PDF = <any>'application/pdf'
}

Bu, “MimeType.PDF” dosyasını “application/pdf” olarak çözer.

TypeScript 2.4’ten beri, string enums belirtmek mümkündür:

enum MimeType {
  JPEG = 'image/jpeg',
  PNG = 'image/png',
  PDF = 'application/pdf',
}

Aynı yöntemi kullanarak açıkça sayısal değerler sağlayabilirsiniz.

enum MyType {
   Value = 3,
   ValueEx = 30,
   ValueEx2 = 300
}

Fancier türleri de çalışır, çünkü const olmayan numaralandırmalar çalışma zamanında gerçek nesnelerdir, örneğin

enum FancyType {
   OneArr = <any>[1],
   TwoArr = <any>[2, 2],
   ThreeArr = <any>[3, 3, 3]
}

olur

var FancyType;
(function (FancyType) {
    FancyType[FancyType["OneArr"] = [1]] = "OneArr";
    FancyType[FancyType["TwoArr"] = [2, 2]] = "TwoArr";
    FancyType[FancyType["ThreeArr"] = [3, 3, 3]] = "ThreeArr";
})(FancyType || (FancyType = {}));

Özel numaralandırma uygulaması olmadan numaralandırmaları genişletme #

enum SourceEnum {
  value1 = <any>'value1',
  value2 = <any>'value2'
}

enum AdditionToSourceEnum {
  value3 = <any>'value3',
  value4 = <any>'value4'
}

// we need this type for TypeScript to resolve the types correctly
type TestEnumType = SourceEnum | AdditionToSourceEnum;
// and we need this value "instance" to use values
let TestEnum = Object.assign({}, SourceEnum, AdditionToSourceEnum);
// also works fine the TypeScript 2 feature
// let TestEnum = { ...SourceEnum, ...AdditionToSourceEnum };

function check(test: TestEnumType) {
  return test === TestEnum.value2;
}

console.log(TestEnum.value1);
console.log(TestEnum.value2 === <any>'value2');
console.log(check(TestEnum.value2));
console.log(check(TestEnum.value3));

Özel numaralandırma uygulaması: numaralandırmalar için genişler #

Bazen Enum’u kendi başınıza uygulamak gerekir. Örneğin. diğer numaralandırmaları genişletmenin net bir yolu yoktur. Özel uygulama buna izin verir:

class Enum {
  constructor(protected value: string) {}

  public toString() {
    return String(this.value);
  }

  public is(value: Enum | string) {
    return this.value = value.toString();
  }
}

class SourceEnum extends Enum {
  public static value1 = new SourceEnum('value1');
  public static value2 = new SourceEnum('value2');
}

class TestEnum extends SourceEnum {
  public static value3 = new TestEnum('value3');
  public static value4 = new TestEnum('value4');
}

function check(test: TestEnum) {
  return test === TestEnum.value2;
}

let value1 = TestEnum.value1;

console.log(value1 + 'hello');
console.log(value1.toString() === 'value1');
console.log(value1.is('value1'));
console.log(!TestEnum.value3.is(TestEnum.value3));
console.log(check(TestEnum.value2));
// this works but perhaps your TSLint would complain
// attention! does not work with ===
// use .is() instead
console.log(TestEnum.value1 == <any>'value1');
