Kurulum ve kurulum #

Arka fon #

TypeScript, doğrudan JavaScript koduna derlenen JavaScript’in yazılı bir üst kümesidir. TypeScript dosyaları genellikle “.ts” uzantısını kullanır. Birçok IDE, başka bir kurulum gerektirmeden TypeScript’i destekler, ancak TypeScript, komut satırından TypeScript Node.JS paketiyle de derlenebilir.

IDE #

Görsel stüdyo #

  • ‘Visual Studio 2015’ TypeScript’i içerir.
  • ‘Visual Studio 2013 Güncelleme 2’ veya üzeri TypeScript’i içerir veya önceki sürümler için TypeScript’i indirebilirsiniz.

Visual Studio Kodu #

  • Visual Studio Code (vscode), TypeScript için bağlamsal otomatik tamamlamanın yanı sıra yeniden düzenleme ve hata ayıklama araçları sağlar. vscode’un kendisi TypeScript’te uygulanmaktadır. Mac OS X, Windows ve Linux için kullanılabilir.

Web Fırtınası #

  • WebStorm 2016.2, TypeScript ve yerleşik bir derleyici ile birlikte gelir. [Web fırtınası ücretsiz değil]

IntelliJ FİKİRİ #

  • IntelliJ IDEA 2016.2, TypeScript desteğine ve eklenti aracılığıyla derleyiciye sahiptir /typescript-support.html) Jetbrains ekibi tarafından sağlanır. [IntelliJ ücretsiz değil]

Atom ve atom-daktilo yazısı #

  • Atom, TypeScript’i atom-typescript paketiyle destekler.

Yüce metin #

  • Sublime Text, TypeScript’i typescript paketiyle destekler.

Komut satırı arayüzünün kurulması #

[Node.js]‘yi yükleyin(https://nodejs.org/) #

npm paketini global olarak kurun #

TypeScript’e herhangi bir dizinden erişmek için global olarak yükleyebilirsiniz.

npm install -g typescript

veya

npm paketini yerel olarak kurun #

TypeScript’i yerel olarak yükleyebilir ve bir dizine sınırlamak için package.json’a kaydedebilirsiniz.

npm install typescript --save-dev

Kurulum kanalları #

Şuradan yükleyebilirsiniz:

  • Kararlı kanal: `npm typescript’i kurun’
  • Beta kanalı: npm install typescript@beta
  • Geliştirme kanalı: `npm typescript@next’i kurun’

TypeScript kodunu derleme #

“tsc” derleme komutu, kodu derlemek için kullanılabilen “typescript” ile birlikte gelir.

tsc my-code.ts

Bu, bir “my-code.js” dosyası oluşturur.

tsconfig.json kullanarak derleyin #

Bir tsconfig.json dosyası aracılığıyla kodunuzla birlikte hareket eden derleme seçenekleri de sağlayabilirsiniz. Yeni bir TypeScript projesi başlatmak için, bir terminal penceresinde projenizin kök dizinine “cd” yazın ve “tsc –init” komutunu çalıştırın. Bu komut, aşağıdakine benzer minimum yapılandırma seçenekleriyle bir ’tsconfig.json’ dosyası oluşturacaktır.

{
    "compilerOptions": {
        "module": "commonjs",
        "target": "es5",
        "noImplicitAny": false,
        "sourceMap": false,
        "pretty": true
    },
    "exclude": [
        "node_modules"
    ]
}

TypeScript projenizin köküne yerleştirilmiş bir ’tsconfig.json’ dosyasıyla, derlemeyi çalıştırmak için ’tsc’ komutunu kullanabilirsiniz.

Temel sözdizimi #

TypeScript, JavaScript’in yazılı bir üst kümesidir; bu, tüm JavaScript kodunun geçerli TypeScript kodu olduğu anlamına gelir. TypeScript, bunun üzerine birçok yeni özellik ekler.

TypeScript, JavaScript’i C# ve Java’ya benzeyen, kesin olarak yazılmış, nesne yönelimli bir dil gibi yapar. Bu, TypeScript kodunun büyük projeler için kullanımının daha kolay olduğu ve bu kodun anlaşılması ve bakımının daha kolay olduğu anlamına gelir. Güçlü yazma ayrıca dilin önceden derlenebileceği (ve oluşturulduğu) ve değişkenlere beyan edilen aralıklarının dışında değerler atanamayacağı anlamına gelir. Örneğin, bir TypeScript değişkeni bir sayı olarak bildirildiğinde, ona bir metin değeri atayamazsınız.

Bu güçlü yazma ve nesne yönelimi, TypeScript’in hata ayıklamasını ve bakımını kolaylaştırır ve bunlar standart JavaScript’in en zayıf noktalarından ikisiydi.

Tür bildirimleri #

Değişkenlere, fonksiyon parametrelerine ve fonksiyon dönüş tiplerine tip bildirimleri ekleyebilirsiniz. Tür, değişken adından sonra iki nokta üst üste işaretinden sonra yazılır, şöyle: var num: number = 5; Derleyici daha sonra derleme sırasında türleri (mümkünse) kontrol edecek ve tür hatalarını rapor edecektir.

var num: number = 5;
num = "this is a string";  // error: Type 'string' is not assignable to type 'number'.

Temel türleri şunlardır:

  • “sayı” (hem tam sayılar hem de kayan noktalı sayılar)

  • “dize”

  • “boole”

  • ‘Dizi’. Bir dizinin öğelerinin türlerini belirtebilirsiniz. Dizi türlerini tanımlamanın iki eşdeğer yolu vardır: “Array” ve “T[]”. Örneğin: - number[] - array of numbers - Array<string> - array of strings

  • Tuple’lar. Tuple’lar, belirli türlere sahip sabit sayıda öğeye sahiptir. - [boolean, string] - tuple where the first element is a boolean and the second is a string. - [number, number, number] - tuple of three numbers.

  • {} - nesne, özelliklerini veya indeksleyicisini tanımlayabilirsiniz

    • {name: string, age: number} - object with name and age attributes
    • {[key: string]: number} - a dictionary of numbers indexed by string

  • enum - { Red = 0, Blue, Green } - sayılarla eşlenen numaralandırma

  • İşlev. Parametreler ve dönüş değeri için türleri belirtirsiniz:

    • (param: number) => string - function taking one number parameter returning string
    • () => number - function with no parameters returning an number.
    • (a: string, b?: boolean) => void - function taking a string and optionally a boolean with no return value.

  • any - Her türe izin verir. ‘any’ içeren ifadeler tip kontrolü yapılmaz.

  • “void” - “hiçbir şey"i temsil eder, bir işlev dönüş değeri olarak kullanılabilir. Yalnızca “boş” ve “tanımsız”, “void” türünün parçasıdır.

  • “asla”

    • let foo: never; -As the type of variables under type guards that are never true.
    • function error(message: string): never { throw new Error(message); } - Asla geri dönmeyen işlevlerin dönüş türü olarak.

  • “null” - “null” değerini yazın. ’null’, katı boş denetimler etkinleştirilmedikçe örtük olarak her türün parçasıdır.

Döküm #

Açılı ayraçlar aracılığıyla açık döküm gerçekleştirebilirsiniz, örneğin:

var derived: MyInterface;
(<ImplementingClass>derived).someSpecificMethod();

Bu örnek, derleyici tarafından “MyInterface” olarak değerlendirilen “türetilmiş” bir sınıfı göstermektedir. İkinci satıra atama olmadan, derleyici ‘someSpecificMethod()‘u anlamadığından bir istisna atar, ancak ‘türetilmiş’ aracılığıyla yayın yapmak derleyiciye ne yapması gerektiğini önerir.

TypeScript’te yayınlamanın başka bir yolu da “as” anahtar sözcüğünü kullanmaktır:

var derived: MyInterface;
(derived as ImplementingClass).someSpecificMethod();

Typescript 1.6’dan beri, varsayılan olarak ‘as’ anahtar sözcüğü kullanılır, çünkü .jsx dosyalarında ‘<>’ kullanımı belirsizdir. Bu, Typescript resmi belgelerinde belirtilmiştir.

Sınıflar #

TypeScript kodunda sınıflar tanımlanabilir ve kullanılabilir. Sınıflar hakkında daha fazla bilgi edinmek için Sınıf belgeleri sayfasına bakın.

Selam Dünya #

class Greeter {
    greeting: string;

    constructor(message: string) {
        this.greeting = message;
    }
    greet(): string {
        return this.greeting;
    }
};

let greeter = new Greeter("Hello, world!");
console.log(greeter.greet());

Burada bir “constructor” ve bir “greet” yöntemi olan bir “Greeter” sınıfımız var. ’new’ anahtar sözcüğünü kullanarak sınıfın bir örneğini oluşturabilir ve ‘greet’ yönteminin konsola çıktısını almasını istediğimiz bir dize iletebiliriz. ‘Greeter’ sınıfımızın örneği, daha sonra ‘greet’ yöntemini çağırmamız için ‘greeter’ değişkeninde saklanır.

Node.js’de TypeScript REPL #

Node.js’de TypeScript REPL kullanmak için tsun paketi kullanabilirsiniz.

ile global olarak kurun

npm install -g tsun

ve terminalinizde veya komut isteminde ’tsun’ komutuyla çalıştırın

Kullanım örneği:

$ tsun
TSUN : TypeScript Upgraded Node
type in TypeScript expression to evaluate
type :help for commands in repl
$ function multiply(x, y) {
..return x * y;
..}
undefined
$ multiply(3, 4)
12

TypeScript’i ts-node kullanarak çalıştırma #

ts-node, kullanıcının “tsc” kullanarak önceden derlemeye gerek kalmadan daktilo dosyalarını doğrudan çalıştırmasını sağlayan bir npm paketidir. Ayrıca REPL sağlar.

ts-node’u kullanarak global olarak kurun

npm install -g ts-node

ts-node, TypeScript derleyicisini paketlemez, bu nedenle yüklemeniz gerekebilir.

npm install -g typescript

Komut dosyasını yürütme

main.ts adlı bir komut dosyasını çalıştırmak için şunu çalıştırın:

ts-node main.ts

// main.ts
console.log("Hello world");

Örnek kullanım

$ ts-node main.ts
Hello world

REPL çalıştırılıyor

REPL’yi çalıştırmak için “ts-node” komutunu çalıştırın.

Örnek kullanım

$ ts-node
> const sum = (a, b): number => a + b;
undefined
> sum(2, 2)
4
> .exit

REPL’den çıkmak için .exit komutunu kullanın veya iki kez CTRL+C tuşlarına basın.
