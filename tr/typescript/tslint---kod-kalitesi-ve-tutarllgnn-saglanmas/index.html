TSLint - kod kalitesi ve tutarlılığının sağlanması

TSLint, kodun statik analizini gerçekleştirir ve koddaki hataları ve olası sorunları tespit eder.

Daha az programlama hatası için yapılandırma #

Bu tslint.json örneği, daha fazla yazma zorlamak, yaygın hataları yakalamak veya hata üretmeye meyilli olan ve daha fazla TypeScript Katkıda Bulunanlar için Kodlama Yönergeleri’ni takip eden kafa karıştırıcı yapıları yakalamak için bir dizi yapılandırma içerir.

Bu kuralları uygulamak için, oluşturma işleminize tslint’i ekleyin ve kodunuzu tsc ile derlemeden önce kontrol edin.

{
  "rules": {
     // TypeScript Specific
     "member-access": true, // Requires explicit visibility declarations for class members.
     "no-any": true, // Diallows usages of any as a type declaration.
     // Functionality
     "label-position": true, // Only allows labels in sensible locations.
     "no-bitwise": true, // Disallows bitwise operators.
     "no-eval": true, // Disallows eval function invocations.
     "no-null-keyword": true, // Disallows use of the null keyword literal.
     "no-unsafe-finally": true, // Disallows control flow statements, such as return, continue, break and throws in finally blocks.
     "no-var-keyword": true, // Disallows usage of the var keyword.
     "radix": true, // Requires the radix parameter to be specified when calling parseInt.
     "triple-equals": true, // Requires === and !== in place of == and !=.
     "use-isnan": true, // Enforces use of the isNaN() function to check for NaN references instead of a comparison to the NaN constant.
     // Style
     "class-name": true, // Enforces PascalCased class and interface names. 
     "interface-name": [ true, "never-prefix" ], // Requires interface names to begin with a capital ‘I’
     "no-angle-bracket-type-assertion": true, // Requires the use of as Type for type assertions instead of <Type>.
     "one-variable-per-declaration": true, // Disallows multiple variable definitions in the same declaration statement.
     "quotemark": [ true, "double", "avoid-escape" ], // Requires double quotes for string literals.
     "semicolon": [ true, "always" ], // Enforces consistent semicolon usage at the end of every statement.
     "variable-name": [true, "ban-keywords", "check-format", "allow-leading-underscore"] // Checks variable names for various errors. Disallows the use of certain TypeScript keywords (any, Number, number, String, string, Boolean, boolean, undefined) as variable or parameter. Allows only camelCased or UPPER_CASED variable names. Allows underscores at the beginning (only has an effect if “check-format” specified).
  }
}

Temel tslint.json kurulumu #

Bu, temel bir “tslint.json” kurulumudur.

  • ‘any’ kullanımını engeller

  • “if”/“else”/“for”/“do”/“while” ifadeleri için kaşlı ayraç gerektirir

  • dizeler için çift tırnak (") kullanılmasını gerektirir

    { “rules”: { “no-any”: true, “curly”: true, “quotemark”: [true, “double”] } }

Varsayılan olarak önceden tanımlanmış bir kural kümesi kullanma #

’tslint’ mevcut bir kural kümesini genişletebilir ve ’tslint:önerilen’ ve ’tslint:latest’ varsayılanlarıyla gönderilir.

’tslint:önerilen’, genel TypeScript programlaması için teşvik ettiğimiz, kararlı, biraz üzerinde düşünülmüş bir kurallar dizisidir. Bu yapılandırma semver’i takip eder, bu nedenle küçük veya yama sürümlerinde kırılma değişiklikleri olmaz.

’tslint:latest’, tslint:recommended’i genişletir ve her TSLint sürümündeki en son kurallar için yapılandırmayı içerecek şekilde sürekli olarak güncellenir. Bu yapılandırmayı kullanmak, kodunuzda tiftik hatalarına neden olan yeni kurallar etkinleştirildiğinden küçük sürümler arasında kırılma değişiklikleri getirebilir. TSLint bir ana sürüm tümseğine ulaştığında, tslint:recommended, tslint:latest ile aynı olacak şekilde güncellenecektir.

Belgeler ve önceden tanımlanmış kural kümesinin kaynak kodu

Yani biri basitçe kullanabilir:

{
  "extends": "tslint:recommended"
}

mantıklı bir başlangıç ​​yapılandırmasına sahip olmak.

Daha sonra “kurallar” aracılığıyla bu ön ayardaki kuralların üzerine yazılabilir, ör. düğüm geliştiricileri için “konsolsuz"u “yanlış” olarak ayarlamak mantıklıydı:

{
  "extends": "tslint:recommended",
  "rules": {
    "no-console": false
  }
}

Kurulum ve kurulum #

tslint yüklemek için komutu çalıştırın

npm install -g tslint

Tslint, tslint.json dosyası aracılığıyla yapılandırılır. Varsayılan yapılandırma çalıştırma komutunu başlatmak için

tslint --init

Dosya çalıştırma komutundaki olası hatalar için dosyayı kontrol etmek için

tslint filename.ts
