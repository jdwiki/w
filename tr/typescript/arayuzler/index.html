Bir arabirim, arabirimi uygulayan herhangi bir sınıfta beklenebilecek alanların ve işlevlerin bir listesini belirtir. Tersine, bir sınıf, arabirimde belirtilen her alana ve işleve sahip olmadığı sürece bir arabirim uygulayamaz.

Arayüzleri kullanmanın birincil faydası, farklı tipteki nesnelerin polimorfik bir şekilde kullanılmasına izin vermesidir. Bunun nedeni, arabirimi uygulayan herhangi bir sınıfın en azından bu alanlara ve işlevlere sahip olmasıdır.

Sözdizimi #

  • arayüz ArayüzAdı {
  •   parametreAdı: parametreTürü;
  •   isteğe bağlıParametreAdı?: parametreTürü;
  • }

Arabirimler ve Tür Takma Adları #

Arayüzler, bir nesnenin şeklini belirtmek için iyidir, örneğin belirtebileceğiniz bir kişi nesnesi için

interface person {
    id?: number;
    name: string;
    age: number;
}

Ancak, örneğin bir kişinin SQL veritabanında saklanma şeklini temsil etmek isterseniz ne olur? Her bir DB girişi bir “[string, string, number]” şeklinden (yani bir dizi veya sayı dizisinden) oluştuğu için, bunu bir nesne şekli olarak göstermenin hiçbir yolu yoktur, çünkü satırın herhangi bir özellik, bu sadece bir dizidir.

Bu, türlerin yararlı olduğu bir durumdur. Bir satır parametresini kabul eden her işlevde “işlev processRow(satır: [dize, dize, sayı])” belirtmek yerine, bir satır için ayrı bir tür takma adı oluşturabilir ve ardından bunu her işlevde kullanabilirsiniz:

type Row = [string, string, number];
function processRow(row: Row)

Resmi arayüz belgeleri #

https://www.typescriptlang.org/docs/handbook/interfaces.html

Genişletilmiş Arayüz #

Bir arayüzümüz olduğunu varsayalım:

interface IPerson {
    name: string;
    age: number;

    breath(): void;
}

Kişiyle aynı özelliklere sahip daha spesifik bir arayüz oluşturmak istiyoruz, bunu “extends” anahtar sözcüğünü kullanarak yapabiliriz:

interface IManager extends IPerson {
    managerId: number;

    managePeople(people: IPerson[]): void;
}

Ek olarak, birden fazla arabirimi genişletmek mümkündür.

Sınıf Arayüzü #

Diğer daktilo kodunun onunla nasıl etkileşime girebileceğini tanımlamak için arabirimde “genel” değişkenler ve yöntemler yazın.

interface ISampleClassInterface {
  sampleVariable: string;

  sampleMethod(): void;
  
  optionalVariable?: string;
}

Burada arayüzü uygulayan bir sınıf oluşturuyoruz.

class SampleClass implements ISampleClassInterface {
  public sampleVariable: string;
  private answerToLifeTheUniverseAndEverything: number;

  constructor() {
    this.sampleVariable = 'string value';
    this.answerToLifeTheUniverseAndEverything = 42;
  }

  public sampleMethod(): void {
    // do nothing
  }
  private answer(q: any): number {
    return this.answerToLifeTheUniverseAndEverything;
  }
}

Örnek, bir “ISampleClassInterface” arabiriminin ve arabirimi “uygulayan” bir “SampleClass” sınıfının nasıl oluşturulacağını gösterir.

Mevcut bir arayüze işlevler veya özellikler ekleyin #

Diyelim ki JQuery tip tanımına bir referansımız var ve onu dahil ettiğimiz ve resmi bir tip tanımı olmayan bir eklentiden ek fonksiyonlara sahip olacak şekilde genişletmek istiyoruz. Eklenti tarafından eklenen işlevleri aynı “JQuery” adıyla ayrı bir arabirim bildiriminde bildirerek kolayca genişletebiliriz:

interface JQuery {
  pluginFunctionThatDoesNothing(): void;

  // create chainable function
  manipulateDOM(HTMLElement): JQuery;
}

Derleyici aynı ada sahip tüm bildirimleri tek bir bildirimde birleştirecektir - daha fazla ayrıntı için bildirim birleştirme bölümüne bakın.

Genel Arayüzler #

Sınıflar gibi arabirimler de polimorfik parametreler (diğer adıyla Generics) alabilir.

Arayüzlerde Genel Parametreleri Bildirme #

interface IStatus<U> {
    code: U;
}

interface IEvents<T> {
    list: T[];
    emit(event: T): void;
    getAll(): T[];
}

Burada, iki arayüzümüzün bazı genel parametreleri, T ve U aldığını görebilirsiniz.

Genel Arayüzleri Uygulamak #

IEvents arayüzünü uygulamak için basit bir sınıf oluşturacağız.

class State<T> implements IEvents<T> {
    
    list: T[];
    
    constructor() {
        this.list = [];
    }
    
    emit(event: T): void {
        this.list.push(event);
    }
    
    getAll(): T[] {
        return this.list;
    }
    
}

State sınıfımızın bazı örneklerini oluşturalım.

Örneğimizde, ‘State’ sınıfı, ‘IStatus’ kullanarak genel bir durumu işleyecektir. Bu şekilde, ‘IEvent’ arabirimi bir ‘IStatus’ da işleyecektir.

const s = new State<IStatus<number>>();

// The 'code' property is expected to be a number, so:
s.emit({ code: 200 }); // works
s.emit({ code: '500' }); // type error 

s.getAll().forEach(event => console.log(event.code));

Burada ‘State’ sınıfımız ‘ISatus’ olarak yazılmıştır.

const s2 = new State<IStatus<Code>>();

//We are able to emit code as the type Code
s2.emit({ code: { message: 'OK', status: 200 } });

s2.getAll().map(event => event.code).forEach(event => {
    console.log(event.message);
    console.log(event.status);
});

‘State’ sınıfımız ‘IStatus’ olarak yazılmıştır. Bu sayede daha karmaşık bir türü emit yöntemimize geçirebiliyoruz.

Gördüğünüz gibi, genel arayüzler, statik olarak yazılan kod için çok kullanışlı bir araç olabilir.

Polimorfizm için Arayüzleri Kullanma #

Arayüzlerin polimorfizm elde etmek için kullanılmasının birincil nedeni ve geliştiricilerin arayüzün yöntemlerini uygulayarak gelecekte kendi yollarında uygulamalarını sağlamaktır.

Bir arayüzümüz ve üç sınıfımız olduğunu varsayalım:

interface Connector{
    doConnect(): boolean;
}

Bu bağlayıcı arayüzüdür. Şimdi bunu Wifi iletişimi için uygulayacağız.

export class WifiConnector implements Connector{

    public doConnect(): boolean{
        console.log("Connecting via wifi");
        console.log("Get password");
        console.log("Lease an IP for 24 hours");
        console.log("Connected");
        return true
    }

}

Burada kendi uygulaması olan WifiConnector isimli beton sınıfımızı geliştirdik. Bu şimdi ‘Bağlayıcı’ yazın.

Şimdi, ‘Connector’ bileşenine sahip ‘Sistemimizi’ oluşturuyoruz. Buna bağımlılık enjeksiyonu denir.

export class System {
    constructor(private connector: Connector){ #inject Connector type
        connector.doConnect()
    }
}

constructor(private connect: Connector) bu satır burada çok önemlidir. “Bağlayıcı” bir arayüzdür ve “doConnect()“e sahip olmalıdır. ‘Connector’ bir arayüz olduğundan, bu ‘Sistem’ sınıfı çok daha fazla esnekliğe sahiptir. ‘Connector’ arayüzünü uygulayan herhangi bir Türü geçebiliriz. Gelecekte geliştirici daha fazla esneklik elde eder. Örneğin, geliştirici şimdi Bluetooth Bağlantı modülü eklemek istiyor:

export class BluetoothConnector implements Connector{

    public doConnect(): boolean{
        console.log("Connecting via Bluetooth");
        console.log("Pair with PIN");
        console.log("Connected");
        return true
    }

}

Wifi ve Bluetooth’un kendi uygulamasına sahip olduğunu görün. Bağlanmanın kendi farklı yolu var. Ancak, bu nedenle her ikisi de Type ‘Connector’ uygulamıştır, şimdi Type ‘Connector’dır. Böylece bunlardan herhangi birini yapıcı parametresi olarak System sınıfına geçirebiliriz. Buna polimorfizm denir. System sınıfı artık Bluetooth / Wifi olup olmadığının farkında değil, sadece Connector arayüzünü uygulayarak Inferade, Bluetooth5 ve benzeri başka bir İletişim modülü ekleyebiliriz.

Buna Ördek yazarak denir. ‘DoConnect()’ sadece bir yer tutucu olduğundan ve geliştirici bunu kendisininmiş gibi uyguladığından, ‘Bağlayıcı’ türü artık dinamiktir.

“constructor(private connect: WifiConnector)” konumunda ise “WifiConnector” somut bir sınıfsa ne olacak? Daha sonra System sınıfı sadece WifiConnector ile sıkı bir şekilde eşleşecektir. Burada arayüz, problemimizi polimorfizm ile çözdü.

Türleri Zorlamak için Arayüzleri Kullanma #

Typescript’in temel faydalarından biri, hataları önlemeye yardımcı olmak için kodunuzda ilettiğiniz veri türlerini zorlamasıdır.

Diyelim ki bir evcil hayvan flört uygulaması yapıyorsunuz.

İki evcil hayvanın birbiriyle uyumlu olup olmadığını kontrol eden bu basit fonksiyona sahipsiniz…

checkCompatible(petOne, petTwo) {
  if (petOne.species === petTwo.species &&
      Math.abs(petOne.age - petTwo.age) <= 5) {
    return true;
  }
}

Bu tamamen işlevsel bir koddur, ancak birisi için, özellikle bu uygulamada çalışan ve bu işlevi yazmayan diğer kişiler için, ’tür’ ve ‘yaş’ olan nesneleri geçirmeleri gerektiğinin farkında olmaması çok kolay olurdu. özellikleri. Yanlışlıkla ‘checkCompatible(petOne.species, petTwo.species)’ dosyasını deneyebilirler ve ardından işlev petOne.species.species veya petOne.species.age’e erişmeye çalıştığında ortaya çıkan hataları bulmaya bırakılabilirler!

Bunun olmasını engellemenin bir yolu, evcil hayvan parametrelerinde istediğimiz özellikleri belirtmektir:

checkCompatible(petOne: {species: string, age: number}, petTwo: {species: string, age: number}) {
    //...
}

Bu durumda, Typescript, işleve iletilen her şeyin ’tür’ ve ‘yaş’ özelliklerine sahip olduğundan emin olur (ek özelliklere sahip olmaları sorun değildir), ancak bu, yalnızca iki özellik belirtilmiş olsa bile, biraz hantal bir çözümdür. Arayüzlerle daha iyi bir yol var!

İlk önce arayüzümüzü tanımlıyoruz:

interface Pet {
  species: string;
  age: number;
  //We can add more properties if we choose.
}

Şimdi tek yapmamız gereken yeni arayüzümüz olarak parametrelerimizin türünü belirtmek, bunun gibi…

checkCompatible(petOne: Pet, petTwo: Pet) {
  //...
}

… ve Typescript, fonksiyonumuza iletilen parametrelerin Pet arayüzünde belirtilen özellikleri içermesini sağlayacaktır!

Örtülü Uygulama ve Nesne Şekli #

TypeScript, arayüzleri destekler, ancak derleyici, JavaScript’i çıkarmaz. Bu nedenle, derleme adımında arabirimler etkin bir şekilde kaybolur. Bu nedenle arabirimlerde tür denetimi, arabirimin gerçekten uygulanıp uygulanmadığına değil, nesnenin arabirimdeki alanları ve işlevleri destekleyip desteklemediği anlamına gelen nesnenin şekline dayanır.

interface IKickable {
  kick(distance: number): void;
}
class Ball {
  kick(distance: number): void {
    console.log("Kicked", distance, "meters!");
  }
}
let kickable: IKickable = new Ball();
kickable.kick(40);

Dolayısıyla, ‘Ball’, ‘IKickable’ı açıkça uygulamasa bile, tür belirtilmiş olsa bile, bir ‘Ball’ örneği bir ‘IKickable’a atanabilir (ve bu şekilde manipüle edilebilir).
