Sözdizimi #

  • @mixin mixin-name ($argument1, $argument, ...){ ... }

Değişken argümanla karıştırma #

Karışımlarda, kullanırken tekli veya çoklu argümanların olabileceği bazı durumlar vardır. “border-radius:4px;” gibi tek bir argümanın veya “border-radius:4px 3px 2px 1px;” gibi birden fazla argümanın olabileceği bir “border-radius” örneğini ele alalım.

Anahtar Kelime Bağımsız Değişkenleri ile Geleneksel karıştırma aşağıdaki gibi olacaktır:-

@mixin radius($rad1, $rad2, $rad3, $rad4){
 -webkit-border-radius: $rad1 $rad2 $rad3 $rad4;
 -moz-border-radius: $rad1 $rad2 $rad3 $rad4;
 -ms-border-radius: $rad1 $rad2 $rad3 $rad4;
 -o-border-radius: $rad1 $rad2 $rad3 $rad4;
 border-radius: $rad1 $rad2 $rad3 $rad4;
}

ve olarak kullanılır

.foo{
    @include radius(2px, 3px, 5px, 6px)
}

Yukarıdaki örnek karmaşıktır (kodlamak, okumak ve sürdürmek) ve yalnızca bir veya iki değer iletemezse, bir hata verir ve bir, iki veya orada değerleri kullanmak için tanımlamanız gerekir. diğer üç karışım.

Değişken Argüman‘ı kullanarak, kaç argüman iletebileceğiniz konusunda endişelenmenize gerek yoktur. Değişken bağımsız değişkenler, bir değişken adı ve ardından üç nokta(…) getirilerek bildirilebilir. Aşağıda değişken argümanına bir örnek verilmiştir.

@mixin radius($radius...)
{  
    -webkit-border-radius: $radius;
    -moz-border-radius: $radius;
    -ms-border-radius: $radius;
    -o-border-radius: $radius;
    border-radius: $radius;
}

ve olarak kullanılır

.foo{
    @include radius(2px 3px 5px 6px)
}
.foo2{
    @include radius(2px 3px)
}
.foo3{
    @include radius(2px)
}

Yukarıdaki örnek çok daha basittir (kodlama, bakım ve okuma), ne kadar argüman geleceği konusunda endişelenmenize gerek yok - bir mi yoksa birden fazla mı?

Birden fazla argüman varsa ve her durumda ikinci argümana erişmek istiyorsanız, bunu propertyname : nth(variable_name, 2) yazarak yapabilirsiniz.

Bir karışım oluşturun ve kullanın #

Bir mixin oluşturmak için @mixin yönergesini kullanın.

@mixin default-box ($color, $borderColor) {
    color: $color;
    border: 1px solid $borderColor;
    clear: both;
    display: block;
    margin: 5px 0;
    padding: 5px 10px;
}

Karışımın adından sonra bir parantez içinde bir argüman listesi belirtebilirsiniz. Değişkenlerinizi $ ile başlatmayı ve virgülle ayırmayı unutmayın.

Karışımı başka bir seçicide kullanmak için @include direktifini kullanın.

footer, header{ @include default-box (#ddd, #ccc); }

Karışımdaki stiller artık “altbilgi” ve “üstbilgi"de kullanılacak, “$color” değişkeni için “#ccc” ve “$borderColor” değişkeni için “#ddd” değeri kullanılacak.

İsteğe bağlı bağımsız değişkenler #

SASS’ın isteğe bağlı bağımsız değişkenleri, yalnızca değerini belirtirseniz bir parametre kullanmanıza izin verir; aksi takdirde göz ardı edilecektir. Aşağıdaki karışıma bir örnek verelim:

@mixin galerie-thumbnail ($img-height:14em, $img-width: null) {
    width: $img-width;
    height: $img-height;
    outline: 1px solid lightgray;
    outline-offset: 5px;
}

yani bir çağrı

.default { 
  @include galerie-thumbnail; 
}
.with-width { 
  @include galerie-thumbnail($img-width: 12em);
}
.without-height { 
  @include galerie-thumbnail($img-height: null);
}

CSS dosyasında basitçe aşağıdakileri çıkarır:

.default {
  height: 14em;
  outline: 1px solid lightgray;
  outline-offset: 5px;
}

.with-width {
  width: 12em;
  height: 14em;
  outline: 1px solid lightgray;
  outline-offset: 5px;
}

.without-height {
  outline: 1px solid lightgray;
  outline-offset: 5px;
}

SASS, değerleri olarak “null” olan özellikleri çıkarmaz; bu, çağrımıza isteğe bağlı bir argüman eklememiz gerektiğinde çok yararlıdır.

@içerik yönergesi #

Mixin’ler bir SASS uyumlu kod bloğundan geçirilebilir ve bu kod daha sonra mixin içinde “@content” yönergesi olarak kullanılabilir hale gelir.

@mixin small-screen {
  @media screen and (min-width: 800px;) {
    @content;
  }
}

@include small-screen {
  .container {
    width: 600px;
  }
}

Ve bu çıktı:

  @media screen and (min-width: 800px;) {
    .container {
      width: 600px;
    }
  }

Mantıklı varsayılanlar #

SASS, elbette üzerine yazmak istedikleriniz dışında herhangi bir parametreyi atlama yeteneği verir. Tekrar “varsayılan kutu” örneğini ele alalım:

@mixin default-box ($color: red, $borderColor: blue) {
    color: $color;
    border: 1px solid $borderColor;
    clear: both;
    display: block;
    margin: 5px 0;
    padding: 5px 10px;
}

Burada şimdi ikinci parametrenin üzerine yazılan karışımı arayacağız

footer, header{ @include default-box ($borderColor: #ccc); }

“$borderColor” değeri “#ccc” iken “$color” değeri “kırmızı” olarak kalır
