Ana Farklılıklar #

İnsanlar genellikle bu CSS önişlemcisinin adı olarak ‘Sass’ deseler de, genellikle ‘SCSS’ sözdizimini ifade ederler. “Sass”, “.sass” dosya uzantısını kullanırken “SCSS”-“Sass”, “.scss” uzantısını kullanır. Her ikisi de “Sass” olarak anılır.

Genel olarak konuşursak, ‘SCSS’ sözdizimi daha yaygın olarak kullanılır. “SCSS”, daha fazla yeteneğe sahip normal CSS’ye benzerken “Sass”, normal CSS’den oldukça farklı görünür. Her iki sözdizimi de aynı yeteneklere sahiptir.

Sözdizimi #

Temel farklar, “Sass"ın, “SCSS"nin yaptığı yerde, küme parantezleri veya noktalı virgül kullanmamasıdır. “Sass” aynı zamanda boşluklara duyarlıdır, yani doğru girinti yapmanız gerekir. ‘SCSS’de, kurallarınızı istediğiniz gibi biçimlendirebilir ve girintileyebilirsiniz.

SCSS: #

// nesting in SCSS
.parent {
  margin-top: 1rem;

  .child {
    float: left;
    background: blue;
  }
}

SASS: #

// nesting in Sass
.parent
  margin-top: 1rem

  .child
    float: left
    background: blue

Derlemeden sonra her ikisi de aynı CSS’yi üretecektir:

.parent {
  margin-top: 1rem;
}
.parent .child {
  float: left;
  background: blue;
}

Haritalar #

Haritalar söz konusu olduğunda, genellikle SCSS daha kolay sözdizimidir. “Sass” girinti tabanlı olduğundan, haritalarınızın tek satırda kaydedilmesi gerekir.

// in Sass maps are "unreadable"
$white: ( white-50: rgba(255, 255, 255, .1), white-100: rgba(255, 255, 255, .2), white-200: rgba(255, 255, 255, .3), white-300: rgba(255, 255, 255, .4), white-400: rgba(255, 255, 255, .5), white-500: rgba(255, 255, 255, .6), white-600: rgba(255, 255, 255, .7), white-700: rgba(255, 255, 255, .8), white-800: rgba(255, 255, 255, .9), white-900: rgba(255, 255, 255, 1 )

SCSS ile kodunuzu birden çok satırda formatlayabildiğiniz için haritalarınızı daha okunaklı olacak şekilde formatlayabilirsiniz.

// in SCSS maps are more readable
$white: (
  white-50: rgba(255, 255, 255, .1),
  white-100: rgba(255, 255, 255, .2),
  white-200: rgba(255, 255, 255, .3),
  white-300: rgba(255, 255, 255, .4),
  white-400: rgba(255, 255, 255, .5),
  white-500: rgba(255, 255, 255, .6),
  white-600: rgba(255, 255, 255, .7),
  white-700: rgba(255, 255, 255, .8),
  white-800: rgba(255, 255, 255, .9),
  white-900: rgba(255, 255, 255, 1)
);

Yorumlar #

Sass vs. Scss’deki yorumlar, çoklu satırlar söz konusu olduğunda büyük ölçüde benzerdir. “SASS” çoklu satırları girintiye duyarlıdır, “SCSS” ise yorum sonlandırıcılara dayanır.

Tek Satırlı Yorum #

stil.scss

// Just this line will be commented!
h1 { color: red; }

stil.sass

// Exactly the same as the SCSS Syntax!
h1
  color: red

Çok Satırlı Yorum #

stil.scss

Başlatıcı: /*

Sonlandırıcı: */

/* This comment takes up
 * two lines.
 */
h1 {
   color: red;
}

Bu, “h1” öğelerini kırmızı rengiyle biçimlendirir.

stil.sass

Şimdi, “SASS"ın iki başlatıcısı var, ancak ilgili sonlandırıcısı yok. “SASS” içindeki çok satırlı yorumlar girinti düzeylerine duyarlıdır.

Başlatıcılar: // ve /*

// This is starts a comment,
   and will persist until you 
   return to the original indentaton level.
h1
  color: red

Bu, “h1” öğelerini kırmızı rengiyle biçimlendirir.

Aynısı /* Başlatıcı ile de yapılabilir:

/* This is starts a comment,
   and will persist until you 
   return to the original indentaton level.
h1
  color: red

İşte orada! ‘SCSS’ ve ‘SASS’ içindeki yorumlar arasındaki temel farklar!

Karışımlar #

“Sass” daha “tembel” sözdizimi olma eğilimindedir. Hiçbir şey bunu, karışımları nasıl tanımladığınızdan ve dahil ettiğinizden daha güzel gösteremez.

Bir karışım tanımlama #

=, Sass içinde bir mixin, SCSSde @mixin nasıl tanımladığınızdır.

// SCSS
@mixin size($x: 10rem, $y: 20rem) {
  width: $x;
  height: $y;
}
  
// Sass
=size($x: 10rem, $y: 20rem)
  width: $x
  height: $y

Karışım dahil #

“+”, “Sass"a, “@include"yi “SCSS"ye nasıl eklediğinizdir.

// SCSS
.element {
  @include size(20rem);
}

// Sass
.element
  +size(20rem)

SCSS ve SASS karşılaştırması #

  • ‘SCSS’ sözdizimi daha çok ‘CSS’ sözdizimine benzer, ancak ‘SASS’ sözdizimi ‘SCSS’den biraz farklıdır ancak her ikisi de tam olarak aynı ‘CSS’ kodunu üretir.
  • SASS da stil özelliklerini noktalı virgül(;) ile sonlandırmıyoruz, ancak SCSS’de stil özelliklerini (;) ile bitiriyoruz.
  • ‘SCSS’de stil özelliklerini kapatmak için ‘{}’ parantezini kullandık ama ‘SASS’da ‘parantez’ kullanmıyoruz.
  • ‘Girinti’, ‘SASS’de çok önemlidir. Öğenin “sınıf” veya “kimliği” içindeki iç içe özellikleri tanımlayacaktır.
  • scssde tek satırda birden çok değişken tanımlayabiliriz ama SASSta yapamıyoruz.

döngü sözdizimi için #

sass 3.3 ve artı sürümün piyasaya sürülmesiyle @if ve else koşulları sözdizimi aynı hale geldi. artık sadece scss ile değil aynı zamanda sass ile de ifadeler kullanabiliriz.

sass sözdizimi

@for $i from 1 through 3 {
  .item-#{$i} { width: 2em * $i; }
}

Derlenmiş

.item-1 {
  width: 2em;
}
.item-2 {
  width: 4em;
}
.item-3 {
  width: 6em;
}

scss sözdizimi

@for $i from 1 through 3 {
  .item-#{$i} { width: 2em * $i; }
}

derlenmiş

.item-1 {
  width: 2em;
}
.item-2 {
  width: 4em;
}
.item-3 {
  width: 6em;
}
