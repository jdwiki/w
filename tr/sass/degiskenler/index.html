Sözdizimi #

  • $değişken_adı: değer;

SCSS #

Tıpkı Sass’ta olduğu gibi, SCSS değişkenleri, bir SCSS belgesi boyunca birden çok kez kullanılacak bir değeri depolamak için kullanılır.

Değişkenler çoğunlukla sık kullanılan özellik değerlerini (yazı tipleri ve renkler gibi) depolamak için kullanılır, ancak herhangi bir özelliğin herhangi bir değeri için kullanılabilir.

SCSS, bir değişken bildirmek için $ sembolünü kullanır.

$font-stack: Helvetica, sans-serif;
$primary-color: #000000;

body {
  font-family: $font-stack;
  color: $primary-color;
}

Bu değişkene henüz atanmamışsa yeni bir değer atamak istiyorsanız, bir değişken bildirirken !default kullanabilirsiniz:

$primary-color: blue;
$primary-color: red !default; // $primary-color is still "blue"
$primary-color: green;        // And now it's green.

küstah #

Değişkenler, bir Sass belgesi boyunca birden çok kez kullanılacak bir değeri bir kez depolamak için kullanılır.

Çoğunlukla yazı tipleri ve renkler gibi şeyleri kontrol etmek için kullanılırlar ancak herhangi bir özelliğin herhangi bir değeri için kullanılabilirler.

Sass, bir şeyi değişken yapmak için $ sembolünü kullanır.

$font-stack: Helvetica, sans-serif
$primary-color: #000000

body
  font-family: $font-stack
  color: $primary-color

Değişken Kapsam #

Değişkenler, JavaScript’te olduğu gibi belirli bir kapsamda bulunur.

Bir bloğun dışında bir değişken bildirirseniz, sayfa boyunca kullanılabilir.

$blue: dodgerblue;

.main {
    background: $blue;

    p {
        background: #ffffff;
        color: $blue;
    }
}

.header {
    color: $blue;
}

Bir blok içinde bir değişken bildirirseniz, sadece o blokta kullanılabilir.

.main {
    $blue: dodgerblue;

    background: $blue;

    p {
        background: #ffffff;
        color: $blue;
    }
}

.header {
    color: $blue; // throws a variable not defined error in SASS compiler
}

Sayfa düzeyinde (bir bloğun dışında) bildirilen değişkenler, eğer içe aktarıldılar ise diğer sayfalarda da kullanılabilir.

@at-root yönergesiyle Değişkenleri Yerelleştirin #

@at-root yönergesi, değişkenleri yerelleştirmek için kullanılabilir.

$color: blue;

@at-root {
  $color: red;

  .a {
    color: $color;
  }
  .b {
    color: $color;
  }
}

.c {
  color: $color;
}

derlenir:

.a {
  color: red;
}

.b {
  color: red;
}

.c {
  color: blue;
}

İnterpolasyon #

Dize enterpolasyonunda değişkenler kullanılabilir. Bu, dinamik olarak seçiciler, özellikler ve değerler oluşturmanıza olanak tanır. Ve bunu yapmak için bir değişken sözdizimi #{$variable} şeklindedir.

$className: widget;
$content: 'a widget';
$prop: content;

.#{$className}-class {
  #{content}: 'This is #{$content}';
}
// Compiles to

.widget-class {
  content: "This is a widget";
}

Bununla birlikte, karışımların veya işlevlerin adlarını dinamik olarak oluşturmak için kullanamazsınız.

SCSS’deki Değişkenler #

SCSS’de değişkenler $ işaretiyle başlar ve CSS özellikleri gibi ayarlanır.

$etiket rengi: #eee;

Yalnızca tanımlandıkları yuvalanmış seçicilerde bulunurlar.

#menu {
    $basic-color: #eee;
    color: $basic-color;
}

Herhangi bir iç içe seçicinin dışında tanımlanırlarsa, her yerde kullanılabilirler.

$width: 5em;

#menu {
  width: $width;
}

#sidebar {
  width: $width;
}

Ayrıca ‘!global’ bayrağıyla da tanımlanabilirler, bu durumda her yerde kullanılabilirler.

#menu {
  $width: 5em !global;
  width: $width;
}

#sidebar {
  width: $width;
}

Değişken adlarının birbirinin yerine kısa çizgi ve alt çizgi kullanabileceğini unutmamak önemlidir. Örneğin, $label-width adında bir değişken tanımlarsanız, ona $label_width olarak erişebilir veya bunun tersini yapabilirsiniz.
