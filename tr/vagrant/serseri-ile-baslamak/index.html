LAMBA projesi #

Bu örnekte Vagrant ile özel bir LAMP proje geliştirme ortamı oluşturulmuştur.

Öncelikle Virtual Box ve Vagrant yüklemeniz gerekecek.

Yukarıdaki dosyayı düzenleyerek sonraki satırları ekleyin veya yorumunu kaldırın:

config.vm.box = "hashicorp/precise64"
config.vm.network :forwarded_port, guest: 80, host: 8080
config.vm.network :private_network, ip: "192.168.33.10"
config.ssh.forward_agent = true
if Vagrant::Util::Platform.windows?
  config.vm.synced_folder "./", "/vagrant"
else
  config.vm.synced_folder "./", "/vagrant", type: "nfs"
end
# https://stefanwrobel.com/how-to-make-vagrant-performance-not-suck
config.vm.provider "virtualbox" do |v|
  host = RbConfig::CONFIG['host_os']

  # Give VM 1/4 system memory & access to all cpu cores on the host
  if host =~ /darwin/
    cpus = `sysctl -n hw.ncpu`.to_i
    # sysctl returns Bytes and we need to convert to MB
    mem = `sysctl -n hw.memsize`.to_i / 1024 / 1024 / 4
  elsif host =~ /linux/
    cpus = `nproc`.to_i
    # meminfo shows KB and we need to convert to MB
    mem = `grep 'MemTotal' /proc/meminfo | sed -e 's/MemTotal://' -e 's/ kB//'`.to_i / 1024 / 4
  else # sorry Windows folks, I can't help you
    cpus = 2
    mem = 1024
  end

  v.customize ["modifyvm", :id, "--memory", mem]
  v.customize ["modifyvm", :id, "--cpus", cpus]
end

İstediğiniz etki alanını serseri VM’ye yönlendirmek için “hosts” dosyanızı düzenleyin. Linux için /etc/hosts, Windows için C:\Windows\System32\Drivers\etc\hosts; ve bu satırı ekleyin:

192.168.33.10 vagrantServer.com

Elbette, vagrantServer.comu herhangi bir adla değiştirebilirsiniz.

# Add bitbucket and github to known hosts
touch /root/.ssh/known_hosts

# Source: https://gist.github.com/winhamwr/7122947
# Sleep until we can successfully SSH into Bitbucket.
# Uses doublinng backoff while waiting 
# with_backoff() adapted from http://stackoverflow.com/a/8351489
# Retries a command a configurable number of times with backoff.
#
# The retry count is given by ATTEMPTS (default 5), the initial backoff
# timeout is given by TIMEOUT in seconds (default 1.)
#
# Successive backoffs double the timeout.
#generatedKey="`cat /root/.ssh/id_rsa.pub`"
echo -n "Generate a SSH key (https://help.github.com/articles/generating-ssh-keys/) 
and add it to your Bitbucket account (Profile -> SHH keys) to continue. "

with_backoff() {
  local max_attempts=${ATTEMPTS-5}
  local timeout=${TIMEOUT-1}
  local attempt=0
  local exitCode=0
 
  while [ $attempt -lt $max_attempts ]
  do
    set +e
    "$@"
    exitCode=$?
    set -e
 
    if [ $exitCode -eq 0 ]
    then
      break
    fi
 
    echo "Failure! Retrying in $timeout.." 1>&2
    sleep $timeout
    attempt=$(( attempt + 1 ))
    timeout=$(( timeout * 2 ))
  done
 
  if [ $exitCode -ne 0 ]
  then
    echo "You've failed me for the last time! ($@)" 1>&2
  fi
 
  return $exitCode
}
 
ATTEMPTS=${ATTEMPTS:-5}
 
export ATTEMPTS
with_backoff ssh -T git@bitbucket.org;

# Clone repositories (replace "yourProjectName" and "yourProjectRepository" with your project data)
cd /var/www
rm -rf yourProjectName/
git clone yourProjectRepository

# Add server names to /etc/hosts (replace "vagrantServer.com" with the domain set above)
echo -e '\n127.0.0.1       vagrantServer.com' >> /etc/hosts

# Enable apache modes
a2enmod rewrite

# Copy sites-available file (you need to add the Apache configuration file for the desired domain in `config/sites-available`. Replace "vagrantServer.conf" with the desired name)
cp /vagrant/config/sites-available/vagrantServer.conf /etc/apache2/sites-available/

# Remove html from document root
sed -i 's/\/var\/www\/html/\/var\/www/g' /etc/apache2/sites-available/*
service apache2 restart

# Enable sites (replace "vagrantServer.conf" with the above file name)
a2ensite vagrantServer.conf

# Install ruby, compass and sass (Optional)
apt-get install -y rubygems
gem install compass
npm install -g sass

# Pull the repo
cd /var/www/yourProjectName

Yukarıdaki dosyayı kontrol edip kaydettikten sonra, tekrar terminalinize gidin, mevcut dizini daha önce oluşturduğunuz ‘vagrant’ ile değiştirin ve ‘vagrant up’ yazın. VM oluşturulacak ve boostrap dosyası VM’den yürütülecek, böylece gerekli tüm şeyler kopyalanacak/kurulacak. Bitirdikten sonra, tarayıcınızı açıp vagrantServer.com’a (veya ona verdiğiniz ad ne olursa olsun) gidebilirsiniz ve serseri VM’den sunulan içeriği görmelisiniz.

Ayrıca proje dosyalarınızı ‘vagrant/yourProjectName’ dizini aracılığıyla da düzenleyebileceksiniz ve ‘vagrant’ dizini içindeki tüm dosyalar sunucunuz ve serseri VM arasında paylaşılacak ve senkronize edilecektir.

Vagrant Box görüntüsünü Amazon AWS AMI’ye yükleyin #

Amazon AWS’ye yüklemek istediğiniz yerel bir serseri kutunuz var. Öncelikle, bir .box dosyası oluşturmanız gerekir:

vagrant package --base my-virtual-machine

This step should take a while depending on the size of your image. Then, you need to get the .vmdk image from the .box file:

gunzip -S .box paketi.box
katran xf paketi

After this step, you should have 4 new files: package, box-disk1.vmdk, Vagrantfile, and box.ovf. Now, to upload to AWS. Assuming you have a AWS account , create an S3 bucket to store the image on Amazon’s servers. You’re going to need Amazon’s EC2 CLI for the next step (as you can’t do this through the console as far as I can tell):

ec2-import-instance box-disk1_1.vmdk -f VMDK -t t2.micro -a x86_64 -b <S3-bucket-name> -o $AWS_ACCESS_KEY -w $AWS_SECRET_KEY -p Linux

Bu komutun bu sonucu biraz zaman alacaktır - büyük resim dosyasını S3’e yüklüyor, ancak komutun kendisi daha hızlı dönüyor. ’ec2-describe-conversion-tasks’ komutunu kullanarak içe aktarmanın ilerlemesini kontrol edebilirsiniz.

Bu bittiğinde, kutunuzun bir örneğinin AWS konsolunda çalıştığını göreceksiniz. Ancak, Genel IP adresine sahip olmadığı ve/veya kendisiyle ilişkilendirilmiş bir .pem dosyası olmadığı için ona erişemeyebilirsiniz. Dolayısıyla bir sonraki adım, örnekten bir AMI oluşturmaktır. Bir AMI oluşturmak için örneği durdurun (sonlandırmayın!) ve örneği sağ tıklayın ve ‘Görüntü’->‘Görüntü Oluştur’ seçeneğine gidin. Bu da biraz zaman almalıdır. Konsolun AMI görünümünde ilerlemeyi kontrol edebilirsiniz. Tamamlandığında, AMI’yi kullanarak bir “.pem” anahtar dosyası ekleyerek bir örneği başlatın ve ardından “ssh” yapabilirsiniz ve hazırsınız.

VirtualBox ve SSH desteği ile Windows için kurulum #

Vagrant’ı Windows platformunda kullanmak için önce sanallaştırma yazılımı ve bir ssh komut satırı aracı yüklemeniz gerekir. Bu örnek ücretsiz VirtualBox ve Cygwin kullanacaktır.

VirtualBox’ı yükleyin #

Resmi indirme sayfasından en son VirtualBox sürümünü indirin ve indirilen dosyayı çalıştırın. Kurulum sırasında ağ bağlantısını geçici olarak kaybedeceğinizi unutmayın.

Ayrıca Oracle’ın ek sürücüler yüklemesine izin vermelisiniz.

Cygwin’i yükleyin #

cygwin.com adresinden alın ve “Paketleri seçin” sayfasını alana kadar kurulumu çalıştırın.

Buradan sadece ssh bin-s’ye ihtiyacımız var:

PATH’e Cygwin ekleyin #

Ayrıca Windows PATH ortam değişkenine C:\cygwin64\bin klasörünü de eklemeniz gerekir. Böylece her yerden ssh komutunu çağırabilirsiniz.

Vagrant’ın kendisini yükleyin #

Vagrant’ı vagrantup.com adresinden indirin ve kurmak için kurulum kılavuzunu takip edin. Bundan sonra bilgisayarınızı yeniden başlatmanız gerekir.

Test yapmak #

Sanallaştırma teknolojisi VTx/VTd etkinleştirilmelidir. (Bunu BIOS’ta güvenlik seçeneklerinde bulabilirsiniz)

Her şeyin doğru şekilde yüklenip yüklenmediğini test etmek için sabit sürücüde bir yerde boş bir klasör oluşturup oluşturmadığını test etmek için komut satırını açın ve az önce oluşturduğunuz klasöre gidin:

cd c:/path/to/your/folder

sonra girin

vagrant init hashicorp/precise64
vagrant up

Bu, Ubuntu 12.04 LTS 64-bit çalıştıran VirtualBox’ı oluşturacak ve başlatacaktır. 32-bit sürümünü başlatmak için vagrant init hashicorp/precise32 kullanın. Başka bir kutuya ihtiyacınız varsa daha fazlasını hashicorp’un web sitesinde bulabilirsiniz.

Ayrıca “vagrant init” komutu, geçerli klasörde bir “Vagrantfile” yapılandırma dosyası oluşturacaktır. Artık onu başka birine gönderebilirsiniz ve o kişi ‘vagrant up’ı aradığında aynı sanal makine onun bilgisayarında yaratılacaktır.

Bu iki komut başarıyla çalıştırıldıktan sonra ssh’yi test etmek için bu komutu aynı klasörde çalıştırın:

vagrant ssh

Her şey yolunda giderse, sanal makinenin içinde ‘serseri’ kullanıcı olarak oturum açmış olacaksınız.

Sonraki adımlar #

ile VM’yi durdurabilirsiniz.

vagrant halt

veya ile silin

vagrant destroy

Daha fazla kutu ve bunların nasıl kurulacağına ilişkin talimatlar vagrantbox.es sayfasında bulunabilir.

Sadece birkaç dakikada sanal bir linux’a sahip olmanın en kolay yolu (3 adımda) #

Aşama 1. #

Ana makinenizde (Windows/Linux/OSX), boş bir “my_project” dizini oluşturun.

Adım 2. #

Bununla ‘Vagrantfile’ adlı bir dosya oluşturun:

Vagrant.configure("2") do |config|
  config.vm.box = "gbarbieru/xenial" #An Ubuntu 16.04 based image
  config.vm.hostname = "my_project"
  config.vm.network :private_network, ip: "172.16.123.10"
end

Aşama 3. #

Makinenizi çalıştırın:

host$ vagrant up
host$ vagrant ssh
virtual$ cd /vagrant

Tamamlandı!

Notlar:

  • Belki başka bir IP adresi atamak istiyorsunuz.
  • Eğer sunucunuz Windows ise, belki ‘putty’den ‘ssh’ yapmak istersiniz. Bunu, 127.0.0.1 ana bilgisayar adına ssh-ing yaparak yapabilirsiniz. ve “2222” bağlantı noktası. Kullanıcı adı ‘vagrant’ ve şifre ‘vagrant’.

tüm klasörleri senkronize etme #

Tüm klasörleri her iki yönde de senkronize etmek için bunu ‘Vagrantfile’ dosyanıza ekleyin.

config.vm.synced_folder "my-project1", "/home/vagrant/my-project1"

klasörleri senkronize ediyor ancak bazı klasörleri hariç tutuyor #

Tüm klasörleri her iki yönde de senkronize etmek için bunu Vagrantfile’nize ekleyin:

config.vm.synced_folder "my-project1", "/home/vagrant/my-project1", type: "rsync",
 :rsync__exclude => ['my-project1/mini_project2/target,my-project1/mini_project2/target,my-project1/mini_project3/target']

Tüm hedef klasörler senkronizasyondan hariç tutulur. Senkronizasyon yalnızca “vagrant up” ve “vagrant yeniden yükleme” sırasında gerçekleşir. Ev sahibinizden misafire her değişikliği senkronize etmek için şunları kullanmanız gerekir:

vagrant rsync-auto
