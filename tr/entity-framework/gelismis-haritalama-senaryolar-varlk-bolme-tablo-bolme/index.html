Gelişmiş haritalama senaryoları varlık bölme, tablo bölme

  • Varlık bölme
  • Masa bölme

Bu sayfada

  • Varlık bölme
  • Masa bölme

Varlık bölmeyi veya tablo bölmeyi desteklemek için EF modelinizi nasıl yapılandırabilirsiniz.

Varlık bölme #

Diyelim ki şöyle bir varlık sınıfınız var:

public class Person  
{
   public int PersonId { get; set; }
   public string Name { get; set; }
   public string ZipCode { get; set; }
   public string City { get; set; }
   public string AddressLine { get; set; }
}

Ardından, bu Kişi varlığını iki tabloya eşlemek istediğinizi varsayalım — Biri Kişi Kimliği ve Ad ile, diğeri ise adres ayrıntılarıyla. Elbette, adresin hangi kişiye ait olduğunu belirlemek için burada da Kişi Kimliğine ihtiyacınız olacaktır. Yani temelde istediğiniz, varlığı iki (veya daha fazla) parçaya bölmek. Bu nedenle isim, varlık bölme. Bunu, özelliklerin her birini farklı bir tabloya eşleyerek yapabilirsiniz:

public class MyDemoContext : DbContext  
{       
  public DbSet<Person> Products { get; set; }

  protected override void OnModelCreating(DbModelBuilder modelBuilder)
  {
      modelBuilder.Entity<Person>().Map(m =>
      {
         m.Properties(t => new { t.PersonId, t.Name });
         m.ToTable("People");
      }).Map(m => 
      {   
        m.Properties(t => new { t.PersonId, t.AddressLine, t.City, t.ZipCode });
        m.ToTable("PersonDetails");
      });
  }
}

Bu iki tablo oluşturacaktır: People ve PersonDetails. Kişi, PersonId ve Name olmak üzere iki alana sahiptir, PersonDetails, PersonId, AddressLine, City ve ZipCode olmak üzere dört sütuna sahiptir. Kişilerde, PersonId birincil anahtardır. PersonDetails’de birincil anahtar aynı zamanda PersonId’dir, ancak aynı zamanda Kişi tablosunda PersonId’e başvuran yabancı bir anahtardır.

People DbSet’i sorgularsanız, EF, varlıkları doldurmak için her iki tablodan verileri almak için PersonIds üzerinde bir birleştirme yapar.

Sütunların adını da değiştirebilirsiniz:

protected override void OnModelCreating(DbModelBuilder modelBuilder)  
{
  modelBuilder.Entity<Person>().Map(m =>
  {
    m.Properties(t => new { t.PersonId });
    m.Property(t => t.Name).HasColumnName("PersonName");
    m.ToTable("People");
  }).Map(m =>
  {
    m.Property(t => t.PersonId).HasColumnName("ProprietorId");
    m.Properties(t => new { t.AddressLine, t.City, t.ZipCode });
    m.ToTable("PersonDetails");
  });
}

Bu aynı tablo yapısını oluşturacaktır, ancak Kişiler tablosunda Ad sütunu yerine bir KişiAdı sütunu olacak ve KişiAyrıntıları tablosunda KişiId sütunu yerine bir ProprietorId olacaktır.

Masa bölme #

Ve şimdi, varlık bölmenin tersini yapmak istediğinizi varsayalım: bir varlığı iki tabloya eşlemek yerine, bir tabloyu iki varlığa eşlemek istiyorsunuz. Buna tablo bölme denir. Diyelim ki beş sütunlu bir tablonuz var: PersonId, Name, AddressLine, City, ZipCode, burada PersonId birincil anahtardır. Ve sonra şöyle bir EF modeli oluşturmak istersiniz:

public class Person  
{
  public int PersonId { get; set; }
  public string Name { get; set; }
  public Address Address { get; set; }
}

public class Address  
{        
  public string ZipCode { get; set; }
  public string City { get; set; }
  public string AddressLine { get; set; }
  public int PersonId { get; set; }
  public Person Person { get; set; }
}

Bir şey hemen atlıyor: Adreste AddressId yok. Bunun nedeni, iki varlığın aynı tabloya eşlenmesidir, bu nedenle aynı birincil anahtara sahip olmaları gerekir. Tablo bölme yaparsanız, bu sadece uğraşmanız gereken bir şeydir. Bu nedenle, tablo bölmenin yanı sıra, Adres varlığını da yapılandırmanız ve birincil anahtarı belirtmeniz gerekir. Ve işte nasıl:

public class MyDemoContext : DbContext  
{
  public DbSet<Person> Products { get; set; }
  public DbSet<Address> Addresses { get; set; }

  protected override void OnModelCreating(DbModelBuilder modelBuilder)
  {
     modelBuilder.Entity<Address>().HasKey(t => t.PersonId);
     modelBuilder.Entity<Person>().HasRequired(t => t.Address)
                                  .WithRequiredPrincipal(t => t.Person);

     modelBuilder.Entity<Person>().Map(m => m.ToTable("People"));
     modelBuilder.Entity<Address>().Map(m => m.ToTable("People"));
  }
}
