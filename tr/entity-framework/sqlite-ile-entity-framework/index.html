SQLite ile Entity Framework

  • Entity Framework’ü bir SQLite sağlayıcısı ile kullanmak için bir proje ayarlama
  • Gerekli Düzeltmeler
  • SQLite bağlantı dizesi ekle

Bu sayfada

  • Entity Framework’ü bir SQLite sağlayıcısı ile kullanmak için bir proje ayarlama
  • Gerekli Düzeltmeler
  • SQLite bağlantı dizesi ekle

SQLite bağımsız, sunucusuz, işlemsel bir SQL veritabanıdır. Hem ücretsiz olarak kullanılabilen bir .NET SQLite kitaplığı hem de Entity Framework SQLite sağlayıcısı kullanılarak bir .NET uygulamasında kullanılabilir. Bu konu, Entity Framework SQLite sağlayıcısının kurulumuna ve kullanımına girecektir.

Entity Framework’ü bir SQLite sağlayıcısı ile kullanmak için bir proje ayarlama #

Entity Framework kitaplığı yalnızca bir SQL Server sağlayıcısıyla birlikte gelir. SQLite kullanmak için ek bağımlılıklar ve yapılandırma gerekir. Gerekli tüm bağımlılıklar NuGet’te mevcuttur.

SQLite Yönetilen Kitaplıkları yükleyin #

Yönetilen tüm bağımlılıklar, NuGet Paket Yönetici Konsolu kullanılarak kurulabilir. Install-Package System.Data.SQLite komutunu çalıştırın.

Yukarıda gösterildiği gibi, System.Data.SQLite kurulurken, ilgili tüm yönetilen kütüphaneler onunla birlikte kurulur. Buna SQLite için EF sağlayıcısı olan System.Data.SQLite.EF6 dahildir. Proje artık SQLite sağlayıcısını kullanmak için gereken derlemelere de başvuruyor.

Yönetilmeyen Kitaplık Dahil #

SQLite tarafından yönetilen kitaplıklar, “SQLite.Interop.dll” adlı yönetilmeyen bir derlemeye bağlıdır. SQLite paketiyle indirilen paket derlemelerine dahildir ve projeyi oluşturduğunuzda bunlar otomatik olarak derleme dizininize kopyalanır. Ancak yönetilmediği için referans listenize dahil edilmeyecektir. Ancak, bu derlemenin en çok SQLite derlemelerinin çalışması için uygulama ile dağıtıldığını unutmayın.

Not: Bu derleme bit bağımlıdır, yani desteklemeyi planladığınız her bitlik (x86/x64) için belirli bir derleme eklemeniz gerekir.

Projenin App.config dosyasını düzenleme #

“app.config” dosyası, SQLite’ın Entity Framework sağlayıcısı olarak kullanılabilmesi için bazı değişiklikler gerektirecektir.

Gerekli Düzeltmeler #

Paketi kurarken, “app.config” dosyası, SQLite ve SQLite EF için gerekli girişleri içerecek şekilde otomatik olarak güncellenir. Ne yazık ki bu girişler bazı hatalar içeriyor. Düzgün çalışması için önce değiştirilmeleri gerekir.

İlk olarak, yapılandırma dosyasında DbProviderFactories öğesini bulun. “system.data” öğesinin içindedir ve aşağıdakileri içerecektir

<DbProviderFactories>
  <remove invariant="System.Data.SQLite.EF6" />
  <add name="SQLite Data Provider (Entity Framework 6)" invariant="System.Data.SQLite.EF6" description=".NET Framework Data Provider for SQLite (Entity Framework 6)" type="System.Data.SQLite.EF6.SQLiteProviderFactory, System.Data.SQLite.EF6" />
  <remove invariant="System.Data.SQLite" /><add name="SQLite Data Provider" invariant="System.Data.SQLite" description=".NET Framework Data Provider for SQLite" type="System.Data.SQLite.SQLiteFactory, System.Data.SQLite" />
</DbProviderFactories>

Bu, tek bir giriş içerecek şekilde basitleştirilebilir

<DbProviderFactories>
    <add name="SQLite Data Provider" invariant="System.Data.SQLite.EF6" description=".NET Framework Data Provider for SQLite" type="System.Data.SQLite.SQLiteFactory, System.Data.SQLite" />
</DbProviderFactories>

Bununla, EF6 SQLite sağlayıcılarının SQLite fabrikasını kullanması gerektiğini belirttik.

SQLite bağlantı dizesi ekle #

Bağlantı dizeleri, kök öğe içindeki yapılandırma dosyasına eklenebilir. SQLite veritabanına erişmek için bir bağlantı dizesi ekleyin.

<connectionStrings>
    <add name="TestContext" connectionString="data source=testdb.sqlite;initial catalog=Test;App=EntityFramework;" providerName="System.Data.SQLite.EF6"/>
</connectionStrings>

Burada dikkat edilmesi gereken önemli nokta “sağlayıcı”dır. System.Data.SQLite.EF6 olarak ayarlanmıştır. Bu, EF’ye bu bağlantı dizesini kullandığımızda SQLite kullanmak istediğimizi söyler. Belirtilen “veri kaynağı” yalnızca bir örnektir ve SQLite veritabanınızın konumuna ve adına bağlı olacaktır.

İlk SQLite DbContext’iniz #

Tüm kurulum ve yapılandırma tamamlandıktan sonra, artık SQLite veritabanınızda çalışacak bir “DbContext” kullanmaya başlayabilirsiniz.

public class TestContext : DbContext
{
    public TestContext()
        : base("name=TestContext") { }
}

name=TestContext belirterek, bağlamı oluşturmak için app.config dosyasında bulunan TestContext bağlantı dizesinin kullanılması gerektiğini belirttim. Bu bağlantı dizesi SQLite kullanacak şekilde yapılandırıldı, bu nedenle bu bağlam bir SQLite veritabanı kullanacaktır.
