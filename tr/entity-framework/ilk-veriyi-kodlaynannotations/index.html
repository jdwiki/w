Entity Framework Code-First, etki alanı sınıflarınıza ve özelliklerine uygulayabileceğiniz bir dizi DataAnnotation özniteliği sağlar. DataAnnotation öznitelikleri, varsayılan Code-First kurallarını geçersiz kılar.

  1. System.ComponentModel.DataAnnotations, sütunun geçersizliğini veya boyutunu etkileyen öznitelikleri içerir.
  2. System.ComponentModel.DataAnnotations.Schema ad alanı, veritabanının şemasını etkileyen öznitelikleri içerir.

Not: DataAnnotations size yalnızca yapılandırma seçeneklerinin bir alt kümesini sunar. Fluent API, Code-First’te bulunan eksiksiz bir yapılandırma seçenekleri seti sağlar.

[Sütun] özelliği #

public class Person
{
    public int PersonID { get; set; }
    
    [Column("NameOfPerson")]
    public string PersonName { get; set; }    
}

Entity Framework’e, özelliğin adını kullanmak yerine belirli bir sütun adını kullanmasını söyler. Ayrıca veritabanı veri türünü ve tablodaki sütunun sırasını da belirleyebilirsiniz:

[Column("NameOfPerson", TypeName = "varchar", Order = 1)]
public string PersonName { get; set; }

[Gerekli] nitelik #

Bir etki alanı sınıfının bir özelliğine uygulandığında, veritabanı bir NOT NULL sütunu oluşturacaktır.

using System.ComponentModel.DataAnnotations;

public class Person
{
    public int PersonID { get; set; }
    
    [Required]
    public string PersonName { get; set; }    
}

[MaxLength] ve [MinLength] özellikleri #

[MaxLength(int)] niteliği, bir etki alanı sınıfının bir dize veya dizi türü özelliğine uygulanabilir. Entity Framework, bir sütunun boyutunu belirtilen değere ayarlayacaktır.

using System.ComponentModel.DataAnnotations;

public class Person
{
    public int PersonID { get; set; }
    
    [MinLength(3), MaxLength(100)]
    public string PersonName { get; set; }    
}

Belirtilen sütun uzunluğuna sahip sonuç sütunu:

[MinLength(int)] özniteliği bir doğrulama özniteliğidir, veritabanı yapısını etkilemez. 3 karakterden daha kısa bir PersonName ile bir Kişi eklemeye/güncellemeye çalışırsak, bu taahhüt başarısız olur. Ele almamız gereken bir DbUpdateConcurrencyException alacağız.

using (var db = new ApplicationDbContext())
{                    
     db.Staff.Add(new Person() { PersonName = "ng" });
     try
     {
          db.SaveChanges();
     }
     catch (DbEntityValidationException ex)
     {
          //ErrorMessage = "The field PersonName must be a string or array type with a minimum length of '3'."
     }
}

Hem [MaxLength] hem de [MinLength] öznitelikleri asp.net-mvc ile doğrulama özniteliği olarak kullanılabilir.

[DatabaseGenerated] özelliği #

Veritabanının özellik için değerleri nasıl oluşturduğunu belirtir. Üç olası değer vardır:

  1. “Yok”, değerlerin veritabanı tarafından üretilmediğini belirtir.
  2. “Kimlik”, sütunun, tipik olarak tamsayı birincil anahtarları için kullanılan bir kimlik sütunu olduğunu belirtir.
  3. “Hesaplamalı”, veritabanının sütun için değer oluşturduğunu belirtir.

Değer “Yok” dışında bir şeyse, Entity Framework, özellikte yapılan değişiklikleri veritabanına geri göndermez.

Varsayılan olarak (StoreGeneratedIdentityKeyConvention temel alınarak) bir tamsayı anahtar özelliği, bir kimlik sütunu olarak değerlendirilir. Bu kuralı geçersiz kılmak ve kimlik dışı bir sütun olarak değerlendirilmeye zorlamak için, “Yok” değeriyle “DatabaseGenerated” özniteliğini kullanabilirsiniz.

using System.ComponentModel.DataAnnotations.Schema;

public class Foo
{
    [Key]
    public int Id { get; set; } // identity (auto-increment) column
}

public class Bar
{
    [Key]
    [DatabaseGenerated(DatabaseGeneratedOption.None)]
    public int Id { get; set; } // non-identity column
}

Aşağıdaki SQL, hesaplanmış bir sütun içeren bir tablo oluşturur:

CREATE TABLE [Person] (
    Name varchar(100) PRIMARY KEY,
    DateOfBirth Date NOT NULL,
    Age AS DATEDIFF(year, DateOfBirth, GETDATE())
)
GO

Yukarıdaki tablodaki kayıtları temsil edecek bir varlık oluşturmak için, ‘Computed’ değeriyle ‘DatabaseGenerated’ özniteliğini kullanmanız gerekir.

[Table("Person")]
public class Person
{
    [Key, StringLength(100)]
    public string Name { get; set; }
    public DateTime DateOfBirth { get; set; }
    [DatabaseGenerated(DatabaseGeneratedOption.Computed)]
    public int Age { get; set; }
}

[YabancıAnahtar(dize)] özelliği #

EF kuralına uymayan bir yabancı anahtar isteniyorsa, özel yabancı anahtar adını belirtir.

public class Person 
{
    public int IdAddress { get; set; }

    [ForeignKey(nameof(IdAddress))]
    public virtual Address HomeAddress { get; set; }
}

Bu, aynı varlık türüyle birden çok ilişkiniz olduğunda da kullanılabilir.

using System.ComponentModel.DataAnnotations.Schema;

public class Customer
{
    ...

    public int MailingAddressID { get; set; }
    public int BillingAddressID { get; set; }

    [ForeignKey("MailingAddressID")]
    public virtual Address MailingAddress { get; set; }
    [ForeignKey("BillingAddressID")]
    public virtual Address BillingAddress { get; set; }
}

‘ForeignKey’ öznitelikleri olmadan, EF onları karıştırabilir ve ‘MailingAddress’i getirirken ‘BillingAddressID’ değerini kullanabilir veya kendi adlandırma kurallarına göre sütun için farklı bir ad bulabilir (‘gibi ’ Address_MailingAddress_Id`) ve bunun yerine bunu kullanmayı deneyin (bunu mevcut bir veritabanıyla kullanıyorsanız bir hataya neden olur).

[TersÖzellik(dize)] özniteliği #

using System.ComponentModel.DataAnnotations.Schema;

public class Department  
{  
    ...

    public virtual ICollection<Employee> PrimaryEmployees { get; set; }  
    public virtual ICollection<Employee> SecondaryEmployees { get; set; }  
}  
  
public class Employee  
{  
    ...

    [InverseProperty("PrimaryEmployees")]  
    public virtual Department PrimaryDepartment { get; set; }  
  
    [InverseProperty("SecondaryEmployees")]  
    public virtual Department SecondaryDepartment { get; set; }  
}

InverseProperty, iki varlık arasında çoklu iki yönlü ilişkiler olduğunda iki yönlü ilişkileri tanımlamak için kullanılabilir.

Entity Framework’e diğer taraftaki özelliklerle hangi gezinme özelliklerini eşleştirmesi gerektiğini söyler.

Entity Framework, iki varlık arasında birden çok çift yönlü ilişki olduğunda diğer tarafta hangi özelliklerle hangi gezinme özelliği eşlemesini bilmiyor.

Parametresi olarak ilgili sınıftaki ilgili navigasyon özelliğinin adına ihtiyaç duyar.

Bu aynı türdeki diğer varlıklarla ilişkisi olan ve özyinelemeli bir ilişki oluşturan varlıklar için de kullanılabilir.

using System.ComponentModel.DataAnnotations;
using System.ComponentModel.DataAnnotations.Schema;

public class TreeNode
{
    [Key]
    public int ID { get; set; }
    public int ParentID { get; set; }

    ...

    [ForeignKey("ParentID")]
    public TreeNode ParentNode { get; set; }
    [InverseProperty("ParentNode")]
    public virtual ICollection<TreeNode> ChildNodes { get; set; }
}

Tablodaki yabancı anahtar için kullanılan sütunu belirtmek için “ForeignKey” özniteliğinin kullanımına da dikkat edin. İlk örnekte, “Employee” sınıfındaki iki özellik, sütun adlarını tanımlamak için “ForeignKey” özniteliğine sahip olabilirdi.

[ComplexType] özelliği #

using System.ComponentModel.DataAnnotations.Schema;

[ComplexType] 
public class BlogDetails 
{ 
    public DateTime? DateCreated { get; set; } 
 
    [MaxLength(250)] 
    public string Description { get; set; } 
}

public class Blog
{
    ...

    public BlogDetails BlogDetail { get; set; }
}

Entity Framework’te sınıfı karmaşık tür olarak işaretleyin.

Karmaşık Türler (Veya Etki Alanına Dayalı Tasarımda Değer Nesneleri) kendi başlarına izlenemezler, ancak bir varlığın parçası olarak izlenirler. Bu nedenle, örnekteki BlogDetails’in bir anahtar özelliği yoktur.

Etki alanı varlıklarını birden çok sınıf arasında tanımlarken ve bu sınıfları eksiksiz bir varlık halinde katmanlarken faydalı olabilirler.

[Anahtar] özelliği #

Anahtar, bir veritabanı tablosundaki her satırı/kaydı benzersiz şekilde tanımlayan bir tablodaki alandır.

Varsayılan İlk Kod kuralını geçersiz kılmak için bu özelliği kullanın. Bir özelliğe uygulanırsa, bu sınıf için birincil anahtar sütun olarak kullanılacaktır.

using System.ComponentModel.DataAnnotations;

public class Person
{
    [Key]
    public int PersonKey { get; set; }        // <- will be used as primary key
     
    public string PersonName { get; set; }    
}

Bileşik birincil anahtar gerekliyse, [Key] özniteliği de birden çok özelliğe eklenebilir. Bileşik anahtar içindeki sütunların sırası [Key, Column(Order = x)] biçiminde sağlanmalıdır.

using System.ComponentModel.DataAnnotations;

public class Person
{
    [Key, Column(Order = 0)]
    public int PersonKey1 { get; set; }    // <- will be used as part of the primary key

    [Key, Column(Order = 1)]
    public int PersonKey2 { get; set; }    // <- will be used as part of the primary key
     
    public string PersonName { get; set; }    
}

[Key] özniteliği olmadan, EntityFramework, sınıfın özelliğini “Id” veya “{ClassName}Id” adlı birincil anahtar olarak kullanmak için varsayılan kurala geri döner.

public class Person
{
    public int PersonID { get; set; }        // <- will be used as primary key
     
    public string PersonName { get; set; }    
}

[Aralık(min,maks)] özelliği #

Bir özellik için sayısal bir minimum ve maksimum aralığı belirtir

using System.ComponentModel.DataAnnotations;   

public partial class Enrollment
{
    public int EnrollmentID { get; set; }
   
    [Range(0, 4)]
    public Nullable<decimal> Grade { get; set; }
}

Değeri aralık dışında olan bir Not eklemeye/güncellemeye çalışırsak, bu taahhüt başarısız olur. Ele almamız gereken bir DbUpdateConcurrencyException alacağız.

using (var db = new ApplicationDbContext())
{
    db.Enrollments.Add(new Enrollment() { Grade = 1000 });

    try
    {
        db.SaveChanges();
    }
    catch (DbEntityValidationException ex)
    {
        // Validation failed for one or more entities
    }
}

Doğrulama özniteliği olarak asp.net-mvc ile de kullanılabilir.

[Eşlenmemiş] özelliği #

Code-First kuralına göre, Entity Framework, desteklenen bir veri türündeki ve hem alıcı hem de ayarlayıcıya sahip her genel özellik için bir sütun oluşturur. [NotMapped] açıklama, bir veritabanı tablosunda sütun İSTEMİYORUZ tüm özelliklere uygulanmalıdır.

Veritabanında saklamak istemeyebileceğimiz bir özellik örneği, bir öğrencinin adı ve soyadına göre tam adıdır. Bu anında hesaplanabilir ve veritabanında saklamaya gerek yoktur.

public string FullName => string.Format("{0} {1}", FirstName, LastName);

“FullName” özelliğinin yalnızca bir alıcısı vardır ve ayarlayıcısı yoktur, bu nedenle varsayılan olarak Entity Framework bunun için bir sütun DEĞİL oluşturur.

Veritabanında saklamak isteyemeyeceğimiz başka bir özellik örneği, bir öğrencinin “Ortalama Notu"dur. OrtalamaGrade’i talep üzerine almak istemiyoruz; bunun yerine başka bir yerde onu hesaplayan bir rutinimiz olabilir.

[NotMapped]
public float AverageGrade { set; get; }

“AverageGrade” [NotMapped] notu olarak işaretlenmelidir, aksi takdirde Entity Framework bunun için bir sütun oluşturur.

using System.ComponentModel.DataAnnotations.Schema;

public class Student
{
    public int Id { set; get; }

    public string FirstName { set; get; }

    public string LastName { set; get; }

    public string FullName => string.Format("{0} {1}", FirstName, LastName);

    [NotMapped]
    public float AverageGrade { set; get; }
}

Yukarıdaki Varlık için DbMigration.cs içinde göreceğiz

CreateTable(
    "dbo.Students",
     c => new
         {
             Id = c.Int(nullable: false, identity: true),
             FirstName = c.String(),
             LastName = c.String(),
         })
     .PrimaryKey(t => t.Id);

[Tablo] özelliği #

[Table("People")]
public class Person
{
    public int PersonID { get; set; }
    public string PersonName { get; set; }    
}

Entity Framework’e, bir tane oluşturmak yerine belirli bir tablo adı kullanmasını söyler (yani, “Kişi” veya “Kişiler”)

[Table] niteliğini kullanarak tablo için bir şema da belirleyebiliriz.

[Table("People", Schema = "domain")]

[Dizin] özelliği #

public class Person
{
    public int PersonID { get; set; }
    public string PersonName { get; set; }

    [Index]
    public int Age { get; set; }
}

Bir sütun veya sütun kümesi için bir veritabanı dizini oluşturur.

[Index("IX_Person_Age")]
public int Age { get; set; }

Bu, belirli bir ada sahip bir dizin oluşturur.

[Index(IsUnique = true)]
public int Age { get; set; }

Bu benzersiz bir dizin oluşturur.

[Index("IX_Person_NameAndAge", 1)]
public int Age { get; set; }

[Index("IX_Person_NameAndAge", 2)]
public string PersonName { get; set; }

Bu, 2 sütun kullanarak bir bileşik dizin oluşturur. Bunu yapmak için aynı dizin adını belirtmeli ve bir sütun sırası sağlamalısınız.

Not: Endeks özniteliği Entity Framework 6.1’de tanıtıldı. Daha eski bir sürüm kullanıyorsanız, bu bölümdeki bilgiler geçerli değildir.

[StringLength(int)] özelliği #

using System.ComponentModel.DataAnnotations;

public class Post
{
    public int Id { get; set; }
    
    [StringLength(100)]
    public string Title { get; set;}

    [StringLength(300)]
    public string Abstract { get; set; }
    
    public string Description { get; set; }
}

Bir dize alanı için maksimum uzunluk tanımlar.

Not: Asp.net-mvc ile doğrulama özelliği olarak da kullanılabilir.

[Zaman damgası] özelliği #

[TimeStamp] niteliği, belirli bir Entity sınıfında yalnızca bir bayt dizisi özelliğine uygulanabilir. Entity Framework, bu özellik için veritabanı tablosunda null olmayan bir zaman damgası sütunu oluşturur. Entity Framework, eşzamanlılık kontrolünde bu Zaman Damgası sütununu otomatik olarak kullanır.

using System.ComponentModel.DataAnnotations.Schema;

public class Student
{
    public int Id { set; get; }

    public string FirstName { set; get; }

    public string LastName { set; get; }    

    [Timestamp]
    public byte[] RowVersion { get; set; }
}

[ConcurrencyCheck] Özelliği #

Bu öznitelik, sınıf özelliğine uygulanır. Eşzamanlılık denetimi için ayrı bir zaman damgası sütunu değil, eşzamanlılık denetimi için mevcut sütunları kullanmak istediğinizde ConcurrencyCheck özniteliğini kullanabilirsiniz.

using System.ComponentModel.DataAnnotations;

public class Author
{
    public int AuthorId { get; set; }
     
    [ConcurrencyCheck]
    public string AuthorName { get; set; }
}

Yukarıdaki örnekte, Yazar sınıfının AuthorName özelliğine ConcurrencyCheck özniteliği uygulanır. Bu nedenle, Code-First, iyimser eşzamanlılığı kontrol etmek için güncelleme komutunda (burada yan tümce) YazarAdı sütununu içerecektir.
