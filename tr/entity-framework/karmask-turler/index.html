Kod İlk Karmaşık Türler #

Karmaşık bir tür, bir veritabanı tablosunun seçili alanlarını ana türün alt öğesi olan tek bir türe eşlemenize olanak tanır.

[ComplexType]
public class Address
{
    public string Street { get; set; }
    public string Street_2 { get; set; }
    public string City { get; set; }
    public string State { get; set; }
    public string ZipCode { get; set; }
}

Bu karmaşık tür daha sonra birden çok varlık türünde kullanılabilir. Hatta aynı varlık türünde birden fazla kez kullanılabilir.

public class Customer
{
    public int Id { get; set; }
    public string Name { get; set; }
    ...
    public Address ShippingAddress { get; set; }
    public Address BillingAddress { get; set; }
}

Bu varlık türü daha sonra veritabanında buna benzer bir tabloda depolanır.

Elbette, bu durumda 1:n ilişkisi (Müşteri-Adres) tercih edilen model olacaktır, ancak örnek, karmaşık türlerin nasıl kullanılabileceğini göstermektedir.
