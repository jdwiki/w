ANTLR v4'e Giriş

  • ANTLR’yi Visual Studio 2015’te yükleme (Nuget kullanarak)
  • Komut Satırı Kullanımı için Yükleme
  • Yapı Otomasyonu araçlarını kullanarak yükleme
  • Eclipse’de kurun ve Hello World oluşturun

Bu sayfada

  • ANTLR’yi Visual Studio 2015’te yükleme (Nuget kullanarak)
  • Komut Satırı Kullanımı için Yükleme
  • Yapı Otomasyonu araçlarını kullanarak yükleme
  • Eclipse’de kurun ve Hello World oluşturun

ANTLR v4, yeni programlama dilleri oluşturmak ve yapılandırılmış metin veya ikili dosyaları işlemek/çevirmek için kullanılan güçlü bir araçtır. ANTLR, bir ayrıştırma ağacı (veya soyut sözdizimi ağacı, AST) oluşturabilen ve arasında dolaşabilen bir ayrıştırıcı oluşturmak için oluşturduğunuz bir dilbilgisini kullanır. Ayrıştırıcı, belirttiğiniz bir hedef dildeki çıktı dosyalarından oluşur. ANTLR v4, Java, C#, JavaScript, Python2 ve Python3 dahil olmak üzere çeşitli hedefleri destekler. C++ desteği üzerinde çalışılıyor. GUI IDE’lerinde çalışmak için Visual Studio, Intellij, NetBeans ve Eclipse için eklentiler vardır.

Genel bilgi için ANTLR web sitesini ziyaret edin. ANTLR hakkında ciddileşmek için, Terrence Parr (ANTLR’yi yaratan adam) tarafından yazılan şiddetle tavsiye edilen kitaba göz atın Kesin ANTLR 4 Referansı.

Önemli Sürüm Bilgisi

  • 4.5: 22/01/15 - JavaScript hedefi eklendi ve C# hedefi yükseltildi. 4.5 Sürüm Notları
  • 4.4: 07/16/14 - Hedef olarak Python2 ve Python3 eklendi. 4.4 Sürüm Notları
  • 4.3: 18/06/14 - Önemli hata düzeltmeleri; yeni hedefler eklemek için hazırlanmıştır. 4.3 Sürüm Notları
  • 4.2: 02/04/14 - Ayrıştırma ağaçlarını seçmek/eşleştirmek için iyileştirilmiş sözdizimi. 4.2 Sürüm Notları
  • 4.1: 30/06/13 - İyileştirilmiş ayrıştırma performansı; AST’leri PNG’ye aktarın. 4.1 Sürüm Notları
  • 4.0: 21/01/13 - İlk sürüm.

ANTLR’yi Visual Studio 2015’te yükleme (Nuget kullanarak) #

  1. Visual Studio 2015’i açın, Araçlar → Uzantılar → Çevrimiçi’ye gidin ve Antlr’ı arayın. ANTLR Dil Desteği (Sam Harwell tarafından düzenlendi) uzantısını indirin ve Visual Studio’yu yeniden başlatın.
  2. Yeni Konsol Uygulama Projesi oluşturun. Çözüm → Çözüm için Nuget Paketlerini Yönet → Gözat’a (Sekme) sağ tıklayın ve Antlr4’ü arayın ve kurun.
  3. Üzerine sağ tıklayarak Projenize Yeni Bir Öğe ekleyin. Ve ANTLR4 Şablonlarını arayın.
  4. ANTLR dosyanızdan (.g4 ile biten) Dosya → Gelişmiş Kaydetme Seçenekleri’ne gidin ve Unicode (UTF-8 imzasız) - Codepage 65001’i arayın ve Tamam’a tıklayın. Bu kadar.

Her şeyin işe yarayıp yaramadığını test edin

  • Bir ANTLR 4 Combined Grammar öğesi oluşturun ve Calculator.g4 olarak adlandırın
  • Bu Github projesinden Hesap Makinesi kaynak kodunu kopyalayıp buraya yapıştırın: Hesap Makinesi, Tom Everett
  • Dilbilgisi hesaplayıcısını dilbilgisi Hesaplayıcı olarak değiştirin
  • Çözüm Gezgini’nde → Tüm Dosyaları Göster’e tıklayın.
  • Projeyi Kaydedin ve Çalıştırın (Başlatın)
  • Solution Explorer’da obj klasörü altında Ziyaretçi ve Dinleyici gibi oluşturulan cs sınıflarını görmelisiniz. Eğer durum buysa, başardınız. Artık ANTLR ile Visual Studio 2015’te çalışmaya başlayabilirsiniz.

Komut Satırı Kullanımı için Yükleme #

ANTLR bir Java Jar dosyası olarak dağıtılır buradan indirilebilir. ANTLR bir jar dosyası olarak derlendiğinden, daha sonra çalışması için Java çalışma zamanı ortamını gerektirir, eğer yoksa buradan indirilebilir.

ANTLR JAR dosyası indirildikten sonra, ANTLR’yi komut satırından herhangi bir JAR dosyasıyla aynı şekilde çalıştırabilirsiniz:

Java -jar antlr-4.5.3-complete.jar

(antlr-4.5.3-complete.jar dosyasıyla aynı dizinde çalıştığınızı varsayarsak).

Bu, şuna benzer bir çıktı vermelidir:

ANTLR Parser Generator  Version 4.5.3
 -o ___              specify output directory where all output is generated
 -lib ___            specify location of grammars, tokens files
 -atn                generate rule augmented transition network diagrams
 -encoding ___       specify grammar file encoding; e.g., euc-jp
 -message-format ___ specify output style for messages in antlr, gnu, vs2005
 -long-messages      show exception details when available for errors and warnings
 -listener           generate parse tree listener (default)
 -no-listener        don't generate parse tree listener
 -visitor            generate parse tree visitor
 -no-visitor         don't generate parse tree visitor (default)
 -package ___        specify a package/namespace for the generated code
 -depend             generate file dependencies
 -D<option>=value    set/override a grammar-level option
 -Werror             treat warnings as errors
 -XdbgST             launch StringTemplate visualizer on generated code
 -XdbgSTWait         wait for STViz to close before continuing
 -Xforce-atn         use the ATN simulator for all predictions
 -Xlog               dump lots of logging info to antlr-timestamp.log

kurulum için önerilen diğer eylemler şunlardır:

     1. Add antlr4-complete.jar to CLASSPATH, either: Permanently: 
Using System Properties dialog > Environment variables > Create or append to CLASSPATH variable Temporarily, at command line: SET CLASSPATH=.;C:\Javalib\antlr4-complete.jar;%CLASSPATH% 
     3.Create batch commands for ANTLR Tool, TestRig in dir in PATH
         antlr4.bat: java org.antlr.v4.Tool %*
         grun.bat:   java org.antlr.v4.gui.TestRig %*

Kurulumdan sonra .g4 dilbilgisi dosyanızı kullanarak bir uygulama oluşturabilirsiniz:

Java -jar antlr-4.5.3-complete.jar yourGrammar.g4

-Dlanguage parametresiyle başka dillerde de bir uygulama oluşturabilirsiniz. Örneğin, C# dosyaları oluşturmak için şöyle bir şey yaparsınız:

java -jar antlr-4.5.3-complete.jar yourGrammar.g4 -Dlanguage=CSharp

Yaygın programlama dilleri için önceden hazırlanmış gramerlerin tam listesi için buraya bakın.

Yapı Otomasyonu araçlarını kullanarak yükleme #

ANTLR’nin en son sürümünü indirin ve bir klasöre çıkarın.

Çalışma zamanına (oluşturulan gramerlerin kullandığı sınıflar) bağlı olarak Maven, Gradle veya diğer derleme araçlarını da kullanabilirsiniz: org.antlr:antlr4-runtime.

Oluşturma sürecinin bir parçası olarak, bir maven projesinde ayrıştırıcıyı otomatik olarak oluşturmak için Maven eklentisini kullanın: “org.antlr:antlr4”.

Eclipse’de kurun ve Hello World oluşturun #

(ANTLR 4.5.3, Eclipse Neon, ANTLR 4 IDE 0.3.5 ve Java 1.8 ile test edilmiştir)

  1. En son ANTLR dosyasını indirin. ANTLR Java ikili dosyaları kavanozunun tamamını aldığınızdan emin olun. Herhangi bir uygun konuma, örneğin diğer Java kitaplıklarının depolandığı klasöre kaydedin. Nerede olduğu önemli değil, sadece konumu hatırla.

  2. ANTLR IDE’yi Eclipse’e kurun.

    • From the Eclipse menu, click Help and select Eclipse Marketplace.
    • In the Find: box, type antlr and click Go.
    • Click Install for ANTLR 4 IDE.
    • Click Finish in the Confirm Selected Features window.
    • If a Security Warning window pops up, click OK.
    • Restart Eclipse.

  3. “Enjektör oluşturulamadı…” hatası için bir çözüm bulun.

    • When accessing ANTLR 4 Preferences in Eclipse or when the environment variable HOME is not set, the following error occurs: Failed to create injector for com.github.jknack.antlr-4ide.Antlr4 for com.github.jknack.antlr-4ide.Antlr4.
    • Make sure the environment variable HOME is set. If not, set it as appropriate for your system.
    • Download Xtext 2.7.3 to the same location as antlr-n.n.n-complete.jar.
    • In Eclipse, click on Help and select Install New Software.
    • Click Add… to get to the Add Repository window.
    • Type a name, xtext 2.7.3 for example, then click on Archive…, navigate to the Xtext 2.7.3 file and select it, then click OK.
    • In the Install window, click the Select All button then click Next> twice, accept the license agreement. and click Finish.
    • Restart Eclipse.

  4. Eclipse/Java’ya ANTLR’nin nerede olduğunu söyleyin.

    • In Eclipse, click on Window and select Preferences.
    • In the left pane, expand Java and Build Path, then select Classpath Variables.
    • In the right pane, click New…, enter a Name, and click File… and browse to your location of antlr-n.n.n-complete.jar. Click OK to get back to the Classpath Variables window.
    • Click OK to exit Preferences.

  5. (İsteğe bağlı) ANTLR IDE tarafından oluşturulan kaynaklar dizinini yapılandırın.

    • In the Eclipse main window, click Window then Preferences.
    • In the left pane, expand ANTLR 4 and select Tool.
    • Under Options, change the directory if desired. For example, java is my target language so I use ./antlr-java.
    • Click OK to close the Preferences window.

  6. Bir ANTLR 4 projesi oluşturun.

    • From the Eclipse main window, go to File, New, Project.
    • In the New Project window, expand General and select ANTLR 4 Project.
    • Click Next, type a project name and click Finish.
    • The default new project contains a Hello.g4 file and will automatically build the standard “Hello World” program.
    • In the Package Explorer, expand the new project folder to see the g4 file and a folder named target (or the name you gave it in Step 5) containing the target source files.
