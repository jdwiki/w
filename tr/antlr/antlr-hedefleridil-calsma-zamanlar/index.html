Dil desteği #

ANTLR, bir dizi programlama dili için ayrıştırıcılar üretebilir:

  1. C# Hedefi
  2. Python Hedefi
  3. JavaScript Hedefi
  4. Java Hedefi

Varsayılan olarak ANTLR, Java programlama dilinde komut satırından bir ayrıştırıcı oluşturur:

Java -jar antlr-4.5.3-complete.jar yourGrammar.g4 //Will output a
    java parser

Hedef dili değiştirmek için işletim sistemi terminalinden/komut satırından aşağıdaki komutu çalıştırabilirsiniz:

    antlr4 -Dlanguage=Python3 yourGrammar.g4 
//with alias
    java -jar antlr-4.5.3-complete.jar -Dlanguage=Python3 yourGrammar.g4 
//without alias

Belirli bir dil için istediğiniz ayrıştırıcıyı oluşturmak için her seferinde komut satırında/terminalde ‘-Dlanguage’ parametresini kullanmak yerine, hedefi global bölüme ekleyerek hedefi .g4 dilbilgisi dosyanızdan seçebilirsiniz:

options {
    language  = "CSharp";
}
//or
options {
    language="Python";
}

Oluşturulan ayrıştırıcı çıktısını kullanmak için, belirtilen dil için ANTLR çalışma zamanına sahip olduğunuzdan emin olun:

  1. CSharp çalışma zamanı
  2. Python 2 çalışma zamanı
  3. python 3 çalışma zamanı

ANTLR çalışma zamanı kitaplıkları hakkında tüm talimatlar ve bilgiler

Python ayrıştırıcı kurulumu #

ANTLR.jar ile gramer .g4 dosyanızı çalıştırdıktan sonra, aşağıdakiler gibi bir dizi dosya oluşturmuş olmanız gerekir:

1.yourGrammarNameListener.py
2.yourGrammarNameParser.py
3.yourGrammarName.tokens
...

Bunları bir python projesinde kullanmak için, geliştirdiğiniz herhangi bir uygulamanın ANTLR kitaplığına erişebilmesi için Python çalışma zamanını çalışma alanınıza ekleyin. Bu, çalışma zamanını mevcut proje klasörünüze çıkararak veya IDE’nizde proje bağımlılıklarınıza aktararak yapılabilir.

#main.py
import yourGrammarNameParser
import sys

#main method and entry point of application

def main(argv):
    """Main method calling a single debugger for an input script"""
    parser = yourGrammarNameParser
    parser.parse(argv)

if __name__ == '__main__':
    main(sys.argv)

Bu kurulum, ayrıştırıcınızı içerir ve parametre olarak geçirilen bir dosyanın işlenmesine izin vermek için komut satırından girdi kabul eder.

#yourGrammarNameParser.py
from yourGrammarNameLexer import yourGrammarNameLexer
from yourGrammarNameListener import yourGrammarNameListener
from yourGrammarNameParser import yourGrammarNameParser
from antlr4 import *
import sys

class yourGrammarNameParser(object):
    """
    Debugger class - accepts a single input script and processes
    all subsequent requirements
    """
def __init__(self): # this method creates the class object.
    pass
        
        
#function used to parse an input file
def parse(argv):
    if len(sys.argv) > 1:
        input = FileStream(argv[1]) #read the first argument as a filestream
        lexer = yourGrammarNameLexer(input) #call your lexer
        stream = CommonTokenStream(lexer)
        parser = yourGrammarNameParser(stream)
        tree = parser.program() #start from the parser rule, however should be changed to your entry rule for your specific grammar.
        printer = yourGrammarNameListener(tree,input)
        walker = ParseTreeWalker()
        walker.walk(printer, tree)
    else:
        print('Error : Expected a valid file')

ANTLR çalışma zamanı ile birleştirilmiş bu dosyalar ve dilbilgisi dosyanızdan oluşturulan dosyalarınız, argüman olarak tek bir dosya adını kabul edecek ve dilbilgisi kurallarınızı okuyacak ve ayrıştıracaktır.

Temel işlevselliği genişletmek için, çalışma zamanı sırasında karşılaşılan belirteçlerle ilgili olayları işlemek için varsayılan dinleyiciyi de genişletmelisiniz.
