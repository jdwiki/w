Kurulum veya Kurulum #

Tanım:

phpMyAdmin, MySQL’in Web üzerinden yönetimini gerçekleştirmeyi amaçlayan PHP ile yazılmış ücretsiz bir yazılım aracıdır. phpMyAdmin, MySQL ve MariaDB üzerinde çok çeşitli işlemleri destekler. Sık kullanılan işlemler (veritabanlarını, tabloları, sütunları, ilişkileri, dizinleri, kullanıcıları, izinleri vb. yönetme), herhangi bir SQL ifadesini doğrudan yürütme olanağına sahipken kullanıcı arabirimi aracılığıyla gerçekleştirilebilir.

Not 1: Çoğu işletim sistemi zaten bir phpMyAdmin paketi içerir ve bu paketi otomatik olarak güncel tutacaktır, ancak bu sürümler bazen biraz eskidir ve bu nedenle en son özellikler eksik olabilir.

Not 2: Bu yöntem, MySQL ile yerel bir web sunucusu kurmuş olduğunuzu varsayar.

Pencereler:

  • Üçüncü Taraf Ürünleri:

    The easiest way to get phpMyAdmin on Windows is using third party products which include phpMyAdmin together with a database and web server such as XAMPP(cross-platform).

  • Çapraz Platform üçüncü taraf ürünlerinin listesi: XAMPP, AMPPS, Zend Server Community Edition

  • Windows’a Özel üçüncü taraf ürünlerin listesi: WampServer, EasyPHP Devserver, Uniform Server, Bitnami WAMP Stack, WPN-XM Server Yığın, WTServer, UwAmp, Vertrigo

  • Manuel Kurulum phpadmin.net:

    You can select any kit from given phpmyadmin kits.

    Example:

    For all languages choose some thing like phpMyAdmin-4.6.4-all-languages.7z or for language specific choose something like phpMyAdmin-4.6.4-english.7z

    After Choosing a phpmyadmin kit extract the folder. Rename the extracted folder to phpMyAdmin it’s easy to reference. Copy the extracted folder to your local web server’s document root. Depending on how you have configured your web server, this could be the www, htdocs, html, or public_html folder. IIS kullanıyorsanız, IIS Yöneticisi’ni açın. Sol gezinme ağacında, “Varsayılan Web Sitesi” üzerine sağ tıklayın ve “Keşfet” öğesini seçin. Çıkarılan phpMyAdmin klasörünü kopyalayıp açılan konuma yapıştırın.

Linux dağıtımları

Not: phpMyAdmin çoğu Linux dağıtımında bulunur. Mümkün olduğunda dağıtım paketlerinin kullanılması önerilir. Genellikle dağıtımınıza entegrasyon sağlarlar ve dağıtımınızdan otomatik olarak güvenlik güncellemeleri alırsınız.

  • Üçüncü Taraf Ürünleri: XAMPP, Anahtar Anahtarı Linux LAMP yığını, Bitnami LAMPStack,

  • Manuel Kurulum: phpadmin.net:

Verilen phpmyadmin kitlerinden herhangi bir kiti seçebilirsiniz.

Örnek:

Tüm diller için phpMyAdmin-4.6.4-all-languages.7z gibi bir şey seçin veya dile özgü bir şey için phpMyAdmin-4.6.4-english.7z gibi bir şey seçin

Bir phpmyadmin kiti seçtikten sonra klasörü çıkarın. Çıkarılan klasörü phpMyAdmin olarak yeniden adlandırın, referans vermesi kolaydır. Çıkarılan klasörü yerel web sunucunuzun belge köküne kopyalayın. Web sunucunuzu nasıl yapılandırdığınıza bağlı olarak bu, htdocs veya public_html klasörü olabilir.

  • Debian:

    Debian’s package repositories include a phpMyAdmin package, but be aware that the configuration file is maintained in /etc/phpmyadmin and may differ in some ways from the official phpMyAdmin documentation. Specifically it does:

    1. Configuration of web server (works for Apache and lighttpd).
    2. Creating of phpMyAdmin configuration storage using dbconfig-common.
    3. Securing setup script, see Setup script on Debian, Ubuntu and derivatives.

  • Terminal: phpmyadmin’i Debian‘a kurmanın en kolay yolu apt-get’tir

      sudo apt-get install phpmyadmin

Kurulum sırasında phpMyAdmin, temel bir yapılandırmada size yol gösterecektir. Sonrasında süreç başlar şu adımları izleyin:

  1. Sunucu için Apache2‘yi seçin

  2. phpmyadmin için veritabanını dbconfig-common ile yapılandırmak isteyip istemediğiniz sorulduğunda EVET‘i seçin.

  3. İstendiğinde MySQL şifrenizi girin

  4. phpmyadmin’de oturum açmak için kullanmak istediğiniz şifreyi girin

Kurulum tamamlandıktan sonra apache yapılandırmasına phpmyadmin’i ekleyin.

sudo nano /etc/apache2/apache2.conf

Dosyaya phpmyadmin yapılandırmasını ekleyin.

Include /etc/phpmyadmin/apache.conf

apache’yi yeniden başlatın:

sudo service apache2 restart

Daha sonra ip-adresiniz/phpmyadmin adresine giderek phpmyadmin’e erişebilirsiniz.

Güvenlik:

Ne yazık ki, phpMyAdmin’in eski sürümlerinde, uzak kullanıcıların sonunda temeldeki sanal özel sunucuda kökten yararlanmalarına izin vermek de dahil olmak üzere ciddi güvenlik açıkları vardı. Bu saldırıların çoğu basit bir işlemle önlenebilir: tüm dizini Apache’nin yerel kullanıcı/parola kısıtlamalarıyla kilitlemek, bu uzak kullanıcıların phpMyAdmin’in eski sürümlerinden yararlanmaya çalışmasına bile engel olur.

.htaccess Dosyasını Kurun:

Bunu ayarlamak için .htaccess dosyasının phpmyadmin dizini içinde çalışmasına izin vererek başlayın. Bunu phpmyadmin yapılandırma dosyasında gerçekleştirebilirsiniz:

sudo nano /etc/phpmyadmin/apache.conf

Dizin bölümünün altına, “Dizin Dizini"nin altına “AllowOverride All” satırını ekleyerek bölümün şu şekilde görünmesini sağlayın:

<Directory /usr/share/phpmyadmin>
        Options FollowSymLinks
        DirectoryIndex index.php
        AllowOverride All
        [...]

.htaccess dosyasını yapılandırın

.htaccess dosyasına izin verildiğinde, phpmyadmin oturum açma sayfasına erişmek için bile oturum açması gereken yerel bir kullanıcı oluşturmaya devam edebiliriz.

phpmyadmin dizininde .htaccess sayfasını oluşturarak başlayın:

sudo nano /usr/share/phpmyadmin/.htaccess

.htaccess dosyasında kullanıcı yetkilendirmesini ayarlayarak takip edin. Aşağıdaki metni kopyalayıp yapıştırın

AuthType Basic
AuthName "Restricted Files"
AuthUserFile /path/to/passwords/.htpasswd
Require valid-user

AuthType: Bu, parolaları kontrol etmek için kullanılacak kimlik doğrulama türünü ifade eder. Şifreler HTTP üzerinden kontrol edilir ve Temel anahtar kelimesi değiştirilmemelidir.

AuthName: Bu, parola isteminde görüntülenecek metindir. Buraya her şeyi koyabilirsiniz.

AuthUserFile: Bu satır, parola dosyasına (bir sonraki adımda oluşturacağımız) sunucu yolunu belirtir. Geçerli kullanıcı gerektir: Bu satır, .htaccess dosyasına yalnızca parola dosyasında tanımlanan kullanıcıların phpMyAdmin oturum açma ekranına erişebileceğini söyler.

htpasswd dosyasını oluşturun:

Şimdi devam edeceğiz ve geçerli kullanıcı bilgilerini oluşturacağız.

Bir htpasswd dosyası oluşturarak başlayın. htpasswd komutunu kullanın ve dosyayı bir tarayıcıdan erişilemediği sürece istediğiniz bir dizine yerleştirin. Parola dosyasını istediğiniz gibi adlandırabilseniz de, genel kural dosyayı .htpasswd olarak adlandırmaktır.

sudo htpasswd -c /path/to/passwords/.htpasswd username

Bir istem, şifrenizi girmenizi ve onaylamanızı isteyecektir.

Kullanıcı adı ve şifre çifti kaydedildiğinde, şifrenin dosyada şifrelendiğini görebilirsiniz.

Apache’yi yeniden başlatarak bitirin:

sudo service apache2 restart

phpMyAdmin’e erişim:

Oturum açma sayfasına yalnızca yetkili kullanıcılar erişebileceğinden phpMyAdmin artık çok daha güvenli olacak. Oluşturduğunuz kullanıcı adı ve şifreyi girin. Giriş yaptıktan sonra MySQL kullanıcı adı ve şifresi ile phpmyadmin’e erişebilirsiniz.

  • Ubuntu:

Başlamak için, varsayılan Ubuntu depolarından phpMyAdmin’i kolayca kurabiliriz.

Bunu yerel paket dizinimizi güncelleyerek ve ardından apt paketleme sistemini kullanarak dosyaları indirip sistemimize yükleyerek yapabiliriz:

sudo apt-get update
sudo apt-get install phpmyadmin

Uyarı:

İlk istem göründüğünde, apache2 vurgulanır, ancak seçildi. Apache’yi seçmek için “SPACE” tuşuna basmazsanız, yükleyici kurulum sırasında gerekli dosyaları taşımayacaktır. “UZAY” a basın, “TAB” ve ardından Apache’yi seçmek için “ENTER”.

  1. Sunucu seçimi için apache2’yi seçin.

  2. Veritabanını kurmak için ‘dbconfig-common’ kullanıp kullanmayacağınız sorulduğunda evet’i seçin.

  3. Veritabanı yöneticinizin şifresini girmeniz istenecektir.

  4. Ardından, phpMyAdmin uygulamasının kendisi için bir parola seçmeniz ve onaylamanız istenecektir.

Yükleme işlemi aslında phpMyAdmin Apache yapılandırma dosyasını /etc/apache2/conf-enabled/ dizinine ekler ve burada otomatik olarak okunur.

Yapmamız gereken tek şey, şunu yazarak yapabileceğimiz ‘php5-mcrypt’ uzantısını açıkça etkinleştirmektir:

sudo php5enmod mcrypt

Ardından, değişikliklerinizin tanınması için Apache’yi yeniden başlatmanız gerekecek:

sudo service apache2 restart

Daha sonra ip-adresiniz/phpmyadmin adresine giderek phpmyadmin’e erişebilirsiniz.

  • Fedora:

Fedora, phpMyAdmin paketini gönderir, ancak yapılandırma dosyasının /etc/phpMyAdmin/ içinde tutulduğunu ve bazı açılardan resmi phpMyAdmin belgelerinden farklı olabileceğini unutmayın.

Terminal:

İlk olarak, basit bir en iyi uygulamayı izleyeceksiniz: yeni bir şey yüklemeden önce mevcut paketlerin listesinin güncel olduğundan emin olun.

dnf -y update

O zaman apt-get aracılığıyla kurulum için sadece bir komut çalıştırma meselesi:

dnf -y install phpmyadmin

Güvenlik:

phpMyAdmin’in güvenliğini sağlamak için belirli bir IP adresine erişimi kilitlememiz gerekir.

phpMyAdmin paketi kurulduğunda, web erişimini yapılandırmak için bir Apache Sanal Ana Bilgisayar dosyası eklenir. O dosyayı düzenleyelim:

vim /etc/httpd/conf.d/phpMyAdmin.conf

Varsayılan olarak, phpMyAdmin yapılandırması yalnızca kurulu olduğu sunucudan erişime izin verir.

Her IP adresini ip adresim nedir? içinde bulduğunuz veya phpMyAdmin’e uzaktan bağlanacak başka bir IP adresiyle değiştirin:

Require ip 127.0.0.1
Allow from 127.0.0.1
Require ip 127.0.0.1
Allow from 127.0.0.1

Apache’yi yeniden başlatın:

systemctl restart httpd

http://the_IP_of_your_server/phpmyadmin adresini ziyaret ederek phpMyAdmin’in çalıştığını doğrulayın Örneğin: http://1.2.3.4/phpmyadmin

  • Besteci:

phpMyAdmin’i Composer kullanarak yükleyebilirsiniz, ancak teknik sınırlamaları nedeniyle şu anda varsayılan Packagist deposunda mevcut değildir.

Kurulum, kendi depomuz https://www.phpmyadmin.net/packages.json eklenerek mümkündür:

composer create-project phpmyadmin/phpmyadmin --repository-url=https://www.phpmyadmin.net/packages.json --no-dev
  • Git’ten yükleme:

Mevcut phpMyAdmin kaynağını şuradan kopyalayabilirsiniz: https://github.com/phpmyadmin/phpmyadmin.git:

git clone https://github.com/phpmyadmin/phpmyadmin.git

Ek olarak, Composer kullanarak bağımlılıkları yüklemeniz gerekir:

composer update

Geliştirmeyi düşünmüyorsanız, aşağıdakileri çağırarak geliştirici araçlarının kurulumunu atlayabilirsiniz:

composer update --no-dev

Mac:

  • Üçüncü Taraf Ürünleri : MAMP, AMPPS, Bitnami MAMPStack, XAMPP

  • Manuel olarak:

phpMyAdmin dosyasını indirin, Tüm diller için phpMyAdmin-4.6.4-all-languages.tar.bz2 gibi bir şey seçin veya dile özel phpMyAdmin-4.6.4-english.tar.gz gibi bir şey seçin

Bir phpmyadmin kiti seçtikten sonra klasörü çıkarın. Çıkarılan klasörü phpMyAdmin olarak yeniden adlandırın, başvurması kolaydır. Çıkarılan klasörü yerel web sunucunuzun belge köküne kopyalayın. Web sunucunuzu nasıl yapılandırdığınıza bağlı olarak bu, htdocs veya public_html klasörü olabilir.

config klasörünü oluşturun

mkdir ~/Sites/phpmyadmin/config

İzinleri değiştir

chmod o+w ~/Sites/phpmyadmin/config

Kurulumu tarayıcıda çalıştırın

http://localhost/~username/phpmyadmin/setup/ veya http://localhost/phpmyadmin/setup/

Yeni bir localhost mysql sunucu bağlantısı oluşturmanız gerekiyor, yeni sunucuya tıklayın.

Kimlik Doğrulama sekmesine geçin ve yerel mysql root kullanıcısını ve parolayı ayarlayın. “root” kullanıcı adını ekleyin (belki zaten doldurulmuştur, MySQL root kullanıcı kurulumu için daha önce kurduğunuz şifreyi ekleyin, kaydet’e tıklayın ve önceki ekrana geri dönersiniz. (Bu, OSX Yönetici veya kök parolası değildir - MySQL kök kullanıcısıdır).

Kaydet’e tıkladığınızdan emin olun, ardından bir config.inc.php dosyası artık phpmyadmin dizininin /config dizinindedir, bu dosyayı /phpmyadmin’in kök düzeyine taşıyın ve ardından şimdi boş/ yapılandırma dizini.

Şimdi http://localhost/~username/phpmyadmin/ adresine gitmek, şimdi MySQL veritabanlarınızla etkileşime girmenize izin verecektir.

Not: localhost’u ip adresiniz ile değiştirin

Bir veritabanı nasıl oluşturulur ve veritabanı kullanıcısı için ayrıcalıklar nasıl verilir. #

  1. http://your_ip/phpmyadmin veya http://localhost/phpmyadmin URL’sine göre phpMyAdmin’e gidin
  2. Kullanıcı adı root ve root şifresini kullanarak giriş yapın.
  3. Veritabanları sekmesine tıklayın.
  4. Veritabanı adını girin, harmanlamayı seçin (varsayılan olarak bırakabilirsiniz) ve oluştur’a tıklayın.
  5. Ayrıcalıklar sekmesine tıklayın ve “Kullanıcı hesabı ekle"yi seçin.
  6. Bağlantı sadece yerelden yapılıyorsa localhost’u seçebilirsiniz. Farklı ana bilgisayarlardan bağlanmayı planlıyorsanız “herhangi bir ana bilgisayar” seçeneğini seçin.
  7. Yeni kullanıcı adı ve şifreyi girin.
  8. Bu kullanıcı için ihtiyaç duyduğunuz ayrıcalıkları seçin.
  9. Veritabanı sekmesine tıklayın ve erişim vermeniz gereken veritabanını seçin (bizim durumumuzda 4. adımda yeni oluşturulan veritabanı). Git’i tıklayın.
