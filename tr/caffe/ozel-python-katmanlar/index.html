Bu öğretici, python kullanarak Caffe için basit bir özel katman oluşturma adımlarında rehberlik edecektir. Sonunda, bazı özel katman örnekleri var. Genellikle, Caffe’de bulunmayan bir işlevi uygulamak için gereksinimlerinize göre ayarlayarak özel bir katman oluşturursunuz.

Bir python özel katmanı oluşturmak, ağınıza bir miktar ek yük getirir ve muhtemelen bir C++ özel katmanı kadar verimli değildir. Ancak bu şekilde tüm kafeyi yeni katmanınızla derlemeniz gerekmeyecek.

Parametreler #

Parametre Ayrıntılar
üst Katmanınızın üst bloblarını içeren bir dizi. top[i].data kullanılarak kendisine iletilen verilere erişin; burada i belirli bir blobun dizinidir
alt Katmanınızın alt bloblarını içeren bir dizi. bottom[i].data kullanılarak kendisine iletilen verilere erişin; burada i belirli bir blobun

- Python katmanı ile Caffe yapısı #

Caffe’nin WITH_PYTHON_LAYER seçeneği ile derlenmesi gerekiyor:

WITH_PYTHON_LAYER=1 make && make pycaffe

- Sınıf dosyasını nereye kaydetmeliyim? #

İki seçeneğiniz var (en azından benim bildiğim). Ya özel katman dosyasını caffe komutunu çalıştıracağınız klasöre kaydedebilirsiniz (muhtemelen prototxt dosyalarınızın olacağı yer). Bir başka yol, benim de favorim, tüm özel katmanlarınızı bir klasöre kaydetmek ve bu klasörü PYTHONPATH’inize eklemek.

Katman Şablonu #

import caffe

class My_Custom_Layer(caffe.Layer):
    def setup(self, bottom, top):
        pass
        
    def forward(self, bottom, top):
        pass
        
    def reshape(self, bottom, top):
        pass

    def backward(self, bottom, top):
        pass

Hatırlanması gereken çok önemli şeyler:

  • Özel katmanınız caffe.Layer‘dan devralmalıdır (bu nedenle caffe’yi içe aktarmayı unutmayın);
  • Şu dört yöntemi tanımlamanız gerekir: kurulum, ileri, yeniden şekillendirme ve geri;
  • Tüm yöntemlerin bir top ve bir alt parametreleri vardır, bunlar katmanınıza iletilen girdi ve çıktıyı depolayan bloblardır. Buna top[i].data veya bottom[i].data kullanarak erişebilirsiniz; burada i, birden fazla üst veya alt blob’unuz olması durumunda blobun dizinidir.

- Kurulum yöntemi #

Kurulum yöntemi, Caffe tüm katmanları başlatırken yürütmenin ömrü boyunca bir kez çağrılır. Parametreleri okuyacağınız, sabit boyutlu arabellekleri başlatacağınız yer burasıdır.

- Yöntemi yeniden şekillendir #

Alt blob (katman girişi) boyutuna bağlı olarak başlatma/kurulum için yeniden şekillendirme yöntemini kullanın. Ağ başlatıldığında bir kez çağrılır.

- İleri yöntemi #

Forward yöntemi, her girdi grubu için çağrılır ve mantığınızın çoğunun olacağı yerdir.

- Geriye doğru yöntem #

Geriye yöntemi, ağın geriye doğru geçişi sırasında çağrılır. Örneğin, evrişim benzeri bir katmanda, gradyanları hesaplayacağınız yer burası olacaktır. Bu isteğe bağlıdır (bir katman yalnızca ileri olabilir).

Prototxt Şablonu #

Tamam, şimdi katmanınızı tasarladınız! .prototxt dosyanızda bunu şu şekilde tanımlarsınız:

layer {
  name: "LayerName"
  type: "Python"
  top: "TopBlobName"
  bottom: "BottomBlobName"
  python_param {
    module: "My_Custom_Layer_File"
    layer: "My_Custom_Layer_Class"
    param_str: '{"param1": 1,"param2":True, "param3":"some string"}'
  }
  include{
        phase: TRAIN
  }
}

Önemli açıklamalar:

  • tür, Python olmalıdır;
  • En azından modül ve katman parametrelerine sahip bir python_param sözlüğünüz olmalıdır;
  • modül, katmanınızı uyguladığınız dosyaya atıfta bulunur (.py olmadan);
  • katman, sınıfınızın adını ifade eder;
  • param_str kullanarak parametreleri katmana iletebilirsiniz (bunlara erişim hakkında daha fazla bilgi aşağıdadır);
  • Her katmanda olduğu gibi, hangi aşamada aktif olmasını istediğinizi tanımlayabilirsiniz (geçerli aşamayı nasıl kontrol edebileceğinizi görmek için örneklere bakın);

Katmanı Ölçün #

Bu örnekte, eğitim sırasında bir ikili problem için doğruluk ve karışıklık matrisi ve test/onaylama sırasında doğruluk, yanlış pozitif oran ve yanlış negatif oranı veren bir “ölçüm” katmanı tasarlayacağız. Caffe’nin zaten bir Doğruluk katmanı olmasına rağmen, bazen F ölçüsü gibi daha fazlasını istersiniz.

Bu benim sınıf tanımımla measureLayer.py:

#Remark: This class is designed for a binary problem, where the first class would be the 'negative'
# and the second class would be 'positive'

import caffe
TRAIN = 0
TEST = 1

class Measure_Layer(caffe.Layer):
    #Setup method
    def setup(self, bottom, top):
        #We want two bottom blobs, the labels and the predictions
        if len(bottom) != 2:
            raise Exception("Wrong number of bottom blobs (prediction and label)") 

        #And some top blobs, depending on the phase
        if self.phase = TEST and len(top) != 3:
            raise Exception("Wrong number of top blobs (acc, FPR, FNR)")
        if self.phase = TRAIN and len(top) != 5:
            raise Exception("Wrong number of top blobs (acc, tp, tn, fp and fn)")
       
        #Initialize some attributes
        self.TPs = 0.0
        self.TNs = 0.0
        self.FPs = 0.0
        self.FNs = 0.0
        self.totalImgs = 0

    #Forward method
    def forward(self, bottom, top):
        #The order of these depends on the prototxt definition
        predictions = bottom[0].data
        labels = bottom[1].data

        self.totalImgs += len(labels)

        for i in range(len(labels)): #len(labels) is equal to the batch size
                pred = predictions[i]   #pred is a tuple with the normalized probability 
                                        #of a sample i.r.t. two classes
                lab = labels[i]
                
                if pred[0] > pred[1]:
                        if lab == 1.0:
                                self.FNs += 1.0
                        else:
                                self.TNs += 1.0
                else:
                        if lab == 1.0:
                                self.TPs += 1.0
                        else:
                                self.FPs += 1.0

        acc = (self.TPs + self.TNs) / self.totalImgs
        
        try: #just assuring we don't divide by 0
                fpr = self.FPs / (self.FPs + self.TNs)
        except:
                fpr = -1.0

        try: #just assuring we don't divide by 0
                fnr = self.FNs / (self.FNs + self.TPs)
        except:
                fnr = -1.0
           
       #output data to top blob
       top[0].data = acc
       if self.phase == TRAIN:
           top[1].data = self.TPs
           top[2].data = self.TNs
           top[3].data = self.FPs
           top[4].data = self.FNs
       elif self.phase == TEST:
           top[1].data = fpr
           top[2].data = fnr
           
    def reshape(self, bottom, top):
        """
        We don't need to reshape or instantiate anything that is input-size sensitive
        """
        pass

    def backward(self, bottom, top):
        """
        This layer does not back propagate
        """
        pass

Veri Katmanı #

Bu örnek, resim yolları içeren bir metin dosyası alan, bir toplu resim yükleyen ve bunları önceden işleyen özel bir veri katmanıdır. Kısa bir ipucu, Caffe zaten geniş bir veri katmanı yelpazesine sahiptir ve sadece basit bir şey istiyorsanız, muhtemelen özel bir katman en verimli yol değildir.

dataLayer.py dosyam şöyle bir şey olabilir:

import caffe

class Custom_Data_Layer(caffe.Layer):
    def setup(self, bottom, top):
        # Check top shape
        if len(top) != 2:
                raise Exception("Need to define tops (data and label)")
        
        #Check bottom shape
        if len(bottom) != 0:
            raise Exception("Do not define a bottom.")
        
        #Read parameters
        params = eval(self.param_str)
        src_file = params["src_file"]
        self.batch_size = params["batch_size"]
        self.im_shape = params["im_shape"]
        self.crop_size = params.get("crop_size", False)
        
        #Reshape top
        if self.crop_size:
            top[0].reshape(self.batch_size, 3, self.crop_size, self.crop_size)
        else:
            top[0].reshape(self.batch_size, 3, self.im_shape, self.im_shape)
            
        top[1].reshape(self.batch_size)
