Bir üretim veritabanı için yedekleme komut dosyası

  • Sözdizimi
  • Parametreler
  • saveProdDb.sh

Bu sayfada

  • Sözdizimi
  • Parametreler
  • saveProdDb.sh

Sözdizimi #

  • Komut dosyası, aşağıdaki sözdizimi ile her yürütme için bir yedekleme dizini oluşturmanıza olanak tanır: Veritabanı yedekleme dizininin adı + yürütme tarihi ve saati
  • Örnek : prodDir22-11-2016-19h55
  • Oluşturulduktan sonra, aşağıdaki sözdizimine sahip iki yedekleme dosyası oluşturur: Veritabanı adı + yürütme tarihi ve saati
  • Örnek :
  • dbprod22-11-2016-19h55.backup (döküm dosyası)
  • dbprod22-11-2016-19h55.sql (sql dosyası)
  • 22-11-2016 @ 19h55 tarihindeki bir yürütmenin sonunda şunu elde ederiz:
  • /save_bd/prodDir22-11-2016-19h55/dbprod22-11-2016-19h55.backup
  • /save_bd/prodDir22-11-2016-19h55/dbprod22-11-2016-19h55.sql

Parametreler #

parametre ayrıntılar
save_db Ana yedekleme dizini
dbProd İkincil yedekleme dizini
TARİH Belirtilen biçimde yedekleme tarihi
dbprod Kaydedilecek veritabanının adı
/opt/postgres/9.0/bin/pg_dump pg_dump ikili dosyasının yolu
-h Sunucunun üzerinde çalıştığı makinenin ana bilgisayar adını belirtir, Örnek : localhost
-p Sunucunun bağlantıları dinlediği TCP bağlantı noktasını veya yerel Unix etki alanı soket dosya uzantısını belirtir, Örnek 5432
-U Olarak bağlanılacak kullanıcı adı.
  1. HDPS veya Symantec Backup gibi bir yedekleme aracı varsa, … Yedekleme dizinini her başlatmadan önce boşaltmak gereklidir.

Yedekleme aracının dağınık olmaması için eski dosyaların yedeğinin yapılması gerekiyor.

Bu özelliği etkinleştirmek için lütfen 3 numaralı satırın yorumunu kaldırın.

rm -R / save_db / *
  1. Bütçenin bir yedekleme aracına sahip olmasına izin vermediği durumlarda, her zaman görev planlayıcı (cron komutu) kullanılabilir.

Mevcut kullanıcı için cron tablosunu düzenlemek için aşağıdaki komut kullanılır.

crontab -e

Komut dosyasının lansmanını takvimle saat 11’de planlayın.

0 23 * * * /saveProdDb.sh

saveProdDb.sh #

Genel olarak, DB’yi pgAdmin istemcisi ile yedekleme eğilimindeyiz. Aşağıdaki, veritabanını (linux altında) iki biçimde kaydetmek için kullanılan bir sh betiğidir:

  • SQL dosyası: PostgreSQL’in herhangi bir sürümündeki verilerin olası bir özgeçmişi için.

  • Döküm dosyası: mevcut sürümden daha yüksek bir sürüm için.

    #!/bin/sh cd /save_db #rm -R /save_db/* DATE=$(date +%d-%m-%Y-%Hh%M) echo -e “Sauvegarde de la base du ${DATE}” mkdir prodDir${DATE} cd prodDir${DATE}

    #dump file /opt/postgres/9.0/bin/pg_dump -i -h localhost -p 5432 -U postgres -F c -b -w -v -f “dbprod${DATE}.backup” dbprod

    #SQL file /opt/postgres/9.0/bin/pg_dump -i -h localhost -p 5432 -U postgres –format plain –verbose -f “dbprod${DATE}.sql” dbprod
