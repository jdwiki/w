Bu sayfada

  • JSONb operatörlerini kullanma
  • Karmaşık JSON belgelerini sorgulama
  • -> ve ->> ile karşılaştırıldığında @> performansı
  • Saf bir JSON tablosu oluşturma

JSON - Java Script Object Notation , Postgresql, 9.2 sürümünden beri JSON Veri türünü destekler. JSON verilerine erişmek için önceden tanımlanmış bazı işlevler ve operatörler vardır.

-> operatörü JSON sütununun anahtarını döndürür. ->> operatörü JSON Sütununun değerini döndürür.

JSONb operatörlerini kullanma #

DB ve Tablo Oluşturma #

DROP DATABASE IF EXISTS books_db;
CREATE DATABASE books_db WITH ENCODING='UTF8' TEMPLATE template0;

DROP TABLE IF EXISTS books;

CREATE TABLE books (
  id SERIAL PRIMARY KEY,
  client TEXT NOT NULL,
  data JSONb NOT NULL
);

DB’yi doldurma #

INSERT INTO books(client, data) values (
    'Joe', 
    '{ "title": "Siddhartha", "author": { "first_name": "Herman", "last_name": "Hesse" } }'
),(
    'Jenny', 
    '{ "title": "Dharma Bums", "author": { "first_name": "Jack", "last_name": "Kerouac" } }'
),(
    'Jenny', 
    '{ "title": "100 años de soledad", "author": { "first_name": "Gabo", "last_name": "Marquéz" } }'
);

Tablo kitaplarının içindeki her şeyi görelim:

SELECT * FROM books;

Çıktı:

-> operatörü JSON sütunlarından değerleri döndürür #

1 sütun seçme:

SELECT client, 
    data->'title' AS title
    FROM books;

Çıktı:

2 sütun seçme:

SELECT client, 
   data->'title' AS title, data->'author' AS author
   FROM books;

Çıktı:

-> ve ->> #

-> operatörü orijinal JSON türünü (bir nesne olabilir) döndürürken, ->> metni döndürür.

NESTED nesneleri döndür #

İç içe bir nesne döndürmek ve böylece operatörleri zincirlemek için -> kullanabilirsiniz:

SELECT client, 
   data->'author'->'last_name' AS author
   FROM books;

Çıktı:

Filtreleme #

JSON’unuzdaki bir değere göre satırları seçin:

 SELECT 
 client,
 data->'title' AS title
 FROM books
  WHERE data->'title' = '"Dharma Bums"';

NEREDE -> kullandığına dikkat edin, bu nedenle JSON `’“Dharma Bums”’’ ile karşılaştırmamız gerekir.

Veya ->> kullanıp Dharma Bums ile karşılaştırabiliriz.

Çıktı:

İç içe filtreleme #

Yuvalanmış bir JSON nesnesinin değerine dayalı satırları bulun:

SELECT 
 client,
 data->'title' AS title
 FROM books
  WHERE data->'author'->>'last_name' = 'Kerouac';

Çıktı:

Gerçek bir dünya örneği #

CREATE TABLE events (
  name varchar(200),
  visitor_id varchar(200),
  properties json,
  browser json
);

Sayfa görüntülemeleri gibi olayları bu tabloda saklayacağız. Her olayın, herhangi bir şey olabilecek (ör. geçerli sayfa) ve ayrıca tarayıcı hakkında bilgi (işletim sistemi, ekran çözünürlüğü vb.) gönderen özellikleri vardır. Bunların her ikisi de tamamen ücretsiz biçimdir ve zamanla değişebilir (izlenecek ekstra şeyler düşündüğümüz için).

Şimdi her şeyi seçelim:

SELECT * FROM events;

Çıktı:

JSON operatörleri + PostgreSQL toplama işlevleri #

JSON operatörlerini, geleneksel PostgreSQL toplama işlevleriyle birleştirerek, istediğimizi çekebiliriz. Emrinizde bir RDBMS’nin tüm gücüne sahipsiniz.

  • Tarayıcı kullanımını görelim:

      SELECT browser->>'name' AS browser, 
    count(browser)
    FROM events
    GROUP BY browser->>'name';

Çıktı:

  • Ziyaretçi başına toplam gelir:

      SELECT visitor_id, SUM(CAST(properties->>'amount' AS integer)) AS total
  FROM events
  WHERE CAST(properties->>'amount' AS integer) > 0
  GROUP BY visitor_id;

Çıktı:

  • Ortalama ekran çözünürlüğü

      SELECT AVG(CAST(browser->'resolution'->>'x' AS integer)) AS width,
    AVG(CAST(browser->'resolution'->>'y' AS integer)) AS height
  FROM events;

Karmaşık JSON belgelerini sorgulama #

Bir tabloda karmaşık bir JSON belgesi almak:

CREATE TABLE mytable (data JSONB NOT NULL);
CREATE INDEX mytable_idx ON mytable USING gin (data jsonb_path_ops);
INSERT INTO mytable VALUES($$
{
    "name": "Alice",
    "emails": [
        "alice1@test.com",
        "alice2@test.com"
    ],
    "events": [
        {
            "type": "birthday",
            "date": "1970-01-01"
        },
        {
            "type": "anniversary",
            "date": "2001-05-05"
        }
    ],
    "locations": {
        "home": {
            "city": "London",
            "country": "United Kingdom"
        },
        "work": {
            "city": "Edinburgh",
            "country": "United Kingdom"
        }
    }
}
$$);

Bir üst düzey öğe için sorgu:

SELECT data->>'name' FROM mytable WHERE data @> '{"name":"Alice"}';

Bir dizideki basit bir öğeyi sorgulayın:

SELECT data->>'name' FROM mytable WHERE data @> '{"emails":["alice1@test.com"]}';

Bir dizideki bir nesneyi sorgulayın:

SELECT data->>'name' FROM mytable WHERE data @> '{"events":[{"type":"anniversary"}]}';

Yuvalanmış bir nesne için sorgu:

SELECT data->>'name' FROM mytable WHERE data @> '{"locations":{"home":{"city":"London"}}}';

-> ve ->> ile karşılaştırıldığında @> performansı #

Sorgunun ‘WHERE’ kısmında ‘@>’, ‘->’ ve ‘-»’ kullanma arasındaki performans farkını anlamak önemlidir. Bu iki sorgu genel olarak eşdeğer gibi görünse de:

SELECT data FROM mytable WHERE data @> '{"name":"Alice"}';
SELECT data FROM mytable WHERE data->'name' = '"Alice"';
SELECT data FROM mytable WHERE data->>'name' = 'Alice';

ilk ifade yukarıda oluşturulan dizini kullanacak, son ikisi ise tam bir tablo taraması gerektirmeyecek.

Sonuç verileri elde edilirken -> operatörünün kullanılmasına hala izin verilmektedir, bu nedenle aşağıdaki sorgular da dizini kullanacaktır:

SELECT data->'locations'->'work' FROM mytable WHERE data @> '{"name":"Alice"}';
SELECT data->'locations'->'work'->>'city' FROM mytable WHERE data @> '{"name":"Alice"}';

Saf bir JSON tablosu oluşturma #

Saf bir JSON tablosu oluşturmak için “JSONB” tipinde tek bir alan sağlamanız gerekir:

CREATE TABLE mytable (data JSONB NOT NULL);

Ayrıca temel bir dizin oluşturmalısınız:

CREATE INDEX mytable_idx ON mytable USING gin (data jsonb_path_ops);

Bu noktada tabloya veri ekleyebilir ve verimli bir şekilde sorgulayabilirsiniz.
