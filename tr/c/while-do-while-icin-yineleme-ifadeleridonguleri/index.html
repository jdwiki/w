Sözdizimi #

  • /* tüm sürümler */
  • için ([ifade]; [ifade]; [ifade]) one_statement
  • için ([ifade]; [ifade]; [ifade]) { zero or several statements }
  • while (ifade) one_statement
  • while (ifade) { sıfır veya birkaç ifade }
  • one_statement while (ifade) yapın;
  • do { bir veya daha fazla ifade } while (ifade);
  • // yukarıdaki forma ek olarak C99’dan beri
  • for (bildirim; [ifade]; [ifade]) one_statement;
  • for (beyan; [ifade]; [ifade]) { zero or several statements }

Yineleme Bildirimi/Döngüler iki kategoriye ayrılır:

  • kafa kontrollü yineleme ifadesi/döngüler
  • ayak kontrollü yineleme ifadesi/döngüler

Baş Kontrollü Yineleme İfadesi/Döngüler #

for ([<expression>]; [<expression>]; [<expression>]) <statement>
while (<expression>) <statement>

for ([declaration expression]; [expression] [; [expression]]) statement
son sürüm

Ayak Kontrollü Yineleme İfadesi/Döngüler #

do <statement> while (<expression>);

Döngü için #

Bir kod bloğunu tekrar tekrar yürütmek için resme döngüler gelir. ‘for’ döngüsü, bir kod bloğunun sabit sayıda çalıştırılacağı zaman kullanılacaktır. Örneğin, n büyüklüğünde bir diziyi kullanıcı girdileriyle doldurmak için, n kere scanf() çalıştırmamız gerekiyor.

#include <stddef.h>          // for size_t

int array[10];               // array of 10 int

for (size_t i = 0; i < 10; i++)    // i starts at 0 and finishes with 9
{
   scanf("%d", &array[i]);
}
son sürüm

Bu şekilde, scanf() fonksiyon çağrısı n defa (örneğimizde 10 defa) yürütülür, ancak sadece bir kez yazılır.

Burada, “i” değişkeni döngü indeksidir ve en iyi şekilde sunulduğu gibi bildirilir. “size_t” (size type) türü, veri nesneleri arasında sayılan veya döngü oluşturan her şey için kullanılmalıdır.

‘for’ içindeki değişkenleri bildirmenin bu yolu, yalnızca C99 standardına güncellenmiş derleyiciler için kullanılabilir. Herhangi bir nedenle hala eski bir derleyiciye takılırsanız, döngü dizinini “for” döngüsünden önce bildirebilirsiniz:

#include <stddef.h>        /* for size_t */
size_t i;
int array[10];             /* array of 10 int */

for (i = 0; i < 10; i++)       /* i starts at 0 and finishes at 9 */
{
   scanf("%d", &array[i]);
}
son sürüm

Döngü sırasında #

Bir koşul doğruyken bir kod parçasını yürütmek için bir “while” döngüsü kullanılır. “While” döngüsü, bir kod bloğu değişken sayıda çalıştırılacağı zaman kullanılacaktır. Örneğin, gösterilen kod, kullanıcı “0” olmayan sayılar girdiği sürece kullanıcı girdisini alır. Kullanıcı “0” eklerse, while koşulu artık doğru değildir, bu nedenle yürütme döngüden çıkar ve sonraki kodlara devam eder:

int num = 1;

while (num != 0)
{
    scanf("%d", &num);
}

Do-While döngüsü #

“for” ve “while” döngülerinden farklı olarak, “do-while” döngüleri, döngünün sonundaki koşulun doğruluğunu kontrol eder, bu da “do” bloğunun bir kez çalışacağı anlamına gelir ve ardından “while"ın durumunu kontrol eder. bloğun alt kısmında. Bu, bir “do-while” döngüsünün her zaman en az bir kez çalışacağı anlamına gelir.

Örneğin, bu “do-while” döngüsü, bu değerlerin toplamı “50"den büyük veya ona eşit olana kadar kullanıcıdan sayılar alacaktır:

int num, sum;
num = sum = 0;

do 
{
  scanf("%d", &num);
  sum += num;

} while (sum < 50);

“do-while” döngüleri çoğu programlama stilinde nispeten nadirdir.

Bir for döngüsünde kontrol yapısı ve akışı #

for ([declaration-or-expression]; [expression2]; [expression3])
{
    /* body of the loop */
}

Bir “for” döngüsünde, döngü koşulunun tümü isteğe bağlı olan üç ifadesi vardır.

  • İlk ifade, “bildirim-veya-ifade”, döngüyü _başlatır. Döngünün başında tam olarak bir kez yürütülür.

Bir döngü değişkeninin bildirimi ve başlatılması veya genel bir ifade olabilir. Bu bir bildirimse, bildirilen değişkenin kapsamı ‘for’ ifadesi ile sınırlandırılır.

son sürüm

C’nin tarihsel sürümleri burada yalnızca bir ifadeye izin veriyordu ve bir döngü değişkeninin bildiriminin ‘for’dan önce yerleştirilmesi gerekiyordu.

son sürüm
  • İkinci ifade olan “ifade2”, test koşuludur. İlk olarak başlatmadan sonra yürütülür. Koşul “doğru” ise, kontrol döngünün gövdesine girer. Değilse, döngü sonunda döngü gövdesinin dışına kayar. Akabinde, bu koşul, body ve update deyiminin her çalıştırılmasından sonra kontrol edilir. “true” olduğunda, kontrol, döngü gövdesinin başına geri döner. Koşulun genellikle, döngü gövdesinin kaç kez yürütüldüğünü kontrol etmesi amaçlanmıştır. Bu, bir döngüden çıkmanın birincil yoludur, diğer yol ise atlama ifadeleri kullanmaktır.
  • Üçüncü ifade, “ifade3”, güncelleme ifadesidir. Döngünün gövdesinin her yürütülmesinden sonra yürütülür. Döngü gövdesinin kaç kez çalıştırıldığının sayısını tutan bir değişkeni artırmak için sıklıkla kullanılır ve bu değişkene iterator adı verilir.

Döngü gövdesinin her yürütme örneğine bir yineleme adı verilir.

Örnek:

for(int i = 0; i < 10 ; i++)
{
    printf("%d", i);
}
son sürüm

Çıktı:

0123456789

Yukarıdaki örnekte, ilk olarak “i = 0” yürütülür ve “i” başlatılır. Ardından, “doğru” olarak değerlendirilen “i < 10” koşulu kontrol edilir. Kontrol, döngünün gövdesine girer ve ‘i’ değeri yazdırılır. Ardından, kontrol ‘i++’ konumuna geçer ve ‘i’ değeri 0’dan 1’e güncellenir. Ardından koşul tekrar kontrol edilir ve işlem devam eder. Bu, ‘i’ değeri 10 olana kadar devam eder. Ardından, ‘i < 10’ koşulu ‘yanlış’ olarak değerlendirilir, ardından kontrol döngüden çıkar.

Sonsuz Döngüler #

Eğer kontrol girer ama asla döngünün gövdesinden ayrılmazsa, bir döngünün sonsuz döngü olduğu söylenir. Bu, döngünün test koşulu hiçbir zaman “yanlış” olarak değerlendirilmediğinde gerçekleşir.

Örnek:

for (int i = 0; i >= 0; )
{
    /* body of the loop where i is not changed*/
}
son sürüm

Yukarıdaki örnekte, yineleyici olan “i” değişkeni 0 olarak başlatılır. Test koşulu başlangıçta “doğru"dur. Ancak, ‘i’ gövdenin herhangi bir yerinde değiştirilmez ve güncelleme ifadesi boştur. Bu nedenle, “i” 0 olarak kalacak ve test koşulu asla “yanlış” olarak değerlendirilmeyecek ve bu da sonsuz bir döngüye yol açacaktır.

atlama ifadesi, olmadığını varsayarsak sonsuz bir döngü oluşturmanın başka bir yolu da koşulu açıkça doğru tutmaktır:

while (true)
{
    /* body of the loop */
}

Bir “for” döngüsünde koşul ifadesi isteğe bağlıdır. Bu durumda, koşul her zaman boş bir şekilde “doğru"dur ve sonsuz bir döngüye yol açar.

for (;;)
{
    /* body of the loop */
}

Bununla birlikte, bazı durumlarda, “break” gibi bir jump deyimi kullanarak döngüden çıkmak amacıyla koşul kasıtlı olarak “true” olarak tutulabilir.

while (true)
{
    /* statements */
    if (condition)
    {
         /* more statements */
         break;
    }
}

Döngü Açma ve Duff’ın Cihazı #

Bazen, düz ileri döngü, döngü gövdesi içinde tamamen yer alamaz. Bunun nedeni, döngünün bazı B ifadeleriyle doldurulması gerektiğidir. Ardından, yineleme bazı A ifadeleriyle başlar, ardından döngüden önce B ile devam eder.

do_B();
while (condition) {
    do_A();
    do_B();
}

B‘yi kodda iki kez yinelemeyle ilgili olası kesme/yapıştırma sorunlarını önlemek için, bir [switch deyimi kullanılarak, döngüyü “while” gövdesinin ortasından başlatmak için Duff’s Device uygulanabilir [2] ve davranıştan düşer.

switch (true) while (condition) {
case false: do_A(); /* FALL THROUGH */
default:    do_B(); /* FALL THROUGH */
}

Duff’s Device aslında döngü açma işlemini uygulamak için icat edildi. Bir bellek bloğuna bir maske uyguladığınızı hayal edin, burada “n”, pozitif bir değere sahip işaretli bir integral türüdür.

do {
    *ptr++ ^= mask;
} while (--n > 0);

’n’ her zaman 4’e bölünebiliyorsa, bunu kolayca şu şekilde açabilirsiniz:

do {
    *ptr++ ^= mask;
    *ptr++ ^= mask;
    *ptr++ ^= mask;
    *ptr++ ^= mask;
} while ((n -= 4) > 0);

Ancak, Duff’s Device ile kod, “n” 4’e bölünemiyorsa, döngünün ortasında doğru yere atlayan bu unrolling deyimini takip edebilir.

switch (n % 4) do {
case 0: *ptr++ ^= mask; /* FALL THROUGH */
case 3: *ptr++ ^= mask; /* FALL THROUGH */
case 2: *ptr++ ^= mask; /* FALL THROUGH */
case 1: *ptr++ ^= mask; /* FALL THROUGH */
} while ((n -= 4) > 0);

Derleyicinin optimizasyon motoru programcı adına döngüleri açabildiğinden, bu tür manuel açma işlemi modern derleyicilerde nadiren gereklidir.
