Yorumlar, kodu okuyan kişiye bir şey belirtmek için kullanılır. Yorumlar derleyici tarafından boşluk gibi değerlendirilir ve kodun gerçek anlamında hiçbir şeyi değiştirmez. C’de yorumlar için kullanılan iki sözdizimi vardır, orijinal /* */ ve biraz daha yeni olan //. Bazı dokümantasyon sistemleri, kod için dokümantasyon oluşturmaya yardımcı olmak için özel olarak biçimlendirilmiş yorumları kullanır.

Sözdizimi #

  • /*...*/
  • //... (yalnızca C99 ve sonrası)

Önişlemciyi kullanarak yorum yapma #

Büyük kod parçaları da “#if 0” ve “#endif” önişlemci yönergeleri kullanılarak “yorumlanabilir”. Bu, kod, aksi takdirde yuvalanmayacak çok satırlı yorumlar içerdiğinde kullanışlıdır.

#if 0 /* Starts the "comment", anything from here on is removed by preprocessor */ 

/* A large amount of code with multi-line comments */  
int foo()
{
    /* lots of code */
    ...

    /* ... some comment describing the if statement ... */
    if (someTest) {
        /* some more comments */
        return 1;
    }

    return 0;
}

#endif /* 0 */

/* code from here on is "uncommented" (included in compiled executable) */
...

/* */ sınırlandırılmış yorumlar #

Bir yorum, bir eğik çizgiyle başlar ve hemen ardından bir yıldız işareti (/*) gelir ve bir yıldız işaretinin hemen ardından bir eğik çizgi (*/) ile karşılaşıldığında biter. Bu karakter kombinasyonları arasındaki her şey bir yorumdur ve derleyici tarafından boş (temelde yoksayılır) olarak değerlendirilir.

/* this is a comment */

Yukarıdaki yorum tek satırlık bir yorumdur. Bu /* türündeki yorumlar, aşağıdaki gibi birden çok satıra yayılabilir:

/* this is a
multi-line
comment */

Kesin olarak gerekli olmasa da, çok satırlı yorumlarla ortak bir stil kuralı, ilk satırdan sonraki satırlara baştaki boşluklar ve yıldız işaretleri ve yeni satırlara da ‘/’ ve ‘/’ karakterleri yerleştirmektir. sıralanmak:

/* 
 * this is a
 * multi-line
 * comment
 */

Hiçbirinde ilgili bir eğik çizgi olmadığından, ekstra yıldızların yorum üzerinde herhangi bir işlevsel etkisi yoktur.

Bu /* türü yorumlar kendi satırlarında, kod satırının sonunda veya hatta kod satırlarında kullanılabilir:

/* this comment is on its own line */
if (x && y) { /*this comment is at the end of a line */
    if ((complexCondition1) /* this comment is within a line of code */
            && (complexCondition2)) {
        /* this comment is within an if, on its own line */
    }
}

Yorumlar yuvalanamaz. Bunun nedeni, sonraki herhangi bir “/“nin yok sayılması (yorumun bir parçası olarak) ve ulaşılan ilk “/“nin yorumu sonlandırıyor olarak değerlendirilmesidir. Aşağıdaki örnekteki yorum çalışmayacak:

/* outer comment, means this is ignored => /* attempted inner comment */ <= ends the comment, not this one => */

Aksi takdirde iç içe olacak bu tür yorumları içeren kod bloklarını yorumlamak için aşağıdaki https://www.wikiod.com/tr/c/yorumlar#Önişlemciyi kullanarak yorum yapma örneğine bakın.

// sınırlandırılmış yorumlar #

C99, C++ tarzı tek satırlı yorumların kullanımını tanıttı. Bu yorum türü iki eğik çizgi ile başlar ve bir satırın sonuna kadar devam eder:

// this is a comment

Bu yorum türü, çok satırlı yorumlara izin vermez, ancak arka arkaya birkaç tek satırlık yorum ekleyerek bir yorum bloğu oluşturmak mümkündür:

// each of these lines are a single-line comment
// note how each must start with
// the double forward-slash

Bu tür bir yorum, kendi satırında veya bir kod satırının sonunda kullanılabilir. Ancak, _satırın sonuna kadar çalıştıkları için, bir kod satırında kullanılamazlar

// this comment is on its own line
if (x && y) { // this comment is at the end of a line
    // this comment is within an if, on its own line
}
son sürüm

Trigraflardan kaynaklanan olası tuzaklar #

// ile sınırlandırılmış yorumlar yazarken, beklenen çalışmasını etkileyen bir yazım hatası yapmak mümkündür. Eğer bir tür:

int x = 20;  // Why did I do this??/

Sondaki / bir yazım hatasıydı ama şimdi \ olarak yorumlanacak. Bunun nedeni, ??/ öğesinin bir trigraf oluşturmasıdır.

??/ trigrafı aslında satır devam sembolü olan \ için uzun bir notasyondur. Bu, derleyicinin bir sonraki satırın mevcut satırın devamı olduğunu, yani yorumun devamı olduğunu düşündüğü anlamına gelir ki bu amaçlanan şey olmayabilir.

int foo = 20; // Start at 20 ??/
int bar = 0;

// The following will cause a compilation error (undeclared variable 'bar')
// because 'int bar = 0;' is part of the comment on the preceding line
bar += foo;
