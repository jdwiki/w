Bir programlama dilindeki operatör, derleyiciye veya yorumlayıcıya belirli bir matematiksel, ilişkisel veya mantıksal işlemi gerçekleştirmesini ve nihai bir sonuç üretmesini söyleyen bir semboldür.

C’nin birçok güçlü operatörü vardır. Birçok C operatörü ikili operatörlerdir, yani iki işlenenleri vardır. Örneğin, “a / b"de “/” iki işleneni (“a”, “b”) kabul eden bir ikili operatördür. Bir işlenen alan bazı tekli operatörler vardır (örneğin: ~, ++), ve yalnızca bir üçlü operatör ? :.

Sözdizimi #

  • ifade1 operatörü
  • operatör ifadesi2
  • ifade1 operatörü ifade2
  • ifade1 ? ifade2: ifade3

Operatörlerin bir arity, bir öncelik ve bir associability vardır.

  • Arity işlenenlerin sayısını gösterir. C’de üç farklı operatör yapısı vardır:

  • Birli (1 işlenen)

  • İkili (2 işlenen)

  • Üçlü (3 işlenen)

  • Öncelik hangi operatörlerin işlenenlerine ilk önce “bağlandığını” gösterir. Yani, hangi operatörün işlenenleri üzerinde işlem yapma önceliği vardır. Örneğin, C dili, çarpma ve bölmenin toplama ve çıkarmaya göre öncelikli olduğu kuralına uyar:

    a * b + c

ile aynı sonucu verir

  (a * b) + c

İstenen bu değilse, tüm operatörler arasında en yüksek önceliğe sahip olduklarından, öncelik parantez kullanılarak zorlanabilir.

  a * (b + c)

Bu yeni ifade, önceki iki ifadeden farklı bir sonuç üretecektir.

C dilinin birçok öncelik düzeyi vardır; Aşağıda tüm operatörlerin azalan öncelik sırasına göre bir tablosu verilmiştir.

**Precedence Table**

Operators                                                     | Associativity
------                                                        | ------
`()` `[]` `->` `.`                                            | left to right
`!` `~` `++` `--` `+` `-` `*` (dereference) `(type)` `sizeof` | right to left
`*` (multiplication) `/` `%`                                  | left to right
`+` `-`                                                       | left to right
`<<` `>>`                                                     | left to right
`<` `<=` `>` `>=`                                             | left to right
`==` `!=`                                                     | left to right
`&`                                                           | left to right
`^`                                                           | left to right
<code>&#124;</code>                                                          | left to right
`&&`                                                          | left to right
<code>&#124;&#124;</code>                                                        | left to right
`?:`                                                          | right to left
`=` `+=` `-=` `*=` `/=` `%=` `&=` `^=` <code>&#124;=</code> `<<=` `>>=`       | right to left
`,`                                                          | left to right

  • Associativity, eşit önceliğe sahip operatörlerin varsayılan olarak nasıl bağlandığını gösterir ve iki tür vardır: Soldan Sağa ve Sağdan Sola. Soldan Sağa bağlama örneği, çıkarma operatörüdür (-). İfade

    a - b - c - d

üç özdeş öncelik çıkarma işlemine sahiptir, ancak aynı sonucu verir

  ((a - b) - c) - d

çünkü en soldaki - iki işlenene önce bağlanır.

Sağdan Sola ilişkilendirmenin bir örneği, başvurudan çıkarma “*” ve artım sonrası “++” operatörleridir. Her ikisi de eşit önceliğe sahiptir, bu nedenle aşağıdaki gibi bir ifadede kullanılırlarsa

  * ptr ++

, bu eşdeğerdir

  * (ptr ++)

çünkü en sağdaki tekli operatör (++) önce tek işlenene bağlanır.

İlişkisel Operatörler #

İlişkisel operatörler, iki işlenen arasındaki belirli bir ilişkinin doğru olup olmadığını kontrol eder. Sonuç 1 (doğru anlamına gelir) veya 0 (yanlış anlamına gelir) olarak değerlendirilir. Bu sonuç genellikle kontrol akışını etkilemek için kullanılır (‘if’, ‘while’, ‘for’ aracılığıyla), ancak değişkenlerde de saklanabilir.

Eşittir “==” #

Sağlanan işlenenlerin eşit olup olmadığını kontrol eder.

1 == 0;         /* evaluates to 0. */
1 == 1;         /* evaluates to 1. */

int x = 5;
int y = 5;
int *xptr = &x, *yptr = &y;
xptr == yptr;   /* evaluates to 0, the operands hold different location addresses. */
*xptr == *yptr; /* evaluates to 1, the operands point at locations that hold the same value. */

Dikkat: Bu operatör atama operatörü (=) ile karıştırılmamalıdır!

“!="’e eşit değildir #

Sağlanan işlenenlerin eşit olup olmadığını kontrol eder.

1 != 0;         /* evaluates to 1. */
1 != 1;         /* evaluates to 0. */

int x = 5;
int y = 5;
int *xptr = &x, *yptr = &y;
xptr != yptr;   /* evaluates to 1, the operands hold different location addresses. */
*xptr != *yptr; /* evaluates to 0, the operands point at locations that hold the same value. */

Bu operatör, eşittir (==) operatörünün sonucunun tersini etkin bir şekilde döndürür.

Değil “!” #

Bir nesnenin “0"a eşit olup olmadığını kontrol edin.

! aynı zamanda doğrudan bir değişkenle aşağıdaki şekilde de kullanılabilir:

!someVal

Bu aynı etkiye sahiptir:

someVal == 0

“>” den büyük #

Sol işlenenin sağ işlenenden daha büyük bir değere sahip olup olmadığını kontrol eder

5 > 4      /* evaluates to 1. */
4 > 5      /* evaluates to 0. */
4 > 4      /* evaluates to 0. */

“<” değerinden daha az #

Sol işlenenin sağ işlenenden daha küçük bir değere sahip olup olmadığını kontrol eder

5 < 4      /* evaluates to 0. */
4 < 5      /* evaluates to 1. */
4 < 4      /* evaluates to 0. */

Büyük veya eşit “>=” #

Sol işlenenin sağ işlenenden daha büyük veya eşit bir değere sahip olup olmadığını kontrol eder.

5 >= 4      /* evaluates to 1. */
4 >= 5      /* evaluates to 0. */
4 >= 4      /* evaluates to 1. */

“<=” değerinden küçük veya eşit #

Sol işlenenin sağ işlenenden daha küçük veya eşit bir değere sahip olup olmadığını kontrol eder.

5 <= 4      /* evaluates to 0. */
4 <= 5      /* evaluates to 1. */
4 <= 4      /* evaluates to 1. */

Koşullu Operatör/Üçlü Operatör #

İlk işlenenini değerlendirir ve elde edilen değer sıfıra eşit değilse ikinci işlenenini değerlendirir. Aksi takdirde, aşağıdaki örnekte gösterildiği gibi üçüncü işlenenini değerlendirir:

a = b ? c : d;

şuna eşittir:

if (b)
    a = c;
else 
    a = d;

Bu sözde kod onu temsil eder: durum? değer_doğruysa_değer: eğer_yanlışsa_değer. Her değer, değerlendirilen bir ifadenin sonucu olabilir.

int x = 5;
int y = 42;
printf("%i, %i\n", 1 ? x : y, 0 ? x : y); /* Outputs "5, 42" */

Koşullu operatör yuvalanabilir. Örneğin, aşağıdaki kod üç sayıdan büyük olanı belirler:

big= a > b ? (a > c ? a : c)
           : (b > c ? b : c);

Aşağıdaki örnek, çift tam sayıları bir dosyaya ve tek tam sayıları başka bir dosyaya yazar:

#include<stdio.h>

int main()
{
    FILE *even, *odds;
    int n = 10;
    size_t k = 0;

    even = fopen("even.txt", "w");
    odds = fopen("odds.txt", "w");

    for(k = 1; k < n + 1; k++)
    {
        k%2==0 ? fprintf(even, "\t%5d\n", k)
               : fprintf(odds, "\t%5d\n", k);
    }
    fclose(even);
    fclose(odds);

    return 0;
}

Koşullu operatör sağdan sola ilişkilendirir. Aşağıdakileri göz önünde bulundur:

exp1 ? exp2 : exp3 ? exp4 : exp5

İlişkilendirme sağdan sola olduğu için yukarıdaki ifade olarak değerlendirilir

exp1 ? exp2 : ( exp3 ? exp4 : exp5 )

Bitsel Operatörler #

Bitsel operatörler, değişkenler üzerinde bit düzeyinde işlem gerçekleştirmek için kullanılabilir. Aşağıda, C’de desteklenen altı bit düzeyinde operatörün bir listesi bulunmaktadır:

Sembol Operatör
& bit düzeyinde VE
| bit düzeyinde kapsayıcı VEYA
^ bit düzeyinde özel VEYA (XOR)
~ bit düzeyinde değil (birinin tamamlayıcısı)
« mantıksal sola kaydırma
>> mantıksal sağa kaydırma

Aşağıdaki program, tüm bitsel operatörlerin kullanımını göstermektedir:

#include <stdio.h>

int main(void)
{
   unsigned int a = 29;    /* 29 = 0001 1101 */  
   unsigned int b = 48;    /* 48 = 0011 0000 */
   int c = 0;           

   c = a & b;              /* 32 = 0001 0000 */ 
   printf("%d & %d = %d\n", a, b, c );

   c = a | b;              /* 61 = 0011 1101 */
   printf("%d | %d = %d\n", a, b, c );

   c = a ^ b;              /* 45 = 0010 1101 */
   printf("%d ^ %d = %d\n", a, b, c );

   c = ~a;                 /* -30 = 1110 0010 */
   printf("~%d = %d\n", a, c );

   c = a << 2;             /* 116 = 0111 0100 */
   printf("%d << 2 = %d\n", a, c );

   c = a >> 2;             /* 7 = 0000 0111 */
   printf("%d >> 2 = %d\n", a, c );

   return 0;
}

İşaretli türlerle bitsel işlemlerden kaçınılmalıdır, çünkü böyle bir bit gösteriminin işaret bitinin özel bir anlamı vardır. Vardiya operatörleri için özel kısıtlamalar geçerlidir:

  • İşaretli bitin 1 bitini sola kaydırmak hatalıdır ve tanımsız davranışa yol açar.

  • Negatif bir değeri sağa kaydırma (işaret biti 1 ile) uygulama tanımlıdır ve bu nedenle taşınabilir değildir.

  • Bir kaydırma operatörünün sağ işleneninin değeri negatifse veya yükseltilmiş sol işlenenin genişliğinden büyük veya ona eşitse, davranış tanımsızdır.

Maskeleme:

Maskeleme, mantıksal bitsel işlemler kullanılarak bir değişkenden istenen bitlerin çıkarılması (veya istenen bitlerin dönüştürülmesi) sürecini ifade eder. Maskelemeyi gerçekleştirmek için kullanılan işlenene (sabit veya değişken) maske adı verilir.

Maskeleme birçok farklı şekilde kullanılır:

  • Bir tamsayı değişkeninin bit düzenine karar vermek.
  • Belirli bir bit deseninin bir kısmını yeni bir değişkene kopyalamak, yeni değişkenin geri kalanı 0’larla doldurulurken (bitsel VE kullanarak)
  • Belirli bir bit deseninin bir kısmını yeni bir değişkene kopyalamak, yeni değişkenin geri kalanı 1s ile doldurulurken (bitsel VEYA kullanılarak).
  • Belirli bir bit düzeninin bir kısmını yeni bir değişkene kopyalamak, orijinal bit düzeninin geri kalanı yeni değişken içinde ters çevrilirken (bit düzeyinde özel VEYA kullanılarak).

Aşağıdaki işlev, bir değişkenin bit modelini görüntülemek için bir maske kullanır:

#include <limits.h>
void bit_pattern(int u)
{
    int i, x, word;
    unsigned mask = 1;
    word = CHAR_BIT * sizeof(int);
    mask = mask << (word - 1);    /* shift 1 to the leftmost position */
    for(i = 1; i <= word; i++)
    {
        x = (u & mask) ? 1 : 0;  /* identify the bit */
        printf("%d", x);         /* print bit value */
        mask >>= 1;              /* shift mask to the right by 1 bit */
    }
}

Aritmetik operatörler #

Temel Aritmetik #

İlişkili matematiksel işlemi kullanarak sol işleneni sağ işlenene uygulamanın sonucu olan bir değer döndürün. Normal matematiksel komütasyon kuralları geçerlidir (yani toplama ve çarpma değişmelidir, çıkarma, bölme ve modül değildir).

Toplama Operatörü #

Toplama operatörü (+) iki işleneni birbirine eklemek için kullanılır. Örnek:

#include <stdio.h>

int main(void)
{
    int a = 5;
    int b = 7;

    int c = a + b; /* c now holds the value 12 */

    printf("%d + %d = %d",a,b,c); /* will output "5 + 7 = 12" */

    return 0;
}

çıkarma operatörü #

Çıkarma operatörü (-), ikinci işleneni birinciden çıkarmak için kullanılır. Örnek:

#include <stdio.h>

int main(void)
{
    int a = 10;
    int b = 7;

    int c = a - b; /* c now holds the value 3 */

    printf("%d - %d = %d",a,b,c); /* will output "10 - 7 = 3" */

    return 0;
}

çarpma operatörü #

Çarpma operatörü (*) her iki işleneni de çarpmak için kullanılır. Örnek:

#include <stdio.h>

int main(void)
{    
    int a = 5;
    int b = 7;

    int c = a * b; /* c now holds the value 35 */

    printf("%d * %d = %d",a,b,c); /* will output "5 * 7 = 35" */

    return 0;
}

* referans operatörüyle karıştırılmamalıdır.

Bölüm Operatörü #

Bölme operatörü (/), ilk işleneni ikinciye böler. Bölmenin her iki işleneni de tamsayıysa, bir tamsayı değeri döndürür ve kalanı atar (geri kalanı hesaplamak ve elde etmek için modulo operatörünü % kullanın).

İşlenenlerden biri kayan noktalı bir değerse, sonuç kesrin bir tahminidir.

Örnek:

#include <stdio.h>

int main (void)
{
    int a = 19 / 2 ; /* a holds value 9   */
    int b = 18 / 2 ; /* b holds value 9   */
    int c = 255 / 2; /* c holds value 127 */
    int d = 44 / 4 ; /* d holds value 11  */
    double e = 19 / 2.0 ; /* e holds value 9.5   */
    double f = 18.0 / 2 ; /* f holds value 9.0   */
    double g = 255 / 2.0; /* g holds value 127.5 */
    double h = 45.0 / 4 ; /* h holds value 11.25 */

    printf("19 / 2 = %d\n", a);    /* Will output "19 / 2 = 9"    */
    printf("18 / 2 = %d\n", b);    /* Will output "18 / 2 = 9"    */
    printf("255 / 2 = %d\n", c);   /* Will output "255 / 2 = 127" */
    printf("44 / 4 = %d\n", d);    /* Will output "44 / 4 = 11"   */
    printf("19 / 2.0 = %g\n", e);  /* Will output "19 / 2.0 = 9.5"    */
    printf("18.0 / 2 = %g\n", f);  /* Will output "18.0 / 2 = 9"      */
    printf("255 / 2.0 = %g\n", g); /* Will output "255 / 2.0 = 127.5" */
    printf("45.0 / 4 = %g\n", h);  /* Will output "45.0 / 4 = 11.25"  */

    return 0;
}

Modulo Operatörü #

Modulo operatörü (%) yalnızca tamsayı işlenenleri alır ve ilk işlenen ikinciye bölündükten sonra kalanı hesaplamak için kullanılır. Örnek:

#include <stdio.h>

int main (void) {
    int a = 25 % 2;    /* a holds value 1  */
    int b = 24 % 2;    /* b holds value 0  */
    int c = 155 % 5;   /* c holds value 0  */
    int d = 49 % 25;   /* d holds value 24 */

    printf("25 % 2 = %d\n", a);     /* Will output "25 % 2 = 1"    */
    printf("24 % 2 = %d\n", b);     /* Will output "24 % 2 = 0"    */
    printf("155 % 5 = %d\n", c);    /* Will output "155 % 5 = 0"   */
    printf("49 % 25 = %d\n", d);    /* Will output "49 % 25 = 24"  */

    return 0;
}

Artırma / Azaltma Operatörleri #

Artırma (a++) ve azaltma (a--) operatörleri, onları uyguladığınız değişkenin değerini bir atama operatörü olmadan değiştirmeleri bakımından farklıdır. Artırma ve eksiltme operatörlerini değişkenden önce veya sonra kullanabilirsiniz. Operatörün yerleşimi, değerin artış/azalma zamanlamasını, değişkene atamadan önce veya sonra değiştirir. Örnek:

#include <stdio.h>

int main(void)
{
    int a = 1;
    int b = 4;
    int c = 1;
    int d = 4;

    a++;
    printf("a = %d\n",a);    /* Will output "a = 2" */
    b--;
    printf("b = %d\n",b);    /* Will output "b = 3" */

    if (++c > 1) { /* c is incremented by 1 before being compared in the condition */
        printf("This will print\n");    /* This is printed */
    } else {
        printf("This will never print\n");    /* This is not printed */
    }

    if (d-- < 4) {  /* d is decremented after being compared */
        printf("This will never print\n");    /* This is not printed */
    } else {
        printf("This will print\n");    /* This is printed */
    }
}

“c” ve “d” örneğinin gösterdiği gibi, her iki operatörün de önek notasyonu ve sonek notasyonu olmak üzere iki formu vardır. Her ikisi de değişkeni artırmada (++) veya azaltmada (--) aynı etkiye sahiptir, ancak döndürdükleri değere göre farklılık gösterir: önek işlemleri önce işlemi yapar ve ardından değeri döndürür, oysa sonek işlemleri önce değeri belirler. döndürülecek değeri seçin ve ardından işlemi yapın.

Bu potansiyel olarak sezgisel olmayan davranış nedeniyle, ifadelerin içinde artırma/azaltma operatörlerinin kullanımı tartışmalıdır.

Virgül Operatörü #

Sol işlenenini değerlendirir, elde edilen değeri atar ve ardından hak işlenenini değerlendirir ve sonuç, en sağ işlenenin değerini verir.

int x = 42, y = 42;
printf("%i\n", (x *= 2, y)); /* Outputs "42". */

Virgül operatörü, işlenenleri arasına bir dizi noktası ekler.

Ayrı argümanları ayıran işlev çağrılarında kullanılan virgül‘ün virgül operatörü OLMADIĞINI, bunun yerine virgül operatöründen farklı bir ayırıcı olarak adlandırıldığını unutmayın. Bu nedenle, virgül operatörünün özelliklerine sahip değildir.

Yukarıdaki printf() çağrısı hem virgül operatörünü hem de ayırıcıyı içerir.

printf("%i\n", (x *= 2, y)); /* Outputs "42". */
/*           ^        ^ this is a comma operator */
/*           this is a separator */

Virgül operatörü, genellikle başlatma bölümünde ve bir “for” döngüsünün güncelleme bölümünde kullanılır. Örneğin:

for(k = 1; k < 10; printf("\%d\\n", k), k += 2);   /*outputs the odd numbers below 9/*

/* outputs sum to first 9 natural numbers */
for(sumk = 1, k = 1; k < 10; k++, sumk += k)
    printf("\%5d\%5d\\n", k, sumk);

Operatörlere Erişim #

Üye erişim operatörleri (nokta ‘.’ ve ok ‘->’) bir ‘yapı’nın bir üyesine erişmek için kullanılır.

nesnenin üyesi #

Erişilen nesnenin üyesi olan nesneyi gösteren değere göre değerlendirir.

struct MyStruct
{
    int x;
    int y;
};

struct MyStruct myObject;
myObject.x = 42;
myObject.y = 123;

printf(".x = %i, .y = %i\n", myObject.x, myObject.y); /* Outputs ".x = 42, .y = 123". */

Sivri uçlu nesnenin üyesi #

Üye erişimi tarafından takip edilen referans kaldırma için sözdizimsel şeker. Etkili bir şekilde, ‘x->y’ formunun bir ifadesi ‘(*x).y’nin kısaltmasıdır - ancak özellikle yapı işaretçileri iç içeyse ok operatörü çok daha açıktır.

struct MyStruct
{
    int x;
    int y;
};

struct MyStruct myObject;
struct MyStruct *p = &myObject;

p->x = 42;
p->y = 123;

printf(".x = %i, .y = %i\n", p->x, p->y); /* Outputs ".x = 42, .y = 123". */
printf(".x = %i, .y = %i\n", myObject.x, myObject.y); /* Also outputs ".x = 42, .y = 123". */

Adresi #

Tekli & operatörü, operatörün adresidir. Elde edilen nesnenin bir değer olması gerektiği yerde verilen ifadeyi değerlendirir. Ardından, türü, elde edilen nesnenin türüne bir işaretçi olan bir nesne olarak değerlendirilir ve elde edilen nesnenin adresini içerir.

int x = 3;
int *p = &x;
printf("%p = %p\n", （void *)&x, (void *)p); /* Outputs "A = A", for some implementation-defined A. */

Referans #

Tekli * operatörü bir işaretçinin başvurusunu kaldırır. Verilen ifadenin değerlendirilmesinden kaynaklanan işaretçinin referansının kaldırılmasından kaynaklanan değeri değerlendirir.

int x = 42;
int *p = &x;
printf("x = %d, *p = %d\n", x, *p); /* Outputs "x = 42, *p = 42". */

*p = 123;
printf("x = %d, *p = %d\n", x, *p); /* Outputs "x = 123, *p = 123". */

İndeksleme #

İndeksleme, işaretçi ekleme ve ardından referans kaldırma için sözdizimsel şekerdir. Etkili olarak, “a[i]” biçiminin bir ifadesi “*(a + i)“ye eşdeğerdir - ancak açık alt simge gösterimi tercih edilir.

int arr[] = { 1, 2, 3, 4, 5 };
printf("arr[2] = %i\n", arr[2]); /* Outputs "arr[2] = 3". */

İndekslemenin değiştirilebilirliği #

Bir tamsayıya işaretçi eklemek değişmeli bir işlemdir (yani işlenenlerin sırası sonucu değiştirmez), bu nedenle ‘işaretçi + tamsayı == tamsayı + işaretçi’.

Bunun bir sonucu olarak ‘arr[3]’ ve ‘3[arr]’ eşdeğerdir.

printf("3[arr] = %i\n", 3[arr]); /* Outputs "3[arr] = 4". */

Kod okunabilirliğini etkilediğinden, “arr[3]” yerine “3[arr]” ifadesinin kullanılması genellikle önerilmez. Gizlenmiş programlama yarışmalarında popüler olma eğilimindedir.

Mantıksal operatörlerin kısa devre davranışı #

Kısa devre, mümkün olduğunda (if/while/…) koşulunun değerlendirme kısımlarını atlayan bir işlevdir. İki işlenen üzerinde mantıksal bir işlem olması durumunda, ilk işlenen değerlendirilir (doğru veya yanlış olarak) ve bir karar varsa (yani, && kullanılırken ilk işlenen yanlış, || kullanılırken ilk işlenen doğrudur) ikinci işlenen değerlendirilir. değerlendirilmedi.

Örnek:

#include <stdio.h>
 
int main(void) {
  int a = 20;
  int b = -5;
 
  /* here 'b == -5' is not evaluated,
     since a 'a != 20' is false. */
  if (a != 20 && b == -5) {
    printf("I won't be printed!\n");
  }
   
  return 0;
}

Kendiniz kontrol edin:

#include <stdio.h>
 
int print(int i) {
  printf("print function %d\n", i);
  return i;
}
 
int main(void) {
  int a = 20;
 
  /* here 'print(a)' is not called,
     since a 'a != 20' is false. */
  if (a != 20 && print(a)) {
    printf("I won't be printed!\n");
  }

  /* here 'print(a)' is called,
     since a 'a == 20' is true. */
  if (a == 20 && print(a)) {
    printf("I will be printed!\n");
  }

  return 0;
}

Çıktı:

$ ./a.out
print function 20
I will be printed!

(Hesaplamalı olarak) maliyetli terimleri değerlendirmekten kaçınmak istediğinizde kısa devre önemlidir. Ayrıca, bu durumda olduğu gibi programınızın akışını büyük ölçüde etkileyebilir: http://stackoverflow.com/questions/26716255/why-does-this-program-print-forked-4-times

Operatör boyutu #

İşlenen olarak bir türle #

Verilen türdeki nesnelerin “size_t” türündeki bayt cinsinden boyutunu değerlendirir. Türün etrafında parantez gerektirir.

printf("%zu\n", sizeof(int)); /* Valid, outputs the size of an int object, which is platform-dependent. */
printf("%zu\n", sizeof int); /* Invalid, types as arguments need to be surrounded by parentheses! */

İşlenen olarak bir ifade ile #

Verilen ifade türündeki nesnelerin bayt cinsinden boyutunu “size_t” türünde değerlendirir. İfadenin kendisi değerlendirilmez. Parantez gerekli değildir; ancak verilen ifade tekli olması gerektiğinden, bunları her zaman kullanmak en iyi uygulama olarak kabul edilir.

char ch = 'a';
printf("%zu\n", sizeof(ch)); /* Valid, will output the size of a char object, which is always 1 for all platforms. */
printf("%zu\n", sizeof ch);  /* Valid, will output the size of a char object, which is always 1 for all platforms. */

Artırma / Azaltma #

Artırma ve eksiltme operatörleri önek ve sonek biçiminde bulunur.

int a = 1;
int b = 1;
int tmp = 0;

tmp = ++a;        /* increments a by one, and returns new value; a == 2, tmp == 2  */
tmp = a++;        /* increments a by one, but returns old value; a == 3, tmp == 2 */
tmp = --b;        /* decrements b by one, and returns new value; b == 0, tmp == 0 */
tmp = b--;        /* decrements b by one, but returns old value; b == -1, tmp == 0 */

Aritmetik işlemlerin sıralama noktalarını tanıtmadığını unutmayın, bu nedenle “++” veya “–” operatörleri içeren belirli ifadeler [tanımsız davranış]( https://www.wikiod.com/tr/c/tanmsz-davrans#Herhangi bir nesneyi iki sıra noktası arasında bir kereden fazla değiştirme).

Cast Operatörü #

Verilen ifadenin değerlendirilmesinden kaynaklanan değerden verilen türe açık bir dönüşüm gerçekleştirir.

int x = 3;
int y = 4;
printf("%f\n", (double)x / y); /* Outputs "0.750000". */

Burada “x"in değeri “double"a dönüştürülür, bölme de “y"nin değerini “double"a yükseltir ve bölmenin sonucu olarak, yazdırma için “printf"e bir “double” iletilir. .

İşlev Çağrısı Operatörü #

İlk işlenen bir işlev işaretçisi olmalıdır (işlevin bir işaretçisine dönüştürüleceği için bir işlev tanımlayıcısı da kabul edilebilir), çağrılacak işlevi tanımlar ve varsa diğer tüm işlenenler topluca işlev çağrısının bağımsız değişkenleri olarak bilinir. . İlgili bağımsız değişkenlerle uygun işlevin çağrılmasından elde edilen dönüş değerini değerlendirir.

int myFunction(int x, int y)
{
    return x * 2 + y;
}

int (*fn)(int, int) = &myFunction;
int x = 42;
int y = 123;

printf("(*fn)(%i, %i) = %i\n", x, y, (*fn)(x, y)); /* Outputs "fn(42, 123) = 207". */
printf("fn(%i, %i) = %i\n", x, y, fn(x, y)); /* Another form: you don't need to dereference explicitly */

Atama Operatörleri #

Sağ işlenenin değerini, sol işlenen tarafından adlandırılan depolama konumuna atar ve değeri döndürür.

int x = 5;      /* Variable x holds the value 5. Returns 5. */ 
char y = 'c';   /* Variable y holds the value 99. Returns 99 
                 * (as the character 'c' is represented in the ASCII table with 99).
                 */
float z = 1.5;  /* variable z holds the value 1.5. Returns 1.5. */
char const* s = "foo"; /* Variable s holds the address of the first character of the string 'foo'. */

Birçok aritmetik işlemin bileşik atama operatörü vardır.

a += b  /* equal to: a = a + b */
a -= b  /* equal to: a = a - b */
a *= b  /* equal to: a = a * b */
a /= b  /* equal to: a = a / b */
a %= b  /* equal to: a = a % b */
a &= b  /* equal to: a = a & b */
a |= b  /* equal to: a = a | b */
a ^= b  /* equal to: a = a ^ b */
a <<= b /* equal to: a = a << b */
a >>= b /* equal to: a = a >> b */

Bu bileşik atamaların önemli bir özelliği, sol taraftaki (‘a’) ifadenin yalnızca bir kez değerlendirilmesidir. Örneğin, “p” bir işaretçiyse

*p += 27;

‘p’ referanslarını yalnızca bir kez, aşağıdakiler ise bunu iki kez yapar.

*p = *p + 27;

Ayrıca, “a = b” gibi bir atamanın sonucunun değer olarak bilinen şey olduğuna da dikkat edilmelidir. Böylece, atama aslında daha sonra başka bir değişkene atanabilecek bir değere sahiptir. Bu, atamaların zincirlenmesinin tek bir ifadede birden çok değişkeni ayarlamasına izin verir.

Bu değer, başka bir ifadenin veya işlev çağrısının sonucunda bazı kodları koruyan “if” ifadelerinin (veya döngüler veya “switch” ifadelerinin) kontrol ifadelerinde kullanılabilir. Örneğin:

char *buffer;
if ((buffer = malloc(1024)) != NULL)
{
    /* do something with buffer */
    free(buffer);
}
else
{
    /* report allocation failure */
}

Bu nedenle, gizemli hatalara yol açabilecek yaygın bir yazım hatasından kaçınmak için özen gösterilmelidir.

int a = 2;
/* ... */
if (a = 1)
    /* Delete all files on my hard drive */

‘a = 1’ her zaman ‘1’ olarak değerlendirileceğinden ve bu nedenle ‘if’ ifadesinin kontrol ifadesi her zaman doğru olacağından bunun feci sonuçları olacaktır (bu yaygın tuzak hakkında daha fazla bilgiyi burada okuyun .com/documentation/c/2006/common-pitfalls/7886/mistakenly-writing-inplace-of-When-comparing#t=201704191400115596587)). Yazar hemen hemen kesinlikle eşitlik operatörünü (==) aşağıda gösterildiği gibi kullanmak istemiştir:

int a = 2;
/* ... */
if (a == 1)
    /* Delete all files on my hard drive */

Operatör İlişkilendirmesi

int a, b = 1, c = 2;
a = b = c;

Bu, “a"ya atanan “b"yi döndüren “c"yi “b"ye atar. Bunun nedeni, tüm atama operatörlerinin doğru ilişkilendirmeye sahip olması, yani ifadedeki en sağdaki işlemin önce değerlendirildiği ve sağdan sola doğru ilerlediği anlamına gelir.

Mantıksal operatörler #

mantıksal VE #

Her iki işlenen de sıfır değilse, 1 döndüren iki işlenenin mantıksal boolean AND-ing’ini gerçekleştirir. Mantıksal AND operatörü “int” türündedir.

0 && 0  /* Returns 0. */
0 && 1  /* Returns 0. */
2 && 0  /* Returns 0. */
2 && 3  /* Returns 1. */

mantıksal VEYA #

İşlenenlerden herhangi biri sıfır değilse, 1 döndüren iki işlenenin mantıksal boolean VEYA’sını gerçekleştirir. Mantıksal VEYA operatörü “int” türündedir.

0 || 0  /* Returns 0. */
0 || 1  /* Returns 1.  */
2 || 0  /* Returns 1.  */
2 || 3  /* Returns 1.  */

mantıksal DEĞİL #

Mantıksal bir olumsuzlama gerçekleştirir. Mantıksal NOT operatörü “int” türündedir. NOT operatörü en az bir bitin 1’e eşit olup olmadığını kontrol eder, eğer öyleyse 0 döndürür. Aksi takdirde 1 döndürür;

!1 /* Returns 0. */
!5 /* Returns 0. */
!0 /* Returns 1. */

Kısa Devre Değerlendirmesi #

Hem && hem de || için ortak olan bazı önemli özellikler vardır:

  • sol işlenen (LHS), sağ işlenen (RHS) hiç değerlendirilmeden önce tamamen değerlendirilir,
  • sol işlenen ile sağ işlenenin değerlendirilmesi arasında bir sıra noktası vardır,
  • ve en önemlisi, sol işlenenin sonucu genel sonucu belirlerse, sağ işlenen hiç değerlendirilmez.

Bunun anlamı şudur ki:

  • LHS ‘doğru’ (sıfır olmayan) olarak değerlendirilirse, ‘||’ RHS’si değerlendirilmeyecektir (çünkü ‘doğru VEYA herhangi bir şey’in sonucu ‘doğru’dur),
  • LHS ‘yanlış’ (sıfır) olarak değerlendirilirse, ‘&&’ RHS’si değerlendirilmeyecektir (çünkü ‘yanlış VE herhangi bir şey’in sonucu ‘yanlış’tır).

Bu, aşağıdaki gibi kod yazmanıza izin verdiği için önemlidir:

const char *name_for_value(int value)
{
    static const char *names[] = { "zero", "one", "two", "three", };
    enum { NUM_NAMES = sizeof(names) / sizeof(names[0]) };
    return (value >= 0 && value < NUM_NAMES) ? names[value] : "infinity";
}

İşleve negatif bir değer iletilirse, “değer >= 0” terimi false olarak değerlendirilir ve “değer < NUM_NAMES” terimi değerlendirilmez.

İşaretçi Aritmetiği #

İşaretçi ekleme #

Bir işaretçi ve bir “N” skaler türü verildiğinde, bellekteki işaret edilen nesneyi doğrudan izleyen, işaret edilen türün “N"inci öğesinin bir işaretçisi olarak değerlendirilir.

int arr[] = {1, 2, 3, 4, 5};
printf("*(arr + 3) = %i\n", *(arr + 3)); /* Outputs "4", arr's fourth element. */

İşaretçinin işlenen değer veya skaler değer olarak kullanılması önemli değildir. Bu, “3 + arr” gibi şeylerin geçerli olduğu anlamına gelir. “arr[k]”, bir dizinin “k+1” üyesiyse, “arr+k”, “arr[k]” için bir işaretçidir. Başka bir deyişle, “arr” veya “arr+0”, “arr[0]” için bir işaretçidir, “arr+1”, “arr[2]” için bir işaretçidir, vb. Genel olarak, “*(dizi+k)”, “dizi[k]” ile aynıdır.

Alışılmış aritmetiğin aksine, bir işaretçiye “int” işaretine “1” eklenmesi, geçerli adres değerine “4” bayt ekleyecektir. Dizi adları sabit işaretçiler olduğundan, dizi adını kullanarak işaretçi notasyonu aracılığıyla bir dizinin üyelerine erişmek için kullanabileceğimiz tek operatör ‘+‘dır. Ancak, bir diziye bir işaretçi tanımlayarak, bir dizideki verileri işlemek için daha fazla esneklik elde edebiliriz. Örneğin, bir dizinin üyelerini aşağıdaki gibi yazdırabiliriz:

#include<stdio.h>
static const size_t N = 5
    
int main()
{
    size_t k = 0;
    int arr[] = {1, 2, 3, 4, 5};
    for(k = 0; k < N; k++)
    {
        printf("\n\t%d", *(arr + k));
    }
    return 0;
}

Diziye bir işaretçi tanımlayarak, yukarıdaki program aşağıdakine eşdeğerdir:

#include<stdio.h>
static const size_t N = 5
    
int main()
{
    size_t k = 0;
    int arr[] = {1, 2, 3, 4, 5};
    int *ptr = arr; /* or int *ptr = &arr[0]; */
    for(k = 0; k < N; k++)
    {
        printf("\n\t%d", ptr[k]);
        /* or   printf("\n\t%d", *(ptr + k)); */
        /* or   printf("\n\t%d", *ptr++); */
    }
    return 0;
}

‘arr’ dizisinin üyelerine ‘+’ ve ‘++’ operatörleri kullanılarak erişildiğini görün. ‘ptr’ işaretçisiyle kullanılabilecek diğer operatörler ‘-’ ve ‘–’ dir.

İşaretçi çıkarma #

Aynı türe iki işaretçi verildiğinde, birinci işaretçinin değerini elde etmek için ikinci işaretçiye eklenmesi gereken skaler değeri tutan ‘ptrdiff_t’ türünde bir nesne olarak değerlendirilir.

int arr[] = {1, 2, 3, 4, 5};
int *p = &arr[2];
int *q = &arr[3];
ptrdiff_t diff = q - p;

printf("q - p = %ti\n", diff); /* Outputs "1". */
printf("*(p + (q - p)) = %d\n", *(p + diff)); /* Outputs "4". */

_Alignof #

Belirtilen tür için hizalama gereksinimini sorgular. Hizalama gereksinimi, türdeki iki nesnenin aralarına tahsis edilebileceği bayt sayısını temsil eden 2’nin pozitif integral gücüdür. C’de, hizalama gereksinimi “size_t” cinsinden ölçülür.

Tür adı, tamamlanmamış bir tür veya bir işlev türü olamaz. Tür olarak bir dizi kullanılırsa, dizi öğesinin türü kullanılır.

Bu operatöre genellikle “<stdalign.h>“den “alignof” kolaylık makrosu aracılığıyla erişilir.

int main(void)
{
    printf("Alignment of char = %zu\n", alignof(char));
    printf("Alignment of max_align_t = %zu\n", alignof(max_align_t));
    printf("alignof(float[10]) = %zu\n", alignof(float[10]));
    printf("alignof(struct{char c; int n;}) = %zu\n",
            alignof(struct {char c; int n;}));    
}

Olası Çıktı:

Alignment of char = 1
Alignment of max_align_t = 16
alignof(float[10]) = 4
alignof(struct{char c; int n;}) = 4
