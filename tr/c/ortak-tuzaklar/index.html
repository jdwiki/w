Aritmetik işlemlerde işaretli ve işaretsiz tam sayıları karıştırma #

Aritmetik işlemlerde “imzalı” ve “imzasız” tam sayıları karıştırmak genellikle iyi bir fikir değildir. Örneğin, aşağıdaki örneğin çıktısı ne olacak?

#include <stdio.h>

int main(void)
{ 
    unsigned int a = 1000;
    signed int b = -1;

    if (a > b) puts("a is more than b");
    else puts("a is less or equal than b"); 

    return 0;
}

1000 -1’den büyük olduğu için çıktının ‘a is büyüktür b’ olmasını beklersiniz, ancak durum böyle olmayacaktır.

Farklı integral türleri arasındaki aritmetik işlemler, olağan aritmetik dönüşümler olarak adlandırılan ortak bir tür içinde gerçekleştirilir (bakınız dil belirtimi, 6.3.1.8).

Bu durumda “ortak tür” ‘unsigned int’ olur, Çünkü Olağan aritmetik dönüşümler‘de belirtildiği gibi,

714  Aksi halde, işaretsiz tamsayı türüne sahip işlenen diğer işlenen türünün sıralamasına eşit veya daha büyük, daha sonra işaretli tamsayı türüne sahip işlenen, türüne dönüştürülür işaretsiz tamsayı tipine sahip işlenen.

Bu, “int” işleneni “b"nin karşılaştırmadan önce “unsigned int"ye dönüştürüleceği anlamına gelir.

-1, bir “işaretsiz int"ye dönüştürüldüğünde, sonuç, 1000’den büyük olan maksimum olası “işaretsiz int” değeridir, yani “a > b” yanlıştır.

Makrolar basit dize değişimleridir #

Makrolar basit dize değişimleridir. (Açıkçası, rastgele dizelerle değil, ön işleme belirteçleriyle çalışırlar.)

#include <stdio.h>

#define SQUARE(x) x*x

int main(void) {
    printf("%d\n", SQUARE(1+2));
    return 0;
}

Bu kodun “9” (‘33’) yazdırmasını bekleyebilirsiniz, ancak makro “1+21+2” olarak genişletileceğinden aslında “5” yazdırılacaktır.

Bu sorunu önlemek için argümanları ve tüm makro ifadesini parantez içine almalısınız.

#include <stdio.h>

#define SQUARE(x) ((x)*(x))

int main(void) {
    printf("%d\n", SQUARE(1+2));
    return 0;
}

Başka bir sorun da, bir makronun argümanlarının bir kez değerlendirilmesinin garanti edilmemesidir; hiç değerlendirilmeyebilirler veya birden çok kez değerlendirilebilirler.

#include <stdio.h>

#define MIN(x, y) ((x) <= (y) ? (x) : (y))

int main(void) {
    int a = 0;
    printf("%d\n", MIN(a++, 10));
    printf("a = %d\n", a);
    return 0;
}

Bu kodda, makro ((a++) <= (10) ? (a++) : (10)) şeklinde genişletilecektir. ‘a++’ (‘0’) ‘10’dan küçük olduğu için, ‘a++’ iki kez değerlendirilecek ve ‘a’ değerini ve ‘MIN’ den döndürülenlerin beklediğinizden farklı olmasını sağlayacaktır.

Bu, işlevler kullanılarak önlenebilir, ancak türlerin işlev tanımıyla sabitleneceğini, oysa makroların türlerle (çok) esnek olabileceğini unutmayın.

#include <stdio.h>

int min(int x, int y) {
    return x <= y ? x : y;
}

int main(void) {
    int a = 0;
    printf("%d\n", min(a++, 10));
    printf("a = %d\n", a);
    return 0;
}

Şimdi çift değerlendirme sorunu düzeltildi, ancak bu “min” işlevi, örneğin, “çift” verileri kesmeden ele alamaz.

Makro yönergeleri iki tür olabilir:

#define OBJECT_LIKE_MACRO     followed by a "replacement list" of preprocessor tokens
#define FUNCTION_LIKE_MACRO(with, arguments) followed by a replacement list

Bu iki makro türünü birbirinden ayıran şey, “#define"den sonra tanımlayıcıyı izleyen karakterdir: bu bir lparen ise, işlev benzeri bir makrodur; aksi takdirde, nesne benzeri bir makrodur. İşlev benzeri bir makro yazmak isteniyorsa, makronun adının sonu ile () arasında boşluk olmamalıdır. Ayrıntılı açıklama için bunu kontrol edin.

C99 veya sonraki sürümlerde, “statik satır içi int min(int x, int y) { … }” kullanabilirsiniz.

son sürüm

C11’de ‘min’ için ’tür-genel’ bir ifade yazabilirsiniz.

#include <stdio.h>

#define min(x, y) _Generic((x), \
                        long double: min_ld, \
                        unsigned long long: min_ull, \
                        default: min_i \
                        )(x, y)

#define gen_min(suffix, type) \
    static inline type min_##suffix(type x, type y) { return (x < y) ? x : y; }

gen_min(ld, long double)
gen_min(ull, unsigned long long)
gen_min(i, int)

int main(void)
{
    unsigned long long ull1 = 50ULL;
    unsigned long long ull2 = 37ULL;
    printf("min(%llu, %llu) = %llu\n", ull1, ull2, min(ull1, ull2));
    long double ld1 = 3.141592653L;
    long double ld2 = 3.141592652L;
    printf("min(%.10Lf, %.10Lf) = %.10Lf\n", ld1, ld2, min(ld1, ld2));
    int i1 = 3141653;
    int i2 = 3141652;
    printf("min(%d, %d) = %d\n", i1, i2, min(i1, i2));
    return 0;
}

Genel ifade, “double”, “float”, “long long”, “unsigned long”, “long”, “unsigned” ve uygun “gen_min” makro çağrıları gibi daha fazla türle genişletilebilir.

realloc’un dönüş değerini geçici bir değere kopyalamayı unutmak #

“realloc” başarısız olursa, “NULL” döndürür. Orijinal arabelleğin değerini ‘realloc’un dönüş değerine atarsanız ve ‘NULL’ döndürürse, orijinal arabellek (eski işaretçi) kaybolur ve bir memory sızıntısı ile sonuçlanır. . Çözüm, geçici bir işaretçiye kopyalamaktır ve bu geçici NULL değilse, o zaman gerçek arabelleğe kopyalayın.

char *buf, *tmp;

buf = malloc(...);
...

/* WRONG */
if ((buf = realloc(buf, 16)) == NULL)
    perror("realloc");

/* RIGHT */
if ((tmp = realloc(buf, 16)) != NULL)
    buf = tmp;
else
    perror("realloc");

\0 için fazladan bir bayt ayırmayı unutmak #

Bir dizgiyi ‘malloc’ed arabelleğe kopyalarken, strlen’e her zaman 1 eklemeyi unutmayın.

char *dest = malloc(strlen(src)); /* WRONG */
char *dest = malloc(strlen(src) + 1); /* RIGHT */

strcpy(dest, src);

Bunun nedeni, “strlen” öğesinin uzunluktaki “\0” sonunu içermemesidir. Eğer ‘YANLIŞ’ (yukarıda gösterildiği gibi) yaklaşımını kullanırsanız, ‘strcpy’yi çağırdıktan sonra programınız tanımsız davranışı başlatır.

Aynı zamanda, ‘stdin’den veya başka bir kaynaktan bilinen maksimum uzunlukta bir dizi okuduğunuz durumlar için de geçerlidir. Örneğin

#define MAX_INPUT_LEN 42

char buffer[MAX_INPUT_LEN]; /* WRONG */
char buffer[MAX_INPUT_LEN + 1]; /* RIGHT */

scanf("%42s", buffer);  /* Ensure that the buffer is not overflowed */

Yanlış anlama dizi bozulması #

Çok boyutlu diziler, işaretçi dizileri vb. kullanan koddaki yaygın bir sorun, “Tür**” ve “Tür[M][N]“nin temelde farklı türler olmasıdır:

#include <stdio.h>

void print_strings(char **strings, size_t n)
{
    size_t i;
    for (i = 0; i < n; i++)
        puts(strings[i]);
}

int main(void)
{
    char s[4][20] = {"Example 1", "Example 2", "Example 3", "Example 4"};
    print_strings(s, 4);
    return 0;
}

Örnek derleyici çıktısı:

file1.c: In function 'main':
file1.c:13:23: error: passing argument 1 of 'print_strings' from incompatible pointer type [-Wincompatible-pointer-types]
         print_strings(strings, 4);
                       ^
file1.c:3:10: note: expected 'char **' but argument is of type 'char (*)[20]'
     void print_strings(char **strings, size_t n)

Hata, “main” işlevindeki “s” dizisinin, aldığından farklı bir işaretçi türü bekleyen “print_strings” işlevine iletildiğini belirtir. Ayrıca, ‘print_strings’ tarafından beklenen türü ve ‘main’den ona geçirilen türü ifade eden bir not içerir.

Sorun, dizi bozulması adlı bir şeyden kaynaklanıyor. İşleve ‘char[4]][20]’ (20 karakterlik 4 diziden oluşan dizi) türüyle ’s’ iletildiğinde, sanki ‘&s[ yazmışsınız gibi ilk öğesine bir işaretçiye dönüşürse ne olur? 0]`, ‘char (*)[20]’ tipine sahip (işaretçi 1 dizi 20 karakter). Bu, bir dizi işaretçi, bir dizi dizi (3-D diziler) ve bir diziye yönelik bir dizi işaretçi dahil olmak üzere herhangi bir dizi için oluşur. void print_strings(char (*strings)[20], size_t n)
/* OR */
void print_strings(char strings[][20], size_t n)

#include <stdio.h>

/*
 * Note the rearranged parameters and the change in the parameter name
 * from the previous definitions:
 *      n (number of strings)
 *   => scount (string count)
 *
 * Of course, you could also use one of the following highly recommended forms
 * for the `strings` parameter instead:
 *
 *    char strings[scount][ccount]
 *    char strings[][ccount]

void print_strings(size_t scount, size_t ccount, char (*strings)[ccount])
{
    size_t i;
    for (i = 0; i < scount; i++)
        puts(strings[i]);
}

int main(void)
{
    char s[4][20] = {"Example 1", "Example 2", "Example 3", "Example 4"};
    print_strings(4, 20, s);
    return 0;

Belleği boşaltmayı unutmak (bellek sızıntıları) #

En iyi programlama uygulaması, doğrudan kendi kodunuzla veya strdup() gibi bir kitaplık API’si gibi dahili veya harici bir işlevi çağırarak dolaylı olarak ayrılmış herhangi bir belleği serbest bırakmaktır. Belleği boşaltamamak, programınız (veya sisteminiz) için kullanılamayan önemli miktarda boşa harcanan bellekte birikebilecek ve muhtemelen çökmelere veya tanımsız davranışlara yol açabilecek bir bellek sızıntısına neden olabilir. Sızıntı bir döngü veya özyinelemeli işlevde art arda meydana gelirse, sorunların ortaya çıkması daha olasıdır. Program hatası riski, sızdıran bir program ne kadar uzun süre çalışırsa o kadar artar. Bazen sorunlar anında ortaya çıkar; diğer zamanlarda, saatlerce hatta yıllarca sürekli çalışma için sorunlar görülmez. Koşullara bağlı olarak, bellek tükenme hataları felaket olabilir.

Aşağıdaki sonsuz döngü, bu belleği boşaltmadan örtük olarak yeni bellek tahsis eden bir işlev olan getline()‘ı çağırarak mevcut bellek sızıntısını tüketecek bir sızıntı örneğidir.

#include <stdlib.h>
#include <stdio.h>

int main(void)
{
    char *line = NULL;
    size_t size = 0;

    /* The loop below leaks memory as fast as it can */

    for(;;) { 
        getline(&line, &size, stdin); /* New memory implicitly allocated */

        /* <do whatever> */

        line = NULL;
    }

    return 0;
}

Buna karşılık, aşağıdaki kod da ‘getline()’ işlevini kullanır, ancak bu sefer, ayrılan bellek doğru şekilde boşaltılır ve bir sızıntı önlenir.

#include <stdlib.h>
#include <stdio.h>

int main(void)
{
    char *line = NULL;
    size_t size = 0;

    for(;;) {
        if (getline(&line, &size, stdin) < 0) {
            free(line);
            line = NULL;

            /* Handle failure such as setting flag, breaking out of loop and/or exiting */
        }

        /* <do whatever> */

        free(line);
        line = NULL;

    }

    return 0;
}

Örneğin, çoğu modern işletim sistemi, bir program sona erdiğinde, program hatasından, “exit()“e yapılan bir sistem çağrısından, işlemin sonlandırılmasından veya “main()“in sonuna ulaşıldığından dolayı, bir program tarafından tüketilen tüm belleği otomatik olarak boşaltır. Yakın program sonlandırma noktasında açıkça belleği boşaltmak, aslında gereksiz olabilir veya bir performans cezasına neden olabilir.

Bağlanırken tanımlanmamış referans hataları #

Derlemedeki en yaygın hatalardan biri bağlantı aşamasında gerçekleşir. Hata şuna benzer:

$ gcc undefined_reference.c 
/tmp/ccoXhwF0.o: In function `main':
undefined_reference.c:(.text+0x15): undefined reference to `foo'
collect2: error: ld returned 1 exit status
$

Şimdi bu hatayı oluşturan koda bakalım:

int foo(void);

int main(int argc, char **argv)
{
    int foo_val;
    foo_val = foo();
    return foo_val;
}

Burada bir foo (int foo();) declaration görüyoruz, ancak bunun definition (gerçek işlev) yok. Bu yüzden derleyiciye fonksiyon başlığını sağladık, ancak hiçbir yerde tanımlanmış böyle bir fonksiyon yoktu, bu yüzden derleme aşaması geçiyor ama linker bir ‘Undefined Reference’ hatasıyla çıkıyor. Küçük programımızda bu hatayı düzeltmek için sadece foo için bir definition eklememiz gerekir:

/* Declaration of foo */
int foo(void);

/* Definition of foo */
int foo(void)
{
    return 5;
}

int main(int argc, char **argv)
{
    int foo_val;
    foo_val = foo();
    return foo_val;
}

Şimdi bu kod derlenecek. “foo()” kaynağının ayrı bir “foo.c” kaynak dosyasında olduğu (ve her iki “foo.c” dosyasında bulunan “foo()“yu bildirmek için bir “foo.h” başlığının bulunduğu) alternatif bir durum ortaya çıkar. veundefined_reference.c`). O zaman düzeltme, hem “foo.c” hem de “undefined_reference.c"den gelen nesne dosyasını bağlamak veya her iki kaynak dosyayı da derlemektir:

$ gcc -c undefined_reference.c 
$ gcc -c foo.c
$ gcc -o working_program undefined_reference.o foo.o
$

Veya:

$ gcc -o working_program undefined_reference.c foo.c
$

/* Return 'true' if the most significant bit is set */

    else

/* Return 'true' if the most significant bit is set */

        return false;

/* Return 'true' if the most significant bit is set */
bool isUpperBitSet(uint8_t bitField)
{
    /* If upper bit is set, result is 0x80 which the if will evaluate as true */
    if (bitField & 0x80)
    {
        return true;
    }
    else
    {
        return false;
    }

İşaretçi aritmetiğinde ekstra ölçeklendirme yapma #

İşaretçi aritmetiğinde, işaretçiye eklenecek veya çıkarılacak tam sayı, adres değişikliği olarak değil, taşınacak öğe sayısı olarak yorumlanır.

#include <stdio.h>

int main(void) {

    int *ptr = &array[0];
    int *ptr2 = ptr + 2;
    printf("%d %d\n", *ptr, *ptr2); /* "1 3" will be printed */
    return 0;
}

Toplamsal operatörleri kullanan açık işaretçi aritmetiği kafa karıştırıcı olabilir, bu nedenle dizi aboneliği kullanmak daha iyi olabilir.

#include <stdio.h>

int main(void) {
    int array[] = {1, 2, 3, 4, 5};
    int *ptr = &array[0];
    int *ptr2 = &ptr[2];
    printf("%d %d\n", *ptr, *ptr2); /* "1 3" will be printed */
    return 0;
}

“E1[E2]”, “(*((E1)+(E2)))” ile aynıdır (N1570 1990.pdf) 6.5.2.1, paragraf 2) ve “&(E1[E2])”, “((E1)+(E2))” ile eşdeğerdir (N1570 6.5.3.2, dipnot 102).

Alternatif olarak, işaretçi aritmetiği tercih edilirse, işaretçiyi farklı bir veri türüne yönlendirmek bayt adreslemeye izin verebilir. Yine de dikkatli olun: endianness bir sorun haline gelebilir ve ‘karakter için işaretçi’ dışındaki türlere döküm yapmak katı takma sorunlarına yol açar.

#include <stdio.h>

int main(void) {
    int array[3] = {1,2,3};  // 4 bytes * 3 allocated
    unsigned char *ptr = (unsigned char *) array;  // unsigned chars only take 1 byte
    /*
     * Now any pointer arithmetic on ptr will match
     * bytes in memory.  ptr can be treated like it
     * was declared as: unsigned char ptr[12];
     */

    return 0;
}

Kayan nokta sayılarını karşılaştırma #

Kayan nokta türleri (“float”, “double” ve “long double”) sonlu kesinliğe sahip oldukları ve değerleri ikili biçimde temsil ettikleri için bazı sayıları tam olarak temsil edemezler. Tıpkı 1/3 gibi kesirler için 10 tabanında tekrar eden ondalık sayılara sahip olmamız gibi, ikili olarak sonlu olarak temsil edilemeyen kesirler de vardır (1/3 gibi, ama daha da önemlisi 1/10 gibi). Kayan nokta değerlerini doğrudan karşılaştırmayın; bunun yerine bir delta kullanın.

#include <float.h> // for DBL_EPSILON and FLT_EPSILON
#include <math.h>  // for fabs()

int main(void)
{
    double a = 0.1; // imprecise: (binary) 0.000110...

    // may be false or true
    if (a + a + a + a + a + a + a + a + a + a == 1.0) {
        printf("10 * 0.1 is indeed 1.0. This is not guaranteed in the general case.\n");
    }

    // Using a small delta value.
    if (fabs(a + a + a + a + a + a + a + a + a + a - 1.0) < 0.000001) {
        // C99 5.2.4.2.2p8 guarantees at least 10 decimal digits
        // of precision for the double type.
        printf("10 * 0.1 is almost 1.0.\n");
    }

    return 0;
}

static inline double rel_diff(double a, double b)
{
    return fabs(a - b) / fmax(fabs(a), fabs(b));
}

}

Kitaplık işlevlerinin dönüş değerlerini yok sayma #

C standart kitaplığındaki hemen hemen her işlev, başarılı bir şey ve hata hakkında başka bir şey döndürür. Örneğin, “malloc”, başarılı olduğunda işlev tarafından ayrılan bellek bloğuna bir işaretçi ve işlev istenen bellek bloğunu tahsis edemezse bir boş işaretçi döndürür. Bu nedenle, daha kolay hata ayıklama için her zaman dönüş değerini kontrol etmelisiniz.

Bu kötü:

char* x = malloc(100000000000UL * sizeof *x);
/* more code */
scanf("%s", x); /* This might invoke undefined behaviour and if lucky causes a segmentation violation, unless your system has a lot of memory */

Bu iyi:

#include <stdlib.h>
#include <stdio.h>

int main(void)
{
    char* x = malloc(100000000000UL * sizeof *x);
    if (x == NULL) {
        perror("malloc() failed");
        exit(EXIT_FAILURE);
    }

    if (scanf("%s", x) != 1) {
        fprintf(stderr, "could not read string\n");
        free(x);
        exit(EXIT_FAILURE);
    }

    /* Do stuff with x. */

    /* Clean up. */
    free(x);

    return EXIT_SUCCESS;
}

Bu şekilde hatanın nedenini hemen bilirsiniz, aksi takdirde tamamen yanlış bir yerde bir hata aramak için saatler harcayabilirsiniz.

Kayan nokta değişmezleri varsayılan olarak double türündedir #

Float türündeki değişkenleri değişmez değerlerle başlatırken veya bunları değişmez değerlerle karşılaştırırken dikkatli olunmalıdır, çünkü “0.1” gibi normal kayan nokta değişmezleri “double” türündedir. Bu sürprizlere yol açabilir:

#include <stdio.h>
int main() {
    float  n;
    n = 0.1;
    if (n > 0.1) printf("Wierd\n");
    return 0;
}
// Prints "Wierd" when n is float

Burada, n başlatılır ve tek kesinliğe yuvarlanır, bu da 0.10000000149011612 değeriyle sonuçlanır. Ardından, “n”, “0.1” değişmez değeriyle (0.100000000000000001’e eşittir) karşılaştırılmak üzere çift duyarlılığa dönüştürülür, bu da bir uyumsuzlukla sonuçlanır.

Yuvarlama hatalarının yanı sıra, “float” değişkenlerini “double” değişmezlerle karıştırmak, çift kesinlik için donanım desteği olmayan platformlarda düşük performansa neden olur.

Dize değişmezleri yerine karakter sabitlerini kullanma veya tam tersi #

C’de karakter sabitleri ve dize değişmezleri farklı şeylerdir.

“a” gibi tek tırnak içine alınmış bir karakter karakter sabitidir. Karakter sabiti, değeri, karakteri temsil eden karakter kodu olan bir tamsayıdır. ‘‘abc’’ gibi birden çok karakter içeren karakter sabitlerinin nasıl yorumlanacağı uygulama tanımlıdır.

“abc”gibi çift tırnak içine alınmış sıfır veya daha fazla karakter bir *dize değişmezidir*. Bir dize değişmezi, öğeleri "char" türünde olan değiştirilemez bir dizidir. Çift tırnak içindeki dize artı boş karakteri sonlandıran içeriktir, bu nedenle“abc” 4 öğeye sahiptir ({‘a’, ‘b’, ‘c’, ‘\0’}`)

Bu örnekte, bir dize değişmezinin kullanılması gereken yerde bir karakter sabiti kullanılır. Bu karakter sabiti, uygulama tanımlı bir şekilde bir işaretçiye dönüştürülecek ve dönüştürülen işaretçinin geçerli olması için çok az şans var, bu nedenle bu örnek tanımsız davranışı çağıracak.

#include <stdio.h>

int main(void) {
    const char *hello = 'hello, world'; /* bad */
    puts(hello);
    return 0;
}

Bu örnekte, bir karakter sabitinin kullanılması gereken yerde bir dize değişmezi kullanılır. Dize değişmezinden dönüştürülen işaretçi, uygulama tanımlı bir şekilde bir tam sayıya dönüştürülür ve uygulama tanımlı bir şekilde ‘char’a dönüştürülür. (Bir tamsayının, dönüştürülecek değeri temsil edemeyen imzalı bir türe nasıl dönüştürüleceği uygulama tarafından tanımlanır ve ‘char’ imzalı olup olmadığı da uygulama tarafından tanımlanır.) Çıktı anlamsız bir şey olacaktır.

#include <stdio.h>

int main(void) {
    char c = "a"; /* bad */
    printf("%c\n", c);
    return 0;
}

Hemen hemen her durumda, derleyici bu karışıklıklardan şikayet edecektir. Olmazsa, daha fazla derleyici uyarı seçeneği kullanmanız gerekir veya daha iyi bir derleyici kullanmanız önerilir.

Özyinelemeli işlev — temel koşulu kaçırma #

Bir sayının faktöriyelini hesaplamak, özyinelemeli bir fonksiyonun klasik bir örneğidir.

Temel Durum Eksik:

#include <stdio.h>

int factorial(int n)
{
       return n * factorial(n - 1);
}

int main()
{
    printf("Factorial %d = %d\n", 3, factorial(3));
    return 0;
}

Tipik çıktı: Bölümleme hatası: 11

Bu işlevin sorunu, sonsuz döngüde olması ve segmentasyon hatasına neden olmasıdır - özyinelemeyi durdurmak için temel bir koşula ihtiyacı vardır.

Beyan Edilen Temel Durum:

#include <stdio.h>

int factorial(int n)
{
    if (n == 1) // Base Condition, very crucial in designing the recursive functions.
    {
       return 1;
    }
    else
    {
       return n * factorial(n - 1);
    }
}

int main()
{
    printf("Factorial %d = %d\n", 3, factorial(3));
    return 0;
}

Örnek çıktı

Factorial 3 = 6

Bu işlev, “n” eşittir 1 koşuluna ulaştığı anda sona erecektir (“n"nin başlangıç ​​değerinin yeterince küçük olması koşuluyla - “int” 32 bitlik bir miktar olduğunda üst sınır “12"dir).

Uyulacak kurallar:

  1. Algoritmayı başlatın. Özyinelemeli programlar genellikle başlamak için bir tohum değerine ihtiyaç duyar. Bu, ya işleve geçirilen bir parametre kullanılarak ya da özyinelemeli olmayan ancak özyinelemeli hesaplama için çekirdek değerleri ayarlayan bir ağ geçidi işlevi sağlayarak gerçekleştirilir.
  2. İşlenmekte olan mevcut değer(ler)in temel durumla eşleşip eşleşmediğini kontrol edin. Eğer öyleyse, değeri işleyin ve döndürün.
  3. Cevabı daha küçük veya daha basit bir alt problem veya alt problemler açısından yeniden tanımlayın.
  4. Algoritmayı alt problem üzerinde çalıştırın.
  5. Cevabın formülasyonunda sonuçları birleştirin.
  6. Sonuçları döndürün.

Bitişik olmayan dizileri “gerçek” çok boyutlu diziler bekleyen işlevlere geçirme #

“malloc”, “calloc” ve “realloc” ile çok boyutlu dizileri tahsis ederken, ortak bir kalıp, iç dizileri birden çok çağrıya tahsis etmektir (çağrı yalnızca bir kez görünse bile, bir döngüde olabilir):

/* Could also be `int **` with malloc used to allocate outer array. */
int *array[4];
int i;

/* Allocate 4 arrays of 16 ints. */
for (i = 0; i < 4; i++)
    array[i] = malloc(16 * sizeof(*array[i]));

İç dizilerden birinin son öğesi ile bir sonraki iç dizinin ilk öğesi arasındaki bayt farkı, “gerçek” çok boyutlu bir dizide olduğu gibi 0 olmayabilir (örneğin, int dizisi[4][16]; ):

/* 0x40003c, 0x402000 */
printf("%p, %p\n", (void *)(array[0] + 15), (void *)array[1]);

‘int’ boyutunu dikkate alarak, 2032 ‘int’ boyutlu dizi öğeleri olan 8128 baytlık (8132-4) bir fark elde edersiniz ve sorun da budur: “gerçek” çok boyutlu bir dizinin aralarında boşluk yoktur. elementler.

“Gerçek” çok boyutlu bir dizi bekleyen bir işleve sahip dinamik olarak ayrılmış bir dizi kullanmanız gerekiyorsa, “int *” türünde bir nesne ayırmalı ve hesaplamaları gerçekleştirmek için aritmetik kullanmalısınız:

void func(int M, int N, int *array);
...

/* Equivalent to declaring `int array[M][N] = {{0}};` and assigning to array4_16[i][j]. */
int *array;
int M = 4, N = 16;
array = calloc(M, N * sizeof(*array));
array[i * N + j] = 1;
func(M, N, array);

“N” bir değişken yerine bir makro veya bir tamsayı sabit değeriyse, kod bir diziye bir işaretçi ayırdıktan sonra daha doğal 2 boyutlu dizi gösterimini kullanabilir:

void func(int M, int N, int *array);
#define N 16
void func_N(int M, int (*array)[N]);
...

int M = 4;
int (*array)[N];
array = calloc(M, sizeof(*array));
array[i][j] = 1;

/* Cast to `int *` works here because `array` is a single block of M*N ints with no gaps,
   just like `int array2[M * N];` and `int array3[M][N];` would be. */
func(M, N, (int *)array);
func_N(M, array);

“N” bir makro veya bir tamsayı değişmezi değilse, “dizi” değişken uzunluklu bir diziye (VLA) işaret edecektir. Bu, ‘int *’ işlevine dönüştürülerek ‘func’ ile hala kullanılabilir ve yeni bir ‘func_vla’ işlevi, ‘func_N’ yerine geçer:

void func(int M, int N, int *array);
void func_vla(int M, int N, int array[M][N]);
...

int M = 4, N = 16;
int (*array)[N];
array = calloc(M, sizeof(*array));
array[i][j] = 1;
func(M, N, (int *)array);
func_vla(M, N, array);

Not: VLA’lar C11’den itibaren isteğe bağlıdır. Uygulamanız C11’i destekliyorsa ve ‘STDC_NO_VLA’ makrosunu 1 olarak tanımlarsa, C99 öncesi yöntemlere takılıp kalırsınız.

Dizi sınırlarını aşma #

Diziler sıfır tabanlıdır, yani dizin her zaman 0’dan başlar ve dizin dizi uzunluğu eksi 1 ile biter. Bu nedenle, aşağıdaki kod dizinin ilk öğesini çıkarmayacak ve yazdırdığı son değer için çöp çıktısı verecektir.

#include <stdio.h>

int main(void)
{
    int x = 0;
    int myArray[5] = {1, 2, 3, 4, 5}; //Declaring 5 elements

    for(x = 1; x <= 5; x++) //Looping from 1 till 5.
       printf("%d\t", myArray[x]);

    printf("\n");
    return 0;
}

Çıktı: ‘2 3 4 5 GarbageValue’

Aşağıda istenen çıktıyı elde etmenin doğru yolu gösterilmektedir:

#include <stdio.h>

int main(void)
{

    for(x = 0; x < 5; x++) //Looping from 0 till 4.
