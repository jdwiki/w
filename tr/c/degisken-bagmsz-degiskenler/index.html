Değişken argümanlar, printf ailesindeki (printf, fprintf, vb.) ve diğerleri tarafından bir fonksiyonun her seferinde farklı sayıda argümanla çağrılmasına izin vermek için kullanılır, bu nedenle varargs adı verilir.

Değişken argümanları özelliğini kullanarak işlevleri uygulamak için #include <stdarg.h> kullanın.

Değişken sayıda argüman alan işlevleri çağırmak için, sondaki üç noktanın kapsamda olduğu tam bir prototip olduğundan emin olun: örneğin void err_exit(const char *format, ...);.

Sözdizimi #

  • void va_start(va_list ap, last); /* Değişken argüman işlemeyi başlat; last, üç noktadan (“…”) önceki son işlev parametresidir */
  • type va_arg(va_list ap, type); /* Listedeki bir sonraki değişken argümanı al; doğru promote türünü ilettiğinizden emin olun */
  • void va_end(va_list ap); /* Argüman işlemeyi sonlandır */
  • void va_copy(va_list dst, va_list src); /* C99 veya üstü: argüman listesini, yani argüman işlemedeki mevcut konumu başka bir listeye kopyala (ör. argümanları birden çok kez iletmek için) */

Parametreler #

parametre Ayrıntılar
“va_list” ap argüman işaretçisi, değişken argümanlar listesindeki geçerli konum
last son değişken olmayan işlev argümanının adı, böylece derleyici değişken argümanları işlemeye başlamak için doğru yeri bulur; bir “kayıt” değişkeni, bir işlev veya bir dizi türü olarak bildirilemez
type terfi edilmiş okunacak değişken argüman türü (örneğin, bir “kısa int” argümanı için “int”)
“va_list” src kopyalamak için geçerli argüman işaretçisi
“va_list” dst doldurulacak yeni argüman listesi

“va_start”, “va_arg”, “va_end” ve “va_copy” işlevleri aslında makrolardır.

daima önce ve yalnızca bir kez “va_start"ı çağırdığınızdan ve işlevin her çıkış noktasında en son ve yalnızca bir kez “va_end"i çağırdığınızdan emin olun. Bunu yapmamak sizin sisteminizde işleyebilir ama kesinlikle taşınabilir değildir** ve bu nedenle hataları davet eder.

İşlevinizi doğru bir şekilde, yani bir prototiple beyan etmeye özen gösterin ve değişken olmayan last argümanındaki kısıtlamalara dikkat edin (bir işlev veya dizi türü değil “kaydet” değil). Argüman işlemeyi başlatmak için en az bir değişken olmayan argüman gerektiğinden, yalnızca değişken argümanlar alan bir fonksiyon bildirmek mümkün değildir.

“va_arg"ı çağırırken, terfi edilmiş argüman tipini talep etmelisiniz, yani:

  • ‘short’, ‘int’ olarak yükseltilir (ve ‘sizeof(unsigned short) == sizeof(int)’ olmadıkça, ‘unsigned short’ da ‘int’ olarak yükseltilir, bu durumda ‘unsigned int’ olarak yükseltilir) .
  • “yüzdürme”, “çift” olarak yükseltilir.
  • “imzalı karakter”, “int” olarak yükseltilir; ‘unsigned char’ ayrıca ‘sizeof(unsigned char) == sizeof(int)’ olmadıkça ‘int’ olarak yükseltilir, ki bu nadiren olur.
  • “char” genellikle “int” olarak yükseltilir.
  • “uint8_t” veya “int16_t” gibi C99 türleri benzer şekilde tanıtılır.

Tarihi (yani K&R) değişken argüman işleme, “<varargs.h>” içinde bildirilir, ancak eski olduğu için kullanılmamalıdır. Standart değişken argüman işleme (burada açıklanan ve <stdarg.h> içinde bildirilen) C89’da tanıtıldı; ‘va_copy’ makrosu C99’da tanıtıldı, ancak ondan önce birçok derleyici tarafından sağlandı.

va_list’in uzunluğunu belirlemek için açık bir sayım bağımsız değişkeni kullanma #

Herhangi bir değişken işlevde, işlev değişken bağımsız değişkenler listesini nasıl yorumlayacağını bilmelidir. ‘printf()’ veya ‘scanf()’ işlevleriyle, biçim dizgisi işleve ne bekleyeceğini söyler.

En basit teknik, diğer argümanların (normalde hepsi aynı türde olan) açık bir sayısını iletmektir. Bu, herhangi bir sayıda tamsayı olabilir ancak bu sayının değişken bağımsız değişken listesinden önce bir bağımsız değişken olarak belirtildiği bir tam sayı dizisinin toplamını hesaplayan aşağıdaki koddaki değişken işlevinde gösterilmiştir.

#include <stdio.h>
#include <stdarg.h>

/* first arg is the number of following int args to sum. */
int sum(int n, ...) {
    int sum = 0;
    va_list it; /* hold information about the variadic argument list. */

    va_start(it, n); /* start variadic argument processing */
    while (n--)
      sum += va_arg(it, int); /* get and sum the next variadic argument */
    va_end(it); /* end variadic argument processing */

    return sum;
}

int main(void)
{
    printf("%d\n", sum(5, 1, 2, 3, 4, 5)); /* prints 15 */
    printf("%d\n", sum(10, 5, 9, 2, 5, 111, 6666, 42, 1, 43, -6218)); /* prints 666 */
    return 0;
}

va_list’in sonunu belirlemek için sonlandırıcı değerleri kullanma #

Herhangi bir değişken işlevde, işlev değişken bağımsız değişkenler listesini nasıl yorumlayacağını bilmelidir. “Geleneksel” yaklaşım (‘printf’ ile örneklendirilmiştir) argüman sayısını önceden belirtmektir. Ancak, bu her zaman iyi bir fikir değildir:

/* First argument specifies the number of parameters; the remainder are also int */
extern int sum(int n, ...);

/* But it's far from obvious from the code. */
sum(5, 2, 1, 4, 3, 6)

/* What happens if i.e. one argument is removed later on? */
sum(5, 2, 1, 3, 6)  /* Disaster */

Bazen, POSIX execlp() ile örneklenen açık bir sonlandırıcı eklemek daha sağlamdır. işlev. İşte bir dizi “çift” sayının toplamını hesaplamak için başka bir fonksiyon:

#include <stdarg.h>
#include <stdio.h>
#include <math.h>

/* Sums args up until the terminator NAN */
double sum (double x, ...) {
  double sum = 0;
  va_list va;

  va_start(va, x);
  for (; !isnan(x); x = va_arg(va, double)) {
    sum += x;
  }
  va_end(va);

  return sum;
}

int main (void) {
  printf("%g\n", sum(5., 2., 1., 4., 3., 6., NAN));
  printf("%g\n", sum(1, 0.5, 0.25, 0.125, 0.0625, 0.03125, NAN));
}

İyi sonlandırıcı değerleri:

  • tamsayı (tümü pozitif veya negatif olmaması gerekir) — “0” veya “-1”
  • kayan nokta türleri — NAN
  • işaretçi türleri — “NULL”
  • numaralandırıcı türleri - bazı özel değerler

printf() benzeri bir arayüzle işlevlerin uygulanması #

Değişken uzunluklu argüman listelerinin yaygın bir kullanımı, işlevleri uygulamaktır. bunlar, “printf()” işlevler ailesinin etrafındaki ince bir sarmalayıcıdır. Böyle bir örnek, bir dizi hata raporlama işlevidir.

errmsg.h

#ifndef ERRMSG_H_INCLUDED
#define ERRMSG_H_INCLUDED

#include <stdarg.h>
#include <stdnoreturn.h>    // C11

void verrmsg(int errnum, const char *fmt, va_list ap);
noreturn void errmsg(int exitcode, int errnum, const char *fmt, ...);
void warnmsg(int errnum, const char *fmt, ...);

#endif

Bu çıplak bir örnektir; bu tür paketler çok ayrıntılı olabilir. Normalde, programcılar ya errmsg()’ ya da ‘warnmsg()’ kullanırlar, bunlar da kendi içlerinde ‘verrmsg()’ kullanır. Yine de birisi daha fazlasını yapma ihtiyacı duyarsa, açıkta kalan ‘verrmsg()’ işlevi yararlı olacaktır. İhtiyacınız olana kadar ifşa etmekten kaçınabilirsiniz (YAGNI — buna ihtiyacınız olmayacak), ancak ihtiyaç eninde sonunda ortaya çıkacaktır ( ihtiyacınız olacak - YAGNI).

errmsg.c

Bu kodun, standart hata çıktısı almak için yalnızca değişken argümanları vfprintf() işlevine iletmesi gerekir. Ayrıca, işlevlere iletilen sistem hata numarasına (errno) karşılık gelen sistem hata mesajını da bildirir.

#include "errmsg.h"
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void
verrmsg(int errnum, const char *fmt, va_list ap)
{
    if (fmt)
        vfprintf(stderr, fmt, ap);
    if (errnum != 0)
        fprintf(stderr, ": %s", strerror(errnum));
    putc('\n', stderr);
}

void
errmsg(int exitcode, int errnum, const char *fmt, ...)
{
    va_list ap;
    va_start(ap, fmt);
    verrmsg(errnum, fmt, ap);
    va_end(ap);
    exit(exitcode);
}

void
warnmsg(int errnum, const char *fmt, ...)
{
    va_list ap;
    va_start(ap, fmt);
    verrmsg(errnum, fmt, ap);
    va_end(ap);
}

errmsg.h kullanma

Şimdi bu işlevleri aşağıdaki gibi kullanabilirsiniz:

#include "errmsg.h"
#include <errno.h>
#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main(int argc, char **argv)
{
    char buffer[BUFSIZ];
    int fd;
    if (argc != 2)
    {
        fprintf(stderr, "Usage: %s filename\n", argv[0]);
        exit(EXIT_FAILURE);
    }
    const char *filename = argv[1];

    if ((fd = open(filename, O_RDONLY)) == -1)
        errmsg(EXIT_FAILURE, errno, "cannot open %s", filename);
    if (read(fd, buffer, sizeof(buffer)) != sizeof(buffer))
        errmsg(EXIT_FAILURE, errno, "cannot read %zu bytes from %s", sizeof(buffer), filename);
    if (close(fd) == -1)
        warnmsg(errno, "cannot close %s", filename);
    /* continue the program */
    return 0;
}

eğer open() veya read() sistem çağrıları başarısız olur, hata standart hataya yazılır ve program çıkış kodu 1 ile çıkar. Eğer close() sistem çağrısı başarısız olur, hata yalnızca bir uyarı mesajı olarak yazdırılır ve program devam ediyor.

printf() biçimlerinin doğru kullanımını kontrol etme

GCC (GNU’nun bir parçası olan GNU C Derleyicisi) kullanıyorsanız Derleyici Koleksiyonu) veya Clang’ı kullanarak derleyiciye sahip olabilirsiniz. hata mesajı işlevlerine ilettiğiniz argümanların eşleştiğini kontrol edin printf() ne bekliyor. Tüm derleyiciler uzantıyı desteklemediğinden derlenmesi gerekir. şartlı olarak, ki bu biraz zor. Ancak sağladığı koruma çabaya değer.

İlk olarak, derleyicinin GCC veya Clang olduğunu nasıl tespit edeceğimizi bilmemiz gerekiyor. GCC’yi taklit ediyor. Cevap, GCC’nin bunu belirtmek için __GNUC__ tanımlamasıdır.

Bkz. ortak işlev nitelikler nitelikler hakkında bilgi için - özellikle “biçim” bağlanmak.

Yeniden yazılmış ’errmsg.h’

#ifndef ERRMSG_H_INCLUDED
#define ERRMSG_H_INCLUDED

#include <stdarg.h>
#include <stdnoreturn.h>    // C11

#if !defined(PRINTFLIKE)
#if defined(__GNUC__)
#define PRINTFLIKE(n,m) __attribute__((format(printf,n,m)))
#else
#define PRINTFLIKE(n,m) /* If only */
#endif /* __GNUC__ */
#endif /* PRINTFLIKE */

void verrmsg(int errnum, const char *fmt, va_list ap);
void noreturn errmsg(int exitcode, int errnum, const char *fmt, ...)
        PRINTFLIKE(3, 4);
void warnmsg(int errnum, const char *fmt, ...)
        PRINTFLIKE(2, 3);

#endif

Şimdi, şöyle bir hata yaparsanız:

errmsg(EXIT_FAILURE, errno, "Failed to open file '%d' for reading", filename);

("%d"nin “%s” olması gerektiği yerde), o zaman derleyici şikayet eder:

$ gcc -O3 -g -std=c11 -Wall -Wextra -Werror -Wmissing-prototypes -Wstrict-prototypes \
>     -Wold-style-definition -c erruse.c
erruse.c: In function ‘main’:
erruse.c:20:64: error: format ‘%d’ expects argument of type ‘int’, but argument 4 has type ‘const char *’ [-Werror=format=]
         errmsg(EXIT_FAILURE, errno, "Failed to open file '%d' for reading", filename);
                                                           ~^
                                                           %s
cc1: all warnings being treated as errors
$

Biçim dizesi kullanma #

Bir biçim dizgisinin kullanılması, açık bir sayım bağımsız değişkenine veya bir sonlandırıcı değerine olan ihtiyacı ortadan kaldıracak şekilde, sonraki değişken bağımsız değişkenlerin beklenen sayısı ve türü hakkında bilgi sağlar.

Aşağıdaki örnek, standart “printf()” işlevini saran ve yalnızca “char”, “int” ve “double” (ondalık kayan nokta biçiminde) türündeki değişken bağımsız değişkenlerin kullanımına izin veren bir işlevi göstermektedir. Burada, printf() ile olduğu gibi, sarma fonksiyonunun ilk argümanı format dizgisidir. Biçim dizgisi ayrıştırıldığında, işlev, beklenen başka bir değişken argüman olup olmadığını ve türünün ne olması gerektiğini belirleyebilir.

#include <stdio.h>
#include <stdarg.h>

int simple_printf(const char *format, ...)
{
    va_list ap; /* hold information about the variadic argument list. */
    int printed = 0; /* count of printed characters */

    va_start(ap, format); /* start variadic argument processing */

    while (*format != '\0') /* read format string until string terminator */
    {
        int f = 0;

        if (*format == '%')
        {
            ++format;
            switch(*format)
            {
                case 'c' :
                    f = printf("%d", va_arg(ap, int)); /* print next variadic argument, note type promotion from char to int */
                    break;
                case 'd' :
                    f = printf("%d", va_arg(ap, int)); /* print next variadic argument */
                    break;

                case 'f' :
                    f = printf("%f", va_arg(ap, double)); /* print next variadic argument */
                    break;
                default :
                    f = -1; /* invalid format specifier */
                    break;
            }
        }
        else
        {
            f = printf("%c", *format); /* print any other characters */
        }

        if (f < 0) /* check for errors */
        {
            printed = f;
            break;
        }
        else
        {
            printed += f;
        }
        ++format; /* move on to next character in string */
    }

    va_end(ap); /* end variadic argument processing */

    return printed;
}

int main (int argc, char *argv[])
{
    int x = 40;
    int y = 0;

    y = simple_printf("There are %d characters in this sentence", x);
    simple_printf("\n%d were printed\n", y);
}
