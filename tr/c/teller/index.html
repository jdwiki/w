C’de bir dize içsel bir tür değildir. Bir C-dizesi, bir boş karakterle, bir “\0” ile sonlandırılan tek boyutlu bir karakter dizisine sahip olma kuralıdır.

Bu, “abc”’ içeriğine sahip bir C-string’in “a”, “b”, “c” ve “\0” olmak üzere dört karaktere sahip olacağı anlamına gelir.

dizelere temel giriş örneğine bakın.

Sözdizimi #

  • char str1[] = “Merhaba dünya!”; /* Değiştirilebilir */
  • char str2[14] = “Merhaba dünya!”; /* Değiştirilebilir */
  • char* str3 = “Merhaba dünya!”; /* Modifiye edilemez*/

Simgeleştirme: strtok(), strtok_r() ve strtok_s() #

‘strtok’ işlevi, bir dizi sınırlayıcı kullanarak bir diziyi daha küçük dizilere veya belirteçlere ayırır.

#include <stdio.h>
#include <string.h>

int main(void)
{
    int toknum = 0;
    char src[] = "Hello,, world!";
    const char delimiters[] = ", !";
    char *token = strtok(src, delimiters);
    while (token != NULL)
    {
        printf("%d: [%s]\n", ++toknum, token);
        token = strtok(NULL, delimiters);
    }
    /* source is now "Hello\0, world\0\0" */
}

Çıktı:

1: [Hello]
2: [world]

Sınırlayıcılar dizisi bir veya daha fazla sınırlayıcı içerebilir ve her strtok çağrısıyla farklı sınırlayıcı diziler kullanılabilir.

Aynı kaynak dizgiyi simgeleştirmeye devam etmek için strtok’a yapılan çağrılar, kaynak dizgiyi tekrar iletmemeli, bunun yerine ilk argüman olarak ‘NULL’ iletmelidir. Aynı kaynak dize * iletilirse, bunun yerine ilk simge yeniden belirtilecektir. Diğer bir deyişle, aynı sınırlayıcılar verildiğinde, “strtok” sadece ilk belirteci tekrar döndürür.

“strtok” belirteçler için yeni bellek ayırmadığından kaynak dizesini değiştirir. Yani, yukarıdaki örnekte, strtok’a yapılan çağrılar tarafından döndürülen işaretçi tarafından başvurulan belirteçleri üretmek için ‘src’ dizgisi manipüle edilecektir. Bu, kaynak dizenin “const” olamayacağı anlamına gelir (bu nedenle, bir dize değişmezi olamaz). Bu ayrıca sınırlayıcı baytın kimliğinin kaybolduğu anlamına gelir (yani örnekte “,” ve “!” kaynak dizeden etkin bir şekilde silinir ve hangi sınırlayıcı karakterin eşleştiğini söyleyemezsiniz).

Ayrıca, kaynak dizedeki birden çok ardışık sınırlayıcının tek olarak ele alındığını unutmayın; örnekte, ikinci virgül yok sayılır.

“strtok”, ayrıştırma sırasında statik bir arabellek kullandığından, ne iş parçacığı güvenlidir ne de yeniden girişlidir. Bu, bir işlev strtok’u çağırırsa, strtok’u kullanırken çağırdığı hiçbir işlevin strtok’u kullanamayacağı ve kendisi strtok’u kullanan herhangi bir işlev tarafından çağrılamayacağı anlamına gelir.

strtok’un yeniden giriş yapmamasının neden olduğu sorunları gösteren bir örnek aşağıdaki gibidir:

char src[] = "1.2,3.5,4.2";
char *first = strtok(src, ","); 

do 
{
    char *part;
    /* Nested calls to strtok do not work as desired */
    printf("[%s]\n", first);
    part = strtok(first, ".");
    while (part != NULL)
    {
        printf(" [%s]\n", part);
        part = strtok(NULL, ".");
    }
} while ((first = strtok(NULL, ",")) != NULL);

Çıktı:

[1.2]
 [1]
 [2]

Beklenen işlem, dış “do while” döngüsünün, her biri için “strtok” olan her bir ondalık sayı dizesinden (“1.2”, “3.5”, “4.2"”) oluşan üç simge oluşturmasıdır. iç döngü için yapılan çağrılar onu ayrı rakam dizgilerine bölmelidir ("1", "2", "3", "5", "4", "2").

Ancak, “strtok” yeniden giriş yapmadığı için bu gerçekleşmez. Bunun yerine, ilk “strtok”, “1.2\0” belirtecini doğru bir şekilde oluşturur ve iç döngü, doğru şekilde ““1” ve "2" belirteçlerini oluşturur. Ancak dış döngüdeki strtok, iç döngü tarafından kullanılan dizgenin sonundadır ve hemen NULL değerini döndürür. “src” dizisinin ikinci ve üçüncü alt dizileri hiç analiz edilmez.

Standart C kitaplıkları iş parçacığı için güvenli veya yeniden giriş yapan bir sürüm içermez, ancak POSIX’ [strtok_r](http://pubs.opengroup.org/onlinepubs/9699919799/functions/strtok_r.html gibi) bazıları içerir. ). MSVC’de “strtok” eşdeğerinin, “strtok_s"nin iş parçacığı için güvenli olduğunu unutmayın.

son sürüm

C11, “strtok_s” adlı iş parçacığı güvenli ve yeniden girişli bir sürüm sunan Ek K adlı isteğe bağlı bir bölüme sahiptir. Özelliği __STDC_LIB_EXT1__ ile test edebilirsiniz. Bu isteğe bağlı kısım yaygın olarak desteklenmemektedir.

“strtok_s” işlevi, belirtilecek dizenin dışında depolamaya karşı koruma sağlaması ve çalışma zamanı kısıtlamalarını kontrol etmesiyle POSIX “strtok_r” işlevinden farklıdır. Doğru yazılmış programlarda ise strtok_s ve strtok_r aynı şekilde davranır.

Örnekte “strtok_s” kullanmak, şu şekilde doğru yanıtı verir:

/* you have to announce that you want to use Annex K */ 
#define __STDC_WANT_LIB_EXT1__ 1
#include <string.h>

#ifndef __STDC_LIB_EXT1__
# error "we need strtok_s from Annex K"
#endif

char src[] = "1.2,3.5,4.2";  
char *next = NULL;
char *first = strtok_s(src, ",", &next);

do 
{
    char *part;
    char *posn;

    printf("[%s]\n", first);
    part = strtok_s(first, ".", &posn);
    while (part != NULL)
    {
        printf(" [%s]\n", part);
        part = strtok_s(NULL, ".", &posn);
    }
} 
while ((first = strtok_s(NULL, ",", &next)) != NULL);

Ve çıktı şöyle olacaktır:

[1.2]
 [1]
 [2]
[3.5]
 [3]
 [5]
[4.2]
 [4]
 [2]
son sürüm

Dize değişmezleri #

Dize değişmezleri, boş sonlandırılmış, static-duration “char” dizilerini temsil eder. Statik depolama sürelerine sahip oldukları için, bir dize değişmez değeri veya aynı temel diziye yönelik bir işaretçi, otomatik bir diziye yönelik bir işaretçinin yapamayacağı çeşitli şekillerde güvenle kullanılabilir. Örneğin, bir işlevden bir dize değişmezi döndürmenin iyi tanımlanmış davranışı vardır:

const char *get_hello() {
    return "Hello, World!";  /* safe */
}

Tarihsel nedenlerden dolayı, bir dize değişmezine karşılık gelen dizinin öğeleri resmi olarak “const” değildir. Bununla birlikte, bunları değiştirmeye yönelik herhangi bir girişimde tanımsız davranış bulunur. Tipik olarak, bir dize değişmezine karşılık gelen diziyi değiştirmeye çalışan bir program çöker veya başka bir şekilde arızalanır.

char *foo = "hello";
foo[0] = 'y';  /* Undefined behavior - BAD! */

Bir işaretçi bir dize değişmezine işaret ettiğinde - veya bazen yapabileceği yerde - yanlışlıkla bu tür tanımsız davranışlara girmekten kaçınmak için işaretçinin referansını “const” olarak bildirmeniz önerilir.

const char *foo = "hello";
/* GOOD: can't modify the string pointed to by foo */

Öte yandan, bir dize değişmezinin altında yatan diziye veya bunun içine bir işaretçi, doğası gereği özel değildir; değeri başka bir şeye işaret edecek şekilde serbestçe değiştirilebilir:

char *foo = "hello";
foo = "World!"; /* OK - we're just changing what foo points to */

Ayrıca, “char” dizileri için başlatıcılar, dize değişmezleri ile aynı forma sahip olabilse de, böyle bir başlatıcının kullanımı, başlatılan dizide bir dize değişmezinin özelliklerini vermez. Başlatıcı, dizinin uzunluğunu ve ilk içeriğini belirtir. Özellikle, açıkça “const” olarak bildirilmemişse öğeler değiştirilebilir:

char foo[] = "hello";
foo[0] = 'y';  /* OK! */

Uzunluğu Hesapla: strlen() #

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(int argc, char **argv) 
{
    /* Exit if no second argument is found. */
    if (argc != 2) 
    {
        puts("Argument missing.");
        return EXIT_FAILURE;
    }

    size_t len = strlen(argv[1]);
    printf("The length of the second argument is %zu.\n", len);

    return EXIT_SUCCESS;
}

Bu program, ikinci girdi argümanının uzunluğunu hesaplar ve sonucu “len"de saklar. Daha sonra bu uzunluğu terminale yazdırır. Örneğin, program_name "Merhaba dünya!" parametreleriyle çalıştırıldığında, program İkinci argümanın uzunluğu 13. sonucunu verecektir çünkü Merhaba, dünya! dizesi 13 karakter uzunluğundadır.

“strlen”, dizenin başlangıcından sonlandırıcı NUL karakteri “\0”‘a kadar olan tüm baytları sayar. Bu nedenle, yalnızca dizenin NUL ile sonlandırılacağı garanti edildiğinde kullanılabilir.

Ayrıca, dize herhangi bir Unicode karakter içeriyorsa, strlen’in size dizede kaç karakter olduğunu söylemeyeceğini unutmayın (çünkü bazı karakterler birden çok bayt uzunluğunda olabilir). Bu gibi durumlarda karakterleri (yani, kod birimleri) kendiniz saymanız gerekir. Aşağıdaki örneğin çıktısını göz önünde bulundurun:

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(void) 
{
    char asciiString[50] = "Hello world!";
    char utf8String[50] = "Γειά σου Κόσμε!"; /* "Hello World!" in Greek */

    printf("asciiString has %zu bytes in the array\n", sizeof(asciiString));
    printf("utf8String has %zu bytes in the array\n", sizeof(utf8String));
    printf("\"%s\" is %zu bytes\n", asciiString, strlen(asciiString));
    printf("\"%s\" is %zu bytes\n", utf8String, strlen(utf8String));
}

Çıktı:

asciiString has 50 bytes in the array
utf8String has 50 bytes in the array
"Hello world!" is 12 bytes
"Γειά σου Κόσμε!" is 27 bytes

Dizelere temel giriş #

C’de bir dize, bir boş karakterle (’\0’) sonlandırılan bir karakter dizisidir.

Çift tırnak içine alınmış karakter dizileri olan dize değişmezlerini kullanarak dizeler oluşturabiliriz; örneğin, "merhaba dünya" dizgesini alın. Dize değişmezleri otomatik olarak boş sonlandırılır.

Birkaç yöntem kullanarak dizeler oluşturabiliriz. Örneğin, bir “char *” bildirebilir ve onu bir dizgenin ilk karakterine işaret edecek şekilde başlatabiliriz:

char * string = "hello world";

Yukarıdaki gibi bir dize sabitine “char *” başlatılırken, dizenin kendisi genellikle salt okunur verilerde ayrılır; “dize”, “h” karakteri olan dizinin ilk öğesini gösteren bir işaretçidir.

Dize değişmezi salt okunur bellekte ayrıldığından, değiştirilemez1. Bunu değiştirmeye yönelik herhangi bir girişim tanımsız davranışa yol açacaktır, bu nedenle böyle bir derleme zamanı hatası almak için “const” eklemek daha iyidir

char const * string = "hello world";

Benzer bir etkiye sahiptir2

char const string_arr[] = "hello world";

Değiştirilebilir bir dize oluşturmak için, bir karakter dizisi bildirebilir ve içeriğini şöyle bir dize değişmezi kullanarak başlatabilirsiniz:

char modifiable_string[] = "hello world";

Bu, aşağıdakilere eşdeğerdir:

char modifiable_string[] = {'h', 'e', 'l', 'l', 'o', ' ', 'w', 'o', 'r', 'l', 'd', '\0'};

İkinci sürüm ayraç içine alınmış başlatıcı kullandığından, karakter dizisine genellikle son öğesi olarak bir “\0” karakteri açıkça dahil edilmedikçe dize otomatik olarak boş sonlandırılmaz.

1 Değiştirilemez, dize değişmezindeki karakterlerin değiştirilemeyeceği anlamına gelir, ancak “dize” işaretçisinin değiştirilebileceğini unutmayın (başka bir yere işaret edebilir veya artırılabilir veya azaltılabilir).

2 Her iki dizenin karakterlerinin değiştirilemeyeceği anlamında her iki dize de benzer etkiye sahiptir. “dize"nin “char” için bir işaretçi olduğuna ve değiştirilebilir bir l-değeri olduğuna dikkat edilmelidir, bu nedenle artırılabilir veya “string_arr” dizisi değiştirilemez bir l değeriyken başka bir konumu işaret edin, değiştirilemez.

Dizeleri kopyalama #

İşaretçi atamaları dizeleri kopyalamaz #

Tam sayıları kopyalamak için = operatörünü kullanabilirsiniz, ancak C’de dizeleri kopyalamak için = operatörünü kullanamazsınız. C’deki dizeler, sonlandırıcı bir boş karaktere sahip karakter dizileri olarak temsil edilir, bu nedenle = operatörünün kullanılması yalnızca bir dizenin adresini (işaretçi) kaydeder.

#include <stdio.h>

int main(void) {
    int a = 10, b;
    char c[] = "abc", *d;

    b = a; /* Integer is copied */
    a = 20; /* Modifying a leaves b unchanged - b is a 'deep copy' of a */
    printf("%d %d\n", a, b); /* "20 10" will be printed */

    d = c; 
    /* Only copies the address of the string - 
    there is still only one string stored in memory */
    
    c[1] = 'x';
    /* Modifies the original string - d[1] = 'x' will do exactly the same thing */

    printf("%s %s\n", c, d); /* "axc axc" will be printed */

    return 0;
}

Yukarıdaki örnek, ‘char d[3]’ yerine ‘char *d’ kullandığımız için derlendi. İkincisini kullanmak bir derleyici hatasına neden olur. C’de dizilere atama yapamazsınız.

#include <stdio.h>

int main(void) {
    char a[] = "abc";
    char b[8];

    b = a; /* compile error */
    printf("%s\n", b);

    return 0;
}

Standart işlevleri kullanarak dizeleri kopyalama #

strcpy() #

Dizeleri gerçekten kopyalamak için, strcpy() işlevi string.h içinde mevcuttur. Kopyalamadan önce hedef için yeterli alan ayrılmalıdır.

#include <stdio.h>
#include <string.h>

int main(void) {
    char a[] = "abc";
    char b[8];

    strcpy(b, a); /* think "b special equals a" */
    printf("%s\n", b); /* "abc" will be printed */

    return 0;
}

snprintf() #

Arabellek taşmasını önlemek için snprintf() kullanılabilir. Şablon dizesini ayrıştırması gerektiğinden performans açısından en iyi çözüm değildir, ancak standart kitaplıkta hazır bulunan ve herhangi bir ekstra adım olmadan kullanılabilen dizeleri kopyalamak için arabellek sınırı güvenli tek işlevdir.

#include <stdio.h>
#include <string.h>

int main(void) {
    char a[] = "012345678901234567890";
    char b[8];

#if 0
    strcpy(b, a); /* causes buffer overrun (undefined behavior), so do not execute this here! */
#endif

    snprintf(b, sizeof(b), "%s", a); /* does not cause buffer overrun */
    printf("%s\n", b); /* "0123456" will be printed */

    return 0;
}

strncat() #

Daha iyi performansa sahip ikinci bir seçenek, strncat() kullanmaktır (strcat() öğesinin arabellek taşması denetimi sürümü) ) - kopyalanacak maksimum bayt sayısını söyleyen üçüncü bir argüman alır:

char dest[32];

dest[0] = '\0';
strncat(dest, source, sizeof(dest) - 1);
    /* copies up to the first (sizeof(dest) - 1) elements of source into dest,
    then puts a \0 on the end of dest */

Bu formülasyonun sizeof(hedef) - 1 kullandığına dikkat edin; bu çok önemlidir, çünkü strncat() her zaman boş bir bayt ekler (iyi), ancak bunu dize boyutunda saymaz (karışıklık ve arabellek üzerine yazma nedeni).

Ayrıca, boş olmayan bir dizeden sonra birleştirme alternatifinin daha da karmaşık olduğunu unutmayın. Düşünmek:

char dst[24] = "Clownfish: ";
char src[] = "Marvin and Nemo";
size_t len = strlen(dst);

strncat(dst, src, sizeof(dst) - len - 1);
printf("%zu: [%s]\n", strlen(dst), dst);

Çıktı:

23: [Clownfish: Marvin and N]

Bununla birlikte, uzunluk olarak belirtilen boyutun, hedef dizinin boyutu değil, terminal boş baytını hesaba katmadan, içinde kalan boşluk miktarı olduğuna dikkat edin. Bu, üzerine büyük yazma sorunlarına neden olabilir. Aynı zamanda biraz israftır; uzunluk argümanını doğru bir şekilde belirlemek için hedefteki verinin uzunluğunu bilirsiniz, bu nedenle mevcut içeriğin sonunda boş baytın adresini belirtebilir ve strncat()ı yeniden taramaktan kurtarabilirsiniz:

    strcpy(dst, "Clownfish: ");
    assert(len < sizeof(dst) - 1);
    strncat(dst + len, src, sizeof(dst) - len - 1);
    printf("%zu: [%s]\n", strlen(dst), dst);

Bu, öncekiyle aynı çıktıyı üretir, ancak “strncat()”, kopyalamaya başlamadan önce “dst"nin mevcut içeriğini taramak zorunda değildir.

strncpy() #

Son seçenek strncpy() işlevidir. Önce gelmesi gerektiğini düşünseniz de, iki ana soruna sahip olan oldukça aldatıcı bir işlevdir:

  1. strncpy() yoluyla kopyalama arabellek sınırına ulaşırsa, sonlandırıcı bir boş karakter yazılmaz.
  2. strncpy() hedefi her zaman, gerekirse boş baytlarla tamamen doldurur.

(Bu tür tuhaf uygulama tarihseldir ve başlangıçta UNIX dosya adlarını işlemek için tasarlanmıştır)

Bunu kullanmanın tek doğru yolu, boş sonlandırmayı manuel olarak sağlamaktır:

strncpy(b, a, sizeof(b)); /* the third parameter is destination buffer size */
b[sizeof(b)/sizeof(*b) - 1] = '\0'; /* terminate the string */
printf("%s\n", b); /* "0123456" will be printed */

O zaman bile, büyük bir arabelleğiniz varsa, ek boş doldurma nedeniyle strncpy() kullanmak çok verimsiz hale gelir.

Bir Dizedeki Karakterler Üzerinde Yineleme #

Dizenin uzunluğunu biliyorsak, karakterlerini yinelemek için bir for döngüsü kullanabiliriz:

char * string = "hello world"; /* This 11 chars long, excluding the 0-terminator. */
size_t i = 0;
for (; i < 11; i++) {
    printf("%c\n", string[i]);    /* Print each character of the string. */
}

Alternatif olarak, eğer dizgenin ne olduğunu bilmiyorsak, bir dizgenin uzunluğunu almak için standart strlen() işlevini kullanabiliriz:

size_t length = strlen(string);
size_t i = 0; 
for (; i < length; i++) {
    printf("%c\n", string[i]);    /* Print each character of the string. */
}

Son olarak, C’deki dizgilerin boş sonlandırılmalarının garanti edilmesi gerçeğinden yararlanabiliriz (önceki örnekte ;-) ‘strlen()’e geçerken bunu zaten yapmıştık). Boyutundan bağımsız olarak diziyi yineleyebilir ve bir boş karaktere ulaştığımızda yinelemeyi durdurabiliriz:

size_t i = 0;
while (string[i] != '\0') {       /* Stop looping when we reach the null-character. */
    printf("%c\n", string[i]);    /* Print each character of the string. */
    i++;
}

Dize Dizileri Oluşturma #

Bir dizi dizi birkaç anlama gelebilir:

  1. Elemanları char * olan bir dizi
  2. Elemanları ‘char’ dizileri olan bir dizi

Bunun gibi bir dizi karakter işaretçisi oluşturabiliriz:

char * string_array[] = {
    "foo",
    "bar",
    "baz"
};

Unutmayın: char *‘a dize değişmezleri atadığımızda, dizelerin kendileri salt okunur bellekte ayrılır. Ancak, string_array dizisi okuma/yazma hafızasında tahsis edilmiştir. Bu, dizideki işaretçileri değiştirebileceğimiz, ancak işaret ettikleri dizeleri değiştiremeyeceğimiz anlamına gelir.

C’de, ana “argv” parametresi (program çalıştırıldığında geçirilen komut satırı argümanları dizisi) bir “char *”: “char * argv[]” dizisidir.

Ayrıca karakter dizileri dizileri de oluşturabiliriz. Dizeler karakter dizileri olduğundan, bir dizi dizisi basitçe öğeleri karakter dizileri olan bir dizidir:

char modifiable_string_array_literals[][4] = {
    "foo",
    "bar",
    "baz"
};

Bu şuna eşdeğerdir:

char modifiable_string_array[][4] = {
    {'f', 'o', 'o', '\0'},
    {'b', 'a', 'r', '\0'},
    {'b', 'a', 'z', '\0'}
};

Dizinin ikinci boyutunun boyutu olarak ‘4’ belirttiğimize dikkat edin; dizimizdeki dizelerin her biri aslında 4 bayttır, çünkü boş sonlandırma karakterini dahil etmemiz gerekir.

Dizeleri Sayıya Dönüştür: atoi(), atof() (tehlikeli, onları kullanmayın) #

Uyarı: “atoi”, “atol”, “atoll” ve “atof” işlevleri doğası gereği güvenli değildir, çünkü: [Sonucun değeri gösterilemiyorsa, davranış tanımsızdır.](http://www. open-std.org/jtc1/sc22/wg14/www/docs/n1124.pdf) (7.20.1p1)

#include <stdio.h>
#include <stdlib.h>

int main(int argc, char** argv)
{
    int val;
    if (argc < 2)
    {
        printf("Usage: %s <integer>\n", argv[0]);
        return 0;
    }

    val = atoi(argv[1]);

    printf("String value = %s, Int value = %d\n", argv[1], val);

    return 0;
}

Dönüştürülecek dize, aralıktaki geçerli bir ondalık tam sayı olduğunda işlev çalışır:

$ ./atoi 100
String value = 100, Int value = 100
$ ./atoi 200
String value = 200, Int value = 200

Bir sayı ile başlayan ve ardından başka bir şey gelen dizeler için yalnızca ilk sayı ayrıştırılır:

$ ./atoi 0x200
0
$ ./atoi 0123x300
123

Diğer tüm durumlarda, davranış tanımsızdır:

$ ./atoi hello
Formatting the hard disk...

Yukarıdaki belirsizlikler ve bu tanımsız davranış nedeniyle, ‘atoi’ işlevler ailesi asla kullanılmamalıdır.

  • ’long int’e dönüştürmek için ‘atol()’ yerine ‘strtol()’ kullanın.
  • ‘double’a dönüştürmek için ‘atof()’ yerine ‘strtod()’ kullanın.
  • ’long long int’e dönüştürmek için ‘atoll()’ yerine ‘strtoll()’ kullanın.

Biçimlendirilmiş verileri dizgeye yaz

int sprintf ( char * str, const char * format, ... );

dizeye kayan veri yazmak için sprintf işlevini kullanın.

#include <stdio.h>
int main ()
{
  char buffer [50];
  double PI = 3.1415926;
  sprintf (buffer, "PI = %.7f", PI);
  printf ("%s\n",buffer);
  return 0;
}

Dizeden biçimlendirilmiş verileri oku

int sscanf ( const char * s, const char * format, ...);

biçimlendirilmiş verileri ayrıştırmak için sscanf işlevini kullanın.

#include <stdio.h>
int main ()
{
  char sentence []="date : 06-06-2012";
  char str [50];
  int year;
  int month;
  int day;
  sscanf (sentence,"%s : %2d-%2d-%4d", str, &day, &month, &year);
  printf ("%s -> %02d-%02d-%4d\n",str, day, month, year);
  return 0;
}

Kopyalama ve Birleştirme: strcpy(), strcat() #

#include <stdio.h>
#include <string.h>

int main(void)
{
  /* Always ensure that your string is large enough to contain the characters
   * and a terminating NUL character ('\0')!
   */
  char mystring[10];

  /* Copy "foo" into `mystring`, until a NUL character is encountered. */
  strcpy(mystring, "foo");
  printf("%s\n", mystring);

  /* At this point, we used 4 chars of `mystring`, the 3 characters of "foo",
   * and the NUL terminating byte.
   */

  /* Append "bar" to `mystring`. */
  strcat(mystring, "bar");
  printf("%s\n", mystring);

  /* We now use 7 characters of `mystring`: "foo" requires 3, "bar" requires 3
   * and there is a terminating NUL character ('\0') at the end.
   */

  /* Copy "bar" into `mystring`, overwriting the former contents. */
  strcpy(mystring, "bar");
  printf("%s\n", mystring);

  return 0;
}

Çıktılar:

foo
foobar
bar

Mevcut bir dizeye ekleme yapıyorsanız veya mevcut bir dizeden kopyalarsanız, bunun NUL ile sonlandırıldığından emin olun!

Dize değişmezleri (örneğin, “foo”`) derleyici tarafından her zaman NUL ile sonlandırılacaktır.

Karşılaştırma: strcmp(), strncmp(), strcasecmp(), strncasecmp() #

strcase*-işlevleri Standart C değil, bir POSIX uzantısıdır.

‘strcmp’ işlevi, iki boş sonlandırılmış karakter dizisini sözlükbilimsel olarak karşılaştırır. İşlevler, sözlükbilimsel sırada ilk bağımsız değişken ikinciden önce görünüyorsa negatif bir değer, eşit olarak karşılaştırılırsa sıfır veya sözlükbilimsel sırada ilk bağımsız değişken ikinciden sonra görünüyorsa pozitif bir değer döndürür.

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs)
{
    int result = strcmp(lhs, rhs); // compute comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "BBB");
    compare("BBB", "CCCCC");
    compare("BBB", "AAAAAA");
    return 0;
}

Çıktılar:

BBB equals BBB
BBB comes before CCCCC
BBB comes after AAAAAA

“strcmp” gibi, “strcasecmp” işlevi de her karakteri küçük harf karşılıklarına çevirdikten sonra argümanlarını sözlükbilimsel olarak karşılaştırır:

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs)
{
    int result = strcasecmp(lhs, rhs); // compute case-insensitive comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "bBB");
    compare("BBB", "ccCCC");
    compare("BBB", "aaaaaa");
    return 0;
}

Çıktılar:

BBB equals bBB
BBB comes before ccCCC
BBB comes after aaaaaa

“strncmp” ve “strncasecmp” en fazla n karakter karşılaştırır:

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs, int n)
{
    int result = strncmp(lhs, rhs, n); // compute comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "Bb", 1);
    compare("BBB", "Bb", 2);
    compare("BBB", "Bb", 3);
    return 0;
}

Çıktılar:

BBB equals Bb
BBB comes before Bb
BBB comes before Bb

Dizeleri güvenle Sayıya dönüştürün: strtoX işlevleri #

C99’dan beri C kitaplığı, bir diziyi sayı olarak yorumlayan bir dizi güvenli dönüştürme işlevine sahiptir. Adları “strtoX” biçimindedir; burada “X”, dönüşümün hedef türünü belirlemek için “l”, “ul”, “d"den biridir.

double strtod(char const* p, char** endptr);
long double strtold(char const* p, char** endptr);

Bir dönüşümün aşırı veya yetersiz olduğunun kontrol edilmesini sağlarlar:

double ret = strtod(argv[1], 0); /* attempt conversion */

/* check the conversion result. */
if ((ret == HUGE_VAL || ret == -HUGE_VAL) && errno == ERANGE) 
    return;  /* numeric overflow in in string */
else if (ret == HUGE_VAL && errno == ERANGE) 
    return; /* numeric underflow in in string */

/* At this point we know that everything went fine so ret may be used */

Dize aslında hiç sayı içermiyorsa, bu “strtod” kullanımı “0.0” değerini döndürür.

Bu tatmin edici değilse, “endptr” ek parametresi kullanılabilir. Dizedeki algılanan sayının sonuna işaret edilecek bir işaretçiden işaretçiye. Yukarıdaki gibi “0” veya “NULL” olarak ayarlanırsa, basitçe yoksayılır.

Bu “endptr” parametresi, başarılı bir dönüşüm olup olmadığını ve öyleyse sayının nerede bittiğini gösterir:

char *check = 0;
double ret = strtod(argv[1], &check); /* attempt conversion */

/* check the conversion result. */
if (argv[1] == check) 
    return; /* No number was detected in string */
else if ((ret == HUGE_VAL || ret == -HUGE_VAL) && errno == ERANGE) 
    return; /* numeric overflow in in string */
else if (ret == HUGE_VAL && errno == ERANGE) 
    return; /* numeric underflow in in string */

/* At this point we know that everything went fine so ret may be used */

Daha geniş tamsayı türlerine dönüştürmek için benzer işlevler vardır:

long strtol(char const* p, char** endptr, int nbase);
long long strtoll(char const* p, char** endptr, int nbase);
unsigned long strtoul(char const* p, char** endptr, int nbase);
unsigned long long strtoull(char const* p, char** endptr, int nbase);

Bu işlevler, sayıyı tutan üçüncü bir “nbase” parametresine sahiptir. sayının yazıldığı taban.

long a = strtol("101",   0, 2 ); /* a = 5L */
long b = strtol("101",   0, 8 ); /* b = 65L */
long c = strtol("101",   0, 10); /* c = 101L */
long d = strtol("101",   0, 16); /* d = 257L */
long e = strtol("101",   0, 0 ); /* e = 101L */
long f = strtol("0101",  0, 0 ); /* f = 65L */
long g = strtol("0x101", 0, 0 ); /* g = 257L */

“nbase” için özel “0” değeri, dizenin, bir C programında sayı değişmezlerinin yorumlandığı şekilde yorumlandığı anlamına gelir: “0x” ön eki onaltılık gösterime karşılık gelir, aksi takdirde baştaki “0” sekizliktir ve diğer tüm sayılar ondalık olarak görülür.

Bu nedenle, bir komut satırı argümanını sayı olarak yorumlamanın en pratik yolu şudur:

int main(int argc, char* argv[] {
    if (argc < 1)
        return EXIT_FAILURE; /* No number given. */

    /* use strtoull because size_t may be wide */
    size_t mySize = strtoull(argv[1], 0, 0);

    /* then check conversion results. */

     ...

    return EXIT_SUCCESS;
}

Bu, programın sekizli, ondalık veya onaltılık bir parametre ile çağrılabileceği anlamına gelir.

strstr #

/* finds the next instance of needle in haystack 
   zbpos: the zero-based position to begin searching from
   haystack: the string to search in
   needle: the string that must be found
   returns the next match of `needle` in `haystack`, or -1 if not found
*/
int findnext(int zbpos, const char *haystack, const char *needle)
{
    char *p; 

    if (((p = strstr(haystack + zbpos, needle)) != NULL)
        return p - haystack;

    return -1;
}

“strstr”, “iğne” ile gösterilen dize için “samanlık” (ilk) bağımsız değişkenini arar. Bulunursa, “strstr”, olayın adresini döndürür. Eğer “iğne” bulamazsa, NULL döndürür. Aynı iğneyi tekrar tekrar bulmamak için zbpos kullanıyoruz. İlk örneği atlamak için bir zbpos ofseti ekliyoruz. Bir Not Defteri klonu, “Sonrakini Bul” diyaloğunu uygulamak için şöyle “findnext” adını verebilir:

/*
    Called when the user clicks "Find Next"
    doc: The text of the document to search
    findwhat: The string to find
*/
void onfindnext(const char *doc, const char *findwhat)
{
    static int i;

    if ((i = findnext(i, doc, findwhat)) != -1)
        /* select the text starting from i and ending at i + strlen(findwhat) */
    else
        /* display a message box saying "end of search" */
}
