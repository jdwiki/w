Numaralandırmalar, “enum” anahtar sözcüğünden ve isteğe bağlı bir tanımlayıcıdan ve ardından parantez içine alınmış bir sayılayıcı-listesinden oluşur.

Bir tanımlayıcı ‘int’ türündedir.

enumerator-list en az bir enumerator öğesine sahiptir.

Bir numaralayıcıya isteğe bağlı olarak “int” türünde sabit bir ifade “atanabilir”.

Sayılayıcı sabittir ve bir “char”, işaretli bir tamsayı veya işaretsiz bir tamsayı ile uyumludur. Hangisi kullanılırsa, uygulama tanımlı olur. Her durumda, kullanılan tür, söz konusu numaralandırma için tanımlanan tüm değerleri temsil edebilmelidir.

Bir enumerator öğesine hiçbir sabit ifade “atanmamış"sa ve bu, enumerator-list içindeki 1st girişiyse, ‘0’ değerini alır, aksi takdirde * Numaralandırıcı listesindeki * önceki giriş artı 1.

Birden çok “atama” kullanmak, aynı numaralandırmanın farklı * numaralandırıcılarının* aynı değerleri taşımasına neden olabilir.

Yinelenen değerle numaralandırma #

Bir numaralandırma değerinin hiçbir şekilde benzersiz olması gerekmez:

#include <stdlib.h> /* for EXIT_SUCCESS */
#include <stdio.h> /* for printf() */


enum Dupes
{
   Base, /* Takes 0 */
   One, /* Takes Base + 1 */
   Two, /* Takes One + 1 */
   Negative = -1,
   AnotherZero /* Takes Negative + 1 == 0, sigh */
};

int main(void)
{
  printf("Base = %d\n", Base);
  printf("One = %d\n", One);
  printf("Two = %d\n", Two);
  printf("Negative = %d\n", Negative);
  printf("AnotherZero = %d\n", AnotherZero);

  return EXIT_SUCCESS;
}

Örnek baskılar:

Base = 0
One = 1
Two = 2
Negative = -1
AnotherZero = 0

typename olmadan numaralandırma sabiti #

Numaralandırma türleri, ad verilmeden de bildirilebilir:

  enum { buffersize = 256, };
  static unsigned char buffer [buffersize] = { 0 };

Bu, bu örnekte olduğu gibi dizi uzunluğu olarak kullanılabilen ‘int’ türünde derleme zaman sabitlerini tanımlamamızı sağlar.

Basit Numaralandırma #

Numaralandırma, kullanıcı tanımlı bir veri türüdür ve integral sabitlerinden oluşur ve her integral sabitine bir ad verilir. Numaralandırılmış veri tipini tanımlamak için “enum” anahtar kelimesi kullanılır.

‘int’ veya ‘string/char*’ yerine ’enum’ kullanırsanız, derleme zamanı denetimini artırır ve geçersiz sabitlerin geçmesinden kaynaklanan hataları önlersiniz ve hangi değerlerin kullanılmasının yasal olduğunu belgelersiniz.

örnek 1 #

enum color{ RED, GREEN, BLUE };

void printColor(enum color chosenColor)
{
    const char *color_name = "Invalid color";
    switch (chosenColor)
    {
       case RED:
         color_name = "RED";
         break;
       
       case GREEN:
        color_name = "GREEN";
        break;    

       case BLUE:
        color_name = "BLUE";
        break;
    }
    printf("%s\n", color_name);
}

Aşağıdaki gibi tanımlanmış bir ana işlevle (örneğin):

int main(){
    enum color chosenColor;
    printf("Enter a number between 0 and 2");
    scanf("%d", (int*)&chosenColor);
    printColor(chosenColor);
    return 0;
}

Örnek 2 #

(Bu örnekte, C99’dan beri standartlaştırılmış atanmış başlatıcılar kullanılmaktadır.)

enum week{ MON, TUE, WED, THU, FRI, SAT, SUN };
     
static const char* const dow[] = { 
  [MON] = "Mon", [TUE] = "Tue", [WED] = "Wed", 
  [THU] = "Thu", [FRI] = "Fri", [SAT] = "Sat", [SUN] = "Sun" };
    
void printDayOfWeek(enum week day) 
{ 
   printf("%s\n", dow[day]);
}

Aralık kontrolünü kullanan aynı örnek:

enum week{ DOW_INVALID = -1, 
  MON, TUE, WED, THU, FRI, SAT, SUN, 
  DOW_MAX };
     
static const char* const dow[] = { 
  [MON] = "Mon", [TUE] = "Tue", [WED] = "Wed", 
  [THU] = "Thu", [FRI] = "Fri", [SAT] = "Sat", [SUN] = "Sun" };
    
void printDayOfWeek(enum week day) 
{ 
   assert(day > DOW_INVALID && day < DOW_MAX);
   printf("%s\n", dow[day]);
}

Typedef numaralandırma #

Bir numaralandırmayı adlandırmak için çeşitli olasılıklar ve kurallar vardır. İlki, “enum” anahtar sözcüğünden hemen sonra bir etiket adı kullanmaktır.

enum color
{ 
    RED, 
    GREEN, 
    BLUE 
};

Bu numaralandırma daha sonra her zaman ve etiketiyle şu şekilde kullanılmalıdır:

enum color chosenColor = RED;

Eğer ’enum’u bildirirken doğrudan ’typedef’ kullanırsak, etiket adını atlayabilir ve ardından türü ’enum’ anahtar kelimesi olmadan kullanabiliriz:

typedef enum 
{ 
    RED, 
    GREEN, 
    BLUE 
} color;

color chosenColor = RED;

Ama bu son durumda onu ’enum color’ olarak kullanamayız, çünkü tanımda etiket adını kullanmadık. Ortak bir kural, aynı adın “enum” anahtar sözcüğüyle veya anahtar sözcüğü olmadan kullanılabilmesi için her ikisini de kullanmaktır. Bu, C++ ile uyumlu olma avantajına sahiptir.

enum color                /* as in the first example */
{ 
    RED, 
    GREEN, 
    BLUE 
};
typedef enum color color; /* also a typedef of same identifier */

color chosenColor  = RED;
enum color defaultColor = BLUE;

İşlev:

void printColor()
{
    if (chosenColor == RED)
    {
        printf("RED\n");
    }
    else if (chosenColor == GREEN)
    {
        printf("GREEN\n");    
    }
    else if (chosenColor == BLUE)
    {
        printf("BLUE\n");
    }
}

’typedef’ hakkında daha fazla bilgi için bkz. https://www.wikiod.com/tr/docs/c/2681/typedef
