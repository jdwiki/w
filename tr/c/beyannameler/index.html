Nesneye veya işleve atıfta bulunan tanımlayıcı bildirimi, genellikle kısaca yalnızca bir nesne veya işlev bildirimi olarak adlandırılır.

Başka bir C dosyasından bir işlevi çağırmak #

foo.h

#ifndef FOO_DOT_H    /* This is an "include guard" */
#define FOO_DOT_H    /* prevents the file from being included twice. */
                     /* Including a header file twice causes all kinds */
                     /* of interesting problems.*/

/**
 * This is a function declaration.
 * It tells the compiler that the function exists somewhere.
 */
void foo(int id, char *name);

#endif /* FOO_DOT_H */

foo.c

#include "foo.h"    /* Always include the header file that declares something
                     * in the C file that defines it. This makes sure that the
                     * declaration and definition are always in-sync.  Put this
                     * header first in foo.c to ensure the header is self-contained.
                     */
#include <stdio.h>
                       
/**
 * This is the function definition.
 * It is the actual body of the function which was declared elsewhere.
 */
void foo(int id, char *name)
{
    fprintf(stderr, "foo(%d, \"%s\");\n", id, name);
    /* This will print how foo was called to stderr - standard error.
     * e.g., foo(42, "Hi!") will print `foo(42, "Hi!")`
     */
}

main.c

#include "foo.h"

int main(void)
{
    foo(42, "bar");
    return 0;
}

Derle ve Bağla

İlk olarak, hem “foo.c” hem de “main.c” dosyalarını nesne dosyalarına *derliyoruz. Burada gcc derleyicisini kullanıyoruz, derleyicinizin adı farklı olabilir ve başka seçeneklere ihtiyacı olabilir.

$ gcc -Wall -c foo.c
$ gcc -Wall -c main.c

Şimdi, son yürütülebilir dosyamızı üretmek için onları birbirine bağlarız:

$ gcc -o testprogram foo.o main.o

Global Değişken Kullanma #

Giriiş #

Beyanname örnekleri şunlardır:

int a; /* declaring single identifier of type int */

Yukarıdaki bildirim, ‘int’ tipindeki bazı nesnelere atıfta bulunan ‘a’ adında tek bir tanımlayıcı bildirir.

int a1, b1; /* declaring 2 identifiers of type int */

İkinci bildirim, aynı “int” tipine sahip olsa da diğer bazı nesnelere atıfta bulunan “a1” ve “b1” adlı 2 tanımlayıcı bildirir.

Temel olarak, bunun çalışma şekli şu şekildedir - önce bazı type koyarsınız, sonra virgül (,) ile ayrılmış tek veya çoklu ifadeler yazarsınız (bu noktada değerlendirilmeyecektir - ve aksi takdirde bu bağlamda beyan edici olarak anılacaktır). Bu tür ifadeleri yazarken, bazı tanımlayıcılara yalnızca dolaylı (*), işlev çağrısı (( )) veya alt simge (veya dizi indeksleme - [ ]) operatörlerini uygulamanıza izin verilir (ayrıca herhangi bir tanımlayıcı kullanamazsınız). operatörler). Kullanılan tanımlayıcının geçerli kapsamda görünür olması gerekli değildir. Bazı örnekler:

/* 1 */ int /* 2 */ (*z) /* 3 */ , /* 4 */ *x , /* 5 */ **c /* 6 */ ;
# Açıklama
1 Tamsayı türünün adı.
2 Değerlendirilmemiş ifade, bazı “z” tanımlayıcılarına dolaylı yoldan aktarma uyguluyor.
3 Aynı bildirimde bir ifadenin daha geleceğini belirten bir virgülümüz var.
4 Değerlendirilmemiş ifade, başka bir `x’ tanımlayıcısına dolaylı yoldan uygulama uyguluyor.
5 Değerlendirilmemiş ifade, (*c) ifadesinin değerine dolaylı olarak uygulanıyor.
6 Bildirimin sonu.

Yukarıdaki tanımlayıcıların hiçbirinin bu bildirimden önce görünür olmadığına ve bu nedenle kullanılan ifadelerin ondan önce geçerli olmayacağına dikkat edin.

Bu tür her ifadeden sonra, içinde kullanılan tanımlayıcı geçerli kapsama dahil edilir. (Tanımlayıcı kendisine bağlantı atadıysa, her iki tanımlayıcının da aynı nesneye veya işleve atıfta bulunması için aynı bağlantı türüyle yeniden bildirilebilir)

Ayrıca, başlatma için eşittir operatör işareti (=) kullanılabilir. Değerlendirilmemiş bir ifadenin (bildirici) ardından bildirim içinde = geliyorsa, tanıtılan tanımlayıcının da başlatıldığını söyleriz. = işaretinden sonra bir kez daha bazı ifadeler koyabiliriz, ancak bu sefer değerlendirilecek ve değeri bildirilen nesne için başlangıç ​​olarak kullanılacaktır.

Örnekler:

int l = 90; /* the same as: */

int l; l = 90; /* if it the declaration of l was in block scope */

int c = 2, b[c]; /* ok, equivalent to: */

int c = 2; int b[c];

Kodunuzun ilerleyen kısımlarında, yeni tanıtılan tanımlayıcının bildirim bölümündeki aynı ifadeyi yazmanıza izin verilir ve tüm erişimlere geçerli değerler atadığınızı varsayarak size bildirimin başında belirtilen türde bir nesne verir. şekilde nesneler. Örnekler:

void f()
{
    int b2; /* you should be able to write later in your code b2 
            which will directly refer to the integer object
            that b2 identifies */
    
    b2 = 2; /* assign a value to b2 */
    
    printf("%d", b2); /*ok - should print 2*/

    int *b3; /* you should be able to write later in your code *b3 */

    b3 = &b2; /* assign valid pointer value to b3 */

    printf("%d", *b3); /* ok - should print 2 */

    int **b4; /* you should be able to write later in your code **b4 */

    b4 = &b3;

    printf("%d", **b4); /* ok - should print 2 */

    void (*p)(); /* you should be able to write later in your code (*p)() */

    p = &f; /* assign a valid pointer value */

    (*p)(); /* ok - calls function f by retrieving the
            pointer value inside p -    p
            and dereferencing it -      *p
            resulting in a function
            which is then called -      (*p)() -

            it is not *p() because else first the () operator is 
            applied to p and then the resulting void object is
            dereferenced which is not what we want here */
}

“b3” bildirimi, bir tamsayı nesnesine erişmek için potansiyel olarak “b3” değerini bir ortalama olarak kullanabileceğinizi belirtir.

Elbette, “b3"e dolaylı (*) uygulamak için, içinde uygun bir değere de sahip olmanız gerekir (daha fazla bilgi için işaretçiler’e bakın). Ayrıca, onu almaya çalışmadan önce bir nesneye bir miktar değer kaydetmelisiniz (bu sorun hakkında daha fazla bilgiyi burada görebilirsiniz). Bunların hepsini yukarıdaki örneklerde yaptık.

int a3(); /* you should be able to call a3 */

Bu, derleyiciye ‘a3’ çağırmayı deneyeceğinizi söyler. Bu durumda ‘a3’ bir nesne yerine işlevi ifade eder. Nesne ve işlev arasındaki bir fark, işlevlerin her zaman bir tür bağlantıya sahip olmasıdır. Örnekler:

void f1()
{
    {
        int f2(); /* 1 refers to some function f2 */
    }
    
    {
        int f2(); /* refers to the exact same function f2 as (1) */
    }
}

Yukarıdaki örnekte, 2 bildirim aynı “f2” işlevine atıfta bulunurken, bu bağlamda nesneleri bildirselerdi (2 farklı blok kapsamına sahip), 2 farklı farklı nesne olurdu.

int (*a3)(); /* you should be able to apply indirection to `a3` and then call it */

Şimdi karmaşıklaşıyor gibi görünebilir, ancak operatörlerin önceliğini biliyorsanız, yukarıdaki bildirimi okurken 0 sorun yaşarsınız. Parantezler gereklidir çünkü * operatörü ( ) operatöründen daha az önceliğe sahiptir.

Alt simge operatörünün kullanılması durumunda, sonuç ifadesi bildirimden sonra gerçekten geçerli olmaz çünkü içinde kullanılan dizin (”[’ ve ‘]’ içindeki değer) her zaman bunun için izin verilen maksimum değerin 1 üzerinde olacaktır. nesne/fonksiyon.

int a4[5]; /* here a4 shouldn't be accessed using the index 5 later on */

Ancak 5’ten daha düşük diğer tüm dizinler tarafından erişilebilir olmalıdır. Örnekler:

a4[0], a4[1]; a4[4];

“a4[5]” UB ile sonuçlanacaktır. Diziler hakkında daha fazla bilgi burada bulunabilir.

int (*a5)[5](); /* here a4 could be applied indirection
                indexed up to (but not including) 5
                and called */

Ne yazık ki bizim için, sözdizimsel olarak mümkün olmasına rağmen, mevcut standart tarafından ‘a5’ bildirimi yasaklanmıştır.

Genel Sabitleri Kullanma #

Tür tanımlı #

Typedef’ler, typedef’in başında ve türünden önce anahtar kelimesine sahip olan bildirimlerdir. Örneğin.:

typedef int (*(*t0)())[5];

(* teknik olarak typedef’i türden sonra da koyabilirsiniz - bu int typedef (*(*t0)())[5]; gibi ama bu önerilmez*)

Yukarıdaki bildirimler, typedef adı için bir tanımlayıcı bildirir. Daha sonra şu şekilde kullanabilirsiniz:

t0 pf;

Hangisi yazmakla aynı etkiye sahip olacak:

int (*(*pf)())[5];

Gördüğünüz gibi typedef adı, bildirimi daha sonra diğer bildirimler için kullanmak üzere bir tür olarak “kaydeder”. Bu şekilde bazı tuş vuruşlarını kaydedebilirsiniz. Ayrıca ’typedef’ kullanan bildirim hala bir bildirim olduğundan, yalnızca yukarıdaki örnekle sınırlı değilsiniz:

t0 (*pf1);

Aynıdır:

int (*(**pf1)())[5];

C bildirimini deşifre etmek için sağ-sol veya spiral kuralı kullanma #

“Sağ-sol” kuralı, C bildirimlerini deşifre etmek için tamamen düzenli bir kuraldır. Bunları oluşturmada da yararlı olabilir.

Açıklamada karşılaştığınız sembolleri okuyun…

*   as "pointer to"          - always on the left side
[]  as "array of"            - always on the right side
()  as "function returning"  - always on the right side

Kural nasıl uygulanır

AŞAMA 1

Tanımlayıcıyı bulun. Bu senin başlangıç ​​noktan. Sonra kendinize “tanımlayıcıdır” deyin. Beyannameye başladın.

ADIM 2

Tanımlayıcının sağındaki sembollere bakın. Diyelim ki orada () bulursanız, bunun bir fonksiyonun bildirimi olduğunu bilirsiniz. Böylece “tanımlayıcı işlev döndürüyor” olur. Veya orada bir [] bulduysanız, “tanımlayıcı şu dizidir” derdiniz. Semboller bitene kadar VEYA sağ parantez ) tuşuna basana kadar sağa devam edin. (Eğer bir sol parantez (, bu bir () sembolünün başlangıcıdır, parantezler arasında bir şeyler olsa bile. Aşağıda bununla ilgili daha fazla bilgi bulabilirsiniz.)

AŞAMA 3

Tanımlayıcının solundaki sembollere bakın. Yukarıdaki sembollerimizden biri değilse (“int” gibi bir şey söyleyin), sadece söyleyin. Aksi takdirde, yukarıdaki tabloyu kullanarak İngilizce’ye çevirin. Semboller bitene kadar VEYA bir sol parantez (.

Şimdi beyanınızı oluşturana kadar 2. ve 3. adımları tekrarlayın.

İşte bazı örnekler:

int *p[];

İlk önce, tanımlayıcıyı bulun:

int *p[];
     ^

“p”

Şimdi, sembollerin dışına çıkana veya sağ parantez isabetine kadar sağa hareket edin.

int *p[];
      ^^

“p,” dizisidir

Artık sağa hareket edemiyor (semboller dışında), bu yüzden sola hareket edin ve şunu bulun:

int *p[];
    ^

“p, işaretçi dizisidir”

Sola devam edin ve şunu bulun:

int *p[];
^^^

“p, int’ye işaretçi dizisidir”.

(veya “p, her öğenin int’ye işaretçi türünde olduğu bir dizidir”)

Başka bir örnek:

int *(*func())();

Tanımlayıcıyı bulun.

int *(*func())();
       ^^^^

“işlev”

Sağa hareket et.

int *(*func())();
           ^^

“işlev, işlevin geri dönmesidir”

Sağ parantez nedeniyle artık sağa hareket edemiyor, bu yüzden sola hareket ettirin.

int *(*func())();
      ^

“işlev, işaretçiyi döndüren işlevdir”

Sol parantez nedeniyle artık sola hareket edemiyor, bu yüzden sağa devam edin.

int *(*func())();
              ^^

“işlev, işlev döndürme işaretçisidir”

Artık sağa hareket edemiyoruz çünkü sembollerimiz tükendi, o yüzden sola gidin.

int *(*func())();
    ^

“işlev, işaretçiyi işleve döndüren işaretçidir”

Ve son olarak, soldan devam edin, çünkü sağda hiçbir şey kalmadı.

int *(*func())();
^^^

“işlev, işaretçiyi işleve döndüren işaretçiyi int’ye döndüren işlevdir”.

Gördüğünüz gibi, bu kural oldukça yararlı olabilir. Beyannameler oluştururken kendinizi akıl sağlığınızı kontrol etmek ve bir sonraki sembolü nereye koyacağınız ve parantezlerin gerekli olup olmadığı konusunda size bir ipucu vermek için de kullanabilirsiniz.

Bazı bildirimler, prototip biçimindeki dizi boyutları ve argüman listeleri nedeniyle olduğundan çok daha karmaşık görünüyor. [3] görürseniz, bu “dizi (boyut 3) of…” şeklinde okunur. (char *,int) ifadesini görürseniz, bu *“işlev bekliyor (char ,int) ve geri dönüyor…” şeklinde okunur.

İşte eğlenceli bir tane:

int (*(*fun_one)(char *,double))[9][20];

Bunu deşifre etmek için her adımdan geçmeyeceğim.

*“fun_one işlevi bekleyen (char ,double) ve int dizisinin (boyut 20) dizisine (boyut 9) döndüren işaretçidir.”

Gördüğünüz gibi dizi boyutlarından kurtulursanız o kadar karmaşık değil. ve argüman listeleri:

int (*(*fun_one)())[][];

Bu şekilde deşifre edebilir ve daha sonra dizi boyutlarını ve argüman listelerini koyabilirsiniz.

Bazı son sözler:

Bu kuralı kullanarak yasa dışı bildirimler yapmak oldukça mümkündür, bu nedenle C’de neyin yasal olduğu hakkında biraz bilgi gereklidir. Örneğin, yukarıdakiler olsaydı:

int *((*fun_one)())[][];

“fun_one is pointer for pointer dizisini int’e döndüren işlev için işaretçidir” okurdu. Bir işlev bir dizi döndüremeyeceğinden, yalnızca bir diziye bir işaretçi döndürdüğünden, bu bildirim geçersizdir.

Yasadışı kombinasyonlar şunları içerir:

[]() - cannot have an array of functions
()() - cannot have a function that returns a function
()[] - cannot have a function that returns an array

Yukarıdaki tüm durumlarda, beyanın yasal olması için bu () ve [] sağ taraftaki semboller arasında soldaki bir * sembolünü bağlamak için bir parantez setine ihtiyacınız olacaktır.

İşte birkaç örnek daha:

Yasal

int i;               an int
int *p;              an int pointer (ptr to an int)
int a[];             an array of ints
int f();             a function returning an int
int **pp;            a pointer to an int pointer (ptr to a ptr to an int)
int (*pa)[];         a pointer to an array of ints
int (*pf)();         a pointer to a function returning an int
int *ap[];           an array of int pointers (array of ptrs to ints)
int aa[][];          an array of arrays of ints
int *fp();           a function returning an int pointer
int ***ppp;          a pointer to a pointer to an int pointer
int (**ppa)[];       a pointer to a pointer to an array of ints
int (**ppf)();       a pointer to a pointer to a function returning an int
int *(*pap)[];       a pointer to an array of int pointers
int (*paa)[][];      a pointer to an array of arrays of ints
int *(*pfp)();       a pointer to a function returning an int pointer
int **app[];         an array of pointers to int pointers
int (*apa[])[];      an array of pointers to arrays of ints
int (*apf[])();      an array of pointers to functions returning an int
int *aap[][];        an array of arrays of int pointers
int aaa[][][];       an array of arrays of arrays of int
int **fpp();         a function returning a pointer to an int pointer
int (*fpa())[];      a function returning a pointer to an array of ints
int (*fpf())();      a function returning a pointer to a function returning an int

Yasadışı

int af[]();          an array of functions returning an int
int fa()[];          a function returning an array of ints
int ff()();          a function returning a function returning an int
int (*pfa)()[];      a pointer to a function returning an array of ints
int aaf[][]();       an array of arrays of functions returning an int
int (*paf)[]();      a pointer to a an array of functions returning an int
int (*pff)()();      a pointer to a function returning a function returning an int
int *afp[]();        an array of functions returning int pointers
int afa[]()[];       an array of functions returning an array of ints
int aff[]()();       an array of functions returning functions returning an int
int *fap()[];        a function returning an array of int pointers
int faa()[][];       a function returning an array of arrays of ints
int faf()[]();       a function returning an array of functions returning an int
int *ffp()();        a function returning a function returning an int pointer
