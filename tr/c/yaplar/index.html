Yapılar, çeşitli türlerdeki bir dizi ilgili değişkeni tek bir bellek biriminde gruplandırmanın bir yolunu sağlar. Bir bütün olarak yapıya tek bir ad veya işaretçi ile başvurulabilir; yapı üyelerine de ayrı ayrı erişilebilir. Yapılar, işlevlere geçirilebilir ve işlevlerden döndürülebilir. ‘struct’ anahtar sözcüğü kullanılarak tanımlanırlar.

Typedef Yapıları #

“typedef"i “struct” ile birleştirmek, kodu daha net hale getirebilir. Örneğin:

typedef struct 
{
    int x, y;
} Point;

aksine:

struct Point 
{
    int x, y;
};

olarak ilan edilebilir:

Point point;

onun yerine:

struct Point point;

Daha da iyisi aşağıdakileri kullanmaktır

typedef struct Point Point;

struct Point 
{
    int x, y;
};

“nokta"nın her iki olası tanımından da yararlanmak için. Böyle bir bildirim, eğer isim belirsiz değilse struct anahtar sözcüğünü atlayabileceğiniz, önce C++ öğrendiyseniz en uygunudur.

Yapıların ’typedef’ adları, programın diğer bölümlerinin diğer tanımlayıcılarıyla çakışabilir. Bazıları bunu bir dezavantaj olarak görür, ancak çoğu insan için bir “yapı” ve başka bir tanımlayıcıya sahip olmak oldukça rahatsız edicidir. Kötü şöhretli, örneğin POSIX’ ‘stat’

int stat(const char *pathname, struct stat *buf);

burada “struct stat” olan bir argümana sahip bir “stat” işlevi görüyorsunuz.

Bir etiket adı olmayan “typedef” yapıları her zaman “struct” bildiriminin tamamının onu kullanan kod tarafından görülebileceğini zorunlu kılar. Tüm “struct” bildirimi daha sonra bir başlık dosyasına yerleştirilmelidir.

Düşünmek:

#include "bar.h"

struct foo 
{
    bar *aBar;
};

Bu nedenle, etiket adı olmayan bir “typedef"d “struct” ile, “bar.h” dosyasının her zaman “bar"ın tam tanımını içermesi gerekir. eğer kullanırsak

typedef struct bar bar;

‘bar.h’ içinde, ‘bar’ yapısının detayları gizlenebilir.

https://www.wikiod.com/tr/c/typedef adresine bakın

Esnek Dizi Üyeleri #

Tip Bildirimi #

En az bir üyeye sahip bir yapı, ek olarak yapının sonunda belirtilmemiş uzunlukta tek bir dizi üyesi içerebilir. Buna esnek dizi üyesi denir:

struct ex1 
{
    size_t foo;
    int flex[];
};

struct ex2_header 
{
    int foo;
    char bar;
};

struct ex2 
{
    struct ex2_header hdr;
    int flex[];
};

/* Merged ex2_header and ex2 structures. */
struct ex3 
{
    int foo;
    char bar;
    int flex[];
};

Boyut ve Dolgu Üzerindeki Etkiler #

Esnek bir dizi üyesi, bir yapının boyutu hesaplanırken boyutsuz olarak kabul edilir, ancak bu üye ile yapının önceki üyesi arasında dolgu hala mevcut olabilir:

/* Prints "8,8" on my machine, so there is no padding. */
printf("%zu,%zu\n", sizeof(size_t), sizeof(struct ex1));

/* Also prints "8,8" on my machine, so there is no padding in the ex2 structure itself. */
printf("%zu,%zu\n", sizeof(struct ex2_header), sizeof(struct ex2));

/* Prints "5,8" on my machine, so there are 3 bytes of padding. */
printf("%zu,%zu\n", sizeof(int) + sizeof(char), sizeof(struct ex3));

Esnek dizi üyesinin eksik dizi tipine sahip olduğu kabul edilir, bu nedenle boyutu “sizeof” kullanılarak hesaplanamaz.

Kullanım #

Esnek bir dizi üyesi içeren bir yapı tipine sahip bir nesneyi bildirebilir ve başlatabilirsiniz, ancak esnek dizi üyesi yokmuş gibi ele alındığından, esnek dizi üyesini başlatmaya çalışmamalısınız. Bunu yapmaya çalışmak yasaktır ve derleme hataları ortaya çıkar.

Benzer şekilde, bu şekilde bir yapı bildirirken esnek dizi üyesinin herhangi bir öğesine değer atamaya çalışmamalısınız, çünkü esnek dizi üyesinin gerektirdiği nesnelere izin vermek için yapının sonunda yeterli dolgu olmayabilir. Derleyici mutlaka bunu yapmanızı engellemeyecektir, bu nedenle bu tanımsız davranışa yol açabilir.

/* invalid: cannot initialize flexible array member */
struct ex1 e1 = {1, {2, 3}};
/* invalid: hdr={foo=1, bar=2} OK, but cannot initialize flexible array member */
struct ex2 e2 = {{1, 2}, {3}};
/* valid: initialize foo=1, bar=2 members */
struct ex3 e3 = {1, 2};

e1.flex[0] = 3; /* undefined behavior, in my case */
e3.flex[0] = 2; /* undefined behavior again */
e2.flex[0] = e3.flex[0]; /* undefined behavior */

Bunun yerine, yapıyı ekstra depolama ile tahsis etmek ve daha sonra serbest bırakmak için ‘malloc’, ‘calloc’ veya ‘realloc’ kullanmayı seçebilirsiniz, bu da esnek dizi üyesini istediğiniz gibi kullanmanıza olanak tanır:

/* valid: allocate an object of structure type `ex1` along with an array of 2 ints */
struct ex1 *pe1 = malloc(sizeof(*pe1) + 2 * sizeof(pe1->flex[0]));

/* valid: allocate an object of structure type ex2 along with an array of 4 ints */
struct ex2 *pe2 = malloc(sizeof(struct ex2) + sizeof(int[4]));

/* valid: allocate 5 structure type ex3 objects along with an array of 3 ints per object */
struct ex3 *pe3 = malloc(5 * (sizeof(*pe3) + sizeof(int[3])));

pe1->flex[0] = 3; /* valid */
pe3[0]->flex[0] = pe1->flex[0]; /* valid */

‘yapı hack’ #

Esnek dizi üyeleri, C99’dan önce mevcut değildi ve hata olarak kabul ediliyordu. Yaygın bir geçici çözüm, ‘struct hack’ adı verilen bir teknik olan uzunluk 1 dizisini bildirmektir:

struct ex1 
{
    size_t foo;
    int flex[1];
};

Ancak bu, gerçek bir esnek dizi üyesinin aksine yapının boyutunu etkiler:

/* Prints "8,4,16" on my machine, signifying that there are 4 bytes of padding. */
printf("%d,%d,%d\n", (int)sizeof(size_t), (int)sizeof(int[1]), (int)sizeof(struct ex1));

“flex” üyeyi esnek bir dizi üyesi olarak kullanmak için, “sizeof(*pe1)” (veya eşdeğeri “sizeof(struct ex1)”) değiştirilecek olması dışında, onu yukarıda gösterildiği gibi “malloc” ile ayırırsınız. “offsetof(struct ex1, flex)” veya daha uzun, türden bağımsız ifade “sizeof(*pe1)-sizeof(pe1->flex)” ile. Alternatif olarak, istenen uzunluğun 0’dan büyük olduğunu varsayarak, yapı boyutuna zaten dahil olduğundan, “esnek” dizinin istenen uzunluğundan 1’i çıkarabilirsiniz. Aynı mantık diğer kullanım örneklerine de uygulanabilir.

Uyumluluk #

Esnek dizi üyelerini desteklemeyen derleyicilerle uyumluluk isteniyorsa, aşağıda “FLEXMEMB_SIZE” gibi tanımlanmış bir makro kullanabilirsiniz:

#if __STDC_VERSION__ < 199901L
#define FLEXMEMB_SIZE 1
#else
#define FLEXMEMB_SIZE /* nothing */
#endif

struct ex1 
{
    size_t foo;
    int flex[FLEXMEMB_SIZE];
};

Nesneleri tahsis ederken, esnek dizi üyelerini ve derleyicileri destekleyen derleyiciler arasında tutarlı kalacak tek ifade olduğundan, yapı boyutuna atıfta bulunmak için (esnek dizi üyesi hariç) “offsetof(struct ex1, flex)” formunu kullanmalısınız. olmayan:

struct ex1 *pe10 = malloc(offsetof(struct ex1, flex) + n * sizeof(pe10->flex[0]));

Alternatif, belirtilen uzunluktan koşullu olarak 1 çıkarmak için önişlemciyi kullanmaktır. Bu formdaki artan tutarsızlık potansiyeli ve genel insan hatası nedeniyle mantığı ayrı bir işleve taşıdım:

struct ex1 *ex1_alloc(size_t n)
{
    struct ex1 tmp;
#if __STDC_VERSION__ < 199901L
    if (n != 0)
        n--;
#endif
    return malloc(sizeof(tmp) + n * sizeof(tmp.flex[0]));
}
...

/* allocate an ex1 object with "flex" array of length 3 */
struct ex1 *pe1 = ex1_alloc(3);

Yapılara yönelik işaretçiler #

Bir “yapı” içeren bir değişkeniniz olduğunda, onun alanlarına nokta operatörünü (.) kullanarak erişebilirsiniz. Ancak, bir “yapı” için bir işaretçiniz varsa, bu işe yaramaz. Alanlarına erişmek için ok operatörünü (->) kullanmanız gerekir. İşte, “yapı” için işaretçiler kullanan ve ok operatörünü gösteren bir yığının çok basit (bazıları “korkunç ve basit” diyebilir) uygulamasına bir örnek.

#include <stdlib.h>
#include <stdio.h>

/* structs */
struct stack
{
    struct node *top;
    int size;
};

struct node
{
    int data;
    struct node *next;
};

/* function declarations */
int push(int, struct stack*);
int pop(struct stack*);
void destroy(struct stack*);

int main(void)
{
    int result = EXIT_SUCCESS;

    size_t i;

    /* allocate memory for a struct stack and record its pointer */
    struct stack *stack = malloc(sizeof *stack);
    if (NULL == stack)
    {
        perror("malloc() failed");
        return EXIT_FAILURE;
    }

    /* initialize stack */
    stack->top = NULL;
    stack->size = 0;

    /* push 10 ints */
    {
        int data = 0;
        for(i = 0; i < 10; i++)
        {
            printf("Pushing: %d\n", data);
            if (-1 == push(data, stack))
            {
                perror("push() failed");
                result = EXIT_FAILURE;
                break;
            }

            ++data;
        }
    }

    if (EXIT_SUCCESS == result)
    {
        /* pop 5 ints */
        for(i = 0; i < 5; i++)
        {
            printf("Popped: %i\n", pop(stack));
        }
    }

    /* destroy stack */
    destroy(stack);

    return result;
}

/* Push a value onto the stack. */
/* Returns 0 on success and -1 on failure. */
int push(int data, struct stack *stack)
{
    int result = 0;

    /* allocate memory for new node */
    struct node *new_node = malloc(sizeof *new_node);
    if (NULL == new_node)
    {
        result = -1;
    }
    else
    {
        new_node->data = data;
        new_node->next = stack->top;
        stack->top = new_node;
        stack->size++;
    }

    return result;
}

/* Pop a value off of the stack. */
/* Returns the value popped off the stack */
int pop(struct stack *stack)
{
    struct node *top = stack->top;
    int data = top->data;
    stack->top = top->next;
    stack->size--;
    free(top);
    return data;
}

/* destroy the stack */
void destroy(struct stack *stack)
{
    /* free all pointers */
    while(stack->top != NULL)
    {
        pop(stack);
    }
}

Yapıları fonksiyonlara geçirme #

C’de, yapılar da dahil olmak üzere tüm argümanlar, değerlere göre işlevlere iletilir. Küçük yapılar için bu iyi bir şeydir, çünkü bir işaretçi aracılığıyla verilere erişmenin bir ek yükü olmadığı anlamına gelir. Bununla birlikte, özellikle programcı, argümanların referans olarak iletildiği diğer dillere alışkınsa, düşük performansla sonuçlanan büyük bir yapıyı kazara iletmeyi çok kolaylaştırır.

struct coordinates
{
    int x;
    int y;
    int z;
};

// Passing and returning a small struct by value, very fast
struct coordinates move(struct coordinates position, struct coordinates movement)
{
    position.x += movement.x;
    position.y += movement.y;
    position.z += movement.z;
    return position;
}

// A very big struct
struct lotsOfData
{
    int param1;
    char param2[80000];
};

// Passing and returning a large struct by value, very slow!
// Given the large size of the struct this could even cause stack overflow
struct lotsOfData doubleParam1(struct lotsOfData value)
{
    value.param1 *= 2;
    return value;
}

// Passing the large struct by pointer instead, fairly fast
void doubleParam1ByPtr(struct lotsOfData *value)
{
    value->param1 *= 2;
}

Yapıları kullanarak nesne tabanlı programlama #

Yapılar, kodu nesne yönelimli bir şekilde uygulamak için kullanılabilir. Bir yapı bir sınıfa benzer, ancak normalde bir sınıfın parçasını oluşturan işlevler eksik, bunları işlev işaretçisi üye değişkenleri olarak ekleyebiliriz. Koordinat örneğimizle kalmak için:

/* coordinates.h */

typedef struct coordinate_s 
{
    /* Pointers to method functions */
    void (*setx)(coordinate *this, int x);
    void (*sety)(coordinate *this, int y);
    void (*print)(coordinate *this);
    /* Data */
    int x;
    int y;
} coordinate;

/* Constructor */
coordinate *coordinate_create(void);
/* Destructor */
void coordinate_destroy(coordinate *this);

Ve şimdi uygulayan C dosyası:

/* coordinates.c */

#include "coordinates.h"
#include <stdio.h>
#include <stdlib.h>

/* Constructor */
coordinate *coordinate_create(void)
{
    coordinate *c = malloc(sizeof(*c));
    if (c != 0)
    {
        c->setx = &coordinate_setx;
        c->sety = &coordinate_sety;
        c->print = &coordinate_print;
        c->x = 0;
        c->y = 0;
    }
    return c;
}

/* Destructor */
void coordinate_destroy(coordinate *this)
{
    if (this != NULL)
    {
        free(this);  
    }  
}

/* Methods */
static void coordinate_setx(coordinate *this, int x)
{
    if (this != NULL)
    {    
        this->x = x;
    }
}

static void coordinate_sety(coordinate *this, int y)
{
    if (this != NULL)
    {
        this->y = y;
    }
}

static void coordinate_print(coordinate *this)
{
    if (this != NULL)
    {
        printf("Coordinate: (%i, %i)\n", this->x, this->y);
    }
    else
    {
        printf("NULL pointer exception!\n");
    }
}

Koordinat sınıfımızın örnek bir kullanımı şöyle olabilir:

/* main.c */

#include "coordinates.h"
#include <stddef.h>

int main(void) 
{
    /* Create and initialize pointers to coordinate objects */
    coordinate *c1 = coordinate_create();
    coordinate *c2 = coordinate_create();
    
    /* Now we can use our objects using our methods and passing the object as parameter */
    c1->setx(c1, 1);
    c1->sety(c1, 2);

    c2->setx(c2, 3);
    c2->sety(c2, 4);

    c1->print(c1);
    c2->print(c2);

    /* After using our objects we destroy them using our "destructor" function */
    coordinate_destroy(c1);
    c1 = NULL;
    coordinate_destroy(c2);
    c2 = NULL;

    return 0;
}

Basit veri yapıları #

Yapı veri türleri, ilgili verileri paketlemenin ve davranmalarını sağlamanın kullanışlı bir yoludur. tek bir değişken gibi.

İki ‘int’ üyesi içeren basit bir ‘struct’ bildirmek:

struct point 
{
    int x;
    int y; 
};

“x” ve “y”, “nokta” yapısının üyeleri (veya alanları) olarak adlandırılır.

Yapıları tanımlama ve kullanma:

struct point p;    // declare p as a point struct
p.x = 5;           // assign p member variables
p.y = 3;

Yapılar tanımda başlatılabilir. Yukarıdaki şuna eşdeğerdir:

struct point p = {5, 3};

Yapılar, [belirlenmiş başlatıcılar][1] kullanılarak da başlatılabilir.

Alanlara erişim de . operatörü kullanılarak yapılır.

printf("point is (x = %d, y = %d)", p.x, p.y);

[1]: https://www.wikiod.com/tr/c/baslatma#Belirlenmiş başlatıcıları kullanma
