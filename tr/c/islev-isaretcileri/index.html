İşlev işaretçileri, veri türleri yerine işlevlere işaret eden işaretçilerdir. Çalışma zamanında çağrılacak fonksiyonda değişkenliğe izin vermek için kullanılabilirler.

Sözdizimi #

  • returnType (*name)(parametreler)

  • typedef returnType (*name)(parametreler)

  • typedef returnType Name(parametreler); İsim *isim;

  • typedef returnType Name(parametreler); typedef Ad *NamePtr;

Bir İşlevden İşlev İşaretçilerini Döndürme #

#include <stdio.h>

enum Op
{
  ADD = '+',
  SUB = '-',
};


/* add: add a and b, return result */    
int add(int a, int b)
{
    return a + b;
}

/* sub: subtract b from a, return result */
int sub(int a, int b)
{
    return a - b;
}

/* getmath: return the appropriate math function */
int (*getmath(enum Op op))(int,int)
{
    switch (op)
    {
        case ADD:
            return &add;
        case SUB:
            return &sub;
        default:
            return NULL;
    }
}

int main(void)
{
    int a, b, c;
    int (*fp)(int,int);

    fp = getmath(ADD);

    a = 1, b = 2;
    c = (*fp)(a, b);
    printf("%d + %d = %d\n", a, b, c);
    return 0;
}

En İyi Uygulamalar #

typedef’i kullanma #

İşlev işaretçisini her seferinde elle bildirmek yerine bir ’typedef’ kullanmak kullanışlı olabilir.

Bir işlev işaretçisi için bir “typedef” bildiren sözdizimi şöyledir:

typedef returnType (*name)(parameters);

Örnek: #

Şu şekilde bir fonksiyona işaret eden bir fonksiyon işaretçisi bekleyen bir fonksiyona, “sıralama"ya sahip olduğumuzu varsayalım:

karşılaştır - İki öğe için bir karşılaştırma işlevi bir sıralama işlevine sağlanacak.

“karşılaştır” ifadesinin, iki öğe kabul edilirse 0 döndürmesi beklenir eşit, geçen ilk öğe bazılarında “daha büyük” ise pozitif bir değer ikinci öğeden daha anlamlıdır ve aksi halde işlev bir negatif değer (birinci öğenin ikincisinden “daha az” olduğu anlamına gelir).

Bir “typedef” olmadan bir işlev işaretçisini bir işleve argüman olarak aşağıdaki şekilde iletirdik:

void sort(int (*compare)(const void *elem1, const void *elem2)) { 
    /* inside of this block, the function is named "compare" */
}

Bir “typedef” ile şunu yazardık:

typedef int (*compare_func)(const void *, const void *);

ve sonra ‘sort’ fonksiyon imzasını şu şekilde değiştirebiliriz:

void sort(compare_func func) { 
    /* In this block the function is named "func" */
}

“sort"un her iki tanımı da formun herhangi bir işlevini kabul eder

int compare(const void *arg1, const void *arg2) {
    /* Note that the variable names do not have to be "elem1" and "elem2" */
}

İşlev işaretçileri, türün işaretçi özelliğini eklemeniz gereken tek yerdir, ör. typedef struct bir şey_struct *something_type gibi tipler tanımlamaya çalışmayın. Bu, örneğin stdio.h ‘FILE’ türü (şimdi fark edeceğiniz gibi bir işaretçi değil) gibi, doğrudan API arayanlar tarafından erişilmesi gerekmeyen üyelere sahip bir yapı için bile geçerlidir.

Bağlam işaretçileri almak. #

Bir işlev işaretçisi, bağlam işaretçisi olarak hemen hemen her zaman kullanıcı tarafından sağlanan bir void * almalıdır.

Örnek #

/* function minimiser, details unimportant */
double findminimum( double (*fptr)(double x, double y, void *ctx), void *ctx)
{
    ...
    /* repeatedly make calls like this */
    temp = (*fptr)(testx, testy, ctx);
}

/* the function we are minimising, sums two cubics */
double *cubics(double x, double y, void *ctx)
{
    double *coeffsx = ctx;
    double *coeffsy = coeffx + 4;

    return coeffsx[0] * x * x * x + coeffsx[1] * x * x + coeffsx[2] * x + coeffsx[3] +
           coeffsy[0] * y * y * y + coeffsy[1] * y * y + coeffsy[2] * y + coeffsy[3];

} 

void caller()
{
    /* context, the coefficients of the cubics */
    double coeffs[8] = {1, 2, 3, 4, 5, 6, 7, 8};
    double min;

    min = findminimum(cubics, coeffs);       
}

Bağlam işaretçisinin kullanılması, fazladan parametrelerin işaret edilen işleve sabit kodlanması gerekmediği veya globallerin kullanılmasını gerektirmediği anlamına gelir.

qsort() kütüphane fonksiyonu bu kuralı takip etmez ve çoğu zaman önemsiz karşılaştırma fonksiyonları için bağlam olmadan kurtulabilir. Ancak daha karmaşık bir şey için bağlam işaretçisi gerekli hale gelir.

Giriiş #

Tıpkı ‘char’ ve ‘int’ gibi, bir işlev C’nin temel bir özelliğidir. Bu nedenle, birine bir işaretçi bildirebilirsiniz: bu, hangi işlev çağrılacak işlevini, işini yapmasına yardımcı olmak için başka bir işleve geçirebileceğiniz anlamına gelir. Örneğin, bir grafiği görüntüleyen bir ‘graph()’ işleviniz varsa, hangi işlevi graf‘a ‘graph()’ işlevine iletebilirsiniz.

// A couple of external definitions to make the example clearer
extern unsigned int screenWidth;
extern void plotXY(double x, double y);

// The graph() function.
// Pass in the bounds: the minimum and maximum X and Y that should be plotted.
// Also pass in the actual function to plot.
void graph(double minX, double minY,
           double maxX, double maxY,
           ???? *fn) {            // See below for syntax

    double stepX = (maxX - minX) / screenWidth;
    for (double x=minX; x<maxX; x+=stepX) {

        double y = fn(x);         // Get y for this x by calling passed-in fn()

        if (minY<=y && y<maxY) {
            plotXY(x, y);         // Plot calculated point
        } // if
    } for
} // graph(minX, minY, maxX, maxY, fn)

Kullanım #

Bu nedenle, yukarıdaki kod, içine aktardığınız işlevin grafiğini çizecektir - bu işlev belirli kriterleri karşıladığı sürece: yani, bir “çift” ilettiğiniz ve bir “çift” çıkış elde ettiğiniz sürece. Bunun gibi birçok işlev vardır - ‘sin()’, ‘cos()’, ’tan()’, ’exp()’ vb. !

Sözdizimi #

Peki hangi işlevleri graph()a geçirebileceğinizi ve hangilerini geçemeyeceğinizi nasıl belirlersiniz? Geleneksel yol, okunması veya anlaşılması kolay olmayan bir sözdizimi kullanmaktır:

double (*fn)(double); // fn is a pointer-to-function that takes a double and returns one

Yukarıdaki problem şu ki, aynı anda tanımlanmaya çalışılan iki şey var: fonksiyonun yapısı ve bunun bir işaretçi olması. Öyleyse, iki tanımı bölün! Ancak ’typedef’ kullanılarak daha iyi bir sözdizimi (okunması ve anlaşılması daha kolay) elde edilebilir.

İşlev İşaretçisi Atama #

#include <stdio.h>

/* increment: take number, increment it by one, and return it */
int increment(int i)
{
    printf("increment %d by 1\n", i);
    return i + 1;
}

/* decrement: take number, decrement it by one, and return it */
int decrement(int i)
{
    printf("decrement %d by 1\n", i);
    return i - 1;
}

int main(void)
{
    int num = 0;          /* declare number to increment */
    int (*fp)(int);       /* declare a function pointer */

    fp = &increment;      /* set function pointer to increment function */
    num = (*fp)(num);     /* increment num */
    num = (*fp)(num);     /* increment num a second time */

    fp = &decrement;      /* set function pointer to decrement function */
    num = (*fp)(num);     /* decrement num */
    printf("num is now: %d\n", num);
    return 0;
}

İşlev işaretçileri yazmak için anımsatıcı #

Tüm C işlevleri, program belleğinde bazı kodların bulunduğu bir noktanın gerçek işaretçileridir. Bir işlev işaretçisinin ana kullanımı, diğer işlevler için bir “geri arama” sağlamaktır (veya sınıfları ve nesneleri simüle etmektir).

Bu sayfanın devamında tanımlandığı gibi bir fonksiyonun sözdizimi şöyledir:

returnType (*name)(parametreler)

Bir işlev işaretçisi tanımı yazmak için bir anımsatıcı aşağıdaki yordamdır:

  1. Normal bir işlev bildirimi yazarak başlayın: returnType name(parameters)
  2. İşlev adını işaretçi sözdizimi ile sarın: returnType (*name)(parameters)

Temel Bilgiler #

Tıpkı bir int, char, float, array/string, struct, vb. için bir işaretçiniz olabileceği gibi - bir işaretçiniz olabilir. bir işlev.

İşaretçinin bildirilmesi işlevin _dönüş değerini, işlevin _adını ve aldığı argüman/parametre türünü alır.

Aşağıdaki işlevin bildirildiğini ve başlatıldığını varsayalım:

int addInt(int n, int m){
    return n+m;
}

Bu işleve bir işaretçi bildirebilir ve başlatabilirsiniz:

int (*functionPtrAdd)(int, int) = addInt; // or &addInt - the & is optional

Bir void işleviniz varsa, şöyle görünebilir:

void Print(void){
    printf("look ma' - no hands, only pointers!\n");
}

O zaman işaretçiyi buna bildirmek:

void (*functionPtrPrint)(void) = Print;

Erişim işlevin kendisi, işaretçinin referansının kaldırılmasını gerektirir:

sum = (*functionPtrAdd)(2, 3); //will assign 5 to sum
(*functionPtrPrint)(); //will print the text in Print function

Bu belgedeki daha gelişmiş örneklerde görüldüğü gibi, bir işleve bir işaretçi bildirmek, işleve birkaç parametreden daha fazla iletilirse dağınık olabilir. Aynı “yapıya” (aynı türde dönüş değeri ve aynı türde parametreler) sahip işlevler için birkaç işaretçiniz varsa, size biraz yazmaktan tasarruf etmek ve kodu daha fazla yapmak için typedef komutunu kullanmak en iyisidir. açık:

typedef int (*ptrInt)(int, int);

int Add(int i, int j){
    return i+j;
}

int Multiply(int i, int j){
    return i*j;
}

int main()
{
    ptrInt ptr1 = Add;
    ptrInt ptr2 = Multiply;

    printf("%d\n", (*ptr1)(2,3)); //will print 5
    printf("%d\n", (*ptr2)(2,3)); //will print 6
    return 0;
}

Ayrıca bir işlev işaretçileri dizisi oluşturabilirsiniz. Tüm işaretçiler aynı “yapıya” sahipse:

int (*array[2]) (int x, int y); // can hold 2 function pointers
array[0] = Add;
array[1] = Multiply;

buradan ve buradan daha fazla bilgi edinebilirsiniz.

Belirli bir işleve erişmek istediğinizde yayınlamayı gerektirse de, farklı türlerde bir dizi işlev işaretçisi tanımlamak da mümkündür. buradan daha fazla bilgi edinebilirsiniz.
