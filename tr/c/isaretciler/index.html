İşaretçi, başka bir nesnenin veya işlevin adresini depolayabilen bir değişken türüdür.

Sözdizimi #

  • <Veri türü> *<Değişken adı>;
  • int *ptrToInt;
  • geçersiz *ptrToVoid; /* C89+ */
  • bazıStruct yapısı *ptrToStruct;
  • int **ptrToPtrToInt;
  • int dizi[uzunluk]; int *ptrToFirstElem = dizi; /* <C99 için ‘uzunluk’ bir derleme zaman sabiti olması gerekir, >=C11 için bir olması gerekebilir. */
  • int *arrayOfPtrsToInt[uzunluk]; /* <C99 için ‘uzunluk’ bir derleme zaman sabiti olması gerekir, >=C11 için bir olması gerekebilir. */

Yıldız işaretinin konumu, tanımın anlamını etkilemez:

/* The * operator binds to right and therefore these are all equivalent. */
int *i;
int * i;
int* i;

Ancak, aynı anda birden çok işaretçi tanımlarken, her birinin kendi yıldız işareti gerekir:

int *i, *j; /* i and j are both pointers */
int* i, j;  /* i is a pointer, but j is an int not a pointer variable */

Dizi değişkeninin adından önce bir yıldız işaretinin verildiği bir dizi işaretçi de mümkündür:

int *foo[2]; /* foo is a array of pointers, can be accessed as *foo[0] and *foo[1] */

Genel hatalar #

İşaretçilerin uygunsuz kullanımı, çoğu zaman segmentasyon hataları nedeniyle güvenlik hatalarını veya program çökmelerini içerebilen bir hata kaynağıdır.

Tahsis hataları kontrol edilmiyor #

Bellek ayırmanın başarılı olacağı garanti edilmez ve bunun yerine bir “NULL” işaretçisi döndürebilir. Döndürülen değeri, ayırmanın başarılı olup olmadığını kontrol etmeden kullanmak, tanımsız davranış‘i çağırır. Bu genellikle bir çökmeye yol açar, ancak bir çöküşün olacağının garantisi yoktur, bu nedenle buna güvenmek de sorunlara yol açabilir.

Örneğin, güvenli olmayan yol:

struct SomeStruct *s = malloc(sizeof *s);
s->someValue = 0; /* UNSAFE, because s might be a null pointer */

Güvenli yol:

struct SomeStruct *s = malloc(sizeof *s);
if (s)
{
    s->someValue = 0; /* This is safe, we have checked that s is valid */
}

Bellek istenirken sizeof yerine değişmez sayıların kullanılması #

Belirli bir derleyici/makine yapılandırması için türlerin bilinen bir boyutu vardır; ancak, belirli bir türün (‘char’ dışında) boyutunun tüm derleyici/makine yapılandırmaları için aynı olacağını tanımlayan herhangi bir standart yoktur. Kod bellek ayırma için “sizeof(int)” yerine 4 kullanıyorsa, orijinal makinede çalışabilir, ancak kodun diğer makinelere veya derleyicilere taşınabilir olması gerekmez. Türler için sabit boyutlar “sizeof(that_type)” veya “sizeof(*var_ptr_to_that_type)” ile değiştirilmelidir.

Taşınabilir olmayan tahsis:

 int *intPtr = malloc(4*1000);    /* allocating storage for 1000 int */
 long *longPtr = malloc(8*1000);  /* allocating storage for 1000 long */

Taşınabilir tahsis:

 int *intPtr = malloc(sizeof(int)*1000);     /* allocating storage for 1000 int */
 long *longPtr = malloc(sizeof(long)*1000);  /* allocating storage for 1000 long */

Ya da daha iyisi:

 int *intPtr = malloc(sizeof(*intPtr)*1000);     /* allocating storage for 1000 int */
 long *longPtr = malloc(sizeof(*longPtr)*1000);  /* allocating storage for 1000 long */

Bellek sızıntıları #

‘Boş’ kullanılarak belleğin tahsisinin kaldırılamaması, artık program tarafından kullanılmayan yeniden kullanılamayan belleğin birikmesine yol açar; buna bellek sızıntısı denir. Bellek, atık bellek kaynaklarını sızdırır ve ayırma hatalarına neden olabilir.

Mantıksal hatalar #

Tüm ayırmalar aynı modeli izlemelidir:

  1. “malloc” (veya “calloc”) kullanarak ayırma
  2. Veri depolamak için kullanım
  3. ‘ücretsiz’ kullanarak tahsisin kaldırılması

“ücretsiz” (sarkan işaretçi) çağrısından sonra veya “malloc” (wild) çağrısından önce bellek kullanma gibi bu kalıba uyulmaması işaretçi), “free"yi iki kez çağırma (“çift ücretsiz”), vb., genellikle bir segmentasyon hatasına neden olur ve programın çökmesine neden olur.

Bu hatalar geçici olabilir ve hata ayıklaması zor olabilir - örneğin, boşaltılan bellek genellikle işletim sistemi tarafından hemen geri alınmaz ve bu nedenle sarkan işaretçiler bir süre devam edebilir ve çalışıyor gibi görünebilir.

Çalıştığı sistemlerde Valgrind, hangi belleğin sızdırıldığını ve ilk olarak nereye tahsis edildiğini belirlemek için paha biçilmez bir araçtır.

Değişkenleri yığınlamak için işaretçiler oluşturma #

Bir işaretçi oluşturmak, işaret edilen değişkenin ömrünü uzatmaz. Örneğin:

int* myFunction() 
{
    int x = 10;
    return &x;
}

Burada “x”, otomatik depolama süresine sahiptir (genellikle yığın ayırma olarak bilinir). Yığına tahsis edildiğinden, ömrü yalnızca “myFunction” yürütüldüğü sürecedir; “myFunction” çıktıktan sonra “x” değişkeni yok edilir. Bu işlev “x” adresini alır ("&x” kullanarak) ve onu arayan kişiye geri vererek, arayana var olmayan bir değişkene bir işaretçi bırakır. Bu değişkene erişmeye çalışmak tanımsız davranış‘i çağırır.

Çoğu derleyici, işlev çıktıktan sonra bir yığın çerçevesini gerçekten temizlemez, bu nedenle döndürülen işaretçinin referansını kaldırmak genellikle size beklenen verileri verir. Ancak başka bir işlev çağrıldığında, işaret edilen hafızanın üzerine yazılabilir ve işaret edilen verinin bozulduğu anlaşılıyor.

Bunu çözmek için, döndürülecek değişkenin deposunu “malloc” yapın ve yeni oluşturulan depoya bir işaretçi döndürün ya da işleve bir tane döndürmek yerine geçerli bir işaretçinin iletilmesini isteyin, örneğin:

#include <stdlib.h>
#include <stdio.h>

int *solution1(void) 
{
    int *x = malloc(sizeof *x);
    if (x == NULL) 
    {
        /* Something went wrong */
        return NULL;
    }

    *x = 10;

    return x;
}

void solution2(int *x) 
{
    /* NB: calling this function with an invalid or null pointer 
       causes undefined behaviour. */

    *x = 10;
}

int main(void) 
{
    { 
        /* Use solution1() */

        int *foo = solution1();  
        if (foo == NULL)
        {
            /* Something went wrong */
            return 1;
        }

        printf("The value set by solution1() is %i\n", *foo);
        /* Will output: "The value set by solution1() is 10" */

        free(foo);    /* Tidy up */
    }

    {
        /* Use solution2() */

        int bar;
        solution2(&bar); 

        printf("The value set by solution2() is %i\n", bar);
        /* Will output: "The value set by solution2() is 10" */
    }

    return 0;
}

Arttırma / azaltma ve referanstan çıkarma #

‘p’ ile gösterileni artırmak için ‘*p++’ yazarsanız, yanılıyorsunuz.

Arttırma/azaltma sonrası, referanstan çıkarmadan önce yürütülür. Bu nedenle, bu ifade ‘p’ işaretçisinin kendisini artıracak ve değiştirmeden artırmadan önce ‘p’ ile gösterileni döndürecektir.

‘p’ ile gösterileni artırmak için ‘(*p)++’ yazmalısınız.

Bu kural, sonradan azaltma için de geçerlidir: “*p–”, “p” ile gösterileni değil, “p” işaretçisini azaltır.

Giriiş #

Bir işaretçi, bir işaretçi olduğunu belirtmek için değişkenin türü ve adı arasına bir yıldız işareti (*) yerleştirilmesi dışında, diğer herhangi bir değişken gibi bildirilir.

int *pointer; /* inside a function, pointer is uninitialized and doesn't point to any valid object yet */

Aynı bildirimde aynı türden iki işaretçi değişkeni bildirmek için, her tanımlayıcıdan önce yıldız işaretini kullanın. Örneğin,

int *iptr1, *iptr2;
int *iptr3,  iptr4;  /* iptr3 is a pointer variable, whereas iptr4 is misnamed and is an int */

Bir ve işareti (’&’) ile gösterilen adresi veya referans operatörü, uygun tipte bir işaretçiye yerleştirilebilecek belirli bir değişkenin adresini verir.

int value = 1;
pointer = &value;

Yıldız işareti (*) ile gösterilen dolaylı veya referanstan çıkarma operatörü, bir işaretçi ile gösterilen bir nesnenin içeriğini alır.

printf("Value of pointed to integer: %d\n", *pointer);
/* Value of pointed to integer: 1 */

İşaretçi bir yapıya veya birleşim tipine işaret ediyorsa, o zaman onun referansını kaldırabilir ve -> operatörünü kullanarak üyelerine doğrudan erişebilirsiniz:

SomeStruct *s = &someObject;
s->someMember = 5; /* Equivalent to (*s).someMember = 5 */

C’de bir işaretçi, yeniden atanabilen ve aksi takdirde kendi başına bir değişken olarak kabul edilen ayrı bir değer türüdür. Örneğin, aşağıdaki örnek, işaretçinin (değişkenin) kendisinin değerini yazdırır.

printf("Value of the pointer itself: %p\n", (void *)pointer);
/* Value of the pointer itself: 0x7ffcd41b06e4 */
/* This address will be different each time the program is executed */

Bir işaretçi değiştirilebilir bir değişken olduğundan, olmaması mümkündür. null olarak ayarlanarak geçerli bir nesneye gelin

pointer = 0;     /* or alternatively */
pointer = NULL;

veya sadece geçerli olmayan rastgele bir bit deseni ekleyerek adres. İkincisi çok kötü bir durum çünkü olamaz işaretçinin referansı kaldırılmadan önce test edildi, yalnızca bir test var bir işaretçinin boş olması durumunda:

if (!pointer) exit(EXIT_FAILURE);

Bir işaretçi yalnızca geçerli bir nesneye işaret ediyorsa başvurudan çıkarılabilir, aksi halde davranış tanımsızdır. Birçok modern uygulama size şu şekilde yardımcı olabilir: segmentasyon hatası ve yürütmeyi sonlandırın, ancak diğerleri programınızı bir geçersiz durum.

Dereference operatörü tarafından döndürülen değer, orijinal değişkene değiştirilebilen bir takma addır, bu nedenle orijinal değişkeni değiştirerek değiştirilebilir.

*pointer += 1;
printf("Value of pointed to variable after change: %d\n", *pointer);
/* Value of pointed to variable after change: 2 */

İşaretçiler de yeniden atanabilir. Bu, bir nesneye işaret eden bir işaretçinin daha sonra aynı türden başka bir nesneye işaret etmek için kullanılabileceği anlamına gelir.

int value2 = 10;
pointer = &value2;
printf("Value from pointer: %d\n", *pointer);
/* Value from pointer: 10 */

Diğer tüm değişkenler gibi, işaretçilerin de belirli bir türü vardır. yapamazsın bir ‘short int’ adresini bir ’long int’ için bir işaretçiye atayın, misal. Bu tür davranışlara tip punning denir ve birkaç istisna olsa da C’de yasaktır.

İşaretçinin belirli bir türde olması gerekse de, her işaretçi türü için ayrılan bellek, işaret edilen türün boyutundan ziyade ortamın adresleri depolamak için kullandığı belleğe eşittir.

#include <stdio.h>

int main(void) {
    printf("Size of int pointer: %zu\n", sizeof (int*));      /* size 4 bytes */
    printf("Size of int variable: %zu\n", sizeof (int));      /* size 4 bytes */
    printf("Size of char pointer: %zu\n", sizeof (char*));    /* size 4 bytes */
    printf("Size of char variable: %zu\n", sizeof (char));    /* size 1 bytes */
    printf("Size of short pointer: %zu\n", sizeof (short*));  /* size 4 bytes */
    printf("Size of short variable: %zu\n", sizeof (short));  /* size 2 bytes */
    return 0;
}

(Not: C99 veya C11 standartlarını desteklemeyen Microsoft Visual Studio kullanıyorsanız, [%Iu](https://msdn.microsoft.com/en-us/library/tcxf1dw6. aspx)1 yukarıdaki örnekte %zu yerine.)

Yukarıdaki sonuçların ortamdan ortama sayı olarak değişebileceğini, ancak tüm ortamların farklı işaretçi türleri için eşit boyutlar göstereceğini unutmayın.

İşaretçiler ve Diziler #

İşaretçiler ve diziler C’de yakından bağlantılıdır. C’deki diziler her zaman bellekte bitişik konumlarda tutulur. İşaretçi aritmetiği her zaman işaret edilen öğenin boyutuna göre ölçeklenir. Bu nedenle, üç ikili dizimiz ve tabana bir işaretçimiz varsa, “ptr” birinci ikiliye, “(ptr + 1)” ikinciye, “*(ptr + 2)” üçüncü ikiliye atıfta bulunur. . Daha uygun bir gösterim, [] dizi gösterimini kullanmaktır.

double point[3] = {0.0, 1.0, 2.0};
double *ptr = point;

/* prints x 0.0, y 1.0 z 2.0 */ 
printf("x %f y %f z %f\n", ptr[0], ptr[1], ptr[2]);

Yani esasen ptr ve dizi adı birbirinin yerine kullanılabilir. Bu kural aynı zamanda bir dizinin bir alt programa geçirildiğinde bir işaretçiye bozunması anlamına gelir.

double point[3] = {0.0, 1.0, 2.0};

printf("length of point is %s\n", length(point));

/* get the distance of a 3D point from the origin */ 
double length(double *pt)
{
   return sqrt(pt[0] * pt[0] + pt[1] * pt[1] + pt[2] * pt[2])
}

Bir işaretçi, bir dizideki herhangi bir öğeye veya son öğenin ötesindeki öğeye işaret edebilir. Bununla birlikte, diziden önceki öğe de dahil olmak üzere başka herhangi bir değere bir işaretçi ayarlamak bir hatadır. (Sebebi, segmentli mimarilerde, ilk elemandan önceki adresin bir segment sınırını geçebilmesidir, derleyici, son eleman artı bir için bunun olmamasını sağlar).

Bir İşaretçinin Başvurusunu Kaldırma #

int a = 1;
int *a_pointer = &a;

‘a_pointer’ referansını kaldırmak ve a’nın değerini değiştirmek için aşağıdaki işlemi kullanırız

*a_pointer = 2;

Bu, aşağıdaki yazdırma ifadeleri kullanılarak doğrulanabilir.

printf("%d\n", a); /* Prints 2 */
printf("%d\n", *a_pointer); /* Also prints 2 */

Ancak, bir ‘NULL’ veya başka bir şekilde geçersiz bir işaretçinin başvurusunu iptal etmek yanlış olur. Bu

int *p1, *p2;

p1 = (int *) 0xbad;
p2 = NULL;

*p1 = 42;
*p2 = *p1 + 1;

genellikle tanımsız davranış‘dir. ‘p1’, geçerli bir adres olmayabilecek bir ‘0xbad’ adresine işaret ettiğinden, referansı kaldırılamaz. Kim bilir ne var? İşletim sistemi belleği veya başka bir programın belleği olabilir. Bunun gibi tek zaman kodu, belirli bilgileri sabit kodlanmış adreslerde depolayan gömülü geliştirmede kullanılır. ‘p2’, geçersiz olan ‘NULL’ olduğundan başvuru kaldırılamaz.

Bir Yapının İşaretçisini Kaldırma #

Diyelim ki aşağıdaki yapıya sahibiz:

struct MY_STRUCT 
{
    int my_int;
    float my_float;
};

MY_STRUCTu struct anahtar sözcüğünü atlayacak şekilde tanımlayabiliriz, böylece onu her kullandığımızda struct MY_STRUCT yazmamız gerekmez. Ancak bu isteğe bağlıdır.

typedef struct MY_STRUCT MY_STRUCT;

Daha sonra bu yapının bir örneğine bir işaretçimiz varsa

MY_STRUCT *instance;

Bu ifade dosya kapsamında görünüyorsa, program başladığında instance bir boş gösterici ile başlatılacaktır. Bu ifade bir fonksiyonun içinde görünüyorsa değeri tanımsızdır. Değişken, başvurunun kaldırılabilmesi için geçerli bir “MY_STRUCT” değişkenine veya dinamik olarak ayrılmış alana işaret edecek şekilde başlatılmalıdır. Örneğin:

MY_STRUCT info = { 1, 3.141593F };
MY_STRUCT *instance = &info;

İşaretçi geçerli olduğunda, iki farklı gösterimden birini kullanarak üyelerine erişmek için referansını kaldırabiliriz:

int a = (*instance).my_int;
float b = instance->my_float;

Her iki yöntem de işe yarasa da, parantez, referans * operatörü ve nokta . operatörünün kombinasyonu yerine ok -> operatörünü kullanmak daha iyi bir uygulamadır, çünkü özellikle okuması ve anlaması daha kolaydır. iç içe kullanımlar.

Bir diğer önemli fark aşağıda gösterilmiştir:

MY_STRUCT copy = *instance;
copy.my_int    = 2;

Bu durumda, “kopya”, “örnek” içeriğinin bir kopyasını içerir. ‘kopya’nın ‘my_int’ini değiştirmek, ‘örnek’te onu değiştirmeyecektir.

MY_STRUCT *ref = instance;
ref->my_int    = 2;

Bu durumda, “başvuru”, “örnek"e yapılan bir başvurudur. Referansı kullanarak “my_int"yi değiştirmek, “örnek"te değiştirecektir.

Yapıların kendileri yerine işlevlerde parametre olarak yapılara işaretçiler kullanmak yaygın bir uygulamadır. Yapıları işlev parametreleri olarak kullanmak, yapı büyükse yığının taşmasına neden olabilir. Bir yapıya işaretçi kullanmak, işaretçi için yalnızca yeterli yığın alanı kullanır, ancak işlev, işleve iletilen yapıyı değiştirirse yan etkilere neden olabilir.

İşlev işaretçileri #

İşaretçiler, işlevleri işaret etmek için de kullanılabilir.

Temel bir işlevi ele alalım:

int my_function(int a, int b)
{
    return 2 * a + 3 * b;
}

Şimdi, bu fonksiyonun türünde bir işaretçi tanımlayalım:

int (*my_pointer)(int, int);

Bir tane oluşturmak için bu şablonu kullanmanız yeterlidir:

return_type_of_func (*my_func_pointer)(type_arg1, type_arg2, ...)

Daha sonra bu işaretçiyi fonksiyona atamalıyız:

my_pointer = &my_function;

Bu işaretçi artık işlevi çağırmak için kullanılabilir:

/* Calling the pointed function */
int result = (*my_pointer)(4, 2);

...

/* Using the function pointer as an argument to another function */
void another_function(int (*another_pointer)(int, int))
{
    int a = 4;
    int b = 2;
    int result = (*another_pointer)(a, b);

    printf("%d\n", result);
}

Bu sözdizimi, temel türlerle daha doğal ve tutarlı görünse de, işlev işaretçilerini ilişkilendirmek ve başvurudan çıkarmak, “&” ve “*” operatörlerinin kullanımını gerektirmez. Bu nedenle, aşağıdaki pasaj eşit derecede geçerlidir:

/* Attribution without the & operator */
my_pointer = my_function;

/* Dereferencing without the * operator */
int result = my_pointer(4, 2);

İşlev işaretçilerinin okunabilirliğini artırmak için typedef’ler kullanılabilir.

typedef void (*Callback)(int a);

void some_function(Callback callback)
{
    int a = 4;
    callback(a);
}

Başka bir okunabilirlik hilesi, C standardının, yukarıdaki gibi argümanlarda (ancak değişken bildiriminde değil) bir işlev işaretçisini bir işlev prototipine benzeyen bir şeye basitleştirmesine izin vermesidir; bu nedenle aşağıdakiler, işlev tanımları ve bildirimleri için eşdeğer olarak kullanılabilir:

void some_function(void callback(int))
{
    int a = 4;
    callback(a);
}

Boş işaretçilerle polimorfik davranış #

qsort() standart kitaplık işlevi, tek bir işlevin çok çeşitli farklı türlerde çalışmasını sağlamak için boşluk işaretçilerinin nasıl kullanılabileceğinin iyi bir örneğidir.

void qsort (
    void *base,                                 /* Array to be sorted */
    size_t num,                                 /* Number of elements in array */
    size_t size,                                /* Size in bytes of each element */
    int (*compar)(const void *, const void *)); /* Comparison function for two elements */

Sıralanacak dizi bir void işaretçisi olarak iletilir, bu nedenle herhangi bir tür öğe dizisi üzerinde çalıştırılabilir. Sonraki iki argüman, qsort()‘a dizide ne kadar eleman beklemesi gerektiğini ve her bir elemanın bayt olarak ne kadar büyük olduğunu söyler.

Son argüman, kendisi iki geçersiz işaretçi alan bir karşılaştırma işlevine yönelik bir işlev işaretçisidir. Arayanın bu işlevi sağlamasını sağlayarak, “qsort()” herhangi bir türdeki öğeleri etkin bir şekilde sıralayabilir.

Floatları karşılaştırmak için böyle bir karşılaştırma fonksiyonuna bir örnek. qsort()‘a iletilen herhangi bir karşılaştırma işlevinin bu tür imzaya sahip olması gerektiğini unutmayın. Polimorfik hale getirilme şekli, void işaretçi argümanlarını karşılaştırmak istediğimiz eleman tipinin işaretçilerine çevirmektir.

int compare_floats(const void *a, const void *b)
{
    float fa = *((float *)a);
    float fb = *((float *)b);
    if (fa < fb)
        return -1;
    if (fa > fb)
        return 1;
    return 0;
}

qsort’un kayan noktaları karşılaştırmak için bu işlevi kullanacağını bildiğimiz için, geçersiz işaretçi argümanlarını, başvuruları kaldırmadan önce kayan nokta işaretçilerine geri çeviriyoruz.

Şimdi, “len” uzunluğunda bir “dizi” dizisinde qsort polimorfik işlevinin kullanımı çok basittir:

qsort(array, len, sizeof(array[0]), compare_floats);

Bir “NULL” işaretçisinin başvurusunu kaldırmanın sonucu tanımsızdır, bu nedenle programın üzerinde çalıştığı işletim sisteminin doğal davranışı bu olsa bile mutlaka bir çökmeye neden olmaz. void* bağımsız değişken olarak işaretçiler ve standart işlevlere değer döndürür #

“void*”, nesne türlerine yönelik işaretçilerin tümünü yakalama türüdür. Bunun kullanımdaki bir örneği, olarak bildirilen ‘malloc’ işlevidir.

void* malloc(size_t);

İşaretçiden geçersize dönüş türü, “malloc"tan bir işaretçiye dönüş değerini başka herhangi bir nesne türüne atamanın mümkün olduğu anlamına gelir:

int* vector = malloc(10 * sizeof *vector);

Değerleri açıkça boşluk işaretçilerinin içine ve dışına dökmemek* genellikle iyi bir uygulama olarak kabul edilir. ‘malloc()‘un özel durumunda bunun nedeni, açık bir atamayla, ‘stdlib.h’yi dahil etmeyi unutursanız, derleyicinin ‘malloc()’ için yanlış bir dönüş türü kabul etmesi, ancak bu konuda uyarı vermemesidir. Ayrıca, DRY (kendini tekrar etme) ilkesine daha iyi uyum sağlamak için void işaretçilerinin doğru davranışını kullanma durumudur; yukarıdakileri aşağıdakiyle karşılaştırın, burada aşağıdaki kod, bir yazım hatasının sorunlara neden olabileceği birkaç gereksiz ek yer içerir:

int* vector = (int*)malloc(10 * sizeof int*);

Benzer şekilde, aşağıdaki gibi işlevler

void* memcpy(void *restrict target, void const *restrict source, size_t size);

bağımsız değişkenleri ‘void *’ olarak belirtilmelidir, çünkü türünden bağımsız olarak herhangi bir nesnenin adresi iletilebilir. Burada ayrıca, bir çağrı bir atama kullanmamalıdır.

unsigned char buffer[sizeof(int)];
int b = 67;
memcpy(buffer, &b, sizeof buffer);

Öncül #

C ve C++‘da işaretçi sözdizimini çevreleyen en kafa karıştırıcı şey, işaretçi sembolü, yıldız işareti (*) bir değişkenle kullanıldığında geçerli olan iki farklı anlamın olmasıdır.

Örnek #

İlk olarak, bir işaretçi değişkenini bildirmek için * kullanırsınız.

int i = 5;
/* 'p' is a pointer to an integer, initialized as NULL */
int *p = NULL;
/* '&i' evaluates into address of 'i', which then assigned to 'p' */
p = &i;
/* 'p' is now holding the address of 'i' */

Bildirmediğiniz (veya çarpmadığınız) zaman, bir işaretçi değişkenini çıkarmak için * kullanılır:

*p = 123;
/* 'p' was pointing to 'i', so this changes value of 'i' to 123 */

Mevcut bir işaretçi değişkeninin başka bir değişkenin adresini tutmasını istediğinizde, * kullanmazsınız**, ancak şöyle yapın:

p = &another_variable;

C programlamaya yeni başlayanlar arasında ortak bir karışıklık, aynı anda bir işaretçi değişkeni bildirip başlattıklarında ortaya çıkar.

int *p = &i;

‘int i = 5;’ ve ‘int i olduğundan; i = 5;aynı sonucu verirse, bazılarıint *p = &i;veint *p; *p = &i;da aynı sonucu verir. Gerçek şu ki, hayır,int *p; p = &i;**başlatılmamış** bir işaretçiyi reddetmeye çalışacak ve bu da UB ile sonuçlanacaktır. Bir işaretçi bildirmiyor veya referansını kaldırmıyorken asla` kullanmayın.

Çözüm #

Yıldız işaretinin (*), kullanıldığı yere bağlı olarak işaretçilerle ilgili olarak C içinde iki farklı anlamı vardır. Bir değişken bildirimi içinde kullanıldığında, eşittir tarafının sağ tarafındaki değer, bellekteki bir adre için bir işaretçi değeri olmalıdır. Zaten bildirilmiş bir değişken ile kullanıldığında, yıldız işareti, bellekteki işaret edilen yere kadar takip ederek ve orada saklanan değerin atanmasına veya alınmasına izin vererek işaretçi değerini referans eder.

Götürmek

İşaretçilerle uğraşırken tabiri caizse P ve Q’larınıza dikkat etmek önemlidir. Yıldız işaretini ne zaman kullandığınıza ve orada kullandığınızda bunun ne anlama geldiğine dikkat edin. Bu küçük ayrıntıyı gözden kaçırmak, gerçekten uğraşmak istemediğiniz hatalı ve/veya tanımsız davranışlara neden olabilir.

İşaretçiden İşaretçiye #

C’de bir işaretçi başka bir işaretçiye başvurabilir.

#include <stdio.h>
#include <stdlib.h>

int main(void) {
  int A = 42;
  int* pA = &A;
  int** ppA = &pA;
  int*** pppA = &ppA;

  printf("%d", ***pppA); /* prints 42 */

  return EXIT_SUCCESS;
}
