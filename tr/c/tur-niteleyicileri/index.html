  • Değiştirilemeyen (const) değişkenler

  • Uçucu değişkenler

Tür niteleyicileri, bir tür hakkında ek anlambilimi açıklayan anahtar sözcüklerdir. Bunlar, tür imzalarının ayrılmaz bir parçasıdır. Hem bir bildirimin en üst düzeyinde (doğrudan tanımlayıcıyı etkileyen) hem de alt düzeylerde (yalnızca işaretçilerle ilgili, işaret edilen değerleri etkileyen) görünebilirler:

Anahtar Kelime Notlar
‘const’ Bildirilen nesnenin mutasyonunu (en üst düzeyde görünerek) veya işaret edilen değerin mutasyonunu (bir işaretçi alt türünün yanında görünerek) önler.
‘uçucu’ Derleyiciye, bildirilen nesnenin (en üst düzeyde) veya işaret edilen değerin (işaretçi alt türlerinde) değerini yalnızca program kontrol akışının bir sonucu olarak değil, dış koşulların bir sonucu olarak değiştirebileceğini bildirir.
‘kısıtla’ Yalnızca işaretçilerle ilgili bir optimizasyon ipucu. İşaretçinin ömrü boyunca, aynı işaret edilen nesneye erişmek için başka hiçbir işaretçi kullanılmayacağını bildirir.

Depolama sınıfı belirteçlerine (‘static’, ’extern’, ‘auto’, ‘register’), tür değiştiricilere (‘signed’, ‘unsigned’, ‘short’, ’long’) ve türe göre tür niteleyicilerin sıralaması belirteçler (“int”, “char”, “double” vb.) zorunlu değildir, ancak bunları yukarıda belirtilen sıraya koymak iyi bir uygulamadır:

static const volatile unsigned long int a = 5; /* good practice */
unsigned volatile long static int const b = 5; /* bad practice */

Üst düzey nitelikler

/* "a" cannot be mutated by the program but can change as a result of external conditions */
const volatile int a = 5;

/* the const applies to array elements, i.e. "a[0]" cannot be mutated */    
const int arr[] = { 1, 2, 3 };

/* for the lifetime of "ptr", no other pointer could point to the same "int" object */
int *restrict ptr;

İşaretçi alt türü nitelikleri

/* "s1" can be mutated, but "*s1" cannot */
const char *s1 = "Hello";

/* neither "s2" (because of top-level const) nor "*s2" can be mutated */
const char *const s2 = "World";

/* "*p" may change its value as a result of external conditions, "**p" and "p" cannot */
char *volatile *p;

/* "q", "*q" and "**q" may change their values as a result of external conditions */
volatile char *volatile *volatile q;

Değiştirilemeyen (const) değişkenler #

const int a = 0; /* This variable is "unmodifiable", the compiler
                    should throw an error when this variable is changed */
int b = 0; /* This variable is modifiable */

b += 10; /* Changes the value of 'b' */
a += 10; /* Throws a compiler error */

‘const’ niteliği yalnızca verileri değiştirme hakkımızın olmadığı anlamına gelir. Değerin arkamızdan değişemeyeceği anlamına gelmez.

_Bool doIt(double const* a) {
   double rememberA = *a;
   // do something long and complicated that calls other functions

   return rememberA == *a;
}

Diğer çağrıların yürütülmesi sırasında ‘*a’ değişmiş olabilir ve bu nedenle bu işlev ‘false’ veya ’true’ döndürebilir.

Uyarı #

“const” niteliğine sahip değişkenler yine de işaretçiler kullanılarak değiştirilebilir:

const int a = 0;

int *a_ptr = (int*)&a; /* This conversion must be explicitly done with a cast */
*a_ptr += 10;          /* This has undefined behavior */

printf("a = %d\n", a); /* May print: "a = 10" */

Ancak bunu yapmak, tanımsız davranışa yol açan bir hatadır. Buradaki zorluk, bunun gibi basit örneklerde beklendiği gibi davranabilmesi, ancak kod büyüdüğünde yanlış gitmesidir.

Uçucu değişkenler #

‘volatile’ anahtar sözcüğü derleyiciye, değişkenin değerinin yalnızca program kontrol akışının bir sonucu olarak değil, dış koşulların bir sonucu olarak herhangi bir zamanda değişebileceğini söyler.

Derleyici, uçucu değişkenle ilgisi olan hiçbir şeyi optimize etmeyecektir.

volatile int foo; /* Different ways to declare a volatile variable */
int volatile foo;

volatile uint8_t * pReg; /* Pointers to volatile variable */
uint8_t volatile * pReg;

Uçucu değişkenleri kullanmanın iki ana nedeni vardır:

  • Bellek eşlemeli G/Ç kayıtları olan donanımla arabirim oluşturmak için.
  • Program kontrol akışının dışında değiştirilen değişkenleri kullanırken (örneğin, bir kesme servis rutininde)

Şu örneği görelim:

int quit = false;

void main() 
{
    ... 
    while (!quit) {
      // Do something that does not modify the quit variable
    } 
    ...
}

void interrupt_handler(void) 
{
  quit = true;
}

Derleyicinin while döngüsünün ‘quit’ değişkenini değiştirmediğini ve döngüyü sonsuz bir ‘while (true)’ döngüsüne dönüştürmediğini fark etmesine izin verilir. “SIGINT” ve “SIGTERM” için sinyal işleyicide “quit” değişkeni ayarlanmış olsa bile, derleyici bunu bilmez.

‘quit’i ‘volatile’ olarak ilan etmek, derleyiciye döngüyü optimize etmemesini söyler ve sorun çözülür.

Bu örnekte gördüğümüz gibi, donanıma erişirken de aynı sorun ortaya çıkıyor:

uint8_t * pReg = (uint8_t *) 0x1717;

// Wait for register to become non-zero 
while (*pReg == 0) { } // Do something else

Optimize edicinin davranışı değişkenin değerini bir kez okumaktır, değer her zaman aynı olacağından tekrar okumaya gerek yoktur. Böylece sonsuz bir döngü elde ederiz. Derleyiciyi istediğimizi yapmaya zorlamak için bildirimi şu şekilde değiştiririz:

uint8_t volatile * pReg = (uint8_t volatile *) 0x1717;
