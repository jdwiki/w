Dinamik olarak ayrılmış belleği yönetmek için, standart C kitaplığı ‘malloc()’, ‘calloc()’, ‘realloc()’ ve ‘free()’ işlevlerini sağlar. C99 ve sonraki sürümlerde ayrıca ‘aligned_alloc()’ vardır. Bazı sistemler ayrıca ‘alloca()’ sağlar.

Sözdizimi #

  • void *aligned_alloc(size_t hizalaması, size_t boyut); /* Sadece C11’den beri */
  • void *calloc(size_t öğeleri, size_t boyut);
  • void free(void *ptr);
  • void *malloc(size_t boyut);
  • void *realloc(void *ptr, size_t boyut);
  • void *alloca(size_t size); /* alloca.h’den, standart değil, taşınabilir değil, tehlikeli. */

Parametreler #

isim açıklama
boyut (malloc, realloc ve aligned_alloc) bayt cinsinden belleğin toplam boyutu. “aligned_alloc” için boyut, hizalamanın tam bir katı olmalıdır.
boyut (calloc) her elemanın boyutu
elementler eleman sayısı
ptr önceden ‘malloc’, ‘calloc’, ‘realloc’ veya ‘aligned_alloc’ tarafından döndürülen ayrılmış belleğe işaretçi
hizalama tahsis edilen hafızanın hizalanması

“aligned_alloc()” öğesinin yalnızca C11 veya üstü için tanımlandığını unutmayın.

son sürüm

POSIX tabanlı sistemler gibi sistemler, hizalanmış bellek ayırmanın başka yollarını sağlar (ör. posix_memalign()) ve ayrıca başka bellek yönetimi seçeneklerine de sahiptir (örneğin, mmap()).

Belleği Boşaltma #

free() öğesini çağırarak dinamik olarak ayrılmış belleği serbest bırakmak mümkündür.

int *p = malloc(10 * sizeof *p); /* allocation of memory */
if (p == NULL) 
{
    perror("malloc failed");
    return -1;
}

free(p); /* release of memory */
/* note that after free(p), even using the *value* of the pointer p
   has undefined behavior, until a new value is stored into it. */

/* reusing/re-purposing the pointer itself */
int i = 42;
p = &i; /* This is valid, has defined behaviour */

“p” ile gösterilen bellek, “free()” çağrısından sonra (ya libc uygulaması ya da temel işletim sistemi tarafından) geri alınır, bu nedenle serbest bırakılan bellek bloğuna “p” aracılığıyla erişmek tanımsız davranış. Serbest bırakılan bellek öğelerine başvuran işaretçiler genellikle [sarkan işaretçiler2 olarak adlandırılır ve bir güvenlik riski taşır. Ayrıca, C standardı, [değere erişmenin](https://www.wikiod.com/tr/c/tanmsz-davrans#Serbest bırakılan işaretçinin değerini oku) bile olduğunu belirtir. Sarkan bir işaretçinin 201608050732042665317) tanımsız davranışı var. ‘p’ işaretçisinin kendisinin yukarıda gösterildiği gibi yeniden kullanılabileceğini unutmayın.

Lütfen, yalnızca ‘malloc()’, ‘calloc()’, ‘realloc()’ ve ‘aligned_alloc()’ işlevlerinden doğrudan döndürülen işaretçilerde ‘free()’ çağırabileceğinizi veya belgelerin size bellek bu şekilde tahsis edilmiştir (strdup() gibi işlevler dikkate değer örneklerdir). Bir işaretçiyi serbest bırakmak,

  • bir değişken üzerinde & operatörü kullanılarak elde edilen veya
  • tahsis edilmiş bir bloğun ortasında,

yasak. Böyle bir hata genellikle derleyiciniz tarafından teşhis edilmez, ancak programın yürütülmesini tanımsız bir durumda yönlendirir.

Bu tür tanımsız davranış örneklerini önlemek için iki yaygın strateji vardır.

İlki ve tercih edileni basittir - artık gerekmediğinde “p"nin kendisinin yok olması, örneğin:

if (something_is_needed())
{

    int *p = malloc(10 * sizeof *p);
    if (p == NULL) 
    {
        perror("malloc failed");
        return -1;
    }

    /* do whatever is needed with p */

    free(p);
}

İçerdiği bloğun (yani, }) bitiminden hemen önce free() çağrıldığında, pnin kendisi ortadan kalkar. Derleyici bundan sonra herhangi bir “p” kullanma girişiminde derleme hatası verecektir.

İkinci bir yaklaşım, işaret ettiği belleği serbest bıraktıktan sonra işaretçinin kendisini de geçersiz kılmaktır:

free(p);
p = NULL;     // you may also use 0 instead of NULL

Bu yaklaşım için argümanlar:

  • Birçok platformda, bir boş göstericinin başvurusunu kaldırma girişimi anında çökmeye neden olur: Segmentasyon hatası. Burada, serbest bırakıldıktan sonra kullanılan değişkene işaret eden en azından bir yığın izi elde ederiz.

İşaretçiyi ‘NULL’ olarak ayarlamadan, sarkan bir işaretçimiz var. Program büyük olasılıkla çökmeye devam edecek, ancak daha sonra işaretçinin işaret ettiği bellek sessizce bozulacağı için. Bu tür hataların izini sürmek zordur çünkü ilk sorunla tamamen alakasız bir çağrı yığınına neden olabilirler.

Dolayısıyla bu yaklaşım fail-fast konseptini takip eder.

  • Bir boş göstericiyi serbest bırakmak güvenlidir. [C Standardı, 4 “free(NULL)” öğesinin hiçbir etkisi olmadığını belirtir:

free işlevi, ptr ile gösterilen alanın serbest bırakılmasına, yani daha fazla tahsis için uygun hale getirilmesine neden olur. ptr bir boş gösterici ise, hiçbir eylem gerçekleşmez. Aksi takdirde, argüman daha önce ‘calloc’, ‘malloc’ veya ‘realloc’ işlevi tarafından döndürülen bir işaretçiyle eşleşmiyorsa veya alan ‘free’ veya ‘realloc’ çağrısıyla serbest bırakıldıysa, davranış şöyle olur: Tanımsız.

  • Bazen ilk yaklaşım kullanılamaz (örneğin, bellek bir işleve tahsis edilir ve çok daha sonra tamamen farklı bir işleve tahsis edilir)

5: https://www.wikiod.com/tr/c/isaretciler#Genel hatalar

Bellek Ayırma #

Standart Tahsis #

C dinamik bellek ayırma işlevleri, <stdlib.h> başlığında tanımlanır. Bir nesne için dinamik olarak bellek alanı ayırmak istenirse, aşağıdaki kod kullanılabilir:

int *p = malloc(10 * sizeof *p);
if (p == NULL) 
{
    perror("malloc() failed");
    return -1;
}

Bu, on “int"in bellekte kapladığı bayt sayısını hesaplar, ardından “malloc"tan bu kadar çok bayt ister ve sonucu (yani, “malloc” kullanılarak yeni oluşturulan bellek yığınının başlangıç ​​adresi) bir nesneye atar. ‘p’ adlı işaretçi.

‘sizeof’un sonucu uygulama tanımlı olduğundan (‘char’, ‘signed char’ ve ‘unsigned char’ olan karakter türleri hariç) talep edilecek bellek miktarını hesaplamak için ‘sizeof’ kullanmak iyi bir uygulamadır. , bunun için “sizeof” her zaman “1” verecek şekilde tanımlanır).

“malloc” isteği yerine getiremeyebileceğinden boş gösterici döndürebilir. Daha sonra boş göstericinin başvurusunu kaldırma girişimlerini önlemek için bunu kontrol etmek önemlidir.

‘malloc()’ kullanılarak dinamik olarak ayrılan bellek, ‘realloc()’ kullanılarak yeniden boyutlandırılabilir veya artık gerekmediğinde ‘free()’ kullanılarak serbest bırakılabilir.

Alternatif olarak, int array[10]; bildirimi aynı miktarda bellek tahsis eder. Ancak, ‘static’ anahtar sözcüğü olmadan bir işlev içinde bildirilirse, yalnızca bildirildiği işlev ve çağırdığı işlevler içinde kullanılabilir (çünkü dizi yığına ayrılacak ve alan bunun için serbest bırakılacaktır). işlev döndüğünde yeniden kullanın). Alternatif olarak, bir fonksiyonun içinde ‘statik’ ile tanımlanmışsa veya herhangi bir fonksiyonun dışında tanımlanmışsa, ömrü programın ömrüdür. İşaretçiler bir işlevden de döndürülebilir, ancak C’deki bir işlev bir dizi döndüremez.

Sıfırlanmış Bellek #

“malloc” tarafından döndürülen bellek, makul bir değere başlatılamayabilir ve belleği “memset” ile sıfırlamaya veya uygun bir değeri hemen içine kopyalamaya özen gösterilmelidir. Alternatif olarak, “calloc”, tüm bitlerin “0” olarak başlatıldığı istenen boyutta bir blok döndürür. Bunun, kayan noktalı sıfır veya boş işaretçi sabitinin temsili ile aynı olması gerekmez.

int *p = calloc(10, sizeof *p);
if (p == NULL) 
{
    perror("calloc() failed");
    return -1;
}

‘calloc’ ile ilgili bir not: Çoğu (yaygın olarak kullanılan) uygulama, performans için ‘calloc()‘u optimize edecektir, bu nedenle ‘malloc()’, ardından ‘memset() çağrılmasından daha hızlı olacaktır. )`, net etki aynı olsa bile.

Hizalanmış Bellek #

C11, verilen hizalamayla boşluk ayıran yeni bir “aligned_alloc()” işlevini tanıttı. Ayrılacak belleğin malloc() veya calloc() tarafından karşılanamayan belirli sınırlarda hizalanması gerekiyorsa kullanılabilir. ‘malloc()’ ve ‘calloc()’ işlevleri, herhangi bir nesne türü için uygun şekilde hizalanmış belleği ayırır (yani, hizalama ‘alignof(max_align_t)’ şeklindedir). Ancak aligned_alloc() ile daha büyük hizalamalar istenebilir.

/* Allocates 1024 bytes with 256 bytes alignment. */
char *ptr = aligned_alloc(256, 1024);
if (ptr) {
    perror("aligned_alloc()");
    return -1;
}
free(ptr);

C11 standardı iki kısıtlama getirir: 1) talep edilen boyut (ikinci argüman) hizalamanın (ilk argüman) tam bir katı olmalıdır ve 2) hizalama değeri, uygulama tarafından desteklenen geçerli bir hizalama olmalıdır . Bunlardan herhangi birinin karşılanmaması tanımsız davranış ile sonuçlanır.

son sürüm

Belleği Yeniden Tahsis Etme #

Bellek ayırdıktan sonra işaretçi depolama alanınızı genişletmeniz veya küçültmeniz gerekebilir. ‘void *realloc(void *ptr, size_t size)’ işlevi, ‘ptr’ ile gösterilen eski nesnenin yerini değiştirir ve ‘size’ ile belirtilen boyuta sahip bir nesneye bir işaretçi döndürür. ‘ptr’, daha önce yeniden tahsis edilecek ‘malloc’, ‘calloc’ veya ‘realloc’ (veya bir boş gösterici) ile ayrılmış bir bellek bloğunun göstericisidir. Orijinal belleğin maksimum olası içeriği korunur. Yeni boyut daha büyükse, eski boyutun ötesindeki herhangi bir ek bellek başlatılmaz. Yeni boyut daha kısaysa, küçülen kısmın içeriği kaybolur. “ptr” NULL ise, yeni bir blok tahsis edilir ve işlev tarafından ona bir işaretçi döndürülür.

#include <stdio.h>
#include <stdlib.h>
 
int main(void)
{
    int *p = malloc(10 * sizeof *p);
    if (NULL == p) 
    {
        perror("malloc() failed");
        return EXIT_FAILURE;
    }
 
    p[0] = 42;
    p[9] = 15;

    /* Reallocate array to a larger size, storing the result into a
     * temporary pointer in case realloc() fails. */
    {
        int *temporary = realloc(p, 1000000 * sizeof *temporary);

        /* realloc() failed, the original allocation was not free'd yet. */
        if (NULL == temporary)
        {
            perror("realloc() failed");
            free(p); /* Clean up. */
            return EXIT_FAILURE;
        }      

        p = temporary;
    }

    /* From here on, array can be used with the new size it was 
     * realloc'ed to, until it is free'd. */

    /* The values of p[0] to p[9] are preserved, so this will print:
       42 15
    */
    printf("%d %d\n", p[0], p[9]);

    free(p);

    return EXIT_SUCCESS;
}

Yeniden tahsis edilen nesne, “*p” ile aynı adrese sahip olabilir veya olmayabilir. Bu nedenle, arama başarılı olursa, yeni adresi içeren ‘realloc’tan dönüş değerini yakalamak önemlidir.

“realloc” dönüş değerini orijinal “p” yerine bir “geçici"ye atadığınızdan emin olun. “realloc”, işaretçinin üzerine yazacak herhangi bir hata durumunda null değerini döndürür. Bu, verilerinizi kaybeder ve bir bellek sızıntısı oluşturur.

realloc(ptr, 0), free(ptr) ile eşdeğer değil #

“realloc”, diğer işaretçideki “malloc + memcpy + free” ile kavramsal olarak eşdeğerdir.

İstenen alanın boyutu sıfırsa, “realloc” davranışı uygulama tanımlıdır. Bu, “0” değerinde bir “size” parametresi alan tüm bellek ayırma işlevleri için benzerdir. Bu tür işlevler aslında boş olmayan bir işaretçi döndürebilir, ancak bu asla başvurudan çıkarılmamalıdır.

Bu nedenle, “realloc(ptr,0)”, “free(ptr)” ile eşdeğer değildir. Olabilir

  • “tembel” bir uygulama olun ve sadece ptr döndürün
  • free(ptr), bir kukla eleman tahsis edin ve bunu döndürün
  • free(ptr) ve 0 döndür
  • sadece başarısızlık için ‘0’ döndürün ve başka bir şey yapmayın.

Bu nedenle, özellikle son iki durum, uygulama koduyla ayırt edilemez.

Bu, “realloc(ptr,0)” öğesinin belleği gerçekten boşaltmayabileceği/tahsis edemeyebileceği ve bu nedenle asla “ücretsiz” yerine kullanılmaması gerektiği anlamına gelir.

Değişken boyutta çok boyutlu diziler #

C99’dan beri C, yalnızca başlatma zamanında bilinen sınırları olan dizileri modelleyen değişken uzunluklu dizilere, VLA’ya sahiptir. Çok büyük VLA ayırmamaya dikkat etmek zorunda olsanız da (yığınınızı parçalayabilirler), *VLA’ya * işaretçiler kullanmak ve bunları ‘sizeof’ ifadelerinde kullanmak iyidir.

double sumAll(size_t n, size_t m, double A[n][m]) {
    double ret = 0.0;
    for (size_t i = 0; i < n; ++i)
       for (size_t j = 0; j < m; ++j)
          ret += A[i][j]
    return ret;
}

int main(int argc, char *argv[argc+1]) {
   size_t n = argc*10;
   size_t m = argc*8;
   double (*matrix)[m] = malloc(sizeof(double[n][m]));
   // initialize matrix somehow
   double res = sumAll(n, m, matrix);
   printf("result is %g\n", res);
   free(matrix);
}

Burada “matris”, “double[m]” türündeki öğelere bir işaretçidir ve “double[n][m]” ile “sizeof” ifadesi, bu tür “n” öğeler için boşluk içermesini sağlar.

Tüm bu alan bitişik olarak tahsis edilir ve böylece tek bir ‘free’ çağrısı ile yeniden tahsis edilebilir.

Dilde VLA’nın varlığı, işlev başlıklarında dizilerin ve işaretçilerin olası bildirimlerini de etkiler. Şimdi, dizi parametrelerinin “[]” içinde genel bir tamsayı ifadesine izin verilir. Her iki işlev için de “[]” içindeki ifadeler, parametre listesinde daha önce bildirilmiş olan parametreleri kullanır. “sumAll” için bunlar, kullanıcı kodunun matris için beklediği uzunluklardır. C’deki tüm dizi işlevi parametrelerine gelince, en içteki boyut bir işaretçi türüne yeniden yazılır, bu nedenle bu, bildirime eşdeğerdir.

  double sumAll(size_t n, size_t m, double (*A)[m]);

Yani, ’n’ gerçekten işlev arayüzünün bir parçası değildir, ancak bilgi dokümantasyon için faydalı olabilir ve ayrıca sınır kontrol derleyicileri tarafından sınır dışı erişim hakkında uyarmak için kullanılabilir.

Benzer şekilde, “ana” için “argc+1” ifadesi, C standardının “argv” argümanı için öngördüğü minimum uzunluktur.

C11’de resmi olarak VLA desteğinin isteğe bağlı olduğunu unutmayın, ancak C11’i uygulayan ve bunlara sahip olmayan hiçbir derleyici bilmiyoruz. Gerekirse __STDC_NO_VLA__ makrosu ile test edebilirsiniz.

son sürüm

alloca: yığında bellek ayırın #

Uyarı: ‘alloca’ burada yalnızca eksiksiz olması açısından bahsedilmiştir. Tamamen taşınabilir değildir (ortak standartlardan herhangi biri kapsamında değildir) ve farkında olmayanlar için onu güvenli olmayan bir dizi potansiyel olarak tehlikeli özelliğe sahiptir. Modern C kodu, onu Değişken Uzunluk Dizileri (VLA) ile değiştirmelidir.

Manuel sayfa

#include <alloca.h>
// glibc version of stdlib.h include alloca.h by default

void foo(int size) {
    char *data = alloca(size);
    /*
      function body;
    */
    // data is automatically freed
}

Çağıranın yığın çerçevesine bellek ayırın, dönen işaretçi tarafından başvurulan alan, arayan işlevi bittiğinde otomatik olarak boş’d olur.

Bu işlev otomatik bellek yönetimi için uygun olsa da, büyük ayırma talep etmenin yığın taşmasına neden olabileceğini ve alloca ile ayrılmış bellekle ücretsiz kullanamayacağınızı unutmayın (bu, yığın taşması ile daha fazla soruna neden olur).

Bu nedenle, bir döngü veya özyinelemeli bir işlev içinde alloca kullanılması önerilmez.

Ve işlev dönüşünde bellek boş’ olduğu için, işaretçiyi bir işlev sonucu olarak döndüremezsiniz (davranış tanımsız olur).

Özet #

  • malloc ile aynı çağrı
  • işlev dönüşünde otomatik olarak serbest bırakılır
  • ücretsiz,realloc işlevleriyle uyumlu değil (tanımsız davranış)
  • işaretçi bir işlev sonucu olarak döndürülemez (tanımsız davranış)
  • ayırma boyutu, (çoğu makinede) “malloc()” tarafından kullanılabilen yığın alanından çok daha küçük olan yığın alanıyla sınırlıdır
  • tek bir işlevde “alloca()” ve VLA’ları (değişken uzunluklu diziler) kullanmaktan kaçının
  • “alloca()”, “malloc()” ve diğerleri kadar taşınabilir değil

Öneri #

  • Yeni kodda alloca() kullanmayın

Modern alternatif.

void foo(int size) {
    char data[size];
    /*
      function body;
    */
    // data is automatically freed
}

Bu, ‘alloca()’ işlevinin çalıştığı yerde ve ‘alloca()’ işlevinin çalışmadığı yerlerde (örneğin döngülerin içinde) çalışır. ‘STDC_NO_VLA’ tanımlamayan bir C99 uygulamasını veya bir C11 uygulamasını varsayar.

Kullanıcı tanımlı bellek yönetimi #

“malloc()” genellikle bellek sayfalarını elde etmek için temel işletim sistemi işlevlerini çağırır. Ancak işlevle ilgili özel bir şey yoktur ve büyük bir statik dizi bildirerek ve ondan tahsis ederek düz C’de uygulanabilir (doğru hizalamayı sağlamakta hafif bir zorluk vardır, pratikte 8 bayta hizalamak neredeyse her zaman yeterlidir).

Basit bir şema uygulamak için, çağrıdan döndürülecek işaretçiden hemen önce bellek bölgesinde bir kontrol bloğu saklanır. Bu, “free()“nin döndürülen işaretçiden çıkarılarak ve tipik olarak blok boyutu olan kontrol bilgisinin okunmasıyla ve bunun serbest listeye geri alınmasına izin veren bazı bilgiler - ayrılmamış bir bağlantılı liste - uygulanabileceği anlamına gelir. bloklar.

Kullanıcı tahsis talebinde bulunduğunda, talep edilen miktarla aynı veya daha büyük bir blok bulunana kadar ücretsiz liste aranır, ardından gerekirse bölünür. Bu, kullanıcı sürekli olarak birçok tahsisat yapıyorsa ve öngörülemeyen boyutta ve ve öngörülemeyen aralıklarla serbest bırakıyorsa (tüm gerçek programlar böyle davranmaz, basit şema genellikle küçük programlar için yeterlidir).

/* typical control block */
struct block
{
   size_t size;         /* size of block */
   struct block *next;  /* next block in free list */ 
   struct block *prev;  /* back pointer to previous block in memory */
   void *padding;       /* need 16 bytes to make multiple of 8 */
}

static struct block arena[10000]; /* allocate from here */
static struct block *firstfree;

Birçok program, aynı boyuttaki küçük nesnelerin çok sayıda tahsisini gerektirir. Bunun uygulanması çok kolaydır. Basitçe bir sonraki işaretçi ile bir blok kullanın. Yani 32 baytlık bir blok gerekliyse:

union block
{
   union block * next;
   unsigned char payload[32];
}  

static union block arena[100];
static union block * head; 
void init(void)
{
    int i;
    for (i = 0; i < 100 - 1; i++)
        arena[i].next = &arena[i + 1];
    arena[i].next = 0; /* last one, null */
    head = &block[0];
}
 
void *block_alloc()
{
    void *answer = head;
    if (answer)
        head = head->next;
    return answer;
}

void block_free(void *ptr)
{
    union block *block = ptr;
    block->next = head;
    head - block;
}

Bu şema son derece hızlı ve verimlidir ve belirli bir netlik kaybıyla genel hale getirilebilir.
