Bir değişkenin veya işlevin kapsamını ayarlamak için bir depolama sınıfı kullanılır. Bir değişkenin depolama sınıfını bilerek, programın çalışma süresi boyunca o değişkenin ömrünü belirleyebiliriz.

Sözdizimi #

  • [otomatik|kayıt|statik|harici] <Veri türü> <Değişken adı>[ = <Değer>];

  • [statik _Thread_local|harici _Thread_local|_Thread_local] <Veri türü> <Değişken adı>[ = <Değer>]; /* >=C11’den beri */

  • Örnekler:

  • typedef int foo;

  • harici int foo[2];

Depolama sınıfı belirteçleri, bir bildirimin üst düzey türünün yanında görünebilen anahtar sözcüklerdir. Bu anahtar sözcüklerin kullanımı, dosya kapsamında mı yoksa blok kapsamında mı bildirildiğine bağlı olarak, bildirilen nesnenin depolama süresini ve bağlantısını etkiler:

Anahtar Kelime Depolama Süresi Bağlantı Açıklamalar
‘statik’ Statik Dahili Dosya kapsamındaki nesneler için dahili bağlantıyı ayarlar; blok kapsamındaki nesneler için statik depolama süresini ayarlar.
“dış” Statik Harici Örtülü ve dolayısıyla dosya kapsamında tanımlanan ve ayrıca bir başlatıcıya sahip olan nesneler için gereksizdir. Başlatıcı olmadan dosya kapsamındaki bir bildirimde kullanıldığında, tanımın başka bir çeviri biriminde bulunacağını ve bağlantı zamanında çözüleceğini gösterir.
‘otomatik’ Otomatik alakasız Örtülü ve dolayısıyla blok kapsamında bildirilen nesneler için gereksiz.
‘kayıt’ Otomatik alakasız Yalnızca otomatik depolama süresi olan nesnelerle ilgilidir. Değişkenin bir kayıt defterinde saklanması gerektiğine dair bir ipucu sağlar. Dayatılan bir kısıtlama, böyle bir nesne üzerinde tekli “&” “adresi” operatörünün kullanılamamasıdır ve bu nedenle nesnenin diğer adı olamaz.
’typedef' alakasız alakasız Pratikte bir depolama sınıfı belirteci değildir, ancak sözdizimsel bir bakış açısından biri gibi çalışır. Tek fark, bildirilen tanımlayıcının bir nesneden ziyade bir tür olmasıdır.
_Thread_local Konu Dahili/harici İş parçacığı depolama süresini temsil etmek için C11’de sunulmuştur. Blok kapsamında kullanılıyorsa, “harici” veya “statik” de içerecektir.

Her nesnenin ilişkili bir depolama süresi (kapsamdan bağımsız olarak) ve bağlantısı (yalnızca dosya kapsamındaki bildirimlerle ilgili) vardır, bu anahtar sözcükler atlanmış olsa bile.

Depolama sınıfı belirteçlerinin üst düzey tür belirteçlerine (‘int’, ‘unsigned’, ‘short’ vb.) ve üst düzey tür niteleyicilerine (‘const’, ‘volatile’) göre sıralaması zorunlu değildir, bu nedenle bu beyanların her ikisi de geçerlidir:

int static const unsigned a = 5; /* bad practice */
static const unsigned int b = 5; /* good practice */

Bununla birlikte, önce depolama sınıfı belirteçlerini, ardından herhangi bir tür niteleyicisini, ardından tür belirtecini (“void”, “char”, “int”, “signed long”, “unsigned long long”, “unsigned long long”, " uzun çift’…).

Tüm depolama sınıfı belirteçleri belirli bir kapsamda yasal değildir:

register int x; /* legal at block scope, illegal at file scope */
auto int y; /* same */

static int z; /* legal at both file and block scope */
extern int a; /* same */

extern int b = 5; /* legal and redundant at file scope, illegal at block scope */

/* legal because typedef is treated like a storage class specifier syntactically */
int typedef new_type_name;

Depolama Süresi

Depolama süresi statik veya otomatik olabilir. Bildirilen bir nesne için, kapsamına ve depolama sınıfı belirteçlerine bağlı olarak belirlenir.

Statik Depolama Süresi

Statik depolama süreli değişkenler, programın tüm yürütülmesi boyunca yaşar ve hem dosya kapsamında (‘statik’ olsun ya da olmasın) hem de blok kapsamında (açıkça ‘statik’ koyarak) bildirilebilir. Genellikle program başlangıcında işletim sistemi tarafından tahsis edilir ve başlatılırlar ve süreç sona erdiğinde geri alınırlar. Pratikte, yürütülebilir formatların bu tür değişkenler için ayrılmış bölümleri vardır (‘data’, ‘bss’ ve ‘rodata’) ve dosyadaki tüm bu bölümler belirli aralıklarda belleğe eşlenir.

İş parçacığı Depolama Süresi

Bu saklama süresi C11'de tanıtıldı. Bu, daha önceki C standartlarında mevcut değildi. Bazı derleyiciler, benzer semantik ile standart olmayan bir uzantı sağlar. Örneğin, gcc, "_Thread_local" içermeyen önceki C standartlarında kullanılabilen "__thread" belirtecini destekler. son sürüm İş parçacığı depolama süresine sahip değişkenler hem dosya kapsamında hem de blok kapsamında bildirilebilir. Blok kapsamında bildirilmişse, aynı zamanda "statik" veya "harici" depolama belirteci kullanacaktır. Ömrü, içinde oluşturulduğu *iş parçacığının* yürütmesinin tamamıdır. Bu, başka bir depolama belirtecinin yanında görünebilen tek depolama belirtecidir.

Otomatik Depolama Süresi

Otomatik saklama süreli değişkenler sadece blok kapsamında (doğrudan bir fonksiyon içinde veya o fonksiyondaki bir blok içinde) bildirilebilir. Yalnızca fonksiyona veya bloğa giriş ve çıkış arasındaki dönemde kullanılabilirler. Değişken kapsam dışına çıktığında (işlevden dönerek veya bloktan ayrılarak), depolaması otomatik olarak serbest bırakılır. İşaretçilerden aynı değişkene yapılan diğer referanslar geçersizdir ve tanımsız davranışa yol açar.

Tipik uygulamalarda, otomatik değişkenler, bir fonksiyonun yığın çerçevesinde veya kayıtlarda belirli ofsetlerde bulunur.

Dış ve Dahili Bağlantı

Bağlantı yalnızca dosya kapsamında bildirilen nesnelerle (işlevler ve değişkenler) ilgilidir ve farklı çeviri birimlerinde görünürlüklerini etkiler. Harici bağlantıya sahip nesneler, diğer tüm çeviri birimlerinde görünür (uygun bildirimin dahil edilmesi şartıyla). Dahili bağlantılı nesneler, diğer çeviri birimlerine maruz kalmaz ve yalnızca tanımlandıkları çeviri biriminde kullanılabilir.

Oto #

Bu depolama sınıfı, bir tanımlayıcının otomatik depolama süresine sahip olduğunu belirtir. Bu, tanımlayıcının tanımlandığı kapsam sona erdiğinde, tanımlayıcı tarafından belirtilen nesnenin artık geçerli olmadığı anlamına gelir.

Global kapsamda yaşamayan veya ‘static’ olarak bildirilmeyen tüm nesneler, tanımlandığında varsayılan olarak otomatik depolama süresine sahip olduğundan, bu anahtar kelime çoğunlukla tarihsel açıdan önemlidir ve kullanılmamalıdır:

int foo(void)
{
    /* An integer with automatic storage duration. */
    auto int i = 3;

    /* Same */
    int j = 5;

    return 0;
} /* The values of i and j are no longer able to be used. */

Kayıt ol #

Derleyiciye bir nesneye erişimin mümkün olduğu kadar hızlı olması gerektiğine dair ipuçları. Derleyicinin ipucunu gerçekten kullanıp kullanmadığı uygulama tanımlıdır; basitçe “auto” ile eşdeğer olarak ele alabilir.

‘Kayıt’ ile bildirilen tüm nesneler için kesin olarak farklı olan tek özellik, adreslerinin hesaplanamamasıdır. Bu nedenle “kayıt”, belirli optimizasyonları sağlamak için iyi bir araç olabilir:

register size_t size = 467;

hiçbir kod, adresini beklenmedik şekilde değiştirilebileceği başka bir işleve iletemeyeceğinden asla takma ad veremeyen bir nesnedir.

Bu özellik aynı zamanda bir dizi

register int array[5];

ilk elemanına bir işaretçiye dönüşemez (yani “dizi"nin “&dizi[0]“ye dönüşmesi). Bu, böyle bir dizinin öğelerine erişilemeyeceği ve dizinin kendisinin bir işleve geçirilemeyeceği anlamına gelir.

Aslında, bir “register” depolama sınıfıyla bildirilen bir dizinin tek yasal kullanımı “sizeof” operatörüdür; başka herhangi bir operatör, dizinin ilk öğesinin adresini gerektirir. Bu nedenle, diziler genellikle ‘register’ anahtar sözcüğüyle bildirilmemelidir, çünkü bu dizileri, ‘register’ anahtar sözcüğü olmadan da kolayca yapılabilen tüm dizinin boyut hesaplaması dışında hiçbir şey için işe yaramaz hale getirir.

register depolama sınıfı, bir blok içinde tanımlanan ve yüksek frekansla erişilen değişkenler için daha uygundur. Örneğin,

/* prints the sum of the first 5 integers*/
/* code assumed to be part of a function body*/ 
{ 
    register int k, sum;
    for(k = 1, sum = 0; k < 6; sum += k, k++);
        printf("\t%d\n",sum);
}

‘_Alignof’ operatörünün, ‘register’ dizileriyle birlikte kullanılmasına da izin verilir.

son sürüm

statik #

“Statik” depolama sınıfı, bildirimin dosyadaki konumuna bağlı olarak farklı amaçlara hizmet eder:

  1. Tanımlayıcıyı yalnızca o çeviri birimi ile sınırlamak için (kapsam=dosya).

    /* No other translation unit can use this variable. */
static int i;

/* Same; static is attached to the function type of f, not the return type int. */
static int f(int n);

  2. Bir fonksiyonun sonraki çağrısıyla (kapsam=blok) kullanmak üzere verileri kaydetmek için:

     void foo()
 {
     static int a = 0; /* has static storage duration and its lifetime is the
                        * entire execution of the program; initialized to 0 on 
                        * first function call */ 
     int b = 0; /* b has block scope and has automatic storage duration and 
                 * only "exists" within function */

     a += 10;
     b += 10; 

     printf("static int a = %d, int b = %d\n", a, b);
 }

 int main(void)
 {
     int i;
     for (i = 0; i < 5; i++)
     {
         foo();
     }

     return 0;
 }

    This code prints:

     static int a = 10, int b = 10
 static int a = 20, int b = 10
 static int a = 30, int b = 10
 static int a = 40, int b = 10
 static int a = 50, int b = 10

Statik değişkenler, birden çok farklı iş parçacığından çağrıldıklarında bile değerlerini korurlar.

  1. Bir diziyi belirtmek için işlev parametrelerinde kullanıldığında, sabit bir minimum öğe sayısına ve boş olmayan bir parametreye sahip olması beklenir:

    /* a is expected to have at least 512 elements. */
void printInts(int a[static 512])
{
    size_t i;
    for (i = 0; i < 512; ++i)
        printf("%d\n", a[i]);
}

Gerekli sayıda öğe (veya boş olmayan bir işaretçi) mutlaka derleyici tarafından kontrol edilmez ve yeterli öğeniz yoksa derleyicilerin sizi herhangi bir şekilde bilgilendirmesi gerekmez. Bir programcı 512’den az öğe veya boş gösterici geçerse, sonuç tanımsız davranıştır. Bunu zorlamak imkansız olduğundan, böyle bir fonksiyona o parametre için bir değer aktarılırken ekstra özen gösterilmelidir.

son sürüm

typedef #

Mevcut bir türe dayalı olarak yeni bir tür tanımlar. Sözdizimi, bir değişken bildiriminin sözdizimini yansıtır.

/* Byte can be used wherever `unsigned char` is needed */
typedef unsigned char Byte;

/* Integer is the type used to declare an array consisting of a single int */
typedef int Integer[1];

/* NodeRef is a type used for pointers to a structure type with the tag "node" */
typedef struct node *NodeRef;

/* SigHandler is the function pointer type that gets passed to the signal function. */
typedef void (*SigHandler)(int);

Teknik olarak bir depolama sınıfı olmasa da, typedef anahtar sözcüğü kullanılırsa diğer depolama sınıflarından hiçbirine izin verilmediğinden derleyici bunu bir sınıf olarak değerlendirecektir.

“typedef” önemlidir ve “#define” makrosu ile değiştirilmemelidir.

typedef int newType; 
newType *ptr;        // ptr is pointer to variable of type 'newType' aka int

Yine de,

#define int newType
newType *ptr;        // Even though macros are exact replacements to words, this doesn't result to a pointer to variable of type 'newType' aka int

dış #

Başka bir yerde tanımlanmış (ve harici bağlantıya sahip) bir nesne veya işlev bildirmek için kullanılır. Genel olarak, ilgili nesnenin veya işlevin tanımlandığı modülde olmayan bir modülde kullanılacak bir nesneyi veya işlevi bildirmek için kullanılır:

/* file1.c */
int foo = 2;  /* Has external linkage since it is declared at file scope. */

/* file2.c */
#include <stdio.h>
int main(void)
{
    /* `extern` keyword refers to external definition of `foo`. */
    extern int foo;
    printf("%d\n", foo);
    return 0;
}

C99’da “inline” anahtar sözcüğünün eklenmesiyle işler biraz daha ilginçleşiyor:

/* Should usually be place in a header file such that all users see the definition */
/* Hints to the compiler that the function `bar` might be inlined */
/* and suppresses the generation of an external symbol, unless stated otherwise. */
inline void bar(int drink)
{
    printf("You ordered drink no.%d\n", drink);
}

/* To be found in just one .c file.
   Creates an external function definition of `bar` for use by other files.
   The compiler is allowed to choose between the inline version and the external
   definition when `bar` is called. Without this line, `bar` would only be an inline
   function, and other files would not be able to call it. */
extern void bar(int);
son sürüm

_Thread_local #

Bu, çoklu iş parçacığı ile birlikte C11’de tanıtılan yeni bir depolama belirteciydi. Bu, önceki C standartlarında mevcut değildir.

iş parçacığı depolama süresini belirtir. ‘_Thread_local’ depolama belirteci ile bildirilen bir değişken, nesnenin o iş parçacığı için yerel olduğunu ve ömrünün, içinde oluşturulduğu iş parçacığının tüm yürütmesi olduğunu belirtir. Ayrıca “statik” veya “harici” ile birlikte görünebilir.

#include <threads.h>
#include <stdio.h>
#define SIZE 5

int thread_func(void *id)
{
    /* thread local variable i. */
    static _Thread_local int i;

    /* Prints the ID passed from main() and the address of the i.
     * Running this program will print different addresses for i, showing
     * that they are all distinct objects. */
    printf("From thread:[%d], Address of i (thread local): %p\n", *(int*)id, (void*)&i);

    return 0;
}

int main(void)
{
    thrd_t id[SIZE];
    int arr[SIZE] = {1, 2, 3, 4, 5};

    /* create 5 threads. */
    for(int i = 0; i < SIZE; i++) {
        thrd_create(&id[i], thread_func, &arr[i]);
    }

    /* wait for threads to complete. */
    for(int i = 0; i < SIZE; i++) {
        thrd_join(id[i], NULL);
    }
}
