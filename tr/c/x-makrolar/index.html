  • printfs için X makrolarının önemsiz kullanımı
  • Uzantı: X makrosunu bağımsız değişken olarak verin
  • Numaralandırma Değeri ve Tanımlayıcı
  • Kod oluşturma
  • Burada, 4 komut içeren bir numaralandırma ve adlarının bir haritasını dize olarak bildirmek için X-makrolarını kullanıyoruz
  • Benzer şekilde, enum değerine göre fonksiyonları çağırmak için bir atlama tablosu oluşturabiliriz.

X-makroları, tekrar eden kodları en aza indirmek ve veri/kod yazışmalarını sürdürmek için önişlemci tabanlı bir tekniktir. Ortak bir veri kümesine dayalı çok sayıda farklı makro genişletme, tüm genişletme grubunu tek bir ana makro aracılığıyla temsil ederek desteklenir; bu makronun değiştirme metni, her bir veri için bir tane olmak üzere bir iç makronun bir dizi genişletmesinden oluşur. İç makro geleneksel olarak ‘X()’ olarak adlandırılır, bu nedenle tekniğin adıdır.

X-makro tarzı bir ana makronun kullanıcısının, iç “X()” makrosu için kendi tanımını sağlaması ve bunun kapsamında ana makroyu genişletmesi beklenir. Master’ın iç makro referansları böylece kullanıcının ‘X()’ tanımına göre genişletilir. Bu şekilde, kaynak dosyadaki tekrarlayan ortak kod kodu miktarı azaltılabilir (‘X()’ yerine geçen metinde yalnızca bir kez görünür), “Kendinizi Tekrar Etmeyin” (DRY) taraftarlarının tercih ettiği gibi. Felsefe.

Ek olarak, “X()“i yeniden tanımlayarak ve ana makroyu bir veya daha fazla kez genişleterek, X makroları karşılık gelen verilerin ve kodun korunmasını kolaylaştırabilir - makronun bir genişletmesi verileri bildirir (örneğin dizi öğeleri veya numaralandırma üyeleri olarak) , ve diğer genişletmeler karşılık gelen kodu üretir.

“X-makro” adı, iç makronun geleneksel adından gelse de, teknik o özel isme bağlı değildir. Yerine herhangi bir geçerli makro adı kullanılabilir.

Eleştiriler şunları içerir:

  • X makrolarına dayanan kaynak dosyalarının okunması daha zordur;
  • tüm makrolar gibi, X makroları da kesinlikle metinseldir – doğal olarak herhangi bir tür güvenliği sağlamazlar; ve
  • X makroları kod oluşturma sağlar. Çağıran işlevlere dayalı alternatiflerle karşılaştırıldığında, X makroları kodu etkili bir şekilde büyütür.

X makrolarının iyi bir açıklaması, Randy Meyers’in Dr. Dobbs’daki [X-Macros] makalesinde bulunabilir (http://www.drdobbs.com/the-new-c-x-macros/184401387).

printfs için X makrolarının önemsiz kullanımı #

/* define a list of preprocessor tokens on which to call X */
#define X_123 X(1) X(2) X(3)

/* define X to use */
#define X(val) printf("X(%d) made this print\n", val);
X_123
#undef X
/* good practice to undef X to facilitate reuse later on */

Bu örnek, önişlemcinin aşağıdaki kodu oluşturmasına neden olacaktır:

printf("X(%d) made this print\n", 1);
printf("X(%d) made this print\n", 2);
printf("X(%d) made this print\n", 3);

Uzantı: X makrosunu bağımsız değişken olarak verin #

X-makro yaklaşımı, “X” makrosunun adını ana makronun bir argümanı yaparak biraz genelleştirilebilir. Bu, makro adı çakışmalarını önlemeye yardımcı olma ve genel amaçlı bir makronun “X” makrosu olarak kullanılmasına izin verme avantajlarına sahiptir.

X makrolarında her zaman olduğu gibi, ana makro, önemi o makroya özgü olan öğelerin bir listesini temsil eder. Bu varyasyonda, böyle bir makro şöyle tanımlanabilir:

/* declare list of items */
#define ITEM_LIST(X) \
      X(item1) \
      X(item2) \
      X(item3) \
/* end of list */

Daha sonra, öğe adlarını şu şekilde yazdırmak için kod üretilebilir:

/* define macro to apply */
#define PRINTSTRING(value) printf( #value "\n");

/* apply macro to the list of items */
ITEM_LIST(PRINTSTRING)

Bu, bu koda genişler:

printf( "item1" "\n"); printf( "item2" "\n"); printf( "item3" "\n");

“X” adının ana makronun yerleşik bir özelliği olduğu standart X makrolarının aksine, bu stille daha sonra argüman olarak kullanılan makronun tanımını kaldırmak gereksiz veya hatta istenmeyen bir durum olabilir (bu örnekte “PRINTSTRING” ).

Numaralandırma Değeri ve Tanımlayıcı #

/* declare items of the enum */
#define FOREACH \
      X(item1) \
      X(item2) \
      X(item3) \
/* end of list */

/* define the enum values */
#define X(id) MyEnum_ ## id,
enum MyEnum { FOREACH };
#undef X

/* convert an enum value to its identifier */
const char * enum2string(int enumValue)
{
    const char* stringValue = NULL;
#define X(id) if (enumValue == MyEnum_ ## id) stringValue = #id;
    FOREACH
#undef X
    return stringValue;
}

Ardından, kodunuzda numaralandırılmış değeri kullanabilir ve tanımlayıcısını aşağıdakileri kullanarak kolayca yazdırabilirsiniz:

printf("%s\n", enum2string(MyEnum_item2));

Kod oluşturma #

X-Makrolar, tekrarlayan kod yazarak kod üretimi için kullanılabilir: bazı görevleri yapmak veya bir dizi sabit, nesne veya işlev bildirmek için bir liste üzerinde yineleme yapın.

Burada, 4 komut içeren bir numaralandırma ve adlarının bir haritasını dize olarak bildirmek için X-makrolarını kullanıyoruz #

Daha sonra enumun string değerlerini yazdırabiliriz.

/* All our commands */
#define COMMANDS(OP) OP(Open) OP(Close) OP(Save) OP(Quit)

/* generate the enum Commands: {cmdOpen, cmdClose, cmdSave, cmdQuit, }; */
#define ENUM_NAME(name) cmd##name,
enum Commands {
  COMMANDS(ENUM_NAME)
};
#undef ENUM_NAME

/* generate the string table */
#define COMMAND_OP(name) #name,
const char* const commandNames[] = {
  COMMANDS(COMMAND_OP)
};
#undef COMMAND_OP

/* the following prints "Quit\n": */
printf("%s\n", commandNames[cmdQuit]());

Benzer şekilde, enum değerine göre fonksiyonları çağırmak için bir atlama tablosu oluşturabiliriz. #

Bu, tüm işlevlerin aynı imzaya sahip olmasını gerektirir. Argüman almazlar ve bir int döndürürlerse, bunu enum tanımıyla bir başlığa koyarız:

/* declare all functions as extern */
#define EXTERN_FUNC(name) extern int doCmd##name(void);
COMMANDS(EXTERN_FUNC)
#undef EXTERN_FUNC

/* declare the function pointer type and the jump table  */
typedef int (*CommandFunc)(void);
extern CommandFunc commandJumpTable[];

Aşağıdakilerin tümü, yukarıdaki bölümün bir başlık olarak dahil edildiği varsayılarak farklı derleme birimlerinde olabilir:

/* generate the jump table */
#define FUNC_NAME(name) doCmd##name,
CommandFunc commandJumpTable[] = {
  COMMANDS(FUNC_NAME)
};
#undef FUNC_NAME

/* call the save command like this: */
int result = commandJumpTable[cmdSave]();

/* somewhere else, we need the implementations of the commands */
int doCmdOpen(void) {/* code performing open command */}
int doCmdClose(void) {/* code performing close command */}
int doCmdSave(void) {/* code performing save command */}
int doCmdQuit(void) {/* code performing quit command */}

Gerçek kodda kullanılan bu tekniğin bir örneği GPU komutu içindir. Chromium’da gönderme.
