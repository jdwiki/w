Sözdizimi #

  • void (*signal(int sig, void (*fonk)(int)))(int);

Parametreler #

parametre Ayrıntılar
imza Sinyal işleyiciyi SIGABRT, SIGFPE, SIGILL, SIGTERM, SIGINT, SIGSEGV veya bazı uygulama tanımlı değerlerden birine ayarlayacak sinyal
işlev Aşağıdakilerden biri olan sinyal işleyici: Varsayılan işleyici için “SIG_DFL”, sinyali yok saymak için “SIG_IGN” veya “void foo(int sig);” imzasına sahip bir işlev işaretçisi.

Sadece C standardından garanti edilen sinyal işleyicilerin kullanımı, kullanıcı tanımlı sinyal işleyicide yapılabilecek veya yapılamayacak çeşitli sınırlamalar getirir.

  • “SIGSEGV”, “SIGFPE”, “SIGILL” veya herhangi bir uygulama tanımlı donanım kesmesini işlerken kullanıcı tanımlı işlev dönerse, davranış C standardı tarafından tanımsızdır. Bunun nedeni, C’nin arayüzünün hatalı durumu değiştirmek için bir araç vermemesidir (örneğin ‘0’a bölündükten sonra) ve bu nedenle program döndürülürken donanım kesintisi meydana gelmeden öncekiyle tamamen aynı hatalı durumda olur.

  • Eğer kullanıcı tanımlı fonksiyon bir “abort” veya “yükselt” çağrısının sonucu olarak çağrıldıysa, sinyal işleyicinin tekrar “yükselt” çağrısı yapmasına izin verilmez.

  • Sinyaller herhangi bir işlemin ortasında gelebilir ve bu nedenle işlemlerin bölünemezliği genellikle garanti edilemez ve sinyal işleme optimizasyon ile iyi çalışmaz. Bu nedenle, bir sinyal işleyicideki verilerde yapılan tüm değişiklikler, değişkenlerde olmalıdır.

  • “sig_atomic_t” türünden (tüm sürümler) veya kilitsiz atom türünden (C11’den beri, isteğe bağlı)

  • “uçucu” nitelikli olanlar.

  • C standart kitaplığındaki diğer işlevler, programın genel durumundaki değişkenleri değiştirebileceklerinden genellikle bu kısıtlamalara uymaz. C standardı yalnızca ‘abort’, ‘_Exit’ (C99’dan beri), ‘quick_exit’ (C11’den beri), ‘sinyal’ (aynı sinyal numarası için) ve bazı atomik işlemler (C11’den beri) için garanti verir.

Yukarıdaki kurallardan herhangi biri ihlal edilirse, davranış C standardı tarafından tanımlanmaz. Platformların belirli uzantıları olabilir, ancak bunlar genellikle bu platformun ötesine taşınabilir değildir.

  • Genellikle sistemlerin, asenkron sinyal güvenli, yani bir sinyal işleyiciden kullanılabilen C kitaplığı işlevlerinden oluşan kendi işlev listesi vardır. Örneğin sıklıkla printf bu işlevler arasındadır.

  • Özellikle C standardı, iş parçacığı arabirimi (C11’den beri) veya POSIX iş parçacıkları gibi herhangi bir platforma özgü iş parçacığı kitaplıkları ile etkileşim hakkında fazla bir şey tanımlamaz. Bu tür platformlar, bu tür iplik kitaplıklarının sinyallerle etkileşimini kendileri belirlemelidir.

“Sinyal()” ile Sinyal İşleme #

Sinyal numaraları programın kendisinde bir arıza nedeniyle tetiklendiklerinde eşzamanlı (“SIGSEGV” - segmentasyon hatası gibi) veya asenkron (“SIGINT” gibi) olabilir. ` - etkileşimli dikkat) programın dışından başlatıldıklarında, örneğin ‘Cntrl-C’ olarak bir tuşa basılarak.

sinyal() işlevi, ISO C standardının bir parçasıdır ve belirli bir sinyali işlemek üzere bir işlev atamak için kullanılabilir.

#include <stdio.h>  /* printf() */
#include <stdlib.h> /* abort()  */
#include <signal.h> /* signal() */

void handler_nonportable(int sig)
{
    /* undefined behavior, maybe fine on specific platform */
    printf("Catched: %d\n", sig);
    
    /* abort is safe to call */
    abort();
}

sig_atomic_t volatile finished = 0;

void handler(int sig)
{
    switch (sig) {
    /* hardware interrupts should not return */
    case SIGSEGV:
    case SIGFPE:
    case SIGILL:

      /* quick_exit is safe to call */
      quick_exit(EXIT_FAILURE);
son sürüm 
      /* use _Exit in pre-C11 */
      _Exit(EXIT_FAILURE);
son sürüm 
    default:
       /* Reset the signal to the default handler, 
          so we will not be called again if things go
          wrong on return. */
      signal(sig, SIG_DFL);
      /* let everybody know that we are finished */
      finished = sig;
      return;
   }
}

int main(void)
{

    /* Catch the SIGSEGV signal, raised on segmentation faults (i.e NULL ptr access */
    if (signal(SIGSEGV, &handler) == SIG_ERR) {
        perror("could not establish handler for SIGSEGV");
        return EXIT_FAILURE;
    }

    /* Catch the SIGTERM signal, termination request */
    if (signal(SIGTERM, &handler) == SIG_ERR) {
        perror("could not establish handler for SIGTERM");
        return EXIT_FAILURE;
    }

    /* Ignore the SIGINT signal, by setting the handler to `SIG_IGN`. */
    signal(SIGINT, SIG_IGN);


    /* Do something that takes some time here, and leaves
       the time to terminate the program from the keyboard. */

    /* Then: */

    if (finished) {
       fprintf(stderr, "we have been terminated by signal %d\n", (int)finished);
        return EXIT_FAILURE;
    }


    /* Try to force a segmentation fault, and raise a SIGSEGV */
    {
      char* ptr = 0;
      *ptr = 0;
    }

    /* This should never be executed */
    return EXIT_SUCCESS;
}

signal() kullanmak, sinyal işleyiciler içinde yapmanıza izin verilen önemli sınırlamalar getirir, daha fazla bilgi için açıklamalara bakın.

POSIX, yeterince belirtilmemiş davranışı ve önemli uygulama varyasyonları nedeniyle ‘signal()’ yerine ‘sigaction()’ kullanılmasını önerir. POSIX ayrıca, programcı tarafından herhangi bir amaç için ücretsiz olarak kullanılabilen “SIGUSR1” ve “SIGUSR2” dahil olmak üzere ISO C standardından çok daha fazla sinyal tanımlar. .
