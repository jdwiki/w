Komut satırı bağımsız değişkenleri

Sözdizimi #

  • int main(int argc, char *argv[])

Parametreler #

parametre Ayrıntılar
argüman argüman sayısı - program adının yanı sıra komut satırından programa verilen boşlukla ayrılmış argümanların sayısıyla başlatılır.
bağımsız değişken argüman vektörü - komut satırında verilen argümanları (ve program adını) içeren bir “char” işaretçileri (dizeler) dizisine başlatılır.

“Barındırılan bir ortamda” çalışan bir C programının (“bağımsız ortamın” aksine normal tür) bir “ana” işlevi olmalıdır. Geleneksel olarak şu şekilde tanımlanır:

int main(int argc, char *argv[])

“argv"nin aynı zamanda “char **argv” olarak da tanımlanabileceğini ve sıklıkla tanımlandığını unutmayın; davranış aynıdır. Ayrıca, parametre adları değiştirilebilir, çünkü bunlar işlev içinde yalnızca yerel değişkenlerdir, ancak ‘argc’ ve ‘argv’ gelenekseldir ve bu adları kullanmalısınız.

Kodun herhangi bir argüman kullanmadığı ‘main’ işlevleri için ‘int main(void)’ kullanın.

Program başladığında her iki parametre de başlatılır:

  • ‘argc’, program adının yanı sıra komut satırından programa verilen boşlukla ayrılmış argümanların sayısıyla başlatılır.
  • “argv”, komut satırında verilen bağımsız değişkenleri (ve program adını) içeren bir “char” işaretçileri (dizeler) dizisidir.
  • bazı sistemler komut satırı argümanlarını “kabukta” genişletir, diğerleri yapmaz. Unix’te, kullanıcı programım *.txt yazarsa, program metin dosyalarının bir listesini alır; Windows’ta “*.txt” dizesini alacaktır.

Not: “argv"yi kullanmadan önce “argc"nin değerini kontrol etmeniz gerekebilir. Teoride, “argc” “0” olabilir ve “argc” sıfırsa, argüman yoktur ve “argv[0]” (“argv[argc]“ye eşdeğer) bir boş göstericidir. Bu sorunla karşılaşırsanız, barındırılan bir ortama sahip olağandışı bir sistem olurdu. Benzer şekilde, çok sıra dışı olsa da, program adı hakkında hiçbir bilgi olmaması mümkündür. Bu durumda, argv[0][0] == '\0' — program adı boş olabilir.

Programı şöyle başlattığımızı varsayalım:

./some_program abba banana mamajam

O zaman “argc”, “4"e eşittir ve komut satırı argümanları:

  • ‘argv[0]’, program adı ana bilgisayar ortamında mevcutsa, "./some_program" (program adı) öğesini gösterir. Aksi takdirde boş bir dize "".
  • ‘argv[1]’, ‘“abba”‘yı işaret eder,
  • ‘argv[2]’, ‘“muz”‘u işaret eder,
  • ‘argv[3]’, ‘“mamajam”‘ı işaret eder,
  • ‘argv[4]’, ‘NULL’ değerini içerir.

Standarttan tam alıntılar için ayrıca bkz.

Argümanları bir programa yazdırın ve tamsayı değerlerine dönüştürün #

Aşağıdaki kod, argümanları programa yazdıracak ve kod, her argümanı bir sayıya (bir uzun):

#include <stdlib.h>
#include <stdio.h>
#include <errno.h>
#include <limits.h>

int main(int argc, char* argv[]) {

    for (int i = 1; i < argc; i++) {
        printf("Argument %d is: %s\n", i, argv[i]);

        errno = 0;
        char *p;
        long argument_numValue = strtol(argv[i], &p, 10);

        if (p == argv[i]) {
            fprintf(stderr, "Argument %d is not a number.\n", i);
        }
        else if ((argument_numValue == LONG_MIN || argument_numValue == LONG_MAX) && errno == ERANGE) {
            fprintf(stderr, "Argument %d is out of range.\n", i);
        }
        else {
            printf("Argument %d is a number, and the value is: %ld\n",
                   i, argument_numValue);
        }
    }
    return 0;
}

Komut satırı argümanlarını yazdırma #

Argümanları aldıktan sonra, bunları aşağıdaki gibi yazdırabilirsiniz:

int main(int argc, char **argv)
{
    for (int i = 1; i < argc; i++)
    {
        printf("Argument %d: [%s]\n", i, argv[i]); 
    }
}

Notlar

  1. ‘argv’ parametresi ‘char *argv[]’ olarak da tanımlanabilir.
  2. argv[0] program adının kendisini içerebilir (programın nasıl yürütüldüğüne bağlı olarak). İlk “gerçek” komut satırı argümanı “argv[1]” konumundadır ve “i” döngü değişkeninin 1 olarak başlatılmasının nedeni budur.
  3. print deyiminde, ‘argv[i]’ yerine ‘*(argv + i)’ kullanabilirsiniz - aynı şeyi değerlendirir, ancak daha ayrıntılıdır.
  4. Argüman değerinin etrafındaki köşeli parantezler, başlangıç ​​ve bitişin belirlenmesine yardımcı olur. Argümanda sondaki boşluklar, yeni satırlar, satır başları veya diğer tuhaf karakterler varsa bu çok değerli olabilir. Bu programdaki bazı varyantlar, argüman listesinin gerçekte ne içerdiğini anlamanız gereken (neredeyse eşdeğer basit kabuk alternatifleri olsa da) kabuk komut dosyalarında hata ayıklamak için kullanışlı bir araçtır.

GNU getopt araçlarını kullanma #

Uygulamalar için komut satırı seçenekleri, C dili tarafından komut satırı argümanlarından farklı bir şekilde ele alınmaz. Linux veya Unix ortamında geleneksel olarak bir tire (-) ile başlayan argümanlardır.

Linux veya Unix ortamında glibc ile komut satırını kolayca tanımlamak, doğrulamak ve ayrıştırmak için getopt araçlarını kullanabilirsiniz. argümanlarınızın geri kalanından seçenekler.

Bu araçlar, seçeneklerinizin GNU Kodlama Standartlarına göre biçimlendirilmesini bekler; bu, POSIX’in aşağıdakiler için belirlediğinin bir uzantısıdır. komut satırı seçeneklerinin biçimi.

Aşağıdaki örnek, GNU getopt araçlarıyla komut satırı seçeneklerinin ele alınmasını gösterir.

#include <stdio.h>
#include <getopt.h>
#include <string.h>

/* print a description of all supported options */
void usage (FILE *fp, const char *path)
{
    /* take only the last portion of the path */
    const char *basename = strrchr(path, '/');
    basename = basename ? basename + 1 : path;

    fprintf (fp, "usage: %s [OPTION]\n", basename);
    fprintf (fp, "  -h, --help\t\t"
                 "Print this help and exit.\n");
    fprintf (fp, "  -f, --file[=FILENAME]\t"
                 "Write all output to a file (defaults to out.txt).\n");
    fprintf (fp, "  -m, --msg=STRING\t"
                 "Output a particular message rather than 'Hello world'.\n");
}

/* parse command-line options and print message */
int main(int argc, char *argv[])
{
    /* for code brevity this example just uses fixed buffer sizes for strings */
    char filename[256] = { 0 };
    char message[256] = "Hello world";
    FILE *fp;
    int help_flag = 0;
    int opt;

    /* table of all supported options in their long form.
     * fields: name, has_arg, flag, val
     * `has_arg` specifies whether the associated long-form option can (or, in
     * some cases, must) have an argument. the valid values for `has_arg` are
     * `no_argument`, `optional_argument`, and `required_argument`.
     * if `flag` points to a variable, then the variable will be given a value
     * of `val` when the associated long-form option is present at the command
     * line.
     * if `flag` is NULL, then `val` is returned by `getopt_long` (see below)
     * when the associated long-form option is found amongst the command-line
     * arguments.
     */
    struct option longopts[] = {
        { "help", no_argument, &help_flag, 1 },
        { "file", optional_argument, NULL, 'f' },
        { "msg", required_argument, NULL, 'm' },
        { 0 }
    };

    /* infinite loop, to be broken when we are done parsing options */
    while (1) {
        /* getopt_long supports GNU-style full-word "long" options in addition
         * to the single-character "short" options which are supported by
         * getopt.
         * the third argument is a collection of supported short-form options.
         * these do not necessarily have to correlate to the long-form options.
         * one colon after an option indicates that it has an argument, two
         * indicates that the argument is optional. order is unimportant.
         */
        opt = getopt_long (argc, argv, "hf:ⓜ️", longopts, 0);

        if (opt == -1) {
            /* a return value of -1 indicates that there are no more options */
            break;
        }

        switch (opt) {
        case 'h':
            /* the help_flag and value are specified in the longopts table,
             * which means that when the --help option is specified (in its long
             * form), the help_flag variable will be automatically set.
             * however, the parser for short-form options does not support the
             * automatic setting of flags, so we still need this code to set the
             * help_flag manually when the -h option is specified.
             */
            help_flag = 1;
            break;
        case 'f':
            /* optarg is a global variable in getopt.h. it contains the argument
             * for this option. it is null if there was no argument.
             */
            printf ("outarg: '%s'\n", optarg);
            strncpy (filename, optarg ? optarg : "out.txt", sizeof (filename));
            /* strncpy does not fully guarantee null-termination */
            filename[sizeof (filename) - 1] = '\0';
            break;
        case 'm':
            /* since the argument for this option is required, getopt guarantees
             * that aptarg is non-null.
             */
            strncpy (message, optarg, sizeof (message));
            message[sizeof (message) - 1] = '\0';
            break;
        case '?':
            /* a return value of '?' indicates that an option was malformed.
             * this could mean that an unrecognized option was given, or that an
             * option which requires an argument did not include an argument.
             */
            usage (stderr, argv[0]);
            return 1;
        default:
            break;
        }
    }

    if (help_flag) {
        usage (stdout, argv[0]);
        return 0;
    }

    if (filename[0]) {
        fp = fopen (filename, "w");
    } else {
        fp = stdout;
    }

    if (!fp) {
        fprintf(stderr, "Failed to open file.\n");
        return 1;
    }

    fprintf (fp, "%s\n", message);

    fclose (fp);
    return 0;
}

“gcc” ile derlenebilir:

gcc example.c -o example

Üç komut satırı seçeneğini destekler (’–help’, ‘–file’ ve ‘–msg’). Hepsinin de bir “kısa biçimi” vardır (-h, -f ve -m). “Dosya” ve “mesaj” seçeneklerinin her ikisi de bağımsız değişkenleri kabul eder. “Msg” seçeneğini belirtirseniz, argümanı gereklidir.

Seçenekler için bağımsız değişkenler şu şekilde biçimlendirilir:

  • `–option=value’ (uzun biçimli seçenekler için)
  • -ovalue veya -o"value" (kısa biçimli seçenekler için)
