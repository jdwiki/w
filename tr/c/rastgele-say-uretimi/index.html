“Rand()“ın kusurları nedeniyle, yıllar içinde birçok başka varsayılan uygulama ortaya çıktı. Bunlar arasında:

  • arc4random() (OS X ve BSD’de mevcuttur)
  • random() (Linux’ta mevcuttur)
  • drand48() (POSIX’te mevcuttur)

Temel Rastgele Sayı Üretimi #

“Rand()” işlevi, “0” ile “RAND_MAX” (“0” ve “RAND_MAX” dahil) arasında bir sözde rastgele tamsayı değeri oluşturmak için kullanılabilir.

srand(int), sözde rasgele sayı üretecini tohumlamak için kullanılır. “Rand()” aynı tohumla her ekildiğinde, aynı değer dizisini üretmelidir. ‘Rand()’ çağrılmadan önce yalnızca bir kez ekilmelidir. Yeni bir sözde rasgele sayı grubu oluşturmak istediğiniz her seferinde tekrar tekrar oluşturulmamalı veya yeniden oluşturulmamalıdır.

Standart uygulama, tohum olarak “time(NULL)” sonucunu kullanmaktır. Rastgele sayı üretecinizin deterministik bir sıraya sahip olması gerekiyorsa, üreteci her program başlangıcında aynı değerle tohumlayabilirsiniz. Bu genellikle sürüm kodu için gerekli değildir, ancak hataların yeniden üretilebilir olması için hata ayıklama çalıştırmalarında kullanışlıdır.

Jeneratörün her zaman tohumlanması tavsiye edilir, tohumlanmadıysa, `srand(1)’ ile tohumlanmış gibi davranır.

#include <stdio.h>
#include <stdlib.h>
#include <time.h>

int main(void) {
    int i;
    srand(time(NULL));
    i = rand();

    printf("Random value between [0, %d]: %d\n", RAND_MAX, i);
    return 0;
}

Olası çıktı:

Random value between [0, 2147483647]: 823321433

Notlar:

C Standardı, üretilen rastgele dizinin kalitesini garanti etmez. Geçmişte, bazı ‘rand()’ uygulamaları, üretilen sayıların dağılımı ve rastgeleliği konusunda ciddi sorunlar yaşıyordu. Kriptografi gibi ciddi rastgele sayı oluşturma ihtiyaçları için “rand()” kullanılması önerilmez.

İzinli Eşgüdümlü Üreteç #

İşte ‘rand()’ veya benzer kitaplık işlevlerine dayanmayan bağımsız bir rastgele sayı üreteci.

Neden böyle bir şey isteyesin ki? Belki platformunuzun yerleşik rasgele sayı üretecine güvenmiyorsunuzdur ya da herhangi bir belirli kitaplık uygulamasından bağımsız olarak yeniden üretilebilir bir rasgelelik kaynağı istiyorsunuz.

Bu kod, mükemmel istatistiksel özelliklere sahip modern, hızlı, genel amaçlı bir RNG olan pcg-random.org kaynaklı PCG32’dir. Kriptografik olarak güvenli değildir, bu yüzden kriptografi için kullanmayın.

#include <stdint.h>

/* *Really* minimal PCG32 code / (c) 2014 M.E. O'Neill / pcg-random.org
 * Licensed under Apache License 2.0 (NO WARRANTY, etc. see website) */

typedef struct { uint64_t state;  uint64_t inc; } pcg32_random_t;

uint32_t pcg32_random_r(pcg32_random_t* rng) {
    uint64_t oldstate = rng->state;
    /* Advance internal state */
    rng->state = oldstate * 6364136223846793005ULL + (rng->inc | 1);
    /* Calculate output function (XSH RR), uses old state for max ILP */
    uint32_t xorshifted = ((oldstate >> 18u) ^ oldstate) >> 27u;
    uint32_t rot = oldstate >> 59u;
    return (xorshifted >> rot) | (xorshifted << ((-rot) & 31));
}

void pcg32_srandom_r(pcg32_random_t* rng, uint64_t initstate, uint64_t initseq) {
    rng->state = 0U;
    rng->inc = (initseq << 1u) | 1u;
    pcg32_random_r(rng);
    rng->state += initstate;
    pcg32_random_r(rng);
}

Ve işte nasıl arayacağınız:

#include <stdio.h>
int main(void) {
    pcg32_random_t rng; /* RNG state */
    int i;

    /* Seed the RNG */
    pcg32_srandom_r(&rng, 42u, 54u);

    /* Print some random 32-bit integers */
    for (i = 0; i < 6; i++)
        printf("0x%08x\n", pcg32_random_r(&rng));
    
    return 0;
}

Oluşturmayı belirli bir aralıkla sınırla #

Genellikle rasgele sayılar üretirken, bir aralık içinde tamsayılar veya 0.0 ile 1.0 arasında bir p değeri oluşturmak yararlıdır. Modül işlemi, çekirdeği düşük bir tamsayıya indirgemek için kullanılabilirken, bu, genellikle kısa bir döngüden geçen düşük bitleri kullanır ve N, RAND_MAX ile orantılı olarak büyükse, dağılımın hafif bir eğrilmesine neden olur.

makro

#define uniform() (rand() / (RAND_MAX + 1.0))

0.0 ila 1.0 - epsilon arasında bir p değeri üretir, bu nedenle

i = (int)(uniform() * N)

“i"yi 0 ila N - 1 aralığında tek tip bir rastgele sayıya ayarlayacaktır.

Maalesef, RAND_MAX’ın “double” türündeki bir değişkenin doğru olarak gösterebileceğinden daha büyük olmasına izin verilmesinde teknik bir kusur var. Bu, “RAND_MAX + 1.0” öğesinin RAND_MAX olarak değerlendirildiği ve işlevin zaman zaman birlik döndürdüğü anlamına gelir. Ancak bu pek olası değildir.

Xorshift Nesil #

Kusurlu ‘rand()’ prosedürlerine iyi ve kolay bir alternatif, George Marsaglia tarafından keşfedilen bir sözde rasgele sayı üreteci sınıfı olan xorshift’tir. xorshift üreteci, kriptografik olarak güvenli olmayan en hızlı rastgele sayı üreteçleri arasındadır. Daha fazla bilgi ve diğer örnek uygulamalar [xorshift Wikipedia sayfasında2 mevcuttur.

Örnek uygulama

#include <stdint.h>

/* These state variables must be initialised so that they are not all zero. */
uint32_t w, x, y, z;

uint32_t xorshift128(void) 
{
    uint32_t t = x;
    t ^= t << 11U;
    t ^= t >> 8U;
    x = y; y = z; z = w;
    w ^= w >> 19U;
    w ^= t;
    return w;
}
