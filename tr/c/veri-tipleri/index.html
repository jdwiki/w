Sabit Genişlik Tamsayı Türleri (C99’dan beri) #

“<stdint.h>” başlığı, birkaç sabit genişlikli tamsayı türü tanımı sağlar. Bu türler isteğe bağlıdır ve yalnızca platformun karşılık gelen genişlikte bir tamsayı türü varsa ve karşılık gelen işaretli türün negatif değerlerin ikinin tümleyeni temsili varsa sağlanır.

Sabit genişlikli tiplerin kullanım ipuçları için açıklamalar bölümüne bakın.

/* commonly used types include */
uint32_t u32 = 32; /* exactly 32-bits wide */

uint8_t u8 = 255;  /* exactly 8-bits wide */

int64_t i64 = -65  /* exactly 64 bit in two's complement representation */

Beyanların Yorumlanması #

C’nin ayırt edici sözdizimsel özelliği, bildirimlerin, normal bir ifadede olacağı gibi, bildirilen nesnenin kullanımını yansıtmasıdır.

Aynı önceliğe ve ilişkilendirilebilirliğe sahip aşağıdaki operatörler, bildiricilerde yeniden kullanılır, yani:

  • bir işaretçiyi ifade eden tekli * “başvuru kaldırma” operatörü;
  • bir diziyi ifade eden ikili [] “dizi aboneliği” operatörü;
  • bir işlevi ifade eden (1+n)-ary () “işlev çağrısı” operatörü;
  • listelenen operatörlerin geri kalanının önceliğini ve ilişkilendirilebilirliğini geçersiz kılan () gruplandırma parantezleri.

Yukarıdaki üç operatör aşağıdaki önceliğe ve ilişkilendirilebilirliğe sahiptir:

Operatör Göreli Öncelik ilişkilendirme
[] (dizi aboneliği) 1 Soldan sağa
() (işlev çağrısı) 1 Soldan sağa
* (başvuru) 2 Sağdan sola

Bildirimleri yorumlarken, tanımlayıcıdan dışa doğru başlamalı ve bitişik operatörleri yukarıdaki tabloya göre doğru sırayla uygulamalıdır. Bir operatörün her uygulaması aşağıdaki İngilizce kelimelerle değiştirilebilir:

| ifade | Tercüme | | ————————- | ————————————————– - | | şey[X] | X büyüklüğünde bir dizi… | | şey(t1, t2, t3) | t1, t2, t3 alan ve dönen bir fonksiyon… | | *şey | bir işaretçi… |

İngilizce yorumun başlangıcı her zaman tanımlayıcı ile başlayacak ve bildirimin sol tarafında duran tür ile bitecektir.

Örnekler

char *names[20];

‘[]’, ‘*’ yerine önceliklidir, dolayısıyla yorum şöyledir: “names”, “char” işaretçisinin 20 boyutunda bir dizisidir.

char (*place)[10];

Önceliği geçersiz kılmak için parantez kullanılması durumunda, önce “*” uygulanır: “yer”, “char” boyutunun 10’luk bir dizisine işaretçidir.

int fn(long, short);

Burada endişelenecek bir öncelik yoktur: ‘fn’, ‘uzun’, ‘kısa’ alan ve ‘int’ döndüren bir işlevdir.

int *fn(void);

Önce “()” uygulanır: “fn”, “void” alan ve bir işaretçiyi “int"e döndüren bir işlevdir.

int (*fp)(void);

“()” önceliğini geçersiz kılmak: “fp”, “void” alan ve “int” döndüren bir işleve yönelik bir işaretçidir.

int arr[5][8];

Çok boyutlu diziler kuralın bir istisnası değildir; ‘[]’ operatörleri, tablodaki ilişkilendirmeye göre soldan sağa sırayla uygulanır: ‘arr’, ‘int’ boyutu 8 olan bir dizinin 5 boyutlu dizisidir.

int **ptr;

İki referans operatörü eşit önceliğe sahiptir, bu nedenle ilişkilendirme etkili olur. Operatörler sağdan sola sırayla uygulanır: “ptr”, bir “int” işaretçisinin işaretçisidir.

Birden Çok Bildirim

Virgül, tek bir ifadede birden çok bildirimi sınırlandırmak için ayırıcı olarak kullanılabilir (virgül operatörü gibi davranmaz*). Aşağıdaki beyan beş beyan içerir: 
int fn(void), *ptr, (*fp)(int), arr[10][20], num;

Yukarıdaki örnekte bildirilen nesneler şunlardır:

  • ‘fn’: ‘void’ alan ve ‘int’ döndüren bir işlev;
  • ‘ptr’: bir ‘int’ işaretçisi;
  • ‘fp’: ‘int’ alan ve ‘int’ döndüren bir işleve işaretçi;
  • “arr”: “int” boyutunda 20 boyutunda bir dizinin 10 boyutunda bir dizi;
  • sayı'': int’’

Alternatif Yorumlama

Bildirimler ayna kullanımı nedeniyle, bir bildirim, nesne üzerinde uygulanabilecek operatörler ve bu ifadenin nihai sonuç türü açısından da yorumlanabilir. Sol tarafta duran tip, tüm operatörler uygulandıktan sonra elde edilen nihai sonuçtur.

/*
 * Subscripting "arr" and dereferencing it yields a "char" result.
 * Particularly: *arr[5] is of type "char".
 */
char *arr[20];

/*
 * Calling "fn" yields an "int" result.
 * Particularly: fn('b') is of type "int".
 */
int fn(char);

/*
 * Dereferencing "fp" and then calling it yields an "int" result.
 * Particularly: (*fp)() is of type "int".
 */
int (*fp)(void);

/*
 * Subscripting "strings" twice and dereferencing it yields a "char" result.
 * Particularly: *strings[5][15] is of type "char"
 */
char *strings[10][20];

Tamsayı türleri ve sabitler #

İşaretli tamsayılar şu türlerde olabilir (“kısa"dan sonraki “int” veya “uzun” isteğe bağlıdır):

signed char c = 127; /* required to be 1 byte, see remarks for further information. */
signed short int si = 32767; /* required to be at least 16 bits. */
signed int i = 32767; /* required to be at least 16 bits */
signed long int li = 2147483647; /* required to be at least 32 bits. */

signed long long int li = 2147483647; /* required to be at least 64 bits */
son sürüm

Bu imzalı tamsayı türlerinin her birinin imzasız bir sürümü vardır.

unsigned int i = 65535;
unsigned short = 2767;
unsigned char = 255;

‘char’ dışındaki tüm türler için, ‘imzalı’ veya ‘imzasız’ kısım atlanırsa, ‘imzalı’ sürüm varsayılır. ‘char’ türü, ‘signed char’ ve ‘unsigned char’dan farklı olarak üçüncü bir karakter türünü oluşturur ve imzalılık (ya da değil) platforma bağlıdır.

Farklı türde tamsayı sabitleri (C jargonunda değişmez değerler olarak adlandırılır), öneklerine veya soneklerine göre farklı tabanlarda ve farklı genişliklerde yazılabilir.

/* the following variables are initialized to the same value: */
int d = 42;   /* decimal constant (base10) */
int o = 052;  /* octal constant (base8) */
int x = 0xaf; /* hexadecimal constants (base16) */
int X = 0XAf; /* (letters 'a' through 'f' (case insensitive) represent 10 through 15) */

Ondalık sabitler her zaman “imzalıdır”. Onaltılık sabitler “0x” veya “0X” ile başlar ve sekizlik sabitler yalnızca “0” ile başlar. Son ikisi, değerin imzalı türe uyup uymadığına bağlı olarak ‘imzalı’ veya ‘imzasız’dır.

/* suffixes to describe width and signedness : */
long int i = 0x32; /* no suffix represent int, or long int */
unsigned int ui = 65535u; /* u or U represent unsigned int, or long int */
long int li = 65536l; /* l or L represent long int */

Son ek olmadan sabit, değerine uyan ilk türe sahiptir, yani “INT_MAX” değerinden daha büyük bir ondalık sabit, mümkünse “long” veya aksi takdirde “long long” türündedir.

<limits.h> başlık dosyası tam sayıların sınırlarını aşağıdaki gibi tanımlar. Uygulama tanımlı değerleri, aynı işaretle aşağıda gösterilenlere eşit veya daha büyük (mutlak değer) olacaktır.

makro Tip Değer
“CHAR_BIT” bit alanı (bayt) olmayan en küçük nesne 8
SCHAR_MIN imzalı karakter -127 / -(27 - 1)
“SCHAR_MAX” imzalı karakter +127 / 27 - 1
“UCHAR_MAX” ‘imzasız karakter’ 255 / 28 - 1
CHAR_MIN “karakter” aşağıya bakınız
“CHAR_MAX” “karakter” aşağıya bakınız
SHRT_MIN kısa int -32767 / -(215 - 1)
“SHRT_MAX” kısa int +32767 / 215 - 1
“USHRT_MAX” ‘imzasız kısa int’ 65535 / 216 - 1
INT_MIN ‘int’ -32767 / -(215 - 1)
“INT_MAX” ‘int’ +32767 / 215 - 1
“UINT_MAX” ‘imzasız int’ 65535 / 216 - 1
LONG_MIN uzun int -2147483647 / -(231 - 1)
‘LONG_MAX’ uzun int +2147483647 / 231 - 1
“ULONG_MAX” ‘imzasız uzun int’ 4294967295 / 232 - 1
makro Tip Değer
LLONG_MIN uzun uzun int -9223372036854775807 / -(263 - 1)
“LLONG_MAX” uzun uzun int +9223372036854775807 / 263 - 1
“ULLONG_MAX” ‘imzasız uzun uzun int’ 18446744073709551615 / 264 - 1

Bir ifadede kullanıldığında “char” işaret-uzanma türündeki bir nesnenin değeri, “CHAR_MIN” değeri “SCHAR_MIN” ile aynı olacak ve “CHAR_MAX” değeri ile aynı olacaktır. “SCHAR_MAX” . “char” türündeki bir nesnenin değeri, bir ifadede kullanıldığında işaret-uzanma yapmıyorsa, “CHAR_MIN” değeri 0 olacak ve “CHAR_MAX” değeri “UCHAR_MAX” ile aynı olacaktır.

C99 standardı, sabit genişlikli tamsayılar için tanımları içeren yeni bir “<stdint.h>” başlığı ekledi. Daha ayrıntılı bir açıklama için sabit genişlikli tamsayı örneğine bakın.

Kayan Nokta Sabitleri #

C dilinin üç zorunlu gerçek kayan nokta türü vardır, ‘float’, ‘double’ ve ’long double’.

float f = 0.314f;        /* suffix f or F denotes type float */
double d = 0.314;        /* no suffix denotes double */
long double ld = 0.314l; /* suffix l or L denotes long double */

/* the different parts of a floating point definition are optional */
double x = 1.; /* valid, fractional part is optional */
double y = .1; /* valid, whole-number part is optional */

/* they can also defined in scientific notation */
double sd = 1.2e3; /* decimal fraction 1.2 is scaled by 10^3, that is 1200.0 */

“<float.h>” başlığı, kayan nokta işlemleri için çeşitli limitler tanımlar.

Kayan nokta aritmetiği uygulama tanımlıdır. Ancak, çoğu modern platform (arm, x86, x86_64, MIPS) IEEE 754 kayan nokta işlemlerini kullanır.

C’de ayrıca yukarıdakilerden türetilen üç isteğe bağlı karmaşık kayan nokta türü vardır.

Dize Değişmezleri #

C’deki bir dize değişmezi, değişmez bir sıfır ile sonlandırılan bir karakter dizisidir.

char* str = "hello, world"; /* string literal */

/* string literals can be used to initialize arrays */
char a1[] = "abc"; /* a1 is char[4] holding {'a','b','c','\0'} */
char a2[4] = "abc"; /* same as a1 */
char a3[3] = "abc"; /* a1 is char[3] holding {'a','b','c'}, missing the '\0' */

Dize değişmezleri değiştirilemez (ve aslında .rodata gibi salt okunur belleğe yerleştirilebilir). Değerlerini değiştirmeye çalışmak tanımsız davranışla sonuçlanır.

char* s = "foobar";
s[0] = 'F'; /* undefined behaviour */

/* it's good practice to denote string literals as such, by using `const` */
char const* s1 = "foobar";
s1[0] = 'F'; /* compiler error! */

Derleme zamanında birden çok dize değişmezi birleştirilir, bu, bunun gibi yapı yazabileceğiniz anlamına gelir.

/* only two narrow or two wide string literals may be concatenated */
char* s = "Hello, " "World";
son sürüm 
/* since C99, more than two can be concatenated */
/* concatenation is implementation defined */
char* s1 = "Hello" ", " "World";

/* common usages are concatenations of format strings */
char* fmt = "%" PRId16; /* PRId16 macro since C99 */
son sürüm

Karakter sabitleriyle aynı olan dize değişmezleri, farklı karakter kümelerini destekler.

/* normal string literal, of type char[] */
char* s1 = "abc";

/* wide character string literal, of type wchar_t[] */
wchar_t* s2 = L"abc";

/* UTF-8 string literal, of type char[] */
char* s3 = u8"abc";

/* 16-bit wide string literal, of type char16_t[] */
char16_t* s4 = u"abc";

/* 32-bit wide string literal, of type char32_t[] */
char32_t* s5 = U"abc";
