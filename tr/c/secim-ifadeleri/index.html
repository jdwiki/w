İç içe if()…else VS if()..else Merdiven #

İç içe “if()…else” ifadeleri, “if()…else” merdivenine kıyasla daha fazla yürütme süresi alır (daha yavaştırlar), çünkü iç içe “if()…else” ifadeleri tüm dış koşullu “if()” ifadesi karşılandığında iç koşullu ifadeler, oysa “if()..else” merdiveni, “if()” veya “else if()” koşullu ifadelerinden herhangi biri olduğunda koşul testini durdurur. Doğrudur.

Bir “if()…else” merdiveni:

#include <stdio.h>

int main(int argc, char *argv[])
{
  int a, b, c;
  printf("\nEnter Three numbers = ");
  scanf("%d%d%d", &a, &b, &c);
  if ((a < b) && (a < c))
  {
    printf("\na = %d is the smallest.", a);
  }
  else if ((b < a) && (b < c))
  {
    printf("\nb = %d is the smallest.", b);
  }
  else if ((c < a) && (c < b))
  {
    printf("\nc = %d is the smallest.", c);
  }
  else
  {
    printf("\nImprove your coding logic");
  }
  return 0;
}

Genel durumda, eşdeğer iç içe “if()…else"den daha iyi olarak kabul edilir:

#include <stdio.h>

int main(int argc, char *argv[])
{
  int a, b, c;
  printf("\nEnter Three numbers = ");
  scanf("%d%d%d", &a, &b, &c);
  if (a < b)
  {
    if (a < c)
      {
        printf("\na = %d is the smallest.", a);
      }
    else
      {
        printf("\nc = %d is the smallest.", c);
      }
  }
  else
  {
    if(b < c)
    {
      printf("\nb = %d is the smallest.", b);
    }
    else
    {
      printf("\nc = %d is the smallest.", c);
    }
  }
  return 0;  
}

if () İfadeleri #

Program akışını kontrol etmenin en basit yollarından biri ‘if’ seçim deyimlerini kullanmaktır. Bir kod bloğunun yürütülüp yürütülmeyeceğine bu ifade ile karar verilebilir.

C’deki ‘if’ seçim ifadesinin sözdizimi aşağıdaki gibi olabilir:

if(cond) 
{
  statement(s);  /*to be executed, on condition being true*/
}

Örneğin,

if (a > 1) {
    puts("a is larger than 1");
}

Burada ‘a > 1’, ‘if’ bloğu içindeki ifadeleri yürütmek için ‘doğru’ olarak değerlendirilmesi gereken bir koşuldur. Bu örnekte “a 1’den büyüktür” yalnızca “a > 1” doğruysa yazdırılır.

“if” seçim ifadeleri, blok içinde yalnızca bir ifade varsa “{” ve “}” sarma parantezlerini atlayabilir. Yukarıdaki örnek şu şekilde yeniden yazılabilir:

if (a > 1)
    puts("a is larger than 1");

Ancak blok içinde birden fazla ifade yürütmek için parantezler kullanılmalıdır.

“if” için koşul birden çok ifade içerebilir. “if” yalnızca ifadenin sonucu doğruysa eylemi gerçekleştirir.

Örneğin

if ((a > 1) && (b > 1)) {
    puts("a is larger than 1");
    a++;
}

“printf” ve “a++” işlemlerini yalnızca hem hem “a” hem de “b” “1"den büyükse yürütür.

geçiş () İfadeleri #

switch ifadeleri, belirli bir test değişkeninin değerine göre programınızın birçok farklı şey yapmasını istediğinizde kullanışlıdır.

“switch” ifadesinin örnek bir kullanımı şöyledir:

int a = 1;

switch (a) {
case 1:
    puts("a is 1");
    break;
case 2:
    puts("a is 2");
    break;
default:
    puts("a is neither 1 nor 2");
    break;
}

Bu örnek eşdeğerdir

int a = 1;

if (a == 1) {
    puts("a is 1");
} else if (a == 2) {
    puts("a is 2");
} else {
    puts("a is neither 1 nor 2");
}

‘switch’ deyimi kullanıldığında ‘a’ değeri 1 ise, ‘a is 1’ yazdırılacaktır. ‘a’ değeri 2 ise, ‘a 2’ yazdırılacaktır. Aksi takdirde, ‘a is ne 1 ne de 2’ yazdırılacaktır.

‘case n:’, ‘switch’ deyimine iletilen değer n olduğunda yürütme akışının nereye sıçrayacağını açıklamak için kullanılır. n derleme zamanı sabiti olmalıdır ve aynı n bir ‘switch’ deyiminde en fazla bir kez bulunabilir.

“default:”, değerin “case n:” seçeneklerinden hiçbiriyle eşleşmediğini açıklamak için kullanılır. Beklenmeyen davranışları yakalamak için her switch ifadesine bir “varsayılan” durum eklemek iyi bir uygulamadır.

‘switch’ bloğundan atlamak için bir ‘break;’ ifadesi gereklidir.

Not: Bir “vaka"nın sonundan sonra yanlışlıkla bir “ara” eklemeyi unutursanız, derleyici sizin “geçersiz” ve sonraki tüm vaka ifadelerini, eğer varsa, niyetinizi varsayacaktır. any, yürütülecektir (sonraki durumlardan herhangi birinde bir break ifadesi bulunmadıkça), sonraki durum ifadesinin/deyimlerinin eşleşip eşleşmediğine bakılmaksızın. Bu özel özellik, [Duff’s Device][3] uygulamasını uygulamak için kullanılır. Bu davranış genellikle C dili belirtiminde bir kusur olarak kabul edilir.

Aşağıda ‘break;’ yokluğunun etkilerini gösteren bir örnek verilmiştir:

int a = 1;

switch (a) {
case 1:
case 2:
    puts("a is 1 or 2");
case 3:
    puts("a is 1, 2 or 3");
    break;
default:
    puts("a is neither 1, 2 nor 3");
    break;
}

“a” değeri 1 veya 2 olduğunda, “a 1 veya 2” ve “a 1, 2 veya 3” yazdırılır. “a” 3 olduğunda, yalnızca “a 1, 2 veya 3” yazdırılacaktır. Aksi takdirde, `a ne 1, ne 2 ne de 3’ yazdırılacaktır.

Özellikle ‘anahtar’da aldığınız değerler dizisi bittiğinde ve derleme zamanında biliniyorsa, ‘varsayılan’ durumun gerekli olmadığını unutmayın.

En iyi örnek, bir “enum” üzerinde bir “anahtar” kullanmaktır.

enum msg_type { ACK, PING, ERROR };
void f(enum msg_type t)
{
  switch (t) {
  case ACK:
    // do nothing
    break;
  case PING:
    // do something
    break;
  case ERROR:
    // do something else
    break;
  }
}

Bunu yapmanın birçok avantajı vardır:

  • bir değeri işlemezseniz çoğu derleyici bir uyarı bildirir (“varsayılan” bir durum varsa bu bildirilmez)
  • aynı nedenle, ’enum’a yeni bir değer eklerseniz, yeni değeri kullanmayı unuttuğunuz tüm yerler size bildirilir (‘varsayılan’ durumda, kodunuzu manuel olarak keşfetmeniz gerekir) bu tür vakaları aramak)
  • Okuyucunun “varsayılan:” tarafından neyin gizlendiğini, başka “enum” değerleri olup olmadığını veya “her ihtimale karşı” için bir koruma olup olmadığını bulması gerekmez. Ve başka “enum” değerleri varsa, kodlayıcı kasıtlı olarak onlar için “varsayılan” durumu mu kullandı yoksa değeri eklediğinde ortaya çıkan bir hata mı var?
  • her bir “enum” değerini işlemek, bir joker karakterin arkasına saklanamayacağınız için kodu açıklayıcı hale getirir, her birini açıkça ele almanız gerekir.

Yine de, birinin aşağıdaki gibi kötü kod yazmasını engelleyemezsiniz:

enum msg_type t = (enum msg_type)666; // I'm evil

Böylece, gerçekten ihtiyacınız varsa, onu algılamak için anahtarınızdan önce fazladan bir kontrol ekleyebilirsiniz.

void f(enum msg_type t)
{
   if (!is_msg_type_valid(t)) {
      // Handle this unlikely error
   }

   switch(t) { 
    // Same code than before
   }
}

[3]: https://www.wikiod.com/tr/c/while-do-while-icin-yineleme-ifadeleridonguleri#Döngü Açma ve Duff’ın Cihazı

if () … else deyimleri ve sözdizimi #

“if” yalnızca koşulu “doğru” olarak değerlendirildiğinde bir eylem gerçekleştirirken, “if” / “else”, koşul “doğru” ve koşul “yanlış” olduğunda farklı eylemleri belirtmenize olanak tanır.

Örnek:

if (a > 1)
    puts("a is larger than 1");
else 
    puts("a is not larger than 1");

Tıpkı ‘if’ deyimi gibi, ‘if’ veya ’else’ içindeki blok sadece bir deyimden oluşuyorsa, o zaman kaşlı ayraçlar atlanabilir (ancak bunu yapmak istenmeden kolayca problemlere yol açabileceğinden önerilmez). Ancak ‘if’ veya ’else’ bloğu içinde birden fazla ifade varsa, o blokta kaşlı ayraçlar kullanılmalıdır.

if (a > 1) 
{
    puts("a is larger than 1");
    a--;
}
else 
{
    puts("a is not larger than 1");
    a++;
}

if()…else Merdiven İki veya daha fazla zincirleme if () … else deyimleri #

“if ()… else” ifadesi, “if ()” içindeki koşul karşılanmadığında ortaya çıkan yalnızca bir (varsayılan) davranışı tanımlamaya izin verirken, iki veya daha fazla “if () … else” zincirini zincirler. ifadeler, varsa “varsayılan” olarak hareket eden son “else” dalına gitmeden önce birkaç davranış daha tanımlamaya izin verir.

Örnek:

int a = ... /* initialise to some value. */

if (a >= 1) 
{
    printf("a is greater than or equals 1.\n");
} 
else if (a == 0) //we already know that a is smaller than 1
{
    printf("a equals 0.\n");
}
else /* a is smaller than 1 and not equals 0, hence: */
{ 
    printf("a is negative.\n");
}
