Selam Dünya #

set serveroutput on

DECLARE
   message constant varchar2(32767):= 'Hello, World!';
BEGIN
   dbms_output.put_line(message);
END;
/

SQL*Plus ve SQL Developer istemcilerinde ‘dbms_output’ çıktısını etkinleştirmek için ‘set serveroutput on’ komutu gereklidir. Komut olmadan hiçbir şey görüntülenmez.

’end;’ satırı, anonim PL/SQL bloğunun sonunu bildirir. Kodu SQL komut satırından çalıştırmak için, kodun son satırından sonraki ilk boş satırın başına / yazmanız gerekebilir. Yukarıdaki kod SQL isteminde çalıştırıldığında aşağıdaki sonucu verir:

Hello, World!

PL/SQL procedure successfully completed.

PLSQL’in Tanımı #

PL/SQL (Procedural Language/Structured Query Language), Oracle Corporation’ın SQL ve Oracle ilişkisel veritabanı için prosedürel uzantısıdır. PL/SQL, Oracle Veritabanında (sürüm 7’den beri), TimesTen bellek içi veritabanında (sürüm 11.2.1’den beri) ve IBM DB2’de (sürüm 9.7’den beri) mevcuttur.

PL/SQL’deki temel birim, üç bölümden oluşan bir blok olarak adlandırılır: bildirimsel bölüm, yürütülebilir bölüm ve istisna oluşturma bölümü.

DECLARE
   <declarations section>
BEGIN
   <executable command(s)>
EXCEPTION
   <exception handling>
END;

Beyanlar - Bu bölüm DECLARE anahtar sözcüğüyle başlar. İsteğe bağlı bir bölümdür ve programda kullanılacak tüm değişkenleri, imleçleri, alt programları ve diğer öğeleri tanımlar.

Yürütülebilir Komutlar - Bu bölüm BEGIN ve END anahtar kelimeleri arasında yer alır ve zorunlu bir bölümdür. Programın yürütülebilir PL/SQL deyimlerinden oluşur. En az bir yürütülebilir kod satırına sahip olmalıdır; bu, hiçbir şeyin yürütülmemesi gerektiğini belirtmek için yalnızca bir NULL komutu olabilir.

İstisna İşleme - Bu bölüm, EXCEPTION anahtar sözcüğüyle başlar. Bu bölüm yine isteğe bağlıdır ve programdaki hataları işleyen istisna(lar) içerir.

Her PL/SQL ifadesi noktalı virgül (;) ile biter. PL/SQL blokları, BEGIN ve END kullanılarak diğer PL/SQL bloklarının içine yerleştirilebilir.

Anonim blokta, bloğun sadece çalıştırılabilir kısmı gereklidir, diğer kısımlar gerekli değildir. Aşağıda, hata bildirimi olmadan hiçbir şey yapmayan basit anonim kod örneği verilmiştir.

BEGIN
    NULL;
END;
/

Eksik yürütülebilir komut bir hataya yol açar, çünkü PL/SQL boş blokları desteklemez. Örneğin, aşağıdaki kodun çalıştırılması bir hataya yol açar:

BEGIN
END;
/

Uygulama hata yükseltecek:

END;
*
ERROR at line 2:
ORA-06550: line 2, column 1:
PLS-00103: Encountered the symbol "END" when expecting one of the following:
( begin case declare exit for goto if loop mod null pragma
raise return select update while with <an identifier>
<a double-quoted delimited-identifier> <a bind variable> <<
continue close current delete fetch lock insert open rollback
savepoint set sql execute commit forall merge pipe purge

“END” anahtar kelimesinin altındaki satırda " * " sembolü; bu blokla biten bloğun boş veya kötü yapılandırılmış olduğu anlamına gelir. Her yürütme bloğu, örneğimizde olduğu gibi, hiçbir şey yapmasa bile yapılması gereken talimatlara ihtiyaç duyar.

%TYPE ve %ROWTYPE arasındaki fark. #

%TYPE: Belirtilen tablonun sütunuyla aynı türde bir alan bildirmek için kullanılır. 

DECLARE
        vEmployeeName   Employee.Name%TYPE;
BEGIN
        SELECT Name 
        INTO   vEmployeeName
        FROM   Employee
        WHERE  RowNum = 1;
        
        DBMS_OUTPUT.PUT_LINE(vEmployeeName);
END;
/

%ROWTYPE: Belirtilen tabloda, görünümde veya imleçte (= birden çok sütun) bulunanla aynı türde bir kayıt bildirmek için kullanılır. 

DECLARE
        rEmployee     Employee%ROWTYPE;
BEGIN
        rEmployee.Name := 'Matt';
        rEmployee.Age := 31;
        
        DBMS_OUTPUT.PUT_LINE(rEmployee.Name);
        DBMS_OUTPUT.PUT_LINE(rEmployee.Age);
END;
/

PLSQL hakkında #

PL/SQL, SQL’e Yordamsal Dil uzantıları anlamına gelir. PL/SQL, diğer yazılım ürünlerinde yalnızca “etkinleştiren bir teknoloji” olarak mevcuttur; bağımsız bir dil olarak mevcut değildir. PL/SQL’i Oracle ilişkisel veritabanında, Oracle Sunucusunda ve Oracle Forms gibi istemci tarafı uygulama geliştirme araçlarında kullanabilirsiniz. PL/SQL’i kullanmanın bazı yolları şunlardır:

  1. Saklı yordamlar oluşturmak için. .
  2. Veritabanı tetikleyicileri oluşturmak için.
  3. Oracle Forms uygulamanızda istemci tarafı mantığını uygulamak için.
  4. World Wide Web ana sayfasını bir Oracle veritabanına bağlamak için.

Bir görünüm oluşturun veya değiştirin #

Bu örnekte bir görünüm oluşturacağız.
Görünüm, çoğunlukla birden çok tablodan veri almanın basit bir yolu olarak kullanılır.

Örnek 1:
Tek bir tabloda bir seçim ile görüntüleyin.

CREATE OR REPLACE VIEW LessonView AS
       SELECT     L.*
       FROM       Lesson L;

Örnek 2:
Birden çok tabloda bir seçimle görüntüleyin.

CREATE OR REPLACE VIEW ClassRoomLessonView AS
       SELECT     C.Id, 
                  C.Name, 
                  L.Subject, 
                  L.Teacher 
       FROM       ClassRoom C, 
                  Lesson L 
       WHERE      C.Id = L.ClassRoomId;

Bu görünümleri bir sorguda çağırmak için bir select ifadesi kullanabilirsiniz. 

SELECT * FROM LessonView;
SELECT * FROM ClassRoomLessonView;

Bir tablo oluşturun #

Aşağıda 3 sütunlu bir tablo oluşturacağız.
‘Id’ sütunu doldurulmalıdır, bu yüzden onu ‘BOŞ DEĞİL’ olarak tanımlarız.
“Sözleşme” sütununa, izin verilen tek değerin “Y” veya “N” olması için bir kontrol de ekleriz. Bir ekleme tamamlandıysa ve ekleme sırasında bu sütun belirtilmezse, varsayılan olarak bir ‘N’ eklenir.

CREATE TABLE Employee (
        Id            NUMBER NOT NULL,
        Name          VARCHAR2(60),
        Contract      CHAR DEFAULT 'N' NOT NULL,
        ---
        CONSTRAINT p_Id PRIMARY KEY(Id),
        CONSTRAINT c_Contract CHECK (Contract IN('Y','N'))
);
