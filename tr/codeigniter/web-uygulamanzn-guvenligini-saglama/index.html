Web uygulamanızın güvenliğini sağlama

CodeIgniter’ın bir geliştirme Çerçevesi olduğunu unutmayın. Uygulamanızı güvenli hale getirmek için çaba göstermez. Size sadece bunu kendiniz yapmanız için gerekli araçları sağlar. CI’nin Güvenlik sayfasına bakarsanız, geliştiricinin Uygulama Güvenliğini anlamasını ve uygulamalarına yerleştirmesini bekledikleri oldukça açıktır.

WebApp güvenliği sizin için nispeten yeniyse, OWASP ile başlardım. Daha açık şeyler yaptığına inandığım Zend veya Cake gibi diğer çerçevelere bakmak avantajlı olabilir.

Sözdizimi #

  • $tazeveri = $bu->güvenlik->xss_clean($user_input_data);

Parametreler #

kullanıcı girişi dizisi boş
xss_filter($kullanıcı girişi dizisi) içine kullanıcı girişi dizisini ekleyin Boş

SQL Enjeksiyon Önleme #

SQL enjeksiyonu, veritabanı sorgusuna yapılan bir saldırıdır. PHP’de bunu önlemek için diğer tekniklerle birlikte mysql_real_escape_string() işlevini kullanıyoruz, ancak CodeIgniter bunu önlemek için dahili işlevler ve kitaplıklar sağlar.

CodeIgniter’da SQL Injection’ı aşağıdaki üç yolla önleyebiliriz -

  • Kaçış Sorguları
  • Sorgu Teklifi
  • Aktif Kayıt Sınıfı

Kaçış Sorguları

<?php
   $username = $this->input->post('username');
   $query = 'SELECT * FROM subscribers_tbl WHERE user_name = '.
   $this->db->escape($email);
   $this->db->query($query);
?>

$this->db->escape() işlevi, verilerin etrafına otomatik olarak tek tırnak işaretleri ekler ve yalnızca dize verilerinden kaçabilmesi için veri türünü belirler.

Sorgu Teklifi

<?php
   $sql = "SELECT * FROM some_table WHERE id = ? AND status = ? AND author = ?";
   $this->db->query($sql, array(3, 'live', 'Rick'));
?>

Yukarıdaki örnekte, soru işareti(?) ‘query()’ fonksiyonunun ikinci parametresindeki dizi ile değiştirilecektir. Sorguyu bu şekilde oluşturmanın ana avantajı, güvenli sorgular üreten değerlerin otomatik olarak kaçmasıdır. CodeIgniter motoru bunu sizin için otomatik olarak yapar, böylece hatırlamanız gerekmez.

Aktif Kayıt Sınıfı

<?php
   $this->db->get_where('subscribers_tbl',array('status'=> active','email' => 'info@arjun.net.in'));
?>

Etkin kayıtları kullanarak, her bir veritabanı bağdaştırıcısı tarafından sorgu sözdizimi oluşturulur. Değerler otomatik olarak kaçtığı için daha güvenli sorgulara da izin verir.

XSS Önleme #

XSS, siteler arası komut dosyası oluşturma anlamına gelir. CodeIgniter, XSS filtreleme güvenliği ile birlikte gelir. Bu filtre, herhangi bir kötü amaçlı JavaScript kodunu veya çerezi ele geçirmeye ve kötü niyetli faaliyetler gerçekleştirmeye çalışan diğer herhangi bir kodu önleyecektir. XSS filtresi aracılığıyla verileri filtrelemek için aşağıda gösterildiği gibi xss_clean() yöntemini kullanın.

$data = $this->security->xss_clean($data);

Bu işlevi yalnızca veri gönderirken kullanmalısınız. İsteğe bağlı ikinci Boole parametresi, görüntü dosyasını XSS ​​saldırısı için kontrol etmek için de kullanılabilir. Bu, dosya yükleme tesisi için kullanışlıdır. Değeri true ise, görüntünün güvenli olduğu anlamına gelir, aksi halde değil.

PHP Hatalarını Gizleme #

Üretim ortamında, genellikle kullanıcılara herhangi bir hata mesajı göstermek istemiyoruz. Hata ayıklama amacıyla geliştirme ortamında etkinleştirilmesi iyidir. Bu hata mesajları, güvenlik nedeniyle site kullanıcılarına göstermememiz gereken bazı bilgiler içerebilir.

Hatalarla ilgili üç CodeIgniter dosyası vardır. PHP Hata Raporlama Seviyesi

Farklı ortamlar, farklı seviyelerde hata raporlaması gerektirir. Varsayılan olarak, geliştirme hataları gösterecek, ancak test ve canlı yayın bunları gizleyecektir. CodeIgniter’ın kök dizininde bu amaçla kullanılan index.php adında bir dosya bulunmaktadır. error_reporting() işlevine argüman olarak sıfır iletirsek, bu tüm hataları gizleyecektir.

CSRF Önleme #

CSRF, siteler arası istek sahteciliği anlamına gelir. Aşağıda gösterildiği gibi application/config/config.php dosyasında bir seçeneği etkinleştirerek bu saldırıyı önleyebilirsiniz.

$config['csrf_protection'] = TRUE;

form_open() işlevini kullanarak bir form oluşturduğunuzda, gizli bir alana otomatik olarak bir CSRF belirteci ekleyecektir. “get_csrf_token_name()” ve “get_csrf_hash()” işlevini kullanarak CSRF belirtecini manuel olarak da ekleyebilirsiniz. Adlarından da anlaşılacağı gibi, ‘get_csrf_token_name()’ işlevi CSRF belirtecinin adını döndürürken, ‘get_csrf_hash()’ hash değerini döndürür.

CSRF belirteci, gönderim için her seferinde yeniden oluşturulabilir veya CSRF tanımlama bilgisinin ömrü boyunca aynı tutabilirsiniz. ‘csrf_regenerate’ yapılandırma seçeneğinin ayarlanması, aşağıda gösterildiği gibi belirtecin yeniden oluşturulmasını zorlayacaktır.

$config['csrf_regenerate'] = TRUE;

URL’leri, aşağıda gösterildiği gibi “csrf_exclude_uris” anahtarını kullanarak yapılandırma dizisinde eşleşmeler ayarlayarak CSRF korumasından beyaz listeye ekleyebilirsiniz. Normal ifadeler de kullanabilirsiniz.

$config['csrf_exclude_uris'] = array('api/person/add');

// XSS Filtering
$data = array(
             'name'=> '<script>Abuse Data</script>'
        );
$data = $this->security->xss_clean($data); // Clean Data

// Escaping Queries
<?php $username = $this->input->post('username'); $query = 'SELECT * FROM subscribers_tbl WHERE user_name = '. $this->db->escape($email); $this->db->query($query); ?>

Kullanıcı Girişinde XSS Önleme #

Herhangi bir kullanıcı girdisine güvenmeyin. kullanıcı <script> etiketi veya herhangi bir javascript alert(); gibi her şeyi girer, bu yüzden tüm verilerin tarayıcımızda çalışmamasını engellememiz gerekir. bu nedenle, güvenli verilerimizi bilgisayar korsanlarının elinde tutmak için xss önleme yöntemini kullanmalıyız ve ayrıca geliştiricinin kullanıcının giriş doğrulaması ve hatayı programlı olarak çözme sorumluluğu vardır.

bu nedenle, bunun CodeIgniter’da bir xss önleme örneği olup olmadığını kontrol edin.

$data = array(
            'name' => "<script>alert('abc')</script>",
            'email' => "useremail@gmail.com"
        );
var_dump($data);
// Print array without xss cleaning/xss filtering

array(2) { ["name"]=> string(29) "" ["email"]=> string(19) "useremail@gmail.com" } // Result with alert

// now print data after xss filtering

$data = $this->security->xss_clean($data);
var_dump($data);

//Print array without xss cleaning/xss filtering
array(2) { ["name"]=> string(38) "[removed]alert('abc')[removed]" ["email"]=> string(19) "useremail@gmail.com" } // Result Without alert

bu nedenle, xss_filtering eklendikten sonra, kullanıcı tarafından girilen herhangi bir kötüye kullanım kodunu çalıştırma sorunumuz yok. ve CodeIgniter bu kötüye kullanım etiketini [removed] anahtar kelimesiyle değiştirir.
