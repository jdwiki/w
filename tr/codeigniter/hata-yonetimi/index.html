Hata yönetimi

Bu sayfada

CodeIgniter, aşağıda açıklanan işlevleri kullanarak uygulamalarınıza hata raporlaması oluşturmanıza olanak tanır. Ayrıca, hata ve hata ayıklama mesajlarının metin dosyaları olarak kaydedilmesine izin veren bir hata günlüğü sınıfına sahiptir.

log_message() #

Bu işlev, günlük dosyalarınıza mesaj yazmanıza olanak tanır. İlk parametrede, mesajın ne tür olduğunu (hata ayıklama, hata, bilgi) belirten üç “düzeyden” birini, ikinci parametrede mesajın kendisi ile sağlamanız gerekir.

Örnek:

if ($some_var == "") {
    log_message('error', 'Some variable did not contain a value.'); 
} 
else {
    log_message('debug', 'Some variable was correctly set'); 
}

log_message('info', 'The purpose of some variable is to provide some value.');

Sözdizimi #

log_message($level, $message);

Parametreler:

  • $level (string) – Günlük seviyesi: ‘hata’, ‘hata ayıklama’ veya ‘bilgi’
  • $message (string) – Günlüğe kaydedilecek mesaj

İade türü: geçersiz

Üç mesaj türü vardır: #

  • Hata mesajları. Bunlar PHP hataları veya kullanıcı hataları gibi gerçek hatalardır.
  • Hata Ayıklama Mesajları. Bunlar hata ayıklamaya yardımcı olan mesajlardır. Örneğin, bir sınıf başlatıldıysa, bunu hata ayıklama bilgisi olarak kaydedebilirsiniz.
  • Bilgilendirme Mesajları. Bunlar en düşük öncelikli mesajlardır, sadece bazı işlemlerle ilgili bilgi verirler. CodeIgniter yerel olarak herhangi bir bilgi mesajı oluşturmaz, ancak uygulamanızda yapmak isteyebilirsiniz.

Not: Günlük dosyasının gerçekten yazılabilmesi için “günlükler” klasör yazılabilir olmalıdır. Ek olarak, “eşiği” ayarlamalısınız. application/config/config.php içinde oturum açın. Örneğin, diğer iki türün değil, yalnızca hata mesajlarının günlüğe kaydedilmesini isteyebilirsiniz. Sıfıra ayarlarsanız, günlük kaydı devre dışı bırakılır.

show_error() #

Bu işlev, aşağıdaki hata şablonunu kullanarak kendisine sağlanan hata mesajını görüntüler:

Yol - application/errors/error_general.php

İsteğe bağlı $status_code parametresi, hatayla birlikte hangi HTTP durum kodunun gönderilmesi gerektiğini belirler.

Sözdizimi #

show_error($message, $status_code, $heading = 'An Error Was Encountered')

Parametreler:

  • $mesaj (karışık) – Hata mesajı
  • $status_code (int) – HTTP Yanıtı durum kodu
  • $heading (string) – Hata sayfası başlığı

İade türü: geçersiz

Kaynak #

  1. codeigniter.com’da show_error

show_404() #

Bu işlev, aşağıdaki hata şablonunu kullanarak kendisine sağlanan 404 hata mesajını görüntüler:

Yol - application/errors/error_404.php

İşlev, kendisine iletilen dizenin, bulunmayan sayfanın dosya yolu olmasını bekler. Denetleyiciler bulunamazsa CodeIgniter’ın otomatik olarak 404 mesajı gösterdiğini unutmayın.

CodeIgniter, tüm show_404() çağrılarını otomatik olarak günlüğe kaydeder. İsteğe bağlı ikinci parametrenin YANLIŞ olarak ayarlanması, günlüğe kaydetmeyi atlayacaktır.

Sözdizimi #

show_404($page = '', $log_error = TRUE)

Parametreler:

  • $sayfa (dize) – URI dizesi
  • $log_error (bool) – Hatanın günlüğe kaydedilip kaydedilmeyeceği

İade türü: geçersiz

Kaynak #

  1. codeigniter.com’da show_404
