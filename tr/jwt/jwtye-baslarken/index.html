İmzasız JWT #

İmzasız bir JWT, “alg: none” başlık değerine ve boş bir JWS (imza) bileşenine sahiptir:

eyJhbGciOiJub25lIn0
.eyJpc3MiOiJqb2UiLA0KICJleHAiOjEzMDA4MTkzODAsDQogImh0dHA6Ly9leGFtcGxlLmNvbS9pc19yb290Ijp0cnVlfQ
.

Sondaki nokta, imzanın boş olduğunu gösterir.

Başlık #

{
  "alg": "none"
}

Yük #

{
  "iss": "joe",
  "exp": 1300819380,
  "http://example.com/is_root": true
}

İmzalı JWT (JWS) #

İmzalı bir JWT, üçüncü bileşen olarak bir Base64 Url Güvenli kodlu imza içerir. İmzayı oluşturmak için kullanılan algoritma başlıkta belirtilmiştir.

eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9
.eyJzdWIiOiJKb2huIERvZSIsImFkbWluIjp0cnVlLCJpYXQiOjE0NzAzNTM5OTQsImV4cCI6MTQ3MDM1NzYyNywianRpIjoiNmU0MDRiYTgtZjg4NS00ZDVmLWJmYTItZTNmNWEwODM4MGE0In0
.7CfBdVP4uKsb0cogYepCvMLm8rcpjBYW1XZzA-a5e44

Başlık #

{
  "typ": "JWT",
  "alg": "HS256"
}

Bu JWT, HMAC-SHA256 algoritması ile imzalanmıştır, dolayısıyla ‘alg: HS256’.

Yük #

{
  "sub": "John Doe",
  "admin": true,
  "iat": 1470353994,
  "exp": 1470357627,
  "jti": "6e404ba8-f885-4d5f-bfa2-e3f5a08380a4"
}

Bu JWT, UTF-8 gizli “notsosecret” ile doğrulanabilir.

JWS veya JWE’niz olup olmadığını nasıl anlarsınız? #

JSON Web Şifreleme spesifikasyonunun (RFC 7516) 9. Bölümünden:

Bir JWS için JOSE Başlığı, “alg” (algoritma) Başlık Parametresi değeri incelenerek JWE için JOSE Başlığından ayırt edilebilir. Değer bir dijital imzayı veya MAC algoritmasını temsil ediyorsa veya “yok” değeriyse, bir JWS içindir; bir Anahtar Şifreleme, Anahtar Sarma, Doğrudan Anahtar Anlaşması, Anahtar Sarma ile Anahtar Anlaşması veya Doğrudan Şifreleme algoritmasını temsil ediyorsa, bir JWE içindir. (İncelenecek “alg” değerinin çıkarılması, JWS Kompakt Serileştirme veya JWE Kompakt Serileştirme kullanılırken basittir ve JWS JSON Serileştirme veya JWE JSON Serileştirme kullanılırken daha zor olabilir.)

Ve

Bir JWS için JOSE Başlığı, bir “enc” (şifreleme algoritması) üyesinin bulunup bulunmadığı belirlenerek JWE için JOSE Başlığından da ayırt edilebilir. “enc” üyesi varsa, bu bir JWE’dir; aksi takdirde, bu bir JWS’dir.

JWS (imzalı)## #

{
  "alg": "HS256"
}

OYNAT (şifreli) #

{
  "alg":"RSA1_5",
  "enc":"A256GCM",
  "iv":"__79_Pv6-fg",
  "x5t":"7noOPq-hJ1_hCnvWh6IeYI2w9Q0"
}

JSON Web Şifreleme (OYNAT) #

JSON Web Encryption (JWE), JavaScript Object Notation (JSON) tabanlı veri yapılarını kullanan şifrelenmiş içeriği temsil eder. Talep verilerinizi şifrelemenin bir yolunu tanımlar, böylece yalnızca hedeflenen alıcı bir belirteçte bulunan bilgileri okuyabilir.

JWE JSON Serileştirmede, bir JWE, bu sekiz üyenin bir kısmını veya tamamını içeren bir JSON nesnesi olarak temsil edilir:

  "protected", with the value BASE64URL(UTF8(JWE Protected Header))
  "unprotected", with the value JWE Shared Unprotected Header
  "header", with the value JWE Per-Recipient Unprotected Header
  "encrypted_key", with the value BASE64URL(JWE Encrypted Key)
  "iv", with the value BASE64URL(JWE Initialization Vector)
  "ciphertext", with the value BASE64URL(JWE Ciphertext)
  "tag", with the value BASE64URL(JWE Authentication Tag)
  "aad", with the value BASE64URL(JWE AAD)

Altı base64url kodlu sonuç dizesi ve korumasız iki JSON nesne değerleri, bir JSON nesnesi içinde üyeler olarak temsil edilir.

Örnek OYUN

Aşağıdaki örnek JWE Header şunu beyan eder:

  • İçerik Şifreleme Anahtarı, JWE Şifreli Anahtarı üretmek için RSA-PKCS1_1.5 algoritması kullanılarak alıcıya şifrelenir

  • Düz Metin, JWE Şifreli Metin oluşturmak için AES-256-GCM algoritması kullanılarak şifrelenir

  • __79_Pv6-fg kodlamalı base64url ile belirtilen 64-bit Başlatma Vektörü kullanıldı

  • JWE’yi şifrelemek için kullanılan anahtara karşılık gelen X.509 sertifikasının parmak izi, 7noOPq-hJ1_hCnvWh6IeYI2w9Q0 kodlaması olan base64url’ye sahiptir.

    { “alg”:“RSA1_5”, “enc”:“A256GCM”, “iv”:"__79_Pv6-fg", “x5t”:“7noOPq-hJ1_hCnvWh6IeYI2w9Q0” }

JWE Başlığının UTF-8 temsilinin baytlarını kodlayan Base64url, bu Kodlanmış JWE Başlığı değerini verir (yalnızca görüntüleme amacıyla satır sonları ile):

eyJhbGciOiJSU0ExXzUiLA0KICJlbmMiOiJBMjU2R0NNIiwNCiAiaXYiOiJfXzc5
X1B2Ni1mZyIsDQogIng1dCI6Ijdub09QcS1oSjFfaENudldoNkllWUkydzlRMCJ

Daha fazla bilgi için JSON Web Şifreleme spesifikasyonunu (RFC 7516) okuyun

JWT’de ne saklanmalı #

JWT RFC, üç iddia sınıfı belirler:

  • “sub”, “iss”, “exp” veya “nbf” gibi tescilli hak talepleri

  • kamusal hak talepleri, “e-posta”, “adres” veya “telefon_numarası” gibi benzersiz olması gereken değerleri içeren genel adlara veya [IANA tarafından kayıtlı2] adlara sahip. Tam listeye bakın

  • Kendi bağlamınızda kullanmak için özel hak talepleri ve değerler çakışabilir

Bu taleplerin hiçbiri zorunlu değildir

Bir JWT bağımsızdır ve kimlik doğrulamayı gerçekleştirmek için gerekli verileri sağlayan sunucu oturumunu kullanmaktan kaçınmalıdır (sunucu depolama ve veritabanı erişimine gerek yoktur). Bu nedenle, “rol” veya “izinler” bilgisi JWT’nin özel taleplerine dahil edilebilir.

Kayıtlı Talepler #

Aşağıdaki Talep Adları, Bölüm 10.1. tarafından oluşturulan IANA “JSON Web Token Claims” kayıt defterinde kayıtlıdır.

  • ‘iss’ (yayınlayan): JWT’yi yayınlayan müdürü tanımlar.
  • ‘sub’ (konu): JWT’nin konusu olan müdürü tanımlar. Eşsiz olmalı
  • “aud” (izleyici): JWT’nin amaçlandığı alıcıları tanımlar (dizi/uri)
  • exp (sona erme süresi): artık bu jetonu kabul etmemeniz gereken sona erme süresini (UTC Unix) tanımlar. Yayınlanan zamandan sonra olmalıdır.
  • nbf(önce değil): JWT’nin kabul edilmemesi gereken UTC Unix zamanını tanımlar
  • “iat” (yayınlanma tarihi): JWT’nin düzenlendiği UTC Unix saatini tanımlar
  • jti (JWT Kimliği): JWT için benzersiz bir tanımlayıcı sağlar.

Örnek #

{
    "iss": "stackoverflow",
    "sub": "joe",
    "aud": ["all"],
    "iat": 1300819370,
    "exp": 1300819380,
    "jti": "3F2504E0-4F89-11D3-9A0C-0305E82C3301"
    "context": {
        "user": {
            "key": "joe",
            "displayName": "Joe Smith"
        },
        "roles":["admin","finaluser"]
    }
}
