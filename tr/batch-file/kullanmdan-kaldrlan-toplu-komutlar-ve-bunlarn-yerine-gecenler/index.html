BITSADMIN #

BITSADMIN belgeleri aktarmak, web sitesi indirmek, web sitelerinden dosya indirmek vb. için kullanıldı… Bu komut, kullanımdan kaldırılmış bir komuttur ve sonraki Windows güncellemelerinde kaldırılabilir. Bu sorunu önlemek için yeni Powershell BIT cmdlet kullanın.

İşte “BITSADMIN” kullanan bir örnek kod.

@echo off
Bitsadmin /create /download Stackoverflow.com
rem download the website StackOverflow.com

HATA AYIKLAMA #

DEBUG komutu, toplu iş dosyasından ikili dosyalar/yürütülebilir dosyalar oluşturmak için kullanıldı. Komut, Windows’un 32bit sürümlerinde hala mevcuttur, ancak yalnızca 16bit talimatlarla ikili dosyalar oluşturabilir ve 64bit makineler için kullanılamaz hale getirir. Şimdi CERTUTIL, HEX veya BASE64 biçimlerinden ikili/ortam dosyalarının kodunun çözülmesine/kodlanmasına izin veren bu amaç için kullanılmaktadır. Simge dosyası oluşturan bir dosya ile örnek:

@echo off

del /q /f pointer.jpg >nul 2>nul
certutil -decode "%~f0" pointer.jpg
hh.exe pointer.jpg
exit /b %errorlevel%

EKLE #

APPEND msdos’ta kaynak/medya dosyalarının aynı dizindeymiş gibi kullanılmasına izin veren komuttu. Komut, Windows’un 32bit sürümlerinde hala kullanılabilir ancak çalışmıyor gibi görünüyor. Bazı kaynaklarda (microsoft’lar dahil), komutun DPATH ile değiştirildiği belirtilir, ancak bu tamamen doğru değildir. DPATH yardım mesajı APPEND komutuna işaret etmesine rağmen, sözdizimi PATH ile aynıdır. DPATH’de listelenen dizinler giriş yönlendirmesi ile veya type komutu olarak kullanılabilir:

@echo off
dpath %windir%

set /p var=<win.ini
echo using dpath with input redirection:
echo %var%
echo.
echo using dpath with type command:
type win.ini
