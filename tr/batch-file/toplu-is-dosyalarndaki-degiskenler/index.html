Değişken Değiştirme #

Diğer programlama dillerinden farklı olarak, bir toplu iş dosyasında bir değişken, toplu iş komut dosyası çalıştırılmadan önceki gerçek değeriyle değiştirilir. Başka bir deyişle, komut dosyası daha sonra run olduğunda değil, komut dosyası komut işlemcisi tarafından belleğe _okunduğunda yapılır.

Bu, değişkenlerin komut dosyası içinde komutlar olarak ve komut dosyasındaki diğer değişken adlarının bir parçası olarak vb. kullanılmasını sağlar. Bu bağlamdaki “kod”, yuvarlak parantezler içinde bir satır veya kod bloğudur: () .

Ancak bu davranış, bir blok içindeki bir değişkenin değerini değiştiremeyeceğiniz anlamına gelir!

SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    ECHO %VAR%
    SET VAR=Goodbye
)

yazdıracak

Hello
Hello

çünkü (gördüğünüz gibi, komut penceresinde komut dosyasını çalıştırırken) şu şekilde değerlendirilir:

SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    echo Hello
    SET VAR=Goodbye
)

Yukarıdaki örnekte, komut dosyası belleğe okunduğunda “ECHO” komutu “Merhaba” olarak değerlendirilir, bu nedenle komut dosyası sonsuza kadar “Merhaba” olarak yankılanır, ancak komut dosyası üzerinden birçok geçiş yapılır.

Daha “geleneksel” değişken davranışını (komut dosyası çalışırken genişletilen değişkenin) elde etmenin yolu, “gecikmeli genişletmeyi” etkinleştirmektir. Bu, komut dosyasına döngü komutundan önce (genellikle bir toplu komut dosyasında bir FOR döngüsü) eklemeyi ve değişken adında yüzde işareti (%) yerine ünlem işareti (!) kullanmayı içerir:

setlocal enabledelayedexpansion 
SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    echo !VAR!
    SET VAR=Goodbye
)
endlocal

yazdıracak

Hello
Goodbye

%%a in (1,1,2) sözdizimi döngünün 2 kez çalışmasına neden olur: ilk durumda, değişken ilk ‘Merhaba’ değerini taşır, ancak ikinci geçişte - yürütüldüğünde 1. geçişte son eylem olarak ikinci SET talimatı - bu, revize edilmiş ‘Hoşçakal’ değerine değişti.

Gelişmiş değişken değiştirme

Şimdi, gelişmiş bir teknik. ‘CALL’ komutunun kullanılması, toplu komut işlemcisinin betiğin aynı satırında bulunan bir değişkeni genişletmesine olanak tanır. Bu, tekrarlanan CALL ve değiştirici kullanımıyla çok düzeyli genişleme sağlayabilir.

Bu, örneğin bir FOR döngüsünde kullanışlıdır. Aşağıdaki örnekte olduğu gibi, numaralandırılmış bir değişken listemiz var:

"c:\MyFiles\test1.txt" "c:\MyFiles\test2.txt" "c:\MyFiles\test3.txt"

Bunu aşağıdaki FOR döngüsünü kullanarak başarabiliriz:

setlocal enabledelayedexpansion
for %%x in (%*) do (
    set /a "i+=1"
    call set path!i!=%%~!i!
    call echo %%path!i!%%
)
endlocal

Çıktı:

c:\MyFiles\test1.txt
c:\MyFiles\test2.txt
c:\MyFiles\test3.txt

‘!i!’ değişkeninin ilk önce başlangıç ​​değeri olan 1’e genişletildiğini, ardından elde edilen %1 değişkeninin gerçek değeri olan ‘c:\MyFiles\test1.txt’ değerine genişletildiğini unutmayın. Bu, “i” değişkeninin çift açılımıdır. Sonraki satırda, ‘i’, ‘%%’ değişken öneki ile birlikte ‘CALL ECHO’ komutu kullanılarak tekrar çift genişletilir, ardından ekrana yazdırılır (yani ekranda görüntülenir).

Döngüden art arda her geçişte, ilk sayı 1 artırılır (‘i+=1’ kodundan dolayı). Böylece döngü boyunca 2. geçişte ‘2’ye ve 3. geçişte ‘3’e yükselir. Böylece ekranda yankılanan dize her geçişte değişir.

Beyanname #

Basit bir değişken oluşturmak ve onu bir değere veya dizgeye atamak için ‘SET’ komutunu kullanın:

SET var=10

Burada kod, “10” değerinde yeni bir “var” değişkeni bildirir. Varsayılan olarak tüm değişkenler dahili olarak dizeler olarak depolanır; bu, “10” değerinin “foo1234” veya “Merhaba, Dünya!” değerinden farklı olmadığı anlamına gelir.

Tırnak işaretleri hakkında notlar #

Kullanılan tırnak işaretleri, değişkenin değerine dahil edilecektir:

SET var="new value"             <-- %var% == '"new value"'

Değişkenlerdeki boşluklar #

Toplu iş dili, boşlukları değişken adlarının kabul edilebilir parçaları olarak kabul eder. Örneğin, “set var = 10”, 10 değerini içeren var adlı bir değişkenle sonuçlanır (var öğesinin sağındaki ve satırın solundaki fazladan boşluğa dikkat edin). 10).

Boşlukları ortadan kaldırmak için tırnak işaretleri kullanma #

Boşlukları önlemek için atamanın tamamında tırnak işaretleri kullanın; değişken adı ve değeri. Bu aynı zamanda satırın sonunda yanlışlıkla boşluk bırakılmasını da önler (␣ karakteri boşluk anlamına gelir):

SET␣var=my␣new␣value␣           <-- '%var%' == 'my new value '
SET␣"var=my␣new␣value"␣         <-- '%var%' == 'my new value'

Ayrıca, birden çok ifadeyi “&” veya “|” ile birleştirirken tırnak işaretleri kullanın - alternatif olarak, sembolü doğrudan değişkenin değerinin sonundan sonra koyun:

SET var=val & goto :next        <-- '%var%' == 'val '
SET "var=val" & goto :next      <-- '%var%' == 'val'
SET var=val& goto :next         <-- '%var%' == 'val'

Birden çok değişken bildir #

Partinin başında birden fazla değişken tanımlandığında, bir değiştirme dizesi kullanılarak kısa bir tanım formu kullanılabilir.

@echo off
set "vars=_A=good,_B=,_E=bad,_F=,_G=ugly,_C=,_H=,_I=,_J=,_K=,_L=,_D=6
set "%vars:,=" & set "%"

for /f %%l in ('set _') do echo %%l
exit /b

_A=good
_D=6
_E=bad
_G=ugly

Yukarıdaki örnekte, ekrana yazdırıldığında değişkenlerin doğal olarak alfabetik olarak sıralandığını unutmayın.

Kullanım #

echo %var%

Bu kod, ‘var’ değerini yansıtacaktır.

“setLocal EnableDelayedExpansion” kullanılırsa, aşağıdaki “var” değerini yansıtır (standart %var% ifadesi bu bağlamda çalışmaz).

echo !var!

Toplu iş dosyalarında değişkenler, komutların bölümleri veya diğer değişkenlerin bölümleri de dahil olmak üzere herhangi bir bağlamda kullanılabilir. Bir değişkeni tanımlamadan önce çağıramazsınız.

Değişkenleri komut olarak kullanma:

set var=echo
%var% This will be echoed

Değişkenleri diğer değişkenlerde kullanma:

set var=part1
set %var%part2=Hello
echo %part1part2%

Bir Değişkeni Dizi Olarak Kullanma #

SET ifadesindeki boşlukları kullanarak bir diziye benzer şekilde davranabilen (gerçek bir dizi nesnesi olmasalar da) bir dizi değişken oluşturmak mümkündür:

@echo off
SET var=A "foo bar" 123
for %%a in (%var%) do (
    echo %%a
)
echo Get the variable directly: %var%

Sonuç:

A
"foo bar"
123
Get the variable directly: A "foo bar" 123

Belirli bilgileri alabilmeniz için değişkeninizi dizinler kullanarak bildirmek de mümkündür. Bu, bir dizi yanılsaması ile birden çok değişken yaratacaktır:

@echo off
setlocal enabledelayedexpansion
SET var[0]=A
SET var[1]=foo bar
SET var[2]=123
for %%a in (0,1,2) do (
    echo !var[%%a]!
)
echo Get one of the variables directly: %var[1]%

Sonuç:

A
foo bar
123
Get one of the variables directly: foo bar

Yukarıdaki örnekte, var kendi içinde bulunmadığından, istenen dizinin ne olduğunu belirtmeden var‘a başvuruda bulunamayacağınızı unutmayın. Bu örnek ayrıca !var[%%a]!‘daki ünlem işaretleri ile birlikte setlocal enabledelayedexpansion‘u kullanır. Bununla ilgili daha fazla bilgiyi Değişken Değiştirme Kapsamı Belgelerinde görüntüleyebilirsiniz.

Bir girdiden değişkenleri ayarlama #

/p anahtarını SET komutuyla kullanarak bir Girişten değişkenler tanımlayabilirsiniz.

Bu giriş bir kullanıcı Girişi (klavye) olabilir:

echo Enter your name : 
set /p name=
echo Your name is %name%

Hangisi bu şekilde basitleştirilebilir:

set /p name=Enter your name :
echo Your name is %name%

Veya girdiyi bir dosyadan alabilirsiniz:

set /p name=< file.txt

bu durumda ilk satırın değerini file.txt dosyasından alırsınız.

Bir dosyadaki çeşitli satırın değerini alma:

(
   set /p line1=
   set /p line2=
   set /p line3=

) < file.txt

Değişkenler Üzerindeki İşlemler #

set var=10
set /a var=%var%+10
echo %var%

Var’ın son değeri 20’dir.

İkinci satır, örneğin bir IF koşulunda veya bir FOR döngüsünde kullanılan bir komut bloğu içinde çalışmıyor, çünkü standart ortam değişkeni genişletmesi yerine gecikmeli genişletme gerekli olacaktır.

İşte bir komut bloğunda da çalışmanın daha iyi bir yolu:

set var=10
set /A var+=10
echo %var%

Komut istemi ortamı, imzalı 32 bit tamsayı değerlerini destekler:

  • + ve += ekleme
  • çıkarma - ve -=
  • çarpma * ve *=
  • / ve /= bölümü
  • modül bölümü % ve %=
  • bit düzeyinde VE &
  • bit düzeyinde VEYA |
  • bit düzeyinde ~ DEĞİL
  • bit düzeyinde XOR ^
  • bit düzeyinde sola kaydırma `«’
  • bit düzeyinde sağa kaydırma >>
  • mantıksal DEĞİL !
  • tekli eksi -
  • ( ve ) ile gruplama

Windows komut yorumlayıcısı, aritmetik ifadelerde 64 bit tam sayı değerlerini veya kayan nokta değerlerini desteklemez.

Not: % operatörü, operatör olarak yorumlanabilmesi için bir toplu iş dosyasına %% olarak yazılmalıdır.

Bir komut istemi penceresinde set /A Value=8 % 3 komut satırı çalıştırıldığında, 2 değerini Değer ortam değişkenine atar ve ek olarak 2 çıktısını verir.

Bir toplu iş dosyasında, ‘2’ değerini ‘Value’ ortam değişkenine atamak için ‘set /A Değeri=%8 %3’ yazılmalıdır ve STDOUT (standart çıktı) işlemek için sırasıyla hiçbir çıktı yazılmaz. Bir toplu iş dosyasındaki set /A Değeri=8 % 3 satırı, toplu iş dosyasının yürütülmesinde Eksik operatör hata mesajına neden olur.

Ortam, sıradan dize değişkenleri için değil, yalnızca aritmetik işlemler için “/A” anahtarını gerektirir.

Set /A’dan sonraki aritmetik ifadedeki her dize, ister bir sayı ister bir operatör olsun, otomatik olarak bir ortam değişkeninin adı olarak yorumlanır.

Bu nedenle, değişken adı sadece kelime karakterlerinden (0-9A-Za-z_) ve ilk karakter rakam olmayan bir karakterden oluştuğunda, bir değişkenin değerine %variable% veya !variable! ile referans vermek gerekli değildir. bu özellikle ( ile başlayan ve eşleşen bir ) ile biten bir komut bloğu içinde yararlıdır.

Sayılar, C/C++ işlevi strtol ile dizeden tam sayıya dönüştürülür ve “taban” sıfırdır, bu da kolayca beklenmedik sonuçlara yol açabilecek otomatik temel belirleme anlamına gelir Sonuçlar.

Örnek:

set Divided=11
set Divisor=3

set /A Quotient=Divided / Divisor
set /A Remainder=Divided %% Divisor

echo %Divided% / %Divisor% = %Quotient%
echo %Divided% %% %Divisor% = %Remainder%

set HexValue1=0x14
set HexValue2=0x0A
set /A Result=(HexValue1 + HexValue2) * -3

echo (%HexValue1% + %HexValue2%) * -3 = (20 + 10) * -3 = %Result%

set /A Result%%=7
echo -90 %%= 7 = %Result%

set OctalValue=020
set DecimalValue=12
set /A Result=OctalValue - DecimalValue

echo %OctalValue% - %DecimalValue% = 16 - 12 = %Result%

Bu örneğin çıktısı:

11 / 3 = 3
11 % 3 = 2
(0x14 + 0x0A) * -3 = (20 + 10) * -3 = -90
-90 %= 7 = -6
020 - 12 = 16 - 12 = 4

Aritmetik ifadenin değerlendirilmesinde tanımlanmayan değişkenler 0 değeri ile değiştirilir.
