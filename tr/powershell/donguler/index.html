Döngü, belirli bir koşula ulaşılana kadar sürekli olarak tekrarlanan bir talimat(lar) dizisidir. Programınızın bir kod bloğunu tekrar tekrar yürütmesini sağlamak, programlamadaki en temel ancak faydalı görevlerden biridir. Bir döngü, yalnızca tekrar ederek önemli ölçüde daha büyük bir sonuç üretmek için çok basit bir ifade yazmanıza izin verir. Eğer koşula ulaşılırsa, bir sonraki talimat sonraki sıralı talimata “düşer” veya döngünün dışında dallanır.

Sözdizimi #

  • için ( <Başlatma>; <Koşul>; <Tekrar> ) { <Script_Block> }

  • <Koleksiyon> | Foreach-Object { <Script_Block_with_$__as_current_item> }

  • foreach ( <Koleksiyon> içindeki <Item> ) { <Script_Block> }

  • while ( <Koşul> ){ <Script_Block> }

  • { <Script_Block> } while ( <Koşul> ) yapın

  • şuna kadar { <Script_Block>} yapın ( <Koşul> )

  • <Koleksiyon>.foreach( { <Script_Block_with_$__as_current_item> } )

Her biri için #

PowerShell’de bir foreach döngüsü çalıştırmanın birden çok yolu vardır ve bunların hepsi kendi avantaj ve dezavantajlarını beraberinde getirir:

Çözüm Avantajlar Dezavantajlar
Foreach ifadesi En hızlı. Statik koleksiyonlarla en iyi şekilde çalışır (bir değişkende saklanır). Boru hattı girişi veya çıkışı yok
ForEach() Yöntemi “Foreach-Object” ile aynı scriptblock sözdizimi, ancak daha hızlı. Statik koleksiyonlarla en iyi şekilde çalışır (bir değişkende saklanır). Boru hattı çıkışını destekler. Boru hattı girişi için destek yok. PowerShell 4.0 veya üstünü gerektirir
Foreach-Object (cmdlet) Boru hattı giriş ve çıkışını destekler. Bağlantıların başlatılması ve kapatılması vb. için başlangıç ​​ve bitiş komut dosyası bloklarını destekler. En esnek çözüm. en yavaş

Verim #

$foreach = Measure-Command { foreach ($i in (1..1000000)) { $i * $i } }
$foreachmethod = Measure-Command { (1..1000000).ForEach{ $_ * $_ } }
$foreachobject = Measure-Command { (1..1000000) | ForEach-Object { $_ * $_ } }

"Foreach: $($foreach.TotalSeconds)"
"Foreach method: $($foreachmethod.TotalSeconds)"
"ForEach-Object: $($foreachobject.TotalSeconds)"

Example output:

Foreach: 1.9039875
Foreach method: 4.7559563
ForEach-Object: 10.7543821

‘Foreach-Object’ en yavaşı olsa da, ardışık düzen desteği, öğeleri geldikleri anda (bir dosya okurken, veri alırken vb.) işlemenize izin verdiği için yararlı olabilir. İşlemden önce tüm verileri belleğe yüklemeniz gerekmediğinden, bu, büyük verilerle ve düşük bellekle çalışırken çok yararlı olabilir.

Her biri için #

“ForEach”, PowerShell’de iki farklı anlama sahiptir. Biri anahtar kelime ve diğeri [ForEach-Object] için bir takma addır. (https://www.wikiod.com/tr/powershell/donguler#ForEach-Object) cmdlet. Birincisi burada açıklanmıştır.

Bu örnek, bir dizideki tüm öğelerin konsol ana bilgisayarına yazdırılmasını gösterir:

$Names = @('Amy', 'Bob', 'Celine', 'David')

ForEach ($Name in $Names)
{
    Write-Host "Hi, my name is $Name!"
}

Bu örnek, bir ForEach döngüsünün çıktısının yakalanmasını gösterir:

$Numbers = ForEach ($Number in 1..20) {
    $Number # Alternatively, Write-Output $Number
}

Son örnekte olduğu gibi, bu örnek de döngüyü kaydetmeden önce bir dizi oluşturmayı gösterir:

$Numbers = @()
ForEach ($Number in 1..20)
{
    $Numbers += $Number
}

İçin #

for($i = 0; $i -le 5; $i++){
    "$i"
}

for döngüsünün tipik bir kullanımı, bir dizideki değerlerin bir alt kümesi üzerinde çalışmaktır. Çoğu durumda, bir dizideki tüm değerleri yinelemek istiyorsanız, bir foreach ifadesi kullanmayı düşünün.

ForEach() Yöntemi #

‘ForEach-Object’ cmdlet’i yerine, burada aynı zamanda bir ‘ForEach’ yöntemini doğrudan nesne dizilerinde kullanma olasılığı da vardır.

(1..10).ForEach({$_ * $_})

veya - istenirse - komut dosyası bloğunun etrafındaki parantezler atlanabilir

(1..10).ForEach{$_ * $_}

Her ikisi de aşağıdaki çıktıyla sonuçlanacaktır

1
4
9
16
25
36
49
64
81
100
son sürüm

ForEach-Object #

“ForEach-Object” cmdlet’i, [‘foreach’](https://www.wikiod.com/tr/powershell/donguler#Her biri için) ifadesine benzer şekilde çalışır, ancak girdisini ardışık düzenden alır .

Temel kullanım #

$object | ForEach-Object {
    code_block
}

Örnek:

$names = @("Any","Bob","Celine","David")
$names | ForEach-Object {
    "Hi, my name is $_!"
}

“Foreach-Object"in iki varsayılan takma adı vardır, “foreach” ve “%” (kısa yol sözdizimi). En yaygın olanı “%“dir, çünkü “foreach” [foreach ifadesi](https://www.wikiod.com/tr/powershell/donguler#Her biri için) ile karıştırılabilir. Örnekler:

$names | % {  
    "Hi, my name is $_!"
} 

$names | foreach {  
    "Hi, my name is $_!"
}

Gelişmiş kullanım #

‘Foreach-Object’, alternatif ‘foreach’ çözümlerinden sıyrılıyor çünkü bu bir cmdlet, yani boru hattını kullanmak için tasarlandı. Bu nedenle, tıpkı bir cmdlet veya gelişmiş işlev gibi üç komut dosyası bloğunu destekler:

  • Başla: İşlem hattından gelen öğeler arasında döngü yapılmadan önce bir kez yürütülür. Genellikle döngüde kullanılmak üzere işlevler oluşturmak, değişkenler oluşturmak, bağlantıları açmak (veritabanı, web +) vb. için kullanılır.
  • Süreç: Ardışık düzenden gelen her öğe için bir kez yürütülür. “Normal” foreach kod bloğu. Bu, parametre belirtilmediğinde yukarıdaki örneklerde kullanılan varsayılandır.
  • Bitiş: Tüm öğeler işlendikten sonra bir kez yürütülür. Genellikle bağlantıları kapatmak, rapor oluşturmak vb. için kullanılır.

Örnek:

"Any","Bob","Celine","David" | ForEach-Object -Begin {
    $results = @()
} -Process {
    #Create and store message
    $results += "Hi, my name is $_!"
} -End {
    #Count messages and output
    Write-Host "Total messages: $($results.Count)"
    $results
}

Devam et #

“Continue” operatörü “For”, “ForEach”, “While” ve “Do” döngülerinde çalışır. En içteki döngünün en üstüne atlayarak döngünün geçerli yinelemesini atlar.

$i =0
while ($i -lt 20) {
    $i++ 
    if ($i -eq 7) { continue }
    Write-Host $I
}

Yukarıdakiler konsola 1’den 20’ye çıktı verir ancak 7 sayısını kaçırır.

Not: Bir ardışık düzen döngüsü kullanırken ‘Devam’ yerine ‘dönüş’ kullanmalısınız.

Kırmak #

‘break’ operatörü bir program döngüsünden hemen çıkacaktır. ‘For’, ‘ForEach’, ‘While’ ve ‘Do’ döngülerinde veya bir ‘Switch’ İfadesinde kullanılabilir.

$i = 0
while ($i -lt 15) {
    $i++ 
    if ($i -eq 7) {break}
    Write-Host $i
}

Yukarıdakiler 15’e kadar sayılacak, ancak 7’ye ulaşıldığında duracaktır.

Not: Bir ardışık düzen döngüsü kullanırken, “break”, “devam” olarak davranacaktır. İşlem hattı döngüsünde ‘break’ simülasyonu yapmak için bazı ek mantık, cmdlet vb. eklemeniz gerekir. ‘break’ kullanmanız gerekiyorsa, ardışık düzen olmayan döngülere bağlı kalmak daha kolaydır.

Etiketleri Kır

Break, bir döngünün örneğinin önüne yerleştirilmiş bir etiketi de çağırabilir:

$i = 0
:mainLoop While ($i -lt 15) {
    Write-Host $i -ForegroundColor 'Cyan'
    $j = 0
    While ($j -lt 15) {
        Write-Host $j -ForegroundColor 'Magenta'
        $k = $i*$j
        Write-Host $k -ForegroundColor 'Green'
        if ($k -gt 100) {
            break mainLoop
        }
        $j++
    }
    $i++
}

Not: Bu kod, “$i"yi “8"e ve “$j"yi “13"e yükselterek “$k"nin “104"e eşit olmasına neden olur. “$k”, “100"ü aştığından, kod her iki döngüden de çıkacaktır.

Süre #

Bir while döngüsü bir koşulu değerlendirecek ve true ise bir eylem gerçekleştirecektir. Koşul doğru olarak değerlendirildiği sürece eylem gerçekleştirilmeye devam edecektir.

while(condition){
  code_block
}

Aşağıdaki örnek, 10’dan 0’a geri sayacak bir döngü oluşturur

$i = 10
while($i -ge 0){
    $i
    $i--
}

Do-While döngüsünden farklı olarak, koşul, eylemin ilk yürütülmesinden önce değerlendirilir. Başlangıç ​​koşulu yanlış olarak değerlendirilirse eylem gerçekleştirilmez.

Not: Koşulu değerlendirirken PowerShell, bir dönüş nesnesinin varlığını doğru olarak değerlendirir. Bu, birkaç şekilde kullanılabilir, ancak aşağıda bir işlemi izlemek için bir örnek verilmiştir. Bu örnek, bir not defteri işlemi oluşturacak ve ardından bu işlem çalıştığı sürece geçerli kabuğu uyuyacaktır. Not defteri örneğini manuel olarak kapattığınızda, while koşulu başarısız olur ve döngü bozulur.

Start-Process notepad.exe
while(Get-Process notepad -ErrorAction SilentlyContinue){
  Start-Sleep -Milliseconds 500
}

Yapmak #

Do döngüleri, her zaman en az bir kez bir kod bloğu çalıştırmak istediğinizde kullanışlıdır. Bir Do-loop, kod bloğunu çalıştırmadan önce yapan while döngüsünün aksine, kod bloğunu çalıştırdıktan sonra durumu değerlendirecektir.

Do döngülerini iki şekilde kullanabilirsiniz:

  • Koşul doğruyken *döngü yapın:

    Do {
    code_block
} while (condition)

  • Koşul doğru olana kadar *döngü yapın, diğer bir deyişle koşul yanlışken döngü yapın:

    Do {
    code_block
} until (condition)

Gerçek Örnekler:

$i = 0

Do {
    $i++
    "Number $i"
} while ($i -ne 3)

Do {
    $i++
    "Number $i"
} until ($i -eq 3)

Do-While ve Do-Until, zıt döngülerdir. Kod aynı ise, durum tersine çevrilir. Yukarıdaki örnek bu davranışı göstermektedir.
