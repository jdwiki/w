Parametreler #

Parametre Açıklama
-Yardım | -? | /? Yardımı gösterir
-Dosya <DosyaYolu> [<Args>] Yürütülmesi gereken komut dosyası yolu ve bağımsız değişkenler (isteğe bağlı)
-Komut { - | <script-block> [-args <arg-dizi>] | <string> [<CommandParameters>] } Argümanlar tarafından takip edilecek komutlar
-EncodedCommand <Base64EncodedCommand> Base64 ile kodlanmış komutlar
-ExecutionPolicy <ExecutionPolicy> Yalnızca bu işlem için yürütme politikasını ayarlar
-InputFormat { Metin | XML} İşleme gönderilen veriler için giriş biçimini ayarlar. Metin (dizeler) veya XML (serileştirilmiş CLIXML)
-Mta PowerShell 3.0+: PowerShell’i çok iş parçacıklı dairede çalıştırır (STA varsayılandır)
-Sta PowerShell 2.0: PowerShell’i tek iş parçacıklı bir dairede çalıştırır (MTA varsayılandır)
-Çıkış Yok Komut dosyasını/komutu yürüttükten sonra PowerShell konsolunu çalışır durumda bırakır
-NoLogo Başlatma sırasında telif hakkı afişini gizler
-Etkileşimsiz Konsolu kullanıcıdan gizler
-Profil Yok Makine veya kullanıcı için PowerShell profillerinin yüklenmesinden kaçının
-OutputFormat { Metin | XML } PowerShell’den döndürülen veriler için çıktı biçimini ayarlar. Metin (dizeler) veya XML (serileştirilmiş CLIXML)
-PSConsoleFile <FilePath> Ortamı yapılandıran önceden oluşturulmuş bir konsol dosyası yükler (“Export-Console” kullanılarak oluşturulur)
-Sürüm <Windows PowerShell sürümü> Çalıştırılacak bir PowerShell sürümü belirtin. Çoğunlukla “2.0” ile kullanılır
-WindowStyle <stil> PowerShell işleminin “normal”, “gizli”, “küçültülmüş” veya “büyütülmüş” bir pencere olarak başlatılıp başlatılmayacağını belirtir.

Komut yürütme #

-Command parametresi, başlatma sırasında yürütülecek komutları belirtmek için kullanılır. Çoklu veri girişlerini destekler.

-Komut <dize> #

Başlatma sırasında yürütülecek komutları bir dize olarak belirtebilirsiniz. Birden çok noktalı virgül ; ile ayrılmış ifadeler yürütülebilir.

>PowerShell.exe -Command "(Get-Date).ToShortDateString()"
10.09.2016

>PowerShell.exe -Command "(Get-Date).ToShortDateString(); 'PowerShell is fun!'"
10.09.2016
PowerShell is fun!

-Komut { scriptblock } #

-Command parametresi ayrıca bir scriptblock girişini de destekler (bir veya birden çok ifade { #code } ayraçlarına sarılır. Bu yalnızca başka bir Windows PowerShell oturumundan PowerShell.exe çağrılırken çalışır.

PS > powershell.exe -Command {
"This can be useful, sometimes..."
(Get-Date).ToShortDateString()
}
This can be useful, sometimes...
10.09.2016

-Komut - (standart giriş) #

Standart girdiden komutları -Command - kullanarak iletebilirsiniz. Standart girdi “echo"dan, bir dosyadan, eski bir konsol uygulamasından vb. gelebilir.

>echo "Hello World";"Greetings from PowerShell" | PowerShell.exe -NoProfile -Command -
Hello World
Greetings from PowerShell

Bir komut dosyası yürütme #

-File parametresini kullanarak başlatma sırasında içeriğini yürütmek için bir ps1 betiğine bir dosya belirtebilirsiniz.

Temel komut dosyası #

MyScript.ps1

(Get-Date).ToShortDateString()
"Hello World"

Çıktı:

>PowerShell.exe -File Desktop\MyScript.ps1
10.09.2016
Hello World

Parametreleri ve argümanları kullanma #

Bunları komut dosyasında kullanmak için dosya yolundan sonra parametreler ve/veya bağımsız değişkenler ekleyebilirsiniz. Argümanlar tanımsız/mevcut kod parametreleri için değerler olarak kullanılacak, geri kalanı $args-dizisinde mevcut olacak

MyScript.ps1

param($Name)

"Hello $Name! Today's date it $((Get-Date).ToShortDateString())"
"First arg: $($args[0])"

Çıktı:

>PowerShell.exe -File Desktop\MyScript.ps1 -Name StackOverflow foo
Hello StackOverflow! Today's date it 10.09.2016
First arg: foo
