Sözdizimi #

  • [[ -OP $dosyaadı ]]
  • [[ $dosya1 -OP $dosya2 ]]
  • [[ -z $dize ]]
  • [[ -n $dize ]]
  • [[ “$string1” == “$string2” ]]
  • [[ “$string1” == $desen ]]

‘[[ … ]]’ sözdizimi, bash yerleşik koşullu ifadelerini çevreler. Köşeli ayraçların her iki tarafında boşluk olması gerektiğini unutmayın.

Koşullu ifadeler, dizelerin, tam sayıların ve dosyaların özelliklerini test etmek için tekli ve ikili operatörleri kullanabilir. Ayrıca &&, || ve ! mantıksal operatörlerini de kullanabilirler.

Dosya türü testleri #

‘-e’ koşullu işleci, bir dosyanın var olup olmadığını test eder (tüm dosya türleri dahil: dizinler, vb.).

if [[ -e $filename ]]; then
  echo "$filename exists"
fi

Belirli dosya türleri için de testler vardır.

if [[ -f $filename ]]; then
  echo "$filename is a regular file"
elif [[ -d $filename ]]; then
  echo "$filename is a directory"
elif [[ -p $filename ]]; then
  echo "$filename is a named pipe"
elif [[ -S $filename ]]; then
  echo "$filename is a named socket"
elif [[ -b $filename ]]; then
  echo "$filename is a block device"
elif [[ -c $filename ]]; then
  echo "$filename is a character device"
fi
if [[ -L $filename ]]; then
  echo "$filename is a symbolic link (to any file type)"
fi

Sembolik bir bağlantı için, “-L” dışında, bu testler hedefe uygulanır ve bozuk bir bağlantı için false döndürür.

if [[ -L $filename || -e $filename ]]; then
  echo "$filename exists (but may be a broken symbolic link)"
fi

if [[ -L $filename && ! -e $filename ]]; then
  echo "$filename is a broken symbolic link"
fi

Dize karşılaştırma ve eşleştirme #

Dize karşılaştırması, alıntılanmış dizeler arasında == operatörünü kullanır. != operatörü karşılaştırmayı reddeder.

if [[ "$string1" == "$string2" ]]; then
  echo "\$string1 and \$string2 are identical"
fi
if [[ "$string1" != "$string2" ]]; then
  echo "\$string1 and \$string2 are not identical"
fi

Sağ taraf alıntılanmamışsa, “$string1” ile eşleşen bir joker karakterdir.

string='abc'
pattern1='a*'
pattern2='x*'
if [[ "$string" == $pattern1 ]]; then
  # the test is true
  echo "The string $string matches the pattern $pattern"
fi
if [[ "$string" != $pattern2 ]]; then
  # the test is false
  echo "The string $string does not match the pattern $pattern"
fi

“<” ve “>” operatörleri, dizeleri sözlükbilimsel sırayla karşılaştırır (dizeler için daha az veya eşit veya daha büyük veya eşit operatörler yoktur).

Boş dize için tekli testler vardır.

if [[ -n "$string" ]]; then
  echo "$string is non-empty"
fi
if [[ -z "${string// }" ]]; then
  echo "$string is empty or contains only spaces"
fi
if [[ -z "$string" ]]; then
  echo "$string is empty"
fi

Yukarıda, “-z” denetimi, “$string” öğesinin ayarlanmadığı veya boş bir dizeye ayarlandığı anlamına gelebilir. Boş ve ayarlanmamış arasında ayrım yapmak için şunu kullanın:

if [[ -n "${string+x}" ]]; then
    echo "$string is set, possibly to the empty string"
fi
if [[ -n "${string-x}" ]]; then
    echo "$string is either unset or set to a non-empty string"
fi
if [[ -z "${string+x}" ]]; then
    echo "$string is unset"
fi
if [[ -z "${string-x}" ]]; then
    echo "$string is set to an empty string"
fi

burada ‘x’ keyfidir. Veya tablo biçiminde:

                        +-------+-------+-----------+
            $string is: | unset | empty | non-empty |
+-----------------------+-------+-------+-----------+
| [[ -z ${string} ]]    | true  | true  | false     |
| [[ -z ${string+x} ]]  | true  | false | false     |
| [[ -z ${string-x} ]]  | false | true  | false     |
| [[ -n ${string} ]]    | false | false | true      |
| [[ -n ${string+x} ]]  | false | true  | true      |
| [[ -n ${string-x} ]]  | true  | false | true      |
+-----------------------+-------+-------+-----------+

Alternatif olarak, durum bir vaka ifadesinde kontrol edilebilir:

case ${var+x$var} in
  (x) echo empty;;
  ("") echo unset;;
  (x*[![:blank:]]*) echo non-blank;;
  (*) echo blank
esac

Burada ‘[:blank:]’ yerel ayara özgü yatay boşluk karakterleridir (sekme, boşluk, vb.).

Bir komutun çıkış durumunu test edin #

Çıkış durumu 0: başarılı
0 dışında çıkış durumu: başarısızlık

Bir komutun çıkış durumunu test etmek için:

if command;then
    echo 'success'
else
    echo 'failure'
fi

Dosya karşılaştırma #

if [[ $file1 -ef $file2 ]]; then
  echo "$file1 and $file2 are the same file"
fi

“Aynı dosya”, dosyalardan birinin yerinde değiştirilmesinin diğerini etkilediği anlamına gelir. İki dosya, farklı adlara sahip olsalar bile aynı olabilir, örneğin bunlar sabit bağlantılar ise veya aynı hedefe sahip sembolik bağlantılar ise veya biri diğerine işaret eden bir sembolik bağlantıysa.

İki dosya aynı içeriğe sahipse, ancak bunlar farklı dosyalarsa (böylece birini değiştirmek diğerini etkilemez), o zaman “-ef” onları farklı olarak bildirir. İki dosyayı bayt bayt karşılaştırmak istiyorsanız, cmp yardımcı programını kullanın.

if cmp -s -- "$file1" "$file2"; then
  echo "$file1 and $file2 have identical contents"
else
  echo "$file1 and $file2 differ"
fi

Metin dosyaları arasında insan tarafından okunabilir bir fark listesi oluşturmak için, “diff” yardımcı programını kullanın.

if diff -u "$file1" "$file2"; then
  echo "$file1 and $file2 have identical contents"
else
  : # the differences between the files have been listed
fi

Dosya erişim testleri #

if [[ -r $filename ]]; then
  echo "$filename is a readable file"
fi
if [[ -w $filename ]]; then
  echo "$filename is a writable file"
fi
if [[ -x $filename ]]; then
  echo "$filename is an executable file"
fi

Bu testler, komut dosyasının (veya komut dosyasından başlatılan programların) dosyaya erişip erişemeyeceğini belirlemek için izinleri ve sahipliği dikkate alır.

Yarış koşullarına (TOCTOU) dikkat edin: Testin şimdi başarılı olması, sonraki satırda hala geçerli olduğu anlamına gelmez. Bir dosyaya erişmeyi denemek ve önce test etmek ve bu arada dosyanın değişmesi durumunda yine de hatayı işlemek yerine hatayı işlemek daha iyidir.

Sayısal karşılaştırmalar #

Sayısal karşılaştırmalar “-eq” operatörlerini ve arkadaşlarını kullanır

if [[ $num1 -eq $num2 ]]; then
  echo "$num1 == $num2"
fi
if [[ $num1 -le $num2 ]]; then
  echo "$num1 <= $num2"
fi

Altı sayısal operatör vardır:

  • -eq eşittir
  • -ne eşit değil
  • -le daha az veya eşit
  • -lt daha az
  • -ge büyük veya eşit
  • -gt daha büyük

“[[ … ]]” içindeki “<” ve “>” operatörlerinin sayıları değil dizeleri karşılaştırdığını unutmayın.

if [[ 9 -lt 10 ]]; then
  echo "9 is before 10 in numeric order"
fi
if [[ 9 > 10 ]]; then
  echo "9 is after 10 in lexicographic order"
fi

İki taraf ondalık (veya başında sıfır olacak şekilde sekizlik) olarak yazılmış sayılar olmalıdır. Alternatif olarak, C/Java/… benzeri bir sözdiziminde tamsayı hesaplamaları yapan ((…)) aritmetik ifade sözdizimini kullanın.

x=2
if ((2*x == 4)); then
  echo "2 times 2 is 4"
fi
((x += 1))
echo "2 plus 1 is $x"

Bir astar testi #

Bunun gibi şeyler yapabilirsiniz:

[[ $s = 'something' ]] && echo 'matched' || echo "didn't match"
[[ $s == 'something' ]] && echo 'matched' || echo "didn't match"
[[ $s != 'something' ]] && echo "didn't match" || echo "matched"
[[ $s -eq 10 ]] && echo 'equal' || echo "not equal"
(( $s == 10 )) && echo 'equal' || echo 'not equal'

Çıkış durumu için bir astar testi:

command && echo 'exited with 0' || echo 'non 0 exit'
cmd && cmd1 && echo 'previous cmds were successful' || echo 'one of them failed'
cmd || cmd1 #If cmd fails try cmd1
