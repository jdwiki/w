Sözdizimi #

  • “function” anahtar sözcüğüyle bir işlev tanımlayın:

     function f {

    }

  • () ile bir fonksiyon tanımlayın:

     f(){

    }

  • Hem “function” anahtar sözcüğü hem de “()” ile bir işlev tanımlayın:

     function f(){

    }

Argümanlı fonksiyonlar #

“helloJohn.sh” içinde:

#!/bin/bash

greet() {
  local name="$1"
  echo "Hello, $name"
}

greet "John Doe"

# running above script
$ bash helloJohn.sh
Hello, John Doe

  1. Eğer argümanı herhangi bir şekilde değiştirmezseniz, onu bir “yerel” değişkene kopyalamanıza gerek yoktur - sadece “echo “Merhaba, $1”`.

  2. Fonksiyonun içindeki argümanlara erişmek için $1, $2, $3 ve benzerlerini kullanabilirsiniz.

    Note: for arguments more than 9 $10 won’t work (bash will read it as $10), you need to do ${10}, ${11} and so on.

  3. $@, bir fonksiyonun tüm argümanlarını ifade eder:

    #!/bin/bash
foo() {
  echo "$@"
}

foo 1 2 3 # output => 1 2 3

    Note: You should practically always use double quotes around "$@", like here.

    Omitting the quotes will cause the shell to expand wildcards (even when the user specifically quoted them in order to avoid that) and generally introduce unwelcome behavior and potentially even security problems.

    foo "string with spaces;" '$HOME' "*"
# output => string with spaces; $HOME *

  4. varsayılan bağımsız değişkenler için ${1:-default_val} kullanın. Örneğin:

#!/bin/bash foo() { local val=${1:-25} echo “$val” }

foo # çıktı => 25 foo 30 # çıktı => 30


5. bir argüman istemek için `${var:?error message}` kullanın
 ```
 foo() {
   local val=${1:?Must provide an argument}
   echo "$val"
 }
 ```

## Basit İşlev
"helloWorld.sh" içinde

 #!/bin/bash

 # Define a function greet
 greet ()
 {
     echo "Hello World!"
 }
 
 # Call the function greet
 greet

Komut dosyasını çalıştırırken mesajımızı görüyoruz

 $ bash helloWorld.sh
 Hello World!

*[Sourcing][1] işlevlere sahip bir dosyanın bunları geçerli bash oturumunuzda kullanılabilir hale getirdiğini unutmayın.

 $ source helloWorld.sh   # or, more portably, ". helloWorld.sh"
 $ greet
 Hello World!

[1]: https://www.wikiod.com/tr/bash/kaynak-bulma


Bazı kabuklarda bir işlevi "dışa aktarabilirsiniz", böylece alt işlemlere maruz kalır.

 bash -c 'greet'  # fails
 export -f greet  # export function; note -f
 bash -c 'greet'  # success

## Bayrakları ve isteğe bağlı parametreleri işleme
*getopts* yerleşik, bayrakları ve isteğe bağlı parametreleri barındıran işlevleri yazmak için işlevlerin içinde kullanılabilir. Bu özel bir zorluk oluşturmaz, ancak *getopts* tarafından dokunulan değerleri uygun şekilde ele almak gerekir. Örnek olarak, *stderr* üzerine bir mesaj yazan ve kod 1 ile veya `-x` seçeneğine parametre olarak sağlanan rastgele bir kodla çıkan bir *failwith* işlevi tanımlıyoruz:

 # failwith [-x STATUS] PRINTF-LIKE-ARGV
 #  Fail with the given diagnostic message
 #
 # The -x flag can be used to convey a custom exit status, instead of
 # the value 1.  A newline is automatically added to the output.

 failwith()
 {
     local OPTIND OPTION OPTARG status
 
     status=1
     OPTIND=1
 
     while getopts 'x:' OPTION; do
         case ${OPTION} in
             x)    status="${OPTARG}";;
             *)    1>&2 printf 'failwith: %s: Unsupported option.\n' "${OPTION}";;
         esac
     done
 
     shift $(( OPTIND - 1 ))
     {
         printf 'Failure: '
         printf "$@"
         printf '\n'
     } 1>&2
     exit "${status}"
 }

Bu işlev aşağıdaki gibi kullanılabilir:

 failwith '%s: File not found.' "${filename}"
 failwith -x 70 'General internal error.'

ve benzeri.

*printf* için değişkenlerin ilk argüman olarak kullanılmaması gerektiğini unutmayın. Yazdırılacak mesaj bir değişkenin içeriğinden oluşuyorsa, bunu yazdırmak için `%s` belirteci aşağıdaki gibi kullanılmalıdır.

 failwith '%s' "${message}"


## Bir fonksiyondan değer döndürme
Bash'deki 'return' ifadesi C-işlevleri gibi bir değer döndürmez, bunun yerine işlevden bir dönüş durumuyla çıkar. Bunu o fonksiyonun çıkış durumu olarak düşünebilirsiniz.

İşlevden bir değer döndürmek istiyorsanız, değeri şu şekilde 'stdout'a gönderin:

 fun() {
     local var="Sample value to be returned"
     echo "$var"
     #printf "%s\n" "$var"
 }

Şimdi, yaparsanız:

 var="$(fun)"

'fun' çıktısı '$var' içinde saklanacaktır.

## Bir işlevin çıkış kodu, son komutunun çıkış kodudur
Bir ana bilgisayarın açık olup olmadığını kontrol etmek için bu örnek işlevi göz önünde bulundurun:

 is_alive() {
     ping -c1 "$1" &> /dev/null
 }

Bu işlev, ilk işlev parametresi tarafından belirtilen ana bilgisayara tek bir ping gönderir.
"ping"in çıktısı ve hata çıktısı "/dev/null" dizinine yönlendirilir,
bu nedenle işlev hiçbir zaman hiçbir şey çıkarmaz.
Ancak 'ping' komutu başarı durumunda 0 çıkış koduna sahip olacaktır,
ve başarısızlıkta sıfır olmayan.
Bu, işlevin son (ve bu örnekte, tek) komutu olduğundan,
ping'in çıkış kodu, işlevin çıkış kodu için yeniden kullanılacaktır.

Bu gerçek koşullu ifadelerde çok kullanışlıdır.

Örneğin, "graucho" ana bilgisayarı çalışıyorsa, "ssh" ile ona bağlanın:

 if is_alive graucho; then
     ssh graucho
 fi

Başka bir örnek: "graucho" ana bilgisayarı çalışana kadar tekrar tekrar kontrol edin ve ardından "ssh" ile ona bağlanın:

 while ! is_alive graucho; do
     sleep 5
 done
 ssh graucho

## İşlev tanımını yazdır
 getfunc() {
     declare -f "$@"
 }
 
 function func(){
     echo "I am a sample function"
 }
 
 funcd="$(getfunc func)" 
 getfunc func # or echo "$funcd"

Çıktı:

 func () 
 { 
     echo "I am a sample function"
 }



## Adlandırılmış parametreleri kabul eden bir işlev
 foo() {
   while [[ "$#" -gt 0 ]]
   do
     case $1 in
       -f|--follow)
         local FOLLOW="following"
         ;;
       -t|--tail)
         local TAIL="tail=$2"
         ;;
     esac
     shift
   done
 
   echo "FOLLOW: $FOLLOW"
   echo "TAIL: $TAIL"
 }

Örnek kullanım:

 foo -f
 foo -t 10
 foo -f --tail 10
 foo --follow --tail 10
