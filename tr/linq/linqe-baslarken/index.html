Kurmak #

LINQ, .NET 3.5 veya sonraki bir sürümünü (veya LINQBridge kullanan .NET 2.0) gerektirir.

Henüz eklenmemişse System.Core‘a bir başvuru ekleyin.

Dosyanın en üstünde ad alanını içe aktarın:

  • C#
  using System;
  using System.Linq;
  • VB.NET
  Imports System.Linq

LINQ’daki farklı birleşimler #

Aşağıdaki örneklerde, aşağıdaki örnekleri kullanacağız:

List<Product> Products = new List<Product>()
{
  new Product()
  {
    ProductId = 1,
    Name = "Book nr 1",
    Price = 25
  },
  new Product()
  {
    ProductId = 2,
    Name = "Book nr 2",
    Price = 15
  },
  new Product()
  {
    ProductId = 3,
    Name = "Book nr 3",
    Price = 20
  },
};
List<Order> Orders = new List<Order>()
{
  new Order()
  {
    OrderId = 1,
    ProductId = 1,
  },
  new Order()
  {
    OrderId = 2,
    ProductId = 1,
  },
  new Order()
  {
    OrderId = 3,
    ProductId = 2,
  },
  new Order()
  {
    OrderId = 4,
    ProductId = NULL,
  },
};

İÇ BİRLEŞİM

Sorgu Sözdizimi

var joined = (from p in Products
              join o in Orders on p.ProductId equals o.ProductId
              select new
              {
                o.OrderId,
                p.ProductId,
                p.Name
              }).ToList();

Yöntem Sözdizimi 

var joined = Products.Join(Orders, p => p.ProductId, 
                                   o => o.OrderId, 
                                     => new 
                                    { 
                                      OrderId   = o.OrderId, 
                                      ProductId = p.ProductId, 
                                      Name      = p.Name 
                                    })
                     .ToList();

Sonuç:

{ 1, 1, "Book nr 1" },
{ 2, 1, "Book nr 1" },
{ 3, 2, "Book nr 2" }

SOL DIŞ KATILMA

var joined = (from p in Products
              join o in Orders on p.ProductId equals o.ProductId into g
              from lj in g.DefaultIfEmpty()
              select new
              {
                //For the empty records in lj, OrderId would be NULL
                OrderId = (int?)lj.OrderId,
                p.ProductId,
                p.Name
              }).ToList();

Sonuç:

{ 1, 1, "Book nr 1" },
{ 2, 1, "Book nr 1" },
{ 3, 2, "Book nr 2" },
{ NULL, 3, "Book nr 3" }

ÇAPRAZ BİRLEŞTİR

var joined = (from p in Products
              from o in Orders
              select new
              {
                o.OrderId,
                p.ProductId,
                p.Name
              }).ToList();

Sonuç:

{ 1, 1, "Book nr 1" },
{ 2, 1, "Book nr 1" },
{ 3, 2, "Book nr 2" },
{ NULL, 3, "Book nr 3" },
{ 4, NULL, NULL }

GRUP KATILIMI

var joined = (from p in Products
              join o in Orders on p.ProductId equals o.ProductId
                into t
              select new
              {
                p.ProductId,
                p.Name,
                Orders = t
              }).ToList();

“Siparişler” Özelliği artık tüm bağlantılı Siparişlerle birlikte bir “IEnumerable<Sipariş>” içeriyor.

Sonuç:

{ 1, "Book nr 1", Orders = { 1, 2 } },
{ 2, "Book nr 2", Orders = { 3 } },
{ 3, "Book nr 3", Orders = { } },

Birden çok koşulda nasıl katılınır

Tek bir koşulla katılırken şunları kullanabilirsiniz:

join o in Orders 
  on p.ProductId equals o.ProductId

Birden çok sisteme katılırken şunları kullanın:

join o in Orders 
  on new { p.ProductId, p.CategoryId } equals new { o.ProductId, o.CategoryId }

Her iki anonim nesnenin de aynı özelliklere sahip olduğundan ve VB.NET’te “Anahtar” olarak işaretlenmeleri gerektiğinden emin olun, ancak VB.NET “And” ile ayrılmış birden çok “Equals” yan tümcesine izin verir:

Join o In Orders 
  On p.ProductId Equals o.ProductId And p.CategoryId Equals o.CategoryId

Sorgu Sözdizimi ve Yöntem Sözdizimi #

Sorgu sözdizimi ve yöntem sözdizimi anlamsal olarak aynıdır, ancak birçok kişi sorgu sözdizimini daha basit ve okunması daha kolay bulur. Diyelim ki bir sayı koleksiyonundan artan sırada sıralanmış tüm çift öğeleri almamız gerekiyor.

C#:

int[] numbers = { 0, 1, 2, 3, 4, 5, 6 };

// Query syntax:
IEnumerable<int> numQuery1 =
            from num in numbers
            where num % 2 == 0
            orderby num
            select num;

// Method syntax:
IEnumerable<int> numQuery2 = numbers.Where(num => num % 2 == 0).OrderBy(n => n);

VB.NET:

Dim numbers() As Integer = { 0, 1, 2, 3, 4, 5, 6 }

' Query syntax: '
Dim numQuery1 = From num In numbers
                 Where num Mod 2 = 0
                 Select num
                 Order By num

' Method syntax: '
Dim numQuery2 = numbers.where(Function(num) num Mod 2 = 0).OrderBy(Function(num) num)

Bazı sorguların yöntem çağrıları olarak ifade edilmesi gerektiğini unutmayın. Örneğin, belirli bir koşulla eşleşen öğelerin sayısını alan bir sorguyu ifade etmek için bir yöntem çağrısı kullanmanız gerekir. Ayrıca, bir kaynak dizideki maksimum değere sahip öğeyi alan bir sorgu için bir yöntem çağrısı kullanmanız gerekir. Bu, kodu daha tutarlı hale getirmek için yöntem sözdizimini kullanmanın bir avantajı olabilir. Ancak, elbette, bir sorgu sözdizimi çağrısından sonra yöntemi her zaman uygulayabilirsiniz:

C#:

int maxNum =
    (from num in numbers
     where num % 2 == 0
     select num).Max();

VB.NET:

Dim maxNum =
    (From num In numbers
     Where num Mod 2 = 0
     Select num).Max();

LINQ yöntemleri ve IEnumerable ve IQueryable karşılaştırması #

‘IEnumerable’ üzerindeki LINQ uzantı yöntemleri, anonim yöntemler olsun, gerçek yöntemleri1 alır:

//C#
Func<int,bool> fn = x => x > 3;
var list = new List<int>() {1,2,3,4,5,6};
var query = list.Where(fn);

'VB.NET
Dim fn = Function(x As Integer) x > 3
Dim list = New List From {1,2,3,4,5,6};
Dim query = list.Where(fn);

veya adlandırılmış yöntemler (bir sınıfın parçası olarak açıkça tanımlanan yöntemler):

//C#
class Program {
    bool LessThan4(int x) {
        return x < 4;
    }

    void Main() {
        var list = new List<int>() {1,2,3,4,5,6};
        var query = list.Where(LessThan4);
    }
}

'VB.NET
Class Program
    Function LessThan4(x As Integer) As Boolean
        Return x < 4
    End Function
    Sub Main
        Dim list = New List From {1,2,3,4,5,6};
        Dim query = list.Where(AddressOf LessThan4)
    End Sub
End Class

Teorik olarak, yöntemin IL’sini ayrıştırmak, yöntemin ne yapmaya çalıştığını anlamak ve bu yöntemin mantığını yalnızca değil, temeldeki herhangi bir veri kaynağına uygulamak mümkündür. bellekteki nesneler. Ancak IL’yi ayrıştırmak kalbin zayıflığı için değildir.

Neyse ki, .NET bu senaryo için ‘IQueryable’ arabirimini ve ‘System.Linq.Queryable’daki uzantı yöntemlerini sağlar. Bu uzantı yöntemleri bir ifade ağacı alır — kodu temsil eden bir veri yapısı — LINQ sağlayıcısının daha sonra ayrıştırabileceği2 ve temel veri kaynağını sorgulamak için daha uygun bir forma dönüştürebileceği gerçek bir yöntem yerine. Örneğin:

//C#
IQueryable<Person> qry = PersonsSet();

// Since we're using a variable of type Expression<Func<Person,bool>>, the compiler 
// generates an expression tree representing this code
Expression<Func<Person,bool>> expr = x => x.LastName.StartsWith("A");
// The same thing happens when we write the lambda expression directly in the call to 
// Queryable.Where

qry = qry.Where(expr);


'VB.NET
Dim qry As IQueryable(Of Person) = PersonSet()

' Since we're using a variable of type Expression(Of Func(Of Person,Boolean)), the compiler 
' generates an expression tree representing this code
Dim expr As Expression(Of Func(Of Person, Boolean)) = Function(x) x.LastName.StartsWith("A")
' The same thing happens when we write the lambda expression directly in the call to 
' Queryable.Where

qry = qry.Where(expr)

(Örneğin) bu sorgu bir SQL veritabanına yönelikse, sağlayıcı bu ifadeyi aşağıdaki SQL ifadesine dönüştürebilir:

SELECT *
FROM Persons
WHERE LastName LIKE N'A%'

ve veri kaynağına karşı yürütün.

Öte yandan, sorgu bir REST API’ye karşıysa, sağlayıcı aynı ifadeyi bir API çağrısına dönüştürebilir:

http://www.example.com/person?filtervalue=A&filtertype=startswith&fieldname=lastname

Bir ifadeye dayalı olarak bir veri isteğini uyarlamanın iki temel faydası vardır (tüm koleksiyonu belleğe yüklemek ve yerel olarak sorgulamak yerine):

  • Temel veri kaynağı genellikle daha verimli sorgulayabilir. Örneğin, ‘LastName’ üzerinde çok iyi bir dizin olabilir. Nesneleri yerel belleğe yüklemek ve bellek içi sorgulamak bu verimliliği kaybeder.
  • Veriler aktarılmadan önce şekillendirilip küçültülebilir. Bu durumda, veri kaynağından sağlanan tüm Kişiler kümesinin aksine, veritabanı/web hizmetinin yalnızca eşleşen verileri döndürmesi gerekir.

Notlar 1. Teknik olarak, aslında yöntem almazlar, bunun yerine yöntemlere işaret eden örnekleri delege edin. Ancak bu ayrım burada önemsizdir. 2. Bu, hataların [like](http ://stackoverflow.com/q/10110266) “LINQ to Entities, ‘System.String ToString()’ yöntemini tanımıyor ve bu yöntem bir mağaza ifadesine çevrilemez.”. LINQ sağlayıcısı (bu durumda Entity Framework sağlayıcısı), bir “ToString” çağrısını nasıl ayrıştıracağını ve eşdeğer SQL’e çevireceğini bilmiyor.
