Büyük tablolara başvuran yabancı anahtarlarla uğraşmak #

Varsayılan olarak, Django “ForeignKey” alanlarını “” alanları yerine otomatik tamamlama alanlarının kullanılmasını sağlar.

görünümler.py #

from dal import autocomplete

class CityAutocomp(autocomplete.Select2QuerySetView):
    def get_queryset(self):
        qs = City.objects.all()
        if self.q:
            qs = qs.filter(name__istartswith=self.q)
        return qs

urls.py #

urlpatterns = [
    url(r'^city-autocomp/$', CityAutocomp.as_view(), name='city-autocomp'),
]

formlar.py #

from dal import autocomplete

class PlaceForm(forms.ModelForm):
    city = forms.ModelChoiceField(
        queryset=City.objects.all(),
        widget=autocomplete.ModelSelect2(url='city-autocomp')
    )

    class Meta:
        model = Place
        fields = ['__all__']

admin.py #

@admin.register(Place)
class PlaceAdmin(admin.ModelAdmin):
    form = PlaceForm

Yönetici sayfası için ek CSS stilleri ve JS komut dosyaları #

Basit bir “Müşteri” modeliniz olduğunu varsayalım:

class Customer(models.Model):
    first_name = models.CharField(max_length=255)
    last_name = models.CharField(max_length=255)
    is_premium = models.BooleanField(default=False)

Django yöneticisine kaydettirirsiniz ve “ilk_adı” ve “son_adı” ile arama alanı eklersiniz:

@admin.register(Customer)
class CustomerAdmin(admin.ModelAdmin):
    list_display = ['first_name', 'last_name', 'is_premium']
    search_fields = ['first_name', 'last_name']

Bunu yaptıktan sonra, arama alanları varsayılan yer tutucuyla yönetici listesi sayfasında görünür: “anahtar kelime”. Peki ya bu yer tutucuyu “Adla Ara” olarak değiştirmek isterseniz?

Bunu, özel Javascript dosyasını admin Mediaya geçirerek yapabilirsiniz:

@admin.register(Customer)
class CustomerAdmin(admin.ModelAdmin):
    list_display = ['first_name', 'last_name', 'is_premium']
    search_fields = ['first_name', 'last_name']
    
    class Media:
        #this path may be any you want, 
        #just put it in your static folder
        js = ('js/admin/placeholder.js', )

Django’nun bu arama çubuğuna hangi kimliği veya sınıfı ayarladığını bulmak için tarayıcı hata ayıklama araç çubuğunu kullanabilir ve ardından js kodunuzu yazabilirsiniz:

$(function () {
   $('#searchbar').attr('placeholder', 'Search by name')
})

Ayrıca Media sınıfı, sözlük nesnesiyle css dosyaları eklemenize izin verir:

class Media:
    css = {
        'all': ('css/admin/styles.css',)
         }

Örneğin, “first_name” sütununun her bir öğesini belirli bir renkte görüntülememiz gerekiyor. Varsayılan olarak Django, ’list_display’ içindeki her öğe için tablo sütunu oluşturur, tüm ‘’ etiketleri ‘field-’list_display_name’’ gibi css sınıfına sahip olacaktır, bizim durumumuzda ‘field_first_name’ olacaktır.

.field_first_name {
     background-color: #e6f2ff;
 }

JS veya bazı css stilleri ekleyerek diğer davranışları özelleştirmek istiyorsanız, tarayıcı hata ayıklama aracında her zaman kimlikleri ve öğe sınıflarını kontrol edebilirsiniz.

Değişim Listesi #

Aşağıdaki modele sahip basit bir “blogum” uygulamanız olduğunu varsayalım:

from django.conf import settings
from django.utils import timezone

class Article(models.Model):
    title = models.CharField(max_length=70)
    slug = models.SlugField(max_length=70, unique=True)
    author = models.ForeignKey(settings.AUTH_USER_MODEL, models.PROTECT)
    date_published = models.DateTimeField(default=timezone.now)
    is_draft = models.BooleanField(default=True)
    content = models.TextField()

Django Admin’in “değişiklik listesi”, belirli bir modelin tüm nesnelerini listeleyen sayfadır.

from django.contrib import admin
from myblog.models import Article

@admin.register(Article)
class ArticleAdmin(admin.ModelAdmin):
    pass

Varsayılan olarak, “name” nesnesini görüntülemek için modelinizin __str__() yöntemini (veya python2 kullanıyorsanız __unicode__()) kullanacaktır. Bu, geçersiz kılmadıysanız, tümü “Makale nesnesi” olarak adlandırılan bir makale listesi göreceğiniz anlamına gelir. Bu davranışı değiştirmek için ‘str()’ yöntemini ayarlayabilirsiniz:

class Article(models.Model):
    def __str__(self):
        return self.title

Şimdi, tüm makaleleriniz farklı bir ada sahip olmalı ve “Makale nesnesi"nden daha açık olmalıdır.

Ancak bu listedeki diğer verileri görüntülemek isteyebilirsiniz. Bunun için ’list_display’ kullanın:

@admin.register(Article)
class ArticleAdmin(admin.ModelAdmin):
    list_display = ['__str__', 'author', 'date_published', 'is_draft']

“list_display”, model alanları ve özellikleriyle sınırlı değildir. aynı zamanda ‘ModelAdmin’inizin bir yöntemi olabilir:

from django.forms.utils import flatatt
from django.urls import reverse
from django.utils.html import format_html

@admin.register(Article)
class ArticleAdmin(admin.ModelAdmin):
    list_display = ['title', 'author_link', 'date_published', 'is_draft']
    
    def author_link(self, obj):
        author = obj.author
        opts = author._meta
        route = '{}_{}_change'.format(opts.app_label, opts.model_name)
        author_edit_url = reverse(route, args=[author.pk])
        return format_html(
            '<a{}>{}</a>', flatatt({'href': author_edit_url}), author.first_name)

    # Set the column name in the change list
    author_link.short_description = "Author"
    # Set the field to use when ordering using this column
    author_link.admin_order_field = 'author__firstname'
