Kurulum #

settings.py #

from django.utils.translation import ugettext_lazy as _

USE_I18N = True  # Enable Internationalization
LANGUAGE_CODE = 'en'  # Language in which original texts are written
LANGUAGES = [  # Available languages
    ('en', _("English")),
    ('de', _("German")),
    ('fr', _("French")),
]

# Make sure the LocaleMiddleware is included, AFTER SessionMiddleware
# and BEFORE middlewares using internationalization (such as CommonMiddleware)
MIDDLEWARE_CLASSES = [
   'django.contrib.sessions.middleware.SessionMiddleware',
   'django.middleware.locale.LocaleMiddleware',
   'django.middleware.common.CommonMiddleware',
]

Dizeleri çevrilebilir olarak işaretleme #

Çevirideki ilk adım, dizeleri çevrilebilir olarak işaretlemektir. Bu onları ‘gettext’ işlevlerinden birinden geçirmektedir (Bkz. Sözdizimi bölümü). Örneğin, işte örnek bir model tanımı:

from django.utils.translation import ugettext_lazy as _
# It is common to import gettext as the shortcut `_` as it is often used
# several times in the same file.

class Child(models.Model):

    class Meta:
        verbose_name = _("child")
        verbose_name_plural = _("children")

    first_name = models.CharField(max_length=30, verbose_name=_("first name"))
    last_name = models.CharField(max_length=30, verbose_name=_("last name"))
    age = models.PositiveSmallIntegerField(verbose_name=_("age"))

‘_()’ içine alınmış tüm dizeler artık çevrilebilir olarak işaretlenmiştir. Yazdırıldıklarında, seçilen dil ne olursa olsun (henüz çeviri olmadığından) her zaman kapsüllenmiş dize olarak görüntülenecektir.

Dizeleri çevirme #

Bu örnek, çeviriye başlamak için yeterlidir. Çoğu zaman, projenizin muhtemel uluslararasılaşmasını öngörmek için yalnızca dizeleri çevrilebilir olarak işaretlemek isteyeceksiniz. Böylece, bu [başka bir örnekte][2] ele alınmıştır.

[2]: https://www.wikiod.com/tr/django/uluslararaslasma#Dizeleri çevirme

Tembel ve Tembel Olmayan çeviri #

Tembel olmayan çeviri kullanıldığında, dizeler hemen çevrilir.

>>> from django.utils.translation import activate, ugettext as _
>>> month = _("June")
>>> month
'June'
>>> activate('fr')
>>> _("June")
'juin'
>>> activate('de')
>>> _("June")
'Juni'
>>> month
'June'

Tembellik kullanıldığında, çeviri yalnızca gerçekten kullanıldığında gerçekleşir.

>>> from django.utils.translation import activate, ugettext_lazy as _
>>> month = _("June")
>>> month
<django.utils.functional.lazy.<locals>.__proxy__ object at 0x7f61cb805780>
>>> str(month)
'June'
>>> activate('fr')
>>> month
<django.utils.functional.lazy.<locals>.__proxy__ object at 0x7f61cb805780>
>>> "month: {}".format(month)
'month: juin'
>>> "month: %s" % month
'month: Juni'

Aşağıdaki durumlarda tembel çeviri kullanmanız gerekir:

  • _("bazı dizeler") değerlendirilirken çeviri etkinleştirilmeyebilir (dil seçilmedi)
  • Bazı dizeler yalnızca başlangıçta değerlendirilebilir (örn. model ve form alanları tanımları gibi sınıf özniteliklerinde)

Şablonlarda çeviri #

Şablonlarda çeviriyi etkinleştirmek için ‘i18n’ kitaplığını yüklemelisiniz.

{% load i18n %}

Temel çeviri trans şablon etiketi ile yapılır.

{% trans "Some translatable text" %}
{# equivalent to python `ugettext("Some translatable text")` #}

“trans” şablon etiketi bağlamı destekler:

{% trans "May" context "month" %}
{# equivalent to python `pgettext("May", "month")` #}

Çeviri dizginize yer tutucuları dahil etmek için aşağıdaki gibi:

_("My name is {first_name} {last_name}").format(first_name="John", last_name="Doe")

blocktrans şablon etiketini kullanmanız gerekecek:

{% blocktrans with first_name="John" last_name="Doe" %}
  My name is {{ first_name }} {{ last_name }}
{% endblocktrans %}

Elbette "John" ve "Doe" yerine değişkenlere ve filtrelere sahip olabilirsiniz:

{% blocktrans with first_name=user.first_name last_name=user.last_name|title %}
  My name is {{ first_name }} {{ last_name }}
{% endblocktrans %}

“first_name” ve “last_name” zaten bağlamınızdaysa, “with” yan tümcesini bile atlayabilirsiniz:

{% blocktrans %}My name is {{ first_name }} {{ last_name }}{% endblocktrans %}

Ancak, yalnızca “üst düzey” bağlam değişkenleri kullanılabilir. Bu çalışmayacak:

{% blocktrans %}
    My name is {{ user.first_name }} {{ user.last_name }}
{% endblocktrans %}

Bunun başlıca nedeni, değişken adının çeviri dosyalarında yer tutucu olarak kullanılmasıdır.

‘blocktrans’ şablon etiketi de çoğullaştırmayı kabul eder.

{% blocktrans count nb=users|length }}
    There is {{ nb }} user.
{% plural %}
    There are {{ nb }} users.
{% endblocktrans %}

Son olarak, ‘i18n’ kitaplığından bağımsız olarak, _("") sözdizimini kullanarak çevrilebilir dizeleri şablon etiketlerine iletebilirsiniz.

{{ site_name|default:_("It works!") }}
{% firstof var1 var2 _("translatable fallback") %}

Bu, bir işlev çağrısı sözdizimini taklit eden bazı sihirli yerleşik Django şablon sistemidir, ancak bu bir işlev çağrısı değildir. “(“Çalışıyor!”)”, “varsayılan” şablon etiketine bir dize olarak “(“Çalışır!”)”’ iletilir ve daha sonra çevrilebilir bir dize ayrıştırılır, tıpkı “name"nin şu şekilde ayrıştırılacağı gibi bir değişken ve "name" bir dizge olarak ayrıştırılır.

Dizeleri çevirme #

Dizeleri çevirmek için çeviri dosyaları oluşturmanız gerekir. Bunu yapmak için Django, ‘makemessages’ yönetim komutuyla birlikte gelir.

$ django-admin makemessages -l fr
processing locale fr

Yukarıdaki komut, yüklü uygulamalarınız içinde çevrilebilir olarak işaretlenmiş tüm dizeleri keşfedecek ve Fransızca çeviri için her uygulama için bir dil dosyası oluşturacaktır. Örneğin, çevrilebilir dizeler içeren yalnızca bir “myapp” uygulamanız varsa, bu “myapp/locale/fr/LC_MESSAGES/django.po” dosyasını oluşturacaktır. Bu dosya aşağıdaki gibi görünebilir:

# SOME DESCRIPTIVE TITLE
# Copyright (C) YEAR THE PACKAGE'S COPYRIGHT HOLDER
# This file is distributed under the same license as the PACKAGE package.
# FIRST AUTHOR <EMAIL@ADDRESS>, YEAR
#
#, fuzzy
msgid ""
msgstr ""
"Project-Id-Version: PACKAGE VERSION\n"
"Report-Msgid-Bugs-To: \n"
"POT-Creation-Date: 2016-07-24 14:01+0200\n"
"PO-Revision-Date: YEAR-MO-DA HO:MI+ZONE\n"
"Last-Translator: FULL NAME <EMAIL@ADDRESS>\n"
"Language-Team: LANGUAGE <LL@li.org>\n"
"Language: \n"
"MIME-Version: 1.0\n"
"Content-Type: text/plain; charset=UTF-8\n"
"Content-Transfer-Encoding: 8bit\n"

#: myapp/models.py:22
msgid "user"
msgstr ""

#: myapp/models.py:39
msgid "A user already exists with this email address."
msgstr ""

#: myapp/templates/myapp/register.html:155
#, python-format
msgid ""
"By signing up, you accept our <a href=\"%(terms_url)s\" "
"target=_blank>Terms of services</a>."
msgstr ""

Önce yer tutucuları doldurmanız gerekecek (büyük harflerle vurgulanmıştır). Ardından dizeleri çevirin. msgid, kodunuzda çevrilebilir olarak işaretlenmiş dizedir. msgstr, dizenin çevirisini hemen yukarıya yazmanız gereken yerdir.

Bir dize yer tutucular içerdiğinde, bunları da çevirinize dahil etmeniz gerekir. Örneğin, en son mesajı aşağıdaki gibi çevireceksiniz:

#: myapp/templates/myapp/register.html:155
#, python-format
msgid ""
"By signing up, you accept our <a href=\"%(terms_url)s\" "
"target=_blank>Terms of services</a>."
msgstr ""
"En vous inscrivant, vous acceptez nos <a href=\"%(terms_url)s\" "
"target=_blank>Conditions d'utilisation</a>"

Çeviri dosyanız tamamlandığında, .po dosyalarını .mo dosyalarına derlemeniz gerekecektir. Bu, ‘derleme iletileri’ yönetim komutu çağrılarak yapılır:

$ django-admin compilemessages

İşte bu, şimdi çeviriler mevcut.

Kodunuzda değişiklik yaptığınızda çeviri dosyalarınızı güncellemek için django-admin makemessages -l fr komutunu yeniden çalıştırabilirsiniz. Bu, mevcut çevirilerinizi koruyarak ve yenilerini ekleyerek .po dosyalarını güncelleyecektir. Silinen dizeler yine de yorumlarda mevcut olacaktır. Tüm diller için “.po” dosyalarını güncellemek için, “django-admin makemessages -a” komutunu çalıştırın. .po dosyalarınız güncellendiğinde, .mo dosyaları oluşturmak için tekrar django-admin compilemessages çalıştırmayı unutmayın.

Noop kullanım örneği #

(u)gettext_noop, bir dizeyi gerçekten çevirmeden çevrilebilir olarak işaretlemenize olanak tanır.

Tipik bir kullanım durumu, geliştiriciler için (İngilizce olarak) bir iletiyi günlüğe kaydetmek istediğinizde ancak aynı zamanda bunu istemciye (istenen dilde) görüntülemek istediğiniz zamandır. ‘gettext’ öğesine bir değişken iletebilirsiniz, ancak içeriği, tanım başına değişken olduğu için çevrilebilir bir dize olarak keşfedilmez..

# THIS WILL NOT WORK AS EXPECTED
import logging 
from django.contrib import messages

logger = logging.getLogger(__name__)

error_message = "Oops, something went wrong!"
logger.error(error_message)
messages.error(request, _(error_message))

Hata mesajı .po dosyasında görünmeyecek ve manuel olarak eklemek için var olduğunu hatırlamanız gerekecek. Bunu düzeltmek için gettext_noop ​​kullanabilirsiniz.

error_message = ugettext_noop("Oops, something went wrong!")
logger.error(error_message)
messages.error(request, _(error_message))

Şimdi "Hata, bir şeyler ters gitti!" dizgisi keşfedilecek ve oluşturulduğunda .po dosyasında mevcut olacak. Ve hata, geliştiriciler için İngilizce olarak kaydedilmeye devam edecek.

Ortak tuzaklar #

bulanık çeviriler #

Bazen “makemessages”, çeviri için bulduğu dizenin zaten var olan çeviriye biraz benzer olduğunu düşünebilir. ‘.po’ dosyasında bunun gibi özel bir ‘bulanık’ yorumla işaretlendiğinde:

#: templates/randa/map.html:91
#, fuzzy
msgid "Country"
msgstr "Länderinfo"

Çeviri doğru olsa veya onu düzeltmek için güncellemiş olsanız bile, ‘bulanık’ yorum satırını kaldırmadığınız sürece projenizi çevirmek için kullanılmayacaktır.

Çok satırlı dizeler #

‘makemessages’, düz metinden python koduna kadar çeşitli biçimlerdeki dosyaları ayrıştırır ve bu biçimlerde çok satırlı dizelere sahip olmak için olası her kuralı takip edecek şekilde tasarlanmamıştır. Çoğu zaman tek satırlık dizelerle gayet iyi çalışır, ancak bunun gibi bir yapınız varsa:

translation = _("firstline"
"secondline"
"thirdline")

Çeviri için yalnızca “ilk satırı” alır. Bunun çözümü, mümkün olduğunda çok satırlı dizeleri kullanmaktan kaçınmaktır.
