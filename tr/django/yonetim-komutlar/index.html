Yönetim komutları, Django projenizde veya temel alınan veritabanında eylemler gerçekleştirebilen güçlü ve esnek komut dosyalarıdır. Çeşitli varsayılan komutlara ek olarak, kendinizinkini de yazmak mümkündür!

Normal Python betikleriyle karşılaştırıldığında, yönetim komut çerçevesini kullanmak, sahne arkasında bazı sıkıcı kurulum işlerinin sizin için otomatik olarak yapıldığı anlamına gelir.

Yönetim komutları şuradan çağrılabilir:

  • django-admin <komut> [seçenekler]
  • python -m Django <komut> [seçenekler]
  • python manager.py <komut> [seçenekler]
  • ‘./manage.py [seçenekler]’, Manage.py’nin yürütme izinlerine sahipse (chmod +x Manage.py)

Yönetim komutlarını Cron ile kullanmak için:

*/10 * * * * pythonuser /var/www/dev/env/bin/python /var/www/dev/manage.py <command> [options] > /dev/null

Yönetim Komutu Oluşturma ve Çalıştırma #

Django’da komut satırı veya diğer hizmetleri (kullanıcı/isteğin kullanılmadığı durumlarda) kullanarak eylemler gerçekleştirmek için ‘yönetim komutlarını’ kullanabilirsiniz.

Django modülleri gerektiğinde alınabilir.

Her komut için ayrı bir dosyanın oluşturulması gerekir: myapp/management/commands/my_command.py
(“yönetim” ve “komutlar” dizinleri boş bir __init__.py dosyasına sahip olmalıdır)

from django.core.management.base import BaseCommand, CommandError

# import additional classes/modules as needed

    help = 'My custom django management command'

       parser.add_argument('author' , nargs='+', type=str)

    def handle(self, *args, **options):

Burada BaseCommand veya alt sınıflarından birini genişleten sınıf adı Command zorunludur.

Yönetim komutunun adı, onu içeren dosyanın adıdır. Yukarıdaki örnekteki komutu çalıştırmak için proje dizininizde aşağıdakini kullanın:

python manage.py my_command

Bir komutu başlatmanın birkaç saniye sürebileceğini unutmayın (çünkü modüllerin içe aktarılması). Bu nedenle bazı durumlarda oluşturulması tavsiye edilir. “yönetim komutları” yerine “arka plan programı” işlemleri.

Yerleşik Yönetim Komutları #

Django, “python manage.py [komut]” kullanarak veya Manage.py +x (yürütülebilir) haklara sahip olduğunda “./manage.py [command]” kullanarak bir dizi yerleşik yönetim komutuyla birlikte gelir. Aşağıdakiler en sık kullanılanlardan bazılarıdır:

Mevcut tüm komutların bir listesini alın

./manage.py help

Django sunucunuzu localhost:8000; üzerinde çalıştırın. yerel testler için gerekli

./manage.py runserver

Projenizin Django ayarları önceden yüklenmiş olarak bir python (veya kuruluysa ipython) konsolu çalıştırın (bunu yapmadan projenizin bölümlerine bir python terminalinde erişmeye çalışmak başarısız olur).

./manage.py shell

Modellerinizde yaptığınız değişiklikleri temel alan yeni bir veritabanı geçiş dosyası oluşturun. Bkz. Taşımalar

./manage.py makemigrations

Uygulanmamış tüm geçişleri geçerli veritabanına uygulayın.

./manage.py migrate

Projenizin test takımını çalıştırın. Bkz. Birim Testi

./manage.py test

Projenizin tüm statik dosyalarını alın ve üretimde sunulabilmesi için STATIC_ROOT içinde belirtilen klasöre koyun.

./manage.py collectstatic

Süper kullanıcı oluşturmaya izin ver.

./manage.py createsuperuser

Belirtilen bir kullanıcının parolasını değiştirin.

./manage.py changepassword username

Manage.py yerine Django-admin kullanma #

‘manage.py’den kurtulabilir ve bunun yerine ‘django-admin’ komutunu kullanabilirsiniz. Bunu yapmak için, “manage.py"nin yaptığını manuel olarak yapmanız gerekir:

  • Proje yolunuzu PYTHONPATH’inize ekleyin

  • DJANGO_SETTINGS_MODULE’u ayarlayın

    export PYTHONPATH="/home/me/path/to/your_project” export DJANGO_SETTINGS_MODULE=“your_project.settings”

Bu, özellikle bu ortam değişkenlerini ‘postactivate’ komut dosyasında ayarlayabileceğiniz bir virtualenv‘de kullanışlıdır.

1: https://www.wikiod.com/tr/django/djangoyu-kullanmaya-baslama#Sanal çevre

django-admin komutu, dosya sisteminizde nerede olursanız olun kullanılabilir olma avantajına sahiptir.

Mevcut komutların listesini al #

Kullanılabilir komutların listesini aşağıdaki şekilde alabilirsiniz:

>>> python manage.py help

Herhangi bir komutu anlamıyorsanız veya isteğe bağlı argümanlar arıyorsanız, bunun gibi -h argümanını kullanabilirsiniz.

>>> python manage.py command_name -h
