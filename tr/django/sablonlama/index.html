Şablonlarda hassas yöntemlerin çağrılmasını önleyin #

Bir nesne şablon bağlamına maruz kaldığında, onun bağımsız değişkensiz yöntemleri kullanılabilir. Bu işlevler “alıcı” olduğunda kullanışlıdır. Ancak bu yöntemlerin bazı verileri değiştirmesi veya bazı yan etkileri olması tehlikeli olabilir. Şablon yazarına büyük olasılıkla güvenseniz bile, bir işlevin yan etkilerinin farkında olmayabilir veya yanlışlıkla yanlış özelliği çağırdığını düşünebilir.

Aşağıdaki model göz önüne alındığında:

class Foobar(models.Model):
    points_credit = models.IntegerField()

    def credit_points(self, nb_points=1):
        """Credit points and return the new points credit value."""
        self.points_credit = F('points_credit') + nb_points
        self.save(update_fields=['points_credit'])
        return self.points_credit

Bunu yanlışlıkla bir şablona yazarsanız:

 You have {{ foobar.credit_points }} points!

Bu, şablon her çağrıldığında nokta sayısını artıracaktır. Ve bunu fark etmeyebilirsiniz bile.

Bunu önlemek için, yan etkileri olan yöntemlerde ‘alters_data’ özniteliğini ‘True’ olarak ayarlamalısınız. Bu, onları bir şablondan çağırmayı imkansız hale getirecektir.

def credit_points(self, nb_points=1):
    """Credit points and return the new points credit value."""
    self.points_credit = F('points_credit') + nb_points
    self.save(update_fields=['points_credit'])
    return self.points_credit
credit_points.alters_data = True

İşlev Tabanlı Görünümlerde Şablon Oluşturma #

İşlev tabanlı bir görünümde bir şablonu aşağıdaki gibi kullanabilirsiniz:

from django.shortcuts import render

def view(request):
    return render(request, "template.html")

Şablon değişkenlerini kullanmak istiyorsanız, bunu aşağıdaki gibi yapabilirsiniz:

from django.shortcuts import render

def view(request):
    context = {"var1": True, "var2": "foo"}
    return render(request, "template.html", context=context)

Ardından, template.html içinde değişkenlerinize şu şekilde başvurabilirsiniz:

<html>
{% if var1 %}
    <h1>{{ var2 }}</h1>
{% endif %}
</html>

Şablon filtreleri #

Django şablon sistemi, içeriği belirli bir şekilde işlemek için şablonun içinde işlevler olan yerleşik etiketler ve filtreler içerir. Borularla birden çok filtre belirtilebilir ve filtrelerin değişken sözdiziminde olduğu gibi argümanları olabilir.

{{ "MAINROAD 3222"|lower }}    # mainroad 3222
{{ 10|add:15}}                 # 25
{{ "super"|add:"glue" }}       # superglue
{{ "A7"|add:"00" }}            # A700
{{ myDate | date:"D d M Y"}}   # Wed 20 Jul 2016

Kullanılabilir yerleşik filtrelerin listesi https://docs.djangoproject.com/en/dev/ref/templates/builtins/#ref-templates-builtins-filters adresinde bulunabilir.

Özel filtreler oluşturma

Kendi şablon filtrelerinizi eklemek için uygulama klasörünüzün içinde “templatetags” adlı bir klasör oluşturun. Ardından bir __init__.py ve filtreleri içerecek dosyanızı ekleyin:

#/myapp/templatetags/filters.py
from django import template

register = template.Library()

@register.filter(name='tostring')
def to_string(value):
    return str(value)

Filtreyi gerçekten kullanmak için şablonunuza yüklemeniz gerekir:

#templates/mytemplate.html
{% load filters %}
{% if customer_id|tostring = customer %} Welcome back {% endif%}

Hileler

Filtreler ilk bakışta basit görünse de, bazı şık şeyler yapmanızı sağlar:

{% for x in ""|ljust:"20" %}Hello World!{% endfor %}    # Hello World!Hello World!Hel...    
{{ user.name.split|join:"_" }} ## replaces whitespace with '_'

Daha fazla bilgi için ayrıca şablon etiketleri bölümüne bakın.

Sınıf Tabanlı Görünümlerde Şablon Oluşturma #

Özel bir değişkendeki bir şablona veri iletebilirsiniz.

“views.py” dosyanızda:

from django.views.generic import TemplateView
from MyProject.myapp.models import Item

class ItemView(TemplateView):
    template_name = "item.html"

    def items(self):
        """ Get all Items """
        return Item.objects.all()

    def certain_items(self):
        """ Get certain Items """
        return Item.objects.filter(model_field="certain")
    
    def categories(self):
        """ Get categories related to this Item """
        return Item.objects.get(slug=self.kwargs['slug']).categories.all()

“item.html” dosyanızdaki basit bir liste:

{% for item in view.items %}
<ul>        
    <li>{{ item }}</li>
</ul>
{% endfor %}

Ayrıca verilerin ek özelliklerini de alabilirsiniz.

“Öğe” modelinizin bir “ad” alanı olduğunu varsayarsak:

{% for item in view.certain_items %}
<ul>        
    <li>{{ item.name }}</li>
</ul>
{% endfor %}

Değişkenler #

Görünüm bağlamınızda sağladığınız değişkenlere çift ayraç gösterimi kullanılarak erişilebilir:

“views.py” dosyanızda:

class UserView(TemplateView):
  """ Supply the request user object to the template """

  template_name = "user.html"

  def get_context_data(self, **kwargs):
    context = super(UserView, self).get_context_data(**kwargs)
    context.update(user=self.request.user)
    return context

“user.html” içinde:

<h1>{{ user.username }}</h1>

<div class="email">{{ user.email }}</div>

Nokta gösterimi aşağıdakilere erişecektir:

  • nesnenin özellikleri, örn. user.username, {{ user.username }} olacak
  • sözlük aramaları, ör. request.GET["search"], {{ request.GET.search }} olacaktır
  • argüman içermeyen yöntemler, örn. users.count(), {{ user.count }} olacak

Şablon değişkenleri, argüman alan yöntemlere erişemez.

Değişkenler ayrıca test edilebilir ve döngüye alınabilir:

{% if user.is_authenticated %}
  {% for item in menu %}
    <li><a href="{{ item.url }}">{{ item.name }}</a></li>
  {% endfor %}
{% else %}
  <li><a href="{% url 'login' %}">Login</a>
{% endif %}

URL’lere ‘{% url ’name’ %}’ biçimi kullanılarak erişilir, burada adlar ‘urls.py’nizdeki adlara karşılık gelir.

{% url 'login' %} - Muhtemelen /accounts/login/ olarak işlenecek {% url 'user_profile' user.id %} - URL’ler için argümanlar sırayla sağlanır {% url next %} - URL’ler değişken olabilir

{% extensions %} , {% include %} ve {% block %} kullanımı #

özet #

  • {% extensions %}: bu, geçerli şablonun ebeveyni olarak bir argüman olarak verilen şablonu bildirir. Kullanım: {% 'parent_template.html' %} öğesini genişletir.

  • {% block %}{% endblock %}: Bu, şablonlarınızdaki bölümleri tanımlamak için kullanılır, böylece başka bir şablon bunu genişletirse, içine yazılan html kodunun yerini alabilir. BT. Bloklar adlarıyla tanımlanır. Kullanım: {% blok içeriği %} <html_code> {% endblock %}.

  • {% include %}: bu, mevcut şablona bir şablon ekleyecektir. Dahil edilen şablonun isteğin bağlamını alacağını ve buna özel değişkenler de verebileceğinizi unutmayın. Temel kullanım: {% include 'şablon_adı.html' %}, değişkenlerle kullanım: {% include 'şablon_adı.html' ile değişken='değer' değişken2=8 %}

Kılavuz #

Tüm kodlar için ortak düzenlere sahip ön uç yan kodunuzu oluşturduğunuzu ve kodu her şablon için tekrarlamak istemediğinizi varsayalım. Django, bunu yapmak için size yerleşik etiketler sağlar. Aynı düzeni paylaşan 3 şablona sahip bir blog web sitemiz olduğunu varsayalım:

project_directory
    ..
    templates
      front-page.html
      blogs.html
      blog-detail.html

  1. base.html dosyasını tanımlayın,

    {% block content %} {% endblock %}

  2. blog.html şeklinde genişletin,

    {% extends ‘base.html’ %}

    {% block content %} # write your blog related code here {% endblock %}

    None of the code written here will be added to the template #

Burada temel düzeni genişlettik, böylece HTML düzeni artık blog.html dosyasında kullanılabilir.{ % blok %} kavramı, tüm öğeleri içeren bir temel “iskelet” şablonu oluşturmanıza izin veren şablon mirasıdır. sitenizin ortak öğeleri ve alt şablonların geçersiz kılabileceği blokları tanımlar.

  1. Şimdi 3 şablonunuzun da bazı popüler gönderileri tanımlayan aynı HTML div’e sahip olduğunu varsayalım. 3 kez yazılmak yerine yeni bir posts.html şablonu oluşturun.

blog.html

{% extends 'base.html' %}

{% block content %}
    # write your blog related code here
    {% include 'posts.html' %} # includes posts.html in blog.html file without passing any data
    <!-- or -->
    {% include 'posts.html' with posts=postdata %} # includes posts.html in blog.html file with passing posts data which is context of view function returns.
{% endblock %}
