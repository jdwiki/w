Bir Projeye Başlamak #

Django, Python tabanlı bir web geliştirme çerçevesidir. Django 1.11 (en son kararlı sürüm) Python 2.7, 3.4, 3.5 veya 3.6 yüklenmesini gerektirir. ‘pip’in mevcut olduğunu varsayarsak, kurulum aşağıdaki komutu çalıştırmak kadar basittir. Aşağıda gösterildiği gibi sürümü çıkarmanın Django’nun en son sürümünü yükleyeceğini unutmayın:

$ pip install django

Django’nun belirli bir sürümünü yüklemek için, sürümün Django 1.10.5 olduğunu varsayalım, aşağıdaki komutu çalıştırın:

$ pip install django==1.10.5

Django kullanılarak oluşturulan web uygulamaları, bir Django projesi içinde bulunmalıdır. Geçerli dizinde yeni bir proje başlatmak için django-admin komutunu kullanabilirsiniz:

$ django-admin startproject myproject

burada “projem”, projeyi benzersiz şekilde tanımlayan bir addır ve sayılar, harfler ve alt çizgi içerebilir.

Bu, aşağıdaki proje yapısını yaratacaktır:

myproject/
    manage.py
    myproject/
        __init__.py
        settings.py
        urls.py
        wsgi.py

Uygulamayı çalıştırmak için geliştirme sunucusunu başlatın

$ cd myproject
$ python manage.py runserver

Artık sunucu çalıştığına göre, web tarayıcınızla http://127.0.0.1:8000/ adresini ziyaret edin. Aşağıdaki sayfayı göreceksiniz:

Varsayılan olarak, “runserver” komutu, geliştirme sunucusunu “8000” bağlantı noktasındaki dahili IP’de başlatır. Siz kodunuzda değişiklik yaptığınızda bu sunucu otomatik olarak yeniden başlatılacaktır. Ancak yeni dosyalar eklemeniz durumunda sunucuyu manuel olarak yeniden başlatmanız gerekecektir.

Sunucunun bağlantı noktasını değiştirmek istiyorsanız, bunu komut satırı argümanı olarak iletin.

$ python manage.py runserver 8080

Sunucunun IP’sini değiştirmek istiyorsanız, bağlantı noktasıyla birlikte iletin.

$ python manage.py runserver 0.0.0.0:8000

“runserver"ın yalnızca hata ayıklama yapıları ve yerel testler için olduğunu unutmayın. Özel sunucu programları (Apache gibi) üretimde her zaman kullanılmalıdır.

Django Uygulaması Ekleme

Bir Django projesi genellikle birden çok “uygulama” içerir. Bu, projenizi daha küçük, bakımı yapılabilir modüller halinde yapılandırmanın bir yoludur. Bir uygulama oluşturmak için proje klasörünüze gidin (“manage.py"nin olduğu yerde) ve “startapp” komutunu çalıştırın (myapp‘ı istediğiniz şekilde değiştirin):

python manage.py startapp myapp

Bu, myapp klasörünü ve sizin için “models.py” ve “views.py” gibi bazı gerekli dosyaları oluşturacaktır.

Django’nun uygulamam‘dan haberdar olmasını sağlamak için, onu settings.py dosyanıza ekleyin:

# myproject/settings.py

# Application definition
INSTALLED_APPS = [
    ...
    'myapp',
]

Bir Django projesinin klasör yapısı tercihinize göre değiştirilebilir. Bazen proje klasörü, klasör adlarının tekrarlanmasını önlemek için /src olarak yeniden adlandırılır. Tipik bir klasör yapısı şöyle görünür:

Sanal çevre #

Kesinlikle gerekli olmasa da, projenize “sanal bir ortamda” başlamanız şiddetle tavsiye edilir. Sanal ortam, belirli bir Python sürümünü ve bir dizi modülü (bağımlılıkları) içeren ve işletim sisteminin yerel Python’unu veya aynı bilgisayardaki diğer projeleri etkilemeyen bir kapsayıcıdır (bir dizin).

Üzerinde çalıştığınız her proje için farklı bir sanal ortam kurarak, çeşitli Django projeleri Python’un farklı sürümlerinde çalışabilir ve çakışma riski olmadan kendi bağımlılık kümelerini koruyabilir.

Python 3.3+ #

Python 3.3+, genellikle “pyvenv” olarak adlandırabileceğiniz standart bir “venv” modülü içerir. ‘pyvenv’ komutunun kullanılamadığı ortamlarda, modülü doğrudan ‘python3 -m venv’ olarak çağırarak aynı işlevselliğe erişebilirsiniz.

Sanal ortamı oluşturmak için:

$ pyvenv <env-folder>
# Or, if pyvenv is not available
$ python3 -m venv <env-folder>

piton 2 #

Python 2 kullanıyorsanız, önce onu pip’ten ayrı bir modül olarak kurabilirsiniz:

$ pip install virtualenv

Ardından bunun yerine ‘virtualenv’ komutunu kullanarak ortamı oluşturun:

$ virtualenv <env-folder>

Etkinleştir (herhangi bir sürüm) #

Sanal ortam şimdi kuruldu. Kullanmak için kullanmak istediğiniz terminalde etkinleştirilmiş olmalıdır.

Sanal ortamı ’etkinleştirmek’ için (herhangi bir Python sürümü)

Linux gibi:

$ source <env-folder>/bin/activate

Windows gibi:

<env-folder>\Scripts\activate.bat

Bu, isteminizi sanal ortamın etkin olduğunu gösterecek şekilde değiştirir. (<env-klasörü>) $

Şu andan itibaren, pip kullanılarak kurulan her şey sistem genelinde değil sanal env klasörünüze yüklenecek.

Sanal ortamdan ayrılmak için “deactivate” komutunu kullanın:

(<env-folder>) $ deactivate

Alternatif olarak: virtualenvwrapper kullanın #

Ayrıca, sanalenv oluşturma ve etkinleştirmeyi çok kullanışlı hale getiren ve kodunuzdan ayırmayı sağlayan virtualenvwrapper kullanmayı da düşünebilirsiniz:

# Create a virtualenv
mkvirtualenv my_virtualenv

# Activate a virtualenv
workon my_virtualenv

# Deactivate the current virtualenv
deactivate

Alternatif olarak: pyenv + pyenv-viritualenv kullanın #

Birden çok Python sürümünü kullanmanız gereken ortamlarda, sanalenv’den pyenv-virtualenv ile birlikte yararlanabilirsiniz:

# Create a virtualenv for specific Python version
pyenv virtualenv 2.7.10 my-virtual-env-2.7.10

# Create a vritualenv for active python verion
pyenv virtualenv venv34

# Activate, deactivate virtualenv
pyenv activate <name>
pyenv deactivate

Virtualenv’leri kullanırken, ‘postactivate’ komut dosyasında ‘PYTHONPATH’ ve ‘DJANGO_SETTINGS_MODULE’ ayarlarınızı yapmak genellikle yararlıdır.

#!/bin/sh
# This hook is sourced after this virtualenv is activated

# Set PYTHONPATH to isolate the virtualenv so that only modules installed
# in the virtualenv are available
export PYTHONPATH="/home/me/path/to/your/project_root:$VIRTUAL_ENV/lib/python3.4"

# Set DJANGO_SETTINGS_MODULE if you don't use the default `myproject.settings`
# or if you use `django-admin` rather than `manage.py`
export DJANGO_SETTINGS_MODULE="myproject.settings.dev"

Proje Yolunuzu Belirleyin #

Proje yolunuzu, temel <env-klasörü> içinde bulunan özel bir .project dosyası içinde ayarlamak da genellikle yararlıdır. Bunu yaparken sanal ortamınızı her etkinleştirdiğinizde, aktif dizini belirtilen yola değiştirecektir.

<env-folder>/.project adında yeni bir dosya oluşturun. Dosyanın içeriği YALNIZCA proje dizininin yolu olmalıdır.

/path/to/project/directory

Şimdi, sanal ortamınızı başlatın (‘source /bin/activate’ veya ‘workon my_virtualenv’ kullanarak) ve terminaliniz dizinleri ‘/path/to/project/directory’ olarak değiştirecektir.

Django Kavramları #

Django-admin, Django ile birlikte gelen bir komut satırı aracıdır. Bir Django projesini başlatmak ve yönetmek için birkaç yararlı komutla birlikte gelir. Komut, proje dizininde olmanız gerekmemesi dışında, ./manage.py ile aynıdır. DJANGO_SETTINGS_MODULE ortam değişkeninin ayarlanması gerekiyor.

Bir Django projesi, bir Django ayarlar dosyası içeren bir Python kod tabanıdır. Django-admin startproject NAME komutuyla Django yöneticisi tarafından bir proje oluşturulabilir. Proje genellikle en üst düzeyde “manage.py” adlı bir dosyaya ve “urls.py” adlı bir kök URL dosyasına sahiptir. ‘manage.py’, ‘django-admin’in projeye özel bir versiyonudur ve o projede yönetim komutları çalıştırmanıza izin verir. Örneğin, projenizi yerel olarak çalıştırmak için python manager.py runserver kullanın. Bir proje Django uygulamalarından oluşur.

Bir Django uygulaması, bir model dosyası (varsayılan olarak “models.py”) ve uygulamaya özel URL’ler ve görünümler gibi diğer dosyaları içeren bir Python paketidir. Django-admin startapp NAME komutuyla bir uygulama oluşturulabilir (bu komut proje dizininizin içinden çalıştırılmalıdır). Bir uygulamanın bir projenin parçası olması için, “settings.py” içindeki INSTALLED_APPS listesine dahil edilmesi gerekir. Standart yapılandırmayı kullandıysanız, Django, kimlik doğrulama gibi şeyleri sizin için halledecek olan, kendi uygulamalarının önceden yüklenmiş birkaç uygulamasıyla birlikte gelir. Uygulamalar birden çok Django projesinde kullanılabilir.

Django ORM, ‘models.py’ içinde tanımlanan tüm veritabanı modellerini toplar ve bu model sınıflarına dayalı veritabanı tabloları oluşturur. Bunu yapmak için önce, settings.py içindeki DATABASES ayarını değiştirerek veritabanınızı kurun. Ardından, veritabanı modellerinizi tanımladıktan sonra, veritabanınızın şemasına dayalı oluşturmak veya güncellemek için python Manage.py makemigrations ve ardından python Manage.py Migration komutunu çalıştırın. senin modellerinde.

Tek Dosya Merhaba Dünya Örneği #

Bu örnek, Django’da Merhaba Dünya sayfası oluşturmanın minimal bir yolunu gösterir. Bu, django-admin startproject example komutunun temelde bir dizi klasör ve dosya oluşturduğunu ve projenizi çalıştırmak için mutlaka bu yapıya ihtiyacınız olmadığını anlamanıza yardımcı olacaktır.

  1. file.py adında bir dosya oluşturun.

  2. Aşağıdaki kodu kopyalayıp o dosyaya yapıştırın.

     import sys

 from django.conf import settings

 settings.configure(
     DEBUG=True,
     SECRET_KEY='thisisthesecretkey',
     ROOT_URLCONF=__name__,
     MIDDLEWARE_CLASSES=(
         'django.middleware.common.CommonMiddleware',
         'django.middleware.csrf.CsrfViewMiddleware',
         'django.middleware.clickjacking.XFrameOptionsMiddleware',
     ),
 )

 from django.conf.urls import url
 from django.http import HttpResponse

 # Your code goes below this line.

 def index(request):
     return HttpResponse('Hello, World!')

 urlpatterns = [
     url(r'^$', index),
 ]

 # Your code goes above this line

 if __name__ == "__main__":
     from django.core.management import execute_from_command_line

     execute_from_command_line(sys.argv)

  3. Terminale gidin ve dosyayı python file.py runserver komutuyla çalıştırın.

  4. Tarayıcınızı açın ve 127.0.0.1:8000’e gidin.

Tam bir merhaba dünya örneği. #

Aşama 1 Django zaten kuruluysa bu adımı atlayabilirsiniz.

pip install Django

Adım 2 Yeni bir proje oluştur

django-admin startproject hello

Bu, aşağıdaki dosyaları içeren “merhaba” adlı bir klasör oluşturacaktır:

hello/
├── hello/
│   ├── __init__.py
│   ├── settings.py
│   ├── urls.py
│   └── wsgi.py
└── manage.py

Aşama 3 “Merhaba” modülünün içinde (”init.py” dosyasını içeren klasör) “views.py” adında bir dosya oluşturun:

hello/
├── hello/
│   ├── __init__.py
│   ├── settings.py
│   ├── urls.py
│   ├── views.py  <- here
│   └── wsgi.py
└── manage.py

ve aşağıdaki içeriği koyun:

from django.http import HttpResponse

def hello(request):
    return HttpResponse('Hello, World')

Buna görünüm işlevi denir.

Adım 4 “hello/urls.py” dosyasını aşağıdaki gibi düzenleyin:

from django.conf.urls import url
from django.contrib import admin
from hello import views

urlpatterns = [
    url(r'^admin/', admin.site.urls),
    url(r'^$', views.hello)
]

bu, “hello()” görüntüleme işlevini bir URL’ye bağlar.

Adım 5 Sunucuyu başlatın.

python manage.py runserver

Adım 6

Bir tarayıcıda http://localhost:8000/ adresine gidin ve şunları göreceksiniz:

Merhaba, Dünya

Docker desteği ile dağıtım dostu proje. #

Varsayılan Django proje şablonu iyidir, ancak kodunuzu dağıtmaya başladığınızda ve örneğin devop’lar ellerini projeye koyduğunda işler karışır. Yapabileceğiniz şey, kaynak kodunuzu havuzunuzda olması gereken diğerlerinden ayırmaktır.

GitHub üzerinde kullanılabilir bir Django proje şablonu bulabilirsiniz.

Proje Yapısı #

PROJECT_ROOT
├── devel.dockerfile
├── docker-compose.yml
├── nginx
│   └── project_name.conf
├── README.md
├── setup.py
└── src
    ├── manage.py
    └── project_name
        ├── __init__.py
        └── service
            ├── __init__.py
            ├── settings
            │   ├── common.py
            │   ├── development.py
            │   ├── __init__.py
            │   └── staging.py
            ├── urls.py
            └── wsgi.py

Tüm projelerimde aynı Dockerfile’ı kullanabildiğim için her proje için ‘service’ adlı ‘service’ dizinini tutmayı seviyorum. Gereksinimlerin ve ayarların ayrımı burada zaten iyi bir şekilde belgelenmiştir: https://www.wikiod.com/tr/django/ayarlar#Birden çok gereksinim dosyası kullanma https://www.wikiod.com/tr/django/ayarlar#Birden çok ayar kullanma

Docker dosyası #

Yalnızca geliştiricilerin Docker’ı kullandığı varsayımıyla (bugünlerde her geliştirici ona güvenmiyor). Bu bir geliştirme ortamı “devel.dockerfile” olabilir:

FROM python:2.7
ENV PYTHONUNBUFFERED 1

RUN mkdir /run/service
ADD . /run/service
WORKDIR /run/service

RUN pip install -U pip
RUN pip install -I -e .[develop] --process-dependency-links

WORKDIR /run/service/src
ENTRYPOINT ["python", "manage.py"]
CMD ["runserver", "0.0.0.0:8000"]

Yalnızca gereksinimlerin eklenmesi, oluşturma sırasında Docker önbelleğinden yararlanacaktır - yalnızca gereksinim değişikliğinde yeniden oluşturmanız gerekir.

Oluştur #

Docker oluşturma, özellikle yerel olarak çalıştırılacak birden çok hizmetiniz olduğunda kullanışlıdır. “docker-compose.yml”:

version: '2'
services:
  web:
    build:
      context: .
      dockerfile: devel.dockerfile
    volumes:
      - "./src/{{ project_name }}:/run/service/src/{{ project_name }}"
      - "./media:/run/service/media"
    ports:
      - "8000:8000"
    depends_on:
      - db
  db:
    image: mysql:5.6
    environment:
      - MYSQL_ROOT_PASSWORD=root
      - MYSQL_DATABASE={{ project_name }}
  nginx:
    image: nginx
    ports:
      - "80:80"
    volumes:
      - "./nginx:/etc/nginx/conf.d"
      - "./media:/var/media"
    depends_on:
      - web

Nginx #

Geliştirme ortamınız mümkün olduğunca prod ortamına yakın olmalı, bu yüzden Nginx’i en başından kullanmayı seviyorum. Örnek bir nginx yapılandırma dosyası:

server {
    listen   80;
    client_max_body_size 4G;
    keepalive_timeout 5;

    location /media/ {
        autoindex on;
        alias /var/media/;
    }

    location / {
        proxy_pass_header Server;
        proxy_set_header Host $http_host;
        proxy_redirect off;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Scheme $scheme;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Ssl on;
        proxy_connect_timeout 600;
        proxy_read_timeout 600;
        proxy_pass http://web:8000/;
    }
}

Kullanım #

$ cd PROJECT_ROOT
$ docker-compose build web  # build the image - first-time and after requirements change
$ docker-compose up  # to run the project
$ docker-compose run --rm --service-ports --no-deps  # to run the project - and be able to use PDB
$ docker-compose run --rm --no-deps <management_command>  # to use other than runserver commands, like makemigrations
$ docker exec -ti web bash  # For accessing django container shell, using it you will be inside /run/service directory, where you can run ./manage shell, or other stuff
$ docker-compose start  # Starting docker containers
$ docker-compose stop  # Stopping docker containers
