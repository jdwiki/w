ara katman yazılımı

  • Django 1.10 ara katman yazılımının yeni stilini anlama
  • IP adresine göre filtrelemek için ara yazılım
  • İsteklere veri ekleyin
  • Genel olarak işleme istisnası

Bu sayfada

  • Django 1.10 ara katman yazılımının yeni stilini anlama
  • IP adresine göre filtrelemek için ara yazılım
  • İsteklere veri ekleyin
  • Genel olarak işleme istisnası

Django’daki ara yazılım, kodun yanıt/istek işlemeye bağlanmasına ve Django’nun girdi veya çıktısını değiştirmesine izin veren bir çerçevedir.

Middleware, yürütmeye dahil edilmeden önce settings.py MIDDLEWARE_CLASSES listenize eklenmelidir. Django’nun yeni bir proje oluştururken sağladığı varsayılan liste aşağıdaki gibidir:

MIDDLEWARE_CLASSES = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.common.CommonMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.auth.middleware.SessionAuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
    'django.middleware.clickjacking.XFrameOptionsMiddleware',
]

Bunların tümü, her istekte sırayla çalışacak olan işlevlerdir (bir kez görünüm kodunuza ‘views.py’ içinde ulaşmadan önce ve bir kez de sürüm 1.10’dan önce ‘process_response’ geri çağrısı için ters sırada). Cross Site Request Forgery (csrf) belirtecini enjekte etmek gibi çeşitli şeyler yaparlar.

Sıra önemlidir, çünkü bazı ara katman yazılımları yeniden yönlendirme yaparsa, sonraki tüm ara katman yazılımları asla çalışmayacaktır. Veya bir ara katman yazılımı, csrf belirtecinin orada olmasını bekliyorsa, “CsrfViewMiddleware"den sonra çalışması gerekir.

Django 1.10 ara katman yazılımının yeni stilini anlama #

Django 1.10, “process_request” ve “process_response” öğelerinin birleştirildiği yeni bir ara katman yazılımı stilini tanıttı.

Bu yeni stilde, bir ara katman yazılımı, başka bir çağrılabilir döndüren bir çağrılabilirdir. Aslında birincisi bir ara katman yazılımı fabrikasıdır ve ikincisi gerçek ara katman yazılımıdır.

Ara katman yazılımı fabrikası, ara katman yazılımı yığınındaki bir sonraki ara katman yazılımını veya yığının sonuna ulaşıldığında görünümün kendisini tek argüman olarak alır.

Ara katman yazılımı, isteği tek bir argüman olarak alır ve her zaman bir “HttpResponse” döndürür.

Yeni stil ara yazılımın nasıl çalıştığını göstermek için en iyi örnek, muhtemelen geriye dönük uyumlu bir ara yazılımın nasıl yapıldığını göstermektir.

class MyMiddleware:

    def __init__(self, next_layer=None):
        """We allow next_layer to be None because old-style middlewares
        won't accept any argument.
        """
        self.get_response = next_layer

    def process_request(self, request):
        """Let's handle old-style request processing here, as usual."""
        # Do something with request
        # Probably return None
        # Or return an HttpResponse in some cases

    def process_response(self, request, response):
        """Let's handle old-style response processing here, as usual."""
        # Do something with response, possibly using request.
        return response

    def __call__(self, request):
        """Handle new-style middleware here."""
        response = self.process_request(request)
        if response is None:
            # If process_request returned None, we must call the next middleware or
            # the view. Note that here, we are sure that self.get_response is not
            # None because this method is executed only in new-style middlewares.
            response = self.get_response(request)
        response = self.process_response(request, response)
        return response

IP adresine göre filtrelemek için ara yazılım #

Birincisi: Yol yapısı

Elinizde yoksa, uygulamanızın içinde aşağıdaki yapıya göre middleware klasörünü oluşturmanız gerekir:

yourproject/yourapp/middleware

Klasör ara yazılımı settings.py, urls, templates… ile aynı klasöre yerleştirilmelidir…

Önemli: Uygulamanızın bu klasörü tanıması için ara katman yazılımı klasörü içinde init.py boş dosyasını oluşturmayı unutmayın

Ara katman yazılımı sınıflarınızı içeren ayrı bir klasöre sahip olmak yerine, işlevlerinizi tek bir dosyaya koymak da mümkündür, projeniz/uygulamanız/middleware.py.

İkinci: Ara katman yazılımını oluşturun

Şimdi özel ara katman yazılımımız için bir dosya oluşturmalıyız. Bu örnekte, kullanıcıları IP adreslerine göre filtreleyen bir ara katman yazılımı istediğimizi varsayalım, filter_ip_middleware.py adında bir dosya oluşturuyoruz:

#yourproject/yourapp/middleware/filter_ip_middleware.py
from django.core.exceptions import PermissionDenied    

class FilterIPMiddleware(object):
    # Check if client IP address is allowed
    def process_request(self, request):
        allowed_ips = ['192.168.1.1', '123.123.123.123', etc...] # Authorized ip's
        ip = request.META.get('REMOTE_ADDR') # Get client IP address
        if ip not in allowed_ips:
            raise PermissionDenied # If user is not allowed raise Error

       # If IP address is allowed we don't do anything
       return None

Üçüncü: Ara yazılımı ‘settings.py’ dosyamıza ekleyin

settings.py içinde MIDDLEWARE_CLASSES aramamız gerekiyor ve oraya ara katman yazılımımızı eklememiz gerekiyor (Son konuma ekleyin). Şunun gibi olmalı:

MIDDLEWARE_CLASSES = (
    'django.middleware.common.CommonMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
     # Above are Django standard middlewares

     # Now we add here our custom middleware
     'yourapp.middleware.filter_ip_middleware.FilterIPMiddleware'
)

Bitti! Artık her müşteriden gelen her istek, özel ara yazılımınızı arayacak ve özel kodunuzu işleyecektir!

İsteklere veri ekleyin #

Django, görünüm içinde kullanım isteklerine ek veri eklemeyi gerçekten kolaylaştırır. Örneğin, isteğin META’sındaki alt etki alanını ayrıştırabilir ve ara katman yazılımı kullanarak istek üzerine ayrı bir özellik olarak ekleyebiliriz.

class SubdomainMiddleware:
    def process_request(self, request):
        """
        Parse out the subdomain from the request
        """
        host = request.META.get('HTTP_HOST', '')
        host_s = host.replace('www.', '').split('.')
        request.subdomain = None
        if len(host_s) > 2:
            request.subdomain = host_s[0]

İsteğinize ara katman yazılımı ile veri eklerseniz, yeni eklenen verilere daha sonra erişebilirsiniz. Burada, hangi kuruluşun uygulamanıza eriştiği gibi bir şeyi belirlemek için ayrıştırılmış alt etki alanını kullanacağız. Bu yaklaşım, tümü tek bir örneğe işaret eden joker karakter alt etki alanlarıyla bir DNS kurulumuyla dağıtılan uygulamalar için kullanışlıdır ve uygulamaya erişen kişi, erişim noktasına bağlı olarak tenli bir sürüm ister.

class OrganizationMiddleware:
    def process_request(self, request):
        """
        Determine the organization based on the subdomain
        """
        try:
            request.org = Organization.objects.get(domain=request.subdomain)
        except Organization.DoesNotExist:
            request.org = None

Ara katman yazılımının birbirine bağlı olması durumunda sıranın önemli olduğunu unutmayın. İstekler için, bağımlı ara yazılımın bağımlılıktan sonra yerleştirilmesini isteyeceksiniz.

MIDDLEWARE_CLASSES = [
    ...
    'myapp.middleware.SubdomainMiddleware',
    'myapp.middleware.OrganizationMiddleware',
    ...
]

Genel olarak işleme istisnası #

Değişiklikleri gönderen istemci en son değişikliklere sahip değilken, veritabanındaki bir nesneyi değiştirme girişimlerini algılamak için bir mantık uyguladığınızı varsayalım. Böyle bir durum olursa, özel bir ConfictError(detailed_message) özel istisnası oluşturursunuz.

Şimdi bu hata oluştuğunda bir HTTP 409 (Confict) durum kodu döndürmek istiyorsunuz. Bu istisnayı ortaya çıkarabilecek her görünümde işlemek yerine, genellikle bunun için ara katman yazılımı olarak kullanabilirsiniz.

class ConfictErrorHandlingMiddleware:
    def process_exception(self, request, exception):
        if not isinstance(exception, ConflictError):
            return  # Propagate other exceptions, we only handle ConflictError
        context = dict(confict_details=str(exception))
        return TemplateResponse(request, '409.html', context, status=409)
