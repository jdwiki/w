Pdb

Pdb ayrıca (Pdb) komut istemine sırasıyla ‘globals()’ veya ’locals()’ yazarak global veya yerel kapsamdaki tüm mevcut değişkenleri yazdırabilir.

Django Hata Ayıklama Araç Çubuğunu Kullanma #

İlk olarak, Django-debug-toolbar yüklemeniz gerekir:

pip install django-debug-toolbar

settings.py:

Ardından, onu projenin yüklü uygulamalarına ekleyin, ancak dikkatli olun - hata ayıklama araç çubuğu olarak yalnızca geliştirme amaçlı uygulamalar ve ara yazılımlar için farklı bir “settings.py” dosyası kullanmak her zaman iyi bir uygulamadır:

# If environment is dev...
DEBUG = True

INSTALLED_APPS += [
    'debug_toolbar',
]

MIDDLEWARE += ['debug_toolbar.middleware.DebugToolbarMiddleware']

Hata ayıklama araç çubuğu ayrıca statik dosyalara dayanır, bu nedenle uygun uygulama da dahil edilmelidir:

INSTALLED_APPS = [
    # ...
    'django.contrib.staticfiles',
    # ...
]

STATIC_URL = '/static/'

# If environment is dev...
DEBUG = True

INSTALLED_APPS += [
    'debug_toolbar',
]

Bazı durumlarda, “settings.py” içinde “INTERNAL_IPS"yi de ayarlamak gerekir:

INTERNAL_IPS = ('127.0.0.1', )

urls.py:

“urls.py” dosyasında, resmi belgelerin önerdiği gibi, bir sonraki snippet, hata ayıklama araç çubuğu yönlendirmesini etkinleştirmelidir:

if settings.DEBUG and 'debug_toolbar' in settings.INSTALLED_APPS:
    import debug_toolbar
    urlpatterns += [
        url(r'^__debug__/', include(debug_toolbar.urls)),
    ]

Kurulumdan sonra araç çubuğunun statiğini toplayın:

python manage.py collectstatic

İşte bu, projenizin sayfalarında hata ayıklama araç çubuğu görünecek ve yürütme süresi, SQL, statik dosyalar, sinyaller vb. hakkında çeşitli yararlı bilgiler sağlayacaktır.

HTML:

Ayrıca, “django-debug-toolbar”, düzgün şekilde oluşturulabilmesi için “text/html”, “” ve “” etiketlerinden oluşan bir İçerik türü gerektirir.

Her şeyi doğru yapılandırdığınızdan emin olmanıza rağmen hata ayıklama araç çubuğu hala oluşturulmadıysa: Bunu anlamaya çalışmak için bu “nükleer” çözümü kullanın.

Python Hata Ayıklayıcıyı Kullanma (Pdb) #

En temel Django hata ayıklama aracı, Python standart kitaplığının bir parçası olan pdb‘dir.

İlk görünüm komut dosyası

Basit bir ‘views.py’ betiğini inceleyelim:

from django.http import HttpResponse


def index(request):
    foo = 1
    bar = 0

    bug = foo/bar

    return HttpResponse("%d goes here." % bug)

Sunucuyu çalıştırmak için konsol komutu:

python manage.py runserver

Dizin sayfasını yüklemeye çalıştığınızda Django’nun bir ‘ZeroDivisionError’ atacağı açıktır, ancak hatanın kodun çok derinlerinde olduğunu varsayarsak, gerçekten kötü olabilir.

Bir kesme noktası ayarlama

Neyse ki, bu hatayı bulmak için bir kesme noktası belirleyebiliriz:

from django.http import HttpResponse

# Pdb import
import pdb


def index(request):
    foo = 1
    bar = 0
    
    # This is our new breakpoint
    pdb.set_trace()
    
    bug = foo/bar
    
    return HttpResponse("%d goes here." % bug)

Sunucuyu pdb ile çalıştırmak için konsol komutu:

python -m pdb manage.py runserver

Şimdi sayfa yükleme kesme noktası, kabukta (Pdb) istemini tetikleyecek ve bu da tarayıcınızı bekleme durumunda asacaktır.

pdb kabuğu ile hata ayıklama

Kabuk aracılığıyla komut dosyasıyla etkileşim kurarak bu görünümde hata ayıklamanın zamanı geldi:

> ../views.py(12)index()
-> bug = foo/bar
# input 'foo/bar' expression to see division results:
(Pdb) foo/bar
*** ZeroDivisionError: division by zero
# input variables names to check their values:
(Pdb) foo
1
(Pdb) bar
0
# 'bar' is a source of the problem, so if we set it's value > 0...
(Pdb) bar = 1
(Pdb) foo/bar
1.0
# exception gone, ask pdb to continue execution by typing 'c':
(Pdb) c
[03/Aug/2016 10:50:45] "GET / HTTP/1.1" 200 111

Son satırda, görüşümüzün bir “Tamam” yanıtı verdiğini ve olması gerektiği gibi çalıştığını görüyoruz.

pdb döngüsünü durdurmak için bir kabuğa q yazmanız yeterlidir.

“Yanlış iddia et” kullanma #

Geliştirme sırasında kodunuza aşağıdaki satırı ekleyin:

assert False, value

bu satır çalıştırıldığında Django’nun bir hata mesajı olarak sağlanan değerle bir AssertionError yükseltmesine neden olur.

Bu, bir görünümde veya bir görünümden çağrılan herhangi bir kodda meydana gelirse ve ‘DEBUG=True’ ayarlanırsa, tarayıcıda birçok hata ayıklama bilgisi içeren tam ve ayrıntılı bir yığın izleme görüntülenir.

İşiniz bittiğinde çizgiyi kaldırmayı unutmayın!

Hata Ayıklayıcı Kullanmak Yerine Daha Fazla Belge, Test, Günlük ve Onay Yazmayı Düşünün #

Hata ayıklama zaman ve çaba gerektirir.

Hata ayıklayıcıyla hataların peşine düşmek yerine, aşağıdakileri yaparak kodunuzu daha iyi hale getirmek için daha fazla zaman harcamayı düşünün:

  • Testler yazın ve çalıştırın. Python ve Django, kodunuzu bir hata ayıklayıcıyla manuel olarak test etmekten çok daha hızlı test etmek için kullanılabilen harika yerleşik test çerçevelerine sahiptir.
  • İşlevleriniz, sınıflarınız ve modülleriniz için uygun dokümantasyon yazma. PEP 257 ve Google’ın Python Stil Kılavuzu, iyi belge dizileri yazmak için iyi uygulamalar sağlar.
  • Günlük Kaydetmeyi Kullan - geliştirme sırasında ve dağıtımdan sonra programınızdan çıktı üretmek için.
  • Kodunuza önemli yerlere “iddialar” ekleyin: Belirsizliği azaltın, sorunları oluştukları anda yakalayın.

Bonus: Dokümantasyon ve testi birleştirmek için doctests yazın!
