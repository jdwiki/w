Google Drive'dan Otomatik İndirmek İçin Google Web App Komut Dosyası

Bu Basit Google Uygulaması Web Komut Dosyası (Bağımsız), kullanıcının yerel bilgisayarına indirilecek dosyalar için Google Drive’ın tekrar yoklanmasına olanak tanır.

Her ikisinin de işlevini sağlamak için bir uygulama komut dosyasının nasıl kullanılacağını gösterir:

  1. Kullanıcı arayüzü (bu örnekte basit olan)
  2. Dosya indirme sayfası.

Nasıl çalıştığına dair daha kapsamlı bir açıklama için “Nasıl Çalışır” Örneği’ni okuyun.

Web Komut Dosyasının çalışması için yayınlanmış olması gerekir.

https://script.google.com için açılır pencereler etkinleştirilmelidir

formlar.html #

<!DOCTYPE html>
<html>
  <head>
    <base target="_top">
    <script>
    
    setInterval(
    function () 
    { 
      document.getElementById('messages').innerHTML = 'Event Timer Fetching';
      google.script.run
        .withSuccessHandler(onSuccess)
        .withFailureHandler(onFailure)
        .fetchFromGoogleDrive();
    }, 60000);
    
    function callFetchGoogleDrive() {
      document.getElementById('messages').innerHTML = 'Fetching';
      google.script.run
        .withSuccessHandler(onSuccess)
        .withFailureHandler(onFailure)
        .fetchFromGoogleDrive();   
    }
      
    function onSuccess(sHref) 
    {
      if(new String(sHref).valueOf() == new String("").valueOf())
      {
        document.getElementById('messages').innerHTML = 'Nothing to download';
      }
      else
      {
        document.getElementById('messages').innerHTML = 'Success';
        document.getElementById('HiddenClick').href = sHref;
        document.getElementById('HiddenClick').click(); // Must enable Pop Ups for https://script.google.com
      }
    }
    
    function onFailure(error) 
    {
      document.getElementById('messages').innerHTML = error.message;
    }
    
    </script>
  </head>
  <body>
    <div id="messages">Waiting to DownLoad!</div>
    <div>
      <a id="HiddenClick" href="" target="_blank" onclick="google.script.host.close" style="visibility: hidden;">Hidden Click</a>
    </div>
    <div>
      <button type="button" onclick='callFetchGoogleDrive();' id="Fetch">Fetch Now!</button> 
    </div>
  </body>
</html>

function doGet(e){
   var serveFile = e.parameter.servefile;
   var id = e.parameter.id;
   
   if(serveFile)
   {
     return downloadFile(id); // and Hyde
   }
   
  return HtmlService.createHtmlOutputFromFile('form.html'); // Jekyll
}

function fetchFromGoogleDrive() { // Jekyll
  var fileslist = DriveApp.searchFiles("your search criteria goes here + and trashed = false"); // the 'and trashed = false' prevents the same file being download more than once
  
  if (fileslist.hasNext()) {
    var afile = fileslist.next();
    var html = ScriptApp.getService().getUrl()+"?servefile=true&id="+afile.getId();
    return html;
  }
  else
  {
    return '';
  }
}
   
function downloadFile(id){ // and Hyde  
  try
  {
    var afile = DriveApp.getFileById(id);
  
    var aname = afile.getName();
    var acontent = afile.getAs('text/plain').getDataAsString();
    
    var output = ContentService.createTextOutput();
    output.setMimeType(ContentService.MimeType.CSV);
    output.setContent(acontent);
    output.downloadAsFile(aname);
    afile.setTrashed(true);
    return output;
  }
  catch (e) {
    return ContentService.createTextOutput('Nothing To Download')
  }
}

Nasıl çalışır #

Dosyaları Drive’dan kullanıcının yerel bilgisayarına (İndirme Klasörü) otomatik olarak indirmek (Anket) için Google Drive (Bağımsız) Web Uygulaması.

DriveApp, dosyaları aramak ve indirmek için mekanizmalar sağlar. Ancak, Google Apps’ın devraldığı istemci/sunucu mimarisi nedeniyle indirme mekanizmasının bazı ciddi sınırlamaları vardır. (Google’ın suçu yok)

Sunucu tarafı DriveApp, yerel PC’ye indirmek için doğrudan bir işlev sağlamaz çünkü sunucunun istemcinin nerede olduğuna dair hiçbir fikri yoktur ve dosyayı sunucunun kendisine indirmek anlamsız olacaktır.

Sunucu tarafı kodu, istemci tarafı koduna dosya verilerini veya dosyaya bir bağlantı sağlamak için bir mekanizmaya ihtiyaç duyar. Bu mekanizmaların her ikisi de sağlanmıştır, ancak birincisinden elde edilen veriler, doğrudan istemci tarafı kodu tarafından kullanılmakla sınırlıdır. İstemcinin, elde edilen verileri yerel PC’ye kaydetme mekanizması yoktur. Böylece verileri web sayfasının kendisinde görüntülemek için kullanılabilir.

İkinci mekanizma, betiğin url’sinin (kendisinin) veya Drive dosyasının url’sinin geri iletilmesine izin verir. Drive dosyası url’si, dosyayı indirmek için doğrudan istemci tarayıcısında kullanılamadığından çok kullanışlı değildir. Bu url’yi bağlantıya yerleştirmek (ve tıklamak) yalnızca açılan ancak aslında hiçbir şey yapmayan (muhtemelen dosyayı çevrimiçi görüntülemek dışında) bir web sayfasıyla sonuçlanır.

Bu komut dosyası url’sini bırakır. Ancak komut dosyası url’si dosyayı değil yalnızca komut dosyasını sağlar.

Bir indirme işlemini başlatmak için, Drive hizmetinden dosyanın, tam olarak Google çevrimiçi kılavuzlarında gösterildiği gibi ContentService createTextOutput kullanılarak sunucu tarafı komut dosyasının doGet / doPost işlevinden döndürülmesi gerekir. Ancak bu, doGet/doPost tarafından döndürülen sonuçlar tarafından oluşturulan web sayfasında başka bir UI öğesi olamayacağı anlamına gelir.

Bu bizi çok çekici olmayan bir çözümle bırakır. Bir sayfayı indiren, kapatan ve başka bir indirme gerektiğinde manuel olarak açılmasını gerektiren, kullanıcı UI öğeleri olmayan boş bir web sayfası.

Açıkçası, başka bir barındırma web sayfası, bu sorunu çözmek için kullanıcı arayüzünü ve Web Uygulaması İndirme komut dosyasının bağlantısını sağlayabilir.

Bu komut dosyası, bu sorunu çözmek için Dr Jekyll ve Mr Hyde yaklaşımını kullanır.

Komut dosyası, GET (doGet) parametresi olmadan açılırsa, varsayılan olarak bir form görüntülemeye geçer. Bu, yayınlanan uygulama bir kullanıcı tarafından ilk açıldığında koşul olacaktır. Bu örnekte sağlanan form son derece basittir.

Komut dosyası servefile=true parametresiyle açılırsa, komut dosyası bir Drive dosyası indirmesi gibi davranır.

İstemci tarafı javascript’i, indirilecek başka bir dosyanın olup olmadığını kontrol etmek için sunucu tarafı komut dosyasını periyodik olarak çağıran bir yoklama mekanizması (olay zamanlayıcı setInterval) içerir.

Sunucu tarafı komut dosyası çalıştırıldığında, arama ölçütleriyle* eşleşen herhangi bir Drive dosyası bulursa, parametrelerle birlikte eklenen komut dosyasının URL’sini döndürür:

?servefile=true&id=the_id_of_the_google_drive_file

(* Bu basit örnekteki arama kriterleri sunucu tarafı komut dosyasına sabit kodlanmıştır. Gerekirse istemciden sunucuya kolayca geçirilebilir.)

Bu bilgi, tanınan withSuccessHandler mekanizması aracılığıyla istemciye bir dize olarak döndürülür.

İstemci java betiği daha sonra bu döndürülen bilgiyle gizli bir çapanın HREF’sini günceller ve ardından çapaya otomatik olarak tıklar.

Bu, uygulamanın/komut dosyasının başka bir çağrısının başlatılmasına neden olur. Uygulamanın yeni çağrılması başlatıldığında doGet, servefile parametresini algılayacak ve kullanıcı arayüzünü döndürmek yerine dosyayı tarayıcıya döndürecektir. Döndürülen dosya, yukarıda açıklanan arama tarafından daha önce döndürülen sağlanan kimlik parametresi tarafından tanımlanan dosya olacaktır.

Sağlanan kimliğe sahip dosyanın hala mevcut olduğu göz önüne alındığında, dosya indirilecek ve uygulamanın yeni çağrısı, bu işlemi tekrarlamak için ilk çağrıyı bırakarak kapanacaktır.

Kullanıcı/test cihazı zamanlayıcıyı beklemekte sabırsızlanırsa, basit arayüzde bir düğme sağlanır, ancak gerekli değildir ve aksi takdirde kaldırılabilir.

Basit biçim, gerekirse daha zengin bir kullanıcı arabirimi sağlamak için genişletilebilir. Dosya arama kriterlerinin sağlanması gibi.
