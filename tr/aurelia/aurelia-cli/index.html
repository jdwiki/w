Bir CLI Uygulamasına Önyükleme Ekleme #

Yaygın olarak kullanılan bir CSS/Javascript kitaplığı Bootstrap. Aurelia CLI tabanlı uygulamanıza kurmak için önce Npm kullanarak kurmanız gerekir.

npm install bootstrap --save

Bootstrap, jQuery’ye sıkı bir bağımlılığa sahip olduğundan, jQuery’nin de kurulu olduğundan emin olmamız gerekir:

npm install jquery --save

Şimdi tercih ettiğiniz IDE/kod düzenleyicide proje dizininizde şu dosyayı açın: aurelia_project/aurelia.json ve dosyanın sonuna doğru build.bundles bölümüne gidin. Paketlerden birine Bootstrap eklemek isteyeceğiz. Bu örnekte, satıcı paketine hem jQuery hem de Bootstrap ekleyeceğiz.

"jquery",
{
  "name": "bootstrap",
  "path": "../node_modules/bootstrap/dist",
  "main": "js/bootstrap.min",
  "deps": ["jquery"],
  "exports": "$",
  "resources": [
    "css/bootstrap.css"
  ]
}

Bu, Bootstrap’i uygulamanızda erişilebilir ve kodunuzdaki ‘import ‘bootstrap’’ yoluyla içe aktarılabilir hale getirecektir (bu, yukarıda tanımlanan ’name’ özelliğidir). Önyükleme tanımımızın “deps”: []" bölümündeki “jquery” referansına dikkat edin. Ayrıca Bootstrap’ın CSS’sini ana vendor-bundle.js dosyamızda "resources":[] özelliğini kullanarak paketlemek istediğimizi belirtiyoruz.

Son olarak, yeni eklenen Bootstrap bağımlılığımızı kullanmak için, önce aşağıdakini dosyanın başına koyarak main.js dosyamızın içindeki Bootstrap kitaplığını içe aktarmak istiyoruz.

import 'bootstrap';

Bootstrap CSS’yi tüm uygulama tarafından küresel olarak erişilebilir olacağı bir yerden eklemek istiyoruz, bu nedenle ‘app.html’nin içinde ‘’ etiketlerinin arasına aşağıdakini koyun.

<require from="bootstrap/css/bootstrap.css"></require>

Bir CLI Uygulamasına Lodash Ekleme #

Yaygın olarak kullanılan bir yardımcı program kitaplığı Lodash. Aurelia CLI tabanlı uygulamanıza kurmak için önce Npm kullanarak kurmanız gerekir.

npm install lodash --save

Şimdi tercih ettiğiniz IDE/kod düzenleyicide proje dizininizde şu dosyayı açın: aurelia_project/aurelia.json ve dosyanın sonuna doğru build.bundles bölümüne gidin. Paketlerden birine Lodash’ı eklemek isteyeceğiz. Bu örnek için, satıcı paketine Lodash ekleyeceğiz.

Lodash, CommonJS kurallarına uyduğu ve birleştirilmiş bir dosyayı dışa aktardığı için, bağımlılıklar dizisine aşağıdaki nesneyi ekleyerek onu kolayca kaydedebiliriz:

{
  "name": "lodash",
  "path": "../node_modules/lodash/lodash.min"
}

Bağımlılık çözülürken otomatik olarak ekleneceği için .js dosya son ekini atladığımızı unutmayın.

Şimdi, ES6 içe aktarma sözdizimini kullanarak basitçe içe aktararak Lodash’ı kullanabiliriz:

``lodash’ dan içe aktar;`

Aurelia-I18N Eklentisini Yükleme #

Resmi I18N Eklentisini CLI Projenize almak için sonraki adımları izleyerek yüklememiz gerekiyor.

İlk önce eklentiyi npm ile kurmak istiyorsunuz:

`npm aurelia-i18n’yi kurun’

Aurelia-I18N, i18next tarafından desteklendiğinden, onu ve seçtiğiniz bir arka uç eklentisini yüklemelisiniz. Örnek olarak, i18next-xhr-backend’den yararlanacağız:

`npm i18next i18next-xhr-backend’i kurun’

Bundan sonra CLI Uygulamamıza yeni bağımlılıklar hakkında bilgi vermemiz gerekiyor. Bunu yapmak için aurelia_project/aurelia.json dosyasını açacağız ve bağımlılıklar adlı bölüme kaydıracağız. Oraya aşağıdaki üç girişi ekleyin:

{
  "name": "i18next",
  "path": "../node_modules/i18next/dist/umd",
  "main": "i18next"
},
{
  "name": "aurelia-i18n",
  "path": "../node_modules/aurelia-i18n/dist/amd",
  "main": "aurelia-i18n"
},
{
  "name": "i18next-xhr-backend",
  "path": "../node_modules/i18next-xhr-backend/dist/umd",
  "main": "i18nextXHRBackend"
}

Harika, şimdi resmi Aurelia-I18N Kılavuzunu takip ederek uygulamanızın kökünde locales adlı bir klasör oluşturuyoruz.

Uygulamanızın barındırılan kökü olduğundan, klasörü kök dizinine (src ile aynı düzeyde) yerleştirmeniz gerekir.

Buraya desteklemek istediğiniz her dil için alt klasörler ekleyin, örneğin İngilizce ve Almanca için en ve de.

Bu klasörlerin her birinin içinde, çeviri anahtarlarınız ve değerlerinizle translation.json adlı bir dosya oluşturun. Ayrıntılı bilgi için resmi kılavuzu takip edin.

Son olarak, eklentiyi uygulamanızın içine bağlamanın zamanı geldi. Bu nedenle src/main.js dosyanıza gidin ve aşağıdaki gibi yapılandırın.

/**********************************************/
/          add the necessary imports           /
/**********************************************/
import environment from './environment';
import Backend from 'i18next-xhr-backend';

//Configure Bluebird Promises.
//Note: You may want to use environment-specific configuration.
Promise.config({
  warnings: {
    wForgottenReturn: false
  }
});

export function configure(aurelia) {
  aurelia.use
    .standardConfiguration()
    .feature('resources');

  if (environment.debug) {
    aurelia.use.developmentLogging();
  }

  if (environment.testing) {
    aurelia.use.plugin('aurelia-testing');
  }

  /**********************************************/
  /             register the plugin              /
  /**********************************************/
  aurelia.use.plugin('aurelia-i18n', (instance) => {
    // register backend plugin
    instance.i18next.use(Backend);

    // adapt options to your needs (see http://i18next.com/docs/options/)
    // make sure to return the promise of the setup method, in order to guarantee proper loading
    return instance.setup({
      backend: {                                  // <-- configure backend settings
        loadPath: './locales/{{lng}}/{{ns}}.json', // <-- XHR settings for where to get the files from
      },
      lng : 'de',
      attributes : ['t','i18n'],
      fallbackLng : 'en',
      debug : false
    });
  });

  aurelia.start().then(() => aurelia.setRoot());
}

Bir CLI Uygulamasına Aurelia Yapılandırması Ekleme #

Aurelia konfigürasyonunun bir cli uygulamasına eklenmesi bazen bir derleme hatası üretir. Bu, eksik bir bağımlılıktan kaynaklanır, bu nedenle bağımlılığı derleme paketine eklememiz yeterlidir.

Takip etmeyi dene:


npm derin genişletmeyi kurun --save
npm aurelia yapılandırmasını kurun --save

Şimdi aşağıdakileri bağımlılıklar dizisindeki aurelia.json dosyasına ekleyin:


{
    "name": "deep-extend",
    "path": "../node_modules/deep-extend/lib/deep-extend"
  },
  {
    "name": "aurelia-configuration",
    "path": "../node_modules/aurelia-configuration/dist/amd",
    "main": "index"
  }
