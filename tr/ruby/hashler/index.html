Hash, benzersiz anahtarların ve değerlerinin sözlük benzeri bir koleksiyonudur. İlişkili diziler olarak da adlandırılırlar, Dizilere benzerler, ancak bir Dizinin dizini olarak tamsayıları kullandığı durumlarda, Hash herhangi bir nesne türünü kullanmanıza izin verir. Anahtarına başvurarak bir Hash’te yeni bir giriş alır veya yeni bir giriş oluşturursunuz.

Sözdizimi #

  • { ad_adı: “Noel”, ikinci_adı: “Edmonds” }

  • { :first_name => “Noel”, :second_name => “Edmonds” }

  • { “İlk Ad” => “Noel”, “İkinci Ad” => “Edmonds” }

  • { birinci_anahtar => birinci_değer, ikinci_anahtar => ikinci_değer }

Ruby’deki karmalar, bir karma tablosu kullanarak değerlerin anahtarlarını eşler.

Herhangi bir hashable nesne anahtar olarak kullanılabilir. Bununla birlikte, azaltılmış nesne tahsisi nedeniyle genellikle birkaç Ruby sürümünde daha verimli olduğu için bir ‘Sembol’ kullanmak çok yaygındır.

{ key1: "foo", key2: "baz"  }

Karma oluşturma #

Ruby’deki bir karma, anahtarları değerlerle eşleyen bir karma tablosu uygulayan bir nesnedir. Ruby, “{}” kullanarak karmaları tanımlamak için belirli bir değişmez sözdizimini destekler:

my_hash = {}  # an empty hash
grades = { 'Mark' => 15, 'Jimmy' => 10, 'Jack' => 10 }

Standart “yeni” yöntemi kullanılarak da bir karma oluşturulabilir:

my_hash = Hash.new  # any empty hash
my_hash = {}        # any empty hash

Karmalar, diziler, nesneler ve diğer karmalar gibi karmaşık türler dahil olmak üzere herhangi bir türden değere sahip olabilir:

mapping = { 'Mark' => 15, 'Jimmy' => [3,4], 'Nika' => {'a' => 3, 'b' => 5} }
mapping['Mark']   # => 15
mapping['Jimmy']  # => [3, 4]
mapping['Nika']   # => {"a"=>3, "b"=>5}

Ayrıca anahtarlar, karmaşık olanlar da dahil olmak üzere herhangi bir türde olabilir:

mapping = { 'Mark' => 15, 5 => 10, [1, 2] => 9 }
mapping['Mark']  # => 15
mapping[[1, 2]]  # => 9

Semboller genellikle karma anahtarlar olarak kullanılır ve Ruby 1.9, özellikle bu süreci kısaltmak için yeni bir sözdizimi sunar. Aşağıdaki karmalar eşdeğerdir:

# Valid on all Ruby versions
grades = { :Mark => 15, :Jimmy => 10, :Jack => 10 }
# Valid in Ruby version 1.9+
grades = { Mark: 15, Jimmy: 10, Jack: 10 }

Aşağıdaki karma (tüm Ruby sürümlerinde geçerlidir) farklıdır, çünkü tüm anahtarlar dizgedir:

grades = { "Mark" => 15, "Jimmy" => 10, "Jack" => 10 }

Her iki sözdizimi sürümü karıştırılabilirken, aşağıdakiler önerilmez.

mapping = { :length => 45, width: 10 }

Ruby 2.2+ ile, sembol tuşlarıyla bir karma oluşturmak için alternatif bir sözdizimi vardır (en çok sembol boşluk içeriyorsa kullanışlıdır):

grades = { "Jimmy Choo": 10, :"Jack Sparrow": 10 }
# => { :"Jimmy Choo" => 10, :"Jack Sparrow" => 10}

Varsayılan Değerleri Ayarlama #

Varsayılan olarak, var olmayan bir anahtarın değerini aramaya çalışmak “nil” değerini döndürür. İsteğe bağlı olarak, var olmayan bir anahtarla karmaya erişildiğinde döndürülecek başka bir değer (veya gerçekleştirilecek bir eylem) belirleyebilirsiniz. Bu, “varsayılan değer” olarak adlandırılsa da, tek bir değer olması gerekmez; örneğin, anahtarın uzunluğu gibi hesaplanmış bir değer olabilir.

Bir karmanın varsayılan değeri, yapıcısına iletilebilir:

h = Hash.new(0)

h[:hi] = 1 
puts h[:hi]  # => 1 
puts h[:bye] # => 0 returns default value instead of nil

Önceden oluşturulmuş bir Hash üzerinde bir varsayılan da belirtilebilir:

my_hash = { human: 2, animal: 1 }
my_hash.default = 0
my_hash[:plant] # => 0

Her yeni anahtara erişildiğinde varsayılan değerin kopyalanmadığına dikkat etmek önemlidir; bu, varsayılan değer bir referans türü olduğunda şaşırtıcı sonuçlara yol açabilir:

# Use an empty array as the default value
authors = Hash.new([])

# Append a book title 
authors[:homer] << 'The Odyssey'

# All new keys map to a reference to the same array:
authors[:plato] # => ['The Odyssey']

Bu sorunu aşmak için Hash yapıcısı, her yeni anahtara erişildiğinde yürütülen bir bloğu kabul eder ve döndürülen değer varsayılan olarak kullanılır:

authors = Hash.new { [] }

# Note that we're using += instead of <<, see below
authors[:homer] += ['The Odyssey']
authors[:plato] # => []

authors # => {:homer=>["The Odyssey"]}

Varsayılan değer karmaya otomatik olarak atanmadığı için yukarıda « yerine += kullanmamız gerektiğine dikkat edin; « kullanmak diziye eklenirdi, ancak yazarlar[:homer] tanımsız kalırdı:

authors[:homer] << 'The Odyssey' # ['The Odyssey']
authors[:homer] # => []
authors # => {}

Erişimde varsayılan değerler atayabilmek ve daha karmaşık varsayılanları hesaplayabilmek için, varsayılan blok hem karma hem de anahtardan geçirilir:

authors = Hash.new { |hash, key| hash[key] = [] }

authors[:homer] << 'The Odyssey'
authors[:plato] # => []

authors # => {:homer=>["The Odyssey"], :plato=>[]}

Ayrıca, bir işlem yapmak ve/veya anahtara (veya diğer bazı verilere) bağlı bir değer döndürmek için varsayılan bir blok kullanabilirsiniz:

chars = Hash.new { |hash,key| key.length }

chars[:test] # => 4

Daha karmaşık karmalar bile oluşturabilirsiniz:

page_views = Hash.new { |hash, key| hash[key] = { count: 0, url: key } }
page_views["http://example.com"][:count] += 1
page_views # => {"http://example.com"=>{:count=>1, :url=>"http://example.com"}}

_Zaten var olan bir karma üzerinde varsayılanı bir Proc olarak ayarlamak için default_proc= kullanın:

authors = {}
authors.default_proc = proc { [] }

authors[:homer] += ['The Odyssey']
authors[:plato] # => []

authors # {:homer=>["The Odyssey"]}

Değerlere Erişim #

Bir karmanın bireysel değerleri, [] ve []= yöntemleri kullanılarak okunur ve yazılır:

my_hash = { length: 4, width: 5 }

my_hash[:length] #=> => 4

my_hash[:height] = 9

my_hash #=> {:length => 4, :width => 5, :height => 9 }

Varsayılan olarak, karmaya eklenmemiş bir anahtara erişmek “nil” değerini döndürür, yani bir anahtarın değerini aramaya çalışmak her zaman güvenlidir:

my_hash = {}

my_hash[:age] # => nil

Karmalar ayrıca dizelerde anahtarlar içerebilir. Bunlara normal olarak erişmeye çalışırsanız, yalnızca bir “nil” döndürür, bunun yerine onlara dize anahtarlarıyla erişirsiniz:

my_hash = { "name" => "user" }

my_hash[:name]    # => nil
my_hash["name"]   # => user

Anahtarların bulunmasının beklendiği veya gerekli olduğu durumlarda, karmaların, var olmayan bir anahtara erişirken bir istisna oluşturacak bir “getirme” yöntemi vardır:

my_hash = {}

my_hash.fetch(:age) #=> KeyError: key not found: :age

“getir”, ikinci bağımsız değişkeni olarak, anahtar önceden ayarlanmamışsa döndürülen varsayılan bir değeri kabul eder:

my_hash =  {}
my_hash.fetch(:age, 45) #=> => 45

“getir”, anahtar daha önce ayarlanmamışsa döndürülen bir bloğu da kabul edebilir:

my_hash = {}
my_hash.fetch(:age) { 21 } #=> 21

my_hash.fetch(:age) do |k|
  puts "Could not find #{k}"
end

#=> Could not find age

Hash’ler ayrıca ‘[]=’ için bir takma ad olarak bir ‘mağaza’ yöntemini de destekler:

my_hash = {}

my_hash.store(:age, 45)

my_hash #=> { :age => 45 }

Ayrıca, “değerler” yöntemini kullanarak bir karmanın tüm değerlerini elde edebilirsiniz:

my_hash = { length: 4, width: 5 }

my_hash.values #=> [4, 5]

Not: Bu yalnızca Ruby 2.3+ içindir ‘#dig’ iç içe ‘Hash’ler için kullanışlıdır. idx nesnelerinin dizisi tarafından belirtilen iç içe değeri, her adımda dig çağırarak, herhangi bir ara adım sıfırsa nil döndürerek çıkarır.

h = { foo: {bar: {baz: 1}}}

h.dig(:foo, :bar, :baz)   # => 1
h.dig(:foo, :zot, :xyz)   # => nil

g = { foo: [10, 11, 12] }
g.dig(:foo, 1)            # => 11

Otomatik olarak Derin Hash oluşturma #

Hash, istenen ancak mevcut olmayan (nil) anahtarlar için varsayılan bir değere sahiptir:

a = {}
p a[ :b ] # => nil

Yeni bir Hash oluştururken, varsayılanı belirtebilirsiniz:

b = Hash.new 'puppy'
p b[ :b ]            # => 'puppy'

Hash.new ayrıca, Perl’in otomatik canlandırma davranışı veya “mkdir -p” gibi iç içe geçmiş karmaları otomatik olarak oluşturmanıza izin veren bir blok alır:

# h is the hash you're creating, and k the key.
#
hash = Hash.new { |h, k| h[k] = Hash.new &h.default_proc }
hash[ :a ][ :b ][ :c ] = 3

p hash # => { a: { b: { c: 3 } } }

Bir Hash Üzerinde Yineleme #

Bir “Hash”, “Sayılandırılabilir#her”, “Sayılandırılabilir#her_çift”, “Sayılandırılabilir#her_anahtar” ve “Sayılandırılabilir#her_değer” gibi birkaç yineleme yöntemi sağlayan “Sayılandırılabilir” modülünü içerir.

“.each” ve “.each_pair”, her bir anahtar/değer çifti üzerinde yinelenir:

h = { "first_name" => "John", "last_name" => "Doe" }
h.each do |key, value|
    puts "#{key} = #{value}"
end

# => first_name = John
#    last_name = Doe

“.each_key” yalnızca anahtarlar üzerinde yinelenir:

h = { "first_name" => "John", "last_name" => "Doe" }
h.each_key do |key|
  puts key
end

# => first_name
#    last_name

“.each_value” yalnızca değerler üzerinde yinelenir:

h = { "first_name" => "John", "last_name" => "Doe" }
h.each_value do |value|
    puts value
end

# => John
#    Doe

“.each_with_index”, öğeler üzerinde yinelenir ve yinelemenin dizinini sağlar:

h = { "first_name" => "John", "last_name" => "Doe" }
h.each_with_index do |(key, value), index|
    puts "index: #{index} | key: #{key} | value: #{value}"
end

# => index: 0 | key: first_name | value: John
#    index: 1 | key: last_name | value: Doe

Dizilere ve Dizilerden Dönüşüm #

Karmalar dizilere ve dizilerden serbestçe dönüştürülebilir. Anahtar/değer çiftlerinin bir diziye dönüştürülmesi, çift için iç içe diziler içeren bir dizi üretecektir:

{ :a => 1, :b => 2 }.to_a # => [[:a, 1], [:b, 2]]

Ters yönde, aynı formattaki bir diziden bir Hash oluşturulabilir:

[[:x, 3], [:y, 4]].to_h # => { :x => 3, :y => 4 }

Benzer şekilde, Hash’ler “Hash[]” ve değişen anahtar ve değerlerin bir listesi kullanılarak başlatılabilir:

Hash[:a, 1, :b, 2] # => { :a => 1, :b => 2 }

Veya her biri iki değer içeren bir dizi dizisinden:

Hash[ [[:x, 3], [:y, 4]] ] # => { :x => 3, :y => 4 }

Hash’ler, “flatten()” kullanılarak bir dizi değişen anahtar ve değere geri dönüştürülebilir:

{ :a => 1, :b => 2 }.flatten # => [:a, 1, :b, 2]

Bir diziye ve diziden kolay dönüştürme, “Hash"ın “toplama” ve “zip” gibi birçok “Sayılandırılabilir” yöntemle iyi çalışmasına olanak tanır:

Hash[('a'..'z').collect{ |c| [c, c.upcase] }] # => { 'a' => 'A', 'b' => 'B', ... }

people = ['Alice', 'Bob', 'Eve']
height = [5.7, 6.0, 4.9]
Hash[people.zip(height)] # => { 'Alice' => 5.7, 'Bob' => '6.0', 'Eve' => 4.9 }

Karmaları filtreleme #

“select”, bloğun “true” olarak değerlendirdiği anahtar/değer çiftleriyle yeni bir “karma” döndürür.

{ :a => 1, :b => 2, :c => 3 }.select { |k, v| k != :a && v.even? } # => { :b => 2 }

Bir filtre bloğunda anahtar veya değer’e ihtiyacınız olmayacaksa, kural, o yerde bir “_” kullanmaktır:

{ :a => 1, :b => 2, :c => 3 }.select { |_, v| v.even? } # => { :b => 2 }
{ :a => 1, :b => 2, :c => 3 }.select { |k, _| k == :c } # => { :c => 3 }

“reddetme”, bloğun “yanlış” olarak değerlendirdiği anahtar/değer çiftleriyle yeni bir “karma” döndürür:

{ :a => 1, :b => 2, :c => 3 }.reject { |_, v| v.even? } # => { :a => 1, :c => 3 }
{ :a => 1, :b => 2, :c => 3 }.reject { |k, _| k == :b } # => { :a => 1, :c => 3 }

Hash’in tüm anahtarlarını veya değerlerini alma #

{foo: 'bar', biz: 'baz'}.keys   # => [:foo, :biz]
{foo: 'bar', biz: 'baz'}.values # => ["bar", "baz"]
{foo: 'bar', biz: 'baz'}.to_a   # => [[:foo, "bar"], [:biz, "baz"]]
{foo: 'bar', biz: 'baz'}.each   #<Enumerator: {:foo=>"bar", :biz=>"baz"}:each>

Karma işlevini geçersiz kılma #

Ruby hash’leri, hash işlemini gerçekleştirmek ve hash’te depolanan nesneleri dahili hash binlere atamak için ‘hash’ ve ’eql?’ yöntemlerini kullanır. Ruby’de ‘karma’nın varsayılan uygulaması mırıltı karma işlevi karma nesnenin tüm üye alanları üzerinde şeklindedir. Bu davranışı geçersiz kılmak için “hash” ve “eql” yöntemlerini geçersiz kılmak mümkündür.

Diğer hash uygulamalarında olduğu gibi, a ve b iki nesnesi, “a.hash == b.hash” ise aynı kovaya hash edilecek ve “a.eql?(b)” ise aynı kabul edilecektir. Bu nedenle, ‘hash’ ve ’eql?‘yi yeniden uygularken, ‘a’ ve ‘b’ ’eql?’ altında eşitse, aynı ‘hash’ değerini döndürmeleri gerektiğinden emin olunmalıdır. Aksi takdirde bu, bir karmada yinelenen girişlere neden olabilir. Tersine, “hash” uygulamasında kötü bir seçim, birçok nesnenin aynı karma paketini paylaşmasına, O(1) arama süresini etkin bir şekilde yok etmesine ve tüm nesnelerde “eql?” çağrısı için O(n)‘ye neden olabilir.

Aşağıdaki örnekte, ilk olarak eklendiği için yalnızca “A” sınıfının örneği anahtar olarak saklanır:

class A
  def initialize(hash_value)
    @hash_value = hash_value
  end
  def hash
    @hash_value # Return the value given externally
  end
  def eql?(b)
    self.hash == b.hash
  end
end

class B < A
end

a = A.new(1)
b = B.new(1)

h = {}
h[a] = 1
h[b] = 2

raise "error" unless h.size == 1
raise "error" unless h.include? b
raise "error" unless h.include? a

Anahtarları ve değerleri değiştirme #

Değiştirilen anahtarlar veya değerlerle yeni bir karma oluşturabilir, gerçekten de inject kullanarak anahtar ekleyebilir veya silebilirsiniz. -inject) (AKA, reduce). Örneğin, dizili anahtarlar ve büyük harf değerleriyle bir karma üretmek için:

fruit = { name: 'apple', color: 'green', shape: 'round' }
# => {:name=>"apple", :color=>"green", :shape=>"round"}

new_fruit = fruit.inject({}) { |memo, (k,v)| memo[k.to_s] = v.upcase; memo }

# => new_fruit is {"name"=>"APPLE", "color"=>"GREEN", "shape"=>"ROUND"}

Hash, bir numaralandırılabilir, özünde bir anahtar/değer çiftleri koleksiyonudur. Bu nedenle ’each’, ‘map’ ve ‘inject’ gibi yöntemlere sahiptir.

Verilen bloktaki her anahtar/değer çifti için değerlendirilir, ilk çalıştırmadaki notun değeri, ‘inject’e geçirilen tohum değeridir, bizim durumumuzda boş bir karma, ‘{}’. Sonraki değerlendirmeler için “memo"nun değeri, önceki blok değerlendirmesinin döndürülen değeridir, bu nedenle “memo"yu bir değere sahip bir anahtar ayarlayarak değiştiririz ve sonunda “memo” döndürürüz. Son blok değerlendirmesinin dönüş değeri, bizim durumumuzda ‘memo’ olan ‘inject’in dönüş değeridir.

Nihai değeri sağlamak zorunda kalmamak için bunun yerine each_with_object kullanabilirsiniz:

new_fruit = fruit.each_with_object({}) { |(k,v), memo| memo[k.to_s] = v.upcase }

Hatta harita:

<!-- if version [gte 1.8] -->
new_fruit = Hash[fruit.map{ |k,v| [k.to_s, v.upcase] }]
<!-- end version if -->

(Karmaların yerinde nasıl değiştirileceği de dahil olmak üzere daha fazla ayrıntı için bu yanıta bakın.)

Hash’lerde İşlemleri Ayarlama #

  • Karmaların Kesişimi

    To get the intersection of two hashes, return the shared keys the values of which are equal:

    hash1 = { :a => 1, :b => 2 }
hash2 = { :b => 2, :c => 3 }
hash1.select { |k, v| (hash2.include?(k) && hash2[k] == v) } # => { :b => 2 }

  • Karmaların birleştirilmesi (birleştirme):

bir karmadaki anahtarlar benzersizdir, birleştirilecek olan her iki karmada da bir anahtar varsa, “birleştirme"nin çağrıldığı karmadan birinin üzerine yazılır:

```
hash1 = { :a => 1, :b => 2 }
hash2 = { :b => 4, :c => 3 }

hash1.merge(hash2) # => { :a => 1, :b => 4, :c => 3 }
hash2.merge(hash1) # => { :b => 2, :c => 3, :a => 1 }
```
