Sözdizimi #

  • sınıf adı
  • #sınıf davranışını açıklayan bazı kodlar
  • son

Ruby’deki sınıf adları Sabitlerdir, bu nedenle ilk harf büyük olmalıdır.

class Cat # correct
end  

class dog # wrong, throws an error
end

Yapıcı #

Bir sınıfın yalnızca bir yapıcısı olabilir, bu “initialize” adı verilen bir yöntemdir. Sınıfın yeni bir örneği oluşturulduğunda yöntem otomatik olarak çağrılır.

class Customer
  def initialize(name)
     @name = name.capitalize 
  end
end

sarah = Customer.new('sarah')
sarah.name #=> 'Sarah'

Bir sınıf oluşturma #

class anahtar sözcüğünü kullanarak yeni bir sınıf tanımlayabilirsiniz.

class MyClass
end

Bir kez tanımlandıktan sonra, .new yöntemini kullanarak yeni bir örnek oluşturabilirsiniz.

somevar = MyClass.new
# => #<MyClass:0x007fe2b8aa4a18>

Erişim Düzeyleri #

Ruby’nin üç erişim seviyesi vardır. Bunlar “kamusal”, “özel” ve “korumalı”dır.

‘private’ veya ‘protected’ anahtar kelimelerini takip eden yöntemler bu şekilde tanımlanır. Bunlardan önce gelen yöntemler, örtük olarak ‘public’ yöntemlerdir.

Genel Yöntemler #

Genel bir yöntem, oluşturulan nesnenin davranışını tanımlamalıdır. Bu yöntemler, oluşturulan nesnenin kapsamı dışından çağrılabilir.

class Cat
  def initialize(name)
    @name = name
  end

  def speak
    puts "I'm #{@name} and I'm 2 years old"
  end
  
  ...
end

new_cat = Cat.new("garfield")
#=> <Cat:0x2321868 @name="garfield">
 
new_cat.speak
#=> I'm garfield and I'm 2 years old

Bu yöntemler genel yakut yöntemleridir, yeni bir kedi başlatma davranışını ve konuşma yönteminin davranışını tanımlarlar.

“public” anahtar sözcüğü gereksizdir, ancak “private” veya “protected” anahtar sözcüklerinden kaçmak için kullanılabilir

def MyClass
  def first_public_method
  end

  private

  def private_method
  end

  public

  def second_public_method
  end
end

Özel Yöntemler #

Özel yöntemlere nesnenin dışından erişilemez. Nesne tarafından dahili olarak kullanılırlar. Kedi örneğini tekrar kullanarak:

class Cat
  def initialize(name)
    @name = name
  end

  def speak
    age = calculate_cat_age # here we call the private method 
    puts "I'm #{@name} and I'm #{age} years old"
  end

  private
     def calculate_cat_age
       2 * 3 - 4 
     end
end

my_cat = Cat.new("Bilbo")
my_cat.speak #=> I'm Bilbo and I'm 2 years old
my_cat.calculate_cat_age #=> NoMethodError: private method `calculate_cat_age' called for #<Cat:0x2321868 @name="Bilbo">

Yukarıdaki örnekte görebileceğiniz gibi, yeni oluşturulan Cat nesnesinin dahili olarak calculate_cat_age yöntemine erişimi vardır. Kedinin adını ve yaşını konsola yazdıran özel ‘calculate_cat_age’ yöntemini çalıştırmanın sonucuna ‘age’ değişkenini atadık.

my_cat nesnesinin dışından Calculate_cat_age yöntemini çağırmaya çalıştığımızda, özel olduğu için bir NoMethodError alırız. Anla?

Korumalı Yöntemler #

Korumalı yöntemler özel yöntemlere çok benzer. Özel yöntemlerin olamayacağı şekilde, nesne örneğinin dışından erişilemezler. Ancak, self ruby ​​yöntemi kullanılarak, korumalı yöntemler aynı türdeki bir nesne bağlamında çağrılabilir.

class Cat
  def initialize(name, age)
    @name = name
    @age = age
  end

  def speak
    puts "I'm #{@name} and I'm #{@age} years old"
  end

  # this == method allows us to compare two objects own ages. 
  # if both Cat's have the same age they will be considered equal.
  def ==(other)
     self.own_age == other.own_age
  end

  protected
     def own_age
        self.age
     end
end

cat1 = Cat.new("ricky", 2)
=> #<Cat:0x007fe2b8aa4a18 @name="ricky", @age=2>

cat2 = Cat.new("lucy", 4)
=> #<Cat:0x008gfb7aa6v67 @name="lucy", @age=4>

cat3 = Cat.new("felix", 2)
=> #<Cat:0x009frbaa8V76 @name="felix", @age=2>

Kedi sınıfına bir yaş parametresi eklediğimizi ve ad ve yaş ile üç yeni kedi nesnesi oluşturduğumuzu görebilirsiniz. Kedi nesnelerimizin yaşlarını karşılaştırmak için ‘own_age’ korumalı yöntemini çağıracağız.

cat1 == cat2
=> false

cat1 == cat3
=> true

Şuna bakın, self.own_age korumalı yöntemini kullanarak cat1’in yaşını alabildik ve cat1’in içindeki cat2.own_age‘u çağırarak bunu cat2’nin yaşıyla karşılaştırabildik.

Alıcılar ve ayarlayıcılarla örnek değişkenlere erişme #

Üç yöntemimiz var:

  1. attr_reader: değişkenin sınıf dışında okunmasına izin vermek için kullanılır.

  2. attr_writer: değişkenin sınıf dışında değiştirilmesine izin vermek için kullanılır.

  3. attr_accessor: her iki yöntemi birleştirir.

    class Cat attr_reader :age # you can read the age but you can never change it attr_writer :name # you can change name but you are not allowed to read attr_accessor :breed # you can both change the breed and read it

    def initialize(name, breed) @name = name @breed = breed @age = 2 end def speak puts “I’m #{@name} and I am a #{@breed} cat” end end

    my_cat = Cat.new(“Banjo”, “birman”)

    reading values: #

    my_cat.age #=> 2 my_cat.breed #=> “birman” my_cat.name #=> Error

    changing values #

    my_cat.age = 3 #=> Error my_cat.breed = “sphynx” my_cat.name = “Bilbo”

    my_cat.speak #=> I’m Bilbo and I am a sphynx cat

Parametrelerin semboller olduğuna dikkat edin. bu bir yöntem oluşturarak çalışır.

class Cat
  attr_accessor :breed
end

Temelde şununla aynıdır:

class Cat
  def breed
    @breed
  end
  def breed= value
    @breed = value
  end
end

Sınıf Yöntemleri türleri #

Sınıfların 3 tür yöntemi vardır: örnek, tekil ve sınıf yöntemleri.

Örnek Yöntemleri #

Bunlar, sınıfın bir örneğinden çağrılabilen yöntemlerdir.

class Thing
  def somemethod
    puts "something"
  end
end

foo = Thing.new # create an instance of the class
foo.somemethod # => something

Sınıf Yöntemi #

Bunlar statik yöntemlerdir, yani o sınıfın bir örneğinde değil, sınıfta çağrılabilirler.

class Thing
  def Thing.hello(name)
    puts "Hello, #{name}!"
  end
end

Sınıf adı yerine ‘self’ kullanmak eşdeğerdir. Aşağıdaki kod, yukarıdaki koda eşdeğerdir:

class Thing
  def self.hello(name)
    puts "Hello, #{name}!"
  end
end

Yöntemi yazarak çağırın

Thing.hello("John Doe")  # prints: "Hello, John Doe!"

Tekton Yöntemleri #

Bunlar yalnızca sınıfın belirli örnekleri için kullanılabilir, ancak hepsi için geçerli değildir.

# create an empty class
class Thing
end

# two instances of the class
thing1 = Thing.new
thing2 = Thing.new

# create a singleton method
def thing1.makestuff
  puts "I belong to thing one"
end

thing1.makestuff # => prints: I belong to thing one
thing2.makestuff # NoMethodError: undefined method `makestuff' for #<Thing>

Hem “singleton” hem de “sınıf” yöntemlerine “eigenclass” adı verilir. Temel olarak, Ruby’nin yaptığı, oluşturulan örneklere müdahale etmeyecek şekilde bu tür yöntemleri içeren anonim bir sınıf oluşturmaktır.

Bunu yapmanın başka bir yolu da ‘class «’ yapıcısıdır. Örneğin:

# a class method (same as the above example)
class Thing
  class << self # the anonymous class
    def hello(name)
      puts "Hello, #{name}!"
    end
  end
end

Thing.hello("sarah") # => Hello, sarah!

# singleton method

class Thing
end

thing1 = Thing.new

class << thing1
  def makestuff
    puts "I belong to thing one"
  end
end

thing1.makestuff # => prints: "I belong to thing one"

Dinamik sınıf oluşturma #

Sınıflar, ‘Class.new’ kullanılarak dinamik olarak oluşturulabilir.

# create a new class dynamically
MyClass = Class.new

# instantiate an object of type MyClass
my_class = MyClass.new

Yukarıdaki örnekte, yeni bir sınıf oluşturulur ve ‘MyClass’ sabitine atanır. Bu sınıf, diğer herhangi bir sınıf gibi somutlaştırılabilir ve kullanılabilir.

‘Class.new’ yöntemi, dinamik olarak oluşturulan sınıfın üst sınıfı olacak bir ‘Sınıf’ı kabul eder.

# dynamically create a class that subclasses another
Staffy = Class.new(Dog)

# instantiate an object of type Staffy
lucky = Staffy.new
lucky.is_a?(Staffy) # true
lucky.is_a?(Dog)    # true

‘Class.new’ yöntemi de bir bloğu kabul eder. Bloğun bağlamı yeni oluşturulan sınıftır. Bu, yöntemlerin tanımlanmasına izin verir.

Duck = 
  Class.new do
    def quack
      'Quack!!'
    end
  end

# instantiate an object of type Duck
duck = Duck.new
duck.quack # 'Quack!!'

Yeni, tahsis et ve başlat #

Birçok dilde, özel bir “new” anahtar sözcüğü kullanılarak bir sınıfın yeni örnekleri oluşturulur. Ruby’de ’new’, bir sınıfın örneklerini oluşturmak için de kullanılır, ancak bu bir anahtar kelime değildir; bunun yerine, diğer herhangi bir statik/sınıf yönteminden farklı olmayan bir statik/sınıf yöntemidir. Tanım kabaca şudur:

class MyClass
   def self.new(*args)
     obj = allocate
     obj.initialize(*args) # oversimplied; initialize is actually private
     obj
   end
end

“ayırma”, sınıfın başlatılmamış bir örneğini yaratmanın gerçek “sihrini” gerçekleştirir

Ayrıca ‘initialize’ dönüş değerinin atıldığını ve bunun yerine obj’nin döndürüldüğünü unutmayın. Bu, en sonunda ‘self’i döndürme konusunda endişelenmeden başlatma yönteminizi neden kodlayabileceğinizi hemen netleştirir.

Tüm sınıfların ‘Class’tan aldığı ’normal’ ‘yeni’ yöntem yukarıdaki gibi çalışır, ancak bunu istediğiniz gibi yeniden tanımlamanız veya farklı çalışan alternatifler tanımlamanız mümkündür. Örneğin:

class MyClass
  def self.extraNew(*args)
    obj = allocate
    obj.pre_initialize(:foo)
    obj.initialize(*args)
    obj.post_initialize(:bar)
    obj
  end
end

Sınıf ve örnek değişkenleri #

Bir sınıfın verileri daha kolay paylaşmak için kullanabileceği birkaç özel değişken türü vardır.

Önünde “@” olan örnek değişkenleri. Aynı değişkeni farklı yöntemlerde kullanmak istiyorsanız kullanışlıdırlar.

class Person
  def initialize(name, age)
    my_age = age # local variable, will be destroyed at end of constructor
    @name = name # instance variable, is only destroyed when the object is
  end

  def some_method
    puts "My name is #{@name}." # we can use @name with no problem
  end
   
  def another_method
    puts "My age is #{my_age}." # this will not work!
  end
end

mhmd = Person.new("Mark", 23)

mhmd.some_method #=> My name is Mark.
mhmd.another_method #=> throws an error

Öncesinde “@@” olan sınıf değişkeni. Bir sınıfın tüm örneklerinde aynı değerleri içerirler.

class Person
  @@persons_created = 0 # class variable, available to all objects of this class
  def initialize(name)
    @name = name

    # modification of class variable persists across all objects of this class
    @@persons_created += 1
  end  
      
  def how_many_persons
    puts "persons created so far: #{@@persons_created}"
  end 
end
    
mark = Person.new("Mark")
mark.how_many_persons #=> persons created so far: 1
helen = Person.new("Helen")

mark.how_many_persons #=> persons created so far: 2
helen.how_many_persons #=> persons created so far: 2
# you could either ask mark or helen

Global Değişkenler, öncesinde “$”. Bunlar programın herhangi bir yerinde mevcuttur, bu yüzden bunları akıllıca kullandığınızdan emin olun.

$total_animals = 0

class Cat
  def initialize
    $total_animals += 1
  end
end

class Dog
  def initialize
    $total_animals += 1
  end
end

bob = Cat.new()
puts $total_animals #=> 1
fred = Dog.new()
puts $total_animals #=> 2
