Kuyruk özyineleme #

Birçok özyinelemeli algoritma yineleme kullanılarak ifade edilebilir. Örneğin, en büyük ortak payda işlevi yinelemeli olarak yazılabilir:

def gdc (x, y)
  return x if y == 0
  return gdc(y, x%y)
end

veya yinelemeli olarak:

def gdc_iter (x, y)
  while y != 0 do
    x, y = y, x%y
  end

  return x
end

İki algoritma teoride eşdeğerdir, ancak özyinelemeli sürüm bir SystemStackError riski taşır. Bununla birlikte, özyinelemeli yöntem kendisine bir çağrı ile sona erdiğinden, yığın taşmasını önlemek için optimize edilebilir. Bunu ifade etmenin başka bir yolu: özyinelemeli algoritma, derleyicinin yöntemin sonunda özyinelemeli yöntem çağrısını aramayı bildiği yinelemeli if ile aynı makine koduyla sonuçlanabilir. Ruby varsayılan olarak kuyruk çağrısı optimizasyonu yapmaz, ancak şununla açabilirsiniz:

RubyVM::InstructionSequence.compile_option = {
  tailcall_optimization: true,
  trace_instruction: false
}

Kuyruk çağrısı optimizasyonunu açmanın yanı sıra talimat izlemeyi de kapatmanız gerekir. Ne yazık ki, bu seçenekler yalnızca derleme zamanında geçerlidir, bu nedenle özyinelemeli yöntemi başka bir dosyadan “gerektirmeniz” veya yöntem tanımını “değerlendirmeniz” gerekir:

RubyVM::InstructionSequence.new(<<-EOF).eval
  def me_myself_and_i
    me_myself_and_i
  end
EOF
me_myself_and_i # Infinite loop, not stack overflow

Son olarak, son dönüş çağrısı yöntemi ve yalnızca yöntemi döndürmelidir. Bu, standart faktöriyel işlevi yeniden yazmanız gerekeceği anlamına gelir:

def fact(x)
  return 1 if x <= 1
  return x*fact(x-1)
end

Şunun gibi bir şeye:

 def fact(x, acc=1)
   return acc if x <= 1
   return fact(x-1, x*acc)
 end

Bu sürüm, birikmiş toplamı, [varsayılan](https://www.wikiod.com/tr/ruby/yontemler#Varsayılan parametreler) bir ikinci (isteğe bağlı) bağımsız değişken aracılığıyla iletir.

Daha fazla okuma: Ruby’de Kuyruk Çağrısı Optimizasyonu ve Tailin’ Ruby.

Özyinelemeli işlev #

Ruby’de özyinelemenin nasıl uygulanabileceğini görmek için basit bir algoritma ile başlayalım.

Bir fırının satacak ürünleri vardır. Ürünler paketlidir. Yalnızca paketlerdeki siparişlere hizmet verir. Paketleme, en büyük paket boyutundan başlar ve daha sonra kalan miktarlar, mevcut olan sonraki paket boyutlarıyla doldurulur.

Örneğin 16 adet sipariş alınırsa fırın 5’li paketten 2, 3’lü paketten 2 adet sipariş vermektedir. 25+23 = 16. Bunun özyinelemede nasıl uygulandığını görelim. “ayırma” burada özyinelemeli işlevdir.

#!/usr/bin/ruby

class Bakery
  attr_accessor :selected_packs

  def initialize
    @packs = [5,3] # pack sizes 5 and 3
    @selected_packs = []
  end

  def allocate(qty)
    remaining_qty = nil

    # ==============================================
    # packs are allocated in large packs first order
    # to minimize the packaging space
    # ==============================================
    @packs.each do |pack|
      remaining_qty = qty - pack

      if remaining_qty > 0
        ret_val = allocate(remaining_qty)
        if ret_val == 0
          @selected_packs << pack
          remaining_qty = 0
          break
        end
      elsif remaining_qty == 0
        @selected_packs << pack
        break
      end
    end

    remaining_qty
  end
end

bakery = Bakery.new
bakery.allocate(16)
puts "Pack combination is: #{bakery.selected_packs.inspect}"

Çıktı:

Paket kombinasyonu: [3, 3, 5, 5]
