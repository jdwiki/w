OptionParser “ARGV"den komut satırı seçeneklerini ayrıştırmak için kullanılabilir.

Zorunlu ve isteğe bağlı komut satırı seçenekleri #

Çok karmaşık bir şey aramıyorsanız, komut satırını elle ayrıştırmak nispeten kolaydır:

# Naive error checking
abort('Usage: ' + $0 + ' site id ...') unless ARGV.length >= 2

# First item (site) is mandatory
site = ARGV.shift

ARGV.each do | id |
  # Do something interesting with each of the ids
end

Ancak seçenekleriniz daha karmaşık hale gelmeye başladığında, muhtemelen OptionParser gibi bir seçenek ayrıştırıcı kullanmanız gerekecektir. :

require 'optparse'

# The actual options will be stored in this hash
options = {}

# Set up the options you are looking for
optparse = OptionParser.new do |opts|
  opts.banner = "Usage: #{$0} -s NAME id ..."

  opts.on("-s", "--site NAME", "Site name") do |s|
    options[:site] = s
  end

  opts.on( '-h', '--help', 'Display this screen' ) do
    puts opts
    exit
  end
end

# The parse! method also removes any options it finds from ARGV.
optparse.parse!

Ayrıca yıkıcı olmayan bir “ayrıştırma” da vardır, ancak “ARGV"de bulunanların geri kalanını kullanmayı planlıyorsanız çok daha az kullanışlıdır.

OptionParser sınıfının zorunlu bağımsız değişkenleri (bu durumda “–site” gibi) zorlamanın bir yolu yoktur. Ancak, parse! komutunu çalıştırdıktan sonra kendi kontrolünüzü yapabilirsiniz:

# Slightly more sophisticated error checking
if options[:site].nil? or ARGV.length == 0
  abort(optparse.help)
end

Daha genel bir zorunlu seçenek işleyicisi için bu yanıta bakın. Açık olmaması durumunda, zorunlu kılmadığınız sürece tüm seçenekler isteğe bağlıdır.

Varsayılan değerler #

OptionsParser ile varsayılan değerleri ayarlamak gerçekten çok kolay. Seçenekleri içinde sakladığınız hash’i önceden doldurmanız yeterlidir:

options = {
  :directory => ENV['HOME']
}

Ayrıştırıcıyı tanımladığınızda, bir kullanıcı bir değer sağlarsa varsayılanın üzerine yazar:

OptionParser.new do |opts|
  opts.on("-d", "--directory HOME", "Directory to use") do |d|
    options[:directory] = d
  end
end

Uzun açıklamalar #

Bazen açıklamanız oldukça uzun olabilir. Örneğin, ‘irb -h’, aşağıdakileri okuyan argümanı listeler:

  --context-mode n  Set n[0-3] to method to create Binding Object,
                    when new workspace was created

Bunun nasıl destekleneceği hemen belli değil. Çoğu çözüm, ikinci ve sonraki satırların girintisini birinciyle aynı hizaya getirmek için ayarlama gerektirir. Neyse ki, “on” yöntemi, bunları ayrı bağımsız değişkenler olarak ekleyerek birden çok açıklama satırını destekler:

  opts.on("--context-mode n",
          "Set n[0-3] to method to create Binding Object,",
          "when new workspace was created") do |n|
    optons[:context_mode] = n
  end

Seçeneği tam olarak açıklamak için istediğiniz kadar açıklama satırı ekleyebilirsiniz.
