Alt sınıflar #

Kalıtım, sınıfların mevcut bir sınıfa dayalı olarak belirli davranışları tanımlamasına olanak tanır.

class Animal
  def say_hello
    'Meep!'
  end

  def eat
    'Yumm!'
  end
end

class Dog < Animal
  def say_hello
    'Woof!'
  end
end

spot = Dog.new
spot.say_hello # 'Woof!'
spot.eat       # 'Yumm!'

Bu örnekte:

  • “Köpek”, “Hayvan"dan miras alarak onu bir Altsınıf yapar.
  • “Köpek”, “Hayvan"dan hem “say_hello” hem de “ye” yöntemlerini alır.
  • “Köpek”, farklı işlevlerle “say_hello” yöntemini geçersiz kılar.

Çoklu Kalıtım #

Çoklu kalıtım, bir sınıfın birden çok sınıftan (yani birden fazla ebeveynden) miras almasına izin veren bir özelliktir. Ruby, çoklu kalıtımı desteklemez. Yalnızca tek devralmayı destekler (yani sınıfın yalnızca bir ebeveyni olabilir), ancak Modüller kullanarak daha karmaşık sınıflar oluşturmak için composition kullanabilirsiniz.

Miras alınan nedir? #

Yöntemler miras alınır

class A
  def boo; p 'boo' end
end

class B < A; end

b = B.new
b.boo # => 'boo'

Sınıf yöntemleri miras alınır

class A
  def self.boo; p 'boo' end
end

class B < A; end

p B.boo # => 'boo'

Sabitler miras alınır

class A
  WOO = 1
end

class B < A; end

p B::WOO # => 1

Ancak dikkat edin, geçersiz kılınabilirler:

class B
  WOO = WOO + 1
end

p B::WOO # => 2

Örnek değişkenleri devralınır:

class A
  attr_accessor :ho
  def initialize
    @ho = 'haha'
  end
end

class B < A; end

b = B.new
p b.ho # => 'haha'

Dikkat edin, örnek değişkenlerini “süper” çağırmadan başlatan yöntemleri geçersiz kılarsanız, bunlar sıfır olacaktır. Yukarıdan devam:

class C < A
  def initialize; end
 end

c = C.new
p c.ho    # => nil

Sınıf örneği değişkenleri devralınmaz:

class A
    @foo = 'foo'
    class << self
        attr_accessor :foo
    end
end

class B < A; end

p B.foo # => nil

# The accessor is inherited, since it is a class method
#
B.foo = 'fob' # possible

Sınıf değişkenleri gerçekten miras alınmaz

Temel sınıf ve tüm alt sınıflar arasında 1 değişken olarak paylaşılırlar:

class A
    @@foo = 0
    def initialize
        @@foo  += 1 
        p @@foo
    end
end

class B < A;end

a = A.new # => 1
b = B.new # => 2

Yani yukarıdan devam etmek:

class C < A
  def initialize
    @@foo = -10
    p @@foo
  end
end

a = C.new # => -10
b = B.new # => -9

Karışımlar #

Mixins, çoklu kalıtıma benzer bir şey elde etmenin güzel bir yoludur. Bir modülde tanımlanan yöntemleri miras almamıza veya daha doğrusu bir sınıfa dahil etmemize izin verir. Bu yöntemler, örnek veya sınıf yöntemleri olarak dahil edilebilir. Aşağıdaki örnek bu tasarımı göstermektedir.

module SampleModule

  def self.included(base)
    base.extend ClassMethods
  end

  module ClassMethods

    def method_static
      puts "This is a static method"
    end

  end

  def insta_method
    puts "This is an instance method"
  end

end

class SampleClass
  include SampleModule
end

sc = SampleClass.new

sc.insta_method

prints "This is an instance method"

sc.class.method_static

prints "This is a static method"

Kalıtım kullanmak için mevcut sınıfları yeniden düzenleme #

Diyelim ki iki sınıfımız var, ‘Cat’ ve ‘Köpek’.

class Cat
  def eat
    die unless has_food?
    self.food_amount -= 1
    self.hungry = false
  end
  def sound
    puts "Meow"
  end
end

class Dog
  def eat
    die unless has_food?
    self.food_amount -= 1
    self.hungry = false
  end
  def sound
    puts "Woof"
  end
end

’eat’ yöntemi bu iki sınıfta tamamen aynıdır. Bu çalışırken, bakımı zordur. Aynı “ye” yöntemine sahip daha fazla hayvan varsa sorun daha da kötüleşecektir. Miras bu sorunu çözebilir.

class Animal
  def eat
    die unless has_food?
    self.food_amount -= 1
    self.hungry = false
  end
  # No sound method
end

class Cat < Animal
  def sound
    puts "Meow"
  end
end

class Dog < Animal
  def sound
    puts "Woof"
  end
end

‘Animal’ adında yeni bir sınıf yarattık ve ’eat’ metodumuzu o sınıfa taşıdık. Daha sonra, ‘Cat’ ve ‘Dog’ öğelerini bu yeni ortak üst sınıftan miras aldık. Bu, kodu tekrarlama ihtiyacını ortadan kaldırır
