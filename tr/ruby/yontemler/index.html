Ruby’deki işlevler, bir dizi eylemi önceden oluşturmak için organize, yeniden kullanılabilir kod sağlar. Fonksiyonlar kodlama sürecini basitleştirir, gereksiz mantığı önler ve kodun takip edilmesini kolaylaştırır. Bu konu, Ruby’de işlevlerin, bağımsız değişkenlerin, parametrelerin, verim ifadelerinin ve kapsamın bildirilmesini ve kullanımını açıklar.

Bir yöntem, bir veya daha fazla nesneyle ilişkilendirilen ve genellikle ada ek olarak bir parametre listesiyle tanımlanan adlandırılmış bir kod bloğudur.

def hello(name)
  "Hello, #{name}"
end

Bir yöntem çağırma, yöntem adını, çağrılacak nesneyi (bazen alıcı olarak adlandırılır) ve adlandırılmış yöntem parametrelerine atanan sıfır veya daha fazla bağımsız değişken değerini belirtir. Yöntemde değerlendirilen son ifadenin değeri, yöntem çağırma ifadesinin değeri olur.

hello("World")
# => "Hello, World"

Alıcı açık olmadığında, ‘kendi’dir.

self
# => main

self.hello("World")
# => "Hello, World"

Ruby Programlama Dili kitabında açıklandığı gibi, birçok dil, ilişkili nesnesi olmayan işlevler ile bir alıcı nesnesinde çağrılan yöntemler arasında ayrım yapar. Ruby tamamen nesne yönelimli bir dil olduğundan, tüm yöntemler gerçek yöntemlerdir ve en az bir nesneyle ilişkilendirilir.

Yöntem Parametrelerine Genel Bakış #

Tip Yöntem İmza Çağrı Örneği Ödevler
Ggerekli def fn(a,b,c) fn(2,3,5) `a=2, b=3, c=5’
Varyadik def fn(*rest) fn(2,3,5) dinlenme=[2, 3, 5]
Dvarsayılan def fn(a=0,b=1) fn(2,3) ‘a=2, b=3’
Kanahtar kelime def fn(a:0,b:1) fn(a:2,b:3) ‘a=2, b=3’

Bu argüman türleri, değişken işlevler oluşturmak için hayal edebileceğiniz hemen hemen her şekilde birleştirilebilir. İşlev için minimum argüman sayısı, imzadaki gerekli argümanların miktarına eşit olacaktır. Önce varsayılan parametrelere, ardından *rest parametresine ekstra argümanlar atanacaktır.

Tip Yöntem İmza Çağrı Örneği Ödevler
R,D,V,R def fn(a,b=1,*mid,z) fn(2,97) `a=2, b=1, orta=[], z=97'
“fn(2,3,97)” `a=2, b=3, orta=[], z=97'
fn(2,3,5,97) `a=2, b=3, orta=[5], z=97'
fn(2,3,5,7,97) `a=2, b=3, orta=[5,7], z=97'
R,K,K def fn(a,g:6,h:7) fn(2) `a=2, g=6, h=7'
fn(2,h:19) a=2, g=6, h=19
fn(2,g:17,h:19) `a=2, g=17, h=19'
VK def fn(**ks) fn(a:2,g:17,h:19) ks={a:2, g:17, s:19}
fn(dört:4,beş:5) ks={dört:4, beş:5}

Bloklara verim #

Yönteminize bir blok gönderebilir ve bu bloğu birden çok kez çağırabilir. Bu, bir proc/lambda veya benzeri göndererek yapılabilir, ancak “verim” ile daha kolay ve hızlıdır:

def simple(arg1,arg2)
  puts "First we are here:  #{arg1}"
  yield
  puts "Finally we are here:  #{arg2}"
  yield
end
simple('start','end') { puts "Now we are inside the yield" }

#> First we are here:  start
#> Now we are inside the yield
#> Finally we are here:  end
#> Now we are inside the yield

{ puts ... } ifadesinin parantez içinde olmadığına, üstü kapalı olarak sonra geldiğine dikkat edin. Bu aynı zamanda yalnızca bir “verim” bloğuna sahip olabileceğimiz anlamına gelir. Argümanları “verim"e aktarabiliriz:

def simple(arg)
  puts "Before yield"
  yield(arg)
  puts "After yield"
end
simple('Dave') { |name| puts "My name is #{name}" }

#> Before yield
#> My name is Dave
#> After yield

Verim ile, yineleyicileri veya diğer kodlarda çalışan herhangi bir işlevi kolayca yapabiliriz:

def countdown(num)
  num.times do |i|
    yield(num-i)
  end
end

geri sayım(5) { |i| "Arama numarası #{i}" } koyar

#> Call number 5
#> Call number 4
#> Call number 3
#> Call number 2
#> Call number 1

Aslında, ‘foreach’, ’each’ ve ’times’ gibi şeyler genellikle sınıflarda ‘verim’ ile uygulanır.

Size bir blok verilip verilmediğini öğrenmek istiyorsanız, block_given? kullanın:

class Employees
  def names
    ret = []
    @employees.each do |emp|
      if block_given?
        yield(emp.name)
      else
        ret.push(emp.name) 
      end
    end

    ret
  end
end

Bu örnek, “Çalışanlar” sınıfının, “name” yöntemini kullanarak çalışan adlarına sahip nesneleri almak için “her” ile yinelenebilen bir “@employees” listesine sahip olduğunu varsayar. Bize bir blok verilirse, o zaman bloğa adını veririz, aksi takdirde onu döndürdüğümüz bir diziye iteriz.

Varsayılan parametreler #

def make_animal_sound(sound = 'Cuack')
    puts sound
end

#

make_animal_sound('Mooo') # Mooo
make_animal_sound         # Cuack

Birden çok bağımsız değişken için varsayılanları dahil etmek mümkündür:

def make_animal_sound(sound = 'Cuack', volume = 11)
    play_sound(sound, volume)
end

make_animal_sound('Mooo') # Spinal Tap cow

Ancak, ikincisini tedarik etmek mümkün değildir. ilk-be) ayrıca ilkini sağlamadan. Konum parametreleri kullanmak yerine anahtar kelime parametrelerini deneyin:

def make_animal_sound(sound: 'Cuack', volume: 11)
    play_sound(sound, volume)
end

make_animal_sound(volume: 1) # Duck whisper

Veya seçenekleri saklayan bir karma parametre:

def make_animal_sound(options = {})
    options[:sound]  ||= 'Cuak'
    options[:volume] ||= 11
    play_sound(sound, volume)
end

make_animal_sound(:sound => 'Mooo')

Varsayılan parametre değerleri herhangi bir Ruby ifadesi ile ayarlanabilir. İfade, yöntem bağlamında çalışacaktır, böylece burada yerel değişkenleri bile bildirebilirsiniz. Not, kod incelemesinden geçmeyecek. [bunu işaret ettiği için] caius’un izniyle 1.

def make_animal_sound( sound = ( raise 'TUU-too-TUU-too...' ) ); p sound; end

make_animal_sound 'blaaaa' # => 'blaaaa'
make_animal_sound          # => TUU-too-TUU-too... (RuntimeError)

İsteğe bağlı parametre(ler) (splat operatörü) #

def welcome_guests(*guests)
    guests.each { |guest| puts "Welcome #{guest}!" }
end

#

welcome_guests('Tom')    # Welcome Tom!
welcome_guests('Rob', 'Sally', 'Lucas') # Welcome Rob!
                                        # Welcome Sally!
                                        # Welcome Lucas!

welcome_guests(['Rob', 'Sally', 'Lucas']) ifadesinin Hoş Geldiniz ["Rob", "Sally", "Lucas"]!' sonucunu vereceğini unutmayın. Bunun yerine, bir listeniz varsa, 'welcome_guests(*['Rob', 'Sally', 'Lucas'])' yapabilirsiniz ve bu 'welcome_guests('Rob', 'Sally', 'Lucas') olarak çalışacaktır. .

Yöntem tanımlama #

Yöntemler, ‘def’ anahtar sözcüğüyle tanımlanır, ardından parantez içinde method name ve isteğe bağlı bir parameter name listesi gelir. ‘def’ ve ’end’ arasındaki Ruby kodu, yöntemin body değerini temsil eder.

def hello(name)
  "Hello, #{name}"
end

Bir yöntem çağırma, yöntem adını, çağrılacak nesneyi (bazen alıcı olarak adlandırılır) ve adlandırılmış yöntem parametrelerine atanan sıfır veya daha fazla bağımsız değişken değerini belirtir.

hello("World")
# => "Hello, World"

Alıcı açık olmadığında, ‘kendi’dir.

Parametre adları, yöntem gövdesi içinde değişkenler olarak kullanılabilir ve bu adlandırılmış parametrelerin değerleri, argümanlardan bir yöntem çağrısına gelir.

hello("World")
# => "Hello, World"
hello("All")
# => "Hello, All"

Tek gerekli parametre #

def say_hello_to(name)
    puts "Hello #{name}"
end

#

say_hello_to('Charles')    # Hello Charles

Gerekli varsayılan isteğe bağlı parametre karışımı #

def my_mix(name,valid=true, *opt)
    puts name
    puts valid
    puts opt
end

Aşağıdaki gibi arayın:

my_mix('me')
# 'me'
# true
# []

my_mix('me', false)
# 'me'
# false
# []

my_mix('me', true, 5, 7) 
# 'me'
# true
# [5,7]

Bir işlevi blok olarak kullanın #

Ruby’deki birçok fonksiyon bir bloğu argüman olarak kabul eder. Örneğin.:

[0, 1, 2].map {|i| i + 1}
 => [1, 2, 3]

Zaten istediğinizi yapan bir fonksiyonunuz varsa, bunu &method(:fn) kullanarak bir bloğa dönüştürebilirsiniz:

def inc(num)
   num + 1
end

[0, 1, 2].map &method(:inc)
 => [1, 2, 3]

Demet Argümanları #

Bir yöntem, bir dizi parametresi alabilir ve onu hemen adlandırılmış yerel değişkenlere dönüştürebilir. Mathias Meyer’in blogunda bulundu.

def feed( amount, (animal, food) )

    p "#{amount} #{animal}s chew some #{food}"

end

feed 3, [ 'rabbit', 'grass' ] # => "3 rabbits chew some grass"

Birden çok gerekli parametre #

def greet(greeting, name)
    puts "#{greeting} #{name}"
end

#

greet('Hi', 'Sophie')    # Hi Sophie

Yöntem Tanımları İfadelerdir #

Ruby 2.x’te bir yöntem tanımlamak, adı temsil eden bir sembol döndürür:

class Example
  puts def hello
  end
end

#=> :hello

Bu, ilginç metaprogramlama tekniklerine izin verir. Örneğin, yöntemler diğer yöntemlerle sarılabilir:

class Class
  def logged(name)
    original_method = instance_method(name)
    define_method(name) do |*args|
      puts "Calling #{name} with #{args.inspect}."
      original_method.bind(self).call(*args)
      puts "Completed #{name}."
    end
  end
end

class Meal
  def initialize
    @food = []
  end
  
  logged def add(item)
    @food << item
  end
end

meal = Meal.new
meal.add "Coffee"
# Calling add with ["Coffee"].
# Completed add.

Bildirilmemiş anahtar kelime argümanlarını yakalama (çift uyarı) #

** operatörü, * operatörüne benzer şekilde çalışır ancak anahtar kelime parametreleri için geçerlidir.

def options(required_key:, optional_key: nil, **other_options)
  other_options
end

options(required_key: 'Done!', foo: 'Foo!', bar: 'Bar!')
#> { :foo => "Foo!", :bar => "Bar!" }

Yukarıdaki örnekte, **other_options kullanılmazsa, bir ArgumentError: bilinmeyen anahtar kelime: foo, bar hatası ortaya çıkar.

def without_double_splat(required_key:, optional_key: nil)
  # do nothing
end

without_double_splat(required_key: 'Done!', foo: 'Foo!', bar: 'Bar!')
#> ArgumentError: unknown keywords: foo, bar

Bu, bir metoda geçmek istediğiniz ve anahtarları filtrelemek istemediğiniz seçeneklerin bir karması olduğunda kullanışlıdır.

def options(required_key:, optional_key: nil, **other_options)
  other_options
end

my_hash = { required_key: true, foo: 'Foo!', bar: 'Bar!' }

options(my_hash)
#> { :foo => "Foo!", :bar => "Bar!" }

** operatörünü kullanarak bir hash’i açmak da mümkündür. Bu, diğer karmalardan gelen değerlere ek olarak, bir yönteme doğrudan anahtar kelime sağlamanıza olanak tanır:

my_hash = { foo: 'Foo!', bar: 'Bar!' }

options(required_key: true, **my_hash)
#> { :foo => "Foo!", :bar => "Bar!" }
