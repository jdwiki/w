Yakut taşları yükleme #

Bu kılavuz, Ruby’nin zaten kurulu olduğunu varsayar. Ruby < 1.9 kullanıyorsanız, yerel olarak dahil edilmeyeceğinden manuel olarak RubyGems yüklemeniz gerekir.

Bir Ruby gem yüklemek için şu komutu girin:

gem install [gemname]

Mücevher bağımlılıkları listesi olan bir proje üzerinde çalışıyorsanız, bunlar ‘Gemfile’ adlı bir dosyada listelenecektir. Projeye yeni bir gem yüklemek için, “Gemfile” içine aşağıdaki kod satırını ekleyin:

gem 'gemname'

Bu “Gemfile”, projenizin gerektirdiği bağımlılıkları yüklemek için Bundler gem tarafından kullanılır, ancak bu, önce Bundler’ı çalıştırarak (henüz yapmadıysanız) kurmanız gerektiği anlamına gelir:

gem install bundler

Dosyayı kaydedin ve ardından şu komutu çalıştırın:

bundle install

Sürümleri belirtme #

Sürüm numarası, canlı komutta “-v” bayrağıyla belirtilebilir, örneğin:

gem install gemname -v 3.14

Bir ‘Gemfile’ içinde sürüm numaralarını belirtirken, birkaç seçeneğiniz vardır:

  • Sürüm belirtilmemiş (gem 'gemname') – Gemfile içindeki diğer değerli taşlarla uyumlu olan en son sürümü yükleyecektir.
  • Tam sürüm belirtildi (“gem ‘gemname’, ‘3.14’’) – Yalnızca ‘3.14’ sürümünü yüklemeye çalışır (ve bu, ‘Gemfile’ içindeki diğer değerli taşlarla uyumlu değilse başarısız olur).
  • İyimser minimum sürüm numarası (gem 'gemname', '>=3.14'') -- Yalnızca 'Gemfile' içindeki diğer değerli taşlarla uyumlu olan *en son* sürümü yüklemeye çalışır ve aşağıdaki durumlarda başarısız olur: "3.14"e eşit veya daha büyük hiçbir sürüm uyumlu değildir. >` operatörü de kullanılabilir.
  • Karamsar minimum sürüm numarası (`gem ‘gemname’, ‘~>3.14’’) – Bu, işlevsel olarak ‘gem ‘gemname’, ‘>=3.14’, ‘<4’’ kullanımına eşdeğerdir. Başka bir deyişle, yalnızca son periyottan sonraki sayının artmasına izin verilir.

En iyi uygulama olarak: rbenv veya rvm gibi Ruby sürüm yönetimi kitaplıklarından birini kullanmak isteyebilirsiniz. /rvm/rvm). Bu kitaplıklar aracılığıyla, buna göre Ruby çalışma zamanlarının ve mücevherlerin farklı sürümlerini yükleyebilirsiniz. Bu nedenle, bir projede çalışırken, projelerin çoğu bilinen bir Ruby sürümüne karşı kodlandığından bu özellikle kullanışlı olacaktır.

github/dosya sisteminden gem kurulumu #

Github’dan veya dosya sisteminden bir mücevher yükleyebilirsiniz. Mücevher git’ten teslim alındıysa veya bir şekilde dosya sisteminde zaten varsa, onu kullanarak yükleyebilirsiniz.

gem install --local path_to_gem/filename.gem

github’dan gem yükleme. Kaynakları github’dan indirin

mkdir newgem
cd newgem
git clone https://urltogem.git

Mücevheri inşa et

gem build GEMNAME.gemspec
gem install gemname-version.gem

Gerekli bir gem’in kod içinden yüklenip yüklenmediğini kontrol etme #

Gerekli bir gem’in kurulu olup olmadığını kontrol etmek için kodunuzdan aşağıdakileri kullanabilirsiniz (örnek olarak nokogiri kullanarak):

begin
  found_gem = Gem::Specification.find_by_name('nokogiri')
  require 'nokogiri'
  ....
  <the rest of your code>
rescue Gem::LoadError
end

Bununla birlikte, bu, kodunuzdaki işlevlerin ayarlanmasında kullanılabilecek bir işleve genişletilebilir.

def gem_installed?(gem_name)
  found_gem = false
  begin
    found_gem = Gem::Specification.find_by_name(gem_name)
  rescue Gem::LoadError
     return false
  else
    return true
  end
end

Artık gerekli gem’in kurulu olup olmadığını kontrol edebilir ve bir hata mesajı yazdırabilirsiniz.

if gem_installed?('nokogiri')
  require 'nokogiri'
else
  printf "nokogiri gem required\n"
  exit 1
end

veya

if gem_installed?('nokogiri')
  require 'nokogiri'
else
  require 'REXML'
end

Gemfile ve Bundler Kullanma #

Bir “Gemfile”, uygulamanızdaki bağımlılıkları düzenlemenin standart yoludur. Temel bir Gemfile şöyle görünecektir:

source 'https://rubygems.org'

gem 'rack'
gem 'sinatra'
gem 'uglifier'

İstediğiniz gem sürümlerini aşağıdaki gibi belirtebilirsiniz:

# Match except on point release. Use only 1.5.X
gem 'rack', '~>1.5.2'
# Use a specific version.
gem 'sinatra', '1.4.7'
# Use at least a version or anything greater.
gem 'uglifier', '>= 1.3.0'

Mücevherleri doğrudan bir git deposundan da çekebilirsiniz:

# pull a gem from github
gem 'sinatra', git: 'https://github.com/sinatra/sinatra.git'
# you can specify a sha
gem 'sinatra', git: 'https://github.com/sinatra/sinatra.git', sha: '30d4fb468fd1d6373f82127d845b153f17b54c51'
# you can also specify a branch, though this is often unsafe
gem 'sinatra', git: 'https://github.com/sinatra/sinatra.git', branch: 'master'

Taşları ne için kullanıldıklarına göre de gruplandırabilirsiniz. Örneğin:

group :development, :test do
    # This gem is only available in dev and test, not production.
    gem 'byebug'
end

Uygulamanızın birden fazla platformda çalışması gerekiyorsa, belirli taşların hangi platformda çalışması gerektiğini belirleyebilirsiniz. Örneğin:

platform :jruby do
  gem 'activerecord-jdbc-adapter'
  gem 'jdbc-postgres'
end

platform :ruby do
  gem 'pg'
end

Tüm mücevherleri bir Gemfile’den yüklemek için şunları yapın:

gem install bundler
bundle install

Paketleyici/satır içi (paketleyici v1.10 ve üstü) #

Bazen birisi için bir senaryo yazmanız gerekir ama onun makinesinde ne olduğundan emin değilsiniz. Senaryonuzun ihtiyaç duyduğu her şey var mı? Endişe etmeyin. Bundler, line olarak adlandırılan harika bir işleve sahiptir.

Bir “gemfile” yöntemi sağlar ve komut dosyası çalıştırılmadan önce indirilir ve gerekli tüm değerli taşları gerektirir. Küçük bir örnek:

require 'bundler/inline' #require only what you need

#Start the bundler and in it use the syntax you are already familiar with
gemfile(true) do 
  source 'https://rubygems.org'
        gem 'nokogiri', '~> 1.6.8.1'
        gem 'ruby-graphviz'
end
