Temel Hata İşleme Bloğu #

Girdisine çok güvenen iki sayıyı bölmek için bir fonksiyon yapalım:

def divide(x, y)
  return x/y
end

Bu, birçok girdi için iyi çalışacaktır:

> puts divide(10, 2)
5

Fakat hepsi değil

> puts divide(10, 0)
ZeroDivisionError: divided by 0

> puts divide(10, 'a')
TypeError: String can't be coerced into Fixnum

Riskli bölme işlemini hataları kontrol etmek için bir ‘başlangıç… bitiş’ bloğuna sararak ve bir mesaj çıkarmak için bir ‘kurtarma’ yan tümcesi kullanarak ve bir sorun varsa ’nil’ döndürerek işlevi yeniden yazabiliriz.

def divide(x, y)
  begin
    return x/y
  rescue
    puts "There was an error"
    return nil
  end
end

> puts divide(10, 0)
There was an error

> puts divide(10, 'a')
There was an error

def divide(x, y)
  begin

    puts "There was a %s (%s)" % [e.class, e.message]
    puts e.backtrace

end

Farklı Hataları Kontrol Etme #

Hatanın türüne göre farklı şeyler yapmak istiyorsanız, argüman olarak her biri farklı bir hata türüne sahip birden çok “kurtarma” cümlesi kullanın.

def divide(x, y)
  begin
    return x/y
  rescue ZeroDivisionError
    puts "Don't divide by zero!"
    return nil
  rescue TypeError
    puts "Division only works on numbers!"
    return nil
  end
end

> divide(10, 0)
Don't divide by zero!

> divide(10, 'a')
Division only works on numbers!

Hatayı “kurtarma” bloğunda kullanmak üzere kaydetmek istiyorsanız:

rescue ZeroDivisionError => e

Başka bir “kurtarma” yan tümcesinde belirtilmeyen türdeki hataları yakalamak için bağımsız değişken içermeyen bir “kurtarma” yan tümcesi kullanın.

def divide(x, y)
  begin
    return x/y
  rescue ZeroDivisionError
    puts "Don't divide by zero!"
    return nil
  rescue TypeError
    puts "Division only works on numbers!"
    return nil
  rescue => e
    puts "Don't do that (%s)" % [e.class]
    return nil
  end
end

> divide(nil, 2)
Don't do that (NoMethodError)

Bu durumda, “nil"i 2’ye bölmeye çalışmak “ZeroDivisionError” veya “TypeError” değildir, bu nedenle varsayılan “kurtarma” yan tümcesi tarafından işlenir ve bize bunun bir “NoMethodError” olduğunu bildiren bir mesaj yazdırır. `.

Yeniden deneniyor #

Bir “kurtarma” yan tümcesinde, muhtemelen hataya neden olan koşulu değiştirdikten sonra, “başlama” yan tümcesini yeniden çalıştırmak için “yeniden dene"yi kullanabilirsiniz.

def divide(x, y)
  begin
    puts "About to divide..."
    return x/y
  rescue ZeroDivisionError
    puts "Don't divide by zero!"
    y = 1
    retry
  rescue TypeError
    puts "Division only works on numbers!"
    return nil
  rescue => e
    puts "Don't do that (%s)" % [e.class]
    return nil
  end
end

TypeError’a neden olacağını bildiğimiz parametreleri iletirsek, ‘begin’ yan tümcesi çalıştırılır (burada “Bölünmek üzere” yazdırılarak işaretlenir) ve hata daha önce olduğu gibi yakalanır ve ’nil’ döndürülür:

> divide(10, 'a')
About to divide...
Division only works on numbers!
 => nil

Ancak ‘ZeroDivisionError’a neden olacak parametreleri geçersek, ‘begin’ yan tümcesi yürütülür, hata yakalanır, bölen 0’dan 1’e değiştirilir ve ardından ‘retry’, ‘begin’ bloğunun yeniden çalıştırılmasına neden olur ( üstten), şimdi farklı bir “y” ile. İkinci seferde hata olmaz ve fonksiyon bir değer döndürür.

> divide(10, 0)
About to divide...     # First time, 10 ÷ 0
Don't divide by zero!
About to divide...     # Second time 10 ÷ 1
=> 10

Hata Oluşup Oluşmadığını Kontrol Etme #

Herhangi bir hata oluşmazsa çalıştırılacak kod için bir ’else’ yan tümcesi kullanabilirsiniz.

def divide(x, y)
  begin
    z = x/y
  rescue ZeroDivisionError
    puts "Don't divide by zero!"
  rescue TypeError
    puts "Division only works on numbers!"
    return nil
  rescue => e
    puts "Don't do that (%s)" % [e.class]
    return nil
  else
    puts "This code will run if there is no error."
    return z
  end
end

Kontrolü “kurtarma” maddelerinden birine aktaran bir hata varsa “else” maddesi çalışmaz:

> divide(10,0)
Don't divide by zero!
=> nil

Ancak herhangi bir hata oluşmazsa, “else” yan tümcesi şu şekilde yürütülür:

> divide(10,2)
This code will run if there is no error.
=> 5

“Başlangıç” tümcesinden dönerseniz, ’else’ yan tümcesinin yürütülmeyeceğini unutmayın.

def divide(x, y)
  begin
    z = x/y
    return z                 # Will keep the else clause from running!
  rescue ZeroDivisionError
    puts "Don't divide by zero!"
  else
    puts "This code will run if there is no error."
    return z
  end
end

> divide(10,2)
=> 5

Her Zaman Çalıştırılması Gereken Kod #

Her zaman yürütmek istediğiniz bir kod varsa, bir “sağla” yan tümcesi kullanın.

def divide(x, y)
  begin
    z = x/y
    return z
  rescue ZeroDivisionError
    puts "Don't divide by zero!"
  rescue TypeError
    puts "Division only works on numbers!"
    return nil
  rescue => e
    puts "Don't do that (%s)" % [e.class]
    return nil
  ensure
    puts "This code ALWAYS runs."
  end
end

Bir hata olduğunda ’ensure’ yan tümcesi yürütülür:

> divide(10, 0)
Don't divide by zero!   # rescue clause
This code ALWAYS runs.   # ensure clause
=> nil

Ve hata olmadığında:

> divide(10, 2)
This code ALWAYS runs.   # ensure clause
=> 5

Teminat yan tümcesi, örneğin dosyaların kapalı olduğundan emin olmak istediğinizde kullanışlıdır.

“else” yan tümcesinden farklı olarak, “ensure” yan tümcesinin, “begin” veya “kurtarma” yan tümceleri bir değer döndürmeden önce yürütüldüğünü unutmayın. Eğer ’ensure’ yan tümcesi, diğer tümcelerin ‘dönüş’ değerini geçersiz kılacak bir ‘dönüş’ içeriyorsa!
