Sözdizimi #

  • $global_variable
  • @@class_variable
  • @örnek değişkeni
  • yerel değişken

Sınıf değişkenleri, sınıf hiyerarşisinde paylaşılır. Bu şaşırtıcı davranışlara neden olabilir.

class A
  @@variable = :x

  def self.variable
    @@variable
  end
end

class B < A
  @@variable = :y
end

A.variable  # :y

Sınıflar nesnelerdir, bu nedenle her sınıfa özgü durumu sağlamak için örnek değişkenler kullanılabilir.

class A
  @variable = :x

  def self.variable
    @variable
  end
end

class B < A
  @variable = :y
end

A.variable  # :x

Sınıf Değişkenleri #

Sınıf değişkenlerinin sınıf çapında bir kapsamı vardır, sınıfın herhangi bir yerinde bildirilebilirler. Bir değişken, önüne `@@’ eklendiğinde bir sınıf değişkeni olarak kabul edilecektir.

class Dinosaur
    @@classification = "Like a Reptile, but like a bird"
    
    def self.classification
        @@classification
    end

    def classification
        @@classification
    end
end

dino = Dinosaur.new
dino.classification
# => "Like a Reptile, but like a bird"
  
Dinosaur.classification
# => "Like a Reptile, but like a bird"

Sınıf değişkenleri ilgili sınıflar arasında paylaşılır ve bir alt sınıftan bunların üzerine yazılabilir

class TRex < Dinosaur
    @@classification = "Big teeth bird!"
end

TRex.classification
# => "Big teeth bird!"

Dinosaur.classification
# => "Big teeth bird!"

Bu davranış çoğu zaman istenmeyen bir durumdur ve sınıf düzeyinde örnek değişkenleri kullanılarak atlatılabilir.

Bir modül içinde tanımlanan sınıf değişkenleri, dahil oldukları sınıf sınıf değişkenlerinin üzerine yazmaz:

module SomethingStrange
    @@classification = "Something Strange"
end

class DuckDinosaur < Dinosaur
    include SomethingStrange
end

DuckDinosaur.class_variables
# => [:@@classification]
SomethingStrange.class_variables
# => [:@@classification]

DuckDinosaur.classification
# => "Big teeth bird!"

Yerel Değişkenler #

Yerel değişkenler (diğer değişken sınıflarından farklı olarak) herhangi bir ön eke sahip değildir.

local_variable = "local"
p local_variable
# => local

Kapsamı nerede bildirildiğine bağlıdır, “bildiri kapsayıcıları” kapsamı dışında kullanılamaz. Örneğin, bir yöntemde yerel bir değişken bildirilmişse, yalnızca o yöntemin içinde kullanılabilir.

def some_method
    method_scope_var = "hi there"
    p method_scope_var
end

some_method
# hi there
# => hi there

method_scope_var
# NameError: undefined local variable or method `method_scope_var'

Tabii ki, yerel değişkenler yöntemlerle sınırlı değildir, genel bir kural olarak, bir do ... end bloğu içinde veya küme parantezleri {} içinde bir değişken bildirir tanımlamaz, bunun olacağını söyleyebilirsiniz. yerel ve içinde bildirildiği bloğa kapsamlı.

2.times do |n|
    local_var = n + 1
    p local_var
end
# 1
# 2
# => 2

local_var
# NameError: undefined local variable or method `local_var'

Ancak, “if” veya “case” bloklarında bildirilen yerel değişkenler üst kapsamda kullanılabilir:

if true
    usable = "yay"
end

p usable
# yay
# => "yay"

Yerel değişkenler, bildirim bloğunun dışında kullanılamazken, bloklara aktarılacaktır:

my_variable = "foo"

my_variable.split("").each_with_index do |char, i|
    puts "The character in string '#{my_variable}' at index #{i} is #{char}"
end
# The character in string 'foo' at index 0 is f
# The character in string 'foo' at index 1 is o
# The character in string 'foo' at index 2 is o
# => ["f", "o", "o"]

Ancak yöntem/sınıf/modül tanımlarına değil

my_variable = "foo"

def some_method
    puts "you can't use the local variable in here, see? #{my_variable}"
end

some_method
# NameError: undefined local variable or method `my_variable'

Blok argümanları için kullanılan değişkenler (elbette) blok için yereldir, ancak önceden tanımlanmış değişkenlerin üzerine yazmadan onları gölgede bırakacaktır.

overshadowed = "sunlight"

["darkness"].each do |overshadowed|
    p overshadowed
end
# darkness
# => ["darkness"]

p overshadowed
# "sunlight"
# => "sunlight"

Genel Değişkenler #

Global değişkenlerin global bir kapsamı vardır ve bu nedenle her yerde kullanılabilir. Kapsamları, tanımlandıkları yere bağlı değildir. Bir değişken, başına “$” işareti eklendiğinde global olarak kabul edilecektir.

$i_am_global = "omg"

class Dinosaur
    def instance_method
       p "global vars can be used everywhere. See? #{$i_am_global}, #{$another_global_var}" 
    end

    def self.class_method
       $another_global_var = "srsly?"
       p "global vars can be used everywhere. See? #{$i_am_global}"
    end
end

Dinosaur.class_method
# "global vars can be used everywhere. See? omg"
# => "global vars can be used everywhere. See? omg"

dinosaur = Dinosaur.new
dinosaur.instance_method
# "global vars can be used everywhere. See? omg, srsly?"
# => "global vars can be used everywhere. See? omg, srsly?"

Global bir değişken her yerde tanımlanabileceğinden ve her yerde görülebileceğinden, “tanımsız” bir global değişkeni çağırmak, bir hata oluşturmak yerine nil döndürür.

p $undefined_var
# nil
# => nil

Global değişkenlerin kullanımı kolay olsa da, sabitler lehine kullanımı kesinlikle önerilmez.

Örnek değişkenler #

Örnek değişkenlerin geniş bir nesne kapsamı vardır, bunlar nesnenin herhangi bir yerinde bildirilebilir, ancak sınıf düzeyinde bildirilen bir örnek değişkeni yalnızca sınıf nesnesinde görünür olacaktır. Bir değişken, başına “@” eklendiğinde örnek değişken olarak kabul edilecektir. Örnek değişkenleri, bir nesnenin özniteliklerini ayarlamak ve almak için kullanılır ve tanımlanmamışsa nil değerini döndürür.

class Dinosaur
    @base_sound = "rawrr"

    def initialize(sound = nil)
        @sound = sound || self.class.base_sound
    end

    def speak
        @sound
    end

    def try_to_speak
        @base_sound
    end

    def count_and_store_sound_length
        @sound.chars.each_with_index do |char, i|
            @sound_length = i + 1
            p "#{char}: #{sound_length}"
        end
    end
    
    def sound_length
        @sound_length
    end

    def self.base_sound
        @base_sound
    end
end

dino_1 = Dinosaur.new
dino_2 = Dinosaur.new "grrr"

Dinosaur.base_sound
# => "rawrr"
dino_2.speak
# => "grrr"

Sınıf düzeyinde bildirilen örnek değişkenine nesne düzeyinde erişilemez:

dino_1.try_to_speak
# => nil

Ancak, yeni yönteme hiçbir ses aktarılmadığında sesi başlatmak için @base_sound örnek değişkenini kullandık:

dino_1.speak
# => "rawwr"

Örnek değişkenleri, bir blok içinde bile nesnenin herhangi bir yerinde bildirilebilir:

dino_1.count_and_store_sound_length
# "r: 1"
# "a: 2"
# "w: 3"
# "r: 4"
# "r: 5"
# => ["r", "a", "w", "r", "r"]

dino_1.sound_length
# => 5

Örnek değişkenleri, aynı sınıfın örnekleri arasında paylaşılmaz

dino_2.sound_length
# => nil

Bu, sınıflar aynı zamanda Ruby’de nesneler olduğundan, bir alt sınıf tarafından üzerine yazılmayan sınıf düzeyinde değişkenler oluşturmak için kullanılabilir.

class DuckDuckDinosaur < Dinosaur
    @base_sound = "quack quack"
end

duck_dino = DuckDuckDinosaur.new
duck_dino.speak
# => "quack quack"
DuckDuckDinosaur.base_sound
# => "quack quack"
Dinosaur.base_sound
# => "rawrr"
