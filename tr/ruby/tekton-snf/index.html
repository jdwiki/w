Sözdizimi #

  • singleton_class = sınıf « nesne; kendi kendine son

Singleton sınıflarının yalnızca bir örneği vardır: karşılık gelen nesneleri. Bu, Ruby’nin ObjectSpace sorgusu yapılarak doğrulanabilir:

instances = ObjectSpace.each_object object.singleton_class

instances.count            # => 1
instances.include? object  # => true

[<][Module#<] kullanılarak, bunların nesnenin gerçek sınıfının alt sınıfları oldukları da doğrulanabilir:

object.singleton_class < object.class  # => true

Referanslar:

  • Ruby’de üç örtük bağlam

Singleton Sınıfının Kalıtımı #

Alt sınıflama ayrıca Singleton Sınıfını da Alt Sınıflar #

class Example
end

Example.singleton_class #=> #<Class:Example>

def Example.foo
  :example
end

class SubExample < Example
end

SubExample.foo #=> :example

SubExample.singleton_class.superclass #=> #<Class:Example>

Bir Modülü Genişletme veya Dahil Etme, Singleton Sınıfını Genişletmez #

module ExampleModule
end

def ExampleModule.foo
  :foo
end

class Example
  extend ExampleModule
  include ExampleModule
end

Example.foo #=> NoMethodError: undefined method

Singleton sınıfları #

Tüm nesneler bir sınıfın örnekleridir. Ancak, gerçeğin tamamı bu değildir. Ruby’de her nesnenin ayrıca bir şekilde gizli bir singleton sınıfı vardır.

Yöntemlerin tek tek nesneler üzerinde tanımlanmasına izin veren şey budur. Singleton sınıfı, nesnenin kendisi ile gerçek sınıfı arasında yer alır, bu nedenle üzerinde tanımlanan tüm yöntemler o nesne için ve yalnızca o nesne için kullanılabilir.

object = Object.new

def object.exclusive_method
  'Only this object will respond to this method'
end

object.exclusive_method
# => "Only this object will respond to this method"

Object.new.exclusive_method rescue $!
# => #<NoMethodError: undefined method `exclusive_method' for #<Object:0xa17b77c>>

Yukarıdaki örnek, define_singleton_method kullanılarak yazılabilirdi:

object.define_singleton_method :exclusive_method do
  "The method is actually defined in the object's singleton class"
end

Bu, “nesne"nin singleton_class üzerindeki yöntemi tanımlamakla aynıdır:

# send is used because define_method is private
object.singleton_class.send :define_method, :exclusive_method do
  "Now we're defining an instance method directly on the singleton class"
end

Ruby’nin çekirdek API’sinin bir parçası olarak “singleton_class"ın varlığından önce, singleton sınıfları metaclasses olarak biliniyordu ve aşağıdaki deyim aracılığıyla erişilebilirdi:

class << object
  self  # refers to object's singleton_class
end

Giriiş #

Ruby’nin üç tür nesnesi vardır:

  • Sınıf Sınıfı veya sınıf Modülü örnekleri olan sınıflar ve modüller.
  • Sınıf örnekleri.
  • Singleton Sınıfları.

Her nesne, yöntemlerini içeren bir sınıfa sahiptir:

class Example
end

object = Example.new

object.class  # => Example
Example.class # => Class
Class.class   # => Class

Nesnelerin kendileri yöntem içeremez, yalnızca sınıfları içerebilir. Ancak singleton sınıfları ile diğer singleton sınıfları da dahil olmak üzere herhangi bir nesneye metot eklemek mümkündür.

def object.foo
  :foo
end
object.foo #=> :foo

‘foo’, ‘object’in tekil sınıfında tanımlanır. Diğer “Örnek” örnekler “foo"ya yanıt veremez.

Ruby, talep üzerine tekil sınıflar oluşturur. Onlara erişmek veya onlara yöntemler eklemek, Ruby’yi onları oluşturmaya zorlar.

Singleton Sınıfına Erişme #

Bir nesnenin singleton sınıfını almanın iki yolu vardır.

  • “singleton_class” yöntemi.

  • Bir nesnenin singleton sınıfını yeniden açma ve ‘self’ döndürme.

    object.singleton_class

singleton_class = class << object
  self
end

Singleton Sınıflarında Örnek/Sınıf Değişkenlerine Erişme #

Singleton sınıfları, örnek/sınıf değişkenlerini nesneleriyle paylaşır.

class Example
  @@foo = :example
end

def Example.foo
  class_variable_get :@@foo
end

Example.foo #=> :example

class Example
  def initialize
    @foo = 1
  end

  def foo
    @foo
  end
end

e = Example.new

e.instance_eval <<-BLOCK
  def self.increase_foo
    @foo += 1
  end
BLOCK

e.increase_foo
e.foo #=> 2

Bloklar, örnek/sınıf değişkenleri hedefinin etrafında kapanır. “class_eval” veya “instance_eval” içindeki bir blok kullanarak örnek veya sınıf değişkenlerine erişmek mümkün değildir. ‘class_eval’ öğesine bir dize iletmek veya ‘class_variable_get’ kullanmak sorunu çözer.

class Foo
  @@foo = :foo
end

class Example
  @@foo = :example 

  Foo.define_singleton_method :foo do
    @@foo
  end
end

Foo.foo #=> :example

Singleton Class ile Mesaj Yayılımı #

Örnekler hiçbir zaman yalnızca veri taşıdıkları bir yöntem içermez. Ancak, bir sınıf örneği de dahil olmak üzere herhangi bir nesne için tek bir sınıf tanımlayabiliriz.

Bir nesneye bir mesaj iletildiğinde (yöntem çağrılır) Ruby ilk önce o nesne için bir singleton sınıfının tanımlanıp tanımlanmadığını ve bu mesaja cevap verip vermediğini kontrol eder, aksi takdirde Ruby örneğin sınıfının ataları zincirini kontrol eder ve üzerinde yürür.

class Example
  def foo
    :example
  end
end

Example.new.foo #=> :example

module PrependedModule
  def foo
    :prepend
  end
end

class Example
  prepend PrependedModule
end

Example.ancestors #=> [Prepended, Example, Object, Kernel, BasicObject]
e = Example.new
e.foo #=> :prepended

def e.foo
  :singleton
end

e.foo #=> :singleton

Yeniden Açma (maymun yaması) Singleton Sınıfları #

Singleton Sınıfını yeniden açmanın üç yolu vardır

  • Bir singleton sınıfında ‘class_eval’ kullanımı.

  • class << bloğunu kullanma.

  • Doğrudan nesnenin singleton sınıfında bir yöntem tanımlamak için ‘def’ kullanma

    class Example end

    Example.singleton_class.class_eval do def foo :foo end end

    Example.foo #=> :foo

class Example
end

class << Example
  def bar
    :bar
  end
end

Example.bar #=> :bar

class Example
end

def Example.baz
  :baz
end

Example.baz #=> :baz

Her nesnenin erişebileceğiniz bir singleton sınıfı vardır.

class Example
end
ex1 = Example.new
def ex1.foobar
  :foobar
end
ex1.foobar #=> :foobar

ex2 = Example.new
ex2.foobar #=> NoMethodError
