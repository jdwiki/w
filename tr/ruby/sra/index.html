Sözdizimi #

  • q = Kuyruk.yeni
  • q.push nesnesi
  • q « nesne #, #push ile aynı
  • q.pop #=> nesne

Birden Fazla Çalışan Tek Lavabo #

Birden fazla Çalışan tarafından oluşturulan verileri toplamak istiyoruz.

İlk önce bir Kuyruk oluşturuyoruz:

sink = Queue.new

Sonra 16 işçinin tümü rastgele bir sayı üretiyor ve onu lavaboya itiyor:

(1..16).to_a.map do
  Thread.new do
    sink << rand(1..100)
  end
end.map(&:join)

Ve verileri almak için bir Kuyruğu Diziye dönüştürün:

data = [].tap { |a| a << sink.pop until sink.empty? }

Bir Kuyruğu Diziye Dönüştürme #

q = Queue.new
q << 1
q << 2

a = Array.new
a << q.pop until q.empty?

Veya bir tek satır:

[].tap { |array| array < queue.pop until queue.empty? }

Tek Kaynaktan Birden Çok Çalışan #

Verileri paralel olarak işlemek istiyoruz.

Kaynağı bazı verilerle dolduralım:

source = Queue.new
data = (1..100)
data.each { |e| source << e }

Ardından verileri işlemek için bazı çalışanlar oluşturun:

(1..16).to_a.map do
  Thread.new do
    until source.empty?
      item = source.pop
      sleep 0.5
      puts "Processed: #{item}"
    end
  end
end.map(&:join)

Tek Kaynak - İş Akışı - Tek Lavabo #

Verileri paralel olarak işlemek ve diğer çalışanlar tarafından işlenmek üzere aşağı itmek istiyoruz.

Workers hem tükettiği hem de ürettiği için iki kuyruk oluşturmamız gerekiyor:

first_input_source = Queue.new
first_output_sink  = Queue.new
100.times { |i| first_input_source << i }

İlk işçi dalgası “ilk_giriş_kaynağı"ndan bir öğe okur, öğeyi işler ve sonuçları “ilk_çıkış_sink"e yazar:

(1..16).to_a.map do
  Thread.new do
    loop do
      item = first_input_source.pop
      first_output_source << item ** 2
      first_output_source << item ** 3
    end
  end
end

İkinci işçi dalgası, giriş kaynağı olarak ‘ilk_çıkış_havuzu’nu kullanır ve okur, işlem daha sonra başka bir çıkış havuzuna yazar:

second_input_source = first_output_sink
second_output_sink  = Queue.new

(1..32).to_a.map do
  Thread.new do
    loop do
      item = second_input_source.pop
      second_output_sink << item * 2
      second_output_sink << item * 3
    end
  end
end

Şimdi ‘second_output_sink’ havuz, onu bir diziye dönüştürelim:

sleep 5 # workaround in place of synchronization
sink = second_output_sink
[].tap { |a| a << sink.pop until sink.empty? }

Verileri Kuyruğa Gönderme - #push #

q = Queue.new
q << "any object including another queue"
# or
q.push :data
  • Yüksek su işareti yoktur, kuyruklar sonsuz büyüyebilir.
  • #push asla engellemez

Bir Kuyruktan Veri Çekme - #pop #

q = Queue.new
q << :data
q.pop #=> :data
  • #pop, bazı veriler mevcut olana kadar engellenir.
  • #pop senkronizasyon için kullanılabilir.

Senkronizasyon - Zaman İçinde Bir Noktadan Sonra #

syncer = Queue.new

a = Thread.new do
  syncer.pop
  puts "this happens at end"
end

b = Thread.new do
  puts "this happens first"
  STDOUT.flush
  syncer << :ok
end

[a, b].map(&:join)

İki Kuyruğu Birleştirme #

  • Sonsuz olarak engellemeyi önlemek için, iş parçacığı birleştirme gerçekleştiğinde kuyruklardan okuma yapılmamalıdır.
  • Senkronizasyonu önlemek veya kuyruklardan biri veri varken sonsuza kadar beklemekten kaçınmak için, aynı iş parçacığında kuyruklardan okuma yapılmamalıdır.

İki sıra tanımlayıp doldurarak başlayalım:

q1 = Queue.new
q2 = Queue.new
(1..100).each { |e| q1 << e }
(101..200).each { |e| q2 << e }

Başka bir kuyruk oluşturmalı ve diğer iş parçacıklarındaki verileri bu sıraya aktarmalıyız:

merged = Queue.new

[q1, q2].map do |q|
  Thread.new do
    loop do
      merged << q.pop
    end
  end
end

Her iki kuyruğu da tamamen tüketebileceğinizi biliyorsanız (tüketim hızı üretimden daha yüksektir, RAM’iniz bitmez) daha basit bir yaklaşım var:

merged = Queue.new
merged << q1.pop until q1.empty?
merged << q2.pop until q2.empty?
