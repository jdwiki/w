Sözdizimi #

  • Proc.new(blok)
  • lambda { |args| kod }
  • ->(arg1, arg2) { kod }
  • nesne.to_proc
  • { |single_arg| kod }
  • yap |arg, (anahtar, değer)| code son

Aşağıdakiler gibi zincirlenmiş birden çok yöntem içeren bir satırınız olduğunda operatör önceliğine dikkat edin:

str = "abcdefg"
puts str.gsub(/./) do |match|
  rand(2).zero? ? match.upcase : match.downcase
end

Beklediğiniz gibi “abCDeFg” gibi bir şey yazdırmak yerine, “#Enumerator:0x00000000af42b28” gibi bir şey yazdırır – bunun nedeni “do … end” öğesinin yöntemlerden daha düşük önceliğe sahip olmasıdır, bu da “gsub” anlamına gelir yalnızca/./` argümanını görür, blok argümanını görmez. Bir numaralandırıcı döndürür. Blok, onu yok sayan ve sadece “gsub(/./)” sonucunu gösteren “puts"a iletilir.

Bunu düzeltmek için “gsub” çağrısını parantez içine alın veya bunun yerine “{ … }” kullanın.

Lambdalar #

# lambda using the arrow syntax
hello_world = -> { 'Hello World!' }
hello_world[]
# 'Hello World!'

# lambda using the arrow syntax accepting 1 argument
hello_world = ->(name) { "Hello #{name}!" }
hello_world['Sven']
# "Hello Sven!"

the_thing = lambda do |magic, ohai, dere|
  puts "magic! #{magic}"
  puts "ohai #{dere}"
  puts "#{ohai} means hello"
end

the_thing.call(1, 2, 3)
# magic! 1
# ohai 3
# 2 means hello

the_thing.call(1, 2)
# ArgumentError: wrong number of arguments (2 for 3)

the_thing[1, 2, 3, 4]
# ArgumentError: wrong number of arguments (4 for 3)

Ayrıca oluşturmak için -> ve lambda çağırmak için .() kullanabilirsiniz.

the_thing = ->(magic, ohai, dere) {
  puts "magic! #{magic}"
  puts "ohai #{dere}"
  puts "#{ohai} means hello"
}

the_thing.(1, 2, 3)
# => magic! 1
# => ohai 3
# => 2 means hello

Burada bir lambda’nın proc ile neredeyse aynı olduğunu görebilirsiniz. Ancak, birkaç uyarı var:

  • Bir lambda’nın argümanlarının doğruluğu uygulanır; bir lambdaya yanlış sayıda argüman iletmek, bir ‘ArgumentError’ ortaya çıkaracaktır. Hala varsayılan parametrelere, uyarı parametrelerine vb. sahip olabilirler.

  • Bir lambda içinden ‘dönüş’, lambda’dan dönerken, bir işlemden ‘dönüş’, çevreleyen kapsamın dışına döner:

    def try_proc
  x = Proc.new {
    return # Return from try_proc
  }
  x.call
  puts "After x.call" # this line is never reached
end

def try_lambda
  y = -> {
    return # return from y
  }
  y.call
  puts "After y.call" # this line is not skipped
end

try_proc # No output
try_lambda # Outputs "After y.call"

Kısmi Uygulama ve Körleme #

Teknik olarak, Ruby’nin işlevleri değil, yöntemleri vardır. Ancak, bir Ruby yöntemi, diğer dildeki işlevlerle neredeyse aynı şekilde davranır:

def double(n)
  n * 2
end

Bu normal yöntem/işlev bir “n” parametresini alır, onu ikiye katlar ve değeri döndürür. Şimdi daha yüksek dereceli bir işlev (veya yöntem) tanımlayalım:

def triple(n)
  lambda {3 * n}
end

Bir sayı döndürmek yerine, “üçlü” bir yöntem döndürür. Interactive Ruby Shell kullanarak test edebilirsiniz:

$ irb --simple-prompt
>> def double(n)
>>   n * 2
>> end
=> :double
>> def triple(n)
>>   lambda {3 * n}
>> end
=> :triple
>> double(2)
=> 4
>> triple(2)
=> #<Proc:0x007fd07f07bdc0@(irb):7 (lambda)>

Üçlü sayıyı gerçekten elde etmek istiyorsanız, lambda’yı aramanız (veya “azaltmanız”) gerekir:

triple_two = triple(2)
triple_two.call # => 6

Veya daha kısaca:

triple(2).call

Köri ve Kısmi Uygulamalar #

Bu, çok temel işlevleri tanımlamak açısından kullanışlı değildir, ancak anında çağrılmayan veya azaltılmayan yöntemlere/işlevlere sahip olmak istiyorsanız kullanışlıdır. Örneğin, belirli bir sayı ile bir sayı ekleyen yöntemler tanımlamak istediğinizi varsayalım (örneğin add_one(2) = 3). Bunlardan bir ton tanımlamanız gerekiyorsa şunları yapabilirsiniz:

def add_one(n)
  n + 1
end 

def add_two(n)
  n + 2
end

Ancak şunu da yapabilirsiniz:

add = -> (a, b) { a + b }
add_one = add.curry.(1)
add_two = add.curry.(2)

Lambda hesabını kullanarak add (λa.(λb.(a+b))) olduğunu söyleyebiliriz. Körleme, _kısmen ’ekleme’ uygulamasının bir yoludur. Yani ‘add.curry.(1)’, ‘(λa.(λb.(a+b)))(1)‘dir ve ‘(λb.(1+b))‘ye indirgenebilir. Kısmi uygulama, bir argümanı ‘add’ öğesine ilettiğimiz, ancak diğer argümanı daha sonra sağlanmak üzere bıraktığımız anlamına gelir. Çıktı özel bir yöntemdir.

Daha faydalı köri örnekleri #

Diyelim ki gerçekten büyük bir genel formülümüz var, ona belirli argümanlar belirtirsek, ondan özel formüller elde edebiliriz. Bu formülü düşünün:

f(x, y, z) = sin(x\*y)*sin(y\*z)*sin(z\*x)

Bu formül üç boyutlu çalışmak için yapılmış ama diyelim ki bu formülü sadece y ve z açısından istiyoruz. Diyelim ki x’i yok saymak için değerini pi/2 olarak ayarlamak istiyoruz. Önce genel formülü yapalım:

f = ->(x, y, z) {Math.sin(x*y) * Math.sin(y*z) * Math.sin(z*x)}

Şimdi yz formülümüzü elde etmek için köri kullanalım:

f_yz = f.curry.(Math::PI/2)

Ardından f_yz içinde saklanan lambdayı çağırmak için:

f_xy.call(some_value_x, some_value_y)

Bu oldukça basit, ama diyelim ki ‘xz’ formülünü almak istiyoruz. Son argüman değilse ‘y’yi ‘Math::PI/2’ olarak nasıl ayarlayabiliriz? Eh, biraz daha karmaşık:

f_xz = -> (x,z) {f.curry.(x, Math::PI/2, z)}

Bu durumda, önceden doldurmadığımız parametre için yer tutucular sağlamamız gerekir. Tutarlılık için “f_xy"yi şöyle yazabiliriz:

f_xy = -> (x,y) {f.curry.(x, y, Math::PI/2)}

Lambda hesabı “f_yz” için şu şekilde çalışır:

f = (λx.(λy.(λz.(sin(x*y) * sin(y*z) * sin(z*x))))
f_yz = (λx.(λy.(λz.(sin(x*y) * sin(y*z) * sin(z*x)))) (π/2) # Reduce => 
f_yz = (λy.(λz.(sin((π/2)*y) * sin(y*z) * sin(z*(π/2))))

Şimdi ‘f_xz’ye bakalım

f = (λx.(λy.(λz.(sin(x*y) * sin(y*z) * sin(z*x))))
f_xz = (λx.(λy.(λz.(sin(x*y) * sin(y*z) * sin(z*x)))) (λt.t) (π/2)  # Reduce =>
f_xz = (λt.(λz.(sin(t*(π/2)) * sin((π/2)*z) * sin(z*t))))

Lambda hesabı hakkında daha fazla bilgi için bunu deneyin.

Yöntemlerin blok argümanları olarak nesneler #

Bir argümanın önüne bir “&” (ve işareti) koymak, onu metodun bloğu olarak iletir. Nesneler, “to_proc” yöntemi kullanılarak bir “Proc"a dönüştürülecektir.

class Greeter
  def to_proc
    Proc.new do |item|
      puts "Hello, #{item}"
    end
  end
end

greet = Greeter.new

%w(world life).each(&greet)

Bu, Ruby’de yaygın bir kalıptır ve birçok standart sınıf bunu sağlar.

Örneğin, [Symbol][symbol]s, kendilerini bağımsız değişkene göndererek to_proc öğesini uygular:

# Example implementation
class Symbol
  def to_proc
    Proc.new do |receiver|
      receiver.send self
    end
  end
end

Bu, [Sayılandırılabilir][sayılandırılabilir] nesnelerle yaygın olarak kullanılan kullanışlı &:symbol deyimini etkinleştirir:

letter_counts = %w(just some words).map(&:length)  # [4, 4, 5]

Proc’a Dönüştürme #

’to_proc’a yanıt veren nesneler, ‘&’ operatörü ile proc’lara dönüştürülebilir (bu, onların blok olarak geçirilmesine de izin verir).

Symbol sınıfı “#to_proc"u tanımlar, bu nedenle aldığı nesnede karşılık gelen yöntemi parametre olarak çağırmaya çalışır.

p [ 'rabbit', 'grass' ].map( &:upcase ) # => ["RABBIT", "GRASS"]

Yöntem nesneleri ayrıca “#to_proc” öğesini de tanımlar.

output = method( :p )

[ 'rabbit', 'grass' ].map( &output ) # => "rabbit\ngrass"

Bloklar #

Bloklar, “{}” (genellikle tek satırlı bloklar için) veya “do..end” (çok satırlı bloklar için kullanılır) ayraçları arasına alınmış kod parçalarıdır.

5.times { puts "Hello world" } # recommended style for single line blocks

5.times do
    print "Hello "
    puts "world"
end   # recommended style for multi-line blocks

5.times {
    print "hello "
    puts "world" } # does not throw an error but is not recommended

Not: kaşlı ayraçlar “do..end"den daha yüksek önceliğe sahiptir

verim #

Bloklar, verim kelimesini kullanan metotlar ve fonksiyonlar içinde kullanılabilir:

def block_caller
    puts "some code"
    yield
    puts "other code"
end
block_caller { puts "My own block" } # the block is passed as an argument to the method.
#some code
#My own block
#other code

Ancak dikkatli olun, eğer ‘verim’ bloksuz çağrılırsa, ‘LocalJumpError’ ortaya çıkar. Bu amaçla Ruby, block_given? adında başka bir yöntem sunar; bu, verimi çağırmadan önce bir bloğun geçip geçmediğini kontrol etmenizi sağlar.

def block_caller
  puts "some code" 
  if block_given? 
    yield
  else
    puts "default"
  end
  puts "other code"
end
block_caller 
# some code
# default
# other code
block_caller { puts "not defaulted"}
# some code
# not defaulted
# other code

‘verim’ de bloğa argümanlar sunabilir

def yield_n(n)
  p = yield n if block_given?
  p || n 
end
yield_n(12) {|n| n + 7 } 
#=> 19 
yield_n(4) 
#=> 4

Bu basit bir örnek olsa da, “verim”, başka bir nesne bağlamında örnek değişkenlere veya değerlendirmelere doğrudan erişime izin vermek için çok yararlı olabilir. Örneğin:

class Application
  def configuration
    @configuration ||= Configuration.new
    block_given? ? yield(@configuration) : @configuration
  end
end
class Configuration; end

app = Application.new 
app.configuration do |config| 
  puts config.class.name
end
# Configuration
#=> nil 
app.configuration
#=> #<Configuration:0x2bf1d30>

Gördüğünüz gibi, “verim"i bu şekilde kullanmak, kodu sürekli olarak “app.configuration.#method_name” çağırmaktan daha okunaklı kılar. Bunun yerine, içerdiği kodu koruyarak tüm yapılandırmayı blok içinde gerçekleştirebilirsiniz.

Değişkenler #

Blok değişkenleri bloğa yereldir (fonksiyonların değişkenlerine benzer), blok yürütüldüğünde ölürler.

my_variable = 8
3.times do |x|
    my_variable = x 
    puts my_variable
end
puts my_variable
#=> 0
# 1
# 2
# 8

Bloklar kaydedilemez, yürütüldükten sonra ölürler. Blokları kaydetmek için ‘procs’ ve ’lambdas’ kullanmanız gerekir.

İşlem #

def call_the_block(&calling); calling.call; end

its_a = proc do |*args|
  puts "It's a..." unless args.empty?
  "beautiful day"
end

puts its_a       #=> "beautiful day"
puts its_a.call  #=> "beautiful day"
puts its_a[1, 2] #=> "It's a..." "beautiful day"

“call_the_block” yöntemini son örnekten kopyaladık. Burada bir blok ile proc metodunu çağırarak bir proc yapıldığını görebilirsiniz. Ayrıca, yöntemler gibi blokların da örtük getirileri olduğunu görebilirsiniz; bu, işlemlerin (ve lambdaların) da yaptığı anlamına gelir. its_a tanımında, blokların normal argümanlar kadar uyarı argümanları da alabildiğini görebilirsiniz; aynı zamanda varsayılan argümanları da alabilirler, ancak bunu çalıştırmanın bir yolunu düşünemedim. Son olarak, bir yöntemi çağırmak için birden fazla sözdizimi kullanmanın mümkün olduğunu görebilirsiniz – ‘call’ yöntemi veya ‘[]’ operatörü.
