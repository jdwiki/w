=~ operatör #

if /hay/ =~ 'haystack'
  puts "There is hay in the word haystack"
end

Not: Sıra önemlidir. ‘‘haystack’ =~ /hay/’ çoğu durumda eşdeğer olsa da, yan etkiler farklılık gösterebilir:

  • Adlandırılmış yakalama gruplarından yakalanan dizeler, yalnızca Regexp#=~ çağrıldığında yerel değişkenlere atanır ([regexp =~ str](http://ruby-doc.org/core-2.3.1/String. html#yöntem-i-3D-7E));
  • Doğru işlenen rastgele bir nesne olabileceğinden, ‘regexp =~ str’ için ‘Regexp#=~’ veya ‘String#=~’ olarak adlandırılacaktır.

Bunun bir doğru/yanlış değeri döndürmediğini, bunun yerine bulunursa eşleşmenin dizinini veya bulunamazsa nil değerini döndürdüğünü unutmayın. Ruby’deki tüm tamsayılar doğru (0 dahil) ve sıfır yanlış olduğundan, bu işe yarar. Boole değeri istiyorsanız, [başka bir örnekte] gösterildiği gibi “#===” kullanın(https://www.wikiod.com/tr/ruby/normal-ifadeler-ve-regex-tabanl-islemler basit-regex-match#t=201607221936139047493).

Vaka İfadelerinde Normal İfadeler #

Bir switch ifadesi kullanarak bir dizenin birkaç normal ifadeyle eşleşip eşleşmediğini test edebilirsiniz.

Örnek #

case "Ruby is #1!"
when /\APython/
    puts "Boooo."
when /\ARuby/
    puts "You are right."
else 
    puts "Sorry, I didn't understand that."
end

Bu işe yarar, çünkü case ifadeleri eşitlik için == operatörü değil === operatörü kullanılarak kontrol edilir. === kullanarak bir karşılaştırmanın sol tarafında bir normal ifade olduğunda, eşleşip eşleşmediğini görmek için bir dize test eder.

Gruplar, adlandırılmış ve başka türlü. #

Ruby, (...) standart grup sözdizimini (?<name>...) adlı bir adlandırılmış grupla genişletir. Bu, kaç grubunuz olduğunu saymak yerine ada göre ayıklamaya izin verir.

name_reg = /h(i|ello), my name is (?<name>.*)/i #i means case insensitive

name_input = "Hi, my name is Zaphod Beeblebrox"

match_data = name_reg.match(name_input) #returns either a MatchData object or nil
match_data = name_input.match(name_reg) #works either way

if match_data.nil? #Always check for nil! Common error.
  puts "No match"
else
  match[0] #=> "Hi, my name is Zaphod Beeblebrox"
  match[1] #=> "i" #the first group, (i|ello)
  match[2] #=> "Zaphod Beeblebrox"
  #Because it was a named group, we can get it by name
  match[:name]  #=> "Zaphod Beeblebrox"
  match["name"] #=> "Zaphod Beeblebrox"
  puts "Hello #{match[:name]}!"
end

Eşleşmenin dizini, sol parantezlerin sırasına göre sayılır (tüm normal ifade, dizin 0’daki ilk gruptur)

reg = /(((a)b)c)(d)/
match = reg.match 'abcd'
match[0] #=> "abcd"
match[1] #=> "abc"
match[2] #=> "ab"
match[3] #=> "a"
match[4] #=> "d"

Niceleyiciler #

Niceleyiciler, tekrarlanan dizelerin sayısını belirtmeye izin verir.

  • Sıfır veya bir:

    /a?/

  • Sıfır veya çok:

    /a*/

  • Bir veya daha fazla:

    /a+/

  • Tam sayı:

    /a{2,4}/ # Two, three or four
/a{2,}/  # Two or more
/a{,4}/  # Less than four (including zero)

Varsayılan olarak, nicelik belirteçleri açgözlüdür, yani hareketsizken alabildikleri kadar çok karakter alırlar maç yapmak. Normalde bu fark edilmez:

/(?<site>.*) Stack Exchange/ =~ 'Motor Vehicle Maintenance & Repair Stack Exchange'

Adlandırılmış yakalama grubu “site” beklendiği gibi “Motorlu Araç Bakım ve Onarım” olarak ayarlanacaktır. Ancak ‘Yığın Değişimi’ dizenin isteğe bağlı bir parçasıysa (çünkü bunun yerine ‘Yığın Taşması’ olabilir), saf çözüm beklendiği gibi çalışmayacaktır:

/(?<site>.*)( Stack Exchange)?/

Bu sürüm yine de eşleşecek, ancak * açgözlülükle bu karakterleri yediğinden, adlandırılmış yakalama ‘Yığın Değişimi’ni içerecek. Çözüm, * tembel yapmak için başka bir soru işareti eklemektir:

/(?<site>.*?)( Stack Exchange)?/

Herhangi bir niceleyiciye ? eklemek onu tembelleştirir.

Karakter sınıfları #

Sembol aralıklarını tanımlar

Sembolleri açıkça sıralayabilirsiniz

/[abc]/ # 'a' or 'b' or 'c'

Veya aralıkları kullanın

/[a-z]/ # from 'a' to 'z'

Aralıkları ve tek sembolleri birleştirmek mümkündür

/[a-cz]/ # 'a' or 'b' or 'c' or 'z'

Baştaki tire (-) karakter olarak kabul edilir

/[-a-c]/ # '-' or 'a' or 'b' or 'c'

“^” ile gelen sembollerden önce geldiğinde sınıflar negatif olabilir

/[^a-c]/ # Not 'a', 'b' or 'c'

Yaygın sınıflar ve özel karakterler için bazı kısayollar ve ayrıca satır sonları vardır.

^  # Start of line
$  # End of line
\A # Start of string
\Z # End of string, excluding any new line at the end of string
\z # End of string
.  # Any single character
\s # Any whitespace character
\S # Any non-whitespace character
\d # Any digit
\D # Any non-digit
\w # Any word character (letter, number, underscore)
\W # Any non-word character
\b # Any word boundary

\n basitçe yeni satır olarak anlaşılacak

“/” veya “[]” gibi ayrılmış herhangi bir karakterden kaçmak için ve diğerleri ters eğik çizgi (sola eğik çizgi) kullanır.

\\ # => \
\[\] # => []

Normal İfade Tanımlama #

Ruby’de bir Regexp üç farklı şekilde oluşturulabilir.

  • eğik çizgi kullanarak: / /

  • %r{} kullanarak

  • ‘Regex.new’ kullanarak

    #The following forms are equivalent
regexp_slash = /hello/
regexp_bracket = %r{hello}
regexp_new = Regexp.new('hello')

string_to_match = "hello world!"

#All of these will return a truthy value
string_to_match =~ regexp_slash    # => 0
string_to_match =~ regexp_bracket  # => 0
string_to_match =~ regexp_new      # => 0

kibrit? - Boole Sonucu #

“$~” ve diğer ilgili değişkenler güncellenmeden normal ifadenin eşleşip eşleşmediğini gösteren “doğru” veya “yanlış” döndürür. İkinci parametre varsa, aramayı başlatmak için dizedeki konumu belirtir.

/R.../.match?("Ruby")    #=> true
/R.../.match?("Ruby", 1) #=> false
/P.../.match?("Ruby")    #=> false

Yakut 2.4+

Ortak hızlı kullanım #

Normal ifadeler genellikle yöntemlerde başka dizelerin olup olmadığını kontrol etmek veya dizeleri aramak ve/veya değiştirmek için parametreler olarak kullanılır.

Aşağıdakileri sık sık görürsünüz:

string = "My not so long string"
string[/so/] # gives so
string[/present/] # gives nil
string[/present/].nil? # gives true

Böylece, bir dizenin bir alt dize içerip içermediğini kontrol etmek için bunu kullanabilirsiniz.

puts "found" if string[/so/]

Daha gelişmiş ama yine de kısa ve hızlı: ikinci parametreyi kullanarak belirli bir grubu arayın, bu örnekte 2 ikincidir, çünkü numaralandırma 0 değil 1’den başlar, grup parantez içine alınan şeydir.

string[/(n.t).+(l.ng)/, 2] # gives long

Ayrıca sıklıkla kullanılır: “sub” veya “gsub” ile arama ve değiştirme, “\1” ilk bulunan grubu, “\2” ikinci grubu verir:

string.gsub(/(n.t).+(l.ng)/, '\1 very \2') # My not very long string

Son sonuç hatırlanır ve aşağıdaki satırlarda kullanılabilir

$2 # gives long
