Semboller #

#thiIsaSymbol  "Symbols are interned strings, used for method and variable names,
                and as values with fast equality checking."
#'hello world' "A symbol with a space in it"
#''            "An empty symbol, not very useful"
#+
#1             "Not the integer 1"

$→     "An unicode character"

Kesirler #

Kesirler, kayan noktalı sayılarla aynı değildir, tam sayılardır (yuvarlama hatası yoktur).

Diziler #

Bayt Dizileri #

Dinamik diziler #

Dinamik diziler ifadelerden oluşturulur. Parantez içindeki her ifade, oluşturulan dizide farklı bir değere göre değerlendirilir.

{self foo. 3 + 2. i * 3}   "A dynamic array built from 3 expressions"

Bloklar #

[ :p | p asString ] "A code block with a parameter p.
                     Blocks are the same as lambdas in other languages"

Bazı notlar: #

Değişmez dizilerin ayırıcı olarak herhangi bir türde ve sayıda boşluk kullandığını unutmayın.

#(256 16rAB1F 3.14s2 2r1001 $A #this)
"is the same as:"
#(256
  16rAB1F
  3.14s2
  2r1001
  $A #this)

Ayrıca değişmezleri oluşturabileceğinizi unutmayın.

#[255 16rFF 8r377 2r11111111]       (four times 255)

#(#[1 2 3] #('string' #symbol))     (arrays of arrays)

Gevşemiş gösterime biraz “tolerans” var

#(symbol) = #(#symbol)              (missing # => symbol)

#('string' ($a 'a'))                (missing # => array)

Ama burada değil:

#([1 2 3]) ~= #(#[1 2 3])           (missing # => misinterpreted)

Yine de

#(true nil false)                   (pseudo variables ok)

Ama burada değil!

#(self) = #(#self)                  (missing # => symbol)

Gördüğünüz gibi, birkaç tutarsızlık var:

  • ’true’, ‘false’ ve ’nil’ sözde değişkenleri diziler içinde değişmez değerler olarak kabul edilirken, “self” ve “super” sözde değişkenleri Semboller olarak yorumlanır (nitelenmemiş dizeler için daha genel kural kullanılır. )

  • Bir dizide iç içe dizi başlatmak için #( yazmak zorunlu değildir ve parantez yeterliyken, iç içe bir ByteArray başlatmak için #[ yazmak zorunludur.

Mesaj gönderme #

Smalltalk’ta yaptığınız hemen hemen her şey nesnelere _mesaj göndermektir (diğer dillerde çağrı yöntemleri olarak anılır). Üç tür mesaj vardır:

Tekli mesajlar:

#(1 2 3) size
"This sends the #size message to the #(1 2 3) array.
 #size is a unary message, because it takes no arguments."

İkili mesajlar:

1 + 2
"This sends the #+ message and 2 as an argument to the object 1.
 #+ is a binary message because it takes one argument (2)
 and it's composed of one or two symbol characters"

Anahtar kelime mesajları:

'Smalltalk’ allButFirst: 5.
"This sends #allButFirst: with argument 5 to the string 'Smalltalk',
 resulting in the new string 'talk'"

3 to: 10 by: 2.
"This one sends the single message #to:by:, which takes two parameters (10 and 2)
 to the number 3.
 The result is a collection with 3, 5, 7, and 9."

Bir ifadedeki birden çok mesaj, öncelik sırasına göre değerlendirilir

unary > binary > keyword

ve soldan sağa.

1 + 2 * 3 " equals 9, because it evaluates left to right"
1 + (2 * 3) " but you can use parenthesis"

1 to:   #(a b c d) size   by:   5 - 4
 "is the same as:"
1 to: ( #(a b c d) size ) by: ( 5 - 4 )

Aynı nesneye çok sayıda mesaj göndermek istiyorsanız, basamaklı ;(noktalı virgül) operatörünü kullanabilirsiniz:

OrderedCollection new
  add: #abc;
  add: #def;
  add: #ghi;
  yourself.

“Bu ilk önce #new mesajını OrderedCollection sınıfına gönderir (#new sadece bir message, not an operator). It results in a new OrderedCollection. It then sends the new collection three times the message #add (with different arguments), and the message yourself.”

Sınıflar ve yöntemler #

Sınıflar ve yöntemler genellikle Smalltalk IDE’de tanımlanır.

sınıflar #

Bir sınıf tanımı tarayıcıda şuna benzer:

XMLTokenizer subclass: #XMLParser
    instanceVariableNames: ''
    classVariableNames: ''
    poolDictionaries: ''
    category: 'XML-Parser'

Bu aslında tarayıcının sistemde yeni bir sınıf oluşturmanız için göndereceği mesajdır. (Bu durumda, #subclass:instanceVariableNames:classVariableNames:poolDictionaries:category:, ancak yeni sınıflar oluşturan başkaları da var).

İlk satır, hangi sınıfı alt sınıflandırdığınızı (bu durumda XMLTokenizer’dır) ve yeni alt sınıfın sahip olacağı adı (#XMLParser) gösterir.

Sonraki üç satır, sınıfın ve örneklerinin sahip olacağı değişkenleri tanımlamak için kullanılır.

yöntemler #

Yöntemler tarayıcıda şöyle görünür:

aKeywordMethodWith: firstArgument and: secondArgument
    "Do something with an argument and return the result."

    ^firstArgument doSomethingWith: secondArgument

^(şapka) dönüş operatörüdür.

** anInteger
    "Raise me to anInteger"
    | temp1 temp2 |

    temp1 := 1.
    temp2 := 1.
    1 to: anInteger do: [ :i | temp1 := temp1 * self + temp2 - i ].
    ^temp1

bu, üs almanın doğru yolu değildir, ancak bir ikili mesaj tanımı (diğer herhangi bir mesaj gibi tanımlanırlar) ve bazı metod geçici değişkenleri (veya yöntem geçicileri, temp1 ve temp2) artı bir blok argümanı (i) gösterir.

Smalltalk’ta Döngüler #

Bu örnekte, öğeler üzerinde döngü oluşturmak için bir “OrderedCollection” nesnesine gönderilebilecek farklı mesajları göstermek için bir “Ordered Collection” kullanılacaktır.
Aşağıdaki kod, mesajı kullanarak boş bir “OrderedCollection” örneğini oluşturacaktır. “yeni” ve ardından “add:” mesajını kullanarak 4 sayı ile doldurun.

anOrderedCollection := OrderedCollection new.
anOrderedCollection add: 1; add: 2; add: 3; add: 4.

Bu mesajların tümü, koleksiyon içindeki öğelerin her biri için değerlendirilecek bir parametre olarak bir blok alacaktır.

  1. do:
    Bu, temel numaralandırma mesajıdır. Örneğin, koleksiyondaki her bir öğeyi yazdırmak istiyorsak, bunu şu şekilde başarabiliriz:

     anOrderedCollection do:[:each | Transcript show: each]. "Prints --> 1234"

    Each of the elements inside the collection will be defined as the user wishes using this syntax: :each This do: loop will print every element in the collection to the Transcript window.

  2. toplayın:
    The collect: message allows you to do something for each item in the collection and puts the result of yeni bir koleksiyondaki eyleminiz
    Örneğin, koleksiyonumuzdaki her öğeyi 2 ile çarpmak ve onu yeni bir koleksiyona eklemek istiyorsak, ‘collect:’ mesajını şu şekilde kullanabiliriz:

     evenCollection := anOrderedCollection collect:[:each | each*2]. "#(2 4 6 8)"

  1. seç:
    select: mesajı, orijinal koleksiyondaki öğelerin bulunduğu bir alt koleksiyon oluşturmanıza olanak tanır. onlar için doğru olan bazı koşullara göre seçilir. Örneğin, koleksiyonumuzdan yeni bir tek sayılar koleksiyonu oluşturmak istersek, select: mesajını şu şekilde kullanabiliriz:

     oddCollection := anOrderedCollection select:[:each | each odd].

    Since each odd returns a Boolean, only the elements that make the Boolean return true will be added to oddCollection which will have #(1 3).

  2. reddet:
    Bu mesaj “select:“in tersi olarak çalışır ve Boolean’ın “true” döndürmesini sağlayan tüm öğeleri reddeder. Veya başka bir deyişle, Boolean’ın ‘yanlış’ döndürmesini sağlayan tüm öğeleri seçecektir. Örneğin, önceki örnekteki gibi aynı oddCollectionı oluşturmak istersek. reddet: ifadesini şu şekilde kullanabiliriz:

     oddCollection := anOrderedCollection reject:[:each | each even].

    oddCollection will again have #(1 3) as its elements.

Bunlar Smalltalk’taki dört temel numaralandırma tekniğidir. Ancak, uygulanabilecek daha fazla mesaj için ‘Koleksiyonlar’ sınıfına göz atmaktan çekinmeyin.
