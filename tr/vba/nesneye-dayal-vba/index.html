Soyutlama #

Soyutlama seviyeleri, şeylerin ne zaman bölüneceğini belirlemeye yardımcı olur. #

Soyutlama, giderek daha ayrıntılı kodlarla işlevsellik uygulanarak elde edilir. Bir makronun giriş noktası, neler olup bittiğini bir bakışta kavramayı kolaylaştıran yüksek soyutlama düzeyine sahip küçük bir prosedür olmalıdır:

Public Sub DoSomething()
    With New SomeForm
        Set .Model = CreateViewModel
        .Show vbModal            
        If .IsCancelled Then Exit Sub
        ProcessUserData .Model
    End With
End Sub

DoSomething prosedürünün yüksek bir soyutlama seviyesi: bir form görüntülediğini ve bir model oluşturduğunu ve bu nesneyi onunla ne yapacağını bilen bir ProcessUserData prosedürüne ilettiğini söyleyebiliriz - modelin nasıl oluşturulduğu başka bir prosedürün işi:

Private Function CreateViewModel() As ISomeModel
    Dim result As ISomeModel
    Set result = SomeModel.Create(Now, Environ$("UserName"))
    result.AvailableItems = GetAvailableItems
    Set CreateViewModel = result
End Function

‘CreateViewModel’ işlevi yalnızca bazı ‘ISomeModel’ örneklerinin oluşturulmasından sorumludur. Bu sorumluluğun bir kısmı, bir dizi mevcut öğe elde etmektir - bu öğelerin nasıl elde edildiği, “GetAvailableItems” prosedürünün arkasında özetlenen bir uygulama detayıdır:

Private Function GetAvailableItems() As Variant
    GetAvailableItems = DataSheet.Names("AvailableItems").RefersToRange
End Function

Buradaki prosedür, bir ‘Veri Sayfası’ çalışma sayfasındaki adlandırılmış bir aralıktan mevcut değerleri okumaktır. Bunları bir veritabanından okumak da olabilir veya değerler sabit kodlanmış olabilir: bu, daha yüksek soyutlama seviyelerinin hiçbiri için endişe etmeyen bir uygulama ayrıntısıdır.

Kapsülleme #

Kapsülleme, uygulama ayrıntılarını istemci kodundan gizler. #

https://www.wikiod.com/tr/vba/kullanc-formlar#QueryClose’u işleme örneği kapsüllemeyi gösterir: formun bir onay kutusu denetimi vardır, ancak istemci kodu doğrudan onunla çalışmaz - onay kutusu bir uygulama ayrıntısıdır, istemci kodunun bilmesi gereken şey, ayarın etkin olup olmadığıdır.

Onay kutusu değeri değiştiğinde, işleyici bir özel alan üyesi atar:

Özel Tip TView Boole Olarak İptal Edildi Boole Olarak BazıDiğerAyarlar ‘kısalık için atlanan diğer özellikler Bitiş Türü Bunu TView Olarak Özelleştir

‘…

Özel Alt SomeOtherSettingInput_Change() this.SomeOtherSetting = CBool(SomeOtherSettingInput.Value) Alt Alt

İstemci kodu bu değeri okumak istediğinde, bir onay kutusu hakkında endişelenmesine gerek yoktur - bunun yerine yalnızca ‘SomeOtherSetting’ özelliğini kullanır:

Kamu Mülkiyetini Boole Olarak SomeOtherSetting() Al SomeOtherSetting = this.SomeOtherSetting Mülkü Sonlandır

‘SomeOtherSetting’ özelliği onay kutusunun durumunu kapsar; istemci kodunun ilgili bir onay kutusu olduğunu bilmesi gerekmez, yalnızca Boolean değerine sahip bir ayar olduğunu bilmesi gerekir. “Boolean” değerini kapsülleyerek, onay kutusunun çevresine bir soyutlama katmanı ekledik.

Değişmezliği zorlamak için arayüzleri kullanma #

Formun modelini özel bir sınıf modülünde kapsülleyerek bunu bir adım daha ileri götürelim. Ancak, ‘KullanıcıAdı’ ve ‘Zaman Damgası’ için bir ‘Kamu Mülkü’ yaparsak, özellikleri değiştirilebilir hale getirerek ‘Mülkiyet İzni’ erişimcilerini açığa çıkarmamız gerekir ve müşteri kodunun değiştirilebilmesini istemeyiz. bu değerler ayarlandıktan sonra.

Abstraksiyon örneğindeki ‘CreateViewModel’ işlevi bir ‘ISomeModel’ sınıfını döndürür: bu bizim arayüzümüz ve şuna benzer:

Option Explicit

Public Property Get Timestamp() As Date
End Property

Public Property Get UserName() As String
End Property

Public Property Get AvailableItems() As Variant
End Property

Public Property Let AvailableItems(ByRef value As Variant)
End Property

Public Property Get SomeSetting() As String
End Property

Public Property Let SomeSetting(ByVal value As String)
End Property

Public Property Get SomeOtherSetting() As Boolean
End Property

Public Property Let SomeOtherSetting(ByVal value As Boolean)
End Property

“Zaman Damgası” ve “KullanıcıAdı” özelliklerinin yalnızca bir “Özellik Alma” erişimcisini gösterdiğine dikkat edin. Şimdi “SomeModel” sınıfı bu arayüzü uygulayabilir:

Option Explicit
Implements ISomeModel

Private Type TModel
    Timestamp As Date
    UserName As String
    SomeSetting As String
    SomeOtherSetting As Boolean
    AvailableItems As Variant
End Type
Private this As TModel

Private Property Get ISomeModel_Timestamp() As Date
    ISomeModel_Timestamp = this.Timestamp
End Property

Private Property Get ISomeModel_UserName() As String
    ISomeModel_UserName = this.UserName
End Property

Private Property Get ISomeModel_AvailableItems() As Variant
    ISomeModel_AvailableItems = this.AvailableItems
End Property

Private Property Let ISomeModel_AvailableItems(ByRef value As Variant)
    this.AvailableItems = value
End Property

Private Property Get ISomeModel_SomeSetting() As String
    ISomeModel_SomeSetting = this.SomeSetting
End Property

Private Property Let ISomeModel_SomeSetting(ByVal value As String)
    this.SomeSetting = value
End Property

Private Property Get ISomeModel_SomeOtherSetting() As Boolean
    ISomeModel_SomeOtherSetting = this.SomeOtherSetting
End Property

Private Property Let ISomeModel_SomeOtherSetting(ByVal value As Boolean)
    this.SomeOtherSetting = value
End Property

Public Property Get Timestamp() As Date
    Timestamp = this.Timestamp
End Property

Public Property Let Timestamp(ByVal value As Date)
    this.Timestamp = value
End Property

Public Property Get UserName() As String
    UserName = this.UserName
End Property

Public Property Let UserName(ByVal value As String)
    this.UserName = value
End Property

Public Property Get AvailableItems() As Variant
    AvailableItems = this.AvailableItems
End Property

Public Property Let AvailableItems(ByRef value As Variant)
    this.AvailableItems = value
End Property

Public Property Get SomeSetting() As String
    SomeSetting = this.SomeSetting
End Property

Public Property Let SomeSetting(ByVal value As String)
    this.SomeSetting = value
End Property

    SomeOtherSetting = this.SomeOtherSetting

    this.SomeOtherSetting = value

Arabirim üyelerinin tümü ‘Özel’dir ve kodun derlenmesi için arabirimin tüm üyelerinin uygulanması gerekir. ‘Public’ üyeleri arabirimin bir parçası değildir ve bu nedenle ‘ISomeModel’ arabirimine karşı yazılan koda maruz kalmazlar.

Bir yapıcıyı simüle etmek için bir Fabrika Yöntemi kullanma #

Bir https://www.wikiod.com/tr/vba/oznitellikler#VB_Önceden Bildirilen Kimlik özniteliği kullanarak, “SomeModel” sınıfının varsayılan bir örneği olmasını sağlayabilir ve çalışan bir işlev yazabiliriz burada yaptığımız gibi, müşteri kodunun ilk önce bir örnek oluşturmaya gerek kalmadan arayabileceği bir tür düzeyi (VB.NET’te ‘Paylaşılan’, C#‘da ‘statik’) üyesi gibi:

ISomeModel Olarak Özel İşlev CreateViewModel() ISOmeModel olarak sonucu karart Sonuç ayarla = SomeModel.Create(Şimdi, Environ$(“KullanıcıAdı”)) sonuç.AvailableItems = GetAvailableItems CreateViewModel = sonucu ayarla Fonksiyonu Bitir

Bu fabrika yöntemi, “ISomeModel” arabiriminden erişildiğinde salt okunur olan özellik değerlerini, burada “Zaman Damgası” ve “KullanıcıAdı” olarak atar:

Public Function Create(ByVal pTimeStamp As Date, ByVal pUserName As String) As ISomeModel
    With New SomeModel
        .Timestamp = pTimeStamp
        .UserName = pUserName
        Set Create = .Self
    End With
End Function

Public Property Get Self() As ISomeModel
    Set Self = Me
End Property

Ve şimdi, “Zaman Damgası” ve “KullanıcıAdı"nı asla yeniden atanamayan salt okunur özellikler olarak gösteren “ISomeModel” arayüzüne karşı kod yazabiliriz (kod arayüze karşı yazıldığı sürece).

Polimorfizm #

Polimorfizm, farklı temel uygulamalar için aynı arayüzü sunma yeteneğidir. #

Arayüzleri uygulama yeteneği, uygulama mantığının kullanıcı arayüzünden veya veritabanından veya şu ya da bu çalışma sayfasından tamamen ayrılmasını sağlar.

Diyelim ki formun kendisinin uyguladığı bir “ISomeView” arayüzünüz var:

Option Explicit

Public Property Get IsCancelled() As Boolean
End Property

Public Property Get Model() As ISomeModel
End Property

Public Property Set Model(ByVal value As ISomeModel)
End Property

Public Sub Show()
End Sub

Formun arka plan kodu şöyle görünebilir:

Option Explicit 
Implements ISomeView

Private Type TView
    IsCancelled As Boolean
    Model As ISomeModel
End Type
Private this As TView

Private Property Get ISomeView_IsCancelled() As Boolean
    ISomeView_IsCancelled = this.IsCancelled
End Property

Private Property Get ISomeView_Model() As ISomeModel
    Set ISomeView_Model = this.Model
End Property

Private Property Set ISomeView_Model(ByVal value As ISomeModel)
    Set this.Model = value
End Property

Private Sub ISomeView_Show()
    Me.Show vbModal
End Sub

Private Sub SomeOtherSettingInput_Change()
    this.Model.SomeOtherSetting = CBool(SomeOtherSettingInput.Value)
End Sub

'...other event handlers...

Private Sub OkButton_Click()
    Me.Hide
End Sub

Private Sub CancelButton_Click()
    this.IsCancelled = True
    Me.Hide
End Sub

Private Sub UserForm_QueryClose(Cancel As Integer, CloseMode As Integer)
    If CloseMode = VbQueryClose.vbFormControlMenu Then
        Cancel = True
        this.IsCancelled = True
        Me.Hide
    End If
End Sub

Ancak hiçbir şey, bir kullanıcı formu olmadan * “ISomeView” arabirimini uygulayan başka bir sınıf modülü oluşturmayı yasaklayamaz - bu bir “SomeViewMock” sınıfı olabilir:

Option Explicit
Implements ISomeView

Private Type TView
    IsCancelled As Boolean
    Model As ISomeModel
End Type
Private this As TView

Public Property Get IsCancelled() As Boolean
    IsCancelled = this.IsCancelled
End Property

Public Property Let IsCancelled(ByVal value As Boolean)
    this.IsCancelled = value
End Property

Private Property Get ISomeView_IsCancelled() As Boolean
    ISomeView_IsCancelled = this.IsCancelled
End Property

Private Property Get ISomeView_Model() As ISomeModel
    Set ISomeView_Model = this.Model
End Property

Private Property Set ISomeView_Model(ByVal value As ISomeModel)
    Set this.Model = value
End Property

Private Sub ISomeView_Show()
    'do nothing
End Sub

Ve şimdi bir “UserForm” ile çalışan kodu değiştirebilir ve “ISomeView” arayüzünde çalışmasını sağlayabiliriz, ör. formu somutlaştırmak yerine parametre olarak vererek:

Public Sub DoSomething(ByVal view As ISomeView)
    With view
        Set .Model = CreateViewModel
        .Show
        If .IsCancelled Then Exit Sub
        ProcessUserData .Model
    End With
End Sub

“DoSomething” yöntemi bir somut sınıfa (ör. belirli bir “UserForm”) değil bir arabirime (yani bir soyutlama) bağlı olduğundan, “ProcessUserData"nın yürütülmemesini sağlayan otomatik bir birim testi yazabiliriz. “view.IsCancelled” “True” olduğunda, testimizin bir “SomeViewMock” örneği oluşturmasını sağlayarak, “IsCancelled” özelliğini “True” olarak ayarlayıp onu “DoSomething"e geçirerek.

Test edilebilir kod, soyutlamalara bağlıdır #

VBA’da birim testleri yazılabilir, hatta onu IDE’ye entegre eden eklentiler var. Ancak kod bir çalışma sayfası, veritabanı, form veya dosya sistemiyle sıkıca birleştiğinde, birim testi gerçek bir çalışma sayfası, veritabanı, form veya dosya sistemi gerektirmeye başlar - ve bu bağımlılıklar yenidir. birim testlerinin gerçek bir çalışma sayfası, veritabanı, form veya dosya sistemi gerektirmemesi için test edilebilir kodun yalıtması gereken kontrol dışı hata noktaları.

Arayüzlere karşı kod yazarak, test kodunun saplama/sahte uygulamaları enjekte etmesine izin verecek şekilde (yukarıdaki “SomeViewMock” örneğinde olduğu gibi), testleri “kontrollü bir ortamda” yazabilir ve her birinin ne olduğunu simüle edebilirsiniz. Bir kez bile form görüntülemeden ve bir form kontrolüne manuel olarak tıklamadan, form verileri üzerindeki kullanıcı etkileşimlerinin 42 olası permütasyonundan.
