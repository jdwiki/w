Sözdizimi #

  • Nitelik VB_Name = “ClassOrModuleName”
  • Nitelik VB_GlobalNameSpace = False ’ Yok sayıldı
  • Nitelik VB_Creatable = False ’ Yok sayıldı
  • Öznitelik VB_PredeclaredId = {True | Yanlış}
  • Öznitelik VB_Exposed = {Doğru | Yanlış}
  • Öznitelik değişkeniName.VB_VarUserMemId = 0 ’ Sıfır, bunun sınıfın varsayılan üyesi olduğunu gösterir.
  • Nitelik değişkeniName.VB_VarDescription = “bazı dize” ’ Metni, bu değişken için Nesne Tarayıcı bilgilerine ekler.
  • Attribute procName.VB_Description = “bazı dize” ’ Metni, prosedür için Nesne Tarayıcı bilgilerine ekler.
  • öznitelik procName.VB_UserMemId = {0 | -4}
  • ’ 0: Fonksiyonu sınıfın varsayılan üyesi yapar.
  • ’ -4: İşlevin bir Numaralandırıcı döndürdüğünü belirtir.

VB_Önceden Bildirilen Kimlik #

Bir sınıfın Genel Varsayılan Eşgörünümünü oluşturur. Varsayılan örneğe, sınıfın adı aracılığıyla erişilir.

beyanname #

VERSION 1.0 CLASS
BEGIN
  MultiUse = -1  'True
END
Attribute VB_Name = "Class1"
Attribute VB_GlobalNameSpace = False
Attribute VB_Creatable = False
Attribute VB_PredeclaredId = True
Attribute VB_Exposed = False
Option Explicit

Public Function GiveMeATwo() As Integer
    GiveMeATwo = 2
End Function

Telefon etmek #

Debug.Print Class1.GiveMeATwo

Bazı yönlerden bu, diğer dillerdeki statik sınıfların davranışını simüle eder, ancak diğer dillerden farklı olarak yine de sınıfın bir örneğini oluşturabilirsiniz.

Dim cls As Class1
Set cls = New Class1
Debug.Print cls.GiveMeATwo

VB_[Var]UserMemId #

‘VB_VarUserMemId’ (modül kapsamı değişkenleri için) ve ‘VB_UserMemId’ (prosedürler için) öznitelikleri VBA’da çoğunlukla iki şey için kullanılır.

Bir sınıfın varsayılan üyesini belirtme #

Bir “Koleksiyon"u içine alan bir “List” sınıfı, bir “Öğe” özelliğine sahip olmak ister, böylece müşteri kodu bunu yapabilir:

For i = 1 To myList.Count 'VBA Collection Objects are 1-based
    Debug.Print myList.Item(i)
Next

Ancak, “Öğe” özelliğinde “VB_UserMemId” özniteliği 0 olarak ayarlandığında, istemci kodu şunu yapabilir:

For i = 1 To myList.Count 'VBA Collection Objects are 1-based
    Debug.Print myList(i)
Next

Herhangi bir sınıfta yalnızca bir üye yasal olarak ‘VB_UserMemId = 0’ değerine sahip olabilir. Özellikler için, “Get” erişimcisinde özniteliği belirtin:

Option Explicit
Private internal As New Collection

Public Property Get Count() As Long
    Count = internal.Count
End Property

Public Property Get Item(ByVal index As Long) As Variant
Attribute Item.VB_Description = "Gets or sets the element at the specified index."
Attribute Item.VB_UserMemId = 0
'Gets the element at the specified index.
    Item = internal(index)    
End Property

Public Property Let Item(ByVal index As Long, ByVal value As Variant)
'Sets the element at the specified index.    
    With internal
        If index = .Count + 1 Then
            .Add item:=value
        ElseIf index = .Count Then
            .Remove index
            .Add item:=value
        ElseIf index < .Count Then
            .Remove index
            .Add item:=value, before:=index
        End If
    End With
End Property

‘ForEvery’ döngü yapısıyla sınıfı yinelenebilir hale getirme #

“-4” sihirli değeriyle, “VB_UserMemId” özelliği VBA’ya bu üyenin bir numaralandırıcı verdiğini söyler - bu da istemci kodunun bunu yapmasına izin verir:

Dim item As Variant
For Each item In myList
    Debug.Print item
Next

Bu yöntemi uygulamanın en kolay yolu, dahili/kapsüllenmiş bir “Koleksiyon” üzerinde gizli “[_NewEnum]” özellik alıcısını çağırmaktır; tanımlayıcı, onu geçersiz bir VBA tanımlayıcısı yapan baştaki alt çizgi nedeniyle köşeli parantez içine alınmalıdır:

Public Property Get NewEnum() As IUnknown
Attribute NewEnum.VB_Description = "Gets an enumerator that iterates through the List."
Attribute NewEnum.VB_UserMemId = -4
Attribute NewEnum.VB_MemberFlags = "40" 'would hide the member in VB6. not supported in VBA.
'Gets an enumerator that iterates through the List.
    Set NewEnum = internal.[_NewEnum]    
End Property

VB_Açık #

Bir sınıfın örnekleme özelliklerini kontrol eder.

Attribute VB_Exposed = False

Sınıfı ‘Private’ yapar. Mevcut projenin dışından erişilemez.

Attribute VB_Exposed = True

Projenin dışında ‘Genel’ sınıfını gösterir. Ancak, VBA’da ‘VB_Createable’ yok sayıldığından, sınıfın örnekleri doğrudan oluşturulamaz. Bu, aşağıdaki VB.Net sınıfına eşdeğerdir.

Public Class Foo
    Friend Sub New()
    End Sub
End Class

Proje dışından bir instance almak için instance oluşturmak için bir fabrikayı açığa çıkarmalısınız. Bunu yapmanın bir yolu, normal bir ‘Public’ modülüdür.

Public Function CreateFoo() As Foo
    CreateFoo = New Foo
End Function

Genel modüllere diğer projelerden erişilebildiğinden, bu, ‘Genel - Oluşturulamaz’ sınıflarımızın yeni örneklerini oluşturmamıza olanak tanır.

VB_Açıklama #

Nesne Gezgini’nde görünür hale gelen bir sınıfa veya modül üyesine bir metin açıklaması ekler. İdeal olarak, bir genel arayüzün / API’nin tüm genel üyelerinin bir açıklaması olmalıdır.

Public Function GiveMeATwo() As Integer
    Attribute GiveMeATwo.VB_Description = "Returns a two!"        
    GiveMeATwo = 2
End Property

Not: bir özelliğin tüm erişimci üyeleri (“Get”, “Let”, “Set”) aynı açıklamayı kullanır.

VB_Adı #

VB_Name, sınıf veya modül adını belirtir.

Attribute VB_Name = "Class1"

Bu sınıfın yeni bir örneği ile oluşturulacak

Dim myClass As Class1
myClass = new Class1

VB_GlobalNameSpace #

VBA’da bu özellik yok sayılır. VB6’dan taşınmadı.

VB6’da, sınıf üyelerine sınıf adını kullanmadan erişilebilmesi için sınıfın Varsayılan Genel Örneği (“kısayol”) oluşturur. Örneğin, “DateTime” (“DateTime.Now"da olduğu gibi) aslında “VBA.Conversion” sınıfının bir parçasıdır.

Debug.Print VBA.Conversion.DateTime.Now
Debug.Print DateTime.Now

VB_Oluşturulabilir #

Bu öznitelik yok sayılır. VB6’dan taşınmadı.

VB6’da, mevcut proje dışındaki sınıfların erişilebilirliğini kontrol etmek için ‘VB_Exposed’ özniteliği ile birlikte kullanıldı.

VB_Exposed=True 
VB_Creatable=True

Diğer projelerden erişilebilen bir ‘Genel Sınıf’ ile sonuçlanır, ancak bu işlevsellik VBA’da mevcut değildir.
