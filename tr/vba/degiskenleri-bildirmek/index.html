Tip İpuçları #

Tip İpuçları şiddetle önerilmez. Tarihsel ve geriye dönük uyumluluk nedenleriyle burada mevcutturlar ve belgelenmiştir. Bunun yerine As [DataType] sözdizimini kullanmalısınız.

Public Sub ExampleDeclaration()

        Dim someInteger% '% Equivalent to "As Integer"
        Dim someLong&    '& Equivalent to "As Long"
        Dim someDecimal@ '@ Equivalent to "As Currency"
        Dim someSingle!  '! Equivalent to "As Single"
        Dim someDouble#  '# Equivalent to "As Double"
        Dim someString$  '$ Equivalent to "As String"

        Dim someLongLong^  '^ Equivalent to "As LongLong" in 64-bit VBA hosts
End Sub

Yazım ipuçları, kodun okunabilirliğini önemli ölçüde azaltır ve aynı zamanda okunabilirliği engelleyen eski bir Macar Notasyonu teşvik eder:

Dim strFile$
Dim iFile%

Bunun yerine, değişkenleri kullanımlarına daha yakın olarak tanımlayın ve türlerini türlerine göre değil, kullanıldıklarına göre adlandırın:

Dim path As String
Dim handle As Integer

Tür ipuçları, belirli bir türü zorlamak için hazır bilgilerde de kullanılabilir. Varsayılan olarak, 32.768’den küçük sayısal bir sabit değer “Tamsayı” sabit değeri olarak yorumlanır, ancak bir tür ipucu ile şunları kontrol edebilirsiniz:

Dim foo 'implicit Variant
foo = 42& ' foo is now a Long
foo = 42# ' foo is now a Double
Debug.Print TypeName(42!) ' prints "Single"

Açık bir türle bildirilen bir değişkene atanacakları veya parametre olarak iletildiklerinde örtük olarak uygun türe dönüştürülecekleri için, tür ipuçlarına genellikle değişmezlerde ihtiyaç duyulmaz. Açık tür dönüştürme işlevlerinden biri kullanılarak örtük dönüştürmelerden kaçınılabilir:

'Calls procedure DoSomething and passes a literal 42 as a Long using a type hint
DoSomething 42&

'Calls procedure DoSomething and passes a literal 42 explicitly converted to a Long
DoSomething CLng(42)

Dize döndüren yerleşik işlevler #

Dizeleri işleyen yerleşik işlevlerin çoğu iki sürümde gelir: Bir “Varyant” döndüren gevşek yazılmış bir sürüm ve bir “Dize” döndüren kesin olarak yazılmış bir sürüm ("$" ile biten). Dönüş değerini bir ‘Varyant’a atamadığınız sürece, ‘Dize’ döndüren sürümü tercih etmelisiniz - aksi halde dönüş değerinin örtük bir dönüşümü olur.

Debug.Print Left(foo, 2)  'Left returns a Variant
Debug.Print Left$(foo, 2) 'Left$ returns a String

Bu işlevler şunlardır:

  • VBA.Conversion.Error -> VBA.Conversion.Error$
  • VBA.Conversion.Hex -> VBA.Conversion.Hex$
  • VBA.Conversion.Oct -> VBA.Conversion.Oct$
  • VBA.Conversion.Str -> VBA.Conversion.Str$
  • VBA.FileSystem.CurDir -> VBA.FileSystem.CurDir$
  • VBA.[_HiddenModule].Input -> VBA.[_HiddenModule].Input$
  • VBA.[_HiddenModule].InputB -> VBA.[_HiddenModule].InputB$
  • VBA.Interaction.Command -> VBA.Interaction.Command$
  • VBA.Interaction.Environ -> VBA.Interaction.Environ$
  • VBA.Strings.Chr -> VBA.Strings.Chr$
  • VBA.Strings.ChrB -> VBA.Strings.ChrB$
  • VBA.Strings.ChrW -> VBA.Strings.ChrW$
  • VBA.Strings.Format -> VBA.Strings.Format$
  • VBA.Strings.LCase -> VBA.Strings.LCase$
  • VBA.Strings.Left -> VBA.Strings.Left$
  • VBA.Strings.LeftB -> VBA.Strings.LeftB$
  • VBA.Strings.LTtrim -> VBA.Strings.LTrim$
  • VBA.Strings.Mid -> VBA.Strings.Mid$
  • VBA.Strings.MidB -> VBA.Strings.MidB$
  • VBA.Strings.Right -> VBA.Strings.Right$
  • VBA.Strings.RightB -> VBA.Strings.RightB$
  • VBA.Strings.RTrim -> VBA.Strings.RTrim$
  • VBA.Strings.Space -> VBA.Strings.Space$
  • VBA.Strings.Str -> VBA.Strings.Str$
  • VBA.Strings.String -> VBA.Strings.String$
  • VBA.Strings.Trim -> VBA.Strings.Trim$
  • VBA.Strings.UCase -> VBA.Strings.UCase$

Bunların işlev takma adları olduğunu, tam olarak tip ipuçları olmadığını unutmayın. ‘Left’ işlevi gizli ‘B_Var_Left’ işlevine karşılık gelirken, ‘Left$’ sürümü gizli ‘B_Str_Left’ işlevine karşılık gelir.

VBA’nın çok erken sürümlerinde $ işareti izin verilen bir karakter değildir ve işlev adının köşeli parantez içine alınması gerekiyordu. Word Basic’te, $ ile biten dizeleri döndüren çok, çok daha fazla işlev vardı.

Değişkenler #

Dürbün #

Bir değişken bildirilebilir (artan görünürlük seviyesinde):

  • Prosedür düzeyinde, herhangi bir prosedürde “Dim” anahtar sözcüğünü kullanmak; bir yerel değişken.
  • Modül düzeyinde, herhangi bir modül türünde “Private” anahtar sözcüğünü kullanmak; bir özel alan.
  • Örnek düzeyinde, herhangi bir sınıf modülü türünde ‘Friend’ anahtar sözcüğünü kullanmak; bir arkadaş alanı.
  • Örnek düzeyinde, herhangi bir sınıf modülü türünde ‘Public’ anahtar sözcüğünü kullanmak; bir kamusal alan.
  • Global olarak, bir standart modülde “Public” anahtar sözcüğünü kullanmak; bir genel değişken.

Değişkenler her zaman mümkün olan en küçük kapsamla bildirilmelidir: genel değişkenler bildirmek yerine parametreleri prosedürlere iletmeyi tercih edin.

Daha fazla bilgi için https://www.wikiod.com/tr/vba/degiskenleri-bildirmek#Erişim Değiştiricileri adresine bakın.

Yerel değişkenler #

Bir yerel değişken bildirmek için “Dim” anahtar sözcüğünü kullanın:

Dim identifierName [As Type][, identifierName [As Type], ...]

Bildirim sözdiziminin “[As Type]” kısmı isteğe bağlıdır. Belirtildiğinde, bu değişkene ne kadar bellek ayrılacağını belirleyen değişkenin veri türünü ayarlar. Bu, bir “Dize” değişkeni bildirir:

Dim identifierName As String

Bir tür belirtilmediğinde, tür örtük olarak “Varyant” olur:

Dim identifierName 'As Variant is implicit

VBA sözdizimi, tek bir ifadede birden çok değişkenin bildirilmesini de destekler:

Dim someString As String, someVariant, someValue As Long

Her değişken için (‘Varyant’ olanlar dışında) “[As Type]” öğesinin belirtilmesi gerektiğine dikkat edin. Bu nispeten yaygın bir tuzaktır:

Dim integer1, integer2, integer3 As Integer 'Only integer3 is an Integer. 
                                            'The rest are Variant.

Statik değişkenler #

Yerel değişkenler ayrıca ‘Statik’ olabilir. VBA’da “Statik” anahtar sözcüğü, bir değişkenin sahip olduğu değeri “hatırlamasını” sağlamak için kullanılır, en son bir prosedür çağrıldığında:

Private Sub DoSomething()
    Static values As Collection
    If values Is Nothing Then
        Set values = New Collection
        values.Add "foo"
        values.Add "bar"
    End If
    DoSomethingElse values
End Sub

Burada ‘değerler’ koleksiyonu ‘Statik’ yerel olarak bildirilir; bir nesne değişkeni olduğundan, “Hiçbir şey” olarak başlatılır. Bildirimi izleyen koşul, nesne başvurusunun daha önce ‘Set’ olup olmadığını doğrular - prosedür ilk kez çalıştırılıyorsa, koleksiyon başlatılır. ‘DoSomethingElse’ öğeleri ekliyor veya kaldırıyor olabilir ve ‘DoSomething’ bir daha çağrıldığında koleksiyonda olmaya devam edecekler.

Alternatif

VBA’nın ‘Statik’ anahtar sözcüğü kolayca yanlış anlaşılabilir - özellikle genellikle diğer dillerde çalışan deneyimli programcılar tarafından. Birçok dilde, bir sınıf üyesinin (alan, özellik, yöntem, …) örnek yerine tür’e ait olmasını sağlamak için ‘statik’ kullanılır. “Statik” bağlamdaki kod, örnek bağlamındaki koda başvuramaz. VBA “Statik” anahtar sözcüğü, çılgınca farklı bir anlama gelir.

Çoğu zaman, bir “Statik” yerel, “Özel”, modül düzeyinde bir değişken (alan) olarak uygulanabilir - ancak bu, bir değişkenin mümkün olan en küçük kapsamda bildirilmesi gerektiği ilkesine meydan okur; içgüdülerinize güvenin, hangisini tercih ederseniz onu kullanın - ikisi de işe yarar… ama ne yaptığını anlamadan ‘Statik’ kullanmak ilginç hatalara yol açabilir.

Dim vs. Özel

Dim anahtar kelimesi prosedür ve modül seviyelerinde yasaldır; modül düzeyinde kullanımı “Private” anahtar sözcüğünü kullanmaya eşdeğerdir:

Option Explicit
Dim privateField1 As Long 'same as Private privateField2 as Long
Private privateField2 As Long 'same as Dim privateField2 as Long

‘Private’ anahtar sözcüğü yalnızca modül düzeyinde yasaldır; bu, yerel değişkenler için ‘Dim’ ayırmayı ve modül değişkenlerini ‘Private’ ile, özellikle de yine de bir public üye bildirmek için kullanılması gereken zıt ‘Public’ anahtar sözcüğüyle, davet ediyor. Alternatif olarak “Dim” her yerde kullanın - önemli olan tutarlılık:

“Özel alanlar”

  • ** YAP** modül düzeyinde bir değişken bildirmek için ‘Özel’ kullanın.
  • DO yerel bir değişken bildirmek için ‘Dim’ kullanın.
  • Modül düzeyinde bir değişken bildirmek için Dim’i KULLANMAYIN.

“Her yer karart”

  • DO herhangi bir şeyi özel/yerel olarak bildirmek için “Dim"i kullanın.
  • Modül düzeyinde bir değişken bildirmek için Özel’i KULLANMAYIN.
  • KAÇININ ‘Genel’ alanları bildirerek.*

*Genel olarak, yine de “Genel” veya “Global” alanları bildirmekten kaçınılmalıdır.

Alanlar #

Modül gövdesinin üst kısmındaki bildirimler bölümünde modül düzeyinde bildirilen bir değişken bir alandır. standart modülde bildirilen bir “Genel” alan bir genel değişkendir:

Public PublicField As Long

Küresel kapsamı olan bir değişkene, bildirildiği projeye referans verecek diğer VBA projeleri de dahil olmak üzere herhangi bir yerden erişilebilir.

Bir değişkeni genel/genel yapmak, ancak yalnızca proje içinden görünür kılmak için “Arkadaş” değiştiricisini kullanın:

Friend FriendField As Long

Bu, özellikle diğer VBA projelerinin eklenti projesine başvurması ve genel API’yi tüketebilmesinin amaçlandığı eklentilerde kullanışlıdır.

Friend FriendField As Long 'public within the project, aka for "friend" code
Public PublicField As Long 'public within and beyond the project

Arkadaş alanları standart modüllerde mevcut değildir.

Örnek Alanları #

Bir sınıf modülünün gövdesinin üst kısmındaki bildirimler bölümünde modül düzeyinde bildirilen bir değişken (‘ThisWorkbook’, ‘ThisDocument’, ‘Worksheet’, ‘UserForm’ ve class modülleri dahil), bir * örnek alanı*: yalnızca sınıfın bir örneği olduğu sürece var olur.

'> Class1
Option Explicit
Public PublicField As Long

'> Module1
Option Explicit
Public Sub DoSomething()
    'Class1.PublicField means nothing here
    With New Class1
        .PublicField = 42
    End With
    'Class1.PublicField means nothing here
End Sub

Kapsülleme alanları #

Örnek verileri genellikle “Gizli” tutulur ve kapsüllenmiş olarak adlandırılır. Özel bir alan, bir “Mülkiyet” prosedürü kullanılarak açığa çıkarılabilir. Özel bir değişkeni arayan kişiye yazma erişimi vermeden genel olarak göstermek için, bir sınıf modülü (veya standart bir modül) bir “Property Get” üyesi uygular:

Option Explicit
Private encapsulated As Long

Public Property Get SomeValue() As Long
    SomeValue = encapsulated
End Property

Public Sub DoSomething()
    encapsulated = 42
End Sub

Sınıfın kendisi kapsüllenmiş değeri değiştirebilir, ancak çağıran kod yalnızca ‘Genel’ üyelere (ve arayan aynı projedeyse ‘Arkadaş’ üyelerine) erişebilir.

Arayanın değişiklik yapmasına izin vermek için:

  • Kapsüllenmiş bir değer, bir modül bir ‘Property Let’ üyesini ortaya çıkarır.
  • Kapsüllenmiş bir nesne referansı, bir modül bir ‘Özellik Kümesi’ üyesini ortaya çıkarır.

Sabitler (Sabit) #

Uygulamanızda hiç değişmeyen bir değeriniz varsa, adlandırılmış bir sabit tanımlayabilir ve onu değişmez bir değer yerine kullanabilirsiniz.

Const’u yalnızca modül veya prosedür düzeyinde kullanabilirsiniz. Bu, bir değişkenin bildirim bağlamının bir sınıf, yapı, modül, prosedür veya blok olması gerektiği ve bir kaynak dosya, ad alanı veya arabirim olamayacağı anlamına gelir.

Public Const GLOBAL_CONSTANT As String = "Project Version #1.000.000.001"
Private Const MODULE_CONSTANT As String = "Something relevant to this Module"

Public Sub ExampleDeclaration()    

    Const SOME_CONSTANT As String = "Hello World"
    
    Const PI As Double = 3.141592653

End Sub

Sabit türleri belirtmek iyi bir uygulama olarak kabul edilebilirken, kesinlikle gerekli değildir. Türü belirtmemek yine de doğru türle sonuçlanacaktır:

Public Const GLOBAL_CONSTANT = "Project Version #1.000.000.001" 'Still a string
Public Sub ExampleDeclaration()

    Const SOME_CONSTANT = "Hello World"           'Still a string
    Const DERIVED_CONSTANT = SOME_CONSTANT        'DERIVED_CONSTANT is also a string
    Const VAR_CONSTANT As Variant = SOME_CONSTANT 'VAR_CONSTANT is Variant/String
    
    Const PI = 3.141592653        'Still a double
    Const DERIVED_PI = PI         'DERIVED_PI is also a double
    Const VAR_PI As Variant = PI  'VAR_PI is Variant/Double
    
End Sub

Bunun Sabitlere özgü olduğunu ve türün belirtilmemesinin bir Varyant türüyle sonuçlandığı değişkenlerin aksine olduğunu unutmayın.

Bir sabiti açıkça Dize olarak bildirmek mümkün olsa da, sabit genişlikli dize sözdizimini kullanarak bir sabiti dize olarak bildirmek mümkün değildir.

'This is a valid 5 character string constant
Const FOO As String = "ABCDE"

'This is not valid syntax for a 5 character string constant
Const FOO As String * 5 = "ABCDE"

Sabit Uzunluklu Dizeleri Bildirme #

VBA’da, Dizeler belirli bir uzunlukta bildirilebilir; beyan edildiği gibi bu uzunluğu korumak için otomatik olarak doldurulur veya kesilir.

Public Sub TwoTypesOfStrings()

    Dim FixedLengthString As String * 5 ' declares a string of 5 characters
    Dim NormalString As String

    Debug.Print FixedLengthString       ' Prints "     "
    Debug.Print NormalString            ' Prints ""

    FixedLengthString = "123"           ' FixedLengthString now equals "123  "
    NormalString = "456"                ' NormalString now equals "456"

    FixedLengthString = "123456"        ' FixedLengthString now equals "12345"
    NormalString = "456789"             ' NormalString now equals "456789"

End Sub

Örtülü ve Açık Beyanname #

Bir kod modülü, modülün en üstünde Option Explicit içermiyorsa, derleyici bunları kullandığınızda otomatik olarak (yani “örtük olarak”) sizin için değişkenler yaratacaktır. Varsayılan olarak ‘Variant’ değişken tipine sahip olacaklardır.

Public Sub ExampleDeclaration()    

    someVariable = 10                  ' 
    someOtherVariable = "Hello World"
    'Both of these variables are of the Variant type.

End Sub

Yukarıdaki kodda, ‘Option Explicit’ belirtilirse, ‘someVariable’ ve ‘someOtherVariable’ için gerekli ‘Dim’ ifadeleri eksik olduğundan kod kesintiye uğrayacaktır.

Option Explicit

Public Sub ExampleDeclaration()   

    Dim someVariable As Long 
    someVariable = 10

    Dim someOtherVariable As String
    someOtherVariable = "Hello World"

End Sub

Tüm değişkenleri bildirdiğinizden emin olmak için kod modüllerinde Option Explicit kullanmak en iyi uygulama olarak kabul edilir.

https://www.wikiod.com/tr/excel-vba/vba-en-iyi-uygulamalar Bu seçeneğin varsayılan olarak nasıl ayarlanacağına bakın.

Erişim Değiştiricileri #

Dim ifadesi yerel değişkenler için ayrılmalıdır. Modül düzeyinde, açık erişim değiştiricilerini tercih edin:

  • Yalnızca tanımlandıkları modül içinde erişilebilen özel alanlar için ‘Private’.
  • Herhangi bir arama koduyla erişilebilen genel alanlar ve genel değişkenler için “Genel”.
  • Proje içinde herkese açık olan ancak diğer referans VBA projelerine erişilemeyen değişkenler için “Arkadaş” (eklentiler için alakalı)
  • ‘Global’, standart modüllerde ‘Genel’ alanlar için de kullanılabilir, ancak sınıf modüllerinde yasa dışıdır ve zaten eskidir - bunun yerine ‘Public’ değiştiricisini tercih edin. Bu değiştirici prosedürler için de yasal değildir.

Erişim değiştiricileri, hem değişkenler hem de prosedürler için geçerlidir.

Private ModuleVariable As String
Public GlobalVariable As String

Private Sub ModuleProcedure()

    ModuleVariable = "This can only be done from within the same Module"

End Sub

Public Sub GlobalProcedure()

    GlobalVariable = "This can be done from any Module within this Project"

End Sub

Seçenek Özel Modül #

Standart modüllerdeki genel parametresiz “Alt” prosedürler makrolar olarak gösterilir ve ana bilgisayar belgesindeki kontrollere ve klavye kısayollarına eklenebilir.

Bunun tersine, standart modüllerdeki genel “İşlev” prosedürleri, ana bilgisayar uygulamasında kullanıcı tanımlı işlevler (UDF’ler) olarak sunulur.

Standart bir modülün üst kısmında ‘Özel Modül Seçeneği’nin belirtilmesi, üyelerinin ana bilgisayar uygulamasına makro ve UDF olarak gösterilmesini engeller.

Statik değişken ne zaman kullanılır #

Yerel olarak bildirilen bir Statik değişken yok edilmez ve Sub prosedüründen çıkıldığında değerini kaybetmez. Prosedüre yapılan sonraki çağrılar, hatırlanan herhangi bir değeri/değerleri ‘sıfırlamak’ isteyebilirsiniz, ancak yeniden başlatma veya atama gerektirmez.

Bunlar özellikle tekrar tekrar çağrılan bir ‘yardımcı’ alt nesneye geç bağlanıldığında kullanışlıdır.

Snippet 2: VBScript.RegExp nesnesini geç bağlayan bir çalışma sayfası UDF’si oluşturun

Option Explicit

Function numbersOnly(str As String, _
                     Optional delim As String = ", ")
    Dim n As Long, nums() As Variant
    Static rgx As Object, cmat As Object

    'with rgx as static, it only has to be created once
    'this is beneficial when filling a long column with this UDF
    If rgx Is Nothing Then
        Set rgx = CreateObject("VBScript.RegExp")
    Else
        Set cmat = Nothing
    End If
    
    With rgx
        .Global = True
        .MultiLine = True
        .Pattern = "[0-9]{1,999}"
        If .Test(str) Then
            Set cmat = .Execute(str)
            'resize the nums array to accept the matches
            ReDim nums(cmat.Count - 1)
            'populate the nums array with the matches
            For n = LBound(nums) To UBound(nums)
                nums(n) = cmat.Item(n)
            Next n
            'convert the nums array to a delimited string
            numbersOnly = Join(nums, delim)
        Else
            numbersOnly = vbNullString
        End If
    End With
End Function

*500K satırı UDF ile doldurmak için geçen süre:
       - Dim rgx As Object ile: 148,74 saniye
       - Statik rgx As Object ile: 26.07 saniye
* Bunlar yalnızca göreli karşılaştırma için düşünülmelidir. Kendi sonuçlarınız, gerçekleştirilen işlemlerin karmaşıklığına ve
    kapsamına göre değişiklik gösterecektir.

Bir UDF’nin bir çalışma kitabının ömrü boyunca bir kez hesaplanmadığını unutmayın. Uçucu olmayan bir UDF bile, referans verdiği aralık(lar) içindeki değerler değişebileceği zaman yeniden hesaplayacaktır. Sonraki her yeniden hesaplama olayı, yalnızca statik olarak bildirilen bir değişkenin faydalarını artırır.

  • Tanımlandığı ve atandığı prosedür veya fonksiyon için değil, modülün ömrü boyunca bir Statik değişken mevcuttur.
  • Statik değişkenler yalnızca yerel olarak bildirilebilir.
  • Statik değişken, özel modül düzeyindeki bir değişkenin aynı özelliklerinin çoğunu tutar, ancak kapsamı daha kısıtlıdır.
