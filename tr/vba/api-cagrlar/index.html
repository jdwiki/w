API, [Uygulama Programlama Arayüzü] anlamına gelir(https://en.wikipedia.org/wiki/Application_programming_interface)

VBA için API’ler, işletim sistemiyle doğrudan etkileşime izin veren bir dizi yöntem anlamına gelir.

DLL dosyalarında tanımlanan prosedürler uygulanarak sistem çağrıları yapılabilir.

Ortak işletim ortamı kitaplığı dosyaları (DLL’ler):

Dinamik Bağlantı Kitaplığı Açıklama
Advapi32.dll API’ler için birçok güvenlik ve Kayıt çağrısını içeren gelişmiş hizmetler kitaplığı
Comdlg32.dll Ortak iletişim API kitaplığı
Gdi32.dll Grafik Aygıt Arayüzü API kitaplığı
Kernel32.dll Çekirdek Windows 32 bit temel API desteği
Lz32.dll 32 bit sıkıştırma rutinleri
Mpr.dll Çoklu Sağlayıcı Yönlendirici kitaplığı
Netapi32.dll 32 bit Ağ API kitaplığı
Shell32.dll 32 bit Kabuk API kitaplığı
User32.dll Kullanıcı arabirimi rutinleri için kitaplık
Sürüm.dll Sürüm kitaplığı
Winmm.dll Windows multimedya kitaplığı
Winspool.drv Yazdırma biriktiricisi API çağrılarını içeren yazdırma biriktirici arabirimi

API bildirimi ve kullanımı #

Bir DLL prosedürü bildirme farklı VBA sürümleriyle çalışmak için:

Option Explicit

#If Win64 Then

    Private Declare PtrSafe Sub xLib "Kernel32" Alias "Sleep" (ByVal dwMilliseconds As Long)

#ElseIf Win32 Then

    Private Declare Sub apiSleep Lib "Kernel32" Alias "Sleep" (ByVal dwMilliseconds As Long)

#End If

Yukarıdaki bildirim, VBA’ya Kernel32.dll dosyasında tanımlanan “Sleep” işlevinin nasıl çağrılacağını söyler.

Win64 ve Win32, [koşullu derleme] için kullanılan önceden tanımlanmış sabitlerdir(https://www.wikiod.com/tr/vba/kosullu-derleme)

Önceden Tanımlanmış Sabitler

Bazı derleme sabitleri önceden tanımlanmıştır. Hangilerinin mevcut olduğu, VBA’yı çalıştırdığınız ofis sürümünün bit boyutuna bağlı olacaktır. Vba7’nin, Office’in 64 bit sürümlerini desteklemek için Office 2010 ile birlikte sunulduğunu unutmayın.

Sabit 16 bit 32 bit 64 bit
Vba6 Yanlış Eğer Vba6 Yanlış
Vba7 Yanlış Vba7 ise Doğru
Win16 Doğru Yanlış Yanlış
Win32 Yanlış Doğru Doğru
Win64 Yanlış Yanlış Doğru
Mac Yanlış Mac ise Mac ise

Bu sabitler, Windows sürümüne değil, Office sürümüne atıfta bulunur. Örneğin, işletim sistemi Windows’un 64 bit sürümü olsa bile, 32 bit Office’te Win32 = DOĞRU.

API’leri bildirirken temel fark, yeni parametre türlerini tanıtan 32 bit ve 64 bit Office sürümleri arasındadır (daha fazla ayrıntı için Açıklamalar bölümüne bakın)

Notlar:

  • Beyanlar modülün en üstüne ve herhangi bir Alt veya Fonksiyonun dışına yerleştirilir
  • Standart modüllerde bildirilen prosedürler varsayılan olarak geneldir
  • Bir modüle özel bir prosedür bildirmek için, bildirimden önce ‘Private’ anahtar kelimesiyle
  • Başka herhangi bir modül türünde bildirilen DLL prosedürleri, o modüle özeldir

Uyku API çağrısı için basit bir örnek:

Public Sub TestPause()

    Dim start As Double

    start = Timer

    Sleep 9000      'Pause execution for 9 seconds

    Debug.Print "Paused for " & Format(Timer - start, "#,###.000") & " seconds"

    'Immediate window result: Paused for 9.000 seconds

End Sub

VBA sarmalayıcılarından sistem işlevlerine kolay erişim sağlamak için özel bir API modülü oluşturulması önerilir – kitaplıklarda kullanılan parametreler ve bu parametrelerin başlatılması gibi gerçek sistem çağrısı için gereken ayrıntıları kapsayan normal VBA Yedekleri veya İşlevleri

Modül, tüm bildirimleri ve bağımlılıkları içerebilir:

  • Yöntem imzaları ve gerekli veri yapıları
  • Giriş doğrulaması yapan ve tüm parametrelerin beklendiği gibi iletilmesini sağlayan sarmalayıcılar

Bir DLL prosedürü bildirmek için, kod penceresinin Beyannameler bölümüne bir “Declare” ifadesi ekleyin.

Prosedür bir değer döndürürse, bunu bir İşlev olarak bildirin:

Declare Function publicname Lib "libname" [Alias "alias"] [([[ByVal] variable [As type] [,[ByVal] variable [As type]]...])] As Type

Bir prosedür bir değer döndürmezse, onu Alt olarak bildirin:

Declare Sub publicname Lib "libname" [Alias "alias"] [([[ByVal] variable [As type] [,[ByVal] variable [As type]]...])]

Mac için Office 2016 korumalı alan

VBA’yı destekleyen diğer Office uygulamaları sürümlerinden farklı olarak, Mac için Office 2016 uygulamaları korumalı alana sahiptir.

Korumalı alan, uygulamaların uygulama kapsayıcısı dışındaki kaynaklara erişmesini kısıtlar. Bu, işlemler arasında dosya erişimi veya iletişimi içeren tüm eklentileri veya makroları etkiler. Aşağıdaki bölümde açıklanan yeni komutları kullanarak korumalı alanın etkilerini en aza indirebilirsiniz. Mac için Office 2016 için yeni VBA komutları

    Const SM_CMONITORS = 80
    dllGetMonitors = GetSystemMetrics32(SM_CMONITORS)

    Const SM_CXVIRTUALSCREEN = 78

    Const SM_CYVIRTUALSCREEN = 79
