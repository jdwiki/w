Bağımsız Değişkenleri ByRef veya ByVal iletme

‘ByRef’ ve ‘ByVal’ değiştiricileri, bir prosedürün imzasının bir parçasıdır ve bir argümanın bir prosedüre nasıl iletildiğini gösterir. VBA’da aksi belirtilmedikçe bir parametre ‘ByRef’ iletilir (yani, yoksa ‘ByRef’ örtüktür).

Not Diğer birçok programlama dilinde (VB.NET dahil), değiştirici belirtilmemişse parametreler örtük olarak değere göre iletilir: olası karışıklığı önlemek için ‘ByRef’ değiştiricilerini açıkça belirtmeyi düşünün.

Geçiş dizileri #

Diziler başvuru yoluyla iletilmelidir. Bu kod derlenir, ancak 424 “Nesne Gerekli” çalışma zamanı hatası oluşturur:

Public Sub Test()
    DoSomething Array(1, 2, 3)
End Sub

Private Sub DoSomething(ByVal foo As Variant)
    foo.Add 42
End Sub

Bu kod derlenmiyor:

Private Sub DoSomething(ByVal foo() As Variant) 'ByVal is illegal for arrays
    foo.Add 42
End Sub

Basit Değişkenleri ByRef ve ByVal Geçirme #

‘ByRef’ veya ‘ByVal’ iletmek, bir argümanın gerçek değerinin ‘CallingProcedure’ tarafından ‘CalledProcedure’a mı iletildiğini veya bir referansın (diğer bazı dillerde işaretçi olarak adlandırılır) ‘CalledProcedure’a mı iletildiğini gösterir.

Bir bağımsız değişken ‘ByRef’ iletilirse, bağımsız değişkenin bellek adresi ‘CalledProcedure’a iletilir ve bu parametrede ‘CalledProcedure’ tarafından yapılan herhangi bir değişiklik, ‘CallingProcedure’daki değere yapılır.

Bir bağımsız değişken “ByVal” iletilirse, değişkene bir referans değil, gerçek değer “CalledProcedure"a iletilir.

Basit bir örnek bunu açıkça gösterecektir:

Sub CalledProcedure(ByRef X As Long, ByVal Y As Long)
    X = 321
    Y = 654
End Sub

Sub CallingProcedure()
    Dim A As Long
    Dim B As Long
    A = 123
    B = 456

    Debug.Print "BEFORE CALL => A: " & CStr(A), "B: " & CStr(B)
    ''Result : BEFORE CALL => A: 123 B: 456

    CalledProcedure X:=A, Y:=B

    Debug.Print "AFTER CALL =  A: " & CStr(A), "B: " & CStr(B)
    ''Result : AFTER CALL => A: 321 B: 456
End Sub

Başka bir örnek:

Sub Main()
    Dim IntVarByVal As Integer
    Dim IntVarByRef As Integer
    
    IntVarByVal = 5
    IntVarByRef = 10
    
    SubChangeArguments IntVarByVal, IntVarByRef '5 goes in as a "copy". 10 goes in as a reference
    Debug.Print "IntVarByVal: " & IntVarByVal 'prints 5 (no change made by SubChangeArguments)
    Debug.Print "IntVarByRef: " & IntVarByRef 'prints 99 (the variable was changed in SubChangeArguments)
End Sub

Sub SubChangeArguments(ByVal ParameterByVal As Integer, ByRef ParameterByRef As Integer)
    ParameterByVal = ParameterByVal + 2 ' 5 + 2 = 7 (changed only inside this Sub)
    ParameterByRef = ParameterByRef + 89 ' 10 + 89 = 99 (changes the IntVarByRef itself - in the Main Sub)
End Sub

ByRef #

Varsayılan değiştirici #

Bir parametre için herhangi bir değiştirici belirtilmemişse, bu parametre dolaylı olarak başvuru yoluyla iletilir.

Public Sub DoSomething1(foo As Long)
End Sub

Public Sub DoSomething2(ByRef foo As Long)
End Sub

“foo” parametresi, hem “DoSomething1” hem de “DoSomething2” içinde “ByRef” iletilir.

Dikkat edin! VBA’ya diğer dillerden edindiğiniz deneyimlerle geliyorsanız, bu büyük olasılıkla alışkın olduğunuz davranışın tam tersidir. Diğer birçok programlama dilinde (VB.NET dahil), örtük/varsayılan değiştirici, parametreleri değere göre iletir.

Referansa göre geçme #

  • Bir değer ‘ByRef’ iletildiğinde, prosedür değere bir referans alır.

    Public Sub Test()
    Dim foo As Long
    foo = 42
    DoSomething foo
    Debug.Print foo
End Sub

Private Sub DoSomething(ByRef foo As Long)
    foo = foo * 2
End Sub

    Calling the above Test procedure outputs 84. DoSomething is given foo and receives a reference to the value, and therefore works with the same memory address as the caller.

  • Bir referans ‘ByRef’ iletildiğinde, prosedür işaretçiye bir referans alır.

    Public Sub Test()
    Dim foo As Collection
    Set foo = New Collection
    DoSomething foo
    Debug.Print foo.Count
End Sub

Private Sub DoSomething(ByRef foo As Collection)
    foo.Add 42
    Set foo = Nothing
End Sub

    The above code raises [run-time error 91](https://www.wikiod.com/tr/vba/vba-calsma-zaman-hatalar#Çalışma zamanı hatası ‘91’: Nesne değişkeni veya Blok değişkeni ayarlanmadı), because the caller is calling the Count member of an object that no longer exists, because DoSomething was given a reference to the object pointer and assigned it to Nothing before returning.

Çağrı sitesinde ByVal’i zorlamak #

Çağrı sitesinde parantez kullanarak “ByRef"i geçersiz kılabilir ve bir argümanın “ByVal” iletilmesini sağlayabilirsiniz:

Public Sub Test()
    Dim foo As Long
    foo = 42
    DoSomething (foo)
    Debug.Print foo
End Sub

Private Sub DoSomething(ByRef foo As Long)
    foo = foo * 2
End Sub

Yukarıdaki kod, “ByRef"in örtük veya açık olarak belirtilmesine bakılmaksızın 42 çıktısını verir.

Dikkat edin! Bu nedenle, prosedür çağrılarında fazladan parantez kullanmak, kolayca hatalara neden olabilir. Prosedür adı ve bağımsız değişken listesi arasındaki boşluklara dikkat edin:

bar = DoSomething(foo) ‘işlev çağrısı, boşluk yok; parens args listesinin bir parçasıdır DoSomething (foo) ‘prosedür çağrısı, boşluk dikkat edin; parens args listesinin parçası DEĞİLDİR DoSomething foo ‘procedure call, foo parametresini ByVal olmaya zorlamaz

ByVal #

Değere göre geçiş #

  • Bir değer ‘ByVal’ iletildiğinde, prosedür değerin bir kopyasını alır.

    Public Sub Test()
    Dim foo As Long
    foo = 42
    DoSomething foo
    Debug.Print foo
End Sub

Private Sub DoSomething(ByVal foo As Long)
    foo = foo * 2
End Sub

    Calling the above Test procedure outputs 42. DoSomething is given foo and receives a copy of the value. The copy is multiplied by 2, and then discarded when the procedure exits; the caller’s copy was never altered.

  • Bir referans ‘ByVal’ iletildiğinde, prosedür işaretçinin bir kopyasını alır.

    Public Sub Test()
    Dim foo As Collection
    Set foo = New Collection
    DoSomething foo
    Debug.Print foo.Count
End Sub

Private Sub DoSomething(ByVal foo As Collection)
    foo.Add 42
    Set foo = Nothing
End Sub

    Calling the above Test procedure outputs 1. DoSomething is given foo and receives a copy of the pointer to the Collection object. Because the foo object variable in the Test scope points to the same object, adding an item in DoSomething adds the item to the same object. Because it’s a copy of the pointer, setting its reference to Nothing does not affect the caller’s own copy.
