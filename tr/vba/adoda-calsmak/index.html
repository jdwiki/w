Bu konuda gösterilen örnekler, netlik için erken bağlama kullanır ve Microsoft ActiveX Data Object x.x Kitaplığına başvuru gerektirir. Kesin olarak yazılan referansları “Object” ile değiştirerek ve “New” kullanarak nesne oluşturmayı uygun olduğunda “CreateObject” ile değiştirerek geç bağlamaya dönüştürülebilirler.

Bir veri kaynağına bağlantı kurma #

ADO aracılığıyla bir veri kaynağına erişmenin ilk adımı, bir ADO `Connection’ nesnesi yaratmaktır. Bu, genellikle veri kaynağı parametrelerini belirtmek için bir bağlantı dizesi kullanılarak yapılır, ancak DSN, kullanıcı kimliği ve parolayı “.Open” yöntemine geçirerek bir DSN bağlantısı açmak da mümkündür.

ADO aracılığıyla bir veri kaynağına bağlanmak için bir DSN’nin gerekli olmadığını unutmayın - ODBC sağlayıcısı olan herhangi bir veri kaynağına uygun bağlantı dizesiyle bağlanabilir. Farklı sağlayıcılar için belirli bağlantı dizeleri bu konunun kapsamı dışında olsa da, ConnectionStrings.com, sağlayıcınız için uygun dizeyi bulmak için mükemmel bir referanstır.

Const SomeDSN As String = "DSN=SomeDSN;Uid=UserName;Pwd=MyPassword;"

Public Sub Example()
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    If Not database Is Nothing Then
        '... Do work.
        database.Close          'Make sure to close all database connections.
    End If
End Sub

Public Function OpenDatabaseConnection(ConnString As String) As ADODB.Connection
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = New ADODB.Connection
    
    With database
        .ConnectionString = ConnString
        .ConnectionTimeout = 10             'Value is given in seconds.
        .Open
    End With
     
    OpenDatabaseConnection = database
     
    Exit Function
Handler:
    Debug.Print "Database connection failed. Check your connection string."
End Function

Veritabanı parolasının yukarıdaki örnekteki bağlantı dizesine yalnızca açıklık amacıyla dahil edildiğini unutmayın. En iyi uygulamalar, veritabanı parolalarının kodda saklanmasını *** değil*** gerektirir. Bu, kullanıcı girişi yoluyla parola alınarak veya Windows kimlik doğrulaması kullanılarak gerçekleştirilebilir.

Parametreli komutlar oluşturma #

Bir ADO bağlantısı aracılığıyla yürütülen SQL’in kullanıcı girdisi içermesi gerektiğinde, SQL enjeksiyon olasılığını en aza indirmek için onu parametreleştirmek en iyi uygulama olarak kabul edilir. Bu yöntem ayrıca uzun birleştirmelerden daha okunabilirdir ve daha sağlam ve sürdürülebilir kodu kolaylaştırır (yani, bir “Parametre” dizisi döndüren bir işlev kullanarak).

Standart ODBC sözdiziminde, parametrelere sorgu metninde “?” “yer tutucuları” verilir ve ardından parametreler, sorguda göründükleri sırayla “Komut"a eklenir.

Aşağıdaki örneğin, [Bir veri kaynağına bağlantı kurma](https://www.wikiod.com/tr/vba/adoda-calsmak#Bir veri kaynağına bağlantı kurma) öğesindeki ‘OpenDatabaseConnection’ işlevini kullandığını unutmayın. -to-a-veri-kaynağı) kısalık için.

Public Sub UpdateTheFoos()
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    
    If Not database Is Nothing Then
        Dim update As ADODB.Command
        Set update = New ADODB.Command
        'Build the command to pass to the data source.
        With update
            .ActiveConnection = database
            .CommandText = "UPDATE Table SET Foo = ? WHERE Bar = ?"
            .CommandType = adCmdText
            
            'Create the parameters.
            Dim fooValue As ADODB.Parameter
            Set fooValue = .CreateParameter("FooValue", adNumeric, adParamInput)
            fooValue.Value = 42
            
            Dim condition As ADODB.Parameter
            Set condition = .CreateParameter("Condition", adBSTR, adParamInput)
            condition.Value = "Bar"
            
            'Add the parameters to the Command
            .Parameters.Append fooValue
            .Parameters.Append condition
            .Execute
        End With
    End If
CleanExit:
    If Not database Is Nothing And database.State = adStateOpen Then
        database.Close
    End If
    Exit Sub
Handler:
    Debug.Print "Error " & Err.Number & ": " & Err.Description
    Resume CleanExit
End Sub

Not: Yukarıdaki örnek, parametreli bir UPDATE deyimini göstermektedir, ancak herhangi bir SQL deyimine parametreler verilebilir.

Sorgu ile kayıtları alma #

Sorgular, her ikisi de döndürülen satırların bir koleksiyonu olan bir ADO `Recordset’ nesnesi döndüren iki şekilde gerçekleştirilebilir. Aşağıdaki örneklerin her ikisinin de [Veri kaynağıyla bağlantı kurma](https://www.wikiod.com/tr/vba/adoda-calsmak#Bir veri kaynağına bağlantı kurma) öğesindeki “OpenDatabaseConnection” işlevini kullandığını unutmayın. -veri kaynağına bağlantı) örneği, kısalık amacıyla. Veri kaynağına iletilen SQL sözdiziminin sağlayıcıya özel olduğunu unutmayın.

İlk yöntem, SQL deyimini doğrudan Connection nesnesine iletmektir ve basit sorguları yürütmek için en kolay yöntemdir:

Public Sub DisplayDistinctItems()
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    
    If Not database Is Nothing Then
        Dim records As ADODB.Recordset
        Set records = database.Execute("SELECT DISTINCT Item FROM Table")
        'Loop through the returned Recordset.
        Do While Not records.EOF      'EOF is false when there are more records.
            'Individual fields are indexed either by name or 0 based ordinal.
            'Note that this is using the default .Fields member of the Recordset.
            Debug.Print records("Item")
            'Move to the next record.
            records.MoveNext
        Loop
    End If
CleanExit:
    If Not records Is Nothing Then records.Close
    If Not database Is Nothing And database.State = adStateOpen Then
        database.Close
    End If
    Exit Sub
Handler:
    Debug.Print "Error " & Err.Number & ": " & Err.Description
    Resume CleanExit
End Sub

İkinci yöntem, yürütmek istediğiniz sorgu için bir ADO Command nesnesi oluşturmaktır. Bu biraz daha fazla kod gerektirir, ancak parametreli sorguları kullanmak için gereklidir:

Public Sub DisplayDistinctItems()
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    
    If Not database Is Nothing Then
        Dim query As ADODB.Command
        Set query = New ADODB.Command
        'Build the command to pass to the data source.
        With query
            .ActiveConnection = database
            .CommandText = "SELECT DISTINCT Item FROM Table"
            .CommandType = adCmdText
        End With
        Dim records As ADODB.Recordset
        'Execute the command to retrieve the recordset.
        Set records = query.Execute()

        Do While Not records.EOF
            Debug.Print records("Item")
            records.MoveNext
        Loop
    End If
CleanExit:
    If Not records Is Nothing Then records.Close
    If Not database Is Nothing And database.State = adStateOpen Then
        database.Close
    End If
    Exit Sub
Handler:
    Debug.Print "Error " & Err.Number & ": " & Err.Description
    Resume CleanExit
End Sub

Veri kaynağına gönderilen komutların kasıtlı veya kasıtsız olarak SQL enjeksiyonuna karşı savunmasız olduğunu unutmayın. Genel olarak, herhangi bir tür kullanıcı girdisini birleştirerek sorgular oluşturulmamalıdır. Bunun yerine, parametrelendirilmeleri gerekir (bkz. [Parametreli komutlar oluşturma](https://www.wikiod.com/tr/vba/adoda-calsmak#Parametreli komutlar oluşturma)).

Skaler olmayan işlevleri yürütme #

ADO bağlantıları, sağlayıcının SQL aracılığıyla desteklediği hemen hemen tüm veritabanı işlevlerini gerçekleştirmek için kullanılabilir. Bu durumda, @@Identity veya benzer SQL komutları ile INSERT deyimlerinden sonra tuş atamaları almak için yararlı olsa da, ‘Execute’ işlevi tarafından döndürülen ‘Recordset’i kullanmak her zaman gerekli değildir. Aşağıdaki örneğin, [Bir veri kaynağına bağlantı kurma](https://www.wikiod.com/tr/vba/adoda-calsmak#Bir veri kaynağına bağlantı kurma) öğesindeki ‘OpenDatabaseConnection’ işlevini kullandığını unutmayın. -to-a-veri-kaynağı) kısalık amacıyla örnek.

Public Sub UpdateTheFoos()
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    
    If Not database Is Nothing Then
        Dim update As ADODB.Command
        Set update = New ADODB.Command
        'Build the command to pass to the data source.
        With update
            .ActiveConnection = database
            .CommandText = "UPDATE Table SET Foo = 42 WHERE Bar IS NULL"
            .CommandType = adCmdText
            .Execute        'We don't need the return from the DB, so ignore it.
        End With
    End If
CleanExit:
    If Not database Is Nothing And database.State = adStateOpen Then
        database.Close
    End If
    Exit Sub
Handler:
    Debug.Print "Error " & Err.Number & ": " & Err.Description
    Resume CleanExit
End Sub

Veri kaynağına gönderilen komutların kasıtlı veya kasıtsız olarak SQL enjeksiyonuna karşı savunmasız olduğunu unutmayın. Genel olarak, SQL ifadeleri herhangi bir türde kullanıcı girdisini birleştirerek oluşturulmamalıdır. Bunun yerine, parametrelendirilmeleri gerekir (bkz. [Parametreli komutlar oluşturma](https://www.wikiod.com/tr/vba/adoda-calsmak#Parametreli komutlar oluşturma)).
