Sözdizimi #

  • işaretsiz uzun millis()

  • işaretsiz uzun mikrolar()

  • geçersiz gecikme (işaretsiz uzun milisaniye)

  • geçersiz gecikmeMikrosaniye(işaretsiz uzun mikrosaniye)

  • Bu sınıfın yapıcıları ve işleçleri için geçenMillis üstbilgisine bakın. Kısacası:

  • geçenMillis geçenMillisObject; yaratıldığından beri veya zaman içinde açıkça belirlenmiş başka bir noktadan beri zamanı takip etmek için bir nesne yaratır

  • geçenMillisObject = 0; nesne tarafından izlenen süreyi “şimdiden beri” olarak sıfırlayın

  • imzasız uzun deltaT = geçenMillisObject; izlenen zamana bakmamıza izin verir

  • elapsedMillisObject += ve -= bunlar beklendiği gibi çalışır

Engelleme ve engelleme olmayan kod #

Çok basit çizimler için, “delay()” ve “delayMicroseconds()” kullanarak engelleme kodu yazmak uygun olabilir. İşler daha karmaşık hale geldiğinde, bu işlevleri kullanmanın bazı dezavantajları olabilir. Bunlardan bazıları:

  • CPU zamanını boşa harcama: Daha karmaşık çizimler, bir LED yanıp sönme süresinin bitmesini beklerken CPU’ya başka bir şey için ihtiyaç duyabilir.
  • beklenmeyen gecikmeler: Açıkça çağrılmayan alt rutinlerde, örneğin eklediğiniz kitaplıklarda “delay()” çağrıldığında.
  • gecikme sırasında meydana gelen ve bir kesme işleyicisi tarafından işlenmeyen eksik olaylar, örneğin yoklamalı düğme basışları: Bir düğmeye 100 ms boyunca basılabilir, ancak bu bir ‘gecikme(500)’ tarafından gölgelenebilir.

Uygulama ayrıntıları #

“millis()” genellikle 1 kHz’den çok daha yüksek bir hızda çalışan bir donanım zamanlayıcısına dayanır. millis() çağrıldığında, uygulama bir miktar değer döndürür, ancak bunun gerçekte kaç yaşında olduğunu bilmiyorsunuz. “Geçerli” milisaniyenin yeni başlamış olması veya bu işlev çağrısından hemen sonra bitmesi mümkündür. Bu, “millis()“den gelen iki sonuç arasındaki farkı hesaplarken, neredeyse sıfır ile neredeyse bir milisaniye arasında herhangi bir farkla çıkabileceğiniz anlamına gelir. Daha yüksek hassasiyet gerekiyorsa micros() kullanın.

’elapsedMillis’in kaynak koduna bakıldığında, zaman içindeki iki noktayı karşılaştırmak için gerçekten de ‘millis()‘i dahili olarak kullandığını ve dolayısıyla bu etkiden muzdarip olduğunu ortaya çıkarır. Yine, aynı kitaplıktan daha yüksek hassasiyet için ’elapsedMicros’ alternatifi var.

millis() ile engellenmeyen göz kırpma #

Bu, arduino dokümanlarından bir örneğe çok yakın:

// set constants for blinking the built-in LED at 1 Hz
#define OUTPIN LED_BUILTIN
#define PERIOD 500  // this is in milliseconds

int ledState = LOW;

// millis() returns an unsigned long so we'll use that to keep track of time
unsigned long lastTime = 0;

void setup() {
  // set the digital pin as output:
  pinMode(OUTPIN, OUTPUT);
}

void loop() {
  unsigned long now = millis();
  if (now - lastTime >= PERIOD) // this will be true every PERIOD milliseconds
  {
    lastTime = now;
    if (ledState == LOW)
    {
      ledState = HIGH;
    }
    else
    {
      ledState = LOW;
    }
    digitalWrite(OUTPIN, ledState);
  }

  // now there's lots of time to do other stuff here
}

“millis()“i bu şekilde kullanmak - işlemleri engellemeyen bir şekilde zamanlamak için - oldukça sık ihtiyaç duyulan bir şeydir, bu nedenle bunun için “elapsedMillis” kitaplığını kullanmayı düşünün.

elapsedMillis kitaplığı (ve sınıfı) ile engellenmeyen göz kırpma #

elapsedMillis library, oluşturulduğundan veya belirli bir değere ayarlandığından bu yana geçen sürenin kaydını tutan aynı ada sahip bir sınıf sağlar:

#include <elapsedMillis.h>

#define OUTPIN LED_BUILTIN
#define PERIOD 500

elapsedMillis ledTime;

bool ledState = false;

void setup() 
{                
  // initialize the digital pin as an output.
  pinMode(OUTPIN, OUTPUT);     
}

void loop()
{
    if (ledTime >= PERIOD) 
    {                
        ledState = !ledState;
        digitalWrite(OUTPIN, ledState);
        ledTime = 0;
    }
    // do other stuff here
}

Örnekte LED pini değiştirildiğinde ledTime nesnesine sıfır atandığını görebilirsiniz. Bu ilk bakışta şaşırtıcı olmayabilir, ancak daha fazla zaman alan şeyler oluyorsa etkisi vardır:

“ledTime” ve “PERIOD” arasındaki karşılaştırmanın 750 milisaniye sonra yapıldığı bir durumu düşünün. Ardından “ledTime"ı sıfıra ayarlamak, aşağıdaki tüm geçiş işlemlerinin 250 ms “geç” olacağı anlamına gelir. Aksine, ‘LEDTime’dan ‘PERIOD’ çıkarılırsa, LED kısa bir süre görecek ve ardından hiçbir şey olmamış gibi yanıp sönmeye devam edecektir.

gecikmeyle yanıp sönmeyi engelleme() #

LED’in yanıp sönmesini sağlamanın en basit yollarından biri şudur: açın, biraz bekleyin, kapatın, tekrar bekleyin ve durmadan tekrarlayın:

// set constants for blinking the built-in LED at 1 Hz
#define OUTPIN LED_BUILTIN
#define PERIOD 500

void setup()
{
  pinMode(OUTPIN, OUTPUT);      // sets the digital pin as output
}

void loop()
{
  digitalWrite(OUTPIN, HIGH);   // sets the pin on
  delayMicroseconds(PERIOD);        // pauses for 500 miliseconds      
  digitalWrite(OUTPIN, LOW);    // sets the pin off
  delayMicroseconds(PERIOD);        // pauses for 500 milliseconds

  // doing other time-consuming stuff here will skew the blinking
}

Bununla birlikte, yukarıdaki örnekte olduğu gibi beklemek, CPU döngülerini boşa harcar, çünkü zamanın belirli bir noktasının geçmesini bekleyen bir döngüde öylece durur. ‘millis()’ veya ’elapsedMillis’ kullanan engellemesiz yolların daha iyi yaptığı şey budur - donanımın yeteneklerinin çoğunu yakmamaları anlamında.

elapsedMillis ve elapsedMicros kullanarak bir şeyin ne kadar sürdüğünü ölçün #

#include <elapsedMillis.h>

void setup() {
  Serial.begin(115200);
  elapsedMillis msTimer;
  elapsedMicros usTimer;

  long int dt = 500;
  delay(dt);

  long int us = usTimer;
  long int ms = msTimer;

  Serial.print("delay(");Serial.print(dt);Serial.println(") took");
  Serial.print(us);Serial.println(" us, or");
  Serial.print(ms);Serial.println(" ms");
}

void loop() {
}

Bu örnekte, bir ’elapsedMillis’ nesnesi ve bir ’elapsedMicros’ nesnesi, bir şeyin ne kadar sürdüğünü ölçmek için kullanılır, bunları zamanlamak istediğimiz ifade yürütülmeden hemen önce oluşturup daha sonra değerlerini alarak. Biraz farklı sonuçlar gösterecekler, ancak milisaniye sonucu bir milisaniyeden fazla kaybolmayacak.

Gecikmeden 1’den fazla görev() #

Farklı aralıklarla art arda yürütmek için 1’den fazla göreviniz varsa, bu örneği başlangıç ​​noktası olarak kullanın:

unsigned long intervals[] = {250,2000}; //this defines the interval for each task in milliseconds
unsigned long last[] = {0,0};           //this records the last executed time for each task

void setup() {
  pinMode(LED_BUILTIN, OUTPUT); //set the built-it led pin as output
  Serial.begin(115200);         //initialize serial
}

void loop() {
  unsigned long now = millis();
  if(now-last[0]>=intervals[0]){ last[0]=now; firstTask(); }
  if(now-last[1]>=intervals[1]){ last[1]=now; secondTask(); }
  
  //do other things here
}

void firstTask(){
  //let's toggle the built-in led
  digitalWrite(LED_BUILTIN, digitalRead(LED_BUILTIN)?0:1);
}

void secondTask(){
  //say hello
  Serial.println("hello from secondTask()");
}

Her 15 saniyede bir yürütülecek başka bir görev eklemek için “intervals” ve “last” değişkenlerini uzatın:

unsigned long intervals[] = {250,2000,15000};
unsigned long last[] = {0,0,0};

Ardından yeni görevi yürütmek için bir ‘if’ ifadesi ekleyin. Bu örnekte, onu ‘üçüncüGörev’ olarak adlandırdım.

if(now-last[2]>=intervals[2]){ last[2]=now; thirdTask(); }

Sonunda işlevi ilan edin:

void thirdTask(){
  //your code here
}
