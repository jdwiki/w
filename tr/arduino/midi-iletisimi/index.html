MIDI THRU Örneği #

MIDI Thru basit ve test edilmesi kolaydır. Düzgün çalışırken Arduino projenizi iki MIDI cihazı arasında, MIDI IN’den MIDI OUT’a kurabileceksiniz ve iki cihazın birlikte çalıştığını doğrulayabileceksiniz. Gecikmeyi ölçme yeteneğiniz varsa, seri arabellek yakalama ve yeniden iletme talimatları nedeniyle bir artış göreceksiniz.

// This is a simple MIDI THRU.  Everything in, goes right out.
// This has been validate on an Arduino UNO and a Olimex MIDI Shield  

boolean byteReady; 
unsigned char midiByte;

void setup() {
    // put your setup code here, to run once:
    //  Set MIDI baud rate:
    Serial.begin(31250);
    byteReady = false;
    midiByte = 0;  
}

// The Loop that always gets called...
void loop() {
   if (byteReady) {
        byteReady = false;
        Serial.write(midiByte);
    }
}

// The little function that gets called each time loop is called.  
// This is automated somwhere in the Arduino code.
void serialEvent() {
  if (Serial.available()) {
    // get the new byte:
    midiByte = (unsigned char)Serial.read();
    byteReady = true;
  }
}

Sıralı MIDI Thru #

// This is a more complex MIDI THRU.  This version uses a queue.  Queues are important because some
// MIDI messages can be interrupted for real time events.  If you are generating your own messages,
// you may need to stop your message to let a "real time" message through and then resume your message.


#define QUEUE_DEPTH 128

// Queue Logic for storing messages
int headQ = 0;
int tailQ = 0;
unsigned char tx_queue[QUEUE_DEPTH];

void setup() {
    // put your setup code here, to run once:
    //  Set MIDI baud rate:
    Serial.begin(31250);
}

// getQDepth checks for roll over.  Folks have told me this 
// is not required.  Feel free to experiment.
int getQDepth() {
int depth = 0;
    if (headQ < tailQ) {
        depth = QUEUE_DEPTH - (tailQ - headQ);
    } else {
        depth = headQ - tailQ;
    }
    return depth;
}

void addQueue (unsigned char myByte) {
    int depth = 0;
    depth = getQDepth();

    if (depth < (QUEUE_DEPTH-2)) {
        tx_queue[headQ] = myByte;
        headQ++;
        headQ = headQ % QUEUE_DEPTH; // Always keep the headQ limited between 0 and 127
    }
}

unsigned char deQueue() {
    unsigned char myByte;
    myByte = tx_queue[tailQ];
    tailQ++;
    tailQ = tailQ % QUEUE_DEPTH;  // Keep this tailQ contained within a limit
    // Now that we dequeed the byte, it must be sent. 
    return myByte;
}

void loop() {
   if (getQDepth>0) {
        Serial.write(deQueue());
    }
}

// The little function that gets called each time loop is called.  
// This is automated somwhere in the Arduino code.
void serialEvent() {
  if (Serial.available()) {
    // get the new byte:
    addQueue((unsigned char)Serial.read());;
  }
}

MIDI Saat Üretimi #

// This is a MiDI clk generator.  This takes a #defined BPM and 
// makes the appropriate clk rate.  The queue is used to let other messages 
// through, but allows a clock to go immediately to reduce clock jitter

#define QUEUE_DEPTH 128
#define BPM 121
#define MIDI_SYSRT_CLK 0xF8

// clock tracking and calculation
unsigned long lastClock;
unsigned long captClock;
unsigned long clk_period_us;

// Queue Logic for storing messages
int headQ = 0;
int tailQ = 0;
unsigned char tx_queue[QUEUE_DEPTH];

void setup() {
    //  Set MIDI baud rate:
    Serial.begin(31250);
    clk_period_us = 60000000 / (24 * BPM);
    lastClock = micros();
}

// getQDepth checks for roll over.  Folks have told me this 
// is not required.  Feel free to experiment.
int getQDepth() {
int depth = 0;
    if (headQ < tailQ) {
        depth = QUEUE_DEPTH - (tailQ - headQ);
    } else {
        depth = headQ - tailQ;
    }
    return depth;
}

void addQueue (unsigned char myByte) {
    int depth = 0;
    depth = getQDepth();

    if (depth < (QUEUE_DEPTH-2)) {
        tx_queue[headQ] = myByte;
        headQ++;
        headQ = headQ % QUEUE_DEPTH; // Always keep the headQ limited between 0 and 127
    }
}

unsigned char deQueue() {
    unsigned char myByte;
    myByte = tx_queue[tailQ];
    tailQ++;
    tailQ = tailQ % QUEUE_DEPTH;  // Keep this tailQ contained within a limit
    // Now that we dequeed the byte, it must be sent. 
    return myByte;
}

void loop() {
    captClock = micros();
    
    if (lastClock > captClock) {
        // we have a roll over condition - Again, maybe we don't need to do this.
        if (clk_period_us <= (4294967295 - (lastClock - captClock))) {
            // Add a the ideal clock period for this BPM to the last measurement value
            lastClock = lastClock + clk_period_us;
            // Send a clock, bypasing the transmit queue
            Serial.write(MIDI_SYSRT_CLK);
        }
    } else if (clk_period_us <= captClock-lastClock) {
        // Basically the same two commands above, but not within a roll over check
        lastClock = lastClock + clk_period_us;
        // Send a clock, bypasing the transmit queue
        Serial.write(MIDI_SYSRT_CLK);
    }

    if (getQDepth>0) {
        Serial.write(deQueue());
    }
}

// The little function that gets called each time loop is called.  
// This is automated somwhere in the Arduino code.
void serialEvent() {
  if (Serial.available()) {
    // get the new byte:
    addQueue((unsigned char)Serial.read());;
  }
}

MIDI Mesajları Tanımlandı #

Genel olarak, MIDI protokolü “mesajlara” bölünür. 4 genel mesaj sınıfı vardır:

  • Kanal Sesi
  • Kanal Modu
  • Sistem Ortak
  • Sistem Gerçek Zamanlı Mesajları

Mesajlar, 0x80’in üzerinde bir bayt değeriyle başlar. 0x7F’nin altındaki herhangi bir değer veri olarak kabul edilir. Etkili bir şekilde, 127’nin tek bir MIDI veri baytına kodlanabilecek maksimum değer olduğu anlamına gelir. Daha büyük değerleri kodlamak için iki veya daha fazla MIDI veri baytı gereklidir.

Unutulmamalıdır ki, mesajların baştan sona kesintisiz olarak gönderilmesi gerekir… HARİÇ… Sistem Gerçek Zamanlı mesajlar, herhangi bir mesajın ortasına enjekte edilebilir, tek bayttır.

Kanal Sesli Mesajları

Durum D7..D0 Veri Baytları Açıklama
1000nnnn 0kkkkkkk 0vvvvvvv Note Off olayı. Bu mesaj, bir not bırakıldığında (bittiğinde) gönderilir. (kkkkkkk) anahtar (not) numarasıdır. (vvvvvvv) hızdır.
1001nnn 0kkkkkkk 0vvvvvvv Etkinlik Üzerine Not. Bu mesaj, bir nota basıldığında (başlat) gönderilir. (kkkkkkk) anahtar (not) numarasıdır. (vvvvvvv) hızdır.
1010nnn 0kkkkkkk 0vvvvvvv Polifonik Tuş Basıncı (Aftertouch). Bu mesaj çoğunlukla “dipten çıktıktan” sonra tuşa basılarak gönderilir. (kkkkkkk) anahtar (not) numarasıdır. (vvvvvvv) basınç değeridir.
1011nnn 0ccccccc 0vvvvvvv Kontrol Değişimi. Bu mesaj, bir kontrolör değeri değiştiğinde gönderilir. Kontrolörler, pedallar ve kollar gibi cihazları içerir. 120-127 numaralı kontrolör “Kanal Modu Mesajları” olarak ayrılmıştır (aşağıda). (ccccccc) denetleyici numarasıdır (0-119). (vvvvvvv) denetleyici değeridir (0-127).
1100nnn 0pppppp Programı Değiştir. Bu mesaj, yama numarası değiştiğinde gönderilir. (ppppppp) yeni program numarasıdır.
1101nnn 0vvvvvvv Kanal Basıncı (Dokunma Sonrası). Bu mesaj çoğunlukla “dipten çıktıktan” sonra tuşa basılarak gönderilir. Bu mesaj, polifonik art dokunuştan farklıdır. Tek en büyük basınç değerini göndermek için bu mesajı kullanın (tüm geçerli basılan tuşların). (vvvvvvv) basınç değeridir.
1110nnnn 0llllll 0mmmmmm Pitch Bend Değişimi. Bu mesaj, hatve bükücüdeki (tipik olarak tekerlek veya kol) bir değişikliği belirtmek için gönderilir. Perde bükücü on dört bitlik bir değerle ölçülür. Merkez (perde değişikliği yok) 2000H’dir. Duyarlılık alıcının bir işlevidir, ancak RPN 0 kullanılarak ayarlanabilir. (llllllll) en az anlamlı 7 bittir. (mmmmmmm) en önemli 7 bittir.

Kanal Modu Mesajları

Durum D7..D0 Veri Baytları Açıklama
1011nnnn 0ccccccc 0vvvvvvv Kanal Modu Mesajları. Bu, Kontrol Değişikliği (yukarıda) ile aynı koddur, ancak 120-127 ayrılmış kontrolör numaralarını kullanarak Mod kontrolünü ve özel mesajı uygular. Komutlar şunlardır:
Tüm Sesler Kapalı. All Sound Off alındığında, tüm osilatörler kapanacak ve ses zarfları mümkün olan en kısa sürede sıfıra ayarlanacaktır. c = 120, v = 0: Tüm Sesler Kapalı
Tüm Denetleyicileri Sıfırla. Reset All Controllers alındığında, tüm kontrolör değerleri varsayılan değerlerine sıfırlanır. (Varsayılanlar için belirli Önerilen Uygulamalara bakın).
c = 121, v = x: Belirli bir Tavsiye Edilen Uygulamada aksi belirtilmedikçe değer yalnızca sıfır olmalıdır.
Yerel kontrol. Yerel Kontrol Kapalı olduğunda, belirli bir kanaldaki tüm cihazlar yalnızca MIDI üzerinden alınan verilere yanıt verecektir. Oynatılan veriler vb. yok sayılır. Yerel Kontrol Açık, normal denetleyicilerin işlevlerini geri yükler.
c = 122, v = 0: Yerel Kontrol Kapalı
c = 122, v = 127: Yerel Kontrol Açık
Tüm Notlar Kapalı. Tüm Notlar Kapalı alındığında, tüm osilatörler kapanacaktır.
c = 123, v = 0: Tüm Notlar Kapalı (Gerçek mod komutlarının açıklaması için metne bakın.)
c = 124, v = 0: Çok Yönlü Mod Kapalı
c = 125, v = 0: Çok Yönlü Mod Açık
c = 126, v = M: Mono Modu Açık (Poli Kapalı) burada M, kanal sayısıdır (Omni Off) veya 0 (Omni On) ​​
c = 127, v = 0: Çoklu Mod Açık (Mono Kapalı) (Not: Bu dört mesaj ayrıca Tüm Notlar Kapalı’ya neden olur)

Sistem Genel Mesajları

Durum D7..D0 Veri Baytları Açıklama
11110000 0iiiiiii [0iiiiiii 0iiiiiii] 0dddddd — — 0dddddd 11110111 Sisteme Özel. Bu mesaj türü, üreticilerin kendi mesajlarını (toplu dökümler, yama parametreleri ve diğer spesifik olmayan veriler gibi) oluşturmalarına izin verir ve ek MIDI Spesifikasyon mesajları oluşturmak için bir mekanizma sağlar. Üreticinin Kimlik kodu (MMA veya AMEI tarafından atanır) ya 1 bayt (0iiiiiii) ya da 3 bayttır (0iiiiiii 0iiiiiii 0iiiiii). 1 Bayt Kimlikten ikisi, üreticiye özel olmayan Evrensel Özel Mesajlar adı verilen uzantılar için ayrılmıştır. Bir cihaz, kimlik kodunu kendi (veya desteklenen bir Evrensel mesaj) olarak tanırsa, mesajın geri kalanını (0ddddddd) dinler. Aksi takdirde mesaj dikkate alınmayacaktır. (Not: Yalnızca Gerçek Zamanlı iletiler, bir Sisteme Özel ile araya eklenebilir.)
11110001 0nndddd MIDI Zaman Kodu Çeyrek Çerçeve. nnn = İleti Türü dddd = Değerler
11110010 0llllll 0mmmmmm Şarkı Konumu İşaretçisi. Bu, şarkının başlangıcından itibaren MIDI vuruşlarının sayısını (1 vuruş = altı MIDI saati) tutan dahili 14 bitlik bir kayıttır. l LSB, MSB m.
11110011 0ssssss Şarkı Seç. Song Select, hangi dizinin veya şarkının çalınacağını belirtir.
11110100 Tanımsız. (Ayrılmış)
11110101 Tanımsız. (Ayrılmış)
11110110 Ayar İsteği. Bir Ayar İsteği aldıktan sonra, tüm analog sentezleyiciler osilatörlerini ayarlamalıdır.
11110111 Exclusive’in Sonu. Sisteme Özel dökümü sonlandırmak için kullanılır (yukarıya bakın).

Sistem Gerçek Zamanlı Mesajları

Durum D7..D0 Veri Baytları Açıklama
11111000 Zamanlama Saati. Senkronizasyon gerektiğinde çeyrek nota başına 24 kez gönderilir (metne bakın).
11111001 Tanımsız. (Ayrılmış)
11111010 Başlama. Geçerli diziyi çalmaya başlayın. (Bu mesaj, Zamanlama Saatleri ile takip edilecektir).
11111011 Devam et. Dizinin Durdurulduğu noktada devam edin.
11111100 Durmak. Geçerli diziyi durdurun.
11111101 Tanımsız. (Ayrılmış)
11111110 Aktif Algılama. Bu mesajın alıcıya bir bağlantının canlı olduğunu bildirmek için tekrar tekrar gönderilmesi amaçlanmıştır. Bu mesajın kullanımı isteğe bağlıdır. İlk alındığında, alıcı her 300ms’de bir (maks) bir başka Aktif Algılama mesajı almayı bekleyecek ve eğer almazsa bağlantının kesildiğini varsayacaktır. Sonlandırmada, alıcı tüm sesleri kapatacak ve normal (aktif olmayan algılama) çalışmasına dönecektir.
11111111 Sıfırla. Sistemdeki tüm alıcıları açılış durumuna sıfırlayın. Bu, tercihen manuel kontrol altında, dikkatli bir şekilde kullanılmalıdır. Özellikle açılışta gönderilmemelidir.
