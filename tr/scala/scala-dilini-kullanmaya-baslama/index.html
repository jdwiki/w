Uygulamayı genişleterek Merhaba Dünya #

object HelloWorld extends App {
  println("Hello, world!")
}

Canlı demo

App özellik‘i genişleterek, açık bir ‘main’ yöntemi tanımlamaktan kaçınabilirsiniz. . “HelloWorld” nesnesinin tüm gövdesi “ana yöntem” olarak kabul edilir.

Gecikmeli Başlatma #

resmi belgelere göre, “Uygulama” Gecikmeli Başlatma. Bu, nesne alanlarının ana yöntem çağrıldıktan sonra başlatıldığı anlamına gelir.

“DelayedInit” artık genel kullanım için kullanımdan kaldırıldı, ancak özel bir durum olarak “Uygulama” için hâlâ destekleniyor. Destek, bir değiştirme özelliğine karar verilinceye ve uygulanıncaya kadar devam edecektir.

son sürüm

“Uygulama"yı genişletirken komut satırı bağımsız değişkenlerine erişmek için “this.args"ı kullanın:

object HelloWorld extends App {
  println("Hello World!")
  for {
    arg <- this.args
  } println(s"Arg=$arg")
}

‘Uygulama’ kullanılırken, nesnenin gövdesi ‘ana’ yöntem olarak yürütülür, ‘ana’yı geçersiz kılmaya gerek yoktur.

Bir ‘ana’ Yöntem Tanımlayarak Merhaba Dünya #

Bu kodu HelloWorld.scala adlı bir dosyaya yerleştirin:

object Hello {
  def main(args: Array[String]): Unit = {
    println("Hello World!")
  }
}

Canlı demo

JVM tarafından yürütülebilir bayt koduna derlemek için:

$ scalac HelloWorld.scala

Çalıştırmak için:

$ scala Hello

Scala çalışma zamanı programı yüklediğinde, ‘main’ yöntemiyle ‘Merhaba’ adlı bir nesne arar. ‘Main’ yöntemi, program giriş noktasıdır ve yürütülür.

Java’dan farklı olarak, Scala’nın nesneleri veya sınıfları içinde bulundukları dosyadan sonra adlandırma gereksinimi olmadığını unutmayın. Bunun yerine, ‘scala Merhaba’ komutunda geçirilen ‘Merhaba’ parametresi, ‘ana’yı içeren aranacak nesneye atıfta bulunur. yöntem yürütülecek. Aynı .scala` dosyasında ana yöntemlerle birden fazla nesneye sahip olmak tamamen mümkündür.

“args” dizisi, varsa, programa verilen komut satırı argümanlarını içerecektir. Örneğin, programı şu şekilde değiştirebiliriz:

object HelloWorld {
  def main(args: Array[String]): Unit = {
    println("Hello World!")
    for {
      arg <- args
    } println(s"Arg=$arg")
  }
}

Derleyin:

$ scalac HelloWorld.scala

Ve sonra yürütün:

$ scala HelloWorld 1 2 3
Hello World!
Arg=1
Arg=2
Arg=3

Senaryo olarak Merhaba Dünya #

Scala bir betik dili olarak kullanılabilir. Göstermek için aşağıdaki içerikle “HelloWorld.scala” oluşturun:

println("Hello")

Komut satırı yorumlayıcısıyla yürütün ($ komut satırı istemidir):

$ scala HelloWorld.scala
Hello

Eğer ‘.scala’yı atlarsanız (örneğin sadece ‘scala HelloWorld’ yazdıysanız), koşucu betiği derlemek ve sonra çalıştırmak yerine bayt kodlu derlenmiş bir ‘.class’ dosyası arayacaktır.

Not: Bir betik dili olarak scala kullanılıyorsa, hiçbir paket tanımlanamaz.

“bash” veya benzer kabuk terminallerini kullanan işletim sistemlerinde, Scala komut dosyaları bir “kabuk önsözü” kullanılarak çalıştırılabilir. HelloWorld.sh adlı bir dosya oluşturun ve içeriği olarak aşağıdakileri yerleştirin:

#!/bin/sh
exec scala "$0" "$@"
!#
println("Hello")

#! ve !# arasındaki kısımlar ‘kabuk önsözüdür’ ve bir bash betiği olarak yorumlanır. Gerisi Scala’dır.

Yukarıdaki dosyayı kaydettikten sonra, ona ‘çalıştırılabilir’ izinler vermelisiniz. Kabukta bunu yapabilirsiniz:

$ chmod a+x HelloWorld.sh

(Bunun herkese izin verdiğini unutmayın: Daha spesifik kullanıcı grupları için nasıl ayarlanacağını öğrenmek için [chmod hakkında 1 okuyun].)

Şimdi betiği şu şekilde çalıştırabilirsiniz:

$ ./HelloWorld.sh

Scala REPL’yi Kullanma #

Ek parametreler olmadan bir terminalde ‘scala’ çalıştırdığınızda, bir REPL (Okuma-Değerlendirme-Baskı Döngüsü) yorumlayıcısı açılır:

nford:~ $ scala
Welcome to Scala 2.11.8 (Java HotSpot(TM) 64-Bit Server VM, Java 1.8.0_66).
Type in expressions for evaluation. Or try :help.

scala>

REPL, Scala’yı bir çalışma sayfası tarzında çalıştırmanıza izin verir: yürütme bağlamı korunur ve tüm bir program oluşturmaya gerek kalmadan komutları manuel olarak deneyebilirsiniz. Örneğin, val poetikası = "Halcyonlar olarak biz olacağız" yazdığınızda şöyle görünürdü:

scala> val poem = "As halcyons we shall be"
poem: String = As halcyons we shall be

Şimdi ‘val’imizi yazdırabiliriz:

scala> print(poem)
As halcyons we shall be

“val"in değişmez olduğunu ve üzerine yazılamadığını unutmayın:

scala> poem = "Brooding on the open sea"
<console>:12: error: reassignment to val
       poem = "Brooding on the open sea"

Ancak REPL’de bir “val"i *yeniden tanımlayabilirsiniz (aynı kapsamda yapılmışsa normal bir Scala programında hataya neden olur):

scala> val poem = "Brooding on the open sea"
poem: String = Brooding on the open sea

REPL oturumunuzun geri kalanında bu yeni tanımlanan değişken, önceden tanımlanan değişkeni gölgeleyecektir. REPL’ler, nesnelerin veya diğer kodların nasıl çalıştığını hızlı bir şekilde görmek için kullanışlıdır. Scala’nın tüm özellikleri mevcuttur: işlevleri, sınıfları, yöntemleri vb. tanımlayabilirsiniz.
