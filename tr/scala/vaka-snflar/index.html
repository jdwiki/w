Sözdizimi #

  • vaka sınıfı Foo() // Parametresiz vaka sınıflarının boş bir listesi olmalıdır
  • vaka sınıfı Foo(a1: A1, …, aN: AN) // a1 … aN alanlarıyla bir vaka sınıfı oluşturun
  • vaka nesnesi Çubuğu // Tek bir vaka sınıfı oluşturun

Oluşturulan Kod Yapıları #

“case” değiştiricisi, Scala derleyicisinin sınıf için otomatik olarak ortak ortak kod oluşturmasına neden olur. Bu kodu manuel olarak uygulamak sıkıcıdır ve bir hata kaynağıdır. Aşağıdaki vaka sınıfı tanımı:

case class Person(name: String, age: Int)

… aşağıdaki kodu otomatik olarak oluşturacaktır:

class Person(val name: String, val age: Int)
  extends Product with Serializable
{
  def copy(name: String = this.name, age: Int = this.age): Person =
    new Person(name, age)

  def productArity: Int = 2

  def productElement(i: Int): Any = i match {
    case 0 => name
    case 1 => age
    case _ => throw new IndexOutOfBoundsException(i.toString)
  }

  def productIterator: Iterator[Any] =
    scala.runtime.ScalaRunTime.typedProductIterator(this)

  def productPrefix: String = "Person"

  def canEqual(obj: Any): Boolean = obj.isInstanceOf[Person]

  override def hashCode(): Int = scala.runtime.ScalaRunTime._hashCode(this)

  override def equals(obj: Any): Boolean = this.eq(obj) || obj match {
    case that: Person => this.name == that.name && this.age == that.age
    case _ => false
  }

  override def toString: String =
    scala.runtime.ScalaRunTime._toString(this)
}

“case” değiştiricisi ayrıca bir eşlik eden nesne oluşturur:

object Person extends AbstractFunction2[String, Int, Person] with Serializable {
  def apply(name: String, age: Int): Person = new Person(name, age)

  def unapply(p: Person): Option[(String, Int)] =
    if(p == null) None else Some((p.name, p.age))
}

Bir “nesne"ye uygulandığında, “vaka” değiştiricisi benzer (daha az dramatik olsa da) etkilere sahiptir. Burada birincil kazançlar, bir “toString” uygulaması ve süreçler arasında tutarlı olan bir “hashCode” değeridir. Vaka nesnelerinin (doğru) referans eşitliğini kullandığını unutmayın:

object Foo extends Product with Serializable {
  def productArity: Int = 0

  def productIterator: Iterator[Any] =
    scala.runtime.ScalaRunTime.typedProductIterator(this)

  def productElement(i: Int): Any =
    throw new IndexOutOfBoundsException(i.toString)

  def productPrefix: String = "Foo"

  def canEqual(obj: Any): Boolean = obj.isInstanceOf[this.type]

  override def hashCode(): Int = 70822 // "Foo".hashCode()

  override def toString: String = "Foo"
}

Aksi takdirde hem sınıfın kendisinde hem de onun eşlik eden nesnesinde “case” değiştiricisi tarafından sağlanacak olan yöntemleri manuel olarak uygulamak hala mümkündür.

Vaka Sınıfı Temelleri #

Normal sınıflara kıyasla – vaka sınıfları gösterimi çeşitli faydalar sağlar:

  • Tüm yapıcı argümanları “genel"dir ve başlatılmış nesnelere erişilebilir (burada gösterildiği gibi normalde durum böyle değildir):

    case class Dog1(age: Int)
val x = Dog1(18)
println(x.age) // 18 (success!)

class Dog2(age: Int)
val x = new Dog2(18)
println(x.age) // Error: "value age is not a member of Dog2"

  • Şu yöntemler için bir uygulama sağlar: ’toString’, ’equals’, ‘hashCode’ (özelliklere göre), ‘copy’, ‘apply’ ve ‘unapply’:

    case class Dog(age: Int)
val d1 = Dog(10)
val d2 = d1.copy(age = 15)

  • Kalıp eşleştirme için uygun bir mekanizma sağlar:

    sealed trait Animal // `sealed` modifier allows inheritance within current build-unit only
case class Dog(age: Int) extends Animal
case class Cat(owner: String) extends Animal
val x: Animal = Dog(18)
x match {
    case Dog(x) => println(s"It's a $x years old dog.")
    case Cat(x) => println(s"This cat belongs to $x.")
}

Vaka Sınıfı Eşitliği #

Vaka sınıfları tarafından ücretsiz olarak sağlanan bir özellik, yalnızca nesnelerin referans eşitliğini kontrol etmek yerine, tüm bireysel üye alanlarının değer eşitliğini kontrol eden, otomatik olarak oluşturulan bir “eşittir” yöntemidir.

Sıradan sınıflarla:

class Foo(val i: Int)
val a = new Foo(3)
val b = new Foo(3)
println(a == b)// "false" because they are different objects

vaka sınıfları ile:

case class Foo(i: Int)
val a = Foo(3)
val b = Foo(3)
println(a == b)// "true" because their members have the same value

Vaka Sınıfları ve Değişmezlik #

Scala derleyicisi, parametre listesindeki her argümanın önüne varsayılan olarak “val” koyar. Bu, varsayılan olarak vaka sınıflarının değişmez olduğu anlamına gelir. Her parametreye bir erişimci yöntemi verilir, ancak mutatör yöntemi yoktur. Örneğin:

case class Foo(i: Int)

val fooInstance = Foo(1)
val j = fooInstance.i       // get
fooInstance.i = 2           // compile-time exception (mutation: reassignment to val)

Bir vaka sınıfındaki bir parametreyi “var” olarak bildirmek, varsayılan davranışı geçersiz kılar ve vaka sınıfını değişken hale getirir:

case class Bar(var i: Int)

val barInstance = Bar(1)
val j = barInstance.i       // get
barInstance.i = 2           // set

Bir vaka sınıfının ‘değişebilir’ olduğu başka bir örnek, vaka sınıfındaki değerin değişken olduğu zamandır:

import scala.collection._

case class Bar(m: mutable.Map[Int, Int])

val barInstance = Bar(mutable.Map(1 -> 2))
barInstance.m.update(1, 3)                  // mutate m
barInstance                                 // Bar(Map(1 -> 3)

Burada meydana gelen ‘mutasyon’un, ’m’nin kendisine değil, ’m’nin işaret ettiği haritada olduğuna dikkat edin. Bu nedenle, başka bir nesnenin üyesi olarak ’m’ olsaydı, o da değişikliği görürdü. Aşağıdaki örnekte “instanceA"yı değiştirmenin “instanceB"yi de nasıl değiştirdiğine dikkat edin:

import scala.collection.mutable

case class Bar(m: mutable.Map[Int, Int])

val m = mutable.Map(1 ->2)
val barInstanceA = Bar(m)
val barInstanceB = Bar(m)
barInstanceA.m.update(1,3)
barInstanceA  // Bar = Bar(Map(1 -> 3))
barInstanceB  // Bar = Bar(Map(1 -> 3))
m  // scala.collection.mutable.Map[Int,Int] = Map(1 -> 3)

Belirli Değişikliklerle Bir Nesnenin Kopyasını Oluşturun #

Vaka sınıfları, belirli değişikliklerle eskisiyle aynı alanları paylaşan yeni bir nesne oluşturan bir “kopya” yöntemi sağlar.

Bu özelliği, aynı özelliklerden bazılarına sahip olan bir önceki nesneden yeni bir nesne oluşturmak için kullanabiliriz. Bu özelliği göstermek için bu basit vaka sınıfı:

case class Person(firstName: String, lastName: String, grade: String, subject: String)
val putu = Person("Putu", "Kevin", "A1", "Math")
val mark = putu.copy(firstName = "Ketut", lastName = "Mark")
// mark: People = People(Ketut,Mark,A1,Math)

Bu örnekte, iki nesnenin benzer özellikleri paylaştığını görebiliriz (“derece = A1”, “konu = Matematik”), kopyada belirtildikleri yerler (“ad” ve “soyad”) dışında.

Tip Güvenliği için Tek Elemanlı Kasa Sınıfları #

Tür güvenliğini sağlamak için bazen etki alanımızda ilkel türlerin kullanılmasından kaçınmak isteriz. Örneğin, ‘adı’ olan bir ‘Kişi’ hayal edin. Tipik olarak, “ad"ı “Dize” olarak kodlardık. Ancak, bir “Kişi"nin “adını” temsil eden bir “Dize” ile bir hata mesajını temsil eden “Dize"yi karıştırmak zor olmayacaktır:

def logError(message: ErrorMessage): Unit = ???
case class Person(name: String)
val maybeName: Either[String, String] = ??? // Left is error, Right is name
maybeName.foreach(logError) // But that won't stop me from logging the name as an error!

Bu tür tuzaklardan kaçınmak için verileri şu şekilde kodlayabilirsiniz:

case class PersonName(value: String)
case class ErrorMessage(value: String)
case class Person(name: PersonName)

ve şimdi ‘PersonName’ ile ‘ErrorMessage’, hatta sıradan bir ‘String’ ile karıştırırsak kodumuz derlenmeyecektir.

val maybeName: Either[ErrorMessage, PersonName] = ???
maybeName.foreach(reportError) // ERROR: tried to pass PersonName; ErrorMessage expected
maybeName.swap.foreach(reportError) // OK

Ancak bu, küçük bir çalışma zamanı ek yüküne neden olur, çünkü şimdi Stringleri PersonName kapsayıcılarına/kutularından çıkarmamız gerekiyor. Bunu önlemek için ‘PersonName’ ve ‘ErrorMessage’ değer sınıfları yapılabilir:

case class PersonName(val value: String) extends AnyVal
case class ErrorMessage(val value: String) extends AnyVal
