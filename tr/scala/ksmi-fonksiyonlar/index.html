Temel sözdizimi #

Scala, normal işlevleri genişleten kısmi işlev adlı özel bir işlev türüne sahiptir. Kısmi işlevler, kalıp eşleştirmede de kullanılan “vaka” sözdizimi kullanılarak anonim olarak tanımlanabilir:

val pf: PartialFunction[Boolean, Int] = {
  case true => 7
}

pf.isDefinedAt(true) // returns true
pf(true) // returns 7

pf.isDefinedAt(false) // returns false
pf(false) // throws scala.MatchError: false (of class java.lang.Boolean)

Örnekte görüldüğü gibi, kısmi bir fonksiyonun ilk parametresinin tüm alanı üzerinde tanımlanmasına gerek yoktur. Standart bir “Function1” örneğinin toplam olduğu varsayılır, bu da her olası argüman için tanımlandığı anlamına gelir.

Toplam işlev olarak kullanım #

Deyimsel Scala’da kısmi işlevler çok yaygındır. Genellikle, özellikler üzerinden toplam işlevleri tanımlamak için uygun “durum” tabanlı sözdizimleri için kullanılırlar:

sealed trait SuperType // `sealed` modifier allows inheritance within current build-unit only
case object A extends SuperType
case object B extends SuperType
case object C extends SuperType

val input: Seq[SuperType] = Seq(A, B, C)

input.map {
  case A => 5
  case _ => 10
} // Seq(5, 10, 10)

Bu, bir “eşleşme” ifadesinin ek sözdizimini normal bir anonim işlevde kaydeder. Karşılaştırmak:

input.map { item => 
  item match {
    case A => 5
    case _ => 10
  }
} // Seq(5, 10, 10)

Ayrıca, bir işleve bir tanımlama grubu veya bir vaka sınıfı geçirildiğinde, kalıp eşleştirmeyi kullanarak bir parametre ayrıştırma gerçekleştirmek için sıklıkla kullanılır:

val input = Seq("A" -> 1, "B" -> 2, "C" -> 3)

input.map { case (a, i) =>
   a + i.toString
} // Seq("A1", "B2", "C3")

’toplama’ ile kullanım #

Kısmi işlevler, genellikle son bir joker karakter eşleşmesi (“durum _”) dahil edilerek toplam işlevler için uygun sözdizimi olarak kullanılırken, bazı yöntemlerde bunların kısmiliği anahtardır. Deyimsel Scala’da çok yaygın bir örnek, [collect](http://www.scala-lang.org/api/current/index.html#scala.collection.TraversableLike@collect[B](pf:PartialFunction[‘dır. A,B]):Scala koleksiyonları kitaplığında tanımlanan Traversable[B]) yöntemi. Burada kısmi işlevler, tek bir kompakt sözdiziminde meydana gelmeleri için bir koleksiyonun öğelerini eşlemek ve/veya filtrelemek için genel işlevlerin incelenmesine izin verir.

Örnek 1

Kısmi fonksiyon olarak tanımlanan bir karekök fonksiyonumuz olduğunu varsayarsak:

val sqRoot:PartialFunction[Double,Double] = { case n if n > 0 => math.sqrt(n) }

Bunu “toplama” birleştiricisi ile çağırabiliriz:

List(-1.1,2.2,3.3,0).collect(sqRoot)

aşağıdakilerle aynı işlemi etkin bir şekilde gerçekleştirmek:

List(-1.1,2.2,3.3,0).filter(sqRoot.isDefinedAt).map(sqRoot)

Örnek 2

sealed trait SuperType // `sealed` modifier allows inheritance within current build-unit only
case class A(value: Int) extends SuperType
case class B(text: String) extends SuperType
case object C extends SuperType

val input: Seq[SuperType] = Seq(A(5), B("hello"), C, A(25), B(""))

input.collect {
  case A(value) if value < 10   => value.toString
  case B(text) if text.nonEmpty => text
} // Seq("5", "hello")

Yukarıdaki örnekte dikkat edilmesi gereken birkaç nokta vardır:

  • Her model eşleşmesinin sol tarafı, işlenecek ve çıktıya dahil edilecek öğeleri etkin bir şekilde seçer. Eşleşen bir “vaka"ya sahip olmayan herhangi bir değer basitçe atlanır.
  • Sağ taraf, uygulanacak vakaya özel işlemi tanımlar.
  • Kalıp eşleştirme, değişkenleri, koruma ifadelerinde (‘if’ yan tümceleri) ve sağ tarafta kullanım için bağlar.

Kompozisyon #

Kısmi işlevler genellikle bir toplam işlevi parçalar halinde tanımlamak için kullanılır:

sealed trait SuperType
case object A extends SuperType
case object B extends SuperType
case object C extends SuperType

val pfA: PartialFunction[SuperType, Int] = {
  case A => 5
}

val pfB: PartialFunction[SuperType, Int] = {
  case B => 10
}

val input: Seq[SuperType] = Seq(A, B, C)

input.map(pfA orElse pfB orElse {
  case _ => 15
}) // Seq(5, 10, 15)

Bu kullanımda, kısmi işlevler, ‘orElse’ yöntemiyle birleştirme sırasına göre denenir. Tipik olarak, kalan tüm durumlarla eşleşen nihai bir kısmi işlev sağlanır. Toplu olarak, bu işlevlerin kombinasyonu toplam bir işlev olarak işlev görür.

Bu model tipik olarak, bir işlevin farklı kod yolları için bir gönderici olarak etkin bir şekilde hareket edebileceği endişeleri ayırmak için kullanılır. Bu, örneğin bir Akka Aktörünün alma yönteminde yaygındır.

1: http://www.scala-lang.org/api/current/index.html#scala.PartialFunction@orElse[A1%3C:A,B1%3E:B](bu:PartialFunction[A1,B1] ]):KısmiFonksiyon[A1,B1] 2: http://doc.akka.io/docs/akka/snapshot/scala/actors.html#Defining_an_Actor_class

Bir harita işlevinde demetleri çıkarmak için kullanım #

Bu üç harita işlevi eşdeğerdir, bu nedenle ekibinizin en okunabilir bulduğu varyasyonu kullanın.

val numberNames = Map(1 -> "One", 2 -> "Two", 3 -> "Three")

// 1. No extraction
numberNames.map(it => s"${it._1} is written ${it._2}" )

// 2. Extraction within a normal function
numberNames.map(it => {
    val (number, name) = it
    s"$number is written $name"
})

// 3. Extraction via a partial function (note the brackets in the parentheses)
numberNames.map({ case (number, name) => s"$number is written $name" })

Kısmi işlev tüm girdilerle eşleşmelidir: eşleşmeyen herhangi bir durum çalışma zamanında bir istisna atar.
