Sözdizimi #

  • {maddeler} gövde için
  • {maddeler} verim gövdesi için
  • (maddeler) gövde için
  • (maddeler) verim gövdesi için

Parametreler #

parametre Ayrıntılar
için Bir for döngüsü/anlama kullanmak için gerekli anahtar sözcük
maddeler for öğesinin üzerinde çalıştığı yineleme ve filtreler.
verim Bir koleksiyon oluşturmak veya ‘vermek’ istiyorsanız bunu kullanın. ‘verim’ kullanmak, ‘for’un dönüş türünün ‘Unit’ yerine bir koleksiyon olmasına neden olur.
gövde Her yinelemede yürütülen for ifadesinin gövdesi.

Anlayışlar için Monadik #

Monadic türünde birden fazla nesneniz varsa, ‘anlama için’ kullanarak değerlerin kombinasyonlarını elde edebiliriz:

for {
   x <- Option(1)
   y <- Option("b")
   z <- List(3, 4)
} {
    // Now we can use the x, y, z variables
    println(x, y, z)
    x  // the last expression is *not* the output of the block in this case!
}

// This prints
// (1, "b", 3)
// (1, "b", 4)

Bu bloğun dönüş tipi Birimdir.

Nesneler aynı monadik “M” türündeyse (örneğin “Seçenek”) o zaman “verim” kullanıldığında “Birim” yerine “M” türünde bir nesne döndürülür.

val a = for {
   x <- Option(1)
   y <- Option("b")
} yield {
    // Now we can use the x, y variables
    println(x, y)
    // whatever is at the end of the block is the output
    (7 * x, y)
}

// This prints:
// (1, "b")
// The val `a` is set:
// a: Option[(Int, String)] = Some((7,b))

Monadik türlerin bir karışımının (‘Option’ ve ‘List’) olduğu orijinal örnekte ‘verim’ anahtar kelimesinin kullanılamayacağını unutmayın. Bunu yapmaya çalışmak, derleme zamanı tipi uyumsuzluk hatası verecektir.

Döngü İçin Yuvalanmış #

Bu, birden çok değişken üzerinde nasıl yineleme yapabileceğinizi gösterir:

for {
  x <- 1 to 2
  y <- 'a' to 'd'
} println("(" + x + "," + y + ")")

(Buradaki ’to’ ifadesinin bir dahil aralık döndüren bir infix işleç yöntemi olduğuna dikkat edin. buradaki tanıma bakın.)

Bu çıktıyı oluşturur:

(1,a)
(1,b)
(1,c)
(1,d)
(2,a)
(2,b)
(2,c)
(2,d)

Bunun parantez yerine parantez kullanan eşdeğer bir ifade olduğuna dikkat edin:

for (
  x <- 1 to 2
  y <- 'a' to 'd'
) println("(" + x + "," + y + ")")

Tüm kombinasyonları tek bir vektörde toplamak için sonucu “verebilir” ve bir “val” olarak ayarlayabiliriz:

val a = for {
  x <- 1 to 2
  y <- 'a' to 'd'
} yield "(%s,%s)".format(x, y)
// a: scala.collection.immutable.IndexedSeq[String] = Vector((1,a), (1,b), (1,c), (1,d), (2,a), (2,b), (2,c), (2,d))

For Döngüsü Kullanarak Koleksiyonlar Arasında Yineleme #

Bu, bir Haritanın her bir öğesinin nasıl yazdırılacağını gösterir.

val map = Map(1 -> "a", 2 -> "b")
for (number <- map) println(number) // prints (1,a), (2,b)
for ((key, value) <- map) println(value) // prints a, b

Bu, bir listenin her bir öğesinin nasıl yazdırılacağını gösterir.

val list = List(1,2,3)
for(number <- list) println(number) // prints 1, 2, 3

Anlamak İçin Şekeri Arındırmak #

Scala’daki ‘for’ kavrayışları sadece sözdizimsel şeker‘dir. Bu anlamalar, konu tiplerinin ‘withFilter’, ‘foreach’, ‘flatMap’ ve ‘map’ yöntemleri kullanılarak gerçekleştirilir. Bu nedenle, yalnızca bu yöntemlerin tanımlandığı türler bir ‘for’ anlamada kullanılabilir.

“pN” kalıpları, “gN” üreteçleri ve “cN” koşullarıyla aşağıdaki formun bir “for” anlayışı:

for(p0 <- x0 if g0; p1 <- g1 if c1) { ??? }

… “withFilter” ve “foreach” kullanarak iç içe çağrıların şekerini giderir:

g0.withFilter({ case p0 => c0  case _ => false }).foreach({
  case p0 => g1.withFilter({ case p1 => c1  case _ => false }).foreach({
    case p1 => ???
  })
})

Aşağıdaki formun bir “for”/“verim” ifadesi ise:

for(p0 <- g0 if c0; p1 <- g1 if c1) yield ???

… “withFilter” ve “flatMap” veya “map” kullanarak iç içe çağrıların şekerini düşürür:

g0.withFilter({ case p0 => c0  case _ => false }).flatMap({
  case p0 => g1.withFilter({ case p1 => c1  case _ => false }).map({
    case p1 => ???
  })
})

(En içteki kavramada ‘map’, her dış kavramada ‘flatMap’in kullanıldığına dikkat edin.)

Şekeri alınmış gösterimin gerektirdiği yöntemleri uygulayan herhangi bir türe bir “for” anlayışı uygulanabilir. Birleştirilebilir oldukları sürece, bu yöntemlerin dönüş türlerinde herhangi bir kısıtlama yoktur.

Döngü İçin Temel #

for (x <- 1 to 10)
  println("Iteration number " + x)

Bu, “1"den “10"a kadar bir “x” değişkenini yinelemeyi ve bu değerle bir şeyler yapmayı gösterir. Bu “for” kavrayışının dönüş türü “Birim"dir.

Anlamak İçin Temel #

Bu, bir for-döngüsü üzerinde bir filtreyi ve bir ‘sıra anlama’ oluşturmak için ‘verim’ kullanımını gösterir:

for ( x <- 1 to 10 if x % 2 == 0)
  yield x

Bunun için çıktı:

scala.collection.immutable.IndexedSeq[Int] = Vector(2, 4, 6, 8, 10)

A for anlama, yineleme ve filtrelerine dayalı olarak yeni bir koleksiyon oluşturmanız gerektiğinde kullanışlıdır.
