Değer ve değişken isimleri küçük büyük harfle yazılmalıdır #

Sabit isimler büyük harfle yazılmalıdır. Yani, eğer üye nihaidir, değişmezdir ve bir paket nesnesine veya bir nesneye aittir, sabit olarak kabul edilebilir

Metod, Değer ve değişken isimleri küçük büyük harfle yazılmalıdır.

Kaynak: http://docs.scala-lang.org/style/naming-conventions.html

Bu derleme:

val (a,b) = (1,2)
// a: Int = 1
// b: Int = 2

ama bu olmaz:

val (A,B) = (1,2)
// error: not found: value A
// error: not found: value B

Sadece val ve var değil #

“val” ve “var” #

scala> val a = 123
a: Int = 123

scala> a = 456
<console>:8: error: reassignment to val
       a = 456

scala> var b = 123
b: Int = 123

scala> b = 321
b: Int = 321
  • “val” referansları değiştirilemez: “Java"daki bir “final” değişkeni gibi, bir kez başlatıldıktan sonra onu değiştiremezsiniz
  • “var” referansları, Java’da basit bir değişken bildirimi olarak yeniden atanabilir

Immutable ve Mutable koleksiyonları #

  val mut = scala.collection.mutable.Map.empty[String, Int]
  mut += ("123" -> 123)
  mut += ("456" -> 456)
  mut += ("789" -> 789)

  val imm = scala.collection.immutable.Map.empty[String, Int]
  imm + ("123" -> 123)
  imm + ("456" -> 456)
  imm + ("789" -> 789)

  scala> mut
    Map(123 -> 123, 456 -> 456, 789 -> 789)

  scala> imm
    Map()

scala> imm + ("123" -> 123) + ("456" -> 456) + ("789" -> 789)
    Map(123 -> 123, 456 -> 456, 789 -> 789)

Scala standart kitaplığı, referansı değil, hem değişmez hem de değişebilir veri yapıları sunar. Değişmez bir veri yapısı her “değiştirildiğinde”, orijinal koleksiyonu yerinde değiştirmek yerine yeni bir örnek üretilir. Koleksiyonun her bir örneği, önemli bir yapıyı başka bir örnekle paylaşabilir.

Değiştirilebilir ve Değişmez Koleksiyon (Resmi Scala Belgeleri)

Ama bu durumda değişmezliği kullanamam! #

Örnek olarak 2 “Harita” alan ve “ma” ve “mb” içindeki her öğeyi içeren bir “Harita” döndüren bir işlev seçelim:

def merge2Maps(ma: Map[String, Int], mb: Map[String, Int]): Map[String, Int]

İlk deneme, “for ((k, v) <-map)” kullanarak haritalardan birinin öğelerini yinelemek ve bir şekilde birleştirilmiş haritayı döndürmek olabilir.

def merge2Maps(ma: ..., mb: ...): Map[String, Int] = {

  for ((k, v) <- mb) {
    ???
  }

}

Bu ilk hareket hemen bir kısıtlama ekler: ‘for’ dışında bir mutasyon artık gerekli. Bu, “for"un şekerini düşürürken daha açıktır:

// this:
for ((k, v) <- map) { ??? }

// is equivalent to:
map.foreach { case (k, v) => ??? }

“Neden mutasyona uğramak zorundayız?” #

“foreach” yan etkilere dayanır. Bir “foreach” içinde bir şeyin olmasını istediğimiz her zaman, “bir şeyi yan etkiye” ihtiyacımız var, bu durumda bir “var result” değişkenini değiştirebiliriz veya değişken bir veri yapısı kullanabiliriz.

“Sonuç” haritasını oluşturma ve doldurma #

“ma” ve “mb"nin “scala.collection.immutable.Map” olduğunu varsayalım, “ma"dan “sonuç” Haritasını oluşturabiliriz:

val result = mutable.Map() ++ ma

Ardından, öğelerini ekleyerek ‘mb’yi yineleyin ve ‘ma’ üzerindeki mevcut öğenin ‘anahtar’ı zaten mevcutsa, onu ‘mb’ ile geçersiz kılalım.

mb.foreach { case (k, v) => result += (k -> v) }

Değişken uygulama #

Şimdiye kadar çok iyi, “değişebilir koleksiyonlar kullanmak zorunda kaldık” ve doğru bir uygulama şöyle olabilir:

def merge2Maps(ma: Map[String, Int], mb: Map[String, Int]): Map[String, Int] = {
  val result = scala.collection.mutable.Map() ++ ma
  mb.foreach { case (k, v) => result += (k -> v) }
  result.toMap // to get back an immutable Map
}

Beklenildiği gibi:

scala> merge2Maps(Map("a" -> 11, "b" -> 12), Map("b" -> 22, "c" -> 23))
  Map(a -> 11, b -> 22, c -> 23)

Kurtarmak için katlanır #

Bu senaryoda ‘foreach’dan nasıl kurtulabiliriz? Tek yapmamız gereken, temelde toplama öğeleri üzerinde yineleme yapmak ve sonucu toplarken bir işlev uygulamaksa, .foldLeft kullanıyor olabiliriz:

def merge2Maps(ma: Map[String, Int], mb: Map[String, Int]): Map[String, Int] = {
  mb.foldLeft(ma) { case (result, (k, v)) => result + (k -> v) }
  // or more concisely mb.foldLeft(ma) { _ + _ }
}

Bu durumda “sonucumuz”, “.foldLeft"in “sıfır"ı olan “ma"dan başlayan birikmiş değerdir.

Ara sonuç #

Açıkçası bu değişmez çözüm, katlama sırasında birçok ‘Harita’ örneğini üretiyor ve yok ediyor, ancak bu örneklerin birikmiş ‘Harita’nın tam bir klonu olmadığını, bunun yerine mevcut örnekle önemli yapı (veri) paylaştığını belirtmekte fayda var.

Daha kolay mantık #

‘.foldLeft’ yaklaşımı kadar açıklayıcıysa, anlambilim hakkında akıl yürütmek daha kolaydır. Değişmez veri yapılarını kullanmak, uygulamamızın akıl yürütmesini kolaylaştırmaya yardımcı olabilir.
