“val” anlamsal olarak statik olduğundan, kodda göründükleri her yerde “yerinde” başlatılırlar. Bu, soyut sınıflarda ve özelliklerde kullanıldığında şaşırtıcı ve istenmeyen davranışlara neden olabilir.

Örneğin, sarmalanmış bir ‘Int’ üzerinde bir artırma işlemini tanımlayan ‘PlusOne’ adında bir özellik yapmak istediğimizi varsayalım. ‘Int’ler değişmez olduğundan, artı bir değer başlatma sırasında bilinir ve daha sonra asla değiştirilmez, bu nedenle anlamsal olarak bir ‘val’dir. Ancak bu şekilde tanımlamak beklenmedik bir sonuç doğuracaktır.

trait PlusOne {
    val i:Int

    val incr = i + 1
}

class IntWrapper(val i: Int) extends PlusOne

IntWrapper’ı hangi değerle oluşturursanız oluşturun, döndürülen nesne üzerinde ‘.incr’ çağrısı her zaman 1 döndürür. o zaman “i” yalnızca varsayılan “0” değerine sahiptir. (Diğer koşullarda, “Nil”, “null” veya benzer bir varsayılan ile doldurulabilir.)

O halde genel kural, soyut bir alana bağlı herhangi bir değerde ‘val’ kullanmaktan kaçınmaktır. Bunun yerine, ihtiyaç duyulana kadar değerlendirme yapmayan ’lazy val’ veya her çağrıldığında değerlendiren ‘def’ kullanın. Ancak, “tembel değer”, başlatma tamamlanmadan önce bir “val” tarafından değerlendirilmeye zorlanırsa, aynı hatanın oluşacağını unutmayın.

Bir keman (Scala-Js ile yazılmıştır, ancak aynı davranış geçerlidir) burada. bulunabilir.

Var, Val ve Def #

oldu #

Bir “var”, Java gibi dillerdeki değişkenlere benzer bir referans değişkenidir. Verilen nesne, “var"ın bildirildiği ile aynı türde olduğu sürece, bir “var"a farklı nesneler serbestçe atanabilir:

scala> var x = 1
x: Int = 1

scala> x = 2
x: Int = 2

scala> x = "foo bar"
<console>:12: error: type mismatch;
 found   : String("foo bar")
 required: Int
       x = "foo bar"
       ^

Yukarıdaki örnekte, “var” türünün ilk değer ataması verilen derleyici tarafından çıkarıldığına dikkat edin.

değer #

Bir “val” sabit bir referanstır. Bu nedenle, önceden atanmış bir ‘val’e yeni bir nesne atanamaz.

scala> val y = 1
y: Int = 1

scala> y = 2
<console>:12: error: reassignment to val
       y = 2
         ^

Ancak, bir “val"in işaret ettiği nesne değil sabitidir. Bu nesne değiştirilebilir:

scala> val arr = new Array[Int](2)
arr: Array[Int] = Array(0, 0)

scala> arr(0) = 1

scala> arr
res1: Array[Int] = Array(1, 0)

tanım #

Bir “def” bir yöntemi tanımlar. Bir yöntem yeniden atanamaz.

scala> def z = 1
z: Int

scala> z = 2
<console>:12: error: value z_= is not a member of object $iw
       z = 2
       ^

Yukarıdaki örneklerde, “val y” ve “def z” aynı değeri döndürür. Bununla birlikte, bir “def” * çağrıldığında* değerlendirilirken, bir “val” veya “var” * atandığında* değerlendirilir. Bu, tanımın yan etkileri olduğunda farklı davranışlara neden olabilir:

scala> val a = {println("Hi"); 1}
Hi
a: Int = 1

scala> def b = {println("Hi"); 1}
b: Int

scala> a + 1
res2: Int = 2

scala> b + 1
Hi
res3: Int = 2

Fonksiyonlar #

Fonksiyonlar değerler olduğu için ‘val’/‘var’/‘def’lere atanabilirler. Diğer her şey yukarıdakiyle aynı şekilde çalışır:

scala> val x = (x: Int) => s"value=$x"
x: Int => String = <function1>

scala> var y = (x: Int) => s"value=$x"
y: Int => String = <function1>

scala> def z = (x: Int) => s"value=$x"
z: Int => String

scala> x(1)
res0: String = value=1

scala> y(2)
res1: String = value=2

scala> z(3)
res2: String = value=3

Tembel val #

“lazy val”, bir “val"in başlatılmasının ilk kez erişilene kadar ertelendiği bir dil özelliğidir. Bu noktadan sonra normal bir “val” gibi davranır.

Bunu kullanmak için, “val"den önce “lazy” anahtar sözcüğünü ekleyin. Örneğin, REPL’yi kullanarak:

scala> lazy val foo = {
     |   println("Initializing")
     |   "my foo value"
     | }
foo: String = <lazy>

scala> val bar = {
     |   println("Initializing bar")
     |   "my bar value"
     | }
Initializing bar
bar: String = my bar value

scala> foo
Initializing
res3: String = my foo value

scala> bar
res4: String = my bar value

scala> foo
res5: String = my foo value

Bu örnek yürütme sırasını gösterir. “lazy val” bildirildiğinde, “foo” değerine kaydedilenlerin tümü henüz değerlendirilmemiş bir tembel işlev çağrısıdır. Normal ‘val’ ayarlandığında, ‘println’ çağrısının yürütüldüğünü ve ‘bar’a değer atandığını görüyoruz. ‘foo’yu ilk değerlendirdiğimizde ‘println’ çalıştırıldığını görüyoruz - ancak ikinci kez değerlendirildiğinde değil. Benzer şekilde, ‘bar’ değerlendirildiğinde ‘println’ yürütmesini görmüyoruz - yalnızca bildirildiğinde.

‘Tembel’ Ne Zaman Kullanılır #

  1. Başlatma işlemi hesaplama açısından pahalıdır ve ‘val’ kullanımı nadirdir.

    lazy val tiresomeValue = {(1 to 1000000).filter(x => x % 113 == 0).sum}
if (scala.util.Random.nextInt > 1000) {
  println(tiresomeValue)
}

“tiresomeValue"nun hesaplanması uzun zaman alır ve her zaman kullanılmaz. Bunu bir “tembel val” yapmak gereksiz hesaplamalardan kurtarır.

  1. Döngüsel bağımlılıkları çözme

Örnekleme sırasında aynı anda bildirilmesi gereken iki nesneli bir örneğe bakalım:

   object comicBook {
     def main(args:Array[String]): Unit = {
       gotham.hero.talk()
       gotham.villain.talk()
     }
   }

   class Superhero(val name: String) {
     lazy val toLockUp = gotham.villain
     def talk(): Unit = {
       println(s"I won't let you win ${toLockUp.name}!")
     }
   }

   class Supervillain(val name: String) {
     lazy val toKill = gotham.hero
     def talk(): Unit = {
       println(s"Let me loosen up Gotham a little bit ${toKill.name}!")
     }
   }

   object gotham {
     val hero: Superhero = new Superhero("Batman")
     val villain: Supervillain = new Supervillain("Joker")
   }

’lazy’ anahtar sözcüğü olmadan, ilgili nesneler bir nesnenin üyesi olamazlar. Böyle bir programın yürütülmesi bir java.lang.NullPointerException ile sonuçlanır. Tembel kullanılarak, referans, başlatılmamış bir değere sahip olma korkusu olmadan başlatılmadan önce atanabilir.

Aşırı Yükleme Tanımı #

İmza farklıysa bir “def"i aşırı yükleyebilirsiniz:

def printValue(x: Int) {
  println(s"My value is an integer equal to $x")
}

def printValue(x: String) {
  println(s"My value is a string equal to '$x'")
}

printValue(1)  // prints "My value is an integer equal to 1"
printValue("1") // prints "My value is a string equal to '1'"

Bu, sınıflar, özellikler, nesneler içinde olsun ya da olmasın aynı işlevi görür.

Adlandırılmış Parametreler #

Bir “def” çağrılırken, parametreler ada göre açıkça atanabilir. Bunu yapmak, doğru şekilde sıralanmalarına gerek olmadığı anlamına gelir. Örneğin, “printUs()” öğesini şu şekilde tanımlayın:

// print out the three arguments in order.
def printUs(one: String, two: String, three: String) = 
   println(s"$one, $two, $three")

Şimdi şu şekillerde çağrılabilir (diğerlerinin yanı sıra):

printUs("one", "two", "three") 
printUs(one="one", two="two", three="three")
printUs("one", two="two", three="three")
printUs(three="three", one="one", two="two")

Bu, her durumda “bir, iki, üç"ün yazdırılmasıyla sonuçlanır.

Tüm bağımsız değişkenler adlandırılmamışsa, ilk bağımsız değişkenler sıraya göre eşleştirilir. Hiçbir konumsal (isimsiz) argüman, adlandırılmış bir argümanın ardından gelemez:

printUs("one", two="two", three="three") // prints 'one, two, three'
printUs(two="two", three="three", "one") // fails to compile: 'positional after named argument'
