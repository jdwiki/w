Parametreler #

parametre Ayrıntılar
seçici Değeri kalıpla eşleştirilen ifade.
alternatifler “büyük/küçük harf” ile sınırlandırılmış alternatiflerin bir listesi.

Basit Desen Eşleştirme #

Bu örnek, bir girdinin birkaç değerle nasıl eşleştirileceğini gösterir:

def f(x: Int): String = x match {
  case 1 => "One"
  case 2 => "Two"
  case _ => "Unknown!"
}

f(2)  // "Two"
f(3)  // "Unknown!"

Canlı demo

Not: “_”, sonuç veya varsayılan durumdur, ancak gerekli değildir.

def g(x: Int): String = x match {
  case 1 => "One"
  case 2 => "Two"
}

g(1)  // "One"
g(3)  // throws a MatchError

Bir istisna atmaktan kaçınmak için, varsayılan durumu (case _ => <dothing>) işlemek burada en iyi işlevsel programlama uygulamasıdır. a case class üzerinden eşleştirmenin, bir vaka eksikse derleyicinin bir uyarı üretmesine yardımcı olabileceğini unutmayın. Aynısı, mühürlü bir özelliği genişleten kullanıcı tanımlı türler için de geçerlidir. Eşleşme toplamsa, varsayılan bir durum gerekli olmayabilir

Satır içi tanımlanmamış değerlerle eşleştirmek de mümkündür. Bunlar, büyük harfli bir ad kullanılarak veya ters tikler ekleyerek elde edilen kararlı tanımlayıcılar olmalıdır.

“Bir” ve “iki” başka bir yerde tanımlandığında veya işlev parametreleri olarak geçirildiğinde:

val One: Int = 1
val two: Int = 2

Aşağıdaki şekilde eşleştirilebilirler:

def g(x: Int): String = x match {
  case One => "One"
  case `two` => "Two"
}

Örneğin Java gibi diğer programlama dillerinden farklı olarak düşme yoktur. Bir vaka bloğu bir girdiyle eşleşirse yürütülür ve eşleştirme tamamlanır. Bu nedenle, en az spesifik durum, son durum bloğu olmalıdır.

def f(x: Int): String = x match {
  case _ => "Default"
  case 1 => "One"
}

f(5) // "Default"
f(1) // "Default"

Bir Seq Üzerinde Desen Eşleştirme #

Koleksiyondaki kesin sayıda öğe olup olmadığını kontrol etmek için

def f(ints: Seq[Int]): String = ints match {
  case Seq() =>
      "The Seq is empty !"
  case Seq(first) =>
      s"The seq has exactly one element : $first"
  case Seq(first, second) =>
      s"The seq has exactly two elements : $first, $second"
  case  s @ Seq(_, _, _) => 
      s"s is a Seq of length three and looks like ${s}"  // Note individual elements are not bound to their own names.
  case s: Seq[Int] if s.length == 4 =>
      s"s is a Seq of Ints of exactly length 4"  // Again, individual elements are not bound to their own names.
  case _ =>
      "No match was found!"
}

Canlı demo

İlk öğe(ler)i çıkarmak ve gerisini bir koleksiyon olarak tutmak için:

def f(ints: Seq[Int]): String = ints match {
  case Seq(first, second, tail @ _*) =>
      s"The seq has at least two elements : $first, $second. The rest of the Seq is $tail"
  case Seq(first, tail @ _*) =>
      s"The seq has at least one element : $first. The rest of the Seq is $tail"
  // alternative syntax
  // here of course this one will never match since it checks
  // for the same thing as the one above
  case first +: tail =>
      s"The seq has at least one element : $first. The rest of the Seq is $tail"
  case _ =>
      "The seq didn't match any of the above, so it must be empty"
}

Genel olarak, bir dizi oluşturmak için kullanılabilecek herhangi bir form, mevcut bir diziye karşı desen eşleştirmesi yapmak için kullanılabilir.

‘Nil’ ve ‘::’ kullanıldığında, desen bir Sırayla eşleştirildiğinde işe yarayacağını, onu bir ‘Liste’ye dönüştürdüğünü ve beklenmedik sonuçlara yol açabileceğini unutmayın. Bundan kaçınmak için kendinizi Seq( ...) ve +: ile sınırlayın.

‘::’ kullanırken ‘WrappedArray’, ‘Vector’ vb. için çalışmayacağını unutmayın, bakınız:

scala> def f(ints:Seq[Int]) = ints match {
     | case h :: t => h
     | case _ => "No match"
     | }
f: (ints: Seq[Int])Any

scala> f(Array(1,2))
res0: Any = No match

Ve +: ile

scala> def g(ints:Seq[Int]) = ints match {
     | case h+:t => h
     | case _ => "No match"
     | }
g: (ints: Seq[Int])Any

scala> g(Array(1,2).toSeq)
res4: Any = 1

Korumalar (ifadeler varsa) #

Örnek eşleştirme sırasında ekstra mantık sağlamak için vaka ifadeleri if ifadeleriyle birleştirilebilir.

def checkSign(x: Int): String = {
    x match {
      case a if a < 0 => s"$a is a negative number"
      case b if b > 0 => s"$b is a positive number"
      case c => s"$c neither positive nor negative"
    }
}

Korumalarınızın ayrıntılı olmayan bir eşleşme oluşturmamasını sağlamak önemlidir (derleyici genellikle bunu yakalamaz):

def f(x: Option[Int]) = x match {
    case Some(i) if i % 2 == 0 => doSomething(i)
    case None    => doSomethingIfNone
}

Bu, tek sayılara bir ‘MatchError’ verir. Tüm durumları hesaba katmalı veya bir joker eşleşme durumu kullanmalısınız:

def f(x: Option[Int]) = x match {
    case Some(i) if i % 2 == 0 => doSomething(i)
    case _ => doSomethingIfNoneOrOdd
}

Regex ile Desen Eşleştirme #

val emailRegex: Regex = "(.+)@(.+)\\.(.+)".r

"name@example.com" match {
  case emailRegex(userName, domain, topDomain) => println(s"Hi $userName from $domain")
  case _ => println(s"This is not a valid email.")
}

Bu örnekte, normal ifade sağlanan e-posta adresiyle eşleşmeye çalışır. Varsa, “kullanıcıAdı” ve “etki alanı” ayıklanır ve yazdırılır. ’topDomain’ de ayıklanmıştır, ancak bu örnekte onunla hiçbir şey yapılmamıştır. Bir Dizgede ‘.r’yi ‘str’ çağırmak, ’new Regex(str)’ ile eşdeğerdir. ‘r’ işlevi, bir örtük dönüştürme aracılığıyla kullanılabilir.

1: https://www.wikiod.com/tr/scala/imalar#Örtük Dönüşüm

Kararlı Tanımlayıcı ile Desen Eşleştirme #

Standart desen eşleştirmede, kullanılan tanımlayıcı, çevreleyen kapsamdaki herhangi bir tanımlayıcıyı gölgeleyecektir. Bazen çevreleyen kapsamın değişkeniyle eşleşmek gerekir.

Aşağıdaki örnek işlev, bir karakter ve bir demet listesi alır ve yeni bir demet listesi döndürür. Karakter, demetlerden birinde ilk eleman olarak mevcutsa, ikinci eleman artırılır. Henüz listede yoksa, yeni bir demet oluşturulur.

def tabulate(char: Char, tab: List[(Char, Int)]): List[(Char, Int)] = tab match {
  case Nil => List((char, 1))
  case (`char`, count) :: tail => (char, count + 1) :: tail
  case head :: tail => head :: tabulate(char, tail)
}

Yukarıda, yöntemin girdisi olan ‘char’ın örüntü eşleşmesinde ‘kararlı’ tutulduğu örüntü eşleşmesi gösterilmektedir: yani, ’tabulate(‘x’, …)’ öğesini çağırırsanız, ilk durum ifadesi yorumlanacaktır. olarak:

case('x', count) => ...

Scala, bir onay işaretiyle sınırlandırılmış herhangi bir değişkeni kararlı bir tanımlayıcı olarak yorumlayacaktır: aynı şekilde büyük harfle başlayan herhangi bir değişkeni de aynı şekilde yorumlayacaktır.

Bir Seçenek Üzerinde Eşleştirme #

Bir Option türünde eşleşiyorsanız:

def f(x: Option[Int]) = x match {
    case Some(i) => doSomething(i)
    case None    => doSomethingIfNone
}

Bu, işlevsel olarak “kat” veya “harita”/“getOrElse” kullanımına eşdeğerdir:

def g(x: Option[Int]) = x.fold(doSomethingIfNone)(doSomething)
def h(x: Option[Int]) = x.map(doSomething).getOrElse(doSomethingIfNone)

Desen Eşleştirme Mühürlü Nitelikler #

Türü mühürlü bir özellik olan bir nesneyi desen eşleştirirken, Scala derleme zamanında tüm durumların ‘kapsamlı olarak eşleştiğini’ kontrol eder:

sealed trait Shape
case class Square(height: Int, width: Int) extends Shape
case class Circle(radius: Int) extends Shape
case object Point extends Shape


def matchShape(shape: Shape): String = shape match {
    case Square(height, width) => "It's a square"
    case Circle(radius)        => "It's a circle"
    //no case for Point because it would cause a compiler warning.
}

Daha sonra ‘Shape’ için yeni bir ‘case class’ eklenirse, ‘Shape’ üzerindeki tüm ‘match’ ifadeleri bir derleyici uyarısı vermeye başlayacaktır. Bu, baştan sona yeniden düzenlemeyi kolaylaştırır: derleyici, geliştiriciyi güncellenmesi gereken tüm kodlar konusunda uyarır.

Desen bağlayıcı (@) #

@ işareti, bir model eşleşmesi sırasında bir değişkeni bir ada bağlar. Bağlı değişken, eşleşen nesnenin tamamı veya eşleşen nesnenin bir parçası olabilir:

sealed trait Shape
case class Rectangle(height: Int, width: Int) extends Shape
case class Circle(radius: Int) extends Shape
case object Point extends Shape

(Circle(5): Shape) match {
  case Rectangle(h, w) => s"rectangle, $h x $w."
  case Circle(r) if r > 9 => s"large circle"
  case c @ Circle(_) => s"small circle: ${c.radius}"  // Whole matched object is bound to c
  case Point => "point"
}

> res0: Dize = küçük daire: 5

Sınırlı tanımlayıcı, koşullu filtrelerde kullanılabilir. Böylece:

case Circle(r) if r > 9 => s"large circle"

şu şekilde yazılabilir:

case c @ Circle(_) if c.radius > 9 => s"large circle"

Ad, eşleşen desenin yalnızca bir kısmına bağlanabilir:

Seq(Some(1), Some(2), None) match {
  // Only the first element of the matched sequence is bound to the name 'c'
  case Seq(c @ Some(1), _*) => head
  case _ => None
}

> res0: Seçenek[Int] = Bazı(1)

Vaka Sınıflarıyla Desen Eşleştirme #

Her vaka sınıfı, model eşleştirme sırasında vaka sınıfının üyelerini yakalamak için kullanılabilecek bir çıkarıcıyı tanımlar:

case class Student(name: String, email: String)

def matchStudent1(student: Student): String = student match {
    case Student(name, email) => s"$name has the following email: $email" // extract name and email
}

Tüm normal kalıp eşleştirme kuralları geçerlidir - eşleşmeyi kontrol etmek için korumaları ve sabit ifadeleri kullanabilirsiniz:

def matchStudent2(student: Student): String = student match {
    case Student("Paul", _) => "Matched Paul" // Only match students named Paul, ignore email
    case Student(name, _) if name == "Paul" => "Matched Paul" // Use a guard to match students named Paul, ignore email
    case s if s.name == "Paul" => "Matched Paul" // Don't use extractor; use a guard to match students named Paul, ignore email
    case Student("Joe", email) => s"Joe has email $email" // Match students named Joe, capture their email
    case Student(name, email) if name == "Joe" => s"Joe has email $email" // use a guard to match students named Joe, capture their email
    case Student(name, email) => s"$name has email $email." // Match all students, capture name and email 
}

Desen Eşleştirme Türleri #

Model eşleştirme, bir örneğin tipini kontrol etmek için isInstanceOf[B] kullanmak yerine de kullanılabilir:

val anyRef: AnyRef = ""
                                                  
anyRef match {
  case _: Number       => "It is a number"
  case _: String       => "It is a string"
  case _: CharSequence => "It is a char sequence"
}
//> res0: String = It is a string

Davaların sırası önemlidir:

anyRef match {
  case _: Number       => "It is a number"
  case _: CharSequence => "It is a char sequence"
  case _: String       => "It is a string"
}
//> res1: String = It is a char sequence

Bu şekilde, geçiş işlevselliği olmadan klasik bir ‘anahtar’ ifadesine benzer. Bununla birlikte, söz konusu türden eşleme ve ‘çıkartma’ değerleri de düzenleyebilirsiniz. Örneğin:

case class Foo(s: String)
case class Bar(s: String)
case class Woo(s: String, i: Int)

def matcher(g: Any):String = {
  g match {
    case Bar(s) => s + " is classy!" 
    case Foo(_) => "Someone is wicked smart!"
    case Woo(s, _) => s + " is adventerous!"
    case _ => "What are we talking about?"
  }
}

print(matcher(Foo("Diana")))  // prints 'Diana is classy!'
print(matcher(Bar("Hadas")))  // prints 'Someone is wicked smart!'
print(matcher(Woo("Beth", 27)))   // prints 'Beth is adventerous!'
print(matcher(Option("Katie")))  // prints 'What are we talking about?'

‘Foo’ ve ‘Woo’ durumunda, ‘bağlanmamış bir değişkeni eşleştirmek’ için alt çizgiyi (’_’) kullandığımızı unutmayın. Diğer bir deyişle, değer (bu durumda sırasıyla ‘Hadas’ ve ‘27’) bir ada bağlı değildir ve dolayısıyla bu durumda işleyicide mevcut değildir. Bu, ‘herhangi bir’ değeri, o değerin ne olduğu konusunda endişelenmeden eşleştirmek için kullanışlı bir kısayoldur.

Tablo anahtarı veya arama anahtarı olarak derlenmiş Desen Eşleştirme #

“@switch” ek açıklaması, derleyiciye “match” ifadesinin bayt kodu düzeyinde tek bir “tableswitch” komutuyla değiştirilebileceğini söyler. Bu, çalışma zamanı sırasında gereksiz karşılaştırmaları ve değişken yükleri kaldırabilen küçük bir optimizasyondur.

“@switch” ek açıklaması yalnızca değişmez sabitlere ve “son değer” tanımlayıcılarına karşı eşleşmeler için çalışır. Kalıp eşleşmesi bir “tableswitch”/“lookupswitch” olarak derlenemiyorsa, derleyici bir uyarı verecektir.

import annotation.switch

def suffix(i: Int) = (i: @switch) match {
  case 1 => "st"
  case 2 => "nd"
  case 3 => "rd"
  case _ => "th"
}

Sonuçlar, normal bir model eşleşmesiyle aynıdır:

scala> suffix(2)
res1: String = "2nd"

scala> suffix(4)
res2: String = "4th"

Scala Belgelerinden (2.8+) – @switch:

Bir eşleşme ifadesine uygulanacak bir açıklama. Varsa, derleyici, eşleşmenin bir tablo anahtarına derlendiğini doğrulayacaktır. veya arama anahtarı ve bunun yerine bir koşullu ifadeler dizisi.

Java Spesifikasyonundan:

  • tableswitch: “İndeks ve atlama ile atlama tablosuna erişin”
  • arama anahtarı: “Anahtar eşleşmesi ve atlama ile atlama tablosuna erişin”

Aynı Anda Birden Fazla Kalıbı Eşleştirme #

|, aynı sonucu elde etmek için birden çok girdiyle eşleşen tek bir durum ifadesinin olması için kullanılabilir:

def f(str: String): String = str match {
  case "foo" | "bar" => "Matched!"
  case _ => "No match."
}

f("foo")  // res0: String = Matched!
f("bar")  // res1: String = Matched!
f("fubar")  // res2: String = No match.

değerleri bu şekilde eşleştirmenin iyi sonuç verdiğini, ancak aşağıdaki türlerin eşleştirilmesinin sorunlara yol açacağını unutmayın:

sealed class FooBar
case class Foo(s: String) extends FooBar
case class Bar(s: String) extends FooBar

val d = Foo("Diana")
val h = Bar("Hadas")

// This matcher WILL NOT work.
def matcher(g: FooBar):String = {
  g match {
    case Foo(s) | Bar(s) => print(s)  // Won't work: s cannot be resolved
    case Foo(_) | Bar(_) => _         // Won't work: _ is an unbound placeholder
    case _ => "Could not match"
  }
}

İkinci durumda (_ ile) ilişkisiz değişkenin değerine ihtiyacınız yoksa ve sadece başka bir şey yapmak istiyorsanız, sorun değil:

def matcher(g: FooBar):String = {
  g match {
    case Foo(_) | Bar(_) => "Is either Foo or Bar."  // Works fine
    case _ => "Could not match"
  }
}

Aksi takdirde, davalarınızı bölmekle kalırsınız:

def matcher(g: FooBar):String = {
  g match {
    case Foo(s) => s 
    case Bar(s) => s
    case _ => "Could not match"
  }
}

Tuple’larda Desen Eşleştirme #

Aşağıdaki “Liste” grupları göz önüne alındığında:

val pastries = List(("Chocolate Cupcake", 2.50), 
                    ("Vanilla Cupcake", 2.25),
                    ("Plain Muffin", 3.25))

Desen eşleştirme, her öğeyi farklı şekilde işlemek için kullanılabilir:

pastries foreach { pastry =>
  pastry match {
    case ("Plain Muffin", price) => println(s"Buying muffin for $price")
    case p if p._1 contains "Cupcake" => println(s"Buying cupcake for ${p._2}")
    case _ => println("We don't sell that pastry")
  }
}

İlk durum, belirli bir dizeyle nasıl eşleştirileceğini ve ilgili fiyatın nasıl alınacağını gösterir. İkinci durum, demetin öğeleriyle eşleştirmek için if ve tüp çıkarma kullanımını gösterir.
