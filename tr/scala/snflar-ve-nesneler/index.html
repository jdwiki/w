Singleton ve Tamamlayıcı Nesneler #

Tekil Nesneler #

Scala, statik üyeleri destekler, ancak Java ile aynı şekilde değildir. Scala, buna Singleton Objects adı verilen bir alternatif sunar. Singleton nesneleri, “new” anahtar sözcüğü kullanılarak somutlaştırılamamaları dışında normal bir sınıfa benzer. Aşağıda örnek bir singleton sınıfı verilmiştir:

object Factorial {
    private val cache = Map[Int, Int]()
    def getCache = cache
}

Tekil nesneyi tanımlamak için (‘sınıf’ veya ‘özellik’ yerine) ‘object’ anahtar sözcüğünü kullandığımıza dikkat edin. Singleton nesneleri somutlaştırılamadığı için parametrelere sahip olamazlar. Bir singleton nesnesine erişmek şöyle görünür:

Factorial.getCache() //returns the cache

Bunun tam olarak bir Java sınıfındaki statik bir yönteme erişmeye benzediğini unutmayın.

Tamamlayıcı Nesneler #

Scala’da tekil nesneler, karşılık gelen bir sınıfın adını paylaşabilir. Böyle bir senaryoda, singleton nesnesine Companion Object adı verilir. Örneğin, ‘Factory’ sınıfının altında tanımlanır ve onun altında bir eşlik eden nesne (‘Factional’ olarak da adlandırılır) tanımlanır. Kural olarak, eşlik eden nesneler, eşlik eden sınıflarıyla aynı dosyada tanımlanır.

class Factorial(num : Int) {

  def fact(num : Int) : Int = if (num <= 1) 1 else (num * fact(num - 1))

  def calculate() : Int = {
    if (!Factorial.cache.contains(num)) {    // num does not exists in cache
      val output = fact(num) // calculate factorial
      Factorial.cache += (num -> output)     // add new value in cache
    }

    Factorial.cache(num)
  }
}

object Factorial {
  private val cache = scala.collection.mutable.Map[Int, Int]()
}

val factfive = new Factorial(5)
factfive.calculate  // Calculates the factorial of 5 and stores it
factfive.calculate  // uses cache this time
val factfiveagain = new Factorial(5)
factfiveagain.calculate  // Also uses cache

Bu örnekte, tekrarlanan sayılar için hesaplama zamanından tasarruf etmek için bir sayının faktöriyelini saklamak için özel bir “önbellek” kullanıyoruz.

Burada “nesne Factorial” bir eşlik eden nesnedir ve “class Factorial” onun karşılık gelen eşlik eden sınıfıdır. Tamamlayıcı nesneler ve sınıflar birbirlerinin “özel” üyelerine erişebilir. Yukarıdaki örnekte, “Factory” sınıfı, eşlik eden nesnesinin özel “cache” üyesine erişiyor.

Sınıfın yeni bir örneğinin yine aynı eşlik eden nesneyi kullanacağını, dolayısıyla o nesnenin üye değişkenlerinde yapılan herhangi bir değişikliğin taşınacağını unutmayın.

Yapıcılar #

Birincil Oluşturucu #

Scala’da birincil kurucu sınıfın gövdesidir. Sınıf adının ardından, yapıcı argümanları olan bir parametre listesi gelir. (Herhangi bir işlevde olduğu gibi, boş bir parametre listesi atlanabilir.)

class Foo(x: Int, y: String) {
    val xy: String = y * x
    /* now xy is a public member of the class */
}

class Bar {
    ...
}

Bir örneğin yapı parametrelerine, “val” anahtar sözcüğüyle örnek üye olarak işaretlenmedikçe, yapıcı gövdesinin dışından erişilemez:

class Baz(val z: String) 
// Baz has no other members or methods, so the body may be omitted

val foo = new Foo(4, "ab")
val baz = new Baz("I am a baz")
foo.x // will not compile: x is not a member of Foo
foo.xy // returns "abababab": xy is a member of Foo
baz.z // returns "I am a baz": z is a member of Baz
val bar0 = new Bar
val bar1 = new Bar() // Constructor parentheses are optional here

Bir nesnenin örneği başlatıldığında gerçekleştirilmesi gereken tüm işlemler, doğrudan sınıfın gövdesine yazılır:

class DatabaseConnection
    (host: String, port: Int, username: String, password: String) {
    /* first connect to the DB, or throw an exception */
    private val driver = new AwesomeDB.Driver()
    driver.connect(host, port, username, password)
    def isConnected: Boolean = driver.isConnected
    ...
}

Yapıcıya mümkün olduğunca az yan etki koymanın iyi bir uygulama olarak kabul edildiğini unutmayın; Yukarıdaki kod yerine, IO programlamasından tüketici kodunun sorumlu olması için “connect” ve “disconnect” yöntemlerine sahip olunması düşünülmelidir.

Yardımcı Yapıcılar #

Bir sınıfın ‘yardımcı kurucular’ adı verilen ek kurucuları olabilir. Bunlar, “def this(…) = e” biçimindeki yapıcı tanımlarıyla tanımlanır; burada “e"nin başka bir kurucu çağırması gerekir:

class Person(val fullName: String) {    
  def this(firstName: String, lastName: String) = this(s"$firstName $lastName")
}

// usage:
new Person("Grace Hopper").fullName // returns Grace Hopper
new Person("Grace", "Hopper").fullName // returns Grace Hopper

Bu, her kurucunun farklı bir değiştiriciye sahip olabileceği anlamına gelir: yalnızca bazıları herkese açık olabilir:

class Person private(val fullName: String) {    
  def this(firstName: String, lastName: String) = this(s"$firstName $lastName")
}

new Person("Ada Lovelace") // won't compile
new Person("Ada", "Lovelace") // compiles

Bu şekilde, tüketici kodunun sınıfı nasıl başlatabileceğini kontrol edebilirsiniz.

Örnek Sınıf Örnekleri #

Scala’daki bir sınıf, bir sınıf örneğinin ‘planıdır’. Bir örnek, o sınıf tarafından tanımlanan durumu ve davranışı içerir. Bir sınıf bildirmek için:

class MyClass{}  // curly braces are optional here as class body is empty

Bir örnek, “new” anahtar sözcüğü kullanılarak başlatılabilir:

var instance = new MyClass()

veya:

var instance = new MyClass

Scala’da argümansız bir oluşturucuya sahip bir sınıftan nesneler oluşturmak için parantezler isteğe bağlıdır. Bir sınıf kurucusu argüman alırsa:

class MyClass(arg : Int)       // Class definition
var instance = new MyClass(2)  // Instance instantiation
instance.arg                   // not allowed

Burada ‘Sınıfım’, yalnızca sınıf için dahili olarak kullanılabilen bir ‘Int’ argümanı gerektirir. ‘arg’, bir alan olarak bildirilmedikçe ‘Sınıfım’ dışında erişilemez:

class MyClass(arg : Int){ 
    val prop = arg  // Class field declaration
} 

var obj = new MyClass(2)
obj.prop     // legal statement

Alternatif olarak, yapıcıda public olarak bildirilebilir:

class MyClass(val arg : Int)   // Class definition with arg declared public
var instance = new MyClass(2)  // Instance instantiation
instance.arg                   //arg is now visible to clients

Parametresiz örnek oluşturma sınıfı: {} vs () #

Diyelim ki yapıcı argümanı olmayan bir MyClass sınıfımız var:

class MyClass

Scala’da aşağıdaki sözdizimini kullanarak somutlaştırabiliriz:

val obj = new MyClass()

Ya da basitçe şunu yazabiliriz:

val obj = new MyClass

Ancak, dikkat edilmezse, bazı durumlarda isteğe bağlı parantez bazı beklenmedik davranışlara neden olabilir. Ayrı bir iş parçacığında çalışması gereken bir görev oluşturmak istediğimizi varsayalım. Örnek kod aşağıdadır:

val newThread = new Thread { new Runnable {
        override def run(): Unit = {
            // perform task
            println("Performing task.")
        }
      }
    }

newThread.start   // prints no output

Bu örnek kodun çalıştırılması halinde Görevi gerçekleştirme yazdıracağını düşünebiliriz, ancak sürpriz bir şekilde hiçbir şey yazdırmayacaktır. Bakalım burada neler oluyor. Daha yakından bakarsanız, “new Thread"den hemen sonra “{}” kaşlı ayraçları kullandık. ‘Thread’i genişleten anonim bir sınıf yarattı:

val newThread = new Thread {
  //creating anonymous class extending Thread
}

Ve sonra bu anonim sınıfın gövdesinde görevimizi tanımladık (yine ‘Runnable’ arayüzünü uygulayan bir anonim sınıf yaratarak). Bu nedenle, ‘public Thread(Runnable target)’ yapıcısını kullandığımızı düşünmüş olabiliriz ama aslında (opsiyonel ‘()‘yı göz ardı ederek) ‘run()’ gövdesinde hiçbir şey tanımlanmamış ‘public Thread()’ yapıcısını kullandık. yöntem. Sorunu düzeltmek için kaşlı ayraçlar yerine parantez kullanmamız gerekiyor.

val newThread = new Thread ( new Runnable {
        override def run(): Unit = {
            // perform task
            println("Performing task.")
        }
      }
    )

Başka bir deyişle, burada “{}” ve “()” değiştirilemez değildir.

Nesneler #

Sınıflar daha çok planlara benzerken, Nesneler statiktir (yani zaten somutlaştırılmıştır):

object Dog {
    def bark: String = "Raf"
}

Dog.bark() // yields "Raf"

Genellikle bir sınıfa eşlik etmek için kullanılırlar, yazmanıza izin verirler:

class Dog(val name: String) {

}

object Dog {
    def apply(name: String): Dog = new Dog(name)
}

val dog = Dog("Barky") // Object
val dog = new Dog("Barky") // Class

Örnek türü kontrolü #

Tür denetimi: variable.isInstanceOf[Type]

Desen eşleştirme ile (bu formda pek kullanışlı değil):

variable match {
  case _: Type => true
  case _ => false
}

Hem “isInstanceOf” hem de kalıp eşleştirme, diziler dışında genel parametresini değil (tür yeniden yapılandırması yok) yalnızca nesnenin türünü kontrol eder:

val list: List[Any] = List(1, 2, 3)             //> list  : List[Any] = List(1, 2, 3)

val upcasting = list.isInstanceOf[Seq[Int]]     //> upcasting  : Boolean = true

val shouldBeFalse = list.isInstanceOf[List[String]]
                                                //> shouldBeFalse  : Boolean = true

Fakat

val chSeqArray: Array[CharSequence] = Array("a") //> chSeqArray  : Array[CharSequence] = Array(a)
val correctlyReified = chSeqArray.isInstanceOf[Array[String]]
                                              //> correctlyReified  : Boolean = false


val stringIsACharSequence: CharSequence = ""    //> stringIsACharSequence  : CharSequence = ""
  
val sArray = Array("a")                         //> sArray  : Array[String] = Array(a)
val correctlyReified = sArray.isInstanceOf[Array[String]]
                                                //> correctlyReified  : Boolean = true

//val arraysAreInvariantInScala: Array[CharSequence] = sArray
//Error: type mismatch;  found   : Array[String]  required: Array[CharSequence]
//Note: String <: CharSequence, but class Array is invariant in type T.
//You may wish to investigate a wildcard type such as `_ <: CharSequence`. (SLS 3.2.10)
//Workaround:
val arraysAreInvariantInScala: Array[_ <: CharSequence] = sArray
                                                //> arraysAreInvariantInScala  : Array[_ <: CharSequence] = Array(a)
  

val arraysAreCovariantOnJVM = sArray.isInstanceOf[Array[CharSequence]]
                                                //> arraysAreCovariantOnJVM  : Boolean = true

Tür atama: variable.asInstanceOf[Type]

[Desen eşleştirme] ile(https://www.wikiod.com/tr/scala/desen-eslestirme):

variable match {
  case _: Type => true
}

Örnekler:

  val x = 3                                       //> x  : Int = 3
  x match {
    case _: Int => true//better: do something
    case _ => false
  }                                               //> res0: Boolean = true
  
  x match {
    case _: java.lang.Integer => true//better: do something
    case _ => false
  }                                               //> res1: Boolean = true
  
  x.isInstanceOf[Int]                             //> res2: Boolean = true
  
  //x.isInstanceOf[java.lang.Integer]//fruitless type test: a value of type Int cannot also be a Integer
  
  trait Valuable { def value: Int}
  case class V(val value: Int) extends Valuable
  
  val y: Valuable = V(3)                          //> y  : Valuable = V(3)
  y.isInstanceOf[V]                               //> res3: Boolean = true
  y.asInstanceOf[V]                               //> res4: V = V(3)

Açıklama: Bu yalnızca JVM’deki davranışla ilgilidir, diğer platformlarda (JS, yerel) tip yayınlama/kontrol farklı davranabilir.
