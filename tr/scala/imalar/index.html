Sözdizimi #

  • örtük değer x: T = ???

Örtük sınıflar, kodlarını değiştirmek zorunda kalmadan mevcut türlere özel yöntemlerin eklenmesine izin verir, böylece kodun kontrolüne ihtiyaç duymadan türleri zenginleştirir.

Varolan bir sınıfı zenginleştirmek için örtük türlerin kullanılmasına genellikle ‘kütüphanemi zenginleştir’ kalıbı denir.

Örtülü Sınıflardaki Kısıtlamalar

  1. Örtülü sınıflar yalnızca başka bir sınıf, nesne veya özellik içinde var olabilir.
  2. Örtülü sınıflar yalnızca bir örtük olmayan birincil kurucu parametresine sahip olabilir.
  3. Aynı kapsamda, örtük sınıfla aynı ada sahip başka bir nesne, sınıf, özellik veya sınıf üyesi tanımı olamaz.

Örtülü Sınıflar #

Örtük sınıflar, önceden tanımlanmış sınıflara yeni yöntemler eklemeyi mümkün kılar.

“String” sınıfının “withoutVowels” yöntemi yoktur. Bu şu şekilde eklenebilir:

object StringUtil {
  implicit class StringEnhancer(str: String) {
    def withoutVowels: String = str.replaceAll("[aeiou]", "")
  }
}

Örtük sınıf, genişletmek istediğiniz türe (String) sahip tek bir kurucu parametreye (str) sahiptir ve türe (withoutVowels) “eklemek” istediğiniz yöntemi içerir. Yeni tanımlanan yöntemler artık doğrudan geliştirilmiş tür üzerinde kullanılabilir (geliştirilmiş tür örtük kapsamda olduğunda):

import StringUtil.StringEnhancer // Brings StringEnhancer into implicit scope

println("Hello world".withoutVowels) // Hll wrld

Başlık altında, örtük sınıflar, geliştirilmiş türden örtük sınıfa bir örtük dönüşüm tanımlar, şöyle:

implicit def toStringEnhancer(str: String): StringEnhancer = new StringEnhancer(str)

Örtük sınıflar, çalışma zamanı nesneleri oluşturmaktan ve dolayısıyla çalışma zamanı ek yükünü ortadan kaldırmaktan kaçınmak için genellikle [Değer sınıfları2 olarak tanımlanır:

implicit class StringEnhancer(val str: String) extends AnyVal {
    /* conversions code here */
}

Yukarıdaki geliştirilmiş tanımla, “withoutVowels” yöntemi her çağrıldığında yeni bir “StringEnhancer” örneğinin oluşturulması gerekmez.

1: https://www.wikiod.com/tr/scala/imalar#Örtük Dönüşüm 2: http://docs.scala-lang.org/overviews/core/value-classes.html

Örtülü Parametreler #

Bir türden bir parametrenin kapsamda bir kez tanımlanması ve ardından o türden bir değer kullanan tüm işlevlere uygulanması gerekiyorsa, örtük parametreler yararlı olabilir.

Normal bir işlev çağrısı şuna benzer:

// import the duration methods
import scala.concurrent.duration._

// a normal method:
def doLongRunningTask(timeout: FiniteDuration): Long = timeout.toMillis

val timeout = 1.second
// timeout: scala.concurrent.duration.FiniteDuration = 1 second

// to call it
doLongRunningTask(timeout) // 1000

Şimdi hepsinin bir zaman aşımı süresine sahip bazı yöntemlerimiz olduğunu ve tüm bu yöntemleri aynı zaman aşımını kullanarak çağırmak istediğimizi varsayalım. Zaman aşımını örtük bir değişken olarak tanımlayabiliriz.

// import the duration methods
import scala.concurrent.duration._

// dummy methods that use the implicit parameter
def doLongRunningTaskA()(implicit timeout: FiniteDuration): Long = timeout.toMillis
def doLongRunningTaskB()(implicit timeout: FiniteDuration): Long = timeout.toMillis

// we define the value timeout as implicit
implicit val timeout: FiniteDuration = 1.second

// we can now call the functions without passing the timeout parameter
doLongRunningTaskA() // 1000
doLongRunningTaskB() // 1000

Bunun çalışma şekli, scalac derleyicisinin kapsamda örtük olarak işaretlenmiş ve türünün örtük parametrelerden biriyle eşleştiği bir değer aramasıdır. Bir tane bulursa, onu örtük parametre olarak uygular.

İki veya daha fazla örtük tanımlarsanız bunun işe yaramayacağını unutmayın. kapsamda aynı tür.

Hata mesajını özelleştirmek için, türde “implicitNotFound” ek açıklamasını kullanın:

@annotation.implicitNotFound(msg = "Select the proper implicit value for type M[${A}]!")
case class M[A](v: A) {}

def usage[O](implicit x: M[O]): O = x.v

//Does not work because no implicit value is present for type `M[Int]`
//usage[Int]   //Select the proper implicit value for type M[Int]!
implicit val first: M[Int] = M(1)
usage[Int]     //Works when `second` is not in scope
implicit val second: M[Int] = M(2)
//Does not work because more than one implicit values are present for the type `M[Int]`
//usage[Int]   //Select the proper implicit value for type M[Int]!

Zaman aşımı bunun için olağan bir kullanım durumudur veya örneğin Akka‘de ActorSystem (çoğu zaman) her zaman aynıdır, bu nedenle genellikle örtük olarak geçirilir. Başka bir kullanım durumu, en yaygın olarak tip sınıflarına dayanan FP kitaplıklarıyla (scalaz, cats veya rapture gibi) kitaplık tasarımı olabilir.

Int, Long, String vb. gibi temel türlerle örtük parametreleri kullanmak, kafa karışıklığı yaratacağı ve kodu daha az okunabilir hale getireceği için genellikle kötü bir uygulama olarak kabul edilir.

Örtülü Dönüşüm #

Bir örtük dönüştürme, derleyicinin bir türden bir nesneyi otomatik olarak başka bir türe dönüştürmesine izin verir. Bu, kodun bir nesneyi başka türden bir nesne olarak ele almasına izin verir.

case class Foo(i: Int)

// without the implicit
Foo(40) + 2    // compilation-error (type mismatch)

// defines how to turn a Foo into an Int
implicit def fooToInt(foo: Foo): Int = foo.i

// now the Foo is converted to Int automatically when needed
Foo(40) + 2    // 42

Dönüştürme tek yönlüdür: bu durumda “42"yi tekrar “Foo(42)“ye dönüştüremezsiniz. Bunu yapmak için ikinci bir örtük dönüşüm tanımlanmalıdır:

implicit def intToFoo(i: Int): Foo = Foo(i)

Bunun, örneğin bir tamsayı değerine bir kayan nokta değerinin eklenebileceği mekanizma olduğuna dikkat edin.

Örtük dönüştürmeler, neler olup bittiğini kararttıkları için idareli kullanılmalıdır. Örtük bir dönüştürme kullanmanın somut bir okunabilirlik kazancı olmadıkça, bir yöntem çağrısı aracılığıyla açık bir dönüştürme kullanmak en iyi uygulamadır.

Örtük dönüşümlerin önemli bir performans etkisi yoktur.

Scala, Java’dan Scala’ya ve geriye tüm dönüşümler dahil scala.Predef içindeki çeşitli örtük dönüşümleri otomatik olarak içe aktarır. Bunlar, varsayılan olarak herhangi bir dosya derlemesine dahil edilir.

‘Örtülü’ Kullanarak Örtülü Parametreleri Çözümleme #

Birden fazla örtük parametre içeren bir örtük parametre listesi varsayarsak:

case class Example(p1:String, p2:String)(implicit ctx1:SomeCtx1, ctx2:SomeCtx2)

Şimdi, örtük örneklerden birinin mevcut olmadığını (‘SomeCtx1’) ve gereken diğer tüm örtülü örneklerin kapsam içinde olduğunu varsayarsak, sınıfın bir örneğini oluşturmak için bir ‘SomeCtx1’ örneği sağlanmalıdır.

Bu, “implicitly” anahtar sözcüğünü kullanarak birbirinin kapsam içi örtük örneğini korurken yapılabilir:

Example("something","somethingElse")(new SomeCtx1(), implicitly[SomeCtx2])

REPL’deki imalar #

Bir REPL oturumu sırasında kapsam içindeki tüm ‘örtükleri’ görüntülemek için:

scala> :implicits

“Predef.scala” içinde tanımlanan örtük dönüştürmeleri de dahil etmek için:

scala> :implicits -v

Bir kişinin bir ifadesi varsa ve ona uygulanan tüm yeniden yazma kurallarının etkisini görmek isterse (örtükler dahil):

scala> reflect.runtime.universe.reify(expr) // No quotes. reify is a macro operating directly on code.

(Örnek:

scala> import reflect.runtime.universe._
scala> reify(Array("Alice", "Bob", "Eve").mkString(", "))
resX: Expr[String] = Expr[String](Predef.refArrayOps(Array.apply("Alice", "Bob", "Eve")(Predef.implicitly)).mkString(", "))

)
