Özellikle dağıtılmış ortamlarda (örneğin, Apache Spark), serileştirme sorunlarından kaçınmak için, tür sınıfı örnekleri için “Seri hale getirilebilir” özelliğini uygulamak en iyi uygulamadır.

Basit Tip Sınıfı #

Bir tür sınıfı, bir veya daha fazla tür parametresine sahip bir “özellik"tir:

trait Show[A] {
  def show(a: A): String
}

Bir tür sınıfını genişletmek yerine, desteklenen her tür için tür sınıfının örtük bir örneği sağlanır. Bu uygulamaları tür sınıfının eşlik eden nesnesine yerleştirmek, örtük çözümlemenin herhangi bir özel içe aktarma olmadan çalışmasına izin verir:

object Show {
  implicit val intShow: Show[Int] = new Show {
    def show(x: Int): String = x.toString
  }

  implicit val dateShow: Show[java.util.Date] = new Show {
    def show(x: java.util.Date): String = x.getTime.toString
  }

  // ..etc
}

Bir işleve iletilen genel bir parametrenin bir tür sınıfı örneğine sahip olduğunu garanti etmek istiyorsanız, örtük parametreleri kullanın:

def log[A](a: A)(implicit showInstance: Show[A]): Unit = {
  println(showInstance.show(a))
}

Ayrıca bir bağlam sınırı kullanabilirsiniz:

def log[A: Show](a: A): Unit = {
  println(implicitly[Show[A]].show(a))
}

Yukarıdaki “log” yöntemini diğer yöntemler gibi çağırın. “Günlüğe” ilettiğiniz “A” için örtük bir “Göster[A]” uygulaması bulunamazsa derleme başarısız olur.

log(10) // prints: "10"
log(new java.util.Date(1469491668401L) // prints: "1469491668401"
log(List(1,2,3)) // fails to compile with
                 // could not find implicit value for evidence parameter of type Show[List[Int]]

Bu örnek, ‘Göster’ tipi sınıfını uygular. Bu, rastgele türlerin rastgele örneklerini “Dize"lere dönüştürmek için kullanılan yaygın bir tür sınıfıdır. Her nesnenin bir “toString” yöntemi olmasına rağmen, “toString"in yararlı bir şekilde tanımlanıp tanımlanmadığı her zaman açık değildir. ‘Show’ tipi sınıfın kullanımıyla, ’log’a geçirilen herhangi bir şeyin ‘String’e iyi tanımlanmış bir dönüşümü olduğunu garanti edebilirsiniz.

Bir Tip Sınıfını Genişletme #

Bu örnek, aşağıdaki tür sınıfının genişletilmesini tartışır.

trait Show[A] {
  def show: String
}

Bir sınıfın you denetimini (ve Scala’da yazılmış) yapmak için tür sınıfını genişletmek için, eşlik eden nesneye bir örtük ekleyin. Bu örnekten ‘Person’ sınıfını nasıl alabileceğimizi gösterelim 201607260225051018393) “Göster"i genişletmek için:

class Person(val fullName: String) {    
  def this(firstName: String, lastName: String) = this(s"$firstName $lastName")
}

“Person"ın eşlik eden nesnesine bir örtük ekleyerek bu sınıfın “Show"u genişletmesini sağlayabiliriz:

object Person {
  implicit val personShow: Show[Person] = new Show {
    def show(p: Person): String = s"Person(${p.fullname})"
  }
}

Tamamlayıcı nesne sınıfla aynı dosyada_ olmalıdır, bu nedenle aynı dosyada hem “sınıf Kişisi"ne hem de “nesne Kişisi"ne ihtiyacınız vardır.

Kontrol etmediğiniz veya Scala’da yazılmamış bir sınıf oluşturmak için, tür sınıfını genişletin, Basit Tür Sınıfı‘da gösterildiği gibi, tür sınıfının tamamlayıcı nesnesine örtük bir öğe ekleyin. /documentation/scala/3835/type-classes/13263/simple-type-class) örneği.

Ne sınıfı ne de tür sınıfını kontrol etmiyorsanız, herhangi bir yerde yukarıdaki gibi bir örtük oluşturun ve onu ‘içe aktarın’. [Basit Tip Sınıf örneğinde](https://www.wikiod.com/tr/scala/tip-snflar#Basit Tip Sınıfı) “log” yöntemini kullanarak:

object MyShow {
  implicit val personShow: Show[Person] = new Show {
    def show(p: Person): String = s"Person(${p.fullname})"
  }
}

def logPeople(persons: Person*): Unit = {
  import MyShow.personShow
  persons foreach { p => log(p) }
}

Türlere tür sınıfı işlevleri ekleyin #

Scala’nın tip sınıfları uygulaması oldukça ayrıntılıdır. Ayrıntıyı azaltmanın bir yolu, “İşlem Sınıfları” olarak adlandırılanları tanıtmaktır. Bu sınıflar, işlevselliği genişletmek için içe aktarıldıklarında bir değişkeni/değeri otomatik olarak sarar.

Bunu göstermek için önce basit bir tip sınıfı oluşturalım:

// The mathematical definition of "Addable" is "Semigroup"
trait Addable[A] {
  def add(x: A, y: A): A
}

Daha sonra özelliği uygulayacağız (tür sınıfını örneklendirelim):

object Instances {

  // Instance for Int
  // Also called evidence object, meaning that this object saw that Int learned how to be added
  implicit object addableInt extends Addable[Int] {
    def add(x: Int, y: Int): Int = x + y
  }

  // Instance for String
  implicit object addableString extends Addable[String] {
    def add(x: String, y: String): String = x + y
  }

}

Şu anda Addable örneklerimizi kullanmak çok zahmetli olacaktır:

import Instances._
val three = addableInt.add(1,2)

Biz sadece 1.add(2) yazmayı tercih ederiz. Bu nedenle, her zaman ‘Addable’ uygulayan bir türün üzerine sarılacak bir “İşlem Sınıfı” (“Ops Sınıfı” olarak da adlandırılır) oluşturacağız.

object Ops {
  implicit class AddableOps[A](self: A)(implicit A: Addable[A]) {
    def add(other: A): A = A.add(self, other)
  }
}

Artık yeni fonksiyonumuz olan ‘add’i, ‘Int’ ve ‘String’in bir parçasıymış gibi kullanabiliriz:

object Main {

  import Instances._ // import evidence objects into this scope
  import Ops._       // import the wrappers

  def main(args: Array[String]): Unit = {

    println(1.add(5))
    println("mag".add("net"))
    // println(1.add(3.141)) // Fails because we didn't create an instance for Double

  }
}

“Ops” sınıfları, simulacrum kitaplığındaki makrolar tarafından otomatik olarak oluşturulabilir:

import simulacrum._

@typeclass trait Addable[A] {
  @op("|+|") def add(x: A, y: A): A
}
