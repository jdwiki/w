Parametreli Yöntemler #

Bir yöntemin dönüş türü, parametrenin türüne bağlı olabilir. Bu örnekte, “x” parametredir, “A”, tür parametresi olarak bilinen “x"in türüdür.

def f[A](x: A): A = x

f(1)         // 1
f("two")     // "two"
f[Float](3)  // 3.0F

Scala, parametrede hangi yöntemlerin çağrılabileceğini sınırlayan dönüş türünü belirlemek için tür çıkarımı kullanır. Bu nedenle, dikkatli olunmalıdır: Aşağıdaki, bir derleme zamanı hatasıdır, çünkü “*” her “A” türü için tanımlanmamıştır:

def g[A](x: A): A = 2 * x  // Won't compile

Genel koleksiyon #

Ints listesini tanımlama #

trait IntList { ... }

class Cons(val head: Int, val tail: IntList) extends IntList { ... }

class Nil extends IntList { ... }

peki ya Boolean, Double vb. listeyi tanımlamamız gerekirse?

Genel liste tanımlama #

trait List[T] {
  def isEmpty: Boolean
  def head: T
  def tail: List[T]
}

class Cons[T](val head: [T], val tail: List[T]) extends List[T] {
  def isEmpty: Boolean = false
}

class Nil[T] extends List[T] {
  def isEmpty: Boolean = true

  def head: Nothing = throw NoSuchElementException("Nil.head")

  def tail: Nothing = throw NoSuchElementException("Nil.tail")
}

Seçenek türü #

Parametreli türe güzel bir örnek, Seçenek türü. Esasen sadece aşağıdaki tanımdır (türde tanımlanmış birkaç yöntem daha vardır):

sealed abstract class Option[+A] {
  def isEmpty: Boolean
  def get: A

  final def fold[B](ifEmpty: => B)(f: A => B): B =
    if (isEmpty) ifEmpty else f(this.get)

  // lots of methods...
}

case class Some[A](value: A) extends Option[A] {
  def isEmpty = false
  def get = value
}

case object None extends Option[Nothing] {
  def isEmpty = true
  def get = throw new NoSuchElementException("None.get")
}

Bunun ayrıca, “B” türünden bir şey döndüren “fold” adlı parametreli bir yöntemi olduğunu da görebiliriz.
