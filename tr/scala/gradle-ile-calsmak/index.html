Kendi Gradle Scala eklentinizi oluşturun #

Temel Kurulum örneğini inceledikten sonra, kendinizi her bir Scala Gradle projesinde bunun çoğunu tekrarlarken bulabilirsiniz. Ortak kod gibi kokuyor…

Gradle tarafından sunulan [Scala eklentisini] uygulamak yerine, tüm ortak yapı mantığınızı yönetmekten ve aynı zamanda mevcut eklentiyi genişletmekten sorumlu olacak kendi Scala eklentinizi uygulayabilseydiniz ne olurdu?

Bu örnek, önceki yapı mantığını yeniden kullanılabilir bir Gradle eklentisine dönüştürecek.

Neyse ki, Gradle’da, [belgelerde] belirtildiği gibi Gradle API’sinin yardımıyla kolayca özel eklentiler yazabilirsiniz. Uygulama dili olarak Scala’nın kendisini ve hatta Java’yı kullanabilirsiniz. Ancak, belgelerde bulabileceğiniz örneklerin çoğu Groovy’de yazılmıştır. Daha fazla kod örneğine ihtiyacınız varsa veya Scala eklentisinin arkasında ne olduğunu anlamak istiyorsanız, örneğin, github repo derecesini kontrol edebilirsiniz.

Eklentiyi yazma #

Gereksinimler #

Özel eklenti, bir projeye uygulandığında aşağıdaki işlevleri ekleyecektir:

  • iki geçersiz kılınabilir varsayılan özelliğe sahip olacak bir “scalaVersion” özellik nesnesi
    • major = “2.12”
    • minor = “0”
  • bir bağımlılık adına uygulanan bir “withScalaVersion” işlevi, ikili uyumluluğu sağlamak için scala ana sürümünü ekleyecektir. (sbt %% operatörü bir zil çalabilir, aksi takdirde devam etmeden önce [buraya] gidin)
  • tam olarak önceki örnekte olduğu gibi, gerekli dizin ağacını oluşturmak için bir “createDirs” görevi

Uygulama kılavuzu #

  1. yeni bir gradle projesi oluşturun ve aşağıdakini build.gradle dosyasına ekleyin
apply plugin: 'scala'
apply plugin: 'maven'

repositories {
    mavenLocal()
    mavenCentral()
}

dependencies {
    compile gradleApi()
    compile "org.scala-lang:scala-library:2.12.0"
}

Notlar:

  • eklenti uygulaması Scala’da yazılmıştır, bu nedenle Gradle Scala Eklentisine ihtiyacımız var
  • Eklentiyi diğer projelerden kullanmak için Gradle Maven Eklentisi kullanılır; bu, proje kavanozunu Maven Yerel Deposuna kaydetmek için kullanılan “kurulum” görevini ekler
  • ‘derleme gradleApi()’, sınıf yoluna ‘gradle-api-<gradle_version>.jar’ı ekler
  1. Eklenti uygulaması için yeni bir Scala sınıfı oluşturun
package com.btesila.gradle.plugins 
 
import org.gradle.api.{Plugin, Project}  
 
class ScalaCustomPlugin extends Plugin[Project] {
    override def apply(project: Project): Unit = {
        project.getPlugins.apply("scala")
    }
}

Notlar:

  • bir Eklentiyi uygulamak için, sadece “Proje” türündeki “Eklenti” özelliğini genişletin ve “uygula” yöntemini geçersiz kılın
  • application yöntemi içinde, eklentinin uygulandığı Project örneğine erişiminiz var ve onu, ona yapı mantığı eklemek için kullanabilirsiniz.
  • bu eklenti zaten var olan Gradle Scala Eklentisini uygulamaktan başka bir şey yapmaz
  1. ‘scalaVersion’ nesne özelliğini ekleyin

İlk olarak, iki sürüm özelliğini tutacak bir ScalaVersion sınıfı oluşturuyoruz.

class ScalaVersion {
  var major: String = "2.12"
  var minor: String = "0"
}

Gradle eklentileriyle ilgili harika bir şey, her zaman belirli özellikleri ekleyebilmeniz veya geçersiz kılabilmenizdir. Bir eklenti, bu tür bir kullanıcı girdisini, bir gradle ‘Project’ örneğine eklenen ‘ExtensionContainer’ aracılığıyla alır. Daha fazla ayrıntı için [bunu] kontrol edin. ‘apply’ yöntemine aşağıdakileri ekleyerek temelde bunu yapıyoruz:

  • projede tanımlanmış bir scalaVersion özelliği yoksa, varsayılan değerlerle bir tane ekliyoruz
  • aksi takdirde, daha fazla kullanmak için mevcut olanı ScalaVersion örneği olarak alırız
var scalaVersion = new ScalaVersion
if (!project.getExtensions.getExtraProperties.has("scalaVersion"))
   project.getExtensions.getExtraProperties.set("scalaVersion", scalaVersion)
başka
   scalaVersion = project.getExtensions.getExtraProperties.get("scalaVersion").asInstanceOf[ScalaVersion]

This is equivalent to writing the following to the build file of the project that applies the plugin:

dış {
    scalaVersion.major = "2.12"
    scalaVersion.minor = "0"

}
  1. add the scala-lang library to the project dependencies, using the scalaVersion
project.getDependencies.add("derleme", s"org.scala-lang:scala-library:${scalaVersion.major}.${scalaVersion.minor}")

This is equivalent to writing the following to the build file of the project that applies the plugin:

"org.scala-lang:scala-library:2.12.0" derleyin
  1. add the withScalaVersion function
val withScalaVersion = (lib: String) => {
    val libComp = lib.split(":")
    libComp.update(1, s"${libComp(1)}_${scalaVersion.major}")
    libComp.mkString(":")
}
project.getExtensions.getExtraProperties.set("withScalaVersion", withScalaVersion)
  1. finally, create the createDirs task and add it to the project
    Implement a Gradle task by extending DefaultTask:
class CreateDirs, DefaultTask'ı genişletir {
@TaskAction
def createDirs(): Birim = {
    val sourceSetContainer = this.getProject.getConvention.getPlugin(classOf[JavaPluginConvention]).getSourceSets

    sourceSetContainer forEach { sourceSet =>
      sourceSet.getAllSource.getSrcDirs.forEach(file => if (!file.getName.contains("java")) file.mkdirs())
    }
  }
}

Note: the SourceSetContainer has information about all source directories present in the project. What the Gradle Scala Plugin does, is to add the extra source sets to the Java ones, as you can see in theplugin docs.

Add the createDir task to the project by appending this to the apply method:

project.getTasks.create("createDirs", classOf[CreateDirs])

In the end, your ScalaCustomPlugin class should look like this:

class ScalaCustomPlugin Eklentiyi genişletiyor[Proje] {
geçersiz kıl def uygula(proje: Proje): Birim = {
    project.getPlugins.apply("scala")

    var scalaVersion = new ScalaVersion
    if (!project.getExtensions.getExtraProperties.has("scalaVersion"))
      project.getExtensions.getExtraProperties.set("scalaVersion", scalaVersion)
    else
      scalaVersion = project.getExtensions.getExtraProperties.get("scalaVersion").asInstanceOf[ScalaVersion]

    project.getDependencies.add("compile", s"org.scala-lang:scala-library:${scalaVersion.major}.${scalaVersion.minor}")

    val withScalaVersion = (lib: String) => {
      val libComp = lib.split(":")
      libComp.update(1, s"${libComp(1)}_${scalaVersion.major}")
      libComp.mkString(":")
    }
    project.getExtensions.getExtraProperties.set("withScalaVersion", withScalaVersion)

    project.getTasks.create("createDirs", classOf[CreateDirs])
  }
}

Installing the plugin project to the local Maven repository #

This is done really easy by running gradle install
You can check the installation by going to local repository directory, usually found at ~/.m2/repository

How does Gradle find our new plugin? #

Each Gradle plugin has an id which is used in the apply statement. For instance, by writing the following to the build file, it translates to a trigger to Gradle to find and apply the plugin with id scala.

eklentiyi uygula: 'scala'

In the same way, we would like to apply our new plugin in the following way,

eklentiyi uygula: "com.btesila.scala.plugin"

meaning that our plugin will have the com.btesila.scala.plugin id.

In order to set this id, add the following file:
src/main/resources/META-INF/gradle-plugin/com.btesil.scala.plugin.properties

uygulama sınıfı=com.btesila.gradle.plugins.ScalaCustomPlugin

Afterwards, run again gradle install.

Using the plugin #

  1. create a new empty Gradle project and add the following to the build file
yapı komut dosyası {
    repositories {
        mavenLocal()
        mavenCentral()
    }

    dependencies {
        //modify this path to match the installed plugin project in your local repository
        classpath 'com.btesila:working-with-gradle:1.0-SNAPSHOT'
    }
}

depolar {
    mavenLocal()
    mavenCentral()
}


eklentiyi uygula: "com.btesila.scala.plugin"
  1. run gradle createDirs - you should now have all the source directories generated
  2. override the scala version by adding this to the build file:
dış {
    scalaVersion.major = "2.11"
    scalaVersion.minor = "8"

}
println(proje.ext.scalaVersion.major)
println(proje.ext.scalaVersion.minor)
  1. add a dependency library that is binary compatible with the Scala version
bağımlılıklar {
    compile withScalaVersion("com.typesafe.scala-logging:scala-logging:3.5.0")
}

That’s it! You can now use this plugin across all your projects without repeating the same old boilerplate.

Basic Setup #

  1. Create a file named SCALA_PROJECT/build.gradle with these contents:

    group 'scala_gradle'
version '1.0-SNAPSHOT'

apply plugin: 'scala'

repositories {
    jcenter()
    mavenCentral()
    maven {
        url "https://repo.typesafe.com/typesafe/maven-releases"
    }
}

dependencies {
    compile group: 'org.scala-lang', name: 'scala-library', version: '2.10.6'
}


task  "create-dirs" << {
    sourceSets*.scala.srcDirs*.each { it.mkdirs() }
    sourceSets*.resources.srcDirs*.each { it.mkdirs() }
}

  2. Run gradle tasks to see available tasks.

  3. Run gradle create-dirs to create a src/scala, src/resources directory.

  4. Run gradle build to build the project and download dependencies.
