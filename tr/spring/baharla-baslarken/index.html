Kurulum (XML Yapılandırması) #

Merhaba Bahar oluşturma adımları:

  1. İhtiyaçlarınızı daha iyi karşılayıp karşılamayacağını görmek için Spring Boot‘i araştırın.
  2. Doğru bağımlılıklarla kurulmuş bir projeye sahip olun. Maven veya Gradle kullanmanız önerilir.
  3. Bir POJO sınıfı oluşturun, örn. Çalışan.java
  4. Sınıfınızı ve değişkenlerinizi tanımlayabileceğiniz bir XML dosyası oluşturun. ör. “beans.xml”
  5. ana sınıfınızı oluşturun, ör. ‘Müşteri.java’
  6. spring-beans (ve geçişli bağımlılıklarını!) bağımlılık olarak ekleyin.

“Çalışan.java”:

package com.test;

public class Employee {

    private String name;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public void displayName() {
        System.out.println(name);
    }
}

“beans.xml”:

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
    http://www.springframework.org/schema/beans/spring-beans-4.3.xsd">
        
    <bean id="employee" class="com.test.Employee">
        <property name="name" value="test spring"></property>
    </bean>

</beans>

“Müşteri.java”:

package com.test;

import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;

public class Customer {
   public static void main(String[] args) {
      ApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");

      Employee obj = (Employee) context.getBean("employee");
      obj.displayName();
   }
}

Örnek olarak Çekirdek Yay Özelliklerini Sergileme #

Tanım #

Bu, aşağıdakileri içeren/gösteren bağımsız çalışan bir örnektir: gerekli minimum bağımlılıklar, Java Yapılandırma, açıklama ve Java Yapılandırması ile Bean bildirimi, Yapıcı ve Özelliğe göre Bağımlılık Enjeksiyonu ve Ön/Sonra kancaları .

bağımlılıklar #

Bu bağımlılıklar sınıf yolunda gereklidir:

  1. yay-çekirdek
  2. bahar bağlamı
  3. bahar fasulyesi
  4. bahar-aop
  5. bahar ifadesi
  6. ortak oturum açma

Ana sınıf #

Sondan başlayarak, bu, Configuration sınıfını işaret ederek Application Context’i başlatan ve belirli işlevleri sergilemek için gereken tüm çeşitli çekirdekleri yükleyen ‘main()’ yöntemi için yer tutucu görevi gören Main sınıfımızdır.

package com.stackoverflow.documentation;

import org.springframework.context.ApplicationContext;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;



public class Main {

    public static void main(String[] args) {

        //initializing the Application Context once per application.
        ApplicationContext applicationContext = 
                new AnnotationConfigApplicationContext(AppConfig.class);

        //bean registered by annotation
        BeanDeclaredByAnnotation beanDeclaredByAnnotation = 
                applicationContext.getBean(BeanDeclaredByAnnotation.class);
        beanDeclaredByAnnotation.sayHello();

        //bean registered by Java configuration file
        BeanDeclaredInAppConfig beanDeclaredInAppConfig = 
                applicationContext.getBean(BeanDeclaredInAppConfig.class);
        beanDeclaredInAppConfig.sayHello();

        //showcasing constructor injection
        BeanConstructorInjection beanConstructorInjection = 
                applicationContext.getBean(BeanConstructorInjection.class);
        beanConstructorInjection.sayHello();

        //showcasing property injection
        BeanPropertyInjection beanPropertyInjection = 
                applicationContext.getBean(BeanPropertyInjection.class);
        beanPropertyInjection.sayHello();

        //showcasing PreConstruct / PostDestroy hooks
        BeanPostConstructPreDestroy beanPostConstructPreDestroy = 
                applicationContext.getBean(BeanPostConstructPreDestroy.class);
        beanPostConstructPreDestroy.sayHello();
    }
}

Uygulama Yapılandırma dosyası #

Konfigürasyon sınıfı, @Configuration ile açıklanır ve başlatılan Uygulama Bağlamında bir parametre olarak kullanılır. Yapılandırma sınıfının sınıf düzeyindeki @ComponentScan ek açıklaması, ek açıklamalar kullanılarak kaydedilen Beans ve bağımlılıklar için taranacak bir pakete işaret eder. Son olarak, @Bean notu, konfigürasyon sınıfında bir bean tanımı olarak hizmet eder.

package com.stackoverflow.documentation;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;

@Configuration
@ComponentScan("com.stackoverflow.documentation")
public class AppConfig {

    @Bean
    public BeanDeclaredInAppConfig beanDeclaredInAppConfig() {
        return new BeanDeclaredInAppConfig();
    }
}

Ek Açıklama ile Fasulye Deklarasyonu #

‘@Component’ ek açıklaması, POJO’yu bileşen taraması sırasında kayıt için mevcut bir Bahar fasulyesi olarak ayırmaya yarar.

@Component
public class BeanDeclaredByAnnotation {

    public void sayHello() {
        System.out.println("Hello, World from BeanDeclaredByAnnotation !");
    }
}

Uygulama Konfigürasyonuna Göre Fasulye Beyanı #

Uygulama Konfigürasyonu sınıf dosyasında fasulye bildirimi/tanımı gerçekleştiğinden, POJO’muza açıklama eklememize veya başka bir şekilde işaretlememize gerek olmadığına dikkat edin.

public class BeanDeclaredInAppConfig {

    public void sayHello() {
        System.out.println("Hello, World from BeanDeclaredInAppConfig !");
    }
}

Yapıcı Enjeksiyon #

@Autowired notunun yapıcı düzeyinde ayarlandığına dikkat edin. Ayrıca, adla açıkça tanımlanmadıkça, varsayılan otomatik kablolamanın fasulyenin türüne bağlı olarak gerçekleştiğine dikkat edin (bu örnekte ‘BeanToBeInjected’).

package com.stackoverflow.documentation;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class BeanConstructorInjection {

    private BeanToBeInjected dependency;

    @Autowired
    public BeanConstructorInjection(BeanToBeInjected dependency) {
        this.dependency = dependency;
    }

    public void sayHello() {
        System.out.print("Hello, World from BeanConstructorInjection with dependency: ");
        dependency.sayHello();
    }
}

Özellik Enjeksiyonu #

@Autowired notunun, adı JavaBeans standardını izleyen ayarlayıcı yöntemini sınırlandırdığına dikkat edin.

package com.stackoverflow.documentation;


import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class BeanPropertyInjection {

    private BeanToBeInjected dependency;

    @Autowired
    public void setBeanToBeInjected(BeanToBeInjected beanToBeInjected) {
        this.dependency = beanToBeInjected;
    }

    public void sayHello() {
        System.out.println("Hello, World from BeanPropertyInjection !");
    }
}

PostConstruct / PreDestroy kancaları #

Bir Bean’in başlatılmasını ve yok edilmesini @PostConstruct ve @PreDestroy kancaları ile durdurabiliriz.

package com.stackoverflow.documentation;

import org.springframework.stereotype.Component;

import javax.annotation.PostConstruct;
import javax.annotation.PreDestroy;

@Component
public class BeanPostConstructPreDestroy {

    @PostConstruct
    public void pre() {
        System.out.println("BeanPostConstructPreDestroy - PostConstruct");
    }

    public void sayHello() {
        System.out.println(" Hello World, BeanPostConstructPreDestroy !");
    }

    @PreDestroy
    public void post() {
        System.out.println("BeanPostConstructPreDestroy - PreDestroy");
    }
}

Spring Framework nedir, neden almalıyız? #

Spring, birçok sınıf sağlayan bir çerçevedir, bunu kullanarak kodumuzda kazan plakası mantığı yazmamıza gerek kalmaz, bu nedenle Spring, J2ee üzerinde soyut bir katman sağlar.

Örneğin, Basit JDBC Uygulamasında Programcı aşağıdakilerden sorumludur:

  1. Sürücü sınıfının yüklenmesi
  2. Bağlantının oluşturulması
  3. İfade nesnesi oluşturma
  4. İstisnaları işlemek
  5. Sorgu oluşturma
  6. Sorgu yürütme
  7. Bağlantının kapatılması

Her programcı aynı kodu yazdığı için ortak kod olarak kabul edilir. Bu nedenle, basitlik için çerçeve, kalıp mantığıyla ilgilenir ve programcının yalnızca iş mantığı yazması gerekir. Böylece, Spring framework kullanarak, minimum kod satırı ile, herhangi bir hata olmadan, hızlı bir şekilde projeler geliştirebiliyoruz, geliştirme maliyeti ve süresi de azaldı.

Öyleyse, payanda olarak neden Bahar’ı seçmelisiniz #

Strut, yalnızca web yönlerine çözüm sağlayan bir çerçevedir ve struts doğası gereği istilacıdır. Yay, payandalara göre birçok özelliğe sahiptir, bu yüzden Bahar’ı seçmeliyiz.

  1. Bahar doğası gereği İstilacı değildir: Bu, herhangi bir sınıfı genişletmeniz veya sınıfınıza herhangi bir arabirim uygulamanız gerekmediği anlamına gelir.
  2. Bahar çok yönlüdür: Bu, projenizdeki mevcut herhangi bir teknolojiyle entegre olabileceği anlamına gelir.
  3. Bahar uçtan uca proje geliştirme sağlar: Bu, iş katmanı, kalıcılık katmanı gibi tüm modülleri geliştirebileceğimiz anlamına gelir.
  4. Yay hafiftir: Yani belirli bir modül üzerinde çalışmak istiyorsanız, o zaman tüm yayı öğrenmeniz gerekmez, sadece o modülü öğrenin (örn. Spring Jdbc, Spring DAO)
  5. Yay, bağımlılık enjeksiyonunu destekler.
  6. Spring, çoklu proje geliştirmeyi destekler, örneğin: Çekirdek Java Uygulaması, Web Uygulaması, Dağıtılmış Uygulama, Kurumsal Uygulama.
  7. Yay, kesişen sorunlar için En Boy Yönelimli Programlamayı destekler.

Sonunda Spring’in Struts’a bir alternatif olduğunu söyleyebiliriz. Ancak Spring, J2EE API’sinin yerine geçmez, Spring tarafından sağlanan sınıflar dahili olarak J2EE API sınıflarını kullanır. Spring geniş bir çerçevedir, bu nedenle birkaç modüle ayrılmıştır. Spring Core dışında hiçbir modül diğerine bağımlı değildir. Bazı Önemli modüller
