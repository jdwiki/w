SimpleJdbcCall'dan SqlRowSet Alma

  • SimpleJdbcCall oluşturma
  • Oracle Veritabanları

Bu sayfada

  • SimpleJdbcCall oluşturma
  • Oracle Veritabanları

Bu, veritabanınızda imleç çıktı parametresi olan saklı bir prosedürle SimpleJdbcCall kullanarak bir SqlRowSet‘in doğrudan nasıl elde edileceğini açıklar,

Bir Oracle veritabanıyla çalışıyorum, diğer veritabanları için çalışması gereken bir örnek oluşturmaya çalıştım, Oracle örneğim Oracle ile ilgili sorunları ayrıntılı olarak anlatıyor.

SimpleJdbcCall oluşturma #

Tipik olarak, SimpleJdbcCalls’ınızı bir Hizmette oluşturmak isteyeceksiniz.

Bu örnek, prosedürünüzün imleç olan tek bir çıktı parametresine sahip olduğunu varsayar; yordamınıza uyması için beyanParameters’ınızı ayarlamanız gerekecektir.

@Service
public class MyService() {

@Autowired
    private DataSource dataSource;
    
    // Autowire your configuration, for example
    @Value("${db.procedure.schema}")
    String schema;
    
    private SimpleJdbcCall myProcCall;
    
    // create SimpleJdbcCall after properties are configured
    @PostConstruct
    void initialize() {
        this.myProcCall = new SimpleJdbcCall(dataSource)
                        .withProcedureName("my_procedure_name")
                        .withCatalogName("my_package")
                        .withSchemaName(schema)
                        .declareParameters(new SqlOutParameter(
                            "out_param_name",
                            Types.REF_CURSOR, 
                            new SqlRowSetResultSetExtractor()));
    }

    public SqlRowSet myProc() {
        Map<String, Object> out = this.myProcCall.execute();
        return (SqlRowSet) out.get("out_param_name");
    }

}

Burada kullanabileceğiniz birçok seçenek var:

  • withoutProcedureColumnMetaDataAccess(), prosedür adlarını aşırı yüklediyseniz veya SimpleJdbcCall’ın veritabanına karşı doğrulama yapmasını istemiyorsanız gereklidir.
  • withReturnValue() prosedürün bir dönüş değeri varsa. beyanParameters’a verilen ilk değer, dönüş değerini tanımlar. Ayrıca, prosedürünüz bir işlevse, yürütürken withFunctionName ve executeFunction kullanın.
  • withNamedBinding() argümanları konum yerine isimler kullanarak vermek istiyorsanız.
  • useInParameterNames() bağımsız değişken sırasını tanımlar. Argümanlarınızı değere bir argüman adı haritası yerine bir liste olarak iletirseniz bunun gerekli olabileceğini düşünüyorum. Ancak, yalnızca WithoutProcedureColumnMetaDataAccess() kullanıyorsanız gerekli olabilir.

Oracle Veritabanları #

Oracle ile ilgili bir takım sorunlar var. İşte bunları nasıl çözeceğiniz.

Prosedür çıktı parametrenizin ‘ref imleç’ olduğunu varsayarsak, bu istisnayı alırsınız.

java.sql.SQLException: Geçersiz sütun türü: 2012

simpleJdbcCall.declareParameters() içindeki Types.REF_CURSOR öğesini OracleTypes.CURSOR olarak değiştirin

OracleTypes’ı Destekleme

Bunu yalnızca verilerinizde belirli sütun türleri varsa yapmanız gerekebilir.

Karşılaştığım sonraki sorun, “oracle.sql.TIMESTAMPTZ” gibi tescilli Türlerin SqlRowSetResultSetExtractor’da bu hataya neden olmasıydı:

Sütun için geçersiz SQL türü; iç içe istisna Java.sql.SQLException: Sütun için geçersiz SQL türü

Bu yüzden Oracle türlerini destekleyen bir ResultSetExtractor oluşturmamız gerekiyor. Şifrenin nedenini bu koddan sonra açıklayacağım.

package com.boost.oracle;

import oracle.jdbc.rowset.OracleCachedRowSet;
import org.springframework.dao.DataAccessException;
import org.springframework.jdbc.core.ResultSetExtractor;
import org.springframework.jdbc.support.rowset.ResultSetWrappingSqlRowSet;
import org.springframework.jdbc.support.rowset.SqlRowSet;

import java.sql.ResultSet;
import java.sql.SQLException;

/**
 * OracleTypes can cause {@link org.springframework.jdbc.core.SqlRowSetResultSetExtractor}
 * to fail due to a Oracle SQL type that is not in the standard {@link java.sql.Types}.
 *
 * Also, types such as {@link oracle.sql.TIMESTAMPTZ} require a Connection when processing
 * the ResultSet; {@link OracleCachedRowSet#getConnectionInternal()} requires a JNDI
 * DataSource name or the username and password to be set.
 *
 * For now I decided to just set the password since changing SpringBoot to a JNDI DataSource
 * configuration is a bit complicated.
 *
 * Created by Arlo White on 2/23/17.
 */
public class OracleSqlRowSetResultSetExtractor implements ResultSetExtractor<SqlRowSet> {

    private String oraclePassword;

    public OracleSqlRowSetResultSetExtractor(String oraclePassword) {
        this.oraclePassword = oraclePassword;
    }

    @Override
    public SqlRowSet extractData(ResultSet rs) throws SQLException, DataAccessException {
        OracleCachedRowSet cachedRowSet = new OracleCachedRowSet();
        // allows getConnectionInternal to get a Connection for TIMESTAMPTZ
        cachedRowSet.setPassword(oraclePassword);
        cachedRowSet.populate(rs);
        return new ResultSetWrappingSqlRowSet(cachedRowSet);
    }

}

Belirli Oracle türleri, bir ResultSet’ten sütun değerini almak için bir Bağlantı gerektirir. TIMESTAMPTZ bu türlerden biridir. Böylece rowSet.getTimestamp(colIndex) çağrıldığında şu istisnayı alırsınız:

Nedeni: Java.sql.SQLException: Bir veya daha fazla kimlik doğrulama RowSet özellikleri ayarlanmadı oracle.jdbc.rowset.OracleCachedRowSet.getConnectionInternal’da(OracleCachedRowSet.Java:560) oracle.jdbc.rowset.OracleCachedRowSet.getTimestamp’ta(OracleCachedRowSet.Java:3717) org.springframework.jdbc.support.rowset.ResultSetWrappingSqlRowSet.getTimestamp adresinde

Bu kodu incelerseniz, OracleCachedRowSet’in Bağlantı almak için parolaya veya JNDI DataSource adına ihtiyacı olduğunu göreceksiniz. JNDI aramasını tercih ederseniz, OracleCachedRowSet’in DataSourceName ayarlanmış olduğunu doğrulamanız yeterlidir.

Bu yüzden Hizmetimde, şifreyi otomatik olarak bağladım ve çıktı parametresini şu şekilde ilan ediyorum:

new SqlOutParameter("cursor_param_name", OracleTypes.CURSOR, new OracleSqlRowSetResultSetExtractor(oraclePassword))
