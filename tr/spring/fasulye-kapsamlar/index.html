Web uyumlu bağlamlarda ek kapsamlar #

Yalnızca web’i tanıyan bir uygulama bağlamında kullanılabilen birkaç kapsam vardır:

  • istek - HTTP isteği başına yeni fasulye örneği oluşturulur
  • oturum - HTTP oturumu başına yeni fasulye örneği oluşturulur
  • application - ‘ServletContext’ başına yeni fasulye örneği oluşturulur
  • globalSession - Portlet ortamında global oturum başına yeni bean örneği oluşturulur (Servlet ortamında global oturum kapsamı, oturum kapsamına eşittir)
  • websocket - WebSocket oturumu başına yeni fasulye örneği oluşturulur

Spring Web MVC ortamında web kapsamlı fasulyeleri bildirmek ve bunlara erişmek için ek kurulum gerekmez.

XML Yapılandırması #

<bean id="myRequestBean" class="OneClass" scope="request"/>
<bean id="mySessionBean" class="AnotherClass" scope="session"/>
<bean id="myApplicationBean" class="YetAnotherClass" scope="application"/>
<bean id="myGlobalSessionBean" class="OneMoreClass" scope="globalSession"/>

Java Yapılandırması (Bahar 4.3’ten önce) #

@Configuration
public class MyConfiguration {

    @Bean
    @Scope(value = WebApplicationContext.SCOPE_REQUEST, proxyMode = ScopedProxyMode.TARGET_CLASS)
    public OneClass myRequestBean() {
        return new OneClass();
    }

    @Bean
    @Scope(value = WebApplicationContext.SCOPE_SESSION, proxyMode = ScopedProxyMode.TARGET_CLASS)
    public AnotherClass mySessionBean() {
        return new AnotherClass();
    }

    @Bean
    @Scope(value = WebApplicationContext.SCOPE_APPLICATION, proxyMode = ScopedProxyMode.TARGET_CLASS)
    public YetAnotherClass myApplicationBean() {
        return new YetAnotherClass();
    }

    @Bean
    @Scope(value = WebApplicationContext.SCOPE_GLOBAL_SESSION, proxyMode = ScopedProxyMode.TARGET_CLASS)
    public OneMoreClass myGlobalSessionBean() {
        return new OneMoreClass();
    }
}

Java Yapılandırması (Bahar 4.3’ten sonra) #

@Configuration
public class MyConfiguration {

    @Bean
    @RequestScope
    public OneClass myRequestBean() {
        return new OneClass();
    }

    @Bean
    @SessionScope
    public AnotherClass mySessionBean() {
        return new AnotherClass();
    }

    @Bean
    @ApplicationScope
    public YetAnotherClass myApplicationBean() {
        return new YetAnotherClass();
    }
}

Prototip kapsamı #

Spring konteyner başlangıcında bir prototip kapsamlı fasulye önceden oluşturulmaz. Bunun yerine, konteynere bu çekirdeği almak için her istek gönderildiğinde yeni bir yeni örnek oluşturulacak. Bu kapsam, durumu diğer bileşenler tarafından paylaşılmayacağından durum bilgisi olan nesneler için önerilir.

Prototip kapsamlı bir bean tanımlamak için, istediğimiz kapsam türünü belirterek @Scope notunu eklememiz gerekiyor.

Aşağıdaki MyBean sınıfı göz önüne alındığında:

public class MyBean {
    private static final Logger LOGGER = LoggerFactory.getLogger(MyBean.class);
    private String property;

    public MyBean(String property) {
        this.property = property;
        LOGGER.info("Initializing {} bean...", property);
    }

    public String getProperty() {
        return this.property;
    }

    public void setProperty(String property) {
        this.property = property;
    }
}

Kapsamını prototip olarak belirten bir fasulye tanımı tanımlıyoruz:

@Configuration
public class PrototypeConfiguration {

    @Bean
    @Scope("prototype")
    public MyBean prototypeBean() {
        return new MyBean("prototype");
    }
}

Nasıl çalıştığını görmek için, bean’i Spring konteynerinden alıyoruz ve özellik alanı için farklı bir değer ayarlıyoruz. Ardından, çekirdeği kaptan tekrar alacağız ve değerini arayacağız:

MyBean prototypeBean1 = context.getBean("prototypeBean", MyBean.class);
prototypeBean1.setProperty("changed property");

MyBean prototypeBean2 = context.getBean("prototypeBean", MyBean.class);

logger.info("Prototype bean 1 property: " + prototypeBean1.getProperty());
logger.info("Prototype bean 2 property: " + prototypeBean2.getProperty());

Aşağıdaki sonuca bakarak her bean isteğinde nasıl yeni bir instance oluşturulduğu görebiliriz:

Initializing prototype bean...
Initializing prototype bean...
Prototype bean 1 property: changed property
Prototype bean 2 property: prototype

Sık yapılan bir hata, çekirdeğin çağrı başına veya iş parçacığı başına yeniden oluşturulduğunu varsaymaktır, durum DEĞİL. Bunun yerine, PER INJECTION (veya bağlamdan alma) bir örnek oluşturulur. Prototip kapsamlı bir fasulye, yalnızca tek bir tekli fasulyeye enjekte edilirse, o Prototip kapsamlı fasulyenin yalnızca bir örneği olacaktır.

Spring, bir prototip çekirdeğinin tüm yaşam döngüsünü yönetmez: kap, bir prototip nesnesini başlatır, yapılandırır, süsler ve başka bir şekilde birleştirir, müşteriye verir ve sonra bu prototip örneği hakkında daha fazla bilgiye sahip olmaz.

Singleton kapsamı #

Bir bean, singleton kapsamı ile tanımlanırsa, Spring konteynerinde yalnızca bir tek nesne örneği başlatılacaktır. Bu bene yapılan tüm istekler aynı paylaşılan örneği döndürür. Bu, bir fasulye tanımlarken varsayılan kapsamdır.

Aşağıdaki MyBean sınıfı göz önüne alındığında:

public class MyBean {
    private static final Logger LOGGER = LoggerFactory.getLogger(MyBean.class);
    private String property;

    public MyBean(String property) {
        this.property = property;
        LOGGER.info("Initializing {} bean...", property);
    }

    public String getProperty() {
        return this.property;
    }

    public void setProperty(String property) {
        this.property = property;
    }
}

@Bean notuyla bir singleton bean tanımlayabiliriz:

@Configuration
public class SingletonConfiguration {

    @Bean
    public MyBean singletonBean() {
        return new MyBean("singleton");
    } 
}

Aşağıdaki örnek, aynı çekirdeği Spring bağlamından iki kez alır:

MyBean singletonBean1 = context.getBean("singletonBean", MyBean.class);
singletonBean1.setProperty("changed property");

MyBean singletonBean2 = context.getBean("singletonBean", MyBean.class);

SingletonBean2 özelliğini günlüğe kaydederken, aynı paylaşılan örneği henüz aldığımız için “değişen özellik” mesajı gösterilecektir.

Örnek farklı bileşenler arasında paylaşıldığından, durumsuz nesneler için tekil kapsamın tanımlanması önerilir.

Tembel tekli fasulye #

Varsayılan olarak, tekli çekirdekler önceden örneklenmiştir. Bu nedenle, Spring konteyner oluşturulduğunda paylaşılan nesne örneği oluşturulacaktır. Uygulamayı başlatırsak, “Singleton bean başlatılıyor…” mesajı gösterilecektir.

Bean’in önceden örneklenmesini istemiyorsak, bean tanımına @Lazy notunu ekleyebiliriz. Bu, ilk talep edilene kadar çekirdeğin oluşturulmasını önleyecektir.

@Bean
@Lazy
public MyBean lazySingletonBean() {
    return new MyBean("lazy singleton");
}

Şimdi, Spring konteynerini başlatırsak, “Lazy singleton bean başlatılıyor…” mesajı görünmeyecektir. Fasulye, ilk kez talep edilene kadar oluşturulmaz:

logger.info("Retrieving lazy singleton bean...");
context.getBean("lazySingletonBean");

Uygulamayı hem singleton hem de lazy singleton fasulye tanımlı olarak çalıştırırsak, aşağıdaki mesajları üretecektir:

Initializing singleton bean...
Retrieving lazy singleton bean...
Initializing lazy singleton bean...
