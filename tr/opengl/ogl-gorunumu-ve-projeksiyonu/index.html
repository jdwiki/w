OGL 4.0 GLSL 400’de bir kamera uygulayın #

Bir sahneye kamerayla çekilmiş gibi bakmak istiyorsak, önce bazı şeyleri tanımlamamız gerekir:

  • Sahnenin görüntülendiği konum, göz konumu “pos”.
  • Sahnede baktığımız nokta (‘hedef’). It is also common to define the direction in which we look. Technically we need a line of sight. One straight in space is mathematically defined either by 2 points or by a point and a vector. The first part of the definition is the eye position and the 2nd is either the target or the line of sight vector los.
  • Yukarı doğru yön “yukarı”.
  • “fov_y” görüş alanı. This means the angle between the two straight lines, starting at the eye position and ending at the leftmost point and the rightmost point, which can be seen simultaneously.
  • “vp” resmimizi yansıttığımız görüntü alanının büyüklüğü ve en boy oranı.
  • yakın uçak “yakın” ve uzak uçak “uzak”. yakın düzlem, göz konumundan düzleme olan mesafedir from where the objects become visible to us. The far plane is the distance from the eye position to the plane to which the objects of the scene are visible to us. An explanation of what the near plane and far plane are needed will follow later..

Bir sahnenin tek tek parçalarını sahnede düzenlemek için model matrisleri kullanılır. Ancak burada bunlardan sadece eksiksiz olması açısından bahsedilmiştir ve burada ele alınmayacaktır.

  • Projeksiyon matrisi: Projeksiyon matrisi, dünyadaki 3B noktalardan, bir iğne deliği kameradan görüldüğü şekliyle, bakış alanının 2B noktalarına eşlemeyi tanımlar.

  • Matrisi görüntüle: Görünüm matrisi, sahnedeki göz konumunu ve görüş yönünü tanımlar.

  • Model matrisi: Model matrisi, sahnedeki bir nesnenin konumunu ve göreli boyutunu tanımlar.

Perspektifi ayarlayın - Projeksiyon matrisi #

Noktalar tarafından tanımlanan yerel AABB’de (eksen hizalı sınırlayıcı kutu) olduğunda, görünüm alanında bir nokta görünür (-1.0, -1.0, -1.0) ve (1.0, 1.0, 1.0). Buna Normalleştirilmiş Cihaz Koordinatları (NDC) denir. (-1.0, -1.0, z) koordinatlarına sahip bir nokta, görünümün sol alt köşesine boyanacak ve bir nokta `(1.0, 1.0, z) koordinatları ile görünümün sağ üst köşesine boyanacaktır. Z-koordinatı (-1.0, 1.0) aralığından (0.0, 1.0) aralığına eşlenir ve Z-tamponuna yazılır.

Sahneden görebildiğimiz tek şey 4 kenarlı bir piramidin içinde. Piramidin tepesi göz pozisyonudur. 4 tarafı piramit, görüş alanı (‘fov_y’) ve en boy oranı (‘vp[0]/vp1’) ile tanımlanır. İzdüşüm matrisi, piramidin içindeki noktaları, noktalar tarafından tanımlanan NDC’ye eşlemelidir. (-1.0, -1.0, -1.0) ve (1.0, 1.0, 1.0). Bu noktada piramidimiz sonsuzdur, derinliğinin bir sonu yoktur ve sonsuz bir uzayı sonlu bir uzaya eşleyemeyiz. Bunun için şimdi yakın düzleme ve uzak düzleme ihtiyacımız var, tepeyi keserek piramidi kesik kesik haline getiriyorlar. ve piramidi derinlikte sınırlamak. C++

using TVec4  = std::array< float, 4 >;
using TMat44 = std::array< TVec4, 4 >;
TMat44 Camera::Perspective( void )
{
    float fn  = far + near;
    float f_n = far - near;
    float r   = (float)vp[0] / vp[1];
    float t   = 1.0f / tan( ToRad( fov_y ) / 2.0f );
    return TMat44{ 
        TVec4{ t / r, 0.0f,  0.0f,                     0.0f },
        TVec4{ 0.0f,  t,     0.0f,                     0.0f },
        TVec4{ 0.0f,  0.0f, -fn / f_n,                -1.0  },
        TVec4{ 0.0f,  0.0f, -2.0f * far * near / f_n,  0.0f } };
}

Sahneye bakışı ayarlayın - Matrisi görüntüleyin #

Görünüm alanındaki koordinat sisteminde, Y ekseni (0, 1, 0) yukarıyı gösterir ve X ekseni sağa (1, 0, 0) işaret eder. Bu, görünüm alanından dışarıyı gösteren bir Z ekseni ile sonuçlanır. ( (0, 0, -1) = çapraz( X ekseni, Y ekseni )).

Sahnede X ekseni doğuyu, Y ekseni kuzeyi ve Z ekseni yukarıyı gösteriyor.

(1, 0, 0) görünümünün X ekseni, (1, 0, 0) sahnesinin Y ekseniyle eşleşir, (0, 1, 0 ) görünümünün Y ekseni, (0, 0, 1) sahnesinin Z ekseniyle eşleşir ve (0, 0, 1 ) görünümünün Z ekseni, (0, -1, 0) sahnesinin olumsuzlanmış Y ekseniyle eşleşir.

Sahnenin referans sisteminden her nokta ve her vektör bu nedenle ilk önce görüntü alanı koordinatlarına dönüştürülmelidir. Bu, skaler vektörlerde bazı takas ve ters çevirme işlemleriyle yapılabilir.

 x  y  z
--------
 1  0  0  | x' =  x
 0  0  1  | y' =  z
 0 -1  0  | z' = -y

Bir görünüm matrisi oluşturmak için, konum “konum”, hedef “hedef” ve yukarı vektör “yukarı”, yukarıda açıklandığı gibi görünüm alanı koordinat sistemine eşlenmelidir. Bu, aşağıdaki kod parçacığında olduğu gibi 2 nokta ‘p’ ve ’t’ ve ‘u’ vektörünü verir. Görünüm matrisinin Z ekseni, ‘p - t’ ile hesaplanan ters görüş hattıdır. Y ekseni yukarı vektör “u"dur. X ekseni, Y ekseni ve Z ekseninin çapraz çarpımı ile hesaplanır. Görünüm matrisini ortonormalleştirmek için, Z ekseninden ve X ekseninden Y eksenini hesaplamak için çapraz çarpım ikinci kez kullanılır (Tabii ki Gram-Schmidt dikleştirmesi de aynı şekilde çalışır). Sonunda, 3 dingilin tümü normalleştirilmeli ve göz konumu ‘pos’, görünüm matrisinin orijini olarak ayarlanmalıdır.

Aşağıdaki kod, sahneye bir bakışı hesaplamak için gerekli adımları tam olarak kapsayan bir matris tanımlar:

  1. Model koordinatlarını görüntü alanı koordinatlarına dönüştürme.
  2. Görüş yönüne doğru döndürün.
  3. Göz pozisyonuna hareket

C++

template< typename T_VEC >
TVec3 Cross( T_VEC a, T_VEC b )
{
    return { a[1] * b[2] - a[2] * b[1], a[2] * b[0] - a[0] * b[2], a[0] * b[1] - a[1] * b[0] };
}

template< typename T_A, typename T_B >
float Dot( T_A a, T_B b )
{ 
    return a[0]*b[0] + a[1]*b[1] + a[2]*b[2]; 
}

template< typename T_VEC >
void Normalize( T_VEC & v )
{ 
    float len = sqrt( v[0] * v[0] + v[1] * v[1] + v[2] * v[2] ); v[0] /= len; v[1] /= len; v[2] /= len; 
}

TMat44 Camera::LookAt( void )
{ 
    TVec3 mz = { pos[0] - target[0], pos[1] - target[1], pos[2] - target[2] };
    Normalize( mz );
    TVec3 my = { up[0], up[1], up[2] };
    TVec3 mx = Cross( my, mz );
    Normalize( mx );
    my = Cross( mz, mx );

    TMat44 v{
        TVec4{ mx[0], my[0], mz[0], 0.0f },
        TVec4{ mx[1], my[1], mz[1], 0.0f },
        TVec4{ mx[2], my[2], mz[2], 0.0f },
        TVec4{ Dot(mx, pos), Dot(my, pos), Dot(TVec3{-mz[0], -mz[1], -mz[2]}, pos), 1.0f }
    };

    return v;
}

piton

def LookAt(self):
    mz = Normalize( (self.pos[0]-self.target[0], self.pos[1]-self.target[1], self.pos[2]-self.target[2]) ) # inverse line of sight
    mx = Normalize( Cross( self.up, mz ) )
    my = Normalize( Cross( mz, mx ) )
    tx = Dot( mx, self.pos )
    ty = Dot( my, self.pos )
    tz = Dot( (-mz[0], -mz[1], -mz[2]), self.pos )   
    return = numpy.matrix( [ 
        [mx[0], my[0], mz[0], 0],
        [mx[1], my[1], mz[1], 0],
        [mx[2], my[2], mz[2], 0],
        [tx, ty, tz, 1] ] )

Matrisler son olarak tek biçimli olarak yazılır ve model konumlarını dönüştürmek için tepe gölgelendiricisinde kullanılır.

Vertex gölgelendirici

Köşe gölgelendiricide birbiri ardına dönüşüm gerçekleştirilir.

  1. Model matrisi, nesneyi (ağ) sahnedeki yerine getirir. (Bu sadece eksiksiz olması için listelenmiştir ve olay mahallinin görünümü ile ilgisi olmadığı için burada belgelenmemiştir)

  2. Görünüm matrisi, sahnenin görüntülendiği yönü tanımlar. Görünüm matrisi ile dönüştürme, sahnenin nesnelerini istenen açıdan görüntülenecek şekilde döndürür. bakış açısının koordinat sistemine referansla bakış yönü.

  3. İzdüşüm matrisi, nesneleri paralel bir görünümden bir perspektif görünüme dönüştürür.

    #version 400

    layout (location = 0) in vec3 inPos; layout (location = 1) in vec3 inCol;

    out vec3 vertCol;

    uniform mat4 u_projectionMat44; uniform mat4 u_viewMat44; uniform mat4 u_modelMat44;

    void main() { vertCol = inCol; vec4 modelPos = u_modelMat44 * vec4( inPos, 1.0 ); vec4 viewPos = u_viewMat44 * modelPos; gl_Position = u_projectionMat44 * viewPos; }

        glUseProgram( self.__prog )
    def SetUniformMat44( self, name, mat ):
        glUniformMatrix4fv( self.__unifomLocation[name], 1, GL_FALSE, mat )
    # read shader program and compile shader
    def CompileShader(self, sourceFileName, shaderStage):
        with open( sourceFileName, 'r' ) as sourceFile:
            sourceCode = sourceFile.read()
        nameMap = { GL_VERTEX_SHADER: 'vertex', GL_FRAGMENT_SHADER: 'fragment' }    
        print( '\n%s shader code:' % nameMap.get( shaderStage, '' ) )
        print( sourceCode )
        shaderObj = glCreateShader( shaderStage )
        glShaderSource( shaderObj, sourceCode )
        glCompileShader( shaderObj )
        result = glGetShaderiv( shaderObj, GL_COMPILE_STATUS )
        if not (result):
            print( glGetShaderInfoLog( shaderObj ) )
            sys.exit()
        return shaderObj
    # linke shader objects to shader program
    def LinkProgram(self, shaderObjs):
        self.__prog = glCreateProgram()
        for shObj in shaderObjs: glAttachShader( self.__prog, shObj )
        glLinkProgram( self.__prog )
        result = glGetProgramiv( self.__prog, GL_LINK_STATUS )
        if not ( result ):
            print( 'link error:' )
            print( glGetProgramInfoLog( self.__prog ) )

class VAObject:
    def __init__( self, dataArrays, tetIndices ):
        self.__obj = glGenVertexArrays( 1 )
        self.__noOfIndices = len( tetIndices )
        self.__indexArr = numpy.array( tetIndices, dtype='uint' )
        noOfBuffers = len( dataArrays )
        buffers = glGenBuffers( noOfBuffers )
        glBindVertexArray( self.__obj )
        for i_buffer in range( 0, noOfBuffers ):
            vertexSize, dataArr = dataArrays[i_buffer]
            glBindBuffer( GL_ARRAY_BUFFER, buffers[i_buffer] )
            glBufferData( GL_ARRAY_BUFFER, numpy.array( dataArr, dtype='float32' ), GL_STATIC_DRAW )
            glEnableVertexAttribArray( i_buffer )
            glVertexAttribPointer( i_buffer, vertexSize, GL_FLOAT, GL_FALSE, 0, None )
    def Draw(self):
        glBindVertexArray( self.__obj )
        glDrawElements( GL_TRIANGLES, self.__noOfIndices, GL_UNSIGNED_INT, self.__indexArr )

    def Fract( self, val ): return val - math.trunc(val)
    def CalcAng( self, currentTime, intervall ): return self.Fract( (currentTime - self.__startTime) / intervall ) * 2.0 *     math.pi
    def CalcMove( self, currentTime, intervall, range ):
        pos = self.Fract( (currentTime - self.__startTime) / intervall ) * 2.0
        pos = pos if pos < 1.0 else (2.0-pos)
        return range[0] + (range[1] - range[0]) * pos
    def EllipticalPosition( self, a, b, angRag ):
        a_b = a * a - b * b
        ea = 0 if (a_b <= 0) else math.sqrt( a_b )
        eb = 0 if (a_b >= 0) else math.sqrt( -a_b )
        return ( a * math.sin( angRag ) - ea, b * math.cos( angRag ) - eb, 0 )
