İletki Yükleme ve Ayarlama (Windows’ta) #

Gereksinimler: İletki, kurulumdan önce aşağıdaki bağımlılıkların yüklenmesini gerektirir:

  • Java JDK 1.7 veya üstü
  • Node.js v4 veya üstü

Kurulum:
Node.js’yi şu URL’den indirin ve kurun: https://nodejs.org/en/

Node.js kurulumunun başarılı olup olmadığını görmek için Ortam değişkenlerine gidip kontrol edebilirsiniz. Sistem Değişkenleri altındaki ‘Yol’ otomatik olarak güncellenecektir.


Aynısını, size kurulu sürümü verecek olan komut istemine npm -version komutunu yazarak da kontrol edebilirsiniz.


Artık İletki iki şekilde kurulabilir: Yerel veya Küresel.

İletkiyi belirtilen bir klasöre veya proje dizini konumuna kurabiliriz. Bir proje dizinine kurarsak, her çalıştırdığımızda sadece o konumdan çalıştırmalıyız.

Proje dizinine yerel olarak yüklemek için proje klasörüne gidin ve komutu yazın

`npm kurulum iletki’


İletkiyi global olarak kurmak için şu komutu çalıştırın:

$ npm install -g protractor

Bu, iki komut satırı aracı, “iletki” ve “web sürücüsü-yöneticisi” kuracaktır. İletkinin başarıyla kurulduğundan emin olmak için “iletki –version” komutunu çalıştırın.

“webdriver-manager”, tarayıcı sürücüsü ikili dosyalarını indirmek ve selenyum sunucusunu başlatmak için kullanılır. Download the browser driver binaries with:

$ webdriver-manager update

Selenyum sunucusunu şununla başlatın:

$ webdriver-manager start

İnternet explorer sürücüsünü indirmek için, komut isteminde “webdriver-manager update –ie” komutunu çalıştırın. Bu, IEDriverServer.exe dosyasını Selenium klasörünüze indirecektir.

İletki kullanarak ilk test #

İletki, ilk testi, spec (test kodu) dosyasını ve yapılandırma dosyasını çalıştırmak için yalnızca iki dosyaya ihtiyaç duyar. Spesifikasyon dosyası test kodunu içerir ve diğeri spesifik dosya yolu, tarayıcı detayları, test url’si, çerçeve parametreleri vb. gibi yapılandırma detaylarını içerir. İlk testi yazmak için sadece Selenium sunucu adresini ve spesifik dosya yolunu sağlayacağız. Tarayıcı gibi diğer parametreler , zaman aşımı, çerçeve varsayılan değerlere alınacaktır.

İletki için varsayılan tarayıcı Chrome’dur.

conf.js - Yapılandırma dosyası

exports.config = {
  seleniumAddress: 'http://localhost:4444/wd/hub',
  specs: ['spec.js']
};

spec.js - Spec (test kodu) dosyası

describe('first test in protractor', function() {
  it('should verify title', function() {
    browser.get('https://angularjs.org');

    expect(browser.getTitle()).toEqual('AngularJS — Superheroic JavaScript MVW Framework');
  });
});

SeleniumAddress - Web sürücüsü sunucusunun çalıştığı sunucunun yolu .

özellikler - Test dosyalarının yolunu içeren bir dizi öğesi. Birden çok yol, virgülle ayrılmış değerlerle belirtilebilir.

açıklama - Jasmine çerçevesinden sözdizimi. ‘açıklama’ sözdizimi sta

Seçici Koşu Testleri #

İletki, tanım() yerine fdescribe() kullanarak test gruplarını seçerek çalıştırabilir.

fdescribe('first group',()=>{
    it('only this test will run',()=>{
        //code that will run
    });
});
describe('second group',()=>{
    it('this code will not run',()=>{
        //code that won't run
    });
});

İletki, it() yerine fit() kullanarak gruplar içinde testleri seçerek çalıştırabilir.

describe('first group',()=>{
    fit('only this test will run',()=>{
        //code that will run
    });
    it('this code will not run',()=>{
        //code that won't run
    });
});

Bir fdescribe() içinde fit() yoksa, o zaman her it() çalışır. Ancak, bir fit() aynı tanım() veya fdescribe() içindeki it() çağrılarını engeller.

fdescribe('first group',()=>{
    fit('only this test will run',()=>{
        //code that will run
    });
    it('this code will not run',()=>{
        //code that won't run
    });
});

Bir fit(), bir fdescribe() yerine bir açıklama() içinde olsa bile çalışır. Ayrıca, bir fdescribe() içinde fit() içermeyen herhangi bir it() çalışacaktır.

fdescribe('first group',()=>{
    it('this test will run',()=>{
        //code that will run
    });
    it('this test will also run',()=>{
        //code that will also run
    });
});
describe('second group',()=>{
    it('this code will not run',()=>{
        //code that won't run
    });
    fit('this code will run',(){
        //code that will run
    });
});

Bekleyen Testler #

İletki, testlerin beklemede olarak ayarlanmasına izin verir. Bu, iletkinin testi yürütmeyeceği, bunun yerine şunu çıkaracağı anlamına gelir:

Pending:
1) Test Name
Temporarily disabled with xit

Veya xdescribe() ile devre dışı bırakıldıysa:

Pending:
1) Test Name
No reason given

Kombinasyonlar #

  • Bir xdescribe() içindeki bir xit(), xit() yanıtını verir.
  • Bir fdescribe() içindeki bir xit(), hala beklemede olarak değerlendirilecektir.
  • Bir xdescribe() içindeki bir fit() çalışmaya devam edecek ve bekleyen hiçbir test herhangi bir sonuç vermeyecektir.

İletki: Kurumsal Açısal Uygulamalar için E2E Testi #

İletki Kurulumu ve Kurulumu

Adım 1: NodeJS’yi buradan indirin ve kurun. Düğümün en son sürümüne sahip olduğunuzdan emin olun. Burada v7.8.0 düğümünü kullanıyorum. Selenium’u çalıştırmak için Java Development Kit’in (JDK) kurulu olması gerekir.

Adım 2: Terminalinizi açın ve iletkiyi global olarak kurmak için aşağıdaki komutu yazın.

npm install -g protractor

Bu, iletki ve web sürücüsü yöneticisi gibi iki araç kuracaktır. İletki Kurulumunuzu şu komutu izleyerek doğrulayabilirsiniz: protractor –version. İletki başarılı bir şekilde kurulursa, sistem kurulu versiyonu gösterecektir.(yani Versiyon 5.1.1).Aksi takdirde kurulumu tekrar kontrol etmeniz gerekecektir. Adım 3: Gerekli ikili dosyaları indirmek için web sürücüsü yöneticisini güncelleyin.

webdriver-manager update

Adım 4: Aşağıdaki komut bir Selenium Sunucusunu başlatacaktır. Bu adım, web sürücüsü yöneticisini arka planda çalıştıracak ve iletki aracılığıyla yapılan testleri dinleyecektir.

web sürücüsü-yöneticisi başlangıcı Sunucunun durumuyla ilgili bilgileri adresinde görebilirsiniz. http://localhost:4444/wd/hub/static/resource/hub.html.

İletki kullanarak İlk Test vakasını yazma:

Test senaryosu yazmaya geçmeden önce, konfigürasyon dosyası ve spec dosyası olmak üzere iki dosya hazırlamalıyız.

Yapılandırma dosyasında:

//In conf.js
exports.config = {
    baseUrl: ‘http://localhost:8800/adminapp’,
    seleniumAddress: ‘http://localhost:4444/wd/hub',
    specs: [‘product/product_test.js’],
    directConnect : true,
    capabilities :{
        browserName: ‘chrome’
    }
}

Yapılandırma dosyasında kullanılan Terminolojilerin Temel Anlayışı:

baseUrl – Test edilen uygulamanız için bir temel URL.

SeleniumAddress – Halihazırda çalışmakta olan bir Selenium Sunucusuna bağlanmak için.

özellikler – Spesifikasyon dosyanızın konumu

directConnect : true – Doğrudan tarayıcı Sürücülerine bağlanmak için.

yetenekler – Tek bir tarayıcıda test yapıyorsanız, yetenekler seçeneğini kullanın. Birden çok tarayıcıda test yapıyorsanız multiCapabilities dizisini kullanın.

Buradan daha fazla yapılandırma seçeneği bulabilirsiniz. Tüm olası terminolojiyi tanımıyla birlikte tanımladılar.

Spec dosyasında:

//In product_test.js

    describe(‘Angular Enterprise Boilerplate’, function() {
      it('should have a title', function() {
        browser.get('http://localhost:8800/adminapp’);
        expect(browser.getTitle()).toEqual(‘Angular Enterprise Boilerplate’);
      });
    });

Spesifikasyon dosyasında kullanılan Terminolojilerin Temel Anlayışı:

Varsayılan olarak İletki, test arayüzü için yasemin çerçevesini kullanır. ‘açıkla’ ve ‘it’ sözdizimi yasemin çerçevesindendir. Buradan daha fazlasını öğrenebilirsiniz. İlk Test vakasını çalıştırma:

Test senaryosunu çalıştırmadan önce, web sürücüsü yöneticinizin ve uygulamanızın terminalinizin farklı sekmelerinde çalıştığından emin olun.

Şimdi, testi şununla çalıştırın:

Protractor app/conf.js

Chrome tarayıcının uygulamanızın url’si ile açıldığını ve kendini kapattığını görmelisiniz. Test çıktısı 1 test, 1 onaylama, 0 başarısızlık olmalıdır.
