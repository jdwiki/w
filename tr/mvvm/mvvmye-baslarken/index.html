C# MVVM Özeti ve Tam Örnek #

Özet:

MVVM, Model, Görünüm ve ViewModel olmak üzere üç farklı bileşenle temsil edilen bir mimari modeldir. Bu üç katmanı anlamak için, her birini kısaca tanımlamak ve ardından birlikte nasıl çalıştıklarına dair bir açıklama yapmak gerekir.

Model, iş mantığını yönlendiren katmandır. ViewModel tarafından tüketilmek üzere herhangi bir veri kaynağından bilgi alır ve depolar.

ViewModel, View ve Model arasında köprü görevi gören katmandır. Model‘deki ham verileri Görünüm için sunulabilir bir forma dönüştürebilir veya dönüştürmeyebilir. Örnek bir dönüşüm şöyle olabilir: modelden görünüm için ‘True’ veya ‘False’ dizesine bir boole bayrağı.

Görünüm, yazılımın (yani GUI’nin) arayüzünü temsil eden katmandır. Rolü, ViewModel‘den* gelen bilgileri kullanıcıya göstermek ve bilgilerdeki değişiklikleri ViewModel*’e* geri iletmektir.

Bu üç bileşen, aşağıdaki şekilde birbirine atıfta bulunarak birlikte çalışır:

  • Görünüm, ViewModel’e atıfta bulunur.
  • ViewModel, Model’e atıfta bulunur.

Görünüm ve ViewModel‘in Veri Bağlamaları olarak bilinen iki yönlü iletişim yeteneğine sahip olduğuna dikkat etmek önemlidir.

İki yönlü iletişim (veri bağlama) için önemli bir bileşen, INotifyPropertyChanged arabirimidir.

Bu mekanizmayı kullanarak, Görünüm, ViewModel içindeki verileri kullanıcı girişi yoluyla değiştirebilir ve ViewModel, Görünüm‘ü işlemler yoluyla güncellenmiş olabilecek verilerle güncelleyebilir. Model‘de veya depodan güncellenmiş verilerle.

MVVM mimarisi, bu katmanların her biri için Endişelerin Ayrılığına büyük önem verir. Katmanları ayırmak bize şu şekilde fayda sağlar:

  • Modülerlik: Her katmanın dahili uygulaması, diğerlerini etkilemeden değiştirilebilir veya değiştirilebilir.
  • Daha yüksek test edilebilirlik: Her katman sahte verilerle Birim Test Edilebilir; bu, ViewModel‘in kodu Görünüm‘ün Arkasındaki Kodda yazılıysa mümkün değildir. .

Yapı:

Yeni bir WPF Uygulaması projesi oluşturun

Çözümünüzde üç yeni klasör oluşturun: Model, ViewModel ve View ve yeni bir başlangıç ​​yapmak için orijinal “MainWindow.xaml” dosyasını silin.

Her biri ayrı bir katmana karşılık gelen üç yeni öğe oluşturun:

  • Model klasörüne sağ tıklayın ve adında bir Sınıf öğesi ekleyin. “HelloWorldModel.cs”.
  • ViewModel klasörüne sağ tıklayın ve bir Sınıf ekleyin ‘HelloWorldViewModel.cs’ adlı öğe.
  • Görünüm klasörüne sağ tıklayın ve HelloWorldView.xaml adlı bir Pencere (WPF) öğesi ekleyin.

“App.xaml” dosyasını yeni Görünüm’e işaret edecek şekilde değiştirin

<Application x:Class="MyMVVMProject.App"
             xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             xmlns:local="clr-namespace:MyMVVMProject"
             StartupUri="/View/HelloWorldView.xaml">
    <Application.Resources>
         
    </Application.Resources>
</Application>

Görünüm Modeli:

Önce ViewModel‘i oluşturmaya başlayın. Sınıf, “INotifyPropertyChanged” arabirimini uygulamalı, bir “PropertyChangedEventHandler” olayı bildirmeli ve olayı yükseltmek için bir yöntem oluşturmalıdır (kaynak: MSDN: Özellik Değişikliği Bildirimi Nasıl Uygulanır). Ardından, özelliğin “set” erişimcisinde “OnPropertyChanged()” yöntemini çağırdığınızdan emin olarak bir alan ve karşılık gelen bir özellik bildirin. Aşağıdaki örnekteki yapıcı, Model‘in ViewModel’e veri sağladığını göstermek için kullanılıyor.

using System;
using System.Collections.Generic;
using System.ComponentModel;
using System.Linq;
using System.Runtime.CompilerServices;
using System.Text;
using System.Threading.Tasks;
using MyMVVMProject.Model;

namespace MyMVVMProject.ViewModel
{
    // Implements INotifyPropertyChanged interface to support bindings
    public class HelloWorldViewModel : INotifyPropertyChanged
    {
        private string helloString;

        public event PropertyChangedEventHandler PropertyChanged;

        public string HelloString
        {
            get
            {
                return helloString;
            }
            set
            {
                helloString = value;
                OnPropertyChanged();
            }
        }

        /// <summary>
        /// Raises OnPropertychangedEvent when property changes
        /// </summary>
        /// <param name="name">String representing the property name</param>
        protected void OnPropertyChanged([CallerMemberName] string name = null)
        {
            PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(name));
        }

        public HelloWorldViewModel()
        {
            HelloWorldModel helloWorldModel = new HelloWorldModel();
            helloString = helloWorldModel.ImportantInfo;
        }
    }
}

Model:

Ardından, Model‘i oluşturun. Daha önce belirtildiği gibi, Model, ViewModel için onu bir havuzdan çekerek (aynı zamanda kaydetme için havuza geri iterek) veri sağlar. Bu, aşağıda basit bir ‘List’ döndürecek olan ‘GetData()’ yöntemiyle gösterilmiştir. Bu katmanda iş mantığı da uygulanır ve ConcatenateData() yönteminde görülebilir. Bu yöntem “Merhaba dünya!” cümlesini oluşturur. daha önce “depomuzdan” döndürülen Liste<string>den.

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace MyMVVMProject.Model
{
    public class HelloWorldModel
    {
        private List<string> repositoryData;
        public string ImportantInfo
        {
            get
            {
                return ConcatenateData(repositoryData);
            }
        }

        public HelloWorldModel()
        {
            repositoryData = GetData();
        }

        /// <summary>
        /// Simulates data retrieval from a repository
        /// </summary>
        /// <returns>List of strings</returns>
        private List<string> GetData()
        {
            repositoryData = new List<string>()
            {
                "Hello",
                "world"
            };
            return repositoryData;
        }

        /// <summary>
        /// Concatenate the information from the list into a fully formed sentence.
        /// </summary>
        /// <returns>A string</returns>
        private string ConcatenateData(List<string> dataList)
        {
            string importantInfo = dataList.ElementAt(0) + ", " + dataList.ElementAt(1) + "!";
            return importantInfo;
        }
    }
}

Görüş:

Son olarak, Görünüm oluşturulabilir. Bu örneğin arkasındaki koda eklenmesi gereken hiçbir şey yoktur, ancak bu, uygulamanın ihtiyaçlarına göre değişebilir. Ancak, XAML’ye eklenen birkaç satır var. “Pencere”, “ViewModel” ad alanına bir başvuruya ihtiyaç duyar. Bu, xmlns:vm="clr-namespace:MyMVVMProject.ViewModel" XAML ad alanıyla eşlenir. Ardından, Pencerenin bir “DataContext"e ihtiyacı vardır. Bu, “vm:HelloWorldViewModel/” olarak ayarlanmıştır. Artık etiket (veya seçtiğiniz kontrol) pencereye eklenebilir. Bu aşamadaki kilit nokta, Binding‘i etiket içeriği olarak görüntülemek istediğiniz ViewModel özelliğine ayarladığınızdan emin olmaktır. Bu durumda, {Binding HelloString} olur.

Alana değil, özelliğe bağlanmak önemlidir, ikinci durumda olduğu gibi Görünüm, değerin değiştiğine dair bildirim almaz, çünkü “OnPropertyChanged()” yöntemi yalnızca “PropertyChangedEvent” değerini yükseltecektir. arazide değil, arazide.

<Window x:Class="MyMVVMProject.View.HelloWorldView"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
        xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
        xmlns:local="clr-namespace:MyMVVMProject.View"
        xmlns:vm="clr-namespace:MyMVVMProject.ViewModel"
        mc:Ignorable="d"
        Title="HelloWorldView" Height="300" Width="300">
    <Window.DataContext>
        <vm:HelloWorldViewModel/>
    </Window.DataContext>
    <Grid>
        <Label x:Name="label" FontSize="30" Content="{Binding HelloString}" HorizontalAlignment="Center" VerticalAlignment="Center"/>
    </Grid>
</Window>
