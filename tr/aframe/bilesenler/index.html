Varlık-bileşen-sistem modelinde bir bileşen, görünüm, davranış ve/veya işlevsellik eklemek için bir varlığa bağladığımız yeniden kullanılabilir ve modüler bir veri yığınıdır.

A-Frame’de bileşenler, sahnedeki 3B nesneler olan varlıkları değiştirir. Karmaşık nesneler oluşturmak için bileşenleri karıştırıyor ve oluşturuyoruz. Üç.js ve JavaScript kodunu, HTML’den bildirimsel olarak kullanabileceğimiz modüllere yerleştirmemize izin veriyorlar. Bileşenler kabaca CSS’ye benzer.

Tanım Yaşam Döngüsü İşleyici Yöntemleri #

Şema anatomi olurken, yaşam döngüsü yöntemleri fizyolojidir; şema, verilerin şeklini, yaşam döngüsü işleyici yöntemlerini kullanımı tanımlar varlığı değiştirmek için veriler. İşleyiciler genellikle Varlık API’si.

YÖNTEMLER #

.init ()

.init (), bileşenin yaşam döngüsünün başlangıcında bir kez çağrılır. Bir varlık, bileşenin “init” işleyicisini çağırabilir:

  • Bileşen, HTML dosyasındaki varlık üzerinde statik olarak ayarlandığında ve sayfa yüklendiğinde.
  • Bileşen, ‘setAttribute’ aracılığıyla ekli bir varlıkta ayarlandığında.
  • Bileşen eklenmemiş bir varlıkta ayarlandığında ve varlık daha sonra appendChild ile sahneye eklenir.

‘init’ işleyicisi genellikle şunlar için kullanılır:

  • Başlangıç ​​durumunu ve değişkenleri ayarlayın
  • Bağlama yöntemleri
  • Olay dinleyicileri ekleyin

Örneğin, bir imleç bileşeninin “init"i durum değişkenlerini ayarlar, bind yöntemler ve olay dinleyicileri ekleyin:

AFRAME.registerComponent('cursor', {
  // ...
  init: function () {
    // Set up initial state and variables.
    this.intersection = null;
    // Bind methods.
    this.onIntersection = AFRAME.utils.bind(this.onIntersection, this);
    // Attach event listener.
    this.el.addEventListener('raycaster-intersection', this.onIntersection);
  }
  // ...
});

.güncelleme (eski Veri)

.update (oldData) bileşenin özellikleri değiştiğinde çağrılır, bileşenin yaşam döngüsünün başlangıcı dahil. Bir varlık, bir bileşenin “güncelleme” işleyicisi:

  • ‘init()’ çağrıldıktan sonra, bileşenin yaşam döngüsünün başında.
  • Bileşenin özellikleri .setAttribute ile güncellendiğinde.

“güncelleme” işleyicisi genellikle şunlar için kullanılır:

  • Varlıkta değişiklik yapma işinin çoğunu “this.data” kullanarak yapın.
  • Bir veya daha fazla bileşen özelliği değiştiğinde varlığı değiştirin.

Varlığa ilişkin ayrıntılı değişiklikler, mevcut öğeyi [farklaştırarak][farklaştırarak] yapılabilir. veri kümesi (this.data), güncellemeden önceki önceki veri kümesiyle (oldData).

A-Frame, hem bir bileşenin yaşam döngüsünün başlangıcında hem de her bir bileşenin verilerinin değiştiği zaman (ör. “setAttribute” sonucunda). bu güncelleme işleyicisi, varlığı değiştirmek için genellikle ’this.data’yı kullanır. güncelleme işleyicisi ilk argümanı aracılığıyla bir bileşenin verilerinin önceki durumuna erişimi vardır. Tam olarak hangisi olduğunu söylemek için bir bileşenin önceki verilerini kullanabiliriz. özellikler ayrıntılı güncellemeler yapmak için değiştirildi.

Örneğin, görünür bileşeninin “güncellemesi”, bileşenin görünürlüğünü ayarlar. varlık.

AFRAME.registerComponent('visible', {
  /**
   * this.el is the entity element.
   * this.el.object3D is the three.js object of the entity.
   * this.data is the component's property or properties.
   */
  update: function (oldData) {
    this.el.object3D.visible = this.data;
  }
  // ...
});

.kaldır ()

“.remove ()”, bileşen varlıktan ayrıldığında çağrılır. Bir varlık, bir bileşenin “kaldır” işleyicisini çağırabilir:

  • Bileşen, ‘removeAttribute’ aracılığıyla varlıktan kaldırıldığında.
  • Varlık olay yerinden ayrıldığında (ör. “removeChild”).

“Kaldır” işleyicisi genellikle şunlar için kullanılır:

  • Bileşenin varlık üzerindeki tüm değişikliklerini kaldırın, geri alın veya temizleyin.
  • Olay dinleyicilerini ayırın.

Örneğin, [ışık bileşeni][ışık] kaldırıldığında, ışık bileşeni daha önce varlığa yerleştirdiği hafif nesneyi kaldıracak, böylece olay yerinden uzaklaştırmak.

AFRAME.registerComponent('light', {
  // ...
  remove: function () {
    this.el.removeObject3D('light');
  }
  // ...
});

.tick (zaman, zamanDelta)

“.tick ()”, sahnenin oluşturma döngüsünün her bir onay işareti veya karesinde çağrılır. Sahne bir bileşenin “kene” işleyicisini çağırır:

  • Render döngüsünün her karesinde.
  • Saniyede 60 ila 120 kez mertebesinde.
  • Varlık veya sahne duraklatılmamışsa (örneğin, Müfettiş açıktır).
  • Varlık hala olay yerine bağlıysa.

“Tick” işleyicisi genellikle şunlar için kullanılır:

  • Her karede veya bir aralıkta varlığı sürekli olarak değiştirin.
  • Koşullar için anket.

‘Tick’ işleyicisi, sahnenin küresel çalışma süresini milisaniye cinsinden sağlar (zaman) ve son kareden bu yana milisaniye cinsinden zaman farkı (zamanDelta). Bunlar enterpolasyon için veya yalnızca belirli bir aralıkta “tick” işleyicisi.

Örneğin, izlenen kontroller bileşeni ilerleyecektir denetleyicinin animasyonlarını, denetleyicinin konumunu ve dönüşünü güncelleyin ve düğme basışlarını kontrol edin.

AFRAME.registerComponent('tracked-controls', {
  // ...
  tick: function (time, timeDelta) {
    this.updateMeshAnimation();
    this.updatePose();
    this.updateButtons();
  }
  // ...
});

.duraklat ()

Varlık veya sahne durakladığında .pause () çağrılır. varlık çağırabilir bileşenin “duraklatma” işleyicisi:

  • Bileşen kaldırılmadan önce, “kaldır” işleyicisi çağrılmadan önce.
  • Varlık Entity.pause () ile duraklatıldığında.
  • Sahne Scene.pause () ile duraklatıldığında (örneğin, Müfettiş açılır).

“Duraklatma” işleyicisi genellikle şunlar için kullanılır:

  • Olay dinleyicilerini kaldırın.
  • Dinamik davranış olasılığını ortadan kaldırın.

Örneğin, ses bileşeni sesi duraklatacak ve bir bir olayda ses çalan olay dinleyicisi:

AFRAME.registerComponent('sound', {
  // ...
  pause: function () {
    this.pauseSound();
    this.removeEventListener();
  }
  // ...
});

.oynat ()

Varlık veya sahne devam ettirildiğinde .play () çağrılır. varlık arayabilir bir bileşenin “oynat” işleyicisi:

  • Bileşen ilk eklendiğinde, ‘güncelleme’ işleyicisi çağrıldıktan sonra.
  • Varlık duraklatıldığında ancak ardından ‘Entity.play ()’ ile devam ettirildiğinde.
  • Sahne duraklatıldığında ancak ardından ‘Scene.play ()’ ile devam ettirildiğinde.

“oynat” işleyicisi genellikle şunları yapmak için kullanılır:

  • Olay dinleyicileri ekleyin.

Örneğin, ses bileşeni sesi çalacak ve sesi güncelleyecektir. bir olayda ses çalacak olay dinleyicisi:

AFRAME.registerComponent('sound', {
  // ...
  play: function () {
    if (this.data.autoplay) { this.playSound(); }
    this.updateEventListener();
  }
  // ...
});

.updateSchema (veri)

.updateSchema(), tanımlanmışsa, her güncellemede çağrılır. şema dinamik olarak değiştirilmelidir.

“updateSchema” işleyicisi genellikle şunlar için kullanılır:

  • Genellikle bir özelliğin değerine bağlı olarak şemayı dinamik olarak güncelleyin veya genişletin.

Örneğin, geometri bileşeni ‘ilkel’ olup olmadığını kontrol eder. özellik, farklı bir şema için şemanın güncellenip güncellenmeyeceğini belirlemek üzere değiştirildi. geometri türü:

AFRAME.registerComponent('geometry', {
  // ...
  updateSchema: (newData) {
    if (newData.primitive !== this.data.primitive) {
      this.extendSchema(GEOMETRIES[newData.primitive].schema);
    }
  }
  // ...
});

BİLEŞEN PROTOTİP YÖNTEMLERİ #

.flushToDOM()

Dizilemede CPU zamanından tasarruf etmek için A-Frame, yalnızca bileşenin gerçek DOM’deki serileştirilmiş gösterimini hata ayıklama modunda güncelleyecektir. FlushToDOM() öğesinin çağrılması, bileşenin verilerini manuel olarak seri hale getirecek ve DOM’yi güncelleyecektir:

document.querySelector('[geometry]').components.geometry.flushToDOM();

Özel bir A-Frame bileşeni kaydedin #

AFRAME.registerComponent (ad, tanım) #

Bir A-Frame bileşeni kaydedin. Bileşenleri içinde herhangi bir yerde kullanmadan önce kaydetmemiz gerekir. Bir HTML dosyasından, bileşenler öncesinde sırayla gelmelidir.

  • {string} adı - Bileşen adı. Bir HTML öznitelik adıyla temsil edilen bileşenin genel API’si.
  • {Nesne} tanımı - Bileşen tanımı. Şema ve yaşam döngüsü işleyici yöntemlerini içerir.

Bileşeni js dosyanızdaki foo dosyasına kaydetme, örneğin foo-component.js #

AFRAME.registerComponent('foo', {
  schema: {},
  init: function () {},
  update: function () {},
  tick: function () {},
  remove: function () {},
  pause: function () {},
  play: function () {}
});

Sahnenizde foo bileşeninin kullanımı #

<html>
  <head>
    <script src="aframe.min.js"></script>
    <script src="foo-component.js"></script>
  </head>
  <body>
    <a-scene>
      <a-entity foo></a-entity>
    </a-scene>
  </body>
</html>

Bileşen HTML Formu #

Bir bileşen, bir veya daha fazla bileşen özelliği biçiminde bir veri kovası tutar. Bileşenler, varlıkları değiştirmek için bu verileri kullanır. Bir motor bileşeni düşünün, beygir gücü veya silindir gibi özellikler tanımlayabiliriz.

HTML öznitelikleri, bileşen adlarını temsil eder ve bu özniteliklerin değeri, bileşen verilerini temsil eder.

Tek Özellikli Bileşen #

Bir bileşen tek özellikli bir bileşense, yani verileri tek bir değerden oluşuyorsa, HTML’de bileşen değeri normal bir HTML niteliği gibi görünür:

<!-- `position` is the name of the position component. -->
<!-- `1 2 3` is the data of the position component. -->
<a-entity position="1 2 3"></a-entity>

Çok Özellikli Bileşen #

Bir bileşen çok özellikli bir bileşense, yani veriler birden çok özellik ve değerden oluşuyorsa, HTML’de bileşen değeri satır içi CSS stillerine benzer:

<!-- `light` is the name of the light component. -->
<!-- The `type` property of the light is set to `point`. -->
<!-- The `color` property of the light is set to `crimson`. -->
<a-entity light="type: point; color: crimson"></a-entity>

Bileşen şema nesnesini tanımlama #

Şema, bileşenin özelliğini veya özelliklerini tanımlayan ve tanımlayan bir nesnedir. Şemanın anahtarları, özelliğin adlarıdır ve şemanın değerleri, özelliğin türlerini ve değerlerini tanımlar (çok özellikli bir bileşen olması durumunda):

Bileşeninizde şema tanımlama

AFRAME.registerComponent('bar', {
  schema: {
    color: {default: '#FFF'},
    size: {type: 'int', default: 5}
  }
}

Tanımlanmış şema varsayılanlarını geçersiz kıl

<a-scene>
  <a-entity bar="color: red; size: 20"></a-entity>
</a-scene>

Tek Nitelikli Şema #

Bir bileşen, tek özellikli bir bileşen olabilir (bir anonim değer) veya çok özellikli bir bileşen (birden çok adlandırılmış değerler). A-Frame, bir bileşenin tek özellikli mi yoksa bileşen mi olduğunu çıkaracaktır. şemanın yapısına dayalı çoklu özellik.

Tek özellikli bir bileşenin şeması, “tür” ve/veya “varsayılan” anahtarlar içerir ve şemanın değerleri nesneler yerine düz değerlerdir:

AFRAME.registerComponent('foo', {
  schema: {type: 'int', default: 5}
});

<a-scene>
  <a-entity foo="20"></a-entity>
</a-scene>

selektörTümü ‘querySelectorAll’ı çağırır ve ‘NodeList’i ‘Array’e dönüştürür (örneğin, ".boxes" [<a-entity class=“boxes”, …]), boş
dize Herhangi bir ayrıştırma yapmaz. '’
vec2 İki sayıyı bir “{x, y}” nesnesine ayrıştırır (ör. “1 -2” ile “{x: 1, y: -2}”). {x: 0, y: 0}
vec3 Üç sayıyı bir “{x, y, z}” nesnesine ayrıştırır (ör. “1 -2 3” ile “{x: 1, y: -2, z: 3}”). {x: 0, y: 0, z: 0}
vec4 Dört sayıyı bir “{x, y, z, w}” nesnesine ayrıştırır (ör. “1 -2 3 -4.5” ile “{x: 1, y: -2, z: 3, w: -4.5}”) {x: 0, y: 0, z: 0, w: 0}

Özellik Türü Çıkarımı #

Şema, yalnızca varsayılan bir değer verilen bir özellik türü çıkarmaya çalışacaktır:

schema: {default: 10}  // type: "number"
schema: {default: "foo"}  // type: "string"
schema: {default: [1, 2, 3]}  // type: "array"

Şema, sağlanmazsa, özellik türü göz önüne alındığında varsayılan bir değer ayarlayacaktır:

schema: {type: 'number'}  // default: 0
schema: {type: 'string'}  // default: ''
schema: {type: 'vec3'}  // default: {x: 0, y: 0, z: 0}

Özel Mülk Türü #

Ayrıca bir ‘parse’ sağlayarak kendi özellik türümüzü veya ayrıştırıcımızı tanımlayabiliriz. “tür” yerine işlev:

schema: {
  // Parse slash-delimited string to an array 
  // (e.g., `foo="myProperty: a/b"` to `['a', 'b']`).
  myProperty: {
    default: [],
    parse: function (value) {
      return value.split('/');
    }
  }
}

Bir Bileşenin Üyelerine ve Yöntemlerine Erişme #

Bir bileşenin üyelerine ve yöntemlerine, varlık aracılığıyla .components nesnesinden erişilebilir. Varlığın bileşen haritasından bileşeni arayın ve bileşenin dahili öğelerine erişebiliriz. Bu örnek bileşeni göz önünde bulundurun:

AFRAME.registerComponent('foo', {
  init: function () {
    this.bar = 'baz';
  },
  qux: function () {
    // ...
  }
});

bar üye ve qux yöntemine erişelim:

var fooComponent = document.querySelector('[foo]').components.foo;
console.log(fooComponent.bar);
fooComponent.qux();
