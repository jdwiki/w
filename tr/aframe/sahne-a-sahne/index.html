Bir sahne, “” öğesiyle temsil edilir. Sahne, genel kök nesnedir ve tüm varlıklar sahne içinde bulunur.

Sahne, Entity sınıfından miras aldığı için tüm özelliklerini, yöntemlerini, bileşen ekleme yeteneğini ve tüm alt düğümlerini (örneğin, “” ve “<a-” bekleme davranışını devralır. varlık>`) oluşturma döngüsünü başlatmadan önce yüklenecek.

Parametreler #

parametre Ayrıntılar
davranışlar Her karede çalıştırılacak kene yöntemlerine sahip bileşen dizisi.
kamera Aktif üç.js kamera.
tuval Tuval öğesine başvuru.
Mobil Ortamın mobil olduğunun tespit edilip edilmediği.
nesne3D ÜÇ. Sahne nesnesi.
işleyici Etkin THREE.WebGLRenderer.
renderBaşladı Sahnenin oluşturulup oluşturulmadığı.
etki Etkin oluşturucuyu THREE.VRffect’e geçirerek oluşturulan VR için oluşturucu.
sistemler Anlık sistemler.
zaman Saniye cinsinden sahnenin küresel çalışma süresi.

Sahne Bileşenlerini Ekleme #

Bileşenler, varlıkların yanı sıra sahneye de eklenebilir. A-Frame, sahneyi yapılandırmak için birkaç bileşenle birlikte gelir:

Bileşen Detaylar
gömülü Tuvalden tam ekran stillerini kaldırın.
olacak O verecek.
klavye kısayolları Klavye kısayollarını değiştirin.
müfettiş A-Frame Inspector’ı enjekte edin.
istatistikler Performans istatistiklerini aç/kapat.
vr-modu-ui VR’ye girmek ve çıkmak için kullanıcı arayüzünü değiştirin.
hata ayıklama Bileşenden DOM’a serileştirmeyi etkinleştirir.

Bunu kullanırken, akıllı telefonunuzun jiroskopunu kullanarak veya bilgisayarda tıklayıp sürükleyerek 360° içeriğin içine bakmak mümkündür.

Hata ayıklama #

Hata ayıklama bileşeni, bileşenden DOM’a serileştirmeyi etkinleştirir.

<a-scene debug></a-scene>

Bileşenden DOM’a Serileştirme #

Varsayılan olarak, performans nedenleriyle A-Frame, DOM’yi bileşen verileriyle güncellemez. Tarayıcının DOM denetçisini açarsak, yalnızca bileşen adlarının (değerlerin değil) görünür olduğunu göreceğiz.

<a-entity geometry material position rotation></a-entity>

A-Frame, bileşen verilerini bellekte saklar. DOM’nin güncellenmesi, dahili bileşen verilerini dizelere dönüştürmek için CPU zamanını alır. Hata ayıklama amacıyla DOM güncellemesini görmek istiyorsak, hata ayıklama bileşenini sahneye ekleyebiliriz. Bileşenler, DOM’a seri hale getirmeye çalışmadan önce etkinleştirilmiş bir hata ayıklama bileşeni olup olmadığını kontrol edecek. Ardından bileşen verilerini DOM’da görüntüleyebileceğiz:

<a-entity geometry="primitive: box" material="color: red" position="1 2 3" rotation="0 180 0"></a-entity>

Bu bileşenin üretimde etkin olmadığından emin olun.

DOM’a Manuel Olarak Serileştirme #

DOM’a manuel olarak seri hale getirmek için Entity.flushToDOM veya Component.flushToDOM kullanın:

document.querySelector('a-entity').components.position.flushToDOM();  // Flush a component.
document.querySelector('a-entity').flushToDOM();  // Flush an entity.
document.querySelector('a-entity').flushToDOM(true);  // Flush an entity and its children.
document.querySelector('a-scene').flushToDOM(true);  // Flush every entity.

İçerik Komut Dosyalarını Sahnede Çalıştırma #

Önerilen yol, bir bileşen yazmak ve onu sahne öğesine eklemektir. Sahne ve alt öğeleri, bu bileşenden önce başlatılacaktır.

AFRAME.registerComponent('do-something', {
  init: function () {
    var sceneEl = this.el;
  }
});

<a-scene do-something></a-scene>

Belirli bir nedenden dolayı özel bir bileşen yazmamayı tercih ederseniz, sahnenin başlatılmasını ve eklenmesini bitirmesini beklemeniz gerekir:

var scene = document.querySelector('a-scene');

if (scene.hasLoaded) {
  run();
} else {
  scene.addEventListener('loaded', run);
}

function run () {
  var entity = scene.querySelector('a-entity');
  entity.setAttribute('material', 'color', 'red');
}
