A-Frame, “” öğesi aracılığıyla bir varlığı temsil eder. Varlık-bileşen-sistem modelinde tanımlandığı gibi varlıklar, onlara görünüm, davranış ve işlevsellik sağlamak için bileşenleri taktığımız yer tutucu nesnelerdir.

Sözdizimi #

  • <a-entity> // Aşağıdaki varlığı göz önünde bulundurun. Kendi başına hiçbir görünümü, davranışı veya işlevi yoktur. Hiçbir şey yapmaz:
  • <a-entity geometri="primitive: box" malzeme="color: red"> // Bir şey oluşturmasını veya bir şey yapmasını sağlamak için ona bileşenler ekleyebiliriz. Ona şekil ve görünüm kazandırmak için geometri ve malzeme bileşenlerini ekleyebiliriz:
  • <a-entity geometri="primitive: box" malzeme="color: red" light="type: point; yoğunluk: 2,0"> // Veya ışık yaymasını sağlamak için, ışık bileşenini daha da ekleyebiliriz:

Parametreler #

parametre Ayrıntılar
bileşenler “.components”, varlığa eklenmiş bileşenlerin bir nesnesidir. Bu, her bileşenin verileri, durumu ve yöntemleri dahil olmak üzere varlığın bileşenlerine erişmemizi sağlar.
oynuyor Varlığın aktif olup olmadığı. Varlığı duraklatırsak, isPlaying false olur.
nesne3D “.object3D”, varlığın üç.js Object3D temsiline bir referanstır. Daha spesifik olarak, nesne3D, kameralar, ağlar, ışıklar veya sesler gibi farklı ‘ÜÇ.Nesne3D’ türleri içerebilen bir ‘ÜÇ.Grup’ nesnesi olacaktır:
object3DMap Bir varlığın object3DMap’i, bileşenlerin ayarladığı farklı ÜÇ.Object3D türlerine (örneğin kamera, kafesler, ışıklar, sesler) erişim sağlayan bir nesnedir.
sahneEl Bir varlığın, sahne öğesine bir referansı vardır.

YÖNTEMLER #

addState (stateName)

addState, varlığa bir durum gönderir. Bu, stateadded olayını yayar ve durum kutusunun varlığını .is kullanarak kontrol edebiliriz:

entity.addEventListener('stateadded', function (evt) {
  if (evt.detail.state === 'selected') {
    console.log('Entity now selected!');
  }
});
entity.addState('selected');
entity.is('selected');  // >> true

yayın (isim, ayrıntı, baloncuklar)

emit, varlık üzerinde özel bir DOM olayı yayar. Örneğin, bir animasyonu tetiklemek için bir olay yayınlayabiliriz:

<a-entity>
  <a-animation attribute="rotation" begin="rotate" to="0 360 0"></a-animation>
</a-entity>

entity.emit('rotate');

İkinci argüman olarak olay detayını veya verilerini de iletebiliriz:

entity.emit('collide', { target: collidingEntity });

Etkinlik varsayılan olarak kabarcıklanacaktır. bubble for bubble ileterek baloncuk yapmamasını söyleyebiliriz:

entity.emit('sink', null, false);

flushToDOM (özyinelemeli)

flushToDOM, bir varlığın bileşenlerinin verilerini manuel olarak seri hale getirecek ve DOM’yi güncelleyecektir.

getAttribute (bileşenAdı)

getAttribute ayrıştırılmış bileşen verilerini alır (karışımlar ve varsayılanlar dahil).

// <a-entity geometry="primitive: box; width: 3">
entity.getAttribute('geometry');
// >> {primitive: "box", depth: 2, height: 2, translate: "0 0 0", width: 3, ...}
entity.getAttribute('geometry').primitive;
// >> "box"
entity.getAttribute('geometry').height;
// >> 2
entity.getAttribute('position');
// >> {x: 0, y: 0, z: 0}

componentName kayıtlı bir bileşenin adı değilse, getAttribute normalde olduğu gibi davranır:

// <a-entity data-position="0 1 1">
entity.getAttribute('data-position');
// >> "0 1 1"

getDOMAttribute (bileşenAdı)

getDOMAttribute yalnızca DOM’de veya setAttribute aracılığıyla açıkça tanımlanan ayrıştırılmış bileşen verilerini alır. bileşenAdı kayıtlı bir bileşenin adıysa, getDOMAttribute yalnızca HTML’de tanımlanan bileşen verilerini ayrıştırılmış bir nesne olarak döndürür. Bileşenler için getDOMAttribute, döndürülen bileşen verileri uygulanan karışımları veya varsayılan değerleri içermediğinden, getAttribute öğesinin kısmi biçimidir:

Yukarıdaki getAttribute örneğinin çıktısını karşılaştırın:

// <a-entity geometry="primitive: box; width: 3">
entity.getDOMAttribute('geometry');
// >> { primitive: "box", width: 3 }
entity.getDOMAttribute('geometry').primitive;
// >> "box"
entity.getDOMAttribute('geometry').height;
// >> undefined
entity.getDOMAttribute('position');
// >> undefined

getObject3D (tür)

getObject3D, object3DMap üzerinde type tarafından başvurulan bir çocuk THREE.Object3D arar.

AFRAME.registerComponent('example-mesh', {
  init: function () {
     var el = this.el;
     el.getOrCreateObject3D('mesh', THREE.Mesh);
     el.getObject3D('mesh');  // Returns THREE.Mesh that was just created.
  }
});

getOrCreateObject3D (tür, Oluşturucu)

Varlık, type altında kayıtlı bir THREE.Object3D‘ye sahip değilse, getOrCreateObject3D, geçirilen Constructor kullanılarak somutlaştırılmış bir THREE.Object3D kaydeder. Varlığın type altında kayıtlı bir THREE.Object3D‘si varsa, getOrCreateObject3D, getObject3D olarak işlev görür:

AFRAME.registerComponent('example-geometry', {
  update: function () {
    var mesh = this.el.getOrCreateObject3D('mesh', THREE.Mesh);
    mesh.geometry = new THREE.Geometry();
  }
});

duraklat ()

pause(), animasyonlar ve bileşenler tarafından tanımlanan tüm dinamik davranışları durduracaktır. Bir varlığı duraklattığımızda, animasyonlarını durduracak ve bileşenlerinin her birinde Component.pause() öğesini çağıracaktır. Bileşenler, genellikle olay dinleyicilerini kaldıran duraklamada olanları uygulamaya karar verir. Bir varlığı duraklattığımızda, bir varlık, alt varlıklarında duraklama() öğesini çağırır.

// <a-entity id="spinning-jumping-ball">
entity.pause();

Örneğin, duraklamadaki görünüm kontrolleri bileşeni, girdiyi dinleyen olay işleyicilerini kaldıracaktır.

oynat ()

play(), animasyonlar ve bileşenler tarafından tanımlanan herhangi bir dinamik davranışı başlatır. Bu, DOM bir varlık eklediğinde otomatik olarak çağrılır. Bir varlık play() olduğunda, varlık, alt varlıklarında play() öğesini çağırır.

entity.pause();
entity.play();

Örneğin, çalınan ses bileşeni sesi çalmaya başlayacaktır.

setAttribute (bileşenAdı, değer, [propertyValue | clobber])

bileşenAdı kayıtlı bir bileşenin adı değilse veya bileşen tek özellikli bir bileşense, setAttribute normalde olduğu gibi davranır:

entity.setAttribute('visible', false);

bileşenAdı kayıtlı bir bileşenin adı olsa da, değer için özel ayrıştırma yapabilir. Örneğin, konum bileşeni tek özellikli bir bileşendir, ancak özellik türü ayrıştırıcısı bir nesne almasına izin verir:

entity.setAttribute('position', { x: 1, y: 2, z: 3 });

setObject3D (tür, nesne)

setObject3D, geçilen nesneyi, bir THREE.Object3D‘yi varlığın object3DMap altında type olarak kaydeder. A-Frame, nesne‘yi varlığın kökünün object3D alt öğesi olarak ekler.

AFRAME.registerComponent('example-orthogonal-camera', {
  update: function () {
    this.el.setObject3D('camera', new THREE.OrthogonalCamera());
  }
});

removeAttribute (bileşenAdı, özellikAdı)

bileşenAdı kayıtlı bir bileşenin adıysa, özniteliği DOM’den kaldırmanın yanı sıra removeAttribute ayrıca bileşeni varlıktan ayırarak bileşenin remove yaşam döngüsü yöntemini çağırır.

entity.removeAttribute('goemetry');  // Detach the geometry component.
entity.removeAttribute('sound');  // Detach the sound component.

propertyName verilirse, removeAttribute, o özelliğin propertyName tarafından belirtilen özellik değerini, özelliğin varsayılan değerine sıfırlar:

entity.setAttribute('material', 'color', 'blue');  // The color is blue.
entity.removeAttribute('material', 'color');  // Reset the color to the default value, white.

removeObject3D (tür)

removeObject3D, type tarafından belirtilen nesneyi varlığın THREE.Group‘undan ve dolayısıyla sahneden kaldırır. Bu, varlığın object3DMap‘ini günceller ve type anahtarının değerini null olarak ayarlar. Bu genellikle bir bileşenden, genellikle kaldırma işleyicisinden çağrılır:

AFRAME.registerComponent('example-light', {
  update: function () {
    this.el.setObject3D('light', new THREE.Light());
    // Light is now part of the scene.
    // object3DMap.light is now a THREE.Light() object.
  },
  remove: function () {
    this.el.removeObject3D('light');
    // Light is now removed from the scene.
    // object3DMap.light is now null.
  }
});

removeState (stateName)

removeState varlıktan bir durum çıkarır. Bu, stateremoved olayını yayacaktır ve .is kullanarak kaldırma durumunu kontrol edebiliriz:

entity.addEventListener('stateremoved', function (evt) {
  if (evt.detail.state === 'selected') {
    console.log('Entity no longer selected.');
  }
});
entity.addState('selected');
entity.is('selected');  // >> true
entity.removeState('selected');
entity.is('selected');  // >> false

Bileşen Değişikliklerini Dinleme #

Varlıktaki değişiklikleri dinlemek için componentchanged olayını kullanabiliriz:

entity.addEventListener('componentchanged', function (evt) {
  if (evt.detail.name === 'position') {
    console.log('Entity has moved from', 
      evt.detail.oldData, 'to', evt.detail.newData, '!');
  }
});

Bağlı ve Ayrılmış Alt Öğeleri Dinleme #

Sahnenin bir varlığı ne zaman eklediğini veya çıkardığını dinlemek için alt-bağlı ve alt-bağlı olaylarını kullanabiliriz:

entity.addEventListener('child-attached', function (evt) {
  if (evt.detail.el.tagName.toLowerCase() === 'a-box') {
    console.log('a box element has been attached');
  };
});

Varlık Çoklu Özellik Bileşen Verileri (setAttribute) #

Çok Özellikli Bileşen Verilerini Güncelleme #

Çok özellikli bir bileşen için bileşen verilerini güncellemek için, kayıtlı bir bileşenin adını bileşenAdı olarak ve bir özellik nesnesini değer olarak iletebiliriz. Bir dize de kabul edilebilir (örneğin, tür: nokta; uzaklık: 30), ancak nesneler A-Frame’i ayrıştırmada bazı çalışmalardan kurtaracaktır:

// Only the properties passed in the object will be overwritten.
entity.setAttribute('light', {
  type: 'spot',
  distance: 30,
  intensity: 2.0
});

Veya çok özellikli bir bileşenin özelliklerini tek tek güncellemek için, kayıtlı bileşenin adını bileşenAdı olarak, bir özellik adını ikinci bağımsız değişken olarak ve özellik değerini üçüncü bağımsız değişken olarak ayarlanacak şekilde iletebiliriz:

// All previous properties for the material component (besides the color)  will be unaffected.
entity.setAttribute('material', 'color', 'crimson');

Dizi özellik türlerinin benzersiz şekilde davrandığını unutmayın:

  • Diziler değişkendir. Dizilerdeki değişiklikler bileşen tarafından görülebilmesi için referansla atanırlar.
  • Dizi türü özelliklerinde yapılan güncellemeler, bileşenin güncelleme yöntemini tetiklemeyecek veya olay yaymayacaktır.

Çok Özellikli Bileşen Verilerini Güncelleme #

true, .setAttribute öğesine üçüncü bağımsız değişken olarak iletilirse, belirtilmeyen özellikler sıfırlanır ve engellenir:

// All previous properties for the light component will be removed and overwritten.
entity.setAttribute('light', {
  type: 'spot',
  distance: 30,
  intensity: 2.0
}, true);

Bir Varlığı Alma #

DOM API’lerini kullanarak bir varlığı kolayca alabiliriz.

<a-entity id="mario"></a-entity>

var el = document.querySelector('#mario');

Bir Varlık bileşenlerini alma #

Örneğin, bir varlığın üç.js kamera nesnesini veya malzeme nesnesini almak istersek, bileşenlerine ulaşabiliriz.

var camera = document.querySelector('a-entity[camera]').components.camera.camera;
var material = document.querySelector('a-entity[material]').components.material.material;

Veya bir bileşen bir API ortaya çıkarırsa, onun yöntemlerini çağırabiliriz:

document.querySelector('a-entity[sound]').components.sound.pause();
