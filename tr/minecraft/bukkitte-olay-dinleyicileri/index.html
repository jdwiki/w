Olay dinleyicisi oluşturma #

Yöntemlerinizi kaydetmek için EventHandler(lar)ı içeren sınıfın Listener arabirimini uygulaması gerekir.

import org.bukkit.event.Listener;

public final class ListenerClass implements Listener {
}

JavaPlugin’i genişleten sınıfta onEnable yönteminize aşağıdaki çağrıyı ekleyerek olay dinleyicisini kaydetmeniz gerekir:

getServer().getPluginManager().registerEvents(new ListenerClass(), this);

Listener sınıfınızdaki herhangi bir olayı dinlemek için, yöntem üzerinde @EventHandler açıklamalı bir yöntem oluşturmalısınız. Olay tipi, metodun tek argümanında Type ile belirtilir. Yöntem istediğiniz gibi adlandırılabilir.

import org.bukkit.event.Listener;
import org.bukkit.event.EventHandler;
import org.bukkit.event.player.PlayerLoginEvent;  

public class ListenerClass implements Listener {
    @EventHandler
    public void onPlayerLogin(PlayerLoginEvent event) {
        event.getPlayer().sendMessage("Welcome to the server!");
    }
}

EventHandler Parametreleri #

“org.bukkit.event.EventHandler” ek açıklaması birkaç parametreyi kabul eder.

öncelik - Dinleyicinizin önceliğini belirtir. Yürütme sırasına göre altı farklı öncelik vardır: DÜŞÜK,DÜŞÜK,NORMAL[varsayılan],YÜKSEK,YÜKSEK,MONİTÖR. Bu sabitler, “org.bukkit.event.EventPriority” numaralandırmasına başvurur.

Bir etkinliğin sonucunu değiştirmek istiyorsanız, DÜŞÜK’ten YÜKSEK’e çok dikkatli bir şekilde seçim yapın. EN DÜŞÜK’te önerilen genelleştirilmiş koruma eklentileri, NORMAL’de daha spesifik eklentiler ve YÜKSEK’te eklentileri geçersiz kılar. Bir olay olduğunda harekete geçmek istiyor, ancak sonucu değiştirmek istemiyorsanız, MONİTÖR’ü kullanın.

Not: MONİTÖR önceliği yalnızca yalnızca okuma için kullanılmalıdır. Bu öncelik, bir olayın sonuçlarını görmek için eklentilerin günlüğe kaydedilmesi için kullanışlıdır ve değerlerin değiştirilmesi bu tür eklentilere müdahale edebilir.

ignoreCancelled - Olayı ele alma sırası dinleyiciye gelmeden önce olay iptal edilirse dinleyicinizin tetiklenip tetiklenmeyeceğini belirten bir boole. Varsayılan olarak yanlış.

import org.bukkit.event.Listener;
import org.bukkit.event.EventHandler;
import org.bukkit.event.EventPriority;
import org.bukkit.event.player.PlayerLoginEvent;

public final class LoginListener implements Listener {
    @EventHandler
    public void normalLogin(PlayerLoginEvent event) {
        // Some code here
    }    

    @EventHandler(priority = EventPriority.HIGH)
    public void highLogin(PlayerLoginEvent event) {
        // Some code here
    }
}

Özel Etkinlikler Oluşturma #

Bazen diğer eklentilerin dinleyebileceği (diğer eklentilerin yanı sıra Apps Kasası bunu yapar) ve hatta iptal edebileceği kendi Etkinliğinizi oluşturmanız gerekir. Bukkit’in Event API’si bunun mümkün olmasını sağlar. Tek yapmanız gereken yeni bir sınıf oluşturmak, ‘Etkinliği’ genişletmesini sağlamak, işleyicileri ve etkinliğinizin ihtiyaç duyduğu nitelikleri (Oyuncu veya mesaj gibi) eklemek.

import org.bukkit.event.Event;
import org.bukkit.event.HandlerList;

public final class CustomEvent extends Event {
    private static final HandlerList handlers = new HandlerList();
    private String message;

    public CustomEvent(String example) {
        message = example;
    }

    public String getMessage() {
        return message;
    }

    public HandlerList getHandlers() {
        return handlers;
    }

    public static HandlerList getHandlerList() {
        return handlers;
    }
}

Özel Etkinliğinizi Çağırma #

Etkinliklerinizi oluşturmak ve çağırmak sizin elinizde, nerede çağıracağınız tamamen size kalmış. İşte bir örnek

// Create the event here
CustomEvent event = new CustomEvent("Sample Message");
// Call the event
Bukkit.getServer().getPluginManager().callEvent(event);
Bukkit.getServer().broadcastMessage(event.getMessage());

Unutmayın: Olaylarınızın kontrolü sizde. Onu çağırmaz ve ona göre hareket etmezsen, olmaz!

Özel Bir Etkinliği Dinleme #

Özel bir olayı dinlemek, normal bir olayı dinlemekle aynıdır.

import org.bukkit.event.Listener;
import org.bukkit.event.EventHandler;

public final class CustomListener implements Listener {
    
    @EventHandler
    public void onCustomEvent(CustomEvent event) {
    // Some code here
    }
}

CustomEvent’inizi İptal Edilebilir Hale Getirme #

Etkinliğinizi iptal edilebilir hale getirmek isterseniz, sadece ‘implements Cancellable’, ‘boolean iptal edildi’ ve bir alıcı ve ayarlayıcı ekleyin:

import org.bukkit.event.Event;
import org.bukkit.event.HandlerList;
import org.bukkit.event.Cancellable;

public final class CustomEvent extends Event implements Cancellable {
    private static final HandlerList handlers = new HandlerList();
    private String message;
    private boolean cancelled;

    public CustomEvent(String example) {
        message = example;
    }

    public String getMessage() {
        return message;
    }

    public boolean isCancelled() {
        return cancelled;
    }

    public void setCancelled(boolean cancel) {
        cancelled = cancel;
    }

    public HandlerList getHandlers() {
        return handlers;
    }

    public static HandlerList getHandlerList() {
        return handlers;
    }
}

Ardından, bir eklentinin özel etkinliği iptal edip etmediğini kontrol edersiniz.

// Create the event here
CustomEvent event = new CustomEvent("Sample Message");
// Call the event
Bukkit.getServer().getPluginManager().callEvent(event);
// Check if the event is not cancelled
if (!event.isCancelled()) {
    Bukkit.getServer().broadcastMessage(event.getMessage());
}

Olayların veya dinleyicilerin kaydını iptal etme #

Tek tek etkinliklerin, tüm dinleyici sınıflarının veya eklentiniz ve hatta diğer eklentiler tarafından kaydedilen tüm etkinliklerin kaydını iptal edebilirsiniz!

Belirli etkinliğin kaydını iptal et #

Her olay sınıfının getHandlerList() statik yöntemi vardır, bunu çağırın ve ardından .unregister() yöntemini kullanabilirsiniz.

PlayerInteractEvent.getHandlerList().unregister(plugin);
// this will unregister all PlayerInteractEvent instances from the plugin
// you can also specify a listener class instead of plugin.

Artık özel etkinliklerinizde neden getHandlerList()’e ihtiyacınız olduğunu biliyorsunuz.

Tüm etkinliklerin kaydını iptal et #

HandlerList sınıfını ve onun unregisterAll() statik yöntemini kullanarak, dinleyici sınıflarından veya eklentilerden olayların kaydını kolayca kaldırabilirsiniz.

HandlerList.unregisterAll(plugin);
// this will unregister all events from the specified plugin
// you can also specify a listener class instead of plugin.
