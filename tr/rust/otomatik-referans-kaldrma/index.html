Nokta operatörü #

Rust’taki . operatörü çok fazla sihirle geliyor! . kullandığınızda, derleyici, deref “ağacından” yöntemi bulmak için gerekli sayıda * (başvuru kaldırma işlemleri) ekler. Bu, derleme zamanında gerçekleştiğinden, yöntemi bulmanın çalışma zamanı maliyeti yoktur.

let mut name: String = "hello world".to_string();
// no deref happens here because push is defined in String itself
name.push('!');

let name_ref: &String = &name;
// Auto deref happens here to get to the String. See below
let name_len = name_ref.len();
// You can think of this as syntactic sugar for the following line:
let name_len2 = (*name_ref).len();

// Because of how the deref rules work,
// you can have an arbitrary number of references. 
// The . operator is clever enough to know what to do.
let name_len3 = (&&&&&&&&&&&&name).len();
assert_eq!(name_len3, name_len);

Otomatik referans kaldırma, std::ops::Deref özelliğini uygulayan her tür için de çalışır.

let vec = vec![1, 2, 3];
let iterator = vec.iter();

Burada ‘iter’, ‘Vec’ yöntemi değil, ‘[T]’ yöntemidir. ‘Vec’ ‘Deref’i ‘Hedef=[T]’ ile uyguladığı için çalışır, bu da ‘*’ operatörü tarafından başvuru yapıldığında ‘Vec‘nin ‘[T]‘ye dönüşmesine izin verir ( derleyicinin bir . sırasında ekleyebileceği).

Deref zorlamaları #

İki tür “T” ve “U” verildiğinde, “&T” yalnızca ve yalnızca “T” “Deref<Target=U>” uygularsa “&U"ya zorlar (dolaylı olarak dönüştürür)

Bu, aşağıdaki gibi şeyler yapmamızı sağlar:

fn foo(a: &[i32]) {
    // code
}

fn bar(s: &str) {
    // code
}

let v = vec![1, 2, 3];
foo(&v); // &Vec<i32> coerces into &[i32] because Vec<T> impls Deref<Target=[T]>

let s = "Hello world".to_string();
let rc = Rc::new(s);
// This works because Rc<T> impls Deref<Target=T> ∴ &Rc<String> coerces into 
// &String which coerces into &str. This happens as much as needed at compile time.
bar(&rc);

İşlev bağımsız değişkenleri için Deref ve AsRef kullanma #

Bir nesne koleksiyonu alması gereken işlevler için dilimler genellikle iyi bir seçimdir:

fn work_on_bytes(slice: &[u8]) {}

Çünkü Vec<T> ve diziler [T; N] Deref<Target=[T]> uygular, kolayca bir dilime zorlanabilirler:

let vec = Vec::new();
work_on_bytes(&vec);

let arr = [0; 10];
work_on_bytes(&arr);

let slice = &[1,2,3];
work_on_bytes(slice); // Note lack of &, since it doesn't need coercing

Ancak, açıkça bir dilim gerektirmek yerine, işlev, dilim olarak kullanılabilen herhangi bir türü kabul edecek şekilde yapılabilir:

fn work_on_bytes<T: AsRef<[u8]>>(input: T) {
    let slice = input.as_ref();
}

Bu örnekte, ‘work_on_bytes’ işlevi, ‘[u8]’ öğesine bir başvuru döndüren ‘as_ref()’ uygulayan herhangi bir ‘T’ türünü alacaktır.

work_on_bytes(vec);
work_on_bytes(arr);
work_on_bytes(slice);
work_on_bytes("strings work too!");
