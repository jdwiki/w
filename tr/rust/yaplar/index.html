Sözdizimi #

  • yapı Foo { field1: Type1, field2: Type2 }
  • foo = Foo { field1: Type1::new(), field2: Type2::new() };
  • yapı Çubuğu ( Type1, Type2 ); // demet türü
  • izin ver _ = Bar(Type1::new(), Type2::new());
  • Baz yapı; // birim benzeri tip
  • izin ver _ = Baz;
  • Foo { field1, .. } = foo; // alan1’i kalıp eşleştirme ile çıkar
  • izin ver Foo { field1: x, .. } = foo; // field1’i x olarak çıkar
  • let foo2 = Foo { field1: Type1::new(), .. foo }; // var olandan inşa et
  • impl Foo { fn fiddle(&self) {} } // Foo için örnek yöntemi bildir
  • impl Foo { fn tweak(&mut self) {} } // Foo için değişken örnek yöntemini bildir
  • impl Foo { fn double(self) {} } // Foo için örnek yöntemine sahip olun
  • impl Foo { fn yeni() {} } // Foo için ilişkili yöntemi bildir

Yapı değerleri oluşturma ve kullanma #

Aşağıdaki “yapı” tanımlarını göz önünde bulundurun:

struct Foo {
    my_bool: bool,
    my_num: isize,
    my_string: String,
}
struct Bar (bool, isize, String);
struct Baz;

Bu türler için yeni yapı değerleri oluşturmak basittir:

let foo = Foo { my_bool: true, my_num: 42, my_string: String::from("hello") };
let bar = Bar(true, 42, String::from("hello"));
let baz = Baz;

. kullanarak bir yapının alanlarına erişin:

assert_eq!(foo.my_bool, true);
assert_eq!(bar.0, true); // tuple structs act like tuples

Bir yapıya değiştirilebilen bir bağlama, alanlarının mutasyona uğramasına neden olabilir:

let mut foo = foo;
foo.my_bool = false;
let mut bar = bar;
bar.0 = false;

Rust’ın kalıp eşleştirme yetenekleri, bir “yapı"nın içine bakmak için de kullanılabilir:

// creates bindings mb, mn, ms with values of corresponding fields in foo
let Foo { my_bool: mb, my_num: mn, my_string: ms } = foo;
assert_eq!(mn, 42);
// .. allows you to skip fields you do not care about
let Foo { my_num: mn, .. } = foo;
assert_eq!(mn, 42);
// leave out `: variable` to bind a variable by its field name
let Foo { my_num, .. } = foo;
assert_eq!(my_num, 42);

Veya Rust’ın güncelleme sözdizimi: ikinci bir yapıyı “şablon” olarak kullanarak bir yapı yapın:

let foo2 = Foo { my_string: String::from("world"), .. foo };
assert_eq!(foo2.my_num, 42);

Yapıları tanımlama #

Rust’taki yapılar ‘struct’ anahtar sözcüğü kullanılarak tanımlanır. En yaygın yapı biçimi, bir dizi adlandırılmış alandan oluşur:

struct Foo {
    my_bool: bool,
    my_num: isize,
    my_string: String,
}

Yukarıdakiler, sırasıyla “bool”, “isize” ve “String” türlerinden “my_bool”, “my_num” ve “my_string” olmak üzere üç alana sahip bir “struct” bildirir.

Rust’ta struct oluşturmanın başka bir yolu da tuple struct: oluşturmaktır.

struct Bar (bool, isize, String);

Bu, sırayla “bool”, “isize” ve “String” türünde üç adsız alana sahip yeni bir tür “Bar” tanımlar. Bu, belirli bir tür için etkin bir şekilde yeni bir “ad” sağladığı için newtype modeli olarak bilinir. Ancak bunu, “type” anahtar sözcüğü kullanılarak oluşturulan takma adlardan daha güçlü bir şekilde yapar; Bar burada tamamen işlevsel bir tiptir, yani onun için kendi yöntemlerinizi yazabilirsiniz (aşağıda).

Son olarak, birim benzeri yapı: adı verilen no alanları olan bir ‘struct’ bildirin.

struct Baz;

Bu, alay etme veya test etme (bir özelliği önemsizce uygulamak istediğinizde) veya bir işaret türü olarak yararlı olabilir. Ancak genel olarak, pek çok birim benzeri yapılarla karşılaşmanız pek olası değildir.

Rust’taki “struct” alanlarının varsayılan olarak özel olduğunu unutmayın — yani, türü tanımlayan modülün dışındaki koddan bunlara erişilemez. O alanı herkese açık hale getirmek için bir alana “pub” anahtar sözcüğünü ön ek olarak koyabilirsiniz. Ek olarak, “struct” türü kendisi özeldir. Türün diğer modüllerde kullanılabilmesi için, “struct” tanımının önüne “pub” eklenmelidir:

pub struct X {
    my_field: bool,
    pub our_field: bool,
}

Genel yapılar #

Yapılar, bir veya daha fazla tip parametresi üzerinden jenerik yapılabilir. Bu türler, türe atıfta bulunulurken <> içinde verilmiştir:

struct Gen<T> {
    x: T,
    z: isize,
}

// ...
let _: Gen<bool> = Gen{x: true, z: 1};
let _: Gen<isize> = Gen{x: 42, z: 2};
let _: Gen<String> = Gen{x: String::from("hello"), z: 3};

Virgül kullanılarak birden çok tür verilebilir:

struct Gen2<T, U> {
    x: T,
    y: U,
}

// ...
let _: Gen2<bool, isize> = Gen2{x: true, y: 42};

Tür parametreleri türün bir parçasıdır, bu nedenle aynı temel türe sahip, ancak farklı parametrelere sahip iki değişken birbirinin yerine kullanılamaz:

let mut a: Gen<bool> = Gen{x: true, z: 1};
let b: Gen<isize> = Gen{x: 42, z: 2};
a = b; // this will not work, types are not the same
a.x = 42; // this will not work, the type of .x in a is bool

Tür parametre atamasından bağımsız olarak bir “yapı” kabul eden bir işlev yazmak istiyorsanız, bu işlevin de genel hale getirilmesi gerekir:

fn hello<T>(g: Gen<T>) {
    println!("{}", g.z); // valid, since g.z is always an isize
}

Peki ya her zaman “g.x” yazdırabilecek bir fonksiyon yazmak istersek? ‘T’yi görüntülenebilecek bir türde olacak şekilde kısıtlamamız gerekir. Bunu tür sınırlarıyla yapabiliriz:

use std::fmt;
fn hello<T: fmt::Display>(g: Gen<T>) {
    println!("{} {}", g.x, g.z);
}

“merhaba” işlevi artık yalnızca, “T” türü “fmt::Display” öğesini uygulayan “Gen” örnekleri için tanımlanmıştır. Örneğin bir Gen<(bool, isize)> iletmeyi deneseydik, derleyici bu tip için merhaba tanımlanmadığından şikayet ederdi.

Ayrıca, o “yapı"yı yalnızca belirli türler için oluşturabileceğinizi belirtmek için “yapı"nın tür parametrelerinde doğrudan tür sınırlarını kullanabiliriz:

use std:#️⃣:Hash;
struct GenB<T: Hash> {
    x: T,
}

Bir “GenB"ye erişimi olan herhangi bir işlev, artık “x” türünün “Hash” uyguladığını bilir ve böylece “.x.hash()” öğesini çağırabilir. Aynı parametre için çoklu tip sınırları, onları ‘+’ ile ayırarak verilebilir.

İşlevlerde olduğu gibi, tür sınırları “where” anahtar sözcüğü kullanılarak “<>“den sonra yerleştirilebilir:

struct GenB<T> where T: Hash {
    x: T,
}

Bu aynı anlamsal anlama sahiptir, ancak karmaşık sınırlarınız olduğunda imzanın okunmasını ve biçimlendirilmesini kolaylaştırabilir.

Tür parametreleri, “yapı"nın örnek yöntemleri ve ilişkili yöntemleri için de kullanılabilir:

// note the <T> parameter for the impl as well
// this is necessary to say that all the following methods only
// exist within the context of those type parameter assignments
impl<T> Gen<T> {
    fn inner(self) -> T {
        self.x
    }
    fn new(x: T) -> Gen<T> {
        Gen{x: x}
    }
}

“Gen"in “T"sinde tür sınırlarınız varsa, bunlar da “impl"nin tür sınırlarına yansıtılmalıdır. Ayrıca, belirli bir yöntemin yalnızca tür belirli bir özelliği karşılıyorsa var olduğunu söylemek için “impl” sınırlarını daha sıkı hale getirebilirsiniz:

impl<T: Hash + fmt::Display> Gen<T> {
    fn show(&self) {
        println!("{}", self.x);
    }
}

// ...
Gen{x: 42}.show(); // works fine
let a = Gen{x: (42, true)}; // ok, because (isize, bool): Hash
a.show(); // error: (isize, bool) does not implement fmt::Display

Yapı yöntemleri #

Bir yapı üzerinde yöntemler (yani, “yapı” üzerinde “çağrılabilen işlevler” veya “yapı” türünün değerleri) üzerinde yöntemler bildirmek için bir “impl” bloğu oluşturun:

impl Foo {
    fn fiddle(&self) {
        // "self" refers to the value this method is being called on
        println!("fiddling {}", self.my_string);
    }
}

// ...
foo.fiddle(); // prints "fiddling hello"

Buradaki “&self”, “fiddle” yöntemini çağırmak için “struct Foo” örneğinin değişmez bir referansının gerekli olduğunu gösterir. Örneği değiştirmek istersek (örneğin, alanlarından birini değiştirmek gibi), bunun yerine bir “&mut self” (yani, değiştirilebilir bir referans) alırdık:

impl Foo {
    fn tweak(&mut self, n: isize) {
        self.my_num = n;
    }
}

// ...
foo.tweak(43);
assert_eq!(foo.my_num, 43);

Son olarak, alıcı olarak “self"i de kullanabiliriz (”&“nin olmamasına dikkat edin). Bu, örneğin çağırana ait olmasını gerektirir ve yöntem çağrılırken örneğin taşınmasına neden olur. Bu, mevcut bir örneği tüketmek, yok etmek veya başka bir şekilde tamamen dönüştürmek istiyorsak faydalı olabilir. Böyle bir kullanım durumunun bir örneği, “zincirleme” yöntemleri sağlamaktır:

impl Foo {
    fn double(mut self) -> Self {
        self.my_num *= 2;
        self
    }
}

// ...
foo.my_num = 1;
assert_eq!(foo.double().double().my_num, 4);

Ayrıca, self’i tekrar döndürmeden önce mutasyona uğratabilmemiz için ‘self’i ‘mut’ ile ön eklediğimizi unutmayın. ‘Çift’ yöntemin dönüş türü de bazı açıklamaları garanti eder. Bir “impl” bloğu içindeki “Self”, “impl"nin geçerli olduğu türü ifade eder (bu durumda “Foo”). Burada, türün imzasını yeniden yazmaktan kaçınmak için çoğunlukla yararlı bir kısayoldur, ancak özelliklerde, belirli bir özelliği uygulayan temel türe atıfta bulunmak için kullanılabilir.

Bir “yapı” için ilişkili bir yöntem (diğer dillerde genellikle “sınıf yöntemi” olarak anılır) bildirmek için, yalnızca “self” argümanını dışarıda bırakın. Bu tür yöntemler, bir örneğinde değil, “yapı” türünün kendisinde çağrılır:

impl Foo {
    fn new(b: bool, n: isize, s: String) -> Foo {
        Foo { my_bool: b, my_num: n, my_string: s }
    }
}

// ...
// :: is used to access associated members of the type
let x = Foo::new(false, 0, String::from("nil"));
assert_eq!(x.my_num, 0);

Yapı yöntemlerinin yalnızca geçerli modülde bildirilen türler için tanımlanabileceğini unutmayın. Ayrıca, alanlarda olduğu gibi, tüm yapı yöntemleri varsayılan olarak özeldir ve bu nedenle yalnızca aynı modülde kod ile çağrılabilir. Tanımları başka bir yerden çağrılabilir kılmak için “pub” anahtar sözcüğüyle ön ek yapabilirsiniz.
