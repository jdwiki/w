Veritabanı tohumlama, veri, genel test verilerini veritabanınıza eklemenizi sağlar. Varsayılan olarak, “database/seeds” altında bir “DatabaseSeeder” sınıfı vardır.

Çalışan ekim makineleri ile yapılabilir

php artisan db:seed

Veya yalnızca tek bir sınıfı işlemek istiyorsanız

php artisan db:seed --class=TestSeederClass

Tüm artisan komutlarında olduğu gibi, api belgelerinde bulunabilecek çok çeşitli yöntemlere erişebilirsiniz.

Bir Ekme Makinesi Oluşturma #

Ekme makineleri oluşturmak için make:seder Artisan komutunu kullanabilirsiniz. Üretilen tüm ekiciler “veritabanı/tohumlar” dizinine yerleştirilecektir.

$ php artisan make:seeder MoviesTableSeeder

Oluşturulan ekme makineleri bir yöntem içerecektir: ‘run’. Bu yöntemde veri tabanınıza veri ekleyebilirsiniz.

<?php

use Illuminate\Database\Seeder;
use Illuminate\Database\Eloquent\Model;

class MoviesTableSeeder extends Seeder
{
    /**
     * Run the database seeds.
     *
     * @return void
     */
    public function run()
    {
        App\Movie::create([
            'name' => 'A New Hope',
            'year' => '1977'
        ]);

        App\Movie::create([
            'name' => 'The Empire Strikes Back',
            'year' => '1980'
        ]);
    }
}

Genelde tüm ekicilerinizi [DatabaseSeeder sınıfının içinde](https://www.wikiod.com/tr/laravel/tohumlama#Diğer ekicileri çağırma) aramak isteyeceksiniz.

Ekicileri yazmayı bitirdikten sonra db:seed komutunu kullanın. Bu, DatabaseSeederın çalıştır fonksiyonunu çalıştıracaktır.

$ php artisan db:seed

Ayrıca, “–class” seçeneğini kullanarak ayrı ayrı çalışacak belirli bir ekme sınıfını çalıştırmayı da belirtebilirsiniz.

$ php artisan db:seed --class=UserSeeder

Tüm geçişleri geri almak ve yeniden çalıştırmak ve ardından yeniden tohumlamak istiyorsanız:

$ php artisan migrate:refresh --seed

migrate:refresh --seed komutu şu 3 komutun kısayoludur:

$ php esnaf göçü:reset # tüm geçişleri geri al
$ php esnaf göç # çalıştırma göçleri
$ php artisan db:seed # ekme makineleri çalıştırın

Veri ekleme #

Veri eklemenin birkaç yolu vardır:

DB Cephesini Kullanma #

public function run()
{
    DB::table('users')
        ->insert([
            'name' => 'Taylor',
            'age'  => 21
        ]);
}

Bir Modelin Örneklenmesi Yoluyla #

public function run()
{
    $user = new User;
    $user->name = 'Taylor';
    $user->save();
}

Oluşturma yöntemini kullanma #

public function run()
{
    User::create([
        'name' => 'Taylor',
        'age'  => 21
    ]);
}

Fabrikayı kullanma #

public function run()
{
    factory(App\User::class, 10)->create();
}

Eski verilerin tohumlanması && silinmesi ve otomatik artışın sıfırlanması #

public function run()
{
    DB::table('users')->delete();
    DB::unprepared('ALTER TABLE users AUTO_INCREMENT=1;');
    factory(App\User::class, 200)->create();
}

Veri ekleme/güncelleme hakkında daha fazla bilgi için Persisting örneğine bakın.

Diğer ekicileri çağırma #

DatabaseSeeder sınıfınız içinde diğer ekicileri arayabilirsiniz.

$this->call(TestSeeder::class)

Bu, ekme makinelerinizi kolayca bulabileceğiniz bir dosya tutmanıza olanak tanır. Yabancı anahtar kısıtlamaları ile ilgili aramalarınızın sırasına dikkat etmeniz gerektiğini unutmayın. Henüz var olmayan bir tabloya başvuramazsınız.

Güvenli yeniden tohumlama #

Daha önce oluşturduğunuz tohumları etkilemeden veritabanınızı yeniden tohumlamak isteyebilirsiniz. Bunun için ekme makinenizde firstOrCreate kullanabilirsiniz:

EmployeeType::firstOrCreate([
    'type' => 'manager',
]);

Ardından veritabanını tohumlayabilirsiniz:

php artisan db:seed

Daha sonra, başka bir çalışan türü eklemek isterseniz, o yeni çalışanı aynı dosyaya ekleyebilirsiniz:

EmployeeType::firstOrCreate([
    'type' => 'manager',
]);
EmployeeType::firstOrCreate([
    'type' => 'secretary',
]);

Ve veritabanınızı sorunsuz bir şekilde yeniden tohumlayın:

php artisan db:seed

İlk aramada kaydı aldığınıza ancak onunla hiçbir şey yapmadığınıza dikkat edin.
