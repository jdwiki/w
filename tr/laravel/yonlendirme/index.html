Temel Yönlendirme #

Yönlendirme, bir tarafta HTTP yöntemleri ve URI’ler ve diğer tarafta eylemler arasında bir harita tanımlar. Rotalar normalde app/Http/routes.php dosyasına yazılır.

En basit biçiminde, bir rota, Route cephesinde karşılık gelen HTTP yöntemi çağrılarak, parametre olarak URI ile eşleşen bir dize (uygulama köküne göre) ve bir geri arama geçirilerek tanımlanır.

Örneğin: “ev” görünümünü döndüren sitenin kök URI’sine giden bir yol şöyle görünür:

Route::get('/', function() {
    return view('home');
});

Gönderi değişkenlerini basitçe yansıtan bir gönderi isteği için bir yol:

Route::post('submit', function() {
   return Input::all();
});

//or

Route::post('submit', function(\Illuminate\Http\Request $request) {
   return $request->all();
});

Bir Denetleyici yöntemine işaret eden yollar #

Geri aramayı satır içinde tanımlamak yerine, yol [ControllerClassName@Method] sözdizimindeki bir controller yöntemine başvurabilir:

Route::get('login', 'LoginController@index');

Birden çok fiil için bir yol #

Belirli bir rota için bir dizi HTTP yöntemini eşleştirmek için “match” yöntemi kullanılabilir:

Route::match(['GET', 'POST'], '/', 'LoginController@index');

Ayrıca, belirli bir rota için herhangi bir HTTP yöntemini eşleştirmek için ‘all’ öğesini kullanabilirsiniz:

Route::all('login', 'LoginController@index');

Rota Grupları #

Kod tekrarını önlemek için rotalar gruplandırılabilir.

Diyelim ki /admin ön ​​ekine sahip tüm URI’ler, ``adminadlı belirli bir ara yazılım kullanıyor ve hepsininApp\Http\Controllers\Admin``` ad alanında yaşıyorlar.

Rota Gruplarını kullanarak bunu temsil etmenin temiz bir yolu aşağıdaki gibidir:

Route::group([
    'namespace' => 'Admin',
    'middleware' => 'admin',
    'prefix' => 'admin'
    ], function () {
        
        // something.dev/admin
        // 'App\Http\Controllers\Admin\IndexController'
        // Uses admin middleware
        Route::get('/', ['uses' => 'IndexController@index']);    
        
        // something.dev/admin/logs
        // 'App\Http\Controllers\Admin\LogsController'
        // Uses admin middleware
        Route::get('/logs', ['uses' => 'LogsController@index']);    
});

Rotalar, bildirildikleri sıraya göre eşleştirilir #

Bu, Laravel rotaları ile ortak bir kazanımdır. Rotalar, bildirildikleri sırayla eşleştirilir. İlk eşleşen rota, kullanılan rotadır.

Bu örnek beklendiği gibi çalışacaktır:

Route::get('/posts/{postId}/comments/{commentId}', 'CommentController@show');
Route::get('/posts/{postId}', 'PostController@show');

‘/posts/1/comments/1’ için bir alma isteği, ‘CommentController@show’u çağırır. /posts/1 için bir alma isteği, PostController@showu çağıracaktır.

Ancak, bu örnek aynı şekilde çalışmayacaktır:

Route::get('/posts/{postId}', 'PostController@show');
Route::get('/posts/{postId}/comments/{commentId}', 'CommentController@show');

‘/posts/1/comments/1’ için bir alma isteği, ‘PostController@show’u çağırır. /posts/1 için bir alma isteği, PostController@showu çağıracaktır.

Laravel ilk eşleşen rotayı kullandığından, /posts/1/comments/1 isteği Route::get('/posts/{postId}', 'PostController@show'); ile eşleşir ve değişkenini atar. $postId, “1/comments/1” değerine dönüştürülür. Bu, ‘CommentController@show’un asla çağrılmayacağı anlamına gelir.

Adlandırılmış Rota #

Adlandırılmış yollar, bir URL oluşturmak veya belirli bir rotaya yönlendirmek için kullanılır. Adlandırılmış bir rota kullanmanın avantajı, gelecekte bir rotanın URI’sini değiştirirsek, adlandırılmış bir rota kullanıyorsak URL’yi veya o rotayı gösteren yönlendirmeleri değiştirmemize gerek kalmamasıdır. Ancak bağlantılar url [ör. url('/admin/login')], o zaman kullanıldığı her yeri değiştirmek zorunda kalırdık.

Adlandırılmış yollar “as” dizi anahtarı kullanılarak oluşturulur

Route::get('login', ['as' => 'loginPage', 'uses' => 'LoginController@index']);

veya “name” yöntemini kullanarak

Route::get('login', 'LoginController@index')->name('loginPage');

Adlandırılmış rotayı kullanarak URL oluştur #

Rotanın adını kullanarak bir url oluşturmak için

$url = route('loginPage');

Yeniden yönlendirme için rota adını kullanıyorsanız

$redirect = Redirect::route('loginPage');

Rota Parametreleri #

URI segmentinin bir kısmını almak için rota parametrelerini kullanabilirsiniz. Bir rota oluştururken isteğe bağlı veya gerekli bir rota parametresi/parametreleri tanımlayabilirsiniz. İsteğe bağlı parametrelerde, parametre adının sonuna bir “?” eklenir. Bu ad, küme parantezleri {} içine alınır

İsteğe Bağlı Parametre #

Route::get('profile/{id?}', ['as' => 'viewProfile', 'uses' => 'ProfileController@view']);

Bu rotaya domain.com/profile/23 adresinden erişilebilir, burada 23 id parametresidir. Bu örnekte, “id”, “ProfileController"ın “view” yönteminde bir parametre olarak iletilir. Bu isteğe bağlı bir parametre olduğundan domain.com/profile dosyasına erişmek gayet iyi çalışıyor.

Gerekli Parametre #

Route::get('profile/{id}', ['as' => 'viewProfile', 'uses' => 'ProfileController@view']);

Gerekli parametre adının sonunda parametre adının sonunda ? olmadığına dikkat edin.

Kontrolördeki parametreye erişim #

Denetleyicinizde, görüntüleme yönteminiz routes.php içindeki ile aynı ada sahip bir parametre alır ve normal bir değişken gibi kullanılabilir. Laravel, değeri enjekte etmekle ilgilenir:

public function view($id){
    echo $id;
}

Tüm rotaları yakala #

Tüm rotaları yakalamak istiyorsanız, gösterildiği gibi normal bir ifade kullanabilirsiniz:

Route::any('{catchall}', 'CatchAllController@handle')->where('catchall', '.*');

Önemli: Başka rotalarınız varsa ve her şeyi yakalamanın araya girmesini istemiyorsanız, onu sona koymalısınız. Örneğin:

Önceden tanımlanmış olanlar dışındaki tüm rotaları yakalamak #

Route::get('login',  'AuthController@login');
Route::get('logout', 'AuthController@logout');
Route::get('home',   'HomeController@home');

// The catch-all will match anything except the previous defined routes.
Route::any('{catchall}', 'CatchAllController@handle')->where('catchall', '.*');

Büyük/küçük harfe duyarsız rotalar #

Laravel’deki rotalar büyük/küçük harfe duyarlıdır. Bunun gibi bir rota anlamına gelir

Route::get('login', ...);

bir GET isteğini /login ile eşleştirecek, ancak bir GET isteğini /Login ile eşleştirmeyecektir.

Rotalarınızı büyük/küçük harfe duyarlı hale getirmek için, istenen URL’leri tanımlanmış rotalarla eşleştirecek yeni bir doğrulayıcı sınıfı oluşturmanız gerekir. Yeni doğrulayıcı ile mevcut olan arasındaki tek fark, derlenmiş yol için büyük/küçük harfe duyarlı olmayan eşleşmeyi etkinleştirmek için normal ifadenin sonuna i değiştiricisini eklemesidir.

<?php namespace Some\Namespace;

use Illuminate\Http\Request;
use Illuminate\Routing\Route;
use Illuminate\Routing\Matching\ValidatorInterface;

class CaseInsensitiveUriValidator implements ValidatorInterface
{
  public function matches(Route $route, Request $request)
  {
    $path = $request->path() == '/' ? '/' : '/'.$request->path();
    return preg_match(preg_replace('/$/','i', $route->getCompiled()->getRegex()), rawurldecode($path));
  }
}

Laravel’in yeni doğrulayıcınızı kullanması için, URL’yi bir rotayla eşleştirmek için kullanılan eşleştiriciler listesini güncellemeniz ve orijinal UriValidator’ı sizinkiyle değiştirmeniz gerekir.

Bunu yapmak için, route.php dosyanızın en üstüne aşağıdakini ekleyin:

<?php
use Illuminate\Routing\Route as IlluminateRoute;
use Your\Namespace\CaseInsensitiveUriValidator;
use Illuminate\Routing\Matching\UriValidator;

$validators = IlluminateRoute::getValidators();
$validators[] = new CaseInsensitiveUriValidator;
IlluminateRoute::$validators = array_filter($validators, function($validator) { 
  return get_class($validator) != UriValidator::class;
});

Bu, orijinal doğrulayıcıyı kaldıracak ve sizinkini doğrulayıcılar listesine ekleyecektir.
