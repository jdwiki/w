Veritabanı geçişi içinde #

Her taşıma bir “up()” yöntemine ve bir “down()” yöntemine sahip olmalıdır. ‘up()’ yönteminin amacı, veritabanı şemasını yeni durumuna getirmek için gerekli işlemleri gerçekleştirmektir ve ‘down()’ yönteminin amacı, ‘up()’ tarafından gerçekleştirilen tüm işlemleri tersine çevirmektir. yöntem. ‘down()’ yönteminin işlemlerinizi doğru şekilde tersine çevirmesini sağlamak, veritabanı şeması değişikliklerini geri alabilmek için çok önemlidir.

Örnek bir taşıma dosyası şöyle görünebilir:

<?php

use Illuminate\Database\Schema\Blueprint;
use Illuminate\Database\Migrations\Migration;

class AddLastLoggedInToUsersTable extends Migration
{
    /**
     * Run the migrations.
     *
     * @return void
     */
    public function up()
    {
        Schema::table('users', function (Blueprint $table) {
            $table->dateTime('last_logged_in')->nullable();
        });
    }

    /**
     * Reverse the migrations.
     *
     * @return void
     */
    public function down()
    {
        Schema::table('users', function (Blueprint $table) {
            $table->dropColumn('last_logged_in');
        });
    }
}

Bu geçişi çalıştırırken, Laravel, veritabanınıza karşı çalıştırmak için aşağıdaki SQL’i oluşturacaktır:

ALTER TABLE `users` ADD `last_logged_in` DATETIME NULL

Göçler #

Veritabanınızı Laravel’de kontrol etmek, geçişleri kullanmaktır. Artisan ile geçiş oluşturun:

php artisan make:migration create_first_table --create=first_table

Bu, CreateFirstTable sınıfını oluşturacaktır. up yönteminin içinde sütunlarınızı oluşturabilirsiniz:

<?php

use Illuminate\Database\Schema\Blueprint;
use Illuminate\Database\Migrations\Migration;

class CreateFirstTable extends Migration
{
    public function up()
    {
        Schema::create('first_table', function (Blueprint $table) {
            $table->increments('id');
            $table->string('first_string_column_name');
            $table->integer('secont_integer_column_name');
            $table->timestamps();
        });
    }

    public function down()
    {
        Schema::drop('first_table');
    }
}

Sonunda tüm geçiş sınıflarınızı çalıştırmak için artisan komutunu çalıştırabilirsiniz:

php artisan migrate

Bu, tablolarınızı ve sütunlarınızı veritabanınızda oluşturacaktır. Diğer faydalı geçiş komutu şunlardır:

  • php artisan move:rollback - Son veritabanı geçişini geri alma
  • php esnaf göçü:sıfırla - Tüm veritabanı geçişlerini geri al
  • php artisan move:refresh - Tüm geçişleri sıfırlayın ve yeniden çalıştırın
  • php esnaf göçü:durum - Her geçişin durumunu göster

Mevcut tabloları değiştirme

Bazen, sütunları yeniden adlandırma/silme gibi mevcut tablo yapınızı değiştirmeniz gerekir. Bunu yeni bir göç oluşturarak başarabilirsiniz. Ve geçişinizin ‘yukarı’ yönteminde.

//Renaming Column.

public function up()
{
    Schema::table('users', function (Blueprint $table) {
        $table->renameColumn('email', 'username');
    });
}

Yukarıdaki örnek, “kullanıcılar tablosu"nun “e-posta sütununu” “kullanıcı adı” olarak yeniden adlandıracaktır. Aşağıdaki kod, “users” tablosundan bir “username” sütununu düşürürken.

ÖNEMLİ : Sütunları değiştirmek için projenin ‘composer.json’ dosyasına ‘doktrin/dbal’ bağımlılığı eklemeniz ve değişiklikleri yansıtmak için ‘besteci güncellemesini’ çalıştırmanız gerekir.

//Droping Column
public function up()
{
    Schema::table('users', function (Blueprint $table) {
        $table->dropColumn('username');
    });
}

Taşıma dosyaları oluşturma #

Şemanızı her değiştirmeniz gerektiğinde doğru dosya adıyla yeni bir geçiş dosyası oluşturmak bir angarya olacaktır. Neyse ki, Laravel’in ‘artisan’ komutu sizin için geçişi oluşturabilir:

php artisan make:migration add_last_logged_in_to_users_table

Yukarıdaki komutla --table ve --create bayraklarını da kullanabilirsiniz. Bunlar isteğe bağlıdır ve yalnızca kolaylık sağlamak içindir ve ilgili ortak kod kodunu geçiş dosyasına ekleyecektir.

php artisan make:migration add_last_logged_in_to_users_table --table=users

php artisan make:migration create_logs_table --create=logs

“–path” seçeneğini kullanarak oluşturulan taşıma için özel bir çıktı yolu belirleyebilirsiniz. Yol, uygulamanın temel yoluna göredir.

php artisan make:migration --path=app/Modules/User/Migrations

Taşıma dosyaları #

Bir Laravel 5 uygulamasındaki geçişler, ‘database/migrations’ dizininde yaşar. Dosya adları belirli bir biçime uygundur:

<year>_<month>_<day>_<hour><minute><second>_<name>.php

Bir geçiş dosyası, belirli bir sorunu çözmek için bir şema güncellemesini temsil etmelidir. Örneğin:

2016_07_21_134310_add_last_logged_in_to_users_table.php

Veritabanı geçişleri, Laravel’in bunları hangi sırayla yürüteceğini bilmesi için kronolojik sırada tutulur. Laravel, her zaman en eskiden en yeniye geçişleri gerçekleştirir.

Taşıma işlemlerini çalıştırma #

Geçişiniz yazıldıktan sonra, onu çalıştırmak, işlemleri veritabanınıza uygulayacaktır.

php artisan migrate

Her şey yolunda giderse, aşağıdakine benzer bir çıktı göreceksiniz:

Migrated: 2016_07_21_134310_add_last_logged_in_to_users_table

Laravel, üretim ortamında geçişleri ne zaman çalıştırdığınızı bilecek kadar akıllıdır. Yıkıcı bir geçiş yaptığınızı algılarsa (örneğin, bir tablodan bir sütunu kaldıran), php artisan move komutu sizden onay isteyecektir. Sürekli teslim ortamlarında bu istenmeyebilir. Bu durumda, onayı atlamak için “–force” işaretini kullanın:

php artisan migrate --force

Yalnızca geçişleri çalıştırdıysanız, veritabanınızda bir “geçişler” tablosunun varlığını görmek için kafanız karışabilir. Bu tablo, Laravel’in hangi geçişlerin halihazırda çalıştırıldığını takip etmek için kullandığı tablodur. ‘migrate’ komutunu verirken, Laravel hangi göçlerin henüz çalışmadığını belirleyecek ve ardından bunları kronolojik sırayla yürütecek ve ardından ‘migrations’ tablosunu uygun şekilde güncelleyecektir.

Ne yaptığınızı kesinlikle bilmiyorsanız, “göçler” tablosunu asla manuel olarak düzenlememelisiniz. Veritabanınızı yanlışlıkla, geçişlerinizin başarısız olacağı bozuk bir durumda bırakmak çok kolaydır.

Taşıma İşlemlerini Geri Alma #

En son geçişi, yani son işlemi geri almak istiyorsanız, harika ‘geri alma’ komutunu kullanabilirsiniz. Ancak bu komutun yalnızca birden çok taşıma dosyası içerebilen son taşımayı geri aldığını unutmayın.

php artisan migrate:rollback

Tüm uygulama geçişlerinizi geri almakla ilgileniyorsanız, aşağıdaki komutu kullanabilirsiniz.

php artisan migrate:reset

Üstelik benim gibi tembelseniz ve tek komutla geri almak ve geçiş yapmak istiyorsanız, bu komutu kullanabilirsiniz.

php artisan migrate:refresh
php artisan migrate:refresh --seed

Ayrıca step seçeneği ile geri alınacak adım sayısını da belirleyebilirsiniz. Bunun gibi 1 adım geri alınacak.

php artisan migrate:rollback --step=1
