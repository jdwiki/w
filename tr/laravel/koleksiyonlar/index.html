Sözdizimi #

  • $koleksiyon = toplama([‘Değer1’, ‘Değer2’, ‘Değer3’]); // Tuşlar varsayılan olarak 0, 1, 2, …,

Illuminate\Support\Collection veri dizileriyle başa çıkmak için akıcı ve kullanışlı bir arayüz sağlar. Bunları bilmeden de kullanmış olabilirsiniz, örneğin, birden fazla kayıt getiren Model sorguları, bir ‘Illuminate\Support\Collection’ örneğini döndürür.

Koleksiyonlarla ilgili güncel belgeler için resmi belgeleri burada bulabilirsiniz.

Koleksiyon Oluşturma #

‘collect()’ yardımcısını kullanarak, aşağıdaki gibi bir diziyi ileterek kolayca yeni koleksiyon örnekleri oluşturabilirsiniz:

$fruits = collect(['oranges', 'peaches', 'pears']);

Yardımcı işlevleri kullanmak istemiyorsanız, doğrudan sınıfı kullanarak yeni bir Koleksiyon oluşturabilirsiniz:

$fruits = new Illuminate\Support\Collection(['oranges', 'peaches', 'pears']);

Açıklamalarda belirtildiği gibi, Modeller varsayılan olarak bir ‘Koleksiyon’ örneği döndürür, ancak gerektiğinde kendi koleksiyonlarınızı oluşturmakta özgürsünüz. Oluşturma sırasında herhangi bir dizi belirtilmezse boş bir Koleksiyon oluşturulur.

nerede() #

where() yöntemini kullanarak bir koleksiyondan belirli öğeleri seçebilirsiniz.

$data = [
    ['name' => 'Taylor',  'coffee_drinker' => true],
    ['name' => 'Matt', 'coffee_drinker' => true]
];

$matt = collect($data)->where('name', 'Matt');

Bu kod parçası, adın ‘Matt’ olduğu koleksiyondaki tüm öğeleri seçecektir. Bu durumda, yalnızca ikinci öğe döndürülür.

Yuvalama #

Laravel’deki çoğu dizi yöntemi gibi, “where()” de iç içe öğelerin aranmasını destekler. İkinci bir dizi ekleyerek yukarıdaki örneği genişletelim:

$data = [
    ['name' => 'Taylor',  'coffee_drinker' => ['at_work' => true, 'at_home' => true]],
    ['name' => 'Matt', 'coffee_drinker' => ['at_work' => true, 'at_home' => false]]
];

$coffeeDrinkerAtHome = collect($data)->where('coffee_drinker.at_home', true);

Bu sadece evde kahve içtiği için Taylor’ı geri getirecektir. Gördüğünüz gibi, nokta gösterimi kullanılarak yuvalama desteklenir.

Eklemeler #

Diziler yerine bir nesne koleksiyonu oluştururken, bunlar “where()” kullanılarak da filtrelenebilir. Koleksiyon daha sonra istenen tüm özellikleri almaya çalışacaktır.

Lütfen, Laravel 5.3’ten bu yana, where() yönteminin, varsayılan olarak değerleri gevşek bir şekilde karşılaştırmaya çalışacağını unutmayın. Bu, (int)1 aranırken, 1 içeren tüm girişlerin de döndürüleceği anlamına gelir. Bu davranışı beğenmiyorsanız, whereStrict() yöntemini kullanabilirsiniz.

Bir koleksiyondan belirli değerleri çıkarmak için Pluck’ı kullanma #

Kendinizi genellikle verilerin yalnızca bölümleriyle ilgilendiğiniz bir veri koleksiyonuyla bulacaksınız.

Aşağıdaki örnekte, bir etkinliğe katılanların bir listesini aldık ve basit bir isim listesi içeren bir tur rehberi sağlamak istiyoruz.

// First we collect the participants
$participants = collect([
    ['name' => 'John', 'age' => 55],
    ['name' => 'Melissa', 'age' => 18],
    ['name' => 'Bob', 'age' => 43],
    ['name' => 'Sara', 'age' => 18],
]);

// Then we ask the collection to fetch all the names
$namesList = $partcipants->pluck('name')
// ['John', 'Melissa', 'Bob', 'Sara'];

Nokta gösterimli nesne koleksiyonları veya iç içe diziler/nesneler için de “puck” kullanabilirsiniz.

$users = User::all(); // Returns Eloquent Collection of all users
$usernames = $users->pluck('username'); // Collection contains only user names

$users->load('profile'); // Load a relationship for all models in collection

// Using dot notation, we can traverse nested properties
$names = $users->pluck('profile.first_name'); // Get all first names from all user profiles

Koleksiyonları genişletmek için makro() kullanma #

“makro()” işlevi, “Illuminate\Support\Collection” nesnelerine yeni işlevler eklemenize olanak tanır.

Kullanım:

Collection::macro("macro_name", function ($parameters) {
    // Your macro
});

Örneğin:

Collection::macro('uppercase', function () {
    return $this->map(function ($item) {
        return strtoupper($item);
    });
});

collect(["hello", "world"])->uppercase();

Sonuç: ["MERHABA", "DÜNYA"]

Dizi Sözdizimini Kullanma #

‘Collection’ nesnesi, ‘ArrayAccess’ ve ‘IteratorAggregate’ arabirimini uygulayarak bir dizi gibi kullanılmasına izin verir.

Erişim toplama öğesi:

 $collection = collect([1, 2, 3]);
 $result = $collection[1];

Sonuç: “2”

Yeni öğe ata:

$collection = collect([1, 2, 3]);
$collection[] = 4;

Sonuç: “$collection”, “[1, 2, 3, 4]” şeklindedir.

Döngü koleksiyonu:

$collection = collect(["a" => "one", "b" => "two"]);
$result = "";
foreach($collection as $key => $value){
    $result .= "(".$key.": ".$value.") ";        
}

Sonuç: “$sonuç”, “(a: bir) (b: iki)“dir

** Diziden Koleksiyona dönüştürme:**

Bir koleksiyonu yerel bir PHP dizisine dönüştürmek için şunu kullanın:

$array = $collection->all();
//or
$array = $collection->toArray()

Bir diziyi koleksiyona dönüştürmek için şunu kullanın:

$collection = collect($array);

Koleksiyonları Dizi İşlevleriyle Kullanma

Koleksiyonların, ‘array_map($callback)’ gibi açıkça diziler gerektiren işlevler tarafından kullanıldığında düzgün şekilde dönüştürülmeyecek normal nesneler olduğunu lütfen unutmayın.

Önce koleksiyonu dönüştürdüğünüzden emin olun veya varsa, bunun yerine ‘Collection’ sınıfı tarafından sağlanan yöntemi kullanın: ‘$collection->map($callback)’

Bir koleksiyonun belirli bir koşulu karşılayıp karşılamadığını kontrol etmek için İçerir’i kullanma #

Yaygın bir sorun, hepsinin belirli bir kriteri karşılaması gereken bir öğe koleksiyonuna sahip olmaktır. Aşağıdaki örnekte bir diyet planı için iki madde topladık ve diyetin sağlıksız yiyecek içerip içermediğini kontrol etmek istiyoruz.

// First we create a collection
$diet = collect([
    ['name' => 'Banana', 'calories' => '89'],
    ['name' => 'Chocolate', 'calories' => '546']
]);

// Then we check the collection for items with more than 100 calories
$isUnhealthy = $diet->contains(function ($i, $snack) {
    return $snack["calories"] >= 100;
});

Yukarıdaki durumda, Çikolata koşulu karşıladığı için “$sağlıksız” değişkeni “doğru” olarak ayarlanır ve bu nedenle diyet sağlıksızdır.

İstatistiksel hesaplamalar için bir koleksiyonda toplam, ortalama, minimum veya maksimum kullanma #

Koleksiyonlar ayrıca size basit istatistiksel hesaplamalar yapmanın kolay bir yolunu sunar.

$books = [
    ['title' => 'The Pragmatic Programmer', 'price' => 20],
    ['title' => 'Continuous Delivery', 'price' => 30],
    ['title' => 'The Clean Coder', 'price' => 10],
]

$min = collect($books)->min('price'); // 10
$max = collect($books)->max('price'); // 30
$avg = collect($books)->avg('price'); // 20
$sum = collect($books)->sum('price'); // 60

azaltma() kullanma #

“Azaltma” yöntemi, her yinelemenin sonucunu sonraki yinelemeye ileterek, koleksiyonu tek bir değere indirger. Lütfen azaltma yöntemi bölümüne bakın.

`````````` yöntemi, bir koleksiyonla her öğe arasında döngü yapar ve bir sonraki yineleme için yeni bir sonuç üretir. Son yinelemenin her sonucu ilk parametreden geçirilir (aşağıdaki örneklerde $carry olarak).

Bu yöntem, büyük veri kümeleri üzerinde çok fazla işlem yapabilir. Örneğin, aşağıdaki örnekler için, aşağıdaki örnek öğrenci verilerini kullanacağız:

 $student = [
    ['class' => 'Math', 'score' => 60],
    ['class' => 'English', 'score' => 61],
    ['class' => 'Chemistry', 'score' => 50],
    ['class' => 'Physics', 'score' => 49],
];

Öğrencinin toplam puanının toplamı

$sum = collect($student)
    ->reduce(function($carry, $item){
        return $carry + $item["score"];
    }, 0);

Sonuç: 220

Açıklama:

  • ``$carry``` son yinelemenin sonucudur.
  • İkinci parametre, yinelemenin ilk turunda $carry için varsayılan değerdir. Bu durumda, varsayılan değer 0’dır.

Tüm puanları >= 50 ise bir öğrenciyi geçin

$isPass = collect($student)
    ->reduce(function($carry, $item){
        return $carry && $item["score"] >= 50;
    }, true);

Sonuç: yanlış

Açıklama:

  • $carry’nin varsayılan değeri doğrudur
  • Tüm puanlar 50’den büyükse, sonuç doğru olacaktır; 50’den küçükse false döndürün.

Herhangi bir puan <50 ise öğrenci başarısız

$isFail = collect($student)
    ->reduce(function($carry, $item){
        return $carry || $item["score"] < 50;
    }, false);

Sonuç: doğru

Açıklamak:

  • $carry’nin varsayılan değeri yanlıştır
  • herhangi bir puan 50’den azsa, doğru döndür; tüm puanlar 50’den büyükse, false döndürün.

En yüksek puana sahip konuyu döndür

$highestSubject = collect($student)
    ->reduce(function($carry, $item){
        return $carry === null || $item["score"] > $carry["score"] ? $item : $carry;
    });

sonuç: [ "konu" => "İngilizce", "puan" => 61 ]

Açıklamak:

  • Bu durumda ikinci parametre sağlanmaz.
  • $carry’nin varsayılan değeri null’dur, bu yüzden bunu koşullumuzda kontrol ederiz.

Arama değerine git veya varsayılanı döndürme özelliğini kullanma #

Kendinizi genellikle değere karşılık gelen bir değişken bulmanız gereken bir durumda bulursunuz ve koleksiyonlar sizi ele geçirdi.

Aşağıdaki örnekte, bir dizide karşılık gelen çağrı kodu atanmış üç farklı yerel ayar elde ettik. Bir yerel ayar sağlayabilmek ve karşılığında ilgili arama kodunu alabilmek istiyoruz. get içindeki ikinci parametre, ilk parametre bulunamazsa varsayılan parametredir.

function lookupCallingCode($locale)
{
    return collect([
        'de_DE' => 49,
        'en_GB' => 44,
        'en_US' => 1,
    ])->get($locale, 44);
}

Yukarıdaki örnekte aşağıdakileri yapabiliriz

lookupCallingCode('de_DE'); // Will return 49
lookupCallingCode('sv_SE'); // Will return 44

Varsayılan değer olarak bir geri arama bile iletebilirsiniz. Belirtilen anahtar mevcut değilse, geri aramanın sonucu döndürülür:

    return collect([
        'de_DE' => 49,
        'en_GB' => 44,
        'en_US' => 1,
    ])->get($locale, function() {
        return 44;
    });

Koleksiyondaki her öğeyi değiştirmek için Haritayı kullanma #

Genellikle bir veri kümesinin yapılandırılma şeklini değiştirmeniz ve belirli değerleri manipüle etmeniz gerekir.

Aşağıdaki örnekte, indirim tutarı eklenmiş bir kitap koleksiyonumuz var. Ama daha çok, fiyatı zaten indirimli olan kitapların bir listesi var.

$books = [
    ['title' => 'The Pragmatic Programmer', 'price' => 20, 'discount' => 0.5],
    ['title' => 'Continuous Delivery', 'price' => 25, 'discount' => 0.1],
    ['title' => 'The Clean Coder', 'price' => 10, 'discount' => 0.75],
];

$discountedItems =  collect($books)->map(function ($book) {
   return ['title' => $book["title"], 'price' => $book["price"] * $book["discount"]];
});

//[
//    ['title' => 'The Pragmatic Programmer', 'price' => 10],
//    ['title' => 'Continuous Delivery', 'price' => 12.5],
//    ['title' => 'The Clean Coder', 'price' => 5],
//]

Bu aynı zamanda tuşları değiştirmek için de kullanılabilir, diyelim ki title anahtarını name olarak değiştirmek istedik bu uygun bir çözüm olacaktır.

Koleksiyonu sıralama #

Bir koleksiyonu sıralamanın birkaç farklı yolu vardır.

Çeşit() #

‘sort’ yöntemi koleksiyonu sıralar:

$collection = collect([5, 3, 1, 2, 4]);

$sorted = $collection->sort();

echo $sorted->values()->all();

returns : [1, 2, 3, 4, 5]

‘Sort’ yöntemi ayrıca kendi algoritmanızla özel bir geri arama iletilmesine de izin verir. Başlık altında sıralama, php’nin usort‘ini kullanır.

$collection = $collection->sort(function ($a, $b) {
    if ($a == $b) {
        return 0;
    }
    return ($a < $b) ? -1 : 1;
});

Göre sırala() #

‘sortBy’ yöntemi, koleksiyonu verilen anahtara göre sıralar:

$collection = collect([
['name' => 'Desk', 'price' => 200],
['name' => 'Chair', 'price' => 100],
['name' => 'Bookcase', 'price' => 150],
]);

$sorted = $collection->sortBy('price');

echo $sorted->values()->all();

returns:  [
        ['name' => 'Chair', 'price' => 100],
        ['name' => 'Bookcase', 'price' => 150],
        ['name' => 'Desk', 'price' => 200],
    ]

‘sortBy’ yöntemi, çok boyutlu bir diziyi sıralamak için daha derin anahtara erişmek için nokta gösterim biçiminin kullanılmasına izin verir.

$collection = collect([
    ["id"=>1,"product"=>['name' => 'Desk', 'price' => 200]],
    ["id"=>2, "product"=>['name' => 'Chair', 'price' => 100]],
    ["id"=>3, "product"=>['name' => 'Bookcase', 'price' => 150]],
    ]);

$sorted = $collection->sortBy("product.price")->toArray();

return: [
  ["id"=>2, "product"=>['name' => 'Chair', 'price' => 100]],
  ["id"=>3, "product"=>['name' => 'Bookcase', 'price' => 150]],
  ["id"=>1,"product"=>['name' => 'Desk', 'price' => 200]],
]

SortByDesc() #

Bu yöntem, “sortBy” yöntemiyle aynı imzaya sahiptir, ancak koleksiyonu ters sırada sıralayacaktır.
