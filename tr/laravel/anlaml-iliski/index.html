İlgili Modelleri Ekleme #

“Yorum” ile “hasMany” ilişkisi olan bir “Post” modeliniz olduğunu varsayalım. Aşağıdakileri yaparak bir gönderiyle ilgili bir “Yorum” nesnesi ekleyebilirsiniz:

$post = Post::find(1);

$commentToAdd = new Comment(['message' => 'This is a comment.']);

$post->comments()->save($commentToAdd);

“saveMany” işlevini kullanarak birden fazla modeli aynı anda kaydedebilirsiniz:

$post = Post::find(1);

$post->comments()->saveMany([
    new Comment(['message' => 'This a new comment']),
    new Comment(['message' => 'Me too!']),
    new Comment(['message' => 'Eloquent is awesome!'])
]);

Alternatif olarak, Eloquent model örneği yerine düz bir PHP dizisini kabul eden bir “create” yöntemi de vardır.

$post = Post::find(1);

$post->comments()->create([
    'message' => 'This is a new comment message'
]);

İlişki Türleri #

Birden Çoka #

Her Gönderinin bir veya daha fazla yorumu olabileceğini ve her yorumun yalnızca tek bir Gönderiye ait olduğunu varsayalım.

bu nedenle yorumlar tablosunda ‘post_id’ olacaktır. Bu durumda ilişkiler aşağıdaki gibi olacaktır.

Post Model

public function comments()
{
   return $this->belongsTo(Post::class);
}

Yabancı anahtar “post_id” dışındaysa, örneğin yabancı anahtar “example_post_id” olur.

public function comments()
{
   return $this->belongsTo(Post::class, 'example_post_id');
}

ve ayrıca, yerel anahtar “id” dışındaysa, örneğin yerel anahtar “other_id"dir

public function comments()
{
   return $this->belongsTo(Post::class, 'example_post_id', 'other_id');
}

Yorum Modeli

birden çoğa tersini tanımlama

public function post()
{
   return $this->hasMany(Comment::class);
}

Bire bir #

İki model arasında nasıl ilişki kurulur (örnek: “Kullanıcı” ve “Telefon” modeli) #

‘Uygulama\Kullanıcı’

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class User extends Model
{
    /**
     * Get the phone record associated with the user.
     */
    public function phone()
    {
        return $this->hasOne('Phone::class', 'foreign_key', 'local_key');
    }
}

‘Uygulama\Telefon’

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class Phone extends Model
{
    /**
     * Get the user that owns the phone.
     */
    public function user()
    {
        return $this->belongsTo('User::class', 'foreign_key', 'local_key');
    }
}

“yabancı_anahtar” : Eloquent varsayılan olarak bu değeri “diğer_model_adı_kimliği” (bu durumda “kullanıcı_kimliği” ve “telefon_kimliği”) olarak varsayar, değilse değiştirin.

’local_key’ : Eloquent varsayılan olarak bu değeri ‘id’ (mevcut model birincil anahtar) olarak kabul edecektir, değilse değiştirin.

Veritabanınız laravel standardına göre dosyalanmışsa, ilişki bildiriminde yabancı anahtar ve yerel anahtar sağlamanıza gerek yoktur.

Açıklama #

Çoktan Çoka #

Diyelim ki roller ve izinler var. Her rol birçok izne ait olabilir ve her izin birçok role ait olabilir. yani 3 masa olacak. iki model ve bir pivot tablo. bir “roller”, “kullanıcılar” ve “permission_role” tablosu.

Rol model

public function permissions()
{
   return $this->belongsToMany(Permission::class);
}

İzin Modeli

public function roles()
{
   return $this->belongsToMany(Roles::class);
}

Not 1

pivot tablo için farklı tablo adı kullanırken aşağıdakileri göz önünde bulundurun.

Eloquent, pivot anahtar adlarını oluşturmak için alfabetik sıra kullandığından, “permission_role” yerine “role_permission” kullanmak istediğinizi varsayalım. pivot tablo adını aşağıdaki gibi ikinci parametre olarak iletmeniz gerekecektir.

Rol model

public function permissions()
{
   return $this->belongsToMany(Permission::class, 'role_permission');
}

İzin Modeli

public function roles()
{
   return $this->belongsToMany(Roles::class, 'role_permission');
}

Not: 2

pivot tabloda farklı anahtar adları kullanırken aşağıdakileri göz önünde bulundurun.

Eloquent, üçüncü ve dördüncü parametre olarak hiçbir anahtar iletilmezse, bunun ‘_id’ ile tekil tablo adları olacağını varsayar. bu nedenle, pivotun “role_id” ve “permission_id” alanlarına sahip olacağını varsayar. Bunların dışındaki anahtarlar kullanılacaksa üçüncü ve dördüncü parametre olarak geçilmelidir.

Diyelim ki ‘role_id’ yerine ‘other_role_id’ ve ‘permission_id’ yerine ‘other_permission_id’ kullanılacak. Yani aşağıdaki gibi olacaktır.

Rol model

public function permissions()
{
   return $this->belongsToMany(Permission::class, 'role_permission', 'other_role_id', 'other_permission_id');
}

İzin Modeli

public function roles()
{
   return $this->belongsToMany(Roles::class, 'role_permission', 'other_permission_id', 'other_role_id');
}

Polimorfik #

Polimorfik ilişkiler, bir modelin tek bir ilişki üzerinde birden fazla başka modele ait olmasına izin verir. İyi bir örnek resimler olabilir, hem kullanıcı hem de ürün bir resme sahip olabilir. Tablo yapısı aşağıdaki gibi görünebilir:

user
    id - integer
    name - string
    email - string

product
    id - integer
    title - string
    SKU - string

image
    id - integer
    url - string
    imageable_id - integer
    imageable_type - string

Bakılması gereken önemli sütunlar resimler tablosundadır. “imageable_id” sütunu, kullanıcının veya ürünün kimlik değerini, “imageable_type” sütunu ise sahip olan modelin sınıf adını içerecektir. Modellerinizde ilişkileri şu şekilde kurarsınız:

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class Image extends Model
{
    /**
     * Get all of the owning imageable models.
     */
    public function imageable()
    {
        return $this->morphTo();
    }
}

class User extends Model
{
    /**
     * Get all of the user's images.
     */
    public function images()
    {
        return $this->morphMany('Image::class', 'imageable');
    }
}

class Product extends Model
{
    /**
     * Get all of the product's images.
     */
    public function images()
    {
        return $this->morphMany('Image::class', 'imageable');
    }
}

Ayrıca, ‘morphTo’ çağrısını gerçekleştiren yöntemin adına erişerek polimorfik modelden bir polimorfik ilişkinin sahibini alabilirsiniz. Bizim durumumuzda bu, Image modelindeki ‘imageable’ yöntemidir. Böylece, bu yönteme dinamik bir özellik olarak erişeceğiz.

$image = App\Image::find(1);

$imageable = $image->imageable;

Bu “görüntülenebilir”, bir Kullanıcı veya Ürün döndürür.

İlişkileri sorgulama #

Eloquent ayrıca aşağıda gösterildiği gibi tanımlanmış ilişkiler üzerinde sorgulama yapmanızı sağlar:

User::whereHas('articles', function (Builder $query) {
    $query->where('published', '!=', true);
})->get();

Bu, bu durumda ilişki yöntemi adınızın “makaleler” olmasını gerektirir. Kapanışta iletilen argüman, ilgili model için Sorgu Oluşturucu’dur, böylece başka bir yerde kullanabileceğiniz herhangi bir sorguyu burada kullanabilirsiniz.

Hevesli Yükleme

Kullanıcı modelinin Makale modeliyle bir ilişkisi olduğunu ve ilgili makaleleri yüklemek istediğinizi varsayalım. Bu, kullanıcı alınırken kullanıcının makalelerinin yükleneceği anlamına gelir.

“makaleler”, Kullanıcı modelindeki ilişki adıdır (yöntem).

User::with('articles')->get();

birden fazla ilişkiniz varsa. örneğin makaleler ve gönderiler.

User::with('articles','posts')->get();

ve iç içe geçmiş ilişkileri seçmek için

User::with('posts.comments')->get();

Birden fazla iç içe ilişki arayın

User::with('posts.comments.likes')->get()

** withPivot() kullanarak Ara tabloya erişme**

Pivot tabloda üçüncü bir ‘permission_assigned_date’ sütununuz olduğunu varsayalım. Varsayılan olarak, pivot nesnede yalnızca model anahtarları bulunur. Şimdi bu sütunu sorgu sonucunda elde etmek için, adı withPivot() işlevine eklemeniz gerekir.

   public function permissions()
        {
           return $this->belongsToMany(Permission::class, 'role_permission', 'other_role_id', 'other_permission_id')->withPivot('permission_assigned_date');
        }

Takma / Çıkarma

Eloquent, ilgili modellerle çalışmayı daha kolay hale getirmek için birkaç ek yardımcı yöntem de sağlar. Örneğin, bir kullanıcının birçok rolü olabileceğini ve bir rolün birçok izne sahip olabileceğini düşünelim. Modelleri birleştiren ara tabloya bir kayıt ekleyerek bir izne rol eklemek için ekleme yöntemini kullanın:

$role= App\Role::find(1);    
$role->permissions()->attach($permissionId);

Bir modele ilişki eklerken, ara tabloya eklenecek bir dizi ek veri de iletebilirsiniz:

$rol->roles()->attach($permissionId, ['permission_assigned_date' => $date]);

Benzer şekilde, bir role karşı belirli bir izni kaldırmak için ayırma işlevini kullanın

$role= App\Role::find(1);
//will remove permission 1,2,3 against role 1
$role->permissions()->detach([1, 2, 3]);

Senkronizasyon İlişkilendirmeleri

Çoktan çoğa ilişkilendirmeler oluşturmak için eşitleme yöntemini de kullanabilirsiniz. Senkronizasyon yöntemi, ara tabloya yerleştirilecek bir dizi kimliği kabul eder. Verilen dizide olmayan tüm kimlikler ara tablodan kaldırılacaktır. Dolayısıyla, bu işlem tamamlandıktan sonra, ara tabloda yalnızca verilen dizideki kimlikler bulunacaktır:

//will keep permission id's 1,2,3 against Role id 1

$role= App\Role::find(1)
$role->permissions()->sync([1, 2, 3]);

Giriiş #

Eloquent ilişkileri, Eloquent model sınıflarınızdaki işlevler olarak tanımlanır. Eloquent modellerinin kendileri gibi, ilişkiler de güçlü sorgu oluşturucular olarak hizmet ettiğinden, ilişkileri işlevler olarak tanımlamak, güçlü yöntem zincirleme ve sorgulama yetenekleri sağlar. Örneğin, bu gönderi ilişkisinde ek kısıtlamalar zincirleyebiliriz:

$user->posts()->where('active', 1)->get();
