Kurulum #

Laravel uygulamaları, popüler bir PHP bağımlılık yöneticisi olan Composer ile kurulur ve yönetilir. Yeni bir Laravel uygulaması oluşturmanın iki yolu vardır.

Besteci aracılığıyla #

$ composer create-project laravel/laravel [foldername]

Veya

$ composer create-project --prefer-dist laravel/laravel [foldername]

[klasör adı] yerine yeni Laravel uygulamanızın yüklenmesini istediğiniz dizinin adını yazın. Kurulumdan önce mevcut olmamalıdır. Composer yürütülebilir dosyasını sistem yolunuza eklemeniz de gerekebilir.

Çerçevenin belirli bir sürümünü kullanarak bir Laravel projesi oluşturmak istiyorsanız, bir sürüm kalıbı sağlayabilirsiniz, aksi takdirde projeniz mevcut en son sürümü kullanacaktır.

Örneğin Laravel 5.2’de bir proje oluşturmak isteseydiniz, şunu çalıştırırdınız:

$ composer create-project --prefer-dist laravel/laravel 5.2.*

Neden –prefer-dist

Bir paketi indirmenin iki yolu vardır: “kaynak” ve “dist”. Kararlı sürümler için Composer varsayılan olarak ‘dist’i kullanır. “Kaynak” bir sürüm kontrol deposudur. --prefer-source etkinleştirilirse, varsa Composer kaynaktan yüklenir.

“–prefer-dist”, “-prefer-source"un tersidir ve Composer’a mümkünse “dist"ten yüklemesini söyler. Bu, yapı sunucularında ve genellikle satıcı güncellemelerini çalıştırmadığınız diğer kullanım durumlarında yüklemeleri önemli ölçüde hızlandırabilir. Ayrıca, uygun bir kurulumunuz yoksa Git ile ilgili sorunlardan kaçınmanıza izin verir.

Laravel yükleyicisi aracılığıyla #

Laravel, hızlı bir şekilde Laravel uygulamaları oluşturmak için yardımcı bir komut satırı yardımcı programı sağlar. İlk önce yükleyiciyi kurun:

$ composer global require laravel/installer

Laravel yükleyicisini çalıştırmak için Composer ikili dosyaları klasörünün $PATH değişkeniniz içinde olduğundan emin olmalısınız.

İlk önce, $PATH değişkeninizde olup olmadığına bakın

echo $PATH

Her şey doğruysa, çıktı şöyle bir şey içermelidir:

Kullanıcılar/kullanıcıadınız/.composer/vendor/bin

Değilse, .bashrc dosyanızı veya ZSH kullanıyorsanız .zshrc dosyanızı, Composer satıcı dizininizin yolunu içerecek şekilde düzenleyin.

Kurulduktan sonra, bu komut belirttiğiniz dizinde yeni bir Laravel kurulumu oluşturacaktır.

laravel new [foldername]

Projeyi bir alt dizin oluşturmadan mevcut çalışma dizininde oluşturmak için [klasöradı] yerine . (nokta) da kullanabilirsiniz.

Uygulamayı çalıştırma #

Laravel, çalıştırılarak başlatılabilen PHP tabanlı bir web sunucusuyla birlikte gelir.

$ php artisan serve

Varsayılan olarak, HTTP sunucusu 8000 numaralı bağlantı noktasını kullanır, ancak bağlantı noktası zaten kullanımdaysa veya aynı anda birden çok Laravel uygulamasını çalıştırmak istiyorsanız, farklı bir bağlantı noktası belirtmek için `–port’ bayrağını kullanabilirsiniz:

$ php artisan serve --port=8080

HTTP sunucusu, uygulamayı çalıştırmak için varsayılan etki alanı olarak “localhost"u kullanır, ancak farklı bir adres belirtmek için “–host” işaretini kullanabilirsiniz:

$ php artisan serve --host=192.168.0.100 --port=8080

Farklı bir sunucu kullanma #

Farklı bir web sunucusu yazılımı kullanmayı tercih ederseniz, projenizin ‘public’ dizini içinde sizin için bazı konfigürasyon dosyaları sağlanır; Apache için .htaccess ve ASP.NET için web.config. NGINX gibi diğer yazılımlar için çeşitli çevrimiçi araçları kullanarak Apache yapılandırmalarını dönüştürebilirsiniz.

Çerçeve, web sunucusu kullanıcısının aşağıdaki dizinlerde yazma izinlerine sahip olmasını gerektirir:

  • /depolama
  • /bootstrap/önbellek

*nix işletim sistemlerinde bu,

chown -R www-data:www-data storage bootstrap/cache
chmod -R ug+rwx storage bootstrap/cache

(burada “www-data”, web sunucusu kullanıcısının adı ve grubudur)

Seçtiğiniz web sunucusu, genellikle belge kökü olarak ayarlanarak yapılan, projenizin `/public’ dizininden içerik sunacak şekilde yapılandırılmalıdır. Projenizin geri kalanı web sunucunuz üzerinden erişilebilir olmamalıdır.

Her şeyi doğru ayarlarsanız, web sitenizin URL’sine gitmek, Laravel’in varsayılan açılış sayfasını göstermelidir.

Merhaba Dünya Örneği (Temel) #

Rota dosyasını açın. Aşağıdaki kodu yapıştırın:

Route::get('helloworld', function () {
    return '<h1>Hello World</h1>';
});

’localhost/helloworld’ rotasına gittikten sonra Merhaba Dünya gösterir.

Rotalar dosyası bulunur:

Web için

routes/web.php

API’ler için

routes/api.php
son sürüm 
app/Http/routes.php
son sürüm 
app/routes.php
son sürüm

Merhaba Dünya Örneği (Denetleyici ve Görünüm Kullanarak) #

  1. Bir Laravel uygulaması oluşturun:

     $ composer create-project laravel/laravel hello-world

  2. Proje klasörüne gidin, örn.

     $ cd C:\xampp\htdocs\hello-world

  3. Bir denetleyici oluşturun:

     $ php artisan make:controller HelloController --resource

Bu, app/Http/Controllers/HelloController.php dosyasını oluşturacaktır. “–resource” seçeneği, denetleyici için CRUD yöntemleri üretecektir, ör. indeksle, oluştur, göster, güncelle.

  1. HelloController’ın ‘index’ yöntemine bir rota kaydedin. Bu satırı app/Http/routes.php (sürüm 5.0 ila 5.2) veya routes/web.php (sürüm 5.3) dosyasına ekleyin:
    Route::get('hello', 'HelloController@index');

Yeni eklenen rotalarınızı görmek için $ php artisan route:list komutunu çalıştırabilirsiniz.

  1. “Görünümler” dizininde bir Blade şablonu oluşturun:

    resources/views/hello.blade.php:

     <h1>Hello world!</h1>

  2. Şimdi indeks yöntemine hello.blade.php şablonunu görüntülemesini söylüyoruz:

    app/Http/Controllers/HelloController.php

    <?php
    
    namespace App\Http\Controllers;
    
    use Illuminate\Http\Request;
    
    use App\Http\Requests;
    
    class HelloController extends Controller
    {
        /**
         * Display a listing of the resource.
         *
         * @return \Illuminate\Http\Response
         */
        public function index()
        {
            return view('hello');
        }
    
        // ... other resources are listed below the index one above

Aşağıdaki PHP Artisan Komutunu kullanarak uygulamanızı sunabilirsiniz: php artisan serve; size uygulamanıza erişebileceğiniz adresi gösterecektir (genellikle varsayılan olarak http://localhost:8000’dedir).

Alternatif olarak, doğrudan tarayıcınızda uygun konuma gidebilirsiniz; XAMPP gibi bir sunucu kullanıyorsanız (ya: http://localhost/hello-world/public/hello, Laravel örneğinizi hello-world, doğrudan xampp/htdocs içine kurmuş olsaydınız). dizin: bu Hello Word’ün 1. adımını komut satırı arabiriminizden yürüterek, xampp/htdocs dizininizi işaret ederek).

LaraDock kullanarak kurulum (Docker için Laravel Homestead) #

LaraDock, Laravel Homestead benzeri bir geliştirme ortamıdır, ancak Vagrant yerine Docker içindir. https://github.com/LaraDock/laradock

Kurulum #

*Git ve Docker gerektirir

LaraDock deposunu klonlayın:

A. Halihazırda bir Laravel projeniz varsa, bu depoyu Laravel kök dizininize kopyalayın:

git submodule add https://github.com/LaraDock/laradock.git

B. Bir Laravel projeniz yoksa ve Docker’dan Laravel’i kurmak istiyorsanız, bu repoyu makinenizin herhangi bir yerine klonlayın:

git clone https://github.com/LaraDock/laradock.git

Temel Kullanım #

  1. Kapsayıcıları Çalıştırın: (Docker-compose komutlarını çalıştırmadan önce laradock klasöründe olduğunuzdan emin olun).

Örnek: NGINX ve MySQL’i çalıştırma: docker-compose up -d nginx mysql

Kendi kombinasyonlarınızı oluşturmak için seçebileceğiniz mevcut kapların bir listesi vardır.

“nginx”, “hhvm”, “php-fpm”, “mysql”, “redis”, “postgres”, “mariadb”, “neo4j”, “mongo”, “apache2”, “caddy”, “memcached”, “beanstalkd”, “beanstalkd-konsol”, “çalışma alanı”

  1. (Artisan, Composer, PHPUnit, Gulp, …) gibi komutları yürütmek için Workspace kapsayıcısına girin.

    docker-compose exec workspace bash

  2. Henüz kurulu bir Laravel projeniz yoksa, bir Docker konteynerinden Laravel’i kurma adımını izleyin.

a. Çalışma Alanı kapsayıcısını girin.

b. Laravel’i yükleyin. besteci yarat-proje laravel/laravel benim-harika-uygulamam "5.3.*"

  1. Laravel konfigürasyonlarını düzenleyin. Laravel’inizin .env dosyasını açın ve DB_HOST’u mysql’inize ayarlayın:

    DB_HOST=mysql

  2. Tarayıcınızı açın ve yerel ana bilgisayar adresinizi ziyaret edin.
