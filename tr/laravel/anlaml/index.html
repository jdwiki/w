Eloquent, Laravel’de bulunan bir ORM’dir (Nesne İlişkisel Model). Etkin kayıt modelini uygular ve ilişkisel veritabanlarıyla etkileşim kurmak için kullanılır.

Tablo adlandırma

Kural, tablo adları için çoğul “snake_case” ve model adları için tekil “StudlyCase” kullanmaktır. Örneğin:

  • Bir “kediler” tablosunun bir “Kedi” modeli olur
  • Bir “jungle_cats” tablosunun “JungleCat” modeli olacaktır
  • Bir “kullanıcı” tablosunun bir “Kullanıcı” modeli olacaktır
  • Bir ‘insanlar’ tablosu ‘Kişi’ modeline sahip olacaktır

Eloquent, modelinizi, yukarıda belirtildiği gibi, model adının çoğulu olan bir tabloyla otomatik olarak bağlamaya çalışacaktır.

Ancak, varsayılan kuralı geçersiz kılmak için bir tablo adı belirtebilirsiniz.

class User extends Model
{ 
    protected $table = 'customers';
}

Giriiş #

Eloquent, Laravel çerçevesinde yerleşik olan ORM‘dir. ActiveRecord kalıbını kullanarak veritabanı tablolarınızla nesne yönelimli bir şekilde etkileşime girmenizi sağlar.

Tek bir model sınıfı genellikle tek bir veritabanı tablosuna ve ayrıca farklı türlerdeki ilişkilere (bire bir, [bire çok][4], [çoktan çoğa][5 ], polimorfik) farklı model sınıfları arasında tanımlanabilir.

Bölüm [Model Yapmak][6], model sınıflarının oluşturulmasını ve tanımını açıklar.

Eloquent modellerini kullanmaya başlamadan önce, config/database.php yapılandırma dosyanızda en az bir veritabanı bağlantısının yapılandırıldığından emin olun.

Geliştirme sırasında anlamlı sorgu oluşturucunun kullanımını anlamak için php artisan ide-helper:generate komutunu kullanabilirsiniz. İşte [bağlantı][7].

Siliniyor #

Verileri veritabanına yazdıktan sonra silebilirsiniz. Bir model örneğini aldıysanız silebilir veya hangi kayıtların silineceğine ilişkin koşulları belirtebilirsiniz.

Bir model örneğini silmek için onu alın ve “delete()” yöntemini çağırın:

$user = User::find(1);
$user->delete();

Alternatif olarak, silmek istediğiniz kayıtların birincil anahtarını (veya birincil anahtar dizisini) “destroy()” yöntemiyle belirtebilirsiniz:

User::destroy(1);
User::destroy([1, 2, 3]);

Ayrıca, sorgulamayı silme ile birleştirebilirsiniz:

User::where('age', '<', 21)->delete();

Bu, koşulla eşleşen tüm kullanıcıları siler.

Not: Eloquent aracılığıyla toplu silme ifadesi yürütülürken, silinen modeller için ‘deleting’ ve ‘deleted’ model olayları tetiklenmeyecektir. Bunun nedeni, silme ifadesi yürütülürken modellerin hiçbir zaman gerçekten alınmamasıdır.

Yumuşak Silme #

Bazen bir kaydı kalıcı olarak silmek istemezsiniz, ancak denetleme veya raporlama amaçlarıyla yanınızda bulundurun. Bunun için Eloquent, soft silme işlevselliği sağlar.

Modelinize geçici silme işlevi eklemek için, “SoftDeletes” özelliğini içe aktarmanız ve Eloquent model sınıfınıza eklemeniz gerekir:

namespace Illuminate\Database\Eloquent\Model;
namespace Illuminate\Database\Eloquent\SoftDeletes;

class User extends Model
{
     use SoftDeletes;
}

Bir modeli silerken, modeliniz için tablodaki “deleted_at” zaman damgası sütununa bir zaman damgası koyacaktır, bu nedenle önce tablonuzda “deleted_at” sütununu oluşturduğunuzdan emin olun. Veya geçişte, ‘deleted_at’ zaman damgasını eklemek için planınızda ‘softDeletes()’ yöntemini çağırmalısınız. Örnek:

Schema::table('users', function ($table) {
    $table->softDeletes();
});

Herhangi bir sorgu, geçici olarak silinen kayıtları atlayacaktır. İsterseniz bunları withTrashed() kapsamını kullanarak zorla gösterebilirsiniz:

User::withTrashed()->get();

Kullanıcıların geçici silme işleminden sonra (yani bir çöp kutusu türü alanda) bir kaydı geri yüklemesine izin vermek istiyorsanız, o zaman ‘restore()’ yöntemini kullanabilirsiniz:

$user = User::find(1);
$user->delete();
$user->restore();

Bir kaydı zorla silmek için, kaydı veritabanından gerçekten kaldıracak olan ‘forceDelete()’ yöntemini kullanın:

$user = User::find(1);
$user->forceDelete();

Birincil anahtarı ve zaman damgalarını değiştirin #

Eloquent modelleri varsayılan olarak birincil anahtarın “id” olarak adlandırılmasını bekler. Durumunuz bu değilse, $primaryKey özelliğini belirterek birincil anahtarınızın adını değiştirebilirsiniz.

class Citizen extends Model
{
    protected $primaryKey = 'socialSecurityNo';

    // ...
}

Şimdi, birincil anahtarınızı kullanan tüm Eloquent yöntemleri (örneğin, “find” veya “findOrFail”) bu yeni adı kullanacaktır.

Ayrıca Eloquent, birincil anahtarın otomatik artan bir tam sayı olmasını bekler. Birincil anahtarınız otomatik artan bir tamsayı değilse (örneğin bir GUID), “$incrementing” özelliğini “false” olarak güncelleyerek Eloquent’e söylemeniz gerekir:

class Citizen extends Model
{
    protected $primaryKey = 'socialSecurityNo';

    public $incrementing = false;

    // ...
}

Varsayılan olarak Eloquent, tablolarınızda “created_at” ve “updated_at” sütunlarının bulunmasını bekler. Bu sütunların Eloquent tarafından otomatik olarak yönetilmesini istemiyorsanız, modelinizdeki $timestamps özelliğini false olarak ayarlayın:

class Citizen extends Model
{
    public $timestamps = false;

    // ...
}

Zaman damgalarını depolamak için kullanılan sütunların adlarını özelleştirmeniz gerekiyorsa, modelinizde “CREATED_AT” ve “UPDATED_AT” sabitlerini ayarlayabilirsiniz:

class Citizen extends Model
{
    const CREATED_AT = 'date_of_creation';
    const UPDATED_AT = 'date_of_last_update';

    // ...
}

Kalıcı #

Eloquent ile veri okumanın yanı sıra, save() yöntemiyle veri eklemek veya güncellemek için de kullanabilirsiniz. Yeni bir model örneği oluşturduysanız, kayıt inserted olacaktır; aksi takdirde, veritabanından bir model aldıysanız ve yeni değerler ayarladıysanız, güncellenir.

Bu örnekte yeni bir “Kullanıcı” kaydı oluşturuyoruz:

$user = new User();
$user->first_name = 'John';
$user->last_name = 'Doe';
$user->email = 'john.doe@example.com';
$user->password = bcrypt('my_password');
$user->save();

Alanları bir dizi veri kullanarak doldurmak için “create” yöntemini de kullanabilirsiniz:

User::create([
    'first_name'=> 'John',
    'last_name' => 'Doe',
    'email'     => 'john.doe@example.com', 
    'password'  => bcrypt('changeme'),
]);

Create yöntemini kullanırken, nitelikleriniz modelinizdeki ‘doldurulabilir’ dizide bildirilmelidir:

class User extends Model
{
    protected $fillable = [
         'first_name',
         'last_name',
         'email',
         'password',
    ];
}

Alternatif olarak, tüm öznitelikleri toplu olarak atanabilir yapmak istiyorsanız, $guarded özelliğini boş bir dizi olarak tanımlayabilirsiniz:

class User extends Model
{
    /**
    * The attributes that aren't mass assignable.
    *
    * @var array
    */
    protected $guarded = [];
}

Ancak, ‘create’ yöntemi yerine ‘forceCreate’ yöntemini kullanarak modelinizdeki ‘fillable’ özniteliğini bile değiştirmeden bir kayıt oluşturabilirsiniz.

User::forceCreate([
    'first_name'=> 'John',
    'last_name' => 'Doe',
    'email'     => 'john.doe@example.com', 
    'password'  => bcrypt('changeme'),
]);

Aşağıda, mevcut bir “Kullanıcı” modelini önce yükleyerek (“bul” kullanarak), değiştirerek ve sonra kaydederek güncelleme örneği verilmiştir:

$user = User::find(1);
$user->password = bcrypt('my_new_password');
$user->save();

Aynı başarıyı tek bir işlev çağrısıyla gerçekleştirmek için “güncelleme” yöntemini kullanabilirsiniz:

$user->update([
    'password' => bcrypt('my_new_password'),
]);

“Create” ve “update” yöntemleri, büyük veri kümeleriyle çalışmayı, aşağıdaki örneklerde gösterildiği gibi, her bir anahtar/değer çiftini ayrı ayrı ayarlamaktan çok daha basit hale getirir:

İstek verilerini toplarken “yalnızca” ve “hariç” kullanımına dikkat edin. Güncellenmesine izin vermek/vermemek istediğiniz anahtarları tam olarak belirtmeniz önemlidir, aksi takdirde bir saldırganın istekleriyle birlikte ek alanlar göndermesi ve istenmeyen güncellemelere neden olması mümkündür.

// Updating a user from specific request data
$data = Request::only(['first_name', 'email']);
$user->find(1);
$user->update($data);

// Create a user from specific request data
$data = Request::except(['_token', 'profile_picture', 'profile_name']);
$user->create($data);

Varlık bulunamazsa 404 atın #

Bir modda bulunmayan bir kaydı ararken otomatik olarak bir istisna atmak istiyorsanız, ikisinden birini kullanabilirsiniz.

Vehicle::findOrFail(1);

veya

Vehicle::where('make', 'ford')->firstOrFail();

‘1’ birincil anahtarına sahip bir kayıt bulunamazsa, bir ‘ModelNotFoundException’ atılır. Hangisi aslında yazmakla aynı (kaynağı görüntüle):

$vehicle = Vehicle::find($id);

if (!$vehicle) {
    abort(404);
}

Klonlama Modelleri #

Kendinizi bir satırı klonlamaya, belki birkaç özelliği değiştirmeye ihtiyaç duyabilirsiniz, ancak işleri KURU tutmak için etkili bir yola ihtiyacınız var. Laravel, bu işlevi yapmanıza izin vermek için bir tür ‘gizli’ yöntem sağlar. Tamamen belgelenmemiş olmasına rağmen, onu bulmak için API’yi aramanız gerekir.

$model->replicate() kullanarak bir kaydı kolayca kopyalayabilirsiniz

$robot = Robot::find(1);
$cloneRobot = $robot->replicate();
// You can add custom attributes here, for example he may want to evolve with an extra arm!
$cloneRobot->arms += 1;
$cloneRobot->save();

Yukarıdakiler, kimliği 1 olan bir robot bulur ve ardından onu klonlar.
