Birden çok veritabanı bağlantısı #

Laravel, kullanıcının birden çok veritabanı bağlantısı üzerinde çalışmasına izin verir. Birden fazla veritabanına bağlanmanız ve bunların birlikte çalışmasını sağlamanız gerekiyorsa, bağlantı kurulumuna dikkat edin.

Dilerseniz aynı uygulamada farklı veritabanlarının kullanılmasına da izin verebilirsiniz.

Varsayılan bağlantı config/database.php içinde, yapılandırma öğesi çağrısını görebilirsiniz:

'default' => env('DB_CONNECTION', 'mysql'),

Bu ad, aşağıdaki bağlantıların mysql adına atıfta bulunur:

'connections' => [

    'sqlite' => [
        'driver' => 'sqlite',
        'database' =>  database_path('database.sqlite'),
        'prefix' => '',
    ],

    'mysql' => [
        'driver' => 'mysql',
        'host' => env('DB_HOST', 'localhost'),
        'port' => env('DB_PORT', '3306'),
        'database' => env('DB_DATABASE', 'forge'),
        'username' => env('DB_USERNAME', 'forge'),
        'password' => env('DB_PASSWORD', ''),
        'charset' => 'utf8',
        'collation' => 'utf8_unicode_ci',
        'prefix' => '',
        'strict' => false,
        'engine' => null,
    ],
],

Diğer kodlarda veya komutlarda veritabanı bağlantısının adını belirtmediyseniz, Laravel varsayılan veritabanı bağlantı adını alacaktır. ancak, birden çok veritabanı bağlantılarında, varsayılan bağlantıyı kursanız bile, kullandığınız veritabanı bağlantısını her yerde daha iyi kurarsınız.

Taşıma dosyası

Taşıma dosyasında, tek bir veritabanı bağlantısı varsa şunları kullanabilirsiniz:

 Schema::create("table",function(Blueprint $table){
     $table->increments('id');
});

Birden çok veritabanı bağlantısında, hangi veritabanı bağlantısını kullandığınızı Laravel’e söylemek için connection() yöntemini kullanacaksınız:

 Schema::connection("sqlite")->create("table",function(Blueprint $table){
     $table->increments('id');
});

Esnaf Göçü

tek veritabanı bağlantısı kullanıyorsanız, şunları çalıştırırsınız:

php artisan migrate

Ancak, birden çok veritabanı bağlantısı için, taşıma verilerini hangi veritabanı bağlantısının tuttuğunu daha iyi bilmelisiniz. böylece aşağıdaki komutu çalıştıracaksınız:

php artisan migrate:install --database=sqlite

Bu komut, geçişi hazırlamak için hedef veritabanına geçiş tablosunu yükleyecektir.

php artisan migrate --database=sqlite

Bu komut, taşıma işlemini çalıştıracak ve taşıma verilerini hedef veritabanına kaydedecektir.

php artisan migrate:rollback --database=sqlite

Bu komut, taşıma işlemini geri alır ve taşıma verilerini hedef veritabanına kaydeder

Belirgin Model

Eloquent kullanarak bir veritabanı bağlantısı belirtmek için ``$connection``` özelliğini tanımlamanız gerekir:

namespace App\Model\Sqlite;
class Table extends Model
{
    protected $table="table";
    protected $connection = 'sqlite';
}

Eloquent kullanarak başka bir (ikinci) veritabanı bağlantısı belirtmek için:

namespace App\Model\MySql;
class Table extends Model
{
    protected $table="table";
    protected $connection = 'mysql';
}

Laravel, config/database.php``` içinde tanımlanan belirtilen bağlantıyı kullanmak için bir modelde tanımlanan ```$connection``` özelliğini kullanır. $connection``` özelliği bir modelde tanımlı değilse, varsayılan kullanılacaktır.

Statik on yöntemini kullanarak başka bir bağlantı da belirtebilirsiniz:

// Using the sqlite connection
Table::on('sqlite')->select(...)->get()
// Using the mysql connection
Table::on('mysql')->select(...)->get()

Veritabanı/Sorgu Oluşturucu

Sorgu oluşturucuyu kullanarak başka bir bağlantı da belirtebilirsiniz:

// Using the sqlite connection
DB::connection('sqlite')->table('table')->select(...)->get()
// Using the mysql connection
DB::connection('mysql')->table('table')->select(...)->get()

Ünite testi

Laravel, veritabanı bağlantı kodunu test etmek için seeInDatabase($table,$fielsArray,$connection) sağlar. Birim test dosyasında şunları yapmanız gerekir:

$this
    ->json(
        'GET',
        'result1/2015-05-08/2015-08-08/a/123'
    )
     ->seeInDatabase("log", ["field"=>"value"], 'sqlite');

Bu şekilde, Laravel hangi veritabanı bağlantısını test edeceğini bilecektir.

Birim Testinde Veritabanı İşlemleri

Laravel, veritabanının testler sırasında tüm değişiklikleri geri almasına izin verir. Birden çok veritabanı bağlantısını test etmek için ```$connectionsToTransact`` özelliklerini ayarlamanız gerekir.

use Illuminate\Foundation\Testing\DatabaseMigrations;

class ExampleTest extends TestCase
{
     use DatabaseTransactions;

     $connectionsToTransact =["mysql","sqlite"] //tell Laravel which database need to rollBack

    public function testExampleIndex()
    {
        $this->visit('/action/parameter')
         ->see('items');
    }
}
