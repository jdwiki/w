Yapılar, birlikte paketlenmiş çeşitli değişkenlerin kümeleridir. Yapının kendisi yalnızca değişkenleri içeren ve onları kolayca erişilebilir kılan bir package‘dir.

C’den farklı olarak, Go’nun yapıları kendilerine eklenmiş yöntemlere sahip olabilir. Ayrıca arayüzleri uygulamalarına izin verir. Bu, Go’nun yapılarını nesnelere benzer kılar, ancak bunlar (muhtemelen kasıtlı olarak) kalıtım gibi nesne yönelimli dillerde bilinen bazı temel özellikleri kaçırırlar.

Yöntemler #

Yapı yöntemleri, işlevlere çok benzer:

type User struct {
    name string
}

func (u User) Name() string {
    return u.name
}

func (u *User) SetName(newName string) {
    u.name = newName
}

Tek fark, yöntem alıcısının eklenmesidir. Türün bir örneği olarak veya türün bir örneğine işaretçi olarak bildirilebilir. “SetName()” örneği mutasyona uğrattığından, örnekte kalıcı bir değişiklik yapabilmek için alıcının bir işaretçi olması gerekir.

Örneğin:

package main

import "fmt"

type User struct {
    name string
}

func (u User) Name() string {
    return u.name
}

func (u *User) SetName(newName string) {
    u.name = newName
}

func main() {
    var me User

    me.SetName("Slim Shady")
    fmt.Println("My name is", me.Name())
}

Oyun Alanına Git

Anonim yapı #

Anonim bir yapı oluşturmak mümkündür:

data := struct {
    Number int 
    Text   string
} { 
    42, 
    "Hello world!",
}

Tam örnek:

package main

import (
    "fmt"
)

func main() {
    data := struct {Number int; Text string}{42, "Hello world!"} // anonymous struct
    fmt.Printf("%+v\n", data)
}

Kompozisyon ve Gömme #

Kompozisyon, kalıtım için bir alternatif sağlar. Bir yapı, bildiriminde ada göre başka bir tür içerebilir:

type Request struct {
    Resource string
}

type AuthenticatedRequest struct {
    Request
    Username, Password string
}

Yukarıdaki örnekte, “AuthenticatedRequest” dört genel üye içerecektir: “Resource”, “Request”, “Username” ve “Password”.

Bileşik yapılar, normal yapılarla aynı şekilde somutlaştırılabilir ve kullanılabilir:

func main() {
    ar := new(AuthenticatedRequest)
    ar.Resource = "example.com/request"
    ar.Username = "bob"
    ar.Password = "P@ssw0rd"
    fmt.Printf("%#v", ar)
}

Gömme #

Önceki örnekte, “İstek” gömülü bir alandır. Kompozisyon, farklı bir tür gömülerek de elde edilebilir. Bu, örneğin bir Struct’u daha fazla işlevsellikle dekore etmek için kullanışlıdır. Örneğin, Kaynak örneğiyle devam edersek, Kaynak alanının içeriğini biçimlendiren bir işlevin önüne “http://” veya “https://” koymasını istiyoruz. İki seçeneğimiz var: AuthenticatedRequest’te yeni yöntemleri oluşturun veya farklı bir yapıdan gömelim:

type ResourceFormatter struct {}

func(r *ResourceFormatter) FormatHTTP(resource string) string {
    return fmt.Sprintf("http://%s", resource)
}
func(r *ResourceFormatter) FormatHTTPS(resource string) string {
    return fmt.Sprintf("https://%s", resource)
}


type AuthenticatedRequest struct {
    Request
    Username, Password string
    ResourceFormatter
}

Ve şimdi ana işlev aşağıdakileri yapabilir:

func main() {
    ar := new(AuthenticatedRequest)
    ar.Resource = "www.example.com/request"
    ar.Username = "bob"
    ar.Password = "P@ssw0rd"

    println(ar.FormatHTTP(ar.Resource))
    println(ar.FormatHTTPS(ar.Resource))

    fmt.Printf("%#v", ar)
}

Gömülü bir ResourceFormatter yapısına sahip olan AuthenticatedRequest’e bakın.

Ama dezavantajı, kompozisyonunuzun dışındaki nesnelere erişememenizdir. Dolayısıyla ‘ResourceFormatter’, ‘AuthenticatedRequest’ten üyelere erişemez.

Dışa Aktarılan ve Dışa Aktarılmayan Alanlar (Özel ve Genel) #

Adları büyük harfle başlayan yapı alanları dışa aktarılır. Diğer tüm isimler ihraç edilmez.

type Account struct {
    UserID      int    // exported
    accessToken string // unexported
}

Dışa aktarılmayan alanlara yalnızca aynı paket içindeki kod ile erişilebilir. Bu nedenle, bir farklı paketinden bir alana erişiyorsanız, adının büyük harfle başlaması gerekir.

package main

import "bank"

func main() {
    var x = &bank.Account{
        UserID: 1,          // this works fine
        accessToken: "one", // this does not work, since accessToken is unexported
    }
}

Ancak, “banka” paketinin içinden hem UserId’e hem de accessToken’a sorunsuzca erişebilirsiniz.

“Banka” paketi şu şekilde uygulanabilir:

package bank

type Account struct {
    UserID int
    accessToken string
}

func ProcessUser(u *Account) {    
    u.accessToken = doSomething(u) // ProcessUser() can access u.accessToken because 
                                   // it's defined in the same package
}

Temel Beyanname #

Temel bir yapı aşağıdaki gibi bildirilir:

type User struct {
    FirstName, LastName string
    Email               string
    Age                 int
}

Her değere alan adı verilir. Alanlar genellikle her satıra bir tane yazılır ve alanın adı türünden önce gelir. Yukarıdaki örnekte ‘FirstName’ ve ‘LastName’ gibi aynı türden ardışık alanlar birleştirilebilir.

Etiketler #

Yapı alanları, kendileriyle ilişkilendirilmiş etiketlere sahip olabilir. Bu etiketler, geliştirici tarafından bir alan hakkında belirtilen özel bilgileri almak için ‘reflect’ paketi tarafından okunabilir.

struct Account {
    Username      string `json:"username"`
    DisplayName   string `json:"display_name"`
    FavoriteColor string `json:"favorite_color,omitempty"`
}

Yukarıdaki örnekte, etiketler, JSON’u sıralarken veya sıralamayı kaldırırken “encoding/json” paketi tarafından kullanılan anahtar adlarını değiştirmek için kullanılır.

Etiket herhangi bir dize değeri olabilse de, boşlukla ayrılmış key:"value" çiftlerini kullanmak en iyi uygulama olarak kabul edilir:

struct StructName {
    FieldName int `package1:"customdata,moredata" package2:"info"`
}

’encoding/xml’ ve ’encoding/json’ paketiyle birlikte kullanılan yapı etiketleri, standart kitaplık boyunca kullanılır.

Yapı Değişmezleri #

Bir yapı türünün değeri, alanları için değerleri belirten bir struct değişmezi kullanılarak yazılabilir.

type Point struct { X, Y int }
p := Point{1, 2}

Yukarıdaki örnek, her alanı doğru sırada belirtir. Bu yararlı değildir, çünkü programcıların tam alanları sırayla hatırlamaları gerekir. Daha sık olarak, bir yapı, alan adlarının bazıları veya tümü ve bunlara karşılık gelen değerleri listeleyerek başlatılabilir.

anim := gif.GIF{LoopCount: nframes}

Atlanan alanlar, türü için sıfır değerine ayarlanır.

Not: İki form aynı kelimede karıştırılamaz.

Yapı kopyaları oluşturma. #

Bir yapı, atama kullanılarak basitçe kopyalanabilir.

type T struct {
    I int
    S string
}

// initialize a struct
t := T{1, "one"}

// make struct copy
u := t // u has its field values equal to t

if u == t { // true
    fmt.Println("u and t are equal") // Prints: "u and t are equal"
}

Yukarıdaki durumda, ‘’t’’ ve ‘u’ artık ayrı nesnelerdir (yapı değerleri).

T alanları olarak herhangi bir referans tipi (dilim, harita, kanal) içermediğinden, yukarıdaki t ve u birbirini etkilemeden değiştirilebilir.

fmt.Printf("t.I = %d, u.I = %d\n", t.I, u.I) // t.I = 100, u.I = 1

Ancak, “T” bir referans türü içeriyorsa, örneğin:

type T struct {
    I  int
    S  string
    xs []int // a slice is a reference type
}

Ardından, atamaya göre basit bir kopyalama, dilim tipi alanının değerini de yeni nesneye kopyalayacaktır. Bu, aynı dilim nesnesine atıfta bulunan iki farklı nesneyle sonuçlanır.

// initialize a struct
t := T{I: 1, S: "one", xs: []int{1, 2, 3}}

// make struct copy
u := t // u has its field values equal to t

Hem u hem de t, kendi alanları xs aracılığıyla aynı dilime atıfta bulunduğundan, bir nesnenin dilimindeki bir değeri güncellemek, diğerindeki değişikliği yansıtacaktır.

// update a slice field in u
u.xs[1] = 500

fmt.Printf("t.xs = %d, u.xs = %d\n", t.xs, u.xs)
// t.xs = [1 500 3], u.xs = [1 500 3]

Bu nedenle, bu referans tipi özelliğinin istenmeyen davranışlara yol açmamasını sağlamak için ekstra özen gösterilmelidir.

Örneğin yukarıdaki nesneleri kopyalamak için dilim alanının açık bir kopyası gerçekleştirilebilir:

// explicitly initialize u's slice field
u.xs = make([]int, len(t.xs))
// copy the slice values over from t
copy(u.xs, t.xs)

// updating slice value in u will not affect t
u.xs[1] = 500

fmt.Printf("t.xs = %d, u.xs = %d\n", t.xs, u.xs)
// t.xs = [1 2 3], u.xs = [1 500 3]

Boş yapı #

Bir yapı, her biri bir ada ve bir türe sahip olan alanlar olarak adlandırılan adlandırılmış öğeler dizisidir. Boş yapı, bu anonim boş yapı gibi alan içermez:

var s struct{}

Veya bu boş yapı türü olarak adlandırılmış gibi:

type T struct{}

Boş yapıyla ilgili ilginç olan şey, boyutunun sıfır olmasıdır (The Go Playground deneyin):

fmt.Println(unsafe.Sizeof(s))

Bu ‘0’ yazdırır, bu nedenle boş yapının kendisi bellek almaz. bu nedenle, kanaldan çıkmak için iyi bir seçenektir, örneğin (The Go Playground‘i deneyin):

package main

import (
    "fmt"
    "time"
)

func main() {
    done := make(chan struct{})
    go func() {
        time.Sleep(1 * time.Second)
        close(done)
    }()

    fmt.Println("Wait...")
    <-done
    fmt.Println("done.")
}
