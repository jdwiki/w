Go, testleri ve kıyaslamaları çalıştırmak için gereken her şeye sahip kendi test tesisleriyle birlikte gelir. Diğer programlama dillerinin çoğundan farklı olarak, bazıları mevcut olmasına rağmen, genellikle ayrı bir test çerçevesine gerek yoktur.

Temel Test #

“main.go”:

package main

import (
    "fmt"
)

func main() {
    fmt.Println(Sum(4,5))
}

func Sum(a, b int) int {
    return a + b
}

“main_test.go”:

package main

import (
    "testing"
)

// Test methods start with `Test`
func TestSum(t *testing.T) {
    got := Sum(1, 2)
    want := 3
    if got != want {
        t.Errorf("Sum(1, 2) == %d, want %d", got, want)
    }
}

Testi çalıştırmak için sadece “go test” komutunu kullanın:

$ go test
ok      test_app    0.005s

Her testin sonuçlarını görmek için “-v” işaretini kullanın:

$ go test -v
=== RUN   TestSum
--- PASS: TestSum (0.00s)
PASS
ok      _/tmp    0.000s

Alt dizinleri tekrar tekrar test etmek için ./... yolunu kullanın:

$ go test -v ./...
ok      github.com/me/project/dir1    0.008s
=== RUN   TestSum
--- PASS: TestSum (0.00s)
PASS
ok      github.com/me/project/dir2    0.008s
=== RUN   TestDiff
--- PASS: TestDiff (0.00s)
PASS

Özel Bir Test Çalıştırın:
Birden fazla test varsa ve belirli bir testi çalıştırmak istiyorsanız, bu şu şekilde yapılabilir:

go test -v -run=<TestName> // will execute only test with this name

Örnek:

go test -v run=TestSum

Tabloya dayalı birim testleri #

Bu test türü, önceden tanımlanmış girdi ve çıktı değerleriyle test etmek için popüler bir tekniktir.

İçeriği olan “main.go” adlı bir dosya oluşturun:

package main

import (
    "fmt"
)

func main() {
    fmt.Println(Sum(4, 5))
}

func Sum(a, b int) int {
    return a + b
}

İle çalıştırdıktan sonra çıktının ‘9’ olduğunu göreceksiniz. Sum işlevi oldukça basit görünse de, kodunuzu test etmek iyi bir fikirdir. Bunu yapmak için, ‘main.go’ ile aynı klasörde ‘main_test.go’ adında aşağıdaki kodu içeren başka bir dosya oluşturuyoruz:

package main

import (
    "testing"
)

// Test methods start with Test
func TestSum(t *testing.T) {
    // Note that the data variable is of type array of anonymous struct,
    // which is very handy for writing table-driven unit tests.
    data := []struct {
        a, b, res int
    }{
        {1, 2, 3},
        {0, 0, 0},
        {1, -1, 0},
        {2, 3, 5},
        {1000, 234, 1234},
    }

    for _, d := range data {
        if got := Sum(d.a, d.b); got != d.res {
            t.Errorf("Sum(%d, %d) == %d, want %d", d.a, d.b, got, d.res)
        }
    }
}

Gördüğünüz gibi, her biri bir dizi girdiye ve beklenen sonuca sahip bir dizi anonim yapı oluşturulur. Bu, çok sayıda test senaryosunun hep birlikte tek bir yerde oluşturulmasına, ardından bir döngüde yürütülmesine, kod tekrarını azaltmasına ve netliği artırmasına olanak tanır.

Kurulum ve sökme işlevini kullanarak test etme #

Bir kurulum ve yırtma işlevi ayarlayabilirsiniz.

  • Bir kurulum işlevi, ortamınızı testlere hazırlar.
  • Bir yırtma işlevi geri alma yapar.

Bu, veritabanınızı değiştiremediğinizde ve veritabanından getirilen bir nesneyi simüle eden bir nesne oluşturmanız gerektiğinde veya her testte bir yapılandırma başlatmanız gerektiğinde iyi bir seçenektir.

Aptal bir örnek olacaktır:

// Standard numbers map
var numbers map[string]int = map[string]int{"zero": 0, "three": 3}

// TestMain will exec each test, one by one
func TestMain(m *testing.M) {
    // exec setUp function
    setUp("one", 1)
    // exec test and this returns an exit code to pass to os
    retCode := m.Run()
    // exec tearDown function
    tearDown("one")
    // If exit code is distinct of zero,
    // the test will be failed (red)
    os.Exit(retCode)
}

// setUp function, add a number to numbers slice
func setUp(key string, value int) {
    numbers[key] = value
}

// tearDown function, delete a number to numbers slice
func tearDown(key string) {
    delete(numbers, key)
}

// First test
func TestOnePlusOne(t *testing.T) {
    numbers["one"] = numbers["one"] + 1

    if numbers["one"] != 2 {
        t.Error("1 plus 1 = 2, not %v", value)
    }
}

// Second test
func TestOnePlusTwo(t *testing.T) {
    numbers["one"] = numbers["one"] + 2

    if numbers["one"] != 3 {
        t.Error("1 plus 2 = 3, not %v", value)
    }
}

Diğer bir örnek, test etmek ve geri almak için veritabanı hazırlamak olacaktır.

 // ID of Person will be saved in database
personID := 12345
// Name of Person will be saved in database
personName := "Toni"

func TestMain(m *testing.M) {
    // You create an Person and you save in database
    setUp(&Person{
            ID:   personID,
            Name: personName,
            Age:  19,
        })
    retCode := m.Run()
    // When you have executed the test, the Person is deleted from database
    tearDown(personID)
    os.Exit(retCode)
}

func setUp(P *Person) {
    // ...
    db.add(P)
    // ...
}

func tearDown(id int) {
    // ...
    db.delete(id)
    // ...
}

func getPerson(t *testing.T) {
    P := Get(personID)
    
    if P.Name != personName {
        t.Error("P.Name is %s and it must be Toni", P.Name)
    }
}

Karşılaştırma testleri #

Karşılaştırmaları ölçmek istiyorsanız, bunun gibi bir test yöntemi ekleyin:

sum.go:

package sum

// Sum calculates the sum of two integers
func Sum(a, b int) int {
    return a + b
}

“sum_test.go”:

package sum

import "testing"

func BenchmarkSum(b *testing.B) {
    for i := 0; i < b.N; i++ {
        _ = Sum(2, 3)
    }
}

Ardından basit bir kıyaslama yapmak için:

$ go test -bench=. 
BenchmarkSum-8    2000000000             0.49 ns/op
ok      so/sum    1.027s

Örnek testler (kendi kendini belgeleyen testler) #

Bu tür testler, kodunuzun düzgün bir şekilde derlendiğinden ve projeniz için oluşturulan belgelerde görüneceğinden emin olur. Buna ek olarak, örnek testler, testinizin uygun çıktı ürettiğini iddia edebilir.

sum.go:

package sum

// Sum calculates the sum of two integers
func Sum(a, b int) int {
    return a + b
}

“sum_test.go”:

package sum

import "fmt"

func ExampleSum() {
    x := Sum(1, 2)
    fmt.Println(x)
    fmt.Println(Sum(-1, -1))
    fmt.Println(Sum(0, 0))

    // Output:
    // 3
    // -2
    // 0
}

Testinizi yürütmek için, bu dosyaları içeren klasörde “go test"i çalıştırın VEYA bu iki dosyayı “sum” adlı bir alt klasöre koyun ve ardından ana klasörden “go test ./sum” komutunu çalıştırın. Her iki durumda da şuna benzer bir çıktı alacaksınız:

ok      so/sum    0.005s

Bunun kodunuzu nasıl test ettiğini merak ediyorsanız, işte gerçekten testi geçemeyen başka bir örnek fonksiyon:

func ExampleSum_fail() {
    x := Sum(1, 2)
    fmt.Println(x)

    // Output:
    // 5
}

“go testi” çalıştırdığınızda, aşağıdaki çıktıyı alırsınız:

$ go test
--- FAIL: ExampleSum_fail (0.00s)
got:
3
want:
5
FAIL
exit status 1
FAIL    so/sum    0.006s

sum paketinizin belgelerini görmek istiyorsanız – şunu çalıştırın:

go doc -http=:6060

ve http://localhost:6060/pkg/FOLDER/sum/ konumuna gidin; burada FOLDER, “sum” paketini içeren klasördür (bu örnekte “so”). Toplam yönteminin belgeleri şöyle görünür:

HTTP isteklerini test etme #

ana.go:

package main

import (
    "fmt"
    "io/ioutil"
    "log"
    "net/http"
)

func fetchContent(url string) (string, error) {
    res, err := http.Get(url)
    if err != nil {
        return "", nil
    }
    defer res.Body.Close()

    body, err := ioutil.ReadAll(res.Body)
    if err != nil {
        return "", err
    }
    return string(body), nil
}

func main() {
    url := "https://example.com/"
    content, err := fetchContent(url)
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println("Content:", content)
}

main_test.go:

package main

import (
    "fmt"
    "net/http"
    "net/http/httptest"
    "testing"
)

func Test_fetchContent(t *testing.T) {
    ts := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprint(w, "hello world")
    }))
    defer ts.Close()

    content, err := fetchContent(ts.URL)
    if err != nil {
        t.Error(err)
    }

    want := "hello world"
    if content != want {
        t.Errorf("Got %q, want %q", content, want)
    }
}

Testlerde Sahte İşlevi Ayarla/Sıfırla #

Bu örnek, birim testimizle ilgisi olmayan bir işlev çağrısının nasıl alay edileceğini ve ardından alaylı işlev çağrısını orijinal işlevine yeniden atamak için ’erteleme’ ifadesinin nasıl kullanılacağını gösterir.

var validate = validateDTD

// ParseXML parses b for XML elements and values, and returns them as a map of 
// string key/value pairs.
func ParseXML(b []byte) (map[string]string, error) {
    // we don't care about validating against DTD in our unit test
    if err := validate(b); err != nil {
        return err
    }

    // code to parse b etc.
}

func validateDTD(b []byte) error {
    // get the DTD from some external storage, use it to validate b etc.
}

Birim testimizde,

func TestParseXML(t *testing.T) {
    // assign the original validate function to a variable.
    originalValidate = validate
    // use the mockValidate function in this test.
    validate = mockValidate
    // defer the re-assignment back to the original validate function.
    defer func() {
       validate = originalValidate
    }()

    var input []byte
    actual, err := ParseXML(input)
    // assertion etc.
}

func mockValidate(b []byte) error {
    return nil // always return nil since we don't care
}

Kod kapsamını HTML biçiminde görüntüleyin #

“Git testi"ni normal şekilde, ancak “kapak profili” bayrağıyla çalıştırın. Ardından sonuçları HTML olarak görüntülemek için go toolu kullanın.

    go test -coverprofile=c.out
    go tool cover -html=c.out
