Go’daki işlevler, bir dizi eylemi gerçekleştirmek için düzenli, yeniden kullanılabilir kod sağlar. Fonksiyonlar kodlama sürecini basitleştirir, gereksiz mantığı önler ve kodun takip edilmesini kolaylaştırır. Bu konu, Go’da işlevlerin, bağımsız değişkenlerin, parametrelerin, dönüş ifadelerinin ve kapsamların bildirimini ve kullanımını açıklar.

Sözdizimi #

  • func() // argümansız ve dönüş değeri olmayan işlev türü
  • func(x int) int // tamsayı kabul eder ve bir tamsayı döndürür
  • func(a, b int, z float32) bool // 2 tamsayı, bir kayan nokta kabul eder ve bir boole döndürür
  • func(önek dizesi, değerler …int) // bir dize ve bir veya daha fazla sayıda tamsayı kabul eden “değişken” işlev
  • func() (int, bool) // iki değer döndüren fonksiyon
  • func(a, b int, z float64, opt …interface{}) (success bool) // 2 tamsayı, bir kayan nokta ve bir veya daha fazla sayıda arabirim kabul eder ve boolean değerini döndürür (zaten içinde başlatılmış olan) işlev)

Değişmez işlevler ve kapanışlar #

stdout’a Merhaba! yazdıran basit bir değişmez işlev:

package main

import "fmt"

func main() {
    func(){
        fmt.Println("Hello!")
    }()
}

oyun alanında oynayın

‘str’ bağımsız değişkenini stdout’a yazdıran değişmez bir işlev:

package main

import "fmt"

func main() {
    func(str string) {
        fmt.Println(str)
    }("Hello!")
}

oyun alanında oynayın

‘str’ değişkenini kapatan değişmez bir işlev:

package main

import "fmt"

func main() {
    str := "Hello!"
    func() {
        fmt.Println(str)
    }()
}

oyun alanında oynayın

Bir değişkene değişmez bir işlev atamak mümkündür:

package main

import (
    "fmt"
)

func main() {
    str := "Hello!"
    anon := func() {
        fmt.Println(str)
    }
    anon()
}

oyun alanında oynayın

Temel Beyanname #

Herhangi bir parametre kabul etmeyen ve herhangi bir değer döndürmeyen basit bir fonksiyon:

func SayHello() {
    fmt.Println("Hello!")
}

Adlandırılmış Dönüş Değerleri #

Dönen değerler yerel bir değişkene atanabilir. Boş bir “dönüş” ifadesi daha sonra mevcut değerlerini döndürmek için kullanılabilir. Bu “çıplak” dönüş olarak bilinir. Naked return deyimleri, daha uzun fonksiyonlarda okunabilirliğe zarar verdiği için sadece kısa fonksiyonlarda kullanılmalıdır:

func Inverse(v float32) (reciprocal float32) {
    if v == 0 {
        return
    }
    reciprocal = 1 / v
    return
}

oyun alanında oynayın

//A function can also return multiple values
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return
}

oyun alanında oynayın

İki önemli şeye dikkat edilmelidir:

  • Dönüş değerlerinin etrafındaki parantezler zorunludur.
  • Her zaman boş bir “dönüş” ifadesi sağlanmalıdır.

Değişken fonksiyonlar #

Değişken bir işlev, herhangi bir sayıda sondaki bağımsız değişkenle çağrılabilir. Bu öğeler bir dilimde saklanır.

package main

import "fmt"

func variadic(strs ...string) {
     // strs is a slice of string
     for i, str := range strs {
         fmt.Printf("%d: %s\n", i, str)
     }
}

func main() {
     variadic("Hello", "Goodbye")
     variadic("Str1", "Str2", "Str3")
}

oyun alanında oynayın

Değişken bir işleve ... ile bir dilim de verebilirsiniz:

func main() {
     strs := []string {"Str1", "Str2", "Str3"}

     variadic(strs...)
}

oyun alanında oynayın

Dönüş Değerleri #

Bir işlev, arayana bir veya daha fazla değer döndürebilir:

func AddAndMultiply(a, b int) (int, int) {
    return a+b, a*b
}

İkinci dönüş değeri de var hatası olabilir:

import errors

func Divide(dividend, divisor int) (int, error) {
    if divisor == 0 {
        return 0, errors.New("Division by zero forbidden")
    }
    return dividend / divisor, nil
}

İki önemli şeye dikkat edilmelidir:

  • Tek bir dönüş değeri için parantez kullanılmayabilir.
  • Her return ifadesi, tüm beyan edilen dönüş değerleri için bir değer sağlamalıdır.

Parametreler #

Bir işlev isteğe bağlı olarak bir dizi parametre bildirebilir:

func SayHelloToMe(firstName, lastName string, age int) {
    fmt.Printf("Hello, %s %s!\n", firstName, lastName)
    fmt.Printf("You are %d", age)
}

“LastName” ile aynı olduğundan “firstName” türünün atlandığına dikkat edin.
