image paketi, 2 boyutlu görüntü ile çalışmak için temel işlevler sağlar. Bu konu, belirli bir görüntü biçimini okuma ve yazma, kırpma, piksele erişme ve değiştirme, renk dönüştürme, yeniden boyutlandırma ve temel görüntü filtreleme gibi görüntüyle çalışırken birkaç temel işlemi açıklar.

Temel konseptler #

Bir görüntü, resim öğelerinin (piksel) dikdörtgen bir ızgarasını temsil eder. image paketinde piksel, image/color içinde tanımlanan renklerden biri olarak temsil edilir. renk/) paketi. Resmin 2 boyutlu geometrisi image.Rectangle, [image.Point](https://golang. org/pkg/image/#Point) ızgaradaki bir konumu belirtir.

Yukarıdaki şekil, paketteki bir görüntünün temel kavramlarını göstermektedir. 15x14 piksel boyutundaki bir görüntü, sol üst köşeden başlayan dikdörtgen bir sınırlara sahiptir (örneğin, yukarıdaki şekilde koordinat (-3, -4)) ve eksenleri sağa ve aşağıya doğru sağ alt köşeye doğru artar (örn. şekilde koordinat (12, 10)). Sınırların ** mutlaka (0,0)** noktasından başlamadığına veya nokta içermediğine dikkat edin.

Resimle ilgili tür #

“Go"da, bir resim her zaman aşağıdaki image.Image arayüzünü uygular

type Image interface {
    // ColorModel returns the Image's color model.
    ColorModel() color.Model
    // Bounds returns the domain for which At can return non-zero color.
    // The bounds do not necessarily contain the point (0, 0).
    Bounds() Rectangle
    // At returns the color of the pixel at (x, y).
    // At(Bounds().Min.X, Bounds().Min.Y) returns the upper-left pixel of the grid.
    // At(Bounds().Max.X-1, Bounds().Max.Y-1) returns the lower-right one.
    At(x, y int) color.Color
}

color.Color arayüzünün şu şekilde tanımlandığı

type Color interface {
    // RGBA returns the alpha-premultiplied red, green, blue and alpha values
    // for the color. Each value ranges within [0, 0xffff], but is represented
    // by a uint32 so that multiplying by a blend factor up to 0xffff will not
    // overflow.
    //
    // An alpha-premultiplied color component c has been scaled by alpha (a),
    // so has valid values 0 <= c <= a.
    RGBA() (r, g, b, a uint32)
}

ve color.Model olarak bildirilen bir arayüzdür

type Model interface {
    Convert(c Color) Color
}

Görüntü boyutuna ve piksele erişme #

“img” değişkeni olarak depolanmış bir görüntümüz olduğunu varsayalım, o zaman boyut ve görüntü pikselini şu şekilde elde edebiliriz:

// Image bounds and dimension
b := img.Bounds()
width, height := b.Dx(), b.Dy()
// do something with dimension ...

// Corner co-ordinates
top := b.Min.Y
left := b.Min.X
bottom := b.Max.Y
right := b.Max.X
    
// Accessing pixel. The (x,y) position must be
// started from (left, top) position not (0,0)
for y := top; y < bottom; y++ {
    for x := left; x < right; x++ {
        cl := img.At(x, y)
        r, g, b, a := cl.RGBA()
        // do something with r,g,b,a color component
    }
}

Paketteki her ‘R,G,B,A’ bileşeninin değerinin ‘0-65535’ (‘0x0000 - 0xffff’) arasında olduğuna, ‘0-255’ değil olduğuna dikkat edin.

Görüntü yükleme ve kaydetme #

Bellekte bir görüntü bir piksel (renk) matrisi olarak görülebilir. Ancak, bir görüntü kalıcı bir depolamada depolandığında, olduğu gibi (RAW formatı), Bitmap veya belirli sıkıştırma algoritması olan diğer görüntü formatları olarak saklanabilir. depolama alanından tasarruf, örn. PNG, JPEG, GIF, vb. Belirli bir formatta bir görüntü yüklerken, görüntünün ilgili algoritma ile image.Image olarak kodunun çözülmesi gerekir. Olarak bildirilen bir image.Decode işlevi

func Decode(r io.Reader) (Image, string, error)

bu özel kullanım için sağlanmıştır. Çeşitli görüntü biçimlerini işleyebilmek için, “image.Decode” işlevini çağırmadan önce, kod çözücünün image.RegisterFormat aracılığıyla kaydedilmesi gerekir. ) fonksiyon olarak tanımlanır

func RegisterFormat(name, magic string, 
    decode func(io.Reader) (Image, error), decodeConfig func(io.Reader) (Config, error))

Şu anda resim paketi üç dosya biçimini desteklemektedir: JPEG, GIF ve PNG. Bir kod çözücüyü kaydetmek için aşağıdakileri ekleyin

import _ "image/jpeg"    //register JPEG decoder

uygulamanın ana paketine. Kodunuzda bir yerde (‘ana’ pakette gerekli değildir), bir JPEG görüntüsü yüklemek için aşağıdaki parçacıkları kullanın:

f, err := os.Open("inputimage.jpg")
if err != nil {
    // Handle error
}
defer f.Close()

img, fmtName, err := image.Decode(f)
if err != nil {
    // Handle error
}

// `fmtName` contains the name used during format registration
// Work with `img` ...

PNG’ye kaydet #

Bir görüntüyü belirli bir formatta kaydetmek için karşılık gelen kodlayıcı açıkça içe aktarılmalıdır, yani.

import "image/png"    //needed to use `png` encoder

daha sonra aşağıdaki snippet’lerle bir görüntü kaydedilebilir:

f, err := os.Create("outimage.png")
if err != nil {
    // Handle error
}
defer f.Close()

// Encode to `PNG` with `DefaultCompression` level
// then save to file
err = png.Encode(f, img)
if err != nil {
    // Handle error
}

DefaultCompression seviyesi dışında bir sıkıştırma seviyesi belirtmek istiyorsanız, bir kodlayıcı oluşturun, ör.

enc := png.Encoder{
    CompressionLevel: png.BestSpeed,
}
err := enc.Encode(f, img)

JPEG’e kaydet #

Resmi kırpma #

image paketindeki çoğu resim türü, “image.Uniform” dışında “SubImage(r Rectangle) Image” yöntemine sahiptir. Bu gerçeğe dayanarak, keyfi bir görüntüyü kırpmak için aşağıdaki gibi bir işlev uygulayabiliriz.

func CropImage(img image.Image, cropRect image.Rectangle) (cropImg image.Image, newImg bool) {
    //Interface for asserting whether `img`
    //implements SubImage or not.
    //This can be defined globally.
    type CropableImage interface {
        image.Image
        SubImage(r image.Rectangle) image.Image
    }

    if p, ok := img.(CropableImage); ok {
        // Call SubImage. This should be fast,
        // since SubImage (usually) shares underlying pixel.
        cropImg = p.SubImage(cropRect)
    } else if cropRect = cropRect.Intersect(img.Bounds()); !cropRect.Empty() {
        // If `img` does not implement `SubImage`,
        // copy (and silently convert) the image portion to RGBA image.
        rgbaImg := image.NewRGBA(cropRect)
        for y := cropRect.Min.Y; y < cropRect.Max.Y; y++ {
            for x := cropRect.Min.X; x < cropRect.Max.X; x++ {
                rgbaImg.Set(x, y, img.At(x, y))
            }
        }
        cropImg = rgbaImg
        newImg = true
    } else {
        // Return an empty RGBA image
        cropImg = &image.RGBA{}
        newImg = true
    }

    return cropImg, newImg
}

Kırpılan görüntünün, temel piksellerini orijinal görüntüyle paylaşabileceğini unutmayın. Bu durumda, kırpılan görüntüde yapılacak herhangi bir değişiklik orijinal görüntüyü etkileyecektir.

Renkli görüntüyü gri tonlamaya dönüştürün #

Kenar algılama gibi bazı dijital görüntü işleme algoritmaları, görüntü yoğunluğu (yani gri tonlama değeri) tarafından taşınan bilgiler yeterlidir. Renk bilgisinin kullanılması (R, G, B kanalı) biraz daha iyi sonuç sağlayabilir, ancak algoritma karmaşıklığı artacaktır. Bu nedenle, bu durumda, böyle bir algoritmayı uygulamadan önce renkli görüntüyü gri tonlu görüntüye dönüştürmemiz gerekir.

Aşağıdaki kod, rastgele görüntüyü 8 bit gri tonlamalı görüntüye dönüştürmenin bir örneğidir. Görüntü, ’net/http’ paketi kullanılarak uzak konumdan alınır, gri tonlamaya dönüştürülür ve son olarak PNG görüntüsü olarak kaydedilir.

package main

import (
    "image"
    "log"
    "net/http"
    "os"

    _ "image/jpeg"
    "image/png"
)

func main() {
    // Load image from remote through http
    // The Go gopher was designed by Renee French. (http://reneefrench.blogspot.com/)
    // Images are available under the Creative Commons 3.0 Attributions license.
    resp, err := http.Get("http://golang.org/doc/gopher/fiveyears.jpg")
    if err != nil {
        // handle error
        log.Fatal(err)
    }
    defer resp.Body.Close()

    // Decode image to JPEG
    img, _, err := image.Decode(resp.Body)
    if err != nil {
        // handle error
        log.Fatal(err)
    }
    log.Printf("Image type: %T", img)

    // Converting image to grayscale
    grayImg := image.NewGray(img.Bounds())
    for y := img.Bounds().Min.Y; y < img.Bounds().Max.Y; y++ {
        for x := img.Bounds().Min.X; x < img.Bounds().Max.X; x++ {
            grayImg.Set(x, y, img.At(x, y))
        }
    }

    // Working with grayscale image, e.g. convert to png
    f, err := os.Create("fiveyears_gray.png")
    if err != nil {
        // handle error
        log.Fatal(err)
    }
    defer f.Close()

    if err := png.Encode(f, grayImg); err != nil {
        log.Fatal(err)
    }
}

image.go içinde uygulanan Set(x, y int, c color.Color) aracılığıyla piksel atanırken renk dönüşümü gerçekleşir. =19292:19335#L691) olarak

func (p *Gray) Set(x, y int, c color.Color) {
    if !(Point{x, y}.In(p.Rect)) {
        return
    }

    i := p.PixOffset(x, y)
    p.Pix[i] = color.GrayModel.Convert(c).(color.Gray).Y
}

burada, “color.GrayModel”, “color.go” dosyasında şu şekilde tanımlanır:

func grayModel(c Color) Color {
    if _, ok := c.(Gray); ok {
        return c
    }
    r, g, b, _ := c.RGBA()

    // These coefficients (the fractions 0.299, 0.587 and 0.114) are the same
    // as those given by the JFIF specification and used by func RGBToYCbCr in
    // ycbcr.go.
    //
    // Note that 19595 + 38470 + 7471 equals 65536.
    //
    // The 24 is 16 + 8. The 16 is the same as used in RGBToYCbCr. The 8 is
    // because the return value is 8 bit color, not 16 bit color.
    y := (19595*r + 38470*g + 7471*b + 1<<15) >> 24

    return Gray{uint8(y)}
}

Yukarıdaki gerçeklere dayanarak, ‘Y’ yoğunluğu aşağıdaki formülle hesaplanır:

Parlaklık: Y = 0,299R + 0,587G + 0,114B

Bir rengi yoğunluğa dönüştürmek için farklı formül/algoritmalar uygulamak istiyorsak, ör.

Ortalama: Y = (R + G + B) / 3 Luma: Y = 0.2126R + 0.7152G + 0.0722B Parlaklık: Y = (min(R, G, B) + maks(R, G, B))/2

daha sonra aşağıdaki snippet’ler kullanılabilir.

// Converting image to grayscale
grayImg := image.NewGray(img.Bounds())
for y := img.Bounds().Min.Y; y < img.Bounds().Max.Y; y++ {
    for x := img.Bounds().Min.X; x < img.Bounds().Max.X; x++ {
        R, G, B, _ := img.At(x, y).RGBA()
        //Luma: Y = 0.2126*R + 0.7152*G + 0.0722*B
        Y := (0.2126*float64(R) + 0.7152*float64(G) + 0.0722*float64(B)) * (255.0 / 65535)
        grayPix := color.Gray{uint8(Y)}
        grayImg.Set(x, y, grayPix)
    }
}

Yukarıdaki hesaplama kayan nokta çarpımı ile yapılır ve kesinlikle en verimli olanı değildir, ancak fikri göstermek için yeterlidir. Diğer bir nokta ise, üçüncü argüman olarak color.Gray ile Set(x, y int, c color.Color) çağrıldığında, önceki ‘grayModel’ fonksiyonunda görüldüğü gibi renk modeli renk dönüşümü gerçekleştirmeyecektir. .
