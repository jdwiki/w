Dilim, programcının bellek yönetimi konusunda endişelenmeden gerektiği kadar öğe ekleyebilmesi için bir diziyi içine alan bir veri yapısıdır. Elde edilen dilimlerin tümü aynı dahili diziye işaret ettiğinden, dilimler çok verimli bir şekilde alt dilimlere kesilebilir. Go programcıları, genellikle diğer birçok programlama dilinde yapılacak olan dizileri kopyalamaktan kaçınmak için bundan yararlanır.

Sözdizimi #

  • dilim := make([]type, len, cap) // yeni bir dilim yarat
  • dilim = ekle(dilim, öğe) // bir dilime bir öğe ekle
  • dilim = append(dilim, öğeler…) // bir dilime öğe dilimi ekle
  • len := len(dilim) // bir dilimin uzunluğunu al
  • cap := cap(dilim) // bir dilimin kapasitesini al
  • elNum := kopyala(dst, dilim) // bir dilimin içeriğini başka bir dilime kopyala

Birlikte İki Dilim Ekleme #

slice1 := []string{"!"}
slice2 := []string{"Hello", "world"}
slice  := append(slice1, slice2...)

Go Playground’da Koş

Uzunluk ve Kapasite #

Dilimlerin hem uzunluğu hem de kapasitesi vardır. Bir dilimin uzunluğu, dilimdeki şu anda olan öğelerin sayısıdır, kapasite ise dilimin yeniden tahsis edilmeden önce tutabileceği öğelerin sayısıdır.

Yerleşik ‘make()’ işlevini kullanarak bir dilim oluştururken, uzunluğunu ve isteğe bağlı olarak kapasitesini belirtebilirsiniz. Kapasite açıkça belirtilmemişse, belirtilen uzunluk olacaktır.

var s = make([]int, 3, 5) // length 3, capacity 5

Bir dilimin uzunluğunu yerleşik len() işleviyle kontrol edebilirsiniz:

var n = len(s) // n == 3

Kapasiteyi yerleşik cap() işleviyle kontrol edebilirsiniz:

var c = cap(s) // c == 5

“make()” tarafından oluşturulan öğeler, dilimin öğe türü için sıfır değerine ayarlanır:

for idx, val := range s {
    fmt.Println(idx, val)
}
// output:
// 0 0
// 1 0
// 2 0

play.golang.org’da çalıştırın

Dizin kapasite dahilinde olsa bile, bir dilim uzunluğunun ötesindeki öğelere erişemezsiniz:

var x = s[3] // panic: runtime error: index out of range

Ancak, kapasite uzunluğu aştığı sürece, yeniden tahsis etmeden yeni öğeler ekleyebilirsiniz:

var t = []int{3, 4}
s = append(s, t) // s is now []int{0, 0, 0, 3, 4}
n = len(s) // n == 5
c = cap(s) // c == 5

Yeni öğeleri kabul etme kapasitesi olmayan bir dilime eklerseniz, temel dizi sizin için yeterli kapasiteyle yeniden tahsis edilecektir:

var u = []int{5, 6}
s = append(s, u) // s is now []int{0, 0, 0, 3, 4, 5, 6}
n = len(s) // n == 7
c = cap(s) // c > 5

Bu nedenle, ilk kez bir dilim oluştururken, ne kadar alana ihtiyacınız olacağını biliyorsanız, gereksiz yeniden tahsislerden kaçınmak için yeterli kapasiteyi tahsis etmek genellikle iyi bir uygulamadır.

Öğeleri kaldırma / “Dilimleme” dilimleri #

Bir dilimden bir veya daha fazla öğeyi çıkarmanız gerekiyorsa veya mevcut başka bir alt dilimle çalışmanız gerekiyorsa; aşağıdaki yöntemi kullanabilirsiniz.

Aşağıdaki örnekler, int dilimini kullanır, ancak bu her tür dilimle çalışır.

Bunun için bir dilime ihtiyacımız var, cadıdan bazı unsurları çıkaracağız:

slice := []int{1, 2, 3, 4, 5, 6} 
// > [1 2 3 4 5 6]

Kaldırmak için öğelerin dizinlerine de ihtiyacımız var:

// index of first element to remove (corresponding to the '3' in the slice)
var first = 2

// index of last element to remove (corresponding to the '5' in the slice)
var last = 4

Ve böylece istenmeyen öğeleri kaldırarak dilimi “dilimleyebiliriz”:

// keeping elements from start to 'first element to remove' (not keeping first to remove), 
// removing elements from 'first element to remove' to 'last element to remove'
// and keeping all others elements to the end of the slice
newSlice1 := append(slice[:first], slice[last+1:]...)
// > [1 2 6]

// you can do using directly numbers instead of variables
newSlice2 := append(slice[:2], slice[5:]...)
// > [1 2 6]

// Another way to do the same
newSlice3 := slice[:first + copy(slice[first:], slice[last+1:])]
// > [1 2 6]

// same that newSlice3 with hard coded indexes (without use of variables)
newSlice4 := slice[:2 + copy(slice[2:], slice[5:])]
// > [1 2 6]

Yalnızca bir öğeyi kaldırmak için, bu öğenin dizinini kaldırılacak ilk VE son dizin olarak koymanız yeterlidir, aynen şöyle:

var indexToRemove = 3
newSlice5 := append(slice[:indexToRemove], slice[indexToRemove+1:]...)
// > [1 2 3 5 6]

// hard-coded version:
newSlice5 := append(slice[:3], slice[4:]...)
// > [1 2 3 5 6]

Ayrıca, öğeleri dilimin başından da kaldırabilirsiniz:

newSlice6 := append(slice[:0], slice[last+1:]...)
// > [6]

// That can be simplified into
newSlice6 := slice[last+1:]
// > [6]

Ayrıca dilimin sonundan bazı öğeleri kaldırabilirsiniz:

newSlice7 := append(slice[:first], slice[first+1:len(slice)-1]...)
// > [1 2]

// That can be simplified into
newSlice7 := slice[:first]
// > [1 2]

Yeni dilimin ilkinden tam olarak aynı öğeleri içermesi gerekiyorsa, aynı şeyi ’last := first-1’ ile kullanabilirsiniz. (Bu, dizinlerinizin önceden hesaplanmış olması durumunda faydalı olabilir)

Dilimler Oluşturma #

Dilimler, programcıların veri listelerini depolamasının tipik yoludur.

Bir dilim değişkeni bildirmek için `[]Type’ sözdizimini kullanın.

var a []int

Bir satırda bir dilim değişkeni bildirmek ve başlatmak için []Tür{değerler} sözdizimini kullanın.

var a []int = []int{3, 1, 4, 1, 5, 9}

Bir dilimi başlatmanın başka bir yolu da “make” işlevidir. Üç bağımsız değişken: dilimin “Türü” (veya harita), “uzunluk” ve “kapasite”.

a := make([]int, 0, 5)

“append"i kullanarak yeni diliminize öğeler ekleyebilirsiniz.

a = append(a, 5)

’len’ kullanarak diliminizdeki eleman sayısını kontrol edin.

length := len(a)

cap kullanarak diliminizin kapasitesini kontrol edin. Kapasite, dilim için şu anda bellekte olmak üzere ayrılmış öğelerin sayısıdır. Go sizin için otomatik olarak daha büyük bir dilim oluşturacağından, her zaman bir dilime kapasitede ekleme yapabilirsiniz.

capacity := cap(a)

Tipik dizin oluşturma sözdizimini kullanarak bir dilimdeki öğelere erişebilirsiniz.

a[0]  // Gets the first member of `a`

Ayrıca, “aralık” içeren dilimler üzerinde bir “for” döngüsü kullanabilirsiniz. İlk değişken, belirtilen dizideki dizindir ve ikinci değişken, dizin değeridir.

for index, value := range a {
    fmt.Println("Index: " + index + " Value: " + value)  // Prints "Index: 0 Value: 5" (and continues until end of slice)
}

Bir dilimi filtreleme #

Yeni bir temel dizi ayırmadan bir dilimi filtrelemek için:

// Our base slice
slice := []int{ 1, 2, 3, 4 }
// Create a zero-length slice with the same underlying array
tmp := slice[:0]

for _, v := range slice {
  if v % 2 == 0 {
    // Append desired values to slice
    tmp = append(tmp, v)
  }
}

// (Optional) Reassign the slice
slice = tmp // [2, 4]

Dilimin sıfır değeri #

Dilimin sıfır değeri, uzunluğu ve kapasitesi “0” olan “nil"dir. Bir “nil” dilimin altında yatan bir dizi yoktur. Ancak, ‘[]int{}’ veya ‘make([]int, 5)[5:]’ gibi sıfır olmayan uzunluk ve kapasite ‘0’ dilimleri de vardır.

Sıfır değeri olan herhangi bir tür, “nil” dilimine dönüştürülebilir:

s = []int(nil)

Bir dilimin boş olup olmadığını test etmek için şunu kullanın:

if len(s) == 0　{
    fmt.Ptintf("s is empty.")
}

Dilime ekleme #

slice = append(slice, "hello", "world")

İçeriği bir dilimden diğerine kopyalama #

Bir dilimin içeriğini başlangıçta boş bir dilime kopyalamak isterseniz, bunu gerçekleştirmek için aşağıdaki adımlar izlenebilir:

  1. Kaynak dilimi oluşturun:

    var sourceSlice []interface{} = []interface{}{“Hello”,5.10,“World”,true}

  2. Hedef dilimi aşağıdakilerle oluşturun:

  • Uzunluk = SourceSlice’ın uzunluğu

    var destinationSlice []interface{} = make([]interface{},len(sourceSlice))

  1. Artık hedef dilimin temel dizisi, kaynak dilimin tüm öğelerini barındıracak kadar büyük olduğuna göre, yerleşik “kopya"yı kullanarak öğeleri kopyalamaya devam edebiliriz:

    copy(destinationSlice,sourceSlice)
