go komutu, Go geliştirmesinin yönetimine izin veren bir komut satırı programıdır. Kod oluşturma, çalıştırma ve test etmenin yanı sıra Go ile ilgili çeşitli diğer görevleri sağlar.

Fmt’ye Git #

go fmt, bir programın kaynak kodunu, okunması ve anlaşılması kolay, düzgün, deyimsel bir şekilde biçimlendirir. Okumayı kolaylaştırmak için herhangi bir kaynakta genel görüntüleme için göndermeden veya bir sürüm kontrol sistemine bağlanmadan önce “go fmt"yi kullanmanız önerilir.

Bir dosyayı biçimlendirmek için:

go fmt main.go

Veya bir dizindeki tüm dosyalar:

go fmt myProject

Ayrıca, herhangi bir kodu basitleştirmeye çalışmak için “gofmt -s” (not “go fmt”) kullanabilirsiniz.

“gofmt” (not “go fmt”) kodu yeniden düzenlemek için de kullanılabilir. Go’yu anlar, bu nedenle basit bir arama ve değiştirme kullanmaktan daha güçlüdür. Örneğin, bu program (main.go) verildiğinde:

package main

type Example struct {
    Name string
}

func (e *Example) Original(name string) {
    e.Name = name
}

func main() {
    e := &Example{"Hello"}
    e.Original("Goodbye")
}

“Orijinal” yöntemini “Refactor” ile “gofmt” ile değiştirebilirsiniz:

gofmt -r 'Original -> Refactor' -d main.go

Hangi farkı üretecek:

-func (e *Example) Original(name string) {
+func (e *Example) Refactor(name string) {
     e.Name = name
 }
 
 func main() {
     e := &Example{"Hello"}
-    e.Original("Goodbye")
+    e.Refactor("Goodbye")
 }

Git koş #

“go run”, yürütülebilir bir dosya oluşturmadan bir programı çalıştırır. Çoğunlukla geliştirme için yararlıdır. “run” yalnızca paket adı ana olan paketleri çalıştırır.

Göstermek için basit bir Merhaba Dünya örneği “main.go” kullanacağız:

package main

import fmt

func main() {
    fmt.Println("Hello, World!")
}

Bir dosyaya derlemeden yürütün:

go run main.go

Çıktı:

Hello, World!

Pakette birden fazla dosya çalıştırın #

Paket main ise ve birden çok dosyaya bölünmüşse, diğer dosyaların ‘run’ komutuna dahil edilmesi gerekir:

go run main.go assets.go

İnşa Et #

“go build”, bir programı yürütülebilir bir dosyada derleyecektir.

Göstermek için basit bir Merhaba Dünya örneği kullanacağız main.go:

package main

import fmt
    
func main() {
    fmt.Println("Hello, World!")
}

Programı derleyin:

go build main.go

“build” yürütülebilir bir program oluşturur, bu durumda: “main” veya “main.exe”. Daha sonra Merhaba Dünya! çıktısını görmek için bu dosyayı çalıştırabilirsiniz. Ayrıca Go’nun kurulu olmadığı benzer bir sisteme kopyalayabilir, çalıştırılabilir hale getirebilir ve orada çalıştırabilirsiniz.

Derlemede İşletim Sistemini veya Mimariyi belirtin: #

“build"den önce “env"yi değiştirerek hangi sistemin veya mimarinin oluşturulacağını belirtebilirsiniz:

env GOOS=linux go build main.go # builds for Linux
env GOARCH=arm go build main.go # builds for ARM architecture

Birden çok dosya oluşturun #

Paketiniz birden çok dosyaya bölünmüşse ve paket adı ana ise (yani içe aktarılabilir bir paket değil), oluşturulacak tüm dosyaları belirtmelisiniz:

go build main.go assets.go # outputs an executable: main

Paket oluşturma #

“main” adlı bir paket oluşturmak için şunları kullanabilirsiniz:

go build . # outputs an executable with name as the name of enclosing folder

Temizle #

“go clean”, bir programda “go build” çağrılırken oluşturulan tüm geçici dosyaları temizler. Ayrıca Makefiles’den kalan dosyaları da temizleyecektir.

Git al #

go get, içe aktarma yollarıyla adlandırılan paketleri indirir. bağımlılıklar. Daha sonra ‘go install’ gibi adlandırılmış paketleri kurar. Get ayrıca kurulumu kontrol etmek için inşa bayraklarını da kabul eder.

github.com/maknahar/phonecountry adresini edinin

Yeni bir paketi teslim alırken get, hedef dizini oluşturur “$GOPATH/src/”. GOPATH birden fazla giriş içeriyorsa, get ilkini kullanır. Benzer şekilde, derlenmiş ikili dosyaları $GOPATH/bin içine kuracaktır.

Bir paketi teslim alırken veya güncellerken, bir dal veya etiket arayın Go’nun yerel olarak yüklenmiş sürümüyle eşleşen. En önemli kural şudur ki, yerel kurulum “go1” sürümünü çalıştırıyorsa, “go1” adlı bir dal veya etiket arar. Böyle bir sürüm yoksa paketin en son sürümünü alır.

‘go get’ kullanılırken, ‘-d’ bayrağı verilen paketi indirmesine ama kurmamasına neden olur. -u bayrağı, paketi ve bağımlılıklarını güncellemesine izin verecektir.

Satıcı dizinlerinde depolanan kodu asla kontrol etmeyin veya güncellemeyin.
