Paket başlatma #

Paket, main’den önce yalnızca bir kez çalıştırılan ‘init’ yöntemlerine sahip olabilir.

package usefull

func init() {
    // init code
}

Paket başlatmayı herhangi bir şeye başvurmadan çalıştırmak istiyorsanız, aşağıdaki içe aktarma ifadesini kullanın.

import _ "usefull"

Paket bağımlılıklarını yönetme #

Go bağımlılıklarını indirmenin yaygın bir yolu, paketi global/paylaşılan “$GOPATH/src” dizinine kaydedecek olan “go get ” komutunu kullanmaktır. Bu, her paketin tek bir sürümünün, onu bir bağımlılık olarak içeren her projeye bağlanacağı anlamına gelir. Bu aynı zamanda, yeni bir geliştirici projenizi dağıttığında, her bağımlılığın en son sürümünü “alacakları” anlamına gelir.

Ancak, bir projenin tüm bağımlılıklarını vendor/ dizinine ekleyerek derleme ortamını tutarlı tutabilirsiniz. Satıcıya bağlı bağımlılıkları projenizin deposuyla birlikte tutmak, proje başına bağımlılık sürümü oluşturmanıza ve yapınız için tutarlı bir ortam sağlamanıza olanak tanır.

Projenizin yapısı şöyle görünecek:

$GOPATH/src/
├── github.com/username/project/
|   ├── main.go 
|   ├── vendor/
|   |   ├── github.com/pkg/errors
|   |   ├── github.com/gorilla/mux

Paketleri içe aktarma #

Şu ifadeyle tek bir paketi içe aktarabilirsiniz:

import "path/to/package"

veya birden çok içe aktarmayı birlikte gruplayın:

import (
    "path/to/package1"
    "path/to/package2"
)

Bu, “.go” dosyaları için “$GOPATH” içindeki ilgili “içe aktarma” yollarına bakacak ve dışa aktarılan adlara “packagename.AnyExportedName” aracılığıyla erişmenizi sağlar.

Paketlerin önüne ./ ekleyerek geçerli klasörün içindeki yerel paketlere de erişebilirsiniz. Bunun gibi bir yapıya sahip bir projede:

project
├── src
│   ├── package1
│   │   └── file1.go
│   └── package2
│       └── file2.go
└── main.go

“file1.go” ve “file2.go” içindeki kodu içe aktarmak için bunu “main.go” içinde çağırabilirsiniz:

import (
    "./src/package1"
    "./src/package2"
)

Paket adları farklı kitaplıklarda çakışabileceğinden, bir paketi yeni bir adla takmak isteyebilirsiniz. Bunu, import ifadenizin önüne kullanmak istediğiniz ismi ekleyerek yapabilirsiniz.

import (
    "fmt" //fmt from the standardlibrary
    tfmt "some/thirdparty/fmt" //fmt from some other library
)

Bu, eski ‘fmt’ paketine ‘fmt.’ kullanarak ve ikinci ‘fmt’ paketine ’tfmt.’ kullanarak erişmenizi sağlar.

Ayrıca paketi kendi ad alanına içe aktarabilirsiniz, böylece ‘paket.’ öneki olmadan tek bir noktayı takma ad olarak kullanarak dışa aktarılan adlara başvurabilirsiniz:

import (
    . "fmt"
)

Yukarıdaki örnek, “fmt"yi global ad alanına içe aktarır ve örneğin doğrudan “Printf"i aramanıza olanak tanır: Playground

Bir paketi içe aktarır ancak dışa aktarılan adlarından hiçbirini kullanmazsanız, Go derleyicisi bir hata mesajı yazdıracaktır. Bunu atlatmak için takma adı alt çizgiye ayarlayabilirsiniz:

import (
    _ "fmt"
)

Bu pakete doğrudan erişmiyorsanız, ancak çalışması için “init” işlevlerine ihtiyacınız varsa, bu yararlı olabilir.

Not:

Paket adları klasör yapısını temel aldığından, klasör adlarındaki ve içe aktarma referanslarındaki (büyük/küçük harf duyarlılığı dahil) herhangi bir değişiklik, Linux ve OS-X’te izlemesi zor olan bir derleme zamanı hatası “büyük/küçük harfe duyarlı olmayan içe aktarma çakışmasına” neden olur. ve düzeltin (hata mesajı, tam tersini iletmeye çalıştığı için ölümlüler için biraz şifrelidir - bu, büyük/küçük harf duyarlılığı nedeniyle karşılaştırma başarısız oldu).

ör: “yol/to/Paket1” vs “yol/to/paket1”

Canlı örnek: https://github.com/akamai-open/AkamaiOPEN-edgegrid-golang/issues/2

Farklı paket ve klasör adı kullanma #

Klasör adından başka bir paket adı kullanmak gayet iyi. Bunu yaparsak, yine de dizin yapısına göre paketi içe aktarmamız gerekir, ancak içe aktarmadan sonra paket yan tümcesinde kullandığımız isimle ona başvurmamız gerekir.

Örneğin, $GOPATH/src/mypck klasörünüz varsa ve içinde bir a.go dosyamız varsa:

package apple

const Pi = 3.14

Bu paketi kullanarak:

package main

import (
    "mypck"
    "fmt"
)

func main() {
    fmt.Println(apple.Pi)
}

Bu işe yarasa bile, paket adını klasör adından saptırmak için iyi bir nedeniniz olmalıdır (veya yanlış anlama ve karışıklık kaynağı olabilir).

Bunun ne faydası var? #

Basit. Paket adı bir Go tanımlayıcı‘dir:

identifier = letter { letter | unicode_digit } .

Bu, tanımlayıcılarda unicode harflerin kullanılmasına izin verir, örn. “αβ”, Go’da geçerli bir tanımlayıcıdır. Klasör ve dosya adları Go tarafından değil İşletim Sistemi tarafından işlenir ve farklı dosya sistemlerinin farklı kısıtlamaları vardır. Aslında tüm geçerli Go tanımlayıcılarının klasör adı olarak kullanılmasına izin vermeyen birçok dosya sistemi vardır, bu nedenle paketlerinizi, aksi takdirde dil özelliklerinin izin vereceği şekilde adlandıramazsınız.

İçerdikleri klasörlerden farklı paket adları kullanma seçeneğine sahip olarak, temel işletim ve dosya sisteminden bağımsız olarak, dil özelliklerinin izin verdiği paketlerinizi gerçekten adlandırma seçeneğiniz vardır.
