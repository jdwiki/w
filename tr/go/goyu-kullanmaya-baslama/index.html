Selam Dünya! #

Aşağıdaki kodu “hello.go” dosya adına yerleştirin:

package main

import "fmt"

func main() {
    fmt.Println("Hello, 世界")
}

Oyun Alanı

Go doğru yüklendiğinde bu program şu şekilde derlenebilir ve çalıştırılabilir:

go run hello.go

Çıktı: #

Hello, 世界

Koddan memnun kaldığınızda, aşağıdakileri çalıştırarak yürütülebilir bir dosyaya derlenebilir:

go build hello.go

Bu, geçerli dizinde işletim sisteminize uygun yürütülebilir bir dosya oluşturacak ve ardından aşağıdaki komutla çalıştırabilirsiniz:

Linux, OSX ve diğer Unix benzeri sistemler

./hello

Pencereler

hello.exe

Not: Go dizelerinin salt okunur bayt dilimleri olarak depolandığını gösterdikleri için Çince karakterler önemlidir.

Ortamın ayarlanması #

Go sisteminizde önceden kurulu değilse, https://golang.org/dl/ adresine gidebilir ve Go’yu indirip kurmak için platformunuzu seçebilirsiniz.

Temel bir Go geliştirme ortamı kurmak için, “go” aracının davranışını etkileyen birçok ortam değişkeninden yalnızca birkaçının ayarlanması gerekir (tam liste için bkz. Go Ortam Değişkenlerini Listeleme) (genel olarak kabuğunuzun ~/.profile dosyasında veya Unix benzeri işletim sistemlerinde eşdeğeri).

‘GOPATH’ #

Sistem PATH ortam değişkeni gibi, Go yolu da Go’nun paketleri arayacağı :(; Windows’ta) ile ayrılmış bir dizin listesidir. go get aracı da paketleri bu listedeki ilk dizine indirecektir.

‘GOPATH’, Go’nun çalışma alanı için gerekli olan ilişkili ‘bin’, ‘pkg’ ve ‘src’ klasörlerini kuracağı yerdir:

  • src — kaynak dosyaların konumu: .go, .c, .g, .s
  • pkg — .a dosyalarını derledi
  • bin — Go tarafından oluşturulmuş yürütülebilir dosyaları içerir

Go 1.8’den itibaren, “GOPATH” ortam değişkeni, ayarlanmamışsa [varsayılan değer][2] değerine sahip olacaktır. Varsayılan olarak Unix/Linux’ta $HOME/go ve Windows’ta %USERPROFILE%/go şeklindedir.

Bazı araçlar, “GOPATH” öğesinin tek bir dizin içereceğini varsayar.

‘GOBIN’ #

“go install” ve “go get” öğelerinin bulunduğu bin dizini, “ana” paketleri oluşturduktan sonra ikili dosyaları yerleştirecektir. Genellikle bu, kurulu ikili dosyaların kolayca çalıştırılabilmesi ve keşfedilebilmesi için PATH sisteminde bir yere ayarlanır.

‘GOROOT’ #

Bu, Go kurulumunuzun konumudur. Standart kütüphaneleri bulmak için kullanılır. Go, yapı yolunu araç zincirine yerleştirirken bu değişkeni ayarlamak zorunda kalmak çok nadirdir. Kurulum dizini yapı dizininden (veya oluşturma sırasında ayarlanan değerden) farklıysa ‘GOROOT’ ayarı gereklidir.

1: https://www.wikiod.com/tr/go/goyu-kullanmaya-baslama#Go Ortamı Değişkenlerini Listeleme [2]: https://golang.org/doc/go1.8#gopath

FizzBuzz #

“Merhaba Dünya” tarzı programların bir başka örneği de FizzBuzz‘dir. Bu, FizzBuzz uygulamasının bir örneğidir. Çok deyimsel Go burada oyunda.

package main

// Simple fizzbuzz implementation

import "fmt"

func main() {
    for i := 1; i <= 100; i++ {
        s := ""       
        if i % 3 == 0 {
            s += "Fizz"
        }
        if i % 5 == 0 {
            s += "Buzz"
        }
        if s != "" {
            fmt.Println(s) 
        } else {
            fmt.Println(i) 
        }
    }
}

Go Ortamı Değişkenlerini Listeleme #

“go” aracını etkileyen ortam değişkenleri, “go env [var …]” komutuyla görüntülenebilir:

$ go env
GOARCH="amd64"
GOBIN="/home/yourname/bin"
GOEXE=""
GOHOSTARCH="amd64"
GOHOSTOS="linux"
GOOS="linux"
GOPATH="/home/yourname"
GORACE=""
GOROOT="/usr/lib/go"
GOTOOLDIR="/usr/lib/go/pkg/tool/linux_amd64"
CC="gcc"
GOGCCFLAGS="-fPIC -m64 -pthread -fmessage-length=0 -fdebug-prefix-map=/tmp/go-build059426571=/tmp/go-build -gno-record-gcc-switches"
CXX="g++"
CGO_ENABLED="1"

Varsayılan olarak listeyi bir kabuk betiği olarak yazdırır; ancak, argüman olarak bir veya daha fazla değişken adı verilirse, adlandırılmış her bir değişkenin değerini yazdırır.

$go env GOOS GOPATH
linux
/home/yourname

Belgelere Çevrimdışı Erişme #

Tam belgeler için şu komutu çalıştırın:

godoc -http=:<port-number>

Bir Go turu için (dilde yeni başlayanlar için şiddetle tavsiye edilir):

go tool tour

Yukarıdaki iki komut, sırasıyla burada ve burada bulunanlara benzer belgelerle web sunucularını başlatacaktır.

Komut satırından hızlı başvuru kontrolü için, örneğin fmt.Print için:

godoc cmd/fmt Print
# or
go doc fmt Print

Genel yardım ayrıca komut satırından da edinilebilir:

go help [command]
