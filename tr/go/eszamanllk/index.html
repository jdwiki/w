Go’da eşzamanlılık, goroutinler kullanılarak sağlanır ve goroutinler arasındaki iletişim genellikle kanallarla yapılır. Ancak, muteksler ve bekleme grupları gibi diğer senkronizasyon araçları mevcuttur ve kanallardan daha uygun olduklarında kullanılmalıdır.

Sözdizimi #

  • git doWork() // doWork işlevini bir goroutine olarak çalıştır
  • ch := make(chan int) // int türünde yeni kanal bildir
  • ch <- 1 // bir kanala gönderme
  • değer = <-ch // bir kanaldan alma

Go’daki goroutines, kullanım açısından diğer dillerdeki threadlere benzer. Dahili olarak, Go bir dizi iş parçacığı oluşturur (“GOMAXPROCS” ile belirtilir) ve ardından goroutinleri iş parçacıkları üzerinde çalışacak şekilde zamanlar. Bu tasarım nedeniyle, Go’nun eşzamanlılık mekanizmaları, bellek kullanımı ve başlatma süresi açısından iş parçacıklarından çok daha verimlidir.

Merhaba Dünya Goroutine #

tek kanal, tek goroutine, tek yazma, tek okuma.

package main

import "fmt"
import "time"

func main() {
    // create new channel of type string
    ch := make(chan string)

    // start new anonymous goroutine
    go func() {
        time.Sleep(time.Second)
        // send "Hello World" to channel
        ch <- "Hello World"
    }()
    // read from channel
    msg, ok := <-ch
    fmt.Printf("msg='%s', ok='%v'\n", msg, ok)
}

Oyun alanında çalıştırın

“ch” kanalı bir arabelleğe alınmamış veya senkronize kanal.

“time.Sleep”, “main()” işlevinin “ch” kanalında bekleyeceğini göstermek için buradadır; bu, bir goroutine olarak yürütülen işlev değişmezi öğesinin bir değer gönderme zamanı olduğu anlamına gelir. bu kanal aracılığıyla: alma operatörü <-ch main() yürütmesini engeller. Olmazsa, ‘main()’ çıktığında goroutine öldürülecek ve değerini göndermek için zamanı olmayacaktı.

Goroutinleri bekliyorum #

Go programları ‘ana’ işlev sona erdiğinde sona erer, bu nedenle tüm goroutinlerin bitmesini beklemek yaygın bir uygulamadır. Bunun için yaygın bir çözüm, bir sync.WaitGroup nesnesi kullanmaktır.

package main

import (
    "fmt"
    "sync"
)

var wg sync.WaitGroup // 1

func routine(i int) {
    defer wg.Done() // 3
    fmt.Printf("routine %v finished\n", i)
}

func main() {
    wg.Add(10) // 2
    for i := 0; i < 10; i++ {
        go routine(i) // *
    }
    wg.Wait() // 4
    fmt.Println("main finished")
}

Örneği oyun alanında çalıştırın

Yürütme sırasına göre WaitGroup kullanımı:

  1. Küresel değişkenin beyanı. Global yapmak, onu tüm işlevler ve yöntemler için görünür kılmanın en kolay yoludur.
  2. Sayacın arttırılması. Bu, ana goroutinde yapılmalıdır çünkü bellek modeli garantiler nedeniyle yeni başlatılan bir goroutinin 4’ten önce yürütüleceğinin garantisi yoktur.
  3. Sayacın azaltılması. Bu, bir goroutinin çıkışında yapılmalıdır. Ertelenmiş bir çağrı kullanarak, nasıl biterse bitsin fonksiyon bittiğinde çağrılacağından emin oluruz.
  4. Sayacın 0’a ulaşmasını beklemek. Bu, tüm goroutinler bitmeden programın çıkmasını önlemek için ana goroutinde yapılmalıdır.

* Parametreler yeni bir goroutine başlamadan önce değerlendirilir. Bu nedenle, olası panik kodunun sayacı artırmaması için, değerlerini ‘wg.Add(10)‘dan önce açıkça tanımlamak gerekir. WaitGroup’a 10 öğe eklemek, böylece wg.Wait kontrolü main() goroutine geri döndürmeden önce 10 öğe bekleyecektir. Burada i’nin değeri for döngüsünde tanımlanır.

Goroutinler oluşturma #

Herhangi bir işlev, çağrısının önüne “go” anahtar kelimesiyle getirilerek bir goroutine olarak çağrılabilir:

func DoMultiply(x,y int) {
    // Simulate some hard work
    time.Sleep(time.Second * 1)
    fmt.Printf("Result: %d\n", x * y)
}

go DoMultiply(1,2) // first execution, non-blocking
go DoMultiply(3,4) // second execution, also non-blocking

// Results are printed after a single second only, 
// not 2 seconds because they execute concurrently:
// Result: 2
// Result: 12

İşlevin dönüş değerinin yok sayıldığını unutmayın.

Bir döngüde goroutin içeren kapakları kullanma #

Bir döngüdeyken, aşağıdaki örnekteki döngü değişkeni (val), döngüden geçerken değeri değiştiren tek bir değişkendir. Bu nedenle, her bir değer değerini goroutine gerçekten geçirmek için aşağıdakiler yapılmalıdır:

for val := range values {
    go func(val interface{}) {
        fmt.Println(val)
    }(val)
}

val’i geçmeden sadece func(val interface{}) { ... }() yapacaksanız, goroutinler gerçekten çalıştığında val değeri val ne olursa olsun olacaktır.

Aynı etkiyi elde etmenin başka bir yolu da şudur:

for val := range values {
    val := val
    go func() {
        fmt.Println(val)
    }()
}

Garip görünen ‘val := val’, her yinelemede yeni bir değişken oluşturur ve bu değişkene daha sonra goroutin tarafından erişilir.

İki goroutine sahip masa tenisi #

package main

import (
    "fmt"
    "time"
)

// The pinger prints a ping and waits for a pong
func pinger(pinger <-chan int, ponger chan<- int) {
    for {
        <-pinger
        fmt.Println("ping")
        time.Sleep(time.Second)
        ponger <- 1
    }
}

// The ponger prints a pong and waits for a ping
func ponger(pinger chan<- int, ponger <-chan int) {
    for {
        <-ponger
        fmt.Println("pong")
        time.Sleep(time.Second)
        pinger <- 1
    }
}

func main() {
    ping := make(chan int)
    pong := make(chan int)

    go pinger(ping, pong)
    go ponger(ping, pong)

    // The main goroutine starts the ping/pong by sending into the ping channel
    ping <- 1

    for {
        // Block the main thread until an interrupt
        time.Sleep(time.Second)
    }
}

Goroutine’leri durdurma #

package main

import (
    "log"
    "sync"
    "time"
)

func main() {
    // The WaitGroup lets the main goroutine wait for all other goroutines
    // to terminate. However, this is no implicit in Go. The WaitGroup must
    // be explicitely incremented prior to the execution of any goroutine 
    // (i.e. before the `go` keyword) and it must be decremented by calling
    // wg.Done() at the end of every goroutine (typically via the `defer` keyword). 
    wg := sync.WaitGroup{}

    // The stop channel is an unbuffered channel that is closed when the main
    // thread wants all other goroutines to terminate (there is no way to 
    // interrupt another goroutine in Go). Each goroutine must multiplex its
    // work with the stop channel to guarantee liveness.
    stopCh := make(chan struct{})


    for i := 0; i < 5; i++ {
        // It is important that the WaitGroup is incremented before we start
        // the goroutine (and not within the goroutine) because the scheduler
        // makes no guarantee that the goroutine starts execution prior to 
        // the main goroutine calling wg.Wait().
        wg.Add(1)
        go func(i int, stopCh <-chan struct{}) {
            // The defer keyword guarantees that the WaitGroup count is 
            // decremented when the goroutine exits.
            defer wg.Done()

            log.Printf("started goroutine %d", i)

            select {
            // Since we never send empty structs on this channel we can 
            // take the return of a receive on the channel to mean that the
            // channel has been closed (recall that receive never blocks on
            // closed channels).   
            case <-stopCh:
                log.Printf("stopped goroutine %d", i)
            }
        }(i, stopCh)
    }

    time.Sleep(time.Second * 5)
    close(stopCh)
    log.Printf("stopping goroutines")
    wg.Wait()
    log.Printf("all goroutines stopped")
}
