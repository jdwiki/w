Go’daki Arayüzler yalnızca sabit yöntem kümeleridir. implicitly türü, yöntem kümesi arabirimin bir üst kümesiyse, bir arabirim uygular. Niyet beyanı yoktur.

Basit arayüz #

Go’da bir arayüz sadece bir dizi metottur. Belirli bir nesnenin davranışını belirtmek için bir arayüz kullanırız.

type Painter interface {
    Paint()
}

Uygulama türü gerekmez, arabirimi uyguladığını bildirir. Aynı imzanın yöntemlerini tanımlamanız yeterlidir.

type Rembrandt struct{}

func (r Rembrandt) Paint() {
    // use a lot of canvas here 
}

Artık yapıyı arayüz olarak kullanabiliriz.

var p Painter
p = Rembrandt{}

Bir arabirim, keyfi sayıda tür tarafından karşılanabilir (veya uygulanabilir). Ayrıca bir tür, isteğe bağlı sayıda arabirim uygulayabilir.

type Singer interface {
     Sing()
}

type Writer interface {
     Write()
}

type Human struct{}

func (h *Human) Sing() {
    fmt.Println("singing")
}

func (h *Human) Write() {
    fmt.Println("writing")
}



type OnlySinger struct{}
func (o *OnlySinger) Sing() {
    fmt.Println("singing")
}

Burada, “Human” yapısı hem “Singer” hem de “Writer” arayüzünü karşılar, ancak “OnlySinger” yapısı sadece “Singer” arayüzünü karşılar.

Boş Arayüz

Hiçbir yöntem içermeyen boş bir arabirim türü var. Bunu ‘arayüz{}’ olarak ilan ediyoruz. Bu, hiçbir yöntem içermez, bu nedenle her “tür” onu karşılar. Bu nedenle boş arayüz herhangi bir tür değeri içerebilir.

var a interface{}
var i int = 5
s := "Hello world"

type StructType struct {
    i, j int
    k string
}


// all are valid statements
a = i
a = s
a = &StructType{1, 2, "hello"}

Arayüzler için en yaygın kullanım durumu, bir değişkenin bir veya daha fazla davranışı desteklediğinden emin olmaktır. Buna karşılık, boş arabirim için birincil kullanım durumu, somut türünden bağımsız olarak herhangi bir değeri tutabilen bir değişken tanımlamaktır.

Bu değerleri orijinal türleri olarak geri almak için yapmamız gereken sadece

i = a.(int)
s = a.(string)
m := a.(*StructType)

veya

i, ok := a.(int)
s, ok := a.(string)
m, ok := a.(*StructType)

“ok”, “a arayüzü"nün belirli bir türe dönüştürülebilir olup olmadığını gösterir. Eğer “ok” yapmak mümkün değilse “false” olacaktır.

Arayüz Değerleri

Bir arabirimin değişkenini bildirirseniz, arabirim tarafından bildirilen yöntemleri uygulayan herhangi bir değer türünü depolayabilir!

Eğer ‘interface Singer’ın ‘h’sini bildirirsek, ‘Human’ veya ‘OnlySinger’ türünde bir değer depolayabilir. Bunun nedeni, hepsinin ‘Singer’ arabirimi tarafından belirtilen yöntemleri uygulamasıdır.

var h Singer
h = &human{}

h.Sing()

Bir türün bir arabirimi karşılayıp karşılamadığını derleme zamanı kontrolü #

Arabirimler ve uygulamalar (bir arabirim uygulayan türler) “ayrıdır”. Bu nedenle, bir tür bir arabirim uygularsa derleme zamanında nasıl kontrol edileceği doğru bir sorudur.

Derleyiciden “T” türünün “I” arabirimini uyguladığını kontrol etmesini istemenin bir yolu, uygun şekilde “T” için sıfır değeri veya “T” işaretçisini kullanarak bir atama girişiminde bulunmaktır. Ve gereksiz çöpleri önlemek için boş tanımlayıcı atamayı seçebiliriz:

type T struct{}

var _ I = T{}       // Verify that T implements I.
var _ I = (*T)(nil) // Verify that *T implements I.

“T” veya “*T”, “I"yi uygulamıyorsa, bu bir derleme zamanı hatası olacaktır.

Bu soru ayrıca resmi SSS’de de yer almaktadır: Türümün bir arayüze uygun olduğunu nasıl garanti edebilirim?

Matematiksel Açıdan Git Arayüzleri #

Matematikte, özellikle Küme Teorisinde, küme adı verilen bir şeyler koleksiyonumuz vardır ve bunları elemanlar olarak adlandırırız. Adı A, B, C, … gibi veya açıkça üyesini ayraç notasyonuna koyarak bir küme gösteririz: {a, b, c, d, e}. İsteğe bağlı bir x elemanımız ve bir Z kümesimiz olduğunu varsayalım, Anahtar soru şudur: “x’in Z’nin üyesi olup olmadığını nasıl anlayabiliriz?”. Matematikçi bu soruya bir kavramla cevap verir: Bir kümenin Karakteristik Özelliği. Bir kümenin Karakteristik Özelliği, kümeyi tamamen tanımlayan bir ifadedir. Örneğin, {0, 1, 2, 3, 4, 5, …} olan Doğal Sayılar kümesini oluşturduk. Bu kümeyi şu ifadeyle tanımlayabiliriz: {an | a0 = 0, an = an-1+1}. Son ifadede a0 = 0, an = an-1+1, doğal sayılar kümesinin karakteristik özelliğidir. Bu ifadeye sahipsek, bu seti tamamen oluşturabiliriz. Çift sayılar kümesini bu şekilde tanımlayalım. Bu kümenin şu sayılarla yapıldığını biliyoruz: {0, 2, 4, 6, 8, 10, …}. Bir bakışta tüm bu sayıların aynı zamanda bir doğal sayı olduğunu anlıyoruz, yani doğal sayıların karakteristik özelliğine bazı ekstra koşullar eklersek, bu kümeyi tanımlayan yeni bir ifade oluşturabiliriz. O halde şu ifadeyle tanımlayabiliriz: {n | n, doğal sayıların bir üyesidir ve 2’deki n’nin hatırlatıcısı sıfırdır}. Şimdi bir kümenin karakteristik özelliğini alan bir filtre oluşturabilir ve kümemizin öğelerini döndürmek için istenen bazı öğeleri filtreleyebiliriz. Örneğin bir doğal sayı filtremiz varsa, hem doğal sayılar hem de çift sayılar bu filtreyi geçebilir, ancak çift sayı filtremiz varsa 3 ve 137871 gibi bazı elemanlar filtreyi geçemez.

Go’da interface’in tanımı, interface’i bir fonksiyonun argümanı olarak kullanmanın karakteristik özelliğini ve mekanizmasını tanımlamak, elemanın istediğimiz kümenin üyesi olup olmadığını tespit eden bir filtre gibidir. Bu yönü kodla açıklayalım:

type Number interface {
    IsNumber() bool // the implementation filter "meysam" from 3.14, 2 and 3
}

type NaturalNumber interface {
    Number
    IsNaturalNumber() bool // the implementation filter 3.14 from 2 and 3
}

type EvenNumber interface {
    NaturalNumber
    IsEvenNumber() bool // the implementation filter 3 from 2
}

‘Number’ın karakteristik özelliği, ‘IsNumber’ yöntemine sahip tüm yapılardır, ‘NaturalNumber’, ‘IsNumber’ ve ‘IsNaturalNumber’ yöntemlerine sahip olan tüm yapılardır ve son olarak ‘EvenNumber’ için, ‘IsNumber’, ’ IsNaturalNumber’ ve ‘IsEvenNumber’ yöntemleri. Arayüzün bu yorumu sayesinde, interface{} herhangi bir karakteristik özelliğe sahip olmadığı için tüm türleri kabul ettiğini (çünkü değerleri ayırt etmek için herhangi bir filtresi olmadığı için) kolayca anlayabiliriz.

Arayüzden temel türü belirleme #

Hareket halindeyken, hangi temel türün iletildiğini bilmek bazen yararlı olabilir. Bu bir tip anahtarı ile yapılabilir. Bu, iki yapıya sahip olduğumuzu varsayar:

type Rembrandt struct{}

func (r Rembrandt) Paint() {}

type Picasso struct{}

func (r Picasso) Paint() {}

Painter arayüzünü uygulayan:

type Painter interface {
    Paint()
}

Ardından, temel türü belirlemek için bu anahtarı kullanabiliriz:


func WhichPainter(painter Painter) {
    switch painter.(type) {
    case Rembrandt:
        fmt.Println("The underlying type is Rembrandt")
    case Picasso:
        fmt.Println("The underlying type is Picasso")
    default:
        fmt.Println("Unknown type")
    }
}

Tip anahtarı #

Tip anahtarları, vakanın tipiyle eşleşen bir değişken elde etmek için de kullanılabilir:

func convint(v interface{}) (int,error) {
    switch u := v.(type) {
    case int:
        return u, nil
    case float64:
        return int(u), nil
    case string:
        return strconv(u)
    default:
        return 0, errors.New("Unsupported type")
    }
}

Tip Onayı #

Type Assertion ile arayüzün gerçek veri tipine ulaşabilirsiniz.

interfaceVariable.(DataType)

“Subber” arabirimini uygulayan “MyType” yapısının örneği:

package main

import (
    "fmt"
)

type Subber interface {
    Sub(a, b int) int
}

type MyType struct {
    Msg string
}
 
//Implement method Sub(a,b int) int
func (m *MyType) Sub(a, b int) int {
    m.Msg = "SUB!!!"

    return a - b;
}

func main() {
    var interfaceVar Subber = &MyType{}
    fmt.Println(interfaceVar.Sub(6,5))
    fmt.Println(interfaceVar.(*MyType).Msg)
}

.(*MyType) olmadan Msg Alanına erişemezdik. interfaceVar.Msgyi denersek derleme hatası gösterecektir:

interfaceVar.Msg undefined (type Subber has no field or method Msg)
