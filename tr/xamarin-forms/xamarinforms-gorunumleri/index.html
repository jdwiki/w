Buton #

Düğme, muhtemelen yalnızca mobil uygulamalarda değil, kullanıcı arayüzüne sahip tüm uygulamalarda en yaygın kontroldür. Bir düğme kavramının burada listelenemeyecek kadar çok amacı var. Genel olarak konuşursak, kullanıcıların uygulamanızda bir tür eylem veya işlem başlatmasına izin vermek için bir düğme kullanacaksınız. Bu işlem, uygulamanızdaki temel gezinmeden İnternet’teki bir web hizmetine veri göndermeye kadar her şeyi içerebilir.

XAML

<Button
    x:Name="MyButton"
    Text="Click Me!"
    TextColor="Red"
    BorderColor="Blue"
    VerticalOptions="Center"
    HorizontalOptions="Center"
    Clicked="Button_Clicked"/>

XAML Kodu Arkası

public void Button_Clicked( object sender, EventArgs args ) 
{
    MyButton.Text = "I've been clicked!";
}

Kod

var button = new Button( ) 
{
    Text = "Hello, Forms !",
    VerticalOptions = LayoutOptions.CenterAndExpand,
    HorizontalOptions = LayoutOptions.CenterAndExpand,
    TextColor = Color.Red,
    BorderColor = Color.Blue,
};

button.Clicked += ( sender, args ) => 
{
    var b = (Button) sender;
    b.Text = "I've been clicked!";
};

Tarih Seçici #

Oldukça sık olarak, mobil uygulamalarda tarihlerle uğraşmak için bir sebep olacaktır. Tarihlerle çalışırken, bir tarih seçmek için muhtemelen bir tür kullanıcı girdisine ihtiyacınız olacaktır. Bu, bir zamanlama veya takvim uygulamasıyla çalışırken ortaya çıkabilir. Bu durumda, kullanıcılara manuel olarak bir tarih yazmalarını gerektirmek yerine, etkileşimli olarak bir tarih seçmelerine olanak tanıyan özel bir denetim sağlamak en iyisidir. DatePicker denetiminin gerçekten yararlı olduğu yer burasıdır.

XAML

<DatePicker Date="09/12/2014" Format="d" />

Kod

var datePicker = new DatePicker{
Date = DateTime.Now,
Format = "d"
};

Giriş #

Giriş Görünümü, kullanıcıların tek bir metin satırı yazmasına izin vermek için kullanılır. Bu tek satırlık metin, temel notlar, kimlik bilgileri, URL’ler ve daha fazlasını girmek dahil olmak üzere birden çok amaç için kullanılabilir. Bu Görünüm çok amaçlı bir Görünümdür, yani normal metin yazmanız gerekiyorsa veya bir parolayı gizlemek istiyorsanız, bunların hepsi bu tek kontrol aracılığıyla yapılır.

XAML

<Entry Placeholder="Please Enter Some Text Here"
HorizontalOptions="Center"
VerticalOptions="Center"
Keyboard="Email"/>

Kod

var entry = new Entry {
Placeholder = "Please Enter Some Text Here",
HorizontalOptions = LayoutOptions.Center,
VerticalOptions = LayoutOptions.Center,
Keyboard = Keyboard.Email
};

Düzenleyici #

Editör, kullanıcıların bazı serbest biçimli metin girmelerine izin vermesi açısından Giriş’e çok benzer. Aradaki fark, Editör’ün çok satırlı girişe izin vermesi, Giriş’in ise yalnızca tek satırlı giriş için kullanılmasıdır. Giriş ayrıca Görünümün daha fazla özelleştirilmesine izin vermek için Düzenleyiciden birkaç özellik daha sağlar.

XAML

<Editor HorizontalOptions="Fill"
VerticalOptions="Fill"
 Keyboard="Chat"/>

Kod

var editor = new Editor {
HorizontalOptions = LayoutOptions.Fill,
VerticalOptions = LayoutOptions.Fill,
Keyboard = Keyboard.Chat
};

Resim #

Görüntüler, herhangi bir uygulamanın çok önemli parçalarıdır. Enjekte etme imkanı sağlarlar. ek görsel unsurların yanı sıra uygulamanıza marka bilinci oluşturma. O görüntülerden bahsetmiyorum bile genellikle metin veya düğmelere bakmaktan daha ilginçtir. Bir Görüntü olarak kullanabilirsiniz uygulamanız içinde bağımsız bir öğedir, ancak diğerlerine bir Image öğesi de eklenebilir. Düğme gibi öğeleri görüntüleyin.

XAML

<Image Aspect="AspectFit" Source="http://d2g29cya9iq7ip.cloudfront.net/co
ntent/images/company/aboutus-video-bg.png?v=25072014072745"/>

Kod

var image = new Image {
Aspect = Aspect.AspectFit,
Source = ImageSource.FromUri(new Uri("http://d2g29cya9iq7ip.cloudfron
t.net/content/images/company/aboutus-video-bg.png?v=25072014072745"))
};

Etiket #

İster inanın ister inanmayın, Label en önemli ancak yeterince takdir edilmeyen View sınıflarından biridir. yalnızca Xamarin.Forms’da, ancak genel olarak UI geliştirmede. Oldukça sıkıcı bir çizgi olarak görülüyor ancak bu metin satırı olmadan belirli fikirleri kullanıcıya iletmek çok zor olurdu. Etiket kontrolleri, kullanıcının bir Düzenleyiciye veya Girişe ne girmesi gerektiğini açıklamak için kullanılabilir. kontrol. Kullanıcı arayüzünün bir bölümünü tanımlayabilir ve ona bağlam verebilirler. göstermek için kullanılabilirler. bir hesap makinesi uygulamasında toplam. Evet, Etiket gerçekten alet çantanızdaki en çok yönlü kontroldür. her zaman çok fazla dikkat çekmeyebilir, ancak orada değilse fark edilen ilk kişidir.

XAML

<Label Text="This is some really awesome text in a Label!"
TextColor="Red"
XAlign="Center"
YAlign="Center"/>

Kod

var label = new Label {
Text = "This is some really awesome text in a Label!",
TextColor = Color.Red,
XAlign = TextAlignment.Center,
YAlign = TextAlignment.Center
};
