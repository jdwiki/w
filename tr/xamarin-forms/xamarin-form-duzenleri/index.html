Mutlak Düzen #

“AbsoluteLayout”, alt öğeleri kendi boyutuna ve konumuna veya mutlak değerlere göre konumlandırır ve boyutlandırır. Alt görünümler, orantılı değerler veya statik değerler kullanılarak konumlandırılabilir ve boyutlandırılabilir ve orantılı ve statik değerler karıştırılabilir.

XAML’de bir “AbsoluteLayout” tanımı şöyle görünür:

<AbsoluteLayout>
    <Label Text="I'm centered on iPhone 4 but no other device"
        AbsoluteLayout.LayoutBounds="115,150,100,100" LineBreakMode="WordWrap"  />
    <Label Text="I'm bottom center on every device."
        AbsoluteLayout.LayoutBounds=".5,1,.5,.1" AbsoluteLayout.LayoutFlags="All"
        LineBreakMode="WordWrap"  />
    <BoxView Color="Olive"  AbsoluteLayout.LayoutBounds="1,.5, 25, 100"
        AbsoluteLayout.LayoutFlags="PositionProportional" />
    <BoxView Color="Red" AbsoluteLayout.LayoutBounds="0,.5,25,100"
        AbsoluteLayout.LayoutFlags="PositionProportional" />
    <BoxView Color="Blue" AbsoluteLayout.LayoutBounds=".5,0,100,25"
        AbsoluteLayout.LayoutFlags="PositionProportional" />
    <BoxView Color="Blue" AbsoluteLayout.LayoutBounds=".5,0,1,25"
        AbsoluteLayout.LayoutFlags="PositionProportional, WidthProportional" />
</AbsoluteLayout>

Aynı düzen kodda şöyle görünür:

Title = "Absolute Layout Exploration - Code";
var layout = new AbsoluteLayout();

var centerLabel = new Label {
Text = "I'm centered on iPhone 4 but no other device.",
LineBreakMode = LineBreakMode.WordWrap};

AbsoluteLayout.SetLayoutBounds (centerLabel, new Rectangle (115, 159, 100, 100));
// No need to set layout flags, absolute positioning is the default

var bottomLabel = new Label { Text = "I'm bottom center on every device.", LineBreakMode = LineBreakMode.WordWrap };
AbsoluteLayout.SetLayoutBounds (bottomLabel, new Rectangle (.5, 1, .5, .1));
AbsoluteLayout.SetLayoutFlags (bottomLabel, AbsoluteLayoutFlags.All);

var rightBox = new BoxView{ Color = Color.Olive };
AbsoluteLayout.SetLayoutBounds (rightBox, new Rectangle (1, .5, 25, 100));
AbsoluteLayout.SetLayoutFlags (rightBox, AbsoluteLayoutFlags.PositionProportional);

var leftBox = new BoxView{ Color = Color.Red };
AbsoluteLayout.SetLayoutBounds (leftBox, new Rectangle (0, .5, 25, 100));
AbsoluteLayout.SetLayoutFlags (leftBox, AbsoluteLayoutFlags.PositionProportional);

var topBox = new BoxView{ Color = Color.Blue };
AbsoluteLayout.SetLayoutBounds (topBox, new Rectangle (.5, 0, 100, 25));
AbsoluteLayout.SetLayoutFlags (topBox, AbsoluteLayoutFlags.PositionProportional);

var twoFlagsBox = new BoxView{ Color = Color.Blue };
AbsoluteLayout.SetLayoutBounds (topBox, new Rectangle (.5, 0, 1, 25));
AbsoluteLayout.SetLayoutFlags (topBox, AbsoluteLayoutFlags.PositionProportional | AbsoluteLayout.WidthProportional);

layout.Children.Add (bottomLabel);
layout.Children.Add (centerLabel);
layout.Children.Add (rightBox);
layout.Children.Add (leftBox);
layout.Children.Add (topBox);

Xamarin.Forms içindeki AbsoluteLayout denetimi, tam olarak nerede olduğunu belirtmenize olanak tanır. alt öğelerin yanı sıra boyut ve şekillerinin (sınırlarının) görünmesini istediğiniz ekranda.

Alt öğelerin sınırlarını belirlemenin birkaç farklı yolu vardır. Bu işlem sırasında kullanılan AbsoluteLayoutFlags numaralandırması. AbsoluteLayoutFlags numaralandırması aşağıdaki değerleri içerir:

  • Tümü: Tüm boyutlar orantılıdır.
  • Yükseklik Orantılı: Yükseklik, yerleşim düzeniyle orantılıdır.
  • Yok: Yorum yapılmaz.
  • PositionProportional: XProportional ve YProportional’ı birleştirir.
  • SizeProportional: Genişlik Oransal ve Yükseklik Orantılı’yı birleştirir.
  • GenişlikOrantılı: Genişlik, yerleşim düzeniyle orantılıdır.
  • XProportional: X özelliği, yerleşim düzeniyle orantılıdır.
  • YProportional: Y özelliği yerleşimle orantılıdır.

AbsoluteLayout kapsayıcısının düzeniyle çalışma süreci biraz görünebilir ilk başta mantıksız gelebilir, ancak biraz kullanımla tanıdık gelecektir. Kendinizi oluşturduktan sonra alt öğeler, bunları kap içinde mutlak bir konuma ayarlamak için izlemeniz gerekecek üç adım. Öğelere atanan bayrakları aşağıdakileri kullanarak ayarlamak isteyeceksiniz: AbsoluteLayout.SetLayoutFlags() yöntemi. Ayrıca kullanmak isteyeceksiniz Öğelere sınırlarını vermek için AbsoluteLayout.SetLayoutBounds() yöntemi. Sonunda, sen alt öğeleri Children koleksiyonuna eklemek isteyecektir. Xamarin.Forms bir Xamarin ve cihaza özel uygulamalar arasındaki soyutlama katmanı, konumsal değerler cihaz piksellerinden bağımsız olabilir. Bu, düzen bayraklarının bahsettiği yer önce oyuna gir. Xamarin.Forms’un düzen işleminin nasıl yapılacağını seçebilirsiniz. kontroller tanımladığınız değerleri yorumlamalıdır.

Kafes #

Satır ve sütunlar halinde düzenlenmiş görünümleri içeren bir düzen.

Bu, XAML’deki tipik bir “Izgara” tanımıdır.

<Grid>
  <Grid.RowDefinitions>
    <RowDefinition Height="2*" />
    <RowDefinition Height="*" />
    <RowDefinition Height="200" />
  </Grid.RowDefinitions>
  <Grid.ColumnDefinitions>
    <ColumnDefinition Width="Auto" />
    <ColumnDefinition Width="*" />
  </Grid.ColumnDefinitions>
    
  <ContentView Grid.Row="0" Grid.Column="0"/>
  <ContentView Grid.Row="1" Grid.Column="0"/>
  <ContentView Grid.Row="2" Grid.Column="0"/>

  <ContentView Grid.Row="0" Grid.Column="1"/>
  <ContentView Grid.Row="1" Grid.Column="1"/>
  <ContentView Grid.Row="2" Grid.Column="1"/>

</Grid>

Kodda tanımlanan aynı “Izgara” şuna benzer:

var grid = new Grid();
grid.RowDefinitions.Add (new RowDefinition { Height = new GridLength(2, GridUnitType.Star) });
grid.RowDefinitions.Add (new RowDefinition { Height = new GridLength (1, GridUnitType.Star) });
grid.RowDefinitions.Add (new RowDefinition { Height = new GridLength(200)});
grid.ColumnDefinitions.Add (new ColumnDefinition{ Width = new GridLength (200) });

Izgaraya öğe eklemek için: “XAML” içinde:

<Grid>
      
   <--DEFINITIONS...--!>
        
   <ContentView Grid.Row="0" Grid.Column="0"/>
   <ContentView Grid.Row="1" Grid.Column="0"/>
   <ContentView Grid.Row="2" Grid.Column="0"/>

   <ContentView Grid.Row="0" Grid.Column="1"/>
   <ContentView Grid.Row="1" Grid.Column="1"/>
   <ContentView Grid.Row="2" Grid.Column="1"/>

</Grid>

C# kodunda:

var grid = new Grid();
//DEFINITIONS...
var topLeft = new Label { Text = "Top Left" };
var topRight = new Label { Text = "Top Right" };
var bottomLeft = new Label { Text = "Bottom Left" };
var bottomRight = new Label { Text = "Bottom Right" };
grid.Children.Add(topLeft, 0, 0);
grid.Children.Add(topRight, 0, 1);
grid.Children.Add(bottomLeft, 1, 0);
grid.Children.Add(bottomRight, 1, 1);

“Yükseklik” ve “Genişlik” için bir dizi birim mevcuttur.

  • Otomatik – içeriği satır veya sütuna sığdırmak için otomatik olarak boyutlandırır. C#‘da GridUnitType.Auto veya XAML’de Otomatik olarak belirtilir.
  • Oransal – sütunları ve satırları kalan boşluğun bir oranı olarak boyutlandırır. C#‘da bir değer ve GridUnitType.Star olarak belirtilir ve

istediğiniz değer olmak üzere XAML’de #* olarak. birini belirtmek #

  • ile satır/sütun, kullanılabilir alanı doldurmasına neden olur.
  • Mutlak – sütunları ve satırları belirli, sabit yükseklik ve genişlik değerleriyle boyutlandırır. C#‘da bir değer ve GridUnitType.Absolute olarak belirtilir ve XAML’de # olarak, # istediğiniz değerdir.

Not: Sütunların genişlik değerleri, Xamarin.Forms’da varsayılan olarak Otomatik olarak ayarlanır; bu, genişliğin alt öğelerin boyutundan belirlendiği anlamına gelir. Bunun, varsayılan genişliğin * olduğu ve kullanılabilir alanı dolduracağı Microsoft platformlarında XAML uygulamasından farklı olduğunu unutmayın.

İçerik Sunucusu #

Şablonlu görünümler için bir düzen yöneticisi. Sunulacak içeriğin nerede göründüğünü işaretlemek için bir ControlTemplate içinde kullanılır.

İçerik Görünümü #

Tek bir içeriğe sahip bir öğe. ContentView’ın kendi kullanımı çok azdır. Amacı, kullanıcı tanımlı bileşik görünümler için bir temel sınıf olarak hizmet etmektir.

XAML

<ContentView>
<Label Text="Hi, I'm a simple Label inside of a simple ContentView"
HorizontalOptions="Center"
VerticalOptions="Center"/>
</ContentView>

Kod

var contentView = new ContentView {
Content = new Label {
 Text = "Hi, I'm a simple Label inside of a simple ContentView",
 HorizontalOptions = LayoutOptions.Center,
 VerticalOptions = LayoutOptions.Center
}
};

Kaydırma Görünümü #

‘İçerik’ gerektiriyorsa kaydırma yapabilen bir öğe.

ScrollView düzenleri içerir ve bunların ekran dışına kaydırılmasını sağlar. “ScrollView”, klavye gösterilirken görünümlerin ekranın görünen kısmına otomatik olarak taşınmasına izin vermek için de kullanılır.

Not: “ScrollViews” iç içe geçmemelidir. Ayrıca, ‘ScrollView’, ‘ListView’ ve ‘WebView’ gibi kaydırma sağlayan diğer kontrollerle iç içe olmamalıdır.

Bir “ScrollView” tanımlaması kolaydır. XAML’de:

<ContentPage.Content>
    <ScrollView>
        <StackLayout>
            <BoxView BackgroundColor="Red" HeightRequest="600" WidthRequest="150" />
            <Entry />
        </StackLayout>
    </ScrollView>
</ContentPage.Content>

Koddaki aynı tanım:

var scroll = new ScrollView();
Content = scroll;
var stack = new StackLayout();
stack.Children.Add(new BoxView { BackgroundColor = Color.Red,   HeightRequest = 600, WidthRequest = 600 });
stack.Children.Add(new Entry());

Çerçeve #

Bazı çerçeveleme seçenekleriyle birlikte tek bir alt öğe içeren bir öğe. Çerçevenin varsayılan Xamarin.Forms.Layout.Padding değeri 20’dir.

XAML

<Frame>
<Label Text="I've been framed!"
HorizontalOptions="Center"
VerticalOptions="Center" />
</Frame>

Kod

var frameView = new Frame {
Content = new Label {
 Text = "I've been framed!",
HorizontalOptions = LayoutOptions.Center,
VerticalOptions = LayoutOptions.Center
},
OutlineColor = Color.Red
};

Şablon Görünümü #

İçeriği bir denetim şablonuyla ve “ContentView” için temel sınıfla görüntüleyen bir öğe.

Göreli Düzen #

Alt öğelerini düzenlemek için “Kısıtlamaları” kullanan bir “Düzen”.

“RelativeLayout”, görünümleri yerleşimin veya kardeş görünümlerin özelliklerine göre konumlandırmak ve boyutlandırmak için kullanılır. ‘AbsoluteLayout’tan farklı olarak, ‘RelativeLayout’ hareketli ankraj kavramına sahip değildir ve yerleşim düzeninin alt veya sağ kenarlarına göre öğeleri konumlandırma olanaklarına sahip değildir. “RelativeLayout”, öğeleri kendi sınırlarının dışında konumlandırmayı destekler.

XAML’de bir RelativeLayout şöyledir:

<RelativeLayout>
    <BoxView Color="Red" x:Name="redBox"
        RelativeLayout.YConstraint="{ConstraintExpression Type=RelativeToParent,
            Property=Height,Factor=.15,Constant=0}"
        RelativeLayout.WidthConstraint="{ConstraintExpression
            Type=RelativeToParent,Property=Width,Factor=1,Constant=0}"
        RelativeLayout.HeightConstraint="{ConstraintExpression
            Type=RelativeToParent,Property=Height,Factor=.8,Constant=0}" />
    <BoxView Color="Blue"
        RelativeLayout.YConstraint="{ConstraintExpression Type=RelativeToView,
            ElementName=redBox,Property=Y,Factor=1,Constant=20}"
        RelativeLayout.XConstraint="{ConstraintExpression Type=RelativeToView,
            ElementName=redBox,Property=X,Factor=1,Constant=20}"
        RelativeLayout.WidthConstraint="{ConstraintExpression
            Type=RelativeToParent,Property=Width,Factor=.5,Constant=0}"
        RelativeLayout.HeightConstraint="{ConstraintExpression
            Type=RelativeToParent,Property=Height,Factor=.5,Constant=0}" />
</RelativeLayout>

Aynı düzen bu kodla gerçekleştirilebilir:

layout.Children.Add (redBox, Constraint.RelativeToParent ((parent) => {
        return parent.X;
    }), Constraint.RelativeToParent ((parent) => {
        return parent.Y * .15;
    }), Constraint.RelativeToParent((parent) => {
        return parent.Width;
    }), Constraint.RelativeToParent((parent) => {
        return parent.Height * .8;
    }));
layout.Children.Add (blueBox, Constraint.RelativeToView (redBox, (Parent, sibling) => {
        return sibling.X + 20;
    }), Constraint.RelativeToView (blueBox, (parent, sibling) => {
        return sibling.Y + 20;
    }), Constraint.RelativeToParent((parent) => {
        return parent.Width * .5;
    }), Constraint.RelativeToParent((parent) => {
        return parent.Height * .5;
    }));

Yığın Düzeni #

“StackLayout”, görünümleri yatay veya dikey olarak tek boyutlu bir çizgide (“yığın”) düzenler. Bir “StackLayout"taki “Görünümler”, yerleşim seçenekleri kullanılarak yerleşimdeki alana göre boyutlandırılabilir. Konumlandırma, yerleşim düzenine eklenen görünümler ve görünümlerin yerleşim seçenekleri ile belirlenir.

XAML’de Kullanım #

<StackLayout>
    <Label Text="This will be on top" />
    <Button Text="This will be on the bottom" />
</StackLayout>

Kodda kullanım #

StackLayout stackLayout = new StackLayout
{
    Spacing = 0,
    VerticalOptions = LayoutOptions.FillAndExpand,
    Children = 
    {
        new Label
        {
            Text = "StackLayout",
            HorizontalOptions = LayoutOptions.Start
        },
        new Label
        {
            Text = "stacks its children",
            HorizontalOptions = LayoutOptions.Center
        },
        new Label
        {
            Text = "vertically",
            HorizontalOptions = LayoutOptions.End
        },
        new Label
        {
            Text = "by default,",
            HorizontalOptions = LayoutOptions.Center
        },
        new Label
        {
            Text = "but horizontal placement",
            HorizontalOptions = LayoutOptions.Start
        },
        new Label
        {
            Text = "can be controlled with",
            HorizontalOptions = LayoutOptions.Center
        },
        new Label
        {
            Text = "the HorizontalOptions property.",
            HorizontalOptions = LayoutOptions.End
        },
        new Label
        {
            Text = "An Expand option allows one or more children " +
                   "to occupy the an area within the remaining " +
                   "space of the StackLayout after it's been sized " +
                   "to the height of its parent.",
            VerticalOptions = LayoutOptions.CenterAndExpand,
            HorizontalOptions = LayoutOptions.End
        },
        new StackLayout
        {
            Spacing = 0,
            Orientation = StackOrientation.Horizontal,
            Children = 
            {
                new Label
                {
                    Text = "Stacking",
                },
                new Label
                {
                    Text = "can also be",
                    HorizontalOptions = LayoutOptions.CenterAndExpand
                },
                new Label
                {
                    Text = "horizontal.",
                },
            }
        }
    }
};
