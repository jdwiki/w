XML Eşlemeleri #

xml eşleme, hazırda bekleme eşleme dosyası olan bir “hbm.xml” dosyası kullanır. Nesne/ilişkisel eşleme için gerekli meta verileri içeren bir sözdizimi xml dosyasıdır. Meta veriler, kalıcı sınıfların bildirimini ve özelliklerin (sütunlara ve diğer varlıklarla yabancı anahtar ilişkilerine) veritabanı tablolarına eşlenmesini içerir.

Projeye Entity.hbm.xml adlı bir dosya ekleyin ve bunu özellikler sekmesinde ‘gömülü kaynak’ olarak ayarlayın. Örnek için Customer.hbm.xml:

<?xml version="1.0" encoding="utf-8" ?>
<hibernate-mapping xmlns="urn:nhibernate-mapping-2.2"
    namespace="Project" assembly="Project">

    <class name="Customer" table="CUSTOMERS">

        <id name="Id">
            <column name="Customer_Id" sql-type="int" not-null="true"/>
            <generator class="native" />
        </id>

        <!-- A cat has to have a name, but it shouldn' be too long. -->
        <property name="Name">
            <column name="Name" length="60" not-null="true" />
        </property>
        <property name="Sex" />
        <property name="Weight" />
        <property name="Active" />
        <property name="Birthday" />
    </class>

</hibernate-mapping>

“hibernate-mapping” etiketi, ad alanı ve derleme proje bilgilerini içerir. ‘class’ etiketi, projedeki varlığın adını ve haritalanmış tabloyu içerir. “id” etiketi, sütunun “column” etiketiyle belirtildiği ve “generator” etiketinin kimliğin nasıl oluşturulacağını tanımladığı “birincil anahtar” için eşlemeyi içerir. “özellik” etiketi, veritabanındaki diğer sütunlar için bilgi içerir.

Akıcı NHibernate Eşlemeleri #

Fluent NHibernate, varlıkları xml eşlemeleri yerine C# kodunu kullanarak eşlemenize yardımcı olacak bir kitaplıktır. Fluent NHibernate, fluent pattern kullanır ve eşlemeleri oluşturmak için konvansiyonlara dayanır ve size görsel stüdyo araçlarının (intellisense gibi) gücünü verir. ) varlıklarınızı haritalama şeklinizi geliştirmek için.

Projenize Fluent NHibernate from Nuget referansını ekleyin ve bir CustomerMap.cs sınıfı ekleyin:

namespace Project.Mappings
{
    public class CustomerMap : ClassMap<Customer>
    {
        public CustomerMap()
        {
            Table("CUSTOMERS");

            Id(x => x.Id).Column("Customer_Id").GeneratedBy.Native();
            
            //map a property while specifying the max-length as well as setting 
            //it as not nullable. Will result in the backing column having
            //these characteristics, but this will not be enforced in the model!
            Map(x => x.Name)
                .Length(16)
                .Not.Nullable();

            Map(x => x.Sex);

            Map(x => x.Weight);

            Map(x => x.Active);

            //Map a property while specifying the name of the column in the database
            Map(x => x.Birthday, "BIRTHDAY");

            //Maps a many-to-one relationship
            References(x => x.Company);

            //Maps a one-to-many relationship, while also defining which 
            //column to use as key in the foreign table.
            HasMany(x => x.Orders).KeyColumn("CustomerPk");
        }
    }
}

“CustomerMap” sınıfı, haritalama için temel sınıf olan “ClassMap“den devralır ve “T” varlığınızın haritasını oluşturmak için gerekli tüm yöntemleri içerir. “Tablo” yöntemi, eşlediğiniz tablo adını tanımlar. ‘Id’ yöntemi, ‘birincil anahtar’ sütununu eşlemek için kullanılır. Map yöntemi diğer sütunları eşlemek için kullanılır.

Modelden Haritaya Örnek #

NHibernate, tablolara veya görünümlere eşlemek için sınıfları kullanır. Bir Düz Eski CLR Nesnesi (bazen Düz Sıradan CLR Nesneleri olarak adlandırılan POCO’lar) oluşturmak kalıcı sınıflar için iyi bir uygulamadır. Bir POCO verisine standart .NET özellik mekanizmaları aracılığıyla erişilebilir ve dahili temsili herkesin görebileceği arabirimden korur.

namespace Project
{
    public class Customer
    {
        public virtual string Id { get; set; }

        public virtual string Name { get; set; }

        public virtual char Sex { get; set; }

        public virtual float Weight { get; set;}

        public virtual bool Active { get; set;}

        public virtual DateTime Birthday { get; set;}

        public Customer()
        {
        }
    }
}

NHibernate, özellik türlerinin kullanımıyla sınırlı değildir: ‘System.Collections’ ve ‘System.Collections.Generics’ ad alanlarındaki sınıflar da dahil olmak üzere tüm .NET türleri ve ilkelleri (string, char ve DateTime gibi) eşlenebilir. Ayrıca, başka bir varlık türüne başvuran özelliklere sahip varlıklar arasında bir ilişki eşleyebilirsiniz. Bunları değerler, değer koleksiyonları veya diğer varlıklarla ilişkilendirmeler olarak eşleyebilirsiniz. Buradaki “Id” adlı özellik, o sınıfın veritabanı tanımlayıcısını (birincil anahtar) temsil eden ve Cat gibi varlıklar için şiddetle tavsiye edilen özel bir özelliktir. NHibernate tanımlayıcıları sınıfta bildirmek zorunda kalmadan yalnızca dahili olarak kullanabilir, ancak uygulama mimarimizdeki esnekliğin bir kısmını kaybederiz.

Kalıcı sınıflar için özel bir arabirim uygulanmamalı veya özel bir kalıcı kalıcı sınıftan alt sınıflamamız gerekmemektedir. NHibernate ayrıca IL manipülasyonu gibi herhangi bir inşa süresi işleme kullanmaz; yalnızca .NET yansımasına ve çalışma zamanı sınıfı geliştirmesine dayanır. Böylece, NHibernate’deki POCO sınıfında herhangi bir bağımlılık olmadan, onu bir veritabanı tablosuna veya görünümüne eşleyebiliriz.

Yukarıda belirtilen çalışma zamanı sınıfı geliştirmesinin çalışması için NHibernate, bir varlık sınıfının tüm genel özelliklerinin ‘sanal’ olarak bildirilmesini gerektirir. NHibernate’in nesneleri oluşturması için varlık sınıfının bağımsız değişkensiz bir oluşturucuya (“korumalı” veya “genel”) sahip olması gerekir.
