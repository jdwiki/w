Fragmanlar ve onların iletişim mekanizmaları hakkında giriş

Sözdizimi #

  • void onActivityCreated(Bundle saveInstanceState) // Parçanın etkinliği oluşturulduğunda ve bu parçanın görünüm hiyerarşisi başlatıldığında çağrılır.

  • void onActivityResult(int requestCode, int resultCode, Intent data) // startActivityForResult(Intent, int) için önceki bir çağrının sonucunu alın.

  • void onAttach(Activity Activity) // Bu yöntem API düzeyi 23’te kullanımdan kaldırıldı. Bunun yerine onAttach(Context) kullanın.

  • void onAttach(Bağlam bağlamı) // Bir parça kendi bağlamına ilk eklendiğinde çağrılır.

  • void onAttachFragment(Fragment childFragment) // Bir parça bu parçanın alt öğesi olarak eklendiğinde çağrılır.

  • void onConfigurationChanged(Configuration newConfig) // Bileşeniniz çalışırken cihaz konfigürasyonu değiştiğinde sistem tarafından çağrılır.

  • void onCreate(Bundle saveInstanceState) // Bir parçanın ilk oluşturulmasını yapmak için çağrılır.

  • View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle saveInstanceState) // Fragmanın kullanıcı arayüzü görünümünü başlatması için çağrılır.

  • void onDestroy() // Parça artık kullanımda olmadığında çağrılır.

  • void onDestroyView() // daha önce onCreateView(LayoutInflater, ViewGroup, Bundle) tarafından oluşturulan görünüm parçadan ayrıldığında çağrılır.

  • void onDetach() // Parça artık etkinliğine bağlı olmadığında çağrılır.

  • void onInflate(Etkinlik etkinliği, AttributeSet öznitelikleri, Bundle saveInstanceState) // Bu yöntem API düzeyi 23’te kullanımdan kaldırıldı. Bunun yerine onInflate(Context, AttributeSet, Bundle) kullanın.

  • void onInflate(Bağlam bağlamı, AttributeSet öznitelikleri, Bundle saveInstanceState) // Tipik olarak bir etkinliğin içerik görünümünü ayarlayarak bir görünüm düzeni enflasyonunun parçası olarak bir parça oluşturulduğunda çağrılır.

  • void onPause() // Fragment artık devam ettirilmediğinde çağrılır.

  • void onResume() // Parça kullanıcı tarafından görüldüğünde ve aktif olarak çalıştığında çağrılır.

  • void onSaveInstanceState(Bundle outState) // Parçanın mevcut dinamik durumunu kaydetmesini istemek için çağrılır, böylece daha sonra yeniden oluşturulabilir, işlemin yeni bir örneğinde yeniden başlatılır.

  • void onStart() // Fragment kullanıcı tarafından görüldüğünde çağrılır.

  • void onStop() // Fragment artık başlatılmadığında çağrılır.

  • void onViewStateRestored(Bundle saveInstanceState) // Kaydedilen tüm durumlar parçanın görünüm hiyerarşisine geri yüklendiğinde çağrılır.

Fragment, bir Aktivitedeki bir davranışı veya kullanıcı arayüzünün bir bölümünü temsil eder. Çok bölmeli bir kullanıcı arabirimi oluşturmak ve bir parçayı birden çok etkinlikte yeniden kullanmak için birden çok parçayı tek bir etkinlikte birleştirebilirsiniz. Bir parçayı, kendi yaşam döngüsüne sahip, kendi girdi olaylarını alan ve aktivite çalışırken ekleyebileceğiniz veya çıkarabileceğiniz (bir tür “alt aktivite” gibi) bir aktivitenin modüler bir bölümü olarak düşünebilirsiniz. farklı etkinliklerde yeniden kullanma).

Yapıcı

Her parçanın bir boş kurucu olması gerekir, bu nedenle faaliyet durumunu geri yüklerken somutlaştırılabilir. Parça yeniden başlatıldığında bu kurucular çağrılmayacağından, alt sınıfların parametreli başka kuruculara sahip olmaması şiddetle tavsiye edilir; bunun yerine, argümanlar çağıran tarafından setArguments(Bundle) ile sağlanabilir ve daha sonra Fragment tarafından getArguments() ile alınabilir.

Bundle kullanarak verileri Activity’den Fragment’a aktarın #

Tüm parçaların boş bir yapıcısı olmalıdır (yani, girdi bağımsız değişkeni olmayan bir yapıcı yöntemi). Bu nedenle, verilerinizi oluşturulan Fragment’e iletmek için `setArguments()’ yöntemini kullanmalısınız. Bu yöntemler, verilerinizi içinde sakladığınız bir paket alır ve Paketi bağımsız değişkenlerde saklar. Daha sonra, bu Paket daha sonra Fragment’in ‘onCreate()’ ve ‘onCreateView()’ geri aramalarında alınabilir.

Aktivite:

 Bundle bundle = new Bundle();
 String myMessage = "Stack Overflow is cool!";
 bundle.putString("message", myMessage );
 FragmentClass fragInfo = new FragmentClass();
 fragInfo.setArguments(bundle);
 transaction.replace(R.id.fragment_single, fragInfo);
 transaction.commit();

Parça:

 @Override
 public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
    String myValue = this.getArguments().getString("message");
    ...
 }

newInstance() kalıbı #

Parametrelerle bir parça oluşturucu oluşturmak mümkün olsa da, Android parçaları yeniden oluştururken sıfır bağımsız değişken yapıcısını dahili olarak çağırır (örneğin, Android’in kendi nedenleriyle öldürüldükten sonra geri yükleniyorlarsa). Bu nedenle, parametreleri olan bir kurucuya güvenmek tavsiye edilmez.

Beklenen parça argümanlarınızın her zaman mevcut olduğundan emin olmak için parçayı oluşturmak için statik bir “newInstance()” yöntemini kullanabilir ve yeni bir örnek oluştururken kullanılabilecek bir pakete istediğiniz parametreleri koyabilirsiniz.

import android.os.Bundle;
import android.support.v4.app.Fragment;

public class MyFragment extends Fragment
{
  // Our identifier for obtaining the name from arguments
  private static final String NAME_ARG = "name";

  private String mName;

  // Required
  public MyFragment(){}

  // The static constructor.  This is the only way that you should instantiate
  // the fragment yourself
  public static MyFragment newInstance(final String name) {
    final MyFragment myFragment = new MyFragment();
    // The 1 below is an optimization, being the number of arguments that will
    // be added to this bundle.  If you know the number of arguments you will add
    // to the bundle it stops additional allocations of the backing map.  If
    // unsure, you can construct Bundle without any arguments
    final Bundle args = new Bundle(1);

    // This stores the argument as an argument in the bundle.  Note that even if
    // the 'name' parameter is NULL then this will work, so you should consider
    // at this point if the parameter is mandatory and if so check for NULL and
    // throw an appropriate error if so
    args.putString(NAME_ARG, name);

    myFragment.setArguments(args);
    return myFragment;
  }

  @Override
  public void onCreate(final Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    final Bundle arguments = getArguments();
    if (arguments == null || !arguments.containsKey(NAME_ARG)) {
      // Set a default or error as you see fit
    } else {
      mName = arguments.getString(NAME_ARG);
    }
  }
}

Şimdi, Etkinlikte:

FragmentTransaction ft = getSupportFragmentManager().beginTransaction();
MyFragment mFragment = MyFragment.newInstance("my name");
ft.replace(R.id.placeholder, mFragment);
//R.id.placeholder is where we want to load our fragment
ft.commit();

Bu model, gerekli tüm argümanların oluşturma sırasında parçalara iletilmesini sağlamak için en iyi uygulamadır. Sistem parçayı yok ettiğinde ve daha sonra yeniden oluşturduğunda, durumunu otomatik olarak geri yükleyeceğini unutmayın - ancak ona bir onSaveInstanceState(Bundle) uygulaması sağlamanız gerekir.

Backstack ve statik kumaş deseni kullanarak parçalar arasında gezinme #

Her şeyden önce, ilk Fragmentimizi başlangıçta eklememiz gerekiyor, bunu Aktivitemizin onCreate() metodunda yapmalıyız:

if (null == savedInstanceState) {
    getSupportFragmentManager().beginTransaction()
      .addToBackStack("fragmentA")
      .replace(R.id.container, FragmentA.newInstance(), "fragmentA")
      .commit();
}

Ardından, backstack’imizi yönetmemiz gerekiyor. En kolay yol, tüm FragmentTransactions için kullanılan etkinliğimize eklenen bir işlevi kullanmaktır.

public void replaceFragment(Fragment fragment, String tag) {
    //Get current fragment placed in container
    Fragment currentFragment = getSupportFragmentManager().findFragmentById(R.id.container);

    //Prevent adding same fragment on top
    if (currentFragment.getClass() == fragment.getClass()) {
        return;
    }

    //If fragment is already on stack, we can pop back stack to prevent stack infinite growth
    if (getSupportFragmentManager().findFragmentByTag(tag) != null) {
        getSupportFragmentManager().popBackStack(tag, FragmentManager.POP_BACK_STACK_INCLUSIVE);
    }

    //Otherwise, just replace fragment
    getSupportFragmentManager()
            .beginTransaction()
            .addToBackStack(tag)
            .replace(R.id.container, fragment, tag)
            .commit();
}

Son olarak, backstack’te mevcut olan son Fragment’ten geri dönerken uygulamadan çıkmak için onBackPressed()i geçersiz kılmalıyız.

@Override
public void onBackPressed() {
    int fragmentsInStack = getSupportFragmentManager().getBackStackEntryCount();
    if (fragmentsInStack > 1) { // If we have more than one fragment, pop back stack
        getSupportFragmentManager().popBackStack();
    } else if (fragmentsInStack == 1) { // Finish activity, if only one fragment left, to prevent leaving empty screen
        finish();
    } else {
        super.onBackPressed();
    }
}

Faaliyette yürütme:

replaceFragment(FragmentB.newInstance(), "fragmentB");

Faaliyet dışında yürütme (“MainActivity"nin bizim faaliyetimiz olduğunu varsayarsak):

((MainActivity) getActivity()).replaceFragment(FragmentB.newInstance(), "fragmentB");

Geri arama arayüzü ile etkinlikleri bir etkinliğe geri gönderme #

Olayları parçadan etkinliğe göndermeniz gerekiyorsa, olası çözümlerden biri geri arama arabirimi tanımlamak ve ana bilgisayar etkinliğinin bunu uygulamasını gerektirmektir.

Örnek

Parçanın düğmesi tıklandığında bir etkinliğe geri arama gönder

Her şeyden önce, geri arama arayüzünü tanımlayın: 
public interface SampleCallback {
    void onButtonClicked();
}

Sonraki adım, bu geri aramayı parçaya atamaktır:

public final class SampleFragment extends Fragment {

    private SampleCallback callback;

    @Override
    public void onAttach(Context context) {
        super.onAttach(context);
        if (context instanceof SampleCallback) {
            callback = (SampleCallback) context;
        } else {
            throw new RuntimeException(context.toString()
                    + " must implement SampleCallback");
        }
    }

    @Override
    public void onDetach() {
        super.onDetach();
        callback = null;
    }

    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
        final View view = inflater.inflate(R.layout.sample, container, false);
        // Add button's click listener
        view.findViewById(R.id.actionButton).setOnClickListener(new View.OnClickListener() {
            public void onClick(View v) {
                callback.onButtonClicked(); // Invoke callback here
            }
        });
        return view;
    }
}

Ve son olarak, aktivitede geri aramayı uygulayın:

public final class SampleActivity extends Activity implements SampleCallback {

    // ... Skipped code with settings content view and presenting the fragment

    @Override
    public void onButtonClicked() {
        // Invoked when fragment's button has been clicked
    }
}

Parçalar arasındaki geçişi canlandırın #

Parçalar arasındaki geçişi canlandırmak veya bir parçayı gösterme veya gizleme sürecini canlandırmak için bir ‘FragmentTransaction’ oluşturmak için ‘FragmentManager’ı kullanırsınız.

Tek bir “FragmentTransaction” için, animasyon gerçekleştirmenin iki farklı yolu vardır: standart bir animasyon kullanabilir veya kendi özel animasyonlarınızı sağlayabilirsiniz.

Standart animasyonlar, “FragmentTransaction.setTransition(int transit)” çağrılarak ve “FragmentTransaction” sınıfında bulunan önceden tanımlanmış sabitlerden biri kullanılarak belirlenir. Yazma sırasında, bu sabitler şunlardır:

FragmentTransaction.TRANSIT_NONE
FragmentTransaction.TRANSIT_FRAGMENT_OPEN
FragmentTransaction.TRANSIT_FRAGMENT_CLOSE
FragmentTransaction.TRANSIT_FRAGMENT_FADE

Tam işlem şöyle görünebilir:

getSupportFragmentManager()
    .beginTransaction()
    .setTransition(FragmentTransaction.TRANSIT_FRAGMENT_FADE)
    .replace(R.id.contents, new MyFragment(), "MyFragmentTag")
    .commit();

Özel animasyonlar, FragmentTransaction.setCustomAnimations(int enter, int çıkış) veya FragmentTransaction.setCustomAnimations(int enter, int çıkış, int popEnter, int popExit) çağrılarak belirlenir.

‘Giriş’ ve ‘çıkış’ animasyonları, arka yığından fragmanlar çıkarmayı içermeyen ‘FragmentTransaction’lar için oynatılacaktır. “popEnter” ve “popExit” animasyonları, arka yığından bir parça çıkarıldığında oynatılacaktır.

Aşağıdaki kod, bir parçayı dışarı kaydırarak ve diğerini yerine kaydırarak bir parçayı nasıl değiştireceğinizi gösterir.

getSupportFragmentManager()
    .beginTransaction()
    .setCustomAnimations(R.anim.slide_in_left, R.anim.slide_out_right)
    .replace(R.id.contents, new MyFragment(), "MyFragmentTag")
    .commit();

XML animasyon tanımları “objectAnimator” etiketini kullanır. Bir slide_in_left.xml örneği şöyle görünebilir:

Parçalar Arası İletişim #

Fragmanlar arasındaki tüm iletişimler bir Aktivite üzerinden yapılmalıdır. Parçalar, bir Etkinlik olmadan birbirleriyle iletişim CANNOT.

Ek kaynaklar

  • https://stackoverflow.com/questions/24777985/how-to-implement-onfragmentinteractionlistener

Bu örnekte, sırasıyla bir “mesaj” (bu durumda basit bir String) göndermek ve almak için “SenderFragment” ve “ReceiverFragment” adlı iki parçayı barındıran bir “MainActivity"ye sahibiz.

SenderFragment içindeki bir Düğme, mesajı gönderme işlemini başlatır. ReceiverFragment içindeki bir TextView, mesaj kendisi tarafından alındığında güncellenir.

Önemli kod satırlarını açıklayan yorumlarla birlikte MainActivity snippet’i aşağıdadır:

// Our MainActivity implements the interface defined by the SenderFragment. This enables
// communication from the fragment to the activity
public class MainActivity extends AppCompatActivity implements SenderFragment.SendMessageListener {

@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
}

/**
 * This method is called when we click on the button in the SenderFragment
 * @param message The message sent by the SenderFragment
 */
@Override
public void onSendMessage(String message) {
    // Find our ReceiverFragment using the SupportFragmentManager and the fragment's id
    ReceiverFragment receiverFragment = (ReceiverFragment)
            getSupportFragmentManager().findFragmentById(R.id.fragment_receiver);

    // Make sure that such a fragment exists
    if (receiverFragment != null) {
        // Send this message to the ReceiverFragment by calling its public method
        receiverFragment.showMessage(message);
    }
}
}

Önemli kod satırlarını açıklayan “SenderFragment” için kod parçacığı aşağıdadır:

public class SenderFragment extends Fragment {

private SendMessageListener commander;

/**
 * This interface is created to communicate between the activity and the fragment. Any activity
 * which implements this interface will be able to receive the message that is sent by this
 * fragment.
 */
public interface SendMessageListener {
    void onSendMessage(String message);
}

/**
 * API LEVEL >= 23
 * <p>
 * This method is called when the fragment is attached to the activity. This method here will
 * help us to initialize our reference variable, 'commander' , for our interface
 * 'SendMessageListener'
 *
 * @param context
 */
@Override
public void onAttach(Context context) {
    super.onAttach(context);
    // Try to cast the context to our interface SendMessageListener i.e. check whether the
    // activity implements the SendMessageListener. If not a ClassCastException is thrown.
    try {
        commander = (SendMessageListener) context;
    } catch (ClassCastException e) {
        throw new ClassCastException(context.toString()
                + "must implement the SendMessageListener interface");
    }
}

/**
 * API LEVEL < 23
 * <p>
 * This method is called when the fragment is attached to the activity. This method here will
 * help us to initialize our reference variable, 'commander' , for our interface
 * 'SendMessageListener'
 *
 * @param activity
 */
@Override
public void onAttach(Activity activity) {
    super.onAttach(activity);
    // Try to cast the context to our interface SendMessageListener i.e. check whether the
    // activity implements the SendMessageListener. If not a ClassCastException is thrown.
    try {
        commander = (SendMessageListener) activity;
    } catch (ClassCastException e) {
        throw new ClassCastException(activity.toString()
                + "must implement the SendMessageListener interface");
    }
}

@Nullable
@Override
public View onCreateView(LayoutInflater inflater, @Nullable ViewGroup container,
                         @Nullable Bundle savedInstanceState) {
    // Inflate view for the sender fragment.
    View view = inflater.inflate(R.layout.fragment_receiver, container, false);

    // Initialize button and a click listener on it
    Button send = (Button) view.findViewById(R.id.bSend);
    send.setOnClickListener(new View.OnClickListener() {
        @Override
        public void onClick(View v) {

            // Sanity check whether we were able to properly initialize our interface reference
            if (commander != null) {

                // Call our interface method. This enables us to call the implemented method
                // in the activity, from where we can send the message to the ReceiverFragment.
                commander.onSendMessage("HELLO FROM SENDER FRAGMENT!");
            }
        }
    });

    return view;
}
}

}

}
}
