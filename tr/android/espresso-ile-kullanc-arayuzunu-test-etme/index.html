Cihaz kurulumu #

Kesintisiz test için cihazlarınızda aşağıdaki ayarları yapmanız önerilir:

  • Geliştirici seçenekleri / Animasyonları Devre Dışı Bırak - testlerin düzensizliğini azaltır
  • Geliştirici seçenekleri / Uyanık kal - testler için özel cihazlarınız varsa bu yararlıdır
  • Geliştirici seçenekleri / Kaydedici arabellek boyutları - telefonunuzda çok büyük test paketleri çalıştırıyorsanız daha yüksek bir sayıya ayarlayın
  • Erişilebilirlik / Dokun ve Tut gecikmesi - espressoda dokunma ile ilgili sorunları önlemek için uzun süre

Gerçek dünyadan oldukça bir kurulum ha? Pekala, şimdi bu yoldan çıktığında, küçük bir testin nasıl kurulacağına bir göz atalım.

Testi yazma #

Aşağıdaki ekrana sahip olduğumuzu varsayalım: Ekran şunları içerir:

  • metin giriş alanı - R.id.textEntry
  • tıklandığında metinle birlikte snackbar’ı gösteren düğme - R.id.shownSnackbarBtn
  • kullanıcı tarafından yazılan metni içermesi gereken snackbar - android.support.design.R.id.snackbar_text

Şimdi akışımızı test edecek bir sınıf oluşturalım:

/**
* Testing of the snackbar activity.
**/
@RunWith(AndroidJUnit4.class)
@LargeTest
public class SnackbarActivityTest{
    //espresso rule which tells which activity to start
    @Rule
    public final ActivityTestRule<SnackbarActivity> mActivityRule = 
        new ActivityTestRule<>(SnackbarActivity.class, true, false);


    @Override
    public void tearDown() throws Exception {
        super.tearDown();
        //just an example how tear down should cleanup after itself
        mDatabase.clear();
        mSharedPrefs.clear();
    }
    
    @Override
    public void setUp() throws Exception {
        super.setUp();
        //setting up your application, for example if you need to have a user in shared
        //preferences to stay logged in you can do that for all tests in your setup
        User mUser = new User();
        mUser.setToken("randomToken");
    }
    
    /**
    *Test methods should always start with "testXYZ" and it is a good idea to 
    *name them after the intent what you want to test
    **/
    @Test
    public void testSnackbarIsShown() {
        //start our activity
        mActivityRule.launchActivity(null);
        //check is our text entry displayed and enter some text to it
        String textToType="new snackbar text";
        onView(withId(R.id.textEntry)).check(matches(isDisplayed()));
        onView(withId(R.id.textEntry)).perform(typeText(textToType));
        //click the button to show the snackbar
        onView(withId(R.id.shownSnackbarBtn)).perform(click());
        //assert that a view with snackbar_id with text which we typed and is displayed
        onView(allOf(withId(android.support.design.R.id.snackbar_text), 
        withText(textToType))) .check(matches(isDisplayed()));
    }
}

Fark ettiğiniz gibi, sık sık geldiğini fark edebileceğiniz 3-4 şey var:

onView(withXYZ) <– viewMatchers ile ekrandaki öğeleri bulabilirsiniz

perform(click()) <– viewActions, daha önce bulduğunuz öğeler üzerinde eylemler yürütebilirsiniz

check(matches(isDisplayed())) <– viewAssertions, daha önce bulduğunuz ekranlarda yapmak istediğiniz kontroller

Bunların tümü ve diğerleri burada bulunabilir: https://google.github.io/android-testing-support-library/docs/espresso/cheatsheet/index.html

İşte bu, şimdi sınıf adına / teste sağ tıklayıp Testi çalıştır’ı seçerek veya komutla testi çalıştırabilirsiniz:

./gradlew connectedFLAVORNAMEAndroidTest

Genel Espresso #

Espresso Kurulumu :

androidTestCompile 'com.android.support.test.espresso:espresso-core:2.2.2'

ViewMatchers – Eşleştirici<? süper Görünüm> arayüzü. Geçerli görünüm hiyerarşisi içinde bir görünümü bulmak için bunlardan bir veya daha fazlasını ‘onView’ yöntemine iletebilirsiniz.

ViewActions – ‘ViewInteraction.perform()’ yöntemine geçirilebilen bir ‘ViewActions’ koleksiyonu (örneğin, ‘click()’).

ViewAssertions – ‘ViewInteraction.check()’ yönteminden geçirilebilen bir ‘ViewAssertions’ koleksiyonu. Çoğu zaman, şu anda seçili olan görünümün durumunu belirtmek için bir Görünüm eşleştirici kullanan eşleşme iddiasını kullanacaksınız.

    return new ViewAction() {
      @Override public Matcher<View> getConstraints() {
        return isAssignableFrom(DrawerLayout.class);
      }

      @Override public String getDescription() {
        return "open drawer";
      }

      @Override public void perform(UiController uiController, View view) {
        ((DrawerLayout) view).openDrawer(GravityCompat.START);
      }
    };
  }

  public static ViewAction actionCloseDrawer() {
    return new ViewAction() {
      @Override public Matcher<View> getConstraints() {
        return isAssignableFrom(DrawerLayout.class);
      }

      @Override public String getDescription() {
        return "close drawer";
      }

      @Override public void perform(UiController uiController, View view) {
        ((DrawerLayout) view).closeDrawer(GravityCompat.START);
      }
    };
  }

Bir görünümde eylem gerçekleştirme #

Perform yöntemini kullanarak bir görünümde ViewActions gerçekleştirmek mümkündür. ‘ViewActions’ sınıfı, aşağıdakiler gibi en yaygın eylemler için yardımcı yöntemler sağlar:

ViewActions.click()
ViewActions.typeText()
ViewActions.clearText()

Örneğin, görünüme tıklamak için:

onView(...).perform(click());
onView(withId(R.id.button_simple)).perform(click());

Bir gerçekleştirme çağrısı ile birden fazla eylemi gerçekleştirebilirsiniz:

onView(...).perform(typeText("Hello"), click());

Çalıştığınız görünüm bir ‘ScrollView’ (dikey veya yatay) içinde bulunuyorsa, görünümün görüntülenmesini gerektiren önceki eylemleri (‘click()’ ve ’typeText()’ gibi) ‘scrollTo()’ ile düşünün. . Bu, diğer eyleme geçmeden önce görünümün görüntülenmesini sağlar:

onView(...).perform(scrollTo(), click());

onView ile bir görünüm bulma #

ViewMatchers ile mevcut görünüm hiyerarşisinde görünümü bulabilirsiniz.

Bir görünüm bulmak için, doğru görünümü seçen bir görünüm eşleştirici ile onView() yöntemini kullanın. onView() yöntemleri, ViewInteraction türünde bir nesne döndürür.

Örneğin, “R.id” ile bir görünüm bulmak şu kadar basittir:

onView(withId(R.id.my_view))

Metin içeren bir görünüm bulma:

onView(withText("Hello World"))

Espresso Test Sınıfı Oluşturun #

Sonraki Java sınıfını src/androidTest/java içine yerleştirin ve çalıştırın.

public class UITest {

  @Test public void Simple_Test() {
    onView(withId(R.id.my_view))         // withId(R.id.my_view) is a ViewMatcher
        .perform(click())                // click() is a ViewAction
        .check(matches(isDisplayed()));  // matches(isDisplayed()) is a ViewAssertion
  }

}

Espresso özel eşleştiriciler #

Espresso, varsayılan olarak, onlarla bazı kontroller veya etkileşimler yapmanız gereken görünümleri bulmanıza yardımcı olan birçok eşleştiriciye sahiptir.

En önemlileri aşağıdaki hile sayfasında bulunabilir:

https://google.github.io/android-testing-support-library/docs/espresso/cheatsheet/

Bazı eşleştirici örnekleri şunlardır:

  • withId(R.id.ID_of_object_you_are_look_for);
  • withText(“Nesnenin olmasını beklediğiniz bazı metinler”);
  • isDisplayed() <– görünümün görünür olup olmadığını kontrol edin
  • yokNotExist() <– görünümün mevcut olmadığını kontrol edin

Bunların hepsi günlük kullanım için çok faydalıdır, ancak daha karmaşık görünümleriniz varsa özel eşleştiricilerinizi yazmak testleri daha okunaklı hale getirebilir ve bunları farklı yerlerde yeniden kullanabilirsiniz.

Genişletebileceğiniz en yaygın 2 eşleştirici türü vardır: TypeSafeMatcher Sınırlı Eşleştirici

TypeSafeMatcher’ı uygulamak, iddia ettiğiniz görünümün örneğini kontrol etmenizi gerektirir, eğer bu doğru türse, bazı özelliklerini bir eşleştiriciye sağladığınız bir değerle eşleştirirsiniz.

Örneğin, bir resim görünümünün doğru çizilebilirliğe sahip olduğunu doğrulayan güvenli eşleştirici yazın:

public class DrawableMatcher extends TypeSafeMatcher<View> {

    private @DrawableRes final int expectedId;
    String resourceName;
    
    public DrawableMatcher(@DrawableRes int expectedId) {
        super(View.class);
        this.expectedId = expectedId;
    }

    @Override
    protected boolean matchesSafely(View target) {
        //Type check we need to do in TypeSafeMatcher
        if (!(target instanceof ImageView)) {
            return false;
        }
        //We fetch the image view from the focused view
        ImageView imageView = (ImageView) target;
        if (expectedId < 0) {
            return imageView.getDrawable() == null;
        }
        //We get the drawable from the resources that we are going to compare with image view source
        Resources resources = target.getContext().getResources();
        Drawable expectedDrawable = resources.getDrawable(expectedId);
        resourceName = resources.getResourceEntryName(expectedId);

        if (expectedDrawable == null) {
            return false;
        }
        //comparing the bitmaps should give results of the matcher if they are equal
        Bitmap bitmap = ((BitmapDrawable) imageView.getDrawable()).getBitmap();
        Bitmap otherBitmap = ((BitmapDrawable) expectedDrawable).getBitmap();
        return bitmap.sameAs(otherBitmap);
    }


    @Override
    public void describeTo(Description description) {
        description.appendText("with drawable from resource id: ");
        description.appendValue(expectedId);
        if (resourceName != null) {
            description.appendText("[");
            description.appendText(resourceName);
            description.appendText("]");
        }
    }
}

Eşleştiricinin kullanımı şu şekilde sarılabilir:

  public static Matcher<View> withDrawable(final int resourceId) {
    return new DrawableMatcher(resourceId);
}

 onView(withDrawable(R.drawable.someDrawable)).check(matches(isDisplayed()));

Sınırlı eşleştiriciler benzerdir, yalnızca tür kontrolünü yapmanız gerekmez, ancak bu sizin için otomatik olarak yapıldığından:

 /**
 * Matches a {@link TextInputFormView}'s input hint with the given resource ID
 *
 * @param stringId
 * @return
 */
public static Matcher<View> withTextInputHint(@StringRes final int stringId) {
    return new BoundedMatcher<View, TextInputFormView>(TextInputFormView.class) {
        private String mResourceName = null;

        @Override
        public void describeTo(final Description description) {
            //fill these out properly so your logging and error reporting is more clear
            description.appendText("with TextInputFormView that has hint ");
            description.appendValue(stringId);
            if (null != mResourceName) {
                description.appendText("[");
                description.appendText(mResourceName);
                description.appendText("]");
            }
        }

        @Override
        public boolean matchesSafely(final TextInputFormView view) {
            if (null == mResourceName) {
                try {
                    mResourceName = view.getResources().getResourceEntryName(stringId);
                } catch (Resources.NotFoundException e) {
                    throw new IllegalStateException("could not find string with ID " + stringId, e);
                }
            }
            return view.getResources().getString(stringId).equals(view.getHint());
        }
    };
}
