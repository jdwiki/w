Lint aracı, Android proje kaynak dosyalarınızı olası hatalar ve doğruluk, güvenlik, performans, kullanılabilirlik, erişilebilirlik ve uluslararasılaştırma için optimizasyon iyileştirmeleri için kontrol eder. Lint’i komut satırından veya Android Studio’dan çalıştırabilirsiniz.

Resmi belgeler: #

https://developer.android.com/studio/write/lint.html

Araçları kullanma:xml dosyalarında yoksay #

’tools:ignore’ özniteliği, tüy bırakmayan uyarıları yok saymak için xml dosyalarında kullanılabilir.

AMA bu teknikle tiftik uyarılarını reddetmek çoğu zaman yanlış bir yoldur.

Tüy bırakmayan bir uyarı anlaşılmalı ve düzeltilmelidir… ancak ve ancak anlamını tam olarak anlarsanız ve onu görmezden gelmek için güçlü bir nedeniniz varsa, göz ardı edilebilir.

Tüy bırakmayan bir uyarıyı görmezden gelmenin meşru olduğu bir kullanım örneği:

  • Bir sistem uygulaması geliştiriyorsunuz (cihaz üreticisi anahtarıyla imzalanmış)
  • Uygulamanızın cihaz tarihini (veya diğer herhangi bir korunan eylemi) değiştirmesi gerekiyor

O zaman bunu manifestinizde yapabilirsiniz: (yani, korumalı izni talep etmek ve tiftik uyarısını görmezden gelmek, çünkü sizin durumunuzda iznin verileceğini biliyorsunuz)

<manifest xmlns:android="http://schemas.android.com/apk/res/android"
      xmlns:tools="http://schemas.android.com/tools"
      ...>
    <uses-permission android:name="android.permission.SET_TIME"
        tools:ignore="ProtectedPermissions"/>

LintOptions’ı gradle ile yapılandırın #

‘build.gradle’ dosyasına bir ’lintOptions’ bölümü ekleyerek tiftiği yapılandırabilirsiniz:

android {

    //.....

    lintOptions {
        // turn off checking the given issue id's
        disable 'TypographyFractions','TypographyQuotes'

        // turn on the given issue id's
        enable 'RtlHardcoded','RtlCompat', 'RtlEnabled'

        // check *only* the given issue id's
        check 'NewApi', 'InlinedApi'

       // set to true to turn off analysis progress reporting by lint
       quiet true
 
       // if true, stop the gradle build if errors are found
       abortOnError false
   
       // if true, only report errors
       ignoreWarnings true
    }
}

Belirli bir varyant için lint çalıştırabilirsiniz (aşağıya bakın), ör. ./gradlew lintRelease veya tüm varyantlar için (./gradlew lint), bu durumda, belirli bir sorunun hangi belirli varyantlar için geçerli olduğunu açıklayan bir rapor üretir.

Mevcut tüm seçenekler için DSL referansı için burayı kontrol edin.

1: http://google.github.io/android-gradle-dsl/current/com.android.build.gradle.internal.dsl.LintOptions.html#com.android.build.gradle.internal.dsl. LintOptions

Java ve XML kaynak dosyalarında tiftik kontrolünü yapılandırma #

Lint kontrolünü Java ve XML kaynak dosyalarınızdan devre dışı bırakabilirsiniz.

Java’da tiftik kontrolünü yapılandırma #

Android projenizde özellikle bir Java sınıfı veya yöntemi için Lint denetimini devre dışı bırakmak için, o Java koduna @SuppressLint ek açıklamasını ekleyin.

Örnek:

@SuppressLint("NewApi")
@Override
public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.main);

Tüm Lint sorunlarını kontrol etmeyi devre dışı bırakmak için:

@SuppressLint("all")

XML’de tiftik kontrolünü yapılandırma #

XML dosyalarınızın belirli bölümleri için Lint denetimini devre dışı bırakmak için ’tools:ignore’ özniteliğini kullanabilirsiniz.

Örneğin:

tools:ignore="NewApi,StringFormatInvalid"

XML öğesindeki tüm Lint sorunlarını denetlemeyi bastırmak için şunu kullanın:

tools:ignore="all"

“Kullanımdan kaldırılmış” hatası olmadan kaynakları içe aktarma #

Android API 23 veya daha üstünü kullanarak, çok sık böyle bir durum görülebilir:

Bu durum, Android API’nin kaynakların alınmasına ilişkin yapısal değişikliğinden kaynaklanmaktadır. Şimdi işlev:

public int getColor(@ColorRes int id, @Nullable Theme theme) throws NotFoundException

kullanılmalıdır. Ancak android.support.v4 kitaplığının başka bir çözümü var.

build.gradle dosyasına aşağıdaki bağımlılığı ekleyin:

com.android.support:support-v4:24.0.0

Ardından destek kitaplığındaki tüm yöntemler kullanılabilir:

ContextCompat.getColor(context, R.color.colorPrimaryDark);
ContextCompat.getDrawable(context, R.drawable.btn_check);
ContextCompat.getColorStateList(context, R.color.colorPrimary);
DrawableCompat.setTint(drawable);
ContextCompat.getColor(context,R.color.colorPrimaryDark));

Ayrıca destek kitaplığından daha fazla yöntem kullanılabilir:

ViewCompat.setElevation(textView, 1F);
ViewCompat.animate(textView);
TextViewCompat.setTextAppearance(textView, R.style.AppThemeTextStyle);
...

lint.xml dosyası nasıl yapılandırılır #

Lint kontrol tercihlerinizi lint.xml dosyasında belirtebilirsiniz. Bu dosyayı manuel olarak oluşturuyorsanız, Android projenizin kök dizinine yerleştirin. Android Studio’da Lint tercihlerini yapılandırıyorsanız, lint.xml dosyası sizin için otomatik olarak oluşturulur ve Android projenize eklenir.

Örnek:

<?xml version="1.0" encoding="UTF-8"?>
    <lint>
        <!-- list of issues to configure -->
</lint>

etiketinde önem derecesi özniteliği değerini ayarlayarak, bir sorun için Lint denetimini devre dışı bırakabilir veya bir sorunun önem düzeyini değiştirebilirsiniz.

Aşağıdaki örnek, bir “lint.xml” dosyasının içeriğini göstermektedir.

<?xml version="1.0" encoding="UTF-8"?>
<lint>
    <!-- Disable the given check in this project -->
    <issue id="IconMissingDensityFolder" severity="ignore" />

    <!-- Ignore the ObsoleteLayoutParam issue in the specified files -->
    <issue id="ObsoleteLayoutParam">
        <ignore path="res/layout/activation.xml" />
        <ignore path="res/layout-xlarge/activation.xml" />
    </issue>

    <!-- Ignore the UselessLeaf issue in the specified file -->
    <issue id="UselessLeaf">
        <ignore path="res/layout/main.xml" />
    </issue>

    <!-- Change the severity of hardcoded strings to "error" -->
    <issue id="HardcodedText" severity="error" />
</lint>

İşaret Bastırma Uyarıları #

Kodunuzda bazı uyarıları işaretlemek iyi bir uygulamadır. Örneğin, testiniz veya eski destek sürümünüz için bazı kullanımdan kaldırılmış yöntemlere ihtiyaç vardır. Ancak Lint kontrolü bu kodu uyarılarla işaretleyecektir. Bu sorunu önlemek için @SuppressWarnings ek açıklamasını kullanmanız gerekir.

Örneğin, kullanımdan kaldırılan yöntemlere uyarılara yoksaymayı ekleyin. Ayrıca ek açıklamalara uyarı açıklamasını da eklemeniz gerekir:

@SuppressWarnings("deprecated");
public void setAnotherColor (int newColor) {
    getApplicationContext().getResources().getColor(newColor)
}

Bu ek açıklamayı kullanarak Lint, Android ve diğerleri dahil tüm uyarıları yok sayabilirsiniz. Suppress Warnings kullanmak, kodun doğru anlaşılmasına yardımcı olur!
