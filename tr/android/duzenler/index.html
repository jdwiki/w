DüzenParamları #

Her bir ViewGroup (ör. LinearLayout, RelativeLayout, CoordinatorLayout vb.), alt özellikleri hakkında bilgi depolaması gerekir. . Çocuklarının ‘ViewGroup’ta nasıl düzenlendiği hakkında. Bu bilgi, ViewGroup.LayoutParams sarmalayıcı sınıfının nesnelerinde depolanır.

Belirli bir düzen türüne özgü parametreleri dahil etmek için, “ViewGroups”, “ViewGroup.LayoutParams” sınıfının alt sınıflarını kullanır.

Örneğin. için

  • ‘LinearLayout’, ‘LinearLayout.LayoutParams’
  • ‘RelativeLayout’ ‘RelativeLayout.LayoutParams’
  • ‘CoordinatorLayout’, ‘CoordinatorLayout.LayoutParams’ -…

Çoğu “ViewGroups”, çocukları için “marjlar” belirleme yeteneğini yeniden kullanır, bu nedenle doğrudan “ViewGroup.LayoutParams” alt sınıfını değil, bunun yerine [“ViewGroup.MarginLayoutParams”8 altsınıfını kullanırlar (ki bu, kendi alt sınıfıdır). ViewGroup.LayoutParams).

xml’de LayoutParams

“LayoutParams” nesneleri, şişirilmiş düzen “xml” dosyasına dayalı olarak oluşturulur.

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
              android:layout_width="match_parent"
              android:layout_height="match_parent"
              android:orientation="vertical">

    <TextView
        android:layout_width="wrap_content"
        android:layout_height="50dp"
        android:layout_gravity="right"
        android:gravity="bottom"
        android:text="Example text"
        android:textColor="@android:color/holo_green_dark"/>

    <ImageView
        android:layout_width="match_parent"
        android:layout_height="0dp"
        android:layout_weight="1"
        android:background="@android:color/holo_green_dark"
        android:scaleType="centerInside"
        android:src="@drawable/example"/>

</LinearLayout>

“layout_” ile başlayan tüm parametreler, kapsayan yerleşimin nasıl çalışması gerektiğini belirtir. Düzen şişirildiğinde, bu parametreler uygun bir ‘LayoutParams’ nesnesine sarılır, bu daha sonra ‘Düzen’ tarafından ‘ViewGroup’ içinde belirli bir ‘Görünüm’ü uygun şekilde konumlandırmak için kullanılacaktır. Bir “Görünüm"ün diğer nitelikleri doğrudan “Görünüm” ile ilgilidir ve “Görünüm"ün kendisi tarafından işlenir.

“Metin Görünümü” için:

  • “layout_width”, “layout_height” ve “layout_gravity” bir “LinearLayout.LayoutParams” nesnesinde depolanacak ve “LinearLayout” tarafından kullanılacak
  • “yerçekimi”, “metin” ve “textColor”, “TextView"ün kendisi tarafından kullanılacaktır

“ImageView” için:

  • “layout_width”, “layout_height” ve “layout_weight”, bir “LinearLayout.LayoutParams” nesnesinde depolanacak ve “LinearLayout” tarafından kullanılacaktır.
  • “background”, “scaleType” ve “src”, “ImageView” tarafından kullanılacaktır

LayoutParams nesnesi alınıyor

getLayoutParams, geçerli bir LayoutParams nesnesinin alınmasına izin veren bir Görünüm yöntemidir.

“LayoutParams” nesnesi doğrudan çevreleyen “ViewGroup” ile ilişkili olduğundan, bu yöntem yalnızca “View”, “ViewGroup"a eklendiğinde boş olmayan bir değer döndürür. Bu nesnenin her zaman mevcut olmayabileceğini aklınızdan çıkarmayın. Özellikle, onu ‘View’in yapıcısı içinde bulundurmaya güvenmemelisiniz.

public class ExampleView extends View {
    
    public ExampleView(Context context) {
        super(context);
        setupView(context);
    }

    public ExampleView(Context context, AttributeSet attrs) {
        super(context, attrs);
        setupView(context);
    }

    public ExampleView(Context context, AttributeSet attrs, int defStyle) {
        super(context, attrs, defStyle);
        setupView(context);
    }

    private void setupView(Context context) {
        if (getLayoutParams().height == 50){  // DO NOT DO THIS!
                                              // This might produce NullPointerException
            doSomething();
        }
    }
    
    //...
}

LayoutParams nesnesine sahip olmak istiyorsanız, bunun yerine onAttachedToWindow yöntemini kullanmalısınız.

public class ExampleView extends View {

    public ExampleView(Context context) {
        super(context);
    }

    public ExampleView(Context context, AttributeSet attrs) {
        super(context, attrs);
    }

    public ExampleView(Context context, AttributeSet attrs, int defStyle) {
        super(context, attrs, defStyle);
    }

    @Override
    protected void onAttachedToWindow() {
        super.onAttachedToWindow();
        if (getLayoutParams().height == 50) { // getLayoutParams() will NOT return null here
            doSomething();
        }
    }

    //...
}

LayoutParams nesnesini yayınlama

Belirli bir “ViewGroup"a özgü özellikleri kullanmanız gerekebilir (örneğin, bir “RelativeLayout"un kurallarını programlı olarak değiştirmek isteyebilirsiniz). Bu amaçla, ViewGroup.LayoutParams nesnesini nasıl düzgün şekilde yayınlayacağınızı bilmeniz gerekecektir.

Bu, aslında başka bir “ViewGroup” olan bir çocuk “View” için bir “LayoutParams” nesnesi alırken biraz kafa karıştırıcı olabilir.

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
              android:id="@+id/outer_layout"
              android:layout_width="match_parent"
              android:layout_height="match_parent"
              android:orientation="vertical">

    <FrameLayout
        android:id="@+id/inner_layout"
        android:layout_width="match_parent"
        android:layout_height="50dp"
        android:layout_gravity="right"/>

</LinearLayout>

ÖNEMLİ: ‘LayoutParams’ nesnesinin türü, doğrudan ENCLOSING ‘ViewGroup’ türüyle ilgilidir.

Yanlış döküm:

FrameLayout innerLayout = (FrameLayout)findViewById(R.id.inner_layout);
FrameLayout.LayoutParams par = (FrameLayout.LayoutParams) innerLayout.getLayoutParams();
                                          // INCORRECT! This will produce ClassCastException

Doğru döküm:

FrameLayout innerLayout = (FrameLayout)findViewById(R.id.inner_layout);
LinearLayout.LayoutParams par = (LinearLayout.LayoutParams) innerLayout.getLayoutParams();
                                         // CORRECT! the enclosing layout is a LinearLayout

Yüzde Düzenleri #

Yüzde Destek Kitaplığı, bir * genel boyutun yüzdesi*.

Bağımlılıklarınıza aşağıdakileri ekleyerek Yüzde Destek Kitaplığını kullanabilirsiniz.

compile 'com.android.support:percent:25.3.1'

Ekranı yatay olarak dolduran, ancak dikey olarak ekranın sadece yarısını dolduran bir görünüm görüntülemek istiyorsanız, aşağıdakileri yapardınız.

<android.support.percent.PercentFrameLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <FrameLayout
        app:layout_widthPercent="100%"
        app:layout_heightPercent="50%"
        android:background="@android:color/black" />

<android.support.percent.PercentFrameLayout>

Yüzdeleri aşağıdaki gibi kodlarla ayrı bir XML dosyasında da tanımlayabilirsiniz:

<fraction name="margin_start_percent">25%</fraction>

Ve mizanpajlarınızda bunlara @fraction/margin_start_percent ile bakın.

Ayrıca, “app:layout_aspectRatio” aracılığıyla özel bir en-boy oranı ayarlama olanağı da içerirler. Bu, yalnızca genişlik gibi yalnızca tek bir boyut ayarlamanıza olanak tanır ve yükseklik, tanımladığınız en boy oranına göre, ister 4:3, ister 16:9, hatta 1:1 kare olsun, otomatik olarak belirlenir. en boy oranı.

Örneğin:

 <ImageView
    app:layout_widthPercent="100%"
    app:layout_aspectRatio="178%"
    android:scaleType="centerCrop"
    android:src="@drawable/header_background"/>

Ağırlığı Görüntüle #

LinearLayout için en çok kullanılan özniteliklerden biri, alt görünümlerinin ağırlığı. Ağırlık, bir görünümün LinearLayout içindeki diğer görünümlere kıyasla ne kadar yer kaplayacağını tanımlar.

Ağırlık, bir bileşene diğerine kıyasla belirli ekran alanı vermek istediğinizde kullanılır.

Temel Özellikler:

  • weightSum, tüm alt görüntülemelerin ağırlıklarının genel toplamıdır. Eğer “weightSum"u belirtmezseniz, sistem tüm ağırlıkların toplamını kendi başına hesaplayacaktır.

  • layout_weight toplam ağırlık toplamındaki boşluk miktarını belirtir widget işgal edecek.

Kod:

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/activity_main"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="horizontal"
    android:weightSum="4">

    <EditText
        android:layout_weight="2"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        android:text="Type Your Text Here" />

    <Button
        android:layout_weight="1"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        android:text="Text1" />

    <Button
        android:layout_weight="1"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        android:text="Text1" />

</LinearLayout>

Çıktı:

Artık cihazın boyutu daha büyük olsa bile, EditText ekran alanının 2/4’ünü alacaktır. Bu nedenle, uygulamanızın görünümü tüm ekranlarda tutarlı görünür.

Not: Widget alanı yatay olarak bölündüğü için burada “layout_width” “0dp” olarak tutulur. Widget’lar dikey olarak hizalanacaksa, ’layout_height’, ‘0dp’ olarak ayarlanacaktır. Bu, kodun verimliliğini artırmak için yapılır, çünkü çalışma zamanında sistem, ağırlık tarafından yönetildiği için sırasıyla genişliği veya yüksekliği hesaplamaya çalışmaz. Bunun yerine “wrap_content"i kullandıysanız, sistem, başka bir hesaplama döngüsüne neden olan ağırlık özelliğini uygulamadan önce genişliği/yüksekliği hesaplamaya çalışır.

Göreli Düzen #

RelativeLayout, alt görünümleri göreli konumlarda görüntüleyen bir “ViewGroup"tur. Varsayılan olarak, tüm alt görünümler yerleşim düzeninin sol üst kısmında çizilir, bu nedenle RelativeLayout.LayoutParams öğesinde bulunan çeşitli düzen özelliklerini kullanarak her bir görünümün konumunu tanımlamanız gerekir. Her bir düzen özelliğinin değeri, üst RelativeLayout’a göre bir düzen konumunu etkinleştirmek için bir boole veya görünümün konumlandırılması gereken düzende başka bir görünüme başvuran bir kimliktir.

Örnek:

<?xml version="1.0" encoding="utf-8"?>
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <ImageView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:id="@+id/imageView"
        android:src="@mipmap/ic_launcher" />

    <EditText
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:id="@+id/editText"
        android:layout_toRightOf="@+id/imageView"
        android:layout_toEndOf="@+id/imageView"
        android:hint="@string/hint" />

</RelativeLayout>

Izgara Düzeni #

GridLayout, adından da anlaşılacağı gibi, Görünümleri bir ızgarada düzenlemek için kullanılan bir düzendir. Bir GridLayout kendisini sütunlara ve satırlara böler. Aşağıdaki örnekte görebileceğiniz gibi, sütunların ve/veya satırların miktarı ‘columnCount’ ve ‘rowCount’ özellikleriyle belirlenir. Bu yerleşime Görünümler eklemek, ilk görünümü birinci sütuna, ikinci görünümü ikinci sütuna ve üçüncü görünümü ikinci satırın ilk sütununa ekleyecektir.

<?xml version="1.0" encoding="utf-8"?>
<GridLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:paddingBottom="@dimen/activity_vertical_margin"
    android:paddingLeft="@dimen/activity_horizontal_margin"
    android:paddingRight="@dimen/activity_horizontal_margin"
    android:paddingTop="@dimen/activity_vertical_margin"
    android:columnCount="2"
    android:rowCount="2">

    <TextView
        android:layout_width="@dimen/fixed"
        android:layout_height="wrap_content"
        android:text="@string/first"
        android:background="@color/colorPrimary"
        android:layout_margin="@dimen/default_margin" />

    <TextView
        android:layout_width="@dimen/fixed"
        android:layout_height="wrap_content"
        android:text="@string/second"
        android:background="@color/colorPrimary"
        android:layout_margin="@dimen/default_margin" />

    <TextView
        android:layout_width="@dimen/fixed"
        android:layout_height="wrap_content"
        android:text="@string/third"
        android:background="@color/colorPrimary"
        android:layout_margin="@dimen/default_margin" />

</GridLayout>

Çerçeve Düzeni #

FrameLayout, tek bir öğenin görüntülenmesi için ekrandaki bir alanı engellemek üzere tasarlanmıştır. Bununla birlikte, bir FrameLayout’a birden çok çocuk ekleyebilir ve android:layout_gravity özniteliğini kullanarak her bir çocuğa yerçekimi atayarak FrameLayout içindeki konumlarını kontrol edebilirsiniz.

Genel olarak, “FrameLayout”, tek bir alt görünümü tutmak için kullanılır. Yaygın kullanım örnekleri, “Etkinlik"te “Parçaları” şişirmek, görünümleri çakıştırmak veya görünümlere ön plan uygulamak için yer tutucular oluşturmaktır.

Örnek:

<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
android:layout_width="match_parent"
android:layout_height="match_parent">

    <ImageView
        android:src="@drawable/nougat"
        android:scaleType="fitCenter"
        android:layout_height="match_parent"
        android:layout_width="match_parent"/>

    <TextView
        android:text="FrameLayout Example"
        android:textSize="30sp"
        android:textStyle="bold"
        android:layout_height="match_parent"
        android:layout_width="match_parent"
        android:gravity="center"/>

</FrameLayout>

Koordinatör Düzeni #

‘CoordinatorLayout’, ‘FrameLayout’a biraz benzer bir kapsayıcıdır, ancak ekstra yeteneklere sahip olduğundan, resmi belgelerde süper güçlü ‘FrameLayout’ olarak adlandırılır.

Bir CoordinatorLayout.Behavior doğrudan CoordinatorLayout alt öğesine ekleyerek, dokunma olaylarını, pencere eklerini, ölçümü, düzeni ve iç içe kaydırmayı durdurabileceksiniz.

Kullanmak için önce gradle dosyanızdaki destek kitaplığına bir bağımlılık eklemeniz gerekir:

compile 'com.android.support:design:25.3.1'

Kütüphanenin en son sürümünün numarası buradan bulunabilir

“CoordinatorLayout"un pratik kullanım örneklerinden biri, “FloatingActionButton” ile bir görünüm oluşturmaktır. Bu özel durumda, üzerinde bir “SwipeRefreshLayout” ve bir “FloatingActionButton” bulunan bir “RecyclerView” oluşturacağız. Bunu şu şekilde yapabilirsiniz:

<?xml version="1.0" encoding="utf-8"?>
<android.support.design.widget.CoordinatorLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:id="@+id/coord_layout"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="horizontal">

    <android.support.v4.widget.SwipeRefreshLayout
        android:id="@+id/swipe_refresh_layout"
        android:layout_width="match_parent"
        android:layout_height="match_parent">

        <android.support.v7.widget.RecyclerView
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:id="@+id/recycler_view"/>

    </android.support.v4.widget.SwipeRefreshLayout>

    <android.support.design.widget.FloatingActionButton
        android:id="@+id/fab"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_margin="16dp"
        android:clickable="true"
        android:color="@color/colorAccent"
        android:src="@mipmap/ic_add_white"
        android:layout_gravity="end|bottom"
        app:layout_anchorGravity="bottom|right|end"/>

</android.support.design.widget.CoordinatorLayout>

FloatingActionButton öğesinin, CoordinatorLayout’a app:layout_anchor="@id/coord_layout” ile nasıl bağlandığına dikkat edin.
