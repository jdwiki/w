“adb shell”, hedef cihazda veya öykünücüde bir Linux kabuğu açar. Bir Android cihazını “adb” aracılığıyla kontrol etmenin en güçlü ve çok yönlü yoludur.

Bu konu, çoğu ‘adb shell’ komutunu içeren örneklerin sınırına ulaşılması nedeniyle https://www.wikiod.com/tr/android/adb-android-hata-ayklama-koprusu ayrıldı.

Sözdizimi #

  • adb shell [-e kaçış] [-n] [-Tt] [-x] [komut]

Parametreler #

parametre Ayrıntılar
-e kaçış karakterini veya “hiçbirini” seçin; varsayılan ‘~’
-n stdin’den okuma
-T PTY tahsisini devre dışı bırak
-t zorla PTY tahsisi
-x uzaktan çıkış kodlarını ve stdout/stderr ayrımını devre dışı bırakın

API 23+ izinlerini verme ve iptal etme #

Güvenlik açığı bulunan izinlerin verilmesine veya iptal edilmesine yardımcı olan tek bir satır.

  • vermek

     adb shell pm grant <sample.package.id> android.permission.<PERMISSION_NAME>

  • iptal etme

     adb shell pm revoke <sample.package.id> android.permission.<PERMISSION_NAME>

  • Kurulumda tüm çalışma zamanı izinlerini aynı anda verme (-g)

     adb install -g /path/to/sample_package.apk

Metin, basılan tuş ve dokunma olaylarını ADB aracılığıyla Android Cihaza gönderin #

metni odaklı bir görünüme eklemek için aşağıdaki komutu yürütün (metin girişini destekliyorsa)

SDK 23+ ile metin gönder

adb shell "input keyboard text 'Paste text on Android Device'"

Cihazınıza ‘adb’ aracılığıyla zaten bağlıysa:

input text 'Paste text on Android Device'
son sürüm

SDK 23’ten önce metin gönderin

adb shell "input keyboard text 'Paste%stext%son%sAndroid%sDevice'"

Boşluklar girdi olarak kabul edilmez, bunları %s ile değiştirin.

son sürüm

Etkinlikleri gönder

Donanım güç tuşuna basmayı simüle etmek için

adb shell input keyevent 26

Veya alternatif olarak

adb shell input keyevent POWER

Bir donanım anahtarınız olmasa bile, eşdeğer eylemi gerçekleştirmek için yine de bir “keyevent” kullanabilirsiniz.

adb shell input keyevent CAMERA

Dokunma etkinliğini giriş olarak gönder

adb shell input tap Xpoint Ypoint

Swipe olayını girdi olarak gönder

adb shell input swipe Xpoint1 Ypoint1 Xpoint2 Ypoint2 [DURATION*]

*SÜRE isteğe bağlıdır, varsayılan=300ms. kaynak

Geliştirici seçeneğinde işaretçi konumunu etkinleştirerek X ve Y puanları alın.

ADB örnek kabuk betiği

Ubuntu’da bir komut dosyası çalıştırmak için, Create script.sh dosyaya sağ tıklayın ve okuma/yazma izni ekleyin ve dosyayı program olarak yürütmeye izin ver‘i işaretleyin.

Terminal öykünücüsünü açın ve ./script.sh komutunu çalıştırın

Script.sh

 for (( c=1; c<=5; c++ ))
    do  
       adb shell input tap X Y
       echo "Clicked $c times"
       sleep 5s
    done

Olay numaralarının kapsamlı bir listesi için

  • birkaç ilginç olayın kısa listesi http://stackoverflow.com/questions/7789826/adb-shell-input-events
  • referans belgeleri https://developer.android.com/reference/android/view/KeyEvent.html#KEYCODE_POWER.

Liste paketleri #

Tüm paketleri, isteğe bağlı olarak yalnızca paket adı <FILTER> içindeki metni içerenleri yazdırır.

adb shell pm list packages [options] <FILTER>

All <FILTER>

adb shell pm list packages

Öznitellikler:

İlişkili dosyalarını görmek için -f.

-i Paketler için yükleyiciye bakın.

-u kaldırılmış paketleri de dahil etmek için.

-u Kaldırılan paketleri de içerir.

Filtreleyen özellikler:

Devre dışı bırakılmış paketler için `-d’.

Etkin paketler için “-e”.

sistem paketleri için `-s’.

Üçüncü taraf paketleri için ‘-3’.

Sorgulanacak belirli bir kullanıcı alanı için --user <USER_ID>.

Ekranı kaydetme #

Android 4.4 (API seviye 19) ve sonraki sürümleri çalıştıran cihazların görüntüsünü kaydetme:

adb shell screenrecord [options] <filename>
adb shell screenrecord /sdcard/demo.mp4

(kaydı durdurmak için Ctrl-C’ye basın)

Dosyayı cihazdan indirin:

adb pull /sdcard/demo.mp4

Not: Ctrl-C tuşlarına basarak ekran kaydını durdurun, aksi takdirde kayıt otomatik olarak üç dakikada veya --zaman-limit tarafından belirlenen zaman sınırında durur.

adb shell screenrecord --size <WIDTHxHEIGHT>

Video boyutunu ayarlar: 1280x720. Varsayılan değer, aygıtın yerel ekran çözünürlüğüdür (destekleniyorsa), desteklenmiyorsa 1280x720. En iyi sonuçlar için cihazınızın Gelişmiş Video Kodlama (AVC) kodlayıcısı tarafından desteklenen bir boyut kullanın.

adb shell screenrecord --bit-rate <RATE>

Video için video bit hızını megabit/saniye olarak ayarlar. Varsayılan değer 4Mbps’dir. Video kalitesini iyileştirmek için bit hızını artırabilirsiniz, ancak bunu yapmak daha büyük film dosyalarıyla sonuçlanır. Aşağıdaki örnek, kayıt bit hızını 5Mbps’ye ayarlar:

adb shell screenrecord --bit-rate 5000000 /sdcard/demo.mp4

adb shell screenrecord --time-limit <TIME>

Maksimum kayıt süresini saniye cinsinden ayarlar. Varsayılan ve maksimum değer 180’dir (3 dakika).

adb shell screenrecord --rotate

Çıkışı 90 derece döndürür. Bu özellik deneyseldir.

adb shell screenrecord --verbose

Komut satırı ekranında günlük bilgilerini görüntüler. Bu seçeneği ayarlamazsanız, yardımcı program çalışırken herhangi bir bilgi görüntülemez.

Not: Bu, bazı cihazlarda çalışmayabilir.

son sürüm

Ekran kayıt komutu, 4.4 öncesi android sürümleriyle uyumlu değil

Ekran kaydı komutu, Android 4.4 (API seviyesi 19) ve üzerini çalıştıran cihazların görüntüsünü kaydetmek için bir kabuk yardımcı programıdır. Yardımcı program, ekran etkinliğini bir MPEG-4 dosyasına kaydeder.

son sürüm

Tarih/Saati adb ile ayarla #

Varsayılan SET formatı AADDhhmm[[CC]YY][.ss] şeklindedir, yani (her biri 2 basamak)

Örneğin, geçerli yılı değiştirmeden 17 Temmuz 10:10’u ayarlamak için şunu yazın:

adb shell 'date 07171010.00'

İpucu 1: tarih değişikliği hemen yansıtılmayacaktır ve ancak sistem saati bir sonraki dakikaya geçtikten sonra fark edilebilir bir değişiklik olacaktır.
Çağrınıza bir “TIME_SET” amaç yayını ekleyerek bir güncellemeyi zorlayabilirsiniz, örneğin:

adb shell 'date 07171010.00 ; am broadcast -a android.intent.action.TIME_SET'

İpucu 2: Android’in saatini yerel makinenizle senkronize etmek için:

Linux:

adb shell date `date +%m%d%H%M%G.%S`

Windows (PowerShell):

$currentDate = Get-Date -Format "MMddHHmmyyyy.ss" # Android's preferred format
adb shell "date $currentDate"

İki ipucu birlikte:

adb shell 'date `date +%m%d%H%M%G.%S` ; am broadcast -a android.intent.action.TIME_SET'
son sürüm

Varsayılan SET formatı ‘YYYYMMDD.HHmmss’

adb shell 'date -s 20160117.095930'

İpucu: Android’in saatini yerel (linux tabanlı) makinenizle senkronize etmek için:

adb shell date -s `date +%G%m%d.%H%M%S`
son sürüm

Geliştirici Seçeneklerini Aç #

adb shell am start -n com.android.settings/.DevelopmentSettings

Cihazınızı/emülatörünüzü ‘Geliştirici Seçenekleri’ bölümüne götürecektir.

Uygulama verilerini yazdır #

Bu komut, ilgili tüm uygulama verilerini yazdırır:

  • sürüm kodu

  • sürüm adı

  • verilen izinler (Android API 23+)

  • vb..

    adb shell dumpsys package <your.package.id>

chmod komutunu kullanarak dosya izinlerini değiştirme #

“Önyükleme Tamamlandı” yayını oluşturma #

Bu, bir “BootListener” uygulayan uygulamalar için geçerlidir. Uygulamanızı öldürerek uygulamanızı test edin ve ardından aşağıdakilerle test edin:

adb shell am broadcast -a android.intent.action.BOOT_COMPLETED -c android.intent.category.HOME -n your.app/your.app.BootListener

(“your.package/your.app.BootListener” ifadesini uygun değerlerle değiştirin).

Harici/ikincil depolama içeriğini görüntüleyin #

İçeriği görüntüle:

adb shell ls \$EXTERNAL_STORAGE
adb shell ls \$SECONDARY_STORAGE

Yolu görüntüle:

adb shell echo \$EXTERNAL_STORAGE
adb shell echo \$SECONDARY_STORAGE

bir Android cihazın içindeki bir işlemi sonlandır #

Bazen Android’in logcat’i, size ait olmayan bazı işlemlerden kaynaklanan, pili tüketen veya yalnızca kodunuzda hata ayıklamayı zorlaştıran hatalarla sonsuza kadar çalışır.

Aygıtı yeniden başlatmadan sorunu çözmenin uygun bir yolu, soruna neden olan işlemi bulmak ve sonlandırmaktır.

Logcat’ten

03-10 11:41:40.010 1550-1627/? E/BazıSüreç: ....

işlem numarasına dikkat edin: 1550

Şimdi bir kabuk açıp işlemi sonlandırabiliriz. ‘root’ sürecini öldüremeyeceğimizi unutmayın.

adb shell

kabuğun içinde kullanarak süreç hakkında daha fazla kontrol edebiliriz

ps -x | grep 1550

ve istersek onu öldürün:

kill -9 1550
