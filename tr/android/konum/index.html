Android Konum API’leri, kullanıcı konumunu bulma, bir kullanıcı genel bir alandan ayrıldığında bilgi verme (Geofencing) ve kullanıcı etkinliğini (yürüme, koşma, araba kullanma vb.) yorumlamaya yardımcı olmak gibi farklı amaçlar için çok çeşitli uygulamalarda kullanılır.

Ancak, Android Konum API’leri, kullanıcı konumu edinmenin tek yolu değildir. Aşağıda, Android’in “LocationManager” ve diğer yaygın konum kitaplıklarının nasıl kullanılacağına ilişkin örnekler verilecektir.

Android’de Konum farkında uygulamalar oluşturmak için iki yol vardır:

  • Android’in yerel açık kaynağı LocationManager
  • Google Play Hizmetlerinin bir parçası olan Google’ın FusedLocationProviderApi

KonumYöneticisi #

Profesyoneller

  • Daha ayrıntılı kontrol
  • Tüm cihazlarda kullanılabilir
  • Android çerçevesinin bir parçası

Eksileri

  • Düzgün yönetilmezse pilin bitmesi bir sorundur
  • Cihaz bir konum bulamıyorsa (ör. bir bina içinde zayıf GPS) konum sağlayıcıları değiştirmek için mantık gerektirir

Özellikler

  • NMEA Dinleyici
  • GPS Durum Dinleyici
  • Sağlayıcı durum değişikliklerini dinleyin (ör. GPS, kullanıcı tarafından kapatılır)
  • Konum kaynağı seçilecek sağlayıcıların listesi

Sağlayıcılar

KÜRESEL KONUMLAMA SİSTEMİ

  • Gerekli İzinler:
  • ACCESS_FINE_LOCATION
  • Doğruluk: 10m - 100m
  • Güç Gereksinimleri: YÜKSEK
  • Kullanılabilirlik: Dünya çapında (açık gökyüzü görünümüyle)
  • NOTLAR:
  • Konum güncellemeleri genellikle saniyede bir gelir, ancak GPS’in bir süredir kullanılmadığı ve A-GPS‘ün kullanılamadığı durumlarda, bir konumun alınması birkaç dakika sürebilir.
  • Gökyüzünün net görüntüsünün engellendiği durumlarda, GPS noktaları çok iyi kümelenmez (konum noktaları “zıplar”) ve “Urban Canyon” etkisi nedeniyle belirli alanlarda doğruluk yanıltıcı olabilir.

Ağ

  • Gerekli İzinler:
  • ACCESS_COARSE_LOCATION veya ACCESS_FINE_LOCATION
  • Doğruluk: 100m - 1000m+
  • Güç Gereksinimleri: DÜŞÜK - ORTA
  • Kullanılabilirlik: Baz istasyonu veya wifi sinyali aralığında
  • NOTLAR:
  • Konum güncellemeleri GPS’den daha az sıklıkta gerçekleşir
  • Konum güncellemeleri tipik olarak iyi kümelenmez (konum noktaları “zıplar”) ve doğruluk, farklı faktörlerin sayısına (wifi sinyallerinin sayısı, sinyal gücü, baz istasyonu tipi vb.)

Pasif

  • Gerekli İzinler:
  • ACCESS_FINE_LOCATION
  • Doğruluk: 10m - 1000m+
  • Güç Gereksinimleri: YOK
  • Kullanılabilirlik: Yalnızca başka bir uygulama GPS veya Ağdan bir konum aldığında
  • NOTLAR:
  • Size sürekli güncellemeler vermek için buna güvenmeyin. Bu, konum istekleri yapan diğer uygulamaları pasif olarak dinler ve bu konumları geri iletir.
  • FusedLocationProviderApi tarafından oluşturulan noktaları döndürmez, yalnızca bunları oluşturmak için kullanılan temel konum noktalarını döndürür.

FusedLocationProviderApi #

Profesyoneller

  • “Kutudan çıktığı gibi” daha az pil tüketimi sunar
  • Zayıf GPS’i iyi idare eder
  • Güncellemeleri daha düzenli olarak alır

Eksileri

  • GPS üzerinde daha az ayrıntılı kontrol
  • Tüm cihazlarda veya belirli ülkelerde mevcut olmayabilir
  • Üçüncü taraf kitaplık bağımlılığı gerektirir

Özellikler

  • Optimum pil tasarrufu için konum sağlayıcıların iyi yönetilen kullanımı
  • Genellikle Ağ Konum Sağlayıcıdan daha doğru noktalar üretir
  • Kütüphanenin daha sık güncellenmesi, daha fazla iyileştirmeye izin verilmesi
  • Ne tür bir sağlayıcı kullanacağınızı belirtmenize gerek yok

LocationRequest Öncelik Düzeyleri

PRIORITY_HIGH_ACCURACY

  • Gerekli İzinler:
  • Daha doğru konum için ACCESS_FINE_LOCATION veya daha az doğru konum için ACCESS_COARSE_LOCATION
  • Doğruluk: 10m - 100m
  • Güç Gereksinimleri: YÜKSEK
  • Kullanılabilirlik: Google Play Hizmetlerinin bulunduğu her yerde.
  • NOTLAR:
  • ACCESS_FINE_LOCATION kullanılmazsa, bu, konum güncellemeleri oluşturmak için GPS kullanmaz, ancak yine de doğru koşullarda oldukça doğru bir nokta bulur.
  • ACCESS_FINE_LOCATION kullanılırsa, çevresel koşullara bağlı olarak cihazı şu anda ne kadar doğru izleyebildiğine bağlı olarak konum noktaları oluşturmak için GPS kullanabilir veya kullanmayabilir.
  • Bu, diğer ayarlardan daha doğru konum güncellemeleri bildirebilse de, yine de “Kentsel Kanyon” etkisine açıktır.

PRIORITY_BALANCED_POWER_ACCURACY

  • Gerekli İzinler:
  • Daha doğru konum için ACCESS_FINE_LOCATION veya daha az doğru konum için ACCESS_COARSE_LOCATION
  • Doğruluk: 100m - 1000m+
  • Güç Gereksinimleri: ORTA
  • Kullanılabilirlik: Google Play Hizmetlerinin bulunduğu her yerde.
  • NOTLAR:
  • “PRIORITY_HIGH_ACCURACY” ile aynı notlar
  • Pek olası olmasa da, bu ayar konum oluşturmak için GPS’i kullanmaya devam edebilir.

PRIORITY_LOW_POWER

  • Gerekli İzinler:
  • ACCESS_FINE_LOCATION veya ACCESS_COARSE_LOCATION
  • Doğruluk: 100m - 1000m+
  • Güç Gereksinimleri: DÜŞÜK
  • Kullanılabilirlik: Google Play Hizmetlerinin bulunduğu her yerde.
  • NOTLAR:
  • Muhtemelen GPS kullanmıyor, ancak şu ana kadar denenmedi.
  • Güncellemeler genellikle çok doğru değildir
  • Genellikle konumdaki önemli değişiklikleri tespit etmek için kullanılır

PRIORITY_NO_POWER

  • Gerekli İzinler:
  • ACCESS_FINE_LOCATION veya ACCESS_COARSE_LOCATION
  • Doğruluk: 10m - 1000m+
  • Güç Gereksinimleri: YOK
  • Kullanılabilirlik: Google Play Hizmetlerinin bulunduğu her yerde.
  • NOTLAR:
  • ‘LocationManager’ ‘PASSIVE_PROVIDER’ ile neredeyse aynı işlevler
  • Google Play Hizmetleri güncellemelerini alındığında geri bildirir; burada “PASSIVE_PROVIDER”, kullanılan temel konum güncellemelerini geri bildirir

Sorun giderme #

OnLocationChanged() Hiç Aranmadı

Bu, Android Konumları almayla ilgili yaygın bir sorun gibi göründüğünden, genel düzeltmelerin hızlı bir listesini vereceğim:

  1. Bildiriminizi kontrol edin!

    One of the most common issues is that the right permissions were never given. If you are using GPS (with or without Network), use <uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>, else use <uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>. Google’s FusedLocationApi requires ACCESS_FINE_LOCATION.

  1. (Android 6+ için) Çalışma zamanı izinlerini kontrol edin!

    Check for and request permissions! If you are never given permissions, you’ll end up with crashes, or worse (if you are catching all exceptions), you’ll end up with no indication of anything! It doesn’t matter if the user grants you permission at the start of the app, always check to see if you have permissions for all calls. The user can easily go to their settings and revoke them.

  1. Kodunuzu iki kez kontrol edin!

    Are you sure you are passing in the right listener? Did you add that BroadcastReceiver or IntentService to your manifest? Are you using PendingIntent.getService() on a BroadcastReceiver class, or getBroadcast() on an IntentService class? Are you sure you are not unregistering your listener somewhere else in your code immediately after requesting?

  1. Cihaz ayarlarını kontrol edin!

    Obviously, make sure you have location services turned on.

    If you are using Network services, did you turn on “Scanning Always Available”? Do you have your location mode set to “Best” (“High Accuracy”) or “Battery Saving” (“Network Only”)?

    If you are using GPS, did you turn on “Best” (“High Accuracy”) or “Device only” in location mode?

  1. Kodunuzu iki kez kontrol edin!

    Yes, this is on here twice. Did you try using a LocationListener instead of a PendingIntent, or vice-versa, to ensure you actually implemented LocationManager properly? Are you sure that the location request isn’t being removed in some part of the Activity or Service lifecycle that you didn’t expect to happen?

  1. Çevrenizi kontrol edin!

    Are you testing GPS on the first floor of a building in the middle of San Francisco? Are you testing Network locations in the middle of nowhere? Do you work in a secret underground bunker void of all radio signals, wondering why your device isn’t getting location? Always double check your surroundings when trying to troubleshoot location problems!

Konumun çalışmamasının daha az belirgin başka nedenleri olabilir, ancak bu ezoterik düzeltmeleri aramadan önce bu hızlı kontrol listesini gözden geçirmeniz yeterli.

LocationRequest ile Faaliyet Kullanımına Örnek #

/*
 * This example is useful if you only want to receive updates in this 
 * activity only, and have no use for location anywhere else.
 */
public class LocationActivity extends AppCompatActivity implements
        GoogleApiClient.ConnectionCallbacks, GoogleApiClient.OnConnectionFailedListener, LocationListener {

    private GoogleApiClient mGoogleApiClient;
    private LocationRequest mLocationRequest;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        mGoogleApiClient = new GoogleApiClient.Builder(this)
                .addConnectionCallbacks(this)
                .addOnConnectionFailedListener(this)
                .addApi(LocationServices.API)
                .build();

        mLocationRequest = new LocationRequest()
                .setPriority(LocationRequest.PRIORITY_HIGH_ACCURACY) //GPS quality location points
                .setInterval(2000) //At least once every 2 seconds
                .setFastestInterval(1000); //At most once a second
    }

    @Override
    protected void onStart(){
        super.onStart();
        mGoogleApiClient.connect();
    }
    
    @Override
    protected void onResume(){
        super.onResume();
        //Permission check for Android 6.0+
        if(ActivityCompat.checkSelfPermission(this, Manifest.permission.ACCESS_FINE_LOCATION) == PackageManager.PERMISSION_GRANTED) {
            if(mGoogleApiClient.isConnected()) {
                LocationServices.FusedLocationApi.requestLocationUpdates(mGoogleApiClient, mLocationRequest, this);
            }
        }
    }
    
    @Override
    protected void onPause(){
        super.onPause();
        //Permission check for Android 6.0+
        if(ActivityCompat.checkSelfPermission(this, Manifest.permission.ACCESS_FINE_LOCATION) == PackageManager.PERMISSION_GRANTED) {
            if(mGoogleApiClient.isConnected()) {
                LocationServices.FusedLocationApi.removeLocationUpdates(mGoogleApiClient, this);
            }
        }
    }
    
    @Override
    protected void onStop(){
        super.onStop();
        mGoogleApiClient.disconnect();
    }

    @Override
    public void onConnected(@Nullable Bundle bundle) {
        if(ActivityCompat.checkSelfPermission(this, Manifest.permission.ACCESS_FINE_LOCATION) == PackageManager.PERMISSION_GRANTED) {
            LocationServices.FusedLocationApi.requestLocationUpdates(mGoogleApiClient, mLocationRequest, this);
        }
    }

    @Override
    public void onConnectionSuspended(int i) {
        mGoogleApiClient.connect();
    }

    @Override
    public void onConnectionFailed(@NonNull ConnectionResult connectionResult) {
    }

    @Override
    public void onLocationChanged(Location location) {
        //Handle your location update code here
    }
}

PendingIntent ve BroadcastReceiver ile Hizmet Kullanımına Örnek #

ÖrnekEtkinlik

Önerilen okuma: LocalBroadcastManager

/*
 * This example is useful if you have many different classes that should be 
 * receiving location updates, but want more granular control over which ones
 * listen to the updates.
 * 
 * For example, this activity will stop getting updates when it is not visible, but a database
 * class with a registered local receiver will continue to receive updates, until "stopUpdates()" is called here.
 *
 */
public class ExampleActivity extends AppCompatActivity {

    private InternalLocationReceiver mInternalLocationReceiver;

    @Override
    protected void onCreate(Bundle savedInstanceState){
        super.onCreate(savedInstanceState);
        
        //Create internal receiver object in this method only.
        mInternalLocationReceiver = new InternalLocationReceiver(this);
    }

    @Override
    protected void onResume(){
        super.onResume();
        
        //Register to receive updates in activity only when activity is visible
        LocalBroadcastManager.getInstance(this).registerReceiver(mInternalLocationReceiver, new IntentFilter("googleLocation"));
    }

    @Override
    protected void onPause(){
        super.onPause();
        
        //Unregister to stop receiving updates in activity when it is not visible.
        //NOTE: You will still receive updates even if this activity is killed.
        LocalBroadcastManager.getInstance(this).unregisterReceiver(mInternalLocationReceiver);
    }

    //Helper method to get updates
    private void requestUpdates(){
        startService(new Intent(this, LocationService.class).putExtra("request", true));
    }

    //Helper method to stop updates
    private void stopUpdates(){

public class LocationService extends Service implements
        GoogleApiClient.ConnectionCallbacks, GoogleApiClient.OnConnectionFailedListener {

    private GoogleApiClient mGoogleApiClient;
    private LocationRequest mLocationRequest;

    @Override
    public void onCreate(){
        super.onCreate();
        mGoogleApiClient = new GoogleApiClient.Builder(this)
                .addConnectionCallbacks(this)
                .addOnConnectionFailedListener(this)
                .addApi(LocationServices.API)
                .build();

        mLocationRequest = new LocationRequest()
                .setPriority(LocationRequest.PRIORITY_HIGH_ACCURACY) //GPS quality location points
                .setInterval(2000) //At least once every 2 seconds
                .setFastestInterval(1000); //At most once a second
    }

    @Override
    public int onStartCommand(Intent intent, int flags, int startId){
        super.onStartCommand(intent, flags, startId);
        //Permission check for Android 6.0+
        if (ContextCompat.checkSelfPermission(this, Manifest.permission.ACCESS_FINE_LOCATION) == PackageManager.PERMISSION_GRANTED) {
            if (intent.getBooleanExtra("request", false)) {
                if (mGoogleApiClient.isConnected()) {
                    LocationServices.FusedLocationApi.requestLocationUpdates(mGoogleApiClient, mLocationRequest, getPendingIntent());
                } else {
                    mGoogleApiClient.connect();
                }
            }
            else if(intent.getBooleanExtra("remove", false)){
                stopSelf();
            }
        }

        return START_STICKY;
    }

    @Override
    public void onDestroy(){
        super.onDestroy();
        if(mGoogleApiClient.isConnected()){
            LocationServices.FusedLocationApi.removeLocationUpdates(mGoogleApiClient, getPendingIntent());
            mGoogleApiClient.disconnect();
        }
    }
    
    private PendingIntent getPendingIntent(){
        
        //Example for IntentService
        //return PendingIntent.getService(this, 0, new Intent(this, **YOUR_INTENT_SERVICE_CLASS_HERE**), PendingIntent.FLAG_UPDATE_CURRENT);
        
        //Example for BroadcastReceiver
        return PendingIntent.getBroadcast(this, 0, new Intent(this, LocationReceiver.class), PendingIntent.FLAG_UPDATE_CURRENT);
    }

    @Override
    public void onConnected(@Nullable Bundle bundle) {
        //Permission check for Android 6.0+
        if(ContextCompat.checkSelfPermission(this, Manifest.permission.ACCESS_FINE_LOCATION) == PackageManager.PERMISSION_GRANTED) {
            LocationServices.FusedLocationApi.requestLocationUpdates(mGoogleApiClient, mLocationRequest, getPendingIntent());
        }
    }

    @Override
    public void onConnectionSuspended(int i) {
        mGoogleApiClient.connect();
    }

    @Override
    public void onConnectionFailed(@NonNull ConnectionResult connectionResult) {

    }

    @Nullable
    @Override
    public IBinder onBind(Intent intent) {
        return null;
    }
}
