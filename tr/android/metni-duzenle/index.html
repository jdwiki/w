EditText’lerle Çalışmak #

EditText, Android uygulamalarındaki standart metin girişi widget’ıdır. Kullanıcının bir uygulamaya metin girmesi gerekiyorsa, bunu yapmanın birincil yolu budur.

Metni düzenle

Bir EditText davranışını özelleştirmek için ayarlanabilecek birçok önemli özellik vardır. Bunlardan birkaçı aşağıda listelenmiştir. Daha fazla giriş alanı ayrıntısı için resmi metin alanları kılavuzuna bakın.

Kullanım

Aşağıdaki XML ile tüm varsayılan davranışlara sahip bir düzene bir EditText eklenir:

<EditText
    android:id="@+id/et_simple"
    android:layout_height="wrap_content"
    android:layout_width="match_parent">
</EditText>

EditText’in TextView’ın basit bir uzantısı olduğunu ve aynı özelliklerin tümünü devraldığını unutmayın.

Değeri Alma

EditText’e girilen metnin değerini almak aşağıdaki gibidir:

EditText simpleEditText = (EditText) findViewById(R.id.et_simple);
String strValue = simpleEditText.getText().toString();

Daha Fazla Giriş Özelleştirmesi

Girişi tek satırlık bir metinle sınırlamak isteyebiliriz (yeni satırlardan kaçının):

<EditText
  android:singleLine="true"
  android:lines="1"
/>

Rakamlar özelliğini kullanarak bir alana girilebilecek karakterleri sınırlayabilirsiniz:

<EditText
  android:inputType="number"
  android:digits="01"
/>

Bu, girilen rakamları yalnızca “0” ve “1” ile sınırlayacaktır. Toplam karakter sayısını şu şekilde sınırlamak isteyebiliriz:

<EditText
  android:maxLength="5"
/>

Bu özellikleri kullanarak metin alanları için beklenen giriş davranışını tanımlayabiliriz.

Renklerin Ayarlanması

Bir EditText içindeki seçili metnin vurgulu arka plan rengini “android:textColorHighlight” özelliğiyle ayarlayabilirsiniz:

<EditText
    android:textColorHighlight="#7cff88"
/>

Yer Tutucu İpuçlarını Görüntüleme

EditText denetiminin kullanıcıdan belirli bir giriş yapmasını istemesi için ipucunu şu şekilde ayarlamak isteyebilirsiniz:

<EditText
    ...
    android:hint="@string/my_hint">
</EditText>

İpuçları

Son satır rengini değiştirme

AppCompat kitaplığını kullandığınızı varsayarsak, colorControlNormal, colorControlActivated ve colorControlHighlight stillerini geçersiz kılabilirsiniz:

<style name="Theme.App.Base" parent="Theme.AppCompat.Light.DarkActionBar">
    <item name="colorControlNormal">#d32f2f</item>
    <item name="colorControlActivated">#ff5722</item>
    <item name="colorControlHighlight">#f44336</item>
</style>

Bir DialogFragment içinde uygulanmış bu stilleri görmüyorsanız, onCreateView() yöntemine geçirilen LayoutInflater kullanılırken bilinen bir hata vardır.

Sorun, AppCompat v23 kitaplığında zaten düzeltildi. Nasıl yükseltileceğiyle ilgili bu kılavuza bakın. Başka bir geçici çözüm, onCreateView() yöntemine iletilen yerine Activity’nin düzen şişiricisini kullanmaktır:

public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
        View view = getActivity().getLayoutInflater().inflate(R.layout.dialog_fragment, container);
  }

EditText Girişi Dinleniyor

Bir EditText’teki değişiklikleri nasıl dinleyeceğinize ve bu değişiklikler meydana geldiğinde bir eylem gerçekleştireceğinize bir göz atmak için temel olay dinleyicileri uçurum notlarına göz atın.

Kayan Etiket Geri Bildirimini Görüntüleme

Geleneksel olarak EditText, kullanıcı yazmaya başladıktan sonra ipucu mesajını (yukarıda açıklanmıştır) gizler. Ayrıca, herhangi bir doğrulama hata mesajının geliştirici tarafından manuel olarak yönetilmesi gerekiyordu.

TextInputLayout ile ipuçlarını ve hata mesajlarını görüntülemek için kayan bir etiket ayarlayabilirsiniz. Daha fazla ayrıntıyı burada bulabilirsiniz.

InputType’ı Özelleştirme #

Metin alanlarının numara, tarih, parola veya e-posta adresi gibi farklı giriş türleri olabilir. Tür, alan içinde ne tür karakterlere izin verileceğini belirler ve sanal klavyeden sık kullanılan karakterler için düzenini optimize etmesini isteyebilir.

Varsayılan olarak, bir ‘EditText’ kontrolü içindeki herhangi bir metin içeriği düz metin olarak görüntülenir. inputType niteliğini ayarlayarak, telefon numaraları ve şifreler gibi farklı bilgi türlerinin girişini kolaylaştırabiliriz:

<EditText
    ...
    android:inputType="phone">
</EditText>

En yaygın giriş türleri şunları içerir:

|Tür | Açıklama | |——————————–|—————————— ——————-| |metinUri | URI olarak kullanılacak metin | |metinE-postaAdresi | E-posta adresi olarak kullanılacak metin | |metinKişiAdı | Bir kişinin adı olan metin | |metinParolası | Gizlenmesi gereken bir parola olan metin | |sayı | Yalnızca sayısal bir alan | |telefon | Telefon numarası girmek için | |tarih | Tarih girmek için | |zaman | Bir zaman girmek için | |metinÇok Satır | Alanda birden çok metin satırına izin ver |

“android:inputType” ayrıca, tüm yeni sözcüklerin büyük harfe dönüştürülmesi veya otomatik tamamlama ve yazım önerileri gibi özelliklerin kullanılması gibi belirli klavye davranışlarını belirlemenize olanak tanır. Klavye davranışlarını tanımlayan yaygın giriş türü değerlerinden bazıları şunlardır:

|Tür | Açıklama | |——————————–|—————————— ——————-| |textCapCapCümleler | Her yeni cümle için ilk harfi büyük yapan normal metin klavyesi| |metinCapWords | Her kelimeyi büyük harf yapan normal metin klavyesi. Unvanlar veya kişi adları için iyi| |metinOtomatik Düzelt | Sıklıkla yanlış yazılan sözcükleri düzelten normal metin klavyesi|

Gerekirse birden çok “inputType” özniteliği ayarlayabilirsiniz (’|’ ile ayırarak). Örnek:

<EditText
    android:id="@+id/postal_address"
    android:layout_width="fill_parent"
    android:layout_height="wrap_content"
    android:hint="@string/postal_address_hint"
    android:inputType="textPostalAddress|
                       textCapWords|
                       textNoSuggestions" />

Mevcut tüm giriş türlerinin listesini burada görebilirsiniz.

SoftKeyboard’u Gizleme #

Softkeyboard’u gizlemek, genellikle EditText ile çalışırken temel bir gereksinimdir. varsayılan olan softkeyboard yalnızca geri düğmesine basılarak kapatılabilir ve bu nedenle çoğu geliştirici, Android’i hideSoftInputFromWindow çağıran sanal klavyeyi gizlemeye zorlamak için InputMethodManager kullanır ve odaklanmış görünüm. Aşağıdakileri yapmak için kod:

public void hideSoftKeyboard()    
{
        InputMethodManager inputMethodManager = (InputMethodManager) getSystemService(Activity.INPUT_METHOD_SERVICE);
        inputMethodManager.hideSoftInputFromWindow(getCurrentFocus().getWindowToken(), 0);    
}

Kod doğrudandır, ancak ortaya çıkan bir diğer önemli sorun, bir olay meydana geldiğinde hide işlevinin çağrılması gerektiğidir. EditText’iniz dışında herhangi bir yere bastığınızda gizlenmiş Softkeyboard’a ihtiyacınız olduğunda ne yapmalısınız? Aşağıdaki kod, onCreate() yönteminizde yalnızca bir kez çağrılması gereken düzgün bir işlev verir.

public void setupUI(View view) 
{
        String s = "inside";
        //Set up touch listener for non-text box views to hide keyboard.
        if (!(view instanceof EditText)) {

            view.setOnTouchListener(new View.OnTouchListener() {

                public boolean onTouch(View v, MotionEvent event) {
                    hideSoftKeyboard();
                    return false;
                }

            });
        }

        //If a layout container, iterate over children and seed recursion.
        if (view instanceof ViewGroup) {

            for (int i = 0; i < ((ViewGroup) view).getChildCount(); i++) {

                View innerView = ((ViewGroup) view).getChildAt(i);

                setupUI(innerView);
            }
        }    
}

Özel Metin Düzenleme içindeki simge veya düğme ve eylemi ve dinleyicileri tıklayın. #

Bu örnek, sağ tarafta simgesi olan Düzenle metninin olmasına yardımcı olacaktır.

Not: Bunda sadece setCompoundDrawablesWithIntrinsicBounds kullanıyorum, Simge konumunu değiştirmek istiyorsanız, bunu kullanarak başarabilirsiniz. setIcon’da setCompoundDrawablesWithIntrinsicBounds.

public class MKEditText extends AppCompatEditText {

    public interface IconClickListener {
        public void onClick();
    }

    private IconClickListener mIconClickListener;

    private static final String TAG = MKEditText.class.getSimpleName();

    private final int EXTRA_TOUCH_AREA = 50;
    private Drawable mDrawable;
    private boolean touchDown;

    public MKEditText(Context context, AttributeSet attrs, int defStyle) {
        super(context, attrs, defStyle);
    }

    public MKEditText(Context context) {
        super(context);
    }

    public MKEditText(Context context, AttributeSet attrs) {
        super(context, attrs);
    }

    public void showRightIcon() {
        mDrawable = ContextCompat.getDrawable(getContext(), R.drawable.ic_android_black_24dp);

        setIcon();
    }

    public void setIconClickListener(IconClickListener iconClickListener) {
        mIconClickListener = iconClickListener;
    }

    private void setIcon() {
        Drawable[] drawables = getCompoundDrawables();

        setCompoundDrawablesWithIntrinsicBounds(drawables[0], drawables[1], mDrawable, drawables[3]);

        setInputType(InputType.TYPE_CLASS_TEXT | InputType.TYPE_TEXT_VARIATION_PASSWORD);
        setSelection(getText().length());
    }

    @Override
    public boolean onTouchEvent(MotionEvent event) {
        final int right = getRight();
        final int drawableSize = getCompoundPaddingRight();
        final int x = (int) event.getX();
        switch (event.getAction()) {
            case MotionEvent.ACTION_DOWN:
                if (x + EXTRA_TOUCH_AREA >= right - drawableSize && x <= right + EXTRA_TOUCH_AREA) {
                    touchDown = true;
                    return true;
                }
                break;
            case MotionEvent.ACTION_UP:
                if (x + EXTRA_TOUCH_AREA >= right - drawableSize && x <= right + EXTRA_TOUCH_AREA && touchDown) {
                    touchDown = false;
                    if (mIconClickListener != null) {
                        mIconClickListener.onClick();
                    }
                    return true;
                }
                touchDown = false;
                break;

        }
        return super.onTouchEvent(event);
    }
}

Dokunma alanını değiştirmek isterseniz, EXTRA_TOUCH_AREA değerleri varsayılan olarak 50 olarak verdim.

Ve Düğmeyi Etkinleştir ve dinleyiciyi tıklatmak için Aktivitenizden veya Fragmentinizden bu şekilde arayabilirsiniz,

MKEditText mkEditText = (MKEditText) findViewById(R.id.password);
mkEditText.showRightIcon();
mkEditText.setIconClickListener(new MKEditText.IconClickListener() {
            @Override
            public void onClick() {
              // You can do action here for the icon.
               
            }
        });
