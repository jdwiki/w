JSON’u Gson ile Ayrıştırma #

Örnek, Google’dan Gson kitaplığı kullanılarak bir JSON nesnesinin ayrıştırılmasını gösterir.

Nesneleri ayrıştırma:

class Robot {
    //OPTIONAL - this annotation allows for the key to be different from the field name, and can be omitted if key and field name are same . Also this is good coding practice as it decouple your variable names with server keys name 
    @SerializedName("version") 
    private String version;

    @SerializedName("age")
    private int age;
    
    @SerializedName("robotName")
    private String name;
    
    // optional : Benefit it allows to set default values and retain them, even if key is missing from Json response. Not required for primitive data types. 

    public Robot{
       version = "";
       name = "";
    }


}

Ardından, ayrıştırmanın gerçekleşmesi gereken yerde aşağıdakileri kullanın:

String robotJson = "{
                        \"version\": \"JellyBean\",
                        \"age\": 3,
                        \"robotName\": \"Droid\"
                   }";

Gson gson = new Gson();
Robot robot = gson.fromJson(robotJson, Robot.class);

Listeyi ayrıştırma:

JSON nesnelerinin bir listesini alırken, genellikle onları ayrıştırmak ve Java nesnelerine dönüştürmek isteyeceksiniz.

Dönüştürmeye çalışacağımız JSON dizesi şudur:

{
  "owned_dogs": [
    {
      "name": "Ron",
      "age": 12,
      "breed": "terrier"
    },
    {
      "name": "Bob",
      "age": 4,
      "breed": "bulldog"
    },
    {
      "name": "Johny",
      "age": 3,
      "breed": "golden retriever"
    }
  ]
}

Bu belirli JSON dizisi üç nesne içerir. Java kodumuzda bu nesneleri ‘Köpek’ nesnelerine eşlemek isteyeceğiz. Bir Dog nesnesi şöyle görünür:

private class Dog {
    public String name;
    public int age;

    @SerializedName("breed")
    public String breedName;
}

JSON dizisini bir “Köpek[]“e dönüştürmek için:

Dog[] arrayOfDogs = gson.fromJson(jsonArrayString, Dog[].class);

Bir “Köpek[]” ​​öğesini bir JSON dizesine dönüştürme:

String jsonArray = gson.toJson(arrayOfDogs, Dog[].class);

JSON dizisini bir ArrayList<Köpek> biçimine dönüştürmek için aşağıdakileri yapabiliriz:

Type typeListOfDogs = new TypeToken<List<Dog>>(){}.getType();
List<Dog> listOfDogs = gson.fromJson(jsonArrayString, typeListOfDogs);

TypeListOfDogs Type nesnesi, Dog nesnelerinin bir listesinin nasıl görüneceğini tanımlar. GSON, JSON dizisini doğru değerlerle eşlemek için bu tür nesneyi kullanabilir.

Alternatif olarak, bir ‘List<Köpek>’ öğesini bir JSON dizisine dönüştürmek de benzer şekilde yapılabilir.

String jsonArray = gson.toJson(listOfDogs, typeListOfDogs);

Gson’a özel bir Dönüştürücü ekleme #

Bazen bazı alanları istediğiniz biçimde serileştirmeniz veya seri durumdan çıkarmanız gerekir, örneğin arka ucunuz tarihler için “YYYY-AA-gg SS:dd” biçimini kullanabilir ve POJOS’unuzun Joda Time’da DateTime sınıfını kullanmasını istersiniz.

Bu dizeleri otomatik olarak DateTimes nesnesine dönüştürmek için özel bir dönüştürücü kullanabilirsiniz.

/**
 * Gson serialiser/deserialiser for converting Joda {@link DateTime} objects.
 */
public class DateTimeConverter implements JsonSerializer<DateTime>, JsonDeserializer<DateTime> {

    private final DateTimeFormatter dateTimeFormatter;

    @Inject
    public DateTimeConverter() {
        this.dateTimeFormatter = DateTimeFormat.forPattern("YYYY-MM-dd HH:mm");
    }

    @Override
    public JsonElement serialize(DateTime src, Type typeOfSrc, JsonSerializationContext context) {
        return new JsonPrimitive(dateTimeFormatter.print(src));
    }

    @Override
    public DateTime deserialize(JsonElement json, Type typeOfT, JsonDeserializationContext context)
            throws JsonParseException {
        
        if (json.getAsString() == null || json.getAsString().isEmpty()) {
            return null;
        }

        return dateTimeFormatter.parseDateTime(json.getAsString());
    }
}

Gson’un yeni oluşturulan dönüştürücüyü kullanmasını sağlamak için, Gson nesnesini oluştururken onu atamanız gerekir:

    DateTimeConverter dateTimeConverter = new DateTimeConverter();
    Gson gson = new GsonBuilder().registerTypeAdapter(DateTime.class, dateTimeConverter)
            .create();

    String s = gson.toJson(DateTime.now());
    // this will show the date in the desired format

Tarihi bu formatta seri durumdan çıkarmak için sadece DateTime formatında bir alan tanımlamanız gerekir:

public class SomePojo {
    private DateTime someDate;    
}

Gson, DateTime türünde bir alanla karşılaştığında, alanı seri durumdan çıkarmak için dönüştürücünüzü arayacaktır.

public class StringList extends ArrayList<String> { }

...

List<String> members = gson.fromJson(json, StringList.class);

Alternatif olarak, istediğiniz türü her zaman alt sınıflara ayırabilir ve ardından o sınıfa geçebilirsiniz. Ancak bu her zaman en iyi uygulama değildir, çünkü size StringList tipinde bir nesne döndürecektir;

JSON’u Gson ile Genel Sınıf Nesnesine Ayrıştırma #

Bir JSON dizimiz olduğunu varsayalım:

["first","second","third"]

Bu JSON dizesini bir String dizisine ayrıştırabiliriz:

Gson gson = new Gson();
String jsonArray = "[\"first\",\"second\",\"third\"]";
String[] strings = gson.fromJson(jsonArray, String[].class);

Ama onu bir ‘List’ nesnesine ayrıştırmak istiyorsak, ‘TypeToken’ kullanmalıyız.

İşte örnek:

Gson gson = new Gson();
String jsonArray = "[\"first\",\"second\",\"third\"]";
List<String> stringList = gson.fromJson(jsonArray, new TypeToken<List<String>>() {}.getType());

Aşağıdaki iki sınıfımız olduğunu varsayalım:

public class Outer<T> {
    public int index;
    public T data;
}

public class Person {
    public String firstName;
    public String lastName;
}

ve bir Outer<Person> nesnesine ayrıştırılması gereken bir JSON dizgemiz var.

Bu örnek, bu JSON dizesinin ilgili genel sınıf nesnesine nasıl ayrıştırılacağını gösterir:

String json = "......";
Type userType = new TypeToken<Outer<Person>>(){}.getType();
Result<User> userResult = gson.fromJson(json,userType);

JSON dizesinin bir “Outer<List>” nesnesine ayrıştırılması gerekiyorsa:

Type userListType = new TypeToken<Outer<List<Person>>>(){}.getType();
Result<List<User>> userListResult = gson.fromJson(json,userListType);

AutoValue ve Gson ile JSON Serileştirme/Seriyi Kaldırma #

Gradle kök dosyanıza içe aktarın

classpath 'com.neenbedankt.gradle.plugins:android-apt:1.8'

Gradle uygulama dosyanıza içe aktarın

apt 'com.google.auto.value:auto-value:1.2'  
apt 'com.ryanharter.auto.value:auto-value-gson:0.3.1'  
provided 'com.jakewharton.auto.value:auto-value-annotations:1.2-update1'  
provided 'org.glassfish:javax.annotation:10.0-b28'

Otomatik değerle nesne oluşturun:

@AutoValue public abstract class SignIn {    
    @SerializedName("signin_token") public abstract String signinToken();
    public abstract String username();

    public static TypeAdapter<SignIn> typeAdapter(Gson gson) {
        return new AutoValue_SignIn.GsonTypeAdapter(gson);
    }

    public static SignIn create(String signin, String username) {
        return new AutoValue_SignIn(signin, username);
    }
}

GsonBuilder ile Gson dönüştürücünüzü oluşturun

Gson gson = new GsonBuilder()
                .registerTypeAdapterFactory(
                    new AutoValueGsonTypeAdapterFactory())
                .create());

Seri durumdan çıkar

String myJsonData = "{
    \"signin_token\": \"mySigninToken\",
    \"username\": \"myUsername\" }";
SignIn signInData = gson.fromJson(myJsonData, Signin.class);

Seri hale getir

Signin myData = SignIn.create("myTokenData", "myUsername");
String myJsonData = gson.toJson(myData);

Gson’u kullanmak, POJO nesnelerini kullanarak Serileştirme ve Serileştirme kodunu basitleştirmenin harika bir yoludur. Yan etki, yansımanın performans açısından maliyetli olmasıdır. Bu nedenle, CustomTypeAdapter oluşturmak için AutoValue-Gson kullanmak, bir api değişikliği meydana geldiğinde güncellemeyi çok basit tutarken bu yansıtma maliyetini önleyecektir.

Diskten bir JSON dosyası yüklemek için Gson’u kullanma. #

Gson’u Retrofit ile serileştirici olarak kullanma #

Öncelikle build.gradle dosyanıza GsonConverterFactoryyi eklemeniz gerekiyor.

compile 'com.squareup.retrofit2:converter-gson:2.1.0'

Ardından, Güçlendirme Hizmetini oluştururken dönüştürücü fabrikasını eklemeniz gerekir:

Gson gson = new GsonBuilder().create();
new Retrofit.Builder()
        .baseUrl(someUrl)
        .addConverterFactory(GsonConverterFactory.create(gson))
        .build()
        .create(RetrofitService.class);

Fabrikaya aktardığınız Gson nesnesini oluştururken özel dönüştürücüler ekleyebilirsiniz. Özel türde dönüşümler oluşturmanıza olanak tanır.

Gson kullanan Özel JSON Seri Ayırıcı #

Tüm yanıtlarda tüm tarihlerin, örneğin /Date(1465935152)/ gibi bazı özel biçimlerde olduğunu ve Java Date örneklerine tüm Json tarihlerini seri durumdan çıkarmak için genel kural uygulamak istediğinizi hayal edin. Bu durumda özel Json Deserializer uygulamanız gerekir.

json’a örnek:

{
  "id": 1,
  "created_on": "Date(1465935152)",
  "updated_on": "Date(1465968945)",
  "name": "Oleksandr"
}

Diyelim ki aşağıdaki sınıfa sahibiz:

class User {
    @SerializedName("id")
    long id;
    @SerializedName("created_on")
    Date createdOn;
    @SerializedName("updated_on")
    Date updatedOn;
    @SerializedName("name")
    String name;
}

Özel seri kaldırıcı:

class DateDeSerializer implements JsonDeserializer<Date> {
    private static final String DATE_PREFIX = "/Date(";
    private static final String DATE_SUFFIX = ")/";

    @Override
    public Date deserialize(JsonElement json, Type typeOfT, JsonDeserializationContext context) throws JsonParseException {
        String dateString = json.getAsString();
        if (dateString.startsWith(DATE_PREFIX) && dateString.endsWith(DATE_SUFFIX)) {
            dateString = dateString.substring(DATE_PREFIX.length(), dateString.length() - DATE_SUFFIX.length());
        } else {
            throw new JsonParseException("Wrong date format: " + dateString);
        }
        return new Date(Long.parseLong(dateString) - TimeZone.getDefault().getRawOffset());
    }
}

Ve kullanım:

Gson gson = new GsonBuilder()
                .registerTypeAdapter(Date.class, new DateDeSerializer())
                .create();
String json = "....";
User user = gson.fromJson(json, User.class);

Jackson JSON dizelerini Date türleriyle seri hale getirin ve seri durumdan çıkarın

Bu, örneğin, Gson Date dönüşümünü Jackson ile uyumlu hale getirmek istediğiniz durum için de geçerlidir.

Jackson genellikle Date’i “epoch’tan beri milisaniye” olarak serileştirirken Gson, Date’i temsil etmek için “31 Ağustos 2016 10:26:17” gibi okunabilir bir biçim kullanır. Bu, bir Jackson biçimi Date serisini kaldırmaya çalıştığınızda Gson’da JsonSyntaxExceptions’a yol açar.

Bunu atlatmak için özel bir seri hale getirici ve özel bir seri hale getirici ekleyebilirsiniz:

JsonSerializer<Date> ser = new JsonSerializer<Date>() {
    @Override
    public JsonElement serialize(Date src, Type typeOfSrc, JsonSerializationContext
                context) {
        return src == null ? null : new JsonPrimitive(src.getTime());
    }
};

JsonDeserializer<Date> deser = new JsonDeserializer<Date>() {
    @Override
    public Date deserialize(JsonElement json, Type typeOfT,
                JsonDeserializationContext context) throws JsonParseException {
        return json == null ? null : new Date(json.getAsLong());
    }
};

Gson gson = new GsonBuilder()
                .registerTypeAdapter(Date.class, ser)
                .registerTypeAdapter(Date.class, deser)
                .create();

Gson’u kalıtımla kullanma #

Gson, kullanıma hazır kalıtımı desteklemez.

Diyelim ki aşağıdaki sınıf hiyerarşisine sahibiz:

public class BaseClass {
    int a;
 
    public int getInt() {
        return a;
   }
}
 
public class DerivedClass1 extends BaseClass {
     int b;
 
     @Override
     public int getInt() {
         return b;
     }
 }
 
public class DerivedClass2 extends BaseClass {
    int c;
 
    @Override
    public int getInt() {
        return c;
    }
}

Ve şimdi “DerivedClass1” örneğini bir JSON dizesine serileştirmek istiyoruz

DerivedClass1 derivedClass1 = new DerivedClass1();
derivedClass1.b = 5;
derivedClass1.a = 10;
 
Gson gson = new Gson();
String derivedClass1Json = gson.toJson(derivedClass1);

Şimdi, başka bir yerde, bu json dizesini alıyoruz ve seri durumdan çıkarmak istiyoruz - ancak derleme zamanında bunun yalnızca bir “BaseClass” örneği olması gerektiğini biliyoruz:

BaseClass maybeDerivedClass1 = gson.fromJson(derivedClass1Json, BaseClass.class);
System.out.println(maybeDerivedClass1.getInt());

Ancak GSON, “derivedClass1Json"un orijinal olarak “DerivedClass1” örneğinin bir örneği olduğunu bilmiyor, bu nedenle bu, 10 yazdıracaktır.

Bu nasıl çözülür?

Bu tür durumları ele alan kendi JsonDeserializerınızı oluşturmanız gerekir. Çözüm tamamen temiz değil, ama daha iyisini bulamadım.

İlk önce, temel sınıfınıza aşağıdaki alanı ekleyin

@SerializedName("type")
private String typeName;

Ve onu temel sınıf yapıcısında başlatın

public BaseClass() {
    typeName = getClass().getName();
}

Şimdi aşağıdaki sınıfı ekleyin:

public class JsonDeserializerWithInheritance<T> implements JsonDeserializer<T> {
 
 @Override
 public T deserialize(
     JsonElement json, Type typeOfT, JsonDeserializationContext context)
     throws JsonParseException {
     JsonObject jsonObject = json.getAsJsonObject();
     JsonPrimitive classNamePrimitive = (JsonPrimitive) jsonObject.get("type");
 
     String className = classNamePrimitive.getAsString();
 
     Class<?> clazz;
     try {
     clazz = Class.forName(className);
     } catch (ClassNotFoundException e) {
     throw new JsonParseException(e.getMessage());
     }
     return context.deserialize(jsonObject, clazz);
 }
}

Geriye kalan tek şey her şeyi bağlamak -

GsonBuilder builder = new GsonBuilder();
 builder
 .registerTypeAdapter(BaseClass.class, new JsonDeserializerWithInheritance<BaseClass>());
 Gson gson = builder.create();

Ve şimdi, aşağıdaki kodu çalıştırarak-

 DerivedClass1 derivedClass1 = new DerivedClass1();
 derivedClass1.b = 5;
 derivedClass1.a = 10;
 String derivedClass1Json = gson.toJson(derivedClass1);
 
 BaseClass maybeDerivedClass1 = gson.fromJson(derivedClass1Json, BaseClass.class);
 System.out.println(maybeDerivedClass1.getInt());

5 yazdıracak.
