Android: Dahili ve Harici Depolama - Terminoloji Açıklaması #

Android geliştiricileri (çoğunlukla yeni başlayanlar), Dahili ve Harici depolama terminolojisi konusunda kafaları karıştı. Bununla ilgili Stackoverflow’ta birçok soru var. Bunun temel nedeni, Google/resmi Android belgelerine göre terminolojinin normal Android işletim sistemi kullanıcısından oldukça farklı olmasıdır. Bu yüzden bunu belgelemenin yardımcı olacağını düşündüm.

Biz ne düşünüyoruz - Kullanıcı Terminolojisi (UT)

Dahili depolama(UT) Harici depolama(UT)
telefonun dahili hafızası çıkarılabilir Secure Digital(SD) kart veya mikro SD depolama
Örnek: Nexus 6P’nin 32 GB dahili belleği. Örnek: samsung, sandisk, stronsiyum, transcend ve diğerleri gibi satıcılar tarafından sağlanan çıkarılabilir SD kartlarda depolama alanı

Ancak, Android Belgelerine/Kılavuzuna göre - Google’ın Terminolojisi (GT)

Dahili depolama(GT):

Varsayılan olarak, dahili depolamaya kaydedilen dosyalar sizin için özeldir. uygulama ve diğer uygulamalar bunlara erişemez (veya kullanıcı).

Harici depolama(GT):

Bu, çıkarılabilir bir depolama ortamı (örneğin bir SD kart) veya bir dahili (çıkarılamaz) depolama.

Harici Depolama(GT) iki türe ayrılabilir:

Birincil Harici Depolama İkincil Harici Depolama veya Çıkarılabilir depolama(GT)
Bu, telefonun dahili dahili belleği (veya) Dahili depolama(UT) ile aynıdır. Bu, çıkarılabilir mikro SD kart depolama (veya) Harici depolama(UT)
Örnek: Nexus 6P’nin 32 GB dahili belleği. Örnek: samsung, sandisk, stronsiyum, transcend ve diğerleri gibi satıcılar tarafından sağlanan çıkarılabilir SD kartlarda depolama alanı
Bu tür depolamaya, telefonunuzu USB kablosuyla PC’ye bağlayıp USB seçenekleri bildiriminde Kamera(PTP) seçerek Windows PC’de erişilebilir. Bu tür depolamaya, telefonunuzu PC’ye bağlayarak Windows PC’de erişilebilir. USB kablosu aracılığıyla ve USB seçenekleri bildiriminde Dosya aktarımı’nı seçerek.

Kısaca,

Harici Depolama(GT) = Dahili Depolama(UT) ve Harici Depolama(UT)

Çıkarılabilir Depolama(GT) = Harici Depolama(UT)

Dahili Depolama(GT)’nin UT’de bir terimi yoktur.

Açıkça anlatayım,

Dahili Depolama(GT): Varsayılan olarak, dahili depolamaya kaydedilen dosyalar uygulamanıza özeldir ve diğer uygulamalar bunlara erişemez. Uygulama kullanıcınız da dosya yöneticisini kullanarak bunlara erişemez; Dosya yöneticisinde “gizli dosyaları göster” seçeneğini etkinleştirdikten sonra bile. Dahili Depolama Alanındaki (GT) dosyalara erişmek için Android telefonunuzu rootlamanız gerekir. Ayrıca, kullanıcı uygulamanızı kaldırdığında bu dosyalar kaldırılır/silinir.

Yani Dahili Depolama (GT), Nexus 6P’nin 32/64 GB dahili belleği olarak düşündüğümüz DEĞİL

Genel olarak, Dahili Depolama (GT) konumu şöyle olur: /data/data/your.application.package.appname/someDirectory/

Harici Depolama(GT):

Her Android uyumlu cihaz, dosyaları kaydetmek için kullanabileceğiniz paylaşılan bir “harici depolamayı” destekler. Harici depolama birimine kaydedilen dosyalar dünya çapında okunabilirdir ve bir bilgisayarda dosyaları aktarmak için USB yığın depolamayı etkinleştirdiklerinde kullanıcı tarafından değiştirilebilir.

Harici Depolama (GT) konumu: Dahili depolama alanınızda (UT) veya çıkarılabilir depolama alanınızda (GT), yani mikro SD kartınızda herhangi bir yerde olabilir. Telefonunuzun OEM’ine ve ayrıca Android işletim sistemi sürümüne bağlıdır.

Harici Depolama (GT) üzerindeki dosyaları okumak veya yazmak için uygulamanızın “READ_EXTERNAL_STORAGE” veya “WRITE_EXTERNAL_STORAGE” sistem izinlerini alması gerekir.

Örneğin:

<manifest ...>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
    ...
</manifest>

Dosyaları hem okumanız hem de yazmanız gerekiyorsa, yalnızca WRITE_EXTERNAL_STORAGE izni, çünkü dolaylı olarak ayrıca okuma erişimi gerektirir.

Harici Depolama(GT) içinde, uygulama-özel olan dosyaları da kaydedebilirsiniz.

Fakat,

Kullanıcı uygulamanızı kaldırdığında, bu dizin ve tüm içeriği silinir.

Uygulamaya özel dosyaları ne zaman Harici Depolama(GT) içine kaydetmeniz gerekir?

Diğer uygulamaların kullanması amaçlanmayan dosyaları işliyorsanız (yalnızca uygulamanız tarafından kullanılan grafik dokular veya ses efektleri gibi), harici depolama biriminde özel bir depolama dizini kullanmalısınız.

Android 4.4’ten başlayarak, uygulamanızın özel dizinlerindeki dosyaları okumak veya yazmak, “READ_EXTERNAL_STORAGE” veya “WRITE_EXTERNAL_STORAGE” izinlerini gerektirmez. Böylece, “maxSdkVersion” özniteliğini ekleyerek iznin yalnızca Android’in alt sürümlerinde istenmesi gerektiğini beyan edebilirsiniz:

<manifest ...>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"
                     android:maxSdkVersion="18" />
    ...
</manifest

Dahili Depolamada(GT) depolama yöntemleri:

Bu yöntemlerin her ikisi de Context sınıfında bulunur

File getDir (String name, int mode)

File getFilesDir ()

Birincil Harici Depolamada depolama yöntemleri, yani Dahili Depolama(UT):

File getExternalStorageDirectory ()

File getExternalFilesDir (String type)

File getExternalStoragePublicDirectory (String type)

Başlangıçta herkes Environment.getExternalStorageDirectory() kullanıyordu, bu da Birincil Harici Depolama’nın köküne işaret ediyordu. Sonuç olarak, Birincil Harici Depolama rastgele içerikle dolduruldu.

Daha sonra, bu iki yöntem eklendi:

  1. ‘Context’ sınıfında, Birincil Harici Depolama üzerindeki uygulamaya özel bir dizine işaret ederek getExternalFilesDir()‘ü eklediler. Uygulama kaldırıldığında bu dizin ve içeriği silinecek.

  2. Fotoğraflar ve filmler gibi iyi bilinen dosya türlerini depolamak için merkezi yerler için Environment.getExternalStoragePublicDirectory(). Uygulama kaldırıldığında bu dizin ve içeriği SİLİNMEZ.

Çıkarılabilir Depolama Birimi’nde (GT) saklama yöntemleri, yani mikro SD kart

API seviyesi 19‘dan önce, SD kartta depolamanın resmi bir yolu yoktu. Ancak çoğu, resmi olmayan kütüphaneler veya API’ler kullanarak bunu yapabilir.

Resmi olarak, API seviyesi 19’da (Android sürüm 4.4 - Kitkat) ‘Context’ sınıfında bir yöntem tanıtıldı.

File[] getExternalFilesDirs (String type)

Uygulamaya özel dizinlere mutlak yollar döndürür. uygulamanın yerleştirebileceği paylaşılan/harici depolama cihazları sahip olduğu kalıcı dosyalar. Bu dosyalar uygulamanın içindedir, ve genellikle kullanıcı tarafından medya olarak görülmez.

Bu, yolları her iki Harici Depolama(GT) - Dahili bellek ve Micro SD kart türüne döndüreceği anlamına gelir. Genellikle ikinci yol, mikro SD kartın depolama yolu olacaktır (ancak her zaman değil). Bu yüzden kodu bu yöntemle çalıştırarak kontrol etmeniz gerekiyor.

Kod snippet’li örnek:

Boş aktivite ile yeni bir android projesi oluşturdum, içine aşağıdaki kodu yazdım

‘MainActivity.java’nın ‘protected void onCreate(Bundle saveInstanceState)’ yöntemi

    File internal_m1 = getDir("custom", 0);
    File internal_m2 = getFilesDir();

    File external_m1 =  Environment.getExternalStorageDirectory();
    
    File external_m2 =  getExternalFilesDir(null);
    File external_m2_Args = getExternalFilesDir(Environment.DIRECTORY_PICTURES);

    File external_m3 = Environment.getExternalStoragePublicDirectory(Environment.DIRECTORY_PICTURES);
    
    File[] external_AND_removable_storage_m1 = getExternalFilesDirs(null);
    File[] external_AND_removable_storage_m1_Args = getExternalFilesDirs(Environment.DIRECTORY_PICTURES);

Yukarıdaki kodu çalıştırdıktan sonra,

Çıktı:

internal_m1: /data/data/your.application.package.appname/app_custom

internal_m2: /data/data/your.application.package.appname/files

external_m1: /storage/emulated/0

external_m2: /storage/emulated/0/Android/data/your.application.package.appname/files

external_m2_Args: /storage/emulated/0/Android/data/your.application.package.appname/files/Pictures

external_m3: /storage/emulated/0/Pictures

external_AND_removable_storage_m1 (first path):
/storage/emulated/0/Android/data/your.application.package.appname/files

external_AND_removable_storage_m1 (second path):    
/storage/sdcard1/Android/data/your.application.package.appname/files

 external_AND_removable_storage_m1_Args (first path):
/storage/emulated/0/Android/data/your.application.package.appname/files/Pictures

external_AND_removable_storage_m1_Args (second path): /storage/sdcard1/Android/data/your.application.package.appname/files/Pictures

Not: Telefonumu Windows PC’ye bağladım; hem geliştirici seçeneklerini hem de USB hata ayıklamasını etkinleştirdi ve ardından bu kodu çalıştırdı. telefonunuzu bağlamazsanız; ancak bunun yerine bunu Android öykünücüsünde çalıştırın, çıktınız değişebilir. Telefon modelim Coolpad Note 3 - Android 5.1’de çalışıyor

Telefonumdaki depolama konumları:

Micro SD depolama konumu: /storage/sdcard1

Dahili Depolama(UT) konumu: /storage/sdcard0.

/sdcard & /storage/emulated/0ın da Dahili Depolamayı(UT) işaret ettiğini unutmayın. Ancak bunlar /storage/sdcard0 için sembolik bağlantılar.

Android’deki farklı depolama yollarını net bir şekilde anlamak için lütfen bu yanıtı gözden geçirin

Sorumluluk reddi: Yukarıda belirtilen tüm depolama yolları benim telefonumdaki yollardır. Dosyalarınız aynı depolama yollarında depolanmayabilir. Çünkü depolama yerleri/yolları satıcınıza, üreticinize ve Android işletim sisteminin farklı sürümlerine bağlı olarak diğer cep telefonlarında değişiklik gösterebilir.

Harici Depolamayı Kullanma #

“Harici” Depolama, dosyaları kullanıcının cihazına kaydetmek için kullanabileceğimiz başka bir depolama türüdür. “Dahili” Depolamadan bazı önemli farklılıkları vardır, yani:

  • Her zaman mevcut değildir. Çıkarılabilir bir ortam (SD kart) durumunda, kullanıcı depolamayı kolayca çıkarabilir.
  • Özel değil. Kullanıcının (ve diğer uygulamaların) bu dosyalara erişimi vardır.
  • Kullanıcı uygulamayı kaldırırsa, getExternalFilesDir() ile alınan dizine kaydettiğiniz dosyalar kaldırılacaktır.

Harici Depolamayı kullanmak için önce uygun izinleri almamız gerekir. Kullanmanız gerekecek:

  • android.permission.WRITE_EXTERNAL_STORAGE okuma ve yazma için
  • android.permission.READ_EXTERNAL_STORAGE sadece okumak için

Bu izinleri vermek için, bunları “AndroidManifest.xml” dosyanızda bu şekilde tanımlamanız gerekir.

<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />

NOT: Bunlar Tehlikeli izinler olduğundan, API Düzeyi 23 veya üzeri kullanıyorsanız, permissions at runtime talep etmeniz gerekir.

Harici Depolamadan yazmaya veya okumaya başlamadan önce, depolama ortamının kullanılabilir olup olmadığını her zaman kontrol etmelisiniz.

String state = Environment.getExternalStorageState();
if (state.equals(Environment.MEDIA_MOUNTED)) {
    // Available to read and write
}
if (state.equals(Environment.MEDIA_MOUNTED) || 
        state.equals(Environment.MEDIA_MOUNTED_READ_ONLY)) {
    // Available to at least read
}

Dosyaları Harici Depolamaya yazarken, dosyanın Genel mi yoksa Özel olarak mı tanınması gerektiğine karar vermelisiniz. Bu tür dosyaların her ikisi de kullanıcı ve cihazdaki diğer uygulamalar tarafından hala erişilebilir olsa da, aralarında önemli bir ayrım vardır.

Kullanıcı uygulamayı kaldırdığında genel dosyalar cihazda kalmalıdır. Genel olarak kaydedilmesi gereken bir dosya örneği, uygulamanız aracılığıyla çekilen fotoğraflar olabilir.

Kullanıcı uygulamayı kaldırdığında özel dosyaların tümü kaldırılmalıdır. Bu tür dosyalar uygulamaya özel olacaktır ve kullanıcı veya diğer uygulamalar için kullanılmayacaktır. Eski. uygulamanız tarafından indirilen/kullanılan geçici dosyalar.

Hem Genel hem de Özel dosyalar için “Belgeler” dizinine nasıl erişeceğiniz aşağıda açıklanmıştır.

Dahili Depolamayı Kullanma #

Varsayılan olarak, Dahili Depolamaya kaydettiğiniz tüm dosyalar uygulamanıza özeldir. Normal şartlar altında diğer uygulamalar veya kullanıcı tarafından erişilemezler. Bu dosyalar, kullanıcı uygulamayı kaldırdığında silinir.

Bir Dosyaya Metin Yazmak için

String fileName= "helloworld";
String textToWrite = "Hello, World!";
FileOutputStream fileOutputStream;

try {
  fileOutputStream = openFileOutput(fileName, Context.MODE_PRIVATE);
  fileOutputStream.write(textToWrite.getBytes());
  fileOutputStream.close();
} catch (Exception e) {
  e.printStackTrace();
}

Mevcut Bir Dosyaya Metin Eklemek İçin

‘openFileOutput’ mod parametresi için ‘Context.MODE_APPEND’ kullanın

fileOutputStream = openFileOutput(fileName, Context.MODE_APPEND);

Model sınıfı oluştur

//create one directory model class
//to store directory title and type in list

public class DirectoryModel {
    String dirName;
    int dirType; // set 1 or 0, where 0 for directory and 1 for file.

    public int getDirType() {
        return dirType;
    }

    public void setDirType(int dirType) {
        this.dirType = dirType;
    }

    public String getDirName() {
        return dirName;
    }

    public void setDirName(String dirName) {
        this.dirName = dirName;
    }
}

Dizin verilerini eklemek için dizin modelini kullanarak liste oluşturun.

//define list to show directory

List<DirectoryModel> rootDir = new ArrayList<>();

Aşağıdaki yöntemi kullanarak dizini alın.

//to fetch device directory

private void getDirectory(String currDir) { // pass device root directory
        File f = new File(currDir);
        File[] files = f.listFiles();
        if (files != null) {
            if (files.length > 0) {
                rootDir.clear();
                for (File inFile : files) {
                    if (inFile.isDirectory()) { //return true if it's directory
                        // is directory
                        DirectoryModel dir = new DirectoryModel();
                        dir.setDirName(inFile.toString().replace("/storage/emulated/0", ""));
                        dir.setDirType(0); // set 0 for directory
                        rootDir.add(dir);
                    } else if (inFile.isFile()) { // return true if it's file
                        //is file
                        DirectoryModel dir = new DirectoryModel();
                        dir.setDirName(inFile.toString().replace("/storage/emulated/0", ""));
                        dir.setDirType(1); // set 1 for file
                        rootDir.add(dir);
                    }
                }
            }
            printDirectoryList();
        }
    }

Günlükte dizin listesini yazdırın.

//print directory list in logs

private void printDirectoryList() {
    for (int i = 0; i < rootDir.size(); i++) {
        Log.e(TAG, "printDirectoryLogs: " + rootDir.get(i).toString());
    }
}
