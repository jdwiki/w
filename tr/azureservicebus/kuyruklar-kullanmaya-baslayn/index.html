Basit kod #

Daha fazla örnek kod burada bulunabilir.

1. Azure portalını kullanarak bir ad alanı oluşturun #

  1. Azure klasik portalında oturum açın.

  2. Portalın sol gezinme bölmesinde Service Bus‘a tıklayın.

  3. Portalın alt bölmesinde Oluştur‘a tıklayın.

  4. Yeni ad alanı ekle iletişim kutusunda bir ad alanı adı girin. Sistem, adın uygun olup olmadığını hemen kontrol eder.

    ![Namespace name][namespace-name]

  5. Ad alanı adının kullanılabilir olduğundan emin olduktan sonra, ad alanınızın barındırılacağı ülke veya bölgeyi seçin.

  6. İletişim kutusundaki diğer alanları varsayılan değerleriyle (Mesajlaşma ve Standart Katman) bırakın, ardından Tamam onay işaretini tıklayın. Sistem şimdi ad alanınızı oluşturur ve etkinleştirir. Sistem, hesabınız için kaynak sağladığı için birkaç dakika beklemeniz gerekebilir.

    ![Created successfully][created-successfully]

###Kimlik bilgilerini alın

  1. Sol gezinme bölmesinde, kullanılabilir ad alanlarının listesini görüntülemek için Service Bus düğümüne tıklayın:

  2. Gösterilen listeden az önce oluşturduğunuz ad alanını seçin:

    ![Select namespace][select-namespace]

  3. Bağlantı Bilgileri’ne tıklayın.

    ![Connection information][connection-information]

  4. Bağlantı bilgilerine erişim bölmesinde, SAS anahtarını ve anahtar adını içeren bağlantı dizesini bulun.

    ![Access connection information][access-connection-information]

  5. Anahtarı not edin veya panoya kopyalayın.

2. Azure portalını kullanarak bir kuyruk oluşturun #

  1. Azure classic portal‘da oturum açın.

  2. Portalın sol gezinme bölmesinde Service Bus‘a tıklayın.

  3. Kuyruğu oluşturmak istediğiniz ad alanını seçin. Bu durumda, “mytestns1” olur.

    ![Selecting a namespace][select-namespace]

  4. Kuyruklar‘ı seçin.

  5. Sol alt köşedeki Yeni‘yi ve ardından Hızlı Oluştur‘u seçin.

  6. Sıra Adı’nı girin ve uygun ad alanının seçildiğinden emin olun.

  7. Yeni bir kuyruk oluştur‘u seçin.

3. Kuyruğa mesaj gönder #

Kuyruğa mesaj göndermek için Visual Studio kullanarak bir C# konsol uygulaması yazacağız.

Konsol uygulaması oluşturun #

  1. Visual Studio’yu başlatın ve yeni bir Konsol uygulaması oluşturun.

Service Bus NuGet paketini ekleyin #

  1. Yeni oluşturulan projeye sağ tıklayın ve NuGet Paketlerini Yönet‘i seçin.

  2. Gözat sekmesine tıklayın, ardından “Microsoft Azure Service Bus”ı arayın ve Microsoft Azure Service Bus öğesini seçin. Kurulumu tamamlamak için Yükle‘ye tıklayın ve ardından bu iletişim kutusunu kapatın.

    ![Select a NuGet package][nuget-pkg]

Kuyruğa mesaj göndermek için bir kod yazın #

  1. Aşağıdaki using ifadesini Program.cs dosyasının en üstüne ekleyin.

    using Microsoft.ServiceBus.Messaging;

  2. Main yöntemine aşağıdaki kodu ekleyin, ad alanı oluşturulurken elde edilen bağlantı dizesi olarak connectionString değişkenini ayarlayın ve sıra oluşturulurken kullanılan kuyruk adı olarak queueName olarak ayarlayın .

    var connectionString = "<Your connection string>";
var queueName = "<Your queue name>";

var client = QueueClient.CreateFromConnectionString(connectionString, queueName);
var message = new BrokeredMessage("This is a test message!");
client.Send(message);

    Here is what your Program.cs should look like.

    using System;
using Microsoft.ServiceBus.Messaging;

namespace GettingStartedWithQueues
{
    class Program
    {
        static void Main(string[] args)
        {
            var connectionString = "<Your connection string>";
            var queueName = "<Your queue name>";

            var client = QueueClient.CreateFromConnectionString(connectionString, queueName);
            var message = new BrokeredMessage("This is a test message!");

            client.Send(message);
        }
    }
}

  3. Programı çalıştırın ve Azure klasik portalını kontrol edin. Kuyruk Uzunluğu değerinin şimdi 1 olması gerektiğine dikkat edin.

    ![Queue length][queue-length-send]

4. Kuyruktan mesaj alın #

  1. Yeni bir konsol uygulaması oluşturun ve yukarıdaki gönderen uygulamaya benzer şekilde Service Bus NuGet paketine bir başvuru ekleyin.

  2. Program.cs dosyasının en üstüne aşağıdaki ‘using’ ifadesini ekleyin.

    using Microsoft.ServiceBus.Messaging;

  3. Main yöntemine aşağıdaki kodu ekleyin, ad alanını oluştururken elde edilen bağlantı dizesi olarak connectionString değişkenini ayarlayın ve oluştururken kullandığınız kuyruk adı olarak queueName öğesini ayarlayın. sıra.

    var connectionString = "";
var queueName = "samplequeue";

var client = QueueClient.CreateFromConnectionString(connectionString, queueName);

client.OnMessage(message =>
{
  Console.WriteLine(String.Format("Message body: {0}", message.GetBody<String>()));
  Console.WriteLine(String.Format("Message id: {0}", message.MessageId));
});

Console.ReadLine();

    Here is what your Program.cs file should look like:

    using System;
using Microsoft.ServiceBus.Messaging;

namespace GettingStartedWithQueues
{
  class Program
  {
    static void Main(string[] args)
    {
      var connectionString = "";
      var queueName = "samplequeue";

      var client = QueueClient.CreateFromConnectionString(connectionString, queueName);

      client.OnMessage(message =>
      {
        Console.WriteLine(String.Format("Message body: {0}", message.GetBody<String>()));
        Console.WriteLine(String.Format("Message id: {0}", message.MessageId));
      });

      Console.ReadLine();
    }
  }
}

  4. Programı çalıştırın ve portalı kontrol edin. Kuyruk Uzunluğu değerinin şimdi 0 olması gerektiğine dikkat edin.

    ![Queue length][queue-length-receive]

Tebrikler! Artık bir kuyruk oluşturdunuz, bir mesaj gönderdiniz ve bir mesaj aldınız.
