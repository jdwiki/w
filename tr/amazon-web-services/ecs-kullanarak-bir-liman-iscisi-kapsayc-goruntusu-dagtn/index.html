ECS kullanarak bir liman işçisi kapsayıcı görüntüsü dağıtın

Bu sayfada

  • Kavram kanıtı olarak AWS ECS hizmetinde örnek bir uygulama dağıtın
  • örnek-görev.json

ECS örneklerini bir kümeye ekleyebilmeniz için önce EC2 Yönetim Konsolu’na gitmeli ve ‘AmazonEC2ContainerServiceforEC2Role’ ilkesinin ekli olduğu bir IAM rolüyle ’ecs için optimize edilmiş’ örnekler oluşturmalısınız.

  1. EC2 Kontrol Panelinize gidin ve ‘Örneği Başlat’ düğmesini tıklayın.

  2. ‘Topluluk AMI’leri’ altında, ’ecs-optimized’ ifadesini arayın ve proje ihtiyaçlarınıza en uygun olanı seçin. Herhangi biri işe yarayacaktır. Sonrakine tıkla.

  3. “Örnek Ayrıntılarını Yapılandır"a ulaştığınızda, “yeni IAM rolü oluştur bağlantısı"na tıklayın ve “ecsInstanceRole” adlı yeni bir rol oluşturun.

  4. “AmazonEC2ContainerServiceforEC2Role” ilkesini bu role ekleyin.

  5. Varsayılan olarak, kapsayıcı örneğiniz “varsayılan” kümenizde başlatılır. Varsayılan yerine kendi kümenize başlamak istiyorsanız, “Gelişmiş Ayrıntılar” listesini seçin ve aşağıdaki komut dosyasını “Kullanıcı verileri” alanına yapıştırın, “kümenizin_adı"nı kümenizin adıyla değiştirin.

    #!/bin/bash echo ECS_CLUSTER=your_cluster_name » /etc/ecs/ecs.config

  6. Ardından, ECS Bulut Sunucunuzu yapılandırmayı tamamlayın.

NOT: Bir web sunucusu oluşturuyorsanız, 80 numaralı bağlantı noktasına erişime izin vermek için bir “güvenlik Grubu” oluşturmak isteyeceksiniz.

  1. Bir Depo Oluşturun: aws ecr create-deposu --depo-adı örnek-deposu
  2. Docker istemcinizin kimliğini kayıt defterinize doğrulayın: `aws ecr get-login –region us-east-1 | ş’
  3. Docker imajınızı oluşturun: docker build -t example-image .
  4. Resminizi etiketleyin, böylece resmi bu havuza gönderebilirsiniz: docker etiketi örnek-resim:en son örnek-ad alanı/örnek-resim:en son
  5. Bu görüntüyü yeni oluşturduğunuz AWS deponuza gönderin: `docker push örnek-ad alanı/örnek-görüntü:en son’
  6. Bir ECS Görev Tanımı kaydedin: aws ecs register-task-definition --cli-input-json example-task.json
  7. Görevi çalıştırın: aws ecs çalıştırma-görev --görev-tanımı örnek-görev

Kavram kanıtı olarak AWS ECS hizmetinde örnek bir uygulama dağıtın #

Kavram kanıtı olarak AWS ECS hizmetinde örnek bir uygulamayı denemek için aşağıdaki adımları izleyin.

  1. AWS yönetim konsolunda oturum açın ve AWS hizmet kataloğu - > Hesaplama - > Ec2‘ye gidin

  2. Amazon linux 64 bit işletim sistemini kullanarak bir VM (EC2 örneği) oluşturun, bunu docker, git, AWS ECS aracı aracı ve diğer araçları yapılandırmak için kullanacağız. Aynı VM’yi, konteyner tabanlı uygulamaları dağıtmak için ECS kümesinde bir düğüm olarak kullanacağız. Bir VM oluşturmak için aşağıdaki adımları izleyin. a) Bir EC2 bulut sunucusu oluşturmak için olağan adımları izleyin, EC2 bulut sunucusu oluşturma sırasında sonraki adımlarda özel vurgu yapın. b) En az takip eden izinlere sahip bir IAM rolü seçin – AmazonEC2ContainerServiceforEC2Role c) VM’de Java’nın kurulu olduğundan emin olun

  3. Docker kurulumu [aşağıdaki komutları yürütün] önce yum paket deposunu güncelleyin

    sudo yum update –y şimdi docker’ı kurmak için yum kurulumunu yürütün

    sudo yum install -y docker

  4. Docker hizmetini başlatın

    sudo service docker start

  5. Docker komutlarını sudo kullanmadan yürütebilmek için ec2 kullanıcısını docker grubuna ekleyin.

    sudo usermod -a -G docker ec2-user

  6. Yeni liman işçisi grubu izinlerini almak için EC2’den çıkış yapın ve tekrar oturum açın.

  7. ec2 kullanıcısının Docker komutlarını sudo olmadan çalıştırabildiğini doğrulayın.

    docker info

  8. Git’i Yükleme

    sudo yum install -y git

  9. Ec2 örneğindeki örnek PHP uygulamasını git’ten klonlayın. Bu uygulamayı POC’miz için kullanacağız.

    git clone https://github.com/awslabs/ecs-demo-php-simple-app

    cd ecs-demo-php-simple-app

dizin içeriğini listeleyerek Dockerfile’ın var olduğunu doğrulayın

ls
  1. AWS hizmet kataloğu -> Hesaplama -> Ec2 Konteyner Hizmeti’ne gidin
  2. Başlayın’a tıklayın
  3. İptal’e tıklayın
  4. Soldaki Depolar menüsünden depolara tıklayın
  5. Başlayın’a tıklayın

15. Depo adını girin ve ileri’ye tıklayın 16. Ec2 araçlarını yapılandırın

aws configure

hesabınıza göre AWS Erişim Anahtarı Kimliği, Gizli Erişim anahtarı, varsayılan bölge adı sağlayın 17. Docker imajını oluşturun, etiketleyin ve itin a) Docker istemcinizin kimliğini kayıt defterinizde doğrulamak için kullanabileceğiniz docker oturum açma komutunu alın:

aws ecr get-login --region us-east-1

b) Komut dönüşünü önceki adımın çıktısı olarak çalıştırın 18. Docker dosyanızdan Docker imajını oluşturun. (Örnek bir docker uygulaması indirdiğiniz Adım 9’u hatırlayın) a)

docker build -t amazon-ecs-sample .

(”.” Geçerli dizin anlamına gelir) b) Görüntünün doğru oluşturulduğunu ve görüntü adının, değişikliklerinizi docker görüntüsüne aktarabileceğiniz bir havuz içerdiğini doğrulamak için docker görüntülerini çalıştırın.

docker images

c) Yeni oluşturulan görüntüyü çalıştırın. -p 80:80 seçeneği, kapsayıcıdaki açıkta kalan 80 numaralı bağlantı noktasını ana bilgisayar sistemindeki 80 numaralı bağlantı noktasıyla eşler (bu durumda Ec2 örneği).

docker run -p 80:80 amazon-ecs-sample

“Apache2: Sunucunun tam etki alanı adı, SunucuAdı için 172.17.0.2 kullanılarak güvenilir bir şekilde belirlenemedi” uyarısını yoksayın. 19. Tarayıcıda örnek uygulama web sayfasına erişmeyi deneyin, örnekle ilişkili güvenlik gruplarında 80 numaralı bağlantı noktasının açık olduğundan emin olun.

http://<ec2-instance-dns-address>

  1. ctrl + c tuşlarına basın, bu docker görüntüsünü durduracaktır. Örnek uygulama erişilebilir olmamalıdır.

  2. Şimdi, örnek docker uygulamamızı başarıyla doğruladıktan sonra, örnek uygulamayı otomatik olarak çalıştırmak için bir küme yapılandırmayı deneyeceğiz. Ayrıca, demo amacıyla mevcut ec2 örneğini kümede bir düğüm olarak kullanmaya çalışacağız. Bu, ec2 bulut sunucusuna bir aracı programı yüklenerek gerçekleştirilebilir.

  3. ec2 bulut sunucusuna Amazon ECS Container Agent yükleme a)

    sudo yum install -y ecs-init

b) Docker arka plan programını yeniden başlatın

sudo service docker restart

c) ecs-init upstart işini başlatın

sudo start ecs

d) (İsteğe bağlı) Aracı iç gözlem API’sı ile aracının çalıştığını doğrulayabilir ve yeni kapsayıcı örneğiniz hakkında bazı bilgileri görebilirsiniz. Güvenlik grubunda 51678 numaralı bağlantı noktasının açık olduğundan emin olun.

curl http://localhost:51678/v1/metadata

  1. AWS hizmet kataloğu -> Hesaplama -> Ec2 Container Service -> Küme’ye gidin ve varsayılan bir kümenin oluşturulduğunu doğrulayın

  2. Şimdi bir görev grubu oluşturmaya ve kümede çalıştırmak için docker imajımızı görev olarak eklemeye devam ediyoruz. a) ecs-demo-php-simple-app klasöründeki simple-app-task-def.json dosyasını inceleyin. b) Simple-app-task-def.json dosyasını düzenleyin ve ücretsiz katmana uygun örnekte çalışabilmesi için anıyı azaltın (birinin bu POC için ücretsiz katmana uygun ec2 örneği kullandığını varsayıyorum, aksi takdirde belleği azaltmaya gerek yoktur) sınır) c) simple-app-task-def.json dosyasındaki tüm olaylarda memory=250 güncelleyin d) simple-app-task-def.json dosyasıyla bir görev tanımı kaydedin.

    aws ecs register-task-definition –cli-input-json file://simple-app-task-def.json

e) ec2 konteyner hizmet sayfasında görev tanımına gidin, kayıtlı görev tanımını bulacaksınız

f) Konsol-örnek-uygulama görev tanımıyla bir görevi çalıştırmak için aşağıdaki AWS CLI komutunu kullanın.

aws ecs run-task --task-definition console-sample-app

g) Örnek web uygulamasını tarayıcıda açın, erişilebilir olmalıdır (19. adıma bakın)

Okuduğunuz için teşekkürler, takip tartışması için yorumlarınızı ve sorularınızı paylaşın.

örnek-görev.json #

{
  "family": "example-task",
  "containerDefinitions": [
    {
        "environment": [],
        "name": "example-container",
        "image": "example-namespace/example-image:latest",
        "cpu": 10,
        "memory": 500,
        "portMappings": [
            {
                "containerPort": 8080,
                "hostPort": 80
            }
        ],
        "entryPoint": [],
        "essential": true
    }
  ]
}
