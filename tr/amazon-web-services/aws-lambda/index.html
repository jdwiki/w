AWS Lambda, sunucuları tedarik etmenize veya yönetmenize gerek kalmadan arka uç kodu çalıştırmanıza izin veren bir hizmettir. AWS Lambda, ölçeklendirme ve yüksek kullanılabilirlik ile ilgilenir. Maliyet, doğrudan kodunuzun ne sıklıkta ve ne kadar süreyle yürütüldüğüne bağlıdır.

AWS Lambda işlevlerinin farklı dillerde nasıl oluşturulacağına ve dağıtılacağına ilişkin örnekler bulacaksınız.

  • AWS Lambda kodu durum bilgisi olmadan yazılmalıdır. Bir lambda örneği korunup yeniden kullanılabilse de, bunu asla beklememelisiniz.

Temel Gradle Java projesi #

Java kodunu AWS Lambda’ya dağıtmak için, çalışma zamanı sırasında ihtiyaç duyulan tüm bağımlılıkları içeren bir dağıtım zip dosyası oluşturmanız gerekir. Gradle’da örnek proje:

apply plugin: 'java'

repositories {
    mavenCentral()
}

dependencies {
    compile 'com.amazonaws:aws-lambda-java-core:1.1.0'
}

task buildZip(type: Zip) {
    from compileJava
    from processResources
    into('lib') {
        from configurations.runtime
    }
}

build.dependsOn buildZip

‘gradle build’i çalıştırmak, kodunuzla birlikte tüm bağımlılıkları içeren ve dağıtılmaya hazır bir zip dosyası oluşturacaktır.

Java’da Temel Lambda Kodu #

Bir lambda, uygulamanıza giriş noktası olarak hizmet edecek bir işleyiciye ihtiyaç duyar. Her işleyicinin ‘RequestHandler<I, O>’ arabirimini uygulaması gerekir; burada ‘I’ giriş tipi ve ‘O’ çıkış tipidir. Girdi türü daha sonra “handleRequest()” yöntemine iletilir ve yöntem çıktı türünü döndürür.

Basit bir örnek şöyle görünebilir:

package com.example.lambda;

import com.amazonaws.services.lambda.runtime.Context;
import com.amazonaws.services.lambda.runtime.RequestHandler;

public class HelloNumber implements RequestHandler<Integer, String> {

    @Override
    public String handleRequest(Integer input, Context context) {
        return "You sent " + input;
    }
}

JavaScript’te temel Lambda kodu (NodeJs) #

Bir lambda, uygulamanıza giriş noktası olarak hizmet edecek bir işleyiciye ihtiyaç duyar. En basit durumda, “bağlam"dan girdinizi alırsınız ve sonucu “callback()” işlevini kullanarak iletirsiniz:

export.handler = (olay, bağlam, geri arama) => { callback(null, ‘You sent ’ + event.number); };

Web GUI’sini (Java) kullanarak AWS Lambda oluşturma #

GUI’yi kullanarak bir Java lambda oluşturmak için önce dağıtım zip’inizin hazır olduğundan emin olun. Ayrıca lambda oluşturabilen bir AWS hesabına ihtiyacınız olacak.

  • AWS Lambda’ya geçin, giriş ekranını kapatın (veya Başlarken belgelerini okuyun) ve “Bir Lambda İşlevi Oluştur” düğmesine basın.
  • Uygun bir plan seçin. Plan, lambda oluşturmanıza yardımcı olabilecek örnek bir şablondur. Şimdilik, bir “Boş İşlev” ile başlayın.
  • Şimdilik tetikleyiciler için endişelenmeyin ve o ekranı atlayın.
  • İşlevinizi adlandırın ve zip dosyasını yükleyin.
  • Aynı yapılandırma ekranında, isteğe bağlı olarak istekleri işleyecek yöntemle, Paketli tam Java sınıfı olarak İşleyici adınızı yazın. Temel Lambda Kodu örneğinden Java arabirimini kullandıysanız, sınıf adı yeterlidir.
  • Aynı ekranda, Şablon(lar)dan yeni bir rol oluştur seçeneğini seçin ve rolünüzü adlandırın. Rol, lambda’nıza diğer AWS kaynaklarına erişim sağlamak için kullanılır. Bu durumda, başka bir kaynağa ihtiyacımız olmadığı için boş bırakın.
  • Kullanılabilir en düşük belleği (128MB) seçin. Bellek de maliyeti etkiler. İşlevinizi kaydedin.

Temel Lambda kodunu test etme #

Lambda’yı başarıyla dağıttıysanız, doğrudan GUI’de de test edebilirsiniz. Mavi Test düğmesine ilk kez tıkladığınızda, size yeni bir test olayının oluşturulmasını sunar.

Java kodunu Java örneğindeki Temel Lambda kodundan test ediyorsanız, tüm gövdeyi silin ve sadece “1” sayısını ekleyin.

Javascript (Düğüm)* örneğindeki *Temel Lambda kodundan Javascript kodunu test ediyorsanız, şuna benzeyen bir json oluşturun:

{
    "number": "1"
}

Testi çalıştırın. Sonucu görmelisiniz:

“1 gönderdin”

AWS API Gateway tetikleyicisi ekleme #

Muhtemelen bir Lambda’yı çağırmanın en yaygın yolu, bir REST çağrısını lambda koduna eşleyen API Ağ Geçidini kullanmaktır. Yeni bir lambda oluştururken de dahil olmak üzere, istediğiniz zaman Lambda’nıza birden çok tetikleyici ekleyebilirsiniz.

API Ağ Geçidi hakkında bilginiz varsa, dağıtılmış bir lambda ile herhangi bir yöntemi ilişkilendirebilirsiniz. Lambda işlevine ayarlanmış Entegrasyon türüyle yeni bir yöntem oluşturmanız ve lambda’nızı göstermeniz yeterlidir. Hem REST girişlerini lambda girişlerine hem de çıkışları REST çıkışlarına eşleyebilirsiniz.

YAPILACAKLAR: API Gateway’i burada mı tanımlayacağınıza veya belgelerin başka bir bölümüne atıfta bulunacağınıza karar verin.
