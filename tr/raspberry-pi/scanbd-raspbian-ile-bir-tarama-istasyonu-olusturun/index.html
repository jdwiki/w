Giriş ve kurulum #

Bu belgenin amacı, herhangi bir kullanıcı arabirimi olmayan bir tarayıcı kullanmaktır. Yaygın bir kullanım, taranan bir PDF dosyasını yalnızca tarayıcı düğmelerine basarak doğrudan Google Drive veya Dropbox’a yüklemektir.

scanbd, tarayıcı düğmelerinden eylemleri tetiklemeye izin verir, aklı başında kullanır.

İşleri biraz daha zorlaştıran şey, tarayıcıyı yoklayarak scanbd’nin cihazı kilitlemesidir. Ayrıca yerel araçlardan (scanimage gibi) tarama yapmamız gerektiğinden, gerektiğinde scandb’yi yoklamadan kesecek bir proxy ayarlamamız gerekiyor. Bu, ağı izleyen ve tarayıcıya herhangi bir yerel erişime scanbd’ye göre öncelik veren scanbm’nin işidir.

Böylece, ihtiyacımız var:

  • Yerel akıllı yapılandırmayı yalnızca ağdan tarama yapacak şekilde ayarlayın (scanbm aracılığıyla)
  • Scanbd’yi yerel tarayıcıya erişecek ve yoklayacak şekilde yapılandırın

kök: olarak

apt-get update
apt-get install libconfuse-dev libsane-dev libudev-dev libusb-dev xinetd

scanbd’nin en son sürümünü burada bulabilirsiniz: https://sourceforge.net/projects/scanbd/files/releases/

# Download the latest version of **scanbd**.
wget https://sourceforge.net/projects/scanbd/files/releases/scanbd-1.4.4.tgz/download

# Uncompress
tar xvzf scanbd-1.4.4.tgz && cd 1.4.4

# Configure with the installation path in /etc/
./configure --disable-Werror

# install it
make
make install

# Add a dbus policy to authorize "saned" user to manage scanbd :
cp integration/scanbd_dbus.conf /etc/dbus-1/system.d/

# Add the init script to manage the daemon :
cp integration/scanbd.debian /etc/init.d/scanbd

scanbd şimdi kuruldu ancak henüz yapılandırılmadı.

Ağdan tarama yapmak için aklı başında olanı yapılandırın #

Sane’nin yerel konfigürasyonu /etc/saned.d içindedir.

/etc/sane.d/dll.conf

# /etc/sane.d/dll.conf - Configuration file for the SANE dynamic backend loader
#
# Backends can also be enabled by configuration snippets under
# /etc/sane.d/dll.d directory -- packages providing backends should drop
# a config file similar to dll.conf in this directory, named after the package.
#

# The next line enables the network backend; comment it out if you don't need
# to use a remote SANE scanner over the network - see sane-net(5) and saned(8)

net

‘dll.conf’ yalnızca ’net’ arka ucunu içerir.

/etc/sane.d/net.conf :

# This is the net backend config file.

## net backend options
# Timeout for the initial connection to saned. This will prevent the backend
# from blocking for several minutes trying to connect to an unresponsive
# saned host (network outage, host down, ...). Value in seconds.
connect_timeout = 3

## saned hosts
# Each line names a host to attach to.
# If you list "localhost" then your backends can be accessed either
# directly or through the net backend.  Going through the net backend
# may be necessary to access devices that need special privileges.

localhost

Bu noktada, scanbm henüz yapılandırılmamış, ağdan hiçbir tarayıcıya ulaşılamıyor. Kurulumu test etmek için scanbd ile birlikte scanbm kurmamız gerekiyor.

Yerel tarayıcıyı yoklamak için scanbd’yi yapılandırın #

Yerel tarayıcıyı tanımlayın #

lsusb kullanarak, productId‘yi (1909 burada) tanımlayın:

pi:# lsusb
pi:# Bus 001 Device 005: ID 04a9:1909 Canon, Inc. CanoScan LiDE 110

Bu productId ile doğru yapılandırma dosyasını grep yapın (tarayıcı modelinize bağlıdır, benim için genesys.conf):

pi:# grep 1909 /etc/sane.d/*conf
pi:# /etc/sane.d/genesys.conf:usb 0x04a9 0x1909

Dosyayı scanbd yapılandırma klasörünüzün içine kopyalayın:

cp /etc/sane.d/genesys.conf /usr/local/etc/scanbd/
cp /etc/sane.d/dll.conf /usr/local/etc/scanbd/

/usr/local/etc/scanbd/dll.conf dosyasını düzenleyin ve “net"i doğru tarayıcı arka ucuyla değiştirin:

# /etc/sane.d/dll.conf - Configuration file for the SANE dynamic backend loader
#
# Backends can also be enabled by configuration snippets under
# /etc/sane.d/dll.d directory -- packages providing backends should drop
# a config file similar to dll.conf in this directory, named after the package.
#

# The next line enables the network backend; comment it out if you don't need
# to use a remote SANE scanner over the network - see sane-net(5) and saned(8)

genesys

Scanbd tarafından yerel tarayıcının bulunduğunu onaylayın #

Şimdi test etmeye hazırız:

scanbd -d7 -f

SANE_CONFIG_DIR env değişkenini değiştirmekten mümkün olduğunca kaçının. Bazı şeyleri bozabilir, varsayılan değer /usr/local/etc/scanbd/scanbd.conf ve init betiğinde zaten yapılandırılmıştır.

Buradaki en önemli şey “bulunan cihaz"dır.

“Bulunan cihaz” görüntülenmezse ne olur? #

Yapılandırmanızı tekrar kontrol etmelisiniz, özellikle bu belgenin tanımlama kısmını ve olası SANE_CONFIG_DIR env değişkeninin yanlış ayarlandığını yazarak temizleyin: “SANE_CONFIG_DIR ayarını kaldır”

** xinetd’yi ağda scanbm dinlemesi yapacak şekilde yapılandırın**

Aşağıdakileri içeren /etc/xinetd.d/sane-port oluşturun:

service sane-port
{
        port        = 6566
        socket_type = stream
        wait        = no
        user        = saned
        group       = scanner
        server      = /usr/local/sbin/scanbm
        server_args = scanbm -c /usr/local/etc/scanbd/scanbd.conf
        disable     = no
}

/!\ scanbm yolunu “hangi scanbm” ile iki kez kontrol edin

Ardından, xinetd‘yi yeniden başlatın ve scanbd‘yi başlatın:

service xinetd restart
/etc/init.d/scanbd start

scanimage -L şimdi tarayıcıyı göstermelidir.

Bu aşamadaki sorunların çoğu izinlerle ilgilidir. emin olun scanimage -L, sudo -u saned' yaparak saned olarak oturum açtığınızda çalışır -s.

Değilse, USB izinlerini kontrol edin: ls -al /dev/bus/usb/***

Tarayıcı düğmelerinden eylemleri tetikleyin #

Eylemler /usr/local/etc/scanbd/scanbd.conf içinde bulunur. Tara, kopyala, e-posta ve dosya olmak üzere 4 düğmem var.

Varsayılan yapılandırma dosyası, varsayılan olarak tüm eylemleri içermez, muhtemelen bloğu manuel olarak eklemek zorunda kalacak. Daha az veya daha fazla düğmeniz olabilir tarayıcı modelinize bağlı olarak.

Her eylem için komut dosyası seçeneği için özel bir yol belirleyeceğiz.

action scan {
    filter = "^scan.*"
    numerical-trigger {
            from-value = 1
            to-value   = 0
    }
    desc   = "Scan to file"
    # script must be an relative path starting from scriptdir (see above), 
    # or an absolute pathname. 
    # It must contain the path to the action script without arguments
    # Absolute path example: script = "/some/path/foo.script 
    script = "/home/pi/scan.sh"
}

scanbd.conf dosyasının sonunda başka herhangi bir varsayılan eylemi yorumlamayı unutmayın:

# devices 
# each device can have actions and functions, you can disable not relevant devices
#include(scanner.d/avision.conf)
#include(scanner.d/fujitsu.conf)
#include(scanner.d/hp.conf)
#include(scanner.d/pixma.conf)
#include(scanner.d/snapscan.conf)
#include(scanner.d/canon.conf)

Artık her eylemi işlemek için özel komut dosyanızı oluşturabilirsiniz:

/sys/class/leds/led0/trigger ile ilgili her satır, neler olup bittiğini izlemek için LED’i kontrol etmek içindir. Ne istersen yapabilirsin, cat /sys/class/leds/led0/trigger size farklı ışık düzenleri verir.

/home/pi/scan.sh

#!/bin/bash

# don't forget to create the folder
scan_dir=/home/pi/scanned-files
datetime=`date +%F_%H%M%S`

echo none >/sys/class/leds/led0/trigger

case $SCANBD_ACTION in
  scan)
    filename=file-$datetime
    logger -t "scanbd: $0" "$SCANBD_DEVICE $SCANBD_ACTION - scanning --resolution 150 --mode Color --depth 8 --format=tiff to $scan_dir/$filename.jpg"
    echo timer >/sys/class/leds/led0/trigger
    scanimage -d $SCANBD_DEVICE --resolution 150 --mode Color --depth 8 --format=tiff  --brightness 5 --contrast 20 | convert tiff:- -compress jpeg $scan_dir/$filename.pdf
    echo none >/sys/class/leds/led0/trigger
    logger -t "scanbd: $0" "Finished scanning"
    ;;
    
  email)
    logger -t "scanbd: $0" "Emailing $scan_dir/file-*pdf"
    echo heartbeat >/sys/class/leds/led0/trigger
    # here are the lines to send the file
    echo none >/sys/class/leds/led0/trigger
