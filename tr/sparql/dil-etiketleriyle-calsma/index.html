Bir hazır bilgiden dil etiketi alma #

RDF’deki değişmez değerler, dil etiketli dizeler olabilir. Bu değişmezlerin bir metin bölümünün yanı sıra bir dil etiketi vardır. Örneğin, “color”@en değişmezi, “color” metin bölümüne ve “en” dil etiketine sahiptir. Bir SPARQL sorgusunda, bir dil etiketine sahip bir değişmezin dilini almak için lang işlevini kullanın. Bir hazır bilgi dil etiketine sahip değilse, lang boş dizeyi döndürür, "".

Dil etiketlerini karşılaştırma #

SPARQL işlevi langMatches, SPARQL sorgularındaki RDF değişmezlerinin dil etiketlerini karşılaştırmak için kullanılabilir. Basit eşitlik operatörü, =, tam dize eşleşmelerini karşılaştırmak için kullanılabilir, ancak bölgesel değişkenleri doğru şekilde dikkate almaz. Örneğin, ?str için dört olası değer:

values ?str { "color"@en-US "color"@en "colour"@en "colour"@en-GB }

tümü İngilizce dil dizeleridir, ancak bunlardan ikisinin bölgesel özellikleri vardır. Bunun anlamı şudur ki

select ?str { 
  values ?str { "color"@en-US "color"@en "colour"@en "colour"@en-GB }
  filter (lang(?str) = 'en')
}

?str değerlerinden yalnızca ikisi dil etiketi olarak “en” içerdiğinden yalnızca iki sonuç döndürür. Yine de,

select ?str { 
  values ?str { "color"@en-US "color"@en "colour"@en "colour"@en-GB }
  filter langMatches(lang(?str), 'en')
}

dört değeri de döndürür.
