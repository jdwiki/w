Prim Algoritmasına Giriş #

Diyelim ki 8 evimiz var. Bu evler arasında telefon hattı kurmak istiyoruz. Evler arasındaki kenar, iki ev arasındaki çizgiyi belirleme maliyetini temsil eder.

Görevimiz, tüm evler birbirine bağlı olacak şekilde hatları kurmak ve tüm bağlantıyı kurmanın maliyeti minimumdur. Şimdi bunu nasıl bulacağız? Prim Algoritmasını kullanabiliriz.

Prim Algoritması, ağırlıklı bir yönsüz grafik için minimum yayılan ağaç bulan açgözlü bir algoritmadır. Bu, ağaçtaki tüm kenarların toplam ağırlığının en aza indirildiği her düğümü içeren bir ağaç oluşturan kenarların bir alt kümesini bulduğu anlamına gelir. Algoritma 1930 yılında Çek matematikçi Vojtěch Jarník tarafından geliştirildi ve daha sonra bilgisayar bilimcisi Robert Clay Prim tarafından yeniden keşfedildi ve yeniden yayınlandı. org/wiki/Robert_C._Prim) 1957’de ve Edsger Wybe Dijkstra 1959’da. DJP algoritması olarak da bilinir, ** Jarnik algoritması**, Prim-Jarnik algoritması veya Prim-Dijsktra algoritması.

Şimdi önce teknik terimlere bakalım. Yönlendirilmemiş bir grafiğin G bazı düğümlerini ve kenarlarını kullanarak S bir grafik oluşturursak, S, G grafiğinin alt grafiği olarak adlandırılır. Şimdi S, yalnızca ve ancak aşağıdaki durumlarda Kapsayan Ağaç olarak adlandırılacaktır:

  • G’nin tüm düğümlerini içerir.
  • Bu bir ağaç, yani döngü yok ve tüm düğümler bağlı.
  • Ağaçta (n-1) kenarlar vardır, burada n, G içindeki düğüm sayısıdır.

Bir grafiğin birçok Spanning Tree‘si olabilir. Ağırlıklı yönsüz bir grafiğin Minimum Yayılan Ağacı, kenarların ağırlığının toplamı minimum olacak şekilde bir ağaçtır. Şimdi minimum yayılma ağacını bulmak için Prim’in algoritmasını kullanacağız, bu, örnek grafiğimizde telefon hatlarını kurulum maliyeti minimum olacak şekilde nasıl kuracağımızdır.

İlk başta bir kaynak düğümü seçeceğiz. Diyelim ki node-1 bizim kaynağımız. Şimdi node-1‘den minimum maliyeti olan kenarı alt grafiğimize ekleyeceğiz. Burada alt grafikteki kenarları mavi rengiyle işaretliyoruz. Burada 1-5 istediğimiz kenardır.

Şimdi düğüm-1 ve düğüm-5 arasındaki tüm kenarları ele alıyoruz ve minimumu alıyoruz. 1-5 zaten işaretlendiğinden 1-2 alırız.

Bu sefer düğüm-1, düğüm-2 ve düğüm-5‘i ele alıyoruz ve 5-4 olan minimum kenarı alıyoruz.

Bir sonraki adım önemlidir. düğüm-1, düğüm-2, düğüm-5 ve düğüm-4’ten minimum kenar 2-4’tür. Ama onu seçersek, alt grafiğimizde bir döngü oluşturacaktır. Bunun nedeni, düğüm-2 ve düğüm-4‘ün zaten alt grafiğimizde olmasıdır. Bu yüzden 2-4 avantaj sağlamak bize fayda sağlamaz. Kenarları, alt grafiğimize yeni bir düğüm ekleyecek şekilde seçeceğiz. Bu yüzden kenarı 4-8 seçiyoruz.

Bu bizim istenen alt grafiğimizdir, bu bize minimum yayılan ağacı verir. Seçmediğimiz kenarları kaldırırsak, şunu elde ederiz:

Bu bizim minimum kapsayan ağacımızdır (MST). Yani telefon bağlantılarını kurmanın maliyeti: 4 + 2 + 5 + 11 + 9 + 2 + **1 ** = 34. Ve evler kümesi ve bağlantıları grafikte gösterilmiştir. Bir grafiğin birden fazla MST‘si olabilir. Seçtiğimiz kaynak düğümüne bağlıdır.

Algoritmanın sözde kodu aşağıda verilmiştir:

Procedure PrimsMST(Graph):     // here Graph is a non-empty connected weighted graph
Vnew[] = {x}                   // New subgraph Vnew with source node x
Enew[] = {}
while Vnew is not equal to V
    u -> a node from Vnew
    v -> a node that is not in Vnew such that edge u-v has the minimum cost
                               // if two nodes have same weight, pick any of them
    add v to Vnew
    add edge (u, v) to Enew
end while
Return Vnew and Enew

Karmaşıklık:

Yukarıdaki saf yaklaşımın zaman karmaşıklığı O(V²)‘dir. Bitişik matris kullanır. Öncelik kuyruğunu kullanarak karmaşıklığı azaltabiliriz. Vnew’e yeni bir düğüm eklediğimizde, komşu kenarlarını öncelik kuyruğuna ekleyebiliriz. Ardından, minimum ağırlıklı kenarı ondan çıkarın. O zaman karmaşıklık şöyle olacaktır: O(ElogE), burada E kenar sayısıdır. Yine, karmaşıklığı O(ElogV)‘ye düşürmek için bir İkili Yığın oluşturulabilir.

Priority Queue kullanan sözde kod aşağıda verilmiştir:

Procedure MSTPrim(Graph, source):
for each u in V
    key[u] := inf
    parent[u] := NULL
end for
key[source] := 0
Q = Priority_Queue()
Q = V
while Q is not empty
    u -> Q.pop
    for each v adjacent to i
        if v belongs to Q and Edge(u,v) < key[v]    // here Edge(u, v) represents
                                                    // cost of edge(u, v)
            parent[v] := u
            key[v] := Edge(u, v)
        end if
    end for
end while

Burada anahtar[], düğüm-v geçişinin minimum maliyetini saklar. parent[] üst düğümü depolamak için kullanılır. Ağacı dolaşmak ve yazdırmak için kullanışlıdır.

Java’da basit bir program aşağıdadır:

import java.util.*;

public class Graph
{
   private static int infinite = 9999999;
   int[][]  LinkCost;
   int      NNodes;
   Graph(int[][] mat)
   {
      int i, j;
      NNodes = mat.length;
      LinkCost = new int[NNodes][NNodes];
      for ( i=0; i < NNodes; i++)
      {
         for ( j=0; j < NNodes; j++)
         {
            LinkCost[i][j] = mat[i][j];
            if ( LinkCost[i][j] == 0 )
               LinkCost[i][j] = infinite;
         }
      }
      for ( i=0; i < NNodes; i++)
      {
         for ( j=0; j < NNodes; j++)
            if ( LinkCost[i][j] < infinite )
               System.out.print( " " + LinkCost[i][j] + " " );
            else
               System.out.print(" * " );
         System.out.println();
      }
   }
   public int unReached(boolean[] r)
   {
      boolean done = true;
      for ( int i = 0; i < r.length; i++ )
         if ( r[i] == false )
            return i;
      return -1;
   }
   public void Prim( )
   {
      int i, j, k, x, y;
      boolean[] Reached = new boolean[NNodes];
      int[] predNode = new int[NNodes];
      Reached[0] = true;
      for ( k = 1; k < NNodes; k++ )
      {
         Reached[k] = false;
      }
      predNode[0] = 0;
      printReachSet( Reached );
      for (k = 1; k < NNodes; k++)
      {
         x = y = 0;
         for ( i = 0; i < NNodes; i++ )
            for ( j = 0; j < NNodes; j++ )
            {
                if ( Reached[i] && !Reached[j] &&
                     LinkCost[i][j] < LinkCost[x][y] )
                {
                   x = i;
                   y = j;
                }
            }
         System.out.println("Min cost edge: (" +
                                + x + "," +
                                + y + ")" +
                                "cost = " + LinkCost[x][y]);
         predNode[y] = x;
         Reached[y] = true;
         printReachSet( Reached );
         System.out.println();
      }
      int[] a= predNode;
   for ( i = 0; i < NNodes; i++ )
          System.out.println( a[i] + " --> " + i );
   }
   void printReachSet(boolean[] Reached )
   {
      System.out.print("ReachSet = ");
      for (int i = 0; i < Reached.length; i++ )
         if ( Reached[i] )
           System.out.print( i + " ");
      //System.out.println();
   }
 public static void main(String[] args)
   {
      int[][] conn = {{0,3,0,2,0,0,0,0,4},  // 0
                      {3,0,0,0,0,0,0,4,0},  // 1
                      {0,0,0,6,0,1,0,2,0},  // 2
                      {2,0,6,0,1,0,0,0,0},  // 3
                      {0,0,0,1,0,0,0,0,8},  // 4
                      {0,0,1,0,0,0,8,0,0},  // 5
                      {0,0,0,0,0,8,0,0,0},  // 6
                      {0,4,2,0,0,0,0,0,0},  // 7
                      {4,0,0,0,8,0,0,0,0}   // 8
                     };
      Graph G = new Graph(conn);
      G.Prim();
   }
}
