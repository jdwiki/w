En Uzun Ortak Ardışık Açıklama #

Dinamik Programlamanın en önemli uygulamalarından biri, En Uzun Ortak Sırayı bulmaktır. Önce bazı temel terminolojileri tanımlayalım.

Sonraki:

Bir alt dizi, kalan öğelerin sırasını değiştirmeden bazı öğeleri silerek başka bir diziden türetilebilen bir dizidir. Diyelim ki bir ABC dizimiz var. Bu dizeden sıfır veya bir veya birden fazla karakter silersek, bu dizenin alt dizisini elde ederiz. Yani ABC dizesinin alt dizileri {“A”, “B”, “C”, “AB”, **“AC” olacaktır. **, “BC”, “ABC”, " “}. Tüm karakterleri kaldırsak bile, boş dize de bir alt dizi olacaktır. Dizideki her karakter için ardışıklığı bulmak için iki seçeneğimiz var - ya karakteri alırız ya da almayız. Yani dizinin uzunluğu n ise, o dizinin 2n alt dizisi vardır.

En Uzun Ortak Sıra:

Adından da anlaşılacağı gibi, iki dize arasındaki tüm ortak diziler arasında en uzun ortak dizi (LCS), maksimum uzunluğa sahip olanıdır. Örneğin: “HELLOM” ve “HMLD” arasındaki ortak alt diziler “H”, “HL”, “HM” vb.‘dir. Burada * “HLL”* uzunluğu 3 olan en uzun ortak dizidir.

Kaba Kuvvet Yöntemi:

backtracking kullanarak iki dizgenin tüm alt dizilerini üretebiliriz. Sonra ortak alt dizileri bulmak için bunları karşılaştırabiliriz. Sonra maksimum uzunluğa sahip olanı bulmamız gerekecek. n uzunluğundaki bir dizenin 2n altdizisi olduğunu daha önce görmüştük. n‘miz 20-25‘i geçerse sorunu çözmek yıllar alacaktı.

Dinamik Programlama Yöntemi:

Yöntemimize bir örnekle yaklaşalım. abcdaf ve acbcf olmak üzere iki dizimiz olduğunu varsayalım. Bunları s1 ve s2 ile gösterelim. Yani bu iki stringin en uzun ortak dizisi “abcf” olacaktır ve uzunluğu 4’tür. “abcf” oluşturmak için, s1‘de “da” ve s2‘de “c” göz ardı ettik. Bunu Dinamik Programlama kullanarak nasıl bulabiliriz?

Bir satırda s1 tüm karakterlerini ve sütunda s2 tüm karakterlerini içeren bir tablo (2B dizi) ile başlayacağız. Burada tablo 0-endekslidir ve karakterleri 1’den sonrasına koyarız. Her satır için tabloyu soldan sağa geçeceğiz. Tablomuz şöyle görünecek:

              0     1     2     3     4     5     6
     +-----+-----+-----+-----+-----+-----+-----+-----+
     | chʳ |     |  a  |  b  |  c  |  d  |  a  |  f  |
     +-----+-----+-----+-----+-----+-----+-----+-----+
  0  |     |     |     |     |     |     |     |     |
     +-----+-----+-----+-----+-----+-----+-----+-----+
  1  |  a  |     |     |     |     |     |     |     |
     +-----+-----+-----+-----+-----+-----+-----+-----+
  2  |  c  |     |     |     |     |     |     |     |
     +-----+-----+-----+-----+-----+-----+-----+-----+
  3  |  b  |     |     |     |     |     |     |     |
     +-----+-----+-----+-----+-----+-----+-----+-----+
  4  |  c  |     |     |     |     |     |     |     |
     +-----+-----+-----+-----+-----+-----+-----+-----+
  5  |  f  |     |     |     |     |     |     |     |
     +-----+-----+-----+-----+-----+-----+-----+-----+

Burada her satır ve sütun, o satırın ve sütunun karakterlerini alıp ondan önceki öneki eklersek, iki dize arasındaki en uzun ortak dizinin uzunluğunu temsil eder. Örneğin: Tablo[2][3], “ac” ve “abc” arasındaki en uzun ortak dizinin uzunluğunu temsil eder.

  1. sütun, s1‘in boş sırasını temsil eder. Benzer şekilde, 0. satır s2’nin boş sırasını temsil eder. Bir dizgenin boş bir alt dizisini alıp başka bir dizge ile eşleştirmeye çalışırsak, ikinci alt dizinin uzunluğu ne olursa olsun, ortak dizinin uzunluğu 0 olacaktır. Böylece 0’ıncı satırları ve 0’ıncı sütunları 0’larla doldurabiliriz. Alırız:

               0     1     2     3     4     5     6
  +-----+-----+-----+-----+-----+-----+-----+-----+
  | chʳ |     |  a  |  b  |  c  |  d  |  a  |  f  |
  +-----+-----+-----+-----+-----+-----+-----+-----+

    0 | | 0 | 0 | 0 | 0 | 0 | 0 | 0 | +—–+—–+—–+—–+—–+—–+—–+—–+ 1 | a | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 2 | c | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 3 | b | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 4 | c | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 5 | f | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ Hadi başlayalım. Tablo[1][1]‘u doldururken kendimize soruyoruz, eğer bir a dizgemiz ve başka bir a dizgimiz olsaydı ve başka hiçbir şey yoksa, en uzun ne olurdu? burada ortak sonuç? Buradaki LCS’nin uzunluğu 1 olacaktır. Şimdi Tablo[1][2]‘ye bakalım. ab dizimiz ve a dizimiz var. LCS’nin uzunluğu 1 olacaktır. Gördüğünüz gibi, sadece a ile abcd, abcda, abcdaf. Böylece tablomuz şöyle görünecek:

               0     1     2     3     4     5     6
  +-----+-----+-----+-----+-----+-----+-----+-----+
  | chʳ |     |  a  |  b  |  c  |  d  |  a  |  f  |
  +-----+-----+-----+-----+-----+-----+-----+-----+

    0 | | 0 | 0 | 0 | 0 | 0 | 0 | 0 | +—–+—–+—–+—–+—–+—–+—–+—–+ 1 | a | 0 | 1 | 1 | 1 | 1 | 1 | 1 | +—–+—–+—–+—–+—–+—–+—–+—–+ 2 | c | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 3 | b | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 4 | c | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 5 | f | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ Şimdi c içerecek olan 2. satır için. Tablo[2][1] için bir tarafta ac ve diğer tarafta a var. Yani LCS’nin uzunluğu 1’dir. Bu 1’i nereden aldık? İki alt dize arasındaki LCS a‘yı gösteren üstten. Yani demek istediğimiz, eğer s1[2] ve s2[1] aynı değilse, o zaman LCS’nin uzunluğu, *üstteki LCS’nin maksimum uzunluğu olacaktır. * veya solda. LCS’nin uzunluğunu en üstte almak, mevcut karakteri s2‘den almadığımızı gösterir. Benzer şekilde, LCS’nin uzunluğunu soldan almak, LCS’yi oluşturmak için mevcut karakteri s1‘den almadığımızı gösterir. Alırız:

               0     1     2     3     4     5     6
  +-----+-----+-----+-----+-----+-----+-----+-----+
  | chʳ |     |  a  |  b  |  c  |  d  |  a  |  f  |
  +-----+-----+-----+-----+-----+-----+-----+-----+

    0 | | 0 | 0 | 0 | 0 | 0 | 0 | 0 | +—–+—–+—–+—–+—–+—–+—–+—–+ 1 | a | 0 | 1 | 1 | 1 | 1 | 1 | 1 | +—–+—–+—–+—–+—–+—–+—–+—–+ 2 | c | 0 | 1 | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 3 | b | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 4 | c | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 5 | f | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ İlk formülümüz şu şekilde olacak:

    if s2[i] is not equal to s1[j] Table[i][j] = max(Table[i-1][j], Table[i][j-1] endif Devam edelim, Tablo[2][2] için ab ve ac dizgemiz var. c ve b aynı olmadığı için üst veya soldaki maksimumu buraya koyarız. Bu durumda yine 1. olur. Bundan sonra Tablo[2][3] için abc ve ac dizgemiz var. Bu sefer hem satır hem de sütunun geçerli değerleri aynıdır. Şimdi LCS’nin uzunluğu, şimdiye kadarki maksimum LCS uzunluğuna + 1’e eşit olacaktır. Şimdiye kadarki maksimum LCS uzunluğunu nasıl elde ederiz? ab ve a arasındaki en iyi eşleşmeyi temsil eden köşegen değeri kontrol ederiz. Bu durumdan, mevcut değerler için s1 ve s2‘ye aynı olan bir karakter daha ekledik. Dolayısıyla LCS’nin uzunluğu elbette artacaktır. 1 + 1 = 2‘yi Tablo[2][3]‘ya koyacağız. anladık

               0     1     2     3     4     5     6
  +-----+-----+-----+-----+-----+-----+-----+-----+
  | chʳ |     |  a  |  b  |  c  |  d  |  a  |  f  |
  +-----+-----+-----+-----+-----+-----+-----+-----+

    0 | | 0 | 0 | 0 | 0 | 0 | 0 | 0 | +—–+—–+—–+—–+—–+—–+—–+—–+ 1 | a | 0 | 1 | 1 | 1 | 1 | 1 | 1 | +—–+—–+—–+—–+—–+—–+—–+—–+ 2 | c | 0 | 1 | 1 | 2 | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 3 | b | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 4 | c | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ 5 | f | 0 | | | | | | | +—–+—–+—–+—–+—–+—–+—–+—–+ Yani ikinci formülümüz şöyle olacak:

    if s2[i] equals to s1[j] Table[i][j] = Table[i-1][j-1] + 1 endif Her iki durumu da tanımladık. Bu iki formülü kullanarak tüm tabloyu doldurabiliriz. Tabloyu doldurduktan sonra şöyle görünecek:

               0     1     2     3     4     5     6
  +-----+-----+-----+-----+-----+-----+-----+-----+
  | chʳ |     |  a  |  b  |  c  |  d  |  a  |  f  |
  +-----+-----+-----+-----+-----+-----+-----+-----+

    0 | | 0 | 0 | 0 | 0 | 0 | 0 | 0 | +—–+—–+—–+—–+—–+—–+—–+—–+ 1 | a | 0 | 1 | 1 | 1 | 1 | 1 | 1 | +—–+—–+—–+—–+—–+—–+—–+—–+ 2 | c | 0 | 1 | 1 | 2 | 2 | 2 | 2 | +—–+—–+—–+—–+—–+—–+—–+—–+ 3 | b | 0 | 1 | 2 | 2 | 2 | 2 | 2 | +—–+—–+—–+—–+—–+—–+—–+—–+ 4 | c | 0 | 1 | 2 | 3 | 3 | 3 | 3 | +—–+—–+—–+—–+—–+—–+—–+—–+ 5 | f | 0 | 1 | 2 | 3 | 3 | 3 | 4 | +—–+—–+—–+—–+—–+—–+—–+—–+ s1 ile s2 arasındaki LCS uzunluğu Tablo[5][6] = 4 olacaktır. Burada 5 ve 6 sırasıyla s2 ve s1 uzunluklarıdır. Sözde kodumuz şöyle olacaktır:

    Procedure LCSlength(s1, s2): Table[0][0] = 0 for i from 1 to s1.length Table[0][i] = 0 endfor for i from 1 to s2.length Table[i][0] = 0 endfor for i from 1 to s2.length for j from 1 to s1.length if s2[i] equals to s1[j] Table[i][j] = Table[i-1][j-1] + 1 else Table[i][j] = max(Table[i-1][j], Table[i][j-1]) endif endfor endfor Return Table[s2.length][s1.length] Bu algoritmanın zaman karmaşıklığı şu şekildedir: O(mn) burada m ve n her bir dizinin uzunluğunu belirtir.

En uzun ortak diziyi nasıl buluruz? Sağ alt köşeden başlayacağız. Değerin nereden geldiğini kontrol edeceğiz. Değer köşegenden geliyorsa, yani Tablo[i-1][j-1], Tablo[i][j] - 1’e eşitse, ya s2’ye basarız [i] veya s1[j] (ikisi de aynıdır) ve çapraz olarak hareket edin. Değer üstten geliyorsa, yani Tablo[i-1][j] Tablo[i][j]‘ye eşitse en üste geçiyoruz. Değer soldan geliyorsa, yani Tablo[i][j-1], Tablo[i][j]‘ye eşitse sola doğru hareket ederiz. En soldaki veya en üstteki sütuna ulaştığımızda aramamız sona eriyor. Ardından değerleri yığından çıkarıyoruz ve yazdırıyoruz. Sözde kod:

Procedure PrintLCS(LCSlength, s1, s2)
temp := LCSlength
S = stack()
i := s2.length
j := s1.length
while i is not equal to 0 and j is not equal to 0
     if Table[i-1][j-1] == Table[i][j] - 1 and s1[j]==s2[i]
        S.push(s1[j])   //or S.push(s2[i])
        i := i - 1
        j := j - 1
    else if Table[i-1][j] == Table[i][j]
        i := i-1
    else
        j := j-1
    endif
endwhile
while S is not empty
    print(S.pop)
endwhile

Dikkat edilmesi gereken nokta: Tablo[i-1][j] ve Tablo[i][j-1], Tablo[i][j] ve *‘a eşitse *Tablo[i-1][j-1]**, Tablo[i][j] - 1’e eşit değildir, o an için iki LCS olabilir. Bu sözde kod bu durumu dikkate almaz. Birden fazla LCS bulmak için bunu özyinelemeli olarak çözmeniz gerekecek.

Bu algoritmanın zaman karmaşıklığı: O(max(m, n)).
