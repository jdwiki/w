Dinamik programlama yaygın olarak kullanılan bir kavramdır ve sıklıkla optimizasyon için kullanılır. Karmaşık bir problemi, genellikle aşağıdan yukarıya yaklaşımla özyinelemeli bir şekilde daha basit alt problemlere bölerek basitleştirmeyi ifade eder. Dinamik programlamanın uygulanabilir olması için bir problemin sahip olması gereken iki temel özellik vardır “Optimal altyapı” ve “Örtüşen alt problemler”. Optimizasyonunu sağlamak için Dynamics programlama, Ezberleme adlı bir kavram kullanır.

Dinamik Programlama, Brute Force’ta yapılan bir iyileştirmedir, Brute Force’tan Dinamik Programlama çözümünün nasıl elde edilebileceğini anlamak için bu örneğe bakın.

Bir Dinamik Programlama Çözümünün 2 ana gereksinimi vardır:

  1. Örtüşen Sorunlar

  2. Optimum Alt Yapı

Örtüşen Altproblemler, sorunun daha küçük sürümlerinin sonuçlarının, orijinal soruna çözüme ulaşmak için birden çok kez yeniden kullanılması anlamına gelir.

Optimal Altyapı, alt problemlerinden bir problem hesaplama yöntemi olduğu anlamına gelir.

Bir Dinamik Programlama Çözümünün 2 ana bileşeni vardır, Devlet ve Geçiş

Devlet, orijinal problemin bir alt problemini ifade eder.

Geçiş, bir problemi alt problemlerine dayalı olarak çözme yöntemidir.

Dinamik Programlama Çözümü tarafından geçen süre Hayır. Devletlerin * Geçiş Süresi. Bu nedenle, bir çözümün ‘N^2’ durumları varsa ve geçiş ‘O(N)’ ise, çözüm kabaca ‘O(N^3)’ zaman alacaktır.

Mesafeyi Düzenle #

Problem ifadesi, eğer bize iki string str1 ve str2 verilmişse, str1’de str2’ye dönüştürüldüğü minimum kaç işlem gerçekleştirilebileceği gibidir.

Java’da uygulama

public class EditDistance {

public static void main(String[] args) {
    // TODO Auto-generated method stub
    String str1 = "march";
    String str2 = "cart";
    
    EditDistance ed = new EditDistance();
    System.out.println(ed.getMinConversions(str1, str2));
}

public int getMinConversions(String str1, String str2){
    int dp[][] = new int[str1.length()+1][str2.length()+1];
    for(int i=0;i<=str1.length();i++){
        for(int j=0;j<=str2.length();j++){
            if(i==0)
                dp[i][j] = j;
            else if(j==0)
                dp[i][j] = i;
            else if(str1.charAt(i-1) == str2.charAt(j-1))
                dp[i][j] = dp[i-1][j-1];
            else{
                dp[i][j] = 1 + Math.min(dp[i-1][j], Math.min(dp[i][j-1], dp[i-1][j-1]));
            }
        }
    }
    return dp[str1.length()][str2.length()];
}

}

Ağırlıklı İş Planlama Algoritması #

Ağırlıklı İş Çizelgeleme Algoritması, Ağırlıklı Etkinlik Seçim Algoritması olarak da ifade edilebilir.

Sorun şu ki, belirli işlerin başlangıç ​​ve bitiş zamanları verildiğinde ve işi bitirdiğinizde elde ettiğiniz kârla, iki iş paralel olarak yürütülemezken elde edebileceğiniz maksimum kâr nedir?

Bu, Açgözlü Algoritma kullanan Etkinlik Seçimi’ne benziyor, ancak ek bir bükülme var. Yani biten iş sayısını maksimize etmek yerine maksimum karı elde etmeye odaklanıyoruz. Burada yapılan işlerin sayısı önemli değil.

Bir örneğe bakalım:

En Uzun Ortak Sıra #

Bize iki dizi verilirse, her ikisinde de bulunan en uzun ortak alt diziyi bulmamız gerekir.

Örnek

“ABCDGH” ve “AEDFHR” giriş Dizileri için LCS, uzunluk 3 “ADH” dir.

“AGGTAB” ve “GXTXAYB” giriş Dizileri için LCS, 4 uzunluğunda “GTAB” dır.

Java’da uygulama

public class LCS {

    public static void main(String[] args) {
        // TODO Auto-generated method stub
        String str1 = "AGGTAB";
        String str2 = "GXTXAYB";
        LCS obj = new LCS();
        System.out.println(obj.lcs(str1, str2, str1.length(), str2.length()));
        System.out.println(obj.lcs2(str1, str2));
    }
    
    //Recursive function
    public int lcs(String str1, String str2, int m, int n){
        if(m==0 || n==0)
            return 0;
        if(str1.charAt(m-1) == str2.charAt(n-1))
            return 1 + lcs(str1, str2, m-1, n-1);
        else
            return Math.max(lcs(str1, str2, m-1, n), lcs(str1, str2, m, n-1));
    }
    
    //Iterative function
    public int lcs2(String str1, String str2){
        int lcs[][] = new int[str1.length()+1][str2.length()+1];
        
        for(int i=0;i<=str1.length();i++){
            for(int j=0;j<=str2.length();j++){
                if(i==0 || j== 0){
                    lcs[i][j] = 0;
                }
                else if(str1.charAt(i-1) == str2.charAt(j-1)){
                    lcs[i][j] = 1 + lcs[i-1][j-1]; 
                }else{
                    lcs[i][j] = Math.max(lcs[i-1][j], lcs[i][j-1]);
                }
            }
        }
        
        return lcs[str1.length()][str2.length()];
    }

}

public int fib(int n){
        int f[] = new int[n+1];
        f[0]=0;f[1]=1;
        for(int i=2;i<=n;i++){
            f[i]=f[i-1]+f[i-2];

Girdi : X = “abcdxyz”, y = “xyzabcd” Çıkış : 4

En uzun ortak alt dize “abcdez"dir ve uzunluk 6.

Java’da uygulama

public int getLongestCommonSubstring(String str1,String str2){
        int arr[][] = new int[str2.length()+1][str1.length()+1];
        int max = Integer.MIN_VALUE;
        for(int i=1;i<=str2.length();i++){
            for(int j=1;j<=str1.length();j++){
                if(str1.charAt(j-1) == str2.charAt(i-1)){
                    arr[i][j] = arr[i-1][j-1]+1;
                    if(arr[i][j]>max)
                        max = arr[i][j];
                }
                else
                    arr[i][j] = 0;
            }
        }
        return max;
    }

Sırt Çantası Sorunu #

0-1 Sırt Çantası

Sırt Çantası Problemi, her biri bir ağırlık, değer ve tam olarak 1 kopya olan bir dizi öğe verildiğinde, toplam ağırlığın veya belirli bir limite eşittir ve toplam değer mümkün olduğu kadar büyüktür.

C++ Örneği: #

Uygulama:

int knapsack(vector<int> &value, vector<int> &weight, int N, int C){
    int dp[C+1];                               
    for (int i = 1; i <= C; ++i){
        dp[i] = -100000000;                   
    }
    dp[0] = 0;
    for (int i = 0; i < N; ++i){
        for (int j = C; j >= weight[i]; --j){
            dp[j] = max(dp[j],dp[j-weight[i]]+value[i]);
        }
    }
    return dp[C];
}

Ölçek:

3 5
5 2
2 1
3 2

Çıktı:

3

Bu, (2,1) ve (3,2) seçilerek elde edilen maksimum değerin 3 olduğu anlamına gelir.

Sınırsız Sırt Çantası

Sınırsız Sırt Çantası Problemi, her biri bir ağırlık, bir değer ve sonsuz kopya içeren bir dizi öğenin, toplam ağırlığın eşit veya daha az olması için bir koleksiyona dahil edilecek her öğenin sayısını belirleyen bir problemdir. belirli bir sınıra kadar ve toplam değer mümkün olduğunca büyüktür.

Python(2.7.11) Örnek: #

Uygulama:

def unbounded_knapsack(w, v, c): # weight, value and capactiy
    m = [0]
    for r in range(1, c+1):
        val = m[r-1]
        for i, wi in enumerate(w):
            if wi > r:
                continue
            val = max(val, v[i] + m[r-wi])
        m.append(val)
    return m[c] # return the maximum value can be achieved

Bu uygulamanın karmaşıklığı, n’nin öğe sayısı olan “O(nC)” dir.

Ölçek:

w = [2, 3, 4, 5, 6]
v = [2, 4, 6, 8, 9]

print unbounded_knapsack(w, v, 13)

Çıktı:

20

Bu, (5, 8), (5, 8) ve (3, 4) seçilerek elde edilebilecek maksimum değerin 20 olduğu anlamına gelir.
