En Uzun Artan Müteakip Temel Bilgiler #

En Uzun Artan Sıra sorunu, altdizinin öğelerinin en düşükten en yükseğe sıralandığı giriş dizisinden altdiziyi bulmaktır. Tüm müteakip diziler bitişik veya benzersiz değildir.

En Uzun Artan Sıranın Uygulanması:

En Uzun Artan Sıra, En Uzun Ortak Sıra gibi algoritmalar Git vb. sürüm kontrol sistemlerinde kullanılır.

Algoritmanın basit biçimi:

  1. Her iki belgede de ortak olan benzersiz satırları bulun.
  2. Bu tür tüm satırları ilk belgeden alın ve ikinci belgedeki görünümlerine göre sıralayın.
  3. Ortaya çıkan dizinin LIS’sini hesaplayın (bir [Sabır Sıralaması2 yaparak), en uzun eşleşen satır dizisini, iki belgenin satırları arasındaki bir yazışmayı elde edin.
  4. Halihazırda eşleşenler arasındaki her satır aralığında algoritmayı yineleyin.

Şimdi LCS probleminin daha basit bir örneğini ele alalım. Burada girdi, a1,a2,...,an. farklı tamsayılardan oluşan yalnızca bir dizidir ve biz de onun içindeki en uzun artan alt diziyi bulmak istiyoruz. Örneğin, girdi 7,3,8,4,2,6 ise en uzun artan dizi 3,4,6 olur.

En kolay yaklaşım, girdi öğelerini artan düzende sıralamak ve orijinal ve sıralanmış dizilere LCS algoritmasını uygulamaktır. Ancak, elde edilen diziye bakarsanız, birçok değerin aynı olduğunu ve dizinin çok tekrarlı göründüğünü fark edeceksiniz. Bu, LIS (en uzun artan alt dizi) probleminin sadece tek boyutlu dizi kullanılarak dinamik programlama algoritması ile yapılabileceğini göstermektedir.

Sahte Kod:

  1. Hesaplamak istediğimiz bir dizi değeri tanımlayın.
    For 1 <= i <= n, let A(i) be the length of a longest increasing sequence of input. Note that the length we are ultimately interested in is max{A(i)|1 ≤ i ≤ n}.
  2. Bir yineleme verin.
    For 1 <= i <= n, A(i) = 1 + max{A(j)|1 ≤ j < i and input(j) < input(i)}.
  3. A’nın değerlerini hesaplayın.
  4. En uygun çözümü bulun.

Aşağıdaki program, optimal bir çözümü hesaplamak için A’yı kullanır. İlk kısım, A(m) girdinin optimal artan alt dizisinin uzunluğu olacak şekilde bir m değerini hesaplar. İkinci kısım, optimal bir artan ardışıklığı hesaplar, ancak kolaylık olması için bunu ters sırada yazdırırız. Bu program O(n) zamanında çalışır, dolayısıyla tüm algoritma O(n^2) zamanında çalışır.

Bölüm 1:

m ← 1 
for i : 2..n 
    if A(i) > A(m) then 
        m ← i 
    end if 
end for

Bölüm 2:

put a
while A(m) > 1 do 
    i ← m−1 
    while not(ai < am and A(i) = A(m)−1) do 
        i ← i−1 
    end while 
    m ← i 
    put a
 end while

Öyinelemeli Çözüm:

Yaklaşım 1:

LIS(A[1..n]):
    if (n = 0) then return 0
    m = LIS(A[1..(n − 1)])
    B is subsequence of A[1..(n − 1)] with only elements less than a[n]
    (* let h be size of B, h ≤ n-1 *)
    m = max(m, 1 + LIS(B[1..h]))
    Output m

Yaklaşım 1’deki zaman karmaşıklığı : O(n*2^n)

Yaklaşım 2:

LIS(A[1..n], x):
    if (n = 0) then return 0
    m = LIS(A[1..(n − 1)], x)
    if (A[n] < x) then
        m = max(m, 1 + LIS(A[1..(n − 1)], A[n]))
    Output m

MAIN(A[1..n]):
    return LIS(A[1..n], ∞)

Yaklaşım 2’de Zaman Karmaşıklığı: O(n^2)

Yaklaşım 3:

LIS(A[1..n]):
    if (n = 0) return 0
    m = 1
    for i = 1 to n − 1 do
        if (A[i] < A[n]) then
            m = max(m, 1 + LIS(A[1..i]))
    return m

MAIN(A[1..n]):
    return LIS(A[1..i])

Yaklaşımda Zaman Karmaşıklığı 3: O(n^2)

Yinelemeli Algoritma:

Değerleri aşağıdan yukarıya yinelemeli olarak hesaplar.

LIS(A[1..n]):
    Array L[1..n]
    (* L[i] = value of LIS ending(A[1..i]) *)
    for i = 1 to n do
        L[i] = 1
        for j = 1 to i − 1 do
            if (A[j] < A[i]) do
                L[i] = max(L[i], 1 + L[j])
    return L

MAIN(A[1..n]):
    L = LIS(A[1..n])
        return the maximum value in L

Yinelemeli yaklaşımda zaman karmaşıklığı: O(n^2)

Yardımcı Boşluk: O(n)

Girdi olarak {0, 8, 4, 12, 2, 10, 6, 14, 1, 9, 5, 13, 3, 11, 7, 15} alalım. Dolayısıyla, verilen girdi için En Uzun Artan Sıra {0, 2, 6, 9, 11, 15}‘dir.

C# Uygulaması #

public class LongestIncreasingSubsequence
{
    private static int Lis(int[] input, int n)
    {
        int[] lis = new int[n];
        int max = 0;
        for(int i = 0; i < n; i++)
        {
            lis[i] = 1;
        }
        for (int i = 1; i < n; i++)
        {
            for (int j = 0; j < i; j++)
            {
                if (input[i] > input[j] && lis[i] < lis[j] + 1)
                    lis[i] = lis[j] + 1;
            }
        }
        for (int i = 0; i < n; i++)
        {
            if (max < lis[i])
                max = lis[i];
        }
        return max;
    }

    public static int Main(int[] input)
    {
        int n = input.Length;
        return Lis(input, n);
    }
}
