DFT’nin Gerçek ve Kompleks formu (Discrete Fourier Ttransforms), herhangi bir ayrık ve periyodik sinyal için frekans analizi veya sentezi gerçekleştirmek için kullanılabilir. FFT (Fast Fourier Transform), modern CPU’larda hızlı bir şekilde gerçekleştirilebilen DFT’nin bir uygulamasıdır.

Radix 2 FFT #

FFT’yi hesaplamak için en basit ve belki de en iyi bilinen yöntem, Radix-2 Zamanda Desimasyon algoritmasıdır. Radix-2 FFT, bir N nokta zaman alanı sinyalini, her biri tek bir noktadan oluşan N zaman alanı sinyaline ayrıştırarak çalışır.

Sinyal ayrıştırma veya ‘zaman içinde kırım’, zaman alanı verileri dizisi için bitlerin ters çevrilmesiyle elde edilir. Böylece, on altı noktalı bir sinyal için, örnek 1 (İkili 0001), örnek 8 (1000) ile değiştirilir, örnek 2 (0010), 4 (0100) ile değiştirilir vb. Bit ters tekniği kullanılarak örnek değiş tokuşu basitçe yazılımda gerçekleştirilebilir, ancak Radix 2 FFT’nin kullanımını N = 2^M uzunluğundaki sinyallerle sınırlar.

Zaman alanındaki 1 noktalı bir sinyalin değeri, frekans alanındaki değerine eşittir, dolayısıyla bu ayrıştırılmış tek zaman alanlı nokta dizisi, bir frekans alanı noktaları dizisi olmak için dönüşüm gerektirmez. N tek noktaları; bununla birlikte, bir N-noktası frekans spektrumunda yeniden yapılandırılması gerekir. Tüm frekans spektrumunun optimal rekonstrüksiyonu kelebek hesaplamaları kullanılarak gerçekleştirilir. Radix-2 FFT’deki her yeniden oluşturma aşaması, benzer bir üstel ağırlıklandırma işlevi seti olan Wn^R’yi kullanarak bir dizi iki noktalı kelebek gerçekleştirir.

FFT, Wn^R’nin periyodikliğinden yararlanarak Ayrık Fourier Dönüşümü’ndeki gereksiz hesaplamaları kaldırır. X[K] veren kelebek hesaplamalarının log2(N) aşamalarında spektral rekonstrüksiyon tamamlanır; dikdörtgen biçiminde gerçek ve hayali frekans alanı verileri. Büyüklük ve faza (kutupsal koordinatlar) dönüştürmek için mutlak değer, √(Re2 + Im2) ve argüman, tan-1(Im/Re) bulunması gerekir.

Sekiz noktalı Radix 2 FFT için tam kelebek akış şeması aşağıda gösterilmiştir. Giriş sinyallerinin daha önce ana hatlarıyla belirtilen zamandaki azalma prosedürüne göre yeniden sıralandığına dikkat edin.

FFT tipik olarak karmaşık girdiler üzerinde çalışır ve karmaşık bir çıktı üretir. Gerçek sinyaller için, sanal kısım sıfıra ve gerçek kısım x[n] giriş sinyaline ayarlanabilir, ancak sadece gerçek verilerin dönüştürülmesini içeren birçok optimizasyon mümkündür. Yeniden oluşturma boyunca kullanılan Wn^R değerleri, üstel ağırlıklandırma denklemi kullanılarak belirlenebilir.

R değeri (üssel ağırlıklandırma gücü), spektral yeniden yapılandırmadaki mevcut aşamayı ve belirli bir kelebek içindeki mevcut hesaplamayı belirler.

Kod Örneği (C/C++)

Radix 2 FFT’yi hesaplamak için bir C/C++ kod örneği aşağıda bulunabilir. Bu, N’nin 2’nin kuvveti olduğu herhangi bir N boyutu için çalışan basit bir uygulamadır. En hızlı FFTw uygulamasından yaklaşık 3 kat daha yavaştır, ancak yine de gelecekteki optimizasyon veya bu algoritmanın nasıl çalıştığını öğrenmek için çok iyi bir temeldir.

#include <math.h>

#define PI       3.1415926535897932384626433832795    // PI for sine/cos calculations
#define TWOPI    6.283185307179586476925286766559     // 2*PI for sine/cos calculations
#define Deg2Rad  0.017453292519943295769236907684886  // Degrees to Radians factor
#define Rad2Deg  57.295779513082320876798154814105    // Radians to Degrees factor
#define log10_2  0.30102999566398119521373889472449   // Log10 of 2 
#define log10_2_INV 3.3219280948873623478703194294948 // 1/Log10(2)

// complex variable structure (double precision)
struct complex
{
public:
    double  Re, Im;        // Not so complicated after all
};

// Returns true if N is a power of 2
bool isPwrTwo(int N, int *M)
{
    *M = (int)ceil(log10((double)N) * log10_2_INV);// M is number of stages to perform. 2^M = N
    int NN = (int)pow(2.0, *M);
    if ((NN != N) || (NN == 0)) // Check N is a power of 2. 
        return false;

    return true;
}

void rad2FFT(int N, complex *x, complex *DFT)
{
    int M = 0;

    // Check if power of two. If not, exit        
    if (!isPwrTwo(N, &M))
        throw "Rad2FFT(): N must be a power of 2 for Radix FFT";

    // Integer Variables

    int BSep;                  // BSep is memory spacing between butterflies
    int BWidth;                // BWidth is memory spacing of opposite ends of the butterfly
    int P;                     // P is number of similar Wn's to be used in that stage
    int j;                     // j is used in a loop to perform all calculations in each stage
    int stage = 1;             // stage is the stage number of the FFT. There are M stages in total (1 to M).
    int HiIndex;               // HiIndex is the index of the DFT array for the top value of each butterfly calc 
    unsigned int iaddr;        // bitmask for bit reversal 
    int ii;                    // Integer bitfield for bit reversal (Decimation in Time)
    int MM1 = M - 1;

    unsigned int i;
    int l;
    unsigned int nMax = (unsigned int)N;

    // Double Precision Variables
    double TwoPi_N = TWOPI / (double)N;    // constant to save computational time.  = 2*PI / N
    double TwoPi_NP;

    // complex Variables (See 'struct complex')
    complex WN;               // Wn is the exponential weighting function in the form a + jb
    complex TEMP;             // TEMP is used to save computation in the butterfly calc
    complex *pDFT = DFT;      // Pointer to first elements in DFT array
    complex *pLo;             // Pointer for lo / hi value of butterfly calcs
    complex *pHi;
    complex *pX;              // Pointer to x[n]


    // Decimation In Time - x[n] sample sorting
    for (i = 0; i < nMax; i++, DFT++)
    {
        pX = x + i;             // Calculate current x[n] from base address *x and index i.
        ii = 0;                 // Reset new address for DFT[n]
        iaddr = i;              // Copy i for manipulations
        for (l = 0; l < M; l++) // Bit reverse i and store in ii...
        {
            if (iaddr & 0x01)     // Detemine least significant bit
                ii += (1 << (MM1 - l));    // Increment ii by 2^(M-1-l) if lsb was 1
            iaddr >>= 1;                // right shift iaddr to test next bit. Use logical operations for speed increase
            if (!iaddr)
                break;
        }
        DFT = pDFT + ii;        // Calculate current DFT[n] from base address *pDFT and bit reversed index ii    
        DFT->Re = pX->Re;       // Update the complex array with address sorted time domain signal x[n]
        DFT->Im = pX->Im;       // NB: Imaginary is always zero
    }

    // FFT Computation by butterfly calculation
    for (stage = 1; stage <= M; stage++) // Loop for M stages, where 2^M = N
    {
        BSep = (int)(pow(2, stage)); // Separation between butterflies = 2^stage
        P = N / BSep;             // Similar Wn's in this stage = N/Bsep
        BWidth = BSep / 2;     // Butterfly width (spacing between opposite points) = Separation / 2.

        TwoPi_NP = TwoPi_N*P;

        for (j = 0; j < BWidth; j++) // Loop for j calculations per butterfly
        {
            if (j != 0)              // Save on calculation if R = 0, as WN^0 = (1 + j0)
            {
                //WN.Re = cos(TwoPi_NP*j)
                WN.Re = cos(TwoPi_N*P*j);     // Calculate Wn (Real and Imaginary)
                WN.Im = -sin(TwoPi_N*P*j);
            }

            for (HiIndex = j; HiIndex < N; HiIndex += BSep) // Loop for HiIndex Step BSep butterflies per stage
            {
                pHi = pDFT + HiIndex;                  // Point to higher value
                pLo = pHi + BWidth;                    // Point to lower value (Note VC++ adjusts for spacing between elements)

                if (j != 0)                            // If exponential power is not zero...
                {
                    //CMult(pLo, &WN, &TEMP);          // Perform complex multiplication of Lovalue with Wn
                    TEMP.Re = (pLo->Re * WN.Re) - (pLo->Im * WN.Im);
                    TEMP.Im = (pLo->Re * WN.Im) + (pLo->Im * WN.Re);

                    //CSub (pHi, &TEMP, pLo);
                    pLo->Re = pHi->Re - TEMP.Re;       // Find new Lovalue (complex subtraction)
                    pLo->Im = pHi->Im - TEMP.Im;

                    //CAdd (pHi, &TEMP, pHi);          // Find new Hivalue (complex addition)
                    pHi->Re = (pHi->Re + TEMP.Re);
                    pHi->Im = (pHi->Im + TEMP.Im);
                }
                else
                {
                    TEMP.Re = pLo->Re;
                    TEMP.Im = pLo->Im;

                    //CSub (pHi, &TEMP, pLo);
                    pLo->Re = pHi->Re - TEMP.Re;       // Find new Lovalue (complex subtraction)
                    pLo->Im = pHi->Im - TEMP.Im;

                    //CAdd (pHi, &TEMP, pHi);          // Find new Hivalue (complex addition)
                    pHi->Re = (pHi->Re + TEMP.Re);
                    pHi->Im = (pHi->Im + TEMP.Im);
                }
            }
        }
    }


    pLo = 0;    // Null all pointers
    pHi = 0;
    pDFT = 0;
    DFT = 0;
    pX = 0;
}

Radix 2 Ters FFT #

Fourier Dönüşümünün güçlü dualitesi nedeniyle, bir ileri dönüşümün çıktısını ayarlamak, ters FFT’yi üretebilir. Frekans alanındaki veriler, aşağıdaki yöntemle zaman alanına dönüştürülebilir:

  1. Tüm K örnekleri için sanal bileşeni ters çevirerek frekans alanı verilerinin karmaşık eşleniğini bulun.
  2. Konjuge frekans alanı verileri üzerinde ileri FFT’yi gerçekleştirin.
  3. Gerçek zaman alanı değerini vermek için bu FFT’nin sonucunun her çıktısını N’ye bölün.
  4. n’nin tüm örnekleri için zaman alanı verilerinin sanal bileşenini ters çevirerek çıktının karmaşık eşleniğini bulun.

Not: hem frekans hem de zaman alanı verileri karmaşık değişkenlerdir. Tipik olarak, bir ters FFT’yi izleyen zaman alanı sinyalinin hayali bileşeni ya sıfırdır ya da yuvarlama hatası olarak yok sayılır. Değişkenlerin kesinliğini 32-bit floattan 64-bit double’a veya 128-bit long double’a yükseltmek, birkaç ardışık FFT işlemi tarafından üretilen yuvarlama hatalarını önemli ölçüde azaltır.

Kod Örneği (C/C++)

#include <math.h>

#define PI       3.1415926535897932384626433832795    // PI for sine/cos calculations
#define TWOPI    6.283185307179586476925286766559     // 2*PI for sine/cos calculations
#define Deg2Rad  0.017453292519943295769236907684886  // Degrees to Radians factor
#define Rad2Deg  57.295779513082320876798154814105    // Radians to Degrees factor
#define log10_2  0.30102999566398119521373889472449   // Log10 of 2 
#define log10_2_INV 3.3219280948873623478703194294948 // 1/Log10(2)

// complex variable structure (double precision)
struct complex
{
public:
    double  Re, Im;        // Not so complicated after all
}; 

void rad2InverseFFT(int N, complex *x, complex *DFT)
{
    // M is number of stages to perform. 2^M = N
    double Mx = (log10((double)N) / log10((double)2));
    int a = (int)(ceil(pow(2.0, Mx)));
    int status = 0;
    if (a != N) // Check N is a power of 2
    {
        x = 0;
        DFT = 0;
        throw "rad2InverseFFT(): N must be a power of 2 for Radix 2 Inverse FFT";
    }

    complex *pDFT = DFT;        // Reset vector for DFT pointers
    complex *pX = x;            // Reset vector for x[n] pointer
    double NN = 1 / (double)N;  // Scaling factor for the inverse FFT        

    for (int i = 0; i < N; i++, DFT++)
        DFT->Im *= -1;          // Find the complex conjugate of the Frequency Spectrum

    DFT = pDFT;                 // Reset Freq Domain Pointer
    rad2FFT(N, DFT, x); // Calculate the forward FFT with variables switched (time & freq)

    int i;
    complex* x;
    for ( i = 0, x = pX; i < N; i++, x++){
        x->Re *= NN;    // Divide time domain by N for correct amplitude scaling
        x->Im *= -1;    // Change the sign of ImX
    }    
}
