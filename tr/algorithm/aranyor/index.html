Ikili arama #

Giriiş #

İkili Arama, bir Böl ve Yönet arama algoritmasıdır. Arama uzayındaki bir elemanın yerini bulmak için O(log n) zamanını kullanır, burada ’n’ arama uzayının boyutudur.

İkili Arama, hedef değeri arama alanının orta değeriyle karşılaştırdıktan sonra her yinelemede arama alanını yarıya indirerek çalışır.

Binary Search’ü kullanmak için, arama alanı bir şekilde sıralanmış (sıralanmış) olmalıdır. Yinelenen girişler (karşılaştırma işlevine göre eşit olarak karşılaştırılanlar), İkili Arama özelliğini ihlal etmeseler de ayırt edilemez.

Geleneksel olarak, karşılaştırma işlevi olarak less (<) kullanırız. a < b ise, true değerini döndürür. a, b’den küçük değilse ve b, a’dan küçük değilse, a ve b eşittir.

Örnek Soru #

Sen bir ekonomistsin, ama oldukça kötü birisin. Size pirinç için denge fiyatını (yani arz = talep olduğu fiyat) bulma görevi verildi.

Unutmayın, bir fiyat ne kadar yüksek belirlenirse, arz o kadar büyük ve talep o kadar az olur

Firmanız piyasa güçlerini hesaplamada çok başarılı olduğu için, pirincin fiyatı belirli bir p fiyatına ayarlandığında, arz ve talebi birim olarak anında alabilirsiniz.

Patronunuz en kısa sürede denge fiyatını istiyor, ancak size denge fiyatının en fazla ‘10^17’ olan bir pozitif tamsayı olabileceğini ve aralıkta tam olarak 1 pozitif tamsayı çözümünün garanti edildiğini söylüyor. O yüzden kaybetmeden önce işine devam et!

Sorunda belirtilenleri tam olarak yapacak olan “getSupply(k)” ve “getDemand(k)” işlevlerini çağırmanıza izin verilir.

Örnek Açıklama #

Burada arama uzayımız ‘1’ ile ‘10^17’ arasındadır. Bu nedenle doğrusal bir arama mümkün değildir.

Ancak, “k” yükseldikçe “getSupply(k)“nin arttığına ve “getDemand(k)“nin azaldığına dikkat edin. Böylece, herhangi bir “x > y” için, “getSupply(x) - getDemand(x) > getSupply(y) - getDemand(y)”). Bu nedenle, bu arama uzayı monotondur ve Binary Search’ü kullanabiliriz.

Aşağıdaki psuedocode, Binary Search’ün kullanımını gösterir:

high = 100000000000000000     <- Upper bound of search space
low = 1                       <- Lower bound of search space
while high - low > 1
    mid = (high + low) / 2    <- Take the middle value
    supply = getSupply(mid)  
    demand = getDemand(mid)
    if supply > demand        
        high = mid             <- Solution is in lower half of search space
    else if demand > supply
        low = mid              <- Solution is in upper half of search space
    else                       <- supply==demand condition
        return mid             <- Found solution

Bu algoritma ~O(log 10^17) zamanında çalışır. Bu, “while” döngüsünün her yinelemesinde arama alanını yarıya indirdiğimiz için (* [düşük:yüksek]‘den [ düşük:orta] veya [orta:yüksek]*).

C Özyinelemeli İkili Aramanın Uygulanması

int binsearch(int a[], int x, int low, int high) {
    int mid;

    if (low > high)
      return -1;

    mid = (low + high) / 2;

    if (x == a[mid]) {
        return (mid);
    } else 
    if (x < a[mid]) {
        binsearch(a, x, low, mid - 1);
    } else {
        binsearch(a, x, mid + 1, high);
    }
}

İkili Arama: Sıralanmış Sayılarda #

Sözde kod kullanarak sayılar üzerinde ikili arama göstermek en kolay yoldur

int array[1000] = { sorted list of numbers };
int N = 100;  // number of entries in search space;
int high, low, mid; // our temporaries
int x; // value to search for

low = 0;
high = N -1;
while(low < high)
{
    mid = (low + high)/2;
    if(array[mid] < x)
        low = mid + 1;
    else
        high = mid;  
}  
if(array[low] == x)
    // found, index is low
else
    // not found

Eşitlik için dizi[mid] ile x’i karşılaştırarak erken dönmeye çalışmayın. Ekstra karşılaştırma sadece kodu yavaşlatabilir. Her zaman aşağı yuvarlanan tamsayı bölümü tarafından tuzağa düşmemek için bir düşük eklemeniz gerektiğini unutmayın.

İlginç bir şekilde, ikili aramanın yukarıdaki versiyonu, dizideki en küçük x oluşumunu bulmanızı sağlar. Dizi, x’in kopyalarını içeriyorsa, yalnızca if koşuluna ekleyerek en büyük x oluşumunu döndürmesi için algoritma biraz değiştirilebilir:

while(low < high)
    {
        mid = low + ((high - low) / 2);
        if(array[mid] < x || (array[mid] == x && array[mid + 1] == x))
            low = mid + 1;
        else
            high = mid;  
    }

Orta = (düşük + yüksek) / 2’ yapmak yerine, Java uygulamaları gibi uygulamalarda gerçekten büyük girdiler için taşma riski.

Doğrusal arama #

Doğrusal arama basit bir algoritmadır. Sorgu bulunana kadar öğeler arasında dolaşır, bu da onu doğrusal bir algoritma yapar - karmaşıklık O(n)‘dir, burada n, geçilecek öğe sayısıdır.

Neden O(n)? En kötü senaryoda, tüm n öğeyi gözden geçirmeniz gerekir.

Bu, bir yığın kitap içinde bir kitap aramakla karşılaştırılabilir - istediğinizi bulana kadar hepsini gözden geçirirsiniz.

Aşağıda bir Python uygulaması verilmiştir:

def linear_search(searchable_list, query):
    for x in searchable_list:
        if query == x:
            return True
    return False

linear_search(['apple', 'banana', 'carrot', 'fig', 'garlic'], 'fig') #returns True

Haham Karp #

Rabin–Karp algoritması veya Karp–Rabin algoritması, bir metindeki bir dizi desen dizisinden herhangi birini bulmak için karma kullanan bir dizi arama algoritmasıdır. Ortalama ve en iyi durum çalışma süresi, O(n+m) uzayında O(n+m)‘dir. p), ancak en kötü durum zamanı O(nm)‘dir; burada n, metnin uzunluğu ve m, desenin uzunluğudur.

t =(d*(t - text.charAt(i)*h) + text.charAt(i+m))%q;

Bu, önce en soldaki karakteri kaldırarak ve ardından metinden yeni karakteri ekleyerek desen için karma değerini yeniden hesaplamak içindir.

Doğrusal aramanın analizi (En Kötü, Ortalama ve En İyi Durumlar) #

Bir algoritmayı analiz etmek için üç durumumuz olabilir:

  1. En Kötü Durum

  2. Ortalama Vaka

  3. En İyi Durum

    #include <stdio.h>

// Linearly search x in arr[].  If x is present then return the index,

// otherwise return -1
int search(int arr[], int n, int x)
{
    int i;
    for (i=0; i<n; i++)
    {
        if (arr[i] == x)
         return i;
    }

    return -1;
}

    /* Driver program to test above functions*/

    int main()
{
    int arr[] = {1, 10, 30, 15};
    int x = 30;
    int n = sizeof(arr)/sizeof(arr[0]);
    printf("%d is present at index %d", x, search(arr, n, x));

    getchar();
    return 0;
}

En Kötü Durum Analizi (Genellikle Yapılır)

En kötü durum analizinde, bir algoritmanın çalışma süresinin üst sınırını hesaplıyoruz. Maksimum sayıda işlemin yürütülmesine neden olan durumu bilmeliyiz. Doğrusal Arama için en kötü durum, aranacak öğe (yukarıdaki kodda x) dizide bulunmadığında ortaya çıkar. x olmadığında, search() işlevleri onu arr[] öğesinin tüm öğeleriyle tek tek karşılaştırır. Bu nedenle, doğrusal aramanın en kötü durum zaman karmaşıklığı Θ(n) olacaktır.

Ortalama Vaka Analizi (Bazen yapılır)

Ortalama durum analizinde, tüm olası girdileri alır ve tüm girdiler için hesaplama süresini hesaplarız. Hesaplanan tüm değerleri toplayın ve toplamı toplam girdi sayısına bölün. Vakaların dağılımını bilmeliyiz (veya tahmin etmeliyiz). Doğrusal arama problemi için, tüm durumların düzgün dağıldığını varsayalım (x’in dizide bulunmaması durumu dahil). Böylece tüm durumları toplarız ve toplamı (n+1)’e böleriz. Ortalama durum zaman karmaşıklığının değeri aşağıdadır.

En İyi Vaka Analizi (Bogus)

En iyi durum analizinde, bir algoritmanın çalışma süresinin alt sınırını hesaplarız. Minimum sayıda işlemin yürütülmesine neden olan durumu bilmeliyiz. Doğrusal arama probleminde en iyi durum, x ilk konumda mevcut olduğunda ortaya çıkar. En iyi durumda işlem sayısı sabittir (n’ye bağlı değildir). Yani en iyi durumda zaman karmaşıklığı Θ(1) olacaktır. Çoğu zaman, algoritmaları analiz etmek için en kötü durum analizi yaparız. En kötü analizde, iyi bilgi olan bir algoritmanın çalışma süresinde bir üst sınır garanti ediyoruz. Pratik vakaların çoğunda ortalama vaka analizi yapmak kolay değildir ve nadiren yapılır. Ortalama durum analizinde, tüm olası girdilerin matematiksel dağılımını bilmemiz (veya tahmin etmemiz) gerekir. En İyi Durum analizi sahtedir. Bir algoritmada daha düşük bir sınır garanti etmek, en kötü durumda olduğu gibi herhangi bir bilgi sağlamaz, bir algoritmanın çalışması yıllar alabilir.

Bazı algoritmalar için tüm durumlar asimptotik olarak aynıdır, yani en kötü ve en iyi durum yoktur. Örneğin, Birleştirme Sıralaması. Merge Sort, her durumda Θ(nLogn) işlemlerini yapar. Diğer sıralama algoritmalarının çoğu en kötü ve en iyi durumlara sahiptir. Örneğin, Hızlı Sıralama’nın tipik uygulamasında (bir köşe elemanı olarak pivotun seçildiği yerde), en kötüsü, giriş dizisi zaten sıralanmış olduğunda ve en iyisi, pivot elemanları diziyi her zaman iki yarıya böldüğünde meydana gelir. Ekleme sıralama için, dizi ters sıralandığında en kötü durum oluşur ve en iyi durum, dizi çıktıyla aynı sırada sıralandığında ortaya çıkar.
