Procedure GenerateSuffixArray(Pattern):
i := 1
j := 0
n := Pattern.length
while i is less than n
    if Pattern[i] is equal to Pattern[j]
        S[i] := j + 1
        j := j + 1
        i := i + 1
    else
        if j is not equal to 0
            j := S[j-1]
        else
            S[i] := 0
            i := i + 1
        end if
    end if
end while

Böylece Pattern[0] ve Pattern[1]‘i güvenle yok sayabilir ve bir sonraki karşılaştırmamıza Pattern[2] ve Text[8'den başlayabiliriz. ]. Devam edersek, Metin içinde Desen‘i bulacağız. Prosedürümüz şöyle görünecek:

Procedure KMP(Text, Pattern)
GenerateSuffixArray(Pattern)
m := Text.Length
n := Pattern.Length
i := 0
j := 0
while i is less than m
    if Pattern[j] is equal to Text[i]
        j := j + 1
        i := i + 1
    if j is equal to n
        Return (j-i)
    else if i < m and Pattern[j] is not equal t Text[i]
        if j is not equal to 0
            j = S[j-1]
        else
            i := i + 1
        end if
    end if
end while
Return -1

Bu algoritmanın Sonek Dizisi Hesaplaması dışında zaman karmaşıklığı O(m) dir. GenerateSuffixArray ‘O(n)’ aldığından, KMP Algoritmasının toplam zaman karmaşıklığı: ‘O(m+n)’ olur.

Not: Metin içinde birden çok Desen örneğini bulmak istiyorsanız, değeri döndürmek yerine değeri yazdırın/saklayın ve j := S[j-1] olarak ayarlayın. Ayrıca herhangi bir olay bulup bulmadığınızı takip etmek için bir ‘bayrak’ bulundurun ve buna göre işlem yapın.

Rabin-Karp Algoritmasına Giriş #

Rabin-Karp Algoritması, Richard M. Karp ._Karp) ve [Michael O. Rabin] (https://en.wikipedia.org/wiki/Michael_O._Rabin) tarafından oluşturulan ve herhangi birini bulmak için karma kullanan bir dizi arama algoritmasıdır. bir metindeki bir dizi desen dizisi.

Bir dizenin alt dizesi, içinde oluşan başka bir dizedir. Örneğin, ver, stackoverflow öğesinin bir alt dizesidir. Alt diziyle karıştırılmamalıdır çünkü kapak aynı dizgenin bir alt dizisidir. Başka bir deyişle, bir dizideki ardışık harflerin herhangi bir alt kümesi, verilen dizenin bir alt dizisidir.

Rabin-Karp algoritmasında, aradığımız kalıpımızın bir karmasını üreteceğiz ve metnin yuvarlanan karmasının kalıp ile eşleşip eşleşmediğini kontrol edeceğiz. Eşleşmiyorsa, metin içinde desen mevcut olmadığını garanti edebiliriz. Ancak, eşleşirse, metin içinde kalıp var olabilir. Bir örneğe bakalım:

Diyelim ki bir metnimiz var: yeminsajid ve metinde nsa kalıbının olup olmadığını öğrenmek istiyoruz. Hash ve haddeleme hashini hesaplamak için bir asal sayı kullanmamız gerekecek. Bu herhangi bir asal sayı olabilir. Bu örnek için asal = 11 alalım. Bu formülü kullanarak hash değerini belirleyeceğiz:

(1st letter) X (prime) + (2nd letter) X (prime)¹ + (3rd letter) X (prime)² X + ......

Şunları belirteceğiz:

a -> 1    g -> 7    m -> 13   s -> 19   y -> 25
b -> 2    h -> 8    n -> 14   t -> 20   z -> 26
c -> 3    i -> 9    o -> 15   u -> 21
d -> 4    j -> 10   p -> 16   v -> 22
e -> 5    k -> 11   q -> 17   w -> 23
f -> 6    l -> 12   r -> 18   x -> 24

nsa’nın karma değeri şöyle olacaktır:

 14 X 11⁰ + 19 X 11¹ + 1 X 11² = 344

Şimdi metnimizin yuvarlanan karmasını buluyoruz. Dönen hash, desenimizin hash değeriyle eşleşirse, dizelerin eşleşip eşleşmediğini kontrol edeceğiz. Modelimiz 3 harfli olduğundan, metnimizden 1. 3 harfi yem alıp hash değerini hesaplayacağız. Alırız:

25 X 11⁰ + 5 X 11¹ + 13 X 11² = 1653

Bu değer, desenimizin karma değeriyle eşleşmiyor. Yani dize burada yok. Şimdi bir sonraki adımı düşünmemiz gerekiyor. Bir sonraki dizgimizin karma değerini hesaplamak için emi. Bunu formülümüzle hesaplayabiliriz. Ama bu oldukça önemsiz olur ve bize daha pahalıya mal olur. Bunun yerine başka bir teknik kullanıyoruz.

  • Önceki Dizenin İlk Harfi değerini mevcut hash değerimizden çıkarıyoruz. Bu durumda, y. ‘1653 - 25 = 1628’ elde ederiz.
  • Farkı, bu örnek için 11 olan asal ile bölüyoruz. ‘1628 / 11 = 148’ elde ederiz.
  • yeni X harfi (asal)ᵐ⁻¹ ekliyoruz, burada m desenin uzunluğudur ve bölüm i = 9 olur. 148 + 9 X 11² = 1237 elde ederiz.

Yeni karma değeri, desen karma değerimize eşit değil. Devam edersek, n için şunu elde ederiz:

Previous String: emi
First Letter of Previous String: e(5)
New Letter: n(14)
New String: "min"
1237 - 5 = 1232
1232 / 11 = 112
112 + 14 X 11² = 1806

Eşleşmiyor. Bundan sonra, s için şunu elde ederiz:

Previous String: min
First Letter of Previous String: m(13)
New Letter: s(19)
New String: "ins"
1806 - 13 = 1793
1793 / 11 = 163
163 + 19 X 11² = 2462

Eşleşmiyor. Ardından, a için şunu elde ederiz:

Previous String: ins
First Letter of Previous String: i(9)
New Letter: a(1)
New String: "nsa"
2462 - 9 = 2453
2453 / 11 = 223
223 + 1 X 11² = 344

Bu bir maç! Şimdi desenimizi mevcut dizge ile karşılaştırıyoruz. Her iki dize de eşleştiğinden, alt dize bu dizede bulunur. Ve alt dizimizin başlangıç ​​pozisyonuna dönüyoruz.

Sözde kod şöyle olacaktır:

Karma Hesaplaması:

Procedure Calculate-Hash(String, Prime, x):
hash := 0                                  // Here x denotes the length to be considered
for m from 1 to x                          // to find the hash value
    hash := hash + (Value of String[m])ᵐ⁻¹
end for
Return hash

Karma Yeniden Hesaplama:

Procedure Recalculate-Hash(String, Curr, Prime, Hash):
Hash := Hash - Value of String[Curr]  //here Curr denotes First Letter of Previous String
Hash := Hash / Prime
m := String.length
New := Curr + m - 1
Hash := Hash + (Value of String[New])ᵐ⁻¹
Return Hash

Dize Maçı:

Procedure String-Match(Text, Pattern, m):
for i from m to Pattern-length + m - 1
    if Text[i] is not equal to Pattern[i]
        Return false
    end if
end for
Return true

Rabin-Karp:

Procedure Rabin-Karp(Text, Pattern, Prime):
m := Pattern.Length
HashValue := Calculate-Hash(Pattern, Prime, m)
CurrValue := Calculate-Hash(Pattern, Prime, m)
for i from 1 to Text.length - m
    if HashValue == CurrValue and String-Match(Text, Pattern, i) is true
        Return i
    end if
    CurrValue := Recalculate-Hash(String, i+1, Prime, CurrValue)
end for
Return -1

Algoritma herhangi bir eşleşme bulamazsa, basitçe -1 döndürür.

Bu algoritma intihal tespitinde kullanılır. Algoritma, kaynak materyal verildiğinde, büyük/küçük harf ve noktalama işaretleri gibi ayrıntıları göz ardı ederek kaynak materyaldeki cümlelerin örneklerini hızlı bir şekilde bir kağıt içinde arayabilir. Aranan dizilerin bolluğu nedeniyle, tek dizili arama algoritmaları burada pratik değildir. Yine, Knuth-Morris-Pratt algoritması veya Boyer-Moore Dizi Arama algoritması, Rabin-Karp’tan daha hızlı tek model dizi arama algoritmasıdır. Ancak, çoklu örüntü araması için tercih edilen bir algoritmadır. Bir metinde çok sayıda, diyelim ki k, sabit uzunluklu desenlerden herhangi birini bulmak istiyorsak, Rabin-Karp algoritmasının basit bir varyantını oluşturabiliriz.

n uzunluğundaki metin ve p birleşik uzunluktaki m desenleri için, ortalama ve en iyi senaryo çalışma süresi uzayda O(n+m)‘dir O(p) ), ancak en kötü durum zamanı O(nm)‘dir.

C’de KMP Algoritması #

Bir txt metni ve bir pat kalıbı verildiğinde, bu programın amacı pat‘ın tüm oluşumunu txt içinde yazdırmak olacaktır.

Örnekler:

Giriş:

  txt[] =  "THIS IS A TEST TEXT"
  pat[] = "TEST"

çıktı:

Pattern found at index 10

Giriş:

  txt[] =  "AABAACAADAABAAABAA"
  pat[] = "AABA"

çıktı:

   Pattern found at index 0
   Pattern found at index 9
   Pattern found at index 13

C Dil Uygulaması:

// C program for implementation of KMP pattern searching 
// algorithm
#include<stdio.h>
#include<string.h>
#include<stdlib.h>
 
void computeLPSArray(char *pat, int M, int *lps);
 
void KMPSearch(char *pat, char *txt)
{
    int M = strlen(pat);
    int N = strlen(txt);
 
    // create lps[] that will hold the longest prefix suffix
    // values for pattern
    int *lps = (int *)malloc(sizeof(int)*M);
    int j  = 0;  // index for pat[]
 
    // Preprocess the pattern (calculate lps[] array)
    computeLPSArray(pat, M, lps);
 
    int i = 0;  // index for txt[]
    while (i < N)
    {
      if (pat[j] == txt[i])
      {
        j++;
        i++;
      }
 
      if (j == M)
      {
        printf("Found pattern at index %d \n", i-j);
        j = lps[j-1];
      }
 
      // mismatch after j matches
      else if (i < N && pat[j] != txt[i])
      {
        // Do not match lps[0..lps[j-1]] characters,
        // they will match anyway
        if (j != 0)
         j = lps[j-1];
        else
         i = i+1;
      }
    }
    free(lps); // to avoid memory leak
}
 
void computeLPSArray(char *pat, int M, int *lps)
{
    int len = 0;  // length of the previous longest prefix suffix
    int i;
 
    lps[0] = 0; // lps[0] is always 0
    i = 1;
 
    // the loop calculates lps[i] for i = 1 to M-1
    while (i < M)
    {
       if (pat[i] == pat[len])
       {
         len++;
         lps[i] = len;
         i++;
       }
       else // (pat[i] != pat[len])
       {
         if (len != 0)
         {
           // This is tricky. Consider the example 
           // AAACAAAA and i = 7.
           len = lps[len-1];
 
           // Also, note that we do not increment i here
         }
         else // if (len == 0)
         {
           lps[i] = 0;
           i++;
         }
       }
    }
}
 
// Driver program to test above function
int main()
{
   char *txt = "ABABDABACDABABCABAB";
   char *pat = "ABABCABAB";
   KMPSearch(pat, txt);
   return 0;
}

KMP algoritmasının Python Uygulaması. #

Saman Yığını: Belirtilen kalıbın aranması gereken dize.
İğne: Aranacak desen.

Zaman karmaşıklığı: Arama kısmı (strstr yöntemi) O(n) karmaşıklığına sahiptir, burada ’n’ samanlığın uzunluğudur, ancak iğne ayrıca önek tablosu oluşturmak için ayrıştırıldığından, önek tablosu oluşturmak için O(m) gereklidir burada “m” iğnenin uzunluğudur.
Bu nedenle, KMP için toplam zaman karmaşıklığı O(n+m)‘dir
Uzay karmaşıklığı: O(m) iğne üzerindeki ön ek tablosu nedeniyle.

Not: Aşağıdaki uygulama, samanlıkta eşleşmenin başlangıç ​​konumunu döndürür (bir eşleşme varsa), aksi takdirde -1 döndürür, örneğin iğne/samanlık boş bir dizeyse veya samanlıkta iğne bulunamadı.

def get_prefix_table(needle):
    prefix_set = set()
    n = len(needle)
    prefix_table = [0]*n
    delimeter = 1
    while(delimeter<n):
        prefix_set.add(needle[:delimeter])
        j = 1
        while(j<delimeter+1):
            if needle[j:delimeter+1] in prefix_set:
                prefix_table[delimeter] = delimeter - j + 1
                break
            j += 1
        delimeter += 1
    return prefix_table

def strstr(haystack, needle):
    # m: denoting the position within S where the prospective match for W begins
    # i: denoting the index of the currently considered character in W.
    haystack_len = len(haystack)
    needle_len = len(needle)
    if (needle_len > haystack_len) or (not haystack_len) or (not needle_len):
        return -1
    prefix_table = get_prefix_table(needle)
    m = i = 0
    while((i<needle_len) and (m<haystack_len)):
        if haystack[m] == needle[i]:
            i += 1
            m += 1
        else:
            if i != 0:
                i = prefix_table[i-1]
            else:
                m += 1
    if i==needle_len and haystack[m-1] == needle[i-1]:
        return m - needle_len
    else:
        return -1
