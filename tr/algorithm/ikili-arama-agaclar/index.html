İkili ağaç, içindeki her düğümün en fazla iki çocuğu olan bir ağaçtır. İkili arama ağacı (BST), öğelerinin özel sıraya göre konumlandırıldığı bir ikili ağaçtır. Her BST’de sol alt ağaçtaki tüm değerler (yani anahtar), sağ alt ağaçtaki değerlerden küçüktür.

İkili Arama Ağacı - Ekleme (Python) #

Bu, Python kullanılarak İkili Arama Ağacı Eklemenin basit bir uygulamasıdır.

Aşağıda bir örnek gösterilmiştir:

Kod parçacığının ardından her görüntü, bu kodun nasıl çalıştığını görselleştirmeyi kolaylaştıran yürütme görselleştirmesini gösterir.

class Node:
    def __init__(self, val):
        self.l_child = None
        self.r_child = None
        self.data = val


def insert(root, node):
    if root is None:
        root = node
    else:
        if root.data > node.data:
            if root.l_child is None:
                root.l_child = node
            else:
                insert(root.l_child, node)
        else:
            if root.r_child is None:
                root.r_child = node
            else:
                insert(root.r_child, node)


def in_order_print(root):
    if not root:
        return
    in_order_print(root.l_child)
    print root.data
    in_order_print(root.r_child)


def pre_order_print(root):
    if not root:
        return        
    print root.data
    pre_order_print(root.l_child)
    pre_order_print(root.r_child)

İkili Arama Ağacı - Silme(C++) #

Silme işlemine başlamadan önce, İkili arama ağacının (BST) ne olduğuna biraz ışık tutmak istiyorum, BST’deki her düğüm en fazla iki düğüme (sol ve sağ çocuk) sahip olabilir. Bir düğümün sol alt ağacının bir anahtar, ana düğümünün anahtarından küçük veya ona eşit. Bir düğümün sağ alt ağacı, üst düğümün anahtarından daha büyük bir anahtara sahiptir.

İkili arama ağacı özelliğini korurken bir ağaçtaki bir düğümü silme.

Bir düğüm silinirken dikkate alınması gereken üç durum vardır.

  • Durum 1: Silinecek düğüm, yaprak düğümdür.(22 değerindeki düğüm).
  • Durum 2: Silinecek düğümün bir çocuğu var.(26 değerine sahip düğüm).
  • Durum 3: Silinecek düğümün her iki çocuğu da var.(49 değerindeki düğüm).

Vakaların açıklaması:

  1. Silinecek düğüm bir yaprak düğüm olduğunda, düğümü silin ve üst düğümüne nullptr iletin.
  2. Silinecek bir düğümün yalnızca bir çocuğu varsa, alt değeri düğüm değerine kopyalayın ve alt öğeyi silin (Durum 1’e dönüştürüldü)
  3. Silinecek bir düğümün iki çocuğu olduğunda, sağ alt ağacındaki minimum değer düğüme kopyalanabilir ve ardından minimum değer düğümün sağ alt ağacından silinebilir (Durum 2’ye Dönüştürüldü)

Not: Sağ alt ağaçtaki minimum, maksimum bir çocuğa sahip olabilir ve sol çocuğu varsa, bu minimum değer olmadığı veya BST özelliğini takip etmediği anlamına gelir.

Bir ağaçtaki düğümün yapısı ve Silme kodu:

struct node
{
    int data;
    node *left, *right;
};

node* delete_node(node *root, int data)
{
  if(root == nullptr) return root;
  else if(data < root->data) root->left  = delete_node(root->left, data);
  else if(data > root->data) root->right = delete_node(root->right, data);

  else
  {
    if(root->left == nullptr && root->right == nullptr) // Case 1
    {
      free(root);
      root = nullptr;
    }
    else if(root->left == nullptr)       // Case 2
    {
       node* temp = root;
       root= root->right;
       free(temp);
    }
    else if(root->right == nullptr)      // Case 2
    {
       node* temp = root;
       root = root->left;
       free(temp);
    }
    else                                 // Case 3
    {
       node* temp = root->right;

       while(temp->left != nullptr) temp = temp->left;

       root->data = temp->data;
       root->right = delete_node(root->right, temp->data);
    }
  }
  return root;
}

Yukarıdaki kodun zaman karmaşıklığı O(h) şeklindedir, burada h ağacın yüksekliğidir.

BST’deki en düşük ortak ata #

BST’yi düşünün:

22 ve 26’nın en küçük ortak atası 24’tür.

26 ve 49’un en küçük ortak atası 46’dır.

22 ve 24’ün en küçük ortak atası 24’tür.

İkili arama ağacı özelliği, düğümlerin en alt atasını bulmak için kullanılabilir

İkili Arama Ağacı - Python #

class Node(object):
    def __init__(self, val):
        self.l_child = None
        self.r_child = None
        self.val = val


class BinarySearchTree(object):
    def insert(self, root, node):
        if root is None:
            return node

        if root.val < node.val:
            root.r_child = self.insert(root.r_child, node)
        else:
            root.l_child = self.insert(root.l_child, node)

        return root


    def in_order_place(self, root):
        if not root:
            return None
        else:
            self.in_order_place(root.l_child)
            print root.val
            self.in_order_place(root.r_child)

    def pre_order_place(self, root):
        if not root:
            return None
        else:
            print root.val
            self.pre_order_place(root.l_child)
            self.pre_order_place(root.r_child)

    def post_order_place(self, root):
        if not root:
            return None
        else:
            self.post_order_place(root.l_child)
            self.post_order_place(root.r_child)
            print root.val

""" Farklı bir düğüm oluşturun ve içine veri ekleyin"""

r = Node(3)
node = BinarySearchTree()
nodeList = [1, 8, 5, 12, 14, 6, 15, 7, 16, 8]

for nd in nodeList:
    node.insert(r, Node(nd))

print "------In order ---------"
print (node.in_order_place(r))
print "------Pre order ---------"
print (node.pre_order_place(r))
print "------Post order ---------"
print (node.post_order_place(r))
