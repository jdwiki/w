Grafik, bazı (muhtemelen boş) alt kümelerini birbirine bağlayan noktalar ve çizgiler topluluğudur. Bir grafiğin noktalarına grafik köşeleri, “düğümler” veya basitçe “noktalar” denir. Benzer şekilde, bir grafiğin köşelerini birleştiren çizgilere grafik kenarları, “yaylar” veya “çizgiler” denir.

G grafiği bir çift (V,E) olarak tanımlanabilir, burada V bir köşeler kümesidir ve E köşeler arasındaki bir kenar kümesidir E ⊆ {(u,v) | u, v ∈ V}.

Grafikler, kenar veya bağlantı kümelerinden üyelerle bağlantılı olabilecek veya olmayabilecek nesne kümelerini modelleyen matematiksel bir yapıdır.

Bir grafik, iki farklı matematiksel nesne kümesi aracılığıyla tanımlanabilir:

  • Bir dizi köşe.
  • Köşe çiftlerini birbirine bağlayan bir dizi kenar.

Grafikler yönlendirilmiş veya yönlendirilmemiş olabilir.

  • Yönlendirilmiş grafikler yalnızca tek yönde “bağlanan” kenarlar içerir.
  • Yönsüz grafikler yalnızca iki köşeyi her iki yönde otomatik olarak birbirine bağlayan kenarları içerir.

Grafikleri Depolama (Bitişlik Matrisi) #

Bir grafiği saklamak için iki yöntem yaygındır:

  • Bitişiklik Matrisi
  • Komşuluk Listesi

Bir komşuluk matrisi, sonlu bir grafiği temsil etmek için kullanılan bir kare matristir. Matrisin elemanları, grafikte köşe çiftlerinin bitişik olup olmadığını gösterir.

Bitişik, ‘başka bir şeyin yanında veya bitişiğinde’ veya bir şeyin yanında olmak anlamına gelir. Örneğin komşularınız yanınızda. Grafik teorisinde, düğüm A‘dan düğüm B‘ye gidebilirsek, düğüm B’nin düğüm A‘ya bitişik olduğunu söyleyebiliriz. Şimdi Bitişiklik Matrisi ile hangi düğümlerin hangisine bitişik olduğunu nasıl depolayacağımızı öğreneceğiz. Bu, hangi düğümlerin aralarında kenar paylaştığını temsil edeceğimiz anlamına gelir. Burada matris 2B dizi anlamına gelir.

Burada grafiğin yanında bir tablo görebilirsiniz, bu bizim komşuluk matrisimiz. Burada Matris[i][j] = 1, i ile j arasında bir kenar olduğunu temsil eder. Kenar yoksa, basitçe Matrix[i][j] = 0 koyarız.

Bu kenarlar, iki şehir arasındaki mesafeyi temsil ediyormuş gibi ağırlıklandırılabilir. Ardından değeri 1 yerine Matrix[i][j] içine koyacağız.

Yukarıda açıklanan grafik Çift yönlü veya Yönsüz‘dür, yani düğüm 2‘den düğüm 1’e gidebilirsek, düğüm’den düğüm 2‘ye de gidebiliriz 1. Grafik Yönlendirilmiş olsaydı, grafiğin bir tarafında ok işareti olurdu. O zaman bile, onu bitişiklik matrisi kullanarak gösterebiliriz.

Kenarı sonsuz olarak paylaşmayan düğümleri temsil ediyoruz. Dikkat edilmesi gereken bir şey, eğer grafik yönsüzse, matris simetrik olur.

Matrisi oluşturmak için sözde kod:

Procedure AdjacencyMatrix(N):    //N represents the number of nodes
Matrix[N][N]
for i from 1 to N
    for j from 1 to N
        Take input -> Matrix[i][j]
    endfor
endfor

Matrix’i bu yaygın yolu kullanarak da doldurabiliriz:

Procedure AdjacencyMatrix(N, E):    // N -> number of nodes
Matrix[N][E]                        // E -> number of edges
for i from 1 to E
    input -> n1, n2, cost
    Matrix[n1][n2] = cost
    Matrix[n2][n1] = cost
endfor

Yönlendirilmiş grafikler için Matrix[n2][n1] = maliyet satırını kaldırabiliriz.

** Bitişiklik Matrisi kullanmanın dezavantajları:**

Hafıza çok büyük bir problem. Kaç tane kenar olursa olsun, her zaman N * N boyutlu matrise ihtiyacımız olacak, burada N düğüm sayısıdır. 10000 düğüm varsa, matris boyutu 381 megabayt civarında 4 * 10000 * 10000 olacaktır. Eğer düşünürsek, bu büyük bir hafıza kaybıdır. birkaç kenarı olan grafikler.

Bir u düğümünden hangi düğüme gidebileceğimizi bulmak istediğimizi varsayalım. Tüm u satırını kontrol etmemiz gerekecek, bu da çok zaman alıyor.

Tek faydası, u-v düğümleri arasındaki bağlantıyı ve bunların maliyetini Bitişiklik Matrisi kullanarak kolayca bulabilmemizdir.

Yukarıdaki sözde kod kullanılarak uygulanan Java kodu:

import java.util.Scanner;
 
public class Represent_Graph_Adjacency_Matrix 
{
    private final int vertices;
    private int[][] adjacency_matrix;
 
    public Represent_Graph_Adjacency_Matrix(int v) 
    {
        vertices = v;
        adjacency_matrix = new int[vertices + 1][vertices + 1];
    }
 
    public void makeEdge(int to, int from, int edge) 
    {
        try 
        {
            adjacency_matrix[to][from] = edge;
        }
        catch (ArrayIndexOutOfBoundsException index) 
        {
            System.out.println("The vertices does not exists");
        }
    }
 
    public int getEdge(int to, int from) 
    {
        try 
        {
            return adjacency_matrix[to][from];
        }
        catch (ArrayIndexOutOfBoundsException index) 
        {
            System.out.println("The vertices does not exists");
        }
        return -1;
    }
 
    public static void main(String args[]) 
    {
        int v, e, count = 1, to = 0, from = 0;
        Scanner sc = new Scanner(System.in);
        Represent_Graph_Adjacency_Matrix graph;
        try 
        {
            System.out.println("Enter the number of vertices: ");
            v = sc.nextInt();
            System.out.println("Enter the number of edges: ");
            e = sc.nextInt();
 
            graph = new Represent_Graph_Adjacency_Matrix(v);
 
            System.out.println("Enter the edges: <to> <from>");
            while (count <= e) 
            {
                to = sc.nextInt();
                from = sc.nextInt();
 
                graph.makeEdge(to, from, 1);
                count++;
            }
 
            System.out.println("The adjacency matrix for the given graph is: ");
            System.out.print("  ");
            for (int i = 1; i <= v; i++)
                System.out.print(i + " ");
            System.out.println();
 
            for (int i = 1; i <= v; i++) 
            {
                System.out.print(i + " ");
                for (int j = 1; j <= v; j++) 
                    System.out.print(graph.getEdge(i, j) + " ");
                System.out.println();
            }
 
        }
        catch (Exception E) 
        {
            System.out.println("Somthing went wrong");
        }
 
        sc.close();
    }
}

Kodu çalıştırmak: Dosyayı kaydedin ve javac Represent_Graph_Adjacency_Matrix.java kullanarak derleyin

Örnek:

$ java Represent_Graph_Adjacency_Matrix

Enter the number of edges:

Grafik Teorisine Giriş #

Graph Theory, nesneler arasındaki ikili ilişkileri modellemek için kullanılan matematiksel yapılar olan grafiklerin incelenmesidir.

Dünya gezegeninin hemen hemen tüm sorunlarının Yol ve Şehir sorunlarına dönüştürülebileceğini ve çözülebileceğini biliyor muydunuz? Grafik Teorisi yıllar önce, hatta bilgisayarın icadından önce icat edildi. Leonhard Euler Königsberg’in Yedi Köprüsü hakkında bir makale yazdı. Grafik Teorisinin ilk makalesi olarak kabul edilir. O zamandan beri insanlar, herhangi bir sorunu bu Şehir-Yol sorununa dönüştürebilirsek, Grafik Teorisi ile kolayca çözebileceğimizi anladılar.

Grafik Teorisinin birçok uygulaması vardır. En yaygın uygulamalardan biri, bir şehirden diğerine en kısa mesafeyi bulmaktır. PC’nize ulaşmak için bu web sayfasının sunucudan birçok yönlendiriciyi gezmesi gerektiğini hepimiz biliyoruz. Grafik Teorisi, çaprazlanması gereken yönlendiricileri bulmasına yardımcı olur. Savaş sırasında, başkenti diğerlerinden ayırmak için hangi caddenin bombalanması gerektiği, Grafik Teorisi kullanılarak da bulunabilir.

Önce Grafik Teorisi ile ilgili bazı temel tanımları öğrenelim.

Grafik:

Diyelim ki 6 şehrimiz var. Bunları 1, 2, 3, 4, 5, 6 olarak işaretliyoruz. Şimdi yolları olan şehirleri birbirine bağlıyoruz.

Bu, bazı şehirlerin onları birbirine bağlayan yollarla gösterildiği basit bir grafiktir. Grafik Teorisinde, bu şehirlerin her birine Düğüm veya Vertex diyoruz ve yollara Kenar denir. Grafik basitçe bu düğümlerin ve kenarların bir bağlantısıdır.

Bir düğüm birçok şeyi temsil edebilir. Bazı grafiklerde düğümler şehirleri, bazıları havaalanlarını, bazıları da satranç tahtasındaki bir kareyi temsil ediyor. Edge, her düğüm arasındaki ilişkiyi temsil eder. Bu ilişki, bir havaalanından diğerine gitme zamanı, bir şövalyenin bir meydandan diğer tüm meydanlara yaptığı hareketler vb. olabilir.

                                             Bir Satranç Tahtasında Şövalye Yolu

Basit bir deyişle, bir Düğüm herhangi bir nesneyi temsil eder ve Kenar iki nesne arasındaki ilişkiyi temsil eder.

** Bitişik Düğüm:**

Bir A düğümü, B düğümü ile bir kenarı paylaşıyorsa, B, A‘ya bitişik olarak kabul edilir. Başka bir deyişle, iki düğüm doğrudan bağlantılıysa, bunlara bitişik düğümler denir. Bir düğüm birden çok bitişik düğüme sahip olabilir.

Yönlendirilmiş ve Yönlendirilmemiş Grafik:

Yönlü grafiklerde kenarların bir tarafında yön işaretleri vardır, bu da kenarların Tek yönlü olduğu anlamına gelir. Öte yandan, yönsüz grafiklerin kenarlarının her iki tarafında da yön işaretleri vardır, bu da onların Çift yönlü oldukları anlamına gelir. Genellikle yönsüz grafikler, kenarların her iki tarafında da işaret olmadan gösterilir.

Diyelim ki devam eden bir parti var. Partideki insanlar düğümlerle temsil edilir ve iki kişi arasında el sıkıştıklarında bir kenar vardır. O halde bu grafik yönsüzdür, çünkü A herhangi bir kişi B kişisiyle tokalaşırsa ve ancak ve ancak B aynı zamanda A ile de tokalaşırsa. Buna karşılık, bir kişiden A başka bir kişiye B kenarları A’nın B hayranlığına karşılık geliyorsa, o zaman bu grafik yönlendirilir, çünkü hayranlığın mutlaka karşılıklı olması gerekmez . İlk grafik türüne yönsüz grafik ve kenarlara yönsüz kenarlar denirken, ikinci grafik türüne yönlü grafik ve kenarlara yönlü kenarlar denir.

Ağırlıklı ve Ağırlıksız Grafik:

Ağırlıklı grafik, her bir kenara bir sayının (ağırlığın) atandığı bir grafiktir. Bu tür ağırlıklar, eldeki soruna bağlı olarak örneğin maliyetleri, uzunlukları veya kapasiteleri temsil edebilir.

Ağırlıksız bir grafik tam tersidir. Tüm kenarların ağırlığının aynı olduğunu varsayıyoruz (muhtemelen 1).

Yol:

Yol, bir düğümden diğerine geçmenin bir yolunu temsil eder. Kenar dizisinden oluşur. İki düğüm arasında birden çok yol olabilir.

Yukarıdaki örnekte, A ile D arasında iki yol vardır. A->B, B->C, C->D bir yoldur. Bu yolun maliyeti 3 + 4 + 2 = 9‘dur. Yine, başka bir yol var A->D. Bu yolun maliyeti 10. Maliyeti en düşük olan yola en kısa yol denir.

Derece:

Bir tepe noktasının derecesi, ona bağlı kenarların sayısıdır. Her iki uçta tepe noktasına bağlanan herhangi bir kenar varsa (bir döngü) iki kez sayılır.

Yönlendirilmiş grafiklerde düğümlerin iki tür derecesi vardır:

  • Derece olarak: Düğüme işaret eden kenarların sayısı.
  • Derece Dışı: Düğümden diğer düğümlere işaret eden kenarların sayısı.

Yönlendirilmemiş grafikler için basitçe derece olarak adlandırılırlar.

Grafikleri Depolama (Bitişlik Listesi) #

Komşuluk listesi, sonlu bir grafiği temsil etmek için kullanılan sırasız listelerin bir koleksiyonudur. Her liste, bir grafikteki bir köşenin komşu kümesini tanımlar. Grafikleri depolamak için daha az bellek gerekir.

Bir grafiği ve onun komşuluk matrisini görelim:

Şimdi bu değerleri kullanarak bir liste oluşturuyoruz.

Buna komşuluk listesi denir. Hangi düğümlerin hangi düğümlere bağlı olduğunu gösterir. Bu bilgiyi 2B dizi kullanarak saklayabiliriz. Ama bize Bitişiklik Matrisi ile aynı belleğe mal olacak. Bunun yerine, bunu depolamak için dinamik olarak ayrılmış belleği kullanacağız.

Birçok dil, bitişiklik listesini depolamak için kullanabileceğimiz Vektör veya Liste‘yi destekler. Bunlar için Listenin boyutunu belirtmemize gerek yoktur. Yalnızca maksimum düğüm sayısını belirtmemiz gerekir.

Sözde kod şöyle olacaktır:

Procedure Adjacency-List(maxN, E):       // maxN denotes the maximum number of nodes
edge[maxN] = Vector()                    // E denotes the number of edges
for i from 1 to E
    input -> x, y                        // Here x, y denotes there is an edge between x, y
    edge[x].push(y)
    edge[y].push(x)
end for
Return edge

Bu bir yönsüz grafik olduğundan, x ile y arasında bir kenar vardır, ayrıca y ile x arasında bir kenar vardır. Yönlendirilmiş bir grafik olsaydı, ikincisini atlardık. Ağırlıklı grafikler için maliyeti de saklamamız gerekir. Bunları depolamak için maliyet[] adında başka bir vektör veya liste oluşturacağız. Sözde kod:

Procedure Adjacency-List(maxN, E):
edge[maxN] = Vector()
cost[maxN] = Vector()
for i from 1 to E
    input -> x, y, w
    edge[x].push(y)
    cost[x].push(w)
end for
Return edge, cost

Buradan, herhangi bir düğüme bağlı toplam düğüm sayısını ve bu düğümlerin ne olduğunu kolayca öğrenebiliriz. Bitişiklik Matrisine göre daha az zaman alır. Ama u ile v arasında bir kenar olup olmadığını bulmamız gerekirse, bir komşuluk matrisi tutsaydık daha kolay olurdu.

Topolojik Sıralama #

Bir topolojik sıralama veya topolojik bir sıralama, köşeleri sıralar. bir çizgi üzerinde yönlendirilmiş bir döngüsel olmayan grafikte, yani bir listede, öyle ki hepsi yönlendirilmiş kenarlar soldan sağa gider. Böyle bir sıralama olamaz grafik yönlendirilmiş bir döngü içeriyorsa, çünkü bir çizgi üzerinde doğru devam edip yine de başladığınız yere geri dönmenizin bir yolu yoktur.

Biçimsel olarak, bir G = (V, E) grafiğinde, tüm değerlerinin doğrusal bir sıralaması köşeler öyledir ki, “G”, “u” köşesinden “v” köşesine bir “(u, v) ∈ E” kenarını içeriyorsa, sıralamada “u”, “v"den önce gelir.

Her DAG’nin en az bir topolojik sıralama

Bir topolojik sıralama oluşturmak için bilinen algoritmalar vardır. doğrusal zamanda herhangi bir DAG için bir örnek:

  1. Her köşe ‘v’ için ‘v.f’ bitiş sürelerini hesaplamak için ‘depth_first_search(G)‘yi çağırın
  2. Her köşe bittiğinde, onu bağlantılı bir listenin önüne ekleyin
  3. şimdi sıralandığı gibi, bağlantılı köşeler listesi.

ϴ(V + E) zamanında topolojik sıralama yapılabilir. derinlik öncelikli arama algoritması ϴ(V + E) zaman alır ve Ω(1) alır (sabit süre) her bir |V| köşesini ön tarafa eklemek için bağlantılı bir liste.

Birçok uygulama, öncelikleri belirtmek için yönlendirilmiş döngüsel olmayan grafikler kullanır. olaylar arasında. Topolojik sıralama kullanıyoruz böylece herhangi birinden önce her bir köşeyi işlemek için bir sipariş alırız. onun halefleri.

Bir grafikteki köşeler, gerçekleştirilecek görevleri ve kenarları temsil edebilir. önce bir görevin gerçekleştirilmesi gereken kısıtlamaları temsil edebilir bir diğer; topolojik sıralama, aşağıdakileri gerçekleştirmek için geçerli bir sıradır. görevler V içinde açıklanan görevler dizisi.

Bir ‘v’ köşesinin bir ‘Görevi(saat_to_complete:int)’ tanımlamasına izin verin, i. e. “Görev(4)”, tamamlanması “4” saat süren bir “Görev"i tanımlar ve “e” kenarı bir “Bekleme(saat: int)“yi tanımlar, öyle ki “Bekleme(3)”, tamamlanması gereken süreyi tanımlar. tamamlanan bir görevden sonra soğuma.

Grafiğimiz “dag” olarak adlandırılsın (yönlendirilmiş asiklik bir grafik olduğu için) ve 5 köşe içermesine izin verin:

A <- dag.add_vertex(Task(4));

C <- dag.add_vertex(Task(3)); 
D <- dag.add_vertex(Task(2));

köşeleri, grafik asiklik olacak şekilde yönlendirilmiş kenarlarla birleştirdiğimizde,

// A ---> C ----+

Thorup’un algoritması #

Thorup’un yönsüz grafik için tek kaynaklı en kısa yol algoritması, Dijkstra’dan daha düşük zaman karmaşıklığına O(m) sahiptir.

Temel fikirler aşağıdaki gibidir. (Üzgünüm, henüz uygulamayı denemedim, bu yüzden bazı küçük ayrıntıları kaçırabilirim. Ve orijinal makale ödeme duvarlı, bu yüzden onu referans alan diğer kaynaklardan yeniden oluşturmaya çalıştım. Doğrulayabilirseniz lütfen bu yorumu kaldırın.)

  • O(m)‘de yayılan ağacı bulmanın yolları vardır (burada açıklanmamıştır). Yayılan ağacı en kısa kenardan en uzuna “büyütmeniz” gerekir ve bu, tamamen büyümeden önce birkaç bağlantılı bileşene sahip bir orman olacaktır.
  • Bir tamsayı b (b>=2) seçin ve yalnızca uzunluk sınırı b^k olan yayılan ormanları göz önünde bulundurun. Tam olarak aynı olan ancak farklı k’ye sahip bileşenleri birleştirin ve minimum k’yi bileşenin düzeyi olarak adlandırın. Ardından mantıksal olarak bileşenleri bir ağaca dönüştürün. u, v’nin ebeveynidir ve u, v’den tamamen farklı olan en küçük bileşendir. Kök, grafiğin tamamıdır ve yapraklar orijinal grafikteki tek köşelerdir (negatif sonsuzluk düzeyiyle). Ağacın hala yalnızca O(n) düğümü var.
  • Her bileşenin kaynağa olan mesafesini koruyun (Dijkstra’nın algoritmasında olduğu gibi). Birden fazla köşesi olan bir bileşenin mesafesi, genişletilmemiş alt öğelerinin minimum mesafesidir. Kaynak tepe noktasının mesafesini 0’a ayarlayın ve ataları buna göre güncelleyin.
  • B tabanındaki mesafeleri göz önünde bulundurun. k düzeyindeki bir düğümü ilk kez ziyaret ederken, çocuklarını, k düzeyindeki tüm düğümler tarafından paylaşılan kovalara koyun (kova sıralamasında olduğu gibi, Dijkstra’nın algoritmasındaki yığını, uzaklığının k ve daha yüksek basamağıyla değiştirin). Bir düğümü her ziyaret ettiğinizde, yalnızca ilk b kovalarını göz önünde bulundurun, her birini ziyaret edin ve kaldırın, mevcut düğümün mesafesini güncelleyin ve yeni mesafeyi kullanarak mevcut düğümü kendi ebeveynine yeniden bağlayın ve bir sonraki ziyareti bekleyin. kovalar.
  • Bir yaprak ziyaret edildiğinde, mevcut mesafe tepe noktasının son mesafesidir. Orijinal grafikte ondan tüm kenarları genişletin ve mesafeleri buna göre güncelleyin.
  • Hedefe ulaşılana kadar kök düğümü (tüm grafiği) tekrar tekrar ziyaret edin.

Bu, yayılan ormanın l uzunluk sınırlaması ile bağlantılı iki bileşeni arasında uzunluğu l’den küçük bir kenarın olmaması gerçeğine dayanmaktadır, bu nedenle, x mesafesinden başlayarak, ulaşana kadar yalnızca bir bağlı bileşene odaklanabilirsiniz. uzaklık x + l. Daha kısa mesafeli köşelerin tümü ziyaret edilmeden önce bazı köşeleri ziyaret edeceksiniz, ancak bu önemli değil çünkü bu köşelerden buraya daha kısa bir yol olmayacağı biliniyor. Diğer parçalar kova sıralama / MSD taban sıralama gibi çalışır ve elbette O(m) kapsayan ağacı gerektirir.

Derinlik İlk Geçişi kullanarak yönlendirilmiş bir grafikte bir döngü algılama #

DFS sırasında keşfedilen bir arka kenar varsa, yönlendirilmiş bir grafikte bir döngü vardır. Arka kenar, bir düğümden kendisine veya bir DFS ağacındaki atalardan birine olan kenardır. Bağlantısı kesilmiş bir grafik için bir DFS ormanı elde ederiz, bu nedenle ayrık DFS ağaçlarını bulmak için grafikteki tüm köşeleri yinelemeniz gerekir.

C++ uygulaması:

    #include <iostream>
     #include <list>
        
    using namespace std; 
    
    #define NUM_V   4

    bool helper(list<int> *graph, int u, bool* visited, bool* recStack)
    {
        visited[u]=true;
        recStack[u]=true;
        list<int>::iterator i;
        for(i = graph[u].begin();i!=graph[u].end();++i)
        {  
            if(recStack[*i]) //if vertice v is found in recursion stack of this DFS traversal
                return true;
            else if(*i==u) //if there's an edge from the vertex to itself
                return true;
            else if(!visited[*i])
            {   if(helper(graph, *i, visited, recStack))
                   return true;
            }
        }
        recStack[u]=false;
        return false;
    }
    /*
    /The wrapper function calls helper function on each vertices which have not been visited. Helper function returns true if it detects a back edge in the subgraph(tree) or false.
*/
    bool isCyclic(list<int> *graph, int V)
    {
      bool visited[V];  //array to track vertices already visited
      bool recStack[V]; //array to track vertices in recursion stack of the traversal.

      for(int i = 0;i<V;i++)
       visited[i]=false, recStack[i]=false;  //initialize all vertices as not visited and not recursed

      for(int u = 0; u < V; u++) //Iteratively checks if every vertices have been visited
      {   if(visited[u]==false)
          {  if(helper(graph, u, visited, recStack)) //checks if the DFS tree from the vertex contains a cycle
               return true;
          }
      }
       return false;
    }
    /*
    Driver function
    */
    int main()
    {
        list<int>* graph = new list<int>[NUM_V];
        graph[0].push_back(1);
        graph[0].push_back(2);
        graph[1].push_back(2);
        graph[2].push_back(0);
        graph[2].push_back(3);
        graph[3].push_back(3);
        bool res = isCyclic(graph, NUM_V);
        cout<<res<<endl;
    }

Sonuç: Aşağıda gösterildiği gibi, grafikte üç arka kenar vardır. Köşe 0 ile 2 arasında bir; köşe 0, 1 ve 2 arasında; ve köşe 3. Aramanın zaman karmaşıklığı O(V+E)‘dir, burada V köşe sayısı ve E kenar sayısıdır.
