Dinamik Zaman Bükülmesine Giriş #

Dynamic Time Warping(DTW), hızı değişebilen iki zamansal dizi arasındaki benzerliği ölçmek için kullanılan bir algoritmadır. Örneğin, bir kişi diğerinden daha hızlı yürüse veya bir gözlem sırasında hızlanma ve yavaşlama olsa bile, yürümedeki benzerlikler DTW kullanılarak tespit edilebilir. Kişi önceden kaydedilmiş örnek sesten daha hızlı veya daha yavaş konuşsa bile, örnek bir sesli komutu diğer komutlarla eşleştirmek için kullanılabilir. DTW, video, ses ve grafik verilerinin zamansal dizilerine uygulanabilir - gerçekten de doğrusal bir diziye dönüştürülebilen herhangi bir veri DTW ile analiz edilebilir.

Genel olarak DTW, belirli kısıtlamalarla verilen iki dizi arasındaki optimal eşleşmeyi hesaplayan bir yöntemdir. Ama burada daha basit noktalara bağlı kalalım. Diyelim ki iki ses dizimiz var Örnek ve Test ve bu iki dizinin eşleşip eşleşmediğini kontrol etmek istiyoruz. Burada ses dizisi, sesinizin dönüştürülmüş dijital sinyalini ifade eder. Söylediğiniz kelimeleri ifade eden sesinizin genliği veya frekansı olabilir. Diyelim ki:

Sample = {1, 2, 3, 5, 5, 5, 6}
Test   = {1, 1, 2, 2, 3, 5}

Bu iki dizi arasındaki en uygun eşleşmeyi bulmak istiyoruz.

İlk önce iki nokta arasındaki mesafeyi tanımlarız, d(x, y) burada x ve y iki noktayı temsil eder. İzin vermek,

d(x, y) = |x - y|     //absolute difference

Bu iki diziyi kullanarak bir 2B matris Tablo oluşturalım. Test‘in her noktasıyla Örnek‘in her noktası arasındaki mesafeleri hesaplayacağız ve aralarındaki en uygun eşleşmeyi bulacağız.

+------+------+------+------+------+------+------+------+
|      |   0  |   1  |   1  |   2  |   2  |   3  |   5  |
+------+------+------+------+------+------+------+------+
|   0  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   1  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   2  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   3  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   6  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+

Burada Tablo[i][j], Örnek[i] ve Test[j]‘ye kadar olan diziyi düşünürsek, iki dizi arasındaki en uygun mesafeyi temsil eder. daha önce gözlemlediğimiz tüm optimal mesafeler.

İlk satır için, Örnek’ten hiçbir değer almazsak, bununla Test arasındaki mesafe sonsuz olacaktır. Bu yüzden ilk satıra sonsuz koyduk. Aynı şey ilk sütun için de geçerli. Test’ten hiçbir değer almazsak, bununla Örnek arasındaki mesafe de sonsuz olacaktır. Ve 0 ile 0 arasındaki mesafe basitçe 0 olacaktır. anladık

+------+------+------+------+------+------+------+------+
|      |   0  |   1  |   1  |   2  |   2  |   3  |   5  |
+------+------+------+------+------+------+------+------+
|   0  |   0  |  inf |  inf |  inf |  inf |  inf |  inf |
+------+------+------+------+------+------+------+------+
|   1  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   2  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   3  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   6  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+

Şimdi her adım için, ilgili her nokta arasındaki mesafeyi dikkate alacağız ve şimdiye kadar bulduğumuz minimum mesafeyle ekleyeceğiz. Bu bize o pozisyona kadar olan iki dizinin en uygun mesafesini verecektir. Formülümüz şu şekilde olacak,

Table[i][j] := d(i, j) + min(Table[i-1][j], Table[i-1][j-1], Table[i][j-1])

İlki için, d(1, 1) = 0, Tablo[0][0] minimumu temsil eder. Dolayısıyla Tablo[1][1] değeri 0 + 0 = 0 olacaktır. İkincisi için, d(1, 2) = 0. Tablo[1][1] minimum değeri temsil eder. Değer şöyle olacaktır: Tablo[1][2] = 0 + 0 = 0. Bu şekilde devam edersek bitirdikten sonra tablo aşağıdaki gibi olacaktır.

+------+------+------+------+------+------+------+------+
|      |   0  |   1  |   1  |   2  |   2  |   3  |   5  |
+------+------+------+------+------+------+------+------+
|   0  |   0  |  inf |  inf |  inf |  inf |  inf |  inf |
+------+------+------+------+------+------+------+------+
|   1  |  inf |   0  |   0  |   1  |   2  |   4  |   8  |
+------+------+------+------+------+------+------+------+
|   2  |  inf |   1  |   1  |   0  |   0  |   1  |   4  |
+------+------+------+------+------+------+------+------+
|   3  |  inf |   3  |   3  |   1  |   1  |   0  |   2  |
+------+------+------+------+------+------+------+------+
|   5  |  inf |   7  |   7  |   4  |   4  |   2  |   0  |
+------+------+------+------+------+------+------+------+
|   5  |  inf |  11  |  11  |   7  |   7  |   4  |   0  |
+------+------+------+------+------+------+------+------+
|   5  |  inf |  15  |  15  |  10  |  10  |   6  |   0  |
+------+------+------+------+------+------+------+------+
|   6  |  inf |  20  |  20  |  14  |  14  |   9  |   1  |
+------+------+------+------+------+------+------+------+

Tablo[7][6]‘daki değer, bu iki dizi arasındaki maksimum mesafeyi temsil eder. Burada 1, Örnek ile Test arasındaki maksimum mesafeyi temsil eder 1.

Şimdi son noktadan başlayarak (0, 0) noktasına kadar geriye gidersek, yatay, dikey ve çapraz hareket eden uzun bir çizgi elde ederiz. Geri izleme prosedürümüz şöyle olacaktır:

if Table[i-1][j-1] <= Table[i-1][j] and Table[i-1][j-1] <= Table[i][j-1]
    i := i - 1
    j := j - 1
else if Table[i-1][j] <= Table[i-1][j-1] and Table[i-1][j] <= Table[i][j-1]
    i := i - 1
else
    j := j - 1
end if

(0, 0)‘a ulaşana kadar buna devam edeceğiz. Her hareketin kendi anlamı vardır:

  • Yatay bir hareket, silmeyi temsil eder. Bu, Test dizimizin bu aralıkta hızlandığı anlamına gelir.
  • Dikey bir hareket, yerleştirmeyi temsil eder. Bu, Test dizisinin bu aralıkta yavaşladığı anlamına gelir.
  • Çapraz bir hareket eşleşmeyi temsil eder. Bu dönemde Test ve Örnek aynıydı.

Sözde kodumuz şöyle olacaktır:

Procedure DTW(Sample, Test):
n := Sample.length
m := Test.length
Create Table[n + 1][m + 1]
for i from 1 to n
    Table[i][0] := infinity
end for
for i from 1 to m
    Table[0][i] := infinity
end for
Table[0][0] := 0
for i from 1 to n
    for j from 1 to m
        Table[i][j] := d(Sample[i], Test[j])
                       + minimum(Table[i-1][j-1],      //match
                                 Table[i][j-1],        //insertion
                                 Table[i-1][j])        //deletion
    end for
end for
Return Table[n + 1][m + 1]

Ayrıca bir yerellik kısıtlaması da ekleyebiliriz. Yani, “Örnek[i]”, “Test[j]” ile eşleştirilirse, “|i - j|” bir pencere parametresi olan w‘den büyük değildir.

Karmaşıklık:

DTW hesaplamanın karmaşıklığı O(m * n)‘dir, burada m ve n her dizinin uzunluğunu temsil eder. DTW hesaplamak için daha hızlı teknikler arasında PrunedDTW, SparseDTW ve FastDTW bulunur.
