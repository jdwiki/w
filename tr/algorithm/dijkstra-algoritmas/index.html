Dijkstra’nın En Kısa Yol Algoritması #

Devam etmeden önce Bitişiklik Matrisi ve BFS hakkında kısa bir fikir edinmenizi tavsiye ederiz

Dijkstra’nın algoritması tek kaynaklı en kısa yol algoritması olarak bilinir. Örneğin yol ağlarını temsil edebilen bir grafikteki düğümler arasındaki en kısa yolları bulmak için kullanılır. Edsger W. Dijkstra tarafından 1956’da tasarlandı ve üç yıl sonra yayınlandı.

En kısa yolu Genişlik İlk Arama (BFS) arama algoritmasını kullanarak bulabiliriz. Bu algoritma iyi çalışıyor, ancak sorun şu ki, her yolu geçme maliyetinin aynı olduğunu varsayıyor, bu da her bir kenarın maliyetinin aynı olduğu anlamına geliyor. Dijkstra’nın algoritması, her yolun maliyetinin aynı olmadığı en kısa yolu bulmamıza yardımcı olur.

İlk başta, Dijkstra’nın algoritmasını yazmak için BFS’nin nasıl değiştirileceğini göreceğiz, ardından onu eksiksiz bir Dijkstra’nın algoritması yapmak için öncelik sırası ekleyeceğiz.

Diyelim ki her bir düğümün kaynağa olan uzaklığı d[] dizisinde tutuluyor. Olduğu gibi, d[3], kaynak’tan düğüm 3’e ulaşmak için d[3] zamanının alındığını gösterir. Mesafeyi bilmiyorsak, sonsuzluğu d[3] içinde saklayacağız. Ayrıca, maliyet[u][v]’nin u-v’nin maliyetini temsil etmesine izin verin. Bu, u düğümünden v düğümüne geçmenin maliyet[u][v] gerektirdiği anlamına gelir.

Edge Relaxation’ı anlamamız gerekiyor. Diyelim ki evinizden, yani kaynak, A yerine gitmek 10 dakika sürüyor. Ve B yerine gitmek 25 dakika sürer. Sahibiz,

d[A] = 10
d[B] = 25

Şimdi A noktasından B noktasına gitmenin 7 dakika sürdüğünü varsayalım, bu şu anlama gelir:

cost[A][B] = 7

Daha sonra kaynak’tan B yerine kaynak’tan A yerine ve sonra A‘dan B noktasına giderek gidebiliriz. , 25 dakika yerine 10 + 7 = 17 dakika sürecektir. Yani,

d[A] + cost[A][B] < d[B]

Sonra güncelliyoruz,

d[B] = d[A] + cost[A][B]

Buna gevşeme denir. u düğümünden v düğümüne gideceğiz ve eğer d[u] + maliyet[u][v] < d[v] ise d[v]‘yi güncelleyeceğiz = d[u] + maliyet[u][v].

BFS’de herhangi bir düğümü iki kez ziyaret etmemize gerek yoktu. Yalnızca bir düğümün ziyaret edilip edilmediğini kontrol ettik. Ziyaret edilmemişse, düğümü kuyruğa ittik, ziyaret edildi olarak işaretledik ve mesafeyi 1 artırdık. Dijkstra’da bir düğümü kuyruğa itebiliriz ve onu ziyaret edilen ile güncellemek yerine rahatlatır veya yenisini güncelleriz. kenar. Bir örneğe bakalım:

Düğüm 1‘in Kaynak olduğunu varsayalım. O zamanlar,

d[1] = 0
d[2] = d[3] = d[4] = infinity (or a large value)

d[2], d[3] ve d[4]‘yi sonsuz olarak belirledik çünkü henüz mesafeyi bilmiyoruz. Ve kaynağın mesafesi elbette 0. Şimdi, source‘dan diğer düğümlere gidiyoruz ve eğer onları güncelleyebilirsek, onları kuyruğa iteceğiz. Örneğin, kenar 1-2‘yi geçeceğimizi varsayalım. d[1] + 2 < d[2] olarak, bu da d[2] = 2 yapacaktır. Benzer şekilde, d[3] = 5 yapan kenar 1-3‘ü geçeceğiz.

5‘in 3. düğüme gitmek için geçebileceğimiz en kısa mesafe olmadığını açıkça görebiliriz. Bu nedenle, BFS gibi bir düğümü yalnızca bir kez geçmek burada çalışmaz. kenar 2-3‘ü kullanarak düğüm 2‘den düğüm 3’e gidersek, d[3] = d[2] + 1 = 3‘ü güncelleyebiliriz. Böylece bir düğümün birçok kez güncellenebileceğini görebiliriz. kaç kere soruyorsun Bir düğümün güncellenebileceği maksimum sayı, bir düğümün derece sayısıdır.

Herhangi bir düğümü birden çok kez ziyaret etmek için sözde kodu görelim. BFS’yi basitçe değiştireceğiz:

procedure BFSmodified(G, source):
Q = queue()
distance[] = infinity
Q.enqueue(source)
distance[source]=0
while Q is not empty
    u <- Q.pop()
    for all edges from u to v in G.adjacentEdges(v) do
        if distance[u] + cost[u][v] < distance[v]
            distance[v] = distance[u] + cost[u][v]
        end if
    end for
end while
Return distance

Bu, kaynaktan tüm düğümlerin en kısa yolunu bulmak için kullanılabilir. Bu kodun karmaşıklığı o kadar iyi değil. İşte neden,

BFS’de node 1‘den diğer tüm node’lara gittiğimizde ilk gelen ilk hizmet yöntemini izliyoruz. Örneğin, 2. düğümü işlemeden önce kaynak tan 3. düğüme gittik. Kaynak’tan 3. düğüme gidersek, 4. düğümü 5 + 3 = 8 olarak güncelleriz. düğüm 3‘ü düğüm 2‘den tekrar güncellediğimizde, düğüm 4‘ü tekrar 3 + 3 = 6 olarak güncellememiz gerekiyor! Böylece düğüm 4 iki kez güncellenir.

Dijkstra önerildi, İlk gelen, ilk hizmet yöntemine gitmek yerine, önce en yakın düğümleri güncellersek, daha az güncelleme alacaktır. Daha önce düğüm 2‘yi işleseydik, düğüm 3 daha önce güncellenirdi ve buna göre düğüm 4 güncellendikten sonra, en kısa mesafeyi kolayca elde ederdik! Buradaki fikir, kaynağa en yakın olan kuyruktan, düğümden seçim yapmaktır. Bu yüzden burada Priority Queue kullanacağız, böylece kuyruğu açtığımızda bize source‘dan en yakın u düğümünü getirecektir. Bunu nasıl yapacak? Bununla d[u] değerini kontrol edecektir.

Sözde kodu görelim:

procedure dijkstra(G, source):
Q = priority_queue()
distance[] = infinity
Q.enqueue(source)
distance[source] = 0
while Q is not empty
    u <- nodes in Q with minimum distance[]
    remove u from the Q
    for all edges from u to v in G.adjacentEdges(v) do
        if distance[u] + cost[u][v] < distance[v]
            distance[v] = distance[u] + cost[u][v]
            Q.enqueue(v)
        end if
    end for
end while
Return distance

Sözde kod, diğer tüm düğümlerin kaynağa olan mesafesini döndürür. Tek bir v düğümünün mesafesini bilmek istiyorsak, sıradan v kuyruktan çıktığında değeri döndürebiliriz.

Şimdi, Dijkstra’nın Algoritması olumsuz bir kenar olduğunda çalışır mı? Negatif bir döngü varsa, her seferinde maliyeti düşürmeye devam edeceği için sonsuzluk döngüsü oluşacaktır. Negatif bir kenar olsa bile, hedef patlatıldıktan hemen sonra geri dönmediğimiz sürece Dijkstra çalışmayacaktır. Ama o zaman, bir Dijkstra algoritması olmayacak. [Bellman–Ford](https://www.wikiod.com/tr/algorithm/bellman-ford-algoritmas#Tek Kaynaklı En Kısa Yol Algoritması (Grafikte negatif bir döngü olduğu göz önüne alındığında) ihtiyacımız olacak Negatif kenarı/döngüyü işlemek için negatif döngü) algoritması.

Karmaşıklık:

BFS’nin karmaşıklığı O(log(V+E))‘dir, burada V düğüm sayısı ve E kenar sayısıdır. Dijkstra için karmaşıklık benzerdir, ancak Priority Queue sıralaması O(logV) alır. Yani toplam karmaşıklık: O(Vlog(V)+E)

Aşağıda, Dijkstra’nın En Kısa Yol Algoritmasını Bitişiklik Matrisi kullanarak çözmek için bir Java örneği verilmiştir.

import java.util.*;
import java.lang.*;
import java.io.*;

class ShortestPath
{
    static final int V=9;
    int minDistance(int dist[], Boolean sptSet[])
    {
        int min = Integer.MAX_VALUE, min_index=-1;

        for (int v = 0; v < V; v++)
            if (sptSet[v] == false && dist[v] <= min)
            {
                min = dist[v];
                min_index = v;
            }

        return min_index;
    }

    void printSolution(int dist[], int n)
    {
        System.out.println("Vertex Distance from Source");
        for (int i = 0; i < V; i++)
            System.out.println(i+" \t\t "+dist[i]);
    }

    void dijkstra(int graph[][], int src)
    {

        Boolean sptSet[] = new Boolean[V];

        for (int i = 0; i < V; i++)
        {
            dist[i] = Integer.MAX_VALUE;
            sptSet[i] = false;
        }

        dist[src] = 0;

        for (int count = 0; count < V-1; count++)
        {
            int u = minDistance(dist, sptSet);

            sptSet[u] = true;

            for (int v = 0; v < V; v++)

                if (!sptSet[v] && graph[u][v]!=0 &&
                        dist[u] != Integer.MAX_VALUE &&
                        dist[u]+graph[u][v] < dist[v])
                    dist[v] = dist[u] + graph[u][v];
        }

        printSolution(dist, V);
    }

    public static void main (String[] args)
    {
    int graph[][] = new int[][]{{0, 4, 0, 0, 0, 0, 0, 8, 0},
                                {4, 0, 8, 0, 0, 0, 0, 11, 0},
                                {0, 8, 0, 7, 0, 4, 0, 0, 2},
                                {0, 0, 7, 0, 9, 14, 0, 0, 0},
                                {0, 0, 0, 9, 0, 10, 0, 0, 0},
                                {0, 0, 4, 14, 10, 0, 2, 0, 0},
                                {0, 0, 0, 0, 0, 2, 0, 1, 6},
                                {8, 11, 0, 0, 0, 0, 1, 0, 7},
                                {0, 0, 2, 0, 0, 0, 6, 7, 0}
                                };
        ShortestPath t = new ShortestPath();
        t.dijkstra(graph, 0);
    }
}
