Gezgin Satıcı Problemi, N köşeleri boyunca tam olarak bir köşe başına seyahat etmenin minimum maliyetini bulma problemidir. ‘i’ köşesinden ‘j’ köşesine seyahat etmenin bir ‘maliyeti[i][j]’ vardır.

Bu sorunu çözmek için 2 tür algoritma vardır: Tam Algoritmalar ve Yaklaşık Algoritmalar

Tam Algoritmalar

  1. Kaba Kuvvet Algoritması
  2. Dinamik Programlama Algoritması

Yaklaşım Algoritmaları

Eklenecek

Kaba Kuvvet Algoritması #

Her tepe noktasından tam olarak bir kez geçen bir yol, tepe noktasını bir şekilde sıralamakla aynıdır. Böylece, her köşeden tam olarak bir kez geçmenin minimum maliyetini hesaplamak için, ‘1’den ‘N’ye kadar olan sayıların ‘N!’ permütasyonlarının her birini kaba kuvvet uygulayabiliriz.

Sahte kod

minimum = INF
for all permutations P

    current = 0                                

    for i from 0 to N-2
        current = current + cost[P[i]][P[i+1]]  <- Add the cost of going from 1 vertex to the next
    
    current = current + cost[P[N-1]][P[0]]      <- Add the cost of going from last vertex to the first

    if current < minimum                        <- Update minimum if necessary
        minimum = current
    
output minimum

Zaman Karmaşıklığı

Geçilmesi gereken ‘N!’ permütasyonları vardır ve her yolun maliyeti ‘O(N)’ cinsinden hesaplanır, bu nedenle bu algoritmanın kesin cevabı vermesi ‘O(N * N!)’ zamanını alır.

Dinamik Programlama Algoritması #

Yolu (sırasıyla) düşünürsek:

(1,2,3,4,6,0,5,7)

ve yol

(1,2,3,5,0,6,7,4)

‘1’ tepe noktasından ‘2’ tepe noktasına ve ‘3’ tepe noktasına gitmenin maliyeti aynı kalıyor, öyleyse neden yeniden hesaplanması gerekiyor? Bu sonuç daha sonra kullanılmak üzere kaydedilebilir.

“dp[bitmask][vertex]”, “bitmask” içindeki karşılık gelen biti “vertex” ile biten “1” olarak ayarlanmış tüm tepe noktalarında seyahat etmenin minimum maliyetini temsil etsin. Örneğin:

dp[12][2] 

   12   =   1 1 0 0
            ^ ^ 
vertices:   3 2 1 0

“12” ikili dosyada “1100"ü temsil ettiğinden, “dp[12][2]”, yol 2’de biten grafikte “2” ve “3” köşelerinden geçmeyi temsil eder.

Böylece aşağıdaki algoritmaya sahip olabiliriz (C++ uygulaması):

int cost[N][N];                        //Adjust the value of N if needed
int memo[1 << N][N];                   //Set everything here to -1
int TSP(int bitmask, int pos){
    int cost = INF;
    if (bitmask == ((1 << N) - 1)){      //All vertices have been explored
        return cost[pos][0];             //Cost to go back
    }
    if (memo[bitmask][pos] != -1){       //If this has already been computed
        return memo[bitmask][pos];       //Just return the value, no need to recompute
    }
    for (int i = 0; i < N; ++i){         //For every vertex 
        if ((bitmask & (1 << i)) == 0){  //If the vertex has not been visited
            cost = min(cost,TSP(bitmask | (1 << i) , i) + cost[pos][i]);   //Visit the vertex
        } 
    }
    memo[bitmask][pos] = cost;           //Save the result
    return cost;
}
//Call TSP(1,0)

Bu satır biraz kafa karıştırıcı olabilir, o yüzden yavaş yavaş üzerinden geçelim:

cost = min(cost,TSP(bitmask | (1 << i) , i) + cost[pos][i]);

İşte, bit maskesi | (1 << i), ‘bitmask’in i. bitini 1’e ayarlar; bu, i. tepe noktasının ziyaret edildiğini gösterir. Virgülden sonraki “i”, bu işlev çağrısındaki yeni “son” köşeyi temsil eden yeni “pos"u temsil eder. “cost[pos][i]”, “pos” köşesinden “i” köşesine seyahat etmenin maliyetini eklemektir.

Böylece, bu çizgi, henüz ziyaret edilmemiş diğer tüm köşelere seyahat etmenin mümkün olan minimum değerine “maliyet” değerini güncellemek içindir.

Zaman Karmaşıklığı

“TSP(bitmask,pos)” işlevi, “bitmask” için “2^N” ve “kon” için “N” değerlerine sahiptir. Her işlevin çalışması için “O(N)” süresi gerekir (“for” döngüsü). Bu nedenle, bu uygulamanın kesin cevabı vermesi O(N^2 * 2^N) zamanını alır.
