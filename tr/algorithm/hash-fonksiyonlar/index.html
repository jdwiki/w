C#‘daki yaygın türler için karma kodları #

Yerleşik için GetHashCode() yöntemi tarafından üretilen karma kodlar ve Sistem ad alanından yaygın C# türleri gösterilir. aşağıda.

Boole #

Değer doğruysa 1, değilse 0.

Byte, [UInt16](https:// github.com/dotnet/coreclr/blob/release/1.1.0/src/mscorlib/src/System/UInt16.cs#L66), [Int32](https://github.com/dotnet/coreclr/blob/release /1.1.0/src/mscorlib/src/System/Int32.cs#L76), [UInt32](https://github.com/dotnet/coreclr/blob/release/1.1.0/src/mscorlib/src/ System/UInt32.cs#L77), Single #

Değer (gerekirse Int32’ye aktarılır).

SByte #

((int)m_value ^ (int)m_value << 8);

Char #

(int)m_value ^ ((int)m_value << 16);

Int16 #

((int)((ushort)m_value) ^ (((int)m_value) << 16));

Int64, [Çift](https:// github.com/dotnet/coreclr/blob/release/1.1.0/src/mscorlib/src/System/Double.cs#L191) #

64 bit sayının alt ve üst 32 biti arasında Xor

(unchecked((int)((long)m_value)) ^ (int)(m_value >> 32));

UInt64, [DateTime](https:// github.com/dotnet/coreclr/blob/release/1.1.0/src/mscorlib/src/System/DateTime.cs#L824), [TimeSpan](https://github.com/dotnet/coreclr/blob/release /1.1.0/src/mscorlib/src/System/TimeSpan.cs#L215) #

((int)m_value) ^ (int)(m_value >> 32);

Ondalık #

((((int *)&dbl)[0]) & 0xFFFFFFF0) ^ ((int *)&dbl)[1];

Nesne #

RuntimeHelpers.GetHashCode(this);

Varsayılan uygulama senkronizasyon blok dizini kullanılır.

Dize #

Karma kodu hesaplaması, platform türüne (Win32 veya Win64), rastgele dize karma kullanma özelliğine, Hata Ayıklama / Yayın moduna bağlıdır. Win64 platformu olması durumunda:

int hash1 = 5381;
int hash2 = hash1;
int c;
char *s = src;
while ((c = s[0]) != 0) {
    hash1 = ((hash1 << 5) + hash1) ^ c;
    c = s[1];
    if (c == 0)
        break;
    hash2 = ((hash2 << 5) + hash2) ^ c;
    s += 2;
}
return hash1 + (hash2 * 1566083941);

ValueType #

Statik olmayan ilk alan, hash kodunu aramak ve almaktır. Türün statik olmayan alanları yoksa, türün karma kodu döner. Statik bir üyenin karma kodu alınamaz çünkü bu üye orijinal türle aynı türdeyse, hesaplama sonsuz bir döngüde sona erer.

Nullable<T> #

return hasValue ? value.GetHashCode() : 0;

Dizi #

int ret = 0;
for (int i = (Length >= 8 ? Length - 8 : 0); i < Length; i++) 
{
    ret = ((ret << 5) + ret) ^ comparer.GetHashCode(GetValue(i));
}

Hash fonksiyonlarına giriş #

Hash işlevi ‘h()’, keyfi boyuttaki ‘x ∈ X’ verilerini sabit boyuttaki ‘y ∈ Y’ değerine eşleyen isteğe bağlı bir işlevdir: ‘y = h(x)’. İyi karma işlevleri aşağıdaki kısıtlamalara sahiptir:

  • hash fonksiyonları düzgün dağılım gibi davranır

  • hash fonksiyonları deterministiktir. “h(x)”, belirli bir “x” için her zaman aynı değeri döndürmelidir

  • hızlı hesaplama (çalışma zamanı O(1) vardır)

Genel olarak, hash fonksiyonunun boyutu girdi verisinin boyutundan daha küçüktür: |y| < |x|. Hash fonksiyonları tersine çevrilemez veya başka bir deyişle çarpışma olabilir: ∃ x1, x2 ∈ X, x1 ≠ x2: h(x1) = h(x2). ‘X’ sonlu veya sonsuz küme olabilir ve ‘Y’ sonlu kümedir.

Hash işlevleri, örneğin yazılım mühendisliği, kriptografi, veritabanları, ağlar, makine öğrenimi vb. gibi bilgisayar biliminin birçok bölümünde kullanılmaktadır. Farklı etki alanına özgü özelliklere sahip birçok farklı türde karma işlevi vardır.

Genellikle karma bir tamsayı değeridir. Hash hesaplamak için programlama dillerinde özel yöntemler vardır. Örneğin, ‘C#’ içinde ‘GetHashCode()’ yöntemi tüm türler için ‘Int32’ değerini (32 bit tam sayı) döndürür. ‘Java’da her sınıf, ‘int’ döndüren ‘hashCode()’ yöntemini sağlar. Her veri türünün kendi veya kullanıcı tanımlı uygulamaları vardır.

Hash yöntemleri #

Hash fonksiyonunu belirlemek için birkaç yaklaşım vardır. Genelliği kaybetmeden, x ∈ X = {z ∈ ℤ: z ≥ 0} pozitif tamsayılar olsun. Genellikle “m” asaldır (2’nin tam kuvvetine çok yakın değildir).

Yöntem Karma işlevi
Bölme yöntemi h(x) = x mod m
Çarpma yöntemi h(x) = ⌊m (xA mod 1)⌋, A ∈ {z ∈ ℝ: 0 < z < 1}

Hash tablosu #

Bir dizi yuvaya indeks hesaplamak için hash tablolarında kullanılan hash fonksiyonları. Hash tablosu, sözlükleri uygulamak için veri yapısıdır (anahtar-değer yapısı). İyi uygulanmış karma tabloların sonraki işlemler için O(1) süresi vardır: anahtarla veri ekleme, arama ve silme. Aynı yuvaya birden fazla anahtar hash olabilir. Çarpışmayı çözmenin iki yolu vardır:

  1. Zincirleme: bağlantılı liste, slotta aynı hash değerine sahip elemanları depolamak için kullanılır

  2. Açık adresleme: her yuvada sıfır veya bir eleman saklanır

Sonraki yöntemler, açık adresleme için gerekli prob dizilerini hesaplamak için kullanılır.

Yöntem formül
Doğrusal problama h(x, i) = (h'(x) + i) mod m
İkinci dereceden sondalama h(x, i) = (h'(x) + c1*i + c2*i^2) mod m
Çift karma h(x, i) = (h1(x) + i*h2(x)) mod m

‘i ∈ {0, 1, …, m-1}’, ‘h’(x), h1(x), h2(x)’ yardımcı özet fonksiyonları olduğunda, ‘c1, c2’ pozitif yardımcı sabitlerdir .
