Sözdizimi #

  • CREATE [GEÇİCİ] [HARİCİ] TABLO [VAR DEĞİLSE] [db_name.]table_name

    [(col_name data_type [COMMENT col_comment], ...)]

[YORUM tablosu_yorum] [PARTITIONED BY (sütun_adı veri_türü [COMMENT sütun_yorum], …)] [KÜMELENEN (sütun_adı, sütun_adı, …) [SIRALI (sütun_adı [ASC|DESC], …)] num_buckets BUCKETS’E GÖRE] [SKEWED BY (col_name, col_name, …) – (Not: Hive 0.10.0 ve sonraki sürümlerde mevcuttur)] ON ((col_value, col_value, …), (col_value, col_value, …), …) [STORED AS DIRECTORIES] [ [SATIR BİÇİMİ satır_biçimi] [file_format OLARAK DEPOLANIR] | STORED BY ‘storage.handler.class.name’ [WITH SERDEPROPERTIES (…)] ] [LOCATION hdfs_path] [TBLPROPERTIES (özellik_adı=özellik_değeri, …)] [AS select_deyimi];

  • CREATE [GEÇİCİ] [HARİCİ] TABLO [VAR DEĞİLSE] [db_name.]table_name MEVCUT_table_or_view_name GİBİ [LOCATION hdfs_path];
  • data_type : ilkel_type,array_type,map_type,struct_type,union_type
  • ilkel_tür: TINYINT, SMALLINT, INT , BIGINT, BOOLEAN, FLOAT, DOUBLE, STRING, BINARY, TIMESTAMP, DECIMAL, DECIMAL(hassas, ölçek), DATE, VARCHAR, CHAR
  • dizi_türü: ARRAY < veri_türü >
  • map_type: HARİTA < ilkel_türü, veri_türü >
  • struct_type: STRUCT < col_name : data_type [COMMENT col_comment], …>
  • union_type: UNIONTYPE < veri_türü, veri_türü, … >
  • row_format: DELIMITED [char [char] TARAFINDAN SONLANDIRILDI ALANLAR]] [char TARAFINDAN SONLANDIRILDIKOLEKSİYON ÖĞELERİ] [MAP KEYS TERMINATED BY char] [LINES TERMINATED BY char] [NULL DEFINED AS char]
    , SERDE serde_adı [SERDE ÖZELLİKLERİYLE (özellik_adı=özellik_değeri, mülk_adı=özellik_değeri, …)]
  • dosya formatı: : SEQUENCEFILE , TEXTFILE , RCFILE , ORC , PARKE , AVRO , INPUTFORMAT input_format_classname OUTPUTFORMAT output_format_classname
  • CREATE (VERİTABANI|ŞEMA) [VAR DEĞİLSE] veritabanı_adı [COMMENT database_comment] [LOCATION hdfs_path] [DBPROPERTIES İLE (özellik_adı=özellik_değeri, …)];

HIVE’da tablolar ve veritabanlarıyla çalışırken. Aşağıdaki noktalar yararlı olabilir.

  • Veritabanını kullanarak değiştirebiliriz use database; command
  • Mevcut çalışan veritabanını bilmek için SELECT current_database() kullanarak alabiliriz
  • Tablo deyimi oluşturmak için kullanılan DDL’yi görmek için SHOW CREATE TABLE tablename kullanabiliriz.
  • Tablonun tüm sütunlarını görmek için kullanılan konum gibi genişletilmiş ayrıntıları göstermek için ’tablo adını AÇIKLA’yı kullanın serde ve diğerleri ‘Biçimlendirilen tablo adını AÇIKLAYIN’. DESCRIBE, DESC olarak da kısaltılabilir.

Tablo Oluştur #

Bölümlü ve sıra dosyası olarak saklanan yönetilen bir tablo oluşturma. Dosyalardaki veri formatının, ‘Ctrl-A (^A)’ ile alanla ve satırla yeni satırla sınırlandırıldığı varsayılır. Aşağıdaki tablo, Hive yapılandırma dosyası ‘Hive-site.xml’ içindeki ‘hive.metastore.warehouse.dir’ anahtarının değerinde belirtilen Hive ambar dizininde oluşturulur.

CREATE TABLE view
(time INT, 
id BIGINT,
url STRING, 
referrer_url STRING,
add STRING COMMENT 'IP of the User')
COMMENT 'This is view table'
PARTITIONED BY(date STRING, region STRING)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY '\001'
STORED AS SEQUENCEFILE;

Bölümler içeren ve bir sıra dosyası olarak saklanan bir harici tablo oluşturma. Dosyalardaki veri formatının ctrl-A ile alanla ve satırla yeni satırla sınırlandırıldığı varsayılır. Aşağıdaki tablo belirtilen konumda oluşturulmuştur ve elimizde veri olduğunda kullanışlıdır. Harici bir tablo kullanmanın avantajlarından biri, verileri silmeden tabloyu bırakabilmemizdir. Örneğin, bir tablo oluşturup şemanın yanlış olduğunu fark edersek, tabloyu güvenle bırakabilir ve veriler hakkında endişelenmeden yeni şema ile yeniden oluşturabiliriz. Diğer bir avantajı da, eğer aynı dosyalar üzerinde domuz gibi başka araçlar kullanıyorsak, tabloyu sildikten sonra bile bunları kullanmaya devam edebiliriz.

CREATE EXTERNAL TABLE view
(time INT, 
id BIGINT,
url STRING, 
referrer_url STRING,
add STRING COMMENT 'IP of the User')
COMMENT 'This is view table'
PARTITIONED BY(date STRING, region STRING)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY '\001'
STORED AS SEQUENCEFILE
LOCATION '<hdfs_location>';

Seçme sorgusu kullanarak tablo oluşturma ve sorgudan sonuçları doldurma, these statements are known as CTAS(Create Table As Select).

CTAS’ta iki bölüm vardır, SELECT bölümü HiveQL tarafından desteklenen herhangi bir SELECT ifadesi olabilir. CTAS’ın CREATE kısmı, SELECT kısmından elde edilen şemayı alır ve SerDe ve depolama formatı gibi diğer tablo özellikleri ile hedef tabloyu oluşturur.

CTA’nın şu kısıtlamaları vardır:

  • Hedef tablo, bölümlenmiş bir tablo olamaz.

  • Hedef tablo harici bir tablo olamaz.

  • Hedef tablo, bir liste kovalama tablosu olamaz.

    CREATE TABLE new_key_value_store ROW FORMAT SERDE “org.apache.hadoop.hive.serde2.columnar.ColumnarSerDe” STORED AS RCFile AS SELECT * FROM page_view SORT BY url, add;

Gibi Tablo Oluştur:

CREATE TABLE**‘ın **LIKE formu, mevcut bir tablo tanımını tam olarak (verilerini kopyalamadan) kopyalamanıza olanak tanır. CTAS’ın aksine, aşağıdaki ifade, tanımı tablo adı dışındaki tüm ayrıntılarda mevcut tabloyla tam olarak eşleşen yeni bir tablo oluşturur. Yeni tablo satır içermiyor.

CREATE TABLE empty_page_views
LIKE page_views;

Veritabanı yarat #

Belirli bir konumda bir veritabanı oluşturma. Depo dizininde oluşturulan veritabanı için herhangi bir konum belirtmezsek.

CREATE DATABASE IF NOT EXISTS db_name 
COMMENT 'TEST DATABASE'
LOCATION /PATH/HDFS/DATABASE/;

HIVE_HBASE Entegrasyonu #

Hive-Hbase entegrasyonu aşağıdaki sürümlerden beri desteklenmektedir. Kovan: 0.11.0 HTemel: 0.94.2 Hadoop: 0.20.2

CREATE TABLE hbase_hive
(id string,
 col1 string,
 col2 string,
 col3 int) 
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES 
("hbase.columns.mapping" = ":key,cf1:col1,cf1:col2,cf1:col3")
TBLPROPERTIES ("hbase.table.name" = "hive_hbase");

Not: 1. sütun anahtar sütun olmalıdır.

Mevcut tablo özelliklerini kullanarak tablo oluşturun. #

CREATE TABLE new_table_name LIKE existing_table_name;

Hive ACID tablosu oluşturma. #

ASİT tabloları Hive 0.14 sürümünden beri desteklenmektedir. Aşağıdaki tablo UPDATE/DELETE/INSERT’i destekler

Hive-site.xml’de aşağıdaki yapılandırma değişiklikleri gerekir.

 hive.support.concurrency = true
 hive.enforce.bucketing = true
 hive.exec.dynamic.partition.mode = nonstrict
 hive.txn.manager =org.apache.hadoop.hive.ql.lockmgr.DbTxnManager
 hive.compactor.initiator.on = true
 hive.compactor.worker.threads = 1

Şu anda yalnızca orc dosyası biçimi desteklenmektedir.

Tablo oluşturma ifadesi.

 create table Sample_Table(
 col1 Int,
 col2 String,
 col3 String) 
 clustered by (col3) into 3 buckets 
 stored as orc 
 TBLPROPERTIES ('transactional'='true');
