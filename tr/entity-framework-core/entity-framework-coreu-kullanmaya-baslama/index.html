Modelleme, Veri Sorgulama ve Kaydetme #

Model #

EF Core ile veri erişimi bir model kullanılarak gerçekleştirilir. Bir model, varlık sınıflarından ve veritabanıyla bir oturumu temsil eden türetilmiş bir bağlamdan oluşur ve verileri sorgulamanıza ve kaydetmenize olanak tanır.

Mevcut bir veritabanından bir model oluşturabilir, bir modeli veritabanınıza uyacak şekilde elle kodlayabilir veya modelinizden bir veritabanı oluşturmak için EF Migrations’ı kullanabilir (ve modeliniz zaman içinde değiştikçe onu geliştirebilirsiniz).

using Microsoft.EntityFrameworkCore;
using System.Collections.Generic;

namespace Intro
{
    public class BloggingContext : DbContext
    {
        public DbSet<Blog> Blogs { get; set; }
        public DbSet<Post> Posts { get; set; }

        protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
        {
            optionsBuilder.UseSqlServer(@"Server=(localdb)\mssqllocaldb;Database=MyDatabase;Trusted_Connection=True;");
        }
    }

    public class Blog
    {
        public int BlogId { get; set; }
        public string Url { get; set; }

        public List<Post> Posts { get; set; }
    }

    public class Post
    {
        public int PostId { get; set; }
        public string Title { get; set; }
        public string Content { get; set; }

        public int BlogId { get; set; }
        public Blog Blog { get; set; }
    }
}

sorgulama #

Varlık sınıflarınızın örnekleri, Dille Tümleşik Sorgu (LINQ) kullanılarak veritabanından alınır.

using (var db = new BloggingContext())
{
    var blogs = db.Blogs
        .Where(b => b.Rating > 3)
        .OrderBy(b => b.Url)
        .ToList();
}

Veri Kaydetme #

Veriler, varlık sınıflarınızın örnekleri kullanılarak veritabanında oluşturulur, silinir ve değiştirilir.

using (var db = new BloggingContext())
{
    var blog = new Blog { Url = "http://sample.com" };
    db.Blogs.Add(blog);
    db.SaveChanges();
}

Veri Silme #

Varlık sınıflarınızın örnekleri, Dille Tümleşik Sorgu (LINQ) kullanılarak veritabanından alınır.

using (var db = new BloggingContext())
{
    var blog = new Blog { Url = "http://sample.com" };
    db.Blogs.Attach(blog);
    db.Blogs.Remove(blog);
    db.SaveChanges();
}

Veri Güncelleme #

Veriler, varlık sınıflarınızın örnekleri kullanılarak veritabanında güncellenir.

using (var db = new BloggingContext())
{
    var blog = new Blog { Url = "http://sample.com" };
    var entity = db.Blogs.Find(blog);
    entity.Url = "http://sample2.com";
    db.SaveChanges();
}

Adım 3 - EF Paketlerini Yükleme #

Class Library’deki project.json dosyanızı açın ve aşağıdakini yapıştırın, ardından dosyayı kaydedin:

{
  "version": "1.0.0-*",

  "dependencies": {
    "Microsoft.EntityFrameworkCore.SqlServer": "1.0.0",
    "Microsoft.EntityFrameworkCore.SqlServer.Design": "1.0.0",
    "Microsoft.EntityFrameworkCore.Tools": "1.0.0-preview2-final",
    "NETStandard.Library": "1.6.0"
  },
  "tools": {
    "Microsoft.EntityFrameworkCore.Tools": "1.0.0-preview2-final"
  },

  "frameworks": {
    "net46": {
    },
    "netcoreapp1.0": {
      "dependencies": {
        "Microsoft.NETCore.App": {
          "type": "platform",
          "version": "1.0.0-*"
        }
      }
    }
  }
}

Bu, ‘Referanslar’ altındaki paketleri geri yüklemelidir.

Bağlantı Dizesi Geçmek #

Buradaki durumumda, her müşterinin kendi Veritabanına sahip olduğu Çok Kiracılı bir Uygulamamız var, ör. Client_1, Client_2, Client_3. Bu yüzden bağlantı dizesi dinamik olmak zorundaydı.

Böylece bir yapıcıya bir bağlantı dizesi özelliği ekledik ve bunu ‘OnConfiguring’ yönteminde Bağlam’a ilettik.

public partial class ClientContext
{
    private readonly string _connectionString;

    public ClientContext(string connectionString) : base()
    {
        _connectionString = connectionString;
    }

    protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
    {
        optionsBuilder.UseSqlServer(_connectionString);
    }
}

ve bu şekilde kullandı:

    public void TestConnection()
    {
        var clientId = 1;

        var connectionString = string.Format("Server=192.168.0.211; Database=Client_{0}; user id= USER; password = PWD;", clientId);

        using (var clientContext = new ClientContext(connectionString))
        {
            var assets = clientContext.Users.Where(s => s.UserId == 1);
        }
    }

Projeye paket ekleme #

Projenize EntityFrameworkCore eklemek için, “project.json” dosyasını güncelleyin (“bağımlılıklar” ve “araçlar” bölümlerine yeni satırlar ekleyin):

"dependencies": {
    ...
    "Microsoft.EntityFrameworkCore.SqlServer": "1.0.0",
    "Microsoft.EntityFrameworkCore.SqlServer.Design": "1.0.0",
    "Microsoft.EntityFrameworkCore.Design": {
      "version": "1.0.0",
      "type": "build"
    },
},
"tools": {
    ...
    "Microsoft.EntityFrameworkCore.Tools": "1.0.0-preview2-final"
}

Bu paketleri internetten gerçekten indirmek için dotnet restore çalıştırmayı unutmayınız.

Microsoft SQLServer dışında bir RDBMS kullanıyorsanız - ‘Microsoft.EntityFrameworkCore.SqlServer’ı doğru sürümle değiştirin (‘Microsoft.EntityFrameworkCore.Sqlite’, ‘Npgsql.EntityFrameworkCore.PostgreSQL’ veya diğer - önerilen sürüm için paket RDBMS belgelerinize bakın. ).
