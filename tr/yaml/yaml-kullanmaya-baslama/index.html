Yorumlar #

# This comment occupies a whole line
- some item # This comment succeeds content of a line
- http://example.com/#nocomment
- "This # does not introduce a comment."
- |
    This is a block scalar.
    A # inside it does not introduce a comment.
   # unless it is less indented than the first line (this is one)

Bir “#” işaretinin bir yorum sunması için ya

  • bir satırın başında meydana gelir veya
  • boşluktan önce gelir.

# her zaman boşluk bırakmalıdır. # içinde alıntılanan skalerler asla yoruma başlamaz. “#”, blok skalerlerinin sonunda yorumlar getirebilir, ancak bu nedenle, blok skalerinin taban girintisinden (genellikle ilk boş olmayan satırının girintisiyle belirlenir) daha az girintili olmalıdır.

Temel Yaml sözdizimi #

YAML, yapılandırılmış verilerin bir hiyerarşide depolanmasına izin veren metin tabanlı bir formattır. YAML, minimum ek yük ile insan ve makine tarafından okunabilecek şekilde tasarlanmıştır. YAML spesifikasyonu yaml.org adresinde bulunabilir. Ayrıca bir referans kartı vardır.

Yorumlar `#’ ile başlar ve yeni satıra kadar devam eder, yorumlar diğer belirteçlerden boşluk ile ayrılmalıdır. Boşluk boş değil, girinti boşluk olmalı, sekme değil. YAML, ana anahtardan daha fazla boşluk içeren ön ekli satırların içinde bulunduğunu dikkate alacaktır. Ayrıca, aynı haritaya ait olmak için tüm satırların önüne aynı miktarda boşluk eklenmelidir.

YAML, koleksiyon türleri olarak dizilere ve eşlemelere sahiptir, her ikisi de akış ve blok stilinde temsil edilebilir.

YAML’de bir dizi skaler dizi şuna benzer:

[ one, two, three ]   # flow style

# or block style

- one
- two
- three

Bir eşleme, anahtar/değer çiftlerinden oluşur:

index: 4  # block style
name: nali

# or 

{ index: 4, name: nali }   # flow style

# or nested (equivalent of { level: { one: { two: fun } } }):

level:

  one:

    two: fun

Metin dizelerini birden çok satıra bölme #

- Without quotes:
   You can just
   split a long piece of text like this.
- With quotes:
    "[But be careful:
     if you \"need\" punctuation, put double quotes around it. You can ev\
     en split without spaces by using backslashes."
- Or single quotes:
    'This works
     but isn''t as flexible'
- If you want to keep those new line characters: | 
    Then do
    it this way with 
    a pipe (|) character. (This string has three \n characters)
- Or you can have just the one final new line: >
    This string has
    just one \n character, at the very end.
- Block indicators:
    Look up >-, >+, |- and |+ for fine tuning.

Kaçan Karakterler #

YAML, üç kaçış gösterimi stilini destekler:

  1. Varlık Kaçışları

a. boşluk: “&#x20;”

b. iki nokta üst üste: “&#58;”

c. ve işareti: “&amp;”

  1. Unicode Kaçışları

a. boşluk: “\u0020”

b. tek alıntı: “\u0027”

c. çift ​​alıntı: “\u0022”

  1. Alıntılanan Kaçışlar

a. tek alıntıda çift alıntı: ‘“Ben her zaman yalan söylerim” doğru bir ifade midir?’

b. iç içe çift alıntı: " "Bıraktım" dedi.

c. iç içe tek alıntı: ’ Dilsizdi: ’’ '

Blok Stili Eşlemeleri #

Örtük anahtarlarla:

key: value
another key:
  - some
  - more
  - values
[1, 2, 3]: last value, which has a flow style key

Örtük ve açık anahtarlarla:

? key
: value
another key:
  - some
  - more
  - values
? [1, 2, 3]
: last value, which has a flow style key

“key”, “another key” ve “[1, 2, 3]” farklı tuş stilleri kullanmalarına rağmen aynı eşlemenin anahtarlarıdır.

İç içe eşlemeler:

first level:
  second level:
    ? third level
    :
      forth level: value of implicit key
    ? third level, second key
    : value of explicit key
  ?
    mapping as: key of
    another: mapping
  : scalar value of mapping key
first level, second key:
  last value
