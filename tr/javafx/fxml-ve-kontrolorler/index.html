Sözdizimi #

  • xmlns:fx=“http://javafx.com/fxml”       // ad alanı bildirimi

Ayar Özellikleri #

fxml’de bir nesneye veri eklemenin birden çok yolu vardır:

<özellik> etiketi #

Bir özelliğin adını taşıyan bir etiket, bir örnek oluşturmak için kullanılan bir öğenin alt öğesi olarak eklenebilir. Bu etiketin alt öğesi, ayarlayıcı kullanılarak özelliğe atanır veya özelliğin içeriğine eklenir (salt okunur liste/harita özellikleri).

Varsayılan özellik #

Bir sınıfa “@DefaultProperty” notu eklenebilir. Bu durumda öğeler, özelliğin adıyla bir öğe kullanılmadan doğrudan alt öğe olarak eklenebilir.

property="değer" özelliği #

Özellikler, nitelik adı olarak özellik adı ve nitelik değeri olarak değer kullanılarak atanabilir. Bu, aşağıdaki öğeyi etiketin alt öğesi olarak eklemekle aynı etkiye sahiptir:

<property>
    <String fx:value="value" />
</property>

statik ayarlayıcı #

Özellikler, ‘statik’ ayarlayıcılar kullanılarak da ayarlanabilir. Bunlar, ilk parametre olarak öğeyi ve ikinci parametre olarak ayarlanacak değeri alan “setProperty” adlı “statik” yöntemlerdir. Bu yöntemler herhangi bir sınıfta okunabilir ve olağan özellik adı yerine ‘ContainingClass.property’ kullanılarak kullanılabilir.

Not: Şu ​​anda, değer türü belirtilmediği sürece, bunun çalışması için karşılık gelen bir statik alıcı yöntemine (yani, öğeyi statik ayarlayıcı ile aynı sınıfta parametre olarak alan “getProperty” adlı statik bir yöntem) sahip olmak gerekli görünmektedir. ‘Dize’dir.

Tip Zorlama #

Atamalar sırasında doğru sınıftan bir nesne elde etmek için aşağıdaki mekanizma kullanılır, örn. bir ayarlayıcı yönteminin parametre tipine uyacak şekilde.

Sınıflar atanabilirse, değerin kendisi kullanılır.

Aksi takdirde değer aşağıdaki gibi dönüştürülür

Hedef türü kullanılan değer (kaynak değer s)
“Boole”, “boole” Boolean.valueOf(s)
‘char’, ‘Karakter’ s.toString.charAt(0)
diğer ilkel tip veya sarmalayıcı tip “s"nin bir “Sayı” olması durumunda, sarmalayıcı türü için “valueOf(s.toString())”, hedef tür için uygun bir yöntemdir
‘BigInteger’ “BigInteger.valueOf(s.longValue())”, “s” bir “Sayı"dır, aksi takdirde “new BigInteger(s.toString())”
“Büyük Ondalık” BigDecimal.valueOf(s.doubleValue()), s bir Sayı, aksi takdirde yeni BigDecimal(s.toString())
Sayı “s.toString()” bir “.” içeriyorsa “Double.valueOf(s.toString())”, aksi takdirde “Long.valueOf(s.toString())”
‘Sınıf’ Class.forName(s.toString()), sınıfı başlatmadan geçerli iş parçacığının ClassLoader bağlamı kullanılarak çağrıldı
numaralandırma “valueOf” yönteminin sonucu, ek olarak, “s” küçük harfle başlayan bir “Dize” ise, her büyük harften önce eklenen “_” ile ayrılmış tamamı büyük bir “Dize"ye dönüştürülür
diğer ’s’ türüyle veya bu türden bir üst sınıfla eşleşen bir parametreye sahip olan, targetType’ta bir ‘statik’ ‘değerOf’ yöntemi tarafından döndürülen değer

Not: Bu davranış iyi belgelenmemiştir ve değişikliğe tabi olabilir.

Örnek #

public enum Location {
    WASHINGTON_DC,
    LONDON;
}

package fxml.sample;

import java.math.BigInteger;
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;
import javafx.beans.DefaultProperty;

@DefaultProperty("items")
public class Sample {
    
    private Location loaction;

    public Location getLoaction() {
        return loaction;
    }

    public void setLoaction(Location loaction) {
        this.loaction = loaction;
    }

    public int getNumber() {
        return number;
    }

    public void setNumber(int number) {
        this.number = number;
    }
    
    int number;

    private final List<Object> items = new ArrayList<>();

    public List<Object> getItems() {
        return items;
    }
    
    private final Map<String, Object> map = new HashMap<>();

    public Map<String, Object> getMap() {
        return map;
    }
    
    private BigInteger serialNumber;

    public BigInteger getSerialNumber() {
        return serialNumber;
    }

    public void setSerialNumber(BigInteger serialNumber) {
        this.serialNumber = serialNumber;
    }

    @Override
    public String toString() {
        return "Sample{" + "loaction=" + loaction + ", number=" + number + ", items=" + items + ", map=" + map + ", serialNumber=" + serialNumber + '}';
    }
    
}

package fxml.sample;

public class Container {

    public static int getNumber(Sample sample) {
        return sample.number;
    }

    public static void setNumber(Sample sample, int number) {
        sample.number = number;
    }

    private final String value;

    private Container(String value) {
        this.value = value;
    }

    public static Container valueOf(String s) {
        return new Container(s);
    }

    @Override
    public String toString() {
        return "42" + value;
    }

}

Parametreleri FXML’ye geçirme - bir controllerFactory kullanarak #

Sorun: Bazı verilerin bir fxml’den yüklenen bir sahneye aktarılması gerekiyor.

Çözüm

Denetleyicileri oluşturmaktan sorumlu bir denetleyici fabrikası belirtin. Verileri fabrika tarafından oluşturulan denetleyici örneğine iletin.

FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<VBox xmlns:fx="http://javafx.com/fxml/1" fx:controller="valuepassing.TestController">
    <children>
        <Text fx:id="target" />
    </children>
</VBox>

Kontrolör

package valuepassing;

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class TestController {

    private final String data;

    public TestController(String data) {
        this.data = data;
    }
    
    @FXML
    private Text target;
    
    public void initialize() {
        // handle data once the fields are injected
        target.setText(data);
    }

}

fxml’yi yüklemek için kullanılan kod

String data = “Merhaba Dünya!”;

Map<Class, Callable<?>> creators = new HashMap<>();
creators.put(TestController.class, new Callable<TestController>() {

    @Override
    public TestController call() throws Exception {
        return new TestController(data);
    }

});

FXMLLoader loader = new FXMLLoader(getClass().getResource("test.fxml"));

loader.setControllerFactory(new Callback<Class<?>, Object>() {

    @Override
    public Object call(Class<?> param) {
        Callable<?> callable = creators.get(param);
        if (callable == null) {
            try {
                // default handling: use no-arg constructor
                return param.newInstance();
            } catch (InstantiationException | IllegalAccessException ex) {
                throw new IllegalStateException(ex);
            }
        } else {
            try {
                return callable.call();
            } catch (Exception ex) {
                throw new IllegalStateException(ex);
            }
        }
    }
});

Parent root = loader.load();

Bu karmaşık görünebilir, ancak fxml’nin hangi denetleyici sınıfına ihtiyaç duyduğuna karar verebilmesi gerekiyorsa yararlı olabilir.

FXML’de örnek oluşturma #

Aşağıdaki sınıf, sınıf örneklerinin nasıl oluşturulabileceğini göstermek için kullanılır:

“Kişi(@NamedArg(“name”) Dize adı)” içindeki ek açıklama, “@NamedArg” ek açıklaması kullanılamadığından kaldırılmalıdır.

son sürüm 
package fxml.sample;

import javafx.beans.NamedArg;
import javafx.beans.property.SimpleStringProperty;
import javafx.beans.property.StringProperty;

public class Person {

    public static final Person JOHN = new Person("John");

    public Person() {
        System.out.println("Person()");
    }
    
    public Person(@NamedArg("name") String name) {
        System.out.println("Person(String)");
        this.name.set(name);
    }
    
    public Person(Person person) {
        System.out.println("Person(Person)");
        this.name.set(person.getName());
    }

    private final StringProperty name = new SimpleStringProperty();

    public final String getName() {
        System.out.println("getter");
        return this.name.get();
    }

    public final void setName(String value) {
        System.out.println("setter");
        this.name.set(value);
    }

    public final StringProperty nameProperty() {
        System.out.println("property getter");
        return this.name;
    }
    
    public static Person valueOf(String value) {
        System.out.println("valueOf");
        return new Person(value);
    }
    
    public static Person createPerson() {
        System.out.println("createPerson");
        return new Person();
    }
    
}

Fxml’yi yüklemeden önce ‘Person’ sınıfının zaten başlatıldığını varsayın.

İthalatla ilgili bir not #

Aşağıdaki fxml örneğinde içe aktarma bölümü dışarıda bırakılacaktır. Ancak fxml ile başlamalıdır

<?xml version="1.0" encoding="UTF-8"?>

ardından fxml dosyasında kullanılan tüm sınıfları içe aktaran bir içe aktarma bölümü gelir. Bu içe aktarmalar, statik olmayan içe aktarmalara benzer, ancak işleme talimatları olarak eklenir. Java.lang paketindeki sınıfların bile içe aktarılması gerekir.

Bu durumda aşağıdaki ithalatlar eklenmelidir:

<?import java.lang.*?>
<?import fxml.sample.Person?>

 

@NamedArg açıklamalı kurucu #

Her parametrenin @NamedArg ile açıklamalı olduğu ve @NamedArg açıklamalarının tüm değerlerinin fxml’de mevcut olduğu bir kurucu varsa, kurucu bu parametrelerle kullanılacaktır.

<Person name="John"/>

<Person xmlns:fx="http://javafx.com/fxml">
    <name>
        <String fx:value="John"/>
    </name>
</Person>

Yüklendiyse, her ikisi de aşağıdaki konsol çıktısıyla sonuçlanır:

Person(String)
son sürüm

 

Args yapıcısı yok #

Uygun @NamedArg açıklamalı kurucu yoksa, parametre almayan kurucu kullanılacaktır.

@NamedArg ek açıklamasını yapıcıdan kaldırın ve yüklemeyi deneyin.

<Person name="John"/>

Bu, yapıcıyı parametresiz kullanacaktır.

Çıktı:

Person()
setter

 

“fx:değer” özelliği #

‘fx:value’ özniteliği, değerini bir ‘String’ parametresi alarak ve kullanılacak örneği döndürerek bir ‘statik’ ‘valueOf’ yöntemine iletmek için kullanılabilir.

Örnek

<Person xmlns:fx="http://javafx.com/fxml" fx:value="John"/>

Çıktı:

valueOf
Person(String)

 

‘fx:fabrika’ #

“fx:factory” özelliği, parametre almayan rastgele “statik” yöntemler kullanarak nesnelerin oluşturulmasına izin verir.

Örnek

<Person xmlns:fx="http://javafx.com/fxml" fx:factory="createPerson">
    <name>
        <String fx:value="John"/>
    </name>
</Person>

Çıktı:

createPerson
Person()
setter

 

<fx:kopya> #

‘fx:copy’ kullanılarak bir kopya oluşturucu çağrılabilir. Bir başkasının “fx:id"sinin belirtilmesi Etiketin “source” özelliği, parametre olarak o nesneyle kopya oluşturucuyu çağırır.

Örnek:

<ArrayList xmlns:fx="http://javafx.com/fxml">
    <Person fx:id="p1" fx:constant="JOHN"/>
    <fx:copy source="p1"/>
</ArrayList>

Çıktı

Person(Person)
getter

 

‘fx:sabit’ #

“fx:constant”, bir “statik final” alanından bir değer alınmasına izin verir.

Örnek

<Person xmlns:fx="http://javafx.com/fxml" fx:constant="JOHN"/>

herhangi bir çıktı üretmez, çünkü bu sadece sınıf başlatılırken oluşturulan ‘JOHN’ye atıfta bulunur.

Örnek FXML #

Bir düğme ve bir etiket düğümü içeren bir ‘AnchorPane’i özetleyen Basit bir FXML belgesi:

<?xml version="1.0" encoding="UTF-8"?>

<?import java.lang.*?>
<?import java.util.*?>
<?import javafx.scene.*?>
<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>

<AnchorPane id="AnchorPane" prefHeight="200" prefWidth="320" xmlns:fx="http://javafx.com/fxml/1" 
        fx:controller="com.example.FXMLDocumentController">
    <children>
        <Button layoutX="126" layoutY="90" text="Click Me!" onAction="#handleButtonAction" fx:id="button" />
        <Label layoutX="126" layoutY="120" minHeight="16" minWidth="69" fx:id="label" />
    </children>
</AnchorPane>

Bu örnek FXML dosyası, bir denetleyici sınıfıyla ilişkilendirilmiştir. FXML ve denetleyici sınıfı arasındaki ilişki, bu durumda, FXML’nin kök öğesindeki ‘fx:controller’ özniteliğinin değeri olarak sınıf adını belirterek yapılır: ‘fx:controller=“com.example. FXMLDocumentController”`. Controller sınıfı, FXML dosyasında tanımlanan UI öğeleri üzerindeki kullanıcı eylemlerine yanıt olarak Java kodunun yürütülmesine izin verir:

package com.example ;

import java.net.URL;
import java.util.ResourceBundle;
import javafx.event.ActionEvent;
import javafx.fxml.FXML;
import javafx.fxml.Initializable;
import javafx.scene.control.Label;

public class FXMLDocumentController {
    
    @FXML
    private Label label;
    
    @FXML
    private void handleButtonAction(ActionEvent event) {
        System.out.println("You clicked me!");
        label.setText("Hello World!");
    }
    
    @Override
    public void initialize(URL url, ResourceBundle resources) {
        // Initialization code can go here. 
        // The parameters url and resources can be omitted if they are not needed
    }    
    
}

FXML dosyasını yüklemek için bir “FXMLLoader” kullanılabilir:

public class MyApp extends Application {

    @Override
    public void start(Stage stage) throws Exception {

        FXMLLoader loader = new FXMLLoader();
        loader.setLocation(getClass().getResource("FXMLDocument.fxml"));
        Parent root = loader.load();
        
        Scene scene = new Scene(root);
        
        stage.setScene(scene);
        stage.show();
    }

}

’load’ yöntemi birkaç eylemi gerçekleştirir ve bunların gerçekleştiği sırayı anlamak yararlıdır. Bu basit örnekte:

  1. ‘FXMLLoader’, FXML dosyasını okur ve ayrıştırır. Dosyada tanımlanan öğelere karşılık gelen nesneler yaratır ve üzerlerinde tanımlanan tüm ‘fx:id’ niteliklerini not eder.

  2. FXML dosyasının kök öğesi bir “fx:controller” özniteliği tanımladığından, “FXMLLoader”, belirttiği sınıfın yeni bir örneğini oluşturur. Varsayılan olarak bu, belirtilen sınıfta bağımsız değişken olmayan yapıcıyı çağırarak gerçekleşir.

  3. Denetleyicide eşleşen alan adlarına sahip alanları olan ve “genel” (önerilmez) veya açıklamalı “@FXML” (önerilir) olan “fx:id” özniteliklerine sahip öğeler, karşılık gelenlere “enjekte edilir” alanlar. Dolayısıyla bu örnekte, FXML dosyasında ‘fx:id=“label”’ ile bir ‘Etiket’ ve denetleyicide şu şekilde tanımlanmış bir alan olduğundan

     @FXML
 private Label label ;

    the label field is initialized with the Label instance created by the FXMLLoader.

  4. Olay işleyicileri, ‘onXXX=”#…”’ özellikleri tanımlanmış FXML dosyasındaki herhangi bir öğeyle kaydedilir. Bu olay işleyicileri, denetleyici sınıfında belirtilen yöntemi çağırır. Bu örnekte, “Button”, “onAction="#handleButtonAction”` değerine sahip olduğundan ve denetleyici bir yöntem tanımladığı için

     @FXML
 private void handleButtonAction(ActionEvent event) { ... }

    when an action is fired on the button (e.g. the user presses it), this method is invoked. The method must have void return type, and can either define a parameter matching the event type (ActionEvent in this example), or can define no parameters.

  5. Son olarak, denetleyici sınıfı bir “initialize” yöntemini tanımlarsa, bu yöntem çağrılır. Bunun, @FXML alanları enjekte edildikten sonra gerçekleştiğine dikkat edin, böylece bu metotta güvenli bir şekilde erişilebilirler ve FXML dosyasındaki öğelere karşılık gelen örneklerle başlatılacaktır. “initialize()” yöntemi hiçbir parametre alamaz veya bir “URL” ve bir “ResourceBundle” alabilir. İkinci durumda, bu parametreler FXML dosyasının konumunu temsil eden “URL” ve “FXMLLoader” üzerinde “loader.setResources(…)” aracılığıyla ayarlanan herhangi bir “ResourceBundle” tarafından doldurulacaktır. Ayarlanmadıysa, bunlardan herhangi biri “boş” olabilir.

İç İçe Denetleyiciler #

Tek bir denetleyici kullanarak tüm kullanıcı arayüzünü tek bir FXML’de oluşturmaya gerek yoktur.

<fx:include> etiketi, bir fxml dosyasını diğerine dahil etmek için kullanılabilir. Dahil edilen fxml’nin denetleyicisi, ‘FXMLLoader’ tarafından oluşturulan diğer herhangi bir nesne gibi dahil edilen dosyanın denetleyicisine enjekte edilebilir.

Bu, “fx:include” öğesine “fx:id” özniteliği eklenerek yapılır. Bu şekilde, dahil edilen fxml’nin denetleyicisi, `<fx:id value>Controller’ adıyla alana enjekte edilecektir.

Örnekler:

fx:id değeri enjeksiyon için alan adı
foo fooDenetleyici
cevap42 answer42Denetleyici
xYz xYzDenetleyici

Örnek fxml’ler

Tezgah

Bu, bir “Metin” düğümü ile bir “StackPane” içeren bir fxml’dir. Bu fxml dosyasının denetleyicisi, sayacı artırmanın yanı sıra geçerli sayaç değerini almaya izin verir:

sayaç.fxml

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<StackPane prefHeight="200" prefWidth="200" xmlns:fx="http://javafx.com/fxml/1" fx:controller="counter.CounterController">
    <children>
        <Text fx:id="counter" />
    </children>
</StackPane>

Kontrolör

package counter;

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class CounterController {
    @FXML
    private Text counter;

    private int value = 0;
    
    public void initialize() {
        counter.setText(Integer.toString(value));
    }
    
    public void increment() {
        value++;
        counter.setText(Integer.toString(value));
    }
    
    public int getValue() {
        return value;
    }
    
}

fxml dahil

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>

<BorderPane prefHeight="500" prefWidth="500" xmlns="http://javafx.com/javafx/8" xmlns:fx="http://javafx.com/fxml/1" fx:controller="counter.OuterController">
    <left>
        <Button BorderPane.alignment="CENTER" text="increment" onAction="#increment" />
    </left>
    <center>
        <!-- content from counter.fxml included here -->
        <fx:include fx:id="count" source="counter.fxml" />
    </center>
</BorderPane>

Dış Denetleyici

Dahil edilen fxml’nin denetleyicisi bu denetleyiciye enjekte edilir. Burada, “Button” için “onAction” olayının işleyicisi, sayacı artırmak için kullanılır.

package counter;

import javafx.fxml.FXML;

public class OuterController {

    // controller of counter.fxml injected here
    @FXML
    private CounterController countController;

    public void initialize() {
        // controller available in initialize method
        System.out.println("Current value: " + countController.getValue());
    }

    @FXML
    private void increment() {
        countController.increment();
    }

}

Kodun “outer.fxml” ile aynı paketteki bir sınıftan çağrıldığı varsayılarak fxml’ler şu şekilde yüklenebilir:

Parent parent = FXMLLoader.load(getClass().getResource("outer.fxml"));

Blokları Tanımla ve fx:reference #

Bazen, fxml’deki olağan nesne yapısının dışında bir öğenin oluşturulması gerekir.

Tanım Blokları işte burada devreye girer:

Bir “fx:define” öğesinin içindeki içerikler, üst öğe için oluşturulan nesneye eklenmez.

“fx:define” öğesinin her alt öğesinin bir “fx:id” özniteliğine ihtiyacı vardır.

Bu şekilde oluşturulan nesnelere daha sonra “fx:reference” öğesi veya ifade bağlama kullanılarak başvurulabilir.

“fx:reference” öğesi, “fx:id” öğesiyle aynı değer kullanılarak “fx:reference” öğesi işlenmeden önce işlenen bir “fx:id” özniteliğine sahip herhangi bir öğeye başvuruda bulunmak için kullanılabilir. “fx:reference” öğesinin “source” özniteliğinde başvurulan öğenin özniteliği.

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import java.lang.*?>
<?import javafx.scene.*?>
<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>


<VBox xmlns:fx="http://javafx.com/fxml/1" prefHeight="300.0" prefWidth="300.0" xmlns="http://javafx.com/javafx/8">
    <children>
        <fx:define>
            <String fx:value="My radio group" fx:id="text" />
        </fx:define>
        <Text>
            <text>
                <!-- reference text defined above using fx:reference -->
                <fx:reference source="text"/>
            </text>
        </Text>
        <RadioButton text="Radio 1">
            <toggleGroup>
                <ToggleGroup fx:id="group" />
            </toggleGroup>
        </RadioButton>
        <RadioButton text="Radio 2">
            <toggleGroup>
                <!-- reference ToggleGroup created for last RadioButton -->
                <fx:reference source="group"/>
            </toggleGroup>
        </RadioButton>
        <RadioButton text="Radio 3" toggleGroup="$group" />
        
        <!-- reference text defined above using expression binding -->
        <Text text="$text" />
    </children>
</VBox>

FXML’ye veri aktarma - mevcut denetleyiciye erişim #

Sorun: Bazı verilerin bir fxml’den yüklenen bir sahneye aktarılması gerekiyor.

Çözüm

“fx:controller” özniteliğini kullanarak bir denetleyici belirtin ve yükleme işlemi sırasında oluşturulan denetleyici örneğini fxml’yi yüklemek için kullanılan “FXMLLoader” örneğinden alın.

Verileri denetleyici örneğine geçirmek için yöntemler ekleyin ve bu yöntemlerdeki verileri işleyin.

FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<VBox xmlns:fx="http://javafx.com/fxml/1" fx:controller="valuepassing.TestController">
    <children>
        <Text fx:id="target" />
    </children>
</VBox>

Kontrolör

package valuepassing;

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class TestController {

    @FXML
    private Text target;

    public void setData(String data) {
        target.setText(data);
    }

}

fxml’yi yüklemek için kullanılan kod

String data = "Hello World!";

FXMLLoader loader = new FXMLLoader(getClass().getResource("test.fxml"));
Parent root = loader.load();
TestController controller = loader.<TestController>getController();
controller.setData(data);

FXML’ye veri aktarma - Denetleyici örneğini belirleme #

Sorun: Bazı verilerin bir fxml’den yüklenen bir sahneye aktarılması gerekiyor.

Çözüm

Denetleyiciyi, fxml’yi yüklemek için daha sonra kullanılan ‘FXMLLoader’ örneğini kullanarak ayarlayın.

fxml’yi yüklemeden önce denetleyicinin ilgili verileri içerdiğinden emin olun.

Not: bu durumda fxml dosyası “fx:controller” özniteliğini içermemelidir.

FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<VBox xmlns:fx="http://javafx.com/fxml/1">
    <children>
        <Text fx:id="target" />
    </children>
</VBox>

Kontrolör

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class TestController {

    private final String data;

    public TestController(String data) {
        this.data = data;
    }
    
    @FXML
    private Text target;
    
    public void initialize() {
        // handle data once the fields are injected
        target.setText(data);
    }

}

fxml’yi yüklemek için kullanılan kod

String data = "Hello World!";

FXMLLoader loader = new FXMLLoader(getClass().getResource("test.fxml"));

TestController controller = new TestController(data);
loader.setController(controller);

Parent root = loader.load();
