Kurulum veya Kurulum #

Ön koşullar

Bu makale, oozie-4.3.0’ın Hadoop 2.7.3’e yüklenmesini gösterdi

  1. Java 1.7+
  2. Hadoop 2.x (burada, 2.7.3)
  3. Maven3+
  4. Unix kutusu

Adım1: Dist dosyası

http://www-eu.apache.org/dist/oozie/4.3.0/ adresinden oozie tar.gz dosyasını alın ve çıkarın

cd $HOME
tar -xvf oozie-4.3.0.tar.gz

2. Adım: Oozie’yi Oluştur

cd $HOME/oozie-4.3.0/bin
./mkdistro.sh -DskipTests

Adım3: Sunucu Kurulumu

Yerleşik ikili dosyaları ana dizine ‘oozie’ olarak kopyalayın

cd $HOME
cp -R $HOME/oozie-4.3.0/distro/target/oozie-4.3.0-distro/oozie-4.3.0 .

Adım 3.1: libext oozie dizini içinde libext dizini oluşturun

 cd $HOME/oozie
 mkdir libext

Not: ExtJS (2.2+) kitaplığı (Oozie web konsolunu etkinleştirmek için isteğe bağlı) Ancak, farklı bir lisans kullandığından ExtJS kitaplığı Oozie ile birlikte verilmez :( Şimdi hadoop kavanozlarını libext dizinine koymanız gerekiyor, aksi takdirde oozie.log dosyasında aşağıdaki hatayı verecektir.

WARN ActionStartXCommand:523 - SUNUCU[data01.teg.io] KULLANICI[hadoop] GRUP[-] TOKEN[] UYGULAMA[map-reduce-wf] İŞ[0000000-161215143751620-oozie-hado-W] EYLEM[0000000-161215143751620-oozie-hado-W@mr-node] Başlatma hatası eylem [mr-node]. ErrorType [GEÇİCİ], HataKodu [JA009], Mesaj [JA009: Küme başlatılamıyor. Lütfen yapılandırmanızı kontrol edin mapreduce.framework.name ve karşılık gelen sunucu adresleri.]

Öyleyse, kavanozları libext dizininin içine koyalım

cp $HADOOP_HOME/share/hadoop/common/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/common/lib/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/hdfs/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/hdfs/lib/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/mapreduce/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/mapreduce/lib/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/yarn/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/yarn/lib/*.jar oozie/libext/

Adım 3.2: Oozie’nin Kimliğine bürünme Oozie’de kimliğe bürünme hatasını önlemek için core-site.xml dosyasını aşağıdaki gibi değiştirin

<!-- OOZIE -->
  <property>
    <name>hadoop.proxyuser.[OOZIE_SERVER_USER].hosts</name>
    <value>[OOZIE_SERVER_HOSTNAME]</value>
  </property>
  <property>
    <name>hadoop.proxyuser.[OOZIE_SERVER_USER].groups</name>
    <value>[USER_GROUPS_THAT_ALLOW_IMPERSONATION]</value>
  </property>

Varsayalım ki, oozie kullanıcım huser ve ana bilgisayar localhost ve grup hadoop

<!-- OOZIE -->
  <property>
    <name>hadoop.proxyuser.huser.hosts</name>
    <value>localhost</value>
  </property>
  <property>
    <name>hadoop.proxyuser.huser.groups</name>
    <value>hadoop</value>
  </property>

Not : Karışıklık olması durumunda tüm değerlerde * kullanabilirsiniz.

Adım 3.3: Savaşı hazırlayın

cd $HOME/oozie/bin
./oozie-setup.sh prepare-war

Bu, oozie dizini içinde oozie.war dosyası oluşturacaktır. Bu savaş daha fazla kullanılacaksa, şu hatayla karşılaşabilirsiniz:

HATA ActionStartXCommand:517 - SUNUCU[data01.teg.io] KULLANICI[hadoop] GRUP[-] TOKEN[] UYGULAMA[map-reduce-wf] İŞ[0000000-161220104605103-oozie-hado-W] EYLEM[0000000-161220104605103-oozie-hado-W@mr-node] Hatası, java.lang.NoSuchFieldError: HADOOP_CLASSPATH

Neden? Niye? çünkü oozie derlemesi, “-Dhadoop.version=2.7.3” seçeneğiyle Hadoop 2.7.3 belirtilirken bile Hadoop 2.6.0 kavanozları üretti.

Yani, bu hatayı önlemek için, oozie.war dosyasını farklı bir dizine kopyalayın

mkdir $HOME/oozie_war_dir
cp $HOME/oozie/oozie.war $HOME/oozie_war_dir
cd $HOME/oozie_war_dir
jar -xvf oozie.war
rm -f oozie.war/WEB-INF/lib/hadoop-*.jar
rm -f oozie.war/WEB-INF/lib/hive-*.jar
rm oozie.war
jar -cvf oozie.war ./*
cp oozie.war $HOME/oozie/

Ardından, bir hazırlık-savaş ile oozie için oozie.war ikili dosyalarını yeniden oluşturun

cd $HOME/oozie/bin
./oozie-setup.sh prepare-war

Adım 3.4: HDFS’de sharelib oluşturun

cd $HOME/oozie/bin
./oozie-setup.sh sharelib create -fs hdfs://localhost:9000

Şimdi, bu sharelib kurulumu size aşağıdaki hatayı verebilir:

org.apache.oozie.service.ServiceException: E0104: Tam olarak yapılamadı hizmeti [org.apache.oozie.service.ShareLibService] başlat, Değil sharelib’i önbelleğe alabilir. Bir Yöneticinin sharelib’i aşağıdakilerle yüklemesi gerekir: oozie-setup.sh ve güncellemek için ‘oozie admin’ CLI komutunu verin paylaşım dosyası

Bunu önlemek için oozie-site.xml dosyasını aşağıdaki gibi değiştirin

cd $HOME/oozie
vi conf/oozie-site.xml

Mülkün altına ekleyin

<property>
    <name>oozie.service.HadoopAccessorService.hadoop.configurations</name> 
    <value>*=/usr/local/hadoop/etc/hadoop/</value>
</property>

Değer, tüm hadoop yapılandırma dosyalarının bulunduğu $HADOOP_HOME/etc/hadoop’unuz olmalıdır.

Adım 3.5 : Oozie DB Oluşturun

cd $HOME/oozie
./bin/ooziedb.sh create -sqlfile oozie.sql -run

Adım 3.6: Daemon’u başlatın

Oozie’yi arka plan programı olarak başlatmak için aşağıdaki komutu kullanın:

./bin/oozied.sh start

durdurmak için

./bin/oozied.sh stop

varsa, günlükleri hatalar için kontrol edin

cd $HOME/oozie/logs
tail -100f oozie.log

Oozie’nin durumunu komut satırından kontrol etmek için aşağıdaki komutu kullanın:

$ ./bin/oozie admin -oozie http://localhost:11000/oozie -status
System mode: NORMAL

Adım 4: İstemci Kurulumu

$ cd
$ cp oozie/oozie-client-4.3.0.tar.gz .
$ tar -xvf oozie-client-4.3.0.tar.gz
$ mv oozie-client-3.3.2 oozie-client
$ cd bin

.bashrc dosyasındaki PATH değişkenine $HOME/oozie-client/bin ekleyin ve terminalinizi yeniden başlatın veya

source $HOME/.bashrc

Kurulumla ilgili daha fazla ayrıntı için bu URL’ye başvurabilirsiniz. https://oozie.apache.org/docs/4.3.0/DG_QuickStart.html

Artık hadoop işlerini terminalinizde oozie’ye gönderebilirsiniz.

Bir örneği çalıştırmak için bu URL’yi takip edebilir ve ilk örneğinizi çalışacak şekilde ayarlayabilirsiniz. https://oozie.apache.org/docs/4.3.0/DG_Examples.html

Yukarıdaki URL’deki harita azaltma örneğini çalıştırırken aşağıdaki hatayla karşılaşabilirsiniz.

java.io.IOException: java.net.ConnectException: Şuradan Arayın localhost.localdomain/127.0.0.1 - 0.0.0.0:10020 bağlantıda başarısız oldu istisna: java.net.ConnectException: Bağlantı reddedildi; Daha fazlası için ayrıntılara bakın: http://wiki.apache.org/hadoop/ConnectionRefused

Çözüm: mr-jobhistory-server.sh’yi başlatın

cd $HADOOP_HOME/sbin
./mr-jobhistory-server.sh start historyserver

Job.properties dosyasını değiştirme konusunda dikkat edilmesi gereken bir diğer nokta da şudur:

nameNode=hdfs://localhost:9000
jobTracker=localhost:8032

senin durumunda bu farklı olabilir, ben apache hadoop kullanıyorum, sen cloudera/hdp/herhangi bir şey kullanıyor olabilirsin

To run spark job, I have tried running in local[*], yarn-client and
yarn-cluster as master, but succeeded in local[*] only
