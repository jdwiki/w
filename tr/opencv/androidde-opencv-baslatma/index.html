Zaman Uyumsuz Başlatma #

Zaman uyumsuz başlatma kullanmak, uygulama geliştirme için önerilen bir yoldur. Hedef sistemde harici olarak kurulan OpenCV kitaplıklarına erişmek için OpenCV Yöneticisi kullanır.

Zaman uyumsuz başlatmayı uygulayan kod parçacığı:

public class MainActivity extends Activity implements CvCameraViewListener2 {

    private BaseLoaderCallback mLoaderCallback = new BaseLoaderCallback(this) {
        @Override
        public void onManagerConnected(int status) {
            switch(status) {
                case LoaderCallbackInterface.SUCCESS:
                    Log.i(TAG,"OpenCV Manager Connected");
                    //from now onwards, you can use OpenCV API
                    Mat m = new Mat(5, 10, CvType.CV_8UC1, new Scalar(0));
                    break;
                case LoaderCallbackInterface.INIT_FAILED:
                    Log.i(TAG,"Init Failed");
                    break;
                case LoaderCallbackInterface.INSTALL_CANCELED:
                    Log.i(TAG,"Install Cancelled");
                    break;
                case LoaderCallbackInterface.INCOMPATIBLE_MANAGER_VERSION:
                    Log.i(TAG,"Incompatible Version");
                    break;
                case LoaderCallbackInterface.MARKET_ERROR:
                    Log.i(TAG,"Market Error");
                    break;
                default:
                    Log.i(TAG,"OpenCV Manager Install");
                    super.onManagerConnected(status);
                    break;
            }
        }
    };

    @Override
    public void onResume() {
        super.onResume();
        OpenCVLoader.initAsync(OpenCVLoader.OPENCV_VERSION_3_1_0, this, mLoaderCallback);
    }

    ...
}

Bu durumda uygulamamız OpenCV Manager ile asenkron olarak çalışmaktadır. Başlatma tamamlandığında, UI iş parçacığında “OnManagerConnected” geri çağrısı çağrılacak.

Lütfen OpenCV çağrılarının kullanılmasına veya yüklenmesine izin verilmediğini unutmayın. Bu geri aramayı başlatmadan önce OpenCV’ye bağlı yerel kütüphaneler. Yükle başarılı OpenCV’den sonra OpenCV’ye bağlı olan kendi yerel kitaplıklarınız başlatma.

Varsayılan “BaseLoaderCallback” uygulaması, uygulama bağlamını “Activity” olarak ele alır ve başlatma hatası durumunda çıkmak için “Activity.finish()” yöntemini çağırır. Bu davranışı geçersiz kılmak için, “BaseLoaderCallback” sınıfının “finish()” yöntemini geçersiz kılmanız ve kendi sonlandırma yönteminizi uygulamanız gerekir.

OpenCV Yöneticisi #

OpenCV Manager, son kullanıcı cihazlarında OpenCV kitaplığı ikili dosyalarını yönetmeyi hedefleyen bir Android hizmetidir. OpenCV dinamik kitaplıklarının aynı cihazdaki uygulamalar arasında paylaşılmasına izin verir.

Yönetici aşağıdaki avantajları sağlar:

  • Daha az bellek kullanımı (yaklaşık 40MB). Tüm uygulamalar hizmetten aynı ikili dosyaları kullanır ve yerel kitaplıkları kendi içlerinde tutmaz.
  • Desteklenen tüm platformlar için donanıma özel optimizasyonlar.
  • Güvenilir OpenCV kitaplığı kaynağı. OpenCV içeren tüm paketler Google Play marketinde yayınlanmaktadır.
  • Düzenli güncellemeler ve hata düzeltmeleri.

Tek dezavantajı, kullanıcıdan ekstra uygulama indirmesi ve ekstra uygulama istemesidir, bu nedenle kullanıcı deneyimi biraz azalır.

Daha fazla bilgi: Android OpenCV Yöneticisi

18/10/16 güncellendi: Play Store üzerinde dağıtılan OpenCV Manager sürümünde bir hata var (21/09/15 güncellendi). Yalnızca OpenCV 3.1.0 sürümünü etkiler. Bazılarını çalıştırdığınızda OpenCV işlevlerinde SIGSEGV hatası alırsınız. Android SDK ile dağıtılan sürüm iyi çalışıyor (OpenCV-android-sdk/apk/OpenCV_3.1.0_Manager_3.10_{platform}.apk). OpenCV web sitesinden indirilebilir. Daha fazla bilgi: Sayı #6247.

Statik Başlatma #

Bu yaklaşıma göre tüm OpenCV ikili dosyaları uygulama paketinize dahil edilir. Çoğunlukla geliştirme ve hata ayıklama amacıyla tasarlanmıştır. Bu yaklaşım üretim kodu için kullanımdan kaldırıldı, zaman uyumsuz başlatma önerilir.

Uygulama projenizin bir JNI parçası yoksa, ilgili OpenCV yerel kitaplıklarını ‘OpenCV-3.1.0-android-sdk/sdk/native/libs’den proje dizininize ‘app/src/main/ klasörüne kopyalamanız yeterlidir. jniLibs`.

Uygulama projesinin JNI parçası olması durumunda, kitaplıkların manuel olarak kopyalanması yerine, “Android.mk” dosyanızı değiştirmeniz gerekir: “include $(CLEAR_VARS)"‘dan sonra ve “include’dan önce aşağıdaki iki kod satırını ekleyin. path_to_OpenCV-3.1.0-android-sdk/sdk/native/jni/OpenCV.mk”`:

OPENCV_CAMERA_MODULES:=on
OPENCV_INSTALL_MODULES:=on

Sonuç aşağıdaki gibi görünmelidir:

include $(CLEAR_VARS)
# OpenCV
OPENCV_CAMERA_MODULES:=on
OPENCV_INSTALL_MODULES:=on
include ../../sdk/native/jni/OpenCV.mk

Bundan sonra OpenCV kitaplıkları, JNI derlemesi sırasında uygulamanızın jniLibs klasörüne kopyalanacaktır.

Uygulamanızda OpenCV’yi etkinleştirmenin son adımı, OpenCV API’sini çağırmadan önce Java başlatma kodudur. Örneğin, Activity sınıfının statik bölümünde yapılabilir:

static {
    if (!OpenCVLoader.initDebug()) {
        // Handle initialization error
    }
}

Uygulamanız OpenCV’ye bağlı başka yerel kitaplıklar içeriyorsa, bunları OpenCV başlatıldıktan sonra yüklemelisiniz:

static {
    if (!OpenCVLoader.initDebug()) {
        // Handle initialization error
    } else {
        System.loadLibrary("my_jni_lib1");
        System.loadLibrary("my_jni_lib2");
    }
}

Not: initDebug() yöntemi, üretim kodu için kullanımdan kaldırılmıştır. Yalnızca deneysel ve yerel geliştirme amaçları için tasarlanmıştır. Uygulamanızı yayınlamak istiyorsanız, zaman uyumsuz başlatma ile yaklaşımı kullanın.
