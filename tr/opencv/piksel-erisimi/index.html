Karşılaştığınız “cv::Mat” türünün farkında olmaya dikkat edin. Örneğin, ‘CV_8UC3’ türünde bir ‘cv::Mat’ dosyanız varsa ve buna ‘image.at(r,c)’ ile erişirseniz, hata oluşmaz, ancak programınız bazı beklenmeyen davranışlara sahip olacaktır.

cv::Mat::ptr işaretçisini kullanarak verimli piksel erişimi #

Verimlilik önemliyse, bir “cv::Mat” nesnesindeki pikselleri yinelemenin hızlı bir yolu, “r” (0 tabanlı dizin).

Matris tipine göre işaretçi farklı bir şablona sahip olacaktır.

  • “CV_8UC1” için: “char* ptr = image.ptr(r);’
  • “CV_8UC3” için: “cv::Vec3b* ptr = image.ptrcv::Vec3b(r);`
  • “CV_32FC1” için: “float* ptr = image.ptr(r);`
  • “CV_32FC3” için: “cv::Vec3f* ptr = image.ptrcv::Vec3f(r);`

Bu “ptr” nesnesi daha sonra “ptr[c]” çağrılarak “r” satırı ve “c” sütunundaki piksel değerine erişmek için kullanılabilir.

Bunu göstermek için, diskten bir görüntü yüklediğimiz ve Piksel piksel çalıştırarak Mavi ve Kırmızı kanallarını ters çevirdiğimiz bir örnek:

#include <opencv2/core.hpp>
#include <opencv2/imgproc.hpp>
#include <opencv2/highgui.hpp>

int main(int argc, char** argv) {
    cv::Mat image = cv::imread("image.jpg", CV_LOAD_IMAGE_COLOR);

    if(!image.data) {
        std::cout << "Error: the image wasn't correctly loaded." << std::endl;
        return -1;
    }

    // We iterate over all pixels of the image
    for(int r = 0; r < image.rows; r++) {
        // We obtain a pointer to the beginning of row r
        cv::Vec3b* ptr = image.ptr<cv::Vec3b>(r);

        for(int c = 0; c < image.cols; c++) {
            // We invert the blue and red values of the pixel
            ptr[c] = cv::Vec3b(ptr[c][2], ptr[c][1], ptr[c][0]);
        }
    }

    cv::imshow("Inverted Image", image);
    cv::waitKey();

    return 0;
}

cv::Mat::at() ile bireysel piksel değerlerine erişin #

Bir OpenCV “cv::Mat” nesnesindeki piksel değerlerine erişmek için önce matrisinizin türünü bilmeniz gerekir.

En yaygın türler şunlardır:

  • 8 bitlik 1 kanallı gri tonlamalı görüntüler için “CV_8UC1”;
  • 32 bit kayan noktalı 1 kanallı gri tonlamalı görüntüler için ‘CV_32FC1’;
  • 8 bitlik 3 kanallı renkli görüntüler için “CV_8UC3”; ve
  • 32 bit kayan noktalı 3 kanallı renkli görüntüler için ‘CV_32FC3’.

“cv::imread” ile varsayılan ayar bir “CV_8UC3” matrisi oluşturacaktır.

Tek tek piksellere erişmek için, en verimli olmasa da en güvenli yol, “r"nin matrisin *satırı olduğu ve “cv::Mat::at(r,c)” yöntemini kullanmaktır. c` sütundur. Şablon argümanı, matrisin türüne bağlıdır.

Diyelim ki bir “cv::Mat resminiz” var. Türüne göre erişim yöntemi ve piksel renk türü farklı olacaktır.

  • “CV_8UC1” için: “char pixelGrayValue = image.at(r,c)’.
  • “CV_8UC3” için: “cv::Vec3b pixelColor = image.atcv::Vec3b(r,c)’. “cv::Vec3b” nesnesi, “uchar” değerlerinin (0 ile 255 arasındaki tam sayılar) üçlüsünü temsil eder.
  • “CV_32FC1” için: “float pixelGrayValue = image.at(r,c)’.
  • “CV_32FC3” için: “cv::Vec3f pixelColor = image.atcv::Vec3f(r,c)’. “cv::Vec3f” nesnesi, “float” değerlerinin üçlüsünü temsil eder.

OpenCV’nin resimleri sıra-ana sırayla temsil ettiğini unutmayın, ör. Matlab veya Cebir’deki sözleşme olarak. Böylece, piksel koordinatlarınız (x,y) ise, piksele image.at<..>(y,x) kullanarak erişeceksiniz.

Alternatif olarak, “at<>” tek bir “cv::Point” bağımsız değişkeni aracılığıyla erişimi de destekler. Bu durumda, erişim sütun-büyük:

image.at<..>(cv::Point(x,y));

Bu yöntemle ilgili daha fazla ayrıntı için OpenCV belgelerine bakın.

Matiterator ile alternatif piksel erişimi #

Pikseller arasında yineleme yapmanın en iyi yolu bu değil; ancak, cv::Mat::at<T>‘den daha iyidir.

Diyelim ki klasörünüzde renkli bir görüntü var ve bu görüntünün her pikselini tekrarlamak ve yeşil ve kırmızı kanalları silmek istiyorsunuz (Bunun bir örnek olduğunu unutmayın, bunu daha optimize yollarla yapabilirsiniz);

#include <opencv2/core/core.hpp>
#include <opencv2/highgui/highgui.hpp> 


int main(int argc, char **argv)
{

// Create a container
cv::Mat im; 

//Create a vector
cv::Vec3b *vec;

// Create an mat iterator
cv::MatIterator_<cv::Vec3b> it;

// Read the image in color format
im = cv::imread("orig1.jpg", 1);

// iterate through each pixel
for(it = im.begin<cv::Vec3b>(); it != im.end<cv::Vec3b>(); ++it)
{
    // Erase the green and red channels 
    (*it)[1] = 0;
    (*it)[2] = 0;
}


// Create a new window
cv::namedWindow("Resulting Image");

// Show the image
cv::imshow("Resulting Image", im);

// Wait for a key
cv::waitKey(0);

return 0;
}

Bunu Cmake ile derlemek için:

cmake_minimum_required(VERSION 2.8)
project(Main)
find_package(OpenCV REQUIRED)
add_executable(Main main.cpp)
target_link_libraries(Main ${OpenCV_LIBS})

Mat’ta Piksel Erişimi #

OpenCV Mat yapısında bireysel piksel erişimi birden fazla yolla sağlanabilir. Nasıl erişileceğini anlamak için önce veri türlerini öğrenmek daha iyidir.

https://www.wikiod.com/tr/opencv/temel-yaplar temel veri türlerini açıklar. Kısaca, CV_<bit-depth>{U|S|F}C(<number_of_channels>) bir türün temel yapısıdır. Bununla birlikte, ‘Vec’ yapılarını anlamak önemlidir.

typedef Vec<type, channels> Vec< channels>< one char for the type>

burada tür ‘uchar, kısa, int, float, double’dan biridir ve her tür için karakterler sırasıyla ‘b, s, i, f, d’dir.

Örneğin, Vec2b “2 kanalın işaretsiz karakter vektörünü” belirtir.

R’nin #satırlar, C’nin #sütunlar ve T’nin tür olduğu ‘Mat mat(R,C,T)‘yi düşünün. “mat"ın (i,j) koordinatına erişim için bazı örnekler:

2D:

If the type is CV_8U or CV_8UC1 ---- //they are alias
mat.at<uchar>(i,j) // --> This will give char value of index (i,j)
//If you want to obtain int value of it
(int)mat.at<uchar>(i,j)

If the type is CV_32F or CV_32FC1 ---- //they are alias
mat.at<float>(i,j) // --> This will give float value of index (i,j)

3 BOYUTLU:

If the type is CV_8UC2 or CV_8UC3 or more channels
mat.at<Vec2b/Vec3b>(i,j)[k] // note that (k < #channels)
//If you want to obtain int value of it
(int)mat.at<uchar>(i,j)[k] 

If the type is CV_64FC2 or CV_64FC3 
mat.at<Vec2d/Vec3d>(i,j)[k] // note that k < #channels

<...> içine doğru türü girmek çok önemlidir, aksi takdirde çalışma zamanı hatası veya istenmeyen sonuçlarla karşılaşabilirsiniz.

Gri bir görüntünün piksel değerlerini C++ ile ayarlama ve alma #

// PixelAccessTutorial.cpp : Defines the entry point for the console 
// Environment: Visual studio 2015, Windows 10
// Assumptions: Opecv is installed configured in the visual studio project
// Opencv version: OpenCV 3.1

#include "stdafx.h"
#include<opencv2/core/core.hpp>
#include<opencv2/highgui/highgui.hpp>
#include<opencv2/imgproc/imgproc.hpp>
#include<string>
#include<iostream>

int main()
{

    cv::Mat imgOriginal;        // input image
    cv::Mat imgGrayscale;        // grayscale of input image

    std::cout << "Please enter an image filename : ";
    std::string img_addr;
    std::cin >> img_addr;

    std::cout << "Searching for " + img_addr << std::endl;

    imgOriginal = cv::imread(img_addr);            // open image

        if (imgOriginal.empty()) {                                    // if unable to open image
            std::cout << "error: image not read from file\n\n";        // show error message on command line
            return(0);                                                // and exit program
        }

    cv::cvtColor(imgOriginal, imgGrayscale, CV_BGR2GRAY);        // convert to grayscale

    const int channels = imgGrayscale.channels();
    printf("Number of channels = %d", channels);

    cv::Mat output ;
    imgGrayscale.copyTo(output); // Just to make sure the Mat objects are of the same size. 

    //Set the threshhold to your desired value
    uchar threshhold = 127; 

    if (channels == 1)
    {
        for (int x = 0; x<imgGrayscale.rows; x++) {
            for (int y = 0; y<imgGrayscale.cols; y++) {
                // Accesssing values of each pixel
                if (imgGrayscale.at<uchar>(x, y) >= threshhold) {
                    // Setting the pixel values to 255 if it's above the threshold
                    output.at<uchar>(x, y) = 254;
                }
                else if (imgGrayscale.at<uchar>(x, y) < threshhold) {
                    // Setting the pixel values to 255 if it's below the threshold
                    output.at<uchar>(x, y) = 0;
                }
                else {
                    // Just in case
                    printf("The value at (%d, %d) are not right. Value: %d\n", x, y, imgGrayscale.at<uchar>(x, y));
                }
            }
        }
    }
    else if (channels == 3)
    {
        // This is only for  gray scale images
        printf("\tThe image has 3 channels. The function does not support images with 3 channels.\n");
    }

    //Create windows to show image
    cv::namedWindow("Gray scale", CV_WINDOW_AUTOSIZE);    
    cv::namedWindow("Binary", CV_WINDOW_AUTOSIZE);        

    cv::imshow("Gray scale", imgGrayscale);        
    cv::imshow("Binary", output);

    cv::waitKey(0);                    // hold windows open until user presses a key

    return 0;
}
