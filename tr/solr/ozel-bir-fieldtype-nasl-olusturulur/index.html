Özel bir fieldType nasıl oluşturulur

Özel Java sınıfı tabanlı alanla ilgili açıklamalar:

Bu, Solr belgelerini özel alan karşılaştırıcısına göre sıralamak için yazılmış büyük bir makalenin harici alan kullanarak Solr’da özel sıralama küçük bir bölümüdür.

Mevcut Solr alanlarından oluşturulan özel alanla ilgili açıklamalar:

Apache bu konuyla ilgili ayrıntılı bir belge oluşturmuştur - Analizörleri, Belirteçleri ve Filtreleri Anlamak.

Kendi özel Java sınıfından özel bir Solr alan türü oluşturun #

Şema değişiklikleri:

Solr şema dosyanızda yeni bir alan türü tanımlamanız gerekecek ve ardından bu türden alanlar oluşturabilirsiniz. Örnek şema snippet’i:

<!-- Source: solr/example/.../conf/schema.xml -->
<?xml version="1.0" encoding="UTF-8" ?>
<schema name="adam" version="1.3">
  <types>
    ...
    <fieldType name="rank_t" class="org.apache.solr.schema.ext.RankFieldType"/>
  </types>
 <fields>
   ...
   <field name="rank" type="rank_t" indexed="true" stored="true"/>
 </fields>
 ...
</schema>

Özel alan türü için Java sınıfı:

// Source: src/java/org/apache/solr/schema/ext/RankFieldType.java
package org.apache.solr.schema.ext;

import java.io.IOException;

import org.apache.lucene.document.Fieldable;
import org.apache.lucene.search.SortField;
import org.apache.solr.response.TextResponseWriter;
import org.apache.solr.schema.FieldType;
import org.apache.solr.schema.SchemaField;
import org.apache.solr.search.ext.RankFieldComparatorSource;

public class RankFieldType extends FieldType {

    @Override
    public SortField getSortField(SchemaField field, boolean top) {
        return new SortField(field.getName(), new RankFieldComparatorSource(), top);
    }

    @Override
    // copied verbatim from GeoHashField method
    public void write(TextResponseWriter writer, String name, Fieldable f) throws IOException {
         writer.writeStr(name, f.stringValue(), false);
    }
}

Mevcut alan türlerinden özel alan türü oluşturun #

Örneğe geçmeden önce biraz teorik bilgi alalım. Burada kullanılan üç önemli terim vardır Analyzers, Tokenizers ve Filters. Böyle özel bir alan oluşturmak için bir belirteç ve bir veya daha fazla filtre içeren bir analizör oluşturmanız gerekecektir. Burada belirtildiği gibi, analizör başına yalnızca bir belirteç oluşturabilirsiniz, ancak bu sınırlamanın üstesinden gelmenin yolları vardır.

<fieldType name="alphaOnlySort" class="solr.TextField" sortMissingLast="true" omitNorms="true">
  <analyzer>
    <tokenizer class="solr.KeywordTokenizerFactory"/>
    <filter class="solr.LowerCaseFilterFactory"/>
    <filter class="solr.TrimFilterFactory"/>
    <filter class="solr.PatternReplaceFilterFactory" replace="all" replacement="" pattern="([^a-z])"/>
  </analyzer>
</fieldType>

Başka bir örnek:

<fieldType name="lowercase_text" class="solr.TextField" positionIncrementGap="150">
  <analyzer>
     <tokenizer class="solr.KeywordTokenizerFactory" />
     <filter class="solr.LowerCaseFilterFactory" />
  </analyzer>
</fieldType>

Açıklamalı bir örnek daha:

<fieldType name="text_stem" class="solr.TextField">
  <analyzer>
    <tokenizer class="solr.StandardTokenizerFactory"/>
    <filter class="solr.StandardFilterFactory"/>
    <filter class="solr.LowerCaseFilterFactory"/>
    <filter class="solr.EnglishPorterFilterFactory"/>
  </analyzer>
</fieldType>

Bu örnek, Solr’un, alanın metnini belirteçlere bölen standart belirteci ile başlar. Bu belirteçler daha sonra Solr’un kısaltmalardan noktaları kaldıran ve birkaç diğer ortak işlemi gerçekleştiren standart filtresinden geçer. Tüm belirteçler daha sonra küçük harfe ayarlanır ve bu, sorgu zamanında büyük/küçük harfe duyarsız eşleştirmeyi kolaylaştıracaktır. Yukarıdaki örnekteki son filtre, Porter kök ayırma algoritmasını kullanan bir kök ayırıcı filtredir. Kök ayırıcı temel olarak, bir sözcüğün çeşitli biçimlerini türettikleri kök ya da kök sözcükle eşleyen bir dizi eşleme kuralıdır. Örneğin, İngilizce’de “sarılmak”, “sarılmak” ve “sarılmak” sözcüklerinin tümü, “sarılmak” kök sözcüğünün biçimleridir. Kök ayırıcı, tüm bu terimleri, dizine eklenecek olan “sarılma” ile değiştirecektir. Bu, “sarılma” sorgusunun “sarılma” terimiyle eşleşeceği, ancak “büyük” ile eşleşmeyeceği anlamına gelir.

Bu tür özel alanın örnek kullanımı:

<field name="keywords" type="text_stem" indexed="true" stored="true" />
