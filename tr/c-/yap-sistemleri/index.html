C++, C gibi, derleme iş akışları ve oluşturma süreçleriyle ilgili uzun ve çeşitli bir geçmişe sahiptir. Bugün, C++, programları derlemek için kullanılan çeşitli popüler derleme sistemlerine sahiptir, bazen tek bir yapı sistemi içinde birden çok platform için. Burada, birkaç yapı sistemi gözden geçirilecek ve analiz edilecektir.

Şu anda, hem popüler hem de platformlar arası C++ için evrensel veya baskın bir yapı sistemi yoktur. Bununla birlikte, büyük platformlara/projelere bağlı birkaç büyük yapı sistemi vardır, bunların en dikkate değer olanı GNU/Linux işletim sistemiyle GNU Make ve Visual C++/Visual Studio proje sistemiyle NMAKE’dir.

Ek olarak, bazı Entegre Geliştirme Ortamları (IDE’ler), yerel IDE ile özel olarak kullanılacak özel yapım sistemlerini de içerir. Belirli yapı sistemi oluşturucuları, Eclipse için CMake ve Microsoft Visual Studio 2012 gibi bu yerel IDE yapı sistemi/proje biçimlerini oluşturabilir.

CMake ile Derleme Ortamı Oluşturma #

CMake, neredeyse tüm derleyiciler veya IDE için tek bir proje tanımından derleme ortamları oluşturur. Aşağıdaki örnekler, platformlar arası “Merhaba Dünya” C++ koduna bir CMake dosyasının nasıl ekleneceğini gösterecektir.

CMake dosyaları her zaman “CMakeLists.txt” olarak adlandırılır ve her projenin kök dizininde (ve muhtemelen alt dizinlerde de) zaten bulunmalıdır. Temel bir CMakeLists.txt dosyası şuna benzer:

cmake_minimum_required(VERSION 2.4)

project(HelloWorld)

add_executable(HelloWorld main.cpp)

Coliru’da canlı görün.

Bu dosya, CMake’e proje adını, hangi dosya sürümünün beklendiğini ve main.cpp gerektiren “HelloWorld” adlı bir yürütülebilir dosya oluşturma talimatlarını söyler.

Komut satırından kurulu derleyiciniz/IDE için bir yapı ortamı oluşturun:

> cmake .

Uygulamayı şununla oluşturun:

> cmake --build .

Bu, işletim sistemine ve kurulu araçlara bağlı olarak sistem için varsayılan yapı ortamını oluşturur. “Kaynak dışı” derlemeleri kullanarak kaynak kodunu tüm derleme yapılarından uzak tutun:

> mkdir build
> cd build
> cmake ..
> cmake --build .

CMake ayrıca platform kabuğunun temel komutlarını önceki örnekten soyutlayabilir:

> cmake -E make_directory build
> cmake -E chdir build cmake .. 
> cmake --build build

CMake, bir dizi yaygın derleme aracı ve IDE için jeneratörler içerir. Visual Studio’nun nmake için makefile dosyaları oluşturmak için:

> cmake -G "NMake Makefiles" ..
> nmake

GNU make ile derleme #

Giriiş #

GNU Make (tarz ‘make’), kabuk komutlarının yürütülmesinin otomasyonuna adanmış bir programdır. GNU Make, Make ailesine giren özel bir programdır. Make, Linux çekirdeği, Mac OS X ve BSD’den türetilenler de dahil olmak üzere Unix benzeri ve POSIX benzeri işletim sistemleri arasında popülerliğini koruyor.

GNU Make, popüler GNU/Linux işletim sistemine bağlı olan GNU Projesine bağlı olmasıyla özellikle dikkate değerdir. GNU Make ayrıca Windows ve Mac OS X’in çeşitli türlerinde çalışan uyumlu sürümlere sahiptir. Aynı zamanda, popülerliğini koruyan tarihsel önemi olan çok kararlı bir sürümdür. Bu nedenlerden dolayı GNU Make genellikle C ve C++ ile birlikte öğretilir.

Temel kurallar #

make ile derlemek için proje dizininizde bir Makefile oluşturun. Makefile’ınız şu kadar basit olabilir:

Yapım dosyası

# Set some variables to use in our command
# First, we set the compiler to be g++
CXX=g++

# Then, we say that we want to compile with g++'s recommended warnings and some extra ones.
CXXFLAGS=-Wall -Wextra -pedantic

# This will be the output file
EXE=app

SRCS=main.cpp

# When you call `make` at the command line, this "target" is called.
# The $(EXE) at the right says that the `all` target depends on the `$(EXE)` target.
# $(EXE) expands to be the content of the EXE variable
# Note: Because this is the first target, it becomes the default target if `make` is called without target
all: $(EXE)

# This is equivalent to saying
# app: $(SRCS)
# $(SRCS) can be separated, which means that this target would depend on each file.
# Note that this target has a "method body": the part indented by a tab (not four spaces).
# When we build this target, make will execute the command, which is:
# g++ -Wall -Wextra -pedantic -o app main.cpp
# I.E. Compile main.cpp with warnings, and output to the file ./app
$(EXE): $(SRCS)
    @$(CXX) $(CXXFLAGS) -o $@ $(SRCS)

# This target should reverse the `all` target. If you call
# make with an argument, like `make clean`, the corresponding target
# gets called.
clean:
    @rm -f $(EXE)

NOT: Girintilerin dört boşluklu değil, sekmeli olduğundan kesinlikle emin olun. Aksi takdirde, `Makefile:10:*** eksik ayırıcı hatası alırsınız. Dur.’

Bunu komut satırından çalıştırmak için aşağıdakileri yapın:

$ cd ~/Path/to/project
$ make
$ ls
app  main.cpp  Makefile

$ ./app
Hello World!

$ make clean
$ ls
main.cpp  Makefile

Artımlı yapılar #

Daha fazla dosyaya sahip olmaya başladığınızda, make daha kullanışlı hale gelir. a.cpp‘yi düzenlerken b.cpp‘yi düzenlemediyseniz ne olur? b.cpp dosyasının yeniden derlenmesi daha fazla zaman alacaktır.

Aşağıdaki dizin yapısıyla:

.
+-- src
|   +-- a.cpp
|   +-- a.hpp
|   +-- b.cpp
|   +-- b.hpp
+-- Makefile

Bu iyi bir Makefile olurdu:

Yapım dosyası

CXX=g++
CXXFLAGS=-Wall -Wextra -pedantic
EXE=app

SRCS_GLOB=src/*.cpp
SRCS=$(wildcard $(SRCS_GLOB))
OBJS=$(SRCS:.cpp=.o)

all: $(EXE)

$(EXE): $(OBJS)
    @$(CXX) -o $@ $(OBJS)

depend: .depend

.depend: $(SRCS)
    @-rm -f ./.depend
    @$(CXX) $(CXXFLAGS) -MM $^>>./.depend

clean:
    -rm -f $(EXE)
    -rm $(OBJS)
    -rm *~
    -rm .depend

include .depend

Sekmeleri tekrar izleyin. Bu yeni Makefile, yalnızca değiştirilen dosyaları yeniden derlemenizi sağlayarak derleme süresini en aza indirir.

SCons ile Bina #

[Scons] kullanarak platformlar arası “Merhaba Dünya” C++ kodunu oluşturabilirsiniz. ](http://scons.org/) - Bir Python dilinde yazılım oluşturma aracı.

İlk olarak, SConstruct adında bir dosya oluşturun (SCons’un varsayılan olarak bu tam ada sahip bir dosya arayacağını unutmayın). Şimdilik dosya, hello.cpp dosyanızın hemen yanındaki bir dizinde olmalıdır. Yeni dosyaya satırı yazın

 Program('hello.cpp')

Şimdi, terminalden “scons” komutunu çalıştırın. gibi bir şey görmelisin

$ scons
scons: Reading SConscript files ...
scons: done reading SConscript files.
scons: Building targets ...
g++ -o hello.o -c hello.cpp
g++ -o hello hello.o
scons: done building targets.

(ayrıntılar işletim sisteminize ve kurulu derleyicinize bağlı olarak değişecek olsa da).

‘Environment’ ve ‘Glob’ sınıfları, ne inşa edeceğinizi daha fazla yapılandırmanıza yardımcı olacaktır. Örneğin, “SConstruct” dosyası

env=Environment(CPPPATH='/usr/include/boost/',
    CPPDEFINES=[],
    LIBS=[],
    SCONS_CXX_STANDARD="c++11"
    )

env.Program('hello', Glob('src/*.cpp'))

“src” içindeki tüm “cpp” dosyalarını kullanarak yürütülebilir “merhaba"yı oluşturur. ‘CPPPATH’, ‘/usr/include/boost’ şeklindedir ve C++11 standardını belirtir.

Otomatik Araçlar (GNU) #

Giriiş #

Autotools, belirli bir yazılım paketi için bir GNU Yapı Sistemi oluşturan bir grup programdır. Makefile (GNU Make ile kullanılacak) gibi çeşitli yapı kaynakları üretmek için birlikte çalışan bir araçlar paketidir. Bu nedenle, Autotools fiili bir yapı sistemi üreticisi olarak kabul edilebilir.

Bazı önemli Autotools programları şunları içerir:

  • Otomatik yapılandırma
  • Automake (‘make’ ile karıştırılmamalıdır)

Genel olarak, Autotools, aşağıdaki komutun çoğu paketi (basit durumda) oluşturmasına (ve yüklemesine) izin vermek için Unix uyumlu komut dosyasını ve Makefile’ı oluşturmayı amaçlar:

./configure && make && make install

Bu nedenle, Autotools’un belirli paket yöneticileriyle, özellikle de POSIX Standart(lar)ına uyan işletim sistemlerine bağlı olanlarla da bir ilişkisi vardır.

Ninja #

Giriiş #

Ninja yapı sistemi, proje web sitesinde [“hıza odaklanan küçük bir yapı sistemi” olarak tanımlanmaktadır.1 Ninja, dosyalarının derleme sistemi dosya oluşturucuları tarafından oluşturulması için tasarlanmıştır ve aşağıdakilere düşük seviyeli bir yaklaşım benimser. CMake veya Meson gibi üst düzey yapı sistemi yöneticilerinin aksine sistemler oluşturun.

Ninja, öncelikle C++ ve Python ile yazılmıştır ve Chromium projesi için SCons derleme sistemine bir alternatif olarak oluşturulmuştur.

NMAKE (Microsoft Program Bakım Yardımcı Programı) #

Giriiş #

NMAKE, öncelikle Microsoft Visual Studio ve/veya Visual C++ komut satırı araçlarıyla birlikte kullanılmak üzere Microsoft tarafından geliştirilmiş bir komut satırı yardımcı programıdır.

NMAKE, Make yapı sistemleri ailesinin altına giren bir yapı sistemidir, ancak Windows’a özgü dosya yolu sözdizimini (kendisi Unix tarzı dosya yollarından farklıdır) desteklemek gibi Unix benzeri Make programlarından ayrılan belirli farklı özelliklere sahiptir.
