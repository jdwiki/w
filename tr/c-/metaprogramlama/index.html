C++‘da Metaprogramlama, derleme zamanında kod oluşturmak için makroların veya şablonların kullanımını ifade eder.

Genel olarak, bu rolde makrolar hoş karşılanmaz ve genel olmasa da şablonlar tercih edilir.

Şablon metaprogramlaması, kod oluşturma hedeflerine ulaşmak için şablonlar veya “constexpr” işlevleri aracılığıyla, genellikle derleme zamanı hesaplamalarını kullanır, ancak derleme zamanı hesaplamaları kendi başına metaprogramlama değildir.

Metaprogramlama (veya daha spesifik olarak, Şablon Metaprogramlama), derleme zamanında sabitler, işlevler veya veri yapıları oluşturmak için şablonlar kullanma uygulamasıdır. Bu, hesaplamaların her çalıştırma zamanında değil, derleme zamanında bir kez yapılmasına izin verir.

Faktöriyelleri Hesaplama #

Faktöriyeller, şablon metaprogramlama teknikleri kullanılarak derleme zamanında hesaplanabilir.

#include <iostream>

template<unsigned int n>
struct factorial
{
    enum
    {
        value = n * factorial<n - 1>::value
    };
};

template<>
struct factorial<0>
{
    enum { value = 1 };
};

int main()
{
    std::cout << factorial<7>::value << std::endl;    // prints "5040"
}

“faktöriyel” bir yapıdır, ancak şablon metaprogramlamasında bir şablon metafonksiyonu olarak ele alınır. Geleneksel olarak, şablon metafonksiyonları, belirli bir üye kontrol edilerek değerlendirilir; türlerle sonuçlanan metafonksiyonlar için “::type” veya değer üreten metafonksiyonlar için “::value”.

Yukarıdaki kodda, şablonu geçmek istediğimiz parametrelerle somutlaştırarak ve değerlendirmenin sonucunu almak için ‘::value’ kullanarak ‘faktöriyel’ metafonksiyonu değerlendiriyoruz.

Meta işlevin kendisi, aynı meta işlevi daha küçük değerlerle özyinelemeli olarak başlatmaya dayanır. “faktöriyel<0>” uzmanlığı, sonlandırma koşulunu temsil eder. Şablon metaprogramlama, bir işlevsel programlama dilinin kısıtlamalarının çoğuna sahiptir, bu nedenle özyineleme birincil “döngü” yapısıdır.

Şablon meta işlevleri derleme zamanında yürütüldüğünden, sonuçları derleme zamanı değerleri gerektiren bağlamlarda kullanılabilir. Örneğin:

int my_array[factorial<5>::value];

Otomatik diziler, derleme zamanı tanımlı bir boyuta sahip olmalıdır. Ve bir metafonksiyonun sonucu bir derleme zamanı sabitidir, yani burada kullanılabilir.

Sınırlama: Derleyicilerin çoğu, bir sınırın ötesinde özyineleme derinliğine izin vermez. Örneğin, “g++” derleyicisi varsayılan olarak özyineleme derinliğini 256 düzeyle sınırlar. g++ durumunda, programcı -ftemplate-depth-X seçeneğini kullanarak özyineleme derinliğini ayarlayabilir.

C++ 11’den beri, std::integral_constant şablonu bu tür şablon hesaplamaları için kullanılabilir:

#include <iostream>
#include <type_traits>

template<long long n>
struct factorial :
  std::integral_constant<long long, n * factorial<n - 1>::value> {};

template<>
struct factorial<0> :
  std::integral_constant<long long, 1> {};

int main()
{
    std::cout << factorial<7>::value << std::endl;    // prints "5040"
}

Ek olarak, “constexpr” işlevleri daha temiz bir alternatif haline gelir.

#include <iostream>

constexpr long long factorial(long long n)
{
  return (n == 0) ? 1 : n * factorial(n - 1);
}

int main()
{
  char test[factorial(3)];
  std::cout << factorial(7) << '\n';
}

Factorial()‘in gövdesi tek bir ifade olarak yazılır çünkü C++11’de ‘constexpr’ işlevleri dilin yalnızca oldukça sınırlı bir alt kümesini kullanabilir.

son sürüm

C++14’ten beri, “constexpr” işlevleri için birçok kısıtlama kaldırıldı ve artık çok daha rahat yazılabilirler:

constexpr long long factorial(long long n)
{
  if (n == 0)
    return 1;
  else
    return n * factorial(n - 1);
}

Ya da:

constexpr long long factorial(int n)
{
  long long result = 1;
  for (int i = 1; i <= n; ++i) {
    result *= i;
  }
  return result;
}
son sürüm

C++ 17’den beri, faktöriyel hesaplamak için kat ifadesi kullanılabilir:

#include <iostream>
#include <utility>

template <class T, T N, class I = std::make_integer_sequence<T, N>>
struct factorial;

template <class T, T N, T... Is>
struct factorial<T,N,std::index_sequence<T, Is...>> {
   static constexpr T value = (static_cast<T>(1) * ... * (Is + 1));
};

int main() {
   std::cout << factorial<int, 5>::value << std::endl;
}

Bir parametre paketi üzerinde yineleme #

Çoğu zaman, değişken bir şablon parametre paketindeki her öğe üzerinde bir işlem gerçekleştirmemiz gerekir. Bunu yapmanın birçok yolu vardır ve çözümlerin C++17 ile okunması ve yazılması daha kolay hale gelir. Bir paketteki her öğeyi yazdırmak istediğimizi varsayalım. En basit çözüm, tekrarlamaktır:

void print_all(std::ostream& os) {
    // base case
}

template <class T, class... Ts>
void print_all(std::ostream& os, T const& first, Ts const&... rest) {
    os << first;
    
    print_all(os, rest...);
}
son sürüm

Bunun yerine, tüm akışı tek bir işlevde gerçekleştirmek için genişletici hilesini kullanabiliriz. Bu, ikinci bir aşırı yüklemeye ihtiyaç duymama avantajına sahiptir, ancak yıldızların okunabilirliğinden daha az dezavantaja sahiptir:

template <class... Ts>
void print_all(std::ostream& os, Ts const&... args) {
    using expander = int[];
    (void)expander{0,
        (void(os << args), 0)...
    };
}
son sürüm

Bunun nasıl çalıştığına dair bir açıklama için TC’nin mükemmel yanıtına bakın.

C++17 ile bu sorunu çözmek için cephaneliğimize iki güçlü yeni araç alıyoruz. Birincisi bir kat ifadesidir:

template <class... Ts>
void print_all(std::ostream& os, Ts const&... args) {
    ((os << args), ...);
}

İkincisi ise orijinal özyinelemeli çözümümüzü tek bir fonksiyonda yazmamıza izin veren ‘if constexpr’dir:

template <class T, class... Ts>
void print_all(std::ostream& os, T const& first, Ts const&... rest) {
    os << first;

    if constexpr (sizeof...(rest) > 0) {        
        // this line will only be instantiated if there are further
        // arguments. if rest... is empty, there will be no call to
        // print_all(os). 
        print_all(os, rest...);
    }
}
son sürüm

std::integer_sequence ile yineleme #

C++ 14’ten beri standart, sınıf şablonunu sağlar

template <class T, T... Ints>
class integer_sequence;

template <std::size_t... Ints>
using index_sequence = std::integer_sequence<std::size_t, Ints...>;

ve bunun için bir üreten metafonksiyon:

template <class T, T N>
using make_integer_sequence = std::integer_sequence<T, /* a sequence 0, 1, 2, ..., N-1 */ >;

template<std::size_t N>
using make_index_sequence = make_integer_sequence<std::size_t, N>;

Bu, C++14’te standart olsa da, bu, C++11 araçları kullanılarak uygulanabilir.

Bu aracı, “std::tuple” argümanlı bir işlevi çağırmak için kullanabiliriz (C++17’de “std::apply” olarak standartlaştırılmıştır):

namespace detail {
    template <class F, class Tuple, std::size_t... Is>
    decltype(auto) apply_impl(F&& f, Tuple&& tpl, std::index_sequence<Is...> ) {
        return std::forward<F>(f)(std::get<Is>(std::forward<Tuple>(tpl))...);
    }
}

template <class F, class Tuple>
decltype(auto) apply(F&& f, Tuple&& tpl) {
    return detail::apply_impl(std::forward<F>(f),
        std::forward<Tuple>(tpl),
        std::make_index_sequence<std::tuple_size<std::decay_t<Tuple>>::value>{});
}


// this will print 3
int f(int, char, double);

auto some_args = std::make_tuple(42, 'x', 3.14);
int r = apply(f, some_args); // calls f(42, 'x', 3.14)

Etiket Gönderimi #

Derleme zamanında işlevler arasında seçim yapmanın basit bir yolu, bir (genellikle son) argüman olarak bir etiketi alan aşırı yüklenmiş bir işlev çiftine bir işlev göndermektir. Örneğin, std::advance() uygulamak için yineleyici kategorisine gönderebiliriz:

namespace details {
    template <class RAIter, class Distance>
    void advance(RAIter& it, Distance n, std::random_access_iterator_tag) {
        it += n;
    }

    template <class BidirIter, class Distance>
    void advance(BidirIter& it, Distance n, std::bidirectional_iterator_tag) {
        if (n > 0) {
            while (n--) ++it;
        }
        else {
            while (n++) --it;
        }
    }

    template <class InputIter, class Distance>
    void advance(InputIter& it, Distance n, std::input_iterator_tag) {
        while (n--) {
            ++it;
        }
    }    
}

template <class Iter, class Distance>
void advance(Iter& it, Distance n) {
    details::advance(it, n, 
            typename std::iterator_traits<Iter>::iterator_category{} );
}

Aşırı yüklenmiş “details::advance” işlevlerinin “std::XY_iterator_tag” bağımsız değişkenleri, kullanılmayan işlev parametreleridir. Gerçek uygulama önemli değil (aslında tamamen boş). Tek amaçları, derleyicinin, ‘details::advance’ etiket sınıfının çağrıldığına bağlı olarak bir aşırı yükleme seçmesine izin vermektir.

Bu örnekte, “advance”, gerçek “Iter” türüne bağlı olarak “iterator_tag” sınıflarından birini döndüren “iterator_traits::iterator_category” meta işlevini kullanır. “iterator_category::type” öğesinin varsayılan olarak oluşturulmuş bir nesnesi, derleyicinin farklı “details::advance” aşırı yüklemelerinden birini seçmesine izin verir. (Bu işlev parametresi, varsayılan olarak oluşturulmuş boş bir “yapı” nesnesi olduğundan ve hiç kullanılmadığından, büyük olasılıkla tamamen optimize edilmiştir.)

Etiket gönderme, size SFINAE ve “enable_if” kullanan eşdeğerlerine göre okunması çok daha kolay bir kod verebilir.

Not: C++17’nin ‘if constexpr’ özellikle ‘advance’ uygulamasını basitleştirebilse de, etiket gönderiminin aksine açık uygulamalar için uygun değildir.

İfadenin Geçerli Olup Olmadığını Tespit Et #

Bir tip üzerinde bir operatörün veya fonksiyonun çağrılabileceğini tespit etmek mümkündür. Bir sınıfın ‘std::hash’ aşırı yüklemesine sahip olup olmadığını test etmek için şunu yapabilirsiniz:

#include <functional> // for std::hash
#include <type_traits> // for std::false_type and std::true_type
#include <utility> // for std::declval

template<class, class = void>
struct has_hash
    : std::false_type
{};

template<class T>
struct has_hash<T, decltype(std::hash<T>()(std::declval<T>()), void())>
    : std::true_type
{};

C++17’den beri, bu tür yapıyı basitleştirmek için `std::void_t’ kullanılabilir

#include <functional> // for std::hash
#include <type_traits> // for std::false_type, std::true_type, std::void_t
#include <utility> // for std::declval

template<class, class = std::void_t<> >
struct has_hash
    : std::false_type
{};

template<class T>
struct has_hash<T, std::void_t< decltype(std::hash<T>()(std::declval<T>())) > >
    : std::true_type
{};

burada ‘std::void_t’ şu şekilde tanımlanır:

template< class... > using void_t = void;
son sürüm

‘Operatör<’ gibi bir operatörün tanımlanıp tanımlanmadığını tespit etmek için sözdizimi hemen hemen aynıdır:

template<class, class = void>
struct has_less_than
    : std::false_type
{};

template<class T>
struct has_less_than<T, decltype(std::declval<T>() < std::declval<T>(), void())>
    : std::true_type
{};

Bunlar, “T”, “std::hash” için aşırı yüklüyse bir “std::unordered_map” kullanmak için kullanılabilir, ancak aksi takdirde bir “std::map” kullanmayı deneyin:

template <class K, class V>
using hash_invariant_map = std::conditional_t<
    has_hash<K>::value,
    std::unordered_map<K, V>,
    std::map<K,V>>;

template<typename T>
struct ValueOrPointer
{

C++11 (ve üstü) ile güç hesaplama #

C++ 11 ve üstü ile derleme zamanında hesaplamalar çok daha kolay olabilir. Örneğin, derleme zamanında verilen bir sayının gücünü hesaplamak aşağıdaki gibi olacaktır:

template <typename T>
constexpr T calculatePower(T value, unsigned power) {
    return power == 0 ? 1 : value * calculatePower(value, power-1);
}

‘constexpr’ anahtar sözcüğü, derleme zamanında işlevi hesaplamaktan sorumludur, ancak o zaman ve ancak o zaman, bunun için tüm gereksinimler karşılandığında (daha fazla bilgi için constexpr anahtar sözcüğü referansına bakın), örneğin tüm argümanlar derleme zamanında bilinmelidir.

Not: C++11’de “constexpr” işlevi yalnızca bir dönüş ifadesinden oluşmalıdır.

Avantajlar: Bunu standart derleme zamanı hesaplama yöntemiyle karşılaştırdığımızda, bu yöntem çalışma zamanı hesaplamaları için de kullanışlıdır. Bunun anlamı, eğer fonksiyonun argümanları derleme zamanında bilinmiyorsa (örneğin, değer ve güç kullanıcı tarafından girdi olarak verilir), o zaman fonksiyon bir derleme zamanında çalıştırılır, bu yüzden bir kodu kopyalamaya gerek yoktur (bizim gibi). eski C++ standartlarında zorlanır).

Örneğin.

void useExample() {
    constexpr int compileTimeCalculated = calculatePower(3, 3); // computes at compile time,
                               // as both arguments are known at compilation time
                               // and used for a constant expression.
    int value;
    std::cin >> value;
    int runtimeCalculated = calculatePower(value, 3);  // runtime calculated,
                                    // because value is known only at runtime.
}

Derleme zamanında gücü hesaplamanın başka bir yolu, katlama ifadesini aşağıdaki gibi kullanabilir:

#include <iostream>
#include <utility>

template <class T, T V, T N, class I = std::make_integer_sequence<T, N>>
struct power;

template <class T, T V, T N, T... Is>
struct power<T, V, N, std::integer_sequence<T, Is...>> {
   static constexpr T value = (static_cast<T>(1) * ... * (V * static_cast<bool>(Is + 1)));
};

int main() {
   std::cout << power<int, 4, 2>::value << std::endl;
}

Herhangi bir T tipi verildiğinde tiplerin manuel ayrımı #

[std::enable_if kullanarak SFINAE uygularken ](https://www.wikiod.com/tr/docs/c%2b%2b/1169/sfinae-substitution-failure-is-not-an-error/3777/enable-if), genellikle yardımcıya erişmek yararlıdır Belirli bir “T” türünün bir dizi ölçütle eşleşip eşleşmediğini belirleyen şablonlar.

Bu konuda bize yardımcı olmak için, standart zaten ’true’ ve ‘false’ için iki tür analog sağlar, bunlar ‘std::true_type’ ve ‘std::false_type’.

Aşağıdaki örnek, bir “T” türünün bir işaretçi olup olmadığının nasıl algılanacağını gösterir, “is_pointer” şablonu, standart “std::is_pointer” yardımcısının davranışını taklit eder:

template <typename T>
struct is_pointer_: std::false_type {};

template <typename T>
struct is_pointer_<T*>: std::true_type {};

template <typename T>
struct is_pointer: is_pointer_<typename std::remove_cv<T>::type> { }

Yukarıdaki kodda üç adım vardır (bazen sadece ikisine ihtiyacınız vardır):

  1. ‘is_pointer_’ öğesinin ilk bildirimi varsayılan durumdur ve ‘std::false_type’ öğesinden miras alınır. varsayılan durum, ““false” koşuluna benzediğinden her zaman “std::false_type” öğesinden türetilmelidir.

  2. İkinci bildirim, ‘T’nin gerçekte ne olduğuyla ilgilenmeden ‘T*’ işaretçisi için ‘is_pointer_’ şablonunu özelleştirir. Bu sürüm ‘std::true_type’ öğesinden devralır.

  3. Üçüncü bildirim (gerçek olan) sadece gereksiz bilgileri “T"den kaldırır (bu durumda “const” ve “volatile” niteleyicilerini kaldırırız) ve daha sonra önceki iki bildirimden birine geri döner.

“is_pointer” bir sınıf olduğundan, değerine erişmek için şunlardan birini yapmanız gerekir:

  • ::value kullanın, ör. “is_pointer::değer” – “değer”, “std::true_type” veya “std::false_type"dan devralınan “bool” türündeki statik bir sınıf üyesidir;
  • Bu türden bir nesne oluşturun, örn. is_pointer<int>{} – Bu işe yarar çünkü “std::is_pointer” varsayılan yapıcısını “std::true_type” veya “std::false_type"dan (“constexpr” yapıcılarına sahip) ve hem “std::true_type” hem de “std::false_type"dan devralır. “constexpr”, “bool"a dönüştürme operatörlerine sahip.

Değere doğrudan erişmenizi sağlayan “yardımcı yardımcı şablonlar” sağlamak iyi bir alışkanlıktır:

template <typename T>
constexpr bool is_pointer_v = is_pointer<T>::value;

C++17 ve üzeri sürümlerde, çoğu yardımcı şablon zaten bir “_v” sürümü sağlar, örneğin:

template< class T > constexpr bool is_pointer_v = is_pointer<T>::value;
template< class T > constexpr bool is_reference_v = is_reference<T>::value;

Değişken argüman sayısı ile Genel Min/Maks #

Şablon meta-programlama ile çeşitli sayısal türleri ve rastgele argüman sayısını kabul eden genel bir işlev (örneğin min) yazmak mümkündür. Bu işlev, iki bağımsız değişken için bir “min” ve daha fazlası için özyinelemeli olarak bildirir.

template <typename T1, typename T2>
auto min(const T1 &a, const T2 &b) 
-> typename std::common_type<const T1&, const T2&>::type
{
    return a < b ? a : b;
}

template <typename T1, typename T2, typename ... Args>
auto min(const T1 &a, const T2 &b, const Args& ... args)
-> typename std::common_type<const T1&, const T2&, const Args& ...>::type
{
    return min(min(a, b), args...);
}

auto minimum = min(4, 5.8f, 3, 1.8, 3, 1.1, 9);
