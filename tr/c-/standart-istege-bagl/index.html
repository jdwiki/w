Bir değerin yokluğunu temsil etmek için seçenekleri kullanma #

C++17’den önce, “nullptr” değerine sahip işaretçilere sahip olmak, genellikle bir değerin yokluğunu temsil ediyordu. Bu, dinamik olarak ayrılmış ve zaten işaretçiler tarafından yönetilen büyük nesneler için iyi bir çözümdür. Ancak bu çözüm, nadiren dinamik olarak tahsis edilen veya işaretçiler tarafından yönetilen ‘int’ gibi küçük veya ilkel türler için iyi çalışmaz. std::opsiyonel bu yaygın soruna uygun bir çözüm sağlar.

Bu örnekte ‘struct Person’ tanımlanmıştır. Bir kişinin evcil hayvanı olması mümkündür, ancak gerekli değildir. Bu nedenle, ‘Person’ öğesinin ‘pet’ üyesi bir ‘std::opsiyonel’ sarmalayıcı ile bildirilir.

#include <iostream>
#include <optional>
#include <string>

struct Animal {
    std::string name;
};

struct Person {
    std::string name;
    std::optional<Animal> pet;
};

int main() {
    Person person;
    person.name = "John";

    if (person.pet) {
        std::cout << person.name << "'s pet's name is " <<
            person.pet->name << std::endl;
    }
    else {
        std::cout << person.name << " is alone." << std::endl;
    }
}

dönüş değeri olarak isteğe bağlı #

std::optional<float> divide(float a, float b) {
  if (b!=0.f) return a/b;
  return {};
}

Burada ya ‘a/b’ kesirini döndürürüz, ancak tanımlanmamışsa (sonsuz olur) bunun yerine boş isteğe bağlı olarak döndürürüz.

Daha karmaşık bir durum:

template<class Range, class Pred>
auto find_if( Range&& r, Pred&& p ) {
  using std::begin; using std::end;
  auto b = begin(r), e = end(r);
  auto r = std::find_if(b, e , p );
  using iterator = decltype(r);
  if (r==e)
    return std::optional<iterator>();
  return std::optional<iterator>(r);
}
template<class Range, class T>
auto find( Range&& r, T const& t ) {
  return find_if( std::forward<Range>(r), [&t](auto&& x){return x==t;} );
}

“find( some_range, 7 )”, “7” sayısına eşit bir şey için kapsayıcıyı veya “some_range” aralığını arar. find_if bunu bir yüklemle yapar.

Bulunamadıysa boş bir isteğe bağlı veya varsa öğeye yineleyici içeren bir isteğe bağlı döndürür.

Bu, şunları yapmanızı sağlar:

if (find( vec, 7 )) {
  // code
}

ya da

if (auto oit = find( vec, 7 )) {
  vec.erase(*oit);
}

başlangıç/bitiş yineleyicileri ve testleri ile uğraşmak zorunda kalmadan.

Giriiş #

İsteğe bağlı seçenekler (Belki türleri olarak da bilinir), içeriği mevcut olan veya olmayan bir türü temsil etmek için kullanılır. C++17’de ‘std::opsiyonel’ sınıfı olarak uygulanırlar. Örneğin, ‘std::opsiyonel’ türünde bir nesne, ‘int’ türünde bir değer içerebilir veya hiçbir değer içermeyebilir.

Opsiyonlar yaygın olarak ya var olmayan bir değeri temsil etmek için ya da anlamlı bir sonuç döndürmeyi başaramayan bir işlevden dönüş türü olarak kullanılır.

İsteğe bağlı diğer yaklaşımlar #

Sorunu çözmek için “std::opsiyonel"in çözdüğü başka birçok yaklaşım vardır, ancak bunların hiçbiri tam değildir: işaretçi kullanmak, sentinel kullanmak veya “pair<bool, T>” kullanmak.

İsteğe bağlı ve İşaretçi #

Bazı durumlarda, mevcut bir nesneye bir işaretçi veya başarısızlığı belirtmek için “nullptr” sağlayabiliriz. Ancak bu, nesnelerin zaten var olduğu durumlarla sınırlıdır - bir değer türü olarak “isteğe bağlı”, bellek ayırmaya başvurmadan yeni nesneleri döndürmek için de kullanılabilir.

İsteğe bağlı ve Sentinel #

Yaygın bir deyim, değerin anlamsız olduğunu belirtmek için özel bir değer kullanmaktır. Bu, integral türleri için 0 veya -1 veya işaretçiler için “nullptr” olabilir. Ancak bu, geçerli değerlerin alanını azaltır (geçerli bir 0 ile anlamsız 0 arasında ayrım yapamazsınız) ve birçok türün nöbetçi değer için doğal bir seçimi yoktur.

İsteğe bağlı ve std::pair<bool, T> #

Diğer bir yaygın deyim, öğelerden birinin değerin anlamlı olup olmadığını belirten bir “bool” olduğu bir çift sağlamaktır.

Bu, bazı türler için mümkün olmayan ve diğerleri için mümkün olan ancak istenmeyen bir hata durumunda değer türünün varsayılan olarak yapılandırılabilir olmasına dayanır. Bir “isteğe bağlı”, hata durumunda herhangi bir şey oluşturmaya gerek duymaz.

değer_veya #

Bir işlevin başarısızlığını temsil etmek için seçenekleri kullanma #

C++17’den önce, bir işlev tipik olarak başarısızlığı birkaç yoldan biriyle temsil ediyordu:

  • Bir boş gösterici döndürüldü.
    • e.g. Calling a function Delegate *App::get_delegate() on an App instance that did not have a delegate would return nullptr.
    • This is a good solution for objects that have been dynamically allocated or are large and managed by pointers, but isn’t a good solution for small objects that are typically stack-allocated and passed by copying.
  • Başarısızlığı belirtmek için dönüş türünün belirli bir değeri ayrılmıştır.
    • e.g. Calling a function unsigned shortest_path_distance(Vertex a, Vertex b) on two vertices that are not connected may return zero to indicate this fact.
  • Değer, döndürülen değerin anlamlı olduğunu belirtmek için bir “bool” ile eşleştirildi.
    • e.g. Calling a function std::pair<int, bool> parse(const std::string &str) with a string argument that is not an integer would return a pair with an undefined int and a bool set to false.

Bu örnekte John’a Fluffy ve Furball adlı iki evcil hayvan verilmiştir. Daha sonra John’un evcil hayvan Bıyıklarını almak için Person::pet_with_name() işlevi çağrılır. John’un Whiskers adında bir evcil hayvanı olmadığından, işlev başarısız olur ve bunun yerine “std::nullopt” döndürülür.

#include <iostream>
#include <optional>
#include <string>
#include <vector>

struct Animal {
    std::string name;
};

struct Person {
    std::string name;
    std::vector<Animal> pets;
    
    std::optional<Animal> pet_with_name(const std::string &name) {
        for (const Animal &pet : pets) {
            if (pet.name == name) {
                return pet;
            }
        }
        return std::nullopt;
    }
};

int main() {
    Person john;
    john.name = "John";
    
    Animal fluffy;
    fluffy.name = "Fluffy";
    john.pets.push_back(fluffy);
    
    Animal furball;
    furball.name = "Furball";
    john.pets.push_back(furball);
    
    std::optional<Animal> whiskers = john.pet_with_name("Whiskers");
    if (whiskers) {
        std::cout << "John has a pet named Whiskers." << std::endl;
    }
    else {
        std::cout << "Whiskers must not belong to John." << std::endl;
    }
}
