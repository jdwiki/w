Bir ifade, ne olduğuna bağlı olarak, “Dynamic_cast”, “static_cast”, “reinterpret_cast” veya “const_cast” kullanılarak “T” türüne açıkça dönüştürülebilir veya cast olabilir. döküm tipi amaçlanmaktadır.

C++ ayrıca işlev stili döküm notasyonunu, T(expr) ve C-stili döküm notasyonunu, (T)expri destekler.

Sözdizimi #

  • basit tip belirteci ( )
  • basit tip belirteci ( ifade listesi )
  • basit tip belirteci çekmeli-init-listesi
  • typename-belirleyici ( )
  • typename-belirleyici ( ifade-listesi )
  • typename-specifier braced-init-list
  • dynamic_cast < type-id > ( ifade )
  • static_cast < type-id > ( ifade )
  • reinterpret_cast < type-id > ( ifade )
  • const_cast < type-id > ( ifade )
  • ( type-id ) cast-ifadesi

Altı döküm gösteriminin hepsinin ortak bir noktası vardır:

  • ‘Dynamic_cast<Derived&>(base)‘de olduğu gibi bir değer referans tipine döküm yapmak, bir değer verir. Bu nedenle, aynı nesneyle bir şey yapmak istediğinizde, ancak onu farklı bir tür olarak ele aldığınızda, bir değer referans türüne atlarsınız.
  • static_cast<string&&>(s)‘de olduğu gibi bir değer referans tipine döküm yapmak, bir değer verir.
  • (int)x’te olduğu gibi referans olmayan bir türe döküm yapmak, dönüştürülmekte olan değerin kopyası olarak düşünülebilecek, ancak orijinalden farklı bir türde bir değer verir.

“reinterpret_cast” anahtar kelimesi, iki farklı türde “güvenli olmayan” dönüşüm gerçekleştirmekten sorumludur:

  • Belleğe erişmek için kullanılabilen “type punning” dönüşümleri sanki farklı bir türdenmiş gibi bir türden.
  • Dönüşümler tamsayı türleri ile işaretçi türleri arasında, her ikisinde de yön.

“static_cast” anahtar kelimesi çeşitli farklı dönüşümler gerçekleştirebilir:

  • Temelden türetilene dönüşümler

  • Hem örtük dönüştürmeler hem de açık bir kurucu veya dönüştürme işlevi çağıran dönüştürmeler dahil olmak üzere, doğrudan başlatma ile yapılabilecek herhangi bir dönüştürme. buraya ve buraya bakın c%2b%2b/3090/explicit-type-conversions/18731/conversion-by-explicit-constructor-or-explicit-conversion-function) daha fazla ayrıntı için.

  • İfadenin değerini atan ‘void’ için.

    // on some compilers, suppresses warning about x being unused
static_cast<void>(x);

  • Aritmetik ve numaralandırma türleri arasında ve farklı numaralandırma türleri arasında. Bkz. enum dönüşümleri

  • İşaretçiden türetilmiş sınıfın üyesine, işaretçiden temel sınıfın üyesine. İşaret edilen türler eşleşmelidir. Bkz. [üyelere yönelik işaretçiler için temel dönüşüme türetilmiş](https://www.wikiod.com/tr/docs/c%2b%2b/3090/explicit-type-conversions/18752/derived-to-base-conversion-for-pointers- üyelere)

  • void* ila T*.

  • std::move’da olduğu gibi bir değerden bir xdeğerine. Bkz. semantiği taşı.

Tutarlılığı ortadan kaldırmak #

Bir const nesnesine yönelik bir işaretçi, “const_cast” anahtar kelimesi kullanılarak const olmayan nesneye bir işaretçiye dönüştürülebilir. Burada, const-doğru olmayan bir işlevi çağırmak için “const_cast” kullanıyoruz. İşaretçi aracılığıyla hiçbir zaman yazmasa da yalnızca const olmayan bir “char*” argümanını kabul eder:

void bad_strlen(char*);
const char* s = "hello, world!";
bad_strlen(s);                    // compile error
bad_strlen(const_cast<char*>(s)); // OK, but it's better to make bad_strlen accept const char*

“const_cast” başvuru tipine, const nitelikli bir değeri const olmayan bir değere dönüştürmek için kullanılabilir.

‘const_cast’ tehlikelidir çünkü C++ tipi sistemin bir const nesnesini değiştirmeye çalışmanızı engellemesini imkansız kılar. Bunu yapmak tanımsız davranışla sonuçlanır.

const int x = 123;
int& mutable_x = const_cast<int&>(x);
mutable_x = 456; // may compile, but produces *undefined behavior*

C tarzı döküm #

C-Style döküm, ‘En iyi çaba’ dökümü olarak kabul edilebilir ve C’de kullanılabilen tek döküm olduğu için adlandırılır. Bu döküm için sözdizimi (NewType)variabledır.

Bu döküm kullanıldığında, aşağıdaki c++ dökümlerinden birini kullanır (sırasıyla):

  • const_cast<NewType>(değişken)
  • static_cast<NewType>(değişken)
  • const_cast<NewType>(static_cast<const NewType>(değişken))
  • reinterpret_cast<const NewType>(değişken)
  • const_cast<NewType>(reinterpret_cast<const NewType>(değişken))

İşlevsel döküm, sözdiziminin sonucu olarak birkaç kısıtlama olsa da çok benzer: NewType(expression). Sonuç olarak, yalnızca boşluk içermeyen türlere yayın yapılabilir.

Yeni c++ dökümünü kullanmak daha iyidir, çünkü daha okunabilirdir ve bir C++ kaynak kodunun herhangi bir yerinde kolayca tespit edilebilir ve hatalar çalışma zamanı yerine derleme zamanında algılanır.

Bu atama, istenmeyen “yeniden yorumlama_atım” ile sonuçlanabileceğinden, genellikle tehlikeli olarak kabul edilir.

Türetilmiş dönüşüme temel #

Temel sınıfa yönelik bir işaretçi, ‘static_cast’ kullanılarak türetilmiş sınıfa bir işaretçiye dönüştürülebilir. ‘static_cast’ herhangi bir çalışma zamanı denetimi yapmaz ve işaretçi gerçekte istenen türü göstermediğinde tanımsız davranışa yol açabilir.

struct Base {};
struct Derived : Base {};
Derived d;
Base* p1 = &d;
Derived* p2 = p1;                        // error; cast required
Derived* p3 = static_cast<Derived*>(p1); // OK; p2 now points to Derived object
Base b;
Base* p4 = &b;
Derived* p5 = static_cast<Derived*>(p4); // undefined behaviour since p4 does not
                                         // point to a Derived object

Benzer şekilde, temel sınıfa yapılan bir referans, ‘static_cast’ kullanılarak türetilmiş sınıfa yapılan bir referansa dönüştürülebilir.

struct Base {};
struct Derived : Base {};
Derived d;
Base& r1 = d;
Derived& r2 = r1;                        // error; cast required
Derived& r3 = static_cast<Derived&>(r1); // OK; r3 now refers to Derived object

Kaynak türü polimorfikse, türetilmiş dönüştürmeye bir temel gerçekleştirmek için “dinamik_gönderme” kullanılabilir. Bir çalışma zamanı denetimi gerçekleştirir ve tanımsız davranış üretmek yerine hata kurtarılabilir. İşaretçi durumunda, başarısızlık durumunda bir boş işaretçi döndürülür. Başvuru durumunda, ‘std::bad_cast’ türünün (veya ‘std::bad_cast’ türünden türetilen bir sınıfın) başarısız olması durumunda bir istisna atılır.

struct Base { virtual ~Base(); }; // Base is polymorphic
struct Derived : Base {};
Base* b1 = new Derived;
Derived* d1 = dynamic_cast<Derived*>(b1); // OK; d1 points to Derived object
Base* b2 = new Base;
Derived* d2 = dynamic_cast<Derived*>(b2); // d2 is a null pointer

İşaretçi ve tamsayı arasında dönüştürme #

Bir nesne işaretçisi (“void*” dahil) veya işlev işaretçisi, “reinterpret_cast” kullanılarak bir tamsayı türüne dönüştürülebilir. Bu, yalnızca hedef türün yeterince uzun olması durumunda derlenir. Sonuç, uygulama tarafından tanımlanır ve tipik olarak, işaretçinin işaret ettiği bellekteki baytın sayısal adresini verir.

Tipik olarak, “uzun” veya “işaretsiz uzun” herhangi bir işaretçi değerini tutacak kadar uzundur, ancak bu standart tarafından garanti edilmez.

“std::intptr_t” ve “std::uintptr_t” türleri varsa, bunların bir “void*” (ve dolayısıyla nesne türüne yönelik herhangi bir işaretçi) tutacak kadar uzun olmaları garanti edilir. Ancak, bir işlev işaretçisini tutacak kadar uzun olmaları garanti edilmez.

son sürüm

Benzer şekilde, bir tamsayı türünü bir işaretçi türüne dönüştürmek için “reinterpret_cast” kullanılabilir. Sonuç yine uygulama tanımlıdır, ancak bir tamsayı türü boyunca bir gidiş-dönüş ile bir işaretçi değerinin değişmemesi garanti edilir. Standart, sıfır değerinin bir boş göstericiye dönüştürülmesini garanti etmez.

void register_callback(void (*fp)(void*), void* arg); // probably a C API
void my_callback(void* x) {
    std::cout << "the value is: " << reinterpret_cast<long>(x); // will probably compile
}
long x;
std::cin >> x;
register_callback(my_callback,
                  reinterpret_cast<void*>(x)); // hopefully this doesn't lose information...

Tip punta dönüştürme #

Bir nesne türüne yönelik bir işaretçi (ilgili başvuru), “yeniden yorumlama_gönderme” kullanılarak herhangi bir başka nesne türüne yönelik bir işaretçiye (ilgili başvuru) dönüştürülebilir. Bu, herhangi bir kurucu veya dönüştürme işlevi çağırmaz.

int x = 42;
char* p = static_cast<char*>(&x);      // error: static_cast cannot perform this conversion
char* p = reinterpret_cast<char*>(&x); // OK
*p = 'z';                              // maybe this modifies x (see below)

“reinterpret_cast"in sonucu, adresin hedef türü için uygun şekilde hizalanması koşuluyla, işlenenle aynı adresi temsil eder. Aksi takdirde sonuç belirsizdir.

int x = 42;
char& r = reinterpret_cast<char&>(x);
const void* px = &x;
const void* pr = &r;
assert(px == pr); // should never fire
son sürüm

“reinterpret_cast"in sonucu belirtilmemiştir, ancak bir işaretçinin (ilgili referans), hedef tipinin hizalama gereksinimi kaynağınkinden daha katı olmadığı sürece, kaynak türünden hedef türüne ve geriye gidiş dönüşte hayatta kalması dışında. tip.

int x = 123;
unsigned int& r1 = reinterpret_cast<unsigned int&>(x);
int& r2 = reinterpret_cast<int&>(r1);
r2 = 456; // sets x to 456

Çoğu uygulamada, “reinterpret_cast” adresi değiştirmez, ancak bu gereksinim C++11’e kadar standartlaştırılmamıştır.

son sürüm

‘reinterpret_cast’ ayrıca bir işaretçiden veri üyesine türünden diğerine veya bir işaretçiden üyeye işlev türünden diğerine dönüştürmek için kullanılabilir.

‘reinterpret_cast’ kullanımı tehlikeli olarak kabul edilir, çünkü ‘reinterpret_cast’ kullanılarak elde edilen bir işaretçi veya referans aracılığıyla okuma veya yazma, kaynak ve hedef türleri ilgisiz olduğunda tanımsız davranışı tetikleyebilir.

Açık yapıcı veya açık dönüştürme işleviyle dönüştürme #

Bir explicit oluşturucu veya dönüştürme işlevinin çağrılmasını içeren bir dönüştürme, dolaylı olarak yapılamaz. Dönüşümün ‘static_cast’ kullanılarak açıkça yapılmasını talep edebiliriz. Anlamı, sonucun geçici olması dışında, doğrudan başlatma ile aynıdır.

class C {
    std::unique_ptr<int> p;
  public:
    explicit C(int* p) : p(p) {}
};
void f(C c);
void g(int* p) {
    f(p);                 // error: C::C(int*) is explicit
    f(static_cast<C>(p)); // ok
    f(C(p));              // equivalent to previous line
    C c(p); f(c);         // error: C is not copyable
}

Örtülü dönüştürme #

‘static_cast’ herhangi bir örtük dönüştürmeyi gerçekleştirebilir. Bu “static_cast” kullanımı, aşağıdaki örneklerde olduğu gibi zaman zaman yararlı olabilir:

  • Bir üç noktaya bağımsız değişkenler iletirken, “beklenen” bağımsız değişken türü statik olarak bilinmez, bu nedenle hiçbir örtük dönüştürme gerçekleşmez.

    const double x = 3.14;
printf("%d\n", static_cast<int>(x)); // prints 3
// printf("%d\n", x); // undefined behaviour; printf is expecting an int here
// alternative:
// const int y = x; printf("%d\n", y);

Açık tür dönüştürme olmadan, üç noktaya bir “çift” nesne geçirilir ve tanımsız davranış ortaya çıkar.

  • Türetilmiş bir sınıf atama operatörü, aşağıdaki gibi bir temel sınıf atama operatörünü çağırabilir:

    struct Base { /* ... */ };
struct Derived : Base {
    Derived& operator=(const Derived& other) {
        static_cast<Base&>(*this) = other;
        // alternative:
        // Base& this_base_ref = *this; this_base_ref = other;
    }
};

Enum dönüşümleri #

‘static_cast’, bir tamsayı veya kayan nokta türünden bir numaralandırma türüne (kapsamlı veya kapsamsız) ve tersine dönüştürebilir. Ayrıca numaralandırma türleri arasında dönüşüm yapabilir.

  • Kapsamsız bir numaralandırma türünden aritmetik bir türe dönüştürme, örtük bir dönüştürmedir; ‘static_cast’ kullanmak mümkündür, ancak gerekli değildir.
  • Kapsamlı bir numaralandırma türü aritmetik bir türe dönüştürüldüğünde:
  • Numaralandırmanın değeri hedef türünde tam olarak gösterilebiliyorsa, sonuç bu değerdir.
  • Aksi takdirde, hedef tipi bir tamsayı tipi ise sonuç belirtilmez.
  • Aksi takdirde, hedef türü bir kayan nokta türüyse, sonuç, temel türe ve ardından kayan nokta türüne dönüştürmeyle aynıdır.

Örnek:

  enum class Format {
      TEXT = 0,
      PDF = 1000,
      OTHER = 2000,
  };
  Format f = Format::PDF;
  int a = f;                         // error
  int b = static_cast<int>(f);       // ok; b is 1000
  char c = static_cast<char>(f);     // unspecified, if 1000 doesn't fit into char
  double d = static_cast<double>(f); // d is 1000.0... probably
son sürüm
  • Bir tamsayı veya numaralandırma türü bir numaralandırma türüne dönüştürüldüğünde:
  • Orijinal değer, hedef numaralandırma aralığı içindeyse, sonuç bu değerdir. Bu değerin tüm numaralandırıcılara eşit olmayabileceğini unutmayın.
  • Aksi takdirde, sonuç belirtilmemiş (<= C++14) veya tanımsız (>= C++17) olur.

Örnek:

  enum Scale {
      SINGLE = 1,
      DOUBLE = 2,
      QUAD = 4
  };
  Scale s1 = 1;                     // error
  Scale s2 = static_cast<Scale>(2); // s2 is DOUBLE
  Scale s3 = static_cast<Scale>(3); // s3 has value 3, and is not equal to any enumerator
  Scale s9 = static_cast<Scale>(9); // unspecified value in C++14; UB in C++17

  • Kayan nokta türü bir numaralandırma türüne dönüştürüldüğünde, sonuç, numaralandırmanın temel alınan türüne ve ardından numaralandırma türüne dönüştürmeyle aynıdır.

    enum Direction {
    UP = 0,
    LEFT = 1,
    DOWN = 2,
    RIGHT = 3,
};
Direction d = static_cast<Direction>(3.14); // d is RIGHT
son sürüm

Üyelere yönelik işaretçiler için temel dönüşüme türetilmiştir #

Türetilmiş sınıfın üyesine yönelik bir işaretçi, ‘static_cast’ kullanılarak temel sınıfın üyesine yönelik bir işaretçiye dönüştürülebilir. İşaret edilen türler eşleşmelidir.

İşlenen üye değerine bir boş gösterici ise, sonuç aynı zamanda üye değerine bir boş göstericidir.

Aksi takdirde, dönüştürme yalnızca işlenen tarafından işaret edilen üye hedef sınıfta gerçekten varsa veya hedef sınıf, işlenen tarafından işaret edilen üyeyi içeren sınıfın bir temel veya türetilmiş sınıfıysa geçerlidir. “static_cast” geçerliliği kontrol etmez. Dönüştürme geçerli değilse, davranış tanımsızdır.

struct A {};
struct B { int x; };
struct C : A, B { int y; double z; };
int B::*p1 = &B::x;
int C::*p2 = p1;                              // ok; implicit conversion
int B::*p3 = p2;                              // error
int B::*p4 = static_cast<int B::*>(p2);       // ok; p4 is equal to p1
int A::*p5 = static_cast<int A::*>(p2);       // undefined; p2 points to x, which is a member
                                              // of the unrelated class B
double C::*p6 = &C::z;
double A::*p7 = static_cast<double A::*>(p6); // ok, even though A doesn't contain z
int A::*p8 = static_cast<int A::*>(p6);       // error: types don't match

void*‘den T*‘ye #

C++‘da ‘void*’ örtük olarak ‘T*’ biçimine dönüştürülemez, burada ‘T’ bir nesne türüdür. Bunun yerine, dönüştürmeyi açıkça gerçekleştirmek için ‘static_cast’ kullanılmalıdır. İşlenen gerçekten bir “T” nesnesine işaret ediyorsa, sonuç o nesneye işaret eder. Aksi takdirde sonuç belirsizdir.

İşlenen bir “T” nesnesine işaret etmese bile, işlenen adresi “T” tipi için uygun şekilde hizalanmış bir baytı işaret ettiği sürece, dönüştürmenin sonucu aynı baytı gösterir.

son sürüm 
// allocating an array of 100 ints, the hard way
int* a = malloc(100*sizeof(*a));                    // error; malloc returns void*
int* a = static_cast<int*>(malloc(100*sizeof(*a))); // ok
// int* a = new int[100];                           // no cast needed
// std::vector<int> a(100);                         // better

const char c = '!';
const void* p1 = &c;
const char* p2 = p1;                           // error
const char* p3 = static_cast<const char*>(p1); // ok; p3 points to c
const int* p4 = static_cast<const int*>(p1);   // unspecified in C++03;
                                               // possibly unspecified in C++11 if
                                               // alignof(int) > alignof(char)
char* p5 = static_cast<char*>(p1);             // error: casting away constness
