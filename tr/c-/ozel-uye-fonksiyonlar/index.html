Varsayılan Yapıcı #

varsayılan yapıcı, çağrıldığında hiçbir parametre gerektirmeyen bir yapıcı türüdür. Adını oluşturduğu türden alır ve onun bir üye işlevidir (tüm kurucular gibi).

class C{
    int i;
public:
    // the default constructor definition
    C()
    : i(0){ // member initializer list -- initialize i to 0
        // constructor function body -- can do more complex things here
    }
};

C c1; // calls default constructor of C to create object c1
C c2 = C(); // calls default constructor explicitly
C c3(); // ERROR: this intuitive version is not possible due to "most vexing parse"
C c4{}; // but in C++11 {} CAN be used in a similar way

C c5[2]; // calls default constructor for both array elements
C* c6 = new C[2]; // calls default constructor for both array elements

“Parametre yok” gereksinimini karşılamanın başka bir yolu, geliştiricinin tüm parametreler için varsayılan değerler sağlamasıdır:

class D{
    int i;
    int j;
public:
    // also a default constructor (can be called with no parameters)
    D( int i = 0, int j = 42 ) 
    : i(i), j(j){
    }
};


D d; // calls constructor of D with the provided default values for the parameters

Bazı durumlarda (yani, geliştirici hiçbir kurucu sağlamaz ve başka hiçbir diskalifiye edici koşul yoktur), derleyici örtük olarak boş bir varsayılan kurucu sağlar:

class C{
    std::string s; // note: members need to be default constructible themselves
};

C c1; // will succeed -- C has an implicitly defined default constructor

Başka türde bir kurucuya sahip olmak, daha önce bahsedilen diskalifiye edici koşullardan biridir:

class C{
    int i;
public:
    C( int i ) : i(i){}
};

C c1; // Compile ERROR: C has no (implicitly defined) default constructor

Örtük varsayılan kurucu oluşturmayı önlemek için yaygın bir teknik, onu “özel” (tanımsız) olarak bildirmektir. Amaç, biri yapıcıyı kullanmaya çalıştığında derleme hatasına neden olmaktır (bu, derleyiciye bağlı olarak bir Özel erişim hatasına veya bir bağlayıcı hatasına neden olur).

Varsayılan bir kurucunun (işlevsel olarak örtük olana benzer) tanımlandığından emin olmak için, bir geliştirici açıkça boş bir tane yazabilir.

son sürüm

C++ 11’de bir geliştirici, derleyicinin varsayılan bir kurucu sağlamasını önlemek için “delete” anahtar sözcüğünü de kullanabilir.

class C{
    int i;
public:
    // default constructor is explicitly deleted
    C() = delete;
};

C c1; // Compile ERROR: C has its default constructor deleted

Ayrıca, bir geliştirici, derleyicinin varsayılan bir kurucu sağlamasını isteme konusunda da açık olabilir.

class C{
    int i;
public:
    // does have automatically generated default constructor (same as implicit one)
    C() = default;

    C( int i ) : i(i){}
};

C c1; // default constructed
C c2( 1 ); // constructed with the int taking constructor
son sürüm

Bir türün varsayılan kurucuya sahip olup olmadığını (veya ilkel bir tür olup olmadığını) “<type_traits>“den “std::is_default_constructible” kullanarak belirleyebilirsiniz:

class C1{ };
class C2{ public: C2(){} };
class C3{ public: C3(int){} };

using std::cout; using std::boolalpha; using std::endl;
using std::is_default_constructible;
cout << boolalpha << is_default_constructible<int>() << endl; // prints true
cout << boolalpha << is_default_constructible<C1>() << endl; // prints true
cout << boolalpha << is_default_constructible<C2>() << endl; // prints true
cout << boolalpha << is_default_constructible<C3>() << endl; // prints false
son sürüm

C++ 11’de, std::is_default_constructible’ın işlevsiz sürümünü kullanmak hala mümkündür:

cout << boolalpha << is_default_constructible<C1>::value << endl; // prints true
son sürüm

Kopyala ve değiştir #

Kaynakları yöneten bir sınıf yazıyorsanız, tüm özel üye işlevlerini uygulamanız gerekir (bkz. Üç/Beş/Sıfır Kuralı. üç-beş-ve-sıfır-kuralı#t=201607231819366817285)). Kopya oluşturucuyu ve atama operatörünü yazmaya en doğrudan yaklaşım şudur:

person(const person &other)
    : name(new char[std::strlen(other.name) + 1])
    , age(other.age)
{
    std::strcpy(name, other.name);
}

person& operator=(person const& rhs) {
    if (this != &other) {
        delete [] name;
        name = new char[std::strlen(other.name) + 1];
        std::strcpy(name, other.name);
        age = other.age;
    }

    return *this;
}

Ama bu yaklaşımın bazı sorunları var. Güçlü istisna garantisinde başarısız olur - “yeni[]” atarsa, “bu"nun sahip olduğu kaynakları zaten temizledik ve kurtaramayız. Kopya atamasında birçok kopya oluşturma mantığını çoğaltıyoruz. Ve genellikle kopyalama işlemine ek yük ekleyen, ancak yine de kritik olan kendi kendine atama kontrolünü hatırlamamız gerekir.

Güçlü istisna garantisini sağlamak ve kod tekrarını önlemek için (sonraki hareket atama operatörüyle iki katı), kopyala ve değiştir deyimini kullanabiliriz:

class person {
    char* name;
    int age;
public:
    /* all the other functions ... */

    friend void swap(person& lhs, person& rhs) {
        using std::swap; // enable ADL

        swap(lhs.name, rhs.name);
        swap(lhs.age, rhs.age);
    }

    person& operator=(person rhs) {
        swap(*this, rhs);
        return *this;
    }
};

Bu neden işe yarıyor? sahip olduğumuzda ne olacağını düşünün.

person p1 = ...;
person p2 = ...;
p1 = p2;

İlk olarak, ‘p2’den ‘rhs’yi kopyalayıp yapılandırıyoruz (ki burada kopyalamak zorunda değildik). Bu işlem atarsa, operator= içinde hiçbir şey yapmıyoruz ve p1 dokunulmadan kalıyor. Ardından, üyeleri *this ve rhs arasında değiştiririz ve ardından rhs kapsam dışına çıkar. “Operatör=” olduğunda, “bu"nun orijinal kaynaklarını örtük olarak temizler (kopyalamamız gerekmeyen yıkıcı aracılığıyla). Kendi kendine atama da işe yarar - kopyala ve değiştir ile daha az verimlidir (ilave bir tahsis ve yeniden tahsis içerir), ancak bu olası olmayan bir senaryoysa, bunu hesaba katmak için tipik kullanım durumunu yavaşlatmıyoruz.

Yukarıdaki formülasyon, hareket ataması için olduğu gibi çalışır.

p1 = std::move(p2);

Burada, ‘rhs’yi ‘p2’den hareket ettiriyoruz-yapılıyoruz ve geri kalan her şey aynı derecede geçerli. Bir sınıf taşınabilir ancak kopyalanabilir değilse, kopya atamasını silmeye gerek yoktur, çünkü bu atama operatörü silinen kopya oluşturucu nedeniyle kötü biçimlendirilmiş olacaktır.

Yıkıcı #

yıkıcı, kullanıcı tanımlı bir nesne yok edilmek üzereyken çağrılan, bağımsız değişkenleri olmayan bir işlevdir. Bir `~’ öneki ile yok ettiği türden sonra adlandırılır.

class C{
    int* is;
    string s;
public:
    C()
    : is( new int[10] ){
    }

    ~C(){  // destructor definition
        delete[] is;
    }
};

class C_child : public C{
    string s_ch;
public:
    C_child(){}
    ~C_child(){} // child destructor
};

void f(){
    C c1; // calls default constructor
    C c2[2]; // calls default constructor for both elements
    C* c3 = new C[2]; // calls default constructor for both array elements

    C_child c_ch;  // when destructed calls destructor of s_ch and of C base (and in turn s)

    delete[] c3; // calls destructors on c3[0] and c3[1]
} // automatic variables are destroyed here -- i.e. c1, c2 and c_ch

Çoğu durumda (yani, bir kullanıcı hiçbir yıkıcı sağlamaz ve başka hiçbir diskalifiye edici koşul yoktur), derleyici örtük olarak bir varsayılan yıkıcı sağlar:

class C{
    int i;
    string s;
};

void f(){
    C* c1 = new C;
    delete c1; // C has a destructor
}


class C{
    int m;
private:
    ~C(){} // not public destructor!
};

class C_container{
    C c;
};

void f(){
    C_container* c_cont = new C_container;
    delete c_cont; // Compile ERROR: C has no accessible destructor
}

C++ 11’de bir geliştirici, derleyicinin varsayılan bir yıkıcı sağlamasını engelleyerek bu davranışı geçersiz kılabilir.

class C{
    int m;
public:
    ~C() = delete; // does NOT have implicit destructor
};

void f{
    C c1; 
} // Compile ERROR: C has no destructor

Ayrıca, bir geliştirici, derleyicinin varsayılan bir yıkıcı sağlamasını isteme konusunda da açık olabilir.

class C{
    int m;
public:
    ~C() = default; // saying explicitly it does have implicit/empty destructor
};

void f(){
    C c1;
} // C has a destructor -- c1 properly destroyed

Bir türün yıkıcı olup olmadığını (veya ilkel bir tür olup olmadığını) “<type_traits>“den “std::is_destructible” kullanarak belirleyebilirsiniz:

class C1{ };
class C2{ public: ~C2() = delete };
class C3 : public C2{ };

using std::cout; using std::boolalpha; using std::endl;
using std::is_destructible;
cout << boolalpha << is_destructible<int>() << endl; // prints true
cout << boolalpha << is_destructible<C1>() << endl; // prints true
cout << boolalpha << is_destructible<C2>() << endl; // prints false
cout << boolalpha << is_destructible<C3>() << endl; // prints false

Örtülü Taşıma ve Kopyalama #

Bir yıkıcı bildirmenin, derleyicinin örtük hareket oluşturucuları ve hareket atama operatörleri oluşturmasını engellediğini unutmayın. Bir yıkıcı bildirirseniz, taşıma işlemleri için uygun tanımları da eklemeyi unutmayın.

Ayrıca, taşıma işlemlerinin bildirilmesi, kopyalama işlemlerinin oluşturulmasını engelleyecektir, bu nedenle bunlar da eklenmelidir (eğer bu sınıfın nesnelerinin kopyalama semantiğine sahip olması gerekiyorsa).

class Movable {
public:
    virtual ~Movable() noexcept = default;

    //    compiler won't generate these unless we tell it to
    //    because we declared a destructor
    Movable(Movable&&) noexcept = default;
    Movable& operator=(Movable&&) noexcept = default;

    //    declaring move operations will suppress generation
    //    of copy operations unless we explicitly re-enable them
    Movable(const Movable&) = default;
    Movable& operator=(const Movable&) = default;
};

Sanal ve Korumalı Yıkıcılar #

Miras alınmak üzere tasarlanmış bir sınıfa Temel sınıf denir. Böyle bir sınıfın özel üye işlevlerine özen gösterilmelidir.

Çalışma zamanında (temel sınıfa bir işaretçi aracılığıyla) polimorfik olarak kullanılmak üzere tasarlanmış bir sınıf, yıkıcıyı ‘sanal’ olarak bildirmelidir. Bu, nesne temel sınıfa bir işaretçi aracılığıyla yok edildiğinde bile, nesnenin türetilmiş parçalarının uygun şekilde yok edilmesini sağlar.

class Base {
public:
    virtual ~Base() = default;

private:
    //    data members etc.
};

class Derived : public Base { //  models Is-A relationship
public:
    //    some methods

private:
    //    more data members
};

//    virtual destructor in Base ensures that derived destructors
//    are also called when the object is destroyed
std::unique_ptr<Base> base = std::make_unique<Derived>();
base = nullptr;  //    safe, doesn't leak Derived's members

Sınıfın polimorfik olması gerekmiyorsa, ancak yine de arabiriminin devralınmasına izin vermesi gerekiyorsa, sanal olmayan bir “korumalı” yıkıcı kullanın.

class NonPolymorphicBase {
public:
    //    some methods

protected:
    ~NonPolymorphicBase() = default; //    note: non-virtual

private:
    //    etc.
};

Böyle bir sınıf, dilimleme nedeniyle sessiz sızıntılardan kaçınarak bir işaretçi aracılığıyla asla yok edilemez.

Bu teknik özellikle “özel” temel sınıflar olarak tasarlanmış sınıflar için geçerlidir. Böyle bir sınıf, özelleştirme noktaları olarak “sanal” yöntemler sağlarken bazı ortak uygulama ayrıntılarını kapsamak için kullanılabilir. Bu tür bir sınıf asla polimorfik olarak kullanılmamalıdır ve “korumalı” bir yıkıcı bu gereksinimi doğrudan kodda belgelemeye yardımcı olur.

Son olarak, bazı sınıflar temel sınıf olarak asla kullanılmamalarını gerektirebilir. Bu durumda, sınıf “final” olarak işaretlenebilir. Bu durumda normal bir sanal olmayan genel yıkıcı iyidir.

class FinalClass final {  //    marked final here
public:
    ~FinalClass() = default;

private:
    //    etc.
};
