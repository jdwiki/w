Depolama sınıfı belirteçleri, bildirimlerde kullanılabilen anahtar kelimelerdir. Bildirimin türünü etkilemezler, ancak tipik olarak varlığın depolanma şeklini değiştirirler.

Hepsi aynı dil sürümünde olmasa da altı depolama sınıfı belirteci vardır: ‘auto’ (C++11’e kadar), ‘register’ (C++17’ye kadar), ‘static’, ’thread_local’ (C++11’den beri), “harici” ve “değişebilir”.

Standarda göre,

Belirli bir decl-belirleyici-seq içinde en fazla bir depolama-sınıfı-belirleyicisi görünür, ancak ’thread_local’ ‘statik’ veya ’extern’ ile görünebilir.

Bir bildirim, depolama sınıfı belirteci içermeyebilir. Bu durumda, dil varsayılan bir davranışı belirtir. Örneğin, varsayılan olarak, blok kapsamında bildirilen bir değişken örtük olarak otomatik depolama süresine sahiptir.

Kayıt ol #

Bir değişkenin yoğun olarak kullanılacağını derleyiciye ima eden bir depolama sınıfı belirteci. “Kayıt” kelimesi, bir derleyicinin böyle bir değişkeni daha az saat döngüsünde erişilebilmesi için bir CPU kaydında saklamayı seçebileceği gerçeğiyle ilgilidir. C++ 11’den başlayarak kullanımdan kaldırıldı.

register int i = 0;
while (i < 100) {
    f(i);
    int g = i*i;
    i += h(i, g);
}

Hem yerel değişkenler hem de işlev parametreleri “kayıt” olarak bildirilebilir. C’den farklı olarak, C++ bir “kayıt” değişkeni ile yapılabileceklere herhangi bir kısıtlama getirmez. Örneğin, bir “kayıt” değişkeninin adresini almak geçerlidir, ancak bu, derleyicinin böyle bir değişkeni bir kayıtta fiilen saklamasını engelleyebilir.

son sürüm

“kayıt” anahtar kelimesi kullanılmamış ve rezerve edilmiştir. “Kayıt” anahtar sözcüğünü kullanan bir program hatalı biçimlendirilmiştir.

son sürüm

dış #

“extern” depolama sınıfı belirteci, bağlama bağlı olarak aşağıdaki üç yoldan biriyle bir bildirimi değiştirebilir:

  1. Bir değişkeni tanımlamadan bildirmek için kullanılabilir. Tipik olarak, bu, ayrı bir uygulama dosyasında tanımlanacak bir değişken için bir başlık dosyasında kullanılır.

    // global scope
int x;             // definition; x will be default-initialized
extern int y;      // declaration; y is defined elsewhere, most likely another TU
extern int z = 42; // definition; "extern" has no effect here (compiler may warn)

  2. “const” veya “constexpr” aksi takdirde dahili bağlantıya sahip olmasına neden olsa bile, ad alanı kapsamındaki bir değişkene harici bağlantı verir.

    // global scope
const int w = 42;            // internal linkage in C++; external linkage in C
static const int x = 42;     // internal linkage in both C++ and C
extern const int y = 42;     // external linkage in both C++ and C
namespace {
    extern const int z = 42; // however, this has internal linkage since
                             // it's in an unnamed namespace
}

  3. Daha önce linkage ile bildirilmişse, blok kapsamındaki bir değişkeni yeniden bildirir. Aksi takdirde, en yakın çevreleyen ad alanının üyesi olan bağlantıya sahip yeni bir değişken bildirir.

    // global scope
namespace {
    int x = 1;
    struct C {
        int x = 2;
        void f() {
            extern int x;           // redeclares namespace-scope x
            std::cout << x << '\n'; // therefore, this prints 1, not 2
        }
    };
}
void g() {
    extern int y; // y has external linkage; refers to global y defined elsewhere
}

Bir işlev ayrıca ’extern’ olarak da bildirilebilir, ancak bunun bir etkisi yoktur. Genellikle burada bildirilen bir fonksiyonun başka bir çeviri biriminde tanımlandığı okuyucuya bir ipucu olarak kullanılır. Örneğin:

 void f();        // typically a forward declaration; f defined later in this TU
 extern void g(); // typically not a forward declaration; g defined in another TU

Yukarıdaki kodda, “f”, “harici” olarak ve “g”, “harici olmayan” olarak değiştirilirse, bu, programın doğruluğunu veya anlamını hiçbir şekilde etkilemez, ancak büyük olasılıkla kodu okuyanın kafasını karıştırırdı.

değişken #

Bir sınıfın statik olmayan, referans olmayan bir veri üyesinin bildirimine uygulanabilen bir belirteç. Bir sınıfın değişken üyesi, nesne “const” olsa bile “const” değildir.

class C {
    int x;
    mutable int times_accessed;
  public:
    C(): x(0), times_accessed(0) {
    }
    int get_x() const {
        ++times_accessed; // ok: const member function can modify mutable data member
        return x;
    }
    void set_x(int x) {
        ++times_accessed;
        this->x = x;
    }
};

C++ 11’de “değişebilir” için ikinci bir anlam eklendi. Bir lambda’nın parametre listesini izlediğinde, lambda’nın işlev çağrısı operatöründeki örtük ‘const’u bastırır. Bu nedenle, değişken bir lambda, kopya tarafından yakalanan varlıkların değerlerini değiştirebilir. Daha fazla ayrıntı için değişken lambdalar adresine bakın.

std::vector<int> my_iota(int start, int count) {
    std::vector<int> result(count);
    std::generate(result.begin(), result.end(),
                  [start]() mutable { return start++; });
    return result;
}

Değişken bir lambda oluşturmak için bu şekilde kullanıldığında “mutable"ın bir depolama sınıfı belirteci olmadığını unutmayın.

son sürüm

statik #

“Statik” depolama sınıfı belirtecinin üç farklı anlamı vardır.

  1. Ad alanı kapsamında bildirilen bir değişkene veya işleve dahili bağlantı verir.

    // internal function; can't be linked to
static double semiperimeter(double a, double b, double c) {
    return (a + b + c)/2.0;
}
// exported to client
double area(double a, double b, double c) {
    const double s = semiperimeter(a, b, c);
    return sqrt(s*(s-a)*(s-b)*(s-c));
}

  2. Bir değişkenin statik depolama süresine sahip olduğunu bildirir (thread_local olmadığı sürece). Ad alanı-kapsam değişkenleri dolaylı olarak statiktir. Statik bir yerel değişken yalnızca bir kez başlatılır, ilk kez denetim tanımından geçer ve kapsamından her çıkıldığında yok edilmez.

    void f() {
    static int count = 0;
    std::cout << "f has been called " << ++count << " times so far\n";
}

  3. Bir sınıf üyesinin bildirimine uygulandığında, o üyenin [statik üye]olduğunu bildirir(https://www.wikiod.com/tr/docs/c%2b%2b/508/classes-structures/15150/static-class -üyeler).

    struct S {
    static S* create() {
        return new S;
    }
};
int main() {
    S* s = S::create();
}

Bir sınıfın statik veri üyesi olması durumunda, hem 2 hem de 3’ün aynı anda geçerli olduğuna dikkat edin: ‘static’ anahtar sözcüğü, üyeyi hem statik bir veri üyesi yapar hem de statik depolama süresi olan bir değişken haline getirir.

Oto #

Otomatik depolama süresine sahip bir değişken bildirir. Otomatik depolama süresi blok kapsamında zaten varsayılan olduğundan ve ad alanı kapsamında otomatik belirteç izin verilmediğinden gereksizdir.

void f() {
    auto int x; // equivalent to: int x;
    auto y;     // illegal in C++; legal in C89
}
auto int z;     // illegal: namespace-scope variable cannot be automatic
son sürüm

C++ 11’de “auto” anlamı tamamen değişti ve artık bir depolama sınıfı belirteci değil, bunun yerine [tür kesintisi] için kullanılıyor (https://www.wikiod.com/tr/docs/c%2b%2b/7863/ tür kesinti/25567/otomatik tür kesinti).
