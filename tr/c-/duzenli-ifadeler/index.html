Normal İfadeler (bazen normal ifadeler veya normal ifadeler olarak adlandırılır), üzerinde çalışılan dizelerde eşleştirilebilecek kalıpları temsil eden metinsel bir sözdizimidir.

c++11 içinde sunulan Normal İfadeler, isteğe bağlı olarak eşleşen dizelerin bir dönüş dizisini veya eşleşen kalıpların nasıl değiştirileceğini tanımlayan başka bir metin sözdizimini destekleyebilir üzerinde çalıştırılan dizelerde.

Sözdizimi #

  • regex_match // İsteğe bağlı olarak bir eşleşme nesnesine yakalayarak, tüm karakter dizisinin normal ifadeyle eşleşip eşleşmediğini döndürür
  • regex_search // Karakter dizisinin bir bölümünün normal ifadeyle eşleşip eşleşmediğini döndürür, isteğe bağlı olarak bir eşleşme nesnesine yakalanır
  • regex_replace // Değiştirme biçimi dizesi aracılığıyla bir regex tarafından değiştirilen giriş karakter dizisini döndürür
  • regex_token_iterator // Yineleyiciler tarafından tanımlanan bir karakter dizisi, yinelenecek bir yakalama dizinleri listesi ve bir normal ifade ile başlatıldı. Dereferencing, normal ifadenin şu anda dizine alınmış eşleşmesini döndürür. Arttırma, sonraki yakalama dizinine hareket eder veya şu anda son dizindeyse, dizini sıfırlar ve karakter dizisinde bir sonraki normal ifade eşleşmesini engeller
  • regex_iterator // Yineleyiciler ve bir regex tarafından tanımlanan bir karakter dizisi ile başlatıldı. Dereferencing, karakter dizisinin tüm normal ifadenin şu anda eşleştiği bölümünü döndürür. Arttırma, karakter dizisinde bir sonraki normal ifade eşleşmesini bulur

Parametreler #

İmza Açıklama

Temel regex_match ve regex_search Örnekleri #

const auto input = "Some people, when confronted with a problem, think \"I know, I'll use regular expressions.\""s;
smatch sm;

cout << input << endl;

// If input ends in a quotation that contains a word that begins with "reg" and another word begining with "ex" then capture the preceeding portion of input
if (regex_match(input, sm, regex("(.*)\".*\\breg.*\\bex.*\"\\s*$"))) {
    const auto capture = sm[1].str();
    
    cout << '\t' << capture << endl; // Outputs: "\tSome people, when confronted with a problem, think\n"
    
    // Search our capture for "a problem" or "# problems"
    if(regex_search(capture, sm, regex("(a|d+)\\s+problems?"))) {
        const auto count = sm[1] == "a"s ? 1 : stoi(sm[1]);
        
        cout << '\t' << count << (count > 1 ? " problems\n" : " problem\n"); // Outputs: "\t1 problem\n"
        cout << "Now they have " << count + 1 << " problems.\n"; // Ouputs: "Now they have 2 problems\n"
    }
}

regex_iterator Örneği #

Yakalamaların tekrar tekrar işlenmesi gerektiğinde, bir “regex_iterator” iyi bir seçimdir. Bir “regex_iterator” başvurusunun kaldırılması, bir “match_result” döndürür. Bu, karşılıklı bağımlılığa sahip koşullu yakalamalar veya yakalamalar için harikadır. Diyelim ki bazı C++ kodlarını belirtmek istiyoruz. Verilen:

enum TOKENS {
    NUMBER,
    ADDITION,
    SUBTRACTION,
    MULTIPLICATION,
    DIVISION,
    EQUALITY,
    OPEN_PARENTHESIS,
    CLOSE_PARENTHESIS
};

Bu dizgiyi belirtebiliriz: const auto input = "42/2 + -8\t=\n(2 + 2) * 2 * 2 -3"s şunun gibi bir ‘regex_iterator’ ile:

vector<TOKENS> tokens;
const regex re{ "\\s*(\\(?)\\s*(-?\\s*\\d+)\\s*(\\)?)\\s*(?:(\\+)|(-)|(\\*)|(/)|(=))" };

for_each(sregex_iterator(cbegin(input), cend(input), re), sregex_iterator(), [&](const auto& i) {
    if(i[1].length() > 0) {
        tokens.push_back(OPEN_PARENTHESIS);
    }
    
    tokens.push_back(i[2].str().front() == '-' ? NEGATIVE_NUMBER : NON_NEGATIVE_NUMBER);
    
    if(i[3].length() > 0) {
        tokens.push_back(CLOSE_PARENTHESIS);
    }        
    
    auto it = next(cbegin(i), 4);
    
    for(int result = ADDITION; it != cend(i); ++result, ++it) {
        if (it->length() > 0U) {
            tokens.push_back(static_cast<TOKENS>(result));
            break;
        }
    }
});

match_results<string::const_reverse_iterator> sm;

if(regex_search(crbegin(input), crend(input), sm, regex{ tokens.back() == SUBTRACTION ? "^\\s*\\d+\\s*-\\s*(-?)" : "^\\s*\\d+\\s*(-?)" })) {
    tokens.push_back(sm[1].length() == 0 ? NON_NEGATIVE_NUMBER : NEGATIVE_NUMBER);
}

Çapalar #

C++ yalnızca 4 bağlantı sağlar:

  • ^ dizenin başlangıcını belirtir
  • dizenin sonunu belirten $
  • \W karakterini veya dizenin başını veya sonunu belirten \b
  • \w karakterini belirten \B

Örneğin, işaretli bir sayı yakalamak istediğimizi varsayalım:

auto input = "+1--12*123/+1234"s;
smatch sm;

if(regex_search(input, sm, regex{ "(?:^|\\b\\W)([+-]?\\d+)" })) {

    do {
        cout << sm[1] << endl;
        input = sm.suffix().str();
    } while(regex_search(input, sm, regex{ "(?:^\\W|\\b\\W)([+-]?\\d+)" }));
}

Bir dizeyi bölme #

std::vector<std::string> split(const std::string &str, std::string regex)
{
    std::regex r{ regex };
    std::sregex_token_iterator start{ str.begin(), str.end(), r, -1 }, end;
    return std::vector<std::string>(start, end);
}

split("Some  string\t with whitespace ", "\\s+"); // "Some", "string", "with", "whitespace"

Niceleyiciler #

Doğrulanacak bir telefon numarası olarak bize “const string input” verildiğini varsayalım. sıfır veya daha fazla niceleyici: regex_match(input, regex("\\d*")) veya bir veya daha fazla niceleyici: regex_match(input , regex("\\d+")) Ancak “input”, “123” gibi geçersiz bir sayısal dize içeriyorsa, bunların ikisi de gerçekten yetersiz kalır: en az 7 basamak:

regex_match(input, regex("\\d{7,}"))

Bu, en az bir telefon rakamı alacağımızı garanti eder, ancak “input”, “123456789012” gibi çok uzun bir sayısal dize de içerebilir. Öyleyse n ile m arasında bir niceleyici ile gidelim, böylece “girdi” en az 7 basamaklı, ancak 11’den fazla değil:

regex_match(input, regex("\\d{7,11}"));

Bu bizi daha da yakınlaştırıyor, ancak [7, 11] aralığındaki geçersiz sayısal dizeler hala kabul ediliyor, örneğin: “123456789” O halde ülke kodunu bir tembel niceleyici ile isteğe bağlı yapalım:

regex_match(input, regex("\\d?\\d{7,10}"))

tembel niceleyicinin olabildiğince az karakterle eşleştiğine dikkat etmek önemlidir, bu nedenle bu karakterin eşleştirilmesinin tek yolu, \d{7,10} ile eşleştirilmiş 10 karakterin olmasıdır. . (İlk karakteri hırsla eşleştirmek için şunu yapmamız gerekirdi: \d{0,1}.) tembel niceleyici başka herhangi bir niceleyiciye eklenebilir.

Şimdi, alan kodunu nasıl isteğe bağlı hale getirebiliriz ve sadece alan kodu mevcutsa bir ülke kodunu nasıl kabul edebiliriz?

regex_match(input, regex("(?:\\d{3,4})?\\d{7}"))

Bu son normal ifadede, \d{7} 7 basamak gerektirir. Bu 7 basamak isteğe bağlı olarak 3 veya 4 basamaktan önce gelir.

tembel niceleyici: \d{3,4}?\d{7} eklemediğimizi unutmayın, \d{3,4}? 3 veya 4 karakter eşleşir, 3 tercih edilir. Bunun yerine yakalamayan grup eşleşmesini en fazla bir kez yapıyoruz, eşleşmemeyi tercih ediyoruz. “Giriş”, “1234567” gibi bir alan kodunu içermiyorsa uyumsuzluğa neden oluyor.

Niceleyici konusunun sonunda, kullanabileceğiniz diğer ek niceleyiciden iyelik niceleyici‘den bahsetmek istiyorum. Ya tembel niceleyici veya iyelik niceleyici herhangi bir niceleyiciye eklenebilir. İyelik niceleyicinin tek işlevi, normal ifade motoruna bunu söyleyerek yardımcı olmaktır, bu karakterleri açgözlülükle almaktır * ve normal ifadenin başarısız olmasına neden olsa bile* asla onlardan vazgeçmemektir. Örneğin, bu pek mantıklı değil: regex_match(input, regex("\\d{3,4}+\\d{7})) Çünkü bir input gibi: “1234567890” \d{3,4}+ olarak eşleştirilenler, 3 eşleştirmesi normal ifadenin başarılı olmasına izin verse bile her zaman 4 karakterle eşleşir.
İyelik niceleyici en iyi şekilde nicelenen belirteç eşleştirilebilir karakter sayısını sınırladığında kullanılır. Örneğin:

regex_match(input, regex("(?:.*\\d{3,4}+){3}"))

“input” aşağıdakilerden herhangi birini içeriyorsa eşleştirmek için kullanılabilir:

123 456 7890
123-456-7890
(123)456-7890
(123) 456 - 7890

Ancak bu regex gerçekten parladığında, “input” yasadışı bir girdi içerdiğinde:

12345 - 67890

İyelik niceleyici olmadan regex motorunun geri dönüp .’ ve 3 veya 4 karakterin* her kombinasyonunu test etmesi ve uyumlu bir kombinasyon bulup bulamayacağını test etmesi gerekir. İyelik niceleyici ile normal ifade, 2.nd iyelik niceleyicinin, ‘0’ karakterinin kaldığı yerden başlar ve normal ifade motoru, ‘.*’ karakterini şu şekilde ayarlamaya çalışır: \d{3,4} eşleşmesine izin ver; normal ifade başarısız olduğunda, daha önceki .* ayarının bir eşleşmeye izin verip vermediğini görmek için geri izleme yapılmaz.
