“auto” anahtar sözcüğü, otomatik olarak türetilmiş bir türü temsil eden bir tür adıdır.

C++98’de zaten ayrılmış bir anahtar kelimeydi, C’den miras alındı. C++‘ın eski sürümlerinde, bir değişkenin otomatik depolama süresine sahip olduğunu açıkça belirtmek için kullanılabilir:

int main()
{
  auto int i = 5; // removing auto has no effect
}

O eski anlam şimdi kaldırıldı.

Temel otomatik örnek #

“auto” anahtar sözcüğü, bir değişken türünün otomatik olarak çıkarılmasını sağlar.

Uzun tip adlarla uğraşırken özellikle kullanışlıdır:

std::map< std::string, std::shared_ptr< Widget > > table;
// C++98
std::map< std::string, std::shared_ptr< Widget > >::iterator i = table.find( "42" );
// C++11/14/17
auto j = table.find( "42" );

döngüler için aralık tabanlı ile:

vector<int> v = {0, 1, 2, 3, 4, 5};
for(auto n: v)
    std::cout << n << ' ';

lambdalar ile:

auto f = [](){ std::cout << "lambda\n"; };
f();

türün tekrarını önlemek için:

auto w = std::make_shared< Widget >();

şaşırtıcı ve gereksiz kopyalardan kaçınmak için:

auto myMap = std::map<int,float>();
myMap.emplace(1,3.14);

std::pair<int,float> const& firstPair2 = *myMap.begin();  // copy!
auto const& firstPair = *myMap.begin();  // no copy!

Kopyanın nedeni, döndürülen türün aslında std::pair<const int,float> olmasıdır!

otomatik ve İfade Şablonları #

“auto”, ifade şablonlarının devreye girdiği yerlerde de sorunlara neden olabilir:

auto mult(int c) {
    return c * std::valarray<int>{1};
}

auto v = mult(3);
std::cout << v[0]; // some value that could be, but almost certainly is not, 3.

Bunun nedeni, “valarray” üzerindeki “operatör*“ün size tembel bir değerlendirme aracı olarak “valarray"e başvuran bir proxy nesnesi vermesidir. ‘auto’ kullanarak, sarkan bir referans oluşturuyorsunuz. ‘mult’ yerine bir ‘std::valarray’ döndürdü, o zaman kod kesinlikle 3 yazdıracaktı.

auto, const ve referanslar #

“auto” anahtar sözcüğü tek başına “int” veya “char"a benzer bir değer türünü temsil eder. Sırasıyla bir const türünü veya bir referans türünü temsil etmek için ‘const’ anahtar sözcüğü ve ‘&’ sembolü ile değiştirilebilir. Bu değiştiriciler birleştirilebilir.

Bu örnekte, “s” bir değer türüdür (türü “std::string” olarak çıkarılacaktır), bu nedenle “for” döngüsünün her yinelemesi vektörden bir dizeyi “s"ye kopyalar.

std::vector<std::string> strings = { "stuff", "things", "misc" };
for(auto s : strings) {
    std::cout << s << std::endl;
}

Döngünün gövdesi ’s’yi değiştirirse (örneğin, ’s.append(” ve şeyler”)’ çağrılarak), “dizeler"in orijinal üyesi değil, yalnızca bu kopya değiştirilir.

Öte yandan, eğer s auto& ile bildirilirse, bu bir referans tipi olacaktır (std::string& olduğu anlaşılır), bu nedenle döngünün her yinelemesinde, bir referansa referans atanacaktır. vektördeki dize:

for(auto& s : strings) {
    std::cout << s << std::endl;
}

Bu döngünün gövdesinde, “s"de yapılan değişiklikler, referans aldığı “dizeler” öğesini doğrudan etkiler.

Son olarak, ’s’, ‘const auto’ olarak bildirilirse, bu bir const referans türü olacaktır, yani döngünün her yinelemesinde vektördeki bir dizeye bir const referansı atanacaktır:

for(const auto& s : strings) {
    std::cout << s << std::endl;
}

Bu döngünün gövdesi içinde, “s” değiştirilemez (yani, üzerinde const olmayan hiçbir yöntem çağrılamaz).

Aralık tabanlı “for” döngüleriyle “auto” kullanıldığında, gereksiz kopyaları önleyeceğinden, döngü gövdesi döngülenen yapıyı değiştirmeyecekse “const auto” kullanmak genellikle iyi bir uygulamadır.

Sondaki dönüş türü #

“auto”, sondaki dönüş türü için sözdiziminde kullanılır:

auto main() -> int {}

hangi eşdeğerdir

int main() {}

“std::declval” yerine parametreleri kullanmak için “decltype” ile birlikte çoğunlukla kullanışlıdır:

template <typename T1, typename T2>
auto Add(const T1& lhs, const T2& rhs) -> decltype(lhs + rhs) { return lhs + rhs; }

Genel lambda (C++14) #

C++ 14, lambda bağımsız değişkeninde “auto” kullanılmasına izin verir

auto print = [](const auto& arg) { std::cout << arg << std::endl; };

print(42);
print("hello world");

Bu lambda çoğunlukla eşdeğerdir

struct lambda {
    template <typename T>
    auto operator ()(const T& arg) const {
        std::cout << arg << std::endl;
    }
};

ve daha sonra

lambda print;

print(42);

otomatik ve proxy nesneleri #

Bazen “auto” bir programcının beklediği gibi davranmayabilir. Tür kesintisi yapılacak doğru şey olmasa bile, tür ifadeyi çıkarır.

Örnek olarak, kodda proxy nesneleri kullanıldığında:

std::vector<bool> flags{true, true, false};
auto flag = flags[0];
flags.push_back(true);

Burada ‘bayrak’, ‘bool’ değil, ‘std::vector::reference’ olacaktır, çünkü ‘vektör’ şablonunun ‘bool’ uzmanlığı için ‘operatör []’, dönüştürme işleci ’ ile bir proxy nesnesi döndürür. operatör bool` tanımlı.

“flags.push_back(true)” kapsayıcıyı değiştirdiğinde, bu sözde referans, artık var olmayan bir öğeye atıfta bulunarak sarkabilir.

Ayrıca bir sonraki durumu mümkün kılar:

void foo(bool b);

std::vector<bool> getFlags();

auto flag = getFlags()[5];
foo(flag);

“vektör” hemen atılır, bu nedenle “bayrak”, atılan bir öğeye sözde referanstır. ‘foo’ çağrısı tanımsız davranışı başlatır.

Bunun gibi durumlarda, bir değişkeni “auto” ile tanımlayabilir ve çıkarılmasını istediğiniz türe yayınlayarak onu başlatabilirsiniz:

auto flag = static_cast<bool>(getFlags()[5]);

ancak bu noktada “auto"yu “bool” ile değiştirmek daha mantıklı.

Proxy nesnelerinin sorunlara neden olabileceği başka bir durum da ifade şablonlarıdır. Bu durumda, şablonlar bazen verimlilik uğruna mevcut tam ifadenin ötesine geçecek şekilde tasarlanmamıştır ve bir sonraki adımda proxy nesnesinin kullanılması tanımsız davranışa neden olur.
