Anahtar sözcüklerin C++ standardı tarafından tanımlanan sabit anlamları vardır ve tanımlayıcı olarak kullanılamazlar. Standart bir kitaplık başlığı içeren herhangi bir çeviri biriminde önişlemciyi kullanarak anahtar sözcükleri yeniden tanımlamak yasa dışıdır. Ancak, anahtar kelimeler, nitelikler içinde özel anlamlarını kaybeder.

Sözdizimi #

  • asm (string-literal);
  • noexcept(ifade) // anlam 1
  • noexcept(sabit ifade) // 2 anlamı
  • noexcept // 2 anlamı
  • sizeof unary-ifadesi
  • sizeof(tür kimliği)
  • sizeof…(tanımlayıcı) // C++11’den beri
  • typename iç içe-isim-belirleyici tanımlayıcı // anlamı 1
  • typename iç içe-isim-belirleyici şablon(opt) simple-template-id // anlam 1
  • typename identifier(opt) // anlamı 2
  • typename… tanımlayıcı(opt) // 2 anlamı; C++11’den beri
  • typename identifier(opt) = type-id // anlamı 2
  • şablon <template-parameter-list> typename …(opt) identifier(opt) // anlam 3
  • şablon <template-parameter-list> typename identifier(opt) = id-expression // anlam 3

“Final” ve “geçersiz kılma” belirteçleri anahtar kelimeler değildir. Tanımlayıcı olarak kullanılabilirler ve yalnızca belirli bağlamlarda özel anlamları vardır.

‘and’, ‘and_eq’, ‘bitand’, ‘bitor’, ‘compl’, ’not’, ’not_eq’, ‘or’, ‘or_eq’, ‘xor’ ve ‘xor_eq’ belirteçleri alternatif yazımlardır. &&, &=, &, |, ~, !, !=, ||, |=, ^ ve ^=, sırasıyla. Standart bunları anahtar kelimeler olarak ele almaz, ancak onları yeniden tanımlamak veya temsil ettikleri operatörlerden başka bir şey ifade etmek için kullanmak imkansız olduğundan, tüm niyet ve amaçlar için anahtar kelimelerdir.

Aşağıdaki konular, temel türleri adlandırma veya yürütme akışını kontrol etme gibi temel amaçlara hizmet eden C++‘daki birçok anahtar kelimenin ayrıntılı açıklamalarını içerir.

asm #

“asm” anahtar sözcüğü, bir dize değişmezi olması gereken tek bir işlenen alır. Uygulama tanımlı bir anlamı vardır, ancak tipik olarak, montajcının çıktısı çeviri birimine dahil edilerek uygulamanın birleştiricisine iletilir.

“asm” ifadesi bir ifade değil bir tanımdır, bu nedenle blok kapsamında veya ad alanı kapsamında (global kapsam dahil) görünebilir. Ancak, satır içi derleme C++ dilinin kurallarıyla sınırlandırılamayacağından, “asm” bir “constexpr” işlevi içinde görünmeyebilir.

Örnek:

[[noreturn]] void halt_system() {
    asm("hlt");
}

açık #

  1. Tek argümanlı bir kurucuya uygulandığında, bu kurucunun örtük dönüştürmeler gerçekleştirmek için kullanılmasını engeller.

    class MyVector {
  public:
    explicit MyVector(uint64_t size);
};
MyVector v1(100);  // ok
uint64_t len1 = 100;
MyVector v2{len1}; // ok, len1 is uint64_t
int len2 = 100;
MyVector v3{len2}; // ill-formed, implicit conversion from int to uint64_t

C++11 başlatıcı listelerini tanıttığından, C++11 ve sonraki sürümlerde “açık”, tek argümanlı durumdakiyle aynı anlama sahip herhangi bir sayıda argüman içeren bir kurucuya uygulanabilir.

   struct S {
       explicit S(int x, int y);
   };
   S f() {
       return {12, 34};  // ill-formed
       return S{12, 34}; // ok
   }

  1. Bir dönüştürme işlevine uygulandığında, bu dönüştürme işlevinin örtük dönüştürmeler gerçekleştirmek için kullanılmasını engeller.

    class C {
    const int x;
  public:
    C(int x) : x(x) {}
    explicit operator int() { return x; }
};
C c(42);
int x = c;                   // ill-formed
int y = static_cast<int>(c); // ok; explicit conversion

istisnasız #

  1. İşleneninin değerlendirmesinin bir istisna yayabileceğini belirleyen birli operatör. Çağrılan işlevlerin gövdelerinin incelenmediğine dikkat edin, bu nedenle “noexcept” yanlış negatifler verebilir. Operand değerlendirilmez.

    #include <iostream>
#include <stdexcept>
void foo() { throw std::runtime_error("oops"); }
void bar() {}
struct S {};
int main() {
    std::cout << noexcept(foo()) << '\n'; // prints 0
    std::cout << noexcept(bar()) << '\n'; // prints 0
    std::cout << noexcept(1 + 1) << '\n'; // prints 1
    std::cout << noexcept(S()) << '\n';   // prints 1
}

Bu örnekte, “bar()” hiçbir zaman bir istisna oluşturamasa da, “noexcept(bar())” yine yanlıştır çünkü “bar()“ın bir istisnayı yayamayacağı gerçeği açıkça belirtilmemiştir.

  1. Bir işlev bildirirken, işlevin bir özel durum oluşturup oluşturamayacağını belirtir. Tek başına, işlevin bir istisna yayamayacağını bildirir. Parantez içine alınmış bir argümanla, fonksiyonun argümanın doğruluk değerine bağlı olarak bir istisna oluşturabileceğini veya yayamayacağını bildirir.

    void f1() { throw std::runtime_error("oops"); }
void f2() noexcept(false) { throw std::runtime_error("oops"); }
void f3() {}
void f4() noexcept {}
void f5() noexcept(true) {}
void f6() noexcept {
    try {
        f1();
    } catch (const std::runtime_error&) {}
}

Bu örnekte, “f4”, “f5” ve “f6"nın istisnaları yayamayacağını beyan ettik. (‘f6’nın yürütülmesi sırasında bir istisna atılabilse de, yakalanır ve fonksiyon dışına yayılmasına izin verilmez.) ‘f2’nin bir istisna yayabileceğini beyan etmiştik. ’noexcept’ belirteci atlandığında, ’noexcept(false)’ ile eşdeğerdir, bu nedenle ‘f1’ ve ‘f3’ün, ‘f3’ün yürütülmesi sırasında istisnalar fiilen atılamasa bile, istisnaları yayabileceğini dolaylı olarak beyan ettik. `.

Bir işlevin “noexcept” olup olmadığı, işlevin türünün bir parçasıdır: yani, yukarıdaki örnekte “f1”, “f2” ve “f3”, “f4”, “f5” ve “f3"ten farklı türlere sahiptir. f6’. Bu nedenle, “noexcept”, işlev işaretçilerinde, şablon argümanlarında ve benzerlerinde de önemlidir.

void g1() {}
void g2() noexcept {}
void (*p1)() noexcept = &g1; // ill-formed, since g1 is not noexcept
void (*p2)() noexcept = &g2; // ok; types match
void (*p3)() = &g1;          // ok; types match
void (*p4)() = &g2;          // ok; implicit conversion

tür adı #

  1. Nitelikli bir ad tarafından takip edildiğinde, ’typename’, bunun bir türün adı olduğunu belirtir. Bu genellikle şablonlarda, özellikle yuvalanmış ad belirteci geçerli örneklemeden farklı bir bağımlı tür olduğunda gereklidir. Bu örnekte, ‘std::decay’, ‘T’ şablon parametresine bağlıdır, bu nedenle iç içe geçmiş türü ’tür’ olarak adlandırmak için, nitelenen adın tamamının önüne ’typename’ koymamız gerekir. Daha fazla ayrıntı için bkz. [“şablon” ve “tür adı” anahtar sözcüklerini nereye ve neden koymak zorundayım?](http://stackoverflow.com/questions/610245/where-and-why-do-i-have- Şablonu-ve-tipi-adı-anahtar kelimeleri koymak için)

    template <class T>
auto decay_copy(T&& r) -> typename std::decay<T>::type;

  2. Bir şablon bildiriminde bir tür parametresi sunar. Bu bağlamda, ‘sınıf’ ile değiştirilebilir.

    template <typename T>
const T& min(const T& x, const T& y) {
    return b < a ? b : a;
}

  1. “typename”, adından önce bir şablon şablon parametresi bildirilirken de kullanılabilir parametrenin, tıpkı “sınıf” gibi.

    template <template <class T> typename U>
void f() {
    U<int>::do_it();
    U<double>::do_it();

boyutu #

Bir ifade veya tür olabilen, işleneninin bayt cinsinden boyutunu veren tekli bir operatör. İşlenen bir ifade ise, değerlendirilmez. Boyut, “std::size_t” türünün sabit bir ifadesidir.

İşlenen bir tür ise, parantez içine alınmalıdır.

  • Bir işlev türüne “sizeof” uygulamak yasa dışıdır.
  • “void” dahil, tamamlanmamış bir türe “sizeof” uygulamak yasa dışıdır.
  • Sizeof, T& veya T&& referans türüne uygulanırsa, sizeof(T) ile eşdeğerdir.
  • Bir sınıf türüne ‘sizeof’ uygulandığında, ortadaki veya sondaki doldurma baytları da dahil olmak üzere, o türdeki tam bir nesnedeki bayt sayısını verir. Bu nedenle, bir “sizeof” ifadesi asla 0 değerine sahip olamaz. Bunun için nesne türlerinin düzenine bakın. daha fazla detay.
  • ‘char’, ‘signed char’ ve ‘unsigned char’ türlerinin boyutu 1’dir. Tersine, bir bayt, bir ‘char’ nesnesini depolamak için gereken bellek miktarı olarak tanımlanır. Bazı sistemlerde 8 bitten daha uzun “char” nesneleri olduğundan, mutlaka 8 bit anlamına gelmez.

ifade bir ifadeyse, “sizeof(*expr*)`, “sizeof(T)” ile eşdeğerdir; burada “T”, ifade türüdür.

int a[100];
std::cout << "The number of bytes in `a` is: " << sizeof a;
memset(a, 0, sizeof a); // zeroes out the array

sizeof... operatörü, bir parametre paketindeki eleman sayısını verir.

template <class... T>
void f(T&&...) {
    std::cout << "f was called with " << sizeof...(T) << " arguments\n";
}

geçersiz C++

  1. Bir işlev döndürme türü olarak kullanıldığında, void anahtar sözcüğü, işlevin bir değer döndürmediğini belirtir. Bir fonksiyonun parametre listesi için kullanıldığında void, fonksiyonun parametre almadığını belirtir. Bir işaretçi bildiriminde kullanıldığında void, işaretçinin “evrensel” olduğunu belirtir.

  2. İşaretçi türü void * ise, işaretçi const veya volatile anahtar sözcüğüyle bildirilmeyen herhangi bir değişkene işaret edebilir. Bir geçersiz işaretçi, başka bir türe dönüştürülmediği sürece başvurudan kaldırılamaz. Bir geçersiz işaretçi, başka herhangi bir veri işaretçisine dönüştürülebilir.

bu işaretçi**

  1. this işaretçisi, yalnızca bir sınıf, yapı veya birleşim türünün statik olmayan üye işlevleri içinde erişilebilen bir işaretçidir. Üye işlevinin çağrıldığı nesneye işaret eder. Statik üye işlevlerinde this işaretçisi yoktur.

     this->member-identifier

Bir nesnenin bu işaretçisi, nesnenin kendisinin bir parçası değildir; nesne üzerindeki sizeof ifadesinin sonucuna yansıtılmaz. Bunun yerine, bir nesne için statik olmayan bir üye işlev çağrıldığında, nesnenin adresi derleyici tarafından işleve gizli bir argüman olarak iletilir. Örneğin, aşağıdaki işlev çağrısı:

myDate.setMonth( 3 );  

can be interpreted this way:


setMonth( &myDate, 3 );  

The object's address is available from within the member function as the this pointer. Most uses of this are implicit. It is legal, though unnecessary, to explicitly use this when referring to members of the class. For example:


void Date::setMonth( int mn )  
{  
   month = mn;            // These three statements  
   this->month = mn;      // are equivalent  
   (*this).month = mn;  
}  

The expression *this is commonly used to return the current object from a member function:


return *this;  

The this pointer is also used to guard against self-reference:


if (&Object != this) {  
// do not execute in cases of self-reference

Deneyin, atın ve İfadeleri yakalayın (C++)**

  1. C++‘da istisna işlemeyi uygulamak için try, throw ve catch ifadelerini kullanırsınız.

  2. İlk olarak, bir istisna oluşturabilecek bir veya daha fazla ifadeyi içine almak için bir try bloğu kullanın.

  3. Bir atma ifadesi, bir try bloğunda istisnai bir durumun (genellikle bir hatanın) oluştuğunu belirtir. Herhangi bir türdeki bir nesneyi, atma ifadesinin işleneni olarak kullanabilirsiniz. Tipik olarak, bu nesne hata hakkında bilgi iletmek için kullanılır. Çoğu durumda, std::exception sınıfını veya standart kitaplıkta tanımlanan türetilmiş sınıflardan birini kullanmanızı öneririz. Bunlardan biri uygun değilse, kendi istisna sınıfınızı std::exception’dan türetmenizi öneririz.

  4. Atılabilecek istisnaları ele almak için, bir try bloğunun hemen ardından bir veya daha fazla yakalama bloğu uygulayın. Her bir yakalama bloğu, işleyebileceği istisna türünü belirtir.

     MyData md;

    try {
    // Code that could throw an exception
    md = GetNetworkResource();
    }
    catch (const networkIOException& e) {
    // Code that executes when an exception of type
    // networkIOException is thrown in the try block
    // …
    // Log error message in the exception object
    cerr « e.what();
    }
    catch (const myDataFormatException& e) {
    // Code that handles another exception type
    // …
    cerr « e.what();
    }

    // The following syntax shows a throw expression
    MyData GetNetworkResource()
    {
    // …
    if (IOSuccess == false)
    throw networkIOException(“Unable to connect”);
    // …
    if (readError)
    throw myDataFormatException(“Format error”);
    // …
    }

try yan tümcesinden sonraki kod, kodun korunan bölümüdür. atma ifadesi bir istisna atar, yani yükseltir. Catch yan tümcesinden sonraki kod bloğu, istisna işleyicisidir. Bu, atma ve yakalama ifadelerindeki türler uyumluysa, oluşturulan istisnayı yakalayan işleyicidir.

    try {  
   throw CSomeOtherException();  
}  
catch(...) {  
   // Catch all exceptions – dangerous!!!  
   // Respond (perhaps only partially) to the exception, then  
   // re-throw to pass the exception to some other handler  
   // ...  
   throw;  
}

arkadaş (C++)

  1. Bazı durumlarda, bir sınıfın üyesi olmayan işlevlere veya ayrı bir sınıftaki tüm üyelere üye düzeyinde erişim vermek daha uygundur. Sadece sınıf uygulayıcısı arkadaşlarının kim olduğunu bildirebilir. Bir işlev veya sınıf, kendisini herhangi bir sınıfın arkadaşı olarak ilan edemez. Bir sınıf tanımında, arkadaş anahtar sözcüğünü ve üye olmayan bir işlevin veya başka bir sınıfın adını kullanarak, sınıfınızın özel ve korumalı üyelerine erişim izni verin. Bir şablon tanımında, bir tür parametresi arkadaş olarak bildirilebilir.

  2. Daha önce bildirilmemiş bir arkadaş işlevi bildirirseniz, bu işlev çevreleyen sınıf dışı kapsama aktarılır.

     class friend F  
 friend F;
 class ForwardDeclared;// Class name is known.  
 class HasFriends  
 {  
    friend int ForwardDeclared::IsAFriend();// C2039 error expected  
 };

arkadaş işlevleri

  1. Bir arkadaş işlevi, bir sınıfın üyesi olmayan ancak sınıfın özel ve korumalı üyelerine erişimi olan bir işlevdir.Arkadaş işlevleri, sınıf üyesi olarak kabul edilmez; bunlar, özel erişim ayrıcalıkları verilen normal harici işlevlerdir.

  2. Arkadaşlar sınıfın kapsamında değildir ve başka bir sınıfın üyesi olmadıkça üye seçim operatörleri (. ve –>) kullanılarak çağrılmazlar.

  3. Erişim sağlayan sınıf tarafından bir arkadaş işlevi bildirilir. Arkadaş bildirimi, sınıf bildiriminde herhangi bir yere yerleştirilebilir. Erişim denetimi anahtar sözcüklerinden etkilenmez.

     #include <iostream>  

 using namespace std;  
 class Point  
 {  
     friend void ChangePrivate( Point & );  
 public:  
     Point( void ) : m_i(0) {}  
     void PrintPrivate( void ){cout << m_i << endl; }  

 private:  
 int m_i;  
 };  

 void ChangePrivate ( Point &i ) { i.m_i++; }  

 int main()  
 {  
    Point sPoint;  
    sPoint.PrintPrivate();  
    ChangePrivate(sPoint);  
    sPoint.PrintPrivate();  
     // Output: 0  
            1  
 }

class B;  

class A {  
public:  
   int Func1( B& b );  

private:  
   int Func2( B& b );  
};  

class B {  
private:  
int _b;  

   // A::Func1 is a friend function to class B  
   // so A::Func1 has access to all members of B  
   friend int A::Func1( B& );  
};  

int A::Func1( B& b ) { return b._b; }   // OK  
int A::Func2( B& b ) { return b._b; }   // C2248
