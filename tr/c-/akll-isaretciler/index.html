Sözdizimi #

  • std::shared_ptr<ClassType> değişkenAdı = std::make_shared<ClassType>(arg1, arg2, ...);
  • std::shared_ptr<ClassType> değişkenAdı (yeni ClassType(arg1, arg2, ...));
  • std::unique_ptr<ClassType> değişkenAdı = std::make_unique<ClassType>(arg1, arg2, ...); // C++14
  • std::unique_ptr<SınıfTürü> değişkenAdı (yeni SınıfTürü(arg1, arg2, ...));

C++, bellek tarafından yönetilen bir dil değildir. Dinamik olarak ayrılmış bellek (yani, “yeni” ile oluşturulan nesneler), açıkça ayrılmamışsa (“sil” ile) “sızdırılır”. Bu nesneye son işaretçiyi atmadan önce dinamik olarak ayrılan belleğin serbest bırakılmasını sağlamak programcının sorumluluğundadır.

Akıllı işaretçiler, dinamik olarak ayrılmış belleğin kapsamını otomatik olarak yönetmek için kullanılabilir (yani, son işaretçi referansı kapsam dışına çıktığında silinir).

Akıllı işaretçiler çoğu durumda “ham” işaretçiler yerine tercih edilir. Bir nesnenin paylaşılmasının mı yoksa benzersiz bir şekilde sahiplenilmesinin mi amaçlandığını kendi adlarına ileterek, dinamik olarak ayrılmış belleğin sahiplik semantiğini açık hale getirirler.

Akıllı işaretçileri kullanabilmek için #include <memory> kullanın.

Benzersiz sahiplik (std::unique_ptr) #

std::unique_ptr, dinamik olarak depolanan bir nesnenin ömrünü yöneten bir sınıf şablonudur. std::shared_ptr‘den farklı olarak, dinamik nesne herhangi bir zamanda bir “std::unique_ptr"nin yalnızca bir örneğine aittir,

// Creates a dynamic int with value of 20 owned by a unique pointer
std::unique_ptr<int> ptr = std::make_unique<int>(20);

(Not: std::unique_ptr C++11’den beri ve `std::make_unique’ C++14’ten beri mevcuttur.)

Yalnızca “ptr” değişkeni, dinamik olarak ayrılmış bir “int” için bir işaretçi tutar. Bir nesneye sahip benzersiz bir işaretçi kapsam dışına çıktığında, sahip olunan nesne silinir, yani nesne sınıf türündeyse yıkıcısı çağrılır ve o nesnenin belleği serbest bırakılır.

‘std::unique_ptr’ ve ‘std::make_unique’ dizi türleriyle birlikte kullanmak için dizi uzmanlıklarını kullanın:

// Creates a unique_ptr to an int with value 59
std::unique_ptr<int> ptr = std::make_unique<int>(59);

// Creates a unique_ptr to an array of 15 ints
std::unique_ptr<int[]> ptr = std::make_unique<int[]>(15);

std::unique_ptr’ye ham bir işaretçi gibi erişebilirsiniz, çünkü bu operatörleri aşırı yükler.

Bir akıllı işaretçinin içeriğinin sahipliğini, orijinal akıllı işaretçinin “nullptr"yi göstermesine neden olacak şekilde “std::move” kullanarak başka bir işaretçiye aktarabilirsiniz.

// 1. std::unique_ptr
std::unique_ptr<int> ptr = std::make_unique<int>();

// Change value to 1
*ptr = 1;

// 2. std::unique_ptr (by moving 'ptr' to 'ptr2', 'ptr' doesn't own the object anymore)
std::unique_ptr<int> ptr2 = std::move(ptr);

int a = *ptr2; // 'a' is 1
int b = *ptr;  // undefined behavior! 'ptr' is 'nullptr'
               // (because of the move command above)

Parametre olarak işlevlere “unique_ptr” geçirilmesi:

void foo(std::unique_ptr<int> ptr)
{
    // Your code goes here
}

std::unique_ptr<int> ptr = std::make_unique<int>(59);
foo(std::move(ptr))

İşlevlerden “unique_ptr” döndürülüyor. Bu, fabrika işlevlerini yazmanın tercih edilen C++11 yoludur, çünkü geri dönüşün sahiplik semantiğini açıkça iletir: arayan, sonuçta ortaya çıkan ‘unique_ptr’nin sahibidir ve bundan sorumludur.

std::unique_ptr<int> foo()
{
    std::unique_ptr<int> ptr = std::make_unique<int>(59);
    return ptr;
}

std::unique_ptr<int> ptr = foo();

Bunu şununla karşılaştırın:

int* foo_cpp03();

int* p = foo_cpp03(); // do I own p? do I have to delete it at some point?
                      // it's not readily apparent what the answer is.
son sürüm

“make_unique” sınıf şablonu C++14’ten beri sağlanmaktadır. C++11 koduna manuel olarak eklemek kolaydır:

template<typename T, typename... Args>
typename std::enable_if<!std::is_array<T>::value, std::unique_ptr<T>>::type
make_unique(Args&&... args)
{ return std::unique_ptr<T>(new T(std::forward<Args>(args)...)); }

// Use make_unique for arrays
template<typename T>
typename std::enable_if<std::is_array<T>::value, std::unique_ptr<T>>::type
make_unique(size_t n)
{ return std::unique_ptr<T>(new typename std::remove_extent<T>::type[n]()); }
son sürüm

dumb akıllı işaretçinin (std::auto_ptr) aksine, unique_ptr vektör tahsisi ile de başlatılabilir (not std::vector). Daha önceki örnekler skaler ayırmalar içindi. Örneğin, 10 öğe için dinamik olarak ayrılmış bir tamsayı dizisine sahip olmak için, şablon türü olarak ‘int[]’ belirtirsiniz (yalnızca ‘int’ değil):

std::unique_ptr<int[]> arr_ptr = std::make_unique<int[]>(10);

Hangisi ile basitleştirilebilir:

auto arr_ptr = std::make_unique<int[]>(10);

Şimdi, “arr_ptr"yi bir diziymiş gibi kullanırsınız:

arr_ptr[2] =  10; // Modify third element

Ayrılma konusunda endişelenmenize gerek yok. Bu şablonun özel sürümü, yapıcıları ve yıkıcıları uygun şekilde çağırır. “unique_ptr"nin vektörlü versiyonunu veya “vektör"ün kendisini kullanmak - kişisel bir seçimdir.

son sürüm

C++ 11’den önceki sürümlerde std::auto_ptr mevcuttu. ‘unique_ptr’den farklı olarak, ‘ptr’ kaynağının içerdiği işaretçinin sahipliğini kaybedeceği ve hedefin onu alacağı auto_ptr’leri kopyalamaya izin verilir.

Sahipliği paylaşma (std::shared_ptr) #

std::shared_ptr sınıf şablonu, bir nesnenin sahipliğini diğer paylaşılan işaretçilerle paylaşabilen paylaşılan bir işaretçi tanımlar. Bu, özel mülkiyeti temsil eden std::unique_ptr ile çelişir.

Paylaşım davranışı, nesneye işaret eden paylaşılan işaretçilerin sayısının yanında depolandığı, referans sayımı olarak bilinen bir teknikle gerçekleştirilir. Bu sayı, son std::shared_ptr örneğinin yok edilmesi veya yeniden atanması yoluyla sıfıra ulaştığında, nesne otomatik olarak yok edilir.

// Creation: 'firstShared' is a shared pointer for a new instance of 'Foo'
std::shared_ptr<Foo> firstShared = std::make_shared<Foo>(/*args*/);

Aynı nesneyi paylaşan birden çok akıllı işaretçi oluşturmak için, ilk paylaşılan işaretçiyi diğer adıyla adlandıran başka bir “shared_ptr” oluşturmamız gerekiyor. İşte bunu yapmanın 2 yolu:

std::shared_ptr<Foo> secondShared(firstShared);  // 1st way: Copy constructing
std::shared_ptr<Foo> secondShared;
secondShared = firstShared;                      // 2nd way: Assigning

Yukarıdaki yollardan herhangi biri, ‘secondShared’i, ‘Foo’ örneğimizin sahipliğini ‘firstShared’ ile paylaşan paylaşılan bir işaretçi yapar.

Akıllı işaretçi, ham işaretçi gibi çalışır. Bu, onları referanstan çıkarmak için * kullanabileceğiniz anlamına gelir. Normal -> operatörü de çalışır:

secondShared->test(); // Calls Foo::test()

Son olarak, son takma ad olan ‘shared_ptr’ kapsam dışına çıktığında, ‘Foo’ örneğimizin yıkıcısı çağrılır.

Uyarı: Bir “shared_ptr” oluşturmak, paylaşılan sahiplik semantiği için fazladan veri ayrılması gerektiğinde bir “bad_alloc” istisnası oluşturabilir. Yapıcı normal bir işaretçi iletilirse, işaret edilen nesnenin sahibi olduğunu varsayar ve bir istisna atılırsa siliciyi çağırır. Bu, “shared_ptr” öğesinin tahsisi başarısız olursa “shared_ptr(new T(args))” öğesinin bir “T” nesnesini sızdırmayacağı anlamına gelir. Ancak, uygulamanın bellek tahsisini optimize etmesini sağlayan make_shared<T>(args) veya allocate_shared<T>(alloc, args) kullanılması tavsiye edilir.

Shared_ptr kullanarak Dizileri Tahsis Etme([])

Ne yazık ki, make_shared<> kullanarak Dizileri tahsis etmenin doğrudan bir yolu yoktur.

’new’ ve ‘std::default_delete’ kullanarak ‘shared_ptr<>’ için diziler oluşturmak mümkündür.

Örneğin, 10 tamsayıdan oluşan bir dizi ayırmak için kodu şu şekilde yazabiliriz:

shared_ptr<int> sh(new int[10], std::default_delete<int[]>());

Ayrılan belleğin ‘delete[]’ kullanılarak doğru şekilde temizlendiğinden emin olmak için burada ‘std::default_delete’ belirtilmesi zorunludur.

Derleme zamanında boyutu biliyorsak, bunu şu şekilde yapabiliriz:

template<class Arr>
struct shared_array_maker {};
template<class T, std::size_t N>
struct shared_array_maker<T[N]> {
  std::shared_ptr<T> operator()const{
    auto r = std::make_shared<std::array<T,N>>();
    if (!r) return {};
    return {r.data(), r};
  }
};
template<class Arr>
auto make_shared_array()
-> decltype( shared_array_maker<Arr>{}() )
{ return shared_array_maker<Arr>{}(); }

sonra ‘make_shared_array<int[10]>’, tümü varsayılan olarak oluşturulmuş 10 inç’e işaret eden bir ‘shared_ptr’ döndürür.

son sürüm

C++17 ile, dizi türleri için shared_ptr özel destek kazandı. Artık dizi siliciyi açıkça belirtmek gerekli değildir ve paylaşılan işaretçi “[]” dizi dizin operatörü kullanılarak başvurudan çıkarılabilir:

std::shared_ptr<int[]> sh(new int[10]);
sh[0] = 42;
son sürüm

Paylaşılan işaretçiler, sahip olduğu nesnenin bir alt nesnesine işaret edebilir:

struct Foo { int x; };
std::shared_ptr<Foo> p1 = std::make_shared<Foo>();
std::shared_ptr<int> p2(p1, &p1->x);

Hem “p2” hem de “p1”, “Foo” türündeki nesneye sahiptir, ancak “p2”, “int” üyesi “x"e işaret eder. Bu, “p1” kapsam dışına çıkarsa veya yeniden atanırsa, temeldeki “Foo” nesnesinin hala canlı olacağı ve “p2"nin sarkmadığından emin olacağı anlamına gelir.

Önemli: Bir “shared_ptr” yalnızca kendisi ve takma ad oluşturucuyla oluşturulan diğer tüm “shared_ptr” hakkında bilgi sahibidir. Aynı “Foo” örneğine referansla oluşturulan diğer tüm “shared_ptr"ler dahil olmak üzere diğer işaretçiler hakkında bir bilgisi yoktur:

Foo *foo = new Foo;
std::shared_ptr<Foo> shared1(foo);
std::shared_ptr<Foo> shared2(foo); // don't do this

shared1.reset(); // this will delete foo, since shared1
                 // was the only shared_ptr that owned it

shared2->test(); // UNDEFINED BEHAVIOR: shared2's foo has been
                 // deleted already!!

Paylaşılan_ptr’nin Mülkiyet Devri

Varsayılan olarak, “shared_ptr” referans sayısını artırır ve sahipliği aktarmaz. Ancak, std::move kullanılarak mülkiyet devri yapılabilir:

shared_ptr<int> up = make_shared<int>();
// Transferring the ownership
shared_ptr<int> up2 = move(up);
// At this point, the reference count of up = 0 and the
// ownership of the pointer is solely with up2 with reference count = 1

Geçici sahiplikle paylaşma (std::weak_ptr) #

std::weak_ptr örnekleri, yalnızca geçici sahipler olurken std::shared_ptr örneklerine ait nesnelere işaret edebilir. Bu, zayıf işaretçilerin nesnenin referans sayısını değiştirmediği ve bu nedenle, nesnenin paylaşılan işaretçilerinin tümü yeniden atandığında veya yok edildiğinde bir nesnenin silinmesini engellemediği anlamına gelir.

Aşağıdaki örnekte, bir ağaç nesnesinin yok edilmesinin engellenmemesi için “std::weak_ptr” örnekleri kullanılmıştır:

#include <memory>
#include <vector>

struct TreeNode {
    std::weak_ptr<TreeNode> parent;
    std::vector< std::shared_ptr<TreeNode> > children;
};

int main() {
    // Create a TreeNode to serve as the root/parent.
    std::shared_ptr<TreeNode> root(new TreeNode);

    // Give the parent 100 child nodes.
    for (size_t i = 0; i < 100; ++i) {
        std::shared_ptr<TreeNode> child(new TreeNode);
        root->children.push_back(child);
        child->parent = root;
    }

    // Reset the root shared pointer, destroying the root object, and
    // subsequently its child nodes.
    root.reset();
}

Alt düğümler, kök düğümün çocuklarına eklendikçe, onların ‘std::weak_ptr’ üyesi ‘parent’ kök düğüme ayarlanır. Kök düğümün referans sayısı artmayacak şekilde, üye ’ebeveyn’, paylaşılan bir işaretçinin aksine zayıf bir işaretçi olarak bildirilir. Kök düğüm “main()” sonunda sıfırlandığında kök yok edilir. Alt düğümlere yönelik geriye kalan tek ‘std::shared_ptr’ referansları kökün ‘children’ koleksiyonunda yer aldığından, tüm alt düğümler de sonradan yok edilir.

Kontrol bloğu uygulama detayları nedeniyle, paylaşılan_ptr tahsisli bellek, ‘shared_ptr’ referans sayacı ve ‘weak_ptr’ referans sayacının her ikisi de sıfıra ulaşana kadar serbest bırakılmayabilir.

#include <memory>
int main()
{
    {
         std::weak_ptr<int> wk;
         {
             // std::make_shared is optimized by allocating only once 
             // while std::shared_ptr<int>(new int(42)) allocates twice.
             // Drawback of std::make_shared is that control block is tied to our integer
             std::shared_ptr<int> sh = std::make_shared<int>(42);
             wk = sh;
             // sh memory should be released at this point...
         }
         // ... but wk is still alive and needs access to control block
     }
     // now memory is released (sh and wk)
}

‘std::weak_ptr’ başvurulan nesnesini canlı tutmadığından, bir ‘std::weak_ptr’ aracılığıyla doğrudan veri erişimi mümkün değildir. Bunun yerine, başvurulan nesneye bir “std::shared_ptr” almaya çalışan bir “lock()” üye işlevi sağlar:

#include <cassert>
#include <memory>
int main()
{
    {
         std::weak_ptr<int> wk;
         std::shared_ptr<int> sp;
         {
             std::shared_ptr<int> sh = std::make_shared<int>(42);
             wk = sh;
             // calling lock will create a shared_ptr to the object referenced by wk
             sp = wk.lock();
             // sh will be destroyed after this point, but sp is still alive
         }
         // sp still keeps the data alive.
         // At this point we could even call lock() again 
         // to retrieve another shared_ptr to the same data from wk
         assert(*sp == 42);
         assert(!wk.expired());
         // resetting sp will delete the data,
         // as it is currently the last shared_ptr with ownership
         sp.reset();
         // attempting to lock wk now will return an empty shared_ptr,
         // as the data has already been deleted
         sp = wk.lock();
         assert(!sp);
         assert(wk.expired());
     }
}

Bir C arayüzüne sarmalayıcı oluşturmak için özel silicileri kullanma #

SDL2 gibi birçok C arabiriminin kendi silme işlevleri vardır. Bu, akıllı işaretçileri doğrudan kullanamayacağınız anlamına gelir:

std::unique_ptr<SDL_Surface> a; // won't work, UNSAFE!

Bunun yerine, kendi silicinizi tanımlamanız gerekir. Buradaki örnekler, SDL_FreeSurface() işlevi kullanılarak serbest bırakılması gereken SDL_Surface yapısını kullanır, ancak bunlar diğer birçok C arabirimine uyarlanabilir olmalıdır.

Silici, bir işaretçi argümanıyla çağrılabilir olmalıdır ve bu nedenle örn. basit bir işlev işaretçisi:

std::unique_ptr<SDL_Surface, void(*)(SDL_Surface*)> a(pointer, SDL_FreeSurface);

Diğer çağrılabilir nesneler de çalışacaktır, örneğin bir “operatör()” içeren bir sınıf:

struct SurfaceDeleter {
    void operator()(SDL_Surface* surf) {
        SDL_FreeSurface(surf);
    }
};

std::unique_ptr<SDL_Surface, SurfaceDeleter> a(pointer, SurfaceDeleter{}); // safe
std::unique_ptr<SDL_Surface, SurfaceDeleter> b(pointer); // equivalent to the above
                                                         // as the deleter is value-initialized

Bu size yalnızca güvenli, sıfır ek yük (unique_ptr kullanıyorsanız) otomatik bellek yönetimi sağlamakla kalmaz, aynı zamanda istisna güvenliği de elde edersiniz.

Silicinin “unique_ptr” türünün bir parçası olduğunu ve uygulamanın boş temel optimizasyonu kullanabileceğini unutmayın. t=201607261453124463486) boş özel siliciler için boyutta herhangi bir değişiklikten kaçınmak için. Böylece std::unique_ptr<SDL_Surface, SurfaceDeleter> ve std::unique_ptr<SDL_Surface, void(*)(SDL_Surface*)> aynı sorunu benzer şekilde çözerken, eski tür hala yalnızca ikinci tip iki işaretçi tutmak zorundayken bir işaretçi: hem SDL_Surface* hem de işlev işaretçisi! Serbest işlevli özel silicilere sahipken, işlevi boş bir türe sarmak tercih edilir.

Referans sayımının önemli olduğu durumlarda, bir “unique_ptr” yerine bir shared_ptr kullanılabilir. shared_ptr her zaman bir silici saklar, bu, API’lerde faydalı olabilecek silicinin türünü siler. “unique_ptr” yerine “shared_ptr” kullanmanın dezavantajları, siliciyi depolamak için daha yüksek bir bellek maliyeti ve referans sayısını korumak için bir performans maliyeti içerir.

// deleter required at construction time and is part of the type
std::unique_ptr<SDL_Surface, void(*)(SDL_Surface*)> a(pointer, SDL_FreeSurface);

// deleter is only required at construction time, not part of the type
std::shared_ptr<SDL_Surface> b(pointer, SDL_FreeSurface);

“template auto” ile özel silicilerimizi sarmayı daha da kolaylaştırabiliriz:

template <auto DeleteFn>
struct FunctionDeleter {
    template <class T>
    void operator()(T* ptr) {
        DeleteFn(ptr);
    }
};

template <class T, auto DeleteFn>
using unique_ptr_deleter = std::unique_ptr<T, FunctionDeleter<DeleteFn>>;

Yukarıdaki örnek basitçe:

unique_ptr_deleter<SDL_Surface, SDL_FreeSurface> c(pointer);

Burada, “auto"nun amacı, “void” (ör. “SDL_FreeSurface”) döndürsün veya döndürmesin (ör. “fclose”) tüm serbest işlevleri ele almaktır.

son sürüm

Hareket semantiği olmadan benzersiz sahiplik (auto_ptr) #

NOT: std::auto_ptr C++11’de kullanımdan kaldırılmıştır ve C++17’de kaldırılacaktır. Bunu yalnızca C++03 veya daha önceki bir sürümü kullanmaya zorlandıysanız ve dikkatli olmaya istekliyseniz kullanmalısınız. ‘std::auto_ptr’ davranışını değiştirmek için ‘std::move’ ile birlikte unique_ptr’ye taşınması önerilir.

std::unique_ptr olmadan önce, hareket anlambiliminden önce std::auto_ptr vardı. std::auto_ptr benzersiz bir sahiplik sağlar ancak sahipliği kopyalandıktan sonra aktarır.

Tüm akıllı işaretçilerde olduğu gibi, std::auto_ptr kaynakları otomatik olarak temizler (bkz. RAII):

{
    std::auto_ptr<int> p(new int(42));
    std::cout << *p;
} // p is deleted here, no memory leaked

ancak yalnızca bir sahibine izin verir:

std::auto_ptr<X> px = ...;
std::auto_ptr<X> py = px; 
  // px is now empty

Bu, std::auto_ptr kullanarak mülkiyeti istenmeden kaybetme tehlikesine karşı açık ve benzersiz tutmak için kullanılmasına izin verir:

void f(std::auto_ptr<X> ) {
    // assumes ownership of X
    // deletes it at end of scope
};

std::auto_ptr<X> px = ...;
f(px); // f acquires ownership of underlying X
       // px is now empty
px->foo(); // NPE!
// px.~auto_ptr() does NOT delete

Mülkiyet devri, “kopya” yapıcısında gerçekleşti. “auto_ptr"nin kopya oluşturucusu ve kopya atama operatörü, işlenenlerini değiştirilebilmesi için “const” olmayan referansla alır. Örnek bir uygulama olabilir:

template <typename T>
class auto_ptr {
    T* ptr;
public:
    auto_ptr(auto_ptr& rhs)
    : ptr(rhs.release())
    { }

    auto_ptr& operator=(auto_ptr& rhs) {
        reset(rhs.release());
        return *this;
    }

    T* release() {
        T* tmp = ptr;
        ptr = nullptr;
        return tmp;
    }

    void reset(T* tmp = nullptr) {
        if (ptr != tmp) {
            delete ptr;
            ptr = tmp;
        }
    }

    /* other functions ... */
};

Bu, bir nesneyi kopyalamanın size nesnenin iki eşdeğer sürümünü bırakmasını gerektiren kopyalama semantiğini bozar. Herhangi bir kopyalanabilir tür olan T için şunları yazabilmeliyim:

T a = ...;
T b(a);
assert(b == a);

Ancak “auto_ptr” için durum böyle değil. Sonuç olarak, “auto_ptr"leri kapsayıcılara koymak güvenli değildir.

son sürüm

std::shared_ptr işaretçilerini yayınlama #

Argüman olarak iletilen işaretçi ile sahipliğini paylaşan bir işaretçiyi almak için “std::shared_ptr” üzerinde “static_cast”, “const_cast”, “dynamic_cast” ve “reinterpret_cast” öğelerini doğrudan kullanmak mümkün değildir. Bunun yerine, ‘std::static_pointer_cast’, ‘std::const_pointer_cast’, ‘std::dynamic_pointer_cast’ ve ‘std::reinterpret_pointer_cast’ işlevleri kullanılmalıdır:

struct Base { virtual ~Base() noexcept {}; };
struct Derived: Base {};
auto derivedPtr(std::make_shared<Derived>());
auto basePtr(std::static_pointer_cast<Base>(derivedPtr));
auto constBasePtr(std::const_pointer_cast<Base const>(basePtr));
auto constDerivedPtr(std::dynamic_pointer_cast<Derived const>(constBasePtr));

“std::reinterpret_pointer_cast” öğesinin yalnızca N3920 tarafından önerildiği ve benimsendiği için C++11 ve C++14’te mevcut olmadığını unutmayın. Şubat 2014’te. Ancak şu şekilde uygulanabilir:

template <typename To, typename From>
inline std::shared_ptr<To> reinterpret_pointer_cast(
    std::shared_ptr<From> const & ptr) noexcept
{ return std::shared_ptr<To>(ptr, reinterpret_cast<To *>(ptr.get())); }

Akıllı işaretçi yazma: value_ptr #

“value_ptr”, bir değer gibi davranan akıllı bir işaretçidir. Kopyalandığında içeriğini kopyalar. Oluşturulduğunda, içeriğini oluşturur.

// Like std::default_delete:
template<class T>
struct default_copier {
  // a copier must handle a null T const* in and return null:
  T* operator()(T const* tin)const {
    if (!tin) return nullptr;
    return new T(*tin);
  }
  void operator()(void* dest, T const* tin)const {
    if (!tin) return;
    return new(dest) T(*tin);
  }
};
// tag class to handle empty case:
struct empty_ptr_t {};
constexpr empty_ptr_t empty_ptr{};
// the value pointer type itself:
template<class T, class Copier=default_copier<T>, class Deleter=std::default_delete<T>,
  class Base=std::unique_ptr<T, Deleter>
>
struct value_ptr:Base, private Copier {
  using copier_type=Copier;
  // also typedefs from unique_ptr

  using Base::Base;

  value_ptr( T const& t ):
    Base( std::make_unique<T>(t) ),
    Copier()
  {}
  value_ptr( T && t ):
    Base( std::make_unique<T>(std::move(t)) ),
    Copier()
  {}
  // almost-never-empty:
      value_ptr():
    Base( std::make_unique<T>() ),
    Copier()
  {}
  value_ptr( empty_ptr_t ) {}

  value_ptr( Base b, Copier c={} ):
    Base(std::move(b)),
    Copier(std::move(c))
  {}

  Copier const& get_copier() const {
    return *this;
  }

  value_ptr clone() const {
    return {
      Base(
        get_copier()(this->get()),
        this->get_deleter()
      ),
      get_copier()
    };
  }
  value_ptr(value_ptr&&)=default;
  value_ptr& operator=(value_ptr&&)=default;

  value_ptr(value_ptr const& o):value_ptr(o.clone()) {}
  value_ptr& operator=(value_ptr const&o) {
    if (o && *this) {
      // if we are both non-null, assign contents:
      **this = *o;
    } else {
      // otherwise, assign a clone (which could itself be null):
      *this = o.clone();
    }
    return *this;
  }
  value_ptr& operator=( T const& t ) {
    if (*this) {
      **this = t;
    } else {
      *this = value_ptr(t);
    }
    return *this;
  }
  value_ptr& operator=( T && t ) {
    if (*this) {
      **this = std::move(t);
    } else {
      *this = value_ptr(std::move(t));
    }
    return *this;
  }
  T& get() { return **this; }
  T const& get() const { return **this; }
  T* get_pointer() {
    if (!*this) return nullptr;
    return std::addressof(get());
  }
  T const* get_pointer() const {
    if (!*this) return nullptr;
    return std::addressof(get());
  }
  // operator-> from unique_ptr
};
template<class T, class...Args>
value_ptr<T> make_value_ptr( Args&&... args ) {
  return {std::make_unique<T>(std::forward<Args>(args)...)};
}

Bu özel değer_ptr, yalnızca onu “boş_ptr_t” ile oluşturursanız veya ondan hareket ederseniz boştur. Bunun bir “benzersiz_ptr” olduğu gerçeğini ortaya çıkarır, bu nedenle “açık operatör bool() const” üzerinde çalışır. .get() bir referans döndürecek şekilde değiştirildi (neredeyse hiçbir zaman boş olmadığından) ve .get_pointer() bunun yerine bir işaretçi döndürür.

Bu akıllı işaretçi, değer semantiği istediğimiz ancak aynı zamanda ‘pImpl’ içeriğini uygulama dosyasının dışında göstermek istemediğimiz ‘pImpl’ durumları için faydalı olabilir.

Varsayılan olmayan bir “Kopyalayıcı” ile, türetilmiş örneklerinin nasıl üretileceğini bilen ve bunları değer türlerine dönüştüren sanal temel sınıfları bile işleyebilir.

Buna atıfta bulunan bir paylaşılan_ptr alma #

“enable_shared_from_this”, “this” için geçerli bir “shared_ptr” örneğini almanızı sağlar.

Sınıfınızı “enable_shared_from_this” sınıf şablonundan türeterek, “shared_ptr” örneğini “this"e döndüren bir “shared_from_this” yöntemini miras alırsınız.

**Öncelikle nesnenin bir “shared_ptr” olarak oluşturulması gerektiğine dikkat edin:

#include <memory>
class A: public enable_shared_from_this<A> {
};
A* ap1 =new A();
shared_ptr<A> ap2(ap1); // First prepare a shared pointer to the object and hold it!
// Then get a shared pointer to the object from the object itself
shared_ptr<A> ap3 = ap1->shared_from_this(); 
int c3 =ap3.use_count(); // =2: pointing to the same object

Not(2) yapıcı içinde “enable_shared_from_this” öğesini çağıramazsınız.

#include <memory> // enable_shared_from_this

class Widget : public std::enable_shared_from_this< Widget >
{
public:
    void DoSomething()
    {
        std::shared_ptr< Widget > self = shared_from_this();
        someEvent -> Register( self );
    }
private:
    ...
};

int main()
{
    ...
    auto w = std::make_shared< Widget >();
    w -> DoSomething();
    ...
}

Yerel otomatik nesne veya genel nesne gibi bir “shared_ptr"ye ait olmayan bir nesnede “shared_from_this()” kullanırsanız, davranış tanımsızdır. C++ 17’den beri bunun yerine ‘std::bad_alloc’ atar.

Bir yapıcıdan “shared_from_this()” kullanmak, onu “shared_ptr"ye ait olmayan bir nesne üzerinde kullanmaya eşdeğerdir, çünkü nesneler, yapıcı geri döndükten sonra “shared_ptr” tarafından ele geçirilir.
