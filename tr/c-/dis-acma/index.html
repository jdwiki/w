Sözdizimi #

  • iplik()
  • iplik(iplik&& diğer)
  • açık iş parçacığı(Function&& func, Args&&… args)

Parametreler #

| parametre | Ayrıntılar | | ——— | ————————————————– —————- | | ‘diğer’ | ‘other’ın sahipliğini alır, ‘other’ artık başlığın sahibi değildir | | “işlev” | Ayrı bir iş parçacığında çağırma işlevi | | “arg” | “Func” için Argümanlar |

Bazı notlar:

  • İki std::thread nesnesi aynı iş parçacığını asla temsil edemez.
  • Bir “std::thread” nesnesi, herhangi bir iş parçacığını temsil etmediği bir durumda olabilir (yani bir hamleden sonra, “join” çağrısından sonra vb.).

std::thread oluşturma #

C++‘da iş parçacıkları std::thread sınıfı kullanılarak oluşturulur. Bir iş parçacığı, ayrı bir yürütme akışıdır; bu, bir yardımcının bir görevi yerine getirirken siz aynı anda başka bir görevi yerine getirmesine benzer. İş parçacığındaki tüm kod yürütüldüğünde, terminates. Bir iş parçacığı oluştururken, üzerinde yürütülecek bir şey iletmeniz gerekir. Bir iş parçacığına iletebileceğiniz birkaç şey:

  • Ücretsiz fonksiyonlar
  • Üye işlevleri
  • Functor nesneleri
  • Lambda ifadeleri

Ücretsiz işlev örneği - ayrı bir iş parçacığında bir işlevi yürütür (Canlı Örnek):

#include <iostream>
#include <thread>
 
void foo(int a)
{
    std::cout << a << '\n';
}
 
int main()
{
    // Create and execute the thread
    std::thread thread(foo, 10); // foo is the function to execute, 10 is the
                                 // argument to pass to it
 
    // Keep going; the thread is executed separately
 
    // Wait for the thread to finish; we stay here until it is done
    thread.join();
 
    return 0;
}

Üye işlevi örneği - ayrı bir ileti dizisinde bir üye işlevi yürütür (Canlı Örnek):

#include <iostream>
#include <thread>
 
class Bar
{
public:
    void foo(int a)
    {
        std::cout << a << '\n';
    }
};
 
int main()
{
    Bar bar;
    
    // Create and execute the thread
    std::thread thread(&Bar::foo, &bar, 10); // Pass 10 to member function
 
    // The member function will be executed in a separate thread
 
    // Wait for the thread to finish, this is a blocking operation
    thread.join();
 
    return 0;
}

Functor nesne örneği (Canlı Örnek):

#include <iostream>
#include <thread>
 
class Bar
{
public:
    void operator()(int a)
    {
        std::cout << a << '\n';
    }
};
 
int main()
{
    Bar bar;
    
    // Create and execute the thread
    std::thread thread(bar, 10); // Pass 10 to functor object
 
    // The functor object will be executed in a separate thread
 
    // Wait for the thread to finish, this is a blocking operation
    thread.join();
 
    return 0;
}

Lambda ifadesi örneği (Canlı Örnek):

#include <iostream>
#include <thread>
 
int main()
{
    auto lambda = [](int a) { std::cout << a << '\n'; };

    // Create and execute the thread
    std::thread thread(lambda, 10); // Pass 10 to the lambda expression
 
    // The lambda expression will be executed in a separate thread
 
    // Wait for the thread to finish, this is a blocking operation
    thread.join();
 
    return 0;
}

Bir konuya referans gönderme #

‘std::thread’ onları kopyalayacak/taşıyacağından, bir referansı (veya ‘const’ referansını) doğrudan bir iş parçacığına iletemezsiniz. Bunun yerine, ‘std::reference_wrapper’ kullanın:

void foo(int& b)
{
    b = 10;
}

int a = 1;
std::thread thread{ foo, std::ref(a) }; //'a' is now really passed as reference

thread.join();
std::cout << a << '\n'; //Outputs 10

void bar(const ComplexObject& co)
{
    co.doCalculations();
}

ComplexObject object;
std::thread thread{ bar, std::cref(object) }; //'object' is passed as const&

thread.join();
std::cout << object.getResult() << '\n'; //Outputs the result

std::thread yerine std::async kullanma #

std::async de iş parçacığı oluşturabilir. ‘std::thread’ ile karşılaştırıldığında, daha az güçlü olarak kabul edilir, ancak yalnızca bir işlevi eşzamansız olarak çalıştırmak istediğinizde kullanımı daha kolaydır.

Eşzamansız olarak bir işlevi çağırma #

#include <future>
#include <iostream>

unsigned int square(unsigned int i){
    return i*i;
}

int main() {
    auto f = std::async(std::launch::async, square, 8);
    std::cout << "square currently running\n"; //do something while square is running
    std::cout << "result is " << f.get() << '\n'; //getting the result from square
}

Ortak tuzaklar #

  • ‘std::async’, işlev tarafından hesaplanacak dönüş değerini tutan bir ‘std::future’ döndürür. Bu ‘gelecek’ yok edildiğinde, iş parçacığı tamamlanana kadar bekler ve kodunuzu etkin bir şekilde tek iş parçacıklı hale getirir. Dönüş değerine ihtiyacınız olmadığında bu kolayca gözden kaçabilir:

    std::async(std::launch::async, square, 5);
//thread already completed at this point, because the returning future got destroyed

  • std::async bir başlatma ilkesi olmadan çalışır, bu nedenle std::async(square, 5); derlenir. Bunu yaptığınızda, sistem bir iş parçacığı oluşturmak isteyip istemediğine karar verir. Buradaki fikir, sistemin verimli bir şekilde çalışabileceğinden daha fazla iş parçacığı çalıştırmadığı sürece bir iş parçacığı oluşturmayı seçmesiydi. Ne yazık ki, uygulamalar genellikle bu durumda bir iş parçacığı oluşturmamayı seçerler, bu nedenle bu davranışı, sistemi bir iş parçacığı oluşturmaya zorlayan ‘std::launch::async’ ile geçersiz kılmanız gerekir.

  • Yarış koşullarına dikkat edin.

Vadeli İşlemler ve Vaatler ile ilgili zaman uyumsuz hakkında daha fazla bilgi

Bir iş parçacığının her zaman birleştirilmesini sağlama #

‘std::thread’ için yıkıcı çağrıldığında, ‘join()’ veya ‘detach()’ zorunlu için bir çağrı yapılmış olmalıdır. Bir iş parçacığı birleştirilmemiş veya ayrılmamışsa, varsayılan olarak std::terminate çağrılır. RAII kullanarak, bunu başarmak genellikle yeterince basittir:

class thread_joiner
{
public:

    thread_joiner(std::thread t)
        : t_(std::move(t))
    { }

    ~thread_joiner()
    {
        if(t_.joinable()) {
            t_.join();
        }
    }

private:

    std::thread t_;
}

Bu daha sonra şöyle kullanılır:

 void perform_work()
 {
     // Perform some work
 }

 void t()
 {
     thread_joiner j{std::thread(perform_work)};
     // Do some other calculations while thread is running
 } // Thread is automatically joined here

Bu ayrıca istisna güvenliği sağlar; iş parçacığımızı normal bir şekilde oluştursaydık ve diğer hesaplamaları yaparken t() içinde yapılan iş bir istisna atmış olsaydı, join() asla iş parçacığımızda çağrılmayacaktı ve işlemimiz sonlandırılacaktı.

Temel Senkronizasyon #

İş parçacığı senkronizasyonu, diğer senkronizasyon ilkellerinin yanı sıra muteksler kullanılarak gerçekleştirilebilir. Standart kitaplık tarafından sağlanan birkaç mutex türü vardır, ancak en basiti ‘std::mutex’tir. Bir mutex’i kilitlemek için üzerinde bir kilit oluşturursunuz. En basit kilit türü std::lock_guarddır:

std::mutex m;
void worker() {
    std::lock_guard<std::mutex> guard(m); // Acquires a lock on the mutex
    // Synchronized code here
} // the mutex is automatically released when guard goes out of scope

‘std::lock_guard’ ile muteks, kilit nesnesinin tüm ömrü boyunca kilitlenir. Kilitleme bölgelerini manuel olarak kontrol etmeniz gereken durumlarda bunun yerine std::unique_lock kullanın:

std::mutex m;
void worker() {
    // by default, constructing a unique_lock from a mutex will lock the mutex
    // by passing the std::defer_lock as a second argument, we
    // can construct the guard in an unlocked state instead and
    // manually lock later.
    std::unique_lock<std::mutex> guard(m, std::defer_lock);
    // the mutex is not locked yet!
    guard.lock();
    // critical section
    guard.unlock();
    // mutex is again released
}

Basit bir iş parçacığı havuzu oluşturun #

C++ 11 iş parçacığı ilkelleri hala nispeten düşük seviyededir. Bir iş parçacığı havuzu gibi daha yüksek seviyeli bir yapı yazmak için kullanılabilirler:

struct tasks {
  // the mutex, condition variable and deque form a single
  // thread-safe triggered queue of tasks:
  std::mutex m;
  std::condition_variable v;
  // note that a packaged_task<void> can store a packaged_task<R>:
  std::deque<std::packaged_task<void()>> work;

  // this holds futures representing the worker threads being done:
  std::vector<std::future<void>> finished;

  // queue( lambda ) will enqueue the lambda into the tasks for the threads
  // to use.  A future of the type the lambda returns is given to let you get
  // the result out.
  template<class F, class R=std::result_of_t<F&()>>
  std::future<R> queue(F&& f) {
    // wrap the function object into a packaged task, splitting
    // execution from the return value:
    std::packaged_task<R()> p(std::forward<F>(f));

    auto r=p.get_future(); // get the return value before we hand off the task
    {
      std::unique_lock<std::mutex> l(m);
      work.emplace_back(std::move(p)); // store the task<R()> as a task<void()>
    }
    v.notify_one(); // wake a thread to work on the task

    return r; // return the future result of the task
  }

  // start N threads in the thread pool.
  void start(std::size_t N=1){
    for (std::size_t i = 0; i < N; ++i)
    {
      // each thread is a std::async running this->thread_task():
      finished.push_back(
        std::async(
          std::launch::async,
          [this]{ thread_task(); }
        )
      );
    }
  }
  // abort() cancels all non-started tasks, and tells every working thread
  // stop running, and waits for them to finish up.
  void abort() {
    cancel_pending();
    finish();
  }
  // cancel_pending() merely cancels all non-started tasks:
  void cancel_pending() {
    std::unique_lock<std::mutex> l(m);
    work.clear();
  }
  // finish enques a "stop the thread" message for every thread, then waits for them:
  void finish() {
    {
      std::unique_lock<std::mutex> l(m);
      for(auto&&unused:finished){
        work.push_back({});
      }
    }
    v.notify_all();
    finished.clear();
  }
  ~tasks() {
    finish();
  }
private:
  // the work that a worker thread does:
  void thread_task() {
    while(true){
      // pop a task off the queue:
      std::packaged_task<void()> f;
      {
        // usual thread-safe queue code:
        std::unique_lock<std::mutex> l(m);
        if (work.empty()){
          v.wait(l,[&]{return !work.empty();});
        }
        f = std::move(work.front());
        work.pop_front();
      }
      // if the task is invalid, it means we are asked to abort:
      if (!f.valid()) return;
      // otherwise, run the task:
      f();
    }
  }
};

“tasks.start(10)” (10 ileti dizisini başlatan) komutunu çalıştırarak ileti dizileri oluşturursunuz.

Geçerli iş parçacığındaki işlemler #

std::this_thread, çağrıldığı fonksiyondan mevcut iş parçacığında ilginç şeyler yapmak için işlevleri olan bir “ad alanı"dır.

| fonksiyon | Açıklama | | ————–| ————————————————– ——- | | get_id | İş parçacığının kimliğini döndürür | | uyku_için | Belirli bir süre uyur | | sleep_until | Belirli bir zamana kadar uyur | | “verim” | Çalışan dizileri yeniden planlayın, diğer dizilere öncelik verin |

std::this_thread::get_id kullanarak mevcut iş parçacığı kimliğini alma:

void foo()
{
    //Print this threads id
    std::cout << std::this_thread::get_id() << '\n';
}

std::thread thread{ foo };
thread.join(); //'threads' id has now been printed, should be something like 12556

foo(); //The id of the main thread is printed, should be something like 2420

std::this_thread::sleep_for kullanarak 3 saniye uyumak:

void foo()
{
    std::this_thread::sleep_for(std::chrono::seconds(3));
}

std::thread thread{ foo };
foo.join();

std::cout << "Waited for 3 seconds!\n";

std::this_thread::sleep_until kullanarak gelecekte 3 saate kadar uyumak:

void foo()
{
    std::this_thread::sleep_until(std::chrono::system_clock::now() + std::chrono::hours(3));
}

std::thread thread{ foo };
thread.join();

std::cout << "We are now located 3 hours after the thread has been called\n";

std::this_thread::yield kullanarak diğer konuların öncelik almasına izin vermek:

void foo(int a)
{
    for (int i = 0; i < al ++i)
        std::this_thread::yield(); //Now other threads take priority, because this thread
                                   //isn't doing anything important

    std::cout << "Hello World!\n";
}

std::thread thread{ foo, 10 };
thread.join();

Koşul Değişkenlerini Kullanma #

Koşul değişkeni, iş parçacıkları arasındaki iletişimi düzenlemek için bir muteks ile birlikte kullanılan bir ilkeldir. Bunu başarmanın ne özel ne de en etkili yolu olmasa da, kalıba aşina olanlar için en basit yollardan biri olabilir.

Bir ‘std::unique_lockstd::mutex’ ile bir ‘std::condition_variable’ üzerinde bekler. Bu, kodun, edinmeye devam edip etmemeye karar vermeden önce paylaşılan durumu güvenli bir şekilde incelemesini sağlar.

Aşağıda, işleri ilginç kılmak için “std::thread”, “std::condition_variable”, “std::mutex” ve birkaç tane daha kullanan bir üretici-tüketici taslağı bulunmaktadır.

#include <condition_variable>
#include <cstddef>
#include <iostream>
#include <mutex>
#include <queue>
#include <random>
#include <thread>


int main()
{
    std::condition_variable cond;
    std::mutex mtx;
    std::queue<int> intq;
    bool stopped = false;

    std::thread producer{[&]()
    {
        // Prepare a random number generator.
        // Our producer will simply push random numbers to intq.
        //
        std::default_random_engine gen{};
        std::uniform_int_distribution<int> dist{};

        std::size_t count = 4006;    
        while(count--)
        {    
            // Always lock before changing
            // state guarded by a mutex and
            // condition_variable (a.k.a. "condvar").
            std::lock_guard<std::mutex> L{mtx};

            // Push a random int into the queue
            intq.push(dist(gen));

            // Tell the consumer it has an int
            cond.notify_one();
        }

        // All done.
        // Acquire the lock, set the stopped flag,
        // then inform the consumer.
        std::lock_guard<std::mutex> L{mtx};

        std::cout << "Producer is done!" << std::endl;

        stopped = true;
        cond.notify_one();
    }};

    std::thread consumer{[&]()
    {
        do{
            std::unique_lock<std::mutex> L{mtx};
            cond.wait(L,[&]()
            {
                // Acquire the lock only if
                // we've stopped or the queue
                // isn't empty
                return stopped || ! intq.empty();
            });

            // We own the mutex here; pop the queue
            // until it empties out.

            while( ! intq.empty())
            {
                const auto val = intq.front();
                intq.pop();

                std::cout << "Consumer popped: " << val << std::endl;
            }

            if(stopped){
                // producer has signaled a stop
                std::cout << "Consumer is done!" << std::endl;
                break;
            }

        }while(true);
    }};

    consumer.join();
    producer.join();
    
    std::cout << "Example Completed!" << std::endl;

    return 0;
}

İplik işlemleri #

Bir iş parçacığı başlattığınızda, bitene kadar yürütülür.

Çoğu zaman, bir noktada (muhtemelen - iş parçacığı zaten yapılmış olabilir) iş parçacığının bitmesini beklemeniz gerekir, çünkü örneğin sonucu kullanmak istersiniz.

int n;
std::thread thread{ calculateSomething, std::ref(n) };

//Doing some other stuff

//We need 'n' now!
//Wait for the thread to finish - if it is not already done
thread.join();

//Now 'n' has the result of the calculation done in the seperate thread
std::cout << n << '\n';

Ayrıca ipliği “ayırabilirsiniz”, serbestçe çalışmasına izin verebilirsiniz:

std::thread thread{ doSomething };

//Detaching the thread, we don't need it anymore (for whatever reason)
thread.detach();

//The thread will terminate when it is done, or when the main thread returns

Konu-yerel depolama #

İş parçacığı yerel depolaması, “thread_local” anahtar sözcüğü kullanılarak oluşturulabilir. ’thread_local’ belirteci ile bildirilen bir değişkenin iş parçacığı depolama süresi olduğu söylenir.

  • Bir programdaki her iş parçacığı, her iş parçacığı yerel değişkeninin kendi kopyasına sahiptir.
  • İşlev (yerel) kapsamına sahip bir iş parçacığı yerel değişkeni, kontrol tanımından ilk kez geçtiğinde başlatılacaktır. Böyle bir değişken, “harici” olarak bildirilmedikçe, dolaylı olarak statiktir.
  • Ad alanı veya sınıf (yerel olmayan) kapsamı olan bir iş parçacığı yerel değişkeni, iş parçacığı başlatmanın bir parçası olarak başlatılacaktır.
  • İş parçacığı yerel değişkenleri, iş parçacığı sonlandırıldığında yok edilir.
  • Bir sınıfın üyesi yalnızca statik ise iş parçacığı yerel olabilir. Bu nedenle, (iş parçacığı, örnek) çifti başına bir kopya yerine, iş parçacığı başına bu değişkenin bir kopyası olacaktır.

Örnek:

void debug_counter() {
    thread_local int count = 0;
    Logger::log("This function has been called %d times by this thread", ++count);
}

İş parçacığı nesnelerini yeniden atama #

Boş thread nesneleri oluşturabilir ve onlara daha sonra iş atayabiliriz.

Bir iş parçacığı nesnesini başka bir etkin, ‘birleştirilebilir’ iş parçacığına atarsak, iş parçacığı değiştirilmeden önce ‘std::terminate’ otomatik olarak çağrılır.

#include <thread>

void foo()
{
    std::this_thread::sleep_for(std::chrono::seconds(3));
}
//create 100 thread objects that do nothing
std::thread executors[100];

// Some code

// I want to create some threads now

for (int i = 0;i < 100;i++)
{
    // If this object doesn't have a thread assigned
    if (!executors[i].joinable())
         executors[i] = std::thread(foo);
}
