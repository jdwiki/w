Derin kopyalama ve taşıma desteği #

Bir tür değer semantiğine sahip olmak istiyorsa ve dinamik olarak ayrılmış nesneleri depolaması gerekiyorsa, kopyalama işlemlerinde türün bu nesnelerin yeni kopyalarını ayırması gerekecektir. Bunu kopya ataması için de yapmalıdır.

Bu tür kopyalamaya “derin kopya” denir. Aksi takdirde referans semantiği olan şeyi etkili bir şekilde alır ve onu değer semantiğine dönüştürür:

struct Inner {int i;};

const int NUM_INNER = 5;
class Value
{
private:
  Inner *array_; //Normally has reference semantics.

public:
  Value() : array_(new Inner[NUM_INNER]){}

  ~Value() {delete[] array_;}

  Value(const Value &val) : array_(new Inner[NUM_INNER])
  {
    for(int i = 0; i < NUM_INNER; ++i)
      array_[i] = val.array_[i];
  }

  Value &operator=(const Value &val)
  {
    for(int i = 0; i < NUM_INNER; ++i)
      array_[i] = val.array_[i];
    return *this;
  }
};

Taşıma semantiği, “Değer” gibi bir türün başvurulan verileri gerçekten kopyalamaktan kaçınmasına izin verir. Kullanıcı değeri bir hareketi tetikleyecek şekilde kullanırsa, nesneden “kopyalanan”, başvurduğu verilerden boş bırakılabilir:

struct Inner {int i;};

constexpr auto NUM_INNER = 5;
class Value
{
private:
  Inner *array_; //Normally has reference semantics.

public:
  Value() : array_(new Inner[NUM_INNER]){}

  //OK to delete even if nullptr
  ~Value() {delete[] array_;}

  Value(const Value &val) : array_(new Inner[NUM_INNER])
  {
    for(int i = 0; i < NUM_INNER; ++i)
      array_[i] = val.array_[i];
  }

  Value &operator=(const Value &val)
  {
    for(int i = 0; i < NUM_INNER; ++i)
      array_[i] = val.array_[i];
    return *this;
  }

  //Movement means no memory allocation.
  //Cannot throw exceptions.
  Value(Value &&val) noexcept : array_(val.array_)
  {
    //We've stolen the old value.
    val.array_ = nullptr;
  }

  //Cannot throw exceptions.
  Value &operator=(Value &&val) noexcept
  {
    //Clever trick. Since `val` is going to be destroyed soon anyway,
    //we swap his data with ours. His destructor will destroy our data.
    std::swap(array_, val.array_);
  }
};

Aslında, nesnenin hareket etmesine izin verirken derin kopyaları yasaklamak istiyorsak, böyle bir türü kopyalanamaz bile yapabiliriz.

struct Inner {int i;};

constexpr auto NUM_INNER = 5;
class Value
{
private:
  Inner *array_; //Normally has reference semantics.

public:
  Value() : array_(new Inner[NUM_INNER]){}

  //OK to delete even if nullptr
  ~Value() {delete[] array_;}

  Value(const Value &val) = delete;
  Value &operator=(const Value &val) = delete;

  //Movement means no memory allocation.
  //Cannot throw exceptions.
  Value(Value &&val) noexcept : array_(val.array_)
  {
    //We've stolen the old value.
    val.array_ = nullptr;
  }

  //Cannot throw exceptions.
  Value &operator=(Value &&val) noexcept
  {
    //Clever trick. Since `val` is going to be destroyed soon anyway,
    //we swap his data with ours. His destructor will destroy our data.
    std::swap(array_, val.array_);
  }
};

Hatta unique_ptr kullanarak Sıfır Kuralını bile uygulayabiliriz:

struct Inner {int i;};

constexpr auto NUM_INNER = 5;
class Value
{
private:
  unique_ptr<Inner []>array_; //Move-only type.

public:
  Value() : array_(new Inner[NUM_INNER]){}

  //No need to explicitly delete. Or even declare.
  ~Value() = default; {delete[] array_;}

  //No need to explicitly delete. Or even declare.
  Value(const Value &val) = default;
  Value &operator=(const Value &val) = default;

  //Will perform an element-wise move.
  Value(Value &&val) noexcept = default;

  //Will perform an element-wise move.
  Value &operator=(Value &&val) noexcept = default;
};

Tanımlar #

Nesnenin gözlemlenebilir durumu, o türün diğer tüm nesnelerinden işlevsel olarak farklıysa, bir türün değer semantiği vardır. Bunun anlamı, bir nesneyi kopyalarsanız, yeni bir nesneniz olur ve yeni nesnede yapılan değişiklikler hiçbir şekilde eski nesneden görünmez.

Çoğu temel C++ türünün değer semantiği vardır:

int i = 5;
int j = i; //Copied
j += 20;
std::cout << i; //Prints 5; i is unaffected by changes to j.

Çoğu standart kitaplık tanımlı türün de değer semantiği vardır:

std::vector<int> v1(5, 12); //array of 5 values, 12 in each.
std::vector<int> v2 = v1; //Copies the vector.
v2[3] = 6; v2[4] = 9;
std::cout << v1[3] << " " << v1[4]; //Writes "12 12", since v1 is unchanged.

Bir türün referans semantiğine sahip olduğu söylenir, eğer o türden bir örnek gözlemlenebilir durumunu başka bir nesneyle (kendisinin dışında) paylaşabiliyorsa, öyle ki bir nesneyi manipüle etmek durumun başka bir nesne içinde değişmesine neden olur.

C++ işaretçileri, işaret ettikleri nesneye göre değer semantiğine sahiptir, ancak işaret ettikleri nesnenin durumuna göre referans semantiğine sahiptir:

int *pi = new int(4);
int *pi2 = pi;
pi = new int(16);
assert(pi2 != pi); //Will always pass.

int *pj = pi;
*pj += 5;
std::cout << *pi; //Writes 9, since `pi` and `pj` reference the same object.

C++ referanslarının da referans semantiği vardır.
