Kasım 2014’te, C++ Standardizasyon Komitesi, doğrudan başlatma sözdizimini kullanan otomatik ve destekli başlatıcılar için özel tür kesinti kuralını ortadan kaldıran N3922 önerisini kabul etti. Bu, C++ standardının bir parçası değildir, ancak bazı derleyiciler tarafından uygulanmıştır.

Yapıcılar için şablon parametre kesintisi #

C++ 17’den önce, şablon kesintisi bir kurucuda sizin için sınıf türünü çıkaramaz. Açıkça belirtilmelidir. Bununla birlikte, bazen, bu türler çok hantal olabilir veya (lambdalar durumunda) adlandırılması imkansız olabilir, bu nedenle tür fabrikalarının çoğalmasını sağladık (‘make_pair()’, ‘make_tuple()’, ‘back_inserter()’, vb.).

Bu artık gerekli değil:

std::pair p(2, 4.5);     // std::pair<int, double>
std::tuple t(4, 3, 2.5); // std::tuple<int, int, double>
std::copy_n(vi1.begin(), 3,
    std::back_insert_iterator(vi2)); // constructs a back_insert_iterator<std::vector<int>>
std::lock_guard lk(mtx); // std::lock_guard<decltype(mtx)>

Yapıcıların sınıf şablonu parametrelerini çıkardığı kabul edilir, ancak bazı durumlarda bu yetersizdir ve açık kesinti kılavuzları sağlayabiliriz:

template <class Iter>
vector(Iter, Iter) -> vector<typename iterator_traits<Iter>::value_type>

int array[] = {1, 2, 3};
std::vector v(std::begin(array), std::end(array)); // deduces std::vector<int>
son sürüm

Otomatik Tip Kesintisi #

‘auto’ anahtar kelime kullanılarak tür kesintisi, Şablon Türü Kesintisi ile hemen hemen aynı şekilde çalışır. Aşağıda birkaç örnek verilmiştir:

auto x = 27;           // (x is neither a pointer nor a reference), x's type is int
const auto cx = x;     // (cx is neither a pointer nor a reference), cs's type is const int
const auto& rx = x;    // (rx is a non-universal reference), rx's type is a reference to a const int

auto&& uref1 = x;      // x is int and lvalue, so uref1's type is int&
auto&& uref2 = cx;     // cx is const int and lvalue, so uref2's type is const int &
auto&& uref3 = 27;     // 27 is an int and rvalue, so uref3's type is int&&

Farklılıklar aşağıda özetlenmiştir:

auto x1 = 27;          // type is int, value is 27
auto x2(27);           // type is int, value is 27
auto x3 = { 27 };      // type is std::initializer_list<int>, value is { 27 }
auto x4{ 27 };         // type is std::initializer_list<int>, value is { 27 }
                       // in some compilers type may be deduced as an int with a 
                       // value of 27. See remarks for more information.
auto x5 = { 1, 2.0 }   // error! can't deduce T for std::initializer_list<t>

Görebileceğiniz gibi, çaprazlı başlatıcılar kullanıyorsanız, auto, std::initializer_list<T> türünde bir değişken oluşturmaya zorlanır. T değerini çıkaramazsa, kod reddedilir.

Bir işlevin dönüş türü olarak “auto” kullanıldığında, işlevin bir sondaki dönüş türü olduğunu belirtir. ).

auto f() -> int {
    return 42;
}
son sürüm

C++14, C++11’de izin verilen otomatik kullanımlara ek olarak aşağıdakilere izin verir:

  1. Sonda dönüş türü olmayan bir işlevin dönüş türü olarak kullanıldığında, işlevin dönüş türünün, varsa işlevin gövdesindeki dönüş ifadelerinden çıkarılması gerektiğini belirtir.

    // f returns int:
auto f() { return 42; }
// g returns void:
auto g() { std::cout << "hello, world!\n"; }

  2. Bir lambdanın parametre tipinde kullanıldığında, lambdayı bir [jenerik lambda] olarak tanımlar(https://www.wikiod.com/tr/docs/c%2b%2b/572/lambdas/1969/generic-lambdas).

    auto triple = [](auto x) { return 3*x; };
const auto x = triple(42); // x is a const int with value 126

‘decltype(auto)’ özel biçimi, ‘auto’ yerine ‘decltype’ tür kesinti kurallarını kullanarak bir tür çıkarır.

int* p = new int(42);
auto x = *p;           // x has type int
decltype(auto) y = *p; // y is a reference to *p
son sürüm

C++03 ve önceki sürümlerde, “auto” anahtar sözcüğünün [depolama sınıfı belirteci] olarak tamamen farklı bir anlamı vardı (https://www.wikiod.com/tr/docs/c%2b%2b/9225/storage-class-specifiers/ 28629/auto) C’den miras alındı.

Şablon Türü Kesintisi #

Şablon Genel Sözdizimi

template<typename T>
void f(ParamType param);

f(expr);

Durum 1: “ParamType” bir Referans veya İşaretçidir, ancak Evrensel veya İleri Referans değildir. Bu durumda tip indirimi bu şekilde çalışır. Derleyici, eğer ’expr’ içinde mevcutsa, referans kısmını yok sayar. Derleyici daha sonra ‘T’yi belirlemek için ’expr’ türünü ‘ParamType’ ile eşleştirir.

template<typename T>
void f(T& param);      //param is a reference

int x = 27;            // x is an int
const int cx = x;      // cx is a const int
const int& rx = x;     // rx is a reference to x as a const int

f(x);                  // T is int, param's type is int&
f(cx);                 // T is const int, param's type is const int&
f(rx);                 // T is const int, param's type is const int&

Durum 2: “ParamType” bir Evrensel Referans veya İleri Referanstır. Bu durumda, “ifade” bir değer ise, tip kesinti 1. durumdakiyle aynıdır. “expr” bir değerse, hem “T” hem de “ParamType”, değer referansları olarak çıkarılır.

template<typename T>
void f(T&& param);     // param is a universal reference

int x = 27;            // x is an int
const int cx = x;      // cx is a const int
const int& rx = x;     // rx is a reference to x as a const int

f(x);                  // x is lvalue, so T is int&, param's type is also int&
f(cx);                 // cx is lvalue, so T is const int&, param's type is also const int&
f(rx);                 // rx is lvalue, so T is const int&, param's type is also const int&
f(27);                 // 27 is rvalue, so T is int, param's type is therefore int&&

Durum 3: “ParamType” Ne İşaretçi ne de Referanstır. Eğer “ifade” bir referans ise referans kısmı yok sayılır. Eğer “ifade” const ise bu da yoksayılır. Uçucuysa, T’nin türü çıkarılırken de göz ardı edilir.

template<typename T>
void f(T param);       // param is now passed by value

int x = 27;            // x is an int
const int cx = x;      // cx is a const int
const int& rx = x;     // rx is a reference to x as a const int

f(x);                  // T's and param's types are both int
f(cx);                 // T's and param's types are again both int
f(rx);                 // T's and param's types are still both int
