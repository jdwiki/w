Sınıf türleri #

“Sınıf” ile, “sınıf” veya “yapı” anahtar sözcüğü kullanılarak tanımlanmış bir türü kastediyoruz (ancak “enum struct” veya “enum class” değil).

  • Boş bir sınıf bile en az bir bayt depolama alanı kaplar; bu nedenle tamamen dolgudan oluşacaktır. Bu, eğer “p” boş bir sınıfın nesnesini gösteriyorsa, “p + 1"in ayrı bir adres olmasını ve ayrı bir nesneyi göstermesini sağlar. Ancak, temel sınıf olarak kullanıldığında boş bir sınıfın 0 boyutuna sahip olması mümkündür. Bkz. boş taban optimizasyonu.

    class Empty_1 {};                               // sizeof(Empty_1)       == 1
class Empty_2 {};                               // sizeof(Empty_2)       == 1
class Derived : Empty_1 {};                     // sizeof(Derived)       == 1
class DoubleDerived : Empty_1, Empty_2 {};      // sizeof(DoubleDerived) == 1
class Holder { Empty_1 e; };                    // sizeof(Holder)        == 1
class DoubleHolder { Empty_1 e1; Empty_2 e2; }; // sizeof(DoubleHolder)  == 2
class DerivedHolder : Empty_1 { Empty_1 e; };   // sizeof(DerivedHolder) == 2

  • Bir sınıf türünün nesne temsili, temel sınıfın ve statik olmayan üye türlerinin nesne temsillerini içerir. Bu nedenle, örneğin, aşağıdaki sınıfta:

    struct S {
    int x;
    char* y;
};

altnesne olarak adlandırılan ve “x” değerini içeren bir “S” nesnesi içinde ardışık bir “sizeof(int)” bayt dizisi ve “sizeof(char*)” baytlarını içeren başka bir alt nesne vardır. “y"nin değeri. İkisi birbirine karıştırılamaz.

  • Bir sınıf türünün üyeleri ve/veya t1, t2,...tN türlerine sahip temel sınıfları varsa, boyut en az sizeof(t1) + sizeof(t2) + ... + sizeof(tN) olmalıdır. verilen önceki noktalar. Ancak, üyelerin ve temel sınıfların hizalama gereksinimlerine bağlı olarak, derleyici alt nesneler arasına veya tam nesnenin başlangıcı veya sonu.

    struct AnInt      { int i; };
  // sizeof(AnInt)        == sizeof(int)
  // Assuming a typical 32- or 64-bit system, sizeof(AnInt)        == 4 (4).
struct TwoInts    { int i, j; };
  // sizeof(TwoInts)      >= 2 * sizeof(int)
  // Assuming a typical 32- or 64-bit system, sizeof(TwoInts)      == 8 (4 + 4).
struct IntAndChar { int i; char c; };
  // sizeof(IntAndChar)   >= sizeof(int) + sizeof(char)
  // Assuming a typical 32- or 64-bit system, sizeof(IntAndChar)   == 8 (4 + 1 + padding).
struct AnIntDerived : AnInt { long long l; };
  // sizeof(AnIntDerived) >= sizeof(AnInt) + sizeof(long long)
  // Assuming a typical 32- or 64-bit system, sizeof(AnIntDerived) == 16 (4 + padding + 8).

  • Hizalama gereksinimleri nedeniyle bir nesneye dolgu eklenirse, boyut, üyelerin ve temel sınıfların boyutlarının toplamından daha büyük olacaktır. ’n’ bayt hizalama ile, boyut tipik olarak tüm üyelerin ve temel sınıfların boyutundan daha büyük olan ’n’nin en küçük katı olacaktır. Her üye “memN” tipik olarak “alignof(memN)“nin katı olan bir adrese yerleştirilir ve “n” tipik olarak tüm üyelerin “alignof"ları arasında en büyük “alignof” olacaktır. Bundan dolayı, daha küçük bir ‘alignof’a sahip bir üyeyi, daha büyük bir ‘alignof’a sahip bir üye takip ediyorsa, ikinci üyenin, öncekinden hemen sonra yerleştirildiğinde düzgün şekilde hizalanmama olasılığı vardır. Bu durumda, dolgu ( hizalama elemanı olarak da bilinir) iki üye arasına yerleştirilecektir, böylece ikinci üye istenen hizalamaya sahip olabilir. Tersine, daha büyük ‘alignof’a sahip bir üyeyi daha küçük ‘alignof’a sahip bir üye takip ediyorsa, genellikle hiçbir dolgu gerekli olmayacaktır. Bu işlem aynı zamanda “paketleme” olarak da bilinir. Sınıflar tipik olarak üyelerinin ‘alignof’unu en büyük ‘alignof’ ile paylaştıkları için, sınıflar tipik olarak doğrudan veya dolaylı olarak içerdikleri en büyük yerleşik türün ‘alignof’una hizalanır.

    // Assume sizeof(short) == 2, sizeof(int) == 4, and sizeof(long long) == 8.
// Assume 4-byte alignment is specified to the compiler.
struct Char { char c; };
  // sizeof(Char)                == 1 (sizeof(char))
struct Int  { int i; };
  // sizeof(Int)                 == 4 (sizeof(int))
struct CharInt { char c; int i; };
  // sizeof(CharInt)             == 8 (1 (char) + 3 (padding) + 4 (int))
struct ShortIntCharInt { short s; int i; char c; int j; };
  // sizeof(ShortIntCharInt)     == 16 (2 (short) + 2 (padding) + 4 (int) + 1 (char) +
  //                                    3 (padding) + 4 (int))
struct ShortIntCharCharInt { short s; int i; char c; char d; int j; };
  // sizeof(ShortIntCharCharInt) == 16 (2 (short) + 2 (padding) + 4 (int) + 1 (char) +
  //                                    1 (char) + 2 (padding) + 4 (int))
struct ShortCharShortInt { short s; char c; short t; int i; };
  // sizeof(ShortCharShortInt)   == 12 (2 (short) + 1 (char) + 1 (padding) + 2 (short) +
  //                                    2 (padding) + 4 (int))
struct IntLLInt { int i; long long l; int j; };
  // sizeof(IntLLInt)            == 16 (4 (int) + 8 (long long) + 4 (int))
  // If packing isn't explicitly specified, most compilers will pack this as
  //   8-byte alignment, such that:
  // sizeof(IntLLInt)            == 24 (4 (int) + 4 (padding) + 8 (long long) +
  //                                    4 (int) + 4 (padding))

// Assume sizeof(bool) == 1, sizeof(ShortIntCharInt) == 16, and sizeof(IntLLInt) == 24.
// Assume default alignment: alignof(ShortIntCharInt) == 4, alignof(IntLLInt) == 8.
struct ShortChar3ArrShortInt {
    short s;
    char c3[3];
    short t;
    int i;
};
  // ShortChar3ArrShortInt has 4-byte alignment: alignof(int) >= alignof(char) &&
  //                                             alignof(int) >= alignof(short)
  // sizeof(ShortChar3ArrShortInt) == 12 (2 (short) + 3 (char[3]) + 1 (padding) +
  //                                      2 (short) + 4 (int))
  // Note that t is placed at alignment of 2, not 4.  alignof(short) == 2.

struct Large_1 {
    ShortIntCharInt sici;
    bool b;
    ShortIntCharInt tjdj;
};
  // Large_1 has 4-byte alignment.
    // alignof(ShortIntCharInt) == alignof(int) == 4
    // alignof(b) == 1
    // Therefore, alignof(Large_1) == 4.
  // sizeof(Large_1) == 36 (16 (ShortIntCharInt) + 1 (bool) + 3 (padding) +
  //                        16 (ShortIntCharInt))
struct Large_2 {
    IntLLInt illi;
    float f;
    IntLLInt jmmj;
};
  // Large_2 has 8-byte alignment.
    // alignof(IntLLInt) == alignof(long long) == 8
    // alignof(float) == 4
    // Therefore, alignof(Large_2) == 8.
  // sizeof(Large_2) == 56 (24 (IntLLInt) + 4 (float) + 4 (padding) + 24 (IntLLInt))

  • Sıkı hizalama “hizalamalar” ile zorlanırsa, aksi takdirde daha küçük olsa bile, türü belirtilen hizalamayı karşılamaya zorlamak için dolgu kullanılacaktır. Örneğin, aşağıdaki tanımla, “Karakterler<5>“in sonuna eklenen üç (veya muhtemelen daha fazla) dolgu baytı olacaktır, böylece toplam boyutu 8 olur. 5 boyutundadır, çünkü bu sınıftan bir dizi oluşturmak imkansız olurdu, bu nedenle boyut, dolgu baytları eklenerek 4’ün katlarına “yuvarlanmalıdır”.

    // This type shall always be aligned to a multiple of 4.  Padding shall be inserted as
// needed.
// Chars<1>..Chars<4> are 4 bytes, Chars<5>..Chars<8> are 8 bytes, etc.
template<size_t SZ>
struct alignas(4) Chars { char arr[SZ]; };

static_assert(sizeof(Chars<1>) == sizeof(Chars<4>), "Alignment is strict.\n");
son sürüm
  • Bir sınıfın statik olmayan iki üyesi aynı erişim belirtecine sahipse, o zaman bir bildirim sırasında daha sonra gelen nesne temsilinde daha sonra gelmesi garanti edilir. Ancak, statik olmayan iki üyenin farklı erişim belirteçleri varsa, bunların nesne içindeki göreli sırası belirtilmez.
  • Temel sınıf alt nesnelerinin bir nesne içinde hangi sırada göründükleri, ardışık olarak mı ortaya çıktıkları ve üye alt nesnelerden önce, sonra veya arasında mı göründükleri belirtilmemiştir.

Aritmetik türleri #

Dar karakter türleri #

“imzasız karakter” türü, ikili bir sayıyı temsil etmek için tüm bitleri kullanır. Bu nedenle, örneğin, “işaretsiz karakter” 8 bit uzunluğundaysa, o zaman bir “char” nesnesinin 256 olası bit modeli, 256 farklı değeri {0, 1, …, 255} temsil eder. 42 sayısının “00101010” bit deseni ile temsil edilmesi garanti edilir.

“imzalı karakter” tipinde dolgu biti yoktur, yani “imzalı karakter” 8 bit uzunluğundaysa, o zaman bir sayıyı temsil etme kapasitesi 8 bittir.

Bu garantilerin dar karakter türleri dışındaki türler için geçerli olmadığını unutmayın.

Tamsayı türleri #

İşaretsiz tamsayı türleri saf bir ikili sistem kullanır, ancak dolgu bitleri içerebilir. Örneğin, “imzasız int"nin 64 bit uzunluğunda olması (olasılıkla olmasa da) mümkündür, ancak yalnızca 0 ile 232 - 1 (dahil) arasındaki tamsayıları depolayabilir. Diğer 32 bit, doğrudan yazılmaması gereken dolgu bitleri olacaktır.

İşaretli tamsayı türleri, işaret biti ve muhtemelen dolgu bitleri olan bir ikili sistem kullanır. İşaretli bir tamsayı türünün ortak aralığına ve karşılık gelen işaretsiz tamsayı türüne ait değerler aynı gösterime sahiptir. Örneğin, “işaretsiz kısa” bir nesnenin “0001010010101011” bit modeli “5291” değerini temsil ediyorsa, “kısa” bir nesne olarak yorumlandığında da “5291” değerini temsil eder.

Her üç sistem de önceki paragraftaki gereksinimi karşıladığından, ikinin tümleyeni, birin tümleyeni veya işaret büyüklüğü gösteriminin kullanılıp kullanılmadığı uygulamayla tanımlanır.

Kayan nokta türleri #

Kayan nokta türlerinin değer temsili, uygulama tanımlıdır. En yaygın olarak, “float” ve “double” türleri IEEE 754’e uygundur ve 32 ve 64 bit uzunluğundadır (bu nedenle, örneğin “float”, 8 üs bitini ve 1 işaret bitini takip edecek olan 23 bit kesinliğe sahip olacaktır). Ancak, standart hiçbir şeyi garanti etmez. Kayan nokta türleri genellikle hesaplamalarda kullanıldıklarında hatalara neden olan “tuzak gösterimleri"ne sahiptir.

Diziler #

Bir dizi türünün öğeleri arasında dolgu yoktur. Bu nedenle, ‘T’ eleman tipine sahip bir dizi, sadece sırayla bellekte düzenlenen ‘T’ nesnelerinin bir dizisidir.

Çok boyutlu bir dizi, bir dizi dizisidir ve yukarıdakiler yinelemeli olarak uygulanır. Örneğin, beyanımız varsa

int a[5][3];

o zaman “a”, 3 “int"den oluşan 5 diziden oluşan bir dizidir. Bu nedenle, ‘a[0][0]’, ‘a0’, ‘a[0][2]’ üç öğesinden oluşan ‘a[0]’, bellekte düzenlenir “a[1][0]”, “a1” ve “a[1][2]“den oluşan “a1“dan önce. Buna satır majör sırası denir.
