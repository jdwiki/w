Sınıflar, işlevler ve (C++14’ten beri) değişkenler şablonlanabilir. Şablon, tüm parametreler belirtildiğinde somut bir sınıf, işlev veya değişken olacak bazı ücretsiz parametreler içeren bir kod parçasıdır. Parametreler türler, değerler veya kendileri şablonlar olabilir. İyi bilinen bir şablon ‘std::vector’ olup, öğe türü belirtildiğinde somut bir kap türü haline gelir, ör. ‘std::vector’.

Sözdizimi #

  • şablon < şablon-parametre-listesi > bildirim
  • dışa aktarma şablonu < template-parameter-list > declaration /* C++11’e kadar */
  • şablon <> bildirim
  • şablon bildirim
  • dış şablon bildirim /* C++11’den beri */
  • şablon < şablon-parametre-listesi > sınıf …(opt) tanımlayıcı(opt)
  • şablon < template-parameter-list > class identifier(opt) = id-ifadesi
  • şablon < template-parameter-list > typename …(opt) identifier(opt) /* C++17’den beri */
  • şablon < template-parameter-list > typename identifier(opt) = id-expression /* C++17’den beri */
  • postfix-ifadesi . şablon kimlik ifadesi
  • postfix-ifadesi -> şablon id-ifadesi
  • iç içe isim belirteci ‘şablon’ basit şablon kimliği ‘::’

“Şablon” kelimesi, bağlama bağlı olarak C++ dilinde beş farklı anlamı olan bir anahtar kelimedir.

  1. Ardından “<>” içine alınmış bir şablon parametreleri listesi geldiğinde, sınıf şablonu gibi bir şablon bildirir. /basic-class-template), bir işlev şablonu veya bir [kısmi uzmanlık](http: //stackoverflow.com/documentation/c%2b%2b/460/templates/6253/partial-template-specialization) mevcut bir şablonun.

    template <class T>
void increment(T& x) { ++x; }

  2. Arkasından boş bir “<>” geldiğinde, bir açık (tam) uzmanlık bildirir. uzmanlık).

    template <class T>
void print(T x);

template <> // <-- keyword used in this sense here
void print(const char* s) {
    // output the content of the string
    printf("%s\n", s);
}

  3. Ardından “<>” içermeyen bir bildirim geldiğinde, bir açık örnekleme bildirimi veya tanımı oluşturur. .

    template <class T>
std::set<T> make_singleton(T x) { return std::set<T>(x); }

template std::set<int> make_singleton(int x); // <-- keyword used in this sense here

  4. Bir şablon parametre listesi içinde, bir şablon şablon parametresi sunar.

    template <class T, template <class U> class Alloc>
//                 ^^^^^^^^ keyword used in this sense here
class List {
    struct Node {
        T value;
        Node* next;
    };
    Alloc<Node> allocator;
    Node* allocate_node() {
        return allocator.allocate(sizeof(T));
    }
    // ...
};

  5. Kapsam çözümleme operatörü :: ve sınıf üyesi erişim operatörleri . ve -> sonra, aşağıdaki adın bir şablon olduğunu belirtir.

    struct Allocator {
    template <class T>
    T* allocate();
};

template <class T, class Alloc>
class List {
    struct Node {
        T value;
        Node* next;
    }
    Alloc allocator;
    Node* allocate_node() {
        // return allocator.allocate<Node>();       // error: < and > are interpreted as
                                                    // comparison operators
        return allocator.template allocate<Node>(); // ok; allocate is a template
        //               ^^^^^^^^ keyword used in this sense here
    }
};

C++ 11’den önce, bir şablon ‘dışa aktarma’ anahtar kelime ile bildirilebiliyordu, bu da onu dışa aktarılmış bir şablon haline getiriyordu. Dışa aktarılan bir şablonun tanımının, şablonun somutlaştırıldığı her çeviri biriminde bulunması gerekmez. Örneğin, aşağıdakilerin çalışması gerekiyordu:

“foo.h”:

#ifndef FOO_H
#define FOO_H
export template <class T> T identity(T x);
#endif

“foo.cpp”:

#include "foo.h"
template <class T> T identity(T x) { return x; }

“main.cpp”:

#include "foo.h"
int main() {
    const int x = identity(42); // x is 42
}

Uygulamanın zorluğu nedeniyle, “export” anahtar sözcüğü çoğu büyük derleyici tarafından desteklenmiyordu. C++ 11’de kaldırıldı; şimdi, “export” anahtar sözcüğünü kullanmak hiçbir şekilde yasa dışıdır. Bunun yerine, şablonları genellikle başlıklarda tanımlamak gerekir (genellikle başlıklarda tanımlanmayan şablon olmayan işlevlerin aksine). http://stackoverflow.com/questions/495021/why-can-templates-only-be-implemented-in-the-header-file adresine bakın

İşlev Şablonları #

Şablonlama, aynı etkiye sahip işlevlere (ve daha geleneksel yapılara) da uygulanabilir.

// 'T' stands for the unknown type
// Both of our arguments will be of the same type.
template<typename T>
void printSum(T add1, T add2)
{
    std::cout << (add1 + add2) << std::endl;
}

Bu daha sonra yapı şablonlarıyla aynı şekilde kullanılabilir.

printSum<int>(4, 5);
printSum<float>(4.5f, 8.9f);

Her iki durumda da şablon argümanı parametre türlerini değiştirmek için kullanılır; sonuç, normal bir C++ işlevi gibi çalışır (parametreler şablon türüyle eşleşmezse, derleyici standart dönüşümleri uygular).

Şablon işlevlerinin ek bir özelliği (şablon sınıflarından farklı olarak), derleyicinin işleve iletilen parametrelere dayalı olarak şablon parametrelerini çıkarabilmesidir.

printSum(4, 5);     // Both parameters are int.
                    // This allows the compiler deduce that the type
                    // T is also int.

printSum(5.0, 4);   // In this case the parameters are two different types.
                    // The compiler is unable to deduce the type of T
                    // because there are contradictions. As a result
                    // this is a compile time error.

Bu özellik, şablon yapılarını ve işlevlerini birleştirdiğimizde kodu basitleştirmemizi sağlar. Standart kitaplıkta, ‘make_X()’ yardımcı işlevini kullanarak ‘şablon yapısı X’ yapmamızı sağlayan ortak bir kalıp vardır.

// The make_X pattern looks like this.
// 1) A template structure with 1 or more template types.
template<typename T1, typename T2>
struct MyPair
{
    T1      first;
    T2      second;
};
// 2) A make function that has a parameter type for
//    each template parameter in the template structure.
template<typename T1, typename T2>
MyPair<T1, T2> make_MyPair(T1 t1, T2 t2)
{
    return MyPair<T1, T2>{t1, t2};
}

Bu nasıl yardımcı olur?

auto val1 = MyPair<int, float>{5, 8.7};     // Create object explicitly defining the types
auto val2 = make_MyPair(5, 8.7);            // Create object using the types of the paramters.
                                            // In this code both val1 and val2 are the same
                                            // type.

Not: Bu, kodu kısaltmak için tasarlanmamıştır. Bu, kodu daha sağlam hale getirmek için tasarlanmıştır. Birden fazla lokasyon yerine tek bir yerde kod değiştirerek tiplerin değiştirilmesine olanak sağlar.

Temel Sınıf Şablonu #

Bir sınıf şablonunun temel fikri, şablon parametresinin derleme zamanında bir tür ile değiştirilmesidir. Sonuç olarak, aynı sınıf birden çok tür için yeniden kullanılabilir. Kullanıcı, sınıfın bir değişkeni bildirildiğinde hangi türün kullanılacağını belirtir. Bunun üç örneği “main()” içinde gösterilmiştir:

#include <iostream>
using std::cout;

template <typename T>         // A simple class to hold one number of any type
class Number {
public:
    void setNum(T n);         // Sets the class field to the given number
    T plus1() const;          // returns class field's "follower"
private:
    T num;                    // Class field
};

template <typename T>         // Set the class field to the given number
void Number<T>::setNum(T n) {
    num = n;
}

template <typename T>         // returns class field's "follower"
T Number<T>::plus1() const {
    return num + 1;
}

int main() {
    Number<int> anInt;        // Test with an integer (int replaces T in the class)
    anInt.setNum(1);
    cout << "My integer + 1 is " << anInt.plus1() << "\n";     // Prints 2

    Number<double> aDouble;   // Test with a double
    aDouble.setNum(3.1415926535897);
    cout << "My double + 1 is " << aDouble.plus1() << "\n";    // Prints 4.14159

    Number<float> aFloat;     // Test with a float
    aFloat.setNum(1.4);
    cout << "My float + 1 is " << aFloat.plus1() << "\n";      // Prints 2.4

    return 0;  // Successful completion
}

Değişken şablon veri yapıları #

Derleme zamanında tanımlanan değişken sayıda ve türde veri üyelerine sahip sınıfları veya yapıları tanımlamak genellikle yararlıdır. Kurallı örnek ‘std::tuple’dır, ancak bazen kendi özel yapılarınızı tanımlamanız gerekir. İşte yapıyı bileşik kullanarak tanımlayan bir örnek (‘std::tuple’da olduğu gibi kalıtım yerine. Daha sonraki uzmanlaşmada yineleme sonlandırma için temel durum olarak da hizmet eden genel (boş) tanımla başlayın:

template<typename ... T>
struct DataStructure {};

Bu, henüz çok kullanışlı olmasa da, DataStructure<> data adında boş bir yapı tanımlamamıza izin veriyor.

Ardından özyinelemeli vaka uzmanlığı gelir:

template<typename T, typename ... Rest>
struct DataStructure<T, Rest ...>
{
    DataStructure(const T& first, const Rest& ... rest)
        : first(first)
        , rest(rest...)
    {}
    
    T first;                                
    DataStructure<Rest ... > rest;
};

Bu, artık DataStructure<int, float, std::string> data(1, 2.1, "hello") gibi rastgele veri yapıları oluşturmamız için yeterlidir.

Yani, ne oluyor? İlk olarak, bunun, “Dinlenme” paketinin özel yapısını umursamadan en az bir değişken şablon parametresinin (yukarıda “T” olarak adlandırılır) mevcut olması şartı olan bir uzmanlık olduğuna dikkat edin. ‘T’nin var olduğunu bilmek, veri üyesinin ‘ilk’ tanımına izin verir. Verilerin geri kalanı özyinelemeli olarak `DataStructure<Rest … > rest’ olarak paketlenir. Yapıcı, ‘rest’ üyeye özyinelemeli bir kurucu çağrısı da dahil olmak üzere bu üyelerin her ikisini de başlatır.

Bunu daha iyi anlamak için bir örnek üzerinde çalışabiliriz: Diyelim ki DataStructure<int, float> data bildiriminiz var. Bildirim ilk olarak uzmanlaşmayla eşleşir ve “int first” ve “DataStructure rest” veri üyelerine sahip bir yapı verir. ‘rest’ tanımı, kendi ‘float first’ ve ‘DataStructure<> rest’ üyelerini oluşturarak yine bu uzmanlıkla eşleşir. Son olarak, bu son “dinlenme”, temel durum tanımıyla eşleşerek boş bir yapı oluşturur.

Bunu şu şekilde görselleştirebilirsiniz:

DataStructure<int, float>
   -> int first
   -> DataStructure<float> rest
         -> float first
         -> DataStructure<> rest
              -> (empty)

Şimdi veri yapısına sahibiz, ancak bireysel veri öğelerine kolayca erişemediğimiz için henüz çok kullanışlı değil (örneğin, kullanmamız gereken DataStructure<int, float, std::string> data'nın son üyesine erişmek için) Tam olarak kullanıcı dostu olmayan data.rest.rest.first`). Bu yüzden ona bir “get” yöntemi ekliyoruz (yalnızca temel durum yapısının “get” için verisi olmadığı için uzmanlıkta gereklidir):

template<typename T, typename ... Rest>
struct DataStructure<T, Rest ...>
{
    ...
    template<size_t idx>
    auto get()
    {
        return GetHelper<idx, DataStructure<T,Rest...>>::get(*this);
    }
    ...
};

Gördüğünüz gibi, bu “get” üye işlevinin kendisi şablonlanmıştır - bu sefer gerekli olan üyenin dizininde (bu nedenle kullanım, “std::tuple” benzeri “data.get<1>()” gibi şeyler olabilir. `). Gerçek çalışma, bir yardımcı sınıf olan ‘GetHelper’ içindeki statik bir işlev tarafından yapılır. Gerekli işlevselliği doğrudan “DataStructure"ın “get"inde tanımlayamamamızın nedeni (az sonra göreceğimiz gibi) “idx” üzerinde uzmanlaşmamız gerekmesidir - ancak bir şablon üyesini uzmanlaştırmak mümkün değildir. içeren sınıf şablonunu uzmanlaştırmadan işlev. Ayrıca burada C++14 stili ‘auto’ kullanımının hayatımızı önemli ölçüde kolaylaştırdığına dikkat edin, aksi takdirde dönüş türü için oldukça karmaşık bir ifadeye ihtiyaç duyacağız.

Yani yardımcı sınıfa. Bu sefer boş bir ileri bildirime ve iki uzmanlığa ihtiyacımız olacak. Önce beyanname:

template<size_t idx, typename T>
struct GetHelper;

Şimdi temel durum (‘idx==0’ olduğunda). Bu durumda sadece ‘ilk’ üyeyi döndürürüz:

template<typename T, typename ... Rest>
struct GetHelper<0, DataStructure<T, Rest ... >>
{
    static T get(DataStructure<T, Rest...>& data)
    {
        return data.first;
    }
};

Özyinelemeli durumda, “idx"i azaltırız ve “rest” üyesi için “GetHelper"ı çağırırız:

template<size_t idx, typename T, typename ... Rest>
struct GetHelper<idx, DataStructure<T, Rest ... >>
{
    static auto get(DataStructure<T, Rest...>& data)
    {
        return GetHelper<idx-1, DataStructure<Rest ...>>::get(data.rest);
    }
};

Bir örnek üzerinde çalışmak için, “DataStructure<int, float> data"ya sahip olduğumuzu ve “data.get<1>()“e ihtiyacımız olduğunu varsayalım. Bu, GetHelper<1, DataStructure<int, float>>::get(data) (2. uzmanlık) öğesini çağırır ve bu da GetHelper<0, DataStructure<float>>::get(data.rest) öğesini çağırır. sonunda (şu anda “idx” 0 olduğu için 1. uzmanlaşmaya göre) “data.rest.first” değerini döndüren .

İşte bu kadar! ‘main’ işlevinde bazı örnek kullanımlarla birlikte tüm çalışma kodu:

#include <iostream>

template<size_t idx, typename T>
struct GetHelper;

template<typename ... T>
struct DataStructure
{
};

template<typename T, typename ... Rest>
struct DataStructure<T, Rest ...>
{
    DataStructure(const T& first, const Rest& ... rest)
        : first(first)
        , rest(rest...)
    {}
    
    T first;
    DataStructure<Rest ... > rest;
    
    template<size_t idx>
    auto get()
    {
        return GetHelper<idx, DataStructure<T,Rest...>>::get(*this);
    }
};

template<typename T, typename ... Rest>
struct GetHelper<0, DataStructure<T, Rest ... >>
{
    static T get(DataStructure<T, Rest...>& data)
    {
        return data.first;
    }
};

template<size_t idx, typename T, typename ... Rest>
struct GetHelper<idx, DataStructure<T, Rest ... >>
{
    static auto get(DataStructure<T, Rest...>& data)
    {
        return GetHelper<idx-1, DataStructure<Rest ...>>::get(data.rest);
    }
};

int main()
{
    DataStructure<int, float, std::string> data(1, 2.1, "Hello");
        
    std::cout << data.get<0>() << std::endl;
    std::cout << data.get<1>() << std::endl;
    std::cout << data.get<2>() << std::endl;
    
    return 0;
}

Argüman iletme #

Şablon, yönlendirme referansı: kullanarak hem değer hem de değer referanslarını kabul edebilir.

template <typename T>
void f(T &&t);

Bu durumda, bağlama göre gerçek ’t’ türü çıkarılacaktır:

struct X { };

X x;
f(x); // calls f<X&>(x)
f(X()); // calls f<X>(x)

İlk durumda, “T” türü, “X"e (‘X&’) referans olarak çıkarılır ve “t” türü, “X"e * değer referansı iken, ikinci durumda, tür of “T”, “X” olarak çıkarılır ve “t” türü, “X” (“X&&”) değerine *değer referansı olarak çıkarılır.

Not: İlk durumda “decltype(t)“nin “T” ile aynı olduğunu, ancak ikinci durumda olmadığını belirtmekte fayda var.

Bir değer veya değer referansı olsun, t’yi başka bir işleve mükemmel bir şekilde iletmek için, ‘std::forward’ kullanılmalıdır:

template <typename T>
void f(T &&t) {
    g(std::forward<T>(t));
}

Yönlendirme referansları değişken şablonlarla kullanılabilir:

template <typename... Args>
void f(Args&&... args) {
    g(std::forward<Args>(args)...);
}

Not: Yönlendirme referansları yalnızca şablon parametreleri için kullanılabilir, örneğin aşağıdaki kodda, “v” bir değer referansıdır, iletme referansı değildir:

#include <vector>

template <typename T>
void f(std::vector<T> &&v);

Şablon Uzmanlığı #

Bir şablon sınıfının/yöntemin belirli örneklemeleri için uygulamayı tanımlayabilirsiniz.

Örneğin:

template <typename T>
T sqrt(T t) { /* Some generic implementation */ }

Daha sonra şunları yazabilirsiniz:

template<>
int sqrt<int>(int i) { /* Highly optimized integer implementation */ }

Daha sonra ‘sqrt(4.0)’ yazan bir kullanıcı genel uygulamayı, ‘sqrt(4)’ ise özel uygulamayı alır.

Kısmi şablon uzmanlığı #

Tam bir şablon uzmanlığının aksine, kısmi şablon uzmanlığı, şablonun mevcut şablonun bazı argümanları sabitlenmiş olarak tanıtılmasına izin verir. Kısmi şablon uzmanlığı yalnızca şablon sınıfı/yapıları için kullanılabilir:

// Common case:
template<typename T, typename U>
struct S {
    T t_val;
    U u_val;
};

// Special case when the first template argument is fixed to int
template<typename V>
struct S<int, V> {
    double another_value;
    int foo(double arg) {// Do something}
};

Yukarıda gösterildiği gibi, kısmi şablon uzmanlıkları, tamamen farklı veri ve işlev üyeleri kümeleri sunabilir.

Kısmen özelleştirilmiş bir şablon başlatıldığında, en uygun uzmanlık seçilir. Örneğin, bir şablon ve iki kısmi uzmanlık tanımlayalım:

template<typename T, typename U, typename V>
struct S {
    static void foo() {
        std::cout << "General case\n";
    }
};

template<typename U, typename V>
struct S<int, U, V> {
    static void foo() {
        std::cout << "T = int\n";
    }
};

template<typename V>
struct S<int, double, V> {
    static void foo() {
        std::cout << "T = int, U = double\n";
    }
};

Şimdi aşağıdaki çağrılar:

S<std::string, int, double>::foo();
S<int, float, std::string>::foo();
S<int, double, std::string>::foo();

yazdıracak

General case
T = int
T = int, U = double

İşlev şablonları yalnızca tamamen özelleştirilebilir:

template<typename T, typename U>
void foo(T t, U u) {
    std::cout << "General case: " << t << " " << u << std::endl;
}

// OK.
template<>
void foo<int, int>(int a1, int a2) {
    std::cout << "Two ints: " << a1 << " " << a2 << std::endl;
}

void invoke_foo() {
    foo(1, 2.1); // Prints "General case: 1 2.1"
    foo(1,2);    // Prints "Two ints: 1 2"
}

// Compilation error: partial function specialization is not allowed.
template<typename U>
void foo<std::string, U>(std::string t, U u) {
    std::cout << "General case: " << t << " " << u << std::endl;
}

Takma ad şablonu #

Temel örnek:

template<typename T> using pointer = T*;

Bu tanım, ‘işaretçi’ öğesini ‘T*’ diğer adı yapar. Örneğin:

pointer<int> p = new int; // equivalent to: int* p = new int;

Takma ad şablonları özelleştirilemez. Bununla birlikte, bu işlevsellik, bir yapıdaki iç içe bir türe başvurmaları sağlanarak dolaylı olarak elde edilebilir:

template<typename T>
 struct nonconst_pointer_helper { typedef T* type; };

template<typename T>
 struct nonconst_pointer_helper<T const> { typedef T* type; };

template<typename T> using nonconst_pointer = nonconst_pointer_helper<T>::type;

Şablon şablonu parametreleri #

Bazen bir şablon tipini, değerlerini düzeltmeden şablona geçirmek isteriz. Şablon şablon parametreleri bunun için oluşturulur. Çok basit şablon şablon parametresi örnekleri:

template <class T>
struct Tag1 { };

template <class T>
struct Tag2 { };

template <template <class> class Tag>
struct IntTag {
   typedef Tag<int> type;
};

int main() {
   IntTag<Tag1>::type t;
}

#include <vector>
#include <iostream>

template <class T, template <class...> class C, class U>
C<T> cast_all(const C<U> &c) {
   C<T> result(c.begin(), c.end());
   return result;
}

int main() {
   std::vector<float> vf = {1.2, 2.6, 3.7};
   auto vi = cast_all<int>(vf);
   for(auto &&i: vi) {
      std::cout << i << std::endl;
   }
}

tip olmayan şablon argümanlarını auto ile bildirme #

C++17’den önce, tip olmayan bir şablon parametresi yazarken, önce onun tipini belirtmeniz gerekiyordu. Böylece ortak bir kalıp şöyle bir şey yazmaya başladı:

template <class T, T N>
struct integral_constant {
    using type = T;
    static constexpr T value = N;
};

using five = integral_constant<int, 5>;

Ancak karmaşık ifadeler için, bunun gibi bir şey kullanmak, şablonları başlatırken ‘decltype(expr), expr’ yazmak zorunda kalmayı gerektirir. Çözüm, bu deyimi basitleştirmek ve basitçe “auto"ya izin vermektir:

template <auto N>
struct integral_constant {
    using type = decltype(N); 
    static constexpr type value = N;
};

using five = integral_constant<5>;

unique_ptr için boş özel silici #

Güzel bir motive edici örnek, boş temel optimizasyonu ‘unique_ptr’ için özel bir siliciyle birleştirmeye çalışmaktan gelebilir. Farklı C API silicilerinin farklı dönüş türleri vardır, ancak umursamıyoruz - sadece herhangi bir işlev için bir şeyin çalışmasını istiyoruz:

template <auto DeleteFn>
struct FunctionDeleter {
    template <class T>
    void operator()(T* ptr) const {
        DeleteFn(ptr);
    }
};

template <T, auto DeleteFn>
using unique_ptr_deleter = std::unique_ptr<T, FunctionDeleter<DeleteFn>>;

Ve şimdi, dönüş türünden bağımsız olarak, tür olmayan bir şablon parametresi olarak “T” türünde bir argüman alabilen herhangi bir işlev işaretçisini kullanabilir ve bundan boyutsuz bir “unique_ptr” ek yükü elde edebilirsiniz:

unique_ptr_deleter<std::FILE, std::fclose> p;

Tür olmayan şablon parametresi #

Şablon parametresi olarak türlerin dışında beyan etmemize izin verilir aşağıdaki kriterlerden birini karşılayan sabit ifadelerin değerleri:

  • integral veya numaralandırma türü,
  • nesneye işaretçi veya işleve işaretçi,
  • nesneye referans değeri veya fonksiyona referans değeri,
  • üyeye işaretçi,
  • std::nullptr_t.

Tüm şablon parametreleri gibi, tür olmayan şablon parametreleri de Şablon Bağımsız Değişken Kesintisi aracılığıyla açıkça belirtilebilir, varsayılan olarak belirlenebilir veya örtük olarak türetilebilir.

Tür dışı şablon parametresi kullanımına örnek:

#include <iostream>

template<typename T, std::size_t size>
std::size_t size_of(T (&anArray)[size])  // Pass array by reference. Requires.
{                                        // an exact size. We allow all sizes
    return size;                         // by using a template "size".
}

int main()
{
    char anArrayOfChar[15];
    std::cout << "anArrayOfChar: " << size_of(anArrayOfChar) << "\n";

    int  anArrayOfData[] = {1,2,3,4,5,6,7,8,9};
    std::cout << "anArrayOfData: " << size_of(anArrayOfData) << "\n";
}

Hem tür hem de tür olmayan şablon parametrelerini açıkça belirtme örneği:

#include <array>
int main ()
{
    std::array<int, 5> foo; // int is a type parameter, 5 is non-type
}

Tip olmayan şablon parametreleri, şablon elde etmenin yollarından biridir. yinelenir ve Metaprogramming yapmayı sağlar.

Açık örnekleme #

Açık bir örnekleme tanımı, henüz kullanmadan, bir şablondan somut bir sınıf, işlev veya değişken oluşturur ve bildirir. Açık bir örneklemeye diğer çeviri birimlerinden başvurulabilir. Bu, yalnızca sonlu bir argüman kümesiyle somutlaştırılacaksa, bir başlık dosyasında bir şablon tanımlamaktan kaçınmak için kullanılabilir. Örneğin:

// print_string.h
template <class T>
void print_string(const T* str);

// print_string.cpp
#include "print_string.h"
template void print_string(const char*);
template void print_string(const wchar_t*);

“print_string” ve “print_string<wchar_t>” açıkça “print_string.cpp” içinde örneklendirildiği için, “print_string” şablonu başlıkta tanımlanmasa bile bağlayıcı bunları bulabilecektir. Bu açık örnekleme bildirimleri mevcut değilse, büyük olasılıkla bir bağlayıcı hatası oluşacaktır. http://stackoverflow.com/questions/495021/why-can-templates-only-be-implemented-in-the-header-file adresine bakın

Açık bir örnekleme tanımından önce “extern” anahtar sözcük geliyorsa, bunun yerine açık bir örnekleme bildirimi olur. Belirli bir uzmanlık için açık bir örnekleme bildiriminin varlığı, geçerli çeviri birimi içinde verilen uzmanlığın örtük olarak somutlaştırılmasını engeller. Bunun yerine, aksi takdirde örtük bir örneklemeye neden olacak bu uzmanlaşmaya yapılan bir referans, aynı veya başka bir TU’daki açık bir somutlaştırma tanımına atıfta bulunabilir.

‘foo.h’

#ifndef FOO_H
#define FOO_H
template <class T> void foo(T x) {
    // complicated implementation
}
#endif

“foo.cpp”

#include "foo.h"
// explicit instantiation definitions for common cases
template void foo(int);
template void foo(double);

“main.cpp”

#include "foo.h"
// we already know foo.cpp has explicit instantiation definitions for these
extern template void foo(double);
int main() {
    foo(42);   // instantiates foo<int> here;
               // wasteful since foo.cpp provides an explicit instantiation already!
    foo(3.14); // does not instantiate foo<double> here;
               // uses instantiation of foo<double> in foo.cpp instead
}

Varsayılan şablon parametre değeri #

Tıpkı fonksiyon argümanlarında olduğu gibi, şablon parametrelerinin de varsayılan değerleri olabilir. Varsayılan değere sahip tüm şablon parametreleri, şablon parametre listesinin sonunda bildirilmelidir. Temel fikir, şablon başlatma sırasında varsayılan değere sahip şablon parametrelerinin atlanabilmesidir.

Varsayılan şablon parametre değeri kullanımına ilişkin basit bir örnek:

template <class T, size_t N = 10>
struct my_array {
    T arr[N];
};

int main() {
    /* Default parameter is ignored, N = 5 */
    my_array<int, 5> a;

    /* Print the length of a.arr: 5 */
    std::cout << sizeof(a.arr) / sizeof(int) << std::endl;

    /* Last parameter is omitted, N = 10 */
    my_array<int> b;

    /* Print the length of a.arr: 10 */
    std::cout << sizeof(b.arr) / sizeof(int) << std::endl;
}
