Bit Operatörleri

  • | - bit düzeyinde VEYA
  • ^ - bitsel XOR (özel VEYA)
  • & - bitsel VE
  • « - sola kaydırma
  • » - sağa kaydırma
  • ~ - bitsel DEĞİL (tekli tamamlayıcı)

Bu sayfada

  • | - bit düzeyinde VEYA
  • ^ - bitsel XOR (özel VEYA)
  • & - bitsel VE
  • « - sola kaydırma
  • » - sağa kaydırma
  • ~ - bitsel DEĞİL (tekli tamamlayıcı)

Bit kaydırma işlemleri tüm işlemci mimarilerinde taşınabilir değildir, farklı işlemciler farklı bit genişliklerine sahip olabilir. Başka bir deyişle, eğer yazdıysan

int a = ~0;
int b = a << 1;

Bu değer, 64 bitlik bir makinede ve 32 bitlik bir makinede veya x86 tabanlı bir işlemciden PIC tabanlı bir işlemciye farklı olacaktır.

Endian-ness, bit bilge işlemlerin kendileri için dikkate alınmasına gerek yoktur, yani, sağa kaydırma (>>) bitleri en az anlamlı bite kaydırır ve bir XOR, özel veya bitler üzerinde gerçekleştirir. . Endian-ness, yalnızca verilerin kendisiyle birlikte dikkate alınmalıdır, yani, endian-ness, uygulamanız için bir endişe ise, bit bilge işlemlerden bağımsız olarak bir endişedir.

| - bit düzeyinde VEYA #

int a = 5;     // 0101b  (0x05)
int b = 12;    // 1100b  (0x0C)
int c = a | b; // 1101b  (0x0D)

std::cout << "a = " << a << ", b = " << b << ", c = " << c << std::endl;

Çıktı

‘a = 5, b = 12, c = 13’

Neden

Biraz bilge ‘VEYA’, bit düzeyinde çalışır ve aşağıdaki Boole doğruluk tablosunu kullanır:

true OR true = true
true OR false = true
false OR false = false

“a” (“0101”) için ikili değer ve “b” (“1100”) için ikili değer birlikte “VEYA” olduğunda, “1101” ikili değerini alırız:

int a = 0 1 0 1
int b = 1 1 0 0 |
        ---------
int c = 1 1 0 1

Bit bilge VEYA, bit bilge atama bileşik operatörü |= kullanımına özel olarak atanmadıkça orijinal değerlerin değerini değiştirmez:

int a = 5;  // 0101b  (0x05)
a |= 12;    // a = 0101b | 1101b

^ - bitsel XOR (özel VEYA) #

int a = 5;     // 0101b  (0x05)
int b = 9;     // 1001b  (0x09)
int c = a ^ b; // 1100b  (0x0C)

std::cout << "a = " << a << ", b = " << b << ", c = " << c << std::endl;

Çıktı

‘a = 5, b = 9, c = 12’

Neden

Biraz bilge ‘XOR’ (özel veya) bit düzeyinde çalışır ve aşağıdaki Boole doğruluk tablosunu kullanır:

true OR true = false
true OR false = true
false OR false = false

Bir XOR işleminde “doğru VEYA doğru = yanlış” olduğuna dikkat edin, burada “doğru VE/VEYA doğru = doğru” işlemlerinde olduğu gibi, XOR işleminin özel doğası bu nedenledir.

Bunu kullanarak, “a” (“0101”) için ikili değer ve “b” (“1001”) için ikili değer birlikte “XOR” olduğunda “1100” ikili değerini elde ederiz:

int a = 0 1 0 1
int b = 1 0 0 1 ^
        ---------
int c = 1 1 0 0

Bit bilge XOR, özellikle bit bilge atama bileşik operatörü ^= kullanımına atanmadıkça orijinal değerlerin değerini değiştirmez:

int a = 5;  // 0101b  (0x05)
a ^= 9;    // a = 0101b ^ 1001b

Bit bilge XOR birçok şekilde kullanılabilir ve genellikle şifreleme ve sıkıştırma için bit maskesi işlemlerinde kullanılır.

Not: Aşağıdaki örnek genellikle güzel bir numara örneği olarak gösterilir. Ancak üretim kodunda kullanılmamalıdır (aynı sonucu elde etmenin daha iyi yolları std::swap() vardır).

Geçici olmadan iki değişkeni değiştirmek için bir XOR işlemi de kullanabilirsiniz:

int a = 42;
int b = 64;

// XOR swap
a ^= b;
b ^= a;
a ^= b;

std::cout << "a = " << a << ", b = " << b << "\n";

Bunu üretime geçirmek için kullanılabileceğinden emin olmak için bir çek eklemeniz gerekir.

void doXORSwap(int& a, int& b)
{
    // Need to add a check to make sure you are not swapping the same
    // variable with itself. Otherwise it will zero the value.
    if (&a != &b)
    {
        // XOR swap
        a ^= b;
        b ^= a;
        a ^= b;
    }
}

Bu nedenle, tek başına güzel bir numara gibi görünse de, gerçek kodda kullanışlı değildir. xor temel bir mantıksal işlem değil, diğerlerinin birleşimidir: a^c=~(a&c)&(a|c)

ayrıca 2015+ derleyicilerinde değişkenler ikili olarak atanabilir:

int cn=0b0111;

& - bitsel VE #

int a = 6;     // 0110b  (0x06)
int b = 10;    // 1010b  (0x0A)
int c = a & b; // 0010b  (0x02)

std::cout << "a = " << a << ", b = " << b << ", c = " << c << std::endl;

Çıktı

‘a = 6, b = 10, c = 2’

Neden

Biraz bilge “VE”, bit düzeyinde çalışır ve aşağıdaki Boole doğruluk tablosunu kullanır:

TRUE  AND TRUE  = TRUE
TRUE  AND FALSE = FALSE
FALSE AND FALSE = FALSE

“a” (“0110”) için ikili değer ve “b” (“1010”) için ikili değer birlikte “VE” olduğunda, “0010” ikili değerini alırız:

int a = 0 1 1 0
int b = 1 0 1 0 &
        ---------
int c = 0 0 1 0

Bit bilge AND, özellikle bit bilge atama bileşik operatörü &= kullanımına atanmadıkça orijinal değerlerin değerini değiştirmez:

int a = 5;  // 0101b  (0x05)
a &= 10;    // a = 0101b & 1010b

« - sola kaydırma #

int a = 1;      // 0001b
int b = a << 1; // 0010b

std::cout << "a = " << a << ", b = " << b << std::endl;

Çıktı

“a = 1, b = 2”

Neden

Sol bit bilge kaydırma, sol taraftaki değerin bitlerini (‘a’) sağda belirtilen sayıya (‘1’) kaydırır, esasen en az anlamlı bitleri 0’larla doldurur, böylece ‘5’ değerini (ikili) kaydırır. 4 kez sola 0000 0101) (ör. 5 << 4) 80 (ikili 0101 0000) değerini verecektir. Bir değeri 1 kez sola kaydırmanın, değeri 2 ile çarpmakla aynı şey olduğunu not edebilirsiniz, örneğin:

int a = 7;
while (a < 200) {
    std::cout << "a = " << a << std::endl;
    a <<= 1;
}

a = 7;
while (a < 200) {
    std::cout << "a = " << a << std::endl;
    a *= 2;
}

Ancak sola kaydırma işleminin işaret biti dahil tüm bitleri sola kaydıracağına dikkat edilmelidir, örnek:

int a = 2147483647; // 0111 1111 1111 1111 1111 1111 1111 1111
int b = a << 1;     // 1111 1111 1111 1111 1111 1111 1111 1110

std::cout << "a = " << a << ", b = " << b << std::endl;

Olası çıktı: a = 2147483647, b = -2

Bazı derleyiciler beklenen gibi sonuçlar verecek olsa da, işaret bitinin etkilenmesi için işaretli bir sayıyı kaydırmayı bırakırsanız sonucun tanımsız olduğuna dikkat edilmelidir. Ayrıca, kaydırmak istediğiniz bit sayısı negatif bir sayıysa veya soldaki türün tutabileceği bit sayısından büyükse, tanımsız olur, örneğin:

int a = 1;
int b = a << -1;  // undefined behavior
char c = a << 20; // undefined behavior

Bit bilge sola kaydırma, özellikle bit bilge atama bileşik operatörü “«=” kullanımına atanmadıkça orijinal değerlerin değerini değiştirmez:

int a = 5;  // 0101b
a <<= 1;    // a = a << 1;

» - sağa kaydırma #

int a = 2;      // 0010b
int b = a >> 1; // 0001b

std::cout << "a = " << a << ", b = " << b << std::endl;

Çıktı

“a = 2, b = 1”

Neden

Sağ bit bilge kaydırma, sol taraftaki değerin (‘a’) bitlerini sağda belirtilen sayıya (‘1’) kaydıracaktır; bir sağa kaydırma işlemi standart olsa da, işaretli bir negatif sayı üzerinde sağa kaydırmanın bitlerine ne olduğu uygulama tanımlıdır ve bu nedenle taşınabilir olması garantilidir, örneğin:

int a = -2;    
int b = a >> 1; // the value of b will be depend on the compiler

Ayrıca, kaydırmak istediğiniz bit sayısının negatif bir sayı olması da tanımsızdır, örneğin:

int a = 1;
int b = a >> -1;  // undefined behavior

Bit bilge sağa kaydırma, özellikle bit düzeyinde atama bileşik operatörü >>= kullanımına atanmadıkça orijinal değerlerin değerini değiştirmez:

int a = 2;  // 0010b
a >>= 1;    // a = a >> 1;

~ - bitsel DEĞİL (tekli tamamlayıcı) #

unsigned char a = 234;  // 1110 1010b  (0xEA)
unsigned char b = ~a;   // 0001 0101b  (0x15)

std::cout << "a = " << static_cast<int>(a) <<
             ", b = " << static_cast<int>(b) << std::endl;

Çıktı

“a = 234, b = 21”

Neden

Biraz bilge ‘NOT’ (birli tamamlayıcı) bit düzeyinde çalışır ve her biti basitçe çevirir. “1” ise “0”, “0” ise “1” olarak değiştirilir. Bit bilge NOT, belirli bir tür için maksimum değere karşı bir değeri XOR’lamakla aynı etkiye sahiptir:

unsigned char a = 234;  // 1110 1010b  (0xEA)
unsigned char b = ~a;   // 0001 0101b  (0x15)
unsigned char c = a ^ ~0;

Bit bilge DEĞİL, belirli bir integral türü için maksimum değeri kontrol etmenin uygun bir yolu olabilir:

unsigned int i = ~0;
unsigned char c = ~0;

std::cout << "max uint = " << i << std::endl <<
             "max uchar = " << static_cast<short>(c) << std::endl;

Bit bilge NOT orijinal değerin değerini değiştirmez ve bileşik atama operatörüne sahip değildir, bu nedenle örneğin a ~= 10 yapamazsınız.

Bit wise NOT (~), mantıksal NOT (!) ile karıştırılmamalıdır; biraz bilge DEĞİL her biti çevirecekse, mantıksal DEĞİL işlemi yapmak için tüm değeri kullanacaktır, başka bir deyişle (!1) != (~1)
