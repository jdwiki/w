C++‘da kullanıcı tanımlı tipler için + ve -> gibi operatörler tanımlamak mümkündür. Örneğin, “” başlığı, dizeleri birleştirmek için bir “+” operatörünü tanımlar. Bu, “operatör” anahtar kelime kullanılarak bir operatör işlevi tanımlanarak yapılır.

Yerleşik türler için operatörler değiştirilemez, operatörler yalnızca kullanıcı tanımlı türler için aşırı yüklenebilir. Yani, işlenenlerden en az biri kullanıcı tanımlı türde olmalıdır.

Aşağıdaki operatörler aşırı yüklenemez:

  • Üye erişimi veya “nokta” operatörü .
  • Üye erişim operatörü .* işaretçisi
  • Kapsam çözümleme operatörü, ::
  • Üçlü koşul operatörü, ?:
  • “Dynamic_cast”, “static_cast”, “reinterpret_cast”, “const_cast”, “typeid”, “sizeof”, “alignof” ve “noexcept”
  • Herhangi bir tür bilgisi mevcut olmadan önce yürütülen # ve ## ön işleme direktifleri.

Aşırı yükleme yapmamanız (zamanın %99,98’i) gereken bazı operatörler vardır:

  • && ve || (bunun yerine, boola örtük dönüştürmeyi kullanmayı tercih edin)
  • ,
  • Operatörün adresi (birli &)

Neden? Niye? Çünkü operatörleri başka bir programcının asla beklemeyeceği şekilde aşırı yüklerler ve bu da beklenenden farklı davranışlara neden olur.

Örneğin, kullanıcı bu operatörlerin && ve || aşırı yüklemelerini kısa devre değerlendirmelerini kaybeder ve kaybetmek onların özel sıralama özellikleri (C++17), sıralama sorunu , operatör aşırı yüklemeleri için de geçerlidir.

Dizi alt simge operatörü #

Hatta dizi indis operatörünü [] aşırı yükleyebilirsiniz.

her zaman (zamanın %99,98’i) bir “const” ve bir “const” olmayan sürüm olmak üzere 2 sürüm uygulamalısınız, çünkü nesne “const” ise, değiştirememesi gerekir ‘[]’ tarafından döndürülen nesne.

Argümanlar değer yerine “const” ile iletilir, çünkü referansa göre geçmek değere göre daha hızlıdır ve “const” ile operatör dizini yanlışlıkla değiştirmez.

Operatörler referansa göre döner, çünkü tasarım gereği [] dönüş nesnesini değiştirebilirsiniz, yani:

std::vector<int> v{ 1 };
v[0] = 2; //Changes value of 1 to 2
          //wouldn't be possible if not returned by reference

Bir “sınıf”/“yapı” içinde yalnızca aşırı yükleme yapabilirsiniz:

//I is the index type, normally an int
T& operator[](const I& index)
{
    //Do something
    //return something
}

//I is the index type, normally an int
const T& operator[](const I& index) const
{
    //Do something
    //return something
}

Çoklu indis operatörleri, [][]..., proxy nesneleri aracılığıyla elde edilebilir. Aşağıdaki basit satır-majör matris sınıfı örneği bunu göstermektedir:

template<class T>
class matrix {
    // class enabling [][] overload to access matrix elements
    template <class C>
    class proxy_row_vector {
        using reference = decltype(std::declval<C>()[0]);
        using const_reference = decltype(std::declval<C const>()[0]);
    public:
        proxy_row_vector(C& _vec, std::size_t _r_ind, std::size_t _cols)
            : vec(_vec), row_index(_r_ind), cols(_cols) {}
        const_reference operator[](std::size_t _col_index) const {
            return vec[row_index*cols + _col_index];
        }
        reference operator[](std::size_t _col_index) {
            return vec[row_index*cols + _col_index];
        }
    private:
        C& vec;
        std::size_t row_index; // row index to access
        std::size_t cols; // number of columns in matrix
    };

    using const_proxy = proxy_row_vector<const std::vector<T>>;
    using proxy = proxy_row_vector<std::vector<T>>;
public:
    matrix() : mtx(), rows(0), cols(0) {}
    matrix(std::size_t _rows, std::size_t _cols)
        : mtx(_rows*_cols), rows(_rows), cols(_cols) {}

    // call operator[] followed by another [] call to access matrix elements
    const_proxy operator[](std::size_t _row_index) const {
        return const_proxy(mtx, _row_index, cols);
    }

    proxy operator[](std::size_t _row_index) {
        return proxy(mtx, _row_index, cols);
    }
private:
    std::vector<T> mtx;
    std::size_t rows;
    std::size_t cols;
};

  1. Orijinal nesneyi değiştiremezsiniz (Nesne foobar = foo + bar; sonuçta fooyu değiştirmemeli, mantıklı olmaz)
  2. Bunu “const” yapamazsınız, çünkü nesneyi değiştirebilmeniz gerekir (çünkü “operatör+”, nesneyi değiştiren “operatör+=” ile uygulanır)

Karmaşık Sayılar Yeniden İncelendi #

Aşağıdaki kod, farklı bir alanın bir üyesiyle dört temel operatörün (+, -, * ve /) uygulanması altında, dilin tür yükseltme kurallarını izleyerek, temel alınan alanın otomatik olarak yükseltildiği çok basit bir karmaşık sayı türünü uygular. (başka bir “karmaşık” veya bir skaler tür olabilir).

Bunun, şablonların temel kullanımının yanı sıra operatör aşırı yüklemesini kapsayan bütünsel bir örnek olması amaçlanmıştır.

#include <type_traits>

namespace not_std{

using std::decay_t;

//----------------------------------------------------------------
// complex< value_t >
//----------------------------------------------------------------

template<typename value_t>
struct complex
{
    value_t x;
    value_t y;

    complex &operator += (const value_t &x)
    {
        this->x += x;
        return *this;
    }
    complex &operator += (const complex &other)
    {
        this->x += other.x;
        this->y += other.y;
        return *this;
    }

    complex &operator -= (const value_t &x)
    {
        this->x -= x;
        return *this;
    }
    complex &operator -= (const complex &other)
    {
        this->x -= other.x;
        this->y -= other.y;
        return *this;
    }

    complex &operator *= (const value_t &s)
    {
        this->x *= s;
        this->y *= s;
        return *this;
    }
    complex &operator *= (const complex &other)
    {
        (*this) = (*this) * other;
        return *this;
    }

    complex &operator /= (const value_t &s)
    {
        this->x /= s;
        this->y /= s;
        return *this;
    }
    complex &operator /= (const complex &other)
    {
        (*this) = (*this) / other;
        return *this;
    }

    complex(const value_t &x, const value_t &y)
    : x{x}
    , y{y}
    {}

    template<typename other_value_t>
    explicit complex(const complex<other_value_t> &other)
    : x{static_cast<const value_t &>(other.x)}
    , y{static_cast<const value_t &>(other.y)}
    {}

    complex &operator = (const complex &) = default;
    complex &operator = (complex &&) = default;
    complex(const complex &) = default;
    complex(complex &&) = default;
    complex() = default;
};

// Absolute value squared
template<typename value_t>
value_t absqr(const complex<value_t> &z)
{ return z.x*z.x + z.y*z.y; }

//----------------------------------------------------------------
// operator - (negation)
//----------------------------------------------------------------

template<typename value_t>
complex<value_t> operator - (const complex<value_t> &z)
{ return {-z.x, -z.y}; }

//----------------------------------------------------------------
// operator +
//----------------------------------------------------------------

template<typename left_t,typename right_t>
auto operator + (const complex<left_t> &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a.x + b.x)>>
{ return{a.x + b.x, a.y + b.y}; }

template<typename left_t,typename right_t>
auto operator + (const left_t &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a + b.x)>>
{ return{a + b.x, b.y}; }

template<typename left_t,typename right_t>
auto operator + (const complex<left_t> &a, const right_t &b)
-> complex<decay_t<decltype(a.x + b)>>
{ return{a.x + b, a.y}; }

//----------------------------------------------------------------
// operator -
//----------------------------------------------------------------

template<typename left_t,typename right_t>
auto operator - (const complex<left_t> &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a.x - b.x)>>
{ return{a.x - b.x, a.y - b.y}; }

template<typename left_t,typename right_t>
auto operator - (const left_t &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a - b.x)>>
{ return{a - b.x, - b.y}; }

template<typename left_t,typename right_t>
auto operator - (const complex<left_t> &a, const right_t &b)
-> complex<decay_t<decltype(a.x - b)>>
{ return{a.x - b, a.y}; }

//----------------------------------------------------------------
// operator *
//----------------------------------------------------------------

template<typename left_t, typename right_t>
auto operator * (const complex<left_t> &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a.x * b.x)>>
{
    return {
        a.x*b.x - a.y*b.y,
        a.x*b.y + a.y*b.x
        };
}

template<typename left_t, typename right_t>
auto operator * (const left_t &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a * b.x)>>
{ return {a * b.x, a * b.y}; }

template<typename left_t, typename right_t>
auto operator * (const complex<left_t> &a, const right_t &b)
-> complex<decay_t<decltype(a.x * b)>>
{ return {a.x * b, a.y * b}; }

//----------------------------------------------------------------
// operator /
//----------------------------------------------------------------

template<typename left_t, typename right_t>
auto operator / (const complex<left_t> &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a.x / b.x)>>
{
    const auto r = absqr(b);
    return {
        ( a.x*b.x + a.y*b.y) / r,
        (-a.x*b.y + a.y*b.x) / r
        };
}

template<typename left_t, typename right_t>
auto operator / (const left_t &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a / b.x)>>
{
    const auto s = a/absqr(b);
    return {
         b.x * s,
        -b.y * s
        };
}

template<typename left_t, typename right_t>
auto operator / (const complex<left_t> &a, const right_t &b)
-> complex<decay_t<decltype(a.x / b)>>
{ return {a.x / b, a.y / b}; }

}// namespace not_std

Tekli operatörler #

2 tekli operatörü aşırı yükleyebilirsiniz:

  • “++foo” ve “foo++”
  • “–foo” ve “foo–”

Aşırı yükleme her iki tür için de aynıdır (++ ve --). Açıklama için aşağı kaydırın

“sınıf”/“yapı” dışında aşırı yükleme:

//Prefix operator ++foo
T& operator++(T& lhs)
{
    //Perform addition
    return lhs;
}

//Postfix operator foo++ (int argument is used to separate pre- and postfix) 
//Should be implemented in terms of ++foo (prefix operator)
T operator++(T& lhs, int)
{
    T t(lhs);
    ++lhs;
    return t;
}

“class”/“struct” içinde aşırı yükleme:

//Prefix operator ++foo
T& operator++()
{
    //Perform addition
    return *this;
}

//Postfix operator foo++ (int argument is used to separate pre- and postfix) 
//Should be implemented in terms of ++foo (prefix operator)
T operator++(int)
{
    T t(*this);
    ++(*this);
    return t;
}

Not: Önek operatörü, üzerinde işlemlere devam edebilmeniz için kendisine bir başvuru döndürür. İlk argüman bir referanstır, önek operatörü nesneyi değiştirdiğinden, “const” olmamasının nedeni de budur (başka türlü değiştiremezsiniz).

Postfix operatörü bir değere göre geçici (önceki değer) döndürür ve bu nedenle geçici bir referans olacağı için bir referans olamaz, bu geçici değişken söndüğü için fonksiyonun sonunda çöp değer olacaktır. kapsamı). Ayrıca ‘const’ olamaz, çünkü onu doğrudan değiştirebilmeniz gerekir.

İlk argüman, “çağıran” nesneye “const” olmayan bir referanstır, çünkü “const” olsaydı, onu değiştiremezdiniz ve bir referans olmasaydı, değiştiremezdiniz. orijinal değer.

Postfix operatörü aşırı yüklemelerinde gereken kopyalama nedeniyle, ‘for’ döngülerinde postfix ++ yerine ++ önekini kullanmayı alışkanlık haline getirmek daha iyidir. ‘for’ döngüsü perspektifinden, genellikle işlevsel olarak eşdeğerdirler, ancak ++ önekini kullanmanın, özellikle kopyalanacak çok sayıda üyeye sahip “yağ” sınıflarında, küçük bir performans avantajı olabilir. Bir for döngüsünde ++ önekini kullanma örneği:

for (list<string>::const_iterator it = tokens.begin();
     it != tokens.end();
     ++it) { // Don't use it++
    ...
}

Karşılaştırma operatörleri #

Tüm karşılaştırma operatörlerini aşırı yükleyebilirsiniz:

  • == ve !=
  • > ve <
  • >= ve <=

Tüm bu operatörleri aşırı yüklemenin önerilen yolu, yalnızca 2 operatör (== ve <) uygulamak ve ardından kalanını tanımlamak için bunları kullanmaktır. Açıklama için aşağı kaydırın

“sınıf”/“yapı” dışında aşırı yükleme:

//Only implement those 2
bool operator==(const T& lhs, const T& rhs) { /* Compare */ }
bool operator<(const T& lhs, const T& rhs) { /* Compare */ }

//Now you can define the rest
bool operator!=(const T& lhs, const T& rhs) { return !(lhs == rhs); }
bool operator>(const T& lhs, const T& rhs) { return rhs < lhs; }
bool operator<=(const T& lhs, const T& rhs) { return !(lhs > rhs); }
bool operator>=(const T& lhs, const T& rhs) { return !(lhs < rhs); }

“class”/“struct” içinde aşırı yükleme:

//Note that the functions are const, because if they are not const, you wouldn't be able
//to call them if the object is const

//Only implement those 2
bool operator==(const T& rhs) const { /* Compare */ }
bool operator<(const T& rhs) const { /* Compare */ }

//Now you can define the rest
bool operator!=(const T& rhs) const { return !(*this == rhs); }
bool operator>(const T& rhs) const { return rhs < *this; }
bool operator<=(const T& rhs) const { return !(*this > rhs); }
bool operator>=(const T& rhs) const { return !(*this < rhs); }

Operatörler açıkça, ilgili işlem için “doğru” veya “yanlış"ı belirten bir “bool” döndürür.

Tüm operatörler argümanlarını const& ile alır, çünkü operatörlerin yaptığı tek şey karşılaştırmaktır, bu yüzden nesneleri değiştirmemeleri gerekir. “&” (referans) ile geçmek değere göre daha hızlıdır ve operatörlerin onu değiştirmediğinden emin olmak için bir “const” referansıdır.

‘class’/‘struct’ içindeki operatörlerin ‘const’ olarak tanımlandığını unutmayın, bunun nedeni, fonksiyonlar ‘const’ olmadan, derleyici olmadığı için ‘const’ nesnelerini karşılaştırmanın mümkün olmayacağıdır. Operatörlerin hiçbir şeyi değiştirmediğini bilin.

Atama operatörü #

Atama operatörü, bir değişkenin durumunu değiştirmenize izin verdiği için en önemli operatörlerden biridir.

Eğer sınıf/yapınız için atama operatörünü aşırı yüklemezseniz, bu otomatik olarak derleyici tarafından oluşturulur: otomatik olarak oluşturulan atama operatörü bir “üyesel atama” gerçekleştirir, yani tüm üyelerde atama operatörlerini çağırarak, böylece bir nesne diğerine kopyalanır, zaman zaman üyedir. Örneğin, bir nesnenin derin bir kopyasını yapmanız gerekiyorsa, basit üye bazında atama ‘sınıfınız’/‘yapınız’ için uygun olmadığında, atama operatörü aşırı yüklenmelidir.

Atama operatörünü = aşırı yüklemek kolaydır, ancak bazı basit adımları izlemelisiniz.

  1. Kendi kendine atama testi. Bu kontrol iki nedenden dolayı önemlidir:

    • a self-assignment is a needless copy, so it does not make sense to perform it;
    • the next step will not work in the case of a self-assignment.

  2. Eski verileri temizleyin. Eski veriler yenileriyle değiştirilmelidir. Şimdi, önceki adımın ikinci nedenini anlayabilirsiniz: nesnenin içeriği yok edildiyse, kendi kendine atama kopyalamayı gerçekleştiremez.

  3. Tüm üyeleri kopyalayın. “Sınıfınız” veya “yapınız” için atama operatörünü aşırı yüklerseniz, bu, derleyici tarafından otomatik olarak oluşturulmaz, bu nedenle tüm üyeleri diğerinden kopyalama sorumluluğunu üstlenmeniz gerekir. nesne.

  4. Geri *bu. Operatör, zincirlemeye izin verdiği için referansla kendi başına döner (yani, int b = (a = 6) + 4; //b == 10).

    //T is some type T& operator=(const T& other) { //Do something (like copying values) return *this; }

Not: ‘diğer’, ‘const&’ tarafından geçirilir, çünkü atanan nesne değiştirilmemelidir ve referansa göre geçiş değere göre daha hızlıdır ve ‘operatör=‘den daha fazlasını değiştirmediğinden emin olmak için yanlışlıkla “const” olur.

Atama operatörü yalnızca ‘sınıf’/‘yapı’ içinde aşırı yüklenebilir, çünkü ‘=’ öğesinin sol değeri her zaman ‘sınıf’/‘yapı’nın kendisidir. Bunu ücretsiz bir işlev olarak tanımlamak bu garantiye sahip değildir ve bu nedenle izin verilmez.

Bunu ‘class’/‘struct’ içinde bildirdiğinizde, soldaki değer örtük olarak ‘class’/‘struct’ın kendisidir, dolayısıyla bununla ilgili bir sorun yoktur.

Adlandırılmış operatörler #

C++‘ı standart C++ işleçleri tarafından “alıntılanan” adlandırılmış işleçlerle genişletebilirsiniz.

İlk önce düzine satırlık bir kitaplık ile başlıyoruz:

namespace named_operator {
  template<class D>struct make_operator{constexpr make_operator(){}};

  template<class T, char, class O> struct half_apply { T&& lhs; };

  template<class Lhs, class Op>
  half_apply<Lhs, '*', Op> operator*( Lhs&& lhs, make_operator<Op> ) {
    return {std::forward<Lhs>(lhs)};
  }

  template<class Lhs, class Op, class Rhs>
  auto operator*( half_apply<Lhs, '*', Op>&& lhs, Rhs&& rhs )
  -> decltype( named_invoke( std::forward<Lhs>(lhs.lhs), Op{}, std::forward<Rhs>(rhs) ) )
  {
    return named_invoke( std::forward<Lhs>(lhs.lhs), Op{}, std::forward<Rhs>(rhs) );
  }
}

bu henüz bir şey yapmıyor.

İlk olarak, vektörleri ekleyerek

namespace my_ns {
  struct append_t : named_operator::make_operator<append_t> {};
  constexpr append_t append{};
  
  template<class T, class A0, class A1>
  std::vector<T, A0> named_invoke( std::vector<T, A0> lhs, append_t, std::vector<T, A1> const& rhs ) {
      lhs.insert( lhs.end(), rhs.begin(), rhs.end() );
      return std::move(lhs);
  }
}

std::vector<int> a {1,2,3};
std::vector<int> b {4,5,6};

auto c = a *append* b;

Buradaki çekirdek, ‘append_t:named_operator::make_operator<append_t>’ türünde bir ‘append’ nesnesi tanımlamamızdır.

Daha sonra sağda ve solda istediğimiz türler için name_invoke( lhs, append_t, rhs ) aşırı yükleniyoruz.

Kitaplık “lhsappend_t” öğesini aşırı yükleyerek geçici bir “half_apply” nesnesi döndürür. Ayrıca, “named_invoke( lhs, append_t, rhs )” öğesini çağırmak için “half_applyrhs” öğesini aşırı yükler.

Sadece uygun ‘append_t’ belirtecini oluşturmamız ve uygun imzanın ADL dostu ’named_invoke’unu yapmamız gerekiyor ve her şey bağlanıyor ve çalışıyor.

Daha karmaşık bir örnek için, bir std::array öğelerinin öğe bazında çarpımına sahip olmak istediğinizi varsayalım:

template<class=void, std::size_t...Is>
auto indexer( std::index_sequence<Is...> ) {
  return [](auto&& f) {
    return f( std::integral_constant<std::size_t, Is>{}... );
  };
}
template<std::size_t N>
auto indexer() { return indexer( std::make_index_sequence<N>{} ); }

namespace my_ns {
  struct e_times_t : named_operator::make_operator<e_times_t> {};
  constexpr e_times_t e_times{};

  template<class L, class R, std::size_t N,
    class Out=std::decay_t<decltype( std::declval<L const&>()*std::declval<R const&>() )>
  >
  std::array<Out, N> named_invoke( std::array<L, N> const& lhs, e_times_t, std::array<R, N> const& rhs ) {
    using result_type = std::array<Out, N>;
    auto index_over_N = indexer<N>();
    return index_over_N([&](auto...is)->result_type {
      return {{
        (lhs[is] * rhs[is])...
      }};
    });
  }
}

Yukarıdaki kitaplıkta küçük bir değişiklikle, ->*then* operatörlerini destekleyebilir ve standart güncellenmeden önce std::function genişletebilir veya monadik ->*bind* yazabiliriz. Ayrıca, aşağıdakilere izin vererek, “Op"u dikkatlice son çağırma işlevine aktardığımız, durum bilgisi olan bir adlandırılmış operatöre sahip olabilir:

named_operator<'*'> append = [](auto lhs, auto&& rhs) {
  using std::begin; using std::end;
  lhs.insert( end(lhs), begin(rhs), end(rhs) );
  return std::move(lhs);
};

C++17’de adlandırılmış bir kapsayıcı ekleme operatörü oluşturma.

İşlev çağrısı operatörü #

() işlev çağrısı operatörünü aşırı yükleyebilirsiniz:

Aşırı yükleme bir “sınıf”/“yapı” içinde yapılmalıdır:

//R -> Return type
//Types -> any different type
R operator()(Type name, Type2 name2, ...)
{
    //Do something
    //return something
}

//Use it like this (R is return type, a and b are variables)
R foo = object(a, b, ...);

Örneğin:

struct Sum
{
    int operator()(int a, int b)
    {
        return a + b;
    }
};

//Create instance of struct
Sum sum;
int result = sum(1, 1); //result == 2

Bitsel DEĞİL operatörü #

Bitsel NOT’u (~) aşırı yüklemek oldukça basittir. Açıklama için aşağı kaydırın

“sınıf”/“yapı” dışında aşırı yükleme:

T operator~(T lhs)
{
    //Do operation
    return lhs;
}

“class”/“struct” içinde aşırı yükleme:

T operator~()
{
    T t(*this);
    //Do operation
    return t;
}

Not: “operatör~” değere göre döner, çünkü yeni bir değer (değiştirilen değer) döndürmesi gerekir, değere bir referans değil (bu, içinde çöp değeri olan geçici nesneye bir referans olacaktır). operatörün işi biter bitmez). ‘const’ değil, çünkü çağıran kodun daha sonra değiştirebilmesi gerekir (yani, ‘int a = ~a + 1;’ mümkün olmalıdır).

class/struct içinde geçici bir nesne yapmanız gerekir, çünkü orijinal nesneyi değiştireceği için `bunu’ değiştiremezsiniz, ki bu böyle olmamalıdır.

G/Ç için bit kaydırma operatörleri #

<< ve >> operatörleri genellikle “yazma” ve “okuma” operatörleri olarak kullanılır:

  • ‘std::ostream’, temel alınan akışa değişkenler yazmak için ‘«’ öğesini aşırı yükler (örnek: ‘std::cout’)
  • ‘std::istream’, temel akıştan bir değişkene okumak için ‘»’ öğesini aşırı yükler (örnek: ‘std::cin’)

Argümanları belirtmenin aynı türden olmaması dışında, bunları “sınıf”/“yapı” dışında “normalde” aşırı yüklemek istiyorsanız, bunu yapma biçimleri benzerdir:

  • Döndürme türü, zincirlemeye izin vermek için (Zincirleme: std::cout << a << b;) referans yoluyla iletilen aşırı yüklemek istediğiniz akıştır (örneğin, std::ostream). Örnek: std::ostream&
  • ’lhs’ dönüş türüyle aynı olur
  • “rhs”, aşırı yüklemeye izin vermek istediğiniz türdür (yani “T”), performans nedeniyle değer yerine “const” ile iletilir (“rhs” zaten değiştirilmemelidir). Örnek: const Vector&.

Örnek:

//Overload std::ostream operator<< to allow output from Vector's
std::ostream& operator<<(std::ostream& lhs, const Vector& rhs)
{
    lhs << "x: " << rhs.x << " y: " << rhs.y << " z: " << rhs.z << '\n';
    return lhs;
}

Vector v = { 1, 2, 3};

//Now you can do
std::cout << v;
