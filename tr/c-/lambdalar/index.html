Sözdizimi #

  • [default-capture, capture-list] (argument-list) değiştirilebilir throw-specation nitelikler -> return-type { lambda-body } // lambda belirteçlerinin sırası ve nitelikler.
  • [capture-list] (argument-list) { lambda-body } // Ortak lambda tanımı.
  • [=] (argument-list) { lambda-body } // Gerekli tüm yerel değişkenleri değere göre yakalar.
  • [&] (argument-list) { lambda-body } // Gerekli tüm yerel değişkenleri referansa göre yakalar.
  • [capture-list] { lambda-body } // Argüman listesi ve belirteçler atlanabilir.

Parametreler #

parametre Ayrıntılar
varsayılan yakalama Listelenmemiş tüm değişkenlerin nasıl yakalandığını belirtir. = (değere göre yakala) veya & (referansa göre yakala) olabilir. Atlanırsa, listelenmemiş değişkenlere lambda-body içinde erişilemez. varsayılan-yakalama, yakalama-listesinden önce gelmelidir.
yakalama listesi lambda-body içinde yerel değişkenlerin nasıl erişilebilir hale getirileceğini belirtir. Ön eki olmayan değişkenler değere göre yakalanır. Ön eki “&” olan değişkenler referansla yakalanır. Bir sınıf yöntemi içinde, tüm üyelerini referans yoluyla erişilebilir kılmak için ‘bu’ kullanılabilir. Listeden önce varsayılan-yakalama gelmedikçe, listelenmeyen değişkenlere erişilemez.
argüman listesi Lambda işlevinin bağımsız değişkenlerini belirtir.
değişken (isteğe bağlı) Normalde değer tarafından yakalanan değişkenler ‘const’tur. “Değişken"in belirtilmesi, onları sabit olmayan hale getirir. Bu değişkenlerde yapılan değişiklikler çağrılar arasında korunur.
atma özelliği (isteğe bağlı) Lambda işlevinin istisna atma davranışını belirtir. Örneğin: “noexcept” veya “throw(std::exception)”.
nitelikler (isteğe bağlı) Lambda işlevi için herhangi bir nitelik. Örneğin, lambda-body her zaman bir istisna atarsa, `[[noreturn]]’ kullanılabilir.
-> dönüş tipi (isteğe bağlı) Lambda işlevinin dönüş türünü belirtir. Dönüş türü derleyici tarafından belirlenemediğinde gereklidir.
lambda gövdesi Lambda işlevinin uygulanmasını içeren bir kod bloğu.

C++17 (geçerli taslak), temel olarak derleme zamanında değerlendirilebilen lambdalar olan “constexpr” lambdaları tanıtır. Bir lambda, “constexpr” gereksinimlerini karşılıyorsa otomatik olarak “constexpr” olur, ancak bunu “constexpr” anahtar sözcüğünü kullanarak da belirtebilirsiniz:

//Explicitly define this lambdas as constexpr
[]() constexpr {
    //Do stuff
}

Lambda ifadesi nedir? #

Bir lambda ifadesi, basit işlev nesneleri oluşturmanın kısa ve öz bir yolunu sağlar. Bir lambda ifadesi, sonuç nesnesi kapatma nesnesi olarak adlandırılan ve bir işlev nesnesi gibi davranan bir değerdir.

‘Lambda ifadesi’ adı, 1930’larda Alonzo Church tarafından mantık ve hesaplanabilirlik ile ilgili soruları araştırmak için icat edilen matematiksel bir formalizm olan lambda hesabından gelmektedir. Lambda hesabı, işlevsel bir programlama dili olan LISP temelini oluşturdu. Lambda hesabı ve LISP ile karşılaştırıldığında, C++ lambda ifadeleri adsız olma ve çevreleyen bağlamdan değişkenleri yakalama özelliklerini paylaşır, ancak işlevleri üzerinde çalışma ve döndürme yeteneklerinden yoksundur.

Bir lambda ifadesi genellikle çağrılabilir bir nesne alan işlevlere argüman olarak kullanılır. Bu, yalnızca argüman olarak iletildiğinde kullanılacak olan adlandırılmış bir işlev oluşturmaktan daha basit olabilir. Bu gibi durumlarda, fonksiyon nesnelerinin satır içinde tanımlanmasına izin verdikleri için genellikle lambda ifadeleri tercih edilir.

Bir lambda tipik olarak üç bölümden oluşur: bir yakalama listesi “[]”, isteğe bağlı bir parametre listesi “()” ve tümü boş olabilen bir gövde “{}”:

[](){}                // An empty lambda, which does and returns nothing

Yakalama listesi

“[]”, yakalama listesidir. Varsayılan olarak, çevreleyen kapsamın değişkenlerine bir lambda tarafından erişilemez. Bir değişkeni yakalamak, onu lambda içinde kopya olarak veya referans olarak erişilebilir hale getirir. Yakalanan değişkenler lambda’nın bir parçası haline gelir; fonksiyon argümanlarının aksine, lambda çağrılırken iletilmeleri gerekmez.

int a = 0;                       // Define an integer variable
auto f = []()   { return a*9; }; // Error: 'a' cannot be accessed
auto f = [a]()  { return a*9; }; // OK, 'a' is "captured" by value
auto f = [&a]() { return a++; }; // OK, 'a' is "captured" by reference
                                 //      Note: It is the responsibility of the programmer
                                 //      to ensure that a is not destroyed before the
                                 //      lambda is called.
auto b = f();                    // Call the lambda function. a is taken from the capture list and not passed here.

Parametre listesi

(), normal işlevlerle neredeyse aynı olan parametre listesidir. Lambda hiçbir argüman almıyorsa, bu parantezler atlanabilir (lambda’yı ‘değişebilir’ olarak bildirmeniz gerekmedikçe). Bu iki lambda eşdeğerdir:

auto call_foo  = [x](){ x.foo(); };
auto call_foo2 = [x]{ x.foo(); };

Parametre listesi, gerçek türler yerine “auto” yer tutucu türünü kullanabilir. Bunu yaparak, bu argüman bir fonksiyon şablonunun şablon parametresi gibi davranır. Bir vektörü genel kodda sıralamak istediğinizde aşağıdaki lambdalar eşdeğerdir:

auto sort_cpp11 = [](std::vector<T>::const_reference lhs, std::vector<T>::const_reference rhs) { return lhs < rhs; }; 
auto sort_cpp14 = [](const auto &lhs, const auto &rhs) { return lhs < rhs; };
son sürüm

Fonksiyon gövdesi

{}, normal işlevlerdekiyle aynı olan gövde‘dir.

Lambda çağırma

Bir lambda ifadesinin sonuç nesnesi, “operatör()” kullanılarak çağrılabilen bir kapatma‘dir (diğer işlev nesnelerinde olduğu gibi):

int multiplier = 5;
auto timesFive = [multiplier](int a) { return a * multiplier; }; 
std::out << timesFive(2); // Prints 10

multiplier = 15;
std::out << timesFive(2); // Still prints 2*5 == 10

İade Türü

Varsayılan olarak, bir lambda ifadesinin dönüş türü çıkarılır.

[](){ return true; };

Bu durumda dönüş türü “bool"dur.

Aşağıdaki sözdizimini kullanarak dönüş türünü manuel olarak da belirleyebilirsiniz:

[]() -> bool { return true; };

Değişken Lambda

Lambda’da değere göre yakalanan nesneler varsayılan olarak değişmezdir. Bunun nedeni, oluşturulan kapatma nesnesinin “operatör()“ünün varsayılan olarak “const” olmasıdır.

auto func = [c = 0](){++c; std::cout << c;};  // fails to compile because ++c
                                              // tries to mutate the state of
                                              // the lambda.

Daha yakın nesnenin “operator()” işlevini “const” olmayan hale getiren “mutable” anahtar sözcüğü kullanılarak değiştirmeye izin verilebilir:

auto func = [c = 0]() mutable {++c; std::cout << c;};

Dönüş türü ile birlikte kullanılırsa, ‘mutable’ ondan önce gelir.

auto func = [c = 0]() mutable -> int {++c; std::cout << c; return c;};

Lambdaların kullanışlılığını gösteren bir örnek

C++ 11’den önce:

// Generic functor used for comparison
struct islessthan
{
    islessthan(int threshold) : _threshold(threshold) {}

    bool operator()(int value) const
    {
        return value < _threshold;
    }
private:
    int _threshold;
};

// Declare a vector
const int arr[] = { 1, 2, 3, 4, 5 };
std::vector<int> vec(arr, arr+5);

// Find a number that's less than a given input (assume this would have been function input)
int threshold = 10;
std::vector<int>::iterator it = std::find_if(vec.begin(), vec.end(), islessthan(threshold));
son sürüm

C++11’den beri:

// Declare a vector
std::vector<int> vec{ 1, 2, 3, 4, 5 };

// Find a number that's less than a given input (assume this would have been function input)
int threshold = 10;
auto it = std::find_if(vec.begin(), vec.end(), [threshold](int value) { return value < threshold; });

Dönüş türünü belirtme #

Tek bir dönüş ifadesine sahip lambdalar veya ifadeleri aynı türden olan birden çok dönüş ifadesine sahip lambdalar için, derleyici dönüş türünü belirleyebilir:

// Returns bool, because "value > 10" is a comparison which yields a Boolean result
auto l = [](int value) {
    return value > 10;
}

Farklı türlerde birden çok dönüş ifadesi içeren lambdalar için, derleyici dönüş türünü çıkaramaz:

// error: return types must match if lambda has unspecified return type
auto l = [](int value) {
    if (value < 10) {
        return 1;
    } else {
        return 1.5;
    }
};

Bu durumda dönüş türünü açıkça belirtmeniz gerekir:

// The return type is specified explicitly as 'double'
auto l = [](int value) -> double {
    if (value < 10) {
        return 1;
    } else {
        return 1.5;
    }
};

Bunun kuralları, “otomatik” tür kesinti kurallarıyla eşleşir. Açıkça belirtilen dönüş türleri olmayan Lambda’lar asla referans döndürmez, bu nedenle bir referans türü isteniyorsa bunun da açıkça belirtilmesi gerekir:

auto copy = [](X& x) { return x; };       // 'copy' returns an X, so copies its input
auto ref  = [](X& x) -> X& { return x; }; // 'ref' returns an X&, no copy

Değere göre yakalama #

Yakalama listesinde değişkenin adını belirtirseniz, lambda onu değere göre yakalayacaktır. Bu, lambda için oluşturulan kapatma türünün değişkenin bir kopyasını sakladığı anlamına gelir. Bu ayrıca değişkenin türünün kopyalanabilir olmasını gerektirir:

int a = 0;

[a]() {
    return a;   // Ok, 'a' is captured by value
};

auto p = std::unique_ptr<T>(...);

[p]() {         // Compile error; `unique_ptr` is not copy-constructible
    return p->createWidget(); 
};
son sürüm

C++ 14’ten itibaren, değişkenleri yerinde başlatmak mümkündür. Bu, lambda’da yalnızca hareket türlerinin yakalanmasına izin verir.

auto p = std::make_unique<T>(...);

[p = std::move(p)]() {
    return p->createWidget(); 
};
son sürüm

Bir lambda, adlarıyla verildiğinde değişkenleri değerine göre yakalasa da, bu tür değişkenler varsayılan olarak lambda gövdesi içinde değiştirilemez. Bunun nedeni, kapatma türünün lambda gövdesini bir “operator() const” bildirimine koymasıdır.

‘const’, kapatma türündeki üye değişkenlere ve kapatmanın üyeleri olan yakalanan değişkenlere erişim için geçerlidir (tüm görünümler tam tersidir):

int a = 0;

[a]() {
    a = 2;      // Illegal, 'a' is accessed via `const`

    decltype(a) a1 = 1; 
    a1 = 2; // valid: variable 'a1' is not const
};

‘const’u kaldırmak için lambda üzerinde ‘mutable’ anahtar kelimesini belirtmeniz gerekir:

int a = 0;

[a]() mutable {
    a = 2;      // OK, 'a' can be modified
    return a;
};

“a” değeri tarafından yakalandığından, lambda çağrılarak yapılan herhangi bir değişiklik “a"yı etkilemeyecektir. “a” değeri, oluşturulduğunda lambdaya kopyalandı, bu nedenle lambda’nın “a” kopyası, harici “a” değişkeninden ayrıdır.

int a = 5 ; 
auto plus5Val = [a] (void) { return a + 5 ; } ; 
auto plus5Ref = [&a] (void) {return a + 5 ; } ; 

a = 7 ; 
std::cout << a << ", value " << plus5Val() << ", reference " << plus5Ref() ;
// The result will be "7, value 10, reference 12"

Özyinelemeli lambdalar #

Diyelim ki Öklid’in gcd() sini lambda olarak yazmak istiyoruz. Bir fonksiyon olarak, şudur:

int gcd(int a, int b) {
    return b == 0 ? a : gcd(b, a%b);
}

Ancak bir lambda özyinelemeli olamaz, kendisini çağırmanın bir yolu yoktur. Bir lambdanın adı yoktur ve bir lambda gövdesi içinde ’this’ kullanılması, yakalanan bir ‘bu’ anlamına gelir (lambda’nın bir üye işlevin gövdesinde oluşturulduğu varsayılır, aksi takdirde bu bir hatadır). Peki bu sorunu nasıl çözeceğiz?

std::function kullanın #

Henüz oluşturulmamış bir std::function referansını bir lambda yakalamasını sağlayabiliriz:

std::function<int(int, int)> gcd = [&](int a, int b){
    return b == 0 ? a : gcd(b, a%b);
};

Bu işe yarar, ancak dikkatli kullanılmalıdır. Yavaştır (şimdi doğrudan işlev çağrısı yerine tür silme kullanıyoruz), kırılgandır (“gcd"yi kopyalamak veya “gcd"yi döndürmek, lambda orijinal nesneye atıfta bulunduğundan bozulur) ve genel ile çalışmaz lambdalar.

İki akıllı işaretçi kullanma: #

auto gcd_self = std::make_shared<std::unique_ptr< std::function<int(int, int)> >>();
*gcd_self = std::make_unique<std::function<int(int, int)>>(
  [gcd_self](int a, int b){
    return b == 0 ? a : (**gcd_self)(b, a%b);
  };
};

Bu, çok fazla dolaylı yol ekler (ki bu ek yüktür), ancak kopyalanabilir/iade edilebilir ve tüm kopyalar durumu paylaşır. Lambda’yı iade etmenize izin verir ve aksi takdirde yukarıdaki çözümden daha az kırılgandır.

Y-birleştirici kullanın #

Kısa bir yardımcı program yapısı yardımıyla tüm bu sorunları çözebiliriz:

template <class F>
struct y_combinator {
    F f; // the lambda will be stored here

    // a forwarding operator():
    template <class... Args>
    decltype(auto) operator()(Args&&... args) const {
        // we pass ourselves to f, then the arguments.
        // the lambda should take the first argument as `auto&& recurse` or similar.
        return f(*this, std::forward<Args>(args)...);
    }
};
// helper function that deduces the type of the lambda:
template <class F>
y_combinator<std::decay_t<F>> make_y_combinator(F&& f) {
    return {std::forward<F>(f)};
}
// (Be aware that in C++17 we can do better than a `make_` function)

gcd’mizi şu şekilde uygulayabiliriz:

auto gcd = make_y_combinator(
  [](auto&& gcd, int a, int b){
    return b == 0 ? a : gcd(b, a%b);
  }
);

“y_combinator”, siz tanımlanana kadar kendinizi adlandırmadan özyineleme yapmanızı sağlayan lambda hesabından bir kavramdır. Bu tam olarak lambdaların sahip olduğu sorundur.

İlk argümanı olarak “özyineleme” alan bir lambda yaratırsınız. Özyinelemek istediğinizde, özyinelemeye argümanları iletirsiniz.

“y_combinator” daha sonra argümanlarıyla, ancak ilk argümanı olarak uygun bir “recurse” nesnesiyle (yani “y_combinator"ın kendisi) bu işlevi çağıran bir işlev nesnesi döndürür. “y_combinator” olarak adlandırdığınız argümanların geri kalanını da lambda’ya iletir.

Kısacası:

auto foo = make_y_combinator( [&](auto&& recurse, some arguments) {
  // write body that processes some arguments
  // when you want to recurse, call recurse(some other arguments)
});

ve ciddi bir kısıtlama veya önemli bir ek yük olmadan bir lambdada özyinelemeniz var.

Varsayılan yakalama #

Varsayılan olarak, yakalama listesinde açıkça belirtilmeyen yerel değişkenlere lambda gövdesi içinden erişilemez. Ancak, lambda gövdesi tarafından adlandırılan değişkenleri örtük olarak yakalamak mümkündür:

int a = 1;
int b = 2;

// Default capture by value
[=]() { return a + b; }; // OK; a and b are captured by value

// Default capture by reference
[&]() { return a + b; }; // OK; a and b are captured by reference

Açık yakalama, örtük varsayılan yakalamanın yanı sıra yine de yapılabilir. Açık yakalama tanımı, varsayılan yakalamayı geçersiz kılar:

int a = 0;
int b = 1;

[=, &b]() {
    a = 2; // Illegal; 'a' is capture by value, and lambda is not 'mutable'
    b = 2; // OK; 'b' is captured by reference
};

Referansa göre yakalayın #

Yerel bir değişkenin adının önüne bir “&” koyarsanız, değişken referansla yakalanacaktır. Kavramsal olarak bu, lambda’nın kapatma tipinin, lambda kapsamı dışından karşılık gelen değişkene referans olarak başlatılan bir referans değişkenine sahip olacağı anlamına gelir. Değişkenin lambda gövdesindeki herhangi bir kullanımı, orijinal değişkene atıfta bulunacaktır:

// Declare variable 'a'
int a = 0;

// Declare a lambda which captures 'a' by reference
auto set = [&a]() {
    a = 1;
};

set();
assert(a == 1);

“a"nın kendisi “const” olmadığı için “mutable” anahtar kelimesine gerek yoktur.

Elbette, referans yoluyla yakalama, lambda’nın yakaladığı değişkenlerin kapsamından kaçmaması gerektiği anlamına gelir. Böylece bir işlev alan işlevleri çağırabilirsiniz, ancak lambda’yı referanslarınızın kapsamı dışında saklayacak bir işlevi çağırmamalısınız. Ve lambdayı iade etmemelisiniz.

Genel lambdalar #

Lambda işlevleri, isteğe bağlı türlerin bağımsız değişkenlerini alabilir. Bu, bir lambda’nın daha genel olmasını sağlar:

auto twice = [](auto x){ return x+x; };

int i = twice(2); // i == 4
std::string s = twice("hello"); // s == "hellohello"

Bu, kapatma tipinin operator() aşırı yüklemesini bir şablon işlevi yaparak C++‘da gerçekleştirilir. Aşağıdaki tür, yukarıdaki lambda kapatma işlemine eşdeğer davranışa sahiptir:

struct _unique_lambda_type
{
  template<typename T>
  auto operator() (T x) const {return x + x;}
};

Genel bir lambdadaki tüm parametrelerin genel olması gerekmez:

[](auto x, int y) {return x + y;}

Burada “x”, ilk işlev argümanına dayalı olarak çıkarılırken “y” her zaman “int” olacaktır.

Genel lambdalar, ‘auto’ ve ‘&’ için olağan kuralları kullanarak, argümanları referansa göre de alabilir. Genel bir parametre auto&& olarak alınırsa, bu argümanda geçirilen bir forwarding referansı ve bir rvalue referansı değil:

auto lamb1 = [](int &&x) {return x + 5;};
auto lamb2 = [](auto &&x) {return x + 5;};
int x = 10;
lamb1(x); // Illegal; must use `std::move(x)` for `int&&` parameters.
lamb2(x); // Legal; the type of `x` is deduced as `int&`.

Lambda işlevleri değişken olabilir ve argümanlarını mükemmel bir şekilde iletebilir:

auto lam = [](auto&&... args){return f(std::forward<decltype(args)>(args)...);};

veya:

auto lam = [](auto&&... args){return f(decltype(args)(args)...);};

bu, yalnızca auto&& türündeki değişkenlerle “düzgün” çalışır.

Jenerik lambdaları kullanmanın güçlü bir nedeni, sözdizimini ziyaret etmektir.

boost::variant<int, double> value;
apply_visitor(value, [&](auto&& e){
  std::cout << e;
});

Burada polimorfik bir şekilde ziyaret ediyoruz; ancak diğer bağlamlarda, geçmekte olduğumuz türün adları ilginç değildir:

mutex_wrapped<std::ostream&> os = std::cout;
os.write([&](auto&& os){
  os << "hello world\n";
});

‘std::ostream’ türünün tekrarlanması burada gürültüdür; her kullandığınızda bir değişkenin türünden bahsetmek zorunda kalmak gibi olurdu. Burada bir ziyaretçi oluşturuyoruz, ancak polimorfik bir ziyaretçi yaratmıyoruz; “auto”, “for(:)” döngüsünde “auto"yu kullanmanızla aynı nedenle kullanılır.

son sürüm

Sınıf lambdaları ve bunun yakalanması #

Bir sınıfın üye işlevinde değerlendirilen bir lambda ifadesi, dolaylı olarak o sınıfın bir arkadaşıdır:

class Foo
{
private:
    int i;
    
public:
    Foo(int val) : i(val) {}
    
    // definition of a member function
    void Test()
    {
        auto lamb = [](Foo &foo, int val)
        {
            // modification of a private member variable
            foo.i = val;
        };
        
        // lamb is allowed to access a private member, because it is a friend of Foo
        lamb(*this, 30);
    }
};

Böyle bir lambda yalnızca o sınıfın bir arkadaşı değildir, içinde bildirildiği sınıfla aynı erişime sahiptir.

Lambda’lar, dış işlevin çağrıldığı nesne örneğini temsil eden ’this’ işaretçisini yakalayabilir. Bu, yakalama listesine “this” eklenerek yapılır:

class Foo
{
private:
    int i;
    
public:
    Foo(int val) : i(val) {}
    
    void Test()
    {
        // capture the this pointer by value
        auto lamb = [this](int val)
        {
            i = val;
        };
        
        lamb(30);
    }
};

‘Bu’ yakalandığında, lambda, içerdiği sınıfın üye adlarını, sanki kendi içeren sınıfındaymış gibi kullanabilir. Bu nedenle, bu tür üyelere örtük bir ‘bu->’ uygulanır.

“bu"nun değere göre tutulduğunu, ancak türün değeriyle yakalanmadığını unutmayın. Bir işaretçi olan ’this’ değeriyle yakalanır. Bu nedenle, lambda “bu"nun *sahibi değildir. Lambda dışarı onu yaratan nesnenin ömrünü yaşıyorsa, lambda geçersiz hale gelebilir.

Bu aynı zamanda lambda’nın “değişebilir” olarak bildirilmeden “bunu” değiştirebileceği anlamına gelir. İşaret edilen nesne değil, “const” olan işaretçidir. Yani, dış üye işlevinin kendisi bir “const” işlevi olmadığı sürece.

Ayrıca, hem “[=]” hem de “[&]” olan varsayılan yakalama yan tümcelerinin, “bunu” örtük olarak ayrıca da yakalayacağını unutmayın. Ve ikisi de onu işaretçinin değerine göre yakalar. Aslında, bir varsayılan verildiğinde yakalama listesinde ‘bu’nun belirtilmesi bir hatadır.

Lambda’lar, lambda oluşturulduğunda oluşturulan ’this’ nesnesinin bir kopyasını yakalayabilir. Bu, yakalama listesine *this eklenerek yapılır:

class Foo
{
private:
    int i;
    
public:
    Foo(int val) : i(val) {}
    
    void Test()
    {
        // capture a copy of the object given by the this pointer
        auto lamb = [*this](int val) mutable
        {
            i = val;
        };
        
        lamb(30); // does not change this->i
    }
};
son sürüm

Genelleştirilmiş yakalama #

Lambdalar yalnızca değişkenler yerine ifadeleri yakalayabilir. Bu, lambdaların yalnızca hareket türlerini saklamasına izin verir:

auto p = std::make_unique<T>(...);

auto lamb = [p = std::move(p)]() //Overrides capture-by-value of `p`.
{
  p->SomeFunc();
};

Bu, dış “p” değişkenini “p” olarak da adlandırılan lambda yakalama değişkenine taşır. “lamb” artık “make_unique” tarafından ayrılan belleğe sahip. Kapak, kopyalanamaz bir tür içerdiğinden, bu, “lamb"ın kendisinin kopyalanamaz olduğu anlamına gelir. Ancak hareket ettirilebilir:

auto lamb_copy = lamb; //Illegal
auto lamb_move = std::move(lamb); //legal.

Şimdi “lamb_move” hafızanın sahibi.

std::function<> saklanan değerlerin kopyalanabilir olmasını gerektirdiğini unutmayın. Kendi yalnızca hareket gerektiren std::function yazabilirsiniz #t=201608061624094409548) veya lambda’yı bir ‘shared_ptr’ sarmalayıcısına koyabilirsiniz:

auto shared_lambda = [](auto&& f){
  return [spf = std::make_shared<std::decay_t<decltype(f)>>(decltype(f)(f))]
  (auto&&...args)->decltype(auto) {
    return (*spf)(decltype(args)(args)...);
  };
};
auto lamb_shared = shared_lambda(std::move(lamb_move));

bizim salt hareketli lambdamızı alır ve durumunu paylaşılan bir işaretçiye doldurur, ardından * kopyalanabilen* bir lambda döndürür ve ardından bir “std::function” veya benzerinde saklanır.

Genelleştirilmiş yakalama, değişkenin türü için “otomatik” tür kesintiyi kullanır. Bu yakalamaları varsayılan olarak değerler olarak bildirir, ancak bunlar aynı zamanda referans olabilirler:

int a = 0;

auto lamb = [&v = a](int add) //Note that `a` and `v` have different names
{
  v += add; //Modifies `a`
};

lamb(20); //`a` becomes 20.

Genelleştirme yakalama, harici bir değişkeni yakalamaya hiç ihtiyaç duymaz. Keyfi bir ifade yakalayabilir:

auto lamb = [p = std::make_unique<T>(...)]()
{
    p->SomeFunc();
}

Bu, lambdalara harici olarak lambdaya bildirmek zorunda kalmadan tutabilecekleri ve potansiyel olarak değiştirebilecekleri keyfi değerler vermek için kullanışlıdır. Tabii ki, bu sadece lambda işini tamamladıktan sonra bu değişkenlere erişmeyi düşünmüyorsanız faydalıdır.

İşlev işaretçisine dönüştürme #

Bir lambda’nın yakalama listesi boşsa, lambda, aynı argümanları alan ve aynı dönüş türünü döndüren bir işlev işaretçisine örtük bir dönüşüme sahiptir:

auto sorter = [](int lhs, int rhs) -> bool {return lhs < rhs;};

using func_ptr = bool(*)(int, int);
func_ptr sorter_func = sorter; // implicit conversion

Böyle bir dönüştürme, unary plus operatörü kullanılarak da uygulanabilir:

func_ptr sorter_func2 = +sorter; // enforce implicit conversion

Bu işlev işaretçisini çağırmak, lambda üzerinde “operator()” işlevini çağırmak gibi davranır. Bu işlev işaretçisi hiçbir şekilde kaynak lambda kapatmasının varlığına bağlı değildir. Bu nedenle, lambda kapanışından daha uzun yaşayabilir.

Bu özellik, esas olarak, C++ işlev nesneleri yerine işlev işaretçileriyle ilgilenen API’lerle lambdaları kullanmak için kullanışlıdır.

Boş bir yakalama listesine sahip genel lambdalar için bir işlev işaretçisine dönüştürme de mümkündür. Gerekirse, doğru uzmanlığı seçmek için şablon argüman kesintisi kullanılacaktır.

auto sorter = [](auto lhs, auto rhs) { return lhs < rhs; };
using func_ptr = bool(*)(int, int);
func_ptr sorter_func = sorter;  // deduces int, int
// note however that the following is ambiguous
// func_ptr sorter_func2 = +sorter;

Satır içi parametre paketini açmak için lambdaları kullanma #

Parametre paketi açma geleneksel olarak, her yapmak istediğinizde bir yardımcı fonksiyon yazmayı gerektirir.

Bu oyuncak örneğinde:

template<std::size_t...Is>
void print_indexes( std::index_sequence<Is...> ) {
  using discard=int[];
  (void)discard{0,((void)(
    std::cout << Is << '\n' // here Is is a compile-time constant.
  ),0)...};
}
template<std::size_t I>
void print_indexes_upto() {
  return print_indexes( std::make_index_sequence<I>{} );
}

“print_indexes_upto”, bir parametre dizini paketi oluşturmak ve paketini açmak istiyor. Bunun için bir yardımcı fonksiyon çağırması gerekir. Oluşturduğunuz bir parametre paketini her açmak istediğinizde, bunu yapmak için özel bir yardımcı işlev oluşturmanız gerekir.

Bu lambdalarla önlenebilir.

Parametre paketlerini aşağıdaki gibi bir lambda çağrıları kümesine açabilirsiniz:

template<std::size_t I>
using index_t = std::integral_constant<std::size_t, I>;
template<std::size_t I>
constexpr index_t<I> index{};

template<class=void, std::size_t...Is>
auto index_over( std::index_sequence<Is...> ) {
  return [](auto&& f){
    using discard=int[];
    (void)discard{0,(void(
      f( index<Is> )
    ),0)...};
  };
}

template<std::size_t N>
auto index_over(index_t<N> = {}) {
  return index_over( std::make_index_sequence<N>{} );
}

Katlama ifadeleriyle, “index_over()” şu şekilde basitleştirilebilir:

template<class=void, std::size_t...Is>
auto index_over( std::index_sequence<Is...> ) {
  return [](auto&& f){
    ((void)(f(index<Is>)), ...);
  };
}
son sürüm

Bunu yaptıktan sonra, parametre paketlerini manuel olarak açma zorunluluğunu başka bir kodda ikinci bir aşırı yükleme ile değiştirmek için kullanabilirsiniz, böylece parametre paketlerini “inline” paketini açmanıza izin verir:

template<class Tup, class F>
void for_each_tuple_element(Tup&& tup, F&& f) {
  using T = std::remove_reference_t<Tup>;
  using std::tuple_size;
  auto from_zero_to_N = index_over< tuple_size<T>{} >();

  from_zero_to_N(
    [&](auto i){
      using std::get;
      f( get<i>( std::forward<Tup>(tup) ) );
    }
  );
}

“index_over” tarafından lambdaya geçirilen “auto i”, bir “std::integral_constant<std::size_t, ???>” şeklindedir. Bunun, “bu"nun durumuna bağlı olmayan “std::size_t"ye bir “constexpr” dönüşümü vardır, bu nedenle onu bir derleme zamanı sabiti olarak kullanabiliriz, örneğin onu “std::get"e ilettiğimizde ` yukarıda.

En üstteki oyuncak örneğine geri dönmek için şu şekilde yeniden yazın:

template<std::size_t I>
void print_indexes_upto() {
  index_over(index<I>)([](auto i){
    std::cout << i << '\n'; // here i is a compile-time constant
  });
}

bu çok daha kısadır ve mantığı onu kullanan kodda tutar.

İşlevleri kullanarak lambda işlevlerini C++03’e taşıma #

C++‘daki Lambda işlevleri, işlevler yazmak için çok kısa bir sözdizimi sağlayan sözdizimsel şekerdir. Bu nedenle, lambda işlevini bir işleve dönüştürerek C++03’te (çok daha ayrıntılı olsa da) eşdeğer işlevsellik elde edilebilir:

// Some dummy types:
struct T1 {int dummy;};
struct T2 {int dummy;};
struct R {int dummy;};

// Code using a lambda function (requires C++11)
R use_lambda(T1 val, T2 ref) {
  // Use auto because the type of the lambda is unknown.
  auto lambda = [val, &ref](int arg1, int arg2) -> R {
    /* lambda-body */
    return R();
  };
  return lambda(12, 27);
}

// The functor class (valid C++03)
// Similar to what the compiler generates for the lambda function.
class Functor {
  // Capture list.
  T1 val;
  T2& ref;

public:
  // Constructor
  inline Functor(T1 val, T2& ref) : val(val), ref(ref) {}

  // Functor body
  R operator()(int arg1, int arg2) const {
    /* lambda-body */
    return R();
  }
};

// Equivalent to use_lambda, but uses a functor (valid C++03).
R use_functor(T1 val, T2 ref) {
  Functor functor(val, ref);
  return functor(12, 27);
}

// Make this a self-contained example.
int main() {
  T1 t1;
  T2 t2;
  use_functor(t1,t2);
  use_lambda(t1,t2);
  return 0;
}

Lambda işlevi “değişebilir” ise, işlevin çağrı operatörünü sabit olmayan yapın, yani:

R operator()(int arg1, int arg2) /*non-const*/ {
  /* lambda-body */
  return R();
}
