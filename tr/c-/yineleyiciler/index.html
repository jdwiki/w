Genel Bakış #

Yineleyiciler Konumlardır #

Yineleyiciler, bir dizi öğe üzerinde gezinme ve çalışma aracıdır ve işaretçilerin genelleştirilmiş bir uzantısıdır. Kavramsal olarak, yineleyicilerin öğeler değil konumlar olduğunu hatırlamak önemlidir. Örneğin, aşağıdaki sırayı alın:

A B C

Dizi üç eleman ve dört konum içerir

+---+---+---+---+
| A | B | C |   |
+---+---+---+---+

Öğeler bir dizi içindeki şeylerdir. Konumlar, diziye anlamlı işlemlerin yapılabileceği yerlerdir. Örneğin, bir öğeye değil, bir önce veya sonra öğe A konumuna eklenir. Bir elemanın (erase(A)) silinmesi bile önce konumunu bularak, sonra onu silerek yapılır.

Yineleyicilerden Değerlere #

Bir konumdan değere dönüştürmek için bir yineleyici dereferenced olur:

auto my_iterator = my_vector.begin(); // position
auto my_value = *my_iterator; // value

Bir yineleyici, dizide atıfta bulunduğu değere başvuruda bulunmaz olarak düşünülebilir. Bu, özellikle bir dizideki “end()” yineleyicisini neden hiçbir zaman referanstan çıkarmamanız gerektiğini anlamak için yararlıdır:

+---+---+---+---+
| A | B | C |   |
+---+---+---+---+
  ↑           ↑
  |           +-- An iterator here has no value. Do not dereference it!
  +-------------- An iterator here dereferences to the value A.

C++ standart kitaplığında bulunan tüm diziler ve kaplarda, ‘begin()’ ilk konuma bir yineleyici döndürür ve ’end()’ bir yineleyiciyi son konumun ötesine döndürür (son konum değil!) . Sonuç olarak, algoritmalardaki bu yineleyicilerin adları çoğu zaman “ilk” ve “son” olarak etiketlenir:

+---+---+---+---+
| A | B | C |   |
+---+---+---+---+
  ↑           ↑
  |           |
  +- first    +- last

Boş bir dizi bile en az bir konum içerdiğinden, herhangi bir diziye bir yineleyici elde etmek de mümkündür:

+---+
|   |
+---+

Boş bir dizide, “begin()” ve “end()” aynı konumda olacaktır ve _hiçbirinin referansı kaldırılamaz:

+---+
|   |
+---+
  ↑
  |
  +- empty_sequence.begin()
  |
  +- empty_sequence.end()

Yineleyicilerin alternatif görselleştirmesi, öğelerin arasındaki konumları işaretlemeleridir:

+---+---+---+
| A | B | C |
+---+---+---+
↑   ^   ^   ↑
|           |
+- first    +- last

ve bir yineleyicinin başvurusunu kaldırmak, yineleyiciden sonra gelen öğeye bir başvuru döndürür. Bu görüşün özellikle yararlı olduğu bazı durumlar şunlardır:

  • ‘insert’ işlemleri, öğeleri yineleyici tarafından belirtilen konuma ekleyecektir,
  • “silme” işlemleri, girilenle aynı konuma karşılık gelen bir yineleyici döndürür,
  • bir yineleyici ve buna karşılık gelen ters yineleyici, öğeler arasında aynı .pozisyonda bulunur

Geçersiz Yineleyici #

Bir yineleyici (örneğin, bir işlem sırasında) konumu artık bir dizinin parçası değilse geçersiz olur. Geçersiz kılınan bir yineleyici, geçerli bir konuma yeniden atanana kadar başvurudan çıkarılamaz. Örneğin:

std::vector<int>::iterator first;
{
    std::vector<int> foo;
    first = foo.begin(); // first is now valid
} // foo falls out of scope and is destroyed
// At this point first is now invalid

C++ standart kitaplığındaki birçok algoritma ve dizi üyesi işlevi, yineleyicilerin ne zaman geçersiz kılınacağını yöneten kurallara sahiptir. Her algoritma, yineleyicileri işleme (ve geçersiz kılma) biçiminde farklıdır.

Yineleyicilerle Gezinme #

Bildiğimiz gibi, yineleyiciler dizilerde gezinmek içindir. Bunu yapmak için bir yineleyicinin konumunu dizi boyunca taşıması gerekir. Yineleyiciler dizide ileriye doğru ilerleyebilir ve bazıları geriye doğru ilerleyebilir:

auto first = my_vector.begin();
++first;                                             // advance the iterator 1 position
std::advance(first, 1);                              // advance the iterator 1 position
first = std::next(first);                            // returns iterator to the next element
std::advance(first, -1);                             // advance the iterator 1 position backwards
first = std::next(first, 20);                        // returns iterator to the element 20 position forward
first = std::prev(first, 5);                         // returns iterator to the element 5 position backward
auto dist = std::distance(my_vector.begin(), first); // returns distance between two iterators.

Not, std::distance’ın ikinci argümanına birinciden ulaşılabilir olmalıdır (veya başka bir deyişle ‘ilk’, ‘saniye’den küçük veya ona eşit olmalıdır).

Yineleyicilerle aritmetik işleçler gerçekleştirebilseniz de, tüm yineleyici türleri için tüm işlemler tanımlanmaz. a = b + 3; Rastgele Erişim Yineleyiciler için işe yarar, ancak b = a; ++b; ++b; ++b;. Bu nedenle, yineleyici türünün ne olduğundan emin değilseniz (örneğin, yineleyiciyi kabul eden bir şablon işlevinde) özel işlevleri kullanmanız önerilir.

Yineleyici Kavramları #

C++ standardı birkaç farklı yineleyici kavramını tanımlar. Bunlar, atıfta bulundukları dizilerde nasıl davrandıklarına göre gruplandırılmıştır. Bir yineleyici models (gibi davranır) kavramını biliyorsanız, ait olduğu diziden bağımsız olarak o yineleyicinin davranışından emin olabilirsiniz. Genellikle en kısıtlayıcıdan en az kısıtlayıcıya doğru sıralanırlar (çünkü bir sonraki yineleyici kavramı öncekinden daha iyi bir adımdır):

  • Girdi Yineleyicileri: Konum başına yalnızca bir kez referans alınabilir. Bir seferde yalnızca ve yalnızca bir pozisyon ilerleyebilir.
  • İleri Yineleyiciler : Herhangi bir sayıda başvurudan çıkarılabilen bir giriş yineleyici.
  • Çift Yönlü Yineleyiciler : Her seferinde bir konum _geriye doğru da ilerleyebilen ileri bir yineleyici.
  • Rastgele Erişim Yineleyicileri: Aynı anda herhangi bir sayıda konum ileri veya geri ilerleyebilen çift yönlü bir yineleyici.
  • Bitişik Yineleyiciler (C++17’den beri): Temel verilerin bellekte bitişik olduğunu garanti eden bir rastgele erişim yineleyici.

Algoritmalar, verildikleri yineleyiciler tarafından modellenen konsepte bağlı olarak değişebilir. Örneğin, “random_shuffle” ileri yineleyiciler için uygulanabilse de, rastgele erişim yineleyicileri gerektiren daha verimli bir değişken sağlanabilir.

Yineleyici özellikleri #

Yineleyici özellikleri, yineleyicilerin özelliklerine tek tip arayüz sağlar. Değer, fark, işaretçi, referans türleri ve ayrıca yineleyici kategorisini almanıza izin verir:

template<class Iter>
Iter find(Iter first, Iter last, typename std::iterator_traits<Iter>::value_type val)  {
    while (first != last) {
        if (*first == val)
            return first;
        ++first;
    }
    return last;
}

Algoritmaları özelleştirmek için yineleyici kategorisi kullanılabilir:

template<class BidirIt>
void test(BidirIt a, std::bidirectional_iterator_tag)  {
    std::cout << "Bidirectional iterator is used" << std::endl;
}
 
template<class ForwIt>
void test(ForwIt a, std::forward_iterator_tag)  {
    std::cout << "Forward iterator is used" << std::endl;
}
 
template<class Iter>
void test(Iter a)  {
    test(a, typename std::iterator_traits<Iter>::iterator_category());
}

Yineleyici kategorileri temel olarak yineleyici kavramlarıdır, ancak Bitişik Yineleyicilerin kodu kırdığı tespit edildiğinden kendi etiketleri yoktur.

Vektör Yineleyici #

“begin”, dizi kapsayıcısındaki ilk öğeye bir “yineleyici” döndürür.

“end”, sondan sonraki ilk öğeye bir “yineleyici” döndürür.

Vektör nesnesi “const” ise, hem “begin” hem de “end” bir “const_iterator” döndürür. Vektörünüz ‘const’ olmasa bile bir ‘const_iterator’ döndürülmesini istiyorsanız, ‘cbegin’ ve ‘cend’ kullanabilirsiniz.

Örnek:

#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = { 1, 2, 3, 4, 5 };  //intialize vector using an initializer_list

    for (std::vector<int>::iterator it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << " ";
    }

    return 0;
}

Çıktı:

1 2 3 4 5

Harita Yineleyici #

Kapsayıcıdaki ilk öğenin yineleyicisi.

Bir harita nesnesi const nitelikliyse, işlev bir “const_iterator” döndürür. Aksi takdirde, bir ‘yineleyici’ döndürür.

// Create a map and insert some values
std::map<char,int> mymap;
mymap['b'] = 100;
mymap['a'] = 200;
mymap['c'] = 300;

// Iterate over all tuples
for (std::map<char,int>::iterator it = mymap.begin(); it != mymap.end(); ++it)
    std::cout << it->first << " => " << it->second << '\n';

Çıktı:

bir => 200 b => 100 c => 300

Ters Yineleyiciler #

Bir liste veya vektör üzerinden geriye doğru yineleme yapmak istiyorsak, bir ‘reverse_iterator’ kullanabiliriz. Bir ters yineleyici, ‘base()’ aracılığıyla erişilebilen bir üye olarak tuttuğu çift yönlü veya rastgele erişim yineleyicisinden yapılır.

Geriye doğru yineleme yapmak için, sırasıyla koleksiyonun sonu ve koleksiyonun başlangıcı için yineleyiciler olarak ‘rbegin()’ ve ‘rend()’ kullanın.

Örneğin, geriye doğru yinelemek için şunu kullanın:

std::vector<int> v{1, 2, 3, 4, 5};
for (std::vector<int>::reverse_iterator it = v.rbegin(); it != v.rend(); ++it)
{
    cout << *it;
} // prints 54321

Bir ters yineleyici, ‘base()’ üye işlevi aracılığıyla bir ileri yineleyiciye dönüştürülebilir. İlişki, ters yineleyicinin “temel()” yineleyiciden sonraki bir öğeye başvurmasıdır:

std::vector<int>::reverse_iterator r = v.rbegin();
std::vector<int>::iterator i = r.base();
assert(&*r == &*(i-1)); // always true if r, (i-1) are dereferenceable
                        // and are not proxy iterators

 +---+---+---+---+---+---+---+
 |   | 1 | 2 | 3 | 4 | 5 |   |
 +---+---+---+---+---+---+---+
   ↑   ↑               ↑   ↑
   |   |               |   |
rend() |         rbegin()  end()
       |                   rbegin().base()
     begin()
     rend().base()

Yineleyicilerin öğeler arasındaki konumları işaretlediği görselleştirmede ilişki daha basittir:

  +---+---+---+---+---+
  | 1 | 2 | 3 | 4 | 5 |
  +---+---+---+---+---+
  ↑                   ↑
  |                   |
  |                 end()
  |                 rbegin()
begin()             rbegin().base()
rend()
rend().base()

Akış Yineleyicileri #

Akış yineleyicileri, bir diziyi okumamız veya bir kapsayıcıdan biçimlendirilmiş verileri yazdırmamız gerektiğinde kullanışlıdır:

// Data stream. Any number of various whitespace characters will be OK.
std::istringstream istr("1\t 2     3 4");
std::vector<int> v;

// Constructing stream iterators and copying data from stream into vector.
std::copy(
    // Iterator which will read stream data as integers.
    std::istream_iterator<int>(istr),
    // Default constructor produces end-of-stream iterator.
    std::istream_iterator<int>(),
    std::back_inserter(v));

// Print vector contents.
std::copy(v.begin(), v.end(),
    //Will print values to standard output as integers delimeted by " -- ".
    std::ostream_iterator<int>(std::cout, " -- "));

Örnek program standart çıktıya 1 -- 2 -- 3 -- 4 -- yazdıracaktır.

C Yineleyiciler (İşaretçiler) #

// This creates an array with 5 values.
const int array[] = { 1, 2, 3, 4, 5 };

#ifdef BEFORE_CPP11

// You can use `sizeof` to determine how many elements are in an array.
const int* first = array;
const int* afterLast = first + sizeof(array) / sizeof(array[0]);

// Then you can iterate over the array by incrementing a pointer until
// it reaches past the end of our array.
for (const int* i = first; i < afterLast; ++i) {
    std::cout << *i << std::endl;
}

#else

// With C++11, you can let the STL compute the start and end iterators:
for (auto i = std::begin(array); i != std::end(array); ++i) {
    std::cout << *i << std::endl;
}

#endif

Bu kod, her satırda bir tane olmak üzere 1’den 5’e kadar olan sayıları şu şekilde verir:

1
2
3
4
5

Parçalamak #

const int array[] = { 1, 2, 3, 4, 5 };

Bu satır, 5 değerli yeni bir tamsayı dizisi oluşturur. C dizileri, her değerin bitişik bir blokta birlikte depolandığı belleğe yalnızca işaretçilerdir.

const int* first = array;
const int* afterLast = first + sizeof(array) / sizeof(array[0]);

Bu çizgiler iki işaretçi oluşturur. İlk işaretçiye, dizideki ilk öğenin adresi olan dizi işaretçisinin değeri verilir. “sizeof” operatörü, bir C dizisinde kullanıldığında dizinin boyutunu bayt olarak döndürür. Bir elemanın boyutuna bölünmesi, dizideki eleman sayısını verir. Diziden sonra bloğun adresini bulmak için bunu kullanabiliriz.

for (const int* i = first; i < afterLast; ++i) {

Burada yineleyici olarak kullanacağımız bir işaretçi oluşturuyoruz. Üzerinde yineleme yapmak istediğimiz ilk öğenin adresi ile başlatılır ve ‘i’, ‘sonradan’ daha küçük olduğu sürece, yani ‘i’ içindeki bir adresi işaret ettiği sürece yinelemeye devam edeceğiz. “dizi”.

    std::cout << *i << std::endl;

Son olarak, döngü içinde ‘i’ yineleyicimizin işaret ettiği değere referansı kaldırarak erişebiliriz. Burada referans operatörü “*”, “i” içindeki adresteki değeri döndürür.

Kendi jeneratör destekli yineleyicinizi yazın #

Diğer dillerdeki yaygın bir kalıp, nesnelerin bir “akışını” üreten bir işleve sahip olmak ve bunun üzerinde döngü yapmak için döngü kodunu kullanabilmektir.

Bunu C++ ile şu şekilde modelleyebiliriz:

template<class T>
struct generator_iterator {
  using difference_type=std::ptrdiff_t;
  using value_type=T;
  using pointer=T*;
  using reference=T;
  using iterator_category=std::input_iterator_tag;
  std::optional<T> state;
  std::function< std::optional<T>() > operation;
  // we store the current element in "state" if we have one:
  T operator*() const {
    return *state;
  }
  // to advance, we invoke our operation.  If it returns a nullopt
  // we have reached the end:
  generator_iterator& operator++() {
    state = operation();
    return *this;        
  }
  generator_iterator operator++(int) {
    auto r = *this;
    ++(*this);
    return r;
  }
  // generator iterators are only equal if they are both in the "end" state:
  friend bool operator==( generator_iterator const& lhs, generator_iterator const& rhs ) {
    if (!lhs.state && !rhs.state) return true;
    return false;
  }
  friend bool operator!=( generator_iterator const& lhs, generator_iterator const& rhs ) {
    return !(lhs==rhs);
  }
  // We implicitly construct from a std::function with the right signature:
  generator_iterator( std::function< std::optional<T>() > f ):operation(std::move(f))
  {
    if (operation)
      state = operation();
  }
  // default all special member functions:
  generator_iterator( generator_iterator && ) =default;
  generator_iterator( generator_iterator const& ) =default;
  generator_iterator& operator=( generator_iterator && ) =default;
  generator_iterator& operator=( generator_iterator const& ) =default;
  generator_iterator() =default;
};

canlı örnek.

Oluşturulan öğeyi erken depolarız, böylece zaten sonunda olup olmadığımızı daha kolay tespit edebiliriz.

Bir son oluşturucu yineleyicinin işlevi hiçbir zaman kullanılmadığından, yalnızca bir kez std::işlevini kopyalayarak bir dizi üreteç yineleyicisi oluşturabiliriz. Varsayılan olarak oluşturulmuş bir oluşturucu yineleyici, kendisiyle ve diğer tüm son oluşturucu yineleyicilerle karşılaştırır.
