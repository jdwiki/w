  • ‘std::map’ veya ‘std::multimap’ten herhangi birini kullanmak için ‘’ başlık dosyası dahil edilmelidir.

  • ‘std::map’ ve ‘std::multimap’ öğelerinin her ikisi de öğelerini artan tuş sırasına göre sıralar. ‘std::multimap’ durumunda, aynı anahtarın değerleri için sıralama yapılmaz.

  • “std::map” ve “std::multimap” arasındaki temel fark, “std::map” öğesinin “std::multimap” öğesinin yaptığı yerde aynı anahtar için yinelenen değerlere izin vermemesidir.

  • Haritalar ikili arama ağaçları olarak uygulanır. Yani search(), insert(), erase() ortalama olarak Θ(log n) zaman alır. Sabit zamanlı işlem için std::unordered_map kullanın.

  • size() ve empty() işlevleri Θ(1) zaman karmaşıklığına sahiptir, bu işlevler her çağrıldığında ağaçta yürümeyi önlemek için düğüm sayısı önbelleğe alınır.

Öğelere erişme #

Bir std::map, (anahtar, değer) çiftlerini girdi olarak alır.

Aşağıdaki std::map başlatma örneğini düşünün:

std::map < std::string, int > ranking { std::make_pair("stackoverflow", 2), 
                                        std::make_pair("docs-beta", 1) };

Bir std::map içinde, öğeler aşağıdaki gibi eklenebilir:

ranking["stackoverflow"]=2;
ranking["docs-beta"]=1;

Yukarıdaki örnekte, ‘stackoverflow’ anahtarı zaten mevcutsa, değeri 2 olarak güncellenecektir. Zaten mevcut değilse, yeni bir giriş oluşturulacaktır.

Bir std::map içinde, anahtar bir dizin olarak verilerek öğelere doğrudan erişilebilir:

std::cout << ranking[ "stackoverflow" ] << std::endl;

Haritada ‘operatör[]’ kullanmanın aslında haritaya sorgulanan anahtarla yeni bir değer ekleyeceğini unutmayın. Bu, anahtar zaten haritada saklanmış olsa bile onu bir ‘const std::map’ üzerinde kullanamayacağınız anlamına gelir. Bu eklemeyi önlemek için, öğenin var olup olmadığını kontrol edin (örneğin find() kullanarak) veya aşağıda açıklandığı gibi at() kullanın.

Bir ‘std::map’ öğelerine ‘at()’ ile erişilebilir:

std::cout << ranking.at("stackoverflow") << std::endl;

Kap, istenen öğeyi içermiyorsa, “at()” öğesinin bir “std::out_of_range” istisnası oluşturacağını unutmayın.

son sürüm

Her iki kapsayıcıda std::map ve std::multimap, öğelere yineleyiciler kullanılarak erişilebilir:

// Example using begin()
std::multimap < int, std::string > mmp { std::make_pair(2, "stackoverflow"),
                                         std::make_pair(1, "docs-beta"),
                                         std::make_pair(2, "stackexchange")  };
auto it = mmp.begin();
std::cout << it->first << " : " << it->second << std::endl; // Output: "1 : docs-beta"
it++;
std::cout << it->first << " : " << it->second << std::endl; // Output: "2 : stackoverflow"
it++;
std::cout << it->first << " : " << it->second << std::endl; // Output: "2 : stackexchange"

// Example using rbegin()
std::map < int, std::string > mp {  std::make_pair(2, "stackoverflow"),
                                    std::make_pair(1, "docs-beta"),
                                    std::make_pair(2, "stackexchange")  };
auto it2 = mp.rbegin();
std::cout << it2->first << " : " << it2->second << std::endl; // Output: "2 : stackoverflow"
it2++;
std::cout << it2->first << " : " << it2->second << std::endl; // Output: "1 : docs-beta"

Öğe ekleme #

Bir öğe, yalnızca anahtarı haritada mevcut değilse, bir ‘std::map’ içine eklenebilir. Örneğin verilen:

std::map< std::string, size_t > fruits_count;

  • ‘insert()’ üye işlevi aracılığıyla bir ‘std::map’ içine bir anahtar/değer çifti eklenir. Argüman olarak bir “çift” gerektirir:

    fruits_count.insert({"grapes", 20});
fruits_count.insert(make_pair("orange", 30));
fruits_count.insert(pair<std::string, size_t>("banana", 40));
fruits_count.insert(map<std::string, size_t>::value_type("cherry", 50));

“insert()” işlevi, bir yineleyici ve bir “bool” değerinden oluşan bir “çift” döndürür:

- If the insertion was successful, the iterator points to the newly inserted element, and the `bool` value is `true`.
- If there was already an element with the same `key`, the insertion fails. When that happens, the iterator points to the element causing the conflict, and the `bool` is value is `false`.

Ekleme ve arama işlemini birleştirmek için aşağıdaki yöntem kullanılabilir:

  auto success = fruits_count.insert({"grapes", 20});
  if (!success.second) {           // we already have 'grapes' in the map
      success.first->second += 20; // access the iterator to update the value
  }

  • Kolaylık sağlamak için, ‘std::map’ kapsayıcı, indis operatörüne haritadaki öğelere erişmesini ve yoksa yenilerini eklemesini sağlar:

    fruits_count["apple"] = 10;

Daha basit olmakla birlikte, kullanıcının öğenin zaten var olup olmadığını kontrol etmesini engeller. Bir öğe eksikse, ‘std:🗺:operator[]’ örtük olarak onu oluşturur, sağlanan değerin üzerine yazmadan önce varsayılan kurucu ile başlatır.

  • ‘insert()’, çaprazlı bir çift listesi kullanarak aynı anda birkaç öğe eklemek için kullanılabilir. insert()‘in bu sürümü void döndürür:

     fruits_count.insert({{"apricot", 1}, {"jackfruit", 1}, {"lime", 1}, {"mango", 7}});

  • “insert()”, “değer_türü” değerlerinin başlangıcını ve sonunu belirten yineleyiciler kullanılarak öğe eklemek için de kullanılabilir:

    std::map< std::string, size_t > fruit_list{ {"lemon", 0}, {"olive", 0}, {"plum", 0}};
fruits_count.insert(fruit_list.begin(), fruit_list.end());

Örnek:

std::map<std::string, size_t> fruits_count;
std::string fruit;
while(std::cin >> fruit){
    // insert an element with 'fruit' as key and '1' as value
    // (if the key is already stored in fruits_count, insert does nothing)
    auto ret = fruits_count.insert({fruit, 1});
    if(!ret.second){            // 'fruit' is already in the map 
        ++ret.first->second;    // increment the counter
    }
}

Bir ekleme işlemi için zaman karmaşıklığı O(log n)‘dir, çünkü “std::map” ağaç olarak uygulanır.

Bir “çift”, “make_pair()” ve “emplace()” kullanılarak açıkça oluşturulabilir:

std::map< std::string , int > runs;
runs.emplace("Babe Ruth", 714);
runs.insert(make_pair("Barry Bonds", 762));

Yeni elemanın nereye ekleneceğini biliyorsak, o zaman bir yineleyici ‘ipucu’ belirtmek için ’emplace_hint()’ kullanabiliriz. Yeni eleman “ipucu"dan hemen önce eklenebilirse, ekleme sabit zamanda yapılabilir. Aksi takdirde, “emplace()” ile aynı şekilde davranır:

std::map< std::string , int > runs;
auto it = runs.emplace("Barry Bonds", 762); // get iterator to the inserted element
// the next element will be before "Barry Bonds", so it is inserted before 'it'
runs.emplace_hint(it, "Babe Ruth", 714);

std::map veya std::multimap içinde arama #

“std::map” veya “std::multimap” içinde bir anahtarı aramanın birkaç yolu vardır.

  • Bir anahtarın ilk oluşumunun yineleyicisini almak için find() işlevi kullanılabilir. Anahtar yoksa end() döndürür.

      std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
  auto it = mmp.find(6);
  if(it!=mmp.end())
      std::cout << it->first << ", " << it->second << std::endl; //prints: 6, 5
  else
      std::cout << "Value does not exist!" << std::endl;

  it = mmp.find(66);
  if(it!=mmp.end())
      std::cout << it->first << ", " << it->second << std::endl; 
  else
      std::cout << "Value does not exist!" << std::endl; // This line would be executed.

  • Bir girdinin ‘std::map’ veya ‘std::multimap’ içinde olup olmadığını bulmanın başka bir yolu, belirli bir anahtarla kaç değerin ilişkilendirildiğini sayan ‘count()’ işlevini kullanmaktır. ‘std::map’ her anahtarla yalnızca bir değer ilişkilendirdiğinden, ‘count()’ işlevi yalnızca 0 (anahtar yoksa) veya 1 (varsa) döndürebilir. ‘std::multimap’ için, aynı anahtarla ilişkili birkaç değer olabileceğinden, ‘count()’ 1’den büyük değerler döndürebilir.

      std::map< int , int > mp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
  if(mp.count(3) > 0) // 3 exists as a key in map
      std::cout << "The key exists!" << std::endl; // This line would be executed.
  else
      std::cout << "The key does not exist!" << std::endl;

Yalnızca bazı öğelerin var olup olmadığına önem veriyorsanız, “bul” kesinlikle daha iyidir: niyetinizi belgeler ve “çoklu haritalar” için ilk eşleşen öğe bulunduğunda durabilir.

  • “std::multimap” durumunda, aynı anahtara sahip birkaç öğe olabilir. Bu aralığı elde etmek için, yineleyici alt sınırı (dahil) ve üst sınırı (hariç) olan ‘std::pair’ değerini döndüren ’equal_range()’ işlevi kullanılır. Anahtar yoksa, her iki yineleyici de “end()“i işaret eder.

      auto eqr = mmp.equal_range(6);
  auto st = eqr.first, en = eqr.second;
  for(auto it = st; it != en; ++it){
      std::cout << it->first << ", " << it->second << std::endl; 
  }
      // prints: 6, 5
      //         6, 7

Bir std::map veya std::multimap başlatma #

“std::map” ve “std::multimap”, virgülle ayrılmış anahtar/değer çiftleri sağlanarak başlatılabilir. Anahtar/değer çiftleri, “{key, value}” ile sağlanabilir veya “std::make_pair(key, value)” ile açıkça oluşturulabilir. ‘std::map’ yinelenen anahtarlara izin vermediğinden ve virgül operatörü sağdan sola çalıştığından, soldaki aynı tuşa sahip çift, sağdaki çiftin üzerine yazılır.

std::multimap < int, std::string > mmp { std::make_pair(2, "stackoverflow"),
                                     std::make_pair(1, "docs-beta"),
                                     std::make_pair(2, "stackexchange")  };
// 1 docs-beta
// 2 stackoverflow
// 2 stackexchange

std::map < int, std::string > mp {  std::make_pair(2, "stackoverflow"),
                                std::make_pair(1, "docs-beta"),
                                std::make_pair(2, "stackexchange")  }; 
// 1 docs-beta
// 2 stackoverflow

Her ikisi de yineleyici ile başlatılabilir.

// From std::map or std::multimap iterator
std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {6, 8}, {3, 4}, 
                               {6, 7} };
                       // {1, 2}, {3, 4}, {3, 4}, {6, 5}, {6, 8}, {6, 7}, {8, 9}
auto it = mmp.begin();
std::advance(it,3); //moved cursor on first {6, 5}
std::map< int, int > mp(it, mmp.end()); // {6, 5}, {8, 9}

//From std::pair array
std::pair< int, int > arr[10];
arr[0] = {1, 3};
arr[1] = {1, 5};
arr[2] = {2, 5};
arr[3] = {0, 1};
std::map< int, int > mp(arr,arr+4); //{0 , 1}, {1, 3}, {2, 5}

//From std::vector of std::pair
std::vector< std::pair<int, int> > v{ {1, 5}, {5, 1}, {3, 6}, {3, 2} };
std::multimap< int, int > mp(v.begin(), v.end()); 
                        // {1, 5}, {3, 6}, {3, 2}, {5, 1}

Öğe sayısını kontrol etme #

‘std::map’ kabı, haritanın boş olup olmamasına bağlı olarak ‘doğru’ veya ‘yanlış’ döndüren ’empty()’ üye işlevine sahiptir. “size()” üye işlevi, bir “std::map” kapsayıcısında depolanan öğe sayısını döndürür:

std::map<std::string , int> rank {{"facebook.com", 1} ,{"google.com", 2}, {"youtube.com", 3}};
if(!rank.empty()){
    std::cout << "Number of elements in the rank map: " << rank.size() << std::endl;
}
else{
    std::cout << "The rank map is empty" << std::endl;
}

Harita Türleri #

Normal Harita #

Harita, anahtar/değer çiftlerini içeren bir ilişkisel kapsayıcıdır.

#include <string>
#include <map>
std::map<std::string, size_t> fruits_count;

Yukarıdaki örnekte, “std::string” anahtar türüdür ve “size_t” bir değerdir.

Anahtar, haritada bir dizin görevi görür. Her anahtar benzersiz olmalı ve sipariş edilmelidir.

  • Aynı anahtara sahip birden fazla öğeye ihtiyacınız varsa, “multimap” kullanmayı düşünün (aşağıda açıklanmıştır)

  • Değer türünüz herhangi bir sıralama belirtmiyorsa veya varsayılan sıralamayı geçersiz kılmak istiyorsanız, şunları sağlayabilirsiniz:

    #include <string>
#include <map>
#include <cstring>
struct StrLess {
    bool operator()(const std::string& a, const std::string& b) {
        return strncmp(a.c_str(), b.c_str(), 8)<0;
               //compare only up to 8 first characters
    }
}
std::map<std::string, size_t, StrLess> fruits_count2;

    If StrLess comparator returns false for two keys, they are considered the same even if their actual contents differ.

Çoklu Harita #

Çoklu harita, aynı anahtara sahip birden çok anahtar/değer çiftinin haritada saklanmasına izin verir. Aksi takdirde, arayüzü ve oluşturulması normal haritaya çok benzer.

 #include <string>
 #include <map>
 std::multimap<std::string, size_t> fruits_count;
 std::multimap<std::string, size_t, StrLess> fruits_count2;

Hash-Map (Sırasız Harita) #

Bir karma harita, normal bir haritaya benzer şekilde anahtar/değer çiftlerini depolar. Yine de öğeleri anahtara göre sıralamaz. Bunun yerine, gerekli anahtar/değer çiftlerine hızlı bir şekilde erişmek için anahtar için bir hash değeri kullanılır.

#include <string>
#include <unordered_map>
std::unordered_map<std::string, size_t> fruits_count;

Sırasız haritalar genellikle daha hızlıdır, ancak öğeler herhangi bir tahmin edilebilir sırada saklanmaz. Örneğin, bir “sırasız_harita” içindeki tüm öğeler üzerinde yineleme yapmak, öğeleri görünüşte rastgele bir sırada verir.

Öğeleri silme #

Tüm öğelerin kaldırılması:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
mmp.clear(); //empty multimap

Yineleyici yardımıyla öğeyi bir yerden kaldırmak:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
                            // {1, 2}, {3, 4}, {3, 4}, {6, 5}, {6, 7}, {8, 9}
auto it = mmp.begin();
std::advance(it,3); // moved cursor on first {6, 5}
mmp.erase(it); // {1, 2}, {3, 4}, {3, 4}, {6, 7}, {8, 9}

Bir aralıktaki tüm öğeleri kaldırma:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
                            // {1, 2}, {3, 4}, {3, 4}, {6, 5}, {6, 7}, {8, 9}
auto it = mmp.begin();
auto it2 = it;
it++; //moved first cursor on first {3, 4}
std::advance(it2,3);  //moved second cursor on first {6, 5}
mmp.erase(it,it2); // {1, 2}, {6, 5}, {6, 7}, {8, 9}

Anahtar olarak sağlanan bir değere sahip tüm öğelerin kaldırılması:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
                            // {1, 2}, {3, 4}, {3, 4}, {6, 5}, {6, 7}, {8, 9}
mmp.erase(6); // {1, 2}, {3, 4}, {3, 4}, {8, 9}

Bir “pred” yüklemini karşılayan öğeleri kaldırma:

std::map<int,int> m;
auto it = m.begin();
while (it != m.end())
{
   if (pred(*it))
       it = m.erase(it);
   else
       ++it;
}

std::map veya std::multimap üzerinde yineleme #

‘std::map’ veya ‘std::multimap’ aşağıdaki yollarla çaprazlanabilir:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
                               
//Range based loop - since C++11
for(const auto &x: mmp) 
    std::cout<< x.first <<":"<< x.second << std::endl;

//Forward iterator for loop: it would loop through first element to last element
//it will be a std::map< int, int >::iterator
for (auto it = mmp.begin(); it != mmp.end(); ++it)
std::cout<< it->first <<":"<< it->second << std::endl; //Do something with iterator

//Backward iterator for loop: it would loop through last element to first element
//it will be a std::map< int, int >::reverse_iterator
for (auto it = mmp.rbegin(); it != mmp.rend(); ++it)
std::cout<< it->first <<" "<< it->second << std::endl; //Do something with iterator

Bir “std::map” veya bir “std::multimap” üzerinde yinelenirken, gereksiz örtük dönüşümlerden kaçınmak için “auto” kullanımı tercih edilir (daha fazla ayrıntı için [bu SO yanıtına1 bakın).

Anahtar olarak kullanıcı tanımlı türlerle std::map oluşturma #

Bir sınıfın bir haritada anahtar olarak kullanılabilmesi için anahtarın ‘kopyalanabilir’ ve ‘atanabilir’ olması yeterlidir. Harita içindeki sıralama, üçüncü argüman tarafından tanımlanır. şablon (ve kullanılıyorsa, yapıcının argümanı). Bu varsayılanlar, varsayılan olarak “<” operatörü olan “std::less” şeklindedir, ancak varsayılanları kullanma zorunluluğu yoktur. Sadece bir karşılaştırma yazın operatör (tercihen işlevsel bir nesne olarak):

struct CmpMyType
{
    bool operator()( MyType const& lhs, MyType const& rhs ) const
    {
        //  ...
    }
};

C++‘da “karşılaştır” yüklemi bir katı zayıf sıralama olmalıdır. Özellikle, “karşılaştırma(X,X)” herhangi bir “X” için “yanlış” döndürmelidir. yani, “CmpMyType()(a, b)” true döndürürse, “CmpMyType()(b, a)” false döndürmelidir ve her ikisi de false döndürürse, öğeler eşit olarak kabul edilir (aynı denklik sınıfının üyeleri).

Kesin Zayıf Sıralama #

Bu, iki nesne arasındaki ilişkiyi tanımlayan matematiksel bir terimdir. Tanımı:

Hem f(x, y) hem de f(y, x) yanlışsa, x ve y iki nesne eşdeğerdir. Bir nesnenin her zaman (dönüşümsüzlük değişmezi ile) kendisine eşdeğer olduğuna dikkat edin.

C++ açısından bu, belirli bir türde iki nesneniz varsa, < operatörüyle karşılaştırıldığında aşağıdaki değerleri döndürmeniz gerektiği anlamına gelir.

X    a;
X    b;

Condition:                  Test:     Result
a is equivalent to b:       a < b     false
a is equivalent to b        b < a     false

a is less than b            a < b     true
a is less than b            b < a     false

b is less than a            a < b     false
b is less than a            b < a     true

Eşdeğer/daha azını nasıl tanımladığınız tamamen nesnenizin türüne bağlıdır.
