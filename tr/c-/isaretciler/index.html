İşaretçi, bellekteki bir konuma atıfta bulunan bir adrestir. Genellikle, işlevlerin veya veri yapılarının, atıfta bulunulan belleği kopyalamak zorunda kalmadan belleği bilmesine ve değiştirmesine izin vermek için kullanılırlar. İşaretçiler hem ilkel (yerleşik) hem de kullanıcı tanımlı türlerle kullanılabilir.

İşaretçiler “başvuru” * , “adresi” & ve “ok” -> operatörlerini kullanır. ‘*’ ve ‘->’ operatörleri işaret edilen belleğe erişmek için kullanılır ve ‘&’ operatörü bellekte bir adres almak için kullanılır.

Sözdizimi #

  • <Veri türü> *<Değişken adı>;
  • <Veri türü> *<Değişken adı> = &<Aynı Veri türünün değişken adı>;
  • <Veri türü> *<Değişken adı> = <Aynı Veri türünün değeri>;
  • int *foo; // Bir tamsayı değerine işaret eden bir işaretçi
  • int *bar = &myIntVar;
  • uzun *çubuk[2];
  • uzun *çubuk[] = {&myLongVar1, &myLongVar2}; //Eşittir: uzun *bar[2]

Aynı satırda birden çok işaretçi bildirirken sorunların farkında olun.

int* a, b, c; //Only a is a pointer, the others are regular ints.

int* a, *b, *c; //These are three pointers!

int *foo[2]; //Both *foo[0] and *foo[1] are pointers.

İşaretçi temelleri #

Not: Aşağıdakilerin hepsinde, C++11 sabiti “nullptr"nin varlığı varsayılır. Daha önceki sürümler için, “nullptr"yi, eskiden benzer bir rol oynayan sabit olan “NULL” ile değiştirin.

son sürüm

Bir işaretçi değişkeni oluşturma #

Belirli bir “*” sözdizimi kullanılarak bir işaretçi değişkeni oluşturulabilir, ör. int *pointer_to_int;. Bir değişken işaretçi türünde (int *) olduğunda, yalnızca bir bellek adresi içerir. Bellek adresi, temel türdeki (`int’) verilerin depolandığı konumdur.

Bir değişkenin boyutu ile aynı türdeki bir işaretçinin boyutu karşılaştırıldığında aradaki fark açıktır:

// Declare a struct type `big_struct` that contains
// three long long ints.
typedef struct {
    long long int foo1;
    long long int foo2;
    long long int foo3;
} big_struct;

// Create a variable `bar` of type `big_struct`
big_struct bar;
// Create a variable `p_bar` of type `pointer to big_struct`.
// Initialize it to `nullptr` (a null pointer).
big_struct *p_bar0 = nullptr;

// Print the size of `bar`
std::cout << "sizeof(bar) = " << sizeof(bar) << std::endl;
// Print the size of `p_bar`.
std::cout << "sizeof(p_bar0) = " << sizeof(p_bar0) << std::endl;

/* Produces:
    sizeof(bar) = 24
    sizeof(p_bar0) = 8
*/

Başka bir değişkenin adresini almak #

İşaretçiler, normal değişkenler gibi kendi aralarında atanabilir; bu durumda, bir işaretçiden diğerine kopyalanan bellek adresidir, işaretçinin işaret ettiği gerçek veriler değil. Ayrıca, boş bir bellek konumunu temsil eden ’nullptr’ değerini alabilirler. “nullptr” değerine eşit bir işaretçi geçersiz bir bellek konumu içerir ve bu nedenle geçerli verilere atıfta bulunmaz.

Belirli bir türdeki bir değişkenin bellek adresini, değişkenin önüne adresi operatörünün & ekleyerek alabilirsiniz. “&” tarafından döndürülen değer, değişkenin bellek adresini içeren temel türe yönelik bir işaretçidir (bu, değişken kapsam dışına çıkmadığı sürece geçerli verilerdir**).

// Copy `p_bar0` into `p_bar_1`.
big_struct *p_bar1 = p_bar0;

// Take the address of `bar` into `p_bar_2`
big_struct *p_bar2 = &bar;

// p_bar1 is now nullptr, p_bar2 is &bar.

p_bar0 = p_bar2;

// p_bar0 is now &bar.

p_bar2 = nullptr;

// p_bar0 == &bar
// p_bar1 == nullptr
// p_bar2 == nullptr

Referansların aksine:

  • iki işaretçi atamak, atanan işaretçinin başvurduğu belleğin üzerine yazmaz;
  • işaretçiler boş olabilir.
  • adresi operatörünün açıkça belirtilmesi gerekir.

Bir işaretçinin içeriğine erişme #

Bir adres almak & gerektirdiğinden, içeriğe erişim de önek olarak başvuru operatörü * kullanılmasını gerektirir. Bir işaretçinin başvurusu kaldırıldığında, temel alınan türün bir değişkeni (aslında ona bir başvuru) olur. Daha sonra “const” değilse okunabilir ve değiştirilebilir.

(*p_bar0).foo1 = 5;

// `p_bar0` points to `bar`. This prints 5.
std::cout << "bar.foo1 = " << bar.foo1 << std::endl;

// Assign the value pointed to by `p_bar0` to `baz`.
big_struct baz;
baz = *p_bar0;

// Now `baz` contains a copy of the data pointed to by `p_bar0`.
// Indeed, it contains a copy of `bar`.

// Prints 5 as well
std::cout << "baz.foo1 = " << baz.foo1 << std::endl;

“*” ve “.” operatörünün birleşimi “->” ile kısaltılır:

std::cout << "bar.foo1 = " << (*p_bar0).foo1 << std::endl; // Prints 5
std::cout << "bar.foo1 = " <<  p_bar0->foo1  << std::endl; // Prints 5

Geçersiz işaretçilerin referansını kaldırma #

Bir işaretçinin başvurusunu kaldırırken, geçerli verilere işaret ettiğinden emin olmalısınız. Geçersiz bir işaretçinin (veya boş bir işaretçinin) referansının kaldırılması, bellek erişim ihlaline veya çöp verilerinin okunmasına veya yazılmasına neden olabilir.

big_struct *never_do_this() {
   // This is a local variable. Outside `never_do_this` it doesn't exist.
   big_struct retval;
   retval.foo1 = 11;
   // This returns the address of `retval`.
   return &retval;
   // `retval` is destroyed and any code using the value returned
   // by `never_do_this` has a pointer to a memory location that
   // contains garbage data (or is inaccessible).
}

Böyle bir senaryoda, “g++” ve “clang++” uyarıları doğru şekilde verir:

(Clang) warning: address of stack memory associated with local variable 'retval' returned [-Wreturn-stack-address]
(Gcc)   warning: address of local variable ‘retval’ returned [-Wreturn-local-addr]

Bu nedenle, işaretçiler boş olabileceğinden, işlevlerin argümanları olduğunda dikkatli olunmalıdır:

void naive_code(big_struct *ptr_big_struct) {
    // ... some code which doesn't check if `ptr_big_struct` is valid.
    ptr_big_struct->foo1 = 12;
}

// Segmentation fault.
naive_code(nullptr);

İşaretçi İşlemleri #

İşaretçiler için iki operatör vardır: Operatörün adresi (&): İşlenenin bellek adresini döndürür. Contents-of (Dereference) operatörü(*): Operatörü tarafından belirtilen adreste bulunan değişkenin değerini döndürür.

int var = 20;
int *ptr;
ptr = &var;

cout << var << endl;
//Outputs 20 (The value of var)

cout << ptr << endl;
//Outputs 0x234f119 (var's memory location)

cout << *ptr << endl;
//Outputs 20(The value of the variable stored in the pointer ptr

Yıldız işareti (*), basit bir işaretçi olduğunu belirtmek amacıyla bir işaretçi bildirirken kullanılır. Bunu, belirtilen adreste bulunan değeri elde etmek için kullanılan dereference operatörüyle karıştırmayın. Onlar sadece aynı işaretle temsil edilen iki farklı şeydir.

İşaretçi Aritmetiği #

Arttırma / Azaltma #

Bir işaretçi artırılabilir veya azaltılabilir (önek ve sonek). Bir işaretçiyi artırmak, işaretçi değerini dizideki öğeye, o anda işaret edilen öğeden bir öğeye kadar ilerletir. Bir işaretçiyi azaltmak, onu dizideki bir önceki öğeye taşır.

İşaretçinin işaret ettiği tür tam değilse işaretçi aritmetiğine izin verilmez. “void” her zaman eksik bir türdür.

char* str = new char[10]; // str = 0x010
++str;                    // str = 0x011  in this case sizeof(char) = 1 byte

int* arr = new int[10];   // arr = 0x00100
++arr;                    // arr = 0x00104 if sizeof(int) = 4 bytes

void* ptr = (void*)new char[10];
++ptr;    // void is incomplete.

Son öğeye bir işaretçi artırılırsa, işaretçi dizinin sonundan sonraki bir öğeye işaret eder. Böyle bir işaretçinin referansı kaldırılamaz, ancak azaltılabilir.

Bir işaretçiyi dizideki sondan bir sonraki öğeye artırmak veya bir dizideki ilk öğeye bir işaretçiyi azaltmak, tanımsız davranışa neden olur.

Dizi olmayan bir nesneye yönelik bir işaretçi, işaretçi aritmetiği amacıyla, sanki 1 boyutunda bir diziymiş gibi ele alınabilir.

Toplama çıkarma #

İşaretçilere tamsayı değerleri eklenebilir; Artan gibi davranırlar, ancak 1 yerine belirli bir sayı ile hareket ederler. Tamsayı değerleri, işaretçi azaltma işlevi görerek işaretçilerden de çıkarılabilir. Arttırma/azaltmada olduğu gibi, işaretçi tam bir türe işaret etmelidir.

char* str = new char[10];  // str = 0x010
str += 2;                  // str = 0x010 + 2 * sizeof(char) = 0x012

int* arr = new int[10];    // arr = 0x100
arr += 2;                  // arr = 0x100 + 2 * sizeof(int) = 0x108, assuming sizeof(int) == 4.

İşaretçi Farkı #

Aynı türden iki işaretçi arasındaki fark hesaplanabilir. İki işaretçi aynı dizi nesnesi içinde olmalıdır; aksi takdirde tanımsız davranış sonuçları.

Aynı dizide “P” ve “Q” olmak üzere iki işaretçi verildiğinde, “P” dizideki “i” öğesiyse ve “Q” “j” öğesiyse, “P - Q” olur “i - j”. Sonuç türü, “” öğesinden “std::ptrdiff_t” şeklindedir.

char* start = new char[10];  // str = 0x010
char* test = &start[5];
std::ptrdiff_t diff = test - start; //Equal to 5.
std::ptrdiff_t diff = start - test; //Equal to -5; ptrdiff_t is signed.
