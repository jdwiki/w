Karmaşıklığı O(n²)‘den O(n)‘ye düşürmek için std::move kullanma #

C++11, bir nesneyi hareket ettirmek için temel dil ve standart kitaplık desteğini tanıttı. Buradaki fikir, bir o nesnesi geçici olduğunda ve kişi mantıksal bir kopya istediğinde, o’nun dinamik olarak ayrılmış bir arabellek gibi kaynaklarını çalmak, o‘yu mantıksal olarak boş ama yine de yok edilebilir bırakmak güvenlidir ve kopyalanabilir.

Temel dil desteği esas olarak

  • değer referansı tür oluşturucu “&&”, ör., “std::string&&”, bir “std::string"e bir değer referansıdır, bu, atıfta bulunulan nesnenin kaynakları çalınabilecek geçici bir nesne olduğunu gösterir (yani taşındı)

  • hareket yapıcısı için özel destek, bu kaynakları fiilen kopyalamak yerine belirtilen diğer nesneden verimli bir şekilde taşıması gereken T( T&& ) ve

  • ayrıca kaynaktan hareket etmesi gereken bir hareket atama operatörü auto operatörü=(T&&) -> T& için özel destek.

Standart kitaplık desteği, esas olarak ‘’ başlığındaki ‘std::move’ işlev şablonudur. Bu işlev, belirtilen nesneye, sanki geçiciymiş gibi taşınabileceğini belirten bir değer referansı üretir.

Bir kap için gerçek kopyalama tipik olarak O(n) karmaşıklığındadır; burada n kaptaki öğelerin sayısıdır, hareket ederken O(1), sabit zamandır. Ve bu kapsayıcıyı n kez mantıksal olarak kopyalayan bir algoritma için, bu, karmaşıklığı genellikle pratik olmayan O(n²)‘den yalnızca doğrusal O(n)‘ye indirebilir.

Andrew Koenig, [19 Eylül 2013’te Dr. Dobbs Journal’daki “Asla Değişmeyen Konteynerler”1] makalesinde, değişkenlerin başlatıldıktan sonra değişmez olduğu bir programlama stili kullanırken algoritmik verimsizliğin ilginç bir örneğini sundu. Bu stilde döngüler genellikle özyineleme kullanılarak ifade edilir. Ve Collatz dizisi oluşturma gibi bazı algoritmalar için özyineleme, bir kapsayıcının mantıksal olarak kopyalanmasını gerektirir:

// Based on an example by Andrew Koenig in his Dr. Dobbs Journal article
// “Containers That Never Change” September 19, 2013, available at
// <url: http://www.drdobbs.com/cpp/containters-that-never-change/240161543>

// Includes here, e.g. <vector>

namespace my {
    template< class Item >
    using Vector_ = /* E.g. std::vector<Item> */;

    auto concat( Vector_<int> const& v, int const x )
        -> Vector_<int>
    {
        auto result{ v };
        result.push_back( x );
        return result;
    }

    auto collatz_aux( int const n, Vector_<int> const& result )
        -> Vector_<int>
    {
        if( n == 1 )
        {
            return result;
        }
        auto const new_result = concat( result, n );
        if( n % 2 == 0 )
        {
            return collatz_aux( n/2, new_result );
        }
        else
        {
            return collatz_aux( 3*n + 1, new_result );
        }
    }

    auto collatz( int const n )
        -> Vector_<int>
    {
        assert( n != 0 );
        return collatz_aux( n, Vector_<int>() );
    }
}  // namespace my

#include <iostream>
using namespace std;
auto main() -> int
{
    for( int const x : my::collatz( 42 ) )
    {
        cout << x << ' ';
    }
    cout << '\n';
}

Çıktı:

42 21 64 32 16 8 4 2

Vektörlerin kopyalanmasından kaynaklanan öğe kopyalama işlemlerinin sayısı burada kabaca O(n²)‘dir, çünkü toplam 1 + 2 + 3 + … n‘dir.

Somut sayılarda, g++ ve Visual C++ derleyicileri ile yukarıdaki “collatz(42)” çağrısı, vektör kopya oluşturucu çağrılarında 8 öğe ve 36 öğe kopyalama işlemi (8*7/2 = 28, artı bazı) içeren bir Collatz dizisiyle sonuçlandı. .

Bu öğe kopyalama işlemlerinin tümü, değerlerine artık ihtiyaç duyulmayan vektörleri basitçe hareket ettirerek kaldırılabilir. Bunu yapmak için, vektörleri değere göre ileterek vektör tipi argümanlar için “const” ve referansı kaldırmak gerekir. İşlev dönüşleri zaten otomatik olarak optimize edilmiştir. Vektörlerin iletildiği ve işlevde daha sonra tekrar kullanılmadığı çağrılar için, bu arabellekleri gerçekten kopyalamak yerine bu arabellekleri hareket ettirmek için sadece ‘std::move’ uygulayın:

using std::move;

auto concat( Vector_<int> v, int const x )
    -> Vector_<int>
{
    v.push_back( x );
    // warning: moving a local object in a return statement prevents copy elision [-Wpessimizing-move]
    // See https://www.wikiod.com/tr/docs/c%2b%2b/2489/copy-elision
    // return move( v );
    return v;
}

auto collatz_aux( int const n, Vector_<int> result )
    -> Vector_<int>
{
    if( n == 1 )
    {
        return result;
    }
    auto new_result = concat( move( result ), n );
    struct result;      // Make absolutely sure no use of `result` after this.
    if( n % 2 == 0 )
    {
        return collatz_aux( n/2, move( new_result ) );
    }
    else
    {
        return collatz_aux( 3*n + 1, move( new_result ) );
    }
}

auto collatz( int const n )
    -> Vector_<int>
{
    assert( n != 0 );
    return collatz_aux( n, Vector_<int>() );
}

Burada, g++ ve Visual C++ derleyicileriyle, vektör kopya oluşturucu çağrıları nedeniyle öğe kopyalama işlemlerinin sayısı tam olarak 0’dı.

Algoritma, üretilen Collatz dizisinin uzunluğunda zorunlu olarak hala O(n)‘dir, ancak bu oldukça çarpıcı bir gelişmedir: O(n²) → Açık*).

Bazı dil desteğiyle, belki de hareket kullanılabilir ve yine de bir değişkenin *başlangıç ​​​​ve son hareketi arasında * değişmezliğini ifade edebilir ve uygulayabilir, bundan sonra bu değişkenin herhangi bir kullanımı bir hata olmalıdır. Ne yazık ki, C++ 14’ten itibaren C++ bunu desteklemiyor. Döngüsüz kod için, hareketten sonra kullanılmama, ilgili adın, yukarıdaki “struct result;” ile olduğu gibi, tamamlanmamış bir “yapı” olarak yeniden bildirilmesi yoluyla zorunlu kılınabilir, ancak bu çirkindir ve başkaları tarafından anlaşılması pek olası değildir. programcılar; ayrıca teşhis oldukça yanıltıcı olabilir.

Özetle, hareket için C++ dili ve kitaplığı desteği, algoritma karmaşıklığında önemli gelişmelere izin verir, ancak desteğin eksikliği nedeniyle, “const"un sağlayabileceği kod doğruluğu garantilerinden ve kod netliğinden vazgeçme pahasına.

Bütünlük için, kopya oluşturucu çağrıları nedeniyle öğe kopyalama işlemlerinin sayısını ölçmek için kullanılan araçlı vektör sınıfı: 
template< class Item >
class Copy_tracking_vector
{
private:
    static auto n_copy_ops()
        -> int&
    {
        static int value;
        return value;
    }
    
    vector<Item>    items_;
    
public:
    static auto n() -> int { return n_copy_ops(); }

    void push_back( Item const& o ) { items_.push_back( o ); }
    auto begin() const { return items_.begin(); }
    auto end() const { return items_.end(); }

    Copy_tracking_vector(){}
    
    Copy_tracking_vector( Copy_tracking_vector const& other )
        : items_( other.items_ )
    { n_copy_ops() += items_.size(); }

    Copy_tracking_vector( Copy_tracking_vector&& other )
        : items_( move( other.items_ ) )
    {}
};

Anlambilimi taşı #

Taşıma semantiği, C++‘da bir nesneyi diğerine taşımanın bir yoludur. Bunun için eski nesneyi boşaltıyoruz ve sahip olduğu her şeyi yeni nesneye yerleştiriyoruz.

Bunun için bir değer referansının ne olduğunu anlamalıyız. Bir değer referansı (‘T&&’, burada T nesne tipidir) normal bir referanstan (‘T&’, şimdi değer referansları olarak adlandırılır) çok farklı değildir. Ancak bunlar 2 farklı tür olarak hareket ederler ve bu nedenle, hareket semantiği ile uğraşırken gerekli olacak, bir türü veya diğerini alan yapıcılar veya işlevler yapabiliriz.

İki farklı türe ihtiyaç duymamızın nedeni iki farklı davranış belirtmektir. Değer referans oluşturucuları kopyalama ile ilgiliyken, değer referans oluşturucuları taşıma ile ilgilidir.

Bir nesneyi taşımak için std::move(obj) kullanacağız. Bu işlev, nesneye bir değer referansı döndürür, böylece o nesneden verileri yeni bir nesneye çalabiliriz. Bunu yapmanın aşağıda tartışılan birkaç yolu vardır.

Unutulmaması gereken önemli nokta, ‘std::move’ kullanımının yalnızca bir değer referansı oluşturmasıdır. Başka bir deyişle, ‘std::move(obj)’ ifadesi nesnenin içeriğini değiştirmezken, ‘auto obj2 = std::move(obj)’ (muhtemelen) değiştirir.

Yapıcıyı taşı #

Diyelim ki bu kod parçacığına sahibiz.

class A {
public:
    int a;
    int b;
       
    A(const A &other) {
        this->a = other.a;
        this->b = other.b;
    }
};

Bir kopya kurucu oluşturmak için, yani bir nesneyi kopyalayan ve yeni bir tane yaratan bir fonksiyon yapmak için normalde yukarıda gösterilen sözdizimini seçerdik, A için A tipi başka bir nesneye referans alan bir kurucumuz olurdu, ve nesneyi yöntemin içine manuel olarak kopyalardık.

Alternatif olarak, kopya oluşturucusunu kullanarak tüm üyeleri otomatik olarak kopyalayan A(const A &) = default; yazabilirdik.

Ancak bir move yapıcısı oluşturmak için burada olduğu gibi bir değer referansı yerine bir değer referansı alacağız.

class Wallet {
public:
    int nrOfDollars;
    
    Wallet() = default; //default ctor

    Wallet(Wallet &&other) {
        this->nrOfDollars = other.nrOfDollars;
        other.nrOfDollars = 0;
    }
};

Lütfen eski değerleri ‘sıfır’ olarak belirlediğimize dikkat edin. Varsayılan taşıma yapıcısı (Wallet(Wallet&&) = default;), bir POD olduğu için nrOfDollars değerini kopyalar.

Hareket semantiği, orijinal örnekten ‘çalma’ durumuna izin verecek şekilde tasarlandığından, bu çalmadan sonra orijinal örneğin nasıl görünmesi gerektiğini düşünmek önemlidir. Bu durumda, değeri sıfıra çevirmeseydik, oyuna giren dolar miktarını ikiye katlamış olurduk.

Wallet a;
a.nrOfDollars = 1;
Wallet b (std::move(a)); //calling B(B&& other);
std::cout << a.nrOfDollars << std::endl; //0
std::cout << b.nrOfDollars << std::endl; //1

Böylece eski bir nesneden bir nesne inşa ettik.

Yukarıdaki basit bir örnek olsa da, move yapıcısının ne yapmak istediğini gösterir. Kaynak yönetimi söz konusu olduğunda olduğu gibi daha karmaşık durumlarda daha kullanışlı hale gelir.

    // Manages operations involving a specified type.
    // Owns a helper on the heap, and one in its memory (presumably on the stack).
    // Both helpers are DefaultConstructible, CopyConstructible, and MoveConstructible.
    template<typename T,
             template<typename> typename HeapHelper,
             template<typename> typename StackHelper>
    class OperationsManager {
        using MyType = OperationsManager<T, HeapHelper, StackHelper>;

        HeapHelper<T>* h_helper;
        StackHelper<T> s_helper;
        // ...

      public:
        // Default constructor & Rule of Five.
        OperationsManager() : h_helper(new HeapHelper<T>) {}
        OperationsManager(const MyType& other)
          : h_helper(new HeapHelper<T>(*other.h_helper)), s_helper(other.s_helper) {}
        MyType& operator=(MyType copy) {
            swap(*this, copy);
            return *this;
        }
        ~OperationsManager() {
            if (h_helper) { delete h_helper; }
        }

        // Move constructor (without swap()).
        // Takes other's HeapHelper<T>*.
        // Takes other's StackHelper<T>, by forcing the use of StackHelper<T>'s move constructor.
        // Replaces other's HeapHelper<T>* with nullptr, to keep other from deleting our shiny
        //  new helper when it's destroyed.
        OperationsManager(MyType&& other) noexcept
          : h_helper(other.h_helper),
            s_helper(std::move(other.s_helper)) {
            other.h_helper = nullptr;
        }

        // Move constructor (with swap()).
        // Places our members in the condition we want other's to be in, then switches members
        //  with other.
        // OperationsManager(MyType&& other) noexcept : h_helper(nullptr) {
        //     swap(*this, other);
        // }

        // Copy/move helper.
        friend void swap(MyType& left, MyType& right) noexcept {
            std::swap(left.h_helper, right.h_helper);
            std::swap(left.s_helper, right.s_helper);
        }
    };

Atamayı taşı #

Değer referansı olan bir nesneye nasıl değer atadığımıza benzer şekilde, onu kopyalayarak, yeni bir tane oluşturmadan değerleri bir nesneden diğerine taşıyabiliriz. Biz buna hareket ataması diyoruz. Değerleri bir nesneden var olan başka bir nesneye taşırız.

Bunun için, kopya atamasında olduğu gibi bir değer referansı alması için değil, bir değer referansı alması için ‘operatör =’ ı aşırı yüklememiz gerekecek.

class A {
    int a;
    A& operator= (A&& other) {
        this->a = other.a;
        other.a = 0;
        return *this;
    }
};

Bu, hareket atamasını tanımlamak için kullanılan tipik sözdizimidir. Operatör =’ i aşırı yükledik, böylece ona bir değer referansı besleyebiliriz ve onu başka bir nesneye atayabiliriz.

A a;
a.a = 1;
A b;
b = std::move(a); //calling A& operator= (A&& other)
std::cout << a.a << std::endl; //0
std::cout << b.a << std::endl; //1

Böylece, bir nesneyi diğerine atamayı hareket ettirebiliriz.
