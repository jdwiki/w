Sözdizimi #

  • [[detaylar]]: Tartışmasız basit öznitelik

  • [[detaylar(argümanlar)]]: Bağımsız değişkenlerle nitelik

  • __attribute(detaylar): Standart olmayan GCC/Clang/IBM’ye özel

  • __declspec(detaylar): Standart olmayan MSVC’ye özgü

[[suya düşmek]] #

Bir “anahtar"da bir “vaka” sona erdiğinde, bir sonraki durumun kodu yürütülür. Bu sonuncusu ‘break’ ifadesi kullanılarak önlenebilir. Bu sözde hata davranışı, amaçlanmadığında hatalara neden olabileceğinden, birkaç derleyici ve statik analizör bu konuda bir uyarı verir.

C++17’den itibaren, kodun düşmesi gerektiğinde uyarının gerekli olmadığını belirtmek için standart bir öznitelik tanıtıldı. Derleyiciler, bir vaka ‘break’ veya ‘[[fallthrough]]’ olmadan sona erdiğinde ve en az bir deyimi olduğunda güvenle uyarı verebilir.

switch(input) {
    case 2011:
    case 2014:
    case 2017:
        std::cout << "Using modern C++" << std::endl;
        [[fallthrough]]; // > No warning
    case 1998:
    case 2003:
        standard = input;
}

“[[fallthrough]]” öğesinin nasıl kullanılabileceğine ilişkin daha ayrıntılı örnekler için teklif bölümüne bakın.

son sürüm

[[kullanımdan kaldırıldı]] ve [[kullanımdan kaldırıldı(“neden”)]] #

C++14, öznitelikler aracılığıyla işlevleri kullanımdan kaldırmanın standart bir yolunu tanıttı. “[[kullanımdan kaldırılmış]]”, bir işlevin kullanımdan kaldırıldığını belirtmek için kullanılabilir. [[deprecated("reason")]] derleyici tarafından gösterilebilecek belirli bir neden eklemeye izin verir.

void function(std::unique_ptr<A> &&a);

// Provides specific message which helps other programmers fixing there code
[[deprecated("Use the variant with unique_ptr instead, this function will be removed in the next release")]]
void function(std::auto_ptr<A> a);

// No message, will result in generic warning if called.
[[deprecated]]
void function(A *a);

Bu öznitelik şunlara uygulanabilir:

  • bir sınıfın beyanı
  • bir typedef adı
  • bir değişken
  • statik olmayan bir veri üyesi
  • bir işlev
  • bir numaralandırma
  • bir şablon uzmanlığı

(ref. c++14 standart taslak: 7.6.5 Kullanımdan kaldırılan özellik)

son sürüm

[[nodiscard]] #

[[nodiscard]] niteliği, bir işlev çağrısı yaptığınızda bir işlevin dönüş değerinin göz ardı edilmemesi gerektiğini belirtmek için kullanılabilir. Eğer dönüş değeri yoksayılırsa derleyici bu konuda bir uyarı vermelidir. Nitelik şuraya eklenebilir:

  • Bir fonksiyon tanımı
  • Bir tip

Bir türe öznitelik eklemek, özniteliği bu türü döndüren her bir işleve eklemekle aynı davranışa sahiptir.

template<typename Function>
[[nodiscard]] Finally<std::decay_t<Function>> onExit(Function &&f);

void f(int &i) {
    assert(i == 0);                    // Just to make comments clear!
    ++i;                               // i == 1
    auto exit1 = onExit([&i]{ --i; }); // Reduce by 1 on exiting f()
    ++i;                               // i == 2
    onExit([&i]{ --i; });              // BUG: Reducing by 1 directly
                                       //      Compiler warning expected
    std::cout << i << std::end;        // Expected: 2, Real: 1
}

“[[nodiscard]]“ın nasıl kullanılabileceğine ilişkin daha ayrıntılı örnekler için teklif adresine bakın. .

Not: Örnekte “Finally”/“onExit” uygulama ayrıntıları atlanmıştır, bkz. [Finally/ScopeExit](https://www.wikiod.com/tr/docs/c%2b%2b/1320/raii- source-acquisition-is-başlatma/4551/nihayet-scopeexit).

son sürüm

[[iade yok]] #

C++ 11, “[[noreturn]]” özniteliğini tanıttı. Bir işlev için, bir return ifadesi yürüterek veya gövdesiyse sona ulaşarak işlevin arayana geri dönmediğini belirtmek için kullanılabilir (bunun “void” işlevleri için geçerli olmadığına dikkat etmek önemlidir, arayana geri döndükleri için herhangi bir değer döndürmezler). Böyle bir işlev, ‘std::terminate’ veya ‘std::exit’ çağrılarak veya bir istisna atılarak sona erebilir. Ayrıca, böyle bir işlevin “longjmp” çalıştırılarak geri dönebileceğini de belirtmekte fayda var.

Örneğin, aşağıdaki işlev her zaman bir istisna atar veya ‘std::terminate’ öğesini çağırır, bu nedenle ‘[[noreturn]]’ için iyi bir adaydır:

[[noreturn]] void ownAssertFailureHandler(std::string message) {
    std::cerr << message << std::endl;
    if (THROW_EXCEPTION_ON_ASSERT)
        throw AssertException(std::move(message));
    std::terminate();
}

Bu tür bir işlevsellik, derleyicinin, kodun asla yürütülmeyeceğini biliyorsa, bir işlevi dönüş ifadesi olmadan sonlandırmasına izin verir. Burada, aşağıdaki kodda “ownAssertFailureHandler” (yukarıda tanımlanmıştır) çağrısı hiçbir zaman geri dönmeyeceğinden, derleyicinin bu çağrının altına kod eklemesine gerek yoktur:

std::vector<int> createSequence(int end) {
    if (end > 0) {
        std::vector<int> sequence;
        sequence.reserve(end+1);
        for (int i = 0; i <= end; ++i)
            sequence.push_back(i);
        return sequence;
    }
    ownAssertFailureHandler("Negative number passed to createSequence()"s);
    // return std::vector<int>{}; //< Not needed because of [[noreturn]]
}

İşlev gerçekten dönecekse tanımsız davranıştır, bu nedenle aşağıdakilere izin verilmez:

[[noreturn]] void assertPositive(int number) {
    if (number >= 0)
        return;
    else
        ownAssertFailureHandler("Positive number expected"s); //< [[noreturn]]
}

“[[noreturn]]” öğesinin çoğunlukla void işlevlerinde kullanıldığını unutmayın. Ancak, bu bir gereklilik değildir ve işlevlerin genel programlamada kullanılmasına izin verir:

template<class InconsistencyHandler>
double fortyTwoDivideBy(int i) {
    if (i == 0)
         i = InconsistencyHandler::correct(i);
    return 42. / i;
}

struct InconsistencyThrower {
    static [[noreturn]] int correct(int i) { ownAssertFailureHandler("Unknown inconsistency"s); }
}

struct InconsistencyChangeToOne {
    static int correct(int i) { return 1; }
}

double fortyTwo = fortyTwoDivideBy<InconsistencyChangeToOne>(0);
double unreachable = fortyTwoDivideBy<InconsistencyThrower>(0);

Aşağıdaki standart kitaplık işlevleri bu özniteliğe sahiptir:

  • std::iptal
  • std::çıkış
  • standart::hızlı_çıkış
  • std::beklenmedik
  • std::sonlandır
  • std::rethrow_exception
  • std::throw_with_nested
  • std::nested_exception::rethrow_nested
son sürüm

[[belki_kullanılmamış]] #

“[[maybe_unused]]’ özelliği, kodda belirli mantığın kullanılamayacağını belirtmek için oluşturulur. Bu, sıklıkla kullanılabileceği veya kullanılmayabileceği önişlemci koşullarıyla bağlantılıysa. Derleyiciler kullanılmayan değişkenler hakkında uyarı verebildiğinden, bu, amacı belirterek onları bastırmanın bir yoludur.

Hata ayıklama yapılarında gerekli olan ve üretimde ihtiyaç duyulmayan değişkenlerin tipik bir örneği, başarıyı gösteren dönüş değerleridir. Hata ayıklama yapılarında koşul belirtilmelidir, ancak üretimde bu iddialar kaldırılmıştır.

[[maybe_unused]] auto mapInsertResult = configuration.emplace("LicenseInfo", stringifiedLicenseInfo);
assert(mapInsertResult.second); // We only get called during startup, so we can't be in the map

Daha karmaşık bir örnek, adsız bir ad alanında bulunan farklı türde yardımcı işlevlerdir. Derleme sırasında bu işlevler kullanılmazsa, bir derleyici bunlara bir uyarı verebilir. İdeal olarak, onları arayanla aynı önişlemci etiketleriyle korumak istersiniz, ancak bu karmaşık hale gelebileceğinden ‘[[belki_unused]]’ özelliği daha sürdürülebilir bir alternatiftir.

namespace {
    [[maybe_unused]] std::string createWindowsConfigFilePath(const std::string &relativePath);
    // TODO: Reuse this on BSD, MAC ...
    [[maybe_unused]] std::string createLinuxConfigFilePath(const std::string &relativePath);
}

std::string createConfigFilePath(const std::string &relativePath) {
#if OS == "WINDOWS"
      return createWindowsConfigFilePath(relativePath);
#elif OS == "LINUX"
      return createLinuxConfigFilePath(relativePath);
#else
#error "OS is not yet supported"
#endif
}

“[[maybe_unused]]” öğesinin nasıl kullanılabileceğine ilişkin daha ayrıntılı örnekler için teklif bölümüne bakın.
