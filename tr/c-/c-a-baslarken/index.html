Selam Dünya #

Bu program standart çıktı akışına ‘Merhaba Dünya!’ yazdırır:

#include <iostream>

int main()
{
    std::cout << "Hello World!" << std::endl;
}

Coliru’da canlı görün.

analiz #

Bu kodun her bir bölümünü ayrıntılı olarak inceleyelim:

  • “#include ”, standart C++ başlık dosyası “iostream"in içeriğini içeren bir önişlemci yönergesidir.

    iostream is a standard library header file that contains definitions of the standard input and output streams. These definitions are included in the std namespace, explained below.

    The standard input/output (I/O) streams provide ways for programs to get input from and output to an external system – usually the terminal.

  • int main() { ... }, main adında yeni bir işlev tanımlar. Geleneksel olarak, programın yürütülmesi üzerine ‘main’ işlevi çağrılır. Bir C++ programında yalnızca bir “ana” işlev olmalıdır ve her zaman “int” türünden bir sayı döndürmelidir.

    Here, the int is what is called the function’s return type. “main” işlevi tarafından döndürülen değer bir çıkış kodudur.

    By convention, a program exit code of 0 or EXIT_SUCCESS is interpreted as success by a system that executes the program. Any other return code is associated with an error.

    If no return statement is present, the main function (and thus, the program itself) returns 0 by default. In this example, we don’t need to explicitly write return 0;.

    All other functions, except those that return the void type, must explicitly return a value according to their return type, or else must not return at all.

  • std::cout << "Merhaba Dünya!" << std::endl; “Merhaba Dünya!” yazdırır. standart çıktı akışına:

    • std is a namespace, and :: is the scope resolution operator that allows look-ups for objects by name within a namespace.

      There are many namespaces. Here, we use :: to show we want to use cout from the std namespace. For more information refer to Scope Resolution Operator - Microsoft Documentation.

    • std::cout is the standard output stream object, defined in iostream, and it prints to the standard output (stdout).

    • << is, in this context, the stream insertion operator, so called because it inserts an object into the stream object.

      The standard library defines the << operator to perform data insertion for certain data types into output streams. stream << content inserts content into the stream and returns the same, but updated stream. This allows stream insertions to be chained: std::cout << "Foo" << " Bar"; prints “FooBar” to the console.

    • "Hello World!" is a character string literal, or a “text literal.” The stream insertion operator for character string literals is defined in file iostream.

    • std::endl is a special I/O stream manipulator object, also defined in file iostream. Inserting a manipulator into a stream changes the state of the stream.

      The stream manipulator std::endl does two things: first it inserts the end-of-line character and then it flushes the stream buffer to force the text to show up on the console. This ensures that the data inserted into the stream actually appear on your console. (Stream data is usually stored in a buffer and then “flushed” in batches unless you force a flush immediately.)

      An alternate method that avoids the flush is:

      std::cout << "Hello World!\n";

      where \n is the character escape sequence for the newline character.

    • The semicolon (;) notifies the compiler that a statement has ended. All C++ statements and class definitions require an ending/terminating semicolon.

Yorumlar #

Bir yorum, C++ derleyicisinin herhangi bir işlevsel anlamla yorumlamasına gerek kalmadan rastgele metni kaynak kodun içine koymanın bir yoludur. Yorumlar, bir programın tasarımı veya yöntemi hakkında fikir vermek için kullanılır.

C++‘da iki tür yorum vardır:

Tek Satırlı Yorumlar #

Çift eğik çizgi dizisi //, yeni satıra kadar tüm metni yorum olarak işaretler:

int main()
{
   // This is a single-line comment.
   int a;  // this also is a single-line comment
   int i;  // this is another single-line comment
}

C-Tarzı/Blok Yorumlar #

‘/’ dizisi yorum bloğunun başlangıcını bildirmek için kullanılır ve ‘/’ dizisi yorumun sonunu bildirmek için kullanılır. Başlangıç ​​ve bitiş dizileri arasındaki tüm metin, metin aksi takdirde geçerli C++ sözdizimi olsa bile yorum olarak yorumlanır. Bunlara bazen “C tarzı” yorumlar denir, çünkü bu yorum sözdizimi C++‘ın önceki dili olan C’den miras alınır:

int main()
{
   /*
    *  This is a block comment.
    */
   int a;
}

Herhangi bir blok yorumda, istediğiniz her şeyi yazabilirsiniz. Derleyici */ sembolüyle karşılaştığında, blok yorumunu sonlandırır:

int main()
{
   /* A block comment with the symbol /*
      Note that the compiler is not affected by the second /*
      however, once the end-block-comment symbol is reached,
      the comment ends.
   */
   int a;
}

Yukarıdaki örnek, geçerli C++ (ve C) kodudur. Ancak, bir blok yorumun içinde ek /* bulunması bazı derleyicilerde bir uyarıya neden olabilir.

Blok yorumları da tek bir satırda başlayıp bitebilir. Örneğin:

void SomeFunction(/* argument 1 */ int a, /* argument 2 */ int b);

Yorumların Önemi #

Tüm programlama dillerinde olduğu gibi, yorumlar çeşitli faydalar sağlar:

  • Okumayı/korumayı kolaylaştırmak için kodun açık dokümantasyonu
  • Kodun amacının ve işlevselliğinin açıklaması
  • Kodun geçmişi veya gerekçesi ile ilgili ayrıntılar
  • Telif hakkı/lisanslarının, proje notlarının, özel teşekkürlerin, katkı kredilerinin vb. doğrudan kaynak koduna yerleştirilmesi.

Ancak, yorumların dezavantajları da vardır:

  • Koddaki herhangi bir değişikliği yansıtacak şekilde muhafaza edilmelidirler.
  • Aşırı yorumlar, kodu daha az okunabilir hale getirme eğilimindedir.

Açık, kendi kendini belgeleyen kodlar yazılarak yorum ihtiyacı azaltılabilir. Basit bir örnek, değişkenler, işlevler ve türler için açıklayıcı adların kullanılmasıdır. Mantıksal olarak ilgili görevleri ayrı işlevlere ayırmak bununla el ele gider.

Kodu devre dışı bırakmak için kullanılan yorum işaretleri #

Geliştirme sırasında, kod bölümlerini silmeden hızlı bir şekilde devre dışı bırakmak için yorumlar da kullanılabilir. Bu, genellikle test etme veya hata ayıklama amaçları için yararlıdır, ancak geçici düzenlemeler dışında hiçbir şey için iyi bir stil değildir. Buna genellikle “yorum yapmak” denir.

Benzer şekilde, bir kod parçasının eski sürümlerini referans amacıyla bir yorumda tutmak, bir sürüm oluşturma sistemi aracılığıyla kodun geçmişini keşfetmeye kıyasla çok az değer sunarken dosyaları karıştırdığı için hoş karşılanmaz.

Standart C++ derleme işlemi #

Yürütülebilir C++ program kodu genellikle bir derleyici tarafından üretilir.

Bir derleyici, kodu bir programlama dilinden (daha fazlası) bir bilgisayar için doğrudan çalıştırılabilen başka bir forma çeviren bir programdır. Kodu çevirmek için bir derleyici kullanmaya derleme denir.

C++, derleme işleminin biçimini “üst” dili olan C’den devralır. Aşağıda, C++‘da derlemenin dört ana adımını gösteren bir liste bulunmaktadır:

  1. C++ önişlemcisi, dahil edilen herhangi bir başlık dosyasının içeriğini kaynak kod dosyasına kopyalar, makro kodu oluşturur ve #define kullanılarak tanımlanan sembolik sabitleri değerleriyle değiştirir.
  2. C++ ön işlemcisi tarafından üretilen genişletilmiş kaynak kod dosyası, platforma uygun derleme dilinde derlenir.
  3. Derleyici tarafından oluşturulan montajcı kodu, platform için uygun nesne koduna birleştirilir.
  4. Birleştirici tarafından oluşturulan nesne kodu dosyası, yürütülebilir bir dosya üretmek için kullanılan herhangi bir kitaplık işlevi için nesne kodu dosyalarıyla bağlantılıdır.
  • Not: Bazı derlenmiş kodlar birbirine bağlıdır, ancak nihai bir program oluşturmak için değil. Genellikle, bu “bağlı” kod, diğer programlar tarafından kullanılabilecek bir biçimde paketlenebilir. Bu “paketlenmiş, kullanılabilir kod paketi”, C++ programcılarının kütüphane olarak adlandırdığı şeydir.

Birçok C++ derleyicisi, kolaylık veya ek analiz için derleme işleminin belirli bölümlerini de birleştirebilir veya birleştirmeyi kaldırabilir. Pek çok C++ programcısı farklı araçlar kullanacak, ancak bir programın üretimine dahil olduklarında tüm araçlar genellikle bu genel süreci izleyecektir.

Aşağıdaki bağlantı bu tartışmayı genişletir ve yardımcı olacak güzel bir grafik sağlar. 1: http://faculty.cs.niu.edu/~mcmahon/CS241/Notes/compile.html

İşlev #

Bir işlev, bir dizi ifadeyi temsil eden bir kod birimidir.

İşlevler argümanları veya değerleri kabul edebilir ve tek bir değer döndürebilir (veya vermeyebilir). Bir işlevi kullanmak için, bağımsız değişken değerlerinde bir işlev çağrısı kullanılır ve işlev çağrısının kendisi, dönüş değeriyle değiştirilir.

Her işlevin bir tür imzası vardır - argümanlarının türleri ve dönüş türünün türü.

Fonksiyonlar, prosedür ve matematiksel fonksiyon kavramlarından ilham alır.

  • Not: C++ işlevleri esasen prosedürlerdir ve matematiksel işlevlerin tam tanımını veya kurallarını takip etmezler.

İşlevler genellikle belirli bir görevi gerçekleştirmek içindir. ve bir programın diğer bölümlerinden çağrılabilir. Bir fonksiyon, bir programda başka bir yerde çağrılmadan önce bildirilmeli ve tanımlanmalıdır.

  • Not: popüler işlev tanımları, dahil edilen diğer dosyalarda gizlenebilir (genellikle kolaylık ve birçok dosyada yeniden kullanım için). Bu, başlık dosyalarının yaygın bir kullanımıdır.

İşlev Bildirimi #

Bir işlev bildirimi, derleyiciye adı ve tür imzasıyla bir işlevin varlığını bildirir. Sözdizimi aşağıdaki gibidir:

int add2(int i); // The function is of the type (int) -> (int)

Yukarıdaki örnekte, int add2(int i) işlevi derleyiciye şunları bildirir:

  • dönüş türü ‘int’dir.
  • İşlevin adı ‘add2’dir.
  • İşlevin argüman sayısı 1’dir:
  • İlk argüman “int” türündedir.
  • İlk bağımsız değişken, işlevin içeriğinde “i” adıyla anılacaktır.

Bağımsız değişken adı isteğe bağlıdır; işlevin bildirimi aşağıdaki gibi de olabilir:

int add2(int); // Omitting the function arguments' name is also permitted.

Tek tanım kuralı uyarınca, belirli bir tür imzaya sahip bir işlev, C++ derleyicisinin görebildiği tüm C++ kod tabanında yalnızca bir kez bildirilebilir veya tanımlanabilir. Başka bir deyişle, belirli bir tür imzaya sahip işlevler yeniden tanımlanamaz - yalnızca bir kez tanımlanmaları gerekir. Bu nedenle, aşağıdakiler geçerli C++ değildir:

int add2(int i);  // The compiler will note that add2 is a function (int) -> int
int add2(int j);  // As add2 already has a definition of (int) -> int, the compiler
                  // will regard this as an error.

Bir işlev hiçbir şey döndürmezse, dönüş türü “void” olarak yazılır. Parametre almıyorsa, parametre listesi boş olmalıdır.

void do_something(); // The function takes no parameters, and does not return anything.
                     // Note that it can still affect variables it has access to.

İşlev Çağrısı #

Bir fonksiyon bildirildikten sonra çağrılabilir. Örneğin, aşağıdaki program ‘main’ işlevi içinde ‘2’ değeriyle ‘add2’yi çağırır:

#include <iostream>

int add2(int i);    // Declaration of add2

// Note: add2 is still missing a DEFINITION.
// Even though it doesn't appear directly in code,
// add2's definition may be LINKED in from another object file.

int main()
{
    std::cout << add2(2) << "\n";  // add2(2) will be evaluated at this point,
                                   // and the result is printed.
    return 0;  
}

Burada, ‘add2(2)’, bir işlev çağrısının sözdizimidir.

Fonksiyon Tanımı #

Bir işlev tanımı*, bir bildirime benzer, ancak işlev kendi gövdesi içinde çağrıldığında yürütülen kodu da içerir.

“add2” için bir işlev tanımı örneği şöyle olabilir:

int add2(int i)       // Data that is passed into (int i) will be referred to by the name i
{                     // while in the function's curly brackets or "scope."
                    
    int j = i + 2;    // Definition of a variable j as the value of i+2.
    return j;         // Returning or, in essence, substitution of j for a function call to
                      // add2.
}

İşlev Aşırı Yüklemesi #

Aynı ada ancak farklı parametrelere sahip birden fazla fonksiyon oluşturabilirsiniz.

int add2(int i)           // Code contained in this definition will be evaluated
{                         // when add2() is called with one parameter.
    int j = i + 2;
    return j;
}

int add2(int i, int j)    // However, when add2() is called with two parameters, the
{                         // code from the initial declaration will be overloaded,
    int k = i + j + 2 ;   // and the code in this declaration will be evaluated
    return k;             // instead.
}

Her iki işlev de aynı adla “add2” olarak adlandırılır, ancak çağrılan asıl işlev doğrudan çağrıdaki parametrelerin miktarına ve türüne bağlıdır. Çoğu durumda, C++ derleyicisi hangi işlevin çağrılacağını hesaplayabilir. Bazı durumlarda, tür açıkça belirtilmelidir.

Varsayılan Parametreler #

İşlev parametreleri için varsayılan değerler yalnızca işlev bildirimlerinde belirtilebilir.

int multiply(int a, int b = 7); // b has default value of 7.
int multiply(int a, int b)
{
    return a * b;               // If multiply() is called with one parameter, the
}                               // value will be multiplied by the default, 7.

Bu örnekte, multiply() bir veya iki parametre ile çağrılabilir. Yalnızca bir parametre verilirse, ‘b’ varsayılan değeri 7 olacaktır. Varsayılan bağımsız değişkenler, işlevin sonraki bağımsız değişkenlerine yerleştirilmelidir. Örneğin:

int multiply(int a = 10, int b = 20); // This is legal 
int multiply(int a = 10, int b);      // This is illegal since int a is in the former

Özel Fonksiyon Çağrıları - Operatörler #

C++‘da ‘işlev_ismi(değer1, değer2, değer3)‘den farklı sözdizimine sahip özel işlev çağrıları vardır. En yaygın örnek operatörlerinkidir.

‘!’, ‘+’, ‘-’, ‘*’, ‘%’ ve ‘«’ gibi derleyici tarafından işlev çağrılarına indirgenecek belirli özel karakter dizileri ve daha fazlası. Bu özel karakterler normalde programlama dışı kullanımla ilişkilendirilir veya estetik için kullanılır (örneğin, “+” karakteri hem C++ programlamasında hem de temel matematikte yaygın olarak toplama sembolü olarak tanınır).

C++ bu karakter dizilerini özel bir sözdizimi ile işler; ancak özünde, bir operatörün her oluşumu bir işlev çağrısına indirgenir. Örneğin, aşağıdaki C++ ifadesi:

3+3

aşağıdaki işlev çağrısına eşdeğerdir:

operator+(3, 3)

Tüm operatör fonksiyon adları ‘operatör’ ile başlar.

C++‘ın öncülü olan C’de, operatör fonksiyon adlarına farklı tip imzalarla ek tanımlar sağlanarak farklı anlamlar yüklenemezken, C++‘da bu geçerlidir. Ek işlev tanımlarını benzersiz bir işlev adı altında “gizlemek”, C++‘da operatör aşırı yüklemesi olarak adlandırılır ve C++‘da nispeten yaygın, ancak evrensel olmayan bir kuraldır.

İşlev prototiplerinin ve bildirimlerinin görünürlüğü #

C++‘da kod, kullanımdan önce bildirilmeli veya tanımlanmalıdır. Örneğin, aşağıdaki bir derleme zamanı hatası üretir:

int main()
{
  foo(2); // error: foo is called, but has not yet been declared
}

void foo(int x) // this later definition is not known in main
{
}

Bunu çözmenin iki yolu vardır: “main()” içindeki kullanımından önce “foo()“nun tanımını veya bildirimini koymak. İşte bir örnek:

void foo(int x) {}  //Declare the foo function and body first

int main()
{
  foo(2); // OK: foo is completely defined beforehand, so it can be called here.
}

Bununla birlikte, kullanımından önce yalnızca bir “prototip” bildirimi koyarak ve ardından işlev gövdesini daha sonra tanımlayarak işlevi “ileri-bildirmek” de mümkündür:

void foo(int);  // Prototype declaration of foo, seen by main
                // Must specify return type, name, and argument list types
int main()
{
  foo(2); // OK: foo is known, called even though its body is not yet defined
}

void foo(int x) //Must match the prototype
{
    // Define body of foo here
}

Prototip, dönüş tipini (void), fonksiyonun adını (foo) ve argüman listesi değişken tiplerini (int) belirtmelidir, ancak argümanların isimleri gerekli DEĞİLDİR .

Bunu kaynak dosyaların organizasyonuna entegre etmenin yaygın bir yolu, tüm prototip bildirimlerini içeren bir başlık dosyası oluşturmaktır:

// foo.h
void foo(int); // prototype declaration

ve sonra başka bir yerde tam tanımı sağlayın:

// foo.cpp --> foo.o
#include "foo.h" // foo's prototype declaration is "hidden" in here
void foo(int x) { } // foo's body definition

ve sonra, derlendikten sonra, ilgili nesne dosyası “foo.o"yu, bağlantı aşamasında kullanıldığı “main.o” derlenmiş nesne dosyasına bağlayın:

// main.cpp --> main.o
#include "foo.h" // foo's prototype declaration is "hidden" in here
int main() { foo(2); } // foo is valid to call because its prototype declaration was beforehand.
// the prototype and body definitions of foo are linked through the object files

prototype ve call işlevi varken, gövde işlevi tanımlanmadığında bir “çözülmemiş harici simge” hatası oluşur. Derleyici hatayı son bağlantı aşamasına kadar rapor etmeyeceğinden ve hatayı göstermek için kodda hangi satıra atlayacağını bilmediğinden, bunları çözmek daha zor olabilir.

Önişlemci #

Önişlemci, derleyicinin önemli bir parçasıdır.

Kaynak kodunu düzenler, bazı bitleri keser, diğerlerini değiştirir ve başka şeyler ekler.

Kaynak dosyalara önişlemci yönergelerini dahil edebiliriz. Bu yönergeler, önişlemciye belirli eylemleri gerçekleştirmesini söyler. Yönerge, yeni bir satırda # ile başlar. Örnek:

#define ZERO 0

Karşılaşacağınız ilk önişlemci yönergesi muhtemelen

#include <something>

direktif. Yaptığı şey, tüm ‘bir şey’i alır ve direktifin olduğu yere dosyanıza ekler. Merhaba dünya programı şu satırla başlar

#include <iostream>

Bu satır, standart girdi ve çıktıyı kullanmanıza izin veren işlevler ve nesneleri ekler.

Önişlemciyi de kullanan C dilinde, C++ dili kadar başlık dosyası yoktur, ancak C++‘da tüm C başlık dosyalarını kullanabilirsiniz.

Bir sonraki önemli direktif muhtemelen

#define something something_else

direktif. Bu, önişlemciye dosya boyunca ilerlerken, her “bir şey” oluşumunu “başka bir şey” ile değiştirmesi gerektiğini söyler. Ayrıca işlevlere benzer şeyler yapabilir, ancak bu muhtemelen gelişmiş C++ olarak sayılır.

“başka bir şey” gerekli değildir, ancak “bir şey"i hiçbir şey olarak tanımlarsanız, önişlemci yönergelerinin dışında, “bir şey"in tüm oluşumları ortadan kalkar.

Bu aslında ‘#if’, ‘#else’ ve ‘#ifdef’ yönergeleri nedeniyle kullanışlıdır. Bunlar için format aşağıdaki gibi olacaktır:

#if something==true
//code
#else
//more code
#endif

#ifdef thing_that_you_want_to_know_if_is_defined
//code
#endif

Bu yönergeler, doğru bitteki kodu ekler ve yanlış bitleri siler. bu, kodun tamamını yeniden yazmak zorunda kalmadan yalnızca belirli işletim sistemlerinde bulunan kod parçalarına sahip olmak için kullanılabilir.
