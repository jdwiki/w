Sözdizimi #

  • değişken.member_var = sabit;
  • değişken.member_function();
  • değişken_işaretçi->üye_var = sabit;
  • değişken_işaretçi->üye_fonksiyonu();

“struct” ve “class” anahtar sözcükleri arasındaki yalnızca farkın, varsayılan olarak, bir “struct"ın üye değişkenleri, üye işlevleri ve temel sınıflarının “genel” olması olduğuna dikkat edin. bir ‘sınıf’ içinde onlar ‘özel’dir. C++ programcıları, yapıcıları ve yıkıcıları varsa ve kendi değişmezlerini zorlama yeteneği varsa, onu bir sınıf olarak adlandırma eğilimindedir; veya sadece basit bir değerler koleksiyonuysa bir yapı, ancak C++ dilinin kendisi hiçbir ayrım yapmaz.

Sınıfla ilgili temel bilgiler #

sınıf, kullanıcı tanımlı bir türdür. “class”, “struct” veya “union” anahtar sözcüğüyle bir sınıf tanıtılır. Konuşma dilindeki kullanımda, “sınıf” terimi genellikle yalnızca birlik dışı sınıflara atıfta bulunur.

Bir sınıf, sınıf üyelerinden oluşan bir koleksiyondur ve şunlar olabilir:

  • üye değişkenler (“alanlar” olarak da adlandırılır),
  • üye işlevleri (“yöntemler” olarak da adlandırılır),
  • üye türleri veya typedef’ler (ör. “iç içe sınıflar”),
  • üye şablonları (her türden: değişken, işlev, sınıf veya takma ad şablonu)

sınıf anahtarları olarak adlandırılan “sınıf” ve “yapı” anahtar sözcükleri, üyeler ve tabanlar için varsayılan erişim belirtecinin “sınıf” anahtarıyla bildirilen bir sınıf için “özel” ve bir sınıf için “genel” olması dışında büyük ölçüde birbirinin yerine kullanılabilir. “struct” veya “union” anahtarıyla bildirilen sınıf (bkz. Erişim değiştiricileri).

Örneğin, aşağıdaki kod parçacıkları aynıdır:

struct Vector
{
    int x;
    int y;
    int z;
};
// are equivalent to
class Vector
{
public:
    int x;
    int y;
    int z;
};

Bir sınıf bildirerek programınıza yeni bir tür eklenir ve o sınıfın nesnelerini şu şekilde başlatmak mümkündür:

Vector my_vector;

Bir sınıfın üyelerine nokta-sözdizimi kullanılarak erişilir.

my_vector.x = 10;
my_vector.y = my_vector.x + 1; // my_vector.y = 11;
my_vector.z = my_vector.y - 4; // my:vector.z = 7;

Son sınıflar ve yapılar #

Bir sınıfın türetilmesi ‘final’ belirteci ile yasaklanmış olabilir. Son bir sınıf ilan edelim:

class A final {
};

Şimdi, herhangi bir alt sınıf denemesi, bir derleme hatasına neden olacaktır:

// Compilation error: cannot derive from final class:
class B : public A {
};

Son sınıf, sınıf hiyerarşisinde herhangi bir yerde görünebilir:

class A {
};

// OK.
class B final : public A {
};

// Compilation error: cannot derive from final class B.
class C : public B {
};

Erişim belirteçleri #

erişim belirteçleri olarak işlev gören üç anahtar kelime vardır. Bunlar, başka bir belirteç erişim düzeyini tekrar değiştirene kadar, belirteci izleyen sınıf üyelerine erişimi sınırlar:

Anahtar Kelime Tanım
‘kamusal’ Herkesin erişimi var
‘korumalı’ Yalnızca sınıfın kendisi, türetilmiş sınıflar ve arkadaşlar erişebilir
‘özel’ Yalnızca sınıfın kendisi ve arkadaşları erişebilir

Tür, ‘class’ anahtar sözcüğü kullanılarak tanımlandığında, varsayılan erişim belirteci ‘private’ olur, ancak tür ‘struct’ anahtar sözcüğü kullanılarak tanımlanırsa, varsayılan erişim belirteci ‘public’ olur:

struct MyStruct { int x; };
class MyClass { int x; };

MyStruct s;
s.x = 9; // well formed, because x is public

MyClass c;
c.x = 9; // ill-formed, because x is private

Erişim belirteçleri çoğunlukla dahili alanlara ve yöntemlere erişimi sınırlamak ve programcıyı belirli bir arabirimi kullanmaya zorlamak için kullanılır, örneğin bir değişkene doğrudan başvurmak yerine alıcıları ve ayarlayıcıları kullanmaya zorlamak için:

class MyClass {

public: /* Methods: */

    int x() const noexcept { return m_x; }
    void setX(int const x) noexcept { m_x = x; }

private: /* Fields: */

    int m_x;

};

“Korumalı"yı kullanmak, türün belirli işlevlerinin yalnızca türetilmiş sınıflar tarafından erişilebilir olmasına izin vermek için yararlıdır, örneğin aşağıdaki kodda, “calculateValue()” yöntemine yalnızca “Plus2Base” temel sınıfından türetilen sınıflar tarafından erişilebilir. , örneğin “Kırkİki”:

struct Plus2Base {
    int value() noexcept { return calculateValue() + 2; }
protected: /* Methods: */
    virtual int calculateValue() noexcept = 0;
};
struct FortyTwo: Plus2Base {
protected: /* Methods: */
    int calculateValue() noexcept final override { return 40; }
};

“Friend” anahtar sözcüğünün, korumalı ve özel üyelere erişim için işlevlere veya türlere erişim istisnaları eklemek için kullanılabileceğini unutmayın.

‘public’, ‘protected’ ve ‘private’ anahtar sözcükleri, temel sınıf alt nesnelerine erişim vermek veya erişimi sınırlamak için de kullanılabilir. Devralma örneğine bakın.

Miras #

Sınıflar/yapılar kalıtım ilişkilerine sahip olabilir.

Bir sınıf/yapı “B”, bir “A” sınıfından/yapısından miras alırsa, bu, “B"nin ebeveyn olarak “A” olduğu anlamına gelir. “B"nin “A"dan türetilmiş bir sınıf/yapı olduğunu ve “A"nın temel sınıf/yapı olduğunu söylüyoruz.

struct A
{
public:
    int p1;
protected:
    int p2;
private:
    int p3;
};

//Make B inherit publicly (default) from A
struct B : A
{
};

Bir sınıf/yapı için 3 tür kalıtım vardır:

  • “kamu”
  • “özel”
  • “korumalı”

Varsayılan mirasın, üyelerin varsayılan görünürlüğüyle aynı olduğuna dikkat edin: “struct” anahtar sözcüğünü kullanıyorsanız “public” ve “class” anahtar sözcüğü için “private”.

Bir “sınıf"ın bir “yapıdan” türetilmesi bile mümkündür (veya tam tersi). Bu durumda, varsayılan miras çocuk tarafından kontrol edilir, bu nedenle bir “sınıf"tan türetilen bir “yapı” varsayılan olarak genel mirasa sahip olacak ve bir “yapı"dan türetilen bir “sınıf”, varsayılan olarak özel mirasa sahip olacaktır.

‘genel’ miras:

struct B : public A // or just `struct B : A`
{
    void foo()
    {
        p1 = 0; //well formed, p1 is public in B
        p2 = 0; //well formed, p2 is protected in B
        p3 = 0; //ill formed, p3 is private in A
    }
};

B b;
b.p1 = 1; //well formed, p1 is public
b.p2 = 1; //ill formed, p2 is protected
b.p3 = 1; //ill formed, p3 is inaccessible

“özel” miras:

struct B : private A
{
    void foo()
    {
        p1 = 0; //well formed, p1 is private in B
        p2 = 0; //well formed, p2 is private in B
        p3 = 0; //ill formed, p3 is private in A
    }
};

B b;
b.p1 = 1; //ill formed, p1 is private
b.p2 = 1; //ill formed, p2 is private
b.p3 = 1; //ill formed, p3 is inaccessible

“korumalı” miras:

struct B : protected A
{
    void foo()
    {
        p1 = 0; //well formed, p1 is protected in B
        p2 = 0; //well formed, p2 is protected in B
        p3 = 0; //ill formed, p3 is private in A
    }
};

B b;
b.p1 = 1; //ill formed, p1 is protected
b.p2 = 1; //ill formed, p2 is protected
b.p3 = 1; //ill formed, p3 is inaccessible

“Korumalı” mirasa izin verilmesine rağmen, bunun gerçek kullanımının nadir olduğunu unutmayın. Uygulamada “korumalı” kalıtımın nasıl kullanıldığına bir örnek, kısmi temel sınıf uzmanlığıdır (genellikle “kontrollü polimorfizm” olarak adlandırılır).

OOP nispeten yeni olduğunda, (genel) kalıtımın sıklıkla bir “IS-A” ilişkisini modellediği söylenirdi. Diğer bir deyişle, genel kalıtım yalnızca türetilmiş sınıfın bir örneği * aynı zamanda temel sınıfın bir* örneğiyse doğrudur.

Bu daha sonra Liskov Değiştirme İlkesi ile rafine edildi: genel kalıtım yalnızca türetilmiş sınıfın bir örneğinin, olası herhangi bir koşulda temel sınıf (ve yine de mantıklı).

Özel mirasın tipik olarak tamamen farklı bir ilişkiyi içerdiği söylenir: “açısından uygulanır” (bazen “HAS-A” ilişkisi olarak adlandırılır). Örneğin, bir “Stack” sınıfı, bir “Vector” sınıfından özel olarak miras alabilir. Özel kalıtım, toplu kalıtımdan çok daha fazla benzerlik taşır.

Korumalı kalıtım neredeyse hiç kullanılmaz ve ne tür bir ilişki içerdiği konusunda genel bir anlaşma yoktur.

Arkadaşlık #

‘Arkadaş’ anahtar sözcüğü, diğer sınıflara ve işlevlere, sınıfın özel ve korumalı üyelerine erişim sağlamak için kullanılır. sınıfın kapsamı dışında tanımlanır.

class Animal{
private:
    double weight;
    double height;
public:
    friend void printWeight(Animal animal);
    friend class AnimalPrinter;
    // A common use for a friend function is to overload the operator<< for streaming. 
    friend std::ostream& operator<<(std::ostream& os, Animal animal);
};

void printWeight(Animal animal)
{
    std::cout << animal.weight << "\n";
}

class AnimalPrinter
{
public:
    void print(const Animal& animal)
    {
        // Because of the `friend class AnimalPrinter;" declaration, we are
        // allowed to access private members here.
        std::cout << animal.weight << ", " << animal.height << std::endl;
    }
}

std::ostream& operator<<(std::ostream& os, Animal animal)
{
    os << "Animal height: " << animal.height << "\n";
    return os;
}

int main() {
    Animal animal = {10, 5};
    printWeight(animal);

    AnimalPrinter aPrinter;
    aPrinter.print(animal);

    std::cout << animal;
}

10, 5

Sanal Miras #

Kalıtımı kullanırken, “sanal” anahtar sözcüğünü belirtebilirsiniz:

struct A{};
struct B: public virtual A{};

“B” sınıfı “A” sanal tabanına sahip olduğunda, “A"nın kalıtım ağacının en çok türetilmiş sınıfında** bulunacağı ve dolayısıyla en çok türetilmiş sınıfın bu sanal tabanı başlatmaktan sorumlu olduğu anlamına gelir:

struct A
{
    int member;
    A(int param)
    {
        member = param;
    }
};

struct B: virtual A
{
    B(): A(5){}
};

struct C: B
{
    C(): /*A(88)*/ {}
};

void f()
{
    C object; //error since C is not initializing it's indirect virtual base `A`
}

/*A(88)*/ yorumunu kaldırırsak, C şimdi dolaylı sanal taban Ayı başlattığı için herhangi bir hata almayacağız.

Ayrıca, “nesne” değişkeni yaratırken, türetilmiş sınıfın çoğunun “C” olduğunu, dolayısıyla “C"nin “A"yı oluşturmaktan (kurucu çağırma) sorumlu olduğunu ve dolayısıyla “A::member"in değerinin “88” olduğunu unutmayın. , 5 değil (B` türünden bir nesne yaratıyor olsaydık olacağı gibi).

elmas problemini çözerken kullanışlıdır.:

  A                                        A   A
 / \                                       |   |
B   C                                      B   C
 \ /                                        \ /
  D                                          D
virtual inheritance                   normal inheritance

“B” ve “C”, “A"dan miras alır ve “D”, “B” ve “C"den miras alır, bu nedenle “D"de “A"nın 2 örneği vardır! Bu derleyicinin o üyeye hangi sınıftan erişmek istediğinizi (‘B’nin devraldığı veya tarafından miras alınır?).

Sanal kalıtım bu sorunu çözer: Sanal temel yalnızca türetilmiş nesnelerin çoğunda bulunduğundan, ‘D’de yalnızca bir ‘A’ örneği olacaktır.

struct A
{
    void foo() {}
};

struct B : public /*virtual*/ A {};
struct C : public /*virtual*/ A {};

struct D : public B, public C
{
    void bar()
    {
        foo(); //Error, which foo? B::foo() or C::foo()? - Ambiguous
    }
};

Yorumların kaldırılması belirsizliği giderir.

Sınıf üyelerine erişme #

Bir sınıfın nesnesinin üye değişkenlerine ve üye işlevlerine erişmek için . operatörü kullanılır:

struct SomeStruct {
  int a;
  int b;
  void foo() {}
};

SomeStruct var;
// Accessing member variable a in var.
std::cout << var.a << std::endl;
// Assigning member variable b in var.
var.b = 1;
// Calling a member function.
var.foo();

Bir sınıfın üyelerine bir işaretçi aracılığıyla erişirken, -> operatörü yaygın olarak kullanılır. Alternatif olarak, örneğin referansı kaldırılabilir ve bu daha az yaygın olmasına rağmen . operatörü kullanılabilir:

struct SomeStruct {
  int a;
  int b;
  void foo() {}
};

SomeStruct var;
SomeStruct *p = &var;
// Accessing member variable a in var via pointer.
std::cout << p->a << std::endl;
std::cout << (*p).a << std::endl;
// Assigning member variable b in var via pointer.
p->b = 1;
(*p).b = 1;
// Calling a member function via a pointer.
p->foo();
(*p).foo();

Statik sınıf üyelerine erişirken, :: operatörü kullanılır, ancak bunun bir örneği yerine sınıfın adında kullanılır. Alternatif olarak, statik üyeye, statik olmayan üyelere erişimle aynı sözdizimi ile sırasıyla . veya -> operatörü kullanılarak bir örnekten veya bir örneğe yönelik bir işaretçiden erişilebilir.

struct SomeStruct {
  int a;
  int b;
  void foo() {}

  static int c;
  static void bar() {}
};
int SomeStruct::c;

SomeStruct var;
SomeStruct* p = &var;
// Assigning static member variable c in struct SomeStruct.
SomeStruct::c = 5;
// Accessing static member variable c in struct SomeStruct, through var and p.
var.a = var.c;
var.b = p->c;
// Calling a static member function.
SomeStruct::bar();
var.bar();
p->bar();

Arka fon #

‘->’ operatörü gereklidir, çünkü ‘.’ üye erişim operatörü, ‘*’ referans kaldırma operatörüne göre önceliğe sahiptir.

“*p.a"nın “p” referansını kaldırması (“p"nin işaret ettiği nesneye bir başvuruyla sonuçlanır) ve ardından “a” üyesine erişmesi beklenir. Ama aslında, ‘p’nin ‘a’ üyesine erişmeye ve sonra onu kaldırmaya çalışır. yani “p.a”, “(p.a)” ile eşdeğerdir. Yukarıdaki örnekte bu, iki gerçek nedeniyle bir derleyici hatasına neden olur: Birincisi, ‘p’ bir işaretçidir ve ‘a’ üyesine sahip değildir. İkinci olarak, “a” bir tamsayıdır ve bu nedenle referansı kaldırılamaz.

Bu soruna nadiren kullanılan çözüm önceliği açıkça kontrol etmek olacaktır: `(*p).a’

Bunun yerine, neredeyse her zaman -> operatörü kullanılır. İlk önce işaretçinin referansını kaldırmak ve sonra ona erişmek için bir kısa eldir. yani “(*p).a”, “p->a” ile tamamen aynıdır.

:: operatörü, bir ad alanının bir üyesine erişmekle aynı şekilde kullanılan kapsam operatörüdür. Bunun nedeni, statik bir sınıf üyesinin o sınıfın kapsamında kabul edilmesi, ancak o sınıfın örneklerinin bir üyesi olarak kabul edilmemesidir. Normal . ve -> kullanımına, tarihsel nedenlerle, örnek üyeler olmasalar da, statik üyeler için de izin verilir; Bu, arayan kişinin belirli bir üye işlevinin statik mi yoksa statik olmayan mı olduğuyla ilgilenmesi gerekmediğinden, şablonlarda genel kod yazmak için kullanılır.

Özel kalıtım: temel sınıf arabirimini kısıtlama #

Özel kalıtım, sınıfın genel arabirimini kısıtlamak gerektiğinde yararlıdır:

class A {
public:
    int move();
    int turn();
};

class B : private A {
public:
    using A::turn; 
};

B b;
b.move();  // compile error
b.turn();  // OK

Bu yaklaşım, A işaretçisine veya referansına yayın yaparak A genel yöntemlerine erişimi verimli bir şekilde önler:

B b; 
A& a = static_cast<A&>(b); // compile error

Genel miras durumunda, bu tür döküm, türetilmiş B’de bunu önlemenin alternatif yollarına rağmen, gizleme gibi tüm A genel yöntemlerine erişim sağlayacaktır:

class B : public A {
private:
    int move();  
};

veya özel kullanarak:

class B : public A {
private:
    using A::move;  
};

o zaman her iki durumda da mümkündür:

B b;
A& a = static_cast<A&>(b); // OK for public inheritance
a.move(); // OK

İç İçe Sınıflar/Yapılar #

Bir “sınıf” veya “yapı”, kendi içinde “iç içe sınıf” olarak adlandırılan başka bir “sınıf”/“yapı” tanımını da içerebilir; bu durumda, içeren sınıfa “kapsayan sınıf” denir. İç içe sınıf tanımı, çevreleyen sınıfın bir üyesi olarak kabul edilir, ancak bunun dışında ayrıdır.

struct Outer {
    struct Inner { };
};

Çevreleyen sınıfın dışından, iç içe sınıflara kapsam operatörü kullanılarak erişilir. Bununla birlikte, çevreleyen sınıfın içinden, iç içe sınıflar niteleyiciler olmadan kullanılabilir:

struct Outer {
    struct Inner { };

    Inner in;
};

// ...

Outer o;
Outer::Inner i = o.in;

İç içe olmayan bir “sınıf”/“yapı"da olduğu gibi, üye işlevler ve statik değişkenler, iç içe geçmiş bir sınıf içinde veya onu çevreleyen ad alanında tanımlanabilir. Ancak, iç içe geçmiş sınıftan farklı bir sınıf olarak kabul edildiğinden, çevreleyen sınıf içinde tanımlanamazlar.

// Bad.
struct Outer {
    struct Inner {
        void do_something();
    };

    void Inner::do_something() {}
};


// Good.
struct Outer {
    struct Inner {
        void do_something();
    };

};

void Outer::Inner::do_something() {}

İç içe olmayan sınıflarda olduğu gibi, iç içe sınıflar, doğrudan kullanılmadan önce tanımlanmaları koşuluyla ileriye dönük olarak bildirilebilir ve daha sonra tanımlanabilir.

class Outer {
    class Inner1;
    class Inner2;

    class Inner1 {};

    Inner1 in1;
    Inner2* in2p;

  public:
    Outer();
    ~Outer();
};

class Outer::Inner2 {};

Outer::Outer() : in1(Inner1()), in2p(new Inner2) {}
Outer::~Outer() {
    if (in2p) { delete in2p; }
}

C++ 11’den önce, iç içe sınıfların yalnızca tür adlarına, ‘statik’ üyelere ve çevreleyen sınıftan numaralandırıcılara erişimi vardı; çevreleyen sınıfta tanımlanan diğer tüm üyeler sınırsızdı.

C++ 11’den itibaren, iç içe sınıflar ve onların üyeleri, onları çevreleyen sınıfın ‘arkadaşları’ gibi muamele görür ve olağan erişim kurallarına göre tüm üyelerine erişebilir; iç içe sınıfın üyeleri, çevreleyen sınıfın bir veya daha fazla statik olmayan üyesini değerlendirme yeteneğini gerektiriyorsa, bu nedenle bir örnek iletilmelidir:

class Outer {
    struct Inner {
        int get_sizeof_x() {
            return sizeof(x); // Legal (C++11): x is unevaluated, so no instance is required.
        }

        int get_x() {
            return x; // Illegal: Can't access non-static member without an instance.
        }

        int get_x(Outer& o) {
            return o.x; // Legal (C++11): As a member of Outer, Inner can access private members.
        }
    };

    int x;
};

Tersine, çevreleyen sınıf, iç içe sınıfın bir arkadaşı olarak kabul edilmez ve bu nedenle, açıkça izin verilmeden özel üyelerine erişemez.

class Outer {
    class Inner {
        // friend class Outer;

        int x;
    };

    Inner in;

  public:
    int get_x() {
        return in.x; // Error: int Outer::Inner::x is private.
        // Uncomment "friend" line above to fix.
    }
};

Yuvalanmış bir sınıfın arkadaşları, otomatik olarak çevreleyen sınıfın arkadaşları olarak kabul edilmez; eğer çevredeki sınıfın da arkadaşı olmaları gerekiyorsa, bu ayrıca bildirilmelidir. Tersine, çevreleyen sınıf otomatik olarak yuvalanmış sınıfın arkadaşı olarak kabul edilmediğinden, çevreleyen sınıfın arkadaşları da yuvalanmış sınıfın arkadaşları olarak kabul edilmez.

class Outer {
    friend void barge_out(Outer& out, Inner& in);

    class Inner {
        friend void barge_in(Outer& out, Inner& in);

        int i;
    };

    int o;
};

void barge_in(Outer& out, Outer::Inner& in) {
    int i = in.i;  // Good.
    int o = out.o; // Error: int Outer::o is private.
}

void barge_out(Outer& out, Outer::Inner& in) {
    int i = in.i;  // Error: int Outer::Inner::i is private.
    int o = out.o; // Good.
}
son sürüm

Diğer tüm sınıf üyelerinde olduğu gibi, iç içe geçmiş sınıflar yalnızca genel erişime sahiplerse sınıfın dışından adlandırılabilir. Ancak, açıkça adlandırmadığınız sürece erişim değiştiricisinden bağımsız olarak bunlara erişmenize izin verilir.

class Outer {
    struct Inner {
        void func() { std::cout << "I have no private taboo.\n"; }
    };

  public:
    static Inner make_Inner() { return Inner(); }
};

// ...

Outer::Inner oi; // Error: Outer::Inner is private.

auto oi = Outer::make_Inner(); // Good.
oi.func();                     // Good.
Outer::make_Inner().func();    // Good.

Ayrıca iç içe sınıf için bir tür takma adı da oluşturabilirsiniz. Çevreleyen sınıfta bir tür diğer adı varsa, iç içe tür ve tür diğer adının farklı erişim değiştiricileri olabilir. Tür diğer adı çevreleyen sınıfın dışındaysa, iç içe sınıfın veya bunun bir “typedef"inin genel olmasını gerektirir.

class Outer {
    class Inner_ {};

  public:
    typedef Inner_ Inner;
};

typedef Outer::Inner  ImOut; // Good.
typedef Outer::Inner_ ImBad; // Error.

// ...

Outer::Inner  oi; // Good.
Outer::Inner_ oi; // Error.
ImOut         oi; // Good.

Diğer sınıflarda olduğu gibi, iç içe sınıflar, diğer sınıflardan hem türetilebilir hem de türetilebilir.

struct Base {};

struct Outer {
    struct Inner : Base {};
};

struct Derived : Outer::Inner {};

Bu, programcının iç içe sınıfı gerektiği gibi güncellemesine izin vererek, çevreleyen sınıfın başka bir sınıftan türetildiği durumlarda faydalı olabilir. Bu, her bir çevreleyen sınıfın iç içe geçmiş sınıfı için tutarlı bir ad sağlamak üzere bir typedef ile birleştirilebilir:

class BaseOuter {
    struct BaseInner_ {
        virtual void do_something() {}
        virtual void do_something_else();
    } b_in;

  public:
    typedef BaseInner_ Inner;

    virtual ~BaseOuter() = default;

    virtual Inner& getInner() { return b_in; }
};

void BaseOuter::BaseInner_::do_something_else() {}

// ---

class DerivedOuter : public BaseOuter {
    // Note the use of the qualified typedef; BaseOuter::BaseInner_ is private.
    struct DerivedInner_ : BaseOuter::Inner {
        void do_something() override {}
        void do_something_else() override;
    } d_in;

  public:
    typedef DerivedInner_ Inner;

    BaseOuter::Inner& getInner() override { return d_in; }
};

void DerivedOuter::DerivedInner_::do_something_else() {}

// ...

// Calls BaseOuter::BaseInner_::do_something();
BaseOuter* b = new BaseOuter;
BaseOuter::Inner& bin = b->getInner();
bin.do_something();
b->getInner().do_something();

// Calls DerivedOuter::DerivedInner_::do_something();
BaseOuter* d = new DerivedOuter;
BaseOuter::Inner& din = d->getInner();
din.do_something();
d->getInner().do_something();

Yukarıdaki durumda, hem “BaseOuter” hem de “DerivedOuter”, sırasıyla “BaseInner_” ve “DerivedInner_” olarak “Inner” üye türünü sağlar. Bu, iç içe türlerin, çevreleyen sınıfın arabirimini bozmadan türetilmesine ve iç içe türün polimorfik olarak kullanılmasına izin verir.

Üye Türleri ve Takma Adlar #

Bir “sınıf” veya “yapı” ayrıca, sınıfın içinde bulunan ve sınıfın üyeleri olarak kabul edilen tür takma adları olan üye türü takma adlarını da tanımlayabilir.

struct IHaveATypedef {
    typedef int MyTypedef;
};

struct IHaveATemplateTypedef {
    template<typename T>
    using MyTemplateTypedef = std::vector<T>;
};

Statik üyeler gibi, bu typedef’lere kapsam operatörü :: kullanılarak erişilir.

IHaveATypedef::MyTypedef i = 5; // i is an int.

IHaveATemplateTypedef::MyTemplateTypedef<int> v; // v is a std::vector<int>.

Normal tür diğer adlarında olduğu gibi, her üye türü diğer adının, tanımından önce tanımlanmış veya diğer ad verilmiş herhangi bir türe atıfta bulunmasına izin verilir, ancak bundan sonra değil. Benzer şekilde, sınıf tanımının dışındaki bir typedef, sınıf tanımından sonra gelmesi koşuluyla, sınıf tanımı içindeki herhangi bir erişilebilir typedef’e başvurabilir.

template<typename T>
struct Helper {
    T get() const { return static_cast<T>(42); }
};

struct IHaveTypedefs {
//    typedef MyTypedef NonLinearTypedef; // Error if uncommented.
    typedef int MyTypedef;
    typedef Helper<MyTypedef> MyTypedefHelper;
};

IHaveTypedefs::MyTypedef        i; // x_i is an int.
IHaveTypedefs::MyTypedefHelper hi; // x_hi is a Helper<int>.

typedef IHaveTypedefs::MyTypedef TypedefBeFree;
TypedefBeFree ii;                  // ii is an int.

Üye türü takma adları herhangi bir erişim düzeyiyle bildirilebilir ve uygun erişim değiştiricisine saygı gösterir.

class TypedefAccessLevels {
    typedef int PrvInt;

  protected:
    typedef int ProInt;

  public:
    typedef int PubInt;
};

TypedefAccessLevels::PrvInt prv_i; // Error: TypedefAccessLevels::PrvInt is private.
TypedefAccessLevels::ProInt pro_i; // Error: TypedefAccessLevels::ProInt is protected.
TypedefAccessLevels::PubInt pub_i; // Good.

class Derived : public TypedefAccessLevels {
    PrvInt prv_i; // Error: TypedefAccessLevels::PrvInt is private.
    ProInt pro_i; // Good.
    PubInt pub_i; // Good.
};

Bu, bir sınıf tasarımcısının, ona dayanan kodu bozmadan iç işleyişini değiştirmesine izin veren bir soyutlama düzeyi sağlamak için kullanılabilir.

class Something {
    friend class SomeComplexType;

    short s;
    // ...

  public:
    typedef SomeComplexType MyHelper;

    MyHelper get_helper() const { return MyHelper(8, s, 19.5, "shoe", false); }

    // ...
};

// ...

Something s;
Something::MyHelper hlp = s.get_helper();

Bu durumda, yardımcı sınıf “SomeComplexType"dan başka bir türe değiştirilirse, yalnızca “typedef” ve “friend” bildiriminin değiştirilmesi gerekir; helper sınıfı aynı işlevi sağladığı sürece, onu adıyla belirtmek yerine Something::MyHelper olarak kullanan herhangi bir kod, genellikle herhangi bir değişiklik yapılmadan çalışmaya devam edecektir. Bu şekilde, temel uygulama değiştirildiğinde değiştirilmesi gereken kod miktarını en aza indiririz, öyle ki tür adının yalnızca bir konumda değiştirilmesi gerekir.

Bu, istenirse “decltype” ile de birleştirilebilir.

class SomethingElse {
    AnotherComplexType<bool, int, SomeThirdClass> helper;

  public:
    typedef decltype(helper) MyHelper;

  private:
    InternalVariable<MyHelper> ivh;

    // ...

  public:
    MyHelper& get_helper() const { return helper; }

    // ...
};

Bu durumda, ‘SomethingElse::helper’ uygulamasının değiştirilmesi, ‘decltype’ nedeniyle typedef’i bizim için otomatik olarak değiştirecektir. Bu, “yardımcı"yı değiştirmek istediğimizde gerekli değişiklik sayısını en aza indirerek insan hatası riskini en aza indirir.

Ancak her şeyde olduğu gibi bu da çok ileri götürülebilir. Örneğin, tür adı dahili olarak yalnızca bir veya iki kez ve harici olarak sıfır kez kullanılıyorsa, bunun için bir takma ad sağlamaya gerek yoktur. Bir proje boyunca yüzlerce veya binlerce kez kullanılıyorsa veya yeterince uzun bir adı varsa, onu her zaman mutlak terimlerle kullanmak yerine typedef olarak sağlamak faydalı olabilir. İleriye dönük uyumluluk ve rahatlığı, yaratılan gereksiz gürültü miktarıyla dengelemek gerekir.

Bu, şablon parametrelerine sınıfın dışından erişim sağlamak için şablon sınıflarıyla da kullanılabilir.

template<typename T>
class SomeClass {
    // ...

  public:
    typedef T MyParam;
    MyParam getParam() { return static_cast<T>(42); }
};

template<typename T>
typename T::MyParam some_func(T& t) {
    return t.getParam();
}

SomeClass<int> si;
int i = some_func(si);

Bu, genellikle öğe türlerini ve diğer yardımcı türlerini üye türü takma adları olarak sağlayan kaplarla birlikte kullanılır. Örneğin, C++ standart kitaplığındaki kapların çoğu, ihtiyaç duyabilecekleri diğer özel türlerle birlikte aşağıdaki 12 yardımcı türü sağlar.

template<typename T>
class SomeContainer {
    // ...

  public:
    // Let's provide the same helper types as most standard containers.
    typedef T                                     value_type;
    typedef std::allocator<value_type>            allocator_type;
    typedef value_type&                           reference;
    typedef const value_type&                     const_reference;
    typedef value_type*                           pointer;
    typedef const value_type*                     const_pointer;
    typedef MyIterator<value_type>                iterator;
    typedef MyConstIterator<value_type>           const_iterator;
    typedef std::reverse_iterator<iterator>       reverse_iterator;
    typedef std::reverse_iterator<const_iterator> const_reverse_iterator;
    typedef size_t                                size_type;
    typedef ptrdiff_t                             difference_type;
};

C++ 11’den önce, özellik henüz mevcut olmadığı için bir tür “template ’typedef’” sağlamak için de yaygın olarak kullanılıyordu; bunlar takma ad şablonlarının tanıtılmasıyla biraz daha az yaygın hale geldi, ancak bazı durumlarda hala kullanışlıdır (ve diğer durumlarda, işlev işaretçisi gibi karmaşık bir türdeki bireysel bileşenleri elde etmek için çok yararlı olabilecek takma ad şablonlarıyla birleştirilir) ). Tür takma adları için genellikle “type” adını kullanırlar.

template<typename T>
struct TemplateTypedef {
    typedef T type;
}

TemplateTypedef<int>::type i; // i is an int.

Bu, bir veya daha fazla parametreyi tanımlayan bir takma ad sağlamak için genellikle birden çok şablon parametresine sahip türlerle kullanıldı.

template<typename T, size_t SZ, size_t D>
class Array { /* ... */ };

template<typename T, size_t SZ>
struct OneDArray {
    typedef Array<T, SZ, 1> type;
};

template<typename T, size_t SZ>
struct TwoDArray {
    typedef Array<T, SZ, 2> type;
};

template<typename T>
struct MonoDisplayLine {
    typedef Array<T, 80, 1> type;
};

OneDArray<int, 3>::type     arr1i; // arr1i is an Array<int, 3, 1>.
TwoDArray<short, 5>::type   arr2s; // arr2s is an Array<short, 5, 2>.
MonoDisplayLine<char>::type arr3c; // arr3c is an Array<char, 80, 1>.

Çoklu Kalıtım #

Tek miras dışında:

class A {};
class B : public A {};

Ayrıca birden fazla mirasa sahip olabilirsiniz:

class A {};
class B {};
class C : public A, public B {};

‘C’ şimdi aynı anda ‘A’ ve ‘B’den miras alacak.

Not: Aynı adlar birden çok kalıtsal “sınıf” veya “yapı"da kullanılıyorsa, bu belirsizliğe yol açabilir. Dikkatli olun!

Bazı durumlarda çoklu kalıtım yardımcı olabilir, ancak bazen çoklu kalıtım kullanılırken garip bir sorunla karşılaşılır.

Örneğin: İki temel sınıf, türetilmiş sınıfta geçersiz kılınmayan aynı ada sahip işlevlere sahiptir ve türetilmiş sınıfın nesnesini kullanarak bu işleve erişmek için kod yazarsanız, derleyici hangi işlevi çağıracağını belirleyemediği için hata gösterir. İşte çoklu kalıtımda bu tür belirsizlik için bir kod.

class base1
{
  public:
     void funtion( )
     { //code for base1 function }  
};
class base2
{
    void function( )
     { // code for base2 function } 
};

class derived : public base1, public base2
{
    
};

    derived obj;
    
  // Error because compiler can't figure out which function to call

Statik sınıf üyeleri #

Bir sınıfın, değişkenler veya işlevler olabilen ‘statik’ üyelere sahip olmasına da izin verilir. Bunlar, sınıfın kapsamında kabul edilir, ancak normal üyeler olarak kabul edilmez; statik depolama süreleri vardır (programın başlangıcından sonuna kadar bulunurlar), sınıfın belirli bir örneğine bağlı değildirler ve tüm sınıf için yalnızca bir kopya vardır.

class Example {
    static int num_instances;      // Static data member (static member variable).
    int i;                         // Non-static member variable.

  public:
    static std::string static_str; // Static data member (static member variable).
    static int static_func();      // Static member function.

    // Non-static member functions can modify static member variables.
    Example() { ++num_instances; }
    void set_str(const std::string& str);
};

int         Example::num_instances;
std::string Example::static_str = "Hello.";

// ...

Example one, two, three;
// Each Example has its own "i", such that:
//  (&one.i != &two.i)
//  (&one.i != &three.i)
//  (&two.i != &three.i).
// All three Examples share "num_instances", such that:
//  (&one.num_instances == &two.num_instances)
//  (&one.num_instances == &three.num_instances)
//  (&two.num_instances == &three.num_instances)

Statik üye değişkenlerin sınıf içinde tanımlandığı, yalnızca bildirildiği ve dolayısıyla tanımlarının sınıf tanımının dışında olduğu kabul edilir; programcının tanımlarında statik değişkenleri başlatmasına izin verilir, ancak zorunlu değildir. Üye değişkenleri tanımlarken, ‘static’ anahtar sözcüğü atlanır.

class Example {
    static int num_instances;               // Declaration.

  public:
    static std::string static_str;          // Declaration.

    // ...
};

int         Example::num_instances;         // Definition.  Zero-initialised.
std::string Example::static_str = "Hello."; // Definition.

Bu nedenle, statik değişkenler, daha sonra tam bir tür olarak tanımlandıkları sürece, tamamlanmamış türler (“void” dışında) olabilir.

struct ForwardDeclared;

class ExIncomplete {
    static ForwardDeclared fd;
    static ExIncomplete    i_contain_myself;
    static int             an_array[];
};

struct ForwardDeclared {};

ForwardDeclared ExIncomplete::fd;
ExIncomplete    ExIncomplete::i_contain_myself;
int             ExIncomplete::an_array[5];

Statik üye işlevleri, normal üye işlevlerinde olduğu gibi sınıf tanımının içinde veya dışında tanımlanabilir. Statik üye değişkenlerinde olduğu gibi, sınıf tanımının dışında statik üye işlevleri tanımlanırken ‘static’ anahtar sözcüğü atlanır.

// For Example above, either...
class Example {
    // ...

  public:
    static int static_func() { return num_instances; }

    // ...

    void set_str(const std::string& str) { static_str = str; }
};

// Or...

class Example { /* ... */ };

int  Example::static_func() { return num_instances; }
void Example::set_str(const std::string& str) { static_str = str; }

Bir statik üye değişkeni “const” olarak bildirilmiş ancak “geçici” değilse ve bir integral veya numaralandırma türündeyse, sınıf tanımı içinde, bildirimde başlatılabilir.

enum E { VAL = 5 };

struct ExConst {
    const static int ci = 5;              // Good.
    static const E ce = VAL;              // Good.
    const static double cd = 5;           // Error.
    static const volatile int cvi = 5;    // Error.

    const static double good_cd;
    static const volatile int good_cvi;
};

const double ExConst::good_cd = 5;        // Good.
const volatile int ExConst::good_cvi = 5; // Good.

C++ 11’den itibaren, “LiteralType” türlerinin (derleme zamanında “constexpr” kurallarına göre oluşturulabilen türler) statik üye değişkenleri de “constexpr” olarak bildirilebilir; eğer öyleyse, sınıf tanımı içinde başlatılmaları gerekir.

struct ExConstexpr {
    constexpr static int ci = 5;                      // Good.
    static constexpr double cd = 5;                   // Good.
    constexpr static int carr[] = { 1, 1, 2 };        // Good.
    static constexpr ConstexprConstructibleClass c{}; // Good.
    constexpr static int bad_ci;                      // Error.
};

constexpr int ExConstexpr::bad_ci = 5;                // Still an error.
son sürüm

Bir “const” veya “constexpr” statik üye değişkeni odr-used ise (resmi olmayan, adresi alınmışsa veya bir referansa atanmışsa), sınıf tanımının dışında yine de ayrı bir tanımı olmalıdır. Bu tanımın bir başlatıcı içermesine izin verilmez.

struct ExODR {
    static const int odr_used = 5;
};

// const int ExODR::odr_used;

const int* odr_user = & ExODR::odr_used; // Error; uncomment above line to resolve.

Statik üyeler belirli bir örneğe bağlı olmadığından, bunlara kapsam operatörü :: kullanılarak erişilebilir.

std::string str = Example::static_str;

Ayrıca normal, statik olmayan üyelermiş gibi erişilebilirler. Bu tarihsel öneme sahiptir, ancak bir üyenin statik olup olmadığı konusunda kafa karışıklığını önlemek için kapsam operatöründen daha az kullanılır.

Example ex;
std::string rts = ex.static_str;

Sınıf üyeleri, statik olmayan sınıf üyelerinde olduğu gibi, kapsamlarını nitelemeden statik üyelere erişebilir.

class ExTwo {
    static int num_instances;
    int my_num;

  public:
    ExTwo() : my_num(num_instances++) {}

    static int get_total_instances() { return num_instances; }
    int get_instance_number() const { return my_num; }
};

int ExTwo::num_instances;

“Değişken” olamazlar ve olmaları da gerekmez; herhangi bir örneğe bağlı olmadıklarından, bir örneğin const olup olmaması statik üyeleri etkilemez.

struct ExDontNeedMutable {
    int immuta;
    mutable int muta;

    static int i;

    ExDontNeedMutable() : immuta(-5), muta(-5) {}
};
int ExDontNeedMutable::i;

// ...

const ExDontNeedMutable dnm;
dnm.immuta = 5; // Error: Can't modify read-only object.
dnm.muta = 5;   // Good.  Mutable fields of const objects can be written.
dnm.i = 5;      // Good.  Static members can be written regardless of an instance's const-ness.

Statik üyeler, tıpkı statik olmayan üyeler gibi erişim değiştiricilerine saygı duyar.

class ExAccess {
    static int prv_int;

  protected:
    static int pro_int;

  public:
    static int pub_int;
};

int ExAccess::prv_int;
int ExAccess::pro_int;
int ExAccess::pub_int;

// ...

int x1 = ExAccess::prv_int; // Error: int ExAccess::prv_int is private.
int x2 = ExAccess::pro_int; // Error: int ExAccess::pro_int is protected.
int x3 = ExAccess::pub_int; // Good.

Belirli bir örneğe bağlı olmadıkları için, statik üye işlevlerin ‘bu’ işaretçisi yoktur; bu nedenle, bir örnek geçmedikçe statik olmayan üye değişkenlere erişemezler.

class ExInstanceRequired {
    int i;

  public:
    ExInstanceRequired() : i(0) {}

    static void bad_mutate() { ++i *= 5; }                         // Error.
    static void good_mutate(ExInstanceRequired& e) { ++e.i *= 5; } // Good.
};

Bir “bu” işaretçisine sahip olmadığı için, adresleri işaretçiler-üye işlevlerinde saklanamaz ve bunun yerine normal işaretçilerden işlevlere depolanır.

struct ExPointer {
           void nsfunc() {}
    static void  sfunc() {}
};

typedef void (ExPointer::* mem_f_ptr)();
typedef void (*f_ptr)();

mem_f_ptr p_sf = &ExPointer::sfunc; // Error.
    f_ptr p_sf = &ExPointer::sfunc; // Good.

Bir “this” işaretçisine sahip olmadıklarından, “const” veya “volatile” olamazlar ve ref niteleyicileri olamazlar. Ayrıca sanal olamazlar.

struct ExCVQualifiersAndVirtual {
    static void   func()                {} // Good.
    static void  cfunc() const          {} // Error.
    static void  vfunc() volatile       {} // Error.
    static void cvfunc() const volatile {} // Error.
    static void  rfunc() &              {} // Error.
    static void rvfunc() &&             {} // Error.

    virtual static void vsfunc()        {} // Error.
    static virtual void svfunc()        {} // Error.
};

Belirli bir örneğe bağlı olmadıkları için, statik üye değişkenleri etkin bir şekilde özel global değişkenler olarak ele alınır; sınıfın herhangi bir örneğinin gerçekten var olup olmadığına bakılmaksızın program başladığında oluşturulurlar ve program çıktığında yok edilirler. Her statik üye değişkenin yalnızca tek bir kopyası mevcuttur (değişken ’thread_local’ (C++11 veya üstü) olarak bildirilmedikçe, bu durumda iş parçacığı başına bir kopya vardır).

Statik üye değişkenler, sınıfın harici veya dahili bağlantıya sahip olup olmadığına bakılmaksızın sınıfla aynı bağlantıya sahiptir. Yerel sınıfların ve adsız sınıfların statik üyelere sahip olmasına izin verilmez.

Statik olmayan üye işlevleri #

Bir sınıf, sınıfın tek tek örnekleri üzerinde çalışan statik olmayan üye işlevlerine sahip olabilir.

class CL {
  public:
    void member_function() {}
};

Bu işlevler, sınıfın bir örneğinde şu şekilde çağrılır:

CL instance;
instance.member_function();

Sınıf tanımının içinde veya dışında tanımlanabilirler; dışarıda tanımlanmışlarsa, sınıfın kapsamında olarak belirtilirler.

struct ST {
    void  defined_inside() {}
    void defined_outside();
};
void ST::defined_outside() {}

Çağrıldıkları örneği nasıl gördüklerini etkileyen CV için uygun ve/veya başvuru için nitelikli olabilirler; işlev, örneği, varsa belirtilen özgeçmiş niteleyicilerine sahip olarak görecektir. Hangi sürümün çağrılacağı, örneğin özgeçmiş niteleyicilerine dayalı olacaktır. Örnekle aynı özgeçmiş niteleyicilerine sahip bir sürüm yoksa, varsa daha fazla özgeçmiş nitelikli bir sürüm çağrılır.

struct CVQualifiers {
    void func()                   {} // 1: Instance is non-cv-qualified.
    void func() const             {} // 2: Instance is const.

    void cv_only() const volatile {}
};

CVQualifiers       non_cv_instance;
const CVQualifiers      c_instance;

non_cv_instance.func(); // Calls #1.
c_instance.func();      // Calls #2.

non_cv_instance.cv_only(); // Calls const volatile version.
c_instance.cv_only();      // Calls const volatile version.

Üye işlevi ref niteleyicileri, işlevin değer örneklerinde çağrılmasının amaçlanıp tasarlanmadığını belirtir ve işlev cv niteleyicileriyle aynı sözdizimini kullanır.

struct RefQualifiers {
    void func() &  {} // 1: Called on normal instances.
    void func() && {} // 2: Called on rvalue (temporary) instances.
};

RefQualifiers rf;
rf.func();              // Calls #1.
RefQualifiers{}.func(); // Calls #2.

CV niteleyicileri ve ref niteleyicileri de gerekirse birleştirilebilir.

struct BothCVAndRef {
    void func() const& {} // Called on normal instances.  Sees instance as const.
    void func() &&     {} // Called on temporary instances.
};
son sürüm

Ayrıca sanal olabilirler; bu, polimorfizm için temeldir ve bir alt sınıf(lar)ın kendi işlevselliklerini sağlarken üst sınıfla aynı arabirimi sağlamasına izin verir.

struct Base {
    virtual void func() {}
};
struct Derived {
    virtual void func() {}
};

Base* bp = new Base;
Base* dp = new Derived;
bp.func(); // Calls Base::func().
dp.func(); // Calls Derived::func().

Daha fazla bilgi için buraya bakın.

    for (const auto& point : points) { 
        std::cout << "{" << point.x << ", " << point.y << "}\n";
    }

    decltype(points[0]) topRightCorner{1, 1};
    auto it = std::find(points, points + 4, topRightCorner);
    std::cout << "top right corner is the "
              << 1 + std::distance(points, it) << "th\n";
}
