İstisnaları yakalama #

İstisnaları yakalamak için bir “try/catch” bloğu kullanılır. “try” bölümündeki kod, bir istisna oluşturabilecek koddur ve “catch” yan tümcesindeki kod, istisnayı ele alır.

#include <iostream>
#include <string>
#include <stdexcept>

int main() {
  std::string str("foo");
  
  try {
      str.at(10); // access element, may throw std::out_of_range
  } catch (const std::out_of_range& e) {
      // what() is inherited from std::exception and contains an explanatory message
      std::cout << e.what();
  }
}

Birden çok istisna türünü işlemek için birden çok “catch” yan tümcesi kullanılabilir. Birden çok “catch” yan tümcesi varsa, istisna işleme mekanizması bunları koddaki görünümlerine göre sırasıyla eşleştirmeye çalışır:

std::string str("foo");
  
try {
    str.reserve(2); // reserve extra capacity, may throw std::length_error
    str.at(10); // access element, may throw std::out_of_range
} catch (const std::length_error& e) {
    std::cout << e.what();
} catch (const std::out_of_range& e) {
    std::cout << e.what();
}

Ortak bir temel sınıftan türetilen istisna sınıfları, ortak temel sınıf için tek bir “catch” yan tümcesi ile yakalanabilir. Yukarıdaki örnek, “std::length_error” ve “std::out_of_range” için iki “catch” yan tümcesini, “std:exception” için tek bir yan tümceyle değiştirebilir:

std::string str("foo");
  
try {
    str.reserve(2); // reserve extra capacity, may throw std::length_error
    str.at(10); // access element, may throw std::out_of_range
} catch (const std::exception& e) {
    std::cout << e.what();
}

‘catch’ yan tümceleri sırayla denendiğinden, önce daha özel yakalama yan tümceleri yazdığınızdan emin olun, aksi takdirde istisna işleme kodunuz hiçbir zaman çağrılmayabilir:

try {
    /* Code throwing exceptions omitted. */
} catch (const std::exception& e) {
    /* Handle all exceptions of type std::exception. */
} catch (const std::runtime_error& e) {
    /* This block of code will never execute, because std::runtime_error inherits
       from std::exception, and all exceptions of type std::exception were already
       caught by the previous catch clause. */
}

Diğer bir olasılık, fırlatılan herhangi bir nesneyi yakalayacak olan tümünü yakalama işleyicisidir:

try {
    throw 10;
} catch (...) {
    std::cout << "caught an exception";
}

Rethrow (yayma) istisnası #

Bazen, yakaladığınız istisna ile bir şeyler yapmak istersiniz (günlüğe yazmak veya bir uyarı yazdırmak gibi) ve işlenecek üst kapsama kadar kabarmasına izin verin. Bunu yapmak için, yakaladığınız herhangi bir istisnayı yeniden gönderebilirsiniz:

try {
    ... // some code here
} catch (const SomeException& e) {
    std::cout << "caught an exception";
    throw;
}

Argümanlar olmadan throw; kullanmak, o anda yakalanan istisnayı yeniden atar.

Yönetilen bir ‘std::exception_ptr’yi yeniden göndermek için, C++ Standart Kitaplığı, programınıza ‘’ başlığını dahil ederek kullanılabilecek ‘rethrow_exception’ işlevine sahiptir.

#include <iostream>
#include <string>
#include <exception>
#include <stdexcept>
 
void handle_eptr(std::exception_ptr eptr) // passing by value is ok
{
    try {
        if (eptr) {
            std::rethrow_exception(eptr);
        }
    } catch(const std::exception& e) {
        std::cout << "Caught exception \"" << e.what() << "\"\n";
    }
}
 
int main()
{
    std::exception_ptr eptr;
    try {
        std::string().at(1); // this generates an std::out_of_range
    } catch(...) {
        eptr = std::current_exception(); // capture
    }
    handle_eptr(eptr);
} // destructor for std::out_of_range called here, when the eptr is destructed

En iyi uygulama: değere göre atma, sabit referansa göre yakalama #

Genel olarak, (işaretçi yerine) değere göre atmak, ancak (const) referansına göre yakalamak iyi bir uygulama olarak kabul edilir.

try {
    // throw new std::runtime_error("Error!");   // Don't do this!
    // This creates an exception object
    // on the heap and would require you to catch the
    // pointer and manage the memory yourself. This can
    // cause memory leaks!
    
    throw std::runtime_error("Error!");
} catch (const std::runtime_error& e) {
    std::cout << e.what() << std::endl;
}

Referans yoluyla yakalamanın iyi bir uygulama olmasının bir nedeni, yakalama bloğuna geçirilirken (veya diğer yakalama bloklarına yayılırken) nesneyi yeniden yapılandırma ihtiyacını ortadan kaldırmasıdır. Başvuruya göre yakalama ayrıca istisnaların polimorfik olarak ele alınmasına izin verir ve nesne dilimlemeyi önler. Ancak, bir istisnayı yeniden oluşturuyorsanız (’throw e;’ gibi, aşağıdaki örneğe bakın), yine de nesne dilimleme elde edebilirsiniz çünkü ’throw e;’ ifadesi, bildirildiği tür ne olursa olsun istisnanın bir kopyasını oluşturur:

#include <iostream>

struct BaseException {
    virtual const char* what() const { return "BaseException"; }
};

struct DerivedException : BaseException {
    // "virtual" keyword is optional here
    virtual const char* what() const { return "DerivedException"; }
};

int main(int argc, char** argv) {
    try {
        try {
            throw DerivedException();
        } catch (const BaseException& e) {
            std::cout << "First catch block: " << e.what() << std::endl;
            // Output ==> First catch block: DerivedException

            throw e; // This changes the exception to BaseException
                     // instead of the original DerivedException!
        }
    } catch (const BaseException& e) {
        std::cout << "Second catch block: " << e.what() << std::endl;
        // Output ==> Second catch block: BaseException
    }
    return 0;
}

İstisnayı değiştirmek için herhangi bir şey yapmayacağınızdan eminseniz (bilgi eklemek veya mesajı değiştirmek gibi), const referansı ile yakalamak derleyicinin optimizasyon yapmasına izin verir ve performansı artırabilir. Ancak bu yine de nesne eklemeye neden olabilir (yukarıdaki örnekte görüldüğü gibi).

Uyarı: Özellikle fazladan bellek veya kaynak ayırmayla ilgili olarak “catch” bloklarında istenmeyen istisnalar oluşturmamaya dikkat edin. Örneğin, “logic_error”, “runtime_error” veya bunların alt sınıflarını oluşturmak, istisna dizesini kopyalarken belleğin bitmesi nedeniyle “bad_alloc” atabilir, G/Ç akışları, ilgili istisna maskeleri seti ile günlük kaydı sırasında atabilir, vb.

standart::uncaught_istisnalar #

C++17, şu anda kaç istisnanın yakalanmadığını bilmek için ‘int std::uncaught_exceptions()’ (sınırlı ‘bool std::uncaught_exception()’ yerine geçer) tanıtır. Bu, bir sınıfın, bir yığının çözülmesi sırasında yok edilip edilmediğini belirlemesine izin verir.

#include <exception>
#include <string>
#include <iostream>

// Apply change on destruction:
// Rollback in case of exception (failure)
// Else Commit (success)
class Transaction
{
public:
    Transaction(const std::string& s) : message(s) {}
    Transaction(const Transaction&) = delete;
    Transaction& operator =(const Transaction&) = delete;
    void Commit() { std::cout << message << ": Commit\n"; }
    void RollBack() noexcept(true) { std::cout << message << ": Rollback\n"; }

    // ...

    ~Transaction() {
        if (uncaughtExceptionCount == std::uncaught_exceptions()) {
            Commit(); // May throw.
        } else { // current stack unwinding
            RollBack();
        }
    }

private:
    std::string message;
    int uncaughtExceptionCount = std::uncaught_exceptions();
};

class Foo
{
public:
    ~Foo() {
        try {
            Transaction transaction("In ~Foo"); // Commit,
                                            // even if there is an uncaught exception
            //...
        } catch (const std::exception& e) {
            std::cerr << "exception/~Foo:" << e.what() << std::endl;
        }
    }
};

int main()
{
    try {
        Transaction transaction("In main"); // RollBack
        Foo foo; // ~Foo commit its transaction.
        //...
        throw std::runtime_error("Error");
    } catch (const std::exception& e) {
        std::cerr << "exception/main:" << e.what() << std::endl;
    }
}

Çıktı:

In ~Foo: Commit
In main: Rollback
exception/main:Error

Özel istisna #

Ham değerleri istisna olarak atmamalısınız, bunun yerine standart istisna sınıflarından birini kullanmalı veya kendinizinkini oluşturmalısınız.

‘std::exception’dan miras alınan kendi istisna sınıfınıza sahip olmak, bunun için iyi bir yoldur. İşte doğrudan “std::exception"dan miras alan özel bir istisna sınıfı:

#include <exception>

class Except: virtual public std::exception {
    
protected:

    int error_number;               ///< Error number
    int error_offset;               ///< Error offset
    std::string error_message;      ///< Error message
    
public:

    /** Constructor (C++ STL string, int, int).
     *  @param msg The error message
     *  @param err_num Error number
     *  @param err_off Error offset
     */
    explicit 
    Except(const std::string& msg, int err_num, int err_off):
        error_number(err_num),
        error_offset(err_off),
        error_message(msg)
        {}

    /** Destructor.
     *  Virtual to allow for subclassing.
     */
    virtual ~Except() throw () {}

    /** Returns a pointer to the (constant) error description.
     *  @return A pointer to a const char*. The underlying memory
     *  is in possession of the Except object. Callers must
     *  not attempt to free the memory.
     */
    virtual const char* what() const throw () {
       return error_message.c_str();
    }
    
    /** Returns error number.
     *  @return #error_number
     */
    virtual int getErrorNumber() const throw() {
        return error_number;
    }
    
    /**Returns error offset.
     * @return #error_offset
     */
    virtual int getErrorOffset() const throw() {
        return error_offset;
    }

};

Örnek bir atış yakalama:

try {
    throw(Except("Couldn't do what you were expecting", -12, -34));
} catch (const Except& e) {
    std::cout<<e.what()
             <<"\nError number: "<<e.getErrorNumber()
             <<"\nError offset: "<<e.getErrorOffset();
}

Sadece aptal bir hata mesajı değil, aynı zamanda hatanın tam olarak ne olduğunu gösteren başka değerler de atmadığınızdan, hata işlemeniz çok daha verimli ve anlamlı hale gelir.

Hata mesajlarını güzelce işlemenizi sağlayan bir istisna sınıfı var :std::runtime_error

Bu sınıftan da miras alabilirsiniz:

#include <stdexcept>

class Except: virtual public std::runtime_error {
    
protected:

    int error_number;               ///< Error number
    int error_offset;               ///< Error offset
    
public:

    /** Constructor (C++ STL string, int, int).
     *  @param msg The error message
     *  @param err_num Error number
     *  @param err_off Error offset
     */
    explicit 
    Except(const std::string& msg, int err_num, int err_off):
        std::runtime_error(msg)
        {
            error_number = err_num;
            error_offset = err_off;
            
        }

    /** Destructor.
     *  Virtual to allow for subclassing.
     */
    virtual ~Except() throw () {}
    
    /** Returns error number.
     *  @return #error_number
     */
    virtual int getErrorNumber() const throw() {
        return error_number;
    }
    
    /**Returns error offset.
     * @return #error_offset
     */
    virtual int getErrorOffset() const throw() {
        return error_offset;
    }

};

Temel sınıftan (‘std::runtime_error’) ‘what()’ işlevini geçersiz kılmadığıma dikkat edin, yani temel sınıfın ‘what()’ sürümünü kullanacağız. Daha fazla gündeminiz varsa geçersiz kılabilirsiniz.

Hangisine eşdeğerdir

Yapıcılar ve yıkıcılar için, catch bloğu yine de bir istisnayı yeniden attığından davranışın farklı olduğuna dikkat edin ( catch blok gövdesinde başka bir atış yoksa yakalanan).

“main” işlevinin diğer işlevler gibi bir işlev try bloğuna sahip olmasına izin verilir, ancak “main” işlevinin try bloğu, yerel olmayan bir statik değişkenin oluşturulması veya herhangi bir statik değişkenin yok edilmesi sırasında meydana gelen istisnaları yakalamaz. . Bunun yerine, ‘std::terminate’ çağrılır.

İç içe istisna #

İstisna işleme sırasında, düşük seviyeli bir işlevden (dosya sistemi hatası veya veri aktarım hatası gibi) genel bir istisna yakaladığınızda ve bazı yüksek seviyeli işlemlerin olabileceğini gösteren daha spesifik bir yüksek seviye istisnası attığınızda yaygın bir kullanım durumu vardır. gerçekleştirilemez (örneğin, Web’de bir fotoğraf yayınlayamamak gibi). Bu, istisna işlemenin yüksek seviyeli işlemlerle belirli sorunlara tepki vermesine izin verir ve ayrıca, yalnızca bir hata mesajıyla programcının uygulamada bir istisnanın oluştuğu bir yer bulmasına izin verir. Bu çözümün dezavantajı, istisna çağrı yığınının kesilmesi ve orijinal istisnanın kaybolmasıdır. Bu, geliştiricileri orijinal istisna metnini yeni oluşturulan bir istisna metnine manuel olarak eklemeye zorlar.

İç içe istisnalar, nedeni açıklayan düşük seviyeli istisnayı, bu özel durumda ne anlama geldiğini açıklayan yüksek seviyeli bir istisnaya ekleyerek sorunu çözmeyi amaçlar.

std::nested_exception, std::throw_with_nested sayesinde istisnaları iç içe geçirmeye izin verir:

#include <stdexcept>
#include <exception>
#include <string>
#include <fstream>
#include <iostream>

struct MyException
{
    MyException(const std::string& message) : message(message) {}
    std::string message;
};

void print_current_exception(int level)
{
    try {
        throw;
    } catch (const std::exception& e) {
        std::cerr << std::string(level, ' ') << "exception: " << e.what() << '\n';
    } catch (const MyException& e) {
        std::cerr << std::string(level, ' ') << "MyException: " << e.message << '\n';
    } catch (...) {
        std::cerr << "Unkown exception\n";
    }
}

void print_current_exception_with_nested(int level =  0)
{
    try {
        throw;
    } catch (...) {
        print_current_exception(level);
    }    
    try {
        throw;
    } catch (const std::nested_exception& nested) {
        try {
            nested.rethrow_nested();
        } catch (...) {
            print_current_exception_with_nested(level + 1); // recursion
        }
    } catch (...) {
        //Empty // End recursion
    }
}

// sample function that catches an exception and wraps it in a nested exception
void open_file(const std::string& s)
{
    try {
        std::ifstream file(s);
        file.exceptions(std::ios_base::failbit);
    } catch(...) {
        std::throw_with_nested(MyException{"Couldn't open " + s});
    }
}
 
// sample function that catches an exception and wraps it in a nested exception
void run()
{
    try {
        open_file("nonexistent.file");
    } catch(...) {
        std::throw_with_nested( std::runtime_error("run() failed") );
    }
}
 
// runs the sample function above and prints the caught exception
int main()
{
    try {
        run();
    } catch(...) {
        print_current_exception_with_nested();
    }
}

Olası çıktı:

exception: run() failed
 MyException: Couldn't open nonexistent.file
  exception: basic_ios::clear

Yalnızca ‘std::exception’dan devralınan istisnalarla çalışıyorsanız, kod basitleştirilebilir bile.

Yapıcıda İşlev Deneme Blokları #

Başlatıcı listesindeki istisnayı yakalamanın tek yolu:

struct A : public B
{
    A() try : B(), foo(1), bar(2)
    {
        // constructor body 
    }
    catch (...)
    {
        // exceptions from the initializer list and constructor are caught here
        // if no exception is thrown here
        // then the caught exception is re-thrown.
    }
 
private:
    Foo foo;
    Bar bar;
};

Yıkıcıda İşlev Deneme Blokları #

struct A
{
    ~A() noexcept(false) try
    {
        // destructor body 
    }
    catch (...)
    {
        // exceptions of destructor body are caught here
        // if no exception is thrown here
        // then the caught exception is re-thrown.
    }
};

Bu mümkün olmasına rağmen, sanki yığın çözme sırasında çağrılan bir yıkıcı bir istisna atar, ‘std::terminate’ çağrılır gibi, yıkıcıdan fırlatırken çok dikkatli olunması gerektiğini unutmayın.
