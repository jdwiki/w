Başlık Dosyaları

  • Temel Örnek
  • Başlık Dosyalarındaki Şablonlar

Bu sayfada

  • Temel Örnek
  • Başlık Dosyalarındaki Şablonlar

C++‘da, C’de olduğu gibi, C++ derleyicisi ve derleme işlemi C önişlemcisini kullanır. GNU C Önişlemci kılavuzunda belirtildiği gibi, bir başlık dosyası aşağıdaki gibi tanımlanır:

Başlık dosyası, C bildirimlerini ve makroyu içeren bir dosyadır. birkaç kaynak dosya arasında paylaşılacak tanımlar (bkz. Makrolar). Bir başlık dosyasını dahil ederek programınızda kullanılmasını talep ettiğinizde, C ön işleme yönergesi ‘#include’ ile.

Başlık dosyaları iki amaca hizmet eder.

  • Sistem başlık dosyaları, arayüzleri işletim sisteminin bölümlerine bildirir. sağlamak için bunları programınıza dahil edersiniz. sistem çağrılarını çağırmak için ihtiyacınız olan tanımlar ve bildirimler ve kütüphaneler.
  • Kendi başlık dosyalarınız, programınızın kaynak dosyaları arasındaki arabirimler için bildirimler içerir. Her zaman bir grubunuz olduğunda tamamı veya çoğu ilgili bildirimler ve makro tanımları birkaç farklı kaynak dosyada gerekliyse, oluşturmak iyi bir fikirdir onlar için bir başlık dosyası.

Ancak, C önişlemcisinin kendisi için bir başlık dosyası, bir kaynak dosyadan farklı değildir.

Başlık/kaynak dosya organizasyon şeması, arayüz ve uygulama arasında ayrım sağlamak için çeşitli yazılım projeleri tarafından belirlenen, güçlü bir şekilde tutulan ve standart bir sözleşmedir.

C++ Standardının kendisi tarafından resmi olarak uygulanmasa da, başlık/kaynak dosya kuralına uyulması şiddetle tavsiye edilir ve pratikte zaten hemen hemen her yerde bulunur.

Başlık dosyalarının, modüllerin gelecek özelliği ile bir proje dosyası yapısı kuralı olarak değiştirilebileceğini unutmayın; bu özellik, yazma anından itibaren gelecekteki bir C++ Standardına (örn.

Temel Örnek #

Aşağıdaki örnek, “// dosyaadı” açıklamalarıyla belirtildiği gibi, birkaç kaynak dosyaya bölünmesi amaçlanan bir kod bloğu içerecektir.

Kaynak dosyaları #

// my_function.h

/* Note how this header contains only a declaration of a function.
 * Header functions usually do not define implementations for declarations
 * unless code must be further processed at compile time, as in templates.
 */

/* Also, usually header files include preprocessor guards so that every header
 * is never included twice.
 *
 * The guard is implemented by checking if a header-file unique preprocessor
 * token is defined, and only including the header if it hasn't been included
 * once before.
 */
#ifndef MY_FUNCTION_H
#define MY_FUNCTION_H

// global_value and my_function() will be
// recognized as the same constructs if this header is included by different files.
const int global_value = 42;
int my_function();

#endif // MY_FUNCTION_H

// my_function.cpp

/* Note how the corresponding source file for the header includes the interface  
 * defined in the header so that the compiler is aware of what the source file is 
 * implementing.
 *
 * In this case, the source file requires knowledge of the global constant
 * global_value only defined in my_function.h. Without inclusion of the header
 * file, this source file would not compile.
 */
#include "my_function.h" // or #include "my_function.hpp"
int my_function() {
  return global_value; // return 42;
}

Başlık dosyaları daha sonra başlık arabirimi tarafından tanımlanan işlevselliği kullanmak isteyen, ancak uygulanması hakkında bilgi gerektirmeyen (böylece kod birleştirmeyi azaltan) diğer kaynak dosyalar tarafından dahil edilir. Aşağıdaki program, yukarıda tanımlandığı gibi “my_function.h” başlığını kullanır:

// main.cpp

#include <iostream>       // A C++ Standard Library header.
#include "my_function.h"  // A personal header

int main(int argc, char** argv) {
  std::cout << my_function() << std::endl;
  return 0;
}

Derleme Süreci #

Başlık dosyaları genellikle bir derleme süreci iş akışının parçası olduğundan, başlık/kaynak dosya kuralından yararlanan tipik bir derleme süreci genellikle aşağıdakileri yapar.

Başlık dosyası ve kaynak kod dosyasının zaten aynı dizinde olduğunu varsayarsak, bir programcı aşağıdaki komutları yürütür:

g++ -c my_function.cpp       # Compiles the source file my_function.cpp
                             # --> object file my_function.o

g++ main.cpp my_function.o   # Links the object file containing the 
                             # implementation of int my_function()
                             # to the compiled, object version of main.cpp
                             # and then produces the final executable a.out

Alternatif olarak, eğer biri önce ‘main.cpp’ dosyasını bir nesne dosyasına derlemek isterse ve daha sonra son adım olarak yalnızca nesne dosyalarını birbirine bağlayın:

g++ -c my_function.cpp
g++ -c main.cpp

g++ main.o my_function.o

Başlık Dosyalarındaki Şablonlar #

Şablonlar, derleme zamanı kodu oluşturmayı gerektirir: örneğin, şablonlanmış bir işlev, kaynak kodda kullanılarak şablonlanmış bir işlev parametreleştirildiğinde etkin bir şekilde birden çok farklı işleve dönüştürülür.

Bu, şablon işlevi, üye işlevi ve sınıf tanımlarının ayrı bir kaynak kod dosyasına devredilemeyeceği anlamına gelir, çünkü herhangi bir şablonlu yapıyı kullanacak herhangi bir kod, genel olarak herhangi bir türev kodu oluşturmak için tanımının bilgisini gerektirir.

Bu nedenle, şablonlanmış kod, başlıklara yerleştirildiyse tanımını da içermelidir. Bunun bir örneği aşağıdadır:

// templated_function.h

template <typename T>
T* null_T_pointer() {
  T* type_point = NULL; // or, alternatively, nullptr instead of NULL
                        // for C++11 or later
  return type_point;
}
