Birden çok kitaplık kullanırken ad çakışmalarını önlemek için kullanılan ad alanı, işlevler, sınıflar, türler vb. için bildirimsel bir önektir.

Sözdizimi #

  • ad alanı tanımlayıcı(opt) { declaration-seq }
  • satır içi ad alanı tanımlayıcı(opt) { declaration-seq } /* C++11’den beri */
  • satır içi(opt) ad alanı attribute-specifier-seq identifier(opt) { declaration-seq } /* C++17’den beri */
  • ad alanı enclosing-namespace-specifier :: identifier { declaration-seq } /* C++17’den beri */
  • ad alanı tanımlayıcı = nitelikli ad-alanı belirticisi;
  • ad alanı iç içe-ad-belirleyici(opt) ad-alanı-adı kullanılması;
  • öznitelik-belirleyici-seq ad alanını kullanarak iç içe-ad-belirleyici(opt) ad-alanı-adı; /* C++11’den beri */

anahtar kelime “ad alanı” bağlama bağlı olarak üç farklı anlama sahiptir:

  1. Ardından isteğe bağlı bir ad ve küme ayracı içine alınmış bir bildirim dizisi geldiğinde, yeni bir ad alanı tanımlar veya [mevcut bir ad alanını genişletir] ad alanı](https://www.wikiod.com/tr/docs/c%2b%2b/495/namespaces/1623/extending-namespaces) bu bildirimlerle birlikte. Ad atlanırsa, ad alanı bir adsız ad alanıdır.

  2. Ardından bir ad ve eşittir işareti geldiğinde, bir ad alanı takma adı bildirir.

  3. Başında ‘using’ ve ardından bir ad alanı adı geldiğinde, bir using yönerge oluşturur. #t=2016080604584467511916), verilen ad alanındaki adların niteliksiz ad aramasıyla bulunmasına izin verir (ancak bu adları geçerli kapsamda yeniden bildirmez). Bir kullanma yönergesi, sınıf kapsamında gerçekleşemez.

kullanma ad alanı std; önerilmez. Neden? Niye? Çünkü ’namespace std’ çok büyük! Bu, isimlerin çakışma ihtimalinin yüksek olduğu anlamına gelir:

//Really bad!
using namespace std;

//Calculates p^e and outputs it to std::cout
void pow(double p, double e) { /*...*/ }

//Calls pow
pow(5.0, 2.0); //Error! There is already a pow function in namespace std with the same signature,
               //so the call is ambiguous

Ad alanları nedir? #

Bir C++ ad alanı, adlarının önüne ad alanının adının eklendiği bir C++ varlıkları (işlevler, sınıflar, değişkenler) topluluğudur. Bir ad alanı içinde kod yazarken, o ad alanına ait adlandırılmış varlıkların ad alanı adının önüne eklenmesi gerekmez, ancak bunun dışındaki varlıkların tam nitelikli adı kullanması gerekir. Tam nitelikli ad, “<ad alanı>::<varlık>” biçimindedir. Örnek:

namespace Example
{
  const int test = 5;

  const int test2 = test + 12; //Works within `Example` namespace
}

const int test3 = test + 3; //Fails; `test` not found outside of namespace.

const int test3 = Example::test + 3; //Works; fully qualified name used.

Ad alanları, ilgili tanımları birlikte gruplandırmak için kullanışlıdır. Bir alışveriş merkezi benzetmesini yapın. Genellikle bir alışveriş merkezi, her biri belirli bir kategoriden ürünler satan birkaç mağazaya bölünür. Bir mağaza elektronik, başka bir mağaza ayakkabı satabilir. Mağaza türlerindeki bu mantıksal ayrımlar, alışveriş yapanların aradıkları ürünleri bulmalarına yardımcı olur. Ad alanları, alışveriş yapanlar gibi c++ programcılarının aradıkları işlevleri, sınıfları ve değişkenleri mantıklı bir şekilde düzenleyerek bulmalarına yardımcı olur. Örnek:

namespace Electronics
{
    int TotalStock;
    class Headphones
    {
        // Description of a Headphone (color, brand, model number, etc.)
    };
    class Television
    {
        // Description of a Television (color, brand, model number, etc.)
    };
}

namespace Shoes
{
    int TotalStock;
    class Sandal
    {
        // Description of a Sandal (color, brand, model number, etc.)
    };
    class Slipper
    {
        // Description of a Slipper (color, brand, model number, etc.)
    };
}

Önceden tanımlanmış tek bir ad alanı vardır; bu, adı olmayan, ancak :: ile gösterilebilen genel ad alanıdır. Örnek:

void bar() {
    // defined in global namespace
}
namespace foo {
    void bar() {
        // defined in namespace foo
    }
    void barbar() {
        bar();   // calls foo::bar()
        ::bar(); // calls bar() defined in global namespace
    }
}

Bağımsız Değişkene Bağlı Arama #

Açık bir ad alanı niteleyicisi olmayan bir işlevi çağırırken, derleyici, o işleve ilişkin parametre türlerinden biri de o ad alanındaysa, bir ad alanı içindeki bir işlevi çağırmayı seçebilir. Buna “Argüman Bağımlı Arama” veya ADL denir:

namespace Test
{
  int call(int i);

  class SomeClass {...};

  int call_too(const SomeClass &data);
}

call(5); //Fails. Not a qualified function name.

Test::SomeClass data;

call_too(data); //Succeeds

“call”, parametre türlerinin hiçbiri “Test” ad alanından gelmediği için başarısız oluyor. ‘call_too’ çalışır çünkü ‘SomeClass’, ‘Test’in bir üyesidir ve bu nedenle ADL kurallarına uygundur.

ADL ne zaman oluşmaz #

Normal niteliksiz arama bir sınıf üyesi, blok kapsamında bildirilmiş bir işlev veya işlev türünde olmayan bir şey bulursa ADL oluşmaz. Örneğin:

void foo();
namespace N {
    struct X {};
    void foo(X ) { std::cout << '1'; }
    void qux(X ) { std::cout << '2'; }
}

struct C {
    void foo() {}
    void bar() {
        foo(N::X{}); // error: ADL is disabled and C::foo() takes no arguments
    }
};

void bar() {
    extern void foo(); // redeclares ::foo
    foo(N::X{});       // error: ADL is disabled and ::foo() doesn't take any arguments
}

int qux;

void baz() {
    qux(N::X{}); // error: variable declaration disables ADL for "qux"
}

Ad alanlarını genişletme #

Ad alanlarının kullanışlı bir özelliği, onları genişletebilmenizdir (buna üyeler ekleyin).

namespace Foo
{
    void bar() {}
}

//some other stuff

namespace Foo
{
    void bar2() {}
}

Yönergeyi kullanma #

‘using’ anahtar kelimesinin üç çeşidi vardır. Anahtar kelime ‘ad alanı’ ile birlikte bir ‘kullanım yönergesi’ yazarsınız:

‘Foo’ ad alanındaki her şeyin önüne ‘Foo::’ yazmak istemiyorsanız, ‘Foo’dan her şeyi almak için ‘using namespace Foo;’ kullanabilirsiniz.

namespace Foo
{
    void bar() {}
    void baz() {}
}

//Have to use Foo::bar()
Foo::bar();

//Import Foo
using namespace Foo;
bar(); //OK
baz(); //OK

Tüm ad alanı yerine bir ad alanındaki seçili varlıkları içe aktarmak da mümkündür:

using Foo::bar;
bar(); //OK, was specifically imported
baz(); // Not OK, was not imported

Dikkatli olun: `başlık dosyalarında ad alanını kullanmak çoğu durumda kötü bir stil olarak görülür. Bu yapılırsa, ad alanı, başlığı içeren her dosyaya aktarılır. Bir ad alanını “kullanmamanın” bir yolu olmadığından, bu ad alanı kirliliğine (küresel ad alanında daha fazla veya beklenmedik semboller) veya daha da kötüsü çatışmalara yol açabilir. Sorunun bir örneği için bu örneğe bakın:

/***** foo.h *****/
namespace Foo
{
    class C;
}

/***** bar.h *****/
namespace Bar
{
    class C;
}

/***** baz.h *****/
#include "foo.h"
using namespace Foo;

/***** main.cpp *****/
#include "bar.h"
#include "baz.h"

using namespace Bar;
C c; // error: Ambiguity between Bar::C and Foo::C

Bir kullanma yönergesi, sınıf kapsamında gerçekleşemez.

Ad alanları oluşturma #

Bir ad alanı oluşturmak gerçekten kolaydır:

//Creates namespace foo
namespace Foo
{
    //Declares function bar in namespace foo
    void bar() {}
}

‘bar’ı çağırmak için, önce ad alanını, ardından kapsam çözümleme operatörünü ‘::’ belirtmeniz gerekir:

Foo::bar();

Bir diğerinde bir ad alanı oluşturmaya izin verilir, örneğin:

namespace A
{
    namespace B
    {
        namespace C
        {
            void bar() {}
        }
    }
}

Yukarıdaki kod aşağıdaki şekilde basitleştirilebilir:

namespace A::B::C
{
    void bar() {}
}
son sürüm

Adsız/anonim ad alanları #

Adların dahili bağlantıya sahip olmasını sağlamak için adsız bir ad alanı kullanılabilir (yalnızca geçerli çeviri birimi tarafından başvurulabilir). Böyle bir ad alanı, diğer ad alanları ile aynı şekilde tanımlanır, ancak ad olmadan:

namespace {
    int foo = 42;
}

“foo” yalnızca göründüğü çeviri biriminde görünür.

Başlık dosyalarında asla adsız ad alanlarının kullanılmaması önerilir, çünkü bu, içerdiği her çeviri birimi için içeriğin bir sürümünü verir. Bu, özellikle const olmayan globaller tanımlarsanız önemlidir.

// foo.h
namespace {
    std::string globalString;
}

// 1.cpp
#include "foo.h" //< Generates unnamed_namespace{1.cpp}::globalString ...

globalString = "Initialize";

// 2.cpp
#include "foo.h" //< Generates unnamed_namespace{2.cpp}::globalString ...

std::cout << globalString; //< Will always print the empty string

Kompakt iç içe ad alanları #

namespace a {
  namespace b {
    template<class T>
    struct qualifies : std::false_type {};
  }
}

namespace other {
  struct bob {};
}

namespace a::b {
  template<>
  struct qualifies<::other::bob> : std::true_type {};
}

Hem “a” hem de “b” ad alanlarını, C++17’den başlayarak “ad alanı a::b” ile tek adımda girebilirsiniz.

son sürüm

Satır içi ad alanı #

“satır içi ad alanı”, çevreleyen ad alanındaki satır içi ad alanının içeriğini içerir, bu nedenle

namespace Outer
{
    inline namespace Inner
    {
        void foo();
    }
}

çoğunlukla eşdeğerdir

namespace Outer
{

    namespace Inner
    {
        void foo();
    }

    using Inner::foo;
}

ancak ‘Outer::Inner::’ öğesinden ve ‘Outer::’ ile ilişkilendirilenler aynıdır.

Yani aşağıdaki eşdeğerdir

Outer::foo();
Outer::Inner::foo();

using namespace Inner; alternatifi, şablon uzmanlığı olarak bazı zor kısımlar için eşdeğer olmayacaktır:

İçin

#include <outer.h> // See below

class MyCustomType;
namespace Outer
{
    template <>
    void foo<MyCustomType>() { std::cout << "Specialization"; }
}

  • Satır içi ad alanı, ‘Outer::foo’nun özelleştirilmesine izin verir

    // outer.h
// include guard omitted for simplification

namespace Outer
{
    inline namespace Inner
    {
        template <typename T>
        void foo() { std::cout << "Generic"; }
    }
}

  • “ad alanını kullanmak”, “Outer::foo"nun uzmanlaşmasına izin vermezken

    // outer.h
// include guard omitted for simplification

namespace Outer
{
    namespace Inner
    {
        template <typename T>
        void foo() { std::cout << "Generic"; }
    }
    using namespace Inner;
    // Specialization of `Outer::foo` is not possible
    // it should be `Outer::Inner::foo`.
}

Satır içi ad alanı, birkaç sürümün birlikte yaşamasına ve varsayılan olarak “satır içi” olana ayarlanmasına izin vermenin bir yoludur

namespace MyNamespace
{
    // Inline the last version
    inline namespace Version2
    {
        void foo(); // New version
        void bar();
    }

    namespace Version1 // The old one
    {
        void foo();
    }

}

Ve kullanım ile

MyNamespace::Version1::foo(); // old version
MyNamespace::Version2::foo(); // new version
MyNamespace::foo();           // default version : MyNamespace::Version1::foo();
son sürüm

Ad alanı takma adı #

Bir ad alanına bir takma ad (yani, aynı ad alanı için başka bir ad) ‘ad alanı’ tanımlayıcı ‘=’ sözdizimi kullanılarak verilebilir. Takma ad alanının üyelerine, takma ad adıyla nitelenerek erişilebilir. Aşağıdaki örnekte, iç içe “AReallyLongName::AnotherReallyLongName” ad alanının yazılması uygun değildir, bu nedenle “qux” işlevi yerel olarak bir “N” diğer adını bildirir. Bu ad alanının üyelerine daha sonra basitçe N:: kullanılarak erişilebilir.

namespace AReallyLongName {
    namespace AnotherReallyLongName {
        int foo();
        int bar();
        void baz(int x, int y);
    }
}
void qux() {
    namespace N = AReallyLongName::AnotherReallyLongName;
    N::baz(N::foo(), N::bar());
}
