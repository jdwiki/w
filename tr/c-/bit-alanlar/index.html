Bit alanları

  • Beyan ve Kullanım

Bu sayfada

  • Beyan ve Kullanım

Bit alanları, boyutu küçültmek için C ve C++ yapılarını sıkıca paketler. Bu ağrısız görünür: üyeler için bit sayısını belirtin ve derleyici bitleri birleştirme işini yapar. Kısıtlama, karma olarak depolandığından, bir bit alanı üyesinin adresini alamamasıdır. “sizeof()” işlevine de izin verilmez.

Üye değerlerini çıkarmak veya değiştirmek için belleğin alınması ve bit düzeyinde işlemlerin uygulanması gerektiğinden, bit alanlarının maliyeti daha yavaş erişimdir. Bu işlemler ayrıca yürütülebilir boyuta da eklenir.

Bitsel işlemler ne kadar pahalı? Basit bir bit olmayan alan yapısı varsayalım:

struct foo {
    unsigned x;
    unsigned y;
}
static struct foo my_var;

Daha sonraki bazı kodlarda:

my_var.y = 5;

sizeof (unsigned) == 4 ise, x yapının başlangıcında saklanır ve y 4 bayt içinde saklanır. Oluşturulan derleme kodu şuna benzer:

loada register1,#myvar     ; get the address of the structure
storei register1[4],#0x05  ; put the value '5' at offset 4, e.g., set y=5

Bu basittir çünkü x, y ile karışmaz. Ancak yapıyı bit alanlarıyla yeniden tanımladığınızı hayal edin:

struct foo {
    unsigned x : 4; /* Range 0-0x0f, or 0 through 15 */
    unsigned y : 4;
}

Hem “x” hem de “y”, tek bir baytı paylaşan 4 bit tahsis edilecektir. Böylece yapı 8 yerine 1 bayt kaplar. Derlemenin üst nibble’da biteceğini varsayarak şimdi ‘y’yi ayarlayacağını düşünün:

loada  register1,#myvar        ; get the address of the structure
loadb  register2,register1[0]  ; get the byte from memory
andb   register2,#0x0f         ; zero out y in the byte, leaving x alone
orb    register2,#0x50         ; put the 5 into the 'y' portion of the byte
stb    register1[0],register2  ; put the modified byte back into memory

Bu yapılardan binlerce veya milyonlarca varsa, bu iyi bir takas olabilir ve belleğin önbellekte kalmasına yardımcı olur veya takası önler - veya bu sorunları kötüleştirmek ve işlemeyi yavaşlatmak için yürütülebilir dosyayı şişirebilir. Her şeyde olduğu gibi, iyi muhakeme kullanın.

Aygıt sürücüsü kullanımı: Aygıt sürücüleri için akıllı bir uygulama stratejisi olarak bit alanlarından kaçının. Bit alanı depolama düzenleri, derleyiciler arasında mutlaka tutarlı değildir ve bu tür uygulamaları taşınabilir yapmaz. Değerleri ayarlamak için oku-değiştir-yaz, cihazların beklediğini yapmayabilir ve beklenmeyen davranışlara neden olabilir.

Beyan ve Kullanım #

struct FileAttributes
{
    unsigned int ReadOnly: 1;    
    unsigned int Hidden: 1;
};

Burada bu iki alanın her biri bellekte 1 bit yer kaplayacaktır. Değişken isimlerinden sonra : 1 ifadesi ile belirtilir. Temel bit alanı türü, herhangi bir integral türü (8 bit int ila 64 bit int) olabilir. ‘Unsigned’ tipinin kullanılması tavsiye edilir, aksi takdirde sürprizler gelebilir.

Daha fazla bit gerekiyorsa, “1” yerine gereken bit sayısı koyun. Örneğin:

struct Date
{
    unsigned int Year : 13; // 2^13 = 8192, enough for "year" representation for long time
    unsigned int Month: 4;  // 2^4 = 16, enough to represent 1-12 month values.
    unsigned int Day:   5;  // 32
};

Tüm yapı sadece 22 bit kullanıyor ve normal derleyici ayarlarıyla bu yapının ‘sizeof’u 4 bayt olacaktır.

Kullanımı oldukça basittir. Sadece değişkeni bildirin ve sıradan yapı gibi kullanın.

Date d;

d.Year = 2016;
d.Month = 7;
d.Day =  22;

std::cout << "Year:" << d.Year << std::endl <<
        "Month:" << d.Month << std::endl <<
        "Day:" << d.Day << std::endl;
