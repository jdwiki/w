this işaretçisi C++ için bir anahtar kelimedir, bu nedenle bunu uygulamak için kitaplığa gerek yoktur. Ve bunun bir işaretçi olduğunu unutmayın! Yani yapamazsınız:

 this.someMember();

Üye işlevlerine veya üye değişkenlerine işaretçilerden -> ok sembolünü kullanarak erişirken:

this->someMember();

“Bu” işaretçisinin daha iyi anlaşılması için diğer yararlı bağlantılar:

bu İşaretçi #

Tüm statik olmayan üye işlevlerin gizli bir parametresi vardır; bu, sınıfın bir örneğine işaret eden, ’this’ olarak adlandırılır; bu parametre, parametre listesinin başına sessizce eklenir ve tamamen derleyici tarafından işlenir. Bir üye fonksiyon içinde sınıfın bir üyesine erişildiğinde, ona sessizce ’this’ aracılığıyla erişilir; bu, derleyicinin tüm örnekler için statik olmayan tek bir üye işlevi kullanmasına ve bir üye işlevinin diğer üye işlevlerini polimorfik olarak çağırmasına izin verir.

struct ThisPointer {
    int i;

    ThisPointer(int ii);

    virtual void func();

    int  get_i() const;
    void set_i(int ii);
};
ThisPointer::ThisPointer(int ii) : i(ii) {}
// Compiler rewrites as:
ThisPointer::ThisPointer(int ii) : this->i(ii) {}
// Constructor is responsible for turning allocated memory into 'this'.
// As the constructor is responsible for creating the object, 'this' will not be "fully"
// valid until the instance is fully constructed.

/* virtual */ void ThisPointer::func() {
    if (some_external_condition) {
        set_i(182);
    } else {
        i = 218;
    }
}
// Compiler rewrites as:
/* virtual */ void ThisPointer::func(ThisPointer* this) {
    if (some_external_condition) {
        this->set_i(182);
    } else {
        this->i = 218;
    }
}

int  ThisPointer::get_i() const { return i; }
// Compiler rewrites as:
int  ThisPointer::get_i(const ThisPointer* this) { return this->i; }

void ThisPointer::set_i(int ii) { i = ii; }
// Compiler rewrites as:
void ThisPointer::set_i(ThisPointer* this, int ii) { this->i = ii; }

Bir kurucuda, ’this’, halihazırda başlatılmış herhangi bir alana veya bir üst sınıftaki herhangi bir alana (örtülü veya açık olarak) erişmek için güvenle kullanılabilir; tersine, (örtük veya açık olarak) henüz başlatılmamış herhangi bir alana veya türetilmiş bir sınıftaki herhangi bir alana erişim güvenli değildir (türetilmiş sınıf henüz oluşturulmadığından ve dolayısıyla alanları başlatılmamış veya mevcut olmadığından). Ayrıca, herhangi bir türetilmiş sınıf işlevi dikkate alınmayacağından (türetilmiş sınıfın henüz oluşturulmaması ve dolayısıyla yapıcısının henüz vtable’ı güncellememesi nedeniyle), yapıcıda ’this’ aracılığıyla sanal üye işlevlerini çağırmak güvenli değildir.

Ayrıca, bir kurucudayken, nesnenin tipinin, o kurucunun inşa ettiği tip olduğuna dikkat edin. Bu, nesne türetilmiş bir tür olarak bildirilse bile geçerlidir. Örneğin, aşağıdaki örnekte, “ctd_good” ve “ctd_bad”, “CtorThisBase()” içinde “CtorThisBase” türündedir ve kurallı türleri “CtorThisDerived” olsa bile, “CtorThis()” içinde “CtorThis” yazın. Daha fazla türetilmiş sınıflar temel sınıf etrafında oluşturulduğundan, örnek, amaçlanan türünün tam olarak oluşturulmuş bir örneği olana kadar sınıf hiyerarşisinden kademeli olarak geçer.

class CtorThisBase {
    short s;

  public:
    CtorThisBase() : s(516) {}
};

class CtorThis : public CtorThisBase {
    int i, j, k;

  public:
    // Good constructor.
    CtorThis() : i(s + 42), j(this->i), k(j) {}

    // Bad constructor.
    CtorThis(int ii) : i(ii), j(this->k), k(b ? 51 : -51) {
        virt_func();
    }

    virtual void virt_func() { i += 2; }
};

class CtorThisDerived : public CtorThis {
    bool b;

  public:
    CtorThisDerived()       : b(true) {}
    CtorThisDerived(int ii) : CtorThis(ii), b(false) {}

    void virt_func() override { k += (2 * i); }
};

// ...

CtorThisDerived ctd_good;
CtorThisDerived ctd_bad(3);

Bu sınıflar ve üye işlevleriyle:

  • İyi kurucuda, “ctd_good” için:
  • ‘CtorThisBase’, ‘CtorThis’ yapıcısı girildiğinde tamamen oluşturulmuştur. Bu nedenle, ‘i’ başlatılırken ’s’ geçerli bir durumdadır ve bu nedenle erişilebilir.
  • “i”, “j(this->i)“ye ulaşılmadan önce başlatılır. Bu nedenle, ‘i’, ‘j’ başlatılırken geçerli bir durumdadır ve bu nedenle erişilebilir.
  • “j”, “k(j)“ye ulaşılmadan önce başlatılır. Bu nedenle, “k” başlatılırken “j” geçerli bir durumdadır ve bu nedenle erişilebilir.
  • Kötü kurucuda, “ctd_bad” için:
  • k, j(this->k) ulaşıldıktan sonra başlatılır. Bu nedenle, ‘j’ başlatılırken ‘k’ geçersiz bir durumda ve buna erişim tanımsız davranışa neden oluyor.
  • ‘CtorThisDerived’, ‘CtorThis’ oluşturuluncaya kadar oluşturulmaz. Bu nedenle, “k” başlatılırken “b” geçersiz bir durumda ve buna erişim tanımsız davranışa neden oluyor.
  • ‘ctd_bad’ nesnesi, ‘CtorThis()‘den ayrılana kadar hala bir ‘CtorThis’dir ve ‘CtorThisDerived()’e kadar ‘CtorThisDerived’in sanal tablosunu kullanmak üzere güncellenmeyecektir. Bu nedenle, ‘virt_func()’, ‘CtorThis::virt_func()’ veya ‘CtorThisDerived::virt_func()’ olarak adlandırılmasından bağımsız olarak ‘CtorThis::virt_func()’ öğesini çağıracaktır.

Üye Verilerine Erişmek için Bu İşaretçiyi Kullanma #

Bu bağlamda, ’this’ işaretçisini kullanmak tamamen gerekli değildir, ancak belirli bir işlevin veya değişkenin sınıfın bir üyesi olduğunu belirterek kodunuzu okuyucu için daha net hale getirecektir. Bu durumda bir örnek:

// Example for this pointer
#include <iostream>
#include <string>

using std::cout;
using std::endl;

class Class
{
  public:
    Class();
    ~Class();
    int getPrivateNumber () const;
  private:
    int private_number = 42;
};

Class::Class(){}
Class::~Class(){}

int Class::getPrivateNumber() const
{
    return this->private_number;
}

int main()
{
    Class class_example;
    cout << class_example.getPrivateNumber() << endl;
}

burada çalışırken görün.

Üye Verileri ve Parametreler Arasında Ayırım Yapmak İçin Bu İşaretçiyi Kullanma #

Bu, üye verilerini parametrelerden ayırt etmek için gerçek bir yararlı stratejidir… Şu örneği ele alalım:

// Dog Class Example
#include <iostream>
#include <string>

using std::cout;
using std::endl;

/*
* @class Dog
*   @member name
*       Dog's name
*   @function bark
*       Dog Barks!
*   @function getName
*       To Get Private
*       Name Variable
*/
class Dog
{
 public:
    Dog(std::string name);
    ~Dog();
    void  bark() const;
    std::string  getName() const;
 private:
    std::string name;
};

Dog::Dog(std::string name)
{
    /*
    *  this->name is the
    *  name variable from 
    *  the class dog . and
    *  name is from the 
    *  parameter of the function
    */
    this->name = name; 
}

Dog::~Dog(){}

void Dog::bark() const
{
  cout << "BARK" << endl;   
}

std::string  Dog::getName() const
{
    return this->name;
}


int main()
{
    Dog dog("Max");
    cout << dog.getName() << endl;
    dog.bark();
}

Aşağıdakileri uyguladığımız yapıcıda burada görebilirsiniz:

this->name = name;

Burada, Dog(this->name) sınıfındaki özel değişkenin adına parametre adını atadığımızı görebilirsiniz.

bu İşaretçi CV Niteleyicileri #

“bu”, diğer işaretçilerle aynı şekilde, özgeçmiş nitelikli olabilir. Ancak this parametresinin parametre listesinde yer almaması nedeniyle bunun için özel bir sözdizimi gereklidir; cv niteleyicileri parametre listesinden sonra, ancak işlevin gövdesinden önce listelenir.

struct ThisCVQ {
    void no_qualifier()                {} // "this" is: ThisCVQ*
    void  c_qualifier() const          {} // "this" is: const ThisCVQ*
    void  v_qualifier() volatile       {} // "this" is: volatile ThisCVQ*
    void cv_qualifier() const volatile {} // "this" is: const volatile ThisCVQ*
};

’this’ bir parametre olduğundan, bir işlev, ‘bu’ cv niteleyicilerine göre aşırı yüklenebilir.

struct CVOverload {
    int func()                { return    3; }
    int func() const          { return   33; }
    int func() volatile       { return  333; }
    int func() const volatile { return 3333; }
};

’this’, ‘const’ olduğunda (“const volatile” dahil), işlev, dolaylı veya açık olarak üye değişkenlere bunun aracılığıyla yazamaz. Bunun tek istisnası, sabitliğe bakılmaksızın yazılabilen ‘değişebilir’ üye değişkenleri‘dir. Bu nedenle, “const”, üye işlevin nesnenin mantıksal durumunu (nesnenin dış dünyaya görünme biçimi), fiziksel durumu değiştirse bile (nesnenin altından görünme biçimini) değiştirmediğini belirtmek için kullanılır. başlık).

Mantıksal durum, nesnenin dış gözlemcilere görünme şeklidir. Doğrudan fiziksel duruma bağlı değildir ve aslında fiziksel durum olarak depolanmayabilir bile. Dış gözlemciler herhangi bir değişiklik göremediği sürece, nesnedeki her bir biti çevirseniz bile mantıksal durum sabittir.

Bitsel durum olarak da bilinen fiziksel durum, nesnenin bellekte nasıl depolandığıdır. Bu, nesnenin özüdür, verilerini oluşturan ham 1’ler ve 0’lardır. Bir nesne, yalnızca bellekteki temsili asla değişmezse fiziksel olarak sabittir.

C++‘ın “sabitliği” fiziksel duruma değil, mantıksal duruma dayandırdığını unutmayın.

class DoSomethingComplexAndOrExpensive {
    mutable ResultType cached_result;
    mutable bool state_changed;

    ResultType calculate_result();
    void modify_somehow(const Param& p);

    // ...

  public:
    DoSomethingComplexAndOrExpensive(Param p) : state_changed(true) {
        modify_somehow(p);
    }

    void change_state(Param p) {
        modify_somehow(p);
        state_changed = true;
    }

    // Return some complex and/or expensive-to-calculate result.
    // As this has no reason to modify logical state, it is marked as "const".
    ResultType get_result() const;
};
ResultType DoSomethingComplexAndOrExpensive::get_result() const {
    // cached_result and state_changed can be modified, even with a const "this" pointer.
    // Even though the function doesn't modify logical state, it does modify physical state
    //  by caching the result, so it doesn't need to be recalculated every time the function
    //  is called.  This is indicated by cached_result and state_changed being mutable.

    if (state_changed) {
        cached_result = calculate_result();
        state_changed = false;
    }

    return cached_result;
}

Teknik olarak ‘bu’ üzerinde ‘const_cast’i özgeçmiş nitelikli olmayan yapmak için _kullanabiliyor olsanız da, gerçekten GERÇEKTEN kullanmamalısınız ve bunun yerine ‘mutable’ kullanmalısınız. Bir “const_cast”, aslında “const” olan bir nesne üzerinde kullanıldığında tanımsız davranışı başlatmakla yükümlüdür, “mutable” ise kullanımı güvenli olacak şekilde tasarlanmıştır. Bununla birlikte, buna son derece eski kodda rastlamanız mümkündür.

Bu kuralın bir istisnası, özgeçmiş nitelikli olmayan erişimcileri ‘const’ erişimcileri olarak tanımlamaktır; cv nitelikli olmayan sürüm çağrılırsa nesnenin ‘const’ olmayacağı garanti edildiğinden, UB riski yoktur.

class CVAccessor {
    int arr[5];

  public:
    const int& get_arr_element(size_t i) const { return arr[i]; }

    int& get_arr_element(size_t i) {
        return const_cast<int&>(const_cast<const CVAccessor*>(this)->get_arr_element(i));
    }
};

Bu, gereksiz kod tekrarını önler.

Normal işaretçilerde olduğu gibi, “bu” “geçici” ise (“const volatile” dahil), önbelleğe alınmak yerine her erişildiğinde bellekten yüklenir. Bu, optimizasyon üzerinde diğer herhangi bir işaretçiyi “uçucu” olarak bildirmekle aynı etkilere sahiptir, bu nedenle dikkatli olunmalıdır.

Bir örnek cv nitelikliyse, erişmesine izin verilen tek üye işlevlerin, “bu” işaretçisi en az örneğin kendisi kadar cv nitelikli olan üye işlevler olduğuna dikkat edin:

  • Özgeçmiş dışı örnekler herhangi bir üye işlevine erişebilir.
  • “const” örnekleri “const” ve “const volatile” işlevlerine erişebilir.
  • “uçucu” örnekler, “uçucu” ve “sabit uçucu” işlevlerine erişebilir.
  • ‘const volatile’ örnekleri, ‘const volatile’ işlevlerine erişebilir.

Bu, const doğruluğunun temel ilkelerinden biridir.

struct CVAccess {
    void    func()                {}
    void  func_c() const          {}
    void  func_v() volatile       {}
    void func_cv() const volatile {}
};

CVAccess cva;
cva.func();    // Good.
cva.func_c();  // Good.
cva.func_v();  // Good.
cva.func_cv(); // Good.

const CVAccess c_cva;
c_cva.func();    // Error.
c_cva.func_c();  // Good.
c_cva.func_v();  // Error.
c_cva.func_cv(); // Good.

volatile CVAccess v_cva;
v_cva.func();    // Error.
v_cva.func_c();  // Error.
v_cva.func_v();  // Good.
v_cva.func_cv(); // Good.

const volatile CVAccess cv_cva;
cv_cva.func();    // Error.
cv_cva.func_c();  // Error.
cv_cva.func_v();  // Error.
cv_cva.func_cv(); // Good.

bu İşaretçi Ref Niteleyicileri #

’this’ cv niteleyicilerine benzer şekilde, ref-qualifiers‘ı ‘*this’e de uygulayabiliriz. Ref niteleyiciler, normal ve değer referans semantiği arasında seçim yapmak için kullanılır, bu da derleyicinin hangisinin daha uygun olduğuna bağlı olarak kopyalama veya taşıma semantiğini kullanmasına izin verir ve ‘bu’ yerine ‘*bu’ya uygulanır.

Referans sözdizimini kullanan ref niteleyicilerine rağmen, “bu"nun kendisinin hala bir işaretçi olduğunu unutmayın. Ayrıca, başvuru niteleyicilerinin `*bu`` türünü gerçekten değiştirmediğini unutmayın; Etkilerini sanki öyleymiş gibi bakarak tanımlamak ve anlamak daha kolay.

struct RefQualifiers {
    std::string s;

    RefQualifiers(const std::string& ss = "The nameless one.") : s(ss) {}

    // Normal version.
    void func() &  { std::cout << "Accessed on normal instance "    << s << std::endl; }
    // Rvalue version.
    void func() && { std::cout << "Accessed on temporary instance " << s << std::endl; }

    const std::string& still_a_pointer() &  { return this->s; }
    const std::string& still_a_pointer() && { this->s = "Bob"; return this->s; }
};

// ...

RefQualifiers rf("Fred");
rf.func();              // Output:  Accessed on normal instance Fred
RefQualifiers{}.func(); // Output:  Accessed on temporary instance The nameless one

Bir üye işlevi, başvuru niteleyicileri olan ve olmayan aşırı yüklemelere sahip olamaz; programcı biri veya diğeri arasında seçim yapmak zorundadır. Neyse ki, özgeçmiş niteleyicileri ref niteleyicileri ile birlikte kullanılabilir ve const doğruluğu kurallarına uyulmasına izin verir.

struct RefCV {
    void func() &                {}
    void func() &&               {}
    void func() const&           {}
    void func() const&&          {}
    void func() volatile&        {}
    void func() volatile&&       {}
    void func() const volatile&  {}
    void func() const volatile&& {}
};
