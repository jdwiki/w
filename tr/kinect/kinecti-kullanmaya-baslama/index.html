Kurulum veya Kurulum #

Kinect kurulumu veya kurulumu hakkında ayrıntılı talimatlar.Kinect Sensörü Kurma

Windows 1.5, 1.6, 1.7, 1.8 için Kinect

Kinect sensörünüzü çalışır duruma getirmek için bazı basit adımlar aşağıda verilmiştir.

Adım 1: Sensörü Sabit Bir Yüzeye Monte Edin

Sensörü, kullanım sırasında düşmeyeceği veya çarpmayacağı bir yerde sabit bir yüzeye yerleştirin. İşte başka ipuçları.

Do not place the Kinect on or in front of a speaker or on a surface that vibrates or makes noise.
Keep the Kinect out of direct sunlight.
Use the Kinect within its specified operating temperature range of 41 to 95 degrees Fahrenheit (5 to 35 degrees Celsius). If the sensor is exposed to an environment outside of its prescribed temperature range, turn it off and allow the temperature to stabilize within the specified range before you use the sensor again.
Do not tilt a Kinect on its base. Manually tilting the Kinect can damage the sensor. The Kinect tilt angle is controlled by software; see Kinect Explorer - WPF C# Sample for an example.

Adım 2: Windows SDK için Kinect’i yükleyin

If you haven't done so already, download and install the free SDK from the Kinect for Windows download page.
If you haven't done so already, download and install the free Developer Toolkit from the Kinect for Windows download page.

Adım 3: Kinect sensörünüzü takın

Connect the power supply for your Kinect to an external power source.
Connect the Kinect to a USB port on your PC and wait for Windows to recognize it.
All the drivers, including audio, will load seamlessly.

Kinect for Windows SDK’yı kurduğunuzda, kurulum paketi Kinect sürücülerini içerir. Kinect sürücüleri Windows tabanlı bir bilgisayara yüklendiğinde, bilgisayarın USB bağlantı noktasına takılı bir Kinect, çok bileşenli bir USB aygıtı olarak görünür.

USB ana bilgisayar denetleyicisi uyumluluğuyla ilgili bilinen bir sorun var.

Sürücüleri yükledikten sonra geliştirme ortamınızı kurmanız gerekir. Visual Studio kullanıyorsanız, ilk C# veya C++ uygulamanızı oluşturmanıza yardımcı olması için Geliştirme Ortamını Yapılandırma ile devam edin.

Sorun giderme ipuçları

Aşağıdaki ipuçları, Kinect’inizi kullanmaya başlamanıza yardımcı olacaktır:

If a non-Microsoft driver for the Kinect is installed on your computer, the Kinect for Windows drivers might not install or function correctly. To fix this, uninstall the non-Microsoft drivers before installing the Kinect for Windows SDK.
Connect the power supply for the Kinect to an external power source; if the Kinect has only power from the USB connection, it will be minimally functional and light the LED, but it must be connected to an external power source to be fully functional.
No tools are required for calibration of audio and video.
Your Kinect should be the only device plugged into a USB hub on your computer. If you have more than one Kinect, connect them to different USB controllers. If 2 hubs are connected to the same controller, only 1 Kinect can work at a time.
The Kinect is protected from overheating by a fan. It is controlled by the sensor's firmware, which turns off the camera at 90 degrees Celsius. There is no software interface for applications to control the fan.
Reasonable lighting, neither extremely dark nor extremely bright, is important for capturing images with the RGB camera. Incandescent, fluorescent, and natural lighting provide no special obstacles, but do not point an intense or constant light source at the camera because this can blind the RGB sensor.
The depth sensor functions adequately in typical and reduced lighting, although in near darkness there is increased noise in the signal.
The depth sensor reads depth information from reflected light. Objects that are highly reflective (mirrors and shiny metal) or highly absorptive (fluffy and/or dark materials) may not be registered by the depth sensor as successfully as other objects.

Ücretsiz/Debian Resmi paketler

Ubuntu 11.10 (Oneiric) ve Debian 7’den (Wheezy) başlayarak, Ubuntu ve Debian resmi libfreenect paketleri sağlar. Bunları bir konsola kolayca yükleyebilirsiniz:

$ sudo apt-get install freenect

Ubuntu 12.04’te gspca çekirdek sürücüsü, libfreenect‘in kullanıcı modunda Kinect cihazını talep etmesini engeller. Modülü kaldırın ve kara listeye alın

$ sudo modprobe -r gspca_kinect 
$ sudo modprobe -r gspca_main
$ echo "blacklist gspca_kinect" |sudo tee -a /etc/modprobe.d/blacklist.conf

veya Ubuntu 12.04 için güncellenmiş libfreenect paketleri ile Florian Echtler libtisch PPA’yı (aşağıya bakın) ekleyerek çekirdek sürücüsünü otomatik olarak ayırabilen libfreenect‘in en son sürümlerini yükleyin.

freenect cihazına ‘plugdev’ grubuna ait herhangi bir kullanıcı erişebilir. Varsayılan olarak, bir masaüstü kullanıcısı plugdev grubuna aittir, ancak onları gruba eklemeniz gerekirse:

$ sudo adduser $USER plugdev

sonra çıkış yapın ve tekrar giriş yapın NeuroDebian deposu

Hangi Debian veya Ubuntu sürümünü kullanırsanız kullanın, libfreenect’in yeni bir sürümünü istiyorsanız, desteklenen tüm Debian ve Ubuntu sürümleri (yani Ubuntu Lucid (10.04), Maverick (10.10), Natty (11.04) için libfreenect’in en son sürümünün destekleri) ), Oneiric (11.10) ve Debian Squeeze ve Wheezy yazı yazılırken) NeuroDebian deposunda mevcuttur. Orada bulunan paketler, resmi Debian paketinin sahipleri tarafından oluşturulur ve Debian/Ubuntu standartlarını takip eder.

NeuroDebian deposunu etkinleştirmek için:

$ wget -O- http://neuro.debian.net/lists/$(lsb_release -cs).us-nh | sudo tee /etc/apt/sources.list.d/neurodebian.sources.list
$ sudo apt-key adv --recv-keys --keyserver pgp.mit.edu 2649A5A9
$ sudo apt-get update

libfreenect kurulumu öncekiyle aynıdır:

$ sudo apt-get install freenect

Cihaza kök ayrıcalıkları olmadan erişmek için kullanıcınızın plugdev grubuna (Masaüstü kullanıcısı için varsayılan) ait olduğundan emin olun. Durum böyle değilse, bunları şu şekilde ekleyin:

$ sudo adduser $USER plugdev

ve çıkış yapın ve tekrar giriş yapın Libya PPA

Lucid (10.04), Maverick (10.10), Natty (11.04), Oneiric (11.10) ve Pangolin (12.04) için Ubuntu Launchpad PP bu link adresinde mevcuttur.

kullanmak için bir konsol açın ve çalıştırın:

$ sudo add-apt-repository ppa:floe/libtisch
$ sudo apt-get update

Bundan sonra şunları kullanabilirsiniz:

$ sudo apt-get install libfreenect libfreenect-dev libfreenect-demos

Bu, libfreenect’i, geliştirme başlıklarını ve demo uygulamalarını yükleyecektir.

Bundan sonra, kendinizi ‘video’ grubuna eklemeniz ve tekrar oturum açmanız gerekir. Paket zaten udev arka plan programı için gerekli kuralları içerir, böylece grup videosundaki kullanıcılar cihaza erişim mümkün olacaktır.

$ sudo adduser $USER video

oturumu kapatıp tekrar açtığınızdan emin olun. Yeniden başlatmanız gerekmez, kinect cihazını hemen şimdi takmanız yeterlidir (zaten bağlıysa, çıkarın ve yeniden takın).

Demo uygulamalarını başlatmak için şunu kullanın:

$ freenect-glview

Cihaza erişimle ilgili sorunlar

Sorun olması durumunda, çalıştırın

$ lsusb | grep Xbox

hangi 3 cihazı listelemelidir

> lsusb | grep Xbox                                                 Bus
> 001 Device 021: ID 045e:02ae Microsoft Corp. Xbox NUI Camera Bus 001
> Device 019: ID 045e:02b0 Microsoft Corp. Xbox NUI Motor Bus 001 Device
> 020: ID 045e:02ad Microsoft Corp. Xbox NUI Audio

Eğer mevcut değillerse, çalıştırın

echo -1 | sudo tee -a /sys/module/usbcore/parameters/autosuspend

ve Kinekt’i yeniden bağlayın ve Kinect Camera listelenmelidir

Freenect- öneki ile başlayan tüm demo uygulamalarını bulabilirsiniz. Ubuntu Manuel Kurulum

Anında çalışmaya başlamak için hızlı kopyala-yapıştır talimatları:

sudo apt-get install git-core cmake libglut3-dev pkg-config build-essential libxmu-dev libxi-dev libusb-1.0-0-dev
git clone git://github.com/OpenKinect/libfreenect.git
cd libfreenect
mkdir build
cd build
cmake ..
make
sudo make install
sudo ldconfig /usr/local/lib64/
sudo freenect-glview

Not: apt-get’in libglut3’ü bulamadığını söyleyen bir hata alırsanız, yerine freeglut3-* olan daha yeni bir Ubuntu sürümünde olabilir libglut3-*, böylece ilk apt-get kurulumunuz şöyle görünür:

sudo apt-get install git-core cmake freeglut3-dev pkg-config build-essential libxmu-dev libxi-dev libusb-1.0-0-dev

Kinect’i root olmayan bir kullanıcı olarak kullanmak için aşağıdakileri yapın:

sudo adduser $USER video

Ayrıca Linux aygıt yöneticisi için kurallar içeren bir dosya oluşturun:

sudo nano /etc/udev/rules.d/51-kinect.rules

Kopyala ve yapıştır:

# ATTR{product}=="Xbox NUI Motor"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02b0", MODE="0666"
# ATTR{product}=="Xbox NUI Audio"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02ad", MODE="0666"
# ATTR{product}=="Xbox NUI Camera"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02ae", MODE="0666"
# ATTR{product}=="Xbox NUI Motor"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02c2", MODE="0666"
# ATTR{product}=="Xbox NUI Motor"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02be", MODE="0666"
# ATTR{product}=="Xbox NUI Motor"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02bf", MODE="0666"

Oturumu kapatıp tekrar açtığınızdan emin olun.

Cihazınıza erişemiyorsanız veya aygıtınızı kullanmak için hala kök ayrıcalıklarına ihtiyacınız varsa: bazı durumlarda yüklü iki sürücünün (libfreenect ve primesense) izinleri arasında çakışmalar olabilir. Durumunuz buysa, primesense’in sensör sürücüsünü yeniden yüklemeyi deneyin ve yalnızca primesense’in kurallar dosyasını /etc/udev/rules.d/55-primesense-usb.rules tutun, /etc/udev/rules.d/51'i kaldırın. -kinect.rules dosyasını siz oluşturduysanız.
