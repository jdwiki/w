Kurulum veya Kurulum #

Düzenlemek #

“npm”, Node.js ile birlikte gelir, dolayısıyla Node.js’yi yüklerseniz otomatik olarak “npm” de yüklenir. Mevcut ve LTS sürümü arasında seçim yapabilirsiniz

Pencereler #

Microsoft Windows için https://nodejs.org/en/download/ adresinden bir MSI yükleyici indirebilirsiniz.

OS KALDIRMA #

Apple OS X için aynı konumdan bir PKG yükleyici indirebilirsiniz https://nodejs.org/en/download/.

Linux #

Linux için Node.js ve npm’yi kurmak için paket yöneticinizi kullanabilirsiniz.

Ayrıca Node.js’yi kaynaktan derleyebilirsiniz ve yine de npm alırsınız. Ayrıca çalıştırabileceğiniz ve “npm"yi yükleyecek bir komut dosyası da vardır:

curl -L https://www.npmjs.com/install.sh | sh

npm’yi en son sürüme yükseltin #

“npm” kurulumunuzu güncellemek için önerilen yöntem, “npm"nin kendisini kurmasıdır:

npm install -g npm@latest

Alternatif olarak, en son sürüm yerine mevcut LTS sürümüne yükseltebilirsiniz:

npm install -g npm@lts

[nvm](https://www.wikiod.com/tr/node-js/nodejsyi-yukleme#Düğüm Sürüm Yöneticisini (nvm) Kullanma ile Node’un (ve npm) herhangi bir sürümünü de yükleyebilirsiniz. -nvm#t=201611281916227729431). Bir “nvm” kurulumu ile “npm” ile global olarak kurulum yaparken, “sudo” (veya Windows’ta Yönetici Olarak Çalıştır) kullanmanıza gerek yoktur.

Global Paketleri Yükleme #

Genel bir paket yükleyin

Global olarak kurulan paketler, modülleri “{prefix}/lib/node_modules” içine bırakır ve yürütülebilir dosyaları “{prefix}/bin” içine koyar, burada “{prefix}” genellikle “/usr/local” gibi bir şeydir. Global bir modül kurmak, onun ikili dosyalarının sizin PATH ortam değişkeninizde biteceği anlamına gelir. Genellikle, bir komut satırı aracıysa veya kabuğunuzda kullanmak istediğiniz bir şeyse, genel bir modül kurmak istersiniz.

npm install --global package-name

Genel bir paketi kaldır

npm uninstall --global package-name

Not: “–global” argümanı “-g” olarak basitleştirilebilir. Bu nedenle, örneğin, ilk komut tam olarak aynı sonuçla npm install -g paket-adı olabilirdi.

Not: *nix sistemlerinde, global paketlerin yüklenmesi süper kullanıcı izinleri gerektirebilir. Bunu yapmamak, “EACCES” ile başarısız olur. Bu durumda, çalıştırın:

sudo npm install --global package-name

Paketleri yükle #

Paketlerin kurulabileceğine dikkat edin Bu komut, adlandırılmış paketlerin mevcut en yeni sürümünü yükler:

hem yerel hem de küresel.

Yerel kurulum, npm’nin paketinizi mevcut çalışma dizinine kurması anlamına gelir. Düğüm modülleri ./node_modules içine, yürütülebilir dosyalar ./node_modules/.bin/ içine gider. Genellikle, bağımlılık olarak programınızda kullanmak için yerel modüller kurmak isteyeceksiniz ve bunlar yalnızca kuruldukları yerde çalışacaklardır.

npm install <package names>

Kısa gösterim:

npm i <package names>

“npm”, “package.json” içinde depolanan “bağımlılıklar” ve “devDependencies” nesneleri (birden çok modül yükleyerek) aracılığıyla, geçerli dizindeki bir “package.json” dosyasıyla çeşitli yararlı şekillerde etkileşime girebilir:

Parametresiz “npm install” komutu

npm install

“package.json” içindeki “bağımlılıklar” ve “devDependencies” nesnelerine nesne anahtarları olarak adlandırılan tüm paketleri, nesne değerleriyle belirtilen anlamsal sürüm oluşturma kısıtlamalarını kullanarak kurar.

Yeni yazılım geliştirirken:

Modülünüze her zaman dahil edilmesi gereken ‘<paket adları>’ ve kurduğunuz npm modüllerinin sürümlerini eklemek için ‘-S’ seçeneğini kullanın. Yüklemeden sonra, “package.json” dosyasında izlenen “bağımlılıklar” listesine eklenir.

npm i <package names> -S

Modülünüzü daha fazla geliştirmek veya test etmek için diğer geliştiriciler tarafından ihtiyaç duyulan “<paket adları>” ve yüklediğiniz npm modüllerinin sürümlerini eklemek için “-D” seçeneğini kullanın. Yüklemeden sonra ‘package.json’ dosyasında izlenen ‘devDependencies’ listesine eklenir.

npm i <package names> -D

“lodash” ve “mocha” paket adlarıdır.

Paketleri güncelleme #

Her uygulamada yaşam döngüsü, bileşenlerinin güncellenmesi gereken bir gün gelir. Herkes her bir bağımlılığı tek tek güncellemenin acısını bilir. Peki burada sadece komutu vermeniz gerekiyor:

npm update (-g)

“-g” oradaysa, npm global paketleri güncelleyecektir.

Bağımlılıkları yönetmek için npm kullanma #

Yani uygulamanızı birden çok siteye dağıtmak mı istiyorsunuz? ve projenizde bunları tek tek yüklemek için çok fazla bağımlılık var mı? Npm’nin bir çözümü var, sadece aşağıdaki komutu verin:

npm init

Projenin kök klasöründe ekrandaki talimatları izleyin (istediğiniz değeri yazın ve enter tuşuna basın) ve ardından bir bağımlılığı kaydetmek istiyorsanız şunu ekleyin:

--save

Sonra senin

npm install

örneğin komutlar:

npm install mypackagename --save

Ve sonra bu bağımlılık kaydedilir, o zaman “node_modules” klasörünü taşımanız gerekmez. Kaydedilen tüm bağımlılık sorununu yüklemek için:

npm install

ve kaydedilen tüm bağımlılıklar yüklenecektir.
