Mac OS’de jHipster kullanarak Spring Boot Uygulaması oluşturun #

jHipster, bir REST API arka ucu ve bir AngularJS ve Twitter Bootstrap ön ucu ile bir Spring Boot web uygulamasını önyüklemenize olanak tanır.

jHipster hakkında daha fazla bilgiyi burada bulabilirsiniz: jHipster Documentation

** Brew yükleyin:**

 /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Brew’in nasıl kurulacağına ilişkin ek bilgileri burada görüntüleyin: Install Brew

Gradle’ı kurun

Gradle bir bağımlılık yönetimi ve yapı sistemidir.

 brew install gradle

Git’i yükleyin

Git bir sürüm kontrol aracıdır

 brew install git

NodeJS’yi yükleyin

NodeJS, diğer araçları kurmak için gereken düğüm paketi yöneticisi olan npm’ye erişmenizi sağlar.

 brew install node

Yeoman’ı yükleyin

Yeoman bir jeneratördür

 npm install -g yo

Bower’ı yükleyin

Bower bir bağımlılık yönetimi aracıdır

 npm install -g bower

Gulp’u yükleyin

Gulp bir görev koşucusudur

 npm install -g gulp

jHipster Yeoman Generator’ı kurun

Bu jHipster jeneratörü

 npm install -g generator-jhipster

Uygulama Oluştur

Bir Terminal penceresi açın.

Projelerinizi tutacağınız kök dizine gidin. Uygulamanızı oluşturacağınız boş bir dizin oluşturun

 mkdir myapplication

O dizine git

 cd myapplication/

Uygulamanızı oluşturmak için şunu yazın

 yo jhipster

Aşağıdaki sorular sorulacak

Hangi türde bir uygulama oluşturmak istersiniz?

Uygulamanızın türü, bir mikro hizmet mimarisi kullanmak isteyip istemediğinize bağlıdır. Mikro hizmetler hakkında tam bir açıklama burada mevcuttur, eğer emin değilseniz varsayılan “Monolitik uygulamayı” kullanın.

Java uygulamanız bunu kök paketi olarak kullanacaktır.

Hangi geliştirme veritabanını kullanmak istersiniz?

Emin değilseniz varsayılan olarak H2’yi kullanın

H2, bellekte çalışıyor. Bu, JHipster’ı kullanmanın en kolay yoludur, ancak sunucunuzu yeniden başlattığınızda verileriniz kaybolacaktır.

Hibernate 2. seviye önbelleği kullanmak istiyor musunuz?

Hazırda Bekletme, JHipster tarafından kullanılan JPA sağlayıcısıdır. Performans nedenleriyle, bir önbellek kullanmanızı ve uygulamanızın ihtiyaçlarına göre ayarlamanızı şiddetle tavsiye ederiz. Bunu yapmayı seçerseniz, ehcache (yerel önbellek) veya Hazelcast (kümelenmiş bir ortamda kullanım için dağıtılmış önbellek) kullanabilirsiniz.

Uygulamanızda bir arama motoru kullanmak ister misiniz? Elasticsearch, Spring Data Elasticsearch kullanılarak yapılandırılacaktır. Elasticsearch kılavuzumuzda daha fazla bilgi bulabilirsiniz.

Emin değilseniz hayır’ı seçin

Kümelenmiş HTTP oturumları kullanmak istiyor musunuz?

Varsayılan olarak JHipster, yalnızca Spring Security’nin kimlik doğrulama ve yetkilendirme bilgilerini depolamak için bir HTTP oturumu kullanır. Elbette, HTTP oturumlarınıza daha fazla veri koymayı seçebilirsiniz. Bir kümede çalışıyorsanız, özellikle “yapışkan oturumlar” içeren bir yük dengeleyici kullanmıyorsanız, HTTP oturumlarını kullanmak sorunlara neden olur. Oturumlarınızı kümenizin içinde çoğaltmak istiyorsanız, Hazelcast’in yapılandırılması için bu seçeneği seçin.

Emin değilseniz hayır’ı seçin

WebSockets kullanmak istiyor musunuz? Web yuvaları, Spring Websocket kullanılarak etkinleştirilebilir. Ayrıca, çerçeveyi nasıl verimli bir şekilde kullanacağınızı göstermek için eksiksiz bir örnek sunuyoruz.

Emin değilseniz hayır’ı seçin

Maven veya Gradle kullanmak ister misiniz? Oluşturduğunuz Java uygulamanızı Maven veya Gradle ile oluşturabilirsiniz. Maven daha kararlı ve daha olgun. Gradle daha esnek, genişletilmesi daha kolay ve daha fazla yutturmaca.

Emin değilseniz Gradle‘ı seçin

CSS’niz için LibSass stil sayfası ön işlemcisini kullanmak ister misiniz? Düğüm-sass, CSS tasarımını basitleştirmek için harika bir çözüm. Verimli bir şekilde kullanmak için, otomatik olarak yapılandırılacak bir Gulp sunucusu çalıştırmanız gerekecektir.

Emin değilseniz hayır’ı seçin

Angular Çeviri ile çeviri desteğini etkinleştirmek ister misiniz? Varsayılan olarak JHipster, hem Angular Translate ile istemci tarafında hem de sunucu tarafında mükemmel uluslararasılaştırma desteği sağlar. Ancak, uluslararasılaştırma biraz ek yük getirir ve yönetimi biraz daha karmaşıktır, bu nedenle bu özelliği yüklememeyi seçebilirsiniz.
