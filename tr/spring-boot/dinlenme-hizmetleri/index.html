Parametreler #

Açıklama Sütun
@Kontrolör Açıklamalı bir sınıfın “Denetleyici” (web denetleyicisi) olduğunu gösterir.
@RequestMapping Web isteklerini belirli işleyici sınıflarına (sınıfla birlikte kullandıysak) ve/veya işleyici yöntemlerine (yöntemlerle birlikte kullandıysak) eşlemek için açıklama.
yöntem = RequestMethod.GET HTTP istek yöntemlerinin türü
YanıtBody Bir yöntem dönüş değerini gösteren açıklama, web yanıt gövdesine bağlanmalıdır
@RestController @Controller + ResponseBody
@ResponseEntity HttpStatus durum kodu ekleyen HttpEntity uzantısı, dönüş http kodunu kontrol edebiliriz

REST-Servis Oluşturma #

  1. STS (Spring Starter Project) veya Spring Initializr (https://start.spring.io adresinde) kullanarak proje oluşturun.
  2. pom.xml dosyanıza bir Web Bağımlılığı ekleyin:
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

veya “Bağımlılıkları ara” arama kutusuna web yazın, web bağımlılığı ekleyin ve sıkıştırılmış projeyi indirin.

  1. Bir Etki Alanı Sınıfı (yani Kullanıcı) oluşturun
 public class User {
    
        private Long id;
    
        private String userName;
    
        private String password;
    
        private String email;
    
        private String firstName;
    
        private String lastName;
    
        public Long getId() {
            return id;
        }
    
        public void setId(Long id) {
            this.id = id;
        }
    
        public String getUserName() {
            return userName;
        }
    
        public void setUserName(String userName) {
            this.userName = userName;
        }
    
        public String getPassword() {
            return password;
        }
    
        public void setPassword(String password) {
            this.password = password;
        }
    
        public String getEmail() {
            return email;
        }
    
        public void setEmail(String email) {
            this.email = email;
        }
    
        public String getFirstName() {
            return firstName;
        }
    
        public void setFirstName(String firstName) {
            this.firstName = firstName;
        }
    
        public String getLastName() {
            return lastName;
        }
    
        public void setLastName(String lastName) {
            this.lastName = lastName;
        }
    
        @Override
        public String toString() {
            return "User [id=" + id + ", userName=" + userName + ", password=" + password + ", email=" + email
                    + ", firstName=" + firstName + ", lastName=" + lastName + "]";
        }
    
        public User(Long id, String userName, String password, String email, String firstName, String lastName) {
            super();
            this.id = id;
            this.userName = userName;
            this.password = password;
            this.email = email;
            this.firstName = firstName;
            this.lastName = lastName;
        }
    
        public User() {}
    }

  1. UserController sınıfı oluşturun ve @Controller, @RequestMapping notlarını ekleyin

     @Controller
 @RequestMapping(value = "api")
 public class UserController {
 }

  2. Veritabanını simüle etmek ve listeye 2 kullanıcı eklemek için statik Liste kullanıcıları değişkenini tanımlayın

     private static List<User> users = new ArrayList<User>();

 public UserController() {
     User u1 = new User(1L, "shijazi", "password", "shijazi88@gmail.com", "Safwan", "Hijazi");
     User u2 = new User(2L, "test", "password", "test@gmail.com", "test", "test");
     users.add(u1);
     users.add(u2);
 }

  3. Tüm kullanıcıları statik listede döndürmek için yeni yöntem oluşturun (getAllUsers)

     @RequestMapping(value = "users", method = RequestMethod.GET)
 public @ResponseBody List<User> getAllUsers() {
      return users;
 }

  4. Uygulamayı [ mvn clean install spring-boot:run] çalıştırın ve bu URL’yi http://localhost:8080/api/users olarak adlandırın.

  5. Sınıfa @RestController ile açıklama ekleyebiliriz ve bu durumda ResponseBody’yi bu sınıftaki tüm yöntemlerden kaldırabiliriz, (@RestController = @Controller + ResponseBody), bir nokta daha http dönüşünü kontrol edebiliriz kodunu ‘ResponseEntity’ kullanırsak, aynı önceki işlevleri uygulayacağız, ancak ‘@RestController’ ve ‘ResponseEntity’yi kullanacağız.

    @RestController @RequestMapping(value = “api2”) public class UserController2 {

     private static List<User> users = new ArrayList<User>();

 public UserController2() {
     User u1 = new User(1L, "shijazi", "password", "shijazi88@gmail.com", "Safwan", "Hijazi");
     User u2 = new User(2L, "test", "password", "test@gmail.com", "test", "test");
     users.add(u1);
     users.add(u2);
 }

 @RequestMapping(value = "users", method = RequestMethod.GET)
 public ResponseEntity<?> getAllUsers() {
    try {
        return new ResponseEntity<>(users, HttpStatus.OK);
    } catch (Exception e) {
        return new ResponseEntity<>(HttpStatus.INTERNAL_SERVER_ERROR);
    }
 }

    }

şimdi uygulamayı çalıştırmayı deneyin ve bu URL’yi http://localhost:8080/api2/users olarak adlandırın

JERSEY ve Spring Boot ile Dinlenme Hizmeti Oluşturma #

Jersey, Rest Services oluşturmak için mevcut birçok çerçeveden biridir, Bu örnek size Jersey ve Spring Boot kullanarak Rest Services’ı nasıl oluşturacağınızı gösterecektir.

1.Proje Kurulumu #

STS kullanarak veya Spring Initializr sayfasını kullanarak yeni bir proje oluşturabilirsiniz. Bir proje oluştururken aşağıdaki bağımlılıkları ekleyin:

  1. Jersey (JAX-RS)

  2. ağ

  3. Bir Denetleyici Oluşturma ======================= Jersey Web Hizmetimiz için bir denetleyici oluşturalım

    @Path("/Welcome") @Component public class MyController { @GET public String welcomeUser(@QueryParam(“user”) String user){ return “Welcome “+user; }
    }

@Path("/Welcome") ek açıklaması, çerçeveye bu denetleyicinin URI yoluna yanıt vermesi gerektiğini belirtir /Hoş Geldiniz

@QueryParam("user") notu, çerçeveye user adında bir sorgu parametresi beklediğimizi belirtir.

  1. Kablolama Jersey Konfigürasyonları ============================== Şimdi Jersey Framework’ü Spring Boot ile yapılandıralım: “org.glassfish.jersey.server.ResourceConfig” öğesini genişleten bir yay bileşeni yerine bir sınıf oluşturun:

    @Component @ApplicationPath("/MyRestService”) public class JerseyConfig extends ResourceConfig { /** * Register all the Controller classes in this method * to be available for jersey framework */ public JerseyConfig() { register(MyController.class); }

    }

@ApplicationPath("/MyRestService") çerçeveye, yalnızca /MyRestService yoluna yönlendirilen isteklerin jersey çerçevesi tarafından ele alınması gerektiğini belirtir, diğer isteklerin yine de yay çerçevesi tarafından ele alınmaya devam etmesi gerekir.

Konfigürasyon sınıfına @ApplicationPath ile açıklama eklemek iyi bir fikirdir, aksi takdirde tüm istekler Jersey tarafından işlenir ve bunu atlayamayacağız ve gerekirse bir yay denetleyicisinin işlemesine izin vermeyeceğiz.

4.Bitti #

Uygulamayı başlatın ve (Yaylı önyüklemeyi 8080 numaralı bağlantı noktasında çalışacak şekilde yapılandırdığınızı varsayarak) gibi bir örnek URL’yi çalıştırın:

http://localhost:8080/MyRestService/Welcome?user=User

RestTemplate (GET) ile REST API Kullanmak #

“RestTemplate” ile bir REST API kullanmak için, Spring boot initialzr ile bir Spring boot projesi oluşturun ve Web bağımlılığının eklendiğinden emin olun:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

Projenizi kurduktan sonra, bir ‘RestTemplate’ çekirdeği oluşturun. Bunu, önceden oluşturulmuş ana sınıf içinde veya ayrı bir yapılandırma sınıfı ("@Configuration” ile açıklamalı bir sınıf) içinde yapabilirsiniz:

@Bean
public RestTemplate restTemplate() {
    return new RestTemplate();
}

Bundan sonra, [REST hizmeti oluştururken][2] yapmanız gerekene benzer bir etki alanı sınıfı oluşturun.

public class User {
    private Long id;
    private String username;
    private String firstname;
    private String lastname;

    public Long getId() {
        return id;
    }

    public String getUsername() {
        return username;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public String getFirstname() {
        return firstname;
    }

    public void setFirstname(String firstname) {
        this.firstname = firstname;
    }

    public String getLastname() {
        return lastname;
    }

    public void setLastname(String lastname) {
        this.lastname = lastname;
    }
}

İstemcinizde, ‘RestTemplate’i otomatik olarak bağlayın:

@Autowired
private RestTemplate restTemplate;

Tek bir kullanıcı döndüren bir REST API’sini kullanmak için artık şunları kullanabilirsiniz:

String url = "http://example.org/path/to/api";
User response = restTemplate.getForObject(url, User.class);

Bir liste veya kullanıcı dizisi döndüren bir REST API’sini kullanmak için iki seçeneğiniz vardır. Ya bir dizi olarak tüketin:

String url = "http://example.org/path/to/api";
User[] response = restTemplate.getForObject(url, User[].class);

Veya ParameterizedTypeReference kullanarak tüketin:

String url = "http://example.org/path/to/api";
ResponseEntity<List<User>> response = restTemplate.exchange(url, HttpMethod.GET, null, new ParameterizedTypeReference<List<User>>() {});
List<User> data = response.getBody();

‘ParameterizedTypeReference’ kullanırken, daha gelişmiş ‘RestTemplate.exchange()’ yöntemini kullanmanız ve bunun bir alt sınıfını oluşturmanız gerekeceğini unutmayın. Yukarıdaki örnekte anonim bir sınıf kullanılmıştır.
