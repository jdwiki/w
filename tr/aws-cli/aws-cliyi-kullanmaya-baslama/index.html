Kurulum ve kurulum #

Kullandığınız işletim sistemi ve ortama bağlı olarak, AWS CLI’yi makinenize kurmanın birkaç farklı yolu vardır:

Microsoft Windows’ta – MSI yükleyicisini kullanın. Linux, OS X veya Unix’te - pip (Python yazılımı için bir paket yöneticisi) kullanın veya birlikte verilen yükleyiciyle manuel olarak kurun.

pip kullanarak yükleyin:

Yüklemek için python’a ihtiyacınız olacak (sürüm 2, 2.6.5+,3 veya 3.3+). ile kontrol edin

python --version

pip --help

Her ikisinin de kurulu olduğu göz önüne alındığında, aws cli’yi kurmak için aşağıdaki komutu kullanın.

sudo pip install awscli

Windows’a yükleyin AWS CLI, Microsoft Windows XP veya sonraki sürümlerde desteklenir. Windows kullanıcıları için MSI yükleme paketi, başka herhangi bir önkoşul yüklemeden AWS CLI’yi yüklemenin tanıdık ve kullanışlı bir yolunu sunar. Windows kullanıcıları, paket yönetimi için zaten pip kullanmıyorlarsa MSI yükleyicisini kullanmalıdır.

  • Windows 32-bit için MSI Yükleyici
  • Windows 64-bit için MSI Yükleyici

İndirilen MSI yükleyicisini çalıştırın. Görüntülenen talimatları izleyin.

AWS CLI’yi birlikte verilen yükleyiciyi kullanarak yüklemek için

Önkoşullar:

  • Linux, OS X veya Unix
  • Python 2 sürüm 2.6.5+ veya Python 3 sürüm 3.3+

  1. wget veya curl kullanarak AWS CLI Bundled Installer’ı indirin.

  2. Paketi açın.

  3. Yürütülebilir yükleme dosyasını çalıştırın.

Linux ve OS X’te her adıma karşılık gelen üç komut şunlardır:

$ curl "https://s3.amazonaws.com/aws-cli/awscli-bundle.zip" -o "awscli-bundle.zip"
$ unzip awscli-bundle.zip
$ sudo ./awscli-bundle/install -i /usr/local/aws -b /usr/local/bin/aws

OS X’te HomeBrew kullanarak yükleyin:

OS X için başka bir seçenek

brew install awscli

AWS CLI Kurulumunu Test Edin

Yardım dosyasını görüntüleyerek CLI’nin doğru şekilde kurulduğunu onaylayın. Bir terminal, kabuk veya komut istemi açın, aws help girin ve Enter’a basın:

$ aws help

AWS CLI’yi yapılandırma

Kurulumu bitirdikten sonra yapılandırmanız gerekir. aws’de hesabınızı oluşturduğunuzda alacağınız erişim anahtarınıza ve gizli anahtarınıza ihtiyacınız olacak. Ayrıca varsayılan bir bölge ad ve varsayılan bir çıktı türü (metin|tablo|json) belirtebilirsiniz.

$ aws configure
AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
Default region name [None]: us-west-2
Default output format [None]: ENTER

CLI aracının güncellenmesi

Amazon, AWS Aracının yeni sürümlerini periyodik olarak yayınlar. Araç Python Pip aracı kullanılarak kurulduysa, aşağıdaki komut uzak depoda güncellemeleri kontrol edecek ve bunu yerel sisteminize uygulayacaktır.

$ pip install awscli --upgrade

Yeni Profil Oluşturma #

“myprofile” adıyla yeni bir kimlik bilgisi profili oluşturmak için:

$ aws configure --profile myprofile
AWS Access Key ID [None]: ACCESSKEY
AWS Secret Access Key [None]: SECRETKEY
Default region name [None]: REGIONNAME
Default output format [None]: text | table | json

AWS erişim anahtarı kimliği ve sırrı için AWS konsolunda bir IAM kullanıcısı oluşturun ve bunun için anahtarlar oluşturun.

Bölge, “eu-west-1” veya “us-east-1” biçimindeki komutlar için varsayılan bölge olacaktır.

Varsayılan çıktı biçimi “metin”, “tablo” veya “json” olabilir.

Artık “–profile” seçeneğini kullanarak profil adını diğer komutlarda kullanabilirsiniz, örneğin:

$ aws ec2 describe-instances --profile myprofile

Diğer diller için AWS kitaplıkları (örneğin, Ruby için “aws-sdk” veya Python için “boto3”), bu yöntemle oluşturduğunuz profili kullanma seçeneklerine sahiptir. Örneğin. boto3 içinde yeni bir oturum oluşturmak şu şekilde yapılabilir, boto3.Session(profile_name:'myprofile') ve profil için oluşturduğunuz kimlik bilgilerini kullanır.

aws-cli yapılandırmanızın ayrıntıları “~/.aws/config” ve “~/.aws/credentials” (linux ve mac-os’ta) içinde bulunabilir. Bu ayrıntılar oradan manuel olarak düzenlenebilir.

S3 paketlerini listeleyin #

aws s3 ls

Adlandırılmış bir profil kullan

aws --profile myprofile s3 ls

Klasörlerdeki nesneler de dahil olmak üzere bir kovadaki tüm nesneleri, boyutu insan tarafından okunabilir biçimde ve sonunda kova özelliklerinin bir özetiyle listeleyin -

aws s3 ls --recursive --summarize --human-readable s3://<bucket_name>/

Bash ile Ubuntu için AWS tamamlayıcısı #

Komutların otomatik olarak tamamlanması için aşağıdaki yardımcı program kullanılabilir:

$ which aws_completer
/usr/bin/aws_completer

$ complete -C '/usr/bin/aws_completer' aws

Gelecekteki kabuk oturumları için bunu ~/.bashrc dosyanıza eklemeyi düşünün.

$ echo "complete -C '/usr/bin/aws_completer' aws" >> ~/.bashrc

Kontrol etmek için şunu yazın:

$ aws ec

[TAB] tuşuna basın, otomatik olarak 2 eklemelidir:

$ aws ec2

aws cli komutlarını kullanma #

aws cli’yi kullanmak için sözdizimi aşağıdaki gibidir:

aws [options] <command> <subcommand> [parameters]

’ec2’ komutunu ve ‘instance-instances’ alt komutunu kullanan bazı örnekler:

aws ec2 describe-instances

aws ec2 describe-instances --instance-ids <your-id>

Sahte kimlikli örnek:

aws ec2 describe-instances --instance-ids i-c71r246a

AWS CLI, AWS kaynaklarını yönetmek için yaygın bir CLI aracıdır. Bu tek araçla tüm aws kaynaklarını yönetebiliriz

sudo apt-get install -y python-dev python-pip

Tek gömlekleri bash #

cat <file> # output a file
tee # split output into a file
cut -f 2 # print the 2nd column, per line
sed -n '5{p;q}' # print the 5th line in a file
sed 1d # print all lines, except the first
tail -n +2 # print all lines, starting on the 2nd
head -n 5 # print the first 5 lines
tail -n 5 # print the last 5 lines

expand # convert tabs to 4 spaces
unexpand -a # convert 4 spaces to tabs
wc # word count
tr ' ' \\t # translate / convert characters to other characters

sort # sort data
uniq # show only unique entries
paste # combine rows of text, by line
join # combine rows of text, by initial column value




Cloudtrail - Günlüğe Kaydetme ve Denetim #

http://docs.aws.amazon.com/cli/latest/reference/cloudtrail/ Kaynak düzeyinde izin desteğiyle birlikte toplam 5 yol

# list all trails
aws cloudtrail describe-trails

# list all S3 buckets
aws s3 ls

# create a new trail
aws cloudtrail create-subscription \
    --name awslog \
    --s3-new-bucket awslog2016

# list the names of all trails
aws cloudtrail describe-trails --output text | cut -f 8

# get the status of a trail
aws cloudtrail get-trail-status \
    --name awslog

# delete a trail
aws cloudtrail delete-trail \
    --name awslog

# delete the S3 bucket of a trail
aws s3 rb s3://awslog2016 --force

# add tags to a trail, up to 10 tags
aws cloudtrail add-tags \
    --resource-id awslog \
    --tags-list "Key=log-type,Value=all"

# list the tags of a trail
aws cloudtrail list-tags \
    --resource-id-list 

# remove a tag from a trail
aws cloudtrail remove-tags \
    --resource-id awslog \
    --tags-list "Key=log-type,Value=all"

Gruplar, İlkeler, Yönetilen İlkeler #

http://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles.html http://docs.aws.amazon.com/cli/latest/reference/iam/

# list all groups
aws iam list-groups

# create a group
aws iam create-group --group-name FullAdmins

# delete a group
aws iam delete-group \
    --group-name FullAdmins

# list all policies
aws iam list-policies

# get a specific policy
aws iam get-policy \
    --policy-arn <value>

# list all users, groups, and roles, for a given policy
aws iam list-entities-for-policy \
    --policy-arn <value>

# list policies, for a given group
aws iam list-attached-group-policies \
    --group-name FullAdmins

# add a policy to a group
aws iam attach-group-policy \
    --group-name FullAdmins \
    --policy-arn arn:aws:iam::aws:policy/AdministratorAccess

# add a user to a group
aws iam add-user-to-group \
    --group-name FullAdmins \
    --user-name aws-admin2

# list users, for a given group
aws iam get-group \
    --group-name FullAdmins

# list groups, for a given user
aws iam list-groups-for-user \
    --user-name aws-admin2

# remove a user from a group
aws iam remove-user-from-group \
    --group-name FullAdmins \
    --user-name aws-admin2

# remove a policy from a group
aws iam detach-group-policy \
    --group-name FullAdmins \
    --policy-arn arn:aws:iam::aws:policy/AdministratorAccess

# delete a group
aws iam delete-group \
    --group-name FullAdmins

anahtar çiftleri #

http://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-key-pairs.html

# list all keypairs
# http://docs.aws.amazon.com/cli/latest/reference/ec2/describe-key-pairs.html
aws ec2 describe-key-pairs

# create a keypair
# http://docs.aws.amazon.com/cli/latest/reference/ec2/create-key-pair.html
aws ec2 create-key-pair \
    --key-name <value>

# create a new private / public keypair, using RSA 2048-bit
ssh-keygen -t rsa -b 2048

# import an existing keypair
# http://docs.aws.amazon.com/cli/latest/reference/ec2/import-key-pair.html
aws ec2 import-key-pair \
    --key-name keyname_test \
    --public-key-material file:///home/apollo/id_rsa.pub

# delete a keypair
# http://docs.aws.amazon.com/cli/latest/reference/ec2/delete-key-pair.html
aws ec2 delete-key-pair \
    --key-name <value>

Güvenlik Grupları #

http://docs.aws.amazon.com/cli/latest/reference/ec2/index.html

# list all security groups
aws ec2 describe-security-groups

# create a security group
aws ec2 create-security-group \
    --vpc-id vpc-1a2b3c4d \
    --group-name web-access \
    --description "web access"

# list details about a securty group
aws ec2 describe-security-groups \
    --group-id sg-0000000

# open port 80, for everyone
aws ec2 authorize-security-group-ingress \
    --group-id sg-0000000 \
    --protocol tcp \
    --port 80 \
    --cidr 0.0.0.0/24

# get my public ip
my_ip=$(dig +short myip.opendns.com @resolver1.opendns.com);
echo $my_ip

# open port 22, just for my ip
aws ec2 authorize-security-group-ingress \
    --group-id sg-0000000 \
    --protocol tcp \
    --port 80 \
    --cidr $my_ip/24

# remove a firewall rule from a group
aws ec2 revoke-security-group-ingress \
    --group-id sg-0000000 \
    --protocol tcp \
    --port 80 \
    --cidr 0.0.0.0/24

# delete a security group
aws ec2 delete-security-group \
    --group-id sg-00000000

Örnekler #

http://docs.aws.amazon.com/cli/latest/reference/ec2/index.html

# http://docs.aws.amazon.com/cli/latest/reference/ec2/describe-instances.html

aws ec2 describe-instance-status \

# Log group names can be between 1 and 512 characters long. Allowed
# characters include a-z, A-Z, 0-9, '_' (underscore), '-' (hyphen),
# '/' (forward slash), and '.' (period).

# create a log stream
# http://docs.aws.amazon.com/cli/latest/reference/logs/create-log-stream.html
aws logs create-log-stream \
    --log-group-name "DefaultGroup" \
    --log-stream-name "syslog"

    --log-stream-name-prefix "syslog"

# http://docs.aws.amazon.com/cli/latest/reference/logs/delete-log-stream.html
aws logs delete-log-stream \
    --log-group-name "DefaultGroup" \
