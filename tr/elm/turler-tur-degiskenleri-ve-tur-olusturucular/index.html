Yeni Tipler Kullanarak Tip Güvenliğini İyileştirme #

Örtüşme türleri, standart metni azaltır ve okunabilirliği artırır, ancak diğer ad türünün kendisinden daha güvenli değildir. Aşağıdakileri göz önünde bulundur:

type alias Email = String

type alias Name = String

someEmail = "holmes@private.com"

someName = "Benedict"

sendEmail : Email -> Cmd msg
sendEmail email = ...

Yukarıdaki kodu kullanarak, sendEmail someName yazabiliriz ve gerçekten olmamasına rağmen derlenir, çünkü isimler ve e-postaların ikisi de String olmasına rağmen, tamamen farklı şeylerdir.

Yeni bir tür oluşturarak tür düzeyinde bir “Dize"yi başka bir “Dize"den gerçekten ayırt edebiliriz. “E-posta"yı “tür takma adı” yerine “tür” olarak yeniden yazan bir örnek:

module Email exposing (Email, create, send)

type Email = EmailAddress String

isValid : String -> Bool
isValid email = 
  -- ...validation logic

create : String -> Maybe Email
create email =
    if isValid email then
        Just (EmailAddress email)
    else
        Nothing

send : Email -> Cmd msg
send (EmailAddress email) = ...

“isValid” işlevimiz, bir dizenin geçerli bir e-posta adresi olup olmadığını belirlemek için bir şeyler yapar. “Oluştur” işlevi, belirli bir “Dize"nin geçerli bir e-posta olup olmadığını kontrol eder ve yalnızca doğrulanmış adresleri döndürmemizi sağlamak için “Belki” ile sarılmış bir “E-posta” döndürür. Doğrulama kontrolünden doğrudan bir “E-posta” oluşturarak “EmailAddress “somestring” yazarak geçebiliriz, ancak modül bildirimimiz burada gösterildiği gibi “EmailAddress” yapıcısını göstermiyorsa

module Email exposing (Email, create, send)

o zaman başka hiçbir modülün ‘EmailAddress’ yapıcısına erişimi olmaz, ancak ek açıklamalarda ‘Email’ türünü kullanmaya devam edebilirler. Bu modülün dışında yeni bir “E-posta” oluşturmanın tek yolu, sağladığı “oluştur” işlevini kullanmaktır ve bu işlev, ilk etapta yalnızca geçerli e-posta adresleri döndürmesini sağlar. Bu nedenle, bu API, kullanıcıyı tür güvenliği aracılığıyla otomatik olarak doğru yola yönlendirir: “gönder” yalnızca “oluştur” tarafından oluşturulan değerlerle çalışır, bu bir doğrulama gerçekleştirir ve bir “Belki E-posta” döndürdüğü için geçersiz e-postaların işlenmesini zorunlu kılar.

“Email” yapıcısını dışa aktarmak isterseniz, şunu yazabilirsiniz:

module Email exposing (Email(EmailAddress), create, send)

Artık “E-posta"yı içe aktaran herhangi bir dosya, yapıcısını da içe aktarabilir. Bu durumda, bunu yapmak, kullanıcıların doğrulamayı atlamasına ve geçersiz e-postalar “göndermesine” izin verir, ancak her zaman böyle bir API oluşturmazsınız, bu nedenle yapıcıları dışa aktarmak faydalı olabilir. Birkaç yapıcısı olan bir türle, yalnızca bazılarını dışa aktarmak isteyebilirsiniz.

Karşılaştırılabilir veri türleri #

Karşılaştırılabilir türler, Temel Bilgiler modülündeki karşılaştırma operatörleri kullanılarak karşılaştırılabilen ilkel türlerdir, örneğin: (<), (>), (<=), (>=), ` max’, ‘min’, ‘karşılaştır’

Elm’deki karşılaştırılabilir türler ‘Int’, ‘Float’, ‘Time’, ‘Char’, ‘String’ ve benzer türlerin demetleri veya listeleridir.

Dokümantasyonda veya tip tanımlarında bunlara özel tip değişken ‘karşılaştırılabilir’ olarak atıfta bulunulur, örn. “Basics.max” işlevi için tür tanımına bakın:

max : comparable -> comparable -> comparable

Yapı Tipleri #

’type alias’ anahtar sözcük kombinasyonu, bir tür için yeni bir ad verir, ancak ’type’ anahtar sözcüğü yalıtılmış olarak yeni bir tür bildirir. Bu türlerin en temellerinden birini inceleyelim: Belki

type Maybe a
    = Just a
    | Nothing

Unutulmaması gereken ilk şey, “Belki” türünün bir tür değişken ile bildirilmiş olmasıdır. /8809/tür değişkenleri) “a”. Dikkat edilmesi gereken ikinci şey, “veya” anlamına gelen | boru karakteridir. Başka bir deyişle, “Belki a” türünden bir şey ya “Yalnızca” ya da “Hiçbir şey"dir.

Yukarıdaki kodu yazdığınızda, ‘Just’ ve ‘Hiçbir şey’ değer oluşturucular olarak kapsama girer ve ‘Belki’, tür oluşturucu olarak kapsam içine girer. İşte imzaları:

Just : a -> Maybe a

Nothing : Maybe a

Maybe : a -> Maybe a -- this can only be used in type signatures

tür değişkeni “a” nedeniyle, herhangi bir tür “Belki” türünün “içine sarılabilir”. Bu nedenle, “Belki Int”, “Belki (List String)” veya “Belki (Belki (List Html))” geçerli türlerdir. Bir “vaka” ifadesi ile herhangi bir “tür” değerini yok ederken, o türün olası her örneğini hesaba katmanız gerekir. “Belki a” türünde bir değer söz konusu olduğunda, hem “Sadece a” durumunu hem de “Hiçbir şey” durumunu hesaba katmanız gerekir:

thing : Maybe Int
thing = 
    Just 3

blah : Int
blah =
    case thing of
        Just n -> 
            n

        Nothing ->
            42

-- blah = 3

Yukarıdaki kodu, “case” ifadesinde “Hiçbir şey” yan tümcesi olmadan yazmayı deneyin: derlenmeyecektir. Bu, ‘Belki’ tipi oluşturucuyu, var olmayan değerleri ifade etmek için harika bir model yapan şeydir, çünkü sizi değerin ‘Hiçbir şey’ olduğu zaman mantığını halletmeye zorlar.

Asla tipi #

“Asla” türü oluşturulamaz (“Temeller” modülü, değer oluşturucusunu dışa aktarmadı ve size “Hiçbir zaman” döndüren başka bir işlev vermedi). ‘asla : Asla’ değeri veya ‘createNever : ?? işlevi yoktur. -> Asla`.

Bunun faydaları vardır: Bir tür sisteminde gerçekleşemeyecek bir olasılığı kodlayabilirsiniz. Bu, bir ‘Int’ ile başarılı olacağını garanti eden ‘Task Never Int’ gibi türlerde görülebilir; veya Elm kodunu JavaScript’ten başlatırken herhangi bir parametre almayan Program Never.

Tip İmzaları #

Elm’de değerler bir ad, eşittir işareti ve ardından gerçek değer yazılarak bildirilir:

someValue = 42

Fonksiyonlar, argüman olarak bir değer veya değerlerin eklenmesiyle birlikte değerlerdir. Genellikle aşağıdaki gibi yazılırlar:

double n = n * 2

Elm’deki her değerin bir türü vardır. Yukarıdaki değerlerin türleri, nasıl kullanıldıklarına bağlı olarak derleyici tarafından çıkarılacaktır. Ancak, herhangi bir üst düzey değerin türünü her zaman açıkça belirtmek ve bunu yapmak için aşağıdaki gibi bir tür imzası yazmak en iyi uygulamadır:

someValue : Int
someValue = 
    42

someOtherValue : Float
someOtherValue =
    42

Gördüğümüz gibi, ‘42’ ya bir ‘Int’ ya da bir ‘Float’ olarak tanımlanabilir. Bu sezgisel olarak mantıklıdır, ancak daha fazla bilgi için Tür Değişkenleri bölümüne bakın.

Tür imzaları, işlevlerle birlikte kullanıldığında özellikle değerlidir. İşte öncekinden ikiye katlama işlevi:

double : Int -> Int
double n =
    n * 2

Bu sefer imzada bir ->, bir ok var ve imzayı “int to int” veya “bir tamsayı alır ve bir tamsayı döndürür” olarak telaffuz ederiz. “->”, “double"a argüman olarak “Int” değeri vererek “double"ın bir “Int” döndüreceğini belirtir. Bu nedenle, bir tamsayıdan bir tamsayı alır:

> double
<function> : Int -> Int

> double 3
6 : Int

Temel tipler #

“elm-repl"de, değerini ve türetilmiş türünü almak için bir kod parçası yazın. Var olan çeşitli türler hakkında bilgi edinmek için aşağıdakileri deneyin:

> 42
42 : number

> 1.987
1.987 : Float

> 42 / 2
21 : Float

> 42 % 2
0 : Int

> 'e'
'e' : Char

> "e"
"e" : String

> "Hello Friend"
"Hello Friend" : String

> ['w', 'o', 'a', 'h']
['w', 'o', 'a', 'h'] : List Char

> ("hey", 42.42, ['n', 'o'])
("hey", 42.42, ['n', 'o']) : ( String, Float, List Char )

> (1, 2.1, 3, 4.3, 'c')
(1,2.1,3,4.3,'c') : ( number, Float, number', Float, Char )

> {}
{} : {}

> { hey = "Hi", someNumber = 43 }
{ hey = "Hi", someNumber = 43 } : { hey : String, someNumber : number }

> ()
() : ()

{} boş Kayıt türüdür ve () boş Tuple türüdür. İkincisi genellikle tembel değerlendirme amacıyla kullanılır. https://www.wikiod.com/tr/elm/fonksiyonlar-ve-ksmi-uygulama içindeki ilgili örneğe bakın.

“Sayı"nın nasıl büyük harfle yazılmadığına dikkat edin. Bu, bunun bir Tür Değişkeni olduğunu gösterir ve ayrıca belirli bir “sayı” kelimesi, “Int” veya “Kayan” olabilen bir Özel Tür Değişkeni anlamına gelir (bkz. daha fazla). Ancak türler her zaman büyük harflidir, örneğin ‘Char’, ‘Float’, ‘List String’ vb.

Özel Tip Değişkenler #

Elm, derleyici için özel bir anlamı olan aşağıdaki özel tip değişkenleri tanımlar:

  • ‘karşılaştırılabilir’: ‘Int’, ‘Float’, ‘Char’, ‘String’ ve bunların demetlerinden oluşur. Bu, < ve > operatörlerinin kullanımına izin verir.

Örnek: Bir listedeki en küçük ve en büyük öğeleri bulmak için bir işlev tanımlayabilirsiniz (“kapsam”). Ne tür bir imza yazacağınızı düşünüyorsunuz. Bir yandan, extentInt : List Int -> Maybe (Int, Int) ve extentChar : List Char -> Maybe (Char, Char) ve diğerini Float ve String için yazabilirsiniz. Bunların uygulanması aynı olacaktır:

  extentInt list =
    let
      helper x (minimum, maximum) = 
        ((min minimum x), (max maximum x))
    in 
      case list of 
        [] ->
          Nothing
        x :: xs ->
          Just <| List.foldr helper (x, x) xs

 You might be tempted to simply write `extent : List a -> Maybe (a, a)`, but the compiler will not let you do this, because the functions `min` and `max` are not defined for these types (NB: these are just simple wrappers around the `<` operator mentioned above). You can solve this by defining `extent : List comparable -> Maybe (comparable, comparable)`. This allows your solution to be *polymorphic*, which just means that it will work for more than one type.
  • sayı: Int ve Floattan oluşur. Bölme dışında aritmetik operatörlerin kullanımına izin verir. Daha sonra örneğin `sum : Liste numarası -> sayı’ tanımlayabilir ve hem ints hem de float için çalışmasını sağlayabilirsiniz.
  • ’eklenebilir’: ‘Dize’, ‘Liste’den oluşur. ++ operatörünün kullanımına izin verir.
  • compappend: Bu bazen görünür, ancak derleyicinin uygulama detayıdır. Şu anda bu, kendi programlarınızda kullanılamaz, ancak bazen bahsedilir.

Şuna benzer bir tür ek açıklamasında: ‘sayı -> sayı -> sayı’ bunların hepsinin aynı türe atıfta bulunduğunu unutmayın, bu nedenle ‘Int -> Float -> Int’ içinden geçmek bir tür hatası olur. Bunu tür değişkenine bir son ek ekleyerek çözebilirsiniz: number -> number' -> number'' daha sonra iyi derlenir.

Bunlar için resmi bir isim yoktur, bazen şöyle denir:

  • Özel Tip Değişkenler
  • Typeclass benzeri Tür Değişkenleri
  • Sözde tip sınıflar

Bunun nedeni, Haskell’in Tip Sınıfları gibi çalışması, ancak kullanıcının bunları tanımlama yeteneği olmamasıdır.

Tür Değişkenleri #

Tür değişkenleri, tür imzalarında büyük harfle kullanılmayan adlardır. “Int” ve “String” gibi büyük harfle yazılan karşılıklarının aksine, tek bir türü değil, herhangi bir türü temsil ederler. Herhangi bir tür veya tür üzerinde çalışabilen genel işlevler yazmak için kullanılırlar ve özellikle ‘Liste’ veya ‘Dict’ gibi kaplar üzerine işlemler yazmak için kullanışlıdırlar. Örneğin, “List.reverse” işlevi aşağıdaki imzaya sahiptir:

reverse : List a -> List a

Bu, herhangi bir tür değer listesi üzerinde çalışabileceği anlamına gelir, bu nedenle ‘List Int’, ‘List (List String)’, hem bunlar hem de diğerleri aynı şekilde ’tersine çevrilebilir’. Bu nedenle, “a” herhangi bir türün yerine geçebilen bir tür değişkenidir.

“reverse” işlevi, “sayı” gibi bir avuç özel türde değişken adı dışında, tür imzasında herhangi bir büyük harf olmayan değişken adını kullanabilirdi (daha fazla bilgi için bununla ilgili örneğe bakın):

reverse : List lol -> List lol

reverse : List wakaFlaka -> List wakaFlaka

Tür değişkenlerinin adları, yalnızca tek bir imza içinde farklı türde değişkenler olduğunda, listelerde ‘harita’ işleviyle örneklenen şekilde anlamlı hale gelir:

map : (a -> b) -> List a -> List b

“harita”, herhangi bir “a” türünden herhangi bir “b” türüne kadar bazı işlevleri, “a” türünden öğeler içeren bir listeyle birlikte alır ve uygulayarak elde ettiği “b” türündeki öğelerin bir listesini döndürür. Listenin her elemanına verilen fonksiyon.

Bunu daha iyi görebilmek için imzayı somutlaştıralım:

plusOne : Int -> Int
plusOne x = 
    x + 1

> List.map plusOne
<function> : List Int -> List Int

Gördüğümüz gibi bu durumda hem a = Int hem de b = Int. Ancak, “map”, “map : (a -> a) -> List a -> List a” gibi bir tür imzasına sahip olsaydı, o zaman yalnızca tek bir tür üzerinde çalışan işlevler üzerinde çalışırdı ve “harita” işlevini kullanarak bir listenin türünü asla değiştiremezsiniz. Ancak “harita"nın tür imzası “a” ve “b” gibi birden çok farklı türde değişkene sahip olduğundan, bir listenin türünü değiştirmek için “harita"yı kullanabiliriz:

isOdd : Int -> Bool
isOdd x =
    x % 2 /= 0

> List.map isOdd
<function> : List Int -> List Bool

Bu durumda, ‘a = Int’ ve ‘b = Bool’. Bu nedenle, farklı tipler alıp döndürebilen fonksiyonları kullanabilmek için farklı tip değişkenler kullanmanız gerekir.

Tür Takma Adları #

Bazen bir türe daha açıklayıcı bir ad vermek isteriz. Diyelim ki uygulamamızın kullanıcıları temsil eden bir veri türü var:

{ name : String, age : Int, email : String }

Ve kullanıcılar üzerindeki fonksiyonlarımız aşağıdaki satırlar boyunca tip imzalarına sahiptir:

prettyPrintUser : { name : String, age : Int, email : String } -> String

Bu, bir kullanıcı için daha büyük bir kayıt türüyle oldukça hantal hale gelebilir, bu nedenle boyutu azaltmak ve bu veri yapısına daha anlamlı bir ad vermek için bir tür takma adı kullanalım:

type alias User =
    { name: String
    , age : Int
    , email : String
    }


prettyPrintUser : User -> String

Tür takma adları, bir uygulama için bir model tanımlamayı ve kullanmayı çok daha temiz hale getirir:

type alias Model =
    { count : Int
    , lastEditMade : Time
    }

‘Tür takma adı’ kullanmak, kelimenin tam anlamıyla bir türe verdiğiniz adla takma ad verir. Yukarıdaki “Model” türünü kullanmak, “{ count : Int, lastEditMade : Time }” kullanmakla tamamen aynıdır. Takma adların temel türlerden nasıl farklı olmadığını gösteren bir örnek:

type alias Bugatti = Int

type alias Fugazi = Int

unstoppableForceImmovableObject : Bugatti -> Fugazi -> Int
unstoppableForceImmovableObject bug fug =
    bug + fug

> unstoppableForceImmovableObject 09 87
96 : Int

Bir kayıt türü için bir tür diğer adı, bildirim sırasına göre her alan için bir bağımsız değişken içeren bir yapıcı işlevi tanımlar.

type alias Point = { x : Int, y : Int }

Point 3 7
{ x = 3, y = 7 } : Point

type alias Person = { last : String, middle : String, first : String }

Person "McNameface" "M" "Namey"
{ last = "McNameface", middle = "M", first = "Namey" } : Person

Her kayıt türü diğer adının, uyumlu bir tür için bile kendi alan sırası vardır.

type alias Person = { last : String, middle : String, first : String }
type alias Person2 = { first : String, last : String, middle : String }

Person2 "Theodore" "Roosevelt" "-"
{ first = "Theodore", last = "Roosevelt", middle = "-" } : Person2

a = [ Person "Last" "Middle" "First", Person2 "First" "Last" "Middle" ]
[{ last = "Last", middle = "Middle", first = "First" },{ first = "First", last = "Last", middle = "Middle" }] : List Person2
