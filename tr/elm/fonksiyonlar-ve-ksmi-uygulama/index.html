Sözdizimi #

  • – argümansız bir fonksiyon tanımlamak, basitçe bir değer tanımlamakla aynı görünüyor dil = “Karaağaç”
  • – bağımsız değişkeni olmayan bir işlevi adını belirterek çağırmak dil
  • – parametreler boşluklarla ayrılır ve işlevin adını takip eder x y = x + y ekleyin
  • – aynı şekilde bir işlevi çağırın 5 2 ekle
  • – sadece bazı parametrelerini sağlayarak bir işlevi kısmen uygular artış = 1 ekle
  • – soldaki ifadeyi sağdaki fonksiyona geçirmek için |> operatörünü kullanın on = 9 |> artış
  • – <| operatör sağdaki ifadeyi soldaki fonksiyona geçirir artış <| 5 4 ekle
  • – » operatörüyle birlikte iki işlevi zincirleme/oluşturma tersYell = String.reverse » String.toUpper
  • – « ters yönde aynı şekilde çalışır geriye doğruYell = String.toUpper « String.reverse
  • – parantez içinde alfasayısal olmayan bir ada sahip bir işlev yeni bir operatör oluşturur (#) x y = x * y on = 5 # 2
  • – herhangi bir infix operatörü, onu parantez içine aldığınızda normal bir fonksiyon haline gelir on = (+) 5 5
  • – isteğe bağlı tür açıklamaları, işlev bildirimlerinin üzerinde görünür isTen : Int -> Bool isTen n = eğer n == 10 ise True else False

Kısmi Uygulama #

Kısmi uygulama, sahip olduğundan daha az argümana sahip bir fonksiyonu çağırmak ve sonucu başka bir fonksiyon olarak kaydetmek (geri kalan argümanları bekler) anlamına gelir.

multiplyBy: Int -> Int -> Int    
multiplyBy x y =
    x * y


multiplyByTwo : Int -> Int  -- one Int has disappeared! we now know what x is.
multiplyByTwo =
    multiplyBy 2


> multiplyByTwo 2
4 : Int

> multiplyByTwo 4
8 : Int

Akademik bir yan not olarak, Elm bunu perde arkasındaki körlük sayesinde yapabilir.

Yerel değişkenler #

Bir fonksiyon içinde yerel değişkenleri tanımlamak mümkündür.

  • kod tekrarını azaltın
  • alt ifadelere isim verin
  • geçirilen argümanların miktarını azaltın.

Bunun yapısı let ... in ... şeklindedir.

bigNumbers =
    let
        allNumbers =
            [1..100]

        isBig number =
            number > 95
    in
        List.filter isBig allNumbers

> bigNumbers
[96,97,98,99,100] : List number

> allNumbers
-- error, doesn't know what allNumbers is!

’let’in ilk bölümündeki tanımların sırası önemli değil!

outOfOrder =
    let
        x =
            y + 1  -- the compiler can handle this

        y =
            100
    in
        x + y

> outOfOrder
201 : number

Genel Bakış #

Elm’deki işlev uygulaması sözdizimi parantez veya virgül kullanmaz ve bunun yerine boşluk duyarlıdır.

Bir işlevi tanımlamak için, adını ‘multiplyByTwo’ ve ‘x’ argümanlarını belirtin, eşittir işaretinden sonraki tüm işlemler ‘=’ bir işlevden döndürülür.

multiplyByTwo x =
    x * 2

Bir işlevi çağırmak için adını ve bağımsız değişkenlerini belirtin:

multiplyByTwo 2  -- 4

multiplyByTwo(2) gibi bir sözdiziminin gerekli olmadığını unutmayın (derleyici şikayet etmese bile). Parantezler yalnızca önceliği çözmeye yarar:

> multiplyByTwo multiplyByTwo 2
-- error, thinks it's getting two arguments, but it only needs one

> multiplyByTwo (multiplyByTwo 2)
4 : number

> multiplyByTwo 2 + 2
6 : number
-- same as (multiplyByTwo 2) + 2

> multiplyByTwo (2 + 2)
8 : number

Lambda ifadeleri #

Elm, lambda ifadeleri veya anonim işlevler için özel bir sözdizimine sahiptir:

\arguments -> returnedValue

Örneğin, ‘List.filter’da görüldüğü gibi:

> List.filter (\num -> num > 1) [1,2,3]
[2,3] : List number

Daha derinde, lambda ifadesinin başlangıcını işaretlemek için geriye doğru bir eğik çizgi “" kullanılır ve işlev gövdesinden bağımsız değişkenleri sınırlamak için “->” oku kullanılır. Daha fazla argüman varsa, bir boşlukla ayrılırlar:

normalFunction x y = x + y
-- is equivalent to
lambdaFunction = \x y -> x + y

> normalFunction 1 2
3 : number

> lambdaFunction 1 2
3 : number

Sıkı ve gecikmeli değerlendirme #

Karaağaçta, son argüman uygulandığında bir fonksiyonun değeri hesaplanır. Aşağıdaki örnekte, f 3 bağımsız değişkenle çağrıldığında veya son bağımsız değişkenle birlikte f’nin curried formu uygulandığında “log"dan tanı yazdırılacaktır.

import String
import Debug exposing (log)

f a b c = String.join "," (log "Diagnostic" [a,b,c]) -- <function> : String -> String -> String -> String

f2 = f "a1" "b2" -- <function> : String -> String

f "A" "B" "C"
-- Diagnostic: ["A","B","C"]
"A,B,C" : String

f2 "c3"
-- Diagnostic: ["a1","b2","c3"]
"a1,b2,c3" : String

Bazen bir işlevin hemen uygulanmasını engellemek isteyeceksiniz. Karaağaçta tipik bir kullanım, işlevlerin ne zaman kontrol edilmesi için bir soyutlama sağlayan Lazy.lazy şeklindedir. uygulanır.

lazy : (() -> a) -> Lazy a

Tembel hesaplamalar, bir “()” veya “Birim” tipi argümanın bir işlevini alır. Birim türü, geleneksel olarak bir yer tutucu bağımsız değişkenin türüdür. Bir bağımsız değişken listesinde, karşılık gelen bağımsız değişken, değerin kullanılmadığını belirten “_” olarak belirtilir. karaağaçtaki birim değeri, kavramsal olarak boş bir demeti veya bir deliği temsil edebilen () özel sembolü ile belirtilir. C, Javascript ve işlev çağrıları için parantez kullanan diğer dillerdeki boş argüman listesine benzer, ancak sıradan bir değerdir.

Örneğimizde, f bir lambda ile hemen değerlendirilmekten korunabilir:

doit f = f () -- <function> : (() -> a) -> a
whatToDo = \_ -> f "a" "b" "c" -- <function> : a -> String
-- f is not evaluated yet

doit whatToDo
-- Diagnostic: ["a","b","c"]
"a,b,c" : String

İşlev değerlendirmesi, bir işlev kısmen uygulandığında ertelenir.

defer a f = \_ -> f a -- <function> : a -> (a -> b) -> c -> b

delayF = f "a" "b" |> defer "c" -- <function> : a -> String

doit delayF
-- Diagnostic: ["a","b","c"]
"a,b,c" : String

Karaağaç, değerlendirmeyi geciktirmek için kullanılamayan bir always işlevine sahiptir. karaağaç, işlev uygulamasının sonucunun kullanılıp kullanılmadığına ve ne zaman kullanıldığına bakılmaksızın tüm işlev bağımsız değişkenlerini değerlendirdiğinden, bir işlev uygulamasını “always” içine sarmak gecikmeye neden olmaz, çünkü “f”, “always” parametresi olarak tam olarak uygulanır.

alwaysF = always (f "a" "b" "c") -- <function> : a -> String
-- Diagnostic: ["a","b","c"] -- Evaluation wasn't delayed.

Infix operatörleri ve infix gösterimi #

Elm, özel infix operatörlerinin tanımlanmasına izin verir.

Infix operatörleri, bir fonksiyonun adı etrafında parantez kullanılarak tanımlanır.

Yapı Tuples 1 => True -- (1, True) için bu infix operatörü örneğini düşünün:

(=>) : a -> b -> ( a, b )
(=>) a b =
    ( a, b )

Elm’deki işlevlerin çoğu önek gösteriminde tanımlanır.

Önemli aksan karakteriyle çevrelenmiş işlev adından önceki ilk argümanı belirterek, infix gösterimini kullanarak herhangi bir işlevi uygulayın:

import List exposing (append)


append [1,1,2] [3,5,8]   -- [1,1,2,3,5,8]
[1,1,2] `append` [3,5,8] -- [1,1,2,3,5,8]
