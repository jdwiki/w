Uygulamalarınızı yapılandırmanın önerilen yolu ‘Elm Mimarisi’ olarak adlandırılır.

En basit program, güncellenebilecek tüm verileri depolayan bir “model” kaydından, programınızın bu verileri güncelleme yollarını tanımlayan bir “Msg” birleşim türünden, modeli alan bir “güncelleme” işlevinden ve bir “Mesaj” döndüren bir işlevden oluşur. yeni model ve bir model alan ve sayfanızın göstereceği HTML’yi döndüren bir işlev ‘görünüm’. Bir işlev bir “Msg” döndürdüğünde, Elm çalışma zamanı onu sayfayı güncellemek için kullanır.

Bayraklı Program #

“programWithFlags"ın “program"dan yalnızca bir farkı vardır.

JavaScript’ten başlatıldığında verileri kabul edebilir:

var root = document.body;
var user = { id: 1, name: "Bob" };
var app = Elm.Main.embed( root, user );

JavaScript’ten iletilen verilere Bayraklar denir.

Bu örnekte, kullanıcı bilgileriyle birlikte bir JavaScript Nesnesini Elm’e iletiyoruz, bayraklar için bir Tür Takma Adı belirtmek iyi bir uygulamadır.

type alias Flags =
    { id: Int
    , name: String
    }

Bayraklar “init” işlevine iletilerek başlangıç ​​durumu oluşturulur:

init : Flags -> ( Model, Cmd Msg )
init flags =
    let
        { id, name } =
            flags
    in
        ( Model id name, Cmd.none )

Tür imzasından farkı fark edebilirsiniz:

programWithFlags :
    { init : flags -> ( model, Cmd msg )          -- init now accepts flags
    , update : msg -> model -> ( model, Cmd msg )
    , subscriptions : model -> Sub msg
    , view : model -> Html msg
    }
    -> Program flags

Başlatma kodu, yalnızca “init” işlevi farklı olduğu için hemen hemen aynı görünüyor.

main =
    programWithFlags
        { init = init
        , update = update
        , view = view
        , subscriptions = subscriptions
        }

Başlangıç ​​programı #

Html, çoğunlukla öğrenme amaçlı “beginnerProgram"a sahiptir.

“beginnerProgram”, Abonelikleri işleme veya Komutları çalıştırma yeteneğine sahip değildir.

Yalnızca DOM Olaylarından kullanıcı girdisini işleyebilir.

Yalnızca “modeli” oluşturmak için bir “görünüm” ve durum değişikliklerini işlemek için bir “güncelleme” işlevi gerektirir.

Örnek #

“BeginnerProgram"ın bu minimal örneğini düşünün.

Bu örnekteki “model”, tek bir “Int” değerinden oluşur.

‘güncelleme’ işlevi, ‘model’de saklanan ‘Int’ değerini artıran yalnızca bir dala sahiptir.

‘Görünüm’ modeli oluşturur ve tıklama DOM Olayı’nı ekler.

[Initialize and build][2] içinde örneğin nasıl oluşturulacağını görün

import Html exposing (Html, button, text)
import Html exposing (beginnerProgram)
import Html.Events exposing (onClick)


main : Program Never
main =
    beginnerProgram { model = 0, view = view, update = update }


-- UPDATE


type Msg
    = Increment

update : Msg -> Int -> Int
update msg model =
    case msg of
        Increment ->
            model + 1


-- VIEW


view : Int -> Html Msg
view model =
    button [ onClick Increment ] [ text ("Increment: " ++ (toString model)) ]

[2]: https://www.wikiod.com/tr/elm/elm-dilini-kullanmaya-baslama#Başlatın ve oluşturun

Program #

Uygulamanız başlatma için herhangi bir harici veri gerektirmediğinde ‘program’ iyi bir seçimdir.

HTTP iletişimi veya JavaScript ile birlikte çalışma gibi G/Ç işlemleri için çok daha fazla fırsat sağlayan Abonelikleri ve Komutları işleme kapasitesine sahiptir.

Modelle birlikte başlatma Komutlarını döndürmek için başlangıç ​​durumu gereklidir.

“Program"ın başlatılması, “model”, “görünüm” ve “güncelleme” ile birlikte “aboneliklerin” sağlanmasını gerektirecektir.

Tür tanımına bakın:

program :
    { init : ( model, Cmd msg )
    , update : msg -> model -> ( model, Cmd msg )
    , subscriptions : model -> Sub msg
    , view : model -> Html msg
    }
    -> Program Never

Örnek #

[Abonelikleri][2] nasıl kullanabileceğinizi göstermenin en basit yolu, JavaScript ile basit bir [Port][3] iletişimi kurmaktır.

Initialize and build / [HTML’ye Gömme][4] içinde örneğin nasıl oluşturulacağını görün

port module Main exposing (..)

import Html exposing (Html, text)
import Html exposing (program)


main : Program Never
main =
    program
        { init = init
        , view = view
        , update = update
        , subscriptions = subscriptions
        }


port input : (Int -> msg) -> Sub msg


-- MODEL


type alias Model =
    Int


init : ( Model, Cmd msg )
init =
    ( 0, Cmd.none )


-- UPDATE


type Msg = Incoming Int


update : Msg -> Model -> ( Model, Cmd msg )
update msg model =
    case msg of
        Incoming x ->
          ( x, Cmd.none )


-- SUBSCRIPTIONS


subscriptions : Model -> Sub Msg
subscriptions model =
    input Incoming


-- VIEW


view : Model -> Html msg
view model =
    text (toString model)

<!DOCTYPE html>
<html>
    <head>
        <script src='elm.js'></script>
</head>
    <body>
    <div id='app'></div>
    <script>var app = Elm.Main.embed(document.getElementById('app'));</script>
    <button onclick='app.ports.input.send(1);'>send</button>
</body>
</html>

Tek yönlü ebeveyn-çocuk iletişimi #

Örnek, bileşen bileşimini ve ebeveynden çocuklara geçen tek yönlü mesajı gösterir.

Bileşen bileşimi, “Html.App.map” ile Mesaj etiketlemeye dayanır

son sürüm

‘0.18.0’ ‘HTML.App’ içinde daraltıldı ‘HTML’

Bileşen bileşimi, “Html.map” ile Mesaj etiketlemeye dayanır

son sürüm

Örnek #

[Initialise and build][2] içinde örneğin nasıl oluşturulacağını görün

module Main exposing (..)

import Html exposing (text, div, button, Html)
import Html.Events exposing (onClick)
import Html.App exposing (beginnerProgram)


main =
    beginnerProgram
        { view = view
        , model = init
        , update = update
        }

{- In v0.18.0 HTML.App was collapsed into HTML
   Use Html.map instead of Html.App.map
-}
view : Model -> Html Msg
view model =
    div []
        [ Html.App.map FirstCounterMsg (counterView model.firstCounter)
        , Html.App.map SecondCounterMsg (counterView model.secondCounter)
        , button [ onClick ResetAll ] [ text "Reset counters" ]
        ]


type alias Model =
    { firstCounter : CounterModel
    , secondCounter : CounterModel
    }


init : Model
init =
    { firstCounter = 0
    , secondCounter = 0
    }


type Msg
    = FirstCounterMsg CounterMsg
    | SecondCounterMsg CounterMsg
    | ResetAll


update : Msg -> Model -> Model
update msg model =
    case msg of
        FirstCounterMsg childMsg ->
            { model | firstCounter = counterUpdate childMsg model.firstCounter }

        SecondCounterMsg childMsg ->
            { model | secondCounter = counterUpdate childMsg model.secondCounter }

        ResetAll ->
            { model
                | firstCounter = counterUpdate Reset model.firstCounter
                , secondCounter = counterUpdate Reset model.secondCounter
            }


type alias CounterModel =
    Int


counterView : CounterModel -> Html CounterMsg
counterView model =
    div []
        [ button [ onClick Decrement ] [ text "-" ]
        , text (toString model)
        , button [ onClick Increment ] [ text "+" ]
        ]


type CounterMsg
    = Increment
    | Decrement
    | Reset


counterUpdate : CounterMsg -> CounterModel -> CounterModel
counterUpdate msg model =
    case msg of
        Increment ->
            model + 1

        Decrement ->
            model - 1

        Reset ->
            0

[2]: https://www.wikiod.com/tr/elm/elm-dilini-kullanmaya-baslama#Başlatın ve oluşturun

Html.App.map ile mesaj etiketleme #

Bileşenler, yayılan DOM Olaylarından sonra gönderilen kendi Mesajlarını tanımlar, örn. Ebeveyn-çocuk iletişimi‘den “CounterMsg”

type CounterMsg
    = Increment
    | Decrement
    | Reset

Bu bileşenin görünümü ‘CounterMsg’ tipinde mesajlar gönderecektir, bu nedenle görünüm tipi imzası ‘Html CounterMsg’dir.

‘counterView’ öğesini ana bileşenin görünümünde yeniden kullanabilmek için, her ‘CounterMsg’ mesajını ebeveynin ‘Msg’sinden geçirmemiz gerekir.

Bu tekniğe mesaj etiketleme denir.

Üst bileşen, alt mesajların iletilmesi için mesajları tanımlamalıdır:

type Msg
    = FirstCounterMsg CounterMsg
    | SecondCounterMsg CounterMsg
    | ResetAll

FirstCounterMsg Artış etiketli bir mesajdır.

Etiketli mesajları göndermek üzere bir ‘counterView’ elde etmek için ‘Html.App.map’ işlevini kullanmalıyız:

Html.map FirstCounterMsg (counterView model.firstCounter)
son sürüm

_HTML.App paketi [daraltıldı][2],v0.18.0da HTML paketine eklendi.

Etiketli mesajları göndermek üzere bir ‘counterView’ elde etmek için ‘Html.map’ işlevini kullanmalıyız:

Html.map FirstCounterMsg (counterView model.firstCounter)
son sürüm

Bu, Html CounterMsg -> Html Msg tür imzasını değiştirir, böylece sayacı üst görünüm içinde kullanmak ve durum güncellemelerini ebeveynin güncelleme işleviyle işlemek mümkündür.
