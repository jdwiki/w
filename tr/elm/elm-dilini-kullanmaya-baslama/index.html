Kurulum #

Elm ile geliştirmeye başlamak için elm-platform adlı bir dizi araç yüklemeniz gerekir.

Şunları içerir: elm-make, elm-reactor, elm-repl ve elm-package.

Bu araçların tümüne CLI üzerinden erişilebilir, yani bunları terminalinizden kullanabilirsiniz.

Elm’i yüklemek için aşağıdaki yöntemlerden birini seçin:

Yükleyiciyi kullanma #

Yükleyiciyi resmi web sitesinden indirin ve kurulum sihirbazını takip edin.

npm’yi kullanma #

Elm platformunu kurmak için Düğüm Paket Yöneticisi’ni kullanabilirsiniz.

Küresel kurulum:

$ npm install elm -g

Yerel kurulum:

$ npm install elm

Yerel olarak kurulmuş Elm platform araçlarına şu yollarla erişilebilir:

$ ./node_modules/.bin/elm-repl  # launch elm-repl from local node_modules/

Homebrew kullanma #

$ brew install elm

Karaağaç kullanımlı sürümler arasında geçiş yapın #

karaağaç kullanımı yükleyin

$ npm install -g elm-use

Daha eski veya daha yeni bir karaağaç sürümüne geçin

$ elm-use 0.18  // or whatever version you want to use

Daha fazla okuma #

İlk projenizi nasıl [Başlatacağınızı ve oluşturacağınızı][8] öğrenin.

[8]: https://www.wikiod.com/tr/elm/elm-dilini-kullanmaya-baslama#Başlatın ve oluşturun

Başlatın ve oluşturun #

Bilgisayarınızda Elm platformunun kurulu olması gerekir, aşağıdaki öğretici, terminale aşina olduğunuz varsayımıyla yazılmıştır.

Başlatma #

Bir klasör oluşturun ve terminalinizle ona gidin:

$ mkdir elm-app
$ cd elm-app/

Elm projesini başlatın ve temel bağımlılıkları kurun:

$ elm-package install -y

Projenizde elm-package.json ve elm-stuff klasörü görünmelidir.

Main.elm uygulamanız için giriş noktası oluşturun ve Merhaba Dünya örneğini buraya yapıştırın.

Projeyi inşa etmek #

İlk projenizi oluşturmak için şunu çalıştırın:

$ elm-make Main.elm

Bu, JavaScript’te derlenmiş ve HTML’de sıralanmış ‘Main.elm’ dosyası (ve tüm bağımlılıklar) ile ‘index.html’ üretecektir. Tarayıcınızda açmayı deneyin!

Bu, ``Html modülünü bulamıyorum hatasıyla başarısız olursa, Elm’in en son sürümünü kullanmıyorsunuz demektir. Elm’i yükselterek ve ilk adımı tekrarlayarak veya aşağıdaki komutla sorunu çözebilirsiniz:

$ elm-package install elm-lang/html -y

Kendi ‘index.html’ dosyanız varsa (örneğin, bağlantı noktalarıyla çalışırken), Elm dosyalarınızı bir JavaScript dosyası olarak da derleyebilirsiniz:

$ elm-make Main.elm --output=elm.js

Stil Kılavuzu ve karaağaç biçimi #

Resmi stil kılavuzu ana sayfada bulunur ve genellikle şunlar için geçerlidir:

  • okunabilirlik (kompaktlık yerine)
  • modifikasyon kolaylığı
  • temiz farklar

Bu, örneğin şu anlama gelir:

homeDirectory : String
homeDirectory =
  "/root/files"


evaluate : Boolean -> Bool
evaluate boolean =
  case boolean of
    Literal bool ->
        bool

    Not b ->
        not (evaluate b)

    And b b' ->
        evaluate b && evaluate b'

    Or b b' ->
        evaluate b || evaluate b'

daha iyi olarak kabul edilir:

homeDirectory = "/root/files"

eval boolean = case boolean of
    Literal bool -> bool
    Not b        -> not (eval b)
    And b b'     -> eval b && eval b'
    Or b b'      -> eval b || eval b'

elm-format aracı, kaynak kodunuzu sizin için otomatik olarak (genellikle kaydetme sırasında), Go dilinin gofmt’sine benzer şekilde biçimlendirerek yardımcı olur. . Yine, temel değer, tutarlı bir stile sahip olmak ve sekmelere karşı boşluklar veya girinti uzunluğu gibi çeşitli sorunlar hakkında argümanları ve alevli savaşları kaydetmektir.

Github deposunda talimatları izleyerek “elm-format"ı yükleyebilirsiniz. biçim). Ardından, Elm dosyalarını otomatik olarak biçimlendirmek için düzenleyicinizi yapılandırın veya “elm-format FILE_OR_DIR –yes” komutunu manuel olarak çalıştırın.

HTML’ye gömme #

Elm kodunu mevcut bir HTML sayfasına eklemek için üç olasılık vardır.

Gövde etiketine yerleştir #

Merhaba Dünya örneğini elm.js dosyasında derlediğinizi varsayarsak, Elm’in <body> etiketini şu şekilde devralmasına izin verebilirsiniz:

<!DOCTYPE html>
<html>
    <body>
        <script src="elm.js"></script>
        <script>
          Elm.Main.fullscreen()
        </script>
    </body>
</html>

UYARI: Bazen bazı krom uzantıları “” ile karışarak uygulamanızın üretimde kesintiye uğramasına neden olabilir. Her zaman belirli bir div’e gömmeniz önerilir. Daha fazla bilgi buradan.

Div’e (veya başka bir DOM düğümüne) göm #

Alternatif olarak, somut HTML öğesi sağlayarak, Elm kodu o belirli sayfa öğesinde çalıştırılabilir:

<!DOCTYPE html>
<html>
    <head>
        <title>Hello World</title>
    </head>
    <body>
        <div id='app'></div>
        <script src="elm.js"></script>
        <script>
            Elm.Main.embed(document.getElementById('app'))
        </script>
    </body>
</html>

Web çalışanı olarak göm (UI yok) #

Elm kodu ayrıca bir çalışan olarak başlatılabilir ve portlar aracılığıyla iletişim kurabilir:

<!DOCTYPE html>
<html>
    <head>
        <title>Hello Worker</title>
    </head>
    <body>
        <script src="elm.js"></script>
        <script>
            var app = Elm.Main.worker();
            app.ports.fromElmToJS.subscribe(function(world) {
                console.log(world)
            });
            app.ports.fromJSToElm.send('hello');
        </script>
    </body>
</html>

TEKRAR #

Elm hakkında bilgi edinmenin iyi bir yolu, REPL’de (Okuma-Değerlendirme-Baskı Döngüsü) bazı ifadeler yazmayı denemektir. elm-app klasörünüzde (Initialize and build aşamasında oluşturduğunuz) bir konsol açın ve aşağıdakileri deneyin:

$ elm repl
---- elm-repl 0.17.1 -----------------------------------------------------------
 :help for help, :exit to exit, more at <https://github.com/elm-lang/elm-repl>
--------------------------------------------------------------------------------
> 2 + 2
4 : number
> \x -> x
<function> : a -> a
> (\x -> x + x)
<function> : number -> number
> (\x -> x + x) 2
4 : number
>

“elm-repl” aslında oldukça güçlü bir araçtır. Diyelim ki “elm-app” klasörünüzün içinde aşağıdaki kodla bir “Test.elm” dosyası oluşturdunuz:

module Test exposing (..)


a = 1


b = "Hello"

Şimdi, REPL’nize (açık kalan) geri dönün ve şunu yazın:

import Test exposing (..)
> a
1 : number
> b
"Hello" : String
>

Daha da etkileyici, eğer Test.elm dosyanıza yeni bir tanım eklerseniz, örneğin:

s = """
Hello,
Goodbye.
"""

Dosyanızı kaydedin, bir kez daha REPL’nize geri dönün ve tekrar `Test’i içe aktarmadan yeni tanım hemen kullanılabilir:

> s
"\nHello,\nGoodbye.\n" : String
>

Birçok satıra yayılan ifadeler yazmak istediğinizde gerçekten kullanışlıdır. Az önce tanımladığınız fonksiyonları hızlı bir şekilde test etmek de çok faydalıdır. Dosyanıza şunları ekleyin:

f x =
  x + x * x

Kaydedin ve REPL’ye geri dönün:

> f
<function> : number -> number
> f 2
6 : number
> f 4
20 : number
>

İçe aktardığınız bir dosyayı her değiştirdiğinizde ve kaydettiğinizde ve REPL’ye geri dönüp herhangi bir şey yapmaya çalıştığınızda, tam dosya yeniden derlenir. Bu nedenle, kodunuzdaki herhangi bir hatayı size söyleyecektir. Bunu ekle:

c = 2 ++ 2

Bunu dene:

> 0
-- TYPE MISMATCH -------------------------------------------------- ././Test.elm

The left argument of (++) is causing a type mismatch.

22|     2 ++ 2
        ^
(++) is expecting the left argument to be a:

    appendable

But the left argument is:

    number

Hint: Only strings, text, and lists are appendable.


>

REPL’ye bu girişi bitirmek için, ’elm-repl’nin, ’elm package install’ ile kurduğunuz paketleri de bildiğini ekleyelim. Örneğin:

> import Html.App
> Html.App.beginnerProgram
<function>
    : { model : a, update : b -> a -> a, view : a -> Html.Html b }
      -> Platform.Program Basics.Never
>

1: https://www.wikiod.com/tr/elm/elm-languagei-kullanmaya-baslama#Başlatın ve oluşturun

Yerel Yapı Sunucusu (Elm Reaktörü) #

Elm Reactor, uygulamanızın prototipini oluşturmak için gerekli araçtır.

Http.App.programWithFlags veya Ports kullanıyorsanız, “Main.elm” dosyasını Elm Reactor ile derleyemeyeceğinizi lütfen unutmayın.

Elm-reactor’ı bir proje dizininde çalıştırmak, her ayrı bileşeni derlemenize izin veren bir proje gezgini içeren bir web sunucusunu başlatır.

Kodunuzda yaptığınız tüm değişiklikler, sayfayı yeniden yüklediğinizde güncellenir.

$ elm-reactor                     # launch elm-reactor on localhost:8000
$ elm-reactor -a=0.0.0.0 -p=3000  # launch elm-reactor on 0.0.0.0:3000
