Kesinlikle Yazılan GridView #

Asp.net 4.5 ile başlayarak web denetimleri, IntelliSense desteği ve derleme zamanı hataları almak için kesin olarak yazılan bağlamadan yararlanabilir.

Modelinizi tutan bir sınıf oluşturun:

public class Album
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string Artist { get; set; }
}

Sayfanızda GridView kontrolünü tanımlayın:

<asp:GridView ID="Grid" runat="server" AutoGenerateColumns="false" ItemType="YourNamespace.Album">
    <Columns>
        <asp:TemplateField HeaderText="Id">
            <ItemTemplate>
                <asp:Label ID="lblName" runat="server" Text="<%# Item.Id %>"></asp:Label>
            </ItemTemplate>
        </asp:TemplateField>
        <asp:TemplateField HeaderText="Name">
            <ItemTemplate>
                <asp:Label ID="lblName" runat="server" Text="<%# Item.Name %>"></asp:Label>
            </ItemTemplate>
        </asp:TemplateField>
        <asp:TemplateField HeaderText="Artist">
            <ItemTemplate>
                <asp:Label ID="lblCity" runat="server" Text="<%# Item.Artist %>"></asp:Label>
            </ItemTemplate>
        </asp:TemplateField>
    </Columns>
</asp:GridView>

Verileri yükleyin ve bağlayın:

var albumList = new List<Album>
{
    new Album {Id = 1, Artist = "Icing (a Cake cover band)", Name = "Toppings Vol. 1"},
    new Album {Id = 2, Artist = "Fleetwood PC", Name = "Best of Windows"},
    new Album {Id = 3, Artist = "this.Bandnames", Name = "TH_ (Pronounced \"Thunderscore\")"},
};

Grid.DataSource = albumList;
Grid.DataBind();

Komut olayını işleme #

GridView’ler, komutların bir GridView satırından gönderilmesine izin verir. Bu, satıra özgü bilgileri bir olay işleyicisine komut bağımsız değişkenleri olarak iletmek için kullanışlıdır.

Bir komut olayına abone olmak için:

<asp:GridView ID="GridView1" ... OnRowCommand="GridView1_RowCommand">

Düğmeler, komutları yükseltmenin en yaygın yoludur. Ayrıca komut argümanlarını belirtmenin bir yolunu da desteklerler. Bu örnekte, argüman, satırın temsil ettiği öğenin bir “ID"sidir.

<TemplateField>
    <ItemTemplate>
        <asp:LinkButton ID="LinkButton1" runat="server"
                        CommandName="SampleCmd"
                        CommandArgument='<%# Eval("ID") %>'>
        </asp:LinkButton>
    </ItemTemplate>
</TemplateField>

Alternatif olarak, en yaygın komut kontrollerini sağlayan bir “CommandField” sütun şablonu kullanılabilir.

Olayın arkasındaki kodda ele alınması:

protected void GridView1_RowCommand(object source, GridViewCommandEventArgs e)
{
    if (e.CommandName == "SampleCmd")
    {
        var id = e.CommandArgument;
    }
}

Bu örnekte kullanılan “KomutAdı"nın isteğe bağlı olduğunu ve geliştiricinin bir seçimi olduğunu unutmayın. Bununla birlikte, GridView’ın kendisinin tanıdığı bir dizi önceden tanımlanmış ad vardır. Bu komutlar tetiklendiğinde ilgili olaylar başlatılır.

Komut Adı Etkinlikler Yükseltildi
İptal Sıra İptal EtmeDüzenle
Sil Sıra Silme, Sıra Silme
Düzenle Satır Düzenleme
Sayfa PageIndexChanging, PageIndexChanged
Seç SelectedIndexChanging, SelectedIndexChanged
Sırala Sıralama, Sıralama
Güncelle RowUpdating, RowUpdated

Bağlanma verileri #

Bir GridView’ı bağlamanın iki yolu vardır. Bunu “DataSource” özelliğini ayarlayarak ve “DataBind()“i çağırarak manuel olarak yapabilir veya “SqlDataSource” gibi bir DataSourceControl kullanabilirsiniz.

Manuel Ciltleme #

GridView’ınızı oluşturun:

<asp:GridView ID="gvColors" runat="server"></asp:GridView>

Önce GridView için kaynak verileri oluşturun veya alın. Ardından, verileri GridView’ın DataSource özelliğine atayın. Son olarak, DataBind() çağırın.

List<string> colors = new List<string>();
colors.Add("Red");
colors.Add("Green");
colors.Add("Blue");

gvColors.DataSource = colors;
gvColors.DataBind();

DataSourceControl #

DataSourceControl’ünüzü oluşturun:

<asp:SqlDataSource ID="sdsColors"
    runat="server"
    ConnectionString="<%$ MyConnectionString %>"
    SelectCommand="SELECT Color_Name FROM Colors">
</asp:SqlDataSource>

GridView’ınızı oluşturun ve “DataSourceID” özelliğini ayarlayın:

<asp:GridView ID="gvColors"
    runat="server"
    DataSourceID="sdsColors">
</asp:GridView>

Sütunlar #

Bir GridView içinde kullanılabilecek yedi farklı sütun türü vardır.

<asp:GridView ID="GridView1" runat="server">
    <Columns>
        ...
    </Columns>
</asp:GridView>

Sınır Alanı:

<asp:BoundField DataField="EmployeeID" HeaderText="Employee ID" />

Düğme Alanı:

<asp:ButtonField ButtonType="Button" HeaderText="Select Employee" Text="Select"/>

CheckBoxField:

<asp:CheckBoxField DataField="IsActive" HeaderText="Is Active" />

Komut Alanı:

<asp:CommandField ShowDeleteButton="true" 
    ShowEditButton="true" 
    ShowInsertButton="true" 
    ShowSelectButton="true" />

HyperLink Alanı:

<asp:HyperLinkField HeaderText="Employee Profile"
    DataNavigateUrlFields="EmployeeID"
    DataNavigateUrlFormatString="EmployeeProfile.aspx?EmployeeID={0}" />

Resim Alanı:

<asp:ImageField HeaderText="Photo"
    DataImageUrlField="EmployeeID"
    DataImageUrlFormatString="/images/{0}" />

Şablon Alanı:

<asp:TemplateField>
    <HeaderTemplate>
        Name
    </HeaderTemplate>
    <ItemTemplate>
        <asp:Label ID="lblEmployeeName"
            runat="server"
            Text='<&# Eval("EmployeeName") %>'></asp:Label>
    </ItemTemplate>
</asp:TemplateField>

Çağrı #

ObjectDataSource #

Bir ObjectDataSource kullanıyorsanız, hemen hemen her şey zaten sizin için işleniyor, sadece GridView’e ‘AllowPaging’i söyleyin ve ona bir ‘PageSize’ verin.

<asp:GridView ID="gvColors"
    runat="server"
    DataSourceID="sdsColors"
    AllowPaging="True"
    PageSize="5">
</asp:GridView>

<asp:SqlDataSource ID="sdsColors"
    runat="server"
    ConnectionString="<%$ MyConnectionString %>"
    SelectCommand="SELECT Color_ID, Color_Name FROM Colors">
</asp:SqlDataSource>

Manuel Ciltleme #

Manuel olarak bağlanıyorsanız, ‘PageIndexChanging’ olayını işlemeniz gerekir. Basitçe ‘DataSource’ ve ‘PageIndex’i ayarlayın ve GridView’ı yeniden bağlayın.

<asp:GridView ID="gvColors"
    runat="server"
    AllowPaging="True"
    PageSize="5"
    OnPageIndexChanging="gvColors_PageIndexChanging">
</asp:GridView>

C#

protected void gvColors_PageIndexChanging(object sender, GridViewPageEventArgs e)
{
    gvColors.DataSource = // Method to retrieve DataSource
    gvColors.PageIndex = e.NewPageIndex;
    gvColors.DataBind();
}

VB.NET

Protected Sub gvColors_PageIndexChanging(sender As Object, e As GridViewPageEventArgs)
{
    gvColors.DataSource = // Method to retrieve DataSource
    gvColors.PageIndex = e.NewPageIndex
    gvColors.DataBind()
}

Satır tıklamasında Gridview’i güncelle #

Görünümü ihtiyacımıza göre güncelleyebilirsek, ızgara görünümleri daha kullanışlıdır. Her satırda kilitleme/kilit açma özelliği olan bir görünüm düşünün. Şu şekilde yapılabilir:

Bir güncelleme paneli ekleyin:

<asp:UpdatePanel ID="UpdatePanel2" runat="server" UpdateMode="Conditional"> </asp:UpdatePanel>

UpdatePanel’inizin içine bir ContentTemplate ve Trigger ekleyin:

<asp:UpdatePanel ID="UpdatePanel2" runat="server" UpdateMode="Conditional"> 
    <ContentTemplate>
    </ContentTemplate>

    <Triggers>
    </Triggers>
</asp:UpdatePanel>

GridView’ınızı ContentTemplate içine ekleyin:

<ContentTemplate>
<asp:GridView ID="GridView1" runat="server">
          <Columns>
                <asp:TemplateField>
                    <ItemTemplate>
                        <asp:ImageButton ID="imgDownload" runat="server" OnClientClick="return confirm('Are you sure want to Lock/Unlock ?');"
                            CommandName="togglelock"
                            CommandArgument='<%#Container.DataItemIndex%>'/>
                        
                    </ItemTemplate>
                </asp:TemplateField>
            </Columns>

</ContentTemplate>

Burada GridView1’imize kilit düğmesi için bir sabit sütun veriyoruz. Dikkat edin, veri bağlama şimdiye kadar gerçekleşmedi.

DataBind Zamanı: (PageLoad’da)

using (SqlConnection con= new SqlConnection(connectionString)) 
{
            SqlCommand sqlCommand = new SqlCommand(" ... ", con);
            SqlDataReader reader = sqlCommand.ExecuteReader();
            GridView1.DataSource = reader;
            GridView1.DataBind();
}

Kilitle/Kilidi Aç görüntüsü, GridView’inizdeki belirli bir sütunun değerine göre farklı olacaktır. Tablonuzun “Durumu Kilitle” başlıklı bir öznitelik/sütun içerdiği bir durumu düşünün. Şimdi (1) bu sütunu DataBind’den hemen sonra ve sayfa oluşturmadan hemen önce gizlemek ve (2) bu gizli sütun değeri temelinde her satıra farklı görüntüler atamak istiyorsunuz, yani bir satır için Kilit Durumu 0 ise, “kilit” olarak atayın. jpg”, durum 1 ise ona “unlock.jpg” atayın. Bunu yapmak için, GridView’in OnRowDataBound seçeneğini kullanacağız, her satırı HTML sayfasına dönüştürmeden hemen önce GridView ile karışıyor.

<ContentTemplate>
<asp:GridView ID="GridView1" runat="server" OnRowDataBound="GridView1_RowDataBound"> ...

cs dosyasında

protected void GridView1_RowDataBound(object sender, GridViewRowEventArgs e)
    {

        if (e.Row.RowType == DataControlRowType.DataRow)
        {
            e.Row.Cells[8].Visible = false; //hiding the desired column which is column number 8 in this case
            GridView1.HeaderRow.Cells[8].Visible = false; //hiding its header
            ImageButton imgDownload = (ImageButton)e.Row.FindControl("imgDownload");
            string lstate = ((CheckBox)e.Row.Cells[8].Controls[0]).Checked.ToString();
            if (lstate == "True")
            { imgDownload.ImageUrl = "images/lock.png"; }
            else
            {
                imgDownload.ImageUrl = "images/unlock.png";
            }
        }
    }

Şimdi GridView istediğimiz gibi oluşturulacak, şimdi bu Lock/Unlock image butonu üzerinde butona tıklama olaylarını uygulayalım. Belirli bir satırda belirli bir işlemi gerçekleştirmek için o satıra bir komut verilmesi gerektiğini ve GridView’ın bize ‘OnRowCommand’ adlı aynı işlevi sağladığını anlayın.

<ContentTemplate>
<asp:GridView ID="GridView1" runat="server" OnRowDataBound="GridView1_RowDataBound" OnRowCommand="GridView1_RowCommand">
...
</ContentTemplate>

Bir “nesne gönderici” ve “GridViewCommandEventArgs e” alan cs dosyasında bir işlev yaratacaktır. e.CommandArgument ile komutu veren satırın indeksini alabiliriz. Burada dikkat edilmesi gereken nokta, bir satırın birden fazla butonu olabileceği ve cs kodunun o satırdan hangi butonun komutu verdiğini bilmesi gerektiğidir. Bu yüzden CommandName kullanacağız

<asp:ImageButton ID="imgDownload" runat="server" OnClientClick="return confirm('Are you sure want to Lock/Unlock ?');"
                            CommandName="togglelock"
                            CommandArgument='<%#Container.DataItemIndex%>'/>

Artık arka uçta komutları farklı satırlardan ve farklı düğmelerden ayırt edebilirsiniz.

protected void GridView1_RowCommand(object sender, GridViewCommandEventArgs e)
    {
        if (e.CommandName == "togglelock")
        {
            using (SqlConnection con= new SqlConnection(connectionString)) 
            {
               int index = Convert.ToInt32(e.CommandArgument);
               SqlCommand sqlCommand = new SqlCommand(" ... ", con);
               SqlDataReader reader = sqlCommand.ExecuteReader();
               GridView1.DataSource = reader;
               GridView1.DataBind();
            }
        }
    }

Trigger öğesine <asp:PostBackTrigger ControlID="GridView1"/> ekleyin ve DataBind tamamlandıktan sonra GridView güncellenecektir.

GridView’ı sayfanın ortasına yerleştirmek için HorizontalAlign="Center" kullanın.
