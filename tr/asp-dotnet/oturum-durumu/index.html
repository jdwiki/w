Sözdizimi #

  • Oturum[“Session_Key”] = Obj_Value;

HTTP durumsuzdur. ASP.NET oturum durumu, HTTP sayfa istekleri arasındaki durumu korumayı kolaylaştıran bir çerçevedir.

Oturum, geri göndermeler ve farklı sayfalar arasında kullanılabilir kalabilme özelliğiyle sınıf düzeyindeki değişkenlerden farklıdır. Örneğin, kullanıcı daha sonra aynı uygulama içinde Sayfa2.aspx’e yönlendirilirse, Sayfa1.aspx’te oluşturulan bir oturum değişkeni kullanılabilir olacaktır.

Ayrıca, sayfa düzeyinde bildirilen statik değişkenlerin aksine, oturum değişkenleri farklı kullanıcılar için bağımsızdır. Yani, bir kullanıcının oturum değişkeninin değerini değiştirmek, diğer kullanıcılar için aynı değişkenin değerini etkilemeyecektir.

ViewState kullanıcının verilerini geçici olarak depolamak için kullanılabilse de, verilerin birden çok sayfada kaydedilmesine izin vermez. Ayrıca, “viewstate” sayfanın bir parçasıdır ve istemciye gönderilir. Sonuç olarak, kullanıcıyla ilgili herhangi bir kritik bilgi ‘ViewState’e kaydedilemez ve bu noktada Oturum değişkenleri kullanışlı hale gelir.

Değerleri saklamak için Session nesnesini kullanma #

System.Web.SessionState.HttpSessionState nesnesi, HTTP istekleri arasında değerleri kalıcı hale getirmenin bir yolunu sağlar. Aşağıdaki örnekte, bir kullanıcının uyarı tercihi oturuma kaydedilmektedir. Daha sonra kullanıcıya başka bir istek sunarken, uygulama bu tercihi oturumdan okuyabilir ve uyarıları gizleyebilir.

public partial class Default : System.Web.UI.Page
{
    public void LoadPreferences(object sender, EventArgs args)
    {
        // ... 
        // ... A DB operation that loads the user's preferences.
        // ...

        // Store a value with the key showWarnings
        HttpContext.Current.Session["showWarnings"] = false;
    }

    public void button2Clicked(object sender, EventArgs args)
    {
        // While processing another request, access this value.
        bool showWarnings = (bool)HttpContext.Current.Session["showWarnings"];
        lblWarnings.Visible = false;
    }
}

Oturum değişkenlerinin tüm kullanıcılar için ortak olmadığını (tıpkı çerezler gibi) ve birden çok geri göndermede kalıcı olduklarını unutmayın.

Oturum, kullanıcı oturumu için bir tanımlayıcı içeren bir tanımlama bilgisi ayarlayarak çalışır. Varsayılan olarak bu tanımlayıcı, kendisine karşı depolanan değerlerle birlikte web sunucusu belleğinde saklanır.

SQL Oturum Mağazası Kullanma #

Oturum durumunu paylaşması gereken birden çok sunucunuz olduğunu fark ederseniz, bunu ASP.NET işlem belleğinde depolamak çalışmayacaktır. Örneğin, istekleri teker teker dağıtan bir yük dengeleyici ile bir web çiftliği ortamına konuşlandırabilirsiniz. Bu ortamda, tek bir kullanıcının istekleri birden fazla sunucu tarafından sunulabilir.

web.config dosyasında bir SQL sunucusu oturum deposunu yapılandırabilirsiniz.

<configuration>
  <system.web>
    <sessionState 
      mode="SQLServer"
      sqlConnectionString="Data Source=localhost;Integrated Security=SSPI"
      cookieless="true"
      timeout="30" />
  </system.web>
</configuration>

Sql şemasını oluşturmak için aspnet_regsql aracını kullanın. [SampleSqlServerName], SQL sunucusunun ana bilgisayar adıdır. -ssadd, araca oturum durumu veritabanını oluşturmasını söyler. -sstype p, araca ASPState varsayılan adıyla yeni bir veritabanı oluşturmasını söyler.

aspnet_regsql.exe -S [SampleSqlServerName] -U [Username] -P [Password] -ssadd -sstype p

Amazon DynamoDB Oturum Mağazasını Kullanma #

SQL sunucusu kullanmak istemiyorsanız, oturum deposu olarak Amazon’un barındırılan Dynamo DB nosql veritabanını kullanabilirsiniz.

AWS SDK’ya ihtiyacınız olacak. Bunu Visual Studio nuget paket yöneticisi konsolundan yüklemek için aşağıdaki komutu kullanın

Install-Package AWSSDK

Daha sonra sessionState sağlayıcınızı özel bir sağlayıcı kullanacak şekilde yapılandırabilirsiniz. Bir profil veya bir IAM erişimi ve gizli anahtar kombinasyonu olarak bölge ve kimlik bilgilerini belirtmelisiniz. Varsayılan olarak bu, ASP.NET_SessionState adlı bir tablo oluşturacaktır.

<configuration>
  <system.web>
    <sessionState
      timeout="20"
      mode="Custom"
      customProvider="DynamoDBSessionStoreProvider">
      <providers>
        <add name="DynamoDBSessionStoreProvider"
             type="Amazon.SessionProvider.DynamoDBSessionStateStore"
             AWSProfileName="[PROFILE]"
             Region="[REGION]"
             CreateIfNotExist="true"
             />
      </providers>
    </sessionState>
  </system.web>
</configuration>
