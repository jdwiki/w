Parametreler #

parametre ayrıntılar
e / düzenle Maya’ya var olan bir özelliğin değerini değiştirmek istediğinizi söyler
q / sorgu Maya’ya mevcut bir mülkün değerini almak istediğinizi söyler

Maya, pencereler, düzenler ve çeşitli kontroller içeren oldukça eksiksiz bir UI araç seti ile birlikte gelir. Bu, C++‘daki QT çerçevesi kullanılarak uygulanır, ancak varsayılan Maya komut seti aracılığıyla MEL ve Python kullanıcılarına sunulur.

QT #

İleri düzey kullanıcılar, Maya kullanıcı arabirimini C++ veya Python kullanarak genişletebilir. 2012’den 2016’ya kadar olan Maya sürümleri Pyside ve QT4’ü kullanır; Maya 2017, Pyside2 ve QT5’i kullanır. Daha fazla ayrıntı burada

Geri arama işlevleri #

Birçok pencere öğesi, kullanıcı pencere öğesiyle etkileşim kurduğunda geri arama işlevlerini başlatabilecek olaylar içerir. Örneğin bir düğmeye basıldığında, bir onay kutusu işaretlendiğinde veya bir açılır menü seçildiğinde bir işlevi tetikleyebilirsiniz.

Bu olayla ilişkilendirilen tam işaret, widget’a bağlıdır, ancak tipik bir geri arama şöyle görünür:

 def callback_fn(_ignore):
     print "button pressed"

 button = cmds.button(label='press me', command = callback_fn)

Düğmeye basmak, dinleyici penceresine “düğmeye basıldı” yazacaktır. Çoğu widget, geri aramaları etkinleştirildiğinde bazı argümanları ateşler - örneğin ‘düğme’ her zaman bir boole değeri içerir - bu nedenle, geri arama işleyicisinin, kullandığınız widget ile gitmek için doğru imzaya sahip olduğundan emin olmanız gerekir. Bu nedenle callback_fn() ihtiyacı olmasa bile bir argüman alır.

Geri Arama Ataması #

Maya, geri arama işlevleri eklemenin iki farklı yolunu destekler:

   # this works, but is not a great idea
   cmds.button(label = 'string reference', command = 'string_name_of_function')
   # use this form whenever possible
   cmds.button(label = 'direct function reference', command = callback_fn)

İlk örnekte geri arama, bir dize değeriyle atanır. Maya, geri aramayı global Python kapsamında bulacaktır - ki bu genellikle düzgün organize edilmiş kod yazarken erişilmesi zordur. Dize adı geri aramalarının çözülmesi de daha yavaştır. İkinci örnek, gerçek Python işlevini geri aramaya iletir - bu form daha hızlı olduğu için tercih edilir ve geri arama işlevine geçerli bir işlev sağlayamadıysanız, kullanıcı arabiriminin ne zaman oluşturulduğu yerine ne zaman oluşturulduğunu bilirsiniz. UI widget’ları aslında kullanılıyor.

Bir geri arama işlevine bir bağımsız değişken değeri iletmek istiyorsanız, bir lambda, bir [kapatma]( kullanabilirsiniz. http://www.shutupandship.com/2012/01/python-closures-explained.html) veya bir functools.partial argümanları geri aramaya bağlayın.

“kısmi” kullanma: #

 from functools import partial
 ....
 def callback_fn(myValue, _ignore):  # _ignore swallows the original button argument
     print myValue

 button = cmds.button(label='press me', command = partial(callback_fn, "fooo"))

“lambda"yı kullanma: #

 def callback_fn(myValue):
     print myValue

 button = cmds.button(label='press me', command = lambda _ignore: callback_fn("fooo"))
 # here the lambda needs to handle the button argument

Kapanışları Kullanma #

b = cmds.button(label = 'press me')
# by defining this function when `b` exists, we can use it later
# without storing it explicitly
def get_button_label(*_):
    print "label of button", b, " is ", cmds.button(b, q=True, l=True)
cmds.button(b, e=True, c=get_button_label)

Temel kullanıcı arayüzü örneği [Python] #

Maya GUI araç takımı, basit, zorunlu bir biçimde çeşitli UI öğeleri oluşturur. GUI widget’ları oluşturmak ve düzenlemek için temel komutlar vardır; widget’lar benzersiz bir dize adıyla tanımlanır.

Tüm gui komutları aynı temel biçimi alır: Bir komut türü ve üzerinde çalışmak veya oluşturmak istediğiniz nesnenin dize adını, widget’ın görünümünü veya davranışını belirten bayraklarla birlikte sağlarsınız. Örneğin, bir düğme oluşturmak için şunları kullanırsınız:

 cmds.button('my_button', label = 'my label')

Bu, yeni bir gui düğmesi oluşturacaktır. Düğmeyi düzenlemek için ’edit’ bayrağıyla aynı komutu kullanırsınız (kısa versiyon sadece ’e’dir). Böylece düğmenin etiketini şu şekilde değiştirebilirsiniz:

cmds.button('my_button', e=True, label = 'a different label')

ve bir özelliğin mevcut değerini “query” veya “q” bayrağıyla sorgulayabilirsiniz:

cmds.button(`my button`, q=True, label=True)
# 'a different label'

Widget adlandırma #

Bir UI komutuyla yeni bir pencere öğesi oluşturduğunuzda, yeni pencere öğesinin almasını istediğiniz adı sağlayabilirsiniz. Ancak, garanti edilmez: Maya, düğmeye istediğiniz adı verecektir – ona tanımadığı bir karakter verdiyseniz veya aynı ada sahip bir pencere öğesi zaten varsa, geri alabilirsiniz. farklı bir isim. Sürprizlerden kaçınmak için oluşturulduğunda yeni bir widget’ın adını yakalamak her zaman iyi bir uygulamadır:

 my_button = cmds.button('my_button') 
 cmds.button(my_button, e=True, label = "a new label")

Pencere oluşturma #

# create a window with a button that closes the window when clicked
window = cmds.window(title='example window')       # create the window
layout = cmds.columnLayout(adjustableColumn=True)  # add a vertical layout

def close_window(*_):
    cmds.deleteUI(window)                          # deletes the window above

button = cmds.button(label= 'press to close", command = close_window)

# show the window
cmds.showWindow(window)

Lambdalar ve döngüler #

Lambda’lar, davranışları GUI öğelerine bağlamak için kullanışlı bir kısayoldur.

b = cmds.button("make a cube", command = lambda _: cmds.polyCube())

Ancak, Python’un lambdaların içindeki değişkenleri yakalama şekli nedeniyle, bir döngü içinde lambdaları kullanarak komutları bağlarsanız beklenmedik sonuçlar alabilirsiniz. Örneğin, bu görünüşe göre farklı boyutlarda küreler oluşturan düğmeler üretmelidir:

# warning: doesn't work like it looks!
for n in range(5):
    b = cmds.button("sphere size %i" % n, command = lambda _: cmds.polySphere(radius=n))

Düğmeler doğru bir şekilde etiketlenecek ancak hepsi aynı yarıçapı (4) kullanacak çünkü döngü kapandığında lambdaların tümü bu değeri yakalayacaktır. TLDR: Bir döngü içinde geri aramalar oluşturuyorsanız, değerleri yakalamak için “functools.partial” veya başka bir yöntem kullanın - lambdalar bu uygulama için çalışmaz.
