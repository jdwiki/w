Ne tür koşul cümleleri kullanılır? #

Koşullu İfadeleri şu şekilde kullanın (sözdizimi “[parantezler]” içindedir):

  • ne zaman [ne zaman:]

    Task:
   - name: run if operating system is debian
     command: echo "I am a Debian Computer"
     when: ansible_os_family == "Debian"

  • döngüler [with_items:]

  • döngüler [with_dicts:]

  • Özel Bilgiler [ ne zaman: my_custom_facts == ‘1234’]

  • Koşullu ithalat

  • Değişkenlere göre dosyaları ve Şablonları seçin

Ne Zaman Koşul #

Temel Kullanım #

Bir görevin veya rolün çalışıp çalışmadığını veya atlanıp atlanmadığını kontrol etmek için ne zaman koşulunu kullanın. Bu normalde hedef sistemdeki gerçeklere dayalı olarak oyun davranışını değiştirmek için kullanılır. Bu oyun kitabını düşünün:

- hosts: all
  tasks:
    - include: Ubuntu.yml
      when: ansible_os_family == "Ubuntu"
    
    - include: RHEL.yml
      when: ansible_os_family == "RedHat"

“Ubuntu.yml” ve “RHEL.yml”, dağıtıma özel bir mantık içerir.

Diğer bir yaygın kullanım, sonuçları belirli Ansible envanter gruplarındaki sonuçlarla sınırlamaktır. Bu envanter dosyasını düşünün:

[dbs]
mydb01

[webservers]
myweb01

Ve bu oyun kitabı:

- hosts: all
  tasks:
    - name: Restart Apache on webservers
      become: yes
      service:
        name: apache2
        state: restarted
      when: webservers in group_names

Bu, group_names sihirli değişken kullanıyor.

Koşullu Sözdizimi ve Mantık #

Tek koşul #

Sözdizimi

ne zaman: (koşul)

Örnek

  • ne zaman: ansible_os_family == "Debian"
  • ne zaman: ansible_pkg_mgr == "uygun"
  • “ne zaman: değişkenadım tanımlı”

Boole Filtresi #

Örnek

“ne zaman: sonuç|başarısız”

Çoklu Koşullar #

Sözdizimi

“Ne zaman: koşul1 ve/veya koşul2”

Örnek (basit)

When: ansible_os_family == "Debian" ve ansible_pkg_mgr == "apt"

Örnek (karmaşık)

Netlik veya önceliği kontrol etmek için parantez kullanın. “VE”, “VEYA"dan daha yüksek önceliğe sahiptir.

Cümleler satırlara yayılabilir:

when:
  ansible_distribution in ['RedHat', 'CentOS', 'ScientificLinux'] and
  (ansible_distribution_version|version_compare('7', '<') or
  ansible_distribution_version|version_compare('8', '>='))
  or
  ansible_distribution == 'Fedora'
  or
  ansible_distribution == 'Ubuntu' and
  ansible_distribution_version|version_compare('15.04', '>=')

İlk dağıtım kontrolünde “veya” gruplamak için parantez kullanımına dikkat edin.

Kurulumla ‘ansible_os_family’ ve ‘ansible_pkg_mgr’ alın #

Kurulum modülünün Ad-Hoc komutu ve filtre ile gerçekleri (ansible_os_family, ansible_pkg_mgr) alabiliriz.

  • ansible_os_family:

      $ ansible all -m setup -a 'filter=ansible_os_family'
  ra.local | SUCCESS => {
      "ansible_facts": {
          "ansible_os_family": "Debian"
      },
      "changed": false
  }

  • ansible_pkg_mgr:

      $ ansible all -m setup -a 'filter=ansible_pkg_mgr'
  debian.local | SUCCESS => {
      "ansible_facts": {
          "ansible_pkg_mgr": "apt"
      },
      "changed": false
  }

Basit “Ne zaman” Örnek(ler)i #

Verilen:

---
variable_name: True

Daha sonra bu görevler ile her zaman çalışır.

- name: This is a conditional task
  module: src=/example/ dest=/example
  when: variable_name 

- name: This is a conditional task
  module: src=/example/ dest=/example
  when: True

Bu görev asla çalışmayacak.

- name: This is a conditional task
  module: src=/example/ dest=/example
  when: False

Yeniden deneme döngüsüne kadar canlı kontrol için kullanma #

Bu, başlamakta olan bir web uygulaması için canlı bir kontrol uygulamak üzere kadar/tekrar deneme/gecikme kullanımına bir örnektir. Web uygulamasının soket bağlantılarını reddettiği bir süre (3 dakikaya kadar) olacağını varsayar. Bundan sonra, /alive sayfasını “OK” kelimesi için kontrol eder. Ayrıca, URL’nin alınmasını ansible çalıştıran localhost’a devreder. Bu, bir dağıtım çalışma kitabındaki son görev olarak anlamlıdır.

---
- hosts: my-hosts
  tasks:
  - action: uri url=http://{{ ansible_all_ipv4_addresses }}:8080/alive return_content=yes
    delegate_to: localhost
    register: result
    until: "'failed' not in result and result.content.find('OK') != -1"
    retries: 18
    delay: 10

Kadar yeniden deneme deseni herhangi bir eylemle kullanılabilir; Ansible belgeleri, belirli bir kabuk komutu istenen sonucu döndürene kadar beklemenin bir örneğini sağlar: http://docs.ansible.com/ansible/playbooks_loops.html#do-until-loops.
