Grunt’u Yükleme #

Önkoşullar #

Grunt, Node.js ve npm’nin yüklenmesini gerektirir. Makinenizde Node.js ve/veya npm kurulu değilse, https://nodejs.org adresine gidin ve işletim sisteminiz için yükleyiciyi veya paketi indirin.

İlk kurulum #

Grunt’u ilk kez kuruyorsanız, öncelikle Grunt komut satırı arabirim paketi ‘grunt-cli’yi global olarak yüklemeniz gerekir.

npm install -g grunt-cli

Bu, Grunt’un yerel sürümünü projenizde çalıştırabilmeniz için genel olarak Grunt için komut satırı arabirimini yükler.

Aşağıdaki komutu çalıştırarak grunt-cli paketinizin kurulu olduğunu doğrulayabilirsiniz:

grunt --version

Bu, en azından “grunt-cli” paketinizin güncel sürümünü yazdırmalıdır.

Grunt’u projenize kurmak #

“grunt-cli"yi kurduktan ve çalıştırdıktan sonra, asıl “grunt” görev çalıştırıcısını ve ilk Grunt paketiniz “grunt-contrib-jshint"i yükleyebilirsiniz:

npm install grunt --save-dev
npm install grunt-contrib-jshint --save-dev

Bu, paketleri NPM paket yöneticisinden indirir ve onları “package.json” dosyanıza “devDependencies” olarak kaydeder.

Gruntfile #

Ardından proje kökünüzde Grunt görevleri için bir yapılandırma dosyası görevi gören bir “Gruntfile.js” dosyasına ihtiyacınız var:

module.exports = function(grunt) {

    grunt.initConfig({
        jshint: {
            files: ['Gruntfile.js'],
        }
    });

    grunt.loadNpmTasks('grunt-contrib-jshint');


    grunt.registerTask('default', ['jshint']);
};

Bu dosya üç şey yapar:

  1. Grunt’a “grunt-contrib-jshint” görevini NPM paketinden yüklemesini söyler
  2. ‘jshint’ görevinin ‘Gruntfile.js’ dosyasına karşı çalıştırılmasını önerir.
  3. ‘jshint’ görevini çalıştıran ‘varsayılan’ adlı bir Grunt görevi oluşturur.

Koşma Grunt #

Projenizi kurduktan sonra, Grunt’un “varsayılan” görevini aşağıdakileri arayarak çalıştırabilirsiniz:

grunt

Bu, “jshint” adlı görevi çalıştırmak üzere yapılandırılmış “varsayılan” adlı bir Grunt görevi arayan yerel “grunt"u çalıştıran “grunt-cli"yi çalıştırır.

Görevleri çalıştırma #

Yapılandırmadaki görevler #

Grunt.initConfig’in tüm özellikleri geçerli görevlerdir, bu nedenle Gruntfile’niz şöyle görünüyorsa:

module.exports = function(grunt) {

    grunt.initConfig({
        jshint: {
            files: ['Gruntfile.js'],
        }
    });

    grunt.loadNpmTasks('grunt-contrib-jshint');


    grunt.registerTask('default', ['jshint']);
};

$ grunt jshint kabuk komutu, jshint görevini çalıştıracaktır.

Hedefli görevler #

Görevlerin farklı hedefleri olabilir. Örneğin bu kod parçasını alın:

    grunt.initConfig({
        jshint: {
            gruntfile: {
                files: ['Gruntfile.js']
            },
            project: {
                files: 'src/**/*.js'
            }
        }
    });

Burada jshint, projenizin gruntfile veya tüm JavaScript dosyalarını hedefleyebilir. $ grunt jshint çalıştırırsak her iki hedef de kullanılacaktır, ancak $ grunt jshint:gruntfile çalıştırırsak linter yalnızca gruntfile’a uygulanacaktır.

Kayıtlı görevler #

Bu grunt.registerTask('default', ['jshint']); gibi kaydedilen varsayılan görevler, $ grunt kabuk komutuyla çalışacaktır.

Yeni kayıtlı görevler, adını grunt’a komut satırı argümanı olarak ileterek çalışır. Örneğin:

grunt.registerTask('gruntfile', ['jshint:gruntfile']);

“$ grunt gruntfile” ile çalışacak.

Yardım #

Aşağıdakileri görmek için grunt -h komutunu çalıştırın:

  • Komut satırı parametreleri
  • Geçerli dizindeki Gruntfile’deki mevcut tüm bağımsız görevler
  • Geçerli dizinin Gruntfile’sindeki her bir görev grubunun tüm alt görevleri
