Sözdizimi #

  • işlev adı = işlev(paramA, paramB, …) gövde; ifade listesi sonunu döndür – basit bir işlev
  • işlev işlev adı(paramA, paramB, …) gövde; dönüş açıklama listesi sonu – yukarıdaki için kısayol
  • yerel işlev adı = işlev(paramA, paramB, …) gövde; dönüş exprlist sonu – bir lambda
  • yerel işlev adı; işlev adı = işlev(paramA, paramB, …) gövde; dönüş exprlist end – özyinelemeli aramalar yapabilen lambda
  • yerel işlev işlev adı(paramA, paramB, …) gövde; dönüş açıklama listesi sonu – yukarıdaki için kısayol
  • funcname(paramA, paramB, …) – bir işlev çağırın
  • yerel var = var veya “Varsayılan” – varsayılan bir parametre
  • sıfır, “hata mesajları” döndür – bir hatayla iptal etmenin standart yolu

İşlevler genellikle “a(b,c) … bitiş” işleviyle ve nadiren bir değişkeni anonim bir işleve (‘a = işlev(a,b) … bitiş’) ayarlayarak ayarlanır. Fonksiyonları parametre olarak iletirken bunun tersi doğrudur, çoğunlukla anonim fonksiyonlar kullanılır ve normal fonksiyonlar o kadar sık ​​kullanılmaz.

Bir fonksiyon tanımlama #

function add(a, b)
    return a + b
end
-- creates a function called add, which returns the sum of it's two arguments

Sözdizimine bakalım. İlk olarak, bir “function” anahtar sözcüğünü görüyoruz. Bu oldukça açıklayıcı. Sonra ‘add’ tanımlayıcısını görüyoruz; isim. Daha sonra (a, b) argümanlarını görüyoruz, bunlar herhangi bir şey olabilir ve yereldirler. Onlara sadece fonksiyon gövdesinin içinde erişebiliriz. Sona geçelim, görüyoruz… peki, ‘son’! Ve arada kalan tek şey işlev gövdesidir; çağrıldığında çalıştırılan kod. ‘return’ anahtar sözcüğü, işlevin aslında bazı yararlı çıktılar vermesini sağlayan şeydir. Onsuz, işlev, sıfır döndürmeye eşdeğer olan hiçbir şey döndürmez. Bu, IO ile etkileşime giren şeyler için elbette yararlı olabilir, örneğin:

function printHello(name)
    print("Hello, " .. name .. "!");
end

Bu fonksiyonda return ifadesini kullanmadık.

İşlevler ayrıca koşullu olarak değerler döndürebilir, yani bir işlevin hiçbir şey (nil) veya bir değer döndürme seçeneği yoktur. Bu, aşağıdaki örnekte gösterilmiştir.

function add(a, b)
    if (a + b <= 100) then
        return a + b -- Returns a value
    else
        print("This function doesn't return values over 100!") -- Returns nil
    end
end

Bir işlevin, gösterildiği gibi virgülle ayrılmış birden çok değer döndürmesi de mümkündür:

function doOperations(a, b)
    return a+b, a-b, a*b
end

added, subbed, multiplied = doOperations(4,2)

İşlevler ayrıca yerel olarak da bildirilebilir

do
    local function add(a, b) return a+b end
    print(add(1,2)) --> prints 3
end
print(add(2, 2)) --> exits with error, because 'add' is not defined here

Tablolara da kaydedilebilirler:

tab = {function(a,b) return a+b end}
(tab[1])(1, 2) --> returns 3

Bir işlevi çağırmak. #

Fonksiyonlar sadece onları çağırabiliyorsak faydalıdır. Bir işlevi çağırmak için aşağıdaki sözdizimi kullanılır:

print("Hello, World!")

“yazdır” işlevini çağırıyoruz. "Merhaba, Dünya" argümanını kullanma. Açıkça görüldüğü gibi, bu çıktı akışına ‘Merhaba, Dünya’ yazdıracaktır. Döndürülen değere erişilebilir, tıpkı diğer değişkenler gibi.

local added = add(10, 50) -- 60

Değişkenler de bir fonksiyonun parametrelerinde kabul edilir.

local a = 10
local b = 60

local c = add(a, b)

print(c)

Bir tablo veya dizge bekleyen işlevler, düzgün bir sözdizimsel şekerle çağrılabilir: çağrıyı çevreleyen parantezler atlanabilir.

print"Hello, world!"
for k, v in pairs{"Hello, world!"} do print(k, v) end

Anonim işlevler #

Anonim işlevler oluşturma #

Anonim işlevler, bir adları olmaması dışında, normal Lua işlevleri gibidir.

doThrice(function()
    print("Hello!")
end)

Görüldüğü gibi fonksiyon “yazdır” veya “ekle” gibi herhangi bir isme atanmamıştır. Anonim bir işlev oluşturmak için tek yapmanız gereken adı atlamaktır. Bu işlevler ayrıca argüman alabilir.

Sözdizimsel şekeri anlama #

Aşağıdaki kodu anlamak önemlidir

function double(x)
    return x * 2
end

aslında sadece bir kısaltmadır

double = function(x)
    return x * 2
end

Ancak, işlev doğrudan bir değişkene atandığından, yukarıdaki işlev anonim değildir!

Fonksiyonlar birinci sınıf değerlerdir #

Bu, bir işlevin sayılar ve dizeler gibi geleneksel değerlerle aynı haklara sahip bir değer olduğu anlamına gelir. Fonksiyonlar değişkenlerde, tablolarda saklanabilir, argüman olarak iletilebilir ve diğer fonksiyonlar tarafından döndürülebilir.

Bunu göstermek için ayrıca bir “yarım” fonksiyon oluşturacağız:

half = function(x)
    return x / 2
end

Şimdi, her ikisi de değer olarak bir fonksiyon içeren iki değişkenimiz var, ‘half’ ve ‘double’. 4 sayısını verilen iki fonksiyona besleyecek ve her iki sonucun toplamını hesaplayacak bir fonksiyon yaratmak istersek ne olur?

Bu işlevi `sumOfTwoFunctions(double, half, 4)’ gibi çağırmak isteyeceğiz. Bu, ‘çift’ işlevi, ‘yarım’ işlevini ve ‘4’ tamsayısını kendi işlevimize besleyecektir.

function sumOfTwoFunctions(firstFunction, secondFunction, input)
    return firstFunction(input) + secondFunction(input)
end

Yukarıdaki “sumOfTwoFunctions” işlevi, işlevlerin bağımsız değişkenler içinde nasıl aktarılabileceğini ve başka bir adla erişilebileceğini gösterir.

Değişken sayıda argüman #

https://www.wikiod.com/tr/lua/degisken-argumanlar

Argüman türlerini kontrol etme #

Bazı işlevler yalnızca belirli bir bağımsız değişken türü üzerinde çalışır:

function foo(tab)
    return tab.bar
end
--> returns nil if tab has no field bar, which is acceptable
--> returns 'attempt to index a number value' if tab is, for example, 3
--> which is unacceptable

function kungfoo(tab)
    if type(tab) ~= "table" then
        return nil, "take your useless " .. type(tab) .." somewhere else!"
    end

    return tab.bar
end

bunun birkaç anlamı vardır:

print(kungfoo(20)) --> prints 'nil, take your useless number somewhere else!'

if kungfoo(20) then print "good" else print "bad" end --> prints bad

foo = kungfoo(20) or "bar" --> sets foo to "bar"

şimdi fonksiyonu istediğimiz parametre ile çağırabiliriz, ve programı çökertmez.

-- if we actually WANT to abort execution on error, we can still do
result = assert(kungfoo({bar=20})) --> this will return 20
result = assert(kungfoo(20)) --> this will throw an error

Peki ya belirli bir sınıfın örneğiyle bir şeyler yapan bir fonksiyonumuz varsa? Bu zordur, çünkü sınıflar ve nesneler genellikle tablolardır, bu nedenle ’type’ işlevi ’table’ değerini döndürür.

local Class = {data="important"}
local meta = {__index=Class}

function Class.new()
    return setmetatable({}, meta)
end
-- this is just a very basic implementation of an object class in lua

object = Class.new()
fake = {}

print(type(object)), print(type(fake)) --> prints 'table' twice

Çözüm: metatabloları karşılaştırın

-- continuation of previous code snippet
Class.is_instance(tab)
    return getmetatable(tab) == meta
end

Class.is_instance(object) --> returns true
Class.is_instance(fake) --> returns false
Class.is_instance(Class) --> returns false
Class.is_instance("a string") --> returns false, doesn't crash the program
Class.is_instance(nil) --> also returns false, doesn't crash either

Kapanışlar #

do
    local tab = {1, 2, 3}
    function closure()
        for key, value in ipairs(tab) do
            print(key, "I can still see you")
        end
    end
    closure()
    --> 1 I can still see you
    --> 2 I can still see you
    --> 3 I can still see you
end

print(tab) --> nil
-- tab is out of scope

closure()
--> 1 I can still see you
--> 2 I can still see you
--> 3 I can still see you
-- the function can still see tab

tipik kullanım örneği #

function new_adder(number)
    return function(input)
        return input + number
    end
end
add_3 = new_adder(3)
print(add_3(2)) --> prints 5

daha gelişmiş kullanım örneği #

function base64.newDecoder(str) -- Decoder factory
    if #str ~= 64 then return nil, "string must be 64 characters long!" end

    local tab = {}
    local counter = 0
    for c in str:gmatch"." do
        tab[string.byte(c)] = counter
        counter = counter + 1
    end

    return function(str)
        local result = ""

        for abcd in str:gmatch"..?.?.?" do
            local a, b, c, d = string.byte(abcd,1,-1)
            a, b, c, d = tab[a], tab[b] or 0, tab[c] or 0, tab[d] or 0
            result = result .. (
                string.char( ((a<<2)+(b>>4))%256 ) ..
                string.char( ((b<<4)+(c>>2))%256 ) ..
                string.char( ((c<<6)+d)%256 )
            )
        end
        return result
    end
end

Varsayılan parametreler #

function sayHello(name)
    print("Hello, " .. name .. "!")
end

Bu işlev basit bir işlevdir ve iyi çalışır. Ama sadece sayHello() diye çağırırsak ne olur?

stdin:2: attempt to concatenate local 'name' (a nil value)
stack traceback:
    stdin:2: in function 'sayHello'
    stdin:1: in main chunk
    [C]: in ?

Bu tam olarak harika değil. Bunu düzeltmenin iki yolu vardır:

  1. İşlevden hemen dönersiniz:

    function sayHello(name)
  if not (type(name) == "string") then
    return nil, "argument #1: expected string, got " .. type(name)
  end -- Bail out if there's no name.
  -- in lua it is a convention to return nil followed by an error message on error

  print("Hello, " .. name .. "!") -- Normal behavior if name exists.
end

  2. Bir varsayılan parametre ayarladınız.

    To do this, simply use this simple expression

    function sayHello(name) name = name or “Jack” – Jack is the default, – but if the parameter name is given, – name will be used instead print(“Hello, " .. name .. “!”) end

isim = isim veya "Jack" deyimi Lua kısa devrelerinde veya olduğu için çalışır. Bir “veya” öğesinin sol tarafındaki öğe “sıfır” veya “yanlış” dışında bir şeyse, sağ taraf hiçbir zaman değerlendirilmez. Öte yandan, “sayHello” parametresiz olarak çağrılırsa, “name” “nil” olur ve böylece “Jack” dizgisi “name"ye atanır. (Bu nedenle, bu deyimin, söz konusu parametre için boolean ‘false’ makul bir değerse çalışmayacağını unutmayın.)

Birden çok sonuç #

Lua’daki işlevler birden çok sonuç döndürebilir.

Örneğin:

function triple(x)
    return x, x, x
end

Bir işlevi çağırırken bu değerleri kaydetmek için aşağıdaki sözdizimini kullanmanız gerekir:

local a, b, c = triple(5)

Bu durumda a = b = c = 5 ile sonuçlanacaktır. Değişkenler listesinde istenen yerde atılabilir değişken ‘_’ kullanılarak döndürülen değerleri yok saymak da mümkündür:

local a, _, c = triple(5)

Bu durumda, ikinci döndürülen değer yok sayılır. Herhangi bir değişkene atamadan dönüş değerlerini yok saymak da mümkündür:

local a = triple(5)

‘a’ değişkenine ilk dönüş değeri atanacak ve kalan ikisi atılacaktır.

Bir işlev tarafından değişken miktarda sonuç döndürüldüğünde, içindeki işlevi yürüterek hepsini bir tabloda saklayabilirsiniz:

local results = {triple(5)}

Bu şekilde, işlevin ne döndürdüğünü görmek için ‘sonuçlar’ tablosu üzerinde yinelenebilir.

Not

Bu, bazı durumlarda sürpriz olabilir, örneğin:

local t = {}
table.insert(t, string.gsub("  hi", "^%s*(.*)$", "%1")) --> bad argument #2 to 'insert' (number expected, got string)

Bunun nedeni, “string.gsub” öğesinin 2 değer döndürmesidir: verilen desen, değiştirilen desen örnekleri ve oluşan toplam eşleşme sayısı.

Bunu çözmek için, ya bir ara değişken kullanın ya da aramanın etrafına () koyun, şöyle:

table.insert(t, (string.gsub("  hi", "^%s*(.*)$", "%1"))) --> works. t = {"hi"}

Bu, aramanın yalnızca ilk sonucunu alır ve gerisini yok sayar.

Adlandırılmış Argümanlar #

local function A(name, age, hobby)
    print(name .. "is " .. age .. " years old and likes " .. hobby)
end
A("john", "eating", 23) --> prints 'john is eating years old and likes 23'
-- oops, seems we got the order of the arguments wrong...
-- this happens a lot, specially with long functions that take a lot of arguments
-- and where the order doesn't follow any particular logic

local function B(tab)
    print(tab.name .. "is " .. tab.age .. " years old and likes " .. tab.hobby)
end
local john = {name="john", hobby="golf", age="over 9000", comment="plays too much golf"}
B(john)
--> will print 'John is over 9000 years old and likes golf'
-- I also added a 'comment' argument just to show that excess arguments are ignored by the function

B({name = "tim"}) -- can also be written as
B{name = "tim"} -- to avoid cluttering the code
--> both will print 'tim is nil years old and likes nil'
-- remember to check for missing arguments and deal with them

function C(tab)
    if not tab.age then return nil, "age not defined" end
    tab.hobby = tab.hobby or "nothing"
    -- print stuff
end

-- note that if we later decide to do a 'person' class
-- we just need to make sure that this class has the three fields
-- age, hobby and name, and it will be compatible with these functions

-- example:
local john = ClassPerson.new("John", 20, "golf") -- some sort of constructor
john.address = "some place" -- modify the object
john:do_something("information") -- call some function of the object
C(john) -- this works because objects are *usually* implemented as tables
