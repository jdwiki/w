Lua’da Boole değerleri, doğruluk ve yanlışlık açıktır. İncelemek için:

  1. Tam olarak iki değere sahip bir boole türü vardır: ’true’ ve ‘false’.
  2. Koşullu bağlamda (‘if’, ’elseif’, ‘while’, ‘until’), bir boole gerekli değildir. Herhangi bir ifade kullanılabilir.
  3. Koşullu bir bağlamda, “yanlış” ve “sıfır” yanlış olarak sayılır ve diğer her şey doğru olarak sayılır.
  4. 3 zaten bunu ima etse de: başka dillerden geliyorsanız, Lua’daki koşullu bağlamlarda “0"ın ve boş dizenin doğru sayıldığını unutmayın.

Boole türü #

Boole değerleri ve diğer değerler #

Lua ile uğraşırken, “doğru” ve “yanlış” boole değerleri ile doğru veya yanlış olarak değerlendirilen değerler arasında ayrım yapmak önemlidir.

Lua’da false olarak değerlendirilen yalnızca iki değer vardır: “nil” ve “false”, sayısal “0” dahil diğer her şey doğru olarak değerlendirilir.

Bunun ne anlama geldiğine dair bazı örnekler:

if 0 then print("0 is true") end --> this will print "true"
if (2 == 3) then print("true") else print("false") end --> this prints "false"
if (2 == 3) == false then print("true") end --> this prints "true"
if (2 == 3) == nil then else print("false") end
--> prints false, because even if nil and false both evaluate to false,
--> they are still different things.

Mantıksal İşlemler #

Lua’daki mantıksal operatörler mutlaka boole değerleri döndürmez:

’ve’, ilk değer doğru olarak değerlendirilirse ikinci değeri döndürür;

“veya”, ilk değer yanlış olarak değerlendirilirse ikinci değeri döndürür;

Bu, diğer dillerde olduğu gibi üçlü operatörü simüle etmeyi mümkün kılar:

local var = false and 20 or 30 --> returns 30
local var = true and 20 or 30 --> returns 20
-- in C: false ? 20 : 30

Bu, mevcut değilse tabloları başlatmak için de kullanılabilir.

tab = tab or {} -- if tab already exists, nothing happens

veya if deyimlerini kullanmaktan kaçınmak için kodun okunmasını kolaylaştırır

print(debug and "there has been an error") -- prints "false" line if debug is false
debug and print("there has been an error") -- does nothing if debug is false
-- as you can see, the second way is preferable, because it does not output
-- anything if the condition is not met, but it is still possible.
-- also, note that the second expression returns false if debug is false,
-- and whatever print() returns if debug is true (in this case, print returns nil)

Değişkenlerin tanımlanıp tanımlanmadığını kontrol etme #

Bir değişkenin var olup olmadığı (tanımlanmışsa) kolayca kontrol edilebilir, çünkü var olmayan değişkenler false olarak değerlendirilen “nil” değerini döndürür.

local tab_1, tab_2 = {}
if tab_1 then print("table 1 exists") end --> prints "table 1 exists"
if tab_2 then print("table 2 exists") end --> prints nothing

Bunun geçerli olmadığı tek durum, bir değişkenin “false” değerini depolamasıdır, bu durumda teknik olarak var olur ancak yine de false olarak değerlendirilir. Bu nedenle, duruma veya girdiye bağlı olarak ‘false’ ve ’nil’ döndüren işlevler oluşturmak kötü bir tasarımdır. Yine de bir “nil” veya “yanlış” olup olmadığını kontrol edebiliriz:

if nil == nil then print("A nil is present") else print("A nil is not present") end
if false == nil then print("A nil is present") else print("A nil is not present") end
-- The output of these calls are:
-- A nil is present!
-- A nil is not present

Koşullu bağlamlar #

Lua’daki koşullu bağlamlar (“if”, “elseif”, “while”, “until”) bir boole gerektirmez. Birçok dilde olduğu gibi, herhangi bir Lua değeri bir durumda görünebilir. Değerlendirme kuralları basittir:

  1. “false” ve “nil” false olarak sayılır.

  2. Diğer her şey doğru sayılır.

    if 1 then
  print("Numbers work.")
end
if 0 then
  print("Even 0 is true")
end

if "strings work" then
  print("Strings work.")
end
if "" then
  print("Even the empty string is true.")
end

Mantıksal operatörler #

Lua’da, boole değerleri mantıksal operatörler aracılığıyla manipüle edilebilir. Bu operatörler arasında ’not’, ‘and’ ve ‘or’ bulunur.

Basit ifadelerde, sonuçlar oldukça basittir:

print(not true) --> false
print(not false) --> true
print(true or false) --> true
print(false and true) --> false

Öncelik sırası #

Öncelik sırası, tekli -, * ve + matematik operatörlerine benzer:

  • “değil”
  • sonra ’ve’
  • sonra ‘veya’

Bu, karmaşık ifadelere yol açabilir:

print(true and false or not false and not true)
print( (true and false) or ((not false) and (not true)) )
    --> these are equivalent, and both evaluate to false

Kısayol Değerlendirmesi #

’ve’ ve ‘veya’ operatörleri, ikincisinin gereksiz olması koşuluyla, yalnızca ilk işlenen kullanılarak değerlendirilebilir:

function a()
    print("a() was called")
    return true
end

function b()
    print("b() was called")
    return false
end

print(a() or b())
    --> a() was called
    --> true
    --  nothing else
print(b() and a())
    --> b() was called
    --> false
    --  nothing else
print(a() and b())
    --> a() was called
    --> b() was called
    --> false

Deyimsel koşullu operatör #

Mantıksal operatörlerin önceliği, kısayol değerlendirmesi ve “yanlış” olmayan ve “sıfır” olmayan değerlerin “doğru” olarak değerlendirilmesi nedeniyle, Lua’da deyimsel bir koşul operatörü mevcuttur:

function a()
    print("a() was called")
    return false
end
function b()
    print("b() was called")
    return true
end
function c()
    print("c() was called")
    return 7
end

print(a() and b() or c())
    --> a() was called
    --> c() was called
    --> 7
    
print(b() and c() or a())
    --> b() was called
    --> c() was called
    --> 7

Ayrıca, ‘x ve a veya b’ yapısının doğası gereği, ‘false’ olarak değerlendirilirse ‘a’ asla döndürülmez, bu koşul, ‘x’ ne olursa olsun her zaman ‘b’ döndürür. .

print(true and false or 1)  -- outputs 1

Doğruluk tabloları #

Lua’daki mantıksal operatörler boolean “döndürmez”, ancak argümanlarından biridir. Yanlış için ’nil’ ve doğru için sayılar kullanıldığında, işte böyle davranırlar.

print(nil and nil)       -- nil
print(nil and 2)         -- nil
print(1 and nil)         -- nil
print(1 and 2)           -- 2

print(nil or nil)        -- nil
print(nil or 2)          -- 2
print(1 or nil)          -- 1
print(1 or 2)            -- 1

Gördüğünüz gibi, Lua her zaman kontrolü başarısız veya başarılı yapan ilk değeri döndürür. İşte bunu gösteren doğruluk tabloları.

  x  |  y  || and            x  |  y  || or
------------------         ------------------
false|false||  x           false|false||  y   
false|true ||  x           false|true ||  y   
true |false||  y           true |false||  x   
true |true ||  y           true |true ||  x

İhtiyacı olanlar için, işte bu mantıksal operatörleri temsil eden iki fonksiyon.

function exampleAnd(value1, value2)
  if value1 then
    return value2
  end
  return value1
end

function exampleOr(value1, value2)
  if value1 then
    return value1
  end
  return value2
end

Üçlü Operatöre ’ve’ ‘veya’ mantıksal operatörleri ile öykünme. #

Lua’da ’ve’ ve ‘veya’ mantıksal operatörleri, bir boole sonucu yerine sonuç olarak işlenenlerden birini döndürür. Sonuç olarak, bu mekanizma, lua’nın dilde ‘gerçek’ bir üçlü operatöre sahip olmamasına rağmen üçlü operatörün davranışını taklit etmek için kullanılabilir.

Sözdizimi #

durum ve truthy_expr veya falsey_expr

Değişken atama/başlatmada kullanım #

local drink = (fruit == "apple") and "apple juice" or "water"

Tablo yapıcısında kullanın #

local menu =
{
  meal  = vegan and "carrot" or "steak",
  drink = vegan and "tea"    or "chicken soup"
}

İşlev argümanı olarak kullan #

print(age > 18 and "beer" or "fruit punch")

dönüş ifadesinde kullanın #

function get_gradestring(student)
  return student.grade > 60 and "pass" or "fail"
end

Uyarı #

Bu mekanizmanın istenen davranışa sahip olmadığı durumlar vardır. Bu durumu göz önünde bulundurun

local var = true and false or "should not happen"

Bir ‘gerçek’ üçlü operatörde, ‘var’ın beklenen değeri ‘yanlış’tır. Bununla birlikte, lua’da, ikinci işlenen yanlış olduğu için ’ve’ değerlendirmesi ‘geçerlidir’. Sonuç olarak ‘var’, bunun yerine ‘olmamalı’ ile biter.

Bu soruna olası iki geçici çözüm, ortadaki işlenenin yanlış olmaması için bu ifadeyi yeniden düzenleyin. Örneğin.

local var = not true and "should not happen" or false

veya alternatif olarak, klasik “if” “then” “else” yapısını kullanın.
