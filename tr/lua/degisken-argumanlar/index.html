Değişken Argümanlar

  • Sözdizimi
  • Temel Bilgiler
  • Gelişmiş Kullanım

Bu sayfada

  • Sözdizimi
  • Temel Bilgiler
  • Gelişmiş Kullanım

Varargs, yaygın olarak bilindiği gibi, işlevlerin belirtim olmadan rastgele sayıda argüman almasına izin verir. Böyle bir işleve verilen tüm argümanlar vararg listesi olarak bilinen tek bir yapı içinde paketlenir; hangi Lua’da ... olarak yazılır. ‘select()’ işlevini kullanarak verilen argümanların sayısını ve bu argümanların değerini çıkarmak için temel yöntemler vardır, ancak daha gelişmiş kullanım kalıpları yapıyı tam faydası için kullanabilir.

Sözdizimi #

  • … – Değişken bir işlev olarak göründüğü argümanları listesi olan işlevi yapar
  • select(ne, …) – Eğer ’ne’, vararg’daki eleman sayısı ile 1 aralığında bir sayıysa, ‘what’th öğesini vararg’daki son elemana döndürür. Endeks sınırların dışındaysa getiri sıfır olacaktır. ‘#’ dizesi ’ne’ ise, vararg’daki öğelerin sayısını döndürür.

Yeterlik

Vararg listesi, dilin PUC-Rio uygulamasında bağlantılı bir liste olarak uygulanır, bu, dizinlerin O(n) olduğu anlamına gelir. Bu, aşağıdaki örnekte olduğu gibi ‘select()’ kullanarak bir vararg’daki öğeler üzerinde yinelemenin bir O(n^2) işlemi olduğu anlamına gelir.

for i = 1, select('#', ...) do
    print(select(i, ...))
end

Bir vararg listesindeki öğeleri yinelemeyi planlıyorsanız, önce listeyi bir tabloya paketleyin. Tablo erişimleri O(1)‘dir, dolayısıyla yineleme toplamda O(n)‘dir. Ya da çok istekliyseniz gelişmiş kullanım bölümünden foldr() örneğine bakın; O(n) içindeki bir vararg listesi üzerinde yineleme yapmak için özyinelemeyi kullanır.

Sıra Uzunluğu Tanımı

Vararg, vararg’ın uzunluğunun, açıkça geçirilen (veya hesaplanan) sıfırlara uyması açısından yararlıdır. Örneğin.

function test(...)
    return select('#', ...)
end

test()             --> 0
test(nil, 1, nil)  --> 3

Ancak bu davranış, uzunluk operatörünün “#” dizilerdeki “delikler” (gömülü sıfırlar) ile çalışmadığı tabloların davranışıyla çelişir. Delikli bir tablonun uzunluğunu hesaplamak tanımsızdır ve buna güvenilemez. Dolayısıyla, ... içindeki değerlere bağlı olarak, {...} uzunluğunu almak ‘doğru’ yanıtla sonuçlanmayabilir. Lua 5.2+ sürümünde bu eksikliği gidermek için table.pack() tanıtıldı (örnekte bu işlevi saf Lua’da uygulayan bir işlev var).

Deyimsel Kullanım

Vararglar uzunlukları boyunca taşındığından, insanlar tablolardaki delikler sorununu önlemek için bunları diziler olarak kullanırlar. Bu onların amaçlanan kullanımı değildi ve Lua’nın referans uygulamasının optimize etmediği bir kullanımdı. Bu tür kullanım örneklerde araştırılmış olsa da, genellikle hoş karşılanmaz.

Temel Bilgiler #

Değişken işlevler, işlev tanımının bağımsız değişken listesindeki ... üç nokta sözdizimi kullanılarak oluşturulur.

function id(...)
    return
end

Bu işlevi ‘id(1, 2, 3, 4, 5)’ olarak çağırırsanız, ‘…’ (vararg listesi olarak da bilinir) ‘1, 2, 3, 4, 5’ değerlerini içerir.

İşlevler gerekli argümanları ve ... alabilir.

function head(x, ...)
    return x
end

Vararg listesinden eleman çekmenin en kolay yolu, sadece değişkenleri listeden atamaktır.

function head3(...)
    local a, b, c = ...
    return a, b, c
end

select(), öğelerin sayısını bulmak ve dolaylı olarak ... öğesinden öğeleri çıkarmak için de kullanılabilir.

function my_print(...)
    for i = 1, select('#', ...) do
        io.write(tostring(select(i, ...)) .. '\t')
    end
    io.write '\n'
end

..., kullanım kolaylığı için {...} kullanılarak bir tabloya paketlenebilir. Bu, tüm argümanları tablonun sıralı bölümüne yerleştirir.

table.pack(...) aynı zamanda vararg listesini bir tabloya paketlemek için de kullanılabilir. table.pack(...) öğesinin avantajı, döndürülen tablonun n alanını select('#', ...) değerine ayarlamasıdır. Argüman listeniz sıfır içeriyorsa bu önemlidir (aşağıdaki açıklamalar bölümüne bakın).

son sürüm 
function my_tablepack(...)
    local t = {...}
    t.n = select('#', ...)
    return t
end

Vararg listesi ayrıca işlevlerden döndürülebilir. Sonuç, çoklu dönüşlerdir.

function all_or_none(...)
    local t = table.pack(...)
    for i = 1, t.n do
        if not t[i] then
            return    -- return none
        end
    end
    return ...    -- return all
end

Gelişmiş Kullanım #

Temel örneklerde belirtildiği gibi, değişken bağlı argümanlara ve değişken argüman listesine (...) sahip olabilirsiniz. Bu gerçeği, diğer dillerde (Haskell gibi) yaptığınız gibi yinelemeli olarak bir listeyi ayırmak için kullanabilirsiniz. Aşağıda, bundan yararlanan bir foldr() uygulaması bulunmaktadır. Her özyinelemeli çağrı, vararg listesinin başını ‘x’e bağlar ve listenin geri kalanını yinelemeli bir çağrıya iletir. Bu, yalnızca bir argüman (select('#', ...) == 0) olana kadar listeyi yok eder. Bundan sonra, her değer, daha önce hesaplanan sonuçla birlikte ‘f’ işlev argümanına uygulanır.

function foldr(f, ...)
    if select('#', ...) < 2 then return ... end
    local function helper(x, ...)
        if select('#', ...) == 0 then
          return x
        end
        return f(x, helper(...))
    end
    return helper(...)
end

function sum(a, b)
    return a + b
end

foldr(sum, 1, 2, 3, 4)
--> 10

Bu programlama stilinden yararlanan diğer işlev tanımlarını burada Sayı #3 ila Sayı #8 arasında bulabilirsiniz.

Lua’nın tek deyimsel veri yapısı tablodur. Bir dizide herhangi bir yerde “nil” varsa, tablo uzunluğu operatörü tanımsızdır. Tablolardan farklı olarak, vararg listesi, temel örneklerde ve açıklamalar bölümünde belirtildiği gibi açık nillere uyar (henüz okumadıysanız lütfen o bölümü okuyun). Az bir çalışma ile vararg listesi, mutasyon dışında bir tablonun yapabileceği her işlemi yapabilir. Bu, vararg listesini değişmez demetleri uygulamak için iyi bir aday yapar.

function tuple(...)
    -- packages a vararg list into an easily passable value
    local co = coroutine.wrap(function(...)
        coroutine.yield()
        while true do
            coroutine.yield(...)
        end
    end)
    co(...)
    return co
end

local t = tuple((function() return 1, 2, nil, 4, 5 end)())

print(t())                 --> 1    2    nil    4    5    | easily unpack for multiple args
local a, b, d = t()        --> a = 1, b = 2, c = nil      | destructure the tuple
print((select(4, t())))    --> 4                          | index the tuple
print(select('#', t()))    --> 5                          | find the tuple arity (nil respecting)

local function change_index(tpl, i, v)
    -- sets a value at an index in a tuple (non-mutating)
    local function helper(n, x, ...)
        if select('#', ...) == 0 then
            if n == i then
                return v
            else
                return x
            end
        else
            if n == i then
                return v, helper(n+1, ...)
            else
                return x, helper(n+1, ...)
            end
        end
    end
    return tuple(helper(1, tpl()))
end

local n = change_index(t, 3, 3)
print(t())                 --> 1    2    nil    4    5
print(n())                 --> 1    2    3    4    5

Yukarıdakiler ile tablolar arasındaki temel fark, tabloların değişken olması ve işaretçi semantiğine sahip olmasıdır, burada demet bu özelliklere sahip değildir. Ek olarak, demetler açık “nil"leri tutabilir ve hiçbir zaman tanımsız uzunluk işlemine sahip olabilir.
