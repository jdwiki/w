Sözdizimi #

  • [[local] mt = ]getmetatable(t) –> ‘t’ için ilişkili metatabloyu al
  • [[local] t = ]setmetatable(t, mt) –> ‘t’ için metatabloyu ‘mt’ olarak ayarlar ve ‘t’ değerini döndürür

Parametreler #

parametre Ayrıntılar
t Bir lua tablosuna atıfta bulunan değişken; tablo değişmezi de olabilir.
mt Metatablo olarak kullanılacak tablo; sıfır veya daha fazla metametod alanı ayarlanmış olabilir.

Burada bahsedilmeyen bazı metametotlar vardır. Tam liste ve kullanımları için lua manual içindeki ilgili girişe bakın.

OOP simülasyonu #

Metatabloların oluşturulması ve kullanımı #

Bir metatablo, bir lua nesnesinin davranışını değiştiren bir dizi işlemi tanımlar. Metatable, özel bir şekilde kullanılan sıradan bir tablodur.

local meta = { } -- create a table for use as metatable

-- a metatable can change the behaviour of many things
-- here we modify the 'tostring' operation:
-- this fields should be a function with one argument.
-- it gets called with the respective object and should return a string
meta.__tostring = function (object)
    return string.format("{ %d, %d }", object.x, object.y)
end

-- create an object
local point = { x = 13, y = -2 }
-- set the metatable
setmetatable(point, meta)

-- since 'print' calls 'tostring', we can use it directly:
print(point) -- prints '{ 13, -2 }'

Çöp toplayıcı - __gc metamethod #

Lua’daki nesneler çöp toplanır. Bazen, bir nesne yok edildiğinde (toplandığında) bazı kaynakları boşaltmanız, bir mesaj yazdırmak veya başka bir şey yapmak istemeniz gerekir. Bunun için nesne yok edildiğinde argüman olarak nesne ile birlikte çağrılan __gc metametodunu kullanabilirsiniz. Bu metametodunu bir tür yıkıcı olarak görebilirsiniz.

Bu örnek, eylem halindeki ‘__gc’ metametodunu gösterir. ’t’ye atanan iç tablo çöp toplandığında, toplanmadan önce bir mesaj yazdırır. Aynı şekilde, komut dosyasının sonuna ulaşıldığında dış tablo için:

local meta =
{
    __gc = function(self)
        print("destroying self: " .. self.name)
    end
}

local t = setmetatable({ name = "outer" }, meta)
do
    local t = { name = "inner" }
    setmetatable(t, meta)
end
son sürüm

Tabloların indekslenmesi #

Meta tabloların belki de en önemli kullanımı, tabloların indekslenmesini değiştirme olasılığıdır. Bunun için dikkate alınması gereken iki eylem vardır: içeriği okumak ve tablonun içeriğini yazmak. Her iki eylemin de yalnızca ilgili anahtar tabloda yoksa tetiklendiğini unutmayın.

Okuma #

local meta = {}

-- to change the reading action, we need to set the '__index' method
-- it gets called with the corresponding table and the used key
-- this means that table[key] translates into meta.__index(table, key)
meta.__index = function(object, index)
    -- print a warning and return a dummy object
    print(string.format("the key '%s' is not present in object '%s'", index, object))
    return -1
end

-- create a testobject
local t = {}

-- set the metatable
setmetatable(t, meta)

print(t["foo"]) -- read a non-existent key, prints the message and returns -1

Bu, var olmayan bir anahtarı okurken bir hata oluşturmak için kullanılabilir:

-- raise an error upon reading a non-existent key
meta.__index = function(object, index)
    error(string.format("the key '%s' is not present in object '%s'", index, object))
end

Yazı #

local meta = {}

-- to change the writing action, we need to set the '__newindex' method
-- it gets called with the corresponding table, the used key and the value
-- this means that table[key] = value translates into meta.__newindex(table, key, value)
meta.__newindex = function(object, index, value)
    print(string.format("writing the value '%s' to the object '%s' at the key '%s'",
                         value, object, index))
    --object[index] = value -- we can't do this, see below
end

-- create a testobject
local t = { }

-- set the metatable
setmetatable(t, meta)

-- write a key (this triggers the method)
t.foo = 42

Şimdi kendinize gerçek değerin tabloda nasıl yazıldığını sorabilirsiniz. Bu durumda, değil. Buradaki sorun, metametotların, sonsuz bir döngü veya daha kesin olarak bir yığın taşması ile sonuçlanacak olan metametotları tetikleyebilmesidir. Peki bunu nasıl çözebiliriz? Bunun çözümüne ham tablo erişimi denir.

Tabloları metametot olarak kullanma #

Bazı metametotların işlev olması gerekmez. Bunun en önemli örneği __index metametodudur. Daha sonra arama olarak kullanılan bir tablo da olabilir. Bu, lua’da sınıfların oluşturulmasında oldukça yaygın olarak kullanılır. Burada, sınıfın tüm işlemlerini (yöntemlerini) tutmak için bir tablo (genellikle metatablonun kendisi) kullanılır:

local meta = {}
-- set the __index method to the metatable.
-- Note that this can't be done in the constructor!
meta.__index = meta

function create_new(name)
    local self = { name = name }
    setmetatable(self, meta)
    return self
end

-- define a print function, which is stored in the metatable
function meta.print(self)
    print(self.name)
end

local obj = create_new("Hello from object")
obj:print()

Daha fazla metametot #

Daha birçok metametot vardır, bazıları aritmetiktir (örneğin toplama, çıkarma, çarpma), bitsel işlemler (ve, veya, xor, kaydırma), karşılaştırma (<, >) ve ayrıca == ve # gibi temel tip işlemler vardır. (eşitlik ve uzunluk). Bu işlemlerin çoğunu destekleyen bir sınıf oluşturalım: rasyonel aritmetik çağrısı. Bu çok temel olsa da, fikri gösterir.

local meta = {
    -- string representation
    __tostring = function(self)
        return string.format("%s/%s", self.num, self.den)
    end,
    -- addition of two rationals
    __add = function(self, rhs)
        local num = self.num * rhs.den + rhs.num * self.den
        local den = self.den * rhs.den
        return new_rational(num, den)
    end,
    -- equality
    __eq = function(self, rhs)
        return self.num == rhs.num and self.den == rhs.den
    end
}

-- a function for the creation of new rationals
function new_rational(num, den)
    local self = { num = num, den = den }
    setmetatable(self, meta)

    return self
end

local r1 = new_rational(1, 2)
print(r1) -- 1/2

local r2 = new_rational(1, 3)
print(r1 + r2) -- 5/6

local r3 = new_rational(1, 2)
print(r1 == r3) -- true
-- this would be the behaviour if we hadn't implemented the __eq metamethod.
-- this compares the actual tables, which are different
print(rawequal(r1, r3)) -- false

Tabloları çağrılabilir hale getirin #

İşlev olarak kullanıldığında nesnenin davranışını tanımlayan ‘__call’ adında bir metametot vardır, örn. “nesne()”. Bu, işlev nesneleri oluşturmak için kullanılabilir:

-- create the metatable with a __call metamethod
local meta = {
    __call = function(self)
        self.i = self.i + 1
    end,
    -- to view the results
    __tostring = function(self)
        return tostring(self.i)
    end
}

function new_counter(start)
    local self = { i = start }
    setmetatable(self, meta)
    return self
end

-- create a counter
local c = new_counter(1)
print(c) --> 1
-- call -> count up
c()
print(c) --> 2

Metametod, karşılık gelen nesne ile çağrılır, kalan tüm argümanlar bundan sonra fonksiyona iletilir:

local meta = {
    __call = function(self, ...)
        print(self.prepend, ...)
    end
}

local self = { prepend = "printer:" }
setmetatable(self, meta)

self("foo", "bar", "baz")

Ham tablo erişimi #

Bazen, metametotları tetiklemek istemezsiniz, ancak erişime sarılmış bazı akıllı işlevler olmadan, tam olarak verilen anahtarı gerçekten yazar veya okursunuz. Bunun için lua size ham tablo erişim yöntemleri sağlar:

-- first, set up a metatable that allows no read/write access
local meta = {
    __index = function(object, index)
        -- raise an error
        error(string.format("the key '%s' is not present in object '%s'", index, object))
    end,
    __newindex = function(object, index, value)
        -- raise an error, this prevents any write access to the table
        error(string.format("you are not allowed to write the object '%s'", object))
    end
}

local t = { foo = "bar" }
setmetatable(t, meta)

-- both lines raise an error:
--print(t[1])
--t[1] = 42

-- we can now circumvent this problem by using raw access:
print(rawget(t, 1)) -- prints nil
rawset(t, 1, 42) -- ok

-- since the key 1 is now valid, we can use it in a normal manner:
print(t[1])

Bununla, değeri tabloya gerçekten yazmak için eski __newindex yöntemini yeniden yazabiliriz:

meta.__newindex = function(object, index, value)
    print(string.format("writing the value '%s' to the object '%s' at the key '%s'",
                         value, object, index))
    rawset(object, index, value)
end
