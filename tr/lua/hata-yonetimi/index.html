Lua’da hataları işleme #

Aşağıdaki fonksiyona sahip olduğumuzu varsayarsak:

function foo(tab)
  return tab.a
end -- Script execution errors out w/ a stacktrace when tab is not a table

biraz geliştirelim

function foo(tab)
  if type(tab) ~= "table" then
    error("Argument 1 is not a table!", 2)
  end
  return tab.a
end -- This gives us more information, but script will still error out

Hata durumunda bile bir fonksiyonun bir programı çökertmesini istemiyorsak, lua’da aşağıdakileri yapmak standarttır:

function foo(tab)
    if type(tab) ~= "table" then return nil, "Argument 1 is not a table!" end
    return tab.a
end -- This never crashes the program, but simply returns nil and an error message

Şimdi böyle davranan bir fonksiyonumuz var, şöyle şeyler yapabiliriz:

if foo(20) then print(foo(20)) end -- prints nothing
result, error = foo(20)
if result then print(result) else log(error) end

Ve eğer bir şeyler ters giderse programın çökmesini istiyorsak, yine de bunu yapabiliriz:

result, error = foo(20)
if not result then error(error) end

Neyse ki tüm bunları her seferinde yazmak zorunda değiliz; lua’nın tam olarak bunu yapan bir işlevi var

result = assert(foo(20))

pcall’ı kullanma #

“pcall”, “korumalı çağrı” anlamına gelir. Fonksiyonlara hata işleme eklemek için kullanılır. “pcall”, diğer dillerdeki “try-catch” ile benzer şekilde çalışır. ‘pcall’ın avantajı, ‘pcall’ ile çağrılan işlevlerde hatalar meydana geldiğinde betiğin tüm yürütmesinin kesintiye uğramamasıdır. “pcall” ile çağrılan bir fonksiyon içinde bir hata meydana gelirse, bir hata atılır ve kodun geri kalanı çalışmaya devam eder.

Sözdizimi:

pcall( f , arg1,···)

Dönüş Değerleri:

iki değer döndürür

  1. durum (boole)
  • İşlev hatasız bir şekilde yürütüldüyse true döndürür.
  • İşlev içinde bir hata meydana gelirse yanlış döndürür.
  1. fonksiyon bloğunun içinde bir hata meydana gelirse fonksiyonun veya hata mesajının değerini döndürür.

pcall çeşitli durumlar için kullanılabilir, ancak yaygın olanı, işlevinize verilen işlevdeki hataları yakalamaktır. Örneğin, bu fonksiyonumuz olduğunu varsayalım:

local function executeFunction(funcArg, times) then
    for i = 1, times do
        local ran, errorMsg = pcall( funcArg )
        if not ran then
            error("Function errored on run " .. tostring(i) .. "\n" .. errorMsg)
        end
    end
end
  1. çalıştırmada verilen fonksiyon hataları olduğunda, hata mesajı kullanıcıya bunun sizin fonksiyonunuzdan değil, bizim fonksiyonumuza verilen fonksiyondan geldiği anlaşılır olacaktır. Ayrıca, bu akılda tutularak, kullanıcıyı bilgilendiren süslü bir BSoD yapılabilir. Ancak, bir API büyük olasılıkla bunu yapmayacağından, bu işlevi uygulayan uygulamaya bağlıdır.

Örnek A - PCall olmadan yürütme

function square(a)
  return a * "a"    --This will stop the execution of the code and throws an error, because of the attempt to perform arithmetic on a string value
end

square(10);

print ("Hello World")    -- This is not being executed because the script was interrupted due to the error

Örnek B - PCall ile yürütme

function square(a)
  return a * "a"
end

local status, retval = pcall(square,10);

print ("Status: ", status)        -- will print "false" because an error was thrown.
print ("Return Value: ", retval)  -- will print "input:2: attempt to perform arithmetic on a string value"
print ("Hello World")    -- Prints "Hello World"

Örnek - Kusursuz kodun yürütülmesi

function square(a)
  return a * a
end

local status, retval = pcall(square,10);

print ("Status: ", status)        -- will print "true" because no errors were thrown 
print ("Return Value: ", retval)  -- will print "100"
print ("Hello World")    -- Prints "Hello World"
