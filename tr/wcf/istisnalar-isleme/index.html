Özel bir hata günlüğü çerçevesi kullanma #

Tüm istisnaların günlüğe kaydedildiğinden emin olmak için özel bir hata günlüğü çerçevesini entegre etmek bazen yararlıdır.

[ServiceContract]
[ErrorHandler]
public interface IMyService
{
       
}

[AttributeUsage(AttributeTargets.Interface)]
public class CustomErrorHandler : Attribute, IContractBehavior, IErrorHandler
{
    public bool HandleError(Exception error)
    {
        return false;
    }

    public void ProvideFault(Exception error, MessageVersion version, ref Message fault)
    {
        if (error == null)
        {
            return;
        }

        //my custom logging framework
    }

    public void ApplyDispatchBehavior(ContractDescription contractDescription, ServiceEndpoint endpoint, DispatchRuntime dispatchRuntime)
    {
        dispatchRuntime.ChannelDispatcher.ErrorHandlers.Add(this);
    }

    public void ApplyClientBehavior(ContractDescription contractDescription, ServiceEndpoint endpoint,
        ClientRuntime clientRuntime)
    {

    }

    public void AddBindingParameters(ContractDescription contractDescription, ServiceEndpoint endpoint,
        BindingParameterCollection bindingParameters)
    {

    }

    public void Validate(ContractDescription contractDescription, ServiceEndpoint endpoint)
    {

    }
}

Bir istisna oluştuğunda ek bilgi gösterme #

Hizmetinizde istisnaları ele almak önemlidir. Hizmeti geliştirirken, WCF’yi daha ayrıntılı bilgi sağlayacak şekilde ayarlayabilir ve bu etiketi genellikle Web.config olmak üzere yapılandırma dosyasına ekleyebilirsiniz:

 <serviceDebug includeExceptionDetailInFaults="true"/>

Bu etiket, genellikle şu şekilde, serviceBehavior etiketinin içine yerleştirilmelidir:

<system.serviceModel>
    <behaviors>
      <serviceBehaviors>
        <behavior>
          <serviceDebug includeExceptionDetailInFaults="true"/>
        </behavior>
      </serviceBehaviors>
    </behaviors>
</system.serviceModel>

Ayrıntılı bilgi örneği:

Sunucu yığını izleme: em System.ServiceModel.Channels.ServiceChannel.HandleReply(ProxyOperationRuntime işlem, ProxyRpc ve rpc) em System.ServiceModel.Channels.ServiceChannel.Call(Dize eylemi, Boolean oneway, ProxyOperationRuntime işlemi, Object[] ins, Nesne[] çıkışları, TimeSpan zaman aşımı) em System.ServiceModel.Channels.ServiceChannelProxy.InvokeService(IMethodCallMessage methodCall, ProxyOperationRuntime işlemi) em System.ServiceModel.Channels.ServiceChannelProxy.Invoke(IMessage mesaj)

[0]‘da yeniden oluşturulan istisna: em System.Runtime.Remoting.Proxies.RealProxy.HandleReturnMessage(IMessage reqMsg, IMessage retMsg) em System.Runtime.Remoting.Proxies.RealProxy.PrivateInvoke(MessageData& msgData, Int32 tipi) em IMyService.GetDataOperation(RequestObterBeneficiario istek) em MyServiceClient.GetDataOpration(RequestData istek)

Bu, müşteriye ayrıntılı bilgilere geri dönecektir. Geliştirme sırasında bu yardımcı olabilir, ancak hizmetiniz üretime geçtiğinde, hizmetiniz veritabanı adınız veya yapılandırmanız gibi hassas veriler gönderebileceğinden artık bunu saklamayacaksınız.

FaultException ile kullanıcı dostu bir mesaj atma #

İstisnaları işleyebilir ve bir FaultException atan ‘Kullanılamayan hizmet’ gibi en dostça bir mesaj atabilirsiniz:

try
{
    // your service logic here
}
catch (Exception ex)
{
    // You can do something here, like log the original exception
    throw new FaultException("There was a problem processing your request");
}

İstemcinizde şu mesajı kolayca alabilirsiniz:

try
{
    // call the service
}
catch (FaultException faultEx)
{
   var errorMessage = faultEx.Message;
   // do something with error message
}

İşlenen istisnayı, kodunuza başka yakalamalar ekleyerek ağ hatası gibi diğer istisnalardan ayırt edebilirsiniz:

try
{
    // call the service
}
catch (FaultException faultEx)
{
   var errorMessage = faultEx.Message;
   // do something with error message
}
catch (CommunicationException commProblem)
{
    // Handle the communication error, like trying another endpoint service or logging
}

Hata Koduyla FaultException Atma #

FaultException ayrıca, istemcinin farklı istisnaları ayırt edebilmesi için ek bilgileri iletmek için kullanabileceğiniz bir dize verisi olan bir FaultCode içerebilir:

try
{
    // your service logic here
}
catch (Exception ex)
{
   throw new FaultException("There was a problem processing your request",
      new FaultCode(("01"));
}

FaultCode’u Alma:

try
{
    // call the service
}
catch (FaultException faultEx)
{
   switch (faultEx.Code.Name)
   {
    case "01":
       // do something
       break;
    case "02":
        // do another something
        break
   }
}

Decoupling ErrorHandlerAttribute hizmet uygulamasından kayıt oluyor #

Hizmetleriniz için tüm için aynı hata günlüğü kodunu ayırmak ve yeniden kullanmak için iki ortak sınıfa ihtiyacınız var ve bunları bir yerde bir kitaplıkta saklayın.

ErrorhandlerAttribute, IServiceBehavior’u uyguluyor. Tüm istisnaları günlüğe kaydeden IErrorhandler uygulayan FaultErrorhandler.

[AttributeUsage(AttributeTargets.Class)]
public class ErrorHandlerAttribute : Attribute, IServiceBehavior
{
    Type mErrorType;
    public ErrorHandlerAttribute(Type t)
    {
        if (!typeof(IErrorHandler).IsAssignableFrom(t))
            throw new ArgumentException("Type passed to ErrorHandlerAttribute constructor must inherit from IErrorHandler");
        mErrorType = t;
    }

    public void AddBindingParameters(ServiceDescription serviceDescription, ServiceHostBase serviceHostBase, Collection<ServiceEndpoint> endpoints, BindingParameterCollection bindingParameters)
    {
    }

    public void ApplyDispatchBehavior(ServiceDescription serviceDescription, ServiceHostBase serviceHostBase)
    {
        foreach (ChannelDispatcher dispatcher in serviceHostBase.ChannelDispatchers)
        {
            dispatcher.ErrorHandlers.Add((IErrorHandler)Activator.CreateInstance(mErrorType));
        }
    }

    public void Validate(ServiceDescription serviceDescription, ServiceHostBase serviceHostBase)
    {
    }
}


class FaultErrorHandler : IErrorHandler
{
    public bool HandleError(Exception error)
    {
        // LOG ERROR
        return false; // false so the session gets aborted
    }

    public void ProvideFault(Exception error, MessageVersion version, ref Message fault)
    {
    }
}

Ardından, hizmet uygulamanızda, FaultErrorHandler’ın Type örneğinde geçen ErrorHandler Özniteliğini ekleyin. ErrorHandler, HandleError’ın çağrıldığı türden bir örnek oluşturacaktır.

[ServiceBehavior]
[ErrorHandler(typeof(FaultErrorHandler))]
public class MyService : IMyService
{
}
