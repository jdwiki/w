Bağımlılık Enjeksiyon Konteyneri (Castle Windsor) Kullanacak Bir WCF Hizmeti Nasıl Yapılandırılır #

Bu örneğin iki bölümü vardır - Windsor Kalesi’ni WCF hizmetinize eklemek için bazı temel adımlar ve ardından Windsor’un kapsayıcısını nasıl yapılandırdığımızı ve kullandığımızı gösteren basit, somut bir örnek.

Bu da örneği biraz uzun kılıyor. Bir DI kabı kullanmayı zaten anlıyorsanız, muhtemelen yalnızca standart adım adımlarını önemsiyorsunuzdur. Bir DI kapsayıcı kullanmak tanıdık değilse, mantıklı olması için biraz daha fazla zaman alır - her şeyin uçtan uca çalıştığını görmek - mantıklı.

Boilerplate Adımları

  1. Castle Windsor WCF entegrasyon tesisi Nuget paketini WCF hizmet uygulamanıza ekleyin. Bu, Castle Windsor’a ve özellikle WCF hizmetleri için bazı bileşenlere referanslar ekleyecektir.

  2. Projenize bir Global Application Class (global.asax) ekleyin: Add > New Item > Visual C# > Web > Global Application Class. Kapsayıcınızı yapılandıran kod, ‘Application_Start’ yönteminden çağrılmalıdır. Düzenli tutmak için tüm konteyner konfigürasyonumuzu ayrı bir sınıfa koyabiliriz. zorunda değiliz. Önemli değil. Bunun farklı örneklerde farklı yapıldığını göreceksiniz. Önemli olan bunun ‘Application_Start’ yönteminden çağrılmasıdır, çünkü bu sizin kompozisyon kökünüz - uygulamanın başladığı yerdir. Buradaki fikir, uygulama başladığında kapsayıcınızı yapılandırmak ve ardından bir daha asla doğrudan dokunmamaktır. Sadece işini yaparken arka planda kalır.

  3. Kapsayıcıyı yapılandırmak için bir sınıf oluşturun. Bu iki şey yapar:

    • ContainerInstance.AddFacility<WcfFacility>() tells Windsor’s WCF code to use this particular container when creating instances of your WCF services.
    • ContainerInstance.Install(FromAssembly.This()) tells Windsor to scan This assembly (in other words, your WCF project) looking for classes that implement IWindsorInstaller. Those classes will provide instructions to tell your container how to resolve dependencies. (We’ll create one a few steps later.)

    public static class Container { private static readonly IWindsorContainer ContainerInstance = new WindsorContainer();

     public static void Configure()
 {
     ContainerInstance.AddFacility<WcfFacility>();
     ContainerInstance.Install(FromAssembly.This());
 }

    }

  4. Bu yöntemi global.asax’ınızdaki ‘Application_Start’tan çağırın:

    public class Global : System.Web.HttpApplication { protected void Application_Start(object sender, EventArgs e) { Container.Configure(); } }

  5. Bir yükleyici oluşturun. Bu sadece “IWindsorInstaller"ı uygulayan bir sınıftır. Bu boş. Hiçbir şey yapmıyor. Birkaç adımda bu sınıfa ekleyeceğiz. ContainerInstance.Install(FromAssembly.This()) çağırdığımızda, konteyner ile bağımlılıkları kaydedebilmemiz için ContainerInstance Install yöntemine geçirilir.

    public class WindsorInstaller : IWindsorInstaller { public void Install(IWindsorContainer container, IConfigurationStore store) { // Nothing here yet! } }

  6. Oluşturduğunuz herhangi bir WCF hizmetinin işaretlemesine bu yönergeyi ekleyin. Bu, uygulamanın hizmetin örneklerini oluşturmak için Windsor’un WCF tesisini kullanacağını gösterir; bu da, bir örnek oluşturulduğunda bağımlılıkları enjekte edebileceği anlamına gelir.

    Factory=“Castle.Facilities.WcfIntegration.DefaultServiceHostFactory, Castle.Facilities.WcfIntegration”

Bu, hizmeti bir Castle Windsor bağımlılık enjeksiyon konteyneri kullanacak şekilde ayarlamak için standart adım adımlarını sona erdirir.

Ancak, bağımlılık enjeksiyonu için en az bir WCF hizmeti yapılandırmadıkça örnek tamamlanmaz. Bunun geri kalanı genel bilgiler değil, sadece basit, somut bir örnek.

  1. GreetingService.svc adında yeni bir WCF servisi oluşturun.

  2. İşaretlemeyi düzenleyin. Şöyle görünmelidir:

    <%@ ServiceHost Language=“C#” Debug=“true” Service=“WcfWindsorDocumentation.GreetingService” CodeBehind=“GreetingService.svc.cs”
    Factory=“Castle.Facilities.WcfIntegration.DefaultServiceHostFactory, Castle.Facilities.WcfIntegration” %>

  3. “IGreetingService"i (hizmet sözleşmesi) şununla değiştirin:

    [ServiceContract] public interface IGreetingService { [OperationContract] string GetGreeting(); }

  4. GreetingService (Tebrik Hizmeti.svc.cs içinde) bu kodla değiştirin. Yapıcının, enjekte etmek için kapsayıcımıza ihtiyaç duyacağımız bir “IGreetingProvider” örneğini gerektirdiğine dikkat edin.

    public class GreetingService : IGreetingService { private readonly IGreetingProvider _greetingProvider;

     public GreetingService(IGreetingProvider greetingProvider)
 {
     _greetingProvider = greetingProvider;
 }

 public string GetGreeting()
 {
     return _greetingProvider.GetGreeting();
 }

    }

  5. Bu IGreetingProvider uygulamasını ekleyin. Ayrıca, tedarik etmek için konteynere ihtiyaç duyacağımız kendine ait birkaç bağımlılığı da var. Bu sınıfların özellikleri çok önemli değil. Sadece takip edilmesi kolay bir şey yaratmak için.

    public interface IGreetingProvider { string GetGreeting(); }

    public interface IComputerNameProvider { string GetComputerName(); }

    public class ComputerNameGreetingProvider : IGreetingProvider { private readonly IComputerNameProvider _computerNameProvider;

     public ComputerNameGreetingProvider(IComputerNameProvider computerNameProvider)
 {
     _computerNameProvider = computerNameProvider;
 }

 public string GetGreeting()
 {
     return string.Concat("Hello from ", _computerNameProvider.GetComputerName());
 }

    }

    public class EnvironmentComputerNameProvider : IComputerNameProvider { public string GetComputerName() { return System.Environment.MachineName; } }

  6. Artık ihtiyacımız olan tüm sınıflara sahibiz. Geriye kalan tek şey, konteynerimize bağımlılıkları kaydetmek. Başka bir deyişle, konteynere hangi sınıfları yaratması gerektiğini söyleyeceğiz, böylece bir “GreetingService” örneğinin nasıl “oluşturulacağını” bilsin. Bu kod, bizim “IWindsorInstaller” uygulamamızda yer alır (ortak kod kodunun 5. adımı.)

    public class WindsorInstaller : IWindsorInstaller { public void Install(IWindsorContainer container, IConfigurationStore store) { container.Register( Component.For<IGreetingService, GreetingService>(), Component.For<IGreetingProvider, ComputerNameGreetingProvider>(), Component.For<IComputerNameProvider, EnvironmentComputerNameProvider>()); } }

Bu, konteynere şunları söyler:

  • Gerektiğinde GreetingService oluşturmaktan sorumludur.
  • “GreetingService” oluşturmaya çalıştığında bir “IGreetingProvider"a ihtiyacı olacak. Gerektiğinde bir ComputerNameGreetingProvider oluşturmalıdır.
  • ‘ComputerNameGreetingProvider’ oluşturmaya çalıştığında, bu sınıf bir ‘IComputerNameProvider’ gerektirir. Bu ihtiyacı karşılamak için bir EnvironmentComputerNameProvider örneği oluşturmalıdır.

“GreetingService” veya bağımlılıklarından birini oluşturma sürecinde bir yerde, kaydetmediğimiz bir bağımlılık gereksinimiyle karşılaşırsa, bunun gibi yararlı bir istisna ile bize bildirecektir.

Eksik bağımlılık. WcfWindsorDocumentation.ComputerNameGreetingProvider bileşeninin bir bağımlılığı var WcfWindsorDocumentation.IComputerNameProvider üzerinde çözüldü. Bağımlılığın bir hizmet olarak kapsayıcıya doğru şekilde kaydedildiğinden veya satır içi bağımsız değişken olarak sağlandığından emin olun.

Bu, bir şeyin “IComputerNameProvider"a bağlı olduğu, ancak bu bağımlılığı yerine getirmek için kayıtlı hiçbir şey olmadığı anlamına gelir.

Bu sadece işin bitmesini sağlar. Gerçek dünya senaryoları için bir bağımlılık enjeksiyon kapsayıcısını doğru şekilde yapılandırmak ve kullanmak için çok daha fazlası vardır. Bu örnek, yalnızca Windsor’u bir WCF hizmet uygulamasına eklemeye özgü olanları kapsar. Autofac veya Unity gibi farklı bir kap kullanırsanız, sözdizimi ve ayrıntılar değişse de, prensipte aynı şeyleri yaptıklarını göreceksiniz ve benzerlikleri kolayca fark edeceksiniz.
