UWP için Birim Testi

  • Test Uygulamasını Yapılandır

  • Bazı işlevlerle alay edin

Bu sayfada

  • Bazı işlevlerle alay edin

Evrensel Windows 10 Uygulaması için Birim Testlerinin nasıl oluşturulacağını göstermek istiyorum. UWP uygulamalarını test etmek için, açıklamalar bölümünde verdiğim bağlantıdan daha fazlasını okuyabileceğiniz xUnit.net Framework kullanacağız.

xUnit Framework hakkında daha fazla bilgi edinebilirsiniz: https://xunit.github.io/docs/getting-started-uwp.html

Test Uygulamasını Yapılandır #

UWP uygulamanızı testlere hazır hale getirdikten sonra, çözümünüze test uygulamasını eklemelisiniz. Bunu yapmak için “sağ” çözüme tıklayın ve “Birim Test Uygulaması (Evrensel Windows)” seçeneğini seçin:

Çözüme ekledikten sonra, yapılandırmak için gereken birkaç adım daha vardır. Hedef ve minimum platform sürümünü seçmeniz istenecektir:

Bunları seçtikten sonra “project.json” dosyasını açın ve aşağıdaki bağımlılıkları ekleyin:

"dependencies": 
 {
  "Microsoft.NETCore.UniversalWindowsPlatform": "5.1.0",
  "xunit.runner.visualstudio": "2.1.0",
  "xunit": "2.1.0",
  "xunit.runner.devices": "2.1.0"
 }

Bunlar, UWP uygulaması için birim testlerini kolaylaştırmak amacıyla NuGet xUnit Framework paketlerini indirmek ve eklemek için kullanılır.

“MSTestFramework.Universal” adlı referansı kaldırın:

Şimdi “UnitTest.cs” dosyasını açın. Aşağıdaki gibi görünecek şekilde değiştirin:

using System;
using Xunit;

namespace UnitTestsForUwp
{
  public class UnitTest1
   {
       [Fact]
       public void TestMethod1()
        {
          Assert.Equal(4, 4);
        }

       [Theory]
       [InlineData(6)]
       public void TestMethod2(int value)
        {
          Assert.True(IsOdd(value));
        }

       bool IsOdd(int value)
        {
          return value % 2 == 1;
        }
     }
   }
}

xUnit öznitelikleri hakkında biraz konuşmak için burada durmakta fayda var:

a. Her zaman doğru olan olgu testleri. Değişmez koşulları test ederler.

b. Teori – yalnızca belirli bir veri kümesi için doğru olan testler.

Şimdi, uygulamayı testler hakkındaki bilgileri gösterecek şekilde hazırlamak istiyoruz, aynı zamanda testleri başlatmak için tek bir iyi yola sahip olmak iyidir. Bunu başarmak için “UnitTestApp.xaml” dosyasında küçük değişiklikler yapmamız gerekiyor. Açın ve tüm kodu aşağıya yapıştırılanlarla değiştirin:

<ui:RunnerApplication
 x:Class="UnitTestsForUwp.App"
 xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
 xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
 xmlns:local="using:UnitTestsForUwp"
 xmlns:ui = "using:Xunit.Runners.UI"
 RequestedTheme="Light">
</ui:RunnerApplication>

“Yerel” ifadesinin ad alanınızla aynı ada sahip olması gerektiğini unutmayın.

Şimdi “UnitTestApp.xaml.cs” dosyasını açın ve kodu aşağıdakiyle değiştirin:

sealed partial class App : RunnerApplication
 {
   protected override void OnInitializeRunner()
    {
      AddTestAssembly(GetType().GetTypeInfo().Assembly);
      InitializeRunner();
    }
    partial void InitializeRunner();
  }

Bu kadar! Şimdi projeyi yeniden oluşturun ve test uygulamasını başlatın. Aşağıda görebileceğiniz gibi tüm testlerinize erişebilirsiniz, bunları başlatabilir ve sonuçları kontrol edebilirsiniz:

Test Uygulamasını hedef uygulama koduyla bağlayın #

Test uygulamanız hazır olduğunda, birim testleri yazmak istediğiniz kodla bağlayabilirsiniz.

PCL’de veya UWP uygulama projesinde (MVVM desenini uyguladığınızı varsayıyorum) kodunuz var, Test Uygulaması projesinde buna referans ekleyin:

Artık Test Uygulamasından tüm kodunuza erişebilirsiniz. İstediğiniz Birim Testlerini oluşturun. Sadece “Gerçekler” veya “Teoriler” kullanın.

Bazı işlevlerle alay edin #

Birim Testlerinizi yazmak için her şeyi hazırladıktan sonra alay etmekten bahsetmeye değer. “SimpleStubs” adında yeni bir çerçeve var, bu da arayüzlere dayalı sahte oluşturmanıza olanak sağlıyor.

GitHub belgelerinden basit durum:

//Sample interface:
public interface IPhoneBook
 {
   long GetContactPhoneNumber(string firstName, string lastName);
   long MyNumber { get; set; }
   event EventHandler<long> PhoneNumberChanged;
 }

//Mocked interface:
var stub = new StubIPhoneBook().GetContactPhoneNumber((firstName, lastName) => 6041234567);

Bununla ilgili daha fazla bilgiyi buradan okuyabilirsiniz: https://github.com/Microsoft/SimpleStubs
