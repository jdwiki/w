Paralel azaltma algoritması tipik olarak, bir dizi öğeyi birleştiren ve tek bir sonuç üreten bir algoritmayı ifade eder. Bu kategoriye giren tipik sorunlar şunlardır:

  • bir dizideki tüm öğeleri toplamak
  • bir dizide maksimum bulma

Genel olarak, paralel indirgeme herhangi bir ikili ilişkisel operatör için uygulanabilir, yani (A*B)*C = A*(B*C). Böyle bir operatör * ile, paralel indirgeme algoritması, dizi argümanlarını çiftler halinde tekrar tekrar gruplandırır. Her çift, diğerleriyle paralel olarak hesaplanır ve bir adımda toplam dizi boyutunu yarıya indirir. İşlem sadece tek bir eleman var olana kadar tekrarlanır.

Operatör, ilişkisel olmasının yanı sıra değişmeli (yani, A*B = B*A) ise, algoritma farklı bir düzende eşlenebilir. Teorik açıdan hiçbir fark yaratmaz, ancak pratikte daha iyi bir bellek erişim modeli sağlar:

Tüm ilişkisel operatörler değişmeli değildir - örneğin matris çarpımını alın.

Değişmeli operatör için tek blok paralel indirgeme #

CUDA’da paralel indirgemeye en basit yaklaşım, görevi gerçekleştirmek için tek bir blok atamaktır:

static const int arraySize = 10000;
static const int blockSize = 1024;

__global__ void sumCommSingleBlock(const int *a, int *out) {
    int idx = threadIdx.x;
    int sum = 0;
    for (int i = idx; i < arraySize; i += blockSize)
        sum += a[i];
    __shared__ int r[blockSize];
    r[idx] = sum;
    __syncthreads();
    for (int size = blockSize/2; size>0; size/=2) { //uniform
        if (idx<size)
            r[idx] += r[idx+size];
        __syncthreads();
    }
    if (idx == 0)
        *out = r[0];
}

...

sumCommSingleBlock<<<1, blockSize>>>(dev_a, dev_out);

Bu, veri boyutu çok büyük olmadığında (birkaç bin öğe civarında) en uygunudur. Bu genellikle, azaltma daha büyük bir CUDA programının parçası olduğunda olur. Giriş, en baştan ‘blockSize’ ile eşleşirse, ilk ‘for’ döngüsü tamamen kaldırılabilir.

İlk adımda, ipliklerden daha fazla öğe olduğunda, şeyleri tamamen bağımsız olarak eklediğimizi unutmayın. Yalnızca sorun “blockSize” değerine düşürüldüğünde, gerçek paralel azaltma tetiklenir. Aynı kod, çarpma, minimum, maksimum, vb. gibi diğer herhangi bir değişmeli, ilişkisel operatöre uygulanabilir.

Algoritmanın, örneğin bir warp düzeyinde paralel indirgeme kullanılarak daha hızlı yapılabileceğini unutmayın.

Değişmeli olmayan operatör için tek blok paralel indirgeme #

Değişmeli olmayan bir operatör için paralel indirgeme yapmak, değişmeli versiyona kıyasla biraz daha karmaşıktır. Örnekte, basitlik adına hala tamsayılar üzerinde bir toplama kullanıyoruz. Örneğin, gerçekten değişmeli olmayan matris çarpımı ile değiştirilebilir. Bunu yaparken, 0’ın çarpmanın nötr bir öğesiyle, yani bir birim matrisiyle değiştirilmesi gerektiğini unutmayın.

static const int arraySize = 1000000;
static const int blockSize = 1024;

__global__ void sumNoncommSingleBlock(const int *gArr, int *out) {
    int thIdx = threadIdx.x;
    __shared__ int shArr[blockSize*2];
    __shared__ int offset;
    shArr[thIdx] = thIdx<arraySize ? gArr[thIdx] : 0;
    if (thIdx == 0)
        offset = blockSize;
    __syncthreads();
    while (offset < arraySize) { //uniform
        shArr[thIdx + blockSize] = thIdx+offset<arraySize ? gArr[thIdx+offset] : 0;
        __syncthreads();
        if (thIdx == 0)
            offset += blockSize;
        int sum = shArr[2*thIdx] + shArr[2*thIdx+1];
        __syncthreads();
        shArr[thIdx] = sum;
    }
    __syncthreads();
    for (int stride = 1; stride<blockSize; stride*=2) { //uniform
        int arrIdx = thIdx*stride*2;
        if (arrIdx+stride<blockSize)
            shArr[arrIdx] += shArr[arrIdx+stride];
        __syncthreads();
    }   
    if (thIdx == 0)
        *out = shArr[0];
}

...

sumNoncommSingleBlock<<<1, blockSize>>>(dev_a, dev_out);

İlk while döngüsünde, threadlerden daha fazla girdi öğesi olduğu sürece yürütülür. Her yinelemede tek bir indirgeme gerçekleştirilir ve sonuç ‘shArr’ dizisinin ilk yarısına sıkıştırılır. İkinci yarı daha sonra yeni verilerle doldurulur.

Tüm veriler gArrdan yüklendikten sonra ikinci döngü yürütülür. Artık sonucu sıkıştırmıyoruz (bu, fazladan bir __syncthreads() maliyeti). Her adımda iş parçacığı n, ‘2n’-inci aktif öğeye erişir ve onu ‘2n+1’-inci öğe ile ekler:

Bu basit örneği daha da optimize etmenin birçok yolu vardır, ör. çarpıtma düzeyinde azalma yoluyla ve paylaşılan bellek bankası çakışmalarını kaldırarak.

Değişmeli operatör için çok bloklu paralel indirgeme #

CUDA’da paralel azalmaya yönelik çok bloklu yaklaşım, tek bloklu yaklaşıma kıyasla ek bir zorluk teşkil eder, çünkü bloklar iletişimde sınırlıdır. Buradaki fikir, her bloğun giriş dizisinin bir bölümünü hesaplamasına izin vermek ve ardından tüm kısmi sonuçları birleştirmek için bir son bloğa sahip olmaktır. Bunu yapmak için, dolaylı olarak ızgara çapında bir senkronizasyon noktası oluşturarak iki çekirdek başlatılabilir.

static const int wholeArraySize = 100000000;
static const int blockSize = 1024;
static const int gridSize = 24; //this number is hardware-dependent; usually #SM*2 is a good number.

__global__ void sumCommMultiBlock(const int *gArr, int arraySize, int *gOut) {
    int thIdx = threadIdx.x;
    int gthIdx = thIdx + blockIdx.x*blockSize;
    const int gridSize = blockSize*gridDim.x;
    int sum = 0;
    for (int i = gthIdx; i < arraySize; i += gridSize)
        sum += gArr[i];
    __shared__ int shArr[blockSize];
    shArr[thIdx] = sum;
    __syncthreads();
    for (int size = blockSize/2; size>0; size/=2) { //uniform
        if (thIdx<size)
            shArr[thIdx] += shArr[thIdx+size];
        __syncthreads();
    }
    if (thIdx == 0)
        gOut[blockIdx.x] = shArr[0];
}

__host__ int sumArray(int* arr) {
    int* dev_arr;
    cudaMalloc((void**)&dev_arr, wholeArraySize * sizeof(int));
    cudaMemcpy(dev_arr, arr, wholeArraySize * sizeof(int), cudaMemcpyHostToDevice);

    int out;
    int* dev_out;
    cudaMalloc((void**)&dev_out, sizeof(int)*gridSize);
    
    sumCommMultiBlock<<<gridSize, blockSize>>>(dev_arr, wholeArraySize, dev_out);
    //dev_out now holds the partial result
    sumCommMultiBlock<<<1, blockSize>>>(dev_out, gridSize, dev_out);
    //dev_out[0] now holds the final result
    cudaDeviceSynchronize();
    
    cudaMemcpy(&out, dev_out, sizeof(int), cudaMemcpyDeviceToHost);
    cudaFree(dev_arr);
    cudaFree(dev_out);
    return out;
}

İdeal olarak, GPU’daki tüm çoklu işlemcileri tam dolulukta doyurmak için yeterli blok başlatmak istiyor. Bu sayıyı aşmak - özellikle dizide ne kadar eleman varsa o kadar iş parçacığı başlatmak - ters etki yapar. Bunu yapmak artık ham hesaplama gücünü artırmaz, ancak çok verimli ilk döngünün kullanılmasını önler.

last-block guard yardımıyla tek bir çekirdek kullanarak da aynı sonucu elde etmek mümkündür:

static const int wholeArraySize = 100000000;
static const int blockSize = 1024;
static const int gridSize = 24;

__device__ bool lastBlock(int* counter) {
    __threadfence(); //ensure that partial result is visible by all blocks
    int last = 0;
    if (threadIdx.x == 0)
        last = atomicAdd(counter, 1);
    return __syncthreads_or(last == gridDim.x-1);
}    

__global__ void sumCommMultiBlock(const int *gArr, int arraySize, int *gOut, int* lastBlockCounter) {
    int thIdx = threadIdx.x;
    int gthIdx = thIdx + blockIdx.x*blockSize;
    const int gridSize = blockSize*gridDim.x;
    int sum = 0;
    for (int i = gthIdx; i < arraySize; i += gridSize)
        sum += gArr[i];
    __shared__ int shArr[blockSize];
    shArr[thIdx] = sum;
    __syncthreads();
    for (int size = blockSize/2; size>0; size/=2) { //uniform
        if (thIdx<size)
            shArr[thIdx] += shArr[thIdx+size];
        __syncthreads();
    }
    if (thIdx == 0)
        gOut[blockIdx.x] = shArr[0];
    if (lastBlock(lastBlockCounter)) {
        shArr[thIdx] = thIdx<gridSize ? gOut[thIdx] : 0;
        __syncthreads();
        for (int size = blockSize/2; size>0; size/=2) { //uniform
            if (thIdx<size)
                shArr[thIdx] += shArr[thIdx+size];
            __syncthreads();
        }
        if (thIdx == 0)
            gOut[0] = shArr[0];            
    }
}

__host__ int sumArray(int* arr) {
    int* dev_arr;
    cudaMalloc((void**)&dev_arr, wholeArraySize * sizeof(int));
    cudaMemcpy(dev_arr, arr, wholeArraySize * sizeof(int), cudaMemcpyHostToDevice);

    int out;
    int* dev_out;
    cudaMalloc((void**)&dev_out, sizeof(int)*gridSize);
    
    int* dev_lastBlockCounter;
    cudaMalloc((void**)&dev_lastBlockCounter, sizeof(int));
    cudaMemset(dev_lastBlockCounter, 0, sizeof(int));
    
    sumCommMultiBlock<<<gridSize, blockSize>>>(dev_arr, wholeArraySize, dev_out, dev_lastBlockCounter);
    cudaDeviceSynchronize();
    
    cudaMemcpy(&out, dev_out, sizeof(int), cudaMemcpyDeviceToHost);
    cudaFree(dev_arr);
    cudaFree(dev_out);
    return out;
}

Değişmeli olmayan operatör için çok bloklu paralel indirgeme #

Paralel indirgemeye çok bloklu yaklaşım, tek bloklu yaklaşıma çok benzer. Genel girdi dizisi, her biri tek bir blokla küçültülmüş bölümlere ayrılmalıdır. Her bloktan kısmi bir sonuç elde edildiğinde, nihai sonucu elde etmek için bir son blok bunları azaltır.

  • sumNoncommSingleBlock Tek blok azaltma örneğinde daha ayrıntılı olarak açıklanmıştır.

  • lastBlock yalnızca kendisine ulaşan son bloğu kabul eder. Bundan kaçınmak istiyorsanız, çekirdeği iki ayrı çağrıya bölebilirsiniz.

    static const int wholeArraySize = 100000000; static const int blockSize = 1024; static const int gridSize = 24; //this number is hardware-dependent; usually #SM*2 is a good number.

    device bool lastBlock(int* counter) { __threadfence(); //ensure that partial result is visible by all blocks int last = 0; if (threadIdx.x == 0) last = atomicAdd(counter, 1); return __syncthreads_or(last == gridDim.x-1); }

    device void sumNoncommSingleBlock(const int* gArr, int arraySize, int* out) { int thIdx = threadIdx.x; shared int shArr[blockSize2]; shared int offset; shArr[thIdx] = thIdx<arraySize ? gArr[thIdx] : 0; if (thIdx == 0) offset = blockSize; __syncthreads(); while (offset < arraySize) { //uniform shArr[thIdx + blockSize] = thIdx+offset<arraySize ? gArr[thIdx+offset] : 0; __syncthreads(); if (thIdx == 0) offset += blockSize; int sum = shArr[2thIdx] + shArr[2thIdx+1]; __syncthreads(); shArr[thIdx] = sum; } __syncthreads(); for (int stride = 1; stride<blockSize; stride=2) { //uniform int arrIdx = thIdxstride2; if (arrIdx+stride<blockSize) shArr[arrIdx] += shArr[arrIdx+stride]; __syncthreads(); }
    if (thIdx == 0) *out = shArr[0]; }

    global void sumNoncommMultiBlock(const int* gArr, int* out, int* lastBlockCounter) { int arraySizePerBlock = wholeArraySize/gridSize; const int* gArrForBlock = gArr+blockIdx.xarraySizePerBlock; int arraySize = arraySizePerBlock; if (blockIdx.x == gridSize-1) arraySize = wholeArraySize - blockIdx.xarraySizePerBlock; sumNoncommSingleBlock(gArrForBlock, arraySize, &out[blockIdx.x]); if (lastBlock(lastBlockCounter)) sumNoncommSingleBlock(out, gridSize, out);
    }

İdeal olarak, GPU’daki tüm çoklu işlemcileri tam dolulukta doyurmak için yeterli blok başlatmak istiyor. Bu sayıyı aşmak - özellikle dizide ne kadar eleman varsa o kadar iş parçacığı başlatmak - ters etki yapar. Bunu yapmak artık ham hesaplama gücünü artırmaz, ancak çok verimli ilk döngünün kullanılmasını önler.

Değişmeli operatör için tek çözgü paralel azaltma #

Bazen indirgeme, daha büyük bir CUDA çekirdeğinin bir parçası olarak çok küçük bir ölçekte gerçekleştirilmelidir. Örneğin, giriş verilerinin tam olarak 32 öğeye sahip olduğunu varsayalım - bir çözgüdeki iş parçacığı sayısı. Böyle bir senaryoda, indirgemeyi gerçekleştirmek için tek bir çözgü atanabilir. Warp’ın mükemmel bir senkronizasyonda yürütüldüğü göz önüne alındığında, birçok __syncthreads() talimatı kaldırılabilir - blok düzeyinde bir indirgeme ile karşılaştırıldığında.

static const int warpSize = 32;

__device__ int sumCommSingleWarp(volatile int* shArr) {
    int idx = threadIdx.x % warpSize; //the lane index in the warp
    if (idx<16) shArr[idx] += shArr[idx+16];
    if (idx<8) shArr[idx] += shArr[idx+8];
    if (idx<4) shArr[idx] += shArr[idx+4];
    if (idx<2) shArr[idx] += shArr[idx+2];
    if (idx==0) shArr[idx] += shArr[idx+1];
    return shArr[0];
}

‘shArr’ tercihen paylaşılan bellekteki bir dizidir. Değer, çözgüdeki tüm iplikler için aynı olmalıdır. “sumCommSingleWarp” birden çok çarpıtma tarafından çağrılırsa, “shArr” çarpıtmalar arasında farklı olmalıdır (her çarpıtma içinde aynı).

“shArr” bağımsız değişkeni, dizideki işlemlerin gerçekten belirtildiği yerde gerçekleştirilmesini sağlamak için “geçici” olarak işaretlenir. Aksi takdirde, “shArr[idx]” öğesine tekrarlayan atama, yalnızca son atama “shArr” için gerçek bir depo olacak şekilde, bir kayda atama olarak optimize edilebilir. Bu olduğunda, anında atamalar diğer iş parçacıkları tarafından görülmez ve yanlış sonuçlar verir. Bir const olmayan değişkeni const parametresi olarak ilettiğinizde olduğu gibi, geçici olmayan bir normal diziyi geçici olanın bir bağımsız değişkeni olarak iletebileceğinizi unutmayın.

İndirgemeden sonra ‘shArr[1..31]’ içeriği umursanmıyorsa, kodu daha da basitleştirebilirsiniz:

static const int warpSize = 32;

__device__ int sumCommSingleWarp(volatile int* shArr) {
    int idx = threadIdx.x % warpSize; //the lane index in the warp
    if (idx<16) {
      shArr[idx] += shArr[idx+16];
      shArr[idx] += shArr[idx+8];
      shArr[idx] += shArr[idx+4];
      shArr[idx] += shArr[idx+2];
      shArr[idx] += shArr[idx+1];
    }
    return shArr[0];
}

Bu kurulumda birçok “if” koşulunu kaldırdık. Ekstra iplikler bazı gereksiz eklemeler yapıyor, ancak ürettikleri içerikler artık umurumuzda değil. Çözgüler SIMD modunda yürütüldüğünden, bu ipliklerin hiçbir şey yapmamasını sağlayarak zamandan tasarruf etmiyoruz. Öte yandan, bu ‘if’ ifadelerinin gövdesi çok küçük olduğundan, koşulların değerlendirilmesi nispeten uzun zaman alır. “shArr[32..47]” 0 ile doldurulursa, ilk “if” ifadesi de kaldırılabilir.

Warp seviyesindeki azalma, blok seviyesindeki azalmayı artırmak için de kullanılabilir:

__global__ void sumCommSingleBlockWithWarps(const int *a, int *out) {
    int idx = threadIdx.x;
    int sum = 0;
    for (int i = idx; i < arraySize; i += blockSize)
        sum += a[i];
    __shared__ int r[blockSize];
    r[idx] = sum;
    sumCommSingleWarp(&r[idx & ~(warpSize-1)]);
    __syncthreads();
    if (idx<warpSize) { //first warp only
        r[idx] = idx*warpSize<blockSize ? r[idx*warpSize] : 0;
        sumCommSingleWarp(r);
        if (idx == 0)
            *out = r[0];
    }
}

“&r[idx & ~(warpSize-1)]” argümanı temelde “r + warpIdx*32” şeklindedir. Bu, ‘r’ dizisini 32 elemanlı parçalara ayırır ve her parçaya ayrı çarpıtma atanır.

Değişmeli olmayan operatör için tek çözgü paralel azaltma #

Bazen indirgeme, daha büyük bir CUDA çekirdeğinin bir parçası olarak çok küçük bir ölçekte gerçekleştirilmelidir. Örneğin, giriş verilerinin tam olarak 32 öğeye sahip olduğunu varsayalım - bir çözgüdeki iş parçacığı sayısı. Böyle bir senaryoda, indirgemeyi gerçekleştirmek için tek bir çözgü atanabilir. Warp’ın mükemmel bir senkronizasyonda yürütüldüğü göz önüne alındığında, birçok __syncthreads() talimatı kaldırılabilir - blok düzeyinde bir indirgeme ile karşılaştırıldığında.

static const int warpSize = 32;

__device__ int sumNoncommSingleWarp(volatile int* shArr) {
    int idx = threadIdx.x % warpSize; //the lane index in the warp
    if (idx%2 == 0) shArr[idx] += shArr[idx+1];
    if (idx%4 == 0) shArr[idx] += shArr[idx+2];
    if (idx%8 == 0) shArr[idx] += shArr[idx+4];
    if (idx%16 == 0) shArr[idx] += shArr[idx+8];
    if (idx == 0) shArr[idx] += shArr[idx+16];
    return shArr[0];
}

‘shArr’ tercihen paylaşılan bellekteki bir dizidir. Değer, çözgüdeki tüm iplikler için aynı olmalıdır. “sumCommSingleWarp” birden çok çarpıtma tarafından çağrılırsa, “shArr” çarpıtmalar arasında farklı olmalıdır (her çarpıtma içinde aynı).

Aksi takdirde, “shArr[idx]” öğesine tekrarlayan atama, yalnızca son atama “shArr” için gerçek bir depo olacak şekilde, bir kayda atama olarak optimize edilebilir. Bu olduğunda, anında atamalar diğer iş parçacıkları tarafından görülmez ve yanlış sonuçlar verir. Bir const olmayan değişkeni const parametresi olarak ilettiğinizde olduğu gibi, geçici olmayan bir normal diziyi geçici olanın bir bağımsız değişkeni olarak iletebileceğinizi unutmayın.

“shArr[1..31]” öğesinin son içeriği umursanmıyorsa ve “shArr[32..47]” öğesini sıfırlarla doldurabiliyorsa, yukarıdaki kodu basitleştirebiliriz:

static const int warpSize = 32;

__device__ int sumNoncommSingleWarpPadded(volatile int* shArr) {
    //shArr[32..47] == 0
    int idx = threadIdx.x % warpSize; //the lane index in the warp
    shArr[idx] += shArr[idx+1];
    shArr[idx] += shArr[idx+2];
    shArr[idx] += shArr[idx+4];
    shArr[idx] += shArr[idx+8];
    shArr[idx] += shArr[idx+16];
    return shArr[0];
}

Bu kurulumda, talimatların yaklaşık yarısını oluşturan tüm ‘if’ koşullarını kaldırdık. Ekstra iş parçacıkları bazı gereksiz eklemeler yaparak sonucu nihai sonuç üzerinde hiçbir etkisi olmayan ‘shArr’ hücrelerinde depolar. Çözgüler SIMD modunda yürütüldüğünden, bu ipliklerin hiçbir şey yapmamasını sağlayarak zamandan tasarruf etmiyoruz.

Yalnızca yazmaçları kullanarak tek çözgü paralel azaltma #

Tipik olarak, indirgeme genel veya paylaşılan dizide gerçekleştirilir. Ancak, indirgeme çok küçük bir ölçekte yapıldığında, daha büyük bir CUDA çekirdeğinin parçası olarak tek bir çözgü ile gerçekleştirilebilir. Bu olduğunda, Keppler veya daha yüksek mimarilerde (CC>=3.0), paylaşılan belleği kullanmaktan kaçınmak için çarpıtma-karıştırma işlevlerini kullanmak mümkündür.

Örneğin, bir çarpıtmadaki her bir ipliğin tek bir girdi veri değeri tuttuğunu varsayalım. Tüm iş parçacıkları birlikte, özetlememiz (veya başka bir ilişkisel işlem gerçekleştirmemiz) gereken 32 öğeye sahiptir.

__device__ int sumSingleWarpReg(int value) {
    value += __shfl_down(value, 1);
    value += __shfl_down(value, 2);
    value += __shfl_down(value, 4);
    value += __shfl_down(value, 8);
    value += __shfl_down(value, 16);
    return __shfl(value,0);
}

Bu sürüm hem değişmeli hem de değişmeli olmayan operatörler için çalışır.
