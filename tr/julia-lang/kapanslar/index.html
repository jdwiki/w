Sözdizimi #

  • x -> [gövde]
  • (x, y) -> [gövde]
  • (xs…) -> [gövde]

Julia’nın eski sürümlerinde, kapatmalar ve anonim işlevlerin çalışma zamanı performans cezası vardı. Bu ceza 0,5’te elimine edilmiştir.

Kapanışlara Giriş #

İşlevler, Julia programlamasının önemli bir parçasıdır. Doğrudan modüller içinde tanımlanabilirler, bu durumda işlevlere üst düzey olarak atıfta bulunulur. Ancak fonksiyonlar, diğer fonksiyonlar içinde de tanımlanabilir. Bu tür işlevlere “kapamalar” denir.

Kapanışlar, değişkenleri dış işlevlerinde yakalar. Bir üst düzey işlev, yalnızca modüllerindeki global değişkenleri, işlev parametrelerini veya yerel değişkenleri kullanabilir:

x = 0  # global
function toplevel(y)
    println("x = ", x, " is a global variable")
    println("y = ", y, " is a parameter")
    z = 2
    println("z = ", z, " is a local variable")
end

Öte yandan, bir kapatma, yakaladığı dış işlevlerden gelen değişkenlere ek olarak tüm bunları kullanabilir:

x = 0  # global
function toplevel(y)
    println("x = ", x, " is a global variable")
    println("y = ", y, " is a parameter")
    z = 2
    println("z = ", z, " is a local variable")

    function closure(v)
        println("v = ", v, " is a parameter")
        w = 3
        println("w = ", w, " is a local variable")
        println("x = ", x, " is a global variable")
        println("y = ", y, " is a closed variable (a parameter of the outer function)")
        println("z = ", z, " is a closed variable (a local of the outer function)")
    end
end

c = toplevel(10) çalıştırırsak sonucu görürüz

julia> c = toplevel(10)
x = 0 is a global variable
y = 10 is a parameter
z = 2 is a local variable
(::closure) (generic function with 1 method)

Bu işlevin kuyruk ifadesinin kendi içinde bir işlev olduğuna dikkat edin; yani bir kapanış. Başka bir işlevmiş gibi kapanışı ‘c’ olarak adlandırabiliriz:

julia> c(11)
v = 11 is a parameter
w = 3 is a local variable
x = 0 is a global variable
y = 10 is a closed variable (a parameter of the outer function)
z = 2 is a closed variable (a local of the outer function)

“c"nin “toplevel” çağrısından “y” ve “z” değişkenlerine hala erişimi olduğunu unutmayın - “toplevel” zaten geri dönmüş olsa bile! Her kapatma, hatta aynı işlev tarafından döndürülenler bile, farklı değişkenler üzerinde kapanır. Tekrar ’toplevel’ diyebiliriz

julia> d = toplevel(20)
x = 0 is a global variable
y = 20 is a parameter
z = 2 is a local variable
(::closure) (generic function with 1 method)

julia> d(22)
v = 22 is a parameter
w = 3 is a local variable
x = 0 is a global variable
y = 20 is a closed variable (a parameter of the outer function)
z = 2 is a closed variable (a local of the outer function)

julia> c(22)
v = 22 is a parameter
w = 3 is a local variable
x = 0 is a global variable
y = 10 is a closed variable (a parameter of the outer function)
z = 2 is a closed variable (a local of the outer function)

“d” ve “c"nin aynı koda sahip olmasına ve aynı argümanlardan geçmesine rağmen çıktılarının farklı olduğuna dikkat edin. Bunlar ayrı kapanışlardır.

Curry Uygulaması #

Kapatmaların bir uygulaması, bir işlevi kısmen uygulamaktır; yani, şimdi bazı argümanlar sağlayın ve kalan argümanları alan bir fonksiyon oluşturun. Currying kısmi başvurunun özel bir şeklidir.

Bir fonksiyona ilk argümanı sağlayacak olan ‘curry(f, x)’ basit fonksiyonu ile başlayalım ve daha sonra ek argümanlar bekleyelim. Tanım oldukça basittir:

curry(f, x) = (xs...) -> f(x, xs...)

Bir kez daha, [anonim işlev sözdizimini](https://www.wikiod.com/tr/julia-lang/fonksiyonlar#Anonim işlevler), bu sefer değişken bağımsız değişken sözdizimi ile birlikte kullanıyoruz.

Bu ‘curry’ işlevini kullanarak bazı temel işlevleri tacit (veya nokta içermeyen) stilinde uygulayabiliriz.

julia> const double = curry(*, 2)
(::#19) (generic function with 1 method)

julia> double(10)
20

julia> const simon_says = curry(println, "Simon: ")
(::#19) (generic function with 1 method)

julia> simon_says("How are you?")
Simon: How are you?

İşlevler beklenen genelliği korur:

julia> simon_says("I have ", 3, " arguments.")
Simon: I have 3 arguments.

julia> double([1, 2, 3])
3-element Array{Int64,1}:
 2
 4
 6

İşlev Bileşimi #

Anonim işlev sözdizimini kullanarak işlev bileşimini gerçekleştirecek bir işlev tanımlayabiliriz. /10470/anonim-işlevler#t=201608292058450143587):

f ∘ g = x -> f(g(x))

Bu tanımın aşağıdaki tanımların her birine eşdeğer olduğunu unutmayın:

∘(f, g) = x -> f(g(x))

veya

function ∘(f, g)
    x -> f(g(x))
end

Julia’da ‘f ∘ g’nin sadece ‘∘(f, g)’ için sözdizimi şekeri olduğunu hatırlatarak.

Bu fonksiyonun doğru bir şekilde oluştuğunu görebiliriz:

julia> double(x) = 2x
double (generic function with 1 method)

julia> triple(x) = 3x
triple (generic function with 1 method)

julia> const sextuple = double ∘ triple
(::#17) (generic function with 1 method)

julia> sextuple(1.5)
9.0

Sürüm v0.5’te bu tanım çok performanslıdır. Oluşturulan LLVM koduna bakabiliriz:

julia> @code_llvm sextuple(1)

define i64 @"julia_#17_71238"(i64) #0 {
top:
  %1 = mul i64 %0, 6
  ret i64 %1
}

İki çarpmanın tek bir çarpma şeklinde katlandığı ve bu fonksiyonun mümkün olduğu kadar verimli olduğu açıktır.

son sürüm

Bu üst düzey işlev nasıl çalışır? Yalnızca kodundan oluşan değil, aynı zamanda kapsamından belirli değişkenleri de izleyen bir kapatma oluşturur. Julia’da üst düzey kapsamda oluşturulmayan tüm işlevler kapanışlardır.

Kapanış alanları aracılığıyla kapatılan değişkenler incelenebilir. Örneğin şunu görüyoruz:

julia> (sin ∘ cos).f
sin (generic function with 10 methods)

julia> (sin ∘ cos).g
cos (generic function with 10 methods)
