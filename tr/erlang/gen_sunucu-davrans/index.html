“gen_server”, Erlang’ın önemli bir özelliğidir ve bu işlevin her yönünü anlamak için bazı ön koşullar gerektirir:

  • Döngü, özyineleme ve durum
  • [Yumurtlama süreçleri2
  • Mesaj geçiyor
  • OTP ilkeleri

Erlang’daki bir özellik hakkında daha fazla bilgi edinmenin iyi bir yolu, kaynak kodunu doğrudan resmi github deposundan okumaktır. “gen_server” davranışı, özünde birçok yararlı bilgi ve ilginç yapı barındırır.

gen_server gen_server.erl içinde tanımlanmıştır ve ilgili belgeleri stdlib Erlang belgelerinde bulunabilir. gen_server bir OTP özelliğidir ve daha fazla bilgi OTP Design Principles and User’s Guide içinde de bulunabilir.

Frank Hebert ayrıca ücretsiz çevrimiçi kitabından ‘gen_server’ için başka bir iyi giriş sunuyor Büyük iyilik için Size Biraz Erlang Öğrenin!

“gen_server” geri araması için resmi belgeler:

  • code_change/3
  • handle_call/3
  • handle_cast/2
  • handle_info/2
  • init/1
  • sonlandır/2

Karşılama Hizmeti #

gen_sunucu davranışını kullanma #

Bir “gen_server”, bir sunucu gibi çalışan belirli bir sonlu durum makinesidir. “gen_server” farklı olay türlerini işleyebilir:

  • “handle_call” ile senkronize istek
  • “handle_cast” ile eşzamansız istek
  • “handle_info” ile diğer mesaj (OTP spesifikasyonunda tanımlanmamıştır)

Senkron ve asenkron mesajlar OTP’de belirtilir ve üzerinde her türlü veri bulunan basit etiketli demetlerdir.

Basit bir “gen_server” şu şekilde tanımlanır:

-module(simple_gen_server).
-behaviour(gen_server).
-export([start_link/0]).
-export([init/1, handle_call/3, handle_cast/2, handle_info/2,
         terminate/2, code_change/3]).

start_link() ->
    Return = gen_server:start_link({local, ?MODULE}, ?MODULE, [], []),
    io:format("start_link: ~p~n", [Return]),
    Return.

init([]) ->
    State = [],
    Return = {ok, State},
    io:format("init: ~p~n", [State]),
    Return.

handle_call(_Request, _From, State) ->
    Reply = ok,
    Return = {reply, Reply, State},
    io:format("handle_call: ~p~n", [Return]),
    Return.

handle_cast(_Msg, State) ->
    Return = {noreply, State},
    io:format("handle_cast: ~p~n", [Return]),
    Return.

handle_info(_Info, State) ->
    Return = {noreply, State},
    io:format("handle_info: ~p~n", [Return]),
    Return.

terminate(_Reason, _State) ->
    Return = ok,
    io:format("terminate: ~p~n", [Return]),
    ok.

code_change(_OldVsn, State, _Extra) ->
    Return = {ok, State},
    io:format("code_change: ~p~n", [Return]),
    Return.

Bu kod basittir: alınan her mesaj standart çıktıya yazdırılır.

gen_sunucu davranışı #

Bir “gen_server” tanımlamak için, bunu kaynak kodunuzda “-behaviour(gen_server)” ile açıkça belirtmeniz gerekir. “davranış"ın ABD (davranış) veya İngiltere’de (davranış) yazılabileceğine dikkat edin.

start_link/0 #

Bu işlev, başka bir işlevi çağırmak için basit bir kısayoldur: gen_server:start_link/3,4.

start_link/3,4 #

start_link() ->
    gen_server:start_link({local, ?MODULE}, ?MODULE, [], []).

Bu işlev, sunucunuzu bir “süpervizör"e veya başka bir işleme bağlı olarak başlatmak istediğinizde çağrılır. start_link/3,4 işleminizi otomatik olarak kaydedebilir (işleminizin benzersiz olması gerektiğini düşünüyorsanız) veya basit bir işlem gibi basitçe ortaya çıkarabilir. Çağrıldığında, bu işlev init/1 yürütür.

Bu fonksiyon şu tanımlı değerleri döndürebilir:

  • {tamam,Pid}
  • “yoksay”
  • {hata,Hata}

init/1 #

init([]) ->
    State = [],
    {ok, State}.

init/1, sunucunuz başlatıldığında ilk yürütülen işlevdir. Bu, uygulamanızın tüm ön koşullarını başlatır ve durumu yeni oluşturulan işleme döndürür.

Bu işlev yalnızca bu tanımlanmış değerleri döndürebilir:

  • {tamam,Eyalet}
  • {tamam, Durum, Zaman Aşımı}
  • {tamam, Durum, hazırda bekletme}
  • {dur, Sebep}
  • “yoksay”

‘Durum’ değişkeni her şey olabilir (örneğin liste, grup, proplists, map, record) ve ortaya çıkan süreç içindeki tüm işlevler tarafından erişilebilir durumda kalır.

handle_call/3 #

handle_call(_Request, _From, State) ->
    Reply = ok,
    {reply, Reply, State}.

gen_server:call/2 bu geri aramayı yürütür. İlk argüman mesajınız (_Request), ikincisi isteğin kaynağı (_From) ve sonuncusu çalışan gen_sunucu davranışınızın mevcut durumudur (State).

Arayana bir cevap istiyorsanız, handle_call/3 aşağıdaki veri yapılarından birini döndürmesi gerekir:

  • {reply,Reply,NewState}
  • {reply,Reply,NewState,Zaman aşımı}
  • {reply,Reply,NewState,hazırda bekletme}

Arayana cevap istemiyorsanız, handle_call/3 aşağıdaki veri yapılarından birini döndürmesi gerekir:

  • {noreply,NewState}
  • {noreply,NewState,Zaman aşımı}
  • {noreply,NewState,hazırda bekletme}

Mevcut gen_sunucunuzun mevcut yürütmesini durdurmak istiyorsanız, handle_call/3 aşağıdaki veri yapılarından birini döndürmesi gerekir:

  • {dur,Neden, Yanıtla,NewState}
  • {dur,Neden,NewState}

handle_cast/2 #

handle_cast(_Msg, State) ->
    {noreply, State}.

gen_server:cast/2 bu geri aramayı yürütür. İlk argüman mesajınızdır (_Msg), ikincisi ise çalışan gen_server davranışınızın mevcut durumudur.

Varsayılan olarak, bu işlev arayana veri gönderemez, bu nedenle yalnızca iki seçeneğiniz vardır, mevcut yürütmeye devam edin:

  • {noreply,NewState}
  • {noreply,NewState,Zaman aşımı}
  • {noreply,NewState,hazırda bekletme}

Veya mevcut “gen_server” işleminizi durdurun:

  • {dur,Neden,NewState}

handle_info/2 #

handle_info(_Info, State) ->
    {noreply, State}.

Dış dünyadan standart olmayan OTP mesajı geldiğinde handle_info/2 yürütülür. Bu, yanıt veremez ve ‘handle_cast/2’ gibi, mevcut yürütmeye devam ederek yalnızca 2 eylem yapabilir:

  • {noreply,NewState}
  • {noreply,NewState,Zaman aşımı}
  • {noreply,NewState,hazırda bekletme}

Veya şu anda çalışan “gen_server” işlemini durdurun:

  • {dur,Neden,NewState}

sonlandır/2 #

terminate(_Reason, _State) ->
    ok.

Bir hata oluştuğunda veya gen_server işleminizi kapatmak istediğinizde terminate/2 çağrılır.

kod_değişimi/3 #

code_change(_OldVsn, State, _Extra) ->
    {ok, State}.

Çalışan kodunuzu yükseltmek istediğinizde code_change/3 işlevi çağrılır.

Bu işlev yalnızca bu tanımlanmış değerleri döndürebilir:

  • {tamam, NewState}
  • {hata, Neden}

Bu işlemi başlatmak #

Kodunuzu derleyebilir ve “simple_gen_server"ı başlatabilirsiniz:

simple_gen_server:start_link().

Sunucunuza mesaj göndermek istiyorsanız şu fonksiyonları kullanabilirsiniz:

% will use handle_call as callback and print:
%   handle_call: mymessage
gen_server:call(simple_gen_server, mymessage).

% will use handle_cast as callback and print:
%   handle_cast: mymessage
gen_server:cast(simple_gen_server, mymessage).

% will use handle_info as callback and print:
%   handle_info: mymessage
erlang:send(whereis(simple_gen_server), mymessage).

Basit Anahtar/Değer Veritabanı #

Bu kaynak kodu, map Erlang veri yapısına dayalı basit bir anahtar/değer deposu hizmeti oluşturur. Öncelikle, gen_server ile ilgili tüm bilgileri tanımlamamız gerekiyor:

-module(cache).
-behaviour(gen_server).

% our API
-export([start_link/0]).
-export([get/1, put/2, state/0, delete/1, stop/0]).

% our handlers
-export([init/1, handle_call/3, handle_cast/2, handle_info/2,
         terminate/2, code_change/3]).

% Defining our function to start `cache` process:

start_link() ->
    gen_server:start_link({local, ?MODULE}, ?MODULE, [], []).

%%%%%%%%%%%%%%
% API

% Key/Value database is a simple store, value indexed by an unique key. 
% This implementation is based on map, this datastructure is like hash 
# in Perl or dictionaries in Python. 

% put/2
% put a value indexed by a key. We assume the link is stable 
% and the data will be written, so, we use an asynchronous call with 
% gen_server:cast/2.

put(Key, Value) ->
    gen_server:cast(?MODULE, {put, {Key, Value}}).

% get/1
% take one argument, a key and will a return the value indexed 
% by this same key. We use a synchronous call with gen_server:call/2.

get(Key) ->
    gen_server:call(?MODULE, {get, Key}).

% delete/1 
% like `put/1`, we assume the data will be removed. So, we use an 
% asynchronous call with gen_server:cast/2.

delete(Key) ->
    gen_server:cast(?MODULE, {delete, Key}).

% state/0 
% This function will return the current state (here the map who contain all 
% indexed values), we need a synchronous call.

state() ->
    gen_server:call(?MODULE, {get_state}).

% stop/0
% This function stop cache server process.

stop() ->
    gen_server:stop(?MODULE).

%%%%%%%%%%%%%%%
% Handlers

% init/1
% Here init/1 will initialize state with simple empty map datastructure.

init([]) ->
    {ok, #{}}.

% handle_call/3
% Now, we need to define our handle. In a cache server we need to get our 
% value from a key, this feature need to be synchronous, so, using 
% handle_call seems a good choice:

handle_call({get, Key}, From, State) ->
    Response = maps:get(Key, State, undefined),
    {reply, Response, State};

% We need to check our current state, like get_fea

handle_call({get_state}, From, State) ->
    Response = {current_state, State},
    {reply, Response, State};

% All other messages will be dropped here.

handle_call(_Request, _From, State) ->
    Reply = ok,
    {reply, Reply, State}.

% handle_cast/2
% put/2 will execute this function.

handle_cast({put, {Key, Value}}, State) ->
    NewState = maps:put(Key, Value, State),
    {noreply, NewState};

% delete/1 will execute this function.

handle_cast({delete, Key}, State) ->
    NewState = maps:remove(Key, State),
    {noreply, NewState};

% All other messages are dropped here.

handle_cast(_Msg, State) ->
    {noreply, State}.

%%%%%%%%%%%%%%%%
% other handlers

% We don't need other features, other handlers do nothing.

handle_info(_Info, State) ->
    {noreply, State}.

terminate(_Reason, _State) ->
    ok.

code_change(_OldVsn, State, _Extra) ->
    {ok, State}.

Önbellek sunucumuzu kullanma #

Artık kodumuzu derleyip ’erl’ ile kullanmaya başlayabiliriz.

% compile cache
c(cache).

% starting cache server
cache:start_link().

% get current store
% will return:
%   #{}
cache:state().

% put some data
cache:put(1, one).
cache:put(hello, bonjour).
cache:put(list, []).

% get current store
% will return:
%   #{1 => one,hello => bonjour,list => []}
cache:state().

% delete a value
cache:delete(1).
cache:state().
%   #{1 => one,hello => bonjour,list => []}

% stopping cache server
cache:stop().
