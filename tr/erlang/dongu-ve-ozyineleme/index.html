Sözdizimi #

  • işlev ( list | iolist | tuple ) -> işlev (kuyruk).

Neden özyinelemeli fonksiyonlar? #

Erlang, işlevsel bir programlama dilidir ve herhangi bir döngü yapısı yoktur. Fonksiyonel programlamadaki her şey veri, tip ve fonksiyonlara dayalıdır. Bir döngü istiyorsanız, kendisini çağıran bir fonksiyon oluşturmanız gerekir.

Zorunlu ve nesne yönelimli dilde geleneksel “while” veya “for” döngüsü Erlang’da şu şekilde temsil edilebilir:

loop() ->
  % do something here
  loop().

Bu kavramı anlamak için iyi bir yöntem, tüm işlev çağrılarını genişletmektir. Bunu diğer örneklerde göreceğiz.

Liste #

Burada list türünden en basit özyinelemeli işlev. Bu işlev, yalnızca başından sonuna kadar bir listede gezinir ve başka bir şey yapmaz.

-spec loop(list()) -> ok.
loop([]) ->
  ok;
loop([H|T]) ->
  loop(T).

Bunu şu şekilde arayabilirsiniz:

loop([1,2,3]). % will return ok.

Burada özyinelemeli fonksiyon genişlemesi:

loop([1|2,3]) ->
  loop([2|3]) ->
    loop([3|]) ->
      loop([]) ->
        ok.

IO eylemleriyle özyinelemeli döngü #

Önceki kod hiçbir şey yapmaz ve oldukça işe yaramaz. Bu nedenle, şimdi bazı eylemleri gerçekleştiren özyinelemeli bir işlev oluşturacağız. Bu kod, lists:foreach/2 ile benzerdir.

-spec loop(list(), fun()) -> ok.
loop([], _) ->
  ok;
loop([H|T], Fun) 
  when is_function(Fun) ->
    Fun(H),
    loop(T, Fun).

Bunu şu şekilde arayabilirsiniz:

Fun = fun(X) -> io:format("~p", [X]) end.
loop([1,2,3]).

Burada özyinelemeli fonksiyon genişlemesi:

loop([1|2,3], Fun(1)) ->
  loop([2|3], Fun(2)) ->
    loop([3|], Fun(3)) ->
      loop([], _) ->
        ok.

lists:foreach/2 çıktısıyla karşılaştırabilirsiniz:

lists:foreach(Fun, [1,2,3]).

Değiştirilmiş listeyi döndüren liste üzerinde özyinelemeli döngü #

lists:map/2 ile benzer başka bir faydalı örnek. Bu fonksiyon bir liste ve bir isimsiz fonksiyon alacaktır. Listedeki bir değer her eşleştirildiğinde, ona fonksiyon uygularız.

-spec loop(A :: list(), fun()) -> list().
loop(List, Fun) 
  when is_list(List), is_function(Fun) ->
    loop(List, Fun, []).

-spec loop(list(), fun(), list()) -> list() | {error, list()}.
loop([], _, Buffer) 
  when is_list(Buffer) ->
    lists:reverse(Buffer);
loop([H|T], Fun, Buffer) 
  when is_function(Fun), is_list(Buffer) ->
    BufferReturn = [Fun(H)] ++ Buffer,
    loop(T, Fun, BufferReturn).

Bunu şu şekilde arayabilirsiniz:

Fun(X) -> X+1 end.
loop([1,2,3], Fun).

Burada özyinelemeli fonksiyon genişlemesi:

loop([1|2,3], Fun(1), [2]) ->
  loop([2|3], Fun(2), [3,2]) ->
    loop([3|], Fun(3), [4,3,2]) ->
      loop([], _, [4,3,2]) ->
        list:reverse([4,3,2]) ->
          [2,3,4].

Bu işlev aynı zamanda “kuyruk özyinelemeli işlev” olarak da adlandırılır, çünkü değiştirilmiş verileri çoklu yürütme bağlamı üzerinden geçirmek için akümülatör gibi bir değişken kullanırız.

Iolist ve Bitstring #

Liste gibi, iolist ve bitstring üzerindeki en basit işlev şudur:

-spec loop(iolist()) -> ok | {ok, iolist} .
loop(<<>>) ->
  ok;
loop(<<Head, Tail/bitstring>>) ->
  loop(Tail);
loop(<<Rest/bitstring>>) ->
  {ok, Rest}

Bunu şu şekilde arayabilirsiniz:

loop(<<"abc">>).

Burada özyinelemeli fonksiyon genişlemesi:

loop(<<"a"/bitstring, "bc"/bitstring>>) ->
  loop(<<"b"/bitstring, "c"/bitstring>>) ->
    loop(<<"c"/bitstring>>) ->
      loop(<<>>) ->
        ok.

Değişken ikili boyut üzerinde özyinelemeli işlev #

Bu kod bit dizgisini alır ve bunun ikili boyutunu dinamik olarak tanımlar. Yani, eğer boyutu ‘4’ olarak ayarlarsak, her ‘4’ bit için bir veri eşleştirilecektir. Bu döngü ilginç bir şey yapmaz, sadece bizim dayanağımızdır.

loop(Bitstring, Size) 
  when is_bitstring(Bitstring), is_integer(Size) ->
    case Bitstring of
      <<>> -> 
        ok;
      <<Head:Size/bitstring,Tail/bitstring>> ->
        loop(Tail, Size);
      <<Rest/bitstring>> ->
        {ok, Rest}
    end.

Bunu şu şekilde arayabilirsiniz:

loop(<<"abc">>, 4).

Burada özyinelemeli fonksiyon genişlemesi:

loop(<<6:4/bitstring, 22, 38, 3:4>>, 4) ->
  loop(<<1:4/bitstring, "bc">>, 4) ->
    loop(<<6:4/bitstring, 38,3:4>>, 4) ->
      loop(<<2:4/bitstring, "c">>, 4) ->
        loop(<<6:4/bitstring, 3:4>>, 4) ->
          loop(<<3:4/bitstring>>, 4) ->
            loop(<<>>, 4) ->
              ok.

Bitstringimiz 7 kalıba bölünmüştür. Neden? Niye? Çünkü Erlang varsayılan olarak ‘8’ bitlik ikili boyutu kullanır, ikiye bölersek ‘4’ bitimiz olur. Dizemiz 8*3=24 bittir. “24/4=6” desenleri. Son kalıp ‘«»’ şeklindedir. loop/2 işlevi 7 kez çağrılır.

eylemlerle değişken ikili boyut üzerinde özyinelemeli işlev #

Şimdi, daha ilginç bir şey yapabiliriz. Bu fonksiyon bir argüman daha alır, isimsiz bir fonksiyon. Bir deseni her eşleştirdiğimizde, bu ona iletilecektir.

-spec loop(iolist(), integer(), function()) -> ok.
loop(Bitstring, Size, Fun) ->
  when is_bitstring(Bitstring), is_integer(Size), is_function(Fun) ->
        case Bitstring of
      <<>> -> 
        ok;
      <<Head:Size/bitstring,Tail/bitstring>> ->
        Fun(Head),
        loop(Tail, Size, Fun);
      <<Rest/bitstring>> ->
        Fun(Rest),
        {ok, Rest}
    end.

Bunu şu şekilde arayabilirsiniz:

Fun = fun(X) -> io:format("~p~n", [X]) end.
loop(<<"abc">>, 4, Fun).

Burada özyinelemeli fonksiyon genişlemesi:

loop(<<6:4/bitstring, 22, 38, 3:4>>, 4, Fun(<<6:4>>) ->
  loop(<<1:4/bitstring, "bc">>, 4, Fun(<<1:4>>)) ->
    loop(<<6:4/bitstring, 38,3:4>>, 4, Fun(<<6:4>>)) ->
      loop(<<2:4/bitstring, "c">>, 4, Fun(<<2:4>>)) ->
        loop(<<6:4/bitstring, 3:4>>, 4, Fun(<<6:4>>) ->
          loop(<<3:4/bitstring>>, 4, Fun(<<3:4>>) ->
            loop(<<>>, 4) ->
              ok.

Değiştirilmiş bit dizgisini döndüren bit dizgisi üzerinde özyinelemeli işlev #

Bu, lists:map/2‘ye benzer, ancak bitstring ve iolist için.

% public function (interface).
-spec loop(iolist(), fun()) -> iolist() | {iolist(), iolist()}.
loop(Bitstring, Fun) ->
  loop(Bitstring, 8, Fun).

% public function (interface).
-spec loop(iolist(), integer(), fun()) -> iolist() | {iolist(), iolist()}.
loop(Bitstring, Size, Fun) ->
  loop(Bitstring, Size, Fun, <<>>)

% private function.
-spec loop(iolist(), integer(), fun(), iolist()) -> iolist() | {iolist(), iolist()}.
loop(<<>>, _, _, Buffer) ->
  Buffer;
loop(Bitstring, Size, Fun, Buffer) ->
  when is_bitstring(Bitstring), is_integer(Size), is_function(Fun) ->
    case Bitstring of
      <<>> -> 
        Buffer;
      <<Head:Size/bitstring,Tail/bitstring>> ->
        Data = Fun(Head),
        BufferReturn = <<Buffer/bitstring, Data/bitstring>>,
        loop(Tail, Size, Fun, BufferReturn);
      <<Rest/bitstring>> ->
        {Buffer, Rest}
    end.

Bu kod daha karmaşık görünüyor. İki işlev eklendi: “loop/2” ve “loop/3”. Bu iki işlev, ’loop/4’ için basit bir arayüzdür.

Bunu şu şekilde yürütebilirsiniz:

Fun = fun(<<X>>) -> << (X+1) >> end.
loop(<<"abc">>, Fun).
% will return <<"bcd">>

Fun = fun(<<X:4>>) -> << (X+1) >> end.
loop(<<"abc">>, 4, Fun).
% will return <<7,2,7,3,7,4>>

loop(<<"abc">>, 4, Fun, <<>>).
% will return <<7,2,7,3,7,4>>

Harita #

Erlang’daki Harita, Perl’deki karmalar veya Python’daki sözlükler’e eşdeğerdir, bu bir anahtar/değer deposudur. Depolanan her değeri listelemek için her anahtarı listeleyebilir ve anahtar/değer çiftini döndürebilirsiniz. Bu ilk döngü size bir fikir verir:

loop(Map) when is_map(Map) -> 
  Keys = maps:keys(Map),
  loop(Map, Keys).

loop(_ , []) ->
  ok;
loop(Map, [Head|Tail]) ->
  Value = maps:get(Head, Map),
  io:format("~p: ~p~n", [Head, Value]),
  loop(Map, Tail).

Bunu şu şekilde yürütebilirsiniz:

Map = #{1 => "one", 2 => "two", 3 => "three"}.
loop(Map).
% will return:
% 1: "one"
% 2: "two"
% 3: "three"

Devlet Yönetimi #

Özyinelemeli işlev, durumlarını döngü için kullanır. Yeni bir süreç oluşturduğunuzda, bu süreç sadece tanımlanmış bir durumu olan bir döngü olacaktır.

Anonim işlev #

Burada, önceki örneğe dayalı özyinelemeli anonim işlevler için 2 örnek. İlk olarak, basit sonsuz döngü:

InfiniteLoop = fun 
  R() -> 
    R() end.

İkincisi, liste üzerinde döngü yapan anonim işlev:

LoopOverList = fun 
  R([]) -> ok;
  R([H|T]) ->
    R(T) end.

Bu iki fonksiyon şu şekilde yeniden yazılabilir:

InfiniteLoop = fun loop/0.

Bu durumda, “loop/0”, açıklamalardan “loop/0"a bir göndermedir. İkincisi, biraz daha karmaşık:

LoopOverLlist = fun loop/2.

Burada, ’loop/2’, liste örneğindeki ’loop/2’ referansıdır. Bu iki gösterim sözdizimsel şekerdir.
