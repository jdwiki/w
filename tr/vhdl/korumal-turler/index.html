Korumalı türler

  • Sözde rastgele bir oluşturucu

Bu sayfada

  • Sözde rastgele bir oluşturucu

VHDL 1993’ten önce, iki eşzamanlı işlem yalnızca sinyallerle iletişim kurabiliyordu. Yalnızca simülasyon adımları arasında sinyalleri güncelleyen dilin simülasyon semantiği sayesinde, bir simülasyonun sonucu deterministikti: süreçleri yürütmek için simülasyon zamanlayıcısı tarafından seçilen sıraya bağlı değildi.

[Aslında bu %100 doğru değil. İşlemler ayrıca dosya giriş/çıkışlarını kullanarak da iletişim kurabilir. Ancak bir tasarımcı, dosyaları kullanarak determinizmden ödün veriyorsa, bu gerçekten bir hata olamaz.]

Böylece dil güvenliydi. Amacı dışında, deterministik olmayan VHDL modelleri tasarlamak neredeyse imkansızdı.

VHDL 1993, paylaşılan değişkenleri tanıttı ve deterministik olmayan VHDL modelleri tasarlamak çok kolaylaştı.

VHDL 2000, korumalı türleri ve paylaşılan değişkenlerin korumalı türde olması gerektiği kısıtlamasını tanıttı.

VHDL’de, korunan türler, Nesne Yönelimli (OO) dillerdeki nesne kavramına en çok benzeyen şeydir. Veri yapılarının kapsüllenmesini ve yöntemlerini uygularlar. Ayrıca veri üyelerine özel ve atomik erişimi garanti ederler. Bu, determinizmi tamamen engellemez, ancak en azından paylaşılan değişkenlere ayrıcalık ve atomiklik ekler.

Korumalı türler, yalnızca simülasyon amaçlı yüksek seviyeli VHDL modelleri tasarlarken çok kullanışlıdır. OO dillerinin birkaç çok iyi özelliğine sahiptirler. Bunları sık kullanmak, kodu daha okunabilir, bakımı yapılabilir ve yeniden kullanılabilir hale getirir.

Notlar:

  • Bazı simülasyon araç zincirleri, varsayılan olarak, yalnızca paylaşılan bir değişken korumalı türde olmadığında uyarı verir.
  • Bazı sentez araçları korumalı türleri desteklemez.
  • Bazı sentez araçları, sınırlı bir paylaşılan değişken desteğine sahiptir.
  • Paylaşılan değişkenlerin donanımı modellemek için kullanılamayacağı ve yan etkileri olmadan kod enstrümantasyonu için ayrılması gerektiği düşünülebilir. Ancak birkaç EDA satıcısı tarafından çok bağlantı noktalı Rastgele Erişimli Belleklerin (RAM) bellek düzlemini modellemek için tavsiye edilen VHDL kalıpları, paylaşılan değişkenleri kullanır. Yani evet, paylaşılan değişkenler belirli durumlarda sentezlenebilir.

Sözde rastgele bir oluşturucu #

Peudo-rastgele üreteçler, simülasyon ortamları tasarlarken sıklıkla kullanışlıdır. Aşağıdaki VHDL paketi, “boolean”, “bit” ve “bit_vector” sözde rasgele bir üreteci tasarlamak için korumalı türlerin nasıl kullanılacağını gösterir. Rastgele “std_ulogic_vektör”, “imzalı”, “imzasız” oluşturmak için kolayca genişletilebilir. Rastgele sınırlara ve tek tip bir dağılıma sahip rasgele tamsayılar oluşturmak için genişletmek biraz daha zor ama yapılabilir.

Paket beyanı #

Korumalı bir tür, tüm genel alt program erişimcilerinin bildirildiği bir bildirime sahiptir. Rastgele oluşturucumuz için, bir tohum başlatma prosedürünü ve rastgele bir “boolean”, “bit” veya “bit_vektör” döndüren üç saf olmayan işlevi herkese açık hale getireceğiz. Aynı parametrelerle herhangi birinin farklı çağrıları farklı değerler döndürebileceğinden, işlevlerin saf olamayacağına dikkat edin.

-- file rnd_pkg.vhd
package rnd_pkg is
    type rnd_generator is protected
        procedure init(seed: bit_vector);
        impure function get_boolean return boolean;
        impure function get_bit return bit;
        impure function get_bit_vector(size: positive) return bit_vector;
    end protected rnd_generator;
end package rnd_pkg;

Paket gövdesi #

Korumalı tip gövdesi, iç veri yapılarını (üyeler) ve alt program gövdelerini tanımlar. Rastgele oluşturucumuz, dört dokunuşla 128 bitlik bir Doğrusal Geri Besleme Kaydırma Kaydı’na (LFSR) dayanmaktadır. ‘durum’ değişkeni, LFSR’nin mevcut durumunu saklar. Özel bir “atma” prosedürü, jeneratör her kullanıldığında LFSR’yi kaydırır.

-- file rnd_pkg.vhd
package body rnd_pkg is
    type rnd_generator is protected body
        constant len: positive := 128;
        constant default_seed: bit_vector(1 to len) := X"8bf052e898d987c7c31fc71c1fc063bc";
        type tap_array is array(natural range <>) of positive range 1 to len;
        constant taps: tap_array(0 to 3) := (128, 126, 101, 99);

        variable state: bit_vector(1 to len) := default_seed;

        procedure throw(n: positive := 1) is
            variable tmp: bit;
        begin
            for i in 1 to n loop
                tmp := '1';
                for j in taps'range loop
                    tmp := tmp xnor state(taps(j));
                end loop;
                state := tmp & state(1 to len - 1);
            end loop;
        end procedure throw;

        procedure init(seed: bit_vector) is
            constant n:   natural            := seed'length;
            constant tmp: bit_vector(1 to n) := seed;
            constant m:   natural            := minimum(n, len);
        begin
            state         := (others => '0');
            state(1 to m) := tmp(1 to m);
        end procedure init;

        impure function get_boolean return boolean is
            constant res: boolean := state(len) = '1';
        begin
            throw;
            return res;
        end function get_boolean;

        impure function get_bit return bit is
            constant res: bit := state(len);
        begin
            throw;
            return res;
        end function get_bit;

        impure function get_bit_vector(size: positive) return bit_vector is
            variable res: bit_vector(1 to size);
        begin
            if size <= len then
                res := state(len + 1 - size to len);
                throw(size);
            else
                res(1 to len) := state;
                throw(len);
                res(len + 1 to size) := get_bit_vector(size - len);
            end if;
            return res;
        end function get_bit_vector;
    end protected body rnd_generator;
end package body rnd_pkg;

Rastgele oluşturucu daha sonra aşağıdaki gibi bir OO tarzında kullanılabilir:

-- file rnd_sim.vhd
use std.env.all;
use std.textio.all;
use work.rnd_pkg.all;

entity rnd_sim is
end entity rnd_sim;

architecture sim of rnd_sim is
    shared variable rnd: rnd_generator;
begin
    process
        variable l: line;
    begin
        rnd.init(X"fe39_3d9f_24bb_5bdc_a7d0_2572_cbff_0117");
        for i in 1 to 10 loop
            write(l, rnd.get_boolean);
            write(l, HT);
            write(l, rnd.get_bit);
            write(l, HT);
            write(l, rnd.get_bit_vector(10));
            writeline(output, l);
        end loop;
        finish;
    end process;
end architecture sim;
