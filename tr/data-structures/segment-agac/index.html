Segment Ağacına Giriş #

Diyelim ki bir dizimiz var:

+-------+-----+-----+-----+-----+-----+-----+
| Index |  0  |  1  |  2  |  3  |  4  |  5  |
+-------+-----+-----+-----+-----+-----+-----+
| Value |  -1 |  3  |  4  |  0  |  2  |  1  |
+-------+-----+-----+-----+-----+-----+-----+

Bu dizi üzerinde bazı sorgular yapmak istiyoruz. Örneğin:

  • Dizin-2 ile dizin-4 arasındaki minimum değer nedir? -> 0
  • Dizin-0 ile dizin-3 arasındaki maksimum değer nedir? -> 4
  • İndeks-1 ile indeks-5 arasındaki toplam nedir? -> 10

Nasıl bulacağız?

Kaba Kuvvet:
Diziyi başlangıç ​​dizininden bitiş dizinine doğru hareket ettirebilir ve sorgumuzu yanıtlayabiliriz. Bu yaklaşımda, her sorgu “O(n)” zamanını alır, burada n başlangıç ​​dizini ile bitiş dizini arasındaki farktır. Peki ya milyonlarca sayı ve milyonlarca sorgu varsa? m sorgular için “O(mn)” zaman alır. Dolayısıyla dizimizdeki 10⁵ değerler için, eğer 10⁵ sorgular yaparsak, en kötü durum için 10¹⁰ öğeleri geçmemiz gerekecek.

Dinamik Programlama:
Farklı aralıklar için değerleri saklamak için 6X6 matris oluşturabiliriz. Minimum aralık sorgusu (RMQ) için dizimiz şöyle görünür:

              0     1     2     3     4     5
     +-----+-----+-----+-----+-----+-----+-----+
     |     |  -1 |  3  |  4  |  0  |  2  |  1  |
     +-----+-----+-----+-----+-----+-----+-----+
  0  |  -1 |  -1 |  -1 |  -1 |  -1 |  -1 |  -1 |
     +-----+-----+-----+-----+-----+-----+-----+
  1  |  3  |     |  3  |  3  |  0  |  0  |  0  |
     +-----+-----+-----+-----+-----+-----+-----+
  2  |  4  |     |     |  4  |  0  |  0  |  0  |
     +-----+-----+-----+-----+-----+-----+-----+
  3  |  0  |     |     |     |  0  |  0  |  0  |
     +-----+-----+-----+-----+-----+-----+-----+
  4  |  2  |     |     |     |     |  2  |  1  |
     +-----+-----+-----+-----+-----+-----+-----+
  5  |  1  |     |     |     |     |     |  1  |
     +-----+-----+-----+-----+-----+-----+-----+

Bu matris yapısına sahip olduğumuzda, tüm RMQ’ları cevaplamak yeterli olacaktır. Burada Matrix[i][j] index-i ile index-j arasındaki minimum değeri depolar. Örneğin: dizin-2 ile dizin-4 arasındaki minimum değer Matris[2][4] = 0‘dır. Bu matris, sorguyu ‘O(1)’ zamanında yanıtlar, ancak bu matrisi oluşturmak O(n²) zaman ve depolamak için ‘O(n²)’ alanı alır. Yani n gerçekten büyük bir sayıysa, bu çok iyi ölçeklenmiyor.

Segment Ağacı:
Bir segment ağacı, aralıkları veya segmentleri depolamak için bir ağaç veri yapısıdır. Depolanan segmentlerden hangisinin belirli bir nokta içerdiğini sorgulamaya izin verir. Bir segment ağacı oluşturmak O(n) zaman alır, bakımı için O(n) boşluk alır ve bir sorguyu O(logn) zamanında cevaplar.

Şimdi bu ağaç üzerinde bir aralık sorgusu yapalım. Aralık sorgusu yapmak için üç koşulu kontrol etmemiz gerekir:

  • Kısmi Örtüşme: Her iki yaprağı da kontrol ediyoruz.
  • Toplam Örtüşme: Düğümde depolanan değeri döndürürüz.
  • Örtüşme Yok: Çok büyük bir değer veya sonsuzluk döndürürüz.

Diyelim ki [2,4] aralığını kontrol etmek istiyoruz, bu da indeks-2 ile 4 arasındaki minimum değeri bulmak istediğimiz anlamına geliyor. Kökten başlayacağız ve düğümlerimizdeki aralığın sorgu aralığımızla örtüşüp örtüşmediğini kontrol edeceğiz. Burada,

  • [2,4] [0,5] tamamen örtüşmez. Bu yüzden her iki yönü de kontrol edeceğiz.
    • At left subtree, [2,4] partially overlaps [0,2]. We’ll check both directions.
      • At left subtree, [2,4] does not overlap [0,1], so this is not going to contribute to our answer. We return infinity.
      • At right subtree, [2,4] totally overlaps [2,2]. We return 4.
        The minimum of these two returned values(4, infinity) is 4. We get 4 from this portion.
    • At right subtree, [2,4] partially overlaps. So we’ll check both directions.
      • At left subtree, [2,4] completely overlaps [3,4]. We return 0.
      • At right subtree, [2,4] doesn’t overlap [5,5]. We return infinity.
        Bu iki döndürülen değerin (0, sonsuz) minimumu 0‘dır. Bu kısımdan 0 alıyoruz.
  • Döndürülen değerlerin (4,0) minimumu 0‘dır. Bu, [2,4] aralığında istenen minimum değerimizdir.

Şimdi sorgumuz ne olursa olsun, bu segment ağacından istenen değeri bulmanın maksimum 4 adım alacağını kontrol edebiliriz.

Dizi Kullanarak Segment Ağacının Uygulanması #

Diyelim ki bir dizimiz var: Item = {-1, 0, 3, 6}. Belirli bir aralıktaki minimum değeri bulmak için SegmentTree dizisini oluşturmak istiyoruz. Segment ağacımız şöyle görünecek:

Düğümlerin altındaki sayılar, SegmentTree dizimizde depolayacağımız her bir değerin dizinlerini gösterir. 4 eleman depolamak için 7 boyutunda bir diziye ihtiyacımız olduğunu görebiliriz. Bu değer şu şekilde belirlenir:

Procedure DetermineArraySize(Item):
multiplier := 1
n := Item.size
while multiplier < n
    multiplier := multiplier * 2
end while
size := (2 * multiplier) - 1
Return size

Yani uzunluğu 5 olan bir dizimiz olsaydı, SegmentTree dizimizin boyutu şöyle olurdu: (8 * 2) - 1 = 15. Şimdi, bir düğümün sol ve sağ alt öğesinin konumunu belirlemek için, eğer bir düğüm i dizinindeyse, konumu:

  • Sol Çocuk şu şekilde gösterilir: (2 * i) + 1.
  • Sağ Çocuk şu şekilde gösterilir: (2 * i) + 2.

Ve i dizinindeki herhangi bir düğümün üstünün dizini şu şekilde belirlenebilir: (i - 1)/2.

Dolayısıyla, örneğimizi temsil eden SegmentTree dizisi şöyle görünür:

   0     1     2     3     4     5     6
+-----+-----+-----+-----+-----+-----+-----+
|  -1 |  -1 |  3  |  -1 |  0  |  3  |  6  |
+-----+-----+-----+-----+-----+-----+-----+

Bu diziyi oluşturmak için sözde koda bakalım:

Procedure ConstructTree(Item, SegmentTree, low, high, position):
if low is equal to high
    SegmentTree[position] := Item[low]
else
    mid := (low+high)/2
    constructTree(Item, SegmentTree, low, mid, 2*position+1)
    constructTree(Item, SegmentTree, mid+1, high, 2*position+2)
    SegmentTree[position] := min(SegmentTree[2*position+1], SegmentTree[2*position+2])
end if

İlk önce, değerlerin girişini alırız ve boyutu olarak Item dizisinin uzunluğunu kullanarak SegmentTree dizisini “infinity” ile başlatırız. Prosedürü aşağıdakileri kullanarak çağırıyoruz:

  • düşük = Öğe dizisinin başlangıç ​​dizini.
  • yüksek = Item dizisinin bitirme indeksi.
  • konum = 0, Segment Ağacımızın kökünü gösterir.

Minimum Aralık Sorgusu Gerçekleştirme #

Bir RMQ gerçekleştirme prosedürü girişte zaten gösterilmiştir. Minimum Aralık Sorgusunu kontrol etmek için sözde kod şöyle olacaktır:

Procedure RangeMinimumQuery(SegmentTree, qLow, qHigh, low, high, position):
if qLow <= low and qHigh >= high        //Total Overlap
    Return SegmentTree[position]
else if qLow > high || qHigh < low      //No Overlap
    Return infinity
else                                    //Partial Overlap
    mid := (low+high)/2
    Return min(RangeMinimumQuery(SegmentTree, qLow, qHigh, low, mid, 2*position+1),
               RangeMinimumQuery(SegmentTree, qLow, qHigh, mid+1, high, 2*position+2))
end if

Burada, qLow = Sorgumuzun alt aralığı', qHigh = Sorgumuzun üst aralığı’. “düşük = Öğe dizisinin başlangıç ​​dizini”, “yüksek = Öğe dizisinin bitiş dizini”, “konum = kök = 0”. Şimdi daha önce oluşturduğumuz örneği kullanarak prosedürü anlamaya çalışalım:

+-----+-----+-----+-----+-----+-----+-----+

[1,3] aralığında minimum değeri bulmak istiyoruz.

Burada, ilk iki koşul karşılanmaz (kısmi örtüşme). +------+-------+-----+------+----------+-----+--------------+

+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  0  |   0  |     3    |     |              |
+------+-------+-----+------+----------+-----+--------------+

Bu özyinelemeli çağrı tamamlandığından, özyineleme tablomuzun bir önceki satırına geri dönüyoruz. Alırız:

+------+-------+-----+------+----------+-----+--------------+
| qLow | qHigh | low | high | position | mid | calling line |
+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  0  |   3  |     0    |  1  |       1      |
+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  0  |   1  |     1    |  0  |       1      |
+------+-------+-----+------+----------+-----+--------------+

Prosedürümüzde RangeMinimumQuery-2 çağrısını yürütüyoruz. Bu sefer ‘düşük = orta+1 = 1’, ‘yüksek = 1’ ve ‘konum = 2*konum+2 = 4’ pasajlarını geçiyoruz. arama hattımız **2** olarak değişti. Alırız:

+------+-------+-----+------+----------+-----+--------------+
| qLow | qHigh | low | high | position | mid | calling line |
+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  0  |   3  |     0    |  1  |       1      |
+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  0  |   1  |     1    |  0  |       2      |
+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  1  |   1  |     4    |     |              |
+------+-------+-----+------+----------+-----+--------------+

Yani önceki düğümün sağ çocuğuna gidiyoruz. Bu sefer tam bir örtüşme var. SegmentAğacı[konum] = SegmentAğacı[4] = 2 değerini döndürürüz.

+------+-------+-----+------+----------+-----+--------------+
| qLow | qHigh | low | high | position | mid | calling line |
+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  0  |   3  |     0    |  1  |       1      |
+------+-------+-----+------+----------+-----+--------------+

Çağıran fonksiyona geri dönersek, iki çağrı fonksiyonunun döndürülen iki değerinin minimumunun ne olduğunu kontrol ediyoruz. Açıkçası minimum 2, çağıran fonksiyona 2 geri dönüyoruz. Özyineleme tablomuz şöyle görünür:

+------+-------+-----+------+----------+-----+--------------+
| qLow | qHigh | low | high | position | mid | calling line |
+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  0  |   3  |     0    |  1  |       1      |
+------+-------+-----+------+----------+-----+--------------+

Segment ağacımızın sol kısmına bakmayı bitirdik. Şimdi buradan RangeMinimumQuery-2yi arayacağız. ‘düşük = orta+1 = 1+1 =2’, ‘yüksek = 3’ ve ‘konum = 2*konum+2 = 2’yi geçeceğiz. Bizim masamız:

+------+-------+-----+------+----------+-----+--------------+
| qLow | qHigh | low | high | position | mid | calling line |
+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  0  |   3  |     0    |  1  |       1      |
+------+-------+-----+------+----------+-----+--------------+
|   1  |   3   |  2  |   3  |     2    |     |              |
+------+-------+-----+------+----------+-----+--------------+

Tam bir örtüşme var. Bu yüzden şu değeri döndürürüz: SegmentTree[konum] = SegmentTree[2] = 0. Bu iki çocuğun çağrıldığı yerden özyinelemeye geri dönüyoruz ve minimum (4,0), yani 0 değerini alıyoruz ve değeri döndürüyoruz.

Prosedürü gerçekleştirdikten sonra, indeks-1‘den indeks-3’e kadar olan minimum olan 0 elde ederiz.

Bu yordamın çalışma zamanı karmaşıklığı “O(logn)” şeklindedir; burada n, Items dizisindeki öğelerin sayısıdır. Bir Aralık Maksimum Sorgusu gerçekleştirmek için satırı değiştirmemiz gerekir:

Return min(RangeMinimumQuery(SegmentTree, qLow, qHigh, low, mid, 2*position+1),
               RangeMinimumQuery(SegmentTree, qLow, qHigh, mid+1, high, 2*position+2))

ile birlikte:

Return max(RangeMinimumQuery(SegmentTree, qLow, qHigh, low, mid, 2*position+1),
               RangeMinimumQuery(SegmentTree, qLow, qHigh, mid+1, high, 2*position+2))

Diyelim ki, zaten bir segment ağacı oluşturdunuz. Dizinin değerlerini güncellemeniz gerekiyor, bu sadece segment ağacınızın yaprak düğümlerini değiştirmekle kalmayacak, aynı zamanda bazı düğümlerdeki minimum/maksimum değerleri de değiştirecektir. Buna bir örnekle bakalım:

Bu, 8 elemanlı minimum segment ağacımızdır. Size hızlı bir hatırlatma yapmak için, her düğüm, yanlarında belirtilen aralığın minimum değerini temsil eder. Diyelim ki dizimizin ilk öğesinin değerini 3 artırmak istiyoruz. Bunu nasıl yapabiliriz? RMQ’yu uyguladığımız yolu takip edeceğiz. Süreç şöyle görünecektir:

  • İlk başta kökün üzerinden geçiyoruz. [0,0] [0,7] ile kısmen örtüşüyor, her iki yöne de gidiyoruz. - At left subtree, [0,0] partially overlaps with [0,3], we go to both directions. - At left subtree, [0,0] partially overlaps with [0,1], we go to both directions. - At left subtree, [0,0] totally overlaps with [0,0], and since its the leaf node, we update the node by increasing it s value by 3. And return the value -1 + 3 = 2. - At right subtree, [1,1] doesn’t overlap with [0,0], we return the value at the node(2).
    The minimum of these two returned values(2, 2) are 2, so we update the value of the current node and return it. - At right subtree [2,3] doesn’t overlap with [0,0], we return the value of the node. (1).
    Since the minimum of these two returned values (2, 1) is 1, we update the value of the current node and return it.
    • At right subtree [4,7] doesn’t overlap with [0,0], we return the value of the node. (1).
  • Son olarak, döndürülen iki değerin (1,1) minimumu 1 olduğundan kök düğümün değeri güncellenir.

Tek bir düğümü güncellemenin ‘O(logn)’ zaman karmaşıklığı gerektirdiğini görebiliriz, burada n yaprak düğümlerinin sayısıdır. Bu nedenle, bazı düğümleri i‘den j‘ye güncellememiz istenirse, O(nlogn) zaman karmaşıklığına ihtiyaç duyacağız. Bu, büyük bir n değeri için hantal hale gelir. Tembel olalım. e., yalnızca gerektiğinde çalışın. Nasıl? Bir aralığı güncellememiz gerektiğinde, bir düğümü güncelleyeceğiz ve alt öğesinin güncellenmesi gerektiğini işaretleyeceğiz ve gerektiğinde güncelleyeceğiz. Bunun için bir segment ağacıyla aynı boyutta lazy dizisine ihtiyacımız var. Başlangıçta lazy dizisinin tüm öğeleri, bekleyen bir güncelleme olmadığını gösteren 0 olacaktır. lazy[i] içinde sıfır olmayan bir öğe varsa, bu öğenin herhangi bir sorgu işlemi yapmadan önce segment ağacındaki i düğümünü güncellemesi gerekir. Bunu nasıl yapacağız? Bir örneğe bakalım:

Diyelim ki örnek ağacımız için bazı sorguları yürütmek istiyoruz. Bunlar:

  • [0,3] artırın 3.
  • [0,3] artırın 1.
  • [0,0] artırın 2.

[0,3]‘ü 3: artır

  • Kök düğümden başlıyoruz. Öncelikle bu değerin güncel olup olmadığını kontrol ediyoruz. Bunun için 0 olan lazy dizimizi kontrol ediyoruz, yani değer güncel. [0,3] kısmen örtüşüyor [0,7]. Yani her iki yöne de gidiyoruz.

    • At the left subtree, there’s no pending update. [0,3] totally overlaps [0,3]. So we update the value of the node by 3. So the value becomes -1 + 3 = 2. This time, we’re not going to go all the way. Instead of going down, we update the corresponding child in the lazy tree of our current node and increment them by 3. We also return the value of the current node.
    • At the right subtree, there’s no pending update. [0,3] doesn’t overlap [4,7]. So we return the value of the current node (1).
      The minimum of two returned values (2, 1) is 1. We update the root node to 1.

Tembel Ağacımızın düğümlerindeki sıfır olmayan değerler temsil eder, bu düğümlerde ve aşağıda bekleyen güncellemeler vardır. Gerekirse bunları güncelleyeceğiz. [0,3] aralığında minimum ne olduğu sorulursa, kök düğümün sol alt ağacına geleceğiz ve bekleyen güncelleme olmadığından 2 döndüreceğiz, hangisi doğru. Dolayısıyla bu işlem, segment ağacı algoritmamızın doğruluğunu etkilemez.

[0,3]‘ü 1: artır

  • Kök düğümden başlıyoruz. Bekleyen güncelleme yok. [0,3] kısmen örtüşüyor [0,7]. Yani her iki yöne de gidiyoruz.
    • In the left subtree, there’s no pending update. [0,3] completely overlaps [0,3]. We update the current node: 2 + 1 = 3. Since this is an internal node, we update its children in the Lazy Tree to be incremented by 1. We’ll also return the value of the current node (3).
    • In the right subtree, there’s no pending update. [0,3] doesn’t overlap [4,7]. We return the value of the current node (1).
  • Döndürülen en az iki değer (3, 1) alarak kök düğümü güncelliyoruz.

Segment Ağacımız ve Tembel Ağacımız şöyle görünecek:

Gördüğünüz gibi, Lazy Tree’deki güncellemeleri topluyoruz, ancak aşağı itmiyoruz. Tembel Yayılımın anlamı budur. Eğer kullanmamış olsaydık, değerleri yapraklara kadar indirmemiz gerekiyordu, bu da bize gereksiz zaman karmaşıklığına mal olacaktı.

[0,0]‘ı 2: artır

  • Kök düğümden başlıyoruz. Kök güncel olduğundan ve [0,0] kısmen [0,7] ile örtüştüğünden, her iki yöne de gidiyoruz.
    • At the left subtree, the current node is up-to-date and [0,0] partially overlaps [0,3], we go to both directions.
      • At the left subtree, the current node in Lazy Tree has a non-zero value. So there is an update which has not been propagated yet to this node. We’re going to first update this node in our Segment Tree. So this becomes -1 + 4 = 3. Then we’re going to propagate this 4 to its children in the Lazy Tree. As we have already updated the current node, we’ll change the value of current node in Lazy Tree to 0. Then [0,0] partially overlaps [0,1], so we go to both directions.
        • At the left node, the value needs to be updated since there is a non-zero value in the current node of Lazy Tree. So we update the value to -1 + 4 = 3. Now, since [0,0] totally overlaps [0,0], we update the value of the current node to 3 + 2 = 5. This is a leaf node, so we need not to propagate the value anymore. We update the corresponding node at the Lazy Tree to 0 since all the values have been propagated up till this node. We return the value of the current node(5).
        • At the right node, the value needs to be updated. So the value becomes: 4 + 2 = 6. Since [0,0] doesn’t overlap [1,1], we return the value of the current node(6). We also update the value in Lazy Tree to 0. No propagation is needed since this is a leaf node.
          We update the current node using the minimum of two returned values(5,6). We return the value of the current node(5).
      • At the right subtree, there’s a pending update. We update the value of the node to 1 + 4 = 5. Since this is not a leaf node, we propagate the value to its children in our Lazy Tree and update the current node to 0. Tembel Yayılımda güncelleme için sözde kod şöyle olacaktır:

Procedure UpdateSegmentTreeLazy(segmentTree, LazyTree, startRange,
                                endRange, delta, low, high, position):
if low > high                                                //out of bounds
    Return
end if
if LazyTree[position] is not equal to 0                      //update needed
    segmentTree[position] := segmentTree[position] + LazyTree[position]
    if low is not equal to high                              //non-leaf node    
        LazyTree[2 * position + 1] := LazyTree[2 * position + 1] + delta
        LazyTree[2 * position + 2] := LazyTree[2 * position + 2] + delta
    end if
    LazyTree[position] := 0
end if
if startRange > low or endRange < high                       //doesn't overlap
    Return
end if
if startRange <= low && endRange >= high                     //total overlap
    segmentTree[position] := segmentTree[position] + delta
    if low is not equal to high
        LazyTree[2 * position + 1] := LazyTree[2 * position + 1] + delta
        LazyTree[2 * position + 2] := LazyTree[2 * position + 2] + delta
    end if
    Return
end if
//if we reach this portion, this means there's a partial overlap
mid := (low + high) / 2
UpdateSegmentTreeLazy(segmentTree, LazyTree, startRange,
                                endRange, delta, low, mid, 2 * position + 1)
UpdateSegmentTreeLazy(segmentTree, LazyTree, startRange,
                                endRange, delta, mid + 1, high, 2 * position + 2)
segmentTree[position] := min(segmentTree[2 * position + 1], 
                                                    segmentTree[2 * position + 2])

Tembel Yayılımdaki RMQ için sözde kod şöyle olacaktır:

Procedure RangeMinimumQueryLazy(segmentTree, LazyTree, qLow, qHigh, low, high, position):
if low > high
    Return infinity
end if
if LazyTree[position] is not equal to 0                      //update needed
    segmentTree[position] := segmentTree[position] + LazyTree[position]
    if low is not equal to high
        segmentTree[position] := segmentTree[position] + LazyTree[position]
    if low is not equal to high                              //non-leaf node    
        LazyTree[2 * position + 1] := LazyTree[2 * position + 1] + LazyTree[position]
        LazyTree[2 * position + 2] := LazyTree[2 * position + 2] + LazyTree[position]
    end if
    LazyTree[position] := 0
end if
if qLow > high and qHigh < low                                //no overlap
    Return infinity
end if
if qLow <= low and qHigh >= high                              //total overlap
    Return segmentTree[position]
end if
//if we reach this portion, this means there's a partial overlap
mid := (low + high) / 2
Return min(RangeMinimumQueryLazy(segmentTree, LazyTree, qLow, qHigh, 
                                        low, mid, 2 * position + 1),
           RangeMinimumQueryLazy(segmentTree, LazyTree, qLow, qHigh,
                                        mid + 1, high, 2 * position + 1)
