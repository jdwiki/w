Birleşim bulma (veya ayrık küme) veri yapısı, basit bir veri yapısıdır ve bir dizi öğenin ayrık kümelere bölünmesidir. Her kümenin, onu diğer kümelerden ayırt etmek için kullanılabilecek bir temsilcisi vardır.

Birçok algoritmada kullanılır, örn. minimum yayılan ağaçları Kruskal’ın algoritması ile hesaplamak, bağlı bileşenleri yönsüz grafiklerde hesaplamak ve daha pek çok şey.

Teori #

Union find veri yapıları aşağıdaki işlemleri sağlar:

  • ‘make_sets(n)’, ’n’ tekil tonlarıyla bir birlik bulma veri yapısını başlatır
  • “find(i)”, “i” öğesi kümesi için bir temsilci döndürür
  • ‘union(i,j)’, ‘i’ ve ‘j’ içeren kümeleri birleştirir

Her “i” öğesi için bir parent öğesi “parent[i]” depolayarak “0” ile “n - 1” arasındaki öğelerin bölümümüzü temsil ediyoruz, bu da sonunda “i"yi içeren kümenin bir temsilcisine yol açar. . Bir öğenin kendisi bir temsilci ise, kendi ebeveynidir, yani ‘parent[i] == i’.

Böylece, tekil kümelerle başlarsak, her eleman kendi temsilcisidir:

Bu ana işaretçileri takip ederek belirli bir kümenin temsilcisini bulabiliriz.

Şimdi kümeleri nasıl birleştirebileceğimizi görelim: 0 ve 1 öğelerini ve 2 ve 3 öğelerini birleştirmek istiyorsak, bunu 0’ın ebeveynini 1’e ve 2’nin ebeveynini 3’e ayarlayarak yapabiliriz.

‘den sonra

Bu basit durumda, yalnızca ana işaretçi öğelerinin kendisi değiştirilmelidir. Bununla birlikte, daha büyük kümeleri birleştirmek istiyorsak, başka bir kümeyle birleştirilecek kümenin temsilcisinin ebeveyn işaretçisini her zaman değiştirmeliyiz: merge(0,3)’ten sonra üst öğeyi ayarladık. 0 içeren kümenin temsilcisinden 3 içeren kümenin temsilcisine

Her küme için temsilciyi doğrudan kendi üst düğümünde depolayarak bunu kolaylaştırabiliriz. Temel uygulama #

Birleşim bulma veri yapısının en temel uygulaması, yapının her öğesi için bir üst öğeyi depolayan bir ‘ana’ dizisinden oluşur. Bir ‘i’ öğesi için bu ebeveyn ‘işaretçilerini’ takip etmek, bizi ‘i’ içeren kümenin ‘j = bul(i)’ temsilcisine götürür, burada ‘parent[j] = j’ bulunur.

using std::size_t;

class union_find {
private:
    std::vector<size_t> parent;  // Parent for every node

public:
    union_find(size_t n) : parent(n) {
        for (size_t i = 0; i < n; ++i)
            parent[i] = i;      // Every element is its own representative
    }

    size_t find(size_t i) const {
        if (parent[i] == i)     // If we already have a representative
            return i;           // return it
        return find(parent[i]); // otherwise return the parent's representative
    }

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi != pj) {        // If the elements are not in the same set: 
            parent[pi] = pj;   // Join the sets by marking pj as pi's parent
        }
    }
};

İyileştirmeler: Yol sıkıştırma #

Birleşim bulma veri yapısı üzerinde birçok “birleştirme” işlemi yaparsak, “üst” işaretçiler tarafından temsil edilen yollar oldukça uzun olabilir. Yol sıkıştırma, teori bölümünde daha önce açıklandığı gibi, bu sorunu gidermenin basit bir yoludur.

Her k-th birleştirme işleminden veya benzer bir şeyden sonra tüm veri yapısı üzerinde yol sıkıştırması yapmayı deneyebiliriz, ancak böyle bir işlemin çalışma süresi oldukça uzun olabilir.

Bu nedenle, yol sıkıştırması çoğunlukla yapının sadece küçük bir bölümünde, özellikle de bir kümenin temsilcisini bulmak için yürüdüğümüz yolda kullanılır. Bu, her özyinelemeli alt çağrıdan sonra “bul” işleminin sonucunu kaydederek yapılabilir:

size_t find(size_t i) const {
    if (parent[i] == i)          // If we already have a representative
        return i;                // return it
    parent[i] = find(parent[i]); // path-compress on the way to the representative
    return parent[i];            // and return it
}

İyileştirmeler: Boyuta göre birleştirme #

Mevcut birleştirme uygulamamızda, kümelerin boyutunu dikkate almadan her zaman sol kümeyi sağ kümenin çocuğu olarak seçiyoruz. Bu kısıtlama olmadan, bir öğeden temsilcisine giden yollar (yol sıkıştırması olmadan) oldukça uzun olabilir, bu nedenle ‘find’ çağrılarında büyük çalışma zamanlarına yol açar.

Bu nedenle bir başka yaygın iyileştirme, tam olarak söylediği şeyi yapan boyuta göre birleştirme buluşsal yöntemidir: İki kümeyi birleştirirken, her zaman daha büyük kümeyi küçük kümenin ebeveyni olarak ayarlarız, böylece en fazla [] yol uzunluğuna yol açarız. ]1 adım:

Sınıfımızda, her eleman için 1 olarak başlatılan ek bir std::vector<size_t> size üyesi saklıyoruz. İki küme birleştirildiğinde, büyük küme küçük kümenin ebeveyni olur ve iki boyutu toplarız:

private:
    ...
    std::vector<size_t> size;

public:
    union_find(size_t n) : parent(n), size(n, 1) { ... }

    ...

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi == pj) {            // If the elements are in the same set: 
            return;                // do nothing
        }
        if (size[pi] > size[pj]) { // Swap representatives such that pj
            std::swap(pi, pj);     // represents the larger set
        }
        parent[pi] = pj;           // attach the smaller set to the larger one
        size[pj] += size[pi];      // update the size of the larger set
    }

İyileştirmeler: Rütbeye göre birlik #

Boyuta göre birleşim yerine yaygın olarak kullanılan başka bir buluşsal yöntem, sıraya göre birleşim buluşsal yöntemidir.

Temel fikri, aslında kümelerin tam boyutunu saklamamız gerekmemesidir, boyutun bir yaklaşımı (bu durumda: kabaca kümenin boyutunun logaritması), boyuta göre birleştirme ile aynı hızı elde etmek için yeterlidir.

Bunun için, aşağıdaki gibi verilen bir kümenin rütbesi kavramını tanıtıyoruz:

  • Singleton’lar rank 0’a sahiptir.
  • Sıralaması eşit olmayan iki küme birleştirilirse, sıra değişmeden bırakılırken daha büyük sıralı küme ebeveyn olur.
  • İki eşit rank kümesi birleştirilirse, biri diğerinin ebeveyni olur (seçim isteğe bağlı olabilir), rankı artırılır.

dereceye göre birleştirmenin bir avantajı, alan kullanımıdır: Maksimum sıralama kabaca gibi arttıkça, tüm gerçekçi girdi boyutları için, sıralama tek bir bayt ( n < 2^255 beri).

Sıralamaya göre birliğin basit bir uygulaması şöyle görünebilir:

private:
    ...
    std::vector<unsigned char> rank;

public:
    union_find(size_t n) : parent(n), rank(n, 0) { ... }

    ...

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi == pj) {
            return;
        }
        if (rank[pi] < rank[pj]) {
            // link the smaller group to the larger one
            parent[pi] = pj;
        } else if (rank[pi] > rank[pj]) {
            // link the smaller group to the larger one
            parent[pj] = pi;
        } else {
            // equal rank: link arbitrarily and increase rank
            parent[pj] = pi;
            ++rank[pi];
        }
    }

Son İyileştirme: Sınır dışı depolama ile derece bazında birleştirme #

Yol sıkıştırma ile birlikte, sıralamaya göre birleştirme, birlik bulma veri yapılarında neredeyse sabit zamanlı işlemler gerçekleştirirken, sıralamayı sınırların dışında depolayarak ‘rank’ depolamasından tamamen kurtulmamızı sağlayan son bir numara vardır. “ebeveyn” dizisinin girdileri. Aşağıdaki gözlemlere dayanmaktadır:

  • Aslında sıralamayı yalnızca temsilciler için saklamamız gerekiyor, diğer öğeler için değil. Bu temsilciler için bir “ebeveyn” kaydetmemiz gerekmez.
  • Şu ana kadar ‘parent[i]’ en fazla ‘size - 1’ boyutundadır, yani daha büyük değerler kullanılmamıştır.
  • Tüm sıralar en fazla ‘dir.

Bu bizi aşağıdaki yaklaşıma getiriyor:

  • parent[i] == i koşulu yerine, şimdi temsilcileri şu şekilde tanımlıyoruz: parent[i] >= size
  • Bu sınır dışı değerleri kümenin sıralarını depolamak için kullanırız, yani temsili ‘i’ sıra ‘parent[i] - boyut’ olan küme
  • Böylece üst diziyi, ‘parent[i] = i’ yerine ‘parent[i] = size’ ile başlatırız, yani her küme, 0. sıradaki kendi temsilcisidir.

Sıralama değerlerini yalnızca “boyut” ile dengelediğimiz için, “birleştirme” uygulamasında “sıra” vektörünü “ana” vektörle değiştirebiliriz ve yalnızca “bul"daki koşul tanımlayıcı temsilcileri değiştirmemiz gerekir:

Sıra ve yol sıkıştırmasına göre birleştirme kullanılarak uygulama tamamlandı:

using std::size_t;

class union_find {
private:
    std::vector<size_t> parent;

public:
    union_find(size_t n) : parent(n, n) {} // initialize with parent[i] = n

    size_t find(size_t i) const {
        if (parent[i] >= parent.size()) // If we already have a representative
            return i;                   // return it
        return find(parent[i]);         // otherwise return the parent's repr.
    }

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi == pj) {
            return;
        }
        if (parent[pi] < parent[pj]) {
            // link the smaller group to the larger one
            parent[pi] = pj;
        } else if (parent[pi] > parent[pj]) {
            // link the smaller group to the larger one
            parent[pj] = pi;
        } else {
            // equal rank: link arbitrarily and increase rank
            parent[pj] = pi;
            ++parent[pi];
        }
    }
};
