XOR Bağlantılı Liste #

Bir XOR Bağlantılı listesi aynı zamanda Bellek-Verimli Bağlantılı Liste olarak da adlandırılır. Çift bağlantılı bir listenin başka bir şeklidir. Bu, büyük ölçüde XOR mantık kapısına ve özelliklerine bağlıdır.

Bunun adı neden Verimli Bellek Bağlantılı Listedir?

Buna böyle denir, çünkü bu, geleneksel bir çift bağlantılı listeden daha az bellek kullanır.

Bu, Çift Bağlantılı Listeden farklı mı?

**Evet öyle.

Bir Çift Bağlantılı Liste, sonraki ve önceki düğümü işaret eden iki işaretçiyi saklar. Temel olarak, geri dönmek istiyorsanız, ‘back’ işaretçisinin gösterdiği adrese gidersiniz. İleri gitmek istiyorsanız, ‘sonraki’ işaretçisinin gösterdiği adrese gidersiniz. Gibi:

Bir Bellek-Verimli Bağlantılı Liste veya diğer bir deyişle XOR Bağlantılı Liste iki yerine yalnızca bir işaretçiye sahip. Bu, önceki adresi (addr (önceki)) XOR (^) sonraki adresi (adr (sonraki)) saklar. Bir sonraki düğüme geçmek istediğinizde belirli hesaplamalar yaparsınız ve bir sonraki düğümün adresini bulursunuz. Bu, önceki düğüme gitmek için aynıdır. Şuna benzer:

Nasıl çalışır?

Bağlantılı listenin nasıl çalıştığını anlamak için XOR’un (^) özelliklerini bilmeniz gerekir:

|-------------|------------|------------|
|    Name     |   Formula  |    Result  |
|-------------|------------|------------|
| Commutative |    A ^ B   |    B ^ A   |
|-------------|------------|------------|
| Associative | A ^ (B ^ C)| (A ^ B) ^ C|
|-------------|------------|------------|
| None (1)    |    A ^ 0   |     A      |
|-------------|------------|------------|
| None (2)    |    A ^ A   |     0      |
|-------------|------------|------------|
| None (3)    | (A ^ B) ^ A|     B      |
|-------------|------------|------------|

Şimdi bunu bir kenara bırakalım ve her bir düğümün neyi sakladığını görelim.

İlk düğüm veya head, önceki düğüm veya adres olmadığından 0 ^ addr (sonraki) değerini depolar. Buna benziyor.

Sonra ikinci düğüm addr (önceki) ^ addr (sonraki) depolar. Buna benziyor.

Listenin kuyruğu bir sonraki düğüme sahip değildir, bu nedenle addr (önceki) ^ 0 değerini depolar. Buna benziyor.

Baştan sonraki düğüme geçiş

Diyelim ki şimdi ilk düğümdesiniz veya A düğümündesiniz. Şimdi B düğümüne geçmek istiyorsunuz. Bunu yapmanın formülü şudur:

Address of Next Node = Address of Previous Node ^ pointer in the current Node

Yani bu olurdu:

addr (next) = addr (prev) ^ (0 ^ addr (next))

Bu başlık olduğundan, önceki adres basitçe 0 olur, yani:

addr (next) = 0 ^ (0 ^ addr (next))

Parantezleri kaldırabiliriz:

addr (next) = 0 ^ 0 addr (next)

none (2) özelliğini kullanarak 0 ^ 0 daima 0 olacaktır diyebiliriz:

addr (next) = 0 ^ addr (next)

’none (1)’ özelliğini kullanarak, bunu şu şekilde basitleştirebiliriz:

addr (next) = addr (next)

Bir sonraki düğümün adresini aldınız!

Bir düğümden sonraki düğüme geçiş

Şimdi diyelim ki bir önceki ve sonraki düğümü olan bir orta düğümdeyiz.

Formülü uygulayalım:

Address of Next Node = Address of Previous Node ^ pointer in the current Node

Şimdi değerleri değiştirin:

addr (next) = addr (prev) ^ (addr (prev) ^ addr (next))

Parantezleri Kaldırın:

addr (next) = addr (prev) ^ addr (prev) ^ addr (next)

’none (2)’ özelliğini kullanarak şunları basitleştirebiliriz:

addr (next) = 0 ^ addr (next)

’none (1)’ özelliğini kullanarak şunları basitleştirebiliriz:

addr (next) = addr (next)

Ve anladın!

Bir düğümden daha önce bulunduğunuz düğüme geçiş

Başlığı anlamıyorsanız, bunun anlamı, X düğümündeyseniz ve şimdi Y düğümüne geçtiyseniz, daha önce ziyaret edilen düğüme veya temelde X düğümüne geri dönmek istersiniz.

Bu zahmetli bir iş değil. Yukarıda bahsettiğimi unutmayın, bulunduğunuz adresi geçici bir değişkende saklıyorsunuz. Yani ziyaret etmek istediğiniz düğümün adresi bir değişkende yatıyor:

addr (prev) = temp_addr

Bir düğümden önceki düğüme geçiş

Bu, yukarıda belirtilenle aynı değildir. Demek istediğim, Z düğümündeydiniz, şimdi Y düğümündesiniz ve X düğümüne gitmek istiyorsunuz.

Bu, bir düğümden bir sonraki düğüme geçmekle neredeyse aynıdır. Sadece bu tam tersi. Bir program yazarken, bir düğümden bir sonraki düğüme geçerken bahsettiğim adımların aynısını kullanacaksınız, sadece listede bir sonraki öğeyi bulmaktan daha önceki öğeyi buluyorsunuz.

C’deki Örnek Kod

/* C/C++ Implementation of Memory efficient Doubly Linked List */
#include <stdio.h>
#include <stdlib.h>
 
// Node structure of a memory efficient doubly linked list
struct node
{
    int data;
    struct node* npx;  /* XOR of next and previous node */
};
 
/* returns XORed value of the node addresses */
struct node* XOR (struct node *a, struct node *b)
{
    return (struct node*) ((unsigned int) (a) ^ (unsigned int) (b));
}
 
/* Insert a node at the begining of the XORed linked list and makes the
   newly inserted node as head */
void insert(struct node **head_ref, int data)
{
    // Allocate memory for new node
    struct node *new_node  = (struct node *) malloc (sizeof (struct node) );
    new_node->data = data;
 
    /* Since new node is being inserted at the begining, npx of new node
       will always be XOR of current head and NULL */
    new_node->npx = XOR(*head_ref, NULL);
 
    /* If linked list is not empty, then npx of current head node will be XOR 
       of new node and node next to current head */
    if (*head_ref != NULL)
    {
        // *(head_ref)->npx is XOR of NULL and next. So if we do XOR of 
        // it with NULL, we get next
        struct node* next = XOR((*head_ref)->npx,  NULL);
        (*head_ref)->npx = XOR(new_node, next);
    }
 
    // Change head
    *head_ref = new_node;
}
 
// prints contents of doubly linked list in forward direction
void printList (struct node *head)
{
    struct node *curr = head;
    struct node *prev = NULL;
    struct node *next;
 
    printf ("Following are the nodes of Linked List: \n");
 
    while (curr != NULL)
    {
        // print current node
        printf ("%d ", curr->data);
 
        // get address of next node: curr->npx is next^prev, so curr->npx^prev
        // will be next^prev^prev which is next
        next = XOR (prev, curr->npx);
 
        // update prev and curr for next iteration
        prev = curr;
        curr = next;
    }
}
 
// Driver program to test above functions
int main ()
{
    /* Create following Doubly Linked List
       head-->40<-->30<-->20<-->10   */
    struct node *head = NULL;
    insert(&head, 10);
    insert(&head, 20);
    insert(&head, 30);
    insert(&head, 40);
 
    // print the created list
    printList (head);
 
    return (0);
}

Yukarıdaki kod iki temel işlevi yerine getirir: ekleme ve çaprazlama.

  • Ekle: Bu, ‘insert’ işlevi tarafından gerçekleştirilir. Bu, başlangıçta yeni bir düğüm ekler. Bu fonksiyon çağrıldığında yeni düğümü baş, bir önceki baş düğümü ikinci düğüm olarak koyar.

  • Geçiş: Bu, ‘printList’ işlevi tarafından gerçekleştirilir. Her düğümden geçer ve değeri yazdırır.

İşaretçilerin XOR’sinin C/C++ standardı tarafından tanımlanmadığını unutmayın. Dolayısıyla yukarıdaki uygulama tüm platformlarda çalışmayabilir.

Bağlantılı Listelere Giriş #

Bağlantılı liste, bir “işaretçi” aracılığıyla diğer düğüm(ler)e bağlanan, düğüm adı verilen veri öğelerinin doğrusal bir koleksiyonudur. Aşağıda, bir baş referansı olan tek başına bağlantılı bir liste bulunmaktadır.

         ┌─────────┬─────────┐   ┌─────────┬─────────┐         
 HEAD ──▶│  data   │"pointer"│──▶│  data   │"pointer"│──▶ null 
         └─────────┴─────────┘   └─────────┴─────────┘

[tek](https://www.wikiod.com/tr/data-structures/baglantl-liste#Tek Bağlantılı Liste) ve [doubly](http:// stackoverflow.com/documentation/data-structures/2967/linked-list/23473/doubly-linked-list) bağlantılı listeler ve dairesel bağlantılı listeler.

Avantajlar

  • Bağlantılı listeler, program çalışırken büyüyüp küçülebilen, bellek ayırabilen ve serbest bırakabilen dinamik bir veri yapısıdır.

  • Düğüm ekleme ve silme işlemleri, bağlantılı bir listede kolayca uygulanır.

  • Yığınlar ve kuyruklar gibi doğrusal veri yapıları, bağlantılı bir liste ile kolayca uygulanır.

  • Bağlantılı listeler erişim süresini azaltabilir ve bellek yükü olmadan gerçek zamanlı olarak genişleyebilir.

Tek Bağlantılı Liste #

Tek Bağlantılı Listeler, bir tür [bağlı liste](https://www.wikiod.com/tr/data-structures/baglantl-liste#Bağlantılı Listelere Giriş). Tek başına bağlantılı bir listenin düğümleri, genellikle “sonraki” olmak üzere başka bir düğüme yalnızca bir “işaretçi"ye sahiptir. Her düğümün başka bir düğüme yalnızca tek bir “işaretçisi” olduğu için buna tek bağlantılı liste denir. Tek başına bağlantılı bir liste bir baş ve/veya kuyruk referansına sahip olabilir. Bir kuyruk referansına sahip olmanın avantajı, O(1) haline gelen ‘getFromBack’, ‘addToBack’ ve ‘removeFromBack’ durumlarıdır.

         ┌─────────┬─────────┐   ┌─────────┬─────────┐         
 HEAD ──▶│  data   │"pointer"│──▶│  data   │"pointer"│──▶ null 
         └─────────┴────△────┘   └─────────┴─────────┘         
          SINGLE        │                                      
          REFERENCE ────┘

Listeyi Atla #

Atlama listeleri, doğru düğüme atlamanıza izin veren bağlantılı listelerdir. Bu, normal tekil bağlantılı listeden çok daha hızlı olan bir yöntemdir. Temelde [tek başına bağlantılı bir listedir](https://www.wikiod.com/tr/data-structures/baglantl-liste#Tek Bağlantılı Liste) ancak işaretçiler bir düğümden sonraki düğüme gitmiyor , ancak birkaç düğüm atlanıyor. Böylece adı “Listeyi Atla”.

Çift Bağlantılı Liste #

Çift Bağlantılı Listeler, bir tür [bağlı liste](https://www.wikiod.com/tr/data-structures/baglantl-liste#Bağlantılı Listelere Giriş). Çift bağlantılı bir listenin düğümleri, diğer düğümlere yönelik iki “işaretçiye” sahiptir, “sonraki” ve “önceki”. Çift bağlantılı liste olarak adlandırılır, çünkü her düğümün diğer düğümlere yalnızca iki “işaretçisi” vardır. Çift bağlantılı bir listenin bir baş ve/veya kuyruk işaretçisi olabilir.

         ┌─────────┬─────────┬─────────┐   ┌─────────┬─────────┬─────────┐         
 null ◀──┤previous │  data   │  next   │◀──┤previous │  data   │  next   │         
         │"pointer"│         │"pointer"│──▶│"pointer"│         │"pointer"│──▶ null 
         └─────────┴─────────┴─────────┘   └─────────┴─────────┴─────────┘         
                          ▲                     △                   △              
                     HEAD │                     │     DOUBLE        │              
                                                └──── REFERENCE ────┘

Çift bağlantılı listeler, tek bağlantılı listelerden daha az alan verimlidir; ancak bazı işlemler için zaman verimliliğinde önemli iyileştirmeler sunarlar. Basit bir örnek, baş ve kuyruk referansı olan çift bağlantılı bir liste için alınacak öğenin dizinine bağlı olarak head veya tail’den başlayacak olan ‘get’ işlevidir. Bir “n”-eleman listesi için, “n/2 + i” indeksli elemanı almak için, baş/kuyruk referansları ile tek başına bağlantılı bir liste “n/2 + i” düğümlerini geçmelidir, çünkü “geri dönemez”. kuyruk. Baş/kuyruk referanslarına sahip çift bağlantılı bir listenin yalnızca “n/2 - i” düğümlerini geçmesi gerekir, çünkü kuyruktan “geri dönebilir” ve listeyi ters sırada geçebilir.

Java’da tekil bağlantılı liste için temel bir uygulama - listenin sonuna tamsayı değerleri ekleyebilen, karşılaşılan ilk değeri listeden kaldırabilen, belirli bir anda bir değerler dizisi döndürebilen ve belirli bir değerin mevcut olup olmadığını belirleyebilen listede.

    private int value;

SinglyLinkedList.java

package com.example.so.ds;


/**
 * <p> Basic single-linked-list implementation </p>
 * @since 20161008
 * @author Ravindra HV
 * @version 0.2
 */
public class SinglyLinkedList {
    
    private Node head;
    private volatile int size;
    
    public int getSize() {
        return size;
    }
    
    public synchronized void append(int value) {
        
        Node entry = new Node(value);
        if(head == null) {
            head=entry;
        }
        else {
            Node temp=head;
            while( temp.getNext() != null) {
                temp=temp.getNext();
            }
            temp.setNext(entry);
        }
        
        size++;
    }
    
    
    public synchronized boolean removeFirst(int value) {
        boolean result = false;
        
        if( head == null ) { // or size is zero..
            // no action
        }
        else if( head.getValue() == value ) {
            head = head.getNext();
            result = true;
        }
        else {

            Node previous = head;
            Node temp = head.getNext();
            while( (temp != null) && (temp.getValue() != value) ) {
                previous = temp;
                temp = temp.getNext();
            }
            
            if((temp != null) && (temp.getValue() == value)) { // if temp is null then not found..
                previous.setNext( temp.getNext() );
                result = true;
            }

        }
        
        if(result) {
            size--;
        }
        
        return result;
        
    }
    
    public synchronized int[] snapshot() {
        Node temp=head;
        int[] result = new int[size];
        for(int i=0;i<size;i++) {
            result[i]=temp.getValue();
            temp = temp.getNext();
        }
        return result;
    }
    
    public synchronized boolean contains(int value) {
        boolean result = false;
        Node temp = head;
        
        while(temp!=null) {
            if(temp.getValue() == value) {
                result=true;
                break;
            }
            temp=temp.getNext();
        }
        return result;
    }
    
}
