Sözdizimi #

  • Cat *cat = [[Cat alloc] init]; // Cat türünde cat nesnesi oluştur
  • Köpek *köpek = [[Köpek tahsisi] init]; // Dog türünde köpek nesnesi oluştur
  • NSObject *someObject = [NSObject tahsisi]; [someObject init]; // bunu yapma
  • XYZObject *nesne = [XYZObject yeni]; // Başlatma için HİÇBİR argüman gerekmiyorsa, nesneler oluşturmak için new kullanın
  • NSString *someString = @“Merhaba, Dünya!”; // literal sözdizimi ile bir NSString oluşturma
  • NSNumber *myFloat = @3.14f; // Değişmez sözdizimi kullanarak bir NSNumber oluşturmak için başka bir örnek
  • NSNumber *myInt = @(84 / 2); // Kutulu bir ifade kullanarak bir nesne oluşturun

Tahsis ve başlatma arasındaki fark #

Çoğu nesne yönelimli dilde, bir nesne için bellek ayırmak ve onu başlatmak atomik bir işlemdir:

// Both allocates memory and calls the constructor
MyClass object = new MyClass();

Objective-C’de bunlar ayrı işlemlerdir. ‘alloc’ sınıf yöntemleri (ve onun tarihsel kardeşi ‘allocWithZone:’), Objective-C çalışma zamanının gerekli belleği ayırmasını sağlar ve onu temizler. Birkaç dahili değer dışında, tüm özellikler ve değişkenler 0/NO/nil olarak ayarlanır.

Bu durumda nesne zaten “geçerlidir” ama biz her zaman nesneyi gerçekten kurmak için bir initializer olarak adlandırdığımız bir yöntemi çağırmak isteriz. Bunlar, diğer dillerdeki constructors ile aynı amaca hizmet eder. Geleneksel olarak, bu yöntemler “init” ile başlar. Dil açısından bakıldığında, bunlar sadece normal yöntemlerdir.

// Allocate memory and set all properties and variables to 0/NO/nil.
MyClass *object = [MyClass alloc];
// Initialize the object.
object = [object init];

// Shorthand:
object = [[MyClass alloc] init];

Başlatma değerleriyle sınıflar oluşturma #

#import <Foundation/Foundation.h>
@interface Car:NSObject {
    NSString *CarMotorCode;
    NSString *CarChassisCode;
}

- (instancetype)initWithMotorValue:(NSString *) motorCode andChassisValue:(NSInteger)chassisCode;
- (void) startCar;
- (void) stopCar;

@end

@implementation Car

- (instancetype)initWithMotorValue:(NSString *) motorCode andChassisValue:(NSInteger)chassisCode{
    CarMotorCode = motorCode;
    CarChassisCode = chassisCode;
    return self;
}

- (void) startCar {...}
- (void) stopCar {...}

@end

Car nesnelerini başlatmak için initWithMotorValue: type andChassisValue: type yöntemi kullanılacaktır.

Singleton Sınıfı #

Singleton Sınıfı nedir?

Bir singleton sınıfı, bir uygulama kaç kez isterse, aynı örneği döndürür. Normal bir sınıfın aksine, A singleton nesnesi, sınıfının kaynaklarına küresel bir erişim noktası sağlar.

Singleton Sınıfları Ne Zaman Kullanılır?

Singleton’lar, bazı genel hizmet veya kaynaklar sunan sınıflar gibi, bu tek kontrol noktasının istendiği durumlarda kullanılır.

Singleton Sınıfları Nasıl Oluşturulur

İlk olarak, Yeni bir dosya oluşturun ve onu ‘NSObject’den alt sınıflayın. Herhangi bir isim verin, burada ‘CommonClass’ kullanacağız. Xcode şimdi sizin için CommonClass.h ve CommonClass.m dosyaları oluşturacak.

CommonClass.h dosyanızda:

#import <Foundation/Foundation.h>

@interface CommonClass : NSObject {
}
+ (CommonClass *)sharedObject;
@property NSString *commonString;
@end

CommonClass.m Dosyanızda:

#import "CommonClass.h"

@implementation CommonClass

+ (CommonClass *)sharedObject {
    static CommonClass *sharedClass = nil;
    static dispatch_once_t onceToken;
    dispatch_once(&onceToken, ^{
        sharedClass = [[self alloc] init];
    });
    return sharedClass;
}

- (id)init {
    if (self = [super init]) {
        self.commonString = @"this is string";
    }
    return self;
}

@end

Singleton Sınıfları Nasıl Kullanılır

Daha önce oluşturduğumuz Singleton Class’a, ilgili modülde CommonClass.h dosyasını import ettiğiniz sürece proje içerisinde herhangi bir yerden erişilebilir olacaktır. Singleton Class’ta paylaşılan verileri değiştirmek ve erişmek için, aşağıdaki gibi “sharedObject” yöntemi kullanılarak erişilebilen o sınıfın paylaşılan Nesnesine erişmeniz gerekir:

[CommonClass sharedObject]

Shared Class’taki öğeleri okumak veya değiştirmek için aşağıdakileri yapın:

NSString *commonString = [[CommonClass sharedObject].commonString; //Read the string in singleton class

NSString *newString = @"New String";
[CommonClass sharedObject].commonString = newString;//Modified the string in singleton class

“örnek türü” dönüş türü #

Objective-C, yalnızca bir yöntem tarafından döndürülen tür olarak kullanılabilen “instancetype” adlı özel bir türü destekler. Alıcı nesnenin sınıfını değerlendirir.

Aşağıdaki sınıf hiyerarşisini göz önünde bulundurun:

@interface Foo : NSObject

- (instancetype)initWithString:(NSString *)string;

@end

@interface Bar : Foo
@end

[[Foo alloc] initWithString:@"abc"] çağrıldığında, derleyici dönüş türünün Foo * olduğu sonucunu çıkarabilir. ‘Bar’ sınıfı, ‘Foo’dan türetildi, ancak başlatıcının bildirimini geçersiz kılmadı. Yine de, “instancetype” sayesinde, derleyici “[[Bar alloc] initWithString:@“xyz”]” öğesinin “Bar *” türünde bir değer döndürdüğü sonucunu çıkarabilir.

“-[Foo initWithString:]” dönüş türünün bunun yerine “Foo *” olduğunu düşünün: “[[Bar alloc] initWithString:]” öğesini çağırırsanız, derleyici bir “Foo *” döndürüldüğünü, bir Bar * olarak geliştiricinin niyetidir. “Instancetype” bu sorunu çözdü.

“instancetype”, başlatıcılar, tekli erişimciler gibi statik yöntemler ve bir “id” döndürmek için gereken alıcı sınıfın bir örneğini döndürmek isteyen diğer yöntemler tanıtılmadan önce. Sorun şu ki, “id”, “herhangi bir tür nesne” anlamına gelir. Bu nedenle derleyici, NSString *wrong = [[Foo alloc] initWithString:@"abc"]; öğesinin yanlış türde bir değişkene atandığını algılayamaz.

Bu sorun nedeniyle, başlatıcılar, dönüş değeri olarak “id” yerine her zaman “instancetype” kullanmalıdır.

Jenerikleri Belirtme #

Tıpkı ‘NSArray’ veya ‘NSDictionary’ gibi jeneriklerle kendi sınıflarınızı geliştirebilirsiniz.

@interface MyClass<__covariant T>

@property (nonnull, nonatomic, strong, readonly) NSArray<T>* allObjects;

- (void) addObject:(nonnull T)obj;

@end
