Yaşam döngüsü yöntemleri, kodu çalıştırmak ve bileşen ömrünün farklı noktalarında bileşeninizle etkileşim kurmak için kullanılmalıdır. Bu yöntemler, bir bileşenin Takılması, Güncellenmesi ve Bağlantısının Kaldırılmasına dayalıdır.

Bileşen Oluşturma #

Bir React bileşeni oluşturulduğunda bir dizi fonksiyon çağrılır:

  • React.createClass (ES5) kullanıyorsanız, 5 adet kullanıcı tanımlı fonksiyon çağrılır.
  • Eğer class Component, React.Component'i genişletir (ES6) kullanıyorsanız, 3 adet kullanıcı tanımlı fonksiyon çağrılır.

getDefaultProps() (yalnızca ES5) #

Bu, çağrılan ilk yöntemdir.

Bileşen başlatıldığında tanımlanmazlarsa, bu işlev tarafından döndürülen prop değerleri varsayılan olarak kullanılacaktır.

Aşağıdaki örnekte, aksi belirtilmedikçe “this.props.name” varsayılan olarak “Bob” olarak ayarlanacaktır:

getDefaultProps() {
  return {
    initialCount: 0,
    name: 'Bob'
  };
}

getInitialState() (yalnızca ES5) #

Bu, çağrılan ikinci yöntemdir.

GetInitialState()‘in dönüş değeri, React bileşeninin ilk durumunu tanımlar. React çerçevesi bu işlevi çağıracak ve dönüş değerini ’this.state’e atayacaktır.

Aşağıdaki örnekte, “this.state.count”, “this.props.initialCount” değeriyle başlatılacaktır:

getInitialState() {
  return {
    count : this.props.initialCount
  };
}

componentWillMount() (ES5 ve ES6) #

Bu çağrılan üçüncü yöntemdir.

Bu işlev, bileşen DOM’a eklenmeden önce son değişiklikleri yapmak için kullanılabilir.

componentWillMount() {
  ...
}

render() (ES5 ve ES6) #

Bu çağrılan dördüncü yöntemdir.

‘render()’ işlevi, bileşenin durumunun ve donanımlarının saf bir işlevi olmalıdır. Oluşturma işlemi sırasında bileşeni temsil eden ve yerel bir DOM bileşeninin (ör. “

”) veya bir bileşik bileşenin temsili olması gereken tek bir öğe döndürür. Hiçbir şey oluşturulmayacaksa, “null” veya “undefined” döndürebilir.

Bu işlev, bileşenin özelliklerinde veya durumunda herhangi bir değişiklikten sonra geri çağrılır.

render() {
  return (
    <div>
      Hello, {this.props.name}!
    </div>
  );
}

componentDidMount() (ES5 ve ES6) #

Bu, beşinci yöntem olarak adlandırılır.

Bileşen monte edilmiştir ve artık bileşenin DOM düğümlerine erişebilirsiniz, ör. “refs” aracılığıyla.

Bu yöntem aşağıdakiler için kullanılmalıdır:

  • Zamanlayıcıların hazırlanması

  • Veri işleniyor

  • Olay dinleyicileri ekleme

  • DOM öğelerini manipüle etme

    componentDidMount() { … }

ES6 Sözdizimi #

Bileşen ES6 sınıfı sözdizimi kullanılarak tanımlanırsa, “getDefaultProps()” ve “getInitialState()” işlevleri kullanılamaz.

Bunun yerine, ‘defaultProps’umuzu sınıf üzerinde statik bir özellik olarak bildiririz ve sınıfımızın yapıcısında durum şeklini ve başlangıç ​​durumunu bildiririz. Bunların her ikisi de, başka bir React yaşam döngüsü işlevi çağrılmadan önce, yapım sırasında sınıfın örneğinde ayarlanır.

Aşağıdaki örnek, bu alternatif yaklaşımı göstermektedir:

class MyReactClass extends React.Component {
  constructor(props){
    super(props);

    this.state = {
      count: this.props.initialCount
    };
  }
  
  upCount() {
    this.setState((prevState) => ({
      count: prevState.count + 1
    }));
  }
  
  render() {
    return (
      <div>
        Hello, {this.props.name}!<br />
        You clicked the button {this.state.count} times.<br />
        <button onClick={this.upCount}>Click here!</button>
      </div>
    );
  }
}

MyReactClass.defaultProps = {
  name: 'Bob',
  initialCount: 0
};

getDefaultProps() değiştiriliyor #

Bileşen özellikleri için varsayılan değerler, sınıfın “defaultProps” özelliği ayarlanarak belirlenir:

MyReactClass.defaultProps = {
  name: 'Bob',
  initialCount: 0
};

getInitialState() değiştiriliyor #

Bileşenin ilk durumunu ayarlamanın deyimsel yolu, yapıcıda “this.state” öğesini ayarlamaktır:

constructor(props){
  super(props);

  this.state = {
    count: this.props.initialCount
  };
}

Bileşen Kaldırma #

bileşenWillUnmount() #

Bu yöntem, bir bileşenin DOM’dan bağlantısı kaldırılmadan önce olarak adlandırılır.

Aşağıdakiler gibi temizlik işlemlerini gerçekleştirmek için iyi bir yerdir:

  • Olay dinleyicilerini kaldırma.

  • Zamanlayıcıları temizleme.

  • Durdurma yuvaları.

  • Redux durumlarını temizleme.

    componentWillUnmount(){ … }

‘componentWillUnMount’ içinde ekli olay dinleyicisinin kaldırılmasına bir örnek

import React, { Component } from 'react';

export default class SideMenu extends Component {

  constructor(props) {
    super(props);
    this.state = {
        ...
      };
    this.openMenu = this.openMenu.bind(this);
    this.closeMenu = this.closeMenu.bind(this);
  }

  componentDidMount() {
    document.addEventListener("click", this.closeMenu);
  }

  componentWillUnmount() {
    document.removeEventListener("click", this.closeMenu);
  }

  openMenu() {
    ...
  }

  closeMenu() {
    ...
  }

  render() {
    return (
      <div>
        <a
          href      = "javascript:void(0)"
          className = "closebtn"
          onClick   = {this.closeMenu}
        >
          ×
        </a>
        <div>
          Some other structure
        </div>
      </div>
    );
  }
}

Bileşen Güncellemesi #

bileşenWillReceiveProps(nextProps) #

Bu, özellik değişikliklerinde çağrılan ilk işlevdir.

Bileşenin özellikleri değiştiğinde, React bu işlevi yeni özellikler ile çağırır. this.props ile eski proplara, nextProps ile yeni proplara ulaşabilirsiniz.

Bu değişkenlerle eski ve yeni proplar arasında bazı karşılaştırma işlemleri yapabilir veya bir özellik değişikliği vb. nedeniyle işlevi çağırabilirsiniz.

componentWillReceiveProps(nextProps){
  if (nextProps.initialCount && nextProps.initialCount > this.state.count){
    this.setState({
      count : nextProps.initialCount
    });
  }
}

shouldComponentUpdate(nextProps, nextState) #

Bu, özellik değişikliklerinde çağrılan ikinci işlev ve durum değişikliklerinde ilk çağrılan işlevdir.

Varsayılan olarak, başka bir bileşen / bileşeniniz bileşeninizin bir özelliğini / durumunu değiştirirse, React bileşeninizin yeni bir sürümünü oluşturur. Bu durumda, bu işlev her zaman true değerini döndürür.

Bu işlevi geçersiz kılabilir ve bileşeninizin güncellenmesi gerekip gerekmediğini daha kesin olarak seçebilirsiniz.

Bu işlev çoğunlukla optimizasyon için kullanılır.

İşlevin false döndürmesi durumunda, güncelleme ardışık düzeni hemen durur.

componentShouldUpdate(nextProps, nextState){
  return this.props.name !== nextProps.name ||
    this.state.count !== nextState.count;
}

componentWillUpdate(nextProps, nextState) #

Bu işlev componentWillMount() gibi çalışır. Değişiklikler DOM’da değil, bu nedenle güncelleme gerçekleştirilmeden hemen önce bazı değişiklikler yapabilirsiniz.

/!\ : this.setState() kullanamazsınız.

componentWillUpdate(nextProps, nextState){}

işle() #

Bazı değişiklikler var, bu yüzden bileşeni yeniden oluşturun.

bileşenDidUpdate(prevProps, prevState) #

‘componentDidMount()’ ile aynı şeyler: DOM yenilendi, yani burada DOM üzerinde biraz çalışabilirsiniz.

componentDidUpdate(prevProps, prevState){}

Farklı durumlarda yaşam döngüsü yöntemi çağrısı #

Bu örnek, yaşam döngüsü yöntemlerinin nasıl kullanılacağı ve yöntemin ne zaman çağrılacağı hakkında konuşan diğer örneklerin tamamlayıcısı olarak hizmet eder.

Bu örnek, hangi yöntemlerin (componentWillMount, componentWillReceiveProps, vb.) çağrılacağını ve bir bileşen için hangi sıranın farklı olacağını farklı durumlarda özetlemektedir:

Bir bileşen başlatıldığında:

  1. getDefaultProps
  2. getInitialState
  3. bileşenWillMount
  4. işlemek
  5. bileşenDidMount

Bir bileşenin durumu değiştiğinde:

  1. gerekirBileşen Güncellemesi
  2. bileşenWillUpdate
  3. render
  4. componentDidUpdate

Bir bileşenin donanımları değiştiğinde:

  1. componentWillReceiveProps
  2. gerekirBileşen Güncellemesi
  3. bileşenWillUpdate
  4. işlemek
  5. bileşenDidUpdate

Bir bileşen sökülürken:

  1. componentWillUnmount

React Bileşen Kabı #

Bir React uygulaması oluştururken, bileşenleri birincil sorumluluklarına göre Sunum ve Kapsayıcı bileşenlere bölmek genellikle istenir. Sunum bileşenleri yalnızca verileri görüntülemekle ilgilidir - bir modeli bir görünüme dönüştüren işlevler olarak kabul edilebilirler ve genellikle bu işlevler olarak uygulanırlar. Tipik olarak herhangi bir dahili durumu korumazlar. Kapsayıcı bileşenleri, verileri yönetmekle ilgilidir. Bu, kendi devletleri aracılığıyla dahili olarak veya Redux gibi bir devlet yönetimi kitaplığı ile aracı olarak hareket ederek yapılabilir. Kapsayıcı bileşeni, verileri doğrudan görüntülemeyecek, bunun yerine verileri bir sunum bileşenine iletecektir.

// Container component
import React, { Component } from 'react';
import Api from 'path/to/api';

class CommentsListContainer extends Component {
    constructor() {
        super();
        // Set initial state
        this.state = { comments: [] }
    }

    componentDidMount() {
        // Make API call and update state with returned comments
        Api.getComments().then(comments => this.setState({ comments }));
    }

    render() {
        // Pass our state comments to the presentational component
        return (
            <CommentsList comments={this.state.comments} />;
        );
    }
}

// Presentational Component
const CommentsList = ({ comments }) => (
    <div>
        {comments.map(comment => (
            <div>{comment}</div>
        )}
    </div>
);

CommentsList.propTypes = {
    comments: React.PropTypes.arrayOf(React.PropTypes.string)
}
