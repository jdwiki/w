Temel Durum #

State in React bileşenlerinde, uygulamanızdaki verileri yönetmek ve iletmek için gereklidir. Bir JavaScript nesnesi olarak temsil edilir ve bileşen düzeyi kapsamına sahiptir, bileşeninizin özel verileri olarak düşünülebilir.

Aşağıdaki örnekte, bileşenimizin ‘constructor’ işlevinde bir başlangıç ​​durumu tanımlıyoruz ve bunu ‘render’ işlevinde kullanıyoruz.

class ExampleComponent extends React.Component {
  constructor(props){
    super(props);

    // Set-up our initial state
    this.state = {
      greeting: 'Hiya Buddy!'
    };
  }

  render() {
    // We can access the greeting property through this.state
    return(
      <div>{this.state.greeting}</div>
    );
  }
}

Ortak Antipattern #

‘props’u ‘state’ içine kaydetmemelisiniz. Bir anti-desen olarak kabul edilir. Örneğin:

export default class MyComponent extends React.Component {
    constructor() {
        super();

        this.state = {
            url: ''
        }

        this.onChange = this.onChange.bind(this);
    }

    onChange(e) {
        this.setState({
            url: this.props.url + '/days=?' + e.target.value
        });
    }

    componentWillMount() {
        this.setState({url: this.props.url});
    }

    render() {
        return (
            <div>
                <input defaultValue={2} onChange={this.onChange} />

                URL: {this.state.url}
            </div>
        )
    }
}

Prop “url”, “durum” üzerine kaydedilir ve ardından değiştirilir. Bunun yerine, değişiklikleri bir duruma kaydetmeyi seçin ve ardından hem “durum” hem de “sahneler"i kullanarak tam yolu oluşturun:

export default class MyComponent extends React.Component {
    constructor() {
        super();

        this.state = {
            days: ''
        }

        this.onChange = this.onChange.bind(this);
    }

    onChange(e) {
        this.setState({
            days: e.target.value
        });
    }

    render() {
        return (
            <div>
                <input defaultValue={2} onChange={this.onChange} />

                URL: {this.props.url + '/days?=' + this.state.days}
            </div>
        )
    }
}

Bunun nedeni, bir React uygulamasında tek bir doğruluk kaynağına sahip olmak istememizdir - yani tüm veriler tek bir bileşenin ve yalnızca bir bileşenin sorumluluğundadır. Verileri kendi durumu içinde saklamak ve props aracılığıyla diğer bileşenlere dağıtmak bu bileşenin sorumluluğundadır.

İlk örnekte, hem MyComponent sınıfı hem de ebeveyni, durumları içinde ‘url’yi koruyor. MyComponent’te state.url’yi güncellersek, bu değişiklikler üst öğeye yansıtılmaz. Tek doğru kaynağımızı kaybettik ve uygulamamız üzerinden veri akışını takip etmek giderek zorlaşıyor. Bunu ikinci örnekle karşılaştırın - url yalnızca ana bileşen durumunda tutulur ve MyComponent’te bir destek olarak kullanılır - bu nedenle tek bir doğruluk kaynağı sağlarız.

setState() #

React uygulamalarınızda UI güncellemeleri yapmanın birincil yolu, setState() işlevine yapılan bir çağrıdır. Bu işlev, sağladığınız yeni durum ile önceki durum arasında bir sığ birleştirme gerçekleştirecek ve bileşeninizin ve tüm ölenlerin yeniden oluşturulmasını tetikleyecektir.

Parametreler

  1. güncelleyici : Durumla birleştirilmesi gereken çok sayıda anahtar/değer çiftine sahip bir nesne veya böyle bir nesneyi döndüren bir işlev olabilir.
  2. callback (opsiyonel): setState() başarıyla yürütüldükten sonra yürütülecek bir fonksiyon. React tarafından setState()’e yapılan çağrıların atomik olduğu garanti edilmediğinden, setState()‘in başarıyla yürütüldüğünden emin olduktan sonra bazı eylemler gerçekleştirmek istiyorsanız bu bazen yararlı olabilir.

Kullanım:

“setState” yöntemi, durumla birleştirilmesi gereken bir dizi anahtar/değer çiftine sahip bir nesne olabilen bir “güncelleyici” argümanını veya “prevState” ve “props"tan hesaplanan böyle bir nesneyi döndüren bir işlevi kabul eder. `.

‘setState()’ öğesini bir Nesne ile ‘güncelleyici’ olarak kullanma #

//
// An example ES6 style component, updating the state on a simple button click.
// Also demonstrates where the state can be set directly and where setState should be used.
//
class Greeting extends React.Component {
    constructor(props) {
        super(props);
        this.click = this.click.bind(this);
        // Set initial state (ONLY ALLOWED IN CONSTRUCTOR)
        this.state = {
            greeting: 'Hello!'
        };
    }
    click(e) {
        this.setState({
              greeting: 'Hello World!'
        });
    }
    render() {
        return(
            <div>
                <p>{this.state.greeting}</p>
                <button onClick={this.click}>Click me</button>
            </div>
        );
    }
    
}

‘setState()’ işlevini ‘güncelleyici’ olarak kullanma #

//
// This is most often used when you want to check or make use 
// of previous state before updating any values.
//

this.setState(function(previousState, currentProps) {
  return {
    counter: previousState.counter + 1
  };
});

Bu, birden çok çağrının React tarafından bir araya toplanıp bir kerede yürütülebildiğinden ve durumu ayarlamak için geçerli destek öğelerini kullanırken tercih edilen yaklaşım olduğundan, “setState()” öğesine birden çok çağrının kullanıldığı bir nesne argümanı kullanmaktan daha güvenli olabilir.

this.setState({ counter: this.state.counter + 1 });
this.setState({ counter: this.state.counter + 1 });
this.setState({ counter: this.state.counter + 1 });

Bu çağrılar React tarafından ‘Object.assign()’ kullanılarak bir araya getirilebilir, bu da sayacın 3 yerine 1 artmasıyla sonuçlanır.

İşlevsel yaklaşım, durum ayarı mantığını bileşenlerin dışına taşımak için de kullanılabilir. Bu durum mantığının izolasyonuna ve yeniden kullanımına izin verir.

// Outside of component class, potentially in another file/module

function incrementCounter(previousState, currentProps) {
    return {
        counter: previousState.counter + 1
    };
}

// Within component

this.setState(incrementCounter);

Object ve geri arama işleviyle setState()‘i çağırma #

//
// 'Hi There' will be logged to the console after setState completes
//

this.setState({ name: 'John Doe' }, console.log('Hi there'));

Durum, Olaylar ve Yönetilen Kontroller #

İşte “yönetilen” giriş alanına sahip bir React bileşeni örneği. Giriş alanının değeri her değiştiğinde, bileşenin durumunu giriş alanının yeni değeriyle güncelleyen bir olay işleyicisi çağrılır. Olay işleyicisinde “setState” çağrısı, dom’daki bileşeni güncellemek için “render” çağrısını tetikleyecektir.

import React from 'react';
import {render} from 'react-dom';


class ManagedControlDemo extends React.Component {

  constructor(props){
    super(props);
    this.state = {message: ""};
  }

  handleChange(e){
    this.setState({message: e.target.value});
  }

  render() {
    return (
      <div>
        <legend>Type something here</legend>
          <input 
            onChange={this.handleChange.bind(this)} 
            value={this.state.message} 
            autoFocus />
        <h1>{this.state.message}</h1>
      </div>
   );
  } 
}


render(<ManagedControlDemo/>, document.querySelector('#app'));

Çalışma zamanı davranışını not etmek çok önemlidir. Bir kullanıcı giriş alanındaki değeri her değiştirdiğinde

  • ‘handleChange’ çağrılacak ve böylece
  • setState çağrılacak ve böylece
  • ‘render’ çağrılacak
