JSX

  • JSX’te Sahne
  • JSX’teki Çocuklar

Bu sayfada

  • JSX’te Sahne
  • JSX’teki Çocuklar

JSX, JavaScript’e XML sözdizimi ekleyen bir önişlemci adımıdır. React’i kesinlikle JSX olmadan kullanabilirsiniz, ancak JSX, React’i çok daha zarif hale getirir.

Tıpkı XML gibi, JSX etiketlerinin de bir etiket adı, nitelikleri ve alt öğeleri vardır. Bir öznitelik değeri tırnak içine alınmışsa, değer bir dizedir. Aksi takdirde, değeri parantez içine alın ve değer, ekteki JavaScript ifadesidir.

Temel olarak, JSX React.createElement(component, props, ...children) işlevi için yalnızca sözdizimsel şeker sağlar.

Yani, aşağıdaki JSX kodu:

class HelloMessage extends React.Component {
  render() {
    return <div>Hello {this.props.name}</div>;
  }
}

ReactDOM.render(<HelloMessage name="Kalo" />, mountNode);

Aşağıdaki JavaScript koduna kadar derler:

class HelloMessage extends React.Component {
  render() {
    return React.createElement(
      "div",
      null,
      "Hello ",
      this.props.name
    );
  }
}

ReactDOM.render(React.createElement(HelloMessage, { name: "Kalo" }), mountNode);

Sonuç olarak, JSX’teki aşağıdaki satırın ne bir dize ne de HTML olduğuna dikkat edin:

const element = <h1>Hello, world!</h1>;

JSX olarak adlandırılır ve JavaScript’in sözdizimi uzantısıdır. JSX size bir şablon dilini hatırlatabilir, ancak JavaScript’in tüm gücüyle birlikte gelir.

React ekibi, belgelerinde, kullanıcı arayüzünün nasıl görünmesi gerektiğini açıklamak için kullanılmasını önerdiklerini söylüyor.

JSX’te Sahne #

JSX’te props belirtmenin birkaç farklı yolu vardır.

JavaScript İfadeleri #

Herhangi bir JavaScript ifadesini, onu “{}” ile çevreleyerek prop olarak iletebilirsiniz. Örneğin, bu JSX’te:

<MyComponent count={1 + 2 + 3 + 4} />

‘MyComponent’ içinde, ‘1 + 2 + 3 + 4’ ifadesi değerlendirildiği için ‘props.count’ değeri ‘10’ olacaktır.

İfadeler ve for döngüleri JavaScript’teki ifadeler değilse, bu nedenle doğrudan JSX’te kullanılamazlar.

Dize Değişmezleri #

Tabii ki, herhangi bir “dize değişmezini” “prop” olarak da iletebilirsiniz. Bu iki JSX ifadesi eşdeğerdir:

<MyComponent message="hello world" />

<MyComponent message={'hello world'} />

Bir dize hazır bilgisini ilettiğinizde, değeri HTML-unescaped olur. Yani bu iki JSX ifadesi eşdeğerdir:

<MyComponent message="&lt;3" />

<MyComponent message={'<3'} />

Bu davranış genellikle alakalı değildir. Burada sadece eksiksiz olması için bahsedilmiştir.

Aksesuar Varsayılan Değeri #

Bir pervane için hiçbir değer iletmezseniz, varsayılan olarak “true” olur. Bu iki JSX ifadesi eşdeğerdir:

<MyTextBox autocomplete />

<MyTextBox autocomplete={true} />

Bununla birlikte, React ekibi belgelerinde bu yaklaşımı kullanmanın tavsiye edilmediğini, çünkü bunun ‘{’ yerine ‘{foo:foo}’ kısaltması olan ES6 nesne kısayolu ‘{foo}’ ile karıştırılabileceğini söylüyor. foo: doğru}`. Bu davranışın, HTML’nin davranışıyla eşleşmesi için orada olduğunu söylüyorlar.

Yayılmış Nitelikler #

Zaten bir nesne olarak props’a sahipseniz ve onu JSX’te iletmek istiyorsanız, props nesnesinin tamamını iletmek için spread operatörü olarak ... kullanabilirsiniz. Bu iki bileşen eşdeğerdir:

function Case1() {
  return <Greeting firstName="Kaloyab" lastName="Kosev" />;
}

function Case2() {
  const person = {firstName: 'Kaloyan', lastName: 'Kosev'};
  return <Greeting {...person} />;
}

JSX’teki Çocuklar #

Hem açılış etiketi hem de kapanış etiketi içeren JSX ifadelerinde, bu etiketler arasındaki içerik özel bir destek olarak iletilir: ‘props.children’. Çocukları geçmenin birkaç farklı yolu vardır:

Dize Değişmezleri #

Açılış ve kapanış etiketleri arasına bir dize koyabilirsiniz ve props.children sadece bu dize olacaktır. Bu, yerleşik HTML öğelerinin çoğu için kullanışlıdır. Örneğin:

<MyComponent>
    <h1>Hello world!</h1>
</MyComponent>

Bu geçerli bir JSX’dir ve MyComponent’teki ‘props.children’ basitçe ‘

Merhaba dünya!

’ olacaktır.

HTML’nin çıkışsız olduğuna dikkat edin, böylece genellikle JSX’i tıpkı HTML yazar gibi yazabilirsiniz.

Unutmayın, bu durumda JSX:

  • satırın başındaki ve sonundaki boşlukları kaldırır;
  • boş satırları kaldırır;
  • etiketlere bitişik yeni satırlar kaldırılır;
  • dize değişmezlerinin ortasında oluşan yeni satırlar tek bir boşlukta yoğunlaştırılır.

JSX Çocukları #

Çocuklar olarak daha fazla JSX öğesi sağlayabilirsiniz. Bu, iç içe bileşenleri görüntülemek için kullanışlıdır:

<MyContainer>
  <MyFirstComponent />
  <MySecondComponent />
</MyContainer>

Farklı çocuk türlerini karıştırabilirsiniz, böylece dize değişmezlerini JSX alt öğeleriyle birlikte kullanabilirsiniz. Bu, JSX’in HTML’ye benzemesinin başka bir yoludur, böylece bu hem geçerli JSX hem de geçerli HTML olur:

<div>
  <h2>Here is a list</h2>
  <ul>
    <li>Item 1</li>
    <li>Item 2</li>
  </ul>
</div>

Bir React bileşeninin birden çok React öğesi döndüremeyeceğini, ancak tek bir JSX ifadesinin birden çok çocuğu olabileceğini unutmayın. Bu nedenle, bir bileşenin birden çok şeyi oluşturmasını istiyorsanız, bunları yukarıdaki örnekte olduğu gibi bir ‘div’ içine sarabilirsiniz.

JavaScript İfadeleri #

Herhangi bir JavaScript ifadesini “{}” içine alarak alt öğe olarak iletebilirsiniz. Örneğin, bu ifadeler eşdeğerdir:

<MyComponent>foo</MyComponent>

<MyComponent>{'foo'}</MyComponent>

Bu genellikle isteğe bağlı uzunluktaki JSX ifadelerinin bir listesini oluşturmak için kullanışlıdır. Örneğin, bu bir HTML listesi oluşturur:

const Item = ({ message }) => (
  <li>{ message }</li>
);

const TodoList = () => {
  const todos = ['finish doc', 'submit review', 'wait stackoverflow review'];
  return (
    <ul>
      { todos.map(message => (<Item key={message} message={message} />)) }
    </ul>
  );
};

JavaScript ifadelerinin diğer çocuk türleri ile karıştırılabileceğini unutmayın.

Çocuk Olarak İşlevler #

Normalde, JSX’e eklenen JavaScript ifadeleri bir dize, bir React öğesi veya bunların bir listesi olarak değerlendirilir. Bununla birlikte, ‘props.children’, yalnızca React’in nasıl oluşturulacağını bildiği türleri değil, her türlü veriyi iletebildiği için diğer herhangi bir prop gibi çalışır. Örneğin, özel bir bileşeniniz varsa, onun “props.children” olarak bir geri arama yapmasını sağlayabilirsiniz:

const ListOfTenThings = () => (
  <Repeat numTimes={10}>
    {(index) => <div key={index}>This is item {index} in the list</div>}
  </Repeat>
);

// Calls the children callback numTimes to produce a repeated component
const Repeat = ({ numTimes, children }) => {
  let items = [];
  for (let i = 0; i < numTimes; i++) {
    items.push(children(i));
  }
  return <div>{items}</div>;
};

Özel bir bileşene geçirilen çocuklar, bu bileşen onları oluşturmadan önce React’in anlayabileceği bir şeye dönüştürdüğü sürece herhangi bir şey olabilir. Bu kullanım yaygın değildir, ancak JSX’in yapabileceklerini genişletmek istiyorsanız işe yarar.

Yok Sayılan Değerler #

“false”, “null”, “undefined” ve “true” öğelerinin geçerli alt öğeler olduğuna dikkat edin. Ama sadece render yapmıyorlar. Bu JSX ifadelerinin tümü aynı şeye dönüşecektir:

<MyComponent />

<MyComponent></MyComponent>

<MyComponent>{false}</MyComponent>

<MyComponent>{null}</MyComponent>

<MyComponent>{true}</MyComponent>

Bu, React öğelerini koşullu olarak oluşturmak için son derece kullanışlıdır. Bu JSX, yalnızca “showHeader” doğruysa bir

oluşturur:

<div>
  {showHeader && <Header />}
  <Content />
</div>

Önemli bir uyarı, “0” sayısı gibi bazı “yanlış” değerlerin hala React tarafından işlenmesidir. Örneğin, bu kod beklediğiniz gibi davranmayacaktır çünkü “props.messages” boş bir dizi olduğunda “0” yazdırılacaktır:

<div>
  {props.messages.length &&
    <MessageList messages={props.messages} />
  }
</div>

Bunu düzeltmeye yönelik bir yaklaşım, “&&“den önceki ifadenin her zaman boolean olduğundan emin olmaktır:

<div>
  {props.messages.length > 0 &&
    <MessageList messages={props.messages} />
  }
</div>

Son olarak, çıktıda “yanlış”, “doğru”, “boş” veya “tanımsız” gibi bir değerin görünmesini istiyorsanız, önce onu bir dizeye dönüştürmeniz gerektiğini unutmayın:

<div>
  My JavaScript variable is {String(myVariable)}.
</div>
