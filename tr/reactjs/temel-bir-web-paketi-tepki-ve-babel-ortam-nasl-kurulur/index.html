Temel bir web paketi, tepki ve babel ortamı nasıl kurulur

  • Görüntülerle özelleştirilmiş bir “Merhaba dünya” için bir ardışık düzen nasıl oluşturulur.

Bu sayfada

  • Görüntülerle özelleştirilmiş bir “Merhaba dünya” için bir ardışık düzen nasıl oluşturulur.

Bu inşa hattı tam olarak “üretime hazır” olarak adlandırdığınız şey değildir, ancak aradığınız geliştirme deneyimini elde etmek için ihtiyacınız olan şeyleri eklemeniz için sağlam bir başlangıç ​​sağlar. Bazı insanların (bazen ben de dahil olmak üzere) aldığı yaklaşım, Yeoman.io’nun veya başka bir yerden tamamen oluşturulmuş bir boru hattını almak ve daha sonra oradaki stile uyana kadar istemedikleri şeyleri çıkarmaktır. Bunda yanlış bir şey yok ama belki yukarıdaki örnekte tam tersi bir yaklaşım seçebilir ve çıplak kemiklerden oluşturabilirsiniz.

Eklemek isteyebileceğiniz bazı şeyler, bir test çerçevesi ve Mocha veya Jasmine ile Karma gibi kapsam istatistikleri gibi şeylerdir. ESLint ile astarlama. Webpack-dev-server’da sıcak modül değişimi, böylece Ctrl+S, F5 geliştirme deneyimini elde edebilirsiniz. Ayrıca mevcut işlem hattı yalnızca geliştirme modunda oluşturulur, bu nedenle bir üretim oluşturma görevi iyi olur.

Anladım!

“webpack.config.js"nin bağlam özelliğinde, yolumuzu tanımlamak için “__dirname"yi “/src” dizesiyle birleştirmek yerine düğüm yolu modülünü kullandığımıza dikkat edin, bunun nedeni windows’un eğik çizgilerden nefret etmesidir ]. Bu nedenle, çözümü platformlar arası daha uyumlu hale getirmek için bize yardımcı olması için kaldıraç düğümünü kullanın.

webpack.config.js özelliklerinin açıklaması

bağlam

Bu, göreli dosya yollarını çözümlemek amacıyla web paketinin kök yolu olarak kullanacağı dosya yoludur. Yani require('./index.html') kullandığımız index.jsx’te bu nokta aslında src/ dizinine çözümlenir çünkü onu bu özellikte böyle tanımladık.

giriş

Çözümü paketlemeye başlamak için web paketinin ilk göründüğü yer. Bu nedenle, index.jsx’te, gereksinimler ve içe aktarmalar ile çözümü bir araya getirdiğimizi göreceksiniz.

çıktı

Burası, web paketinin paket olarak bulduğu dosya dosyalarını nereye bırakması gerektiğini tanımladığımız yerdir. Paketlenmiş javascript ve stillerimizin bırakılacağı dosya için de bir isim tanımladık.

devSunucu

Bunlar webpack-dev-server’a özel ayarlardır. contentBase sunucunun kökünü nereye yapması gerektiğini tanımlar, burada dist/ klasörünü temelimiz olarak tanımladık. “port”, sunucunun barındırılacağı bağlantı noktasıdır. ‘open’, webpack-dev-server’a sunucuyu çalıştırdıktan sonra sizin için varsayılan tarayıcınızı açması talimatını vermek için kullanılan şeydir.

modül > yükleyiciler

Bu, web paketinin kullanması için bir eşleme tanımlar, böylece farklı dosyalar bulduğunda ne yapacağını bilir. “test” özelliği, web paketinin bu modülü uygulayıp uygulamayacağını belirlemek için kullanması için normal ifade verir, çoğu durumda dosya uzantılarında eşleşmeler vardır. ’loader’ veya ’loaders’, dosyayı web paketine yüklemek için kullanmak istediğimiz yükleyici modülünün adını verir ve bu yükleyicinin o dosya türünün paketlenmesiyle ilgilenmesine izin verir. Ayrıca javascript’te bir ‘query’ özelliği vardır, bu sadece yükleyiciye bir sorgu dizgisi sağlar, bu yüzden isteseydik muhtemelen html yükleyicide de bir sorgu özelliği kullanabilirdik. Bu sadece işleri yapmanın farklı bir yolu.

Görüntülerle özelleştirilmiş bir “Merhaba dünya” için bir ardışık düzen nasıl oluşturulur. #

1. Adım: Node.js’yi yükleyin

Oluşturacağınız derleme hattı Node.js’ye dayalıdır, bu nedenle ilk etapta bunu kurduğunuzdan emin olmalısınız. Node.js’nin nasıl kurulacağına ilişkin talimatlar için bununla ilgili SO belgelerine göz atabilirsiniz buradan

Adım 2: Projenizi bir düğüm modülü olarak başlatın

Proje klasörünüzü komut satırında açın ve aşağıdaki komutu kullanın:

npm init

Bu örneğin amaçları doğrultusunda, varsayılanları almakta özgürsünüz veya tüm bunların ne anlama geldiği hakkında daha fazla bilgi istiyorsanız, paket yapılandırmasının ayarlanmasıyla ilgili [bu][2] SO belgesine göz atabilirsiniz.

Adım 3: Gerekli npm paketlerini kurun

Bu örnek için gerekli paketleri kurmak için komut satırında aşağıdaki komutu çalıştırın:

npm install --save react react-dom

Ardından dev bağımlılıkları için şu komutu çalıştırın:

npm install --save-dev babel-core babel-preset-react babel-preset-es2015 webpack babel-loader css-loader style-loader file-loader image-webpack-loader

Son olarak, webpack ve webpack-dev-server, projenizin bir bağımlılığı olarak değil, küresel olarak yüklenmeye değer şeylerdir, eğer bir bağımlılık olarak eklemeyi tercih ederseniz, o zaman işe yarayacaktır, ben yapmam. İşte çalıştırılacak komut:

npm install --global webpack webpack-dev-server

Adım 3: Projenizin kök dizinine bir .babelrc dosyası ekleyin

Bu, babel’i yeni yüklediğiniz ön ayarları kullanacak şekilde ayarlayacaktır. .babelrc dosyanız şöyle görünmelidir:

{
  "presets": ["react", "es2015"]
}

Adım 4: Proje dizini yapısını kurun

Kendinize dizininizin kök dizininde aşağıdakine benzeyen bir dizin yapısı kurun:

|- node_modules
|- src/
  |- components/
  |- images/
  |- styles/
  |- index.html
  |- index.jsx
|- .babelrc
|- package.json

NOT: ’node_modules’, ‘.babelrc’ ve ‘package.json’, önceki adımlarda zaten orada olmalıydı, nereye uyduklarını görebilmeniz için onları ekledim.

Adım 5: Projeyi Hello World proje dosyalarıyla doldurun

Bu, bir işlem hattı oluşturma süreci için gerçekten önemli değil, bu yüzden size bunların kodunu vereceğim ve bunları kopyalayıp yapıştırabilirsiniz:

src/components/HelloWorldComponent.jsx

import React, { Component } from 'react';

class HelloWorldComponent extends Component {
  constructor(props) {
    super(props);
    this.state = {name: 'Student'};
    this.handleChange = this.handleChange.bind(this);
  }

  handleChange(e) {
    this.setState({name: e.target.value});
  }

  render() {
    return (
      <div>
        <div className="image-container">
          <img src="./images/myImage.gif" />
        </div>
        <div className="form">
          <input type="text" onChange={this.handleChange} />
          <div>
            My name is {this.state.name} and I'm a clever cloggs because I built a React build pipeline
          </div>
        </div>
      </div>
    );
  }
}

export default HelloWorldComponent;

src/images/myImage.gif

Bunu istediğiniz herhangi bir resimle değiştirmekten çekinmeyin, sadece resimleri bir araya getirebileceğimizi kanıtlamak için orada. Kendi resminizi sağlar ve ona farklı bir ad verirseniz, değişikliklerinizi yansıtmak için HelloWorldComponent.jsx dosyasını güncellemeniz gerekir. Aynı şekilde, farklı bir dosya uzantısına sahip bir resim seçerseniz, webpack.config.js’deki resim yükleyicinin “test” özelliğini yeni dosya uzantınıza uyacak şekilde uygun normal ifadeyle değiştirmeniz gerekir.

src/styles/styles.css

.form {
  margin: 25px;
  padding: 25px;
  border: 1px solid #ddd;
  background-color: #eaeaea;
  border-radius: 10px;
}

.form div {
  padding-top: 25px;
}

.image-container {
  display: flex;
  justify-content: center;
}

index.html

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Learning to build a react pipeline</title>
</head>
<body>
  <div id="content"></div>
  <script src="app.js"></script>
</body>
</html>

index.jsx

import React from 'react';
import { render } from 'react-dom';
import HelloWorldComponent from './components/HelloWorldComponent.jsx';

require('./images/myImage.gif');
require('./styles/styles.css');
require('./index.html');

render(<HelloWorldComponent />, document.getElementById('content'));

6. Adım: Web paketi yapılandırması oluşturun

Projenizin kök dizininde webpack.config.js adlı bir dosya oluşturun ve bu kodu içine kopyalayın:

webpack.config.js

var path = require('path');

var config = {
  context: path.resolve(__dirname + '/src'),
  entry: './index.jsx',
  output: {
    filename: 'app.js',
    path: path.resolve(__dirname + '/dist'),
  },
  devServer: {
    contentBase: path.join(__dirname + '/dist'),
    port: 3000,
    open: true,
  },
  module: {
    loaders: [
      {
        test: /\.(js|jsx)$/,
        exclude: /node_modules/,
        loader: 'babel-loader'
      },
      {
        test: /\.css$/,
        loader: "style!css"
      },
      {
        test: /\.gif$/,
        loaders: [
          'file?name=[path][name].[ext]',
          'image-webpack',
        ]
      },
      { test: /\.(html)$/,
        loader: "file?name=[path][name].[ext]"
      }
    ],
  },
};

module.exports = config;

Adım 7: İşlem hattınız için npm görevleri oluşturun

Bunu yapmak için projenizin kökündeki package.json dosyasında tanımlanan JSON’un komut dosyası anahtarına iki özellik eklemeniz gerekir. Komut dosyalarınızın anahtarının şöyle görünmesini sağlayın:

  "scripts": {
    "start": "webpack-dev-server",
    "build": "webpack",
    "test": "echo \"Error: no test specified\" && exit 1"
  },

’test’ betiği zaten orada olacak ve onu tutup tutmamayı seçebilirsiniz, bu örnek için önemli değil.

Adım 8: Ardışık düzeni kullanın

Komut satırından, proje kök dizinindeyseniz, şimdi şu komutu çalıştırabilmelisiniz:

npm run build

Bu, oluşturduğunuz küçük uygulamayı bir araya getirecek ve proje klasörünüzün kök dizininde oluşturacağı dist/ dizinine yerleştirecektir.

Komutu çalıştırırsanız:

npm start

Ardından, oluşturduğunuz uygulama, bir web paketi geliştirme sunucusu örneğinin içindeki varsayılan web tarayıcınızda sunulacaktır.
