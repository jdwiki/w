Giriiş #

“props”, verileri ve yöntemleri bir ana bileşenden bir alt bileşene aktarmak için kullanılır.

‘sahneler’ hakkında ilginç şeyler

  1. Değişmezler.
  2. Yeniden kullanılabilir bileşenler oluşturmamıza izin veriyorlar.

Temel örnek

class Parent extends React.Component{
  doSomething(){
     console.log("Parent component");
  }
  render() {
    return <div>
         <Child 
           text="This is the child number 1"
           title="Title 1" 
           onClick={this.doSomething} />
         <Child 
           text="This is the child number 2"
           title="Title 2" 
           onClick={this.doSomething} />
      </div>
  }
}

class Child extends React.Component{
  render() {
    return <div>
       <h1>{this.props.title}</h1>
       <h2>{this.props.text}</h2>
      </div>
  }
}

Örnekte görebileceğiniz gibi, ‘props’ sayesinde yeniden kullanılabilir bileşenler oluşturabiliyoruz.

Varsayılan aksesuarlar #

“defaultProps”, “props” bileşeniniz için varsayılan veya geri dönüş değerlerini ayarlamanıza olanak tanır. ‘defaultProps’, bileşenleri farklı görünümlerden sabit aksesuarlarla çağırdığınızda kullanışlıdır, ancak bazı görünümlerde farklı değer iletmeniz gerekir.

Sözdizimi

ES5

var MyClass = React.createClass({
  getDefaultProps: function() {
    return {
      randomObject: {},
      ...
    };
  }
}
son sürüm

ES6

class MyClass extends React.Component {...}
  
MyClass.defaultProps = {
    randomObject: {},
    ...
}
son sürüm

ES7

class MyClass extends React.Component {  
    static defaultProps = {
        randomObject: {},
        ...
    };
}
son sürüm

“getDefaultProps()” veya “defaultProps” sonucunun sonucu önbelleğe alınır ve ana bileşen tarafından belirtilmemişse “this.props.randomObject” öğesinin bir değere sahip olmasını sağlamak için kullanılır.

PropTypes #

‘propTypes’, bileşeninizin ihtiyaç duyduğu ‘sahnelerin’ ve olması gereken türü belirtmenize olanak tanır. Bileşeniniz ‘propTypes’ ayarlamadan çalışacaktır, ancak bileşeninizi daha okunabilir hale getireceğinden, bileşeninizi okuyan diğer geliştiriciler için belge görevi göreceğinden ve geliştirme sırasında, onun için belirlediğiniz tanımdan farklı bir tipte bir prop ayarlamaya çalışın.

Bazı ilkel “propTypes” ve yaygın olarak kullanılabilen “propTypes” şunlardır:

  optionalArray: React.PropTypes.array,
  optionalBool: React.PropTypes.bool,
  optionalFunc: React.PropTypes.func,
  optionalNumber: React.PropTypes.number,
  optionalObject: React.PropTypes.object,
  optionalString: React.PropTypes.string,
  optionalSymbol: React.PropTypes.symbol

Herhangi bir “propType” öğesine “isRequired” eklerseniz, o bileşenin örneğini oluştururken o pervane sağlanmalıdır. gerekli ‘propTypes’ değerini sağlamazsanız, bileşen örneği oluşturulamaz.

Sözdizimi

ES5

var MyClass = React.createClass({
  propTypes: {
    randomObject: React.PropTypes.object,
    callback: React.PropTypes.func.isRequired,
    ...
  }
}
son sürüm

ES6

class MyClass extends React.Component {...}
  
MyClass.propTypes = {
    randomObject: React.PropTypes.object,
    callback: React.PropTypes.func.isRequired,
    ...
};
son sürüm

ES7

class MyClass extends React.Component {  
     static propTypes = {
        randomObject: React.PropTypes.object,
        callback: React.PropTypes.func.isRequired,
        ...
    };
}
son sürüm

Daha karmaşık donanım doğrulaması

Aynı şekilde, “PropTypes” daha karmaşık doğrulama belirlemenize olanak tanır.

Bir nesneyi doğrulama

...
    randomObject: React.PropTypes.shape({
        id: React.PropTypes.number.isRequired,
        text: React.PropTypes.string,
    }).isRequired,
...

Nesne dizisinde doğrulama

...
    arrayOfObjects: React.PropTypes.arrayOf(React.PropTypes.shape({
        id: React.PropTypes.number.isRequired,
        text: React.PropTypes.string,
    })).isRequired,
...
son sürüm

Yayılma operatörünü kullanarak sahne öğelerini aktarma #

Onun yerine

var component = <Component foo={this.props.x} bar={this.props.y} />;

Her özelliğin tek bir prop değeri olarak iletilmesi gerektiğinde, tüm değerlerinizi iletmek için ES6’daki diziler için desteklenen spread operatörünü ... kullanabilirsiniz. Bileşen şimdi böyle görünecek.

var component = <Component {...props} />;

İlettiğiniz nesnenin özelliklerinin bileşenin props’larına kopyalandığını unutmayın.

Sıra önemlidir. Daha sonraki özellikler öncekileri geçersiz kılar.

 var props = { foo: 'default' };
 var component = <Component {...props} foo={'override'} />;
 console.log(component.props.foo); // 'override'

Başka bir durum da, props’ın yalnızca parçalarını alt bileşenlere iletmek için spread operatörünü de kullanabilmenizdir, ardından props’tan yıkım sözdizimini tekrar kullanabilirsiniz.

Çocuk bileşenlerin çok sayıda sahneye ihtiyaç duyduğu ancak bunları birer birer geçmek istemediğinde çok kullanışlıdır.

 const { foo, bar, other } = this.props // { foo: 'foo', bar: 'bar', other: 'other' };
 var component = <Component {...{foo, bar}} />;
 const { foo, bar } = component.props
 console.log(foo, bar); // 'foo bar'

Props.children ve bileşen bileşimi #

Bir bileşenin “alt” bileşenleri, özel bir destek olan ‘props.children’ üzerinde mevcuttur. Bu destek, bileşenleri birlikte “Birleştirmek” için çok kullanışlıdır ve JSX işaretlemesini daha sezgisel veya DOM’nin amaçlanan nihai yapısını yansıtıcı hale getirebilir:

var SomeComponent = function () {
    return (
        <article className="textBox">
            <header>{this.props.heading}</header>
            <div className="paragraphs">
                {this.props.children}
            </div>
        </article>
    );
}

Bu, bileşeni daha sonra kullanırken isteğe bağlı sayıda alt öğe eklememize izin verir:

var ParentComponent = function () {
    return (
        <SomeComponent heading="Amazing Article Box" >
            <p className="first"> Lots of content </p>
            <p> Or not </p>
        </SomeComponent>
    );
}

Props.children ayrıca bileşen tarafından manipüle edilebilir. props.children bir dizi olabilir veya olmayabilir olduğundan, React onlar için React.Children olarak yardımcı işlevler sağlar. Önceki örnekte, her paragrafı kendi “

” öğesine sarmak isteyip istemediğimizi düşünün:

var SomeComponent = function () {
    return (
        <article className="textBox">
            <header>{this.props.heading}</header>
            <div className="paragraphs">
                {React.Children.map(this.props.children, function (child) {
                    return (
                        <section className={child.props.className}>
                            React.cloneElement(child)
                        </section>
                    );
                })}
            </div>
        </article>
    );
}

Alt <p> etiketinden propları kaldırmak için React.cloneElement kullanımına dikkat edin - props değişmez olduğundan, bu değerler doğrudan değiştirilemez. Bunun yerine, bu aksesuarlara sahip olmayan bir klon kullanılmalıdır.

Ek olarak, döngülere öğe eklerken, React’in yeniden oluşturma sırasında çocukları nasıl uzlaştırdığını unutmayın ve bir döngüye eklenen alt öğelere genel olarak benzersiz bir “anahtar” desteği eklemeyi kesinlikle düşünün.

Çocuk bileşenlerinin türünü algılama #

Bazen, bunlar arasında yineleme yaparken alt bileşenin türünü bilmek gerçekten yararlıdır. Alt bileşenleri yinelemek için React Children.map util işlevini kullanabilirsiniz:

React.Children.map(this.props.children, (child) => {
  if (child.type === MyComponentType) {
    ...
  }
});

Alt nesne, belirli bir bileşenle karşılaştırabileceğiniz “type” özelliğini ortaya çıkarır.
