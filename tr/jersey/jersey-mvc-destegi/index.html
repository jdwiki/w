  • Jersey MVC Merhaba Dünya

Spring MVC gibi MVC Çerçeveleri, dinamik web sayfalarına hizmet eden web uygulamaları oluşturmak için kullanılıyor. Jersey, REST Framework olarak bilinmesine rağmen, MVC modülünü kullanarak dinamik web sayfaları oluşturma desteğine de sahiptir.

Jersey MVC Merhaba Dünya #

Başlamak için yeni bir Maven web uygulaması oluşturun (bunun nasıl yapılacağı bu örneğin kapsamı dışındadır). pom.xml dosyanıza aşağıdaki iki bağımlılığı ekleyin

<dependency>
    <groupId>org.glassfish.jersey.containers</groupId>
    <artifactId>jersey-container-servlet</artifactId>
    <version>2.25.1</version>
</dependency>
<dependency>
    <groupId>org.glassfish.jersey.ext</groupId>
    <artifactId>jersey-mvc-jsp</artifactId>
    <version>2.25.1</version>
</dependency>

Ayrıca pom’a, geliştirme sırasında uygulamayı çalıştıracağımız ‘jetty-maven-plugin’i ekleyin.

<build>
    <finalName>jersey-mvc-hello-world</finalName>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>2.5.1</version>
            <inherited>true</inherited>
            <configuration>
                <source>1.8</source>
                <target>1.8</target>
            </configuration>
        </plugin>

        <plugin>
            <groupId>org.eclipse.jetty</groupId>
            <artifactId>jetty-maven-plugin</artifactId>
            <version>9.3.8.v20160314</version>
        </plugin>
    </plugins>
</build>

Artık controllerlarımızı oluşturabiliriz. Herhangi bir MVC çerçevesinde, kavramlar genellikle aynıdır. Bir şablonunuz var ve şablonu oluşturmak için kullanılacak bir modeli doldurmak için bir denetleyici kullanıyorsunuz. Buradaki “render” terimi, şablon ve modeli birleştirerek son HTML sayfasını oluşturmak için kullanılır. Örneğin bu şablonu alın

src/main/webapp/WEB-INF/jsp/index.jsp

<html>
    <head>
        <title>JSP Page</title>
    </head>
    <body>
        <h1>${it.hello} ${it.world}</h1>
    </body>
</html>

Bu bir JSP dosyasıdır. JSP, Jersey tarafından desteklenen şablon motorlarından yalnızca biridir. Burada “merhaba” ve “dünya” olmak üzere iki model değişkeni kullanıyoruz. Bu iki değişkenin bu şablonu oluşturmak için kullanılan modelde olması bekleniyor. Yani denetleyiciyi ekleyelim

package com.example.controller;

import org.glassfish.jersey.server.mvc.Viewable;

import javax.ws.rs.GET;
import javax.ws.rs.Path;
import javax.ws.rs.Produces;
import javax.ws.rs.core.MediaType;
import java.util.HashMap;
import java.util.Map;

@Path("/")
public class HomeController {

    @GET
    @Produces(MediaType.TEXT_HTML)
    public Viewable index() {
        Map<String, String> model = new HashMap<>();
        model.put("hello", "Hello");
        model.put("world", "World");
        return new Viewable("/index", model);
    }
}

Burada modeli ‘merhaba’ ve ‘dünya’ özellikleriyle doldurduğumuzu görebilirsiniz. Ayrıca controller yöntemi, bu durumda ‘index’ olarak kullanılacak görünüm şablonunun adını döndürür. Bununla çerçeve, “indeks” şablonunu almayı ve onu oluşturmak için sağlanan modeli kullanmayı bilir.

Şimdi sadece onu yapılandırmamız gerekiyor. Aşağıdakilerle bir ‘ResourceConfig’ alt sınıfı ekleyin

package com.example;

import org.glassfish.jersey.server.ResourceConfig;
import org.glassfish.jersey.server.mvc.jsp.JspMvcFeature;

public class AppConfig extends ResourceConfig {

    public AppConfig() {
        packages("com.example.controller");
        property(JspMvcFeature.TEMPLATE_BASE_PATH, "/WEB-INF/jsp");
        register(JspMvcFeature.class);
    }
}

Burada olan üç şey var:

  1. Jersey’e ‘com.example.controller’ paketini ‘@Path’ ile açıklamalı sınıflar için taramasını söylemek için ‘paketleri’ kullanıyoruz. Bu durumda HomeControllerımızı kaydeder.

  2. Çerçevenin şablonları çözmesi için temel yolu ayarlıyoruz. Bu durumda Jersey’e şablonlar için ‘WEB-INF/jsp’ye bakmasını söylüyoruz. Bu direktörde yukarıdaki index.jsp örneğini görebilirsiniz. Ayrıca denetleyicide sadece ‘index’ şablon adını döndürürüz. Bu, yapılandırma temel yolunun önüne eklenerek ve örtük bir ‘.jsp’ sonuna ekleyerek şablonu bulmak için kullanılacaktır.

  3. JSP işlemeyi işleyen özelliği kaydetmemiz gerekiyor. Daha önce de belirtildiği gibi JSP, Jersey tarafından desteklenen tek işleme motoru değildir. Kutunun dışında desteklenen birkaç tane daha var.

Yapmamız gereken son şey, web.xml’de Jersey’i yapılandırmak

<filter>
    <filter-name>Jersey</filter-name>
    <filter-class>org.glassfish.jersey.servlet.ServletContainer</filter-class>
    <init-param>
        <param-name>javax.ws.rs.Application</param-name>
        <param-value>com.example.AppConfig</param-value>
    </init-param>
</filter>

<filter-mapping>
    <url-pattern>/*</url-pattern>
    <filter-name>Jersey</filter-name>
</filter-mapping>

Burada sadece Jersey’i “AppConfig” sınıfımızı kullanacak şekilde yapılandırıyoruz. Burada belirtilmesi gereken çok önemli bir nokta, normalde göreceğiniz şey olan ‘’ yerine ‘’ kullanılmasıdır. Bu, şablon motoru olarak JSP kullanıldığında gereklidir.

Şimdi çalıştırabiliriz. Komut satırından mvn jetty:run komutunu çalıştırın. Bu, daha önce yapılandırdığımız Maven Jetty eklentisini çalıştıracaktır. “Jetty Server Başlatıldı” yazısını gördüğünüzde sunucu hazır demektir. http://localhost:8080/ tarayıcı URL’sine gidin. Voila, “Merhaba Dünya”. Zevk almak.
