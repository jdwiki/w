Bir Linux sunucusunda AEM’nin Hizmet olarak nasıl kurulacağına ilişkin adım adım kılavuz.

CentOS 7’de AEM 6.x kurulumu #

Önkoşullar #

  1. AEM, sunucunuza yüklendi. Kurulumun yolunu kopyalayın (örneğin: /mnt/crx)
  2. AEM’yi (ör. java -jar cq-quickstart-author-p4502.jar) bir kez başlatın. Bu, özellikle komut dosyalarının gerektirdiği /mnt/crx/crx-quickstart/bin olmak üzere tüm gerekli klasörleri oluşturacaktır.
  3. Hizmete erişimi olacak bir kullanıcı oluşturun. (bkz: am)

Adım adım rehber #

  1. Kök erişimine ihtiyacınız olacak
  2. Bu 2 dosyayı oluşturun

  • aem

          #!/bin/bash
      #
      # /etc/rc.d/init.d/aem6
      #
      #
      # # of the file to the end of the tags section must begin with a #
      # character. After the tags section, there should be a blank line.
      # This keeps normal comments in the rest of the file from being
      # mistaken for tags, should they happen to fit the pattern.>
      #
      # chkconfig: 35 85 15
      # description: This service manages the Adobe Experience Manager java process.
      # processname: aem6
      # pidfile: /crx-quickstart/conf/cq.pid

      # Source function library.
      . /etc/rc.d/init.d/functions

      SCRIPT_NAME=`basename $0`
      AEM_ROOT=/opt/aem6
      AEM_USER=aem

      ########
      BIN=${AEM_ROOT}/crx-quickstart/bin
      START=${BIN}/start
      STOP=${BIN}/stop
      STATUS="${BIN}/status"

      case "$1" in
      start)
      echo -n "Starting AEM services: "
      su - ${AEM_USER} ${START}
      touch /var/lock/subsys/$SCRIPT_NAME
      ;;
      stop)
      echo -n "Shutting down AEM services: "
      su - ${AEM_USER} ${STOP}
      rm -f /var/lock/subsys/$SCRIPT_NAME
      ;;
      status)
      su - ${AEM_USER} ${STATUS}
      ;;
      restart)
      su - ${AEM_USER} ${STOP}
      su - ${AEM_USER} ${START}
      ;;
      reload)
      ;;
      *)
      echo "Usage: $SCRIPT_NAME {start|stop|status|reload}"
      exit 1
      ;;
      esac

  • aem.servis

      [Unit]
  Description=Adobe Experience Manager

  [Service]
  Type=simple
  ExecStart=/usr/bin/aem start
  ExecStop=/usr/bin/aem stop
  ExecReload=/usr/bin/aem restart
  RemainAfterExit=yes

  [Install]
  WantedBy=multi-user.target
  1. aem komut dosyasını açın ve aşağıdakileri güncelleyin
  • AEM_ROOT (ör. ‘/mnt/crx’ köktür, burada ‘/mnt/crx/crx-quickstart’ tam yoldur)
  • AEM_USER (ör. “aem”)
  1. Bu dosyaları sunucuya SCP
  • ‘aem’i ‘/usr/bin/aem’ dizinine kopyalayın
    • Example: From terminal on your desktop $ scp <filename> user@1.1.1.1:/usr/bin/aem
  • ‘aem.service’ dosyasını ‘/etc/system.d/system/aem.system’ dizinine kopyalayın
    • Example: From terminal on your desktop $ scp <filename> user@1.1.1.1:/etc/system.d/system/aem.system
  1. Sunucunuza SSH
  • ssh kullanıcısı@1.1.1.1
  1. Dosyalara izin verin
  • sudo chmod u+rwx /usr/bin/aem
  • sudo chmod u+rwx /etc/system.d/system/aem.system
  1. Güncelleme
  • cd /etc/system.d/system
  • systemctl aem.system'i etkinleştirir
  1. AEM’yi başlatmak için sunucuyu yeniden başlatabilir veya aşağıdaki komutları çalıştırabilirsiniz. Bu komutu çalıştırmadan önce Ön koşul Adım 2‘yi çalıştırdığınızdan emin olun.

AEM’yi BAŞLATMA, YENİDEN BAŞLATMA ve DURDURMA komutları #

  1. AEM’yi başlatın - sudo service aem start
  2. AEM’yi yeniden başlatın - sudo service aem restart
  3. AEM’yi Durdur - sudo service aem stop

Notlar #

  1. Yukarıdaki örnek CentOS 7 üzerinde test edilmiştir.
  2. AEM 6.3 sürümü kullanıldı. Yukarıdaki işlemin AEM 6.x için çalışması gerekmesine rağmen
