REST API uç noktalarını korumak için Bahar Güvenliği #

Aşağıdaki girişleri pom.xml içine ekleyin.

<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-web</artifactId>
    <version>3.1.0.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-config</artifactId>
    <version>3.1.0.RELEASE</version>
</dependency>

3.1’den büyük Spring sürümü için önemlidir:

‘org.springframework.security.filterChains’ için fasulye oluşturma hatası, 3.1’den daha yüksek Spring sürümünü kullandığınızda ve ‘spring-aop’, ‘spring-jdbc’, ‘spring-tx’ ve ‘spring’ için manuel olarak bağımlılık eklemediğinizde ortaya çıkıyor. “pom.xml” dosyanızdaki -ifadeler.

Spring bağlamında aşağıdaki girişleri ekleyin. İki REST uç noktasını (merhaba dünya ve hoşçakal) korumak istiyoruz. XSD versiyonunu Spring versiyonuna göre ayarlayın.

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
   xmlns:security="http://www.springframework.org/schema/security"
   xmlns:context="http://www.springframework.org/schema/context"
   xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans-3.1.xsd
                       http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context-3.1.xsd
                       http://www.springframework.org/schema/security
                       http://www.springframework.org/schema/security/spring-security-3.1.xsd">
                               
    <security:http auto-config='true' create-session="never">
        <security:intercept-url pattern="/helloworld/**" access="ROLE_USER" />
        <security:intercept-url pattern="/goodbye/**" access="ROLE_ADMIN" />
        <security:intercept-url pattern="/**" access="IS_AUTHENTICATED_ANONYMOUSLY" />
        <security:http-basic />
    </security:http>

    <security:authentication-manager>
        <security:authentication-provider>
            <security:user-service>
                <security:user name="username1" password="password1"
                    authorities="ROLE_USER" />
                <security:user name="username2" password="password2"
                    authorities="ROLE_ADMIN" />
            </security:user-service>
        </security:authentication-provider>
    </security:authentication-manager>
</beans>

Aşağıdaki girişleri “web.xml” içine ekleyin.

<!-- Spring security-->
<filter>
    <filter-name>springSecurityFilterChain</filter-name>
    <filter-class>org.springframework.web.filter.DelegatingFilterProxy</filter-class>
</filter>
<filter-mapping>
    <filter-name>springSecurityFilterChain</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>

<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>

<context-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>classpath:security-context.xml</param-value>
</context-param>

Spring-boot ve JDBC Authentication kullanan Spring-Security #

Yetkisiz kullanıcıların sayfaya erişmesini engellemek istediğinizi varsayalım, o zaman erişim yetkisi vererek onlara engel koymalısınız. Bunu, tüm HTTP uç noktalarını güvence altına alarak temel kimlik doğrulaması sağlayan yay güvenliğini kullanarak yapabiliriz. Bunun için projenize bahar güvenliği bağımlılığı eklemeniz gerekiyor, maven’de bağımlılığı şu şekilde ekleyebiliriz:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>

İşte yalnızca kimliği doğrulanmış kullanıcıların erişebilmesini sağlayan bir güvenlik yapılandırması.

@Configuration
@Order(SecurityProperties.ACCESS_OVERRIDE_ORDER)
public class SecurityConfig extends WebSecurityConfigurerAdapter {

@Autowired
DataSource datasource;

@Override
protected void configure(HttpSecurity http) throws Exception {
    http
        .authorizeRequests()
            .anyRequest()
            .fullyAuthenticated()
            .and()
        .formLogin()
            .loginPage("/login")
            .failureUrl("/login?error")
            .permitAll()
            .and()
        .logout()
            .logoutUrl("/logout")
            .logoutSuccessUrl("/login?logout")
            .permitAll()
            .and()
        .csrf();
}

@Override
protected void configure(AuthenticationManagerBuilder auth) throws Exception {
    auth.jdbcAuthentication().dataSource(datasource).passwordEncoder(passwordEncoder());
}

@Bean
public PasswordEncoder passwordEncoder() {
    PasswordEncoder encoder = new BCryptPasswordEncoder();
    return encoder;
}
}

Yapılandırma |Açıklama —————–|——————————– ————————– @Configuration |Sınıfın Spring IoC konteyneri tarafından fasulye tanımları kaynağı olarak kullanılabileceğini gösterir. @Order (SecurityProperties.ACCESS_OVERRIDE_ORDER)|Otomatik olarak yapılandırılmış diğer özellikleri değiştirmeden erişim kurallarını geçersiz kılın. Daha düşük değerler daha yüksek önceliğe sahiptir. WebSecurityConfigurerAdapter|SecurityConfig sınıfı, güvenlik yapılandırmasının bazı özelliklerini ayarlamak için birkaç yöntemini genişletir ve geçersiz kılar. @Autowired of DataSource|Fiziksel veri kaynağına bağlantılar için fabrika sağlayın. configure(HttpSecurity) |Geçersiz kılınan yöntem, hangi URL yollarının güvenli hale getirilip hangilerinin korunmayacağını tanımlar. .authorizeRequests().anyRequest() .fulAuthenticated()|Uygulamamıza yönelik tüm isteklerin kimliğinin doğrulanması gerektiğini belirtir. .formLogin()|Forma dayalı bir oturum açmayı yapılandırır .loginPage("/login").failureUrl("/login?error").permitAll()|Oturum açma sayfasının konumunu belirtir ve tüm kullanıcıların sayfaya erişmesine izin verilmelidir. .logout().logoutUrl("/logout") .logoutSuccessUrl("/login?logout").permitAll()|Oturum kapatıldıktan sonra yönlendirilecek URL. Varsayılan, /login?logout’tur. .csrf()|Siteler Arası İstek Sahteciliğini önlemek için kullanılır, CSRF koruması etkindir (varsayılan). configure(AuthenticationManagerBuilder){}|Kullanıcıların kimliğinin nasıl doğrulanacağını tanımlamak için geçersiz kılınan yöntem. .jdbcAuthentication().dataSource(datasource)|JDBC kimlik doğrulamasını kullandığımızı bahara gösterir .passwordEncoder(passwordEncoder())|Parolalarımızı kodlamak için bir parola kodlayıcı kullandığımızı bahara gösterir. (Parola Encoder seçimini döndürmek için bir fasulye oluşturulur, bu durumda BCrypt kullanıyoruz) Kullanılacak herhangi bir tablo adı veya herhangi bir sorgu yapılandırmadığımıza dikkat edin, bunun nedeni yay güvenliğinin varsayılan olarak aşağıdaki tabloları aramasıdır:

create table users (   
  username varchar(50) not null primary key,
  password varchar(255) not null,
  enabled boolean not null) ;

create table authorities (
  username varchar(50) not null,
  authority varchar(50) not null,
  foreign key (username) references users (username),
  unique index authorities_idx_1 (username, authority));

Aşağıdaki satırları yukarıdaki tablolara ekleyin:

INSERT INTO authorities(username,authority) 
VALUES ('user', 'ROLE_ADMIN');

INSERT INTO users(username,password,enabled)
VALUES('user', '$2a$10$JvqOtJaDys0yoXPX9w47YOqu9wZr/PkN1dJqjG9HHAzMyu9EV1R4m', '1');

Bizim durumumuzda kullanıcı adı ‘user’ ve parola da BCrypt algoritması ile şifrelenmiş ‘kullanıcı’dır.

Son olarak, yaylı önyüklemenin kullanması için application.properties içinde bir veri kaynağı yapılandırın:

spring.datasource.url = jdbc:mysql://localhost:3306/spring
spring.datasource.username = root
spring.datasource.password = Welcome123

Not: Bir oturum açma denetleyicisi oluşturun ve yapılandırın ve bunu /login yoluna eşleyin ve oturum açma sayfanızı bu denetleyiciye yönlendirin

Merhaba Bahar Güvenliği #

Not 1: Java sunucu uygulaması hakkında biraz ön bilgiye ihtiyacınız var page(JSP) ve Apache Maven buna başlamadan önce örnekler.

Web sunucusunu (Apache tomcat gibi) mevcut web projesiyle başlatın veya bir tane oluşturun.

index.jsp sayfasını ziyaret edin.

Bu sayfaya herkes erişebilir, güvenli değil!

Güvenli uygulama #

  1. Maven bağımlılıklarını güncelleyin

pom.xml dosyanıza bağımlılıklar ekleme

pom.xml

<dependency>
  <groupId>org.springframework.security</groupId>
  <artifactId>spring-security-web</artifactId>
  <version>4.0.1.RELEASE</version>
</dependency>
<dependency>
  <groupId>org.springframework.security</groupId>
  <artifactId>spring-security-config</artifactId>
  <version>4.0.1.RELEASE</version>
</dependency>

Not 1: Projenizde daha önce “Bahar” kullanmıyorsanız, “bahar bağlamı” ile ilgili bağımlılık. Bu örnek, “bahar bağlamı” ile xml yapılandırmasını kullanacaktır. Öyleyse bu bağımlılığı da ekleyin.

<bağımlılık> org.springframework yay bağlamı <sürüm>4.2.2.RELEASE </bağımlılık>

Not 2: Projenizde daha önce JSTL kullanmıyorsanız, bu konuda bağımlılık. Bu örnek, jsp sayfasında JSTL kullanacaktır. Öyleyse bu bağımlılığı da ekleyin.

<bağımlılık> org.glassfish.web javax.servlet.jsp.jstl <sürüm>1.2.1</sürüm> </bağımlılık>

  1. Spring Security Konfigürasyon Dosyası Yapın

“WEB-INF” klasörü içinde klasör adını “spring” yapın ve security.xml dosyası yapın. Sonraki kodlardan kopyalayıp yapıştırın.

WEB-INF/spring/security.xml

<b:beans xmlns="http://www.springframework.org/schema/security"
         xmlns:b="http://www.springframework.org/schema/beans"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
                             http://www.springframework.org/schema/security http://www.springframework.org/schema/security/spring-security.xsd">

  <http />

  <user-service>
    <user name="stackoverflow" password="pwd" authorities="ROLE_USER" />
  </user-service>

</b:beans>
  1. web.xml’i güncelleyin

“WEB-INF” klasörü içindeki web.xml dosyanızı güncelleyin

WEB-INF/web.xml

<filter>
  <filter-name>springSecurityFilterChain</filter-name>
  <filter-class>org.springframework.web.filter.DelegatingFilterProxy</filter-class>
</filter>
<filter-mapping>
  <filter-name>springSecurityFilterChain</filter-name>
  <url-pattern>/*</url-pattern>
</filter-mapping>

Not: Daha önce projenizde “Bahar” kullanmıyorsanız, Spring bağlamlarının yüklenmesiyle ilgili yapılandırmalar. Bu parametreyi ekleyin ve dinleyici de.

<bağlam-param> contextConfigLocation <param-değeri> /WEB-INF/bahar/*.xml </param-değeri> </bağlam-param>

org.springframework.web.context.ContextLoaderListener

Güvenli web uygulamasını çalıştırma #

Web sunucunuzu çalıştırdıktan ve index.jsp adresini ziyaret ettikten sonra, bahar güvenliği tarafından oluşturulan varsayılan giriş sayfasını göreceksiniz. Çünkü kimliğiniz doğrulanmadı.

giriş yapabilirsiniz

username : stackoverflow
password : pwd

Not: WEB-INF/spring/security.xml üzerinde kullanıcı adı ve şifre ayarı

Kullanıcı adı gösteriliyor #

“Merhaba” dan sonra kullanıcı adını yazdıran jstl etiketi ekleme

index.jsp

<h1>Hello <c:out value="${pageContext.request.remoteUser}" />!!</h1>

Çıkış yapmak #

index.jsp

Form ekleme, “Merhaba kullanıcı adı"ndan sonra giriş etiketleri, gönderen, bahar güvenliğinden çıkış url /logout oluşturur.

<h1>Hello <c:out value="${pageContext.request.remoteUser}" />!!</h1>

<form action="/logout" method="post">
  <input type="submit" value="Log out" />
  <input type="hidden" name="${_csrf.parameterName}" value="${_csrf.token}" />
</form>

Başarıyla çıkış yaptığınızda, otomatik olarak oluşturulan giriş sayfasını tekrar görürsünüz. Çünkü şimdi kimliğiniz doğrulanmadı.
