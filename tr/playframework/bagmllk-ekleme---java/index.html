Guice ile bağımlılık ekleme - 2.4, 2.5 oyna #

Guice, Play’in varsayılan bağımlılık ekleme (ayrıca DI) çerçevesidir. Diğer çerçeveler de kullanılabilir, ancak Guice’i kullanmak geliştirme çabalarını kolaylaştırır, çünkü Play perdenin altındaki şeylerle ilgilenir.

Play API’lerinin Enjeksiyonu #

Play 2.5’ten başlayarak, önceki sürümlerde statik olan birkaç API DI ile oluşturulmalıdır. Bunlar örneğin Yapılandırma, JPAApi, CacheApi vb.‘dir.

Play API’lerinin enjeksiyon yöntemi, Play tarafından otomatik olarak enjekte edilen bir sınıf için ve özel bir sınıf için farklıdır. Otomatik olarak enjekte edilen bir sınıfta enjeksiyon, alana veya kurucuya uygun @Inject notunu koymak kadar basittir. Örneğin, özellik enjeksiyonlu bir denetleyiciye Yapılandırma enjekte etmek için:

@Inject 
private Configuration configuration;

veya yapıcı enjeksiyonu ile:

private Configuration configuration;
@Inject
public MyController(Configuration configuration) {
    this.configuration = configuration;
}

DI için kayıtlı bir custom sınıfındaki enjeksiyon, tıpkı otomatik olarak enjekte edilen sınıf için yapıldığı gibi yapılmalıdır - @Inject notu ile.

DI için bağlı olmayan bir custom sınıfından enjeksiyon, Play.current().injector() ile bir enjektöre açık çağrı ile yapılmalıdır. Örneğin, özel bir sınıfa Yapılandırma enjekte etmek için şöyle bir yapılandırma verisi üyesi tanımlayın:

private Configuration configuration = Play.current().injector().instanceOf(Configuration.class);

Özel enjeksiyon bağlama #

Özel enjeksiyon bağlama, @ImplementedBy notu ile veya Guice modülü ile programlı bir şekilde yapılabilir.

@ImplementedBy açıklamalı enjeksiyon #

@ImplementedBy açıklamalı enjeksiyon en basit yoldur. Aşağıdaki örnek, önbellek için bir cephe sağlayan bir hizmeti göstermektedir.

  1. Hizmet, CacheProvider arabirimi tarafından aşağıdaki gibi tanımlanır:

    @ImplementedBy(RunTimeCacheProvider.class)
public interface CacheProvider {  
   CacheApi getCache();  
}

  2. Hizmet, RunTimeCacheProvider sınıfı tarafından uygulanır:

    public class RunTimeCacheProvider implements CacheProvider {  
   @Inject  
   private CacheApi appCache;  
   @Override  
   public public CacheApi getCache() {  
     return appCache;  
   }  
}

Not: appCache veri üyesi, bir RunTimeCacheProvider örneği oluşturulduktan sonra enjekte edilir. 3. Önbellek denetçisi @Inject ek açıklamalı bir denetleyicinin üyesi olarak tanımlanır ve denetleyiciden çağrılır:

public class HomeController extends Controller {
  @Inject
  private CacheProvider cacheProvider;
  ...      
  public Result getCacheData() {
        Object cacheData = cacheProvider.getCache().get("DEMO-KEY");
        return ok(String.format("Cache content:%s", cacheData));  
  }

@ImplementedBy ek açıklamasıyla yapılan enjeksiyon, sabit bağlamayı oluşturur: Yukarıdaki örnekte CacheProvider her zaman RunTimeCacheProvider ile başlatılır. Bu tür bir yöntem, yalnızca tek bir uygulamaya sahip bir arabirim olduğunda bir durum için uygundur. Birkaç uygulamaya sahip bir arayüze veya soyut arayüz olmadan tekil olarak uygulanan bir sınıfa yardımcı olamaz. Dürüst olmak gerekirse, @ImplementedBy, nadir durumlarda hepsi için kullanılacaktır. Guice modülü ile programlı bağlama kullanılması daha olasıdır.

Varsayılan Play modülü ile enjeksiyon bağlama #

Varsayılan Play modülü, kök proje dizininde şu şekilde tanımlanan Module adlı bir sınıftır:

import com.google.inject.AbstractModule;  
public class Module extends AbstractModule {  
  @Override  
  protected void configure() {  
      // bindings are here           
  }  
}

Not: Yukarıdaki snippet, yapılandırma içindeki bağlamayı gösterir, ancak elbette diğer tüm bağlama yöntemlerine saygı duyulacaktır.

CacheProvider‘ın RunTimeCacheProvider‘a programlı bağlanması için:

  1. @ImplementedBy ek açıklamasını CacheProvider: tanımından kaldırın

    public interface CacheProvider {
    CacheApi getCache();
    }

  2. Modülü yapılandır‘ı aşağıdaki gibi uygulayın:

    public class Module extends AbstractModule {
    @Override
    protected void configure() {
    bind(CacheProvider.class).to(RunTimeCacheProvider.class); }
    }

Varsayılan Play modülü ile esnek enjeksiyon bağlama

RunTimeCacheProvider sahte uygulamalı JUnit testlerinde iyi çalışmıyor (birim testleri konusuna bakın). Bu nedenle, birim testleri için farklı CacheProvider uygulaması talep edilir. Enjeksiyon bağlama ortama göre yapılmalıdır.

Bir örnek görelim.

  1. FakeCache sınıfı, testler yürütülürken kullanılmak üzere CacheApi’nin saplama uygulamasını sağlar (uygulaması o kadar ilginç değildir - sadece bir haritadır).

  2. FakeCacheProvider sınıfı, testler yürütülürken kullanılacak CacheProvider‘ı uygular:

    public class FakeCacheProvider implements CacheProvider {
    private final CacheApi fakeCache = new FakeCache(); @Override
    public CacheApi getCache() {
    return fakeCache; }
    }

  3. Modül aşağıdaki gibi uygulanır:

    public class Module extends AbstractModule { private final Environment environment; public Module(Environment environment, Configuration configuration) { this.environment = environment; } @Override
    protected void configure() {
    if (environment.isTest() ) { bind(CacheProvider.class).to(FakeCacheProvider.class); } else { bind(CacheProvider.class).to(RuntimeCacheProvider.class);
    } }
    }
    Örnek sadece eğitim amaçlıdır. Modül içindeki testler için bağlama, uygulama ve testler arasında çift olduğundan, en iyi uygulama değildir. Testler için bağlama, testlerin kendisi tarafından yapılmalı ve modül, teste özel uygulama hakkında bilgi sahibi olmamalıdır. Bunu daha iyi nasıl yapacağınızı görün … .

Özel bir modül ile enjeksiyon bağlama #

Özel bir modül, varsayılan Oynatma modülüne çok benzer. Aradaki fark, herhangi bir ada sahip olabilmesi ve herhangi bir pakete ait olmasıdır. Örneğin, OnStartupModule modülü paket modüllerine aittir.

package modules;  
import com.google.inject.AbstractModule;  
public class OnStartupModule extends AbstractModule {  
   @Override  
   protected void configure() {  
       ...            
   }  
}

Play tarafından çağrı için özel bir modül açıkça etkinleştirilmelidir. OnStartupModule modülü için application.conf: dosyasına aşağıdakiler eklenmelidir:

play.modules.enabled += "modules.OnStartupModule"
