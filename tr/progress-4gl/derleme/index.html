Derleme İlerleme kodunu “r-kodu” olarak adlandırır ve normalde .r uzantılı bir dosyaya kaydedilir. Derlemenin birkaç farklı yolu vardır: ‘COMPILE’ ifadesini kullanmak veya Linux veya AppBuilder’da: yerleşik Uygulama Derleyicisi. Developer Studio (Eclipse ortamı), derleme işleminde yerleşik olarak derlemeye sahiptir.

Veritabanını güncelleyen 4GL programlarını derlemek için 4GL Development veya OpenEdge Studio’nun kurulu olması gerekir.

Sözdizimi #

  • DERLEME programı.p KAYDET. //program.p’yi derleyin ve r-kodunu kaydedin
  • DERLEME DEĞER(var) KAYDET. //“var” değişkeninde kayıtlı adı derleyin ve r-kodunu kaydedin
  • DERLEME prog.p XREF prog.xref LİSTE prog.list. //prog.p’yi derleyin ve xref ve liste dosyaları oluşturun. R kodunu kaydetmeyin.
  • DERLEME programı.p HATA YOK KAYDET. //program.p’yi derleyin, r-kodunu kaydedin ve yürütmeyi durdurmak için hataları bastırın.

Uygulama Derleyici #

Windows Uygulama Oluşturucu

Windows Appbuilder’da Uygulama Derleyicisi Araçlar Menüsü’nde bulunur.

Prosedür Düzenleyicisi (Linux - “pro” veya Windows “pro.exe”

Prosedür Düzenleyicisinde (hem Linux hem de Windows), Araçlar menüsünde bulunursa Derleyici.

Uygulama Derleyici

İşletim sisteminden bağımsız olarak, derleyicinin işlevselliği aynıdır. Dizinler ve/veya dosyalar ekleyebilir ve bunları derleyebilirsiniz.

Ana ayarlar (daha aşağıda):

  • Yeni .r Dosyasını kaydedin. İşaretlenmezse, dosyalar derlenir ancak kaydedilmez. Örneğin hata takibi için kullanışlıdır.
  • Alt Dizinlere bakın. Aksi takdirde alt dizinlerin eklenmesi gerekecektir.
  • Eski .r Dosyalarını kaldırın. Eski .r dosyasının üzerine yazın.
  • .r Dosyası Yok ise Onlu Derleme. Yalnızca derlenmemiş dosyaları derler.

Seçenekler:

  • Propath - size propath’ı gösterir ve buradan derlenecek dizinleri seçmenize izin verir.
  • Ekle - bir dizin veya dosya girmenizi sağlar.
  • Değiştir - mevcut bir girişi değiştirmenizi sağlar.
  • Sil - Bir girişi siler.
  • Derlemeyi Başlat - Derleyiciyi başlatır. Kısayol: F2

Ana menü seçenekleri:

  • Dosya -> Çıkış: Derleyiciden çıkar
  • Derleme -> Derlemeyi Başlat : Derleyiciyi başlatır. Kısayol: F2
  • Araçlar -> Diğer araçlara erişim
  • Seçenek -> Derleyici… : Ayarlar, aşağıya bakın.
  • Yardım -> OpenEdge Yardımı (Yalnızca Windows). Çevrimiçi yardım. Kısayol: F1

Ayarlar

  • Varsayılan Dosya Özelliği: Derlenecek dosya adı uzantıları
  • Mesaj Günlük Dosyası: Mesajları, uyarıları ve hataları kaydetmek için dosya
  • Şuraya kaydedin: .r dosyasının saklanacağı yer. Kodla aynı dizini boş bırakırsanız.
  • Diller: çeviriler için. Burada kapsanmamıştır.
  • V6Frame: Eski ve kullanışsız…
  • Steam-IO: Derleyici çıktısını yazdırmak istiyorsanız. Büyük olasılıkla değil.
  • Listeleme Dosyası: Derleyicinin bir liste dosyası oluşturmasını istiyorsanız. Hata ayıklama için kullanışlıdır
  • Ekle: mevcut liste dosyasına ekleyin. Aksi takdirde üzerine yazın.
  • Sayfa Genişliği + Uzunluk: Listeleme dosyasının formatı.
  • Xref Dosyası: Derleyicinin bir “XREF” oluşturmasını istiyorsanız. Hata ayıklama, dizin kullanımını kontrol etme vb. için kullanışlıdır.
  • XML ​​Formatı: Eğer derleyici xref bir xml olmalıdır. Aksi takdirde “düz” metin.
  • Ekle: mevcut xref dosyasına ekleyin. Aksi takdirde üzerine yazın.
  • Hata Ayıklama Dosyası: Hata ayıklayıcı listeleme dosyası.
  • Şifreleme Anahtarı: Kaynak dosya xcode kullanılarak şifrelenmişse anahtarı buraya girin.
  • R-kodu Boyutunu Küçült: r-kodunu küçük tutmak için bazı hata ayıklama bilgilerini kaldırın.
  • MD-5 oluşturun: Çoğunlukla WebClient derlemesi için.

Temel kullanım

  1. Derleyiciyi başlatın
  2. Bir yol ekleyin (son oturumdan daha önce kaydedilmemişse)
  3. Derlemek için F2 tuşuna basın.
  4. Hataları gözlemleyin.
  5. Çıkış

DERLEME ifadesi #

Derleme ifadesi, Progress ABL’deki programları derlemenizi sağlar:

Temel kullanım:

COMPILE hello-world.p SAVE.

Bir değişken ile:

DEFINE VARIABLE prog AS CHARACTER   NO-UNDO.

prog = "hello.p".

COMPILE VALUE(prog) SAVE.

‘DERLEME’ ifadesi için birkaç seçenek vardır:

‘KAYDET’, .r kodunun ileride kullanılmak üzere kaydedilmesi gerektiğini belirtir.

COMPILE hello-world.p SAVE.

SAVE INTO dir OR SAVE INTO VALUE(dir-variable) r-kodunu belirtilen dizine kaydeder:

COMPILE hello-world.p SAVE INTO /usr/sources.

LİSTE dosyası. Bloklar, içermeler vb. ile ilgili hata ayıklama bilgilerini içeren bir liste dosyası oluşturur.

COMPILE program.p SAVE LISTING c:\temp\listing.txt.

Listeleme, dosya, sayfa boyutu ve sayfa genişliği eklemek için birkaç seçeneğe sahiptir:

APPEND PAGE-SIZE num PAGE-WIDTH num

XREF xreffile, dize ve indeks kullanımı vb. hakkında bilgi içeren bir derleyici xref dosyası kaydedecektir. Bunu ayrıca `EKLE’ yapabilirsiniz.

COMPILE checkFile.p SAVE XREF c:\directory\xref-file.txt.

‘XREF-XML xreffile-or-dir’, ‘XREF’ ile aynı şeyi yapacak, ancak dosyayı bunun yerine xml formatında kaydedecektir. Bir dizin kullanıyorsanız, xref dosyası programname.xref.xml olarak adlandırılacaktır.

COMPILE file.p SAVE XREF c:\temp\.

‘HATA YOK’, programınızın durdurulmasından kaynaklanan tüm hataları bastıracaktır.

COMPILE program SAVE NO-ERROR.

“DEBUG-LIST file” satır numaralarına sahip bir hata ayıklama dosyası oluşturur.

COMPILE checkFile.p SAVE DEBUG-LIST c:\temp\debug.txt.

‘ÖN PROSES dosyası’ önce tüm önişlemcileri çevirecek ve ardından derlemeden önce kodla yeni bir .p dosyası oluşturacaktır.

 COMPILE checkFile.p SAVE PREPROCESS c:\temp\PREPROC.txt.

“XCODE anahtarı”, anahtar olarak “anahtar” ile şifrelenmiş bir kaynak kodu derler. XCODE’u XREF, XREF-XML, STRING-XREF veya LISTING seçenekleriyle birlikte kullanamazsınız.

COMPILE program.p SAVE XCODE myKey.

Birkaç seçeneği birleştirebilirsiniz:

COMPILE prog.p SAVE INTO /usr/r-code XREF /usr/xrefs/xref.txt APPEND LISTING /usr/listings.txt APPEND NO-ERROR.

DERLEME sistem tanıtıcısı #

‘COMPILER’ sistem tanıtıcısı, yeni bir derlemeyle ilgili bilgilere bakmanıza izin verir.

“ok-program.p"nin herhangi bir hatası veya uyarısı olmayan bir program olduğunu varsayarsak:

COMPILE ok-program.p SAVE NO-ERROR.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

Bu şunları sağlayacaktır:

Uyarı ile program derleme:

/* program-with-warning.p */
DEFINE VARIABLE c AS CHARACTER   NO-UNDO.
DEFINE VARIABLE i AS INTEGER     NO-UNDO.

c = "hello".
DISPLAY c.
//This RETURN makes the program exit here and the code below unreachable.
RETURN.

IF TRUE THEN DO:
    i = 10. 
END.

Programın Derlenmesi:

COMPILE program-with-warning.p SAVE.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

DO iError = 1 TO COMPILER:NUM-MESSAGES:
    DISPLAY 
        COMPILER:GET-FILE-NAME(iError)    LABEL "Filename" FORMAT "x(20)"
        COMPILER:GET-MESSAGE(iError)      LABEL "Message"  FORMAT "x(50)"
        COMPILER:GET-NUMBER(iError)       LABEL "Msg#"  
        COMPILER:GET-ERROR-COLUMN(iError) LABEL "Column"
        COMPILER:GET-ERROR-ROW(iError)    LABEL "Row"
            WITH FRAME fr1 SIDE-LABELS 1 COLUMNS.
END.

Sonuç:

Hatalı bir program derleme

DEFINE VARIABLE c AS CHARACTER   NO-UNDO.
DEFINE VARIABLE i AS INTEGER     NO-UNDO.

c = "hello".
DISPLAY c.
//Casting should be required below...
IF TRUE THEN DO:
    i = c.
END.

Programın Derlenmesi:

//Use no-errors to supress any error messages from interrupting us.
COMPILE c:\temp\program-with-error.p SAVE NO-ERROR.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

DO iError = 1 TO COMPILER:NUM-MESSAGES:
    DISPLAY 
        COMPILER:GET-FILE-NAME(iError)    LABEL "Filename" FORMAT "x(20)"
        COMPILER:GET-MESSAGE(iError)      LABEL "Message"  FORMAT "x(50)"
        COMPILER:GET-NUMBER(iError)       LABEL "Msg#"  
        COMPILER:GET-ERROR-COLUMN(iError) LABEL "Column"
        COMPILER:GET-ERROR-ROW(iError)    LABEL "Row"
            WITH FRAME fr1 SIDE-LABELS 1 COLUMNS 20 DOWN.

    DOWN WITH FRAME fr1.
END.
