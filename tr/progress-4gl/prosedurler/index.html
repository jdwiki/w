Progress ABL’de iki tür prosedür vardır: Windows dll’lerine veya Unix/Linux paylaşımlı kitaplık prosedürlerine cepheler olan dahili prosedürler ve prosedür prototipleri.

Tıpkı fonksiyonlarda olduğu gibi, prosedürler iç içe olamaz. İşlevleri prosedürlere yerleştiremezsiniz ve bunun tersi de geçerlidir.

‘RUN’ ifadesi ile bir prosedür çağrılır.

Sözdizimi #

  • ÇALIŞTIR prosedür adı. //Prosedüradı adlı bir prosedürü çalıştırır.

  • RUN proc1(INPUT “HELLO”). // HELLO dizesini proc1’e girer

  • RUN proc2(GİRİŞ değişken1, çıkış değişken2). //var1’i girdiler ve var2’yi proc2’ye/proc2’den çıktılar

  • RUN proc3(giriş “isim = ‘joe’”, ÇIKIŞ TABLOSU ttResult). // name=joe girer ve bir tablodaki kayıtları çıkarır

  • PROSEDÜR proc: // proc adında bir prosedür bildirir

  • SON PROSEDÜR. // Mevcut prosedürü sonlandırır

Temel bir dahili prosedür #

Fonksiyonların aksine, bir prosedür bildirmek için iletmeye gerek yoktur. RUN kullanarak aramadan önce veya sonra kodunuzun herhangi bir yerine yerleştirilebilir.

RUN proc.

//Procedure starts here
PROCEDURE proc:

//Procedure ends here
END PROCEDURE.

Prosedür adının ardından bunun bir bloğun başlangıcı olduğunu söyleyen iki nokta üst üste işareti gelir. Blok ‘END PROCEDURE’ ile biter (ancak bu basitçe ‘END.’ ile değiştirilebilir).

GİRİŞ ve ÇIKIŞ parametreleri #

Bir prosedür farklı türde parametrelere sahip olabilir: girdi, çıktı, girdi-çıktı (çift yönlü) ve ayrıca geçici tablolar ve veri kümeleri gibi bazı özel türler).

run deyiminde ‘INPUT’ (varsayılan olarak kabul edilir) bildirmek isteğe bağlıdır - diğer tüm yönergeler özel olarak bildirilmelidir.

Girdi olarak iki tamsayı alan ve bir ondalık sayı veren bir prosedür.

PROCEDURE divideAbyB:
    DEFINE INPUT  PARAMETER piA       AS INTEGER     NO-UNDO.
    DEFINE INPUT  PARAMETER piB       AS INTEGER     NO-UNDO.
    DEFINE OUTPUT PARAMETER pdeResult AS DECIMAL     NO-UNDO.

    pdeResult = piA / piB.

END PROCEDURE.

DEFINE VARIABLE de AS DECIMAL     NO-UNDO.

RUN divideAbyB(10, 2, OUTPUT de).

DISPLAY de. //5.00

Parametreler tamamen isteğe bağlıdır. İstediğiniz şekilde karıştırıp eşleştirebilirsiniz. Parametrelerin sırası size bağlıdır, ancak girişle başlayıp çıkışla bitirmek kullanışlıdır - bunları çalıştırma ifadesinde doğru sıraya koymanız gerekir ve karıştırma talimatları can sıkıcı olabilir.

Özyineleme - özyinelemeye bakın #

Özyineleme kolaydır - prosedürün kendisini prosedürün içinden ‘ÇALIŞTIRIN’. Ancak çok fazla özyineleme yaparsanız yığında yer kalmaz.

Bir prosedür hesaplama faktöriyel.

PROCEDURE factorial:
    DEFINE INPUT  PARAMETER piNum AS INTEGER     NO-UNDO.
    DEFINE OUTPUT PARAMETER piFac AS INTEGER     NO-UNDO.

    DEFINE VARIABLE iFac AS INTEGER     NO-UNDO.

    IF piNum = 1 THEN DO:
        pifac = 1.
    END.
    ELSE DO:
        RUN factorial(piNum - 1, OUTPUT iFac).
        piFac = piNum * iFac.
    END.

END PROCEDURE.

DEFINE VARIABLE f AS INTEGER     NO-UNDO.

RUN factorial(7, OUTPUT f).

DISPLAY f.

Dürbün #

Prosedürün kendi kapsamı vardır. Dış kapsam prosedüre “sızar”, ancak bunun tersi olmaz.

DEFINE VARIABLE i AS INTEGER     NO-UNDO INIT 1.
DEFINE VARIABLE j AS INTEGER     NO-UNDO.

PROCEDURE p:

    MESSAGE i VIEW-AS ALERT-BOX. // 1
    MESSAGE j VIEW-AS ALERT-BOX. // 0

    j = 2.

END PROCEDURE.

RUN p.

MESSAGE i VIEW-AS ALERT-BOX. // 1
MESSAGE j VIEW-AS ALERT-BOX. // 2

Bir prosedür içinde, dışarıdaki parametreyle aynı ada sahip bir değişken bildirmek, yalnızca yerel bir değişken oluşturacaktır.

DEFINE VARIABLE i AS INTEGER     NO-UNDO INIT 1.
DEFINE VARIABLE j AS INTEGER     NO-UNDO.

PROCEDURE p:
    
    DEFINE VARIABLE i AS INTEGER     NO-UNDO INIT 5.

    MESSAGE i VIEW-AS ALERT-BOX. // 5
    MESSAGE j VIEW-AS ALERT-BOX. // 0

    j = 2.

END PROCEDURE.

RUN p.

MESSAGE i VIEW-AS ALERT-BOX. // 1
MESSAGE j VIEW-AS ALERT-BOX. // 2

Bir prosedürün içinde oluşturulan herhangi bir değişkene yalnızca o prosedür için erişilebilir.

Bu bir derleyici hatası üretecektir:

PROCEDURE p:
    
    DEFINE VARIABLE i AS INTEGER     NO-UNDO INIT 5.

END PROCEDURE.

RUN p.

MESSAGE i VIEW-AS ALERT-BOX. // Unknown Field or Variable name i - error 201
