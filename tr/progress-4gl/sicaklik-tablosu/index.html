“TEMP-TABLE”, Progress ABL’nin çok güçlü bir özelliğidir. Karmaşık mantık yazmak için kullanılabilecek geçici bir bellek içi (çoğunlukla en azından) tablodur. Prosedürlere, fonksiyonlara ve diğer programlara giriş/çıkış parametreleri olarak kullanılabilir. Bir veya daha fazla geçici tablo, bir “DATASET"in (genellikle ProDataset olarak adlandırılır) temelini oluşturabilir.

Yerel bir İlerleme veritabanı tablosuyla yapılabilecek hemen hemen her şey geçici bir tabloyla yapılabilir.

Basit bir geçici tablo tanımlama #

Bu, üç alanlı ttTempTable adlı bir “TEMP-TABLE” tanımıdır. “GERİ ALMA YOK”, geri alma işleminin gerekmediğini belirtir (bu, genellikle tam tersine ihtiyacınız olmadıkça yapmak istediğiniz şeydir).

DEFINE TEMP-TABLE ttTempTable NO-UNDO
    FIELD field1 AS INTEGER
    FIELD field2 AS CHARACTER
    FIELD field3 AS LOGICAL.

İndeksli bir geçici tablo #

Eğer onlara karşı sorgu çalıştırmayı planlıyorsanız, geçici tablolar indekslerle oluşturulabilir (ve oluşturulmalıdır).

Bu tablo, bir alan (alan1) içeren bir dizine (indeks1) sahiptir. Bu dizin birincil ve benzersizdir (yani, iki kayıt aynı alan1 içeriğine sahip olamaz).

DEFINE TEMP-TABLE ttTempTable NO-UNDO
    FIELD field1 AS INTEGER
    FIELD field2 AS CHARACTER
    FIELD field3 AS LOGICAL
    INDEX index1 IS PRIMARY UNIQUE field1 .

Diğer dizinler - dizinler… #

Her geçici tablo için birden çok dizin tanımlayabilirsiniz. Onlara ihtiyacınız varsa - tanımlayın. Temel olarak, sorgunuzla ve/veya sıralama düzeninizle eşleşen bir dizin, performansa yardımcı olacaktır!

DEFINE TEMP-TABLE ttWithIndex NO-UNDO
    FIELD field1 AS INTEGER
    FIELD field2 AS CHARACTER
    FIELD field3 AS LOGICAL
    INDEX field1 field1.

DEFINE TEMP-TABLE ttWithoutIndex NO-UNDO
    FIELD field1 AS INTEGER
    FIELD field2 AS CHARACTER
    FIELD field3 AS LOGICAL.

DEFINE VARIABLE i              AS INTEGER     NO-UNDO.
DEFINE VARIABLE iWithCreate    AS INTEGER     NO-UNDO.
DEFINE VARIABLE iWithFind      AS INTEGER     NO-UNDO.
DEFINE VARIABLE iWithoutCreate AS INTEGER     NO-UNDO.
DEFINE VARIABLE iWithoutFind   AS INTEGER     NO-UNDO.

ETIME(TRUE).
DO i = 1 TO 1000:
    CREATE ttWithIndex.
    ttWithIndex.field1 = i.
END.
iWithCreate = ETIME.

ETIME(TRUE).
DO i = 1 TO 1000:
    CREATE ttWithoutIndex.
    ttWithoutIndex.field1 = i.
END.
iWithoutCreate = ETIME.

RELEASE ttWithIndex.
RELEASE ttWithoutIndex.

ETIME(TRUE).
DO i = 1 TO 1000:
    FIND FIRST ttWithIndex WHERE ttWithIndex.field1 = i NO-ERROR.
END.
iWithFind = ETIME.

ETIME(TRUE).
DO i = 1 TO 1000:
    FIND FIRST ttWithoutIndex WHERE ttWithoutIndex.field1 = i NO-ERROR.
END.
iWithoutFind = ETIME.

MESSAGE 
    "With index took" iWithFind "ms to find and" iWithCreate "ms to create" SKIP 
    "Without index took" iWithoutFind "ms to find and" iWithoutCreate "ms to create"
    VIEW-AS ALERT-BOX.

Dizin ile arama, dizinsiz arama ile karşılaştırıldığında yaklaşık 70 kat daha hızlıydı! Bu elbette sadece bir çalışmadır, bu nedenle bilimsel bir kanıt değildir, ancak dizin kurulumunuz etki yaratacaktır.

Geçici tabloları girme ve çıktı alma #

Geçici tabloları programların, prosedürlerin ve işlevlerin içine ve dışına geçirmek çok basittir.

Bu, bir dizgede veya benzerinde kolayca saklayabileceğinizden daha fazla sayıda veriyi işlemek için bir prosedür istiyorsanız kullanışlı olabilir. Geçici tabloları INPUT, OUTPUT ve INPUT-OUTPUT verileri olarak iletebilirsiniz.

Bir geçici tablonun girilmesi ve diğerinin çıktısının alınması:

DEFINE TEMP-TABLE ttRequest NO-UNDO
    FIELD fieldA AS CHARACTER
    FIELD fieldB AS CHARACTER.

/* Define a temp-table with the same fields and indices */
DEFINE TEMP-TABLE ttResponse NO-UNDO LIKE ttRequest.

/* A procedure that simply swap the values of fieldA and fieldB */
PROCEDURE swapFields:
    DEFINE INPUT  PARAMETER TABLE FOR ttRequest.
    DEFINE OUTPUT PARAMETER TABLE FOR ttResponse.

    FOR EACH ttRequest:
        CREATE ttResponse.
        ASSIGN 
            ttResponse.fieldA = ttRequest.fieldB
            ttResponse.fieldB = ttRequest.fieldA.
    END.
END PROCEDURE.

CREATE ttRequest.
ASSIGN ttRequest.fieldA = "A"
       ttRequest.fieldB = "B".

CREATE ttRequest.
ASSIGN ttRequest.fieldA = "B"
       ttRequest.fieldB = "C".

CREATE ttRequest.
ASSIGN ttRequest.fieldA = "C"
       ttRequest.fieldB = "D".

/* Call the procedure */
RUN swapFields ( INPUT  TABLE ttRequest
               , OUTPUT TABLE ttResponse).

FOR EACH ttResponse:
    DISPLAY ttResponse.
END.

Sonuç:

fieldA--------fieldB--------

B             A
C             B
D             C

Bir geçici tablonun girdi-çıktısı:

DEFINE TEMP-TABLE ttCalculate NO-UNDO
    FIELD num1     AS INTEGER
    FIELD num2     AS INTEGER
    FIELD response AS DECIMAL.

PROCEDURE pythagoras:
    DEFINE INPUT-OUTPUT PARAMETER TABLE FOR ttCalculate.

    FOR EACH ttCalculate:
        ttCalculate.response = SQRT( EXP(num1, 2) + EXP(num2, 2)).
    END.

END PROCEDURE.

CREATE ttCalculate.
ASSIGN ttCalculate.num1 = 3
       ttCalculate.num2 = 4.

CREATE ttCalculate.
ASSIGN ttCalculate.num1 = 6
       ttCalculate.num2 = 8.

CREATE ttCalculate.
ASSIGN ttCalculate.num1 = 12
       ttCalculate.num2 = 16.

/* Call the procedure */
RUN pythagoras ( INPUT-OUTPUT  TABLE ttCalculate ).

FOR EACH ttCalculate:
    DISPLAY ttCalculate.
END.

Sonuç:

----------num1-- ----------num2-- -------response-

         3                4             5.00
         6                8            10.00
        12               16            20.00

İşlevlere geçiş

DEFINE TEMP-TABLE ttNumbers NO-UNDO
    FIELD num1     AS INTEGER
    FIELD num2     AS INTEGER
    INDEX index1 num1 num2.

DEFINE VARIABLE iNum AS INTEGER     NO-UNDO.

/* Forward declare the function */
FUNCTION hasAPair RETURNS LOGICAL (INPUT TABLE ttNumbers) FORWARD.

DO iNum = 1 TO 100:
    CREATE ttNumbers.
    ASSIGN ttNumbers.num1 = RANDOM(1,100)
           ttNumbers.num2 = RANDOM(1,100).
END.

MESSAGE hasAPair(INPUT TABLE ttNumbers) VIEW-AS ALERT-BOX.


/* Function to check if two records has the same value in num1 and num2 */
FUNCTION hasAPair RETURNS LOGICAL (INPUT TABLE ttNumbers):

    FIND FIRST ttNumbers WHERE ttNumbers.num1 = ttNumbers.num2 NO-ERROR.
    IF AVAILABLE ttNumbers THEN
        RETURN TRUE.
    ELSE 
        RETURN FALSE.

END FUNCTION.

Program dosyalarına geçiş

Geçici tabloları diğer .p programlarına ve programlarından diğer prosedürlere ilettiğiniz şekilde iletirsiniz. Tek fark, hem çağıran hem de çağrılan programın aynı geçici tablo bildirimine sahip olması gerektiğidir. Kolay bir yol, geçici tablo programını üçüncü bir dosyada saklamaktır - her iki programda da kullanılan bir içerme.

Geçici tablo tanımını içeren dosyayı dahil et: /* ttFile.i */ DEFINE TEMP-TABLE ttFile NO-UNDO FIELD fName AS CHARACTER FORMAT “x(20)” FIELD isADirectory AS LOGICAL.

Bir geçici tablodaki tüm dosyaları kontrol eden program. Onlar dizin mi?

/* checkFiles.p */
{ttFile.i}

DEFINE INPUT-OUTPUT PARAMETER TABLE FOR ttFile.

FOR EACH ttFile:
    FILE-INFO:FILE-NAME = ttFile.fName.

    IF FILE-INFO:FILE-TYPE BEGINS "D" THEN
        ttFile.isADirectory = TRUE.
END.

Ana program:

{ttFile.i}

CREATE ttFile.
ASSIGN ttFile.fname = "c:\temp\".

CREATE ttFile.
ASSIGN ttFile.fname = "c:\Windows\".

CREATE ttFile.
ASSIGN ttFile.fname = "c:\Windoose\".

RUN checkFiles.p(INPUT-OUTPUT TABLE ttFile).

FOR EACH ttFile:
    DISPLAY ttFile.
END.

Sonuç:

fName----------------- isADirector

c:\temp\               yes
c:\Windows\            yes
c:\Windoose\           no
