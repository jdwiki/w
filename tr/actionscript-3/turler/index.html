Türleri kontrol etme #

Bir değerin belirli bir türde olup olmadığını doğrulamak için “is” operatörünü kullanabilirsiniz:

var sprite:Sprite = new Sprite();

trace(sprite is Sprite); // true
trace(sprite is DisplayObject); // true, Sprite inherits DisplayObject
trace(sprite is IBitmapDrawable); // true, DisplayObject implements IBitmapDrawable
trace(sprite is Number); // false
trace(sprite is Bitmap); // false, Bitmap inherits DisplayObject
                         // but is not inherited by Sprite.

Ayrıca, uygulanan arabirimleri* ve int/uint türlerini kontrol ederken *‘false’ döndürmesi dışında, “is” ile neredeyse aynı şekilde çalışan bir “instanceof” operatörü (kullanımdan kaldırılmış) vardır.

‘as’ operatörü de tıpkı ‘is’ operatörü gibi kullanılabilir. Bu, özellikle size açık nesne türünün tüm olası özelliklerinin bir listesini verecek olan FlashDevelop gibi bazı akıllı IDE kullanıyorsanız kullanışlıdır. Örnek:

for (var i:int = 0; i < a.length; i++){
    var d:DisplayObject = a[i] as DisplayObject;
    if (!d) continue;
    d.//get hints here
    stage.addChild(d);
}

‘is’ ile aynı etkiyi elde etmek için şunu yazarsınız (biraz daha az uygun):

for (var i:int = 0; i < a.length; i++){
    if (a[i] is DisplayObject != true) continue;
    var d:DisplayObject = a[i] as DisplayObject;
    stage.addChild(d);
}

‘as’ operatörü ile kodlar kontrol edilirken, verilen değerin belirtilen türe ilk kez dönüştürüleceğini ve ardından bu işlemin sonucunun yanlış değilse kontrol edileceğini unutmayın, bu nedenle olası yanlış/NaN değerleriyle kullanırken dikkatli olun:

if(false as Boolean) trace("This will not be executed");
if(false as Boolean != null) trace("But this will be");

Aşağıdaki tablo, tür operatörlerinin sonucuyla birlikte bazı temel değerleri ve türleri gösterir. Yeşil hücreler doğru, kırmızıdan yanlışa, gri ise derleme/çalışma zamanı hatalarına neden olur.

Tip Döküm #

Tip dökümü ‘as’ operatörü ile yapılır:

var chair:Chair = furniture as Chair;

Veya değeri Type() içine sararak:

var chair:Chair = Chair(furniture);

Eğer “as” ile döküm başarısız olursa, bu dökümün sonucu “null” olur. ‘Type()’ içine sarılarak döküm başarısız olursa, bir ‘TypeError’ atılır.

İşlev türü #

İşlevler “İşlev” türündedir:

function example():void { }
trace(example is Function); // true

Bunlara “Function” türündeki diğer değişkenler tarafından başvurulabilir:

var ref:Function = example;
ref(); // ref.call(), ref.apply(), etc.

Ve türü ‘Fonksiyon’ olan parametreler için argümanlar olarak iletilebilirler:

function test(callback:Function):void {
    callback();
}

test(function() {
    trace('It works!');
}); // Output: It works!

Sınıf türü #

Sınıf bildirimlerine yapılan başvurular “Sınıf” olarak yazılır:

var spriteClass:Class = Sprite;

Bu sınıfın örneklerini başlatmak için “Class” türündeki değişkenleri kullanabilirsiniz:

var sprite:Sprite = new spriteClass();

Bu, sağlanan sınıfı oluşturabilecek ve örneğini oluşturabilecek bir işleve “Sınıf” türünde bir argüman iletmek için yararlı olabilir:

function create(type:Class, x:int, y:int):* {
    var thing:* = new type();
    
    thing.x = x;
    thing.y = y;
    
    return thing;
}

var sprite:Sprite = create(Sprite, 100, 100);

Yazılan diziler #

Ne yazık ki ActionScript 3’ün bir jenerik kavramı yoktur, bu nedenle, yazılan bir diziyi ‘Array’ olarak tanımlamanın bir yolu yoktur. Bununla birlikte, benzer bir şekilde çalışan özel bir sınıf Vector.<T> vardır; vektörü başlatırken somut bir sınıfa* referans sağlayın. Bu, “Vector.” türünün (örneğin, genişletin ve yeni işlevler ekleyin) üzerine soyutlamalar oluşturmanın bir yolu olmadığı anlamına gelir ki bu büyük bir dezavantajdır.

Bakmanın daha basit bir yolu, tanımladığınız her sınıfın otomatik olarak Vector.<NameOfYourClass> adında bir tamamlayıcı sınıfı varmış gibi olmasıdır.

Bununla birlikte, geleneksel bir diziye göre ‘Vector.’ türünün hala büyük avantajları vardır:

  • Diziler1‘a karşı Vector.<T> ile çalışırken performans artışı oluyor.
  • Koleksiyona T olmayan değerler eklemeye çalışırsanız derleme türü TypeErrors alırsınız.
  • IDE’ler, bir ‘Vector.’ örneğindeki nesneler için yararlı tür ipucu bilgileri sağlar.

Bir “Vektör.” oluşturma örnekleri:

var strings:Vector.<String> = new Vector.<String>(); // or
var numbers:Vector.<Number> = new <Number>[];

1 Vektörler aslında yalnızca ilkel türlerle (‘String’, ‘int’, ‘uint’, ‘Number’ vb.) çalışırken dizilere göre kayda değer performans iyileştirmeleri sağlar.

Açıklama ekleme türleri #

Derleyiciye bir değerin türünü :Type ile açıklama ekleyerek söyleyebilirsiniz:

var value:int = 10; // A property "value" of type "int".

İşlev parametreleri ve dönüş türleri de açıklanabilir:

// This function accepts two ints and returns an int.
function sum(a:int, b:int):int {
    return a + b;
}

Eşleşmeyen tipte bir değer atamaya çalışmak ‘TypeError’ ile sonuçlanacaktır:

var sprite:Sprite = 10; // 10 is not a Sprite.
