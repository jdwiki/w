Flash’tan istek yapma #

URLRequest ve [URLLoader](http://help.adobe. com/en_US/FlashPlatform/reference/actionscript/3/flash/net/URLLoader.html) sınıfları, Flash’tan harici kaynaklara istek yapmak için birlikte çalışır. “URLRequest”, istekle ilgili bilgileri tanımlar; istek gövdesi ve istek yöntemi türü ve “URLLoader”, asıl isteği gerçekleştirmek ve kaynaktan bir yanıt alındığında bilgilendirilmenin bir yolunu sağlamak için buna başvurur.

Örnek:

var request:URLRequest = new URLRequest('http://stackoverflow.com');
var loader:URLLoader = new URLLoader();

loader.addEventListener(Event.COMPLETE, responseReceived);
loader.load(request);

function responseReceived(event:Event):void {
    trace(event.target.data); // or loader.data if you have reference to it in
                              // this scope.
}

İsteğinize değişken ekleme #

URLVariables sınıfı, bir “URLRequest” ile birlikte gönderilecek verileri tanımlamanıza olanak tanır. .

Örnek:

var variables:URLVariables = new URLVariables();

variables.prop = "hello";
variables.anotherProp = 10;

var request:URLRequest = new URLRequest('http://someservice.com');
request.data = variables;

İsteği bir “URLLoader” aracılığıyla gönderebilir veya navigateToURL kullanarak sorgu dizesine eklenen değişkenlerle istek URL’sini açabilirsiniz. flash/net/package.html#navigateToURL()).

HTTP yöntemini değiştirme (GET, POST, PUT, vb.) #

URLRequestMethod sınıfı, yapabileceğiniz çeşitli istek türleri için sabitler içerir. Bu sabitler, “URLRequest"in method özelliğine tahsis edilecektir:

var request:URLRequest = new URLRequest('http://someservice.com');
request.method = URLRequestMethod.POST;

AIR çalışma zamanı dışında yalnızca “GET” ve “POST” öğelerinin mevcut olduğunu unutmayın.

Yanıt verilerim her zaman boş, “eşzamansız” ne anlama geliyor? #

Flash, harici bir kaynaktan veri isteğinde bulunduğunda, bu işlem zaman uyumsuz olur. Bunun ne anlama geldiğinin en temel açıklaması, verilerin “arka planda” yüklenmesi ve alındığında “Event.COMPLETE” a atadığınız olay işleyicisini tetiklemesidir. Bu, uygulamanızın kullanım ömrü boyunca herhangi bir noktada olabilir.

Verileriniz, ‘URLLoader’ınızdaki ’load()’ çağrıldıktan hemen sonra ÇIKARILMAYACAK. “Event.COMPLETE” için bir olay dinleyicisi **** eklemeli ve oradaki yanıtla etkileşim kurmalısınız.

var request:URLRequest = new URLRequest('http://someservice.com');
var loader:URLLoader = new URLLoader();

loader.addEventListener(Event.COMPLETE, responseReceived);
loader.load(request);

trace(loader.data); // Will be null.

function responseReceived(event:Event):void {
    trace(loader.data); // Will be populated with the server response.
}

trace(loader.data); // Will still be null.

“setTimeout” veya benzerini kullanmak gibi küçük numaralarla bu sorunu çözemezsiniz:

setTimeout(function() {
    trace(loader.data); // Will be null if the data hasn't finished loading
                        // after 1000ms (which you can't guarantee).
}, 1000);

Etki alanları arası istekler #

Flash, uygulamanızın üzerinde çalıştığı etki alanından başka bir etki alanından, bu etki alanının, etki alanının kökünde (ör. Security.loadPolicyFile() ile hedefleyebilirsiniz. crossdomain.xml dosyası, sunucunuzdan bir Flash uygulamasından veri isteyebilecek etki alanlarını belirtebileceğiniz yerdir.

en izin verilen crossdomain.xml örneği:

<?xml version="1.0" ?>
<cross-domain-policy>
  <allow-access-from domain="*"/>
  <allow-http-request-headers-from domain="*" headers="*"/>
</cross-domain-policy>

Bu örneğin üretim ortamlarında kullanılmaması gerektiğine dikkat edin, daha kısıtlayıcı bir örnek kullanın.

Daha kısıtlayıcı bir özel crossdomain.xml örneğin şöyle görünecektir:

<?xml version="1.0"?>
<!DOCTYPE cross-domain-policy SYSTEM "http://www.macromedia.com/xml/dtds/cross-domain-policy.dtd">
<cross-domain-policy>
    <site-control permitted-cross-domain-policies="master-only" />

    <allow-access-from domain="*.domain.com" to-ports="80,843,8011" /> 
    <allow-access-from domain="123.123.123.123" to-ports="80,843,8011" /> 
</cross-domain-policy>
