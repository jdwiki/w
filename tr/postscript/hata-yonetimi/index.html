Sözdizimi #

  • { -code- } durduruldu { -error- }{ -no-error- } ifelse % hata yakalama çerçevesi

  • $error /errorname % stackunderflow typecheck rangecheck vb. olsun $error /newerror % bool olsun. hatayı devre dışı bırakmak için false koy $error /ostack, hata noktasında işlenen yığınının % kopyasını alır

  • errordict /stackoverflow { -additional-code- /stackoverflow signalerror} koymak % hata türleri için ek kod yürütür (burada, /stackoverflow hatası).

Postscript’te hata işlemenin iki düzeyi vardır. Bu ikilik, hem yorumlayıcının hataları ele alma biçimi hem de kullanıcının (programcı) işlemeyi kontrol etmek için kullanabileceği araçlar için geçerlidir.

Alt seviye, alışılmadık bir kontrol yapısıdır dur ... durdu. “durduruldu”, yürütme yığını üzerinde “çıkış” operatörü (bir döngü için) veya “durdur” operatörü (“durduruldu” bağlamı için) ise atlanabilecek bir işaret oluşturması bakımından bir döngü yapısı gibi davranır. denir. Döngü yapısından farklı olarak, “durduruldu”, yığın üzerinde “dur"un çağrılıp çağrılmadığını gösteren bir Boole verir (aksi halde “durduruldu"ya iletilen prosedürün tamamlanana kadar yürütüldüğü bilinir.

Bir PostScript hatası oluştuğunda, örneğin ‘stackunderflow’ gibi, yorumlayıcı hatanın adını ‘systemdict’ içinde bulunan ’errordict’ içinde arar. Kullanıcı ’errordict’ içindeki prosedürü değiştirmediyse, varsayılan hata prosedürü tüm yığının anlık görüntülerini alacak ve bunları ‘systemdict’ içindeki başka bir sözlük olan ‘$error’ içine yerleştirecektir. Son olarak, varsayılan prosedür, kullanıcı programını yürütme yığınından çıkaran ve ’errordict’ içinde ‘handleerror’ olarak adlandırılan yorumlayıcının hata yazdırma prosedürünü yürüten ‘stop’u çağıracaktır.

Tüm bu bilgileri kullanarak, kodun bir bölümünü { ... } durduruldu içine sararak hataları yakalayabilirsiniz. “Dur” diyerek bir hatayı yeniden atabilirsiniz. $error /errorname get ile ne tür bir hata oluştuğunu belirleyebilirsiniz.

Prosedürü “errordict” içinde bu adla değiştirerek belirli bir hata türü için varsayılan davranışı da değiştirebilirsiniz. Veya ’errordict’ içindeki ‘/handleerror’ ifadesini değiştirerek hata raporunun yazdırılma biçimini değiştirin.

Bir hata bildirildiğinde olayların sırası #

Bir hatanın sırası genellikle:

  1. hata, ’errordict’ içinde hata adının aranması ve bu prosedürün yürütülmesiyle tetiklenir.
  2. “errordict” prosedürü “signalerror” adını vererek hata adını verir.
  3. ‘signalerror’, yığınların anlık görüntülerini alır, anlık görüntüleri ‘$hata’ya kaydeder ve ardından ‘dur’u çağırır.
  4. stop, durdurulan operatör tarafından oluşturulan en yakın çevreleyen durdurulan içeriğe kadar yürütme yığınını açar.
  5. Program hatayı yakalamak için kendi durdurulmuş bağlamını oluşturmamışsa, başlangıç ​​kodu tarafından tüm kullanıcı programını parantez içine almak için çağrılan bir dış düzey stopped { errordict /handleerror get exec } if tarafından yakalanacaktır. .
  6. ‘handleerror’, bir hata raporu yazdırmak için ‘$error’ içindeki bilgileri kullanır.

Bir hata sinyali (fırlatma) #

Araçların çoğu, hata atmak için operatörün adı dışında standartlaştırılmıştır. Adobe tercümanlarında buna .error denir. Ghostscript’te buna “signalerror” denir. Bu satırla, Adobe yorumlayıcıları veya Ghostscript veya xpost için postscript kodunda ‘signalerror’ kullanabilirsiniz.

/.error where {pop /signalerror /.error load def} if

komut adı hataadı     sinyal hatası     — "$hata” ve ardından “dur” içindeki yığının anlık görüntülerini alın.

Örneğin.

% my proc only accepts integer
/proc {
    dup type /integertype ne {
        /proc cvx /typecheck signalerror
    } if
    % ... rest of proc ...
} def

Bir akım noktası var mı? #

“currentpoint” başarılı bir şekilde yürütülürse yığında “true” veya bir “/nocurrentpoint” hatası sinyali verirse “false” verir.

{currentpoint pop pop} stopped not  % bool

Hata yakalama #

Varsayılan hata işleyicinin son eylemi “dur"u çağırmak olduğundan, kodu bir “{ … } durduruldu” yapısına ekleyerek operatörlerden gelen hataları yakalayabilirsiniz.

{
    0 array
    1 get
} stopped {
    $error /errorname get =
} if

dizin o dizi için kabul edilebilir aralığın dışında olduğunda “get” tarafından bildirilen hata “rangecheck” yazdıracaktır.

Yeniden atma hataları #

Bu pasaj, bir postscript döngü operatörü gibi davranan bir prosedür uygular. Kullanıcı ‘proc’, ’exit’i çağırırsa, sonunda ’end’ için dikte yığınını düzeltmek için ‘invalidexit’ hatasını yakalar. ‘invalidexit’ dışındaki herhangi bir hata, ‘stop’ çağrılarak yeniden atılır.

% array n proc  .  -
% Like `forall` but delivers length=n subsequences produced by getinterval
/fortuple { 4 dict begin
    0 {offset proc n arr} {exch def} forall
    /arr load length n idiv
    {
        {
            /arr load offset n getinterval
            [ /proc load currentdict end /begin cvx ] cvx exec
            /offset offset n add def
        } stopped {
            $error /errorname get /invalidexit eq
            { 1 dict begin exit }{ stop } ifelse
        } if
    } repeat
end
} def
