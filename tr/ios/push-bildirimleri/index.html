Anında Bildirimler için cihazı kaydetme #

Cihazınızı push bildirimlerine kaydetmek için, didFinishLaunchingWithOptions yönteminde AppDelegate dosyanıza aşağıdaki kodu ekleyin:

Süratli #

func application(application: UIApplication, didFinishLaunchingWithOptions launchOptions: [NSObject: AnyObject]?) -> Bool {
    // Override point for customization after application launch.
    if UIDevice.currentDevice().systemVersion.compare(v, options: .NumericSearch) == NSOrderedAscending { 
        // Register for Push Notitications, if running iOS < 8
        if application.respondsToSelector("registerUserNotificationSettings:") {
            let types:UIUserNotificationType = (.Alert | .Badge | .Sound)
            let settings:UIUserNotificationSettings = UIUserNotificationSettings(forTypes: types, categories: nil)

            application.registerUserNotificationSettings(settings)
            application.registerForRemoteNotifications()
        } else {      
            // Register for Push Notifications before iOS 8
            application.registerForRemoteNotificationTypes(.Alert | .Badge | .Sound)
        }
    } else {
        var center = UNUserNotificationCenter.currentNotificationCenter()
        center.delegate = self
        center.requestAuthorizationWithOptions((UNAuthorizationOptionSound | UNAuthorizationOptionAlert | UNAuthorizationOptionBadge)) {(granted: Bool, error: NSError) -> Void in
            if !error {
                UIApplication.sharedApplication().registerForRemoteNotifications()
                // required to get the app to do anything at all about push notifications
                print("Push registration success.")
            } else {
                print("Push registration FAILED")
                print("ERROR: \(error.localizedFailureReason!) - \(error.localizedDescription)")
                print("SUGGESTIONS: \(error.localizedRecoveryOptions) - \(error.localizedRecoverySuggestion!)")
            })
    }

    return true
}

Push Bildirimleri ile kullanım için Uygulama Kimliğini Kaydetme #

İhtiyacınız olan şeyler #

  • Ücretli bir Apple Geliştirici Programı Üyeliği
  • Daha önce hiçbir yerde kullanılmayan, uygulamanız için geçerli bir Uygulama Kimliği ve tanımlayıcı (com.example.MyApp gibi)
  • developer.apple.com ve Üye Merkezine erişim
  • Test edilecek bir iOS Aygıtı (Push Bildirimleri Simülatörde çalışmadığından)

Apple Developer Center’da Uygulama Kimliği için APN erişimini etkinleştirme #

1- Developer.apple.com Üye Merkezi’ne giriş yapın (ana sayfadaki Hesap bağlantısı)

2- “Sertifikalar” a gidin

3- Sol panelden “Uygulama Kimliği"ni seçin

4- Sağ üstteki “+” işaretine tıklayın

5- Push Bildirimleri seçeneği işaretliyken Uygulama Kimliği ekleyin

6- Oluşturulan Uygulama Kimliğine tıklayın ve Düzenle’yi seçin

7- Anında Bildirimler panelinde Yapılandır’a tıklayın

8- Mac’inizde Keychain Access uygulamasını açın

9- Anahtarlık Erişimi menüsünden Sertifika Yardımcısı -> Bir Sertifika Yetkilisinden Sertifika İste’ye tıklayın

10- İlk metin alanına postanızı girin

11- İkinci metin alanına adınızı girin

12- CA E-posta Adresini boş bırakın

13- CA’ya E-postayla Gönderilmek yerine Diske kaydedildi’yi seçin

14- Devam’a tıklayın ve oluşturulan dosyayı yükleyin

15- Oluşturulan dosyayı Apple tarafından indirin ve Keychain Access açıkken açın

Xcode’da APN erişimini etkinleştirme #

1- Projenizi seçin

2- Yetenekler sekmesini açın

3- Push Bildirimlerini bulun ve açın

4-Arka Plan Modlarını bulun ve açın ve Uzaktan Bildirimleri kontrol edin

Anında iletme bildirimlerini test etme #

Sunucu tarafınızı onlar için hazırlamadan önce bile push bildirimlerinin nasıl çalıştığını test etmek her zaman iyi bir uygulamadır, sadece sizin tarafınızda her şeyin doğru şekilde ayarlandığından emin olmak için. Aşağıdaki PHP betiğini kullanarak kendinize anında iletme bildirimi göndermek oldukça kolaydır.

  1. Komut dosyasını bir dosya olarak (örneğin send_push.php) sertifikanızla (geliştirme veya üretim) aynı klasöre kaydedin.

  2. Cihazınızın jetonunu, sertifikadan şifreyi koymak için düzenleyin

  3. Bağlantı, dev_path veya prod_path açmak için doğru yolu seçin (komut dosyasında ‘APNS sunucusuna bağlantı aç’ burada gerçekleşir)

  4. Terminaldeki klasöre cd atın ve ‘php send_push’ komutunu çalıştırın

  5. Bildirimi cihazınıza alın

    $alert, 'sound' => 'default', 'event' => $event ); // Encode the payload as JSON $payload = json_encode($body); // Build the binary notification $msg = chr(0) . pack('n', 32) . pack('H*', $deviceToken) . pack('n', strlen($payload)) . $payload; // Send it to the server $result = fwrite($fp, $msg, strlen($msg)); if (!$result) echo 'Message not delivered' . PHP_EOL; else echo 'Message successfully delivered' . PHP_EOL; // Close the connection to the server fclose($fp);

Uygulamanızın Push Bildirimi için zaten kayıtlı olup olmadığını kontrol etme #

Süratli #

let isPushEnabled = UIApplication.sharedApplication().isRegisteredForRemoteNotifications()

Sunucu geliştiricisine iletmek için .cer dosyanızdan bir .pem sertifikası oluşturma #

  1. aps.cer’ı bir klasöre kaydedin

  2. “Anahtarlık erişimi"ni açın ve bu sertifikanın altındaki anahtarı bir .p12 dosyasına aktarın (buna key.p12 olarak adlandırın). Bunu yapmak için sağ tıklayın ve Dışa Aktar’ı seçin. Adım 1 ile aynı klasöre kaydedin. Dışa aktarma sırasında bir parola girmeniz istenecektir. Bir şeyler uydur ve ezberle.

  3. Terminaldeki o klasöre cd atın ve aşağıdaki komutları çalıştırın:

  4. .cer’i bir .pem sertifikasına dönüştürün

    openssl x509 -in aps.cer -inform der -out aps.pem

  5. Anahtarınızı .pem biçimine dönüştürün. Anahtarı açmak için 2. adımda anahtar zincirinden dışa aktardığınız parolayı girin. Ardından, dışa aktarılan dosyayı koruyacak başka bir parola girin. Onay için iki kez girmeniz istenecektir.

    openssl pkcs12 -nocerts -out key.pem -in key.p12

  6. Dosyaları son bir dosyada birleştirin

    cat key.pem aps.pem > final_cert.pem

  7. final_cert.pem nihai sonuçtur. Korumalı sertifikayı kullanabilmeleri için 5. adımdaki parolayla sunucu geliştiricilerine iletin.

Anında Bildirimlerden Kaydı Kaldırma #

Uzaktan Bildirimlerden programlı olarak kaydınızı iptal etmek için şunları kullanabilirsiniz:

Amaç-C #

[[UIApplication sharedApplication] unregisterForRemoteNotifications];

Süratli #

UIApplication.sharedApplication().unregisterForRemoteNotifications()

bu, telefonunuzun ayarlarına girip uygulama için Bildirimleri manuel olarak kapatmaya benzer.

NOT: Buna ihtiyaç duyabileceğiniz nadir durumlar olabilir (ör. uygulamanız artık push bildirimlerini desteklemediğinde)

Yalnızca kullanıcının Bildirimleri geçici olarak devre dışı bırakmasına izin vermek istiyorsanız. Sunucunuzdaki veritabanındaki cihaz belirtecini kaldırmak için bir yöntem uygulamalısınız. Aksi takdirde, Bildirimi yalnızca cihazınızda yerel olarak devre dışı bırakırsanız, sunucunuz yine de mesaj gönderir.

Uygulama simgesi rozet numarasını ayarlama #

Başvurunuzun içinden rozet numarasını ayarlamak için aşağıdaki kod parçasını kullanın (varsayalım ‘someNumber’ daha önce bildirilmiş):

Hedef-C

[UIApplication sharedApplication].applicationIconBadgeNumber = someNumber;

Süratli

UIApplication.shared.applicationIconBadgeNumber = bazıNumber

Rozeti tamamen kaldırmak için someNumber = 0 olarak ayarlamanız yeterlidir.

(Etkileşimli Olmayan) Anında Bildirim için Kaydolma #

Geri arama işlevleri (başarılı, başarısız) olarak adlandırılacağından, push bildirimine kaydolma mantığının ‘AppDelegate.swift’ içine eklenmesi önerilir. Kayıt olmak için aşağıdakileri yapmanız yeterlidir:

let application = UIApplication.sharedApplication()
let settings = UIUserNotificationSettings(forTypes: [.Alert, .Badge, .Sound], categories: nil)
application.registerUserNotificationSettings(settings)

Ardından didRegisterUserNotificationSettings geri çağırma işlevi çağrılır ve bu durumda kaydı şu şekilde tetiklersiniz:

func application(application: UIApplication, didRegisterUserNotificationSettings notificationSettings: UIUserNotificationSettings) {
    application.registerForRemoteNotifications()
}

Ve bu durumda, push bildirimi almak için izin isteyen sistem uyarısı gösterilecektir. Aşağıdaki geri arama işlevlerinden biri çağrılacak:

func application(application: UIApplication, didRegisterForRemoteNotificationsWithDeviceToken deviceToken: NSData) {
    let tokenChars = UnsafePointer<CChar>(deviceToken.bytes)
    var tokenString = ""
    
    for i in 0..<deviceToken.length {
        tokenString += String(format: "%02.2hhx", arguments: [tokenChars[i]])
    }
    
    print("Push token: \(tokenString)")
}

func application(application: UIApplication, didFailToRegisterForRemoteNotificationsWithError error: NSError) {
    print("didFailToRegisterForRemoteNotificationsWithError: \(error)")
            
}

Çok nadir durumlarda, ne başarılı ne de başarısız geri arama işlevleri çağrılmaz. Bu, internet bağlantı sorunlarınız olduğunda veya APNS Korumalı Alanı kapalı olduğunda gerçekleşir. Sistem, bir miktar doğrulama yapmak için APNS’ye bir API çağrısı yapar, bunun yapılmaması, iki geri arama işlevinin hiçbirinin çağrılmayacağına yol açacaktır. Sorunsuz olduğundan emin olmak için Apple sistem durumu sayfasını ziyaret edin.

Anında Bildirimi İşleme #

Kullanıcı bir push bildirimine tıkladığında, aşağıdaki geri arama işlevi çağrılır. Derin bağlantı oluşturmanıza yardımcı olacak arka uçtan gönderilen belirli bilgileri almak için JSON’u ayrıştırabilirsiniz:

Hızlı

func application(application: UIApplication, didReceiveRemoteNotification userInfo: [NSObject : AnyObject]) {
    print("Received notification: \(userInfo)")
}

    NSLog(@"Received notification: %@", userInfo);
}
