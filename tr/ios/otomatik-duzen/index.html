Otomatik Düzen, görünüm hiyerarşinizdeki tüm görünümlerin boyutunu ve konumunu, bu görünümlere yerleştirilen kısıtlamalara dayalı olarak dinamik olarak hesaplar. Kaynak

Sözdizimi #

  • NSLayoutConstraint(item: Any, öznitelik: NSLayoutAttribute, RelatedBy: NSLayoutRelation, toItem: Any?, öznitelik: NSLayoutAttribute, çarpan: CGFloat, sabit: CGFloat) // Programlı olarak bir kısıtlama oluşturun

Uzay Görünümü Eşittir #

İki görünümün yan yana, denetimlerinde ortalanmış olmasını istemek yaygındır. Stack Overflow’ta verilen ortak cevap, bu iki görünümü bir ‘UIView’ içine gömmek ve ‘UIView’i ortalamaktır. Bu gerekli veya tavsiye edilmez. UILayoutGuide dokümanlarından:

Görünüm hiyerarşinize sahte görünümler eklemekle ilgili bir takım maliyetler vardır. İlk olarak, görünümün kendisini oluşturmanın ve sürdürmenin maliyeti vardır. İkincisi, yapay görünüm, görünüm hiyerarşisinin tam bir üyesidir, yani hiyerarşinin gerçekleştirdiği her göreve ek yük ekler. Hepsinden kötüsü, görünmez kukla görünüm, diğer görünümlere yönelik mesajları engelleyebilir ve bulması çok zor sorunlara neden olabilir.

Düğmeleri gereksiz bir ‘UIView’a eklemek yerine, bunu yapmak için ‘UILayoutGuide’ı kullanabilirsiniz. Bir “UIlayoutGuide”, esasen Otomatik Düzen ile etkileşime girebilen dikdörtgen bir alandır. Düğmelerin sol ve sağ taraflarına bir ‘UILayoutGuide’ koyup genişliklerini eşit olacak şekilde ayarladınız. Bu, düğmeleri ortalayacaktır. İşte kodda nasıl yapılacağı:

Görsel Biçim Dili stili

view.addSubview(button1)
view.addSubview(button2)

let leftSpace = UILayoutGuide()
view.addLayoutGuide(leftSpace)

let rightSpace = UILayoutGuide()
view.addLayoutGuide(rightSpace)

let views = [
    "leftSpace" : leftSpace,
    "button1" : button1,
    "button2" : button2,
    "rightSpace" : rightSpace
]

// Lay the buttons and layout guides out horizontally in a line. 
// Put the layout guides on each end.
NSLayoutConstraint.activateConstraints(NSLayoutConstraint.constraintsWithVisualFormat("H:|[leftSpace][button1]-[button2][rightSpace]|", options: [], metrics: nil, views: views))

// Now set the layout guides widths equal, so that the space on the 
// left and the right of the buttons will be equal
leftSpace.widthAnchor.constraintEqualToAnchor(rightSpace.widthAnchor).active = true

Çapa Stili

let leadingSpace = UILayoutGuide()
let trailingSpace = UILayoutGuide()
view.addLayoutGuide(leadingSpace)
view.addLayoutGuide(trailingSpace)

leadingSpace.widthAnchor.constraintEqualToAnchor(trailingSpace.widthAnchor).active = true
    
leadingSpace.leadingAnchor.constraintEqualToAnchor(view.leadingAnchor).active = true
leadingSpace.trailingAnchor.constraintEqualToAnchor(button1.leadingAnchor).active = true
    
trailingSpace.leadingAnchor.constraintEqualToAnchor(button2.trailingAnchor).active = true
trailingSpace.trailingAnchor.constraintEqualToAnchor(view.trailingAnchor).active = true

Buna dikey kısıtlamalar da eklemeniz gerekecek, ancak bu, herhangi bir “sahte” görünüm eklemeden görünümdeki düğmeleri ortalayacaktır! Bu, sistemi “sahte” görünümleri göstererek CPU zamanını boşa harcamaktan kurtaracaktır. Bu örnek, düğmeleri kullanır, ancak kısıtlamalar koymak istediğiniz herhangi bir görünüm için düğmeleri değiştirebilirsiniz.

iOS 8 veya önceki bir sürümünü destekliyorsanız, bu düzeni oluşturmanın en kolay yolu gizli yapay görünümler eklemektir. iOS 9 ile sahte görünümleri düzen kılavuzlarıyla değiştirebilirsiniz.

Not: Arayüz Oluşturucu henüz yerleşim kılavuzlarını desteklememektedir (Xcode 7.2.1). Bu nedenle, bunları kullanmak istiyorsanız, kısıtlamalarınızı kodda oluşturmalısınız. Kaynak.

Alternatif olarak, kısıtlamaları programlı olarak ekleyebilirsiniz. Kısıtlamaları oluşturun ve bunları aşağıdaki örnekte açıklandığı gibi istenen UI öğelerine ve görünümlerine ekleyin, burada bir düğme oluşturup onu merkeze, yatay ve dikey olarak denetimine göre hizalıyoruz:

Hedef-C

- (void)viewDidLoad
{
    [super viewDidLoad];
    UIButton *yourButton = [[UIButton alloc] initWithFrame:CGRectMake(0, 0, 100, 18)];
    [yourButton setTitle:@"Button" forState:UIControlStateNormal];


    [self.view addConstraint:[NSLayoutConstraint constraintWithItem:yourButton attribute:NSLayoutAttributeCenterY relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeCenterY multiplier:1 constant:0]]; //Align veritcally center to superView

    [self.view addConstraint:[NSLayoutConstraint constraintWithItem:yourButton attribute:NSLayoutAttributeCenterX relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeCenterX multiplier:1 constant:0]]; //Align horizontally center to superView

    [self.view addSubview:yourButton]; //Add button to superView
}

Süratli

override func viewDidLoad() 
{
    super.viewDidLoad()
    let yourButton: UIButton = UIButton(frame: CGRect(x: 0, y: 0, width: 100, height: 18))
    yourButton.setTitle("Button", forState: .Normal)

    let centerVertically = NSLayoutConstraint(item: yourButton,
                                   attribute: .CenterX,
                                   relatedBy: .Equal,
                                      toItem: view,
                                   attribute: .CenterX,
                                  multiplier: 1.0,
                                    constant: 0.0)
    let centerHorizontally = NSLayoutConstraint(item: yourButton,
                                   attribute: .CenterY,
                                   relatedBy: .Equal,
                                      toItem: view,
                                   attribute: .CenterY,
                                  multiplier: 1.0,
                                    constant: 0.0)
    NSLayoutConstraint.activateConstraints([centerVertically, centerHorizontally])
}

Kısıtlamaları programlı olarak ayarlama #

Kazan plakası kod örneği

override func viewDidLoad() {
    super.viewDidLoad()

    let myView = UIView()
    myView.backgroundColor = UIColor.blueColor()
    myView.translatesAutoresizingMaskIntoConstraints = false
    view.addSubview(myView)

    // Add constraints code here
    // ...
}

Aşağıdaki örneklerde Anchor Style, ‘NSLayoutConstraint’ Style yerine tercih edilen yöntemdir, ancak yalnızca iOS 9’da mevcuttur, bu nedenle iOS 8’i destekliyorsanız yine de ‘NSLayoutConstraint’ Style’ı kullanmalısınız.

Sabitleme #

Çapa Stili

let margins = view.layoutMarginsGuide
myView.leadingAnchor.constraintEqualToAnchor(margins.leadingAnchor, constant: 20).active = true
  • ’leadingAnchor’a ek olarak, ’trailingAnchor’, ’topAnchor’ ve ‘bottomAnchor’ da vardır.

NSLayoutKısıtlama Stili

NSLayoutConstraint(item: myView, attribute: NSLayoutAttribute.Leading, relatedBy: NSLayoutRelation.Equal, toItem: view, attribute: NSLayoutAttribute.LeadingMargin, multiplier: 1.0, constant: 20.0).active = true
  • “.Leading"e ek olarak “.Trailing”, “.Top” ve “.Bottom” da vardır.
  • “.LeadingMargin"e ek olarak “.TrailingMargin”, “.TopMargin” ve “.BottomMargin” vardır.

Görsel Biçim Dili stili

NSLayoutConstraint.constraintsWithVisualFormat("H:|-20-[myViewKey]", options: [], metrics: nil, views: ["myViewKey": myView])

Genişlik ve yükseklik #

Çapa Stili

myView.widthAnchor.constraintEqualToAnchor(nil, constant: 200).active = true
myView.heightAnchor.constraintEqualToAnchor(nil, constant: 100).active = true

NSLayoutKısıtlama Stili

NSLayoutConstraint(item: myView, attribute: NSLayoutAttribute.Width, relatedBy: NSLayoutRelation.Equal, toItem: nil, attribute: NSLayoutAttribute.NotAnAttribute, multiplier: 1, constant: 200).active = true
NSLayoutConstraint(item: myView, attribute: NSLayoutAttribute.Height, relatedBy: NSLayoutRelation.Equal, toItem: nil, attribute: NSLayoutAttribute.NotAnAttribute, multiplier: 1, constant: 100).active = true

Görsel Biçim Dili stili

NSLayoutConstraint.constraintsWithVisualFormat("H:[myViewKey(200)]", options: [], metrics: nil, views: ["myViewKey": myView])
NSLayoutConstraint.constraintsWithVisualFormat("V:[myViewKey(100)]", options: [], metrics: nil, views: ["myViewKey": myView])

Kapta ortala #

Çapa Stili

myView.centerXAnchor.constraintEqualToAnchor(view.centerXAnchor).active = true
myView.centerYAnchor.constraintEqualToAnchor(view.centerYAnchor).active = true

NSLayoutKısıtlama Stili

NSLayoutConstraint(item: myView, attribute: NSLayoutAttribute.CenterX, relatedBy: NSLayoutRelation.Equal, toItem: view, attribute: NSLayoutAttribute.CenterX, multiplier: 1, constant: 0).active = true
NSLayoutConstraint(item: myView, attribute: NSLayoutAttribute.CenterY, relatedBy: NSLayoutRelation.Equal, toItem: view, attribute: NSLayoutAttribute.CenterY, multiplier: 1, constant: 0).active = true

Görsel Biçim Dili stili

NSLayoutConstraint.constraintsWithVisualFormat("V:[viewKey]-(<=0)-[myViewKey]", options: NSLayoutFormatOptions.AlignAllCenterX, metrics: nil, views: ["myViewKey": myView, "viewKey": view])
NSLayoutConstraint.constraintsWithVisualFormat("H:[viewKey]-(<=0)-[myViewKey]", options: NSLayoutFormatOptions.AlignAllCenterY, metrics: nil, views: ["myViewKey": myView, "viewKey": view])

UILabel Temel Boyut #

Bir metne resim ön ekine sahip olacak bir görünüm oluşturmalıyız. metin değişken uzunlukta olabilir. Görüntü + metninde her zaman bir üst görünümün merkezinde olduğu bir sonuca ulaşmak zorundayız.

Adım 1: Önce tek bir görünüm projesi oluşturun ve seçtiğiniz bir şeyi adlandırın ve storyboard’un ilk görünümünü açın. Makul bir boyutta bir görünüm sürükleyin ve arka plan rengini sarıya ayarlayın.Görüntü denetleyicimi 3.5 olarak yeniden boyutlandırdım. ″. Ortaya çıkan görünüm şöyle görünmelidir

Adım 2: Şimdi sarı görünüme kısıtlamalar ekleyeceğiz.Başlamak için genişlik ve yükseklik kısıtlamaları ekleyeceğiz (Bir dakika bekleyin, bu görünümün dinamik genişliğe sahip olacağını söylememiş miydik?Tamam, buna geri döneceğiz Daha sonra) Aşağıdaki kısıtlamaları aşağıdaki resme göre ekleyin genişlik değeriyle uğraşmayın, genişlik için herhangi bir değer yeterli olacaktır, sadece otomatik yerleşimleri düzgün bir şekilde ekleyebilmemiz için onu yeterince büyük tutun.

Bu iki kısıtlamayı ekledikten sonra, XCode’un size aşağıdaki resimdeki gibi hatalar verdiğini göreceksiniz, bunları görelim ve anlayalım.

İki hatamız var (kırmızı, hata anlamına gelir)Yukarıda tartışıldığı gibi belirsizlik kısmını tekrar gözden geçirelim

Eksik Kısıtlamalar : Şunlar için gerekli kısıtlamalar: X konumu :- Yukarıda tartışıldığı gibi, görünüme bir genişlik ve yükseklik verdik, bu nedenle “SINIRI” tanımlandı, ancak kökenini vermedik, bu nedenle “ÇERÇEVE” tanımlanmadı . Otomatik yerleşim, sarı görünümümüzün X konumunun ne olacağını belirleyemiyor

Eksik Kısıtlamalar : Şunun için gerekli kısıtlamalar: Y konumu :- Yukarıda tartışıldığı gibi, görünüme bir genişlik ve yükseklik verdik, bu nedenle “SINIRI” tanımlandı, ancak kökenini vermedik, bu nedenle “ÇERÇEVE” tanımlanmadı . Otomatik yerleşim, sarı görünümümüzün Y konumunun ne olacağını belirleyemiyor Bunu çözmek için autolayout’a X ve Y’yi reole etmek için bir şeyler vermeliyiz. Çerçeveleri ayarlayamadığımız için bunu autolayout yöntemiyle yapacağız. Aşağıda verilen resme göre aşağıdaki kısıtlamaları ekleyin, daha sonra açıklayacağım.

Yaptığımız şey, bir “Dikey Merkez” ve “Yatay Merkez” ekledik, bu kısıtlamalar otomatik yerleşime sarı görünümümüzün her zaman ortada olacağını söylüyor Yatay: yani X’in aynısı dikey kısıtlama ile belirlenir ve Y belirlenir.(sen çerçeveyi ayarlamanız gerekebilir).

Adım 3: Artık temel sarı görünümümüz hazır. Aşağıdaki kısıtlamalarla sarı görünümümüzün alt görünümü olarak önek resmini ekleyeceğiz. İstediğiniz herhangi bir resmi seçebilirsiniz.

Ön ek görüntümüz için sabit bir boyuta sahip olduğumuzdan, bu görüntü görünümü için sabit genişlik yüksekliğine sahip olacağız. Kısıtlamaları ekleyin ve sonraki adıma geçin.

Adım4: Sarı görünümümüzün alt görünümü olarak bir UILabel ekleyin ve aşağıdaki kısıtlamaları ekleyin

Gördüğünüz gibi UILabel’imize sadece göreceli kısıtlamalar verdim.Önek görüntüsünden 8 puan ve sarı görünümden 0,0,0 üst ve alttan geliyor.Genişliğin dinamik olmasını istediğimiz için genişlik veya yükseklik kısıtlaması vermeyeceğiz. .

S: Neden şimdi hata alamıyoruz, genişlik ve yükseklik vermedik mi? Cvp:- Sadece otomatik yerleşim, ekranda bir görünüm oluşturmak için gereken herhangi bir şeyi çözemediğinde hata veya uyarı alıyoruz. Yükseklik genişliği veya orijin olsun. Etiketimiz sarı görünüm ve önek resmi ve çerçevelerine göre olduğundan iyi tanımlanmış otomatik yerleşim, Etiketimizin çerçevesini hesaplayabilir.

Adım 5: Şimdi hatırlayacak olursak, sarı görünüme sabit bir görünüm verdiğimizi ancak bunun etiketimizin metnine bağlı olarak dinamik olmasını istediğimizi fark edeceğiz. Bu yüzden sarı görünümün Genişlik Kısıtlamamızı değiştireceğiz. Belirsizliği gidermek için sarı görünümün genişliği gereklidir, ancak UILabel’in içeriğine bağlı olarak çalışma zamanında geçersiz kılınmasını istiyoruz. Bu yüzden sarı görünümümüzü seçeceğiz ve Size denetçisine gideceğiz ve genişlik kısıtlamasının önceliğini 1’e düşüreceğiz, böylece aşırı yönetilecek. Aşağıda verilen resmi takip edin.

Adım 6: UILabel’in metne göre genişlemesini ve sarı görünümümüzü zorlamasını istiyoruz. Bu nedenle sarı görünüm genişliğinin önceliğini azalttık. Şimdi UILabel’imizin metin sıkıştırma direncinin önceliğini artıracağız. azaltmak için görüntüleyin, böylece UILabel’in içerik kucaklama önceliğini artıracağız.Aşağıdaki resmi takip edin

Gördüğünüz gibi, içerik kucaklama önceliğini 500’e ve sıkıştırma direnci önceliğini 751’e çıkardık, bu da genişlik kısıtlamasının 1 önceliğini başarıyla yönetecek.

Şimdi derleyin ve çalıştırın, aşağıdaki gibi bir şey göreceksiniz.

Görsel Biçim Dilinin Temelleri: Koddaki Kısıtlamalar! #

HVFL, UI öğelerini basit ve hızlı bir şekilde kısıtlamak için tasarlanmış bir dildir. Genel olarak, VFL, çok daha okunabilir, erişilebilir ve kompakt olduğu için Arayüz Oluşturucu’daki geleneksel UI özelleştirmesine göre bir avantaja sahiptir.

Burada, üç UIView’un soldan sağa sınırlandırıldığı ve “superView.width"i “aGradeView” ile doldurduğu bir VFL örneği verilmiştir.

"H:|[bgView][aGradeView(40)][bGradeView(40)]|"

UI Nesnelerini yatay ve dikey olarak sınırlayabileceğimiz iki eksen vardır.

VFL’nin her satırı her zaman “H:” veya “V:” ile başlar. Hiçbiri yoksa, varsayılan seçenek H: şeklindedir.

Devam edelim, bir boru hattımız var. | Bu sembol veya boru, denetime atıfta bulunur. Yukarıdaki VFL kodunun snippet’ine daha yakından bakarsanız, bu boru hatlarından ikisini fark edeceksiniz.

Bu, denetimin iki yatay ucunu, dış sol ve dış sağ sınırları belirtir.

Sırada bazı köşeli parantezler göreceksiniz, ilk köşeli parantez içinde bgView var. Köşeli parantezlerimiz olduğunda, bu bir UI öğesine atıfta bulunur, şimdi ad ile gerçek UI öğesi arasında nasıl bir bağlantı kurduğumuzu merak edebilirsiniz, belki bir çıkış?

Bunu yazının sonunda ele alacağım.

İkinci köşeli parantez [aGradeView(50)] çiftine bakarsanız, içinde bazı parantezler de vardır, bu mevcut olduğunda, bu durumda eksenlere bağlı olarak genişliği/yüksekliği tanımlar. genişliği 50 pikseldir.

İlk köşeli parantez ‘[bgView]’ açıkça tanımlanmış bir genişliğe sahip değildi, yani mümkün olduğu kadar genişleyecektir.

Pekala, temel bilgiler bu kadar, başka bir örnekte gelişmiş şeyler hakkında daha fazla bilgi.

örneğin: 

    // 1. create views
    UIView *blueView = [[UIView alloc] init];
    blueView.backgroundColor = [UIColor blueColor];
    [self.view addSubview:blueView];

    UIView *redView = [[UIView alloc] init];
    redView.backgroundColor = [UIColor redColor];
    [self.view addSubview:redView];

    // 2. forbid Autoresizing
    blueView.translatesAutoresizingMaskIntoConstraints = NO;
    redView.translatesAutoresizingMaskIntoConstraints = NO;

    // 3. make contraints
    // horizontal
    NSArray *blueH = [NSLayoutConstraint constraintsWithVisualFormat:@"H:|-20-[blueView]-20-|" options:NSLayoutFormatAlignAllLeft metrics:nil views:@{@"blueView" : blueView}];
    [self.view addConstraints:blueH];

    // vertical
    NSArray *blueVandRedV = [NSLayoutConstraint constraintsWithVisualFormat:@"V:|-20-[blueView(50)]-20-[redView(==blueView)]" options:NSLayoutFormatAlignAllTrailing metrics:nil views:@{@"blueView" : blueView, @"redView" : redView}];
    [self.view addConstraints:blueVandRedV];

    NSLayoutConstraint *redW = [NSLayoutConstraint constraintWithItem:redView attribute:NSLayoutAttributeWidth relatedBy:NSLayoutRelationEqual toItem:blueView attribute:NSLayoutAttributeWidth multiplier:0.5 constant:0];
    [self.view addConstraint:redW];

Otomatik Mizanpaj ile nasıl animasyon yapılır #

Otomatik Mizanpaj olmadan, bir görünümün karesini zaman içinde değiştirerek animasyon gerçekleştirilir. Otomatik Mizanpaj ile, kısıtlamalar görünüm çerçevesini belirler, dolayısıyla bunun yerine kısıtlamaları canlandırmanız gerekir. Bu yönlendirme, animasyonun görselleştirilmesini zorlaştırır.

Otomatik Mizanpaj ile canlandırmanın yolları şunlardır:

  1. Periyodik çağrılar (CADisplayLink, dispatch_source_t, dispatch_after, NSTimer) kullanarak oluşturulduktan sonra kısıtlamanın sabitini değiştirin. Ardından kısıtlamayı güncellemek için layoutIfNeeded çağırın. Örnek:

Hedef-C:

self.someConstraint.constant = 10.0;
[UIView animateWithDuration:0.25 animations:^{
    [self.view layoutIfNeeded];
}];

Süratli:

self.someConstraint.constant = 10.0
UIView.animate(withDuration: 0.25, animations: self.view.layoutIfNeeded)

  1. Kısıtlamaları değiştirin ve bir animasyon bloğu içinde [view layoutIfNeeded] öğesini çağırın. Bu, animasyon sırasında kısıtlamaları göz ardı ederek iki konum arasında enterpolasyon yapar.

    [UIView animateWithDuration:0.5 animations:^{ [view layoutIfNeeded]; }]

  2. Kısıtlamaların önceliğini değiştirin. Bu, kısıtlamaları eklemek ve kaldırmaktan daha az CPU yoğundur.

  3. Tüm kısıtlamaları kaldırın ve otomatik boyutlandırma maskeleri kullanın. Sonrası için view.translatesAutoresizingMaskIntoConstraints = YES ayarlamanız gerekir.

  4. Amaçlanan animasyonu engellemeyen kısıtlamalar kullanın.

  5. Bir kapsayıcı görünümü kullanın. Denetimi kısıtlamaları kullanarak konumlandırın. Ardından, animasyonla savaşmayan kısıtlamalara sahip bir alt görünüm ekleyin, örneğin: denetime göre bir merkez. Bu, denetimin kısıtlamalarının bir kısmını kaldırır, böylece alt görünümdeki animasyonla savaşmazlar.

  6. ** Görünümler yerine katmanları canlandırın**. Katman dönüşümleri Otomatik Düzen’i tetiklemez.

    CABasicAnimation* ba = [CABasicAnimation animationWithKeyPath:@“transform”]; ba.autoreverses = YES; ba.duration = 0.3; ba.toValue = [NSValue valueWithCATransform3D:CATransform3DMakeScale(1.1, 1.1, 1)]; [v.layer addAnimation:ba forKey:nil];

  7. Düzen Alt Görünümlerini geçersiz kıl. “[super layoutSubviews]” öğesini çağırın ve kısıtlamalara ince ayar yapın.

  8. viewDidLayoutSubviews içindeki çerçeveyi değiştirin. Otomatik Mizanpaj, ’layoutSubviews’ içinde uygulanır, bu nedenle bir kez yapıldığında, ‘viewDidLayoutSubviews’ içinde değiştirin.

  9. Otomatik Düzenden çıkın ve görünümleri manuel olarak ayarlayın. Bu geçersiz kılma “layoutSubviews”/“layout"u, süper sınıfın uygulamasını çağırmadan yapabilirsiniz.

Hızlı ipucu: animasyonlu görünümün ebeveyni enterpolasyon yapmıyorsa (yani, animasyon başlangıçtan bitiş durumuna atlarsa), animasyonlu görünümün ebeveyni olan en derin görünümde (içinde) “layoutIfNeeded()” öğesini çağırın. diğer bir deyişle, animasyondan etkilenmez). Bunun neden işe yaradığını tam olarak bilmiyorum.

UILabel Öncelik Çakışmasını Çözün #

Sorun: Bir görünüm içinde çok sayıda etiket kullandığınızda bir uyarı alabilirsiniz:

Bu uyarıyı nasıl düzeltebiliriz?

Çözüm: Öncelikleri hesaplayıp sıraya koyuyoruz. Öncelikler, etiketlerden farklı olmalıdır. Bu, önemli olanın daha yüksek önceliğe sahip olacağı anlamına gelir. Örneğin, benim durumumda etiketlerim için dikey öncelikleri şöyle belirledim:

  1. etiket için en yüksek önceliği ve 4. etiket için en düşük önceliği ayarladım.

Bir ViewController’da, bu önceliklerin etkisini görmenin zor olduğunu düşünüyorum. Bununla birlikte, UITableViewCell + tahmin hücre yüksekliği ile çok açık bir şekilde.

Orantılı Düzen #

Kısıt şu şekilde oluşturuldu:

NSLayoutConstraint(item: myView, attribute: NSLayoutAttribute.Leading, relatedBy: NSLayoutRelation.Equal, toItem: view, attribute: NSLayoutAttribute.LeadingMargin, multiplier: 1.0, constant: 20.0)

veya matematik açısından:

view.attribute * multiplier + constant          (1)

Farklı boyut faktörü için orantılı düzen oluşturmak için çarpanı kullanabilirsiniz.

Örnek:

Turkuaz Görünüm (V1), 1:1.1 oranına sahip genişlik orantılı denetim genişliğine sahip bir karedir.

Gary square(V2), V1’in bir alt görünümüdür. Alt boşluk, sabit = 60 ile ayarlandı, Son boşluk, çarpan = 1.125 ve sabit = 0 ile ayarlandı

Sondaki boşluk orantılı olarak ayarlandı, alt boşluk sabit olarak ayarlandı.

Not: view.attribute 0’a eşitse (örneğin baştaki boşluk), kısıtlama formülü (1) 0’a eşit olacaktır. View.attribute != öğesini görüntülemek için ikinci sınırlama öğesini değiştirmeniz veya kenar boşluğuna göre sınırlama ayarlamanız gerekir. 0.

NSLayoutConstraint: Koddaki kısıtlamalar! #

Bir çerçeve üzerinde çalışırken, kısıtlamalar çok karmaşık değilse, yeterince küçültmek için Masonry veya SnapKit’i içe aktarmak yerine Interface Builder veya NSLayoutConstraint kodunu kodda kullansak iyi olur.

örneğin:

  • Amaç-C
    // 1. create views
    UIView *blueView = [[UIView alloc] init];
    blueView.backgroundColor = [UIColor blueColor];
    [self.view addSubview:blueView];

    UIView *redView = [[UIView alloc] init];
    redView.backgroundColor = [UIColor redColor];
    [self.view addSubview:redView];

    // 2. forbid Autoresizing
    blueView.translatesAutoresizingMaskIntoConstraints = NO;
    redView.translatesAutoresizingMaskIntoConstraints = NO;

    // 3. make contraints
    // 3.1 blueView
    NSLayoutConstraint *blueLeft = [NSLayoutConstraint constraintWithItem:blueView attribute:NSLayoutAttributeLeft relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeLeft multiplier:1 constant:20];
    [self.view addConstraint:blueLeft];

    NSLayoutConstraint *blueTop = [NSLayoutConstraint constraintWithItem:blueView attribute:NSLayoutAttributeTop relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeTop multiplier:1 constant:20];
    [self.view addConstraint:blueTop];

    NSLayoutConstraint *blueRight = [NSLayoutConstraint constraintWithItem:blueView attribute:NSLayoutAttributeRight relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeRight multiplier:1 constant:-20];
    [self.view addConstraint:blueRight];

    NSLayoutConstraint *blueHeight = [NSLayoutConstraint constraintWithItem:blueView attribute:NSLayoutAttributeHeight relatedBy:NSLayoutRelationEqual toItem:nil attribute:NSLayoutAttributeNotAnAttribute multiplier:1 constant:50];
    [self.view addConstraint:blueHeight];

    // 3.2 redView
    NSLayoutConstraint *redTop = [NSLayoutConstraint constraintWithItem:redView attribute:NSLayoutAttributeTop relatedBy:NSLayoutRelationEqual toItem:blueView attribute:NSLayoutAttributeBottom multiplier:1 constant:20];
    [self.view addConstraint:redTop];

    NSLayoutConstraint *redRight = [NSLayoutConstraint constraintWithItem:redView attribute:NSLayoutAttributeRight relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeRight multiplier:1 constant:-20];
    [self.view addConstraint:redRight];

    NSLayoutConstraint *redHeight = [NSLayoutConstraint constraintWithItem:redView attribute:NSLayoutAttributeHeight relatedBy:NSLayoutRelationEqual toItem:blueView attribute:NSLayoutAttributeHeight multiplier:1 constant:0];
    [self.view addConstraint:redHeight];

    NSLayoutConstraint *redWidth = [NSLayoutConstraint constraintWithItem:redView attribute:NSLayoutAttributeWidth relatedBy:NSLayoutRelationEqual toItem:blueView attribute:NSLayoutAttributeWidth multiplier:0.5 constant:0];
    [self.view addConstraint:redWidth];

Otomatik Düzen nasıl kullanılır #

Otomatik düzen, görünümleri herhangi bir cihazda ve yönlendirmede iyi görünecek şekilde düzenlemek için kullanılır. Kısıtlamalar, her şeyin nasıl ortaya konması gerektiğini söyleyen kurallardır. Diğer şeylerin yanı sıra kenarları sabitleme, merkezleme ve boyutları ayarlamayı içerir.

Otomatik düzen varsayılan olarak etkindir, ancak bunu iki kez kontrol edebilirsiniz. Proje Gezgini’nde Main.storyboard‘a tıklarsanız ve ardından Dosya denetçisini gösterirseniz. Otomatik Düzen ve Boyut Sınıflarının işaretli olduğundan emin olun:

Otomatik düzen kısıtlamaları, Arayüz Oluşturucu’da veya kodda ayarlanabilir. Arayüz Oluşturucu’da, sağ altta Otomatik Düzen araçlarını bulacaksınız. Onlara tıklamak, bir görünümdeki kısıtlamaları ayarlamak için farklı seçenekleri ortaya çıkaracaktır.

Farklı cihaz boyutları veya yönelimleri için farklı kısıtlamalara sahip olmak istiyorsanız, bunları alt ortada bulunan wAny hAny Size Class seçeneklerinde ayarlayabilirsiniz.

Otomatik Düzenin Otomatik Olmayan Düzen ile karışık kullanımı #

Bazen, “UIKit"in kendisi tarafından yapılan Otomatik Düzen hesaplamalarına bazı ek işlemler yapmak isteyebilirsiniz.

Örnek: “maskLayer” içeren bir “UIView"a sahip olduğunuzda, Otomatik Düzen “UIView"un “çerçevesini” değiştirir değiştirmez “maskLayer"ı güncellemeniz gerekebilir

// CustomView.m
- (void)layoutSubviews {
    [super layoutSubviews];
    // now you can assume Auto Layout did its job
    // you can use view's frame in your calculations
    CALayer maskLayer = self.maskLayer;
    maskLayer.bounds = self.bounds;
    ...
}

veya ViewController içindeki Otomatik Düzen için bazı ek işlemler yapmak istiyorsanız

- (void)viewDidLayoutSubviews {
    [super viewDidLayoutSubviews];
    // now you can assume all your subviews are positioned/resized correctly
    self.customView.frame = self.containerView.frame;
}
