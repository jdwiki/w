UILabel sınıfı, salt okunur bir metin görünümü uygular. Bu sınıfı, kullanıcı arabiriminizin diğer bölümlerini tanımlamak için kullanabilecekleriniz gibi bir veya daha fazla statik metin satırı çizmek için kullanabilirsiniz. Temel UILabel sınıfı, etiket metninin hem basit hem de karmaşık stili için destek sağlar. Ayrıca, etiketin gölge kullanıp kullanmadığı veya vurgulu çizip çizmediği gibi görünümün özelliklerini de kontrol edebilirsiniz. Gerekirse, alt sınıflara ayırarak metninizin görünümünü daha da özelleştirebilirsiniz.

Sözdizimi #

  • UILabel.numberOfLines: Int // etiketin sahip olabileceği maksimum satır sayısını al veya ayarla. 0 sınırsız
  • UILabel.text: Dize? // etiketin gösterdiği metni al veya ayarla
  • UILabel.textColor: UIColor! // etiketteki metnin rengini al veya ayarla
  • UILabel.tintColor: UIColor! // etiketin renk tonunu al veya ayarla
  • UILabel.attributedText: NSAttributedString? // etiketin atfedilen metnini al veya ayarla
  • UILabel.font: UIFont! // etiketteki metnin yazı tipini al veya ayarla
  • UILabel.textAlignment: NSTextAlignment // metnin hizalamasını al veya ayarla

UILabel’ler, bir veya daha fazla metin satırını görüntülemek için kullanılabilen görünümlerdir. Gölgeler, metin renkleri ve yazı tipleri gibi metni stilize etmenin birden çok yolunu içerir.

UILabels ayrıca, metnin bölümlerine stiller uygulamak için metin + satır içi işaretleme olan Nitelikli Dizeleri de görüntüleyebilir.

UILabel, UIAppearance protokolüne uymadığından, UILabels’in görünümünü özelleştirmek için UIAppearance proxy yöntemlerini kullanamazsınız. Daha fazlası için bu [tartışmaya2 bakın.

Bir UIL Etiketi oluşturun #

Çerçeveli #

Etiketiniz için tam olarak ayarlamak istediğiniz boyutları bildiğinizde, bir “CGRect” çerçeveli bir “UILabel” başlatabilirsiniz.

Süratli #

let frame = CGRect(x: 0, y: 0, width: 200, height: 21)
let label = UILabel(frame: frame)
view.addSubview(label)

Amaç-C #

CGRect frame = CGRectMake(0, 0, 200, 21);
UILabel *label = [[UILabel alloc] initWithFrame:frame];
[view addSubview:label];

Otomatik Düzen ile #

iOS’un çalışma zamanında çerçevesini dinamik olarak hesaplamasını istediğinizde, bir “UILabel"e kısıtlamalar ekleyebilirsiniz.

Süratli #

let label = UILabel()
label.backgroundColor = .red
label.translatesAutoresizingMaskIntoConstraints = false
view.addSubview(label)

NSLayoutConstraint.activate([
    //stick the top of the label to the top of its superview:
    label.topAnchor.constraint(equalTo: view.topAnchor)

    //stick the left of the label to the left of its superview
    //if the alphabet is left-to-right, or to the right of its 
    //superview if the alphabet is right-to-left:
    label.leadingAnchor.constraint(equalTo: view.leadingAnchor)

    //stick the label's bottom to the bottom of its superview:
    label.bottomAnchor.constraint(equalTo: view.bottomAnchor)

    //the label's width should be equal to 100 points:
    label.widthAnchor.constraint(equalToConstant: 100)
])

Amaç-C #

UILabel *label = [[UILabel alloc] init];

Objective-c + Görsel Biçim Dili (VFL) ile #

  UILabel *label = [UILabel new];
  label.translatesAutoresizingMaskIntoConstraints = NO;
  [self.view addSubview label];
  // add horizontal constraints with 5 left and right padding from the leading and trailing
 
  [self.view addConstraints:[NSLayoutConstraint constraintsWithVisualFormat:@"V:|-5-[labelName]-5-|"
                                                                    options:0
                                                                    metrics:nil
                                                                      views:@{@"labelName":label}]];
  // vertical constraints that will use the height of the superView with no padding on top and bottom
  [self.view addConstraints:[NSLayoutConstraint constraintsWithVisualFormat:@"H:|[labelName]|"
                                                                    options:0
                                                                    metrics:nil
                                                                      views:@{@"labelName":label}]]

Özellikler iletişim kutusunda, “UILabel” adını ayarlayabilir ve “güçlü” veya “zayıf” olarak ayarlayabilirsiniz. “Güçlü” ve “zayıf” hakkında daha fazla bilgi için bkz. bu,

@IBOutlet weak var nameLabel : UILabel!

Amaç-C #

Satır Sayısı #

Bir etiket yaptığınızda ve metnini görüntüleyebileceği tek bir satırdan fazla olacak şekilde ayarladığınızda, kesilecek ve üç nokta (…) ile biten yalnızca bir metin satırı göreceksiniz. Bunun nedeni, numberOfLines adlı bir özelliğin ayarlanmış olmasıdır. 1 ve bu nedenle yalnızca bir satır görüntülenecektir. ‘UILabel’leri ele alırken yaygın bir hatadır ve birçok kişi bunu bir hata olarak düşünür veya bir metin satırından fazlasını göstermek için birden fazla etiket kullanabilirler, ancak sadece bu özelliği düzenleyerek şunu söyleyebiliriz. Belirtilen sayıda satırı kabul etmek için “UILabel”. Örneğin, bu özellik 5 olarak ayarlanırsa etiket 1, 2, 3, 4 veya 5 satırlık veri gösterebilir.

Değeri programlı olarak ayarlama #

Bu özelliği ayarlamak için ona yeni bir tamsayı atamanız yeterlidir:

Süratli #

label.numberOfLines = 2

Amaç-C #

label.numberOfLines = 2;

Not #

Bu özelliği 0’a ayarlamak mümkündür. Ancak bu, herhangi bir satırı kabul etmeyeceği anlamına gelmez, bunun yerine etiketin gerektiği kadar satıra sahip olabileceği anlamına gelir (“Sonsuz”):

Süratli #

label.numberOfLines = 0

Amaç-C #

label.numberOfLines = 0;

Not #

Etiketin bir yükseklik kısıtlaması varsa, kısıtlamaya uyulacaktır. Bu durumda label.numberOfLines = 0 beklendiği gibi çalışmayabilir.

Not #

Daha karmaşık çok satırlı bir metin için UITextView daha uygun olabilir.*

İçerik Sınırlarını Hesaplayın (yani dinamik hücre yükseklikleri için) #

Bir etiketin alacağı çerçeveyi hesaplamak istemek için yaygın bir kullanım durumu, tablo görünümü hücrelerini uygun şekilde boyutlandırmaktır. Bunu yapmanın önerilen yolu, boundingRectWithSize:options:attributes:context: NSString yöntemini kullanmaktır.

options, Dize çizim seçeneklerini alır:

  • Birden çok satıra sahip etiketler için NSStringDrawingUsesLineFragmentOrigin kullanılmalıdır
  • Maksimum sayıda satır varsa, | operatörü kullanılarak NSStringDrawingTruncatesLastVisibleLine eklenmelidir

“attributes”, atfedilen dizeleri etkileyen özniteliklerin bir “NSDictionary"sidir (tam liste: Apple Docs), ancak yüksekliği etkileyen faktörler şunları içerir:

  • NSFontAttributeName: Çok önemlidir, boyut ve yazı tipi ailesi, etiketin görüntülenen boyutunun kritik bir parçasıdır.

  • NSParagraphStyleAttributeName: Metnin nasıl görüntüleneceğini özelleştirmek için. Buna satır aralığı, metin hizalama, kesme stili ve diğer birkaç seçenek dahildir. Bu değerlerden herhangi birini açıkça değiştirmediyseniz, bu kadar endişelenmenize gerek yoktur, ancak IB’deki bazı değerleri değiştirdiyseniz önemli olabilir.

Birincil ‘NSStringDrawingContext’ kullanım durumu, yazı tipinin belirli bir doğruya sığdırmak için yeniden boyutlandırılmasına izin vermek için olduğundan, dinamik bir yükseklik hesaplıyorsak durum böyle olmamalıdır.

Hedef C

- (CGFloat)tableView:(UITableView *)tableView heightForRowAtIndexPath:(NSIndexPath *)indexPath {
    
    UITableViewCell *cell = [tableView cellForRowAtIndexPath:indexPath];

Metni sığdırmak için etiketi otomatik boyutlandırma #

Bu örnek, metin içeriği değiştiğinde bir etiketin genişliğinin nasıl otomatik olarak yeniden boyutlandırılabileceğini gösterir.

Sol ve üst kenarları sabitleyin #

Etiketin sol ve üst taraflarını sabitlemek için kısıtlamalar eklemek için otomatik düzeni kullanın.

Bundan sonra otomatik olarak yeniden boyutlandırılacaktır.

Notlar #

  • Bu örnek, bu Yığın Taşması yanıtından alınmıştır.

  • Genişlik ve yükseklik için kısıtlamalar eklemeyin. Etiketler, metin içeriğine göre öz bir boyuta sahiptir.

  • Otomatik düzeni kullanırken sizeToFit ayarlamanıza gerek yok. Örnek proje için tam kod burada:

      import UIKit
  class ViewController: UIViewController {

      @IBOutlet weak var myLabel: UILabel!

      @IBAction func changeTextButtonTapped(sender: UIButton) {
          myLabel.text = "my name is really long i want it to fit in this box"
      }
  }

  • Bu yöntem, bu örnekte gibi birden çok etiketi yatay olarak doğru şekilde yerleştirmek için de kullanılabilir.

  • Etiketinizin satır kaydırmasını istiyorsanız, o zaman IB’de satır sayısını 0 olarak ayarlayın ve koda `myLabel.preferredMaxLayoutWidth = 150 // veya her neyse’ ekleyin. (Düğme ayrıca etiketin altına sabitlenmiştir, böylece etiket yüksekliği arttığında aşağı hareket eder.)

Bilinmeyen metin uzunluğundan dinamik etiket çerçevesi #

Bazen, metin uzunluğunun bilinmediği dinamik içeriğe dayalı olarak bir UILabel’i yeniden boyutlandırmamız gerekir. Bu örnekte, UILabel’in genişliği 280 noktada sabitlenmiştir ve yüksekliği sonsuzdur, diyelim ki 9999. Çerçeveyi metin stiline ve maximumLabelSize’a göre tahmin etmek.

Hedef-C #

UILabel * label = [[UILabel alloc] init];

NSString *message = @"Some dynamic text for label";

//set the text and style if any.
label.text = message;

label.numberOfLines = 0;

CGSize maximumLabelSize = CGSizeMake(280, 9999); //280:max width of label and 9999-max height of label.

// use font information from the UILabel to calculate the size
CGSize expectedLabelSize = [label sizeThatFits:maximumLabelSize];

//Deprecated in iOS 7.0
//CGSize expectedLabelSize = [message sizeWithFont:label.font constrainedToSize:maximumLabelSize lineBreakMode:NSLineBreakByWordWrapping];

// create a frame that is filled with the UILabel frame data
CGRect newFrame = label.frame;

// resizing the frame to calculated size
newFrame.size.height = expectedLabelSize.height;

// put calculated frame into UILabel frame
label.frame = newFrame;

Süratli #

var message: String = "Some dynamic text for label"
//set the text and style if any.
label.text = message
label.numberOfLines = 0
var maximumLabelSize: CGSize = CGSize(width: 280, height: 9999)

// resizing the frame to calculated size

// put calculated frame into UILabel frame
label.frame = newFrame
