Komut satırı değişken ataması #

Komut satırından değişken atamak için -v kullanılabilir:

$ awk -v myvar="hello" 'BEGIN {print myvar}'
hello

Eşittir işaretinin etrafında boşluk olmadığına dikkat edin.

Bu, kabuk değişkenlerinin kullanılmasına izin verir:

$ shell_var="hello"
$ awk -v myvar="$shell_var" 'BEGIN {print myvar}'
hello

Ayrıca, bu, “awk"yi kontrol eden yerleşik değişkenlerin ayarlanmasına izin verir:

“FS” (alan ayırıcı) içeren bir örneğe bakın:

$ cat file
1,2;3,4
$ awk -v FS="," '{print $2}' file
2;3
$ awk -v FS=";" '{print $2}' file
3,4

Veya “OFS” (çıkış alanı ayırıcısı) ile:

$ echo "2 3" | awk -v OFS="--" '{print $1, $2}'
2--3
$ echo "2 3" | awk -v OFS="+" '{print $1, $2}'
2+3

-v seçeneğini kullanarak parametreleri bir programa iletmek #

-v seçeneği ve ardından değişken=değer biçiminde bir atama, parametreleri bir awk programına geçirmek için kullanılabilir. Bu, görevi “sınıfta konuşmayacağım” cümlesinin katını yazmak olan aşağıdaki ceza programı ile gösterilmiştir. standart çıktıda. Aşağıdaki örnek, öğretmenler arasında çok popüler olan 100 değerini kullanmaktadır:

awk -v count=100 'BEGIN {
  for(i = 1; i <= count; ++i) {
    print("I shall not talk in class.")
  }
  exit
}'

‘-v’ bayrağının tekrarlı kullanımı ile birden fazla parametre iletmek mümkündür:

awk -v count=100 -v "sentence=I shall not talk in class." 'BEGIN {
  for(i = 1; i <= count; ++i) {
    print(sentence)
  }
  exit
}'

Dizi veya liste parametreleri için yerleşik destek yoktur, bunların manuel olarak işlenmesi gerekir. Bir liste parametresini iletmek için klasik bir yaklaşım, bir sınırlayıcı kullanarak listeyi bitiştirmektir, popüler seçenekler :, | veya ,‘dir. split işlevi daha sonra listeyi bir awk dizisi olarak kurtarmaya izin verir:

awk -v 'serialised_list=a🅱️c:d:e:f' 'BEGIN {
  list_sz = split(serialised_list, list, ":")
  for(i = 1; i <= list_sz; ++i) {
    printf("list: %d: %s\n", i, list[i])
  }
  exit
}'

Bu awk programının çıktısı

list: 1: a
list: 2: b
list: 3: c
list: 4: d
list: 5: e
list: 6: f

Bazen liste öğelerini bir awk dizisinin anahtarları olarak kurtarmak daha uygundur, çünkü bu, üyelik doğrulamasını kolaylaştırır. Örneğin, aşağıdaki program, ilk sözcüğü sabit bir istisna listesine ait olmayan her satırı yazdırır:

awk -v 'serialised_exception_list=apple🍐cherry' 'BEGIN {
  _list_sz = split(serialised_exception_list, _list, ":")
  for(i = 1; i <= _list_sz; ++i) {
    exception[_list[i]]
  }
}

! ($1 in exception) { print }' <<EOF
apple Apples are yummy, I like them.
pineapple Do you like pineapple?
EOF

Bu programın çıktısı

pineapple Do you like pineapple?

Son bir örnek olarak, bir kabuk betiğinin parametreleri yardımcı bir awk betiğine nasıl aktardığını gösterdiğinden, ceza programının bir kabuk betiğine nasıl sarılacağını gösteriyoruz:

#!/bin/sh

usage()
{
   cat <<EOF
Usage: punishment [-c COUNT][-s SENTENCE]
 Prepare your punishments for you
EOF
}

punishment_count='100'
punishment_sentence='I shall not talk in class.'
while getopts "c:hs:" OPTION; do
  case "${OPTION}" in
    c) punishment_count="${OPTARG}";;
    s) punishment_sentence="${OPTARG}";;
    h) usage; exit 0;;
    *) usage; exit 64;;
  esac
done

awk -v "count=${punishment_count}" -v "sentence=${punishment_sentence}" 'BEGIN {
  for(i = 1; i <= count; ++i) {
    print(sentence)
  }
  exit
}'

Yerel değişkenler #

awk dili, işlevlerde yerel değişkenleri doğrudan desteklemez. Bununla birlikte, işlevlere fazladan argümanlar ekleyerek onları taklit etmek kolaydır. Gerçek parametreler olmadıklarını belirtmek için bu değişkenlerin önüne bir “_” koymak gelenekseldir.

Bu tekniği, bir dize etrafına tek tırnak işaretleri ekleyen bir “tek_teklif” işlevinin tanımıyla gösteriyoruz:

# single_quote(TEXT)
#  Return a string made of TEXT surrounded by single quotes

function single_quote(text, _quote) {
  _quote = sprintf("%c", 39)
  return sprintf("%s%s%s", _quote, text, _quote);
}

sprintf("'%s'", text) kullanmanın daha basit yaklaşımı pratik sorunlara yol açar çünkü awk komut dosyaları genellikle awk programına tek tırnaklı argümanlar olarak iletilir.

Atama Bağımsız Değişkenleri #

Atama bağımsız değişkenleri, bir “awk” çağrısının sonunda, dosya değişkenleriyle aynı alanda görünür, hem “-v” atamaları hem de bağımsız değişken atamaları, aşağıdaki normal ifadeyle eşleşmelidir. (bir POSIX yerel ayarı olduğu varsayılarak)

^[[:alpha:]_][[:alnum:]_]*=

Aşağıdaki örnek, aşağıdakileri içeren bir dosya “dosyası” olduğunu varsayar: “1 2 3” (boşlukla ayrılmış)

$ awk '{$1=$1}1' file OFS=, file OFS=- file
1 2 3
1,2,3
1-2-3
