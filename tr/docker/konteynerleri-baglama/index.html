Parametreler #

parametre Ayrıntılar
tty:true docker-compose.yml dosyasında, “tty: true” bayrağı, kapsayıcının “sh” komutunu girdi beklerken çalışır durumda tutar.

“Ana bilgisayar” ve “köprü” ağ sürücüleri, kapsayıcıları tek bir liman işçisi ana bilgisayarına bağlayabilir. Kapsayıcıların bir makinenin ötesinde iletişim kurmasına izin vermek için bir yer paylaşımlı ağ oluşturun. Ağı oluşturma adımları, liman işçisi ana bilgisayarlarınızın nasıl yönetildiğine bağlıdır.

  • Sürü Modu: docker network create --driver overlay
  • docker/swarm: bir harici anahtar/değer deposu gerektirir

Docker ağı #

Aynı liman işçisi ağındaki kapsayıcıların açıkta kalan bağlantı noktalarına erişimi vardır.

docker network create sample
docker run --net sample --name keys consul agent -server -client=0.0.0.0 -bootstrap

Consul’s Dockerfile ‘8500’, ‘8600’ ve birkaç bağlantı noktası daha gösterir. Göstermek için aynı ağda başka bir kapsayıcı çalıştırın:

docker run --net sample -ti alpine sh
/ # wget -qO- keys:8500/v1/catalog/nodes

Burada konsolos konteyneri, ilk komutta verilen isim olan ‘keys’ten çözülür. Docker, kapsayıcıları `–name’lerine göre bulmak için bu ağda dns çözümlemesi sağlar.

Docker-oluşturma #

Ağlar, bir oluşturma dosyasında (v2) belirtilebilir. Varsayılan olarak tüm kapsayıcılar paylaşılan bir ağdadır.

Bu dosyayla başlayın: example/docker-compose.yml:

version: '2'
services:
  keys:
    image: consul
    command: agent -server -client=0.0.0.0 -bootstrap
  test:
    image: alpine
    tty: true
    command: sh

Bu yığını “docker-compose up -d” ile başlatmak, bu durumda “example_default” adlı üst dizinden sonra adlandırılan bir ağ oluşturacaktır. docker network ls ile kontrol edin

 > docker network ls
NETWORK ID          NAME                    DRIVER              SCOPE
719eafa8690b        example_default         bridge              local

Kapsayıcıların çözebildiğini ve iletişim kurabildiğini doğrulamak için dağ kapsayıcısına bağlanın:

 > docker exec -ti example_test_1 sh
/ # nslookup keys
...
/ # wget -qO- keys:8500/v1/kv/?recurse
...

Bir oluşturma dosyası, docker network komutundan ağ adını, sürücüyü ve diğer seçenekleri belirtmek için bir “ağlar:” üst düzey bölümüne sahip olabilir.

Konteyner Bağlantısı #

Docker --link argümanı ve link: bölümleri docker-compose diğer kapsayıcılara takma adlar yapar.

docker network create sample
docker run -d --net sample --name redis redis

Bağlantı ile, orijinal ad veya eşleme, redis kapsayıcısını çözecektir.

> docker run --net sample --link redis:cache -ti python:alpine sh -c "pip install redis && python"
>>> import redis
>>> r = redis.StrictRedis(host='cache')
>>> r.set('key', 'value')
True

Docker ‘1.10.0’ konteyner bağlantısından önce ağ bağlantısı da kurulur - davranış şimdi liman işçisi ağı tarafından sağlanır. Sonraki sürümlerdeki bağlantılar, varsayılan köprü ağında yalnızca eski etkisi sağlar.
