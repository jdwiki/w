Sürü, toplu olarak hizmetleri dağıtan bir dizi Docker Motorudur (veya düğümler). Swarm, işlemeyi birçok fiziksel, sanal veya bulut makineye dağıtmak için kullanılır.

Sözdizimi #

  • Bir sürüyü başlat: docker swarm init [SEÇENEKLER]

  • Düğüm ve/veya yönetici olarak bir sürüye katılın: docker sürüsü birleştirme [SEÇENEKLER] HOST:PORT

  • Yeni bir hizmet oluştur: liman işçisi hizmeti oluştur [SEÇENEKLER] GÖRÜNTÜ [KOMUTU] [ARG…]

  • Bir veya daha fazla hizmetle ilgili ayrıntılı bilgileri görüntüleyin: liman işçisi hizmetini inceleyin [SEÇENEKLER] HİZMET [SERVICE…]

  • Hizmetleri listeleyin: liman işçisi hizmeti ls [SEÇENEKLER]

  • Bir veya daha fazla hizmeti kaldırın: docker service rm SERVICE [SERVICE…]

  • Bir veya birden fazla çoğaltılan hizmeti ölçeklendirin: liman işçisi hizmet ölçeği SERVICE=REPLIKAS [SERVICE=REPLICAS…]

  • Bir veya daha fazla hizmetin görevlerini listeleyin: liman işçisi hizmeti ps [SEÇENEKLER] HİZMET [SERVICE…]

  • Bir hizmeti güncelleyin: liman işçisi hizmeti güncellemesi [SEÇENEKLER] HİZMET

Sürü modu aşağıdaki özellikleri uygular:

  • Docker Engine ile entegre küme yönetimi
  • Merkezi olmayan tasarım
  • Bildirimli hizmet modeli
  • Ölçekleme
  • İstenen devlet uzlaşması
  • Çok ana bilgisayar ağı
  • Servis keşfi
  • Yük dengeleme
  • Varsayılan olarak güvenli tasarım
  • Sürekli güncellemeler

Swarm ile ilgili daha resmi Docker belgeleri için: Swarm moduna genel bakış


Swarm Modu CLI Komutları #

Belgeler için komut açıklamasına tıklayın

Bir sürü başlatma

docker swarm init [OPTIONS]

Düğüm ve/veya yönetici olarak bir sürüye katılın

docker swarm join [OPTIONS] HOST:PORT

Yeni bir hizmet oluşturun

docker service create [OPTIONS] IMAGE [COMMAND] [ARG...]

Bir veya daha fazla hizmetle ilgili ayrıntılı bilgileri görüntüleyin

docker service inspect [OPTIONS] SERVICE [SERVICE...]

Hizmetleri listeleyin

docker service ls [OPTIONS]

Bir veya daha fazla hizmeti kaldırın

docker service rm SERVICE [SERVICE...]

Bir veya daha fazla çoğaltılmış hizmeti ölçeklendirin

docker service scale SERVICE=REPLICAS [SERVICE=REPLICAS...]

Bir veya daha fazla hizmetin görevlerini listeleyin

docker service ps [OPTIONS] SERVICE [SERVICE...]

Bir hizmeti güncelleme

docker service update [OPTIONS] SERVICE

Docker-machine ve VirtualBox kullanarak Linux’ta bir sürü oluşturun #

# Create the nodes
# In a real world scenario we would use at least 3 managers to cover the fail of one manager.
docker-machine create -d virtualbox manager
docker-machine create -d virtualbox worker1

# Create the swarm
# It is possible to define a port for the *advertise-addr* and *listen-addr*, if none is defined the default port 2377 will be used.
docker-machine ssh manager \
    docker swarm init \
    --advertise-addr $(docker-machine ip manager)
    --listen-addr $(docker-machine ip manager)

# Extract the Tokens for joining the Swarm
# There are 2 different Tokens for joining the swarm.
MANAGER_TOKEN=$(docker-machine ssh manager docker swarm join-token manager --quiet)
WORKER_TOKEN=$(docker-machine ssh manager docker swarm join-token worker --quiet)


# Join a worker node with the worker token
docker-machine ssh worker1 \
    docker swarm join \
    --token $WORKER_TOKEN \
    --listen-addr $(docker-machine ip worker1) \
    $(docker-machine ip manager):2377

Düğüm Kullanılabilirliği #

Sürü Modu Düğüm Kullanılabilirliği:

  • Etkin, zamanlayıcının bir düğüme görevler atayabileceği anlamına gelir.
  • Duraklatma, zamanlayıcının düğüme yeni görevler atamadığı, ancak mevcut görevler çalışmaya devam eder.
  • Boşaltma, zamanlayıcının düğüme yeni görevler atamadığı anlamına gelir. bu zamanlayıcı, mevcut görevleri kapatır ve bunları bir kullanılabilir düğüm

Mod Kullanılabilirliğini değiştirmek için:

#Following commands can be used on swarm manager(s)
docker node update --availability drain node-1
#to verify:
docker node ls

Sürü Düğümlerini Yükselt veya Azalt #

Bir düğümü veya düğüm kümesini yükseltmek için, bir yönetici düğümünden ‘docker düğümü yükseltmesini’ çalıştırın:

docker node promote node-3 node-2

Node node-3 promoted to a manager in the swarm.
Node node-2 promoted to a manager in the swarm.

Bir düğümü veya düğüm kümesini düşürmek için, bir yönetici düğümünden “docker düğümü indirgeme” komutunu çalıştırın:

docker node demote node-3 node-2

Manager node-3 demoted in the swarm.
Manager node-2 demoted in the swarm.

Çalışan ve yönetici katılım belirtecini öğrenin #

Bir sürüye yeni düğümlerin sağlanmasını otomatikleştirirken, yöneticinin reklamı yapılan adresinin yanı sıra sürü için doğru birleştirme belirtecinin ne olduğunu bilmeniz gerekir. Bunu, mevcut yönetici düğümlerinden herhangi birinde aşağıdaki komutları çalıştırarak öğrenebilirsiniz:

# grab the ipaddress:port of the manager (second last line minus the whitespace)
export MANAGER_ADDRESS=$(docker swarm join-token worker | tail -n 2 |  tr -d '[[:space:]]')

# grab the manager and worker token
export MANAGER_TOKEN=$(docker swarm join-token manager -q)
export WORKER_TOKEN=$(docker swarm join-token worker -q)

-q seçeneği yalnızca belirteci çıkarır. Bu seçenek olmadan, bir sürüye kaydolmak için tam komutu alırsınız.

Ardından, yeni sağlanan düğümlerde, kullanarak sürüye katılabilirsiniz.

docker swarm join --token $WORKER_TOKEN $MANAGER_ADDRESS

Merhaba dünya uygulaması #

Genellikle, çoğaltılmış ve düzenlenmiş bir uygulama oluşturmak için bir hizmet yığını oluşturmak istersiniz.

Tipik bir modern web uygulaması, bir veritabanı, api, ön uç ve ters proxy’den oluşur.

Süreklilik

Veritabanının kalıcılığa ihtiyacı var, bu nedenle bir sürüdeki tüm düğümler arasında paylaşılan bazı dosya sistemlerine ihtiyacımız var. NAS, NFS sunucusu, GFS2 veya başka bir şey olabilir. Bunu ayarlamak burada kapsam dışındadır. Şu anda Docker, bir sürüde kalıcılığı içermiyor ve yönetmiyor. Bu örnek, tüm düğümlere monte edilmiş /nfs/ paylaşılan konumu olduğunu varsayar.

Ağ

Birbiriyle iletişim kurabilmek için bir sürüdeki servislerin aynı ağ üzerinde olması gerekir.

Bir IP aralığı (burada ‘10.0.9.0/24’) ve ağ adı (‘merhaba-ağ’) seçin ve bir komut çalıştırın:

docker network create \
  --driver overlay \
  --subnet 10.0.9.0/24 \
  --opt encrypted \
  hello-network

Veri tabanı

İhtiyacımız olan ilk hizmet bir veritabanıdır. Örnek olarak postgresql kullanalım. nfs/postgres içinde bir veritabanı için bir klasör oluşturun ve şunu çalıştırın:

docker service create --replicas 1 --name hello-db \
       --network hello-network -e PGDATA=/var/lib/postgresql/data \
       --mount type=bind,src=/nfs/postgres,dst=/var/lib/postgresql/data \
       kiasaki/alpine-postgres:9.5

“–network merhaba-network” ve “–mount” seçeneklerini kullandığımıza dikkat edin.

API

API oluşturmak bu örneğin kapsamı dışındadır, bu yüzden username/hello-api altında bir API imajınız olduğunu varsayalım.

docker service create --replicas 1 --name hello-api \
       --network hello-network \
       -e NODE_ENV=production -e PORT=80 -e POSTGRESQL_HOST=hello-db \
       username/hello-api

Veritabanı hizmetimizin adını geçtiğimize dikkat edin. Docker sürüsü, yerleşik bir döngüsel DNS sunucusuna sahiptir, bu nedenle API, DNS adını kullanarak veritabanına bağlanabilecektir.

Ters proxy

API’mizi bir dış dünyaya sunmak için nginx hizmeti oluşturalım. Paylaşılan bir konumda nginx yapılandırma dosyaları oluşturun ve şunu çalıştırın:

docker service create --replicas 1 --name hello-load-balancer \
       --network hello-network \
       --mount type=bind,src=/nfs/nginx/nginx.conf,dst=/etc/nginx/nginx.conf \
       -p 80:80 \
       nginx:1.10-alpine

Bir bağlantı noktası yayınlamak için -p seçeneğini kullandığımıza dikkat edin. Bu bağlantı noktası, bir sürüdeki herhangi bir düğüm için kullanılabilir olacaktır.

Sürüden Ayrılmak #

Çalışan Düğümü:

#Run the following on the worker node to leave the swarm.

docker swarm leave
Node left the swarm.

Düğüm Yönetici rolüne sahipse, Yönetici nisabını koruma konusunda bir uyarı alırsınız. Yönetici düğümünden çıkmak için –force kullanabilirsiniz:

#Manager Node

docker swarm leave --force
Node left the swarm.

Swarm’dan ayrılan düğümler, ‘docker node ls’ çıktısında görünmeye devam edecek.

Düğümleri listeden kaldırmak için:

docker node rm node-2
