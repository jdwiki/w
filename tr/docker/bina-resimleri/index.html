Dockerfile’dan görüntü oluşturma #

Bir Dockerfile’ınız olduğunda, `docker build’i kullanarak ondan bir görüntü oluşturabilirsiniz. Bu komutun temel biçimi şudur:

docker build -t resim-adı yol

Dockerfile’niz Dockerfile olarak adlandırılmamışsa, oluşturulacak Dockerfile adını vermek için -f bayrağını kullanabilirsiniz.

docker build -t image-name -f Dockerfile2 .

Örneğin, geçerli çalışma dizinindeki bir “Dockerfile"dan “dockerbuild-example:1.0.0” adlı bir görüntü oluşturmak için:

$ ls
Dockerfile Dockerfile2

$ docker build -t dockerbuild-example:1.0.0 .

$ docker build -t dockerbuild-example-2:1.0.0 -f Dockerfile2 .

Daha fazla seçenek ve ayar için [docker build kullanım belgelerine] bakın.

Yaygın bir hata, kullanıcı ana dizininde (~) bir Dockerfile oluşturmaktır. Bu kötü bir fikir çünkü docker build -t mytag sırasında bu mesaj uzun süre görünecek:

Bağlam yükleniyor

Bunun nedeni, docker arka plan programının tüm kullanıcının dosyalarını (hem ana dizini hem de alt dizinlerini) kopyalamaya çalışmasıdır. Dockerfile için her zaman bir dizin belirleyerek bundan kaçının.

Derleme dizinine bir .dockerignore dosyası eklemek iyi bir uygulamadır. Sözdizimi “.gitignore” dosyalarına benzer ve derleme bağlamı olarak yalnızca istenen dosya ve dizinlerin yüklenmesini sağlar.

ENTRYPOINT ve CMD arasındaki fark #

Yerleşik görüntülerde varsayılan olarak hangi komutun çalıştırılacağını belirtmek için iki “Dockerfile” yönergesi vardır. Yalnızca “CMD"yi belirtirseniz, docker bu komutu “/bin/sh -c” olan varsayılan “GİRİŞ NOKTASI"nı kullanarak çalıştırır. Yerleşik görüntüyü başlattığınızda, giriş noktasını ve/veya komutu veya her ikisini birden geçersiz kılabilirsiniz. Her ikisini de belirtirseniz, “GİRİŞ NOKTASI”, kapsayıcı işleminizin yürütülebilir dosyasını belirtir ve bu yürütülebilir dosyanın parametreleri olarak “CMD” sağlanır.

Örneğin, “Dockerfile” dosyanız şunları içeriyorsa:

FROM ubuntu:16.04
CMD ["/bin/date"]

O zaman /bin/sh -c varsayılan ENTRYPOINT yönergesini kullanıyorsunuz ve bu varsayılan giriş noktasıyla /bin/date komutunu çalıştırıyorsunuz. Kapsayıcı işleminizin komutu /bin/sh -c /bin/date olacaktır. Bu görüntüyü çalıştırdığınızda, varsayılan olarak geçerli tarihi yazdıracaktır.

$ docker build -t test .
$ docker run test
Tue Jul 19 10:37:43 UTC 2016

Komut satırında “CMD"yi geçersiz kılabilirsiniz, bu durumda belirttiğiniz komutu çalıştıracaktır.

$ docker run test /bin/hostname
bf0274ec8820

Bir “ENTRYPOINT” yönergesi belirtirseniz, Docker bu yürütülebilir dosyayı kullanır ve “CMD” yönergesi komutun varsayılan parametrelerini belirtir. Dolayısıyla, “Dockerfile” dosyanız şunları içeriyorsa:

FROM ubuntu:16.04
ENTRYPOINT ["/bin/echo"]
CMD ["Hello"]

Sonra çalıştırmak üretecek

$ docker build -t test .
$ docker run test
Hello

İsterseniz farklı parametreler sağlayabilirsiniz, ancak hepsi /bin/echo komutunu çalıştıracaktır.

$ docker run test Hi
Hi

Docker dosyanızda listelenen giriş noktasını geçersiz kılmak istiyorsanız (yani, bu kapsayıcıda “echo"dan farklı bir komut çalıştırmak istiyorsanız), komut satırında “–entrypoint” parametresini belirtmeniz gerekir:

$ docker run --entrypoint=/bin/hostname test
b2c70e74df18

Genellikle, çalıştırmak istediğiniz ana uygulamanıza işaret etmek için ‘ENTRYPOINT’ yönergesini ve varsayılan parametrelere ‘CMD’yi kullanırsınız.

Basit bir Docker dosyası #

FROM node:5

FROM yönergesi, başlamak için bir görüntü belirtir. Herhangi bir geçerli [resim referansı] kullanılabilir.

WORKDIR /usr/src/app

‘WORKDIR’ yönergesi, konteynerin içinde ‘cd’ çalıştırmaya eşdeğer olan geçerli çalışma dizinini konteyner içinde ayarlar. (Not: RUN cd geçerli çalışma dizinini değiştirmez.)

RUN npm install cowsay knock-knock-jokes

‘RUN’, verilen komutu kapsayıcının içinde yürütür.

COPY cowsay-knockknock.js ./

“KOPYALA”, derleme bağlamından ilk bağımsız değişkende belirtilen dosya veya dizini kopyalar (docker build path< öğesine iletilen yol /code>) ikinci bağımsız değişken tarafından belirtilen kapsayıcıdaki konuma.

CMD node cowsay-knockknock.js

“CMD”, görüntü [çalıştır] olduğunda ve hiçbir komut verilmediğinde yürütülecek bir komutu belirtir. [‘docker run’a bir komut ileterek] geçersiz kılınabilir.

Daha birçok talimat ve seçenek var; tam bir liste için [Dockerfile referansı]’na bakın.

Bir Görüntüyü Docker Hub’a veya başka bir Kayıt Defterine Gönderme ve Çekme #

Yerel olarak oluşturulan görüntüler, Docker Hub’e veya kayıt olarak bilinen diğer herhangi bir docker repo ana bilgisayarına gönderilebilir. Mevcut bir liman işçisi hub hesabında oturum açmak için “docker login"i kullanın.

docker login

Login with your Docker ID to push and pull images from Docker Hub.
If you don't have a Docker ID, head over to https://hub.docker.com to create one.

Username: cjsimon
Password:
Login Succeeded

Bir sunucu adı belirtilerek farklı bir liman işçisi kayıt defteri kullanılabilir. Bu aynı zamanda özel veya kendi kendine barındırılan kayıtlar için de geçerlidir. Ayrıca, güvenlik için bir harici kimlik bilgisi deposu kullanmak mümkündür.

docker login quay.io

Ardından görüntüleri etiketleyebilir ve oturum açtığınız kayıt defterine gönderebilirsiniz. Deponuz “sunucu/kullanıcı adı/repoadı:etiket” olarak belirtilmelidir. Sunucuyu atlamak, şu anda varsayılan olarak Docker Hub’dır. (Varsayılan kayıt defteri başka bir sağlayıcıyla değiştirilemez ve bu özelliği uygulamak için plan yok yoktur.)

docker tag mynginx quay.io/cjsimon/mynginx:latest

Aynı görüntünün farklı sürümlerini veya dallarını temsil etmek için farklı etiketler kullanılabilir. Birden fazla farklı etikete sahip bir resim, her etiketi aynı depoda görüntüler.

Yeni etiketlenmiş görüntünüz de dahil olmak üzere yerel makinenize yüklenen yüklü görüntülerin bir listesini görmek için ‘docker görüntüleri’ni kullanın. Ardından, kayıt defterine yüklemek için Push’u kullanın ve görüntüyü indirmek için çekin.

docker push quay.io/cjsimon/mynginx:latest

Bir görüntünün tüm etiketleri -a seçeneği belirtilerek çekilebilir.

docker pull quay.io/cjsimon/mynginx:latest

Dockerfile’de Bir Bağlantı Noktasını Gösterme #

EXPOSE <port> [<port>...]

Docker’ın belgelerinden:

EXPOSE komutu Docker’a konteynerin çalışma zamanında belirtilen ağ portlarını dinlediğini bildirir. “EXPOSE”, konteynerin bağlantı noktalarını ana bilgisayar tarafından erişilebilir hale getirmez. Bunu yapmak için, bir dizi bağlantı noktasını yayınlamak için “-p” işaretini veya açıktaki tüm bağlantı noktalarını yayınlamak için “-P” işaretini kullanmalısınız. Bir bağlantı noktası numarasını açığa çıkarabilir ve başka bir numara altında harici olarak yayınlayabilirsiniz.

Örnek: #

Docker dosyanızın içinde:

EXPOSE 8765

Bu bağlantı noktasına ana makineden erişmek için, bu argümanı “docker run” komutunuza ekleyin:

-p 8765:8765

ENTRYPOINT ve CMD fiil ve parametre olarak görülüyor #

İle biten bir Docker dosyanız olduğunu varsayalım.

GİRİŞ NOKTASI [ "nethogs"] CMD ["wlan0"]

bu görüntüyü bir

docker yerleşik -t denetçisi .

böyle bir Dockerfile ile oluşturulmuş görüntüyü aşağıdaki gibi bir komutla başlatın

docker run -it --net=host --rm denetçisi

,nethogs wlan0 adlı arayüzü izleyecek

Şimdi, eth0 (veya wlan1 veya ra1…) arayüzünü izlemek istiyorsanız, şöyle bir şey yapacaksınız.

docker run -it --net=host --rm denetçisi eth0

veya

docker run -it --net=host --rm denetçisi wlan1

Proxy kullanarak oluşturma #

Genellikle bir Docker görüntüsü oluştururken, Dockerfile, Internet’ten kaynakları almak için programları çalıştıran yönergeler içerir (örneğin, GitHub’da bir program ikili derlemesini çekmek için “wget”).

Docker’a, bu tür programların bu getirmeleri bir proxy aracılığıyla gerçekleştirmesi için ayarlanmış ortam değişkenlerini iletmesi talimatını vermek mümkündür:

$ docker build --build-arg http_proxy=http://myproxy.example.com:3128 \
               --build-arg https_proxy=http://myproxy.example.com:3128 \
               --build-arg no_proxy=internal.example.com \
               -t test .

“build-arg”, yalnızca derleme sırasında kullanılabilen ortam değişkenleridir.
