Docker’ı Windows’a Yükleme #

Gereksinimler: Donanım Sanallaştırma Teknolojisini destekleyen ve etkinleştirilen bir makinede Windows 7 veya üzeri 64 bit sürümü.

Docker ikili dosyası Windows’ta yerel olarak çalışabilirken, kapsayıcıları oluşturmak ve barındırmak için kutuda bir Linux sanal makinesi çalıştırmanız gerekir.

1.12 sürümünden bu yana, Docker, arka uç görevi görecek küçük bir Linux makinesini başlatmak için Windows’un yerel Hyper-V işlevselliğini kullanabileceğinden, kurulacak ayrı bir VM’ye ihtiyacınız yoktur.

Docker’ı yüklemek için aşağıdaki adımları izleyin:

  1. Windows için Docker’e gidin
  2. Yükleyiciyi indirin ve çalıştırın.
  3. Varsayılan seçeneklerle yükleyiciye devam edin ve istendiğinde hesap kimlik bilgilerinizi girin.

Kurulum hakkında daha fazla bilgi için burayı kontrol edin.

son sürüm

1.11 sürümüne kadar bu Linux sanal makinesini çalıştırmanın en iyi yolu Docker, VirtualBox ve Linux konuk makinesini yükleyen Docker Toolbox’ı kurmaktır.

Docker araç kutusunu yüklemek için aşağıdaki adımları izleyin:

  1. [Docker Araç Kutusu]’na gidin(https://www.docker.com/toolbox)
  2. Windows bağlantısını tıklayın ve yükleyiciyi çalıştırın.
  3. Varsayılan seçeneklerle yükleyiciye devam edin ve istendiğinde hesap kimlik bilgilerinizi girin.

Bu, Docker ikili dosyalarını Program Dosyalarına yükleyecek ve mevcut Sanal Kutu kurulumlarını güncelleyecektir. Yükleme hakkında daha fazla bilgi için burayı kontrol edin.

son sürüm

Kurulumu Doğrulamak için:

  1. Henüz başlatılmadıysa Başlat menüsünden Dockerı başlatın ve çalıştığından emin olun. Ardından herhangi bir terminali açın (‘cmd’ veya PowerShell)
son sürüm
  1. Masaüstünüzde Docker Toolbox simgesini bulun. Docker Toolbox terminalini başlatmak için simgeye tıklayın.
son sürüm

  1. Terminal açık tip olduğunda

    docker run hello-world

  2. Her şey yolundaysa, bu, kurulumun başarılı olduğunu doğrulayan bir karşılama mesajı yazdırmalıdır.

Docker’ı Mac OS X’e Yükleme #

Gereksinimler: Docker’ı çalıştırmak için OS X 10.8 “Mountain Lion” veya daha yenisi gerekir.

Docker ikili dosyası yerel olarak Mac OS X’te çalışabilirken, kapsayıcıları oluşturmak ve barındırmak için kutuda bir Linux sanal makinesi çalıştırmanız gerekir.

1.12 sürümünden bu yana, kurulacak ayrı bir VM’ye ihtiyacınız yoktur, çünkü Docker, arka uç görevi görecek küçük bir Linux makinesini başlatmak için OSX’in yerel ‘Hypervisor.framework’ işlevini kullanabilir.

Docker’ı yüklemek için aşağıdaki adımları izleyin:

  1. Mac için Docker’e gidin
  2. Yükleyiciyi indirin ve çalıştırın.
  3. Varsayılan seçeneklerle yükleyiciye devam edin ve istendiğinde hesap kimlik bilgilerinizi girin.

Kurulum hakkında daha fazla bilgi için burayı kontrol edin.

son sürüm

1.11 sürümüne kadar bu Linux sanal makinesini çalıştırmanın en iyi yolu Docker, VirtualBox ve Linux konuk makinesini yükleyen Docker Toolbox’ı kurmaktır.

Docker araç kutusunu yüklemek için aşağıdaki adımları izleyin:

  1. [Docker Araç Kutusu]’na gidin(https://www.docker.com/toolbox)
  2. Mac bağlantısını tıklayın ve yükleyiciyi çalıştırın.
  3. Varsayılan seçeneklerle yükleyiciye devam edin ve istendiğinde hesap kimlik bilgilerinizi girin.

Bu, Docker ikili dosyalarını /usr/local/bin içine kuracak ve mevcut herhangi bir Sanal Kutu kurulumunu güncelleyecektir. Yükleme hakkında daha fazla bilgi için burayı kontrol edin.

son sürüm

Kurulumu Doğrulamak için:

  1. Uygulamalar klasöründen Docker.app dosyasını başlatın ve çalıştığından emin olun. Ardından Terminal’i açın.
son sürüm
  1. Bir terminal açacak ve onu Docker komutları için kullanıma hazırlayacak olan Docker Hızlı Başlangıç ​​Terminalini açın.
son sürüm

  1. Terminal açık tip olduğunda

    $ docker run hello-world

  2. Her şey yolundaysa, bu, kurulumun başarılı olduğunu doğrulayan bir karşılama mesajı yazdırmalıdır.

Ubuntu Linux’ta docker kurulumu #

Docker, Ubuntu Linux’un aşağıdaki 64-bit sürümlerinde desteklenir:

*Ubuntu Xenial 16.04 (LTS)

  • Ücretsiz Wily 15.10
  • Ubuntu Güvenilir 14.04 (LTS)
  • Ubuntu Kesin 12.04 (LTS)

Birkaç not:

Aşağıdaki talimatlar, yalnızca Docker paketleri kullanılarak kurulumu içerir ve bu, Docker‘ın en son resmi sürümünün alınmasını sağlar. Yalnızca “Ubuntu tarafından yönetilen” paketleri kullanarak yüklemeniz gerekiyorsa, Ubuntu belgelerine bakın (Belirgin nedenlerle başka türlü önerilmez).

Ubuntu Utopic 14.10 ve 15.04, Docker’ın APT deposunda bulunur ancak bilinen güvenlik sorunları nedeniyle artık resmi olarak desteklenmemektedir.

Ön koşullar

  • Docker yalnızca 64 bit Linux kurulumunda çalışır.
  • Docker, Linux çekirdeği sürüm 3.10 veya üzerini gerektirir (sürüm 3.13 veya üzeri gerektiren Ubuntu Precise 12.04 hariç). 3.10’dan eski çekirdekler, Docker kapsayıcılarını çalıştırmak için gereken bazı özelliklerden yoksundur ve belirli koşullar altında veri kaybına ve sıklıkla paniğe neden olan bilinen hataları içerir. ‘uname -r’ komutuyla mevcut çekirdek sürümünü kontrol edin. ‘Ubuntu Precise (12.04 LTS)’ çekirdeğinizi daha aşağı kaydırarak güncellemeniz gerekiyorsa bu gönderiyi kontrol edin. Buna WikiHow bakın diğer Ubuntu kurulumları için en son sürümü edinmek için gönderin.

APT kaynaklarını güncelleyin

Docker deposundaki paketlere erişmek için bunun yapılması gerekir.

  1. Makinenizde “sudo” veya “root” ayrıcalıklarına sahip bir kullanıcı olarak oturum açın.

  2. Bir terminal penceresi açın.

  3. Paket bilgilerini güncelleyin, APT’nin https yöntemiyle çalıştığından ve CA sertifikalarının kurulu olduğundan emin olun.

    $ sudo apt-get update $ sudo apt-get install
    apt-transport-https
    ca-certificates
    curl
    software-properties-common

  4. Docker’ın resmi GPG anahtarını ekleyin:

     $ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

Anahtar parmak izinin 9DC8 5822 9FC7 DD38 854A E2D8 8D81 803C 0EBF CD88 olduğunu doğrulayın.

    $ sudo apt-key fingerprint 0EBFCD88

        
        pub   4096R/0EBFCD88 2017-02-22
              Key fingerprint = 9DC8 5822 9FC7 DD38 854A  E2D8 8D81 803C 0EBF CD88
        uid                  Docker Release (CE deb) <docker@docker.com>
        sub   4096R/F273FCD8 2017-02-22
  1. Aşağıdaki tabloda Ubuntu sürümünüze karşılık gelen girişi bulun. Bu, APT’nin Docker paketlerini nerede arayacağını belirler. Mümkün olduğunda, Ubuntu’nun uzun vadeli bir destek (LTS) sürümünü çalıştırın.
Ubuntu Sürümü Depo
Hassas 12.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-precise main
Güvenilir 14.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-güvenilir ana
Wily 15.10 deb https://apt.dockerproject.org/repo ubuntu-wily main
Xenial 16.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-xenial main

Not: Docker, tüm mimariler için paket sağlamaz. İkili yapılar her gece oluşturulur ve bunları “https://master.dockerproject.org” adresinden indirebilirsiniz. Docker’ı çok mimarili bir sisteme kurmak için, girişe bir [arch=...] yan tümcesi ekleyin. Ayrıntılar için Debian Multiarch wiki‘ye bakın.

  1. <REPO> yer tutucusu yerine işletim sisteminizin girişini değiştirerek aşağıdaki komutu çalıştırın.

    $ echo “” | sudo tee /etc/apt/sources.list.d/docker.list

  2. “Sudo apt-get update” komutunu çalıştırarak “APT” paket dizinini güncelleyin.

  3. “APT"nin doğru depodan çekildiğini doğrulayın.

Aşağıdaki komutu çalıştırdığınızda, yüklemeniz için kullanılabilen her Docker sürümü için bir girdi döndürülür. Her girişin https://apt.dockerproject.org/repo/ URL’si olmalıdır. Şu anda yüklü olan sürüm *** ile işaretlenmiştir.Aşağıdaki örneğin çıktısına bakın.

$ apt-cache policy docker-engine

  docker-engine:
    Installed: 1.12.2-0~trusty
    Candidate: 1.12.2-0~trusty
    Version table:
   *** 1.12.2-0~trusty 0
          500 https://apt.dockerproject.org/repo/ ubuntu-trusty/main amd64 Packages
          100 /var/lib/dpkg/status
       1.12.1-0~trusty 0
          500 https://apt.dockerproject.org/repo/ ubuntu-trusty/main amd64 Packages
       1.12.0-0~trusty 0
          500 https://apt.dockerproject.org/repo/ ubuntu-trusty/main amd64 Packages

Şu andan itibaren ‘apt-get upgrade’i çalıştırdığınızda, ‘APT’ yeni depodan çeker.

Ubuntu Sürümüne göre Önkoşullar

Ubuntu Trusty (14.04), Wily (15.10) ve Xenial (16.04) için, “aufs” depolama sürücüsünü kullanmanıza izin veren “linux-image-extra-*” çekirdek paketlerini kurun.

linux-image-extra-* paketlerini kurmak için:

  1. Ubuntu sunucunuzda bir terminal açın.

  2. Paket yöneticinizi sudo apt-get update komutuyla güncelleyin.

  3. Önerilen paketleri kurun.

    $ sudo apt-get install linux-image-extra-$(uname -r) linux-image-extra-virtual

  4. Docker kurulumuna geçin

Ubuntu Precise (12.04 LTS) için Docker, 3.13 çekirdek sürümünü gerektirir. Çekirdek sürümünüz 3.13’ten daha eskiyse, yükseltmeniz gerekir. Ortamınız için hangi paketlerin gerekli olduğunu görmek için bu tabloya bakın:

Paket Açıklama
linux-image-generic-lts-trusty Genel Linux çekirdek görüntüsü. Bu çekirdeğin yerleşik ‘AUFS’si vardır. Bu, Docker’ı çalıştırmak için gereklidir.
linux-headers-generic-lts-trusty Bunlara bağlı olan “ZFS” ve “VirtualBox konuk eklemeleri” gibi paketlere izin verir. Mevcut çekirdeğiniz için başlıkları yüklemediyseniz, “güvenilir” çekirdek için bu başlıkları atlayabilirsiniz. Emin değilseniz, güvenlik için bu paketi eklemelisiniz.
`xserver-xorg-lts-güvenilir’ Unity/Xorg olmadan grafiksel olmayan ortamlarda isteğe bağlıdır. Gerekli Docker’ı bir grafik ortamla makinede çalıştırırken.
`ligbl1-mesa-glx-lts-güvenilir’ Bu paketlerin nedenleri hakkında daha fazla bilgi edinmek için, özellikle LTS Etkinleştirme Yığını olmak üzere, desteklenen çekirdeklerin kurulum talimatlarını okuyun. Her versiyonun altındaki not 5’e bakın.

Çekirdeğinizi yükseltmek ve ek paketleri kurmak için aşağıdakileri yapın:

  1. Ubuntu sunucunuzda bir terminal açın.

  2. Paket yöneticinizi sudo apt-get update komutuyla güncelleyin.

  3. Gereken ve isteğe bağlı paketleri kurun.

    $ sudo apt-get install linux-image-generic-lts-trusty

  4. Kurmanız gereken diğer paketler için bu adımı tekrarlayın.

  5. Güncellenmiş çekirdeği kullanmak için “sudo yeniden başlatma” komutunu kullanarak ana makinenizi yeniden başlatın.

  6. Yeniden başlattıktan sonra devam edin ve Docker’ı yükleyin.

En son sürümü yükleyin

Ön koşulları karşıladığınızdan emin olun, ancak bundan sonra aşağıdaki adımları izleyin.

Not: Üretim sistemleri için, Docker’ı yanlışlıkla güncellememeniz için belirli bir sürüm yüklemeniz önerilir. Üretim sistemleri için yükseltmeleri dikkatli bir şekilde planlamalısınız.

  1. Ubuntu kurulumunuzda “sudo” ayrıcalıklarına sahip bir kullanıcı olarak oturum açın. (Muhtemelen “sudo -su” çalıştırıyor).

  2. “sudo apt-get update” komutunu çalıştırarak APT paket dizininizi güncelleyin.

  3. Docker Community Edition’ı “sudo apt-get install docker-ce” komutuyla kurun.

  4. “docker” arka planını “sudo service docker start” komutuyla başlatın.

  5. Hello-world imajını çalıştırarak “docker"ın doğru kurulduğunu doğrulayın.

     $ sudo docker run hello-world

Bu komut, bir test görüntüsü indirir ve bir kapsayıcıda çalıştırır. Konteyner çalıştığında, bir bilgi mesajı yazdırır ve çıkar.

Docker’ı root olmayan bir kullanıcı olarak yönetin

Docker komutunu kullandığınızda sudo kullanmak istemiyorsanız, docker adında bir Unix grubu oluşturun ve buna kullanıcılar ekleyin. Docker arka plan programı başladığında, Unix soketinin sahipliğini docker grubu tarafından okunabilir/yazılabilir hale getirir.

docker grubunu oluşturmak ve kullanıcınızı eklemek için:

  1. Ubuntu’da “sudo” ayrıcalıklarına sahip bir kullanıcı olarak oturum açın.

  2. ‘sudo groupadd docker’ komutuyla ‘docker’ grubunu oluşturun.

  3. Kullanıcınızı docker grubuna ekleyin.

     $ sudo usermod -aG docker $USER

  4. Grup üyeliğinizin yeniden değerlendirilmesi için çıkış yapın ve tekrar giriş yapın.

  5. “sudo” izni olmadan “docker” komutlarını kullanabileceğinizi doğrulayın.

     $ docker run hello-world

Bu başarısız olursa, bir hata görürsünüz:

     Cannot connect to the Docker daemon. Is 'docker daemon' running on this host?

‘DOCKER_HOST’ ortam değişkeninin kabuğunuz için ayarlanıp ayarlanmadığını kontrol edin.

    $ env | grep DOCKER_HOST

Ayarlanmışsa, yukarıdaki komut bir sonuç döndürür. Eğer öyleyse, ayarını kaldırın.

    $ unset DOCKER_HOST

‘DOCKER_HOST’ değişkeninin hatalı ayarlanmasını önlemek için ortamınızı ‘~/.bashrc’ veya ‘~/.profile’ gibi dosyalarda düzenlemeniz gerekebilir.

Docker’ı Ubuntu’ya Yükleme #

Gereksinimler: Docker, çekirdeği en az 3.10 sürümü olan herhangi bir Linux’a yüklenebilir. Docker, Ubuntu Linux’un aşağıdaki 64 bit sürümlerinde desteklenir:

*Ubuntu Xenial 16.04 (LTS)

  • Ücretsiz Wily 15.10
  • Ubuntu Güvenilir 14.04 (LTS)
  • Ubuntu Kesin 12.04 (LTS)

Kolay kurulum

Not: Docker’ı varsayılan Ubuntu deposundan yüklemek, Docker’ın eski bir sürümünü yükleyecektir.

Docker deposunu kullanarak Docker’ın en son sürümünü yüklemek için, Docker tarafından sağlanan kurulum komut dosyasını alıp çalıştırmak için ‘curl’ kullanın:

$ curl -sSL https://get.docker.com/ | sh

Alternatif olarak, Docker’ı kurmak için “wget” kullanılabilir:

$ wget -qO- https://get.docker.com/ | sh

Docker şimdi yüklenecek.

Manuel Kurulum

Ancak, kurulum komut dosyasını çalıştırmak bir seçenek değilse, Docker’ın en son sürümünü resmi depodan manuel olarak yüklemek için aşağıdaki talimatlar kullanılabilir.

$ sudo apt-get update
$ sudo apt-get install apt-transport-https ca-certificates

GPG anahtarını ekleyin:

$ sudo apt-key adv --keyserver hkp://p80.pool.sks-keyservers.net:80 \
  --recv-keys 58118E89F3A912897C070ADBF76221572C52609D

Ardından, favori düzenleyicinizde /etc/apt/sources.list.d/docker.list dosyasını açın. Dosya yoksa, oluşturun. Mevcut girişleri kaldırın. Ardından, sürümünüze bağlı olarak aşağıdaki satırı ekleyin:

  • Ubuntu Kesin 12.04 (LTS):

    deb https://apt.dockerproject.org/repo ubuntu-precise main

  • Ubuntu Trusty 14.04 (LTS)

    deb https://apt.dockerproject.org/repo ubuntu-trusty main

  • Ücretsiz Wily 15.10

    deb https://apt.dockerproject.org/repo ubuntu-wily main

-Ubuntu Xenial 16.04 (LTS)

`deb https://apt.dockerproject.org/repo ubuntu-xenial main`

Dosyayı kaydedin ve çıkın, ardından paket dizininizi güncelleyin, yüklü Docker sürümlerini kaldırın ve “apt” öğesinin doğru depodan çekildiğini doğrulayın:

$ sudo apt-get update
$ sudo apt-get purge lxc-docker
$ sudo apt-cache policy docker-engine

Ubuntu sürümünüze bağlı olarak bazı ön koşullar gerekli olabilir:

  • Ubuntu Xenial 16.04 (LTS), Ubuntu Wily 15.10, Ubuntu Trusty 14.04 (LTS)

    sudo apt-get update && sudo apt-get install linux-image-extra-$(uname -r)

  • Ubuntu Kesin 12.04 (LTS)

    This version of Ubuntu requires kernel version 3.13. You may need to install additional packages depending on your environment:

    linux-image-generic-lts-trusty

    Generic Linux kernel image. This kernel has AUFS built in. This is required to run Docker.

    linux-headers-generic-lts-trusty

    Allows packages such as ZFS and VirtualBox guest additions which depend on them. If you didn’t install the headers for your existing kernel, then you can skip these headers for the trusty kernel. If you’re unsure, you should include this package for safety.

    xserver-xorg-lts-trusty

    libgl1-mesa-glx-lts-trusty

    These two packages are optional in non-graphical environments without Unity/Xorg. Required when running Docker on machine with a graphical environment.

    To learn more about the reasons for these packages, read the installation instructions for backported kernels, specifically the LTS Enablement Stack — refer to note 5 under each version.

    Install the required packages then reboot the host:

    $ sudo apt-get install linux-image-generic-lts-trusty

    $ sudo reboot

Son olarak, “apt” paket dizinini güncelleyin ve Docker’ı kurun:

$ sudo apt-get update
$ sudo apt-get install docker-engine

Daemon’u başlatın:

$ sudo service docker start

Şimdi bir test görüntüsü başlatarak docker’ın düzgün çalıştığını doğrulayın:

$ sudo docker run hello-world

Bu komut, kurulumun başarılı olduğunu doğrulayan bir karşılama mesajı yazdırmalıdır.

Google Cloud’da bir liman işçisi kapsayıcısı oluşturun #

Docker daemon (motor) kullanmadan, bulut sağlayıcıları kullanarak docker kullanabilirsiniz. Bu örnekte, hesabınıza bağlı bir “gcloud” (Google Cloud util) olmalıdır.

docker-machine create --driver google --google-project `your-project-name` google-machine-type f1-large fm02

Bu örnek, Google Cloud konsolunuzda yeni bir örnek oluşturacaktır. ‘f1-large’ makine zamanını kullanma

Docker’ı Ubuntu’ya yükleyin #

Docker, Ubuntu Linux’un aşağıdaki 64-bit sürümlerinde desteklenir:

*Ubuntu Xenial 16.04 (LTS)

  • Ücretsiz Wily 15.10
  • Ubuntu Güvenilir 14.04 (LTS)
  • Ubuntu Kesin 12.04 (LTS)

Birkaç not:

Aşağıdaki talimatlar, yalnızca Docker paketleri kullanılarak kurulumu içerir ve bu, Docker‘ın en son resmi sürümünün alınmasını sağlar. Yalnızca “Ubuntu tarafından yönetilen” paketleri kullanarak yüklemeniz gerekiyorsa, Ubuntu belgelerine bakın (Belirgin nedenlerle başka türlü önerilmez).

Ubuntu Utopic 14.10 ve 15.04, Docker’ın APT deposunda bulunur ancak bilinen güvenlik sorunları nedeniyle artık resmi olarak desteklenmemektedir.

Ön koşullar

  • Docker yalnızca 64 bit Linux kurulumunda çalışır.
  • Docker, Linux çekirdeği sürüm 3.10 veya üzerini gerektirir (sürüm 3.13 veya üzeri gerektiren Ubuntu Precise 12.04 hariç). 3.10’dan eski çekirdekler, Docker kapsayıcılarını çalıştırmak için gereken bazı özelliklerden yoksundur ve belirli koşullar altında veri kaybına ve sıklıkla paniğe neden olan bilinen hataları içerir. ‘uname -r’ komutuyla mevcut çekirdek sürümünü kontrol edin. ‘Ubuntu Precise (12.04 LTS)’ çekirdeğinizi daha aşağı kaydırarak güncellemeniz gerekiyorsa bu gönderiyi kontrol edin. Buna WikiHow bakın diğer Ubuntu kurulumları için en son sürümü edinmek için gönderin.

APT kaynaklarını güncelleyin

Docker deposundaki paketlere erişmek için bunun yapılması gerekir.

  1. Makinenizde “sudo” veya “root” ayrıcalıklarına sahip bir kullanıcı olarak oturum açın.

  2. Bir terminal penceresi açın.

  3. Paket bilgilerini güncelleyin, APT’nin https yöntemiyle çalıştığından ve CA sertifikalarının kurulu olduğundan emin olun.

    $ sudo apt-get update $ sudo apt-get install apt-transport-https ca-certificates

  4. Yeni “GPG” anahtarını ekleyin. Bu komut, “58118E89F3A912897C070ADBF76221572C52609D” kimliğine sahip anahtarı “hkp://ha.pool.sks-keyservers.net:80” anahtar sunucusundan indirir ve “adv anahtar zincirine” ekler. Daha fazla bilgi için “man apt-key” çıktısına bakın.

     $ sudo apt-key adv \
        --keyserver hkp://ha.pool.sks-keyservers.net:80 \
        --recv-keys 58118E89F3A912897C070ADBF76221572C52609D

  5. Aşağıdaki tabloda Ubuntu sürümünüze karşılık gelen girişi bulun. Bu, APT’nin Docker paketlerini nerede arayacağını belirler. Mümkün olduğunda, Ubuntu’nun uzun vadeli bir destek (LTS) sürümünü çalıştırın.

Ubuntu Sürümü Depo
Hassas 12.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-precise main
Güvenilir 14.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-güvenilir ana
Wily 15.10 deb https://apt.dockerproject.org/repo ubuntu-wily main
Xenial 16.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-xenial main

Not: Docker, tüm mimariler için paket sağlamaz. İkili yapılar her gece oluşturulur ve bunları “https://master.dockerproject.org” adresinden indirebilirsiniz. Docker’ı çok mimarili bir sisteme kurmak için, girişe bir [arch=...] yan tümcesi ekleyin. Ayrıntılar için Debian Multiarch wiki‘ye bakın.

  1. <REPO> yer tutucusu yerine işletim sisteminizin girişini değiştirerek aşağıdaki komutu çalıştırın.

    $ echo “” | sudo tee /etc/apt/sources.list.d/docker.list

  2. “Sudo apt-get update” komutunu çalıştırarak “APT” paket dizinini güncelleyin.

  3. “APT"nin doğru depodan çekildiğini doğrulayın.

Aşağıdaki komutu çalıştırdığınızda, yüklemeniz için kullanılabilen her Docker sürümü için bir girdi döndürülür. Her girişin https://apt.dockerproject.org/repo/ URL’si olmalıdır. Şu anda yüklü olan sürüm *** ile işaretlenmiştir.Aşağıdaki örneğin çıktısına bakın.

$ apt-cache policy docker-engine

  docker-engine:
    Installed: 1.12.2-0~trusty
    Candidate: 1.12.2-0~trusty
    Version table:
   *** 1.12.2-0~trusty 0
          500 https://apt.dockerproject.org/repo/ ubuntu-trusty/main amd64 Packages
          100 /var/lib/dpkg/status
       1.12.1-0~trusty 0
          500 https://apt.dockerproject.org/repo/ ubuntu-trusty/main amd64 Packages
       1.12.0-0~trusty 0
          500 https://apt.dockerproject.org/repo/ ubuntu-trusty/main amd64 Packages

Şu andan itibaren ‘apt-get upgrade’i çalıştırdığınızda, ‘APT’ yeni depodan çeker.

Ubuntu Sürümüne göre Önkoşullar

Ubuntu Trusty (14.04), Wily (15.10) ve Xenial (16.04) için, “aufs” depolama sürücüsünü kullanmanıza izin veren “linux-image-extra-*” çekirdek paketlerini kurun.

linux-image-extra-* paketlerini kurmak için:

  1. Ubuntu sunucunuzda bir terminal açın.

  2. Paket yöneticinizi sudo apt-get update komutuyla güncelleyin.

  3. Önerilen paketleri kurun.

    $ sudo apt-get install linux-image-extra-$(uname -r) linux-image-extra-virtual

  4. Docker kurulumuna geçin

Ubuntu Precise (12.04 LTS) için Docker, 3.13 çekirdek sürümünü gerektirir. Çekirdek sürümünüz 3.13’ten daha eskiyse, yükseltmeniz gerekir. Ortamınız için hangi paketlerin gerekli olduğunu görmek için bu tabloya bakın:

Paket Açıklama
linux-image-generic-lts-trusty Genel Linux çekirdek görüntüsü. Bu çekirdeğin yerleşik ‘AUFS’si vardır. Bu, Docker’ı çalıştırmak için gereklidir.
linux-headers-generic-lts-trusty Bunlara bağlı olan “ZFS” ve “VirtualBox konuk eklemeleri” gibi paketlere izin verir. Mevcut çekirdeğiniz için başlıkları yüklemediyseniz, “güvenilir” çekirdek için bu başlıkları atlayabilirsiniz. Emin değilseniz, güvenlik için bu paketi eklemelisiniz.
`xserver-xorg-lts-güvenilir’ Unity/Xorg olmadan grafiksel olmayan ortamlarda isteğe bağlıdır. Gerekli Docker’ı bir grafik ortamla makinede çalıştırırken.
`ligbl1-mesa-glx-lts-güvenilir’ Bu paketlerin nedenleri hakkında daha fazla bilgi edinmek için, özellikle LTS Etkinleştirme Yığını olmak üzere, desteklenen çekirdeklerin kurulum talimatlarını okuyun. Her versiyonun altındaki not 5’e bakın.

Çekirdeğinizi yükseltmek ve ek paketleri kurmak için aşağıdakileri yapın:

  1. Ubuntu sunucunuzda bir terminal açın.

  2. Paket yöneticinizi sudo apt-get update komutuyla güncelleyin.

  3. Gereken ve isteğe bağlı paketleri kurun.

    $ sudo apt-get install linux-image-generic-lts-trusty

  4. Kurmanız gereken diğer paketler için bu adımı tekrarlayın.

  5. Güncellenmiş çekirdeği kullanmak için “sudo yeniden başlatma” komutunu kullanarak ana makinenizi yeniden başlatın.

  6. Yeniden başlattıktan sonra devam edin ve Docker’ı yükleyin.

En son sürümü yükleyin

Ön koşulları karşıladığınızdan emin olun, ancak bundan sonra aşağıdaki adımları izleyin.

Not: Üretim sistemleri için, Docker’ı yanlışlıkla güncellememeniz için belirli bir sürüm yüklemeniz önerilir. Üretim sistemleri için yükseltmeleri dikkatli bir şekilde planlamalısınız.

  1. Ubuntu kurulumunuzda “sudo” ayrıcalıklarına sahip bir kullanıcı olarak oturum açın. (Muhtemelen “sudo -su” çalıştırıyor).

  2. “sudo apt-get update” komutunu çalıştırarak APT paket dizininizi güncelleyin.

  3. Docker’ı “sudo apt-get install docker-engine” komutuyla kurun.

  4. “docker” arka planını “sudo service docker start” komutuyla başlatın.

  5. Hello-world imajını çalıştırarak “docker"ın doğru kurulduğunu doğrulayın.

     $ sudo docker run hello-world

Bu komut, bir test görüntüsü indirir ve bir kapsayıcıda çalıştırır. Konteyner çalıştığında, bir bilgi mesajı yazdırır ve çıkar.

Docker’ı root olmayan bir kullanıcı olarak yönetin

Docker komutunu kullandığınızda sudo kullanmak istemiyorsanız, docker adında bir Unix grubu oluşturun ve buna kullanıcılar ekleyin. Docker arka plan programı başladığında, Unix soketinin sahipliğini docker grubu tarafından okunabilir/yazılabilir hale getirir.

docker grubunu oluşturmak ve kullanıcınızı eklemek için:

  1. Ubuntu’da “sudo” ayrıcalıklarına sahip bir kullanıcı olarak oturum açın.

  2. ‘sudo groupadd docker’ komutuyla ‘docker’ grubunu oluşturun.

  3. Kullanıcınızı docker grubuna ekleyin.

     $ sudo usermod -aG docker $USER

  4. Grup üyeliğinizin yeniden değerlendirilmesi için çıkış yapın ve tekrar giriş yapın.

  5. “sudo” izni olmadan “docker” komutlarını kullanabileceğinizi doğrulayın.

     $ docker run hello-world

Bu başarısız olursa, bir hata görürsünüz:

     Cannot connect to the Docker daemon. Is 'docker daemon' running on this host?

‘DOCKER_HOST’ ortam değişkeninin kabuğunuz için ayarlanıp ayarlanmadığını kontrol edin.

    $ env | grep DOCKER_HOST

Ayarlanmışsa, yukarıdaki komut bir sonuç döndürür. Eğer öyleyse, ayarını kaldırın.

    $ unset DOCKER_HOST

‘DOCKER_HOST’ değişkeninin hatalı ayarlanmasını önlemek için ortamınızı ‘~/.bashrc’ veya ‘~/.profile’ gibi dosyalarda düzenlemeniz gerekebilir.

CentOS’ta Docker-ce VEYA Docker-ee’yi Yükleme #

Docker aşağıdaki sürümleri duyurdu:

-Docker-ee (Enterprise Edition), Docker-ce(Community Edition) ve Docker (Ticari Destek) ile birlikte

Bu belge, CentOS’ta Docker-ee ve Docker-ce sürümünün kurulum adımlarında size yardımcı olacaktır.

Docker-ce Kurulumu

Docker-ce sürümünü yükleme adımları aşağıdadır

  1. yum-config-manager yardımcı programını sağlayan yum-utils’i kurun:

    $ sudo yum install -y yum-utils

  2. Kararlı depoyu kurmak için aşağıdaki komutu kullanın:

$ sudo yum-config-manager \
    --add-repo \
    https://download.docker.com/linux/centos/docker-ce.repo

  1. İsteğe bağlı: Kenar deposunu etkinleştirin. Bu depo, yukarıdaki docker.repo dosyasına dahil edilmiştir ancak varsayılan olarak devre dışıdır. Kararlı deponun yanında etkinleştirebilirsiniz.

     $ sudo yum-config-manager --enable docker-ce-edge
  • ‘–disable’ bayrağıyla ‘yum-config-manager’ komutunu çalıştırarak kenar deposunu devre dışı bırakabilirsiniz. Yeniden etkinleştirmek için “–enable” bayrağını kullanın. Aşağıdaki komut, kenar deposunu devre dışı bırakır. 
     $ sudo yum-config-manager --disable docker-ce-edge

  1. yum paket dizinini güncelleyin.

     $ sudo yum makecache fast

  2. Aşağıdaki komutu kullanarak docker-ce’yi kurun:

     $ sudo yum install docker-ce-17.03.0.ce

  3. Docker-ce parmak izini onaylayın

    060A 61C5 1B55 8A7F 742B 77AA C52F EB6B 621E 9F35

    If you want to install some other version of docker-ce you can use following command:

    $ sudo yum install docker-ce-VERSION

“SÜRÜM” numarasını belirtin

  1. Her şey yolunda giderse, docker-ce artık sisteminize yüklenmiştir, başlatmak için aşağıdaki komutu kullanın:

     $ sudo systemctl start docker

  2. Docker kurulumunuzu test edin:

     $ sudo docker run hello-world

    you should get following message:

     Hello from Docker! 
 This message shows that your installation appears to be working correctly.

-Docker-ee (Enterprise Edition) Kurulumu

Enterprise Edition (EE) için, <DOCKER-EE-URL>’nizi almak için kaydolmanız gerekir.

  1. Kaydolmak için https://cloud.docker.com/ adresine gidin. Bilgilerinizi girin ve e-posta kimliğinizi onaylayın. Onaylandıktan sonra size bir <DOCKER-EE-URL> verilir ve bunu kuruluma tıkladıktan sonra kontrol panelinizde görebilirsiniz.

  2. /etc/yum.repos.d/ dizinindeki mevcut Docker depolarını kaldırın

  3. Docker EE veri havuzu URL’nizi /etc/yum/vars/ içindeki bir yum değişkeninde saklayın. <DOCKER-EE-URL> ilk adımda not ettiğiniz URL ile.

     $ sudo sh -c 'echo "<DOCKER-EE-URL>" > /etc/yum/vars/dockerurl'

  4. yum-config-manager yardımcı programını sağlayan yum-utils’i yükleyin:

     $ sudo yum install -y yum-utils

  5. Kararlı depoyu eklemek için aşağıdaki komutu kullanın:

     $ sudo yum-config-manager \
 --add-repo \
 <DOCKER-EE-URL>/docker-ee.repo

  6. yum paket dizinini güncelleyin.

     $ sudo yum makecache fast

  7. Docker-ee’yi kurun

     sudo yum install docker-ee

  8. Docker-ee’yi aşağıdaki komutu kullanarak başlatabilirsiniz:

    $ sudo systemctl start docker
