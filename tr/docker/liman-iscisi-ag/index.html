Kapsayıcının ana bilgisayar ipi nasıl bulunur #

Örneğin, içinde çalışan web sunucusuna bağlanabilmeniz için, ana bilgisayarda çalışan kapsayıcının IP adresini bulmanız gerekir.

“docker-machine”, MacOSX ve Windows’ta kullanılan şeydir.

İlk olarak, makinelerinizi listeleyin:

$ docker-machine ls

NAME      ACTIVE   DRIVER       STATE     URL                         SWARM
default   *        virtualbox   Running   tcp://192.168.99.100:2376

Ardından makinelerden birini seçin (varsayılana varsayılan denir) ve:

$ docker-machine ip default

192.168.99.100

Docker ağı oluşturma #

docker network create app-backend

Bu komut, “appBackend” adında basit bir köprülü ağ oluşturacaktır. Varsayılan olarak bu ağa hiçbir kapsayıcı eklenmez.

Listeleme Ağları #

docker network ls

Bu komut, yerel Docker ana bilgisayarında oluşturulmuş tüm ağları listeler. Varsayılan köprü “köprü” ağını, ana bilgisayar “ana bilgisayar” ağını ve boş “boş” ağını içerir. Varsayılan olarak tüm kapsayıcılar, varsayılan köprü ‘köprü’ ağına bağlıdır.

Ağa kapsayıcı ekle #

docker network connect app-backend myAwesomeApp-1

Bu komut, “myAwesomeApp-1” kapsayıcısını “app-backend” ağına ekler. Kullanıcı tanımlı bir ağa bir kapsayıcı eklediğinizde, yerleşik DNS çözümleyicisi (tam özellikli bir DNS sunucusu değildir ve dışa aktarılamaz), ağdaki her bir kapsayıcının aynı ağdaki diğer kapsayıcıyı çözmesine olanak tanır. Bu basit DNS çözümleyici, varsayılan köprü “köprü” ağında mevcut değildir.

Kapsayıcıyı ağdan ayırın #

docker network disconnect app-backend myAwesomeApp-1

Bu komut, “myAwesomeApp-1” kapsayıcısını “app-backend” ağından ayırır. Kapsayıcı artık bağlantısının kesildiği ağdaki diğer kapsayıcılarla iletişim kuramayacak veya ayrıldığı ağdaki diğer kapsayıcıları aramak için yerleşik DNS çözümleyicisini kullanamayacak.

Docker ağını kaldırın #

docker network rm app-backend

Bu komut, kullanıcı tanımlı “app-backend” ağını Docker ana bilgisayarından kaldırır. Başka bir ağ üzerinden bağlanmayan ağdaki tüm kapsayıcılar, diğer kapsayıcılarla iletişimi kaybeder. Varsayılan köprü “köprü” ağını, “ana bilgisayar” ana bilgisayar ağını veya “boş” boş ağı kaldırmak mümkün değildir.

Docker ağını inceleyin #

docker network inspect app-backend

Bu komut, “app-backend” ağıyla ilgili ayrıntıları verir.

Bu komutun çıktısı şuna benzemelidir:

[

        "Name": "foo",
        "Id": "a0349d78c8fd7c16f5940bdbaf1adec8d8399b8309b2e8a969bd4e3226a6fc58",

                }
            ]

    }
