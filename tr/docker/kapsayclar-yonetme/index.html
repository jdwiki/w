Sözdizimi #

  • docker rm [SEÇENEKLER] KONTEYNER [KONTEYNER…]
  • liman işçisi [SEÇENEKLER] KONTEYNER eki
  • docker exec [SEÇENEKLER] KONTEYNER KOMUTU [ARG…]
  • liman işçisi ps [SEÇENEKLER]
  • liman işçisi günlükleri [SEÇENEKLER] KONTEYNER
  • liman işçisi [SEÇENEKLER] KONTEYNER|GÖRÜNTÜ [KONTEYNER|GÖRÜNTÜ…]
  • Yukarıdaki örneklerde, konteyner docker komutunun bir parametresi olduğunda, “” veya “container id” veya “<CONTAINER_NAME>” olarak belirtilir. Tüm bu yerlerde, bir kapsayıcı belirtmek için bir kapsayıcı adı veya kapsayıcı kimliği iletebilirsiniz.

Listeleme kapsayıcıları #

$ docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS                     NAMES
2bc9b1988080        redis               "docker-entrypoint.sh"   2 weeks ago         Up 2 hours          0.0.0.0:6379->6379/tcp    elephant-redis
817879be2230        postgres            "/docker-entrypoint.s"   2 weeks ago         Up 2 hours          0.0.0.0:65432->5432/tcp   pt-postgres

“docker ps” kendi başına yalnızca şu anda çalışan kapsayıcıları yazdırır. Tüm kapsayıcıları (durdurulmuş olanlar dahil) görüntülemek için “-a” bayrağını kullanın:

$ docker ps -a
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS                    PORTS                     NAMES
9cc69f11a0f7        docker/whalesay     "ls /"                   26 hours ago        Exited (0) 26 hours ago                             berserk_wozniak
2bc9b1988080        redis               "docker-entrypoint.sh"   2 weeks ago         Up 2 hours                0.0.0.0:6379->6379/tcp    elephant-redis
817879be2230        postgres            "/docker-entrypoint.s"   2 weeks ago         Up 2 hours                0.0.0.0:65432->5432/tcp   pt-postgres

Belirli bir duruma sahip kapsayıcıları listelemek için sonuçları filtrelemek için -f komut satırı seçeneğini kullanın. Çıkış yapan tüm kapsayıcıların listelenmesine ilişkin bir örnek:

$ docker ps -a -f status=exited
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS                    PORTS                     NAMES
9cc69f11a0f7        docker/whalesay     "ls /"                   26 hours ago        Exited (0) 26 hours ago

-q anahtarı ile sadece Konteyner Kimliklerini listelemek de mümkündür. Bu, diğer Unix yardımcı programlarıyla (“grep” ve “awk” gibi) sonuç üzerinde çalışmayı çok kolaylaştırır:

$ docker ps -aq
9cc69f11a0f7
2bc9b1988080
817879be2230

docker run --name mycontainer1 ile bir konteyner başlatırken rastgele bir isim değil belirli bir isim verirsiniz (mostalgic_stallman gibi mood_ Famous biçiminde) ve böyle bir komutla onları bulmak kolay olabilir

liman işçisi ps -f isim=container1

Kapsayıcıları kaldırın, silin ve temizleyin #

“docker rm” aşağıdaki gibi belirli bir kapsayıcıyı kaldırmak için kullanılabilir:

docker rm <container name or id>

Tüm kapsayıcıları kaldırmak için bu ifadeyi kullanabilirsiniz:

docker rm $(docker ps -qa)

Varsayılan olarak docker, çalışan bir kapsayıcıyı silmez. Çalışan herhangi bir kapsayıcı bir uyarı mesajı üretecek ve silinmeyecektir. Diğer tüm kapsayıcılar silinecek.

Alternatif olarak “xargs” kullanabilirsiniz:

docker ps -aq -f status=exited | xargs -r docker rm

“docker ps -aq -f status=exited”, durumu “Çıkış” olan kapsayıcıların kapsayıcı kimliklerinin bir listesini döndürür.

Uyarı: Yukarıdaki örneklerin tümü yalnızca ‘durdurulmuş’ kapsayıcıları kaldıracaktır.

Bir kapsayıcıyı kaldırmak için durdurulmuş olsun ya da olmasın, -f kuvvet bayrağını kullanabilirsiniz:

docker rm -f <container name or id>

Durumdan bağımsız olarak tüm kapsayıcıları kaldırmak için:

docker rm -f $(docker ps -qa)

Yalnızca “ölü” durumundaki kapları kaldırmak istiyorsanız:

docker rm $(docker ps --all -q -f status=dead)

Yalnızca “çıkış” durumundaki kapsayıcıları kaldırmak istiyorsanız:

docker rm $(docker ps --all -q -f status=exited)

Bunların tümü, [konteynerler [1] listelenirken kullanılan filtrelerin permütasyonlarıdır.

[Sürüm 1.3][2]‘den sonra alan kullanan hem istenmeyen kapsayıcıları hem de sarkan görüntüleri kaldırmak için aşağıdakileri kullanın (Unix aracı “df"ye benzer):

$ docker system df

Kullanılmayan tüm verileri kaldırmak için:

$ docker system prune

[1]: https://www.wikiod.com/tr/docker/kapsayclar-yonetme#Kapsayıcıları listeleme [2]: https://blog.docker.com/2017/01/whats-new-in-docker-1-13/

Referans kapsayıcıları #

Bir kapsayıcı adını alan Docker komutları üç farklı biçimi kabul eder:

Tip Örnek
Tam UUID ‘9cc69f11a0f76073e87f25cb6eaf0e079fbfbd1bc47c063bcd25ed3722a8cc4a’
Kısa UUID ‘9cc69f11a0f7’
İsim ‘berserk_wozniak’

Sisteminizdeki kapsayıcılar için bu değerleri görüntülemek için docker ps kullanın.

UUID, Docker tarafından oluşturulur ve değiştirilemez. docker run --name <verilen ad> <image> başlattığınızda kapsayıcıya bir ad verebilirsiniz. Kapsayıcıyı başlatırken bir ad belirtmezseniz, Docker kapsayıcı için rastgele bir ad oluşturur.

NOT: UUID’nin (veya bir ‘kısa’ UUID’nin) değeri, verilen değer bir kapsayıcıya özgü olduğu sürece herhangi bir uzunlukta olabilir

Daemon olarak çalışan bir örneğe bağlanın #

Bunu başarmanın iki yolu vardır, bunlardan ilki ve en bilineni şudur:

docker attach --sig-proxy=false <container>

Bu, bash’inizi tam anlamıyla kapsayıcı bash’a ekler, yani çalışan bir betiğiniz varsa, sonucu görürsünüz.

Ayırmak için şunu yazmanız yeterlidir: Ctl-P Ctl-Q

Ancak daha kolay bir yola ihtiyacınız varsa ve yeni bash örnekleri oluşturabilmek için aşağıdaki komutu çalıştırmanız yeterlidir:

docker exec -it <container> bash

Belirli bir kapsayıcı bulma #

docker ps --filter name=myapp_1

Dosyaları kapsayıcılardan/kapsayıcılara kopyalama #

kapsayıcıdan ana bilgisayara

docker cp CONTAINER_NAME:PATH_IN_CONTAINER PATH_IN_HOST

ana bilgisayardan konteynere

docker cp PATH_IN_HOST CONTAINER_NAME:PATH_IN_CONTAINER

jess/transmission kullanırsam

https://hub.docker.com/r/jess/transmission/builds/bsn7eqxrkzrhxazcuytbmzp/

, kapsayıcıdaki dosyalar /transmission/download konumunda

ve ana bilgisayardaki mevcut dizinim /home/$USER/abc, sonra

docker cp iletim_kimliği_veya_adı:/transmission/download .

Dosyaları kopyalayacağım

/home/$USER/abc/transmission/download

docker cp kullanarak sadece bir dosya kopyalayamazsınız, dizin ağacını ve dosyaları kopyalarsınız.

Kapsayıcıları başlatma ve durdurma #

Çalışan bir kapsayıcıyı durdurmak için:

docker stop <container> [<container>...]

Bu, kapsayıcıdaki ana süreci bir SIGTERM, ardından ödemesiz süre içinde durmazsa bir SIGKILL gönderir. Her konteynerin adı durdukça yazdırılır.

Durdurulan bir kapsayıcıyı başlatmak için:

docker start <container> [<container>...]

Bu, arka planda geçirilen her kapsayıcıyı başlatacaktır; her kapsayıcının adı başlarken yazdırılır. Konteyneri ön planda başlatmak için -a (--attach) bayrağını iletin.

Özel biçimli kapsayıcıları listeleyin #

docker ps --format 'table {{.ID}}\t{{.Names}}\t{{.Status}}'

Kapsayıcı IP’sini bulun #

Kapsayıcınızın IP adresini öğrenmek için şunu kullanın:

docker inspect <container id> | grep IPAddress

veya liman işçisi incelemesini kullanın

docker inspect --format '{{ .NetworkSettings.IPAddress }}' ${CID}

Docker kapsayıcısını yeniden başlatma #

docker restart <container> [<container>...]

Seçenek –zaman : Kapsayıcıyı öldürmeden önce durmayı beklemek için saniyeler (varsayılan 10)

docker restart <container> --time 10

Komutu zaten var olan bir liman işçisi kapsayıcısında çalıştır #

docker exec -it <konteyner kimliği> /bin/bash

Bazı hızlı testler yapmak veya uygulamanın ne yaptığını görmek için zaten çalışan bir kapsayıcıda oturum açmak yaygındır. Genellikle, günlükler nedeniyle kötü kapsayıcı kullanım uygulamalarını belirtir ve değiştirilen dosyalar birimlere yerleştirilmelidir. Bu örnek, kapsayıcıda oturum açmamızı sağlar. Bu, /bin/bash’in kapta mevcut olduğunu varsayar, /bin/sh veya başka bir şey olabilir.

docker exec <container id> tar -czvf /tmp/backup.tgz /data
docker cp <container id>:/tmp/backup.tgz .

Bu örnek, bir tar içindeki veri dizininin içeriğini arşivler. Sonra docker cp ile onu geri alabilirsiniz.

Docker birimlerini kaldırın, silin ve temizleyin #

Docker birimleri, bir kapsayıcı durdurulduğunda otomatik olarak kaldırılmaz. Bir kapsayıcıyı durdurduğunuzda ilişkili birimleri kaldırmak için:

docker rm -v <container id or name>

‘-v’ bayrağı belirtilmemişse, birim diskte ‘sarkan birim’ olarak kalır. Tüm sarkan birimleri silmek için:

docker volume rm $(docker volume ls -qf dangling=true)

“docker volume ls -qf dangling=true” filtresi, etiketlenmemiş olanlar da dahil olmak üzere bir konteynere eklenmemiş liman işçisi birimleri adlarının bir listesini döndürür.

Alternatif olarak, “xargs” kullanabilirsiniz:

docker volume ls -f dangling=true -q | xargs --no-run-if-empty docker volume rm

Docker kapsayıcı dosya sistemlerini dışa ve içe aktarın #

Bir Docker konteynerinin dosya sistemi içeriğini bir tarball arşiv dosyasına kaydetmek mümkündür. Bu, örneğin bir veritabanı kapsayıcısında önemli değişiklikler varsa ve bu değişiklikleri başka bir yerde çoğaltmak mümkün değilse, kapsayıcı dosya sistemlerini farklı ana bilgisayarlara taşımak için bir tutamda yararlıdır. Lütfen unutmayın, bir kapsayıcının dosya sistemini dışa aktarmak ve taşımak yerine, bir “docker run” komutu veya “docker-compose.yml” dosyası kullanarak güncellenmiş bir görüntüden tamamen yeni bir kapsayıcı oluşturmanın tercih edildiğini unutmayın. Docker’ın gücünün bir kısmı, görüntü ve kapsayıcı oluşturma konusundaki bildirimsel tarzının denetlenebilirliği ve hesap verebilirliğidir. ‘Docker export’ ve ‘docker import’ kullanılarak, bir konteynerin dosya sistemi içinde yapılan değişikliklerin orijinal durumundan gizlenmesi nedeniyle bu güç azaltılır.

docker export -o redis.tar redis

Yukarıdaki komut boş bir görüntü oluşturacak ve ardından ‘redis’ kabının dosya sistemini bu boş görüntüye aktaracaktır. Tarball arşivinden içe aktarmak için şunu kullanın:

docker import ./redis.tar redis-imported:3.0.7

Bu komut, kapsayıcıların oluşturulabileceği “redis-imported:3.0.7” görüntüsünü oluşturacaktır. Ayrıca, içe aktarmada değişiklikler oluşturmak ve bir taahhüt mesajı ayarlamak da mümkündür:

docker import -c="ENV DEBUG true" -m="enable debug mode" ./redis.tar redis-changed

‘-c’ komut satırı seçeneğiyle kullanılabilen Dockerfile yönergeleri, ‘CMD’, ‘ENTRYPOINT’, ‘ENV’, ‘EXPOSE’, ‘ONBUILD’, ‘USER’, ‘VOLUME’, ‘WORKDIR’ şeklindedir.
