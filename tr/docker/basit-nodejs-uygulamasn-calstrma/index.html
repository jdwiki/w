Bir Konteyner içinde Temel Node.js uygulamasını çalıştırma #

Tartışacağım örnek, sisteminizde çalışan bir Docker kurulumuna ve Node.js ile nasıl çalışılacağına dair temel bir anlayışa sahip olduğunuzu varsayar. Docker ile nasıl çalışmanız gerektiğinin farkındaysanız, sisteminizde Node.js çerçevesinin kurulmasına gerek olmadığı açık olmalıdır, bunun yerine Docker’da bulunan “node” görüntüsünün “en son” sürümünü kullanıyor olacağız. Bu nedenle, gerekirse docker pull node komutuyla görüntüyü önceden indirebilirsiniz. (Komut, “düğüm” görüntüsünün en son sürümünü docker’dan otomatik olarak “çeker”.)

  1. Tüm çalışan uygulama dosyalarınızın bulunacağı bir dizin oluşturmaya devam edin. Bu dizinde, uygulamanızı ve bağımlılıkları açıklayan bir “package.json” dosyası oluşturun. “package.json” dosyanız şöyle görünmelidir:

     {
   "name": "docker_web_app",
   "version": "1.0.0",
   "description": "Node.js on Docker",
   "author": "First Last <first.last@example.com>",
   "main": "server.js",
   "scripts": {
     "start": "node server.js"
   },
   "dependencies": {
     "express": "^4.13.3"
   }
 }

  1. Node.js ile çalışmamız gerekirse, genellikle bir web uygulamasını tanımlayan bir “sunucu” dosyası oluştururuz. Bu durumda “Express.js” çerçevesini kullanırız (sürüm “4.13.3” sonrası). Temel bir “server.js” dosyası şuna benzer:

     var express = require('express');
 var PORT = 8080;
 var app = express();
 app.get('/', function (req, res) {
   res.send('Hello world\n');
 });

 app.listen(PORT);
 console.log('Running on http://localhost:' + PORT);

  2. Docker’a aşina olanlar için bir ‘Dockerfile’ ile karşılaşmışsınızdır. “Dockerfile”, uygulamanız için uyarlanmış özel bir görüntü oluşturmak için gereken tüm komutları içeren bir metin dosyasıdır.

Geçerli dizinde Dockerfile adlı boş bir metin dosyası oluşturun. Windows’ta bir tane oluşturma yöntemi basittir. Linux’ta, uygulamanız için gerekli tüm dosyaları içeren dizinde `dockerfile’ye dokunmak isteyebilirsiniz. Dockerfile’ı herhangi bir metin düzenleyiciyle açın ve aşağıdaki satırları ekleyin:

FROM node:latest
RUN mkdir -p /usr/src/my_first_app
WORKDIR /usr/src/my_first_app
COPY package.json /usr/src/my_first_app/
RUN npm install
COPY . /usr/src/my_first_app
EXPOSE 8080

  • FROM node:latest Docker daemon’a hangi imajdan oluşturmak istediğimizi bildirir. Bu durumda, Docker Hub‘da bulunan resmi Docker görüntüsü “düğüm"ünün “en son” sürümünü kullanırız.

  • Bu görüntünün içinde gerekli tüm dosyaları içeren bir çalışma dizini oluşturmaya devam ediyoruz ve arka plan programına bu dizini uygulamamız için istenen çalışma dizini olarak ayarlamasını söylüyoruz. Bunun için ekliyoruz

      RUN mkdir -p /usr/src/my_first_app
  WORKDIR /usr/src/my_first_app

  • Ardından, ilk önce ‘package.json’ dosyasını (bağımlılıklar dahil uygulama bilgilerini belirtir) görüntüdeki ‘/usr/src/my_first_app’ çalışma dizinine taşıyarak uygulama bağımlılıklarını yüklemeye devam ederiz. Biz bunu

      COPY package.json /usr/src/my_first_app/
  RUN npm install

  • Ardından COPY yazıyoruz. /usr/src/my_first_app tüm uygulama dosyalarını ve kaynak kodunu görüntüdeki çalışma dizinine eklemek için.

  • Daha sonra, uygulama ‘8080’ bağlantı noktasına bağlandığından, arka plan programına, ortaya çıkan konteynerin ‘8080’ bağlantı noktasını görünür hale getirmesi talimatını vermek için ‘EXPOSE’ yönergesini kullanırız (bir konteyner-ana bilgisayar eşlemesi yoluyla).

  • Son adımda, temel ’npm start’ komutunu çalıştırarak arka plan programına ’node server.js’ komutunu görüntü içinde çalıştırmasını söylüyoruz. Bunun için komutları argüman olarak alan CMD direktifini kullanıyoruz.

      CMD [ "npm", "start" ]

  1. Daha sonra Dockerfile ile aynı dizinde bir .dockerignore dosyası oluşturuyoruz. “node_modules” kopyamızın ve Node.js sistem kurulumumuz tarafından kullanılan günlüklerin Docker görüntüsüne kopyalanmasını önlemek için. .dockerignore dosyası aşağıdaki içeriğe sahip olmalıdır:

     node_modules
 npm-debug.log

  2. İmajınızı oluşturun #

Dockerfile’ı içeren dizine gidin ve Docker görüntüsünü oluşturmak için aşağıdaki komutu çalıştırın. “-t” bayrağı, daha sonra docker images komutunu kullanarak daha kolay bulmak için resminizi etiketlemenizi sağlar:

    $ docker build -t <your username>/node-web-app .

Resminiz artık Docker tarafından listelenecek. Aşağıdaki komutu kullanarak görüntüleri görüntüleyin:

$ docker images

REPOSITORY                      TAG        ID              CREATED
node                            latest     539c0211cd76    10 minutes ago
<your username>/node-web-app    latest     d64d3505b0d2    1 minute ago

  1. Resmi çalıştırma #

Artık uygulama içeriklerini, ’node’ temel imajını ve ‘Dockerfile’ı kullanarak oluşturduğumuz imajı çalıştırabiliriz. Şimdi yeni oluşturulan <kullanıcı adınız>/node-web-app resmimizi çalıştırmaya devam ediyoruz. “docker run” komutuna “-d” anahtarının sağlanması, kapsayıcıyı ayrılmış modda çalıştırır, böylece kapsayıcı arka planda çalışır. ‘-p’ bayrağı, genel bir bağlantı noktasını kapsayıcı içindeki özel bir bağlantı noktasına yönlendirir. Bu komutu kullanarak daha önce oluşturduğunuz görüntüyü çalıştırın:

$ docker run -p 49160:8080 -d <your username>/node-web-app

  1. Terminalinizde docker ps çalıştırarak uygulamanızın çıktısını yazdırın. Çıktı böyle bir şeye benzemelidir.

     CONTAINER ID        IMAGE                         COMMAND             CREATED             STATUS              PORTS                     NAMES
 7b701693b294      <your username>/node-web-app   "npm start"         20 minutes ago       Up 48 seconds       0.0.0.0:49160->8080/tcp   loving_goldstine

docker logs <CONTAINER ID> girerek uygulama çıktısını alın. Bu durumda docker günlükleri 7b701693b294 şeklindedir.

Çıktı: ‘http://localhost:8080’de çalışıyor

  1. ‘docker ps’ çıktısından, elde edilen port eşlemesi ‘0.0.0.0:49160->8080/tcp’ şeklindedir. Bu nedenle Docker, konteynerin içindeki ‘8080’ bağlantı noktasını ana makinedeki 49160 bağlantı noktasıyla eşleştirdi. Tarayıcıda artık localhost:49160 girebiliriz.

Uygulamamızı curl kullanarak da arayabiliriz:

$ curl -i localhost:49160

HTTP/1.1 200 OK
X-Powered-By: Express
Content-Type: text/html; charset=utf-8
Content-Length: 12
Date: Sun, 08 Jan 2017 14:00:12 GMT
Connection: keep-alive

Hello world
