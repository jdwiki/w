Konsolu bir liman işçisi 1.12 sürüsünde çalıştırın #

Bu, konsolu Docker 1.12’de yeni sürü moduyla bir liman işçisi sürüsünde kümelenmiş modda çalıştırmak için resmi konsolos liman işçisi görüntüsüne dayanır. Bu örnekte http://qnib.org/2016/08/11/consul-service/ temel alınmıştır. Kısaca fikir, birbiriyle konuşan iki liman işçisi sürüsü hizmeti kullanmaktır. Bu, bireysel konsolos konteynırlarının ips’lerini önceden bilememeniz sorununu çözer ve liman işçisi sürüsünün dns’lerine güvenmenizi sağlar.

Bu, en az üç düğüme sahip çalışan bir liman işçisi 1.12 küme kümeniz olduğunu varsayar.

Neler olduğunu görebilmek için docker arka plan programlarınızda bir günlük sürücüsü yapılandırmak isteyebilirsiniz. Bunun için syslog sürücüsünü kullandım: dockerd’da --log-driver=syslog seçeneğini ayarlayın.

İlk önce consul için bir yer paylaşımlı ağ oluşturun:

docker network create consul-net -d overlay

Şimdi kümeyi yalnızca 1 düğümle önyükleyin (varsayılan –replikas 1’dir):

docker service create --name consul-seed \
  -p 8301:8300 \
  --network consul-net \
  -e 'CONSUL_BIND_INTERFACE=eth0' \
  consul agent -server -bootstrap-expect=3  -retry-join=consul-seed:8301 -retry-join=consul-cluster:8300

Artık 1 düğümlü bir kümeniz olmalıdır. Şimdi ikinci hizmeti getirin:

docker service create --name consul-cluster \
  -p 8300:8300 \
  --network consul-net \
  --replicas 3 \
  -e 'CONSUL_BIND_INTERFACE=eth0' \
  consul agent -server -retry-join=consul-seed:8301 -retry-join=consul-cluster:8300

Artık dört düğümlü bir konsolos kümeniz olmalıdır. Bunu, herhangi bir liman işçisi kapsayıcısında çalıştırarak doğrulayabilirsiniz:

docker exec <containerid> consul members
