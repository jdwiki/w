Docker ile Iptable'lar

Bu sayfada

  • Sözdizimi
  • Parametreler
  • Docker kapsayıcılarına erişimi bir dizi IP ile sınırlayın
  • Docker arka plan programı başladığında kısıtlama erişimini yapılandırın
  • Bazı özel iptables kuralları

Bu konu, iptables kullanarak dış dünyadan docker kapsayıcılarınıza erişimi nasıl sınırlayacağınızla ilgilidir.

Sabırsız olanlar için örnekleri inceleyebilirsiniz. Diğerleri için, yeni kuralların nasıl oluşturulacağını anlamak için lütfen açıklama bölümünü okuyun.

Sözdizimi #

  • iptables -I DOCKER [KURAL …] [KABUL|DROP] // DOCKER tablosunun en üstüne bir kural eklemek için
  • iptables -D DOCKER [KURAL …] [KABUL|DROP] // DOCKER tablosundan bir kuralı kaldırmak için
  • ipset restore < /etc/ipfriends.conf // ipset ipfriends dosyanızı yeniden yapılandırmak için

Parametreler #

parametreler Ayrıntılar
ext_if Docker ana bilgisayarındaki harici arayüzünüz.
XXX.XXX.XXX.XXX Docker kapsayıcılarına erişimin verilmesi gereken belirli bir IP.
YYY.YYY.YYY.YYY Docker kapsayıcılarına erişim için başka bir IP verilmelidir.
iparkadaşlar Docker kapsayıcılarınıza erişmesine izin verilen IP’leri tanımlayan ipset adı.

Sorun #

Docker kapsayıcıları için iptables kurallarını yapılandırmak biraz zor. İlk başta, “klasik” güvenlik duvarı kurallarının hile yapması gerektiğini düşünürdünüz.

Örneğin, HTTPS aracılığıyla bazı kişisel web hizmetlerini açığa çıkarmak için bir nginx-proxy kapsayıcı + birkaç hizmet kapsayıcı yapılandırdığınızı varsayalım. O zaman bunun gibi bir kural, web hizmetlerinize yalnızca IP XXX.XXX.XXX.XXX için erişim vermelidir.

$ iptables -A INPUT -i eth0 -p tcp -s XXX.XXX.XXX.XXX -j ACCEPT
$ iptables -P INPUT DROP

İşe yaramaz, konteynerlerinize hala herkes erişebilir.

Gerçekten de, Docker kapsayıcıları ana bilgisayar hizmetleri değildir. Ana makinenizdeki sanal bir ağa güvenirler ve ana bilgisayar bu ağ için bir ağ geçidi görevi görür. Ve ağ geçitleriyle ilgili olarak, yönlendirilen trafik INPUT tablosu tarafından değil, daha önce gönderilen kuralı etkisiz hale getiren FORWARD tablosu tarafından işlenir.

Ama hepsi bu değil. Aslında, Docker arka plan programı, kapsayıcı ağ bağlantısıyla ilgili sihrini yapmaya başladığında birçok iptables kuralı oluşturur. Özellikle, trafiği FORWARD tablosundan bu yeni tabloya ileterek kapsayıcılarla ilgili kuralları işlemek için bir DOCKER tablosu oluşturulur.

$ iptables -L
Chain INPUT (policy ACCEPT)
target     prot opt source               destination

Chain FORWARD (policy DROP)
target     prot opt source               destination
DOCKER-ISOLATION  all  --  anywhere             anywhere
DOCKER     all  --  anywhere             anywhere
ACCEPT     all  --  anywhere             anywhere             ctstate RELATED,ESTABLISHED
ACCEPT     all  --  anywhere             anywhere
ACCEPT     all  --  anywhere             anywhere
DOCKER     all  --  anywhere             anywhere
ACCEPT     all  --  anywhere             anywhere             ctstate RELATED,ESTABLISHED
ACCEPT     all  --  anywhere             anywhere
ACCEPT     all  --  anywhere             anywhere

Chain OUTPUT (policy ACCEPT)
target     prot opt source               destination

Chain DOCKER (2 references)
target     prot opt source               destination
ACCEPT     tcp  --  anywhere             172.18.0.4           tcp dpt:https
ACCEPT     tcp  --  anywhere             172.18.0.4           tcp dpt:http

Chain DOCKER-ISOLATION (1 references)
target     prot opt source               destination
DROP       all  --  anywhere             anywhere
DROP       all  --  anywhere             anywhere
RETURN     all  --  anywhere             anywhere

Çözüm #

Resmi belgeleri kontrol ederseniz (https://docs.docker.com/v1.5/articles/networking/), Docker konteyner erişimini belirli bir IP ile sınırlamak için ilk çözüm verilir.

$ iptables -I DOCKER -i ext_if ! -s 8.8.8.8 -j DROP

Gerçekten de, DOCKER tablosunun en üstüne bir kural eklemek iyi bir fikirdir. Docker tarafından otomatik olarak yapılandırılan kurallara müdahale etmez ve basittir. Ama iki büyük eksiklik:

  • İlk olarak, bir yerine iki IP’den erişmeniz gerekirse? Burada sadece bir src IP’si kabul edilebilir, diğeri bunu engellemenin hiçbir yolu olmadan bırakılacaktır.
  • İkinci olarak, liman işçinizin İnternet’e erişmesi gerekiyorsa ne olur? Sadece sunucu 8.8.8.8 onlara yanıt verebileceğinden, pratik olarak hiçbir istek başarılı olmayacaktır.
  • Son olarak, başka mantıklar eklemek isterseniz ne olur? Örneğin, HTTP protokolünde hizmet veren web sunucunuza herhangi bir kullanıcıya erişim verin, ancak diğer her şeyi belirli IP ile sınırlayın.

İlk gözlem için ipset kullanabiliriz. Yukarıdaki kuralda tek bir IP’ye izin vermek yerine, önceden tanımlanmış ipset’ten tüm IP’lere izin veriyoruz. Bonus olarak ipset, iptable kuralını yeniden tanımlamaya gerek kalmadan güncellenebilir.

$ iptables -I DOCKER -i ext_if -m set ! --match-set my-ipset src -j DROP

İkinci gözlem için, bu güvenlik duvarları için kurallı bir sorundur: bir güvenlik duvarı aracılığıyla bir sunucuyla iletişim kurmanıza izin veriliyorsa, güvenlik duvarı sunucuyu isteğinize yanıt vermesi için yetkilendirmelidir. Bu, kurulmuş bir bağlantıyla ilgili paketleri yetkilendirerek yapılabilir. Docker mantığı için şunu verir:

$ iptables -I DOCKER -i ext_if -m state --state ESTABLISHED,RELATED -j ACCEPT

Son gözlem bir noktaya odaklanıyor: iptables kuralları esastır. Aslında, bazı bağlantıları KABUL ETMEK için ek mantık (KURULAN bağlantılarla ilgili olanlar dahil), ipset ile eşleşmeyen kalan tüm bağlantıları reddeden DROP kuralından önce DOCKER tablosunun en üstüne yerleştirilmelidir.

Tablonun en üstüne kurallar ekleyen iptable’ın -I seçeneğini kullandığımızdan, önceki iptables kuralları ters sırada eklenmelidir:

// Drop rule for non matching IPs
$ iptables -I DOCKER -i ext_if -m set ! --match-set my-ipset src -j DROP
// Then Accept rules for established connections
$ iptables -I DOCKER -i ext_if -m state --state ESTABLISHED,RELATED -j ACCEPT 
$ iptables -I DOCKER -i ext_if ... ACCEPT // Then 3rd custom accept rule
$ iptables -I DOCKER -i ext_if ... ACCEPT // Then 2nd custom accept rule
$ iptables -I DOCKER -i ext_if ... ACCEPT // Then 1st custom accept rule

Tüm bunları göz önünde bulundurarak, şimdi bu konfigürasyonu gösteren örnekleri inceleyebilirsiniz.

Docker kapsayıcılarına erişimi bir dizi IP ile sınırlayın #

Önce gerekirse ipset kurun. Nasıl yapılacağını öğrenmek için lütfen dağıtımınıza bakın. Örnek olarak, Debian benzeri dağıtımlar için komut buradadır.

$ apt-get update
$ apt-get install ipset

Ardından, Docker kapsayıcılarınıza erişim açmak istediğiniz IP’leri içeren bir ipset tanımlamak için bir yapılandırma dosyası oluşturun.

$ vi /etc/ipfriends.conf
# Recreate the ipset if needed, and flush all entries
create -exist ipfriends hash:ip family inet hashsize 1024 maxelem 65536
flush
# Give access to specific ips
add ipfriends XXX.XXX.XXX.XXX
add ipfriends YYY.YYY.YYY.YYY

Bu ipset’i yükleyin.

$ ipset restore < /etc/ipfriends.conf

Docker arka plan programınızın çalıştığından emin olun: aşağıdaki komutu girdikten sonra hiçbir hata gösterilmemelidir.

$ docker ps

iptables kurallarınızı eklemeye hazırsınız. Düzene ****** saygı göstermelisiniz.

// All requests of src ips not matching the ones from ipset ipfriends will be dropped.
$ iptables -I DOCKER -i ext_if -m set ! --match-set ipfriends src -j DROP
// Except for requests coming from a connection already established.
$ iptables -I DOCKER -i ext_if -m state --state ESTABLISHED,RELATED -j ACCEPT

Yeni kurallar oluşturmak istiyorsanız, yenilerini eklemeden önce eklediğiniz tüm özel kuralları kaldırmanız gerekir.

$ iptables -D DOCKER -i ext_if -m set ! --match-set ipfriends src -j DROP
$ iptables -D DOCKER -i ext_if -m state --state ESTABLISHED,RELATED -j ACCEPT
