Birden çok liman işçisi motoru ana bilgisayarının uzaktan yönetimi.

“docker-machine”, Docker çalıştıran uzak ana bilgisayarları yönetir.

docker-machine komut satırı aracı, sağlayıcıya özel sürücüleri kullanarak tüm makinenin yaşam döngüsünü yönetir. “Aktif” bir makine seçmek için kullanılabilir. Bir kez seçildiğinde, etkin bir makine, yerel Docker Motoruymuş gibi kullanılabilir.

Mevcut Docker Machine ortam bilgilerini alın #

Bütün bunlar kabuk komutlarıdır.

Mevcut varsayılan liman işçisi-makine yapılandırmasını almak için “docker-machine env”

eval $(docker-machine env) mevcut liman işçisi-makine yapılandırmasını almak ve mevcut kabuk ortamını bu liman işçisi-makinesini .

Kabuğunuz bir proxy kullanacak şekilde ayarlanmışsa, docker makinenize bağlanırken proxy’yi atlamak için –no-proxy seçeneğini belirtebilirsiniz: eval $(docker-machine env --no-proxy)

Birden çok liman işçisi makineniz varsa, makine adını argüman olarak belirtebilirsiniz: eval $(docker-machine env --no-proxy makineadı)

Docker makinesine SSH #

Bütün bunlar kabuk komutlarıdır

  • Çalışan bir liman işçisi makinesinde doğrudan oturum açmanız gerekiyorsa, bunu yapabilirsiniz:

Varsayılan liman işçisi makinesine ssh yapmak için “docker-machine ssh”

Varsayılan olmayan bir liman işçisi makinesine ssh yapmak için “docker-machine ssh machinename”

  • Sadece tek bir komut çalıştırmak istiyorsanız, bunu yapabilirsiniz. Varsayılan liman işçisi makinesinde ne kadar süredir çalıştığını görmek için “çalışma süresi"ni çalıştırmak için, “docker-machine ssh default uptime” komutunu çalıştırın.

Docker makinesi oluşturun #

Docker’ı bir makineye kurmanın en iyi yöntemi docker-machine kullanmaktır. Karşılıklı kimlik doğrulama ve SSH anahtarları için benzersiz bir çift SSL sertifikası oluşturmak da dahil olmak üzere mevcut en iyi güvenlik ayarlarını otomatik olarak uygulayacaktır.

Virtualbox kullanarak yerel bir makine oluşturmak için:

docker-machine create --driver virtualbox docker-host-1

Docker’ı mevcut bir makineye kurmak için jenerik sürücüyü kullanın:

docker-machine -D create -d generic --generic-ip-address 1.2.3.4 docker-host-2

“–driver” seçeneği, docker’a makineyi nasıl oluşturacağını söyler. Desteklenen sürücülerin listesi için bkz.

  • resmen destekleniyor
  • üçüncü taraf

Liman işçisi makinelerini listeleyin #

Liman işçisi makinelerinin listelenmesi, her bir liman işçisi makinesinin Docker’ın durumunu, adresini ve sürümünü döndürür.

 docker-machine ls

Şunun gibi bir şey yazdıracak:

NAME             ACTIVE   DRIVER    STATE     URL                          SWARM   DOCKER    ERRORS
docker-machine-1 -        ovh       Running   tcp://1.2.3.4:2376                   v1.11.2   
docker-machine-2 -        generic   Running   tcp://1.2.3.5:2376                   v1.11.2

Çalışan makineleri listelemek için:

docker-machine ls --filter state=running

Hata makinelerini listelemek için:

docker-machine ls --filter state=

Adı ‘yan proje-’ ile başlayan makineleri listelemek için Golang filtresini kullanın:

docker-machine ls --filter name="^side-project-"

Yalnızca makinenin URL’lerinin listesini almak için:

docker-machine ls --format '{{ .URL }}'

Tam komut referansı için https://docs.docker.com/machine/reference/ls/ adresine bakın.

Docker Makinesini Yükselt #

Bir liman işçisi makinesini yükseltmek, bir aksama süresi anlamına gelir ve planya gerektirebilir. Bir liman işçisi makinesini yükseltmek için şunu çalıştırın:

docker-machine upgrade docker-machine-name

Bu komutun seçenekleri yok

Docker makinesinin IP adresini alın #

Bir liman işçisi makinesinin IP adresini almak için bunu şu komutla yapabilirsiniz:

docker-machine ip machine-name
