Docker Görüntüsü Kullanarak Üç Düğümlü Mongo Çoğaltma Nasıl Kurulur ve Chef kullanılarak Sağlanır

  • Oluşturma Adımı

Bu sayfada

  • Oluşturma Adımı

Bu Belgeler, Docker Image kullanılarak ve Chef kullanılarak otomatik olarak sağlanan üç düğümlü bir Mongo replika setinin nasıl oluşturulacağını açıklar.

Oluşturma Adımı #

Adımlar:

  1. Mongo düğüm kimlik doğrulaması için bir Base 64 anahtar dosyası oluşturun. Bu dosyayı şef data_bags içine yerleştirin

  2. Chef suppermarket’e gidin ve liman işçisi yemek kitabını indirin. Özel bir yemek kitabı oluşturun (örneğin custom_mongo) ve yemek kitabınızın metadata.rb’sine bağımlı ‘docker’, ‘~> 2.0’ ekleyin

  3. Özel yemek kitabınızda bir nitelik ve tarif oluşturun

  4. Rep Set kümesini oluşturmak için Mongo’yu başlatın

1. Adım: Anahtar dosyası oluşturun

mongo-keyfile adlı data_bag ve keyfile adlı öğeyi oluşturun. Bu, şefin data_bags dizininde olacaktır. Ürün içeriği aşağıdaki gibi olacaktır

openssl rand -base64 756 > <path-to-keyfile>

anahtar dosyası öğesi içeriği

{
  "id": "keyfile",
  "comment": "Mongo Repset keyfile",
  "key-file": "generated base 64 key above"
}

Adım 2: Şef akşam yemeği pazarından liman işçisi yemek kitabını indirin ve ardından custom_mongo yemek kitabı oluşturun

knife cookbook site download docker 
knife cookbook create custom_mongo

custom_mongo eklentisinin metadat.rb’sinde

depends          'docker', '~> 2.0'

Adım 3: özellik ve tarif oluşturun

Öznitellikler

default['custom_mongo']['mongo_keyfile'] = '/data/keyfile' 
default['custom_mongo']['mongo_datadir'] = '/data/db'
default['custom_mongo']['mongo_datapath'] = '/data'
default['custom_mongo']['keyfilename'] = 'mongodb-keyfile'

Yemek tarifi

#
# Cookbook Name:: custom_mongo
# Recipe:: default
#
# Copyright 2017, Innocent Anigbo
#
# All rights reserved - Do Not Redistribute
#

data_path = "#{node['custom_mongo']['mongo_datapath']}"
data_dir = "#{node['custom_mongo']['mongo_datadir']}"
key_dir = "#{node['custom_mongo']['mongo_keyfile']}"
keyfile_content = data_bag_item('mongo-keyfile', 'keyfile')
keyfile_name = "#{node['custom_mongo']['keyfilename']}"

#chown of keyfile to docker user
execute 'assign-user' do
 command "chown 999 #{key_dir}/#{keyfile_name}"
 action :nothing
end

#Declaration to create Mongo data DIR and Keyfile DIR
%W[ #{data_path} #{data_dir} #{key_dir} ].each do |path|
directory path do
  mode '0755'
  end
end

#declaration to copy keyfile from data_bag to keyfile DIR on your mongo server
file "#{key_dir}/#{keyfile_name}" do
  content keyfile_content['key-file']
  group 'root'
  mode '0400'
  notifies :run, 'execute[assign-user]', :immediately
end

#Install docker
docker_service 'default' do
  action [:create, :start]
end

#Install mongo 3.4.2
docker_image 'mongo' do
  tag '3.4.2'
  action :pull
end

Rol dizininde mongo-role adlı Rol oluşturun

{
  "name": "mongo-role",
  "description": "mongo DB Role",
  "run_list": [
    "recipe[custom_mongo]"
  ]
}

Üç mongo düğümü çalıştırma listesine yukarıdaki rolü ekleyin

knife node run_list add FQDN_of_node_01 'role[mongo-role]'
knife node run_list add FQDN_of_node_02 'role[mongo-role]'
knife node run_list add FQDN_of_node_03 'role[mongo-role]'

Adım 4: repset oluşturmak için üç düğüm Mongo’yu başlatın

Yukarıdaki rolün üç Mongo düğümünün tamamında zaten uygulanmış olduğunu varsayıyorum. Yalnızca 01 düğümünde, kimlik doğrulamasını etkinleştirmek için Mongo’yu –auth ile başlatın

docker run --name mongo -v /data/db:/data/db -v /data/keyfile:/opt/keyfile --hostname="mongo-01.example.com" -p 27017:27017 -d mongo:3.4.2 --keyFile /opt/keyfile/mongodb-keyfile --auth

01 düğümünde çalışan docker kapsayıcısının etkileşimli kabuğuna erişin ve Yönetici kullanıcısı oluşturun

docker exec -it mongo /bin/sh
    mongo
    use admin
    db.createUser( {
         user: "admin-user",
         pwd: "password",
         roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
       });

Kök kullanıcı oluştur

db.createUser( {
         user: "RootAdmin",
         pwd: "password",
         roles: [ { role: "root", db: "admin" } ]
       });

Yukarıda 01 düğümünde oluşturulan Docker kapsayıcısını Durdurun ve Silin. Bu, ana bilgisayar DIR’sindeki verileri ve anahtar dosyasını etkilemez. Sildikten sonra Mongo’yu 01 düğümünde tekrar başlatın, ancak bu sefer repset bayrağıyla

docker rm -fv mongo
docker run --name mongo-uat -v /data/db:/data/db -v /data/keyfile:/opt/keyfile --hostname="mongo-01.example.com" -p 27017:27017 -d mongo:3.4.2 --keyFile /opt/keyfile/mongodb-keyfile --replSet "rs0"

şimdi mongo’yu düğüm 02 ve 03’te rep set bayrağıyla başlatın

docker run --name mongo -v /data/db:/data/db -v /data/keyfile:/opt/keyfile --hostname="mongo-02.example.com" -p 27017:27017 -d mongo:3.4.2 --keyFile /opt/keyfile/mongodb-keyfile --replSet "rs0"
docker run --name mongo -v /data/db:/data/db -v /data/keyfile:/opt/keyfile --hostname="mongo-03.example.com" -p 27017:27017 -d mongo:3.4.2 --keyFile /opt/keyfile/mongodb-keyfile --replSet "rs0"

Düğüm 01’deki kök kullanıcıyla kimlik doğrulaması yapın ve çoğaltma kümesini başlatın

use admin
db.auth("RootAdmin", "password");
rs.initiate()

01 düğümünde, repset0 kümesini oluşturmak için Çoğaltma Kümesine Düğüm 2 ve 3’ü ekleyin

rs.add("mongo-02.example.com")
rs.add("mongo-03.example.com")

Test yapmak

Birincil çalıştırmada db.printSlaveReplicationInfo() öğesini çalıştırın ve birincil zamanın SyncedTo ve Behind öğelerini gözlemleyin. Daha sonra aşağıdaki gibi 0 sn olmalıdır

Çıktı

 rs0:PRIMARY> db.printSlaveReplicationInfo()
    source: mongo-02.example.com:27017
            syncedTo: Mon Mar 27 2017 15:01:04 GMT+0000 (UTC)
            0 secs (0 hrs) behind the primary
    source: mongo-03.example.com:27017
            syncedTo: Mon Mar 27 2017 15:01:04 GMT+0000 (UTC)
            0 secs (0 hrs) behind the primary
