Sözdizimi #

  • liman işçisi görüntüleri [SEÇENEKLER] [DEPO[:TAG]]
  • liman işçisi [SEÇENEKLER] KONTEYNER|GÖRÜNTÜ [KONTEYNER|GÖRÜNTÜ…]
  • liman işçisi [SEÇENEKLER] ADI[:TAG|@DIGEST] çeker
  • docker rmi [SEÇENEKLER] GÖRÜNTÜ [GÖRÜNTÜ…]
  • liman işçisi etiketi [SEÇENEKLER] GÖRÜNTÜ[:TAG] [REGISTRYHOST/][USERNAME/]NAME[:TAG]

Docker Hub’dan bir görüntü alma #

Normalde görüntüler Docker Hub‘dan otomatik olarak çekilir. Docker, Docker ana bilgisayarında halihazırda mevcut olmayan herhangi bir görüntüyü Docker Hub’dan çekmeye çalışır. Örneğin, “ubuntu” görüntüsü Docker ana bilgisayarında olmadığında “docker run ubuntu"nun kullanılması, Docker’ın en son “ubuntu” görüntüsünün çekilmesini başlatmasına neden olur. Docker Hub’dan bir görüntüyü manuel olarak almak veya güncellemek için “docker pull” kullanılarak bir görüntüyü ayrı olarak çekmek mümkündür.

docker pull ubuntu
docker pull ubuntu:14.04

Farklı bir görüntü kayıt defterinden çekmek veya bir görüntünün belirli bir sürümünü çekmek için ek seçenekler mevcuttur. Tam görüntü adı ve isteğe bağlı sürüm kullanılarak alternatif bir kayıt defteri belirtilir. Örneğin, aşağıdaki komut ‘ubuntu:14.04’ görüntüsünü ‘registry.example.com’ kayıt defterinden almaya çalışacaktır:

docker pull registry.example.com/username/ubuntu:14.04

Referans resimler #

Bir görüntünün adını alan Docker komutları dört farklı biçimi kabul eder:

Tip Örnek
Kısa Kimlik 693bce725149
İsim “merhaba-dünya” (varsayılanı “:latest” etiketidir)
Ad+etiket “merhaba-dünya:en son”
Özet hello-world@sha256:e52be8ffeeb1f374f440893189cd32f44cb166650e7ab185fa7735b7dc48d619

Not: Bir görüntüye yalnızca, o görüntü orijinal olarak o özet kullanılarak çekilmişse, özetine göre başvurabilirsiniz. Bir görüntünün özetini görmek için (eğer varsa) “docker images –digests” komutunu çalıştırın.

Resimleri Kaldırma #

Görüntüleri kaldırmak için “docker rmi” komutu kullanılır:

docker rmi <image name>

Bir resmi kaldırmak için tam resim adı kullanılmalıdır. Görüntü, kayıt defteri adını kaldırmak için etiketlenmemişse, belirtilmesi gerekir. Örneğin:

docker rmi registry.example.com/username/myAppImage:1.3.5

Bunun yerine görüntüleri kimliklerine göre kaldırmak da mümkündür:

docker rmi 693bce725149

Kolaylık sağlamak için, belirtilen alt dize net olduğu sürece, resim kimliğinin yalnızca ilk birkaç karakterini belirterek resimleri resim kimliklerine göre kaldırmak mümkündür:

docker rmi 693

Not: Görüntüler, o görüntüyü kullanan mevcut kapsayıcılar olsa bile kaldırılabilir; docker rmi, görüntünün “etiketini kaldırır”.

Hiçbir kap bir görüntü kullanmıyorsa, çöp toplanır. Bir kap bir görüntü kullanıyorsa, onu kullanan tüm kapsayıcılar kaldırıldıktan sonra görüntü çöp olarak toplanır. Örneğin:

$ docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED                  STATUS                     PORTS               NAMES
5483657ee07b        hello-world         "/hello"            Less than a second ago   Exited (0) 2 seconds ago                       small_elion

$ docker rmi hello-world
Untagged: hello-world:latest

$ docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED                  STATUS                      PORTS               NAMES
5483657ee07b        693bce725149        "/hello"            Less than a second ago   Exited (0) 12 seconds ago                       small_elion

Başlamamış Kapsayıcılarla Tüm Resimleri Kaldır

Başlatılmış kapsayıcıya sahip olmayan tüm yerel görüntüleri kaldırmak için, görüntülerin bir listesini parametre olarak sağlayabilirsiniz:

docker rmi $(docker images -qa)

Tüm Resimleri Kaldır

Başlatılmış bir kapsayıcıya sahip olup olmadıklarına bakılmaksızın görüntüleri kaldırmak istiyorsanız, kuvvet bayrağını (-f) kullanın:

docker rmi -f $(docker images -qa)

** Sarkan Resimleri Kaldır**

Bir resim etiketlenmemişse ve herhangi bir kapsayıcı tarafından kullanılmıyorsa, ‘sarkandır’ ve şu şekilde kaldırılabilir:

docker images -q --no-trunc -f dangling=true | xargs -r docker rmi

Görüntüler için Docker Hub’da arama yapın #

Search komutunu kullanarak Docker Hub görüntüleri arayabilirsiniz:

docker search <term>

Örneğin:

Görüntüleri inceleme #

docker inspect <image>

Çıktı JSON formatındadır. Yalnızca istenen tuşları ayrıştırmak ve yazdırmak için jq komut satırı yardımcı programını kullanabilirsiniz.

docker inspect <image> | jq -r '.[0].Author'

Yukarıdaki komut, resimlerin yazar adını gösterecektir.

Resimleri etiketleme #

Bir görseli etiketlemek, farklı görsel versiyonlarını takip etmek için kullanışlıdır:

docker tag ubuntu:latest registry.example.com/username/ubuntu:latest

Başka bir etiketleme örneği:

docker tag myApp:1.4.2 myApp:latest
docker tag myApp:1.4.2 registry.example.com/company/myApp:1.4.2

Docker görüntülerini kaydetme ve yükleme #

docker save -o ubuntu.latest.tar ubuntu:latest

Bu komut, ‘ubuntu:latest’ görüntüsünü geçerli dizinde ‘ubuntu.latest.tar’ adıyla bir tarball arşivi olarak kaydeder. Bu tarball arşivi daha sonra örneğin “rsync” kullanılarak başka bir ana bilgisayara taşınabilir veya depoda arşivlenebilir.

Tarball taşındığında, aşağıdaki komut dosyadan bir görüntü oluşturacaktır:

docker load -i /tmp/ubuntu.latest.tar

Artık her zamanki gibi ‘ubuntu:latest’ görüntüsünden kaplar oluşturmak mümkündür.
