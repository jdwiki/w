Docker’ın her yönünü kendi uygulamalarınızdan kontrol etmenize, Docker’da çalışan uygulamaları yönetmek ve izlemek için araçlar oluşturmanıza ve hatta onu Docker’ın kendisinde uygulamalar oluşturmak için kullanmanıza olanak tanıyan bir API.

systemd çalıştıran Linux üzerinde Docker API’sine Uzaktan erişimi etkinleştir #

Linux çalıştıran systemd, Ubuntu 16.04 gibi, /etc/default/docker öğesine -H tcp://0.0.0.0:2375 eklenmesi eskisi gibi etki yapmıyor.

Bunun yerine, 4243 numaralı bağlantı noktasındaki bir TCP soketinde docker’ı kullanılabilir hale getirmek için /etc/systemd/system/docker-tcp.socket adlı bir dosya oluşturun:

[Unit]
Description=Docker Socket for the API  
[Socket]
ListenStream=4243  
Service=docker.service  
[Install]
WantedBy=sockets.target

Ardından yeni soketi etkinleştirin:

systemctl enable docker-tcp.socket
systemctl enable docker.socket
systemctl stop docker
systemctl start docker-tcp.socket
systemctl start docker

Şimdi Uzak API’nin çalışıp çalışmadığını doğrulayın:

curl -X GET http://localhost:4243/images/json

Systemd’de TLS ile Uzaktan Erişimi Etkinleştir #

Paket yükleyici birim dosyasını /etc dizinine kopyalayın, burada yükseltme sırasında değişikliklerin üzerine yazılmaz:

cp /lib/systemd/system/docker.service /etc/systemd/system/docker.service

ExecStart’taki seçeneklerinizle /etc/systemd/system/docker.service’i güncelleyin:

ExecStart=/usr/bin/dockerd -H fd:// -H tcp://0.0.0.0:2376 \
  --tlsverify --tlscacert=/etc/docker/certs/ca.pem \
  --tlskey=/etc/docker/certs/key.pem \
  --tlscert=/etc/docker/certs/cert.pem

“dockerd"ın 1.12 daemon adı olduğuna ve öncesinde “docker daemon” olduğuna dikkat edin. Ayrıca 2376’nın liman işçisi standart TLS bağlantı noktası olduğunu, 2375’in standart şifrelenmemiş bağlantı noktası olduğunu unutmayın. Kendi TLS imzalı CA’nızı, sertifikanızı ve anahtarınızı oluşturma adımları için bu sayfaya bakın.

Systemd birim dosyalarında değişiklik yaptıktan sonra systemd yapılandırmasını yeniden yüklemek için aşağıdakileri çalıştırın:

systemctl daemon-reload

Ardından, docker’ı yeniden başlatmak için aşağıdakileri çalıştırın:

systemctl restart docker

Bu bağlantı noktasına ağ erişimi olan herkes ana bilgisayarda tam kök erişimine sahip olduğundan, Docker bağlantı noktasını açığa çıkarırken TLS şifrelemesini atlamak kötü bir fikirdir.

Linux’ta Docker API’sine Uzaktan Erişimi Etkinleştir #

/etc/init/docker.conf dosyasını düzenleyin ve DOCKER_OPTS değişkenini aşağıdaki şekilde güncelleyin:

DOCKER_OPTS='-H tcp://0.0.0.0:4243 -H unix:///var/run/docker.sock'

Docker deamon’u yeniden başlatın

service docker restart

Remote API’nin çalışıp çalışmadığını doğrulayın

curl -X GET http://localhost:4243/images/json

Go ile yazılmış ilerleme çubuklarıyla resim çekme #

Burada, ‘Go’ ve ‘Docker Engine API’ ve ‘CLI’de ‘docker pull your_image_name’ komutunu çalıştırdığınızda gösterilen ilerleme çubuklarıyla aynı ilerleme çubuklarını kullanarak görüntü çekme örneği verilmiştir. İlerleme çubuklarının amaçları için bazı ANSI kodları kullanılır.

package yourpackage

import (
    "context"
    "encoding/json"
    "fmt"
    "io"
    "strings"

    "github.com/docker/docker/api/types"
    "github.com/docker/docker/client"
)

// Struct representing events returned from image pulling
type pullEvent struct {
    ID             string `json:"id"`
    Status         string `json:"status"`
    Error          string `json:"error,omitempty"`
    Progress       string `json:"progress,omitempty"`
    ProgressDetail struct {
        Current int `json:"current"`
        Total   int `json:"total"`
    } `json:"progressDetail"`
}

// Actual image pulling function
func PullImage(dockerImageName string) bool {
    client, err := client.NewEnvClient()

    if err != nil {
        panic(err)
    }

    resp, err := client.ImagePull(context.Background(), dockerImageName, types.ImagePullOptions{})

    if err != nil {
        panic(err)
    }

    cursor := Cursor{}
    layers := make([]string, 0)
    oldIndex := len(layers)

    var event *pullEvent
    decoder := json.NewDecoder(resp)

    fmt.Printf("\n")
    cursor.hide()

    for {
        if err := decoder.Decode(&event); err != nil {
            if err == io.EOF {
                break
            }

            panic(err)
        }

        imageID := event.ID

        // Check if the line is one of the final two ones
        if strings.HasPrefix(event.Status, "Digest:") || strings.HasPrefix(event.Status, "Status:") {
            fmt.Printf("%s\n", event.Status)
            continue
        }

        // Check if ID has already passed once
        index := 0
        for i, v := range layers {
            if v == imageID {
                index = i + 1
                break
            }
        }

        // Move the cursor
        if index > 0 {
            diff := index - oldIndex

            if diff > 1 {
                down := diff - 1
                cursor.moveDown(down)
            } else if diff < 1 {
                up := diff*(-1) + 1
                cursor.moveUp(up)
            }

            oldIndex = index
        } else {
            layers = append(layers, event.ID)
            diff := len(layers) - oldIndex

            if diff > 1 {
                cursor.moveDown(diff) // Return to the last row
            }

            oldIndex = len(layers)
        }

        cursor.clearLine()

        if event.Status == "Pull complete" {
            fmt.Printf("%s: %s\n", event.ID, event.Status)
        } else {
            fmt.Printf("%s: %s %s\n", event.ID, event.Status, event.Progress)
        }

    }

    cursor.show()

    if strings.Contains(event.Status, fmt.Sprintf("Downloaded newer image for %s", dockerImageName)) {
        return true
    }

    return false
}

Daha iyi okunabilirlik için, ANSI kodlarıyla imleç eylemleri şuna benzeyen ayrı bir yapıya taşınır:

package yourpackage

import "fmt"

// Cursor structure that implements some methods
// for manipulating command line's cursor
type Cursor struct{}

func (cursor *Cursor) hide() {
    fmt.Printf("\033[?25l")
}

func (cursor *Cursor) show() {
    fmt.Printf("\033[?25h")
}

func (cursor *Cursor) moveUp(rows int) {
    fmt.Printf("\033[%dF", rows)
}

func (cursor *Cursor) moveDown(rows int) {
    fmt.Printf("\033[%dE", rows)
}

func (cursor *Cursor) clearLine() {
    fmt.Printf("\033[2K")
}

Bundan sonra ana paketinizde, çekmek istediğiniz resim adını geçerek PullImage fonksiyonunu çağırabilirsiniz. Elbette çağırmadan önce görüntünün bulunduğu Docker kayıt defterine giriş yapmanız gerekiyor.

Bazı karmaşık yapılardan geçerek cURL isteği yapmak #

‘Docker API’sine yapılan bazı sorgular için ‘cURL’yi kullanırken, bazı karmaşık yapıları iletmek biraz zor olabilir. Diyelim ki, görüntülerin bir listesini almak, filtrelerin bir sorgu parametresi olarak kullanılmasına izin veriyor; bu, ‘harita[string][]dize’nin bir ‘JSON’ temsili olmalıdır (‘Go’ içindeki haritalar hakkında şunları yapabilirsiniz: daha fazlasını burada bulabilirsiniz).
Bunu nasıl başaracağınız aşağıda açıklanmıştır:

curl --unix-socket /var/run/docker.sock \
    -XGET "http:/v1.29/images/json" \
    -G \
    --data-urlencode 'filters={"reference":{"yourpreciousregistry.com/path/to/image": true}, "dangling":{"true": true}}'

Burada “-G” bayrağı, “–data-urlencode” parametresindeki verilerin, aksi takdirde kullanılacak olan “POST” isteği yerine bir “HTTP GET” isteğinde kullanılacağını belirtmek için kullanılır. Veriler URL’ye bir “?” ayırıcısı ile eklenecektir.
