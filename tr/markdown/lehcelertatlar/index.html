Farklı söz dizimine sahip Markdown varyasyonlarına “tatlar” denir. Markdown aromaları https://github.com/jgm/CommonMark/wiki/Markdown-Flavors adresinde listelenmiştir.

Bir Markdown aroması, çeşitli programlama dillerinde ve yazılım uygulamalarında uygulanabilir. Markdown ayrıştırıcısı uygulayan kitaplıklar https://github.com/markdown/markdown.github.com/wiki/Implementations ve https://www.w3.org/community/markdown/wiki/MarkdownImplementations adresinde listelenmiştir.

Çeşitli Markdown aromaları ve aralarındaki farklar http://flavoredmarkdown.com adresinde belgelenmiştir.

Markdown kodunu birçok farklı tatta ve bu tatların uygulamalarını test etmek ve karşılaştırmak için çevrimiçi aracı Babelmark kullanabilirsiniz.

Bu, Stack Overflow ve diğer Stack Exchange siteleri tarafından kullanılan işaretlemenin tadıdır. Bir soruyu yanıtladığınızda veya belge eklediğinizde bu işaretlemeyi kullanırsınız. Bu cevap SO işaretlemesinden yapılmıştır

Doxygen Markdown #

Doxygen Markdown kullanımını da destekleyen yaygın olarak kullanılan bir kod belgeleme aracıdır (C++, C# ve Java dahil diller için). Standart Markdown sözdizimine ek olarak, bir dizi Doxygen’e özgü öğe vardır.

Birincil özellikler, referanslar için “@ref” etiketlerinin ve bunların başvurabileceği “@page”, “@section/@subsection” ve “@anchor” öğelerinin kullanılmasıdır.

@ref <ad> [“metin”] #

Bu öğe, belgede başka bir yerde tanımlanmış olan adlandırılmış bir bölüm, alt bölüm, sayfa veya bağlantıya bir bağlantı (yani referans) oluşturur. (bkz. Doxygen referansı)

İlk parametre (“name”), bağlantı vermek istediğiniz bölümün, alt bölümün, sayfanın veya bağlantının adıyla eşleşmelidir.

İkinci isteğe bağlı parametre ("text") çift tırnak içine alınmalıdır ve referans bağlantısının sayfada nasıl görüneceğini tanımlayacaktır. Kullanılmazsa, bağlantı referansta kullanılan başlık olarak görünecektir.

@section <bölüm-adı> (bölüm başlığı) #

Bu öğe bir bölüm adını tanımlar. Görsel olarak Markdown’daki bir ‘#’ öğesine eşdeğerdir, ancak belgelerinizin diğer bölümlerinden bağlantı kurulabilecek bir referans da tanımlayacaktır. (bkz. Doxygen referansı)

İlk parametre olan “section-name”, “@ref” öğesi tarafından kullanılabilecek referans adını tanımlar. Bu boşluk içeremez.

İkinci parametre olan “bölüm başlığı”, sayfanızda bölüm başlığının nasıl görüneceğini tanımlayan (boşluklarla ayrılabilen) bir sözcük dizisidir.

Örnek #

@section Intro Introduction

This is some text in my introduction.

@section Body Body Paragraph

This is some text in my body, where I refer to the @ref Intro.

Çıktı #

Giriiş #

Bu benim tanıtımımda bir metin.

Gelişme paragrafı #

Bu, Giriş’e atıfta bulunduğum, vücudumdaki bir metin.

Not: Yukarıdaki Giriş kelimesi Giriş başlığına atlayacak bir bağlantı olarak görünecektir.

GitHub Aromalı Markdown #

GitHub Flavored Markdown (bazen GFM olarak kısaltılır), GitHub.com üzerinde markdown ile çalışmayı kolaylaştırır.

GFM’nin temel özellikleri şunları içerir:

  • kod girintisi
  • görev listesi desteği
  • kolay GitHub sorunu referansı
  • otomatik GitHub kullanıcı adı ve SHA algılama
  • otomatik url algılama
  • emoji desteği
