Drush nedir? #

Drush, Drupal siteleri için bir komut satırı komut dosyası arabirimidir. Drupal sitelerinin komut satırı yönetimine izin verir.

Drush komutları #

Aşırı Durum #

drush status

Bu size Drupal sitenize genel bir bakış sağlayacaktır. Sürüm, URI, veritabanı konumu, dosya yolları, varsayılan tema vb. Bu komutu kullanırsanız ve bu bilgileri görmüyorsanız, yanlış bir klasördesiniz ve Drush hangi Drupal sitesinden bahsettiğinizi bilmiyor demektir.

Herhangi bir kullanıcı için şifre sıfırlama #

drush upwd admin --password="newpassword"

Burada “admin” mevcut bir kullanıcı adı ve “newpassword” istenen şifredir.

Tek seferlik bir yönetici giriş URL’si oluşturun #

drush uli

Yönetici bölümüne giriş yapmak için kullanılabilecek bir URL oluşturur. URL’nin tek kullanımlık bir belirteci var. Sonuç şöyle görünmelidir:

http://example.com/user/reset/1/1469178712/MEn1QOXo3YGKAUHCknFQF0rEPJ_itkS-a6I8LJwaNYs/login

Bazen ana bilgisayar adı veya IP çözülemez ve sonuç şuna benzer bir uyarıdır:

default does not appear to be a resolvable hostname or IP, not starting browser.   [warning]
You may need to use the --uri option in your command or site alias to indicate
the correct URL of this site.
http://default/user/reset/1/1469178629/-zFS_0u8is2N2uCKuLUdGBpJ3cZzV9am5_irsbtVAOs/login

Çözüm, aşağıdaki örnekte olduğu gibi “–url” parametresini kullanıyor:

drush uli --uri="http://example.com/"

Önbellekleri temizleme #

drush cache-rebuild

Drupal 8 için önbelleği yeniden oluşturur. drupal 7 için şunları kullanabilirsiniz.

drush cache-clear

Bu komutlar ayrıca daha kısa

drush cr

veya

drush cc // optionally pass all to clear all the caches

Modülleri etkinleştir #

drush pm-enable mymodule

Yönetici arayüzünde bir modülü etkinleştirmek gibi ‘mymodule’ özelliğini etkinleştirin

Bu komutlar ayrıca daha kısa

drush en mymodule // optionally pass the -y option to avoid the interactive question

Bakım #

Drush kullanarak bakım modunu etkinleştirmek için şu komutu kullanabilirsiniz:

drush vset maintenance_mode 1 // pass 0 to disable the maintenance

Bakım modunu etkinleştirdikten / devre dışı bıraktıktan sonra önbellekleri temizlemeyi unutmayın.

Yapılandırmayı Dışa Aktar #

Drupal 7 ve daha düşük sürümlerde, yapılandırmanız muhtemelen Features modülü kullanılarak depolanır. Veritabanlarındaki değişikliklerle bir özelliği güncellemek için bu komutu kullanın:

drush features-update [feature-name] // e.g. drush features-update content_type_news

Bu kısayolu da kullanabilirsiniz:

drush fu [feature-name]

Drupal 8. “Yapılandırma Yönetimi” kullanarak. Yapılandırmanızı drush ile dışa aktarmak için bu komutu kullanın

drush config-export // optionally add -y to not have to verify it

Ayrıca steno komutunu da kullanabilirsiniz.

drush cex -y

Drush’ı yükleyin #

Manuel Global Kurulum #

OS X ve Linux için:

  1. Terminal, Bash veya normal kabuğunuzu açın.

  2. Aşağıdakileri Terminal/Bash’e yazın:

    # Download latest stable release using the code below or browse to github.com/drush-ops/drush/releases.
php -r "readfile('http://files.drush.org/drush.phar');" > drush
# Or use our upcoming release: php -r "readfile('http://files.drush.org/drush-unstable.phar');" > drush

# Test your install.
php drush core-status

# Make `drush` executable as a command from anywhere. Destination can be anywhere on $PATH.
chmod +x drush
sudo mv drush /usr/local/bin

# Optional. Enrich the bash startup file with completion and aliases.
drush init

Pencereler için:

  1. GitHub’dan Drush dosyasını indirin.
  2. Sıkıştırılmış dosyayı istediğiniz sürücüye çıkarın, örn. 
    C:\
  3. Drush’ı kurun, Apache, PHP ve MySQLi de içermesi gereken ortam yolu değişkeninde ayıklanan klasör yolunu tanımlayın. 
    C:\xampp\apache\bin;C:\xampp\mysql\bin;C:\xampp\php;C:\drush;
  4. Drush’ın çalıştığını doğrulayın: 
    drush status

Besteci Küresel Kurulum #

  1. Composer’ı global olarak yükleyin.
  2. export PATH="$HOME/.composer/vendor/bin:$PATH" öğesini ~/.bash_profile (OS X) veya ~/.bashrc (Linux) içine yerleştirerek Composer’ın bin dizinini sistem yoluna ekleyin ).
  3. Drush’ı yükleyin: 
    composer global require drush/drush
  4. Drush’ın çalıştığını doğrulayın: 
    drush status

Drush’ı yükleyin #

Manuel kurulum #

Terminalde aşağıdaki komutları yazın.

php -r "readfile('http://files.drush.org/drush.phar');" > drush
chmod +x drush
sudo mv drush /usr/local/bin
drush init # Add alias in bash startup file.

Besteci #

Composer’ın kurulu olduğunu varsayarsak.

composer global require drush/drush:dev-master
