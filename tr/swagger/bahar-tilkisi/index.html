Varsayılan yanıt mesajlarını geçersiz kıl #

Springfox, varsayılan olarak tüm API denetleyicilerine uygulanan bir dizi varsayılan yanıt mesajı tanımlar. Bu, örn. “201 - Oluşturuldu” ve “204 - İçerik Yok” ve ayrıca birkaç “40x” yanıtı. Varsayılan yanıt mesajlarının API’niz için geçerli olmadığı durumlar olabilir. Bununla başa çıkmak için olasılıklar oluşturmalısınız:

  • Varsayılan yanıt mesajını kapatabilir ve @ApiResponses notunu kullanarak kendinizinkini tanımlayabilirsiniz.
  • Global olarak kendi yanıt mesajlarınızı tanımlayabilirsiniz

Varsayılan yanıt mesajlarının dönüşü #

docket.useDefaultResponseMessages(false);

Artık bireysel yanıt mesajlarınızı her denetleyici düzeyinde ayarlayabilirsiniz. Örneğin.

 @ApiResponses(value = {
        @ApiResponse(code=400, message = "This is a bad request, please stick to the API description", response = RestApiExceptionModel.class),
        @ApiResponse(code=401, message = "Your request cannot be authorized.", response = RestApiExceptionModel.class)
 })

Kendi varsayılan yanıt mesajlarınızı ayarlayın #

ModelRef errorModel = new ModelRef("RestApiExceptionModel");
List<ResponseMessage> responseMessages = Arrays.asList(
        new ResponseMessageBuilder().code(401).message("Unauthorized").responseModel(errorModel).build(),
        new ResponseMessageBuilder().code(403).message("Forbidden").responseModel(errorModel).build(),
        new ResponseMessageBuilder().code(404).message("NotFound").responseModel(errorModel).build());

docket.globalResponseMessage(RequestMethod.POST, responseMessages)
        .globalResponseMessage(RequestMethod.PUT, responseMessages)
        .globalResponseMessage(RequestMethod.GET, responseMessages)
        .globalResponseMessage(RequestMethod.DELETE, responseMessages);

Spring-boot’ta swagger-ui kullanarak springfox’u kurun #

  1. Springfox’u Maven veya Gradle kullanarak uygulamanıza dahil edin
  2. Uygulamanızda yeni bir Docket bean oluşturun ve yapılandırın
  3. API’nizi ihtiyaçlarınıza göre belgeleyin
  4. Uygulamanızı başlatın ve elde ettiğiniz sonuçları görün

#1 Maven ile Springfox’u Almak #

pom.xml dosyanıza swagger2 ve swagger-ui bağımlılıklarını ekleyin

<dependency>
    <groupId>io.springfox</groupId>
    <artifactId>springfox-swagger2</artifactId>
    <version>2.6.0</version>
</dependency>
<dependency>
    <groupId>io.springfox</groupId>
    <artifactId>springfox-swagger-ui</artifactId>
    <version>2.6.0</version>
</dependency>

#2 Uygulamanızı swagger kullanacak şekilde yapılandırın #

‘@SpringBootApplication’ açıklamalı ana sınıfınıza ‘@EnableSwagger2’ ek açıklamasını ekleyin ve bu (veya başka herhangi bir) yapılandırma sınıfında bir swagger Docket bean oluşturun.

@Bean
public Docket api() {                
    return new Docket(DocumentationType.SWAGGER_2)          
      .select()                                       
      .apis(RequestHandlerSelectors.any())
      .paths(PathSelectors.any())                     
      .build();
}

Bu konfigürasyon, uygulamanızdaki tüm yay kontrolörleri üzerinde bir API dokümantasyonu oluşturacaktır. API belgelerinin üretimini belirli denetleyicilerle sınırlamanız gerekirse, çeşitli RequestHandlerSelectors arasından seçim yapabilirsiniz. Örneğin. API belgelerinizi, RequestHandlerSelectors.basePackage("your.package.structure") kullanarak paket yapınıza veya RequestHandlerSelectors.withClassAnnotation(Api.class) kullanarak açıklama eklediğiniz belirli sınıflara dayalı olarak oluşturabilirsiniz.

#3 API’nizi belgeleyin #

Ek bilgilerle denetleyici sınıflarınızı ve yöntemlerinizi geliştirmek için belgelerde açıklanan ek açıklamaları kullanın. Genel başlık, açıklama veya sürüm gibi API’nizin genel bilgilerini açıklamak için Docket bean’inizdeki ApiInfoBuilder()‘ı kullanın.

ApiInfoBuilder kullanılarak meta veri tanımı için örnek:

// Within your configuration class
public static ApiInfo metadata(){
    return new ApiInfoBuilder()
            .title("Your Title")
            .description("Your Description")
            .version("1.x")
            .build();
}

// Within your method that definies the Docket bean...
docket.apiInfo(metadata());
