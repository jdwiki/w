REST WS Kavramları (JAX-RS) Ek açıklamaların kullanımı Web uygulaması REST API standartları Maven ve bağımlılığı

swagger-ui forması REST WS ile #

Swagger‘in resmi web sitesinde dediği gibi:

Swagger, hem insanların hem de bilgisayarların hizmetin yeteneklerini kaynak koduna, belgelere veya ağ trafiği denetimi aracılığıyla keşfetmesine ve anlamasına olanak tanıyan, REST API’lerine yönelik standart, dilden bağımsız bir arabirim tanımlamaktır. Swagger aracılığıyla doğru bir şekilde tanımlandığında, bir tüketici, minimum miktarda uygulama mantığıyla uzak hizmeti anlayabilir ve onunla etkileşime girebilir. Daha düşük seviyeli programlama için arayüzlerin yaptıklarına benzer şekilde Swagger, hizmeti çağırırken tahminde bulunmayı ortadan kaldırır.

Maven ve jersey kullandığınızı varsayarsak, aşağıdaki bağımlılığın eklenmesi gerekir: Maven Swagger Dependency ve JAX-RS bağımlılıkları. Şimdi “maven webapp” oluşturmanız gerekiyor ve webapp’ın altına bu URL‘de bulunan içeriği yapıştırmanız gerekiyor. Şimdi, Swagger tarafından sağlanan Ek Açıklamaları kullanacağımız gerçek koda geçeceğiz:

Aşağıdaki gibi bir bean Employee.java oluşturun:

import io.swagger.annotations.ApiModel;
import io.swagger.annotations.ApiModelProperty;

@ApiModel("Employee bean")
public class Employee {

    private String name;
    private String id;
    private String dept;

    @ApiModelProperty(value = "Name of employee", example = "Test Employee")
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @ApiModelProperty(value = "Id of employee", example = "123456")
    public String getId() {
        return id;
    }

    public void setId(String id) {
        this.id = id;
    }

    @ApiModelProperty(value = "Department of employee", example = "IT Division", allowableValues = "IT, Sales, Admin")
    public String getDept() {
        return dept;
    }

    public void setDept(String dept) {
        this.dept = dept;
    }
}

Burada kullandığımız Swagger bileşenleri şunlardır:

  1. @ApiModel("Employee bean") - Bu, Swagger UI’da görüntülenmesi gereken Bean sınıfının adına karar verecektir.
  2. @ApiModelProperty(value ="ABC", example="DeptName") - Bu, çekirdekte kullanılan her alan hakkında bilgi sağlayacaktır. değer, alanın açıklamasını sağlar ve örnek, o alanın örnek değerini sağlar.

Şimdi aşağıdaki gibi bir GET web servisi oluşturmak için aşağıdaki gibi bir REST Controller oluşturacağız:

import javax.ws.rs.GET;
import javax.ws.rs.Path;
import javax.ws.rs.Produces;
import javax.ws.rs.core.GenericEntity;
import javax.ws.rs.core.MediaType;
import javax.ws.rs.core.Response;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.swagger.ws.bean.Employee;
import org.swagger.ws.service.EmployeeService;

import io.swagger.annotations.Api;
import io.swagger.annotations.ApiOperation;
import io.swagger.annotations.ApiResponse;
import io.swagger.annotations.ApiResponses;

@Path("/employee")
@Api(tags = {"Employee"})
public class EmployeeController {
    private static final Logger LOGGER = LoggerFactory.getLogger(EmployeeController.class);

@GET
@Produces(MediaType.APPLICATION_JSON + ";charset=utf-8")
@ApiOperation(produces="application/json", value = "Fetch employee details", httpMethod="GET", notes = "<br>This service fetches Employee details", response = Employee.class)
@ApiResponses(value = { @ApiResponse(code = 200,response = Employee.class, message = "Successful operation"),@ApiResponse(code = 400, message = "Bad Request", response = Error.class), @ApiResponse(code = 422, message = "Invalid data", response = Error.class), @ApiResponse(code = 500, message = "Internal Server Error", response = Error.class) })
public Response getEmployee()  {
        EmployeeService employeeService = new EmployeeService();
        Employee empDetails = employeeService.getEmployee();
        Response response;
        LOGGER.debug("Fetching employee");
        GenericEntity<Employee> entity = new GenericEntity<Employee>(empDetails){}; 
        response = Response.status(200).entity(entity).build();
        return response;
    }
}

Burada kullandığımız Swagger bileşenleri şunlardır:

  1. @Api(tags = {"Çalışan"}) - Bu ek açıklama Swagger’a web hizmetinin başlığının ne olması gerektiğini söyleyecektir. Bu durumda başlık “Çalışan” dır. Lütfen, REST WS yazarken birkaç standardın, Swagger Belgeleri yazarken de geçerli olduğunu unutmayın, başlık gibi “GetEmployee” veya “DeleteEmployee” gibi olmamalıdır.
  2. @ApiOperation(produces="application/json", value = "Çalışan ayrıntılarını getir", httpMethod="GET", notlar = "<br>Bu hizmet Çalışan ayrıntılarını getirir", yanıt = Çalışan.class) - Bu ek açıklama, web hizmetiniz hakkında kısa bir fikir sağlar. - produces describes format of response, - value describes brief idea about the webservice - notes describes detailed information about this webservice in case of any.
  3. @ApiResponses(değer = { @ApiResponse(kod = 200,yanıt = Çalışan.sınıf, mesaj = "Başarılı işlem"),@ApiResponse(kod = 400, mesaj = "Hatalı İstek", yanıt = Hata.sınıf) , @ApiResponse(kod = 422, mesaj = "Geçersiz veri", yanıt = Hata.sınıf), @ApiResponse(kod = 500, mesaj = "Dahili Sunucu Hatası", yanıt = Error.class) }) - Bu ek açıklamalar şunları sağlar: bize bu web servisini kullanırken yanıt olarak alınabilecek farklı HTTP Durum kodlarını ele almanın bir yolu. Hata kodunu, buna karşı özel mesajı ayarlamamızı ve hatta gerekirse bu hatayı ayrı bir hata sınıfında yakalamamızı sağlar.

Son olarak, web servisi istemci tarafından tüketildiğinde çalışanların ayrıntılarını alacak gerçek bir Hizmet sınıfı oluşturmamız gerekiyor. Gerektiği gibi uygulayabilirsiniz. Demo amaçlı örnek hizmet aşağıdadır:

public class EmployeeService {

public Employee getEmployee() {
    Employee employee = new Employee();
    employee.setId("1");
    employee.setName("Test");
    employee.setDept("IT");
    return employee;
    }
}
