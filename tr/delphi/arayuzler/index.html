Arayüzler, açık uygulama bilgisi sağlamadan, gerekli bilgileri ve yöntem ve sınıfların beklenen çıktısını tanımlamak için kullanılır.

Sınıflar arabirimleri uygulayabilir ve arabirimler birbirlerinden devralabilir. Bir sınıf bir arabirimi uyguluyorsa, bu, arabirim tarafından sunulan tüm işlevlerin ve prosedürlerin sınıfta mevcut olduğu anlamına gelir.

Delphi’deki arayüzlerin özel bir yönü, arayüz örneklerinin referans sayımına dayalı bir ömür yönetimine sahip olmasıdır. Sınıf örneklerinin ömrü manuel olarak yönetilmelidir.

Tüm bu hususlar göz önüne alındığında, farklı hedeflere ulaşmak için arayüzler kullanılabilir:

  • İşlemler için birden fazla farklı uygulama sağlayın (örneğin bir dosyaya kaydetme, veritabanına kaydetme veya E-Posta olarak gönderme, tümü “SaveData” Arabirimi olarak)
  • Bağımlılıkları azaltın, ayrıştırmayı iyileştirin ve böylece kodu daha iyi sürdürülebilir ve test edilebilir hale getirin
  • Ömür boyu yönetimle uğraşmadan birden fazla birimde örneklerle çalışın (burada bile tuzaklar olsa da, dikkatli olun!)

Arayüz tanımlama ve uygulama #

Bir arabirim, bir sınıf gibi bildirilir, ancak erişim değiştiricileri (‘public’, ‘private’, …). Ayrıca, hiçbir tanımlamaya izin verilmez, bu nedenle değişkenler ve sabitler kullanılamaz.

Arayüzlerde her zaman, Ctrl + Shift + G tuşlarına basılarak oluşturulabilen bir Benzersiz Tanımlayıcı olmalıdır.

IRepository = interface
    ['{AFCFCE96-2EC2-4AE4-8E23-D4C4FF6BBD01}']
    function  SaveKeyValuePair(aKey: Integer; aValue: string): Boolean;
end;

Bir arabirimi uygulamak için, arabirimin adı temel sınıfın arkasına eklenmelidir. Ayrıca, sınıf ‘TInterfacedObject’in alt öğesi olmalıdır (bu ömür boyu yönetim için önemlidir).

TDatabaseRepository = class(TInterfacedObject, IRepository)
    function  SaveKeyValuePair(aKey: Integer; aValue: string): Boolean;
end;

Bir sınıf bir arabirimi uyguladığında, arabirimde bildirilen tüm yöntemleri ve işlevleri içermelidir, aksi takdirde derlenmez.

Kayda değer bir şey, arayan arayüz ile çalışıyorsa, erişim değiştiricilerinin herhangi bir etkisinin olmamasıdır. Örneğin, arabirimin tüm işlevleri ‘katı özel’ üyeler olarak uygulanabilir, ancak arabirimin bir örneği kullanılıyorsa yine de başka bir sınıftan çağrılabilir.

Arayüzlerdeki özellikler #

Arayüzlerde değişkenlerin bildirilmesi mümkün olmadığından, özellikleri tanımlamanın “hızlı” yolu (property Value: TObject read FValue write FValue;) kullanılamaz. Bunun yerine, Getter ve setter (her biri yalnızca gerekirse) arayüzde de bildirilmelidir.

IInterface = interface(IInterface)
    ['{6C47FF48-3943-4B53-8D5D-537F4A0DEC0D}']
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject;

    property Value: TObject read GetValue write SetValue;
end;

Kayda değer bir şey, uygulayıcı sınıfın özelliği bildirmek zorunda olmadığıdır. Derleyici bu kodu kabul eder:

TImplementer = class(TInterfacedObject, IInterface)
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject
end;

Ancak bir uyarı, bu şekilde özelliğe sınıfın kendisi aracılığıyla değil, yalnızca arabirimin bir örneği aracılığıyla erişilebileceğidir. Ayrıca, özelliğin sınıfa eklenmesi okunabilirliği artırır.

Birden çok arayüz uygulama #

Delphi sınıfları için mümkün olmayan birden fazla sınıftan (Çoklu Kalıtım) miras almanın aksine, sınıflar birden fazla arabirim uygulayabilir. Bunu başarmak için, tüm arayüzlerin adı, temel sınıfın arkasına virgülle ayrılmış olarak eklenmelidir.

Elbette, uygulayıcı sınıf, arayüzlerin her biri tarafından bildirilen işlevleri de tanımlamalıdır.

IInterface1 = interface
    ['{A2437023-7606-4551-8D5A-1709212254AF}']
    procedure Method1();
    function Method2(): Boolean;
end;

IInterface2 = interface
    ['{6C47FF48-3943-4B53-8D5D-537F4A0DEC0D}']
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject;

    property Value: TObject read GetValue write SetValue;
end;

TImplementer = class(TInterfacedObject, IInterface1, IInterface2)
    // IInterface1
    procedure Method1();
    function Method2(): Boolean;

    // IInterface2
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject

    property Value: TObject read GetValue write SetValue;
end;

Arayüzler için kalıtım #

Arayüzler, tıpkı sınıfların yaptığı gibi, birbirlerinden miras alabilir. Bir uygulayıcı sınıf, bu nedenle, arabirimin ve tüm temel arabirimlerin işlevlerini uygulamak zorundadır. Ancak bu şekilde derleyici, implenting sınıfının temel arabirimi de uyguladığını bilmez, yalnızca açıkça listelenen arabirimleri bilir. Bu nedenle ‘TImplementer’ üzerinde ‘ISüperInterface olarak’ kullanmak işe yaramaz. Bu aynı zamanda, tüm temel arayüzleri de açıkça uygulamak için yaygın bir uygulama ile sonuçlanır (bu durumda TImplementer = class(TInterfacedObject, IDescendantInterface, ISuperInterface)).

ISuperInterface = interface
    ['{A2437023-7606-4551-8D5A-1709212254AF}']
    procedure Method1();
    function Method2(): Boolean;
end;

IDescendantInterface = interface(ISuperInterface)
    ['{6C47FF48-3943-4B53-8D5D-537F4A0DEC0D}']
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject;

    property Value: TObject read GetValue write SetValue;
end;

TImplementer = class(TInterfacedObject, IDescendantInterface)
    // ISuperInterface
    procedure Method1();
    function Method2(): Boolean;

    // IDescendantInterface
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject

    property Value: TObject read GetValue write SetValue;
end;
