HDFS’nin ve nasıl çalıştığının iyi bir açıklaması.

Sözdizimi, HDFS’de kullanılabilecek komutları içermelidir.

HDFS’de dosya bulma #

Hadoop Dağıtılmış dosya sisteminde bir dosya bulmak için:

hdfs dfs -ls -R / | grep [search_term]

Yukarıdaki komutta,

-ls dosyaları listelemek içindir

‘-R’ özyinelemeli (alt dizinler arasında yineleme) içindir

/ kök dizinden anlamına gelir

| ilk komutun çıktısını ikinciye aktarmak için

eşleşen dizeleri çıkarmak için grep komutu

hadoop dosya sistemindeki tüm dosyalar listesinde aranacak [search_term] dosya adı.

Alternatif olarak aşağıdaki komut da bazı ifadeleri bul ve uygula olarak kullanılabilir:

hadoop fs -find / -name testi -print

Belirtilen ifadeyle eşleşen tüm dosyaları bulur ve seçilen eylemleri bunlara uygular. Herhangi bir yol belirtilmemişse, varsayılan olarak geçerli çalışma dizinine geçer. Herhangi bir ifade belirtilmemişse, varsayılan olarak -print olur.

Aşağıdaki birincil ifadeler tanınır:

  • “isim kalıbı”
  • “isim kalıbı”

Dosyanın temel adı, standart dosya sistemi küreselleştirmesini kullanan kalıpla eşleşirse doğru olarak değerlendirilir. -iname kullanılırsa, eşleşme büyük/küçük harfe duyarsızdır.

  • “yazdır”
  • print0Always

Doğru olarak değerlendirir. Geçerli yol adının standart çıktıya yazılmasına neden olur. “-print0” ifadesi kullanılırsa, bir ASCII NULL karakteri eklenir.

Aşağıdaki operatörler tanınır:

expression -a expression
expression -and expression
expression expression

HDFS - Hadoop Dağıtılmış Dosya Sistemi #

Hadoop Dağıtılmış Dosya Sistemi (HDFS), büyük ticari sunucu kümelerini kapsayacak şekilde tasarlanmış, ölçeklenebilir ve güvenilir veri depolaması sağlayan Java tabanlı bir dosya sistemidir. HDFS, MapReduce ve YARN, Apache™ Hadoop®’un özünü oluşturur.

HDFS, belirli bir veri bloğunun birden çok kopyasının (varsayılan olarak 3) birden çok düğümde kaydedilmesiyle elde edilen, hataya son derece dayanıklı olacak şekilde tasarlanmıştır.

HDFS’yi Bloklar ve Böller #

  1. HDFS’de Blok Boyutu ve Bloklar : HDFS, bir dosya yüklendiğinde verileri bloklar halinde saklama konseptine sahiptir. Bloklar, HDFS’deki (veya bu konuda başka herhangi bir dosya sistemindeki) verilerin fiziksel bölümleridir.

    Whenever a file is loaded onto the HDFS, it is splitted physically (yes, the file is divided) into different parts known as blocks. The number of blocks depend upon the value of dfs.block.size in hdfs-site.xml

    Ideally, the block size is set to a large value such as 64/128/256 MBs (as compared to 4KBs in normal FS). The default block size value on most distributions of Hadoop 2.x is 128 MB. The reason for a higher block size is because Hadoop is made to deal with PetaBytes of data with each file ranging from a few hundred MegaBytes to the order of TeraBytes.

    Say for example you have a file of size 1024 MBs. if your block size is 128 MB, you will get 8 blocks of 128MB each. This means that your namenode will need to store metadata of 8 x 3 = 24 files (3 being the replication factor).

    Consider the same scenario with a block size of 4 KBs. It will result in 1GB / 4KB = 250000 blocks and that will require the namenode to save the metadata for 750000 blocks for just a 1GB file. Since all these metadata related information is stored in-memory, larger block size is preferred to save that bit of extra load on the NameNode.

    Now again, the block size is not set to an extremely high value like 1GB etc because, ideally, 1 mapper is launched for each block of data. So if you set the block size to 1GB, you might lose parallelism which might result in a slower throughput overall.

2.) HDFS’de Bölünmüş Boyut : Splits in Hadoop Processing are the logical chunks of data. When files are divided into blocks, hadoop doesn’t respect any file bopundaries. It just splits the data depending on the block size. Say if you have a file of 400MB, with 4 lines, and each line having 100MB of data, you will get 3 blocks of 128 MB x 3 and 16 MB x 1. But when input splits are calculated while the prceossing of data, file/record boundaries are kept in mind and in this case we will have 4 input splits of 100 MB each, if you are using, say, NLineInputFormat.

Bölünmüş Boyut ayrıca mapreduce.input.fileinputformat.split.maxsize özelliği kullanılarak iş başına ayarlanabilir.
