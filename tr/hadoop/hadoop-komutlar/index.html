Sözdizimi #

  • Hadoop v1 komutları: hadoop fs -<komut>

  • Hadoop v2 komutları: hdfs dfs -<komut>

Hadoop v1 Komutları #

1. Hadoop sürümünü yazdırın #

hadoop version

2. HDFS’deki kök dizinin içeriğini listeleyin #

#

hadoop fs -ls /

3. Kullanılan alan miktarını bildirin ve #

şu anda takılı dosya sisteminde mevcut #

#

hadoop fs -df hdfs:/

4. Altındaki dizinlerin, dosyaların ve baytların sayısını sayın. #

belirtilen dosya kalıbıyla eşleşen yollar #

#

hadoop fs -count hdfs:/

5. Bir DFS dosya sistemi denetleme yardımcı programı çalıştırın #

#

hadoop fsck – /

6. Bir küme dengeleme yardımcı programı çalıştırın #

#

hadoop balancer

7. Altında “hadoop” adında yeni bir dizin oluşturun. #

/user/training dizini HDFS’de. sen olduğundan beri #

şu anda “eğitim” kullanıcı kimliğiyle oturum açmış, #

/user/training, HDFS’deki ana dizininizdir. #

#

hadoop fs -mkdir /user/training/hadoop

8. Yerel dizinden örnek bir metin dosyası ekleyin #

HDFS’de oluşturduğunuz yeni dizine “veri” adlı #

önceki adımda. #

#

hadoop fs -put data/sample.txt /user/training/hadoop

9. Bu yeni dizinin içeriğini HDFS’de listeleyin. #

#

hadoop fs -ls /user/training/hadoop

10. “Perakende” adlı yerel dizinin tamamını #

/user/training dizini HDFS’de. #

#

hadoop fs -put data/retail /user/training/hadoop

11. /user/training, HDFS’deki ana dizininiz olduğundan, #

mutlak yolu olmayan herhangi bir komut #

bu dizine göre yorumlanır. Sonraki #

komutu bu nedenle ana dizininizi listeler ve #

orada yeni eklediğiniz öğeleri göstermelidir. #

#

hadoop fs -ls

12. Bu dizinin HDFS’de ne kadar yer kapladığını görün. #

#

hadoop fs -du -s -h hadoop/retail

13. “Perakende” dizininden bir “müşteriler” dosyasını silin. #

#

hadoop fs -rm hadoop/retail/customers

14. Bu dosyanın artık HDFS’de olmadığından emin olun. #

#

hadoop fs -ls hadoop/retail/customers

15. Bir joker karakter kullanarak “perakende” dizinindeki tüm dosyaları silin. #

#

hadoop fs -rm hadoop/retail/*

16. Çöpü boşaltmak için #

#

hadoop fs -expunge

17. Son olarak, tüm perakende dizinini kaldırın ve #

içeriğinin HDFS’deki içeriği. #

#

hadoop fs -rm -r hadoop/retail

18. Hadoop dizinini tekrar listeleyin #

#

hadoop fs -ls hadoop

19. Yerel dizinden satın almalar.txt dosyasını ekleyin #

HDFS’de oluşturduğunuz hadoop dizinine “/home/training/” adlı #

#

hadoop fs -copyFromLocal /home/training/purchases.txt hadoop/

20. Satın almalar.txt metin dosyanızın içeriğini görüntülemek için #

hadoop dizininizde bulunan. #

#

hadoop fs -cat hadoop/purchases.txt

21. HDFS dizininde bulunan “hadoop” dizininden satın almalar.txt dosyasını ekleyin #

yerel dizininizde bulunan “veri” dizinine #

#

hadoop fs -copyToLocal hadoop/purchases.txt /home/training/data

22. cp, HDFS’de bulunan dizinler arasında dosya kopyalamak için kullanılır #

#

hadoop fs -cp /user/training/*.txt /user/training/hadoop

23. ‘-get’ komutu, ‘-copyToLocal’ komutuna alternatif olarak kullanılabilir. #

#

hadoop fs -get hadoop/sample.txt /home/training/

24. “purchases.txt” dosyasının son kilobaytını stdout’a görüntüleyin. #

#

hadoop fs -tail hadoop/purchases.txt

25. HDFS’de varsayılan dosya izinleri 666’dır. #

Bir dosyanın izinlerini değiştirmek için ‘-chmod’ komutunu kullanın #

#

hadoop fs -ls hadoop/purchases.txt
sudo -u hdfs hadoop fs -chmod 600 hadoop/purchases.txt

26. Sahip ve grubun varsayılan adları eğitim, eğitimdir. #

Sahip adını ve grup adını aynı anda değiştirmek için ‘-chown’ kullanın #

#

hadoop fs -ls hadoop/purchases.txt
sudo -u hdfs hadoop fs -chown root:root hadoop/purchases.txt

27. Grubun varsayılan adı eğitimdir #

Grup adını değiştirmek için ‘-chgrp’ komutunu kullanın #

#

hadoop fs -ls hadoop/purchases.txt
sudo -u hdfs hadoop fs -chgrp training hadoop/purchases.txt

28. Bir dizini bir konumdan diğerine taşıyın #

#

hadoop fs -mv hadoop apache_hadoop

29. Bir dosyaya varsayılan çoğaltma faktörü 3’tür. #

Bir dosyanın çoğaltma faktörünü değiştirmek için ‘-setrep’ komutunu kullanın #

#

hadoop fs -setrep -w 2 apache_hadoop/sample.txt

30. Kümedeki bir düğümden diğerine bir dizini kopyalayın #

Kopyalamak için ‘-distcp’ komutunu kullanın, #

-mevcut bir dosyanın üzerine yazmak için üzerine yazma seçeneği #

-her iki dizini de senkronize etmek için güncelleme komutu #

#

hadoop fs -distcp hdfs://namenodeA/apache_hadoop hdfs://namenodeB/hadoop

31. Ad düğümünün güvenli moddan çıkmasını sağlayan komut #

#

hadoop fs -expunge
sudo -u hdfs hdfs dfsadmin -safemode leave

32. Tüm hadoop dosya sistemi kabuk komutlarını listeleyin #

#

hadoop fs

33. hdfs kota değerlerini ve kullanımdaki mevcut ad ve bayt sayısını alın. #

#

hadoop fs -count -q [-h] [-v] <directory>...<directory>

34. Son olarak, her zaman yardım isteyin! #

#

hadoop fs -help

Hadoop v2 Komutları #

appendToFile: Yerel dosya sisteminden hedef dosya sistemine tek kaynak veya birden çok kaynak ekleyin. Ayrıca stdin’den gelen girişi okur ve hedef dosya sistemine ekler. dosyasını - olarak tut

 hdfs dfs -appendToFile [localfile1 localfile2 ..] [/HDFS/FILE/PATH..]

cat: Kaynak yollarını stdout’a kopyalar.

 hdfs dfs -cat URI [URI …]

chgrp: Dosyaların grup ilişkilendirmesini değiştirir. -R ile dizin yapısı üzerinden yinelemeli olarak değişiklik yapar. Kullanıcı, dosya sahibi veya süper kullanıcı olmalıdır.

 hdfs dfs -chgrp [-R] GROUP URI [URI …]

chmod: Dosyaların izinlerini değiştirir. -R ile dizin yapısı üzerinden yinelemeli olarak değişiklik yapar. Kullanıcı, dosya sahibi veya süper kullanıcı olmalıdır

 hdfs dfs -chmod [-R] <MODE[,MODE]... | OCTALMODE> URI [URI …]

chown: Dosyaların sahibini değiştirir. -R ile dizin yapısı üzerinden yinelemeli olarak değişiklik yapar. Kullanıcı süper kullanıcı olmalıdır.

 hdfs dfs -chown [-R] [OWNER][:[GROUP]] URI [URI ]

copyFromLocal: Kaynağın yerel bir dosya referansıyla sınırlandırılması dışında put komutuna benzer şekilde çalışır.

 hdfs dfs -copyFromLocal <localsrc> URI

copyToLocal: Hedefin yerel bir dosya referansıyla sınırlandırılması dışında get komutuna benzer şekilde çalışır.

 hdfs dfs -copyToLocal [-ignorecrc] [-crc] URI <localdst>

count: Belirtilen dosya modeliyle eşleşen yolların altındaki dizin, dosya ve bayt sayısını sayar.

 hdfs dfs -count [-q] [-h] <paths>

cp: Belirtilen bir kaynaktan belirli bir hedefe bir veya daha fazla dosyayı kopyalar. Birden çok kaynak belirtirseniz, belirtilen hedef bir dizin olmalıdır.

 hdfs dfs -cp URI [URI …] <dest>

du: Belirtilen dosyanın boyutunu veya belirtilen dizinde bulunan dosya ve dizinlerin boyutlarını görüntüler. -s seçeneğini belirtirseniz, tek tek dosya boyutları yerine dosya boyutlarının toplu bir özetini görüntüler. -h seçeneğini belirtirseniz, dosya boyutlarını “insan tarafından okunabilir” bir şekilde biçimlendirir.

 hdfs dfs -du [-s] [-h] URI [URI …]

dus: Dosya boyutlarının bir özetini görüntüler; hdfs dfs -du –s’ye eşdeğerdir.

 hdfs dfs -dus <args>

expunge: Çöp kutusunu boşaltır. Bir dosyayı sildiğinizde, dosya HDFS’den hemen kaldırılmaz, ancak /trash dizinindeki bir dosya olarak yeniden adlandırılır. Dosya orada kaldığı sürece, fikrinizi değiştirirseniz silme işlemini geri alabilirsiniz, ancak silinen dosyanın yalnızca en son kopyası geri yüklenebilir.

 hdfs dfs –expunge

get: Dosyaları yerel dosya sistemine kopyalar. -ignorecrc seçeneğini belirtirseniz, döngüsel artıklık denetiminde (CRC) başarısız olan dosyalar yine de kopyalanabilir. CRC, veri iletim hatalarını tespit etmek için yaygın bir tekniktir. CRC sağlama toplamı dosyaları .crc uzantısına sahiptir ve başka bir dosyanın veri bütünlüğünü doğrulamak için kullanılır. -crc seçeneğini belirtirseniz bu dosyalar kopyalanır.

 hdfs dfs -get [-ignorecrc] [-crc] <src> <localdst>

getmerge: src içindeki dosyaları birleştirir ve sonucu belirtilen yerel hedef dosyaya yazar. Her dosyanın sonuna yeni satır karakteri eklemek için addnl seçeneğini belirtin.

 hdfs dfs -getmerge <src> <localdst> [addnl]

ls: Belirtilen dosyalar veya dizinler için istatistikleri döndürür.

 hdfs dfs -ls <args>

lsr: ls’nin özyinelemeli sürümü olarak hizmet eder; Unix komutu ls -R’ye benzer.

 hdfs dfs -lsr <args>

mkdir: Belirtilen bir veya daha fazla yol üzerinde dizinler oluşturur. Davranışı, zaten yoksa belirtilen dizine giden tüm dizinleri oluşturan Unix mkdir -p komutuna benzer.

 hdfs dfs -mkdir <paths>

moveFromLocal: Kaynağın kopyalandıktan sonra silinmesi dışında put komutuna benzer şekilde çalışır.

 hdfs dfs -moveFromLocal <localsrc> <dest>

mv: Belirtilen bir kaynaktan belirli bir hedefe bir veya daha fazla dosyayı taşır. Birden çok kaynak belirtirseniz, belirtilen hedef bir dizin olmalıdır. Dosya sistemleri arasında dosya taşımaya izin verilmez.

 hdfs dfs -mv URI [URI …] <dest>

put: Dosyaları yerel dosya sisteminden hedef dosya sistemine kopyalar. Bu komut ayrıca stdin’den girdi okuyabilir ve hedef dosya sistemine yazabilir.

 hdfs dfs -put <localsrc> ... <dest>

rm: Belirtilen bir veya daha fazla dosyayı siler. Bu komut boş dizinleri veya dosyaları silmez. Çöp kutusunu atlamak (etkinleştirilmişse) ve belirtilen dosyaları hemen silmek için -skipTrash seçeneğini belirtin.

 hdfs dfs -rm [-skipTrash] URI [URI …]

rm r: –rm’nin özyinelemeli versiyonu olarak hizmet eder.

 hdfs dfs -rm -r [-skipTrash] URI [URI …]

setrep: Belirtilen bir dosya veya dizin için çoğaltma faktörünü değiştirir. -R ile dizin yapısı üzerinden yinelemeli olarak değişiklik yapar.

 hdfs dfs -setrep <rep> [-R] <path>

stat: Belirtilen yol hakkındaki bilgileri görüntüler.

 hdfs dfs -stat URI [URI …]

kuyruk: Belirtilen bir dosyanın stdout’a son kilobaytını görüntüler. Sözdizimi, belirtilen dosyanın izlenmesini sağlayan Unix -f seçeneğini destekler. Dosyaya başka bir işlem tarafından yeni satırlar eklendikçe tail, ekranı günceller.

 hdfs dfs -tail [-f] URI

test: Belirtilen dosya veya dizinin özniteliklerini döndürür. Dosyanın mı yoksa dizinin mi var olduğunu belirlemek için -e’yi belirtir; -z dosyanın mı yoksa dizinin mi boş olduğunu belirlemek için; ve -d URI’nin bir dizin olup olmadığını belirlemek için.

 hdfs dfs -test -[ezd] URI

metin: Belirtilen bir kaynak dosyanın çıktısını metin biçiminde verir. Geçerli girdi dosyası biçimleri zip ve TextRecordInputStream’dir.

 hdfs dfs -text <src>

touchz: Belirtilen yolda 0 boyutunda yeni, boş bir dosya oluşturur.

 hdfs dfs -touchz <path>
