Selenium WebDriver’da Bekleme Türleri #

Herhangi bir web uygulamasını çalıştırırken yükleme süresini dikkate almak gerekir. Kodunuz henüz yüklenmemiş herhangi bir öğeye erişmeye çalışırsa, WebDriver bir istisna atar ve betiğiniz durur.

Üç tür Bekleme vardır -

  • Örtülü Beklemeler
  • Açık Beklemeler
  • Akıcı Beklemeler

Açık beklemeler yalnızca belirli bölümlerde kullanılırken, program boyunca bekleme süresini ayarlamak için örtük beklemeler kullanılır.

örtük bekle #

Örtük bir bekleme, WebDriver’a, bir öğeyi veya öğeleri hemen mevcut değilse, bulmaya çalışırken belirli bir süre boyunca DOM’yi yoklamasını söylemektir. Örtülü beklemeler, temelde WebDriver’a, WebDriver’ın aradığı belirtilen web öğesinin mevcut olup olmadığını görmek istediğiniz gecikmeyi söyleme yönteminizdir. Varsayılan ayar 0’dır. Bir kez ayarlandığında, örtük bekleme, WebDriver nesne örneğinin ömrü için ayarlanır. Örtük bekleme, aşağıdaki kod parçası kullanılarak kodun örnekleme bölümünde bildirildi.

Java‘daki örnek:

driver.manage().timeouts().implicitlyWait(15, TimeUnit.SECONDS);
// You need to import the following class - import java.util.concurrent.TimeUnit;

C# içindeki örnek:

driver.Manage().Timeouts().ImplicitlyWait(TimeSpan.FromSeconds(15));

Bu durumda, WebDriver’a, belirtilen öğenin UI’de (DOM) bulunmadığı durumlarda 15 saniye beklemesi gerektiğini söylüyorsunuz.

açık bekleme #

Bazı öğelerin yüklenmesinin daha uzun sürdüğü durumlarla karşılaşabilirsiniz. Bu gibi durumlar için örtük beklemeyi ayarlamak mantıklı değildir, çünkü tarayıcı her öğe için gereksiz yere aynı süreyi bekleyecek ve otomasyon süresini artıracaktır. Açık bekleme, bazı belirli öğeler için örtük beklemeyi tamamen atlayarak burada yardımcı olur.

Açık beklemeler, belirli bir web öğesiyle sınırlı olan akıllı beklemelerdir. Açık beklemeleri kullanarak, temel olarak WebDriver’a pes etmeden önce X birim zaman beklemek olduğunu söylüyorsunuz.

Açık beklemeler, WebDriverWait ve ExpectedConditions sınıfları kullanılarak yapılır. Aşağıdaki örnekte, bir sonraki komuta geçmeden önce kullanıcı adı olan bir elemanın görünür hale gelmesi için 10 saniye kadar bekleyeceğiz. İşte adımlar.

Java‘daki örnek:

//Import these two packages:
import org.openqa.selenium.support.ui.ExpectedConditions;
import org.openqa.selenium.support.ui.WebDriverWait;

//Declare a WebDriverWait variable. In this example, we will use myWaitVar as the name of the variable.
WebDriverWait myWaitVar = new WebDriverWait(driver, 30);

//Use myWaitVar with ExpectedConditions on portions where you need the explicit wait to occur. In this case, we will use explicit wait on the username input before we type the text tutorial onto it.
myWaitVar.until(ExpectedConditions.visibilityOfElementLocated(By.id(“username”)));
driver.findElement(By.id(“username”)).sendKeys(“tutorial”);

BeklenenKoşullar sınıfı, bir öğeyi beklemek için önceden tanımlanmış bazı ortak koşullara sahiptir. Java bağlamasında bu koşulların listesini görmek için buraya tıklayın.

C# içindeki örnek:

using OpenQA.Selenium;
using OpenQA.Selenium.Support.UI;
using OpenQA.Selenium.PhantomJS;

// You can use any other WebDriver you want, such as ChromeDriver.
using (var driver = new PhantomJSDriver())
{
    driver.Navigate().GoToUrl("http://somedomain/url_that_delays_loading");

    // We aren't going to use it more than once, so no need to declare this a variable.
    new WebDriverWait(driver, TimeSpan.FromSeconds(10))
        .Until(ExpectedConditions.ElementIsVisible(By.Id("element-id")));

    // After the element is detected by the previous Wait, 
    // it will display the element's text
    Console.WriteLine(driver.FindElement(By.Id("element-id")).Text);
}

Bu örnekte sistem, eleman görünene kadar 10 saniye bekleyecektir. Öğe zaman aşımından sonra görünmezse, WebDriver bir `WebDriverTimeoutException’ atacaktır.

Lütfen dikkat: Öğe 10 saniyelik zaman aşımından önce görünürse, sistem hemen sonraki işlemlere geçecektir.

akıcı bekleyiş #

Örtük ve açık beklemenin aksine, akıcı bekleme iki parametre kullanır. Zaman aşımı değeri ve yoklama sıklığı. Diyelim ki 30 saniye zaman aşımı değeri ve 2 saniye yoklama frekansımız var. WebDriver, zaman aşımı değerine (30 saniye) kadar her 2 saniyede bir öğeyi kontrol edecektir. Herhangi bir sonuç olmadan zaman aşımı değeri aşıldıktan sonra istisna atılır. Aşağıda, akıcı beklemenin uygulanmasını gösteren örnek bir kod verilmiştir.

Java‘daki örnek:

Wait wait = new FluentWait(driver).withTimeout(30, SECONDS).pollingEvery(2, SECONDS).ignoring(NoSuchElementException.class);


WebElement testElement = wait.until(new Function() {
    public WebElement apply(WebDriver driver) {
        return driver.findElement(By.id("testId"));
    }
});

Akıcı beklemeyi kullanmanın bir başka avantajı da, beklerken belirli istisna türlerini (Örn. NoSuchElementExceptions) göz ardı edebilmemizdir. Tüm bu hükümler nedeniyle, akıcı bekleme, AJAX uygulamalarında ve ayrıca eleman yükleme süresinin sık sık dalgalandığı senaryolarda yardımcı olur. Akıcı beklemenin stratejik kullanımı, otomasyon çabalarını önemli ölçüde geliştirir.

Farklı türde açık bekleme koşulları #

Açık beklemede, bir koşulun olmasını beklersiniz. Örneğin, bir öğe tıklanabilir olana kadar beklemek istiyorsunuz.

İşte birkaç yaygın problemin bir gösterimi.

Lütfen dikkat: Bu örneklerin tümünde, “sınıf adı”, “xpath”, “bağlantı metni”, “etiket adı” veya “cssSelector” gibi herhangi bir “By"yi konum belirleyici olarak kullanabilirsiniz

Öğe görünene kadar bekleyin #

Örneğin, web sitenizin yüklenmesi biraz zaman alıyorsa, sayfanın yüklenmesi tamamlanana ve öğeniz Web Sürücüsü tarafından görünene kadar bekleyebilirsiniz.

C#

WebDriverWait wait = new WebDriverWait(driver, TimeSpan.FromSeconds(10));
wait.Until(ExpectedConditions.ElementIsVisible(By.Id("element-id")));

Java

WebDriverWait wait = new WebDriverWait(driver, 10);
wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("element-id")));

Öğe artık görünmeyene kadar bekleyin #

Daha önce olduğu gibi, ancak tersine döndü.

C#

WebDriverWait wait = new WebDriverWait(driver, TimeSpan.FromSeconds(10));
wait.Until(ExpectedConditions.InvisibilityOfElementLocated(By.Id("element-id")));

Java

WebDriverWait wait = new WebDriverWait(driver, 10);
wait.until(ExpectedConditions.invisibilityOfElementLocated(By.id("element-id")));

Belirtilen öğede metin bulunana kadar bekleyin

C#

IWebElement element = driver.FindElement(By.Id("element-id"));

WebDriverWait wait = new WebDriverWait(driver, TimeSpan.FromSeconds(10));
wait.Until(ExpectedConditions.TextToBePresentInElement(element, "text"));

Java

WebElement element = driver.findElement(By.id("element-id"));

WebDriverWait wait = new WebDriverWait(driver, 10);
wait.until(ExpectedConditions.textToBePresentInElement(element, "text"));

Yukarıda verilen bağlantıya giderseniz, orada tüm bekleme koşullarını göreceksiniz.

Bu bekleme koşullarının kullanımı arasındaki fark, giriş parametrelerindedir.

Bu, giriş parametresi WebElement ise WebElement’i geçmeniz gerektiği anlamına gelir, giriş parametresi olarak By bulucu alırsa öğe bulucuyu geçmeniz gerekir.

Ne tür bir bekleme koşulu kullanmak istediğinizi akıllıca seçin.

Ajax İsteklerinin Tamamlanmasını Bekliyor #

C# #

using OpenQA.Selenium
using OpenQA.Selenium.Chrome;
using System.Threading;

namespace WebDriver Tests
{
    class WebDriverWaits
    {
        static void Main()
        {
            IWebDriver driver = new ChromeDriver(@"C:\WebDriver");
            
            driver.Navigate().GoToUrl("page with ajax requests");
            CheckPageIsLoaded(driver);

            // Now the page is fully loaded, you can continue with further tests.
        }

        private void CheckPageIsLoaded(IWebDriver driver)
        {
            while (true)
            {
                bool ajaxIsComplete = (bool)(driver as IJavaScriptExecutor).ExecuteScript("return jQuery.active == 0");
                if (ajaxIsComplete)
                    return;
                Thread.Sleep(100);
            }
        }
    }
}

Bu örnek, ajax isteklerinin yapıldığı sayfalar için kullanışlıdır, burada kendi JavaScript kodumuzu çalıştırmak için IJavaScriptExecutor kullanıyoruz. Bir ‘while’ döngüsü içinde olduğu için, ‘ajaxIsComplete == true’ olana kadar çalışmaya devam edecek ve dönüş ifadesi çalıştırılacaktır.

“jQuery.active"in “0"a eşit olduğunu onaylayarak tüm ajax isteklerinin tamamlandığını kontrol ederiz. Bu işe yarar, çünkü her yeni ajax isteği yapıldığında ‘jQuery.active’ artırılır ve her istek onu tamamladığında azaltılır, bundan ‘jQuery.active == 0’ olduğunda tüm ajax isteklerinin tamamlanması gerektiği sonucunu çıkarabiliriz.

Akıcı Bekleme #

Akıcı bekleme, bekleme işlevine bir argüman kabul edebildiğinden daha yapılandırılabilir olan açık beklemenin (WebDriverWait) bir üst sınıfıdır. Bunu önlemek için bir en iyi uygulama olduğu için örtük beklemeyi ileteceğim.

Kullanım (Java):

Wait wait = new FluentWait<>(this.driver)
        .withTimeout(driverTimeoutSeconds, TimeUnit.SECONDS)
        .pollingEvery(500, TimeUnit.MILLISECONDS)
        .ignoring(StaleElementReferenceException.class)
        .ignoring(NoSuchElementException.class)
        .ignoring(ElementNotVisibleException.class);

WebElement foo = wait.until(ExpectedConditions.presenceOfElementLocated(By.yourBy));

// or use your own predicate:
WebElement foo = wait.until(new Function() {
  public WebElement apply(WebDriver driver) {
    return element.getText().length() > 0;
  }
});

Açık bekleme varsayılanlarıyla birlikte kullandığınızda, bu yalnızca varsayılanları: ‘DEFAULT_SLEEP_TIMEOUT = 500;’ olan ve ‘NotFoundException’ yok sayılan bir ‘FluentWait’ olur.

2: http://grepcode.com/file/repo1.maven.org/maven2/org.seleniumhq.selenium/selenium-support/2.47.1/org/openqa/selenium/support/ui/WebDriverWait.java# WebSürücüsüBekle

Akıcı bekleme #

Her FluentWait örneği, bir koşul için beklenecek maksimum sürenin yanı sıra koşulun kontrol edilme sıklığını tanımlar. Ayrıca, kullanıcı, bekleme sırasında sayfada bir öğe ararken NoSuchElementExceptions gibi belirli istisna türlerini yoksaymak için beklemeyi yapılandırabilir. Bu, sürücüyle ilişkilidir.

Wait<WebDriver> wait = new FluentWait<WebDriver>(driver)
.withTimeout(30, SECONDS) //actuall wait for the element to pe present
.pollingEvery(5, SECONDS) //selenium will keep looking for the element after every 5seconds
.ignoring(NoSuchElementException.class); //while ignoring this condition
wait.until(ExpectedConditions.visibilityOf(driver.findElement(By.id("username"));
