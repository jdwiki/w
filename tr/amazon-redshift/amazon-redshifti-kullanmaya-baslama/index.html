Ön Koşulları Ayarlama #

Ön Koşulların Ayarlanması, AWS hesabına Kaydolmayı ve SQL Workbench/J gibi SQL İstemci Sürücülerini ve Araçlarını yüklemeyi içerir

SQL Workbench J’yi kurmak ve yapılandırmak için:

  • Tüm sistemler için genel paketi indirmek için bu bağlantıyı kullanın (sql-workbench) . Tüm önkoşullarının kurulu olduğunu varsayarsak.
  • Uygun JDBC/ODBC Sürücüsünü alın.
  • Kurulum dizinindeki komut dosyasını kullanarak SQL-Workbench’i başlatın.
  • Bağlantı Penceresini Aç > Sürücüleri Yönet > Gözat ve sürücü dosyasını seçin.
  • Bağlantı Profili Seç Penceresinde. Sürücü kutusunda oluşturulan sürücüyü seçin. Oluşturulan kırmızıya kayma kümesindeki URL’yi (Amazon Redshift konsolundan JDBC URL’si) ve ana kullanıcı Kullanıcı Adı , Parola’yı yapıştırın.

Örnek Bir Amazon Redshift Kümesi Başlatın #

  • Amazon Redshift Dashboard’dan Kümeyi Başlat’a tıklayın
  • Küme ayrıntıları sayfasında herhangi bir Küme Tanımlayıcı, Veritabanı Adı, Veritabanı Bağlantı Noktası seçin (Küme oluşturulduktan sonra bağlantı noktası numarasını değiştiremeyeceğiniz için güvenlik duvarınızda açık bir bağlantı noktası seçin.), Ana Kullanıcı Adı, Ana Kullanıcı Parolası. Düğüm Yapılandırma sayfasında Sürüm (Kırmızıya Kaydırma Sürümü), Düğüm Türü , Küme Türü ve Hesaplama Düğümü Sayısını seçin.
  • Seçtiğiniz EC2-VPC veya EC2 Classic platformuna bağlı olarak, kümenizi yetkilendirmek için güvenlik adımları değişiklik gösterir. Geri kalan sayfalar için şimdilik varsayılan ayarları kullanabilirsiniz.

Kümeyi Yetkilendir #

EC2-VPC Platformu için navigasyon panelinden Redshift cluster sekmesini açtıktan sonra oluşturulan cluster adına tıklayın ve konfigürasyon sekmesine gidin. Küme özelliklerinde güvenlik grubunu seçin. Gelen ve Giden Sekmelerinden gelen ve giden kurallarını (Protokol, Port Aralığı ,Kaynak) gereksinimlerinize göre düzenleyin.

EC2-Classic Platform için navigasyon panelinden Redshift cluster sekmesini açtıktan sonra oluşturulan cluster adına tıklayın ve konfigürasyon sekmesine gidin. Küme Güvenlik Grupları için Küme Özellikleri altında varsayılanı seçin. Ardından, küme güvenlik grubu listesindeki Güvenlik Grupları sekmesinden küme güvenlik grubunu seçin. Güvenlik grubu bağlantıları sekmesindeki bağlantı türünden CIDR/IP’yi seçin ve bir IP/Port ile yetkilendirin.

var asyncjs = require('async');
hsqldb.reserve(function(err, connObj) {
    if (connObj) {
        console.log("Connection: " + connObj.uuid);
        var conn = connObj.conn;
        asyncjs.series([
            function(callback) {
                conn.createStatement(function(err, statement) {
                    if (err) {
                        callback(err);
                    } else {
                        statement.setFetchSize(100, function(err) {
                            if (err) {
                                callback(err);
                            } else {
                              statement.executeQuery("SELECT * FROM test.sample_data", function(err, resultset) {
                                resultset.toObjArray(function(err,sresults){
                                  console.log(sresults);
                                });   
                              });   
                           }
                        })
                      }
                    })
                 }
              ]) 
         }
      })
