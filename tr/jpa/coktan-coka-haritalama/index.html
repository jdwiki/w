Bir “ManyToMany” eşlemesi, her ikisinin de birbirinin birden fazla örneğiyle ilişkili olabileceği varlıklar arasındaki ilişkiyi tanımlar ve “@ManyToMany” ek açıklamasıyla tanımlanır.

Varlık tablosundaki bir yabancı anahtar sütununun kullanılabileceği ‘@OneToMany’den farklı olarak, ‘ManyToMany’, varlıkları birbirine eşleyen bir birleştirme tablosu gerektirir.

Parametreler #

Açıklama Amaç
@TableGenerator GeneratedValue ek açıklaması için bir oluşturucu öğesi belirtildiğinde adıyla başvurulabilecek bir birincil anahtar oluşturucu tanımlar
@GeneratedValue Birincil anahtarların değerleri için üretim stratejilerinin belirtilmesini sağlar. Id ek açıklamasıyla birlikte bir varlığın veya eşlenmiş üst sınıfın birincil anahtar özelliğine veya alanına uygulanabilir.
@ManyToMany Çalışan ve Proje varlıkları arasındaki ilişkiyi, birçok çalışanın birden çok projede çalışabileceği şekilde belirtir.
mappedBy="projeler" Çalışan ve Proje
@JoinColumn Dernek sahibi olarak kabul edilecek Varlığa atıfta bulunacak sütunun adını belirtir
@JoinTable Veritabanında, çalışanı yabancı anahtarlar kullanarak proje ilişkilerine tutacak tabloyu belirtir
  • Otomatik kimlik için @TableGenerator ve @GeneratedValue kullanılır jpa tablo oluşturucu kullanarak oluşturma.
  • @ManyToMany ek açıklaması, Çalışan arasındaki ilişkiyi belirtir ve Proje varlıkları.
  • @JoinTable, birleştirme tablosu jpa olarak kullanılacak tablonun adını belirtir Çalışan ve Proje arasında isim kullanarak çoktan çoğa eşleme = “çalışan_projesi”. Bu yapılır çünkü belirlemenin bir yolu yoktur veritabanı tablolarının yaptığı gibi bir jpa’nın birçok eşlemeye sahip olması diğer tabloya başvurmak için yabancı anahtarlar içermez.
  • @JoinColumn, aşağıdakilere atıfta bulunacak sütunun adını belirtir. Derneğin sahibi olarak kabul edilecek kişi, @inverseJoinColumn, ters tarafının adını belirtir ilişki. (Sahip olarak kabul edilmek için herhangi bir tarafı seçebilirsiniz. Sadece bu tarafların ilişkide olduğundan emin olun). Bizim durumumuzda var Sahibi olarak Çalışan seçildi, bu nedenle @JoinColumn idemployee’yi ifade eder birleştirme tablosundaki çalışan_proje sütunu ve @InverseJoinColumn başvuruyor jpa’nın ters tarafı olan idproject’e birçok eşleme.
  • Proje varlığındaki @ManyToMany ek açıklaması ters ilişkiyi gösterir bu nedenle, Çalışan içindeki alana atıfta bulunmak için mappedBy=projects kullanır. varlık.

Çalışanın Çoktan Çoka Haritalama Projelendirmesi #

Çalışan varlığı.

package com.thejavageek.jpa.entities;

import java.util.List;

import javax.persistence.CascadeType;
import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;
import javax.persistence.JoinColumn;
import javax.persistence.JoinTable;
import javax.persistence.ManyToMany;
import javax.persistence.TableGenerator;

@Entity
public class Employee {

    @TableGenerator(name = "employee_gen", table = "id_gen", pkColumnName = "gen_name", valueColumnName = "gen_val", allocationSize = 100)
    @Id
    @GeneratedValue(strategy = GenerationType.TABLE, generator = "employee_gen")
    private int idemployee;
    private String name;

    @ManyToMany(cascade = CascadeType.PERSIST)
    @JoinTable(name = "employee_project", joinColumns = @JoinColumn(name = "idemployee"), inverseJoinColumns = @JoinColumn(name = "idproject"))
    private List<Project> projects;

    public int getIdemployee() {
        return idemployee;
    }

    public void setIdemployee(int idemployee) {
        this.idemployee = idemployee;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public List<Project> getProjects() {
        return projects;
    }

    public void setProjects(List<Project> projects) {
        this.projects = projects;
    }

}

Proje Varlığı:

package com.thejavageek.jpa.entities;

import java.util.List;

import javax.persistence.CascadeType;
import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;
import javax.persistence.ManyToMany;
import javax.persistence.TableGenerator;

@Entity
public class Project {

    @TableGenerator(name = "project_gen", allocationSize = 1, pkColumnName = "gen_name", valueColumnName = "gen_val", table = "id_gen")
    @Id
    @GeneratedValue(generator = "project_gen", strategy = GenerationType.TABLE)
    private int idproject;
    private String name;

    @ManyToMany(mappedBy = "projects", cascade = CascadeType.PERSIST)
    private List<Employee> employees;

    public int getIdproject() {
        return idproject;
    }

    public void setIdproject(int idproject) {
        this.idproject = idproject;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public List<Employee> getEmployees() {
        return employees;
    }

    public void setEmployees(List<Employee> employees) {
        this.employees = employees;
    }

}

Katıştırılabilir ek açıklama olmadan bileşik anahtar nasıl kullanılır? #

Eğer varsa

| roleId | name | discription |

Yukarıdaki senaryoda “rightrole” tablosu bileşik anahtara sahiptir ve buna JPA’da erişmek için kullanıcının “Gömülü” ek açıklamalı varlık oluşturması gerekir.

Gömülü açıklama tek nesne için iyidir ancak toplu kayıtlar eklenirken sorun çıkaracaktır.

Sorun, kullanıcı “haklar” ile yeni bir “rol” oluşturmak istediğinde, ilk kullanıcının “depolama (kalıcı)” “rol” nesnesini ve ardından kullanıcının rol için yeni oluşturulan “kimliği” almak için “yıkama” yapması gerekir. sonra ve sonra kullanıcı onu ‘doğru rol’ varlığının nesnesine koyabilir.

Bu kullanıcıyı çözmek için varlığı biraz farklı bir şekilde yazabilir.

Rol tablosu için varlık:

@Entity
@Table(name = "role")
public class Role {

    private static final long serialVersionUID = 1L;

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Basic(optional = false)
    @NotNull
    @Column(nullable = false)
    private Long roleID;

    
    @OneToMany(cascade = CascadeType.ALL, mappedBy = "role", fetch = FetchType.LAZY)
    private List<RightRole> rightRoleList;
 
    @ManyToMany(cascade = {CascadeType.PERSIST})
    @JoinTable(name = "rightrole",
            joinColumns = {
                @JoinColumn(name = "roleID", referencedColumnName = "ROLE_ID")},
            inverseJoinColumns = {
                @JoinColumn(name = "rightID", referencedColumnName = "RIGHT_ID")})
    private List<Right> rightList;
.......
}

@JoinTable ek açıklaması, bir varlık olmadan bile “rightrole” tablosuna eklenmesiyle ilgilenecektir (bu tablonun yalnızca rol ve sağ kimlik sütunlarına sahip olduğu sürece).

Kullanıcı daha sonra basitçe şunları yapabilir:

Role role = new  Role();
List<Right> rightList = new ArrayList<>();
Right right1 = new Right();
Right right2 = new Right();
rightList.add(right1);
rightList.add(right2);
role.setRightList(rightList);

inverseJoinColumns’a @ManyToMany(cascade = {CascadeType.PERSIST}) yazmanız gerekir, aksi takdirde çocuk silinirse ebeveyn verileriniz silinir.
