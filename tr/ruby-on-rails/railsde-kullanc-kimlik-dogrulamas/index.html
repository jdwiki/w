Devise çok güçlü bir mücevherdir, kurulumdan hemen sonra kaydolmanıza, oturum açmanıza ve oturum kapatmanıza olanak tanır. Ayrıca kullanıcı, uygulamalarına kimlik doğrulama ve kısıtlamalar ekleyebilir. Kullanıcı kullanmak isterse, Devise ayrıca kendi görünümleriyle birlikte gelir. Bir kullanıcı ayrıca kayıt olma ve oturum açma formlarını ihtiyaca ve gereksinime göre özelleştirebilir. Devise’in, Rails konusunda yeniyseniz, kendi oturum açma bilgilerinizi uygulamanızı önerdiği unutulmamalıdır.

“Rails create devise:install” kullanılarak aygıt yapılandırmaları oluşturulurken aygıt, terminalde izlenecek bir dizi talimatı listeler.

Halihazırda bir ‘KULLANICI’ modeliniz varsa, ‘raylar oluşturucu KULLANICI’ komutunu çalıştırmak, mevcut ‘KULLANICI’ modelinize gerekli sütunları ekleyecektir.

“Kullanıcı"nın oturum açıp açmadığını kontrol etmek için denetleyicinizin üst kısmındaki “before_action :authenticate_user!” yardımcı yöntemini kullanın. değilse, oturum açma sayfasına yönlendirileceklerdir.

Devise kullanarak kimlik doğrulama #

Gemfile’e gem ekleyin:

`mücevher ’tasarlamak’

Ardından “paket yükleme” komutunu çalıştırın.

Gerekli yapılandırma dosyasını oluşturmak için $ Rails create devise:install komutunu kullanın.

Her bir ortamda Devise mailer için varsayılan URL seçeneklerini ayarlayın Geliştirme ortamında şu satırı ekleyin:

config.action_mailer.default_url_options = { host: 'localhost', port: 3000 }

config/environments/development.rb dosyanıza

benzer şekilde üretimde bu config/environments/production.rb dosyasını düzenleyin ve ekleyin

config.action_mailer.default_url_options = { host: 'your-site-url'}

Ardından, şunu kullanarak bir model oluşturun: `$ Rails, USER oluşturur. “KULLANICI”, kimlik doğrulamasını uygulamak istediğiniz sınıf adıdır.

Son olarak, “rake db:migrate” komutunu çalıştırın ve hazırsınız.

Özel görünümler #

Görünümlerinizi yapılandırmanız gerekiyorsa, tüm görünümleri uygulamanıza kopyalayacak olan ‘rails create devise:views’ oluşturucusunu kullanabilirsiniz. Ardından bunları istediğiniz gibi düzenleyebilirsiniz.

Uygulamanızda birden fazla Devise modeliniz varsa (örneğin Kullanıcı ve Yönetici), Devise’ın tüm modeller için aynı görünümleri kullandığını fark edeceksiniz. Devise, görünümleri özelleştirmek için kolay bir yol sunar. “config/initializers/devise.rb” dosyasının içinde “config.scoped_views = true” değerini ayarlayın.

Kapsamlı görünümler oluşturmak için oluşturucuyu da kullanabilirsiniz: ‘raylar aygıt oluşturur: görünüm kullanıcıları’

Kaydedilebilir ve onaylanabilir modül için olanlar gibi yalnızca birkaç görünüm kümesi oluşturmak istiyorsanız, -v işaretini kullanın: rails oluşturma aygıt:görünümler -v kayıt onayları

Denetleyici Filtreleri ve Yardımcıları Tasarlayın #

Devise kullanarak kullanıcı kimlik doğrulamalı bir denetleyici ayarlamak için, bunu Before_action: (cihaz modelinizin ‘Kullanıcı’ olduğunu varsayarak) ekleyin:

before_action :authenticate_user!

Bir kullanıcının oturum açıp açmadığını doğrulamak için aşağıdaki yardımcıyı kullanın:

user_signed_in?

Mevcut oturum açmış kullanıcı için bu yardımcıyı kullanın:

“current_user”

Bu kapsam için oturuma erişebilirsiniz:

“kullanıcı_oturum”

  • Devise modelinizin adı “Kullanıcı” yerine “Üye” ise, yukarıdaki “kullanıcı"yı “üye” ile değiştirin.

Omniauth #

Önce yetkilendirme stratejinizi seçin ve onu Gemfile dosyanıza ekleyin. Burada stratejilerin bir listesini bulabilirsiniz: https://github.com/intridea/omniauth/wiki/List-of-Strategies

gem 'omniauth-github', :github => 'intridea/omniauth-github'
gem 'omniauth-openid', :github => 'intridea/omniauth-openid'

Bunu Rails ara katman yazılımınıza şu şekilde ekleyebilirsiniz:

Rails.application.config.middleware.use OmniAuth::Builder do
  require 'openid/store/filesystem' 
  provider :github, ENV['GITHUB_KEY'], ENV['GITHUB_SECRET']
  provider :openid, :store => OpenID::Store::Filesystem.new('/tmp')
end

Varsayılan olarak, OmniAuth rotalarınıza /auth/:provider ekler ve bu yolları kullanarak başlayabilirsiniz.

Varsayılan olarak, bir hata varsa, omniauth /auth/failure konumuna yönlendirilecektir.

has_secure_password #

Kullanıcı Modeli Oluştur #

rails model Kullanıcı e-postası oluşturur:string password_digest:string

has_secure_password modülünü Kullanıcı modeline ekleyin #

class User < ActiveRecord::Base
  has_secure_password
end

Artık şifre ile yeni bir kullanıcı oluşturabilirsiniz.

user = User.new email: 'bob@bob.com', password: 'Password1', password_confirmation: 'Password1'

Kimlik doğrulama yöntemiyle şifreyi doğrulayın

user.authenticate('somepassword')

has_secure_token #

Kullanıcı Modeli Oluştur

# Schema: User(token:string, auth_token:string)
class User < ActiveRecord::Base
  has_secure_token
  has_secure_token :auth_token
end

Artık yeni bir kullanıcı oluşturduğunuzda, bir jeton ve auth_token otomatik olarak oluşturulur

user = User.new
user.save
user.token # => "pX27zsMN2ViQKta1bGfLmVJE"
user.auth_token # => "77TMHrHJFvFDwodq8w7Ev2m7"

“regenerate_token” ve “regenerate_auth_token” kullanarak belirteçleri güncelleyebilirsiniz.

user.regenerate_token # => true
user.regenerate_auth_token # => true
