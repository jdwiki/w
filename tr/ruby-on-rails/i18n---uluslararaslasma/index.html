Sözdizimi #

  • I18n.t(“anahtar”)
  • I18n.translate(“anahtar”) # I18n.t("anahtar") ile eşdeğer
  • I18n.t(“anahtar”, say: 4)
  • I18n.t(“anahtar”, param1: “Bir şey”, param2: “Başka”)
  • I18n.t(“yok_var”, varsayılan: “anahtar”) # anahtar eksikse bir varsayılan belirtin
  • I18n.locale #=> :tr
  • I18n.locale = :tr
  • I18n.default_local #=> :tr
  • I18n.default_locale = :tr
  • t(".key") # I18n.t("key") ile aynı, ancak çağrıldığı eylem/şablonun kapsamı

argümanlarla I18n #

Parametreleri I18n t yöntemine iletebilirsiniz:

# Example config/locales/en.yml
en:
  page:
    users: "%{users_count} users currently online"

# In models, controller, etc...
I18n.t('page.users', users_count: 12)

# In views

# ERB
<%= t('page.users', users_count: 12) %>

#SLIM
= t('page.users', users_count: 12)

# Shortcut in views - DRY!
# Use only the dot notation
# Important: Consider you have the following controller and view page#users

# ERB Example app/views/page/users.html.erb
<%= t('.users', users_count: 12) %>

Ve aşağıdaki çıktıyı alın:

"12 users currently online"

ActiveRecord model niteliklerini çevirme #

globalize gem, ActiveRecord modellerinize çeviri eklemek için harika bir çözümdür. Bunu Gemfile dosyanıza ekleyerek yükleyebilirsiniz:

gem 'globalize', '~> 5.0.0'

“Rails 5” kullanıyorsanız, “activemodel-serializers-xml” eklemeniz de gerekir.

gem 'activemodel-serializers-xml'

Model çevirileri, modellerinizin öznitelik değerlerini çevirmenize olanak tanır, örneğin:

class Post < ActiveRecord::Base
  translates :title, :text
end

I18n.locale = :en
post.title # => Globalize rocks!

I18n.locale = :he
post.title # => גלובאלייז2 שולט!

Çevrilmesi gereken model niteliklerinizi tanımladıktan sonra, geçiş yoluyla bir çeviri tablosu oluşturmanız gerekir. globalize, create_translation_table! ve drop_translation_table! sağlar.

Bu geçiş için ‘yukarı’ ve ‘aşağı’ kullanmanız ve değil ‘değiştirmemeniz’ gerekir. Ayrıca, bu geçişi başarılı bir şekilde çalıştırmak için, yukarıda gösterildiği gibi, önce modelinizde çevrilmiş öznitelikleri tanımlamanız gerekir. Önceki “Post” modeli için uygun bir geçiş şudur:

class CreatePostsTranslationTable < ActiveRecord::Migration
  def up
    Post.create_translation_table! title: :string, text: :text
  end

  def down
    Post.drop_translation_table!
  end
end

Aşağıdakiler gibi belirli seçenekler için seçenekleri de iletebilirsiniz:

class CreatePostsTranslationTable < ActiveRecord::Migration
  def up
    Post.create_translation_table! title: :string,
      text: { type: :text, null: false, default: "Default text" }
  end

  def down
    Post.drop_translation_table!
  end
end

Çeviri ihtiyacı sütunlarınızda zaten mevcut verileriniz varsa, geçişinizi ayarlayarak bunları kolayca çeviriler tablosuna taşıyabilirsiniz:

class CreatePostsTranslationTable < ActiveRecord::Migration
  def up
    Post.create_translation_table!({
      title: :string,
      text: :text
    }, {
      migrate_data: true
    })
      
  end

  def down
    Post.drop_translation_table! migrate_data: true
  end
end

Bütün verileriniz güvenli bir şekilde taşındıktan sonra çevrilmiş sütunları üst tablodan çıkardığınızdan emin olun. Veri geçişinden sonra çevrilen sütunları üst tablodan otomatik olarak kaldırmak için, geçişe “remove_source_columns” seçeneğini ekleyin:

class CreatePostsTranslationTable < ActiveRecord::Migration
  def up
    Post.create_translation_table!({
      title: :string,
      text: :text
    }, {
      migrate_data: true,
      remove_source_columns: true
    })
      
  end

  def down
    Post.drop_translation_table! migrate_data: true
  end
end

Ayrıca önceden oluşturulmuş bir çeviri tablosuna yeni alanlar ekleyebilirsiniz:

class Post < ActiveRecord::Base
  # Remember to add your attribute here too.
  translates :title, :text, :author
end

class AddAuthorToPost < ActiveRecord::Migration
  def up
    Post.add_translation_fields! author: :text
  end

  def down
    remove_column :post_translations, :author
  end
end

Çoğullaştırma #

I18n’nin sizin için çoğullaştırmayı işlemesine izin verebilirsiniz, sadece ‘count’ argümanını kullanın.

Yerel ayar dosyanızı şu şekilde ayarlamanız gerekir:

# config/locales/en.yml
en:
  online_users:
    one: "1 user is online"
    other: "%{count} users are online"

Ardından, “count” argümanını “I18n.t” yardımcısına ileterek az önce oluşturduğunuz anahtarı kullanın:

I18n.t("online_users", count: 1)
#=> "1 user is online"

I18n.t("online_users", count: 4)
#=> "4 users are online"

İstekler aracılığıyla yerel ayarı ayarlayın #

Çoğu durumda, I18n yerel ayarı yapmak isteyebilirsiniz. Geçerli oturum, geçerli kullanıcı veya bir URL parametresine dayalı olarak yerel ayarı ayarlamak isteyebilirsiniz. Bu, denetleyicilerinizden birinde bir “before_action” veya tüm kontrolörler.

class ApplicationController < ActionController::Base
  before_action :set_locale

  protected

  def set_locale
    # Remove inappropriate/unnecessary ones
    I18n.locale = params[:locale] ||    # Request parameter
      session[:locale] ||               # Current session
      (current_user.preferred_locale if user_signed_in?) ||  # Model saved configuration
      extract_locale_from_accept_language_header ||          # Language header - browser config
      I18n.default_locale               # Set in your config files, english by super-default
  end

  # Extract language from request header
  def extract_locale_from_accept_language_header
    if request.env['HTTP_ACCEPT_LANGUAGE']
      lg = request.env['HTTP_ACCEPT_LANGUAGE'].scan(/^[a-z]{2}/).first.to_sym
      lg.in?([:en, YOUR_AVAILABLE_LANGUAGES]) ? lg : nil
    end
  end

URL tabanlı #

“locale” parametresi bunun gibi bir URL’den gelebilir

http://yourapplication.com/products?locale=en

Veya

http://yourapplication.com/en/products

İkincisini elde etmek için, bir “kapsam” ekleyerek “rotalarınızı” düzenlemeniz gerekir:

# config/routes.rb
scope "(:locale)", locale: /en|fr/ do
  resources :products
end

Bunu yaparak, “http://yourapplication.com/en/products” adresini ziyaret ettiğinizde yerel ayarınız “:en” olarak ayarlanır. Bunun yerine, “http://yourapplication.com/fr/products” adresini ziyaret etmek, onu “:fr” olarak ayarlayacaktır. Ayrıca, “:locale” parametresini kaçırdığınızda bir yönlendirme hatası almazsınız, çünkü “http://yourapplication.com/products” ziyareti varsayılan I18n yerel ayarını yükleyecektir.

Oturum tabanlı veya kalıcılık tabanlı #

Bu, kullanıcının dili değiştirmek için bir düğmeye/dil bayrağına tıklayabileceğini varsayar. Eylem, oturumu geçerli dile ayarlayan bir denetleyiciye yönlendirilebilir (ve sonunda, kullanıcı bağlıysa değişiklikleri bir veritabanında sürdürür)

class SetLanguageController < ApplicationController
  skip_before_filter :authenticate_user!
  after_action :set_preferred_locale

  # Generic version to handle a large list of languages
  def change_locale
    I18n.locale = sanitize_language_param
    set_session_and_redirect
  end

Sanitize_language_param’ı mevcut diller listenizle tanımlamanız ve sonunda dilin mevcut olmaması durumunda hataları halletmeniz gerekir.

Çok az diliniz varsa, bunları şu şekilde tanımlamanız faydalı olabilir:

def fr
  I18n.locale = :fr
  set_session_and_redirect
end

def en
  I18n.locale = :en
  set_session_and_redirect
end

private

  def set_session_and_redirect
    session[:locale] = I18n.locale
    redirect_to :back
  end

  def set_preferred_locale
    if user_signed_in?
      current_user.preferred_locale = I18n.locale.to_s
      current_user.save if current_user.changed?
    end
  end
end

Not: “dil değiştir” eylemlerinize bazı yollar eklemeyi unutmayın

Varsayılan Yerel Ayar #

Uygulamanızın varsayılan yerel ayarını yapmanız gerektiğini unutmayın. Bunu, “config/application.rb” içinde ayarlayarak yapabilirsiniz:

config.i18n.default_locale = :de

veya “config/initializers” klasöründe bir başlatıcı oluşturarak:

# config/initializers/locale.rb
I18n.default_locale = :it

HTTP isteğinden yerel ayarı alın #

Bazen uygulama yerel ayarını istek IP’sine göre ayarlamak faydalı olabilir. Bunu Geocoder kullanarak kolayca yapabilirsiniz. Geocoder‘ın yaptığı pek çok şey arasında, bir “isteğin” “konumunu” da söyleyebilir.

İlk olarak, Gemfile dosyanıza Geocoder ekleyin

# Gemfile
gem 'geocoder'

‘Geocoder’, standart ‘Rack::Request’ nesnesine ’location’ ve ‘safe_location’ yöntemlerini ekler, böylece herhangi bir HTTP isteğinin konumunu IP adresine göre kolayca arayabilirsiniz. Bu yöntemleri, “ApplicationController” içindeki bir “before_action” içinde kullanabilirsiniz:

class ApplicationController < ActionController::Base
  before_action :set_locale_from_request

  def set_locale_from_request
    country_code = request.location.data["country_code"] #=> "US"
    country_sym = country_code.underscore.to_sym #=> :us

    # If request locale is available use it, otherwise use I18n default locale
    if I18n.available_locales.include? country_sym
      I18n.locale = country_sym
    end
end

“0.0.0.0” ve “localhost” gibi şeyler geçerli geçerli İnternet IP adresleri olduğundan, bunun “geliştirme” ve “test” ortamlarında çalışmayacağını unutmayın.

Sınırlamalar ve alternatifler #

Geocoder çok güçlü ve esnektir, ancak bir coğrafi kodlama hizmeti ile çalışacak şekilde yapılandırılması gerekir (bkz. daha fazla ayrıntı); bunların çoğu kullanıma sınır koyar. Ayrıca, her istekte harici bir hizmeti aramanın performansı etkileyebileceğini de akılda tutmakta fayda var.

Bunları ele almak için şunları da dikkate almaya değer olabilir:

1. Çevrimdışı bir çözüm #

GeoIP gibi bir mücevher kullanmak ([buraya2 bakın) yerel bir veri dosyasına karşı arama yapılmasına izin verir. Bu veri dosyalarının güncel tutulması gerektiğinden doğruluk açısından bir ödünleşim olabilir.

2. CloudFlare’ı kullanın #

CloudFlare aracılığıyla sunulan sayfalar, başlığa (HTTP_CF_IPCOUNTRY) eklenen ülke koduyla birlikte şeffaf bir şekilde coğrafi olarak kodlanma seçeneğine sahiptir. Daha fazla ayrıntı burada bulunabilir.

I18n’yi HTML Etiketleri ve Sembolleriyle Kullanın #

# config/locales/en.yml
en:
  stackoverflow:
    header:
      title_html: "Use <strong>I18n</strong> with Tags &amp; Symbols"

“Başlık” adından sonra fazladan “_html” eklenmesine dikkat edin.

Ve Görünümlerde,

# ERB
<h2><%= t(:title_html, scope: [:stackoverflow, :header]) %></h2>

Görünümlerde I18n kullanın #

Bu YAML yerel ayar dosyasına sahip olduğunuzu varsayarsak:

# config/locales/en.yml
en:
  header:
    title: "My header title"

ve başlık dizinizi görüntülemek istiyorsanız, bunu yapabilirsiniz

# in ERB files
<%= t('header.title') %>

# in SLIM files
= t('header.title')
