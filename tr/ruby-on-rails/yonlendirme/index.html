Rails yönlendiricisi, URL’leri tanır ve bunları bir denetleyicinin eylemine gönderir. Ayrıca, görünümlerinizde dizeleri sabit kodlama ihtiyacını ortadan kaldırarak yollar ve URL’ler oluşturabilir.

Genel olarak “Yönlendirme”, URL’lerin uygulamanız tarafından nasıl “işlendiği"dir. Rails durumunda, genellikle hangi denetleyicinin ve bu denetleyicinin hangi eyleminin belirli bir gelen URL’yi ele alacağıdır. Rails uygulamalarında rotalar genellikle config/routes.rb dosyasına yerleştirilir.

Kaynak Yönlendirme (Temel) #

Rotalar config/routes.rb içinde tanımlanır. Bunlar genellikle resources veya source yöntemleri.

“resources :users”, tümü “UsersController” eylemleriyle eşlenen aşağıdaki yedi yolu oluşturur:

get       '/users',          to: 'users#index'
post      '/users',          to: 'users#create'
get       '/users/new',      to: 'users#new'
get       '/users/:id/edit', to: 'users#edit'
get       '/users/:id',      to: 'users#show'
patch/put '/users/:id',      to: 'users#update'
delete    '/users/:id',      to: 'users#destroy'

Eylem adları, yukarıdaki “to” parametresinde “#” işaretinden sonra gösterilir. Aynı ada sahip yöntemler, “app/controllers/users_controller.rb” içinde aşağıdaki gibi tanımlanmalıdır:

class UsersController < ApplicationController
  def index
  end

  def create
  end

  # continue with all the other methods…
end

Oluşturulan eylemleri “yalnızca” veya “hariç” ile sınırlayabilirsiniz:

resources :users, only:   [:show]
resources :users, except: [:show, :index]

Aşağıdakileri çalıştırarak uygulamanızın tüm rotalarını istediğiniz zaman görüntüleyebilirsiniz:

son sürüm 
$ rake routes
# OR
$ rails routes

          PATCH  /users/:id(.:format)      users#update
          PUT    /users/:id(.:format)      users#update
          DELETE /users/:id(.:format)      users#destroy

-g seçeneğini kullanarak rotalar arasında arama yapabilirsiniz. Bu, yardımcı yöntem adı, URL yolu veya HTTP fiiliyle kısmen eşleşen herhangi bir yolu gösterir:

$ rake routes -g new_user     # Matches helper method
$ rake routes -g POST         # Matches HTTP Verb POST
son sürüm

$ rake routes -g POST         # Matches HTTP Verb POST 
# OR
$ rails routes -g new_user    # Matches helper method
$ rails routes -g POST        # Matches HTTP Verb POST
son sürüm

Routes.rb’de ‘resources’ yerine ‘resource’ yöntemi kullanılarak rotalar yalnızca üyeler için (koleksiyonlar değil) kullanılabilir olarak bildirilebilir. “Kaynak” ile, varsayılan olarak bir “indeks” yolu oluşturulmaz, ancak yalnızca açıkça şöyle bir yol istendiğinde:

resource :orders, only: [:index, :create, :show]

Kısıtlamalar #

Kısıtlamaları kullanarak hangi rotaların mevcut olduğunu filtreleyebilirsiniz.

Aşağıdakiler dahil olmak üzere kısıtlamaları kullanmanın birkaç yolu vardır:

  • segment kısıtlamaları,
  • isteğe dayalı kısıtlamalar
  • gelişmiş kısıtlamalar

Örneğin, yalnızca belirli bir IP adresinin bir rotaya erişmesine izin vermek için istenen tabanlı bir kısıtlama:

constraints(ip: /127\.0\.0\.1$/) do
  get 'route', to: "controller#action"
end

Diğer benzer örneklere bakın ActionDispatch::Routing::Mapper::Scoping.

Daha karmaşık bir şey yapmak istiyorsanız, daha gelişmiş kısıtlamalar kullanabilir ve mantığı sarmak için bir sınıf oluşturabilirsiniz:

# lib/api_version_constraint.rb
class ApiVersionConstraint
  def initialize(version:, default:)
    @version = version
    @default = default
  end

  def version_header
    "application/vnd.my-app.v#{@version}"
  end

  def matches?(request)
    @default || request.headers["Accept"].include?(version_header)
  end
end

# config/routes.rb
require "api_version_constraint"

Rails.application.routes.draw do
  namespace :v1, constraints: ApiVersionConstraint.new(version: 1, default: true) do
    resources :users # Will route to app/controllers/v1/users_controller.rb
  end

  namespace :v2, constraints: ApiVersionConstraint.new(version: 2) do
    resources :users # Will route to app/controllers/v2/users_controller.rb
  end
end

Bir form, birkaç gönder düğmesi

Ayrıca, bir formun gönderme etiketlerinin değerini, farklı bir eyleme yönlendirmek için bir kısıtlama olarak kullanabilirsiniz. Birden fazla gönderme düğmesi olan bir formunuz varsa (örneğin, “önizleme” ve “gönder”), bu kısıtlamayı, form hedef URL’sini değiştirmek için javascript yazmak yerine doğrudan routes.rb dosyanızda yakalayabilirsiniz. Örneğin commit_param_routing gem ile, submit_tag raylarından yararlanabilirsiniz.

Rails ‘submit_tag’ ilk parametresi, form taahhüt parametrenizin değerini değiştirmenize izin verir

# app/views/orders/mass_order.html.erb
<%= form_for(@orders, url: mass_create_order_path do |f| %>
    <!-- Big form here -->
  <%= submit_tag "Preview" %>
  <%= submit_tag "Submit" %>
  # => <input name="commit" type="submit" value="Preview" />
  # => <input name="commit" type="submit" value="Submit" />
  ...
<% end %>

# config/routes.rb
resources :orders do
  # Both routes below describe the same POST URL, but route to different actions 
  post 'mass_order', on: :collection, as: 'mass_order',
    constraints: CommitParamRouting.new('Submit'), action: 'mass_create' # when the user presses "submit"
  post 'mass_order', on: :collection,
    constraints: CommitParamRouting.new('Preview'), action: 'mass_create_preview' # when the user presses "preview"
  # Note the `as:` is defined only once, since the path helper is mass_create_order_path for the form url
  # CommitParamRouting is just a class like ApiVersionContraint
end

Kapsam belirleme yolları #

Rails, rotalarınızı düzenlemek için çeşitli yollar sunar.

URL’ye göre kapsam:

scope 'admin' do
  get 'dashboard', to: 'administration#dashboard'
  resources 'employees'
end

Bu, aşağıdaki yolları oluşturur

get       '/admin/dashboard',          to: 'administration#dashboard'
post      '/admin/employees',          to: 'employees#create'
get       '/admin/employees/new',      to: 'employees#new'
get       '/admin/employees/:id/edit', to: 'employees#edit'
get       '/admin/employees/:id',      to: 'employees#show'
patch/put '/admin/employees/:id',      to: 'employees#update'
delete    '/admin/employees/:id',      to: 'employees#destroy'

Bazı görünümleri farklı bir alt klasörde tutmak, yönetici görünümlerini kullanıcı görünümlerinden ayırmak sunucu tarafında daha mantıklı olabilir.

Modele göre kapsam

scope module: :admin do
  get 'dashboard', to: 'administration#dashboard'
end

“modül”, verilen adın alt klasörünün altındaki denetleyici dosyalarını arar

get       '/dashboard',          to: 'admin/administration#dashboard'

Bir “as” parametresi ekleyerek yol yardımcıları önekini yeniden adlandırabilirsiniz.

scope 'admin', as: :administration do
  get 'dashboard'
end

# => administration_dashboard_path

Rails, ’namespace’ yöntemini kullanarak yukarıdakilerin tümünü yapmak için uygun bir yol sağlar. Aşağıdaki bildirimler eşdeğerdir

namespace :admin do
end

scope 'admin', module: :admin, as: :admin

Kontrolöre göre kapsam

scope controller: :management do
  get 'dashboard'
  get 'performance'
end

Bu, bu rotaları oluşturur

get       '/dashboard',          to: 'management#dashboard'
get       '/performance',        to: 'management#performance'

Sığ Yuvalama

Kaynak yolları, mümkün olduğunda URL’leri kısaltmaya yardımcı olan bir “:sığ” seçeneğini kabul eder. Kaynaklar birden fazla düzeyde iç içe geçmemelidir. Bundan kaçınmanın bir yolu sığ yollar oluşturmaktır. Amaç, ihtiyaç duyulmayan üst koleksiyon URL segmentlerini dışarıda bırakmaktır. Sonuç olarak, oluşturulan iç içe rotalar yalnızca ‘:index’ , ‘:create’ ve ‘:new’ eylemleri içindir. Gerisi kendi sığ URL bağlamında tutulur. Özel sığ rotalara kapsam için iki seçenek vardır:

  • :shallow_path: Belirtilen bir parametreyle üye yollarının önüne ekler

    scope shallow_path: "sekret" do
  resources :articles do
    resources :comments, shallow: true
  end
end

  • :shallow_prefix: Belirtilen parametreleri adlandırılmış yardımcılara ekleyin

    scope shallow_prefix: "sekret" do
  resources :articles do
    resources :comments, shallow: true
  end
end

Ayrıca “sığ” rotaları şu şekilde de gösterebiliriz:

resources :auctions, shallow: true do
  resources :bids do
   resources :comments
  end
end

alternatif olarak aşağıdaki gibi kodlanmıştır (eğer bloktan memnunsanız):

resources :auctions do
 shallow do
   resources :bids do
     resources :comments
   end
 end
end

yeni_açık artırma AL /açık artırmalar/yeni(.:format)

Oluşturulan rotaları dikkatli bir şekilde analiz ederseniz, URL’nin iç içe bölümlerinin yalnızca hangi verilerin görüntüleneceğini belirlemek için gerektiğinde dahil edildiğini fark edeceksiniz.

Endişeler #

İç içe rotalarda tekrardan kaçınmak için endişeler, yeniden kullanılabilir ortak kaynakları paylaşmanın harika bir yolunu sağlar. Bir endişe oluşturmak için, ‘routes.rb’ dosyası içindeki ’endişe’ yöntemini kullanın. Yöntem bir sembol ve blok bekler:

concern :commentable do
  resources :comments
end

Bu kod, kendisi herhangi bir rota oluşturmazken, bir kaynakta :endişeler özniteliğinin kullanılmasına izin verir. En basit örnek şöyle olacaktır:

resource :page, concerns: :commentable

Eşdeğer iç içe kaynak şöyle görünür:

resource :page do
  resource :comments
end

Bu, örneğin aşağıdaki yolları oluşturacaktır:

/pages/#{page_id}/comments
/pages/#{page_id}/comments/#{comment_id}

Endişelerin anlamlı olması için, endişeyi kullanan birden fazla kaynak olmalıdır. Ek kaynaklar, endişeyi çağırmak için aşağıdaki söz dizimlerinden herhangi birini kullanabilir:

resource :post, concerns: %i(commentable)
resource :blog do
  concerns :commentable
end

Kök yolu #

‘root’ yöntemiyle uygulamanıza bir ana sayfa rotası ekleyebilirsiniz.

# config/routes.rb
Rails.application.routes.draw do
  root "application#index"
  # equivalent to:
  # get "/", "application#index"  
end

# app/controllers/application_controller.rb
class ApplicationController < ActionController::Base
  def index
    render "homepage"
  end
end

Ve terminalde, ’tırmık yolları’ (Rails 5’teki ‘ray yolları’) şunları üretecektir:

root     GET    /         application#index

Ana sayfa genellikle en önemli rota olduğundan ve rotalar göründükleri sıraya göre önceliklendirildiğinden, ‘root’ rotası genellikle rota dosyanızdaki ilk rota olmalıdır.

Ek RESTful eylemler #

resources :photos do
  member do
    get 'preview'
  end
  collection do
    get 'dashboard'
  end
end

Bu, varsayılan 7 RESTful rotaya** ek olarak aşağıdaki rotaları oluşturur:

get       '/photos/:id/preview',          to: 'photos#preview'
get       '/photos/dashboards',           to: 'photos#dashboard'

Bunu tek satırlar için yapmak istiyorsanız, şunları kullanabilirsiniz:

resources :photos do
  get 'preview',   on: :member
  get 'dashboard', on: :collection
end

/new yoluna da bir eylem ekleyebilirsiniz:

resources :photos do
  get 'preview', on: :new
end

Hangisi yaratacak:

get       '/photos/new/preview',          to: 'photos#preview'

RESTful rotalarınıza eylemler eklerken dikkatli olun, muhtemelen başka bir kaynağı kaçırıyorsunuz!

Rotaları birden çok dosyaya ayırın #

Rota dosyanız çok büyükse, rotalarınızı birden fazla dosyaya koyabilir ve dosyaların her birini Ruby’nin require_relative yöntemiyle dahil edebilirsiniz:

config/routes.rb: #

YourAppName::Application.routes.draw do
  require_relative 'routes/admin_routes'
  require_relative 'routes/sidekiq_routes'
  require_relative 'routes/api_routes'
  require_relative 'routes/your_app_routes'
end

config/routes/api_routes.rb: #

YourAppName::Application.routes.draw do
  namespace :api do
    # ...
  end
end

Üye ve Toplama Yolları #

Bir kaynağın içinde bir üye bloğu tanımlamak, o kaynağa dayalı yolun tek bir üyesi üzerinde hareket edebilen bir yol oluşturur:

resources :posts do
  member do
    get 'preview'
  end
end

Bu, aşağıdaki üye yolunu oluşturur:

get '/posts/:id/preview', to: 'posts#preview'
# preview_post_path

Toplama rotaları, bir kaynak nesneleri koleksiyonu üzerinde işlem yapabilen rotalar oluşturmaya izin verir:

resources :posts do
  collection do
    get 'search'
  end
end

Bu, aşağıdaki toplama yolunu oluşturur:

get '/posts/search', to: 'posts#search'
# search_posts_path

Alternatif bir sözdizimi:

resources :posts do
  get 'preview', on: :member
  get 'search',  on: :collection
end

Yönlendirme #

Rails rotalarında yeniden yönlendirmeyi aşağıdaki gibi yapabilirsiniz:

get '/stories', to: redirect('/posts')
son sürüm 
match "/abc" => redirect("http://example.com/abc")
son sürüm

Ayrıca tüm bilinmeyen yolları belirli bir yola yönlendirebilirsiniz:

match '*path' => redirect('/'), via: :get
# or
get '*path' => redirect('/')
son sürüm 
match '*path' => redirect('/')
son sürüm

Başka bir uygulama bağlayın #

mount, başka bir uygulamayı (temelde raf uygulaması) monte etmek için kullanılır veya mevcut uygulamada kullanılacak motorları raylar

sözdizimi:

mount SomeRackApp, at: "some_route"

Artık, “some_rack_app_path” veya “some_rack_app_url” yol yardımcısını kullanarak yukarıdaki takılı uygulamaya erişebilirsiniz.

Ancak bu yardımcı adını yeniden adlandırmak istiyorsanız, bunu şu şekilde yapabilirsiniz:

mount SomeRackApp, at: "some_route", as: :myapp

Bu, bu takılı uygulamaya gitmek için kullanılabilecek “myapp_path” ve “myapp_url” yardımcılarını oluşturacaktır.

Kapsam mevcut yerel ayarlar #

Uygulamanız farklı dillerde mevcutsa, genellikle geçerli yerel ayarı URL’de gösterirsiniz.

scope '/(:locale)', locale: /#{I18n.available_locales.join('|')}/ do
    root 'example#root'
    # other routes
end

Kökünüze “I18n.available_locales” içinde tanımlanan yerel ayarlar aracılığıyla erişilebilir.

Yönlendirmeler ve Joker Karakterli Yollar #

Kullanıcınıza kolaylık sağlamak için bir URL sağlamak, ancak bunu doğrudan kullanmakta olduğunuz başka bir URL ile eşlemek istiyorsanız. Bir yönlendirme kullanın:

# config/routes.rb
TestApp::Application.routes.draw do
  get 'courses/:course_name' => redirect('/courses/%{course_name}/lessons'), :as => "course"
end

Bu hızla ilginçleşti. Buradaki temel ilke, bir rotayı başka bir rotaya göndermek için sadece ‘#redirect’ yöntemini kullanmaktır. Rotanız oldukça basitse, gerçekten basit bir yöntemdir. Ancak orijinal parametreleri de göndermek istiyorsanız, parametreyi %{burada} içine alarak biraz jimnastik yapmanız gerekir. Her şeyin etrafındaki tek tırnak işaretine dikkat edin.

Yukarıdaki örnekte, kolaylık olması için :as parametresiyle bir takma ad kullanarak yolu da yeniden adlandırdık. Bu, bu adı #_path yardımcıları gibi yöntemlerde kullanmamıza izin verir. Yine, sorularla ‘$ komisyon rotalarınızı’ test edin.

noktalı URL parametreleri #

Bir id numarasından daha karmaşık bir url parametresini desteklemek istiyorsanız, değer bir nokta içeriyorsa ayrıştırıcı ile sorun yaşayabilirsiniz. Bir noktadan sonra gelen her şeyin bir format olduğu varsayılacaktır (yani json, xml).

Kabul edilen girdiyi _genişletmek için bir kısıtlama kullanarak bu sınırlamayı çözebilirsiniz.

Örneğin, url’de bir kullanıcı kaydına e-posta adresiyle başvuruda bulunmak istiyorsanız:

resources :users, constraints: { id: /.*/ }
