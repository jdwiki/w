CanCan, Rails için kullanıcı rollerinden ayrıştırılmış basit bir yetkilendirme stratejisidir. Tüm izinler tek bir yerde saklanır.

CanCan’ı kullanmadan önce, gem ile veya manuel olarak Kullanıcılar oluşturmayı unutmayın. CanCan’ın maksimum işlevselliğini elde etmek için bir Yönetici kullanıcı oluşturun.

CanCan’ı kullanmaya başlama #

CanCan, belirli kaynaklara kullanıcı erişimini kısıtlayan Ruby on Rails için popüler bir yetkilendirme kitaplığıdır. En son mücevher (CanCanCan), ölü projenin CanCan devamıdır.

İzinler ‘Yetenek’ sınıfında tanımlanır ve denetleyicilerden, görünümlerden, yardımcılardan veya koddaki herhangi bir yerden kullanılabilir.

Bir uygulamaya yetkilendirme desteği eklemek için CanCanCan gem’i “Gemfile” dosyasına ekleyin:

gem 'cancancan'

Ardından yetenek sınıfını tanımlayın:

# app/models/ability.rb
class Ability
  include CanCan::Ability

  def initialize(user)
  end
end

Ardından, yetkili modelleri denetleyiciye yüklemek için “load_and_authorize_resource” kullanarak yetkilendirmeyi kontrol edin:

class ArticlesController < ApplicationController
  load_and_authorize_resource

  def show
    # @article is already loaded and authorized
  end
end

yetkilendirmeyi kontrol etmek veya bir istisna oluşturmak için yetkilendir!

def show
  @article = Article.find(params[:id])
  authorize! :read, @article
end

Bir nesnenin denetleyicilerde, görünümlerde veya yardımcılarda herhangi bir yerde belirli bir eyleme karşı yetkilendirilip yetkilendirilmediğini kontrol etmek için ‘can?’

<% if can? :update, @article %>
  <%= link_to "Edit", edit_article_path(@article) %>
<% end %>

Not: Bu, imzalı kullanıcının “current_user” yöntemiyle sağlandığını varsayar.

Çok sayıda yetenekle başa çıkma #

Yetenek tanımlarının sayısı artmaya başladığında, Yetenek dosyasını işlemek giderek daha zor hale gelir.

Bu sorunu ele almanın ilk stratejisi, şu örneğe göre yetenekleri anlamlı yöntemlere taşımaktır:

class Ability
  include CanCan::Ability

  def initialize(user)
    anyone_abilities 

    if user
      if user.admin?
        admin_abilities
      else
        authenticated_abilities
      end
    else
      guest_abilities
    end
  end

  private
  
  def anyone_abilities
    # define abilities for everyone, both logged users and visitors
  end

  def guest_abilities
    # define abilities for visitors only
  end

  def authenticated_abilities
    # define abilities for logged users only
  end

  def admin_abilities
    # define abilities for admins only
  end
end

Bu sınıf yeterince büyüdüğünde, aşağıdaki gibi farklı sorumlulukları yerine getirmek için onu farklı sınıflara ayırmayı deneyebilirsiniz:

# app/models/ability.rb
class Ability
  include CanCan::Ability

  def initialize(user)
    self.merge Abilities::Everyone.new(user)

    if user
      if user.admin?
        self.merge Abilities::Admin.new(user)
      else
        self.merge Abilities::Authenticated.new(user)
      end
    else
      self.merge Abilities::Guest.new(user)
    end
  end
end

ve sonra bu sınıfları şu şekilde tanımlayın:

# app/models/abilities/guest.rb
module Abilities
  class Guest
    include CanCan::Ability

    def initialize(user)
      # Abilities for anonymous visitors only
    end
  end
end

vb. “Yetenekler::Kimlik Doğrulandı”, “Yetenekler::Yönetici” veya başka herhangi biriyle.

Yetenekleri tanımlama #

Yetenekler, ‘can’ ve ‘cannot’ yöntemleri kullanılarak ‘Ability’ sınıfında tanımlanır. Temel referans için aşağıdaki yorumlanmış örneği göz önünde bulundurun:

class Ability
  include CanCan::Ability

  def initialize(user)
    # for any visitor or user
    can :read, Article

    if user
      if user.admin?
        # admins can do any action on any model or action
        can :manage, :all
      else
        # regular users can read all content
        can :read, :all
        # and edit, update and destroy their own user only
        can [:edit, :destroy], User, id: user_id
        # but cannot read hidden articles
        cannot :read, Article, hidden: true
      end
    else
      # only unlogged visitors can visit a sign_up page:
      can :read, :sign_up
    end
  end
end

Bir yeteneği hızlıca test edin #

Bir yetenek sınıfının doğru izinleri verip vermediğini hızlı bir şekilde test etmek istiyorsanız, konsolda veya yüklenen Rails ortamıyla başka bir bağlamda bir yeteneği başlatabilirsiniz, test etmek için bir kullanıcı örneğini iletmeniz yeterlidir:

test_ability = Ability.new(User.first)
test_ability.can?(:show, Post) #=> true
other_ability = Ability.new(RestrictedUser.first)
other_ability.cannot?(:show, Post) #=> true

Daha fazla bilgi: https://github.com/ryanb/cancan/wiki/Testing-Abilities
