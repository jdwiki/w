Bir özelliğin uzunluğunu doğrulama #

class Person < ApplicationRecord
  validates :name, length: { minimum: 2 }
  validates :bio, length: { maximum: 500 }
  validates :password, length: { in: 6..20 }
  validates :registration_number, length: { is: 6 }
end

Olası uzunluk kısıtlaması seçenekleri şunlardır:

  • :minimum - Nitelik, belirtilen uzunluktan daha az olamaz.
  • :maximum - Nitelik belirtilen uzunluktan fazla olamaz.
  • :in (veya :in) - Öznitelik uzunluğu belirli bir aralığa dahil edilmelidir. Bu seçeneğin değeri bir aralık olmalıdır.
  • :is - Öznitelik uzunluğu verilen değere eşit olmalıdır.

Bir özelliğin varlığını doğrulama #

Bu yardımcı, belirtilen özniteliklerin boş olmadığını doğrular.

class Person < ApplicationRecord
  validates :name, presence: true
end

Person.create(name: "John").valid? # => true
Person.create(name: nil).valid? # => false

Belirtilen özniteliklerin olmadığını doğrulamak için “yokluk” yardımcısını kullanabilirsiniz. Sıfır veya boş değerleri kontrol etmek için mevcut? yöntemini kullanır.

class Person < ApplicationRecord
  validates :name, :login, :email, absence: true
end

Not: Özniteliğin bir “boolean” olması durumunda, olağan mevcudiyet doğrulamasını kullanamazsınız (öznitelik, “yanlış” bir değer için geçerli olmaz). Bunu bir dahil etme doğrulaması kullanarak yapabilirsiniz:

validates :attribute, inclusion: [true, false]

Özel doğrulamalar #

‘ActiveModel::Validator’ veya ‘ActiveModel::EachValidator’dan devralınan yeni sınıflar ekleyerek kendi doğrulamalarınızı ekleyebilirsiniz. Her iki yöntem de benzerdir ancak biraz farklı şekillerde çalışırlar:

ActiveModel::Validator ve validates_with #

Bir kaydı argüman olarak alan ve üzerinde doğrulama yapan ‘validate’ yöntemini uygulayın. Ardından modeldeki sınıfla birlikte ‘validates_with’ kullanın.

# app/validators/starts_with_a_validator.rb
class StartsWithAValidator < ActiveModel::Validator
  def validate(record)
    unless record.name.starts_with? 'A'
      record.errors[:name] << 'Need a name starting with A please!'
    end
  end
end
 
class Person < ApplicationRecord
  validates_with StartsWithAValidator
end

ActiveModel::EachValidator ve validate #

Yeni doğrulayıcınızı tek bir parametre üzerinde ortak “validate” yöntemini kullanarak kullanmayı tercih ederseniz, “ActiveModel::EachValidator"dan miras alan bir sınıf oluşturun ve üç bağımsız değişken alan “validate_each” yöntemini uygulayın: “record”, “attribute” ve “değer”:

class EmailValidator < ActiveModel::EachValidator
  def validate_each(record, attribute, value)
    unless value =~ /\A([^@\s]+)@((?:[-a-z0-9]+\.)+[a-z]{2,})\z/i
      record.errors[attribute] << (options[:message] || 'is not an email')
    end
  end
end
 
class Person < ApplicationRecord
  validates :email, presence: true, email: true
end

Ray kılavuzları hakkında daha fazla bilgi.

Bir özelliğin biçimini doğrular #

“Format” ve “with” seçeneğini kullanarak bir özniteliğin değerinin normal bir ifadeyle eşleştiğini doğrulayın.

class User < ApplicationRecord
  validates :name, format: { with: /\A\w{6,10}\z/ }
end

Ayrıca bir sabit tanımlayabilir ve değerini normal bir ifadeye ayarlayabilir ve bunu with: seçeneğine iletebilirsiniz. Bu, gerçekten karmaşık düzenli ifadeler için daha uygun olabilir

PHONE_REGEX = /\A\(\d{3}\)\d{3}-\d{4}\z/
validates :phone, format: { with: PHONE_REGEX }

Varsayılan hata mesajı “geçersiz"dir. Bu, :message seçeneği ile değiştirilebilir.

validates :bio, format: { with: /\A\D+\z/, message: "Numbers are not allowed" }

Tersi de yanıt verir ve bir değerin ‘without:’ seçeneğiyle normal bir ifadeyle eşleşmemesi gerektiğini belirtebilirsiniz.

Bir özelliğin sayısallığını doğrulama #

Bu doğrulama, yalnızca sayısal değerlerin eklenmesini kısıtlar.

class Player < ApplicationRecord
  validates :points, numericality: true
  validates :games_played, numericality: { only_integer: true }
end

Bu yardımcı, “:only_integer"in yanı sıra, kabul edilebilir değerlere kısıtlamalar eklemek için aşağıdaki seçenekleri de kabul eder:

  • :greer_than - Değerin sağlanan değerden büyük olması gerektiğini belirtir. Bu seçenek için varsayılan hata mesajı “%{count} değerinden büyük olmalıdır” şeklindedir.
  • :greer_than_or_equal_to - Değerin sağlanan değerden büyük veya ona eşit olması gerektiğini belirtir. Bu seçenek için varsayılan hata mesajı “%{count} değerinden büyük veya buna eşit olmalıdır” şeklindedir.
  • :equal_to - Değerin sağlanan değere eşit olması gerektiğini belirtir. Bu seçenek için varsayılan hata mesajı “%{count}‘a eşit olmalıdır” şeklindedir.
  • :less_than - Değerin sağlanan değerden küçük olması gerektiğini belirtir. Bu seçenek için varsayılan hata mesajı “%{count}‘dan az olmalıdır” şeklindedir.
  • :less_than_or_equal_to - Değerin sağlanan değerden küçük veya ona eşit olması gerektiğini belirtir. Bu seçenek için varsayılan hata mesajı “%{count} değerinden küçük veya buna eşit olmalıdır” şeklindedir.
  • :other_than - Değerin sağlanan değerden farklı olması gerektiğini belirtir. Bu seçenek için varsayılan hata mesajı “%{count} dışında olmalıdır” şeklindedir.
  • :odd - true olarak ayarlanırsa değerin tek sayı olması gerektiğini belirtir. Bu seçenek için varsayılan hata mesajı “tek olmalı” şeklindedir.
  • :even - true olarak ayarlanırsa değerin çift sayı olması gerektiğini belirtir. Bu seçenek için varsayılan hata mesajı “eşit olmalıdır” şeklindedir.

Varsayılan olarak, sayısallık sıfır değerlere izin vermez. İzin vermek için allow_nil: true seçeneğini kullanabilirsiniz.

Gruplama doğrulaması #

Bir özelliğin dahil edildiğini doğrular #

inclusion: yardımcısını kullanarak bir diziye bir değerin dahil edilip edilmediğini kontrol edebilirsiniz. ‘:in’ seçeneği ve diğer adı ‘:in’, kabul edilebilir değerler kümesini gösterir.

class Country < ApplicationRecord
  validates :continent, inclusion: { in: %w(Africa Antartica Asia Australia
                                            Europe North America South America) }
end

Bir değerin bir diziye dahil edilip edilmediğini kontrol etmek için hariç tutma: yardımcısını kullanın

class User < ApplicationRecord
  validates :name, exclusion: { in: %w(admin administrator owner) }
end

Bir özelliğin benzersizliğini doğrulayın #

Bu yardımcı, nesne kaydedilmeden hemen önce özniteliğin değerinin benzersiz olduğunu doğrular.

class Account < ApplicationRecord
  validates :email, uniqueness: true
end

Benzersizlik kontrolünü sınırlamak için kullanılan bir veya daha fazla özniteliği belirtmek için kullanabileceğiniz bir “:scope” seçeneği vardır:

class Holiday < ApplicationRecord
  validates :name, uniqueness: { scope: :year,
    message: "should happen once per year" }
end

Ayrıca, benzersizlik kısıtlamasının büyük/küçük harfe duyarlı olup olmayacağını tanımlamak için kullanabileceğiniz bir “:büyük harfe duyarlı” seçeneği de vardır. Bu seçenek varsayılan olarak “doğru"dur.

class Person < ApplicationRecord
  validates :name, uniqueness: { case_sensitive: false }
end

Doğrulamaları Atlama #

Doğrulamaları atlamak istiyorsanız aşağıdaki yöntemleri kullanın. Bu yöntemler, geçersiz olsa bile nesneyi veritabanına kaydeder.

  • azalma!
  • azalan_sayac
  • artış!
  • artım_sayac
  • değiştir!
  • dokunmak
  • update_all
  • update_attribute
  • update_column
  • update_columns
  • update_counters

Ayrıca, “kaydetme” argümanını “kaydetme” argümanı olarak ileterek kaydederken doğrulamayı atlayabilirsiniz.

User.save(validate: false)

Koşullu doğrulama #

Bazen kaydı yalnızca belirli koşullar altında doğrulamanız gerekebilir.

class User < ApplicationRecord
  validates :name, presence: true, if: :admin? 

  def admin?
    conditional here that returns boolean value
  end
end

Koşullu gerçekten küçükseniz, bir Proc kullanabilirsiniz:

class User < ApplicationRecord
  validates :first_name, presence: true, if: Proc.new { |user| user.last_name.blank? }
end

Negatif koşul için “olmazsa” kullanabilirsiniz:

class User < ApplicationRecord
  validates :first_name, presence: true, unless: Proc.new { |user| user.last_name.present? }
end

“instance_eval” aracılığıyla yürütülecek bir dize de iletebilirsiniz:

class User < ApplicationRecord
  validates :first_name, presence: true, if: 'last_name.blank?'
end

Özelliğin onayı #

Tam olarak aynı içeriği alması gereken iki metin alanınız olduğunda bunu kullanmalısınız. Örneğin, bir e-posta adresini veya parolayı onaylamak isteyebilirsiniz. Bu doğrulama, adı “_confirmation” eklenerek onaylanması gereken alanın adı olan bir sanal özniteliği oluşturur.

class Person < ApplicationRecord
  validates :email, confirmation: true
end

Not Bu kontrol yalnızca “email_confirmation” sıfır değilse gerçekleştirilir.

Onay istemek için, onay özniteliği için bir mevcudiyet kontrolü eklediğinizden emin olun.

class Person < ApplicationRecord
  validates :email,      confirmation: true
  validates :email_confirmation, presence: true
end

Kaynak

:on seçeneğini kullanma #

:on seçeneği, doğrulamanın ne zaman gerçekleşeceğini belirtmenize izin verir. Tüm yerleşik doğrulama yardımcıları için varsayılan davranış, kaydetme sırasında çalıştırılmalıdır (hem yeni bir kayıt oluştururken hem de güncellerken).

class Person < ApplicationRecord
  # it will be possible to update email with a duplicated value
  validates :email, uniqueness: true, on: :create
 
  # it will be possible to create the record with a non-numerical age
  validates :age, numericality: true, on: :update
 
  # the default (validates on both create and update)
  validates :name, presence: true
end
