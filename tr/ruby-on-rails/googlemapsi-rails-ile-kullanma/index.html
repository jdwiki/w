google maps javascript etiketini düzen başlığına ekleyin #

Google haritalarının turbolinks ile düzgün çalışması için javascript etiketini bir görünüme eklemek yerine doğrudan düzen başlığına ekleyin.

# app/views/layouts/my_layout.html.haml
!!!
%html{:lang => 'en'}
  %head
    - # ...
    = google_maps_api_script_tag

“google_maps_api_script_tag” en iyi bir yardımcıda tanımlanır.

# app/helpers/google_maps_helper.rb
module GoogleMapsHelper
  def google_maps_api_script_tag
    javascript_include_tag google_maps_api_source
  end

  def google_maps_api_source
    "https://maps.googleapis.com/maps/api/js?key=#{google_maps_api_key}"
  end

  def google_maps_api_key
    Rails.application.secrets.google_maps_api_key
  end
end

Uygulamanızı google’a kaydettirebilir ve api anahtarınızı google api konsolunda alabilirsiniz. Google’da kısa bir google maps javascript api için bir api anahtarının nasıl isteneceği vardır.

API anahtarı secrets.yml dosyasında saklanır:

# config/secrets.yml
development:
  google_maps_api_key: '...'
  # ...
production:
  google_maps_api_key: '...'
  # ...

.gitignore dosyanıza config/secrets.yml dosyasını eklemeyi unutmayın ve api anahtarını depoya göndermediğinizden emin olun.

Modeli coğrafi olarak kodlayın #

Kullanıcılarınızın ve/veya gruplarınızın profilleri olduğunu ve bir google haritasında adres profili alanlarını görüntülemek istediğinizi varsayalım.

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  # Attributes: 
  # label, e.g. "Work address"
  # value, e.g. "Willy-Brandt-Straße 1\n10557 Berlin"
end

Adresleri coğrafi olarak kodlamanın, yani “boylam” ve “enlem” sağlamanın harika bir yolu geocoder gem.

Geocoder’ı ‘Gemfile’ dosyanıza ekleyin ve yüklemek için ‘bundle’ı çalıştırın.

# Gemfile
gem 'geocoder', '~> 1.3'

Konumu veritabanına kaydetmek için “enlem” ve “boylam” için veritabanı sütunları ekleyin. Bu, konuma her ihtiyaç duyduğunuzda coğrafi kodlama hizmetini sorgulamaktan daha verimlidir. Daha hızlıdır ve sorgu sınırına o kadar çabuk ulaşmıyorsunuz.

➜ bin/rails generate migration add_latitude_and_longitude_to_profile_fields \
    latitude:float longitude:float
➜ bin/rails db:migrate  # Rails 5, or:
➜ rake db:migrate       # Rails 3, 4

Modelinize coğrafi kodlama mekanizmasını ekleyin. Bu örnekte, adres dizgisi “değer” özniteliğinde saklanır. Coğrafi kodlamayı, kayıt değiştiğinde ve yalnızca bir değer mevcut olduğunda gerçekleştirilecek şekilde yapılandırın:

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  geocoded_by :value
  after_validation :geocode, if: ->(address_field){ 
    address_field.value.present? and address_field.value_changed? 
  }

end

Varsayılan olarak, geocoder arama hizmeti olarak google’ı kullanır. Mesafe hesaplamaları veya yakınlık araması gibi birçok ilginç özelliğe sahiptir. Daha fazla bilgi için geocoder README sayfasına bakın.

Profil görünümünde bir google haritasında adresleri göster #

Profil görünümünde, bir google haritasındaki adres alanlarının yanı sıra bir kullanıcı veya grubun profil alanlarını bir listede gösterin.

- # app/views/profiles/show.html.haml
%h1 Contact Information
.profile_fields
  = render @profile_fields
.google_map{data: address_fields: @address_fields.to_json }

Uygun “@profile_fields” ve “@address_fields” denetleyicide ayarlanır:

# app/controllers/profiles_controller.rb
class ProfilesController < ApplicationController
  def show
    # ...
    @profile_fields = @user_or_group.profile_fields
    @address_fields = @profile_fields.where(type: 'ProfileFields::Address')
  end
end

Haritayı başlatın, işaretçileri yerleştirin, javascript ile yakınlaştırma ve diğer harita ayarlarını yapın.

Javascript ile haritadaki işaretçileri ayarlayın #

Diyelim ki harita olacak ve ‘veri’ özelliği olarak işaretçi olarak gösterilecek adres alanlarına sahip bir ‘.google_map’ div var.

Örneğin:

<!-- http://localhost:3000/profiles/123 -->
<div class="google_map" data-address-fields="[
  {label: 'Work address', value: 'Willy-Brandt-Straße 1\n10557 Berlin', 
  position: {lng: ..., lat: ...}},
  ...
]"></div>

Turbolinks olaylarını elle yönetmeden turbolinks ile “$(document).ready” olayından yararlanmak için [jquery.turbolinks gem](https: //github.com/kossnocorp/jquery.turbolinks).

Haritayla daha sonra filtreleme veya bilgi pencereleri gibi başka işlemler yapmak isterseniz, haritanın bir kahve komut dosyası sınıfı tarafından yönetilmesi uygundur.

# app/assets/javascripts/google_maps.js.coffee
window.App = {} unless App?
class App.GoogleMap
  constructor: (map_div)->
    # TODO: initialize the map
    # TODO: set the markers

Varsayılan olarak ad alanlı birkaç kahve komut dosyası dosyası kullanırken, tüm kahve komut dosyası dosyaları tarafından paylaşılan genel bir “Uygulama” ad alanı tanımlamak uygundur.

Ardından, (muhtemelen birkaç) “.google_map” div’leri arasında dolaşın ve her biri için “App.GoogleMap” sınıfının bir örneğini oluşturun.

# app/assets/javascripts/google_maps.js.coffee
# ...
$(document).ready ->
  App.google_maps = []
  $('.google_map').each ->
    map_div = $(this)
    map = new App.GoogleMap map_div
    App.google_maps.push map

Bir kahve komut dosyası sınıfı kullanarak haritayı başlatın. #

Bir “App.GoogleMap” kahve komut dosyası sınıfı sağlandığında, google haritası şu şekilde başlatılabilir:

# app/assets/javascripts/google_maps.js.coffee
# ...
class App.GoogleMap
  map_div: {}
  map: {}
  
  constructor: (map_div)->
    @map_div = map_div
    @init_map()
    @reference_the_map_as_data_attribute

  # To access the GoogleMap object or the map object itself
  # later via the DOM, for example 
  # 
  #     $('.google_map').data('GoogleMap')
  #
  # store references as data attribute of the map_div.
  #
  reference_the_map_as_data_attribute: ->
    @map_div.data 'GoogleMap', this
    @map_div.data 'map', @map

  init_map: ->
    @map = new google.maps.Map(@dom_element, @map_configuration) if google?

  # `@map_div` is the jquery object. But google maps needs
  # the real DOM element.
  #
  dom_element: ->
    @map_div.get(0)

  map_configuration: -> {
    scrollWheel: true
  }

Olası “map_configuration” seçenekleri hakkında daha fazla bilgi edinmek için google’ın MapOptions belgelerine ve bunların [kılavuzuna] bakın. kontrol öğeleri ekleme](https://developers.google.com/maps/documentation/javascript/controls).

Başvuru için, google.maps.Mapburada kapsamlı bir şekilde belgelenmiştir.

Bir kahve komut dosyası sınıfı kullanarak harita işaretçilerini başlatın #

Bir “App.GoogleMap” kahve komut dosyası sınıfı ve haritanın “.google_map” divinin “data-address-fields” özelliğinde saklanan işaretçi bilgileri sağlandı işaretçiler haritada şu şekilde başlatılabilir:

# app/assets/javascripts/google_maps.js.coffee
# ...
class App.GoogleMap
  # ...
  markers: []

  constructor: (map_div)->
    # ...
    @init_markers()

  address_fields: ->
    @map_div.data('address-fields')

  init_markers: ->
    self = this  # to reference the instance as `self` when `this` is redefined.
    self.markers = []
    for address_field in self.address_fields()
      marker = new google.maps.Marker {
        map: self.map,
        position: {
          lng: address_field.longitude,
          lat: address_field.latitude
        },
        # # or, if `position` is defined in `ProfileFields::Address#as_json`:
        # position: address_field.position,
        title: address_field.value
      }
      self.markers.push marker

İşaretçi seçenekleri hakkında daha fazla bilgi edinmek için google’ın MarkerOptions belgelerine ve bunların işaretçi kılavuzuna göz atın. ://developers.google.com/maps/documentation/javascript/markers).

Kahve komut dosyası sınıfı kullanarak bir haritayı otomatik yakınlaştırın #

“google.maps.Map"in “@map” ve “google.maps.Marker"ın depolandığı bir “App.GoogleMap” kahve komut dosyası sınıfı sağladı “@markers” olarak, harita otomatik olarak yakınlaştırılabilir, yani tüm işaretçilerin görüneceği şekilde ayarlanabilir, örneğin: haritada şöyle:

# app/assets/javascripts/google_maps.js.coffee
# ...
class App.GoogleMap
  # ...
  bounds: {}

  constructor: (map_div)->
    # ...
    @auto_zoom()

  auto_zoom: ->
    @init_bounds()
    # TODO: Maybe, adjust the zoom to have a maximum or 
    # minimum zoom level, here.

  init_bounds: ->
    @bounds = new google.maps.LatLngBounds()
    for marker in @markers
      @bounds.extend marker.position
    @map.fitBounds @bounds

Sınırlar hakkında daha fazla bilgi edinmek için google’ın LatLngBounds belgelerine bakın.

Model özelliklerini json olarak gösterme #

Bir google haritasında adres profili alanlarını işaretçi olarak görüntülemek için, adres alanı nesnelerinin javascript’e json nesneleri olarak iletilmesi gerekir.

Normal veritabanı özellikleri #

Bir ApplicationRecord nesnesinde to_json çağrılırken, veritabanı öznitelikleri otomatik olarak açığa çıkar.

“label”, “value”, “longitude” ve “latitude” özniteliklerine sahip bir “ProfileFields::Address” modeli verildiğinde, “address_field.as_json” bir “Karma” ile sonuçlanır, ör. temsil,

address_field.as_json  # =>
  {label: "Work address", value: "Willy-Brandt-Straße 1\n10557 Berlin",
    longitude: ..., latitude: ...}

bu, ’to_json’ tarafından bir json dizgisine dönüştürülür:

address_field.to_json  # =>
  "{\"label\":\"Work address\",\"value\":\"Willy-Brandt-Straße 1\\n
    10557 Berlin\",\"longitude\":...,\"latitude\":...}"

Bu, örneğin harita işaretçileri için araç ipuçlarını göstermek için javascript’te daha sonra “etiket” ve “değer” kullanımına izin verdiği için kullanışlıdır.

Diğer özellikler #

Diğer sanal öznitelikler, “as_json” yöntemini geçersiz kılarak açığa çıkarılabilir.

Örneğin, bir “başlık” niteliğini ortaya çıkarmak için, onu birleştirilmiş “as_json” karma değerine dahil edin:

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  # ...

  # For example: "John Doe, Work address"
  def title
    "#{self.parent.name}, #{self.label}"
  end

  def as_json
    super.merge {
      title: self.title
    }
  end
end

Yukarıdaki örnek, nesnenin orijinal öznitelik karmasını döndüren ve onu gerekli konumla birleştiren orijinal “as_json” yöntemini çağırmak için super kullanır. doğramak.

“as_json” ve “to_json” arasındaki farkı anlamak için jjulian’ın bu blog gönderisine bir göz atın.

Durum #

İşaretçileri oluşturmak için, google maps api, varsayılan olarak, boylam ve enlemin sırasıyla “lng” ve “enlem” olarak saklandığı bir “konum” karmasını gerektirir.

Bu konum karması, javascript’te, daha sonra veya adres alanının json gösterimini tanımlarken burada oluşturulabilir:

Bu “pozisyonu” adres alanının json niteliği olarak sağlamak için, modelde “as_json” yöntemini geçersiz kılmanız yeterlidir.

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  # ...

  def as_json
    super.merge {
      # ...
      position: {
        lng: self.longitude,
        lat: self.latitude
      }
    }
  end
end
