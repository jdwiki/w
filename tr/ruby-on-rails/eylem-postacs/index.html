Action Mailer, mailler sınıflarını ve görünümlerini kullanarak uygulamanızdan e-posta göndermenize olanak tanır. Postalar, denetleyicilere çok benzer şekilde çalışır. ActionMailer::Base’den devralırlar ve uygulamada/postalarda yaşarlar ve uygulamada/görünümlerde görünen ilişkili görünümlere sahiptirler.

Web sunucunuzu bağlamamak için e-posta gönderimini eşzamansız olarak işlemeniz önerilir. Bu, “delayed_job” gibi çeşitli servisler aracılığıyla yapılabilir.

Temel Posta Göndericisi #

Bu örnek dört farklı dosya kullanır:

  • Kullanıcı modeli
  • Kullanıcı postası
  • E-posta için html şablonu
  • E-posta için düz metin şablonu

Bu durumda, kullanıcı modeli postada ‘onaylı’ yöntemi çağırır ve onaylanan ‘gönderiyi’ iletir (modeldeki ‘onaylı’ yöntem, bir denetleyici yönteminden, vb. bir geri çağrı ile çağrılabilir). Ardından, posta gönderici, iletilen “gönderinin” (ör. başlık) bilgilerini kullanarak html veya düz metin şablonundan e-postayı oluşturur. Varsayılan olarak, posta göndericisi, posta göndericisindeki yöntemle aynı ada sahip şablonu kullanır (bu nedenle hem posta gönderici yönteminin hem de şablonların adı ‘onaylı’dır).

user_mailer.rb #

class UserMailer < ActionMailer::Base
  default from: "donotreply@example.com"

  def approved(post)
      @title = post.title
      @user = post.user
      mail(to: @user.email, subject: "Your Post was Approved!")
  end
end

kullanıcı.rb #

def approved(post)
    UserMailer.approved(post)
end

onaylı.html.erb #

<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
<html>
    <head>
      <meta http-equiv="Content-Type" content="text/html; charset=utf-8">
      <title>Post Approved</title>
    </head>
    <body>
        <h2>Congrats <%= @user.name %>! Your post (#<%= @title %>) has been approved!</h2>
        <p>We look forward to your future posts!</p>
    </body>
</html>

onaylı.metin.erb #

Congrats <%= @user.name %>! Your post (#<%= @title %>) has been approved!
We look forward to your future posts!

Yeni bir posta gönderici oluşturma #

Yeni bir mailer oluşturmak için aşağıdaki komutu girin

rails generate mailer PostMailer

Bu, “app/mailers/post_mailer.rb” içinde PostMailer adında boş bir şablon dosyası oluşturacaktır.

class PostMailer < ApplicationMailer
end

E-posta görünümü için, biri html formatı ve diğeri metin formatı için olmak üzere iki düzen dosyası da oluşturulacaktır.

Jeneratörü kullanmamayı tercih ederseniz, kendi mailer’larınızı oluşturabilirsiniz. ‘ActionMailer::Base’den miras aldıklarından emin olun

Ek Ekleme #

ActionMailer ayrıca dosya eklemeye de izin verir.

attachments['filename.jpg'] = File.read('/path/to/filename.jpg')

Varsayılan olarak, ekler “Base64” ile kodlanacaktır. Bunu değiştirmek için, ekleri yöntemine bir karma ekleyebilirsiniz.

attachments['filename.jpg'] = {
  mime_type: 'application/gzip',
  encoding: 'SpecialEncoding',
  content: encoded_content
}

Ayrıca satır içi ekler de ekleyebilirsiniz.

attachments.inline['image.jpg'] = File.read('/path/to/image.jpg')

ActionMailer Engelleyici #

Action Mailer, önleme yöntemlerine kancalar sağlar. Bunlar, posta teslimi yaşam döngüsü sırasında çağrılan sınıfları kaydetmenize olanak tanır.

Bir interceptor sınıfı, e-posta gönderilmeden önce çağrılacak olan :delivering_email(message) yöntemini uygulamalıdır ve bu, e-posta dağıtım aracılarına ulaşmadan önce değişiklik yapmanıza olanak tanır. Sınıfınız, gerekli değişiklikleri doğrudan Mail::Message örneğinde iletilen örnekte yapmalıdır.

Geliştiricilerin gerçek kullanıcılara değil kendilerine e-posta göndermeleri yararlı olabilir.

Bir actionmailer interceptor kaydetme örneği:

# config/initializers/override_mail_recipient.rb

if Rails.env.development? or Rails.env.test?
  class OverrideMailRecipient
    def self.delivering_email(mail)
      mail.subject = 'This is dummy subject'
      mail.bcc = 'test_bcc@noemail.com'
      mail.to = 'test@noemail.com'
    end
  end
  ActionMailer::Base.register_interceptor(OverrideMailRecipient)
end

class Haber BültenleriDenetleyici < Uygulama Denetleyicisi
    skip_before_action :authenticate_user!

    def create
      @newsletter = Newsletter.create(newsletter_params)
      if @newsletter.save
        redirect_to blog_index_path
      else
        redirect_to root_path
      end
    end

    private

      def set_newsletter
        @newsletter = Newsletter.find(params[:id])
      end

      def newsletter_params
        params.require(:newsletter).permit(:name, :email)
      end

Bu, geliştirme ortamınızda e-posta teslimi ayarlamanıza gerek olmadığı ve artık başka birinin adresine yanlışlıkla bir test e-postası gönderme konusunda endişelenmenize gerek olmadığı anlamına gelir.

Önce gem’i geliştirme ortamınıza ekleyin ve yüklemek için paket komutunu çalıştırın.

Ardından Geliştirme Ortamında teslim yöntemini ayarlayın:

Şimdi Cron Job‘un başarıyla oluşturulduğunu test etmek için, CRON SYNTAX’ta planlanmış işimiz olan terminalden beri okumak için bir sonraki komutu kullanabiliriz:

your_project your_mac_user$ ne zaman

30 4 * * * /bin/bash -l -c 'cd /Users/your_mac_user/Desktop/your_project && RAILS_ENV=üretim paketi yürütme rakeweek_newsletter_email --silent'

Şimdi, testi Geliştirme Ortamında çalıştırmak için, uygulamanın kullanacağı modellerin nerede olduğunu bilmesini sağlamak için application.rb ana dosyasındaki bir sonraki satırı ayarlamak akıllıca olacaktır.

subl projeniz/config/application.rb

config.action_mailer.default_url_options = { :host => "http://localhost:3000/" }

Şimdi Capistrano V3‘ün yeni Cron Job‘u sunucu içinde ve bu görevin yürütülmesini başlatacak tetikleyiciyi kaydetmesine izin vermek için bir sonraki gereksinimi eklemeliyiz:

subl your_project/Capfile

'ne zaman/capistrano' gerektirir

Ve deploy dosyasına CRON JOB‘un ortam hakkında kullanacağı tanımlayıcıyı ve uygulamanın adını ekleyin.

subl projeniz/config/deploy.rb

set :whileever_identifier, ->{ "#{fetch(:application)}_#{fetch(:rails_env)}" }

Ve hazır, her dosyadaki değişiklikleri kaydettikten sonra capistrano dağıtım komutunu çalıştırın:

Üretim dağıtımı üst sınırı

Ve şimdi JOB’unuz oluşturuldu ve benim istediğim ve bu dosyalarda belirlediğimiz zaman aralığında Mailer Metodu’nu çalıştırmak için takvimlendirildi.
