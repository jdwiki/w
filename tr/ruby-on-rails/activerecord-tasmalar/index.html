Sütun türü Açıklama
:birincil_anahtar Birincil anahtar
:dize Daha kısa dize veri türü. Maksimum karakter sayısı için limit seçeneğine izin verir.
:metin Daha uzun metin miktarı. Maksimum bayt sayısı için ’limit’ seçeneğine izin verir.
`:tamsayı' Tamsayı. Maksimum bayt sayısı için ’limit’ seçeneğine izin verir.
:büyük Daha büyük tamsayı
:yüzer Şamandıra
:ondalık Değişken hassasiyetli ondalık sayı. “Hassas” ve “ölçek” seçeneklerine izin verir.
:sayısal “Hassas” ve “ölçek” seçeneklerine izin verir.
:tarihsaat Tarihler/saatler için DateTime nesnesi.
:zaman Times için zaman nesnesi.
:tarih Tarihler için tarih nesnesi.
:ikili Ikili veri. Maksimum bayt sayısı için ’limit’ seçeneğine izin verir.
: boolean Boole

  • Çoğu taşıma dosyası db/migrate/ dizininde bulunur. Dosya adlarının başında bir UTC zaman damgasıyla tanımlanırlar: “YYYYMMDDHHMMSS_create_products.rb”.

  • ‘Rails üret’ komutu, ‘Rails g’ olarak kısaltılabilir.

  • Bir alana bir `:type’ geçirilmezse, varsayılan olarak bir dizge olur.

Bir tabloya birden çok sütun ekleme #

Bir tabloya birden çok sütun eklemek için, “raylar oluşturma geçişi” komutunu kullanırken “field:type” çiftlerini boşluklarla ayırın.

Genel sözdizimi şöyledir:

rails generate migration NAME [field[:type][:index] field[:type][:index]] [options]

Örneğin, “kullanıcılar” tablosuna “ad”, “maaş” ve “e-posta” alanları eklenecektir:

rails generate migration AddDetailsToUsers name:string salary:decimal email:string

Hangisi aşağıdaki göçü oluşturur:

class AddDetailsToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :name, :string
    add_column :users, :salary, :decimal
    add_column :users, :email, :string
  end
end

Tabloya referans sütunu ekleme #

“Kullanıcılar” tablosuna bir “ekibe” referans eklemek için şu komutu çalıştırın:

$ rails generate migration AddTeamRefToUsers team:references

Bu, aşağıdaki taşımayı oluşturur:

class AddTeamRefToUsers < ActiveRecord::Migration[5.0]
  def change
    add_reference :users, :team, foreign_key: true
  end
end

Bu geçiş, “users” tablosunda bir “team_id” sütunu oluşturacaktır.

Eklenen sütuna uygun bir ‘index’ ve ‘yabancı_anahtar’ eklemek istiyorsanız, komutu ‘rails taşıma AddTeamRefToUsers ekibi:references:index’ olarak değiştirin. Bu, aşağıdaki taşımayı üretecektir:

class AddTeamRefToUsers < ActiveRecord::Migration
  def change
    add_reference :users, :team, index: true
    add_foreign_key :users, :teams
  end
end

Referans sütununuzu Rails’in otomatik olarak oluşturduğundan farklı bir ad vermek istiyorsanız, taşıma işleminize aşağıdakini ekleyin: (Örn.: “Yazı"yı oluşturan “Kullanıcı"yı “Yazı” tablosunda “Yazar” olarak adlandırmak isteyebilirsiniz. )

class AddAuthorRefToPosts < ActiveRecord::Migration
  def change
    add_reference :posts, :author, references: :users, index: true
  end
end

Geri alma geçişleri #

En son geçişi “geri almak” için, “değiştir” yöntemini geri alarak veya “aşağı” yöntemini çalıştırarak. Çalıştır komutu:

rake db:rollback
son sürüm 
rails db:rollback
son sürüm

Son 3 taşıma işlemini geri alın #

rake db:rollback STEP=3
son sürüm 
rails db:rollback STEP=3
son sürüm

“ADIM”, geri alınacak taşıma sayısını sağlar.

Tüm taşıma işlemlerini geri al #

rake db:rollback VERSION=0
son sürüm 
rails db:rollback VERSION=0
son sürüm

Dizin içeren yeni bir sütun ekleyin #

“users” tablosuna yeni bir indexed “email” sütunu eklemek için şu komutu çalıştırın:

rails generate migration AddEmailToUsers email:string:index

Bu, aşağıdaki taşımayı üretecektir:

class AddEmailToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :email, :string
    add_index :users, :email
  end
end

Belirli bir taşımayı çalıştır #

Belirli bir taşımayı yukarı veya aşağı çalıştırmak için “db:migrate:up” veya “db:migrate:down” kullanın.

Belirli bir taşıma yukarı:

rake db:migrate:up VERSION=20090408054555
son sürüm 
rails db:migrate:up VERSION=20090408054555
son sürüm

Belirli bir geçiş aşağı:

rake db:migrate:down VERSION=20090408054555
son sürüm 
rails db:migrate:down VERSION=20090408054555
son sürüm

Yukarıdaki komutlardaki sürüm numarası, taşımanın dosya adındaki sayısal önektir. Örneğin, “20160515085959_add_name_to_users.rb” geçişine geçmek için sürüm numarası olarak “20160515085959” kullanırsınız.

Taşıma işlemlerini yeniden yap #

redo komutunu kullanarak geri alabilir ve ardından tekrar geçiş yapabilirsiniz. Bu temelde “geri alma” ve “geçiş” görevlerini birleştiren bir kısayoldur.

Çalıştır komutu:

rake db:migrate:redo
son sürüm 
rails db:migrate:redo
son sürüm

Birden fazla sürüme geri dönmek için STEP parametresini kullanabilirsiniz.

Örneğin, 3 taşıma işlemine geri dönmek için:

rake db:migrate:redo STEP=3
son sürüm 
rails db:migrate:redo STEP=3
son sürüm

Tabloya yeni bir sütun ekleyin #

“users” tablosuna yeni bir “name” sütunu eklemek için şu komutu çalıştırın:

rails generate migration AddNameToUsers name

Bu, aşağıdaki taşımayı oluşturur:

class AddNameToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :name, :string
  end
end

Taşıma adı “AddXXXToTABLE_NAME” biçiminde ve ardından veri türlerini içeren sütunlar listesinde olduğunda, oluşturulan taşıma uygun “add_column” ifadelerini içerecektir.

Tablodan mevcut bir sütunu kaldırma #

“Kullanıcılar” tablosundan mevcut “ad” sütununu kaldırmak için şu komutu çalıştırın:

rails generate migration RemoveNameFromUsers name:string

Bu, aşağıdaki taşımayı üretecektir:

class RemoveNameFromUsers < ActiveRecord::Migration[5.0]
  def change
    remove_column :users, :name, :string
  end
end

Taşıma adı “RemoveXXXFromYYY” biçimindeyse, ardından veri türlerini içeren sütunların listesi gelirse, oluşturulan taşıma uygun “remove_column” ifadelerini içerecektir.

Veri türünü (ör. “:string”) “remove_column” parametresi olarak belirtmek gerekli olmasa da, şiddetle tavsiye edilir. Veri türü belirtilmemişse, taşıma işlemi geri alınamaz.

Taşıma işlemlerini farklı ortamlarda çalıştırma #

“Test” ortamında geçişleri çalıştırmak için şu kabuk komutunu çalıştırın:

rake db:migrate RAILS_ENV=test

Rails 5.0’dan başlayarak, “tırmık” yerine “raylar” kullanabilirsiniz:

rails db:migrate RAILS_ENV=test
son sürüm

Yeni bir tablo oluşturun #

“name” ve “maaş” sütunlarıyla yeni bir “kullanıcılar” tablosu oluşturmak için şu komutu çalıştırın:

rails generate migration CreateUsers name:string salary:decimal

Bu, aşağıdaki taşımayı üretecektir:

class CreateUsers < ActiveRecord::Migration[5.0]
  def change
    create_table :users do |t|
      t.string :name
      t.decimal :salary  
    end
  end
end

Taşıma adı “CreateXXX” biçiminde ve ardından veri türlerine sahip sütunların listesi olduğunda, listelenen sütunlarla “XXX” tablosunu oluşturan bir taşıma oluşturulur.

Taşıma işlemlerini çalıştırma #

Çalıştır komutu:

rake db:migrate
son sürüm 
rails db:migrate
son sürüm

Hedef sürümün belirtilmesi, belirtilen sürüme ulaşana kadar gerekli geçişleri (yukarı, aşağı, değişiklik) çalıştıracaktır. Burada “sürüm numarası”, taşımanın dosya adındaki sayısal önektir.

rake db:migrate VERSION=20080906120000
son sürüm 
rails db:migrate VERSION=20080906120000
son sürüm

Mevcut bir sütunun türünü değiştirin #

Rails’de var olan bir sütunu geçişle değiştirmek için aşağıdaki komutu çalıştırın:

rails g migration change_column_in_table

Bu, ‘db/migration’ dizininde (zaten yoksa) yeni bir taşıma dosyası oluşturacaktır, bu dosya ön eki zaman damgası ve aşağıdaki içeriği içeren taşıma dosyası adını içerecektir:

def change
  change_column(:table_name, :column_name, :new_type)
end

Ray Kılavuzu – Sütunları Değiştirme

Daha uzun ama daha güvenli bir yöntem #

Yukarıdaki kod, kullanıcının geçişi geri almasını engeller. “change” yöntemini ayrı “up” ve “down” yöntemlerine bölerek bu sorunu önleyebilirsiniz:

def up
  change_column :my_table, :my_column, :new_type
end

def down
  change_column :my_table, :my_column, :old_type
end

Varsayılan değerle sütun ekle #

Aşağıdaki örnek, “kullanıcılar” tablosuna bir “yönetici” sütunu ekler ve bu sütuna varsayılan “yanlış” değerini verir.

class AddDetailsToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :admin, :boolean, default: false
  end
end

Varsayılanlarla geçişler, örneğin PostgreSQL gibi büyük tablolarda uzun zaman alabilir. Bunun nedeni, her satırın yeni eklenen sütun için varsayılan değerle güncellenmesi gerekmesidir. Bunu atlatmak (ve dağıtımlar sırasında kapalı kalma süresini azaltmak) için geçişinizi üç adıma bölebilirsiniz:

  1. Yukarıdakine benzer bir “add_column” geçişi ekleyin, ancak varsayılan olarak ayarlamayın
  2. Uygulamanız çalışırken sütunu bir komisyon görevinde veya konsolda dağıtın ve güncelleyin. Uygulamanızın yeni/güncellenmiş satırlar için bu sütuna zaten veri yazdığından emin olun.
  3. Başka bir ‘change_column’ geçişi ekleyin, ardından bu sütunun varsayılanını istenen varsayılan değere değiştirir

Bir birleştirme tablosu oluşturun #

“öğrenciler” ve “dersler” arasında bir birleştirme tablosu oluşturmak için şu komutu çalıştırın:

$ rails g migration CreateJoinTableStudentCourse student course

Bu, aşağıdaki taşımayı üretecektir:

class CreateJoinTableStudentCourse < ActiveRecord::Migration[5.0]
  def change
    create_join_table :students, :courses do |t|
      # t.index [:student_id, :course_id]
      # t.index [:course_id, :student_id]
    end
  end
end

Bir hstore sütunu oluşturun #

“Hstore” sütunları, ayarları saklamak için faydalı olabilir. Uzantıyı etkinleştirdikten sonra PostgreSQL veritabanlarında kullanılabilirler.

class CreatePages < ActiveRecord::Migration[5.0]
  def change
    create_table :pages do |t|
      enable_extension 'hstore' unless extension_enabled?('hstore')
      t.hstore :settings
      t.timestamps
    end
  end
end

Kendi kendine referans ekle #

Hiyerarşik bir ağaç oluşturmak için öz referans yararlı olabilir. Bu, bir taşıma işleminde “add_reference” ile gerçekleştirilebilir.

class AddParentPages < ActiveRecord::Migration[5.0]
  def change
    add_reference :pages, :pages
  end
end

Yabancı anahtar sütunu “pages_id” olacaktır. Yabancı anahtar sütun adı hakkında karar vermek istiyorsanız, önce sütunu oluşturmanız ve ardından referansı eklemeniz gerekir.

class AddParentPages < ActiveRecord::Migration[5.0]
  def change
    add_column :pages, :parent_id, :integer, null: true, index: true
    add_foreign_key :pages, :pages, column: :parent_id
  end
end

Bir dizi sütunu oluşturun #

Bir “dizi” sütunu PostgreSQL tarafından desteklenir. Rails, bir PostgreSQL dizisini otomatik olarak bir Ruby dizisine ve bunun tersini de yapacaktır.

Bir “dizi” sütunu olan bir tablo oluşturun:

create_table :products do |t|
  t.string :name
  t.text :colors, array: true, default: []
end

Mevcut bir tabloya bir “dizi” sütunu ekleyin:

add_column :products, :colors, array: true, default: []

Bir “dizi” sütunu için bir dizin ekleyin:

add_index :products, :colors, using: 'gin'

Tabloları Değiştirme #

Yanlış şemaya sahip bir tablo oluşturduysanız, sütunları ve özelliklerini değiştirmenin en kolay yolu ‘change_table’dır. Aşağıdaki örneği inceleyin:

change_table :orders do |t|
  t.remove :ordered_at # removes column ordered_at
  t.string :skew_number # adds a new column 
  t.index  :skew_number #creates an index
  t.rename :location, :state #renames location column to state
end

Yukarıdaki geçiş, bir “siparişler” tablosunu değiştirir. İşte değişikliklerin satır satır açıklaması:

  1. ’t.remove :ordered_at’, ‘orders’ tablosundan ‘ordered_at’ sütununu kaldırır.
  2. ’t.string :skew_number’, ‘orders’ tablosuna ‘skew_number’ adında yeni bir string tipi sütun ekler.
  3. “t.index :skew_number”, “siparişler” tablosundaki “skew_number” sütununa bir dizin ekler.
  4. ’t.rename :location, :state’, ‘orders’ tablosundaki ’location’ sütununu ‘state’ olarak yeniden adlandırır.

Bir tabloya benzersiz bir sütun ekleyin #

“users"a yeni bir benzersiz “email” sütunu eklemek için aşağıdaki komutu çalıştırın:

rails generate migration AddEmailToUsers email:string:uniq

Bu, aşağıdaki taşımayı yaratacaktır:

class AddEmailToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :email, :string
    add_index :users, :email, unique: true
  end
end

Boş değerleri yasakla #

Tablo sütunlarınızda “null” değerleri yasaklamak için, taşıma işleminize “:null” parametresini şu şekilde ekleyin:

class AddPriceToProducts < ActiveRecord::Migration
  def change
    add_column :products, :float, null: false
  end
end

Mevcut verilere NOT NULL kısıtlaması ekleme #

“Kullanıcılar” tablosuna bir yabancı anahtar “şirket_kimliği” eklemek istediğinizi ve bunun üzerinde bir “BOŞ DEĞİL” kısıtlamasına sahip olmak istediğinizi varsayalım. “Kullanıcılar"da zaten verileriniz varsa, bunu birkaç adımda yapmanız gerekecektir.

class AddCompanyIdToUsers < ActiveRecord::Migration
  def up
    # add the column with NULL allowed
    add_column :users, :company_id, :integer

    # make sure every row has a value
    User.find_each do |user|
      # find the appropriate company record for the user
      # according to your business logic
      company = Company.first
      user.update!(company_id: company.id)
    end

    # add NOT NULL constraint
    change_column_null :users, :company_id, false
  end

  # Migrations that manipulate data must use up/down instead of change
  def down
    remove_column :users, :company_id
  end
end
