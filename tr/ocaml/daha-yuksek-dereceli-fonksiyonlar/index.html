Sözdizimi #

  • `val (|>) : ‘a -> (‘a -> ‘b) -> ‘b’
  • `val (@@) : (‘a -> ‘b) -> ‘a -> ‘b’

Genel algoritmalar #

Genel algoritmaları uygulamak için daha yüksek dereceli işlevler kullanılabilir ve kullanıcıya son ayrıntıları sağlama sorumluluğundan vazgeçilir. Örneğin, ‘List.sort’, çeşitli sıralama yöntemlerinin uygulanmasına izin veren bir karşılaştırma işlevi bekler. Burada, büyük/küçük harfe duyarlı olmayan dizeleri sıralıyoruz:

let string_case_insensitive_sort lst =
  let case_insensitive_compare a b =
    String.compare (String.lowercase a) (String.lowercase b)
  in
  List.sort case_insensitive_compare lst

Standart kitaplıkta, özellikle Liste modülünde zengin bir üst düzey işlevler listesi vardır, örneğin ‘List.fold_left’ ve ‘List.sort’a bakın. Üçüncü taraf kitaplıklarında daha gelişmiş örnekler bulunabilir. İyi bir örnek, ocaml-gsl‘de uygulanan simüle edilmiş tavlama‘dir. Simüle edilmiş tavlama, problemin durumlarını keşfetmek için kullanılan bir fonksiyon ve bir hata fonksiyonu (burada enerji fonksiyonu olarak adlandırılır) tarafından parametrelendirilen genel bir optimizasyon prosedürüdür. .

C++‘a aşina olan kullanıcılar bunu Strateji modeliyle karşılaştırabilir.

Bir istisna oluşturulduğunda bile sistem kaynaklarını atın #

Bir işlem bir istisna oluştursa bile sistem kaynaklarının atılmasını sağlamak için daha yüksek dereceli işlevler kullanılabilir. ‘with_output_file’ tarafından kullanılan model, endişelerin net bir şekilde ayrılmasını sağlar: daha yüksek dereceli ‘with_output_file’ işlevleri, ‘f’ işlemi yalnızca çıktı kanalını tüketirken dosya manipülasyonuna bağlı sistem kaynaklarının yönetimiyle ilgilenir.

let with_output_file path f =
  let c = open_out path in
  try
    let answer = f c in
    (close_out c; answer)
  with exn -> (close_out c; raise exn)

Bir dosyaya dize yazan bir işlevi uygulamak için bu üst düzey işlevi kullanalım:

let save_string path s =
  (with_output_file path) (fun c -> output_string c s)

fun c -> output_string c s işlevinden daha gelişmiş işlevleri kullanarak daha karmaşık değerleri kaydetmek mümkündür. Örneğin standart kitaplıktaki Marshal modülüne veya [Yojson](http:// mjambon.com/yojson.html) Martin Jambon’un kitaplığı.

Kompozisyon operatörleri #

İki kullanışlı üst düzey işlev, ikili application (@) ve reverse-application veya “pipe” (|>) operatörleridir. 4.01‘den beri ilkel olarak mevcut olmalarına rağmen, onları burada tanımlamak yine de öğretici olabilir:

let (|>) x f = f x
let (@@) f x = f x

3’ün karesini artırma problemini ele alalım. Bu hesaplamayı ifade etmenin bir yolu şudur:

(* 1 -- Using parentheses *)
succ (square 3)
(* - : int = 10 *)

(* where `square` is defined as: *)
let square x = x * x

Basitçe “son kare 3"ü yapamayacağımıza dikkat edin, çünkü ([sol-ilişkilendirme2 nedeniyle) bu anlamsız “(öz kare) 3"e indirgenir. (@@) uygulamasını kullanarak, parantez olmadan şunu ifade edebiliriz:

(* 2 -- Using the application operator *)
succ @@ square 3
(* - : int = 10 *)

Gerçekleştirilecek son işlemin (yani “succ”) ifadede ilk olarak nasıl gerçekleştiğine dikkat edin? reverse-application operatörü (|>) bunu tersine çevirmemizi sağlar:

(* 3 -- Using the reverse-application operator *)
3 |> square |> succ
(* - : int = 10 *)

3 sayısı artık “kare” ve ardından “succ” ile “borulanır”, “succ"un uygulandığı bir sonuç vermek için “square"e uygulanmaz.
