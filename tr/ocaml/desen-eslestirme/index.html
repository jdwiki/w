Model Eşleştirmeyi Kullanan Faktöryel İşlev #

let rec factorial n = match n with
| 0 | 1 -> 1
| n -> n * (factorial (n - 1))

Bu işlev hem 0 hem de 1 değerleriyle eşleşir ve bunları özyinelemeli tanımımızın temel durumuna eşler. Ardından diğer tüm sayılar bu işlevin özyinelemeli çağrısına eşlenir.

Boole ifadelerinin değerlendirilmesi #

Atomları dizelerle tanımlanan boole ifadelerinin türünü şu şekilde tanımlarız:

type expr =
| Atom of string
| Not of expr
| And of expr * expr
| Or of expr * expr

ve bir “oracle : string -> bool” kullanarak bu ifadeleri değerlendirebilir ve atomların değerlerini aşağıdaki gibi verir:

let rec eval oracle = function
| Atom(name) -> oracle name
| Not(expr) -> not(eval oracle expr)
| And(expr1, expr2) -> (eval oracle expr1) && (eval oracle expr2)
| Or(expr1, expr2)  -> (eval oracle expr1) || (eval oracle expr2)

İşlevin ne kadar net ve okunması kolay olduğunu görün. Model eşleştirmenin doğru kullanımı sayesinde, bu işlevi okuyan bir programcının, doğru şekilde uygulandığından emin olmak için çok az zamana ihtiyacı vardır.

Olumsuzlama normal formu : derin kalıp eşleştirme #

Model eşleştirme, karmaşık değerlerin yapısını çözmeye izin verir ve hiçbir şekilde bir değerin temsilinin “en dıştaki” düzeyi ile sınırlı değildir. Bunu göstermek için, bir boole ifadesini, tüm olumsuzlamaların yalnızca atomlarda olduğu, sözde olumsuzlama normal formu ve bu formdaki ifadeleri tanıyan bir yüklem olduğu bir boole ifadesine dönüştüren işlevi uygularız:

Atomları dizelerle tanımlanan boole ifadelerinin türünü şu şekilde tanımlarız:

type expr =
| Atom of string
| Not of expr
| And of expr * expr
| Or of expr * expr

Önce olumsuzlama normal biçimindeki ifadeleri tanıyan bir yüklem tanımlayalım:

let rec is_nnf = function
| (Atom(_) | Not(Atom(_))) -> true
| Not(_) -> false
| (And(expr1, expr2) | Or(expr1, expr2)) -> is_nnf expr1 && is_nnf expr2

Gördüğünüz gibi Not(Atom(_)) gibi iç içe kalıplarla eşleştirme yapmak mümkün. Şimdi, bir boole ifadesini, olumsuzlama normal biçiminde eşdeğer bir boole ifadesine eşleyen bir işlev uyguluyoruz:

let rec nnf = function
| (Atom(_) | Not(Atom(_))) as expr -> expr
| Not(And(expr1, expr2)) -> Or(nnf(Not(expr1)),nnf(Not(expr2)))
| Not(Or(expr1, expr2)) -> And(nnf(Not(expr1)),nnf(Not(expr2)))
| And(expr1, expr2) -> And(nnf expr1, nnf expr2)
| Or(expr1, expr2) -> Or(nnf expr1, nnf expr2)
| Not(Not(expr)) -> nnf expr

Bu ikinci işlev, iç içe desenlerin daha da fazla kullanılmasını sağlar. Sonunda, bir imanın olumsuzlanması konusunda kodumuzu en üst düzeyde test edebiliriz:

# let impl a b =
Or(Not(a), b);;
  val impl : expr -> expr -> expr = <fun>
# let expr = Not(impl (Atom "A") (Atom "B"));;
val expr : expr = Not (Or (Not (Atom "A"), Atom "B"))
# nnf expr;;
- : expr = And (Atom "A", Not (Atom "B"))
# is_nnf (nnf expr);;
- : bool = true

OCaml tipi sistem, bir model eşleştirmesinin eksiksizliğini doğrulayabilir. Örneğin, ’nnf’ işlevinde ‘Not(Veya(ifade1, ifade2)’ durumunu atlarsak, derleyici bir uyarı verir:

# let rec non_exhaustive_nnf = function
| (Atom(_) | Not(Atom(_))) as expr -> expr
| Not(And(expr1, expr2)) -> Or(nnf(Not(expr1)),nnf(Not(expr2)))
| And(expr1, expr2) -> And(nnf expr1, nnf expr2)
| Or(expr1, expr2) -> Or(nnf expr1, nnf expr2)
| Not(Not(expr)) -> nnf expr;;

Eşleşen kayıt alanları #

Kalıp eşleştirme, kayıtları bozmak için kullanılabilir. Bunu bir metin dosyasındaki konumları temsil eden bir kayıt türüyle gösteriyoruz, ör. bir programın kaynak kodu.

type location = {
  filename : string;
  line: int;
  column: int;
  offset: int;
}

Konum türünde bir “x” değeri şu şekilde çözülebilir:

let { filename; line; column; offset; } = x

İşlevleri tanımlamak için benzer bir sözdizimi kullanılabilir, örneğin konumları yazdırmak için bir işlev:

let print_location { filename; line; column; offset; } =
  Printf.printf "%s: %d: %d" filename line column

Veya alternatif olarak

let print_location = function { filename; line; column; offset; } ->
  Printf.printf "%s: %d: %d" filename line column

Kayıtlarla eşleşen kalıpların bir kaydın tüm alanlarından bahsetmesine gerek yoktur. İşlev “offset” alanını kullanmadığından, onu dışarıda bırakabiliriz:

let print_location { filename; line; column; } =
  Printf.printf "%s: %d: %d" filename line column

Kayıt bir modülde tanımlandığında, kalıpta meydana gelen ilk alanı nitelemek yeterlidir:

module Location =
struct
  type t = {
      filename : string;
      line: int;
      column: int;
      offset: int;
    }
end

let print_location { Location.filename; line; column; } =
  Printf.printf "%s: %d: %d" filename line column

Desen eşleştirme ile özyinelemeli liste işleme #

Burada, OCaml’in örüntü eşleştirme sözdizimini kullanarak listelerin özyinelemeli olarak nasıl işleneceğini gösteriyoruz.

let rec map f lst =
  match lst with
  | [] -> []
  | hd::tl -> (f hd)::(map f tl)

Bu durumda, “[]” kalıbı boş listeyle eşleşirken, “hd::tl” en az bir elemanı olan herhangi bir listeyle eşleşir ve listenin ilk elemanını “hd"ye ve geri kalanını atayacaktır. listenin (boş olabilir) tl.

“hd::tl"nin çok genel bir kalıp olduğunu ve boş olmayan herhangi bir listeyle eşleşeceğini unutmayın. Belirli sayıda öğeye sahip listelerde eşleşen desenler de yazabiliriz:

(* Return the last element of a list. Fails if the list is empty. *)
let rec last lst =
  match lst with
  | [] -> failwith "Empty list"
  | [x] -> x (* Equivalent to x::[], [x] matches a list with only one element *)
  | hd::tl -> last tl

(* The second to last element of a list. *)
let rec second_to_last lst =
  match lst with
  | [] -> failwith "Empty list"
  | x::[] -> failwith "Singleton list"
  | fst::snd::[] -> snd
  | hd::tl -> second_to_last tl

Ek olarak, OCaml, listelerin kendi öğelerinde kalıp eşleştirmeyi destekler. Bir liste içindeki elemanların yapısı hakkında daha spesifik olabiliriz ve OCaml doğru fonksiyon tipini çıkaracaktır:

(* Assuming a list of tuples, return a list with first element of each tuple. *)
let rec first_elements lst =
  match lst with
  | [] -> []
  | (a, b)::tl -> a::(first_elements tl)
(* val first_elements : ('a * 'b) list -> 'a list = <fun> *)

Bu kalıpları bir araya getirerek, herhangi bir rastgele karmaşık listeyi işleyebiliriz.

Model eşleştirmeyi kullanarak bir fonksiyon tanımlama #

‘function’ anahtar sözcüğü, bir işlevin son argümanında kalıp eşleştirmeyi başlatmak için kullanılabilir. Örneğin, bir tamsayı listesinin toplamını bu şekilde hesaplayan ‘sum’ adında bir fonksiyon yazabiliriz.

let rec sum = function
  | [] -> 0
  | h::t -> h + sum t
;;

val sum : int list -> int = <fun>
