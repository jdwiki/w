resmi belgeler: SQLite Sürüm 3’teki Veri Tipleri

Tarih/saat türleri #

SQLite, tarih veya saat değerleri için ayrı bir veri türüne sahip değildir.

ISO8601 dizileri #

“CURRENT_DATE”, “CURRENT_TIME” ve “CURRENT_TIMESTAMP” yerleşik anahtar sözcükleri, dizeleri ISO8601 biçiminde döndürür:

> SELECT CURRENT_DATE, CURRENT_TIME, CURRENT_TIMESTAMP;
CURRENT_DATE  CURRENT_TIME  CURRENT_TIMESTAMP  
------------  ------------  -------------------
2016-07-08    12:34:56      2016-07-08 12:34:56

Bu tür değerler, tüm yerleşik tarih/saat işlevleri tarafından da anlaşılır:

> SELECT strftime('%Y', '2016-07-08');
2016

Julian gün numaraları #

Yerleşik tarih/saat işlevleri, sayıları Julian günleri olarak yorumlar:

> SELECT datetime(2457578.02425926);
2016-07-08 12:34:56

“julianday()” işlevi, desteklenen herhangi bir tarih/saat değerini bir Jülyen gün numarasına dönüştürür:

> SELECT julianday('2016-07-08 12:34:56');
2457578.02425926

Unix zaman damgaları #

Yerleşik tarih/saat işlevleri, sayıları Unix zaman damgaları olarak yorumlayabilir. “unixepoch” değiştiricisi:

> SELECT datetime(0, 'unixepoch');
1970-01-01 00:00:00

strftime() işlevi, desteklenen herhangi bir tarih/saat değerini bir Unix zaman damgasına dönüştürebilir:

> SELECT strftime('%s', '2016-07-08 12:34:56');
1467981296

desteklenmeyen biçim #

Tarih/saat değerlerini veritabanında başka herhangi bir biçimde saklamak mümkün olabilir, ancak yerleşik tarih/saat işlevleri bunları ayrıştırmaz ve NULL döndürür:

> SELECT time('1:30:00');   -- not two digits

> SELECT datetime('8 Jul 2016');
 

Sütun türlerini zorunlu kılma #

SQLite, dinamik yazma kullanır ve bildirilen sütun türlerini yok sayar:

> CREATE TABLE Test (
      Col1 INTEGER,
      Col2 VARCHAR(2),       -- length is ignored, too
      Col3 BLOB,
      Col4,                  -- no type required
      Col5 FLUFFY BUNNIES    -- use whatever you want
  );
> INSERT INTO Test VALUES (1, 1, 1, 1, 1);
> INSERT INTO Test VALUES ('xxx', 'xxx', 'xxx', 'xxx', 'xxx');
> SELECT * FROM Test;
1   1   1   1   1         
xxx xxx xxx xxx xxx

(Ancak, tür benzeşimi için bildirilen sütun türleri kullanılır.)

Türleri zorlamak için [typeof() function](https://www.wikiod.com/tr/sqlite/veri-tipleri#TÜR işlevi) ile bir kısıtlama eklemeniz gerekir:

CREATE TABLE Tab (
    Col1 TEXT   CHECK (typeof(Col1) = 'text' AND length(Col1) <= 10),
    [...]
);

(Böyle bir sütun NULL olabilirse, “null"a açıkça izin vermeniz gerekir.)

TÜR işlevi #

sqlite> SELECT TYPEOF(NULL);
null
sqlite> SELECT TYPEOF(42);
integer
sqlite> SELECT TYPEOF(3.141592653589793);
real
sqlite> SELECT TYPEOF('Hello, world!');
text
sqlite> SELECT TYPEOF(X'0123456789ABCDEF');
blob

Booleanları kullanma #

Booleanlar için SQLite “0” ve “1” tamsayılarını kullanır:

sqlite> SELECT 2 + 2 = 4;
1
sqlite> SELECT 'a' = 'b';
0
sqlite> SELECT typeof('a' = 'b');
integer

> CREATE TABLE Users ( Name, IsAdmin );
> INSERT INTO Users VALUES ('root', 1);
> INSERT INTO Users VALUES ('john', 0);
> SELECT Name FROM Users WHERE IsAdmin;
root
