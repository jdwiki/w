Sözdizimi #

  • fun TypeName.extensionName(params, …) { /* gövde */ } // Bildirim
  • fun <T: Any> TypeNameWithGenerics<T>.extensionName(params, …) { /* body */ } // Generics ile Bildirim
  • myObj.extensionName(args, …) // çağırma

Uzantılar statik olarak çözümlenir. Bu, kullanılacak uzantı yönteminin, erişmekte olduğunuz değişkenin referans türüne göre belirlendiği anlamına gelir; çalışma zamanında değişkenin türü ne olursa olsun, her zaman aynı uzantı yöntemi çağrılır. Bunun nedeni, bir uzantı yöntemi bildirmenin, alıcı türüne gerçekte bir üye eklememesidir.

Potansiyel Tuzak: Uzantılar Statik Olarak Çözümlendi #

Çağrılacak uzantı yöntemi, erişilen değişkenin referans türüne göre derleme zamanında belirlenir. Çalışma zamanında değişkenin türü ne olursa olsun, her zaman aynı uzantı yöntemi çağrılır.

open class Super

class Sub : Super()

fun Super.myExtension() = "Defined for Super"

fun Sub.myExtension() = "Defined for Sub"

fun callMyExtension(myVar: Super) {
    println(myVar.myExtension())
}

callMyExtension(Sub())

Yukarıdaki örnekte “Defined for Super” yazdırılacaktır, çünkü “myVar” değişkeninin bildirilen türü “Süper"dir.

Üst Düzey Uzantılar #

Üst düzey uzantı yöntemleri bir sınıf içinde yer almaz.

fun IntArray.addTo(dest: IntArray) {
    for (i in 0 .. size - 1) {
        dest[i] += this[i]
    }
}

Yukarıda IntArray tipi için bir uzatma metodu tanımlanmıştır. Uzantı yönteminin tanımlandığı nesneye (alıcı olarak adlandırılır) ’this’ anahtar sözcüğü kullanılarak erişildiğini unutmayın.

Bu uzantı şöyle çağrılabilir:

val myArray = intArrayOf(1, 2, 3)
intArrayOf(4, 5, 6).addTo(myArray)

Tembel uzantı özelliği geçici çözümü #

Hesaplaması pahalı olan bir uzantı özelliği oluşturmak istediğinizi varsayalım. Bu nedenle, tembel mülk temsilcisi kullanarak hesaplamayı önbelleğe almak ve mevcut örneğe (‘bu’) atıfta bulunmak istiyorsunuz, ancak bunu Kotlin sorunları [KT-9686][2’de açıklandığı gibi yapamazsınız. ] ve KT-13053. Ancak, burada sağlanan resmi bir geçici çözüm vardır.

Örnekte, extension özelliği “color” dur. Açık bir “colorCache” kullanır ve “bu” ile birlikte “tembel” gerekmediği için kullanılabilir:

class KColor(val value: Int)

private val colorCache = mutableMapOf<KColor, Color>()

val KColor.color: Color
    get() = colorCache.getOrPut(this) { Color(value, true) }

Java 7+ Path sınıfını genişleten örnek #

Uzantı yöntemleri için yaygın bir kullanım durumu, mevcut bir API’yi geliştirmektir. Java 7+ “Path” sınıfına “exist”, “notExists” ve “deleteRecursively” ekleme örnekleri:

fun Path.exists(): Boolean = Files.exists(this)
fun Path.notExists(): Boolean = !this.exists()
fun Path.deleteRecursively(): Boolean = this.toFile().deleteRecursively()

Şimdi bu örnekte çağrılabilir:

val dir = Paths.get(dirName)
if (dir.exists()) dir.deleteRecursively()

İnsan tarafından okunabilir bir dize oluşturmak için uzun süren örnek #

İnsan tarafından okunabilir bir dize oluşturmak için “Int” veya “Long” türünde herhangi bir değer verildiğinde:

fun Long.humanReadable(): String {
    if (this <= 0) return "0"
    val units = arrayOf("B", "KB", "MB", "GB", "TB", "EB")
    val digitGroups = (Math.log10(this.toDouble())/Math.log10(1024.0)).toInt();
    return DecimalFormat("#,##0.#").format(this/Math.pow(1024.0, digitGroups.toDouble())) + " " + units[digitGroups];
}

fun Int.humanReadable(): String {
    return this.toLong().humanReadable()
}

Daha sonra kolayca şu şekilde kullanılır:

println(1999549L.humanReadable())
println(someInt.humanReadable())

ISO formatlı bir dize oluşturmak için Java 8 Temporal sınıflarını genişletme örneği #

Bu beyanname ile:

fun Temporal.toIsoString(): String = DateTimeFormatter.ISO_INSTANT.format(this)

Artık basitçe şunları yapabilirsiniz:

val dateAsString = someInstant.toIsoString()

Okunabilirliği artırmak için uzantı işlevlerini kullanma #

Kotlin’de şöyle bir kod yazabilirsiniz:

val x: Path = Paths.get("dirName").apply { 
    if (Files.notExists(this)) throw IllegalStateException("The important file does not exist")
}

Ancak ‘apply’ kullanımı, niyetinize göre o kadar açık değil. Bazen, eylemi yeniden adlandırmak ve daha açık hale getirmek için benzer bir uzantı işlevi oluşturmak daha açıktır. Bunun kontrolden çıkmasına izin verilmemeli, ancak doğrulama gibi çok yaygın eylemler için:

infix inline fun <T> T.verifiedBy(verifyWith: (T) -> Unit): T {
    verifyWith(this)
    return this
}

infix inline fun <T: Any> T.verifiedWith(verifyWith: T.() -> Unit): T {
    this.verifyWith()
    return this
}

Artık kodu şu şekilde yazabilirsiniz:

val x: Path = Paths.get("dirName") verifiedWith {
    if (Files.notExists(this)) throw IllegalStateException("The important file does not exist")
}

Bu da insanların lambda parametresi içinde ne beklemeleri gerektiğini bilmelerini sağlar.

‘verifiedBy’ için ‘T’ tür parametresinin ‘T: Any?’ ile aynı olduğuna dikkat edin; bu, null yapılabilir türlerin bile uzantının bu sürümünü kullanabileceği anlamına gelir. ‘verifiedWith’ null olamaz gerektirse de.

Tamamlayıcı Nesnelere genişletme işlevleri (Statik işlevlerin görünümü) #

Bir sınıfı statik bir işlevmişsiniz gibi genişletmek istiyorsanız, örneğin ‘Something’ sınıfı için ‘fromString’ statik görünümlü işlev ekleyin, bu yalnızca sınıfın bir tamamlayıcı nesnesi varsa çalışabilir .org/docs/reference/object-declarations.html#companion-objects) ve uzantı işlevinin eşlik eden nesne üzerinde bildirildiğini:

class Something {
    companion object {}
}

class SomethingElse {
}

fun Something.Companion.fromString(s: String): Something = ... 

fun SomethingElse.fromString(s: String): SomethingElse = ... 

fun main(args: Array<String>) {
    Something.fromString("") //valid as extension function declared upon the
                             //companion object

    SomethingElse().fromString("") //valid, function invoked on instance not
                                   //statically

    SomethingElse.fromString("") //invalid
}

inline fun <reified T : View> Activity.findOptional(id: Int): T? = findViewById(id) as? T
inline fun <reified T : View> Fragment.findOptional(id: Int): T? = view?.findViewById(id) as? T

val yourButton by lazy { find<Button>(R.id.yourButtonId) }
