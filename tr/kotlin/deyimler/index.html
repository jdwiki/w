Kotlin’de seri hale getirilebilir ve serialVersionUid #

Kotlin’de bir sınıf için ‘serialVersionUID’ oluşturmak için, tümü sınıfın eşlik eden nesnesine üye eklemeyi içeren birkaç seçeneğiniz vardır.

En özlü bayt kodu, içeren sınıfta özel bir statik değişken olacak bir “özel sabit değer"den gelir, bu durumda “MySpecialCase”:

class MySpecialCase : Serializable {
    companion object {
        private const val serialVersionUID: Long = 123
    }
}

Bu formları da kullanabilirsiniz, her biri serileştirme için gerekli olmayan alıcı/ayarlayıcı yöntemlerine sahip olmanın bir yan etkisi olan…

class MySpecialCase : Serializable {
    companion object {
        private val serialVersionUID: Long = 123
    }
}

Bu, statik alanı yaratır, ancak aynı zamanda eşlik eden nesne üzerinde gereksiz olan bir “getSerialVersionUID” alıcısını da oluşturur.

class MySpecialCase : Serializable {
    companion object {
        @JvmStatic private val serialVersionUID: Long = 123
    }
}

Bu, statik alanı oluşturur, ancak aynı zamanda ‘MySpecialCase’ içeren sınıf üzerinde gereksiz olan ‘getSerialVersionUID’ statik bir alıcı da oluşturur.

Ancak tümü, bir “Serileştirilebilir” sınıfa “serialVersionUID” ekleme yöntemi olarak çalışır.

Genel kurucuda sağlamadan bir sınıfa yetki verin #

Bir sınıfa yetki vermek istediğinizi, ancak yapıcı parametresinde temsilci sınıfı sağlamak istemediğinizi varsayalım. Bunun yerine, onu özel olarak inşa etmek ve kurucu arayanın bundan habersiz olmasını istersiniz. İlk başta bu imkansız görünebilir, çünkü sınıf delegasyonu yalnızca kurucu parametrelerine yetki verilmesine izin verir. Ancak, [bu cevap2‘da verildiği gibi bunu yapmanın bir yolu vardır:

class MyTable private constructor(table: Table<Int, Int, Int>) : Table<Int, Int, Int> by table {

    constructor() : this(TreeBasedTable.create()) // or a different type of table if desired

}

Bununla, ‘MyTable’ yapıcısını şöyle çağırabilirsiniz: ‘MyTable()’. “MyTable” delegelerinin özel olarak oluşturulacağı “Table<Int, Int, Int>”. Yapıcı arayan bu konuda hiçbir şey bilmiyor.

Bu örnek, bu SO sorusuna dayanmaktadır.

Null yapılabilir nesnelerle çalışmayı basitleştirmek için let veya ayrıca kullanın #

Kotlin’deki ’let’, çağrıldığı nesneden yerel bir bağlama oluşturur. Örnek:

val str = "foo"
str.let {
    println(it) // it
}

Bu, "foo" yazdıracak ve Birim döndürecektir.

’let’ ve ‘ayrıca’ arasındaki fark, bir ’let’ bloğundan herhangi bir değer döndürebilmenizdir. Öte yandan ‘ayrıca’ her zaman ‘Birim’i yeniden döndürür.

Şimdi bu neden faydalı, soruyorsun? Çünkü ’null’ döndürebilen bir metot çağırırsanız ve sadece bu dönüş değeri ’null’ olmadığında bazı kodları çalıştırmak istiyorsanız, ’let’ veya ‘ayrıca’ şu şekilde kullanabilirsiniz:

val str: String? = someFun()
str?.let {
    println(it)
}

Bu kod parçası, yalnızca “str” ​​"boş” olmadığında “let” bloğunu çalıştıracaktır. “boş” güvenlik operatörüne (?) dikkat edin.

Listeyi filtreleme #

val list = listOf(1,2,3,4,5,6)

//filter out even numbers

val even = list.filter { it % 2 == 0 }

println(even) //returns [2,4]

Kotlin’de akıcı yöntemler #

Kotlin’deki akıcı yöntemler Java ile aynı olabilir:

fun doSomething() {
   someOtherAction()
   return this
}

Ancak, aşağıdaki gibi bir uzantı işlevi oluşturarak bunları daha işlevsel hale getirebilirsiniz:

fun <T: Any> T.fluently(func: ()->Unit): T {
    func()
    return this
}

Hangi daha sonra açıkça akıcı işlevlere izin verir:

fun doSomething() {
   return fluently { someOtherAction() }
}

Nesneleri başlatmak veya yöntem zincirleme elde etmek için Apply’ı kullanın #

‘Apply’ dokümantasyonu şunları söylüyor:

alıcısı olarak this değeri ile belirtilen fonksiyon bloğunu çağırır ve this değerini döndürür.

kdoc pek yardımcı olmasa da ‘apply’ gerçekten de kullanışlı bir işlevdir. Layman’ın terimleriyle “apply”, “bu"nun sizin “apply” adını verdiğiniz nesneye bağlı olduğu bir kapsam oluşturur. Bu, daha sonra geri döneceğiniz bir nesne üzerinde birden fazla yöntem çağırmanız gerektiğinde bazı kodlar ayırmanıza olanak tanır. Örnek:

File(dir).apply { mkdirs() }

Bunu yazmakla aynı şey:

fun makeDir(String path): File {
    val result = new File(path)
    result.mkdirs()
    return result
}

DTO’lar (POJO’lar/POCO’lar) oluşturma #

Kotlin’deki veri sınıfları, verileri tutmaktan başka bir şey yapmayan sınıflardır. Bu tür sınıflar “veri” olarak işaretlenir:

data class User(var firstname: String, var lastname: String, var age: Int)

Yukarıdaki kod, aşağıdakilerin otomatik olarak oluşturulduğu bir “Kullanıcı” sınıfı oluşturur:

  • Tüm özellikler için Alıcılar ve Ayarlayıcılar (yalnızca valler için alıcılar)
  • eşittir()
  • hashcode()
  • toString()
  • kopyala()
  • “bileşenN()” (burada “N”, bildirim sırasına göre karşılık gelen özelliktir)

Tıpkı bir işlevde olduğu gibi, varsayılan değerler de belirtilebilir:

data class User(var firstname: String = "Joe", var lastname: String = "Bloggs", var age: Int = 20)

Daha fazla ayrıntı burada Veri Sınıfları bulunabilir.
