Varsayılan uygulamalara sahip arayüz #

Kotlin’deki bir arabirim, işlevler için varsayılan uygulamalara sahip olabilir:

interface MyInterface {
    fun withImplementation() {
      print("withImplementation() was called")
    }
}

Bu tür arabirimleri uygulayan sınıflar, bu işlevleri yeniden uygulamadan kullanabilecektir.

class MyClass: MyInterface {
    // No need to reimplement here
}
val instance = MyClass()
instance.withImplementation()

Özellikleri #

Varsayılan uygulamalar, mülk alıcıları ve ayarlayıcılar için de çalışır:

interface MyInterface2 {
    val helloWorld
        get() = "Hello World!"
}

Arabirim erişimcileri uygulamaları, destek alanlarını kullanamaz

interface MyInterface3 {
    // this property won't compile!
    var helloWorld: Int
        get() = field
        set(value) { field = value }
}

Çoklu uygulamalar #

Birden çok arabirim aynı işlevi uyguladığında veya tümü bir veya daha fazla uygulama ile tanımlandığında, türetilmiş sınıfın uygun çağrıyı manuel olarak çözmesi gerekir.

interface A {
    fun notImplemented()
    fun implementedOnlyInA() { print("only A") }
    fun implementedInBoth() { print("both, A") }
    fun implementedInOne() { print("implemented in A") }
}

interface B {
    fun implementedInBoth() { print("both, B") }
    fun implementedInOne() // only defined
}

class MyClass: A, B {
    override fun notImplemented() { print("Normal implementation") }

    // implementedOnlyInA() can by normally used in instances

    // class needs to define how to use interface functions
    override fun implementedInBoth() {
        super<B>.implementedInBoth()
        super<A>.implementedInBoth()
    }

    // even if there's only one implementation, there multiple definitions
    override fun implementedInOne() {
        super<A>.implementedInOne()
        print("implementedInOne class implementation")
    }
}

Temel Arayüz #

Bir Kotlin arabirimi, durumu depolayamasalar da, soyut yöntemlerin bildirimlerini ve varsayılan yöntem uygulamalarını içerir.

interface MyInterface {
    fun bar()
}

Bu arayüz artık bir sınıf tarafından aşağıdaki gibi uygulanabilir:

class Child : MyInterface {
   override fun bar() {
       print("bar() was called")
   }
}

Arayüzlerdeki Özellikler #

Arayüzlerde özellikleri bildirebilirsiniz. Bir arabirimin durumu olamayacağından, bir özelliği yalnızca soyut olarak veya erişimciler için varsayılan uygulama sağlayarak bildirebilirsiniz.

interface MyInterface {
    val property: Int // abstract

    val propertyWithImplementation: String
        get() = "foo"

    fun foo() {
        print(property)
    }
}

class Child : MyInterface {
    override val property: Int = 29
}

süper anahtar kelime #

interface MyInterface {
    fun funcOne() {
        //optional body
        print("Function with default implementation")
    }
}

Arayüzdeki yöntemin kendi varsayılan uygulaması varsa, ona erişmek için super anahtar sözcüğünü kullanabiliriz.

super.funcOne()

Varsayılan Uygulamalarla Birden Çok Arayüz Uygulanırken Çakışmalar #

Varsayılan uygulamaları içeren aynı ada sahip yöntemlere sahip birden fazla arabirim uygularken, hangi uygulamanın kullanılması gerektiği derleyici için belirsizdir. Bir çakışma olması durumunda, geliştiricinin çakışan yöntemi geçersiz kılması ve özel bir uygulama sağlaması gerekir. Bu uygulama, varsayılan uygulamalara yetki vermeyi veya seçmemeyi seçebilir.

interface FirstTrait {
    fun foo() { print("first") }
    fun bar()
}

interface SecondTrait {
    fun foo() { print("second") }
    fun bar() { print("bar") }
}

class ClassWithConflict : FirstTrait, SecondTrait {
    override fun foo() {
        super<FirstTrait>.foo()  // delegate to the default implementation of FirstTrait
        super<SecondTrait>.foo() // delegate to the default implementation of SecondTrait
    }

    // function bar() only has a default implementation in one interface and therefore is ok.
}
