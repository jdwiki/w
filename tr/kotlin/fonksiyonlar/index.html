Sözdizimi #

  • eğlenceli Ad(Params) = …
  • eğlenceli Ad(Params) {…}
  • eğlenceli Ad(Params): Tür {…}
  • fun <Tür Argümanı> Ad(Params): Tür {…}
  • satır içi eğlence Ad(Params): Tür {…}
  • { ArgName: ArgType -> … }
  • { ArgName -> … }
  • { ArgNames -> … }
  • { (ArgName: ArgType): Tür -> … }

Parametreler #

Parametre Ayrıntılar
Ad Fonksiyonun adı
Params Bir isim ve tip ile fonksiyona verilen değerler: Name:Type
Type Fonksiyonun tipini döndür
Tür Argümanı Genel programlamada kullanılan tür parametresi (mutlaka dönüş türü değildir)
ArgName Fonksiyona verilen değerin adı
ArgType ArgName için tür belirteci
ArgNames Virgülle ayrılmış ArgName listesi

İşlev Referansları #

Bir işleve, işlevin adının önüne :: ekleyerek onu çağırmadan başvuruda bulunabiliriz. Bu daha sonra başka bir işlevi parametre olarak kabul eden bir işleve geçirilebilir.

fun addTwo(x: Int) = x + 2
listOf(1, 2, 3, 4).map(::addTwo) # => [3, 4, 5, 6]

Alıcısı olmayan işlevler (ParamTypeA, ParamTypeB, ...) -> ReturnTypea dönüştürülür; burada ParamTypeA, ParamTypeB … işlev parametrelerinin türüdür ve `ReturnType1 işlev dönüş değerinin türüdür .

fun foo(p0: Foo0, p1: Foo1, p2: Foo2): Bar {
    //...
}
println(::foo::class.java.genericInterfaces[0]) 
// kotlin.jvm.functions.Function3<Foo0, Foo1, Foo2, Bar>
// Human readable type: (Foo0, Foo1, Foo2) -> Bar

Alıcılı işlevlerin (uzantı işlevi veya üye işlevi olabilir) farklı bir sözdizimi vardır. Çift kolondan önce alıcının tip adını eklemelisiniz:

class Foo
fun Foo.foo(p0: Foo0, p1: Foo1, p2: Foo2): Bar {
    //...
}
val ref = Foo::foo
println(ref::class.java.genericInterfaces[0]) 
// kotlin.jvm.functions.Function4<Foo, Foo0, Foo1, Foo2, Bar>
// Human readable type: (Foo, Foo0, Foo1, Foo2) -> Bar
// takes 4 parameters, with receiver as first and actual parameters following, in their order

// this function can't be called like an extension function, though
val ref = Foo::foo
Foo().ref(Foo0(), Foo1(), Foo2()) // compile error

class Bar {
    fun bar()
}
print(Bar::bar) // works on member functions, too.

Bununla birlikte, bir fonksiyonun alıcısı bir nesne olduğunda, alıcı parametre listesinden çıkarılır, çünkü bunlar bu tipte yalnızca bir örnektir.

object Foo
fun Foo.foo(p0: Foo0, p1: Foo1, p2: Foo2): Bar {
    //...
}
val ref = Foo::foo
println(ref::class.java.genericInterfaces[0]) 
// kotlin.jvm.functions.Function3<Foo0, Foo1, Foo2, Bar>
// Human readable type: (Foo0, Foo1, Foo2) -> Bar
// takes 3 parameters, receiver not needed

object Bar {
    fun bar()
}
print(Bar::bar) // works on member functions, too.

Kotlin 1.1’den beri, fonksiyon referansı, daha sonra sınırlı fonksiyon referansı olarak adlandırılan bir değişkene sınırlı olabilir.

fun makeList(last: String?): List<String> {
    val list = mutableListOf("a", "b", "c")
    last?.let(list::add)
    return list
}
son sürüm

Bu örneğin yalnızca sınırlı işlev başvurusunun nasıl çalıştığını göstermek için verildiğine dikkat edin. Diğer tüm açılardan kötü bir uygulamadır.

Ancak özel bir durum var. Üye olarak bildirilen bir uzantı işlevine başvurulamaz.

class Foo
class Bar {
    fun Foo.foo() {}
    val ref = Foo::foo // compile error
}

Temel fonksiyonlar #

İşlevler, ‘fun’ anahtar sözcüğü ve ardından bir işlev adı ve herhangi bir parametre kullanılarak bildirilir. Varsayılan olarak “Birim” olan bir işlevin dönüş türünü de belirtebilirsiniz. Fonksiyonun gövdesi parantez {} içine alınır. Dönüş türü “Birim"den farklıysa, gövde, gövde içindeki her sonlandırıcı dal için bir dönüş ifadesi yayınlamalıdır.

fun sayMyName(name: String): String {
    return "Your name is $name" 
}

Aynı şeyin kısa bir versiyonu:

fun sayMyName(name: String): String = "Your name is $name"

Ve tür, çıkarılabildiği için atlanabilir:

fun sayMyName(name: String) = "Your name is $name"

Satır İçi İşlevler #

İşlevler “satır içi” öneki kullanılarak satır içi olarak bildirilebilir ve bu durumda C’deki makrolar gibi davranırlar - çağrılmak yerine, derleme zamanında işlevin gövde koduyla değiştirilirler. Bu, bazı durumlarda, özellikle lambdaların fonksiyon parametreleri olarak kullanıldığı durumlarda, performans faydalarına yol açabilir.

inline fun sayMyName(name: String) = "Your name is $name"

C makrolarından bir fark, satır içi işlevlerin çağrıldıkları kapsama erişememeleridir:

inline fun sayMyName() = "Your name is $name"

fun main() {
    val name = "Foo"
    sayMyName() # => Unresolved reference: name
}

Lambda İşlevleri #

Lambda işlevleri, genellikle bir işlev çağrısı sırasında işlev parametresi olarak işlev görmek için oluşturulan anonim işlevlerdir. Bunlar, {parantez} ile çevrelenen ifadelerle bildirilir - argüman gerekirse, bunlar bir ok -> önüne konur.

{ name: String ->
    "Your name is $name" //This is returned
}

Lambda işlevindeki son ifade otomatik olarak dönüş değeridir.

Lambda’yı derleyicinin türleri çıkarabileceği bir yere koyarsanız, türler isteğe bağlıdır.

Çoklu argümanlar:

{ argumentOne:String, argumentTwo:String ->
    "$argumentOne - $argumentTwo"
}

Lambda işlevi yalnızca bir argümana ihtiyaç duyuyorsa, argüman listesi atlanabilir ve bunun yerine tek argümana “it” kullanılarak başvurulabilir.

{ "Your name is $it" }

Bir işlevin tek argümanı bir lambda işleviyse, işlev çağrısından parantezler tamamen çıkarılabilir.

# These are identical
listOf(1, 2, 3, 4).map { it + 2 }
listOf(1, 2, 3, 4).map({ it + 2 })

Diğer İşlevleri Alan İşlevler #

“Lambda Functions” bölümünde görüldüğü gibi, fonksiyonlar parametre olarak diğer fonksiyonları alabilir. Diğer işlevleri alan işlevleri bildirmeniz gereken “işlev türü” aşağıdaki gibidir:

# Takes no parameters and returns anything
() -> Any?

# Takes a string and an integer and returns ReturnType
(arg1: String, arg2: Int) -> ReturnType

Örneğin, bir lambda işlevini iki kez yürüten bir işlevi bildirmek için en belirsiz tür olan () -> Any? kullanabilirsiniz:

fun twice(x: () -> Any?) {
    x(); x();
}

fun main() {
    twice {
        println("Foo")
    } # => Foo
      # => Foo
}

Operatör işlevleri #

Kotlin, önceden tanımlanmış bir dizi operatör için sabit sembolik temsili ("+” veya “*” gibi) ve sabit önceliği olan uygulamalar sağlamamıza izin verir. Bir operatörü uygulamak için, karşılık gelen tip için bir üye işlevi veya sabit bir adla bir uzantı işlevi sağlıyoruz. Operatörleri aşırı yükleyen işlevlerin “operatör” değiştiricisiyle işaretlenmesi gerekir:

data class IntListWrapper (val wrapped: List<Int>) {
    operator fun get(position: Int): Int = wrapped[position]
}

val a = IntListWrapper(listOf(1, 2, 3))
a[1] // == 2

Daha fazla operatör işlevi burada içinde bulunabilir

Steno Fonksiyonları #

Bir işlev yalnızca bir ifade içeriyorsa, parantezleri atlayabilir ve bunun yerine değişken ataması gibi bir eşittir kullanabiliriz. İfadenin sonucu otomatik olarak döndürülür.

fun sayMyName(name: String): String = "Your name is $name"
