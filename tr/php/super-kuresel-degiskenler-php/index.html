Süper küreseller, her zaman tüm kapsamlarda bulunan yerleşik değişkenlerdir.

PHP’de önceden tanımlanmış birkaç değişken “süper küreseldir”, yani bir komut dosyası boyunca tüm kapsamlarda kullanılabilirler. Bunlara işlevler veya yöntemler içinde erişmek için global $variable; yapmaya gerek yoktur.

Suberglobals açıkladı #

Giriiş #

Basitçe söylemek gerekirse, bunlar komut dosyalarınızda all kapsamında bulunan değişkenlerdir.

Bu, onları fonksiyonlarınızda parametre olarak geçirmenize veya farklı kapsamlarda kullanılabilir olmaları için bir kod bloğunun dışında saklamanıza gerek olmadığı anlamına gelir.

Süper küresel nedir?? #

Bunların süper kahramanlar gibi olduğunu düşünüyorsanız - değiller.

PHP 7.1.3 sürümünden itibaren 9 süper küresel değişken vardır. Bunlar aşağıdaki gibidir:

  • $GLOBALS - Global kapsamda mevcut olan tüm değişkenlere referans verir
  • $_SERVER - Sunucu ve yürütme ortamı bilgileri
  • “$_GET” - HTTP GET değişkenleri
  • $_POST - HTTP POST değişkenleri
  • $_FILES - HTTP Dosya Yükleme değişkenleri
  • $_COOKIE - HTTP Çerezleri
  • $_SESSION - Oturum değişkenleri
  • “$_REQUEST” - HTTP İsteği değişkenleri
  • $_ENV - Ortam değişkenleri

Belgelere bakın.

Gres referansı için üzgünüm! Bağlantı

Bu süperkahramanlargloballer hakkında biraz açıklama yapmanın zamanı geldi.

$KÜRESEL #

Komut dosyasının global kapsamında halihazırda tanımlanmış olan tüm değişkenlere referansları içeren bir ilişkisel dizi. Değişken isimleri dizinin anahtarlarıdır.

kod

$myGlobal = "global"; // declare variable outside of scope

function test()
{
    $myLocal = "local"; // declare variable inside of scope
    // both variables are printed
    var_dump($myLocal);
    var_dump($GLOBALS["myGlobal"]);
}

test(); // run function
// only $myGlobal is printed since $myLocal is not globally scoped
var_dump($myLocal);
var_dump($myGlobal);

Çıktı

string 'local' (length=5)
string 'global' (length=6)
null
string 'global' (length=6)

Yukarıdaki örnekte $myLocal, test() işlevi içinde bildirildiği ve işlev kapatıldıktan sonra yok edildiği için ikinci kez görüntülenmez.

Küresel olmak #

Bunu gidermek için iki seçenek vardır.

Birinci seçenek: global anahtar kelime

function test()
{
    global $myLocal;
    $myLocal = "local";
    var_dump($myLocal);
    var_dump($GLOBALS["myGlobal"]);
}

“Global” anahtar sözcüğü, onu global kapsamın parçası olmaya zorlayan bir değişkenin önekidir.

Global anahtar kelimeyle aynı ifadede bir değişkene değer atayamayacağınızı unutmayın. Bu nedenle, neden altına bir değer atamak zorunda kaldım. (Yeni satırları ve boşlukları kaldırırsanız mümkündür, ancak bunun düzgün olduğunu düşünmüyorum. global $myLocal; $myLocal = "local").

İkinci seçenek: $GLOBALS dizisi

function test()
{
    $GLOBALS["myLocal"] = "local";
    $myLocal = $GLOBALS["myLocal"];
    var_dump($myLocal);
    var_dump($GLOBALS["myGlobal"]);
}

Bu örnekte, ilişkisel dizi yerine bir değişken adı yazmayı daha kolay bulduğumdan, $GLOBAL["myLocal"] değerini $myLocala yeniden atadım.

$_SERVER, başlıklar, yollar ve komut dosyası konumları gibi bilgileri içeren bir dizidir. Bu dizideki girişler web sunucusu tarafından oluşturulur. Her web sunucusunun bunlardan herhangi birini sağlayacağının garantisi yoktur; sunucular bazılarını atlayabilir veya burada listelenmeyen diğerlerini sağlayabilir.   • HTTP_USER_AGENT - Kullanıcı aracısının içeriği. Bu, işletim sistemi de dahil olmak üzere istemcinin tarayıcısıyla ilgili tüm bilgileri içeren bir dizedir.

  • “REQUEST_TIME” - İsteğin başlangıcının zaman damgası. PHP 5.1.0’dan beri mevcuttur.

“$_GET” #

Geçerli komut dosyasına URL parametreleri aracılığıyla iletilen ilişkisel bir değişken dizisi.

“$_GET”, tüm URL parametrelerini içeren bir dizidir; bunlar sonra ne var? URL’de.

Örnek olarak http://www.example.com/index.php?myVar=myVal kullanılarak. Bu URL’deki bu bilgiler, $_GET["myVar"] biçiminde erişilerek elde edilebilir ve bunun sonucu myVal olacaktır.

Okumayı sevmeyenler için bazı kodlar kullanmak.

// URL = http://www.example.com/index.php?myVar=myVal
echo $_GET["myVar"] == "myVal" ? "true" : "false"; // returns "true"

Yukarıdaki örnek, [üçlü operatör](https://www.wikiod.com/tr/php/operatorler#Üçlü operatör (?:)) kullanır.

Bu, süper küresel “$_GET” kullanarak URL’deki değere nasıl erişebileceğinizi gösterir.

Şimdi başka bir örnek! gasp

// URL = http://www.example.com/index.php?myVar=myVal&myVar2=myVal2
echo $_GET["myVar"]; // returns "myVal"
echo $_GET["myVar2"]; // returns "myVal2"

URL üzerinden birden fazla değişkeni ve işareti (&) ile ayırarak göndermek mümkündür.

Güvenlik riski Bilgisayarın geçmişinde kalacağından ve o tarayıcıya erişebilen herkes tarafından görüleceğinden, hassas bilgilerin URL yoluyla gönderilmemesi çok önemlidir.

$_POST #

İstekte HTTP İçerik Türü olarak application/x-www-form-urlencoded veya multipart/form-data kullanılırken HTTP POST yöntemi aracılığıyla geçerli komut dosyasına geçirilen bir ilişkisel değişken dizisi.

Verilerin bir yerden diğerine gönderilmesinde “$_GET"e çok benzer.

Doğrudan bir örneğe girerek başlayacağım. (Formun bulunduğu sayfaya bilgi göndereceği için action niteliğini atladım).

<form method="POST">
    <input type="text" name="myVar" value="myVal" />
    <input type="submit" name="submit" value="Submit" />
</form>

Yukarıda, verilerin gönderilebileceği temel bir form verilmiştir. Gerçek bir ortamda ‘değer’ niteliği ayarlanmaz, bu da formun boş olacağı anlamına gelir. Bu daha sonra kullanıcı tarafından girilen bilgileri gönderir.

echo $_POST["myVar"]); // returns "myVal"

Güvenlik riski POST yoluyla veri göndermek de güvenli değildir. HTTPS kullanmak, verilerin daha güvenli tutulmasını sağlayacaktır.

$_FILES #

HTTP POST yöntemi aracılığıyla geçerli komut dosyasına yüklenen ilişkilendirilebilir öğeler dizisi. Bu dizinin yapısı POST yöntemi yüklemeleri bölümünde özetlenmiştir.

Temel bir formla başlayalım.

<form method="POST" enctype="multipart/form-data">
    <input type="file" name="myVar" />
    <input type="submit" name="Submit" />
</form>

‘action’ özniteliğini atladığımı unutmayın (yine!). Ayrıca, enctype="multipart/form-data" ekledim, bu dosya yüklemeleriyle uğraşacak herhangi bir form için önemlidir.

// ensure there isn't an error
if ($_FILES["myVar"]["error"] == UPLOAD_ERR_OK)
{
    $folderLocation = "myFiles"; // a relative path. (could be "path/to/file" for example)
    
    // if the folder doesn't exist then make it
    if (!file_exists($folderLocation)) mkdir($folderLocation);

    // move the file into the folder
    move_uploaded_file($_FILES["myVar"]["tmp_name"], "$folderLocation/" . basename($_FILES["myVar"]["name"]));
}

Bu, bir dosya yüklemek için kullanılır. Bazen birden fazla dosya yüklemek isteyebilirsiniz. Bunun için bir nitelik var, buna ‘multiple’ denir. Hemen hemen herhangi bir şey için bir özellik var. Üzgünüm

Aşağıda, birden çok dosya gönderen bir form örneği verilmiştir.

<form method="POST" enctype="multipart/form-data">
    <input type="file" name="myVar[]" multiple="multiple" />
    <input type="submit" name="Submit" />
</form>

Burada yapılan değişiklikleri not edin; sadece birkaç tane var.

  • giriş ismi köşeli parantezler içindedir. Bunun nedeni, artık bir dizi dosya olması ve bu nedenle forma seçilen dosyaların bir dizisini yapmasını söylüyoruz. Köşeli parantezlerin atlanması, ikinci en çok dosyanın $_FILES["myVar"] olarak ayarlanmasına neden olur.

  • multiple="multiple" özelliği. Bu sadece tarayıcıya kullanıcıların birden fazla dosya seçebileceğini söyler.

“$_ÇEREZ” #

HTTP Çerezleri aracılığıyla geçerli komut dosyasına iletilen ilişkisel bir değişken dizisi.

Çerezler, veri içeren ve müşterinin bilgisayarında depolanan değişkenlerdir.

Yukarıda bahsedilen süper küresellerin aksine, tanımlama bilgileri bir işlevle oluşturulmalıdır (ve bir değer atamamalıdır). Sözleşme aşağıdadır.

setcookie("myVar", "myVal", time() + 3600);

Bu örnekte tanımlama bilgisi için bir ad belirtilir (bu örnekte “myVar"dır), bir değer verilir (bu örnekte “myVal"dir, ancak değerini tanımlama bilgisine atamak için bir değişken iletilebilir), ve sonra bir son kullanma süresi verilir (bu örnekte 3600 saniye bir dakika olduğu için bir saattir).

Çerez oluşturma kuralı farklı olsa da, çereze diğerleriyle aynı şekilde erişilir.

echo $_COOKIE["myVar"]; // returns "myVal"

Bir çerezi yok etmek için setcookie tekrar çağrılmalıdır, ancak sona erme süresi geçmişte _herhangi bir zaman olarak ayarlanmıştır. Aşağıya bakınız.

setcookie("myVar", "", time() - 1);
var_dump($_COOKIE["myVar"]); // returns null

Bu, çerezleri kaldıracak ve istemci bilgisayarından kaldıracaktır.

$_SESSION #

Geçerli komut dosyası için kullanılabilen oturum değişkenlerini içeren bir ilişkisel dizi. Bunun nasıl kullanıldığı hakkında daha fazla bilgi için Oturum işlevleri belgelerine bakın.

Oturumlar, sunucu tarafı olmaları dışında çerezlere çok benzer.

Oturumları kullanmak için, oturumların kullanılmasına izin vermek için komut dosyalarınızın en üstüne session_start() eklemelisiniz.

Bir oturum değişkeni ayarlamak, diğer herhangi bir değişkeni ayarlamakla aynıdır. Aşağıdaki örneğe bakın.

$_SESSION["myVar"] = "myVal";

Bir oturum başlatırken, çerez olarak bir rastgele kimlik belirlenir ve “PHPSESSID” olarak adlandırılır ve o geçerli oturum için oturum kimliğini içerecektir. Buna session_id() işlevi çağrılarak erişilebilir.

“unset” işlevini kullanarak oturum değişkenlerini yok etmek mümkündür (öyle ki “unset($_SESSION[“myVar”])” bu değişkeni yok eder). Alternatif, session_destory() çağırmaktır. Bu, tüm oturumu yok edecektir, yani tüm oturum değişkenleri artık mevcut olmayacaktır.

“$_REQUEST” #

Varsayılan olarak $_GET, $_POST içeriğini içeren bir ilişkisel dizi php.net/manual/en/reserved.variables.post.php) ve $_COOKIE.

PHP belgelerinin belirttiği gibi, bu yalnızca “$_GET”, “$_POST” ve “$_COOKIE” öğelerinin tek bir değişkende harmanlanmasıdır.

Bu dizilerin üçünün de aynı ada sahip bir dizine sahip olması mümkün olduğundan, php.ini dosyasında ‘request_order’ adında, üçünden hangisinin önceliğe sahip olduğunu belirleyebilen bir ayar vardır. Örneğin, “GPC”‘ye ayarlanmışsa, soldan sağa okunduğu için “$_COOKIE” değeri kullanılacaktır, yani “$_REQUEST” değerini “$_GET” olarak ayarlayacaktır, sonra $_POST ve sonra $_COOKIE ve $_COOKIE son olduğundan bu, $_REQUEST içindeki değerdir. [Bu soruya] bakın(http://stackoverflow.com/questions/43157933/what-is-the-request-precedence).

$_ENV #

Ortam yöntemi aracılığıyla geçerli komut dosyasına iletilen ilişkisel bir değişken dizisi.

Bu değişkenler, PHP ayrıştırıcısının çalıştığı ortamdan PHP’nin global ad alanına aktarılır. Çoğu, PHP’nin altında çalıştığı kabuk tarafından sağlanır ve farklı sistemler muhtemelen farklı türde kabuklar çalıştırır, kesin bir liste imkansızdır. Tanımlanmış ortam değişkenlerinin bir listesi için lütfen kabuğunuzun belgelerine bakın.

Diğer ortam değişkenleri, PHP’nin bir sunucu modülü veya CGI işlemcisi olarak çalışmasına bakılmaksızın oraya yerleştirilen CGI değişkenlerini içerir.

$_ENV içinde depolanan her şey PHP’nin çalıştığı ortamdandır.

“$_ENV” yalnızca “php.ini” izin veriyorsa doldurulur. “$_ENV” öğesinin neden doldurulmadığı hakkında daha fazla bilgi için bu yanıta bakın.

$_ENV: Bu SuperGlobal Değişken Kabuk Ortamı Değişkeni, PHP’nin altında çalıştığı ayrıntılar.

$_COOKIE: Bu Süper Küresel Değişken, verilen Anahtar ile Çerez değerini almak için kullanılır.

<?php
$cookie_name = "data";
$cookie_value = "Foo Bar";
setcookie($cookie_name, $cookie_value, time() + (86400 * 30), "/"); // 86400 = 1 day
if(!isset($_COOKIE[$cookie_name])) {
    echo "Cookie named '" . $cookie_name . "' is not set!";
}
else {
    echo "Cookie '" . $cookie_name . "' is set!<br>";
    echo "Value is: " . $_COOKIE[$cookie_name];
}

/**
    Output
    Cookie 'data' is set!
    Value is: Foo Bar
*/
?>
