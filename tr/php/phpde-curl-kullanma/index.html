Sözdizimi #

  • kaynak curl_init ([ string $url = NULL ] )
  • bool curl_setopt (kaynak $ch, int $seçenek, karışık $değer)
  • bool curl_setopt_array (kaynak $ch, dizi $seçenekler)
  • karışık curl_exec (kaynak $ch )
  • geçersiz curl_close (kaynak $ch )

Parametreler #

parametre Ayrıntılar
curl_init – Bir cURL oturumu başlatma
url cURL isteğinde kullanılacak url
curl_setopt – cURL aktarımı için bir seçenek belirleyin
ch cURL tanıtıcısı (curl_init()‘den dönüş değeri)
seçenek CURLOPT_XXX ayarlanacak - seçenekler ve kabul edilebilir değerler listesi için PHP belgelerine bakın
değer Verilen seçenek için cURL tutamağında ayarlanacak değer
curl_exec – Bir cURL oturumu gerçekleştirin
ch cURL tanıtıcısı (curl_init()‘den dönüş değeri)
curl_close – Bir cURL oturumunu kapatın
ch cURL tanıtıcısı (curl_init()‘den dönüş değeri)

POST İstekleri #

HTML formu POST eylemini taklit etmek istiyorsanız, cURL kullanabilirsiniz.

// POST data in array
$post = [
    'a' => 'apple',
    'b' => 'banana'
];

// Create a new cURL resource with URL to POST
$ch = curl_init('http://www.example.com');

// We set parameter CURLOPT_RETURNTRANSFER to read output
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

// Let's pass POST data
curl_setopt($ch, CURLOPT_POSTFIELDS, $post);

// We execute our request, and get output in a $response variable
$response = curl_exec($ch);

// Close the connection
curl_close($ch);

Temel Kullanım (GET İstekleri) #

cURL, URL sözdizimi ile veri aktarımı için bir araçtır. HTTP, FTP, SCP ve diğerlerini destekler (curl >= 7.19.4). Unutmayın, kullanmak için cURL uzantısını yüklemeniz ve etkinleştirmeniz gerekir.

// a little script check is the cURL extension loaded or not
if(!extension_loaded("curl")) {
    die("cURL extension not loaded! Quit Now.");
}

// Actual script start

// create a new cURL resource
// $curl is the handle of the resource
$curl = curl_init();

// set the URL and other options
curl_setopt($curl, CURLOPT_URL, "http://www.example.com");

// execute and pass the result to browser
curl_exec($curl);

// close the cURL resource
curl_close($curl);

Çerezleri Kullanma #

cURL, sonraki isteklerde kullanılmak üzere yanıtlarda alınan çerezleri tutabilir. Bellekte basit oturum tanımlama bilgisi işleme için bu, tek bir kod satırıyla sağlanır:

curl_setopt($ch, CURLOPT_COOKIEFILE, "");

cURL tutamacı yok edildikten sonra tanımlama bilgilerini saklamanız gereken durumlarda, bunları depolamak için dosyayı belirtebilirsiniz:

curl_setopt($ch, CURLOPT_COOKIEJAR, "/tmp/cookies.txt");

Ardından bunları tekrar kullanmak istediğinizde çerez dosyası olarak iletin:

curl_setopt($ch, CURLOPT_COOKIEFILE, "/tmp/cookies.txt");

Ancak, farklı cURL tutamaçları arasında tanımlama bilgileri taşımanız gerekmedikçe bu iki adımın gerekli olmadığını unutmayın. Çoğu kullanım durumu için, tek ihtiyacınız olan, “CURLOPT_COOKIEFILE” öğesini boş dizeye ayarlamaktır.

Çerez işleme, örneğin, oturum açmayı gerektiren bir web sitesinden kaynakları almak için kullanılabilir. Bu tipik olarak iki aşamalı bir prosedürdür. İlk olarak, giriş sayfasına POST.

<?php

# create a cURL handle
$ch  = curl_init();

# set the URL (this could also be passed to curl_init() if desired)
curl_setopt($ch, CURLOPT_URL, "https://www.example.com/login.php");

# set the HTTP method to POST
curl_setopt($ch, CURLOPT_POST, true);

# setting this option to an empty string enables cookie handling
# but does not load cookies from a file
curl_setopt($ch, CURLOPT_COOKIEFILE, "");

# set the values to be sent
curl_setopt($ch, CURLOPT_POSTFIELDS, array(
    "username"=>"joe_bloggs",
    "password"=>"$up3r_$3cr3t",
));

# return the response body
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

# send the request
$result = curl_exec($ch);

İkinci adım (standart hata kontrolü yapıldıktan sonra) genellikle basit bir GET isteğidir. Önemli olan, ikinci istek için mevcut cURL tanıtıcısını yeniden kullanmaktır. Bu, ilk yanıttaki çerezlerin otomatik olarak ikinci isteğe dahil edilmesini sağlar.

# we are not calling curl_init()

# simply change the URL
curl_setopt($ch, CURLOPT_URL, "https://www.example.com/show_me_the_foo.php");

# change the method back to GET
curl_setopt($ch, CURLOPT_HTTPGET, true);

# send the request
$result = curl_exec($ch);

# finished with cURL
curl_close($ch);

# do stuff with $result...

Bu, yalnızca tanımlama bilgisi işlemenin bir örneği olarak tasarlanmıştır. Gerçek hayatta, işler genellikle daha karmaşıktır. Genellikle, POST’unuza eklenmesi gereken bir giriş belirtecini çekmek için giriş sayfasının ilk GET’ini yapmanız gerekir. Diğer siteler, User-Agent dizesine göre cURL istemcisini engelleyebilir ve değiştirmenizi gerektirebilir.

Çoklu POST istekleri yapmak için multi_curl kullanma #

Bazen bir veya daha fazla farklı uç noktaya çok sayıda POST isteği yapmamız gerekir. Bu senaryo ile başa çıkmak için multi_curl kullanabiliriz.

Öncelikle basit örnekteki gibi tam olarak gerektiği kadar istek oluşturup bir diziye koyuyoruz.

curl_multi_init’i kullanıyoruz ve her bir tutamacı ona ekliyoruz.

Bu örnekte 2 farklı uç nokta kullanıyoruz:

//array of data to POST
$request_contents = array();
//array of URLs
$urls = array();
//array of cURL handles
$chs = array();

//first POST content
$request_contents[] = [
    'a' => 'apple',
    'b' => 'banana'
];
//second POST content
$request_contents[] = [
    'a' => 'fish',
    'b' => 'shrimp'
];
//set the urls
$urls[] = 'http://www.example.com';
$urls[] = 'http://www.example2.com';

//create the array of cURL handles and add to a multi_curl
$mh = curl_multi_init();
foreach ($urls as $key => $url) {
    $chs[$key] = curl_init($url);
    curl_setopt($chs[$key], CURLOPT_RETURNTRANSFER, true);
    curl_setopt($chs[$key], CURLOPT_POST, true);
    curl_setopt($chs[$key], CURLOPT_POSTFIELDS, $request_contents[$key]);

    curl_multi_add_handle($mh, $chs[$key]);
}

Ardından istekleri göndermek için curl_multi_exec kullanırız.

//running the requests
$running = null;
do {
  curl_multi_exec($mh, $running);
} while ($running);

//getting the responses
foreach(array_keys($chs) as $key){
    $error = curl_error($chs[$key]);
    $last_effective_URL = curl_getinfo($chs[$key], CURLINFO_EFFECTIVE_URL);
    $time = curl_getinfo($chs[$key], CURLINFO_TOTAL_TIME);
    $response = curl_multi_getcontent($chs[$key]);  // get results
    if (!empty($error)) {
      echo "The request $key return a error: $error" . "\n";
    }
    else {
      echo "The request to '$last_effective_URL' returned '$response' in $time seconds." . "\n";
    }

    curl_multi_remove_handle($mh, $chs[$key]);
}

// close current handler
curl_multi_close($mh);

Bu örnek için olası bir geri dönüş şöyle olabilir:

‘http://www.example.com’ isteği 2 saniye içinde ‘meyveler’ döndürdü.

‘http://www.example2.com’ isteği 5 saniye içinde ‘deniz ürünleri’ döndürdü.

Özel bir yöntemle istek oluşturma ve gönderme #

Varsayılan olarak, PHP Curl “GET” ve “POST” isteklerini destekler. “CURLOPT_CUSTOMREQUEST” parametresini kullanarak “DELETE”, “PUT” veya “PATCH” (veya hatta standart olmayan yöntemler) gibi özel istekler göndermek de mümkündür.

$method = 'DELETE'; // Create a DELETE request

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, 1);
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, $method);
$content = curl_exec($ch);
curl_close($ch);

Tek bir istekte CurlFile ile çok boyutlu veri ve birden çok dosya gönderme #

Diyelim ki aşağıdaki gibi bir formumuz var. Verileri AJAX aracılığıyla web sunucumuza ve oradan da harici bir sunucuda çalışan bir komut dosyasına göndermek istiyoruz.

Yani normal girdilerimiz, çoklu seçim alanımız ve birden fazla dosya yükleyebileceğimiz bir dosya bırakma alanımız var.

AJAX POST isteğinin başarılı olduğunu varsayarsak PHP sitesinde aşağıdaki verileri alırız:

// print_r($_POST)

Array
(
    [first_name] => John
    [last_name] => Doe
    [activities] => Array
        (
            [0] => soccer
            [1] => hiking
        )
)

ve dosyalar şöyle görünmelidir

// print_r($_FILES)

Array
(
    [upload] => Array
        (
            [name] => Array
                (
                    [0] => my_photo.jpg
                    [1] => my_life.pdf
                )

            [type] => Array
                (
                    [0] => image/jpg
                    [1] => application/pdf
                )

            [tmp_name] => Array
                (
                    [0] => /tmp/phpW5spji
                    [1] => /tmp/phpWgnUeY
                )

            [error] => Array
                (
                    [0] => 0
                    [1] => 0
                )

            [size] => Array
                (
                    [0] => 647548
                    [1] => 643223
                )

Çok uzak çok iyi. Şimdi bu verileri ve dosyaları CurlFile Class ile cURL kullanarak harici sunucuya göndermek istiyoruz.

cURL yalnızca basit bir diziyi kabul ettiğinden, çok boyutlu bir diziyi kabul etmediğinden, önce $_POST dizisini düzleştirmemiz gerekir.

Bunu yapmak için, size aşağıdakileri veren örneğin bu işlevi kullanabilirsiniz:

// print_r($new_post_array)

Array
(
    [first_name] => John
    [last_name] => Doe
    [activities[0]] => soccer
    [activities[1]] => hiking
)

Sonraki adım, yüklenen dosyalar için CurlFile Nesneleri oluşturmaktır. Bu, aşağıdaki döngü tarafından yapılır:

$files = array();

foreach ($_FILES["upload"]["error"] as $key => $error) {
    if ($error == UPLOAD_ERR_OK) {

        $files["upload[$key]"] = curl_file_create(
            $_FILES['upload']['tmp_name'][$key],
            $_FILES['upload']['type'][$key],
            $_FILES['upload']['name'][$key]
        );
    }
}

curl_file_create, CurlFile Sınıfının bir yardımcı işlevidir ve CurlFile nesnelerini oluşturur. Her nesneyi iki dosyamız için “upload[0]” ve “upload[1]” adlı anahtarlarla $files dizisine kaydederiz.

Şimdi düzleştirilmiş yazı dizisini ve dosyalar dizisini birleştirmeli ve bunu şöyle $data olarak kaydetmeliyiz:

$data = $new_post_array + $files;

Son adım, cURL isteğini göndermektir:

$ch = curl_init();

curl_setopt_array($ch, array(
    CURLOPT_POST => 1,
    CURLOPT_URL => "https://api.externalserver.com/upload.php",
    CURLOPT_RETURNTRANSFER => 1,
    CURLINFO_HEADER_OUT => 1,
    CURLOPT_POSTFIELDS => $data
));

$result = curl_exec($ch);

curl_close ($ch);

$data artık basit (düz) bir dizi olduğundan, cURL bu POST isteğini İçerik Türü ile otomatik olarak gönderir: multipart/form-data

Harici sunucudaki upload.php’de, normalde yaptığınız gibi artık gönderi verilerini ve dosyaları $_POST ve $_FILES ile alabilirsiniz.
