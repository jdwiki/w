Bu uzantı, verileri doğrulayarak veya sterilize ederek filtreler. Bu, özellikle veri kaynağı, kullanıcı tarafından sağlanan girdi gibi bilinmeyen (veya yabancı) veriler içerdiğinde kullanışlıdır. Örneğin, bu veriler bir HTML formundan gelebilir.

Sözdizimi #

  • karışık filtre_var ( karışık $değişken [, int $filtre = FILTER_DEFAULT [, karışık $seçenekler ]] )

Parametreler #

parametre Ayrıntılar
değişken Filtrelenecek değer. Skaler değerlerin filtrelenmeden önce dahili olarak dizeye dönüştürüldüğünü unutmayın.
—— ——
filtre Uygulanacak filtrenin kimliği. Filtre türleri kılavuz sayfası, kullanılabilir filtreleri listeler. Atlanırsa, FILTER_UNSAFE_RAW’a eşdeğer olan FILTER_DEFAULT kullanılır. Bu, varsayılan olarak hiçbir filtreleme yapılmamasına neden olur.
—— ——
seçenekler İlişkili seçenekler dizisi veya bayrakların bit düzeyinde ayrılması. Eğer filtre seçenekleri kabul ederse, dizideki “flags” alanında bayraklar sağlanabilir. “Geri arama” filtresi için çağrılabilir tür geçirilmelidir. Geri arama, filtrelenecek değer olan bir argümanı kabul etmeli ve filtreden/temizlendikten sonra değeri döndürmelidir.

E-posta adresini doğrula #

Bir e-posta adresini filtrelerken, filter_var(), filtrelenmiş verileri, bu durumda e-posta adresini veya geçerli bir e-posta adresi bulunamazsa false döndürür:

var_dump(filter_var('john@example.com', FILTER_VALIDATE_EMAIL));
var_dump(filter_var('notValidEmail', FILTER_VALIDATE_EMAIL));

Sonuçlar:

string(16) "john@example.com"
bool(false)

Bu işlev latin olmayan karakterleri doğrulamaz. Uluslararasılaştırılmış alan adı, “xn–” biçiminde doğrulanabilir.

Bir e-posta göndermeden önce e-posta adresinin doğru olup olmadığını bilemeyeceğinizi unutmayın. MX kaydı kontrolü gibi bazı ekstra kontroller yapmak isteyebilirsiniz, ancak bu gerekli değildir. Bir onay e-postası gönderirseniz, kısa bir süre sonra kullanılmayan hesapları kaldırmayı unutmayın.

Bir Değeri Doğrulamak Bir Tam Sayıdır #

Bir tamsayı olması gereken bir değeri filtrelerken, “filter_var()” filtrelenmiş verileri, bu durumda tamsayıyı veya değer bir tamsayı değilse false değerini döndürür. Şamandıralar değil tam sayılardır:

var_dump(filter_var('10', FILTER_VALIDATE_INT));
var_dump(filter_var('a10', FILTER_VALIDATE_INT));
var_dump(filter_var('10a', FILTER_VALIDATE_INT));
var_dump(filter_var(' ', FILTER_VALIDATE_INT));
var_dump(filter_var('10.00', FILTER_VALIDATE_INT));
var_dump(filter_var('10,000', FILTER_VALIDATE_INT));
var_dump(filter_var('-5', FILTER_VALIDATE_INT));
var_dump(filter_var('+7', FILTER_VALIDATE_INT));

Sonuçlar:

int(10)
bool(false)
bool(false)
bool(false)
bool(false)
bool(false)
int(-5)
int(7)

Yalnızca rakam bekliyorsanız, normal bir ifade kullanabilirsiniz:

if(is_string($_GET['entry']) && preg_match('#^[0-9]+$#', $_GET['entry']))
    // this is a digit (positive) integer
else
    // entry is incorrect

Bu değeri bir tamsayıya çevirirseniz, bu kontrolü yapmanıza gerek kalmaz ve böylece filter_var kullanabilirsiniz.

Bir Aralığa Düşen Bir Tamsayı Doğrulama #

Bir tamsayının bir aralığa düştüğünü doğrularken kontrol, minimum ve maksimum sınırları içerir:

$options = array(
    'options' => array(
        'min_range' => 5,
        'max_range' => 10,
    )
);
var_dump(filter_var('5', FILTER_VALIDATE_INT, $options));
var_dump(filter_var('10', FILTER_VALIDATE_INT, $options));
var_dump(filter_var('8', FILTER_VALIDATE_INT, $options));
var_dump(filter_var('4', FILTER_VALIDATE_INT, $options));
var_dump(filter_var('11', FILTER_VALIDATE_INT, $options));
var_dump(filter_var('-6', FILTER_VALIDATE_INT, $options));

Sonuçlar:

int(5)
int(10)
int(8)
bool(false)
bool(false)
bool(false)

Bir URL’yi doğrulama #

Bir URL’yi filtrelerken “filter_var()”, filtrelenmiş verileri, bu durumda URL’yi veya geçerli bir URL bulunamazsa false döndürür:

URL: “example.com”

var_dump(filter_var('example.com', FILTER_VALIDATE_URL));
var_dump(filter_var('example.com', FILTER_VALIDATE_URL, FILTER_FLAG_SCHEME_REQUIRED));
var_dump(filter_var('example.com', FILTER_VALIDATE_URL, FILTER_FLAG_HOST_REQUIRED));
var_dump(filter_var('example.com', FILTER_VALIDATE_URL, FILTER_FLAG_PATH_REQUIRED));
var_dump(filter_var('example.com', FILTER_VALIDATE_URL, FILTER_FLAG_QUERY_REQUIRED));

Sonuçlar:

bool(false)
bool(false)
bool(false)
bool(false)
bool(false)

URL: “http://example.com”

var_dump(filter_var('http://example.com', FILTER_VALIDATE_URL));

var_dump(filter_var('http://example.com', FILTER_VALIDATE_URL, FILTER_FLAG_HOST_REQUIRED));

bool(false)

Filtreleri dezenfekte edin #

Değişkenimizi ihtiyacımıza göre sterilize etmek için filtreler kullanabiliriz.

Örnek

$string = "<p>Example</p>";
$newstring = filter_var($string, FILTER_SANITIZE_STRING);
var_dump($newstring); // string(7) "Example"

yukarıdaki html etiketlerini $string değişkeninden kaldıracaktır.

Boole Değerlerini Doğrulama #

var_dump(filter_var(true, FILTER_VALIDATE_BOOLEAN, FILTER_NULL_ON_FAILURE)); // true
var_dump(filter_var(false, FILTER_VALIDATE_BOOLEAN, FILTER_NULL_ON_FAILURE)); // false
var_dump(filter_var(1, FILTER_VALIDATE_BOOLEAN, FILTER_NULL_ON_FAILURE)); // true
var_dump(filter_var(0, FILTER_VALIDATE_BOOLEAN, FILTER_NULL_ON_FAILURE)); // false

var_dump(filter_var('0', FILTER_VALIDATE_BOOLEAN, FILTER_NULL_ON_FAILURE)); // false
var_dump(filter_var('', FILTER_VALIDATE_BOOLEAN, FILTER_NULL_ON_FAILURE)); // false
var_dump(filter_var(' ', FILTER_VALIDATE_BOOLEAN, FILTER_NULL_ON_FAILURE)); // false
var_dump(filter_var('true', FILTER_VALIDATE_BOOLEAN, FILTER_NULL_ON_FAILURE)); // true
