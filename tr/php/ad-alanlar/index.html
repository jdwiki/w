PHP belgelerinden:

Ad alanları nedir? En geniş tanımda ad alanları, öğeleri kapsüllemenin bir yoludur. Bu birçok yerde soyut bir kavram olarak görülebilir. Örneğin, herhangi bir işletim sisteminde dizinler, ilgili dosyaları gruplandırmaya hizmet eder ve içindeki dosyalar için bir ad alanı görevi görür. Somut bir örnek olarak, foo.txt dosyası hem /home/greg dizininde hem de /home/other dizininde bulunabilir, ancak foo.txt dosyasının iki kopyası aynı dizinde bir arada bulunamaz. Ek olarak, /home/greg dizini dışında foo.txt dosyasına erişmek için, /home/greg/foo.txt dosyasını almak için dizin ayırıcıyı kullanarak dizin adını dosya adının başına eklemeliyiz. Bu aynı ilke, programlama dünyasındaki ad alanlarına da uzanır.

Üst düzey ad alanlarının ‘PHP’ ve ‘php’ PHP dilinin kendisi için ayrıldığına dikkat edin. Herhangi bir özel kodda kullanılmamalıdırlar.

Ad alanlarını bildirme #

Bir ad alanı bildirimi aşağıdaki gibi görünebilir:

  • namespace MyProject; - MyProject ad alanını bildir
  • namespace MyProject\Security\Cryptography; - İç içe bir ad alanı bildir
  • namespace MyProject { ... } - Parantez içine alan bir ad alanı bildirin.

Tek bir dosyada istediğiniz kadar tanımlayabilmenize rağmen, dosya başına yalnızca tek bir ad alanı bildirmeniz önerilir:

namespace First {
    class A { ... }; // Define class A in the namespace First.
}

namespace Second {
    class B { ... }; // Define class B in the namespace Second.
}

namespace {
    class C { ... }; // Define class C in the root namespace.
}

Her ad alanı bildirdiğinizde, bundan sonra tanımladığınız sınıflar o ad alanına ait olacaktır:

namespace MyProject\Shapes;

class Rectangle { ... }
class Square { ... }
class Circle { ... }

Bir ad alanı bildirimi, farklı dosyalarda birden çok kez kullanılabilir. Yukarıdaki örnek, tek bir dosyada “MyProject\Shapes” ad alanındaki üç sınıfı tanımladı. Tercihen bu, her biri namespace MyProject\Shapes; ile başlayan üç dosyaya bölünür. Bu, PSR-4 standart örneğinde daha ayrıntılı olarak açıklanmıştır.

Ad alanında bir sınıfa veya işleve başvurma #

Ad Alanlarını Bildirme‘de gösterildiği gibi, bir ad alanında bir sınıfı aşağıdaki gibi tanımlayabiliriz:

namespace MyProject\Shapes;

class Rectangle { ... }

Bu sınıfa başvurmak için tam yolun (ad alanı dahil) kullanılması gerekir:

$rectangle = new MyProject\Shapes\Rectangle();

Bu, sınıfı “use” ifadesi aracılığıyla içe aktararak kısaltılabilir:

// Rectangle becomes an alias to MyProject\Shapes\Rectangle
use MyProject\Shapes\Rectangle;

$rectangle = new Rectangle();

PHP 7.0’a gelince, çeşitli ‘use’ ifadelerini parantez kullanarak tek bir ifadede gruplayabilirsiniz:

use MyProject\Shapes\{
    Rectangle,         //Same as `use MyProject\Shapes\Rectangle`
    Circle,            //Same as `use MyProject\Shapes\Circle`
    Triangle,          //Same as `use MyProject\Shapes\Triangle`
    
    Polygon\FiveSides, //You can also import sub-namespaces
    Polygon\SixSides   //In a grouped `use`-statement
};

$rectangle = new Rectangle();

Bazen iki sınıf aynı ada sahiptir. Farklı bir ad alanındalarsa bu bir sorun değildir, ancak bunları “use” ifadesi ile içe aktarmaya çalışırken sorun olabilir:

use MyProject\Shapes\Oval;
use MyProject\Languages\Oval; // Apparantly Oval is also a language!
// Error!

Bu, “as” anahtar sözcüğünü kullanarak takma ad için kendiniz bir ad tanımlayarak çözülebilir:

use MyProject\Shapes\Oval as OvalShape;
use MyProject\Languages\Oval as OvalLanguage;

Geçerli ad alanı dışındaki bir sınıfa başvuruda bulunmak için, bir `' ile kaçılması gerekir, aksi takdirde geçerli ad alanından göreli bir ad alanı yolu varsayılır:

namespace MyProject\Shapes;

// References MyProject\Shapes\Rectangle. Correct!
$a = new Rectangle();

// References MyProject\Shapes\Rectangle. Correct, but unneeded!
$a = new \MyProject\Shapes\Rectangle(); 

// References MyProject\Shapes\MyProject\Shapes\Rectangle. Incorrect!
$a = new MyProject\Shapes\Rectangle(); 


// Referencing StdClass from within a namespace requires a \ prefix
// since it is not defined in a namespace, meaning it is global.

// References StdClass. Correct!
$a = new \StdClass(); 

// References MyProject\Shapes\StdClass. Incorrect!
$a = new StdClass();

Alt ad alanları bildirme #

Hiyerarşi ile tek bir ad alanı bildirmek için aşağıdaki örneği kullanın:

namespace MyProject\Sub\Level;

const CONNECT_OK = 1;
class Connection { /* ... */ }
function connect() { /* ... */  }

Yukarıdaki örnek oluşturur:

sabit Projem\Alt\Seviye\BAĞLANTI_TAMAM

sınıf `Projem\Alt\Seviye\Bağlantı’ ve

işlev Projem\Alt\Seviye\bağlan

Ad Alanları nedir? #

PHP topluluğu, çok sayıda kod oluşturan birçok geliştiriciye sahiptir. Bu, bir kitaplığın PHP kodunun başka bir kitaplıkla aynı sınıf adını kullanabileceği anlamına gelir. Her iki kitaplık da aynı ad alanında kullanıldığında çarpışır ve sorun yaratır.

Ad alanları bu sorunu çözer. PHP başvuru kılavuzunda açıklandığı gibi, ad alanları, ad alanı dosyalarının bulunduğu işletim sistemi dizinleriyle karşılaştırılabilir; aynı ada sahip iki dosya ayrı dizinlerde bir arada bulunabilir. Aynı şekilde, aynı ada sahip iki PHP sınıfı, ayrı PHP ad alanlarında bir arada bulunabilir.

Diğer geliştiriciler tarafından diğer kitaplıklarla çarpışma korkusu olmadan kullanılabilmesi için kodunuzu adlandırmanız önemlidir.
