Değişkenleri boşaltma #

var_dump işlevi, hata ayıklama için bir değişkenin (tür ve değer) içeriğini boşaltmanıza olanak tanır.

Örnek:

$array = [3.7, "string", 10, ["hello" => "world"], false, new DateTime()];
var_dump($array);

Çıktı:

array(6) {
  [0]=>
  float(3.7)
  [1]=>
  string(6) "string"
  [2]=>
  int(10)
  [3]=>
  array(1) {
    ["hello"]=>
    string(5) "world"
  }
  [4]=>
  bool(false)
  [5]=>
  object(DateTime)#1 (3) {
    ["date"]=>
    string(26) "2016-07-24 13:51:07.000000"
    ["timezone_type"]=>
    int(3)
    ["timezone"]=>
    string(13) "Europe/Berlin"
  }
}

Hataları görüntüleme #

PHP’nin sayfada çalışma zamanı hatalarını görüntülemesini istiyorsanız, ya ‘php.ini’ içinde ya da ‘ini_set’ işlevini kullanarak ‘display_errors’‘i etkinleştirmeniz gerekir.

E_* sabitlerini kabul eden error_reporting (veya ini’deki) işleviyle bitsel operatörler kullanılarak hangi hataların görüntüleneceğini seçebilirsiniz.

PHP, html_errors ayarına bağlı olarak hataları metin veya HTML biçiminde görüntüleyebilir.

Örnek:

ini_set("display_errors", true);
ini_set("html_errors", false); // Display errors in plain text
error_reporting(E_ALL & ~E_USER_NOTICE); // Display everything except E_USER_NOTICE

trigger_error("Pointless error"); // E_USER_NOTICE
echo $nonexistentVariable; // E_NOTICE
nonexistentFunction(); // E_ERROR

Düz metin çıktısı: (HTML biçimi, uygulamalar arasında farklılık gösterir)

Notice: Undefined variable: nonexistentVariable in /path/to/file.php on line 7

Fatal error: Uncaught Error: Call to undefined function nonexistentFunction() in /path/to/file.php:8
Stack trace:
#0 {main}
  thrown in /path/to/file.php on line 8

NOT: php.ini’de hata raporlamayı devre dışı bırakırsanız ve çalışma zamanı sırasında etkinleştirirseniz, çalışma zamanı ayarı uygulanmadan önce meydana geldikleri için bazı hatalar (ayrıştırma hataları gibi) görüntülenmez.

’error_reporting’i işlemenin yaygın yolu, geliştirme sırasında ‘E_ALL’ sabitiyle tamamen etkinleştirmek ve komut dosyalarınızın içindekileri gizlemek için üretim aşamasında ‘display_errors’ ile genel olarak görüntülemeyi devre dışı bırakmaktır.

phpinfo() #

Uyarı #

phpinfonun yalnızca geliştirme ortamında kullanılması zorunludur. “phpinfo” içeren kodu asla bir üretim ortamına yayınlamayın

Giriiş #

Bununla birlikte, çalıştığınız PHP ortamını (işletim sistemi, yapılandırma, sürümler, yollar, modüller) anlamada, özellikle bir hatayı takip ederken yararlı bir araç olabilir. Basit bir yerleşik işlevdir:

phpinfo();

Çıktının özelleştirilmesine izin veren bir “$what” parametresine sahiptir. Varsayılan “INFO_ALL” olup, tüm bilgileri görüntülemesine neden olur ve geliştirme sırasında PHP’nin mevcut durumunu görmek için yaygın olarak kullanılır.

Özelleştirilmiş bir liste görmek için bitsel operatörlerle birlikte [INFO_* sabitleri2] parametresini iletebilirsiniz.

Güzel biçimlendirilmiş ayrıntılı bir görünüm için tarayıcıda çalıştırabilirsiniz. Ayrıca PHP CLI’de de çalışır, burada daha kolay görüntüleme için onu “daha az"a aktarabilirsiniz.

Örnek #

phpinfo(INFO_CONFIGURATION | INFO_ENVIRONMENT | INFO_VARIABLES);

Bu, PHP yönergelerinin (ini_get), ortam ($_ENV) ve ön tanımlı değişkenlerin bir listesini görüntüler.

Hata ayıklama #

Xdebug, hata ayıklama ve profil oluşturma yetenekleri sağlayan bir PHP uzantısıdır.
DBGp hata ayıklama protokolünü kullanır.

Bu araçta bazı güzel özellikler var:

  • hatalar üzerinde yığın izleri
  • maksimum yuvalama seviyesi koruması ve zaman takibi
  • değişkenleri görüntülemek için standart var_dump() işlevinin yararlı bir şekilde değiştirilmesi
  • parametreler ve dönüş değerleri de dahil olmak üzere tüm işlev çağrılarını farklı formatlarda bir dosyaya kaydetmeye izin verir
  • kod kapsamı analizi
  • profil bilgileri
  • uzaktan hata ayıklama (çalışan PHP komut dosyalarıyla etkileşime giren hata ayıklayıcı istemciler için arabirim sağlar)

Gördüğünüz gibi bu uzantı geliştirme ortamı için mükemmel bir şekilde uygundur. Özellikle uzaktan hata ayıklama özelliği, çok sayıda var_dump olmadan php kodunuzu hata ayıklamanıza ve C++ veya Java dillerinde olduğu gibi normal hata ayıklama sürecini kullanmanıza yardımcı olabilir.

Genellikle bu uzantının yüklenmesi çok basittir:

pecl install xdebug # install from pecl/pear

Ve onu php.ini’nizde etkinleştirin:

zend_extension="/usr/local/php/modules/xdebug.so"

Daha karmaşık durumlarda, bu [talimatlara2 bakın

Bu aracı kullanırken şunları hatırlamanız gerekir:
XDebug, üretim ortamları için uygun değildir

phpversiyon() #

Giriiş #

Çeşitli kitaplıklar ve bunlarla ilişkili gereksinimlerle çalışırken, genellikle mevcut PHP ayrıştırıcısının sürümünü veya paketlerinden birinin sürümünü bilmek gerekir.

Bu işlev, uzantı adı biçiminde tek bir isteğe bağlı parametreyi kabul eder: phpversion('extension'). Söz konusu uzantı kuruluysa, işlev, sürüm değerini içeren bir dize döndürür. Ancak, uzantı yüklenmemişse ‘YANLIŞ’ döndürülür. Uzantı adı verilmezse, işlev PHP ayrıştırıcısının sürümünü döndürür.

Örnek #

print "Current PHP version: " . phpversion();
// Current PHP version: 7.0.8

print "Current cURL version: " . phpversion( 'curl' );
// Current cURL version: 7.0.8
// or
// false, no printed output if package is missing

Hata Raporlama (ikisini de kullanın) #

// this sets the configuration option for your environment
ini_set('display_errors', '1');

//-1 will allow all errors to be reported
error_reporting(-1);
