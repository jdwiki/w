Sözdizimi #

  • __getFunctions() // Hizmet için işlev dizisini döndürür (yalnızca WSDL modu)
  • __getTypes() // Hizmet için tür dizisini döndürür (yalnızca WSDL modu)
  • __getLastRequest() // Son istekten XML döndürür (‘izleme’ seçeneği gerektirir)
  • __getLastRequestHeaders() // Son istekten başlıkları döndürür (‘izleme’ seçeneği gerektirir)
  • __getLastResponse() // Son yanıttan XML döndürür (‘izleme’ seçeneği gerektirir)
  • __getLastResponseHeaders() // Son yanıttan başlıkları döndürür (‘izleme’ seçeneği gerektirir)

Parametreler #

parametre Ayrıntılar
$wsdl WSDL URI’si veya WSDL olmayan mod kullanılıyorsa “NULL”
$seçenekler SoapClient için seçenekler dizisi. WSDL olmayan mod, ayarlamak için “konum” ve “uri” gerektirir, diğer tüm seçenekler isteğe bağlıdır. Olası değerler için aşağıdaki tabloya bakın.

‘SoapClient’ sınıfı bir ‘__call’ yöntemiyle donatılmıştır. Bu doğrudan çağrılacak *değildir. Bunun yerine şunları yapmanızı sağlar:

$soap->requestInfo(['a', 'b', 'c']);

Bu, “requestInfo” SOAP yöntemini çağıracaktır.

Olası “$options” değerleri tablosu (Anahtar/değer çiftleri dizisi):

Seçenek Ayrıntılar
konum SOAP sunucusunun URL’si. Gerekli WSDL olmayan modda. URL’yi geçersiz kılmak için WSDL modunda kullanılabilir.
uri SOAP hizmetinin hedef ad alanı. Gerekli WSDL olmayan modda.
stil Olası değerler “SOAP_RPC” veya “SOAP_DOCUMENT” şeklindedir. Yalnızca WSDL olmayan modda geçerlidir.
kullanım Olası değerler “SOAP_ENCODED” veya “SOAP_LITERAL"dir. Yalnızca WSDL olmayan modda geçerlidir.
sabun_versiyonu Olası değerler “SOAP_1_1” (varsayılan) veya “SOAP_1_2”‘dir.
kimlik doğrulama HTTP kimlik doğrulamasını etkinleştirin. Olası değerler “SOAP_AUTHENTICATION_BASIC” (varsayılan) veya “SOAP_AUTHENTICATION_DIGEST” şeklindedir.
giriş HTTP kimlik doğrulaması için kullanıcı adı
şifre HTTP kimlik doğrulaması için parola
proxy_host proxy sunucusunun URL’si
proxy_port Proxy sunucu bağlantı noktası
proxy_login Proxy için kullanıcı adı
proxy_parola Proxy için parola
local_cert HTTPS istemci sertifikasına giden yol (kimlik doğrulama için)
parola HTTPS istemci sertifikası için parola
sıkıştırma İsteği / yanıtı sıkıştır. Değer, “SOAP_COMPRESSION_GZIP” veya “SOAP_COMPRESSION_DEFLATE” içeren bir “SOAP_COMPRESSION_ACCEPT” bit maskesidir. Örneğin: SOAP_COMPRESSION_ACCEPT | SOAP_COMPRESSION_GZIP.
kodlama Dahili karakter kodlaması (TODO: olası değerler)
iz Boolean, varsayılan olarak “YANLIŞ"tır. Hataların geri izlenebilmesi için isteklerin izlenmesini sağlar. __getLastRequest(), __getLastRequestHeaders(), __getLastResponse() ve __getLastResponseHeaders() kullanımına izin verir.
sınıf haritası WSDL türlerini PHP sınıflarına eşleyin. Değer, anahtar olarak WSDL türlerini ve değer olarak PHP sınıf adlarını içeren bir dizi olmalıdır.
istisnalar Boole değeri. SOAP hataları istisnaları olmalıdır (`SoapFault türünden).
bağlantı_zaman aşımı SOAP hizmetine bağlantı için zaman aşımı (saniye cinsinden).
tip haritası Tür eşlemeleri dizisi. Dizi, şu anahtarlarla birlikte anahtar/değer çiftleri olmalıdır: ’type_name’, ’type_ns’ (ad alanı URI’si), ‘from_xml’ (bir dize parametresini kabul eden geri arama) ve ’to_xml’ (bir nesne parametresini kabul eden geri arama).
cache_wsdl WSDL dosyası nasıl (eğer varsa) önbelleğe alınmalıdır. Olası değerler “WSDL_CACHE_NONE”, “WSDL_CACHE_DISK”, “WSDL_CACHE_MEMORY” veya “WSDL_CACHE_BOTH” şeklindedir.
user_agent “User-Agent” başlığında kullanılacak dize.
akış_bağlam Bir bağlam için bir kaynak.
özellikler “SOAP_SINGLE_ELEMENT_ARRAYS”, “SOAP_USE_XSI_ARRAY_TYPE”, “SOAP_WAIT_ONE_WAY_CALLS” bit maskesi.
keep_alive (PHP sürümü >= 5.4 yalnızca) Boolean değeri. ‘Connection: Keep-Alive’ başlığını (‘TRUE’) veya ‘Connection: Close’ başlığını (‘FALSE’) gönderin.
ssl_yöntemi (PHP sürümü >= 5.5 yalnızca) Hangi SSL/TLS sürümünün kullanılacağı. Olası değerler “SOAP_SSL_METHOD_TLS”, “SOAP_SSL_METHOD_SSLv2”, “SOAP_SSL_METHOD_SSLv3” veya “SOAP_SSL_METHOD_SSLv23” şeklindedir.

32 bit PHP ile ilgili sorun: 32 bit PHP’de, otomatik olarak “xs:long” ile tamsayıya dönüştürülen 32 bitten büyük sayısal dizeler, onun 32 bit sınırına ulaşmasına neden olur, döküm ‘2147483647’ olarak ayarlayın. Bu soruna geçici bir çözüm bulmak için, dizeleri __soapCall() öğesine geçirmeden önce yüzecek şekilde çevirin.

WSDL Modu #

İlk olarak, URL’yi WSDL dosyasına ve isteğe bağlı olarak bir dizi seçenek ileterek yeni bir ‘SoapClient’ nesnesi oluşturun.

// Create a new client object using a WSDL URL
$soap = new SoapClient('https://example.com/soap.wsdl', [
    # This array and its values are optional
    'soap_version' => SOAP_1_2,
    'compression' => SOAP_COMPRESSION_ACCEPT | SOAP_COMPRESSION_GZIP,
    'cache_wsdl' => WSDL_CACHE_BOTH,
    # Helps with debugging
    'trace' => TRUE,
    'exceptions' => TRUE
]);

Ardından, SOAP yöntemlerinizi çağırmak için $soap nesnesini kullanın.

$result = $soap->requestData(['a', 'b', 'c']);

WSDL Olmayan Mod #

Bu, WSDL moduna benzer, ancak WSDL dosyası olarak ‘NULL’ geçirmemiz ve ’location’ ve ‘uri’ seçeneklerini ayarladığınızdan emin olmamız dışında.

$soap = new SoapClient(NULL, [
    'location' => 'https://example.com/soap/endpoint',
    'uri' => 'namespace'
]);

Sınıf Haritaları #

PHP’de bir SOAP İstemcisi oluştururken, yapılandırma dizisinde bir “sınıf haritası” anahtarı da ayarlayabilirsiniz. Bu ‘sınıf haritası’, varsayılan ‘StdClass’ yerine WSDL’de tanımlanan hangi tiplerin gerçek sınıflarla eşleştirilmesi gerektiğini tanımlar. Bunu yapmak istemenizin nedeni, normal `StdClass’ta hangi alanların ayarlandığını tahmin etmek yerine, bu sınıflarda alanların ve yöntem çağrılarının otomatik olarak tamamlanmasını alabilmenizdir.

class MyAddress {
    public $country;
    public $city;
    public $full_name;
    public $postal_code; // or zip_code
    public $house_number;
}

class MyBook {
    public $name;
    public $author;

    // The classmap also allows us to add useful functions to the objects
    // that are returned from the SOAP operations.
    public function getShortDescription() {
        return "{$this->name}, written by {$this->author}";
    }
}

$soap_client = new SoapClient($link_to_wsdl, [
    // Other parameters
    "classmap" => [
        "Address" => MyAddress::class, // ::class simple returns class as string
        "Book" => MyBook::class,
    ]
]);

Sınıf haritasını yapılandırdıktan sonra, “Adres” veya “Kitap” tipi döndüren belirli bir işlem gerçekleştirdiğinizde, SoapClient o sınıfı başlatır, alanları verilerle doldurur ve işlem çağrısından döndürür.

// Lets assume 'getAddress(1234)' returns an Address by ID in the database
$address = $soap_client->getAddress(1234);

// $address is now of type MyAddress due to the classmap
echo $address->country;

// Lets assume the same for 'getBook(1234)'
$book = $soap_client->getBook(124);

// We can not use other functions defined on the MyBook class
echo $book->getShortDescription();

// Any type defined in the WSDL that is not defined in the classmap
// will become a regular StdClass object
$author = $soap_client->getAuthor(1234);

// No classmap for Author type, $author is regular StdClass.
// We can still access fields, but no auto-completion and no custom functions
// to define for the objects.
echo $author->name;

SOAP isteğini ve yanıtını izleme #

Bazen SOAP isteğinde neyin gönderilip alındığına bakmak isteriz. Aşağıdaki yöntemler, istek ve yanıtta XML’i döndürür:

SoapClient::__getLastRequest()
SoapClient::__getLastRequestHeaders()
SoapClient::__getLastResponse()
SoapClient::__getLastResponseHeaders()

Örneğin, bir “ENVIRONMENT” sabitimiz olduğunu ve bu sabitin değeri “DEVELOPMENT” olarak ayarlandığında, “getAddress” çağrısı bir hata verdiğinde tüm bilgileri yankılamak istediğimizi varsayalım. Bir çözüm olabilir:

try {
    $address = $soap_client->getAddress(1234);
} catch (SoapFault $e) {
    if (ENVIRONMENT === 'DEVELOPMENT') {
        var_dump(
            $soap_client->__getLastRequestHeaders()
            $soap_client->__getLastRequest(),
            $soap_client->__getLastResponseHeaders(),
            $soap_client->__getLastResponse()
        );
    }
    ...
}
