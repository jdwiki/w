SQLSRV sürücüsü, Microsoft SQL Server ve SQL Azure veritabanlarına erişmenizi sağlayan Microsoft tarafından desteklenen bir PHP uzantısıdır. PHP 5.3’ten itibaren kullanımdan kaldırılan ve PHP 7’den kaldırılan MSSQL sürücüleri için bir alternatiftir.

SQLSRV uzantısı aşağıdaki işletim sistemlerinde kullanılabilir:

  • Windows Vista Service Pack 2 veya üzeri
  • Windows Server 2008 Service Pack 2 veya üzeri -Windows Server 2008 R2
  • Windows 7

SQLSRV uzantısı, Microsoft SQL Server 2012 Native Client’ın PHP çalıştıran aynı bilgisayara yüklenmesini gerektirir. Microsoft SQL Server 2012 Yerel İstemcisi zaten kurulu değilse, “Gereksinimler” dokümantasyon sayfasındaki uygun bağlantıya tıklayın.

En son SQLSRV sürücülerini indirmek için aşağıdakilere gidin: İndir

SQLSRV Sürücüleri için sistem gereksinimlerinin tam listesi burada bulunabilir: Sistem Gereksinimleri

SQLSRV 3.1+ kullananlar SQL Server için Microsoft ODBC Sürücüsü 11‘ü indirmelidir.

PHP7 kullanıcıları en son sürücüleri GitHub adresinden indirebilir.

SQL Server için Microsoft® ODBC Sürücüsü 13 Microsoft SQL Server 2008, SQL Server 2008 R2, SQL Server 2012’yi destekler, SQL Server 2014, SQL Server 2016 (Önizleme), Analitik Platform Sistemi, Azure SQL Veritabanı ve Azure SQL Veri Ambarı.

Sorgu Sonuçlarını Alma #

Bir sorgudan sonuç almanın 3 ana yolu vardır:

sqlsrv_fetch_array() #

sqlsrv_fetch_array() sonraki satırı bir dizi olarak alır.

$stmt = sqlsrv_query($conn, $query);

while($row = sqlsrv_fetch_array($stmt)) {
    echo $row[0];
    $var = $row["name"];
    //...
}

sqlsrv_fetch_array() farklı dizi türlerini geri getirmek için isteğe bağlı ikinci bir parametreye sahiptir: SQLSRV_FETCH_ASSOC, SQLSRV_FETCH_NUMERIC ve SQLSRV_FETCH_BOTH(varsayılan) kullanılabilir; her biri sırasıyla ilişkisel, sayısal veya ilişkisel ve sayısal dizileri döndürür.

sqlsrv_fetch_object() #

sqlsrv_fetch_object() sonraki satırı bir nesne olarak alır.

$stmt = sqlsrv_query($conn, $query);

while($obj = sqlsrv_fetch_object($stmt)) {
    echo $obj->field; // Object property names are the names of the fields from the query
    //...
}

sqlsrv_fetch() #

sqlsrv_fetch() sonraki satırı okunabilir hale getirir.

$stmt = sqlsrv_query($conn, $query);

while(sqlsrv_fetch($stmt) === true) {
    $foo = sqlsrv_get_field($stmt, 0); //gets the first field -
}

Hata Mesajlarını Alma #

Bir sorgu yanlış gittiğinde, sorunun nedenini belirlemek için sürücü tarafından döndürülen hata mesaj(lar)ını getirmek önemlidir. Sözdizimi:

sqlsrv_errors([int $errorsOrWarnings]);

Bu, aşağıdakileri içeren bir dizi döndürür:

Anahtar Açıklama
SQLSTATE SQL Server / OBDC Sürücüsünün bulunduğu durum
kod SQL Server hata kodu
mesaj Hatanın açıklaması

Yukarıdaki işlevi şu şekilde kullanmak yaygındır:

$brokenQuery = "SELECT BadColumnName FROM Table_1";
$stmt = sqlsrv_query($conn, $brokenQuery);

if ($stmt === false) {
    if (($errors = sqlsrv_errors()) != null) {
        foreach ($errors as $error) {
            echo "SQLSTATE: ".$error['SQLSTATE']."<br />";
            echo "code: ".$error['code']."<br />";
            echo "message: ".$error['message']."<br />";
        }
    }
}

Bağlantı Oluşturma #

$dbServer = "localhost,1234"; //Name of the server/instance, including optional port number (default is 1433)
$dbName = "db001"; //Name of the database
$dbUser = "user"; //Name of the user
$dbPassword = "password"; //DB Password of that user

$connectionInfo = array(
    "Database" => $dbName, 
    "UID" => $dbUser,
    "PWD" => $dbPassword
);

$conn = sqlsrv_connect($dbServer, $connectionInfo);

SQLSRV ayrıca bir PDO Sürücüsüne sahiptir. PDO kullanarak bağlanmak için:

$conn = new PDO("sqlsrv:Server=localhost,1234;Database=db001", $dbUser, $dbPassword);

Basit Bir Sorgu Yapma #

//Create Connection
$conn = sqlsrv_connect($dbServer, $connectionInfo);

$query = "SELECT * FROM [table]"; 
$stmt = sqlsrv_query($conn, $query);

Not: köşeli parantez [] * kullanımı, * [ayrılmış bir kelime] olduğu için * tablo' kelimesinden kaçmaktır (https://msdn.microsoft.com/en-us/library/ms189822 .aspx). Bunlar, MySQL*'deki backticks* `` ` *do ile aynı şekilde çalışır.

Saklı Prosedürü Çağırma #

Sunucuda saklı bir yordamı çağırmak için:

$query = "{call [dbo].[myStoredProcedure](?,?,?)}"; //Parameters '?' includes OUT parameters

$params = array(
    array($name, SQLSRV_PARAM_IN),
    array($age, SQLSRV_PARAM_IN),
    array($count, SQLSRV_PARAM_OUT, SQLSRV_PHPTYPE_INT) //$count must already be initialised
);

$result = sqlsrv_query($conn, $query, $params);

Parametreli Sorgu Oluşturma #

$conn = sqlsrv_connect($dbServer, $connectionInfo);

$query = "SELECT * FROM [users] WHERE [name] = ? AND [password] = ?";
$params = array("joebloggs", "pa55w0rd");

$stmt = sqlsrv_query($conn, $query, $params);

Aynı sorgu ifadesini farklı parametrelerle bir kereden fazla kullanmayı planlıyorsanız, aşağıda gösterildiği gibi sqlsrv_prepare() ve sqlsrv_execute() işlevleriyle aynısı yapılabilir:

$cart = array(
    "apple" => 3,
    "banana" => 1,
    "chocolate" => 2
);

$query = "INSERT INTO [order_items]([item], [quantity]) VALUES(?,?)";
$params = array(&$item, &$qty); //Variables as parameters must be passed by reference

$stmt = sqlsrv_prepare($conn, $query, $params);

foreach($cart as $item => $qty){
    if(sqlsrv_execute($stmt) === FALSE) {
        die(print_r(sqlsrv_errors(), true));
    }
}
