Sınıflar ve Nesneler, benzer görevleri gruplayarak kodunuzu daha verimli ve daha az tekrarlı hale getirmek için kullanılır.

Nesneleri oluşturmak için kullanılan eylemleri ve veri yapısını tanımlamak için bir sınıf kullanılır. Nesneler daha sonra bu önceden tanımlanmış yapı kullanılarak oluşturulur.

Sözdizimi #

  • class <ClassName> [ <ParentClassName> öğesini genişletir ] [ <Interface1> [, <Interface2>, ... ] { } uygular // Sınıf bildirimi
  • arayüz <ArayüzAdı> [ <ParentInterface1>'i genişletir [, <ParentInterface2>, ...] ] { } // Arayüz bildirimi
  • `<Özellik1> [, <Özellik2>, …]’ kullanın; // özellikleri kullan
  • [ genel | korumalı | özel ] [ statik ] $<varName>; // Nitelik bildirimi
  • const <CONST_NAME>; // Sabit bildirim
  • [ genel | korumalı | özel ] [ statik ] işlev <yöntemAdı>([args...]) { } // Yöntem bildirimi

Sınıflar ve Arayüz bileşenleri #

Sınıfların özellikleri, sabitleri ve yöntemleri olabilir.

  • Özellikler, nesne kapsamındaki değişkenleri tutar. Bildirimde başlatılabilirler, ancak yalnızca ilkel bir değer içeriyorlarsa.

  • Sabitler, bildirimde başlatılmalıdır ve yalnızca ilkel bir değer içerebilir. Sabit değerler derleme zamanında sabitlenir ve çalışma zamanında atanamayabilir.

  • Yöntemler, yöntem soyut olarak bildirilmediği sürece, bir gövdeye, hatta boş bir gövdeye sahip olmalıdır.

    class Foo { private $foo = ‘foo’; // OK private $baz = array(); // OK private $bar = new Bar(); // Error! }

Arayüzlerin özellikleri olamaz, ancak sabitleri ve yöntemleri olabilir.

  • Arabirim sabitleri, bildirimde başlatılmalıdır ve yalnızca ilkel bir değer içerebilir. Sabit değerler derleme zamanında sabitlenir ve çalışma zamanında atanamayabilir.

  • Arayüz yöntemlerinin gövdesi yoktur.

    interface FooBar { const FOO_VALUE = ‘bla’; public function doAnything(); }

Sınıf Sabitleri #

Sınıf sabitleri, bir programda sabit değerleri tutmak için bir mekanizma sağlar. Yani, “3.14” veya “Apple” gibi bir değere bir ad (ve ilişkili derleme zamanı denetimi) vermenin bir yolunu sağlarlar. Sınıf sabitleri yalnızca “const” anahtar sözcüğü ile tanımlanabilir - define işlevi bu bağlamda kullanılamaz.

Örnek olarak, bir program boyunca π değeri için bir kestirme temsile sahip olmak uygun olabilir. ‘const’ değerlerine sahip bir sınıf, bu tür değerleri tutmanın basit bir yolunu sağlar.

class MathValues {
    const PI = M_PI;
    const PHI = 1.61803;
}

$area = MathValues::PI * $radius * $radius;

Sınıf sabitlerine, statik değişkenlere çok benzer şekilde, bir sınıf üzerinde çift kolon operatörü (kapsam çözümleme operatörü olarak adlandırılır) kullanılarak erişilebilir. Bununla birlikte, statik değişkenlerin aksine, sınıf sabitlerinin değerleri derleme zamanında sabitlenir ve yeniden atanamazlar (örneğin, “MathValues::PI = 7” önemli bir hata üretir).

Sınıf sabitleri, daha sonra değiştirilmesi gerekebilecek bir sınıfın içindeki şeyleri tanımlamak için de yararlıdır (ancak, örneğin bir veritabanında depolamayı garanti edecek kadar sık ​​değişmez). Buna “self” kapsam çözümleyicisini (hem örneklenmiş hem de statik uygulamalarda çalışan) kullanarak dahili olarak başvurabiliriz.

class Labor {
    /** How long, in hours, does it take to build the item? */
    const LABOR_UNITS = 0.26;
    /** How much are we paying employees per hour? */
    const LABOR_COST = 12.75;

    public function getLaborCost($number_units) {
         return (self::LABOR_UNITS * self::LABOR_COST) * $number_units;
    }
}

Sınıf sabitleri yalnızca < 5.6 sürümlerinde skaler değerler içerebilir

PHP 5.6’dan itibaren, sabitleri olan ifadeler kullanabiliriz, yani matematik ifadeleri ve bitiştirilmiş dizeler kabul edilebilir sabitlerdir.

class Labor {
    /** How much are we paying employees per hour? Hourly wages * hours taken to make */
    const LABOR_COSTS = 12.75 * 0.26;

    public function getLaborCost($number_units) {
         return self::LABOR_COSTS * $number_units;
    }
}

PHP 7.0’dan itibaren, ‘define’ ile bildirilen sabitler artık diziler içerebilir.

define("BAZ", array('baz'));

Sınıf sabitleri, yalnızca matematiksel kavramları depolamaktan daha fazlası için kullanışlıdır. Örneğin, bir pasta hazırlarken, farklı meyve çeşitlerini alabilen tek bir ‘Pie’ sınıfının olması uygun olabilir.

class Pie {
    protected $fruit;

    public function __construct($fruit) {
        $this->fruit = $fruit;
    }
}

Daha sonra ‘Pie’ sınıfını şu şekilde kullanabiliriz.

$pie = new Pie("strawberry");

Burada ortaya çıkan sorun, ‘Pie’ sınıfını somutlaştırırken, kabul edilebilir değerlere ilişkin hiçbir rehberlik sağlanmamasıdır. Örneğin, “boisenberry” pastası yaparken “boisenberry” yanlış yazılmış olabilir. Ayrıca, bir erik turtasını desteklemeyebiliriz. Bunun yerine, bir yerde tanımlanmış kabul edilebilir meyve türlerinin bir listesinin olması, onları aramak mantıklı olacaktır. ‘Meyve’ adında bir sınıf söyleyin:

class Fruit {
    const APPLE = "apple";
    const STRAWBERRY = "strawberry";
    const BOYSENBERRY = "boysenberry";
}

$pie = new Pie(Fruit::STRAWBERRY);

Kabul edilebilir değerleri sınıf sabitleri olarak listelemek, bir yöntemin kabul ettiği kabul edilebilir değerler konusunda değerli bir ipucu sağlar. Ayrıca yazım hatalarının derleyiciyi geçememesini de sağlar. ’new Pie(‘aple’)’ ve ’new Pie(‘apple’)’ her ikisi de derleyici için kabul edilebilir olsa da, ’new Pie(Fruit::APLE)’ bir derleyici hatası üretecektir.

Son olarak, sınıf sabitlerinin kullanılması, sabitin gerçek değerinin tek bir yerde değiştirilebileceği ve sabiti kullanan herhangi bir kodun otomatik olarak değişikliğin etkilerine sahip olduğu anlamına gelir.

Bir sınıf sabitine erişmenin en yaygın yöntemi ‘Sınıfım::CONSTANT_NAME’ olsa da, buna şu şekilde de erişilebilir:

echo MyClass::CONSTANT;

$classname = "MyClass";
echo $classname::CONSTANT; // As of PHP 5.3.0

PHP’deki sınıf sabitleri, geleneksel olarak tümü büyük harflerle, alt çizgiler sözcük ayırıcılar olarak adlandırılır, ancak geçerli herhangi bir etiket adı bir sınıf sabiti adı olarak kullanılabilir.

PHP 7.1’den itibaren, sınıf sabitleri artık varsayılan genel kapsamdan farklı görünürlüklerle tanımlanabilir. Bu, sınıf sabitlerinin gereksiz yere genel kapsama sızmasını önlemek için artık hem korumalı hem de özel sabitlerin tanımlanabileceği anlamına gelir (bkz. https://www.wikiod.com/tr/php/snflar-ve-nesneler#Yöntem ve Özellik Görünürlüğü -özellik-görünürlük). Örneğin:

class Something {
    const PUBLIC_CONST_A = 1;
    public const PUBLIC_CONST_B = 2;
    protected const PROTECTED_CONST = 3;
    private const PRIVATE_CONST = 4;
}

sınıf sabitlerine karşı tanımla #

Bu geçerli bir yapı olmasına rağmen:

function bar() { return 2; };

define('BAR', bar());

Aynısını sınıf sabitleriyle yapmaya çalışırsanız, bir hata alırsınız:

function bar() { return 2; };

class Foo {
    const BAR = bar(); // Error: Constant expression contains invalid operations
}

Ama şunları yapabilirsiniz:

function bar() { return 2; };

define('BAR', bar());

class Foo {
    const BAR = BAR; // OK
}

Daha fazla bilgi için bkz. kılavuzdaki sabitler.

Sınıfın adını almak için ::class kullanma #

PHP 5.5, ad alanı kapsamını ve ‘use’ ifadelerini hesaba katarak tam sınıf adını almak için `::class’ sözdizimini tanıttı.

namespace foo;
use bar\Bar;
echo json_encode(Bar::class); // "bar\\Bar"
echo json_encode(Foo::class); // "foo\\Foo"
echo json_encode(\Foo::class); // "Foo"

Yukarıdakiler, sınıflar tanımlanmasa bile çalışır (yani bu kod parçacığı tek başına çalışır).

Bu sözdizimi, bir sınıf adı gerektiren işlevler için kullanışlıdır. Örneğin, bir sınıfın var olup olmadığını kontrol etmek için ‘class_exists’ ile birlikte kullanılabilir. Bu snippet’te dönüş değerinden bağımsız olarak hiçbir hata oluşturulmaz:

class_exists(ThisClass\Will\NeverBe\Loaded::class, false);

Soyut Sınıflar #

Soyut bir sınıf, somutlaştırılamayan bir sınıftır. Soyut sınıflar, herhangi bir gövdesi olmayan yöntemler olan soyut yöntemleri tanımlayabilir, yalnızca bir tanım:

abstract class MyAbstractClass {
    abstract public function doSomething($a, $b);
}

Soyut sınıflar, daha sonra bu soyut yöntemlerin uygulanmasını sağlayabilecek bir alt sınıf tarafından genişletilmelidir.

Bunun gibi bir sınıfın temel amacı, alt sınıfların bir yapıyı bağlı kalmaya “zorlayarak” miras almasına izin veren bir tür şablon sağlamaktır. Bunu bir örnekle detaylandıralım:

Bu örnekte bir ‘İşçi’ arayüzü uygulayacağız. İlk önce arayüzü tanımlıyoruz:

interface Worker {
    public function run();
}

Daha fazla Worker uygulamasının geliştirilmesini kolaylaştırmak için soyut bir işçi sınıfı oluşturacağız. bu, arayüzden “run()” yöntemini zaten sağlar, ancak herhangi bir alt sınıf tarafından doldurulması gereken bazı soyut yöntemleri belirtir:

abstract class AbstractWorker implements Worker {
    protected $pdo;
    protected $logger;

    public function __construct(PDO $pdo, Logger $logger) {
        $this->pdo = $pdo;
        $this->logger = $logger;
    }

    public function run() {
        try {
            $this->setMemoryLimit($this->getMemoryLimit());
            $this->logger->log("Preparing main");
            $this->prepareMain();
            $this->logger->log("Executing main");
            $this->main();
        } catch (Throwable $e) {
            // Catch and rethrow all errors so they can be logged by the worker
            $this->logger->log("Worker failed with exception: {$e->getMessage()}");
            throw $e;
        }
    }

    private function setMemoryLimit($memoryLimit) {
        ini_set('memory_limit', $memoryLimit);
        $this->logger->log("Set memory limit to $memoryLimit");
    }

    abstract protected function getMemoryLimit();

    abstract protected function prepareMain();

    abstract protected function main();
}

Her şeyden önce, getMemoryLimit() soyut bir yöntem sağladık. ‘AbstractWorker’dan uzanan herhangi bir sınıfın bu yöntemi sağlaması ve bellek limitini döndürmesi gerekir. AbstractWorker daha sonra hafıza limitini ayarlar ve günlüğe kaydeder.

İkinci olarak, “AbstractWorker”, çağrıldıklarını günlüğe kaydettikten sonra “prepareMain()” ve “main()” yöntemlerini çağırır.

Son olarak, tüm bu yöntem çağrıları bir “try” - “catch” bloğunda gruplandırılmıştır. Bu nedenle, alt sınıf tarafından tanımlanan soyut yöntemlerden herhangi biri bir istisna atarsa, bu istisnayı yakalar, günlüğe kaydeder ve yeniden atarız. Bu, tüm alt sınıfların bunu kendileri uygulamak zorunda kalmasını önler.

Şimdi AbstractWorkerdan genişleyen bir alt sınıf tanımlayalım:

class TranscactionProcessorWorker extends AbstractWorker {
    private $transactions;

    protected function getMemoryLimit() {
        return "512M";
    }

    protected function prepareMain() {
        $stmt = $this->pdo->query("SELECT * FROM transactions WHERE processed = 0 LIMIT 500");
        $stmt->execute();
        $this->transactions = $stmt->fetchAll();
    }

    protected function main() {
        foreach ($this->transactions as $transaction) {
            // Could throw some PDO or MYSQL exception, but that is handled by the AbstractWorker
            $stmt = $this->pdo->query("UPDATE transactions SET processed = 1 WHERE id = {$transaction['id']} LIMIT 1");
            $stmt->execute();
        }
    }
}

Gördüğünüz gibi, TransactionProcessorWorker uygulaması oldukça kolaydı, çünkü sadece bellek sınırını belirtmemiz ve gerçekleştirmesi gereken gerçek eylemler hakkında endişelenmemiz yeterliydi. “TransactionProcessorWorker"da hata işlemeye gerek yoktur çünkü bu “AbsractWorker"da işlenir.

Soyut bir sınıftan miras alınırken, tüm yöntemler soyut olarak işaretlenir. ebeveynin sınıf bildirimi çocuk (veya çocuğun kendisi) tarafından tanımlanmalıdır ayrıca özet olarak işaretlenmelidir); ayrıca, bu yöntemler aynı (veya daha az) ile tanımlanmalıdır. kısıtlı) görünürlük. Örneğin, soyut yöntem tanımlanmışsa korumalı olarak, işlev uygulaması şu şekilde tanımlanmalıdır: korumalı veya genel, ancak özel değil.

Geç statik bağlama #

PHP 5.3+ ve üzeri sürümlerde, bir statik özelliğin veya yöntemin hangi sınıftan çağrılacağını kontrol etmek için geç statik bağlama kullanabilirsiniz. ‘self::’ kapsam çözümleyicisinin doğasında bulunan sorunun üstesinden gelmek için eklendi. Aşağıdaki kodu alın

class Horse {
    public static function whatToSay() {
         echo 'Neigh!';
    }

    public static function speak() {
         self::whatToSay();
    }
}

class MrEd extends Horse {
    public static function whatToSay() {
         echo 'Hello Wilbur!';
    }
}

‘MrEd’ sınıfının, ‘whatToSay()’ üst işlevini geçersiz kılmasını beklersiniz. Ama bunu çalıştırdığımızda beklenmedik bir şey elde ederiz.

Horse::speak(); // Neigh!
MrEd::speak(); // Neigh!

Sorun şu ki, “self::whatToSay();” yalnızca “Horse” sınıfına başvurabilir, yani “MrEd"e uymaz. static:: kapsam çözümleyicisine geçersek, bu sorunu yaşamıyoruz. Bu yeni yöntem, sınıfa kendisini çağıran örneğe uymasını söyler. Böylece beklediğimiz mirası alıyoruz

class Horse {
    public static function whatToSay() {
         echo 'Neigh!';
    }

    public static function speak() {
         static::whatToSay(); // Late Static Binding
    }
}

Horse::speak(); // Neigh!
MrEd::speak(); // Hello Wilbur!

Ad Alanı ve Otomatik Yükleme #

Teknik olarak, otomatik yükleme, bir PHP sınıfı gerekli olduğunda ancak bulunamadığında bir geri arama yürüterek çalışır. Bu tür geri aramalar genellikle bu sınıfları yüklemeye çalışır.

Genel olarak, otomatik yükleme, bir sınıf gerektiğinde, PHP dosyalarını (özellikle bir PHP kaynak dosyasının belirli bir sınıfa tahsis edildiği PHP sınıf dosyalarını) sınıfın tam nitelikli adına (FQN) göre uygun yollardan yükleme girişimi olarak anlaşılabilir. .

Diyelim ki bu sınıflarımız var:

“application\controllers\Base” için sınıf dosyası:

<?php
namespace application\controllers { class Base {...} }

“application\controllers\Control” için sınıf dosyası:

<?php
namespace application\controllers { class Control {...} }

“application\models\Page” için sınıf dosyası:

<?php
namespace application\models { class Page {...} }

Kaynak klasör altında, bu sınıflar sırasıyla FQN’leri olarak yollara yerleştirilmelidir:

  • Kaynak dosya
  • ‘uygulamalar’
    • controllers
      • Base.php
      • Control.php
    • models
      • Page.php

Bu yaklaşım, bu işlevi kullanarak sınıf dosyası yolunu FQN’ye göre programlı olarak çözmeyi mümkün kılar:

function getClassPath(string $sourceFolder, string $className, string $extension = ".php") {
    return $sourceFolder . "/" . str_replace("\\", "/", $className) . $extension; // note that "/" works as a directory separator even on Windows
}

“spl_autoload_register” işlevi, gerektiğinde kullanıcı tanımlı bir işlev kullanarak bir sınıf yüklememize izin verir:

const SOURCE_FOLDER = __DIR__ . "/src";
spl_autoload_register(function (string $className) {
    $file = getClassPath(SOURCE_FOLDER, $className);
    if (is_readable($file)) require_once $file;
});

Bu işlev, geri yükleme yöntemlerini kullanmak için daha da genişletilebilir:

const SOURCE_FOLDERS = [__DIR__ . "/src", "/root/src"]);
spl_autoload_register(function (string $className) {
    foreach(SOURCE_FOLDERS as $folder) {
        $extensions = [
            // do we have src/Foo/Bar.php5_int64?
            ".php" . PHP_MAJOR_VERSION . "_int" . (PHP_INT_SIZE * 8),
            // do we have src/Foo/Bar.php7?
            ".php" . PHP_MAJOR_VERSION,
            // do we have src/Foo/Bar.php_int64?
            ".php" . "_int" . (PHP_INT_SIZE * 8),
            // do we have src/Foo/Bar.phps?
            ".phps"
            // do we have src/Foo/Bar.php?
            ".php"
        ];
        foreach($extensions as $ext) {
            $path = getClassPath($folder, $className, $extension);
            if(is_readable($path)) return $path;
        }
    }
});

Bu sınıfı kullanan bir dosya yüklendiğinde PHP’nin sınıfları yüklemeye çalışmadığını unutmayın. Bir betiğin ortasına veya hatta kapatma işlevlerine yüklenebilir . Bu, geliştiricilerin, özellikle de otomatik yüklemeyi kullananların, çalışma zamanında, özellikle phar dosyalarında yürütülen kaynak dosyaları değiştirmekten kaçınmasının nedenlerinden biridir.

Arayüzler #

Giriiş #

Arabirimler, arabirimi karşılamak için sınıfların uygulaması gereken genel API’lerin tanımlarıdır. “Sözleşmeler” olarak çalışırlar, bir dizi alt sınıfın ne yaptığını belirtirler, ancak nasıl yaptıklarını belirtmezler.

Arayüz tanımı, sınıf tanımına çok benzer, “sınıf” anahtar sözcüğünü “arayüz” olarak değiştirir:

interface Foo {

}

Arayüzler yöntemler ve/veya sabitler içerebilir, ancak öznitelik içermez. Arayüz sabitleri, sınıf sabitleriyle aynı kısıtlamalara sahiptir. Arayüz yöntemleri dolaylı olarak soyuttur:

interface Foo {
    const BAR = 'BAR';

    public function doSomething($param1, $param2);
}

Not: arabirimler yapıcılar veya yıkıcılar bildirmemelidir, çünkü bunlar sınıf düzeyinde uygulama ayrıntılarıdır.

Gerçekleşme #

Bir arabirim uygulaması gereken herhangi bir sınıf, bunu “implements” anahtar sözcüğünü kullanarak yapmalıdır. Bunu yapmak için sınıfın, arayüzde bildirilen her yöntem için aynı imzaya saygı gösteren bir uygulama sağlaması gerekir.

Tek bir sınıf aynı anda birden fazla arabirim yapabilir.

interface Foo {
    public function doSomething($param1, $param2);
}

interface Bar {
    public function doAnotherThing($param1);
}


class Baz implements Foo, Bar {
    public function doSomething($param1, $param2) {
        // ...
    }

    public function doAnotherThing($param1) {
        // ...
    }
}

Soyut sınıflar arabirimleri uygularken, tüm yöntemleri uygulamaları gerekmez. Temel sınıfta uygulanmayan herhangi bir yöntem, onu genişleten somut sınıf tarafından uygulanmalıdır:

abstract class AbstractBaz implements Foo, Bar {
    // Partial implementation of the required interface...
    public function doSomething($param1, $param2) {
        // ...
    }
}

class Baz extends AbstractBaz {
    public function doAnotherThing($param1) {
        // ...
    }
}

Arayüz gerçekleştirmenin kalıtsal bir özellik olduğuna dikkat edin. Bir arabirim uygulayan bir sınıfı genişletirken, örtük olduğu için onu somut sınıfta yeniden bildirmeniz gerekmez.

Not: PHP 5.3.9’dan önce, bir sınıf, bir yöntem belirten iki arabirim uygulayamazdı aynı isimle, çünkü belirsizliğe neden olur. PHP’nin daha yeni sürümleri izin verir bu, yinelenen yöntemler aynı imzaya sahip olduğu sürece[1].

Miras #

Sınıflar gibi, aynı ’extends’ anahtar sözcüğünü kullanarak arayüzler arasında bir kalıtım ilişkisi kurmak mümkündür. Temel fark, arayüzler için çoklu kalıtımın izin verilmesidir:

interface Foo {

}

interface Bar {

}

interface Baz extends Foo, Bar {

}

Örnekler #

Aşağıdaki örnekte bir araç için basit bir örnek arayüzümüz var. Araçlar ileri ve geri gidebilir.

interface VehicleInterface {
    public function forward();

    public function reverse();

    ...
}

class Bike implements VehicleInterface {
    public function forward() {
        $this->pedal();
    }

    public function reverse() {
        $this->backwardSteps();
    }

    protected function pedal() {
        ...
    }

    protected function backwardSteps() {
        ...
    }

    ...
}

class Car implements VehicleInterface {
    protected $gear = 'N';

    public function forward() {
        $this->setGear(1);
        $this->pushPedal();
    }

    public function reverse() {
        $this->setGear('R');
        $this->pushPedal();
    }

    protected function setGear($gear) {
        $this->gear = $gear;
    }

    protected function pushPedal() {
        ...
    }

    ...
}

Ardından arayüzü uygulayan iki sınıf oluşturuyoruz: Bike ve Car. Bisiklet ve Araba dahili olarak çok farklıdır, ancak her ikisi de araçtır ve VehicleInterface’in sağladığı aynı genel yöntemleri uygulamalıdır.

Yazı ipucu, yöntemlerin ve işlevlerin Arayüzler istemesine izin verir. Her türden aracın bulunduğu bir kapalı otopark sınıfımız olduğunu varsayalım.

class ParkingGarage {
    protected $vehicles = [];

    public function addVehicle(VehicleInterface $vehicle) {
        $this->vehicles[] = $vehicle;
    }
}

“addVehicle”, “VehicleInterface” türünde bir “$vehicle” gerektirdiğinden —somut bir uygulama değil— ParkingGarage’ın manipüle edebileceği ve kullanabileceği hem Bisikletleri hem de Arabaları girebiliriz.

Yöntem ve Mülk Görünürlüğü #

Bir sınıf içindeki yöntemlere (sınıf/nesne işlevleri) ve özelliklere (sınıf/nesne değişkenleri) uygulayabileceğiniz ve uygulandıkları yöntem veya özellik için erişim denetimi sağlayan üç görünürlük türü vardır.

Bunlar hakkında PHP Documentation for OOP Visibility içinde kapsamlı bir şekilde okuyabilirsiniz.

Halk #

Bir yöntemin veya özelliğin “genel” olarak bildirilmesi, yönteme veya özelliğe aşağıdakiler tarafından erişilmesini sağlar:

  • Bunu ilan eden sınıf.
  • Beyan edilen sınıfı genişleten sınıflar.
  • Sınıf hiyerarşisi dışındaki tüm harici nesneler, sınıflar veya kodlar.

Bu “genel” erişime bir örnek:

class MyClass {
    // Property
    public $myProperty = 'test';

    // Method
    public function myMethod() {
        return $this->myProperty;
    }
}

$obj = new MyClass();
echo $obj->myMethod();
// Out: test

echo $obj->myProperty;
// Out: test

Korumalı #

Bir yöntemin veya özelliğin “korumalı” olarak bildirilmesi, yönteme veya özelliğe aşağıdakiler tarafından erişilmesini sağlar:

  • Bunu ilan eden sınıf.
  • Beyan edilen sınıfı genişleten sınıflar.

Bu, sınıf hiyerarşisi dışındaki harici nesnelerin, sınıfların veya kodun bu yöntemlere veya özelliklere erişmesine izin vermez. Bu yöntemi/özelliği kullanan bir şeyin ona erişimi yoksa, mevcut olmayacak ve bir hata atılacaktır. Yalnızca beyan edilen selfin örnekleri (veya alt sınıfları) buna erişebilir.

Bu “korumalı” erişime bir örnek:

class MyClass {
    protected $myProperty = 'test';

    protected function myMethod() {
        return $this->myProperty;
    }
}

class MySubClass extends MyClass {
    public function run() {
        echo $this->myMethod();
    }
}

$obj = new MySubClass();
$obj->run(); // This will call MyClass::myMethod();
// Out: test

$obj->myMethod(); // This will fail.
// Out: Fatal error: Call to protected method MyClass::myMethod() from context ''

Yukarıdaki örnek, yalnızca kendi kapsamındaki “korumalı” öğelere erişebileceğinizi belirtir. Esasen: “Evin içindekilere yalnızca evin içinden erişilebilir.”

Özel #

Bir yöntemin veya özelliğin “özel” olarak bildirilmesi, yönteme veya özelliğe aşağıdakiler tarafından erişilmesini sağlar:

  • Yalnızca olarak bildiren sınıf (alt sınıflar değil).

Bir “özel” yöntem veya özellik yalnızca onu oluşturan sınıf içinde görünür ve erişilebilirdir.

Aynı türdeki nesnelerin, aynı örnekler olmasalar bile birbirlerinin özel ve korumalı üyelerine erişimi olacağını unutmayın.

class MyClass {
    private $myProperty = 'test';

    private function myPrivateMethod() {
        return $this->myProperty;
    }

    public function myPublicMethod() {
        return $this->myPrivateMethod();
    }

    public function modifyPrivatePropertyOf(MyClass $anotherInstance) {
        $anotherInstance->myProperty = "new value";
    }
}

class MySubClass extends MyClass {
    public function run() {
        echo $this->myPublicMethod();
    }

    public function runWithPrivate() {
        echo $this->myPrivateMethod();
    }
}

$obj = new MySubClass();
$newObj = new MySubClass();

// This will call MyClass::myPublicMethod(), which will then call
// MyClass::myPrivateMethod();
$obj->run(); 
// Out: test

    
$obj->modifyPrivatePropertyOf($newObj);

$newObj->run();
// Out: new value

echo $obj->myPrivateMethod(); // This will fail.
// Out: Fatal error: Call to private method MyClass::myPrivateMethod() from context ''

echo $obj->runWithPrivate(); // This will also fail.
// Out: Fatal error: Call to private method MyClass::myPrivateMethod() from context 'MySubClass'

Belirtildiği gibi, “özel” yönteme/özelliğe yalnızca tanımlı sınıfından erişebilirsiniz.

Son Anahtar Kelime #

tanım: Son Anahtar Kelime, tanımın önüne final ekleyerek alt sınıfların bir yöntemi geçersiz kılmasını önler. Sınıfın kendisi final olarak tanımlanıyorsa, genişletilemez

Son Yöntem

class BaseClass {
   public function test() {
       echo "BaseClass::test() called\n";
   }
   
   final public function moreTesting() {
       echo "BaseClass::moreTesting() called\n";
   }
}

class ChildClass extends BaseClass {
   public function moreTesting() {
       echo "ChildClass::moreTesting() called\n";
   }
}
// Results in Fatal error: Cannot override final method BaseClass::moreTesting()

Son Sınıf:

final class BaseClass {
   public function test() {
       echo "BaseClass::test() called\n";
   }

   // Here it doesn't matter if you specify the function as final or not
   final public function moreTesting() {
       echo "BaseClass::moreTesting() called\n";
   }
}

class ChildClass extends BaseClass {
}
// Results in Fatal error: Class ChildClass may not inherit from final class (BaseClass)

Son sabitler: Java’dan farklı olarak, PHP’de sınıf sabitleri için “final” anahtar sözcüğü kullanılmaz. Bunun yerine “const” anahtar sözcüğünü kullanın.

Neden ‘final’ kullanmak zorundayım?

  1. Devasa bir kalıtım zincirini önlemek
  2. Teşvik edici kompozisyon
  3. Geliştiriciyi genel kullanıcı API’sı hakkında düşünmeye zorlayın
  4. Geliştiriciyi bir nesnenin genel API’sini küçültmeye zorlayın
  5. Bir “son” sınıf her zaman genişletilebilir hale getirilebilir
  6. Kapsüllemeyi “genişletir”
  7. Bu esnekliğe ihtiyacınız yok
  8. Kodu değiştirmekte özgürsünüz

“final"den ne zaman kaçınılır: Son sınıflar yalnızca aşağıdaki varsayımlar altında etkin bir şekilde çalışır:

  1. Son sınıfın uyguladığı bir soyutlama (arayüz) vardır.
  2. Son sınıfın tüm genel API’leri bu arayüzün bir parçasıdır

Otomatik yükleme #

Bir sınıf veya kalıtım her kullanıldığında kimse “gerektirmek” veya “dahil etmek” istemez. Acı verici olabileceği ve unutulması kolay olduğu için PHP, otomatik yükleme denilen şeyi sunuyor. Zaten Composer kullanıyorsanız, Composer kullanarak otomatik yükleme hakkında bilgi edinin.

Otomatik yükleme tam olarak nedir?

İsim temelde her şeyi söylüyor. İstenen sınıfın depolandığı dosyayı almanız gerekmez, ancak PHP *otomatik olarak yükler.

Bunu temel PHP’de üçüncü taraf kodu olmadan nasıl yapabilirim?

[__autoload][2] işlevi vardır, ancak [spl_autoload_register][3] kullanmak daha iyi bir uygulama olarak kabul edilir. Bu işlevler, verilen alan içinde bir sınıf tanımlanmadığında PHP tarafından dikkate alınacaktır. Bu nedenle, mevcut bir projeye otomatik yükleme eklemek sorun değil, çünkü tanımlanmış sınıflar (‘require’ ile yani) daha önce olduğu gibi çalışacaktır. Kesinlik açısından, aşağıdaki örnekler anonim işlevleri kullanacaktır, eğer PHP < 5.3 kullanıyorsanız, işlevi tanımlayabilir ve adını spl_autoload_register argümanı olarak iletebilirsiniz.

Örnekler

spl_autoload_register(function ($className) {
    $path = sprintf('%s.php', $className);
    if (file_exists($path)) {
        include $path;
    } else {
        // file not found
    }
});

Yukarıdaki kod, [sprintf][4] kullanarak sınıf adına ve eklenen “.php” uzantısına sahip bir dosya adı eklemeye çalışır. “FooBar"ın yüklenmesi gerekiyorsa, “FooBar.php"nin var olup olmadığına bakar ve varsa onu içerir.

Elbette bu, projenin bireysel ihtiyacına uyacak şekilde genişletilebilir. Gruplamak için bir sınıf adının içindeki ‘_’ kullanılıyorsa, ör. ‘User_Post’ ve ‘User_Image’ her ikisi de ‘User’ anlamına gelir, her iki sınıf da “Kullanıcı” adlı bir klasörde tutulabilir:

spl_autoload_register(function ($className) {
    //                        replace _ by / or \ (depending on OS)
    $path = sprintf('%s.php', str_replace('_', DIRECTORY_SEPARATOR, $className) );
    if (file_exists($path)) {
        include $path;
    } else {
        // file not found
    }
});

“User_Post” sınıfı şimdi “User/Post.php” vb.‘den yüklenecektir.

spl_autoload_register çeşitli ihtiyaçlara göre uyarlanabilir. Sınıflı tüm dosyalarınız “class.CLASSNAME.php” olarak mı adlandırılıyor? Sorun değil. Çeşitli yerleştirme (User_Post_Content => “User/Post/Content.php”)? Sorun da yok.

Daha ayrıntılı bir otomatik yükleme mekanizması istiyorsanız ve yine de Composer’ı dahil etmek istemiyorsanız, üçüncü taraf kitaplıkları eklemeden çalışabilirsiniz.

spl_autoload_register(function ($className) {
    $path = sprintf('%1$s%2$s%3$s.php',
        // %1$s: get absolute path
        realpath(dirname(__FILE__)),
        // %2$s: / or \ (depending on OS)
        DIRECTORY_SEPARATOR,
        // %3$s: don't wory about caps or not when creating the files
        strtolower(
            // replace _ by / or \ (depending on OS)
            str_replace('_', DIRECTORY_SEPARATOR, $className)
        )
    );

    if (file_exists($path)) {
        include $path;
    } else {
        throw new Exception(
            sprintf('Class with name %1$s not found. Looked in %2$s.',
                $className,
                $path
            )
        );
    }
});

Bunun gibi otomatik yükleyicileri kullanarak, mutlu bir şekilde şöyle kod yazabilirsiniz:

require_once './autoload.php'; // where spl_autoload_register is defined

$foo = new Foo_Bar(new Hello_World());

Sınıfları kullanma:

class Foo_Bar extends Foo {}

class Hello_World implements Demo_Classes {}

Bu örnekler ‘foo/bar.php’, ‘foo.php’, ‘hello/world.php’ ve ‘demo/classes.php’ sınıflarını içerecektir.

Dinamik Bağlama #

Yöntem geçersiz kılma olarak da adlandırılan dinamik bağlama, birden çok sınıf aynı yöntemin farklı uygulamalarını içerdiğinde ortaya çıkan bir çalışma zamanı polimorfizmi örneğidir, ancak yöntemin çağrılacağı nesne bilinmiyor çalışma süresi kadar.

Bu, belirli bir koşul, bir eylemi gerçekleştirmek için hangi sınıfın kullanılacağını belirlerse, eylemin her iki sınıfta da aynı şekilde adlandırıldığı durumlarda yararlıdır.

interface Animal {
    public function makeNoise();
}

class Cat implements Animal {
    public function makeNoise
    {
        $this->meow();
    }
    ...
}

class Dog implements Animal {
    public function makeNoise {
        $this->bark();
    }
    ...
}

class Person {
    const CAT = 'cat';
    const DOG = 'dog';

    private $petPreference;
    private $pet;

    public function isCatLover(): bool {
        return $this->petPreference == self::CAT;
    }

    public function isDogLover(): bool {
        return $this->petPreference == self::DOG;
    }

    public function setPet(Animal $pet) {
        $this->pet = $pet;
    }

    public function getPet(): Animal {
        return $this->pet;
    }
}

if($person->isCatLover()) {
    $person->setPet(new Cat());
} else if($person->isDogLover()) {
    $person->setPet(new Dog());
}

$person->getPet()->makeNoise();

Yukarıdaki örnekte, ‘User’ sınıfı içindeki özelliğe bağlı olarak ‘makeNoise’ yapacak ‘Animal’ sınıfı (‘Köpek|Cat’) çalışma zamanına kadar bilinmemektedir.

Bir alt öğe başlatılırken bir üst kurucunun çağrılması #

Alt sınıflarla ilgili yaygın bir tuzak, ebeveyninizin ve çocuğunuzun her ikisi de bir yapıcı(__construct()) yöntemi içeriyorsa, yalnızca alt sınıf yapıcısının çalışacağıdır. Çocuğundan ebeveyn __construct() yöntemini çalıştırmanız gereken durumlar olabilir. Bunu yapmanız gerekiyorsa, parent:: kapsam çözümleyicisini kullanmanız gerekecektir:

parent::__construct();

Şimdi bunu gerçek dünyadaki bir durumda kullanmak şöyle bir şeye benzeyecektir:

class Foo {

    function __construct($args) { 
        echo 'parent'; 
    }

}

class Bar extends Foo {

    function __construct($args) {
        parent::__construct($args);
    }
}

Yukarıdakiler, ‘__construct()’ üst öğesini çalıştıracak ve ’echo’nun çalıştırılmasına neden olacaktır.

$this, self ve static artı singleton #

Geçerli nesneye başvurmak için $this kullanın. atıfta bulunmak için ‘self’i kullanın. mevcut sınıf. Başka bir deyişle, statik olmayan için $this->üye kullanın üyeler, statik üyeler için self::$member kullanın.

Aşağıdaki örnekte sayHello() ve sayGoodbye() self kullanıyor ve $this farkı burada gözlemlenebilir.

class Person {
    private $name;

    public function __construct($name) {
        $this->name = $name;
    }

    public function getName() {
        return $this->name;
    }

    public function getTitle() {
        return $this->getName()." the person";
    }

    public function sayHello() {
        echo "Hello, I'm ".$this->getTitle()."<br/>";
    }

    public function sayGoodbye() {
        echo "Goodbye from ".self::getTitle()."<br/>";
    }
}

class Geek extends Person {
    public function __construct($name) {
        parent::__construct($name);
    }

    public function getTitle() {
        return $this->getName()." the geek";
    }
}

$geekObj = new Geek("Ludwig");
$geekObj->sayHello();
$geekObj->sayGoodbye();

“Statik”, hiyerarşide yöntemi çağırdığınız herhangi bir sınıfı ifade eder. Sınıflar miras alındığında statik sınıf özelliklerinin daha iyi yeniden kullanılmasına izin verir.

Aşağıdaki kodu göz önünde bulundurun:

class Car {
    protected static $brand = 'unknown';
    
    public static function brand() {
         return self::$brand."\n";
    }
}

class Mercedes extends Car {
    protected static $brand = 'Mercedes';
}

class BMW extends Car {
    protected static $brand = 'BMW';
}

echo (new Car)->brand();
echo (new BMW)->brand();
echo (new Mercedes)->brand();

Bu, istediğiniz sonucu üretmez:

bilinmeyen bilinmeyen bilinmeyen

Bunun nedeni, “brand()” yöntemi çağrıldığında “self"in “Car” sınıfını ifade etmesidir.

Doğru sınıfa başvurmak için bunun yerine “statik” kullanmanız gerekir:

class Car {
    protected static $brand = 'unknown';
    
    public static function brand() {
         return static::$brand."\n";
    }
}

class Mercedes extends Car {
    protected static $brand = 'Mercedes';
}

class BMW extends Car {
    protected static $brand = 'BMW';
}

echo (new Car)->brand();
echo (new BMW)->brand();
echo (new Mercedes)->brand();

Bu istenen çıktıyı üretir:

bilinmeyen BMW Mercedes

Tekil #

Yeniden kullanmak istediğiniz bazı harici kaynaklara bir bağlantı oluşturması veya temsil etmesi pahalı olan bir nesneniz varsa, yani bağlantı havuzunun olmadığı bir veritabanı bağlantısı veya başka bir sisteme bir soket varsa, “statik” ve “kendini” kullanabilirsiniz. ` bir sınıftaki anahtar kelimeleri bir singleton yapmak için. Singleton modelinin kullanılması gerekip gerekmediği konusunda güçlü görüşler var, ancak kullanımları var.

class Singleton {
    private static $instance = null;

    public static function getInstance(){
        if(!isset(self::$instance)){
            self::$instance = new self();
        }
        
        return self::$instance;
    }
    
    private function __construct() {
        // Do constructor stuff
    }
}

Örnek kodda görebileceğiniz gibi, nesne referansını tutmak için $instance özel bir statik özelliği tanımlıyoruz. Bu statik olduğundan, bu başvuru bu türdeki TÜM nesneler arasında paylaşılır.

getInstance()yöntemi, nesnenin oluşturulmasını mümkün olan en son ana kadar geciktirmek için tembel örnekleme olarak bilinen bir yöntem kullanır, çünkü bellekte kullanılmayan nesnelerin asla kullanılması amaçlanmamasını istersiniz. Ayrıca, gerekenden daha fazla nesne yüklemek zorunda kalmadan sayfa yükünde zaman ve CPU tasarrufu sağlar. Yöntem, nesnenin ayarlanmış olup olmadığını kontrol etmek, değilse oluşturmak ve geri döndürmek. Bu, bu türden yalnızca bir nesnenin yaratılmasını sağlar.

Ayrıca, kimsenin onu “new” anahtar sözcüğüyle dışarıdan oluşturmamasını sağlamak için yapıcıyı özel olacak şekilde ayarlıyoruz. Bu sınıftan miras almanız gerekiyorsa, ‘private’ anahtar kelimelerini ‘protected’ olarak değiştirmeniz yeterlidir.

Bu nesneyi kullanmak için aşağıdakileri yazmanız yeterlidir:

$singleton = Singleton::getInstance();

Şimdi, gevşek bir şekilde bağlanmış nesneleri hedefleyebileceğiniz ve hedefleyebileceğiniz bağımlılık enjeksiyonunu kullanmanızı rica ediyorum, ancak bazen bu makul değildir ve singleton modeli kullanılabilir.

Temel Sınıf Tanımlama #

PHP’deki bir nesne, değişkenler ve işlevler içerir. Nesneler tipik olarak, bu sınıfın tüm nesnelerinin içereceği değişkenleri ve işlevleri tanımlayan bir sınıfa aittir.

Bir sınıfı tanımlamak için sözdizimi şöyledir:

class Shape {
    public $sides = 0;
    
    public function description() {
        return "A shape with $this->sides sides.";
    }
}

Bir sınıf tanımlandıktan sonra, aşağıdakileri kullanarak bir örnek oluşturabilirsiniz:

$myShape = new Shape();

Nesne üzerindeki değişkenlere ve işlevlere şu şekilde erişilir:

$myShape = new Shape();
$myShape->sides = 6;

print $myShape->description(); // "A shape with 6 sides"

yapıcı #

Sınıflar, nesne oluşturmanın bir parçası olarak yürütülen özel bir __construct() yöntemini tanımlayabilir. Bu genellikle kullanılır bir nesnenin başlangıç ​​değerlerini belirtin:

class Shape {
    public $sides = 0;
    
    public function __construct($sides) {
        $this->sides = $sides;
    }
    
    public function description() {
        return "A shape with $this->sides sides.";
    }
}

$myShape = new Shape(6);

print $myShape->description(); // A shape with 6 sides

Başka Bir Sınıfı Genişletme #

Sınıf tanımları, mevcut sınıf tanımlarını genişletebilir, yeni değişkenler ve işlevler ekleyebilir ve üst sınıfta tanımlananları değiştirebilir.

İşte önceki örneği genişleten bir sınıf:

class Square extends Shape {
    public $sideLength = 0;
    
    public function __construct($sideLength) {
       parent::__construct(4);
       
       $this->sideLength = $sideLength;
    }
    
    public function perimeter() {
        return $this->sides * $this->sideLength;
    }

    public function area() {
        return $this->sideLength * $this->sideLength;
    }
}

“Square” sınıfı, hem “Shape” hem de “Square” sınıfı için değişkenler ve davranışlar içerir:

$mySquare = new Square(10);

print $mySquare->description()/ // A shape with 4 sides

print $mySquare->perimeter() // 40

print $mySquare->area() // 100

Anonim Sınıflar #

Anonim sınıflar, tek seferlik hızlı nesnelerin kolayca oluşturulmasını sağlamak için PHP 7’ye dahil edildi. Yapıcı argümanları alabilir, diğer sınıfları genişletebilir, arayüzleri uygulayabilir ve normal sınıflar gibi özellikleri kullanabilirler.

En temel biçiminde, anonim bir sınıf aşağıdaki gibi görünür:

new class("constructor argument") {
    public function __construct($param) {
        var_dump($param);
    }
}; // string(20) "constructor argument"

Anonim bir sınıfı başka bir sınıfın içine yerleştirmek, o sınıfın o dış sınıfın özel veya korumalı yöntemlerine veya özelliklerine erişmesine izin vermez. Dış sınıfın korumalı yöntemlere ve özelliklerine erişim, dış sınıfı anonim sınıftan genişleterek elde edilebilir. Dış sınıfın özel özelliklerine erişim, bunları anonim sınıfın yapıcısına ileterek elde edilebilir.

Örneğin:

class Outer {
    private $prop = 1;
    protected $prop2 = 2;

    protected function func1() {
        return 3;
    }

    public function func2() {
        // passing through the private $this->prop property
        return new class($this->prop) extends Outer {
            private $prop3;

            public function __construct($prop) {
                $this->prop3 = $prop;
            }

            public function func3() {
                // accessing the protected property Outer::$prop2
                // accessing the protected method Outer::func1()
                // accessing the local property self::$prop3 that was private from Outer::$prop
                return $this->prop2 + $this->func1() + $this->prop3;
            }
        };
    }
}

echo (new Outer)->func2()->func3(); // 6
