session_start() Seçenekler #

PHP Sessions ile başlayarak, session-based php.ini options içeren bir diziyi session_start işlevine iletebiliriz.

Örnek

<?php
   if (version_compare(PHP_VERSION, '7.0.0') >= 0) {
       // php >= 7 version
       session_start([
          'cache_limiter' => 'private',
          'read_and_close' => true,
       ]);
   } else {
       // php < 7 version
       session_start();
   }
?>

Bu özellik ayrıca, varsayılan olarak “true” olan ve oturum verilerinin yalnızca değişirse yeniden yazılacağı anlamına gelen “session.lazy_write” adlı yeni bir “php.ini” ayarı sunar.

Referans: https://wiki.php.net/rfc/session-lock-ini

Oturum Kilitleme #

Hepimizin bildiği gibi PHP, oturum verilerini sunucu tarafında bir dosyaya yazar. Oturumu session_start() aracılığıyla başlatan php betiğine bir istek yapıldığında, PHP bu oturum dosyasını kilitler ve bu, aynı session_id için gelen diğer isteklerin tamamlanmasını engeller/bekler, çünkü diğer istekler takılıp kalır. oturum dosyası kilitli serbest bırakılıncaya kadar veya yayınlanmadığı sürece session_start()

Oturum dosyası, komut dosyası tamamlanana veya oturum manuel olarak kapatılana kadar kilitli kalır. Bu durumdan kaçınmak için * yani. birden fazla isteğin bloke edilmesini önlemek için*, oturumu başlatabilir ve oturumu kapatarak kilidi oturum dosyasından kaldırabilir ve kalan isteklerin devam etmesine izin verebiliriz.

// php < 7.0 
// start session 
session_start();

// write data to session
$_SESSION['id'] = 123; // session file is locked, so other requests are blocked

// close the session, release lock
session_write_close();

Şimdi oturum kapalıysa oturum değerlerini nasıl okuyacağımızı, oturum kapandıktan sonra bile güzelleştireceğimizi düşünecek, oturum hala müsait. Yani, yine de oturum verilerini okuyabiliyoruz.

echo $_SESSION['id'];    // will output  123

php >= 7.0‘da salt_okunur oturumu, okuma_yazma oturumu ve lazy_write oturumu olabilir, bu nedenle session_write_close() kullanılması gerekmeyebilir.

Oturum verilerini değiştirme #

$_SESSION değişkeni bir dizidir ve onu normal bir dizi gibi alabilir veya değiştirebilirsiniz.

<?php
// Starting the session
session_start();

// Storing the value in session
$_SESSION['id'] = 342;

// conditional usage of session values that may have been set in a previous session
if(!isset($_SESSION["login"])) {
    echo "Please login first";
    exit;
}
// now you can use the login safely
$user = $_SESSION["login"];

// Getting a value from the session data, or with default value, 
//     using the Null Coalescing operator in PHP 7
$name = $_SESSION['name'] ?? 'Anonymous';

Ayrıca bir dizi üzerinde nasıl çalışılacağına ilişkin daha fazla referans için https://www.wikiod.com/tr/php/bir-diziyi-manipule-etme adresine bakın.

Bir oturumda bir nesne saklarsanız, yalnızca bir sınıf otomatik yükleyiciniz varsa veya sınıfı zaten yüklediyseniz, zarif bir şekilde alınabileceğini unutmayın. Aksi takdirde, nesne ‘__PHP_Incomplete_Class’ türü olarak ortaya çıkar ve bu daha sonra çökmelere yol açabilir. Otomatik yükleme hakkında https://www.wikiod.com/tr/php/snflar-ve-nesneler#Ad Alanı ve Otomatik Yükleme adresine bakın.

Uyarı: #

Oturum verileri ele geçirilebilir. Bu, şu bölümde özetlenmiştir: Pro PHP Güvenliği: Uygulama Güvenliği İlkelerinden XSS Savunmasının Uygulanmasına - Bölüm 7: Oturum Ele Geçirmeyi Önleme Bu nedenle, hiçbir kişisel bilgiyi $_SESSION içinde saklamamanız şiddetle tavsiye edilebilir. . Bu, en kritik olarak kredi kartı numaralarını, devlet tarafından verilen kimlikleri ve şifreleri içerir; ancak aynı zamanda adlar, e-postalar, telefon numaraları vb. gibi daha az varsayımsal verilere de uzanır ve bu da bir bilgisayar korsanının meşru bir kullanıcının kimliğine bürünmesine/ödün vermesine olanak tanır. Genel bir kural olarak, oturum verilerinde sayısal tanımlayıcılar gibi değersiz/kişisel olmayan değerler kullanın.

Tüm oturumu yok et #

Yok etmek istediğiniz bir oturumunuz varsa, bunu session_destroy() ile yapabilirsiniz.

/*
    Let us assume that our session looks like this:
    Array([firstname] => Jon, [id] => 123)

    We first need to start our session:
*/
session_start();

/*
    We can now remove all the values from the `SESSION` superglobal:
    If you omitted this step all of the global variables stored in the 
    superglobal would still exist even though the session had been destroyed.
*/
$_SESSION = array();

// If it's desired to kill the session, also delete the session cookie.
// Note: This will destroy the session, and not just the session data!
if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(session_name(), '', time() - 42000,
        $params["path"], $params["domain"],
        $params["secure"], $params["httponly"]
    );
}

//Finally we can destroy the session:
session_destroy();

“session_destroy()” kullanımı, “$_SESSION = array();” gibi bir şey kullanmaktan farklıdır; bu, “SESSION” süper küreselinde depolanan tüm değerleri kaldıracak, ancak oturumun gerçek depolanmış sürümünü yok etmeyecektir.

Not: “session_unset()” yerine “$_SESSION = array();” kullanıyoruz çünkü kılavuz şunları şart koşuyor:

$_SESSION kullanmayan eski kullanımdan kaldırılmış kod için yalnızca session_unset() kullanın.

Oturum adı #

Oturum çerezlerinin oluşturulup oluşturulmadığını kontrol etme #

Oturum adı, oturumları depolamak için kullanılan tanımlama bilgisinin adıdır. Bunu, kullanıcı için bir oturum için çerezlerin oluşturulup oluşturulmadığını tespit etmek için kullanabilirsiniz:

if(isset($_COOKIE[session_name()])) {
    session_start();
}

Gereksiz yere tanımlama bilgileri oluşturmak istemiyorsanız, bu yöntemin genellikle yararlı olmadığını unutmayın.

Oturum adını değiştirme #

Oturum adını session_name() çağırarak güncelleyebilirsiniz.

//Set the session name
session_name('newname');
//Start the session
session_start();

session_name() içinde herhangi bir argüman sağlanmazsa, geçerli oturum adı döndürülür.

Yalnızca alfanümerik karakterler içermelidir; kısa ve açıklayıcı olmalıdır (yani, çerez uyarıları etkinleştirilmiş kullanıcılar için). Oturum adı sadece rakamlardan oluşamaz, en az bir harf bulunmalıdır. Aksi takdirde, her seferinde yeni bir oturum kimliği oluşturulur.

Hatasız Güvenli Oturum Başlatma #

Pek çok geliştirici, büyük projeler üzerinde çalışırken, özellikle de eklentiler, eklentiler, bileşenler vb. üzerinde bazı modüler CMS üzerinde çalışıyorlarsa, bu sorunu yaşarlar. Burada, ilk önce PHP sürümünün tüm sürümleri kapsayacak şekilde işaretlendiği ve daha sonra işaretlendiği güvenli oturum başlangıcı için bir çözümdür. oturum başlatılırsa. Oturum yoksa, oturumu güvenli başlatırım. Oturum varsa hiçbir şey olmaz.

if (version_compare(PHP_VERSION, '7.0.0') >= 0) {
    if(session_status() == PHP_SESSION_NONE) {
        session_start(array(
          'cache_limiter' => 'private',
          'read_and_close' => true,
       ));
    }
}
else if (version_compare(PHP_VERSION, '5.4.0') >= 0)
{
    if (session_status() == PHP_SESSION_NONE) {
        session_start();
    }
}
else
{
    if(session_id() == '') {
        session_start();
    }
}

Bu, “session_start” hatasını önlemek için size çok yardımcı olabilir.
