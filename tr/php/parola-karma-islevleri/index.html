Daha güvenli web hizmetleri, parolaları düz metin biçiminde saklamaktan kaçındığından, PHP gibi diller, daha güvenli endüstri standardını desteklemek için çeşitli (şifresi çözülemeyen) karma işlevleri sağlar. Bu konu, PHP ile uygun karma için belgeler sağlar.

Sözdizimi #

  • string password_hash ( string $parola , tamsayı $algo [, dizi $seçenekler ] )
  • boolean password_verify ( string $password , string $hash )
  • boolean password_needs_rehash ( string $hash , tamsayı $algo [, dizi $options ] )
  • dizi password_get_info (dize $hash )

PHP 5.5’ten önce, “parola_*” işlevlerini sağlamak için uyumluluk paketini kullanabilirsiniz. Yapabiliyorsanız, uyumluluk paketini kullanmanız şiddetle tavsiye edilir.

Uyumluluk paketi olsun veya olmasın, crypt() aracılığıyla doğru Bcrypt işlevselliği PHP 5.3.7+ sürümüne dayanır, aksi takdirde şifreleri ASCII- ile kısıtlamalısınız sadece karakter kümeleri.

Not: PHP 5.5 veya altı kullanıyorsanız, artık herhangi bir güvenlik güncellemesi almayan PHP’nin desteklenmeyen bir sürümünü kullanıyorsunuz demektir. Mümkün olan en kısa sürede güncelleyin, daha sonra şifre karmalarınızı güncelleyebilirsiniz.

Algoritma Seçimi #

Güvenli algoritmalar #

  • bcrypt, hash hesaplama süresini artırmak için anahtar uzatmayı kullandığınız sürece en iyi seçeneğinizdir, çünkü [kaba kuvvet saldırılarını son derece yavaşlatır](http://arstechnica.com/security/2015/08/cracking- tüm-saldırıya uğramış-ashley-madison-parolaları-bir ömür boyu sürebilir/).
  • argon2, [PHP 7.2’de sunulacak olan] başka bir seçenektir(https://wiki.php.net/rfc/argon2_password_hash).

Güvenli olmayan algoritmalar #

Aşağıdaki karma algoritmalar güvensizdir veya amaca uygun değildir ve bu nedenle kullanılmamalıdır. Yavaş ve kaba kuvvetle zorlanan parola karmaları yerine hızlı özetler için tasarlandıkları için parola karmaları için asla uygun değildiler.

Bunlardan herhangi birini kullanırsanız, tuzlar dahil, önerilen güvenli algoritmalardan birine en kısa sürede geçmelisiniz**.

Güvensiz kabul edilen algoritmalar:

  • MD4 - çarpışma saldırısı 1995’te bulundu
  • MD5 - 2005’te bulunan çarpışma saldırısı
  • SHA-1 - 2015’te gösterilen çarpışma saldırısı

Bazı algoritmalar, gerçekliği kanıtlamak için mesaj özeti algoritması olarak güvenle kullanılabilir, ancak asla şifre karma algoritması olarak kullanılamaz:

  • SHA-2
  • SHA-3

SHA256 ve SHA512 gibi güçlü karmaların kesintisiz ve sağlam olduğunu unutmayın, ancak bcrypt veya argon2 karma işlevlerini kullanmak genellikle daha güvenlidir, çünkü bu algoritmalara karşı kaba kuvvet saldırıları klasik bilgisayarlar için çok daha zordur.

Şifre karması oluşturma #

Mevcut sektördeki en iyi uygulama standart karma veya anahtar türetmesini kullanmak için password_hash() kullanarak parola karmaları oluşturun. Yazma sırasında standart bcrypt‘dir; bu, “PASSWORD_DEFAULT” öğesinin “PASSWORD_BCRYPT” ile aynı değeri içerdiği anlamına gelir.

$options = [
    'cost' => 12,
];

$hashedPassword = password_hash($plaintextPassword, PASSWORD_DEFAULT, $options);

Üçüncü parametre zorunlu değildir.

“Maliyet” değeri, üretim sunucunuzun donanımına göre seçilmelidir. Bunu artırmak, parolanın oluşturulmasını daha maliyetli hale getirecektir. Üretmek ne kadar maliyetli olursa, onu kırmaya çalışan herkesin onu üretmesi de o kadar uzun sürer. Maliyet ideal olarak mümkün olduğunca yüksek olmalıdır, ancak pratikte her şeyi çok fazla yavaşlatmayacak şekilde ayarlanmalıdır. 0.1 ile 0.4 saniye arasında bir yerde sorun olmaz. Şüpheniz varsa varsayılan değeri kullanın.

5.5.0’dan daha düşük PHP’de password_* işlevleri kullanılamaz. Bu işlevleri değiştirmek için uyumluluk paketini kullanmalısınız. Uyumluluk paketinin PHP 5.3.7 veya daha yüksek bir sürüm gerektirdiğine veya “$2y” düzeltmesinin geri yüklendiği bir sürümün (RedHat’ın sağladığı gibi) gerektirdiğine dikkat edin.

Bunları kullanamıyorsanız, crypt() ile parola karmasını uygulayabilirsiniz. işlevsellik.

// this is a simple implementation of a bcrypt hash otherwise compatible
// with `password_hash()`
// not guaranteed to maintain the same cryptographic strength of the full `password_hash()`
// implementation

// if `CRYPT_BLOWFISH` is 1, that means bcrypt (which uses blowfish) is available
// on your system
if (CRYPT_BLOWFISH == 1) {
    $salt = mcrypt_create_iv(16, MCRYPT_DEV_URANDOM);
    $salt = base64_encode($salt);
    // crypt uses a modified base64 variant
    $source = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/';
    $dest = './ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789';
    $salt = strtr(rtrim($salt, '='), $source, $dest);
    $salt = substr($salt, 0, 22);
    // `crypt()` determines which hashing algorithm to use by the form of the salt string
    // that is passed in
    $hashedPassword = crypt($plaintextPassword, '$2y$10$'.$salt.'$');
}
son sürüm

Şifre karması için tuz #

Kript algoritmasının güvenilirliğine rağmen gökkuşağı tablolarına karşı hala güvenlik açığı bulunmaktadır. Bu nedenle tuz kullanılması tavsiye edilir.

Tuz, kaynak dizeyi benzersiz kılmak için karma oluşturmadan önce parolaya eklenen bir şeydir. İki özdeş parola verildiğinde, elde edilen karmalar da benzersiz olacaktır, çünkü bunların tuzları benzersizdir.

Rastgele bir tuz, şifre güvenliğinizin en önemli parçalarından biridir. Bu, bilinen parola karmalarının bir arama tablosunda bile, rastgele bir tuz kullanıldığından, bir saldırganın kullanıcı parola karmasını veritabanı parola karmalarıyla eşleştiremeyeceği anlamına gelir. Her zaman rastgele ve kriptografik olarak güvenli tuzlar kullanmalısınız. Daha fazlasını okuyun

password_hash() bcrypt algoritması ile, düz metin tuzu, elde edilen hash ile birlikte saklanır; bu, hash’in farklı sistemler ve platformlar arasında aktarılabileceği ve yine de orijinal şifreyle eşleştirilebileceği anlamına gelir.

Bu önerilmediğinde bile, kendi rastgele tuzunuzu tanımlamak için “tuz” seçeneğini kullanabilirsiniz.

 $options = [
        'salt' => $salt, //see example below
 ];

Önemli. Bu seçeneği atlarsanız, her şifre karma işlemi için password_hash() tarafından rastgele bir tuz oluşturulur. Bu, amaçlanan çalışma modudur.

son sürüm

Tuz seçeneği PHP 7.0.0’dan itibaren kullanımdan kaldırıldı. Artık sadece varsayılan olarak üretilen tuzun kullanılması tercih edilmektedir.

son sürüm

Mevcut bir parola karma değerinin daha güçlü bir algoritmaya yükseltilip yükseltilemeyeceğini belirleyin #

Sistemin parolalarınızı hash etmek için en iyi algoritmayı seçmesine izin vermek için “PASSWORD_DEFAULT” yöntemini kullanıyorsanız, varsayılanın gücü arttıkça, kullanıcılar oturum açarken eski parolaları yeniden düzenlemek isteyebilirsiniz.

<?php
// first determine if a supplied password is valid
if (password_verify($plaintextPassword, $hashedPassword)) {

    // now determine if the existing hash was created with an algorithm that is
    // no longer the default
    if (password_needs_rehash($hashedPassword, PASSWORD_DEFAULT)) {

        // create a new hash with the new default
        $newHashedPassword = password_hash($plaintextPassword, PASSWORD_DEFAULT);

        // and then save it to your data store
        //$db->update(...);
    }
}
?>

Sisteminizde password_* işlevleri mevcut değilse (ve aşağıdaki açıklamalarda bağlantısı verilen uyumluluk paketini kullanamıyorsanız), aşağıdakine benzer bir yöntemle orijinal karma oluşturmak için kullanılan algoritmayı ve kullanılan algoritmayı belirleyebilirsiniz:

<?php
if (substr($hashedPassword, 0, 4) == '$2y$' && strlen($hashedPassword) == 60) {
    echo 'Algorithm is Bcrypt';
    // the "cost" determines how strong this version of Bcrypt is
    preg_match('/\$2y\$(\d+)\$/', $hashedPassword, $matches);
    $cost = $matches[1];
    echo 'Bcrypt cost is '.$cost;
}
?>

Bir karmaya karşı parolayı doğrulama #

‘password_verify()’, bir parolanın bilinen bir karmaya karşı geçerliliğini doğrulamak için sağlanan yerleşik işlevdir (PHP 5.5’ten itibaren).

<?php
if (password_verify($plaintextPassword, $hashedPassword)) {
    echo 'Valid Password';
}
else {
    echo 'Invalid Password.';
}
?>

Desteklenen tüm karma algoritmalar, karmanın kendisinde hangi karmanın kullanıldığını tanımlayan bilgileri depolar, bu nedenle düz metin parolasını kodlamak için hangi algoritmayı kullandığınızı belirtmenize gerek yoktur.

Sisteminizde password_* işlevleri mevcut değilse (ve aşağıdaki açıklamalarda bağlantısı verilen uyumluluk paketini kullanamıyorsanız), ‘crypt()’ işleviyle parola doğrulaması uygulayabilirsiniz. Zamanlama saldırılarından kaçınmak için özel önlemler alınması gerektiğini lütfen unutmayın.

<?php
// not guaranteed to maintain the same cryptographic strength of the full `password_hash()`
// implementation
if (CRYPT_BLOWFISH == 1) {
    // `crypt()` discards all characters beyond the salt length, so we can pass in
    // the full hashed password
    $hashedCheck = crypt($plaintextPassword, $hashedPassword);

    // this a basic constant-time comparison based on the full implementation used
    // in `password_hash()`
    $status = 0;
    for ($i=0; $i<strlen($hashedCheck); $i++) {
        $status |= (ord($hashedCheck[$i]) ^ ord($hashedPassword[$i]));
    }

    if ($status === 0) {
        echo 'Valid Password';
    }
    else {
        echo 'Invalid Password';
    }
}
?>
