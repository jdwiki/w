Konu, tüm makine öğrenimi algoritmaları için PHP-ML kullanır. Kütüphanenin kurulumu kullanılarak yapılabilir

composer require php-ai/php-ml

Aynısı için github deposu burada bulunabilir.

Ayrıca verilen örneklerin sadece gösterim amacıyla çok küçük veri seti olduğunu belirtmekte fayda var. Gerçek veri seti bundan daha kapsamlı olmalıdır.

PHP-ML kullanarak sınıflandırma #

Makine Öğreniminde sınıflandırma, yeni bir gözlemin hangi kategorilere ait olduğunu belirleyen problemdir. Sınıflandırma, “Denetimli Makine Öğrenimi” kategorisine girer.

Sınıflandırma uygulayan herhangi bir algoritma sınıflandırıcı olarak bilinir.

PHP-ML’de desteklenen sınıflandırıcılar şunlardır:

  • SVC (Destek Vektör Sınıflandırması)
  • k-En Yakın Komşular
  • Naif bayanlar

’train’ ve ‘predict’ yöntemi tüm sınıflandırıcılar için aynıdır. Tek fark, kullanılan temel algoritmada olacaktır.

SVC (Destek Vektör Sınıflandırması) #

Yeni bir gözlemi tahmin etmeye başlamadan önce sınıflandırıcımızı eğitmemiz gerekiyor. Aşağıdaki kodu göz önünde bulundurun

// Import library
use Phpml\Classification\SVC;
use Phpml\SupportVectorMachine\Kernel;

// Data for training classifier
$samples = [[1, 3], [1, 4], [2, 4], [3, 1], [4, 1], [4, 2]];  // Training samples
$labels = ['a', 'a', 'a', 'b', 'b', 'b'];

// Initialize the classifier
$classifier = new SVC(Kernel::LINEAR, $cost = 1000);
// Train the classifier
$classifier->train($samples, $labels);

Kod oldukça yalındır. Yukarıda kullanılan “$cost”, her bir eğitim örneğini yanlış sınıflandırmaktan ne kadar kaçınmak istediğimizin bir ölçüsüdür. Daha küçük bir “$cost” değeri için yanlış sınıflandırılmış örnekler alabilirsiniz. Varsayılan olarak 1.0 olarak ayarlanmıştır

Artık sınıflandırıcıyı eğittiğimize göre bazı gerçek tahminler yapmaya başlayabiliriz. Tahminler için sahip olduğumuz aşağıdaki kodları göz önünde bulundurun

$classifier->predict([3, 2]); // return 'b'
$classifier->predict([[3, 2], [1, 5]]); // return ['b', 'a']

Yukarıdaki durumda sınıflandırıcı, sınıflandırılmamış örnekler alabilir ve oradaki etiketleri tahmin edebilir. ‘predict’ yöntemi, tek bir örneğin yanı sıra bir dizi örnek alabilir.

k-En Yakın Komşular #

Bu algoritma için sınıflandırıcı iki parametre alır ve şu şekilde başlatılabilir:

$classifier = new KNearestNeighbors($neighbor_num=4);
$classifier = new KNearestNeighbors($neighbor_num=3, new Minkowski($lambda=4));

$neighbor_num knn algoritmasında taranacak en yakın komşuların sayısıdır, ikinci parametre ise ilk durumda varsayılan olarak ` olan uzaklık ölçüsüdür. Öklidyen. Minkowski hakkında daha fazla bilgiyi burada bulabilirsiniz.

Aşağıda bu sınıflandırıcının nasıl kullanılacağına dair kısa bir örnek verilmiştir.

// Training data
$samples = [[1, 3], [1, 4], [2, 4], [3, 1], [4, 1], [4, 2]];
$labels = ['a', 'a', 'a', 'b', 'b', 'b'];

// Initialize classifier
$classifier = new KNearestNeighbors();
// Train classifier
$classifier->train($samples, $labels);

// Make predictions
$classifier->predict([3, 2]); // return 'b'
$classifier->predict([[3, 2], [1, 5]]); // return ['b', 'a']

NaiveBayes Sınıflandırıcı #

‘NaiveBayes Sınıflandırıcı’, ‘Bayes’ teoremine dayanır ve yapıcıda herhangi bir parametreye ihtiyaç duymaz.

Aşağıdaki kod, basit bir tahmin uygulamasını gösterir

// Training data
$samples = [[5, 1, 1], [1, 5, 1], [1, 1, 5]];
$labels = ['a', 'b', 'c'];

// Initialize classifier
$classifier = new NaiveBayes();
// Train classifier
$classifier->train($samples, $labels);

// Make predictions
$classifier->predict([3, 1, 1]); // return 'a'
$classifier->predict([[3, 1, 1], [1, 4, 1]); // return ['a', 'b']

Pratik vaka #

Şimdiye kadar tüm durumumuzda sadece tamsayı dizileri kullandık, ancak gerçek hayatta durum böyle değil. Bu nedenle, sınıflandırıcıların nasıl kullanılacağına dair pratik bir durumu açıklamaya çalışayım.

Doğadaki çiçeklerin özelliklerini saklayan bir uygulamanız olduğunu varsayalım. Basitlik adına, yaprakların rengini ve uzunluğunu düşünebiliriz. Dolayısıyla verilerimizi eğitmek için iki özellik kullanılacaktır. color, her birine bir int değeri atayabileceğiniz daha basit olanıdır ve uzunluk için (0 mm,10 mm)=1 , (10 mm,20 mm)=2 gibi bir aralığınız olabilir. İlk verilerle sınıflandırıcınızı eğitin. Şimdi, bir kullanıcınızın arka bahçesinde yetişen çiçek türünü tanımlaması gerekiyor. Yaptığı şey çiçeğin “rengini” seçmek ve taç yapraklarının uzunluğunu eklemek. Çalışan sınıflandırıcınız çiçeğin türünü algılayabilir (“Yukarıdaki örnekte etiketler”)

Gerileme #

PHP-ML kullanarak sınıflandırmada yeni gözleme etiketler atadık. Gerileme, çıktı değerinin bir sınıf etiketi değil, sürekli bir değer olması farkıyla hemen hemen aynıdır. Tahmin ve tahmin için yaygın olarak kullanılır. PHP-ML aşağıdaki regresyon algoritmalarını destekler

  • Vektör Regresyonunu Destekleyin
  • En Küçük Kareler Doğrusal Regresyon

Regresyon, sınıflandırmada kullanılanla aynı “tren” ve “tahmin” yöntemlerine sahiptir.

Destek Vektör Regresyonu #

Bu, SVM (Support Vector Machine) için regresyon versiyonudur. Sınıflandırmada olduğu gibi ilk adım modelimizi eğitmektir.

// Import library
use Phpml\Regression\SVR;
use Phpml\SupportVectorMachine\Kernel;

// Training data
$samples = [[60], [61], [62], [63], [65]];
$targets = [3.1, 3.6, 3.8, 4, 4.1];

// Initialize regression engine
$regression = new SVR(Kernel::LINEAR);
// Train regression engine
$regression->train($samples, $targets);

Regresyonda “$hedefler”, sınıflandırmanın aksine sınıf etiketleri değildir. Bu, ikisi için farklılaştırıcı faktörlerden biridir. Modelimizi verilerle eğittikten sonra gerçek tahminlerle başlayabiliriz.

$regression->predict([64])  // return 4.03

Tahminlerin hedefin dışında bir değer döndürdüğünü unutmayın.

En Küçük Kareler Doğrusal Regresyon #

Bu algoritma, yaklaşık çözüm için “en küçük kareler yöntemini” kullanır. Aşağıda basit bir eğitim ve tahmin kodu gösterilmektedir

// Training data
$samples = [[60], [61], [62], [63], [65]];
$targets = [3.1, 3.6, 3.8, 4, 4.1];

// Initialize regression engine
$regression = new LeastSquares();
// Train engine
$regression->train($samples, $targets);
// Predict using trained engine
$regression->predict([64]); // return 4.06

PHP-ML ayrıca ‘Çoklu Doğrusal Regresyon’ seçeneği sunar. Aynısı için örnek bir kod aşağıdaki gibi olabilir

$samples = [[73676, 1996], [77006, 1998], [10565, 2000], [146088, 1995], [15000, 2001], [65940, 2000], [9300, 2000], [93739, 1996], [153260, 1994], [17764, 2002], [57000, 1998], [15000, 2000]];
$targets = [2000, 2750, 15500, 960, 4400, 8800, 7100, 2550, 1025, 5900, 4600, 4400];

$regression = new LeastSquares();
$regression->train($samples, $targets);
$regression->predict([60000, 1996]) // return 4094.82

“Çoklu Doğrusal Regresyon”, özellikle birden fazla faktör veya özellik sonucu belirlediğinde kullanışlıdır.

Pratik vaka #

Şimdi gerçek hayat senaryosunda bir regresyon uygulamasını ele alalım.

Diyelim ki çok popüler bir web sitesi işletiyorsunuz, ancak trafik değişmeye devam ediyor. Herhangi bir zamanda dağıtmanız gereken sunucu sayısını tahmin edecek bir çözüm istiyorsunuz. Barındırma sağlayıcınızın size sunucuları oluşturmanız için bir api verdiğini ve her sunucunun önyüklemesinin 15 dakika sürdüğünü varsayalım. Önceki trafik verilerine ve regresyona dayanarak, herhangi bir zamanda uygulamanıza çarpacak trafiği tahmin edebilirsiniz. Bu bilgiyi kullanarak, dalgalanmadan 15 dakika önce bir sunucuyu başlatabilir, böylece uygulamanızın çevrimdışı duruma geçmesini önleyebilirsiniz.

Kümeleme #

Kümeleme, benzer nesneleri birlikte gruplandırmakla ilgilidir. Örüntü tanıma için yaygın olarak kullanılır. “Kümeleme”, “denetimsiz makine öğrenimi” kapsamındadır, bu nedenle eğitim gerekmez. PHP-ML, aşağıdaki kümeleme algoritmalarını destekler

  • k-Araçlar
  • dbs taraması

k-Araçlar #

k-Ortalamalar, verileri eşit varyanslı “n” gruplarına ayırır. Bu, çözümümüzde ihtiyaç duyduğumuz küme sayısı olan bir ’n’ sayısını geçmemiz gerektiği anlamına gelir. Aşağıdaki kod daha fazla netlik getirmeye yardımcı olacaktır

// Our data set
$samples = [[1, 1], [8, 7], [1, 2], [7, 8], [2, 1], [8, 9]];

// Initialize clustering with parameter `n`
$kmeans = new KMeans(3);
$kmeans->cluster($samples); // return [0=>[[7, 8]], 1=>[[8, 7]], 2=>[[1,1]]]

Çıktının 3 dizi içerdiğine dikkat edin, çünkü bu, “KMeans” yapıcısındaki “n"nin değeriydi. Yapıcıda “başlatma yöntemi” olacak isteğe bağlı bir ikinci parametre de olabilir. Örneğin düşünün

$kmeans = new KMeans(4, KMeans::INIT_RANDOM);

“INIT_RANDOM”, kümeleri belirlemeye çalışırken tamamen rastgele bir merkez nokta yerleştirir. Ancak, merkezin verilerden çok uzak olmasını önlemek için, verilerin uzay sınırlarıyla sınırlandırılmıştır.

Varsayılan kurucu “başlatma yöntemi”, süreci hızlandırmak için centroid’i akıllı bir şekilde seçen kmeans++‘dir.

DBSCAN #

‘KMeans’ın aksine, ‘DBSCAN’ yoğunluk tabanlı bir kümeleme algoritmasıdır, yani sonucumuzda istediğimiz küme sayısını belirleyecek olan ’n’yi geçmeyeceğiz. Öte yandan, bunun çalışması için iki parametre gerekir

  1. $minSamples : Bir kümede bulunması gereken minimum nesne sayısı
  2. $epsilon : Aynı kümede kabul edilmeleri için iki örnek arasındaki maksimum uzaklık hangisidir.

Aynısı için hızlı bir örnek aşağıdaki gibidir

// Our sample data set
$samples = [[1, 1], [8, 7], [1, 2], [7, 8], [2, 1], [8, 9]];

$dbscan = new DBSCAN($epsilon = 2, $minSamples = 3);
$dbscan->cluster($samples); // return [0=>[[1, 1]], 1=>[[8, 7]]]

Kod hemen hemen açıklayıcıdır. Önemli bir fark, çıktı dizisindeki öğelerin sayısını KMeans’in aksine bilmenin bir yolu olmamasıdır.

Pratik Vaka #

Şimdi gerçek hayat senaryosunda kümeleme kullanımına bir göz atalım

Kümeleme, “kalıp tanıma” ve “veri madenciliği"nde yaygın olarak kullanılmaktadır. Bir içerik yayınlama uygulamanız olduğunu düşünün. Artık kullanıcılarınızı elde tutmak için sevdikleri içeriğe bakmaları gerekiyor. Basitlik adına, belirli bir web sayfasında bir dakikadan daha uzun süre kalırlarsa ve aşağıya inerlerse, o içeriği sevdiklerini varsayalım. Artık içeriğinizin her biri onunla benzersiz bir tanımlayıcıya sahip olacak ve kullanıcı da öyle. Buna göre küme yapın ve hangi kullanıcı segmentinin benzer bir içerik zevkine sahip olduğunu öğreneceksiniz. Bu da, aynı kümedeki bazı kullanıcılar makaleyi severse diğerlerinin de seveceğini ve bunun uygulamanızda öneriler olarak gösterilebileceğini varsayabileceğiniz öneri sisteminde kullanılabilir.
