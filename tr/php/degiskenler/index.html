Sözdizimi #

  • $değişken = ‘değer’; // Genel değişken ata
  • $nesne->özellik = ‘değer’; // Bir nesne özelliği ata
  • SınıfAdı::$özellik = ‘değer’; // Statik bir sınıf özelliği atayın
  • $dizi[0] = ‘değer’; // Bir dizinin indeksine bir değer atayın
  • $dizi[] = ‘değer’; // Bir dizinin sonundaki bir öğeyi itin
  • $dizi[‘anahtar’] = ‘değer’; // Bir dizi değeri ata
  • echo $değişken; // Değişken bir değeri yankı (yazdır)
  • bazı_fonksiyon($değişken); // Değişkeni fonksiyon parametresi olarak kullan
  • unset($değişken); // Bir değişkenin ayarını kaldır
  • $$değişken = ‘değer’; // Değişken bir değişkene atama
  • isset($değişken); // Bir değişkenin ayarlanıp ayarlanmadığını kontrol edin
  • boş($değişken); // Bir değişkenin boş olup olmadığını kontrol edin

Tip kontrolü #

Değişkenler ve türlerle ilgili bazı belgeler PHP’nin kullanmadığından bahseder. statik yazma. Bu doğrudur, ancak PHP, işlev/yöntem parametreleri ve dönüş değerleri söz konusu olduğunda (özellikle PHP 7 ile) bazı tür kontrolleri yapar.

PHP 7’de tip ipucunu aşağıdaki gibi kullanarak parametre ve değer tipi kontrolünü zorunlu kılabilirsiniz:

<?php

/**
 * Juggle numbers and return true if juggling was
 * a great success.
 */
function numberJuggling(int $a, int $b) : bool
{
    $sum = $a + $b;

    return $sum % 2 === 0;
}

Not: PHP’nin tamsayılar ve booleler için gettype() değeri sırasıyla ‘integer’ ve ‘boolean’dır. Ancak bu tür değişkenler için tip ipucu için ‘int’ ve ‘bool’ kullanmanız gerekir. Aksi takdirde PHP size bir sözdizimi hatası vermez, ancak ‘integer’ ve ‘boolean’ *classes’ın iletilmesini bekler.

Yukarıdaki örnek, sayısal olmayan değerin “$a” veya “$b” parametresi olarak verilmesi ve işlev “doğru” veya “yanlış” dışında bir değer döndürmesi durumunda hata verir. Yukarıdaki örnek “gevşek"tir, çünkü “$a” veya “$b"ye bir kayan nokta değeri verebilirsiniz. Katı türleri zorlamak istiyorsanız, yani yalnızca tamsayılar girebilirsiniz ve kayan değerler giremezsiniz, aşağıdakileri PHP dosyanızın en başına ekleyin:

<?php
declare('strict_types=1');

PHP 7’den önce işlevler ve yöntemler, aşağıdaki türler için tür ipuçlarına izin veriyordu:

  • “çağrılabilir” (çağrılabilir bir işlev veya yöntem)
  • “dizi” (diğer dizileri de içerebilen herhangi bir dizi türü)
  • Arayüzler (Tam Nitelikli Sınıf Adı veya FQDN)
  • Sınıflar (FQDN)

Ayrıca bakınız: https://www.wikiod.com/tr/php/bir-degiskenin-degerini-ckarma

Bir Değişkene Ada Göre Dinamik Olarak Erişme (Değişken değişkenleri) #

Değişkenlere dinamik değişken isimleri ile erişilebilir. Bir değişkenin adı başka bir değişkende saklanabilir, bu da ona dinamik olarak erişilmesini sağlar. Bu tür değişkenler değişken değişkenler olarak bilinir.

Bir değişkeni değişken değişkene dönüştürmek için değişkeninizin önüne fazladan bir $ koyarsınız.

$variableName = 'foo';
$foo = 'bar';

// The following are all equivalent, and all output "bar":
echo $foo;
echo ${$variableName};
echo $$variableName;

//similarly,
$variableName  = 'foo';
$$variableName = 'bar';

// The following statements will also output 'bar'
echo $foo; 
echo $$variableName; 
echo ${$variableName};

Değişken değişkenler, işlev/yöntem çağrılarını eşlemek için kullanışlıdır:

function add($a, $b) {
    return $a + $b;
}

$funcName = 'add';

echo $funcName(1, 2); // outputs 3

Bu özellikle PHP sınıflarında yardımcı olur:

class myClass {
    public function __construct() {
        $functionName = 'doSomething';
        $this->$functionName('Hello World');
    }

    private function doSomething($string) {
        echo $string; // Outputs "Hello World"
    }
}

“{}” arasına “$değişkenAdı” koymak mümkündür, ancak gerekli değildir:

${$variableName} = $value;

Aşağıdaki örnekler hem eşdeğerdir hem de çıktı “baz"dır:

$fooBar = 'baz';
$varPrefix = 'foo';

echo $fooBar;               // Outputs "baz"
echo ${$varPrefix . 'Bar'}; // Also outputs "baz"

{} kullanmak yalnızca değişkenin adının kendisi bir ifade olduğunda zorunludur, örneğin:

${$variableNamePart1 . $variableNamePart2} = $value;

Yine de her zaman {} kullanılması önerilir, çünkü daha okunabilirdir.

Bunu yapmanız tavsiye edilmese de, bu davranışı zincirlemek mümkündür:

$$$$$$$$DoNotTryThisAtHomeKids = $value;

Değişken değişkenlerin aşırı kullanımının birçok geliştirici tarafından kötü bir uygulama olarak kabul edildiğini belirtmek önemlidir. Modern IDE’ler tarafından statik analiz için çok uygun olmadıklarından, birçok değişken değişkene (veya dinamik yöntem çağrılarına) sahip büyük kod tabanlarının bakımı hızla zorlaşabilir.

PHP5 ve PHP7 arasındaki farklar #

Her zaman {} veya () kullanmanın bir başka nedeni de, PHP5 ve PHP7’nin dinamik değişkenlerle uğraşırken biraz farklı bir yönteminin olmasıdır, bu da bazı durumlarda farklı bir sonuçla sonuçlanır.

PHP7’de dinamik değişkenler, özellikler ve yöntemler, PHP5’teki özel durumların karışımının aksine, artık kesinlikle soldan sağa sırayla değerlendirilecektir. Aşağıdaki örnekler, değerlendirme sırasının nasıl değiştiğini göstermektedir.

1. Durum : $$foo['bar']['baz'] #

  • PHP5 yorumu: ${$foo['bar']['baz']}
  • PHP7 yorumu: ($$foo)['bar']['baz']

2. Durum : $foo->$bar['baz'] #

  • PHP5 yorumu: $foo->{$bar['baz']}
  • PHP7 yorumu: ($foo->$bar)['baz']

Durum 3: $foo->$bar['baz']() #

  • PHP5 yorumu: $foo->{$bar['baz']}()
  • PHP7 yorumu: ($foo->$bar)['baz']()

Durum 4 : Foo::$bar['baz']() #

  • PHP5 yorumu: Foo::{$bar['baz']}()
  • PHP7 yorumu: (Foo::$bar)['baz']()

Veri tipleri #

Farklı amaçlar için farklı veri türleri vardır. PHP’nin açık tür tanımları yoktur, ancak bir değişkenin türü, atanan değerin türüne veya dönüştürüldüğü türe göre belirlenir. Bu, türler hakkında kısa bir genel bakıştır, ayrıntılı belgeler ve örnekler için PHP türleri konusuna bakın.

PHP’de şu veri türleri vardır: null, boolean, tamsayı, kayan nokta, dize, nesne, kaynak ve dizi.

Hükümsüz #

Null herhangi bir değişkene atanabilir. Değeri olmayan bir değişkeni temsil eder.

$foo = null;

Bu, değişkeni geçersiz kılar ve çağrılırsa değeri tanımsız veya geçersiz olur. Değişken bellekten temizlenir ve çöp toplayıcı tarafından silinir.

Boole #

Bu, yalnızca iki olası değere sahip en basit türdür.

$foo = true;
$bar = false;

Boolean’lar, kod akışını kontrol etmek için kullanılabilir.

$foo = true;

if ($foo) {
    echo "true";
} else {
    echo "false";
}

Tamsayı #

Bir tam sayı, pozitif veya negatif bir tam sayıdır. Herhangi bir sayı tabanı ile kullanılabilir. Bir tamsayının boyutu platforma bağlıdır. PHP, işaretsiz tam sayıları desteklemez.

$foo = -3;  // negative
$foo = 0;   // zero (can also be null or false (as boolean)
$foo = 123; // positive decimal
$bar = 0123; // octal = 83 decimal
$bar = 0xAB; // hexadecimal = 171 decimal
$bar = 0b1010; // binary = 10 decimal
var_dump(0123, 0xAB, 0b1010); // output: int(83) int(171) int(10)

Batmadan yüzmek #

Kayan nokta sayıları, “çiftler” veya basitçe “kayan noktalar” olarak adlandırılan ondalık sayılardır.

$foo = 1.23;
$foo = 10.0;
$bar = -INF;
$bar = NAN;

Dizi #

Bir dizi, bir değerler listesi gibidir. Bir dizinin en basit biçimi tamsayı ile indekslenir ve ilk eleman indeks 0’da olacak şekilde indeks tarafından sıralanır.

$foo = array(1, 2, 3); // An array of integers
$bar = ["A", true, 123 => 5]; // Short array syntax, PHP 5.4+

echo $bar[0];    // Returns "A"
echo $bar[1];    // Returns true
echo $bar[123];  // Returns 5
echo $bar[1234]; // Returns null

Diziler ayrıca bir tamsayı dizini dışında bir anahtarı bir değerle ilişkilendirebilir. PHP’de tüm diziler, sahne arkasındaki ilişkisel dizilerdir, ancak bir ‘ilişkisel dizi’den açıkça bahsettiğimizde, genellikle tamsayı olmayan bir veya daha fazla anahtar içeren bir diziyi kastederiz.

$array = array();
$array["foo"] = "bar";
$array["baz"] = "quux";
$array[42] = "hello";
echo $array["foo"]; // Outputs "bar"
echo $array["bar"]; // Outputs "quux"
echo $array[42]; // Outputs "hello"

Sicim #

Bir dize, bir dizi karakter gibidir.

$foo = "bar";

Bir dizi gibi, bir dize de kendi karakterlerini döndürmek için dizine alınabilir:

$foo = "bar";
echo $foo[0]; // Prints 'b', the first character of the string in $foo.

Nesne #

Bir nesne, bir sınıfın bir örneğidir. Değişkenlerine ve yöntemlerine -> operatörü ile erişilebilir.

$foo = new stdClass(); // create new object of class stdClass, which a predefined, empty class
$foo->bar = "baz";
echo $foo->bar; // Outputs "baz"
// Or we can cast an array to an object:
$quux = (object) ["foo" => "bar"];
echo $quux->foo; // This outputs "bar".

Kaynak #

Kaynak değişkenleri, açılan dosyalar, veritabanı bağlantıları, akışlar, resim tuvali alanları ve benzerleri için özel tutamaçları tutar ([kılavuzda](https://secure.php.net/manual/en/language.types.resource içinde belirtildiği gibi) .php#language.types.resource.casting)).

$fp = fopen('file.ext', 'r'); // fopen() is the function to open a file on disk as a resource.
var_dump($fp); // output: resource(2) of type (stream)

Bir değişkenin türünü dizge olarak almak için gettype() işlevini kullanın:

echo gettype(1); // outputs "integer"
echo gettype(true); // "boolean"

Global değişken en iyi uygulamaları #

Bu sorunu aşağıdaki sözde kodla gösterebiliriz.

function foo() {
    global $bob;
    $bob->doSomething();
}

Buradaki ilk sorunuz bariz bir soru

$bob nereden geldi?

Kafan mı karıştı? İyi. Globallerin neden kafa karıştırıcı olduğunu ve kötü bir uygulama olarak değerlendirildiğini öğrendiniz.

Bu gerçek bir program olsaydı, bir sonraki eğlenceniz “$bob"un tüm örneklerini takip etmek ve doğru olanı bulmanızı ummaktır (eğer her yerde “$bob” kullanılıyorsa bu daha da kötüleşir). Daha da kötüsü, bir başkası gidip `$bob’u tanımlarsa (veya o değişkeni unuttuysanız ve yeniden kullandınız) kodunuz bozulabilir (yukarıdaki kod örneğinde, yanlış nesneye sahip olmak veya hiç nesne olmaması önemli bir hataya neden olur).

Hemen hemen tüm PHP programları include('file.php'); gibi bir kod kullandığından, bunun gibi kodu koruma işiniz, ne kadar çok dosya eklerseniz o kadar katlanarak zorlaşır.

Ayrıca bu, uygulamalarınızı test etme görevini çok zorlaştırır. Veritabanı bağlantınızı tutmak için global bir değişken kullandığınızı varsayalım:

$dbConnector = new DBConnector(...);

function doSomething() {
    global $dbConnector;
    $dbConnector->execute("...");
}

Bu işlevi birim test etmek için, global “$dbConnector” değişkenini geçersiz kılmanız, testleri çalıştırmanız ve ardından hataya çok açık olan orijinal değerine sıfırlamanız gerekir:

/**
 * @test
 */
function testSomething() {
     global $dbConnector;

     $bkp = $dbConnector; // Make backup
     $dbConnector = Mock::create('DBConnector'); // Override

     assertTrue(foo());

     $dbConnector = $bkp; // Restore
}

Küresellerden nasıl kaçınırız?

Globallerden kaçınmanın en iyi yolu Bağımlılık Enjeksiyonu adlı bir felsefedir. İhtiyacımız olan araçları işleve veya sınıfa aktardığımız yer burasıdır.

function foo(\Bar $bob) {
    $bob->doSomething();
}

Bunu anlamak ve sürdürmek çok daha kolaydır. “$bob"un nerede kurulduğuna dair bir tahmin yok çünkü arayan bunu bilmekle sorumlu (bilmemiz gerekeni bize aktarıyor). Daha da iyisi, geçirilenleri kısıtlamak için tür bildirimlerini kullanabiliriz.

Dolayısıyla, ‘$bob’un ya ‘Bar’ sınıfının bir örneği ya da ‘Bar’ın bir alt öğesinin örneği olduğunu biliyoruz, yani o sınıfın yöntemlerini kullanabileceğimizi biliyoruz. Standart bir otomatik yükleyici ile birlikte (PHP 5.3’ten beri mevcuttur), artık Barın tanımlandığı yere gidebiliriz. PHP 7.0 veya sonraki sürümler, skaler türleri de kullanabileceğiniz genişletilmiş tip bildirimleri içerir (‘int’ veya ‘string’ gibi).

PHP’deki süper globaller, her zaman kullanılabilir olan önceden tanımlanmış değişkenlerdir ve komut dosyası boyunca herhangi bir kapsamdan erişilebilir.

Global $değişkeni yapmaya gerek yoktur; bunlara işlevler/yöntemler, sınıflar veya dosyalar içinde erişmek için.

Bu PHP süper küresel değişkenleri aşağıda listelenmiştir:

  • $KÜRESEL
  • $_SERVER

  • $_POST
  • $_GET
  • $_FILES
  • $_ENV
  • $_COOKIE
  • $_SESSION

Başlatılmamış değişkenlerin varsayılan değerleri #

PHP’de gerekli olmasa da, değişkenleri başlatmak çok iyi bir uygulamadır. Başlatılmamış değişkenler, kullanıldıkları bağlama bağlı olarak türlerinin varsayılan değerine sahiptir:

Ayarlanmamış VE başvurulmamış

var_dump($unset_var); // outputs NULL

Boole

echo($unset_bool ? "true\n" : "false\n"); // outputs 'false'

Sicim

$unset_str .= 'abc';
var_dump($unset_str); // outputs 'string(3) "abc"'

Tamsayı

$unset_int += 25; // 0 + 25 => 25
var_dump($unset_int); // outputs 'int(25)'

Yüzer/çift

$unset_float += 1.25;
var_dump($unset_float); // outputs 'float(1.25)'

Dizi

$unset_arr[3] = "def";
var_dump($unset_arr); //  outputs array(1) {  [3]=>  string(3) "def" }

Nesne

$unset_obj->foo = 'bar';
var_dump($unset_obj); // Outputs: object(stdClass)#1 (1) {  ["foo"]=>  string(3) "bar" }

Başlatılmamış bir değişkenin varsayılan değerine güvenmek, aynı değişken adını kullanan bir dosyanın diğerine dahil edilmesi durumunda sorunludur.

Değişken Değer Doğruluğu ve Özdeş Operatör #

PHP’de, değişken değerlerinin ilişkili bir “doğruluğu” vardır, bu nedenle boolean olmayan değerler bile “doğru” veya “yanlış"a eşit olacaktır. Bu, herhangi bir değişkenin koşullu bir blokta kullanılmasına izin verir, örn.

if ($var == true) { /* explicit version */ }
if ($var) { /* $var == true is implicit */ }

Farklı değişken değerleri türleri için bazı temel kurallar şunlardır:

  • Sıfır olmayan uzunluktaki dizeler, "” "” gibi yalnızca boşluk içeren dizeler de dahil olmak üzere “true” değerine eşittir.

  • Boş dizeler '', false değerine eşittir.

    $var = ‘’; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

    $var = ’ ‘; $var_is_true = ($var == true); // true $var_is_false = ($var == false); // false

  • Tamsayılar sıfırdan farklıysa “doğru"ya, sıfır ise “yanlış"a eşittir.

    $var = -1; $var_is_true = ($var == true); // true $var = 99; $var_is_true = ($var == true); // true $var = 0; $var_is_true = ($var == true); // false

  • null, false değerine eşittir

    $var = null; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

  • Boş dizeler '' ve dize sıfır '0', false değerine eşittir.

    $var = ‘’; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

    $var = ‘0’; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

  • Kayan nokta değerleri, sıfırdan farklıysa “doğru"ya, sıfır değerleri ise “yanlış"a eşittir.

    • NAN (PHP’s Not-a-Number) equates to true, i.e. NAN == true is true. This is because NAN is a nonzero floating-point value.
    • Zero-values include both +0 and -0 as defined by IEEE 754. PHP does not distinguish between +0 and -0 in its double-precision floating-point, i.e. floatval('0') == floatval('-0') is true.
      • In fact, floatval('0') === floatval('-0').
      • Additionally, both floatval('0') == false and floatval('-0') == false.

    $var = NAN; $var_is_true = ($var == true); // true $var_is_false = ($var == false); // false

    $var = floatval(’-0’); $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

    $var = floatval(‘0’) == floatval(’-0’); $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

KİMLİK OPERATÖR

Karşılaştırma Operatörleri için PHP Dokümantasyonunda, Özdeş Operatör === vardır. Bu operatör, bir değişkenin bir referans değeriyle özdeş olup olmadığını kontrol etmek için kullanılabilir:

$var = null;
$var_is_null = $var === null; // true
$var_is_true = $var === true; // false
$var_is_false = $var === false; // false

Karşılık gelen bir aynı olmayan operatörüne sahiptir !==:

$var = null;
$var_is_null = $var !== null; // false
$var_is_true = $var !== true; // true
$var_is_false = $var !== false; // true

Aynı operatör, “is_null()” gibi dil işlevlerine alternatif olarak kullanılabilir.

strpos() İLE KULLANIM ÖRNEKLERİ

‘strpos($haystack, $needle)’ dil işlevi, ‘$haystack’ içinde ‘$needle’ın meydana geldiği dizini veya bunun gerçekleşip gerçekleşmediğini bulmak için kullanılır. strpos() işlevi büyük/küçük harf duyarlıdır; büyük/küçük harfe duyarlı olmayan bul, ihtiyacınız olan şeyse stripos($haystack, $needle) ile gidebilirsiniz

‘strpos’ & ‘stripos’ işlevi ayrıca, belirtilirse, aramanın dizenin başlangıcından itibaren sayılan bu sayıda karakteri başlatacak olan üçüncü ‘offset’ (int) parametresini de içerir. strrpos ve strripos’tan farklı olarak, ofset negatif olamaz

İşlev şunu döndürebilir:

  • “$haystack"ın başında “$iğne” bulunursa “0”;
  • ‘$haystack’ içinde başlangıçtan başka bir yerde ‘$needle’ bulunursa dizini belirten sıfır olmayan bir tam sayı;
  • ve “$iğne”, “$haystack” içinde herhangi bir yerde bulunmuyorsa “false” değeri.

PHP’de hem ‘0’ hem de ‘yanlış’ doğruluk ‘yanlış’ olduğundan, ancak ‘strpos()’ için farklı durumları temsil ettiğinden, bunları birbirinden ayırmak ve tam olarak ‘yanlış’ı aramak için aynı operatörü ‘===’ kullanmak önemlidir. ’ ve yalnızca ‘yanlış’a eşit olan bir değer değil.

$idx = substr($haystack, $needle);
if ($idx === false) 
{
    // logic for when $needle not found in $haystack
} 
else
{
    // logic for when $needle found in $haystack
}

Alternatif olarak, aynı değil operatörünü kullanarak:

$idx = substr($haystack, $needle);
if ($idx !== false) 
{
    // logic for when $needle found in $haystack
} 
else
{
    // logic for when $needle not found in $haystack
}

Tanımlanmış tüm değişkenleri alma #

get_defined_vars(), işlevin çağrıldığı kapsamda tanımlanan değişkenlerin tüm adlarını ve değerlerini içeren bir dizi döndürür. Verileri yazdırmak istiyorsanız, print_r veya var_dump gibi insan tarafından okunabilir verilerin çıktısını almak için standart işlevleri kullanabilirsiniz.

var_dump(get_defined_vars());

Not: Bu işlev genellikle yalnızca 4 süper küresel döndürür: $_GET,$_POST,$_COOKIE,$_FILES. Diğer süper küreseller, yalnızca kodun bir yerinde kullanılmışlarsa döndürülür. Bunun nedeni, varsayılan olarak etkin olan auto_globals_jit yönergesidir. Etkinleştirildiğinde, “$_SERVER” ve “$_ENV” değişkenleri, komut dosyası başladığında değil, ilk kullanıldıklarında (Tam Zamanında) oluşturulur. Bu değişkenler bir komut dosyası içinde kullanılmazsa, bu yönergenin açık olması performans artışıyla sonuçlanacaktır.
