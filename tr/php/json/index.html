JSON (JavaScript Object Notation), nesneleri düz metne serileştirmenin platformdan ve dilden bağımsız bir yoludur. Web’de ve PHP’de sıklıkla kullanıldığı için, PHP’de JSON ile çalışmak için bir temel uzantı vardır.

Sözdizimi #

  • string json_encode ( karışık $değer [, int $seçenekler = 0 [, int $derinlik = 512 ]] )
  • karışık json_decode ( string $json [, bool $assoc = false [, int $derinlik = 512 [, int $options = 0 ]]] )

Parametreler #

parametre Ayrıntılar
json_encode -
değer Kodlanan değer. Kaynak dışında herhangi bir tür olabilir. Tüm dize verileri UTF-8 kodlu olmalıdır.
seçenekler JSON_HEX_QUOT, JSON_HEX_TAG, JSON_HEX_AMP, JSON_HEX_APOS, JSON_NUMERIC_CHECK, JSON_PRETTY_PRINT, JSON_UNESCAPED_SLASHES, JSON_FORCE_APJECT, JSON_PORCE_OBJECT, JSON_PRESTIC, JSON_PRESTIC,JSON_PRETY_PRETY_PRUT. Bu sabitlerin davranışı JSON sabitleri sayfasında açıklanmıştır.
derinlik Maksimum derinliği ayarlayın. Sıfırdan büyük olmalıdır.
json_decode -
json Json dizesinin kodu çözülüyor. Bu işlev yalnızca UTF-8 kodlu dizelerle çalışır.
assoc İşlev nesneler yerine ilişkisel dizi döndürmeli.
seçenekler JSON kod çözme seçeneklerinin bit maskesi. Şu anda yalnızca JSON_BIGINT_AS_STRING desteklenmektedir (varsayılan, büyük tamsayıları kayan nokta olarak yayınlamaktır)
  • json_decode geçersiz JSON’un işlenmesi çok düzensizdir ve kod çözmenin başarılı olup olmadığını güvenilir bir şekilde belirlemek çok zordur, json_decode geçersiz girdi için null değerini döndürür, ancak null ayrıca JSON’un kodunu çözmesi için tamamen geçerli bir nesnedir. Bu tür sorunları önlemek için, her kullandığınızda her zaman json_last_error‘u aramalısınız.

JSON dizesinin kodunu çözme #

json_decode() işlevi, ilk parametresi olarak JSON kodlu bir dize alır ve onu bir PHP değişkenine ayrıştırır.

Normalde, JSON nesnesindeki en üst düzey öğe bir sözlükse, “json_decode()” bir \stdClass nesnesi veya JSON nesnesi bir diziyse dizine alınmış dizi döndürür. . Ayrıca, basit dizeler, ““true””, ““false”’ ve ‘“null”’ gibi belirli skaler değerler için skaler değerler veya ‘NULL’ döndürür. Ayrıca herhangi bir hatada ‘NULL’ döndürür.

// Returns an object (The top level item in the JSON string is a JSON dictionary)
$json_string = '{"name": "Jeff", "age": 20, "active": true, "colors": ["red", "blue"]}';
$object = json_decode($json_string);
printf('Hello %s, You are %s years old.', $object->name, $object->age);
#> Hello Jeff, You are 20 years old.

// Returns an array (The top level item in the JSON string is a JSON array)
$json_string = '["Jeff", 20, true, ["red", "blue"]]';
$array = json_decode($json_string);
printf('Hello %s, You are %s years old.', $array[0], $array[1]);

Yukarıda kodunu çözdüğümüz nesnedeki her bir özelliğin türlerini ve değerlerini görüntülemek için var_dump() kullanın.

// Dump our above $object to view how it was decoded
var_dump($object);

Çıktı (değişken türlerini not edin):

class stdClass#2 (4) {
 ["name"] => string(4) "Jeff"
 ["age"] => int(20)
 ["active"] => bool(true)
 ["colors"] =>
   array(2) {
     [0] => string(3) "red"
     [1] => string(4) "blue"
   }
}

Not: JSON’daki types değişkeni PHP eşdeğerine dönüştürülmüştür.

Bir nesne döndürmek yerine JSON nesneleri için bir ilişkisel dizi döndürmek için, “true” değerini ikinci parametre olarak “json_decode()” öğesine iletin.

$json_string = '{"name": "Jeff", "age": 20, "active": true, "colors": ["red", "blue"]}';
$array = json_decode($json_string, true); // Note the second parameter
var_dump($array);

Çıktı (dizi ilişkisel yapısını not edin):

array(4) {
  ["name"] => string(4) "Jeff"
  ["age"] => int(20)
  ["active"] => bool(true)
  ["colors"] =>
  array(2) {
    [0] => string(3) "red"
    [1] => string(4) "blue"
  }
}

Döndürülecek değişken bir nesne değilse, ikinci parametrenin ($assoc) hiçbir etkisi yoktur.

Not: $assoc parametresini kullanırsanız, boş dizi ve boş nesne arasındaki farkı kaybedersiniz. Bu, kodu çözülmüş çıktınızda tekrar “json_encode()” çalıştırmanın farklı bir JSON yapısıyla sonuçlanacağı anlamına gelir.

JSON dizesi özyinelemede 512’den fazla öğeye (5.2.3’ten eski sürümlerde 20 öğe veya 5.2.3 sürümünde 128 öğe) sahipse, “json_decode()” işlevi “NULL” döndürür. 5.3 veya sonraki sürümlerde, bu sınır, aşağıda tartışıldığı gibi üçüncü parametre ($depth) kullanılarak kontrol edilebilir.

Kılavuza göre:

PHP, orijinal » RFC 4627‘de belirtildiği gibi bir JSON üst kümesini uygular - ayrıca skaler türleri ve NULL’u kodlar ve kodunu çözer. RFC 4627, bu değerleri yalnızca bir dizi veya nesne içinde iç içe olduklarında destekler. Bu üst küme, daha yeni » RFC 7159 (RFC 4627’nin yerini almayı amaçlar) ve » ECMA-404’teki genişletilmiş “JSON metni” tanımıyla tutarlı olsa da, bu, tek bir skaler değeri kodlarken kesinlikle RFC 4627’ye bağlı olan eski JSON ayrıştırıcıları.

Bu, örneğin basit bir dizenin PHP’de geçerli bir JSON nesnesi olarak kabul edileceği anlamına gelir:

$json = json_decode('"some string"', true);
var_dump($json, json_last_error_msg());

Çıktı:

string(11) "some string"
string(8) "No error"

Ancak bir dizide veya nesnede olmayan basit dizeler RFC 4627 standardının parçası değildir. Sonuç olarak, JSLint, JSON Formatter & Validator (RFC 4627 modunda) gibi çevrimiçi denetleyiciler size bir hata verecektir.

Özyineleme derinliği için üçüncü bir “$depth” parametresi vardır (varsayılan değer “512"dir), bu, kodu çözülecek orijinal nesnenin içindeki iç içe nesnelerin miktarı anlamına gelir.

Dördüncü bir “$options” parametresi var. Şu anda yalnızca bir değeri kabul ediyor, “JSON_BIGINT_AS_STRING”. Varsayılan davranış (bu seçeneği devre dışı bırakır), büyük tamsayıları dizeler yerine kayan noktalara çevirmektir.

true, false ve null değişmezlerin geçersiz küçük harfli olmayan değişkenleri artık geçerli girdi olarak kabul edilmiyor.

Yani bu örnek:

var_dump(json_decode('tRue'), json_last_error_msg());
var_dump(json_decode('tRUe'), json_last_error_msg());
var_dump(json_decode('tRUE'), json_last_error_msg());
var_dump(json_decode('TRUe'), json_last_error_msg());
var_dump(json_decode('TRUE'), json_last_error_msg());
var_dump(json_decode('true'), json_last_error_msg());

bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"

Ve sonra:

NULL
string(12) "Syntax error"
NULL

string(12) "Syntax error"
NULL
string(12) "Syntax error"
NULL
string(12) "Syntax error"
bool(true)
string(8) "No error"

Benzer davranış “yanlış” ve “boş” için de ortaya çıkar.

Dize dönüştürülemiyorsa, “json_decode()” öğesinin “NULL” döndüreceğini unutmayın.

$json = "{'name': 'Jeff', 'age': 20 }" ;  // invalid json 

$person = json_decode($json);
echo $person->name;    //  Notice: Trying to get property of non-object: returns null
echo json_last_error();     
#  4 (JSON_ERROR_SYNTAX)
echo json_last_error_msg(); 
#  unexpected character

Hataları saptamak için yalnızca “NULL” olan dönüş değerine güvenmek güvenli değildir. Örneğin, JSON dizesi “boş” dışında hiçbir şey içermiyorsa, “json_decode()”, hiçbir hata oluşmasa bile “boş” döndürür.

JSON dizesini kodlama #

json_encode işlevi, bir PHP dizisini (veya PHP 5.4’ten beri, JsonSerializable arabirimini uygulayan bir nesneyi) JSON kodlu bir dizeye dönüştürür. Başarı durumunda JSON kodlu bir dize veya başarısızlık durumunda FALSE döndürür.

$array = [
    'name' => 'Jeff',
    'age' => 20,
    'active' => true,
    'colors' => ['red', 'blue'],
    'values' => [0=>'foo', 3=>'bar'],
];

Kodlama sırasında, PHP veri türleri dize, tamsayı ve boole, JSON eşdeğerlerine dönüştürülür. İlişkili diziler JSON nesneleri olarak kodlanır ve - varsayılan bağımsız değişkenlerle çağrıldıklarında - dizine alınmış diziler JSON dizileri olarak kodlanır. (Dizi anahtarları 0’dan başlayan sürekli bir sayısal dizi olmadığı sürece, bu durumda dizi bir JSON nesnesi olarak kodlanacaktır.)

echo json_encode($array);

Çıktı:

{"name":"Jeff","age":20,"active":true,"colors":["red","blue"],"values":{"0":"foo","3":"bar"}}

Argümanlar #

PHP 5.3’ten beri, json_encode’un ikinci argümanı, aşağıdakilerden biri veya daha fazlası olabilen bir bit maskesidir.

Herhangi bir bit maskesinde olduğu gibi, ikili VEYA operatörü | ile birleştirilebilirler.

JSON_FORCE_OBJECT #

Dizi yerine nesne oluşturmaya zorlar

$array = ['Joel', 23, true, ['red', 'blue']];
echo json_encode($array);
echo json_encode($array, JSON_FORCE_OBJECT);

Çıktı:

["Joel",23,true,["red","blue"]]
{"0":"Joel","1":23,"2":true,"3":{"0":"red","1":"blue"}}

JSON_HEX_TAG, JSON_HEX_AMP, JSON_HEX_APOS, [JSON_HEX_QUOT][json-hex -alıntı] #

Kodlama sırasında aşağıdaki dönüşümleri sağlar:

Sabit Giriş Çıktı
JSON_HEX_TAG < \u003C
JSON_HEX_TAG > `\u003E’
JSON_HEX_AMP & \u0026
JSON_HEX_APOS ’’’ \u0027
JSON_HEX_QUOT " \u0022 
$array = ["tag"=>"<>", "amp"=>"&", "apos"=>"'", "quot"=>"\""];
echo json_encode($array);
echo json_encode($array, JSON_HEX_TAG | JSON_HEX_AMP | JSON_HEX_APOS | JSON_HEX_QUOT);

Çıktı:

{"tag":"<>","amp":"&","apos":"'","quot":"\""}
{"tag":"\u003C\u003E","amp":"\u0026","apos":"\u0027","quot":"\u0022"}
son sürüm

JSON_NUMERIC_CHECK #

Sayısal dizelerin tamsayılara dönüştürülmesini sağlar.

$array = ['23452', 23452];
echo json_encode($array);
echo json_encode($array, JSON_NUMERIC_CHECK);

Çıktı:

["23452",23452]    
[23452,23452]
son sürüm

JSON_PRETTY_PRINT #

JSON’u kolayca okunabilir hale getirir

$array = ['a' => 1, 'b' => 2, 'c' => 3, 'd' => 4];
echo json_encode($array);
echo json_encode($array, JSON_PRETTY_PRINT);

Çıktı:

{"a":1,"b":2,"c":3,"d":4}
{
    "a": 1,
    "b": 2,
    "c": 3,
    "d": 4
}

JSON_UNESCAPED_SLASHES #

Çıktıda çıkışsız / eğik çizgi içerir

$array = ['filename' => 'example.txt', 'path' => '/full/path/to/file/'];
echo json_encode($array);
echo json_encode($array, JSON_UNESCAPED_SLASHES);

Çıktı:

{"filename":"example.txt","path":"\/full\/path\/to\/file"}
{"filename":"example.txt","path":"/full/path/to/file"}

JSON_UNESCAPED_UNICODE #

Çıktıda “\u” kodlu dizeler yerine UTF8 kodlu karakterler içerir

$blues = ["english"=>"blue", "norwegian"=>"blå", "german"=>"blau"];
echo json_encode($blues);
echo json_encode($blues, JSON_UNESCAPED_UNICODE);

Çıktı:

{"english":"blue","norwegian":"bl\u00e5","german":"blau"}
{"english":"blue","norwegian":"blå","german":"blau"}
son sürüm

[JSON_PARTIAL_OUTPUT_ON_ERROR][json-kısmi-çıktı-hata üzerine] #

Bazı kodlanamayan değerlerle karşılaşıldığında kodlamanın devam etmesine izin verir.

$fp = fopen("foo.txt", "r");
$array = ["file"=>$fp, "name"=>"foo.txt"];
echo json_encode($array); // no output
echo json_encode($array, JSON_PARTIAL_OUTPUT_ON_ERROR);

Çıktı:

{"file":null,"name":"foo.txt"}
son sürüm

JSON_PRESERVE_ZERO_FRACTION #

Floatların her zaman float olarak kodlanmasını sağlar.

$array = [5.0, 5.5];
echo json_encode($array);
echo json_encode($array, JSON_PRESERVE_ZERO_FRACTION);

Çıktı:

[5,5.5]
[5.0,5.5]
son sürüm

JSON_UNESCAPED_LINE_TERMINATORS #

“JSON_UNESCAPED_UNICODE” ile kullanıldığında, eski PHP sürümlerinin davranışına geri döner ve U+2028 SATIR AYIRICI ve U+2029 PARAGRAPH AYIRICI karakterlerinden kaçmaz. JSON’da geçerli olmasına rağmen, bu karakterler JavaScript’te geçerli değildir, bu nedenle ‘JSON_UNESCAPED_UNICODE’un varsayılan davranışı 7.1 sürümünde değiştirildi.

$array = ["line"=>"\xe2\x80\xa8", "paragraph"=>"\xe2\x80\xa9"];
echo json_encode($array, JSON_UNESCAPED_UNICODE);
echo json_encode($array, JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_LINE_TERMINATORS);

Çıktı:

{"line":"\u2028","paragraph":"\u2029"}
{"line":" ","paragraph":" "}
son sürüm

JSON hatalarında hata ayıklama #

“json_encode” veya “json_decode” sağlanan dizeyi ayrıştırmadığında “false” döndürür. PHP’nin kendisi bu olduğunda herhangi bir hata veya uyarı vermez, json_last_error() ve json_last_error_msg() işlevlerini kullanmak kullanıcıdadır. uygulamanızda bir hata oluşup oluşmadığını kontrol etmek ve buna göre hareket etmek (hata ayıklayın, bir hata mesajı gösterin, vb.).

Aşağıdaki örnek, JSON ile çalışırken yaygın olarak görülen bir hatayı, bir JSON dizesinin kodunun çözülmemesini/kodlanmamasını (örneğin, hatalı bir UTF-8 kodlu dizenin iletilmesi nedeniyle) gösterir.

// An incorrectly formed JSON string
$jsonString = json_encode("{'Bad JSON':\xB1\x31}");

if (json_last_error() != JSON_ERROR_NONE) {
    printf("JSON Error: %s", json_last_error_msg());
}

#> JSON Error: Malformed UTF-8 characters, possibly incorrectly encoded

json_last_error_msg #

json_last_error_msg(), bir dizeyi kodlamaya/kodunu çözmeye çalışırken meydana gelen son hatanın insan tarafından okunabilir bir mesajını döndürür.

  • Bu işlev, hiçbir hata oluşmasa bile her zaman bir dize döndürür. Varsayılan hata olmayan dize “Hata Yok” şeklindedir.
  • Başka bir (bilinmeyen) hata oluşursa ‘false’ döndürür
  • Bunu döngülerde kullanırken dikkatli olun, çünkü json_last_error_msg her yinelemede geçersiz kılınacaktır.

Bu işlevi yalnızca mesajı görüntülemek için kullanmalısınız, kontrol ifadelerinde test etmek için değil.

// Don't do this:
if (json_last_error_msg()){} // always true (it's a string)
if (json_last_error_msg() != "No Error"){} // Bad practice

// Do this: (test the integer against one of the pre-defined constants)
if (json_last_error() != JSON_ERROR_NONE) {
    // Use json_last_error_msg to display the message only, (not test against it)
    printf("JSON Error: %s", json_last_error_msg());
}

Bu işlev PHP 5.5’ten önce mevcut değildir. İşte bir polyfill uygulaması:

if (!function_exists('json_last_error_msg')) {
    function json_last_error_msg() {
        static $ERRORS = array(
            JSON_ERROR_NONE => 'No error',
            JSON_ERROR_DEPTH => 'Maximum stack depth exceeded',
            JSON_ERROR_STATE_MISMATCH => 'State mismatch (invalid or malformed JSON)',
            JSON_ERROR_CTRL_CHAR => 'Control character error, possibly incorrectly encoded',
            JSON_ERROR_SYNTAX => 'Syntax error',
            JSON_ERROR_UTF8 => 'Malformed UTF-8 characters, possibly incorrectly encoded'
        );

        $error = json_last_error();
        return isset($ERRORS[$error]) ? $ERRORS[$error] : 'Unknown error';
    }
}

json_last_error #

json_last_error(), PHP tarafından sağlanan önceden tanımlanmış sabitlerden biriyle eşlenmiş bir tamsayı döndürür.

Sabit Anlamı
“JSON_ERROR_NONE” Hata oluşmadı
“JSON_ERROR_DEPTH” Maksimum yığın derinliği aşıldı
“JSON_ERROR_STATE_MISMATCH” Geçersiz veya hatalı biçimlendirilmiş JSON
“JSON_ERROR_CTRL_CHAR” Kontrol karakteri hatası, muhtemelen yanlış kodlanmış
“JSON_ERROR_SYNTAX” Sözdizimi hatası (PHP 5.3.3’ten beri)
“JSON_ERROR_UTF8” Hatalı biçimlendirilmiş UTF-8 karakterleri, muhtemelen yanlış kodlanmış (PHP 5.5.0’dan beri)
“JSON_ERROR_RECURSION” Kodlanacak değerde bir veya daha fazla özyinelemeli başvuru
“JSON_ERROR_INF_OR_NAN” Kodlanacak değerde bir veya daha fazla NAN veya INF değeri
“JSON_ERROR_UNSUPPORTED_TYPE” Kodlanamayan türden bir değer verildi

Bir Nesnede JsonSerializable Kullanımı #

REST API’leri oluşturduğunuzda, istemci uygulamasına iletilecek bir nesnenin bilgilerini azaltmanız gerekebilir. Bu amaçla, bu örnek, “JsonSerialiazble” arabiriminin nasıl kullanılacağını gösterir.

Bu örnekte, “Kullanıcı” sınıfı, aslında varsayımsal bir ORM’nin bir DB model nesnesini genişletir.

class User extends Model implements JsonSerializable {
    public $id;
    public $name;
    public $surname;
    public $username;
    public $password;
    public $email;
    public $date_created;
    public $date_edit;
    public $role;
    public $status;

    public function jsonSerialize() {
        return [
            'name' => $this->name,
            'surname' => $this->surname,
            'username' => $this->username
        ];
    }
}

“jsonSerialize()” yöntemini sağlayarak sınıfa “JsonSerializable” uygulamasını ekleyin.

public function jsonSerialize()

Şimdi, uygulama denetleyicinizde veya komut dosyanızda, Kullanıcı nesnesini ‘json_encode()’ öğesine iletirken, tüm nesne yerine ‘jsonSerialize()’ yönteminin dönüş json kodlu dizisini alacaksınız.

json_encode($User);

Geri dönücek:

{"name":"John", "surname":"Doe", "username" : "TestJson"}

özellik değerleri örneği. #

Bu, hem RESTful bir uç noktadan döndürülen veri miktarını azaltacak hem de nesne özelliklerinin bir json gösteriminden hariç tutulmasına izin verecektir.

json_encode() ile Özel ve Korumalı Özellikleri Kullanma

JsonSerializable kullanmaktan kaçınmak için, “json_encode()” çıktısından sınıf bilgilerini gizlemek için özel veya korumalı özellikler kullanmak da mümkündür. O zaman Sınıfın \JsonSerializable’ı uygulaması gerekmez.

json_encode() işlevi yalnızca bir JSON’a sınıflandırın.

<?php

class User {
    // private properties only within this class
    private $id;
    private $date_created;
    private $date_edit;

    // properties used in extended classes
    protected $password;
    protected $email;
    protected $role;
    protected $status;

    // share these properties with the end user        
    public $name;
    public $surname;
    public $username;

    // jsonSerialize() not needed here
}        

$theUser = new User();

var_dump(json_encode($theUser));

Çıktı: #

string(44) "{"name":null,"surname":null,"username":null}"
son sürüm

Başlık json ve döndürülen yanıt #

JSON olarak içerik türüne sahip bir başlık ekleyerek:

<?php
 $result = array('menu1' => 'home', 'menu2' => 'code php', 'menu3' => 'about');

//return the json response :
header('Content-Type: application/json');  // <-- header declaration
echo json_encode($result, true);    // <--- encode
exit();

Başlık oradadır, böylece uygulamanız hangi verilerin döndürüldüğünü ve onu nasıl işlemesi gerektiğini algılayabilir.
Şunu unutmayın: içerik başlığı yalnızca döndürülen verilerin türü hakkında bilgidir.

UTF-8 kullanıyorsanız, şunları kullanabilirsiniz:

header("Content-Type: application/json;charset=utf-8");
