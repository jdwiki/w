Sözdizimi #

  • $foo = 1; $bar = &$foo; // hem $foo hem de $bar aynı değeri gösteriyor: 1
  • $var = 1; function calc(&$var) { $var *= 15; } hesapla($var); echo $var;

Referansa göre iki değişken atarken, her iki değişken de aynı değeri gösterir. Aşağıdaki örneği alın:

$foo = 1;
$bar = &$foo;

“$foo” "$bar"ı göstermiyor. “$foo” ve “$bar”, “1” olan aynı “$foo” değerine işaret eder. Örneklemek için:

$baz = &$bar;
unset($bar);
$baz++;

Eğer bir ’nokta ile’ ilişkimiz olsaydı, bu şimdi ‘unset()’ten sonra bozulurdu; bunun yerine, “$foo” ve “$baz” yine aynı değeri, yani “2"yi gösterir.

Referansa Göre Ata #

Bu, referans vermenin ilk aşamasıdır. Esasen referansla ata yaptığınızda, iki değişkenin bu şekilde aynı değeri paylaşmasına izin vermiş olursunuz.

$foo = &$bar;

Burada “$foo” ve “$bar” eşittir. Birbirlerini göstermezler. Aynı yeri gösterirler (* “değer”*).

array() dil yapısı içinde referansa göre de atayabilirsiniz. Kesinlikle referans olarak bir atama olmamakla birlikte.

$foo = 'hi';
$bar = array(1, 2);
$array = array(&$foo, &$bar[0]);

Ancak, dizilerin içindeki referansların potansiyel olarak tehlikeli. Normal (referansla değil) bir atama yapmak sağ taraftaki referans, sol tarafı bir referans, ancak dizilerin içindeki referanslar bu normallerde korunur ödevler. Bu aynı zamanda dizinin bulunduğu işlev çağrıları için de geçerlidir. değere göre geçti.

Başvuruya göre atama yalnızca değişkenler ve dizilerle sınırlı değildir, aynı zamanda işlevler ve tüm “referansa göre” ilişkilendirmeler için de mevcuttur.

function incrementArray(&$arr) {
    foreach ($arr as &$val) {
        $val++;
    }
}

function &getArray() {
    static $arr = [1, 2, 3];
    return $arr;
}

incrementArray(getArray());
var_dump(getArray()); // prints an array [2, 3, 4]

Atama, yukarıdaki gibi işlev tanımında anahtardır. Bir ifadeyi başvuruya göre iletemezsiniz, yalnızca bir değer/değişken iletebilirsiniz. Bu nedenle, ‘bar()’ içindeki ‘$a’ örneği.

Referansa Göre Dönüş #

Bazen, dolaylı olarak referansa göre geri dönmeniz için zaman gelir.

Referansa göre geri dönmek, bir işlevi kullanmak istediğinizde yararlıdır. bir referansın hangi değişkene bağlanması gerektiğini bulun. Kullanmayın performansı artırmak için referansa göre dönüş. motor bunu kendi başına otomatik olarak optimize eder. Yalnızca şu durumlarda referansları döndür: bunu yapmak için geçerli bir teknik nedeniniz var.

PHP Documentation for Returning By Reference‘den alınmıştır.

Aşağıdaki örnek de dahil olmak üzere, referansla döndürmenin alabileceği birçok farklı form vardır:

function parent(&$var) {
    echo $var;
    $var = "updated";
}

function &child() {
    static $a = "test";
    return $a;
}

parent(child()); // returns "test"
parent(child()); // returns "updated"

Referansa göre dönüş sadece fonksiyon referansları ile sınırlı değildir. Ayrıca, işlevi dolaylı olarak çağırma olanağına da sahipsiniz:

function &myFunction() {
    static $a = 'foo';
    return $a;
}

$bar = &myFunction();
$bar = "updated"
echo myFunction();

Bir işlev çağrısını doğrudan referans yapamazsınız, onu kullanmadan önce bir değişkene atanması gerekir. Bunun nasıl çalıştığını görmek için echo &myFunction(); komutunu deneyin.

Notlar #

  • Kullanmayı düşündüğünüz her iki yerde de bir referans (&) belirtmeniz gerekmektedir. Bunun anlamı, işlev tanımınız (function &myFunction() {...) ve çağıran referansta (function callFunction(&$variable) {... veya &myFunction();).
  • Bir değişkeni yalnızca referans olarak döndürebilirsiniz. Bu nedenle, yukarıdaki örnekte “$a” örneğinin somutlaştırılması. Bu, bir ifade döndüremeyeceğiniz anlamına gelir, aksi takdirde bir E_NOTICE PHP hatası oluşturulur (Uyarı: ...... içindeki referans tarafından yalnızca değişken referansları döndürülmelidir).
  • Referansla iadenin meşru kullanım durumları vardır, ancak bunların yalnızca aynı amaca ulaşmak için diğer tüm potansiyel seçenekleri keşfettikten sonra dikkatli bir şekilde kullanılmaları gerektiği konusunda uyarmalıyım.

Referansa Göre Geçiş #

Bu, orijinal değişkeni değiştirmenize izin veren bir işleve veya öğeye başvurarak bir değişkeni iletmenize olanak tanır.

Referans yoluyla aktarma yalnızca değişkenlerle sınırlı değildir, aşağıdakiler de referans yoluyla iletilebilir:

  • Yeni ifadeler, ör. foo(yeni SomeClass)
  • Fonksiyonlardan döndürülen referanslar

Diziler #

“Referansa göre geçiş”‘in yaygın bir kullanımı, yeni diziler oluşturma veya ad alanınızı kirletme kapsamına girmeden bir dizi içindeki başlangıç ​​değerlerini değiştirmektir. Referansa göre geçmek, değişkenin önüne bir “&” => “&$myElement” eklemek/öneklemek kadar basittir.

Aşağıda, bir diziden bir öğenin kullanılmasına ve yalnızca ilk değerine 1 eklenmesine ilişkin bir örnek verilmiştir.

$arr = array(1, 2, 3, 4, 5);

foreach($arr as &$num) {
    $num++;
}

Şimdi “$arr” içindeki herhangi bir öğeyi çalıştırdığınızda, referans artırıldıkça orijinal öğe güncellenecektir. Bunu şu şekilde doğrulayabilirsiniz:

print_r($arr);

Not

Döngüler içinde referansa göre geçişi kullanırken not almalısınız. Yukarıdaki döngünün sonunda, “$num” hala dizinin son elemanına bir referans tutar. Bunu döngü sonrası atamak, son dizi öğesini manipüle etmekle sonuçlanacaktır! Döngü sonrası unset()’ yaparak bunun olmamasını sağlayabilirsiniz:

$myArray = dizi(1, 2, 3, 4, 5);

foreach($myArray as &$num) { $sayı++; } ayarlanmadı($sayı); Yukarıdakiler herhangi bir sorunla karşılaşmamanızı sağlayacaktır. Bununla ilgili olabilecek sorunlara bir örnek, StackOverflow’daki bu soru içinde mevcuttur.

Fonksiyonlar #

Referansa göre geçiş için başka bir yaygın kullanım, işlevler içindedir. Orijinal değişkeni değiştirmek şu kadar basittir:

$var = 5;
// define
function add(&$var) {
    $var++;
}
// call
add($var);

Bu, orijinal değişkeni “eko” ederek doğrulanabilir.

echo $var;

PHP belgelerinde aşağıda belirtildiği gibi, işlevlerle ilgili çeşitli kısıtlamalar vardır:

Not: Bir işlev çağrısında referans işareti yoktur - yalnızca işlevde tanımlar. İşlev tanımları tek başına doğru bir şekilde geçmek için yeterlidir referansa göre argüman. PHP 5.3.0’dan itibaren bir uyarı alacaksınız & kullandığınızda “referansa göre arama süresi"nin kullanımdan kaldırıldığını söyleyerek içinde foo(&$a);. Ve PHP 5.4.0’dan itibaren, çağrı zamanı geçiş referansı şu şekildeydi: kaldırıldı, bu nedenle onu kullanmak ölümcül bir hataya neden olacaktır.
