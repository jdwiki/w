Tip Karşılaştırma #

İki tür karşılaştırma vardır: == ile gevşek karşılaştırma ve === ile katı karşılaştırma. Kesin karşılaştırma, operatörün her iki tarafının hem tipinin hem de değerinin aynı olmasını sağlar.

// Loose comparisons
var_dump(1 == 1); // true
var_dump(1 == "1"); // true
var_dump(1 == true); // true
var_dump(0 == false); // true

// Strict comparisons
var_dump(1 === 1); // true
var_dump(1 === "1"); // false
var_dump(1 === true); // false
var_dump(0 === false); // false

// Notable exception: NAN — it never is equal to anything
var_dump(NAN == NAN); // false
var_dump(NAN === NAN); // false

Ayrıca, !== kullanarak tür ve değerin eşleşmediğini kontrol etmek için güçlü karşılaştırmayı da kullanabilirsiniz.

== operatörünün yeterli olmadığı tipik bir örnek, arama kelimesi bulunamazsa false döndüren strpos gibi farklı türler döndürebilen işlevler ve eşleşme konumudur. (int) aksi halde:

if(strpos('text', 'searchword') == false)
  // strpos returns false, so == comparison works as expected here, BUT:
if(strpos('text bla', 'text') == false)
  // strpos returns 0 (found match at position 0) and 0==false is true.
  // This is probably not what you expect!
if(strpos('text','text') === false)
  // strpos returns 0, and 0===false is false, so this works as expected.

Boole #

Boolean, “doğru” veya “yanlış” olarak gösterilen iki değere sahip bir türdür.

Bu kod, “$foo"nun değerini “true” ve “$bar"ın değerini “false” olarak ayarlar:

$foo = true;
$bar = false;

’true’ ve ‘false’ büyük/küçük harfe duyarlı değildir, bu nedenle ‘TRUE’ ve ‘FALSE’ da kullanılabilir, ‘FaLsE’ bile mümkündür. Küçük harf kullanımı en yaygın olanıdır ve çoğu kod stili kılavuzunda önerilir, ör. PSR-2.

Boolean ifadeleri aşağıdaki gibi if ifadelerinde kullanılabilir:

if ($foo) { //same as evaluating if($foo == true)
    echo "true";
}

PHP’nin zayıf yazılmış olması nedeniyle, yukarıdaki “$foo”, “doğru” veya “yanlış” dışındaysa, otomatik olarak bir boole değerine zorlanır. Aşağıdaki değerler “yanlış” ile sonuçlanır:

  • sıfır değeri: “0” (tamsayı), “0.0” (kayan) veya “0” (dize)
  • boş bir '' dizesi veya [] dizisi
  • “null” (ayarlanmamış bir değişkenin içeriği veya bir değişkene atanmış)

Başka herhangi bir değer “true” ile sonuçlanır.

Bu gevşek karşılaştırmayı önlemek için, * ve türü * karşılaştıran === kullanarak güçlü karşılaştırmayı zorunlu kılabilirsiniz. Ayrıntılar için [Tip Karşılaştırma][3] bölümüne bakın.

Bir türü boolea dönüştürmek için, türden önce (bool) veya (boolean) dökümünü kullanabilirsiniz.

var_dump((bool) "1"); //evaluates to true

veya [boolval][4] işlevini çağırın:

var_dump( boolval("1") ); //evaluates to true

Bir dizeye Boole dönüşümü (“yanlış"ın boş bir dize verdiğini unutmayın):

var_dump( (string) true ); // string(1) "1"
var_dump( (string) false ); // string(0) ""

Bir tamsayıya Boole dönüşümü:

var_dump( (int) true ); // int(1)
var_dump( (int) false ); // int(0)

Bunun tersinin de mümkün olduğunu unutmayın:

var_dump((bool) "");        // bool(false)
var_dump((bool) 1);         // bool(true)

Ayrıca tüm sıfır olmayanlar true değerini döndürür:

var_dump((bool) -2);        // bool(true)
var_dump((bool) "foo");     // bool(true)
var_dump((bool) 2.3e5);     // bool(true)
var_dump((bool) array(12)); // bool(true)
var_dump((bool) array());   // bool(false)
var_dump((bool) "false");   // bool(true)

[3]: https://www.wikiod.com/tr/php/turler#Tip Karşılaştırma [4]: http://php.net/manual/en/function.boolval.php

Batmadan yüzmek #

$float = 0.123;

Tarihsel nedenlerden dolayı, bir kayan nokta durumunda gettype() tarafından “double” döndürülür ve yalnızca “kayan” değil

Float’lar, düz tamsayılardan daha fazla çıktı hassasiyetine izin veren kayan nokta sayılarıdır.

PHP’nin değişken türlerini gevşek dökümü nedeniyle, kayan noktalar ve tam sayılar birlikte kullanılabilir:

$sum = 3 + 0.14;

echo $sum; // 3.14

php, diğer diller gibi kayan noktayı kayan sayı olarak göstermez, örneğin:

$var = 1;
echo ((float) $var); //returns 1 not 1.0

Uyarı #

Kayan nokta hassasiyeti

(PHP kılavuz sayfasından)

Kayan noktalı sayıların kesinliği sınırlıdır. Sisteme bağlı olmasına rağmen, PHP tipik olarak 1.11e-16 mertebesinde yuvarlama nedeniyle maksimum bağıl hata verir. Temel olmayan aritmetik işlemler daha büyük hatalar verebilir ve birkaç işlem bir araya getirildiğinde hata yayılımı dikkate alınmalıdır.

Ek olarak, 0,1 veya 0,7 gibi 10 tabanında tam olarak kayan noktalı sayılar olarak gösterilebilen rasyonel sayılar, boyutu ne olursa olsun dahili olarak kullanılan taban 2’de (ikili) kayan nokta sayıları olarak tam bir temsile sahip değildir. mantis Bu nedenle, küçük bir hassasiyet kaybı olmadan dahili ikili karşılıklarına dönüştürülemezler. Bu, kafa karıştırıcı sonuçlara yol açabilir: örneğin, floor((0.1+0.7)*10), dahili gösterim 7.9999999999999999991118 gibi bir şey olacağından beklenen 8 yerine genellikle 7 döndürür.

Bu nedenle, kayan sayı sonuçlarına asla son basamağa güvenmeyin ve kayan noktalı sayıları eşitlik için doğrudan karşılaştırmayın. Daha yüksek hassasiyet gerekliyse, isteğe bağlı kesinlik matematik işlevleri ve gmp işlevleri kullanılabilir.

Tek Alıntı #

Her şeyi neredeyse tamamen “olduğu gibi” görüntüler. Değişkenler ve çoğu kaçış dizisi yorumlanmayacaktır. Bunun istisnası, gerçek tek bir alıntıyı görüntülemek için, ondan bir ters eğik çizgi ' ile kaçılabilir ve bir ters eğik çizgi görüntülemek için başka bir ters eğik çizgi ile kaçılabilir \

$my_string = 'Nothing is parsed, except an escap\'d apostrophe or backslash. $foo\n';
var_dump($my_string);

/*
string(68) "Nothing is parsed, except an escap'd apostrophe or backslash. $foo\n"
*/

Çift Alıntı #

Tek tırnaklı bir dizeden farklı olarak, dizelerdeki basit değişken adları ve kaçış dizileri değerlendirilecektir. Kıvrımlı parantezler (son örnekte olduğu gibi) karmaşık değişken adlarını izole etmek için kullanılabilir.

$variable1 = "Testing!";
$variable2 = [ "Testing?", [ "Failure", "Success" ] ];
$my_string = "Variables and escape characters are parsed:\n\n";
$my_string .= "$variable1\n\n$variable2[0]\n\n";
$my_string .= "There are limits: $variable2[1][0]";
$my_string .= "But we can get around them by wrapping the whole variable in braces: {$variable2[1][1]}";
var_dump($my_string);

/*
string(98) "Variables and escape characters are parsed:

Testing!

Testing?

There are limits: Array[0]"

But we can get around them by wrapping the whole variable in braces: Success

*/

Heredoc #

Heredoc dizesinde, değişken adları ve kaçış dizileri, çift tırnaklı dizelere benzer şekilde ayrıştırılır, ancak karmaşık değişken adları için parantezler kullanılamaz. Dizenin başlangıcı <<< tanımlayıcı ile ve sonu tanımlayıcı ile sınırlandırılır, burada tanımlayıcı herhangi bir geçerli PHP adıdır. Bitiş tanımlayıcısı tek başına bir satırda görünmelidir. Tanımlayıcıdan önce veya sonra boşluk bırakılmasına izin verilmez, ancak PHP’deki herhangi bir satır gibi, noktalı virgülle de sonlandırılmalıdır.

$variable1 = "Including text blocks is easier";
$my_string = <<< EOF
Everything is parsed in the same fashion as a double-quoted string,
but there are advantages. $variable1; database queries and HTML output
can benefit from this formatting.
Once we hit a line containing nothing but the identifier, the string ends.
EOF;
var_dump($my_string);

/*
string(268) "Everything is parsed in the same fashion as a double-quoted string,
but there are advantages. Including text blocks is easier; database queries and HTML output
can benefit from this formatting.
Once we hit a line containing nothing but the identifier, the string ends."
*/

Nowdoc #

Nowdoc dizesi, en temel kaçış dizileri bile değerlendirilmese de, heredoc’un tek tırnaklı sürümü gibidir. Dizenin başındaki tanımlayıcı tek tırnak içine alınır.

$my_string = <<< 'EOF'
A similar syntax to heredoc but, similar to single quoted strings,
nothing is parsed (not even escaped apostrophes \' and backslashes \\.)
EOF;
var_dump($my_string);

/*
string(116) "A similar syntax to heredoc but, similar to single quoted strings,
nothing is parsed (not even escaped apostrophes \' and backslashes \\.)"
*/

Aranabilir #

Callables, geri arama olarak adlandırılabilecek herhangi bir şeydir. “Geri arama” olarak adlandırılabilecek şeyler şunlardır:

  • Anonim işlevler

  • Standart PHP işlevleri (not: dil yapıları değil)

  • Statik Sınıflar

    $obj = new MyClass(); call_user_func([$obj, ‘myCallbackMethod’]);

PHP 5.4’ten itibaren geri aramalar “callable” tür ipucu ile gösterilebilir.

$callable = function () {
    return 'value';
};

function call_something(callable $fn) {
    call_user_func($fn);
}

call_something($callable);

Hükümsüz #

PHP, null anahtar kelimesiyle “değer yok"u temsil eder. C dilindeki boş göstericiye ve SQL’deki NULL değerine biraz benzer.

Değişkeni null olarak ayarlamak:

$nullvar = null; // directly

function doSomething() {} // this function does not return anything
$nullvar = doSomething(); // so the null is assigned to $nullvar

Değişkenin null olarak ayarlanıp ayarlanmadığını kontrol etme:

if (is_null($nullvar)) { /* variable is null */ }

if ($nullvar === null) {  /* variable is null */ }

Null vs tanımsız değişken #

Değişken tanımlanmadıysa veya ayarlanmadıysa, boş değere karşı yapılan tüm testler başarılı olacaktır, ancak aynı zamanda bir ‘Uyarı: Tanımsız değişken: nullvar’ oluşturacaktır:

$nullvar = null;
unset($nullvar);
if ($nullvar === null) {  /* true but also a Notice is printed */ }
if (is_null($nullvar)) {  /* true but also a Notice is printed */ }

Bu nedenle tanımsız değerler isset ile kontrol edilmelidir:

if (!isset($nullvar)) {  /* variable is null or is not even defined */  }

Tip Döküm #

PHP genellikle, kullanmayı düşündüğünüz veri türünü, kullanıldığı bağlamdan doğru bir şekilde tahmin eder, ancak bazen bir türü manuel olarak zorlamak yararlıdır. Bu, bildirimin önüne parantez içinde gerekli türün adını ekleyerek gerçekleştirilebilir:

$bool = true;
var_dump($bool); // bool(true)

$int = (int) true;
var_dump($int); // int(1)

$string = (string) true;
var_dump($string); // string(1) "1"
$string = (string) false;
var_dump($string); // string(0) ""

$float = (float) true;
var_dump($float); // float(1)

$array = ['x' => 'y'];
var_dump((object) $array); // object(stdClass)#1 (1) { ["x"]=> string(1) "y" } 

$object = new stdClass();
$object->x = 'y';
var_dump((array) $object); // array(1) { ["x"]=> string(1) "y" }

$string = "asdf";
var_dump((unset)$string); // NULL

Ancak dikkatli olun: Her tür yayın, beklendiği gibi çalışmaz:

// below 3 statements hold for 32-bits systems (PHP_INT_MAX=2147483647)
// an integer value bigger than PHP_INT_MAX is automatically converted to float:
var_dump(       999888777666 ); // float(999888777666)
// forcing to (int) gives overflow:
var_dump((int)  999888777666 ); // int(-838602302)
// but in a string it just returns PHP_INT_MAX
var_dump((int) "999888777666"); // int(2147483647)

var_dump((bool) []);      // bool(false) (empty array)
var_dump((bool) [false]); // bool(true)  (non-empty array)

Kaynaklar #

resource, dosya, soket, akış gibi harici bir kaynağa başvuran özel bir değişken türüdür. belge veya bağlantı.

$file = fopen('/etc/passwd', 'r');

echo gettype($file);
# Out: resource

echo $file;
# Out: Resource id #2

Farklı (alt) kaynak türleri vardır. get_resource_type() kullanarak kaynak türünü kontrol edebilirsiniz:

$file = fopen('/etc/passwd', 'r');
echo get_resource_type($file);
#Out: stream

$sock = fsockopen('www.google.com', 80);
echo get_resource_type($sock);
#Out: stream

Yerleşik kaynak türlerinin tam listesini burada bulabilirsiniz.

Tip Hokkabazlık #

PHP zayıf yazılmış bir dildir. Veri türlerinin açık bir şekilde bildirilmesini gerektirmez. Değişkenin kullanıldığı bağlam, veri türünü belirler; dönüşüm otomatik olarak yapılır:

$a = "2";             // string 
$a = $a + 2;          // integer (4) 
$a = $a + 0.5;        // float (4.5)
$a = 1 + "2 oranges"; // integer (3)

Tamsayılar #

PHP’deki tamsayılar doğal olarak taban 2 (ikili), taban 8 (sekizlik), taban 10 (ondalık) veya taban 16 (onaltılık) olarak belirtilebilir.

$my_decimal = 42;
$my_binary = 0b101010;
$my_octal = 052;
$my_hexadecimal = 0x2a;

echo ($my_binary + $my_octal) / 2;
// Output is always in decimal: 42

Tam sayılar, platforma bağlı olarak 32 veya 64 bit uzunluğundadır. “PHP_INT_SIZE” sabiti, tamsayı boyutunu bayt olarak tutar. “PHP_INT_MAX” ve (PHP 7.0’dan beri) “PHP_INT_MIN” de mevcuttur.

printf("Integers are %d bits long" . PHP_EOL, PHP_INT_SIZE * 8);
printf("They go up to %d" . PHP_EOL, PHP_INT_MAX);

Tamsayı değerleri, gerektiğinde kayan değerler, booleanlar ve dizelerden otomatik olarak oluşturulur. Açık bir typecast gerekiyorsa, bu (int) veya (integer) dökümü ile yapılabilir:

$my_numeric_string = "123";
var_dump($my_numeric_string);
// Output: string(3) "123"
$my_integer = (int)$my_numeric_string;
var_dump($my_integer);
// Output: int(123)

Tamsayı taşması, bir şamandıraya dönüştürülerek ele alınacaktır:

$too_big_integer = PHP_INT_MAX + 7;
var_dump($too_big_integer);
// Output: float(9.2233720368548E+18)

PHP’de tamsayı bölme operatörü yoktur, ancak her zaman sadece kayan kısmı atarak ‘yuvarlanan’ örtük bir döküm kullanılarak simüle edilebilir. PHP sürüm 7’den itibaren bir tamsayı bölme işlevi eklendi.

$not_an_integer = 25 / 4;
var_dump($not_an_integer);
// Output: float(6.25)
var_dump((int) (25 / 4)); // (see note below)
// Output: int(6)
var_dump(intdiv(25 / 4)); // as of PHP7
// Output: int(6)

((int) dökümü bölmeden daha yüksek önceliğe sahip olduğu için(25/4) etrafındaki fazladan parantezlerin gerekli olduğunu unutmayın)
