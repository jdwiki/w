Sözdizimi #

  • int readfile ( string $dosyaadı [, bool $use_include_path = false [, kaynak $bağlam ]] )

Parametreler #

parametre Açıklama
dosya adı Dosya adı okunuyor.
use_include_path Dosyayı include_path içinde de aramak istiyorsanız, isteğe bağlı ikinci parametreyi kullanabilir ve DOĞRU olarak ayarlayabilirsiniz.

Dosya adı sözdizimi #

Bu konudaki işlevlere iletilen çoğu dosya adı şunlardır:

  1. Doğada dizeler.
  • Dosya adları doğrudan aktarılabilir. Diğer türlerin değerleri iletilirse, dizeye dönüştürülürler. Bu, özellikle “DirectoryIterator” yinelemesindeki değer olan “SplFileInfo” için kullanışlıdır.
  1. Göreceli veya mutlak.
  • Mutlak olabilirler. Unix benzeri sistemlerde, mutlak yollar / ile başlar, ör. /home/user/file.txt, Windows’tayken, mutlak yollar sürücü ile başlar, ör. C:/Kullanıcılar/kullanıcı/dosya.txt
  • Ayrıca, getcwd değerine bağlı olan ve chdir tarafından değiştirilebilir olan göreli de olabilirler.
  1. Protokolleri kabul edin.
  • Yönetilecek protokol sarmalayıcıyı belirtmek için scheme:// ile başlayabilirler. Örneğin, file_get_contents("http://example.com") http://example.com adresinden içerik alır.
  1. Eğik çizgi uyumlu.
  • Windows’ta DIRECTORY_SEPARATOR ters eğik çizgi iken ve sistem varsayılan olarak yollar için ters eğik çizgi döndürürken, geliştirici yine de dizin ayırıcı olarak / kullanabilir. Bu nedenle, uyumluluk için geliştiriciler / karakterini tüm sistemlerde dizin ayırıcı olarak kullanabilirler, ancak işlevler tarafından döndürülen değerlerin (ör. realpath) ters eğik çizgi içerebileceğini unutmayın.

Dosyaları ve dizinleri silme #

Dosyaları silme #

[bağlantıyı kaldır][bağlantıyı kaldır] işlevi, tek bir dosyayı siler ve işlemin başarılı olup olmadığını döndürür.

$filename = '/path/to/file.txt';

if (file_exists($filename)) {
    $success = unlink($filename);
    
    if (!$success) {
         throw new Exception("Cannot delete $filename");
    }
}

Özyinelemeli silme ile dizinleri silme #

Öte yandan, dizinler [rmdir][rmdir] ile silinmelidir. Ancak bu işlev yalnızca boş dizinleri siler. Dosya içeren bir dizini silmek için önce dizinlerdeki dosyaları silin. Dizin alt dizinler içeriyorsa, recursion gerekli olabilir.

Aşağıdaki örnek, bir dizindeki dosyaları tarar, üye dosyaları/dizinleri tekrar tekrar siler ve silinen dosyaların (dizinlerin değil) sayısını döndürür.

function recurse_delete_dir(string $dir) : int {
    $count = 0;

    // ensure that $dir ends with a slash so that we can concatenate it with the filenames directly
    $dir = rtrim($dir, "/\\") . "/";

    // use dir() to list files
    $list = dir($dir);

    // store the next file name to $file. if $file is false, that's all -- end the loop.
    while(($file = $list->read()) !== false) {
        if($file === "." || $file === "..") continue;
        if(is_file($dir . $file)) {
            unlink($dir . $file);
            $count++;
        } elseif(is_dir($dir . $file)) {
            $count += recurse_delete_dir($dir . $file);
        }
    }

    // finally, safe to delete directory!
    rmdir($dir);

    return $count;
}

[bağlantıyı kaldır]: http://php.net/bağlantıyı kaldır [rmdir]: http://php.net/rmdir

Kolaylık işlevleri #

Ham doğrudan IO #

file_get_contents ve file_put_contents, tek bir çağrıda bir PHP dizesine/dosyadan/dosyadan/dosyadan okuma/yazma yeteneği sağlar.

file_put_contents, dosyaya kesmek ve üzerine yazmak yerine eklemek için FILE_APPEND bit maskesi bayrağıyla da kullanılabilir. Yazmaya devam ederken dosyaya özel bir kilit almak için LOCK_EX bit maskesi ile birlikte kullanılabilir. Bit maskesi bayrakları | bitsel VEYA operatörü ile birleştirilebilir.

$path = "file.txt";
// reads contents in file.txt to $contents
$contents = file_get_contents($path);
// let's change something... for example, convert the CRLF to LF!
$contents = str_replace("\r\n", "\n", $contents);
// now write it back to file.txt, replacing the original contents
file_put_contents($path, $contents);

‘FILE_APPEND’ günlük dosyalarına ekleme yapmak için kullanışlıdır, ‘LOCK_EX’ ise birden çok işlemden dosya yazmanın yarış durumunu önlemeye yardımcı olur. Örneğin, geçerli oturum hakkında bir günlük dosyasına yazmak için:

file_put_contents("logins.log", "{$_SESSION["username"]} logged in", FILE_APPEND | LOCK_EX);

CSV GÇ #

fgetcsv($file, $length, $separator)

fgetcsv, csv alanları için açık dosya kontrolünden satırı ayrıştırır. Başarı durumunda bir dizideki CSV alanları veya başarısızlık durumunda ‘YANLIŞ’ döndürür.

Varsayılan olarak, CSV dosyasının yalnızca bir satırını okuyacaktır.

$file = fopen("contacts.csv","r");
print_r(fgetcsv($file));    
print_r(fgetcsv($file,5," "));
fclose($file);

    [3] => Norway
)

    [0] => Hege,

Bir dosyayı doğrudan stdout’a okumak #

readfile bir dosyayı çıktı arabelleğine kopyalar. readfile(), büyük dosyalar gönderirken bile kendi başına herhangi bir bellek sorunu oluşturmaz.

$file = 'monkey.gif';

if (file_exists($file)) {
    header('Content-Description: File Transfer');
    header('Content-Type: application/octet-stream');
    header('Content-Disposition: attachment; filename="'.basename($file).'"');
    header('Expires: 0');
    header('Cache-Control: must-revalidate');
    header('Pragma: public');
    header('Content-Length: ' . filesize($file));
    readfile($file);
    exit;
}

Veya bir dosya işaretçisinden #

Alternatif olarak, dosyada stdout’a kopyalamaya başlamak üzere bir nokta aramak için bunun yerine fpassthru kullanın. Aşağıdaki örnekte, son 1024 bayt stdout’a kopyalanmıştır:

$fh = fopen("file.txt", "rb");
fseek($fh, -1024, SEEK_END); 
fpassthru($fh);

Bir diziye dosya okuma #

[file][file], bir dizide geçirilen dosyadaki satırları döndürür. Dizinin her bir elemanı dosyadaki bir satıra karşılık gelir ve yeni satır hala eklidir.

print_r(file("test.txt"));

test.txt

Welcome to File handling
This is to test file handling

Çıktı:

Array 
( 
    [0] => Welcome to File handling 
    [1] => This is to test file handling 
)

Dosya bilgilerini alma #

Yolun bir dizin mi yoksa dosya mı olduğunu kontrol edin #

is_dir işlevi, argümanın bir dizin olup olmadığını döndürürken, is_file, argümanın bir dosya olup olmadığını döndürür. Her ikisinin de olup olmadığını kontrol etmek için file_exists kullanın.

$dir  = "/this/is/a/directory";
$file = "/this/is/a/file.txt";

echo is_dir($dir) ? "$dir is a directory" : "$dir is not a directory", PHP_EOL,
    is_file($dir) ? "$dir is a file" : "$dir is not a file", PHP_EOL,
    file_exists($dir) ? "$dir exists" : "$dir doesn't exist", PHP_EOL,
    is_dir($file) ? "$file is a directory" : "$file is not a directory", PHP_EOL,
    is_file($file) ? "$file is a file" : "$file is not a file", PHP_EOL,
    file_exists($file) ? "$file exists" : "$file doesn't exist", PHP_EOL;

Bu verir:

/this/is/a/directory is a directory
/this/is/a/directory is not a file
/this/is/a/directory exists
/this/is/a/file.txt is not a directory
/this/is/a/file.txt is a file
/this/is/a/file.txt exists

Dosya türü kontrol ediliyor #

Bir dosyanın türünü kontrol etmek için filetype kullanın, bu olabilir:

  • “fifo”
  • “karakter”
  • “dir”
  • “blok”
  • “bağ”
  • “dosya”
  • “soket”
  • “bilinmeyen”

Dosya adını doğrudan filetype öğesine iletmek:

echo filetype("~"); // dir

“filetype"ın false döndürdüğünü ve dosya yoksa bir “E_WARNING"i tetiklediğini unutmayın.

Okunabilirliği ve yazılabilirliği kontrol etme #

Dosya adını is_writable ve is_readable işlevlerine iletmek, dosyanın sırasıyla yazılabilir mi yoksa okunabilir mi olduğunu kontrol eder.

Dosya mevcut değilse, işlevler düzgün bir şekilde ‘false’ döndürür.

Dosya erişim/değiştirme zamanı kontrol ediliyor #

filemtime ve [fileatime][fileatime] kullanımı, dosyaya yapılan son değişikliğin veya erişimin zaman damgasını döndürür. Dönüş değeri bir Unix zaman damgasıdır - ayrıntılar için bkz. https://www.wikiod.com/tr/php/tarihler-ve-saatlerle-calsma

echo "File was last modified on " . date("Y-m-d", filemtime("file.txt"));
echo "File was last accessed on " . date("Y-m-d", fileatime("file.txt"));

fileinfo ile yol parçalarını alın #

$fileToAnalyze = ('/var/www/image.png');

$filePathParts = pathinfo($fileToAnalyze);

echo '<pre>';
   print_r($filePathParts);
echo '</pre>';

Bu örnek çıktı verecektir:

Array
(
    [dirname] => /var/www
    [basename] => image.png
    [extension] => png
    [filename] => image
)

Hangi olarak kullanılabilir:

$filePathParts['dirname']
$filePathParts['basename']
$filePathParts['extension']
$filePathParts['filename']
parametre Ayrıntılar
$yol Ayrıştırılacak dosyanın tam yolu
$seçenek Mevcut dört seçenekten biri [PATHINFO_DIRNAME, PATHINFO_BASENAME, PATHINFO_EXTENSION veya PATHINFO_FILENAME]
  • Bir seçenek (ikinci parametre) iletilmezse, ilişkisel bir dizi döndürülür, aksi takdirde bir dize döndürülür.
  • Dosyanın var olduğunu doğrulamaz.
  • Basitçe dizeyi parçalara ayırır. Dosya üzerinde doğrulama yapılmaz (mime tipi kontrol vs. yok)
  • Uzantı basitçe ‘$path’ öğesinin son uzantısıdır ‘image.jpg.png’ dosyasının yolu, teknik olarak bir ‘.jpg’ dosyası olsa bile ‘.png’ olacaktır. Uzantısı olmayan bir dosya, dizide bir uzantı öğesi döndürmez.

Büyük dosyalarla uğraşırken bellek kullanımını en aza indirin #

Büyük bir dosyayı ayrıştırmamız gerekirse, ör. Milyonlarca satır içeren 10 Mbayttan fazla bir CSV, bazıları ‘file’ veya ‘file_get_contents’ işlevlerini kullanır ve sonunda ‘memory_limit’ ayarına basar.

İzin verilen XXXXX baytlık bellek boyutu tükendi

hata. Aşağıdaki kaynağı göz önünde bulundurun (top-1m.csv tam olarak 1 milyon satıra sahiptir ve yaklaşık 22 Mbyte boyutundadır)

var_dump(memory_get_usage(true));
$arr = file('top-1m.csv');
var_dump(memory_get_usage(true));

Bu çıktı:

int(262144)
int(210501632)

yorumlayıcının tüm satırları $arr dizisinde tutması gerektiğinden, ~200 Mbyte RAM tüketmiştir. Dizinin içeriğiyle ilgili hiçbir şey yapmadığımızı unutmayın.

Şimdi aşağıdaki kodu göz önünde bulundurun:

var_dump(memory_get_usage(true));
$index = 1;
if (($handle = fopen("top-1m.csv", "r")) !== FALSE) {
    while (($row = fgetcsv($handle, 1000, ",")) !== FALSE) {
        file_put_contents('top-1m-reversed.csv',$index . ',' . strrev($row[1]) . PHP_EOL, FILE_APPEND);
        $index++;
    }
    fclose($handle);
}
var_dump(memory_get_usage(true));

hangi çıktılar

int(262144)
int(262144)

bu yüzden fazladan tek bir bayt bellek kullanmıyoruz, ancak tüm CSV’yi ayrıştırıyoruz ve 2. sütunun değerini tersine çevirerek başka bir dosyaya kaydediyoruz. Bunun nedeni, “fgetcsv"nin yalnızca bir satır okuması ve her döngüde “$row"un üzerine yazılmasıdır.

Akış tabanlı dosya IO #

Akış açma #

fopen, okuma, yazma, arama ve bunun üzerindeki diğer işlevler için çeşitli işlevlerle kullanılabilen bir dosya akışı tanıtıcısı açar. Bu değer “kaynak” türündedir ve işlevselliğini sürdüren diğer iş parçacıklarına iletilemez.

$f = fopen("errors.log", "a"); // Will try to open errors.log for writing

İkinci parametre, dosya akışının modudur:

Mod Açıklama
r Dosyanın başından başlayarak salt okunur modda aç
r+ Dosyanın başından başlayarak okuma ve yazma için aç
w dosyanın başından başlayarak yalnızca yazmaya açık. Dosya varsa, dosyayı boşaltacaktır. Eğer mevcut değilse, onu yaratmaya çalışacaktır.
w+ dosyanın başından başlayarak okuma ve yazma için açın. Dosya varsa, dosyayı boşaltacaktır. Eğer mevcut değilse, onu yaratmaya çalışacaktır.
“bir” dosyanın sonundan başlayarak yalnızca yazmak için bir dosya açın. Dosya yoksa, onu oluşturmaya çalışacaktır
bir+ dosyanın sonundan başlayarak okuma ve yazma için bir dosya açın. Dosya yoksa, onu oluşturmaya çalışacaktır
x yalnızca yazmak için bir dosya oluşturun ve açın. Dosya varsa, “fopen” çağrısı başarısız olur
x+ okuma ve yazma için bir dosya oluşturun ve açın. Dosya varsa, “fopen” çağrısı başarısız olur
c dosyayı sadece yazmak için açın. Dosya yoksa, onu oluşturmaya çalışacaktır. Dosyanın başında yazmaya başlar, ancak dosyayı yazmadan önce boşaltmaz
c+ dosyayı okumak ve yazmak için açın. Dosya yoksa, onu oluşturmaya çalışacaktır. Dosyanın başında yazmaya başlar, ancak dosyayı yazmadan önce boşaltmaz

Windows’ta modun arkasına bir “t” eklemek (örneğin “a+b”, “wt” vb.), dosyayla çalışırken “"\n” satır sonlarını ""\r\n" olarak çevirir. Bu amaçlanmamışsa, özellikle ikili bir dosyaysa, modun arkasına b ekleyin.

PHP uygulaması, “Çok fazla açık dosya” hatasını önlemek için artık kullanılmadığında fclose kullanarak akışları kapatmalıdır. Akışlar yalnızca çalışma zamanı kapandığında kapatıldığından, bu özellikle CLI programlarında önemlidir - bu, web sunucularında bunun gerekli olmadığı_ (ancak kaynak sızıntısını önlemek için bir uygulama olarak yine de _gerektiği) anlamına gelir. işlemin uzun süre çalışmasını beklemiyorsanız akışları kapatın ve çok fazla akış açmayacak.

Okuma #

fread kullanılması, dosya işaretçisinden veya bir EOF karşılanana kadar verilen bayt sayısını okuyacaktır.

Okuma satırları #

fgets kullanılması, bir EOL’ye ulaşılana veya verilen uzunluk okunana kadar dosyayı okuyacaktır.

Hem fread hem de fgets, okuma sırasında dosya işaretçisini hareket ettirecektir.

Kalan her şeyi okumak #

stream_get_contents kullanılması, akışta kalan tüm baytları bir dizeye dönüştürür ve döndürür.

Dosya işaretçi konumunu ayarlama #

İlk olarak akışı açtıktan sonra, dosya işaretçisi dosyanın başındadır (veya “a” kipi kullanılıyorsa, sonundadır). fseek işlevinin kullanılması, dosya işaretçisini üç değerden birine göre yeni bir konuma taşır:

  • SEEK_SET: Bu varsayılan değerdir; dosya konumu ofseti dosyanın başlangıcına göre olacaktır.
  • SEEK_CUR: Dosya konumu ofseti mevcut konuma göre olacaktır.
  • SEEK_END: Dosya konumu ofseti dosyanın sonuna göre olacaktır. Negatif ofset iletmek, bu değerin en yaygın kullanımıdır; dosya konumunu, dosyanın sonundan önce belirtilen bayt sayısına taşıyacaktır.

[rewind][rewind], fseek($fh, 0, SEEK_SET)nin kullanışlı bir kısayoludur.

ftell kullanımı, dosya işaretçisinin mutlak konumunu gösterecektir.

Örneğin, aşağıdaki komut dosyası ilk 10 baytı atlar, sonraki 10 baytı okur, 10 baytı atlar, sonraki 10 baytı ve ardından file.txt’deki son 10 baytı okur:

$fh = fopen("file.txt", "rb");
fseek($fh, 10); // start at offset 10
echo fread($fh, 10); // reads 10 bytes
fseek($fh, 10, SEEK_CUR); // skip 10 bytes
echo fread($fh, 10); // read 10 bytes
fseek($fh, -10, SEEK_END); // skip to 10 bytes before EOF
echo fread($fh, 10); // read 10 bytes
fclose($fh);

Yazı #

fwrite kullanılması, sağlanan dizeyi geçerli dosya işaretçisinden başlayarak dosyaya yazar.

fwrite($fh, "Some text here\n");

Dosyaları ve dizinleri Taşıma ve Kopyalama #

Dosyalar kopyalanıyor #

[kopyala][kopyala], birinci bağımsız değişkendeki kaynak dosyayı ikinci bağımsız değişkendeki hedefe kopyalar. Çözümlenen hedefin önceden oluşturulmuş bir dizinde olması gerekir.

if (copy('test.txt', 'dest.txt')) {
    echo 'File has been copied successfully';
} else {
    echo 'Failed to copy file to destination given.'
}

Dizinleri özyinelemeli kopyalama #

Dizinleri kopyalamak, dizinleri silmeye oldukça benzer, ancak dosyalar için [‘bağlantıyı kaldır’][bağlantıyı kaldır] yerine [‘kopyala’][kopyala] kullanılırken, dizinler için [ yerine [‘mkdir’][mkdir] kullanılır. rmdir][rmdir], fonksiyonun sonunda olmak yerine başında kullanılır.

function recurse_delete_dir(string $src, string $dest) : int {
    $count = 0;

    // ensure that $src and $dest end with a slash so that we can concatenate it with the filenames directly
    $src = rtrim($dest, "/\\") . "/";
    $dest = rtrim($dest, "/\\") . "/";

    // use dir() to list files
    $list = dir($src);

    // create $dest if it does not already exist
    @mkdir($dest);

    // store the next file name to $file. if $file is false, that's all -- end the loop.
    while(($file = $list->read()) !== false) {
        if($file === "." || $file === "..") continue;
        if(is_file($src . $file)) {
            copy($src . $file, $dest . $file);
            $count++;
        } elseif(is_dir($src . $file)) {
            $count += recurse_copy_dir($src . $file, $dest . $file);
        }
    }

    return $count;
}

Yeniden Adlandırma/Taşıma #

Dosyaları ve dizinleri yeniden adlandırmak/taşımak çok daha basittir. Tüm dizinler, [rename][rename] işlevi kullanılarak tek bir çağrıda taşınabilir veya yeniden adlandırılabilir.

  • rename("~/file.txt", "~/file.html");
  • rename("~/dir", "~/old_dir");
  • rename("~/dir/file.txt", "~/dir2/file.txt");
