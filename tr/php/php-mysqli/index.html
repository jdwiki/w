mysqli arayüzü, sürümde kullanımdan kaldırılan mysql arayüzünün bir geliştirmesidir (“MySQL İyileştirme uzantısı” anlamına gelir). 5.5 ve 7.0 sürümünde kaldırılmıştır. Mysqli uzantısı veya bazen bilindiği gibi, MySQL geliştirilmiş uzantısı, MySQL sistemlerinin 4.1.3 ve daha yeni sürümlerinde bulunan yeni özelliklerden yararlanmak için geliştirilmiştir. mysqli uzantısı PHP sürüm 5 ve sonraki sürümlerde bulunur.

Özellikler #

mysqli arabiriminin bir dizi avantajı vardır, mysql uzantısına göre önemli geliştirmeler şunlardır:

  • Nesneye yönelik arayüz
  • Hazırlanan Bildiriler için Destek
  • Çoklu İfadeler için Destek
  • İşlemler için Destek
  • Gelişmiş hata ayıklama yetenekleri
  • Gömülü sunucu desteği

Bir ikili arayüz: daha eski, prosedürel stil ve yeni bir nesne yönelimli programlama (OOP) stili. Kullanımdan kaldırılan “mysql” yalnızca prosedürel bir arayüze sahipti, bu nedenle nesne yönelimli stil genellikle tercih edilir. Bununla birlikte, yeni stil, OOP’nin gücü nedeniyle de uygundur.

Alternatifler #

Veritabanlarına erişmek için mysqli arayüzüne bir alternatif, daha yeni PHP Data Objects (PDO) arayüzüdür. Bu, yalnızca OOP stili programlamaya sahiptir ve yalnızca MySQL türü veritabanlarından daha fazlasına erişebilir.

Yakın bağlantı #

Veritabanını sorgulamayı bitirdiğimizde, kaynakları boşaltmak için bağlantıyı kapatmamız önerilir.

Nesne yönelimli stil

$conn->close();

Prosedür stili

mysqli_close($conn);

Not: Daha önce açık olarak bağlantı kapat işlevi çağrılarak kapatılmadığı sürece, komut dosyasının yürütülmesi biter bitmez sunucuyla bağlantı kapatılacaktır.

Kullanım Örneği: Komut dosyamızın sonucu aldıktan sonra gerçekleştirmesi gereken yeterli miktarda işlem varsa ve tam sonuç kümesini aldıysa, bağlantıyı kesinlikle kapatmalıyız. Bunu yapmasaydık, web sunucusu yoğun kullanım altındayken MySQL sunucusunun bağlantı sınırına ulaşma şansı vardır.

MySQLi bağlantısı #

Nesne yönelimli stil

Sunucuya bağlan

$conn = new mysqli("localhost","my_user","my_password");

Varsayılan veritabanını ayarlayın: $conn->select_db("my_db");

Veritabanına Bağlan

$conn = new mysqli("localhost","my_user","my_password","my_db");

Prosedür stili

Sunucuya bağlan

$conn = mysqli_connect("localhost","my_user","my_password");

Varsayılan veritabanını ayarlayın: mysqli_select_db($bağlantı, "my_db");

Veritabanına Bağlan

$conn = mysqli_connect("localhost","my_user","my_password","my_db");

Veritabanı Bağlantısını Doğrulayın

Nesneye yönelik stil

if ($conn->connect_errno > 0) {
    trigger_error($db->connect_error);
} // else: successfully connected

prosedür stili

if (!$conn) {
   trigger_error(mysqli_connect_error());
} // else: successfully connected

MySQLi sonuçları arasında dolaşın #

PHP, sonuçlarınızdan veri almayı ve bir ‘while’ deyimi kullanarak üzerinde döngü oluşturmayı kolaylaştırır. Bir sonraki satırı alamadığında ‘false’ döndürür ve döngünüz sona erer. Bu örnekler ile çalışır

  • mysqli_fetch_assoc - Anahtar olarak sütun adlarını içeren ilişkisel dizi
  • mysqli_fetch_object - Değişken olarak sütun adlarına sahip `stdClass’ nesnesi
  • mysqli_fetch_array - İlişkisel VE Sayısal dizi (birini veya diğerini almak için bağımsız değişkenleri kullanabilir)
  • mysqli_fetch_row - Sayısal dizi

Nesne yönelimli stil

while($row = $result->fetch_assoc()) {
    var_dump($row);
}

Prosedür stili

while($row = mysqli_fetch_assoc($result)) {
    var_dump($row);
}

Sonuçlardan kesin bilgi almak için şunları kullanabiliriz:

while ($row = $result->fetch_assoc()) {
    echo 'Name and surname: '.$row['name'].' '.$row['surname'].'<br>';
    echo 'Age: '.$row['age'].'<br>'; // Prints info from 'age' column
}

MySQLi’de hazırlanmış ifadeler #

Hazırlanan ifadelerin SQL ifadelerinizi SQL Enjeksiyon saldırılarından korumanıza neden yardımcı olduğuna dair eksiksiz bir tartışma için lütfen https://www.wikiod.com/tr/php/pdo#Parametreli Sorgularla SQL Injection’ı Önleme bölümünü okuyun.

Buradaki $conn değişkeni bir MySQLi nesnesidir. Daha fazla ayrıntı için MySQLi bağlantı örneği bölümüne bakın.

Her iki örnek için de “$sql"nin

$sql = "SELECT column_1 
    FROM table 
    WHERE column_2 = ? 
        AND column_3 > ?";

? daha sonra vereceğimiz değerleri temsil eder. Lütfen, türü ne olursa olsun yer tutucular için alıntılara ihtiyacımız olmadığını unutmayın. Ayrıca, sorgunun yalnızca “SET”, “DEĞERLER” ve “NEREDE” anlamına gelen veri bölümlerinde yer tutucular sağlayabiliriz. ‘SEÇ’ veya ‘FROM’ bölümlerinde yer tutucuları kullanamazsınız.

Nesne yönelimli stil

if ($stmt = $conn->prepare($sql)) {
  $stmt->bind_param("si", $column_2_value, $column_3_value);
  $stmt->execute();

  $stmt->bind_result($column_1);
  $stmt->fetch();
  //Now use variable $column_1 one as if it were any other PHP variable
  $stmt->close();
}

Prosedür stili

if ($stmt = mysqli_prepare($conn, $sql)) {
  mysqli_stmt_bind_param($stmt, "si", $column_2_value, $column_3_value);
  mysqli_stmt_execute($stmt);
  // Fetch data here
  mysqli_stmt_close($stmt);
}

“$stmt->bind_param” öğesinin ilk parametresi veya “mysqli_stmt_bind_param” öğesinin ikinci parametresi, SQL sorgusunda karşılık gelen parametrenin veri türü tarafından belirlenir:

parametre Bağlı parametrenin veri türü
ben tamsayı
d çift ​​
s dize
b damla

Parametre listenizin, sorgunuzda belirtilen sırada olması gerekir. Bu örnekte “si”, ilk (column_2 = ?) parametresinin dize ve ikinci parametrenin (column_3 > ?) tamsayı olduğu anlamına gelir.

Veri almak için bkz. https://www.wikiod.com/tr/php/php-mysqli#Hazırlanmış bir ifadeden veri nasıl alınır

1: https://www.wikiod.com/tr/php/php-mysqli#MySQLi bağlantısı

Kaçan Dizeler #

Kaçan dizeler, bir sorguya eklenmek üzere verileri güvenceye almak için daha eski (ve daha az güvenli) bir yöntemdir. Verileri işlemek ve sterilize etmek (başka bir deyişle) için MySQL’in mysql_real_escape_string() işlevini kullanarak çalışır (başka bir deyişle , PHP kaçış yapmıyor). MySQLi API, bu işleve doğrudan erişim sağlar

$escaped = $conn->real_escape_string($_GET['var']);
// OR
$escaped = mysqli_real_escape_string($conn, $_GET['var']);

Bu noktada, MySQL’in doğrudan sorguda kullanmak için güvenli olduğunu düşündüğü bir dizginiz var.

$sql = 'SELECT * FROM users WHERE username = "' . $escaped . '"';
$result = $conn->query($sql);

Peki bu neden [hazırlanmış ifadeler](https://www.wikiod.com/tr/php/php-mysqli#MySQLi’de hazırlanan ifadeler) kadar güvenli değil? MySQL’i güvenli olduğunu düşündüğü bir dize üretmesi için kandırmanın yolları vardır. Aşağıdaki örneği göz önünde bulundurun

$id = mysqli_real_escape_string("1 OR 1=1");    
$sql = 'SELECT * FROM table WHERE id = ' . $id;

‘1 OR 1=1’ MySQL’in kaçacağı verileri temsil etmez, ancak bu yine de SQL enjeksiyonunu temsil eder. Güvenli olmayan verileri döndürdüğü yerleri temsil eden başka örnekler de vardır. Sorun, MySQL’in kaçış işlevinin verileri SQL sözdizimiyle uyumlu hale getirmek için tasarlanmış olmasıdır. MySQL’in SQL talimatları için kullanıcı verilerini karıştırmayacağından emin olmak için TASARLANMAMIŞTIR.

MySQLi sorgusu #

“query” işlevi geçerli bir SQL dizesi alır ve onu doğrudan “$conn” veritabanı bağlantısına karşı yürütür.

Nesne yönelimli stil

$result = $conn->query("SELECT * FROM `people`");

Prosedür stili

$result = mysqli_query($conn, "SELECT * FROM `people`");

DİKKAT**

Buradaki yaygın bir sorun, insanların sorguyu basitçe yürütüp çalışmasını beklemesidir (yani, bir mysqli_stmt nesnesi döndürme). Bu işlev yalnızca bir dize aldığından, sorguyu önce kendiniz oluşturuyorsunuz. SQL’de herhangi bir hata varsa, MySQL derleyicisi başarısız olur, bu noktada bu işlev ‘false’ döndürür.

$result = $conn->query('SELECT * FROM non_existent_table'); // This query will fail
$row = $result->fetch_assoc();

Yukarıdaki kod bir “E_FATAL” hatası üretecektir çünkü “$sonuç” bir nesne değil “yanlış"tır.

PHP Ölümcül hata: Bir nesne olmayan bir üye işlevi fetch_assoc() çağrısı

Prosedürel hata benzerdir, ancak ölümcül değildir, çünkü biz sadece işlevin beklentilerini ihlal ediyoruz.

$row = mysqli_fetch_assoc($result); // same query as previous

PHP’den aşağıdaki mesajı alacaksınız

mysqli_fetch_array() parametre 1’in boolean verilen mysqli_result olmasını bekler

Önce bir test yaparak bundan kaçınabilirsiniz.

if($result) $row = mysqli_fetch_assoc($result);

MySQLi’de SQL’de Hata Ayıklama #

Yani sorgunuz başarısız oldu (’$conn’u nasıl yaptığımızı öğrenmek için https://www.wikiod.com/tr/php/php-mysqli#MySQLi bağlantısı bakın)

$result = $conn->query('SELECT * FROM non_existent_table'); // This query will fail

Ne olduğunu nasıl öğreneceğiz? “$sonuç” “yanlış” olduğundan bunun bir yardımı olmaz. Neyse ki connect $conn bize MySQL’in başarısızlık hakkında ne söylediğini söyleyebilir.

trigger_error($conn->error);

veya prosedürel

trigger_error(mysqli_error($conn));

Şuna benzer bir hata almalısınız

‘my_db.non_existent_table’ tablosu mevcut değil

Hazırlanmış bir ifadeden veri nasıl alınır #

Hazırlanan ifadeler #

Bir sorgunun nasıl hazırlanacağını ve yürütüleceğini öğrenmek için [MySQLi’de hazırlanan ifadeler](https://www.wikiod.com/tr/php/php-mysqli#MySQLi’de hazırlanan ifadeler) bölümüne bakın.

Sonuçların bağlanması #

Nesneye yönelik stil

$stmt->bind_result($forename);

Prosedür stili

mysqli_stmt_bind_result($stmt, $forename);

bind_result kullanmanın sorunu, kullanılacak sütunları belirtmek için ifadenin gerektirmesidir. Bu, yukarıdakilerin çalışması için sorgunun şu şekilde görünmesi gerektiği anlamına gelir: “Kullanıcılardan ön ad SEÇ”. Daha fazla sütun eklemek için bunları “bind_result” işlevine parametre olarak eklemeniz yeterlidir (ve bunları SQL sorgusuna eklediğinizden emin olun).

Her iki durumda da, “adı” sütununu “$ önad” değişkenine atıyoruz. Bu işlevler, atamak istediğiniz sütunlar kadar argüman alır. Atama yalnızca bir kez yapılır, çünkü işlev başvuruya göre bağlanır.

Daha sonra aşağıdaki gibi döngü yapabiliriz:

Nesneye yönelik stil

while ($stmt->fetch())
    echo "$forename<br />";

Prosedür stili

while (mysqli_stmt_fetch($stmt))
    echo "$forename<br />";

Bunun dezavantajı, aynı anda birçok değişken atamanız gerektiğidir. Bu, büyük sorguları takip etmeyi zorlaştırır. MySQL Native Driver (mysqlnd) kuruluysa, tek yapmanız gereken get_result kullanmaktır. .net/manual/en/mysqli-stmt.get-result.php).

Nesneye yönelik stil

$result = $stmt->get_result();

Prosedür stili

$result = mysqli_stmt_get_result($stmt);

Bununla çalışmak çok daha kolay çünkü şimdi bir mysqli_result nesnesi alıyoruz. Bu, [mysqli_query’nin döndürdüğü](https://www.wikiod.com/tr/php/php-mysqli#MySQLi sorgusu) aynı nesnedir. Bu, verilerinizi almak için bir [normal sonuç döngüsü](https://www.wikiod.com/tr/php/php-mysqli#MySQLi sonuçları arasında dolaşın) kullanabileceğiniz anlamına gelir.

Ya mysqlnd yükleyemezsem? #

Durum buysa, @Sophivorus sizi bu harika yanıtı ele aldı.

Bu işlev, sunucuya kurulmadan “get_result” görevini gerçekleştirebilir. Sadece sonuçlar arasında dolaşır ve ilişkisel bir dizi oluşturur

function get_result(\mysqli_stmt $statement)
{
    $result = array();
    $statement->store_result();
    for ($i = 0; $i < $statement->num_rows; $i++)
    {
        $metadata = $statement->result_metadata();
        $params = array();
        while ($field = $metadata->fetch_field())
        {
            $params[] = &$result[$i][$field->name];
        }
        call_user_func_array(array($statement, 'bind_result'), $params);
        $statement->fetch();
    }
    return $result;
}

Daha sonra, tıpkı mysqli_fetch_assoc() kullanıyormuşuz gibi, bunun gibi sonuçlar elde etmek için işlevi kullanabiliriz.

<?php
$query = $mysqli->prepare("SELECT * FROM users WHERE forename LIKE ?");
$condition = "J%";
$query->bind_param("s", $condition);
$query->execute();
$result = get_result($query);

while ($row = array_shift($result)) {
    echo $row["id"] . ' - ' . $row["forename"] . ' ' . $row["surname"] . '<br>';
}

Yüklenmesi gerekmemesi dışında, mysqlnd sürücüsünü kullanıyormuşsunuz gibi aynı çıktıya sahip olacaktır. Söz konusu sürücüyü sisteminize yükleyemiyorsanız, bu çok kullanışlıdır. Sadece bu çözümü uygulayın.

MySQLi Kimlik Ekle #

[AUTO_INCREMENT][3] sütunlu bir tabloda INSERT sorgusu tarafından oluşturulan son kimliği alın.

Nesneye Yönelik Stil

$id = $conn->insert_id;

Prosedür Stili

$id = mysqli_insert_id($conn);

Bağlantıda önceki sorgu yoksa veya sorgu bir AUTO_INCREMENT değerini güncellemediyse sıfır döndürür.

Satırları güncellerken kimlik ekleyin

Normalde bir “UPDATE” ifadesi bir ekleme kimliği döndürmez, çünkü “AUTO_INCREMENT” kimliği yalnızca yeni bir satır kaydedildiğinde (veya eklendiğinde) döndürülür. Yeni kimliğe güncelleme yapmanın bir yolu, güncelleme için INSERT ... ON DUPLICATE KEY UPDATE sözdizimini kullanmaktır.

İzlenecek örnekler için kurulum:

CREATE TABLE iodku (
    id INT AUTO_INCREMENT NOT NULL,
    name VARCHAR(99) NOT NULL,
    misc INT NOT NULL,
    PRIMARY KEY(id),
    UNIQUE(name)
) ENGINE=InnoDB;

INSERT INTO iodku (name, misc)
    VALUES
    ('Leslie', 123),
    ('Sally', 456);
Query OK, 2 rows affected (0.00 sec)
Records: 2  Duplicates: 0  Warnings: 0
+----+--------+------+
| id | name   | misc |
+----+--------+------+
|  1 | Leslie |  123 |
|  2 | Sally  |  456 |
+----+--------+------+

Bir “güncelleme” gerçekleştiren IODKU ve ilgili “id"yi alan “LAST_INSERT_ID()” durumu:

$sql = "INSERT INTO iodku (name, misc)
    VALUES
    ('Sally', 3333)            -- should update
    ON DUPLICATE KEY UPDATE    -- `name` will trigger "duplicate key"
    id = LAST_INSERT_ID(id),
    misc = VALUES(misc)";
$conn->query($sql);
$id = $conn->insert_id;        -- picking up existing value (2)

IODKU’nun bir “insert” gerçekleştirdiği ve “LAST_INSERT_ID()” öğesinin yeni “id"yi aldığı durum:

$sql = "INSERT INTO iodku (name, misc)
    VALUES
    ('Dana', 789)            -- Should insert
    ON DUPLICATE KEY UPDATE
    id = LAST_INSERT_ID(id),
    misc = VALUES(misc);
$conn->query($sql);
$id = $conn->insert_id;      -- picking up new value (3)

Ortaya çıkan tablo içeriği:

SELECT * FROM iodku;
+----+--------+------+
| id | name   | misc |
+----+--------+------+
|  1 | Leslie |  123 |
|  2 | Sally  | 3333 |  -- IODKU changed this
|  3 | Dana   |  789 |  -- IODKU added this
+----+--------+------+
