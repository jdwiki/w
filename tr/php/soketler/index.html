TCP istemci soketi #

TCP (İletim Kontrol Protokolü) kullanan bir soket oluşturma #

$socket = socket_create(AF_INET, SOCK_STREAM, SOL_TCP);

Soketin başarıyla oluşturulduğundan emin olun. ‘onSocketFailure’ işlevi, bu konudaki https://www.wikiod.com/tr/php/soketler#Soket hatalarını işleme örneğinden gelir.

if(!is_resource($socket)) onSocketFailure("Failed to create socket");

Soketi belirtilen adrese bağlayın #

Bağlantı başarısız olursa ikinci satır düzgün bir şekilde başarısız olur.

socket_connect($socket, "chat.stackoverflow.com", 6667)
        or onSocketFailure("Failed to connect to chat.stackoverflow.com:6667", $socket);

Sunucuya veri gönderme #

“socket_write” işlevi, bir soket aracılığıyla bayt gönderir. PHP’de bir bayt dizisi, normalde kodlamaya duyarlı olmayan bir dize ile temsil edilir.

socket_write($socket, "NICK Alice\r\nUSER alice 0 * :Alice\r\n");

Sunucudan veri alma #

Aşağıdaki kod parçası, “socket_read” işlevini kullanarak sunucudan bazı veriler alır.

Üçüncü parametre olarak PHP_NORMAL_READ geçmek, bir \r/\n bayta kadar okur ve bu bayt dönüş değerine dahil edilir.

Aksine, ‘PHP_BINARY_READ’ geçmek, akıştan gerekli miktarda veriyi okur.

Önceden “socket_set_nonblock” çağrıldıysa ve “PHP_BINARY_READ” kullanıldıysa, “socket_read” hemen “false” döndürür. Aksi takdirde yöntem, yeterli veri (ikinci parametredeki uzunluğa ulaşmak veya bir satır sonuna ulaşmak için) alınana veya soket kapatılana kadar bloke eder.

Bu örnek, sözde bir IRC sunucusundan veri okur.

while(true) {
    // read a line from the socket
    $line = socket_read($socket, 1024, PHP_NORMAL_READ);
    if(substr($line, -1) === "\r") {
        // read/skip one byte from the socket
        // we assume that the next byte in the stream must be a \n.
        // this is actually bad in practice; the script is vulnerable to unexpected values
        socket_read($socket, 1, PHP_BINARY_READ);
    }

    $message = parseLine($line);
    if($message->type === "QUIT") break;
}

Soketin kapatılması #

Soketin kapatılması, soketi ve onunla ilişkili kaynakları serbest bırakır.

socket_close($socket);

TCP sunucu soketi #

Soket oluşturma #

TCP’yi kullanan bir soket oluşturun. İstemci soketi oluşturmakla aynıdır.

$socket = socket_create(AF_INET, SOCK_STREAM, SOL_TCP);

Soket bağlama #

Belirli bir bağlantı noktası (parametre 3) için belirli bir ağdan (parametre 2) bağlantıları sokete bağlayın.

İkinci parametre genellikle tüm ağlardan bağlantıyı kabul eden “0.0.0.0”`dır. Ayrıca

‘socket_bind’ kaynaklı hataların yaygın bir nedeni, belirtilen adresin zaten başka bir işleme bağlı olmasıdır. Diğer işlemler genellikle öldürülür (genellikle kritik işlemlerin yanlışlıkla öldürülmesini önlemek için manuel olarak), böylece soketler serbest bırakılır.

socket_bind($socket, "0.0.0.0", 6667) or onSocketFailure("Failed to bind to 0.0.0.0:6667");

Dinlemek için bir soket ayarlayın #

Socket_listen’i kullanarak soketin gelen bağlantıları dinlemesini sağlayın. İkinci parametre, kabul edilmeden önce kuyruğa alınmasına izin verecek maksimum bağlantı sayısıdır.

socket_listen($socket, 5);

Bağlantıyı işleme #

TCP sunucusu aslında alt bağlantıları işleyen bir sunucudur. “socket_accept” yeni bir alt bağlantı oluşturur.

$conn = socket_accept($socket);

‘socket_accept’ bağlantısı için veri aktarımı, https://www.wikiod.com/tr/php/soketler#TCP istemci soketi için olanla aynıdır.

Bu bağlantının kapatılması gerektiğinde, doğrudan socket_close($conn); öğesini çağırın. Bu, orijinal TCP sunucu soketini etkilemeyecektir.

Sunucuyu Kapatmak #

Öte yandan, sunucu artık kullanılmadığında socket_close($socket); çağrılmalıdır. Bu, TCP adresini de serbest bırakacak ve diğer işlemlerin adrese bağlanmasına izin verecektir.

UDP sunucu soketi #

UDP (kullanıcı datagram protokolü) sunucusu, TCP’den farklı olarak akış tabanlı değildir. Paket tabanlıdır, yani bir istemci, sunucuya “paket” adı verilen birimler halinde veri gönderir ve istemci, istemcileri adreslerine göre tanımlar. Aynı istemciden gönderilen farklı paketleri ilişkilendiren yerleşik bir işlev yoktur (aynı istemciden gelen verilerin “socket_accept” tarafından oluşturulan belirli bir kaynak tarafından işlendiği TCP’nin aksine). Her UDP paketi geldiğinde yeni bir TCP bağlantısının kabul edilip kapatıldığı düşünülebilir.

UDP sunucu soketi oluşturma #

$socket = socket_create(AF_INET, SOCK_DGRAM, SOL_UDP);

Soketi bir adrese bağlama #

Parametreler, bir TCP sunucusu için olanlarla aynıdır.

socket_bind($socket, "0.0.0.0", 9000) or onSocketFailure("Failed to bind to 0.0.0.0:9000", $socket);

Paket gönderme #

Bu satır, bir UDP paketindeki “$veri"yi “$address”:"$port"a gönderir.

socket_sendto($socket, $data, strlen($data), 0, $address, $port);

Paket alma #

Aşağıdaki kod parçası, UDP paketlerini istemci dizinli bir şekilde yönetmeye çalışır.

$clients = [];
while (true){
    socket_recvfrom($socket, $buffer, 32768, 0, $ip, $port) === true
            or onSocketFailure("Failed to receive packet", $socket);
    $address = "$ip:$port";
    if (!isset($clients[$address])) $clients[$address] = new Client();
    $clients[$address]->handlePacket($buffer);
}

Sunucuyu Kapatmak #

“socket_close” UDP sunucu soket kaynağında kullanılabilir. Bu, UDP adresini serbest bırakacak ve diğer işlemlerin bu adrese bağlanmasına izin verecektir.

Soket hatalarını işleme #

socket_last_error, soket uzantısından son hatanın hata kimliğini almak için kullanılabilir.

“socket_strerror”, kimliği insan tarafından okunabilir dizelere dönüştürmek için kullanılabilir.

function onSocketFailure(string $message, $socket = null) {
    if(is_resource($socket)) {
        $message .= ": " . socket_strerror(socket_last_error($socket));
    }
    die($message);
}
