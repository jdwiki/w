PDO (PHP Veri Nesneleri) uzantısı, geliştiricilerin çok sayıda farklı türde veritabanlarına bağlanmasına ve bunlara karşı tek tip, nesne yönelimli sorgular yürütmesine olanak tanır. tavır.

Sözdizimi #

  • PDO::LastInsertId()
  • PDO::LastInsertId($columnName) // bazı sürücülerin sütun adına ihtiyacı var

Uyarı lastInsertId() kullanırken istisnaları kontrol etmeyi unutmayın. Aşağıdaki hatayı verebilir:

SQLSTATE IM001 : Sürücü bu işlevi desteklemiyor

Bu yöntemi kullanarak istisnaları nasıl düzgün bir şekilde kontrol etmeniz gerektiği aşağıda açıklanmıştır:

// Retrieving the last inserted id
$id = null;

try {
    $id = $pdo->lastInsertId(); // return value is an integer    
}
catch( PDOException $e ) {
    echo $e->getMessage();
}

Parametreli Sorgularla SQL Injection’ı Önleme #

SQL enjeksiyonu, kötü niyetli bir kullanıcının SQL sorgusunu değiştirmesine ve buna istenmeyen komutlar eklemesine izin veren bir saldırı türüdür. Örneğin, aşağıdaki kod savunmasızdır:

// Do not use this vulnerable code!
$sql = 'SELECT name, email, user_level FROM users WHERE userID = ' . $_GET['user'];
$conn->query($sql);

Bu, bu betiğin herhangi bir kullanıcısının veritabanımızı temelde istediği gibi değiştirmesine izin verir. Örneğin, aşağıdaki sorgu dizesini göz önünde bulundurun:

page.php?user=0;%20TRUNCATE%20TABLE%20users;

Bu, örnek sorgumuzun şöyle görünmesini sağlar

SELECT name, email, user_level FROM users WHERE userID = 0; TRUNCATE TABLE users;

Bu uç bir örnek olsa da (çoğu SQL enjeksiyon saldırısı verileri silmeyi amaçlamaz ve çoğu PHP sorgu yürütme işlevi çoklu sorguyu desteklemez), bu, SQL enjeksiyon saldırısının dikkatsiz bir derleme ile nasıl mümkün hale getirilebileceğinin bir örneğidir. sorgu. Ne yazık ki, bunun gibi saldırılar çok yaygındır ve verilerini korumak için uygun önlemleri almayan kodlayıcılar nedeniyle oldukça etkilidir.

SQL enjeksiyonunun oluşmasını önlemek için hazırlanmış ifadeler önerilen çözümdür. Kullanıcı verilerini doğrudan sorguya birleştirmek yerine, bunun yerine bir yer tutucu kullanılır. Veriler daha sonra ayrı olarak gönderilir, bu da SQL motorunun bir dizi talimat için kullanıcı verilerini karıştırma şansı olmadığı anlamına gelir.

Buradaki konu PDO olsa da, lütfen PHP MySQLi uzantısının [hazırlanan ifadeleri desteklediğini] unutmayın (https://www.wikiod.com/tr/php/php-mysqli#MySQLi’de hazırlanan ifadeler)

PDO iki tür yer tutucuyu destekler (yer tutucular sütun veya tablo adları için kullanılamaz, yalnızca değerler için kullanılabilir):

  1. Adlandırılmış yer tutucular. İki nokta üst üste (:), ardından belirgin bir ad (ör. :user)

    // using named placeholders
$sql = 'SELECT name, email, user_level FROM users WHERE userID = :user';
$prep = $conn->prepare($sql);
$prep->execute(['user' => $_GET['user']]); // associative array
$result = $prep->fetchAll();

  2. ? olarak gösterilen geleneksel SQL konumsal yer tutucuları:

    // using question-mark placeholders
$sql = 'SELECT name, user_level FROM users WHERE userID = ? AND user_level = ?';
$prep = $conn->prepare($sql);
$prep->execute([$_GET['user'], $_GET['user_level']]); // indexed array
$result = $prep->fetchAll();

Tablo veya sütun adlarını dinamik olarak değiştirmeniz gerekirse, bunun kendi güvenlik riskleriniz olduğunu ve kötü bir uygulama olduğunu bilin. Yine de, dize birleştirme ile yapılabilir. Bu tür sorguların güvenliğini artırmanın bir yolu, bir izin verilen değerler tablosu ayarlamak ve birleştirmek istediğiniz değeri bu tabloyla karşılaştırmaktır.

Bağlantı karakter kümesini yalnızca DSN aracılığıyla ayarlamanın önemli olduğunu unutmayın, aksi takdirde uygulamanız bazı garip kodlamalar kullanılıyorsa [belirsiz bir güvenlik açığı][öykünme-hazırlama-enjeksiyon] eğilimli olabilir. 5.3.6’dan önceki PDO sürümleri için, DSN aracılığıyla karakter kümesi ayarlamak mevcut değildir ve bu nedenle tek seçenek, oluşturulduktan hemen sonra bağlantıda “PDO::ATTR_EMULATE_PREPARES” niteliğini “yanlış” olarak ayarlamaktır.

$conn->setAttribute(PDO::ATTR_EMULATE_PREPARES, false);

Bu, PDO’nun yalnızca öykünmek yerine temel alınan DBMS’nin yerel olarak hazırlanmış deyimlerini kullanmasına neden olur.

Ancak, PDO’nun MySQL’in yerel olarak hazırlayamayacağı ifadeleri taklit etmek için [sessizce geri dönüş yapacağınıpdo-geri dönüş yapacağını unutmayın: Yapabilecekleri kılavuzda listelenmiştir (kaynak ).

Temel PDO Bağlantısı ve Alma #

PHP 5.0’dan beri PDO bir veritabanı erişim katmanı olarak mevcuttur. Veritabanından bağımsızdır ve bu nedenle aşağıdaki bağlantı örneği kodu, yalnızca DSN’yi değiştirerek [desteklenen veritabanlarından2 herhangi biri için çalışmalıdır.

// First, create the database handle

//Using MySQL (connection via local socket):
$dsn = "mysql:host=localhost;dbname=testdb;charset=utf8";

//Using MySQL (connection via network, optionally you can specify the port too):
//$dsn = "mysql:host=127.0.0.1;port=3306;dbname=testdb;charset=utf8";

//Or Postgres
//$dsn = "pgsql:host=localhost;port=5432;dbname=testdb;";

//Or even SQLite
//$dsn = "sqlite:/path/to/database"

$username = "user";
$password = "pass";
$db = new PDO($dsn, $username, $password);

// setup PDO to throw an exception if an invalid query is provided
$db->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

// Next, let's prepare a statement for execution, with a single placeholder
$query = "SELECT * FROM users WHERE class = ?";
$statement = $db->prepare($query);

// Create some parameters to fill the placeholders, and execute the statement
$parameters = [ "221B" ];
$statement->execute($parameters);

// Now, loop through each record as an associative array
while ($row = $statement->fetch(PDO::FETCH_ASSOC)) {
    do_stuff($row);
}

prepare işlevi, sorgu dizesinden bir PDOStatement nesnesi oluşturur. Sorgunun yürütülmesi ve sonuçların alınması bu döndürülen nesne üzerinde gerçekleştirilir. Bir arıza durumunda, işlev ya “yanlış” döndürür ya da bir “istisna” (PDO bağlantısının nasıl yapılandırıldığına bağlı olarak) atar.

PDO ile Veritabanı İşlemleri #

Veritabanı işlemleri, bir dizi veri değişikliğinin ancak her ifadenin başarılı olması durumunda kalıcı olmasını sağlar. Bir işlem sırasındaki herhangi bir sorgu veya kod hatası yakalanabilir ve ardından denenen değişiklikleri geri alma seçeneğiniz vardır.

PDO, işlemleri başlatmak, gerçekleştirmek ve geri almak için basit yöntemler sağlar.

$pdo = new PDO(
    $dsn, 
    $username, 
    $password, 
    array(PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION)
);

try {
    $statement = $pdo->prepare("UPDATE user SET name = :name");

    $pdo->beginTransaction();

    $statement->execute(["name"=>'Bob']);
    $statement->execute(["name"=>'Joe']);

    $pdo->commit();
} 
catch (\Exception $e) {
    if ($pdo->inTransaction()) {
        $pdo->rollback();
        // If we got here our two data updates are not in the database
    }
    throw $e;
}

Bir işlem sırasında, yapılan herhangi bir veri değişikliği sadece aktif bağlantı tarafından görülebilir. ‘SEÇ’ ifadeleri, henüz veritabanına bağlanmamış olsalar bile, değiştirilmiş değişiklikleri döndürür.

Not: İşlem desteğiyle ilgili ayrıntılar için veritabanı satıcı belgelerine bakın. Bazı sistemler işlemleri hiç desteklemez. Bazıları iç içe işlemleri desteklerken diğerleri desteklemez.

PDO ile İşlemleri Kullanan Pratik Örnek

Aşağıdaki bölümde, işlemlerin kullanımının veritabanının tutarlılığını sağladığı pratik bir gerçek dünya örneği gösterilmektedir.

Aşağıdaki senaryoyu hayal edin, diyelim ki bir e-ticaret sitesi için alışveriş sepeti oluşturuyorsunuz ve siparişleri iki veritabanı tablosunda tutmaya karar verdiniz. Biri “order_id”, “name”, “address”, “telephone” ve “created_at” alanlarıyla “siparişler” adında. Ve ‘order_id’, ‘product_id’ ve ‘miktar’ alanlarıyla birlikte ‘orders_products’ adlı ikincisi. İlk tablo siparişin meta verilerini, ikincisi ise sipariş edilen gerçek ürünleri içerir.

Veritabanına yeni bir sipariş ekleme

Veritabanına yeni bir sipariş eklemek için iki şey yapmanız gerekir. Öncelikle, siparişin meta verilerini (‘ad’, ‘adres’, vb.) Ardından, siparişe dahil olan ürünlerin her biri için ‘sipariş_ürünleri’ tablosuna bir kayıt ‘EKLEME’ yapmanız gerekir.

Bunu, aşağıdakine benzer bir şey yaparak yapabilirsiniz:

// Insert the metadata of the order into the database
$preparedStatement = $db->prepare(
    'INSERT INTO `orders` (`name`, `address`, `telephone`, `created_at`)
     VALUES (:name, :address, :telephone, :created_at)'
);

$preparedStatement->execute([
    'name' => $name,
    'address' => $address,
    'telephone' => $telephone,
    'created_at' => time(),
]);

// Get the generated `order_id`
$orderId = $db->lastInsertId();

// Construct the query for inserting the products of the order
$insertProductsQuery = 'INSERT INTO `orders_products` (`order_id`, `product_id`, `quantity`) VALUES';

$count = 0;
foreach ( $products as $productId => $quantity ) {
    $insertProductsQuery .= ' (:order_id' . $count . ', :product_id' . $count . ', :quantity' . $count . ')';
    
    $insertProductsParams['order_id' . $count] = $orderId;
    $insertProductsParams['product_id' . $count] = $productId;
    $insertProductsParams['quantity' . $count] = $quantity;
    
    ++$count;
}

// Insert the products included in the order into the database
$preparedStatement = $db->prepare($insertProductsQuery);
$preparedStatement->execute($insertProductsParams);

Bu, beklenmedik bir şey gerçekleşene ve herhangi bir nedenle ikinci “INSERT” sorgusu başarısız olana kadar veritabanına yeni bir sipariş eklemek için harika çalışır. Böyle bir durumda, ‘siparişler’ tablosunda kendisiyle ilişkili hiçbir ürün olmayacak yeni bir sipariş alacaksınız. Neyse ki, düzeltme çok basit, tek yapmanız gereken sorguları tek bir veritabanı işlemi şeklinde yapmak.

Bir işlemle veritabanına yeni bir sipariş ekleme

‘PDO’ kullanarak bir işlem başlatmak için tek yapmanız gereken, veritabanınıza herhangi bir sorgu yürütmeden önce ‘beginTransaction’ yöntemini çağırmak. Ardından ‘INSERT’ ve/veya ‘UPDATE’ sorgularını çalıştırarak verilerinizde istediğiniz değişikliği yaparsınız. Ve son olarak, değişiklikleri kalıcı kılmak için ‘PDO’ nesnesinin ‘commit’ yöntemini çağırırsınız. Siz “taahhüt” yöntemini çağırana kadar, bu noktaya kadar verilerinizde yaptığınız her değişiklik henüz kalıcı değildir ve yalnızca “PDO” nesnesinin “geri alma” yöntemini çağırarak kolayca geri alınabilir.

Aşağıdaki örnekte, aynı zamanda verilerin tutarlılığını sağlarken, veritabanına yeni bir sipariş eklemek için işlemlerin kullanımı gösterilmektedir. İki sorgudan biri başarısız olursa, tüm değişiklikler geri alınır.

// In this example we are using MySQL but this applies to any database that has support for transactions
$db = new PDO('mysql:host=' . $host . ';dbname=' . $dbname . ';charset=utf8', $username, $password);    

// Make sure that PDO will throw an exception in case of error to make error handling easier
$db->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

try {
    // From this point and until the transaction is being committed every change to the database can be reverted
    $db->beginTransaction();    
    
    // Insert the metadata of the order into the database
    $preparedStatement = $db->prepare(
        'INSERT INTO `orders` (`order_id`, `name`, `address`, `created_at`)
         VALUES (:name, :address, :telephone, :created_at)'
    );
    
    $preparedStatement->execute([
        'name' => $name,
        'address' => $address,
        'telephone' => $telephone,
        'created_at' => time(),
    ]);
    
    // Get the generated `order_id`
    $orderId = $db->lastInsertId();

    // Construct the query for inserting the products of the order
    $insertProductsQuery = 'INSERT INTO `orders_products` (`order_id`, `product_id`, `quantity`) VALUES';
    
    $count = 0;
    foreach ( $products as $productId => $quantity ) {
        $insertProductsQuery .= ' (:order_id' . $count . ', :product_id' . $count . ', :quantity' . $count . ')';
        
        $insertProductsParams['order_id' . $count] = $orderId;
        $insertProductsParams['product_id' . $count] = $productId;
        $insertProductsParams['quantity' . $count] = $quantity;
        
        ++$count;
    }
    
    // Insert the products included in the order into the database
    $preparedStatement = $db->prepare($insertProductsQuery);
    $preparedStatement->execute($insertProductsParams);
    
    // Make the changes to the database permanent
    $db->commit();
}
catch ( PDOException $e ) { 
    // Failed to insert the order into the database so we rollback any changes
    $db->rollback();
    throw $e;
}

PDO: MySQL/MariaDB sunucusuna bağlanma #

Altyapınıza bağlı olarak MySQL/MariaDB sunucusuna bağlanmanın iki yolu vardır.

Standart (TCP/IP) bağlantı #

$dsn = 'mysql:dbname=demo;host=server;port=3306;charset=utf8';
$connection = new \PDO($dsn, $username, $password);

// throw exceptions, when SQL error is caused
$connection->setAttribute(\PDO::ATTR_ERRMODE, \PDO::ERRMODE_EXCEPTION);
// prevent emulation of prepared statements
$connection->setAttribute(\PDO::ATTR_EMULATE_PREPARES, false);

PDO, eski MySQL sunucu sürümleriyle (hazır deyimler için desteği olmayan) uyumlu olacak şekilde tasarlandığından, öykünmeyi açıkça devre dışı bırakmanız gerekir. Aksi takdirde, genellikle hazırlanmış beyanlar kullanılarak sağlanan ek enjeksiyon önleme avantajlarını kaybedersiniz.

Akılda tutmanız gereken bir diğer tasarım uzlaşması, varsayılan hata işleme davranışıdır. Başka şekilde yapılandırılmamışsa, PDO herhangi bir SQL hatası belirtisi göstermez.

Bunu “istisna modu"na ayarlamanız şiddetle tavsiye edilir, çünkü bu size kalıcılık soyutlamaları yazarken ek işlevsellik kazandırır (örneğin: “EŞSİZ” kısıtlamasını ihlal ederken bir istisna olması).

Soket bağlantısı #

$dsn = 'mysql:unix_socket=/tmp/mysql.sock;dbname=demo;charset=utf8';
$connection = new \PDO($dsn, $username, $password);

// throw exceptions, when SQL error is caused
$connection->setAttribute(\PDO::ATTR_ERRMODE, \PDO::ERRMODE_EXCEPTION);
// prevent emulation of prepared statements
$connection->setAttribute(\PDO::ATTR_EMULATE_PREPARES, false);

Unix benzeri sistemlerde, ana bilgisayar adı “localhost” ise, sunucuya bağlantı bir etki alanı soketi üzerinden yapılır.

PDO: Bir sorgudan etkilenen satır sayısını alın #

PDO sınıfının bir örneği olan $db ile başlıyoruz. Bir sorguyu yürüttükten sonra, genellikle bundan etkilenen satır sayısını belirlemek isteriz. ‘PDOStatement’ın ‘rowCount()’ yöntemi iyi çalışacaktır:

$query = $db->query("DELETE FROM table WHERE name = 'John'");
$count = $query->rowCount();

echo "Deleted $count rows named John";

NOT: Bu yöntem yalnızca INSERT, DELETE ve UPDATE ifadelerinden etkilenen satır sayısını belirlemek için kullanılmalıdır. Bu yöntem SELECT deyimleri için de işe yarayabilir, ancak tüm veritabanlarında tutarlı değildir.

PDO::lastInsertId() #

Veritabanı tablonuza yeni eklediğiniz bir satır için otomatik artan kimlik değerini alma ihtiyacını sık sık bulabilirsiniz. Bunu lastInsertId() yöntemiyle başarabilirsiniz.

// 1. Basic connection opening (for MySQL)
$host = 'localhost';
$database = 'foo';
$user = 'root'
$password = '';
$dsn = "mysql:host=$host;dbname=$database;charset=utf8";
$pdo = new PDO($dsn, $user, $password);

// 2. Inserting an entry in the hypothetical table 'foo_user'
$query = "INSERT INTO foo_user(pseudo, email) VALUES ('anonymous', 'anonymous@example.com')";
$query_success = $pdo->query($query);

// 3. Retrieving the last inserted id
$id = $pdo->lastInsertId(); // return value is an integer

Postgresql ve Oracle’da, o anda eklenen / değiştirilen satırların belirtilen sütunlarını döndüren RETURNING Anahtar Kelimesi vardır. İşte bir giriş eklemek için örnek:

// 1. Basic connection opening (for PGSQL)
$host = 'localhost';
$database = 'foo';
$user = 'root'
$password = '';
$dsn = "pgsql:host=$host;dbname=$database;charset=utf8";
$pdo = new PDO($dsn, $user, $password);

// 2. Inserting an entry in the hypothetical table 'foo_user'
$query = "INSERT INTO foo_user(pseudo, email) VALUES ('anonymous', 'anonymous@example.com') RETURNING id";
$statement = $pdo->query($query);

// 3. Retrieving the last inserted id
$id = $statement->fetchColumn();  // return the value of the id column of the new row in foo_user
