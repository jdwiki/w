Sözdizimi #

  • dize serileştirme (karışık $değer)

Parametreler #

parametre Ayrıntılar
değer Serileştirilecek değer. serialize(), resource tipi hariç tüm türleri işler. Hatta kendisine referanslar içeren dizileri serialize() yapabilirsiniz. Serileştirdiğiniz dizi/nesne içindeki dairesel referanslar da saklanacaktır. Diğer tüm referanslar kaybolacaktır. Nesneleri serileştirirken PHP, serileştirmeden önce __sleep() üye işlevini çağırmaya çalışır. Bu, nesnenin serileştirilmeden önce herhangi bir son dakika temizliği vb. yapmasına izin vermek içindir. Benzer şekilde, nesne unserialize() kullanılarak geri yüklendiğinde __wakeup() üye işlevi çağrılır. Nesnenin özel üyeleri, üye adının önüne sınıf adını ekler; korumalı üyelerin başına bir ‘*’ eklenir. Başa eklenen bu değerlerin her iki tarafında boş baytlar bulunur.

Serileştirme, aşağıdaki dize yapılarını kullanır:

[..] yer tutuculardır.

Tip Yapı
dize s:[dizenin boyutu]:[değer]
Tamsayı i:[değer]
Çift Kişilik d:[değer]
Boole `b:[değer (doğru = 1 ve yanlış = 0)]’
Boş “N”
Nesne O:[nesne adı boyutu]:[nesne adı]:[nesne boyutu]:{[özellik adı dize tanımı]:[özellik değeri tanımı];(her özellik için tekrarlanır)}
dizi a:[dizi boyutu]:{[anahtar tanımı];[değer tanımı];(her anahtar değer çifti için tekrarlanır)}

Farklı türlerin seri hale getirilmesi #

Bir değerin depolanabilir bir temsilini oluşturur.

Bu, PHP değerlerini türlerini ve yapılarını kaybetmeden depolamak veya aktarmak için kullanışlıdır.

Serileştirilmiş dizeyi yeniden bir PHP değerine dönüştürmek için unserialize() kullanın.

Bir diziyi seri hale getirme #

$string = "Hello world";    
echo serialize($string);

// Output: 
// s:11:"Hello world";

Bir çiftin seri hale getirilmesi #

$double = 1.5;    
echo serialize($double);

// Output: 
// d:1.5;

Bir şamandırayı seri hale getirme #

Float, çiftler olarak serileştirilir.

Bir tamsayıyı seri hale getirme #

$integer = 65;    
echo serialize($integer);

// Output: 
// i:65;

Bir boole’u seri hale getirme #

$boolean = true;    
echo serialize($boolean);

// Output: 
// b:1;

$boolean = false;    
echo serialize($boolean);

// Output: 
// b:0;

Serileştirme null #

$null = null;    
echo serialize($null);

// Output: 
// N;

Bir diziyi seri hale getirme #

$array = array(
    25,
    'String',
    'Array'=> ['Multi Dimension','Array'],
    'boolean'=> true,
    'Object'=>$obj, // $obj from above Example
    null,
    3.445
); 


// This will throw Fatal Error
// $array['function'] = function() { return "function"; };

echo serialize($array);

// Output:
// a:7:{i:0;i:25;i:1;s:6:"String";s:5:"Array";a:2:{i:0;s:15:"Multi Dimension";i:1;s:5:"Array";}s:7:"boolean";b:1;s:6:"Object";O:3:"abc":1:{s:1:"i";i:1;}i:2;N;i:3;d:3.4449999999999998;}

Bir nesneyi seri hale getirme #

Ayrıca Nesneleri seri hale getirebilirsiniz.

Nesneleri serileştirirken PHP, serileştirmeden önce __sleep() üye işlevini çağırmaya çalışır. Bu, nesnenin serileştirilmeden önce herhangi bir son dakika temizliği vb. yapmasına izin vermek içindir. Benzer şekilde, nesne unserialize() kullanılarak geri yüklendiğinde __wakeup() üye işlevi çağrılır.

class abc {
    var $i = 1;
    function foo() {
        return 'hello world';
    }
}

$object = new abc(); 
echo serialize($object);

// Output:
// O:3:"abc":1:{s:1:"i";i:1;}

Closure’ların serileştirilemeyeceğini unutmayın: #

$function = function () { echo 'Hello World!'; };
$function(); // prints "hello!"

$serializedResult = serialize($function);  // Fatal error: Uncaught exception 'Exception' with message 'Serialization of 'Closure' is not allowed'

Serileştirme ile ilgili Güvenlik Sorunları #

Kullanıcı girişinden gelen verileri seri hale getirmek için “serileştirme” işlevini kullanmak tehlikeli olabilir.

php.net’ten bir uyarı

Uyarı Güvenilmeyen kullanıcı girdisini unserialize() işlevine iletmeyin. Serileştirme, aşağıdakilerden dolayı kodun yüklenmesine ve yürütülmesine neden olabilir: nesne başlatma ve otomatik yükleme ve kötü niyetli bir kullanıcı bundan yararlanmak için. gibi güvenli, standart bir veri değişim formatı kullanın. Geçmeniz gerekiyorsa JSON (json_decode() ve json_encode() aracılığıyla) kullanıcıya serileştirilmiş veriler.

Olası Saldırılar #

  • PHP Nesne Enjeksiyonu

PHP Nesne Enjeksiyonu #

PHP Object Injection, bir saldırganın bağlama bağlı olarak Code Injection, SQL Injection, Path Traversal ve Application Denial of Service gibi farklı türde kötü niyetli saldırılar gerçekleştirmesine olanak verebilecek uygulama düzeyinde bir güvenlik açığıdır. Güvenlik açığı, kullanıcı tarafından sağlanan girdi unserialize() PHP işlevine geçirilmeden önce uygun şekilde temizlenmediğinde ortaya çıkar. PHP nesne serileştirmeye izin verdiğinden, saldırganlar geçici serileştirilmiş dizeleri savunmasız bir unserialize() çağrısına geçirebilir ve bu da uygulama kapsamına rastgele bir PHP nesnesi/nesnesi enjeksiyonu ile sonuçlanır.

Bir PHP Nesne Enjeksiyonu güvenlik açığından başarıyla yararlanmak için iki koşulun karşılanması gerekir:

  • Uygulama, kötü niyetli saldırılar gerçekleştirmek veya bir “POP zinciri” başlatmak için kullanılabilecek bir PHP sihirli yöntemini (’__wakeup’ veya ‘__destruct’ gibi) uygulayan bir sınıfa sahip olmalıdır.
  • Saldırı sırasında kullanılan tüm sınıflar, savunmasız ‘unserialize()’ çağrıldığında bildirilmelidir, aksi takdirde bu tür sınıflar için nesne otomatik yükleme desteklenmelidir.

Örnek 1 - Yol Geçiş Saldırısı

Aşağıdaki örnek, sömürülebilir bir ‘__destruct’ yöntemine sahip bir PHP sınıfını göstermektedir:

class Example1
{
   public $cache_file;

   function __construct()
   {
      // some PHP code...
   }

   function __destruct()
   {
      $file = "/var/www/cache/tmp/{$this->cache_file}";
      if (file_exists($file)) @unlink($file);
   }
}

// some PHP code...

$user_data = unserialize($_GET['data']);

// some PHP code...

Bu örnekte, bir saldırgan Path Traversal saldırısı yoluyla rastgele bir dosyayı silebilir, ör. aşağıdaki URL’yi istemek:

http://testsite.com/vuln.php?data=O:8:"Example1":1:{s:10:"cache_file";s:15:"../../index.php";}

Örnek 2 - Kod Enjeksiyonu saldırısı

Aşağıdaki örnek, kullanılabilir bir __wakeup yöntemine sahip bir PHP sınıfını göstermektedir:

class Example2
{
   private $hook;

   function __construct()
   {
      // some PHP code...
   }

   function __wakeup()
   {
      if (isset($this->hook)) eval($this->hook);
   }
}

// some PHP code...

$user_data = unserialize($_COOKIE['data']);

// some PHP code...

Bu örnekte bir saldırgan, aşağıdaki gibi bir HTTP isteği göndererek bir Kod Ekleme saldırısı gerçekleştirebilir:

GET /vuln.php HTTP/1.0
Host: testsite.com
Cookie: data=O%3A8%3A%22Example2%22%3A1%3A%7Bs%3A14%3A%22%00Example2%00hook%22%3Bs%3A10%3A%22phpinfo%28%29%3B%22%3B%7D
Connection: close

“veri” tanımlama bilgisi parametresinin aşağıdaki komut dosyası tarafından oluşturulduğu yer:

class Example2
{
   private $hook = "phpinfo();";
}

print urlencode(serialize(new Example2));
