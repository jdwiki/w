Tip Hokkabazlık Nedir? #

PHP gevşek yazılmış bir dildir. Bu, varsayılan olarak, bir ifadedeki işlenenlerin aynı (veya uyumlu) türde olmasını gerektirmediği anlamına gelir. Örneğin, bir dizeye bir sayı ekleyebilir ve çalışmasını bekleyebilirsiniz.

var_dump ("This is example number " . 1);

Çıktı olacaktır:

string(24) “Bu, 1 numaralı örnektir”

PHP bunu, uyumsuz değişken türlerini, istenen işlemin gerçekleşmesine izin veren türlere otomatik olarak çevirerek gerçekleştirir. Yukarıdaki durumda, 1 tamsayısını bir dizgeye çevirecektir, yani önceki dizge değişmezine birleştirilebileceği anlamına gelir. Buna tip hokkabazlığı denir. Bu PHP’nin çok güçlü bir özelliğidir fakat aynı zamanda farkında değilseniz sizi çok fazla saç çekmenize, hatta güvenlik sorunlarına bile yol açabilecek bir özelliktir.

Aşağıdakileri göz önünde bulundur:

if (1 == $variable) {
    // do something
}

Amaç, programcının bir değişkenin 1 değerine sahip olup olmadığını kontrol etmesi gibi görünüyor. Ancak bunun yerine $değişkeninin değeri “1 buçuk” olursa ne olur? Cevap sizi şaşırtabilir.

$variable = "1 and a half";
var_dump (1 == $variable);

Sonuç:

bool(doğru)

Bu neden oldu? Bunun nedeni, PHP’nin “1 buçuk” dizesinin bir tamsayı olmadığını, ancak tamsayı 1 ile karşılaştırmak için olması gerektiğini fark etmesidir. Başarısız olmak yerine, PHP, tür hokkabazlığını başlatır ve değişkeni tamsayı. Bunu, tamsayıya dönüştürülebilecek dizenin başlangıcındaki tüm karakterleri alarak ve bunları yayınlayarak yapar. Sayı olarak kabul edilemeyecek bir karakterle karşılaştığı anda durur. Bu nedenle “1 buçuk” tamsayı 1’e dönüştürülür.

Kabul, bu çok yapmacık bir örnek, ancak sorunu göstermeye hizmet ediyor. Sonraki birkaç örnek, gerçek yazılımda meydana gelen tür hokkabazlığından kaynaklanan hatalarla karşılaştığım bazı durumları kapsayacaktır.

Bir dosyadan okuma #

Bir dosyadan okurken, o dosyanın sonuna ne zaman ulaştığımızı bilmek isteriz. Dosyanın sonunda fgets()‘ın false döndürdüğünü bilerek, bunu bir döngü koşulu olarak kullanabiliriz. Ancak, son okumadan döndürülen veriler boolean ‘yanlış’ olarak değerlendirilen bir şey olursa, dosya okuma döngümüzün zamanından önce sonlandırılmasına neden olabilir.

$handle = fopen ("/path/to/my/file", "r");

if ($handle === false) {
    throw new Exception ("Failed to open file for reading");
}

while ($data = fgets($handle)) {
    echo ("Current file line is $data\n");
}

fclose ($handle);

Okunmakta olan dosya boş bir satır içeriyorsa, boş dize boolean “false” olarak değerlendirildiğinden “while” döngüsü o noktada sonlandırılacaktır.

Bunun yerine, katı eşitlik operatörlerini kullanarak boolean “yanlış” değerini açıkça kontrol edebiliriz:

while (($data = fgets($handle)) !== false) {
    echo ("Current file line is $data\n");
}

Bunun yapmacık bir örnek olduğuna dikkat edin; gerçek hayatta aşağıdaki döngüyü kullanırdık:

while (!feof($handle)) {
    $data = fgets($handle);
    echo ("Current file line is $data\n");
}

Veya her şeyi şununla değiştirin:

$filedata = file("/path/to/my/file");
foreach ($filedata as $data) {
    echo ("Current file line is $data\n");
}

1: https://www.wikiod.com/tr/php/operatorler#Karşılaştırma Operatörleri

Sürprizleri değiştir #

Switch ifadeleri, eşleşmeleri belirlemek için katı olmayan karşılaştırma kullanır. Bu, bazı kötü sürprizlere yol açabilir. Örneğin, aşağıdaki ifadeyi göz önünde bulundurun:

switch ($name) {
    case 'input 1':
        $mode = 'output_1';
        break;
    case 'input 2':
        $mode = 'output_2';
        break;
    default:
        $mode = 'unknown';
        break;
}

Bu çok basit bir ifadedir ve $name bir dize olduğunda beklendiği gibi çalışır, ancak aksi takdirde sorunlara neden olabilir. Örneğin, “$name” tamsayı “0” ise, karşılaştırma sırasında tür hokkabazlığı gerçekleşir. Ancak, dengelenen, switch ifadesindeki koşul değil, case ifadesindeki değişmez değerdir. “input 1”` dizesi, “0” tamsayısının giriş değeriyle eşleşen “0” tamsayısına dönüştürülür. Bunun sonucu, “0” tamsayı değeri sağlarsanız, ilk durum her zaman yürütülür.

Bu sorunun birkaç çözümü var:

Açık döküm #

Değer, karşılaştırmadan önce bir dizgeye [typecast][2] olabilir:

switch ((string)$name) {
...
}

Veya bir dize döndürdüğü bilinen bir işlev de kullanılabilir:

switch (strval($name)) {
...
}

Bu yöntemlerin her ikisi de değerin “case” ifadelerindeki değerle aynı türde olmasını sağlar.

“geçiş” yapmaktan kaçının #

Bir “if” ifadesi kullanmak, karşılaştırmanın nasıl yapıldığı üzerinde kontrol sağlayarak [katı karşılaştırma operatörleri][3] kullanmamıza izin verir:

if ($name === "input 1") {
    $mode = "output_1";
} elseif ($name === "input 2") {
    $mode = "output_2";
} else {
    $mode = "unknown";
}

[2]: https://www.wikiod.com/tr/php/turler#Tip Döküm [3]: https://www.wikiod.com/tr/php/operatorler#Karşılaştırma Operatörleri

Sıkı yazım #

PHP 7.0’dan beri, tip hokkabazlığının bazı zararlı etkileri sıkı tipleme ile hafifletilebilir. PHP, bu ‘declare’ ifadesini dosyanın ilk satırı olarak dahil ederek, bir ‘TypeError’ istisnası atarak parametre tipi bildirimlerini ve dönüş tipi bildirimlerini zorlayacaktır.

declare(strict_types=1);

Örneğin, parametre türü tanımlarını kullanan bu kod, çalıştırıldığında ‘TypeError’ türünden yakalanabilir bir istisna atar:

<?php
declare(strict_types=1);

function sum(int $a, int $b) {
    return $a + $b;
}

echo sum("1", 2);

Benzer şekilde, bu kod bir dönüş türü bildirimi kullanır; ayrıca bir tamsayı dışında bir şey döndürmeye çalışırsa bir istisna atar:

<?php
declare(strict_types=1);

function returner($a): int {
    return $a;
}

returner("this is a string");
