Operatör, bir veya daha fazla değer (veya programlama jargonunda ifadeler) alan ve başka bir değer veren (yapının kendisi bir ifade haline gelecek şekilde) bir şeydir.

Operatörler aldıkları değer sayısına göre gruplandırılabilir.

Operatörler bir ("!$a" ve “++$a” gibi tekli operatörler), iki ("$a + $b" veya “$a » $b” gibi ikili operatörler) üzerinde “işler” veya işlem yapar veya üç (tek üçlü operatör $a ? $b : $c şeklindedir) ifadeler.

Operatör önceliği, operatörlerin nasıl gruplanacağını etkiler (sanki parantezler varmış gibi). Aşağıda öncelik sırasına göre operatörlerin bir listesi bulunmaktadır (ikinci sütundaki operatörler). Bir satırda birden çok operatör varsa, gruplandırma, ilk sütunun ilişkilendirilebilirliği belirttiği kod sırasına göre belirlenir (örneklere bakın).

Tam bilgi Stack Overflow‘da.

İşlevlerin ve dil yapılarının (örneğin, “yazdır”) her zaman önce değerlendirildiğini, ancak herhangi bir dönüş değerinin yukarıdaki öncelik/birleşim kurallarına göre kullanılacağını unutmayın. Bir dil yapısından sonra parantezler atlanırsa özel dikkat gerekir. Örneğin. eko 2. print 3 + 4; echo’s “721”: “print” bölümü “3 + 4” değerini değerlendirir, “7” sonucunu yazdırır ve “1” değerini döndürür. Bundan sonra, ‘yazdır’ın (‘1’) dönüş değeriyle birleştirilen ‘2’ yankılanır.

Boş Birleştirme Operatörü (??) #

Null birleştirme, PHP 7’de tanıtılan yeni bir operatördür. Bu operatör, ‘NULL’ değil de ayarlanmışsa ilk işlenenini döndürür. Aksi takdirde, ikinci işlenenini döndürür.

Aşağıdaki örnek:

$name = $_POST['name'] ?? 'nobody';

ikisine de eşdeğerdir:

if (isset($_POST['name'])) {
    $name = $_POST['name'];
} else {
    $name = 'nobody';
}

ve:

$name = isset($_POST['name']) ? $_POST['name'] : 'nobody';

Bu operatör ayrıca zincirlenebilir (sağ ilişkisel anlambilim ile):

$name = $_GET['name'] ?? $_POST['name'] ?? 'nobody';

şuna eşdeğerdir:

if (isset($_GET['name'])) {
    $name = $_GET['name'];
} elseif (isset($_POST['name'])) {
    $name = $_POST['name'];
} else {
    $name = 'nobody';
}

Not:
Dize birleştirmede birleştirme operatörünü kullanırken parantez () kullanmayı unutmayın 

$firstName = "John";
$lastName = "Doe";
echo $firstName ?? "Unknown" . " " . $lastName ?? "";

Bu, yalnızca “John” çıktısını verir ve $firstName null ve $lastName “Doe” ise, “Bilinmeyen Doe” çıktısını verir. ‘John Doe’ çıktısını almak için bunun gibi parantezler kullanmalıyız.

$firstName = "John";
$lastName = "Doe";
echo ($firstName ?? "Unknown") . " " . ($lastName ?? "");

Bu, yalnızca “John” yerine “John Doe” çıktısını verir.

Uzay Gemisi Operatörü (<=>) #

PHP 7, ifadeleri karşılaştırmak için kullanılabilecek yeni bir tür operatör sunar. Bu operatör, ilk ifade ikinci ifadeden küçük, ona eşit veya ondan büyükse -1, 0 veya 1 döndürür.

// Integers
print (1 <=> 1); // 0
print (1 <=> 2); // -1
print (2 <=> 1); // 1

// Floats
print (1.5 <=> 1.5); // 0
print (1.5 <=> 2.5); // -1
print (2.5 <=> 1.5); // 1
 
// Strings
print ("a" <=> "a"); // 0
print ("a" <=> "b"); // -1
print ("b" <=> "a"); // 1

Nesneler karşılaştırılabilir değildir ve böyle yapmak tanımsız davranışla sonuçlanacaktır.

Bu operatör, özellikle “usort”, “uasort” veya “uksort” kullanarak kullanıcı tanımlı bir karşılaştırma işlevi yazarken kullanışlıdır. Örneğin, “ağırlık” özelliklerine göre sıralanacak bir dizi nesne verildiğinde, anonim bir işlev, sıralama işlevlerinin beklediği değeri döndürmek için “<=>” kullanabilir.

usort($list, function($a, $b) { return $a->weight <=> $b->weight; });

PHP 5’te bunun için daha ayrıntılı bir ifade gerekirdi.

usort($list, function($a, $b) {
    return $a->weight < $b->weight ? -1 : ($a->weight == $b->weight ? 0 : 1);
});

Üçlü operatör (?:) #

Üçlü operatör, satır içi bir “if” ifadesi olarak düşünülebilir. Üç bölümden oluşur. “İşleç” ve iki sonuç. Sözdizimi aşağıdaki gibidir:

$value = <operator> ? <true value> : <false value>

Eğer ‘operatör’ ’true’ olarak değerlendirilirse, ilk bloktaki değer döndürülür (’’), aksi takdirde ikinci bloktaki değer döndürülür (’’). Üçlü operatörümüzün sonucuna $değer ayarladığımız için, döndürülen değeri saklayacaktır.

Örnek:

$action = empty($_POST['action']) ? 'default' : $_POST['action'];

“$action”, “empty($_POST[‘action’])” doğru olarak değerlendirilirse “varsayılan” dizesini içerir. Aksi takdirde, $_POST['action'] değerini içerecektir.

`(ifade1) ? (ifade2) : (ifade3)’, ‘ifade1’ ‘doğru’ olarak değerlendirilirse ‘ifade2’ ve ‘ifade1’ ‘yanlış’ olarak değerlendirilirse ‘ifade3’ olarak değerlendirilir.

Üçlü operatörün orta kısmını dışarıda bırakmak mümkündür. “ifade1?: ifade3” ifadesi, “ifade1” DOĞRU olarak değerlendirilirse “ifade1”, aksi takdirde “ifade3” döndürür. ?: genellikle Elvis operatörü olarak adlandırılır.

Bu, Null Coalescing operatörü ? gibi davranır, ancak ??, sol işlenenin tam olarak null olmasını gerektirirken ?: sol işleneni bir boole olarak çözmeye çalışır ve kontrol edip etmediğini kontrol eder. boolean “false” olarak çözümlenir.

Örnek:

function setWidth(int $width = 0){
    $_SESSION["width"] = $width ?: getDefaultWidth();
}

Bu örnekte, “setWidth”, genişlik oturum değerini değiştirmek için bir genişlik parametresini veya varsayılan 0’ı kabul eder. “$width” 0 ise ("$width" sağlanmazsa), yanlış boolean olarak çözülür, bunun yerine “getDefaultWidth()” değeri kullanılır. “$width” yanlış boole değerine çözümlenmediyse, “getDefaultWidth()” işlevi çağrılmaz.

Değişkenlerin boole dönüşümü hakkında daha fazla bilgi için https://www.wikiod.com/tr/php/turler adresine bakın.

1: https://www.wikiod.com/tr/php/operatorler#Null Birleştirme Operatörü (??)

Artırma (++) ve Azaltma Operatörleri (–) #

Değişkenler sırasıyla ‘++’ veya ‘–’ ile 1 artırılabilir veya azaltılabilir. Değişkenlerden önce veya sonra gelebilirler ve aşağıda gösterildiği gibi anlamsal olarak biraz farklılık gösterebilirler.

$i = 1;
echo $i; // Prints 1

// Pre-increment operator increments $i by one, then returns $i
echo ++$i; // Prints 2

// Pre-decrement operator decrements $i by one, then returns $i
echo --$i; // Prints 1

// Post-increment operator returns $i, then increments $i by one
echo $i++; // Prints 1 (but $i value is now 2)

// Post-decrement operator returns $i, then decrements $i by one
echo $i--; // Prints 2 (but $i value is now 1)

Artırma ve azaltma operatörleri hakkında daha fazla bilgi resmi belgelerde bulunabilir.

Yürütme İşleci (``) #

PHP yürütme operatörü, geri tepmelerden (``) oluşur ve kabuk komutlarını çalıştırmak için kullanılır. Komutun çıktısı döndürülecek ve bu nedenle bir değişkende saklanabilir.

// List files
$output = `ls`;
echo "<pre>$output</pre>";

Yürütme operatörünün ve shell_exec() işleminin aynı sonucu vereceğine dikkat edin.

Mantıksal Operatörler (&&/AND ve ||/OR) #

PHP’de mantıksal AND ve OR operatörlerinin iki versiyonu vardır.

Operatör Doğru ise
$a ve $b Hem “$a” hem de “$b” doğrudur
$a && $b Hem “$a” hem de “$b” doğrudur
$a veya $b “$a” veya “$b” doğru
$a || $b “$a” veya “$b” doğru

“&&” ve “||” operatörlerinin “ve” ve “veya"dan daha yüksek öncelik olduğuna dikkat edin. Aşağıdaki tabloya bakınız:

değerlendirme $e Sonucu olarak değerlendirildi
$e = false || doğru doğru $e = (yanlış || doğru)
`$e = yanlış veya doğru’ Yanlış ($e = yanlış) veya doğru

Bu nedenle, ’ve’ ve ‘veya’ yerine ‘&&’ ve ‘||’ kullanmak daha güvenlidir.

Dize Operatörleri (. ve .=) #

Yalnızca iki dize operatörü vardır:

  • İki dizenin birleştirilmesi (nokta):

    $a = "a";
$b = "b";
$c = $a . $b; // $c => "ab"

  • Birleştirme ataması (nokta=):

    $a = "a";
$a .= "b"; // $a => "ab"

Nesne ve Sınıf Operatörleri #

Nesnelerin veya sınıfların üyelerine, nesne operatörü (->) ve sınıf operatörü (::) kullanılarak erişilebilir.

class MyClass {
    public $a = 1;
    public static $b = 2;
    const C = 3;
    public function d() { return 4; }
    public static function e() { return 5; }
}

$object = new MyClass();
var_dump($object->a);   // int(1)
var_dump($object::$b);  // int(2)
var_dump($object::C);   // int(3)
var_dump(MyClass::$b);  // int(2)
var_dump(MyClass::C);   // int(3)
var_dump($object->d()); // int(4)
var_dump($object::d()); // int(4)
var_dump(MyClass::e()); // int(5)
$classname = "MyClass";
var_dump($classname::e()); // also works! int(5)

Nesne operatöründen sonra “$” yazılmaması gerektiğini unutmayın ("$object->$a” yerine “$object->a”). Sınıf operatörü için durum böyle değildir ve $ gereklidir. Sınıfta tanımlanan bir sabit için $ asla kullanılmaz.

Ayrıca, “var_dump(MyClass::d());” işlevine yalnızca “d()” işlevi nesneye referans vermiyorsa izin verildiğine dikkat edin:

class MyClass {
    private $a = 1;
    public function d() {
        return $this->a;
    }
}

$object = new MyClass();
var_dump(MyClass::d());   // Error!

Bu, ‘PHP Ölümcül hata: Yakalanmayan Hata: Nesne bağlamında olmadığında $this kullanma’ya neden olur

Bu operatörler, ‘zincirleme’ için kullanılabilecek sol ilişkilendirmeye sahiptir:

class MyClass {
    private $a = 1;
    
    public function add(int $a) {
        $this->a += $a;
        return $this;
    }
    
    public function get() {
        return $this->a;
    }
}

$object = new MyClass();
var_dump($object->add(4)->get());  // int(5)

Bu operatörler en yüksek önceliğe sahiptirler (kılavuzda bunlardan bahsedilmezler), hatta “klon"dan daha yüksektir. Böylece:

class MyClass {
    private $a = 0;
    public function add(int $a) {
        $this->a += $a;
        return $this;
    }
    public function get() {
        return $this->a;
    }
}

$o1 = new MyClass();
$o2 = clone $o1->add(2);
var_dump($o1->get()); // int(2)
var_dump($o2->get()); // int(2)

‘$o1’ değeri, nesne klonlanmadan önce _öncesine eklenir!

Önceliği etkilemek için parantez kullanmanın PHP sürüm 5 ve daha eski sürümlerde çalışmadığını unutmayın (PHP 7’de çalışır):

// using the class MyClass from the previous code
$o1 = new MyClass();
$o2 = (clone $o1)->add(2);  // Error in PHP 5 and before, fine in PHP 7
var_dump($o1->get()); // int(0) in PHP 7
var_dump($o2->get()); // int(2) in PHP 7

Birleşik Atama (+= vb.) #

Birleştirilmiş atama operatörleri, bazı değişkenler üzerindeki bir işlem için bir kısayoldur ve ardından bu yeni değeri o değişkene atar.

Aritmetik:

$a = 1;   // basic assignment
$a += 2; // read as '$a = $a + 2'; $a now is (1 + 2) => 3
$a -= 1; // $a now is (3 - 1) => 2
$a *= 2; // $a now is (2 * 2) => 4
$a /= 2; // $a now is (16 / 2) => 8
$a %= 5; // $a now is (8 % 5) => 3 (modulus or remainder)

// array +
$arrOne = array(1);
$arrTwo = array(2);
$arrOne += $arrTwo;

Birden Çok Diziyi Birlikte İşlemek

$a **= 2; // $a now is (4 ** 2) => 16 (4 raised to the power of 2)

Bir dizenin birleşik bitiştirilmesi ve atanması:

$a = "a";
$a .= "b"; // $a => "ab"

Birleşik ikili bit düzeyinde atama operatörleri:

$a = 0b00101010;  // $a now is 42
$a &= 0b00001111; // $a now is (00101010 & 00001111) => 00001010 (bitwise and)
$a |= 0b00100010; // $a now is (00001010 | 00100010) => 00101010 (bitwise or)
$a ^= 0b10000010; // $a now is (00101010 ^ 10000010) => 10101000 (bitwise xor)
$a >>= 3;         // $a now is (10101000 >> 3) => 00010101 (shift right by 3)
$a <<= 1;         // $a now is (00010101 << 1) => 00101010 (shift left by 1)

Operatör önceliğini değiştirme (parantez içinde) #

Operatörlerin değerlendirildiği sıra, operatör önceliği tarafından belirlenir (Ayrıca Açıklamalar bölümüne bakın).

İçinde

$a = 2 * 3 + 4;

“$a” 10 değerini alır çünkü önce “2 * 3” değerlendirilir (çarpma, toplamadan daha yüksek önceliğe sahiptir), 10’a eşit olan “6 + 4” alt sonucunu verir.

Öncelik parantez kullanılarak değiştirilebilir:

$a = 2 * (3 + 4);

$a 14 değerini alır çünkü önce (3 + 4) değerlendirilir.

Temel Atama (=) #

$a = "some string";

“$a"nın “bir dizi” değerine sahip olmasıyla sonuçlanır.

Bir atama ifadesinin sonucu, atanan değerdir. Tek bir eşittir işareti = karşılaştırma için DEĞİLDİR!

$a = 3;
$b = ($a = 5);

şunları yapar:

  1. Satır 1, “3"ü “$a"ya atar.
  2. Satır 2, “5"i “$a"ya atar. Bu ifade de “5” değerini verir.
    1. satır daha sonra parantez içindeki (‘5’) ifadenin sonucunu ‘$b’ye atar.

Böylece: hem “$a” hem de “$b” artık “5” değerine sahiptir.

Dernek #

sol ilişkilendirme #

İki operatörün önceliği eşitse, ilişkilendirme gruplandırmayı belirler (ayrıca Açıklamalar bölümüne bakın):

$a = 5 * 3 % 2; // $a now is (5 * 3) % 2 => (15 % 2) => 1

* ve % eşit önceliğe ve sol ilişkilendirilebilirliğe sahiptir. Çarpma önce (solda) gerçekleştiği için gruplandırılmıştır.

$a = 5 % 3 * 2; // $a now is (5 % 3) * 2 => (2 * 2) => 4

Şimdi, modül operatörü önce (solda) gerçekleşir ve böylece gruplanır.

Doğru ilişkilendirme #

$a = 1;
$b = 1;
$a = $b += 1;

Hem “$a” hem de “$b” artık “2” değerine sahiptir, çünkü “$b += 1” gruplanır ve ardından sonuç ("$b”, “2"dir) “$a"ya atanır.

Karşılaştırma Operatörleri #

eşitlik #

Temel eşitlik testi için eşit operatör == kullanılır. Daha kapsamlı kontroller için, aynı === operatörünü kullanın.

Özdeş operatör, eşit operatörle aynı şekilde çalışır, işlenenlerinin aynı değere sahip olmasını gerektirir, ancak aynı veri türüne sahip olmalarını da gerektirir.

Örneğin, aşağıdaki örnek ‘a ve b eşittir’ ifadesini gösterecek, ancak ‘a ve b özdeştir’ ifadesini göstermeyecektir.

$a = 4;
$b = '4';
if ($a == $b) {
    echo 'a and b are equal'; // this will be printed
}
if ($a === $b) {
    echo 'a and b are identical'; // this won't be printed
}

Eşit işlecini kullanırken, sayısal diziler tamsayılara dönüştürülür.

Nesnelerin karşılaştırılması #

===, tam olarak aynı örnek olup olmadıklarını kontrol ederek iki nesneyi karşılaştırır. Bu, “new stdClass() === new stdClass()” öğesinin, aynı şekilde oluşturulmuş olsalar bile (ve tamamen aynı değerlere sahip olsalar) false olarak çözümlendiği anlamına gelir.

==, eşit olup olmadıklarını tekrar tekrar kontrol ederek iki nesneyi karşılaştırır (deep equals). Bunun anlamı, “$a” ve “$b” şuysa, “$a == $b” için:

  1. aynı sınıftan
  2. dinamik özellikler de dahil olmak üzere aynı özelliklere sahip olmak
  3. $property kümesindeki her özellik için, $a->property == $b->property doğrudur (dolayısıyla tekrar tekrar kontrol edilir).

Diğer yaygın olarak kullanılan operatörler #

Onlar içerir:

  1. Büyüktür (>)

  2. Küçüktür (<)

  3. Büyüktür veya Eşittir (>=)

  4. Küçüktür veya Eşittir (<=)

  5. Eşit Değil (!=)

  6. Özdeş Olarak Eşit Değil (!==)

  7. Büyüktür: “$a > $b”, “$a"nın değeri “$b"den büyükse “true”, aksi takdirde false döndürür.

Örnek:

var_dump(5 > 2); // prints bool(true)
var_dump(2 > 7); // prints bool(false)
  1. Küçüktür: ‘$a < $b’, ‘$a’nın değeri ‘$b’ değerinden küçükse ’true’, aksi takdirde false döndürür.

Örnek:

var_dump(5 < 2); // prints bool(false)
var_dump(1 < 10); // prints bool(true)
  1. Büyüktür veya Eşittir: “$a >= $b”, “$a"nın değeri “$b"den büyük veya “$b"ye eşitse “doğru” değerini döndürür, aksi halde ‘yanlış’ döndürür.

Örnek:

var_dump(2 >= 2); // prints bool(true)
var_dump(6 >= 1); // prints bool(true)
var_dump(1 >= 7); // prints bool(false)
  1. Daha Küçük Veya Eşittir: ‘$a <= $b’, ‘$a’nın değeri ‘$b’den küçük veya ‘$b’ye eşitse ‘doğru’ değerini döndürür, aksi takdirde ‘yanlış’ döndürür.

Örnek:

var_dump(5 <= 5); // prints bool(true)
var_dump(5 <= 8); // prints bool(true)
var_dump(9 <= 1); // prints bool(false)

5/6. Eşit Değil/Özdeşle: Eşitlikle ilgili önceki örneği yeniden özetlemek için, aşağıdaki örnek ‘a ve b özdeş değildir’ ifadesini gösterecek, ancak ‘a ve b eşit değildir’ göstermeyecektir.

$a = 4;
$b = '4';
if ($a != $b) {
    echo 'a and b are not equal'; // this won't be printed
}
if ($a !== $b) {
    echo 'a and b are not identical'; // this will be printed
}

instanceof (tür operatörü) #

Bazı nesnelerin belirli bir sınıftan olup olmadığını kontrol etmek için, PHP sürüm 5’ten beri (ikili) ‘instanceof’ operatörü kullanılabilir.

İlk (sol) parametre test edilecek nesnedir. Bu değişken bir nesne değilse, “instanceof” her zaman “false” değerini döndürür. Sabit bir ifade kullanılırsa, bir hata atılır.

İkinci (sağ) parametre, karşılaştırılacak sınıftır. Sınıf, sınıf adının kendisi, sınıf adını içeren bir dize değişkeni (bir dize sabiti değil!) veya o sınıfın bir nesnesi olarak sağlanabilir.

class MyClass {
}

$o1 = new MyClass();
$o2 = new MyClass();
$name = 'MyClass';

// in the cases below, $a gets boolean value true
$a = $o1 instanceof MyClass;
$a = $o1 instanceof $name;
$a = $o1 instanceof $o2;

// counter examples:
$b = 'b';
$a = $o1 instanceof 'MyClass'; // parse error: constant not allowed
$a = false instanceof MyClass; // fatal error: constant not allowed
$a = $b instanceof MyClass;    // false ($b is not an object)

“instanceof”, bir nesnenin başka bir sınıfı genişleten veya bir arabirim uygulayan bir sınıftan olup olmadığını kontrol etmek için de kullanılabilir:

interface MyInterface {
}

class MySuperClass implements MyInterface {
}

class MySubClass extends MySuperClass {
}

$o = new MySubClass();

// in the cases below, $a gets boolean value true    
$a = $o instanceof MySubClass;
$a = $o instanceof MySuperClass;
$a = $o instanceof MyInterface;

Bir nesnenin bir sınıfa ait olup olmadığını kontrol etmek için not operatörü (!) kullanılabilir:

class MyClass {
}

class OtherClass {
}

$o = new MyClass();
$a = !$o instanceof OtherClass; // true

‘instanceof’, ‘!‘den daha yüksek önceliğe sahip olduğundan, ‘$o ​​instanceof MyClass’ etrafındaki parantezlerin gerekli olmadığını unutmayın, ancak bu, kodu parantezlerle daha iyi okunabilir hale getirebilir.

uyarılar #

Bir sınıf yoksa, sınıfı tanımlamaya çalışmak için kayıtlı otomatik yükleme işlevleri çağrılır (bu, Belgelerin bu bölümünün kapsamı dışında kalan bir konudur!). 5.1.0’dan önceki PHP sürümlerinde, ‘instanceof’ operatörü de bu çağrıları tetikler, böylece aslında sınıfı tanımlardı (ve eğer sınıf tanımlanamazsa, ölümcül bir hata meydana gelirdi). Bunu önlemek için bir dize kullanın:

// only PHP versions before 5.1.0!
class MyClass {
}

$o = new MyClass();
$a = $o instanceof OtherClass; // OtherClass is not defined!
// if OtherClass can be defined in a registered autoloader, it is actually
// loaded and $a gets boolean value false ($o is not a OtherClass)
// if OtherClass can not be defined in a registered autoloader, a fatal
// error occurs.

$name = 'YetAnotherClass';
$a = $o instanceof $name; // YetAnotherClass is not defined!
// $a simply gets boolean value false, YetAnotherClass remains undefined.

PHP sürüm 5.1.0’dan itibaren, bu durumlarda kayıtlı otomatik yükleyiciler artık çağrılmamaktadır.

PHP’nin eski sürümleri (5.0 öncesi) #

PHP’nin eski sürümlerinde (5.0’dan önce), bir nesnenin bir sınıftan olup olmadığını belirlemek için ‘is_a’ işlevi kullanılabilir. Bu işlev PHP sürüm 5’te ve PHP sürüm 5.3.0’da kullanımdan kaldırılmıştır.

Bitsel Operatörler #

Önek bitsel operatörler #

Bitsel operatörler, mantıksal operatörler gibidir, ancak boole değeri yerine bit başına yürütülür.

// bitwise NOT ~: sets all unset bits and unsets all set bits
printf("%'06b", ~0b110110); // 001001

Bit maskesi-bit maskesi operatörleri #

Bitsel VE &: bir bit yalnızca her iki işlenende de ayarlanmışsa ayarlanır

printf("%'06b", 0b110101 & 0b011001); // 010001

Bitsel VEYA |: işlenenlerden birinde veya her ikisinde de ayarlanmışsa bir bit ayarlanır

printf("%'06b", 0b110101 | 0b011001); // 111101

Bitsel XOR ^: bir işlenende ayarlanmışsa ve başka bir işlenende ayarlanmamışsa bir bit ayarlanır, yani yalnızca bu bit iki işlenende farklı durumdaysa

printf("%'06b", 0b110101 ^ 0b011001); // 101100

Bit maskelerinin örnek kullanımları #

Bu operatörler, bit maskelerini değiştirmek için kullanılabilir. Örneğin:

file_put_contents("file.log", LOCK_EX | FILE_APPEND);

Burada, iki bit maskesini birleştirmek için | operatörü kullanılır. ‘+’ aynı etkiye sahip olsa da, ‘|’ iki normal skaler tamsayı eklemek yerine bit maskelerini birleştirdiğinizi vurgular.

class Foo{
    const OPTION_A = 1;
    const OPTION_B = 2;
    const OPTION_C = 4;
    const OPTION_A = 8;

    private $options = self::OPTION_A | self::OPTION_C;

    public function toggleOption(int $option){
        $this->options ^= $option;
    }

    public function enable(int $option){
        $this->options |= $option; // enable $option regardless of its original state
    }

    public function disable(int $option){
        $this->options &= ~$option; // disable $option regardless of its original state,
                                    // without affecting other bits
    }

    /** returns whether at least one of the options is enabled */
    public function isOneEnabled(int $options) : bool{
        return $this->options & $option !== 0;
        // Use !== rather than >, because 
        // if $options is about a high bit, we may be handling a negative integer
    }

    /** returns whether all of the options are enabled */
    public function areAllEnabled(int $options) : bool{
        return ($this->options & $options) === $options;
        // note the parentheses; beware the operator precedence
    }
}

Bu örnek ("$option” öğesinin her zaman yalnızca bir bit içerdiği varsayılarak) şunları kullanır:

  • bit maskelerini rahatça değiştirmek için ^ operatörü.
  • | operatörü, orijinal durumunu veya diğer bitleri biraz ihmal ederek ayarlamak için
  • ~ operatörü, yalnızca bir biti ayarlı bir tamsayıyı, yalnızca bir biti ayarlanmamış bir tam sayıya dönüştürür
  • & operatörü, & şu özelliklerini kullanarak biraz ayarını kaldırır:
  • Bir set biti olan &= hiçbir şey yapmayacağından ((1 & 1) === 1, (0 & 1) === 0), &= sadece ile bir tamsayı ile yapmak bir bit ayarlanmadı, yalnızca o bitin ayarını kaldırır, diğer bitleri etkilemez.
  • &= ayarlanmamış bir bit ile o bitin ayarını kaldırır ((1 & 0) === 0, (0 & 0) === 0)
  • & operatörünü başka bir bit maskesiyle kullanmak, o bit maskesinde ayarlanmayan diğer tüm bitleri filtreleyecektir.
  • Çıkışta herhangi bir bit ayarlanmışsa, seçeneklerden herhangi birinin etkinleştirildiği anlamına gelir.
  • Çıktıda bit maskesinin tüm bitleri varsa, bu, bit maskesindeki tüm seçeneklerin etkinleştirildiği anlamına gelir.

Şu karşılaştırma operatörlerinin: (< > <= >= == === != !== <> <=> ) şu bit maskesi-bit maskesi operatörlerinden daha yüksek önceliğe sahiptir: (| ^ &). Bit düzeyinde sonuçlar genellikle bu karşılaştırma operatörleri kullanılarak karşılaştırıldığından, bu, bilinmesi gereken yaygın bir tuzaktır.

Bit kaydırma operatörleri #

Bitsel sola kaydırma <<: tüm bitleri verilen adım sayısı kadar sola (daha anlamlı) kaydırın ve int boyutunu aşan bitleri atın

“« $x”, en yüksek “$x” bitlerinin ayarını kaldırmaya ve 2’nin “$x” kuvvetiyle çarpmaya eşdeğerdir.

printf("%'08b", 0b00001011<< 2); // 00101100

assert(PHP_INT_SIZE === 4); // a 32-bit system
printf("%x, %x", 0x5FFFFFFF << 2, 0x1FFFFFFF << 4); // 7FFFFFFC, FFFFFFFF

Bit düzeyinde sağa kaydırma >>: en düşük kaydırmayı atın ve kalan bitleri sağa kaydırın (daha az önemli)

>> $x, 2’nin $x kuvvetine bölmeye ve tamsayı olmayan kısmı atmaya eşdeğerdir

printf("%x", 0xFFFFFFFF >> 3); // 1FFFFFFF

Bit kaydırmanın örnek kullanımları: #

16’ya hızlı bölme (’/= 16’dan daha iyi performans)

$x >>= 4;

32 bit sistemlerde bu, tamsayıdaki tüm bitleri atar ve değeri 0 olarak ayarlar. 64 bit sistemlerde bu, en önemli 32 bitin ayarını kaldırır ve en az

$x = $x << 32 >> 32;

önemli 32 bit, “$x & 0xFFFFFFFF"ye eşdeğer

Not: Bu örnekte printf("%'06b") kullanılmıştır. Değeri 6 ikili basamak olarak verir.
