Özel ve korumalı üye değişkenlerine erişim #

Yansıma genellikle, sahte nesnelerin çalışma zamanı oluşturma/örnekleme gibi yazılım testinin bir parçası olarak kullanılır. Aynı zamanda herhangi bir zamanda bir nesnenin durumunu incelemek için de harikadır. Burada, korumalı bir sınıf üyesinin beklenen değeri içerdiğini doğrulamak için bir birim testinde Reflection kullanımına bir örnek verilmiştir.

Aşağıda bir Araba için çok temel bir sınıf var. Arabanın rengini temsil eden değeri içerecek korumalı bir üye değişkeni vardır. Üye değişken korumalı olduğundan, ona doğrudan erişemeyiz ve değerini almak ve ayarlamak için sırasıyla bir alıcı ve ayarlayıcı yöntemi kullanmalıyız.

class Car
{
    protected $color
    
    public function setColor($color)
    {
        $this->color = $color;
    }
    
    public function getColor($color)
    {
        return $this->color;
    }
}

Bunu test etmek için birçok geliştiricinin bir Car nesnesi oluşturacağını, arabanın rengini Car::setColor() kullanarak ayarlayın, Car::getColor() kullanarak rengi alın ve bu değeri ayarladıkları renkle karşılaştırın:

/**
 * @test
 * @covers     \Car::setColor
 */
public function testSetColor()
{
    $color = 'Red';

    $car = new \Car();
    $car->setColor($color);
    $getColor = $car->getColor();
        
    $this->assertEquals($color, $reflectionColor);
}

Yüzeyde bu iyi görünüyor. Sonuçta, Car::getColor() tüm yaptığı, korunan Car::$color üye değişkeninin değerini döndürmektir. Ancak bu test iki yönden kusurludur:

  1. Bu testin kapsamı dışında kalan Car::getColor() alıştırmalarını yapar.
  2. Testin yanlış pozitif veya negatif olmasına neden olabilecek bir hataya sahip olabilecek Car::getColor()a bağlıdır.

Birim testimizde neden Car::getColor() kullanmamamız ve bunun yerine Reflection kullanmamız gerektiğine bakalım. Diyelim ki bir geliştiriciye her araba rengine “Metalik” eklemek için bir görev verildi. Bu nedenle, arabanın rengine “Metalik” eklemek için Car::getColor() öğesini değiştirmeye çalışırlar:

class Car
{
    protected $color
    
    public function setColor($color)
    {
        $this->color = $color;
    }
    
    public function getColor($color)
    {
        return "Metallic "; $this->color;
    }
}

Hatayı görüyor musun? Geliştirici, otomobilin rengine “Metalik” eklemek için birleştirme operatörü yerine noktalı virgül kullandı. Sonuç olarak, ne zaman Car::getColor() çağrılsa, arabanın gerçek rengi ne olursa olsun “Metallic” döndürülür. Sonuç olarak, Car::setColor() birim testimiz, Car::setColor() mükemmel şekilde çalışmasına ve bu değişiklikten etkilenmemesine rağmen* başarısız olacaktır*.

Peki, Car::$color öğesinin Car::setColor() aracılığıyla ayarladığımız değeri içerdiğini nasıl doğrulayacağız? Korumalı üye değişkenini doğrudan incelemek için Refelection’ı kullanabiliriz. Peki bunu nasıl yapacağız*? Değeri alabilmesi için korumalı üye değişkenini kodumuz için erişilebilir kılmak için Refelection’ı kullanabiliriz.

Önce kodu görelim ve sonra parçalayalım:

/**
 * @test
 * @covers     \Car::setColor
 */
public function testSetColor()
{
    $color = 'Red';

    $car = new \Car();
    $car->setColor($color);
    
    $reflectionOfCar = new \ReflectionObject($car);
    $protectedColor = $reflectionOfForm->getProperty('color');
    $protectedColor->setAccessible(true);
    $reflectionColor = $protectedColor->getValue($car);
    
    $this->assertEquals($color, $reflectionColor);
}

Yukarıdaki kodda Car::$color değerini elde etmek için Reflection’ı şu şekilde kullanıyoruz:

  1. Car nesnemizi temsil eden yeni bir ReflectionObject oluşturuyoruz
  2. “Car::$color” için bir ReflectionProperty elde ederiz (bu, “Car::$color” değişkenini “temsil eder”)
  3. `Car::$color``u erişilebilir hale getiriyoruz
  4. Car::$color değerini alırız

Gördüğünüz gibi, Reflection kullanarak, ‘Car::$color’ değerini, ‘Car::getColor()’ veya geçersiz test sonuçlarına neden olabilecek başka herhangi bir erişim fonksiyonunu çağırmadan alabiliriz. Artık Car::setColor() için birim testimiz güvenli ve doğrudur.

Sınıfların veya nesnelerin özellik algılaması #

Sınıfların özellik tespiti kısmen ‘property_exists’ ve ‘method_exists’ fonksiyonları ile yapılabilir.

class MyClass {
    public $public_field;
    protected $protected_field;
    private $private_field;
    static $static_field;
    const CONSTANT = 0;
    public function public_function() {}
    protected function protected_function() {}
    private function private_function() {}
    static function static_function() {}
}

// check properties
$check = property_exists('MyClass', 'public_field');    // true
$check = property_exists('MyClass', 'protected_field'); // true
$check = property_exists('MyClass', 'private_field');   // true, as of PHP 5.3.0
$check = property_exists('MyClass', 'static_field');    // true
$check = property_exists('MyClass', 'other_field');     // false

// check methods
$check = method_exists('MyClass', 'public_function');    // true
$check = method_exists('MyClass', 'protected_function');    // true
$check = method_exists('MyClass', 'private_function');    // true
$check = method_exists('MyClass', 'static_function');    // true

// however...
$check = property_exists('MyClass', 'CONSTANT');  // false
$check = property_exists($object, 'CONSTANT');    // false

Bir “ReflectionClass” ile sabitler de tespit edilebilir:

$r = new ReflectionClass('MyClass');
$check = $r->hasProperty('public_field');  // true
$check = $r->hasMethod('public_function'); // true
$check = $r->hasConstant('CONSTANT');      // true
// also works for protected, private and/or static members.

Not: “özellik_var” ve “yöntem_var” için, sınıf adı yerine ilgilenilen sınıfın bir nesnesi de sağlanabilir. Yansıma kullanarak, ‘ReflectionClass’ yerine ‘ReflectionObject’ sınıfı kullanılmalıdır.

Özel/korumalı yöntemleri test etme #

Bazen genel yöntemlerin yanı sıra özel ve korumalı yöntemleri de test etmek yararlıdır.

class Car
{
    /**
     * @param mixed $argument
     *
     * @return mixed
     */
    protected function drive($argument)
    {
        return $argument;
    }

    /**
     * @return bool
     */
    private static function stop()
    {
        return true;
    }
}

Sürüş yöntemini test etmenin en kolay yolu yansıma kullanmaktır

class DriveTest
{
    /**
     * @test
     */
    public function testDrive()
    {
        // prepare
        $argument = 1;
        $expected = $argument;
        $car = new \Car();

        $reflection = new ReflectionClass(\Car::class);
        $method = $reflection->getMethod('drive');
        $method->setAccessible(true);

        // invoke logic
        $result = $method->invokeArgs($car, [$argument]);

        // test
        $this->assertEquals($expected, $result);
    }
}

Yöntem statik ise, sınıf örneğinin yerine null değerini geçersiniz.

class StopTest
{
    /**
     * @test
     */
    public function testStop()
    {
        // prepare
        $expected = true;

        $reflection = new ReflectionClass(\Car::class);
        $method = $reflection->getMethod('stop');
        $method->setAccessible(true);

        // invoke logic
        $result = $method->invoke(null);

        // test
        $this->assertEquals($expected, $result);
    }
}
