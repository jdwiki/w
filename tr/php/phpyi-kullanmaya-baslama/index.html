Web sunucusundan HTML çıktısı #

PHP, HTML dosyalarına içerik eklemek için kullanılabilir. HTML doğrudan bir web tarayıcısı tarafından işlenirken, PHP komut dosyaları bir web sunucusu tarafından yürütülür ve elde edilen HTML tarayıcıya gönderilir.

Aşağıdaki HTML işaretlemesi, çıktıya Merhaba Dünya! ekleyecek bir PHP ifadesi içerir:

<!DOCTYPE html>
<html>
    <head>
        <title>PHP!</title>
    </head>
    <body>
        <p><?php echo "Hello world!"; ?></p>
    </body>
</html>

Bu bir PHP betiği olarak kaydedildiğinde ve bir web sunucusu tarafından yürütüldüğünde, kullanıcının tarayıcısına aşağıdaki HTML gönderilir:

<!DOCTYPE html>
<html>
    <head>
        <title>PHP!</title>
    </head>
    <body>
        <p>Hello world!</p>
    </body>
</html>

’echo’ ayrıca bir değeri hemen yazdırmanıza izin veren bir kısayol sözdizimine sahiptir. PHP 5.4.0’dan önce bu kısa sözdizimi yalnızca short_open_tag yapılandırma ayarı etkinken çalışır.

Örneğin, aşağıdaki kodu göz önünde bulundurun:

<p><?= "Hello world!" ?></p>

Çıktısı, aşağıdakilerin çıktısıyla aynıdır:

<p><?php echo "Hello world!"; ?></p>
son sürüm

Gerçek dünya uygulamalarında, XSS (Siteler arası komut dosyası çalıştırma) saldırılarını veya metin bozulmasını önlemek için PHP tarafından bir HTML sayfasına gönderilen tüm veriler düzgün şekilde escaped olmalıdır.

Ayrıca, kısa etiketlerin (<?= ... ?>) doğru kullanımı da dahil olmak üzere en iyi uygulamaları açıklayan Dizeler ve PSR-1 bölümüne bakın.

2: https://www.wikiod.com/tr/php/guvenlik#Siteler Arası Komut Dosyası Çalıştırma (XSS) 3: https://www.wikiod.com/tr/php/turler#Teller 4: http://www.php-fig.org/psr/psr-1/

Selam Dünya! #

PHP’de çıktı yazdırmak için en yaygın kullanılan dil yapısı “echo"dur:

echo "Hello, World!\n";

Alternatif olarak, “yazdır"ı da kullanabilirsiniz:

print "Hello, World!\n";

Her iki ifade de küçük farklılıklarla aynı işlevi görür:

  • “echo"nun bir “void” dönüşü vardır, “print” ise “1” değerinde bir “int” döndürür
  • “echo” birden çok argüman alabilir (yalnızca parantez olmadan), “print” ise yalnızca bir argüman alır
  • “echo”, “yazdır"dan biraz daha hızlı

Hem “echo” hem de “print”, işlev değil, dil yapılarıdır. Bu, argümanlarının etrafında parantez gerektirmedikleri anlamına gelir. İşlevlerle kozmetik tutarlılık için parantezler dahil edilebilir. “echo” ve “print” kullanımının kapsamlı örnekleri başka bir yerde mevcuttur.

Aşağıdaki örnekte olduğu gibi C stili printf ve ilgili işlevler de mevcuttur:

printf("%s\n", "Hello, World!");

PHP’de değişkenlerin çıktısını almanın kapsamlı bir tanıtımı için Bir değişkenin değerinin çıktısını almak bölümüne bakın.

3: https://www.wikiod.com/tr/php/bir-degiskenin-degerini-ckarma#yankı ve baskı 4: https://www.wikiod.com/tr/php/bir-degiskenin-degerini-ckarma

Web sunucusundan HTML olmayan çıktı #

Bazı durumlarda, bir web sunucusuyla çalışırken, web sunucusunun varsayılan içerik türünü geçersiz kılmak gerekebilir. Örneğin verileri “düz metin”, “JSON” veya “XML” olarak göndermeniz gereken durumlar olabilir.

[header()][header] işlevi ham bir HTTP başlığı gönderebilir. Tarayıcıya gönderdiğimiz içeriği bildirmek için Content-Type başlığını ekleyebilirsiniz.

“İçerik Türü"nü “metin/düz” olarak ayarladığımız aşağıdaki kodu göz önünde bulundurun:

header("Content-Type: text/plain");
echo "Hello World";

Bu, aşağıdaki içeriğe sahip bir düz metin belgesi üretecektir:

Merhaba Dünya

JSON içeriği oluşturmak için bunun yerine “application/json” içerik türünü kullanın:

header("Content-Type: application/json");

// Create a PHP data array.
$data = ["response" => "Hello World"];

// json_encode will convert it to a valid JSON string.
echo json_encode($data);

Bu, aşağıdaki içeriğe sahip “application/json” türünde bir belge üretecektir:

{“response”:“Merhaba Dünya”}

PHP herhangi bir çıktı üretmeden önce header() fonksiyonunun çağrılması gerektiğine dikkat edin, aksi takdirde web sunucusu yanıt için zaten başlıklar göndermiş olacaktır. Bu nedenle, aşağıdaki kodu göz önünde bulundurun:

// Error: We cannot send any output before the headers
echo "Hello";

// All headers must be sent before ANY PHP output
header("Content-Type: text/plain");
echo "World";

Bu bir uyarı üretecektir:

Uyarı: Başlık bilgisi değiştirilemiyor - satırda /dir/example.php içinde (/dir/example.php:2 adresinden çıktı başlatıldı) tarafından zaten gönderilen başlıklar 3

header() kullanıldığında, çıktısının sunucudan gönderilen ilk bayt olması gerekir. Bu nedenle, PHP açılış etiketi <?php öncesinde dosyanın başında boş satırlar veya boşluklar olmaması önemlidir. Aynı nedenle, yalnızca PHP içeren dosyalardan ve bir dosyanın en sonundaki PHP kod bloklarından PHP kapanış etiketini ?> çıkartmak en iyi uygulama olarak kabul edilir (bkz. PSR-2). .

İçeriğinizi daha sonra, örneğin üstbilgilerin çıktısını aldıktan sonra çıktı almak üzere bir değişkene nasıl ‘yakalayacağınızı’ öğrenmek için [çıktı arabelleğe alma bölümünü][çıktı arabelleğe alma]‘yı görüntüleyin.

PHP yerleşik sunucusu #

PHP 5.4+, yerleşik bir geliştirme sunucusuyla birlikte gelir. Nginx veya Apache gibi bir üretim HTTP sunucusu kurmak zorunda kalmadan uygulamaları çalıştırmak için kullanılabilir. Yerleşik sunucu yalnızca geliştirme ve test amacıyla kullanılmak üzere tasarlanmıştır.

-S bayrağı kullanılarak başlatılabilir:

php -S <host/ip>:<port>

Örnek kullanım #

  1. Aşağıdakileri içeren bir “index.php” dosyası oluşturun:

  2. Komut satırından php -S localhost:8080 komutunu çalıştırın. “http://” eklemeyin. Bu, belge kökü olarak içinde bulunduğunuz dizini kullanarak 8080 numaralı bağlantı noktasını dinleyen bir web sunucusunu başlatır.

  3. Tarayıcıyı açın ve http://localhost:8080 konumuna gidin. “Merhaba Dünya” sayfanızı görmelisiniz.

Yapılandırma #

Varsayılan belge kökünü (yani geçerli dizini) geçersiz kılmak için -t bayrağını kullanın:

php -S <host/ip>:<port> -t <directory>

Örneğin. projenizde bir public/ dizini varsa, projenizi bu dizinden php -S localhost:8080 -t public/ kullanarak sunabilirsiniz.

[Mon Aug 15 18:20:19 2016] ::1:52455 [200]: /

PHP CLI #

PHP, doğrudan CLI (Komut Satırı Arayüzü) kullanılarak komut satırından da çalıştırılabilir.

CLI, standart girdi ve çıktı açısından bazı farklılıklar dışında, temelde web sunucularındaki PHP ile aynıdır.

Tetikleme #

PHP CLI, PHP kodunu çalıştırmanın dört yolunu sağlar:

  1. Standart giriş. php komutunu herhangi bir argüman olmadan çalıştırın, ancak PHP kodunu buna aktarın: 
    echo ‘<?php echo “Merhaba dünya!”;’ | php
  2. Argüman olarak dosya adı. İlk argüman olarak bir PHP kaynak dosyasının adıyla php komutunu çalıştırın: 
    php merhaba_world.php
  3. Argüman olarak kodlayın. ‘php’ komutunda ‘-r’ seçeneğini ve ardından çalıştırılacak kodu kullanın. Argümandaki her şey PHP kodu olarak kabul edildiğinden <?php açık etiketleri gerekli değildir: 
    php -r ’echo “Merhaba dünya!”;’
  4. Etkileşimli kabuk. Etkileşimli bir kabuk başlatmak için ‘php’ komutundaki ‘-a’ seçeneğini kullanın. Ardından PHP kodunu yazın (veya yapıştırın) ve return‘ye basın: 
    $ php -a
Etkileşimli mod etkin
php > echo “Merhaba dünya!”;
Merhaba dünya!

Çıktı #

PHP web sunucusunda HTML çıktısı üreten tüm işlevler veya kontroller, stdout akışında çıktı üretmek için kullanılabilir (dosya tanımlayıcı 1) ve web sunucusunda hata günlüklerinde çıktı üreten tüm eylemler PHP, stderr akışında çıktı üretecektir (dosya tanımlayıcı 2).

Örnek.php #

<?php
echo "Stdout 1\n";
trigger_error("Stderr 2\n");
print_r("Stdout 3\n");
fwrite(STDERR, "Stderr 4\n");
throw new RuntimeException("Stderr 5\n");
?>
Stdout 6

Kabuk komut satırı #

$ php Example.php 2>stderr.log >stdout.log;\
> echo STDOUT; cat stdout.log; echo;\
> echo STDERR; cat stderr.log\

STDOUT
Stdout 1
Stdout 3

STDERR
Stderr 4
PHP Notice:  Stderr 2
 in /Example.php on line 3
PHP Fatal error:  Uncaught RuntimeException: Stderr 5
 in /Example.php:6
Stack trace:
#0 {main}
  thrown in /Example.php on line 6

Talimat Ayırma #

Diğer birçok C-tarzı dilde olduğu gibi, her ifade noktalı virgül ile sonlandırılır. Ayrıca, PHP bloğunun son kod satırını sonlandırmak için bir kapatma etiketi kullanılır.

PHP kodunun son satırı noktalı virgülle bitiyorsa, bu son kod satırından sonra herhangi bir kod yoksa kapanış etiketi isteğe bağlıdır. Örneğin, aşağıdaki örnekte echo "No error";dan sonra kapanış etiketini dışarıda bırakabiliriz:

<?php echo "No error"; // no closing tag is needed as long as there is no code below

Ancak, PHP kod bloğunuzu takip eden başka bir kod varsa, kapanış etiketi artık isteğe bağlı değildir:

<?php echo "This will cause an error if you leave out the closing tag"; ?>
<html>
    <body>
    </body>
</html>

Eğer o kod bloğunda bir kapanış etiketi varsa, PHP kod bloğundaki son ifadenin noktalı virgülünü de dışarıda bırakabiliriz:

<?php echo "I hope this helps! :D";
echo "No error" ?>

Bu PHP kod bloğunu takip eden başka kod yoksa, son PHP kod bloğu hariç her PHP kod bloğu için her zaman bir noktalı virgül ve bir kapanış etiketi kullanılması genellikle önerilir.

Yani, kodunuz temel olarak şöyle görünmelidir:

<?php
    echo "Here we use a semicolon!";
    echo "Here as well!";
    echo "Here as well!";
    echo "Here we use a semicolon and a closing tag because more code follows";
?>
<p>Some HTML code goes here</p>
<?php
    echo "Here we use a semicolon!";
    echo "Here as well!";
    echo "Here as well!";
    echo "Here we use a semicolon and a closing tag because more code follows";
?>
<p>Some HTML code goes here</p>
<?php
    echo "Here we use a semicolon!";
    echo "Here as well!";
    echo "Here as well!";
    echo "Here we use a semicolon but leave out the closing tag";

PHP Etiketleri #

Bir dosyadaki PHP bloklarını belirtmek için üç tür etiket vardır. PHP ayrıştırıcısı, yorumlanacak kodu sınırlamak için açılış ve (varsa) kapanış etiketlerini arar.

Standart Etiketler #

Bu etiketler, PHP kodunu bir dosyaya gömmek için standart yöntemdir.

<?php
    echo "Hello World";
?>

Yankı Etiketleri #

Bu etiketler tüm PHP sürümlerinde mevcuttur ve PHP 5.4’ten beri her zaman etkindir. Önceki sürümlerde, yankı etiketleri yalnızca kısa etiketlerle birlikte etkinleştirilebiliyordu.

<?= "Hello World" ?>
son sürüm

Kısa Etiketler #

Bu etiketleri short_open_tag seçeneği ile devre dışı bırakabilir veya etkinleştirebilirsiniz.

<?
    echo "Hello World";
?>

Kısa etiketler:

  • tüm önemli PHP kodlama standartlarında izin verilmez
  • resmi belgelerde önerilmez
  • çoğu dağıtımda varsayılan olarak devre dışıdır
  • satır içi XML’in işleme talimatlarına müdahale edin
  • çoğu açık kaynak projesi tarafından kod gönderimlerinde kabul edilmez

ASP Etiketleri #

asp_tags seçeneğini etkinleştirerek, ASP tarzı etiketler kullanılabilir.

<%
    echo "Hello World";
%>

Bunlar tarihi bir tuhaflıktır ve asla kullanılmamalıdır. PHP 7.0’da kaldırıldılar.
