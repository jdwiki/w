Sözdizimi #

  • fonksiyon f(SınıfAdı $param) {}
  • fonksiyon f(bool $param) {}
  • fonksiyon f(int $param) {}
  • fonksiyon f(kayan $param) {}
  • fonksiyon f(string $param) {}
  • fonksiyon f(self $param) {}
  • fonksiyon f(çağrılabilir $param) {}
  • fonksiyon f(dizi $param) {}
  • işlev f(?type_name $param) {}
  • f() işlevi: tür_adı {}
  • f() işlevi: geçersiz {}
  • işlev f() : ?type_name {}

Tür ipuçları veya tür bildirimleri, bir işlevin parametrelerinin belirli bir türde olmasını sağlayan savunma amaçlı bir programlama uygulamasıdır. Bu, özellikle bir arabirim için tür ipucu verirken kullanışlıdır, çünkü işlevin sağlanan bir parametrenin arabirimde gerekli olan yöntemlerle aynı yöntemlere sahip olmasını garanti etmesine olanak tanır.

Bir tür ipucu işlevine yanlış türün geçirilmesi, önemli bir hataya yol açacaktır:

Önemli hata: Yakalanmayan TypeError: foo() öğesine iletilen X bağımsız değişkeni aşağıdakilerden olmalıdır verilen tür RequiredType, ProvidedType

Tip ipucu skaler tipler, diziler ve çağrılabilirler #

Tip ipucu dizi parametreleri (ve PHP 7.1’den sonraki dönüş değerleri) desteği PHP 5.1’e ‘dizi’ anahtar kelimesiyle eklendi. Herhangi bir boyut ve türdeki tüm diziler ve boş diziler geçerli değerlerdir.

PHP 5.4’te tür ipucu çağrılabilirleri için destek eklendi. ‘is_callable()’ olan herhangi bir değer, ‘çağrılabilir’, yani ‘Closure’ nesneleri, işlev adı dizeleri ve ‘dizi(sınıf_adı|nesne, yöntem_adı)’ ima edilen parametreler ve dönüş değerleri için geçerlidir.

İşlev adında “is_callable()” olmayacak şekilde bir yazım hatası olursa, daha az belirgin bir hata mesajı görüntülenir:

Önemli hata: Yakalanmayan TypeError: foo() öğesine iletilen bağımsız değişken 1, verilen çağrılabilir, dize/dizi türünde olmalıdır

function foo(callable $c) {}
foo("count"); // valid
foo("Phar::running"); // valid
foo(["Phar", "running"); // valid
foo([new ReflectionClass("stdClass"), "getName"]); // valid
foo(function() {}); // valid

foo("no_such_function"); // callable expected, string given

Statik olmayan yöntemler, statik biçimde çağrılabilirler olarak da geçirilebilir, bu da sırasıyla PHP 7 ve 5’te bir kullanımdan kaldırma uyarısı ve E_STRICT düzeyinde hata ile sonuçlanır.

Yöntem görünürlüğü dikkate alınır. ‘çağrılabilir’ parametreli_ yöntemin _bağlamının sağlanan çağrılabilir öğeye erişimi yoksa, yöntem yokmuş gibi sona erecektir.

class Foo{
  private static function f(){
    echo "Good" . PHP_EOL;
  }

  public static function r(callable $c){
    $c();
  }
}

function r(callable $c){}

Foo::r(["Foo", "f"]);
r(["Foo", "f"]);

Çıktı:

Önemli hata: Yakalanmayan TypeError: r() öğesine iletilen bağımsız değişken 1 çağrılabilir olmalı, dizi verilmelidir

PHP 7’de skaler tip ipucu desteği eklendi. Bu, ‘boolean’, ‘integer’, ‘float’ ve ‘string’ için tip ipucu desteği aldığımız anlamına geliyor.

<?php

function add(int $a, int $b) {
    return $a + $b;
}

var_dump(add(1, 2)); // Outputs "int(3)"

Varsayılan olarak, PHP, sağlanan herhangi bir argümanı tür ipucuna uyacak şekilde yayınlamaya çalışır. Çağrıyı ‘add(1.5, 2)’ olarak değiştirmek, tam olarak aynı çıktıyı verir, çünkü ‘1.5’ kayan noktası PHP tarafından ‘int’e dönüştürülür.

Bu davranışı durdurmak için, bunu gerektiren her PHP kaynak dosyasının başına ‘declare(strict_types=1);’ eklenmelidir.

<?php

declare(strict_types=1);

function add(int $a, int $b) {
    return $a + $b;
}

var_dump(add(1.5, 2));

Yukarıdaki komut dosyası artık önemli bir hata üretiyor:

Önemli hata: Yakalanmayan TypeError: add() öğesine iletilen bağımsız değişken 1 tamsayı türünde olmalıdır, verilen kayan nokta

Bir İstisna: Özel Tipler #

Bazı PHP işlevleri ‘resource’ türünde bir değer döndürebilir. Bu skaler bir tür değil, özel bir tür olduğundan ipucu yazmak mümkün değildir.

Örnek olarak, “curl_init()” bir “kaynak” ve “fopen()” döndürür. Elbette bu iki kaynak birbiriyle uyumlu değil. Bu nedenle, PHP 7 açıkça “kaynak” ipucu verirken aşağıdaki TypeError’ı daima atar:

TypeError: sample() öğesine iletilen Argüman 1, verilen kaynak, kaynağın bir örneği olmalıdır

Tip ipucu sınıfları ve arayüzler #

PHP 5’te sınıflar ve arayüzler için tip ipucu eklendi.

Sınıf türü ipucu #

<?php

class Student
{
    public $name = 'Chris';
}

class School
{
    public $name = 'University of Edinburgh';
}

function enroll(Student $student, School $school)
{
    echo $student->name . ' is being enrolled at ' . $school->name;
}

$student = new Student();
$school = new School();

enroll($student, $school);

Yukarıdaki komut dosyası çıktıları:

Chris, Edinburgh Üniversitesi’ne kaydoluyor

Arayüz türü ipucu #

<?php

interface Enrollable {};
interface Attendable {};

class Chris implements Enrollable
{
    public $name = 'Chris';
}

class UniversityOfEdinburgh implements Attendable
{
    public $name = 'University of Edinburgh';
}

function enroll(Enrollable $enrollee, Attendable $premises)
{
    echo $enrollee->name . ' is being enrolled at ' . $premises->name;
}

$chris = new Chris();
$edinburgh = new UniversityOfEdinburgh();

enroll($chris, $edinburgh);

Yukarıdaki örnek, öncekiyle aynı çıktıyı verir:

Chris, Edinburgh Üniversitesi’ne kaydoluyor

Kendi kendine yazma ipuçları #

“self” anahtar sözcüğü, değerin yöntemi bildiren sınıfın bir örneği olması gerektiğini belirtmek için bir tür ipucu olarak kullanılabilir.

Null yapılabilir tür ipuçları #

Parametreler #

PHP 7.1’de, tür ipucundan önce ? operatörü kullanılarak null yapılabilir tip ipucu eklendi.

function f(?string $a) {}
function g(string $a) {}

f(null); // valid
g(null); // TypeError: Argument 1 passed to g() must be of the type string, null given

PHP 7.1’den önce, bir parametrenin tür ipucu varsa, boş değerleri kabul etmek için varsayılan bir ’null’ değeri bildirmelidir.

function f(string $a = null) {}
function g(string $a) {}

f(null); // valid
g(null); // TypeError: Argument 1 passed to g() must be of the type string, null given

Dönüş değerleri #

PHP 7.0’da, dönüş türü olan işlevler null döndürmemelidir.

PHP 7.1’de işlevler, null yapılabilir bir dönüş türü ipucu bildirebilir. Ancak, işlev yine de boş değil, boş dönmelidir (yok/boş dönüş deyimleri).

function f() : ?string {
    return null;
}

function g() : ?string {}
function h() : ?string {}

f(); // OK
g(); // TypeError: Return value of g() must be of the type string or null, none returned
h(); // TypeError: Return value of h() must be of the type string or null, none returned

Genel nesneleri ipucu yazın #

PHP nesneleri herhangi bir temel sınıftan (stdClass dahil) miras almadığından, genel bir nesne tipine ipucu veren tip desteği yoktur.

Örneğin, aşağıdaki çalışmayacaktır.

<?php

function doSomething(object $obj) {
    return $obj;
}

class ClassOne {}
class ClassTwo {}

$classOne= new ClassOne();
$classTwo= new ClassTwo();

doSomething($classOne);
doSomething($classTwo);

Ve ölümcül bir hata atacak:

Önemli hata: Yakalanmayan TypeError: doSomething() öğesine iletilen bağımsız değişken 1, bir nesne örneği, verilen OperationOne örneği olmalıdır

Bunun için bir geçici çözüm, hiçbir yöntem tanımlamayan dejenere bir arabirim bildirmek ve tüm nesnelerinizin bu arabirimi uygulamasını sağlamaktır.

<?php

interface Object {}

function doSomething(Object $obj) {
    return $obj;
}

class ClassOne implements Object {}
class ClassTwo implements Object {}

$classOne = new ClassOne();
$classTwo = new ClassTwo();

doSomething($classOne);
doSomething($classTwo);

Tip İpuçları İade Yok(Void) #

PHP 7.1’de “void” dönüş türü eklendi. PHP’nin gerçek bir “void” değeri olmamasına rağmen, programlama dillerinde genellikle hiçbir şey döndürmeyen bir işlevin “void” döndürdüğü anlaşılır. Bu, “null” döndürmekle karıştırılmamalıdır, çünkü “null” döndürülebilen bir değerdir.

function lacks_return(): void {
    // valid
}

Bir “void” dönüşü bildirirseniz, herhangi bir değer döndüremeyeceğinizi veya önemli bir hata alacağınızı unutmayın:

function should_return_nothing(): void {
    return null; // Fatal error: A void function must not return a value
}

Ancak, işlevden çıkmak için return kullanmak geçerlidir:

function returns_nothing(): void {
    return; // valid
}
