ASP.NET Core MVC ile Minimal ASP.NET Core Web API #

ASP.NET Core 1.0 ile MVC ve Web API çerçevesi, ASP.NET Core MVC adlı tek bir çerçevede birleştirildi. MVC ve Web API birçok işlevi paylaştığı için bu iyi bir şeydir, ancak her zaman küçük farklılıklar ve kod tekrarları olmuştur.

Ancak, bu ikisini bir çerçeve içinde birleştirmek, birini diğerinden ayırt etmeyi de zorlaştırdı. Örneğin, “Microsoft.AspNet.WebApi”, yenisini değil, Web API 5.x.x çerçevesini temsil eder. Ancak, Microsoft.AspNetCore.Mvc (sürüm 1.0.0) eklediğinizde, tam kapsamlı paketi alırsınız. Bu, MVC çerçevesinin sunduğu kullanıma hazır özelliklerin tümünü içerecektir. Razor, etiket yardımcıları ve model bağlama gibi.

Yalnızca bir Web API’si oluşturmak istediğinizde, tüm bu özelliklere ihtiyacımız yoktur. Peki, minimalist bir Web API’sini nasıl oluştururuz? Cevap: Microsoft.AspNetCore.Mvc.Core. Yeni dünyada MVC birden çok pakete bölünmüştür ve bu paket, yönlendirme ve yetkilendirme gibi MVC çerçevesinin yalnızca temel bileşenlerini içerir.

Bu örnek için minimal bir MVC API oluşturacağız. JSON biçimlendirici ve CORS dahil. Boş bir ASP.NET Core 1.0 Web Uygulaması oluşturun ve şu paketleri projenize ekleyin.json:

"Microsoft.AspNetCore.Mvc.Core": "1.0.0",
"Microsoft.AspNetCore.Mvc.Cors": "1.0.0",
"Microsoft.AspNetCore.Mvc.Formatters.Json": "1.0.0"

Şimdi başlangıç ​​sınıfında AddMvcCore() kullanarak MVC’yi kaydedebiliriz:

public void ConfigureServices(IServiceCollection services)
{
    services.AddMvcCore()
            .AddCors()
            .AddJsonFormatters();
}

“AddMvcCore”, daha fazla oluşturmaya izin veren bir “IMvcCoreBuilder” örneği döndürür. Ara yazılımı yapılandırmak her zamankiyle aynıdır:

public void Configure(IApplicationBuilder app)
{
    app.UseCors(policy =>
    {
        policy.AllowAnyOrigin();
    });
    app.UseMvc();
}

Kontrolörler #

‘Eski’ Web API, kendi denetleyici temel sınıfıyla birlikte gelir: ‘ApiController’. Yeni dünyada böyle bir şey yok, sadece varsayılan ‘Controller’ sınıfı var. Ne yazık ki, bu oldukça büyük bir temel sınıftır ve model bağlama, görünümler ve JSON.NET’e bağlıdır.

Neyse ki, yeni çerçevede denetleyici sınıflarının, yönlendirme mekanizması tarafından alınması için ‘Denetleyici’den türetilmesi gerekmez. Sadece adı Controller ile eklemek yeterlidir. Bu, kendi denetleyici temel sınıfımızı oluşturmamızı sağlar. Eski zamanların hatırına, ona ‘ApiController’ diyelim:

/// <summary>
/// Base class for an API controller.
/// </summary>
[Controller]
public abstract class ApiController
{
    [ActionContext]
    public ActionContext ActionContext { get; set; }

    public HttpContext HttpContext => ActionContext?.HttpContext;

    public HttpRequest Request => ActionContext?.HttpContext?.Request;

    public HttpResponse Response => ActionContext?.HttpContext?.Response;

    public IServiceProvider Resolver => ActionContext?.HttpContext?.RequestServices;
}

“[Denetleyici]” özelliği, türün veya türetilmiş herhangi bir türün, varsayılan denetleyici bulma mekanizması tarafından denetleyici olarak kabul edildiğini belirtir. “[ActionContext]” özelliği, MVC denetleyiciyi oluşturduğunda özelliğin geçerli “ActionContext” ile ayarlanması gerektiğini belirtir. ActionContext mevcut istek hakkında bilgi sağlar.

ASP.NET Core MVC ayrıca bir ControllerBase sağlar. yalnızca görünüm desteği olmayan bir denetleyici temel sınıfı. Yine de bizimkinden çok daha büyük. Uygun bulursanız kullanın.

Çözüm #

Artık yeni ASP.NET Core MVC çerçevesini kullanarak minimal bir Web API’si oluşturabiliriz. Modüler paket yapısı, sadece ihtiyacımız olan paketleri çekmemize ve yalın ve basit bir uygulama oluşturmamıza olanak tanır.

Komut satırından yeni bir proje oluşturun #

Dotnet komutunu kullanarak tamamen komut satırından yeni bir ASP.NET Core projesi oluşturmak mümkündür.

dotnet new web
dotnet restore
dotnet run

dotnet new web yeni bir “boş” web projesinin iskeletini oluşturur. “web” parametresi, “dotnet” aracına “ASP.NET Core Empty” şablonunu kullanmasını söyler. Şu anda yüklü olan tüm kullanılabilir şablonları göstermek için dotnet new -all kullanın. Diğer anahtar şablonlar arasında “console”, “classlib”, “mvc” ve “xunit” bulunur.

Şablon iskeleden çıkarıldıktan sonra, projeyi çalıştırmak için gereken paketleri geri yükleyebilir (“dotnet restore”) ve derleyip başlatabilirsiniz (“dotnet run”).

Proje çalıştıktan sonra varsayılan bağlantı noktasında mevcut olacaktır: http://localhost:5000

ASP.NET Core’da ortam değişkenini ayarlama [Windows] #

=> Orijinal Gönderi <=

ASP.NET Core, geçerli ortamı belirlemek için “ASPNETCORE_ENVIRONMENT” ortam değişkenini kullanır. Varsayılan olarak, uygulamanızı bu değeri ayarlamadan çalıştırırsanız, otomatik olarak ‘Üretim’ ortamına varsayılan olacaktır.

> dotnet run
Project TestApp (.NETCoreApp,Version=v1.0) was previously compiled. Skipping compilation.

Hosting environment: Production  
Content root path: C:\Projects\TestApp  
Now listening on: http://localhost:5000  
Application started. Press Ctrl+C to shut down.

Windows’ta ortam değişkenini ayarlama

Komut satırında

Windows’ta bulunan setx.exe komutunu kullanarak bir komut isteminden kolayca bir ortam değişkeni ayarlayabilirsiniz. Bir kullanıcı değişkenini kolayca ayarlamak için kullanabilirsiniz:

>setx ASPNETCORE_ENVIRONMENT "Development"

SUCCESS: Specified value was saved.

Ortam değişkeninin geçerli açık pencerede ayarlanmadığına dikkat edin. Güncellenen ortamı görmek için yeni bir komut istemi açmanız gerekecek. Bir yönetici komut istemi açar ve /M anahtarını eklerseniz, sistem değişkenlerini (yalnızca kullanıcı değişkenleri yerine) ayarlamak da mümkündür:

>setx ASPNETCORE_ENVIRONMENT "Development" /M

SUCCESS: Specified value was saved.

PowerShell’i Kullanma Alternatif olarak, değişkeni ayarlamak için PowerShell’i kullanabilirsiniz. PowerShell’de normal kullanıcı ve sistem değişkenlerinin yanı sıra $Env: komutunu kullanarak geçici bir değişken de oluşturabilirsiniz:

$Env:ASPNETCORE_ENVIRONMENT = "Development"

Oluşturulan değişken yalnızca PowerShell oturumunuz boyunca sürer - pencereyi kapattığınızda ortam varsayılan değerine geri döner.

Alternatif olarak, kullanıcı veya sistem ortamı değişkenlerini doğrudan ayarlayabilirsiniz. Bu yöntem, geçerli oturumdaki ortam değişkenlerini değiştirmez, bu nedenle değişikliklerinizi görmek için yeni bir PowerShell penceresi açmanız gerekir. Daha önce olduğu gibi, sistem (Makine) değişkenlerini değiştirmek, yönetici erişimi gerektirecektir.

[Environment]::SetEnvironmentVariable("ASPNETCORE_ENVIRONMENT", "Development", "User")
[Environment]::SetEnvironmentVariable("ASPNETCORE_ENVIRONMENT", "Development", "Machine")

Windows kontrol panelini kullanma Komut isteminin hayranı değilseniz, farenizi kullanarak değişkenlerinizi kolayca güncelleyebilirsiniz! Windows başlat menüsü düğmesini tıklayın (veya Windows tuşuna basın), “ortam değişkenleri"ni arayın ve bunun için Ortam değişkenlerini düzenle’yi seçin. hesabınız:

Bu seçeneğin seçilmesi, Sistem Özellikleri iletişim kutusunu açacaktır.

Sisteminizdeki mevcut ortam değişkenlerinin listesini görüntülemek için Ortam Değişkenleri’ne tıklayın.

ASPNETCORE_ENVIRONMENT adlı bir değişkeniniz olmadığını varsayarak, Yeni… düğmesini tıklayın ve yeni bir hesap ortam değişkeni ekleyin: Tüm değişikliklerinizi kaydetmek için Tamam’ı tıklayın. Yeni ortam değişkenlerinin yüklendiğinden emin olmak için tüm komut pencerelerini yeniden açmanız gerekecektir.
