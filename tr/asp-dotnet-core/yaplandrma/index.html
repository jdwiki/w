Asp.net çekirdeği yapılandırma soyutlamaları sağlar. Çeşitli kaynaklardan yapılandırma ayarlarını yüklemenize ve daha sonra uygulamanız tarafından tüketilebilecek son bir yapılandırma modeli oluşturmanıza olanak tanırlar.

Sözdizimi #

  • “Yapılandırma”
  • string this[string key] { get; Ayarlamak; }
  • IEnumerable<IConfigurationSection> GetChildren();
  • IConfigurationSection GetSection(dize anahtarı);

Başlarken #

Bu örnekte, yeni bir proje için iskele kurduğunuzda ne olduğunu anlatacağız.

İlk olarak, projenize aşağıdaki bağımlılıklar eklenecektir (şu anda project.json dosyası):

"Microsoft.Extensions.Configuration.EnvironmentVariables": "1.0.0",
"Microsoft.Extensions.Configuration.Json": "1.0.0",

Ayrıca, ‘Startup.cs’ dosyanızda, ‘ConfigurationBuilder’ akıcı API’sini kullanarak yapılandırmayı oluşturmaktan sorumlu olacak bir kurucu oluşturacaktır:

  1. Önce yeni bir ConfigurationBuilder oluşturur.
  2. Daha sonra, diğer dosyaların mutlak yolunu hesaplamak için kullanılacak bir temel yol belirler.
  3. Yapılandırma oluşturucuya isteğe bağlı bir “appsettings.json” ekler ve değişikliklerini izler
  4. Opsiyonel bir ortamla ilgili appsettings.environementName.json yapılandırma dosyası ekler.
  5. Ardından ortam değişkenlerini ekler.
public Startup(IHostingEnvironment env)
{
    var builder = new ConfigurationBuilder()
        .SetBasePath(env.ContentRootPath)
        .AddJsonFile("appsettings.json", optional: true, reloadOnChange: true)
        .AddJsonFile($"appsettings.{env.EnvironmentName}.json", optional: true)
        .AddEnvironmentVariables();

    Configuration = builder.Build();
}

Birkaç kaynakta aynı ayar yapılırsa, eklenen en son kaynak kazanır ve değeri seçilir.

Yapılandırma daha sonra dizin oluşturucu özelliği kullanılarak tüketilebilir. İki nokta üst üste : karakteri bir yol sınırlayıcıya hizmet eder.

Configuration["AzureLogger:ConnectionString"]

Bu, bir “AzureLogger” bölümünde “ConnectionString” yapılandırma değerini arayacaktır.

Ortam Değişkenleriyle Çalışma #

ConfigurationBuilder üzerinde .AddEnvironmentVariables() öğesini çağırarak yapılandırmayı ortam değişkenlerinden kaynaklayabilirsiniz.

APPSETTING_ ön ekine sahip ortam değişkenlerini yükleyecektir. Ardından, anahtar yolu ayırıcı olarak iki nokta üst üste : kullanacaktır.

Bu şu anlama gelir: aşağıdaki ortam ayarları:

APPSETTING_Security:Authentication:UserName = a_user_name
APPSETTING_Security:Authentication:Password = a_user_password

Bu json’un eşdeğeri olacak:

{
    "Security" : {
       "Authentication" : {
           "UserName" : "a_user_name",
           "Password" : "a_user_password" 
        } 
    }
}

** Azure Hizmetinin ayarları ortam değişkenleri olarak ileteceğini unutmayın. Önek sizin için şeffaf bir şekilde ayarlanacaktır. Aynısını Azure’da yapmak için AppSettings dikey penceresinde iki Uygulama Ayarı belirlemeniz yeterlidir:

Security:Authentication:UserName         a_user_name
Security:Authentication:Password         a_user_password

Seçenek modeli ve yapılandırması #

Büyük konfigürasyon değer setleriyle uğraşırken, bir tane satın almak için onları yüklemek oldukça kullanışlı olabilir.

Asp.net ile birlikte gelen seçenek modeli, bir “bölümü” dotnet “poco” ile eşleştirmek için uygun bir yol sunar: Örneğin, ‘StorageOptions’ doğrudan bir yapılandırma bölümünden hidratlanabilir b ‘Microsoft.Extensions.Options.ConfigurationExtensions’ paketini ekleyerek ve ‘Configure(IConfiguration config)’ uzantı yöntemini çağırarak.

services.Configure<StorageOptions>(Configuration.GetSection("Storage"));

Bellek yapılandırma kaynağında #

Ayrıca yapılandırmayı, “Dictionary<string,string>” gibi bir bellek nesnesinden de kaynaklayabilirsiniz.

.AddInMemoryCollection(new Dictionary<string, string>
{
    ["akey"] = "a value"
})

Bu, entegrasyon/birim testi senaryolarında yardımcı olabilir.

Bağımlılık Enjeksiyonu Kullanarak Yapılandırmaya Erişme #

Önerilen yaklaşım, bunu yapmaktan kaçınmak ve bunun yerine IOptions<TOptions> ve IServiceCollection.Configure<TOptions> kullanmaktır.

Bununla birlikte, “IConfigurationRoot” uygulamasını geniş çapta kullanılabilir hale getirmek hala oldukça basittir.

Startup.cs yapıcısında, yapılandırmayı oluşturmak için aşağıdaki koda sahip olmalısınız,

 Configuration = builder.Build();

Burada ‘Configuration’, ‘IConfigurationRoot’un bir örneğidir, Ve bu örneği ConfigureServices yöntemindeki Startup.cs içindeki hizmet koleksiyonuna Singleton olarak ekleyin.

 public void ConfigureServices(IServiceCollection services)
 {
    services.AddSingleton<IConfigurationRoot>(provider => Configuration);

Örneğin, artık bir Denetleyici/Hizmet içindeki yapılandırmaya erişebilirsiniz.

  public MyController(IConfigurationRoot config){
      var setting1= config.GetValue<string>("Setting1")
  }
