Aspnet çekirdeği, temel temel kavramlarından biri olarak Dependency Injection ile oluşturulmuştur. Yerleşik olanı seçtiğiniz bir üçüncü taraf kapsayıcı ile değiştirebilmeniz için uygun bir kapsayıcı soyutlaması sunar.

Sözdizimi #

  • IServiceCollection.Add(ServiceDescriptor öğesi);
  • IServiceCollection.AddScoped(ServiceType yazın);
  • IServiceCollection.AddScoped(Type serviceType, Type applicationType);
  • IServiceCollection.AddScoped(Type serviceType, Func<IServiceProvider, nesne> uygulamaFactory);
  • IServiceCollection.AddScoped<TService>()
  • IServiceCollection.AddScoped<TService>(Func<IServiceProvider, TService> uygulamaFactory)
  • IServiceCollection.AddScoped<TService, TImplementation>()
  • IServiceCollection.AddScoped<TService, TImplementation>(Func<IServiceProvider, TImplementation> applicationFactory)
  • IServiceCollection.AddSingleton(ServiceType yazın);
  • IServiceCollection.AddSingleton(Type serviceType, Func<IServiceProvider, nesne> uygulamaFactory);
  • IServiceCollection.AddSingleton(Type serviceType, Type applicationType);
  • IServiceCollection.AddSingleton(Type serviceType, nesne uygulamaÖrneği);
  • IServiceCollection.AddSingleton<TService>()
  • IServiceCollection.AddSingleton<TService>(Func<IServiceProvider, TService> uygulamaFactory)
  • IServiceCollection.AddSingleton<TService>(TService uygulamaÖrneği)
  • IServiceCollection.AddSingleton<TService, TImplementation>()
  • IServiceCollection.AddSingleton<TService, TImplementation>(Func<IServiceProvider, TImplementation> applicationFactory)
  • IServiceCollection.AddTransient(ServiceType yazın);
  • IServiceCollection.AddTransient(Type serviceType, Func<IServiceProvider, nesne> uygulamaFactory);
  • IServiceCollection.AddTransient(Type serviceType, Type applicationType);
  • IServiceCollection.AddTransient<TService>()
  • IServiceCollection.AddTransient<TService>(Func<IServiceProvider, TService> uygulamaFactory)
  • IServiceCollection.AddTransient<TService, TImplementation>()
  • IServiceCollection.AddTransient<TService, TImplementation>(Func<IServiceProvider, TImplementation> applicationFactory)
  • IServiceProvider.GetService(ServiceType Türü)
  • IServiceProvider.GetService<T>()
  • IServiceProvider.GetServices(Tip serviceType)
  • IServiceProvider.GetServices<T>()

“IServiceProvider” yöntemlerinin genel türevlerini kullanmak için aşağıdaki ad alanını eklemeniz gerekir:

using Microsoft.Extensions.DependencyInjection;

Bağımlılıkları kaydet #

Yerleşik kapsayıcı, bir dizi yerleşik özellik ile birlikte gelir:

Ömür boyu kontrol #

public void ConfigureServices(IServiceCollection services)
    {
        // ...
    
        services.AddTransient<ITestService, TestService>();
        // or
        services.AddScoped<ITestService, TestService>();
        // or
        services.AddSingleton<ITestService, TestService>();
        // or
        services.AddSingleton<ITestService>(new TestService());
    }
  • AddTransient: Her çözüldüğünde oluşturulur
  • AddScoped: İstek başına bir kez oluşturulur
  • AddSingleton: Uygulama başına bir kez tembelce oluşturulur
  • AddSingleton (örnek): Uygulama başına önceden oluşturulmuş bir örnek sağlar

Numaralandırılabilir bağımlılıklar #

Numaralandırılabilir bağımlılıkları kaydetmek de mümkündür:

 services.TryAddEnumerable(ServiceDescriptor.Transient<ITestService, TestServiceImpl1>());
 services.TryAddEnumerable(ServiceDescriptor.Transient<ITestService, TestServiceImpl2>());

Daha sonra bunları aşağıdaki gibi tüketebilirsiniz:

public class HomeController : Controller
{
    public HomeController(IEnumerable<ITestService> services)
    {
        // do something with services.
    }
}

Genel bağımlılıklar #

Ayrıca genel bağımlılıkları da kaydedebilirsiniz:

services.Add(ServiceDescriptor.Singleton(typeof(IKeyValueStore<>), typeof(KeyValueStore<>)));

Ve sonra aşağıdaki gibi tüketin:

public class HomeController : Controller
{
    public HomeController(IKeyValueStore<UserSettings> userSettings)
    {
        // do something with services.
    }
}

Kaydolun ve manuel olarak çözün #

Bağımlılıkları tanımlamanın tercih edilen yolu, Açık Bağımlılıklar İlkesi izleyen yapıcı enjeksiyonunu kullanmaktır:

ITestService.cs

public interface ITestService
{
    int GenerateRandom();
}

TestService.cs

public class TestService : ITestService
{
    public int GenerateRandom()
    {
        return 4;
    }
}

Startup.cs (ConfigureServices)

public void ConfigureServices(IServiceCollection services)
{
    // ...

    services.AddTransient<ITestService, TestService>();
}

HomeController.cs

using Microsoft.Extensions.DependencyInjection;

namespace Core.Controllers
{
    public class HomeController : Controller
    {
        public HomeController(ITestService service)
        {
            int rnd = service.GenerateRandom();
        }
    }
}

Bir Denetleyici Eylemine bağımlılık enjekte etme #

Daha az bilinen bir yerleşik özellik, “FromServicesAttribute” kullanılarak Denetleyici Eylemi enjeksiyonudur.

[HttpGet]
public async Task<IActionResult> GetAllAsync([FromServices]IProductService products)
{
     return Ok(await products.GetAllAsync());
}

Önemli bir not, [FromServices] genel “Property Injection” veya “Method Injection” mekanizması olarak kullanılamaz! Yalnızca bir denetleyici eyleminin veya denetleyici yapıcısının yöntem parametrelerinde kullanılabilir (yapıcıda eskidir, ancak ASP.NET Core DI sistemi zaten yapıcı enjeksiyonunu kullandığından ve fazladan işaretleyici gerekli değildir).

Kontrolörler, kontrolör eylemi dışında hiçbir yerde kullanılamaz. Ayrıca ASP.NET Core MVC’ye çok özeldir ve Microsoft.AspNetCore.Mvc.Core derlemesinde bulunur.

Bu öznitelikle ilgili olarak ASP.NET Core MVC GitHub sorunundan ([[FromServices]‘i yalnızca parametrelere1 uygulanacak şekilde sınırlayın) orijinal alıntı:

@rynowak:

@Eilon:

Özelliklerle ilgili sorun, birçok kişiye, herhangi bir nesnenin herhangi bir özelliğine uygulanabilirmiş gibi görünmesidir.

Kabul edildi, bu özelliğin nasıl kullanılması gerektiği konusunda kafa karışıklığı yaşayan kullanıcılar tarafından yayınlanan bir dizi sorunla karşılaştık. Hem “[FromServices] tuhaf ve bundan hoşlanmadım” hem de “[FromServices] beni şaşırttı” türünden oldukça fazla miktarda geri bildirim oldu. Bir tuzak gibi geliyor ve takımın bundan yıllar sonra hala soruları yanıtlayacağı bir şey.

[FromServices] için en değerli senaryonun, yalnızca o yerde ihtiyacınız olan bir hizmet için bir eylemin yöntem parametresinde olduğunu düşünüyoruz.

/cc @danroth27 - doküman değişiklikleri

Mevcut [FromServices]’e aşık olan herkese, özellik enjeksiyonu yapabilen bir DI sistemine bakmayı şiddetle tavsiye ederim (örneğin Autofac).

Notlar:

  • .NET Core Dependency Injection sistemine kayıtlı tüm** hizmetler, “[FromServices]” özniteliği kullanılarak bir denetleyici eyleminin içine enjekte edilebilir.
  • En alakalı durum, yalnızca tek bir eylem yönteminde bir hizmete ihtiyacınız olduğunda ve denetleyicinizin yapıcısını yalnızca bir kez kullanılacak başka bir bağımlılıkla karıştırmak istemediğiniz zamandır.
  • ASP.NET Core MVC (yani saf .NET Framework veya .NET Core konsol uygulamaları) dışında kullanılamaz, çünkü Microsoft.AspNetCore.Mvc.Core derlemesinde bulunur.
  • Özellik veya yöntem enjeksiyonu için mevcut üçüncü taraf IoC kaplarından birini (Autofac, Unity, vb.) kullanmalısınız.

Uygulama başlatma / Veritabanı Tohumlama sırasında kapsamlı hizmetleri kullanma #

Kapsamlı hizmetleri uygulama başlatma sırasında çözümlemek zor olabilir, çünkü istek ve dolayısıyla kapsamlı hizmet yoktur.

Kapsamlı bir hizmetin uygulama başlatma sırasında app.ApplicationServices.GetService<AppDbContext>() aracılığıyla çözümlenmesi sorunlara neden olabilir, çünkü global kapsayıcı kapsamında oluşturulacak ve bu da onu uygulamanın ömrü boyunca etkin bir şekilde tek bir hale getirecektir. “DbContext enjekte edilirken ASP.NET Core’da atılan bir nesneye erişilemiyor” gibi istisnalara yol açabilir.

Aşağıdaki kalıp, sorunu önce yeni bir kapsam oluşturup ardından kapsamlı hizmetleri ondan çözerek, ardından iş tamamlandıktan sonra kapsamlı kapsayıcıyı atarak çözer.

public Configure(IApplicationBuilder app)
{
    // serviceProvider is app.ApplicationServices from Configure(IApplicationBuilder app) method
    using (var serviceScope = app.ApplicationServices.GetRequiredService<IServiceScopeFactory>().CreateScope())
    {
        var db = serviceScope.ServiceProvider.GetService<AppDbContext>();

        if (await db.Database.EnsureCreatedAsync())
        {
            await SeedDatabase(db);
        }
    }
}

Bu, Entity Framework çekirdek ekibinin uygulama başlatma sırasında verileri tohumlamak için yarı resmi bir yoludur ve MusicStore örneği uygulamasında yansıtılır.

Bir Denetleyicideki bağımlılıkları alın #

Kaydedildikten sonra, Denetleyici yapıcısına parametreler eklenerek bir bağımlılık alınabilir.

// ...
using System;
using Microsoft.Extensions.DependencyInjection;

namespace Core.Controllers
{
    public class HomeController : Controller
    {
        public HomeController(ITestService service)
        {
            int rnd = service.GenerateRandom();
        }
    }
}

Seçenekler kalıbı / Seçenekleri hizmetlere enjekte etme #

ASP.NET Core ile Microsoft ekibi ayrıca, güçlü yazılan seçeneklere sahip olmanızı sağlayan ve seçenekleri hizmetlerinize enjekte etme yeteneğini yapılandıran Seçenekler modelini de tanıttı.

İlk önce, konfigürasyonumuzu tutacak güçlü bir yazı tipiyle başlıyoruz.

public class MySettings 
{
    public string Value1 { get; set; }
    public string Value2 { get; set; }
}

Ve ‘appsettings.json’ içinde bir giriş.

{
  "mysettings" : {
    "value1": "Hello",
    "value2": "World"
  }
}

Ardından Startup sınıfında başlatıyoruz. Bunu yapmanın iki yolu var

  1. Doğrudan appsettings.json “mysettings” bölümünden yükleyin

    services.Configure<MySettings>(Configuration.GetSection("mysettings"));

  2. Manuel olarak yapın

    services.Configure<MySettings>(new MySettings 
{
    Value1 = "Hello",
    Value2 = Configuration["mysettings:value2"]
});

“appsettings.json” dosyasının her hiyerarşi düzeyi bir “:” ile ayrılır. “değer2”, “mysettings” nesnesinin bir özelliği olduğundan, ona “mysettings:value2” aracılığıyla erişiriz.

Son olarak, IOptions<T> arayüzünü kullanarak seçenekleri hizmetlerimize enjekte edebiliriz.

public class MyService : IMyService
{
    private readonly MySettings settings;

    public MyService(IOptions<MySettings> mysettings) 
    {
        this.settings = mySettings.Value;
    }
}

Notlar #

“IOptions” başlatma sırasında yapılandırılmazsa, “IOptions” öğesinin enjekte edilmesi, “T” sınıfının varsayılan örneğini enjekte edecektir.

Dependency Injection ile Denetleyicileri, ViewComponents’ı ve TagHelper’ları çözümleyin #

Varsayılan olarak Denetleyiciler, ViewComponents ve TagHelpers, bağımlılık ekleme kapsayıcısı aracılığıyla kaydedilmez ve çözümlenmez. Bu, AutoFac gibi bir 3. taraf Kontrol Tersine Çevirme (IoC) kapsayıcısı kullanırken, yani özellik enjeksiyonunun yapılamamasına neden olur.

ASP.NET Core MVC’nin bu Türleri IoC aracılığıyla da çözmesini sağlamak için, ‘Startup.cs’ dosyasına aşağıdaki kayıtların eklenmesi gerekir (GitHub’daki resmi ControllersFromService örneğinden alınmıştır)

public void ConfigureServices(IServiceCollection services)
{
    var builder = services
        .AddMvc()
        .ConfigureApplicationPartManager(manager => manager.ApplicationParts.Clear())
        .AddApplicationPart(typeof(TimeScheduleController).GetTypeInfo().Assembly)
        .ConfigureApplicationPartManager(manager =>
        {
            manager.ApplicationParts.Add(new TypesPart(
              typeof(AnotherController),
              typeof(ComponentFromServicesViewComponent),
              typeof(InServicesTagHelper)));

            manager.FeatureProviders.Add(new AssemblyMetadataReferenceFeatureProvider());
        })
        .AddControllersAsServices()
        .AddViewComponentsAsServices()
        .AddTagHelpersAsServices();

    services.AddTransient<QueryValueService>();
    services.AddTransient<ValueService>();
    services.AddSingleton<IHttpContextAccessor, HttpContextAccessor>();
}

Düz Bağımlılık Enjeksiyonu örneği (Startup.cs olmadan) #

Bu, Microsoft.Extensions.DependencyInjection nuget paketini kestrel’den “WebHostBuilder” kullanmadan nasıl kullanacağınızı gösterir (örn. bir webApp’tan başka bir şey oluşturmak için):

internal class Program
{
    public static void Main(string[] args)
    {
        var services = new ServiceCollection(); //Creates the service registry
        services.AddTransient<IMyInterface, MyClass>(); //Add registration of IMyInterface (should create an new instance of MyClass every time)
        var serviceProvider = services.BuildServiceProvider(); //Build dependencies into an IOC container
        var implementation = serviceProvider.GetService<IMyInterface>(); //Gets a dependency

        //serviceProvider.GetService<ServiceDependingOnIMyInterface>(); //Would throw an error since ServiceDependingOnIMyInterface is not registered
        var manualyInstaniate = new ServiceDependingOnIMyInterface(implementation); 

        services.AddTransient<ServiceDependingOnIMyInterface>();
        var spWithService = services.BuildServiceProvider(); //Generaly its bad practise to rebuild the container because its heavey and promotes use of anti-pattern.
        spWithService.GetService<ServiceDependingOnIMyInterface>(); //only now i can resolve
    }
}

interface IMyInterface
{
}

class MyClass : IMyInterface
{
}

class ServiceDependingOnIMyInterface
{
    private readonly IMyInterface _dependency;

    public ServiceDependingOnIMyInterface(IMyInterface dependency)
    {
        _dependency = dependency;
    }
}

Microsoft.Extensions.DependencyInjection’ın iç işleyişi #

IServiceKoleksiyon #

Microsoft’un DI nuget paketiyle bir IOC kapsayıcı oluşturmaya başlamak için bir “IServiceCollection” oluşturmaya başlayın. Halihazırda sağlanan Koleksiyonu kullanabilirsiniz: ServiceCollection:

var services = new ServiceCollection();

Bu ‘IServiceCollection’, ‘IList, ICollection, IEnumerable, IEnumerable’ın uygulanmasından başka bir şey değildir.

Aşağıdaki yöntemlerin tümü, yalnızca “ServiceDescriptor” örneklerini listeye eklemek için kullanılan uzantı yöntemleridir:

services.AddTransient<Class>(); //add registration that is always recreated
services.AddSingleton<Class>(); // add registration that is only created once and then re-used
services.AddTransient<Abstract, Implementation>(); //specify implementation for interface
services.AddTransient<Interface>(serviceProvider=> new Class(serviceProvider.GetService<IDependency>())); //specify your own resolve function/ factory method.
services.AddMvc(); //extension method by the MVC nuget package, to add a whole bunch of registrations.
// etc..

//when not using an extension method:
services.Add(new ServiceDescriptor(typeof(Interface), typeof(Class)));

IServiceSağlayıcı #

Hizmet sağlayıcı, hızlı bir şekilde kullanılabilecekleri şekilde tüm kayıtları ‘Derleyen’dir, bu, temelde aşağıdakiler için bir uzantı yöntemi olan ‘services.BuildServiceProvider()’ ile yapılabilir:

var provider = new ServiceProvider( services, false); //false is if it should validate scopes

Perde arkasında, ‘IServiceCollection’daki her ‘ServiceDescriptor’, nesnenin dönüş tipi olduğu ve şu şekilde olduğu ‘Func<ServiceProvider, object>’ fabrika yöntemine derlenir: Uygulama türünün oluşturulmuş örneği, Singleton veya kendi tanımlı fabrikanız yöntem.

Bu kayıtlar, temelde bir “ConcurrentDictionary” olan “ServiceTable"a, anahtarı “ServiceType” ve yukarıda tanımlanan Factory yönteminin değeri olan “ServiceTable"a eklenir.

Sonuç #

Şimdi, bizim için Hizmetler oluşturmak için aynı anda kullanabileceğimiz bir `ConcurrentDictionary<Type, Func<ServiceProvider, object»’ var. Bunun nasıl görünebileceğine dair temel bir örnek göstermek için.

  var serviceProvider = new ConcurrentDictionary<Type, Func<ServiceProvider, object>>();
  var factoryMethod = serviceProvider[typeof(MyService)];
  var myServiceInstance = factoryMethod(serviceProvider)

Bu iş böyle değil!

Bu “ConcurrentDictionary”, “ServiceProvider"ın bir özelliği olan “ServiceTable"ın bir özelliğidir.
