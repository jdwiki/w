.NET Core projesinde Angular 2 bileşenleri oluşturulurken beklenen hatalar (sürüm 0.8.3) #

Bir .NET Core projesinde yeni Angular 2 bileşenleri oluştururken aşağıdaki hatalarla karşılaşabilirsiniz (0.8.3 sürümünden itibaren):

Error locating module for declaration
        SilentError: No module files found

VEYA

No app module found. Please add your new Class to your component. 
        Identical ClientApp/app/app.module.ts

[ÇÖZÜM]

  1. app.module.client.ts’yi app.client.module.ts olarak yeniden adlandırın

  2. app.client.module.ts’yi açın: bildirimin başına 3 nokta “…” ekleyin ve bildirimi parantez içine alın.

    For example: [...sharedConfig.declarations, <MyComponent>]

  3. boot-client.ts’yi açın: yeni dosyayı kullanmak için içe aktarmanızı güncelleyin. app.client.module reference.

    For example: import { AppModule } from './app/app.client.module';

  4. Şimdi yeni bileşeni oluşturmaya çalışın: ng component my-bileşenim

[AÇIKLAMA]

Angular CLI, projenizde app.module.ts adlı bir dosya arar ve bileşeni içe aktarmak için bildirimler özelliği için bir başvuru bulmaya çalışır. Bu bir dizi olmalıdır (sharedConfig.declarations olduğu gibi), ancak değişiklikler uygulanmaz
