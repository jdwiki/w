EJB’yi JBoss AS 7.1 ile kurma #

1. genel bakış

Bu yazıda Enterprise JavaBeans’e (EJB) nasıl başlayacağınızı tartışacağız. JBoss AS 7.1.1.Final’i kullanacağız, ancak istediğiniz sunucuyu kullanmakta özgürsünüz.

2. Bean için Maven Bağımlılıkları

EJB’yi kullanmak için, pom.xml dosyanızın bağımlılıklar bölümüne EJB’nin en son sürümünü eklediğinizden emin olun:

<dependency>
    <groupId>org.jboss.spec.javax.ejb</groupId>
    <artifactId>jboss-ejb-api_3.2_spec</artifactId>
    <version>1.0.0.Final</version>
</dependency>

Bu eğitimde JBoss’u uygulama sunucumuz olarak kullanacağımızdan, JBoss bağımlılıklarını doğru şekilde eklediğinizden emin olun. Eğitimin sonraki bölümünde, proje için maven yapısının nasıl kurulacağını ayrıntılı olarak tartışacağız.

3. EJB Uzaktan Kumanda

İlk olarak HelloWorldRemote isimli Bean Interface’i oluşturalım.

public interface HelloWorldRemote {
    public String getHelloWorld();
}

Şimdi yukarıdaki arayüzü uygulayacağız ve onu HelloWorldBean olarak adlandıracağız.

@Stateless
public class HelloWorldBean implements HelloWorldRemote {

    public HelloWorldBean() {
    
    }
    
    @Override
    public String getHelloWorld(){
        return ("Hello World");
    }
}

Sınıf bildiriminin üstündeki @Stateless gösterimine dikkat edin. Durum bilgisi olmayan bir oturum çekirdeğini belirtir.

4. Remote Bean için Maven Kurulumu

Bu bölümde, uygulamayı sunucuda oluşturmak ve çalıştırmak için maven’in nasıl kurulacağını tartışacağız.

Eklentileri tek tek inceleyelim.

4.1. Derleyici Eklentisi

Maven-compiler-plugin, projemizin kaynaklarını derlemek için kullanılır.

Burada eklentinin 2.3.1 sürümünü, kaynak ve hedef JDK’nın konfigürasyon altında 1.7’ye ayarlandığı şekilde kullandık.

Bu ayarları etiketinin içindeki özellikler olarak tanımladık ve onu ${property} üzerinden yönlendirdik.

<version.compiler.plugin>2.3.1</version.compiler.plugin>
<!-- maven-compiler-plugin -->
<maven.compiler.target>1.7</maven.compiler.target>
<maven.compiler.source>1.7</maven.compiler.source>

4.2 EJB Eklentisi

Bu eklenti, Bean dosyasını ve ilişkili istemci kavanozunu oluşturur.

ejb versiyonunu 3.2 olarak belirledik ve client’ı oluşturan createClient özelliği true olarak ayarlandı.

4.3 JBoss’ta Dağıtım

jboss-as-maven-plugin, uygulamayı JBoss AS 7’de dağıtmak, yeniden dağıtmak, dağıtmak veya çalıştırmak için kullanılır.

Bu yapılandırmada, yapı dosyası adını, bizim durumumuzda ejb-remote-1.0-SNAPSHOT varsayılan olarak yapay-sürüm biçiminde olan proje derleme dosya adıyla aynı şekilde belirtiriz.

4.4 EJB için Gerekli Maven Bağımlılıkları

jboss-javaee-6.0, kullanmak istediğimiz JBoss’un Java EE 6 API’lerinin sürümünü tanımlar.

JBoss, bir Malzeme Listesi (BOM) dahil olmak üzere eksiksiz bir Java EE 6 API seti dağıtır.

Malzeme Listesi, bir yapı yığınının (veya koleksiyonunun) sürümlerini belirtir. Her zaman doğru yapıt sürümlerini alabilmemiz için bunu etiketinde belirtiriz. Bu bağımlılığın türü, gerekli bağımlılıkları içeren bir pom.

<dependency>
    <groupId>org.jboss.spec</groupId>
    <artifactId>jboss-javaee-6.0</artifactId>
    <version>${version.org.jboss.spec.jboss.javaee.6.0}</version>
    <type>pom</type>
    <scope>import</scope>
</dependency>

4.5 Ek Açıklamalar

Aşağıdakiler ek açıklama bağımlılığını alacaktır:

<dependency>
    <groupId>org.jboss.spec.javax.annotation</groupId>
    <artifactId>jboss-annotations-api_1.1_spec</artifactId>
    <scope>provided</scope>
</dependency>

4.6 EJB sürüm 3.2

Aşağıdaki kod parçasında, spesifikasyonların en son sürümünü alıyoruz:

<dependency>
    <groupId>org.jboss.spec.javax.ejb</groupId>
    <artifactId>jboss-ejb-api_3.2_spec</artifactId>
    <version>1.0.0.Final</version>
</dependency>

Yukarıdaki projeyi bir JBoss sunucusunda çalıştırmak için önce şunu çalıştırmamız gerekiyor:

mvn clean install

Ardından, aşağıdaki maven komutunu çalıştırarak onu çalışan bir JBoss sunucusuna dağıtmamız gerekiyor:

jboss-as:deploy

Şimdi jboss sunucusunda konuşlandırılan jar dosyasını görmelisiniz.

Alternatif olarak, mevcut kavanozu projedeki hedef klasörden kopyalayabilir ve sunucunun webapp klasörüne yapıştırabilirsiniz.

5. İstemci Projesini Ayarlama

Remote bean’i oluşturduktan sonra, bir client oluşturarak konuşlandırılmış bean’i test etmeliyiz.

Öncelikle proje için maven kurulumunu tartışalım.

5.1 Maven Eklentileri kullanıldı

Maven-compiler-plugin, projenizin kaynaklarını derlemek için kullanılır.

Kaynak ve hedef sınıflar için jdk 1.7 sürümünü belirledik.

İstemcimiz bir Java programıdır, onu çalıştırmak için sistem ve Java programlarını çalıştırmaya yardımcı olan ’exec-maven-plugin’i kullanırız. Yürütülebilir dosyayı (yani Java), sınıf yolunu ve Java sınıfını (com.baeldung.ejb.client.Client) belirtmemiz gerekiyor.

Eklenti, sağlanan bağımlılıklara dayalı olarak gerekli sınıf yolu argümanlarını içerdiğinden sınıf yolu boş bırakılır.

EJB3 istemcisi için 5.2 Maven Bağımlılıkları

EJB3 istemcisini çalıştırmak için aşağıdaki bağımlılıkları eklememiz gerekiyor.

İstemciyi çalıştırmak için bu uygulamanın EJB uzak iş arayüzlerine güveniyoruz. Bu yüzden ejb client jar bağımlılığını belirtmemiz gerekiyor. Bu projenin EJB istemci kavanozuna bağımlılığını belirtmek için “ejb-client” değerine sahip etiketi kullanılır.

<dependency>
    <groupId>com.theopentutorials.ejb3</groupId>
    <artifactId>ejbmavendemo</artifactId>
    <type>ejb-client</type>
    <version>${project.version}</version>
</dependency>

“jboss-transaction-api_1.1_spec”, “jboss-ejb-api_3.1_spec”, “jboss-ejb-client”, “xnio-api”, “xnio-nio”, “jboss-remoting”, “jboss” bağımlılıkları -sasl, jboss-marshalling-river` çalışma zamanı kapsamına sahiptir çünkü bunlar derleme zamanı değil, gerekli çalışma zamanıdır.

bağımlılık Yönetimi altındaki jboss-javaee-6.0 ve jboss-as-ejb-client-bom bağımlılıkları içe aktarma kapsamına sahiptir. Bu, uzak bir POM’dan geçerli projeye bağımlılık yönetimi bilgilerini dahil etmek için kullanılır. Bu uzak POM’lar, istemciyi çalıştırmak için gerekli bağımlılıkları içeren JBoss tarafından sağlanır.

5.3 JBoss EJB İstemci özellikleri

“src/main/resources” altında bir dosya oluşturun ve onu jboss-ejb-client.properties olarak adlandırın.

remote.connectionprovider.create.options.org.xnio.Options.SSL_ENABLED=false
remote.connections=default
remote.connection.default.host=localhost
remote.connection.default.port = 4447
remote.connection.default.connect.options.org.xnio.Options.SASL_POLICY_NOANONYMOUS=false

6. İstemci Sınıfını Oluşturma

İlk önce bir ClientUtility sınıfı oluşturuyoruz:

public class ClientUtility {
    private static Context initialContext;
    private static final String PKG_INTERFACES = "org.jboss.ejb.client.naming";

    public static Context getInitialContext() throws NamingException {
        if (initialContext == null) {
            Properties properties = new Properties();
            properties.put(Context.URL_PKG_PREFIXES, PKG_INTERFACES);
            initialContext = new InitialContext(properties);
         }
        return initialContext;
    }
}

Şimdi sunucuya yerleştirdiğimiz bean’i tüketecek gerçek Client sınıfını oluşturalım:

public class Client {
    
    //The lookup method to get the EJB name
    private static HelloWorldRemote doLookup() {
        Context context = null;
        HelloWorldRemote bean = null;
        try {
            // 1. Obtaining Context
            context = ClientUtility.getInitialContext();
            // 2. Generate JNDI Lookup name
            String lookupName = getLookupName();
            // 3. Lookup and cast
            bean = (HelloWorldRemote) context.lookup(lookupName);
 
        } catch (NamingException e) {
            e.printStackTrace();
        }
        return bean;
    }
 
    private static String getLookupName() {
        
         // The app name is the EAR name of the deployed EJB without .ear suffix.
         // Since we haven't deployed the application as a .ear, the app name for
         // us will be an empty string
         
        String appName = "";
 
        
         // The module name is the JAR name of the deployed EJB without the .jar
         // suffix.
        String moduleName = "ejb-remote-0.0.1-SNAPSHOT";
 
        
        // AS7 allows each deployment to have an (optional) distinct name. This
        // can be an empty string if distinct name is not specified.
        String distinctName = "";
 
        // The EJB bean implementation class name
        String beanName = "HelloWorldBean";
 
        // Fully qualified remote interface name
        final String interfaceName = "com.baeldung.ejb.tutorial.HelloWorldRemote";
 
        // Create a look up string name
        String name = "ejb:" + appName + "/" + moduleName + "/" + distinctName
                + "/" + beanName + "!" + interfaceName;
        
        return name;
    }
}

Client sınıfı fasulyeyi tüketir ve sonucu verir.

7. Çözüm

Böylece bir EJB sunucusu ve hizmeti tüketen bir istemci oluşturduk. Proje herhangi bir Uygulama Sunucusunda çalıştırılabilir.
