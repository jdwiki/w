Web Uygulamanızı Oturum Açma ve Oturumu Kapatma ile koruyun #

Bu örnek çok basit bir Spring Boot uygulamasıdır.

Maven Bağımlılıkları #

İlk önce projenize aşağıdaki bağımlılıkları ekleyin. Yeni bir proje oluşturduğunuzda Spring Initializr önerilir.

<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>1.5.1.RELEASE</version>
    <relativePath/>
</parent>

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-security</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-thymeleaf</artifactId>
    </dependency>
    <dependency>
        <groupId>org.thymeleaf.extras</groupId>
        <artifactId>thymeleaf-extras-springsecurity4</artifactId>
    </dependency>
</dependencies>

Web Uygulamanızı oluşturun #

Web siteleri ve denetleyici ile bir web uygulaması oluşturun. Örneğin, yalnızca bir sayfa (index.html) ve giriş sayfası için bir giriş içeren bu çok küçük web uygulaması.

@Configuration
public class MvcConfig extends WebMvcConfigurerAdapter{
    
    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addRedirectViewController("/", "index");
        registry.addViewController("/index").setViewName("index");
        registry.addViewController("/login").setViewName("login");
    }
}

Web Uygulamanızı güvenli hale getirin #

Web uygulamanızın güvenliğini sağlamak için Spring Security’yi yapılandırın. Örneğin. yalnızca kimliği doğrulanmış kullanıcıların herhangi bir isteğine izin verin. js ve css gibi statik kaynaklara izin verin, aksi takdirde kimliği doğrulanmamış kullanıcılar için yüklenmeyecektir. Oturum açma ve oturum kapatma sayfasını bu kuralın dışında tutun ve bir test kullanıcısı oluşturun:

@Configuration
@EnableWebSecurity
public class WebSecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.authorizeRequests()
                .antMatchers("/css/*.css", "/js/*.js").permitAll()
                .anyRequest().authenticated()
                .and()
                .formLogin()
                .loginPage("/login")
                .permitAll()
                .and()
                .logout()
                .permitAll();
    }

    @Autowired 
    public void configureGlobal(AuthenticationManagerBuilder auth) throws Exception { 
        auth.inMemoryAuthentication() 
                .withUser("user").password("password").roles("USER"); 
    } 
}

Giriş sayfasını oluşturun #

Giriş sayfasının, “/login” için gönderi isteğinde bulunan bir forma sahip olması gerekir:

<!DOCTYPE html>
<html xmlns="http://www.w3.org/1999/xhtml"
      xmlns:th="http://www.thymeleaf.org">

    <head>
        <title>Login</title>
        <link th:href="@{/css/stylesheet.css}" rel="stylesheet" type="text/css"/>
        <script type="text/javascript" th:src="@{/js/login.js}"></script>
    </head>
    <body>
        <!-- show notification on error -->
        <div th:if="${param.error}">
            Invalid username or password.
        </div>

        <!-- show notification of logout -->
        <div th:if="${param.logout}">
            You have been logged out.
        </div>

        <!-- login form -->
        <div>
            <form th:action="@{/login}" method="post">
                <h2 >Please sign in</h2>
                <label >User Name</label>
                <input type="text" name="username" th:required="required" th:autofocus="autofocus"/>
                <br/>
                
                <label>Password</label>
                <input type="password" name="password" th:required="required" />
                <br/>
               
                <input type="submit" value="Sign In"/>
            </form>
        </div>
    </body>
</html>

Kullanıcı yanlış kullanıcı adı/şifre girdiğinde hata parametresi ayarlanır. Kullanıcı oturumu kapattığında, oturum kapatma parametresi ayarlanır. Bu, ilgili mesajları göstermek için kullanılır.

Kullanıcı özelliklerine erişin #

Başarılı bir oturum açtıktan sonra kullanıcı index.html‘ye yönlendirilir. Spring Security Dialect, kullanıcı adı gibi kullanıcı özelliklerine erişmemizi sağlar:

<!DOCTYPE html>
<html xmlns="http://www.w3.org/1999/xhtml"
      xmlns:th="http://www.thymeleaf.org">
    <head>
        <title>Index</title>
    </head>
    <body>
        <div>
            <h3>Welcome <span th:text="${#authentication.name}"/></h3>
            <form th:action="@{/logout}" method="post"> 
                <input type="submit" value="Logout"/> 
            </form>
        </div>
    </body>
</html>

“/logout” için gönderilen istek yoluyla bir çıkış yapılır

Yalnızca kimliği doğrulanmış kullanıcılar için bir şeyler gösteriliyor #

<div sec:authorize="isAuthenticated()">
    This text is displayed for authenticated users.    
</div>

Kullanıcı adını göster #

Kimliği doğrulanmış kullanıcılar için kullanıcı adını gösterebilirsiniz

<div sec:authorize="isAuthenticated()">
    Welcome, <span sec:authentication="name">Username</span>     
</div>

Farklı içeriklere farklı roller göster #

sec:authorize özniteliği, öznitelik ifadesi doğru olarak değerlendirildiğinde içeriğini işler.

<div sec:authorize="hasRole('ROLE_ADMIN')">
    Content for administrators
</div>
<div sec:authorize="hasRole('ROLE_USER')">
    Content for users
</div>

sec:authentication niteliği, günlüğe kaydedilen kullanıcı rollerini yazdırmak için kullanılır:

Roles: <span sec:authentication="principal.authorities">ROLE_USER, ROLE_ADMIN</span>
