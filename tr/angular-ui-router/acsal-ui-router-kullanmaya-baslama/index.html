Merhaba Dünya Örneği #

ADIM 1: Kurulum

Angular-UI Router’ı kullanmadan önce, projenize AngularJS’nin kendisini dahil etmelisiniz. Bununla ilgili ayrıntılı bir kılavuz için bu belgelere bakın.

Angular-UI Router’ı sırasıyla GitHub-Page veya NuGet, NPM, Bower’dan indirebilirsiniz.

JS dosyasını web sayfanıza ekledikten sonra uygulamanızın içine ui.router modülünü ekleyebilirsiniz. Komut dosyanızda şöyle bir şeye sahip olmalısınız:

var app = angular.module('app', []);

ve şimdi Angular-UI Router’ı kendi uygulamamıza şu şekilde enjekte edeceğiz:

var app = angular.module('app', ['ui.router']);

Şimdi Angular-UI Router uygulamamız ile birlikte yüklenecektir. Aşağıdaki adımlar, Angular-UI Router’ın temellerini açıklayacak ve bazı temel işlevleri gösterecektir.

ADIM 2: Basit durumları tanımlama

UI-Router’ı Angular config işlevi içinde yapılandırabilirsiniz. Durumlarınızı tanımlamak için $stateProvider kullanın. Aşağıdaki örnekte, her durumun bir url’si, denetleyicisi ve bir şablonu vardır.

(function() {
  var app = angular.module('app', ['ui.router']);
  
  app.config(['$stateProvider', function($stateProvider) {
      $stateProvider
        .state('home', {
          url: "/home",
          templateUrl: "home.html",
          controller: "homeCtrl"
        })
        .state('kitchen', {
          url: "/kitchen",
          templateUrl: "kitchen.html",
          controller: "kitchenCtrl"
        })
        .state('den', {
          url: "/denCtrl",
          templateUrl: "den.html",
          controller: "denCtrl"
        })
        .state('itemDetail', {
          url: "/items/:itemName",
          templateUrl: "item.html",
          controller: "itemDetailCtrl"
        })

    }])
})();

HTML’nizde, durum görünümlerinin içeride doldurulabilmesi için ‘ui-view’ yönergesine ihtiyacınız olacak.

<div ui-view></div>

ADIM 3: Durumlara erişim

$stateProvider içinde tanımlanan bir duruma erişmenin hep birlikte 3 yolu vardır.

1. “ui-sref” yönergesi aracılığıyla

‘ui-sref’ yönergesini kullanarak HTML’nizin içindeki durumlara erişebilirsiniz.

<li ui-sref-active="active">
    <a ui-sref="kitchen">Go to the Kitchen</a>
</li>
<li ui-sref-active="active">
    <a ui-sref="den">Enter the den</a>
</li>
<li ui-sref-active="active">
    <a ui-sref="itemDetail({itemName:'keyboard'})">Key Board</a>
</li>

2. Denetleyicideki ‘$state’ hizmeti aracılığıyla

“.go” yöntemiyle denetleyiciye sağlanan “$state” öğesini kullanarak denetleyicinizin içindeki diğer durumlara da gidebilirsiniz.

.controller(function($scope, $state) {
    // ...
    $scope.navigateTo = function(stateName) {
        $state.go(stateName); // i.e. $state.go('den'); 
    };
})

3. Tarayıcıdaki url aracılığıyla

Bunun gibi tanımlanmış ‘mutfak’ adlı bir durumunuz olduğunu varsayarsak:

$stateProvider
  .state("kitchen", {
    url: "/kitchenUrl",
    ...
  });

Ardından, tarayıcınızda URL olarak appdomain/kitchenUrl’ye erişmek, iç içe geçmiş durumların olmadığını ve “appdomain"in uygulamanızı barındıran sunucu olduğunu varsayarak, “mutfak” durumunuza gidecektir.

Temel Görünüm #

index.html

<html>
    <head>
        <title>Angular-UI Router Example</title>
        <script type="text/javascript" src="https://cdnjs.cloudflare.com/ajax/libs/angular.js/1.4.9/angular.js"></script>
        <script type="text/javascript" src="https://cdnjs.cloudflare.com/ajax/libs/angular-ui-router/0.3.1/angular-ui-router.js"></script>
        <script type="text/javascript" src="../js/script.js"></script>
    </head>
    <body ui-view="mainView"> <!-- Defining a container for our view -->
    </body>
</html>

script.js

var app = angular.module('app', ['ui.router']);
app.config(['$stateProvider', function($stateProvider){
    $stateProvider.state('home', {                        // Creating a state called 'home'
        url: '',                                          // An empty URL means that this state will be loaded on the main URL when no other state is called
        views: {
            'mainView': {                                 // Section for our view-container that we defined in the HTML
                template: '<h1>It works!</h1>'            // Setting a template for this view
                /*templateUrl: '../templates/home.html'*/ //templateUrl would load the file and uses it's content as the template
             }
        }
    });
}])

Çoklu görünüm ile bir durum tanımlama #

ui-router’da bir durum, her biri kendi denetleyicisine ve bir şablona sahip birden fazla görünüm tutabilir.

.state('dashboard', {
     name: 'dashboard',
     url: '/dashboard',
     views: {
         "view1": {
             templateUrl: "path/to/view1.html",
             controller: "view1Controller"
         },
         "view2": {
             templateUrl: "path/to/view2.html",
             controller: "view2Controller"
         }
     }
 })

Ardından, eyaletinizin HTML’sinin içinde bu görünümleri bağlayabilirsiniz.

<div ui-view="view1"></div>
<div ui-view="view2"></div>

Verileri bir duruma çözümleme #

Durumunuza geçiş yaptığınızda verileri kendi durumunuza ‘çözebilirsiniz’, genellikle durumun bu verileri kullanması gerektiğinde veya sağlanan bazı girdilerin kimliğinin doğrulanması gerektiğinde bir duruma çözümlemek için yararlıdır.

Durumlarınızı tanımladığınızda, “.resolve” özelliğine çözümlenecek bir değerler haritası sağlamanız gerekecek, çözümlenen her değerin bir “söz” döndüren bir işlevi olmalıdır.

.state('main', {
     url: "/main",
     templateUrl: "path/to/main.html",
     controller: 'mainCtrl',
     resolve: {
         serverData: function ($http) {
             return $http.get('some/url');
         }
     }
});

Şimdi, “mainCtrl” içinde verilere erişebilirsiniz (yani “$http” çağrısı başarıyla çözüldüyse).

.controller("mainCtrl", function($scope, serverData) {
    $scope.resolvedData = serverData.then(resp=> resp.data);
    ....
})

Geçiş olaylarını kullanma #

UI-Router, geçiş hatalarını işlemek, belirli parametre değerlerine dayalı geçişleri işlemek/engellemek, özel kimlik doğrulama vb. için yardımcı olabilecek geçiş olaylarını ortaya çıkarır.

Bu olaylar, global bir etki için “$rootScope"a veya denetleyici başına bir etki için “$scope"a bağlanabilir.

$stateChangeError - Bu olay, durumu değiştirme girişimi başarısız olduğunda ve atıldığında ve hata oluştuğunda yayınlanır, bu olay aşağıdaki imza ile bir geri arama işlevi başlatır:

geri arama(olay, toState, toParams, fromState, fromParams, hata)

olay: olay nesnesi

toState: hedef durum

toParams: hedef duruma geçirilen parametreler

fromState: mevcut durum

fromParams: geçerli duruma geçirilen parametreler

hata: hata nesnesi

$stateChangeStart - Bu olay, bir durum geçişi başladığında yayınlanır, bu olay aşağıdaki imzaya sahip bir geri arama işlevi başlatır:

geri arama(olay, toState, toParams, fromState, fromParams, options)

seçenekler: durum seçenekleri nesnesi

$stateChangeSuccess - Bu olay, bir durum geçişi tamamlandığında yayınlanır, bu olay aşağıdaki imzaya sahip bir geri arama işlevi başlatır:

geri arama(olay, toState, toParams, fromState, fromParams, options)

$stateNotFound - Bu olay, geçiş yapmak istediğiniz bir durum bulunamadığında yayınlanır, bu olay aşağıdaki imzaya sahip bir geri arama işlevi başlatır:

geri arama(olay, unfoundState, fromParams, fromState)

unfoundState - bulunamayan durumu temsil eden bir nesne

Örnek:

$rootScope.$on('$stateChangeSuccess', function (event, toState, toParams, fromState, fromParams, options) {
    $log.debug("$stateChangeSuccess: event: %o toState: %o, toParams: %o, fromState: %o, fromParams: %o, options: %o", event, toState, toParams, fromState, fromParams, options);
    // runs when the state has successfully changed
});

$rootScope.$on('$stateChangeStart', function (event, toState, toParams, fromState, fromParams, options) {
    $log.debug("$stateChangeStart: event: %o toState: %o, toParams: %o, fromState: %o, fromParams: %o, options: %o", event, toState, toParams, fromState, fromParams, options);
    // runs when the state has just started to transition
});

$rootScope.$on('$stateNotFound', function (event, unfoundState, fromParams, fromState) {
    $log.debug("$stateNotFound: event: %o unfoundState: %o, fromParams: %o, fromState: %o", event, unfoundState, fromParams, fromState);
    // runs when the state wsa not found
});

$rootScope.$on('$stateChangeError', function (event, toState, toParams, fromState, fromParams, error) {
    $log.debug("$stateChangeError: event: %o toState: %o, toParams: %o, fromState: %o, fromParams: %o, error: %o", event, toState, toParams, fromState, fromParams, error);
    // runs when there was an error while attempting to transition
});
