Sözdizimi #

  • unittest { … } - yalnızca “unittesting” modunda çalıştırılan bir blok
  • assert(<bir boole olarak değerlendirilen ifade>, <isteğe bağlı hata mesajı>)

Birim test blokları #

Testler, kararlı, hatasız uygulamalar sağlamak için mükemmel bir yoldur. Etkileşimli bir belge görevi görürler ve işlevselliği bozma korkusu olmadan kodu değiştirmeye izin verirler. D, D dilinin bir parçası olarak “unittest” bloğu için uygun ve yerel bir sözdizimi sağlar. D modülündeki herhangi bir yerde “unittest” blokları, kaynak kodun işlevselliğini test etmek için kullanılabilir.

/**
Yields the sign of a number.
Params:
    n = number which should be used to check the sign
Returns:
    1 for positive n, -1 for negative and 0 for 0.
*/
T sgn(T)(T n)
{
    if (n == 0)
        return 0;
    return (n > 0) ? 1 : -1;
}

// this block will only be executed with -unittest
// it will be removed from the executable otherwise
unittest
{
    // go ahead and make assumptions about your function
    assert(sgn(10)  ==  1);
    assert(sgn(1)   ==  1);
    assert(sgn(-1)  == -1);
    assert(sgn(-10) == -1);
}

Unittest yürütme #

-unittest bayrağı D derleyicisine iletilirse, tüm unittest bloklarını çalıştırır. Derleyicinin saplanmış bir “ana” işlev oluşturmasına izin vermek genellikle yararlıdır. Derleme ve çalıştırma sarmalayıcısı “rdmd"yi kullanarak, D programınızı test etmek şu kadar kolay olur:

rdmd -main -unittest yourcode.d

Tabii ki, isterseniz bu işlemi iki adıma da bölebilirsiniz:

dmd -main -unittest yourcode.d
./yourcode

dub projeleri için tüm dosyaların derlenmesi ve en birim bloklarının çalıştırılması ile rahatlıkla yapılabilir.

dub test

Profesyonel ipucu: bahşişten tasarruf etmek için "tdmd"yi kabuk takma adı olarak tanımlayın. 
alias tdmd="rdmd -main -unittest"

ve ardından dosyalarınızı aşağıdakilerle test edin:

tdmd yourcode.d

Açıklamalı birim test #

Şablonlu kod için, işlev özniteliklerinin (örn.

@safe @nogc pure nothrow unittest
{
    import std.math;
    assert(exp(0)  ==  1);
    assert(log(1)  ==  0);
}

Elbette D’de her bloğa niteliklerle açıklama eklenebileceğini ve derleyicilerin elbette bunların doğru olduğunu doğruladığını unutmayın. Yani örneğin aşağıdaki örneğe benzer olacaktır:

unittest
{
    import std.math;
    @safe {
        assert(exp(0)  ==  1);
        assert(log(1)  ==  0);
    }
}
