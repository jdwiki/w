7 Basit Performans İyileştirmeleri #

1) ng-repeat’i idareli kullanın

Görünümlerde ng-repeat kullanılması, özellikle iç içe ng-repeat‘ler olduğunda genellikle düşük performansa neden olur.

Bu çok yavaş!

<div ng-repeat="user in userCollection">
  <div ng-repeat="details in user">
    {{details}}
  </div>
</div>

Mümkün olduğunca iç içe tekrarlardan kaçınmaya çalışın. ’ng-repeat’ performansını artırmanın bir yolu, ‘$index ile izle’ (veya başka bir kimlik alanı) kullanmaktır. Varsayılan olarak, “ng-repeat” tüm nesneyi izler. “Track by” ile, Angular nesneyi yalnızca “$index” veya nesne kimliği ile izler.

<div ng-repeat="user in userCollection track by $index">
  {{user.data}}
</div>

**pagination, sanal kaydırma gibi diğer yaklaşımları kullanın. com/questions/33259241/how-can-i-make-an-virtual-scroll-with-angularjs), [sonsuz kaydırma](http://stackoverflow.com/questions/21674266/angularjs-infinite-scroll-in- a-container) veya limitTo: start büyük koleksiyonlar üzerinde yinelemeyi önlemek için mümkün olduğunda.

2) Bir kez bağla #

Angular, çift yönlü veri bağlama özelliğine sahiptir. Çok fazla kullanıldığında yavaş olmanın bir bedeli vardır.

Daha Yavaş Performans

<!-- Default data binding has a performance cost -->
<div>{{ my.data }}</div>

Daha Hızlı Performans (AngularJS >= 1.3)

<!-- Bind once is much faster -->
<div>{{ ::my.data }}</div>

<div ng-bind="::my.data"></div>

<!-- Use single binding notation in ng-repeat where only list display is needed  -->
<div ng-repeat="user in ::userCollection">
  {{::user.data}}
</div>

“Bir kez bağla” notasyonunu kullanmak, Angular’a, ilk özet döngü serisinden sonra değerin sabitlenmesini beklemesini söyler. Angular, bu değeri DOM’da kullanacak, ardından tüm izleyicileri kaldıracak, böylece statik bir değer haline gelecek ve artık modele bağlı olmayacak.

{{}} çok daha yavaştır.

Bu “ng-bind” bir yönergedir ve geçirilen değişkene bir izleyici yerleştirir. Bu nedenle, “ng-bind” yalnızca iletilen değer gerçekten değiştiğinde geçerli olacaktır.

Öte yandan, parantezler, gerekli olmasa bile, her `$digest’te kontrol edilecek ve yenilenecektir.

3) Kapsam işlevleri ve filtreler zaman alır #

AngularJS bir özet döngüsüne sahiptir. Tüm işlevleriniz bir görünümdedir ve özet döngüsü her çalıştığında filtreler yürütülür. Model her güncellendiğinde özet döngüsü yürütülür ve uygulamanızı yavaşlatabilir (sayfa yüklenmeden önce filtreye birden çok kez basılabilir).

Bundan kaçının:

<div ng-controller="bigCalulations as calc">
  <p>{{calc.calculateMe()}}</p>
  <p>{{calc.data | heavyFilter}}</p>
</div>

Daha iyi yaklaşım

<div ng-controller="bigCalulations as calc">
  <p>{{calc.preCalculatedValue}}</p>
  <p>{{calc.data | lightFilter}}</p>
</div>

Kontrolör nerede olabilir:

app.controller('bigCalulations', function(valueService) {
    // bad, because this is called in every digest loop
    this.calculateMe = function() {
        var t = 0;
        for(i = 0; i < 1000; i++) {
            t += i;
        }
        return t;
    }
    // good, because this is executed just once and logic is separated in service to    keep the controller light
    this.preCalulatedValue = valueService.valueCalculation(); // returns 499500
});

4 izleyici #

İzleyiciler performansı büyük ölçüde düşürür. Daha fazla izleyiciyle, özet döngüsü daha uzun sürecek ve kullanıcı arayüzü yavaşlayacaktır. İzleyici değişiklik algılarsa, özet döngüsünü başlatır ve görünümü yeniden oluşturur.

Angular’da değişken değişiklikleri manuel olarak izlemenin üç yolu vardır.

$watch() - değer değişikliklerini izler

$watchCollection() - koleksiyondaki değişiklikleri izler (normal $watchdan fazlasını izler)

$watch(..., true) - Bundan mümkün olduğunca kaçının, “derin izleme” gerçekleştirir ve performansı düşürür (“watchCollection"dan daha fazla izler)

Görünümde değişkenleri bağlıyorsanız, yeni saatler oluşturduğunuzu unutmayın - özellikle döngülerde saat oluşturmayı önlemek için {{::variable}} kullanın.

Sonuç olarak, kaç izleyici kullandığınızı izlemeniz gerekir. İzleyicileri bu komut dosyasıyla sayabilirsiniz (@Words Like Jared İzleyici sayısı’na kredi verin)

(function() {
    var root = angular.element(document.getElementsByTagName('body')),
        watchers = [],
        f = function(element) {
        angular.forEach(['$scope', '$isolateScope'], function(scopeProperty) {
            if(element.data() && element.data().hasOwnProperty(scopeProperty)) {
                angular.forEach(element.data()[scopeProperty].$$watchers, function(watcher) {
                watchers.push(watcher);
                });
            }
        });

        angular.forEach(element.children(), function(childElement) {
            f(angular.element(childElement));
        });
    };
 
    f(root);
 
    // Remove duplicate watchers
    var watchersWithoutDuplicates = [];
    angular.forEach(watchers, function(item) {
        if(watchersWithoutDuplicates.indexOf(item) < 0) {
            watchersWithoutDuplicates.push(item);
        }
    });
    console.log(watchersWithoutDuplicates.length);
})();

5) of-if / of-show #

Bu işlevler davranış olarak çok benzer. ng-if öğeleri DOM’den kaldırırken ng-show yalnızca öğeleri gizler ancak tüm işleyicileri tutar. Kodun göstermek istemediğiniz kısımları varsa ng-if kullanın.

Kullanım türüne bağlıdır, ancak çoğu zaman biri diğerinden daha uygundur.

  • Eleman gerekli değilse, ’ng-if’ kullanın

  • Hızlı bir şekilde açmak/kapatmak için ’ng-show/ng-hide’ kullanın

    <div ng-repeat="user in userCollection">
  <p ng-if="user.hasTreeLegs">I am special<!-- some complicated DOM --></p>
  <p ng-show="user.hasSubscribed">I am awesome<!-- switch this setting on and off --></p>
</div>

Şüpheniz varsa - ’ng-if’ kullanın ve test edin!

6) Hata ayıklamayı devre dışı bırak #

Varsayılan olarak, bağlama yönergeleri ve kapsamları, çeşitli hata ayıklama araçlarına yardımcı olmak için kodda fazladan sınıflar ve işaretlemeler bırakır. Bu seçeneği devre dışı bırakmak, özet döngüsü sırasında bu çeşitli öğeleri artık oluşturmayacağınız anlamına gelir.

angular.module('exampleApp', []).config(['$compileProvider', function ($compileProvider) {
    $compileProvider.debugInfoEnabled(false);
}]);

7) Kaynaklarınızı ortaya çıkarmak için bağımlılık enjeksiyonunu kullanın #

Bağımlılık Enjeksiyonu, nesnenin kendisini oluşturması yerine bir nesneye bağımlılıklarının verildiği bir yazılım tasarım modelidir. Bu, sabit kodlanmış bağımlılıkları kaldırmak ve gerektiğinde bunları değiştirmeyi mümkün kılmakla ilgilidir.

Tüm enjekte edilebilir işlevlerin bu tür dize ayrıştırmasıyla ilişkili performans maliyetini merak edebilirsiniz. Angular, ilk seferden sonra $inject özelliğini önbelleğe alarak bununla ilgilenir. Yani bu, bir fonksiyonun her çağrılması gerektiğinde gerçekleşmez.

PROFESYONELLER İÇİN İPUCU: En iyi performansa sahip yaklaşımı arıyorsanız, $inject özellik açıklama yaklaşımını kullanın. Bu yaklaşım, işlev tanımının ayrıştırılmasını tamamen önler, çünkü bu mantık, açıklama işlevinde aşağıdaki denetime sarılır: if (!($inject = fn.$inject)). $inject zaten mevcutsa, ayrıştırma gerekmez!

var app = angular.module('DemoApp', []);

var DemoController = function (s, h) {
    h.get('https://api.github.com/users/angular/repos').success(function (repos) {
        s.repos = repos;
    });
}
// $inject property annotation
DemoController['$inject'] = ['$scope', '$http'];

app.controller('DemoController', DemoController);

PRO İPUCU 2: Bir hizmet örtük açıklamaları kullanmaya çalıştığında bir hata verecek olan katı DI modunu seçmek için “ng-app” ile aynı öğeye bir “ng-strict-di” yönergesi ekleyebilirsiniz. Örnek:

<html ng-app="DemoApp" ng-strict-di>

Veya manuel önyükleme kullanıyorsanız:

angular.bootstrap(document, ['DemoApp'], {
    strictDi: true
});

Bir Kez Bağla #

Angular, harika çift yönlü veri bağlamaya sahip olma ününe sahiptir. Varsayılan olarak, Angular, modelde veya görünüm bileşeninde herhangi bir veri değiştiğinde, model ve görünüm bileşenleri arasında bağlı değerleri sürekli olarak senkronize eder.

Bu, çok fazla kullanıldığında biraz yavaş olma maliyeti ile birlikte gelir. Bunun daha büyük bir performans isabeti olacak:

Kötü performans: {{my.data}}

Tek seferlik bağlamayı kullanmak için değişken adından önce iki nokta üst üste :: ekleyin. Bu durumda, değer yalnızca my.data tanımlandıktan sonra güncellenir. Açıkça veri değişikliklerini izlememeye işaret ediyorsunuz. Angular, herhangi bir değer kontrolü yapmaz, bu da her bir özet döngüsünde daha az ifadenin değerlendirilmesine neden olur.

Tek seferlik ciltleme kullanan iyi performans örnekleri

{{::my.data}}
<span ng-bind="::my.data"></span>
<span ng-if="::my.data"></span>
<span ng-repeat="item in ::my.data">{{item}}</span>
<span ng-class="::{ 'my-class': my.data }"></div>

Not: Ancak bu, ‘my.data’ için çift yönlü veri bağlamasını kaldırır, bu nedenle uygulamanızda bu alan her değiştiğinde, aynısı görünüme otomatik olarak yansıtılmaz. Bu nedenle yalnızca uygulamanızın ömrü boyunca değişmeyecek değerler için kullanın.

of-if vs of-show #

Bu işlevler davranış olarak çok benzer. Aradaki fark, “ng-if” öğesinin DOM’den öğeleri kaldırmasıdır. Kodun gösterilmeyecek büyük bölümleri varsa, o zaman ’ng-if’ gitmenin yoludur. ’ng-show’ yalnızca öğeleri gizleyecek, ancak tüm işleyicileri tutacaktır.

ifs #

ngIf yönergesi, bir ifadeye dayalı olarak DOM ağacının bir bölümünü kaldırır veya yeniden oluşturur. ngIf öğesine atanan ifade yanlış bir değer olarak değerlendirilirse öğe DOM’den kaldırılır, aksi takdirde öğenin bir klonu DOM’a yeniden eklenir.

gösteri #

ngShow yönergesi, ngShow özniteliğine sağlanan ifadeye göre verilen HTML öğesini gösterir veya gizler. Öğe, ng-hide CSS sınıfı kaldırılarak veya öğeye eklenerek gösterilir veya gizlenir.

Örnek #

<div ng-repeat="user in userCollection">
  <p ng-if="user.hasTreeLegs">I am special
    <!-- some complicated DOM -->
  </p>
  <p ng-show="user.hasSubscribed">I am aweosme
    <!-- switch this setting on and off -->
  </p>
</div>

Çözüm #

Kullanım türüne bağlıdır, ancak genellikle biri diğerinden daha uygundur (örneğin, öğeye zamanın %95’inde ihtiyaç duyulmuyorsa, “ng-if” kullanın; DOM öğesinin görünürlüğünü değiştirmeniz gerekiyorsa, şunu kullanın: ng-show).

Şüphe duyduğunuzda ’ng-if’ kullanın ve test edin!

Not: “ng-if” yeni bir yalıtılmış kapsam oluştururken “ng-show” ve “ng-hide” oluşturmaz. Üst kapsam özelliğine doğrudan erişilemiyorsa $parent.property kullanın.

İzleyiciler #

Bir değeri izlemek ve bu değerin değiştiğini tespit etmek için izleyicilere ihtiyaç vardır.

$watch() veya $watchCollection çağrısından sonra yeni izleyici, mevcut kapsamda dahili izleyici koleksiyonuna eklenir.

Peki, izleyici nedir? #

İzleyici, her özet döngüsünde çağrılan ve bir değer döndüren basit bir işlevdir. Angular, döndürülen değeri önceki çağrıdakiyle aynı değilse kontrol eder - ikinci parametrede $watch() veya $watchCollection işlevine geçirilen bir geri arama yürütülür.

(function() {
  angular.module("app", []).controller("ctrl", function($scope) {
    $scope.value = 10;
    $scope.$watch(
      function() { return $scope.value; },
      function() { console.log("value changed"); }
    );
  }
})();

İzleyiciler performans katilleridir. Ne kadar çok izleyiciniz varsa, bir özet döngüsü oluşturmaları o kadar uzun sürer, kullanıcı arayüzü o kadar yavaş olur. Bir izleyici değişiklik algılarsa, özet döngüsünü başlatır (tüm ekranda yeniden hesaplama)

Angular’da değişken değişiklikleri manuel olarak izlemenin üç yolu vardır.

$watch() - sadece değer değişikliklerini izler

$watchCollection() - koleksiyondaki değişiklikleri izler (normal $watch’dan fazlasını izler)

$watch(..., true) - Bundan mümkün olduğunca kaçının, “derin izleme” gerçekleştirir ve performansı öldürür (watchCollection’dan daha fazla izler)

Görünümde değişkenleri bağlıyorsanız, yeni izleyiciler oluşturduğunuzu unutmayın - özellikle döngülerde izleyici oluşturmak için {{::variable}} kullanın

Sonuç olarak, kaç izleyici kullandığınızı izlemeniz gerekir. İzleyicileri bu komut dosyasıyla sayabilirsiniz (@Words Like Jared - http://stackoverflow.com/questions/18499909/)

(function() {
  var root = angular.element(document.getElementsByTagName("body")),
      watchers = [];

  var f = function(element) {

    angular.forEach(["$scope", "$isolateScope"], function(scopeProperty) {
      if(element.data() && element.data().hasOwnProperty(scopeProperty)) {
        angular.forEach(element.data()[scopeProperty].$$watchers, function(watcher) {
          watchers.push(watcher);
        });
      }
    });

    angular.forEach(element.children(), function(childElement) {
      f(angular.element(childElement));
    });

  };

  f(root);

  // Remove duplicate watchers
  var watchersWithoutDuplicates = [];
  angular.forEach(watchers, function(item) {
    if(watchersWithoutDuplicates.indexOf(item) < 0) {
      watchersWithoutDuplicates.push(item);
    }
  });

  console.log(watchersWithoutDuplicates.length);

})();

Kendi komut dosyanızı oluşturmak istemiyorsanız, sayfaya gömülü gerçek zamanlı bir grafik kullanan ng-stats adlı açık kaynaklı bir yardımcı program vardır. Angular’ın yönettiği saatlerin sayısı ve zaman içindeki özet döngülerinin sıklığı ve süresi hakkında size fikir vermek için. Yardımcı program, grafiğin nasıl çalışmasını istediğinizi yapılandırmak için arayabileceğiniz ‘showAngularStats’ adlı genel bir işlev sunar.

showAngularStats({
  "position": "topleft",
  "digestTimeThreshold": 16,
  "autoload": true,
  "logDigest": true,
  "logWatches": true
});

Geçerli kapsam dışındaki diğer kapsamlarda kayıtlı dinleyicilerin kaydını her zaman sil #

Aşağıda gösterildiği gibi, geçerli kapsamınız dışındaki kapsamların kaydını her zaman silmelisiniz:

//always deregister these
$rootScope.$on(...);
$scope.$parent.$on(...);

Angular bununla ilgileneceğinden, mevcut kapsamdaki dinleyicilerin kaydını silmeniz gerekmez:

//no need to deregister this
$scope.$on(...);

Başka bir denetleyiciye giderseniz $rootScope.$on dinleyicileri bellekte kalacaktır. Denetleyici kapsam dışına çıkarsa, bu bir bellek sızıntısı yaratacaktır.

Yapma

angular.module('app').controller('badExampleController', badExample);
badExample.$inject = ['$scope', '$rootScope'];

function badExample($scope, $rootScope) {
    $rootScope.$on('post:created', function postCreated(event, data) {});
}

Yapmak

angular.module('app').controller('goodExampleController', goodExample);
goodExample.$inject = ['$scope', '$rootScope'];

function goodExample($scope, $rootScope) {
    var deregister = $rootScope.$on('post:created', function postCreated(event, data) {});

    $scope.$on('$destroy', function destroyScope() {
        deregister();
    });
}

Kapsam işlevleri ve filtreler #

AngularJS, özet döngüsüne sahiptir ve bir görünümdeki tüm işlevleriniz ve filtreler, özet döngüsü her çalıştırıldığında yürütülür. Model güncellendiğinde özet döngüsü yürütülür ve uygulamanızı yavaşlatabilir (sayfa yüklenmeden önce filtreye birden çok kez basılabilir).

Bundan kaçınmalısınız:

<div ng-controller="bigCalulations as calc">
  <p>{{calc.calculateMe()}}</p>
  <p>{{calc.data | heavyFilter}}</p>
</div>

Daha iyi yaklaşım

<div ng-controller="bigCalulations as calc">
  <p>{{calc.preCalculatedValue}}</p>
  <p>{{calc.data | lightFilter}}</p>
</div>

Kontrolör örneğinin bulunduğu yer:

.controller("bigCalulations", function(valueService) {
  // bad, because this is called in every digest loop
  this.calculateMe = function() {
    var t = 0;
    for(i = 0; i < 1000; i++) {
      t = t + i;
    }
    return t;
  }
  //good, because it is executed just once and logic is separated in service to    keep the controller light
  this.preCalulatedValue = valueService.caluclateSumm(); // returns 499500
});

Modelinizin Sekmesini Kaldırın #

<div ng-controller="ExampleController">
    <form name="userForm">
        Name:
        <input type="text" name="userName"
           ng-model="user.name"
           ng-model-options="{ debounce: 1000 }" />
            <button ng-click="userForm.userName.$rollbackViewValue();   user.name=''">Clear</button><br />
    </form>
    <pre>user.name = </pre>
</div>

Yukarıdaki örnekte, 1 saniye olan 1000 milisaniyelik bir geri dönme değeri ayarlıyoruz. Bu önemli bir gecikmedir, ancak girdinin birçok “$digest” döngüsüyle tekrar tekrar “ng-model"i atmasını önleyecektir.

Debounce’ı giriş alanlarınızda ve anında güncellemenin gerekli olmadığı herhangi bir yerde kullanarak, Angular uygulamalarınızın performansını önemli ölçüde artırabilirsiniz. Sadece zamana göre geciktirmekle kalmaz, aynı zamanda eylem tetiklendiğinde de geciktirebilirsiniz. Ng-modelinizi her tuş vuruşunda güncellemek istemiyorsanız, bulanıklıkta da güncelleme yapabilirsiniz.
