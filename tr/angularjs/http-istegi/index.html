Kontrolör içinde $http kullanma #

$http hizmeti, bir HTTP isteği oluşturan ve bir söz veren bir işlevdir.

Genel kullanım

// Simple GET request example:
$http({
  method: 'GET',
  url: '/someUrl'
}).then(function successCallback(response) {
    // this callback will be called asynchronously
    // when the response is available
  }, function errorCallback(response) {
    // called asynchronously if an error occurs
    // or server returns response with an error status.
  });

Kontrolör içinde kullanım

appName.controller('controllerName',
    ['$http', function($http){

    // Simple GET request example:
    $http({
        method: 'GET',
        url: '/someUrl'
    }).then(function successCallback(response) {
        // this callback will be called asynchronously
        // when the response is available
    }, function errorCallback(response) {
        // called asynchronously if an error occurs
        // or server returns response with an error status.
    });
}])

Kısayol Yöntemleri

$http hizmetinin de kısayol yöntemleri vardır. http yöntemlerini buradan okuyun

Sözdizimi

$http.get('/someUrl', config).then(successCallback, errorCallback);
$http.post('/someUrl', data, config).then(successCallback, errorCallback);

Kısayol Yöntemleri

Bir hizmette $http isteğini kullanma #

HTTP istekleri, her web uygulamasında tekrar tekrar yaygın olarak kullanılır, bu nedenle her ortak istek için bir yöntem yazmak ve ardından bunu uygulama boyunca birden çok yerde kullanmak akıllıca olacaktır.

Bir “httpRequestsService.js” oluşturun

httpRequestsService.js

appName.service('httpRequestsService', function($q, $http){

    return {
        // function that performs a basic get request
        getName: function(){
            // make sure $http is injected
            return $http.get("/someAPI/names")
                .then(function(response) {
                    // return the result as a promise
                    return response;
                }, function(response) {
                    // defer the promise
                    return $q.reject(response.data);
                });
        },

        // add functions for other requests made by your app
        addName: function(){
            // some code...
        }
    }
})

Yukarıdaki hizmet, hizmetin içinde bir alma isteği gerçekleştirecektir. Bu, hizmetin enjekte edildiği herhangi bir kontrolör için geçerli olacaktır.

Örnek kullanım

appName.controller('controllerName',
    ['httpRequestsService', function(httpRequestsService){

        // we injected httpRequestsService service on this controller
        // that made the getName() function available to use.
        httpRequestsService.getName()
            .then(function(response){
                // success
            }, function(error){
                // do something with the error
            })
    }])

Bu yaklaşımı kullanarak artık httpRequestsService.js‘yi her zaman ve herhangi bir denetleyicide kullanabiliriz.

$http isteğinin zamanlaması #

$http istekleri, sunucuya bağlı olarak değişen süre gerektirir, bazıları birkaç milisaniye, bazıları ise birkaç saniye sürebilir. Genellikle bir istekten veri almak için gereken süre kritiktir. Yanıt değerinin bir ad dizisi olduğunu varsayarsak, aşağıdaki örneği göz önünde bulundurun:

Yanlış

$scope.names = [];

$http({
    method: 'GET',
    url: '/someURL'
}).then(function successCallback(response) {
            $scope.names = response.data;
        },
        function errorCallback(response) {
            alert(response.status);
        });

alert("The first name is: " + $scope.names[0]);

$http isteğinin hemen altındaki $scope.names[0] dosyasına erişmek genellikle bir hata verir - bu kod satırı, sunucudan yanıt alınmadan önce yürütülür.

Doğru

$scope.names = [];

$scope.$watch('names', function(newVal, oldVal) {
    if(!(newVal.length == 0)) {
        alert("The first name is: " + $scope.names[0]);
    }
});

$http({
    method: 'GET',
    url: '/someURL'
}).then(function successCallback(response) {
            $scope.names = response.data;
        },
        function errorCallback(response) {
            alert(response.status);
        });

[$watch][1] hizmetini kullanarak $scope.names dizisine yalnızca yanıt alındığında erişiriz. Başlatma sırasında, “$scope.names” daha önce başlatılmış olmasına rağmen işlev çağrılır, bu nedenle “newVal.length"in 0’dan farklı olup olmadığını kontrol etmek gerekir. Dikkat edin - $scope.names üzerinde yapılan herhangi bir değişiklik izleme işlevini tetikleyecektir.

[1]: https://www.wikiod.com/tr/angularjs/ozet-dongusu#$özet ve $izle
