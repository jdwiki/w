ngRoute kullanarak yönlendirme

  • Bireysel rotalar için özel davranış tanımlama

  • Temel örnek
  • Bireysel rotalar için özel davranış tanımlama
  • Rota parametreleri örneği

’ngRoute’, açısal uygulamalar için yönlendirme ve derin bağlantı hizmetleri ve yönergeleri sağlayan yerleşik bir modüldür.

’ngRoute’ ile ilgili tüm belgeler https://docs.angularjs.org/api/ngRoute adresinde mevcuttur.

Temel örnek #

Bu örnek, “controllerAs” sözdizimini kullanarak, her biri kendi görünümüne ve denetleyicisine sahip 3 yolla küçük bir uygulama kurmayı gösterir.

Yönlendiricimizi açısal .config işlevinde yapılandırıyoruz

  1. .config dosyasına $routeProvider enjekte ediyoruz
  2. Bir rota tanımlama nesnesi ile “.When” metodunda rota isimlerimizi tanımlıyoruz.
  3. “.When” yöntemini “şablon” veya “templateUrl”, “controller” ve “controllerAs"ı belirten bir nesne ile sağlıyoruz.

app.js

angular.module('myApp', ['ngRoute'])
  .controller('controllerOne', function() {
    this.message = 'Hello world from Controller One!';
  })
  .controller('controllerTwo', function() {
    this.message = 'Hello world from Controller Two!';
  })
  .controller('controllerThree', function() {
    this.message = 'Hello world from Controller Three!';
  })
  .config(function($routeProvider) {
    $routeProvider
    .when('/one', {
      templateUrl: 'view-one.html',
      controller: 'controllerOne',
      controllerAs: 'ctrlOne'
    })
    .when('/two', {
      templateUrl: 'view-two.html',
      controller: 'controllerTwo',
      controllerAs: 'ctrlTwo'
    })
    .when('/three', {
      templateUrl: 'view-three.html',
      controller: 'controllerThree',
      controllerAs: 'ctrlThree'
    })
    // redirect to here if no other routes match
    .otherwise({
      redirectTo: '/one'
    });
  });

Ardından HTML’mizde ‘href’ ile ‘’ öğelerini kullanarak navigasyonumuzu tanımlarız, ‘helloRoute’ rota adı için ‘Rotam’ olarak yönlendiririz

Ayrıca görüşümüze bir konteyner ve rotalarımızı enjekte etmek için ng-view yönergesini sağlıyoruz.

index.html

<div ng-app="myApp">
  <nav>
    <!-- links to switch routes -->
    <a href="#/one">View One</a>
    <a href="#/two">View Two</a>
    <a href="#/three">View Three</a>
  </nav>
  <!-- views will be injected here -->
  <div ng-view></div>
  <!-- templates can live in normal html files -->
  <script type="text/ng-template" id="view-one.html">
    <h1>{{ctrlOne.message}}</h1>
  </script>

  <script type="text/ng-template" id="view-two.html">
    <h1>{{ctrlTwo.message}}</h1>
  </script>

  <script type="text/ng-template" id="view-three.html">
    <h1>{{ctrlThree.message}}</h1>
  </script>
</div>

Bireysel rotalar için özel davranış tanımlama #

Bireysel rotalar için özel davranışı tanımlamanın en basit yolu oldukça kolay olacaktır.

Bu örnekte, bir kullanıcının kimliğini doğrulamak için kullanıyoruz:

1) “routes.js”: istenen herhangi bir rota için yeni bir özellik (“requireAuth” gibi) oluşturun

angular.module('yourApp').config(['$routeProvider', function($routeProvider) {
    $routeProvider
        .when('/home', {
            templateUrl: 'templates/home.html',
            requireAuth: true
        })
        .when('/login', {
            templateUrl: 'templates/login.html',
        })
        .otherwise({
            redirectTo: '/home'
        });
}])

2) “ng-view” içindeki bir öğeye bağlı olmayan üst düzey bir denetleyicide (açısal “$routeProvider” ile çakışmayı önlemek için), “newUrl” öğesinin “requireAuth” özelliğine sahip olup olmadığını kontrol edin ve uygun davran

angular.module('YourApp').controller('YourController', ['$scope', 'session', '$location',
    function($scope, session, $location) {

        $scope.$on('$routeChangeStart', function(angularEvent, newUrl) {
            
            if (newUrl.requireAuth && !session.user) {
                // User isn’t authenticated
                $location.path("/login");
            }
            
        });
    }
]);

Rota parametreleri örneği #

Bu örnek, kontrol cihazında kullanmak için rotadaki parametreleri geçen temel örneği genişletir.

Bunu yapmak için şunları yapmalıyız:

  1. Rota adındaki parametre konumunu ve adını yapılandırın
  2. Kontrolörümüze $routeParams hizmetini enjekte edin

app.js

angular.module('myApp', ['ngRoute'])
  .controller('controllerOne', function() {
    this.message = 'Hello world from Controller One!';
  })
  .controller('controllerTwo', function() {
    this.message = 'Hello world from Controller Two!';
  })
  .controller('controllerThree', ['$routeParams', function($routeParams) {
    var routeParam = $routeParams.paramName

    if ($routeParams.message) {
        // If a param called 'message' exists, we show it's value as the message
        this.message = $routeParams.message;
    } else {
        // If it doesn't exist, we show a default message
        this.message = 'Hello world from Controller Three!';
    }
  }])
  .config(function($routeProvider) {
    $routeProvider
    .when('/one', {
      templateUrl: 'view-one.html',
      controller: 'controllerOne',
      controllerAs: 'ctrlOne'
    })
    .when('/two', {
      templateUrl: 'view-two.html',
      controller: 'controllerTwo',
      controllerAs: 'ctrlTwo'
    })
    .when('/three', {
      templateUrl: 'view-three.html',
      controller: 'controllerThree',
      controllerAs: 'ctrlThree'
    })
    .when('/three/:message', { // We will pass a param called 'message' with this route
      templateUrl: 'view-three.html',
      controller: 'controllerThree',
      controllerAs: 'ctrlThree'
    })
    // redirect to here if no other routes match
    .otherwise({
      redirectTo: '/one'
    });
  });

Ardından, şablonlarımızda herhangi bir değişiklik yapmadan, sadece özel mesajlı yeni bir bağlantı ekleyerek, yeni özel mesajı görünümümüzde görebiliriz.

index.html

<div ng-app="myApp">
  <nav>
    <!-- links to switch routes -->
    <a href="#/one">View One</a>
    <a href="#/two">View Two</a>
    <a href="#/three">View Three</a>
    <!-- New link with custom message -->
    <a href="#/three/This-is-a-message">View Three with "This-is-a-message" custom message</a>
  </nav>
  <!-- views will be injected here -->
  <div ng-view></div>
  <!-- templates can live in normal html files -->
  <script type="text/ng-template" id="view-one.html">
    <h1>{{ctrlOne.message}}</h1>
  </script>

  <script type="text/ng-template" id="view-two.html">
    <h1>{{ctrlTwo.message}}</h1>
  </script>

  <script type="text/ng-template" id="view-three.html">
    <h1>{{ctrlThree.message}}</h1>
  </script>
</div>
