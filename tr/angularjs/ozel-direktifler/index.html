Parametreler #

parametre Ayrıntılar
kapsam Direktifin kapsamını ayarlamak için özellik. Yanlış, doğru veya izole bir kapsam olarak ayarlanabilir: { @, =, <, & }.
kapsam: sahte Yönerge ana kapsamı kullanır. Yönerge için kapsam oluşturulmadı.
kapsam: doğru Yönerge, ana kapsamı prototip olarak yeni bir alt kapsam olarak devralır. Aynı öğe üzerinde yeni bir kapsam talep eden birden fazla yönerge varsa, bunlar yeni bir kapsamı paylaşacaktır.
kapsam: { @ } Yönerge kapsamı özelliğinin bir DOM öznitelik değerine tek yönlü bağlanması. Nitelik değeri ebeveyne bağlı olduğundan, direktif kapsamında değişecektir.
kapsam: { = } Yönerge özniteliği değişirse üst öğedeki özniteliği değiştiren çift yönlü öznitelik bağlama ve bunun tersi.
kapsam: { < } Bir yönerge kapsamı özelliğinin ve bir DOM öznitelik ifadesinin tek yönlü bağlanması. İfade ebeveynde değerlendirilir. Bu, üst değerin kimliğini izler, böylece üst öğedeki bir nesne özelliğinde yapılan değişiklikler yönergeye yansıtılmaz. Bir yönergedeki bir nesne özelliğinde yapılan değişiklikler, her ikisi de aynı nesneye başvurduğu için üst öğeye yansıtılacaktır
kapsam: { & } Yönergenin, üst öğede değerlendirilecek bir ifadeye veri iletmesine izin verir.
derleme: işlev Bu işlev, bağlantı işlevi çalıştırılmadan önce yönerge şablonunda DOM dönüşümü gerçekleştirmek için kullanılır. ’tElement’ (yönerge şablonu) ve ’tAttr’ (yönergede bildirilen niteliklerin listesi) kabul eder. Kapsama erişimi yoktur. Bir ‘bağlantı sonrası’ işlevi olarak kaydedilecek bir işlev döndürebilir veya ‘ön bağlantı’ ve ‘bağlantı sonrası’ işlevleri olarak kaydedilecek olan ‘pre’ ve ‘post’ özelliklerine sahip bir nesne döndürebilir. .
bağlantı: işlev/nesne link özelliği bir fonksiyon veya nesne olarak konfigüre edilebilir. Şu argümanları alabilir: kapsam(yönerge kapsamı), iElement(yönergenin uygulandığı DOM öğesi), iAttrs(DOM öğesi niteliklerinin toplanması), controller(yönergenin gerektirdiği denetleyici dizisi), transcludeFn. Esas olarak DOM dinleyicilerini ayarlamak, değişiklikler için model özelliklerini izlemek ve DOM’yi güncellemek için kullanılır. Şablon klonlandıktan sonra yürütülür. Derleme işlevi yoksa bağımsız olarak yapılandırılır.
ön bağlantı işlevi Herhangi bir alt bağlantı işlevinden önce yürütülen bağlantı işlevi. Varsayılan olarak, alt yönerge bağlantı işlevleri, ana yönerge bağlantı işlevlerinden önce yürütülür ve ön bağlantı işlevi, önce ana öğenin bağlantı kurmasını sağlar. Bir kullanım durumu, çocuğun ebeveynden veri gerektirmesidir.
bağlantı sonrası işlevi Alt öğelerden sonra yöneticilerin üst öğeye bağlandığı bağlantı işlevi. Genellikle olay işleyicileri eklemek ve alt yönergelere erişmek için kullanılır, ancak alt yönerge zaten bağlantılı olacağından alt yönergenin gerektirdiği veriler burada ayarlanmamalıdır.
kısıtla: dize Yönergenin DOM içinden nasıl çağrılacağını tanımlar. Olası değerler (Yönerge adımızın ‘demoDirective’ olduğunu varsayarsak): ‘E’ - Öğe adı (’’), ‘A’ - Attribute (’
</ div>), C - Eşleşen sınıf (
), M - Yoruma göre ( ). 'restrict' özelliği birden çok seçeneği de destekleyebilir, örneğin - 'restrict: "AC" yönergeyi Attribute VEYA Class ile sınırlayacaktır. Atlanırsa, varsayılan değer ““EA”`dır (Öğe veya Nitelik).
gerektirir: ‘demoDirective’ Geçerli öğe üzerinde demoDirective’in denetleyicisini bulun ve denetleyicisini bağlantı işlevine dördüncü argüman olarak enjekte edin. Bulunamazsa bir hata atın.
gerektirir: ‘?demoDirective’ demoDirective’in denetleyicisini bulmaya çalışın veya bulunamazsa fn bağlantısına null değerini iletin.
gerektirir: ‘^demoDirective’ Öğeyi ve üst öğelerini arayarak demoDirective’in denetleyicisini bulun. Bulunamazsa bir hata atın.
gerektirir: ‘^^demoDirective’ Öğenin üst öğelerini arayarak demoDirective’in denetleyicisini bulun. Bulunamazsa bir hata atın.

Özel yönergeler oluşturma ve kullanma #

Yönergeler, angularjs’nin en güçlü özelliklerinden biridir. Özel angularjs yönergeleri, bir html etiketine belirli bir davranış sağlamak için yeni html öğeleri veya özel nitelikler oluşturarak html’nin işlevselliğini genişletmek için kullanılır.

direktif.js

// Create the App module if you haven't created it yet
var demoApp= angular.module("demoApp", []);

// If you already have the app module created, comment the above line and create a reference of the app module
var demoApp = angular.module("demoApp"); 


// Create a directive using the below syntax
// Directives are used to extend the capabilities of html element 
// You can either create it as an Element/Attribute/class      
// We are creating a directive named demoDirective. Notice it is in CamelCase when we are defining the directive just like ngModel
// This directive will be activated as soon as any this element is encountered in html

demoApp.directive('demoDirective', function () {
    
  // This returns a directive definition object
  // A directive definition object is a simple JavaScript object used for configuring the directive’s behaviour,template..etc
  return {
    // restrict: 'AE', signifies that directive is Element/Attribute directive, 
    // "E" is for element, "A" is for attribute, "C" is for class, and "M" is for comment. 
    // Attributes are going to be the main ones as far as adding behaviors that get used the most.
    // If you don't specify the restrict property it will default to "A"
    restrict :'AE',  

    // The values of scope property decides how the actual scope is created and used inside a directive. These values can be either “false”, “true” or “{}”. This creates an isolate scope for the directive.
    // '@' binding is for passing strings. These strings support {{}} expressions for interpolated values.
    // '=' binding is for two-way model binding. The model in parent scope is linked to the model in the directive's isolated scope.
    // '&' binding is for passing a method into your directive's scope so that it can be called within your directive. 
    // The method is pre-bound to the directive's parent scope, and supports arguments.
    scope: { 
      name: "@",  // Always use small casing here even if it's a mix of 2-3 words
    },                  

    // template replaces the complete element with its text. 
    template: "<div>Hello {{name}}!</div>",
                
    // compile is called during application initialization. AngularJS calls it once when html page is loaded.
    compile: function(element, attributes) {
      element.css("border", "1px solid #cccccc");
                    
      // linkFunction is linked with each element with scope to get the element specific data.
      var linkFunction = function($scope, element, attributes) {
        element.html("Name: <b>"+$scope.name +"</b>");
        element.css("background-color", "#ff00ff");
      };
      return linkFunction;
    }
  };
});

Bu yönerge daha sonra Uygulamada şu şekilde kullanılabilir:

<html>
   
   <head>
      <title>Angular JS Directives</title>
   </head>
   <body>

   <script src="directive.js"></script>   
    <div ng-app = "demoApp">
       <!-- Notice we are using Spinal Casing here -->  
       <demo-directive name="World"></demo-directive>
     
      </div>
   </body>
</html>

demoApp.directive('demoDirective', function () {
  var directiveDefinitionObject = {
    multiElement:  
    priority:
    terminal:
    scope: {},  
    bindToController: {},
    controller:
    controllerAs:
    require:
    restrict:
    templateNamespace:
    template:
    templateUrl:
    transclude:
    compile: 
    link: function(){}                                  
  };
  return directiveDefinitionObject;
});
  1. multiElement - true olarak ayarlanır ve yönerge adının başlangıcı ile bitişi arasındaki tüm DOM düğümleri, yönerge öğeleri olarak toplanır ve gruplandırılır
  2. priority - tek bir DOM öğesi üzerinde birden fazla direktif tanımlandığında, direktifleri uygulama sırasının belirtilmesine izin verir. Daha yüksek numaralı direktifler önce derlenir.
  3. terminal - true olarak ayarlanır ve mevcut öncelik, yürütülecek son direktif seti olur
  4. scope - direktifin kapsamını belirler
  5. denetleyiciye bağlan - kapsam özelliklerini doğrudan direktif kontrolörüne bağlar
  6. controller - kontrolör yapıcı işlevi
  7. require - başka bir yönerge gerektirir ve denetleyicisini bağlantı işlevine dördüncü argüman olarak enjekte eder
  8. controllerAs - kontrolörün direktif şablonundan referans alınmasına izin vermek için direktif kapsamındaki kontrolörün isim referansı.
  9. restrict - Yönergeyi Öğe, Nitelik, Sınıf veya Yorumla sınırlayın
  10. templateNameSpace - yönerge şablonu tarafından kullanılan belge türünü ayarlar: html, svg veya matematik. html varsayılandır
  11. şablon - varsayılan olarak direktifin öğesinin içeriğini değiştiren veya transclude true ise direktif öğesinin içeriğini saran html işaretlemesi
  12. templateUrl - şablon için eşzamansız olarak sağlanan url
  13. transclude - Direktifin göründüğü öğenin içeriğini çıkartın ve direktif için kullanılabilir hale getirin. İçerikler derlenir ve bir dönüştürme işlevi olarak yönergeye sunulur.
  14. derleme - DOM şablonunu dönüştürme işlevi
  15. bağ - yalnızca derleme özelliği tanımlanmamışsa kullanılır. Bağlantı işlevi, DOM dinleyicilerini kaydetmenin yanı sıra DOM’yi güncellemekten de sorumludur. Şablon klonlandıktan sonra yürütülür.

Yönerge kullanılarak yeniden kullanılabilir bileşen nasıl oluşturulur #

AngularJS yönergeleri, bir AngularJS uygulamasında HTML’nin oluşturulmasını kontrol eden şeydir. Bir Html öğesi, öznitelik, sınıf veya yorum olabilirler. Yönergeler, DOM’yi işlemek, HTML öğelerine yeni davranış eklemek, veri bağlama ve daha pek çok şey için kullanılır. Angular’ın sağladığı bazı yönerge örnekleri ng-model, ng-hide, ng-if’dir.

Benzer şekilde, kişi kendi özel yönergesini oluşturabilir ve bunları yeniden kullanılabilir hale getirebilir. Özel yönergeler oluşturmak için Referans. Yeniden kullanılabilir yönergeler oluşturmanın ardındaki anlam, angularjs’nin angular.js kullanmamızı sağladığı gibi, sizin tarafınızdan yazılan bir dizi yönerge/bileşen yapmaktır. Bu yeniden kullanılabilir yönergeler, tutarlı bir davranış, görünüm ve his gerektiren uygulamalar/uygulamalar paketiniz olduğunda özellikle çok yardımcı olabilir. Bu tür yeniden kullanılabilir bileşene bir örnek, uygulamanızda veya farklı uygulamalarda kullanmak isteyebileceğiniz, ancak bunların aynı şekilde davranmasını veya aynı görünmesini istediğiniz basit bir araç çubuğu olabilir.

Öncelikle uygulamanızın Klasöründe resuableComponents adında bir klasör oluşturun ve reusableModuleApp.js yapın

yeniden kullanılabilirModuleApp.js:

(function(){

 var reusableModuleApp = angular.module('resuableModuleApp', ['ngSanitize']);

 //Remember whatever dependencies you have in here should be injected in the app module where it is intended to be used or it's scripts should be included in your main app  
                               //We will be injecting ng-sanitize

 resubaleModuleApp.directive('toolbar', toolbar)

 toolbar.$inject=['$sce'];

 function toolbar($sce){

     return{ 
     restrict :'AE',
        //Defining below isolate scope actually provides window for the directive to take data from app that will be using this.
        scope : {
                value1: '=',
                value2: '=',
                },

         }
     template : '<ul>  <li><a ng-click="Add()" href="">{{value1}}</a></li>  <li><a ng-click="Edit()" href="#">{{value2}}</a></li> </ul> ',
         link : function(scope, element, attrs){
           
              //Handle's Add function
              scope.Add = function(){


              };

              //Handle's Edit function
              scope.Edit = function(){

              };
     }
  }

});

mainApp.js:

(function(){
   var mainApp = angular.module('mainApp', ['reusableModuleApp']); //Inject resuableModuleApp in your application where you want to use toolbar component
   
   mainApp.controller('mainAppController', function($scope){
      $scope.value1 = "Add";
      $scope.value2 = "Edit"; 
   
   });
 
 });

index.html:

 <!doctype html>
 <html ng-app="mainApp">
 <head>
  <title> Demo Making a reusable component
 <head>
  <body ng-controller="mainAppController">
   
     <!-- We are providing data to toolbar directive using mainApp'controller -->
     <toolbar value1="value1" value2="value2"></toolbar>
  
    <!-- We need to add the dependent js files on both apps here -->
     <script src="js/angular.js"></script>
     <script src="js/angular-sanitize.js"></script>
    
     <!-- your mainApp.js should be added afterwards --->
      <script src="mainApp.js"></script>

     <!-- Add your reusable component js files here -->
      <script src="resuableComponents/reusableModuleApp.js"></script>

  </body>
 </html>

Yönerge, varsayılan olarak yeniden kullanılabilir bileşenlerdir. Ayrı açısal modülde yönergeler yaptığınızda, aslında onu farklı angularJs uygulamalarında dışa aktarılabilir ve yeniden kullanılabilir hale getirir. Yeni yönergeler reusableModuleApp.js içine kolayca eklenebilir ve reusableModuleApp davranışı tanımlamak için yönergenin içinde kendi denetleyicisine, hizmetlerine, DDO nesnesine sahip olabilir.

Temel Yönerge örneği #

superman-directive.js

angular.module('myApp', [])
  .directive('superman', function() {
    return {
      // restricts how the directive can be used
      restrict: 'E',
      templateUrl: 'superman-template.html',
      controller: function() {
        this.message = "I'm superman!"
      },
      controllerAs: 'supermanCtrl',
      // Executed after Angular's initialization. Use commonly 
      // for adding event handlers and DOM manipulation
      link: function(scope, element, attributes) {
        element.on('click', function() {
          alert('I am superman!')
        });
      }
    }
  });

süpermen-template.html

<h2>{{supermanCtrl.message}}</h2>

index.html

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Document</title>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.5.0/angular.js"></script>
  <script src="superman-directive.js"><script/>
</head>
<body>
<div ng-app="myApp">
  <superman></superman>
</div>
</body>
</html>

Yönergenin “kısıtlama” ve “bağlantı” işlevleri hakkında daha fazla bilgiyi AngularJS’nin Direktiflerle ilgili resmi dokümantasyonunda kontrol edebilirsiniz.

Şablonlu ve yalıtılmış bir kapsam içeren temel yönerge #

İsolated kapsam ile özel bir yönerge oluşturmak, yönergemizin yanlışlıkla üst kapsamdaki verileri değiştirmesini ve özel verileri okumasını kısıtlamasını önlemek için yönergenin kapsamını içerideki yönergenin kapsamını dış kapsamından ayırır. ebeveyn kapsamı.

Yalıtılmış bir kapsam oluşturmak ve yine de özel yönergemizin dış kapsamla iletişim kurmasına izin vermek için, yönergenin iç kapsamının dış kapsamla nasıl eşleştirileceğini açıklayan “kapsam” seçeneğini kullanabiliriz.

Gerçek bağlamalar, direktife eklenen ekstra nitelikler ile yapılır. Bağlama ayarları, “kapsam” seçeneği ve anahtar/değer çiftlerine sahip bir nesne ile tanımlanır:

  • Yönergemizin yalıtılmış kapsam özelliğine karşılık gelen bir anahtar
  • Angular’a yönergenin iç kapsamını eşleşen bir özniteliğe nasıl bağlayacağını söyleyen bir değer

Yalıtılmış kapsamı olan bir direktifin basit örneği:

var ProgressBar = function() {
  return {
    scope: { // This is how we define an isolated scope
      current: '=', // Create a REQUIRED bidirectional binding by using the 'current' attribute
      full: '=?maxValue' // Create an OPTIONAL (Note the '?'): bidirectional binding using 'max-value' attribute to the `full` property in our directive isolated scope 
    }
    template: '<div class="progress-back">' +
              '  <div class="progress-bar"' +
              '       ng-style="{width: getProgress()}">' +
              '  </div>' + 
              '</div>',
    link: function(scope, el, attrs) {
      if (scope.full === undefined) {
        scope.full = 100;
      }
      scope.getProgress = function() {
        return (scope.current / scope.size * 100) + '%';
      }
    }
  }
}

ProgressBar.$inject = [];
angular.module('app').directive('progressBar', ProgressBar);

Bu yönergenin nasıl kullanılacağına ve denetleyicinin kapsamından direktifin iç kapsamına verilerin nasıl bağlanacağına ilişkin örnek:

Kontrolör:

angular.module('app').controller('myCtrl', function($scope) {
    $scope.currentProgressValue = 39;
    $scope.maxProgressBarValue = 50;
});

Görüş:

<div ng-controller="myCtrl">
    <progress-bar current="currentProgressValue"></progress-bar>
    <progress-bar current="currentProgressValue" max-value="maxProgressBarValue"></progress-bar>
</div>

Yeniden kullanılabilir bir bileşen oluşturma #

Yönergeler, yeniden kullanılabilir bileşenler oluşturmak için kullanılabilir. İşte bir “kullanıcı kutusu” bileşeni örneği:

userBox.js

angular.module('simpleDirective', []).directive('userBox', function() {
  return {
    scope: {
        username: '=username',
        reputation: '=reputation'
    },
    templateUrl: '/path/to/app/directives/user-box.html'
  };
});

Controller.js

var myApp = angular.module('myApp', ['simpleDirective']);

myApp.controller('Controller', function($scope) {

    $scope.user = "John Doe";
    $scope.rep = 1250;

    $scope.user2 = "Andrew";
    $scope.rep2 = 2850;

});

sayfam.js

<html lang="en" ng-app="myApp">
  <head>
    <script src="/path/to/app/angular.min.js"></script>
    <script src="/path/to/app/js/controllers/Controller.js"></script>
    <script src="/path/to/app/js/directives/userBox.js"></script>
  </head>

  <body>
  
    <div ng-controller="Controller">
        <user-box username="user" reputation="rep"></user-box>
        <user-box username="user2" reputation="rep2"></user-box>
    </div>
    
  </body>
</html>

kullanıcı kutusu.html

<div>{{username}}</div>
<div>{{reputation}} reputation</div>

Sonuç:

John Doe
1250 reputation
Andrew
2850 reputation

Yönerge dekoratörü #

Bazen bir direktiften ek özelliklere ihtiyacınız olabilir. Yönergeyi yeniden yazmak (kopyalamak) yerine, yönergenin nasıl davranacağını değiştirebilirsiniz.

Dekoratör $inject aşamasında çalıştırılacaktır.

Bunu yapmak için modülünüze bir .config sağlayın. Yönerge myDirective olarak adlandırılır, bu nedenle myDirectiveDirective’i yapılandırmanız gerekir. (bu, açısal bir sözleşmede [sağlayıcılar hakkında bilgi edinin]).

Bu örnek, yönergenin templateUrl’sini değiştirecektir:

angular.module('myApp').config(function($provide){
        $provide.decorator('myDirectiveDirective', function($delegate){
             var directive = $delegate[0]; // this is the actual delegated, your directive
             directive.templateUrl = 'newTemplate.html'; // you change the directive template
        return $delegate;
    })
});

Bu örnek, tıklandığında yönerge öğesine bir onClick olayı ekler, bu derleme aşamasında gerçekleşir.

angular.module('myApp').config(function ($provide) {
        $provide.decorator('myDirectiveTwoDirective', function ($delegate) {
            var directive = $delegate[0];
            var link = directive.link; // this is directive link phase
            directive.compile = function () { // change the compile of that directive
                return function (scope, element, attrs) {
                    link.apply(this, arguments); // apply this at the link phase
                    element.on('click', function(){ // when add an onclick that log hello when the directive is clicked.
                            console.log('hello!');
                    }); 
                };
            };
            return $delegate;
        });

    });

Benzer yaklaşım hem Sağlayıcılar hem de Hizmetler için kullanılabilir.

Yönerge devralma ve birlikte çalışabilirlik #

Angular js yönergeleri iç içe yerleştirilebilir veya birlikte çalışabilir hale getirilebilir.

Bu örnekte, Adir yönergesi, Bdir Adir gerektirdiğinden, denetleyicisinin $ kapsamını Bdir yönergesine gösterir.

angular.module('myApp',[]).directive('Adir', function () {
        return {
            restrict: 'AE',
            controller: ['$scope', function ($scope) {
                    $scope.logFn = function (val) {
                            console.log(val);
                        }
                  }]
            }
    })

require: ‘^Adir’ ayarını yaptığınızdan emin olun (açısal belgelere bakın, bazı sürümler ^ karakteri gerektirmez).

.directive('Bdir', function () {
        return {
            restrict: 'AE',
            require: '^Adir', // Bdir require Adir
            link: function (scope, elem, attr, Parent) { 
            // Parent is Adir but can be an array of required directives.
                elem.on('click', function ($event) {
                    Parent.logFn("Hello!"); // will log "Hello! at parent dir scope
                    scope.$apply(); // apply to parent scope.
                });
            }
        }
    }]);

Yönergenizi şu şekilde yuvalayabilirsiniz:

<div a-dir><span b-dir></span></div>
<a-dir><b-dir></b-dir> </a-dir>

Yönergelerin HTML’nizde iç içe olması gerekli değildir.
