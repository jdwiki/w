Sözdizimi #

  • $scope : ng.IScope - TypeScript’te türü tanımlamanın yolu budur belirli bir değişken için.

Paketleme / Küçültmeyi Kullanma #

$scope’un denetleyicinin yapıcı işlevlerine enjekte edilme şekli, açısal bağımlılık enjeksiyonu temel seçeneğini göstermenin ve kullanmanın bir yoludur, ancak küçültülemeyeceği için üretime hazır değildir. Bunun nedeni, küçültme sisteminin değişken adlarını değiştirmesi ve anguar’ın bağımlılık enjeksiyonunun neyin enjekte edilmesi gerektiğini bilmek için parametre adlarını kullanmasıdır. Bu nedenle, bir örnek için ExampleController’ın yapıcı işlevi aşağıdaki koda küçültülür.

function n(n){this.setUpWatches(n)

ve “$scope”, “n” olarak değiştirildi!
Bunun üstesinden gelmek için bir $inject dizisi(string[]) ekleyebiliriz. Böylece açısalın DI’si, denetleyici yapıcı işlevi hangi konumda enjekte edileceğini bilir.
Yani yukarıdaki yazı tipi şu şekilde değişir:

module App.Controllers {
    class Address {
        line1: string;
        line2: string;
        city: string;
        state: string;
    }
    export class SampleController {
        firstName: string;
        lastName: string;
        age: number;
        address: Address;
        setUpWatches($scope: ng.IScope): void {
            $scope.$watch(() => this.firstName, (n, o) => {
                //n is string and so is o
            });
        };
        static $inject : string[] = ['$scope'];
        constructor($scope: ng.IScope) {
            this.setUpWatches($scope);
        }
    }
}

TypeScript’te Açısal Denetleyiciler #

AngularJS’de tanımlandığı gibi Belgeler

DOM’a ng-denetleyici aracılığıyla bir Denetleyici eklendiğinde yönergesi, Angular, aşağıdakileri kullanarak yeni bir Controller nesnesi başlatır: Belirtilen Kontrolörün yapıcı işlevi. Yeni bir çocuk kapsamı enjekte edilebilir bir parametre olarak oluşturuldu ve kullanıma sunuldu Kontrolörün yapıcı işlevi $ kapsamı olarak.

TypeScript sınıfları kullanılarak kontrolörler çok kolay bir şekilde yapılabilir.

module App.Controllers {
    class Address {
        line1: string;
        line2: string;
        city: string;
        state: string;
    }
    export class SampleController {
        firstName: string;
        lastName: string;
        age: number;
        address: Address;
        setUpWatches($scope: ng.IScope): void {
            $scope.$watch(() => this.firstName, (n, o) => {
                //n is string and so is o
            });
        };
        constructor($scope: ng.IScope) {
            this.setUpWatches($scope);
        }
    }
}

Ortaya Çıkan Javascript

var App;
(function (App) {
    var Controllers;
    (function (Controllers) {
        var Address = (function () {
            function Address() {
            }
            return Address;
        }());
        var SampleController = (function () {
            function SampleController($scope) {
                this.setUpWatches($scope);
            }
            SampleController.prototype.setUpWatches = function ($scope) {
                var _this = this;
                $scope.$watch(function () { return _this.firstName; }, function (n, o) {
                    //n is string and so is o
                });
            };
            ;
            return SampleController;
        }());
        Controllers.SampleController = SampleController;
    })(Controllers = App.Controllers || (App.Controllers = {}));
})(App || (App = {}));
//# sourceMappingURL=ExampleController.js.map

Denetleyici sınıfını yaptıktan sonra, denetleyiciyle ilgili açısal js modülünün sınıfı kullanarak basit bir şekilde yapılabilmesine izin verin.

app
 .module('app')
 .controller('exampleController', App.Controller.SampleController)

Denetleyiciyi ControllerAs Sözdizimi ile Kullanma #

Yaptığımız Controller, controller as Syntax kullanılarak somutlaştırılabilir ve kullanılabilir. Bunun nedeni, değişkeni $scope’a değil, doğrudan controller sınıfına koymamızdır.

“controller as someName” kullanmak, denetleyiciyi “$scope"un kendisinden ayırmaktır. Bu nedenle, denetleyicide bağımlılık olarak $scope enjekte etmeye gerek yoktur.

Geleneksel yol :

// we are using $scope object.
app.controller('MyCtrl', function ($scope) {
  $scope.name = 'John';
});

<div ng-controller="MyCtrl">
  {{name}}
</div>

Şimdi, denetleyici olarak ile Sözdizimi :

// we are using the "this" Object instead of "$scope"
app.controller('MyCtrl', function() {
  this.name = 'John';
});

<div ng-controller="MyCtrl as info">
  {{info.name}}
</div>

JavaScript’te bir “sınıf” başlatırsanız, şunu yapabilirsiniz:

var jsClass = function () {
  this.name = 'John';
}
var jsObj = new jsClass();

Böylece, artık jsClass herhangi bir yöntem veya özelliğine erişmek için jsObj örneğini kullanabiliriz.

Açısal olarak, aynı tür şeyleri yaparız. Örnekleme için sözdizimi olarak denetleyici kullanırız.

Neden ControllerAs Sözdizimi ? #

Denetleyici İşlevi #

Denetleyici işlevi, yalnızca bir JavaScript oluşturucu işlevinden başka bir şey değildir. Bu nedenle, bir görünüm yüklediğinde ‘işlev bağlamı’(** ‘bu’**) denetleyici nesnesine ayarlanır.

Dava 1 :

this.constFunction = function() { ... }

‘$scope’ üzerinde değil, ‘denetleyici nesnesinde’ oluşturulur. görünümler, denetleyici nesnesinde tanımlanan işlevlere erişemez.

Örnek :

<a href="#123" ng-click="constFunction()"></a> // It will not work

Durum 2:

$scope.scopeFunction = function() { ... }

“Denetleyici nesnesi” üzerinde değil, “$ kapsam nesnesi” içinde oluşturulur. görünümler yalnızca $scope nesnesinde tanımlanan işlevlere erişebilir.

Örnek :

<a href="#123" ng-click="scopeFunction()"></a> // It will work

Neden ControllerAs ? #

  • ControllerAs sözdizimi, nesnelerin nerede manipüle edildiğini çok daha net hale getirir.birCtrl.name ve anotherCtrl.name olması, birden çok farklı kontrolör tarafından atanan bir adınız' olduğunu belirlemeyi çok daha kolaylaştırır. farklı amaçlar ancak her ikisi de aynı "$scope.name"yi kullanıyorsa ve bir sayfada her ikisi de **{{name}}`** ile bağlantılı iki farklı HTML öğesine sahipse, hangisinin hangi denetleyiciden olduğunu belirlemek zordur .

  • “$ kapsamını” gizlemek ve üyeleri bir “ara nesne” aracılığıyla denetleyiciden görünüme göstermek. this.* ayarını yaparak, kontrol cihazından görünüme göstermek istediğimiz şeyi tam olarak gösterebiliriz.

      <div ng-controller="FirstCtrl">
      {{ name }}
      <div ng-controller="SecondCtrl">
          {{ name }}
          <div ng-controller="ThirdCtrl">
              {{ name }}
          </div>
      </div>
  </div>

Burada, yukarıdaki durumda {{ isim }} kullanmak çok kafa karıştırıcı olacaktır ve ayrıca hangisinin hangi kontrolörle ilişkili olduğunu da bilmiyoruz.

<div ng-controller="FirstCtrl as first">
    {{ first.name }}
    <div ng-controller="SecondCtrl as second">
        {{ second.name }}
        <div ng-controller="ThirdCtrl as third">
            {{ third.name }}
        </div>
    </div>
</div>

Neden $ kapsam ? #

  • $watch, $digest, $emit, $http gibi bir veya daha fazla $scope yöntemine erişmeniz gerektiğinde $scope kullanın.
  • hangi özelliklerin ve/veya yöntemlerin “$kapsam"a maruz kalacağını sınırlayın, ardından bunları gerektiği gibi açıkça “$kapsam"a geçirin.
