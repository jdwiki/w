Perceptron modelini C++‘da uygulama #

Bu örnekte, nasıl çalıştığına dair daha iyi bir fikir edinebilmeniz için algılayıcı modelinin C++‘da uygulanmasını inceleyeceğim.

Öncelikle yapmak istediğimiz şeyin basit bir algoritmasını yazmak iyi bir uygulamadır.

algoritma:

  1. Ağırlıklar için bir vektör yapın ve 0’a sıfırlayın (Yarılık terimini eklemeyi unutmayın)
  2. 0 hata veya düşük hata sayısı elde edene kadar ağırlıkları ayarlamaya devam edin.
  3. Görünmeyen veriler üzerinde tahminler yapın.

Süper basit bir algoritma yazdıktan sonra, şimdi ihtiyaç duyacağımız bazı fonksiyonları yazalım.

  • Ağın girdisini hesaplamak için bir fonksiyona ihtiyacımız olacak (e.i x * wT** girdileri ağırlıklarla çarparak)
  • 1 veya -1 tahminini elde etmemiz için bir adım işlevi
  • Ve ağırlıklar için ideal değerleri bulan bir fonksiyon.

O halde daha fazla uzatmadan hemen konuya girelim.

fit işlevinin nasıl < float > vektörünün bir vektörünü argüman olarak aldığına dikkat edin. Bunun nedeni, eğitim veri setimizin bir girdi matrisi olmasıdır. Esasen, matrisin bir çift x vektörünün birbirinin üzerine yığıldığını ve bu Matrix’in her sütununun bir özellik olduğunu hayal edebiliriz.

Son olarak sınıfımızın sahip olması gereken değerleri ekleyelim. Ağırlıkları tutmak için w vektörü, eğitim veri seti üzerinde yapacağımız geçiş sayısını gösteren epochs sayısı gibi. Ve bu değeri çevirerek antrenman prosedürünü daha hızlı hale getirmek için her ağırlık güncellemesini çarpacağımız öğrenme oranı olan eta sabiti veya eta çok yüksekse yapabiliriz. ideal sonucu elde etmek için aşağı çevirin (algılayıcının çoğu uygulaması için eta 0,1 değerini öneririm).

class perceptron
{
public:
    perceptron(float eta,int epochs);
    float netInput(vector<float> X);
    int predict(vector<float> X);
    void fit(vector< vector<float> > X, vector<float> y);
private:
    float m_eta;
    int m_epochs;
    vector < float > m_w;
};

Şimdi sınıf setimiz ile. Fonksiyonların her birini yazmanın zamanı geldi.

Yapıcıdan başlayacağız ( perceptron(float eta,int epochs);)

perceptron::perceptron(float eta, int epochs)
{
    m_epochs = epochs; // We set the private variable m_epochs to the user selected value
    m_eta = eta; // We do the same thing for eta
}

Gördüğünüz gibi yapacağımız şey çok basit şeyler. Öyleyse başka bir basit fonksiyona geçelim. Tahmin işlevi( int tahmin(vector X); ). all predict işlevinin yaptığının, net girdiyi alıp netInput 0’dan büyükse ve -1’den büyükse 1 değerini döndürmek olduğunu unutmayın.

int perceptron::predict(vector<float> X)
{
    return netInput(X) > 0 ? 1 : -1; //Step Function
}

Hayatlarımızı kolaylaştırmak için satır içi if ifadesi kullandığımıza dikkat edin. Satır içi if ifadesi şu şekilde çalışır:

şart ? if_true : başka

Çok uzak çok iyi. netInput işlevini uygulamaya geçelim( float netInput(vector X); )

netInput şunları yapar; girdi vektörünü ağırlık vektörünün devriği ile çarpar

x * wT

Başka bir deyişle, x girdi vektörünün her bir öğesini, w ağırlık vektörünün karşılık gelen öğesiyle çarpar ve sonra bunların toplamını alır ve sapmayı ekler.

(x1 * w1 + x2 * w2 + … + xn * wn) + sapma

önyargı = 1 * w0

float perceptron::netInput(vector<float> X)
{
    // Sum(Vector of weights * Input vector) + bias
    float probabilities = m_w[0]; // In this example I am adding the perceptron first
    for (int i = 0; i < X.size(); i++)
    {
        probabilities += X[i] * m_w[i + 1]; // Notice that for the weights I am counting
        // from the 2nd element since w0 is the bias and I already added it first.
    }
    return probabilities;
}

Pekala, şimdi hemen hemen bitirdik, yapmamız gereken son şey, ağırlıkları değiştiren fit fonksiyonunu yazmak.

void perceptron::fit(vector< vector<float> > X, vector<float> y)
{
    for (int i = 0; i < X[0].size() + 1; i++) // X[0].size() + 1 -> I am using +1 to add the bias term
    {
        m_w.push_back(0); // Setting each weight to 0 and making the size of the vector
        // The same as the number of features (X[0].size()) + 1 for the bias term
    }
    for (int i = 0; i < m_epochs; i++) // Iterating through each epoch
    {
        for (int j = 0; j < X.size(); j++) // Iterating though each vector in our training Matrix
        {
            float update = m_eta * (y[j] - predict(X[j])); //we calculate the change for the weights
            for (int w = 1; w < m_w.size(); w++){ m_w[w] += update * X[j][w - 1]; } // we update each weight by the update * the training sample
            m_w[0] = update; // We update the Bias term and setting it equal to the update
        }
    }
}

Yani esasen buydu. Sadece 3 fonksiyonla, artık tahmin yapmak için kullanabileceğimiz çalışan bir algılayıcı sınıfımız var!

Aşağıdaki işlevlere sahip sınıf .cpp dosyası:

perceptron::perceptron(float eta, int epochs)
{
    m_epochs = epochs;
    m_eta = eta;
}

void perceptron::fit(vector< vector<float> > X, vector<float> y)
{
    for (int i = 0; i < X[0].size() + 1; i++) // X[0].size() + 1 -> I am using +1 to add the bias term
    {
        m_w.push_back(0);
    }
    for (int i = 0; i < m_epochs; i++)
    {
        int errors = 0;
        for (int j = 0; j < X.size(); j++)
        {
            float update = m_eta * (y[j] - predict(X[j]));
            for (int w = 1; w < m_w.size(); w++){ m_w[w] += update * X[j][w - 1]; }
            m_w[0] = update;
            errors += update != 0 ? 1 : 0;
        }
        m_errors.push_back(errors);
    }
}

float perceptron::netInput(vector<float> X)
{
    // Sum(Vector of weights * Input vector) + bias
    float probabilities = m_w[0];
    for (int i = 0; i < X.size(); i++)
    {
        probabilities += X[i] * m_w[i + 1];
    }
    return probabilities;
}

int perceptron::predict(vector<float> X)
{
    return netInput(X) > 0 ? 1 : -1; //Step Function
}

void perceptron::printErrors()
{
    printVector(m_errors);
}

void perceptron::exportWeights(string filename)
{
    ofstream outFile;
    outFile.open(filename);

    for (int i = 0; i < m_w.size(); i++)
    {
        outFile << m_w[i] << endl;
    }

    outFile.close();
}

void perceptron::importWeights(string filename)
{
    ifstream inFile;
    inFile.open(filename);

    for (int i = 0; i < m_w.size(); i++)
    {
        inFile >> m_w[i];
    }
}

void perceptron::printWeights()
{
    cout << "weights: ";
    for (int i = 0; i < m_w.size(); i++)
    {
        cout << m_w[i] << " ";
    }
    cout << endl;
}

Bir algılayıcı tam olarak nedir? #

Özünde bir algılayıcı modeli, ikili sınıflandırma için en basit denetimli öğrenme algoritmalarından biridir. Bir tür doğrusal sınıflandırıcı, yani tahminlerini bir dizi ağırlık ile özellik vektörünü birleştiren doğrusal bir tahmin işlevine dayalı olarak yapan bir sınıflandırma algoritmasıdır. Düşünmenin daha sezgisel bir yolu, Yalnızca bir nöronlu bir Sinir Ağı gibidir.

Çalışma şekli çok basittir. Her öğenin veri setimizin bir özelliği olduğu bir x girdi değerleri vektörü alır.

Bir örnek: #

Bir cismin bisiklet mi yoksa araba mı olduğunu sınıflandırmak istediğimizi varsayalım. Bu örnek için 2 özellik seçmek istediğimizi varsayalım. Nesnenin yüksekliği ve genişliği. Bu durumda x = [x1, x2] burada x1 yükseklik ve x2 genişliktir.

Sonra x girdi vektörümüze sahip olduğumuzda, o vektördeki her elemanı bir ağırlıkla çarpmak istiyoruz. Genellikle ağırlık değeri ne kadar yüksek olursa, özellik o kadar önemli olur. Örneğin, x3 özelliği olarak renk kullandıysak ve kırmızı bir bisiklet ve kırmızı bir araba varsa, algılayıcı ona çok düşük bir ağırlık ayarlayacaktır, böylece renk durumu etkilemeyecektir. son tahmin.

Pekala, 2 vektörü x ve w ile çarptık ve bir vektör elde ettik. Şimdi bu vektörün elemanlarını toplamamız gerekiyor. Bunu yapmanın akıllıca bir yolu, x‘ı w ile çarpmak yerine, x‘ı wT ile çarpabiliriz, burada **T ** devrik anlamına gelir. Bir vektörün devrini vektörün döndürülmüş bir versiyonu olarak hayal edebiliriz. Daha fazla bilgi için Wikipedia sayfasını okuyabilirsiniz. Esasen w vektörünün devriğini alarak 1xN yerine Nx1 vektörünü elde ederiz. Böylece şimdi 1xN boyutundaki girdi vektörümüzü bu Nx1 ağırlık vektörü ile çarparsak, bir 1x1 vektörü (veya sadece tek bir değer) elde ederiz. x1 * w1 + x2 * w2 + … + xn * wn‘a eşittir. Bunu yaptıktan sonra, şimdi tahminimize sahibiz. Ama son bir şey var. Bu tahmin muhtemelen yeni bir örneği sınıflandırabilmek için basit bir 1 veya -1 olmayacaktır. Yani yapabileceğimiz şey basitçe şunu söylemektir: Tahminimiz 0’dan büyükse, örneğin sınıf 1’e ait olduğunu söyleriz, aksi takdirde tahmin sıfırdan küçükse örneğin sınıf -1’e ait olduğunu söyleriz. Buna adım işlevi denir.

Fakat doğru tahminler yapabilmek için doğru ağırlıkları nasıl elde ederiz? Başka bir deyişle, algılayıcı modelimizi nasıl eğitiyoruz?

Pekala, algılayıcı durumunda, modelimizi eğitmek için süslü matematik denklemlerine ihtiyacımız yok. Ağırlıklarımız aşağıdaki denklemle ayarlanabilir:

Δw = eta * (y - tahmin) * x(i)

burada x(i) özelliğimizdir (örneğin ağırlık 1 için x1, w2 için x2 vb.).

Ayrıca öğrenme oranı olan eta adlı bir değişken olduğunu fark ettim. Öğrenme oranını, ağırlık değişiminin ne kadar büyük olmasını istediğimizi hayal edebilirsiniz. İyi bir öğrenme oranı, hızlı bir öğrenme algoritması ile sonuçlanır. Çok yüksek bir eta değeri, her çağda artan miktarda hataya neden olabilir ve modelin gerçekten kötü tahminler yapmasına ve asla yakınsamamasına neden olabilir. Çok düşük bir öğrenme oranı, sonuç olarak modelin yakınsaması çok fazla zaman alabilir. (Genellikle algılayıcı modeli için eta‘yı ayarlamak için iyi bir değer 0,1’dir, ancak durumdan duruma farklılık gösterebilir).

Son olarak, bazılarınız ilk girdinin bir sabit ( 1 ) olduğunu ve w0 ile çarpıldığını fark etmiş olabilir. Peki bu tam olarak nedir? İyi bir tahmin elde etmek için bir önyargı eklememiz gerekiyor. Ve işte bu sabit tam olarak budur.

Önyargı teriminin ağırlığını değiştirmek için, diğer ağırlıklar için yaptığımızla aynı denklemi kullanırız, ancak bu durumda onu girdi ile çarpmayız (çünkü girdi sabit bir 1’dir ve dolayısıyla buna gerek yoktur):

Δw = eta * (y - tahmin)

Basit bir algılayıcı modeli temelde böyle çalışır! Ağırlıklarımızı çalıştırdığımızda ona yeni veriler verebilir ve tahminlerimizi alabiliriz.

NOT: #

Perceptron modelinin önemli bir dezavantajı var! Veriler doğrusal olarak ayrılabilir değilse, hiçbir zaman yakınsamayacaktır (e.i mükemmel ağırlıkları bulur), bu, bir özellik alanındaki 2 sınıfı düz bir çizgiyle ayırabilmek anlamına gelir. Bundan kaçınmak için, modelin asla mükemmel şekilde ayarlanamayacak ağırlıkları ayarlamaya takılıp kalmaması için sabit sayıda yineleme eklemek iyi bir uygulamadır.
