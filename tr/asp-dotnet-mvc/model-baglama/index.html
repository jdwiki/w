Model bağlama, tipik olarak bir GET isteğinin Sorgu Dizesinde veya POST gövdesi içinde HTTP parametrelerini alma ve daha sonra Denetleyici eylemlerine gerek kalmadan doğrulanabilen ve nesne yönelimli bir şekilde tüketilebilen bir nesneye uygulama sürecidir. HTTP parametrelerinin nasıl alınacağı konusunda samimi bilgiye sahip olmak.

Başka bir deyişle, model bağlama, MVC’de eylemlerin, ister değer türü ister nesne olsun, parametre(ler)e sahip olmasına izin veren şeydir.

Eylemde örnek oluşturmaya çalışmak için, bağlama modeli süreci çeşitli yerlerde veri arayacaktır:

  • Form verisi
  • Rota Verileri
  • Sorgu Dizesi
  • Dosyalar Özel (örneğin çerezler)

Dosya yükleme #

Model:

public class SampleViewModel
{
    public HttpPostedFileBase file {get;set;}
}

Görüş:

@model HelloWorldMvcApp.SampleViewModel

@using (Html.BeginForm("Index","Home",FormMethod.Post, new { enctype = "multipart/form-data" }))
{
    <div class="form-group">
        @Html.TextBoxFor(model => model.file, new {@class="form-control", type="file"}) 
        @Html.ValidationMessageFor(model => model.file)
    </div>

    <button type="submit" class="btn btn-success submit">Upload</button>
}

Eylem:

[HttpPost]
public ActionResult Index(SampleViewModel model)
{                
    
    if (model.file.ContentLength > 0) 
    {
        string fileName = Path.GetFileName(model.file.FileName);
        string fileLocation = "~/App_Data/uploads/"+ fileName;
        model.file.SaveAs(Server.MapPath(fileLocation));
    }
    return View(model);
}

PostModel’de bağlanmayı engelle #

Bir (post) model göz önüne alındığında:

public class User
{
    public string FirstName { get; set; }
    public bool IsAdmin { get; set; }
}

Şöyle bir görünümle:

@using (Html.BeginForm()) {
    @Html.EditorFor(model => model.FirstName)
    <input type="submit" value="Save" />       
}

Kötü niyetli bir kullanıcının IsAdmin atamasını önlemek için eylemde Bind özniteliğini kullanabilirsiniz:

[HttpPost]
public ViewResult Edit([Bind(Exclude = "IsAdmin")] User user)
{
    // ...
}

Rota değeri bağlama #

{controller=Home}/{action=Index}/{id?} gibi bazı varsayılan yönlendirmeler verildiğinde https://stackoverflow.com/questions/1558902 url’niz varsa

Bu, QuestionsController’a gider ve 1558902 değeri, bir dizin eyleminin id parametresiyle eşlenir, yani.

public ActionResult Index(int? id){
     //id would be bound to id of the route
}

Sorgu dizesi bağlama #

Rotayı genişletmek için https://stackoverflow.com/questions/1558902?sort=desc gibi bir url’niz olduğunu söyleyin.

ve {controller=Home}/{action=Index}/{id?} gibi yönlendirme

public ActionResult Index(int? id, string sort){
     //sort would bind to the value in the query string, i.e. "desc"
}

Nesnelere bağlanma #

Genellikle asp.net-mvc’de viewmodel sınıfları ile çalışıyor olursunuz ve bunlar üzerindeki özelliklere bağlanmak istersiniz. Bu, bireysel parametrelerle eşlemeye benzer şekilde çalışır.

Bunun gibi basit bir görünüm modeli çağrınız olduğunu söyleyin PostViewModel

public class PostViewModel{
  public int Id {get;set;}
  public int SnappyTitle {get;set;}
}

Ardından, http isteğindeki bir formdan Id ve SnappyTitle değerlerini gönderdiniz, ardından modelin kendisi eylem parametresiyse, örn.

public ActionResult UpdatePost(PostViewModel viewModel){
  //viewModel.Id would have our posted value
}

Bağlamanın parametre ve özellik adları için büyük/küçük harfe duyarlı olmadığını belirtmekte fayda var. Ayrıca, mümkün olan yerlerde değerler verecektir. Belirli örnekler için daha fazla uç durum bırakıyorum

Ajax bağlama #

Bunlar, POST yöntemini kullanarak HTTP isteğine giren form değerleridir. (jQuery POST istekleri dahil).

Gibi bir ajax gönderisi yaptığınızı söyleyin

$.ajax({
    type: 'POST',
    url: window.updatePost,
    data:  { id: 21, title: 'snappy title' },
    //kept short for clarity
});

Burada json, id ve title içindeki iki değer, eşleşen eyleme bağlanır, ör.

public JsonResult UpdatePost(int id, string title) {
    ...
}

Genel, Oturum tabanlı model bağlama #

Bazen tüm modeli korumamız ve onu eylemler ve hatta denetleyiciler arasında aktarmamız gerekir. Modeli oturumda saklamak, bu tür gereksinimler için iyi bir çözüm. Bunu MVC’nin güçlü model bağlama özellikleriyle birleştirirsek, bunu yapmanın zarif bir yolunu buluruz. Üç kolay adımda genel oturum tabanlı model bağlama oluşturabiliriz:

Birinci adım: Model bağlayıcı oluşturun

Bir model bağlayıcının kendisi oluşturun. Şahsen /Infrastructure/ModelBinders klasöründe SessionDataModelBinder sınıfını oluşturdum.

using System;
using System.Web.Mvc;

public class SessionDataModelBinder<TModel>
    : IModelBinder
    where TModel : class
{
    private string SessionKey { get; set; }

    public SessionDataModelBinder(string sessionKey)
    {
        if (string.IsNullOrEmpty(sessionKey))
            throw new ArgumentNullException(nameof(sessionKey));
        SessionKey = sessionKey;
    }

    public object BindModel(
        ControllerContext controllerContext, 
        ModelBindingContext bindingContext)
    {
        // Get model from session
        TModel model = controllerContext
            .HttpContext
            .Session[SessionKey] as TModel;
        // Create model if it wasn't found from session and store it
        if (model == null)
        {
            model = Activator.CreateInstance<TModel>();
            controllerContext.HttpContext.Session[SessionKey] = model;
        }
        // Return the model
        return model;
    }
}

İkinci adım: ciltleyiciyi kaydedin

Aşağıdaki gibi bir modelimiz varsa:

public class ReportInfo
{
    public int ReportId { get; set; }
    public ReportTypes TypeId { get; set; }
}

public enum ReportTypes
{
    NotSpecified,
    Monthly, Yearly
}

Bu model için oturum tabanlı model bağlayıcıyı Application_Start yönteminde Global.asax içinde kaydedebiliriz:

protected void Application_Start()
{
    .........

    // Model binders.
    // Remember to specy unique SessionKey
    ModelBinders.Binders.Add(typeof(ReportInfo), 
        new SessionDataModelBinder<ReportInfo>("ReportInfo"));
}

Üçüncü adım: kullanın!

Artık bu model bağlayıcıdan sadece aksiyonlarımıza parametre ekleyerek:

public class HomeController : Controller
{
    public ActionResult Index(ReportInfo reportInfo)
    {
        // Simply set properties
        reportInfo.TypeId = ReportTypes.Monthly;

        return View();
    }

    public ActionResult About(ReportInfo reportInfo)
    {
        // reportInfo.TypeId is Monthly now because we set
        // it previously in Index action.
        ReportTypes currentReportType = reportInfo.TypeId;

        return View();
    }
}

Model bağlayıcı kullanarak tarih alanlarını dinamik biçimlerle manuel olarak doğrulama #

Farklı kullanıcıların farklı tarih saat biçimine ihtiyacı varsa, gelen tarih dizenizi biçime göre gerçek tarihe ayrıştırmanız gerekebilir. Bu durumda bu snippet size yardımcı olabilir.

public class DateTimeBinder : DefaultModelBinder 
{
    public override object BindModel(ControllerContext controllerContext, ModelBindingContext bindingContext)
    {

                var value = bindingContext.ValueProvider.GetValue(bindingContext.ModelName);
                DateTime date;
                var displayFormat = Session["DateTimeFormat"];
                if (value.AttemptedValue != "")
                {
                    if (DateTime.TryParseExact(value.AttemptedValue, displayFormat, CultureInfo.InvariantCulture, DateTimeStyles.None, out date))
                    {
                        return date;
                    }
                    else
                    {
                        bindingContext.ModelState.AddModelError(bindingContext.ModelName, "Invalid date format");
                    }
                }
            }

        return base.BindModel(controllerContext, bindingContext);
    }
