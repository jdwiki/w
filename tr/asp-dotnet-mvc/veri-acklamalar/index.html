Uzaktan doğrulama #

Remote Validation, giriş kontrolüne girilen içeriğin geçerli olup olmadığını kontrol etmek için sunucu tarafına bir ajax isteği göndererek kontrol etmek için kullanılır. #

Çalışma

‘RemoteAttribute’, doğrulanan alanın değeriyle istemciden bir denetleyici eylemine bir AJAX çağrısı yaparak çalışır. Denetleyici eylemi daha sonra doğrulamanın başarılı veya başarısız olduğunu belirten bir “JsonResult” yanıtı döndürür. İşleminizden “true” döndürmek, doğrulamanın geçtiğini gösterir. Diğer herhangi bir değer başarısızlığı gösterir. “false” döndürürseniz, öznitelikte belirtilen hata mesajı kullanılır. Dize veya tamsayı gibi başka bir şey döndürürseniz, hata mesajı olarak görüntülenecektir. Hata mesajınızın dinamik olması gerekmiyorsa, doğru veya yanlış döndürmek ve doğrulayıcının öznitelikte belirtilen hata mesajını kullanmasına izin vermek mantıklıdır.

Görünüm Modeli

public class ViewModel
{
    [Remote("IsEmailAvailable", "Group", HttpMethod = "POST", ErrorMessage = "Email already exists. Please enter a different email address.")]
    public string Email{ get; set; }
}

Kontrolör

[HttpPost]
public JsonResult IsEmailAvailable(string Email)
{
    // Logic to check whether email is already registered or Not.
    var emailExists = IsEmailRegistered();
    return Json(!emailExists);         
}

Özel Doğrulama Özelliği #

Genel veri doğrulaması olmayan bazı kuralların doğrulanması söz konusu olduğunda, örneğin bir alanın gerekli olduğundan emin olmak veya bazı değerler aralığı ancak iş mantığınıza özgüdür, o zaman kendi Özel Doğrulayıcı’nızı oluşturabilirsiniz. Özel bir doğrulama özniteliği oluşturmak için, “ValidationAttribute” sınıfını “devralmanız” ve “IsValid” yöntemini “geçersiz kılmanız” yeterlidir. IsValid yöntemi iki parametre alır; birincisi, ‘değer’ olarak adlandırılan bir ’nesne’ ve ikincisi, ‘validationContext’ olarak adlandırılan bir ‘ValidationContext nesnesidir. Değer, özel doğrulayıcınızın doğrulayacağı alandaki gerçek değeri ifade eder.

“E-posta"yı “Özel Doğrulayıcı” aracılığıyla doğrulamak istediğinizi varsayalım.

public class MyCustomValidator : ValidationAttribute
{
    private static string myEmail= "admin@dotnetfiddle.net";

    protected override ValidationResult IsValid(object value, ValidationContext validationContext)
    {
        string Email = value.ToString();
        if(myEmail.Equals(Email))
            return new ValidationResult("Email Already Exist");
        return ValidationResult.Success;
    }
}

public class SampleViewModel
{
    [MyCustomValidator]
    [Required]
    public string Email { get; set; }

    public string Name { get; set; }
}

Gerekli Özellik #

Required özelliği, bir özelliğin gerekli olduğunu belirtir. Öznitelikte ErrorMessage özelliği kullanıldığında bir hata mesajı belirtilebilir .

İlk önce ad alanını ekleyin:

using System.ComponentModel.DataAnnotations;

Ve niteliği bir mülke uygulayın.

public class Product
{
   [Required(ErrorMessage = "The product name is required.")]
   public string Name { get; set; }

   [Required(ErrorMessage = "The product description is required.")]
   public string Description { get; set; }
}

Globalleştirilmiş uygulamalar için hata mesajındaki kaynakları kullanmak da mümkündür. Bu durumda, ErrorMessageResourceName, kaynak anahtarıyla belirtilmelidir. [ErrorMessageResourceType](https://msdn.microsoft.com/library/system.componentmodel.dataannotations.validationattribute.errormessageresourcetype(v=vs.110) üzerinde ayarlanması gereken kaynak sınıfı (resx dosyası). aspx):

public class Product
{
   [Required(ErrorMessageResourceName = "ProductNameRequired", 
             ErrorMessageResourceType = typeof(ResourceClass))]
   public string Name { get; set; }

   [Required(ErrorMessageResourceName = "ProductDescriptionRequired", 
             ErrorMessageResourceType = typeof(ResourceClass))]
   public string Description { get; set; }
}

Aralık Özelliği #

Normal İfade Özelliği #

[RegularExpression] özelliği, herhangi bir özelliği veya genel alanı süsleyebilir ve özelliğin geçerli sayılması için eşleşmesi gereken bir normal ifade belirtir.

[RegularExpression(validationExpression)]
public string Property { get; set; }

Ek olarak, geçersiz veri girildiğinde kullanıcı tarafından alınan mesajı ayarlamak için kullanılabilecek isteğe bağlı bir ‘ErrorMessage’ özelliğini kabul eder:

[RegularExpression(validationExpression, ErrorMessage = "{your-error-message}")]
public string Property { get; set; }

Örnek(ler)

[RegularExpression(@"^[a-z]{8,16}?$", ErrorMessage = "A User Name must consist of 8-16 lowercase letters")]
public string UserName{ get; set; }
[RegularExpression(@"^\d{5}(-\d{4})?$", ErrorMessage = "Please enter a valid ZIP Code (e.g. 12345, 12345-1234)")]
public string ZipCode { get; set; }
