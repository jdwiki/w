Yönlendirme, ASP.NET MVC’nin bir URI’yi bir eylemle nasıl eşleştirdiğidir. Yönlendirme modülü, gelen tarayıcı isteklerini belirli MVC denetleyici eylemlerine eşlemekten sorumludur.

MVC 5, öznitelik yönlendirme adı verilen yeni bir yönlendirme türünü destekler. Adından da anlaşılacağı gibi, öznitelik yönlendirme, rotaları tanımlamak için öznitelikleri kullanır. Öznitelik yönlendirme, web uygulamanızdaki URI’ler üzerinde size daha fazla kontrol sağlar.

MVC’de öznitelik yönlendirme #

Klasik rota tanımlama yöntemiyle birlikte MVC WEB API 2 ve ardından MVC 5 çerçeveleri “Öznitelik yönlendirme"yi tanıttı:

public class RouteConfig
{
    public static void RegisterRoutes(RouteCollection routes)
    {
        routes.IgnoreRoute("{resource}.axd/{*pathInfo}");
 
        // This enables attribute routing and must go  before other routes are added to the routing table.
        // This makes attribute routes have higher priority
        routes.MapMvcAttributeRoutes();  
    }
}

Bir denetleyici içinde aynı öneki olan rotalar için, “RoutePrefix” özniteliğini kullanarak denetleyici içindeki tüm eylem yöntemleri için ortak bir önek ayarlayabilirsiniz.

[RoutePrefix("Custom")]
public class CustomController : Controller
{
    [Route("Index")]
    public ActionResult Index()
    {
        ...
    }
}

“RoutePrefix” isteğe bağlıdır ve URL’nin denetleyicinin tüm eylemlerinin önüne eklenen bölümünü tanımlar.

Birden fazla rotanız varsa, eylemi parametre olarak yakalayarak varsayılan bir rota ayarlayabilir ve ardından varsayılan rotayı geçersiz kılan belirli eylem yöntem(ler)inde belirli bir ‘Route’ özelliği tanımlanmadıkça bunu tüm denetleyici için uygulayabilirsiniz.

[RoutePrefix("Custom")]
[Route("{action=index}")]
public class CustomController : Controller
{
    public ActionResult Index()
    {
        ...
    }

    public ActionResult Detail()
    {
        ...
    }
}

Tümünü yakalama rotası #

Aşağıdaki gibi sınırsız sayıda segmente izin veren bir rotaya sahip olmak istediğimizi varsayalım:

  • http://example.com/Products/ (tüm ürünleri görüntüleyin)
  • http://example.com/Products/IT
  • http://example.com/Products/IT/Laptops
  • http://example.com/Products/IT/Laptops/Ultrabook
  • http://example.com/Products/IT/Laptops/Ultrabook/Asus
  • vb.

Normalde rota tablosunun sonuna bir rota eklememiz gerekir, çünkü bu muhtemelen aşağıdaki gibi tüm istekleri yakalayacaktır:

routes.MapRoute("Final", "Route/{*segments}",
      new { controller = "Product", action = "View" });

Denetleyicide, bunu halledebilecek bir eylem şunlar olabilir:

public void ActionResult View(string[] segments /* <- the name of the parameter must match the name of the route parameter */)
{
    // use the segments to obtain information about the product or category and produce data to the user
    // ...
}

Yönlendirme temelleri #

Mvc’de özel rota ekleme #

Kullanıcı, bir denetleyicideki belirli bir eyleme bir URL eşleyerek özel rota ekleyebilir. Bu, arama motoru optimizasyonu amacıyla kullanılır ve URL’leri okunabilir hale getirir.

routes.MapRoute(
  name: "AboutUsAspx", // Route name
  url: "AboutUs.aspx",  // URL with parameters
  defaults: new { controller = "Home", action = "AboutUs", id = UrlParameter.Optional }  // Parameter defaults
);

İstemci tarafı yönlendirmeyi etkinleştirmek için tümünü yakalama yolu #

URL’de Tek Sayfalı Uygulamanın (SPA) durumunu kodlamak iyi bir uygulamadır:

my-app.com/admin-spa/users/edit/id123

Bu, uygulama durumunu kaydetmeye ve paylaşmaya izin verir.
Kullanıcı url’yi tarayıcının adres çubuğuna koyduğunda ve sunucuya girdiğinde, istenen url’nin istemci tarafı kısmını yok saymalıdır. SPA’nızı statik bir html dosyası yerine işlenmiş bir Razor görünümü (denetleyici eyleminin çağrılmasının sonucu) olarak sunarsanız, bir tümünü yakalama yolu kullanabilirsiniz:

public class AdminSpaController
{
    [Route("~/admin-spa/{clienSidePart*}")]
    ActionResult AdminSpa()
    {
        ...
    }
}

Bu durumda sunucu sadece SPA döndürür ve daha sonra kendisini rotaya göre başlatır. Bu yaklaşım, url-rewrite modülüne bağlı olmadığı için daha esnektir.

Alanlarda Öznitelik Yönlendirme #

Alanlarda Nitelik Yönlendirmeyi kullanmak için alanların kaydedilmesi ve [RouteArea(...)] tanımları gereklidir.

“RouteConfig.cs” içinde:

public class RouteConfig
{
    public static void RegisterRoutes(RouteCollection routes)
    {
        routes.IgnoreRoute("{resource}.axd/{*pathInfo}");
        routes.MapMvcAttributeRoutes();
        AreaRegistration.RegisterAllAreas();
    }
}

Örnek bir alan denetleyici özniteliği yönlendirme tanımında:

[RouteArea("AreaName", AreaPrefix = "AreaName")]
[RoutePrefix("SampleAreaController")]
public class SampleAreaController : Controller
{
    [Route("Index")]
    public ActionResult Index()
    {
        return View();
    }
}

Alanlarda ‘Url.Action’ bağlantılarını kullanmak için:

@Url.Action("Index", "SampleAreaController", new { area = "AreaName" })

Özel Yönlendirme #

Özel yönlendirme, belirli gelen istekleri işlemek için özel yönlendirme gereksinimi sağlar.

Özel rotalar tanımlamak için rota tablosuna eklediğiniz rotaların sırasının önemli olduğunu unutmayın.

public static void RegisterRoutes(RouteCollection routes)
{
    routes.IgnoreRoute("{resource}.axd/{*pathInfo}");
    
    // this is an advanced custom route
    // you can define custom URL with custom parameter(s) point to certain action method
    routes.MapRoute(
    "CustomEntry", // Route name
    "Custom/{entryId}", // Route pattern
    new { controller = "Custom", action = "Entry" } // Default values for defined parameters above
    );

    // this is a basic custom route
    // any custom routes take place on top before default route
    routes.MapRoute(
    "CustomRoute", // Route name
    "Custom/{controller}/{action}/{id}", // Route pattern
    new { controller = "Custom", action = "Index", id = UrlParameter.Optional } // Default values for defined parameters above
    );

    routes.MapRoute(
    "Default", // Route name
    "{controller}/{action}/{id}", // Route pattern
    new { controller = "Home", action = "Index", id = UrlParameter.Optional } // Default values for defined parameters above
    );
}

“controller” ve “action” isimleri saklıdır. Varsayılan olarak MVC, URL’nin “{controller}” bölümünü “Controller” sınıfıyla eşler ve ardından herhangi bir sonek eklemeden “” adında bir yöntem arar.

“{controller}/{action}/{parameter}” şablonunu kullanarak bir rota ailesi oluşturmak cazip gelse de, bunu yaparak uygulamanızın yapısını ifşa ettiğinizi ve denetleyicinin adını değiştirdiğiniz için URL’leri biraz kırılgan hale getirdiğinizi düşünün. rotayı değiştirir ve kullanıcı tarafından kaydedilen bağlantıları keser.

Açık rota ayarını tercih edin:

routes.MapRoute(
    "CustomRoute", // Route name
    "Custom/Index/{id}", // Route pattern
    new { controller = "Custom", action = nameof(CustomController.Index), id = UrlParameter.Optional }
);

(ilave “Kontrolör” son ekine sahip olacağından, denetleyici adı için “nameof” operatörünü kullanamazsınız) rotada denetleyici adını ayarlarken atlanması gerekir.
