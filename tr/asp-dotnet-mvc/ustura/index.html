Jilet nedir?

Razor, sunucu tabanlı kodu (Visual Basic ve C#) web sayfalarına yerleştirmenizi sağlayan bir biçimlendirme sözdizimidir.

Sunucu tabanlı kod, tarayıcıya bir web sayfası yazılırken anında dinamik web içeriği oluşturabilir. Bir web sayfası çağrıldığında, sunucu, sayfayı tarayıcıya döndürmeden önce sayfanın içindeki sunucu tabanlı kodu yürütür. Kod, sunucu üzerinde çalışarak veritabanlarına erişmek gibi karmaşık görevleri gerçekleştirebilir.

Sözdizimi #

  • @{ … }
  • @değişken ismi
  • @(değişken ismi)
  • @için(…){ }
  • @(Açık İfade)
  • @* yorumlar *@

ASP.NET Razor, hem C# hem de VB için görünüm motorları içerir.

C# görünüm motoru, ‘.cshtml’ uzantılı dosyaları işlerken, VB görünüm motoru ‘.vbhtml’ dosyalarıyla çalışır.

Temel Sözdizimi #

Razor kodu, HTML kodu içinde herhangi bir yere eklenebilir. Jilet kod blokları @{ ... } içine alınır. Satır içi değişken ve işlevler @ ile başlar. Razor parantezlerinin içindeki kod, normal C# veya VB kurallarını takip eder.

Tek satırlık ifade:

@{ var firstNumber = 1; }

Çok satırlı kod bloğu:

@{
    var secondNumber = 2;
    var total = firstNumber + secondNumber;
}

Satır içi değişken kullanma:

<h1>The total count is @total</h1>

** Açıkça satır içi bir değişken kullanma**:

<h2>Item@(item.Id)</h2>

Bu özel örnek için, “Item@item.Id” bir e-postaya benzediği ve Razor tarafından bu şekilde oluşturulacağı için örtük sözdizimini kullanamayacağız.

Kodu kontrol akışı ifadelerinin içine alın:

<h1>Start with some HTML code</h1>

@for (int i = 0; i < total; i++){
    Console.Write(i);
}

<p>Mix in some HTML code for fun!</p>
<p>Add a second paragraph.</p>

@if (total > 3)
{
    Console.Write("The total is greater than 3");
}
else
{
    Console.Write("The total is less than 3");
}

Bu aynı sözdizimi, “for”, “foreach”, “while”, “if”, “switch” gibi tüm ifadeler için kullanılacaktır.

Kodun içine kod ekleme:

@if (total > 3)
{
    if(total == 10)
    {
        Console.Write("The total is 10")
    }
}

İkinci “if” yerine “@” yazmanız gerekmediğini unutmayın. Koddan sonra, mevcut kodun arkasına başka bir kod yazabilirsiniz.

Bir HTML öğesinden sonra kod eklemek istiyorsanız, bir @ yazmanız yapmanız gerekir.

Yorum ekle #

Razor’ın @* ile başlayan ve *@ ile biten kendi yorum sözdizimi vardır.

Satır İçi Yorum:

<h1>Comments can be @*hi!*@ inline</h1>

Çok Satırlı Yorum:

@* Comments can spread
   over multiple
   lines *@

HTML Yorumu

Razor görünümlerinde “” ile biten normal HTML yorum sözdizimini de kullanabilirsiniz. Ancak diğer yorumlardan farklı olarak, bir HTML yorumu içindeki Razor kodu hala normal şekilde yürütülür.

@{
    var hello = "Hello World!"; 
} 
<!-- @hello -->

Yukarıdaki örnek, aşağıdaki HTML çıktısını üretir:

<!-- Hello World! -->

Bir kod bloğu içindeki yorumlar:

@{
    // This is a comment
    var Input = "test";
}

Adında - (tire) bulunan özel bir öznitelik ekleme #

Adında - (tire) olan bir jilet aracılığıyla bir öznitelik eklemeniz gerekiyorsa, bunu basitçe yapamazsınız.

@Html.DropDownListFor(m => m.Id, Model.Values, new { @data-placeholder = "whatever" })

derlemeyecek. data-* öznitelikleri, öğelere fazladan değerler eklemek için html5’te geçerlidir ve yaygındır.

Ancak aşağıdakiler işe yarayacak

@Html.DropDownListFor(m => m.Id, Model.Values, new { @data_placeholder = "whatever" })

“_”, işlendiğinde “-” ile değiştirildiğinden.

Bu, html’deki öznitelik adlarında alt çizgiler kabul edilmediğinden iyi çalışır.

@functions kullanarak satır içi sınıflar ve yöntemler oluşturun #

Razor @functions anahtar sözcüğünü kullanmak, bir Razor dosyasında satır içi kullanım için sınıfları ve yöntemleri tanıtma yeteneği verir:

@functions
{
    string GetCssClass(Status status)
    {
        switch (status)
        {
            case Status.Success:
                return "alert-success";
            case Status.Info:
                return "alert-info";
            case Status.Warning:
                return "alert-warning";
            case Status.Danger:
            default:
                return "alert-danger";
         }
     }
}


<label class="alert @GetCssClass(status)"></label>

Aynı şey sınıflar için de yapılabilir:

@functions
{
    class Helpers
    {
        //implementation
    }
}

Razor kod bloğunda HTML’yi görüntüle #

Bir Razor kod bloğunun içindeyken, tarayıcı HTML kodunu yalnızca koddan kaçarsa tanır.

Tek satır için @: kullanın:

@foreach(int number in Model.Numbers)
{
    @:<h1>Hello, I am a header!</h1>
}

Çok satırlı için <text> ... </text> kullanın:

@{
    var number = 1;

    <text>
        Hello, I am text
        <br / >
        Hello, I am more text!
    </text>
}

Razor’un bir kod bloğunun içindeyken HTML etiketlerini anlayacağını unutmayın. Bu nedenle, HTML etiketlerinin etrafına “metin” etiketi eklemek gereksizdir (yine de doğru olsa da), örneğin:

@{
    var number = 1;
    <text>
        <div>
            Hello, I am text
            <br / >
            Hello, I am more text!
        </div>
    </text>
}

@ karakterden kaçma #

Çoğu durumda, Razor ayrıştırıcısı, bir e-posta adresi gibi bir şeyin parçası olmak yerine, “@” işaretinin ne zaman kodun bir parçası olarak kullanılması gerektiğini anlayacak kadar akıllıdır. Aşağıdaki örnekte, @ işaretinden kaçmak gerekli değildir:

<p>Reach out to us at contact@mail.com</p>

Bununla birlikte, bazı durumlarda, @ işaretinin kullanımı daha belirsizdir ve aşağıdaki örnekte olduğu gibi, “@@” ile açıkça kaçılmalıdır:

<p>Join us @@ Stack Overflow!</p>

Alternatif olarak, HTML kodlu @ karakterini kullanabiliriz.

<p>Join us &#64; Stack Overflow!</p>

@helpers’ı görünümler arasında paylaşın #

@Helpers görünümler arasında paylaşılabilir.

App_Code klasöründe oluşturulmaları gerekir.

@helper CreatePrimaryBootstrapButton(string label)
{
    <button type="button" class="btn btn-primary">@label</button>
}

//call

@MenuHelpers.CreatePrimaryBootstrapButton("my button")

@Url ve @Html globalleri, App_code’da tanımlanan @Helper’da varsayılan olarak mevcut değildir. Bunları aşağıdaki gibi ekleyebilirsiniz (App_code klasörünüzdeki her .cshtml için)

@*  Make @Html and @Url available *@
@functions
{
    private new static HtmlHelper<object> Html
    {
        get { return ((WebViewPage)CurrentPage).Html; }
    }

    private static UrlHelper Url
    {
        get { return ((WebViewPage)CurrentPage).Url; }
    }
 }

Düzenleyici Şablonları #

Düzenleyici şablonları, Razor kodunu yeniden kullanmanın iyi bir yoludur. Düzenleyici şablonlarını Razor kısmi görünümleri olarak tanımlayabilir ve ardından bunları diğer görünümlerde kullanabilirsiniz.

Düzenleyici şablonları genellikle Views/Shared/EditorTemplates/ klasöründe bulunur, ancak Views/ControllerName/EditorTemplates/ klasörüne de kaydedilebilirler. Görünümün adı, genellikle, “.cshtml” gibi, şablonu kullanmak istediğiniz nesnenin adıdır.

İşte DateTime için basit bir düzenleyici şablonu:

@model DateTime
<div>
   <span>
      @Html.TextBox("", Model.ToShortDateString(), new { data_date_picker="true" })
   </span>
</div>

Dosyayı Views/Shared/EditorTemplate/DateTime.cshtml olarak kaydedin.

Ardından, bu şablon kodunu başka bir görünümde çağırmak için EditorFor kullanın:

@Html.EditorFor(m => m.CreatedDate)

Dosya adını belirtmek için bir UIHint niteliği de vardır:

public class UiHintExampleClass
{
    [UIHint("PhoneNumber")]
    public string Phone { get; set; }
}

Bu telefon numarası şablonunu Views/Shared/EditorTemplates/PhoneNumber.cshtml içinde tanımlayın.

Düzenleyici şablonları, Özel Türler için de tanımlanabilir.

İşte “SubModel” adlı özel bir tür:

public class SubModel
{
    public Guid Id { get; set;} 
    public string FirstName { get; set; }
    public string LastName { get; set; }
}

public class Model
{
    public Guid Id { get; set; }
    public DateTime Created {get; set; }
    
    public SubModel SubModel{get; set; }
}

Bu, SubModel için EditorTemplate:

@model SubModel
<div class="form-group">
    @Html.LabelFor(m => m.FirstName)
    @Html.TextBoxFor(m => m.FirstName)
</div>
<div class="form-group">
    @Html.LabelFor(m => m.LastName)
    @Html.TextBoxFor(m => m.LastName)
</div>

Şimdi, Model için Görünüm basitçe şöyle olur:

@model Model
@Html.EditorFor(m => m.CreatedDate)
@Html.EditorFor(m => m.SubModel, new { @Prefix = "New"}) 
@* the second argument is how you can pass viewdata to your editor template*@

Razor içeriğini bir @helper’a iletin #

Bir @helper’a bir Razor parçası gönderin, örneğin bir HTML div.

@helper WrapInBox(Func<Object, HelperResult> content)
{
    <div class="box">@content(null) </div>
}

//call 
@WrapInBox(@<div>
                I'm a inner div
            </div>)
