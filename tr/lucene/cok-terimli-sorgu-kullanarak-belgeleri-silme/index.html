Çok Terimli Sorgu Kullanarak Belgeleri Silme

Belgenizde birincil anahtar alanınız varsa (geleneksel SQL veritabanlarında olduğu gibi) bir Lucene dizininden belgeleri silmek kolaydır.

Ancak, bazen ihtiyacınız olan şey, belgedeki birden çok alana dayalı olarak bir dizi belgeyi silmektir. Lucene API, silme için kullanılacak bir sorgu belirleyerek bunu başarmanızı sağlar.

Bunu yapmak için doğru Analyzer’ı seçin, sorguyu oluşturun, belgeleri silmek için sorguyu indexWriter’a iletin.

Sözdizimi #

  1. indexWriter.deleteDocuments(multiTermQuery);
  2. Sorgu multiTermQuery = new QueryParser("", Analyzer).parse(“field_name1:"field value 1" AND field_name2:"field value 2"”);
  3. BooleanQuery multiTermQuery = new BooleanQuery(); multiTermQuery.add(new TermQuery(new Term(“field_name1”, “field value 1”)), BooleanClause.Occur.MUST); multiTermQuery.add(new TermQuery(new Term(“field_name2”, “field value 2”)), BooleanClause.Occur.MUST);

Analizör Seçimiyle İlgili Uyarılar #

Hemen belli olmaz, ancak kullandığınız analizör, sorgunuzun çalıştırılma biçiminde büyük bir fark yaratır. Bunun nedeni, StandardAnalyzer’ın “the” ve “a” gibi yaygın İngilizce sözcükleri filtrelemesidir. Tam olarak eşleşmesi için farklı bir çözümleyici (KeywordAnalyzer gibi) seçmek isteyebilirsiniz. Bu tabii ki Lucene uygulamanıza bağlı.

Analizör Seçimi #

Her şeyden önce, hangi analizörü kullandığınıza dikkat edin. StandardAnalyzer’ın ’the’ ve ‘a’ gibi yaygın sözcükleri filtrelediğini fark ettiğimde bir süre şaşırdım. Alanınız ‘A’ değerine sahip olduğunda bu bir sorundur. KeywordAnalyzer’ı düşünmek isteyebilirsiniz:

Analizörün etrafındaki bu gönderiye bakın.

// Create an analyzer:
// NOTE: We want the keyword analyzer so that it doesn't strip or alter any terms:
// In our example, the Standard Analyzer removes the term 'A' because it is a common English word.
// http://stackoverflow.com/a/9071806/231860
KeywordAnalyzer analyzer = new KeywordAnalyzer();

Sorgu Ayrıştırıcı #

Ardından, QueryParser’ı kullanarak sorgunuzu oluşturabilirsiniz:

Varsayılan operatörün geçersiz kılınmasıyla ilgili bu gönderiye bakın.

// Create a query parser without a default field in this example (the first argument):
QueryParser queryParser = new QueryParser("", analyzer);

// Optionally, set the default operator to be AND (we leave it the default OR):
// http://stackoverflow.com/a/9084178/231860
// queryParser.setDefaultOperator(QueryParser.Operator.AND);

// Parse the query:
Query multiTermQuery = queryParser.parse("field_name1:\"field value 1\" AND field_name2:\"field value 2\"");

Sorgu API’sı #

Veya API’lerini kullanarak sorguyu kendiniz oluşturarak da aynısını yapabilirsiniz:

BooleanQuery oluşturmayla ilgili bu eğiticiye bakın.

BooleanQuery multiTermQuery = new BooleanQuery();
multiTermQuery.add(new TermQuery(new Term("field_name1", "field value 1")), BooleanClause.Occur.MUST);
multiTermQuery.add(new TermQuery(new Term("field_name2", "field value 2")), BooleanClause.Occur.MUST);

Sorguyla Eşleşen Belgeleri Sil #

Ardından, sorguyla eşleşen belgeleri silmek için sorguyu yazara iletiyoruz:

Bu sorunun yanıtına bakın.

Burada API’ye bakın

// Remove the document by using a multi key query:
// http://www.avajava.com/tutorials/lessons/how-do-i-combine-queries-with-a-boolean-query.html
indexWriter.deleteDocuments(multiTermQuery);
