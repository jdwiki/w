Pencereler #

  1. En son nodeJS LTS sürümünün kurulu olduğundan emin olun
  2. Komut istemini açın ve $ npm install -g nativescript yazın
  3. Komut istemine $ @powershell -NoProfile -ExecutionPolicy Bypass -Command "iex ((new-object net.webclient).DownloadString('https://www.nativescript.org/setup/win'))" yazın - bu biraz zaman alabilir
  4. Yukarıdakilerin işe yaradığını doğrulamak için komut istemine `$ tns doktor’ yazın (cmd’niz)
  5. Herhangi bir hata varsa, sorun giderme kılavuzunu takip edin.

Kurulum veya Kurulum #

Nativescript’i kurmaya veya yüklemeye ilişkin ayrıntılı talimatlar.

Aşağıdaki örnekler, bir Windows veya OSX sistemi kurmak için gerekli adımları gösterir ve ardından herhangi bir sorun yaşamanız durumunda sorun giderme kılavuzlarına gönderin.

Ayrıca, önerilen iş akışlarının, IDE’lerin ve öykünücülerin nasıl kurulacağına ilişkin örnekler de vardır.

Mac os işletim sistemi #

  1. en yeni Node.js LTS’nin kurulu olduğundan emin olun. Homebrew kullanıyorsanız bu, ‘brew install node4-lts’ ile yapılabilir.
  2. Terminal’i açın ve npm install -g nativescript yazın. Bir “EACCES” hatası alırsanız, “sudo npm install -g nativescript” kullanın.
  3. Komut istemine ruby -e "$(curl -fsSL https://www.nativescript.org/setup/mac)" yazın. (Bu biraz zaman alabilir.)
  4. Yukarıdakilerin işe yaradığını doğrulamak için Terminal’e `tns doktor’ yazın.
  5. Herhangi bir hata varsa, sorun giderme kılavuzunu takip edin.

NativeScript geliştirme için Visual Studio Code’u kullanma #

Visual Studio Code, Microsoft’un açık kaynaklı ve zengin özelliklere sahip bir kod düzenleyicisidir. Bunu NativeScript geliştirmesi için ayarlamak için Komut Paletini açın (F1 veya ⌘+Shift+P) ve “ext install NativeScript” yazın.

NativeScript uzantısı yüklendikten sonra hata ayıklayıcı, kodunuzda kesme noktaları ayarlamanıza izin vermelidir. Bir cihaz bağlandığında veya bir öykünücü çalışırken, uygulamanızı Hata Ayıklama sekmesinden başlatabilirsiniz.

İlk Merhaba Dünya programınız #

$ mkdir hello-world
$ cd hello-world
$ tns create hello-world --ng
$ tns platform add android #You can only add ios on an OSX machine

Ardından, bağlı bir cihazınız olduğundan veya bir emülatörün çalıştığından emin olun (eğer yapmazsanız, varsayılan emülatör başlamalıdır veya bir hata ortaya çıkacaktır. Android için genymotion’u tavsiye ederim).

$ tns run android

Varsayılan Android öykünücüsünü kullanmak istiyorsanız, “–emulator” bayrağını ekleyin.

tns 2.5’ten itibaren livesync, artık dosya değişikliklerini kaydettiğinizde otomatik olarak yeniden derlenecek olan tns run <platform> için varsayılan eylemdir. Bu, geliştirme sürenizi önemli ölçüde artırabilir, ancak eklentilerinizde değişiklik yaparsanız, düzgün bir şekilde yeniden derlemeniz gerekir.

WiFi üzerinden nativescript-Android Uygulamasında Hata Ayıklama (Kök olmadan) #

1-Cihazınızı USB kablosu ile bilgisayarınıza bağlamanız gerekmektedir. USB hata ayıklamanın çalıştığından emin olun. “adb devices” (veya “tns device”) çalıştırıldığında görünüp görünmediğini kontrol edebilirsiniz.

2-‘adb tcpip 5555’i çalıştırın

3-Cihazınızın bağlantısını kesin (USB kablosunu çıkarın).

4-Telefonunuzun IP adresini görüntülemek için Ayarlar -> Telefon hakkında -> Durum bölümüne gidin.

5-adb connect <cihazınızın IP adresi>:5555 komutunu çalıştırın

6-Eğer tekrar adb devices (veya tns device) çalıştırırsanız cihazınızı görmelisiniz.

7- Artık tns run android ,tns livesync android komutlarını kullanabilirsiniz.

NOTLAR :

1-WiFi ağı değiştiğinde 1’den 3’e kadar olan adımları tekrarlamanız gerekmez (bu, telefonunuzu wifi hata ayıklama moduna ayarlar). 4 ila 6 arasındaki adımları uygulayarak telefonunuza tekrar bağlanmanız gerekir.

2-Android telefonlar yeniden başlatıldığında wifi hata ayıklama modunu kaybediyor. Bu nedenle, piliniz biterse baştan başlamanız gerekir. Aksi takdirde, pilinize dikkat eder ve telefonunuzu yeniden başlatmazsanız haftalarca kablosuz yaşayabilirsiniz!

UYARI :

seçeneği etkin bırakmak tehlikelidir, veri ağında olsanız bile ağınızdaki herkes hata ayıklama sırasında cihazınıza bağlanabilir. Bunu yalnızca güvenilir bir Wi-Fi’ye bağlıyken yapın ve bittiğinde bağlantısını kesmeyi unutmayın!
