Lisp genellikle öğrencilerin özyinelemeli algoritmaları anlamayı ve uygulamayı öğrendiği eğitim bağlamlarında kullanılır.

Common Lisp’te veya taşınabilir kodda yazılan üretim kodunun özyinelemeyle ilgili birkaç sorunu vardır: Bunlar, kuyruk çağrısı optimizasyonu gibi uygulamaya özel özelliklerden yararlanmazlar, bu da genellikle özyinelemeyi tamamen önlemeyi gerekli kılar. Bu durumlarda, uygulamalar:

  • Yığın boyutlarındaki sınırlamalar nedeniyle genellikle bir yineleme derinliği sınırı vardır. Bu nedenle özyinelemeli algoritmalar yalnızca sınırlı boyuttaki veriler için çalışacaktır.
  • Özellikle dinamik kapsamlı işlemlerle birlikte kuyruk çağrılarının optimizasyonunu her zaman sağlamayın.
  • Yalnızca belirli optimizasyon seviyelerinde kuyruk çağrılarının optimizasyonunu sağlar.
  • Genellikle kuyruk çağrısı optimizasyonu sağlamaz.
  • Genellikle belirli platformlarda kuyruk çağrısı optimizasyonu sağlamaz. Örneğin, JVM’nin kendisi kuyruk çağrısı optimizasyonunu desteklemediğinden JVM üzerindeki uygulamalar bunu yapmayabilir.

Kuyruk çağrılarını atlamalarla değiştirmek genellikle hata ayıklamayı daha da zorlaştırır; Atlamalar eklemek, yığın çerçevelerinin hata ayıklayıcıda kullanılamamasına neden olur. Alternatif olarak Common Lisp şunları sağlar:

  • ‘DO’, ‘DOTIMES’, ‘LOOP’ ve diğerleri gibi yineleme yapıları
  • “HARİTA”, “AZALT” ve diğerleri gibi üst düzey işlevler
  • Düşük seviyeli “git” dahil olmak üzere çeşitli kontrol yapıları

Özyineleme şablonu 1 tek koşul tek kuyruk özyineleme #

(defun fn (x)
  (cond (test-condition the-value)
        (t (fn reduced-argument-x))))

Bir listenin öğelerini yinelemeli olarak yazdırın #

;;Recursively print the elements of a list
(defun print-list (elements)
    (cond
        ((null elements) '()) ;; Base case: There are no elements that have yet to be printed. Don't do anything and return a null list.
        (t
            ;; Recursive case
            ;; Print the next element.
            (write-line (write-to-string (car elements)))
            ;; Recurse on the rest of the list.
            (print-list (cdr elements))
        )
    )
)

Bunu test etmek için şunu çalıştırın:

(setq test-list '(1 2 3 4))
(print-list test-list)

1
2
3
4

Bir tam sayının faktöriyelini hesaplayın #

Özyinelemeli bir işlev olarak uygulanması kolay bir algoritma faktöriyeldir.

;;Compute the factorial for any n >= 0. Precondition: n >= 0, n is an integer.
(defun factorial (n)
    (cond
        ((= n 0) 1) ;; Special case, 0! = 1
        ((= n 1) 1) ;; Base case, 1! = 1
        (t
            ;; Recursive case
            ;; Multiply n by the factorial of n - 1.
            (* n (factorial (- n 1)))
        )
    )
)

n’inci Fibonacci sayısını hesapla #

;;Find the nth Fibonacci number for any n > 0.
;; Precondition: n > 0, n is an integer. Behavior undefined otherwise.
(defun fibonacci (n)
    (cond
        (                                     ;; Base case.
             ;; The first two Fibonacci numbers (indices 1 and 2) are 1 by definition.
            (<= n 2)                          ;; If n <= 2
            1                                 ;; then return 1.
        )
        (t                                    ;; else
            (+                                ;; return the sum of
                                              ;; the results of calling 
                (fibonacci (- n 1))           ;; fibonacci(n-1) and
                (fibonacci (- n 2))           ;; fibonacci(n-2).
                                              ;; This is the recursive case.
            )
        )
    )
)
