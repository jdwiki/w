Parametreler #

Açıklama Ayrıntılar
@OneToOne Karşılık gelen bir nesneyle bire bir ilişki belirtir.
@OneToMany Birçok nesneyle eşleşen tek bir nesneyi belirtir.
@ManyToOne Tek bir nesneyle eşleşen nesneler koleksiyonunu belirtir.
@Entity Bir veritabanı tablosuyla eşleşen bir nesneyi belirtir.
@Tablo Bu nesnenin de hangi veritabanı tablosuyla eşleştiğini belirtir.
@JoinColumn Bir foregin anahtarının hangi sütunda depolandığını belirtir.
@JoinTable Yabancı anahtarları depolayan bir ara tablo belirtir.

Çift Yönlü Çoktan Çoka, kullanıcı tarafından yönetilen birleştirme tablosu nesnesini kullanma #

@Entity
@Table(name="FOO")
public class Foo {
    private UUID fooId;
    
    @OneToMany(mappedBy = "bar")
    private List<FooBar> bars;
}

@Entity
@Table(name="BAR")
public class Bar {
    private UUID barId;
    
    @OneToMany(mappedBy = "foo")
    private List<FooBar> foos;
}

@Entity
@Table(name="FOO_BAR")
public class FooBar {
    private UUID fooBarId;

    @ManyToOne
    @JoinColumn(name = "fooId")
    private Foo foo;

    @ManyToOne
    @JoinColumn(name = "barId")
    private Bar bar;

    //You can store other objects/fields on this table here.
}

Kullanıcının yönettiği bir ara birleştirme tablosunu kullanarak birçok “Foo” nesnesi ile birçok “Bar” nesnesi arasında iki yönlü bir ilişki belirtir.

‘Foo’ nesneleri, ‘FOO’ adlı bir tabloda satırlar olarak depolanır. ‘Bar’ nesneleri, ‘BAR’ adlı bir tabloda satırlar olarak depolanır. ‘Foo’ ve ‘Bar’ nesneleri arasındaki ilişkiler, ‘FOO_BAR’ adlı bir tabloda saklanır. Uygulamanın bir parçası olarak bir ‘FooBar’ nesnesi var.

İlişkinin oluşturulduğu tarih gibi birleştirme nesnesinde fazladan bilgi depolamak istediğinizde yaygın olarak kullanılır.

İki Yönlü Hazırda Bekletme yönetilen birleştirme tablosunu kullanan Çoka Çok Yönlü #

@Entity
@Table(name="FOO")
public class Foo {
    private UUID fooId;
    
    @OneToMany
    @JoinTable(name="FOO_BAR",
        joinColumns = @JoinColumn(name="fooId"),
        inverseJoinColumns = @JoinColumn(name="barId"))
    private List<Bar> bars;
}

@Entity
@Table(name="BAR")
public class Bar {
    private UUID barId;
    
    @OneToMany
    @JoinTable(name="FOO_BAR",
        joinColumns = @JoinColumn(name="barId"),
        inverseJoinColumns = @JoinColumn(name="fooId"))
    private List<Foo> foos;
}

Hazırda Bekletme’nin yönettiği bir ara birleştirme tablosunu kullanarak birçok “Foo” nesnesi ile birçok “Bar” nesnesi arasındaki ilişkiyi belirtir.

‘Foo’ nesneleri, ‘FOO’ adlı bir tabloda satırlar olarak depolanır. ‘Bar’ nesneleri, ‘BAR’ adlı bir tabloda satırlar olarak depolanır. ‘Foo’ ve ‘Bar’ nesneleri arasındaki ilişkiler, ‘FOO_BAR’ adlı bir tabloda saklanır. Ancak bu, uygulamanın bir parçası olarak ‘FooBar’ nesnesi olmadığı anlamına gelir.

Yabancı anahtar eşleme kullanarak Çift Yönlü Bire Çok İlişki #

@Entity
@Table(name="FOO") 
public class Foo {
    private UUID fooId;
    
    @OneToMany(mappedBy = "bar")
    private List<Bar> bars;
}

@Entity
@Table(name="BAR")
public class Bar {
    private UUID barId;
    
    @ManyToOne
    @JoinColumn(name = "fooId")
    private Foo foo;
}

Bir “Foo” nesnesi ile bir yabancı anahtar kullanan birçok “Bar” nesnesi arasında iki yönlü bir ilişki belirtir.

‘Foo’ nesneleri, ‘FOO’ adlı bir tabloda satırlar olarak depolanır. ‘Bar’ nesneleri, ‘BAR’ adlı bir tabloda satırlar olarak depolanır. Yabancı anahtar, ‘BAR’ tablosunda ‘fooId’ adlı bir sütunda saklanır.

Foo.class tarafından yönetilen Çift Yönlü Bire Bir İlişki #

@Entity
@Table(name="FOO")    
public class Foo {
    private UUID fooId;
    
    @OneToOne(cascade = CascadeType.ALL)
    @JoinColumn(name = "barId")
    private Bar bar;
}

@Entity
@Table(name="BAR")
public class Bar {
    private UUID barId;
    
    @OneToOne(mappedBy = "bar")
    private Foo foo;
}

Yabancı anahtar kullanan bir “Foo” nesnesi ile bir “Bar” nesnesi arasında iki yönlü bir ilişki belirtir.

‘Foo’ nesneleri, ‘FOO’ adlı bir tabloda satırlar olarak depolanır. ‘Bar’ nesneleri, ‘BAR’ adlı bir tabloda satırlar olarak depolanır. Yabancı anahtar, ‘FOO’ tablosunda ‘barId’ adlı bir sütunda saklanır.

“mappedBy” değerinin, sütun adı değil, nesne üzerindeki alan adı olduğuna dikkat edin.

Kullanıcı tarafından yönetilen birleştirme tablosunu kullanan Tek Yönlü Bire Çok İlişki #

@Entity
@Table(name="FOO")
public class Foo {
    private UUID fooId;
    
    @OneToMany
    @JoinTable(name="FOO_BAR",
        joinColumns = @JoinColumn(name="fooId"),
        inverseJoinColumns = @JoinColumn(name="barId", unique=true))
    private List<Bar> bars;
}

@Entity
@Table(name="BAR")
public class Bar {
    private UUID barId;

    //No Mapping specified here.
}

@Entity
@Table(name="FOO_BAR")
public class FooBar {
    private UUID fooBarId;

    @ManyToOne
    @JoinColumn(name = "fooId")
    private Foo foo;

    @ManyToOne
    @JoinColumn(name = "barId", unique = true)
    private Bar bar;

    //You can store other objects/fields on this table here.
}

Kullanıcının yönettiği bir ara birleştirme tablosunu kullanarak bir “Foo” nesnesi ile birçok “Bar” nesnesi arasında tek yönlü bir ilişki belirtir.

Bu, “ManyToMany” ilişkisine benzer, ancak hedef yabancı anahtara “benzersiz” bir kısıtlama eklerseniz, bunun “OneToMany” olmasını zorunlu kılabilirsiniz.

‘Foo’ nesneleri, ‘FOO’ adlı bir tabloda satırlar olarak depolanır. ‘Bar’ nesneleri, ‘BAR’ adlı bir tabloda satırlar olarak depolanır. ‘Foo’ ve ‘Bar’ nesneleri arasındaki ilişkiler, ‘FOO_BAR’ adlı bir tabloda saklanır. Uygulamanın bir parçası olarak bir ‘FooBar’ nesnesi var.

“Bar” nesnelerinin “Foo” nesnelerine eşlenmediğine dikkat edin. ‘Bar’ nesneleri, ‘Foo’ nesnelerini etkilemeden serbestçe değiştirilebilir.

Spring Security ile, gerçekleştirebilecekleri bir “Rol” listesine sahip bir “Kullanıcı” nesnesi kurarken çok yaygın olarak kullanılır. Bir kullanıcıya, “Rol"leri silen basamaklar hakkında endişelenmenize gerek kalmadan roller ekleyebilir ve kaldırabilirsiniz.

Tek Yönlü Bire Bir İlişki #

@Entity
@Table(name="FOO")
public class Foo {
    private UUID fooId;
    
    @OneToOne
    private Bar bar;
}

@Entity
@Table(name="BAR")
public class Bar {
    private UUID barId;
    //No corresponding mapping to Foo.class
}

Bir “Foo” nesnesi ile bir “Bar” nesnesi arasında tek yönlü bir ilişki belirtir.

‘Foo’ nesneleri, ‘FOO’ adlı bir tabloda satırlar olarak depolanır. ‘Bar’ nesneleri, ‘BAR’ adlı bir tabloda satırlar olarak depolanır.

“Bar” nesnelerinin “Foo” nesnelerine eşlenmediğine dikkat edin. ‘Bar’ nesneleri, ‘Foo’ nesnelerini etkilemeden serbestçe değiştirilebilir.
