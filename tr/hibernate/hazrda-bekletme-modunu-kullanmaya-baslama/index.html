Hazırda Beklet’i ayarlamak için XML Yapılandırmasını Kullanma #

Sınıf yolunda bir yerde “database-servlet.xml” adında bir dosya oluşturuyorum.

Başlangıçta yapılandırma dosyanız şöyle görünecektir:

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
xmlns:jdbc="http://www.springframework.org/schema/jdbc"
xmlns:tx="http://www.springframework.org/schema/tx"
xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans-3.2.xsd
    http://www.springframework.org/schema/jdbc http://www.springframework.org/schema/jdbc/spring-jdbc-3.2.xsd
    http://www.springframework.org/schema/tx http://www.springframework.org/schema/tx/spring-tx-3.2.xsd">

</beans>

“tx” ve “jdbc” Spring ad alanlarını içe aktardığımı fark edeceksiniz. Bunun nedeni, onları bu yapılandırma dosyasında oldukça yoğun bir şekilde kullanacağımızdır.

Yapmak istediğiniz ilk şey, açıklama tabanlı işlem yönetimini (@Transactional) etkinleştirmektir. İnsanların Bahar’da Hazırda Bekletme modunu kullanmasının ana nedeni, Spring’in tüm işlemlerinizi sizin için yönetmesidir. Yapılandırma dosyanıza aşağıdaki satırı ekleyin:

<tx:annotation-driven />

Bir veri kaynağı oluşturmamız gerekiyor. Veri kaynağı temel olarak Hazırda Bekletme’nin nesnelerinizi sürdürmek için kullanacağı veritabanıdır. Genellikle bir işlem yöneticisinin bir veri kaynağı olacaktır. Hazırda Bekletme modunun birden çok veri kaynağıyla konuşmasını istiyorsanız, birden çok işlem yöneticiniz vardır.

<bean id="dataSource" 
    class="org.springframework.jdbc.datasource.DriverManagerDataSource">
    <property name="driverClassName" value="" />
    <property name="url" value="" />
    <property name="username" value="" />
    <property name="password" value="" />
</bean>

Bu fasulyenin sınıfı, ‘javax.sql.DataSource’u uygulayan herhangi bir şey olabilir, böylece kendinizinkini yazabilirsiniz. Bu örnek sınıf, Spring tarafından sağlanmıştır, ancak kendi iş parçacığı havuzuna sahip değildir. Popüler bir alternatif, Apache Commons org.apache.commons.dbcp.BasicDataSource‘tur, ancak daha birçokları vardır. Aşağıdaki özelliklerin her birini açıklayacağım:

  • driveClassName: JDBC sürücünüzün yolu. Bu, sınıf yolunuzda bulunması gereken veritabanına özgü bir JAR’dır. En güncel sürüme sahip olduğunuzdan emin olun. Bir Oracle veritabanı kullanıyorsanız, bir OracleDriver’a ihtiyacınız olacaktır. Bir MySQL veritabanınız varsa, bir MySQLDriver’a ihtiyacınız olacaktır. İhtiyacınız olan sürücüyü burada bulabilecek misiniz bir bakın, ancak hızlı bir google size doğru sürücüyü vermelidir.

  • url: Veritabanınızın URL’si. Genellikle bu, “jdbc:oracle:thin:\path\to\your\veritabanı” veya “jdbc:mysql://path/to/your/database” gibi bir şey olacaktır. Kullanmakta olduğunuz veritabanının varsayılan konumu için Google’da arama yaparsanız, bunun ne olması gerektiğini öğrenebilmelisiniz. ‘org.hibernate.HibernateException: ‘hibernate.dialect’ ayarlanmadığında bağlantı boş olamaz’ mesajıyla birlikte bir ‘HibernateException’ alıyorsanız ve bu kılavuzu izliyorsanız, URL’nizin yanlış olma olasılığı %90’dır, %5 olasılıkla veritabanınızın başlatılmamış olması ve %5 olasılıkla kullanıcı adınızın/parolanızın yanlış olması.

  • kullanıcı adı: Veritabanı ile kimlik doğrulaması yapılırken kullanılacak kullanıcı adı.

  • parola: Veritabanı ile kimlik doğrulaması yapılırken kullanılacak parola.

Bir sonraki şey, SessionFactoryyi kurmaktır. Bu, Hazırda Bekletme’nin işlemlerinizi oluşturmak ve yönetmek için kullandığı şeydir ve aslında veritabanıyla konuşur. Aşağıda açıklamaya çalışacağım epeyce yapılandırma seçeneğine sahip.

<bean id="sessionFactory"
    class="org.springframework.orm.hibernate4.LocalSessionFactoryBean">
    <property name="dataSource" ref="dataSource" />
    <property name="packagesToScan" value="au.com.project />
    <property name="hibernateProperties">
        <props>
            <prop key="hibernate.use_sql_comments">true</prop>
            <prop key="hibernate.hbm2ddl.auto">validate</prop>
        </props>
    </property>
</bean>

  • dataSource: Veri kaynağınız. DataSource’un kimliğini değiştirdiyseniz, buradan ayarlayın.

  • packagesToScan: JPA açıklamalı nesnelerinizi bulmak için taranacak paketler. Bunlar, oturum fabrikasının yönetmesi gereken, genellikle POJO’lar olacak ve @Entity ile açıklamalı nesnelerdir. Hazırda Bekletme modunda nesne ilişkilerinin nasıl kurulacağı hakkında daha fazla bilgi için buraya bakın.

  • annotatedClasses (gösterilmemiştir): Hepsi aynı pakette değilse, Hazırda Bekletme’nin taraması için sınıfların bir listesini de sağlayabilirsiniz. “packagesToScan” veya “annotatedClasses” kullanmalısınız, ancak ikisini birden kullanmamalısınız. Deklarasyon şöyle görünür:

<property name="annotatedClasses">
    <list>
        <value>foo.bar.package.model.Person</value>
        <value>foo.bar.package.model.Thing</value>
    </list>
</property>
  • hibernateProperties: Bunların hepsi sevgiyle burada belgelenmiştir sayısız vardır. Kullanacağınız başlıcaları aşağıdaki gibidir:
  • hibernate.hbm2ddl.auto: En sıcak Hazırda Bekleme sorularından biri bu özelliği detaylandırır. Daha fazla bilgi için bakın. Genelde validate kullanırım ve veritabanımı SQL komut dosyalarını (bellek içi için) kullanarak kurarım veya veritabanını önceden oluştururum (mevcut veritabanı).
  • hibernate.show_sql: Boole bayrağı, eğer doğruysa Hazırda Bekletme, ürettiği tüm SQL’i stdouta yazdırır. Günlük yöneticinizde log4j.logger.org.hibernate.type=TRACE log4j.logger.org.hibernate.SQL=DEBUG ayarını yaparak, günlükçünüzü sorgulara bağlı değerleri gösterecek şekilde de yapılandırabilirsiniz. (log4j kullanıyorum).
  • hibernate.format_sql: Boolean bayrağı, Hibernate’in SQL’inizi stdout’a güzel bir şekilde yazdırmasına neden olur.
  • hibernate.dialect (Gerekli bir nedenle gösterilmemiştir): Dışarıdaki birçok eski öğretici, veritabanınızla iletişim kurmak için kullanacağı Hazırda Bekletme lehçesini nasıl ayarlayacağınızı gösterir. Hazırda Bekletme, kullandığınız JDBC sürücüsüne bağlı olarak hangi diyalektin kullanılacağını otomatik olarak algılayabilir. Yaklaşık 3 farklı Oracle lehçesi ve 5 farklı MySQL lehçesi olduğundan, bu kararı Hazırda Bekletme’ye bırakırdım. Hibernate’in desteklediği lehçelerin tam listesi için [buraya bakın] 5.

Beyan etmeniz gereken son 2 fasulye:

<bean class="org.springframework.dao.annotation.PersistenceExceptionTranslationPostProcessor"
    id="PersistenceExceptionTranslator" />

<bean id="transactionManager" 
    class="org.springframework.orm.hibernate4.HibernateTransactionManager">
    <property name="sessionFactory" ref="sessionFactory" />
</bean>

‘PersistenceExceptionTranslator’, veritabanına özel ‘HibernateException’ veya ‘SQLExceptions’ı, uygulama bağlamı tarafından anlaşılabilen Spring istisnalarına çevirir.

‘TransactionManager’ çekirdeği, işlemleri ve geri alma işlemlerini kontrol eden şeydir.

Not: SessionFactory fasulyenizi DAO’larınıza otomatik olarak bağlamanız gerekir.

XML’siz Hazırda Bekletme yapılandırması #

Bu örnek buradan alınmıştır.

package com.reborne.SmartHibernateConnector.utils;

import org.hibernate.HibernateException;
import org.hibernate.Session;
import org.hibernate.SessionFactory;
import org.hibernate.cfg.Configuration;

public class LiveHibernateConnector implements IHibernateConnector {

    private String DB_DRIVER_NAME = "";
    private String DB_URL = "jdbc:h2:~/liveDB;MV_STORE=FALSE;MVCC=FALSE";
    private String DB_USERNAME = "sa";
    private String DB_PASSWORD = "";
    private String DIALECT = "org.hibernate.dialect.H2Dialect";
    private String HBM2DLL = "create";
    private String SHOW_SQL = "true";
    
    private static Configuration config;
    private static SessionFactory sessionFactory;
    private Session session;
    
    private boolean CLOSE_AFTER_TRANSACTION = false;

    public LiveHibernateConnector() {
        
        config = new Configuration();

        config.setProperty("hibernate.connector.driver_class",         DB_DRIVER_NAME);
        config.setProperty("hibernate.connection.url",                 DB_URL);
        config.setProperty("hibernate.connection.username",         DB_USERNAME);
        config.setProperty("hibernate.connection.password",         DB_PASSWORD);
        config.setProperty("hibernate.dialect",                     DIALECT);
        config.setProperty("hibernate.hbm2dll.auto",                 HBM2DLL);
        config.setProperty("hibernate.show_sql",                    SHOW_SQL);
    
        /*
         * Config connection pools
         */

        config.setProperty("connection.provider_class", "org.hibernate.connection.C3P0ConnectionProvider");
        config.setProperty("hibernate.c3p0.min_size", "5");
        config.setProperty("hibernate.c3p0.max_size", "20");
        config.setProperty("hibernate.c3p0.timeout", "300");
        config.setProperty("hibernate.c3p0.max_statements", "50");
        config.setProperty("hibernate.c3p0.idle_test_period", "3000");
        
        
        /**
         * Resource mapping
         */
        
//        config.addAnnotatedClass(User.class);
//        config.addAnnotatedClass(User.class);
//        config.addAnnotatedClass(User.class);
    
        sessionFactory = config.buildSessionFactory();
    }


    public HibWrapper openSession() throws HibernateException {
        return new HibWrapper(getOrCreateSession(), CLOSE_AFTER_TRANSACTION);
    }


    public Session getOrCreateSession() throws HibernateException {
        if (session == null) {
            session = sessionFactory.openSession();
        }
        return session;
    }

    public void reconnect() throws HibernateException {
        this.sessionFactory = config.buildSessionFactory();
    }

    
}

Lütfen, en son Hazırda Bekletme ile bu yaklaşımın iyi çalışmadığını unutmayın (Hazırda Bekletme 5.2 sürümü hala bu yapılandırmaya izin verir)

XML kullanarak basit hazırda bekletme örneği #

Yapılandırmalar için XML kullanarak basit bir hazırda bekletme projesi oluşturmak için hibernate.cfg.xml, her varlık için bir POJO ve her varlık için bir EntityName.hbm.xml olmak üzere 3 dosyaya ihtiyacınız vardır. İşte MySQL kullanan her birine bir örnek:

hibernate.cfg.xml

<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE hibernate-configuration PUBLIC 
"-//Hibernate/Hibernate Configuration DTD 3.0//EN"
"http://hibernate.sourceforge.net/hibernate-configuration-3.0.dtd">

<hibernate-configuration>
   <session-factory>
   <property name="hibernate.dialect">
      org.hibernate.dialect.MySQLDialect
   </property>
   <property name="hibernate.connection.driver_class">
      com.mysql.jdbc.Driver
   </property>

   <property name="hibernate.connection.url">
      jdbc:mysql://localhost/DBSchemaName
   </property>
   <property name="hibernate.connection.username">
      testUserName
   </property>
   <property name="hibernate.connection.password">
      testPassword
   </property>

   <!-- List of XML mapping files -->
   <mapping resource="HibernatePractice/Employee.hbm.xml"/>

</session-factory>
</hibernate-configuration>

DBSchemaName, testUserName ve testPassword değiştirilecektir. Bir paketteyse tam kaynak adını kullandığınızdan emin olun.

çalışan.java

package HibernatePractice;

public class Employee {
    private int id;
    private String firstName;
    private String middleName;
    private String lastName;
    
    public Employee(){
        
    }
    public int getId(){
        return id;
    }
    public void setId(int id){
        this.id = id;
    }
    public String getFirstName(){
        return firstName;
    }
    public void setFirstName(String firstName){
        this.firstName = firstName;
    }
    public String getMiddleName(){
        return middleName;
    }
    public void setMiddleName(String middleName){
        this.middleName = middleName;
    }
    public String getLastName(){
        return lastName;
    }
    public void setLastName(String lastName){
        this.lastName = lastName;
    }
}

Çalışan.hbm.xml

<hibernate-mapping>
   <class name="HibernatePractice.Employee" table="employee">
      <meta attribute="class-description">
         This class contains employee information. 
      </meta>
      <id name="id" type="int" column="empolyee_id">
         <generator class="native"/>
      </id>
      <property name="firstName" column="first_name" type="string"/>
      <property name="middleName" column="middle_name" type="string"/>
      <property name="lastName" column="last_name" type="string"/>
   </class>
</hibernate-mapping>

Yine, sınıf bir paketteyse, paketAdı.sınıfAdı tam sınıf adını kullanın.

Bu üç dosyaya sahip olduktan sonra, projenizde hazırda bekletme modunu kullanmaya hazırsınız.
