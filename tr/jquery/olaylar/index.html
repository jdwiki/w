jQuery, olayları addEventListener işlevi aracılığıyla dahili olarak işler. Bu, aynı DOM öğesi için aynı olaya bağlı birden fazla işleve sahip olmanın tamamen yasal olduğu anlamına gelir.

Yetkilendirilen Etkinlikler #

Örnekle başlayalım. İşte çok basit bir örnek HTML.

Örnek HTML

<html>
    <head>
    </head>
    <body>
        <ul>
            <li>
                <a href="some_url/">Link 1</a>
            </li>
            <li>
                <a href="some_url/">Link 2</a>
            </li>
            <li>
                <a href="some_url/">Link 3</a>
            </li>
        </ul>
    </body>
</html>

Sorun

Şimdi bu örnekte, tüm <a> öğelerine bir olay dinleyicisi eklemek istiyoruz. Sorun, bu örnekteki listenin dinamik olmasıdır. “

  • ” öğeleri zaman geçtikçe eklenir ve çıkarılır. Ancak, sayfa değişiklikler arasında yenilenmez, bu da bağlantı nesnelerine yönelik basit tıklama olay dinleyicilerini kullanmamıza izin verir (yani, $('a').click()).

    Elimizdeki problem, gelip giden <a> öğelerine nasıl olay ekleyeceğimizdir.

    Arka plan bilgisi - Olay yayılımı

    Temsil edilen olaylar, yalnızca olay yayılımı (genellikle olay köpürmesi olarak adlandırılır) nedeniyle mümkündür. Ne zaman bir olay başlatılsa, sonuna kadar (belge köküne kadar) kabaracaktır. Bir olayın işlenmesini değişmeyen bir üst öğeye delege eder, dolayısıyla “temsil edilen” olaylar adını alırlar.

    Bu nedenle, yukarıdaki örnekte, “” öğe bağlantısının tıklanması, bu öğelerde şu sırayla ’tıklama’ olayını tetikleyecektir:

    • a -li
    • ul
    • gövde
    • html
    • Doküman kaynağı

    Çözüm

    Olay köpürmesinin ne yaptığını bilerek, HTML’miz aracılığıyla yayılan istenen olaylardan birini yakalayabiliriz.

    Bu örnekte onu yakalamak için iyi bir yer “

      ” öğesidir, çünkü bu öğe dinamik değildir:

      $('ul').on('click', 'a', function () {
  console.log(this.href); // jQuery binds the event function to the targeted DOM element
                          // this way `this` refers to the anchor and not to the list
  // Whatever you want to do when link is clicked
});

      Yukarılarda:

      • Bu olay dinleyicisinin alıcısı olan ‘ul’ var
      • İlk parametre (’tıklama’), hangi olayları algılamaya çalıştığımızı tanımlar.
      • İkinci parametre (‘a’), olayın nereden başlaması gerektiğini bildirmek için kullanılır (bu olay dinleyicisinin alıcısı altındaki tüm alt öğelerden, ul).
      • Son olarak üçüncü parametre, birinci ve ikinci parametrelerin gereksinimleri karşılandığında çalıştırılan koddur.

      Çözümün nasıl çalıştığını ayrıntılı olarak

      1. Kullanıcı “” öğesini tıklar
      2. Bu, <a> öğesinde tıklama olayını tetikler.
      3. Olay, belge köküne doğru köpürmeye başlar.
      4. Olay önce <li> öğesine ve ardından <ul> öğesine baloncuk yapar.
      5. Olay dinleyicisi, <ul> öğesinin bağlı olduğu olay dinleyicisine sahip olduğu için çalıştırılır.
      6. Olay dinleyicisi önce tetikleyici olayı algılar. Köpüren olay ’tıklama’dır ve dinleyici ’tıklama’ya sahiptir, bu bir geçiştir.
      7. Dinleyici, ikinci parametreyi (‘a’) baloncuk zincirindeki her bir öğeyle eşleştirmeye çalışır. Zincirdeki son öğe bir ‘a’ olduğundan, bu filtreyle eşleşir ve bu da bir geçiştir.
      8. Üçüncü parametredeki kod, bu olduğu için eşleşen öğe kullanılarak çalıştırılır. Eğer fonksiyon stopPropagation() için bir çağrı içermiyorsa, olay yukarı doğru köke (“belge”) doğru yayılmaya devam edecektir.

      Not: Değişmeyen uygun bir ata mevcut değilse/uygun değilse, “belge” kullanmalısınız. Alışkanlık olarak “beden"i aşağıdaki nedenlerle kullanmayın:

      • “gövde"nin stil ile ilgili bir hatası var, bu da fare olaylarının baloncuk oluşturmadığı anlamına gelebilir. Bu, tarayıcıya bağlıdır ve hesaplanan gövde yüksekliği 0 olduğunda (örneğin, tüm alt öğelerin mutlak konumları olduğunda) gerçekleşebilir. Fare olayları her zaman “belge"ye dönüşür.
      • “belge” her zaman betiğinizde bulunur, böylece bir DOM’ye hazır işleyicinin dışında “belge"ye yetki verilmiş işleyiciler ekleyebilir ve bunların çalışmaya devam edeceğinden emin olabilirsiniz.

      Olay İşleyicileri Ekleme ve Çıkarma #

      Olay İşleyicisi Ekle #

      1.7 sürümünden beri jQuery, .on() olay API’sine sahiptir. Bu şekilde, herhangi bir standart javascript olayı veya özel olay, seçili olan jQuery öğesine bağlanabilir. .click() gibi kısayollar vardır, ancak .on() size daha fazla seçenek sunar.

      HTML #

      <button id="foo">bar</button>

      jQuery #

      $( "#foo" ).on( "click", function() {
  console.log( $( this ).text() ); //bar
});

      Bir Olay İşleyicisini Ayırın #

      Doğal olarak, olayları jQuery nesnelerinizden de ayırma olanağına sahipsiniz. Bunu .off( olaylar [, seçici ] [, işleyici ] ) kullanarak yaparsınız.

      HTML #

      <button id="hello">hello</button>

      jQuery #

      $('#hello').on('click', function(){
    console.log('hello world!');
    $(this).off();
});

      $(this) düğmesine tıklandığında, mevcut jQuery nesnesine atıfta bulunulacak ve ekli tüm olay işleyicileri ondan kaldırılacaktır. Hangi olay işleyicisinin kaldırılması gerektiğini de belirtebilirsiniz.

      jQuery #

      $('#hello').on('click', function(){
    console.log('hello world!');
    $(this).off('click');
});

$('#hello').on('mouseenter', function(){
    console.log('you are about to click');
});

      Bu durumda mouseenter olayı tıkladıktan sonra da çalışmaya devam edecektir.

      Belge Yükleme Olayı .load() #

      Komut dosyanızın bir resim veya PDF gibi belirli bir kaynak yüklenene kadar beklemesini istiyorsanız, .on( "load", işleyici) kısayolunun kısayolu olan .load() kullanabilirsiniz.

      HTML

      <img src="image.jpeg" alt="image" id="image">

      jQuery

      $( "#image" ).load(function() {
  // run script
});

      Kimlikleri kullanmadan öğeleri tekrarlamak için etkinlikler #

      Sorun

      Sayfada, belirli bir örnekle bir şeyler yapmak için hangisinde bir olayın meydana geldiğini bilmeniz gereken bir dizi yinelenen öğe vardır.

      Çözüm

      • Tüm ortak öğelere ortak bir sınıf verin
      • Olay dinleyicisini bir sınıfa uygulayın. ’this’ içindeki olay işleyicisi, olayın gerçekleştiği eşleşen seçici öğedir
      • ’this’ten başlayarak, o örnek için en çok yinelenen dış kaba geçin
      • Bu örneğe özgü diğer öğeleri yalıtmak için o kap içinde find() kullanın

      HTML

      <div class="item-wrapper" data-item_id="346">
   <div class="item"><span class="person">Fred</span></div>
   <div class="item-toolbar">
      <button class="delete">Delete</button>
   </div>   
</div>
<div class="item-wrapper" data-item_id="393">
   <div clss="item"><span class="person">Wilma</span></div>
   <div class="item-toolbar">
      <button class="delete">Delete</button>
   </div>   
</div>

      jQuery

      $(function() {
  $('.delete').on('click', function() {
    // "this" is element event occured on
    var $btn = $(this);
    // traverse to wrapper container
    var $itemWrap = $btn.closest('.item-wrapper');
    // look within wrapper to get person for this button instance
    var person = $itemWrap.find('.person').text();
    // send delete to server and remove from page on success of ajax
    $.post('url/string', { id: $itemWrap.data('item_id')}).done(function(response) {
      $itemWrap.remove()
    }).fail(function() {
      alert('Ooops, not deleted at server');
    });
  });
});

      orijinalEtkinlik #

      Bazen jQuery olayında bulunmayan özellikler olabilir. Temel özelliklere erişmek için Event.originalEvent kullanın

      Kaydırma Yönünü Al #

      $(document).on("wheel",function(e){
    console.log(e.originalEvent.deltaY)
    // Returns a value between -100 and 100 depending on the direction you are scrolling
})

      Belirli olayları jQuery aracılığıyla açma ve kapatma. (Adlandırılmış Dinleyiciler) #

      Bazen önceden kaydedilmiş tüm dinleyicileri kapatmak istersiniz.

      //Adding a normal click handler
$(document).on("click",function(){
    console.log("Document Clicked 1")
});
//Adding another click handler
$(document).on("click",function(){
    console.log("Document Clicked 2")
});
//Removing all registered handlers.
$(document).off("click")

      Bu yöntemle ilgili bir sorun, diğer eklentiler vb. tarafından “belgeye” bağlanan TÜM dinleyicilerin de kaldırılacak olmasıdır.

      Çoğu zaman, yalnızca bizim tarafımızdan bağlanan tüm dinleyicileri ayırmak istiyoruz.

      Bunu başarmak için adlandırılmış dinleyicileri şu şekilde bağlayabiliriz:

      //Add named event listener.
$(document).on("click.mymodule",function(){
    console.log("Document Clicked 1")
});
$(document).on("click.mymodule",function(){
    console.log("Document Clicked 2")
});

//Remove named event listener.
$(document).off("click.mymodule");

      Bu, başka herhangi bir tıklama dinleyicisinin yanlışlıkla değiştirilmemesini sağlar.
