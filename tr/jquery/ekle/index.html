Parametreler #

parametreler Ayrıntılar
içerik Olası türler: Öğe, HTML dizesi, metin, dizi, nesne ve hatta bir dize döndüren bir işlev.

  • .append() & .after() potansiyel olarak kodu çalıştırabilir. Bu kod çalıştıran komut dosyası etiketlerinin enjeksiyonu veya HTML niteliklerinin kullanımı (örneğin, ). eklemek için bu yöntemleri kullanmayın. URL sorgu parametreleri gibi güvenilmeyen kaynaklardan elde edilen dizeler, çerezler veya form girdileri. Bunu yapmak, siteler arası komut dosyası çalıştırmayı tanıtabilir (XSS) güvenlik açıkları. Eklemeden önce herhangi bir kullanıcı girişini kaldırın veya çıkın belgenin içeriği.

  • jQuery, SVG’yi resmi olarak desteklemiyor. SVG’de jQuery yöntemlerini kullanma belgeler, bu yöntem için açıkça belgelenmedikçe, beklenmedik davranışlar Şu andan itibaren SVG’yi destekleyen yöntem örnekleri jQuery 3.0, addClass ve removeClass’tır.

Verimli ardışık .append() kullanımı #

Başlayan:

HTML #

<table id='my-table' width='960' height='500'></table>

JS #

var data = [
    { type: "Name", content: "John Doe" },
    { type: "Birthdate", content: "01/01/1970" },
    { type: "Salary", content: "$40,000,000" },
    // ...300 more rows...
    { type: "Favorite Flavour", content: "Sour" }
];

Bir döngü içine ekleme

Az önce büyük bir veri dizisi aldınız. Şimdi döngüye girme ve sayfada oluşturma zamanı.

İlk düşünceniz şöyle bir şey yapmak olabilir:

var i;                        // <- the current item number
var count = data.length;      // <- the total
var row;                      // <- for holding a reference to our row object

// Loop over the array
for ( i = 0; i < count; ++i ) {
    row = data[ i ];

    // Put the whole row into your table
    $('#my-table').append(
        $('<tr></tr>').append(
            $('<td></td>').html(row.type),
            $('<td></td>').html(row.content)
        )
    );
}

Bu kesinlikle geçerlidir ve tam olarak beklediğiniz şeyi verecektir, ancak…

Bunu yapma. #

Bu 300+ veri satırını hatırlıyor musunuz?

Her biri, tarayıcıyı diğer stiller ile birlikte her öğenin genişliğini, yüksekliğini ve konumlandırma değerlerini yeniden hesaplamaya zorlar - bunlar bir düzen sınırı ile ayrılmadıkça, maalesef bu örnek için (olduğu gibi). bir “

” öğesinin torunlarıdır), yapamazlar.

Küçük miktarlarda ve birkaç sütunda, bu performans cezası kesinlikle ihmal edilebilir olacaktır. Ancak her milisaniyenin sayılmasını istiyoruz.

Daha iyi seçenekler

  1. Ayrı bir diziye ekleyin, döngü tamamlandıktan sonra ekleyin
/** 
 * Repeated DOM traversal (following the tree of elements down until you reach
 * what you're looking for - like our <table>) should also be avoided wherever possible.
 */

// Keep the table cached in a variable then use it until you think it's been removed
var $myTable = $('#my-table');

// To hold our new <tr> jQuery objects
var rowElements = [];

var count = data.length;
var i;
var row;

// Loop over the array
for ( i = 0; i < count; ++i ) {
    rowElements.push(
        $('<tr></tr>').append(
            $('<td></td>').html(row.type),
            $('<td></td>').html(row.content)
        )
    );
}

// Finally, insert ALL rows at once
$myTable.append(rowElements);

Bu seçeneklerden en çok jQuery’ye güvenir.

İşlevsel olarak ondan öncekine eşdeğer, yalnızca okunması daha kolay.

  1. HTML dizelerini kullanma (yerleşik jQuery yöntemleri yerine)
// ...
var rowElements = data.map(function ( row ) {
    var rowHTML = '<tr><td>';
    rowHTML += row.type;
    rowHTML += '</td><td>';
    rowHTML += row.content;
    rowHTML += '</td></tr>';
    return rowHTML;
});

// Using .join('') here combines all the separate strings into one
$myTable.append(rowElements.join(''));

Tamamen geçerli ama yine de önerilmez. Bu, jQuery’yi çok büyük miktarda metni bir kerede ayrıştırmaya zorlar ve gerekli değildir. jQuery, doğru kullanıldığında yaptığı işte çok iyidir.

  1. Öğeleri manuel olarak oluşturun, belge parçasına ekleyin
var $myTable = $(document.getElementById('my-table'));

/**
 * Create a document fragment to hold our columns
 * - after appending this to each row, it empties itself
 *   so we can re-use it in the next iteration.
 */
var colFragment = document.createDocumentFragment();

/**
 * Loop over the array using .reduce() this time.  
 * We get a nice, tidy output without any side-effects.
 *  - In this example, the result will be a 
 *    document fragment holding all the <tr> elements.
 */
var rowFragment = data.reduce(function ( fragment, row ) {

    // Create a row
    var rowEl = document.createElement('tr');

    // Create the columns and the inner text nodes
    var typeEl = document.createElement('td');
    var typeText = document.createTextNode(row.type);
    typeEl.appendChild(typeText);

    var contentEl = document.createElement('td');
    var contentText = document.createTextNode(row.content);
    contentEl.appendChild(contentText);

    // Add the columns to the column fragment
    // - this would be useful if columns were iterated over separately
    //   but in this example it's just for show and tell.
    colFragment.appendChild(typeEl);
    colFragment.appendChild(contentEl);
    
    rowEl.appendChild(colFragment);

    // Add rowEl to fragment - this acts as a temporary buffer to 
    // accumulate multiple DOM nodes before bulk insertion
    fragment.appendChild(rowEl);

    return fragment;
}, document.createDocumentFragment());

// Now dump the whole fragment into your table
$myTable.append(rowFragment);

Kişisel favorim. Bu, jQuery’nin daha düşük bir düzeyde ne yaptığına dair genel bir fikri gösterir.

Bir kapsayıcıya öğe ekleme #

Çözüm 1:

$('#parent').append($('#child'));

Çözüm 2:

$('#child').appendTo($('#parent'));

Her iki çözüm de “#child” öğesini (sona ekleyerek) “#parent” öğesini ekliyor.

Önceki:

<div id="parent">
  <span>other content</span>
</div>
<div id="child">

</div>

Sonrasında:

<div id="parent">
  <span>other content</span>
  <div id="child">

  </div>
</div>

Not: Belgede zaten var olan içeriği eklediğinizde, bu içerik orijinal üst kapsayıcısından kaldırılacak ve yeni üst kapsayıcıya eklenecektir. Yani bir öğeyi klonlamak için .append() veya .appendTo() kullanamazsınız. Bir klona ihtiyacınız varsa .clone() -> http://api.jquery.com/clone/ kullanın
