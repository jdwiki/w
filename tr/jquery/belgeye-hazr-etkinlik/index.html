Belgeye hazır nedir ve nasıl kullanmalıyım? #

jQuery kodu genellikle jQuery(function($) { ... }); içine sarılır, böylece yalnızca DOM yüklemeyi bitirdikten sonra çalışır.

<script type="text/javascript"> 
  jQuery(function($) {
    // this will set the div's text to "Hello".
    $("#myDiv").text("Hello");
  });
</script>

<div id="myDiv">Text</div>

Bu önemlidir, çünkü jQuery (ve genel olarak JavaScript) sayfada oluşturulmamış bir DOM öğesini seçemez.

<script type="text/javascript">
  // no element with id="myDiv" exists at this point, so $("#myDiv") is an
  // empty selection, and this will have no effect
  $("#myDiv").text("Hello");
</script>

<div id="myDiv">Text</div>

.ready() yöntemine özel bir işleyici ileterek jQuery ad alanına diğer ad verebileceğinizi unutmayın. Bu, başka bir JS kitaplığının bir çakışma yaratan jQuery ile aynı kısaltılmış $ takma adını kullandığı durumlarda kullanışlıdır. Bu çakışmayı önlemek için $.noConflict(); çağırmalısınız - Bu sizi yalnızca varsayılan jQuery ad alanını kullanmaya zorlar (kısa $ takma adı yerine). .ready() işleyicisine özel bir işleyici ileterek, jQuery kullanmak için takma ad seçebileceksiniz.

$.noConflict();

jQuery( document ).ready(function( $ ) {
   // Here we can use '$' as jQuery alias without it conflicting with other 
   // libraries that use the same namespace 
   $('body').append('<div>Hello</div>')
});

jQuery( document ).ready(function( jq ) {
   // Here we use a custom jQuery alias 'jq' 
   jq('body').append('<div>Hello</div>')
});

jQuery kodunuzu sayfanın en altına koymak yerine, $(document).ready işlevini kullanmak, tüm HTML öğelerinin oluşturulmasını ve Belge Nesne Modelinin (DOM) tamamının JavaScript kodunun yürütülmesi için hazır olmasını sağlar.

jQuery 2.2.3 ve önceki sürümler #

Bunların hepsi eşdeğerdir, blokların içindeki kod, belge hazır olduğunda çalışacaktır:

$(function() {
  // code
});

$().ready(function() {
  // code
});

$(document).ready(function() {
  // code
});

Bunlar eşdeğer olduklarından, ilki önerilen biçimdir, aşağıda aynı sonuçları veren ‘$’ yerine ‘jQuery’ anahtar kelimesine sahip bir sürüm verilmiştir:

jQuery(function() {
  // code
});

jQuery 3.0 #

Gösterim #

jQuery 3.0’dan itibaren yalnızca bu form önerilir:

jQuery(function($) {
  // Run when document is ready
  // $ (first argument) will be internal reference to jQuery
  // Never rely on $ being a reference to jQuery in the global namespace
});

Diğer tüm belgeye hazır işleyiciler jQuery 3.0’da kullanımdan kaldırılmıştır.

Eşzamansız #

jQuery 3.0’dan itibaren, hazır işleyici her zaman eşzamansız olarak çağrılacak. Bu, aşağıdaki kodda, belgenin yürütme noktasında hazır olup olmadığına bakılmaksızın, ‘dış işleyici’ günlüğünün her zaman önce görüntüleneceği anlamına gelir.

$(function() {
  console.log("inside handler");
});
console.log("outside handler");

> dış işleyici > iç işleyici

$(belge).ready() ve $(window).load() arasındaki fark #

$(window).load() jQuery sürüm 1.8’de kullanımdan kaldırıldı (ve jQuery 3.0‘den tamamen kaldırıldı) ve bu nedenle artık kullanılmamalıdır. Kullanımdan kaldırmanın nedenleri bu olayla ilgili jQuery sayfasında belirtilmiştir.

Resimlerle kullanıldığında load olayının uyarıları

Geliştiricilerin .load() kısayolunu kullanarak çözmeye çalıştıkları yaygın bir zorluk, bir görüntü (veya görüntü koleksiyonu) tamamen yüklendiğinde bir işlevi yürütmektir. Bununla ilgili dikkat edilmesi gereken birkaç bilinen uyarı vardır. Bunlar:

  • Tarayıcılar arasında tutarlı ve güvenilir bir şekilde çalışmıyor
  • “src” görüntüsü öncekiyle aynı “src"ye ayarlanmışsa WebKit’te doğru şekilde tetiklenmez
  • DOM ağacını doğru şekilde kabartmıyor
  • Tarayıcının önbelleğinde zaten yaşayan resimler için ateşlenmeyi durdurabilir

Hala load() kullanmak istiyorsanız, aşağıda belgelenmiştir:

$(document).ready() tam DOM kullanılabilir olana kadar bekler – HTML’deki tüm öğeler ayrıştırılır ve belgede bulunur. Ancak resimler gibi kaynaklar bu noktada tam olarak yüklenmemiş olabilir. Tüm kaynaklar yüklenene kadar beklemek önemliyse, $(window).load() ve bu olayın önemli sınırlamalarının farkındaysanız, bunun yerine aşağıdaki kullanılabilir:

$(document).ready(function() {
  console.log($("#my_large_image").height()); // may be 0 because the image isn't available
});

$(window).load(function() {
  console.log($("#my_large_image").height()); // will be correct
});

Olayları ekleme ve hazır içindeki DOM’yi manipüle etme () #

$(document).ready() için örnek kullanımlar:

  1. Olay işleyicileri ekleme jQuery olay işleyicileri ekleyin

    $(document).ready(function() { $(“button”).click(function() { // Code for the click function }); });

  2. Sayfa yapısı oluşturulduktan sonra jQuery kodunu çalıştırın

    jQuery(function($) { // set the value of an element. $("#myElement”).val(“Hello”); });

  3. Yüklenen DOM yapısını değiştirin Örneğin: sayfa ilk kez yüklendiğinde bir ‘div’i gizleyin ve bir düğmenin tıklama olayında göster

    $(document).ready(function() { $("#toggleDiv").hide(); $(“button”).click(function() { $("#toggleDiv").show(); }); });

jQuery(fn) ile kodunuzu öncesinde çalıştırma arasındaki fark #

Belgeye hazır olayı kullanmak, ~300 ms’ye kadar gecikmeli yürütme ile küçük performans dezavantajlarına sahip olabilir. Bazen aynı davranış, </body> kapanış etiketinden hemen önce kodun çalıştırılmasıyla da elde edilebilir:

<body>
  <span id="greeting"></span> world!
  <script>
    $("#greeting").text("Hello");
  </script>
</body>

benzer davranış üretecek, ancak şurada olduğu gibi belgeye hazır olay tetikleyicisini beklemediğinden daha erken çalışacak:

<head>
  <script>
    jQuery(function($) {
      $("#greeting").text("Hello");
    });
  </script>
</head>
<body>
  <span id="greeting"></span> world!
</body>

İlk örneğin, sayfanız hakkındaki bilginize ve komut dosyasının </body> kapanış etiketinden hemen önce ve özellikle span etiketinden sonra yerleştirilmesine dayandığını vurgulayın.
