Başlarken #

Aşağıdaki içeriğe sahip bir “hello.html” dosyası oluşturun:

<!DOCTYPE html>
<html>
<head>
    <title>Hello, World!</title>
</head>
<body>
    <div>
        <p id="hello">Some random text</p>
    </div>
    <script src="https://code.jquery.com/jquery-2.2.4.min.js"></script>
    <script>
        $(document).ready(function() {
            $('#hello').text('Hello, World!');
        });
    </script>
</body>
</html>

JSBin’de Canlı Demo

Bu dosyayı bir web tarayıcısında açın. Sonuç olarak, Merhaba Dünya! yazan bir sayfa göreceksiniz.

Kodun açıklaması #

  1. jQuery kitaplığını jQuery CDN‘den yükler:

    <script src="https://code.jquery.com/jquery-2.2.4.min.js"></script>

Bu, “jQuery” işlevi ve ad alanı için bir takma ad olan “$” global değişkenini tanıtır.

Dikkat ederken yapılan en yaygın hatalardan birinin jQuery, başka herhangi bir komut dosyasından ÖNCE kitaplığı yükleyemiyor veya ona bağlı olabilecek veya ondan yararlanabilecek kütüphaneler.

  1. DOM’nin (Belge Nesne Modeli) jQuery tarafından [“hazır”][belgeye hazır] olduğu tespit edildiğinde yürütülecek bir işlevi erteler:

    // When the `document` is `ready`, execute this function `...`
$(document).ready(function() { ... });

// A commonly used shorthand version (behaves the same as the above)
$(function() { ... });

  2. DOM hazır olduğunda, jQuery yukarıda gösterilen geri arama işlevini yürütür. Fonksiyonumuzun içinde 2 ana işi yapan tek bir çağrı vardır:

  3. ‘id’ özniteliği ‘merhaba’ya eşit olan öğeyi alır (selektörümüz ‘#hello’). Bir seçiciyi iletilen argüman olarak kullanmak, jQuery’nin işlevselliğinin ve adlandırmasının özüdür; kitaplığın tamamı esasen document.querySelectorAllMDN uzantısından evrimleşmiştir.

  4. Seçili öğenin içindeki [text()][text] öğesini Merhaba, Dünya! olarak ayarlayın.

       #    ↓ - Pass a `selector` to `$` jQuery, returns our element
   $('#hello').text('Hello, World!');
   #             ↑ - Set the Text on the element

Daha fazla bilgi için [jQuery - Documentation][jQuery - Documentation] sayfasına bakın.

Ad alanı çakışmalarından kaçınma #

jQuery dışındaki kitaplıklar da takma ad olarak $ kullanabilir. Bu, bu kitaplıklar ve jQuery arasında parazite neden olabilir.

$ diğer kitaplıklarla kullanmak üzere serbest bırakmak için:

jQuery.noConflict();

Bu işlevi çağırdıktan sonra, “$” artık “jQuery” için bir takma ad değildir. Ancak, jQuery işlevlerine erişmek için yine de “jQuery” değişkenini kullanabilirsiniz:

jQuery('#hello').text('Hello, World!');

İsteğe bağlı olarak, jQuery için takma ad olarak farklı bir değişken atayabilirsiniz:

var jqy = jQuery.noConflict();
jqy('#hello').text('Hello, World!');

Tersine, diğer kitaplıkların jQuery ile karışmasını önlemek için jQuery kodunuzu bir [hemen çağrılan işlev ifadesi (IIFE)](https://www.wikiod.com/tr/javascript/fonksiyonlar#Hemen Çağrılan İşlev İfadeleri) içine sarabilirsiniz. -function-expressions#t=201607152249399289216) ve bağımsız değişken olarak “jQuery"yi iletin:

(function($) {
    $(document).ready(function() {
        $('#hello').text('Hello, World!');
    });
})(jQuery);

Bu IIFE içinde, $ yalnızca jQuery için bir takma addır.

jQuery’nin $ takma adının güvenliğini sağlamanın ve DOM’nin hazır olduğundan emin olmanın başka bir basit yolu:

jQuery(function( $ ) { // DOM is ready
   // You're now free to use $ alias
   $('#hello').text('Hello, World!');
});

Özetlemek,

  • jQuery.noConflict() : $ artık jQuery’ye atıfta bulunmazken, jQuery değişkeni yapar.
  • var jQuery2 = jQuery.noConflict() - $ artık jQuery’ye atıfta bulunmazken, jQuery değişkeni ve aynı şekilde jQuery2 değişkeni de öyle.

Şimdi, üçüncü bir senaryo var - Ya jQuery’nin yalnızca `jQuery2’ içinde kullanılabilir olmasını istiyorsak? Kullanmak,

var jQuery2 = jQuery.noConflict(true)

Bu, jQuery’ye atıfta bulunan “$” veya “jQuery” ile sonuçlanmaz.

Bu, jQuery’nin birden çok sürümü aynı sayfaya yüklenecek olduğunda kullanışlıdır.

<script src='https://code.jquery.com/jquery-1.12.4.min.js'></script>
<script>
    var jQuery1 = jQuery.noConflict(true);
</script>
<script src='https://code.jquery.com/jquery-3.1.0.min.js'></script>
<script>
    // Here, jQuery1 refers to jQuery 1.12.4 while, $ and jQuery refers to jQuery 3.1.0.
</script>

https://learn.jquery.com/using-jquery-core/avoid-conflicts-other-libraries/

jQuery Ad Alanı (“jQuery” ve “$”) #

“jQuery”, herhangi bir jQuery kodu yazmak için başlangıç ​​noktasıdır. “jQuery(…)” işlevi veya “jQuery.foo” değişkeni olarak kullanılabilir.

‘$’, ‘jQuery’ için bir takma addır ve ikisi genellikle birbiriyle değiştirilebilir (‘jQuery.noConflict();’ öğesinin kullanıldığı yerler dışında - bkz. Ad alanı çakışmalarından kaçınma).

Bu HTML parçacığına sahip olduğumuzu varsayarsak -

<div id="demo_div" class="demo"></div>

Bu div’e bir miktar metin içeriği eklemek için jQuery kullanmak isteyebiliriz. Bunu yapmak için jQuery text() işlevini kullanabiliriz. Bu, “jQuery” veya “$” kullanılarak yazılabilir. yani -

jQuery("#demo_div").text("Demo Text!");

Veya -

$("#demo_div").text("Demo Text!");

Her ikisi de aynı nihai HTML ile sonuçlanacaktır -

<div id="demo_div" class="demo">Demo Text!</div>

$, jQueryden daha özlü olduğu için, jQuery kodu yazmak için genellikle tercih edilen yöntemdir.

jQuery, CSS seçicilerini kullanır ve yukarıdaki örnekte bir ID seçici kullanılmıştır. jQuery’deki seçiciler hakkında daha fazla bilgi için bkz. seçici türleri.

Konsol aracılığıyla jQuery’yi olmayan bir sayfaya yükleme. #

Bazen, çoğu geliştiricinin kullanışlı “jQuery"ye sahip olmasına alışkınken, “jQuery” kullanmayan sayfalarla çalışmak gerekir.

Bu gibi durumlarda, aşağıdakileri çalıştırarak yüklenen bir sayfaya manuel olarak “jQuery” eklemek için “Chrome Geliştirici Araçları” konsolu (F12) kullanılabilir:

var j = document.createElement('script');
j.onload = function(){ jQuery.noConflict(); };
j.src = "https://ajax.googleapis.com/ajax/libs/jquery/1.12.4/jquery.min.js";
document.getElementsByTagName('head')[0].appendChild(j);

İstediğiniz sürüm yukarıdakinden farklı olabilir(1.12.4) için [ihtiyacınız olanın] bağlantısını buradan alabilirsiniz]1.

HTML sayfasının başına komut dosyası etiketi ekle #

jQuery‘yi resmi CDN‘den yüklemek için jQuery web sitesine gidin. Mevcut farklı sürümlerin ve biçimlerin bir listesini göreceksiniz.

Şimdi, yüklemek istediğiniz jQuery sürümünün kaynağını kopyalayın. Diyelim ki jQuery 2.X‘i yüklemek istiyorsunuz, size şöyle bir şey gösterecek olan sıkıştırılmamış veya küçültülmüş etiketine tıklayın:

[

Kodun tamamını kopyalayın (veya kopyala simgesine tıklayın) ve html’nizin “” veya “” kısmına yapıştırın.

En iyi uygulama, harici JavaScript kitaplıklarını “async” özniteliğiyle head etiketine yüklemektir. İşte bir gösteri:

<!DOCTYPE html>
    <html>
      <head>
         <title>Loading jquery-2.2.4</title>
         <script src="https://code.jquery.com/jquery-2.2.4.min.js" async></script>
      </head>
      <body>
          <p>This page is loaded with jquery.</p>
      </body>
   </html>

‘async’ niteliğini kullanırken, javascript kitaplıkları daha sonra eşzamansız olarak yüklendiğinden ve mümkün olan en kısa sürede yürütüldüğünden bilinçli olun. İkinci kitaplığın birinci kitaplığa bağımlı olduğu yerde iki kitaplık varsa bu durumda ikinci kitaplık ilk kitaplıktan önce yüklenip çalıştırılırsa hata verebilir ve uygulama bozulabilir.

jQuery Nesnesi #

$() veya jQuery() kullanılarak jQuery her çağrıldığında, dahili olarak jQuerynin yeni bir örneğini oluşturur. Bu, yeni örneği gösteren kaynak kodu:

// Define a local copy of jQuery
jQuery = function( selector, context ) {

    // The jQuery object is actually just the init constructor 'enhanced'
    // Need init if jQuery is called (just allow error to be thrown if not included)
    return new jQuery.fn.init( selector, context );
}

Dahili olarak jQuery, prototipine “.fn” olarak atıfta bulunur ve burada bir jQuery nesnesini dahili olarak başlatmak için kullanılan stil, arayan tarafından açıkça “new” kullanılmadan bu prototipin gösterilmesine izin verir.

Bir örnek ayarlamaya ek olarak (’.each’, ‘children’, ‘filter’ vb. gibi jQuery API’si bu şekilde açığa çıkar), jQuery dahili olarak sonuçla eşleşmesi için dizi benzeri bir yapı da oluşturur. seçicinin (argüman olarak hiçbir şey, “tanımsız”, “boş” veya benzerinden başka bir şey iletilmesi şartıyla). Tek bir öğe olması durumunda, bu dizi benzeri yapı yalnızca o öğeyi tutacaktır.

Basit bir gösterim, kimliği olan bir öğe bulmak ve ardından temeldeki DOM öğesini döndürmek için jQuery nesnesine erişmek olacaktır (bu, birden çok öğe eşleştirildiğinde veya mevcut olduğunda da işe yarar).

var $div = $("#myDiv");//populate the jQuery object with the result of the id selector
var div = $div[0];//access array-like structure of jQuery object to get the DOM Element

Ad alanlı jQuery eklentileri yükleniyor #

Genellikle eklentileri yüklerken, her zaman sonra jQuery eklentisini eklediğinizden emin olun.

<script src="https://code.jquery.com/jquery-3.1.1.min.js"></script>
<script src="some-plugin.min.js"></script>

Eğer jQuery’nin birden fazla sürümünü gerekiyorsa, o zaman eklentiyi/eklentileri gerekli jQuery sürümünden sonra yüklediğinizden ve ardından [jQuery.noConflict(true)](http:/ /api.jquery.com/jQuery.noConflict/); sonra jQuery’nin sonraki sürümünü ve ilişkili eklenti(ler)i yükleyin:

<script src="https://code.jquery.com/jquery-1.7.0.min.js"></script>
<script src="plugin-needs-1.7.min.js"></script>
<script>
// save reference to jQuery v1.7.0
var $oldjq = jQuery.noConflict(true);
</script>
<script src="https://code.jquery.com/jquery-3.1.1.min.js"></script>
<script src="newer-plugin.min.js"></script>

Şimdi eklentileri başlatırken, ilişkili jQuery sürümünü kullanmanız gerekecek.

<script>
// newer jQuery document ready
jQuery(function($){
  // "$" refers to the newer version of jQuery
  // inside of this function

  // initialize newer plugin
  $('#new').newerPlugin();
});

// older jQuery document ready
$oldjq(function($){
  // "$" refers to the older version of jQuery
  // inside of this function

  // initialize plugin needing older jQuery
  $('#old').olderPlugin();
});
</script>

Her iki eklentiyi de başlatmak için yalnızca bir belgeye hazır işlevi kullanmak mümkündür, ancak belgeye hazır işlevi içindeki herhangi bir fazladan jQuery koduyla ilgili karışıklığı ve sorunları önlemek için referansları ayrı tutmak daha iyi olur.
