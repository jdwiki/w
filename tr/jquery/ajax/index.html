Sözdizimi #

  • var jqXHR = $.ajax( url [,ayarlar] )
  • var jqXHR = $.ajax( [ayarlar] )
  • jqXHR.done(işlev( veri, textStatus, jqXHR ) {});
  • jqXHR.fail(işlev( jqXHR, textStatus, errorThown ) {});
  • jqXHR.always(işlev( jqXHR ) {});

Parametreler #

parametre Ayrıntılar
url İsteğin gönderileceği URL’yi belirtir
ayarlar isteğin davranışını etkileyen çok sayıda değer içeren bir nesne
tip İstek için kullanılacak HTTP yöntemi
veri Talep tarafından gönderilecek veriler
başarı İstek başarılı olursa çağrılacak bir geri arama işlevi
hata Hatayı işlemek için bir geri arama
durumKodu Yanıt karşılık gelen koda sahip olduğunda çağrılacak sayısal HTTP kodları ve işlevleri nesnesi
dataType Sunucudan geri beklediğiniz veri türü
içerikTürü Sunucuya gönderilecek verilerin içerik türü. Varsayılan “application/x-www-form-urlencoded; charset=UTF-8” şeklindedir.
bağlam Geri aramalarda kullanılacak bağlamı belirtir, genellikle geçerli hedefi ifade eden “bu”.

AJAX, Asenkron JavaScript and XML anlamına gelir. AJAX, bir web sayfasının, tüm sayfayı yeniden yüklemeye gerek kalmadan sunucuya eşzamansız bir HTTP (AJAX) isteği gerçekleştirmesine ve bir yanıt almasına izin verir.

HTTP Yanıt Kodlarını $.ajax() ile İşleme #

İsteğin başarılı olup olmamasına bağlı olarak tetiklenen ‘.done’, ‘.fail’ ve ‘.always’ söz geri çağrılarına ek olarak, belirli bir [HTTP Durum Kodu] olduğunda bir işlevi tetikleme seçeneği vardır. 1] sunucudan döndürülür. Bu, “statusCode” parametresi kullanılarak yapılabilir.

$.ajax({
    type: {POST or GET or PUT etc.},
    url:  {server.url},
    data: {someData: true},
    statusCode: {
        404: function(responseObject, textStatus, jqXHR) {
            // No content found (404)
            // This code will be executed if the server returns a 404 response
        },
        503: function(responseObject, textStatus, errorThrown) {
            // Service Unavailable (503)
            // This code will be executed if the server returns a 503 response
        }           
    }
})
.done(function(data){
    alert(data);
})
.fail(function(jqXHR, textStatus){
    alert('Something went wrong: ' + textStatus);
})
.always(function(jqXHR, textStatus) {
   alert('Ajax request was finished')
});

İstek başarılı olursa durum kodu işlevleri aynı başarı geri araması olarak parametreler; bir hatayla sonuçlanırsa (3xx yönlendirme dahil), “hata” ile aynı parametreleri alırlar. geri arama.

Form Göndermek için Ajax Kullanma #

Bazen bir formunuz olabilir ve onu ajax kullanarak göndermek isteyebilirsiniz.

Diyelim ki bu basit forma sahipsiniz -

<form id="ajax_form" action="form_action.php">
  <label for="name">Name :</label>
  <input name="name" id="name" type="text" />
  <label for="name">Email :</label>
  <input name="email" id="email" type="text" />
  <input type="submit" value="Submit" />
</form>

Aşağıdaki jQuery kodu kullanılabilir (bir $(document).ready çağrısı içinde) -

$('#ajax_form').submit(function(event){
  event.preventDefault();
  var $form = $(this);

  $.ajax({
      type: 'POST',
      url: $form.attr('action'),
      data: $form.serialize(),
      success: function(data) {
        // Do something with the response
      },
      error: function(error) {
        // Do something with the error
      }
  });
});

Açıklama

  • var $form = $(this) - yeniden kullanım için önbelleğe alınan form
  • $('#ajax_form').submit(function(event){ - Kimliği olan form “ajax_form” gönderilir, bu işlevi çalıştırın ve olayı bir parametre.
  • event.preventDefault(); - Formun normal şekilde gönderilmesini engeller (Alternatif olarak, ajax({}); ifadesinden sonra aynı etkiye sahip olacak ``return false` kullanabiliriz)
  • url: $form.attr('action'), - Formun “action” özniteliğinin değerini alın ve “url” özelliği için kullanın.
  • data: $form.serialize(), - Form içindeki girdileri sunucuya göndermek için uygun bir dizgeye dönüştürür. Bu durumda “name=Bob&email=bob@bobsemailaddress.com” gibi bir şey döndürür.

Hepsi bir arada örnekler #

Ajax’ı Al:

Çözüm 1:

$.get('url.html', function(data){
    $('#update-box').html(data);
});

Çözüm 2:

 $.ajax({
     type: 'GET',
     url: 'url.php',  
 }).done(function(data){
     $('#update-box').html(data);
 }).fail(function(jqXHR, textStatus){
     alert('Error occured: ' + textStatus);
 });

Ajax Yükü: Basitlik için oluşturulmuş başka bir ajax alma yöntemi

$('#update-box').load('url.html');

.load ayrıca ek verilerle çağrılabilir. Veri bölümü, dize veya nesne olarak sağlanabilir.

$('#update-box').load('url.php', {data: "something"});
$('#update-box').load('url.php', "data=something");

.load bir geri arama yöntemiyle çağrılırsa, sunucuya yapılan istek bir gönderi olacaktır.

$('#update-box').load('url.php', {data: "something"}, function(resolve){
    //do something
});

Ajax Dosya Yüklemeleri #

1. Basit Bir Tam Örnek #

Bu örnek kodu, her yeni dosya seçimi yapıldığında kullanıcı tarafından seçilen dosyaları yüklemek için kullanabiliriz.

<input type="file" id="file-input" multiple>

 

var files;
var fdata = new FormData();
$("#file-input").on("change", function (e) {
    files = this.files;

    $.each(files, function (i, file) {
        fdata.append("file" + i, file);
    });

    fdata.append("FullName", "John Doe");
    fdata.append("Gender", "Male");
    fdata.append("Age", "24");

    $.ajax({
        url: "/Test/Url",
        type: "post",
        data: fdata, //add the FormData object to the data parameter
        processData: false, //tell jquery not to process data
        contentType: false, //tell jquery not to set content-type
        success: function (response, status, jqxhr) {
            //handle success
        },
        error: function (jqxhr, status, errorMessage) {
            //handle error
        }
    });
});

Şimdi bunu parçalayalım ve parça parça inceleyelim.

2. Dosya Girişleriyle Çalışma #

Bu MDN Belgesi ( Web uygulamalarından dosyaları kullanma ), dosya girişlerinin nasıl ele alınacağına dair çeşitli yöntemler hakkında iyi bir okumadır. Bu yöntemlerden bazıları bu örnekte de kullanılacaktır.

Dosyaları yüklemeye başlamadan önce, kullanıcıya yüklemek istediği dosyaları seçmesi için bir yol vermeliyiz. Bu amaçla bir `dosya girişi’ kullanacağız. ‘Multiple’ özelliği birden fazla dosyanın seçilmesine izin verir, kullanıcının bir seferde bir dosya seçmesini istiyorsanız bunu kaldırabilirsiniz.

<input type="file" id="file-input" multiple>

Dosyaları yakalamak için girdinin ‘change olayı’nı kullanacağız.

var files;
$("#file-input").on("change", function(e){
    files = this.files;
});

İşleyici işlevinin içinde, girdimizin files özelliği aracılığıyla dosyalara erişiriz. Bu bize bir dizi benzeri nesne olan bir FileList verir.

3. FormData’nın Oluşturulması ve Doldurulması #

Ajax ile dosya yüklemek için FormData kullanacağız.

var fdata = new FormData();

Önceki adımda elde ettiğimiz FileList nesne gibi bir dizidir ve for loop, for…of loop ve jQuery.each dahil olmak üzere çeşitli yöntemler kullanılarak yinelenebilir. . Bu örnekte jQuery’ye bağlı kalacağız.

$.each(files, function(i, file) {
  //...
});

Dosyaları formdata nesnemize eklemek için FormData’nın ekleme yöntemini kullanacağız.

$.each(files, function(i, file) {
  fdata.append("file" + i, file);
});

Aynı şekilde göndermek istediğimiz diğer verileri de ekleyebiliriz. Diyelim ki kullanıcıdan aldığımız bazı kişisel bilgileri dosyalarla birlikte göndermek istiyoruz. Bu bilgiyi formdata nesnemize ekleyebiliriz.

fdata.append("FullName", "John Doe");
fdata.append("Gender", "Male");
fdata.append("Age", "24");
//...

4. Dosyaları Ajax ile Gönderme #

$.ajax({
    url: "/Test/Url",
    type: "post",
    data: fdata, //add the FormData object to the data parameter
    processData: false, //tell jquery not to process data
    contentType: false, //tell jquery not to set content-type
    success: function (response, status, jqxhr) {
        //handle success
    },
    error: function (jqxhr, status, errorMessage) {
        //handle error
    }
});

“processData” ve “contentType” özelliklerini “false” olarak ayarladık. Bu, dosyaların sunucuya gönderilebilmesi ve sunucu tarafından doğru bir şekilde işlenebilmesi için yapılır.

JSON verilerini gönderme #

jQuery, jSON yanıtlarını işlemeyi zahmetsiz hale getirir, ancak belirli bir istek JSON biçiminde veri göndermenizi istediğinde biraz daha fazla çalışma gerekir:

 $.ajax("/json-consuming-route", {
      data: JSON.stringify({author: {name: "Bullwinkle J. Moose", 
                                     email: "bullwinkle@example.com"} }),
      method: "POST",
      contentType: "application/json"
   });

Bulduğumuz veriler için doğru “contentType"ı belirttiğimize dikkat edin. gönderme; bu genel olarak iyi bir uygulamadır ve gönderi yaptığınız API tarafından gerekli olabilir - ancak aynı zamanda jQuery’ye ‘%20’ ile ‘+’ arasındaki varsayılan dönüşümü gerçekleştirmemesi talimatını verme gibi bir yan etkisi vardır. “contentType”, “application/x-www-form-urlencoded” varsayılan değerinde bırakılsaydı yapardı. Herhangi bir nedenle contentType’ı varsayılan olarak bırakmanız gerekiyorsa, bunu önlemek için “processData” öğesini false olarak ayarladığınızdan emin olun.

[JSON.stringify](https://www.wikiod.com/tr/javascript/json#Bir değeri seri hale getirme) çağrısından burada kaçınılabilir, ancak verileri bir JavaScript nesnesi biçiminde sağlamamıza olanak tanır (böylece, özellik adlarının alıntılanmaması gibi utanç verici JSON sözdizimi hatalarından kaçınılır).
