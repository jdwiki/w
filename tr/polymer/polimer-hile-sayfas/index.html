Bu, Polymer 2.x kitaplığı için bir hile sayfasıdır. Monica Dinculescu’dan post çatalı.

Öğe tanımlama #

Dokümanlar: 1.x -> 2.x yükseltme kılavuzu, bir öğe kaydetme, paylaşılan stil modülleri.

<link rel="import" href="bower_components/polymer/polymer-element.html">
<dom-module id="element-name">
  <template>
    <!-- Use one of these style declarations, but not both -->
    <!-- Use this if you don’t want to include a shared style -->
    <style></style>
    <!-- Use this if you want to include a shared style -->
    <style include="some-style-module-name"></style>
  </template>
  <script>
    class MyElement extends Polymer.Element {
      static get is() { return 'element-name'; }
      // All of these are optional. Only keep the ones you need.
      static get properties() { ... }
      static get observers() { ... }
    }

    // Associate the new class with an element name
    customElements.define(MyElement.is, MyElement);
  </script>
</dom-module>

Belirli bir özel etiketin sınıf tanımını almak için şunu kullanabilirsiniz: customElements.get('eleman-adı').

Bir öğeyi genişletme #

Dokümanlar: genişleyen öğeler, [devralınan şablonlar](https://www.polymer-project .org/2.0/docs/devguide/dom-template#inherited-templates).

“Polymer.Element” yerine özel bir öğe, farklı bir öğeyi genişletebilir:

class ParentElement extends Polymer.Element {
  /* ... */
}
class ChildElement extends ParentElement {
  /* ... */
}

Ebeveynin şablonunu değiştirmek veya şablona eklemek için “şablon” alıcısını geçersiz kılın:

<dom-module id="child-element">
  <template>
    <style> /* ... */ </style>
    <span>bonus!</span>
   </template>
  <script>
    var childTemplate;
    var childTemplate = Polymer.DomModule.import('child-element', 'template');
    var parentTemplate = ParentElement.template.cloneNode(true);
    // Or however you want to assemble these.
    childTemplate.content.insertBefore(parentTemplate.firstChild, parentTemplate);

    class ChildElement extends ParentElement {
      static get is() { return 'child-element'; }
      // Note: the more work you do here, the slower your element is to
      // boot up. You should probably do the template assembling once, in a
      // static method outside your class (like above).
      static get template() {
        return childTemplate;
      }
    }
    customElements.define(ChildElement.is, ChildElement);
  </script>
</dom-module>

Ana sınıfı bilmiyorsanız, şunları da kullanabilirsiniz:

class ChildElement extends customElements.get('parent-element') {
  /* ... */
}

Bir karışım tanımlama #

Dokümanlar: mixins, karma öğeler.

Uygulamayı farklı öğeler arasında paylaşmak için bir sınıf ifadesi karışımı tanımlama:

<script>
  MyMixin = function(superClass) {
    return class extends superClass {
      // Code that you want common to elements.
      // If you're going to override a lifecycle method, remember that a) you
      // might need to call super but b) it might not exist.
      connectedCallback() {
        if (super.connectedCallback) {
          super.connectedCallback();
        }
        /* ... */
      }
    }
  }
</script>

Bir öğe tanımında karışımı kullanma:

<dom-module id="element-name">
  <template><!-- ... --></template>
  <script>
    // This could also be a sequence:
    // class MyElement extends AnotherMixin(MyMixin(Polymer.Element)) { … }
    class MyElement extends MyMixin(Polymer.Element) {
      static get is() { return 'element-name' }
      /* ... */
    }
    customElements.define(MyElement.is, MyElement);
  </script>
</dom-module>

Karma davranışları (1.x sözdiziminde tanımlanmıştır) mixins olarak kullanma:

<dom-module id="element-name">
  <template><!-- ... --></template>
  <script>
    class MyElement extends Polymer.mixinBehaviors([MyBehavior, MyBehavior2], Polymer.Element) {
     static get is() { return 'element-name' }
     /* ... */
    }
    customElements.define('element-name', MyElement);
  </script>
</dom-module>

Yaşam döngüsü yöntemleri #

Dokümanlar: yaşam döngüsü geri aramaları, hazır.

class MyElement extends Polymer.Element {
 constructor() { super(); /* ... */ }
 ready() { super.ready(); /* ... */ }
 connectedCallback() { super.connectedCallback(); /* ... */ }
 disconnectedCallback() { super.disconnectedCallback(); /* ... */ }
 attributeChangedCallback() { super.attributeChangedCallback(); /* ... */ }
}

Bağlanma verileri #

Dokümanlar: veri bağlama, öznitelik bağlama, dizi öğelerine bağlama, hesaplanmış bağlamalar.

Unutmayın: Polimer camel-cases özellikleri, yani JavaScript’te “myProperty” kullanıyorsanız, HTML’de “mülkümü” kullanırsınız.

Tek yönlü bağlama: “benim Mülküm” değiştiğinde, “mülkiyetleri” güncellenir:

<some-element their-property="[[myProperty]]"></some-element>

İki yönlü bağlama: “benim Mülküm” değiştiğinde, “mülkiyetleri” güncellenir, ve tersi:

<some-element their-property="{{myProperty}}"></some-element>

Özellik bağlama: “myProperty” “true” olduğunda, öğe gizlenir; olduğu zaman “false”, öğe görünür. Nitelik ve özellik arasındaki fark bağlama, özellik bağlamanın “someElement.someProp = value” ile eşdeğer olmasıdır, öznitelik bağlama şuna eşdeğerdir: someElement.setAttribute(someProp, value)

<some-element hidden$="[[myProperty]]"></some-element>

Bilgisayarlı bağlama: “sınıf” özniteliğine bağlanma, şu durumlarda stilleri yeniden derleyecektir: “myProperty” değişiklikleri:

<some-element class$="[[_computeSomething(myProperty)]]"></some-element>
<script>
_computeSomething: function(prop) {
  return prop ? 'a-class-name' : 'another-class-name';
}
</script>

Gözlemciler #

Dokümanlar: gözlemciler, çok özellikli gözlemciler, dizi mutasyonlarını gözlemleme, dinamik olarak gözlemci ekleme.

“Özellikler” bloğuna bir “gözlemci” eklemek, bir mülkün değeri:

static get properties() {
  return {
    myProperty: {
      observer: '_myPropertyChanged'
    }
  }
}

// The second argument is optional, and gives you the
// previous value of the property, before the update:
_myPropertyChanged(value, /*oldValue */) { /* ... */ }

“Gözlemciler” bloğunda:

static get observers() {
  return [
    '_doSomething(myProperty)',
    '_multiPropertyObserver(myProperty, anotherProperty)',
    '_observerForASubProperty(user.name)',
    // Below, items can be an array or an object:
    '_observerForABunchOfSubPaths(items.*)'
  ]
}

‘otherProperty’ özelliği için dinamik olarak bir gözlemci ekleme:

// Define a method.
_otherPropertyChanged(value) { /* ... */ }
// Call it when `otherPropety` changes.
this._createPropertyObserver('otherProperty', '_otherPropertyChanged', true);
