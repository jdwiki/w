.NET için Cloud Resource Manager API İstemcisi ile Yeni Proje

  • Herhangi bir Google hizmetinde herhangi bir yöntemi (!) çağırmak (!)

Bu örnek için .NET için Google API İstemci Kitaplıkları kullanacağız.

Başka kütüphaneler var. Ayrıntılar için lütfen Google Müşteri Kitaplıkları Açıklaması sayfasına bakın.

Proje Oluşturma ve Yönetme için Cloud Resource Manager API’sini kullanacağız oluşturma-yönetme-projeler).

Başlayalım.

Hepsini bir araya koy…

İki dosyanız olmalı. İlk dosya ya “packages.config” ya da “project.json” olarak adlandırılır.

İkinci dosyaya Program.cs adını verelim. Yukarıdaki bölümlerde yer alan tüm kodlar tek bir ana yönteme yapıştırılabilir:

using System;
using System.Collections.Generic;
using System.Threading.Tasks;

using Google.Apis.Auth.OAuth2;
using Google.Apis.CloudResourceManager.v1;
using Google.Apis.Services;

using Data = Google.Apis.CloudResourceManager.v1.Data;

namespace OurFirstProject
{
    public class Program
    {
        private const string projectId = [YOUR-PROJECT-ID];
        private const string applicationName = "Test";

        public static void Main(string[] args)
        {

            var scopes = new String[] {
                CloudResourceManagerService.Scope.CloudPlatform
            };

            GoogleCredential credential = Task.Run(
                () => GoogleCredential.GetApplicationDefaultAsync()
            ).Result;

            if (credential.IsCreateScopedRequired)
            {
                credential = credential.CreateScoped(scopes);
            }

            CloudResourceManagerService service = new CloudResourceManagerService(
                new BaseClientService.Initializer()
                {
                    HttpClientInitializer = credential,
                    ApplicationName = applicationName
                }
            );

            Console.WriteLine("1. Create Project");
            Data.Operation operation1 = service.Projects.Create(
                new Data.Project()
                {
                    ProjectId = projectId,
                }
            ).Execute();

            Console.Write("2. Awaiting Operation Completion");
            Data.Operation operation2;
            do
            {
                operation2 = service.Operations.Get(operation1.Name).Execute();
                Console.WriteLine(operation2.Done.ToString());
                System.Threading.Thread.Sleep(1000);
            } while (operation2.Done != true);

            Console.WriteLine();
            Console.WriteLine("Enter to continue");
            Console.ReadLine();

            Console.WriteLine("3. Deleting Project");
            var operation3 = service.Projects.Delete(projectId).Execute();
        }
    }
}

“Bekliyor” adımı, “Doğru” ile biten (umarım) boş satırlar içerecektir.

Başlarken #

Kod Windows ve Linux üzerinde çalışacaktır.

.NET Core veya .NET Standard üzerinde çalışır. Tek fark, .NET Core üzerinde çalışmak istiyorsanız, project.json kullanmanız gerektiğidir. .NET Standard üzerinde çalıştırmak istiyorsanız, packages.config kullanmalısınız.

Cloud Resource Manager API için API İstemci Kitaplığı, nuget’te mevcuttur:

https://www.nuget.org/packages/Google.Apis.CloudResourceManager.v1/

Yazıldığı gibi sürüm: 1.22.0.809

Bunu 2 şekilde yapacağız:

  • .NET Standard ile Windows; ve
  • .NET Core ile Linux.

Pencereler

Visual Studio kullanıyorsanız, yeni bir “Visual C#” “Konsol Uygulaması” oluşturun. Aksi takdirde, proje için bir dizin oluşturun ve içinde packages.config adında bir dosya oluşturun. package.config, .NET Standard içindir, ancak Windows ile .NET Standard kullanıyoruz. .NET Standard’ı yalnızca Windows’ta çalıştırabilirsiniz. “packages.config” içeriğini şununla değiştirin:

﻿<?xml version="1.0" encoding="utf-8"?>
<packages>
  <package id="Google.Apis" version="1.22.0"
    targetFramework="net452" />
  <package id="Google.Apis.Auth" version="1.22.0"
    targetFramework="net452" />
  <package id="Google.Apis.CloudResourceManager.v1" version="1.22.0.809" 
    targetFramework="net452" />
  <package id="Google.Apis.Core" version="1.22.0"
    targetFramework="net452" />
</packages>

Linux

Proje için bir dizin oluşturun ve içinde project.json adında bir dosya oluşturun. “project.json”, .NET Core içindir, ancak bu örnekte Linux ile .NET Core kullanıyoruz. .NET Core’u Linux veya Windows üzerinde çalıştırabilirsiniz. ‘project.json’ içeriğini şununla değiştirin:

{
  "version": "1.0.0-*",
  "buildOptions": {
    "debugType": "portable",
    "emitEntryPoint": true
  },
  "dependencies": {},
  "frameworks": {
    "netcoreapp1.1": {
      "dependencies": {
        "Microsoft.NETCore.App": {
          "type": "platform",
          "version": "1.1.0"
        },
        "Google.Apis.CloudResourceManager.v1":"1.22.0.809"
      },
      "imports": "dnxcore50"
    }
  }
}

Uygulama Varsayılan Kimlik Bilgileri #

Hepsini burada tekrar etmeyeceğim: “Uygulama Varsayılan Kimlik Bilgileri, Google API’lerini çağırmak için yetkilendirme kimlik bilgilerini almanın basit bir yolunu sağlar. "

Uygulama Varsayılan Kimlik Bilgilerini kullanabiliyorsanız, yapın.

Makinenizden API’leri çağırırken, Uygulama Varsayılan Kimlik Bilgilerini kimliğiniz olarak kullanmadan önce gerçekleştirmeniz gereken ek bir adım vardır:

gcloud auth application-default login [yourname@gmail.com]

Bu nedenle Uygulama Varsayılan Kimlik Bilgilerini kullanmayı tercih edeceksiniz:

var scopes = new String[] {
    CloudResourceManagerService.Scope.CloudPlatform
};

GoogleCredential credential = Task.Run(
    () => GoogleCredential.GetApplicationDefaultAsync()
).Result;

if (credential.IsCreateScopedRequired)
{
    credential = credential.CreateScoped(scopes);
}

…Google Cloud API’ye (herhangi bir) yapılan çağrıları yetkilendirmek için ihtiyacınız olan kod bu kadar!

Google hizmetine karşı arama yapmak için bir sonraki adımda credential nesnesini kullanacağız…

Herhangi bir Google hizmetinde herhangi bir yöntemi (!) çağırmak (!) #

Bir Google hizmetinde bir yöntemi çağırma kodunu öğrendikten sonra, _herhangi bir Google hizmetinde any yöntemini nasıl çağıracağınızı çıkarabileceksiniz.

İlk olarak, önceki örnekte somutlaştırılan ‘credential’ nesnesini kullanarak hizmetle bağlantı kurarız:

CloudResourceManagerService service = new CloudResourceManagerService(
    new BaseClientService.Initializer()
    {
        HttpClientInitializer = credential,
        ApplicationName = "Our First Google API Client"
    }
);

Daha sonra servis tarafından sağlanan metotları çağırabiliriz. Hangi yöntemler mevcuttur?

https://cloud.google.com/resource-manager/docs/apis

“Projects.Create” için temel alınan REST isteği nedir?

https://cloud.google.com/resource-manager/reference/rest/v1/projects/create

Tamam… Kodu yazalım.

Kod, “projectId” için bir dize değeri bekliyor. Proje kimlikleri benzersiz tanımlayıcılardır. Projelerinizi tanımlamanıza yardımcı olacak bir isimlendirme sistemi kullanmanızı tavsiye ederim.

“Projects.Create”, bir “Data.Project” nesnesi bekliyor. Bu nesne bir zorunlu özelliktir, sağlayacağımız tek şey Proje Kimliğidir, ancak aynı zamanda bir Proje Adı, Etiketleri, Projenin ana bileşeninin ayrıntılarını vb. sağlayabiliriz.

Data.Operation operation1 = service.Projects.Create(
    new Data.Project()
    {
        ProjectId = projectId,
    }
).Execute();

Proje oluşturma eşzamansız olarak işlenir. Projenin ne zaman oluşturulacağını belirlemek için sorgulamamız gereken bir ‘Operation’ nesnesi verildi. İşlemler, işlemi benzersiz olarak tanımlayan bir Ad özelliğine sahiptir. Kodun bir sonraki bölümü, “Henüz bitirmedik mi?” platformunu yoklar. Proje, yeni işlemimiz “True” olan bir “Done” özelliği içerdiğinde oluşturulacaktır.

Data.Operation operation2;
do
{
    operation2 = service.Operations.Get(operation1.Name).Execute();
    System.Threading.Thread.Sleep(1000);
} while (operation2.Done != true);

Tamamlanması için ve umarım bundan sonra uzun yıllar projenizi çok mutlu kullandıktan sonra projenizi silmeniz gerekebilir. Biz sadece Projects.Delete’i ararız ve Proje Kimliğimizi sağlarız. Bu aynı zamanda bir işlem döndürür ve kesin olarak tamamlanana kadar bu işlemi de yoklamamız gerekir. Ardından projemiz silinecektir.

var operation3 = service.Projects.Delete(projectId).Execute();

Bu kadar!
