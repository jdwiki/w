Python için Cloud Resource Manager API İstemcisi ile Yeni Proje

Bu örnek için Python için Google API İstemci Kitaplıkları kullanacağız.

Başka kütüphaneler var. Ayrıntılar için lütfen Google İstemci Kitaplıkları Açıklaması sayfasına bakın.

Proje Oluşturma ve Yönetme için Cloud Resource Manager API’sini kullanacağız oluşturma-yönetme-projeler).

Başlayalım.

Hepsini bir araya koy…

Yukarıdaki örnekleri izlediyseniz, ‘my_project_folder’ adlı bir dizinde olmalısınız ve umarım ‘venv’ adlı bir alt dizin içerir.

Virtualenv’inizin etkinleştirildiğinden emin olun.

Tüm kod tek bir dosyaya yerleştirilebilir, buna create_project.py diyelim.

“my_project_folder” içinde “create_project.py” dosyasını oluşturun.

import json
import time

from apiclient.discovery import build
from oauth2client.client import GoogleCredentials

SERVICE_NAME = "cloudresourcemanager"
SERVICE_VERSION = "v1"

# Don't forget to replace YOUR-PROJECT-ID with your choice of Project ID
# Even though the code deletes the Project, change this value each time
PROJECT_ID = "YOUR-PROJECT-ID"

credentials = GoogleCredentials.get_application_default()

service = build(
    SERVICE_NAME,
    SERVICE_VERSION,
    credentials=credentials)

operation1 = service.projects().create(
    body={
        "project_id": PROJECT_ID
    }
).execute()

print(json.dumps(
    operation1,
    sort_keys=True,
    indent=3))

name = operation1["name"]
while True:
    operation2 = service.operations().get(
        name=name
    ).execute()
    print(json.dumps(
        operation2,
        sort_keys=True,
        indent=3))
    if "done" in operation2:
        if (operation2["done"]):
            break
    time.sleep(1)

raw_input("Press Enter to delete the Project...")

operation3 = service.projects().delete(
    projectId=PROJECT_ID
).execute()

Dosyayı kaydedin ve komut istemi türünden:

python create_project.py

Başlarken #

Python’u çalıştırabilmeniz gerekecek.

Python, Linux, Mac OS X ve Windows için kullanılabilir.

pip ve virtualenv öneririm.

Pip, Python paketlerini kurmak için önerilen araçtır.

Google Cloud API Kitaplıkları, pip paketleri olarak mevcuttur.

Sanal Ortam (aka “sanalenv”) “farklı projelerin gerektirdiği bağımlılıkları ayrı yerlerde tutmak için bir araçtır”.

Projeniz için bir dizin oluşturun ve ardından:

cd my_project_folder
virtualenv venv

Virtualenv klasörünüze “venv” dışında bir ad verebilirsiniz, ancak bu ad, dizinin amacına ilişkin yararlı bir hatırlatıcıdır. virtualenv şuna benzer bir şey göstermelidir:

New python executable in venv/bin/python
Installing setuptools, pip, wheel...done.

Virtualenv oluşturulur, ancak onu kullanmak için onu ’etkinleştirmemiz’ gerekir. Onu kullanmayı bitirdiğimizde, onu “devre dışı bırakmak” da (zorunlu olmasa da) iyi bir uygulamadır.

source venv/bin/activate

Venv adlı sanal ortamda olduğumuzu belirtmek için Linux makinemdeki komut istemi değişiyor. Proje dizininizde kalacaksınız. Kilometreniz değişebilir, ancak:

(venv) user@host

Şimdi pip’i yükseltelim ve Google API İstemci Kitaplıklarını yükleyelim

pip install --upgrade pip
pip install --upgrade google-api-python-client

Her şey yolundaysa, “pip frost” yazarsanız, bu listeye benzer bir şey görmelisiniz. Sürümleriniz daha yüksek olabilir:

pip freeze
google-api-python-client==1.6.2
httplib2==0.10.3
oauth2client==4.0.0
pyasn1==0.2.3
pyasn1-modules==0.0.8
rsa==3.4.2
six==1.10.0
uritemplate==3.0.0

Python ortamınız artık hazır, Google API İstemci Kitaplıkları yüklendi. Python kodumuzu yazabiliriz. Lütfen bir sonraki örneğe devam edin.

Projeyi bitirdiğinizde, sanal ortamı ‘devre dışı bırakmak’ iyi bir uygulamadır. Lütfen bunu şimdi yapmayın, çünkü biraz kod yazacağız. Ancak işiniz bittiğinde lütfen buraya dönün ve:

deactivate

Virtualenv’e geri dönmek istiyorsanız, ‘activate’ komutunu yeniden çalıştırmanız yeterlidir.

Uygulama Varsayılan Kimlik Bilgileri #

Hepsini burada tekrar etmeyeceğim: “Uygulama Varsayılan Kimlik Bilgileri, Google API’lerini çağırmak için yetkilendirme kimlik bilgilerini almanın basit bir yolunu sağlar. "

Uygulama Varsayılan Kimlik Bilgilerini kullanabiliyorsanız, yapın.

Makinenizden API’leri çağırırken, Uygulama Varsayılan Kimlik Bilgilerini kimliğiniz olarak kullanmadan önce gerçekleştirmeniz gereken ek bir adım vardır:

gcloud auth application-default login [yourname@gmail.com]

Bu nedenle Uygulama Varsayılan Kimlik Bilgilerini kullanmayı tercih edeceksiniz:

scopes = [
    "https://www.googleapis.com/auth/cloud-platform"
]
credentials = GoogleCredentials.get_application_default()
if credentials.create_scoped_required():
    credentials = credentials.create_scoped(scopes)

Gerçekte, genellikle sadece şunlardan kurtulabilirsiniz:

credentials = GoogleCredentials.get_application_default()

…Google Cloud API’ye (herhangi bir) yapılan çağrıları yetkilendirmek için ihtiyacınız olan kod bu kadar!

Google hizmetine karşı arama yapmak için bir sonraki adımda credential nesnesini kullanacağız…

Herhangi bir Google hizmetinde herhangi bir yöntemi (!) çağırmak (!) #

Bir Google hizmetinde bir yöntemi çağırma kodunu öğrendikten sonra, herhangi bir Google hizmetinde herhangi bir yöntemi nasıl çağıracağınızı çıkarabileceksiniz.

İlk olarak, önceki örnekte somutlaştırılan ‘credential’ nesnesini kullanarak hizmetle bağlantı kurarız:

service = build(
    SERVICE_NAME,
    SERVICE_VERSION,
    credentials=credentials)

Daha sonra servisin sağladığı metotları çağırabiliriz. Hangi yöntemler mevcuttur?

https://cloud.google.com/resource-manager/docs/apis

Projects.Create için temel alınan REST isteği nedir?

https://cloud.google.com/resource-manager/reference/rest/v1/projects/create

Tamam… Kodu yazalım.

‘create’ yöntemi, minimum olarak Proje Kimliğini içeren bir gövde bekler. Proje kimlikleri benzersiz tanımlayıcılardır. Projelerinizi tanımlamanıza yardımcı olacak bir isimlendirme sistemi kullanmanızı tavsiye ederim. Yöntem ayrıca bir Proje Adını, Etiketleri, Projenin ebeveynlerinin ayrıntılarını vb. kabul eder.

operation1 = service.projects().create(
  body={
    "project_id": PROJECT_ID
  }
).execute()

Proje oluşturma eşzamansız olarak işlenir. Projenin ne zaman oluşturulacağını belirlemek için sorgulamamız gereken bir Operation nesnesi verildi. İşlemler, işlemi benzersiz olarak tanımlayan bir Ad özelliğine sahiptir. Kodun bir sonraki bölümü, “Henüz bitirmedik mi?” platformunu yoklar. Proje, yeni işlemimiz “True” olan bir “Done” özelliği içerdiğinde oluşturulacaktır.

name = operation1["name"]
while True:
    operation2 = service.operations().get(
        name=name
    ).execute()
    if "done" in operation2:
        if (operation2["done"]):
            break
    time.sleep(1)

Tamamlanması için ve umarım bundan sonra uzun yıllar projenizi çok mutlu kullandıktan sonra projenizi silmeniz gerekebilir. Sadece silme yöntemini çağırır ve Proje Kimliğimizi sağlarız. Bu aynı zamanda bir işlem döndürür ama tamamlanana kadar işlemi yoklamayı size bırakacağım

operation3 = service.projects().delete(
    projectId=PROJECT_ID
).execute()

Bu kadar!
