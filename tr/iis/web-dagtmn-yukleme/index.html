Web Deploy’u yüklemek, Visual Studio kullanılarak doğrudan geliştirme ortamından web uygulamalarının hızlı bir şekilde yüklenmesine olanak tanır.

Kurulum türleri:

çevrimiçi - hedef Windows sunucusunun İnternet erişimi var - kurulum, aşağıdaki İnternet bağlantısına sahip sunucularda Web dağıtımını kurma ve yapılandırmayı kullanarak basittir.

çevrimdışı - hedef Windows sunucusu çevrimdışı - birkaç bileşen ve yapılandırmanın manuel olarak yapılması (veya komut dosyasıyla) yapılması gerektiğinden kurulum oldukça karmaşıktır

WebDeploy’u ASP.NET ile IIS’ye Yükleme #

  1. Web Sunucusu Rolünü Kurun
  2. WebDeploy 3.6’yı MSDN‘den yükleyin
  3. Web Sunucusu (IIS) > Web Sunucusu > Uygulama Geliştirme altında ASP.NET 4.6‘yı etkinleştirin

İnternet bağlantısı olan sunucularda Web dağıtımını yükleme ve yapılandırma #

Web projesi değişikliklerini bir Web Sunucusuna doğrudan dağıtabilmek için aşağıdaki adımlar izlenmelidir. Hedef sunucunun İnternet erişimi varsa, Microsoft’un ihtiyaç duyulan hemen hemen her şeyi yapmak için bir Web Platformu paketi olduğundan işlem oldukça basittir.

  1. Temizlik

C:\Program Files\IIS öğesinin Microsoft Web Deploy’un eski sürümünü içermediğinden emin olun. İçeriyorsa, (onları) Program Ekle/Kaldır’dan kaldırın ve kalan dosyaları kaldırın

  1. Web platformu yükleyici

Web platformu yükleyicisini (WPI) buradan yükleyin

  1. Web dağıtım kurulumu

WPI’yi çalıştırın, Barındırma Sunucuları için Web Deploy 3.6’yı arayın ve içerdiği her şeyi yükleyin. Gerekli tüm bağımlılıkları (örneğin SQL dağıtımı için) kurmaya ve ayrıca IIS’yi Web dağıtımları için yapılandırılmaya hazır hale getirmeye özen gösterecektir.

  1. IIS yapılandırması

IIS, bir veya daha fazla Web sitesinde web dağıtımlarına izin vermek için yapılandırılmalıdır.

  • Haklar: web sitesine -> IIS yönetici izinlerine erişin ve hangi kullanıcıların yayınlamasına izin verileceğini yapılandırın. NOT: belirli bir yapılandırmada, bir kullanıcı seçmeye çalışmak IIS yöneticisinin çökmesine neden olabilir. Bu nadir durumlarda, kullanıcı adını manuel olarak girmeniz yeterlidir - ör. etki alanı\kullanıcı adı

  • Web dağıtımı Yayınlama yapılandırması - Web sitesine sağ tıklayın -> Dağıt… -> Web Dağıtımı Yayınlamasını Yapılandır

NOT: Yayının çalışması için seçilen bağlantı noktasının (varsayılan 8172’dir) açılması gerekir (bazı şirketlerde bloke olabilir).

İnternet bağlantısı olmayan sunucularda Web dağıtımını yükleme ve yapılandırma #

İnternet bağlantısı olmadan (çevrimdışı) Web dağıtımının tam yapılandırmasını yapmak daha zordur çünkü Web platformu yükleyicisi (UI) bir İnternet konumundan paket listesi ve içeriği sorgulayarak çalışır.

  1. Temizlik C:\Program Files\IIS’nin Microsoft Web Deploy’un eski sürümünü içermediğinden emin olun. İçeriyorsa, (onları) Program Ekle/Kaldır’dan kaldırın ve kalan dosyaları kaldırın

  2. Web Platform Installer v5 Komut Satırı (WebPICMD.exe), “çevrimiçi” gereksinim olmadan paketleri dışa aktarmaya ve yüklemeye olanak tanır. Tam kurulum ve kullanım talimatları burada bulunabilir.

  3. Web Dağıtımı için gerekli paketler

  • Web Deploy’u bir “çevrimiçi” makineye kurun

  • WPI Komut Satırını kullanarak bunun için paketleri dışa aktarın

    C:\Program Files\Microsoft\Web Platform Installer>WebPICMD.exe /Offline /Product s:“WDeployPS” /Path:C:\OfflineCache

Dosyaları ve Klasörleri Web Dağıtımından Hariç Tutma #

Tipik olarak, dağıtım gerçekleştirilirken Web Uygulamasındaki bazı dosyaların üzerine yazılmamalıdır (ör. “web.config”). Bu şu şekilde gerçekleştirilebilir:

1) Çıktıdan hariç tutma - bu, Oluşturma eylemi’nin Yok olarak ayarlanması anlamına gelir. Bu en kolay yoldur, ancak uygulamanın yerel olarak çalışması için çıktıda olması gereken bazı belirli dosya veya klasörler için çalışmayabilir.

2) Dosyaları ve Klasörleri Web Paketinden Çıkarma Web uygulaması kök klasöründe özel bir xml dosyası oluşturarak. Örneğin.:

File name = [project name].wpp.targets
File content = 
    
<Project ToolsVersion="4.0" 
         xmlns="http://schemas.microsoft.com/developer/msbuild/2003">
  <ItemGroup>   
    <!-- this will exclude all files from Internal folder -->
    <ExcludeFromPackageFolders Include="Internal">
      <FromTarget>ContactManager.Mvc.wpp.targets</FromTarget>
    </ExcludeFromPackageFolders>

    <!-- this will exclude specified files -->
    <ExcludeFromPackageFiles Include="Scripts\jquery-1.4.4-
vsdoc.js;Scripts\jquery-1.4.4.js;Scripts\jquery-ui.js;Scripts\jquery.unobtrusive-ajax.js;Scripts\jquery.validate-vsdoc.js;Scripts\jquery.validate.js;Scripts\jquery.validate.unobtrusive.js;Scripts\MicrosoftAjax.debug.js;Scripts\MicrosoftMvcValidation.debug.js">
      <FromTarget>ContactManager.Mvc.wpp.targets</FromTarget>
    </ExcludeFromPackageFiles>
  </ItemGroup>
</Project>

Daha fazla ayrıntı burada bulunabilir.

Web dağıtımı otomatik yedeklemeler #

Web dağıtımı, dağıtım sırasında hedef Web sitesini (hedef Web uygulamasını değil!) otomatik olarak yedekleme seçeneği sunar. Bu, web uygulamasının geri alınmasına izin vermek için önerilir.

Otomatik yedeklemeleri yapılandırmak için aşağıdaki adımlar izlenmelidir:

1) Yedeklemeleri etkinleştir

Powershell’de %programfiles%\IIS\Microsoft Web Deploy V3\scripts\BackupScripts.ps1 dosyasını açın

Aşağıdaki komutları yürütün:

# Turns on all backup functionality
TurnOn-Backups -On $true

# Turns off all backup functionality
TurnOn-Backups -On $false

# Changes default global backup behavior to enabled
Configure-Backups -Enabled $true

# Changes default backup behavior for site "foo" to enabled
Configure-Backups -SiteName "foo" -Enabled $true

# Changes the path of where backups are stored to a sibling directory named "siteName_snapshots".  
# For more information about path variables, see the "backupPath" attribute in the section 
# "Configuring  Backup Settings on the Server for Global usage manually in IIS Config"
Configure-Backups -BackupPath "{SitePathParent}\{siteName}_snapshots"

# Configures default backup limit to 5 backups
Configure-Backups -NumberOfBackups 5

# Configures sync behavior to fail if a sync fails for any reason
Configure-Backups -ContinueSyncOnBackupFailure $false

# Adds providers to skip when performing a backup
Configure-Backups -AddExcludedProviders @("dbmysql","dbfullsql")

# Allows a site administrator to enable backups and set the number of backups at the site level
Configure-BackupSettingsProvider -CanSetEnabled $true -CanSetNumBackups $true

# Allows a site administrator to control which providers they want to skip in a backup, as 
# well as whether they can continue a sync after a backup failure
Configure-BackupSettingsProvider -CanSetContinueSyncOnBackupFailure $true -CanAddExcludedProviders $true

2) Global veya site düzeyinde yedekleme ayarlarını kontrol edin

Get-BackupSettings
Get-BackupSettings -SiteName "Default Web Site"

3) Daha fazla yedekleme özelleştirmesi

Yedekleme ayarları her web sitesi için yapılandırılabilir. applicationHost.config dosyasını açın ve belirli konumu için yedekleme ayarlarını ekleyin:

<location path="siteName">
    <system.webServer>
        <wdeploy>
            <backup enabled="true" numberOfBackups="4">
                <excludedProviders>
                    <clear />
                    <provider name="dbfullsql" />
                </excludedProviders>
            </backup>
        </wdeploy>
    </system.webServer>
</location>

Güvenlikle ilgili bilgiler ve komut satırı kullanımıyla ilgili diğer bilgiler için bu makaleye erişin.
