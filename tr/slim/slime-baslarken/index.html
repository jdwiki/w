Kurulum veya Kurulum #

Slim çerçevesini kurmanın önerilen yolu, besteci kullanmaktır.

  1. Boş bir dizin oluşturun

Bu dizin, Slim uygulamamızın çalışması için gerekli tüm dosyaları içerecektir. Bu dizine root dizini diyoruz, böylece diğer tüm uygulama dosyalarını ve dizinlerini kök dizine göre adresleyebiliriz.

mkdir slim-app

  1. Besteci kullanarak Slim çerçevesini ve gereksinimlerini yükleyin

    cd slim-app composer require slim/slim “^3.0”

Şu andan itibaren, bunun bizim çalışma dizinimiz olduğunu varsayıyoruz.

Besteci gerekli dosyaları indirmeyi bitirdikten sonra, iki dosya composer.json ve composer.lock ve besteci tarafından indirilen dosyaları içeren vendor adlı bir dizin olmalıdır. Artık uygulamamızı oluşturmaya hazırız. Uygulamamızı düzenlemek için başka bir dizin oluşturuyoruz:

mkdir public

Buna public dizini diyoruz ve web sunucumuza uygulamamızı bu dizinden sunucuya göndermesini söyleyeceğiz.

  1. Örnek “Merhaba Dünya” uygulaması

Slim’i kullanmak için genel dizinde aşağıdaki kodla bir index.php oluşturun:

genel/index.php

<?php

include "../vendor/autoload.php";

$app = new \Slim\App();

$app->get('/', function ($request, $response, $args) {
    $response->getBody()->write("Hello world!");
});

$app->run();
  1. Sunucuda yerleşik PHP’yi başlatın

Artık uygulamamıza hizmet etmek için sunucuda yerleşik PHP kullanabiliriz:

php -S localhost:8080 -t public

ve bu adresi bir web tarayıcısında açarak projeyi çalıştırın:

http://localhost:8080

Çıktı

Merhaba dünya!

Şimdi tüm istekler bu dosyaya iletilecek şekilde web sunucusunu yapılandırın:

Temiz URL’ler için Apache yapılandırması (İsteğe bağlı)

Bu gerekli değildir, ancak API URL’sindeki index.php’yi kaldırmak için ince projeler için önerilir.

index.php dosyanızın bulunduğu klasörde .htaccess oluşturun. Dosya aşağıdaki kodu içermelidir:

RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^ index.php [QSA,L]

Apache sanal ana makinenizin “AllowOverride” seçeneğiyle yapılandırıldığından emin olun, böylece “.htaccess” tarafından bildirilen yeniden yazma kuralları gerçekten kullanılabilir:

AllowOverride All

İnce Çerçeveye Başlarken #

Kurulum veya Kurulum İnce çerçeve

  • Besteci Yükle
  • cmd’yi aç
  • Proje Klasörünüzün Kök dizinine gidin ve Aşağıdaki Komutu Çalıştırın.

besteci ince/ince “^3.0” gerektirir

Artık projenizde satıcı dizininiz olacak

Sonraki Kök klasörde Index.php oluşturun ve aşağıdaki kodu ekleyin

<?php
use \Psr\Http\Message\ServerRequestInterface as Request;
use \Psr\Http\Message\ResponseInterface as Response;

require 'vendor/autoload.php';

$app = new \Slim\App;
$app->get('/hello/{name}', function (Request $request, Response $response) {
    $name = $request->getAttribute('name');
    $response->getBody()->write("Hello, $name");

    return $response;
});
$app->run();

Ardından Project’i Localhost’ta çalıştırın ve aşağıdaki komutu deneyin

http://localhost/project-root/index.php/hello/any-thing

Veritabanı tablosundan Json Verilerini Alın (REST API) #

use \Psr\Http\Message\ServerRequestInterface as Request;
use \Psr\Http\Message\ResponseInterface as Response;

require 'vendor/autoload.php';

$app = new \Slim\App;

$app->get('/employee/view', function ($req, $res) {

    $con = new mysqli('localhost','USERNAME','PASSWORD','DATABASE');

    $query = $con->query("SELECT * FROM employee"); 
    while ($row = $query->fetch_assoc()) {
        $data[] = $row;
    }
    return $res->withJson($data); 
});

$app->run();

include "vendor/autoload.php";

‘besteci’ ve @akrabats ‘Slim-Skeleton’ ile iskele iskeleti projesi #

  1. composer‘i global olarak mı yoksa yerel olarak mı kurduğunuza bağlı olarak.

    • Locally: php composer.phar create-project slim/slim-skeleton {my-app-name}
    • Globally: composer create-project slim/slim-skeleton {my-app-name}

  2. Bir web sunucusu çalıştırıyorsanız (örn. Apache veya nginx), sanal ana makine belgenizin kökünü yeni uygulamalar ortak klasörünüze yönlendirin. “{uygulama-adım}/genel”.

  3. “logs/” öğesinin web sunucusu kurulumunuz tarafından yazılabilir olduğundan emin olun.

Ayrıca, tüm shebang’ı şu şekilde dev modunda çalıştırabilirsiniz:

  • Yerel olarak: php composer.phar start
  • Küresel olarak: “besteci başlangıcı”
