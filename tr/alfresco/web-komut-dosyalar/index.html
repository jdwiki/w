Web betikleri, Alfresco’da sadece bazı bilgileri gösterebilen veya ayrıca Alfresco içinde bazı şeyler yapabilen (örneğin, iş akışını çalıştırma, dosyalar, kullanıcılar, gruplar veya diğer varlıklarla çalışma) yapabilen işlevsel modüllerdir.

Her webscript’in iki ana bölümü vardır: kod (.js, .java) ve Freemaker şablonu (.ftl)

Webscript, içinde kullanılan metin dizeleriyle birlikte ek .properties dosyasına da sahip olabilir. Parçalar, context.xml dosyasında eşleştirilir (Spring çerçeve mantığı).

Bu dosyada açıklanan herhangi bir URL’ye bağlanır.

Merhaba Dünya Web Komut Dosyası #

Merhaba dünya web komut dosyası yapalım. Web komut dosyalarının bir tanımlayıcısı, denetleyicisi ve isteğe bağlı olarak bir görünümü vardır. Bu dosyalar bir adlandırma kuralına uymalıdır.

Bu tanımlayıcı merhabaworld.get.desc.xml olarak adlandırılır.

<webscript>
  <shortname>Hello World</shortname>
  <description>Hello world web script</description>
  <url>/example/helloworld?name={nameArgument}</url>
  <authentication>user</authentication>  
</webscript>

Tanımlayıcının, bu web komut dosyasının bir URL, “/example/helloworld” ile eşleneceğini ve kullanıcı kimlik doğrulaması gerektirdiğini bildirdiğini görebilirsiniz. Tanımlayıcı ayrıca ad adında bir bağımsız değişken bildirir.

İşte kontrolör. Helloworld.get.js olarak adlandırılır.

model.foo = "bar";

Bu denetleyici JavaScript’te yazılmıştır, ancak denetleyiciler Java’da da yazılabilir. Biraz daha fazla çalışma ile diğer dillerde de kontrolörler yazabilirsiniz.

Bu kontrolör pek bir şey yapmaz. Sadece “foo” adlı modele yeni bir değişken ekler ve ona “bar” değerini verir.

Denetleyicinizin, tümü resmi belgelerde belgelenen çeşitli kök kapsamlı değişkenlere erişimi vardır.

Son olarak manzaraya bakalım. Helloworld.get.html.ftl olarak adlandırılmıştır.

<html>
  <body>
    <p>Hello, ${args.name!"name not specified"}!</p>
    <p>Foo: ${foo}</p>
  </body>
</html>

Adından bu görünümün bir Freemarker şablonu olarak uygulandığını ve HTML çıktısını aldığını görebilirsiniz. Bu görünüm, modelden “foo” değerini alır ve ayrıca web komut dosyasına geçirilen ad argümanını da alır. Bir ad bağımsız değişkeni belirtilmemişse, şablon bazı varsayılan metinler sağlar.

Bunun yerine XML veya JSON üretmek istiyorsanız, sadece adı değiştirin, ardından şablon uygulamanızı buna göre güncelleyin.

Dağıtım

Web komut dosyaları sınıf yoluna konuşlandırılabilir veya depoya yüklenebilir. Örneğin, bu web komut dosyasını depoya yükleyerek dağıtmak için şu adımları izleyin:

  1. Bu üç dosyayı Veri Sözlüğü/Web Komut Dosyası Uzantılarına yükleyin
  2. http://localhost:8080/alfresco/s/index adresine gidip “Web Komut Dosyalarını Yenile"yi tıklayarak Web Komut Dosyalarını yenileyin.
  3. http://localhost:8080/alfresco/s/example/helloworld?name=Jeff adresine giderek web komut dosyasına gidin

Klasör Oluşturucu: POST’u işleyen bir Web Komut Dosyası #

Hello World Web Komut Dosyası, GET HTTP yöntemlerini işler. Peki ya sunucuda veri oluşturmak istiyorsanız? Bunun için web komut dosyanız POST’u işlemelidir.

İşte Company Home’da yeni klasörler oluşturan basit bir örnek. Şuna benzeyen bir JSON gövdesiyle bir POST çağrısı yaparak çağrılır:

{'name':'testfolder'}

İsteğe bağlı olarak, bunları gövdenin bir parçası olarak ileterek klasöre bir başlık veya açıklama ekleyebilirsiniz, örneğin:

{
  'name':'testfolder',
  'title':'test title',
  'description':'test description'
}

Tanımlayıcının adı foldermaker.post.desc.xml:

<webscript>
  <shortname>Folder Maker</shortname>
  <description>Creates folders</description>
  <family>Examples</family>
  <url>/example/folders</url>
  <format default="json"></format>
  <authentication>user</authentication>
</webscript>

İsteğe bağlı “aile” öğesi, web komut dosyalarını web komut dosyası dizininde gruplandırmanın uygun bir yoludur. “format” öğesi, bu web betiğinin JSON döndürdüğünü bildirir.

Denetleyicinin adı foldermaker.post.json.js:

var name = title = desc = null;

var name = json.get('name');

try {
  title = json.get('title');
} catch (err) {}

try {
  desc = json.get('description');
} catch (err) {}

var folder = companyhome.createFolder(name);

var needsSave = false;

if (title != null) {
  folder.properties['cm:title'] = title;
  needsSave = true;
}

if (desc != null) {
  folder.properties['cm:description'] = desc;
  needsSave = true;
}

if (needsSave) {
  folder.save();
}

model.id = folder.nodeRef.toString();
model.name = name;
model.title = title;
model.description = desc;

Bu denetleyicinin adında “json” olduğuna dikkat edin. Bu, Alfresco’ya bir JSON yükü beklemesini söyler. Alfresco, JSON’u otomatik olarak ayrıştırır ve onu “json” adlı bir kök değişkene koyar.

Denetleyici JSON’dan adı, başlığı ve açıklamayı alır ve “companyhome” adlı kök kapsam değişkenini kullanarak klasörü oluşturur.

Bu özelliklerde bir başlık veya açıklama iletilirse kaydedilir.

Yeni klasörün düğüm referansının yanı sıra iletilen değerler, kontrolü görünüme teslim etmeden önce modelde ayarlanır.

Görünüm, foldermaker.post.json.ftl olarak adlandırılır:

{
  <#if title??>
  "title": "${title}",
  </#if>
  <#if description??>
  "description": "${description}",
  </#if>
  "id": "${id}",
  "name": "${name}"
}

Bu freemarker, modelde ayarlanan değerleri JSON olarak geri yansıtır. Başlık ve açıklama her zaman mevcut olmayabilir, bu nedenle görünüm, ayarlanmamışlarsa geri döndürmekten kaçınmak için bir if ifadesinde yerleşik Freemarker boş denetimi kullanır.

Bu web betiğini test etmek için herhangi bir HTTP istemcisi kullanılabilir. Curl kullanmanın nasıl görüneceği aşağıda açıklanmıştır:

jpotts$ curl -uadmin:admin -H "content-type: application/json" -X POST "http://localhost:8080/alfresco/s/example/folders" -d "{'name':'testfolder','title':'test title', 'description':'test desc'}"
{
  "title": "test title",
  "description": "test desc"
  "id": "workspace://SpacesStore/cc26a12f-306b-41f1-a859-668f11fc2a54",
  "name": "testfolder"
}

curl’nin temel auth kimlik bilgilerini geçtiğini unutmayın. Bu durumda “admin” kullanıyor. Bu örnek Şirket Ana Sayfasında öğeler oluşturduğundan, bunu yapmak için uygun izinlere sahip bir kullanıcı kullanmalısınız.

Bu web betiğinde gerçek bir hata denetimi yoktur. Bir ad girmezseniz veya daha önce kullanılmış bir ad girerseniz bir hata görürsünüz.
