İlkel bir tür kullanarak bir değişken bildirme ve atama #

Visual Basic’teki değişkenler, ‘Dim’ anahtar sözcüğü kullanılarak bildirilir. Örneğin, bu, “Tamsayı” veri türüyle “sayaç” adlı yeni bir değişken bildirir:

Dim counter As Integer

Değişken bildirimi ayrıca “Genel”, “Korumalı”, “Arkadaş” veya “Özel” gibi bir erişim değiştirici içerebilir. `. Bu, erişilebilirliğini belirlemek için değişkenin kapsam ile birlikte çalışır.

Erişim Değiştirici Anlam
Genel Çevreleyen türe erişebilen tüm türler
Korumalı Yalnızca çevreleyen sınıf ve ondan miras alanlar
Arkadaş Aynı derlemedeki çevreleyen türe erişebilen tüm türler
Korumalı Arkadaş Çevreleyen sınıf ve onun mirasçıları, veya aynı derlemedeki çevreleyen sınıfa erişebilen türler
Özel Sadece çevreleyen tip
Statik Yalnızca yerel değişkenlerde ve yalnızca bir kez başlatılır.

Kısaca, “Dim” anahtar sözcüğü, değişkenin bildiriminde erişim değiştiricisi ile değiştirilebilir:

Public TotalItems As Integer
Private counter As Integer

Date(Time) nesneleri, değişmez sözdizimi kullanılarak da tanımlanabilir: Karanlık örnek As Tarih = #26.07.2016 12:8 PM#

Bir değişken bildirildiğinde, örnek olarak, “Alt” veya “İşlev” bildirilen içeren türün Kapsam içinde bulunacaktır:

Public Function IncrementCounter() As Integer
    Dim counter As Integer = 0
    counter += 1

    Return counter
End Function

Sayaç değişkeni yalnızca ‘End Function’a kadar var olacak ve ardından kapsam dışı kalacaktır. Bu sayaç değişkenine fonksiyonun dışında ihtiyaç duyulursa, onu sınıf/yapı veya modül düzeyinde tanımlamanız gerekecektir.

Public Class ExampleClass

    Private _counter As Integer
   
    Public Function IncrementCounter() As Integer
       _counter += 1
       Return _counter
    End Function

End Class

Alternatif olarak, yerel bir değişkenin, çevreleyen yönteminin çağrıları arasında değerini korumasına izin vermek için “Statik” (“Paylaşılan” ile karıştırılmamalıdır) değiştiricisini kullanabilirsiniz:

Function IncrementCounter() As Integer
    Static counter As Integer = 0
    counter += 1

    Return counter
End Function

Bildirim seviyeleri – Yerel ve Üye değişkenleri #

Yerel değişkenler - Bir sınıfın (veya başka bir yapının) bir prosedürü (alt rutin veya işlev) içinde bildirilenler. Bu örnekte, “exampleLocalVariable”, “ExampleFunction()” içinde bildirilen yerel bir değişkendir:

Public Class ExampleClass1

    Public Function ExampleFunction() As Integer
        Dim exampleLocalVariable As Integer = 3
        Return exampleLocalVariable
    End Function

End Class

‘Statik’ anahtar sözcüğü, yerel bir değişkenin tutulmasına ve sonlandırmadan sonra değerini korumasına izin verir (burada genellikle, içeren prosedür sona erdiğinde yerel değişkenlerin varlığı sona erer).

Bu örnekte konsol ‘024’tür. “Main()“den “ExampleSub()“a yapılan her çağrıda, statik değişken önceki çağrının sonunda sahip olduğu değeri korur:

Module Module1

    Sub Main()
        ExampleSub()
        ExampleSub()
        ExampleSub()
    End Sub

    Public Sub ExampleSub()
        Static exampleStaticLocalVariable As Integer = 0
        Console.Write(exampleStaticLocalVariable.ToString)
        exampleStaticLocalVariable += 2
    End Sub

End Module

Üye değişkenler - Herhangi bir prosedürün dışında, sınıf (veya diğer yapı) seviyesinde beyan edilir. Bunlar, içeren sınıfın her bir örneğinin o değişkenin kendi ayrı kopyasına sahip olduğu örnek değişkenleri veya sınıfın kendisiyle ilişkili tek bir değişken olarak var olan, bağımsız, bağımsız “Paylaşılan” değişkenler olabilir. herhangi bir örneğin.

Burada, “ÖrnekSınıf2” iki üye değişken içerir. “ÖrnekSınıf2"nin her bir örneğinin, sınıf referansı aracılığıyla erişilebilen ayrı bir “ÖrnekInstanceVariable"ı vardır. Paylaşılan ExampleSharedVariable değişkenine ancak sınıf adı kullanılarak erişilir:

Module Module1

    Sub Main()

        Dim instance1 As ExampleClass4 = New ExampleClass4
        instance1.ExampleInstanceVariable = "Foo"

        Dim instance2 As ExampleClass4 = New ExampleClass4
        instance2.ExampleInstanceVariable = "Bar"

        Console.WriteLine(instance1.ExampleInstanceVariable)
        Console.WriteLine(instance2.ExampleInstanceVariable)
        Console.WriteLine(ExampleClass4.ExampleSharedVariable)

    End Sub

    Public Class ExampleClass4

        Public ExampleInstanceVariable As String
        Public Shared ExampleSharedVariable As String = "FizzBuzz"

    End Class

End Module

Erişim Değiştiricileri Örneği #

Aşağıdaki örnekte, iki projeyi barındıran bir çözümünüz olduğunu düşünün: ConsoleApplication1 ve SampleClassLibrary. İlk proje SampleClass1 ve SampleClass2 sınıflarına sahip olacaktır. İkincisinde SampleClass3 ve SampleClass4 olacaktır. Başka bir deyişle, her biri iki sınıflı iki meclisimiz var. ConsoleApplication1, SampleClassLibrary‘ye referansa sahiptir.

SampleClass1.MethodA’nın diğer sınıflar ve yöntemlerle nasıl etkileşime girdiğini görün.

ÖrnekSınıf1.vb:

SampleClassLibrary'yi içe aktarır

Genel Sınıf ÖrnekSınıf1
    Public Sub MethodA()
        'MethodA can call any of the following methods because
        'they all are in the same scope.
        MethodB()
        MethodC()
        MethodD()
        MethodE()

        'Sample2 is defined as friend. It is accessible within
        'the type itself and all namespaces and code within the same assembly.
        Dim class2 As New SampleClass2() 
        class2.MethodA()
        'class2.MethodB() 'SampleClass2.MethodB is not accessible because
                          'this method is private. SampleClass2.MethodB
                          'can only be called from SampleClass2.MethodA,
                          'SampleClass2.MethodC, SampleClass2.MethodD
                          'and SampleClass2.MethodE
        class2.MethodC()
        'class2.MethodD() 'SampleClass2.MethodD is not accessible because
                          'this method is protected. SampleClass2.MethodD
                          'can only be called from any class that inherits
                          'SampleClass2, SampleClass2.MethodA, SampleClass2.MethodC,
                          'SampleClass2.MethodD and SampleClass2.MethodE
        class2.MethodE()

        Dim class3 As New SampleClass3() 'SampleClass3 resides in other
                                         'assembly and is defined as public.
                                         'It is accessible anywhere.
        class3.MethodA()
        'class3.MethodB() 'SampleClass3.MethodB is not accessible because
                          'this method is private. SampleClass3.MethodB can
                          'only be called from SampleClass3.MethodA,
                          'SampleClass3.MethodC, SampleClass3.MethodD
                          'and SampleClass3.MethodE

        'class3.MethodC() 'SampleClass3.MethodC is not accessible because
                          'this method is friend and resides in another assembly.
                          'SampleClass3.MethodC can only be called anywhere from the
                          'same assembly, SampleClass3.MethodA, SampleClass3.MethodB,
                          'SampleClass3.MethodD and SampleClass3.MethodE

        'class4.MethodD() 'SampleClass3.MethodE is not accessible because
                          'this method is protected friend. SampleClass3.MethodD
                          'can only be called from any class that resides inside
                          'the same assembly and inherits SampleClass3,
                          'SampleClass3.MethodA, SampleClass3.MethodB,
                          'SampleClass3.MethodC and SampleClass3.MethodD      

        'Dim class4 As New SampleClass4() 'SampleClass4 is not accessible because
                                          'it is defined as friend and resides in
                                          'other assembly.
    End Sub

    Private Sub MethodB()
        'Doing MethodB stuff...
    End Sub

    Friend Sub MethodC()
        'Doing MethodC stuff...
    End Sub

    Protected Sub MethodD()
        'Doing MethodD stuff...
    End Sub

    Protected Friend Sub MethodE()
        'Doing MethodE stuff...
    End Sub
