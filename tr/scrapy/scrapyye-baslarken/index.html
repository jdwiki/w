Scrapy Kurulumu #

scrapy kurulumunun ön koşulu:

  • Python 2.7 veya üzeri 3.3
  • pip ve setuptools Python paketleri.
  • lxml
  • OpenSSL.

Scrapy’yi pip kullanarak kurabilirsiniz. pip komutunu kullanarak yüklemek için:

pip install Scrapy

Platforma özel kurulum

Anakonda

Scrapy’yi yüklemek için önerilen yol budur.

Anaconda veya Miniconda’yı zaten kurduysanız, Scrapinghub şirketi Linux, Windows ve OS X için resmi conda paketlerini korur.

Conda kullanarak Scrapy’yi kurmak için şunu çalıştırın:

conda install -c scrapinghub scrapy

Ubuntu 9.10 veya üzeri

Zaten sizin için tüm bağımlılıkları çözen ve en son hata düzeltmeleriyle sürekli güncellenen resmi Ubuntu Paketlerini kullanın.

Sistem paketlerine güvenmek yerine python bağımlılıklarını yerel olarak oluşturmayı tercih ederseniz, önce gerekli python olmayan bağımlılıklarını yüklemeniz gerekir:

sudo apt-get install python-dev python-pip libxml2-dev libxslt1-dev zlib1g-dev libffi-dev libssl-dev

Bundan sonra Scrapy’yi pip ile kurabilirsiniz:

pip install Scrapy

Archlinux

Genel talimatları takip edebilir veya Scrapy’yi AUR Scrapy paketinden kurabilirsiniz:

yaourt -S scrapy

Pencereler

Python 3 ile Scrapy henüz Windows’ta desteklenmemektedir.

Windows’a scrapy yüklemek için bu adımları izleyin:

  • Python 2.7’yi yükleyin

  • Python yürütülebilir dosyasına ve ek komut dosyalarına giden yolları dahil etmek için PATH ortam değişkenini ayarlayın. PATH’e aşağıdaki yolların eklenmesi gerekir:

    C:\Python27;C:\Python27\Scripts;

  • pywin32’yi buradan yükleyin

  • hadi Scrapy’yi yükleyelim:

      pip install Scrapy

Mac OS X

Scrapy’nin bağımlılıklarını oluşturmak, bir C derleyicisinin ve geliştirme başlıklarının varlığını gerektirir. OS X’te bu, genellikle Apple’ın Xcode geliştirme araçları tarafından sağlanır. Xcode komut satırı araçlarını kurmak için bir terminal penceresi açın ve şunu çalıştırın:

xcode-select --install

“pip"in sistem paketlerini güncellemesini engelleyen bir bilinen sorun var. Scrapy ve bağımlılıklarını başarılı bir şekilde kurmak için bunun ele alınması gerekir. İşte önerilen bazı çözümler:

  • (Önerilen) Sistem python’u kullanmayın, sisteminizin geri kalanıyla çakışmayan yeni, güncellenmiş bir sürüm yükleyin. Homebrew paket yöneticisini kullanarak bunu nasıl yapacağınız aşağıda açıklanmıştır:

    • Install homebrew following the instructions in http://brew.sh/

    • Update your PATH variable to state that homebrew packages should be used before system packages (Change .bashrc to .zshrc accordantly if you’re using zsh as default shell):

      echo "export PATH=/usr/local/bin:/usr/local/sbin:$PATH" >> ~/.bashrc

    • Reload .bashrc to ensure the changes have taken place:

      source ~/.bashrc

    • Install python:

      brew install python

    • Latest versions of python have pip bundled with them so you won’t need to install it separately. If this is not the case, upgrade python:

      brew update; brew upgrade python

  • (İsteğe bağlı) Scrapy’yi yalıtılmış bir python ortamına kurun.

Bu yöntem, yukarıdaki OS X sorunu için bir geçici çözümdür, ancak bağımlılıkları yönetmek için genel olarak iyi bir uygulamadır ve ilk yöntemi tamamlayabilir.

virtualenv python’da sanal ortamlar oluşturmak için kullanabileceğiniz bir araçtır. Başlamak için http://docs.python-guide.org/en/latest/dev/virtualenvs/ gibi bir öğretici okumanızı öneririz.

Bu geçici çözümlerden herhangi birinin ardından Scrapy’yi yükleyebilmelisiniz:

pip install Scrapy

Proje oluşturma #

Scrapy ile çalışmaya başlamadan önce, kodunuzu saklamak istediğiniz bir projeye başlamalısınız. Dizine girin ve bu kodu çalıştırın

scrapy startproject helloProject

Bu kodun üçüncü kısmı proje adıdır. Bu kod, aşağıdaki içeriğe sahip bir “merhabaProje” dizini oluşturacaktır:

helloProject/
    scrapy.cfg            # deploy configuration file

    helloProject/         # project's Python module, you'll import your code from here
        __init__.py

        items.py          # project items file

        pipelines.py      # project pipelines file

        settings.py       # project settings file

        spiders/          # a directory where you'll later put your spiders
            __init__.py
