GitHub çekme istekleri başarısız #

GitHub’dan özel bir depo çekmeye çalışırken, Jenkins’in Git ve SSH eklentileriyle hazır kurulumları çalışmayacaktır.

Microsoft’tan PSExec.exe PS Aracı #

Bulduğum bu sorunu düzeltmenin ilk adımı, PSTools dosyasını indirmek ve araçları derleme sunucusunda uygun bir konuma çıkarmaktı (örn.

PSExec veya PSExec64 kullanarak yalnızca Jenkins için yeni bir SSH anahtarı oluşturun #

  1. Önce Komut istemini ve “Yönetici Olarak Çalıştır"ı açın.

  2. Komut istemi açıldığında, PSTools dizinine gidin.

  3. Komut isteminden, Jenkins’in varsayılan olarak derleme sunucusunda çalıştırdığı Yerel Hizmet olarak PSExec veya PSExec64 kullanarak git-bash çalıştırmamız gerekiyor.

  4. PSExec’i etkileşimli olarak çalıştırmak için -i anahtarını ve yerel hizmet olarak git-bash’i çalıştırmak için -s anahtarını kullanacağız.

  5. GitHub’da bir ssh anahtarı oluşturma talimatlarını izleyin - Yeni bir SSH anahtarı oluşturma ve ssh aracısına ekleme

  6. 64 bit Windows sistemindeyseniz, .ssh klasörünü C:\Windows\SysWOW64\config\systemprofile.ssh konumuna kopyalayın (64bit Windows sistemimde buna gerek yoktu, ancak orada .ssh dosyaları orada saklanmalıdır, hala sorun yaşıyorsanız aklınızda bulundurmanız gereken bir şey).

  7. Genel SSH anahtarını github anahtarlarınıza ekleyin.

    Your Commandline should look similar to this:

    C:\Programs\PSTools> PSExec.exe -i -s C:\Programs\Git\git-bash
