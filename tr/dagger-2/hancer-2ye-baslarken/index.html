Android örneği #

Dagger kullanarak bir Android uygulaması yazmanın temel zorluklarından biri, birçok Android çerçeve sınıfının, “Activity” ve “Fragment” gibi işletim sisteminin kendisi tarafından başlatılmasıdır, ancak Dagger, enjekte edilen tüm nesneleri oluşturabiliyorsa en iyi sonucu verir. Bunun yerine, bir yaşam döngüsü yönteminde üye enjeksiyonu yapmanız gerekir. 2.10 sürümünden itibaren hançer, android bileşenleriyle hançer kullanımını basitleştiren ‘dagger.android’ kullanımına izin verir.

Etkinlik nesnelerini enjekte etme

  1. Bu temel türler için gerekli tüm bağlamaların kullanılabilir olduğundan emin olmak için uygulama bileşeninize ‘AndroidInjectionModule’ yükleyin.

     @Component(modules = {AndroidInjectionModule.class})
 public interface AppComponent {}

  2. [AndroidInjector][AndroidInjector]‘u uygulayan bir “@Subcomponent” ve [AndroidInjector.Builder][AndroidInjector.Builder] öğesini genişleten bir “@Subcomponent.Builder” yazarak başlayın:

     @Subcomponent
 public interface MainActivityComponent extends AndroidInjector<MainActivity> {
     @Subcomponent.Builder
     abstract class Builder extends AndroidInjector.Builder<MainActivity> {}
 }

  3. Alt bileşeni tanımladıktan sonra, alt bileşen oluşturucuyu bağlayan bir modül tanımlayarak ve onu “Uygulamanızı” enjekte eden bileşene ekleyerek bileşen hiyerarşinize ekleyin:

     @Module(subcomponents = MainActivityComponent.class)
 public abstract class MainActivityModule {

     @Binds @IntoMap @ActivityKey(MainActivity.class)
     abstract AndroidInjector.Factory<? extends Activity>
     bindMainActivityInjectorFactory(MainActivityComponent.Builder builder);
 }

  4. Ardından, ActivityInjector() yönteminden dönmek için ‘Application’ınızı ‘HasDispatchingActivityInjector’ ve ‘@Inject’ bir ‘DispachingAndroidInjector’ uygulamasını yapın:

     public class MyApp extends Application implements HasDispatchingActivityInjector     {

     @Inject
     DispatchingAndroidInjector<Activity> dispatchingActivityInjector;

     @Override
     public void onCreate() {
        super.onCreate();
        DaggerAppComponent.create().inject(this);
     }

     @Override
     public DispatchingAndroidInjector<Activity> activityInjector() {
         return dispatchingActivityInjector;
     }
 }

  5. Son olarak, Activity.onCreate() yönteminizde, super.onCreate(); öğesini çağırmadan önce AndroidInjection.inject(this) öğesini çağırın:

     public class MainActivity extends Activity {
     public void onCreate(Bundle savedInstanceState) {
         AndroidInjection.inject(this);
         super.onCreate(savedInstanceState);
     }
 }

Bu örnek, resmi hançer belgelerine dayanmaktadır. Çalışma örneği github adresinde bulunabilir.

Açıklama ve Kurulum #

Hançer 2 nedir?

Web sitesi kendisini şöyle tanımlıyor:

Dagger, tamamen statik, derleme zamanı bağımlılık enjeksiyon çerçevesidir

Kitaplık, bağımlılık grafiklerini modellemenin yanı sıra nesneleri yeniden kullanmayı da kolaylaştırır. Yansıma, açıklama işlemenin bir parçası olarak yalnızca derleme zamanında kullanıldığından, Dagger 2, bağımlılık enjeksiyonu için geliştirilmiş hıza sahiptir.

Kurmak

1- Ek açıklama işleme desteği ekleyin:

Android

Üst düzey “build.gradle” komut dosyası:

 repositories {
    mavenCentral()
  }
  dependencies {
    classpath 'com.neenbedankt.gradle.plugins:android-apt:1.8'
  }

Modül düzeyinde “build.gradle” komut dosyası:

apply plugin: 'com.neenbedankt.android-apt'

Java

plugins {
  id "net.ltgt.apt" version "0.5"
}

2- Hançer 2 bağımlılıklarını ekleyin

 dependencies {
      compile 'com.google.dagger🗡2.x'
      apt 'com.google.dagger:dagger-compiler:2.x'
    }

Basit bir örnekle Dagger2’yi öğrenin #

Bir sürü farklı Dagger2 öğreticisi okudum ve izledim ama çoğu çok uzun veya anlaşılması zor bu yüzden Dagger2 için yeni bir basit ve kısa eğitim yazmaya karar verdim, umarım beğenirsiniz.

Neden ihtiyacımız var? #

  • Paylaşılan örneklere erişimi basitleştirir: Paylaşılan örneklere referanslar elde etmek için basit bir yol sağlar, örneğin Dagger’da “SharedPrefrences” gibi tekli örneklerimizi bildirdikten sonra alanları basit bir “@Inject” notuyla bildirebiliriz.
  • Daha kolay birim ve entegrasyon testi: Ağ yanıtları veren ve bu davranışı taklit eden modülleri kolayca değiştirebiliriz.

Basit bir örnekle başlayalım #

Tam örnek kaynağı GitHub hesabımda mevcut.

Dagger2 bağımlılıkları ekleyin #

Öncelikle, modül düzeyindeki build.gradle dosyanıza Dagger2 bağımlılıklarını eklememiz gerekiyor.

compile "com.google.dagger🗡$dagger_version"
compile "com.google.dagger:dagger-android:$dagger_version"
compile "com.google.dagger:dagger-android-support:$dagger_version"
annotationProcessor "com.google.dagger:dagger-compiler:$dagger_version"

‘:app’ projesinde Error:Conflict with ‘com.google.code.findbugs:jsr305’ bağımlılığı gibi bir hata alıyorsanız, ana app/build.gradle dosyanıza aşağıdakileri eklemelisiniz.

configurations.all {
   resolutionStrategy.force 'com.google.code.findbugs:jsr305:3.0.1'
}

İki basit sınıf #

İki sınıfımız var (Vehicle ve Motor), Vehicle sınıfının çalışması için Motor sınıfına ve MainActivity’nin Vehicle sınıfına ihtiyacı var. Bu örnekleri sağlamak için Dagger2’yi kullanacağız.

class Vehicle {
   private Motor motor;

  @Inject
  Vehicle(Motor motor) {
     this.motor = motor;
  }

  void increaseSpeed(int value) {
     motor.accelerate(value);
  }

  void decreaseSpeed(int value) {
     motor.decelerate(value);
  }

  void stop() {
     motor.brake();
  }

  int getSpeed() {
     return motor.getRpm();
  }
}

Motor sınıfı:

class Motor {
  private int rpm;

  Motor() {
    this.rpm = 0;
  }

  int getRpm() {
    return rpm;
  }

  void accelerate(int value) {
    rpm += value;
  }

  void decelerate(int value) {
    rpm -= value;
  }

  void brake() {
    rpm = 0;
  }
}

Modül sınıfı #

Modül sınıfı, enjekte edilebilecek nesneleri sağlamaktan sorumludur. Bu örnekte, Araç sınıfına Motor sınıfını ve MainActivity’ye Araç sınıfını enjekte etmek istiyoruz, bu nedenle bu örnekleri sağlamak için MyModule oluşturmalıyız.

@Module
class MyModule {

  @Provides
  @Singleton
  Motor provideMotor() {
    return new Motor();
  }

  @Provides
  @Singleton
  Vehicle provideVehicle() {
    return new Vehicle(new Motor());
  }
}

@Ek açıklama sağlayın: Bu yöntemden döndürülen nesne, bağımlılık ekleme için kullanılabilir.

@Bileşen arayüzü #

Dagger2, sınıflarımızdan nasıl örnekler oluşturması gerektiğini bilmek için bileşen arayüzüne ihtiyaç duyar.

@Singleton
@Component(modules = {MyModule.class})
interface MyComponent {
  Vehicle provideVehicle();

  void inject(MainActivity main);
}

@Component interface: nesnenin sağlayıcısı ile bir bağımlılığı ifade eden nesneler arasındaki bağlantı.

Yapıcıya Bağımlılık Ekle #

@Inject ek açıklaması ekleyerek, hançer2, Vehicle sınıfındaki örnek Motor nesnemiz gibi bu nesneden otomatik olarak bir örnek oluşturabilir.

MainClass’ta bağımlılık enjekte etme #

Dagger2, yapıcılara bağımlılıkları otomatik olarak enjekte edebilir, ancak Android bileşenleri (aktiviteler, parçalar, vb.) Android çerçevesi tarafından başlatılır, bu da üzerlerinde bağımlılık enjeksiyonunu kullanmayı zorlaştırır, bu yüzden bunları aşağıdaki kod gibi manuel olarak enjekte etmeliyiz:

public class MainActivity extends AppCompatActivity {
  @Inject
  Vehicle vehicle;

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    MyComponent component = DaggerMyComponent.builder().build();
    component.inject(this);
  }
}

İşte bu, umarım beğenirsiniz.

Temel Örnek #

Bir modül tanımlayın (bağımlılık modeli ve grafiği):

@Module
public class CoffeeModule{

    @Provides
    public CoffeeMaker provideCoffeeMaker(){
         return new CoffeeMaker();
    }

    @Provides
    public Coffee provideCoffee(CoffeeMaker coffeeMaker){
        return new Coffee(coffeeMaker);
    }

}

Bir bileşen tanımlayın:

@Component(
    modules={
        CoffeeModule.class
    }
)
interface CoffeeComponent {

        DeveloperActivity inject(DeveloperActivity developerActivity);
}

Bağımlılıkları enjekte edin:

class DeveloperActivity extends ...{

    @Inject
    Coffee myCoffee;

    @Override
    protected void onCreate(Bundle savedInstanceState) {           
        super.onCreate(savedInstanceState);

        DaggerCoffeeComponent.builder()
                .coffeeModule(new CoffeeModule())
                .build()
                .inject();
       
    }

}
