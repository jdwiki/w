SSH anahtarınızı oluşturma #

ssh anahtarınızı ssh-keygen kullanarak oluşturabilirsiniz, ssh kurulumunun bir parçası olan bir programdır. Bunu yapmak için çalıştırın ve ekrandaki talimatları izleyin.

İşte bir örnek:

$ ssh-keygen
Generating public/private rsa key pair.

ssh anahtar çiftinin kaydedileceği varsayılan dizin, ana dizininizdeki .ssh klasörünün içindedir (bunu geçerli bir yol belirterek değiştirebilirsiniz) ve anahtar çiftinin varsayılan adı özel anahtar için id_rsadır ve ortak anahtar için “id_rsa.pub”:

Enter file in which to save the key (/home/nasreddine/.ssh/id_rsa): /home/my_folder/my_ssh_key

Bir parola girerek SSH anahtarınızı yetkisiz kullanıma karşı koruyabilirsiniz. Bu isteğe bağlıdır, ancak bir parola kullanmanız önerilir. Diğer komut programlarında olduğu gibi, parolanızı girerken ekranda hiçbir şey gösterilmeyeceğini, ancak kaydedildiğini unutmayın:

Enter passphrase (empty for no passphrase):
Enter same passphrase again:

Parolanızı girdikten sonra ssh-keygen bir anahtar oluşturacak ve onu seçtiğiniz yola kaydedecektir:

Your identification has been saved in /home/my_folder/my_ssh_key.
Your public key has been saved in /home/my_folder/my_ssh_key.pub.

Yapılmıştı. Artık ssh anahtarımız kullanıma hazır.

Bir makineye SSH nasıl yapılır #

SSH ile makinede bir kullanıcının hesabına giriş yapmak için ssh username@ip_address komutunu kullanabilirsiniz. Şifre isteyecektir. Doğru parolayı yazarsanız, o makinede o kullanıcının kabuğuna bağlanırsınız. Aksi takdirde tekrar şifre isteyecektir.

Örneğin

root@dev10:~# ssh root@10.11.50.3
root@10.11.50.3's password: 
Welcome to Ubuntu 16.04 LTS (GNU/Linux 4.4.0-31-generic x86_64)
    
Last login: Fri Jul 22 18:33:27 2016 from 10.11.50.10
root@dev2:~#

Bir makineye bağlanmak için belirli bir ssh anahtarı kullanmak istiyorsanız, ssh -i /path/to/ssh_secret_key username@host kullanın.

Bir makineye ilk kez bağlandığınızda, hedef makinenin parmak izini doğrulamanızı isteyecektir. Bu, bir ortadaki adam saldırısı önlemek için bir güvenlik mekanizmasıdır. Hedef makinede bu komutu vererek hedef makinenin parmak izini görebilirsiniz.

ssh-keygen -l -E md5 -f /etc/ssh/ssh_host_ecdsa_key.pub

Her ikisi de aynıysa “evet” yazabilirsiniz. Şifre istemine geçecektir.

Örnek:

root@dev10:~# ssh root@10.11.50.3
The authenticity of host '10.11.50.3 (10.11.50.3)' can't be established.
ECDSA key fingerprint is dd:a3:de💿5b:01:cd:0b:b6:bc:b3:09:c2:c8:1a:68.
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added '10.11.50.3' (ECDSA) to the list of known hosts.
root@10.11.50.3's password: 

Last login: Fri Jul 22 17:45:09 2016 from 10.11.1.71
root@dev2:~#

Kurulum veya Kurulum #

SSH protokolü uygulamasının ücretsiz sürümü olan OpenSSH, tüm Linux dağıtımlarında mevcuttur. Sunucu ve istemci paketlerinden oluşur.

Kurulum #

Debian tabanlı Linux’ta, kullanarak openssh yükleyebilirsiniz. #

# apt-get install openssh-server openssh-client

RHEL/CentOS’ta “yum” kullanmanız gerekir: #

# yum install openssh-server openssh-clients

Mevcut Fedora, “yum” yerine “dnf” kullanıyor.

Arch Linux’ta pacman kullanın: #

# pacman -S openssh

OSX’te ‘openssh’ zaten kurulu olmalıdır. #

Daha yeni sürümü kullanmak istiyorsanız, brew’dan openssh yüklemeniz gerekir:

# brew install openssh --with-brewed-openssl --with-keychain-support

Windows için YAPILACAKLAR Talimatları #

Kurmak #

openssh istemcisi herhangi bir özel kurulum gerektirmez ve kurulumdan hemen sonra kullanıma hazırdır. Bunu ‘ssh remote’ çalıştırmayı deneyebilirsiniz, burada ‘uzaktan’, ‘ssh’ sunucusunu çalıştıran uzak ana bilgisayardır.

openssh sunucusu genellikle kurulum ve varsayılan kurulum uygulandıktan sonra başlatılır. Değilse, kullanarak ‘systemd’ tabanlı sistemlerde başlatabilirsiniz.

systemd ile Debian tabanlı Linux’ta:

# systemctl start ssh

RHEL/CentOS/Fedora ve Arch Linux’ta:

# systemctl start sshd

Veya kullanan upstart sistemlerinde

# service sshd start

Yapılandırma #

‘openssh’, ‘/etc/ssh/’ altında yapılandırma dosyalarına sahiptir. İstemci ayrıca “~/.ssh/config” içindeki istemci yapılandırmasını da okuyor. Sunucu, varsayılan değerlerin çoğunu ve basit anahtar/değer çiftlerini içeren bir sshd_config dosyası kullanıyor. Örnek:

Protocol 2
PasswordAuthentication yes
ChallengeResponseAuthentication no
UsePAM yes
AcceptEnv LANG LC_CTYPE LC_NUMERIC LC_TIME LC_COLLATE LC_MONETARY
X11Forwarding yes
Subsystem    sftp    /usr/libexec/openssh/sftp-server

Genel anahtarınızı sunucu kullanıcısının yetkili anahtarları listesine ekleme #

Bir sunucuya “ssh” yapabilmek için kimliğinizin genel anahtarının güvenilir anahtarlar listesine eklenmesi gerekir. En yaygın olarak bu, kullanıcı başına yapılır:

ssh-copy-id -i ~/.ssh/<identity>.pub <user>@<hostname>

Hangi manuel olarak da yapılabilir:

cat ~/.ssh/<identity>.pub | ssh <user>@<hostname> 'cat >> ~/.ssh/authorized_keys'

Bunu yaptıktan sonra, kimlik dosyasını ssh çağrısına geçirirken kullanıcının şifresini vermenize gerek kalmadan giriş yapabilmelisiniz.

Yapılandırma dosyası #

OpenSSH yapılandırma dosyaları, ssh istemcisi her çalıştırıldığında uygulanması gereken yapılandırma için kullanılır. Çoğu komut satırı seçeneğini yapılandırma dosyalarına yerleştirmek mümkündür.

OpenSSH, aşağıdaki kaynaklardan gelen yapılandırmayı sırayla kullanır:

  1. Komut satırı seçenekleri
  2. Kullanıcının yapılandırma dosyası ~/.ssh/config
  3. Sistem genelinde yapılandırma dosyası /etc/ssh/ssh_config

Yapılandırma seçenekleri yapılandırma dosyalarında tek tek listelenir.

# This is a comment.

# Parameter can be specified like this, separated with white space.
StrictHostKeyChecking ask

# Or parameter key and value may be separated with white space and =.  
ForwardX11 = yes

# The parameter value can be quoted if it contains white space.    
IdentityFile "/file system/path with/white space"

Olası yapılandırma parametrelerinin tam listesi burada mevcuttur.

Yapılandırma dosyasının en kullanışlı özelliklerinden biri, ana bilgisayar adı veya adresine göre bölümlere ayrılabilmesidir. Bu şekilde farklı hostlar için farklı konfigürasyonlara sahip olabilirsiniz.

# Based on host name.    

Host host1.domain.com
    User user1

Host host2.domain.com
    User user2

# Or wildcard matching name or ip.

Host *elastic-cloud.com 10.201.4.?
    User user3

Şifre kullanarak komut dosyasından bağlanma #

Gerçekten ‘ssh’ bağlantısını kodlamanız gerektiğinde, şifreyi ‘ssh’ komutuna aktarmak işe yaramaz (’echo passw0rd | ssh host’). Bunun nedeni, parolanın standart girişten değil, doğrudan TTY’den (teleprinter, teletypewriter, geçmiş nedenlerle Teletype) okunmasıdır.

Ancak bu sorunu çözen ‘sshpass’ aracı var. Parolayı parametreden, dosyadan veya ortam değişkeninden okuyabilir. Ancak bu seçeneklerin hiçbirinin bir parola için güvenlik gereksinimlerini karşılamadığını unutmayın!

$ sshpass -p passw0rd ssh host
$ sshpass -f /secret/filename ssh host
$ SSHPASS=passw0rd sshpass -e ssh host

Komut satırı seçenekleri diğer kullanıcılar tarafından ‘ps’ içinde görülebilir (çalışma zamanı sırasında maskelenir, ancak başlangıç ​​zamanında değil ve buna güvenemezsiniz):

... 23624  6216 pts/5    Ss   Aug30   0:00  \_ /bin/bash
... 12812  1988 pts/5    S+   08:50   0:00  |   \_ sshpass -p passw0rd ssh host
... 45008  5796 pts/15   Ss+  08:50   0:00  |       \_ ssh host

Bir işlemin environemnet değişkenlerine, /proc/PID/environ dosyası kullanılarak sistemdeki diğer işlemler tarafından da erişilebileceğini unutmayın.

Son olarak, parolayı dosyada saklamak olası en iyi fikir gibi görünebilir, ancak yine de diğer örneklerde açıklandığı gibi anahtarları kullanmak ‘ssh’ kullanmak için tercih edilen yoldur.

SSH Aracısı #

Uzun bir parola belirlediyseniz ve sunucuya her bağlanmak istediğinizde tekrar girmek istemiyorsanız, bilgisayarınızda oturum açarken parolanızı saklamak için SSH-Agent kullanabilirsiniz.

ssh-agent’ı arka planda başlatın:

eval "$(ssh-agent -s)"
# Agent pid 59566

Ve anahtarınızı ssh-agent’a ekleyin, parolanızı girmeniz istenecektir:

ssh-add ~/.ssh/matrix.ac
# Enter passphrase for /home/osaka/.ssh/matrix.ac: 
# Identity added: /home/osaka/.ssh/matrix.ac (/home/osaka/.ssh/matrix.ac)

Tamamlandı. Şimdi ssh user@matrix.ac ile bağlanın ve parolanızı girmek zorunda kalmamalısınız. Gnome-anahtarlık/denizatı, anahtarlık ve diğer anahtar yöneticiler gibi programları kullanarak bunu genişletebilirsiniz.
