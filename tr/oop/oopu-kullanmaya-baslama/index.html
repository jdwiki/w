OOP Giriş #

Giriş #

Nesne Yönelimli Programlama (çoğunlukla OOP olarak anılır), sorunları çözmek için bir programlama paradigmasıdır.
Bir OO (nesne yönelimli) programın güzelliği, programı belirli siparişleri izleyen sıralı bir komut dosyası yerine birbiriyle iletişim kuran bir grup nesne olarak düşünmemizdir.

OOP’yi destekleyen birçok programlama dili vardır, bunlardan bazıları şunlardır:

  • Java
  • C++
  • c#

Python’un ayrıca OOP’yi desteklediği bilinmektedir, ancak birkaç özelliği yoktur.

OOP Terminolojisi #

OOP’deki en temel terim bir sınıf‘dir.
Bir sınıf temelde bir durumu olan ve durumuna göre çalışan bir nesnedir.

Bir diğer önemli terim de örnek‘dir.
Bir sınıfı, kendisinin örneklerini oluşturmak için kullanılan bir şablon olarak düşünün. Sınıf bir şablondur ve örnek(ler) somut nesnelerdir.

A sınıfından oluşturulan bir örneğe genellikle ’tip A’ denir, tıpkı 5 türünün int olması ve “abcd” türünün bir string olması gibi.

(class) ClassA: tipinde insance1 adlı bir örnek oluşturma örneği

Java #

ClassA örneği1 = yeni ClassA();

C++ #

SınıfA örneği1; veya

SınıfA * örnek1 = yeni SınıfA (); # Yığın üzerinde

Python #

örnek1 = SınıfA()

Yukarıdaki örnekte de görebileceğiniz gibi, her durumda sınıfın adı geçiyordu ve ondan sonra boş parantezler vardı (eğer boşsa parantezlerin bırakılabileceği C++ hariç). Bu parantezler içinde sınıfımızın kurucusuna “argümanları” iletebiliriz.

Yapıcı, her örnek oluşturulduğunda çağrılan bir sınıfın yöntemidir. Argüman alabilir veya almayabilir. Programcı oluşturduğu bir sınıf için herhangi bir kurucu belirtmezse, boş bir kurucu oluşturulacaktır (hiçbir şey yapmayan bir kurucu).
Çoğu dilde yapıcı, dönüş türünü tanımlamadan ve aynı sınıf adıyla (birkaç bölümdeki örnek) bir yöntem olarak tanımlanır.

(class) ClassB türünde b1 adlı bir örnek oluşturma örneği. ClassB yapıcısı int: tipinde bir argüman alır.

Java #

ClassA örneği1 = yeni ClassA(5); veya

int ben = 5; ClassA örneği1 = yeni ClassA(i);

C++ #

SınıfA örneği1(5);

Python #

örnek1 = SınıfA(5) Gördüğünüz gibi, bir örnek oluşturma süreci, bir işlevi çağırma sürecine çok benzer.

Fonksiyonlar ve Yöntemler #

Hem işlevler hem de yöntemler çok benzerdir, ancak Nesne Yönelimli Tasarımda (OOD) her birinin kendi anlamı vardır.
Yöntem, bir sınıfın örneği üzerinde gerçekleştirilen bir işlemdir. Yöntemin kendisi genellikle çalışmak için örneğin durumunu kullanır.
Bu arada, bir işlev belirli bir örneğe değil, bir sınıfa aittir. Bu, sınıfın durumunu veya bir örnekte depolanan herhangi bir veriyi kullanmadığı anlamına gelir.

Şu andan itibaren örneklerimizi yalnızca Java dilinde göstereceğiz, çünkü OOP bu dilde çok nettir, ancak aynı ilkeler başka herhangi bir OOP dilinde de çalışır.

Java’da bir fonksiyonun tanımında static kelimesi bulunur, şöyle:

// File's name is ClassA
public static int add(int a, int b) {
    return a + b;
}

Bu, onu komut dosyasında herhangi bir yerden arayabileceğiniz anlamına gelir.

// From the same file
System.out.println(add(3, 5));

// From another file in the same package (or after imported)
System.out.println(ClassA.add(3, 5));

Fonksiyonu başka bir dosyadan çağırdığımızda, ait olduğu sınıfın adını kullanırız (Java’da bu aynı zamanda dosyanın adıdır) bu, fonksiyonun herhangi bir örneğine değil, sınıfa ait olduğu sezgisini verir.

Buna karşılık, ClassA‘da şöyle bir yöntem tanımlayabiliriz:

// File's name is ClassA
public int subtract(int a, int b){
    return a - b;
}

Bu bildirimden sonra bu yöntemi şöyle çağırabiliriz:

ClassA a = new ClassA();
System.out.println(a.subtract(3, 5));

Burada, yöntemini çıkarma işlemini çağırmak için bir ClassA örneği oluşturmamız gerekiyordu. Dikkat edin, aşağıdakileri ** YAPAMAZ**:

System.out.println(ClassA.subtract(3, 5));

Bu satır, bu statik olmayan yöntemi örneksiz olarak adlandırdığımızdan şikayet eden bir derleme hatası üretecektir.

Bir sınıfın durumunu kullanma #

Diyelim ki * çıkarma* yöntemimizi tekrar uygulamak istiyoruz, ancak bu sefer her zaman aynı sayıyı (her örnek için) çıkarmak istiyoruz. Aşağıdaki sınıfı oluşturabiliriz:

class ClassB {

    private int sub_amount;

    public ClassB(int sub_amount) {
        this.sub_amount = sub_amount;
    }

    public int subtract(int a) {
        return a - sub_amount;
    }

    public static void main(String[] args) {
        ClassB b = new ClassB(5);
        System.out.println(b.subtract(3)); // Ouput is -2
    }
}

Bu kodu çalıştırdığımızda, ClassB sınıfının b adlı yeni bir örneği oluşturulur ve yapıcısı 5 değeriyle beslenir.
Yapıcı şimdi verilen sub_amount değerini alır ve onu sub_amount olarak da adlandırılan kendi özel alanı olarak saklar (bu kural Java’da çok bilinir, argümanları alanlarla aynı şekilde adlandırır).
Daha sonra b üzerinde 3 değeri ile subtract metodunu çağırmanın sonucunu konsola yazdırıyoruz.

  • çıkarma* uygulamasında, yapıcıdaki gibi this. kullanmadığımıza dikkat edin.
    Java’da this sadece o kapsamda tanımlanan aynı ada sahip başka bir değişken olduğunda yazılmalıdır. Aynı şey Python’un “self"i için de geçerlidir.
    Yani çıkarmada sub_amount kullandığımızda, her sınıf için farklı olan özel alana başvuruyoruz.

Vurgulamak için başka bir örnek.
Yukarıdaki koddaki ana işlevi aşağıdaki gibi değiştirelim:

ClassB b1 = new ClassB(1);
ClassB b2 = new ClassB(2);

System.out.println(b1.subtract(10)); // Output is 9
System.out.println(b2.subtract(10)); // Output is 8

Gördüğümüz gibi b1 ve b2 bağımsızdır ve her birinin kendi durumu vardır.

Arayüzler ve Kalıtım #

Bir arayüz bir sözleşmedir, bir sınıfın hangi yöntemlere sahip olacağını ve dolayısıyla yeteneklerini tanımlar. Bir arayüzün bir uygulaması yoktur, sadece yapılması gerekenleri tanımlar.
Java’da bir örnek şöyle olacaktır:

interface Printalbe {
    public void print();
}

Printalbe arabirimi, print adlı bir yöntemi tanımlar, ancak uygulamasını sağlamaz (Java için oldukça garip). Kendisini bu arabirimi ‘uygulayan’ olarak ilan eden her sınıf, draw yöntemine bir uygulama sağlamalıdır. Örneğin:

class Person implements Printalbe {

    private String name;

    public Person(String name) {
        this.name = name;
    }

    public void print() {
        System.out.println(name);
    }
}

Person kendini Drawable uyguladığını bildirirse, ancak print için bir uygulama sağlamazsa, bir derleme hatası olur ve program derlenmez.

Kalıtım, bir sınıfa başka bir sınıfı genişletmeye işaret eden bir terimdir. Örneğin, diyelim ki artık yaşı olan bir insanımız var. Böyle bir kişiyi uygulamanın bir yolu, Person sınıfını kopyalamak ve aynı alanlara ve yöntemlere sahip olan ancak başka bir özelliği olan AgedPerson adlı yeni bir sınıf yazmaktır -age.
Sınıfımıza basit bir özellik eklemek için tüm kodumuzu çoğalttığımız için bu çok kötü olurdu.
Person‘dan miras almak için kalıtımı kullanabilir ve böylece tüm özelliklerini alabilir, ardından bunları yeni özelliğimizle geliştirebiliriz, örneğin:

class AgedPerson extends Person {

    private int age;

    public AgedPerson(String name, int age) {
        super(name);
        this.age = age;
    }

    public void print() {
        System.out.println("Name: " + name + ", age:" + age);
    }
}

Devam eden birkaç yeni şey var:

  • Kişiden miras aldığımızı belirtmek için kaydedilen ’extends’ kelimesini kullandık (ve ayrıca Printable olarak uygulanmasını, bu nedenle tekrar ‘Printable’ uygulamasını bildirmemize gerek yok).
  • Person‘ın yapıcısını çağırmak için kaydetme kelimesini ‘süper’ kullandık.
  • Person‘ın print yöntemini yenisiyle geçersiz kıldık.

Bu oldukça Java tekniği olmaya başladı, bu yüzden bu konuya daha fazla girmeyeceğim. Ancak kalıtım ve arayüzler hakkında bunları kullanmaya başlamadan önce öğrenilmesi gereken birçok ekstrem durum olduğundan bahsedeceğim. Örneğin, hangi yöntemler ve işlevler miras alınır? Bir sınıftan miras alınırken özel/genel/korunan alanlara ne olur? ve benzeri.

Soyut Sınıf #

Bir soyut sınıf, OOP’de hem arayüzlerin hem de kalıtımın bir kombinasyonunu tanımlayan oldukça gelişmiş bir terimdir. İçinde hem uygulanmış hem de uygulanmamış yöntemleri/işlevleri olan bir sınıf yazmanıza izin verir. Java’da bu, ‘abstract’ anahtar sözcüğü kullanılarak yapılır ve bunu hızlı bir örnekten daha fazla açıklamayacağım:

abstract class AbstractIntStack {

    abstract public void push(int element);

    abstract public void pop();

    abstract public int top();

    final public void replaceTop(int element) {
        pop();
        push(element);
    }
}

Not: “final” anahtar sözcüğü, bu sınıftan miras aldığınızda bu yöntemi geçersiz kılamayacağınızı belirtir. Bir sınıf final olarak ilan edilirse, hiçbir sınıf ondan miras alamaz.
