Kurulum veya Kurulum #

OrientDB iki sürümde mevcuttur:

  • Topluluk Sürümü Apache 2 lisansı altında açık kaynaklı bir proje olarak yayınlandı. 2.0.html). Bu lisans, hem açık kaynak hem de ticari projeler için sınırsız ücretsiz kullanıma izin verir.

  • Enterprise Edition, Community Edition üzerine kurulmuş ticari bir yazılımdır. Enterprise, OrientDB motorunu geliştiren ekip tarafından geliştirilmiştir. Aşağıdakiler gibi Kurumsal özellikler sağlayarak Community Edition’ın bir uzantısı olarak hizmet eder:

    • Non-Stop Backup and Restore
    • Scheduled FULL and Incremental Backups
    • Query Profiler
    • Distributed Clustering configuration
    • Metrics Recording
    • Live Monitoring with configurable Alerts

Topluluk Sürümü, indirmek için ikili paket olarak veya GitHub’da kaynak kodu olarak mevcuttur. Enterprise Edition lisansı, Support satın alımlarına dahildir.

Ön koşullar

OrientDB’nin her iki sürümü de Java Sanal makinesini (JVM) uygulayan herhangi bir işletim sisteminde çalışır. Bunlara örnekler:

  • Linux, ARM dahil tüm dağıtımlar (Raspberry Pi vb.)
  • Mac OS X
  • Microsoft Windows, 95/NT ve sonrası
  • Solaris
  • HP-UX -IBM AIX

OrientDB, Java, sürüm 1.7 veya üzeri gerektirir.

Not**: OSGi kapsayıcılarında, OrientDB, LRU tabanlı önbelleği oluşturmak için concurrentlinkedhashmap tarafından sağlanan bir “ConcurrentLinkedHashMap” uygulamasını kullanır. Bu kitaplık, genellikle bir sistem paketi olarak gösterilmeyen sun.misc paketini aktif olarak kullanır. Bu sınırlamanın üstesinden gelmek için, çerçeve özellikleri listenize ‘sun.misc’ değerine sahip ‘org.osgi.framework.system.packages.extra’ özelliğini eklemelisiniz.

Platformu başlatan sanal makineye bir argüman iletmek kadar basit olabilir:

$ Java -Dorg.osgi.framework.system.packages.extra=sun.misc

Docker’ı kullanın #

Bilgisayarınızda Docker kuruluysa, OrientDB’yi çalıştırmanın en kolay yolu budur. Komut satırı türünden:

$ docker run -d –-name orientdb -p 2424:2424 -p 2480:2480
   -e ORIENTDB_ROOT_PASSWORD=root orientdb:latest

“Kök” yerine, kullanmak istediğiniz kökün şifresini yazın.

İkili Kurulum #

OrientDB, veritabanını sisteminize kurmak için önceden derlenmiş bir ikili paket sağlar. İşletim sisteminize bağlı olarak bu, OrientDB’yi çalıştırmak için ihtiyacınız olan tüm ilgili dosyaları içeren tarlı veya sıkıştırılmış bir pakettir. Masaüstü kurulumları için OrientDB İndirmeleri adresine gidin ve sisteminize en uygun paketi seçin.

Sunucu kurulumlarında wget yardımcı programını kullanabilirsiniz:

$ wget https://orientdb.com/download.php?file=orientdb-community-2.2.0.tar.gz

İster web tarayıcınızı, ister wget kullanın, indirilen dosyayı kullanımınız için uygun bir dizine açın veya çıkarın (örneğin, Linux’ta /opt/orientdb/). Bu, sisteminizde OrientDB’yi çalıştırmanız gerekecek olan ilgili dosya ve komut dosyaları ile ‘orientdb-community-2.2.0’ adlı bir dizin oluşturur.

Kaynak Kodu Kurulumu #

İkili paketleri indirmeye ek olarak, OrientDB’yi GitHub’da bulunan Community Edition kaynak kodundan derleme seçeneğine de sahipsiniz. Bu işlem, sisteminize Git ve Apache Maven yüklemenizi gerektirir.

OrientDB’yi kaynak koddan derlemek için Community Edition deposunu klonlayın, ardından yeni oluşturulan dizinde Maven’i (mvn) çalıştırın:

$ git clone https://github.com/orientechnologies/orientdb
$ git checkout develop
$ cd orientdb
$ mvn clean install

Testleri atlamak mümkündür:

$ mvn clean install -DskipTests

Geliştirme dalı, OrientDB’nin bir sonraki sürümü için kod içerir. Kararlı sürümler ana dalda etiketlenir. Her bakımlı sürüm için OrientDB’nin kendi “düzeltme” dalı vardır. Bu notlar yazılırken şubelerin durumu şöyle:

  • geliştirme: sonraki 3.0.x sürümü için devam eden çalışma (3.0.x-SNAPSHOT)
  • 2.2.x: sonraki 2.2.x kararlı sürümü için düzeltme (2.2.x-SNAPSHOT)
  • 2.1.x: sonraki 2.1.x kararlı sürümü için düzeltme (2.1.x-SNAPSHOT)
  • 2.0.x: sonraki 2.0.x kararlı sürümü için düzeltme (2.0.x-SNAPSHOT)
  • master’daki son etiket 2.2.0’dır

Derleme işlemi, tüm jar’ları yerel maven deposuna kurar ve “hedef” dizini içindeki “dağıtım” modülü altında arşivler oluşturur. Yazma sırasında, 2.1.x şubesinden bina şunları verdi:

$ls -l distribution/target/
total 199920
    1088 26 Jan 09:57 archive-tmp
     102 26 Jan 09:57 databases
     102 26 Jan 09:57 orientdb-community-3.0.0-SNAPSHOT.dir
48814386 26 Jan 09:57 orientdb-community-3.0.0-SNAPSHOT.tar.gz
53542231 26 Jan 09:58 orientdb-community-3.0.0-SNAPSHOT.zip
$

orientdb-community-3.0.0-SNAPSHOT.dir dizini, sıkıştırılmamış OrientDB dağıtımını içerir. Katılmak istiyorsanız Contribute to OrientDB sayfasına bir göz atın.

Her dağıtım paketi, GratefulDeadConcerts adlı örnek bir veritabanı içerir. Veritabanını oluşturmak mümkündür:

$ cd distribution/
$ mvn  prepare-package
[INFO] Scanning for projects...
[INFO]

Done: imported 809 vertices and 8049 edges

Veritabanı hedef dizinde oluşturulur

$ ls target/databases/
GratefulDeadConcerts/

İzinleri Güncelle #

Linux, Mac OS X ve UNIX tabanlı işletim sistemi için kaynaktan derledikten sonra bazı dosyaların izinlerini değiştirmeniz gerekir.

$ chmod 755 bin/*.sh
$ chmod -R 777 config

Bu komutlar, “config/” dizinindeki dosyaların yürütme izinlerini ve “bin/” içindeki kabuk komut dosyalarını güncelleyerek, derlediğiniz komut dosyalarını veya programları çalıştırabilmenizi sağlar.

Kurulum Sonrası Görevler #

İkili dosyayı yükleyen masaüstü kullanıcıları için, OrientDB şimdi yüklenmiştir ve kurulumun bin paketinde bulunan kabuk komut dosyaları aracılığıyla çalıştırılabilir. Sunucular için, bir hizmet olarak OrientDB için veritabanı sunucusunu yönetmek için yapmanız gereken bazı ek adımlar vardır. Bunun için prosedür, işletim sisteminize bağlı olarak değişir.

  • Unix, Linux ve Mac OS X’te Hizmet Olarak Yükleyin
  • Microsoft Windows’ta Hizmet Olarak Yükle

Yükseltme #

OrientDB’nin daha yeni bir sürümüne yükseltme zamanı geldiğinde, yöntemler ilk etapta nasıl kurmayı seçtiğinize bağlı olarak değişir. İkili indirmelerden yüklediyseniz, yukarıdaki indirme işlemini tekrarlayın ve yeni dizine işaret etmek için tüm sembolik bağlantıları veya kısayolları güncelleyin.

OrientDB’nin kaynaktan oluşturulduğu sistemler için en son kaynak kodunu aşağı çekin ve kaynaktan derleyin.

$ git pull origin master
$ mvn clean install

Kaynaktan derleme yaptığınızda, Git’i kullanarak OrientDB’nin farklı sürümlerini oluşturmak için dallar arasında geçiş yapabileceğinizi unutmayın. Örneğin,

$ git checkout 2.2.x
$ mvn clean install

“master” yerine “2.2.x” dalını oluşturur.

OrientDB ile tek bir yürütülebilir kavanoz oluşturma #

Motorlar, operatörler, fabrikalar gibi dahili bileşenler için OrientDB, Java SPI Servis Sağlayıcı Arayüzü kullanır. Bu, OrientDB kavanozlarının, bileşenlerin uygulanmasını içeren “META-INF/services” içindeki dosyalarla birlikte gönderildiği anlamına gelir. Tek bir yürütülebilir jar oluştururken, aynı ada sahip dosyaların içeriğini farklı orientdb-*.jar içinde birleştirmeniz gerektiğini unutmayın. Maven Shade Plugin kullanıyorsanız, Service Resource Transformer kullanabilirsiniz. maven-shade-plugin/examples/resource-transformers.html#ServicesResourceTransformer) bunu yapmak için.
