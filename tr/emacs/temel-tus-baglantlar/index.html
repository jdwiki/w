Emacs’tan Çık #

Aşağıdaki tuş bağlama ile Emacs’tan çıkabilirsiniz:

C-x C-c

“C”, “kontrol” anahtarıdır.

Emacs'i Askıya Al

Aşağıdaki tuş bağlamayı kullanarak Emacs’ı askıya alabilirsiniz:

C-z

Sizi kabuğunuza geri döndürür. Emacs oturumunuzu sürdürmek istiyorsanız, terminalinize fg girin.

Geçerli komutu iptal et #

Çoğu zaman, devam eden kısmen yazılmış bir komut dizisine sahip olduğunuz bir duruma girersiniz, ancak bunu iptal etmek istersiniz. Aşağıdaki tuş bağlantılarından biriyle iptal edebilirsiniz:

C-g

EscEscEsc

Tamponlar #

  • Bir arabellek listesi örneği

    CRM Buffer                Size  Mode              Filename[/Process]
. * .emacs                3294  Emacs-Lisp        ~/.emacs
 %  *Help*                 101  Help
    search.c             86055  C                 ~/cvs/emacs/src/search.c
 %  src                  20959  Dired by name     ~/cvs/emacs/src/
  * *mail*                  42  Mail
 %  HELLO                 1607  Fundamental       ~/cvs/emacs/etc/HELLO
 %  NEWS                481184  Outline           ~/cvs/emacs/etc/NEWS
    *scratch*              191  Lisp Interaction
  * *
Messages*            1554  Messages

    The first field of a line indicates:

    • ‘.’ the buffer is current.
    • ‘%’ a read-only buffer.
    • ‘*’ the buffer is modified.

  • Arabelleği seçin. Aşağıdaki tuş bağlama ile herhangi bir açık arabellek arasından seçim yapabilirsiniz:

C-x b

Geçiş yapmak istediğiniz arabellek adı sorulacak.

  • Liste tamponları:

    C-x C-b

  • Bazı arabelleği kaydet, hangi arabelleğin kaydedilip kaydedilmeyeceğini seçme:

    C-x s

  • Bir arabelleği öldür:

    C-x k

  • İşaretli arabelleklerdeki işlemler:

    <kbd>S</kbd>  Save the marked buffers                                                                  

<kbd>A</kbd>  View the marked buffers in this frame.                                                   

<kbd>H</kbd>  View the marked buffers in another frame.                                                

<kbd>V</kbd> Revert the marked buffers.                                                               

<kbd>T</kbd> Toggle read-only state of marked buffers.                                                

<kbd>D</kbd> Kill the marked buffers.                                                                 

<kbd>M-s a C-s</kbd> Do incremental search in the marked buffers.                                     

<kbd>M-s a C-M-s</kbd> Isearch for regexp in the marked buffers.                                      

<kbd>U</kbd> Replace by regexp in each of the marked buffers.                                                                                                                  

<kbd>Q</kbd> Query replace in each of the marked buffers.                                             

<kbd>I</kbd> As above, with a regular expression.                                                     

<kbd>P</kbd> Print the marked buffers.                                                                

<kbd>O</kbd> List lines in all marked buffers which match a given regexp (like the function `occur').                                                                                    

<kbd>X</kbd> Pipe the contents of the marked buffers to a shell command.                                                   

<kbd>N</kbd> Replace the contents of the marked buffers with the output of a shell command.  

<kbd>!</kbd> Run a shell command with the buffer's file as an argument.                                                             

<kbd>E</kbd> Evaluate a form in each of the marked buffers. This is a very flexible command.  For example, if you want to make all of the marked buffers read only, try using (read-only-mode 1) as the input form.                                  

<kbd>W</kbd> - As above, but view each buffer while the form is evaluated.                                           

<kbd>k</kbd> - Remove the marked lines from the *Ibuffer* buffer, but don't kill the associated buffer.                                      

<kbd>x</kbd> - Kill all buffers marked for deletion.

  • Bazı arabelleği kaydet, hangi arabelleğin kaydedilip kaydedilmeyeceğini seçme:

    C-x s

  • Bir sonraki ara belleğe geçin:

C-x SAĞ

  • Önceki ara belleğe geç:

C-x SOL

Dosya yönetimi #

  • Açık dosyayı aynı dosya adı altında yeniden kaydedin (Kaydet):

    C-x C-s

  • filename (Farklı Kaydet) olarak yazın:

    C-x C-w filename

Yeni dosya adı mini arabellekte sorulacak.

  • Yeni dosya oluştur veya mevcut dosyayı yükle (Yeni / Yükle):

    C-x C-f filename

    With the mnemonic here for f meaning file. You will be prompted for a file path in the minibuffer.

  • Alternatif dosyayı ziyaret edin

    C-x C-f

    If the file does not exist yet, you will be prompted the path of the file to create in the minibuffer.

İmleç (nokta) hareketi #

Ana Sayfa, Bitiş, Sayfa yukarı ve Sayfa aşağı ok tuşlarını kullanan imleç hareketlerine ek olarak emacs, imleci daha küçük veya daha büyük metin parçaları üzerinde hareket ettirebilen bir dizi tuş vuruşu tanımlar:

Karakter bazında:

  • Geri karakter: C-b
  • İleri karakter: C-f

Kelime ile

  • Geri kelime: M-b (i.e. Alt b veya Meta b)
  • İleri kelime: M-f

Satıra göre:

  • Geçerli satırın başlangıcı: C-a
  • Geçerli satırın ilk (boşluk olmayan) karakterinin başlangıcı:M-m
  • Geçerli satırın sonu: C-e
  • Önceki satır: C-p
  • Sonraki satır: C-n

Tüm arabellek:

  • Arabelleğin başlangıcı: M-«/kbd>
  • Arabelleğin sonu: M->

İçeriğe (mod) bağlı olarak ‘blok’ ile:

Tipik anahtar bağlamaları:

  • Geriye dönük cümle/ifade: M-a
  • İleri cümle/ifade: M-e
  • İşlevin başlangıcı: M-C-a
  • İşlevin sonu: M-C-e

Önek bağımsız değişkenleri

Birkaç ‘adım’ı aynı anda hareket ettirmek için, hareket komutlarına ESC veya C-u tuşlarına basılarak bir önek argümanı ve listelenen tuş vuruşlarından önce bir sayı verilebilir. C-u için sayı isteğe bağlıdır ve varsayılan olarak 4’tür. Örn. ESC 3 C-n 3 satır aşağı, C-u M-f 4 kelime ileri gider.

Anahtar bağlama gösterimi #

Emacs’ın belgeleri, burada açıklanan tüm anahtar bağlamalar için tutarlı bir gösterim kullanır:

Anahtar akorlar #

İki veya daha fazla tuşa aynı anda basılarak bir “anahtar akoru” elde edilir. Tuş akorları, tüm tuşların tire (-) ile ayrılmasıyla gösterilir. Genellikle öne çıkan değiştirici tuşları içerirler:

  • C-: kontrol;
  • S-: vardiya;
  • M-: alt (“M”, tarihsel nedenlerle “Meta” anlamına gelir).

Diğer tuşlar basitçe adlarıyla belirtilir, örneğin:

  • a: “a” tuşu;
  • sol: sol ok tuşu;
  • SPC: boşluk tuşu;
  • RET: dönüş anahtarı.

Bu nedenle anahtar akor örnekleri şunları içerir:

  • C-a: aynı anda kontrol ve a tuşlarına basın;
  • S-sağ: aynı anda shift ve sağa tuşlarına basın;
  • C-M-a: aynı anda kontrol, alt ve a tuşlarına basın.

Tuş dizileri #

“Anahtar dizileri”, birbiri ardına yazılması gereken anahtar dizileridir (veya anahtar akorları). Tüm anahtar (veya akor) notasyonlarını bir boşlukla ayırarak belirtilirler.

Örnekler şunları içerir:

  • C-x b: aynı anda kontrol ve x tuşlarına basın, ardından bunları bırakın ve b tuşuna basın;
  • C-x C-f: aynı anda kontrol ve x tuşlarına basın, ardından x‘yi bırakın ve f</kbd’ye basın > (her iki akor da kontrol değiştiricisini içerdiğinden, onu serbest bırakmak gerekli değildir).

Alt yerine ESC kullanma #

Alt değiştiriciyi kullanan tuş akorları, ESC ile başlayan bir tuş dizisi olarak da girilebilir. Bu, Alt tuş akorlarını iletmeyen bir uzak bağlantı üzerinden Emacs kullanılırken veya bu tuş kombinasyonları bir pencere yöneticisi tarafından _ör.

Örnek:

M-x, ESC x olarak girilebilir.

Emacs lisp dosyalarındaki anahtar bağlantılarını açıklama #

Burada açıklanan gösterimin aynısı, Emacs lisp dosyalarında anahtar bağlamaları tanımlanırken kullanılabilir.

Örnek:

(global-set-key (kbd “C-x C-b”) ’tampon menüsü) C-x C-b tuş dizisini “buffer-menu” komutuna bağlar

Birden çok pencere veya çerçeve #

Emacs’taki “Pencere”, aksi takdirde “bölme” veya “ekran bölümü” olarak adlandırılabilecek şeyi ifade eder. Bazı pencere işleme komutları şunları içerir:

  • Geçerli pencereyi yatay olarak böl: C-x 2
  • Geçerli pencereyi dikey olarak böl: C-x 3
  • Sonraki pencereyi seçin: C-x o
  • Geçerli pencereyi kapat: C-x 0
  • Geçerli pencere hariç diğer tüm pencereleri kapatın: C-x 1

Emacs’taki bir “çerçeve”, aksi takdirde “pencere” olarak adlandırılabilecek şeydir. Çerçeveler şu komutlar kullanılarak değiştirilir:

  • Yeni çerçeve oluştur: C-x 5 2
  • Geçerli kareyi sil: C-x 5 0
  • Diğer kareleri sil: C-x 5 1

Anahtarlama pencereleri kullanılarak elde edilebilir

  • S-sol, S-sağ, S-yukarı, S-aşağı (yani, <kbd Bir yönde komşu pencereye geçmek için bir ok tuşuyla birlikte >Shift veya
  • Bir sonraki pencereye geçmek için C-x o.

Bölge - Kes, Kopyala, Yapıştır #

  • İmleç konumunda işareti ayarlayın:

    C-space or C-@

  • Bölgeyi öldür (Kes):

    C-w

  • Yüzüğü öldürmek için bölgeyi kopyalayın:

    M-w or Esc-w

  • Yank (Yapıştır) en son öldürülen:

    C-y

  • Yank (Yapıştır) sonraki son öldürme:

    M-y or Esc-y

Öldürmek #

kill, Emacs tarafından metin silmek için kullanılan komuttur. “kill” komutu, Windows’taki “cut” komutuna benzer. Bir kelimeyi (M-d), satırın geri kalanını (C-k) veya daha büyük metin bloklarını ‘öldüren’ çeşitli komutlar mevcuttur. Silinen metin, daha sonra içinden çekilebileceği “öldürme halkasına” eklenir.

Seç ve kes (öldür) #

Öldürme ve çekme Windows’taki ‘seç ve kes’ özelliğine benzer şekilde, burada C-spc var. C-spc anahtar bağlaması seçimi başlatır, kullanıcı seçim yapmak için ok tuşları veya başka bir komut yardımıyla işareti hareket ettirebilir. Seçim tamamlandığında - seçilen metni öldürmek için C-w‘ye basın

‘kill’ komutu için hızlı başvuru olarak kullanılabilecek bazı temel komutlar (Emacs eğitiminden alınmıştır)

    M-DEL       Kill the word immediately before the cursor
    M-d         Kill the next word after the cursor

    C-k         Kill from the cursor position to end of line
    M-k         Kill to the end of the current sentence

Çekmek #

yank daha önce silinmiş metnin eklenmesini tanımlar, ör. en son öldürülen metni çeken C-y kullanılarak. “Yank” komutu, Windows’taki “yapıştır” komutuna benzer.

Yank metin daha önce öldürüldü #

‘kill’ komutunun, öldürülen metni bir ‘kill-ring’e eklediğini biliyoruz. Silinen metni ‘kill-ring’den almak için, istenen metin çekilinceye kadar M-y komutunu art arda kullanın.

(Not: M-y tuşunun çalışması için önceki komutun ‘YANK’ olması gerekir, aksi halde çalışmaz)

Geri alma #

Az önce yaptığınız bir şeyi geri almak için:

C-_ veya C-x u veya C-/

Ara ve Değiştir #

Emacs’ta, temel arama aracı (“I-Arama”), imlecinizin bulunduğu yerden önce veya sonra arama yapmanızı sağlar.

  • İmlecinizin konumundan sonra bir metin aramak için (ileriye doğru arama') <kbd> C-s *sometext*</kbd> tuşuna basın. *sometext`* öğesinin bir sonraki oluşumuna gitmek istiyorsanız, tekrar C-s tuşuna basın (ve sonraki oluşumlar için bu şekilde devam edin). İmleç doğru konuma geldiğinde, arama isteminden çıkmak için Enter tuşuna basın.

  • İmlecinizin bulunduğu yerden önce arama yapmak için (`geriye doğru arama’), daha önce kullandığınız şekilde C-r‘yi kullanın.

  • “Geri arama"dan “ileri arama"ya geçmek için 2 kez C-s tuşuna basın. Ve ‘ileriye doğru arama’ istemindeyken ‘geriye doğru aramak’ için C-r tuşuna 2 kez basın.

  • Ara ve değiştir:

    M-% (or Esc-% ) oldtext Enter newtext Enter

    • Confirm: y
    • Skip: n
    • Quit: q
    • Replace all: !

Dava #

  • Kelimeyi büyük harfle yaz: M-c

  • Kelimeyi büyük harfe çevir: M-u

  • Kelimeyi küçük harfe çevir: M-l
