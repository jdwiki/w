O Entity Framework Code-First fornece um conjunto de atributos DataAnnotation, que você pode aplicar às suas classes e propriedades de domínio. Os atributos DataAnnotation substituem as convenções de Code-First padrão.

  1. System.ComponentModel.DataAnnotations inclui atributos que afetam a nulidade ou o tamanho da coluna.
  2. O namespace System.ComponentModel.DataAnnotations.Schema inclui atributos que afetam o esquema do banco de dados.

Observação: DataAnnotations fornece apenas um subconjunto de opções de configuração. A API Fluent fornece um conjunto completo de opções de configuração disponíveis no Code-First.

atributo [Coluna] #

public class Person
{
    public int PersonID { get; set; }
    
    [Column("NameOfPerson")]
    public string PersonName { get; set; }    
}

Diz ao Entity Framework para usar um nome de coluna específico em vez de usar o nome da propriedade. Você também pode especificar o tipo de dados do banco de dados e a ordem da coluna na tabela:

[Column("NameOfPerson", TypeName = "varchar", Order = 1)]
public string PersonName { get; set; }

Atributo [Obrigatório] #

Quando aplicado a uma propriedade de uma classe de domínio, o banco de dados criará uma coluna NOT NULL.

using System.ComponentModel.DataAnnotations;

public class Person
{
    public int PersonID { get; set; }
    
    [Required]
    public string PersonName { get; set; }    
}

Atributos [MaxLength] e [MinLength] #

[MaxLength(int)] atributo pode ser aplicado a uma propriedade do tipo string ou array de uma classe de domínio. O Entity Framework definirá o tamanho de uma coluna para o valor especificado.

using System.ComponentModel.DataAnnotations;

public class Person
{
    public int PersonID { get; set; }
    
    [MinLength(3), MaxLength(100)]
    public string PersonName { get; set; }    
}

A coluna resultante com o comprimento de coluna especificado:

[MinLength(int)] atributo é um atributo de validação, não afeta a estrutura do banco de dados. Se tentarmos inserir/atualizar uma Pessoa com PersonName com comprimento menor que 3 caracteres, este commit falhará. Receberemos um DbUpdateConcurrencyException que precisaremos manipular.

using (var db = new ApplicationDbContext())
{                    
     db.Staff.Add(new Person() { PersonName = "ng" });
     try
     {
          db.SaveChanges();
     }
     catch (DbEntityValidationException ex)
     {
          //ErrorMessage = "The field PersonName must be a string or array type with a minimum length of '3'."
     }
}

Os atributos [MaxLength] e [MinLength] também podem ser usados ​​com asp.net-mvc como um atributo de validação.

atributo [DatabaseGenerated] #

Especifica como o banco de dados gera valores para a propriedade. Existem três valores possíveis:

  1. Nenhum especifica que os valores não são gerados pelo banco de dados.
  2. “Identidade” especifica que a coluna é uma coluna de identidade, que normalmente é usada para chaves primárias inteiras.
  3. Computed especifica que o banco de dados gera o valor para a coluna.

Se o valor for diferente de None, o Entity Framework não confirmará as alterações feitas na propriedade de volta ao banco de dados.

Por padrão (com base na StoreGeneratedIdentityKeyConvention), uma propriedade de chave inteira será tratada como uma coluna de identidade. Para substituir esta convenção e forçá-la a ser tratada como uma coluna sem identidade, você pode usar o atributo DatabaseGenerated com um valor de None.

using System.ComponentModel.DataAnnotations.Schema;

public class Foo
{
    [Key]
    public int Id { get; set; } // identity (auto-increment) column
}

public class Bar
{
    [Key]
    [DatabaseGenerated(DatabaseGeneratedOption.None)]
    public int Id { get; set; } // non-identity column
}

O SQL a seguir cria uma tabela com uma coluna computada:

CREATE TABLE [Person] (
    Name varchar(100) PRIMARY KEY,
    DateOfBirth Date NOT NULL,
    Age AS DATEDIFF(year, DateOfBirth, GETDATE())
)
GO

Para criar uma entidade para representar os registros na tabela acima, você precisaria usar o atributo DatabaseGenerated com um valor de Computed.

[Table("Person")]
public class Person
{
    [Key, StringLength(100)]
    public string Name { get; set; }
    public DateTime DateOfBirth { get; set; }
    [DatabaseGenerated(DatabaseGeneratedOption.Computed)]
    public int Age { get; set; }
}

[ForeignKey(string)] atributo #

Especifica o nome da chave estrangeira personalizada se uma chave estrangeira não seguir a convenção do EF for desejada.

public class Person 
{
    public int IdAddress { get; set; }

    [ForeignKey(nameof(IdAddress))]
    public virtual Address HomeAddress { get; set; }
}

Isso também pode ser usado quando você tem vários relacionamentos com o mesmo tipo de entidade.

using System.ComponentModel.DataAnnotations.Schema;

public class Customer
{
    ...

    public int MailingAddressID { get; set; }
    public int BillingAddressID { get; set; }

    [ForeignKey("MailingAddressID")]
    public virtual Address MailingAddress { get; set; }
    [ForeignKey("BillingAddressID")]
    public virtual Address BillingAddress { get; set; }
}

Sem os atributos ForeignKey, o EF pode misturá-los e usar o valor de BillingAddressID ao buscar o MailingAddress, ou pode apenas apresentar um nome diferente para a coluna com base em suas próprias convenções de nomenclatura (como Address_MailingAddress_Id) e tente usar isso (o que resultaria em um erro se você estiver usando isso com um banco de dados existente).

[InverseProperty(string)] atributo #

using System.ComponentModel.DataAnnotations.Schema;

public class Department  
{  
    ...

    public virtual ICollection<Employee> PrimaryEmployees { get; set; }  
    public virtual ICollection<Employee> SecondaryEmployees { get; set; }  
}  
  
public class Employee  
{  
    ...

    [InverseProperty("PrimaryEmployees")]  
    public virtual Department PrimaryDepartment { get; set; }  
  
    [InverseProperty("SecondaryEmployees")]  
    public virtual Department SecondaryDepartment { get; set; }  
}

InverseProperty pode ser usado para identificar relacionamentos bidirecional quando existem relacionamentos múltiplos bidirecional entre duas entidades.

Ele informa ao Entity Framework quais propriedades de navegação ele deve corresponder às propriedades do outro lado.

O Entity Framework não sabe qual mapa de propriedades de navegação com quais propriedades do outro lado quando existem vários relacionamentos bidirecionais entre duas entidades.

Ele precisa do nome da propriedade de navegação correspondente na classe relacionada como parâmetro.

Isso também pode ser usado para entidades que possuem relacionamento com outras entidades do mesmo tipo, formando um relacionamento recursivo.

using System.ComponentModel.DataAnnotations;
using System.ComponentModel.DataAnnotations.Schema;

public class TreeNode
{
    [Key]
    public int ID { get; set; }
    public int ParentID { get; set; }

    ...

    [ForeignKey("ParentID")]
    public TreeNode ParentNode { get; set; }
    [InverseProperty("ParentNode")]
    public virtual ICollection<TreeNode> ChildNodes { get; set; }
}

Observe também o uso do atributo ForeignKey para especificar a coluna que é usada para a chave estrangeira na tabela. No primeiro exemplo, as duas propriedades na classe Employee poderiam ter o atributo ForeignKey aplicado para definir os nomes das colunas.

Atributo [ComplexType] #

using System.ComponentModel.DataAnnotations.Schema;

[ComplexType] 
public class BlogDetails 
{ 
    public DateTime? DateCreated { get; set; } 
 
    [MaxLength(250)] 
    public string Description { get; set; } 
}

public class Blog
{
    ...

    public BlogDetails BlogDetail { get; set; }
}

Marque a classe como tipo complexo no Entity Framework.

Tipos complexos (ou objetos de valor no design orientado a domínio) não podem ser rastreados por conta própria, mas são rastreados como parte de uma entidade. É por isso que BlogDetails no exemplo não tem uma propriedade de chave.

Eles podem ser úteis ao descrever entidades de domínio em várias classes e camadas dessas classes em uma entidade completa.

[Chave] atributo #

Chave é um campo em uma tabela que identifica exclusivamente cada linha/registro em uma tabela de banco de dados.

Use este atributo para substituir a convenção Code-First padrão. Se aplicada a uma propriedade, ela será usada como a coluna de chave primária dessa classe.

using System.ComponentModel.DataAnnotations;

public class Person
{
    [Key]
    public int PersonKey { get; set; }        // <- will be used as primary key
     
    public string PersonName { get; set; }    
}

Se uma chave primária composta for necessária, o atributo [Key] também poderá ser adicionado a várias propriedades. A ordem das colunas na chave composta deve ser fornecida no formato [Key, Column(Order = x)].

using System.ComponentModel.DataAnnotations;

public class Person
{
    [Key, Column(Order = 0)]
    public int PersonKey1 { get; set; }    // <- will be used as part of the primary key

    [Key, Column(Order = 1)]
    public int PersonKey2 { get; set; }    // <- will be used as part of the primary key
     
    public string PersonName { get; set; }    
}

Sem o atributo [Key], EntityFramework retornará à convenção padrão que é usar a propriedade da classe como uma chave primária chamada “Id” ou “{ClassName}Id”.

public class Person
{
    public int PersonID { get; set; }        // <- will be used as primary key
     
    public string PersonName { get; set; }    
}

atributo [Range(min,max)] #

Especifica um intervalo numérico mínimo e máximo para uma propriedade

using System.ComponentModel.DataAnnotations;   

public partial class Enrollment
{
    public int EnrollmentID { get; set; }
   
    [Range(0, 4)]
    public Nullable<decimal> Grade { get; set; }
}

Se tentarmos inserir/atualizar um Grau com valor fora do intervalo, este commit falhará. Receberemos um DbUpdateConcurrencyException que precisaremos manipular.

using (var db = new ApplicationDbContext())
{
    db.Enrollments.Add(new Enrollment() { Grade = 1000 });

    try
    {
        db.SaveChanges();
    }
    catch (DbEntityValidationException ex)
    {
        // Validation failed for one or more entities
    }
}

Também pode ser usado com asp.net-mvc como um atributo de validação.

Atributo [Não mapeado] #

Por convenção Code-First, o Entity Framework cria uma coluna para cada propriedade pública que seja de um tipo de dados com suporte e tenha um getter e um setter. [NotMapped] anotação deve ser aplicada a todas as propriedades para as quais NÃO queremos uma coluna em uma tabela de banco de dados.

Um exemplo de propriedade que talvez não queiramos armazenar no banco de dados é o nome completo de um aluno com base em seu nome e sobrenome. Isso pode ser calculado em tempo real e não há necessidade de armazená-lo no banco de dados.

public string FullName => string.Format("{0} {1}", FirstName, LastName);

A propriedade “FullName” tem apenas um getter e nenhum setter, portanto, por padrão, o Entity Framework NÃO criará uma coluna para ele.

Outro exemplo de uma propriedade que talvez não queiramos armazenar no banco de dados é o “AverageGrade” de um aluno. Não queremos obter o AverageGrade sob demanda; em vez disso, podemos ter uma rotina em outro lugar que o calcule.

[NotMapped]
public float AverageGrade { set; get; }

O “AverageGrade” deve ser marcado com a anotação [NotMapped], senão o Entity Framework criará uma coluna para ele.

using System.ComponentModel.DataAnnotations.Schema;

public class Student
{
    public int Id { set; get; }

    public string FirstName { set; get; }

    public string LastName { set; get; }

    public string FullName => string.Format("{0} {1}", FirstName, LastName);

    [NotMapped]
    public float AverageGrade { set; get; }
}

Para a Entidade acima veremos dentro de DbMigration.cs

CreateTable(
    "dbo.Students",
     c => new
         {
             Id = c.Int(nullable: false, identity: true),
             FirstName = c.String(),
             LastName = c.String(),
         })
     .PrimaryKey(t => t.Id);

e no SQL Server Management Studio

Atributo [Tabela] #

[Table("People")]
public class Person
{
    public int PersonID { get; set; }
    public string PersonName { get; set; }    
}

Diz ao Entity Framework para usar um nome de tabela específico em vez de gerar um (ou seja, Person ou Persons)

Também podemos especificar um esquema para a tabela usando o atributo [Table]

[Table("People", Schema = "domain")]

[Índice] atributo #

public class Person
{
    public int PersonID { get; set; }
    public string PersonName { get; set; }

    [Index]
    public int Age { get; set; }
}

Cria um índice de banco de dados para uma coluna ou conjunto de colunas.

[Index("IX_Person_Age")]
public int Age { get; set; }

Isso cria um índice com um nome específico.

[Index(IsUnique = true)]
public int Age { get; set; }

Isso cria um índice exclusivo.

[Index("IX_Person_NameAndAge", 1)]
public int Age { get; set; }

[Index("IX_Person_NameAndAge", 2)]
public string PersonName { get; set; }

Isso cria um índice composto usando 2 colunas. Para fazer isso, você deve especificar o mesmo nome de índice e fornecer uma ordem de coluna.

Observação: O atributo Index foi introduzido no Entity Framework 6.1. Se você estiver usando uma versão anterior, as informações nesta seção não se aplicam.

atributo [StringLength(int)] #

using System.ComponentModel.DataAnnotations;

public class Post
{
    public int Id { get; set; }
    
    [StringLength(100)]
    public string Title { get; set;}

    [StringLength(300)]
    public string Abstract { get; set; }
    
    public string Description { get; set; }
}

Define um comprimento máximo para um campo de string.

Observação: Também pode ser usado com asp.net-mvc como atributo de validação.

Atributo [carimbo de data e hora] #

O atributo [TimeStamp] pode ser aplicado a apenas uma propriedade de matriz de bytes em uma determinada classe Entity. O Entity Framework criará uma coluna de carimbo de data/hora não anulável na tabela de banco de dados para essa propriedade. O Entity Framework usará automaticamente esta coluna TimeStamp na verificação de simultaneidade.

using System.ComponentModel.DataAnnotations.Schema;

public class Student
{
    public int Id { set; get; }

    public string FirstName { set; get; }

    public string LastName { set; get; }    

    [Timestamp]
    public byte[] RowVersion { get; set; }
}

[ConcurrencyCheck] Atributo #

Este atributo é aplicado à propriedade de classe. Você pode usar o atributo ConcurrencyCheck quando quiser usar colunas existentes para verificação de simultaneidade e não uma coluna de carimbo de data/hora separada para simultaneidade.

using System.ComponentModel.DataAnnotations;

public class Author
{
    public int AuthorId { get; set; }
     
    [ConcurrencyCheck]
    public string AuthorName { get; set; }
}

Do exemplo acima, o atributo ConcurrencyCheck é aplicado à propriedade AuthorName da classe Author. Portanto, Code-First incluirá a coluna AuthorName no comando de atualização (cláusula where) para verificar a simultaneidade otimista.
