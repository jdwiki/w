Existem duas maneiras gerais de especificar COMO o Entity Framework mapeará classes POCO para tabelas de banco de dados, colunas, etc.: Data Annotations e Fluent API.

Embora as anotações de dados sejam simples de ler e entender, elas não possuem certos recursos, como especificar o comportamento “Cascade on Delete” para uma entidade. A API Fluent, por outro lado, é um pouco mais complexa de usar, mas fornece um conjunto de recursos muito mais avançado.

Modelos de mapeamento #

Chave primária composta #

Usando o método .HasKey(), um conjunto de propriedades pode ser configurado explicitamente como a chave primária composta da entidade.

using System.Data.Entity;    
// ..

public class PersonContext : DbContext
{
    // ..

    protected override void OnModelCreating(DbModelBuilder modelBuilder)
    {
        // ..

        modelBuilder.Entity<Person>().HasKey(p => new { p.FirstName, p.LastName });
    }
}

Comprimento máximo #

Usando o método .HasMaxLength(), a contagem máxima de caracteres pode ser configurada para uma propriedade.

using System.Data.Entity;    
// ..

public class PersonContext : DbContext
{
    // ..

    protected override void OnModelCreating(DbModelBuilder modelBuilder)
    {
        // ..

        modelBuilder.Entity<Person>()
                    .Property(t => t.Name)
                    .HasMaxLength(100);
    }
}

Chave primária #

Usando o método .HasKey(), uma propriedade pode ser configurada explicitamente como chave primária da entidade.

using System.Data.Entity;    
// ..

public class PersonContext : DbContext
{
    // ..

    protected override void OnModelCreating(DbModelBuilder modelBuilder)
    {
        // ..

        modelBuilder.Entity<Person>().HasKey(p => p.PersonKey);
    }
}

Propriedades obrigatórias (NÃO NULL) #

Usando o método .IsRequired(), as propriedades podem ser especificadas como obrigatórias, o que significa que a coluna terá uma restrição NOT NULL.

using System.Data.Entity;    
// ..

public class PersonContext : DbContext
{
    // ..

    protected override void OnModelCreating(DbModelBuilder modelBuilder)
    {
        // ..

        modelBuilder.Entity<Person>()
                    .Property(t => t.Name)
                    .IsRequired();
    }
}

Nomenclatura de chave estrangeira explícita #

Quando uma propriedade de navegação existir em um modelo, o Entity Framework criará automaticamente uma coluna de chave estrangeira. Se um nome de chave estrangeira específico for desejado, mas não estiver contido como uma propriedade no modelo, ele poderá ser definido explicitamente usando a API Fluent. Ao utilizar o método Map ao estabelecer o relacionamento de Chave Estrangeira, qualquer nome exclusivo pode ser usado para Chaves Estrangeiras.

public class Company
{
    public int Id { get; set; }
}

public class Employee
{
    property int Id { get; set; }
    property Company Employer { get; set; }
}

public class EmployeeContext : DbContext
{
    protected override void OnModelCreating(DbModelBuilder modelBuilder)
    {
        modelBuilder.Entity<Employee>()
                    .HasRequired(x => x.Employer)
                    .WithRequiredDependent()
                    .Map(m => m.MapKey("EmployerId"));
    }
}

Após especificar o relacionamento, o método Map permite que o nome da chave estrangeira seja definido explicitamente executando MapKey. Neste exemplo, o que resultaria em um nome de coluna de Employer_Id agora é EmployerId.
