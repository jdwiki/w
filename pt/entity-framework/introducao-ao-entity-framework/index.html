Instalando o pacote NuGet do Entity Framework #

Em seu Visual Studio, abra a janela Solution Explorer e clique com o botão direito em seu projeto e escolha Gerenciar Pacotes NuGet no menu:

Na janela que se abre, digite EntityFramework na caixa de pesquisa no canto superior direito.

Ou se você estiver usando o Visual Studio 2015, verá algo assim:

Em seguida, clique em Instalar.

Também podemos instalar o framework de entidade usando o console do gerenciador de pacotes. Para fazer isso, você precisa primeiro abri-lo usando o menu Ferramentas -> Gerenciador de Pacotes NuGet -> Console do Gerenciador de Pacotes e digite isto:

Install-Package EntityFramework

Isso instalará o Entity Framework e adicionará automaticamente uma referência ao assembly em seu projeto.

Usando o Entity Framework do C# (Code First) #

O código primeiro permite que você crie suas entidades (classes) sem usar um designer de GUI ou um arquivo .edmx. Ele é chamado de Code first, porque você pode criar seus modelos first e Entity framework criará o banco de dados de acordo com os mapeamentos para você automaticamente. Ou você também pode usar essa abordagem com o banco de dados existente, que é chamado de codifique primeiro com o banco de dados existente Por exemplo, se você quiser que uma tabela contenha uma lista de planetas:

public class Planet
{
    public string Name { get; set; }
    public decimal AverageDistanceFromSun { get; set; }
}

Agora crie seu contexto que é a ponte entre suas classes de entidade e o banco de dados. Dê a ele uma ou mais propriedades DbSet<>:

using System.Data.Entity;

public class PlanetContext : DbContext
{
    public DbSet<Planet> Planets { get; set; }
}

Podemos usar isso fazendo o seguinte:

using(var context = new PlanetContext())
{
    var jupiter = new Planet 
    {
        Name = "Jupiter", 
        AverageDistanceFromSun = 778.5
    };

    context.Planets.Add(jupiter);
    context.SaveChanges();
}

Neste exemplo, criamos um novo Planet com a propriedade Name com o valor de "Júpiter" e a propriedade AverageDistanceFromSun com o valor de 778.5

Podemos então adicionar este Planet ao contexto usando o método Add() do DbSet e confirmar nossas alterações no banco de dados usando o método SaveChanges().

Ou podemos recuperar linhas do banco de dados:

using(var context = new PlanetContext())
{
    var jupiter = context.Planets.Single(p => p.Name == "Jupiter");
    Console.WriteLine($"Jupiter is {jupiter.AverageDistanceFromSun} million km from the sun.");
}

O que é Entity Framework? #

Escrever e gerenciar código ADO.Net para acesso a dados é um trabalho tedioso e monótono. A Microsoft forneceu uma estrutura O/RM chamada “Entity Framework” para automatizar as atividades relacionadas ao banco de dados para seu aplicativo.

A estrutura de entidade é uma estrutura de mapeamento de objeto/relacional (O/RM). É um aprimoramento do ADO.NET que oferece aos desenvolvedores um mecanismo automatizado para acessar e armazenar os dados no banco de dados.

O que é O/RM?

ORM é uma ferramenta para armazenar dados de objetos de domínio para o banco de dados relacional como MS SQL Server, de forma automatizada, sem muita programação. O/RM inclui três partes principais:

  1. Objetos de classe de domínio
  2. Objetos de banco de dados relacional
  3. Mapeamento de informações sobre como os objetos de domínio são mapeados para objetos de banco de dados relacionais (e.x tabelas, visualizações e procedimentos armazenados)

O ORM nos permite manter nosso design de banco de dados separado de nosso design de classe de domínio. Isso torna o aplicativo sustentável e extensível. Ele também automatiza a operação CRUD padrão (Criar, Ler, Atualizar e Excluir) para que o desenvolvedor não precise escrevê-lo manualmente.
