Entrar no Azure #

Modo clássico (Gerenciamento de serviços):

Add-AzureAccount

Isso o autenticará usando o Azure Active Directory e o PowerShell obterá um token de acesso que expira após cerca de 12 horas. Portanto, você deve repetir a autenticação após 12 horas.

Uma alternativa é executar o seguinte cmdlet:

Get-AzurePublishSettingsFile

Isso abre uma janela do navegador onde você pode baixar um arquivo de configurações de publicação. Este arquivo contém um certificado que permite a autenticação do PowerShell. Em seguida, você pode importar seu arquivo de configurações de publicação usando o seguinte:

Import-AzurePublishSettingsFile

Lembre-se de que o arquivo de configurações de publicação contém um certificado com privilégios de administrador efetivos em sua assinatura. Mantenha-o seguro ou exclua-o depois de usá-lo.

Gerente de Recursos

No lado do Resource Manager, só podemos usar a autenticação do Azure Active Directory com os tokens de acesso de 12 horas. Existem dois comandos alternativos atualmente que você pode usar:

Login-AzureRmAccount
Add-AzureRmAccount

Selecionando assinatura #

Quando você tem várias assinaturas em sua conta do Azure; é importante que você selecione o que deseja operar (e use-o como padrão); para evitar acidentes com recursos na assinatura errada.

Modo clássico

Set-AzureSubscription
Select-AzureSubscription

Gerente de Recursos

Select-AzureRmSubscription

Informações da assinatura

Os comandos acima solicitam que você especifique informações (por exemplo, o ID da assinatura) para identificar a assinatura para a qual deseja mudar. Para listar essas informações para as assinaturas às quais você tem acesso, execute este comando:

Get-AzureSubscription

Modo clássico vs modo ARM #

Ao trabalhar com o Azure usando o PowerShell, há 2 maneiras diferentes das quais você deve estar ciente e verá muita documentação da Microsoft referindo-se a ambas:

“Modo clássico (Gerenciamento de serviços)”

Essa é a maneira antiga de operar o Azure e gerenciar o Azure. Ainda existem alguns serviços no Azure que só podem ser gerenciados usando o modo clássico, embora mais e mais serviços estejam migrando para o novo modo ARM.

Para listar os módulos instalados em suas máquinas operando no modo clássico, você pode fazer o seguinte:

Get-Module -ListAvailable Azure.*

“Gerenciador de recursos (ARM)”

Esta é a nova maneira de gerenciar o Azure (com base nas APIs REST fornecidas). A maioria dos serviços que você pode gerenciar no Azure a partir do modo clássico do Powershell agora pode ser gerenciada usando o novo modo, com algumas exceções. Essa deve ser a maneira preferencial de gerenciar seus recursos do Azure, a menos que você tenha determinados serviços que ainda não tenham suporte no modo ARM.

Para listar os módulos instalados em sua máquina contendo comandos para operar no modo ARM, você pode fazer o seguinte:

Get-Module -ListAvailable AzureRM*

Obtenha a versão atual do Azure PowerShell #

Para determinar a versão do Azure PowerShell que você instalou, execute o seguinte:

Get-Module -ListAvailable -Name Azure -Refresh

Esse comando retorna a versão instalada mesmo quando você não carregou o módulo do Azure PowerShell em sua sessão atual do PowerShell.

Manipular ativos do Azure #

Os Cmdlets do Azure permitem que você execute algumas das mesmas ações em ativos do Azure por meio do PowerShell que você faria usando o código C# ou o portal do Azure.

Por exemplo, estas etapas permitem baixar o conteúdo de um blob do Azure em um diretório local:

New-Item -Path .\myblob -ItemType Directory
$context = New-AzureStorageContext -StorageAccountName MyAccountName -StorageAccountKey {key from the Azure portal}
$blob = Get-AzureStorageBlob -Container MyContainerName -Context $context
$blob | Get-AzureStorageBlobContent -Destination .\myblob\

Gerenciando Gerenciadores de Tráfego #

Com o Azure PowerShell, você pode obter determinadas funcionalidades atualmente indisponíveis no Portal do Azure, como:

  • Reconfigure todos os endpoints do Traffic Manager de uma só vez
  • Aborde outros serviços por meio do “ResourceId” do Azure em vez do nome de domínio, para que você não precise definir o local manualmente para os pontos de extremidade do Azure

Pré-requisitos #

Para começar, você precisa [login](https://www.wikiod.com/pt/azure/azure-powershell#Entrar no Azure) e [selecionar assinatura RM](http://stackoverflow.com/ documentação/azure/3961/azure-powershell/13825/selecting-subscription).

Obtenha o perfil do TrafficManager #

As operações com Gerenciadores de Tráfego via PowerShell são feitas em três etapas:

  1. Obtenha o perfil TM:
    $profile = Get-AzureRmTrafficManagerProfile -ResourceGroupName my-resource-group -Name my-traffic-manager
    Ou crie um novo como [neste artigo](https://docs. microsoft.com/en-us/powershell/resourcemanager/azurerm.trafficmanager/v2.2.0/new-azurermtrafficmanagerprofile).
  2. Explore e modifique o perfil TM
    Verifique os campos $profile e $profile.Endpoints para ver a configuração de cada endpoint.
  3. Salve as alterações via Set-AzureRmTrafficManagerProfile -TrafficManagerProfile $profile.

Alterar pontos de extremidade #

Todos os endpoints atuais são armazenados na lista $profile.Endpoints, então você pode alterá-los diretamente por índice
$profile.Endpoints[0].Weight = 100
ou por nome
$profile .Pontos de extremidade | ?{ $_.Name -eq 'meu-terminal' } | %{ $_.Peso = 100 }

Para limpar todos os endpoints, use
$profile.Endpoints.Clear()

Para excluir um endpoint específico, use
Remove-AzureRmTrafficManagerEndpointConfig -TrafficManagerProfile $profile -EndpointName 'my-endpoint'

Para adicionar um novo endpoint, use
Add-AzureRmTrafficManagerEndpointConfig -TrafficManagerProfile $profile -EndpointName "my-endpoint" -Type AzureEndpoints -TargetResourceId "/subscriptions/00000000-0000-0000-0000-000000000000/resourceGroups/my-resource-group/providers/Microsoft.ClassicCompute/domainNames/ my-azure-service" -EndpointStatus Enabled -Weight 100

Como você pode ver, no último caso, abordamos nosso serviço azure por meio de ResourceId em vez de nome de domínio.

Tenha em mente #

Suas alterações na TM e seus endpoints não são aplicadas até que você invoque Set-AzureRmTrafficManagerProfile -TrafficManagerProfile $profile. Isso permite reconfigurar totalmente a TM em uma operação.

O Gerenciador de Tráfego é uma implementação de DNS e o endereço IP dado aos clientes tem algum tempo de vida (também conhecido como TTL, você pode ver sua duração em segundos no arquivo $ campo profile.Ttl). Portanto, depois de reconfigurar a TM, alguns clientes continuarão a usar terminais antigos armazenados em cache até que o TTL expire.
