Força HTTPS #

.htaccess pode ser usado para forçar seu site HTTP a redirecionar para HTTPS.

Aqui está uma maneira rápida que não requer a edição do código do seu domínio:

RewriteEngine On
RewriteCond %{HTTPS} =off
RewriteRule ^ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

Aviso: O código acima assume que você pode confiar em %{HTTP_HOST} para apontar para seu domínio.

Se você precisar ter certeza de que o local de redirecionamento é seu domínio, substitua %{HTTP_HOST} por seu domínio.

O código acima faz isso:

  1. Ative [RewriteEngine][1].
  2. Continue se a solicitação atual não estiver usando HTTPS.
  3. Faça um redirecionamento HTTP 301 para https://%{HTTP_HOST}%{REQUEST_URI}, onde
    • %{HTTP_HOST} is the host requested by the browser and
    • %{REQUEST_URI} is the URI requested by the browser (everything after the domain).

Aviso: seu aplicativo da web deve ser capaz de lidar com solicitações HTTPS, e o Apache para seu host deve ser configurado com um certificado de site válido.

Observe que é significativamente mais eficiente simplesmente fazer um Redirect no http vhost do que fazer essas comparações múltiplas por solicitação em um arquivo .htaccess. Veja http://wiki.apache.org/httpd/RedirectSSL para uma discussão mais aprofundada desta técnica.

[1]: https://www.wikiod.com/pt/apache/arquivos-htaccess-no-apache#Reescrever mecanismo

Reescrever mecanismo #

O módulo RewriteEngine dentro do Apache é usado para reescrever dinamicamente URLs e caminhos dependendo das várias expressões fornecidas:

<IfModule mod_rewrite.c>
RewriteEngine On
RewriteBase /
RewriteRule ^index\.php$ - [L]
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.php [END]
</IfModule>

As regras acima irão reescrever os arquivos PHP para que não mostrem mais sua extensão, e então index.php irá apenas mostrar como um domínio nu (semelhante ao comportamento normalmente visto em index.html). A regra acima é fornecida com o WordPress.

Observe que no Apache httpd 2.2.16 e posterior, esse bloco inteiro pode ser substituído por uma única linha usando a diretiva FallbackResource:

FallbackResource /index.php

Ativar CORS #

Para habilitar o Cross-Origin Resource Sharing (CORS) no Apache, você precisará definir pelo menos um cabeçalho HTTP que o altere (o comportamento padrão é bloquear o CORS). No exemplo a seguir, vamos definir este cabeçalho HTTP dentro de .htaccess, mas ele também pode ser definido no arquivo your-site.conf do seu site ou no arquivo de configuração do Apache. Independentemente da aparência de sua configuração, você pode definir os cabeçalhos HTTP relevantes em qualquer bloco de configuração do Apache, ou seja, <VirtualHost>, <Directory>, <Location> e <Files>.

Existem alguns cabeçalhos HTTP relacionados ao CORS que você pode retornar na resposta:

Access-Control-Allow-Origin

Access-Control-Allow-Credentials
Access-Control-Allow-Methods
Access-Control-Max-Age
Access-Control-Allow-Headers
Access-Control-Expose-Headers

Alguns dos itens acima são necessários para solicitações de “comprovação”. Alguns clientes HTTP (ou seja, navegadores modernos) executam uma solicitação antes da solicitação desejada apenas para ver se eles têm autorização para fazer a solicitação real no servidor. Consulte https://en.wikipedia.org/wiki/Cross-origin_resource_sharing para obter mais informações sobre a solicitação de comprovação.

O cabeçalho HTTP principal que precisamos é Access-Control-Allow-Origin e é isso que vamos definir. No entanto, o mesmo princípio se aplica a praticamente todos eles (você só precisa saber o que devolver).

O exemplo a seguir define o cabeçalho HTTP necessário em um bloco de configuração <Directory> para habilitar um nome de domínio qualificado completo (FQDN) de cliente protegido por SSL:

<Directory /path/to/your/site/>
        Header set Access-Control-Allow-Origin "https://my.CLIENT.domain"
</Directory>

Depois de definir isso no servidor, agora podemos realizar uma solicitação de https://my.client.domain para nosso servidor e ele deve responder.

Nota: Muitas pessoas usam Access-Control-Allow-Origin: "*" que é um curinga, para significar que solicitações de TODOS domínios devem ser aceitas. Isso geralmente é desaconselhável, a menos que você esteja executando algum tipo de API pública ou repositório de arquivos. Além disso, observe o contexto da configuração do seu cabeçalho HTTP. Você pode querer permitir solicitações HTTP para uma API, mas não para “hotlinking” de imagens etc. Você pode definir esse cabeçalho em qualquer lugar que desejar em seu fluxo de configuração do Apache para somente configurá-lo em situações específicas. Por exemplo, o seguinte somente definiria o cabeçalho HTTP CORS quando o caminho solicitado não for um arquivo ou diretório (adequa-se a uma API pública que não permite hotlinking de imagem):

<Directory /path/to/your/site/>
    Options +FollowSymlinks
    Options +Indexes
    RewriteEngine On

    #Make sure it's not a specific file or directory that they're trying to reach
    RewriteCond %{SCRIPT_FILENAME} !-f
    RewriteCond %{SCRIPT_FILENAME} !-d
    Header set Access-Control-Allow-Origin "*"
    RewriteRule ^(.*)$ index.php/$1 [L]
</Directory>

Pré-requisitos #

Você precisa ter mod_headers instalado e habilitado: a2enmod headers

Redirecionamento 301 por Htaccess #

O código de status de resposta HTTP 301 Moved Permanently é usado para redirecionamento de URL permanente, o que significa que os links ou registros atuais usando o URL para o qual a resposta é recebida devem ser atualizados. A nova URL deve ser fornecida no campo Local incluído na resposta. O redirecionamento 301 é considerado uma prática recomendada para atualizar usuários de HTTP para HTTPS. escreva este código no arquivo htaccess para PHP-APACHE

Redirect 301 /oldpage/ /newpage/

Aqui está um exemplo usando um arquivo htaccess para redirecionar para um não www com um SSL anexado ao domínio.

RewriteEngine On
RewriteCond %{HTTPS} off
RewriteCond %{HTTP_HOST} ^www\.(.*)$ [NC]
RewriteRule ^(.*)$ http://%1/$1 [R=301,L]

RewriteCond %{HTTPS} on
RewriteCond %{HTTP_HOST} ^www\.(.*)$ [NC]
RewriteRule ^(.*)$ https://%1/$1 [R=301,L]

RewriteEngine On
RewriteCond %{SERVER_PORT} 80
RewriteRule ^(.*)$ https://example.com/$1 [R,L]
