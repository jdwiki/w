Visão geral de todo o processo #

  1. Baixe o código-fonte mais recente.
  2. Configure o kernel.
  3. Construa o mundo e o kernel.
  4. Configure o sistema de arquivos raiz do seu novo FreeBSD.
  5. Instale o mundo e o kernel.

Obtém o número de processadores #

Uma maneira fácil de acelerar o processo de construção e instalação do novo sistema é usar mais processadores para aumentar o poder computacional.

Para descobrir qual é o número de processadores que você tem que acelerar o processo:

sysctl hw.ncpu

Por exemplo:

hw.ncpu: 1

Vamos definir a variável de ambiente $NUMBER_OF_PROCESSORS então:

export $NUMBER_OF_PROCESSORS=$(sysctl hw.ncpu | tr -d 'a-z.: ')

Baixe o código-fonte mais recente #

SVN #

O projeto FreeBSD usa SVN como SCM padrão. A fonte pode ser baixada com o software svnlite.

Atualize #

cd /usr/src
svnlite checkout https://svn.freebsd.org/base/head .

Obter lançamentos #

cd /usr/src
svnlite checkout https://web.freebsd.org/base/release/11.0.0 .

Tarball (http e ftp) #

Você também pode obter a fonte do tarball congelado com o comando fetch

http #

cd /tmp
fetch http://ftp.freebsd.org/pub/FreeBSD/releases/amd64/11.0-RELEASE/src.txz
cd /usr/src
tar xJvf /tmp/src.txz

##ftp

cd /tmp
fetch ftp://ftp.freebsd.org/pub/FreeBSD/releases/amd64/11.0-RELEASE/src.txz
cd /usr/src
tar xJvf /tmp/src.txz

Git #

GitHub #

git clone https://github.com/freebsd/freebsd freebsdsrc

Configure o kernel #

  1. Vá para o diretório com o código fonte:
cd freebsdsrc
  1. Acesse o diretório com o código de configuração do kernel:
# Se o seu sistema for de 32 bits.
cd sys/i386/conf/
# Se o seu sistema for de 64 bits.
cd sys/amd64/conf/
  1. Obtenha uma cópia do kernel GENERIC (vamos chamá-lo de MODEDKERNEL). Será a base de suas personalizações.
cp GENÉRICO MODEDKERNEL
  1. Modifique o arquivo MODEDKERNEL à sua vontade.

Construa o mundo e o kernel #

Construir o mundo #

Vá para o freebsdsrc/ (o diretório raiz da árvore de fontes do FreeBSD que você já baixou) e construa o mundo:

sudo make -j${NUMBER_OF_PROCESSORS} buildworld KERNCONF=MODEDKERNEL -DNO_CLEAN

Tempo estimado #

  • Tempo estimado no Hasee Q540S rodando em um processador: 8 horas.
  • Tempo estimado no Dell L702X rodando em 8 processadores: 98 minutos.

Construir o kernel #

Para construir o kernel execute:

sudo make -j${NUMBER_OF_PROCCESORS} buildkernel KERNCONF=UFFIE -DNO_CLEAN

Tempo estimado #

  • Tempo estimado no Hasee Q540S rodando em um processador: 2 horas.
  • Tempo estimado no Dell L702X rodando em 8 processadores: 19 minutos.

Configure o sistema de arquivos raiz do seu novo FreeBSD #

Vamos configurar o diretório de destino para o sistema de arquivos raiz de seu novo FreeBSD (por exemplo /usr/home/beastie/MODEDKERNEL).

  1. Adicione as seguintes linhas ao /etc/src.conf para configurá-lo:
.if ${KERNCONF} == "MODEDKERNEL"
       DESTDIR?=/usr/home/beastie/MODEDKERNEL
       MODULES_OVERRIDE=md ufs
.fim se

Lembre-se de usar espaços e não tabulações se desejar recuar o código.

  1. Crie o sistema de arquivos raiz agora:

  • Faça diretórios de distribuição:

    sudo make distrib-dirs KERNCONF=MODEDKERNEL

    Estimated time on Hasee Q540S: a few seconds.

  • Faça a distribuição:

    sudo make distribution KERNCONF=UFFIE

    Estimated time on Hasee Q540S: 3 minutes.

Instale o mundo e o kernel #

#Instala o mundo

sudo make installworld KERNCONF=MODEDKERNEL

Tempo estimado em Hasee Q540S: 5 minutos.

#Instala o kernel

sudo make installkernel KERNCONF=MODEDKERNEL

Tempo estimado no Hasee Q540S: alguns segundos.
