Ação de desconto do produto #

Um dia eu conversei com um amigo meu que usa o framework PHP Laravel em seu trabalho. Quando eu disse a ele que o Django tem seu próprio sistema HTML CRUD com tudo incluído, para interagir com o banco de dados, chamado Django admin, seus olhos saltaram! Ele me disse: “*Levei meses para construir uma interface Admin para meu aplicativo da web atual e você está dizendo que tem tudo isso sem ter que escrever uma única linha de código?”. Eu respondi “Sim!”

O administrador do Django é um recurso poderoso do Django que fornece muitos recursos. Uma delas são ações.

Mas o que são “ações”?

Suponha que você tenha um modelo e já tenha adicionado algumas entradas (talvez centenas, talvez milhares) nele. Agora, você deseja aplicar uma ação de regra a pelo menos um deles, sem usar o console (python manage.py shell):

# python manage.py shell (interactive console)

from math import ceil

from my_app.models import Product


DISCOUNT = 10  # percentage

for product in Product.objects.filter(is_active=True):
    """ Set discount to ALL products that are flagged as active """
    multiplier = DISCOUNT / 100.  # DISCOUNT / 100 in python 3 (without dot)
    old_price = product.price
    new_price = ceil(old_price - (old_price * multiplier))  # seller wins :)
    product.price = new_price
    product.save(update_fields=['price'])

Você notou que aplicamos o desconto em todos os produtos. E se quiséssemos aplicar essa lógica a outras específicas? Ou se quiséssemos inserir manualmente o valor do desconto e depois aplicar esse valor em alguns produtos? Tudo isso através do admin do Django! Você está no caminho certo. Ações do Django FTW. Vejamos um exemplo completo.

O básico: #

  • Python 3.4.3
  • Django 1.10
  • SQLite (construído em Python, sem necessidade de configuração de instalação extra)

O modelo: #

  • Representando um produto para nossa loja virtual
  • O preço de um produto seria números inteiros (não decimais)

O objetivo: #

  • Poder, através da interface do Django Admin, aplicar um desconto fixo em uma ou mais entradas de produtos.

Configuração (aplicativo, modelo e administrador do Django) #

Supondo que você já tenha iniciado um projeto, vá para o diretório onde o manage.py reside e crie um aplicativo com o nome stock, digitando:

python manage.py createapp stock

O Django criará automaticamente uma estrutura de diretórios para você. Vá e edite o arquivo models.py e adicione:

# models.py

from django.db import models


class Product(models.Model):
    name = models.CharField('Product name', max_length=100)  # required field
    price = models.PositiveIntegerField('Product price')

    def __str__(self):  # __unicode__ in Python 2
        return self.name

Nosso modelo Product foi criado, mas nada no banco de dados ainda. Para que as migrações funcionem, nosso aplicativo deve ser incluído na lista INSTALLED_APPS.

Edite seu arquivo settings.py e na lista INSTALLED_APPS adicione:

# settings.py

INSTALLED_APPS = [
    # ... previous Django apps
    'stock.apps.StockConfig',
]

Agora execute:

python manage.py makemigrations
python manage.py migrate

Após o comando migrate, seu banco de dados agora tem uma tabela chamada product que contém três colunas, id, name e price. Até agora tudo bem!

Se você não alterou nada em seu arquivo ROOT_URLCONF, que geralmente fica dentro da pasta <your_project_name>/< your_project_name>/, então a URL que aponta para o site de administração do Django deve ser:

# urls.py

urlpatterns = [
    # ... other URLs
    url(r'^admin/', admin.site.urls),
]

Até agora, não procuramos nada específico sobre as ações de administração do Django. Dê um passo final e adicione-os dentro do seu arquivo stock/admin.py:

# admin.py

from django.contrib import admin

from .models import Product


@admin.register(Product)
class ProductAdmin(admin.ModelAdmin):
    pass

OK. A configuração está feita. Apenas para certificar-se de que tudo funciona, execute:

python manage.py runserver

e com seu navegador favorito visite a página 127.0.0.1:8000/admin/. Você deve ver a página de administração do Django brilhante-glamorous-terrific, que permite que você Create-Read-Update-Delete seu modelo Product! Se, por acaso, a página acima solicitar um nome de usuário/senha e você não tiver nenhum, não há problema. Você simplesmente não criou um Usuário para fazer login no admin. Basta executar:

python manage.py createsuperuser

digite seu nome, e-mail, nome de usuário e senha (duas vezes) e pronto.


Gere alguns produtos falsos #

Até agora, criamos o modelo, mas ainda não há entradas (sem produtos). Precisamos de algumas entradas para esclarecer o poder das ações de administração do Django.

Vamos criar 100 produtos e trabalhar com eles. Mas, em vez de pressionar manualmente o botão ADICIONAR e inserir um ’nome’ e um ‘preço’, escreveremos um script para fazer o trabalho para nós.

Execute python manage.py shell e digite o seguinte:

# python manage.py shell

from stock.models import Product    


for i in range(1, 101):
    p = Product.objects.create(name='Product %s' % i, price=i)

O loop for acima cria (o que significa que os dados são salvos no banco de dados) 100 produtos (entradas) com nomes Produto 1, Produto 2, … Produto 100 e preços 1, 2 , …, 100.

Para visualizar esses produtos através da página de administração do Django, basta visitar novamente 127.0.0.1:8000/admin/ e clicar no link Produtos:

Aproveite seus 100 produtos gerados automaticamente:

Isso é conhecido como a página change list do Django. Agora, para que isso fique muito mais bonito, edite seu arquivo stock/admin.py e digite:

@admin.register(Product)
class ProductAdmin(admin.ModelAdmin):
    list_display = ('name', 'price')

Agora, clique em atualizar e você verá uma segunda coluna exibindo os preços.


Ufa! Finalize as ações #

Para recapitular, temos o modelo e temos as entradas. Em seguida, queremos criar uma ação que, uma vez selecionada, fará um desconto de 30% no(s) produto(s) selecionado(s).

Você notou que já existe uma caixa de seleção no topo da página change list, com o rótulo Action? Django, adiciona automaticamente uma ação padrão em cada entrada que executa a ação Delete.

Django admin actions são escritas como funções simples. Vamos mergulhar. Edite o arquivo stock/admin.py e adicione o seguinte:

# admin.py

@admin.register(Product)
class ProductAdmin(admin.ModelAdmin):
    list_display = ('name', 'price')
    actions = ['discount_30']

    def discount_30(self, request, queryset):
        from math import ceil
        discount = 30  # percentage

        for product in queryset:
            """ Set a discount of 30% to selected products """
            multiplier = discount / 100.  # discount / 100 in python 3
            old_price = product.price
            new_price = ceil(old_price - (old_price * multiplier))
            product.price = new_price
            product.save(update_fields=['price'])
    discount_30.short_description = 'Set 30%% discount'

Algumas coisas a serem observadas aqui:

  • A classe ProductAdmin tem um atributo extra (actions) que é uma lista de strings (cada string é o nome da função que representa a ação).
  • A função de ação que é um método da classe ProductAdmin. Ele recebe como argumentos a instância ModelAdmin (self já que este é um método), o objeto HTTP request e o queryset (uma lista dos objetos-entradas-produtos selecionados).
  • A última linha é um atributo de função (short_description) que define o nome exibido dentro da caixa de seleção de ações (há um % duplo lá para escapar do %).

Dentro da função-ação, iteramos em cada produto (que foi selecionado) e definimos seu valor reduzido em 30%. Então chamamos o método save() com o argumento [update_fields](https:/ /docs.djangoproject.com/en/dev/ref/models/instances/#specifying-which-fields-to-save) para forçar um UPDATE nos campos que estão incluídos na lista update_fields (em vez de um UPDATE em todos os campos do modelo) no banco de dados, por questões de desempenho (não é um ganho de desempenho neste exemplo, com apenas 2 colunas, mas você entendeu).

Agora, clique em atualizar na página change list e você deverá ver sua ação na página delete. Vá em frente e selecione alguns produtos (usando a caixa de seleção à esquerda de cada um ou todos os produtos usando a caixa de seleção mais à esquerda), selecione a ação Definir 30% de desconto e clique no botão Ir. É isso!

Claro, isso não é muito útil na maioria das situações, pois essa ação não permite que você insira um valor diferente de desconto. Você deve editar o arquivo admin.py cada vez que quiser aplicar um desconto diferente. No próximo exemplo veremos como fazer isso.
