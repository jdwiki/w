Usando parfor para paralelizar um loop #

Você pode usar parfor para executar as iterações de um loop em paralelo:

Exemplo:

poolobj = parpool(2);       % Open a parallel pool with 2 workers 

s = 0;                      % Performing some parallel Computations
parfor i=0:9
    s = s + 1;
end
disp(s)                     % Outputs '10'

delete(poolobj);            % Close the parallel pool

Nota: parfor não pode ser aninhado diretamente. Para o aninhamento de parfor use uma função no primeiro parfor e adicione o segundo parfor nessa função.

Exemplo:

parfor i = 1:n
[op] = fun_name(ip);
end

function [op] = fun_name(ip)
parfor j = 1:length(ip)
% Some Computation
end

Quando usar parfor #

Basicamente, parfor é recomendado em dois casos: muitas iterações em seu loop (por exemplo, 1e10), ou se cada iteração demorar muito tempo (por exemplo, eig(magic(1e4) )). No segundo caso, você pode considerar usar spmd . O motivo do parfor ser mais lento que um loop for para intervalos curtos ou iterações rápidas é a sobrecarga necessária para gerenciar todos os trabalhadores corretamente, em vez de apenas fazer o cálculo.

Além disso, muitas funções têm [multi-threading implícito embutido] 3, tornando um loop parfor não mais eficiente, ao usar essas funções, do que um loop serial for, já que todos os núcleos já estão sendo usados. parfor será realmente um prejuízo neste caso, já que tem a sobrecarga de alocação, sendo tão paralelo quanto a função que você está tentando usar.

Considere o exemplo a seguir para ver o comportamento de for em oposição ao de parfor. Primeiro, abra o pool paralelo, se ainda não o fez:

gcp; % Opens a parallel pool using your current settings

Em seguida, execute alguns loops grandes:

n = 1000; % Iteration number
EigenValues = cell(n,1); % Prepare to store the data
Time = zeros(n,1);
for ii = 1:n
tic
    EigenValues{ii,1} = eig(magic(1e3)); % Might want to lower the magic if it takes too long
Time(ii,1) = toc; % Collect time after each iteration
end

figure; % Create a plot of results
plot(1:n,t)
title 'Time per iteration'
ylabel 'Time [s]'
xlabel 'Iteration number[-]';

Então faça o mesmo com parfor em vez de for. Você notará que o tempo médio por iteração aumenta. No entanto, perceba que o parfor usou todos os trabalhadores disponíveis, portanto, o tempo total (sum(Time)) deve ser dividido pelo número de núcleos em seu computador.

Assim, enquanto o tempo para fazer cada iteração separada aumenta usando parfor em relação ao uso de for, o tempo total diminui consideravelmente.

Executando comandos em paralelo usando uma instrução “Single Program, Multiple Data” (SPMD) #

Ao contrário de um loop for paralelo (parfor), que recebe as iterações de um loop e as distribui entre várias threads, uma instrução de programa único, vários dados (spmd) recebe uma série de comandos e os distribui para **todos ** as threads, para que cada thread execute o comando e armazene os resultados. Considere isto:

poolobj = parpool(2);    % open a parallel pool with two workers

spmd
    q = rand(3);         % each thread generates a unique 3x3 array of random numbers
end

q{1}             % q is called like a cell vector
q{2}             % the values stored in each thread may be accessed by their index

delete(poolobj)  % if the pool is closed, then the data in q will no longer be accessible

É importante notar que cada thread pode ser acessada durante o bloco spmd por seu índice de thread (também chamado de índice de laboratório, ou labindex):

poolobj = parpool(2);        % open a parallel pool with two workers

spmd
    q = rand(labindex + 1);  % each thread generates a unique array of random numbers
end

size(q{1})                   % the size of q{1} is 2x2
size(q{2})                   % the size of q{2} is 3x3

delete(poolobj)              % q is no longer accessible

Em ambos os exemplos, q é um objeto composto, que pode ser inicializado com o comando q ​​= Composite(). É importante observar que os objetos compostos são acessíveis apenas enquanto o pool está em execução.

Usando o comando batch para fazer vários cálculos em paralelo #

Para usar multi-threading no MATLAB pode-se usar o comando batch. Observe que você deve ter a caixa de ferramentas Parallel Computing instalada.

Para um script demorado, por exemplo,

for ii=1:1e8
   A(ii)=sin(ii*2*pi/1e8);
end

para executá-lo no modo de lote, usaria o seguinte:

job=batch("da")

que permite que o MATLAB seja executado no modo de lote e torna possível usar o MATLAB enquanto isso para fazer outras coisas, como adicionar mais processos em lote.

Para recuperar os resultados após terminar o trabalho e carregar o array A no espaço de trabalho:

load(job, 'A')

Por fim, abra o “monitor job gui” em Home → Environment → Parallel → Monitor jobs e exclua o job através de:

delete(job)

Para carregar uma função para processamento em lote, simplesmente use esta instrução onde fcn é o nome da função, N é o número de arrays de saída e x1, ..., xn são arrays de entrada:

 j=batch(fcn, N, {x1, x2, ..., xn})
