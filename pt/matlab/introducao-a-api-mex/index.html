Verifica o número de entradas/saídas em um arquivo C++ MEX #

Neste exemplo vamos escrever um programa básico que verifica o número de entradas e saídas passadas para uma função MEX.

Como ponto de partida, precisamos criar um arquivo C++ implementando o “gateway MEX”. Esta é a função executada quando o arquivo é chamado do MATLAB.

##testinputs.cpp ##

// MathWorks provided header file
#include "mex.h"

// gateway function
void mexFunction(int nlhs, mxArray *plhs[], int nrhs, const mxArray *prhs[])
{
    // This function will error if number of inputs its not 3 or 4
    // This function will error if number of outputs is more than 1

    // Check inputs:
    if (nrhs < 3 || nrhs > 4) {
        mexErrMsgIdAndTxt("Testinputs:ErrorIdIn",
            "Invalid number of inputs to MEX file.");
    }

    // Check outputs:
    if (nlhs > 1) {
        mexErrMsgIdAndTxt("Testinputs:ErrorIdOut",
                "Invalid number of outputs to MEX file.");
    }
}

Primeiro, incluímos o cabeçalho mex.h que contém definições de todas as funções e tipos de dados necessários para trabalhar com a API MEX. Então implementamos a função mexFunction como mostrado, onde sua assinatura não deve mudar, independente das entradas/saídas realmente utilizadas. Os parâmetros da função são os seguintes:

  • nlhs: Número de saídas solicitadas.
  • *plhs[]: Array contendo todas as saídas no formato MEX API.
  • nrhs: Número de entradas passadas.
  • *prhs[]: Array contendo todas as entradas no formato MEX API.

Em seguida, verificamos o número de argumentos de entradas/saídas e, se a validação falhar, um erro é lançado usando a função mexErrMsgIdAndTxt (ela espera o identificador de formato somename:iD, um simples “ID” não funcionará).

Uma vez que o arquivo é compilado como mex testinputs.cpp, a função pode ser chamada no MATLAB como:

>> testinputs(2,3)
Error using testinputs. Invalid number of inputs to MEX file.

>> testinputs(2,3,5)

>> [~,~] = testinputs(2,3,3)
Error using testinputs. Invalid number of outputs to MEX file.

Insira uma string, modifique-a em C e imprima-a #

Neste exemplo, ilustramos a manipulação de strings no MATLAB MEX. Vamos criar uma função MEX que aceita uma string como entrada do MATLAB, copiar os dados para a string C, modificá-lo e convertê-lo de volta para mxArray retornado para o lado do MATLAB.

O principal objetivo deste exemplo é mostrar como strings podem ser convertidas para C/C++ a partir do MATLAB e vice-versa.

stringIO.cpp #

#include "mex.h"
#include <cstring>

void mexFunction(int nlhs, mxArray *plhs[], int nrhs, const mxArray *prhs[])
{
    // check number of arguments
    if (nrhs != 1 || nlhs > 1) {
        mexErrMsgIdAndTxt("StringIO:WrongNumArgs", "Wrong number of arguments.");
    }

    // check if input is a string
    if (mxIsChar(prhs[0])) {
        mexErrMsgIdAndTxt("StringIO:TypeError", "Input is not a string");
    }

    // copy characters data from mxArray to a C-style string (null-terminated)
    char *str = mxArrayToString(prhs[0]);

    // manipulate the string in some way
    if (strcmp("theOneString", str) == 0) {
        str[0] = 'T';  // capitalize first letter
    } else {
        str[0] = ' ';  // do something else?
    }

    // return the new modified string
    plhs[0] = mxCreateString(str);

    // free allocated memory
    mxFree(str);
}

As funções relevantes neste exemplo são:

  • mxIsChar para testar se um mxArray é do tipo mxCHAR.
  • mxArrayToString para copiar os dados de uma string mxArray para um buffer char *.
  • mxCreateString para criar uma string mxArray a partir de um char*.

Como uma nota lateral, se você quiser apenas ler a string, e não modificá-la, lembre-se de declará-la como const char* para velocidade e robustez.

Finalmente, uma vez compilado, podemos chamá-lo do MATLAB como:

>> mex stringIO.cpp

>> strOut = stringIO('theOneString')
strOut = 
TheOneString

>> strOut = stringIO('somethingelse')
strOut=
omethingelse

Passando uma estrutura por nomes de campo #

Este exemplo ilustra como ler entradas de estrutura de vários tipos do MATLAB e passá-las para variáveis ​​de tipo equivalentes em C.

Embora seja possível e fácil descobrir a partir do exemplo como carregar campos por números, aqui é obtido comparando os nomes dos campos com as strings. Assim, os campos struct podem ser endereçados por seus nomes de campo e variáveis ​​nele podem ser lidas por C.

structIn.c #

#include "mex.h"
#include <string.h> // strcmp


void mexFunction (int nlhs, mxArray *plhs[],
                  int nrhs, const mxArray *prhs[])
{
  // helpers
  double* double_ptr;
  unsigned int i; // loop variable
  
  // to be read variables
  bool optimal;
  int randomseed;
  unsigned int desiredNodes;

  if (!mxIsStruct(prhs[0])) {
    mexErrMsgTxt("First argument has to be a parameter struct!");
  }
  for (i=0; i<mxGetNumberOfFields(prhs[0]); i++) {
    if (0==strcmp(mxGetFieldNameByNumber(prhs[0],i),"randomseed")) {
      mxArray *p = mxGetFieldByNumber(prhs[0],0,i);
      randomseed = *mxGetPr(p);
    }
    if (0==strcmp(mxGetFieldNameByNumber(prhs[0],i),"optimal")) {
      mxArray *p = mxGetFieldByNumber(prhs[0],0,i);
      optimal = (bool)*mxGetPr(p);
    }
    if (0==strcmp(mxGetFieldNameByNumber(prhs[0],i),"numNodes")) {
      mxArray *p = mxGetFieldByNumber(prhs[0],0,i);
      desiredNodes = *mxGetPr(p);
    }
  }
}

O loop sobre i percorre todos os campos na estrutura fornecida, enquanto as partes if(0==strcmp) comparam o nome do campo matlab com a string fornecida. Se for uma correspondência, o valor correspondente é extraído para uma variável C.

Passe uma matriz 3D do MATLAB para C #

Neste exemplo, ilustramos como pegar uma matriz 3D de tipo real duplo do MATLAB e passá-la para um array C double*.

Os principais objetivos deste exemplo são mostrar como obter dados de arrays MATLAB MEX e destacar alguns pequenos detalhes no armazenamento e manuseio de matrizes.

matrizIn.cpp #

#include "mex.h"

void mexFunction(int  nlhs , mxArray *plhs[],
        int nrhs, mxArray const *prhs[]){
   // check amount of inputs
   if (nrhs!=1) {
        mexErrMsgIdAndTxt("matrixIn:InvalidInput", "Invalid number of inputs to MEX file.");
    }
   
   // check type of input
   if( !mxIsDouble(prhs[0]) || mxIsComplex(prhs[0])){
        mexErrMsgIdAndTxt("matrixIn:InvalidType",  "Input matrix must be a double, non-complex array.");
   }
   
   // extract the data
   double const * const matrixAux= static_cast<double const *>(mxGetData(prhs[0]));

   // Get matrix size
   const mwSize *sizeInputMatrix= mxGetDimensions(prhs[0]);

   // allocate array in C. Note: its 1D array, not 3D even if our input is 3D
   double*  matrixInC= (double*)malloc(sizeInputMatrix[0] *sizeInputMatrix[1] *sizeInputMatrix[2]* sizeof(double));

  
   // MATLAB is column major, not row major (as C). We need to reorder the numbers
   // Basically permutes dimensions   

   // NOTE: the ordering of the loops is optimized for fastest memory access! 
   // This improves the speed in about 300% 

    const int size0 = sizeInputMatrix[0]; // Const makes compiler optimization kick in
    const int size1 = sizeInputMatrix[1];
    const int size2 = sizeInputMatrix[2];
    
    for (int j = 0; j < size2; j++)
    {
        int jOffset = j*size0*size1; // this saves re-computation time
        for (int k = 0; k < size0; k++)
        {
            int kOffset = k*size1; // this saves re-computation time
            for (int i = 0; i < size1; i++)
            {
                int iOffset = i*size0; 
                matrixInC[i + jOffset + kOffset] = matrixAux[iOffset + jOffset + k];
            }
        }
    }

    // we are done!

    // Use your C matrix here

    // free memory
    free(matrixInC);
    return;
}

Os conceitos relevantes a ter em conta:

  • Matrizes MATLAB são todas 1D na memória, não importa quantas dimensões elas tenham quando usadas no MATLAB. Isso também é verdade para a maioria (se não todas) das representações de matriz principal em bibliotecas C/C++, pois permite otimização e execução mais rápida.

  • Você precisa copiar explicitamente as matrizes do MATLAB para C em um loop.

  • Matrizes MATLAB são armazenadas em ordem de coluna principal, como em Fortran, mas C/C++ e linguagens mais modernas são de linha principal. É importante permutar a matriz de entrada, ou então os dados ficarão completamente diferentes.

As funções relevantes neste exemplo são:

  • mxIsDouble verifica se a entrada é do tipo double.
  • mxIsComplex verifica se a entrada é real ou imaginária.
  • mxGetData retorna um ponteiro para os dados reais na matriz de entrada. NULL se não houver dados reais.
  • mxGetDimensions retorna um ponteiro para um array mwSize, com o tamanho da dimensão em cada índice.
