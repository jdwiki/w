Círculos #

A opção mais fácil para desenhar um círculo é - obviamente - a função rectangle.

%// radius
r = 2;

%// center
c = [3 3];

pos = [c-r 2*r 2*r];
rectangle('Position',pos,'Curvature',[1 1])
axis equal

mas a curvatura do retângulo deve ser definida como 1!

O vetor position define o retângulo, os dois primeiros valores x e y são o canto inferior esquerdo do retângulo. Os dois últimos valores definem a largura e a altura do retângulo.

pos = [ [x y] width height ]

O canto inferior esquerdo do círculo - sim, este círculo tem cantos, embora imaginários - é o centro c = [3 3] menos o raio r = 2 que é [x y] = [1 1]. Largura e altura são iguais ao diâmetro do círculo, então width = 2*r; altura = largura;

Caso a suavidade da solução acima não seja suficiente, não há como usar a maneira óbvia de desenhar um círculo real usando funções trigonométricas.

%// number of points
n = 1000;

%// running variable
t = linspace(0,2*pi,n);

x = c(1) + r*sin(t);
y = c(2) + r*cos(t);

%// draw line
line(x,y)

%// or draw polygon if you want to fill it with color
%// fill(x,y,[1,1,1])
axis equal

Setas; flechas #

Em primeiro lugar, pode-se usar quiver, onde não é necessário lidar com unidades de figura normalizadas inúteis usando anotação

drawArrow = @(x,y) quiver( x(1),y(1),x(2)-x(1),y(2)-y(1),0 )    

x1 = [10 30];
y1 = [10 30];

drawArrow(x1,y1); hold on

x2 = [25 15];
y2 = [15 25];

drawArrow(x2,y2)

Importante é o 5º argumento de quiver: 0 que desabilita uma escala padrão, já que esta função é normalmente usada para plotar campos vetoriais. (ou use o par de valores de propriedade 'AutoScale','off')

Pode-se também adicionar recursos adicionais:

drawArrow = @(x,y,varargin) quiver( x(1),y(1),x(2)-x(1),y(2)-y(1),0, varargin{:} )       
drawArrow(x1,y1); hold on
drawArrow(x2,y2,'linewidth',3,'color','r')

Se forem desejadas pontas de seta diferentes, é necessário usar anotações (esta resposta pode ser útil [Como faço para alterar o estilo da ponta da seta no quiver plot?](http://stackoverflow.com/questions/18776172/in-matlab-how- faça-mudar-o-estilo-cabeça-de-seta-em-quiver-plot)).

O tamanho da ponta da seta pode ser ajustado com a propriedade 'MaxHeadSize'. Não é consistente, infelizmente. Os limites dos eixos precisam ser definidos posteriormente.

x1 = [10 30];
y1 = [10 30];
drawArrow(x1,y1,{'MaxHeadSize',0.8,'Color','b','LineWidth',3}); hold on

x2 = [25 15];
y2 = [15 25];
drawArrow(x2,y2,{'MaxHeadSize',10,'Color','r','LineWidth',3}); hold on

xlim([1, 100])
ylim([1, 100])

Há outro ajuste para pontas de seta ajustáveis:

function [ h ] = drawArrow( x,y,xlimits,ylimits,props )

xlim(xlimits)
ylim(ylimits)

h = annotation('arrow');
set(h,'parent', gca, ...
    'position', [x(1),y(1),x(2)-x(1),y(2)-y(1)], ...
    'HeadLength', 10, 'HeadWidth', 10, 'HeadStyle', 'cback1', ...
    props{:} );

end

que você pode chamar do seu script da seguinte maneira:

drawArrow(x1,y1,[1, 100],[1, 100],{'Color','b','LineWidth',3}); hold on
drawArrow(x2,y2,[1, 100],[1, 100],{'Color','r','LineWidth',3}); hold on

Elipse #

Para plotar uma elipse, você pode usar sua equação. Uma elipse tem um eixo maior e um eixo menor. Também queremos ser capazes de traçar a elipse em diferentes pontos centrais. Portanto, escrevemos uma função cujas entradas e saídas são:

Inputs:
    r1,r2: major and minor axis respectively
    C: center of the ellipse (cx,cy)
Output:
    [x,y]: points on the circumference of the ellipse

Você pode usar a seguinte função para obter os pontos em uma elipse e, em seguida, plotar esses pontos.

function [x,y] = getEllipse(r1,r2,C)
beta = linspace(0,2*pi,100);
x = r1*cos(beta) - r2*sin(beta);
y = r1*cos(beta) + r2*sin(beta);
x = x + C(1,1);
y = y + C(1,2);
end

Exemplo:

[x,y] = getEllipse(1,0.3,[2 3]);
plot(x,y);

Polígono(s) #

Crie vetores para manter as localizações x e y dos vértices, alimente-os em patch.

Polígono Único #

X=rand(1,4); Y=rand(1,4);
h=patch(X,Y,'red');

Vários polígonos #

Os vértices de cada polígono ocupam uma coluna de cada um de X, Y.

X=rand(4,3); Y=rand(4,3);
for i=2:3
  X(:,i)=X(:,i)+(i-1); % create horizontal offsets for visibility
end

h=patch(X,Y,'red');

Plotagem pseudo 4D #

Uma matriz (m x n) pode ser representada por uma superfície usando surf;

A cor da superfície é definida automaticamente em função dos valores na matriz (m x n). Se o colormap não for especificado, o padrão será aplicado.

Uma barra de cores pode ser adicionada para exibir o mapa de cores atual e indicar o mapeamento de dados valores no mapa de cores.

No exemplo a seguir, a matriz z (m x n) é gerada pela função:

z=x.*y.*sin(x).*cos(y);

no intervalo [-pi,pi]. Os valores x e y podem ser gerados usando o meshgrid função e a superfície é renderizada da seguinte forma:

% Create a Figure
figure
% Generate the `x` and `y` values in the interval `[-pi,pi]`
[x,y] = meshgrid([-pi:.2:pi],[-pi:.2:pi]);
% Evaluate the function over the selected interval
z=x.*y.*sin(x).*cos(y);
% Use surf to plot the surface
S=surf(x,y,z);
xlabel('X Axis');
ylabel('Y Axis');
zlabel('Z Axis');
grid minor
colormap('hot')
colorbar

Figura 1

Agora pode ser que informações adicionais estejam vinculadas aos valores da matriz z e eles são armazenados em outra matriz (m x n)

É possível adicionar essas informações adicionais no gráfico modificando o forma como a superfície é colorida.

Isso permitirá ter uma espécie de plotagem 4D: para a representação 3D da superfície gerada pela primeira matriz (m x n), a quarta dimensão será representada pela dados contidos na segunda matriz (m x n).

É possível criar tal plot chamando surf com 4 entradas:

surf(x,y,z,C)

onde o parâmetro C é a segunda matriz (que deve ser do mesmo tamanho de z) e é usado para definir a cor da superfície.

No exemplo a seguir, a matriz C é gerada pela função:

C=10*sin(0.5*(x.^2.+y.^2))*33;

no intervalo [-pi,pi]

Figura 2

Agora podemos chamar surf com quatro entradas:

figure
surf(x,y,z,C)
% shading interp
xlabel('X Axis');
ylabel('Y Axis');
zlabel('Z Axis');
grid minor
colormap('hot')
colorbar

Figura 3

Comparando a Figura 1 e a Figura 3, podemos notar que:

  • a forma da superfície corresponde aos valores z (a primeira matriz (m x n))
  • a cor da superfície (e seu alcance, dado pela barra de cores) corresponde ao Valores C (a primeira matriz (m x n))

figure
surf(x,y,C,z)
% shading interp
xlabel('X Axis');
ylabel('Y Axis');
zlabel('Z Axis');
grid minor
colormap('hot')
colorbar

Desenho rápido #

Existem três maneiras principais de fazer gráficos ou animações sequenciais: plot(x,y), set(h , 'XData' , y, 'YData' , y) e animatedline. Se você deseja que sua animação seja suave, você precisa de um desenho eficiente e os três métodos não são equivalentes.

% Plot a sin with increasing phase shift in 500 steps
x = linspace(0 , 2*pi , 100);

figure
tic
for thetha = linspace(0 , 10*pi , 500)
    y = sin(x + thetha);
    plot(x,y)
    drawnow
end
toc

Eu recebo ‘5,278172 segundos’. A função plot basicamente exclui e recria o objeto de linha a cada vez. Uma maneira mais eficiente de atualizar um gráfico é usar as propriedades XData e YData do objeto Line.

tic
h = [];   % Handle of line object
for thetha = linspace(0 , 10*pi , 500)
    y = sin(x + thetha);
    
    if isempty(h)
        % If Line still does not exist, create it
        h = plot(x,y);
    else
        % If Line exists, update it
        set(h , 'YData' , y)
    end
    drawnow
end
toc

Agora eu recebo 2.741996 segundos, muito melhor!

animatedline é uma função relativamente nova, introduzida em 2014b. Vamos ver como se sai:

tic
h = animatedline;
for thetha = linspace(0 , 10*pi , 500)
    y = sin(x + thetha);
    clearpoints(h)
    addpoints(h , x , y)
    drawnow
end
toc

3.360569 segundos, não tão bom quanto atualizar um gráfico existente, mas ainda melhor que plot(x,y).

Claro, se você tiver que plotar uma única linha, como neste exemplo, os três métodos são quase equivalentes e fornecem animações suaves. Mas se você tiver gráficos mais complexos, atualizar os objetos Line existentes fará a diferença.
