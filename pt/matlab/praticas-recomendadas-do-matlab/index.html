Recue o código corretamente #

A indentação adequada dá não apenas a aparência estética, mas também aumenta a legibilidade do código.

Por exemplo, considere o seguinte código:

%no need to understand the code, just give it a look
n = 2;
bf = false;
while n>1
for ii = 1:n
for jj = 1:n
if ii+jj>30
bf = true;
break
end
end
if bf
break 
end
end
if bf
break 
end
n = n + 1;
end

Como você pode ver, você precisa dar uma olhada cuidadosa para ver qual loop e instruções if estão terminando onde.
Com o recuo inteligente, você terá esta aparência:

n = 2;
bf = false;
while n>1
    for ii = 1:n
        for jj = 1:n
            if ii+jj>30
                bf = true;
                break
            end
        end
        if bf
            break
        end
    end
    if bf
        break
    end
    n = n + 1;
end

Isso indica claramente o início e o fim da instrução loops/if.

Você pode fazer recuo inteligente por:
*• selecionando todo o seu código (Ctrl+A)
*• e então pressionando Ctrl+I ou clicando em na barra de edição.

Mantenha as linhas curtas #

Use o caractere de continuação (reticências) ... para continuar a instrução longa.

Exemplo:

MyFunc( parameter1,parameter2,parameter3,parameter4, parameter5, parameter6,parameter7, parameter8, parameter9)

pode ser substituído por:

MyFunc( parameter1, ...
        parameter2, ...
        parameter3, ...
        parameter4, ...
        parameter5, ...
        parameter6, ...
        parameter7, ...
        parameter8, ...
        parameter9)

Use assert #

O Matlab permite que alguns erros muito triviais ocorram silenciosamente, o que pode fazer com que um erro seja gerado muito mais tarde na execução - dificultando a depuração. Se você presumir algo sobre suas variáveis, valide.

function out1 = get_cell_value_at_index(scalar1,cell2)
assert(isscalar(scalar1),'1st input must be a scalar')
assert(iscell(cell2),'2nd input must be a cell array')

assert(numel(cell2) >= scalar1),'2nd input must have more elements than the value of the 1st input')
assert(~isempty(cell2{scalar1}),'2nd input at location is empty')

out1 = cell2{scalar1};

Evite loops #

Na maioria das vezes, os loops são computacionalmente caros com o Matlab. Seu código será muito mais rápido se você usar a vetorização. Muitas vezes, também torna seu código mais modular, facilmente modificável e mais fácil de depurar. A maior desvantagem é que você precisa de tempo para planejar as estruturas de dados, e os erros de dimensão são mais fáceis de encontrar.

Exemplos #

Não escreva

for t=0:0.1:2*pi
    R(end+1)=cos(t);
end

mas

t=0:0.1:2*pi;
R=cos(t)

Não escreva

for i=1:n
    for j=1:m
        c(i,j)=a(i)+2*b(j);
    end
end

Mas algo parecido com

c=repmat(a.',1,m)+2*repmat(b,n,1)

Para obter mais detalhes, consulte vetorização

Operador de comentário de bloco #

É uma boa prática adicionar comentários que descrevam o código. É útil para outros e até para o codificador quando devolvido posteriormente. Uma única linha pode ser comentada usando o símbolo % ou usando a tecla de atalho Ctrl+R. Para descomentar uma linha comentada anteriormente, remova o símbolo % ou use a tecla de atalho Crtl+T.

Enquanto comentar um bloco de código pode ser feito adicionando um símbolo % no início de cada linha, as versões mais recentes do MATLAB (após 2015a) permitem que você use o Block Comment Operator %{ code %}. Este operador aumenta a legibilidade do código. Ele pode ser usado para comentários de código e documentação de ajuda de função. O Bloco pode ser dobrado e desdobrado para aumentar a legibilidade do código.

Como pode ser visto os operadores %{ e %} devem aparecer sozinhos nas linhas. Não inclua nenhum outro texto nestas linhas.

function y = myFunction(x)
%{
myFunction  Binary Singleton Expansion Function
y = myFunction(x) applies the element-by-element binary operation
specified by the function handle FUNC to arrays A and B, with implicit
expansion enabled.
%}

%%  Compute z(x, y) = x.*sin(y) on a grid:
% x = 1:10;
y = x.';

%{
z = zeros(numel(x),numel(y));
for ii=1:numel(x)
    for jj=1:numel(y)
        z(ii,jj) = x(ii)*sin(y(jj));
    end
end
%}

z = bsxfun(@(x, y) x.*sin(y), x, y);
y = y + z;

end

Criar nome exclusivo para arquivo temporário #

Ao codificar um script ou uma função, pode acontecer que um ou mais de um arquivo temporário seja necessário para, por exemplo, armazenar alguns dados.

Para evitar sobrescrever um arquivo existente ou ocultar uma função MATLAB a função tempname pode ser usado para gerar um nome exclusivo para um arquivo temporário na pasta temporária do sistema.

my_temp_file=tempname

O nome do arquivo é gerado sem a extensão; pode ser adicionado concatenando a extensão desejada para o nome gerado por tempname

my_temp_file_with_ext=[tempname '.txt']

A localização da pasta temporária do sistema pode ser recuperada chamando o tempdir.

Se, durante a execução da função/script, o arquivo temporário não estiver mais necessário, ele pode ser excluído usando a função delete

Como delete não pede confirmação, pode ser útil definir on opção para mover o arquivo a ser deletado na pasta recycle.

Isso pode ser feito usando a função recycle this caminho:

recycle('on')

No exemplo a seguir, um possível uso das funções tempname, delete e recycle é proposto.

%
% Create some example data
%
theta=0:.1:2*pi;
x=cos(theta);
y=sin(theta);
%
% Generate the temporary filename
%
my_temp_file=[tempname '.mat'];
%
% Split the filename (path, name, extension) and display them in a message box
[tmp_file_path,tmp_file_name, tmp_file_ext]=fileparts(my_temp_file)
uiwait(msgbox(sprintf('Path= %s\nName= %s\nExt= %s', ...
              tmp_file_path,tmp_file_name,tmp_file_ext),'TEMPORARY FILE'))
%
% Save the varaibles in a temporary file
%
save(my_temp_file,'x','y','theta')
%
% Load the varaibles from the temporary file
%
load(my_temp_file)
%
% Set the reclycle option on
%
recycle('on')
%
% Delete the temporary file
%
delete(my_temp_file)

Embargo

O nome do arquivo temporário é gerado usando o método java.util.UUID.randomUUID (randomUUID).

Se o MATLAB for executado sem JVM, o nome do arquivo temporário será gerado usando matlab.internal.timing.timing baseado no contador de CPU e tempo. Nesse caso o nome do arquivo temporário não é garantido como exclusivo.

Use validar atributos #

A função validateattributes pode ser usado para validar uma matriz em relação a um conjunto de especificações

Ele pode, portanto, ser usado para validar a entrada fornecida a uma função.

No exemplo a seguir, a função test_validateattributes requer três entrada

função test_validateattributes(input_1,input_2,input_3)

As especificações de entrada são:

  • array_1:

  • classe: dupla

  • tamanho: [3,2]

  • valores: os elementos não devem ser NaN

  • char_array:

  • classe: char

  • valor: a string não deve estar vazia

  • array_3

  • classe: dupla

  • tamanho: [5 1]

  • valores: os elementos devem ser reais

Para validar as três entradas, a função validateattributes pode ser chamada com a seguinte sintaxe:

validateattributes(A,classes,attributes,funcName,varName,argIndex)

Onde:

  • A é o array a ser validado
  • classes: é o type do array (por exemplo, single, double, logical)
  • attributes: são os atributos que o array de entrada deve corresponder (por exemplo, [3,2], size para especificar o tamanho do array, nonnan para especificar que o array não deve ter valores NaN)
  • funcName: é o nome da função na qual ocorre a validação. Este argumento é usado na geração da mensagem de erro (se houver)
  • varName: é o nome do array em validação. Este argumento é usado na geração da mensagem de erro (se houver)
  • argIndex: é a posição do array de entrada na lista de entrada. Este argumento é usado na geração da mensagem de erro (se houver)

Caso uma ou mais entradas não correspondam à especificação, um erro mensagem é gerada.

No caso de mais de uma entrada inválida, a validação para quando a primeira incompatibilidade é encontrada.

Esta é a função test_validateattributes na qual a validação de entrada foi implementado.

Como a função requer três entradas, uma primeira verificação do número de entradas fornecido é realizado usando a função nargin.

function test_validateattributes(array_1,char_array_1,array_3)
%
% Check for the number of expected input: if the number of input is less
% than the require, the function exits with an error message
%
if(nargin ~= 3)
   error('Error: TEST_VALIDATEATTRIBUTES requires 3 input, found %d',nargin)
end
%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
% Definition of the expected characteristics of the first input %
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%
% INPUT #1 name (only used in the generation of the error message)
%
input_1_name='array_1';
%
% INPUT #1 position (only used in the generation of the error message)
%
input_1_position=1;
%
% Expected CLASS of the first input MUST BE "double"
%
input_1_class={'double'};
%
% Expected ATTRIBUTES of the first input
%   SIZE: MUST BE [3,2]
%
input_1_size_attribute='size';
input_1_size=[3,2];
%
%   VALUE CHECK: the element MUST BE NOT NaN
%
input_1_value_type='nonnan';
%
% Build the INPUT 1 attributes
%
input_1_attributes={input_1_size_attribute,input_1_size, ...
                    input_1_value_type};
%
% CHECK THE VALIDITY OF THE FIRST INPUT
%
validateattributes(array_1, ...
                   input_1_class,input_1_attributes,'', ...
                   input_1_name,input_1_position);

%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
% Definition of the expected characteristics of the second input %
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%
% INPUT #1 name (only used in the generation of the error message)
%
input_2_name='char_array_1';
%
% INPUT #2 position (only used in the generation of the error message)
%
input_2_position=2;
%
% Expected CLASS of the first input MUST BE "string"
%
input_2_class={'char'};
%
%   VALUE CHECK: the element must be not NaN
%
input_2_size_attribute='nonempty';
%
% Build the INPUT 2 attributes
%
input_2_attributes={input_2_size_attribute};
%
% CHECK THE VALIDITY OF THE SECOND INPUT
%
validateattributes(char_array_1, ...
                   input_2_class,input_2_attributes,'', ...
                   input_2_name,input_2_position);
%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
% Definition of the expected characteristics of the third input %
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%
% INPUT #3 name (only used in the generation of the error message)
%
input_3_name='array_3';
%
% INPUT #3 position (only used in the generation of the error message)
%
input_3_position=3;
%
% Expected CLASS of the first input MUST BE "double"
%
input_3_class={'double'};
%
% Expected ATTRIBUTES of the first input
%   SIZE: must be [5]
input_3_size_attribute='size';
input_3_size=[5 1];
%   VALUE CHECK: the elements must be real
input_3_value_type='real';
%
% Build the INPUT 3 attributes
%
input_3_attributes={input_3_size_attribute,input_3_size, ...
                    input_3_value_type};
%
% CHECK THE VALIDITY OF THE FIRST INPUT
%
validateattributes(array_3, ...
                   input_3_class,input_3_attributes,'', ...
                   input_3_name,input_3_position);

disp('All the three input are OK')

O script a seguir pode ser usado para testar a implementação do procedimento de validação.

Gera as três entradas necessárias e, aleatoriamente, as torna inválidas.

%
% Generate the first input
%
n_rows=randi([2 3],1);
n_cols=2;
input_1=randi([20 30],n_rows,n_cols);
%
% Generate the second input
%
if(rand > 0.5)
   input_2='This is a string';
else
   input_2='';
end
%
% Generate the third input
%
input_3=acos(rand(5,1)*1.2);
%
% Call the test_validateattributes function with the above generated input
%
input_1
input_2
input_3
%
test_validateattributes(input_1,input_2,input_3)
