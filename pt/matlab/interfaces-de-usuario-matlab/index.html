Passando dados pela interface do usuário #

A maioria das interfaces de usuário avançadas exige que o usuário seja capaz de passar informações entre as várias funções que compõem uma interface de usuário. O MATLAB tem vários métodos diferentes para fazer isso.

[guiado] 1 #

O próprio GUI Development Environment (GUIDE) do MATLAB prefere usar um struct chamado handles para passar dados entre callbacks. Este struct contém todos os identificadores gráficos para os vários componentes da interface do usuário, bem como dados especificados pelo usuário. Se você não estiver usando um retorno de chamada criado por GUIDE que passa automaticamente handles, você pode recuperar o valor atual usando guidata

% hObject is a graphics handle to any UI component in your GUI
handles = guidata(hObject);

Se você quiser modificar um valor armazenado nesta estrutura de dados, você pode modificá-lo, mas deve armazená-lo de volta no hObject para que as alterações sejam visíveis por outros retornos de chamada. Você pode armazená-lo especificando um segundo argumento de entrada para guidata.

% Update the value
handles.myValue = 2;

% Save changes
guidata(hObject, handles)

O valor de hObject não importa desde que seja um componente de interface do usuário dentro da mesma figura porque, em última análise, os dados são armazenados na figura que contém hObject.

Melhor para:

  • Armazenando a estrutura handles, na qual você pode armazenar todos os alças de seus componentes GUI.
  • Armazenar “pequenas” outras variáveis ​​que precisam ser acessadas pela maioria dos callbacks.

Não recomendado para:

  • Armazenar grandes variáveis ​​que não precisam ser acessadas por todos retornos de chamada e subfunções (use setappdata/getappdata para esses).

setappdata/getappdata #

Semelhante à abordagem guidata, você pode usar setappdata e getappdata para armazenar e recuperar valores de dentro de um identificador gráfico. A vantagem de usar esses métodos é que você pode recuperar somente o valor desejado em vez de um struct inteiro contendo todos os dados armazenados. É semelhante a um armazenamento de chave/valor.

Para armazenar dados em um objeto gráfico

% Create some data you would like to store
myvalue = 2

% Store it using the key 'mykey'
setappdata(hObject, 'mykey', myvalue)

E para recuperar esse mesmo valor de dentro de um callback diferente

value = getappdata(hObject, 'mykey');

Nota: Se nenhum valor foi armazenado antes de chamar getappdata, ele retornará um array vazio ([]).

Semelhante ao guidata, os dados são armazenados na figura que contém hObject.

Melhor para:

  • Armazenar grandes variáveis ​​que não precisam ser acessadas por todos retornos de chamada e subfunções.

UserData #

Cada identificador gráfico tem uma propriedade especial, UserData que pode conter qualquer dado que você desejar. Ele pode conter uma matriz de células, um struct ou até mesmo um escalar. Você pode aproveitar esta propriedade e armazenar quaisquer dados que deseja associar a um determinado identificador gráfico neste campo. Você pode salvar e recuperar o valor usando os métodos padrão get/set para objetos gráficos ou notação de ponto se estiver usando R2014b ou mais recente.

% Create some data to store
mydata = {1, 2, 3};

% Store it within the UserData property
set(hObject, 'UserData', mydata)

% Of if you're using R2014b or newer:
% hObject.UserData = mydata;

Em seguida, de dentro de outro retorno de chamada, você pode recuperar esses dados:

their_data = get(hObject, 'UserData');

% Or if you're using R2014b or newer:
% their_data = hObject.UserData;

Melhor para:

  • Armazenar variáveis ​​com escopo limitado (variáveis ​​que provavelmente serão usadas apenas pelo objeto em que estão armazenadas ou objetos que tenham uma relação direta com ele).

Funções aninhadas #

No MATLAB, uma função aninhada pode ler e modificar qualquer variável definida na função pai. Dessa forma, se você especificar um retorno de chamada para ser uma função aninhada, ele poderá recuperar e modificar quaisquer dados armazenados na função principal.

function mygui()    
    hButton = uicontrol('String', 'Click Me', 'Callback', @callback);

    % Create a counter to keep track of the number of times the button is clicked
    nClicks = 0;

    % Callback function is nested and can therefore read and modify nClicks
    function callback(source, event)
        % Increment the number of clicks
        nClicks = nClicks + 1;

        % Print the number of clicks so far
        fprintf('Number of clicks: %d\n', nClicks);
    end
end

Melhor para:

  • GUIs pequenas e simples. (para prototipagem rápida, não precisa implementar os métodos guidata e/ou set/getappdata).

Não recomendado para:

  • GUIs médias, grandes ou complexas.

  • GUI criada com GUIDE.

Argumentos de entrada explícitos #

Se você precisar enviar dados para uma função de retorno de chamada e não precisar modificar os dados dentro do retorno de chamada, sempre considere passar os dados para o retorno de chamada usando uma definição de retorno de chamada cuidadosamente elaborada.

Você pode usar uma função anônima que adiciona entradas

% Create some data to send to mycallback
data = [1, 2, 3];

% Pass data as a third input to mycallback
set(hObject, 'Callback', @(source, event)mycallback(source, event, data))

Ou você pode usar a sintaxe da matriz de células para especificar um retorno de chamada, especificando novamente entradas adicionais.

set(hObject, 'Callback', {@mycallback, data})

Melhor para:

  • Quando o callback precisa de data para realizar algumas operações mas a variável data não precisa ser modificada e salva em um novo estado.

Criando um botão em sua interface do usuário que pausa a execução de retorno de chamada #

Às vezes, gostaríamos de pausar a execução do código para inspecionar o estado do aplicativo (consulte Depuração). Ao executar o código através do editor MATLAB, isso pode ser feito usando o botão “Pausar” na interface do usuário ou pressionando Ctrl+c (no Windows). No entanto, quando uma computação foi iniciada a partir de uma GUI (através do retorno de chamada de algum uicontrol), esse método não funciona mais, e o retorno de chamada deve ser interrompido por meio de outro retorno de chamada. Abaixo está uma demonstração deste princípio:

function interruptibleUI
dbclear in interruptibleUI % reset breakpoints in this file
figure('Position',[400,500,329,160]); 

uicontrol('Style', 'pushbutton',...
          'String', 'Compute',...
          'Position', [24 55 131 63],...
          'Callback', @longComputation,...
          'Interruptible','on'); % 'on' by default anyway
      
uicontrol('Style', 'pushbutton',...
          'String', 'Pause #1',...
          'Position', [180 87 131 63],...
          'Callback', @interrupt1);       

uicontrol('Style', 'pushbutton',...
          'String', 'Pause #2',...
          'Position', [180 12 131 63],...
          'Callback', @interrupt2);    

end

function longComputation(src,event)
  superSecretVar = rand(1);
  pause(15);
  print('done!'); % we'll use this to determine if code kept running "in the background".
end

function interrupt1(src,event) % depending on where you want to stop
  dbstop in interruptibleUI at 27 % will stop after print('done!');
  dbstop in interruptibleUI at 32 % will stop after **this** line.
end

function interrupt2(src,event) % method 2
  keyboard;
  dbup; % this will need to be executed manually once the code stops on the previous line.
end

Para ter certeza de que você entendeu este exemplo, faça o seguinte:

  1. Cole o código acima em um novo arquivo chamado e salve-o como interruptibleUI.m, de forma que o código comece na primeira linha do arquivo (isso é importante para que o primeiro método funcione).
  2. Execute o script.
  3. Clique em Compute e logo depois clique em Pause #1 ou em Pause #2.
  4. Certifique-se de encontrar o valor de superSecretVar.

Passando dados usando a estrutura “handles” #

Este é um exemplo de uma GUI básica com dois botões que alteram um valor armazenado na estrutura handles da GUI.

function gui_passing_data()
    % A basic GUI with two buttons to show a simple use of the 'handles'
    % structure in GUI building

    %  Create a new figure.
    f = figure();

    % Retrieve the handles structure
    handles = guidata(f);

    % Store the figure handle
    handles.figure = f;

    % Create an edit box and two buttons (plus and minus), 
    % and store their handles for future use
    handles.hedit  = uicontrol('Style','edit','Position',[10,200,60,20] , 'Enable', 'Inactive');

    handles.hbutton_plus  = uicontrol('Style','pushbutton','String','+',...
               'Position',[80,200,60,20] , 'Callback' , @ButtonPress);

    handles.hbutton_minus  = uicontrol('Style','pushbutton','String','-',...
               'Position',[150,200,60,20] , 'Callback' , @ButtonPress);

    % Define an initial value, store it in the handles structure and show
    % it in the Edit box
    handles.value = 1;
    set(handles.hedit , 'String' , num2str(handles.value))

    % Store handles
    guidata(f, handles);



function  ButtonPress(hObject, eventdata)
    % A button was pressed
    % Retrieve the handles
    handles = guidata(hObject);

    % Determine which button was pressed; hObject is the calling object
    switch(get(hObject , 'String'))
        case '+'
            % Add 1 to the value
            handles.value = handles.value + 1;
            set(handles.hedit , 'String', num2str(handles.value))
        case '-'
            % Substract 1 from the value
            handles.value = handles.value - 1;
    end

    % Display the new value
    set(handles.hedit , 'String', num2str(handles.value))

    % Store handles
    guidata(hObject, handles);

Para testar o exemplo, salve-o em um arquivo chamado gui_passing_data.m e execute-o com F5. Por favor, note que em um caso tão simples, você nem precisaria armazenar o valor na estrutura handles porque você poderia acessá-lo diretamente da propriedade String da caixa de edição.

Problemas de desempenho ao passar dados pela interface do usuário #

Duas técnicas principais permitem passar dados entre funções GUI e Callbacks: setappdata/getappdata e guidata (leia mais sobre isso). O primeiro deve ser usado para variáveis ​​maiores, pois é mais eficiente em termos de tempo. O exemplo a seguir testa a eficiência dos dois métodos.

Uma GUI com um botão simples é criada e uma grande variável (10000x10000 double) é armazenada tanto com guidata quanto com setappdata. O botão recarrega e armazena de volta a variável usando os dois métodos enquanto cronometra sua execução. O tempo de execução e a porcentagem de melhoria usando setappdata são exibidos na janela de comando.

function gui_passing_data_performance()
    % A basic GUI with a button to show performance difference between
    % guidata and setappdata

    %  Create a new figure.
    f = figure('Units' , 'normalized');

    % Retrieve the handles structure
    handles = guidata(f);

    % Store the figure handle
    handles.figure = f;

    handles.hbutton = uicontrol('Style','pushbutton','String','Calculate','units','normalized',...
               'Position',[0.4 , 0.45 , 0.2 , 0.1] , 'Callback' , @ButtonPress);

    % Create an uninteresting large array
    data = zeros(10000);

    % Store it in appdata
    setappdata(handles.figure , 'data' , data);

    % Store it in handles
    handles.data = data;

    % Save handles
    guidata(f, handles);



function  ButtonPress(hObject, eventdata)

    % Calculate the time difference when using guidata and appdata
    t_handles = timeit(@use_handles);
    t_appdata = timeit(@use_appdata);

    % Absolute and percentage difference
    t_diff = t_handles - t_appdata;
    t_perc = round(t_diff / t_handles * 100);

    disp(['Difference: ' num2str(t_diff) ' ms / ' num2str(t_perc) ' %'])




function  use_appdata()  

    % Retrieve the data from appdata
    data = getappdata(gcf , 'data');

    % Do something with data %

    % Store the value again
    setappdata(gcf , 'data' , data);


function use_handles()

    % Retrieve the data from handles
    handles = guidata(gcf);
    data = handles.data;

    % Do something with data %

    % Store it back in the handles
    handles.data = data;
    guidata(gcf, handles);

No meu Xeon W3530@2,80 GHz eu recebo Diferença: 0,00018957 ms / 73 %, assim usando getappdata/setappdata eu recebo uma melhoria de desempenho de 73%! Observe que o resultado não muda se uma variável dupla 10x10 for usada, no entanto, o resultado mudará se handles contiver muitos campos com dados grandes.
