Sintaxe #

  1. Y = fft(X) % calcula a FFT do Vetor ou Matriz X usando um comprimento de transformação padrão de 256 (a ser confirmado para a versão)

  2. Y = fft(X,n) % calcula a FFT de X usando n como Transform Length, n deve ser um número baseado em 2 potências. Se o comprimento de X for menor que n, o Matlab preencherá automaticamente X com zeros de tal forma que comprimento(X) = n

  3. Y = fft(X,n,dim) % calcula a FFT de X usando n como Transform Length ao longo da dimensão dim (pode ser 1 ou 2 para horizontal ou vertical, respectivamente)

  4. Y = fft2(X) % Calcular a FFT 2D de X

  5. Y = fftn(X, dim) % Calcular a FFT dim-dimensional de X, em relação ao vetor de dimensões dim.

  6. y = ifft(X) % calcula o Inverso de FFT de X (que é uma matriz/vetor de números) usando o padrão 256 Transform Length

  7. y = ifft(X,n) % calcula o IFFT de X usando n como Transform Length

  8. y = ifft(X,n,dim) % calcula o IFFT de X usando n como Transform Length over dimension dim (pode ser 1 ou 2 para horizontal ou vertical, respectivamente)

  9. y = ifft(X,n,dim,‘symmetric’) %A opção Simétrica faz com que ifft trate X como conjugado simétrico ao longo da dimensão ativa. Esta opção é útil quando X não é exatamente conjugado simétrico, apenas devido ao erro de arredondamento.

  10. y = ifft2(X) % Calcular o inverso 2D ft de X

  11. y= ifftn(X,dim) %Calcule a dim-dimensional inversa fft de X.

Parâmetros #

Parâmetro Descrição
X este é o seu sinal de entrada no domínio do tempo, deve ser um vetor de números.
n este é o parâmetro NFFT conhecido como Transform Length, pense nisso como a resolução do seu resultado FFT, DEVE ser um número que seja uma potência de 2 (ou seja, 64.128.256…2^N)
escurecer esta é a dimensão na qual você deseja calcular a FFT, use 1 se quiser calcular sua FFT na direção horizontal e 2 se quiser calcular sua FFT na direção vertical - Observe que esse parâmetro geralmente é deixado em branco, pois a função é capaz de detectar a direção do seu vetor.

Matlab FFT é um processo muito paralelizado capaz de lidar com grandes quantidades de dados. Ele também pode usar a GPU com grande vantagem.

ifft(fft(X)) = X

A afirmação acima é verdadeira se os erros de arredondamento forem omitidos.

Implemente uma Transformada de Fourier simples no Matlab #

A Transformada de Fourier é provavelmente a primeira lição em Processamento Digital de Sinais, sua aplicação está em todos os lugares e é uma ferramenta poderosa quando se trata de analisar dados (em todos os setores) ou sinais. Matlab tem um conjunto de poderosas caixas de ferramentas para Fourier Transform. Neste exemplo, usaremos a Transformada de Fourier para analisar um sinal básico de onda senoidal e gerar o que às vezes é conhecido como Periodograma usando FFT:

%Signal Generation
A1=10;                % Amplitude 1
A2=10;                % Amplitude 2
w1=2*pi*0.2;          % Angular frequency 1
w2=2*pi*0.225;        % Angular frequency 2
Ts=1;                 % Sampling time
N=64;                 % Number of process samples to be generated
K=5;                  % Number of independent process realizations
sgm=1;                % Standard deviation of the noise
n=repmat([0:N-1].',1,K);             % Generate resolution
phi1=repmat(rand(1,K)*2*pi,N,1);     % Random phase matrix 1
phi2=repmat(rand(1,K)*2*pi,N,1);     % Random phase matrix 2
x=A1*sin(w1*n*Ts+phi1)+A2*sin(w2*n*Ts+phi2)+sgm*randn(N,K);   % Resulting Signal

NFFT=256;            % FFT length
F=fft(x,NFFT);       % Fast Fourier Transform Result
Z=1/N*abs(F).^2;     % Convert FFT result into a Periodogram

Observe que a Transformada Discreta de Fourier é implementada pela Transformada Rápida de Fourier (fft) no Matlab, ambas produzirão o mesmo resultado, mas a FFT é uma implementação rápida da DFT.

figure
w=linspace(0,2,NFFT);
plot(w,10*log10(Z)),grid;
xlabel('w [\pi rad/s]')
ylabel('Z(f) [dB]')
title('Frequency Range: [ 0 ,  \omega_s ]')

Imagens e FTs multidimensionais #

Em imagens médicas, espectroscopia, processamento de imagens, criptografia e outras áreas da ciência e engenharia, muitas vezes se deseja computar transformadas de Fourier multidimensionais de imagens. Isso é bastante direto no Matlab: as imagens (multidimensionais) são apenas matrizes n-dimensionais, afinal, e as transformadas de Fourier são operadores lineares: apenas iterativamente transformamos Fourier ao longo de outras dimensões. Matlab fornece fft2 e ifft2 para fazer isso em 2-d, ou fftn em n-dimensões.

Uma armadilha potencial é que a transformada de Fourier de imagens geralmente é mostrada “ordenada centrada”, ou seja, com a origem do espaço k no meio da imagem. O Matlab fornece o comando fftshift para trocar a localização dos componentes DC da transformada de Fourier apropriadamente. Esta notação de ordenação torna substancialmente mais fácil realizar técnicas comuns de processamento de imagem, uma das quais é ilustrada abaixo.

Preenchimento zero #

Uma maneira “rápida e suja” de interpolar uma imagem pequena para um tamanho maior é transformá-la de Fourier, preencher a transformada de Fourier com zeros e, em seguida, fazer a transformação inversa. Isso interpola efetivamente entre cada pixel com uma função de base em forma de sinc e é comumente usado para aumentar a escala de dados de imagens médicas de baixa resolução. Vamos começar carregando um exemplo de imagem integrado

%Load example image 
I=imread('coins.png'); %Load example data -- coins.png is builtin to Matlab 
I=double(I); %Convert to double precision -- imread returns integers 
imageSize = size(I); % I is a 246 x 300 2D image

%Display it
imagesc(I); colormap gray; axis equal; 
%imagesc displays images scaled to maximum intensity

Agora podemos obter a transformada de Fourier de I. Para ilustrar o que o fftshift faz, vamos comparar os dois métodos:

% Fourier transform 
%Obtain the centric- and non-centric ordered Fourier transform of I
k=fftshift(fft2(fftshift(I))); 
kwrong=fft2(I); 

%Just for the sake of comparison, show the magnitude of both transforms: 
figure; subplot(2,1,1); 
imagesc(abs(k),[0 1e4]); colormap gray; axis equal; 
subplot(2,1,2); 
imagesc(abs(kwrong),[0 1e4]); colormap gray; axis equal;
%(The second argument to imagesc sets the colour axis to make the difference clear).

Agora obtivemos o 2D FT de uma imagem de exemplo. Para preenchê-lo com zero, queremos pegar cada espaço k, preencher as arestas com zeros e, em seguida, fazer a transformação de volta:

%Zero fill
kzf = zeros(imageSize .* 2); %Generate a  492x600 empty array to put the result in 
kzf(end/4:3*end/4-1,end/4:3*end/4-1) = k; %Put k in the middle 
kzfwrong = zeros(imageSize .* 2); %Generate a  492x600 empty array to put the result in 
kzfwrong(end/4:3*end/4-1,end/4:3*end/4-1) = kwrong; %Put k in the middle 

%Show the differences again 
%Just for the sake of comparison, show the magnitude of both transforms: 
figure; subplot(2,1,1); 
imagesc(abs(kzf),[0 1e4]); colormap gray; axis equal; 
subplot(2,1,2); 
imagesc(abs(kzfwrong),[0 1e4]); colormap gray; axis equal;
%(The second argument to imagesc sets the colour axis to make the difference clear).

Neste ponto, o resultado bastante normal:

Uma vez que tomamos as transformações de volta, podemos ver que (corretamente!) os dados de preenchimento zero fornecem um método sensato para interpolação:

% Take the back transform and view 
Izf = fftshift(ifft2(ifftshift(kzf))); 
Izfwrong = ifft2(kzfwrong);

figure; subplot(1,3,1); 
imagesc(abs(Izf)); colormap gray; axis equal; 
title('Zero-filled image'); 
subplot(1,3,2); 
imagesc(abs(Izfwrong)); colormap gray; axis equal;
title('Incorrectly zero-filled image'); 
subplot(1,3,3); 
imagesc(I); colormap gray; axis equal; 
title('Original image'); 
set(gcf,'color','w');

Observe que o tamanho da imagem preenchida com zero é o dobro do original. Pode-se zerar o preenchimento por mais de um fator de dois em cada dimensão, embora obviamente isso não aumente arbitrariamente o tamanho de uma imagem.

Dicas, dicas, 3D e muito mais #

O exemplo acima vale para imagens 3D (como geralmente são geradas por técnicas de imagem médica ou microscopia confocal, por exemplo), mas exigem que fft2 seja substituído por fftn(I, 3), por exemplo. Devido à natureza um tanto complicada de escrever fftshift(fft(fftshift(...) várias vezes, é bastante comum definir funções como fft2c localmente para fornecer uma sintaxe mais fácil localmente – como:

function y = fft2c(x)

y = fftshift(fft2(fftshift(x)));

Observe que a FFT é rápida, mas grandes transformações de Fourier multidimensionais ainda levarão tempo em um computador moderno. Além disso, é inerentemente complexo: a magnitude do espaço k foi mostrada acima, mas a fase é absolutamente vital; as traduções no domínio da imagem são equivalentes a uma rampa de fase no domínio de Fourier. Existem várias operações muito mais complexas que podem ser feitas no domínio de Fourier, como filtrar frequências espaciais altas ou baixas (multiplicando-as por um filtro) ou mascarar pontos discretos correspondentes ao ruído. Existe correspondentemente uma grande quantidade de código gerado pela comunidade para lidar com operações comuns de Fourier disponíveis no site principal do repositório da comunidade do Matlab, o File Exchange.

Transformadas Inversas de Fourier #

Um dos principais benefícios da Transformação de Fourier é sua capacidade de reverter para o Domínio do Tempo sem perder informações. Vamos considerar o mesmo sinal que usamos no exemplo anterior:

A1=10;                % Amplitude 1
A2=10;                % Amplitude 2
w1=2*pi*0.2;          % Angular frequency 1
w2=2*pi*0.225;        % Angular frequency 2
Ts=1;                 % Sampling time
N=64;                 % Number of process samples to be generated
K=1;                  % Number of independent process realizations
sgm=1;                % Standard deviation of the noise
n=repmat([0:N-1].',1,K);             % Generate resolution
phi1=repmat(rand(1,K)*2*pi,N,1);     % Random phase matrix 1
phi2=repmat(rand(1,K)*2*pi,N,1);     % Random phase matrix 2
x=A1*sin(w1*n*Ts+phi1)+A2*sin(w2*n*Ts+phi2)+sgm*randn(N,K);   % Resulting Signal

NFFT=256;            % FFT length
F=fft(x,NFFT);       % FFT result of time domain signal

Se abrirmos F no Matlab, descobriremos que é uma matriz de números complexos, uma parte real e uma parte imaginária. Por definição, para recuperar o sinal original no Domínio do Tempo, precisamos tanto do Real (que representa a variação da Magnitude) quanto do Imaginário (que representa a variação da Fase), então para retornar ao Domínio do Tempo, basta:

TD = ifft(F,NFFT);   %Returns the Inverse of F in Time Domain

Observe aqui que TD retornado teria o comprimento 256 porque configuramos NFFT para 256, no entanto, o comprimento de x é apenas 64, então o Matlab preencherá zeros no final da transformação TD. Por exemplo, se a NFFT for 1024 e o comprimento for 64, o TD retornado será 64 + 960 zeros. Observe também que, devido ao arredondamento de ponto flutuante, você pode obter algo como 3,1 * 10e-20, mas para fins gerais: Para qualquer X, ifft(fft(X)) é igual a X para dentro do erro de arredondamento.

Digamos por um momento que após a transformação, fizemos algo e ficamos apenas com a parte REAL da FFT:

R = real(F);         %Give the Real Part of the FFT
TDR = ifft(R,NFFT);  %Give the Time Domain of the Real Part of the FFT

Isso significa que estamos perdendo a parte imaginária de nossa FFT e, portanto, estamos perdendo informações nesse processo reverso. Para preservar o original sem perder informações, você deve sempre manter a parte imaginária da FFT usando imag e aplicar suas funções para ambas ou para a parte real.

figure
subplot(3,1,1)
plot(x);xlabel('time samples');ylabel('magnitude');title('Original Time Domain Signal')
subplot(3,1,2)
plot(TD(1:64));xlabel('time samples');ylabel('magnitude');title('Inverse Fourier Transformed - Time Domain Signal')
subplot(3,1,3)
plot(TDR(1:64));xlabel('time samples');ylabel('magnitude');title('Real part of IFFT transformed Time Domain Signal')
