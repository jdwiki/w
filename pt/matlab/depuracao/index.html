Sintaxe #

  • dbstop no arquivo no local se expressão

Parâmetros #

Parâmetro Detalhes
arquivo Nome do arquivo .m (sem extensão), ex. ajustado. Este parâmetro é (Obrigatório) a menos que defina tipos especiais de ponto de interrupção condicional como dbstop if error ou dbstop if naninf.
localização Número da linha onde o ponto de interrupção deve ser colocado. Se a linha especificada não contiver código executável, o ponto de interrupção será colocado na primeira linha válida após a especificada.
expressão Qualquer expressão ou combinação das mesmas que resulte em um valor booleano. Exemplos: ind == 1, nargin < 4 && isdir('Q:\').

Trabalhando com pontos de interrupção #

Definição #

No desenvolvimento de software, um ponto de interrupção é um ponto de parada ou pausa intencional em um programa, implementado para fins de depuração.

Mais geralmente, um breakpoint é um meio de adquirir conhecimento sobre um programa durante sua execução. Durante a interrupção, o programador inspeciona o ambiente de teste (registradores de uso geral, memória, logs, arquivos, etc.) para descobrir se o programa está funcionando conforme o esperado. Na prática, um ponto de interrupção consiste em uma ou mais condições que determinam quando a execução de um programa deve ser interrompida.

-Wikipédia

Pontos de interrupção no MATLAB #

Motivação #

No MATLAB, quando a execução é pausada em um ponto de interrupção, as variáveis ​​existentes no espaço de trabalho atual (também conhecido como scope) ou em qualquer um dos espaços de trabalho de chamada podem ser inspecionadas (e geralmente também modificadas).

Tipos de pontos de interrupção #

O MATLAB permite que os usuários coloquem dois tipos de pontos de interrupção em arquivos .m:

  • Pontos de interrupção padrão (ou “unrestricted”) (mostrados em vermelho) - pausam a execução sempre que a linha marcada é alcançada.
  • Breakpoints “condicionais” (mostrados em amarelo) - pausa a execução sempre que a linha marcada for alcançada E a condição definida no breakpoint for avaliada como true.

Colocando pontos de interrupção #

Ambos os tipos de pontos de interrupção podem ser criados de várias maneiras:

  • Usando a GUI do Editor MATLAB, clicando com o botão direito do mouse na linha horizontal ao lado do número da linha.

  • Usando o comando dbstop:

    % Create an unrestricted breakpoint:
dbstop in file at location
% Create a conditional breakpoint:
dbstop in file at location if expression

% Examples and special cases: 
dbstop in fit at 99 % Standard unrestricted breakpoint.

dbstop in fit at 99 if nargin==3 % Standard conditional breakpoint.

dbstop if error % This special type of breakpoint is not limited to a specific file, and
                % will trigger *whenever* an error is encountered in "debuggable" code.

dbstop in file % This will create an unrestricted breakpoint on the first executable line
               % of "file".

dbstop if naninf % This special breakpoint will trigger whenever a computation result 
                 % contains either a NaN (indicates a division by 0) or an Inf

  • Usando atalhos de teclado: a tecla padrão para criar um breakpoint padrão no Windows é F12; a chave padrão para pontos de interrupção condicionais é unset.

Desativando e reativando pontos de interrupção #

Desabilite um breakpoint para ignorá-lo temporariamente: breakpoints desabilitados não pausam a execução. Desabilitar um ponto de interrupção pode ser feito de várias maneiras:

  • Clique com o botão direito do mouse no círculo vermelho/amarelo do ponto de interrupção > Desativar ponto de interrupção.
  • Clique com o botão esquerdo em um ponto de interrupção condicional (amarelo).
  • Na guia Editor > Pontos de interrupção > Ativar\Desativar.

Removendo pontos de interrupção #

Todos os pontos de interrupção permanecem em um arquivo até serem removidos, manualmente ou automaticamente. Os pontos de interrupção são limpos automaticamente ao encerrar a sessão do MATLAB (ou seja, encerrar o programa). A limpeza manual de pontos de interrupção é feita de uma das seguintes maneiras:

  • Usando o comando dbclear:

    dbclear all
dbclear in file   
dbclear in file at location    
dbclear if condition

  • Clicar com o botão esquerdo em um ícone de ponto de interrupção padrão ou em um ícone de ponto de interrupção condicional desativado.

  • Clique com o botão direito do mouse em qualquer ponto de interrupção > Limpar ponto de interrupção.

  • Na guia Editor > Pontos de interrupção > Limpar tudo.

  • Nas versões pré-R2015b do MATLAB, usando o comando clear.

Retomando a execução #

Quando a execução é pausada em um ponto de interrupção, há duas maneiras de continuar executando o programa:

  • Executar a linha atual e pausar novamente antes da próxima linha.

F101 no Editor, dbstep na Janela de Comando, “Etapa” na Faixa de Opções > Editor > DEBUG.

  • Executar até o próximo breakpoint (se não houver mais breakpoints, a execução prossegue até o final do programa).

    F121 in the Editor, dbcont in the Command Window, “Continue” in Ribbon > Editor > DEBUG.

1 - padrão no Windows.

Depurando o código Java invocado pelo MATLAB #

Visão geral #

Para depurar classes Java que são chamadas durante a execução do MATLAB, é necessário realizar duas etapas:

  1. Execute o MATLAB no modo de depuração JVM.
  2. Anexe um depurador Java ao processo MATLAB.

Quando o MATLAB é iniciado no modo de depuração da JVM, a seguinte mensagem aparece na janela de comando:

JVM is being started with debugging enabled.
Use "jdb -connect com.sun.jdi.SocketAttach:port=4444" to attach debugger.

Fim do MATLAB #

Ao executar o MATLAB usando este atalho, a depuração da JVM será habilitada.

Alternativamente, o arquivo java.opts pode ser criado/atualizado. Este arquivo é armazenado em “matlab-root\bin\arch”, onde “matlab-root” é o diretório de instalação do MATLAB e “arch” é a arquitetura (por exemplo, “win32”).

O seguinte deve ser adicionado no arquivo:

-Xdebug
-Xrunjdwp:transport=dt_socket,address=1044,server=y,suspend=n
