Uso de bsxfun #

Muitas vezes, a razão pela qual o código foi escrito em um loop ‘for’ é para calcular valores de ‘próximos’. A função bsxfun muitas vezes pode ser usada para fazer isso de forma mais sucinta.

Por exemplo, suponha que você deseja realizar uma operação coluna a coluna na matriz B, subtraindo a média de cada coluna dela:

B = round(randn(5)*10);                  % Generate random data 
A = zeros(size(B));                      % Preallocate array
for col = 1:size(B,2);                    % Loop over columns
    A(:,col) = B(:,col) - mean(B(:,col));   % Subtract means 
end

Este método é ineficiente se B for grande, muitas vezes devido ao MATLAB ter que mover o conteúdo das variáveis ​​na memória. Usando bsxfun, pode-se fazer o mesmo trabalho de forma organizada e fácil em apenas uma única linha:

A = bsxfun(@minus, B, mean(B));

Aqui, @minus é um identificador de função para o operador minus (-) e será aplicado entre os elementos das duas matrizes B e mean(B) . Outros handles de função, mesmo os definidos pelo usuário, também são possíveis.

Em seguida, suponha que você queira adicionar o vetor de linha v a cada linha na matriz A:

v = [1,  2,  3];

A = [8,  1,  6
     3,  5,  7
     4,  9,  2];

A abordagem ingênua é usar um loop (faça não faça isso):

B = zeros(3);
for row = 1:3
    B(row,:) = A(row,:) + v;
end

Outra opção seria replicar v com repmat (faça não faça isso também):

>> v = repmat(v,3,1)
v =
     1     2     3
     1     2     3
     1     2     3

>> B = A + v;

Em vez disso, use bsxfun para esta tarefa:

>> B = bsxfun(@plus, A, v);
B =
     9     3     9
     4     7    10
     5    11     5

Sintaxe #

bsxfun(@fun, A, B)

onde @fun é uma das funções suportadas e as duas matrizes A e B respeitam as duas condições abaixo .

O nome bsxfun ajuda a entender como a função funciona e significa Binary FUNction com Singleton eXpansion. Em outras palavras, se:

  1. duas matrizes compartilham as mesmas dimensões, exceto uma
  2. e a dimensão discordante é um singleton (ou seja, tem um tamanho de 1) em qualquer uma das duas matrizes

então a matriz com a dimensão singleton será expandida para corresponder à dimensão da outra matriz. Após a expansão, uma função binária é aplicada elemento a elemento nas duas matrizes.

Por exemplo, seja A um array M-by-N-byK e B seja um array M-by-N. Em primeiro lugar, suas duas primeiras dimensões têm tamanhos correspondentes. Em segundo lugar, A tem camadas K enquanto B tem implicitamente apenas 1, portanto, é um singleton. Todas as condições são atendidas e B será replicado para corresponder à 3ª dimensão de A.

Em outras linguagens, isso é comumente chamado de broadcasting e acontece automaticamente em Python (numpy) e Octave.

A função, @fun, deve ser uma função binária, o que significa que deve receber exatamente duas entradas.

Observações #

Internamente, o bsxfun não replica o array e executa um loop eficiente.

Expansão de array implícita (transmissão) [R2016b] #

O MATLAB R2016b apresentou uma generalização de seu mecanismo de expansão escalar1,2, para também oferecer suporte a certas operações elementares entre arrays de tamanhos diferentes, como desde que suas dimensões sejam compatíveis. Os operadores que suportam expansão implícita são1:

  • Operadores aritméticos elementares: +, -, .*, .^, ./, .\.
  • Operadores relacionais: <, <=, >, >=, ==, ~=.
  • Operadores lógicos: &, |, xor.
  • Funções bit a bit: bitand, bitor, bitxor.
  • Funções matemáticas elementares: max, min, mod, rem, hypot, atan2, atan2d.

As operações binárias acima mencionadas são permitidas entre arrays, desde que tenham “tamanhos compatíveis”. Os tamanhos são considerados “compatíveis” quando cada dimensão em uma matriz é exatamente igual à mesma dimensão na outra matriz ou é igual a 1. Observe que as dimensões de singleton à direita (ou seja, de tamanho ‘1’) são omitidas pelo MATLAB, embora teoricamente haja uma quantidade infinita delas. Em outras palavras - as dimensões que aparecem em uma matriz e não aparecem na outra, são implicitamente adequadas para expansão automática.

Por exemplo, nas versões do MATLAB antes do R2016b isso aconteceria:

>> magic(3) + (1:3)
Error using  + 
Matrix dimensions must agree.

Considerando que a partir de R2016b a operação anterior será bem-sucedida:

>> magic(3) + (1:3)
ans =

     9     3     9
     4     7    10
     5    11     5

Exemplos de tamanhos compatíveis: #

Descrição 1st Tamanho da matriz 2nd Tamanho da matriz Tamanho do resultado
Vetorial e escalar [3x1] [1x1] [3x1]
Vetores de linha e coluna [1x3] [2x1] [2x3]
Vetor e matriz 2D [1x3] [5x3] [5x3]
Matrizes N-D e K-D [1x3x3] [5x3x1x4x2] [5x3x3x4x2]

Exemplos de tamanhos incompatíveis: #

Descrição 1st Tamanho da matriz 2nd Tamanho da matriz Possível solução alternativa
Vetores onde uma dimensão é um múltiplo da mesma dimensão na outra matriz. [1x2] [1x8] transpor
Matrizes com dimensões que são múltiplas umas das outras. [2x2] [8x8] repmat, remodelar
Arrays N-D que têm a quantidade certa de dimensões singleton, mas estão na ordem errada (#1). [2x3x4] [2x4x3] permutar
Arrays N-D que têm a quantidade certa de dimensões singleton, mas estão na ordem errada (#2). [2x3x4x5] [5x2] permutar

IMPORTANTE:
O código baseado nesta convenção NÃO é compatível com versões anteriores de qualquer versão anterior do MATLAB. Portanto, a invocação explícita de bsxfun1,[2][5] (que atinge o mesmo efeito) deve ser usada se o código precisar ser executado em versões mais antigas do MATLAB. Se tal preocupação não existir, notas de versão do MATLAB R2016 encorajam os usuários a mudar de bsxfun:

Em comparação com o uso de bsxfun, a expansão implícita oferece velocidade de execução mais rápida, melhor uso de memória e melhor legibilidade do código.

Leitura relacionada:

  • Documentação do MATLAB em “[Compatible Array Sizes for Basic Operations][6]”.
  • Transmissão do NumPy[1][7],[2][8].
  • Uma comparação entre a [velocidade de computação usando bsxfun vs. expansão de array implícita] [9].

[5]: https://www.wikiod.com/pt/matlab/vetorizacao#Uso de bsxfun [6]: http://www.mathworks.com/help/matlab/matlab_prog/compatible-array-sizes-for-basic-operations.html [7]: http://docs.scipy.org/doc/numpy/user/basics.broadcasting.html [8]: https://www.wikiod.com/pt/numpy/matrizes#Operações de matriz de transmissão [9]: http://stackoverflow.com/questions/42559922/how-much-faster-is-implicit-expansion-compared-with-bsxfun

Operações elementares #

MATLAB suporta (e encoraja) operações vetorizadas em vetores e matrizes. Por exemplo, suponha que temos A e B, duas matrizes n-by-m e queremos que C seja o produto elemento a elemento dos elementos correspondentes (ou seja, C(i, j) = A(i,j)*B(i,j)).

A maneira não vetorizada, usando loops aninhados, é a seguinte:

C = zeros(n,m);
for ii=1:n
    for jj=1:m
        C(ii,jj) = A(ii,jj)*B(ii,jj);
    end
end

No entanto, a maneira vetorizada de fazer isso é usando o operador element-wise .*:

C = A.*B;
  • Para mais informações sobre a multiplicação elemento a elemento no MATLAB veja a documentação de times.
  • Para obter mais informações sobre a diferença entre operações de matriz e matriz, consulte Operações de matriz versus matriz na documentação do MATLAB.

Mascaramento Lógico #

O MATLAB suporta o uso de mascaramento lógico para realizar a seleção em uma matriz sem o uso de laços for ou instruções if.

Uma máscara lógica é definida como uma matriz composta apenas por 1 e 0.

Por exemplo:

mask = [1 0 0; 0 1 0; 0 0 1];

é uma matriz lógica que representa a matriz identidade.

Podemos gerar uma máscara lógica usando um predicado para consultar uma matriz.

A = [1 2 3; 4 5 6; 7 8 9];
B = A > 4;

Primeiro criamos uma matriz 3x3, A, contendo os números de 1 a 9. Em seguida, consultamos A para valores maiores que 4 e armazenamos o resultado em uma nova matriz chamada B.

B é uma matriz lógica da forma:

B = [0 0 0
     0 1 1
     1 1 1]

Ou 1 quando o predicado A > 4 era verdadeiro. E ‘0’ quando era falso.

Podemos usar matrizes lógicas para acessar elementos de uma matriz. Se uma matriz lógica for usada para selecionar elementos, índices onde um 1 aparece na matriz lógica serão selecionados na matriz que você está selecionando.

Usando o mesmo B de cima, poderíamos fazer o seguinte:

C = [0 0 0; 0 0 0; 0 0 0];
C(B) = 5;

Isso selecionaria todos os elementos de C onde B tem um 1 nesse índice. Esses índices em C são então definidos como 5.

Nosso C agora se parece com:

C = [0 0 0
     0 5 5
     5 5 5]

Podemos reduzir blocos de código complicados contendo if e for usando máscaras lógicas.

Pegue o código não vetorizado:

A = [1 3 5; 7 9 11; 11 9 7];
for j = 1:length(A)
  if A(j) > 5
    A(j) = A(j) - 2;
  end
end

Isso pode ser encurtado usando mascaramento lógico para o seguinte código:

A = [1 3 5; 7 9 11; 11 9 7];
B = A > 5;
A(B) = A(B) - 2;

Ou ainda mais curto:

A = [1 3 5; 7 9 11; 11 9 7];
A(A > 5) = A(A > 5) - 2;

Soma, média, prod & co #

Dado um vetor aleatório

v = rand(10,1);

se você quiser a soma de seus elementos, NÃO use um loop

s = 0;
for ii = 1:10
    s = s + v(ii);
end

mas use o recurso vetorizado da função sum()

s = sum(v);

Funções como sum(), [mean()](http://uk.mathworks.com/help /matlab/ref/mean.html), prod() e outros, têm a capacidade de operar diretamente ao longo de linhas, colunas ou outras dimensões.

Por exemplo, dada uma matriz aleatória

A = rand(10,10);

a média para cada coluna é

m = mean(A,1);

a média para cada linha é

m = mean(A,2)

Todas as funções acima funcionam apenas em uma dimensão, mas e se você quiser somar toda a matriz? Você poderia usar:

s = sum(sum(A))

Mas e se tiver uma matriz ND? aplicar sum em sum em sum… não parece ser a melhor opção, em vez disso, use o operador : para vetorizar seu array:

s = sum(A(:))

e isso resultará em um número que é a soma de todo o seu array, não importa quantas dimensões ele tenha.

Obtém o valor de uma função de dois ou mais argumentos #

Em muitas aplicações é necessário calcular a função de dois ou mais argumentos.

Tradicionalmente, usamos for-loops. Por exemplo, se precisarmos calcular o f = exp(-x^2-y^2) (não use isso se precisar de simulações rápidas):

% code1
x = -1.2:0.2:1.4;
y = -2:0.25:3;
for nx=1:lenght(x)
   for ny=1:lenght(y)
      f(nx,ny) = exp(-x(nx)^2-y(ny)^2);
   end
end

Mas a versão vetorizada é mais elegante e rápida:

% code2
[x,y] = ndgrid(-1.2:0.2:1.4, -2:0.25:3);
f = exp(-x.^2-y.^2);

do que podemos visualizar:

surf(x,y,f)

Nota1 - Grades: Normalmente, o armazenamento de matrizes é organizado linha por linha. Mas no MATLAB, é o armazenamento coluna por coluna como no FORTRAN. Assim, existem duas funções simuladas ndgrid e meshgrid no MATLAB para implementar os dois modelos mencionados. Para visualizar a função no caso de meshgrid, podemos usar:

surf(y,x,f)

Nota2 - Consumo de memória: Deixe o tamanho de x ou y ser 1000. Assim, precisamos armazenar elementos 1000*1000+2*1000 ~ 1e6 para *code1 não vetorizado *. Mas precisamos de elementos 3*(1000*1000) = 3e6 no caso de code2 vetorizado. No caso 3D (deixe que z tenha o mesmo tamanho que x ou y), o consumo de memória aumenta drasticamente: 4*(1000*1000*1000) (~32GB para duplos) no caso do vetorizado code2 vs ~1000*1000*1000 (apenas ~8GB) no caso de code1. Assim, temos que escolher a memória ou a velocidade.
