Os operadores de transposição #

  • .' é a maneira correta de transpor um vetor ou matriz no MATLAB.
  • ' é a maneira correta de obter a transposição de conjugado complexo (também conhecido como conjugado Hermitiano) de um vetor ou matriz no MATLAB.

Observe que para a transposição .', há um período na frente do apóstrofo. Isso está de acordo com a sintaxe para as outras operações elementares no MATLAB: * multiplica matrizes, .* multiplica elementos de matrizes juntos. Os dois comandos são muito semelhantes, mas conceitualmente muito distintos. Como outros comandos do MATLAB, esses operadores são “açúcar sintático” que se transforma em uma chamada de função “apropriada” em tempo de execução. Assim como == se torna uma avaliação da função eq, pense em .' como a abreviação de transpose. Se você escrever apenas ' (sem o ponto), você está de fato usando o comando ctranspose, que calcula a transposição conjugada complexa, que também é conhecida como Hermitian conjugate, frequentemente usado em física. Desde que o vetor ou matriz transposta seja de valor real, os dois operadores produzem o mesmo resultado. Mas assim que lidarmos com números complexos, inevitavelmente teremos problemas se não usarmos a abreviação “correta”. O que é “correto” depende da sua aplicação.

Considere o seguinte exemplo de uma matriz C contendo números complexos:

>> C = [1i, 2; 3*1i, 4]
C =
   0.0000 + 1.0000i   2.0000 + 0.0000i
   0.0000 + 3.0000i   4.0000 + 0.0000i

Vamos fazer a transposição usando a abreviação .' (com o ponto). A saída é como esperado, a forma transposta de C.

>> C.'
ans =
   0.0000 + 1.0000i   0.0000 + 3.0000i
   2.0000 + 0.0000i   4.0000 + 0.0000i

Agora, vamos usar ' (sem o ponto). Vemos que, além da transposição, os valores complexos também foram transformados em seus conjugados complexos.

>> C'
ans =
   0.0000 - 1.0000i   0.0000 - 3.0000i
   2.0000 + 0.0000i   4.0000 + 0.0000i

Para resumir, se você pretende calcular o conjugado Hermitiano, o conjugado complexo transpõe, então use ' (sem o ponto). Se você quiser apenas calcular a transposição sem conjugar os valores complexos, use .' (com o ponto).

Não nomeie uma variável com um nome de função existente #

Já existe uma função sum(). Como resultado, se nomearmos uma variável com o mesmo nome

sum = 1+3;

e se tentarmos usar a função enquanto a variável ainda existir na área de trabalho

A = rand(2);
sum(A,1)

obteremos o erro enigmático:

Subscript indices must either be real positive integers or logicals.

clear() a variável primeiro e depois use a função

clear sum

sum(A,1)
ans =
       1.0826       1.0279

Como podemos verificar se já existe uma função para evitar esse conflito?

Use which() com o sinalizador -all:

which sum -all
sum is a variable.
built-in (C:\Program Files\MATLAB\R2016a\toolbox\matlab\datafun\@double\sum)   % Shadowed double method
...

Esta saída está nos dizendo que sum é primeiro uma variável e que os seguintes métodos (funções) são sombreados por ela, ou seja, MATLAB primeiro tentará aplicar nossa sintaxe à variável, em vez de usar o método.

Esteja ciente da imprecisão do ponto flutuante #

Os números de ponto flutuante não podem representar todos os números reais. Isso é conhecido como imprecisão de ponto flutuante.

Existem infinitos números de pontos flutuantes e eles podem ser infinitamente longos (por exemplo, π), portanto, ser capaz de representá-los perfeitamente exigiria uma quantidade infinita de memória. Vendo que isso era um problema, foi projetada uma representação especial para armazenamento de “número real” em computador, o padrão IEEE 754. Em suma, descreve como os computadores armazenam esse tipo de número, com um expoente e mantissa, como,

``float = sinal * 2^ expoente * mantissa''

Com uma quantidade limitada de bits para cada um deles, apenas uma precisão finita pode ser alcançada. Quanto menor o número, menor o intervalo entre os números possíveis (e vice-versa!). Você pode tentar seus números reais nesta demonstração online.

Esteja ciente desse comportamento e tente evitar todas as comparações de pontos flutuantes e seu uso como condições de parada em loops. Veja abaixo dois exemplos:

Exemplos: comparação de ponto flutuante feita ERRADA: #

>> 0.1 + 0.1 + 0.1  == 0.3

ans =

  logical

   0

É uma prática ruim usar a comparação de ponto flutuante conforme mostrado pelo exemplo anterior. Você pode superá-lo tomando o valor absoluto de sua diferença e comparando-o com um nível de tolerância (pequeno).

Abaixo está outro exemplo, onde um número de ponto flutuante é usado como condição de parada em um loop while:**

k = 0.1;
while k <= 0.3 
  disp(num2str(k));
  k = k + 0.1;
end

% --- Output: ---
0.1
0.2

Ele perde o último loop esperado (0.3 <= 0.3).

Exemplo: comparação de ponto flutuante concluída DIREITA: #

x = 0.1 + 0.1 + 0.1;
y = 0.3;
tolerance = 1e-10; % A "good enough" tolerance for this case.

if ( abs( x - y ) <= tolerance )
  disp('x == y');
else
  disp('x ~= y');
end

% --- Output: ---
x == y

Várias coisas a serem observadas:

  • Como esperado, agora x e y são tratados como equivalentes.
  • No exemplo acima, a escolha da tolerância foi feita de forma arbitrária. Assim, o valor escolhido pode não ser adequado para todos os casos (especialmente ao trabalhar com números muito menores). A escolha do limite inteligente pode ser feita usando a função eps, ou seja, N*eps(max(x,y)), onde N é algum número específico do problema. Uma escolha razoável para N, que também é permissiva o suficiente, é 1E2 (embora, no problema acima, N=1 também seja suficiente).

O que você vê NÃO é o que você obtém: char vs cellstring na janela de comando #

Este é um exemplo básico voltado para novos usuários. Ele não se concentra em explicar a diferença entre char e cellstring.

Pode acontecer que você queira se livrar do ' em suas strings, embora você nunca os tenha adicionado. Na verdade, esses são artefatos que a janela de comando usa para distinguir entre alguns tipos.

Uma string será impressa

s = 'dsadasd'
s =
dsadasd

Uma cellstring imprimirá

c = {'dsadasd'};
c = 
    'dsadasd'

Observe como as aspas simples e o recuo são artefatos para nos notificar que c é uma cellstring em vez de um char. A string está de fato contida na célula, ou seja,

c{1}
ans =
dsadasd

Função indefinida ou método X para argumentos de entrada do tipo Y #

Esta é a maneira prolixa do MATLAB de dizer que não pode encontrar a função que você está tentando chamar. Há vários motivos pelos quais você pode receber esse erro:

Essa função foi introduzida depois da sua versão atual do MATLAB #

A documentação online do MATLAB fornece um recurso muito bom que permite determinar em qual versão uma determinada função foi introduzida. Ele está localizado no canto inferior esquerdo de cada página da documentação:

Compare esta versão com sua própria versão atual (ver) para determinar se esta função está disponível em sua versão específica. Se não for, tente pesquisar as versões arquivadas da documentação para encontrar uma alternativa adequada em sua versão.

Você não tem essa caixa de ferramentas! #

A instalação básica do MATLAB possui um grande número de funções; no entanto, funcionalidades mais especializadas são empacotadas em caixas de ferramentas e vendidas separadamente pelo Mathworks. A documentação para todas as caixas de ferramentas é visível se você tem a caixa de ferramentas ou não, então certifique-se de verificar e ver se você tem a caixa de ferramentas apropriada.

Para verificar a qual caixa de ferramentas uma determinada função pertence, olhe no canto superior esquerdo da documentação online para ver se uma caixa de ferramentas específica é mencionada.

Você pode então determinar quais caixas de ferramentas sua versão do MATLAB instalou emitindo o comando ver que imprimirá uma lista de todas as caixas de ferramentas instaladas.

Se você não tiver essa caixa de ferramentas instalada e quiser usar a função, precisará adquirir uma licença para essa caixa de ferramentas específica do The Mathworks.

MATLAB não consegue localizar a função #

Se o MATLAB ainda não encontrar sua função, ela deve ser uma função definida pelo usuário. É possível que ele resida em outro diretório e esse diretório deva ser [adicionado ao caminho de pesquisa](http://www.mathworks.com/help/matlab/matlab_env/add-remove-or-reorder-folders-on- the-search-path.html) para que seu código seja executado. Você pode verificar se o MATLAB pode localizar sua função usando which que deve retornar o caminho para o arquivo de origem.

O uso de “i” ou “j” como unidade imaginária, índices de loop ou variável comum. #

Recomendação #

Como os símbolos i e j podem representar coisas significativamente diferentes no MATLAB, seu uso como índices de loop dividiu a comunidade de usuários do MATLAB desde muito tempo. Embora alguns motivos históricos de desempenho possam ajudar o equilíbrio a se inclinar para um lado, esse não é mais o caso e agora a escolha depende inteiramente de você e das práticas de codificação que você escolher seguir.

As recomendações oficiais atuais do Mathworks são:

  • Como i é uma função, ela pode ser substituída e usada como uma variável. No entanto, é melhor evitar usar i e j para nomes de variáveis ​​se você pretende usá-los em aritmética complexa.
  • Para velocidade e robustez melhorada em aritmética complexa, use 1i e 1j em vez de i e j.

Predefinição #

No MATLAB, por padrão, as letras i e [j](http://uk.mathworks.com/help /matlab/ref/j.html) são nomes de função embutidos, que se referem à unidade imaginária no domínio complexo.

Então, por padrão i = j = sqrt(-1).

>> i
ans =
   0.0000 + 1.0000i
>> j
ans =
   0.0000 + 1.0000i

e como você deve esperar:

>> i^2
ans =
    -1

Usando-os como uma variável (para índices de loop ou outra variável)

O MATLAB permite usar o nome da função integrada como uma variável padrão. Nesse caso, o símbolo usado não apontará mais para a função interna, mas para sua própria variável definida pelo usuário. Essa prática, no entanto, geralmente não é recomendada, pois pode levar a confusão, depuração e manutenção difíceis (veja outro exemplo [do-not-name-a-variable-with-an-existing-function-name](http:// stackoverflow.com/documentation/matlab/973/common-mistakes-and-errors/3168/do-not-name-a-variable-with-an-existing-function-name)).

Se você for ultra pedante em respeitar as convenções e as melhores práticas, evitará usá-las como índices de loop nessa linguagem. No entanto, é permitido pelo compilador e perfeitamente funcional, então você também pode optar por manter hábitos antigos e usá-los como iteradores de loop.

>> A = nan(2,3);
>> for i=1:2        % perfectly legal loop construction
       for j = 1:3
        A(i, j) = 10 * i + j;
       end
   end

Observe que os índices de loop não saem do escopo no final do loop, portanto, eles mantêm seu novo valor.

>> [ i ; j ]
ans =
     2
     3

No caso de você usá-los como variáveis, certifique-se de eles foram inicializados antes de serem usados. No loop acima, o MATLAB os inicializa automaticamente quando prepara o loop, mas se não for inicializado corretamente, você pode ver rapidamente que pode introduzir inadvertidamente números complexos em seu resultado.

Se mais tarde você precisar desfazer o sombreamento da função embutida (= por exemplo, você quer que i e j representem a unidade imaginária novamente), você pode limpar as variáveis:

>> clear i j

Você entende agora a reserva do Mathworks sobre usá-los como índices de loop se você pretende usá-los em aritmética complexa. Seu código estaria cheio de inicializações de variáveis ​​e comandos clear, a melhor maneira de confundir o programador mais sério (sim, você aí!…) e acidentes de programa esperando para acontecer.

Se nenhuma aritmética complexa for esperada, o uso de i e j é perfeitamente funcional e não há penalidade de desempenho.

Usando-os como unidade imaginária: #

Se o seu código tem que lidar com números complexos, então i e j certamente serão úteis. No entanto, por uma questão de desambiguação e até mesmo para performances, recomenda-se usar a forma completa em vez da sintaxe abreviada. A forma completa é 1i (ou 1j).

>> [ i ; j ; 1i ; 1j]
ans =
   0.0000 + 1.0000i
   0.0000 + 1.0000i
   0.0000 + 1.0000i
   0.0000 + 1.0000i

Eles representam o mesmo valor sqrt(-1), mas a forma posterior:

  • é mais explícito, de forma semântica.
  • é mais sustentável (alguém olhando seu código mais tarde não tem que ler o código para descobrir se i ou j era uma variável ou a unidade imaginária).
  • é mais rápido (fonte: Mathworks).

Observe que a sintaxe completa 1i é válida com qualquer número antes do símbolo:

>> a = 3 + 7.8j
a =
   3.0000 + 7.8000i

Esta é a única função que você pode manter com um número sem um operador entre eles.

Armadilhas #

Embora seu uso como unidade imaginária OR variável seja perfeitamente legal, aqui está apenas um pequeno exemplo de como pode ficar confuso se os dois usos forem misturados:

Vamos substituir i e torná-lo uma variável:

>> i=3
i =
     3

Agora i é uma variável (mantendo o valor 3), mas apenas sobrescrevemos a notação shorthand da unidade imaginária, a forma completa ainda é interpretada corretamente:

>> 3i
ans =
   0.0000 + 3.0000i

O que agora nos permite construir as formulações mais obscuras. Eu deixo você avaliar a legibilidade de todas as seguintes construções:

>> [ i ; 3i ; 3*i ; i+3i ; i+3*i ]
ans =
   3.0000 + 0.0000i
   0.0000 + 3.0000i
   9.0000 + 0.0000i
   3.0000 + 3.0000i
  12.0000 + 0.0000i

Como você pode ver, cada valor no array acima retorna um resultado diferente. Embora cada resultado seja válido (desde que essa tenha sido a intenção inicial), a maioria de vocês admitirá que seria um verdadeiro pesadelo ler um código repleto de tais construções.

Não há argumentos de entrada suficientes #

Muitas vezes, os desenvolvedores iniciantes do MATLAB usam o editor do MATLAB para escrever e editar código, em particular funções personalizadas com entradas e saídas. Há um botão Executar na parte superior que está disponível nas versões recentes do MATLAB:

Uma vez que o desenvolvedor termina com o código, muitas vezes eles são tentados a apertar o botão Run. Para algumas funções, isso funcionará bem, mas para outras, elas receberão um erro ‘Não há argumentos de entrada suficientes’ e ficarão confusos sobre o motivo pelo qual o erro ocorre.

A razão pela qual esse erro pode não acontecer é porque você escreveu um script MATLAB ou uma função que não recebe argumentos de entrada. Usar o botão Executar executará um script de teste ou executará uma função assumindo que não há argumentos de entrada. Se sua função requer argumentos de entrada, o erro Não há argumentos de entrada suficientes ocorrerá porque você escreveu uma função que espera que as entradas entrem na função. Portanto, você não pode esperar que a função seja executada simplesmente pressionando o botão Executar.

Para demonstrar esse problema, suponha que tenhamos uma função mult que simplesmente multiplique duas matrizes:

function C = mult(A, B)
    C = A * B;
end

Nas versões recentes do MATLAB, se você escreveu esta função e pressionou o botão Executar, ela apresentará o erro que esperamos:

>> mult
Not enough input arguments.

Error in mult (line 2)
    C = A * B;

Existem duas maneiras de resolver esse problema:

Método #1 - Através do Prompt de Comando #

Simplesmente crie as entradas que você precisa no prompt de comando e execute a função usando as entradas que você criou:

A = rand(5,5);
B = rand(5,5);
C = mult(A,B);

Método #2 - Interativamente através do Editor #

Abaixo do botão Run, há uma seta preta escura. Se você clicar nessa seta, poderá especificar as variáveis ​​que gostaria de obter da área de trabalho do MATLAB digitando a maneira como deseja chamar a função exatamente como viu no método #1. Certifique-se de que as variáveis ​​que você está especificando dentro da função existem na área de trabalho do MATLAB:

Usando length para arrays multidimensionais #

Um erro comum dos codificadores MATLAB é usar a função length para matrizes (ao contrário de vetores, para os quais ela se destina). A função length, conforme mencionado em sua documentação, “retorna o comprimento da maior dimensão do array” da entrada.

Para vetores, o valor de retorno de length tem dois significados diferentes:

  1. O número total de elementos no vetor.
  2. A maior dimensão do vetor.

Ao contrário dos vetores, os valores acima não seriam iguais para matrizes de mais de uma dimensão não-singleton (ou seja, cujo tamanho é maior que 1). É por isso que usar length para matrizes é ambíguo. Em vez disso, o uso de uma das seguintes funções é incentivado, mesmo ao trabalhar com vetores, para deixar a intenção do código perfeitamente clara:

  1. size(A) - retorna um vetor de linha cujos elementos contêm a quantidade de elementos ao longo da dimensão correspondente de A .
  2. numel(A) - retorna o número de elementos em A. Equivalente a prod(tamanho(A)).
  3. ndims(A) - retorna o número de dimensões na matriz A. Equivalente a numel(tamanho(A)).

Isso é especialmente importante ao escrever funções de biblioteca “à prova de futuro”, vetorizadas, cujas entradas não são conhecidas antecipadamente e podem ter vários tamanhos e formas.

Cuidado com as mudanças no tamanho do array #

Algumas operações comuns no MATLAB, como diferenciação ou integração, geram resultados com uma quantidade de elementos diferente da dos dados de entrada. Este fato pode ser facilmente ignorado, o que normalmente causaria erros como ‘As dimensões da matriz devem concordar’. Considere o seguinte exemplo:

t = 0:0.1:10;        % Declaring a time vector
y = sin(t);          % Declaring a function

dy_dt = diff(y);     % calculates dy/dt for y = sin(t)

Digamos que queremos plotar esses resultados. Damos uma olhada nos tamanhos dos arrays e vemos:

size(y) is 1x101
size(t) is 1x101

Mas:

size(dy_dt) is 1x100

A matriz é um elemento menor!

Agora imagine que você tenha dados de medição de posições ao longo do tempo e queira calcular jerk(t), você obterá um array 3 elementos a menos que o array de tempo (porque o jerk é a posição diferenciada 3 vezes).

vel = diff(y);       % calculates velocity vel=dy/dt for y = sin(t)  size(vel)=1x100
acc = diff(vel);     % calculates acceleration acc=d(vel)/dt         size(acc)=1x99
jerk = diff(acc);    % calculates jerk jerk=d(acc)/dt                size(jerk)=1x98

E então operações como:

x = jerk .* t;          % multiplies jerk and t element wise

retornam erros, porque as dimensões da matriz não concordam.

Para calcular operações como acima, você deve ajustar o tamanho maior do array para caber no menor. Você também pode executar uma regressão (polyfit) com seus dados para obter um polinômio para seus dados.

Erros de incompatibilidade de dimensão #

Os erros de Incompatibilidade de dimensão geralmente aparecem quando:

  • Não prestar atenção ao formato das variáveis ​​retornadas das chamadas de função/método. Em muitas funções embutidas do MATLAB, as matrizes são convertidas em vetores para acelerar os cálculos, e a variável retornada ainda pode ser um vetor em vez da matriz que esperávamos. Esse também é um cenário comum quando mascaramento lógico está envolvido.
  • Usando tamanhos de array incompatíveis ao invocar [expansão de array implícita] 2.
