reactive, reactiveValue e eventReactive, observe e observeEvent em Shiny

reactive, reactiveValue e eventReactive são vários tipos de expressões reativas no Shiny. Eles produzem uma saída que pode ser usada como entrada em outras expressões, que por sua vez terão uma dependência da expressão reativa.

observe e observeEvent são semelhantes a expressões reativas. A grande diferença é que os observadores não produzem nenhum resultado e, portanto, são úteis apenas por seus efeitos colaterais.

Exemplos de seu uso são dados neste documento.

reativo #

Um reativo pode ser usado para tornar a saída dependente de outra expressão. No exemplo abaixo, o elemento output$text é dependente de text_reactive, que por sua vez é dependente de input$user_text. Sempre que input$user_text muda, o elemento output$text e text_reactive se tornam invalidados. Eles são recalculados com base no novo valor para input$user_text.

library(shiny)

ui <- fluidPage(
  headerPanel("Example reactive"),
  
  mainPanel(
    
    # input field
    textInput("user_text", label = "Enter some text:", placeholder = "Please enter some text."),
    
    # display text output
    textOutput("text"))
)

server <- function(input, output) {
  
  # reactive expression
  text_reactive <- reactive({
    input$user_text
  })
  
  # text output
  output$text <- renderText({
    text_reactive()
  })
}

shinyApp(ui = ui, server = server)

eventoReativo #

eventReactives são semelhantes aos reactivos, eles são construídos da seguinte forma:

eventReactive( event {
code to run
})

eventReactives não são dependentes de todas as expressões reativas em seu corpo (’code to run’ no snippet acima). Em vez disso, eles dependem apenas das expressões especificadas na seção evento.

No exemplo abaixo, adicionamos um botão de envio e criamos um eventReactive. Sempre que input$user_text muda, o eventReactive não é invalidado, pois o eventReactive depende apenas do actionButton input$submit. Sempre que esse botão é pressionado, text_reactive e subsequentemente output$text são invalidados e serão recalculados com base no input$user_text atualizado.

library(shiny)

ui <- fluidPage(
  headerPanel("Example eventReactive"),
  
  mainPanel(
    
    # input field
    textInput("user_text", label = "Enter some text:", placeholder = "Please enter some text."),

    # submit button
    actionButton("submit", label = "Submit"),
    
    # display text output
    textOutput("text"))
)

server <- function(input, output) {
  
  # reactive expression
  text_reactive <- eventReactive( input$submit, {
    input$user_text
  })
  
  # text output
  output$text <- renderText({
    text_reactive()
  })
}

shinyApp(ui = ui, server = server)

reactiveValues #

reactiveValues ​​pode ser usado para armazenar objetos, dos quais outras expressões podem ter uma dependência.

No exemplo abaixo, um objeto reactiveValues ​​é inicializado com o valor “Nenhum texto foi enviado ainda.”. Um observador separado é criado para atualizar o objeto reactiveValues ​​sempre que o botão de envio é pressionado. Observe que o próprio reactiveValues ​​não depende das expressões em seu corpo.

library(shiny)

ui <- fluidPage(
  headerPanel("Example reactiveValues"),
  
  mainPanel(
    
    # input field
    textInput("user_text", label = "Enter some text:", placeholder = "Please enter some text."),
    actionButton("submit", label = "Submit"),
    
    # display text output
    textOutput("text"))
)

server <- function(input, output) {
  
  # observe event for updating the reactiveValues
  observeEvent(input$submit,
               {
    text_reactive$text <- input$user_text
  })
  
  # reactiveValues
  text_reactive <- reactiveValues(
    text = "No text has been submitted yet."
  )
  
  # text output
  output$text <- renderText({
    text_reactive$text
  })
}

shinyApp(ui = ui, server = server)

observeEvento #

Um objeto observeEvent pode ser usado para acionar um trecho de código quando ocorre um determinado evento. É construído como:

observeEvent( event {
code to run
})

O observeEvent será dependente apenas da seção ’event’ no pequeno pedaço de código acima. Não será dependente de nada na parte ‘code to run’. Um exemplo de implementação pode ser encontrado abaixo:

library(shiny)

ui <- fluidPage(
  headerPanel("Example reactive"),
  
  mainPanel(
    
    # action buttons
    actionButton("button1","Button 1"),
    actionButton("button2","Button 2")
  )
)

server <- function(input, output) {
  
  # observe button 1 press.
  observeEvent(input$button1, {
    # The observeEvent takes no dependency on button 2, even though we refer to the input in the following line.
    input$button2  
    showModal(modalDialog(
      title = "Button pressed",
      "You pressed one of the buttons!"
    ))
  })
}

shinyApp(ui = ui, server = server)

observar #

Uma expressão de observação é acionada toda vez que uma de suas entradas muda. A principal diferença em relação a uma expressão reativa é que ela não produz saída e deve ser usada apenas para seus efeitos colaterais (como modificar um objeto reactiveValues ​​ou acionar um pop-up).

Além disso, observe que observe não ignora NULLs, portanto, ele será acionado mesmo que suas entradas ainda sejam NULL. observeEvent por padrão ignora NULL, como é quase sempre desejável.

library(shiny)

ui <- fluidPage(
  headerPanel("Example reactive"),
  
  mainPanel(
    
    # action buttons
    actionButton("button1","Button 1"),
    actionButton("button2","Button 2")
  )
)

server <- function(input, output) {
  
  # observe button 1 press.
  observe({
    input$button1
    input$button2  
    showModal(modalDialog(
      title = "Button pressed",
      "You pressed one of the buttons!"
    ))
  })
}

shinyApp(ui = ui, server = server)
