Existem várias funções e instruções incorporadas para acessar o sistema operacional.

OS-COMMAND #

Executa um comando do SO.

OS-COMMAND sem nenhuma opção iniciará um novo shell e não sairá dele - assim, você deixará em OS gráficos: es uma janela “pendurada”.

DEFINE VARIABLE cmd AS CHARACTER   NO-UNDO.

cmd = "dir".

OS-COMMAND VALUE(cmd).

Existem três opções: SILENT, NO-WAIT e NO-CONSOLE.

SILENCIOSO

Depois de processar um comando do sistema operacional, o shell AVM pausa. Para sair da janela em plataformas Windows GUI, você deve digitar exit. Para sair da janela nas plataformas de caracteres do Windows, você deve digitar exit e pressionar RETURN ou BARRA DE ESPAÇOS. Você pode usar a opção SILENT para eliminar essa pausa. Use esta opção apenas se tiver certeza de que o programa, comando ou arquivo em lote não gera nenhuma saída na tela. Não pode ser usado com NO-WAIT.

OS-COMMAND SILENT VALUE("runprogram.exe").

NÃO AGUARDE

Em um ambiente multitarefa, faz com que o AVM passe imediatamente o controle de volta para a próxima instrução após o OS-COMMAND sem esperar que o comando do sistema operacional termine. Não pode ser usado com SILENT. Esta opção é suportada apenas no Windows.

OS-COMMAND NO-WAIT VALUE("DIR > dirfile.txt").

No Linux/Unix, você terá que conseguir isso precedendo o comando com um sinal &:

OS-COMMAND VALUE("ls >> file.txt &").

SEM CONSOLE

Ao processar um comando do sistema operacional, o AVM cria uma janela de console. A janela do console não pode ser limpa após a execução do comando. Você pode usar a opção NO-CONSOLE para evitar que esta janela seja criada em primeiro lugar.

OS-COMMAND NO-CONSOLE VALUE("startbach.bat").

Nenhum erro é retornado do OS-COMMAND para o Progress ABL, então você deve verificar os erros de outra maneira, possivelmente gravando-os em um arquivo em um script de shell ou similar.

OPSYS #

A função OPSYS retorna em qual sistema operacional o programa está sendo executado:

MESSAGE OPSYS VIEW-AS ALERT-BOX.

Resultado:

Ele pode ser usado para selecionar qual utilitário do SO chamar:

IF OPSYS = "LINUX" THEN 
    OS-COMMAND VALUE("ls -l").
ELSE 
    OS-COMMAND VALUE("dir").

OS-ERROR #

Retorna um erro de uma chamada anterior de OS-* representada por um inteiro. As chamadas que podem retornar um OS-ERROR são:

  • OS-APPEND
  • CÓPIA DO SO
  • OS-CREATE-DIR
  • OS-DELETE
  • OS-RENAME
  • SALVAR CACHE

Observe que o OS-COMMAND está ausente. Você mesmo precisa lidar com erros no OS-COMMAND.

Número do erro Descrição
0 Sem erro
1 Não proprietário
2 Nenhum arquivo ou diretório
3 Chamada de sistema interrompida
4 Erro de E/S
5 Número de arquivo incorreto
6 Sem mais processos
7 Memória de núcleo insuficiente
8 Permissão negada
9 Endereço incorreto
10 O arquivo existe
11 Sem tal dispositivo
12 Não é um diretório
13 É um diretório
14 Estouro da tabela de arquivos
15 Muitos arquivos abertos
16 Arquivo muito grande
17 Não há espaço no dispositivo
18 Diretório não vazio
999 Erro não mapeado (padrão ABL)

Função OS-GETENV #

Retorna o valor de qualquer variável de ambiente do SO.

MESSAGE OS-GETENV ("OS") VIEW-AS ALERT-BOX.

Em uma máquina Windows:

MESSAGE OS-GETENV ("SHELL") VIEW-AS ALERT-BOX.

Resultado em uma máquina Linux com Bash como shell atual:

                        ┌────── Message ───────┐
                        │      /bin/bash       │
                        │ ──────────────────── │
                        │         <OK>         │
                        └──────────────────────┘

OS-COPY #

Copiar um arquivo

COPIAR arquivo de origem arquivo de destino

Copie c:\temp\source-file.txt para c:\temp\target-file.txt. Você precisa verificar OS-ERROR para o sucesso ou a falta dele.

OS-COPY VALUE("c:\temp\source-file.txt")  VALUE("c:\temp\target-file.txt").
IF OS-ERROR <> 0 THEN DO:
    MESSAGE "An error occured" VIEW-AS ALERT-BOX ERROR.
END.

OS-DELETE #

Exclui um arquivo ou uma árvore de arquivos.

Tal como acontece com muitos outros utilitários OS-*, você deve verificar o status em OS-ERROR.

OS-DELETE arquivo-ou-dir-para-excluir [ RECURSIVE ]

Exclua toda a árvore /tmp/dir:

OS-DELETE VALUE("/tmp/dir") RECURSIVE.

Exclua o arquivo chamado c:\dir\file.txt

OS-DELETE VALUE("c:\dir\file.txt").

OS-CREATE-DIR #

Cria um diretório, o status está em OS-ERROR

Diretório OS-CREATE-DIR

Crie um diretório chamado /usr/local/appData

OS-CREATE-DIR VALUE("/usr/local/appData").

OS-APPEND #

Anexar um arquivo a outro. O status é verificado em OS-ERROR

Destino de origem OS-APPEND

Anexa targetfile.txt com sourcefile.txt:

OS-APPEND VALUE("sourcefile.txt") VALUE("targetfile.txt").

OS-RENAME #

Renomeie um arquivo ou diretório. O status está em “OS-ERROR”. Também pode ser usado para mover arquivos (ou mover e renomear).

OS-RENAME nome antigo novo nome

Renomeie /tmp/old-name para /tmp/new-name:

OS-RENAME VALUE("/tmp/old-name") VALUE("/tmp/new-name").

Mova o arquivo c:\temp\old.txt para c:\new-dir\old.txt:

OS-RENAME VALUE("c:\temp\old.txt") VALUE("c:\new-dir\old.txt").

OS-DRIVES (somente Windows) #

Retorna uma lista de todas as unidades em um sistema.

MESSAGE OS-DRIVES VIEW-AS ALERT-BOX.

Resultado com quatro unidades, C a F:

No Linux, a lista estará simplesmente vazia, pois, por definição, não há “unidades” conectadas. A listagem de diretórios é feita de outra maneira (INPUT FROM OS-DIR)
