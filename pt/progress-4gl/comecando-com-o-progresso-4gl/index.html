FizzBuzz #

Outro exemplo de programas no estilo “Hello World” é FizzBuzz.

DEFINE VARIABLE i AS INTEGER     NO-UNDO.
DEFINE VARIABLE cOut AS CHARACTER   NO-UNDO.

DO i = 1 TO 100:

    /* Dividable by 3: fizz */
    IF i MODULO 3 = 0 THEN
        cOut = "Fizz".
    /* Dividable by 5: buzz */
    ELSE IF i MODULO 5 = 0 THEN
        cOut = "Buzz".
    /* Otherwise just the number */
    ELSE 
        cOut = STRING(i).
    
    /* Display the output */
    DISPLAY cOut WITH FRAME x1 20 DOWN.
    /* Move the display position in the frame down 1 */
    DOWN WITH FRAME x1.
END.

Instalação ou configuração #

Instalando o Progresso

Baixe sua distribuição do Progress. Se você quiser uma licença de demonstração, você precisa entrar em contato com eles. Certifique-se de baixar um arquivo tar de 64 bits e não de 32 bits (a menos que você execute uma máquina de 32 bits).

Janelas

O download será um arquivo zip. Descompacte-o e simplesmente execute setup.exe. A instalação será gráfica, mas exatamente como a descrita abaixo.

Linux/Unix/HP-UX etc

Coloque o arquivo tar em seu sistema Progress. Digamos que você o tenha em seu diretório inicial:

/home/user/PROGRESSFILENAME.tar

Extraia:

cd /home/user
tar xvf PROGRESSFILENAME.tar

Ele irá criar um diretório chamado

proinst

Altere o diretório para outro destino e crie um diretório temporário lá. Por exemplo:

cd /tmp
mkdir proinst116
cd proinst116

Quando a instalação estiver concluída, este diretório conterá informações sobre a instalação, bem como arquivos que você pode salvar e usar para futuras repetições automáticas da mesma instalação.

Agora execute o script de instalação (chamado “proinst” no diretório “proinst”):

/home/user/proinst/proinst

Instalação silenciosa

A instalação armazenou um arquivo chamado /usr/dlc/install/response.ini (ou seu diretório de instalação). Este arquivo pode ser usado para repetir exatamente a mesma instalação novamente em uma instalação “silenciosa” que pode ser scriptet e executada sem qualquer interação.

Para executar uma instalação silenciosa, basta fazer:

/path-to-proinst/proinst -b /path-to-response-file/response.ini -l /path-to-store-log/silent.log

Olá Mundo! #

Depois de iniciar seu editor de progresso de escolha (há algumas opções), basta escrever:

DISPLAY "Hello, World!".

E execute pressionando a tecla ou item de menu correspondente:

No Windows no AppBuilder: F1 (Compile -> Executar)

No Linux/Unix no editor 4GL: F2 (ou ctrl+X) (Compile -> Run)

No Windows no Developer Studio: alt+shift+X, seguido por G (Executar -> Executar como progresso Aplicativo Open Edge)

Configurando o ambiente #

Linux/Unix

Depois de instalar o Progress, é muito fácil executá-lo.

Você só precisa de algumas variáveis ​​de ambiente. O diretório onde o Progress foi instalado (padrão /usr/dlc mas pode ser outra coisa) precisa estar na variável DLC

DLC=/usr/dlc

E você também pode querer o subdiretório "bin" de DLC em seu PATH:

PATH=$PATH:$DLC/bin

Agora você está definido!

Há também um script instalado chamado proenv que fará isso (e um pouco mais) para você. Sua localização padrão é /usr/dlc/bin/proenv.

Alguns utilitários:

showcfg

Isso listará todos os seus produtos instalados.

pro

Isso iniciará o “Editor de Procedimentos”, onde você poderá editar e executar seus programas.

pro program.p

Abrirá o program.p para edição, se existir. Caso contrário, um erro será exibido.

pro -p program.p

Isso executará “program.p”. Se houver um arquivo compilado (program.r) presente ele será executado, caso contrário será compilado temporariamente e depois executado. O arquivo compilado não será salvo.

Criando o banco de dados de demonstração “sports2000” a partir da linha de comando #

Isso mostra como criar o banco de dados de demonstração usado em grande parte da documentação do Progress: sports2000.

Isso pressupõe que você instalou os produtos Progress com pelo menos um tipo de licença de banco de dados.

Execute o script/bat-file proenv que lhe dará um prompt com todas as variáveis ​​de ambiente definidas.

Crie um diretório.

Este exemplo é para Windows. A manipulação de diretórios, etc., pode ser diferente em outro sistema operacional.

proenv> cd \
proenv> mkdir db
proenv> cd db
proenv> mkdir sports2000
proenv> cd sports2000

Crie um banco de dados sports2000 usando “prodb”

proenv> prodb mySportsDb sports2000

Sintaxe de prodb:

prodb nome do novo banco de dados nome e caminho do banco de dados de origem

Isso criará um banco de dados chamado “mySportsDb” no diretório atual. Esse banco de dados é uma cópia exata do banco de dados sports2000 que acompanha a instalação do Progress. Como o banco de dados sports2000 de origem está localizado no diretório de instalação do Progress, você não precisa especificar o caminho.

Agora você pode acessar o banco de dados diretamente simplesmente digitando pro mySportsDb. Isso iniciará um Editor de Progresso conectado ao banco de dados. Esta será uma conexão de usuário único para que ninguém mais possa acessar o banco de dados ao mesmo tempo.

No editor você pode simplesmente digitar:

FOR EACH bill NO-LOCK: 
  DISPLAY bill. 
END.

Para acessar o banco de dados. Pressione Ctrl+X para executar. Isso exibirá todo o conteúdo da tabela “bill”. Se você quiser cancelar, você pode pressionar Ctrl+C.

Arquivos de Programas #

O código Progress ABL é normalmente armazenado em arquivos com diferentes finais dependendo do que eles contêm. As terminações são opcionais, mas sim um padrão de fato:

Extensão do nome do arquivo Contém
.p Um programa Progresso. Pode conter vários procedimentos internos, funções etc.
.i Incluir arquivo a ser incluído em outros arquivos
.w Um arquivo que contém uma representação gráfica de uma janela ou caixa de diálogo, baseada em WinForm.
.r O resultado compilado de qualquer arquivo contendo Progress 4GL. Chamado r-code.
.cls Uma Classe Orientada a Objetos Progress
.wrx Um contêiner para dados ActiveX sempre que necessário (gerado pela compilação em “AppBuilder”).

Para executar um arquivo de programa em Progress 4GL, a instrução RUN é usada:

RUN program.p. //Will run program.p without parameters.
RUN program.w (INPUT true). //Will run program.w with input parameter set to true.

RUN program. //Will run program.r if present otherwise internal procedure "program".

Para incluir outro arquivo em um programa Progress, a diretiva {} é usada:

{program.i} //Includes program.i in the current program

Running sports2000 como serviço #

Uma vez que o banco de dados sports2000 foi instalado, é hora de executá-lo como um servidor autônomo (e não se conectar a ele como um arquivo).

Inicie o proenv (proenv no startmeny no Windows ou /usr/install-directory/bin/proenv no Linux/Unix).

Este exemplo é do Windows. O Linux é o mesmo, mas você precisa alterar os caminhos etc. para corresponder à sua instalação.

proenv> proserve mySportsDb -H localhost -S 9999

proenv> proserve mySportsDb -H 192.168.1.10 -S 9999.

Conectando e desconectando

Assim que seu banco de dados estiver funcionando, você poderá se conectar a ele em seu editor de progresso:

CONNECT mySportsDb -H localhost -S 9999.

ou

CONNECT "-db mySportsDb -H localhost -S 9999".

Se você receber uma mensagem de erro, você obteve algumas informações erradas no comando ou o banco de dados não está funcionando. Você também pode ter um firewall de software ou interferência semelhante.

Você pode verificar o arquivo de log do banco de dados (mySportsDb.lg neste exemplo) para quaisquer pistas.

Desconectar é tão fácil quanto:

DISCONNECT mySportDb.

ou

DISCONNECT "mySportsDb".

Encerrando o banco de dados (ou desconectando usuários

Para encerrar o banco de dados, você pode executar o comando proshut do proenv:

proenv> proshut mySportsDb
OpenEdge Release 11.6 as of Fri Oct 16 19:01:51 EDT 2015
usr    pid    time of login           user id     Type  tty                  Limbo?
 24   7044 Wed Feb 01 20:22:57 2017   xyz         REMC  XYZ-PC               no
            1  Disconnect a User
            2  Unconditional Shutdown
            3  Emergency Shutdown (Kill All)
            x  Exit
  1. Use 1 para desconectar usuários específicos.
  2. Use 2 para encerrar o banco de dados. Observação: sem perguntas, o desligamento começa diretamente!
  3. Use 3 somente se você não puder derrubar o banco de dados de outra forma. Isso pode corromper seus dados.
  4. Use x para sair do utilitário proshut.

Você também pode desligar o banco de dados diretamente da linha de comando:

proenv>proshut mySportsDb -by

Ou desconecte um usuário da linha de comando (supondo que você saiba o número do usuário, usr na lista acima):

proenv>proshut mySportsDb -C disconnect 24
