Compile o código Progress como chamado “r-code” e normalmente é salvo em um arquivo com a extensão .r. Existem algumas maneiras diferentes de compilar: usando a instrução COMPILE ou no Linux ou AppBuilder: o compilador de aplicativos embutido. O Developer Studio (o ambiente Eclipse) tem compilação embutida em seu processo de construção.

Você deve ter 4GL Development ou OpenEdge Studio instalado para compilar programas 4GL que atualizam o banco de dados.

Sintaxe #

  • COMPILAR programa.p SAVE. //Compila program.p e salva seu r-code
  • COMPILAR VALOR(var) SALVAR. //Compila o nome salvo na variável “var” e salva seu r-code
  • COMPILAR prog.p XREF prog.xref LISTING prog.list. //Compila prog.p e cria xref e arquivos de listagem. Não salve o código-r.
  • COMPILE program.p SAVE NO-ERROR. //Compila o program.p, salva o r-code e suprime os erros para parar a execução.

Compilador de aplicativos #

Windows AppBuilder

No Construtor de Aplicativos do Windows, o Compilador de Aplicativos é encontrado no Menu Ferramentas.

Editor de procedimentos (Linux - pro ou Windows pro.exe

No Editor de Procedimentos (Linux e Windows) o Compilador se encontra no menu Ferramentas.

Compilador de aplicativos

Independentemente do sistema operacional, a funcionalidade do compilador é a mesma. Você pode adicionar diretórios e/ou arquivos e compilá-los.

Configurações principais (mais abaixo):

  • Salvar novo arquivo .r. Se não estiver marcado, os arquivos serão simplesmente compilados, mas não salvos. Útil para rastreamento de erros, por exemplo.
  • Procure em Subdiretórios. Caso contrário, subdiretórios terão que ser adicionados.
  • Remova arquivos .r antigos. Substitua o arquivo .r antigo.
  • Compilar Onlu se não houver arquivo .r. Apenas compila arquivos não compilados.

Opções:

  • Propath - mostra o propath e permite que você selecione diretórios para compilar a partir dele.
  • Adicionar - permite inserir um diretório ou arquivo.
  • Modificar - permite modificar uma entrada existente.
  • Excluir - Exclui uma entrada.
  • Iniciar Compilação - Inicia o compilador. Atalho: F2

As opções do menu principal:

  • Arquivo -> Sair: Sai do compilador
  • Compile -> Start Compile : Inicia o compilador. Atalho: F2
  • Ferramentas -> Acesso a outras ferramentas
  • Opção -> Compilador… : Configurações, veja abaixo.
  • Ajuda -> Ajuda do OpenEdge (somente Windows). Ajuda online. Atalho: F1

Definições

  • Especificação de arquivo padrão: extensões de nome de arquivo para compilar
  • Arquivo de Log de Mensagens: Arquivo para salvar mensagens, avisos e erros em
  • Salvar em: Onde armazenar o arquivo .r. Se em branco o mesmo diretório que o código.
  • Idiomas: para traduções. Não coberto aqui.
  • V6Frame: Antigo e inútil…
  • Steam-IO: Se você deseja imprimir a saída do compilador. Mais provável que não.
  • Arquivo de listagem: Se você deseja que o compilador crie um arquivo de listagem. Útil para depuração
  • Anexar: adicionar ao arquivo de listagem existente. Caso contrário, sobrescreva.
  • Largura da página + Comprimento: Formato do arquivo de listagem.
  • Arquivo Xref: Se você quiser que o compilador crie um XREF. Útil para depuração, verificação de uso de índice etc.
  • Formato XML: Se o compilador xref deve ser um xml. Caso contrário, texto “simples”.
  • Anexar: adicionar ao arquivo xref existente. Caso contrário, sobrescreva.
  • Arquivo de depuração: arquivo de listagem do depurador.
  • Chave de Criptografia: Se o arquivo fonte for criptografado usando xcode insira a chave aqui.
  • Minimize o tamanho do código R: remova algumas informações de depuração para manter o código r pequeno.
  • Gerar MD-5: Principalmente para compilação WebClient.

Uso básico

  1. Inicie o compilador
  2. Adicione um caminho (se ainda não tiver sido salvo da última sessão)
  3. Pressione F2 para compilar.
  4. Observe quaisquer erros.
  5. Sair

COMPILE declaração #

A instrução compile permite compilar programas em Progress ABL:

Uso básico:

COMPILE hello-world.p SAVE.

Com uma variável:

DEFINE VARIABLE prog AS CHARACTER   NO-UNDO.

prog = "hello.p".

COMPILE VALUE(prog) SAVE.

Existem várias opções para a instrução COMPILE:

SAVE indica que o código .r deve ser salvo para uso futuro.

COMPILE hello-world.p SAVE.

SAVE INTO dir OR SAVE INTO VALUE(dir-variable) salva o r-code no diretório especificado:

COMPILE hello-world.p SAVE INTO /usr/sources.

Arquivo de LISTAGEM. Cria um arquivo de listagem contendo informações de depuração sobre blocos, inclusões etc.

COMPILE program.p SAVE LISTING c:\temp\listing.txt.

A listagem tem algumas opções para anexar arquivos, tamanho da página e largura da página:

APPEND PAGE-SIZE num PAGE-WIDTH num

XREF xreffile salvará um arquivo xref do compilador contendo informações sobre o uso de strings e índices, etc. Você também pode APPEND este.

COMPILE checkFile.p SAVE XREF c:\directory\xref-file.txt.

XREF-XML xreffile-or-dir fará a mesma coisa que XREF, mas salvará o arquivo em um formato xml. Se você usar um diretório, o arquivo xref será nomeado programname.xref.xml.

COMPILE file.p SAVE XREF c:\temp\.

NO-ERROR suprimirá quaisquer erros de parar seu programa.

COMPILE program SAVE NO-ERROR.

DEBUG-LIST file gera um arquivo de depuração com números de linha.

COMPILE checkFile.p SAVE DEBUG-LIST c:\temp\debug.txt.

PREPROCESS file primeiro traduzirá todos os pré-processadores e então criará um novo arquivo .p com o código antes de compilar.

 COMPILE checkFile.p SAVE PREPROCESS c:\temp\PREPROC.txt.

XCODE key compilará um código fonte criptografado com key como chave. Você não pode usar XCODE com as opções XREF, XREF-XML, STRING-XREF ou LISTING juntas.

COMPILE program.p SAVE XCODE myKey.

Você pode combinar várias opções:

COMPILE prog.p SAVE INTO /usr/r-code XREF /usr/xrefs/xref.txt APPEND LISTING /usr/listings.txt APPEND NO-ERROR.

Manipulação do sistema COMPILER #

O handle do sistema COMPILER permite que você veja informações sobre uma compilação recente.

Assumindo que ok-program.p é um programa sem nenhum erro ou aviso:

COMPILE ok-program.p SAVE NO-ERROR.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

Isso produzirá:

Compilando um programa com um aviso:

/* program-with-warning.p */
DEFINE VARIABLE c AS CHARACTER   NO-UNDO.
DEFINE VARIABLE i AS INTEGER     NO-UNDO.

c = "hello".
DISPLAY c.
//This RETURN makes the program exit here and the code below unreachable.
RETURN.

IF TRUE THEN DO:
    i = 10. 
END.

Compilando o programa:

COMPILE program-with-warning.p SAVE.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

DO iError = 1 TO COMPILER:NUM-MESSAGES:
    DISPLAY 
        COMPILER:GET-FILE-NAME(iError)    LABEL "Filename" FORMAT "x(20)"
        COMPILER:GET-MESSAGE(iError)      LABEL "Message"  FORMAT "x(50)"
        COMPILER:GET-NUMBER(iError)       LABEL "Msg#"  
        COMPILER:GET-ERROR-COLUMN(iError) LABEL "Column"
        COMPILER:GET-ERROR-ROW(iError)    LABEL "Row"
            WITH FRAME fr1 SIDE-LABELS 1 COLUMNS.
END.

Resultado:

Compilando um programa com erro

DEFINE VARIABLE c AS CHARACTER   NO-UNDO.
DEFINE VARIABLE i AS INTEGER     NO-UNDO.

c = "hello".
DISPLAY c.
//Casting should be required below...
IF TRUE THEN DO:
    i = c.
END.

Compilando o programa:

//Use no-errors to supress any error messages from interrupting us.
COMPILE c:\temp\program-with-error.p SAVE NO-ERROR.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

DO iError = 1 TO COMPILER:NUM-MESSAGES:
    DISPLAY 
        COMPILER:GET-FILE-NAME(iError)    LABEL "Filename" FORMAT "x(20)"
        COMPILER:GET-MESSAGE(iError)      LABEL "Message"  FORMAT "x(50)"
        COMPILER:GET-NUMBER(iError)       LABEL "Msg#"  
        COMPILER:GET-ERROR-COLUMN(iError) LABEL "Column"
        COMPILER:GET-ERROR-ROW(iError)    LABEL "Row"
            WITH FRAME fr1 SIDE-LABELS 1 COLUMNS 20 DOWN.

    DOWN WITH FRAME fr1.
END.
