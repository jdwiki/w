Crie um blastdb de DNA #

Para comparar sequências de consulta com sequências de referência, você deve criar um blastdb de sua(s) referência(s). Isso é feito usando makeblastdb que é incluído quando você instala o blast.

makeblastdb -in <input fasta> -dbtype nucl -out <label for database>

Então, se você tivesse um arquivo reference.fasta contendo os seguintes registros:

>reference_1
ATCGATAAA
>reference_2
ATCGATCCC

Você executaria o seguinte:

makeblastdb -in reference.fasta -dbtype nucl -out my_database

Isso criaria os seguintes arquivos:

  • my_database.nhr
  • my_database.nin
  • my_database.nsq

Observe que os arquivos de banco de dados são rotulados com o argumento -out.

Extraia sequências fasta de um nucl blastdb #

Você pode extrair a sequência fasta de um blastdb construído a partir de um arquivo fasta usando blastdbcmd que deve ser instalado quando você instala o makeblastdb.

blastdbcmd -entry all -db <database label> -out <outfile>

Se você tivesse um banco de dados chamado my_database que continha os arquivos my_database.nhr, my_database.nsq, my_database.nin e você quisesse que seu arquivo de saída fasta fosse chamado reference.fasta, você executaria o seguinte :

blastdbcmd -entry all -db my_database -out reference.fasta

Instale o blast no Ubuntu #

apt-get install ncbi-blast+

Você pode verificar a versão que será instalada com antecedência aqui:

http://packages.ubuntu.com/xenial/ncbi-blast+

Extrair GI e taxiar do blastdb #

Os dados podem ser extraídos de um blastdb usando blastdbcmd que deve ser incluído em uma instalação do blast. Você pode especificar a partir das opções abaixo como parte de -outfmt quais metadados incluir e em que ordem.

Da página man:

 -outfmt <String>
   Output format, where the available format specifiers are:
       %f means sequence in FASTA format
       %s means sequence data (without defline)
       %a means accession
       %g means gi
       %o means ordinal id (OID)
       %i means sequence id
       %t means sequence title
       %l means sequence length
       %h means sequence hash value
       %T means taxid
       %X means leaf-node taxids
       %e means membership integer
       %L means common taxonomic name
       %C means common taxonomic names for leaf-node taxids
       %S means scientific name
       %N means scientific names for leaf-node taxids
       %B means BLAST name
       %K means taxonomic super kingdom
       %P means PIG

O snippet de exemplo mostra como gi e taxid podem ser extraídos do blastdb. O NCBI 16SMicrobial (ftp) blastdb foi escolhido para este exemplo:

# Example:
# blastdbcmd -db <db label> -entry all -outfmt "%g %T" -out <outfile>
blastdbcmd -db 16SMicrobial -entry all -outfmt "%g %T" -out 16SMicrobial.gi_taxid.tsv

Que irá produzir um arquivo 16SMicrobial.gi_taxid.tsv que se parece com isso:

939733319 526714
636559958 429001
645319546 629680
