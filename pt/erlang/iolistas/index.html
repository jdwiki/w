iolistas

Enquanto uma string Erlang é uma lista de inteiros, uma “iolist” é uma lista cujos elementos são inteiros, binários ou outros iolists, por exemplo ["foo", $b, $a, $r, <<"baz">>]. Esse iolist representa a string "foobarbaz".

Embora você possa converter uma iolist para um binário com iolist_to_binary/1, muitas vezes você não precisa, pois as funções da biblioteca Erlang como file:write_file/2 e gen_tcp:send/2 aceitam iolists assim como strings e binários.

Sintaxe #

  • -type iolist() :: lista_imprópria(byte() | binary() | iolist(), binary() | []).

O que é um iolista?

É qualquer binário. Ou qualquer lista contendo números inteiros entre 0 e 255. Ou qualquer lista aninhada arbitrariamente contendo qualquer uma dessas duas coisas.

Artigo original

Use listas de inteiros e binários profundamente aninhadas para representar dados de E/S para evitar a cópia ao concatenar strings ou binários.

Eles são eficientes mesmo ao combinar grandes quantidades de dados. Por exemplo, combinar dois binários de cinquenta kilobytes usando a sintaxe binária <<B1/binary, B2/binary>> normalmente exigiria a realocação de ambos em um novo binário de 100kb. Usando listas IO [B1, B2] apenas aloca a lista, neste caso três palavras. Uma lista usa uma palavra e outra palavra por elemento, veja aqui para mais informações.

Usar o operador ++ teria criado uma lista totalmente nova, em vez de apenas uma nova lista de dois elementos. Recriar listas para adicionar elementos ao final pode se tornar caro quando a lista é longa.

Nos casos em que os dados binários são pequenos, alocar listas de E/S pode ser maior do que anexar os binários. Se os dados binários puderem ser pequenos ou grandes, geralmente é melhor aceitar o custo consistente das listas de E/S.

Observe que a adição de binários é otimizada conforme descrito aqui. Em suma, um binário pode ter espaço extra e oculto alocado. Isso será preenchido se outro binário for anexado a ele que caiba no espaço livre. Isso significa que nem todo anexo binário causará uma cópia completa de ambos os binários.

A lista IO pode ser convertida em um binário #

<<"Guten tag, Hello">> = iolist_to_binary(["Guten tag, ",<<"Hello">>]).

Uma lista IO pode ser convertida em um binário usando a função iolist_to_binary/1. Se os dados forem armazenados por um longo período ou enviados como uma mensagem para outros processos, pode fazer sentido convertê-los em um binário. O custo único de converter para um binário pode ser mais barato do que copiar a lista de E/S muitas vezes, na coleta de lixo de um único processo ou na passagem de mensagens para outros.

Cuidado com listas impróprias #

["Guten tag " | <<"Hello">>].

No shell isso será impresso como ["Guten tag "|<<"Hello">>] em vez de ["Guten tag ",<<"Hello">>]. O operador pipe criará uma lista imprópria se o último elemento à direita não for uma lista. Enquanto uma lista imprópria cuja “cauda” é um binário ainda é uma iolista válida, listas impróprias podem causar problemas porque muitas funções recursivas esperam que uma lista vazia seja o último elemento, e não, como neste caso, um binário.

As listas de E/S são normalmente usadas para criar saída para uma porta, por exemplo um arquivo ou soquete de rede. #

file:write_file("myfile.txt", ["Hi " [<<"there">>], $\n]).

Adicione os tipos de dados permitidos à frente de uma lista de E/S, criando uma nova. #

["Guten Tag " | [<<"Hello">>]].
[<<"Guten Tag ">> | [<<"Hello">>]].
[$G, $u, $t, $e, $n , $T, $a, $g | [<<"Hello">>]].
[71,117,116,101,110,84,97,103,<<"Hello">>].

Os dados de E/S podem ser adicionados com eficiência ao final de uma lista. #

Data_1 = [<<"Hello">>].
Data_2 = [Data_1,<<" Guten Tag ">>].

Obtenha o tamanho da lista de E/S #

Data = ["Guten tag ",<<"Hello">>],
Len = iolist_size(Data),
[<<Len:32>> | Data].

O tamanho de um iolist pode ser calculado usando o iolist_size/1. Esse trecho calcula o tamanho de uma mensagem e a cria e anexa à frente como um binário de quatro bytes. Esta é uma operação típica em protocolos de mensagens.
