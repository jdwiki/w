Olá Mundo #

Há duas coisas que você precisa saber ao escrever um aplicativo “hello world” em Erlang:

  1. O código-fonte está escrito na linguagem de programação erlang usando o editor de texto de sua escolha
  2. A aplicação é então executada na máquina virtual erlang. Neste exemplo vamos interagir com a VM erlang através do shell erlang.

Primeiro o código fonte do aplicativo: #

Crie um novo arquivo hello.erl contendo o seguinte:

-module(hello).
-export([hello_world/0]).

hello_world() ->
  io:format("Hello, World!~n", []).

Vamos dar uma olhada rápida no que isso significa:

  • -module(hello). Todas as funções erlang existem dentro de um módulo. Os módulos são então usados ​​para construir aplicativos, que são uma coleção de módulos. Esta primeira linha é para identificar este módulo, ou seja, hello. Os módulos podem ser comparados aos pacotes do Java
  • -export([hello_world/0]). Diz ao compilador quais funções tornar “públicas” (quando comparadas com linguagens OO), e a aridade da função relevante. A aridade é o número de argumentos que a função recebe. Já que em erlang uma função com 1 argumento é vista como uma função diferente de uma com 2 argumentos mesmo que o nome possa ser exatamente o mesmo. Ou seja, hello_world/0 é uma função completamente diferente de hello_world/1 por exemplo.
  • hello_world() Este é o nome da função. O -> indica a transição para a implementação (corpo) da função. Isso pode ser lido como “hello_world() é definido como …”. Observe que hello_world() (sem argumentos) é identificado por hello_world/0 na VM e hello_world(Some_Arg) como hello_world/1.
  • io:format("Hello, World!~n", []) Do módulo io, a função format/2 é chamada, que é a função para saída padrão. ~n é um especificador de formato que significa imprimir uma nova linha. O [] é uma lista de variáveis ​​a serem impressas indicadas por especificadores de formato na string de saída, que neste caso não é nada.
  • Todas as instruções erlang devem terminar com um . (ponto).

Em Erlang, o resultado da última instrução em uma função é retornado.

Agora, vamos executar nossa aplicação: #

Inicie o shell erlang no mesmo diretório que o arquivo hello.erl:

$ erl

Você deve obter um prompt parecido com isto (sua versão pode ser diferente):

Eshell V8.0  (abort with ^G)
1>

Agora digite os seguintes comandos:

1> c(hello).
{ok,hello}
2> hello:hello_world().
Hello, World!
ok

Vamos percorrer cada linha uma a uma:

  • c(hello) - este comando chama a função c em um átomo hello. Isso efetivamente diz ao Erlang para encontrar o arquivo hello.erl, compilá-lo em um módulo (um arquivo chamado hello.beam será gerado no diretório) e carregá-lo no ambiente.
  • {ok, hello} - este é o resultado da chamada da função c acima. É uma tupla contendo um átomo ok e um átomo hello. As funções Erlang geralmente retornam {ok, Something} ou {error, Reason}.
  • hello:hello_world() - chama a função hello_world() do módulo hello.
  • Hello, World! - isto é o que nossa função imprime.
  • ok - isso é o que nossa função retornou. Como Erlang é uma linguagem de programação funcional, cada função retorna alguma coisa. No nosso caso, embora não tenhamos retornado nada em hello_world(), a última chamada nessa função foi para io:format(...) e essa função retornou ok, que por sua vez é o que nossa função retornou.

Compreensão da lista #

As compreensões de lista são uma construção sintática para criar uma lista com base em listas existentes. Em erlang uma compreensão de lista tem a forma [Expr || Qualificador1, ..., QualificadorN]. Onde os qualificadores são geradores Pattern <- ListExpr ou filtros como integer(X) avaliando como true ou false.

O exemplo a seguir mostra uma compreensão de lista com um gerador e dois filtros.

[X || X <- [1,2,a,3,4,b,5,6], integer(X), X > 3].

O resultado é uma lista contendo apenas números inteiros maiores que 3.

[4,5,6]

Módulos #

Um módulo erlang é um arquivo com algumas funções agrupadas. Este arquivo geralmente tem extensão .erl.

Um módulo “Hello World” com o nome hello.erl é mostrado abaixo

-module(hello).
-export([hello_world/0]).

hello_world() ->
  io:format("Hello, World!~n", []).

No arquivo, é necessário declarar o nome do módulo. Como mostrado antes na linha 1. O nome do módulo e o nome do arquivo antes da extensão .erl devem ser os mesmos.

Função #

Função é um conjunto de instruções, que são agrupadas. Essas instruções agrupadas realizam determinada tarefa. Em erlang, todas as funções retornarão um valor quando forem chamadas.

Abaixo está um exemplo de uma função que soma dois números

add(X, Y)-> X + Y.

Essa função executa uma operação de adição com valores X e Y e retorna o resultado. A função pode ser usada como abaixo

add(2,5).

As declarações de função podem consistir em várias cláusulas, separadas por um ponto e vírgula. Os Argumentos em cada uma dessas cláusulas são avaliados por correspondência de padrões. A seguinte função retornará ’tupla’ se o Argumento for uma tupla no Formulário: {teste, X} onde X pode ser qualquer valor. Ele retornará ’lista’, se o Argumento for uma lista de comprimento 2 no formato [“teste”, X], e retornará ‘{erro, “Motivo”}’ em qualquer outro caso:

function({test, X}) -> tuple;
function(["test", X]) -> list;
function(_) -> {error, "Reason"}.

Se o argumento não for uma tupla, a segunda cláusula será avaliada. Se o argumento não for uma lista, a terceira cláusula será avaliada.

As declarações de função podem consistir nos chamados ‘Guards’ ou ‘Guard Sequences’. Esses Guards são expressões que limitam a avaliação de uma função. Uma função com Guards só é executada quando todas as Guard Expressions produzem um valor verdadeiro. Múltiplos Guardas podem ser separados por um ponto e vírgula.

function_name(Argument) when Guard1; Guard2; ... GuardN -> (...).

A função ‘function_name’ só será avaliada, quando a Sequência de Guarda for verdadeira. A função a seguir retornará true somente se o argumento X estiver no intervalo apropriado (0..15):

in_range(X) when X>=0; X<16 -> true;
in_range(_) -> false.

Iniciando e Parando o Shell Erlang #

Iniciando o shell Erlang

Em um sistema UNIX, você inicia o shell Erlang a partir de um prompt de comando com o comando erl

Exemplo:

$ erl
Erlang/OTP 18 [erts-7.0] [source] [64-bit] [smp:8:8] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.0  (abort with ^G)
1>

O texto que mostra quando você inicia o shell informa informações sobre qual versão do Erlang você está executando, bem como outras informações úteis sobre o sistema erlang.

Para iniciar o shell no Windows, clique no ícone Erlang no menu Iniciar do Windows.

Parando o shell Erlang

Para uma saída controlada do shell erlang, digite:

Erlang/OTP 18 [erts-7.0] [source] [64-bit] [smp:8:8] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.0  (abort with ^G)
1> q().

Você também pode sair do shell Erlang pressionando Ctrl+C em sistemas UNIX ou Ctrl+Break no Windows, o que o leva ao seguinte prompt:

Erlang/OTP 18 [erts-7.0] [source] [64-bit] [smp:8:8] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.0  (abort with ^G)
1> 
BREAK: (a)bort (c)ontinue (p)roc info (i)nfo (l)oaded
       (v)ersion (k)ill (D)b-tables (d)istribution

Se você pressionar a (para abortar), sairá do shell diretamente.

Outras formas de sair do shell erlang são: init:stop() que faz a mesma coisa que q() ou erlang:halt().

Correspondência de padrões #

Uma das operações mais comuns em erlang é a correspondência de padrões. Ele é usado ao atribuir um valor a uma variável, em declarações de função e em estruturas de fluxo de controle como instruções case e ​​receive. Uma operação de correspondência de padrões precisa de pelo menos 2 partes: um padrão e um termo com o qual o padrão é correspondido.

Uma atribuição de variável em erlang se parece com isso:

X = 2.

Na maioria das linguagens de programação a semântica desta operação é direta: Ligue um valor (2) a um nome de sua escolha (a variável – neste caso X). Erlang tem uma abordagem ligeiramente diferente: combine o padrão do lado esquerdo (X) com o termo do lado direito (2). Neste caso, o efeito é o mesmo: a variável X agora está vinculada ao valor 2. No entanto, com a correspondência de padrões, você pode realizar atribuições mais estruturadas.

{Type, Meta, Doc} = {document, {author, "Alice"}, {text, "Lorem Ipsum"}}.

Essa operação de correspondência é realizada analisando a estrutura do termo do lado direito e aplicando todas as variáveis ​​do lado esquerdo aos valores apropriados do termo, de modo que o lado esquerdo seja igual ao lado direito. Neste exemplo, Type está vinculado ao termo: document, Meta a {author, "Alice"} e Doc a {text, "Lorem Ipsum"}. Neste exemplo em particular, as variáveis: Type, Meta e Doc são assumidas como unbound, para que cada variável possa ser usada.

As correspondências de padrões também podem ser construídas, usando variáveis ​​vinculadas.

Identifier = error.

A variável Identifier agora está vinculada ao valor error. A operação de correspondência de padrões a seguir funciona porque a estrutura corresponde, e a variável vinculada Identifier tem o mesmo valor como a parte apropriada do lado direito do termo.

{Identifier, Reason} = {error, "Database connection timed out."}.

Uma operação de correspondência de padrões falha quando há uma incompatibilidade entre o termo do lado direito e o padrão do lado esquerdo. A correspondência a seguir falhará, porque Identifier está vinculado ao valor error, que não possui expressão apropriada no termo do lado direito.

{Identifier, Reason} = {fail, "Database connection timed out."}.
> ** exception error: no match of right hand side value {fail,"Database ..."}
