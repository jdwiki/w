Construa e instale Erlang/OTP no FreeBSD #

Os exemplos a seguir mostram 3 métodos principais para instalar Erlang/OTP no FreeBSD.

Método 1 - Pacote Binário Pré-construído #

Use pkg para instalar o pacote binário pré-construído:

$ pkg install erlang

Teste sua nova instalação:

$ erl
Erlang/OTP 18 [erts-7.3.1] [source] [64-bit] [smp:2:2] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.3.1  (abort with ^G)

Método 2 - Compile e instale usando a coleção de portas (recomendado) #

Compile e instale a porta como de costume:

$ make -C /usr/ports/lang/erlang install clean

Teste sua nova instalação:

$ erl
Erlang/OTP 18 [erts-7.3.1] [source] [64-bit] [smp:2:2] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.3.1  (abort with ^G)

Isto irá buscar o tarball de lançamento do site oficial, aplicar alguns patches se necessário, construir o lançamento e instalá-lo. Obviamente, levará algum tempo.

Método 3 - Compile e instale a partir do tarball de lançamento #

Nota: construir a versão manualmente funciona, mas usar os dois métodos acima deve ser preferido, já que a coleção de portas incorpora patches que tornam a versão mais amigável ao FreeBSD.

Baixe o arquivo de lançamento:

$ fetch 'http://erlang.org/download/otp_src_18.3.tar.gz'

Verifique se sua soma MD5 está correta:

$ fetch 'http://erlang.org/download/MD5'
MD5                                           100% of   24 kB  266 kBps 00m00s

$ grep otp_src_18.3.tar.gz MD5
MD5(otp_src_18.3.tar.gz)= 7e4ff32f97c36fb3dab736f8d481830b

$ md5 otp_src_18.3.tar.gz
MD5 (otp_src_18.3.tar.gz) = 7e4ff32f97c36fb3dab736f8d481830b

Extraia o tarball:

$ tar xzf otp_src_18.3.tar.gz

Configurar:

$ ./configure --disable-hipe

Se você quiser construir Erlang com HiPe, você precisará aplicar os patches da coleção de portas.

Construir:

$ gmake

Instalar:

$ gmake install

Teste sua nova instalação:

$ erl
Erlang/OTP 18 [erts-7.3] [source] [64-bit] [smp:2:2] [async-threads:10] [kernel-poll:false]

Eshell V7.3  (abort with ^G)

Compile e instale usando kerl #

kerl é uma ferramenta que ajuda a construir e instalar versões Erlang/OTP.

Instale o curl:

$ make -C /usr/ports/ftp/curl install clean

Baixar cara:

$ fetch 'https://raw.githubusercontent.com/kerl/kerl/master/kerl'
$ chmod +x kerl

Atualize a lista de lançamentos disponíveis:

$ ./kerl update releases
The available releases are:
R10B-0 R10B-10 R10B-1a R10B-2 R10B-3 R10B-4 R10B-5 R10B-6 R10B-7 R10B-8 R10B-9 R11B-0 R11B-1 R11B-2 R11B-3 R11B-4 R11B-5 R12B-0 R12B-1 R12B-2 R12B-3 R12B-4 R12B-5 R13A R13B01 R13B02-1 R13B02 R13B03 R13B04 R13B R14A R14B01 R14B02 R14B03 R14B04 R14B R14B_erts-5.8.1.1 R15B01 R15B02 R15B02_with_MSVCR100_installer_fix R15B03-1 R15B03 R15B R16A_RELEASE_CANDIDATE R16B01 R16B02 R16B03-1 R16B03 R16B 17.0-rc1 17.0-rc2 17.0 17.1 17.3 17.4 17.5 18.0 18.1 18.2 18.2.1 18.3 19.0

Compile a versão necessária:

$ ./kerl build 18.3 erlang-18.3

Verifique se o build está presente na lista de builds:

$ ./kerl list builds
18.3,erlang-18.3

Instale a compilação em algum lugar:

$ ./kerl install erlang-18.3 ./erlang-18.3

Fonte do arquivo activate se você estiver executando o bash ou o shell de peixe. Se você estiver executando um cshell, adicione o diretório bin de compilação ao PATH:

$ setenv PATH "/some/where/erlang-18.3/bin/:$PATH"

Teste sua nova instalação:

$ which erl
/some/where/erlang-18.3/bin//erl

$ erl
Erlang/OTP 18 [erts-7.3] [source] [64-bit] [smp:2:2] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.3  (abort with ^G)

Outros lançamentos #

Se você quiser construir outra versão do Erlang/OTP, procure as outras portas na coleção:

  • lang/erlang-runtime15
  • lang/erlang-runtime16
  • lang/erlang-runtime17
  • lang/erlang-runtime18

Compile e instale Erlang/OTP no Ubuntu #

Os exemplos a seguir mostram dois métodos principais para instalar Erlang/OTP no Ubuntu.

Método 1 - Pacote Binário Pré-construído #

Basta executar este comando e ele fará o download e instalará a versão estável mais recente do Erlang de Erlang Solutions.

$ sudo apt-get install erlang

Método 2 - Construir e instalar a partir da fonte #

Baixe o arquivo tar:

$ wget http://erlang.org/download/otp_src_19.0.tar.gz

Extraia o arquivo tar:

$ tar -zxf otp_src_19.0.tar.gz

Entre no diretório extraído e defina ERL_TOP para ser o caminho atual:

$ cd otp_src_19.0
$ export ERL_TOP=`pwd`

Agora, antes de configurar o build, você quer ter certeza de que tem todas as dependências necessárias para instalar o Erlang:

Dependências básicas:

$ sudo apt-get install autoconf libncurses-dev build-essential

Outras dependências de aplicativos

Aplicação Instalação de dependência
HiPE $ sudo apt-get install m4
ODBC $ sudo apt-get install unixodbc-dev
OpenSSL $ sudo apt-get install libssl-dev
wxWidgets $ sudo apt-get install libwxgtk3.0-dev libglu-dev
Documentação $ sudo apt-get install fop xsltproc
Orber e outros projetos C++ $ sudo apt-get install g++
jinterface $ sudo apt-get install default-jdk

Configurar e construir: Você pode definir suas próprias opções ou deixá-lo em branco para executar a configuração padrão. Configuração avançada e compilação para Erlang/OTP.

$ ./configure [ options ]
$ make

Testando a compilação:

$ make release_tests
$ cd release/tests/test_server
$ $ERL_TOP/bin/erl -s ts install -s ts smoke_test batch -s init stop

Após executar esses comandos, abra $ERL_TOP/release/tests/test_server/index.html com seu navegador web e verifique se não há falhas. Se todos os testes forem aprovados, podemos continuar com a instalação.

Instalando:

$ make install

Construa e instale Erlang/OTP no OpenBSD #

Erlang no OpenBSD está atualmente quebrado nas arquiteturas alpha, sparc e hppa.

Método 1 - Pacote Binário Pré-construído #

O OpenBSD permite que você escolha a versão desejada que deseja instalar em seu sistema:

######################################################################
# free-choice:
######################################################################
$ pkg_add erlang
# a       0: <None>
#   1: erlang-16b.03p10v0
#   2: erlang-17.5p6v0
#   3: erlang-18.1p1v0
#   4: erlang-19.0v0

######################################################################
# manual-choice:
######################################################################
pkg_add erlang%${version}
# example: 
pkg_add erlang%19

O OpenBSD pode suportar várias versões do Erlang. Para facilitar o uso, cada binário é instalado na versão Erlang em seu nome. Então, se você instalou o erlang-19.0v0, seu binário erl será erl19.

Se você quiser usar erl, você pode criar um link simbólico:

ln -s /usr/local/bin/erl19 /usr/local/bin/erl

ou crie um alias em seu arquivo de configuração do shell ou no arquivo .profile:

echo 'alias erl="erl19"' >> ~/.profile
# or
echo 'alias erl="erl19"' >> ~/.shrc

Agora você pode executar erl:

erl19
# or if you have an alias or symlink
erl
# Erlang/OTP 19 [erts-8.0] [source] [async-threads:10] [kernel-poll:false]
# 
# Eshell V8.0  (abort with ^G)

Método 2 - Construir e instalar usando portas #

 RELEASE=OPENBSD_$(uname -r  | sed 's/\./_/g')
 cd /usr
 cvs -qz3 -danoncvs@anoncvs.openbsd.org:/cvs co -r${RELEASE}
 cd /usr/ports/lang/erlang
 ls -p 
 # 16/ 17/ 18/ 19/  CVS/  Makefile  Makefile.inc  erlang.port.mk
 cd 19
 make && make install

Método 3 - Construir a partir da fonte #

Construir a partir da fonte requer pacotes adicionais:

  • git
  • gmake
  • autoconf-2.59
pkg_add git gmake autoconf%2.59
git clone https://github.com/erlang/otp.git
cd otp
AUTOCONF_VERSION="2.59" ./build_build all
