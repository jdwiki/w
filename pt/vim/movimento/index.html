Movimento Básico #

Observações #

  • Todo movimento pode ser utilizado após um comando do operador, então o comando opera sobre o texto compreendido pelo alcance do movimento.
  • Assim como os comandos do operador, os movimentos podem incluir uma contagem, então você pode mover por 2wordas, por exemplo.

Setas; flechas #

No Vim, as teclas normais de seta/cursor (←↓↑→) funcionam como esperado. No entanto, para digitadores de toque, é mais fácil usar as teclas alternativas hjkl. Em um teclado típico, eles estão localizados próximos um do outro na mesma linha e facilmente acessíveis com a mão direita. A técnica mnemônica para lembrar qual é qual entre eles é assim:

  • h/l — essas estão localizadas “mais à esquerda/direita” entre as quatro letras do teclado, então são equivalentes a “ir à esquerda/direita” respectivamente;
  • j — o “j” minúsculo tem sua cauda indo “para baixo” abaixo das letras típicas, como uma pequena seta - então é equivalente a “descer”;
  • k — por outro lado, “k” minúsculo tem seu “ascendente” indo “para cima” acima das letras típicas, como um pequeno ponteiro - então é equivalente a “subindo”.

Movimentos básicos #

Todos os comandos abaixo devem ser executados em modo normal.

Comando Descrição
h ou esquerda vá [count] caracteres para a esquerda
j ou para baixo vá [count] caracteres abaixo
k ou acima vá [count] caracteres acima
l ou direita vá [count] caracteres para a direita
gg vá para a primeira linha, ou [contagem]‘ª linha, se fornecida
H vá para a primeira linha na tela visível
M vá para a linha do meio na tela visível
L vá para a última linha na tela visível
G vá para a última linha, ou [contagem]‘ª linha, se fornecida
Início ou 0 ir para o primeiro caractere da linha
^ ir para o primeiro caractere não em branco da linha
+ descer uma linha para o primeiro caractere não em branco
- subir uma linha até o primeiro caractere não em branco
$ ou Fim vá para o final da linha (se [count] for fornecido, vá [count - 1] linhas para baixo)
| vá para o [count]‘º caractere ou vá para o início da linha se count não for especificado
f{char} vá para [count]‘ª ocorrência de {char} à direita inclusive
F{char} vá para [count]‘ª ocorrência de {char} à esquerda inclusive
t{char} vá para [count]‘ª ocorrência de {char} à direita exclusivo
T{char} vá para [count]‘ª ocorrência de {char} à esquerda exclusivo
; repetir as últimas f, t, F ou T [count] vezes
, repita as últimas f, t, F ou T, na direção oposta, [count]< /kbd> vezes
w ir para o início da próxima palavra
b ir para o início da palavra anterior
e ir para o final da próxima palavra
ge ir para o final da palavra anterior
% vá para pares correspondentes, por exemplo, (), [], {}, /* */ ou #if, #ifdef, #else, #elif, #endif
{ } parágrafo anterior/seguinte
[{]} início/fim do bloco
’{char} Ir para marcar (marcar com m{char})
<C-B><C-F> página anterior/seguinte
<C-O><C-I> Voltar ou avançar na “lista de saltos” (requer o recurso jumplist, veja :help jumps)

Nota: b, e e w consideram uma palavra como letras, números e sublinhados por padrão (isso pode ser configurado com o configuração iskeyword). Cada um deles também pode ser capitalizado, fazendo com que eles pulem qualquer coisa que não seja espaço em branco também.

Nota: O Vim reconhece dois tipos de movimento: movimento do operador (:help movement) e saltos (:help jumplist). Movimentos como aqueles executados com g (gg, G, g,) contam como saltos, assim como mudanças. As alterações obtêm sua própria lista de atalhos, que é navegável conforme mencionado acima via g, e g; (veja :help changelist). Os saltos não são tratados como comandos de movimento pelo Vim

Ao mover-se para cima ou para baixo nas linhas, o cursor retém sua coluna como seria de esperar. Se a nova linha for muito curta, o cursor se moverá para o final da nova linha. Se a coluna estiver além do final da linha, o cursor será exibido no final da linha. O número da coluna inicial ainda é retido até que uma ação seja executada para alterá-lo (como editar texto ou mover explicitamente a coluna).

Se o comprimento de uma linha exceder a largura da tela, o texto será quebrado (nas configurações padrão, esse comportamento pode ser configurado). Para percorrer as linhas exibidas na tela, em vez de linhas dentro do arquivo, adicione g na frente do comando usual. Por exemplo, gj moverá o cursor para a posição exibida uma linha abaixo de sua posição atual, mesmo que esta esteja na mesma linha do arquivo.

Procurando #

Saltar para os personagens #

f{char} - move para a próxima ocorrência de {char} à direita do cursor na mesma linha

F{char} - move para a próxima ocorrência de {char} à esquerda do cursor na mesma linha

t{char} - move para a esquerda da próxima ocorrência de {char} para a direita do cursor na mesma linha

T{char} - move para a direita da próxima ocorrência de {char} para a esquerda do cursor na mesma linha

Saltar para frente/para trás entre os ‘resultados’ via ; e ,.

Além disso, você pode pesquisar palavras inteiras via /<searchterm>Enter.

Procurando por cordas #

* - move para a próxima ocorrência da palavra sob o cursor

# - move para a ocorrência anterior da palavra sob o cursor

/searchtermEnter leva você para a próxima correspondência (busca avançada). Se você usar ? em vez de /, a pesquisa retrocede.

Salte entre as partidas via n (próximo) e N (anterior).

Para visualizar/editar suas pesquisas anteriores, digite / e pressione a tecla de seta para cima.

Úteis também são estas configurações: (observe que :se é igual a :set)

  • :se hls HighLightSearch, destaca todas as correspondências de pesquisa; use :noh para desativar temporariamente o realce de pesquisa/marca (:set noh ou :set nohls desativa.)
  • :se is ou :set incs ativa a Pesquisa Incremental, o cursor pula para a próxima correspondência automaticamente. (:se nois desliga.)
  • :se ic IgnoreCase, desativa a diferenciação entre maiúsculas e minúsculas. (:se noic liga novamente.)
  • :se scs SmartCaSe, pode ser usado quando IgnoreCase estiver definido; torna a (in)sensibilidade a maiúsculas inteligente! por exemplo. /the procurará por the, The, ThE, etc. enquanto /The apenas procurará por The.

Navegando para o início de uma palavra específica #

Ao editar texto, uma tarefa comum é navegar até uma palavra específica na tela. Nestes exemplos, exploramos como podemos navegar para a palavra atualizado. Por uma questão de consistência entre os exemplos, pretendemos pousar na primeira letra da palavra.

Salto no meio da tela M$B

Essa abordagem é rápida, usando apenas 3 pressionamentos de tecla. A desvantagem, no entanto, é que não é muito geral, pois não é comum que nossa linha de destino fique bem no meio da tela. Ainda assim, é um movimento útil ao fazer movimentos menos granulares.

Usando uma contagem 3jfu;;

À primeira vista, isso pode parecer um passo atrás da primeira abordagem devido ao número de pressionamentos de tecla. Mas como usamos uma contagem aqui em vez de M, é mais flexível. Podemos identificar rapidamente a contagem correta a ser usada se [relativenumber][3] estiver ativado. Para mover para a palavra alvo, usar f em combinação com ; pode ser surpreendentemente eficaz - e certamente melhor do que pressionar repetidamente w. Se você ultrapassar seu alvo com ;, você pode retroceder com ,.

Pesquisa explícita /para cimaEnternn

[![digite a descrição da imagem aqui][4]][4]

Navegar através de / pode ser muito poderoso. Muitas vezes, podemos pular diretamente para a nossa palavra-alvo digitando-a. Aqui digitamos apenas os dois primeiros caracteres na esperança de que correspondam exclusivamente à nossa palavra. Infelizmente, existem várias correspondências, mas podemos pular rapidamente para a próxima com n.

Pesquisa implícita /yEspaçoEnterw

[![digite a descrição da imagem aqui][5]][5]

Em alguns casos, pode ser mais eficiente pular perto do nosso alvo em vez de tentar ir diretamente para ele. Aqui observamos que há uma letra que ocorre com pouca frequência, y, bem ao lado do alvo. Podemos adicionar um Espaço ao nosso termo de pesquisa para diminuir as chances de encontrarmos algum outro caractere y ao longo do caminho. Isso também pode ser usado com grande efeito com f{char}, como no exemplo Usando uma contagem.

Procurando por padrão #

O Vim suporta o uso de expressões regulares ao pesquisar em um arquivo.

O caractere para indicar que você deseja realizar uma pesquisa é /.

A pesquisa mais simples que você pode realizar é a seguinte

/if

Isto irá pesquisar o arquivo inteiro para todas as instâncias de if. No entanto, nossa busca if é na verdade uma expressão regular que irá corresponder a qualquer ocorrência da palavra if, incluindo aquelas dentro de outras palavras.

Por exemplo, nossa pesquisa diria que todas as palavras a seguir correspondem à nossa pesquisa: if, spiffy, endif, etc.

Podemos fazer pesquisas mais complicadas usando expressões regulares mais complicadas.

Se nossa busca fosse:

/\<if\>

então nossa busca retornará apenas correspondências exatas para a palavra completa if. O spiffy e o endif acima não seriam retornados pela pesquisa, apenas if.

Também podemos usar intervalos. Dado um arquivo:

hello1
hello2
hello3
hello4

Se quisermos procurar aquelas linhas contendo “hello” seguido por um dígito entre 1 e 3, diríamos:

/hello[1-3]

Outro exemplo:

/(?:\d*\.)?\d+

encontraria todos os números inteiros e decimais no arquivo.

Usando marcas para se movimentar #

Marcas são como marcadores; eles ajudam você a encontrar lugares que você já esteve.

#TLDR Defina-os no modo normal com m{a-zA-Z} e salte para eles no modo normal ou visual com '{a-zA-Z} (aspas simples) ou `{a- zA-Z} (backtick). Letras minúsculas são para marcas dentro de um buffer, e letras maiúsculas e dígitos são globais. Veja suas marcas atualmente definidas com :marks, e para mais informações veja :help mark.

Coloca uma marca #

A ajuda interna do Vim diz:

m{a-zA-Z}               Set mark {a-zA-Z} at cursor position (does not move
                        the cursor, this is not a motion command).

A marca manterá o controle de qual linha e coluna foi colocada. Não há confirmação visual de que uma marca foi definida ou se uma marca tinha um valor anterior e foi substituída.

Saltar para uma marca #

A ajuda interna do Vim diz:

Jumping to a mark can be done in two ways:  
1. With ` (backtick):     The cursor is positioned at the specified location
                          and the motion is exclusive.
2. With ' (single quote): The cursor is positioned on the first non-blank
                          character in the line of the specified location and
                          the motion is linewise.

Backtick usa a posição da coluna, enquanto aspas simples não. A diferença entre simplesmente permite que você ignore a posição da coluna da sua marca, se desejar.

Você pode alternar entre marcas não globais no modo visual, além do modo normal, para permitir a seleção de texto com base nas marcas.

Marcas Globais #

Marcas globais (letras maiúsculas) permitem saltar entre arquivos. O que isso significa é se, por exemplo, a marca A for definida em foo.txt, então de bar.txt (em qualquer lugar no meu sistema de arquivos), se eu pular para marcar A, meu buffer atual será substituído por foo.txt. O Vim solicitará para salvar as alterações.

Saltar para uma marca em outro arquivo não é considerado um movimento, e seleções visuais (entre outras coisas) não funcionarão como pular para marcas dentro de um buffer.

Para voltar ao arquivo anterior (bar.txt neste caso), use :b[uffer] # (ou seja, :b# ou :buffer#).

Observação:

Marcas especiais #

Existem certas marcas que o Vim define automaticamente (que você pode sobrescrever sozinho, mas provavelmente não precisará).

Por exemplo (parafraseado da ajuda do Vim):

`[` and `]`: jump to the first or last character of the previously changed or 
             yanked text.  {not in Vi}

`<` and `>`: jump to the first or last line (with `'`) or character (with 
             <code>`</code>) of the last selected Visual area in the current 
             buffer.  For block mode it may also be the last character in the 
             first line (to be able to define the block).  {not in Vi}.

Mais, da ajuda integrada do Vim:

''  ``              To the position before the latest jump, or where the
                    last "m'" or "m`" command was given.  Not set when the
                    :keepjumps command modifier was used.
                    Also see restore-position.


'"  `"              To the cursor position when last exiting the current
                    buffer.  Defaults to the first character of the first
                    line.  See last-position-jump for how to use this
                    for each opened file.
                    Only one position is remembered per buffer, not one
                    for each window.  As long as the buffer is visible in
                    a window the position won't be changed.
                    {not in Vi}.

'.  `.              To the position where the last change was made.  The
                    position is at or near where the change started.
                    Sometimes a command is executed as several changes,
                    then the position can be near the end of what the
                    command changed.  For example when inserting a word,
                    the position will be on the last character.
                    {not in Vi}

'"  `"              To the cursor position when last exiting the current
                    buffer.  Defaults to the first character of the first
                    line.  See last-position-jump for how to use this
                    for each opened file.
                    Only one position is remembered per buffer, not one
                    for each window.  As long as the buffer is visible in
                    a window the position won't be changed.
                    {not in Vi}.

'^  `^              To the position where the cursor was the last time
                    when Insert mode was stopped.  This is used by the
                    gi command.  Not set when the :keepjumps command
                    modifier was used.  {not in Vi}

Além disso, os caracteres (,),{ e } são marcas que saltam para a mesma posição que seus comandos de modo normal - ou seja, '} faz a mesma coisa no modo normal como }.

Saltar para uma linha específica #

Para saltar para uma linha específica com dois pontos. Para pular para a primeira linha de um arquivo, use

:1

Para pular para a linha 23

:23
