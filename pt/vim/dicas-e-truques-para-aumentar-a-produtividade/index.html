Sintaxe #

  • :definir número relativo
  • :definir número
  • :set nonumber / :set nove
  • :pwd

Este recarregamento automático só acontecerá se você editar seu vimrc em uma versão completa do vim que suporte autocmd.

Ativar números de linha relativos #

Para excluir algumas linhas de texto quando você não sabe o número exato de linhas a serem excluídas, tente 10dd , 5dd , 3dd até remover todas as linhas.

Números de linha relativos resolve este problema, suponha que temos um arquivo contendo:

sometimes, you see a block of 
text. You want to remove 
it but you 
cannot directly get the 
exact number of 
lines to delete 
so you try 
10d , 5d 
3d until 
you
remove all the block.

Entre no modo NORMAL: Esc

Agora, execute :set relativenumber. Feito isso, o arquivo será exibido como:

<pré> 3 às vezes, você vê um bloco de 2 texto. Você deseja remover 1 mas você 0 não pode obter diretamente o 1 número exato de 2 linhas para excluir 3 então você tenta 4 10d , 5d 5 3d até 6 você 7 remova todo o bloco.

onde 0 é o número da linha atual e também mostra o número real da linha na frente do número relativo, então agora você não precisa estimar o número de linhas da linha atual para cortar ou excluir ou pior contagem eles um por um.

Agora você pode executar seu comando usual como 6dd e você tem certeza sobre o número de linhas.

Você também pode usar a forma abreviada do mesmo comando :set rnu para ativar os números relativos e :set nornu para desativar o mesmo.

Se você também :set number ou já tiver :set number ativado, você obterá o número da linha na qual o cursor está.

<pré> 3 às vezes, você vê um bloco de 2 texto. Você deseja remover 1 mas você 4 não pode obter diretamente o 1 número exato de 2 linhas para excluir 3 então você tenta 4 10d , 5d 5 3d até 6 você 7 remova todo o bloco.

Mapeamentos para sair do modo Inserir #

Muitos usuários do Vim acham o Esc muito difícil de alcançar e acabam encontrando outro mapeamento que é fácil de alcançar a partir da linha inicial. Observe que Ctrl-[ pode ser equivalente a Esc em um teclado em inglês e é muito mais fácil de alcançar.

jk #

inoremap jk <ESC>

Este é realmente fácil de acionar; apenas esmague seus dois primeiros dedos na linha da casa ao mesmo tempo. Também é difícil acionar acidentalmente, pois “jk” nunca aparece em nenhuma palavra em inglês e, se você estiver no modo normal, não fará nada. Se você não digitar muito “blackjack”, considere também adicionar inoremap kj <ESC> para que você não precise se preocupar com o tempo das duas teclas.

Caps Lock #

##Linux No Linux, você pode usar o xmodmap para fazer o Caps Lock fazer a mesma coisa que o Esc. Coloque isso em um arquivo:

!! No clear Lock 
clear lock
!! make caps lock an escape key
keycode 0x42 = Escape

Em seguida, execute xmodmap file. Isso remapeia Caps Lock para Esc.

Janelas #

No Windows, você pode usar SharpKey ou AutoHotkey.

Mac OS #

Se você tiver o macOS 10.12.1 ou posterior, poderá remapear o Caps Lock para Escape usando as Preferências do Sistema. Selecione Teclado, vá para a guia Teclado e clique em Teclas modificadoras.

Usando o recurso de conclusão de caminho dentro do Vim #

Isso é muito comum, você memoriza um caminho para um arquivo ou pasta, abre o Vim e tenta escrever o que acabou de memorizar, mas não tem 100% de certeza de que está correto, então fecha o editor e começa de novo.

Quando você deseja o recurso de conclusão de caminho e tem um arquivo /home/ubuntu/my_folder/my_file e está editando outro arquivo referenciando o caminho do anterior:

Entre no modo de inserção: insira ou faça do jeito que você quiser. Em seguida, escreva /h. Quando o cursor está sob h, pressione Ctrlx e então Ctrlf para que o editor complete para /home/ porque o padrão /h é único

Agora, suponha que você tenha duas pastas dentro de /home/ubuntu/ chamadas my_folder_1 my_folder_2

e você quer o caminho /home/ubuntu/my_folder_2

como sempre:

Entre no modo de inserção

escreva /h e pressione Ctrlx e então Ctrlf . Agora você tem /home/ Em seguida, adicione u após /home/ e pressione Ctrlx e então Ctrlf . Agora, você tem /home/ubuntu/ porque esse caminho é único. Agora, escreva my depois de /home/ubuntu/ e pressione Ctrlx e então Ctrlf . O editor irá completar sua palavra até my_folder_ e você verá a árvore de diretórios, então use as setas do teclado para escolher o que você quer.

Visualizando números de linha #

Para visualizar os números de linha da visualização padrão, insira

:set number

Para ocultar números de linha

:set nonumber

Há também um atalho para acima. nu é o mesmo que number.

:set nonu

Para ocultar números de linha, também podemos usar

:set nu!

Escreva um arquivo se você esquecer de sudo antes de iniciar o vim #

Este comando salvará o arquivo aberto com direitos sudo

:w !sudo tee % >/dev/null

Você também pode mapear w!! para escrever um arquivo como root

:cnoremap w!! w !sudo tee % >/dev/null

Macros “descartáveis” rápidas #

Adicione isso ao seu vimrc:

nnoremap Q @q

Para começar a gravar a macro “descartável”, use qq. Para finalizar a gravação aperte q (no modo normal para ambos).

Para executar a macro gravada, use Q.

Isso é útil para macros que você precisa repetir muitas vezes seguidas, mas provavelmente não será usada novamente depois.

Como saber o diretório e/ou o caminho do arquivo que você está editando #

Para saber o caminho do diretório em que seu arquivo está você pode usar:

Esc para entrar no modo de comando

:pwd

Isso imprimirá o caminho para o diretório na parte inferior do editor, assim

I'm a ninja
~
~
~
~
~
/home/ubuntu/myfolder                                                1,5         All

Agora, se você quiser saber o nome do arquivo que está editando em relação ao diretório de trabalho do vim, você pode usar:

Esc para entrar no modo de comando CTRLG

I'm a ninja
~
~
~
~
~
"myfile"[Modified][New file] 1 line --100%--                         1,5         All

Finalmente, para obter o caminho absoluto para o arquivo que você está editando, use

Esc para entrar no modo de comando,

1 e, em seguida, CTRLG

I'm a ninja
~
~
~
~
~
"~/myfolder/myfile"[Modified][New file] 1 line --100%--              1,5         All

Recarrega automaticamente o vimrc ao salvar #

Para recarregar automaticamente o vimrc ao salvar, adicione o seguinte ao seu vimrc:

if has('autocmd') " ignore this section if your vim does not support autocommands
    augroup reload_vimrc
        autocmd!
        autocmd! BufWritePost $MYVIMRC,$MYGVIMRC nested source %
    augroup END
endif

e então, pela última vez, digite:

:so $MYVIMRC

Na próxima vez que você salvar seu vimrc, ele será recarregado automaticamente.

nested é útil se você estiver usando vim-airline. O processo de carregamento da linha aérea aciona alguns comandos automáticos, mas como você está executando um comando automático, eles são ignorados. nested permite acionar comandos automáticos aninhados e permite que a companhia aérea carregue corretamente.

Pesquisa dentro de um bloco de funções #

Para pesquisar o texto foo dentro de um bloco {} ao redor do cursor, use o seguinte comando (<ESC> - tecla escape, <CR> - tecla enter):

vi{<ESC>/\%Vfoo<CR>

agora você pode pular entre as correspondências dentro do bloco pressionando n e p. Se você tiver a opção hlsearch habilitada, isso irá destacar todas as correspondências. \%V é uma parte especial do padrão de pesquisa, que diz ao vim para pesquisar apenas na área selecionada visualmente. Você também pode fazer um mapeamento assim:

:vnoremap g/ <ESC>/\%V

Depois disso, o comando acima é encurtado para o seguinte:

vi{g/foo<CR>

Outro truque útil é imprimir todas as linhas que contêm o padrão:

vi{
:'<,'>g/foo/#

O intervalo '<,'> é inserido automaticamente.

Veja :help range e :help :g.

Copiar, mover ou excluir a linha encontrada #

Muitos usuários se encontram em uma situação em que desejam apenas copiar, mover ou excluir uma linha rapidamente e voltar para onde estavam.

Normalmente, se você deseja mover uma linha que contém a palavra lot abaixo da linha atual, digite algo como:

/lot<Esc>dd<C-o>p

Mas para aumentar a produtividade você pode usar este atalho nestes casos:

" It's recommended to turn on incremental search when doing so
set incsearch

" copy the found line
cnoremap $t <CR>:t''<CR>
" move the found line
cnoremap $m <CR>:m''<CR>
" delete the found line
cnoremap $d <CR>:d<CR>``

Então uma pesquisa assim:

/lot$m

moveria a linha que contém lot abaixo da linha em que seu cursor estava quando você iniciou a pesquisa.
