Saindo do Vim #

Para sair do Vim, primeiro certifique-se de estar no modo Normal pressionando Esc.

  • :q Enter (irá evitar que você saia se tiver alterações não salvas - abreviação de :quit)

Para descartar alterações e sair do Vim:

  • :q! Enter para forçar a saída e descartar as alterações (abreviação de :quit!, não deve ser confundido com :!q),
  • ZQ é um atalho que faz o mesmo que :q!,
  • :cq Enter quit e retornar erro (descartar todas as alterações para que o compilador não recompile este arquivo)

Para salvar as alterações e sair do Vim:

  • :wq Enter (abreviação de :write e :quit),
  • :x Enter (o mesmo que :wq, mas não escreverá se o arquivo não foi alterado),
  • ZZ é um atalho que faz o mesmo que :x (Salvar área de trabalho e sair do editor),
  • :[range]wq! Enter (escreva as linhas em [range])

Para fechar vários buffers de uma vez (mesmo em várias janelas e/ou abas), anexe a letra a a qualquer um dos Comandos acima (os que começam com :). Por exemplo, para escrever e sair de todas as janelas, você pode usar:

  • :wqa Enter ou
  • :xa Enter — Escreva todos os buffers alterados e saia do Vim. Se houver buffers sem nome de arquivo, que sejam somente leitura ou que não possam ser gravados por outro motivo, o Vim não será encerrado
  • :xa! Enter — Escreva todos os buffers alterados, mesmo aqueles que são somente leitura, e saia do Vim. Se houver buffers sem nome de arquivo ou que não possam ser gravados por outro motivo, o Vim não será encerrado
  • :qa Enter — tente sair, mas pare se houver algum arquivo não salvo;
  • :qa! Enter — sair sem salvar (descartar alterações em qualquer arquivo não salvo)

Se você abriu o Vim sem especificar um arquivo e deseja salvá-lo antes de sair, você receberá a mensagem E32: No file name. Você pode salvar seu arquivo e sair usando:

  • :wq filename Enter ou;
  • :x filename Enter

Explicação: #

O pressionamento de tecla : na verdade abre o modo Command. O comando q é uma abreviação de quit, w, de write e x, de exit (você também pode digitar :quit, :write e :exit se você quer). Atalhos não começando com : como ZZ e ZQ referem-se aos mapeamentos de teclas do modo Normal. Você pode pensar neles como atalhos.

O pressionamento de tecla ! às vezes é usado no final de um comando para forçar sua execução, o que permite descartar alterações no caso de :q!. Colocar o ! no início do comando tem um significado diferente. Por exemplo, pode-se digitar :!q errado em vez de :q! e o vim terminaria com um erro 127.

Uma maneira fácil de lembrar disso é pensar em ! como uma forma de insistir em executar algo. Assim como quando você escreve: “Eu quero sair!”

Fundamentos #

Execute tutoriais do vim interativos quantas vezes forem necessárias para se sentir confortável com o básico.

O Vim possui vários modos, por exemplo. modo normal, modo de inserção e modo de linha de comando.

Modo normal é para edição e navegação de texto. Neste modo h, j, k e l correspondem às teclas de cursor ←, ↓, ↑ e →. A maioria dos comandos no modo normal podem ser prefixados com uma “contagem”, por exemplo 3j desce 3 linhas.

Modo de inserção é para inserir o texto diretamente, neste modo o vim é semelhante a outros editores de texto mais simples. Para entrar no modo de inserção, pressione i no modo normal. Para sair pressione <ESC> (tecla Escape).

Modo de linha de comando é para executar comandos mais complexos, como salvar o arquivo e sair do vim. Pressione : para iniciar o modo de linha de comando. Para sair deste modo, você também pode pressionar <ESC>. Para salvar as alterações no arquivo use :w (ou :write). Para sair do vim sem salvar suas alterações, use :q! (ou :quit!).

Estes são alguns dos comandos mais úteis no vim:

Comando Descrição
eu (inserir) entra no modo de inserção antes da posição atual do cursor
’eu' entra no modo de inserção antes do primeiro caractere imprimível da linha atual
a (anexar) entra no modo de inserção depois da posição atual do cursor
A entra no modo de inserção depois do último caractere imprimível da linha atual
x excluir caractere na posição atual do cursor
X excluir caractere à esquerda para a posição atual do cursor
w passar para a próxima palavra
b mover para a palavra anterior
0 mover para o início da linha
$ mover para o fim da linha
r replace – entra no modo de substituição para um caractere. O próximo caractere que você digitar substituirá o caractere sob o cursor.
R entra no modo de substituição indefinidamente. Cada caractere que você digitar substituirá o caractere sob o cursor e avançará o cursor em um.
s substituto – exclui o caractere na posição atual do cursor e, em seguida, entra no modo de inserção
S exclua a linha atual em que o cursor está atualmente e entre no modo de inserção
<Esc>, <C-c> sai do modo de inserção e volta ao modo normal
u desfazer
<C-r> refazer
dd, dw, dl, d$ corte a linha atual, do cursor para a próxima palavra, ou o caractere, posição atual até o final da linha atual, respectivamente, note: D é o equivalente a d$
cc, cw, cl alterar a linha atual, do cursor para a próxima palavra, ou o caractere, respectivamente
yy, yw, yl, y$ puxe (“copia”) a linha atual, do cursor para a próxima palavra, ou o caractere, posição atual para o final da linha atual, respectivamente
p, P put (“colar”) depois ou antes da posição atual, respectivamente
o, O para criar uma nova linha vazia, depois ou antes da atual e entrar no modo de inserção
:w gravar o buffer atual no disco
:q!, ZQ sair sem escrever
:x, :wq, ZZ escrever e sair
:ajuda abra uma janela com o arquivo de ajuda
:ajuda {assunto} mostrar ajuda para um assunto específico
qz comece a gravar ações para registrar z, q para finalizar a gravação, @z para reproduzir as ações. z pode ser qualquer letra: q é frequentemente usado por conveniência. Leia mais: Macros

Salvando um arquivo somente leitura editado no Vim #

Às vezes, podemos abrir um arquivo que não temos permissão para escrever no Vim sem usar o sudo.

Use este comando para salvar um arquivo somente leitura editado no Vim.

:w !sudo tee > /dev/null %

Que você poderia mapear para :w!! em seu .vimrc:

cmap w!! w !sudo tee > /dev/null %

Você será apresentado um prompt como mostrado na imagem.

.

Pressione O e o arquivo será salvo. Ele permanece aberto no vi/vim para mais edição ou leitura e você pode sair normalmente digitando :q! já que o arquivo ainda está aberto como somente leitura.

Explicação do comando #

:w ............................ isn't modifying your file in this case, 
   ............................ but sends the current buffer contents to 
   ............................ a substituted shell command
   !sudo ...................... call the shell 'sudo' command
         tee .................. the output of the vi/vim write command is redirected 
                                using the 'tee' command
             > /dev/null ...... throws away the standard output, since we don't need 
                                to pass it to other commands
                         % .... expands to the path of the current file

Tutoriais interativos do Vim (como vimtutor) #

vimtutor é um tutorial interativo que cobre os aspectos mais básicos da edição de texto.

No sistema do tipo UNIX, você pode iniciar o tutorial com:

$ vimtutor

No Windows, “Vim tutor” pode ser encontrado no diretório “Vim 7.x” em “Todos os Programas” no menu do Windows.

Veja :help vimtutor para mais detalhes.

Outros tutoriais interativos incluem estes baseados em navegador:

  • Vim Adventures – Uma versão interativa do jogo vimtutor disponível na web. Apenas os primeiros níveis são gratuitos.
  • Open Vim – Um terminal interativo que ensina os comandos básicos com feedback.
  • Vim Genius – Outro terminal interativo que também inclui lições intermediárias e avançadas, incluindo macros e arglist.

Instalação #

O Vim em sua máquina - se houver - é muito provável que seja uma compilação “pequena” que carece de recursos úteis como suporte à área de transferência, realce de sintaxe ou até mesmo a capacidade de usar plugins.

Isso não é um problema se tudo o que você precisa é uma maneira rápida de editar arquivos de configuração, mas em breve você encontrará várias paredes se pretender fazer do Vim seu editor principal.

Portanto, geralmente é recomendado instalar uma compilação completa.

Instalação em Linux/BSD #

Nesses sistemas, o truque é simplesmente instalar a versão GUI que vem com um comando gvim para iniciar a GUI e um comando vim para iniciar a TUI.

Distribuições baseadas em Arch e Arch #

$ sudo pacman -R vim
$ sudo pacman -S gvim

Debian e distribuições baseadas em Debian #

$ sudo apt-get update
$ sudo apt-get install vim-gtk

Distribuições baseadas em Gentoo e Gentoo #

$ sudo emerge --sync
$ sudo emerge app-editors/gvim

Distribuições baseadas em RedHat e RedHat #

$ sudo yum check-update
$ sudo yum install vim-X11

Fedora #

$ sudo dnf check-update
$ sudo dnf install vim-X11

Distribuições baseadas em Slackware e Slackware #

$ sudo slackpkg update
$ sudo slackpkg install-new vim-gvim

Distribuições baseadas em OpenBSD e OpenBSD #

$ sudo pkg_add vim-x11

Distribuições baseadas em FreeBSD e FreeBSD #

$ sudo pkg install editors/vim

Instalação no Mac OS X #

A estratégia é semelhante ao Mac OS X: instalamos a versão GUI para obter a GUI e a TUI. Ao final, devemos ser capazes de:

  • clique duas vezes no ícone MacVim no Finder,
  • clique no ícone MacVim no Dock,
  • emita $ mvim no shell para abrir a GUI do MacVim,
  • emita $ mvim -v no shell para abrir a TUI do MacVim.

Instalação normal #

Baixe e instale um instantâneo oficial como faria com qualquer outro aplicativo do Mac OS X.

Coloque o script mvim que vem junto com o MacVim em algum lugar em seu $PATH.

Gerenciador de pacotes #

Portas Mac: #

$ sudo port selfupdate
$ sudo port install macvim

Cerveja caseira: #

$ brew install macvim

Para tornar o MacVim o console padrão do Vim:

$ brew install macvim --with-override-system-vim

Instalação no Windows #

Não há Vim nos sistemas Windows por padrão. Você pode baixar e instalar o Vim do site do Tuxproject para compilações mais atualizadas e completas ou pode baixar e instalar o Vim do [site do Vim] oficial] 2.

Chocolate #

> choco install vim

Construindo o Vim a partir da fonte #

Se os métodos acima não atenderem às suas necessidades, ainda é possível construir o Vim você mesmo, com somente as opções que você precisa.

Este tópico será discutido em sua própria seção (atualmente em rascunho).

Suspendendo vim #

Ao usar o vim na linha de comando, você pode suspender o vim e voltar ao seu prompt, sem realmente sair do vim. Portanto, mais tarde você poderá recuperar sua sessão vim a partir do mesmo prompt.

Quando estiver no modo Normal (se não estiver, pressione esc para chegar lá), execute um destes comandos:

:stenter

:susenter

:stopenter

:suspenderentrar

Alternativamente, em alguns sistemas, quando no modo Normal ou Visual, emitir Ctrl+Z terá o mesmo efeito.

Observação: Se autowrite for definido, os buffers com alterações e nomes de arquivos serão gravados. Adicione um ! antes de enter para evitar, por exemplo. :st!enter.

Mais tarde, quando você quiser retornar à sua sessão vim, se você não tiver suspendido nenhum outro trabalho, emitir o seguinte restaurará o vim como seu trabalho em primeiro plano.

fgentrar

Caso contrário, você precisará encontrar o ID do job da sua sessão vim emitindo jobsenter e então colocando em primeiro plano os jobs correspondentes fg %[job ID]enter ex. fg %1enter.

O que fazer em caso de falha #

O Vim salva todas as suas edições não salvas em um arquivo de troca, um arquivo extra que é excluído assim que as alterações são confirmadas salvando. O nome do arquivo de troca é geralmente o nome do arquivo que está sendo editado precedido por um . e com um sufixo .swp (você pode vê-lo com :sw).

Portanto, caso seu processo vim termine antes de você ter a chance de salvar suas edições, você pode recuperar seu trabalho aplicando as alterações contidas no arquivo de troca ao seu arquivo atual usando a opção de linha de comando -r. Por exemplo, se myFile for o arquivo que você estava editando, use:

$ vi -r myFile

para recuperar as alterações não confirmadas.

Se existir um arquivo de troca, o vim deve solicitar opções de recuperação de qualquer maneira

$ vi myFile
E325: ATTENTION
Found a swap file by the name ".myFile.swp"
...
Swap file ".myFile.swp" already exists!
[O]pen Read-Only, (E)dit anyway, (R)ecover, (D)elete it, (Q)uit, (A)bort:

Se você escolher (R)ecover, as alterações do arquivo swp serão aplicadas, mas o arquivo de troca não será excluído, então não se esqueça de excluir o arquivo de troca posteriormente se estiver satisfeito com a recuperação.
