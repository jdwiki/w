Um plugin é um script ou conjunto de scripts que altera o comportamento padrão do Vim, adicionando recursos inexistentes ou estendendo recursos existentes.

Muitas vezes adicionados “recursos inexistentes” incluem:

  • comentando,
  • detecção de recuo,
  • preenchimento automático,
  • correspondência difusa,
  • suporte para um idioma específico,
  • etc

Muitas vezes, os “recursos existentes” estendidos incluem:

  • omnicompleção,
  • objetos de texto e movimentos,
  • puxando e colocando,
  • linha de estado,
  • pesquisar e substituir,
  • comutação de página de buffer/janela/aba,
  • dobrando,
  • etc

Coelho #

Vundle é um gerenciador de plugins para o Vim.

Instalando o Vundle #

(Os detalhes completos da instalação podem ser encontrados no Vundle Quick Start)

  1. Instale o Git e clone o Vundle em ~/.vim/bundle/Vundle.vim.

  2. Configure plugins adicionando o seguinte no topo do seu .vimrc, adicionando ou removendo plugins conforme necessário (os plugins na lista são meramente ilustrativos)

     set nocompatible              " be iMproved, required
 filetype off                  " required

 " set the runtime path to include Vundle and initialize
 set rtp+=~/.vim/bundle/Vundle.vim
 call vundle#begin()
 " alternatively, pass a path where Vundle should install plugins
 "call vundle#begin('~/some/path/here')

 " let Vundle manage Vundle, required
 Plugin 'VundleVim/Vundle.vim'

 " All of your Plugins must be added before the following line
 call vundle#end()            " required
 filetype plugin indent on    " required
 " To ignore plugin indent changes, instead use:
 "filetype plugin on

 "place non-Plugin stuff after this line

  3. Instale os plugins: iniciando o Vim e executando :PluginInstall.

Formatos de plug-in suportados #

A seguir estão exemplos de diferentes formatos suportados. Mantenha os comandos do Plugin entre vundle#begin e vundle#end.

Localização do plug-in Uso
plugin no GitHub Plugin 'tpope/vim-fugitive'
plugin de http://vim-scripts.org/vim/scripts.html Plugin 'L9'
Plugin Git não hospedado no GitHub Plugin 'git://git.wincent.com/command-t.git'
git repos em sua máquina local (ou seja, ao trabalhar em seu próprio plugin) Plugin 'file:///home/gmarik/path/to/plugin'
O script sparkup vim está em um subdiretório deste repositório chamado vim. Passe o caminho para definir o runtimepath corretamente. Plugin 'rstacruz/sparkup', {'rtp': 'vim/'}
Instale o L9 e evite um conflito de nomes se você já instalou uma versão diferente em outro lugar. Plugin 'ascenator/L9', {'name': 'newL9'}

Trabalhar em uma conta compartilhada, por exemplo, no nó principal do cluster pode levantar problemas do ponto de uso do disco pelo diretório .vim. Existem alguns pacotes que ocupam uma quantidade considerável de espaço em disco, por exemplo YCM. Então, por favor, escolha seu diretório de plugins Vundle com sabedoria, e é muito fácil fazer isso configurando rtp. E também se você está planejando instalar qualquer plugin vim, não faça diretamente git clone no diretório bundle. Use o caminho Vundle.

Patógeno #

vim-pathogen é um gerenciador de runtimepath criado por Tim Pope para facilitar a instalação de plugins e arquivos de tempo de execução em seus próprios diretórios privados.

Instalando o Patógeno #

  1. Coloque patógeno em ~/.vim/bundle (aqui com Git, mas não é obrigatório):

    git clone https://github.com/tpope/vim-pathogen.git

  2. Adicione as seguintes linhas no topo do seu .vimrc:

     " enable vim-pathogen
 runtime bundle/vim-pathogen/autoload/pathogen.vim
 execute pathogen#infect()
  • a diretiva runtime especifica o caminho para o script de autoload do vim-pathogen;
  • execute pathogen#infect() o inicia.

Uma vez iniciado, o Pathogen iniciará automaticamente uma varredura nas pastas em ~/.vim/bundle e carregará o plugin de cada uma delas.

Usando Patógeno #

  1. Coloque o diretório de nível superior do seu plugin em ~/.vim/bundle/ para disponibilizá-lo na próxima vez que você iniciar o Vim.
  2. Execute :Helptags para indexar a documentação do seu novo plugin.

Benefícios #

  • Cada plugin reside em seu próprio diretório em ~/.vim/bundle/.
  • Seu .vimrc fica limpo da configuração necessária para carregar plugins.

O esforço necessário para “gerenciar” um plugin é assim reduzido a:

  • coloque seu diretório de nível superior em ~/.vim/bundle/ para instalá-lo,
  • substitua seu diretório de nível superior para atualizá-lo,
  • exclua seu diretório de nível superior para desinstalá-lo.

Como você executa essas três ações (manualmente, por meio de uma ferramenta de automação, com Git/Svn/Hg/whatever…) depende completamente de você.

Como funcionam os plugins #

Um plug-in pode se apresentar como um único arquivo contendo 30 linhas de vimscript ou como 20 MB de vimscript/python/ruby/whatever dividido em muitos arquivos em uma dúzia de diretórios que dependem de várias ferramentas externas.

O primeiro é obviamente fácil de instalar e gerenciar, mas o último pode ser um grande desafio.

O princípio #

A opção 'runtimepath' informa ao Vim onde procurar por scripts de tempo de execução. O valor padrão faz com que o Vim procure scripts nos seguintes diretórios em ordem:

  • em sistemas do tipo UNIX
  • $HOME/.vim/
  • $VIM/vimfiles/
  • $VIMRUNTIME/
  • $VIM/vimfiles/after/
  • $HOME/.vim/after/
  • no Windows
  • $HOME/vimfiles/
  • $VIM/vimfiles/
  • $VIMRUNTIME/
  • $VIM/vimfiles/after/
  • $HOME/vimfiles/after/

Dos diretórios acima, instale plugins apenas nos que estão em negrito. Os outros causarão instabilidade sem uma boa razão. Instalar um plugin se resume a colocar cada um de seus componentes no diretório correto em $HOME/.vim/ ou $HOME/vimfiles/.

O método manual #

Plug-in de arquivo único #

Coloque o arquivo em $HOME/.vim/plugin ou $HOME/vimfiles/plugin

Isso forneceria o plug-in na inicialização do Vim. Agora o usuário pode usar tudo definido nele. Se o plugin, no entanto, precisar de ativação, o usuário deve executar o comando sempre que quiser usá-lo ou adicionar o comando ao .vimrc

Pacote #

Um pacote é uma estrutura de diretório que o plug-in usa. Consiste em todos os arquivos do plugin nos subdiretórios apropriados.

Para instalar tal plugin, os subdiretórios devem ser mesclados com seus equivalentes em $HOME/.vim/plugin. Essa abordagem, no entanto, leva à mistura de arquivos de plugins diferentes nos mesmos diretórios e pode levar a problemas de namespace.

Outra abordagem é copiar o diretório inteiro em $HOME/.vim/bundle.

Ao usar esta abordagem deve haver pelo menos um arquivo .vim sob o diretório $HOME/.vim/bundle/autoload. Esses arquivos seriam originados pelo vim na inicialização.

Nota: Dependendo do sistema operacional do usuário, o prefixo de todos os caminhos pode ser $HOME/vimfiles. Para mais detalhes veja Como funcionam os plugins
