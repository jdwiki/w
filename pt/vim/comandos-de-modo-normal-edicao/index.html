Copiar, Recortar e Colar #

No Vim, essas operações são tratadas de forma diferente do que você pode estar acostumado em quase qualquer outro editor ou processador de texto moderno (Ctrl-C, Ctrl-X, Ctrl-V). Para entender, você precisa saber um pouco sobre registros e movimentos.

Observação: esta seção não abrangerá a cópia e o corte do Modo Visual ou a remoção de alcance, pois estão além do escopo do Modo Normal e da edição básica.

Registros #

O Vim usa o conceito de registers para lidar com a movimentação de texto dentro do próprio programa. O Windows possui uma única área de transferência para esse propósito, que é análoga a um único registro no Vim. Ao copiar, recortar e colar no Vim, existem maneiras de usar um fluxo de trabalho de edição similarmente simples (no qual você não precisa pensar em registros), mas também há possibilidades muito mais complexas.

Um registro é direcionado para a entrada/saída de um comando prefixando o comando com " e um nome de letra minúscula.

Moções #

Um movimento no Vim é qualquer comando que mova a posição do cursor para outro lugar. Ao copiar, recortar e colar no Modo Normal, as possibilidades de seleção de texto para movimento são limitadas apenas pelo seu conhecimento de movimentos. Alguns serão ilustrados a seguir.

Copiando e recortando #

Os comandos básicos para copiar e cortar operações são y (‘yank’, para copiar) e d (’excluir’, para cortar). Você verá as semelhanças na tabela a seguir.

| Comando | : | Descrição | | | —— | —— | —— | | y{motion} | | Copie (‘yank’) o texto indicado pelo movimento no registro padrão | aa | | Copie a linha atual para o registro padrão, linewise | S | | Copie a linha atual no registro padrão (sinônimo de yy) | “ayiw | | Copie a palavra sobre a qual o cursor está no registrador ‘a’ | 20"byy | | Copie vinte linhas, começando do cursor, no registrador ‘b’ | d{motion} | | Cortar (’excluir’) o texto indicado pelo movimento no registro padrão | dd | | Corte a linha atual no registro padrão, linewise | D | | Corta do cursor até o final da linha no registro padrão (NÃO é sinônimo de dd) | “adiw | | Corta a palavra onde o cursor está no registrador ‘a’ | 20"bdd | | Corte vinte linhas, começando do cursor, no registrador ‘b’

Nota: quando algo é copiado ou recortado linewise, o comportamento de colagem mostrado abaixo colocará o texto antes ou depois da line atual (em vez do cursor). Seguem exemplos para esclarecer.

Colando #

Existem várias maneiras de colar no Vim, dependendo do que você está tentando realizar.

| Comando | : | Descrição | | | —— | —— | —— | | p | | Cole o que estiver no registro padrão depois do cursor | P | | Cole o que estiver no registro padrão antes do cursor | “ap | | Cole o conteúdo do registrador ‘a’ após o cursor | “cP | | Cole o conteúdo do registrador ‘c’ antes do cursor

Então, como faço um corte e colagem realmente simples?

Se eu tiver o seguinte texto:

1 This line should be second
2 This line should be first

Eu posso fazer o mais simples recortar e colar colocando meu cursor em algum lugar na linha 1 e digitando ddp. Aqui estão os resultados:

1 This line should be first
2 This line should be second

O que aconteceu? dd ‘Corta’ a primeira linha (em sentido de linha) no registro padrão - que conterá apenas uma coisa de cada vez, como a área de transferência do Windows - e p cola a linha após o atual, que acabou de ser alterado devido ao comando dd.

Aqui está um exemplo não tão simples. Eu preciso mover um par de palavras ao redor. (Isso é artificial e desnecessário, mas você pode aplicar esse princípio a pedaços maiores de código.)

1 These words order out are of

Eu posso repetir w para chegar ao ‘o’ na frente de ‘pedido’ (ou b se eu apenas digitei e percebi meu erro).

Então “adaw para colocar ‘ordem’ no registrador ‘a’.

Então w para chegar ao ‘a’ em ‘are’.

Depois disso, eu digitaria “bdaw para colocar ‘are’ no registro ‘b’. Agora eu tenho isso exibido:

1 These words out of

Para ser claro, agora ‘order’ está no registro ‘a’ e ‘are’ está no registro ‘b’, como duas pranchetas separadas.

Para organizar as palavras corretamente, eu digito b para chegar ao ‘o’ em ‘out’, e então “bP para colocar ‘are’ do registro ‘b’ no frente de ‘fora’:

1 These words are out of

Agora eu digito A para chegar ao final da linha, seguido por EspaçoEsc (assumindo que não havia espaço após ‘de’) e “ap para colocar ‘ordem’ onde ele pertence.

1 These words are out of order

Desfazer e Refazer Básico #

Desfazer #

| Comando | : | Descrição | | | —— | —— | —— | | u |u,desfazer | Desfazer a alteração mais recente | 5u | | Desfaça as cinco alterações mais recentes (use qualquer número)

Por favor, esteja ciente de que no Vim, a ‘alteração mais recente’ varia de acordo com o modo em que você está. Se você entrar no Modo de Inserção (i) e digitar um parágrafo inteiro antes de retornar ao Modo Normal Esc), esse parágrafo inteiro é considerado a mudança mais recente.

Refazer #

| Comando | : | Descrição | | | —— | —— | —— | | Ctrl-R |red,redo | Refazer a alteração desfeita mais recente | 2Ctrl-R| | Refazer as duas alterações desfeitas mais recentes (use qualquer número)

Existe uma outra maneira de desfazer e refazer alterações no Vim que é tratada de maneira um pouco diferente. Quando você desfaz uma mudança com u, você percorre de volta os nós em uma ‘árvore’ de suas mudanças, e pressionando Ctrl-R volta por esses nós em ordem. (A árvore de desfazer é um tópico separado e é muito complexo para ser abordado aqui.)

Você pode também usar U (ou seja, maiúscula) para remover todas as últimas alterações em uma única linha (a linha onde suas últimas alterações foram feitas). Isso não atravessa os nós da árvore da mesma forma que u. Usar U na verdade conta como uma mudança em si - outro nó forward na árvore - de modo que se você pressionar U uma segunda vez imediatamente após o primeiro, ele atuará como um comando Refazer .

Cada um tem seus usos, mas u / :desfazer deve cobrir a maioria dos casos simples.

Introdução - Nota rápida sobre o modo normal #

No Modo Normal, os comandos podem ser inseridos por combinações diretas de teclas (digitando u para desfazer a última alteração, por exemplo). Esses comandos geralmente têm equivalentes no modo ’ex’, acessados ​​digitando dois pontos :, que o coloca em um buffer de linha única na parte inferior da janela do Vim.

No modo ’ex’, após digitar os dois pontos você digita um nome de comando ou sua abreviação seguido de Enter para executar o comando. Então, :undoEnter realiza a mesma coisa que digitar diretamente u no Modo Normal.

Você pode ver que os comandos diretos geralmente serão mais rápidos (uma vez aprendidos) do que os comandos ’ex’ para edição simples, mas para ser completo, sempre que possível na documentação a seguir, se ambos estiverem disponíveis para uso, ambos serão mostrados.

A maioria desses comandos também pode ser precedida por um count prefixando ou intercalando um número - digitar 3dd no modo normal, por exemplo, exclui três linhas (começando na posição atual do cursor).

Repita a última alteração #

O comando Repetir, executado com a tecla ponto ou ponto (.), é mais útil do que parece à primeira vista. Uma vez aprendido, você se verá usando-o com frequência.

| Comando | : | Descrição | | | —— | —— | —— | | . | | Repita a última alteração | 10. | | Repita a última alteração 10 vezes

Então, para um exemplo muito simples, se você fizer uma alteração na linha 1 digitando iIEsc, com o seguinte resultado:

1 I made a mistake
2  made a mistake
3  made a mistake

Seu cursor estará na posição 1 da linha 1, e tudo que você precisa fazer para corrigir as próximas duas linhas é pressionar j. duas vezes - isto é, j para mover para baixo uma linha e . para repetir a última alteração, que foi a adição do I. Não há necessidade de voltar ao modo de inserção duas vezes para corrigir essas linhas.

Torna-se muito mais poderoso quando usado para repetir macros.

Conclusão #

A conclusão pode ser usada para combinar palavras usadas em um documento. Ao digitar uma palavra, Ctrlp ou Ctrln corresponderá a palavras semelhantes anteriores ou próximas no documento.

Isso pode até ser combinado com o modo Ctrl-X para completar linhas inteiras. Por exemplo digite algo como:

This is an example sentence.

então vá para a próxima linha e comece a digitar a mesma frase:

Thi

e pressione Ctrlp que resultará em:

This

Agora ainda no modo de inserção, pressione Ctrlx Ctrlp e a próxima palavra será concluída resultando em:

This is

Continue pressionando Ctrlx Ctrlp até que a linha inteira esteja completa.

Se você sabe que deseja completar uma linha inteira, digite isto:

This is an example sentence.

então na próxima linha digite:

Thi

e pressione x Ctrll para completar a linha.

Se a conclusão que está sendo feita for um nome de arquivo Ctrlx Ctrlf pode ser usado para completar esse diretório. Modelo:

~/Deskt

então pressione Ctrlx Ctrlf e:

~/Desktop

será concluído (se nesse local). Ctrlx Ctrlf pode ser usado repetidamente para listar os arquivos na área de trabalho.
