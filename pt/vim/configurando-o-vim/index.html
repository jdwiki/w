Arquivos e diretórios #

O que quer que você faça para personalizar o Vim, isso NUNCA deve acontecer fora de $HOME:

  • no Linux, BSD e Cygwin, $HOME geralmente é /home/username/,
  • no Mac OS X, $HOME é /Users/username/,
  • no Windows, $HOME geralmente é C:\Users\username\.

A localização canônica para seu diretório vimrc e vim está em a raiz desse diretório $HOME:

  • em sistemas do tipo Unix

    $HOME/.vimrc       <-- the file
$HOME/.vim/        <-- the directory

  • no Windows

    $HOME\_vimrc       <-- the file
$HOME\vimfiles\    <-- the directory

O layout acima é garantido para funcionar, agora e no futuro.

O Vim 7.4 tornou possível manter seu adorável vimrc dentro do seu vim diretório. É realmente uma boa ideia, mesmo porque torna mais fácil mova sua configuração ao redor.

Se você usar o 7.4 exclusivamente, o seguinte será suficiente:

  • em sistemas do tipo Unix

    $HOME/.vim/vimrc

  • no Windows

    $HOME\vimfiles\vimrc

Se você deseja os benefícios de um vim/ independente, mas usa tanto a versão 7.4 quanto uma versão mais antiga, ou apenas uma versão mais antiga, a solução mais simples e à prova de futuro é colocar isso linha e somente esta linha:

runtime vimrc

neste arquivo:

  • em sistemas do tipo Unix

    $HOME/.vimrc

  • no Windows

    $HOME\_vimrc

e faça sua configuração em $HOME/.vim/vimrc ou $HOME\vimfiles\vimrc.

Mapeamentos #

  • Não coloque comentários após os mapeamentos, isso vai quebrar as coisas.
  • Use :map <F6> para ver o que está mapeado para <F6> e em qual modo.
  • Use :verbose map <F6> para ver também onde foi mapeado pela última vez.
  • :map e :map! são muito genéricos. Use :n[nore]map para mapeamentos de modo normal, :i[nore]map para modo de inserção, :x[nore]map para modo visual, etc.

Mapeamentos recursivos #

Use mapeamentos recursivos somente se pretender usar outros mapeamentos em seus mapeamentos:

nnoremap b     B
nmap     <key> db

Neste exemplo, b é feito para funcionar como B no modo normal. Como usamos b em um mapeamento recursivo, pressionar <key> funcionará como dB, não como db.

Mapeamentos não recursivos #

Use mapeamentos não recursivos somente se pretender usar comandos padrão em seus mapeamentos, o que quase sempre é o que você deseja:

nnoremap <key> db

Neste exemplo, usamos b em um mapeamento não recursivo, portanto, pressionar a tecla sempre funcionará como db, quer tenhamos remapeado b ou não.

Executando um comando de um mapeamento #

nnoremap <key> :MyCommand<CR>

Executando vários comandos de um mapeamento #

nnoremap <key> :MyCommand <bar> MyOtherCommand <bar> SomeCommand<CR>

Chamando uma função de um mapeamento #

nnoremap <key> :call SomeFunction()<CR>

Mapeando um mapeamento <Plug> #

map <key> <Plug>name_of_mapping

Veja :help map-commands, :help key-notation e :help <plug>.

consulte https://www.wikiod.com/pt/vim/mapeamentos-de-teclas-no-vim para ler mais

Alternar enumeração de linhas #

Para habilitar - digite:

:set number ou :set nu.

Para desabilitar - digite:

:set nonumber ou :set nove.

Para habilitar a enumeração relativa à localização do cursor - digite:

:definir número relativo.

Para desabilitar a enumeração relativa ao local do cursor - digite:

:set norelativenumber.

Nota: Para alterar se a linha atual mostra o número da linha real ou 0, use os comandos :set number e :set nonumber enquanto o atributo relativenumber estiver ativo.

O arquivo vimrc #

O arquivo .vimrc (pronuncia-se Vim-wreck) é um arquivo de configuração do Vim. Ele contém comandos que serão executados pelo Vim toda vez que for iniciado.

Por padrão o arquivo está vazio ou inexistente; você pode usá-lo para personalizar seu ambiente Vim.

Para descobrir onde o Vim espera que o arquivo vimrc seja armazenado, abra o Vim e execute:

:echo $MYVIMRC

Unix: em um sistema Unix como Mac ou Linux, seu vimrc será chamado .vimrc e geralmente estará localizado em seu diretório pessoal ($HOME/.vimrc).

Windows: no Windows será chamado _vimrc e localizado em seu diretório pessoal (%HOMEPATH%/_vimrc).

Na inicialização, o Vim pesquisará em vários lugares por um arquivo vimrc. O primeiro que existe é usado, os outros são ignorados. Para uma referência completa veja o artigo de documentação :h $MYVIMRC.

Quais opções posso usar? #

Se você não sabe quais opções você deve usar, você pode estar interessado no comando :options.

Isso abrirá uma divisão com todas as opções do Vim listadas e com seu valor atual exibido. Existem 26 seções para exibir todas as opções que você pode experimentar.

por exemplo.

4 displaying text

scroll    number of lines to scroll for CTRL-U and CTRL-D
    (local to window)
     set scr=20
scrolloff    number of screen lines to show around the cursor
     set so=5
wrap    long lines wrap
     set nowrap    wrap

...

Em uma linha de valor (por exemplo, set nowrap) você pode pressionar CR para alternar o valor (se for um valor binário). Em uma linha de opção (por exemplo, wrap long line wrap) você pode pressionar CR para acessar a documentação desta opção.

Funções #

  • Não se esqueça do estrondo para permitir que o Vim sobrescreva essa função na próxima vez que você recarregar o script onde a função está definida.
  • As funções personalizadas devem começar com um caractere maiúsculo (funções globais), ou com s: (funções locais de script), ou devem ser prefixadas com o nome associado ao plug-in de carregamento automático onde são definidas (por exemplo, em {&rtp }/autoload/foo/bar.vim poderíamos definir foo#bar#functionname()).
  • Para poder usar os parâmetros na função, use a:parameter_name. Funções variáveis ​​podem ser definidas com as reticências ..., para acessar os parâmetros use a:000 (lista de todos os parâmetros), ou a:0 (número de parâmetros igual a len(a:000) ), a:1 primeiros parâmetros sem nome e assim por diante.
  • As funções podem ser chamadas assim: :call MyFunction(param1, param2)
  • Cada linha em uma função começa implicitamente com um :, portanto, todos os comandos são comandos de dois pontos
  • Para evitar que a função continue sua execução em caso de erro, é melhor anotar a assinatura da função com abort

Exemplo #

function! MyFunction(foo, bar, ... ) abort
    return a:foo . a:bar . (a:0 > 0 ? a:1 : '')
endfunction

Funções de script #

Se você planeja usar sua função apenas no arquivo onde ela está definida (seja porque você quebrou uma função maior em partes menores, ou porque você a usará em um comando, um mapeamento, …), você pode prefixar com s:, evitando sujar seu namespace global com funções internas inúteis:

function! s:my_private_function() " note we don't need to capitalize the first letter this time
    echo "Hi!"
endfunction

Usando s:funções de mapeamentos #

Se sua função local de script for usada em um mapeamento, você precisa referenciá-la usando o prefixo especial <SID>:

nnoremap <your-mapping-key> :call <SID>my_private_function()<CR>

Veja :help user-functions.

Observe, no entanto, que desde o Vim 7, é considerado uma prática recomendada definir abreviações de mapeamentos, comandos e menus em plug-ins (ft) e definir funções em plug-ins de carregamento automático - exceto as funções que os plug-ins precisam usar quando são carregados. Isso significa que hoje em dia a necessidade de chamar scripts de funções locais a partir de mapeamentos não é tão pertinente quanto antigamente.

Esquema de cores #

O Vim vem com vários esquemas de cores pré-instalados. No Linux, os esquemas de cores que vêm com o Vim são armazenados em /usr/share/vim/vim74/colors/ (onde 74 é o número da sua versão, sem pontos); O MacVim os armazena em /Applications/MacVim.app/Contents/Resources/vim/runtime/colors.

Alterando Esquemas de Cores #

O comando colorscheme muda o esquema de cores atual.

Por exemplo, para definir o esquema de cores para “robokai”:

:colorscheme robokai

O esquema de cores padrão é chamado de forma criativa de default, portanto, para retornar a ele, use

:colorscheme default

Para visualizar todos os esquemas de cores atualmente instalados, digite :colorscheme seguido por espaço e então tab ou ctrld< /kbd>.

Instalando Esquemas de Cores #

Esquemas de cores instalados pelo usuário podem ser colocados em ~/.vim/colors/. Uma vez que um esquema de cores é adicionado a este diretório, ele aparecerá como uma opção para o comando colorscheme.

Para encontrar novos esquemas de cores, existem sites como vimcolors que contêm uma variedade de esquemas de cores. Também existem ferramentas como vim.ink e Vivify para ajudá-lo a criar seus próprios esquemas de cores, ou você pode criar eles à mão.

Opções de configuração #

Comumente você usaria :set para definir opções ao seu gosto em seu .vimrc. Existem muitas opções que podem ser alteradas.

Por exemplo, para usar espaços para recuo:

:set expandtab
:set shiftwidth=4
:set softtabstop=4

Realce de sintaxe #

Ative o realce de sintaxe quando o terminal tiver cores

if &t_Co > 2 || has("gui_running")
    syntax on
end

Mostrar espaços em branco e guias à direita. Mostrar guias pode ser especialmente útil ao procurar erros em Makefiles.

set list listchars=tab:\|_,trail:.
highlight SpecialKey ctermfg=DarkGray

Opções #

Existem três tipos de opções:

  • opções booleanas,
  • opções de strings,
  • opções de número.

Para verificar o valor de uma opção,

  • use :set option? para verificar o valor de uma opção,
  • use :verbose set option? para ver também onde foi definido pela última vez.

Configurando opções booleanas #

set booloption      " Set booloption.
set nobooloption    " Unset booloption.

set booloption!     " Toggle booloption.

set booloption&     " Reset booloption to its default value.

Configurando opções de string #

set stroption=baz   " baz

set stroption+=buzz " baz,buzz
set stroption^=fizz " fizz,baz,buzz
set stroption-=baz  " fizz,buzz

set stroption=      " Unset stroption.

set stroption&      " Reset stroption to its default value.

Configurando opções de número #

set numoption=1     " 1

set numoption+=2    " 1 + 2 == 3
set numoption-=1    " 3 - 1 == 2
set numoption^=8    " 2 * 8 == 16

Usando uma expressão como valor #

  • usando concatenação:

    execute "set stroption=" . my_variable

  • usando :let:

    let &stroption = my_variable

Veja :help :set e :help :let.

Variáveis #

Como a maioria das linguagens de script, o vimscript possui variáveis.

Pode-se definir uma variável com o comando :let:

let variable = value

e exclua-o com :unlet:

unlet variable

No Vim, as variáveis ​​podem ser delimitadas adicionando uma única letra e dois pontos ao seu nome. Os autores de plugins usam esse recurso para imitar opções:

let g:plugin_variable = 1

Veja :help internal-variables.

Comandos #

  • Não se esqueça do bang para permitir que o Vim sobrescreva esse comando na próxima vez que você recarregar seu vimrc.
  • Os comandos personalizados devem começar com um caractere maiúsculo.

Exemplos #

command! MyCommand call SomeFunction()
command! MyOtherCommand command | Command | command
  • Veja :help user-commands.

Grupos de comando automático #

  • Grupos de comando automático são bons para organização, mas também podem ser úteis para depuração. Pense neles como pequenos namespaces que você pode ativar/desativar à vontade.

Exemplo #

augroup MyGroup
    " Clear the autocmds of the current group to prevent them from piling
    " up each time you reload your vimrc.
    autocmd!

    " These autocmds are fired after the filetype of a buffer is defined to
    " 'foo'. Don't forget to use 'setlocal' (for options) and '<buffer>'
    " (for mappings) to prevent your settings to leak in other buffers with
    " a different filetype.
    autocmd FileType foo setlocal bar=baz
    autocmd FileType foo nnoremap <buffer> <key> :command<CR>

    " This autocmd calls 'MyFunction()' everytime Vim tries to create/edit
    " a buffer tied to a file in /'path/to/project/**/'.
    autocmd BufNew,BufEnter /path/to/project/**/* call MyFunction()
augroup END

Veja :help autocommand.

Condicionais #

if v:version >= 704
    " Do something if Vim is the right version.
endif

if has('patch666')
    " Do something if Vim has the right patch-level.
endif

if has('feature')
    " Do something if Vim is built with 'feature'.
endif

Veja :help has-patch e :help feature-list.
