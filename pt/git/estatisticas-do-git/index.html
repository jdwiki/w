Sintaxe #

  • git log [<options>] [<revision range>] [[–] <path>]
  • git log –pretty=short | git shortlog [<opções>]
  • git shortlog [<options>] [<revision range>] [[–] <path>]

Parâmetros #

Parâmetro Detalhes
-n, --numbered Classifique a saída de acordo com o número de commits por autor em vez de ordem alfabética
-s, --summary Forneça apenas um resumo da contagem de commits
-e, --email Mostrar o endereço de e-mail de cada autor
--format[=<formato>] Em vez do assunto do commit, use algumas outras informações para descrever cada commit. <format> pode ser qualquer string aceita pela opção --format do git log.
-w[<largura>[,<indent1>[,<indent2>]]] Enrole a saída envolvendo cada linha em width. A primeira linha de cada entrada é recuada pelo número de espaços indent1, e as linhas subsequentes são recuadas por espaços indent2.
<intervalo de revisão> Mostrar apenas commits no intervalo de revisão especificado. Padrão para todo o histórico até o commit atual.
[--] <caminho> Mostrar apenas commits que explicam como os arquivos correspondentes ao path surgiram. Os caminhos podem precisar ser prefixados com “–” para separá-los das opções ou do intervalo de revisão.

Linhas de código por desenvolvedor #

git ls-tree -r HEAD | sed -Ee 's/^.{53}//' | \
while read filename; do file "$filename"; done | \
grep -E ': .*text' | sed -E -e 's/: .*//' | \
while read filename; do git blame --line-porcelain "$filename"; done | \
sed -n 's/^author //p' | \
sort | uniq -c | sort -rn

Listando cada branch e a data de sua última revisão #

for k in `git branch -a | sed s/^..//`; do echo -e `git log -1 --pretty=format:"%Cgreen%ci %Cblue%cr%Creset" $k --`\\t"$k";done | sort

Commits por desenvolvedor #

Git shortlog é usado para resumir as saídas do git log e agrupar os commits por autor.

Por padrão, todas as mensagens de commit são mostradas, mas o argumento --summary ou -s pula as mensagens e fornece uma lista de autores com seu número total de commits.

--numbered ou -n altera a ordem de ordem alfabética (por autor crescente) para número de commits decrescente.

git shortlog -sn        #Names and Number of commits

git shortlog -sne       #Names along with their email ids and the Number of commits

ou

git log --pretty=format:%ae \
| gawk -- '{ ++c[$0]; } END { for(cc in c) printf "%5d %s\n",c[cc],cc; }'

Observação: os commits da mesma pessoa não podem ser agrupados quando seu nome e/ou endereço de e-mail foram escritos de forma diferente. Por exemplo, John Doe e Johnny Doe aparecerão separadamente na lista. Para resolver isso, consulte o recurso .mailmap.

Confirmações por data #

git log --pretty=format:"%ai" | awk '{print " : "$1}' | sort -r | uniq -c

Número total de commits em um branch #

git log  --pretty=oneline |wc -l

Lista todos os commits em formato bonito #

git log --pretty=format:"%Cgreen%ci %Cblue%cn  %Cgreen%cr%Creset %s"

Isso dará uma boa visão geral de todos os commits (1 por linha) com data, usuário e mensagem de commit.

A opção --pretty tem muitos espaços reservados, cada um começando com %. Todas as opções podem ser encontradas aqui

Encontre todos os repositórios Git locais no computador #

Para listar todos os locais do repositório git no seu, você pode executar o seguinte

find $HOME -type d -name ".git"

Supondo que você tenha locate, isso deve ser muito mais rápido:

locate .git |grep git$

Se você tiver gnu locate ou mlocate, isso selecionará apenas os diretórios git:

locate -ber \\.git$

Mostra o número total de commits por autor #

Para obter o número total de commits que cada desenvolvedor ou contribuidor fez em um repositório, você pode simplesmente usar o git shortlog:

git shortlog -s

que fornece os nomes dos autores e o número de commits de cada um.

Além disso, se você quiser que os resultados sejam calculados em todas as ramificações, adicione o sinalizador --all ao comando:

git shortlog -s --all
