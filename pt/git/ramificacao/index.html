Sintaxe #

  • git branch [--set-upstream | --track | --no-track] [-l] [-f] <branchname> [<start-point>]
  • git branch (--set-upstream-to=<upstream> | -u <upstream>) [<branchname>]
  • git branch --unset-upstream [<branchname>]
  • git branch (-m | -M) [<oldbranch>] <newbranch>
  • git branch (-d | -D) [-r] <branchname>…​
  • git branch --edit-description [<branchname>]
  • git branch [--color[=<quando>] | --sem cor] [-r | -a] [--list] [-v [--abbrev=<comprimento> | --no-abbrev]] [--column[=<opções>] | --no-column] [(--merged | --no-merged | --contains) [<commit>]] [--sort=<key>] [--points-at <object>] [<pattern >…​]

Parâmetros #

Parâmetro Detalhes
-d, –delete Excluir uma ramificação. O branch deve ser totalmente mesclado em seu branch upstream ou em HEAD se nenhum upstream foi definido com --track ou --set-upstream
-D Atalho para --delete --force
-m, –move Mover/renomear um branch e o reflog correspondente
-M Atalho para --move --force
-r, –remotos Liste ou exclua (se usado com -d) as ramificações de rastreamento remoto
-a, –todos Liste as ramificações de rastreamento remoto e as ramificações locais
–lista Ative o modo de lista. git branch <pattern> tentaria criar um branch, use git branch --list <pattern> para listar os branches correspondentes
–set-upstream Se o branch especificado ainda não existe ou se --force foi dado, age exatamente como --track. Caso contrário, configura a configuração como –track faria ao criar a ramificação, exceto que para onde a ramificação aponta não é alterada

Todo repositório git tem um ou mais branchs. Um branch é uma referência nomeada ao HEAD de uma sequência de commits.

Um repositório git tem um branch current (indicado por um * na lista de nomes de branchs impressa pelo comando git branch), Sempre que você cria um novo commit com o comando git commit, seu novo commit se torna o HEAD do branch atual, e o HEAD anterior se torna o pai do novo commit.

Uma nova ramificação terá o mesmo ‘HEAD’ da ramificação a partir da qual foi criada até que algo seja confirmado na nova ramificação.

Criando e verificando novas ramificações #

Para criar uma nova ramificação, permanecendo na ramificação atual, use:

git branch <name>

Geralmente, o nome da ramificação não deve conter espaços e está sujeito a outras especificações listadas aqui. Para alternar para uma ramificação existente:

git checkout <name>

Para criar um novo branch e mudar para ele:

git checkout -b <name>

Para criar um branch em um ponto diferente do último commit do branch atual (também conhecido como HEAD), use um destes comandos:

git branch <name> [<start-point>]
git checkout -b <name> [<start-point>]

O <start-point> pode ser qualquer revisão conhecida pelo git (por exemplo, outro nome de branch, commit SHA ou uma referência simbólica como HEAD ou um nome da etiqueta):

git checkout -b <name> some_other_branch
git checkout -b <name> af295
git checkout -b <name> HEAD~5
git checkout -b <name> v1.0.5

Para criar uma ramificação a partir de uma ramificação remota (o padrão <nome_remote> é origem):

git branch <name> <remote_name>/<branch_name>
git checkout -b <name> <remote_name>/<branch_name>

Se um determinado nome de ramificação for encontrado apenas em um controle remoto, você pode simplesmente usar

git checkout -b <branch_name>

que é equivalente a

git checkout -b <branch_name> <remote_name>/<branch_name>

Às vezes, você pode precisar mover vários de seus commits recentes para uma nova ramificação. Isso pode ser alcançado ramificando e “revertendo”, assim:

git branch <new_name>
git reset --hard HEAD~2 # Go back 2 commits, you will lose uncommitted work.
git checkout <new_name>

Aqui está uma explicação ilustrativa desta técnica:

 Initial state       After git branch <new_name>    After git reset --hard HEAD~2
                             newBranch                        newBranch
                                 ↓                                ↓
A-B-C-D-E (HEAD)         A-B-C-D-E (HEAD)                 A-B-C-D-E (HEAD)
        ↑                        ↑                            ↑
      master                   master                       master

Listando filiais #

O Git fornece vários comandos para listar branches. Todos os comandos usam a função de git branch, que fornecerá uma lista de determinados ramos, dependendo de quais opções são colocadas na linha de comando. O Git irá, se possível, indicar o branch atualmente selecionado com uma estrela ao lado dele.

Gol Comando
Listar filiais locais git branch
Listar ramificações locais detalhada git branch -v
Listar filiais remotas e locais git branch -a OU git branch --all
Listar ramificações remotas e locais (verbose) git branch -av
Listar ramificações remotas git branch -r
Listar branches remotos com o commit mais recente git branch -rv
Listar filiais mescladas git branch --merged
Listar filiais não mescladas git branch --no-merged
Listar branches contendo commit git branch --contém [<commit>]

Notas:

  • Adicionando um v adicional a -v, por exemplo $ git branch -avv ou $ git branch -vv imprimirá o nome do branch upstream também.
  • As ramificações mostradas em vermelho são ramificações remotas

Excluir uma ramificação remota #

Para excluir uma ramificação no repositório remoto origin, você pode usar para Git versão 1.5.0 e mais recente

git push origin :<branchName>

e a partir do Git versão 1.7.0, você pode excluir um branch remoto usando

git push origin --delete <branchName>

Para excluir uma ramificação de rastreamento remoto local:

git branch --delete --remotes <remote>/<branch>
git branch -dr <remote>/<branch> # Shorter

git fetch <remote> --prune # Delete multiple obsolete tracking branches
git fetch <remote> -p      # Shorter

Para excluir uma ramificação localmente. Observe que isso não excluirá a ramificação se houver alterações não mescladas:

git branch -d <branchName>

Para excluir uma ramificação, mesmo que tenha alterações não mescladas:

git branch -D <branchName>

Troca rápida para a ramificação anterior #

Você pode alternar rapidamente para a ramificação anterior usando

git checkout -

Confira um novo branch rastreando um branch remoto #

Existem três maneiras de criar um novo branch feature que rastreia o branch remoto origin/feature:

  • git checkout --track -b feature origin/feature,
  • git checkout -t origem/recurso,
  • git checkout feature - assumindo que não existe um branch local feature e que existe apenas um remoto com o branch feature.

Para configurar o upstream para rastrear o branch remoto - digite:

  • git branch --set-upstream-to=<remote>/<branch> <branch>
  • git branch -u <remote>/<branch> <branch>

Onde:

  • <remote> pode ser: origin, develop ou aquele criado pelo usuário,
  • <branch> é a ramificação do usuário para rastrear no controle remoto.

Para verificar quais ramificações remotas suas filiais locais estão rastreando:

  • git branch -vv

Excluir uma ramificação localmente #

$ git branch -d dev

Exclui a ramificação chamada dev se suas alterações forem mescladas com outra ramificação e não serão perdidas. Se a ramificação dev contiver alterações que ainda não foram mescladas que seriam perdidas, git branch -d falhará:

$ git branch -d dev
error: The branch 'dev' is not fully merged.
If you are sure you want to delete it, run 'git branch -D dev'.

De acordo com a mensagem de aviso, você pode forçar a exclusão da ramificação (e perder quaisquer alterações não mescladas nessa ramificação) usando o sinalizador -D:

$ git branch -D dev

Crie um branch órfão (ou seja, um branch sem commit pai) #

git checkout --orphan new-orphan-branch

O primeiro commit feito neste novo branch não terá pais e será a raiz de um novo histórico totalmente desconectado de todos os outros branches e commits.

Renomear uma ramificação #

Renomeie o branch que você fez check-out:

git branch -m new_branch_name

Renomeie outro branch:

git branch -m branch_you_want_to_rename new_branch_name

Enviar branch para remoto #

Use para enviar commits feitos em seu branch local para um repositório remoto.

O comando git push recebe dois argumentos:

  • Um nome remoto, por exemplo, origin
  • Um nome de filial, por exemplo, mestre

Por exemplo:

git push  <REMOTENAME> <BRANCHNAME>

Como exemplo, você normalmente executa o git push origin master para enviar suas alterações locais para o seu repositório online.

Usar -u (abreviação de --set-upstream) configurará as informações de rastreamento durante o push.

git push -u <REMOTENAME> <BRANCHNAME>

Por padrão, o git envia o branch local para um branch remoto com o mesmo nome. Por exemplo, se você tiver um local chamado new-feature, se você enviar o branch local, ele também criará um branch remoto new-feature. Se você quiser usar um nome diferente para a ramificação remota, anexe o nome remoto após o nome da ramificação local, separado por ::

git push <REMOTENAME> <LOCALBRANCHNAME>:<REMOTEBRANCHNAME>

Pesquisando nas filiais #

Para listar branches locais que contêm um commit ou tag específico

git branch --contains <commit>

Para listar branches locais e remotos que contêm um commit ou tag específico

git branch -a --contains <commit>

Move o branch atual HEAD para um commit arbitrário #

Um branch é apenas um ponteiro para um commit, então você pode movê-lo livremente. Para fazer com que o branch se refira ao commit aabbcc, emita o comando

git reset --hard aabbcc

Observe que isso substituirá o commit atual do seu branch e, portanto, todo o seu histórico. Você pode perder algum trabalho emitindo este comando. Se for esse o caso, você pode usar o reflog para recuperar os commits perdidos. Pode ser aconselhável executar este comando em uma nova ramificação em vez da atual.

No entanto, esse comando pode ser particularmente útil ao fazer rebase ou fazer outras grandes modificações no histórico.

Sobrescrever um único arquivo no diretório de trabalho atual com o mesmo de outra ramificação #

O arquivo em check-out substituirá as alterações ainda não confirmadas que você fez neste arquivo.

Este comando irá verificar o arquivo file.example (que está localizado no diretório path/to/) e sobrescreverá quaisquer alterações que você possa ter feito neste arquivo.

git checkout some-branch path/to/file

some-branch pode ser qualquer coisa tree-ish conhecida pelo git (veja Revision Selection e gitrevisions para mais detalhes)

Você deve adicionar -- antes do caminho se seu arquivo puder ser confundido com um arquivo (opcional, caso contrário). Nenhuma outra opção pode ser fornecida após o --.

git checkout some-branch -- some-file

O segundo some-file é um arquivo neste exemplo.
