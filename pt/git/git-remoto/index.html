Sintaxe #

  • git remote [-v | --verbose]
  • git remote add [-t <branch>] [-m <master>] [-f] [--[no-]tags] [--mirror=<fetch|push>]<name> <url>
  • git remote renomear <antigo> <novo>
  • git remote remove <name>
  • git remote set-head <name> (-a | --auto | -d | --delete | <branch>)
  • git remote set-branchs [--add] <name> <branch>...
  • git remote set-url [--push] <name> <newurl> [<oldurl>]
  • git remote set-url --add [--push] <name> <newurl>
  • git remote set-url --delete [--push] <name> <url>
  • git remote [-v | --verbose] show [-n] <nome>...
  • git remoção remota [-n | --dry-run] <name>...
  • git remote [-v | --verbose] atualização [-p | --prune] [(<grupo> | <remoto>)...]
  • git remote show <name>

Parâmetros #

Parâmetro Detalhes
-v, –verbose Execute detalhadamente.
-m <mestre> Define head para o branch <master> do remoto
–mirror=buscar As refs não serão armazenadas no namespace refs/remotes, mas serão espelhadas no repositório local
–mirror=push git push se comportará como se –mirror fosse passado
–sem etiquetas git fetch <name> não importa tags do repositório remoto
-t <ramo> Especifica o controle remoto para rastrear somente <ramificação>
-f git fetch <name> é executado imediatamente após a configuração remota
–etiquetas git fetch <name> importa cada tag do repositório remoto
-a, –auto O HEAD da referência simbólica é definido para a mesma ramificação que o HEAD do controle remoto
-d, –delete Todas as referências listadas são excluídas do repositório remoto
–adicionar Adiciona <name> à lista de branches atualmente rastreados (set-braches)
–adicionar Em vez de alterar algum URL, um novo URL é adicionado (set-url)
–todos Empurre todos os ramos.
–delete Todos os URLs correspondentes a <url> são excluídos. (set-url)
–empurre URLs de push são manipulados em vez de URLs de busca
-n As cabeças remotas não são consultadas primeiro com git ls-remote <nome>, as informações em cache são usadas em vez disso
–funcionamento a seco informe quais galhos serão podados, mas não os poda de fato
–podar Remover ramificações remotas que não possuem uma contraparte local

Exibir repositórios remotos #

Para listar todos os repositórios remotos configurados, use git remote.

Ele mostra o nome abreviado (aliases) de cada identificador remoto que você configurou.

$ git remote
premium
premiumPro
origin

Para mostrar informações mais detalhadas, o sinalizador --verbose ou -v pode ser usado. A saída incluirá a URL e o tipo de controle remoto (push ou pull):

$ git remote -v
premiumPro    https://github.com/user/CatClickerPro.git (fetch)
premiumPro    https://github.com/user/CatClickerPro.git (push)
premium    https://github.com/user/CatClicker.git (fetch)
premium    https://github.com/user/CatClicker.git (push)
origin    https://github.com/ud/starter.git (fetch)
origin    https://github.com/ud/starter.git (push)

Altere o URL remoto do seu repositório Git #

Você pode querer fazer isso se o repositório remoto for migrado. O comando para alterar o URL remoto é:

git remote set-url

São necessários 2 argumentos: um nome remoto existente (origem, upstream) e a url.

Verifique seu URL remoto atual:

git remote -v
origin    https://bitbucket.com/develop/myrepo.git (fetch)
origin    https://bitbucket.com/develop/myrepo.git (push)

Altere seu URL remoto:

git remote set-url origin https://localserver/develop/myrepo.git

Verifique novamente seu URL remoto:

git remote -v
origin    https://localserver/develop/myrepo.git (fetch)
origin    https://localserver/develop/myrepo.git (push)

Remover um repositório remoto #

Remova o controle remoto chamado <nome>. Todas as ramificações de rastreamento remoto e definições de configuração para o controle remoto são removidas.

Para remover um repositório remoto dev:

git remote rm dev

Adicionar um repositório remoto #

Para adicionar um controle remoto, use git remote add na raiz do seu repositório local.

Para adicionar um repositório Git remoto <url> como um nome curto fácil <name> use

git remote add <name> <url>

O comando git fetch <name> pode então ser usado para criar e atualizar branches de rastreamento remoto <name>/<branch>.

Mostrar mais informações sobre repositório remoto #

Você pode ver mais informações sobre um repositório remoto por git remote show <remote repository alias>

git remote show origin

resultado:

remote origin
Fetch URL:  https://localserver/develop/myrepo.git
Push  URL:  https://localserver/develop/myrepo.git
HEAD branch: master
Remote branches:
  master      tracked
Local branches configured for 'git pull':
  master      merges with remote master
Local refs configured for 'git push':
  master      pushes to master      (up to date)

Renomear um repositório remoto #

Renomeie o controle remoto chamado <old> para <new>. Todas as ramificações de rastreamento remoto e as definições de configuração do controle remoto são atualizadas.

Para renomear um nome de branch remoto dev para dev1 :

git remote rename dev dev1
