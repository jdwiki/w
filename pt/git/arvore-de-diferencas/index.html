Compara o conteúdo e o modo de blobs encontrados por meio de dois objetos de árvore.

Veja os arquivos alterados em um commit específico #

git diff-tree --no-commit-id --name-only -r COMMIT_ID

Uso #

git diff-tree [--stdin] [-m] [-c] [--cc] [-s] [-v] [--pretty] [-t] [-r] [--root] [<common-diff-options>] <tree-ish> [<tree-ish>] [<path>...]
Opção Explicação
-r diff recursivamente
–raiz inclua o commit inicial como diff contra /dev/null

Opções de diferenças comuns #

Opção Explicação
-z saída diff-raw com linhas terminadas com NUL.
-p formato de patch de saída.
-u sinônimo de -p.
–patch-com-raw produza um patch e o formato diff-raw.
–stat mostre diffstat em vez de patch.
–numstat mostre o diffstat numérico em vez do patch.
–patch-com-stat saída de um patch e prefixe seu diffstat.
–somente nome mostrar apenas os nomes dos arquivos alterados.
–nome-status mostrar nomes e status de arquivos alterados.
–full-index mostre o nome completo do objeto nas linhas de índice.
–abbrev= abreviar nomes de objetos no cabeçalho diff-tree e diff-raw.
-R trocar pares de arquivos de entrada.
-B detectar reescritas completas.
-M detectar renomeações.
-C detectar cópias.
–encontrar-cópias-mais difícil tente arquivos inalterados como candidatos para detecção de cópia.
-l limite tentativas de renomeação até caminhos.
-O reordene as diferenças de acordo com o .
-S encontre o par de arquivos cujo único lado contém a string.
–picareta-tudo mostre todos os arquivos diff quando -S é usado e hit é encontrado.
-a –texto tratar todos os arquivos como texto.
