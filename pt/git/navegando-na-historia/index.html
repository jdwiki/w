Sintaxe #

  • git log [opções] [intervalo de revisão] [[–] caminho…]

Parâmetros #

Parâmetro           Explicação
-q, –quieto Silencioso, suprime a saída do diferencial
–fonte Mostra a fonte do commit
–use-mailmap Use o arquivo de mapa de correio (altera as informações do usuário para confirmar o usuário)
–decorar[=…] Opções de decoração
–L <n,m:arquivo> Mostra o log para um intervalo específico de linhas em um arquivo, contando a partir de 1. Começa na linha n, vai até a linha m. Também mostra dif.
–show-assinatura Exibir assinaturas de commits assinados
-i, –regexp-ignore-case Corresponder aos padrões de limitação de expressão regular sem considerar maiúsculas e minúsculas

Referências e documentação atualizada: git-log documentação oficial

Log mais bonito #

Para ver o log em uma estrutura gráfica mais bonita, use:

git log --decorate --oneline --graph

saída de amostra:

* e0c1cea (HEAD -> maint, tag: v2.9.3, origin/maint) Git 2.9.3
*   9b601ea Merge branch 'jk/difftool-in-subdir' into maint
|\  
| * 32b8c58 difftool: use Git::* functions instead of passing around state
| * 98f917e difftool: avoid $GIT_DIR and $GIT_WORK_TREE
| * 9ec26e7 difftool: fix argument handling in subdirs
* |   f4fd627 Merge branch 'jk/reset-ident-time-per-commit' into maint
...

Como é um comando muito grande, você pode atribuir um alias:

git config --global alias.lol "log --decorate --oneline --graph"

Para usar a versão do alias:

# history of current branch :
git lol

# combined history of active branch (HEAD), develop and origin/master branches :
git lol HEAD develop origin/master

# combined history of everything in your repo :
git lol --all

Registro Git “regular” #

git log

exibirá todos os seus commits com o autor e o hash. Isso será mostrado em várias linhas por confirmação. (Se você deseja mostrar uma única linha por commit, veja onelineing). Use a tecla q para sair do log.

Por padrão, sem argumentos, o git log lista os commits feitos naquele repositório em ordem cronológica inversa – ou seja, o mais recente os commits aparecem primeiro. Como você pode ver, este comando lista cada commit com seu checksum SHA-1, nome e e-mail do autor, data escrito e a mensagem de confirmação. - fonte

Se você deseja limitar seu comando ao último log de n commits, você pode simplesmente passar um parâmetro. Por exemplo, se você deseja listar os últimos 2 logs de commits

git log -2

Registro on-line #

git log --oneline

irá mostrar todos os seus commits com apenas a primeira parte do hash e a mensagem de commit. Cada commit estará em uma única linha, como sugere o sinalizador oneline.

A opção oneline imprime cada commit em uma única linha, o que é útil se você estiver vendo muitos commits. - fonte

Se você deseja limitar seu comando ao último log de n commits, você pode simplesmente passar um parâmetro. Por exemplo, se você deseja listar os últimos 2 logs de commits

git log -2 --oneline

Colorir Logs #

git log --graph --pretty=format:'%C(red)%h%Creset -%C(yellow)%d%Creset %s %C(green)(%cr) %C(yellow)<%an>%Creset'

A opção format permite que você especifique seu próprio formato de saída de log:

Parâmetro Detalhes
%C(color_name) opção colore a saída que vem depois
%h ou %H abrevia commit hash (use %H para hash completo)
%Creset redefine a cor para a cor padrão do terminal
%d nomes de referência
%s assunto [enviar mensagem]
%cr data do committer, relativa à data atual
%an nome do autor

Pesquisa de registros #

git log -S"#define SAMPLES"

Pesquisa por adição ou remoção de uma string específica ou a string correspondente fornecida REGEXP. Neste caso estamos procurando a adição/remoção da string #define SAMPLES. Por exemplo:

+#define SAMPLES  100000

ou

-#define SAMPLES  100000

git log -G"#define SAMPLES"

Pesquisa por alterações em linhas contendo uma string específica ou a string correspondente fornecida REGEXP. Por exemplo:

-#define SAMPLES  100000
+#define SAMPLES  100000000

Lista todas as contribuições agrupadas por nome de autor #

git shortlog resume o git log e agrupa por autor

Se nenhum parâmetro for fornecido, uma lista de todos os commits feitos por committer será mostrada em ordem cronológica.

$ git shortlog
Committer 1 (<number_of_commits>):
    Commit Message 1
    Commit Message 2
    ...
Committer 2 (<number_of_commits>):
    Commit Message 1
    Commit Message 2
    ...

Para simplesmente ver o número de commits e suprimir a descrição do commit, passe a opção de resumo:

-s

--resumo

$ git shortlog -s
<number_of_commits> Committer 1
<number_of_commits> Committer 2

Para classificar a saída por número de commits em vez de alfabeticamente pelo nome do committer, passe a opção numerada:

-n

--numerado

Para adicionar o e-mail de um committer, adicione a opção de e-mail:

-e

--email

Uma opção de formato personalizado também pode ser fornecida se você quiser exibir informações além do assunto do commit:

--formato

Isso pode ser qualquer string aceita pela opção --format do git log.

Consulte Colorizing Logs acima para obter mais informações sobre isso.

1: https://www.wikiod.com/pt/git/navegando-na-historia#Colorir registros

Pesquisando a string de commit no git log #

Pesquisando git log usando alguma string no log:

git log [options] --grep "search_string"

Exemplo:

git log --all --grep "removed file"

Procurará a string arquivo removido em todos os logs em todos os ramos.

A partir do git 2.4+, a busca pode ser invertida usando a opção --invert-grep.

Exemplo:

git log --grep="add file" --invert-grep

Mostrará todos os commits que não contêm add file.

Filtrar registros #

git log --after '3 days ago'

Datas específicas também funcionam:

git log --after 2016-05-01

Assim como outros comandos e sinalizadores que aceitam um parâmetro de data, o formato de data permitido é suportado pela data GNU (altamente flexível).

Um alias para --after é --since.

Existem sinalizadores para o inverso também: --before e --until.

Você também pode filtrar logs por autor. por exemplo.

git log --author=author

git log --patch
