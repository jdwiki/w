Sintaxe #

  • git am [–signoff] [–keep] [–[no-]keep-cr] [–[no-]utf8] [–3way] [–interactive] [–committer-date-is-author-date] [–ignore-date] [–ignore-space-change | –ignore-whitespace] [–whitespace=<option>] [-C<n>] [-p<n>] [–directory=<dir>] [–exclude=<path>] [–include=<path>] [–reject] [-q | –quiet] [–[no-]scissors] [-S[<keyid>]] [–patch-format=<format>] [(<mbox> | <Maildir>)…]
  • git am (–continue | –skip | –abort)

Parâmetros #

Parâmetro Detalhes
(<mbox>|<Maildir>)… A lista de arquivos de caixa de correio para ler os patches. Se você não fornecer esse argumento, o comando lerá a partir da entrada padrão. Se você fornecer diretórios, eles serão tratados como Maildirs.
-s, –signoff Adicione uma linha Signed-off-by: à mensagem de commit, usando a sua identidade de commiter.
-q, –quieto Fique quieto. Imprima apenas mensagens de erro.
-u, –utf8 Passe o sinalizador -u para git mailinfo. A mensagem de log de confirmação proposta retirada do e-mail é recodificada na codificação UTF-8 (a variável de configuração i18n.commitencoding pode ser usada para especificar a codificação preferida do projeto se não for UTF-8). Você pode usar --no-utf8 para substituir isso.
–no-utf8 Passe -n sinalizador para git mailinfo.
-3, –3way Quando o patch não se aplica corretamente, volte para a mesclagem de 3 vias se o patch registrar a identidade dos blobs aos quais ele deve se aplicar e tivermos esses blobs disponíveis localmente.
–ignore-date, –ignore-space-change, –ignore-whitespace, –whitespace=<option>, -C<n>, -p<n>, –directory=<dir >, –exclude=<caminho>, –include=<caminho>, –reject Esses sinalizadores são passados ​​para o programa git apply que aplica o patch.
–patch-formato Por padrão, o comando tentará detectar o formato do patch automaticamente. Esta opção permite que o usuário ignore a detecção automática e especifique o formato do patch como o(s) patch(es) deve(m) ser interpretado(s). Os formatos válidos são mbox, stgit, stgit-series e hg.
-i, –interativo Execute interativamente.
–data-committer-é-autor-data Por padrão, o comando registra a data da mensagem de e-mail como a data do autor do commit e usa a hora da criação do commit como a data do committer. Isso permite que o usuário minta sobre a data do commiter usando o mesmo valor que a data do autor.
–ignore-data Por padrão, o comando registra a data da mensagem de e-mail como a data do autor do commit e usa a hora da criação do commit como a data do committer. Isso permite que o usuário minta sobre a data do autor usando o mesmo valor da data do committer.
–pular Pule o patch atual. Isso só é significativo ao reiniciar um patch abortado.
-S[<keyid>], –gpg-sign[=<keyid>] Compromissos de assinatura GPG.
–continua, -r, –resolvido Após uma falha de patch (por exemplo, tentativa de aplicar patch conflitante), o usuário o aplicou manualmente e o arquivo de índice armazena o resultado da aplicação. Faça um commit usando a autoria e o log de commit extraídos da mensagem de e-mail e do arquivo de índice atual e continue.
–resolvemsg=<msg> Quando ocorre uma falha de patch, <msg> será impresso na tela antes de sair. Isso substitui a mensagem padrão informando que você deve usar --continue ou --skip para lidar com a falha. Isto é somente para uso interno entre git rebase e ​​git am.
–abortar Restaure a ramificação original e aborte a operação de correção.

Criando um patch #

Para criar um patch, há duas etapas.

  1. Faça suas alterações e confirme-as.
  2. Execute git format-patch <commit-reference> para converter todos os commits desde o commit <commit-reference> (não incluindo) em arquivos de patch.

Por exemplo, se os patches devem ser gerados a partir dos dois últimos commits:

git format-patch HEAD~~

Isso criará 2 arquivos, um para cada commit desde HEAD~~, assim:

0001-hello_world.patch
0002-beginning.patch

Aplicando patches #

Podemos usar git apply some.patch para que as alterações do arquivo .patch sejam aplicadas ao seu diretório de trabalho atual. Eles não serão encenados e precisam ser comprometidos.

Para aplicar um patch como um commit (com sua mensagem de commit), use

git am some.patch

Para aplicar todos os arquivos de patch à árvore:

git am *.patch
