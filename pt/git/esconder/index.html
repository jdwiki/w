Sintaxe #

  • git stash list [<options>]
  • git stash show [<stash>]
  • git stash drop [-q|--quiet] [<stash>]
  • git stash ( pop | apply ) [--index] [-q|--quiet] [<stash>]
  • git stash branch <branchname> [<stash>]
  • git stash [salvar [-p|--patch] [-k|--[no-]keep-index] [-q|--quiet] [-u|--include-untracked] [-a| --all] [<mensagem>]]
  • git stash clear
  • git stash create [<message>]
  • git stash store [-m|--message <message>] [-q|--quiet] <commit>

Parâmetros #

Parâmetro Detalhes
mostrar Mostre as alterações registradas no stash como uma diferença entre o estado stashed e seu pai original. Quando nenhum <stash> é fornecido, mostra o mais recente.
lista Liste os stashes que você tem atualmente. Cada stash é listado com seu nome (por exemplo, stash@{0} é o stash mais recente, stash@{1} é o anterior, etc.), o nome do branch atual quando o stash foi criado e um pequeno descrição do commit no qual o stash foi baseado.
pop Remova um único estado stash da lista stash e aplique-o no topo do estado atual da árvore de trabalho.
apply Como pop, mas não remova o estado da lista de stash.
claro Remova todos os estados stashed. Observe que esses estados estarão sujeitos a poda e podem ser impossíveis de recuperar.
soltar Remova um único estado stash da lista stash. Quando nenhum <stash> é fornecido, ele remove o mais recente. ou seja, stash@{0}, caso contrário, <stash> deve ser uma referência de log de stash válida no formato stash@{<revision>}.
criar Crie um stash (que é um objeto de confirmação regular) e retorne seu nome de objeto, sem armazená-lo em qualquer lugar no namespace ref. Isso se destina a ser útil para scripts. Provavelmente não é o comando que você deseja usar; veja “salvar” acima.
loja Armazene um determinado stash criado via git stash create (que é um commit de merge pendente) na stash ref, atualizando o stash reflog. Isso se destina a ser útil para scripts. Provavelmente não é o comando que você deseja usar; veja “salvar” acima.

Stashing nos permite ter um diretório de trabalho limpo sem perder nenhuma informação. Então, é possível começar a trabalhar em algo diferente e/ou mudar de ramo.

Criar esconderijo #

Salve o estado atual do diretório de trabalho e o índice (também conhecido como área de preparo) em uma pilha de stashes.

git stash

Para incluir todos os arquivos não rastreados no stash, use os sinalizadores --include-untracked ou -u.

git stash --include-untracked

Para incluir uma mensagem com seu stash para torná-la mais facilmente identificável posteriormente

git stash save "<whatever message>"

Para deixar a área de teste no estado atual após o stash, use os sinalizadores --keep-index ou -k.

git stash --keep-index

Aplicar e remover stash #

Para aplicar o último stash e removê-lo da pilha - digite:

git stash pop

Para aplicar um stash específico e removê-lo da pilha - digite:

git stash pop stash@{n}

Aplicar stash sem removê-lo #

Aplica o último stash sem removê-lo da pilha

git stash apply

Ou um estoque específico

git stash apply stash@{n}

Mostrar estoque #

Mostra as alterações salvas no último stash

git stash show

Ou um estoque específico

git stash show stash@{n}

Para mostrar o conteúdo das alterações salvas para o stash específico

git stash show -p stash@{n}

O que é Stashing? #

Ao trabalhar em um projeto, você pode estar no meio de uma mudança de ramificação de recurso quando um bug é levantado contra o mestre. Você não está pronto para confirmar seu código, mas também não quer perder suas alterações. É aqui que o git stash vem a calhar.

Execute git status em uma ramificação para mostrar suas alterações não confirmadas:

(master) $ git status
On branch master
Your branch is up-to-date with 'origin/master'.
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

    modified:   business/com/test/core/actions/Photo.c

no changes added to commit (use "git add" and/or "git commit -a")

Em seguida, execute git stash para salvar essas alterações em uma pilha:

(master) $ git stash
Saved working directory and index state WIP on master: 
2f2a6e1 Merge pull request #1 from test/test-branch
HEAD is now at 2f2a6e1 Merge pull request #1 from test/test-branch

Se você adicionou arquivos ao seu diretório de trabalho, eles também podem ser armazenados. Você só precisa encenar-los primeiro.

(master) $ git stash
Saved working directory and index state WIP on master:
(master) $ git status
On branch master
Untracked files:
  (use "git add <file>..." to include in what will be committed)

        NewPhoto.c

nothing added to commit but untracked files present (use "git add" to track)
(master) $ git stage NewPhoto.c
(master) $ git stash
Saved working directory and index state WIP on master:
(master) $ git status
On branch master
nothing to commit, working tree clean
(master) $

Seu diretório de trabalho agora está limpo de todas as alterações que você fez. Você pode ver isso executando novamente o git status:

(master) $ git status
On branch master
Your branch is up-to-date with 'origin/master'.
nothing to commit, working directory clean

Para aplicar o último stash, execute git stash apply (além disso, você pode aplicar e remover o último stash alterado com git stash pop):

(master) $ git stash apply
On branch master
Your branch is up-to-date with 'origin/master'.
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

    modified:   business/com/test/core/actions/Photo.c

no changes added to commit (use "git add" and/or "git commit -a")

Observe, no entanto, que o stashing não lembra a ramificação em que você estava trabalhando. Nos exemplos acima, o usuário estava stashing em master. Se eles mudarem para o branch dev, dev, e executarem git stash apply, o último stash será colocado no branch dev.

(master) $ git checkout -b dev
Switched to a new branch 'dev'
(dev) $ git stash apply
On branch dev
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

    modified:   business/com/test/core/actions/Photo.c

no changes added to commit (use "git add" and/or "git commit -a")

Estoque parcial #

Se você quiser armazenar apenas algumas diferenças em seu conjunto de trabalho, você pode usar um armazenamento parcial.

git stash -p

E, em seguida, selecione interativamente quais pedaços armazenar.

A partir da versão 2.13.0, você também pode evitar o modo interativo e criar um stash parcial com um pathspec usando a nova palavra-chave push.

git stash push -m "My partial stash" -- app.config

Listar stashes salvos #

git stash list

Isso listará todos os stashes na pilha em ordem cronológica inversa. Você receberá uma lista parecida com esta:

stash@{0}: WIP on master: 67a4e01 Merge tests into develop
stash@{1}: WIP on master: 70f0d95 Add user role to localStorage on user login

Você pode se referir a um stash específico pelo nome, por exemplo stash@{1}.

Mova seu trabalho em andamento para outro branch #

Se durante o trabalho você perceber que está no branch errado e ainda não criou nenhum commit, você pode facilmente mover seu trabalho para o branch correto usando o stashing:

git stash
git checkout correct-branch
git stash pop

Lembre-se de que git stash pop aplicará o último stash e o excluirá da lista de stash. Para manter o stash na lista e aplicar apenas em algum branch você pode usar:

git stash apply

Remover esconderijo #

Remover todo o estoque

git stash clear

Remove o último stash

git stash drop

Ou um estoque específico

git stash drop stash@{n}

Aplicar parte de um stash com checkout #

Você fez um stash e deseja fazer check-out de apenas alguns dos arquivos nesse stash.

git checkout stash@{0} -- myfile.txt

Stashing interativo #

Recupere um stash descartado #

Se você acabou de abri-lo e o terminal ainda está aberto, você ainda terá o valor de hash impresso por git stash pop na tela:

$ git stash pop
[...]
Dropped refs/stash@{0} (2ca03e22256be97f9e40f08e6d6773c7d41dbfd1)

(Observe que git stash drop também produz a mesma linha.)

Caso contrário, você pode encontrá-lo usando isso:

git fsck --no-reflog | awk '/dangling commit/ {print $3}'

Isso mostrará a você todos os commits nas pontas do seu gráfico de commit que não são mais referenciados de qualquer branch ou tag – cada commit perdido, incluindo todo commit stash que você já criou, estará em algum lugar nesse gráfico.

A maneira mais fácil de encontrar o commit do stash que você deseja é provavelmente passar essa lista para o gitk:

gitk --all $( git fsck --no-reflog | awk '/dangling commit/ {print $3}' )

Isso iniciará um navegador de repositório mostrando a você todos os commits no repositório de todos os tempos, independentemente de estar acessível ou não.

Você pode substituir o gitk por algo como git log --graph --oneline --decorate se preferir um bom gráfico no console em vez de um aplicativo GUI separado.

Para identificar commits do stash, procure por mensagens de commit deste formulário:

        WIP em algum branch: commithash Alguma mensagem antiga de commit

Depois de saber o hash do commit que você deseja, você pode aplicá-lo como um stash:

git stash apply $stash_hash

Ou você pode usar o menu de contexto no gitk para criar ramificações para quaisquer commits inacessíveis que você estiver interessado. Depois disso, você pode fazer o que quiser com eles com todas as ferramentas normais. Quando terminar, apenas exploda esses galhos novamente.
