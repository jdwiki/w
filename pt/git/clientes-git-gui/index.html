gitk e git-gui #

Quando você instala o Git, você também obtém suas ferramentas visuais, gitk e git-gui.

gitk é um visualizador de histórico gráfico. Pense nisso como um poderoso shell de GUI sobre git log e git grep. Esta é a ferramenta a ser usada quando você está tentando encontrar algo que aconteceu no passado ou visualizar o histórico do seu projeto.

Gitk é mais fácil de invocar a partir da linha de comando. Apenas cd em um repositório Git e digite:

$ gitk [opções de log do git]

Gitk aceita muitas opções de linha de comando, a maioria das quais são passadas para a ação subjacente do git log. Provavelmente um dos mais úteis é o sinalizador --all, que diz ao gitk para mostrar commits alcançáveis ​​de qualquer referência, não apenas HEAD. A interface do Gitk se parece com isso:

Figura 1-1. O visualizador de histórico do gitk.

No topo está algo que se parece um pouco com a saída de git log –graph; cada ponto representa um commit, as linhas representam relacionamentos pai e as referências são mostradas como caixas coloridas. O ponto amarelo representa HEAD e o ponto vermelho representa as alterações que ainda não se tornaram um commit. Na parte inferior está uma visão do commit selecionado; os comentários e o patch à esquerda e uma visualização de resumo à direita. No meio há uma coleção de controles usados ​​para pesquisar o histórico.

Você pode acessar muitas funções relacionadas ao git clicando com o botão direito do mouse em um nome de branch ou em uma mensagem de commit. Por exemplo, verificar uma ramificação diferente ou escolher um commit é feito facilmente com um clique.

git-gui, por outro lado, é principalmente uma ferramenta para criar commits. Também é mais fácil invocar a partir da linha de comando:

$ git gui

E fica mais ou menos assim:

A ferramenta de submissão git-gui.

Figura 1-2. A ferramenta de confirmação git-gui.

À esquerda está o índice; as alterações não encenadas estão na parte superior, as alterações encenadas na parte inferior. Você pode mover arquivos inteiros entre os dois estados clicando em seus ícones ou selecionar um arquivo para visualização clicando em seu nome.

No canto superior direito está a visualização diff, que mostra as alterações para o arquivo selecionado no momento. Você pode encenar pedaços individuais (ou linhas individuais) clicando com o botão direito do mouse nesta área.

No canto inferior direito está a área de mensagem e ação. Digite sua mensagem na caixa de texto e clique em “Commit” para fazer algo semelhante ao git commit. Você também pode optar por alterar o último commit escolhendo o botão de opção “Amend”, que atualizará a área “Staged Changes” com o conteúdo do último commit. Então você pode simplesmente preparar ou desfazer algumas alterações, alterar a mensagem do commit e clicar em “Commit” novamente para substituir o commit antigo por um novo.

gitk e git-gui são exemplos de ferramentas orientadas a tarefas. Cada um deles é adaptado para um propósito específico (visualização do histórico e criação de commits, respectivamente), e omite os recursos não necessários para essa tarefa.

Fonte: https://git-scm.com/book/en/v2/Git-in-Other-Environments-Graphical-Interfaces
