Introdução ao Access SQL #

Ao usar o Access, você pode recuperar dados usando consultas. Essas consultas são criadas usando a Linguagem de Consulta Estruturada (SQL). Compreender o SQL é importante porque pode ajudar a criar consultas melhores e mais úteis.

Consultas de União (Mesclar) #

Quando você deseja combinar os resultados de várias tabelas ou consultas com campos semelhantes em um único conjunto de dados resultante sem realizar nenhuma junção relacional (ou seja, você deseja listar um conjunto de dados imediatamente após o outro), você usará uma consulta UNION. No entanto, é notável que essas consultas devem ser criadas manualmente no SQL View.

A sintaxe de uma consulta UNION é

SELECT
  floatingpoint_field AS floatptfld,
  text_field
FROM first_table
  UNION
SELECT
  integer_field,
  decimal_field
FROM a_saved_query
  UNION
SELECT
  1.0,
  "hi there Jack"

e retornará um conjunto de dados de dois campos com nomes de campo (coluna): floatptfld e text_field

É fundamental que os tipos de dados (e estilos de dados) para campos de tabelas mescladas subsequentemente sejam compatíveis com a primeira consulta da série. Em outras palavras, se a primeira consulta SELECT gerar um número para a primeira coluna, a segunda consulta também deve retornar um número na primeira coluna. Além de combinar os tipos de campos em ordem, as instruções SELECT devem retornar o mesmo número de campos. Os nomes dos campos da folha de dados resultante são herdados da primeira definição de tabela.

A consulta a seguir NÃO seria legal, pois o texto não pode ser transformado em dados decimais nem os números de ponto flutuante podem ser convertidos em inteiros (sem truncamento ou arredondamento explícitos e conversão de tipo).

SELECT
  integer_field AS this_really_wont_turn_out_well,
  decimal_field
FROM a_saved_query
  UNION
SELECT
  floatingpoint_field,
  text_field
FROM first_table

A função COUNT() #

Você pode usar a função COUNT() para retornar o número de registros que correspondem a uma consulta. A tabela ‘Employee’ a seguir contém os números de identificação do funcionário e o número de identificação do gerente associado.

Employee_ID Manager_ID
12 37
22 37
37 63
42 45
45 63
57 45
59 45
63

Uma instrução COUNT() pode ser usada para descobrir quantos funcionários têm um gerente específico:

SELECT COUNT(*) AS CNT FROM Employees WHERE Employee.Manager_ID = 37;

retorna

CNT
2

A função também pode ser combinada em consultas mais complicadas. Para descobrir quantos funcionários são supervisionados diretamente por uma pessoa específica, pode-se aplicar o seguinte:

SELECT T1.Employee_ID,
    (SELECT COUNT(*) AS CNT FROM Employees AS T2 WHERE T2.Manager_ID =
        T1.Employee_ID) AS Supervised_Count
FROM Employees AS T1;

retorna:

Employee_ID Supervisionado_Contagem
12 0
22 0
37 2
42 0
45 3
57 0
59 0
63 2
