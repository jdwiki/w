Listar alvos, esquemas e configurações de compilação disponíveis #

Para listar todos os esquemas disponíveis para o projeto em seu diretório atual

xcodebuild -list

Opcionalmente, você pode passar um caminho para um projeto ou arquivo de espaço de trabalho

xcodebuild -list -workspace ./MyApp.xcworkspace
xcodebuild -list -project ./MyApp.xcodeproj

Exemplo de saída

    Targets:
        Themoji
        ThemojiUITests
        Unit

    Build Configurations:
        Debug
        Release

Executando testes #

Para executar seus testes de unidade no simulador usando xcodebuild use

Se você tiver um espaço de trabalho (por exemplo, ao usar CocoaPods)

xcodebuild \
  -workspace MyApp.xcworkspace \
  -scheme "MyScheme" \
  -sdk iphonesimulator \
  -destination 'platform=iOS Simulator,name=iPhone 6,OS=9.1' \
  test

Se você tiver um arquivo de projeto

xcodebuild \
  -project MyApp.xcproj \
  -scheme "MyScheme" \
  -sdk iphonesimulator \
  -destination 'platform=iOS Simulator,name=iPhone 6,OS=9.1' \
  test

Os valores alternativos de “destino” são

  -destination 'platform=iOS,id=REAL_DEVICE_UDID'
  -destination 'platform=iOS,name=IPHONE NAME'

Compilar e assinar esquema #

Limpeza e compilação de código para iPhone, no projeto MyProject para schema Qa:

xcrun xcodebuild clean \
    -workspace "MyProject.xcworkspace" \
    -scheme "YourScheme" \
    -sdk iphoneos \
    -configuration Debug \
    archive \
    -archivePath builds/MyProject.xcarchive

A configuração pode ser Debug ou Release.

Assinando o código compilado anteriormente:

xcrun xcodebuild -exportArchive \
    -archivePath builds/MyProject-Qa.xcarchive \
    -exportOptionsPlist config.plist \
    -exportPath builds

config.plist contém as informações sobre como empacotar e assinar o aplicativo, para uso de builds de desenvolvimento:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
        <key>method</key>
        <string>development</string>
        <key>uploadSymbols</key>
        <true/>
</dict>
</plist>

Uma plist de lançamento da App Store deve conter algo como:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
        <key>teamID</key>
        <string>xxxxxxxxxxx</string>
        <key>method</key>
        <string>app-store</string>
        <key>uploadSymbols</key>
        <true/>
</dict>
</plist>

Onde o ID da equipe pode ser obtido no seu chaveiro.

Todos os parâmetros disponíveis

  • compileBitcode
  • embedOnDemandResourcesAssetPacksInBundle
  • iCloudContainerEnvironment
  • manifesto
  • método
  • onDemandResourcesAssetPacksBaseURL
  • teamID
  • desbaste
  • uploadBitcode
  • uploadSymbols

Para obter mais informações sobre cada um dos parâmetros execute xcodebuild --help

Acesse qualquer ferramenta de linha de comando no pacote de aplicativos Xcode (xcrun) #

xcrun usa a versão Xcode padrão do sistema (definida via xcode-select) para localizar e executar ferramentas de linha de comando do pacote de aplicativos Xcode, por exemplo, llvm-cov.

# Generate code coverage reports via llvm-cov 
# /Applications/Xcode.app/Contents/Developer/Toolchains/XcodeDefault.xctoolchain/usr/bin
xcrun llvm-cov [parameters]

# Execute xcodebuild 
# /Applications/Xcode.app/Contents/Developer/usr/bin
xcrun xcodebuild [parameters]

# Use Xcode's version of git, e.g., if you have installed a newer version
# /Applications/Xcode.app/Contents/Developer/usr/bin 
xcrun git [parameters]

Alternando ferramentas de linha de comando com xcode-select #

Imprima o caminho para o diretório do desenvolvedor ativo (Xcode selecionado)

xcode-select -p

Selecione uma versão diferente do Xcode, por exemplo Beta

sudo xcode-select -s /Applications/Xcode-beta.app

Redefinir para a versão padrão do Xcode

sudo xcode-select -r

Isso é equivalente a executar sudo xcode-select -s /Applications/Xcode.app

Para mais detalhes: man xcode-select
