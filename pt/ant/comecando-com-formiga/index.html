Bootstrap Apache Ivy #

Olá Mundo #

Adicione o seguinte a um arquivo chamado build.xml no diretório do seu projeto:

<?xml version="1.0" encoding="UTF-8"?>
<project name="HelloWorld" default="main">
    <target name="main" description="this is target main">
        <echo message="Hello World" />
    </target>
</project>

A partir de um prompt de comando em um computador com Windows, a execução de ant main será exibida de forma semelhante à seguinte:

$ ant main
Buildfile: C:\Users\<me>\Projects\HelloWorld\build.xml

main:
     [echo] Hello World

BUILD SUCCESSFUL

Além disso, o usuário agora pode executar o comando ant como nome de destino default adicionado ao projeto. Quando o comando ant é executado, ele procura o destino default do projeto e o executa.

$ ant 
Buildfile: C:\Users\<me>\Projects\HelloWorld\build.xml

main:
     [echo] Hello World

BUILD SUCCESSFUL

Se o script de compilação for escrito por outra pessoa e o usuário final quiser ver qual destino ele pode executar, execute o comando que mostrará todos os destinos que possuem descrições.

$ ant -p

Criar pacote jar #

O seguinte irá criar dist/output.jar a partir do código fonte em src e as bibliotecas em lib, e usará src/Main.java como a classe principal.

<project name="Project" default="main" basedir=".">

    <property name="src.dir"     value="src"/>
    <property name="build.dir"   value="build"/>
    <property name="dist.dir"    value="dist"/>

    <path id="classpath">
        <fileset dir="lib" includes="**/*.jar"/>
        <pathelement path="${build.dir}"/>
    </path>

    <target name="clean">
      <delete dir="${build.dir}"/>
      <delete dir="${dist.dir}"/>
    </target>
    

    <target name="compile">
        <mkdir dir="${build.dir}"/>
        <javac srcdir="${src.dir}" destdir="${build.dir}" classpathref="classpath"/>
        <copy todir="${build.dir}">
            <fileset dir="${src.dir}" excludes="**/*.java"/>
        </copy>
    </target>

    <target name="jar" depends="compile">
        <mkdir dir="${dist.dir}"/>
        <jar destfile="${dist.dir}/${ant.project.name}.jar" basedir="${build.dir}">
            <fileset dir="${build.dir}"/>
            <restrict>
                <archives>
                    <zips>
                        <fileset dir="lib" includes="**/*.jar"/>
                    </zips>
                </archives>
            </restrict>
            <manifest>
                <attribute name="Main-Class" value="Main"/>
            </manifest>
        </jar>
    </target>

    <target name="main" depends="clean,jar"/>
</project>

Instalação ou Configuração #

Instalar o Ant é muito simples. Siga as etapas abaixo para instalar o Ant na plataforma Windows:

  1. Baixe a versão mais recente do ant do site do Apache

  2. Descompacte o arquivo em sua máquina.

  3. Defina ANT_HOME nas variáveis ​​de ambiente

  4. Adicione %ANT_HOME%\bin à sua variável de ambiente PATH.

  5. Defina CLASSPATH=%ANT_HOME%\lib;%CLASSPATH%

  6. Agora abra o prompt de comando e digite o comando ant. Você deve ver abaixo:

    Buildfile: build.xml does not exist!
 Build failed

Como alternativa, usando o Homebrew no macOS ou Linuxbrew no Linux, você pode simplesmente executar: brew install form

Ao usar brew não é necessário configurar as variáveis ​​de ambiente.

Várias distribuições Linux também suportam a instalação do Ant a partir de seus respectivos gerenciadores de pacotes.

Para testar se o Ant está instalado corretamente, navegue até o prompt de comando e execute

ant -version

Este comando imprimirá a versão do Ant e também mostrará que o Ant foi instalado com sucesso.

A página de instruções de instalação do próprio Ant está disponível no site do Apache Ant.
