Instalação ou Configuração #

Descrição:

phpMyAdmin é uma ferramenta de software livre escrita em PHP, destinada a lidar com a administração do MySQL pela Web. O phpMyAdmin suporta uma ampla variedade de operações no MySQL e no MariaDB. As operações usadas com frequência (gerenciamento de bancos de dados, tabelas, colunas, relações, índices, usuários, permissões, etc.) podem ser realizadas através da interface do usuário, enquanto você ainda tem a capacidade de executar diretamente qualquer instrução SQL.

Nota 1: Muitos sistemas operacionais já incluem um pacote phpMyAdmin e o manterão atualizado automaticamente, no entanto, essas versões às vezes estão um pouco desatualizadas e, portanto, podem estar faltando os recursos mais recentes.

Nota 2: Este método assume que você já configurou um servidor web local com MySQL.

Janelas:

  • Produtos de terceiros:

    The easiest way to get phpMyAdmin on Windows is using third party products which include phpMyAdmin together with a database and web server such as XAMPP(cross-platform).

  • Lista de produtos de terceiros multiplataforma: XAMPP, AMPPS, Zend Server Community Edition

  • Lista de produtos de terceiros específicos do Windows: WampServer, EasyPHP Devserver, Uniform Server, Bitnami WAMP Stack, WPN-XM Server Stack, WTServer, UwAmp, Vertrigo

  • Instalação Manual phpadmin.net:

    You can select any kit from given phpmyadmin kits.

    Example:

    For all languages choose some thing like phpMyAdmin-4.6.4-all-languages.7z or for language specific choose something like phpMyAdmin-4.6.4-english.7z

    After Choosing a phpmyadmin kit extract the folder. Rename the extracted folder to phpMyAdmin it’s easy to reference. Copy the extracted folder to your local web server’s document root. Depending on how you have configured your web server, this could be the www, htdocs, html, or public_html folder. Se você estiver usando o IIS, abra o Gerenciador do IIS. Na árvore de navegação à esquerda, clique com o botão direito em “Default Web Site” e selecione “Explore”. Copie e cole a pasta phpMyAdmin extraída no local que é aberto.

Distribuições Linux

Nota: phpMyAdmin está incluído na maioria das distribuições Linux. Recomenda-se usar pacotes de distribuição quando possível. Eles geralmente fornecem integração à sua distribuição e você receberá automaticamente atualizações de segurança da sua distribuição.

  • Produtos de terceiros: XAMPP, TurnKey Linux LAMP stack, Bitnami LAMPStack,

  • Instalação Manual: phpadmin.net:

Você pode selecionar qualquer kit dos kits phpmyadmin fornecidos.

Exemplo:

Para todos os idiomas, escolha algo como phpMyAdmin-4.6.4-all-languages.7z ou para idiomas específicos, escolha algo como phpMyAdmin-4.6.4-english.7z

Depois de escolher um kit phpmyadmin extraia a pasta. Renomeie a pasta extraída para phpMyAdmin, é fácil fazer referência. Copie a pasta extraída para a raiz de documentos do seu servidor web local. Dependendo de como você configurou seu servidor web, pode ser a pasta htdocs ou public_html.

  • Debian:

    Debian’s package repositories include a phpMyAdmin package, but be aware that the configuration file is maintained in /etc/phpmyadmin and may differ in some ways from the official phpMyAdmin documentation. Specifically it does:

    1. Configuration of web server (works for Apache and lighttpd).
    2. Creating of phpMyAdmin configuration storage using dbconfig-common.
    3. Securing setup script, see Setup script on Debian, Ubuntu and derivatives.

  • Terminal: A maneira mais fácil de instalar o phpmyadmin no Debian é através do apt-get

      sudo apt-get install phpmyadmin

Durante a instalação, o phpMyAdmin o guiará por uma configuração básica. Depois o processo começa siga estes passos:

  1. Selecione Apache2 para o servidor

  2. Escolha YES quando perguntado sobre configurar o banco de dados para phpmyadmin com dbconfig-common

  3. Digite sua senha do MySQL quando solicitado

  4. Digite a senha que você deseja usar para fazer login no phpmyadmin

Após a conclusão da instalação, adicione phpmyadmin à configuração do apache.

sudo nano /etc/apache2/apache2.conf

Adicione a configuração do phpmyadmin ao arquivo.

Include /etc/phpmyadmin/apache.conf

Reinicie o apache:

sudo service apache2 restart

Você pode então acessar o phpmyadmin indo para your-ip-address/phpmyadmin.

Segurança:

Infelizmente, as versões mais antigas do phpMyAdmin tiveram sérias vulnerabilidades de segurança, incluindo permitir que usuários remotos explorassem o root no servidor privado virtual subjacente. Pode-se evitar a maioria desses ataques através de um processo simples: bloquear todo o diretório com as restrições nativas de usuário/senha do Apache, o que impedirá que esses usuários remotos tentem explorar versões mais antigas do phpMyAdmin.

Configure o arquivo .htaccess:

Para configurar isso, comece permitindo que o arquivo .htaccess funcione dentro do diretório phpmyadmin. Você pode fazer isso no arquivo de configuração do phpmyadmin:

sudo nano /etc/phpmyadmin/apache.conf

Na seção do diretório, adicione a linha “AllowOverride All” em “Directory Index”, fazendo com que a seção fique assim:

<Directory /usr/share/phpmyadmin>
        Options FollowSymLinks
        DirectoryIndex index.php
        AllowOverride All
        [...]

Configure o arquivo .htaccess

Com o arquivo .htaccess permitido, podemos prosseguir para configurar um usuário nativo cujo login seria necessário até mesmo para acessar a página de login do phpmyadmin.

Comece criando a página .htaccess no diretório phpmyadmin:

sudo nano /usr/share/phpmyadmin/.htaccess

Faça o acompanhamento configurando a autorização do usuário no arquivo .htaccess. Copie e cole o seguinte texto em

AuthType Basic
AuthName "Restricted Files"
AuthUserFile /path/to/passwords/.htpasswd
Require valid-user

AuthType: Refere-se ao tipo de autenticação que será utilizado para verificar as senhas. As senhas são verificadas via HTTP e a palavra-chave Basic não deve ser alterada.

AuthName: Este é o texto que será exibido no prompt de senha. Você pode colocar qualquer coisa aqui.

AuthUserFile: Esta linha designa o caminho do servidor para o arquivo de senha (que criaremos na próxima etapa). Requer valid-user: Esta linha informa ao arquivo .htaccess que somente usuários definidos no arquivo de senha podem acessar a tela de login do phpMyAdmin.

Crie o arquivo htpasswd:

Agora iremos em frente e criaremos as informações válidas do usuário.

Comece criando um arquivo htpasswd. Use o comando htpasswd e coloque o arquivo em um diretório de sua escolha, desde que não seja acessível a partir de um navegador. Embora você possa nomear o arquivo de senha como preferir, a convenção é nomeá-lo como .htpasswd.

sudo htpasswd -c /path/to/passwords/.htpasswd username

Um prompt solicitará que você forneça e confirme sua senha.

Depois que o par de nome de usuário e senha for salvo, você poderá ver que a senha está criptografada no arquivo.

Termine reiniciando o apache:

sudo service apache2 restart

Acessando o phpMyAdmin:

O phpMyAdmin agora será muito mais seguro, pois somente usuários autorizados poderão acessar a página de login. Preencha-a com o nome de usuário e senha que você gerou. Depois de fazer o login, você pode acessar o phpmyadmin com o nome de usuário e a senha do MySQL.

  • Ubuntu:

Para começar, podemos simplesmente instalar o phpMyAdmin dos repositórios padrão do Ubuntu.

Podemos fazer isso atualizando nosso índice de pacotes local e, em seguida, usando o sistema de empacotamento apt para baixar os arquivos e instalá-los em nosso sistema:

sudo apt-get update
sudo apt-get install phpmyadmin

Aviso:

Quando o primeiro prompt aparece, o apache2 é realçado, mas não selecionado. Se você não clicar em “ESPAÇO” para selecionar o Apache, o instalador não moverá os arquivos necessários durante a instalação. Clique em “ESPAÇO”, “TAB” e depois “ENTER” para selecionar Apache.

  1. Para a seleção do servidor, escolha apache2.

  2. Selecione sim quando perguntado se deseja usar dbconfig-common para configurar o banco de dados

  3. Você será solicitado a inserir a senha do administrador do banco de dados

  4. Você será solicitado a escolher e confirmar uma senha para o próprio aplicativo phpMyAdmin

O processo de instalação realmente adiciona o arquivo de configuração phpMyAdmin Apache no diretório /etc/apache2/conf-enabled/, onde é lido automaticamente.

A única coisa que precisamos fazer é habilitar explicitamente a extensão php5-mcrypt, que podemos fazer digitando:

sudo php5enmod mcrypt

Depois, você precisará reiniciar o Apache para que suas alterações sejam reconhecidas:

sudo service apache2 restart

Você pode então acessar o phpmyadmin indo para your-ip-address/phpmyadmin.

  • Fedora:

O Fedora envia o pacote phpMyAdmin, mas esteja ciente de que o arquivo de configuração é mantido em /etc/phpMyAdmin/ e pode diferir em alguns aspectos da documentação oficial do phpMyAdmin.

Terminal:

Primeiro, você seguirá uma prática recomendada simples: garantir que a lista de pacotes disponíveis esteja atualizada antes de instalar qualquer coisa nova.

dnf -y update

Então é só rodar um comando para instalação via apt-get:

dnf -y install phpmyadmin

Segurança:

Para proteger o phpMyAdmin, devemos bloquear o acesso a um endereço IP específico.

Quando o pacote phpMyAdmin é instalado, um arquivo Apache Virtual Host é adicionado para configurar o acesso à web. Vamos editar esse arquivo:

vim /etc/httpd/conf.d/phpMyAdmin.conf

Por padrão, a configuração do phpMyAdmin só permite o acesso a partir do servidor em que está instalado.

Altere cada endereço IP para aquele que você encontrou em qual é meu endereço IP? ou outro endereço IP que se conectará remotamente ao phpMyAdmin:

Require ip 127.0.0.1
Allow from 127.0.0.1
Require ip 127.0.0.1
Allow from 127.0.0.1

Reiniciar o Apache:

systemctl restart httpd

Verifique se o phpMyAdmin está funcionando visitando http://the_IP_of_your_server/phpmyadmin Por exemplo: http://1.2.3.4/phpmyadmin

  • Compositor:

Você pode instalar o phpMyAdmin usando o Composer, mas atualmente não está disponível no repositório padrão do Packagist devido às suas limitações técnicas.

A instalação é possível adicionando nosso próprio repositório https://www.phpmyadmin.net/packages.json:

composer create-project phpmyadmin/phpmyadmin --repository-url=https://www.phpmyadmin.net/packages.json --no-dev
  • Instalando do Git:

Você pode clonar a fonte atual do phpMyAdmin de https://github.com/phpmyadmin/phpmyadmin.git:

git clone https://github.com/phpmyadmin/phpmyadmin.git

Além disso, você precisa instalar dependências usando o Composer:

composer update

Se você não pretende desenvolver, pode pular a instalação das ferramentas do desenvolvedor invocando:

composer update --no-dev

Mac:

  • Produtos de Terceiros: MAMP, AMPPS, Bitnami MAMPStack, XAMPP

  • Manualmente:

Baixe phpMyAdmin , Para todos os idiomas, escolha algo como phpMyAdmin-4.6.4-all-languages.tar.bz2 ou para idiomas específicos, escolha algo como phpMyAdmin-4.6.4-english.tar.gz

Depois de escolher um kit phpmyadmin extraia a pasta. Renomeie a pasta extraída para phpMyAdmin, é fácil fazer referência. Copie a pasta extraída para a raiz de documentos do seu servidor web local. Dependendo de como você configurou seu servidor web, esta pode ser a pasta htdocs ou public_html.

Crie a pasta config

mkdir ~/Sites/phpmyadmin/config

Altere as permissões

chmod o+w ~/Sites/phpmyadmin/config

Execute a configuração no navegador

http://localhost/~username/phpmyadmin/setup/ ou http://localhost/phpmyadmin/setup/

Você precisa criar uma nova conexão com o servidor mysql localhost, clique em novo servidor.

Mude para a aba Authentication e defina o usuário root local do mysql e a senha. Adicione o nome de usuário “root” (talvez já preenchido, adicione a senha que você configurou anteriormente para a configuração do usuário root do MySQL, clique em salvar e você retornará à tela anterior. (Esta não é a senha de administrador ou root do OSX – é o usuário root do MySQL).

Certifique-se de clicar em salvar, então um config.inc.php está agora no diretório /config do diretório phpmyadmin, mova este arquivo para o nível raiz de /phpmyadmin e remova o agora vazio / diretório de configuração.

Agora indo para http://localhost/~username/phpmyadmin/ agora permitirá que você interaja com seus bancos de dados MySQL.

Observação: Altere o host local com your-ip-address

Como criar um banco de dados e conceder privilégios para o usuário do banco de dados. #

  1. Navegue até o phpMyAdmin pela URL http://your_ip/phpmyadmin ou http://localhost/phpmyadmin
  2. Faça login usando o nome de usuário root e a senha do root.
  3. Clique na guia Bancos de dados.
  4. Digite o nome do banco de dados, selecione o agrupamento (você pode deixar como padrão) e clique em criar.
  5. Clique na guia Privilégios e selecione “Adicionar conta de usuário”.
  6. Você pode selecionar localhost se a conexão for feita somente de local. Selecione “qualquer host” se estiver planejando se conectar de diferentes hosts.
  7. Digite o novo nome de usuário e senha.
  8. Selecione os privilégios que você precisa para esse usuário.
  9. Clique na guia Banco de Dados e selecione o banco de dados para o qual você precisa dar acesso (no nosso caso o banco de dados recém-criado na etapa 4). Clique em Ir.
