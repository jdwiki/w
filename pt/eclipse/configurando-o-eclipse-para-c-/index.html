Linux + CMake (gerador “Unix Makefiles”) + Qt (opcional) #

Você deve ter um projeto CMake simples myproject, e vamos criar um espaço de trabalho Eclipse fora dele: <pré> myproject/ .git/ CMakeLists.txt src/ main.cpp workspace/ myproject/ Release/ Debug/

Qt (opcional) #

  • Obtenha o mais recente CDT do Eclipse e instale o pacote Qt nele através de “Ajuda -> Instalar Novo Software”.

Área de trabalho #

  • Crie um diretório “espaço de trabalho” vazio ao lado do diretório de origem do projeto CMake.
  • Inicie o Eclipse e alterne para esse diretório “espaço de trabalho”.
  • Crie um projeto C++ (para Qt com Eclipse anterior ao Neon: crie “Qt Makefile Project” e exclua o arquivo *.pro, makefile e main.cpp dele)

Anexando fontes ao projeto #

  • Vá para Propriedades do Projeto -> Caminhos e Símbolos -> Local de Origem -> Pasta de Link.
  • Marque “Avançado” e vincule a pasta de origem do projeto CMake assim: ../../myproject/src/. Funciona porque o espaço de trabalho está fora do diretório do projeto CMake.

Gerador CMake #

  • Crie a pasta Release no projeto.
  • Vá para a visualização “Make Target” (Ctrl+3 e digite “Make Target” se for difícil de encontrar). A visualização “Make Target” é semelhante à visualização do projeto.
  • Clique com o botão direito do mouse na pasta “Release” e clique em “New…”.
  • Desmarque “Igual ao nome do destino”.
  • Desmarque “Usar configurações do construtor”.
  • Digite “Release” no campo “Target name”.
  • Deixe “Make target” vazio.
  • Defina “comando de compilação” para algo como cmake ../../../myproject/.
  • Clique OK.
  • Dê um duplo clique neste alvo de make “Release” que acabou de ser criado na pasta Release. Isso executará a geração cmake.

Construir #

  • Vá para Propriedades do Projeto e crie uma configuração “Release”.
  • Ative a configuração “Release”.
  • Para a configuração “Release”, desmarque “Gerar Makefiles automaticamente”.
  • Defina o diretório de compilação como “Release”.
  • Ative a compilação paralela.

Agora, você pode construir o projeto do Eclipse com um usual Ctrl+b “Build”.

Executando novamente o CMake (para gerar novamente os makefiles) #

  • Remova tudo do diretório “Release”.
  • Vá para a visualização “Make Target”.
  • Clique duas vezes no alvo “Release”.
