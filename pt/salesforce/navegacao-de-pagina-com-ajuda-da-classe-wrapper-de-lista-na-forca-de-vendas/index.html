Navegação de página com ajuda da classe wrapper de lista na força de vendas.

  • Controlador de paginação

Nesta página

  • Controlador de paginação

Slaesforce StandardSetController mantém apenas a Lista de sObject, não contém a lista de objetos da classe wrapper. O código-fonte abaixo demonstra o uso da paginação usando a classe wrapper na força de vendas.

Controlador de paginação #

Exemplo de código: Agora comece com a criação do controlador de paginação

public with sharing class Pagination {

}

vou mostrar todos os contatos na forma de paginação e deve haver uma caixa de seleção para cada contato para selecionar ou desmarcar contato para realizar a operação de exclusão nos contatos. Então eu preciso criar uma classe wrapper para manter contato e variável booleana para seleção. Primeiro crie a classe Wrapper para a paginação do controlador Insira o código abaixo no controlador de paginação.

 public class contactWrapper{
        public Contact cont {get;set;}
        public Boolean isSelected{get;set;}
        public contactWrapper(contact c,Boolean s)
        {
            cont=c;
            isSelected=s;
        }
    }

Agora recuperando dados no Apex para paginar com a ajuda de StandardSetController. O StandardSetController é uma ferramenta extremamente poderosa com funcionalidade integrada que você pode usar para simplificar bastante o código personalizado em suas páginas do Visualforce. Como o servidor retorna apenas os dados da página que está sendo solicitada, o StandardSetController pode reduzir significativamente o estado de exibição, especialmente em comparação com o estado de exibição que você obteria ao usar SOQL.

 Public Integer noOfRecords{get; set;} // Future reference in Visual force Page
    Public Integer size{get;set;}  // Future reference in Visual force Page
    public final Integer Page_Size=10; // Number records in a Page should be displayed 

 public ApexPages.StandardSetController setCon {
        get{
            if(setCon == null){
                size=Page_Size;
                string queryString = 'Select Id,Name, Email, Birthdate, Phone, MobilePhone from Contact order by Name';
                setCon = new ApexPages.StandardSetController(Database.getQueryLocator(queryString));
                setCon.setPageSize(size);
                noOfRecords = setCon.getResultSize();
            }
            return setCon;
        }set;
    }

Agora você tem os contatos na Variável setCon , sempre que seu pedido for setCon.getRecords() ele irá recuperar os 10 primeiros registros de contato do setCon. Aqui eu criei uma classe wrapper simples para mostrar a demonstração. Você pode criar sua própria classe wrapper com base no requisito. Mas sempre você deve estar ciente de que ApexPages.StandardSetController contém apenas List of sObject, ele não conterá a lista de classe wrapper object . Essa é a razão pela qual escrevi código extra abaixo para realizar esse futuro de maneira diferente. O código abaixo converte a lista de contatos em uma lista de objetos da classe wrapper, para que, sempre que você chamar os contatos na página visual force, ele receba a lista de objetos da classe wrapper.

 public list<contactWrapper> contWpr{get;set;} 
    public set<id> selectedContactIds{ get;private set;} // to maintain state of the selected contact
                                                         // through out paginating 

    public Pagination() {
       selectedContactIds=new  set<id>();
    }

      Public list<contactWrapper> getContacts(){
         contWpr =new list<contactWrapper>();
        for(Contact c: (List<Contact>)setCon.getRecords())
            if(selectedContactIds.contains(c.id))
                contWpr.add(new contactWrapper(c,true));
                else
                contWpr.add(new contactWrapper(c,false));
        return contWpr;
    }

Agora você escreveu o código para gerar o resultado Mas como navegar pela página? Isso pode ser feito com um passo fácil com ApexPages.StandardSetController.Veja a beleza do código abaixo do StandardSetController, Não há necessidade de manter o número da página, deslocamento e limite etc. Basta usar os métodos StandardSetController. Copie o código abaixo no controlador de paginação.

public Boolean hasNext {
        get {
            return setCon.getHasNext();
        }
        set;
    }
    public Boolean hasPrevious {
        get {
            return setCon.getHasPrevious();
        }
        set;
    }

    public Integer pageNumber {
        get {
            return setCon.getPageNumber();
        }
        set;
    }

    public void first() {
        setCon.first();
        // do you operation here 
    }

    public void last() {
        setCon.last();
            // do you operation here 
    }

    public void previous() {
        setCon.previous();
            // do you operation here 
    }

    public void next() {
        setCon.next();
            // do you operation here 
    }

Você está quase terminando de paginar os contatos. Últimos métodos que adicionei para cumprir toda a funcionalidade da minha página. Como mencionei anteriormente, temos uma caixa de seleção adicional para selecionar o contato e executar a operação de exclusão nos contatos selecionados.

public void contactSelection()
    {
        Id id=(Id)ApexPages.currentPage().getParameters().get('cId');
        if(selectedContactIds.contains(id))
            selectedContactIds.remove(id);
            else
            selectedContactIds.add(id);
    }

 public void deleteContacts()
    {
        List<contact> contactToDelete=[select id from contact where id in :selectedContactIds];
        if(contactToDelete.size()!=0)  //   if(!contactToDelete.isEmpty()) // Best Practice 
           { 
                try {  delete contactToDelete; }  // You may get Exception if you try to delete the 
                                                // related contact ,include try block to avoid error.
                catch(exception ex){ System.debug(ex); }
                refresh();
           }
    }   

 public pageReference refresh() {
        setCon = null;
        selectedContactIds=new set<id>();
        getContacts();
        setCon.setPageNumber(1);
        return null;
    }
