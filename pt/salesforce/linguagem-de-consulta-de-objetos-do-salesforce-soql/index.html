Sintaxe #

  • SELECT Id FROM Account
  • SELECT Id, Nome FROM Account
  • SELECT Id FROM Account WHERE Name = ‘SomeAccountName’
  • SELECT Id, Name, (SELECT Id, Name FROM Contacts) FROM Account
  • SELECT Id, Name FROM Account WHERE Id = :apexVariableName

Consulta SOQL básica #

SELECT Id, Name FROM Account

Isso retornará os campos Id e Name da tabela Account. Nenhuma filtragem ou classificação será aplicada.

Consulta SOQL para referenciar os campos do objeto pai #

Quando os objetos são vinculados por uma relação de pesquisa ou mestre-detalhe, os campos de registros pai podem ser referenciados a partir do registro filho ou ‘objeto base’ em uma consulta. Isso também é conhecido como travessia ascendente.

SELECT FirstName, Account.Name, Account.Category__c FROM Contact

É possível percorrer cinco registros para cima.

SELECT Account.Owner.Profile.CreatedBy.Name FROM Contact

Quando o objeto base for um campo de pesquisa personalizado, o __c no nome do campo Primary_Influencer__c, por exemplo, será alterado para __r.

SELECT Primary_Influencer__r.Nickname__c FROM Contact

Consulta SOQL para obter registros filhos

SELECT Id, Name, (SELECT Id, FirstName, LastName FROM Contacts) FROM Account

Consulta SOQL com pedido #

SELECT Id, Name FROM User ORDER BY LastName

SELECT Id, Name FROM Contact ORDER BY LastModifiedDate DESC

SELECT Name, Title FROM User ORDER BY Title ASC NULLS FIRST

SELECT Id FROM Contact ORDER BY LastName ASC NULLS LAST, FirstName ASC NULLS FIRST

Usando SOQL para construir um mapa #

Um recurso muito útil que muitas pessoas ignoram é a capacidade de construir um mapa usando uma consulta SOQL.

Map<Id, Account> accounts = new Map<Id, Account>([SELECT Id, Name FROM Account]);
System.debug(accounts);

Quando você executa esse código, accounts contém um mapa de seus objetos Account, digitados em Id. A saída para o log de depuração seria semelhante a esta:

11:15:10:025 USER_DEBUG [13]|DEBUG|{
    XXXXXXXXXXXXXXXXXX=Account:{Id=XXXXXXXXXXXXXXXXXX, Name=Account 1}, 
    YYYYYYYYYYYYYYYYYY=Account:{Id=YYYYYYYYYYYYYYYYYY, Name=Account 2}, 
    ZZZZZZZZZZZZZZZZZZ=Account:{Id=ZZZZZZZZZZZZZZZZZZ, Name=Account 3}, 
    ...
}

Agora você pode pesquisar os objetos Account usando seu ID. Além disso, se você quiser uma coleção de IDs exclusivos, você pode chamar a função keySet() da classe Map, assim:

System.debug(accounts.keySet());

que se parece com isso no log de depuração:

11:23:21:010 USER_DEBUG [15]|DEBUG|{XXXXXXXXXXXXXXXXXX, YYYYYYYYYYYYYYYYYY, ZZZZZZZZZZZZZZZZZZ, ...}

Isso é muito útil quando você precisa consultar para obter registros e acessá-los repetidamente em seu código.

Consulta SOQL com filtragem #

SELECT Name FROM User WHERE IsActive = true

Isso retornará o nome de todos os usuários ativos.

SELECT Name, Phone FROM Contact WHERE CreatedDate >= 2016-01-01T00:00:00.000Z

Isso retornará os contatos criados a partir de 1º de janeiro de 2016.

SELECT Id, Name FROM Account LIMIT 100

Isso retornará as primeiras 100 contas de uma lista não ordenada.

SELECT Id, Name, Phone FROM Lead WHERE Phone LIKE '(%) %-%'

Isso retornará Leads com um número de telefone correspondente ao formato especificado. “%” atua como um caractere curinga.

Usar LIKE '% %' também permite que um desenvolvedor replique uma fórmula CONTAINS( ).

SELECT Email FROM Lead WHERE LeadSource LIKE '%Google%'

Retornará leads com uma fonte de lead que contém o Google, ou seja, ‘Google AdWords’ e ‘Google Natural Search’.

SOQL dinâmico #

Você pode executar uma consulta de banco de dados a partir de uma String em vez de uma expressão SOQL regular:

String tableName = 'Account';
String queryString = 'SELECT Id FROM ' + tableName + ' WHERE CreatedDate >= YESTERDAY';
List<SObject> objects = Database.query(queryString);

Como as consultas SOQL dinâmicas não são compiladas, suas referências de esquema não são validadas, portanto, é preferível usar a interpolação de variáveis ​​do Apex usando a sintaxe :variable sempre que possível.

Consultas SOQL no Apex #

Para realizar uma consulta no Apex, coloque a consulta entre colchetes. O resultado pode ser atribuído a uma lista ou a um único objeto.

List<Account> allAccounts = [SELECT Id, Name FROM Account];
Account oldestAccount = [SELECT Id, Name FROM Account ORDER BY CreatedDate LIMIT 1];

Referências de variáveis ​​em consultas SOQL do Apex #

Para fazer referência a uma variável em uma consulta, adicione dois pontos (:) antes do nome da variável.

Datetime targetDate = Datetime.now().addDays(-7);
List<Lead> recentLeads = [SELECT Id FROM Lead WHERE CreatedDate > :targetDate];

string targetName = 'Unknown';
List<Contact> incompleteContacts = [SELECT Id FROM Contact WHERE FirstName = :targetName];

Exceções potenciais em consultas SOQL do Apex #

Ao atribuir a um único objeto, uma consulta que retorna algo diferente de uma única linha lançará um QueryException.

try {
    Account a = [SELECT Id FROM Account WHERE Name = 'Non-existent Account'];  
} catch (QueryException e) {
    // List has no rows for assignment to SObject
}

try {
    Account a = [SELECT Id FROM Account];  
} catch (QueryException e) {
    // List has more than 1 row for assignment to SObject
}

A tentativa de usar um campo que você não incluiu na consulta lançará um SObjectException

Account a = [SELECT Id FROM Account LIMIT 1];
try {
    System.debug( a.Name );
} catch (SObjectException e) {
    // SObject row was retrieved via SOQL without querying the requested field: Name
}

Usando uma semijunção #

Selecionando todas as contas que possuem registros de oportunidades abertas sob elas

SELECT Id, Name FROM Account WHERE AccountId IN 
    (SELECT Id FROM Opportunity WHERE IsClosed = false)
