Esta é uma parte muito pequena, mas ainda essencial se você quiser se comunicar de forma eficiente com outras pessoas usando o Sublime Text 3.

Mais para vir: settings, keymap, mousemap etc…

Temas #

Um tema altera a aparência global do Sublime Text 3. Ele altera as guias, barra lateral, painéis rápidos (ir para, paleta de comandos), barra de status, etc…

Aqui está uma lista não exaustiva de temas (top 100):

  • [Antes do amanhecer](https://packagecontrol.io/packages/Antes do amanhecer)
  • Tema do Material
  • Tema - Spacegray
  • Seti_UI
  • Tema - Brogrammer
  • Tema - Flatland
  • Tema - Refrigerante

Nota: a convenção para o nome dos pacotes do tema é que eles começam com Tema -.

Theme - Focus
    img/
    focus.light.sublime-theme
    focus.dark.sublime-theme

Como você pode ver, essa convenção não é realmente respeitada.

Os temas são definidos em um arquivo .sublime-theme.

Esquema de cores #

Os esquemas de cores alteram as cores do código. Eles podem alterar, por exemplo, com que cor as palavras-chave são destacadas, com que cor a linha em que o acento circunflexo está destacada (precisa que highlight_line seja definido como true nas preferências (não faz parte do esquema de cojor)) , qual é a cor do acento circunflexo, etc.

Muitos temas vêm com esquemas de cores ou muitos pacotes contêm muitos deles.

precisa de exemplo de pacotes

O esquema de cores é definido em um arquivo .tmTheme. Este é um arquivo XML no formato de lista de propriedades, que é usado por muitos editores de texto.

A convenção para o nome dos packages dos esquemas de cores é que eles começam com Color Scheme -

Color Scheme - Focus
    first.tmTheme
    second.tmTheme
    ...

Ferramentas #

  • você pode usar este aplicativo da web para criar/editar seus esquemas de cores: tmTheme editor.
  • Outra ferramenta é o ColorSchemeEditor que permite editar seu esquema de cores de Sublime Text.
  • Você também pode alternar rapidamente o esquema de cores usando Schemr

Plugins #

Plugins são arquivos .py que alteram o comportamento do Sublime Text 3. Eles são escritos, como você provavelmente adivinhou, em Python.

Como existem muitos tipos diferentes de plugins (manipulação de texto, linting, formatação, visualização, etc), não daremos nenhum exemplo para não preencher esta página com uma lista enorme. Dê uma olhada na página de navegação do Controle de Pacotes se estiver interessado em otimizar seu fluxo de trabalho com alguns plugins (você deveria estar).

Pacotes #

Um pacote é uma pasta que pode conter qualquer coisa listada nestes exemplos (e outras coisas que não podem ter nada a ver com o Sublime Text 3, como um gulpfile.js se você estiver automatizando algumas tarefas).

Você pode instalar qualquer pacote usando o Package Control.

Um .sublime-package ? #

Talvez você já tenha visto que, ao instalar um pacote usando o controle de pacotes, você só obtém um .sublime-package na pasta Install Packages… Como é que você não recebe o código? Na verdade, um .sublime-package é um arquivo .zip renomeado. Você pode extrair ou visualizar alguns arquivos dentro de um .sublime-package usando PackageResourceViewer.

Definições #

As configurações são, como muitas coisas no Sublime Text 3, arquivos simples JSON. Porém, a extensão do arquivo não é .json mas sim .sublime-settings. Como você provavelmente entendeu, as configurações alteram como os pacotes afetam o Sublime Text 3. É importante estar ciente do máximo de configurações possível para otimizar seu fluxo de trabalho.

O sistema para a maioria dos pacotes é o seguinte: um padrão e um usuário que substitui o padrão.

Para entender a prioridade das configurações (o que é altamente recomendável), dê uma olhada na documentação não oficial, mas incrível: <http://docs.sublimetext.info/en/latest/customization/settings.html#the-settings -hierarquia>
