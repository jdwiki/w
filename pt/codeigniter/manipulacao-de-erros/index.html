Manipulação de erros

  • log_message()

O CodeIgniter permite que você crie relatórios de erros em seus aplicativos usando as funções descritas abaixo. Além disso, possui uma classe de registro de erros que permite que mensagens de erro e depuração sejam salvas como arquivos de texto.

log_message() #

Esta função permite gravar mensagens em seus arquivos de log. Você deve fornecer um dos três “níveis” no primeiro parâmetro, indicando que tipo de mensagem é (debug, error, info), com a própria mensagem no segundo parâmetro.

Exemplo:

if ($some_var == "") {
    log_message('error', 'Some variable did not contain a value.'); 
} 
else {
    log_message('debug', 'Some variable was correctly set'); 
}

log_message('info', 'The purpose of some variable is to provide some value.');

Sintaxe #

log_message($level, $message);

Parâmetros:

  • $level (string) – Nível de log: ’error’, ‘debug’ ou ‘info’
  • $message (string) – Mensagem para log

Tipo de retorno: void

Existem três tipos de mensagens: #

  • Mensagens de erro. Estes são erros reais, como erros de PHP ou erros do usuário.
  • Mensagens de depuração. Estas são mensagens que auxiliam na depuração. Por exemplo, se uma classe foi inicializada, você pode registrar isso como informações de depuração.
  • Mensagens informativas. Estas são as mensagens de menor prioridade, simplesmente dando informações sobre algum processo. O CodeIgniter não gera nativamente nenhuma mensagem de informação, mas você pode querer em seu aplicativo.

Observação: para que o arquivo de log seja realmente gravado, os “logs” a pasta deve ser gravável. Além disso, você deve definir o “limite” para logando em application/config/config.php. Você pode, por exemplo, querer apenas que as mensagens de erro sejam registradas, e não os outros dois tipos. Se você defini-lo como zero, o log será desabilitado.

show_error() #

Esta função exibirá a mensagem de erro fornecida a ela usando o seguinte modelo de erro:

Caminho - application/errors/error_general.php

O parâmetro opcional $status_code determina qual código de status HTTP deve ser enviado com o erro.

Sintaxe #

show_error($message, $status_code, $heading = 'An Error Was Encountered')

Parâmetros:

  • $message (mixed) – Mensagem de erro
  • $status_code (int) – Código de status de resposta HTTP
  • $heading (string) – Erro no cabeçalho da página

Tipo de retorno: void

Fonte #

  1. show_error em codeigniter.com

show_404() #

Esta função exibirá a mensagem de erro 404 fornecida a ela usando o seguinte modelo de erro:

Caminho - application/errors/error_404.php

A função espera que a string passada a ela seja o caminho do arquivo para a página que não foi encontrada. Observe que o CodeIgniter mostra automaticamente as mensagens 404 se os controladores não forem encontrados.

O CodeIgniter registra automaticamente qualquer chamada show_404(). Definir o segundo parâmetro opcional como FALSE ignorará o registro.

Sintaxe #

show_404($page = '', $log_error = TRUE)

Parâmetros:

  • $page (string) – string URI
  • $log_error (bool) – Se o erro deve ser registrado

Tipo de retorno: void

Fonte #

  1. show_404 em codeigniter.com
