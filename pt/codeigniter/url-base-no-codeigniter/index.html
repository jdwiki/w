Configurando seu URL base no Codeigniter #

Você precisará definir sua URL base em application/config/config.php

Se não estiver definido, o CodeIgniter tentará adivinhar o protocolo e o caminho para sua instalação, mas devido a questões de segurança, o nome do host será definido para $_SERVER['SERVER_ADDR'] se disponível, ou localhost caso contrário. O mecanismo de detecção automática existe apenas por conveniência durante desenvolvimento e NÃO DEVE ser usado em produção!

$config['base_url'] = '';

Deve ser arquivado como

$config['base_url'] = 'http://localhost/projectname/';

$config['base_url'] = 'http://www.example.com/';

Sempre bom usar / no final de base_url

Quando você não define sua URL base, pode encontrar alguns erros em que não pode carregar seu CSS, imagens e outros itens de ativos. E também você pode ter problemas para enviar formulários, como alguns usuários encontraram.

Atualizar

Se você não quiser definir sua URL base, outra maneira é.

Crie um novo arquivo principal em application/core/MY_Config.php

E cole este código

<?php

class MY_Config extends CI_Config {

    public function __construct() {

        $this->config =& get_config();

        log_message('debug', "Config Class Initialized");

        // Set the base_url automatically if none was provided

        if ($this->config['base_url'] == '')
        {
            if (isset($_SERVER['HTTP_HOST']))
            {
                $base_url = isset($_SERVER['HTTPS']) && strtolower($_SERVER['HTTPS']) !== 'off' ? 'https' : 'http';
                $base_url .= '://'. $_SERVER['HTTP_HOST'];
                $base_url .= str_replace(basename($_SERVER['SCRIPT_NAME']), '', $_SERVER['SCRIPT_NAME']);
            }

            else
            {
                $base_url = 'http://localhost/';
            }

            $this->set_item('base_url', $base_url);

        }
    }
}

Algo mais sobre base_url #

O que acontece se eu não definir base_url ? #

Você não receberá nenhum erro de impotência para definir isso e prosseguir. Você pode continuar sem configurar, mas deve saber sobre injeção de cabeçalho HTTP

Se eu não configurar o que vai aparecer? #

Você obterá http://[::1]/ em vez de sua URL real.

O que isso significa http://[::1]/ ?? #

Este é um URL temporário definido pelo CI por padrão. Isso apontará a raiz do seu documento.

::1 - Endereço do servidor (localhost) Leia mais sobre isso

Como definir o base_url() adequado? #

A URL base deve sempre apontar para a raiz da pasta do seu projeto. (fora da pasta do aplicativo)

$config['base_url'] = 'http://localhost/path/to/project'; # If localhost
$config['base_url'] = 'http://stackoverflow.com/'; # If live
$config['base_url'] = 'https://www.wikiod.com/pt/docs/'; # If live & inside subdomain (assume documentation is subfolder/subdomain)

Como usar base_url()?? #

O uso mais comum é encontrar o caminho certo para seus arquivos js ou css.

<link rel="stylesheet" href="<?php echo base_url('styles/style.css');?>" />
<script src="<?php echo base_url('vendor/jquery/jquery.min.js');?>"></script>

Adicionar o código acima em sua visualização produzirá HTML como abaixo:

<link rel="stylesheet" href="http://localhost/path/to/project/styles/style.css" />
<script src="http://localhost/path/to/project/vendor/jquery/jquery.min.js"></script>

Maneira inteligente de configurar o base_url #

As seguintes linhas de código são uma maneira mais inteligente de configurar o base_url no codeigniter:

$config['base_url'] = ((isset($_SERVER['HTTPS']) && $_SERVER['HTTPS'] == "on") ? "https" : "http");
$config['base_url'] .= "://".$_SERVER['HTTP_HOST'];
$config['base_url'] .= str_replace(basename($_SERVER['SCRIPT_NAME']),"",$_SERVER['SCRIPT_NAME']);

Recomendado é

$config['base_url'] = 'https://stackoverflow.com/';

Porque todos conhecem o espaço de hospedagem. Portanto, se você definir assim, pode impedir a injeção no seu site/host.
