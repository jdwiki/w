Instalando Rails no mac. #

Você precisaria instalar o ruby antes de poder instalar os trilhos.

O Mac já vem com o ruby instalado baseado em quão recente é o seu macOS? Dependendo de qual versão do ruby você deseja para o seu desenvolvimento, a melhor maneira de instalar o Ruby é usar o RVM. No seu terminal, digite o comando abaixo listado nas etapas:

  1. Instale o rvm

    curl -sSL https://get.rvm.io | bash -s stable --ruby

  2. Para Rails 3, a melhor versão para instalar é ruby 1.9.3

    rvm install 1.9.3
ruby -v #=> 1.9.3

  3. Defina sua versão do Ruby

    rvm use 1.9.3 --default

  4. Instale o Rails (esta versão do Rails requer a versão ruby ​​>=1.9.3)

    gem install rails -v 4.2.7.1
rails -v #=> 4.2.7.1

  5. Instale o aplicativo Rails

    rails new my_first_app #(this will install the app for you.)
cd my_first_app
rails s #(run the server)

  6. Abra o navegador e digite abaixo o seu URL.

    http://localhost:3000

    Message saying ‘Welcome to rails’ will be displayed or similar.

Olá Mundo em Rails #

  1. Diga “Olá”, Rails

    To get Rails saying “Hello”, you need to create at minimum a controller and a view.

    A controller’s purpose is to receive specific requests for the application. Routing decides which controller receives which requests. Often, there is more than one route to each controller, and different routes can be served by different actions. Each action’s purpose is to collect information to provide it to a view.

    A view’s purpose is to display this information in a human readable format. An important distinction to make is that it is the controller, not the view, where information is collected. The view should just display that information. By default, view templates are written in a language called eRuby (Embedded Ruby) which is processed by the request cycle in Rails before being sent to the user.

    To create a new controller, you will need to run the “controller” generator and tell it you want a controller called “Welcome” with an action called “index”, just like this:

    $ bin/rails generate controller Welcome index

    Rails will create several files and a route for you.

    create  app/controllers/welcome_controller.rb
 route  get 'welcome/index'
invoke  erb
create    app/views/welcome
create    app/views/welcome/index.html.erb
invoke  test_unit
create    test/controllers/welcome_controller_test.rb
invoke  helper
create    app/helpers/welcome_helper.rb
invoke  assets
invoke    coffee
create      app/assets/javascripts/welcome.coffee
invoke    scss
create      app/assets/stylesheets/welcome.scss

  2. Os mais importantes são, obviamente, o controlador, localizado em app/controllers/welcome_controller.rb e a visualização, localizada em app/views/welcome/index.html.erb.

    Open the app/views/welcome/index.html.erb file in your text editor. Delete all of the existing code in the file, and replace it with the following single line of code:

    <h1>Hello, Rails!</h1>

  3. Agora que criamos o controller e a view, precisamos dizer ao Rails quando queremos “Hello, Rails!” aparecer. No nosso caso, queremos que ele apareça quando navegarmos para a URL raiz do nosso site, http://localhost:3000.

Em seguida, você deve informar ao Rails onde está localizada sua home page real. Edite o arquivo adicionando a linha de código root ‘welcome#index’. Deve ser algo como o seguinte:

   Rails.application.routes.draw do
     get 'welcome/index'

     root 'welcome#index'
   end

  1. root welcome#index diz ao Rails para mapear requisições para a raiz da aplicação para a ação index do controlador welcome e get welcome/index diz ao Rails para mapear requisições para http://localhost:3000/welcome/index para a ação de índice do controlador de boas-vindas. Isso foi criado anteriormente quando você executou o gerador do controlador (bin/rails geram o índice de boas-vindas do controlador).

  2. Yay, agora o momento da verdade. Inicie o servidor web após reiniciar seu servidor Rails e navegue até http://localhost:3000 em seu navegador. Você verá a mensagem “Hello, Rails!” que você colocou em app/views/welcome/index.html.erb, indicando que esta nova rota está realmente indo para a ação index do WelcomeController e está renderizando a visão corretamente.
