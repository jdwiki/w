Defina o estilo de código para um projeto #

O PhpStorm oferece configurações padrão para estilo de código para uma grande quantidade de idiomas com base nas melhores práticas e padrões comuns. Mas você pode personalizar o estilo de cada idioma em uma base por projeto nas Configurações do PhpStorm > Editor > Estilo de código.

Esquemas

Esquemas são coleções de diretrizes e configurações de estilo de código. Você pode selecionar um esquema para um projeto e ele será aplicado instantaneamente. Há também um esquema de projeto que só está disponível enquanto você tem um projeto aberto. O esquema Projeto não salva as diretrizes nas configurações gerais do usuário, mas nas próprias configurações do projeto.

Se você clicar no botão Gerenciar, poderá adicionar um novo esquema para configurar rapidamente um novo conjunto de diretrizes de estilo de código. A caixa Gerenciar também pode ser usada para exportar ou importar esquemas, o que é bastante útil se você quiser compartilhar esquemas com seus amigos ou colegas.

Muda o estilo para um idioma específico #

Para alterar as configurações de estilo de código para um idioma específico, basta clicar no idioma disponível na barra lateral. Em seguida, será apresentada uma página de configurações diferente para cada idioma.

Por exemplo, a linguagem PHP terá configurações para tabulações e espaços, chaves ou PHP Docs.

Se você quiser saber o que cada configuração diferente significa, você pode procurá-las na documentação oficial.

Documentação de estilo de código para PhpStorm 2016 e superior

Definir estilos de código de um padrão predefinido como PSR-2 #

O PhpStorm já vem com muitos esquemas de linguagem predefinidos que são baseados em diretrizes e padrões comuns de estilo de código como PSR-2. Há um recurso oculto nas páginas de configurações de estilo de código onde você pode importar esses padrões e defini-los como sua configuração atual. Para fazer isso, basta escolher seu idioma de codificação no painel esquerdo. Em seguida, há um pequeno link no canto superior direito chamado Set from…

Ao clicar neste link, o PhpStorm apresentará um pequeno pop-up onde você pode escolher entre os padrões predefinidos na guia Estilo predefinido.

Imponha um estilo de código especificado para um projeto em vários membros da equipe #

Atualmente, não existe um método de botão de um clique para realmente aplicar as diretrizes de estilo de código em uma equipe, mas existem dois métodos para garantir que um determinado estilo de código seja aplicado ao seu produto.

Importar esquemas de estilo de código PhpStorm #

A primeira e mais fácil solução é configurar um esquema de estilo de código em sua própria instância do PhpStorm, exportar o esquema para uma unidade portátil ou unidade de rede e importar o esquema em todas as máquinas de desenvolvimento. Desta forma, um desenvolvedor pode usar facilmente os atalhos de teclado Cmd+Shift+L (MacOS) ou Ctl+< kbd>Alt+L (Windows / Linux) para formatar automaticamente o código fonte completo.

Verificações automatizadas de formato de código com um servidor CI #

Uma maneira muito rígida de controlar a formatação do código-fonte é implementar um servidor de integração contínua como Jenkins que é capaz de verificar se o código-fonte corresponde a um estilo de código predefinido.

Vamos supor que um desenvolvedor trabalhou em um novo recurso em sua própria ramificação de desenvolvimento e deseja enviar suas alterações para o repositório principal. Primeiro, ele envia as alterações para sua própria ramificação, onde o novo recurso será verificado pelo servidor de CI. Se a verificação falhou porque o código não está formatado corretamente, o desenvolvedor será notificado para que ele possa corrigir os problemas.

Existem várias maneiras de configurar verificações de qualidade e formatação de código com todos os diferentes servidores de integração, portanto, explicar como configurar um servidor com verificações deve ser feito nas tags correspondentes.
