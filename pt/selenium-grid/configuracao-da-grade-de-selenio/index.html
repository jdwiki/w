Instalação ou configuração #

Antes de configurar uma grade de Selenium, você precisa ter certeza de ter o Java instalado e configurado no caminho do ambiente do seu computador.

Configurar o Hub #

  • Faça o download da versão estável mais recente do Selenium Server.
  • Inicie o prompt de comando e navegue até o local em que você colocou o arquivo jar do servidor Selenium.
  • Tipo: (FYI: seu número de versão pode ser diferente do meu) java –jar selenium-server-standalone-2.53.0.jar –role hub
  • Agora deve ficar assim:

Basicamente, o que aconteceu é que o servidor web do selenium foi iniciado e agora está escutando em uma porta - neste caso, o padrão 4444 (FYI - Este número de porta pode ser alterado passando o parâmetro -port seguido pelo número da porta na qual você deseja para executar o servidor).

  • Agora abra um navegador e navegue até http://localhost:4444/grid/console
  • Se tudo estiver funcionando, o servidor deve aparecer e você verá algo assim:

Em seguida, precisamos configurar algumas máquinas de nós.

Configure os nós #

  • Assim como baixamos o Selenium Server para o Hub, também precisamos fazer o download em todas as nossas máquinas Node .
  • Depois de ter o arquivo jar do servidor Selenium na máquina do nó, navegue até o diretório onde o jar foi baixado e abra o prompt cmd.
  • Tipo: java –jar selenium-server-standalone-2.53.0.jar –role node –hub http://hubIP:4444/grid/register
    • hubIP :- in case the hub and nodes are running on a different machine
    • localhost :- in case the hub and nodes are running on same machine

Como você pode ver, o nó agora está registrado no hub, por padrão, o nó começa em -port 5555, mas você pode alterar o mesmo usando o parâmetro -port seguido pelo número da porta.

Se tudo funcionar como esperado, você deverá ver agora o endereço IP do nó que acabou de iniciar e registrar na visualização do console do hub:

Coisas a observar #

  • Se não especificarmos o seleniumProtocol, o Node será registrado com o Remote Control (Legacy) e o Webdriver Protocol (como visto na imagem acima).
  • Se o tipo de navegador e o número de instâncias não forem mencionados, o Node iniciará 5 instâncias do Firefox, 5 instâncias do Chrome e 1 instância do driver IE.

Isso é tudo que você precisaria fazer para uma Selenium Grid em funcionamento.

Configuração Json #

Um exemplo de configuração para um hub:

java -jar selenium-server-standalone-<versão>.jar -role hub -hubConfig hubConfig.json

{
    "_comment" : "Configuration for Hub - hubConfig.json",
    "host": ip,
    "maxSessions": 5,
    "port": 4444,
    "cleanupCycle": 5000,
    "timeout": 300000,
    "newSessionWaitTimeout": -1,
    "servlets": [],

Configuração e uso em C# #

Configuração #

Nos parágrafos seguintes haverá um exemplo por navegador para configuração em Json e configuração em C#.

Este exemplo espera que você tenha todos os drivers em sua variável de caminho e navegadores instalados.

Microsoft borda #

Código C# para criar um webdriver remoto

// Defining webdriver variable
RemoteWebDriver _webDriver;
// Creating Capabilities and chosing browser
capabiliteiten = DesiredCapabilities.Edge();
// Setting platform 
capabiliteiten.Platform = new Platform(PlatformType.Windows);
// Requesting remote webdriver
_webDriver = new RemoteWebDriver(_gridServerUri, capabiliteiten);

Configuração de nós em Json

{
    "browserName":"MicrosoftEdge",
    "platform": "WINDOWS",
    "maxIstances": 1,
    "seleniumProtocol": "WebDriver"
}

Ópera #

Código C# para criar um webdriver remoto Isto é para OperaChromium

// Defining webdriver variable
RemoteWebDriver _webDriver;
// Creating Capabilities
capabiliteiten = new DesiredCapabilities();
// Setting platform 
capabiliteiten.Platform = new Platform(PlatformType.Windows);
// Chosing browser
capabiliteiten.SetCapability(CapabilityType.BrowserName, "operablink");
// Requesting remote webdriver
_webDriver = new RemoteWebDriver(_gridServerUri, capabiliteiten);

Configuração de nós em Json

{
    "browserName": "operablink",
    "platform": "WINDOWS",
    "maxInstances": 5,
    "seleniumProtocol": "WebDriver"
}

O tipo de plataforma pode ser um dos seguintes:

  • PlatformType.Android;
  • PlatformType.Any;
  • PlatformType.Linux;
  • PlatformType.Mac;
  • PlatformType.Unix;
  • PlatformType.Vista;
  • PlatformType.Windows;
  • PlatformType.WinNT;
  • PlatformType.XP;
