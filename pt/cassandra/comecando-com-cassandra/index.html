Instalação ou Configuração #

Instalação de nó único #

  1. Pré-instale NodeJS, Python e Java
  2. Selecione seu documento de instalação com base em sua plataforma http://docs.datastax.com/en/cassandra/3.x/cassandra/install/installTOC.html
  3. Baixe os binários do Cassandra em http://cassandra.apache.org/download/
  4. Descompacte o arquivo baixado em <local de instalação>
  5. Inicie o cassandra usando <local de instalação>/bin/cassandra OU inicie o Cassandra como um serviço - [sudo] service cassandra start
  6. Verifique se o cassandra está funcionando usando <local de instalação>/bin/nodetool status.

Ex:

  1. No ambiente Windows execute o arquivo cassandra.bat para iniciar o servidor Cassandra e cqlsh.bat para abrir o terminal do cliente CQL para executar comandos CQL.

Existem duas maneiras de realizar a instalação de um Single Node.

Você deve ter Oracle Java 8 ou OpenJDk 8 (preferido para versões do Cassandra > 3.0)

1. Instalando um pacote Debian (instala o Cassandra como um serviço) #

Adicione a versão do Cassandra ao repositório (substitua o 22x por sua própria versão por exemplo para 2.7 use 27x)

echo "deb-src http://www.apache.org/dist/cassandra/debian 22x main" | sudo tee -a /etc/apt/sources.list.d/cassandra.sources.list
# Update the repository  
sudo apt-get update
# Then install it
sudo apt-get install cassandra cassandra-tools

Agora o Cassandra pode ser iniciado e interrompido usando:

sudo service cassandra start
sudo service cassandra stop

Verifique o status usando:

nodetool status

Os diretórios de logs e dados são /var/log/cassandra e /var/lib/cassandra respectivamente.

2. Instalando qualquer versão do Cassandra na forma de tarball binário (instala o Cassandra como um processo autônomo) #

Baixe a versão do Datastax:

curl -L  http://downloads.datastax.com/community/dsc-cassandra-version_number-bin.tar.gz | tar xz

Ou Apache Cassandra binário tarball manualmente (do site http://www.apache.org/dist/cassandra/)

Agora descompacte isso:

tar -xvzf dsc-cassandra-version_number-bin.tar.gz

Altere o diretório para o local de instalação:

cd install_location

Inicie o Cassandra usando:

sudo sh ./bin/cassandra

Pare de usar:

sudo kill -9 pid

Verificar:

./bin/nodetool status

E viola, você tem um cluster de teste de nó único para Cassandra. Então, basta usar cqlsh no terminal para o shell Cassandra.

A configuração do Cassandra pode ser feita em cassandra.yaml na pasta conf em install_location.
