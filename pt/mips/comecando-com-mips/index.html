Simulador de MIPS de MARTE #

O simulador MARS MIPS é um editor de linguagem assembly, montador, simulador e depurador para o processador MIPS, desenvolvido por Pete Sanderson e Kenneth Vollmar na Missouri State University (src).

Você obtém o MARS de graça aqui. Quanto à instalação da versão 4.5, você pode precisar do Java SDK adequado para o seu sistema aqui

Antes de montar, o ambiente deste simulador pode ser dividido de forma simplista para três segmentos: o editor no canto superior esquerdo onde todo o código está sendo escrito, o compilador/saída logo abaixo do editor e a lista de registradores que representam a “CPU” do nosso programa.

Após a montagem (simplesmente pressionando F3) o ambiente muda, com dois novos segmentos recebendo a posição do editor: o segmento de texto onde

i) cada linha de código assembly é limpa de “pseudoinstruções” (falaremos sobre elas em um segundo) na coluna “básica” e

ii) o código de máquina para cada instrução na coluna “código”,

e o segmento de dados onde podemos ver uma representação da memória de um processador com ordem little-endian.

Após a montagem, podemos executar nosso código de uma só vez (F5) ou passo a passo (F7), além de retroceder a execução várias etapas para trás (F8).

Agora, vamos ver o código de exemplo acima e explicar cada linha:

.text
.globl main
main:            #main function

li    $v0, 11    #11=system code for printing a character, $v0=register that gets the system code for printing as value
la    $a0, 'a'   #'a'=our example character, $a0=register that accepts the character for printing
syscall          #Call to the System to execute our instructions and print the character at the a0 register 

li $v0, 10       #11=system code for terminating, $v0=register that gets the system code for terminating (optional, but desirable)
syscall          #Call to the System to terminate the execution

MARS aceita e exporta arquivos com o tipo de arquivo .asm

Mas o código acima imprime apenas um caractere, e o bom e velho “Hello World”? Que tal, não sei, adicionar um número ou algo assim? Bem, podemos mudar o que tínhamos um pouco para isso:

.data               #data section 
str: .asciiz "Hello world\n"  
number: .word 256
 
.text                 #code section 
.globl main 
main: 
li       $v0, 4                #system call for printing strings 
la       $a0, str              #loading our string from data section to the $a0 register
syscall  
       
la       $t0, number        #loading our number from data section to the $t0 register
lw       $s1, 0($t0)        #loading our number as a word to another register, $s1 

addi     $t2, $s1, 8         #adding our number ($s1) with 8 and leaving the sum to register $t2

sw       $t2, 0($t0)        #storing the sum of register $t2 as a word at the first place of $t0

li       $v0, 10               # system call for terminating the execution
syscall

Antes de ilustrar os resultados através do MARS, é necessário um pouco mais de explicação sobre esses comandos:

  • Chamadas do sistema são um conjunto de serviços fornecidos pelo sistema operacional. Para usar uma chamada de sistema, um código de chamada é necessário para ser colocado $v0 registrar para a operação necessária. Se uma chamada de sistema tiver argumentos, esses são colocados nos registradores $a0-$a2. Aqui são todas as chamadas do sistema.

  • li (load imediato) é uma pseudo-instrução (falaremos sobre isso depois) que carrega instantaneamente um registrador com um valor. la (carregar address) também é uma pseudo-instrução que carrega um endereço para um registro. Com li $v0, 4 o registrador $v0 agora tem 4 como value, enquanto la $a0, str carrega a string de str para o registro $a0.

  • Uma palavra é (por mais que estejamos falando de MIPS) um 32 bits sequência, com o bit 31 sendo o Bit Mais Significativo e o bit 0 sendo o Bit Menos Significativo.

  • lw (load word) transfere da memória para um registrador, enquanto sw (palavra de armazenamento) é transferida de um registrador para a memória. Com o lw comando $s1, 0($t0), carregamos para $s1 registrar o valor que foi no LSB do registrador $t0 (é o que o 0 simboliza aqui, o deslocamento da palavra), também conhecido como 256. $t0 aqui tem o endereço, enquanto $s1 tem o valor. sw $t2, 0($t0) faz exatamente o oposto trabalho.

  • MARS usa o Little Endian, o que significa que o LSB de uma palavra é armazenado no menor endereço de byte da memória.

  • O MIPS usa endereços de byte, portanto, um endereço é separado de seu anterior e a seguir por 4.

    .text
main:
    li    $s0,0x30
loop:
    move    $a0,$s0        # copy from s0 to a0
    
    li    $v0,11        # syscall with v0 = 11 will print out
    syscall            # one byte from a0 to the Run I/O window

    addi    $s0,$s0,3    # what happens if the constant is changed?
    
    li    $t0,0x5d
    bne    $s0,$t0,loop
    nop            # delay slot filler (just in case)

stop:    j    stop        # loop forever here
    nop            # delay slot filler (just in case)

Pressione F3 para montá-lo e, em seguida, pressione executar. Agora você começou a compilar e executar o código MIPS.

Instruções detalhadas sobre como configurar ou instalar mips.

QtSpim para windows #

  1. baixe o QtSpim aqui 32,6 MB
  2. instale fácil instalação
  3. crie seu primeiro arquivo de montagem (.s) ou use o exemplo C:\Program Files (x86)\QtSpim\helloworld.s
  4. execute o programa a partir do atalho da área de trabalho ou C:\Program Files (x86)\QtSpim\QtSpim.exe

existem duas janelas para o programa a principal rotulada QtSpim aqui você vê o programa que está executando (rotulado texto), a memória (rotulado dados), os valores dos registradores (rotulado FP Regs para ponto flutuante e Int Regs para inteiro) e o controle para o simulador

a outra janela chamada console é onde você verá a saída e inserirá a entrada do seu programa se houver alguma

  1. carregue o arquivo usando Arquivo -> Carregar arquivo
  2. você pode usar click run (f5) para ver o resultado final ou ir passo a passo (p10) para ver o estado do registrador e memória enquanto o programa está executando para depurar
