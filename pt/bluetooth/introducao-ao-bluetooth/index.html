Perfis #

A especificação Bluetooth contém várias especificações de perfil. Um perfil descreve como usar e implementar uma função.

Eles podem depender um do outro, aqui está um layout básico das dependências de perfil mais comuns

Todos os perfis podem ser encontrados em BT SIG, esteja ciente de que versões diferentes podem conter funcionalidades diferentes. Observe também que alguns dos perfis contêm várias categorias, às vezes opcionais, portanto, certifique-se de que seu dispositivo seja compatível com a categoria em questão. Aqui estão alguns dos perfis de smartphone mais comuns e suas especificações

A2DP - Perfil avançado de distribuição de áudio
O Advanced Audio Distribution Profile (A2DP) define os protocolos e procedimentos que realizam a distribuição de conteúdo de áudio de alta qualidade em mono ou estéreo em canais ACL. Um caso de uso típico é o streaming de conteúdo de música de um player de música estéreo para fones de ouvido ou alto-falantes. Os dados de áudio são compactados em um formato adequado para uso eficiente da largura de banda limitada.
Dependências: GAVDP, GAP

AVRCP - Perfil de controle remoto de áudio/vídeo
O perfil de controle remoto de áudio/vídeo (AVRCP) define os recursos e procedimentos necessários para garantir a interoperabilidade entre dispositivos Bluetooth com funções de controle de áudio/vídeo nos cenários de distribuição de áudio/vídeo. Este perfil adota o modelo de dispositivo AV/C e formato de comando para mensagens de controle, e essas mensagens são transportadas pelo Protocolo de Transporte de Controle de Áudio/Vídeo (AVCTP). Nesse perfil, o controlador traduz a ação do usuário detectada para o sinal de controle A/V e, em seguida, o transmite para um dispositivo Bluetooth remoto. Além disso, o perfil usa extensões específicas do Bluetooth para suportar a transferência de metadados relacionados ao conteúdo a ser transferido entre dispositivos Bluetooth. O controle remoto descrito neste perfil foi projetado especificamente para controle A/V.
Dependências: GAP

HFP - Perfil mãos-livres
Este documento define os protocolos e procedimentos que devem ser usados ​​pelos dispositivos que implementam o Perfil Hands-Free. Os exemplos mais comuns de tais dispositivos são as unidades Hands-Free no carro usadas em conjunto com telefones celulares ou fones de ouvido sem fio vestíveis. O perfil define como dois dispositivos que suportam o Perfil Hands-Free devem interagir entre si ponto a ponto. Uma implementação do Perfil Mãos Livres normalmente permite que um fone de ouvido ou uma unidade viva-voz incorporada se conecte, sem fio, a um telefone celular com a finalidade de atuar como mecanismo de entrada e saída de áudio do telefone celular e permitir que funções típicas de telefonia sejam realizado sem acesso ao telefone real.
Dependências: SPP, GAP

HSP - Perfil do fone de ouvido
Este perfil de fone de ouvido define os protocolos e procedimentos que devem ser usados ​​por dispositivos que exigem uma conexão de áudio full-duplex combinada com comandos mínimos de controle do dispositivo. Os exemplos mais comuns de tais dispositivos são fones de ouvido, computadores pessoais, PDAs e telefones celulares, embora a maioria dos telefones celulares prefira usar um perfil mais avançado, como o Perfil Hands-Free. O fone de ouvido pode ser conectado sem fio para atuar como mecanismo de entrada e saída de áudio do dispositivo, fornecendo áudio full duplex.
Dependências: SPP, GAP

PBAP - Perfil de acesso à agenda
O Phone Book Access Profile (PBAP) define os protocolos e procedimentos que devem ser usados ​​pelos dispositivos para a recuperação de objetos da lista telefônica. Ele é baseado em um modelo de interação Cliente-Servidor em que o dispositivo Cliente extrai objetos do catálogo telefônico do dispositivo Servidor. Este perfil é especialmente adaptado para o caso de uso Hands-Free (ou seja, implementado em combinação com o “Hands-Free Profile” ou o “SIM Access Profile”). Ele fornece vários recursos que permitem o manuseio avançado de objetos de lista telefônica, conforme necessário no ambiente do carro. Em particular, é muito mais rico do que o Object Push Profile (que pode ser usado para enviar a entrada do catálogo telefônico formatado com vCard de um dispositivo para outro). Esse perfil também pode ser aplicado a outros casos de uso em que um dispositivo Cliente deve extrair objetos de catálogo telefônico de um dispositivo Servidor. Observe, no entanto, que este perfil só permite a consulta de objetos da lista telefônica (somente leitura). Não é possível alterar o conteúdo do objeto original da lista telefônica (leitura/gravação).
Dependências: GOEP, SPP, GAP
