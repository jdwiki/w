Sobre o Laravel #

Criado por Taylor Otwell como um PHP web framework, Laravel de código aberto gratuito, tem como objetivo facilitar e acelerar o processo de desenvolvimento de aplicativos da web com um ótimo gosto pela simplicidade.

Ele segue o padrão de arquitetura model–view–controller (MVC), bem como o padrão de codificação PSR-2 e o padrão de carregamento automático PSR-4.

Executar um Desenvolvimento Orientado a Testes (TDD) em Laravel é divertido e fácil de implementar.

Hospedado no GitHub e disponível em https://github.com/laravel/laravel, o Laravel possui uma arquitetura de microsserviços, tornando-o tremendamente extensível e isso, com facilidade, com o uso de pacotes personalizados e/ou de terceiros existentes.

Principais características #

MVC #

O Laravel usa o modelo MVC, portanto, existem três partes principais do framework que trabalham juntas: models, views e controllers. Os controladores são a parte principal onde a maior parte do trabalho é feito. Eles se conectam a modelos para obter, criar ou atualizar dados e exibir os resultados em visualizações, que contêm a estrutura HTML real do aplicativo.

Mecanismo de modelagem de lâminas #

O Laravel é fornecido com um mecanismo de modelagem conhecido como Blade. Blade é bastante fácil de usar, mas poderoso. Uma característica que o mecanismo de modelagem Blade não compartilha com outros populares é sua permissividade; permitindo o uso de código PHP simples em arquivos do mecanismo de modelagem Blade.

É importante notar que os arquivos do mecanismo de modelagem Blade têm .blade anexado aos nomes dos arquivos logo antes do .php usual, que nada mais é do que a extensão real do arquivo. Como tal, .blade.php é a extensão de arquivo resultante para arquivos de modelo Blade. Os arquivos do mecanismo de modelo blade são armazenados no diretório resources/views.

Roteamento e middleware #

Você pode definir as URLs do seu aplicativo com a ajuda de rotas. Essas rotas podem conter dados variáveis, conectar-se a controladores ou podem ser agrupadas em middlewares. Middelware é um mecanismo para filtrar solicitações HTTP. Eles podem ser usados ​​para interagir com solicitações antes que elas cheguem aos controladores e, assim, podem modificar ou rejeitar solicitações.

Artesão #

Artisan é a ferramenta de linha de comando que você pode usar para controlar partes do Laravel. Existem muitos comandos disponíveis para criar modelos, controladores e outros recursos necessários para o desenvolvimento. Você também pode escrever seus próprios comandos para estender a ferramenta de linha de comando Artisan.

ORM Eloquente #

Para conectar seus modelos a vários tipos de bancos de dados, o Laravel oferece seu próprio ORM com um grande conjunto de funções para trabalhar. A estrutura também fornece migração e propagação e também apresenta reversões.

Manipulação de eventos #

A estrutura é capaz de manipular eventos em todo o aplicativo. Você pode criar ouvintes de eventos e manipuladores de eventos semelhantes aos do NodeJs.
