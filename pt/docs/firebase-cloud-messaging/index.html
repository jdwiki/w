Tutorial do firebase-cloud-messaging

Uma pergunta comum notória é “como enviar notificações de dispositivo para dispositivo”, infelizmente a resposta é: você não pode. O FCM precisa ser acionado para enviar notificações push. Isso pode ser feito de 3 maneiras diferentes:

  1. Diretamente no console da Web do Firebase
  2. Definir um listener do Firebase Functions e acionar o FCM
  3. Um servidor solicita ao FCM para enviar uma notificação por push

Uma notificação por push é uma carga útil de informações enviada do FCM. Existem 3 tipos de notificações push: notificação, dados, notificação e dados. Esta informação pode ser representada como um JSON:

  {
    "to" : "APA91bHun4MxP5egoKMwt2KZFBaFUH-1RYqx...",
    "notification" : {
      "body" : "great match!",
      "title" : "Portugal vs. Denmark",
      "icon" : "myicon"
    },
    "data" : {
      "Nick" : "Mario",
      "Room" : "PortugalVSDenmark"
    }
  }

O exemplo acima é para o terceiro tipo, notification e data combinados. É isso que será pedido ao FCM para enviar.

  1. O console pode enviar notification e notification com data, mas nunca apenas com data
  2. Funções e qualquer Servidor podem enviar os 3 tipos

A importância do tipo notification é que permite que os aplicativos recebam pushes padrão, capacitando outras equipes, como marketing, a aumentar o crescimento de aplicativos simplesmente usando o console da web sem necessidade de codificação adicional além de adicionar a biblioteca ao projeto.

Por favor, não confunda a notificação push, o tipo notificação e a notificação visual, esta última corresponde a uma classe Android (comumente NotificationCompat).

O comportamento do push é diferente de acordo com o tipo e se o aplicativo está em primeiro plano ou em segundo plano. Não em primeiro plano significa, minimizado ou fechado.

  1. notification acionará uma notificação visual padrão se o aplicativo não estiver em primeiro plano, esta notificação pode ser personalizada no manifesto, consulte documentação. Se o aplicativo estiver em primeiro plano, temos que personalizar o comportamento dentro do método onMessageReceived.
  2. O comportamento do tipo data deve sempre ser personalizado.
  3. A combinação de notificação e dados se o aplicativo não estiver em primeiro plano acionará a notificação visual padrão e a carga útil de dados estará disponível quando o usuário clicar. Como a Activity do launcher é acionada quando a notificação visual é clicada, então você tem que literalmente getIntent().getStringExtra("yourKey"); nessa Activity para obter os dados. Se o aplicativo estiver ativo (em primeiro plano), você deve personalizar o comportamento dentro do método onMessageReceived e obter acesso à carga útil data imediatamente.

Para obter o payload da informação você deve fazê-lo dentro do método onMessageReceived, onde o único argumento disponível é a mensagem:

  1. Para obter a notificação você tem que remoteMessage.getNotification() então você pode obter o corpo ou título com os métodos correspondentes
  2. Para obter os dados você tem que remoteMessage.getData().get("yourKey").

É uma boa ideia adicionar todas as verificações not null, haverá vários tipos de notificações chegando em aplicativos avançados. Uma boa estratégia é verificar se cada um, a notificação e os dados não são nulos. Uma estratégia útil consequente será sempre usar uma chave type nas notificações data para fazer algum controle de fluxo.

Para enviar data do console da Web do Firebase, as opções avançadas devem ser abertas.

As chaves de notificação são limitadas e indicadas na documentação. Os valores em qualquer tipo podem ser apenas String.

Se você tiver problemas para encontrar qualquer documentação no Firebase, vá até o final da página e altere o idioma para “inglês”, as documentações são mais finas em alguns outros idiomas.
