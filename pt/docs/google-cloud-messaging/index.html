Tutorial do google-cloud-messaging

Google Cloud Messaging: visão geral

O Google Cloud Messaging (GCM) é um serviço gratuito que permite aos desenvolvedores enviar mensagens entre servidores e aplicativos clientes. Isso inclui mensagens downstream de servidores para aplicativos cliente e mensagens upstream de aplicativos cliente para servidores.

Por exemplo, uma mensagem downstream leve pode informar a um aplicativo cliente que há novos dados a serem buscados no servidor, como no caso de uma notificação de “novo email”. Para casos de uso como mensagens instantâneas, uma mensagem do GCM pode transferir até 4 KB de carga útil para o aplicativo cliente. O serviço GCM lida com todos os aspectos de enfileiramento de mensagens e entrega de e para o aplicativo cliente de destino.

Visão geral da arquitetura

Uma implementação do GCM inclui um servidor de conexão do Google, um servidor de aplicativos em seu ambiente que interage com o servidor de conexão por meio do protocolo HTTP ou XMPP e um aplicativo cliente.

Veja como esses componentes interagem:

  • Os GCM Connection Servers do Google aceitam mensagens downstream do seu servidor de aplicativos e as enviam para um aplicativo cliente. O servidor de conexão XMPP também pode aceitar mensagens enviadas upstream do aplicativo cliente e encaminhá-las para o servidor do aplicativo. Para obter mais informações, consulte Sobre o GCM Connection Server.
  • Em seu Servidor de aplicativos, você implementa o HTTP e/ou [XMPP](https://developers.google. com/cloud-messaging/ccs) para se comunicar com o(s) servidor(es) de conexão GCM. Os servidores de aplicativos enviam mensagens downstream para um servidor de conexão GCM; o servidor de conexão enfileira e armazena a mensagem e a envia para o aplicativo cliente. Se você implementar o XMPP, seu servidor de aplicativos poderá receber mensagens enviadas do aplicativo cliente.
  • O Aplicativo cliente é um aplicativo cliente habilitado para GCM. Para receber e enviar mensagens do GCM, este aplicativo deve se registrar no GCM e obter um identificador exclusivo chamado token de registro. Para obter mais informações sobre como implementar o aplicativo cliente, consulte a documentação da sua plataforma.

Conceitos chave

Abaixo resume os principais termos e conceitos envolvidos no GCM. Ele é dividido nessas categorias:

  • Componentes — As entidades que desempenham um papel principal no GCM.
  • Credenciais — Os IDs e tokens usados ​​no GCM para garantir que todas as partes tenham sido autenticadas e que a mensagem esteja indo para o local correto.

Componentes e credenciais do GCM.

Componentes

  • GCM Connection Servers - Servidores do Google envolvidos no envio de mensagens entre o servidor do aplicativo e o aplicativo cliente.
  • Aplicativo cliente - Um aplicativo cliente habilitado para GCM que se comunica com seu servidor de aplicativos.
  • App Server - Um servidor de aplicativos que você escreve como parte da implementação do GCM. O servidor de aplicativos envia dados para um aplicativo cliente por meio do servidor de conexão GCM. Se o seu servidor de aplicativos implementar o protocolo XMPP, ele também poderá receber mensagens enviadas upstream de aplicativos clientes.

Credenciais

  • ID do remetente
    Um valor numérico exclusivo criado ao configurar seu projeto de API. O ID do remetente é usado no processo de registro para identificar um servidor de aplicativos com permissão para enviar mensagens ao aplicativo cliente.

  • Chave do servidor
    Uma chave salva no servidor de aplicativos que dá ao servidor de aplicativos acesso autorizado aos serviços do Google. No HTTP, a chave do servidor é incluída no cabeçalho das solicitações POST que enviam mensagens. No XMPP, a chave do servidor é usada na solicitação de autenticação SASL PLAIN como uma senha para autenticar a conexão. Não inclua a chave do servidor em nenhum lugar no código do cliente. Você obtém a chave do servidor ao criar seu projeto de API.

  • ID do aplicativo O aplicativo cliente que está se registrando para receber mensagens. Como isso é implementado depende da plataforma:

  • Android: use o nome do pacote do manifesto do aplicativo.

  • iOS: use o identificador de pacote do aplicativo.

  • Chrome: use o nome da extensão do Chrome.

  • Token de registro
    Um ID emitido pelos servidores de conexão do GCM para o aplicativo cliente que permite receber mensagens. Observe que os tokens de registro devem ser mantidos em segredo.

Fluxo do ciclo de vida

  • Registre-se para ativar o GCM. Uma instância de um aplicativo cliente se registra para receber mensagens. Para mais discussões, consulte Registrar aplicativos cliente.
  • Enviar e receber mensagens downstream.
  • Envie uma mensagem. O servidor de aplicativos envia mensagens para o aplicativo cliente:
    1. The app server sends a message to GCM connection servers.
    2. The GCM connection server enqueues and stores the message if the device is offline.
    3. When the device is online, the GCM connection server sends the message to the device.
    4. On the device, the client app receives the message according to the platform-specific implementation. See your platform-specific documentation for details.
  • Receber uma mensagem. Um aplicativo cliente recebe uma mensagem de um servidor de conexão do GCM. Consulte a documentação específica da plataforma para obter detalhes sobre como um aplicativo cliente nesse ambiente processa as mensagens que recebe.
  • Enviar e receber mensagens upstream. Esse recurso só está disponível se você estiver usando o servidor de conexão XMPP.
  • Envie uma mensagem. Um aplicativo cliente envia mensagens para o servidor de aplicativos:
    1. On the device, the client app sends messages to the XMPP connection server. See your platform-specific documentation for details on how a client app can send a message via XMPP.
    2. The XMPP connection server enqueues and stores the message if the server is disconnected.
    3. When the app server is re-connected, the XMPP connection server sends the message to the app server.
  • Receber uma mensagem. Um servidor de aplicativos recebe uma mensagem do servidor de conexão XMPP e faz o seguinte:
    1. Parses the message header to verify client app sender information.
    2. Sends “ack” to the XMPP connection server to acknowledge receiving the message.
    3. Optionally parses the message payload, as defined by the client app.
