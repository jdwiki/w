Tutorial de recursão

Recursão refere-se a algo sendo definido em termos de si mesmo. No contexto da programação, é a prática de usar funções que chamam a si mesmas ou tipos recursivos.

Funções recursivas #

Existem duas partes para uma função recursiva:

  • Um ou mais casos base
  • Um passo recursivo

Como uma função recursiva chama a si mesma, a sequência de chamadas de função pode durar para sempre. Para garantir que a função termine, os casos base devem ser incluídos. Estes dirão à função quando retornar um valor em vez de realizar outra chamada recursiva. Normalmente, as funções recursivas verificarão suas entradas em relação ao caso base e retornarão se o caso base for atingido. Se o caso base não for alcançado, a função prosseguirá para a etapa recursiva.

A etapa recursiva é quando uma função chama a si mesma. Muitas funções recursivas retornarão algum cálculo realizado no resultado da chamada recursiva. O exemplo da sequência de Fibonacci demonstra isso. Essa função chama a si mesma recursivamente duas vezes e depois retorna a soma dos resultados dessas chamadas

Um uso comum da recursão é navegar e realizar cálculos em grandes estruturas de dados, como árvores e gráficos. Isso funciona bem porque a recursão divide o problema para resolver problemas menores e se baseia nessas soluções para resolver o problema maior. Outro uso comum para a recursão é realizar operações iterativas (loop) em linguagens (tipicamente linguagens funcionais) que não possuem estruturas de loop integradas.

Tipos Recursivos #

Um tipo recursivo é aquele cujos valores são compostos de valores do mesmo tipo, ou expressos de forma diferente: um tipo recursivo é definido em termos de si mesmo. Por exemplo, uma lista é um tipo recursivo, pois cada subconjunto da lista é uma lista. As árvores são outro exemplo, à medida que os nós são removidos de uma árvore, a construção restante ainda é uma árvore.

Formalmente, o conjunto de valores de um tipo recursivo, T, será definido por uma equação de conjunto recursiva na forma:

T = … T …

Uma equação de conjunto recursiva pode ter muitas soluções, mas tal equação sempre tem uma solução mínima que é um subconjunto de todas as outras soluções.

Como exemplo prático, considere esta definição de Haskell para um tipo de lista,

data List a = Nil | Cons a (List a)

significando que uma lista de a’s é uma lista vazia ou uma célula de contras contendo um ‘a’ (a “cabeça” da lista) e outra lista (a “cauda”).
