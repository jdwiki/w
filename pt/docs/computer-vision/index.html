tutorial de visão computacional

Processamento Digital de Imagens e Visão Computacional é um campo interessante, pois está muito bem localizado entre a Matemática e a Ciência da Computação. Portanto, é muito útil entender os fundamentos e aplicá-los usando programação para entender o tópico.

As imagens digitais são discretização de sinais de 2 ou 3 dimensões. Em outras palavras, imagens digitais são conjuntos amostrados de pixels ou voxels de domínios contínuos.

                                        f : R² ⊃ Ω → R

Onde o f é a imagem digital sobre Ω : Domínio de imagem retangular

Por uma questão de simplicidade, discutiremos apenas imagens digitais bidimensionais, como as de nossos avatares do StackOverflow.

Sobre pixels: Uma nota rápida sobre valores de pixels antes de começarmos a discutir sobre os tipos de imagens. Como regra geral, os pixels começam com o valor 0, que denota ausência de luz (preto), atinge 1, intensidade máxima (por exemplo, branco) e são representados em números inteiros.

Imagens binárias: Imagens somente em preto e branco. Cada pixel é 0 ou 1, cada pixel pode ser representado por um bit. Eles não são muito conhecidos popularmente, pois geralmente são usados ​​em aplicações científicas ou para outras operações de processamento de imagens como máscara por exemplo.

Um exemplo de imagem binária. (Aviso que os valores de pixel de imagem deste arquivo não são necessariamente binários, isso é para demonstração, também esta é Lena, a estrela do mundo de Processamento de Imagens)

Imagens em tons de cinza: graças aos filtros online, todo mundo ainda conhece muito bem essas imagens. Estas imagens são geralmente um byte por pixel, sendo 0 preto e 255 branco, tudo entre é um tom diferente de cinza, dado que os humanos só podem distinguir 40 tons de cinza, esta faixa é suficiente para muitas aplicações (Observe que os valores de pixels aqui são mapeados de 0 a 1 para valores de byte 0 - 255)

Imagens coloridas: Por fim, o tipo de imagem digital mais comum, imagens coloridas. Temos que mencionar o conceito de canais aqui. As imagens digitais também possuem canais, na verdade, as imagens binárias e em tons de cinza descritas acima também possuem canais. A descrição mais comum seria o modelo RGB (Vermelho-Verde-Azul), nesse caso a imagem tem 3 canais (não confunda com dimensões, estas ainda são imagens 2D) para descrever a vermelhidão, azul e verde da imagem. Neste caso, cada pixel é um tripleto, um valor entre 0 - 255 (Sem vermelho para mais vermelho), 0 - 255 (Sem verde para mais verde), 0 - 255 (Sem azul para mais azul). Para este modelo, o pixel {0,0,0} é preto, {255,255,255} é branco, {255,0,0} é vermelho, {255, 255, 0} é amarelo. No entanto, a cor é um tópico muito amplo, e você pode verificar as referências para obter mais informações.

Imagens hiperespectrais:

Depois de discutirmos os canais, é mais fácil falar sobre imagens hiperespectrais. Essas imagens podem ter centenas de canais e são comumente usadas em microscopia, imagens de satélite etc.
