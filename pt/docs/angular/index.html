Tutorial Angular

Angular (comumente referido como “Angular 2+” ou “Angular 2”) é uma estrutura da Web de front-end de código aberto baseada em TypeScript liderada pelo Equipe Angular no Google e por uma comunidade de indivíduos e empresas para abordar todas as partes do fluxo de trabalho do desenvolvedor ao criar aplicativos da Web complexos. Angular é uma reescrita completa da mesma equipe que construiu AngularJS. ¹

A estrutura consiste em várias bibliotecas, algumas delas principais (@angular/core por exemplo) e algumas opcionais (@angular/animations).

Você escreve aplicativos Angular compondo HTML templates com marcação Angularized, escrevendo component classes para gerenciar esses modelos, adicionando lógica de aplicativo em services e boxing componentes e serviços em [modules] 11.

Em seguida, você inicia o aplicativo bootstrapping pelo módulo raiz. O Angular assume o controle, apresentando o conteúdo do seu aplicativo em um navegador e respondendo às interações do usuário de acordo com as instruções fornecidas.

Indiscutivelmente, a parte mais fundamental do desenvolvimento de aplicativos Angular são os componentes. Um componente é a combinação de um modelo HTML e uma classe de componente que controla uma parte da tela. Aqui está um exemplo de um componente que exibe uma string simples:

src/app/app.component.ts

importe { Componente } de ‘@angular/core’;

@Component({ seletor: ‘meu-aplicativo’, modelo: <h1>Olá {{name}}</h1> }) exportar classe AppComponent { nome = ‘Angular’; }

Cada componente começa com uma função decoradora @Component que recebe um objeto metadados. O objeto de metadados descreve como o modelo HTML e a classe de componente funcionam juntos.

A propriedade selector diz ao Angular para exibir o componente dentro de uma tag <my-app> personalizada no arquivo index.html.

index.html (dentro da tag body)

Carregando conteúdo do AppComponent aqui…

A propriedade template define uma mensagem dentro de um cabeçalho <h1>. A mensagem começa com “Hello” e termina com {{name}}, que é uma expressão Angular interpolation binding. Em tempo de execução, o Angular substitui {{name}} pelo valor da propriedade name do componente. A vinculação de interpolação é um dos muitos recursos do Angular que você descobrirá nesta documentação. No exemplo, altere a propriedade name da classe do componente de 'Angular' para 'World' e veja o que acontece.

Este exemplo foi escrito em TypeScript, um superconjunto de JavaScript. O Angular usa o TypeScript porque seus tipos facilitam o suporte à produtividade do desenvolvedor com ferramentas. Além disso, quase todo o suporte é para TypeScript e, portanto, usar JavaScript simples para escrever seu aplicativo será difícil. No entanto, escrever código Angular em JavaScript é possível; este guia explica como.
