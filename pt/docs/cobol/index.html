COBOL é a linguagem COmmon **Business Oorientada para programação L.

Embora tenha se tornado um nome pronunciável, COBOL ainda é tratado como um acrônimo pelo comitê de padrões, e COBOL é a grafia preferida pelos órgãos de padrões ISO e INCITS.

Especificação padrão #

A especificação atual é

ISO/IEC 1989:2014 Tecnologia da informação – Linguagens de programação, seus ambientes e interfaces de software do sistema – Linguagem de programação COBOL

Esse documento foi publicado em maio de 2014 e pode ser adquirido em diversas filiais de órgãos padrão, oficialmente sediados em

http://www.iso.org/iso/home/store/catalogue_tc/catalogue_detail.htm?csnumber=51416

Campo principal de uso #

Orientado à Negócios. Isso geralmente significa processamento de transações. Bancos, agências governamentais e o setor de seguros são as principais áreas de implantação de aplicativos COBOL. Os sistemas de mainframe IBM geralmente têm um compilador COBOL instalado. Existem mais de 300 dialetos COBOL, com talvez 10 ou mais versões tomando a maior parte das implantações. A maioria desses compiladores são sistemas proprietários, mas o software livre COBOL também está disponível.

Categoria #

COBOL é uma linguagem de programação processual, imperativa e compilada. A partir da especificação COBOL 2002, recursos Orientados a Objetos foram adicionados ao padrão.

Por intenção de design, COBOL é uma linguagem de programação muito detalhada. Embora a forma algébrica seja permitida:

COMPUTE I = R * B

a intenção inicial era usar palavras completas para descrições computacionais e manipulação de dados:

MULTIPLY INTEREST-RATE BY BALANCE GIVING CURRENT-INTEREST ROUNDED MODE IS NEAREST-EVEN

Esta decisão de design tem tanto defensores quanto detratores. Alguns acham que é muito detalhado, enquanto outros argumentam que a sintaxe permite maior legibilidade em um ambiente de negócios.

Matemática Decimal #

COBOL é projetado em torno da aritmética decimal, ao contrário da maioria das linguagens que usam uma representação interna binária. A especificação COBOL exige cálculos decimais de ponto fixo muito precisos, um aspecto da linguagem que tem sido bem visto nos setores financeiros. COBOL também permite USAGE BINARY, mas tende para representações decimais (base-10).

História #

COBOL remonta ao final da década de 1950, com implementações iniciais publicadas em 1960.

A contra-almirante da Marinha dos EUA Grace Hopper é frequentemente associada ao COBOL e defendida em nome da linguagem durante os estágios iniciais de desenvolvimento. Ela não foi a única pessoa envolvida no design e desenvolvimento do COBOL, mas é muitas vezes referida como a Mãe do COBOL.

Devido ao apoio inicial de governos e grandes corporações, o COBOL tem sido amplamente utilizado por muitas décadas. Continua sendo motivo de orgulho para alguns e um espinho para outros, que o consideram ultrapassado. A verdade provavelmente está em algum lugar entre essas visões extremas. Quando aplicado ao processamento de transações, o COBOL está em casa. Quando aplicado a telas da Web modernas e aplicativos de rede, pode não parecer tão confortável.

Estrutura #

Os programas COBOL são escritos em quatro divisões separadas.

  • DIVISÃO DE IDENTIFICAÇÃO
  • DIVISÃO DE MEIO AMBIENTE
  • DIVISÃO DE DADOS
  • DIVISÃO DE PROCEDIMENTOS

Descrições de dados #

Sendo projetado para lidar com dados decimais, o COBOL permite descrições de dados baseadas em PICTURE, em hierarquias agrupadas.

01 record-group.
   05 balance        pic s9(8)v99.
   05 rate           pic 999v999.
   05 show-balance   pic $Z(7)9.99.

Isso define “saldo” como um valor de oito dígitos com sinal com dois dígitos assumidos após o ponto decimal. rate é três dígitos antes e três dígitos depois de um ponto decimal assumido. show-balance é um campo de edição numérica que terá um cifrão inicial, sete dígitos (zero suprimido) com pelo menos um dígito mostrado antes de dois dígitos após um ponto decimal.

balance pode ser usado em cálculos, show-balance é apenas para fins de exibição e não pode ser usado em instruções computacionais.

Declarações processuais #

COBOL é uma linguagem pesada de palavras-chave reservadas. MOVER, COMPUTE, MULTIPLICAR, PERFORM, palavras de formato longo compõem a maior parte da especificação padrão. Mais de 300 palavras-chave e 47 instruções operacionais na especificação COBOL 2014. Muitas implementações de compiladores adicionam ainda mais à lista de palavras reservadas.
