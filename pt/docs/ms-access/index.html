Tutorial de acesso ms

Desculpas pela parede de texto. Edições futuras provavelmente adicionarão capturas de tela e outros elementos visuais.

As consultas no Microsoft Access podem ser criadas por qualquer um dos quatro métodos

  1. Usando um construtor Query Wizard passo a passo através da GUI, que fará uma série de perguntas sobre quais dados você deseja exibir e como vários bits de dados estão relacionados entre si.
  2. Usando a GUI no Design View, na qual você seleciona tabelas e campos específicos com o mouse. A ordenação visual dos vários campos pode ser especificada arrastando as colunas relevantes no painel inferior e as propriedades adicionais para cada campo podem ser especificadas no painel Propriedades (direita).
  3. Mudando de Design View para SQL View e especificando o código de consulta SQL bruto. Na maioria das circunstâncias, você pode alternar livremente entre o Modo Design e o Modo SQL - o Access incorporará as alterações que você fez em um modo de exibição para o outro. A sintaxe do Access SQL é semelhante, mas não idêntica, àquela usada no MySQL, PostgreSQL, Oracle ou MS SQL Server (tSQL).
  4. Usando a programação Visual Basic for Applications que pode ser acessada através do grupo Macro da faixa Database Tools (Access 2007+). A manipulação do banco de dados ocorre por meio das bibliotecas ADO ou DAO e usa a mesma sintaxe do SQL View no aplicativo principal, com a exceção de que vários caracteres especiais precisam ser “escapados”. As consultas criadas por meio desse método não são acessíveis diretamente do painel de navegação, mas devem ser colocadas em uma função ou subprocedimento e acionadas por outros elementos (por exemplo, por um botão em um formulário) por meio de macros ou executadas diretamente na interface GUI do VBA.

A edição de um valor de registro de uma consulta no modo de folha de dados resultará em uma alteração no valor de registro subjacente, supondo que o campo de consulta não seja uma agregação ou concatenação de várias fontes de informações.

Formulários e Relatórios podem ser usados ​​para exibir informações de consultas em um formulário alternativo a uma visualização simples de “Folha de Dados” que parece semelhante a uma planilha no estilo Excel. Os formulários são direcionados para exibição na tela, enquanto os relatórios são direcionados para os impressos em papel.
