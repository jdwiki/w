tutorial webgl

WebGL é uma API de rasterização que geralmente é executada em sua GPU, oferecendo a capacidade de desenhar rapidamente gráficos 2D e 3D. O WebGL também pode ser usado para fazer cálculos em matrizes de dados.

WebGL é uma API de nível muito baixo. Em um nível básico, o WebGL é um mecanismo que executa 2 funções fornecidas pelo usuário na GPU. Uma função é chamada de vertex shader. O trabalho de um sombreador de vértice é calcular as posições dos vértices. Com base nas posições das saídas da função, o WebGL pode rasterizar vários tipos de primitivas, incluindo pontos, linhas ou triângulos. Ao rasterizar esses primitivos, ele chama uma segunda função fornecida pelo usuário chamada fragment shader. O trabalho de um sombreador de fragmento é calcular uma cor para cada pixel do primitivo que está sendo desenhado no momento.

Essas funções são escritas em uma linguagem chamada GLSL que é um pouco parecida com C/C++ e estritamente tipada.

Cabe ao programador fornecer essas funções para fazer o WebGL desenhar 2d, 3d ou computar algo. Quase todas as partes do WebGL tratam de configurar essas 2 funções e, em seguida, fornecer dados a elas.

Os dados podem ser fornecidos a partir de 4 fontes.

  • Uniformes

Uniformes são entradas para funções de sombreador muito parecido com parâmetros de função ou variáveis ​​globais. Eles são definidos uma vez antes de um sombreador ser executado e permanecem constantes durante a execução

  • Atributos

Os atributos fornecem dados apenas para sombreadores de vértice. Os atributos definem como extrair dados dos buffers. Por exemplo, você pode colocar posições, coordenadas normais e de textura em um buffer. Os atributos permitem que você diga ao WebGL como extrair esses dados de seus buffers e fornecê-los a um sombreador de vértice. Um vertex shader é chamado um número de vezes especificado pelo usuário chamando gl.drawArrays ou gl.drawElements e especificando uma contagem. Cada vez que o sombreador de vértice atual é chamado, o próximo conjunto de dados será extraído dos buffers especificados pelo usuário e colocado nos atributos

  • Texturas

Texturas são matrizes 2D de dados de até 4 canais. Mais comumente, esses 4 canais são vermelho, verde, azul e alfa de uma imagem. O WebGL não se importa com os dados. Ao contrário de atributos e buffers, os shaders podem ler valores de texturas com acesso aleatório.

  • Variações

Variações são uma maneira de um sombreador de vértice passar dados para um sombreador de fragmento. Variações são interpoladas entre os valores de saída do vertex shader enquanto uma primitiva é rasterizada usando um fragment shader
