C é uma linguagem de programação de computador imperativa de uso geral, que suporta programação estruturada, escopo de variável léxica e recursão, enquanto um sistema de tipo estático evita muitas operações não intencionais. Por design, C fornece construções que mapeiam de forma eficiente para instruções de máquina típicas e, portanto, encontrou uso duradouro em aplicativos que anteriormente eram codificados em linguagem assembly, incluindo sistemas operacionais, bem como vários softwares de aplicativos para computadores, desde supercomputadores a sistemas embarcados. .

Apesar de seus recursos de baixo nível, a linguagem foi projetada para incentivar a programação entre plataformas. Um programa C escrito portátil e compatível com os padrões pode ser compilado para uma ampla variedade de plataformas de computador e sistemas operacionais com poucas alterações em seu código-fonte. A linguagem tornou-se disponível em uma ampla gama de plataformas, de microcontroladores embutidos a supercomputadores.

C foi originalmente desenvolvido por Dennis Ritchie entre 1969 e 1973 no Bell Labs e usado para reimplementar os sistemas operacionais Unix. Desde então, tornou-se uma das linguagens de programação mais usadas de todos os tempos, com compiladores C de vários fornecedores disponíveis para a maioria das arquiteturas de computador e sistemas operacionais existentes.

Compiladores comuns #

O processo para compilar um programa C difere entre compiladores e sistemas operacionais. A maioria dos sistemas operacionais vem sem um compilador, então você terá que instalar um. Algumas escolhas comuns de compiladores são:

  • GCC, a coleção de compiladores GNU
  • clang: um front-end da família de linguagem C para LLVM
  • MSVC, ferramentas de compilação do Microsoft Visual C/C++

Os documentos a seguir devem fornecer uma boa visão geral sobre como começar a usar alguns dos compiladores mais comuns:

  • Introdução ao Microsoft Visual C
  • Introdução ao GCC

Suporte à versão C do compilador #

Observe que os compiladores têm níveis variados de suporte para o padrão C, com muitos ainda não suportando completamente o C99. Por exemplo, a partir da versão 2015, o MSVC suporta grande parte do C99, mas ainda tem algumas exceções importantes para suporte da própria linguagem (por exemplo, o pré-processamento parece não estar em conformidade) e para a biblioteca C (por exemplo, <tgmath.h>) , nem documentam necessariamente as suas “escolhas dependentes da implementação”. Wikipedia tem uma tabela mostrando o suporte oferecido por alguns compiladores populares.

Alguns compiladores (principalmente GCC) ofereceram, ou continuam a oferecer, extensões de compilador que implementam recursos adicionais que os produtores do compilador consideram necessários, úteis ou acreditam que podem se tornar parte de uma versão C futura, mas que atualmente não fazem parte de nenhuma versão C. padrão. Como essas extensões são específicas do compilador, elas podem ser consideradas não compatíveis entre si e os desenvolvedores do compilador podem removê-las ou alterá-las em versões posteriores do compilador. O uso de tais extensões geralmente pode ser controlado por sinalizadores do compilador.

Além disso, muitos desenvolvedores têm compiladores que suportam apenas versões específicas de C impostas pelo ambiente ou plataforma que estão direcionando.

Ao selecionar um compilador, é recomendável escolher um compilador que tenha o melhor suporte para a versão mais recente de C permitida para o ambiente de destino.

Estilo de código (fora do tópico aqui): #

Como o espaço em branco é insignificante em C (ou seja, não afeta a operação do código), os programadores geralmente usam o espaço em branco para tornar o código mais fácil de ler e compreender, isso é chamado de estilo de código. É um conjunto de regras e diretrizes usadas ao escrever o código-fonte. Abrange questões como como as linhas devem ser recuadas, se espaços ou tabulações devem ser usados, como as chaves devem ser colocadas, como os espaços devem ser usados ​​em torno de operadores e colchetes, como as variáveis ​​devem ser nomeadas e assim por diante.

O estilo de código não é coberto pelo padrão e é principalmente baseado em opiniões (pessoas diferentes acham estilos diferentes mais fáceis de ler), como tal, geralmente é considerado off-topic no SO. O conselho primordial sobre estilo no próprio código é que a consistência é primordial - escolha, ou faça, um estilo e cumpra-o. Basta explicar que existem vários estilos nomeados em uso comum que geralmente são escolhidos pelos programadores em vez de criar seu próprio estilo.

Alguns estilos de recuo comuns são: estilo K & R, estilo Allman, estilo GNU e assim por diante. Alguns desses estilos têm variantes diferentes. Allman, por exemplo, é usado como Allman regular ou a variante popular, Allman-8. Informações sobre alguns dos estilos populares podem ser encontradas na Wikipedia. Esses nomes de estilo são retirados dos padrões que os autores ou organizações geralmente publicam para uso de muitas pessoas que contribuem com seu código, para que todos possam ler facilmente o código quando conhecerem o estilo, como o guia de formatação GNU que faz parte do documento padrões de codificação GNU.

Algumas convenções de nomenclatura comuns são: UpperCamelCase, lowerCamelCase, lower_case_with_underscore, ALL_CAPS, etc. Esses estilos são combinados de várias maneiras para uso com diferentes objetos e tipos (por exemplo, macros geralmente usam o estilo ALL_CAPS)

O estilo K&R geralmente é recomendado para uso na documentação SO, enquanto os estilos mais esotéricos, como Pico, são desencorajados.
