O que é wxPython #

Simplificando, wxPython é um conjunto de ligações para a biblioteca de GUI de plataforma cruzada wxWidgets C++.

Ok, o que é wxWidgets #

A biblioteca wxWidgets fornece um conjunto de abstrações gratuito, gratuito e de código aberto para os vários elementos da GUI para que os controles nativos ainda sejam usados, quando disponíveis, mantendo a aparência, a sensação e a velocidade nativas. Como tal, ele fornece uma abstração para criação de GUI e vários outros utilitários em uma plataforma que permite aos desenvolvedores criar aplicativos para Windows, Mac OS X, Linux e outras plataformas usando uma única base de código. wxWidgets foi iniciado em 1992 e você pode ver um histórico detalhado aqui. A biblioteca wxWidgets é distribuída sob a licença wxWindows, que é baseada na L-GPL, mas com uma cláusula de exceção. A cláusula de exceção permite que você vincule seu aplicativo dinamicamente ou estaticamente ao wxWidgets sem o requisito de distribuir a fonte para seu próprio aplicativo. Em outras palavras, você pode usar wxWidgets para projetos gratuitos ou comerciais, sem custo. A licença incentiva você a devolver os aprimoramentos feitos na própria biblioteca wxWidgets.

Os destaques, observe que wxWidgets compreende centenas de classes para desenvolvimento de aplicativos multiplataforma:

  • Layout de janela usando dimensionadores
  • Contextos do dispositivo (junto com canetas, pincéis e fontes)
  • Sistema abrangente de tratamento de eventos
  • Visualizador de Ajuda HTML
  • Reprodução de som e vídeo
  • Suporte Unicode e Internacionalização
  • Arquitetura de Documentos/Visualizações
  • Arquitetura de impressão
  • Tomadas
  • Multithreading
  • Manipulação de arquivos e diretórios
  • Ajuda online e sensível ao contexto
  • Renderização HTML
  • Recipientes Básicos
  • Carregamento, Salvamento, Desenho e Manipulação de Imagens
  • Biblioteca de data-hora e temporizadores
  • Manipulação de erros
  • Área de transferência e arrastar e soltar

Observe que algumas dessas facilidades, ex. threading, não são realmente relacionados à GUI, mas fornecem uma abstração de plataforma cruzada útil para que, no caso de threading, por exemplo, um conjunto de código de aplicativo funcione em qualquer plataforma suportada.

Por muitos anos, a biblioteca wxWidgets produziu 4 compilações separadas, além de compilações de depuração de um conjunto de código-fonte, bibliotecas estáticas e dinâmicas criadas para ASCII e Unicode. Geralmente está disponível pré-construído nas variantes mais comuns e como código-fonte para construir com as várias opções para o ambiente de destino e com a cadeia de ferramentas C++ dos desenvolvedores com várias cadeias de ferramentas sendo suportadas.

As ligações python para esta biblioteca e algumas adições formam wxPython.

Voltar para O que é wxPython, (o que ele me dá)? #

O wxPython oferece ao desenvolvedor uma maneira de se beneficiar de uma biblioteca GUI de plataforma cruzada, com uma licença clara, ao mesmo tempo em que oferece os benefícios do Python. Assim como o wxWidgets e o Python, o wxPython é gratuito, gratuito e de código aberto e está disponível para uso e distribuição em projetos gratuitos e comerciais sem a necessidade de distribuir seu código-fonte.

  • Suite GUI completa, incluindo (mas não limitado a):
  • Windows (incluindo Windows MDI)
  • Assistentes
  • Molduras e MiniQuadros
  • Diálogos, Padrão, Avançado e Personalizado
  • Controles de exibição de livros, árvores, grades e dados
  • Medidores, Sliders, Spinners, Animações, Área de Transferência, Arrastar e Soltar
  • Suporte a HTML, PDF e visualizador de imagens
  • Os componentes da GUI podem ser posicionados de forma absoluta, mas é altamente recomendável usar o layout baseado em dimensionador que suporta dimensionamento automático, etc.
  • Plataforma cruzada - GUIs de suporte para Windows, OS-X e Linux com uma única base de código sem declarações condicionais em seu código
  • Velocidade nativa, aparência e sensação.
  • Protótipo, teste e depuração rápidos - lembre-se que isso é python
  • Execute e edite amostras de quase tudo no pacote de demonstração.
  • Licença clara para uso gratuito mesmo em produtos comerciais.
  • Se necessário, sua GUI python pode ser refatorada para uma GUI C++ wxWidgets posteriormente, pois já a está usando.
  • Grande, ativa e útil comunidade de usuários e desenvolvedores tanto no [StackOverflow][14] quanto nas [listas de e-mail][15].

Observe que, onde o próprio python fornece um mecanismo de plataforma cruzada para implementar as funções utilitárias do wxWidgets, threading novamente sendo um bom exemplo, ele é omitido intencionalmente do wxPython.

wxPython também possui um conjunto muito grande de demonstrações que podem ser executadas, testadas e editadas a partir do pacote Documents and Demo.

Sabores do wxPython #

ASCII vs Unicode: #

Por muitos anos, assim como com wxWidgets, os desenvolvedores tiveram que escolher entre compilações ASCII e Unicode, além de precisar de uma compilação para sua versão específica do python, bem como as opções de 32/64 bits. A partir do wxPython 2.8.9, a compilação somente ASCII do wxPython foi descartada, portanto, o suporte a Unicode está sempre disponível.

Clássico vs. Phoenix: #

Desde o wxPython 3.0.0 existe a compilação “Clássica” lançada do wxPython e uma compilação do Phoenix atualmente não lançada. A compilação clássica tende a ficar atrás das compilações wxWidgets dos mesmos números e o pacote de documentação é o C++ - está disponível para download para várias plataformas, (veja [Instalação do Classic][3]), no caso do windows como um instalador executável. As ligações do Phoenix, sendo em grande parte geradas automaticamente, devem seguir mais de perto as compilações do wxWidgets e também incluir a documentação específica do wxPython - é construível a partir da fonte ou compilações noturnas como rodas podem ser obtidas usando pip, (consulte [Instalação do Phoenix][4]).

Em wxPython, mas não em wxWidgets #

wxPython estende a biblioteca wxWidgets com vários recursos, os seguintes são apenas alguns, que não estão disponíveis no wxWidgets:

  • Editores e shells de programadores: [crust][5], [crustslices][6], [AlaCart & AlaMode][7], [AlaModeTest][8]
  • [Intérprete][9] & [magia][10]
  • Inspeção - permite que você abra uma janela para navegar por todos os componentes da GUI de seus aplicativos.
  • Um extenso conjunto de demos
