O que é android-gradle #

android-gradle é um gradle plugin mantido oficialmente pela equipe de desenvolvedores do Google Tools e é a ferramenta de compilação oficial desde o anúncio em 16 de maio de 2013 no Google I/O.

Aprenda o básico lendo Configure your build with Gradle.

Principais características #

Os principais recursos do plug-in Android Gradle são:

  • Gerenciamento de dependências
  • Projetos modulares com bibliotecas
  • Variantes através de Sabores e Tipos de Construção
  • Construções independentes de IDE

Visão geral #

  1. Baixe e instale o Android Studio
  2. abra-o e crie um novo projeto com todas as configurações padrão

Em teoria, você pode instalar o gradle diretamente, construir os arquivos de configuração e a estrutura de diretórios por conta própria. Na prática ninguém faz isso.

Uma estrutura de pastas de projeto normalmente se parece com isso:

Plugin android-gradle #

Um projeto gradle geralmente é dividido em subprojetos ou módulos, cada um contendo um script de compilação dedicado.

A dependência do plugin geralmente é declarada no arquivo build.gradle de nível principal/superior:

buildscript {
    // maven repositories for dependencies
    repositories {
        jcenter()
    }
    // build script dependencies
    dependencies {
        // this is the dependency to the android build tools
        classpath 'com.android.tools.build:gradle:2.1.2'
    }
}

allprojects {
    // maven repositories for all sub-project / modules
    repositories {
        jcenter()
    }
}

Neste exemplo, a versão do plugin android-gradle é 2.1.2 como você pode ver nesta linha:

classpath 'com.android.tools.build:gradle:2.1.2'

Módulos #

O projeto é dividido em módulos, cada um contendo um script build.gradle dedicado. O arquivo settings.gradle lista estes módulos:

include ':app'

Os dois pontos : são usados ​​como um delimitador de pasta.

Para usar o plugin ele deve ser aplicado no topo do arquivo build.gradle de cada módulo (app no exemplo).

Para um aplicativo Android:

apply plugin: 'com.android.application'

Para uma biblioteca Android:

apply plugin: 'com.android.library'

E então configurado em sua tag android:

android {
  // gradle-android plugin configuration
}

Configuração básica do aplicativo Android #

O build.gradle gerado pelo Android Studio para um aplicativo é assim:

apply plugin: 'com.android.application'

android {
    // setup which version of the SDK to build against and
    // with what version of the build tools
    compileSdkVersion 23
    buildToolsVersion "23.0.2"

    // default app configurations
    defaultConfig {
        // this is your app unique ID
        applicationId "com.example.myapp"
        
        // devices with lower SDK version can't install the app
        minSdkVersion 14
        // target SDK version, should be the last available one and
        // match the compile one
        targetSdkVersion 23
        
        // integer and string version of your app
        versionCode 1
        versionName "1.0"
    }
    
    // default build types are "debug" and "release"
    buildTypes {
        release {
            // enable / disable proguard optimization
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}

// app dependencies
dependencies {
    // any jar file in the libs folder
    compile fileTree(dir: 'libs', include: ['*.jar'])
    // test dependency
    testCompile 'junit:junit:4.12'
    // runtime dependency on the support library
    compile 'com.android.support:appcompat-v7:24.0.0'
}

Configure sua compilação com Gradle ensine a você configurações e opções mais avançadas do Android Gradle Plugin e aprofunde o significado dessa configuração.

O defaultConfig é chamado assim porque pode ser substituído por Product Flavors.

A tag buildTypes permite que você configure como construir seu aplicativo habilitando a otimização (como proguard), você pode aprender mais lendo Build Types. Ele também pode ser usado para configurar a assinatura do seu aplicativo.

Você também deve aprender mais sobre como Declarar Dependências. Como você pode ver, a tag dependencies está fora da tag android: isso significa que não é definida pelo plugin Android, mas é padrão gradle.

O Embrulho Gradle #

O Android Studio também instalará, por padrão, um gradle wrapper. Esta é uma ferramenta que você pode executar diretamente da linha de comando e fará o download de uma versão local específica do gradle na primeira vez que você a executar.

Para iniciar a compilação do aplicativo, você pode iniciar o wrapper gradle

Linux/Mac:

./gradlew assemble

Janelas:

gradlew assemble

O script inicia o wrapper, contido em uma pasta gradle no diretório raiz do seu projeto:

  • gradle-wrapper.jar: o código do wrapper para baixar o gradle e executá-lo
  • gradle-wrapper.properties define qual versão do gradle o wrapper deve baixar
