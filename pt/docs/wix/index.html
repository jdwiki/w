Tutorial wix

O que é WiX? #

O WiX toolset permite que os desenvolvedores criem instaladores para o Windows Installer, o mecanismo de instalação do Windows. É código aberto e parte da .NET Foundation.

O núcleo do WiX é um conjunto de ferramentas de compilação que compilam pacotes do Windows Installer usando os mesmos conceitos de compilação do restante do seu produto: o código-fonte é compilado e vinculado para criar executáveis; neste caso, pacotes de instalação .exe, pacotes de instalação .msi, módulos de mesclagem .msm e patches .msp. As ferramentas de compilação de linha de comando do WiX funcionam com qualquer sistema de compilação automatizado. Além disso, o MSBuild é compatível com a linha de comando, Visual Studio e Team Build.

O WiX inclui várias extensões que oferecem funcionalidades além do Windows Installer. Por exemplo, o WiX pode instalar sites IIS, criar bancos de dados SQL Server e registrar exceções no Firewall do Windows, entre outros.

Com o Burn, o bootstrapper do WiX, você pode criar pacotes de instalação que instalam pré-requisitos como o .NET Framework e outros tempos de execução junto com seu próprio produto. Burn permite baixar pacotes ou combiná-los em um único .exe para download.

O SDK do WiX inclui bibliotecas gerenciadas e nativas que facilitam a escrita de código que funciona com o Windows Installer, incluindo ações personalizadas em C# e C++.

Como funciona o WiX? #

O código-fonte do WiX é escrito no formato XML com uma extensão de arquivo .wxs. As ferramentas WiX seguem o modelo tradicional de compilação e link usado para criar executáveis ​​a partir do código-fonte.

No momento da compilação, os arquivos de origem do WiX são validados em relação ao esquema principal do WiX e, em seguida, processados ​​por um pré-processador, compilador e vinculador para criar o resultado final. Há um conjunto de ferramentas WiX que podem ser usadas para produzir diferentes tipos de saída.

Requisitos do sistema WiX #

O WiX suporta .NET 3.5 e 4.0 e posterior. O suporte ao MSBuild do WiX requer o .NET 3.5, que não é instalado por padrão no Windows 8 e no Windows Server 2012 e posterior.

Na próxima versão do WiX (v3.11), o .NET 4.0 será necessário; a construção usando .NET 3.5 não será mais suportada.
