Verificando tipos #

Você pode usar o operador is para validar se um valor é de um determinado tipo:

var sprite:Sprite = new Sprite();

trace(sprite is Sprite); // true
trace(sprite is DisplayObject); // true, Sprite inherits DisplayObject
trace(sprite is IBitmapDrawable); // true, DisplayObject implements IBitmapDrawable
trace(sprite is Number); // false
trace(sprite is Bitmap); // false, Bitmap inherits DisplayObject
                         // but is not inherited by Sprite.

Há também um operador instanceof (obsoleto) que funciona quase idêntico ao is exceto que retorna false ao verificar interfaces implementadas e tipos int/uint.

O operador as também pode ser usado como o operador is. Isso é especialmente útil se você usar algum IDE inteligente como o FlashDevelop, que fornecerá uma lista de todas as propriedades possíveis do tipo de objeto explícito. Exemplo:

for (var i:int = 0; i < a.length; i++){
    var d:DisplayObject = a[i] as DisplayObject;
    if (!d) continue;
    d.//get hints here
    stage.addChild(d);
}

Para obter o mesmo efeito com is, você escreveria (um pouco menos conveniente):

for (var i:int = 0; i < a.length; i++){
    if (a[i] is DisplayObject != true) continue;
    var d:DisplayObject = a[i] as DisplayObject;
    stage.addChild(d);
}

Apenas tenha em mente que ao verificar as coditions com o operador as, o valor fornecido será primeiro convertido para o tipo especificado e, em seguida, o resultado dessa operação será verificado se não for falso, portanto, tenha cuidado ao usá-lo com possíveis valores falsos/NaN:

if(false as Boolean) trace("This will not be executed");
if(false as Boolean != null) trace("But this will be");

A tabela abaixo mostra alguns valores básicos e tipos com resultado de operadores de tipo. As células verdes serão avaliadas como verdadeiras, as vermelhas como falsas e as cinzas causarão erros de compilação/tempo de execução.

Tipo de fundição #

A conversão de tipo é feita com o operador as:

var chair:Chair = furniture as Chair;

Ou envolvendo o valor em Type():

var chair:Chair = Chair(furniture);

Se a conversão falhar com as, o resultado dessa conversão será null. Se a conversão falhar ao envolver em Type(), um TypeError será lançado.

O tipo de função #

As funções são do tipo Função:

function example():void { }
trace(example is Function); // true

Eles podem ser referenciados por outras variáveis ​​com o tipo Function:

var ref:Function = example;
ref(); // ref.call(), ref.apply(), etc.

E eles podem ser passados ​​como argumentos para parâmetros cujo tipo é Function:

function test(callback:Function):void {
    callback();
}

test(function() {
    trace('It works!');
}); // Output: It works!

O tipo de classe #

Referências a declarações de classe são digitadas como Class:

var spriteClass:Class = Sprite;

Você pode usar variáveis ​​do tipo Class para instanciar instâncias dessa classe:

var sprite:Sprite = new spriteClass();

Isso pode ser útil para passar um argumento do tipo Class para uma função que pode criar uma instância da classe fornecida:

function create(type:Class, x:int, y:int):* {
    var thing:* = new type();
    
    thing.x = x;
    thing.y = y;
    
    return thing;
}

var sprite:Sprite = create(Sprite, 100, 100);

Matrizes digitadas #

Infelizmente, o ActionScript 3 não tem um conceito de genéricos, portanto, não há como definir um array tipado como Array<T>. No entanto, existe uma classe especial Vector.<T> que funciona de maneira semelhante, exceto que você deve fornecer uma referência a uma classe concreta ao instanciar o vetor. Isso significa que não há como construir abstrações em cima do tipo Vector.<T> (por exemplo, estendê-lo e adicionar novas funcionalidades), o que é uma grande desvantagem.

Uma maneira mais simples de ver é como se cada classe que você define automaticamente tivesse uma classe associada chamada Vector.<NameOfYourClass>.

Com isso dito, ainda há grandes vantagens para o tipo Vector.<T> sobre um array convencional:

  • Há um aumento de desempenho ao trabalhar com Vector.<T> vs arrays1.
  • Você recebe TypeErrors do tipo de compilação se tentar inserir valores não-T na coleção.
  • IDEs fornecem informações úteis de dicas de tipo para os objetos dentro de uma instância Vector.<T>.

Exemplos de criação de um Vector.<T>:

var strings:Vector.<String> = new Vector.<String>(); // or
var numbers:Vector.<Number> = new <Number>[];

1 Na verdade, os vetores só fornecem melhorias notáveis ​​de desempenho em relação aos arrays ao trabalhar com tipos primitivos (String, int, uint, Number, etc).

Tipos de anotação #

Você pode dizer ao compilador o tipo de um valor anotando-o com :Type:

var value:int = 10; // A property "value" of type "int".

Parâmetros de função e tipos de retorno também podem ser anotados:

// This function accepts two ints and returns an int.
function sum(a:int, b:int):int {
    return a + b;
}

A tentativa de atribuir um valor com um tipo incompatível resultará em um TypeError:

var sprite:Sprite = 10; // 10 is not a Sprite.
