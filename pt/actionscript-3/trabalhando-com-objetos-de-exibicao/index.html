Sintaxe #

  1. addChild(child) - adiciona um novo item à árvore filho deste objeto como o elemento mais alto.
  2. addChildAt(filho, índice) - adiciona um novo item à árvore filho deste objeto em uma posição especificada. O item mais abaixo tem índice de 0.
  3. getChildAt(index) - retorna um filho com determinado índice.
  4. getChildIndex(child) retorna o índice de um filho direto deste objeto. Caso contrário, uma exceção é lançada.
  5. removeChild(filho) - remove o filho direto especificado da árvore filho deste objeto. Lança exceção se o pai do filho fornecido não for igual a this.
  6. removeChildAt(index) - remove um filho selecionado por índice em vez de referência. Lança exceção se a árvore filha não for tão larga.
  7. removeChildren(beginIndex:int = 0, endIndex:int = 0x7fffffff)) - adicionado no Flash Player 11, remove um subconjunto de filhos por intervalo de índice ou todos os filhos se chamados sem parâmetros.
  8. setChildIndex(child,index) - altera o índice do filho para o novo valor, deslocando todos os filhos entre eles para ocupar o local liberado.
  9. swapChildren(child1,child2) - troca as posições dos dois filhos na lista de exibição, não afetando as posições dos outros filhos.
  10. swapChildrenAt(index1,index2) - troca os filhos localizados por seus índices.

A lista de exibição é na verdade uma árvore e é visualizada com o algoritmo de profundidade primeiro. Qualquer objeto listado anteriormente será exibido anteriormente e pode ser obscurecido por objetos listados posteriormente. Todas as técnicas que podem ser usadas em uma árvore podem ser aplicadas ao trabalho com lista de exibição.

Remova todos os objetos da lista de exibição #

Se segmentar o Flash Player 11+, o método interno removeChildren é a melhor maneira de remover todos os filhos:

removeChildren(); //a start and end index can be passed

Para aplicativos legados, o mesmo pode ser feito com um loop:

while (numChildren > 0) {
    removeChildAt(0);
}

Introdução à lista de exibição #

No AS3, os ativos de exibição não ficam visíveis até que sejam adicionados à Lista de exibição.

O tempo de execução do AIR/Flash tem uma estrutura de exibição hierárquica (relacionamento pai-filho onde os filhos podem ter seus próprios filhos), com o ‘palco’ sendo o pai de nível superior.

Para adicionar algo à lista de exibição, você usa addChild ou addChildAt. Aqui está um exemplo básico de desenhar um círculo e adicioná-lo à lista de exibição:

var myCircle:Shape = new Shape();
        
myCircle.graphics.beginFill(0xFF0000); //red
myCircle.graphics.drawCircle(25, 25, 50);
myCircle.graphics.endFill();
        
this.addChild(myCircle); //add the circle as a child of `this`

Para ver o objeto no exemplo acima, this (o contexto do código) também deve estar na lista de exibição, assim como quaisquer pais que ele possa ter. No AS3, o stage é o pai mais importante.

Os objetos de exibição só podem ter um pai. Portanto, se um filho já tiver um pai e você o adicionar a outro objeto, ele será removido do pai anterior.

Ordem Z/Estratificação #

Digamos que você tenha replicado o código do exemplo anterior para ter 3 círculos:

var redCircle:Shape = new Shape();
redCircle.graphics.beginFill(0xFF0000); //red
redCircle.graphics.drawCircle(50, 50, 50); //graphics.endFill is not required

var greenCircle:Shape = new Shape();
greenCircle.graphics.beginFill(0x00FF00); //green
greenCircle.graphics.drawCircle(75, 75, 50);
        
var blueCircle:Shape = new Shape();
blueCircle.graphics.beginFill(0x0000FF); //blue
blueCircle.graphics.drawCircle(100, 100, 50);

this.addChild(redCircle); 
this.addChild(greenCircle); 
this.addChild(blueCircle);

Como o método addChild adiciona o filho em cima de tudo no mesmo pai, você obterá esse resultado com os itens em camadas na mesma ordem em que usar addChild:

Se você queria um filho em camadas diferentes em relação aos irmãos, você pode usar addChildAt. Com addChildAt, você passa outro parâmetro que indica o índice (ordem z) em que o filho deve estar. 0 sendo a posição/camada mais inferior.

this.addChild(redCircle); 
this.addChild(greenCircle); 
this.addChildAt(blueCircle,0); //This will add the blue circle at the bottom

Agora o círculo azul está sob seus irmãos. Se mais tarde você quiser alterar o índice de um filho, você pode usar o método setChildIndex (no pai do filho).

this.setChildIndex(redCircle, this.numChildren - 1); //since z-index is 0 based, the top most position is amount of children less 1.

Isso irá reorganizar o círculo vermelho para que fique acima de tudo. O código acima produz exatamente o mesmo resultado que this.addChild(redCircle).

Removendo objetos de exibição #

Para remover objetos, você tem os métodos inversos removeChild e removeChildAt bem como o método removeChildren.

removeChild(redCircle); //this will take redCircle off the display list

removeChildAt(0); //this will take the bottom most object off the display list 

removeChildren(); //this will clear all children from the display list

removeChildren(1); //this would remove all children except the bottom most

removeChildren(1,3); //this would remove the children at indexes 1, 2 & 3

Eventos #

Quando um filho é adicionado à lista de exibição, alguns eventos são acionados nesse filho.

  • Evento.ADICIONADO
  • Event.ADDED_TO_STAGE

Por outro lado, também existem os eventos de remoção:

  • Evento. REMOVIDO
  • Event.REMOVED_FROM_STAGE

Adobe Animate/Flash Professional #

Ao lidar com linhas de tempo do FlashProfessional/Adobe Animate, adicionar algo à linha de tempo lida com as nuances da lista de exibição automaticamente. Eles foram adicionados e removidos da lista de exibição automaticamente pela linha do tempo.

No entanto, é bom ter em mente que:

Se você manipular por meio de código o parentesco de um objeto de exibição criado pela linha do tempo (usando addChild/setChildIndex), esse filho não será mais removido automaticamente pela linha do tempo e precisará ser removido via código.

Camadas #

Pode haver situações em que você decida que um conjunto de objetos de exibição deve estar sempre acima de outro conjunto de objetos, por exemplo, setas sobre cabeças, explosões sobre algo que acabou de explodir, etc. Para fazer isso da forma mais simples possível, você precisa designar e crie um conjunto de Spritess, organize-os em ordem de baixo para cima, depois é só adicionar todos os objetos do conjunto “acima” a uma camada acima daquela usada para objetos do conjunto “abaixo”.

var monsters:Vector.<Monster>;
var bullets:Vector.<Bullet>; // desired: bullets strictly above monsters
var monsterLayer:Sprite=new Sprite();
var bulletLayer:Sprite=new Sprite();
addChild(monsterLayer);
addChild(bulletLayer);

Então, sempre que você adicionar um Monster à lista de exibição, adicione-o a monsterLayer, e sempre que você adicionar um Bullet, adicione a bulletLayer para obter o efeito desejado.

Transitando de frames para troca manual de conteúdo #

Logo no início, um desenvolvedor Flash usa frames, pois eles estão disponíveis nativamente no Flash player, para hospedar várias telas de seu aplicativo (na maioria das vezes é um jogo). Eventualmente, eles podem se deparar com um problema de que algo dá errado exatamente porque eles usaram quadros e ignoraram as dificuldades que surgem disso, e buscar maneiras de manter sua estrutura de quadros, mas também remover o obstáculo de usar quadros com suas complicações. A solução é usar classes descendentes Sprite, ou quadros exportados como MovieClips com um único quadro (para aqueles que projetam em Adobe Flash CS), e alternar manualmente o conteúdo com addChild() e removeChild() .

A classe gerenciadora deve ter todas as suas classes de quadro filho prontas, e sempre que uma transição é chamada, uma função semelhante a esta pode ser usada:

var frames:Vector.<DisplayObject>; // this holds instances to ALL children
var currentFrame_alt:int; // current frame. Can't use the property
function changeFrame(frame:int):void {
    removeChild(frames[currentFrame_alt]);
    addChild(frames[frame]);
    currentFrame_alt=frame;
}

Todos os filhos podem despachar e ouvir eventos com Event.ADDED_TO_STAGE usado como um ponto de entrada para o que quer que aconteça após gotoAndStop() atingir esse quadro, e quaisquer transições de saída podem ser codificadas como eventos com base em strings, que estão sendo ouvidas na classe Main, que então executa a transição.

frames[0].addEventListener("startGame",startGame); // assuming frame 0 is a "Play" button
function startGame(e:Event):void {
    changeFrame(1); // switch to frame 1 - will display frames[1]
}

Claro, o conjunto de strings deve ser predefinido, por exemplo, a tela de introdução pode ter dois botões para iniciar o jogo, “Iniciar jogo” e “Iniciar sem som”, por exemplo, e os botões devem despachar eventos diferentes, que serão tratados diferente na classe de gerente.

Esse padrão pode ser tão profundo quanto você precisar. Se algum quadro do projeto contiver um MovieClip com vários quadros, ele também poderá ser desacoplado em sprites com esse método.
