No mangusto, Middlewares também são chamados de ganchos pre e post.

Existem dois tipos de middleware #

Ambos os middlewares suportam ganchos pré e pós.

  1. Middleware de documentos

    Its supported for document functions init, validate, save and remove

  2. Middleware de consulta

    Its supported for query functions count, find, findOne, findOneAndRemove, findOneAndUpdate, insertMany and update.

Ganchos pré e pós #

Existem dois tipos de ganchos Pré

  1. série

    As the name suggests, Its executed in serial order i..e one after another

  2. paralelo

O middleware paralelo oferece controle de fluxo mais refinado e o método hooked não é executado até que done seja chamado por todos os middlewares paralelos.

Post Middleware são executados após o método hooked e todos os seus pré middleware serem concluídos.

métodos hooked são as funções suportadas pelo middleware de documentos. iniciar, validar, salvar, remover

Middleware para hash da senha do usuário antes de salvá-la #

Este é um exemplo de Serial Document Middleware

Neste exemplo, escreveremos um middleware que converterá a senha de texto simples em uma senha com hash antes de salvá-la no banco de dados.

Esse middleware será ativado automaticamente ao criar um novo usuário ou atualizar os detalhes do usuário existente.

FILENAME: User.js

// lets import mongoose first
import mongoose from 'mongoose'

// lets create a schema for the user model
const UserSchema = mongoose.Schema(
  {
    name: String,
    email: { type: String, lowercase: true, requird: true },
    password: String,
  },
);


/**
 * This is the middleware, It will be called before saving any record
 */
UserSchema.pre('save', function(next) {

  // check if password is present and is modified.
  if ( this.password && this.isModified('password') ) {

    // call your hashPassword method here which will return the hashed password.
    this.password = hashPassword(this.password);

  }

  // everything is done, so let's call the next callback.
  next();

});


// lets export it, so we can import it in other files.
export default mongoose.model( 'User', UserSchema );

Agora, toda vez que salvamos um usuário, este middleware verificará se a senha está definida e está modificada (isso é para que não façamos hash da senha dos usuários se não for modificada).

FILENAME: App.js

import express from 'express';
import mongoose from 'mongoose';

// lets import our User Model
import User from './User';

// connect to MongoDB
mongoose.Promise = global.Promise;
mongoose.connect('mongodb://127.0.0.1:27017/database');


let app = express();
/* ... express middlewares here .... */


app.post( '/', (req, res) => {

  /*
    req.body = {
      name: 'John Doe',
      email: 'john.doe@gmail.com',
      password: '!trump'
    }
   */

  // validate the POST data

  let newUser = new User({
    name: req.body.name,
    email: req.body.email,
    password: req.body.password,
  });

  newUser.save( ( error, record ) => {
    if (error) {
      res.json({
        message: 'error',
        description: 'some error occoured while saving the user in database.'
      });
    } else {
      res.json({
        message: 'success',
        description: 'user details successfully saved.',
        user: record
      });
    }
  });

});



let server = app.listen( 3000, () => {
    console.log(`Server running at http://localhost:3000` );
  }
);
