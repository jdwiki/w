Instalação do CMake #

Vá para a página de download CMake e obtenha um binário para o seu sistema operacional, por exemplo Windows, Linux ou Mac OS X. No Windows, clique duas vezes no binário para instalar. No Linux, execute o binário a partir de um terminal.

No Linux, você também pode instalar os pacotes do gerenciador de pacotes da distribuição. No Ubuntu 16.04, você pode instalar a linha de comando e o aplicativo gráfico com:

sudo apt-get install cmake
sudo apt-get install cmake-gui

No FreeBSD você pode instalar a linha de comando e o aplicativo gráfico baseado em Qt com:

pkg install cmake
pkg install cmake-gui

No Mac OSX, se você usar um dos gerenciadores de pacotes disponíveis para instalar seu software, sendo os mais notáveis ​​MacPorts (MacPorts) e Homebrew (Homebrew), você também poderá instalar o CMake por meio de um dos eles. Por exemplo, no caso de MacPorts, digite o seguinte

sudo port install cmake

instalará o CMake, enquanto no caso de você usar o gerenciador de pacotes Homebrew, você digitará

brew install cmake

Depois de instalar o CMake, você pode verificar facilmente fazendo o seguinte

cmake --version

Você deve ver algo semelhante ao seguinte

cmake version 3.5.1

CMake suite maintained and supported by Kitware (kitware.com/cmake).

Projeto Simples “Hello World” #

Dado um arquivo de origem C++ main.cpp definindo uma função main(), um arquivo CMakeLists.txt acompanhante (com o conteúdo a seguir) instruirá o CMake a gerar as instruções de compilação apropriadas para o sistema atual e o compilador C++ padrão .

main.cpp (C++ Hello World Example)

#include <iostream>

int main()
{
    std::cout << "Hello World!\n";
    return 0;
}

CMakeLists.txt

cmake_minimum_required(VERSION 2.4)

project(hello_world)

add_executable(app main.cpp)

Veja ao vivo no Coliru

  1. cmake_minimum_required(VERSION 2.4) define uma versão mínima do CMake necessária para avaliar o script atual.

  2. project(hello_world) inicia um novo projeto CMake. Isso acionará muita lógica interna do CMake, especialmente a detecção do compilador C e C++ padrão.

  3. Com add_executable(app main.cpp) um destino de compilação app é criado, que invocará o compilador configurado com alguns sinalizadores padrão para a configuração atual para compilar um app executável do arquivo fonte fornecido main.cpp.

Linha de comando (In-Source-Build, não recomendado)

> cmake .
...
> cmake --build .

cmake . faz a detecção do compilador, avalia o CMakeLists.txt no diretório . fornecido e gera o ambiente de compilação no diretório de trabalho atual.

O comando cmake --build . é uma abstração para a chamada build/make necessária.

Linha de comando (Fora da fonte, recomendado)

Para manter seu código-fonte limpo de qualquer artefato de compilação, você deve fazer compilações “fora da fonte”.

> mkdir build
> cd build
> cmake ..
> cmake --build .

Ou o CMake também pode abstrair os comandos básicos do shell de sua plataforma do exemplo acima:

> cmake -E make_directory build
> cmake -E chdir build cmake .. 
> cmake --build build

“Hello World” como biblioteca #

Este exemplo mostra como implantar o programa “Hello World” como uma biblioteca e como vinculá-lo a outros destinos.

Digamos que temos o mesmo conjunto de arquivos de origem/cabeçalho que em https://www.wikiod.com/pt/docs/cmake/862/getting-started-with-cmake/22391/hello-world-with-multiple-source-files #t=201610310659039267444 exemplo. Em vez de construir a partir de vários arquivos de origem, podemos primeiro implantar o foo.cpp como uma biblioteca usando add_library() e depois vinculá-lo ao programa principal com target_link_libraries().

Modificamos CMakeLists.txt para

cmake_minimum_required(VERSION 2.4)

project(hello_world)

include_directories(${PROJECT_SOURCE_DIR})
add_library(applib foo.cpp)
add_executable(app main.cpp)
target_link_libraries(app applib)

e seguindo os mesmos passos, obteremos o mesmo resultado.

Alternando entre os tipos de compilação, por exemplo depurar e liberar #

O CMake conhece vários tipos de compilação, que geralmente influenciam os parâmetros padrão do compilador e do vinculador (como informações de depuração que estão sendo criadas) ou caminhos de código alternativos.

Por padrão, o CMake é capaz de lidar com os seguintes tipos de compilação:

  • Depuração: geralmente uma compilação de depuração clássica, incluindo informações de depuração, sem otimização etc.
  • Release: seu build de lançamento típico sem informações de depuração e otimização completa.
  • RelWithDebInfo:: Igual a Release, mas com informações de depuração.
  • MinSizeRel: Uma compilação especial Release otimizada para tamanho.

Como as configurações são tratadas depende do gerador que está sendo usado.

Alguns geradores (como o Visual Studio) dão suporte a várias configurações. O CMake irá gerar todas as configurações de uma vez e você pode selecionar no IDE ou usando --config CONFIG (com cmake --build) qual configuração você deseja construir. Para esses geradores, o CMake fará o possível para gerar uma estrutura de diretório de compilação, de modo que os arquivos de configurações diferentes não entrem um no outro.

Os geradores que suportam apenas uma única configuração (como Unix Makefiles) funcionam de maneira diferente. Aqui a configuração atualmente ativa é determinada pelo valor da variável CMake CMAKE_BUILD_TYPE.

Por exemplo, para escolher um tipo de compilação diferente, pode-se emitir os seguintes comandos de linha de comando:

cmake -DCMAKE_BUILD_TYPE=Debug path/to/source
cmake -DCMAKE_BUILD_TYPE=Release path/to/source

Um script CMake deve evitar definir o próprio CMAKE_BUILD_TYPE, pois geralmente é considerado responsabilidade do usuário fazê-lo.

Para alternar geradores de configuração única, a configuração requer a reexecução do CMake. Uma compilação subsequente provavelmente substituirá os arquivos de objeto produzidos pela configuração anterior.

“Hello World” com vários arquivos de origem #

Primeiro podemos especificar os diretórios dos arquivos de cabeçalho por include_directories(), então precisamos especificar os arquivos de origem correspondentes do executável de destino por add_executable(), e ter certeza de que há exatamente uma função main() nos arquivos de origem.

A seguir está um exemplo simples, todos os arquivos são assumidos colocados no diretório PROJECT_SOURCE_DIR.

main.cpp

#include "foo.h"

int main()
{
    foo();
    return 0;
}

foo.h

void foo();

foo.cpp

#include <iostream>
#include "foo.h"

void foo()
{
    std::cout << "Hello World!\n";
}

CMakeLists.txt

cmake_minimum_required(VERSION 2.4)

project(hello_world)

include_directories(${PROJECT_SOURCE_DIR})
add_executable(app main.cpp foo.cpp)  # be sure there's exactly one main() function in the source files

Podemos seguir o mesmo procedimento no exemplo acima para construir nosso projeto. Em seguida, a execução de app imprimirá

>./app
Hello World!
