A simplicidade das variáveis ​​básicas do CMake contradiz a complexidade da sintaxe completa da variável. Esta página documenta os vários casos de variáveis, com exemplos, e aponta as armadilhas a serem evitadas.

Sintaxe #

  • set(variable_name valor [cache type description [FORCE]])

Os nomes de variáveis ​​diferenciam maiúsculas de minúsculas. Seus valores são do tipo string. O valor de uma variável é referenciado via:

${variable_name}

e é avaliado dentro de um argumento entre aspas

"${variable_name}/directory"

Variáveis ​​e o Cache de Variáveis ​​Globais #

Principalmente você usará “variáveis ​​normais”:

set(VAR TRUE)
set(VAR "main.cpp")
set(VAR1 ${VAR2})

Mas o CMake também conhece “variáveis ​​em cache” globais (persistentes em CMakeCache.txt). E se existirem variáveis ​​normais e em cache com o mesmo nome no escopo atual, as variáveis ​​normais ocultam as em cache:

cmake_minimum_required(VERSION 2.4)
project(VariablesTest)

set(VAR "CACHED-init" CACHE STRING "A test")
message("VAR = ${VAR}")

set(VAR "NORMAL")
message("VAR = ${VAR}")

set(VAR "CACHED" CACHE STRING "A test" FORCE)
message("VAR = ${VAR}")

Saída da primeira execução

VAR = CACHED-init
VAR = NORMAL
VAR = CACHED

Saída da segunda execução

VAR = CACHED
VAR = NORMAL
VAR = CACHED

Nota: A opção FORCE também desmarca/remove a variável normal do escopo atual.

Casos de uso para variáveis ​​em cache #

Normalmente, existem dois casos de uso (por favor, não os use mal para variáveis ​​globais):

  1. Um valor em seu código deve ser modificável pelo usuário do seu projeto, por exemplo. com o cmakegui, [ccmake](https://cmake.org/cmake/help /latest/manual/ccmake.1.html) ou com a opção cmake -D ...:

CMakeLists.txt / MyToolchain.cmake

   set(LIB_A_PATH "/some/default/path" CACHE PATH "Path to lib A")

Linha de comando

   $ cmake -D LIB_A_PATH:PATH="/some/other/path" ..

Isso pré-configura esse valor no cache e a linha acima não o modificará.

CMake GUI

Na GUI, o usuário primeiro inicia o processo de configuração, depois pode modificar qualquer valor armazenado em cache e termina com o início da geração do ambiente de construção.

  1. Além disso, o CMake armazena em cache os resultados de pesquisa/teste/identificação do compilador (portanto, ele não precisa fazer isso novamente sempre que executar novamente as etapas de configuração/geração)

    find_path(LIB_A_PATH libA.a PATHS "/some/default/path")

Aqui LIB_A_PATH é criado como uma variável em cache.

Strings e Listas #

É importante saber como o CMake distingue entre listas e strings simples. Quando você escreve:

set(VAR "a b c")

você cria uma string com o valor "a b c". Mas quando você escreve esta linha sem aspas:

set(VAR a b c)

Você cria uma lista de três itens: "a", "b" e "c".

Variáveis ​​que não são de lista também são listas (de um único elemento).

As listas podem ser operadas com o comando list(), que permite concatenar listas, pesquisá-las, acessar elementos arbitrários e assim por diante ([documentação de list()](https://cmake.org/cmake/help/latest /comando/lista.html)).

Um pouco confuso, uma lista também é uma string. A linha

set(VAR a b c)

é equivalente a

set(VAR "a;b;c")

Portanto, para concatenar listas também se pode usar o comando set():

set(NEW_LIST "${OLD_LIST1};${OLD_LIST2})"

Variável em cache (global) #

set(my_global_string "a string value"
    CACHE STRING "a description about the string variable")
set(my_global_bool TRUE
    CACHE BOOL "a description on the boolean cache entry")

Caso uma variável armazenada em cache já esteja definida no cache quando o CMake processa a respectiva linha (por exemplo, quando o CMake é executado novamente), ela não é alterada. Para substituir o padrão, acrescente FORCE como último argumento:

set(my_global_overwritten_string "foo"
    CACHE STRING "this is overwritten each time CMake is run" FORCE)

Adicionando sinalizadores de perfil ao CMake para usar gprof #

A série de eventos aqui deve funcionar da seguinte forma:

  1. Compile o código com a opção -pg
  2. Código do link com a opção -pg
  3. Execute o programa
  4. O programa gera o arquivo gmon.out
  5. Execute o programa gprof

Para adicionar sinalizadores de criação de perfil, você deve adicionar ao seu CMakeLists.txt:

SET(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -pg")
SET(CMAKE_EXE_LINKER_FLAGS "${CMAKE_EXE_LINKER_FLAGS} -pg")
SET(CMAKE_SHARED_LINKER_FLAGS "${CMAKE_SHARED_LINKER_FLAGS} -pg")

Isso deve adicionar sinalizadores para compilar e vincular e usar após executar o programa:

gprof ./my_exe

Se você receber um erro como:

gmon.out: No such file or directory

Isso significa que a compilação não adicionou informações de perfil corretamente.

Variável Local #

set(my_variable "the value is a string")

Por padrão, uma variável local é definida apenas no diretório atual e quaisquer subdiretórios adicionados através do comando add_subdirectory.

Para estender o escopo de uma variável, existem duas possibilidades:

  1. CACHE, que o tornará disponível globalmente

  2. use PARENT_SCOPE, que o tornará disponível no escopo pai. O escopo pai é o arquivo CMakeLists.txt no diretório pai ou o chamador da função atual.

    Technically the parent directory will be the CMakeLists.txt file that included the current file via the add_subdirectory command.
