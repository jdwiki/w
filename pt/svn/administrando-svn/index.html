Criar novo usuário #

Para adicionar usuário, use o seguinte comando

htpasswd /etc/subversion/passwd user_name

Especifique user_name com o nome de usuário que você deseja adicionar no comando acima. Ele pedirá para fornecer a senha para o usuário.

Se você está criando o primeiro usuário, você precisa adicionar –c switch no comando acima, que criará o arquivo.

htpasswd -c /etc/subversion/passwd user_name

Você pode verificar a existência do arquivo ou lista de usuários configurados usando o seguinte comando cat /etc/subversion/passwd

Você pode precisar executar os comandos acima como superusuário.

Crie grupos de usuários #

Os grupos podem ser definidos no arquivo /etc/subversion/svn_access_control.

Crie/edite o arquivo usando o seguinte comando

nano /etc/subversion/svn_access_control

Use a sintaxe especificada abaixo para definir grupos e atribuir membros.

[groups]
groupname = <list of users, comma separated>

por exemplo.

[groups]
myproject-dev = john, peter
myproject-support = maria, cristine

O exemplo acima criará dois grupos chamados myproject-dev e myproject-support. Ele adicionará os usuários john e peter ao grupo myproject-dev e os usuários maria e cristine ao grupo myproject-support.

Os grupos podem ser usados ​​para gerenciar o acesso ao repositório

Gerenciando permissões de repositório #

As especificações de acesso para repositórios do subversion são especificadas no arquivo etc/subversion/svn_access_control

Crie/edite o arquivo usando o seguinte comando

nano /etc/subversion/svn_access_control

Use a seguinte sintaxe para configurar permissões de acesso para repositórios para grupo/membros

[Repository:<Path>]
@groupname = r/rw
User = r

por exemplo.

[myproject:/]
@myproject-dev = rw
@myproject-support = r
jack = r
            
[myproject:/branches/support]
@myproject-support = rw
patrick = r

A configuração de exemplo acima concederá acesso de leitura e gravação a todo o repositório myproject para usuários pertencentes ao grupo myproject-dev, enquanto o acesso somente leitura é concedido a usuários pertencentes ao grupo myproject-support e usuário específico jack. Observe que os nomes dos grupos são precedidos por @.

Da mesma forma, ele atribuirá acesso de leitura/gravação à ramificação support do repositório myproject para todos os usuários pertencentes a myproject-support e acesso somente leitura a patrick.

Criando um novo repositório #

Novo repositório pode ser criado com duas opções diferentes:

1. Usando linha de comando #

Execute o seguinte comando. Ele criará um diretório para o repositório, mas o caminho pai deve estar presente. ou seja, no exemplo a seguir, /var/svn já deve estar lá, enquanto criará o diretório my_repository.

svnadmin create /var/svn/my_repository

Se você estiver usando o TortoiseSVN, você pode usar a GUI para criar o repositório.

  1. Abra o diretório no qual deseja criar um novo repositório.
  2. Clique com o botão direito na pasta e selecione TortoiseSVN -> Criar repositório aqui...
  3. Um repositório é então criado dentro da pasta selecionada. Não edite esses arquivos você mesmo! Se você receber algum erro, certifique-se de que a pasta esteja vazia e não protegida contra gravação.
