O que são Registros?

O processador pode operar com valores numéricos (números), mas estes devem ser armazenados em algum lugar primeiro. Os dados são armazenados principalmente na memória, ou dentro do opcode da instrução (que é armazenado geralmente na memória também), ou em uma memória especial no chip colocada diretamente no processador, que é chamada de registro.

Para trabalhar com valor no registrador, você não precisa endereçá-lo por endereço, mas “nomes” mnemônicos especiais são usados, como por exemplo ax em x86, ou A em Z80, ou r0 em ARM.

Alguns processadores são construídos de uma forma, onde quase todos os registradores são iguais e podem ser usados ​​para todos os propósitos (geralmente grupo RISC de processadores), outros possuem especialização distinta, quando apenas alguns registradores podem ser usados ​​para aritmética (“acumulador” em CPUs anteriores) e outros registradores apenas para endereçamento de memória, etc.

Esta construção usando a memória diretamente no chip do processador tem uma grande implicação de desempenho, adicionar dois números de registradores armazenando-o de volta ao registrador geralmente é feito no menor tempo possível por esse processador (Exemplo no processador ARM: ADD r2,r0,r1 sets r2 para o valor (r0 + r1), em ciclo de processador único).

Ao contrário, quando um dos operandos está referenciando um local de memória, o processador pode travar por algum tempo, esperando que o valor chegue do chip de memória (no x86 isso pode variar de zero espera por valores no cache L0 a centenas de A CPU dá um ciclo quando o valor não está em nenhum cache e precisa ser lido diretamente do chip DRAM da memória).

Portanto, quando o programador está criando algum código de processamento de dados, ele geralmente deseja ter todos os dados durante o processamento em registradores para obter o melhor desempenho. Se isso não for possível e forem necessárias leituras/gravações de memória, elas devem ser minimizadas e formar um padrão que coopere com a arquitetura de caches/memória da plataforma específica.

O tamanho nativo do registrador em bits é frequentemente usado para agrupar processadores, como Z80 sendo “processador de 8 bits” e 80386 sendo “processador de 32 bits” - embora esse agrupamento raramente seja claro. Por exemplo, Z80 opera também com pares de registradores, formando valor nativo de 16 bits, e a CPU 80686 de 32 bits possui instruções MMX para trabalhar com registradores de 64 bits nativamente.

Registros x64 #

A arquitetura x64 é a evolução da arquitetura x86 mais antiga, manteve a compatibilidade com seu antecessor (os registros x86 ainda estão disponíveis), mas também introduziu novos recursos:

  • Os registradores passaram a ter capacidade de 64 bits;
  • Existem mais 8 registradores de uso geral;
  • Os registradores de segmento são forçados a 0 no modo 64 bits;
  • Os 32, 16 e 8 bits inferiores de cada registro já estão disponíveis.

Observação

Os sufixos usados ​​para endereçar os bits mais baixos dos novos registradores significam:

  • Byte B, 8 bits;
  • palavra W, 16 bits;
  • Palavra dupla D, 32 bits.

Registros Zilog Z80 #

Registros: 8 bits: A, B, C, D, E, H, L, F, I, R, 16 bits: SP , PC, IX, IY, e sombras de alguns registradores 8b: A', B', C', D', E', H' , ‘L’ e ‘F’’.

A maioria dos registradores de 8 bits podem ser usados ​​também em pares como registradores de 16 bits: AF, BC, DE e HL.

SP é um ponteiro de pilha, marcando a parte inferior da memória da pilha (usado pelas instruções PUSH/POP/CALL/RET). PC é contador de programa, apontando para a instrução atualmente executada. I é Registro de interrupção, fornecendo byte alto de endereço da tabela de vetores para o modo de interrupção IM 2. R é refresh register, ele incrementa cada vez que a CPU busca um opcode (ou prefixo de opcode). Algumas instruções não oficiais existem em alguns processadores Z80 para manipular partes de 8 bits de IX como IXH:IXL e IY como IYH:IYL.

Variantes de sombra não podem ser acessadas diretamente por nenhuma instrução, a instrução EX AF,AF' irá alternar entre AF e AF', e a instrução EXX irá trocar BC,DE,HL por BC ',DE',HL'.

Carregando valor em um registrador:

    ; from other register
    LD   I,A        ; copies value in A into I (8 bit)
    LD   BC,HL      ; copies value in HL into BC (16 bit)
    ; directly with value encoded in instruction machine code
    LD   B,d8       ; 8b value d8 into B
    LD   DE,d16     ; 16b value d16 into DE
    ; from a memory (ROM/RAM)
    LD   A,(HL)     ; value from memory addressed by HL into A
    LD   A,(a16)    ; value from memory with address a16 into A
    LD   HL,(a16)   ; 16b value from memory with address a16 into HL
    POP  IX         ; 16b value popped from stack into IX
    LD   A,(IY+a8)  ; IX and IY allows addressing with 8b offset
    ; from I/O port (for writing value at I/O port use "OUT")
    IN   A,(C)      ; reads I/O port C, value goes to A

As combinações corretas de possíveis operandos de origem e destino são limitadas (por exemplo, LD H,(a16) não existe).

Armazenando valor em uma memória:

    LD   (HL),D     ; value D stored into memory addressed by HL
    LD   (a16),A    ; value A into memory with address a16
    LD   (a16),HL   ; value HL into 16b of memory with address a16
    LD   (IX+a8),d8 ; value d8 into memory at address IX+a8
    LD   (IY+a8),B  ; value B into memory at address IY+a8
    ; specials ;)
    PUSH DE         ; 16b value DE pushed to stack
    CALL a16        ; while primarily used for execution branching
      ; it also stores next instruction address into stack

Registros x86 #

No mundo de 32 bits, os registradores de uso geral se dividem em três classes gerais: os registradores de uso geral de 16 bits, os registradores estendidos de 32 bits registradores de uso geral e as metades de registradores de 8 bits. Essas três aulas não representam três conjuntos inteiramente distintos de registros. Os de 16 bits e Os registradores de 8 bits são, na verdade, nomes de regiões dentro dos registradores de 32 bits. Registro O crescimento na família de CPUs x86 ocorreu ao estendendo os registros existentes em CPUs mais antigas

Existem oito registradores de uso geral de 16 bits: AX, BX, CX, DX, BP, SI, DI, e SP; e você pode colocar qualquer valor neles que possa ser expresso em 16 bits ou menos.

Quando a Intel expandiu a arquitetura x86 para 32 bits em 1986, dobrou o tamanho de todos os oito registros e deu-lhes novos nomes prefixando um E na frente de cada nome de registro, resultando em EAX, EBX, ECX, EDX, EBP, ESI, EDI e ESP.

Com x86_64 veio outra duplicação do tamanho do registro, bem como a adição de alguns novos registros. Esses registradores têm 64 bits de largura e são nomeados (barra usada para mostrar o nome do registrador alternativo): RAX/r0, RBX/r3, RCX/r1, RDX/r2, RBP/r5, RSI/r6, RDI/r7, RSP/r4 , R8, R9, R10, R11, R12, R13, R14, R15.

Embora os registradores de uso geral possam ser tecnicamente usados ​​para qualquer coisa, cada registrador também tem um objetivo alternativo/principal:

  • AX (acumulador) é usado em operações aritméticas.
  • CX (contador) é usado nas instruções de deslocamento e rotação e usado para loops.
  • DX (dados) é usado em operações aritméticas e de E/S.
  • BX (base) usado como um ponteiro para dados (especificamente como um deslocamento para o registro de segmento DS quando em modo segmentado).
  • SP (pilha) aponta para o topo da pilha.
  • BP (base da pilha) aponta para a base da pilha.
  • SI (source) aponta para uma fonte na memória para operações de fluxo (por exemplo, lodsb).
  • DI (destino) aponta para um destino na memória para operações de fluxo (por exemplo, stosb).

Os registradores de segmento, usados ​​no modo segmentado, apontam para diferentes segmentos na memória. Cada registrador de segmento de 16 bits dá uma visão de 64k (16 bits) de dados. Depois que um registrador de segmento foi definido para apontar para um bloco de memória, registradores (como BX, SI e DI) podem ser usados ​​como deslocamentos para o registrador de segmento para que locais específicos no espaço de 64k possam ser acessado.

Os seis registradores de segmento e seus usos são:
