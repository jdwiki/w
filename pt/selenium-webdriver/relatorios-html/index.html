Este tópico aborda a criação de relatórios HTML para testes de selênio. Existem vários tipos de plugins disponíveis para relatórios e os mais utilizados são Allure, ExtentReports e ReportNG.

Relatórios de extensão #

Este exemplo cobre a implementação de ExtentReports no Selenium usando TestNG, Java e Maven.

ExtentReports estão disponíveis em duas versões, comunitária e comercial. Para fins de facilidade e demonstração, usaremos a versão da comunidade.

1. Dependência

Adicione a dependência em seu arquivo maven pom.xml para relatórios de extensão.

   <dependency>
        <groupId>com.aventstack</groupId>
        <artifactId>extentreports</artifactId>
        <version>3.0.6</version>
    </dependency>

2. Configurar plug-ins

Configure o plugin maven surefire conforme abaixo em pom.xml

    <build>
    <defaultGoal>clean test</defaultGoal>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.6.1</version>
            <configuration>
                <source>${jdk.level}</source>
                <target>${jdk.level}</target>
            </configuration>
        </plugin>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-surefire-plugin</artifactId>
            <version>2.19.1</version>
            <configuration>
                <suiteXmlFiles>
                    <suiteXmlFile>testng.xml</suiteXmlFile>
                </suiteXmlFiles>
            </configuration>
        </plugin>
    </plugins>
</build>

3. Teste de amostra com ExtentReports

Agora, crie um teste com o nome test.java

public class TestBase {
    WebDriver driver;

    ExtentReports extent;
    ExtentTest logger;
    ExtentHtmlReporter htmlReporter;
    String htmlReportPath = "C:\\Screenshots/MyOwnReport.html"; //Path for the HTML report to be saved

    @BeforeTest
    public void setup(){
        htmlReporter = new ExtentHtmlReporter(htmlReportPath);
        extent = new ExtentReports();
        extent.attachReporter(htmlReporter);

        System.setProperty("webdriver.chrome.driver", "pathto/chromedriver.exe");
        driver = new ChromeDriver();
    
    }

    @Test
    public void test1(){
        driver.get("http://www.google.com/");
        logger.log(Status.INFO, "Opened site google.com");
        assertEquals(driver.getTitle()), "Google");
        logger.log(Status.PASS, "Google site loaded");
    }

    @AfterMethod
    public void getResult(ITestResult result) throws Exception {
        if (result.getStatus() == ITestResult.FAILURE)
        {
            logger.log(Status.FAIL, MarkupHelper.createLabel(result.getName() + " Test case FAILED due to below issues:", ExtentColor.RED));
            logger.fail(result.getThrowable());
        } 
        else if (result.getStatus() == ITestResult.SUCCESS)
        {
            logger.log(Status.PASS, MarkupHelper.createLabel(result.getName() + " Test Case PASSED", ExtentColor.GREEN));
        } 
        else if (result.getStatus() == ITestResult.SKIP)
        {
            logger.log(Status.SKIP, MarkupHelper.createLabel(result.getName() + " Test Case SKIPPED", ExtentColor.BLUE));
        }
     }

    @AfterTest
    public void testend() throws Exception {
        extent.flush();
    }

    @AfterClass
    public void tearDown() throws Exception {
        driver.close();
    }

Relatórios de atração #

Este exemplo abrange a implementação do Allure Reports no Selenium usando TestNG, Java e Maven.

#Configuração Maven#

Repositório #

Adicione o seguinte código para configurar o repositório jcenter

<repository>
            <id>jcenter</id>
            <name>bintray</name>
            <url>http://jcenter.bintray.com</url>
</repository>

Dependência #

Adicione as seguintes dependências ao seu pom.xml

<dependency>
        <groupId>org.aspectj</groupId>
        <artifactId>aspectjweaver</artifactId>
        <version>${aspectj.version}</version>
    </dependency>
    <dependency>
        <groupId>ru.yandex.qatools.allure</groupId>
        <artifactId>allure-testng-adaptor</artifactId>
        <version>1.5.4</version>
    </dependency>

Configuração do Plugin Surefire #

<plugin>
        <groupId> org.apache.maven.plugins</groupId>
        <artifactId>maven-surefire-plugin</artifactId>
        <version>2.20</version>
        <configuration>
            <argLine>-javaagent:${settings.localRepository}/org/aspectj/aspectjweaver/${aspectj.version}/aspectjweaver-${aspectj.version}.jar
            </argLine>
            <properties>
                <property>
                    <name>listener</name>
                    <value>ru.yandex.qatools.allure.testng.AllureTestListener</value>
                </property>
            </properties>
            <suiteXmlFiles>testng.xml</suiteXmlFiles>
            <testFailureIgnore>false</testFailureIgnore>
        </configuration>
    </plugin>

Teste de amostra para o Relatório Allure #

Crie um teste de amostra com o nome test.java

public class test{
    WebDriver driver;
    WebDriverWait wait;

    @BeforeMethod
    public void setup() {
        System.setProperty("webdriver.chrome.driver", "path to/chromedriver.exe");
        driver = new ChromeDriver();
        driver.get("https://www.google.com/");
        wait = new WebDriverWait(driver,50);
    }

    @Title("Title check")
    @Description("Checking the title of the loaded page.")
    @Test
    public void searchTest(){
        String title = driver.getTitle();
        LogUtil.log("Title Fetched: "+title);
        assertEquals(title,"Google");
        LogUtil.log("Test Passed. Expected: Google | Actual: "+title);
        System.out.println("Page Loaded");
    }

    @AfterMethod
    public void teardown(){
        driver.close();
    }
}

Na classe acima usamos a classe LogUtil. Isso é feito simplesmente para registrar Etapas em nosso teste. Abaixo está o código para o mesmo

LogUtil.java

public final class LogUtil {

    private LogUtil() {
    }

    @Step("{0}")
    public static void log(final String message){
        //intentionally empty
    }
}

Aqui

@Title("") adicionará o título ao seu teste no Relatório Allure

@Description("") adicionará a descrição ao seu teste

@Step("") adicionará uma etapa no relatório de atração para o teste

Na execução será gerado um arquivo xml na pasta “target/allure-results/”

Relatório final com Jenkins

Se você estiver executando no Jenkins com o plug-in Allure Report instalado, o Jenkins renderizará automaticamente o relatório em seu trabalho.

Relatório final sem Jenkins

Para aqueles que não possuem um Jenkins, use a seguinte linha de comando para criar o relatório html. Allure CLI é um aplicativo Java, portanto, está disponível para todas as plataformas. Você precisa instalar manualmente o Java 1.7+ antes de usar o Allure CLI.

Debian

Para repositórios baseados em Debian, fornecemos um PPA para que a instalação seja simples: Instale o Allure CLI para debian

$ sudo apt-add-repository ppa:yandex-qatools/allure-framework
$ sudo apt-get update 
$ sudo apt-get install allure-commandline

As distribuições suportadas são: Trusty e Precise. Após a instalação, você terá o comando allure disponível.

Mac OS

Você pode instalar o Allure CLI via Homebrew.

$ brew tap qatools/formulas 
$ brew install allure-commandline

Após a instalação, você terá o comando allure disponível.

Windows e outros Unix

  1. Baixe a versão mais recente como arquivo zip de https://github.com/allure-framework/allure-core/releases/latest.
  2. Descompacte o arquivo no diretório allure-commandline. Navegue até a lixeira directory.
  3. Use allure.bat para Windows e allure para outros Unix platforms.

Na linha de comando/terminal agora basta digitar a seguinte sintaxe e o relatório será gerado na pasta allure-report

$ allure generate directory-with-results/
