Exceções no Selenium-WebDriver

  • Exceções do Python

Nesta página

  • Exceções do Python

Há uma série de exceções que podem ser lançadas ao usar um webdriver. Os exemplos abaixo pretendem dar uma ideia do que eles significam.

Exceções do Python #

Documentação de exceção do Selenium

ElementNotInteractableException: Lançado quando um elemento está presente no DOM, mas as interações com esse elemento atingem outro elemento devido à ordem de pintura

  • ElementNotSelectableException: Lançado ao tentar selecionar um elemento não selecionável. Exemplos de elementos não selecionáveis:
    • script
  • ElementNotVisibleException: Lançado quando um elemento está presente no DOM, mas não é visível e, portanto, não pode interagir. Mais comumente encontrado ao tentar clicar ou ler o texto de um elemento que está oculto da visualização.
  • ErrorInResponseException: Lançado quando ocorre um erro no lado do servidor. Isso pode acontecer ao se comunicar com a extensão do firefox ou o servidor de driver remoto.
  • ImeActivationFailedException: Lançado quando a ativação de um mecanismo IME falhou.
  • ImeNotAvailableException: Lançado quando o suporte IME não está disponível. Essa exceção é lançada para cada chamada de método relacionada a IME se o suporte a IME não estiver disponível na máquina.
  • InvalidArgumentException: Os argumentos passados ​​para um comando são inválidos ou malformados.
  • InvalidCookieDomainException: Lançado ao tentar adicionar um cookie em um domínio diferente do URL atual.
  • InvalidElementStateException: Lançado quando uma ação resultaria em um estado inválido para um elemento. Subclasses:
    • ElementNotInteractableException
    • ElementNotSelectableException
    • ElementNotVisibleException
  • InvalidSelectorException: Lançado quando o seletor usado para localizar um elemento não retorna um WebElement. Atualmente, isso só acontece quando o seletor é uma expressão xpath e é sintaticamente inválido (ou seja, não é uma expressão xpath) ou a expressão não seleciona WebElements (por exemplo, “count(//input)”).
  • InvalidSwitchToTargetException: Lançado quando o alvo de quadro ou janela a ser trocado não existe.
  • MoveTargetOutOfBoundsException: Lançado quando o destino fornecido ao método ActionsChains move() é inválido, ou seja, fora do documento.
  • NoAlertPresentException: Lançado ao alternar para nenhum alerta apresentado. Isso pode ser causado pela chamada de uma operação na classe Alert() quando um alerta ainda não está na tela.
  • NoSuchAttributeException: Lançado quando o atributo do elemento não pôde ser encontrado. Você pode querer verificar se o atributo existe no navegador específico que você está testando. Alguns navegadores podem ter nomes de propriedades diferentes para a mesma propriedade. (.innerText do IE8 vs. Firefox .textContent)
  • NoSuchElementException: Lançado quando o elemento não pôde ser encontrado. Se você encontrar essa exceção, verifique o seguinte:
    • Check your selector used in your find_by…
    • Element may not yet be on the screen at the time of the find operation, (webpage is still loading) see selenium.webdriver.support.wait.WebDriverWait() for how to write a wait wrapper to wait for an element to appear.
  • NoSuchFrameException: Lançado quando o alvo do quadro a ser trocado não existe.
  • NoSuchWindowException: Lançado quando o destino da janela a ser alternada não existe. Para encontrar o conjunto atual de identificadores de janela ativos, você pode obter uma lista dos identificadores de janela ativos da seguinte maneira:
    print driver.window_handles
  • RemoteDriverServerException:
  • StaleElementReferenceException: Lançado quando uma referência a um elemento agora está “obsoleta”. Stale significa que o elemento não aparece mais no DOM da página. As possíveis causas de StaleElementReferenceException incluem, mas não se limitam a:
    • You are no longer on the same page, or the page may have refreshed since the element was located.
    • The element may have been removed and re-added to the screen, since it was located. Such as an element being relocated. This can happen typically with a javascript framework when values are updated and the node is rebuilt.
    • Element may have been inside an iframe or another context which was refreshed.
  • TimeoutException: Lançado quando um comando não é concluído em tempo suficiente.
  • UnableToSetCookieException: Lançado quando um driver falha ao definir um cookie.
  • UnexpectedAlertPresentException: Lançado quando um alerta inesperado é exibido. Geralmente gerado quando um modal esperado está bloqueando o formulário do webdriver executando mais comandos.
  • UnexpectedTagNameException: Lançado quando uma classe de suporte não obteve um elemento da web esperado.
  • WebDriverException: exceção de webdriver base. Todas as exceções de webdriver usam WebDriverException ou InvalidStateException como a classe pai.
