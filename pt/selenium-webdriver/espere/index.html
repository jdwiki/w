Tipos de espera no Selenium WebDriver #

Ao executar qualquer aplicativo da web, é necessário levar em consideração o tempo de carregamento. Se seu código tentar acessar qualquer elemento que ainda não esteja carregado, o WebDriver lançará uma exceção e seu script será interrompido.

Existem três tipos de esperas -

  • Esperas Implícitas
  • Esperas explícitas
  • Esperas Fluentes

As esperas implícitas são usadas para definir o tempo de espera ao longo do programa, enquanto as esperas explícitas são usadas apenas em partes específicas.

Espera implícita #

Uma espera implícita é dizer ao WebDriver para pesquisar o DOM por um determinado período de tempo ao tentar localizar um elemento ou elementos se eles não estiverem disponíveis imediatamente. Esperas implícitas são basicamente sua maneira de informar ao WebDriver a latência que você deseja ver se o elemento da Web especificado não estiver presente que o WebDriver está procurando. A configuração padrão é 0. Uma vez definida, a espera implícita é definida para a vida útil da instância do objeto WebDriver. A espera implícita é declarada na parte de instanciação do código usando o snippet a seguir.

Exemplo em Java:

driver.manage().timeouts().implicitlyWait(15, TimeUnit.SECONDS);
// You need to import the following class - import java.util.concurrent.TimeUnit;

Exemplo em C#:

driver.Manage().Timeouts().ImplicitlyWait(TimeSpan.FromSeconds(15));

Então, neste caso, você está dizendo ao WebDriver que ele deve esperar 15 segundos caso o elemento especificado não esteja disponível na UI (DOM).

Espera explícita #

Você pode encontrar casos em que algum elemento leva mais tempo para carregar. Definir espera implícita para esses casos não faz sentido, pois o navegador aguardará desnecessariamente o mesmo tempo para cada elemento, aumentando o tempo de automação. A espera explícita ajuda aqui, ignorando completamente a espera implícita para alguns elementos específicos.

Esperas explícitas são esperas inteligentes confinadas a um determinado elemento da web. Usando esperas explícitas, você está basicamente dizendo ao WebDriver no máximo que é esperar por X unidades de tempo antes de desistir.

As esperas explícitas são feitas usando as classes WebDriverWait e ExpectedConditions. No exemplo abaixo, devemos esperar até 10 segundos para que um elemento cujo id seja username fique visível antes de prosseguir para o próximo comando. Aqui estão os passos.

Exemplo em Java:

//Import these two packages:
import org.openqa.selenium.support.ui.ExpectedConditions;
import org.openqa.selenium.support.ui.WebDriverWait;

//Declare a WebDriverWait variable. In this example, we will use myWaitVar as the name of the variable.
WebDriverWait myWaitVar = new WebDriverWait(driver, 30);

//Use myWaitVar with ExpectedConditions on portions where you need the explicit wait to occur. In this case, we will use explicit wait on the username input before we type the text tutorial onto it.
myWaitVar.until(ExpectedConditions.visibilityOfElementLocated(By.id(“username”)));
driver.findElement(By.id(“username”)).sendKeys(“tutorial”);

A classe ExpectedConditions tem algumas condições comuns predefinidas para aguardar um elemento. Clique aqui para ver a lista dessas condições na vinculação Java.

Exemplo em C#:

using OpenQA.Selenium;
using OpenQA.Selenium.Support.UI;
using OpenQA.Selenium.PhantomJS;

// You can use any other WebDriver you want, such as ChromeDriver.
using (var driver = new PhantomJSDriver())
{
    driver.Navigate().GoToUrl("http://somedomain/url_that_delays_loading");

    // We aren't going to use it more than once, so no need to declare this a variable.
    new WebDriverWait(driver, TimeSpan.FromSeconds(10))
        .Until(ExpectedConditions.ElementIsVisible(By.Id("element-id")));

    // After the element is detected by the previous Wait, 
    // it will display the element's text
    Console.WriteLine(driver.FindElement(By.Id("element-id")).Text);
}

Neste exemplo, o sistema aguardará 10 segundos até que o elemento fique visível. Se o elemento não estiver visível após o tempo limite, o WebDriver lançará um WebDriverTimeoutException.

Observação: se o elemento estiver visível antes do tempo limite de 10 segundos, o sistema prosseguirá imediatamente para o processo adicional.

Espera fluente #

Ao contrário da espera implícita e explícita, a espera fluente usa dois parâmetros. Valor de tempo limite e frequência de sondagem. Digamos que temos um valor de tempo limite de 30 segundos e uma frequência de pesquisa de 2 segundos. O WebDriver verificará o elemento a cada 2 segundos até o valor do tempo limite (30 segundos). Depois que o valor do tempo limite é excedido sem nenhum resultado, a exceção é lançada. Abaixo está um código de exemplo que mostra a implementação da espera fluente.

Exemplo em Java:

Wait wait = new FluentWait(driver).withTimeout(30, SECONDS).pollingEvery(2, SECONDS).ignoring(NoSuchElementException.class);


WebElement testElement = wait.until(new Function() {
    public WebElement apply(WebDriver driver) {
        return driver.findElement(By.id("testId"));
    }
});

Outra vantagem de usar a espera fluente é que podemos ignorar tipos específicos de exceções (por exemplo, NoSuchElementExceptions) enquanto aguardamos. Devido a todas essas disposições, a espera fluente é útil em aplicativos AJAX, bem como em cenários em que o tempo de carregamento do elemento flutua com frequência. O uso estratégico da espera fluente melhora significativamente os esforços de automação.

Diferentes tipos de condições de espera explícitas #

Na espera explícita, você espera que uma condição aconteça. Por exemplo, você deseja esperar até que um elemento seja clicável.

Aqui está uma demonstração de alguns problemas comuns.

Observação: Em todos esses exemplos, você pode usar qualquer By como localizador, como classname, xpath, link text, tag name ou cssSelector

Aguarde até que o elemento esteja visível #

Por exemplo, se o seu site demorar um pouco para carregar, você pode esperar até que a página termine de carregar e seu elemento fique visível para o WebDriver.

C#

WebDriverWait wait = new WebDriverWait(driver, TimeSpan.FromSeconds(10));
wait.Until(ExpectedConditions.ElementIsVisible(By.Id("element-id")));

Java

WebDriverWait wait = new WebDriverWait(driver, 10);
wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("element-id")));

Aguarde até que o elemento não esteja mais visível #

Igual ao anterior, mas invertido.

C#

WebDriverWait wait = new WebDriverWait(driver, TimeSpan.FromSeconds(10));
wait.Until(ExpectedConditions.InvisibilityOfElementLocated(By.Id("element-id")));

Java

WebDriverWait wait = new WebDriverWait(driver, 10);
wait.until(ExpectedConditions.invisibilityOfElementLocated(By.id("element-id")));

Aguarde até que o texto esteja presente no elemento especificado

C#

IWebElement element = driver.FindElement(By.Id("element-id"));

WebDriverWait wait = new WebDriverWait(driver, TimeSpan.FromSeconds(10));
wait.Until(ExpectedConditions.TextToBePresentInElement(element, "text"));

Java

WebElement element = driver.findElement(By.id("element-id"));

WebDriverWait wait = new WebDriverWait(driver, 10);
wait.until(ExpectedConditions.textToBePresentInElement(element, "text"));

Se você acessar o link fornecido acima, verá todas as condições de espera lá.

A diferença entre o uso dessas condições de espera está em seu parâmetro de entrada.

Isso significa que você precisa passar o WebElement se seu parâmetro de entrada for WebElement, você precisa passar o localizador de elemento se ele usar o localizador By como seu parâmetro de entrada.

Escolha sabiamente que tipo de condição de espera você deseja usar.

Aguardando a conclusão das solicitações do Ajax #

C# #

using OpenQA.Selenium
using OpenQA.Selenium.Chrome;
using System.Threading;

namespace WebDriver Tests
{
    class WebDriverWaits
    {
        static void Main()
        {
            IWebDriver driver = new ChromeDriver(@"C:\WebDriver");
            
            driver.Navigate().GoToUrl("page with ajax requests");
            CheckPageIsLoaded(driver);

            // Now the page is fully loaded, you can continue with further tests.
        }

        private void CheckPageIsLoaded(IWebDriver driver)
        {
            while (true)
            {
                bool ajaxIsComplete = (bool)(driver as IJavaScriptExecutor).ExecuteScript("return jQuery.active == 0");
                if (ajaxIsComplete)
                    return;
                Thread.Sleep(100);
            }
        }
    }
}

Este exemplo é útil para páginas onde são feitas requisições ajax, aqui usamos o IJavaScriptExecutor para rodar nosso próprio código JavaScript. Como está dentro de um loop while, ele continuará a ser executado até ajaxIsComplete == true e assim a instrução return é executada.

Verificamos se todas as requisições ajax estão completas confirmando que jQuery.active é igual a 0. Isso funciona porque cada vez que uma nova requisição ajax é feita jQuery.active é incrementado e cada vez que uma requisição complementa é diminuída, a partir disso podemos deduzir que quando jQuery.active == 0 todas as requisições ajax devem estar completas.

Espera Fluente #

Fluent wait é uma superclasse de explicit wait (WebDriverWait) que é mais configurável, pois pode aceitar um argumento para a função wait. Vou passar a espera implícita, já que é uma prática recomendada evitá-la.

Uso (Java):

Wait wait = new FluentWait<>(this.driver)
        .withTimeout(driverTimeoutSeconds, TimeUnit.SECONDS)
        .pollingEvery(500, TimeUnit.MILLISECONDS)
        .ignoring(StaleElementReferenceException.class)
        .ignoring(NoSuchElementException.class)
        .ignoring(ElementNotVisibleException.class);

WebElement foo = wait.until(ExpectedConditions.presenceOfElementLocated(By.yourBy));

// or use your own predicate:
WebElement foo = wait.until(new Function() {
  public WebElement apply(WebDriver driver) {
    return element.getText().length() > 0;
  }
});

Quando você usa Explicit wait com seus padrões é simplesmente um FluentWait<WebDriver> com padrões de: DEFAULT_SLEEP_TIMEOUT = 500; e ignorando NotFoundException.

Espera fluente #

Cada instância de FluentWait define o tempo máximo de espera por uma condição, bem como a frequência com que verificar a condição. Além disso, o usuário pode configurar a espera para ignorar tipos específicos de exceções enquanto espera, como NoSuchElementExceptions ao procurar um elemento na página. Está associado ao driver.

Wait<WebDriver> wait = new FluentWait<WebDriver>(driver)
.withTimeout(30, SECONDS) //actuall wait for the element to pe present
.pollingEvery(5, SECONDS) //selenium will keep looking for the element after every 5seconds
.ignoring(NoSuchElementException.class); //while ignoring this condition
wait.until(ExpectedConditions.visibilityOf(driver.findElement(By.id("username"));
