O que é o Selenium WebDriver? #

Selenium é um conjunto de ferramentas projetadas para automatizar navegadores. É comumente usado para testes de aplicativos da Web em várias plataformas. Existem algumas ferramentas disponíveis sob o guarda-chuva do Selenium, como o Selenium WebDriver (ex-Selenium RC), o Selenium IDE e o Selenium Grid.

WebDriver é uma interface de controle remoto que permite manipular elementos DOM em páginas da web, bem como comandar o comportamento de agentes do usuário. Essa interface fornece um [protocolo de fio] de linguagem neutra 2 que foi implementado para várias plataformas, como:

  • GeckoDriver (Mozilla Firefox)
  • ChromeDriver (Google Chrome)
  • SafariDriver (Apple Safari)
  • InternetExplorerDriver (MS InternetExplorer)
  • MicrosoftWebDriver ou EdgeDriver (MS Edge)
  • OperaChromiumDriver (navegador Opera)

bem como outras implementações:

  • EventFiringWebDriver
  • HtmlUnitDriver
  • PhantomJSDriver
  • RemoteWebDriver

Selenium WebDriver é uma das ferramentas Selenium que fornece APIs Orientadas a Objetos em uma variedade de linguagens para permitir mais controle e a aplicação de práticas padrão de desenvolvimento de software. Para simular com precisão a maneira como um usuário interagirá com um aplicativo da Web, ele usa “Eventos de nível de sistema operacional nativo” em oposição a “Eventos de JavaScript sintetizados”.

Links para referência:

  • http://www.seleniumhq.org/
  • http://www.aosabook.org/en/selenium.html
  • https://www.w3.org/TR/webdriver/

Instalação ou configuração #

Para começar a usar o WebDriver, você precisará obter o driver relevante no site do Selenium: Downloads do Selenium HQ. A partir daqui, você precisa baixar o driver relevante para o(s) navegador(es) e/ou plataforma(s) em que está tentando executar o WebDriver, por exemplo, se você estava testando no Chrome, o site do Selenium o direcionará para:

https://sites.google.com/a/chromium.org/chromedriver/

Para baixar chromedriver.exe.

Por fim, antes de poder usar o WebDriver, você precisará baixar as ligações de idioma relevantes, por exemplo. se estiver usando C#, você pode acessar o download da página Downloads do Selenium HQ para obter os arquivos .dll necessários ou, alternativamente, baixá-los como pacotes no Visual Studio por meio do gerenciador de pacotes NuGet.

Os arquivos necessários devem ser baixados agora, para informações sobre como começar a usar o WebDriver, consulte a outra documentação do selenium-webdriver.

Para Visual Studio [NuGet] #

A maneira mais fácil de instalar o Selenium WebDriver é usando um gerenciador de pacotes NuGet.

Em seu projeto, clique com o botão direito do mouse em “Referências” e clique em “Gerenciar pacotes NuGet”, conforme mostrado:

Em seguida, digite na caixa de pesquisa “webdriver”. Você deve então ver algo assim:

Instale “Selenium.WebDriver” e “Selenium.Support” (o pacote de suporte inclui recursos adicionais, como Aguardar) clicando no botão Instalar no lado direito.

Em seguida, você pode instalar os WebDrivers que deseja usar, como um destes:

  • Selenium.WebDriver.ChromeDriver (Google Chrome)

  • PhantomJS (sem cabeça)

Instalação ou configuração para Java #

Para escrever testes usando Selenium Webdriver e Java como linguagem de programação, você precisará baixar arquivos JAR do Selenium Webdriver do site da Selenium.

Existem várias maneiras de configurar um projeto Java para o webdriver Selenium, uma das mais fáceis de todas é usando o Maven. O Maven baixa as ligações Java necessárias para o Selenium webdriver, incluindo todas as dependências. A outra maneira é baixar os arquivos JAR e importá-los para o seu projeto.

Etapas para configurar o projeto Selenium Webdriver usando o Maven:

  1. Instale o maven na caixa do Windows seguindo este documento: https://maven.apache.org/install.html

  2. Crie uma pasta com o nome selenium-learing

  3. Crie um arquivo na pasta acima usando qualquer editor de texto com o nome pom.xml

  4. Copie o conteúdo abaixo para pom.xml

     <project xmlns="http://maven.apache.org/POM/4.0.0"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
         <modelVersion>4.0.0</modelVersion>
         <groupId>SeleniumLearning</groupId>
         <artifactId>SeleniumLearning</artifactId>
         <version>1.0</version>
         <dependencies>
             <dependency>
                 <groupId>org.seleniumhq.selenium</groupId>
                 <artifactId>selenium-learning</artifactId>
                 <version>3.0.0-beta1</version>
             </dependency>
         </dependencies>
 </project>

Observação: Certifique-se de que a versão especificada acima seja a mais recente. Você pode verificar a versão mais recente aqui: http://docs.seleniumhq.org/download/maven.jsp

  1. Usando a linha de comando, execute o comando abaixo no diretório do projeto.

    mvn clean install

O comando acima fará o download de todas as dependências necessárias e as adicionará ao projeto. 6. Escreva o comando abaixo para gerar um projeto Eclipse que você pode importar para o Eclipse IDE.

`mvn eclipse:eclipse`
  1. Para importar o projeto para o Eclipse ide, você pode seguir os passos abaixo

Abrir Elipse -> Arquivo -> Importar -> Geral -> Projeto Existente no Workspace -> Próximo -> Navegar -> Localize a pasta que contém pom.xml -> Ok -> Concluir

Instale o plugin m2eclipse clicando com o botão direito do mouse em seu projeto e selecione Maven -> Enable Dependency Management.

Etapas para configurar o projeto Selenium Webdriver usando arquivos Jar

  1. Crie um novo projeto no Eclipse seguindo as etapas abaixo.

Abrir Elipse -> Arquivo -> Novo -> Projeto Java -> Forneça um nome (selenium-learning) -> Concluir

  1. Faça download dos arquivos jar em http://www.seleniumhq.org/download/. Você precisa baixar Selenium Standalone Server e Selenium Client & WebDriver Language Bindings. Como este documento está falando sobre Java, você precisa baixar apenas o jar da seção Java. Dê uma olhada na captura de tela anexada.

Nota: O Selenium Standalone Server só é necessário se quiser usar o servidor remoto para executar os testes. Como este documento está acima da configuração do projeto, é melhor ter tudo no lugar.

  1. Os jars serão baixados em arquivo zip, descompacte-os. Você deve ser capaz de ver .jar diretamente.
  1. No eclipse, clique com o botão direito do mouse no projeto que você criou na etapa 1 e siga os passos abaixo.

Propriedades -> Caminho de compilação Java -> Selecione a guia Bibliotecas -> Clique em Adicionar Jars Externos -> Localize a pasta jar descompactada que você baixou acima -> Selecione todos os jars da pasta lib -> Clique em Ok -> Novamente clique em Adicionar Jars Externos -> Localize a mesma pasta descompactada -> Selecione o jar que está fora da pasta lib (client-combined-3.0.0-beta1-nodeps.jar) -> Ok

Da mesma forma, adicione o Selenium Standalone Server seguindo o passo acima.

  1. Agora você pode começar a escrever código de selênio em seu projeto.

PS: A documentação acima é baseada na versão beta do selenium-3.0.0, portanto, os nomes dos arquivos jar especificados podem mudar com a versão.
