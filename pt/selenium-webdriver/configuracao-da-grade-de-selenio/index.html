O Selenium Grid é uma estrutura para executar testes distribuídos em uma variedade de dispositivos de teste. É usado para testar aplicativos da web. É possível escrever testes em diferentes linguagens de programação populares, incluindo C#, Groovy, Java, Perl, PHP, Python e Ruby. Os testes podem ser executados em vários navegadores da Web em plataformas como Windows, Linux e OS X.

É um software de código aberto, lançado sob a licença Apache 2.0: os desenvolvedores da web podem baixá-lo e usá-lo gratuitamente.

Sintaxe #

  • para executar o arquivo jar, o seguinte é a sintaxe para cada arquivo jar
  • java -jar <jar-file-full-name>.jar -<your parameters if any>

Parâmetros #

Parâmetros Detalhes
papel É o que diz ao selênio qual era hub ou node
porto Isto é para especificar qual porta o hub ou node deve estar escutando.
centro Este parâmetro é usado em node para especificar o hub url
nome do navegador Tem sido usado em node para especificar o nome do navegador como firefox, chrome, internet explorer
maxInstances É onde a instância do navegador está sendo especificada, por exemplo. 5 significa que haverá 5 instâncias do navegador que o usuário especificou estará presente.
nodeConfig Um arquivo de configuração Json para o nó. Você pode especificar a função, porta etc. aqui
hubConfig Um arquivo de configuração Json para o nó. Você pode especificar a função, porta, instâncias máximas etc. aqui

Criando um hub e nó do Selenium Grid #

Criando um hub #

Uma configuração rápida para uma configuração de hub e nó na grade de selênio. Para obter mais informações, consulte: Documentos da grade 2

Requisitos #

Para configurar um hub de grade, você precisa do fluxo:

  • Selenium-server-standalone-<Versão>.jar

Criando o hub #

Para criar um Hub, você precisa executar o servidor selenium.

  1. Faça o download do Selenium-server-standalone-.jar
  2. Abra seu terminal e navegue até a pasta onde o Selenium-server-standalone-.jar está
  3. Execute o seguinte comando:
    1. For default configuration java -jar selenium-server-standalone-<Version>.jar -role hub
    2. For Json configuration java -jar selenium-server-standalone-<Version>.jar -role hub -hubConfig hubConfig.json
  4. Abra http://localhost:4444/ você verá uma mensagem a seguir

Ao clicar em console -> View config para ver a configuração dos detalhes do hub.

Criando um nó #

Requisitos #

Para configurar um hub de grade, você precisa do fluxo:

  • Selenium-server-standalone-<Versão>.jar
  • Webdrivers
    • Chrome driver
    • FireFox driver
    • Microsoft Edge driver
  • Navegadores
    • Chrome
    • FireFox
    • Microsoft Edge (Windows 10)

Criando o nó #

Agora para criar nós para o Hub

  1. Faça o download do Selenium-server-standalone-.jar
  2. Baixe os navegadores em que deseja testar
  3. Baixe os drivers para os navegadores que deseja testar
  4. Abra o novo terminal e navegue até o local do arquivo jar do servidor selenium
  5. Execute o seguinte comando:
    1. for default configuration java -jar selenium-server-standalone-<VERSION NUMBER>.jar -role node
    2. For Json configuration java -jar selenium-server-standalone-<Version>.jar -role node -nodeConfig nodeConfig.json
  6. Agora vá para http://localhost:4444/grid/console para ver os detalhes do nó

Código Java para Selenium Grid #

String hubUrl = "http://localhost:4444/wd/hub"       
DesiredCapabilities capability = DesiredCapabilities.firefox(); //or which browser you want
RemoteWebDriver driver = new RemoteWebDriver(hubUrl, capability);
