Criando sua primeira página hello world #

Crie um novo aplicativo Rails hello-world a partir do comando no Windows e Terminal no Linux.

rails new hello-world

Agora mova para o novo diretório de aplicativos

cd hello-world

Agora gere um controlador

rails generate controller hello_world index

Aqui index é o nome do método no controlador hello_world. Você pode verificar abrindo o arquivo app/controllers/hello_world_controller.rb no diretório de sua aplicação. Código fica assim:

class HelloWorldController < ApplicationController
  def index
  end
end

Um route é adicionado automaticamente em seu arquivo config/routes.rb que aponta para o seu método. Veja o código em seu arquivo routes.rb.

    Rails.application.routes.draw do
      get 'hello_world/index'

      # For details on the DSL available within this file, see http://guides.rubyonrails.org/routing.html
   end

Agora abra o arquivo app/views/hello_world/index.html.rb Limpe todo o conteúdo e escreva

Olá, esta é minha primeira página do Rails.

Inicie o servidor Rails:

rails server

Abra este URL no seu navegador:

http://localhost:3000/hello_world/

Você deveria ver:

Hello, this is my first rails page

Faça sua nova página, sua página inicial. No arquivo routes.rb na pasta config remova a linha get 'hello_world/index' e adicione:

root 'hello_world#index'

Agora abra: http://localhost:3000/ Você verá: ‘Olá, este é meu primeiro trilho’ Você terminou.

Configurar Ruby On Rails no Ubuntu 17.04 Zesty Zapus #

Isso levará cerca de 30 minutos. Estaremos configurando o ambiente de desenvolvimento Ruby on Rails no Ubuntu 16.10 Yakkety Yak. Você vai querer baixar a versão mais recente do Desktop aqui: http://releases.ubuntu.com/17.04/

Abra seu terminal usando Ctrl + Alt + T.

Instalando Ruby #

O primeiro passo é instalar as dependências para Ruby.

sudo apt-get update
sudo apt-get install git-core curl zlib1g-dev build-essential libssl-dev libreadline-dev libyaml-dev libsqlite3-dev sqlite3 libxml2-dev libxslt1-dev libcurl4-openssl-dev python-software-properties libffi-dev nodejs

Estaremos usando Ruby versão 2.4.0 e é recomendado. Instale Ruby usando um dos três métodos. Cada um tem seus próprios benefícios, a maioria das pessoas prefere o rbenv, mas se você estiver familiarizado com o rvm, vá em frente.

Nota: Apenas usuários rbenv precisam executar rbenv rehash após instalar o bundler.

Instalando com rbenv: #

cd
git clone https://github.com/rbenv/rbenv.git ~/.rbenv
echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(rbenv init -)"' >> ~/.bashrc
exec $SHELL

git clone https://github.com/rbenv/ruby-build.git ~/.rbenv/plugins/ruby-build
echo 'export PATH="$HOME/.rbenv/plugins/ruby-build/bin:$PATH"' >> ~/.bashrc
exec $SHELL

rbenv install 2.4.0
rbenv global 2.4.0
ruby -v

Em seguida, instale o Bundler:

gem install bundler

Use rbenv rehash após instalar o bundler.

Instalando com rvm: #

sudo apt-get install libgdbm-dev libncurses5-dev automake libtool bison libffi-dev
gpg --keyserver hkp://keys.gnupg.net --recv-keys 409B6B1796C275462A1703113804BB82D39DC0E3
curl -sSL https://get.rvm.io | bash -s stable
source ~/.rvm/scripts/rvm
rvm install 2.4.0
rvm use 2.4.0 --default
ruby -v

Em seguida, instale o Bundler:

gem install bundler

Instalando com fonte: #

cd
wget http://ftp.ruby-lang.org/pub/ruby/2.4/ruby-2.4.0.tar.gz
tar -xzvf ruby-2.4.0.tar.gz
cd ruby-2.4.0/
./configure
make
sudo make install
ruby -v

Então o último passo é instalar o Bundler:

gem install bundler

Instalando Trilhos #

Rails vem com tantas dependências hoje em dia, vamos precisar instalar um run-time JavaScript como o NodeJS. Isso permite que você use o Coffee-script e o Asset Pipeline no Rails que combina e minimiza seu JavaScript para fornecer um ambiente de produção mais rápido.

Instale o NodeJS usando o repositório oficial:

curl -sL https://deb.nodesource.com/setup_4.x | sudo -E bash -
sudo apt-get install -y nodejs

Agora instale o Rails 5.0.1:

gem install rails -v 5.0.1

Se você estiver usando rbenv, execute o seguinte comando para disponibilizar o executável do Rails:

 rbenv rehash

Agora o Rails está instalado, execute rails -v para garantir que o Rails esteja instalado corretamente:

rails -v
# Rails 5.0.1

Se você estiver obtendo um resultado diferente por algum motivo, isso significa que seu ambiente pode não estar configurado corretamente.

Configurando o PostgreSQL #

Rails vem com sqlite3 como banco de dados padrão. É provável que você não queira usá-lo porque está armazenado como um arquivo simples no disco. Você provavelmente vai querer algo mais robusto como MySQL ou PostgreSQL.

Há muita documentação sobre ambos, então você pode escolher um que pareça mais confortável.

sudo sh -c "echo 'deb http://apt.postgresql.org/pub/repos/apt/ xenial-pgdg main' > /etc/apt/sources.list.d/pgdg.list"
wget --quiet -O - http://apt.postgresql.org/pub/repos/apt/ACCC4CF8.asc | sudo apt-key add -
sudo apt-get update
sudo apt-get install postgresql-common
sudo apt-get install postgresql-9.5 libpq-dev

A instalação do PostgreSQL não configura um usuário para você, portanto, você precisará seguir estas etapas para criar um usuário com permissão para criar bancos de dados. Sinta-se à vontade para substituir Hizqeel pelo seu nome de usuário.

sudo -u postgres createuser hizqeel -s

# If you would like to set a password for the user, you can do the following
sudo -u postgres psql
postgres=# \password hizqeel

Instalação ou configuração #

Na plataforma Windows

Etapa 1: Instalação do Ruby

Se você já instalou o Ruby no seu PC, pode pular esta etapa.

  • Vá para a página de download do rubyinstaller.org e baixe uma das versões estáveis ​​do ruby ​​correspondente à arquitetura da sua plataforma Windows.
  • Baixe novamente o kit de desenvolvimento para sua versão ruby ​​formulário aqui.
  • Agora instale o ruby.
  • Este é o momento de aprimorar sua instalação do ruby ​​com o kit de desenvolvimento do ruby. Isso é exigido por algumas das gemas para compilar no seu pc.
  • Extraia o arquivo do kit de desenvolvimento ruby ​​em uma pasta próxima ao caminho de instalação do ruby ​​C:\Ruby-**. Por exemplo C:\rubyDevkit.
  • Agora abra cmd.exe e vá para o diretório onde você extraiu seu devkit.
  • Em seguida, execute este comando ruby dk.rb init. Isso inicializará sua instalação. Então execute ruby dk.rb install.

Etapa 2: Instalar trilhos

  • após a instalação bem sucedida do ruby. O próximo passo é instalar trilhos. Antes de instalar o Rails, instale o bundler gem executando o gem install bundler na janela do prompt de comando.

  • Após instalar o bundler agora execute gem install rails -v version_of_rails por exemplo -v 5.0.0.1.

  • Se todo o processo for concluído sem erros, você instalou os trilhos com sucesso.
