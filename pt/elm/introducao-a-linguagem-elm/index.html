Instalação #

Para iniciar o desenvolvimento com Elm, você precisa instalar um conjunto de ferramentas chamado elm-platform.

Inclui: elm-make, elm-reator, elm-repl e elm-package.

Todas essas ferramentas estão disponíveis através do CLI, ou seja, você pode usá-las a partir do seu terminal.

Escolha um dos seguintes métodos para instalar o Elm:

Usando o instalador #

Baixe o instalador do site oficial e siga o assistente de instalação.

Usando npm #

Você pode usar o Node Package Manager para instalar a plataforma Elm.

Instalação global:

$ npm install elm -g

Instalação local:

$ npm install elm

As ferramentas da plataforma Elm instaladas localmente são acessíveis por meio de:

$ ./node_modules/.bin/elm-repl  # launch elm-repl from local node_modules/

Usando homebrew #

$ brew install elm

Alternar entre versões com elm-use #

Instale o elm-use

$ npm install -g elm-use

Mudar para uma versão mais antiga ou mais recente do elm

$ elm-use 0.18  // or whatever version you want to use

Saiba como [Inicializar e construir][8] seu primeiro projeto.

Inicialize e construa #

Você deve ter a plataforma Elm instalada em seu computador, o tutorial a seguir foi escrito com a suposição de que você está familiarizado com o terminal.

Inicialização #

Crie uma pasta e navegue até ela com seu terminal:

$ mkdir elm-app
$ cd elm-app/

Inicialize o projeto Elm e instale as dependências principais:

$ elm-package install -y

As pastas elm-package.json e elm-stuff devem aparecer em seu projeto.

Crie o ponto de entrada para seu aplicativo Main.elm e cole o exemplo Hello World nele.

Construindo o projeto #

Para construir seu primeiro projeto, execute:

$ elm-make Main.elm

Isso produzirá index.html com o arquivo Main.elm (e todas as dependências) compilado em JavaScript e embutido no HTML. Experimente abri-lo em seu navegador!

Se isso falhar com o erro ‘Não consigo encontrar o módulo ‘Html’.` significa que você não está usando a versão mais recente do Elm. Você pode resolver o problema atualizando o Elm e refazendo a primeira etapa ou com o seguinte comando:

$ elm-package install elm-lang/html -y

Caso você tenha seu próprio arquivo index.html (por exemplo, ao trabalhar com ports), você também pode compilar seus arquivos Elm para um arquivo JavaScript:

$ elm-make Main.elm --output=elm.js

Mais informações no exemplo Embedding into HTML.

Guia de estilo e formato olmo #

O guia de estilo oficial está localizado na página inicial e geralmente vai para:

  • legibilidade (em vez de compacidade)
  • facilidade de modificação
  • diferenças limpas

Isso significa que, por exemplo, isso:

homeDirectory : String
homeDirectory =
  "/root/files"


evaluate : Boolean -> Bool
evaluate boolean =
  case boolean of
    Literal bool ->
        bool

    Not b ->
        not (evaluate b)

    And b b' ->
        evaluate b && evaluate b'

    Or b b' ->
        evaluate b || evaluate b'

é considerado melhor do que:

homeDirectory = "/root/files"

eval boolean = case boolean of
    Literal bool -> bool
    Not b        -> not (eval b)
    And b b'     -> eval b && eval b'
    Or b b'      -> eval b || eval b'

A ferramenta elm-format ajuda a formatar seu código-fonte para você automaticamente (normalmente ao salvar), de maneira semelhante ao gofmt da linguagem Go . Novamente, o valor subjacente é ter um estilo consistente e salvar argumentos e flamewars sobre vários problemas, como tabs x espaços ou comprimento de recuo.

Você pode instalar o elm-format seguindo as instruções no [Github repo](https://github.com/avh4/elm- formato). Então configure seu editor para formatar os arquivos Elm automaticamente ou execute elm-format FILE_OR_DIR --yes manualmente.

Incorporando em HTML #

Existem três possibilidades para inserir o código Elm em uma página HTML existente.

Incorporar na tag do corpo #

Supondo que você compilou o exemplo Hello World no arquivo elm.js, você pode deixar Elm assumir a tag <body> assim:

<!DOCTYPE html>
<html>
    <body>
        <script src="elm.js"></script>
        <script>
          Elm.Main.fullscreen()
        </script>
    </body>
</html>

AVISO: às vezes, algumas extensões do Chrome interferem no <body>, o que pode fazer com que seu aplicativo interrompa a produção. Recomenda-se sempre incorporar em uma div específica. Mais informações aqui.

Incorporar em um Div (ou outro nó DOM) #

Alternativamente, fornecendo um elemento HTML concreto, o código Elm pode ser executado nesse elemento de página específico:

<!DOCTYPE html>
<html>
    <head>
        <title>Hello World</title>
    </head>
    <body>
        <div id='app'></div>
        <script src="elm.js"></script>
        <script>
            Elm.Main.embed(document.getElementById('app'))
        </script>
    </body>
</html>

Incorporar como um Web worker (sem UI) #

O código Elm também pode ser iniciado como um trabalhador e se comunicar através de portas:

<!DOCTYPE html>
<html>
    <head>
        <title>Hello Worker</title>
    </head>
    <body>
        <script src="elm.js"></script>
        <script>
            var app = Elm.Main.worker();
            app.ports.fromElmToJS.subscribe(function(world) {
                console.log(world)
            });
            app.ports.fromJSToElm.send('hello');
        </script>
    </body>
</html>

REPL #

Uma boa maneira de aprender sobre o Elm é tentar escrever algumas expressões no REPL (Read-Eval-Print Loop). Abra um console em sua pasta elm-app (que você criou na fase Initialize and build) e tente o seguinte:

$ elm repl
---- elm-repl 0.17.1 -----------------------------------------------------------
 :help for help, :exit to exit, more at <https://github.com/elm-lang/elm-repl>
--------------------------------------------------------------------------------
> 2 + 2
4 : number
> \x -> x
<function> : a -> a
> (\x -> x + x)
<function> : number -> number
> (\x -> x + x) 2
4 : number
>

elm-repl é na verdade uma ferramenta bastante poderosa. Digamos que você crie um arquivo Test.elm dentro de sua pasta elm-app com o seguinte código:

module Test exposing (..)


a = 1


b = "Hello"

Agora, você volta ao seu REPL (que permaneceu aberto) e digita:

import Test exposing (..)
> a
1 : number
> b
"Hello" : String
>

Ainda mais impressionante, se você adicionar uma nova definição ao seu arquivo Test.elm, como

s = """
Hello,
Goodbye.
"""

Salve seu arquivo, volte mais uma vez ao seu REPL, e sem importar Test novamente, a nova definição estará disponível imediatamente:

> s
"\nHello,\nGoodbye.\n" : String
>

É realmente conveniente quando você deseja escrever expressões que abrangem muitas linhas. Também é muito útil testar rapidamente as funções que você acabou de definir. Adicione o seguinte ao seu arquivo:

f x =
  x + x * x

Salve e volte para o REPL:

> f
<function> : number -> number
> f 2
6 : number
> f 4
20 : number
>

Cada vez que você modifica e salva um arquivo que importou e volta ao REPL e tenta fazer qualquer coisa, o arquivo completo é recompilado. Portanto, ele informará sobre qualquer erro em seu código. Adicione isso:

c = 2 ++ 2

Tente isso:

> 0
-- TYPE MISMATCH -------------------------------------------------- ././Test.elm

The left argument of (++) is causing a type mismatch.

22|     2 ++ 2
        ^
(++) is expecting the left argument to be a:

    appendable

But the left argument is:

    number

Hint: Only strings, text, and lists are appendable.


>

Para concluir esta introdução ao REPL, vamos acrescentar que o elm-repl também conhece os pacotes que você instalou com o elm package install. Por exemplo:

> import Html.App
> Html.App.beginnerProgram
<function>
    : { model : a, update : b -> a -> a, view : a -> Html.Html b }
      -> Platform.Program Basics.Never
>

Servidor de compilação local (Elm Reactor) #

Elm Reactor é a ferramenta essencial para prototipar sua aplicação.

Observe que você não poderá compilar Main.elm com Elm Reactor, se estiver usando Http.App.programWithFlags ou Ports

A execução do elm-reactor em um diretório de projetos iniciará um servidor web com um explorador de projetos, que permite compilar cada componente separado.

Todas as alterações feitas no seu código são atualizadas quando você recarrega a página.

$ elm-reactor                     # launch elm-reactor on localhost:8000
$ elm-reactor -a=0.0.0.0 -p=3000  # launch elm-reactor on 0.0.0.0:3000
