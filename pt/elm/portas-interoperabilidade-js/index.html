Sintaxe #

  • Elm (recebendo): port functionName : (valor -> msg) -> Sub msg
  • JS (enviando): app.ports.functionName.send(value)
  • Elm (enviando): port functionName : args -> Cmd msg
  • JS (recebendo): app.ports.functionName.subscribe(function(args) { … });

Consulte http://guide.elm-lang.org/interop/javascript.html do The Elm Guide para ajudar a entender esses exemplos.

Extrovertido #

As portas de saída são usadas como Comandos, que você retorna da sua função update.

Lado do olmo #

Defina a porta de saída:

port output : () -> Cmd msg

Neste exemplo, enviamos uma Tupla vazia, apenas para acionar uma assinatura no lado do JavaScript.

Para isso, temos que aplicar a função output com uma Tupla vazia como argumento, para obter um Comando para enviar os dados de saída do Elm.

update msg model =
    case msg of
        TriggerOutgoing data ->
            ( model, output () )

lado do JavaScript #

Inicialize o aplicativo:

var root = document.body;
var app = Elm.Main.embed(root);

Assine uma porta com um nome correspondente:

app.ports.output.subscribe(function () {
    alert('Outgoing message from Elm!');
});

Observação #

A partir de 0.17.0, a mensagem de saída imediata para JavaScript do seu estado inicial não terá efeito.

init : ( Model, Cmd Msg )
init =
    ( Model 0, output () ) -- Nothing will happen

Veja a solução alternativa no exemplo abaixo.

Entrada #

Os dados de entrada do JavaScript estão passando por Assinaturas.

Lado do olmo #

Primeiro, precisamos definir uma porta de entrada, usando a seguinte sintaxe:

port input : (Int -> msg) -> Sub msg

Podemos usar Sub.batch se tivermos várias assinaturas, este exemplo conterá apenas uma assinatura para porta de entrada

subscriptions : Model -> Sub Msg
subscriptions model =
    input Get

Então você tem que passar as subscriptions para o seu Html.program:

main =
    Html.program
        { init = init
        , view = view
        , update = update
        , subscriptions = subscriptions
        }

lado do JavaScript #

Inicialize o aplicativo:

var root = document.body;
var app = Elm.Main.embed(root);

Envie a mensagem para Elm:

var counter = 0;
        
document.body.addEventListener('click', function () {
    counter++;
    app.ports.input.send(counter);
});

Observação #

Observe que a partir de 0.17.0 o app.ports.input.send(counter); imediato após a inicialização do aplicativo não terá efeito!

Passe todos os dados necessários para a inicialização como sinalizadores usando Html.programWithFlags

Iniciar #

index.html

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>Trying out ports</title>
  </head>
  <body>
    <div id="app"></div>
    <script src="elm.js"></script>
    <script>
    
      var node = document.getElementById('app');
      var app = Elm.Main.embed(node);
      
      // subscribe to messages from Elm
      app.ports.toJs.subscribe(function(messageFromElm) {
        alert(messageFromElm);
        // we could send something back by
        // app.ports.fromJs.send('Hey, got your message! Sincerely, JS');
      });
      
      // wait three seconds and then send a message from JS to Elm
      setTimeout(function () {
        app.ports.fromJs.send('Hello from JS');
      }, 3000);
      
    </script>
  </body>
</html>

Main.elm

port module Main exposing (..)

import Html

port toJs : String -> Cmd msg
port fromJs : (String -> msg) -> Sub msg

main =
   Html.program
        { init = (Nothing, Cmd.none) -- our model will be the latest message from JS (or Nothing for 'no message yet')
        , update = update
        , view = view
        , subscriptions = subscriptions
        }

type Msg
    = GotMessageFromJs String

update msg model =
    case msg of
        GotMessageFromJs message ->
            (Just message, toJs "Hello from Elm")

view model =
    case model of
        Nothing ->
            Html.text "No message from JS yet :("
        Just message ->
            Html.text ("Last message from JS: " ++ message)

subscriptions model =
    fromJs GotMessageFromJs

Instale o pacote elm-lang/html se você ainda não o fez por elm-package install elm-lang/html --yes.

Compile este código usando elm-make Main.elm --yes --output elm.js para que o arquivo HTML o encontre.

Se tudo correr bem, você poderá abrir o arquivo index.html com o texto “Sem mensagem” exibido. Após três segundos o JS envia uma mensagem, Elm a recebe, muda seu modelo, envia uma resposta, JS a recebe e abre um alerta.

Mensagem de saída imediata na inicialização em 0.17.0 #

Para enviar uma mensagem imediata com dados para JavaScript, você precisa acionar uma ação do seu init.

init : ( Model, Cmd Msg )
init =
    ( Model 0, send SendOutgoing )


send : msg -> Cmd msg
send msg =
    Task.perform identity identity (Task.succeed msg)

Visão geral #

Um módulo que está usando Ports deve ter a palavra-chave port em sua definição de módulo.

port module Main exposing (..)

É impossível usar ports com Html.App.beginnerProgram, pois não permite o uso de Subscriptions ou Commands.

As portas são integradas ao loop de atualização de Html.App.program ou Html.App.programWithFlags.

Observação #

program e programWithFlags no elm 0.18 estão dentro do pacote Html em vez de Html.App.
