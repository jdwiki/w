Sintaxe #

  • – definir uma função sem argumentos parece o mesmo que simplesmente definir um valor idioma = “Olmo”
  • – chamando uma função sem argumentos informando seu nome Língua
  • – os parâmetros são separados por espaços e seguem o nome da função adicione x y = x + y
  • – chama uma função da mesma forma adicionar 5 2
  • – aplica parcialmente uma função fornecendo apenas alguns de seus parâmetros incrementar = adicionar 1
  • – use o operador |> para passar a expressão da esquerda para a função da direita dez = 9 |> incremento
  • – o <| operador passa a expressão à direita para a função à esquerda incrementar <| adicionar 5 4
  • – encadeia/compõe duas funções junto com o operador » para trásYell = String.reverse » String.toUpper
  • – o « funciona da mesma forma no sentido inverso para trásYell = String.toUpper « String.reverse
  • – uma função com um nome não alfanumérico entre parênteses cria um novo operador (#) x y = x * y dez = 5 # 2
  • – qualquer operador infixo se torna uma função normal quando você o coloca entre parênteses dez = (+) 5 5
  • – anotações de tipo opcionais aparecem acima das declarações de função isTen : Int -> Bool isTen n = se n == 10 então Verdadeiro senão Falso

Aplicação Parcial #

Aplicação parcial significa chamar uma função com menos argumentos do que tem e salvar o resultado como outra função (que aguarda o restante dos argumentos).

multiplyBy: Int -> Int -> Int    
multiplyBy x y =
    x * y


multiplyByTwo : Int -> Int  -- one Int has disappeared! we now know what x is.
multiplyByTwo =
    multiplyBy 2


> multiplyByTwo 2
4 : Int

> multiplyByTwo 4
8 : Int

Como uma nota acadêmica, Elm pode fazer isso por causa de currying nos bastidores.

Variáveis ​​locais #

É possível definir variáveis ​​locais dentro de uma função para

  • reduzir a repetição de código
  • dar nome a subexpressões
  • reduz a quantidade de argumentos passados.

A construção para isso é let ... in ....

bigNumbers =
    let
        allNumbers =
            [1..100]

        isBig number =
            number > 95
    in
        List.filter isBig allNumbers

> bigNumbers
[96,97,98,99,100] : List number

> allNumbers
-- error, doesn't know what allNumbers is!

A ordem das definições na primeira parte de let não importa!

outOfOrder =
    let
        x =
            y + 1  -- the compiler can handle this

        y =
            100
    in
        x + y

> outOfOrder
201 : number

Visão geral #

A sintaxe do aplicativo de função no Elm não usa parênteses ou vírgulas e, em vez disso, é sensível a espaços em branco.

Para definir uma função, especifique seu nome multiplyByTwo e argumentos x, qualquer operação após o sinal de igual = é o que retorna de uma função.

multiplyByTwo x =
    x * 2

Para chamar uma função, especifique seu nome e argumentos:

multiplyByTwo 2  -- 4

Note que sintaxe como multiplyByTwo(2) não é necessária (mesmo que o compilador não reclame). Os parênteses servem apenas para resolver a precedência:

> multiplyByTwo multiplyByTwo 2
-- error, thinks it's getting two arguments, but it only needs one

> multiplyByTwo (multiplyByTwo 2)
4 : number

> multiplyByTwo 2 + 2
6 : number
-- same as (multiplyByTwo 2) + 2

> multiplyByTwo (2 + 2)
8 : number

Expressões lambda #

Elm tem uma sintaxe especial para expressões lambda ou funções anônimas:

\arguments -> returnedValue

Por exemplo, como visto em List.filter:

> List.filter (\num -> num > 1) [1,2,3]
[2,3] : List number

Mais detalhadamente, uma barra invertida, \, é usada para marcar o início da expressão lambda, e a seta, ->, é usada para delimitar argumentos do corpo da função. Se houver mais argumentos, eles serão separados por um espaço:

normalFunction x y = x + y
-- is equivalent to
lambdaFunction = \x y -> x + y

> normalFunction 1 2
3 : number

> lambdaFunction 1 2
3 : number

Avaliação rigorosa e demorada #

Em elm, o valor de uma função é calculado quando o último argumento é aplicado. No exemplo abaixo, o diagnóstico de log será impresso quando f for invocado com 3 argumentos ou uma forma curry de f for aplicada com o último argumento.

import String
import Debug exposing (log)

f a b c = String.join "," (log "Diagnostic" [a,b,c]) -- <function> : String -> String -> String -> String

f2 = f "a1" "b2" -- <function> : String -> String

f "A" "B" "C"
-- Diagnostic: ["A","B","C"]
"A,B,C" : String

f2 "c3"
-- Diagnostic: ["a1","b2","c3"]
"a1,b2,c3" : String

Às vezes você vai querer evitar que uma função seja aplicada imediatamente. Um uso típico em elm é Lazy.lazy que fornece uma abstração para controlar quando as funções são aplicados.

lazy : (() -> a) -> Lazy a

Computações lentas usam uma função de um argumento do tipo () ou Unit. O tipo de unidade é convencionalmente o tipo de um argumento de espaço reservado. Em uma lista de argumentos, o argumento correspondente é especificado como _, indicando que o valor não é usado. O valor unitário em elm é especificado pelo símbolo especial () que pode representar conceitualmente uma tupla vazia ou um buraco. Assemelha-se à lista de argumentos vazia em C, Javascript e outras linguagens que usam parênteses para chamadas de função, mas é um valor comum.

Em nosso exemplo, f pode ser protegido de ser avaliado imediatamente com um lambda:

doit f = f () -- <function> : (() -> a) -> a
whatToDo = \_ -> f "a" "b" "c" -- <function> : a -> String
-- f is not evaluated yet

doit whatToDo
-- Diagnostic: ["a","b","c"]
"a,b,c" : String

A avaliação da função é atrasada sempre que uma função é parcialmente aplicada.

defer a f = \_ -> f a -- <function> : a -> (a -> b) -> c -> b

delayF = f "a" "b" |> defer "c" -- <function> : a -> String

doit delayF
-- Diagnostic: ["a","b","c"]
"a,b,c" : String

Elm tem uma função always, que não pode ser usada para atrasar a avaliação. Como o elm avalia todos os argumentos da função independentemente de se e quando o resultado do aplicativo de função é usado, agrupar um aplicativo de função em always não causará atraso, porque f é totalmente aplicado como um parâmetro para always.

alwaysF = always (f "a" "b" "c") -- <function> : a -> String
-- Diagnostic: ["a","b","c"] -- Evaluation wasn't delayed.

Operadores infixos e notação infixa #

Elm permite a definição de operadores infixos personalizados.

Os operadores infixos são definidos usando parênteses em torno do nome de uma função.

Considere este exemplo de operador infixo para construção de Tuplas 1 => True -- (1, True):

(=>) : a -> b -> ( a, b )
(=>) a b =
    ( a, b )

A maioria das funções em Elm são definidas em notação de prefixo.

Aplique qualquer função usando notação infixa especificando o primeiro argumento antes do nome da função entre o caractere de acento grave:

import List exposing (append)


append [1,1,2] [3,5,8]   -- [1,1,2,3,5,8]
[1,1,2] `append` [3,5,8] -- [1,1,2,3,5,8]
