Dicionários #

Os dicionários são implementados em uma biblioteca central Dict.

Um dicionário mapeando chaves exclusivas para valores. As chaves podem ser qualquer tipo comparável. Isso inclui Int, Float, Time, Char, String e tuplas ou listas de tipos comparáveis.

As operações de inserção, remoção e consulta levam tempo O(log n).

Diferentemente das Tuplas e Registros, os Dicionários podem alterar sua estrutura, ou seja, é possível adicionar e remover chaves.

É possível atualizar um valor por uma chave.

Também é possível acessar ou atualizar um valor usando chaves dinâmicas.

Acessando valores #

Você pode recuperar um valor de um dicionário usando uma função Dict.get.

Definição de tipo de Dict.get:

get : comparable -> Dict comparable v -> Maybe v

Sempre retornará Talvez v, pois é possível tentar obter um valor por uma chave inexistente.

import Dict

initialUsers =
      Dict.fromList [ (1, "John"), (2, "Brad") ]

getUserName id =
  initialUsers
  |> Dict.get id
  |> Maybe.withDefault "Anonymous"
  
getUserName 2 -- "Brad"
getUserName 0 -- "Anonymous"

Atualizando valores #

A operação de atualização em um Dicionário é realizada usando Maybe.map, pois a chave solicitada pode estar ausente.

import Dict

initialUsers =
  Dict.fromList [ (1, "John"), (2, "Brad") ]

updatedUsers =
  Dict.update 1 (Maybe.map (\name -> name ++ " Johnson")) initialUsers

Maybe.withDefault "No user" (Dict.get 1 updatedUsers) -- "John Johnson"

Registros #

Record é um conjunto de pares chave-valor.

greeter =
    { isMorning: True
    , greeting: "Good morning!"
    }

É impossível acessar um valor por uma chave inexistente.

É impossível modificar dinamicamente a estrutura do Record.

Os registros só permitem atualizar valores por chaves constantes.

Acessando valores #

Os valores não podem ser acessados ​​usando uma chave dinâmica para evitar possíveis erros em tempo de execução:

isMorningKeyName =
    "isMorning "


greeter[isMorningKeyName] -- Compiler error
greeter.isMorning -- True

A sintaxe alternativa para acessar o valor permite extrair o valor, enquanto canaliza o Registro:

greeter
    |> .greeting
    |> (++) " Have a nice day!" -- "Good morning! Have a nice day!"

Tipos de extensão #

Às vezes, você deseja que a assinatura de um parâmetro restrinja os tipos de registro que você passa para as funções. A extensão de tipos de registro torna desnecessária a ideia de supertipos. O exemplo a seguir mostra como esse conceito pode ser implementado:

type alias Person =
    { name : String
    }


type alias Animal =
    { name : String
    }


peter : Person
peter =
    { name = "Peter" }


dog : Animal
dog =
    { name = "Dog" }


getName : { a | name : String } -> String
getName livingThing =
    livingThing.name


bothNames : String
bothNames =
    getName peter ++ " " ++ getName dog

Poderíamos até levar a extensão de registros um passo adiante e fazer algo como:

type alias Named a = { a | name : String }
type alias Totalled a = { a | total : Int }


totallyNamed : Named ( Totalled { age : Int })
totallyNamed =
  { name = "Peter Pan"
  , total = 1337
  , age = 14
  }

Agora temos maneiras de passar esses tipos parciais em funções:

changeName : Named a -> String -> Named a
changeName a newName =
  { a | name = newName }
  
cptHook = changeName totallyNamed "Cpt. Hook" |> Debug.log "who?"

Atualizando valores #

Elm tem uma sintaxe especial para atualizações de registros:

model =
    { id: 1
    , score: 0
    , name: "John Doe"
    }


update model =
    { model
       | score = model.score + 100
       | name = "John Johnson"
    }

Tuplas #

Tuplas são listas ordenadas de valores de qualquer tipo.

(True, "Hello!", 42)

É impossível alterar a estrutura de uma Tupla ou atualizar o valor.

Tuplas em Elm são consideradas um tipo de dados primitivo, o que significa que você não precisa importar nenhum módulo para usar Tuplas.

Acessando valores #

O módulo Basics possui duas funções auxiliares para acessar valores de uma Tupla com comprimento de dois ( a, b ) sem usar correspondência de padrões:

fst (True, "Hello!") -- True
snd (True, "Hello!") -- "Hello!"

O acesso a valores de tuplas com maior comprimento é feito por correspondência de padrões.

Correspondência de padrões #

As tuplas são extremamente úteis em combinação com a correspondência de padrões:

toggleFlag: (Sting, Bool) -> (Sting, Bool)
toggleFlag (name, flag) =
    (name, not flag)

Observações sobre Tuplas #

As tuplas contêm menos de 7 valores do tipo de dados “comparável”
