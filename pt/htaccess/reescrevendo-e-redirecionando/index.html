Antes que os URLs possam ser reescritos, um módulo chamado mod_rewrite.c precisa ser habilitado. Normalmente, está desabilitado na configuração por padrão.

mod_rewrite pode ser habilitado executando o comando

$ sudo a2enmod mod_rewrite
$ sudo service apache2 restart

ou comentando as linhas

#LoadModule rewrite_module modules/mod_rewrite.so
#AddModule mod_rewrite.c

no arquivo httpd.conf.

Sinalizadores de reescrita populares #

F|proibido #

Semelhante a Deny, este sinalizador força o servidor a retornar imediatamente um código de status 403 Forbidden para o navegador ou cliente solicitante da solicitação.

Exemplo: Negar acesso a solicitações que terminam com exe:

RewriteRule .exe$ - [F]

G|foi #

Se um recurso solicitado estava disponível no passado, mas não está mais disponível, você pode usar esse sinalizador para forçar o servidor a retornar imediatamente um código de status 410 Gone para o navegador ou cliente solicitante da solicitação.

Exemplo: informar a um visitante que um produto antigo não existe mais:

RewriteRule ^old-product.html$ - [G]

L|último #

Na maioria dos contextos, exceto .htaccess, este sinalizador instrui mod_rewrite a parar de processar a condição/conjunto de regras atual, da mesma forma que last e break (Perl e C, respectivamente) fazem.

No entanto, no contexto .htaccess ou <Directory>, uma solicitação que foi reescrita usando uma RewriteRule com este sinalizador será passada de volta ao mecanismo de análise de URL para processamento adicional. Como tal, é possível que o URI reescrito seja tratado pelo mesmo contexto e talvez alterado ainda mais.

Uma recomendação geral é usar o sinalizador END não apenas para interromper o processamento da condição/conjunto de regras atual, mas também para evitar qualquer reescrita nestes contextos.

Nota: Os sinalizadores F e G, discutidos acima, ambos usam L implicitamente, então você não precisa especificá-los separadamente.

N|próximo #

Este sinalizador executará novamente o processo de reescrita desde o início, começando novamente com a primeira condição/regra definida. Desta vez, o URL a ser correspondido não é mais o URI original, mas sim o URI reescrito retornado pelo último conjunto de regras. Use este sinalizador para reiniciar o processo de regravação.

Um aviso: Use este sinalizador com cuidado, pois pode resultar em um loop infinito!

NC|nocase #

Isso instrui mod_rewrite a corresponder ao Pattern de uma RewriteRule sem diferenciar maiúsculas de minúsculas. Para esclarecer, MyIndex.html e myindex.html seriam considerados pelo módulo como a mesma coisa. Além disso, este sinalizador permite que você use a-z em vez de A-Za-z em uma expressão regular.

R|redirecionar #

Este sinalizador é usado para enviar uma resposta de redirecionamento HTTP para o navegador/cliente solicitante.

Por padrão, se nenhum código for fornecido, uma resposta de redirecionamento com o código de status 302 Found (semelhante a um redirecionamento temporário) será retornada. Se você deseja usar um redirecionamento mais permanente, deve usar o código de status 302 (301 Movido Permanentemente).

Geralmente, apenas códigos de status no intervalo 300-399 devem ser usados ​​com esse sinalizador. Se códigos de status fora desse intervalo forem usados ​​(o que é perfeitamente aceitável), a string de substituição será descartada e a reescrita será interrompida como se o sinalizador L fosse usado. Em alguns casos, essa é uma maneira prática de forçar respostas 404 Not Found, mesmo que a solicitação aponte para um recurso existente.

Exemplo: emita uma resposta de redirecionamento 302 Found:

RewriteRule ^bus$ /train [R,L]

Exemplo: emita uma resposta de redirecionamento 301 movido permanentemente:

RewriteRule ^speed-train$ /hyperloop [R=301,L]

Exemplo: Forçar um 404 não encontrado:

RewriteRule ^blip$ - [R=404,L]

Redirecionar com/sem parâmetros de consulta #

Redirecionar sem parâmetros de consulta:

RewriteRule ^route$ /new_route_without_query [L,R=301,QSD]

Redirecionar com parâmetros de consulta:

RewriteCond %{QUERY_STRING} ^$
RewriteRule ^/?route$ %{REQUEST_URI}?query=param1&query2=param2 [NC,L,R=301]

Redirecionamentos www e não www #

Redirecione qualquer domínio simples para www.[seu_domínio].tld:

# Start Apache Rewriting engine
RewriteEngine On
# Make sure you're not already using www subdomain
# and that the host string is not empty
RewriteCond %{HTTP_HOST} !^$
RewriteCond %{HTTP_HOST} !^www\.
# We check for http/https connection protocol
RewriteCond %{HTTPS}s ^on(s)|
# In case the previous conditions matches, redirect to www
RewriteRule ^(.*)$ http%1://www.%{HTTP_HOST}/$1 [R=301,L]

Redirecione www.[seu_domínio].tld para [seu_domínio].tld

# Start Apache Rewriting engine
RewriteEngine On
# We check if we're on the www subdomain
RewriteCond %{HTTP_HOST} ^www\.([^\.]+\.[^\.]+)$
# In case the previous condition matches, redirect to non-www
RewriteRule ^(.*)$ http://%1/$1 [R=301,L]

Redirecione qualquer nível de subdomínios aninhados para seu domínio principal:

# Start Apache Rewriting engine
RewriteEngine On
# We check if there's a subdomain
RewriteCond %{HTTP_HOST} \.([^.]+\.[^.]+)$
# redirect to the main domain name
RewriteRule ^ http://%1%{REQUEST_URI} [R=301,L]

URLs amigáveis ​​para SEO #

Os mecanismos de pesquisa não indexarão seus produtos se você tiver um URL como o seguinte:

http://www.yourdomain.com/product.php?id=123

O URL amigável para SEO se pareceria com http://www.seudominio.com/123/nome-do-produto/. O código a seguir ajuda a conseguir isso sem ter que alterar o código product.php.

RewriteEngine On
RewriteRule ^product/([0-9]+)/product-name-slug/?$ product.php?id=$1

Adicionando uma barra no final #

RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_URI} !(.*)/$
RewriteRule ^(.*)$ /$1/ [L,R=301]

O primeiro RewriteCond ajuda a excluir os arquivos. O segundo RewriteCond verifica se já existe uma barra final. Se for o caso, RewriteRule não é aplicado.

Se você tiver algum URL que não deva ser reescrito, você pode adicionar mais um RewriteCond.

RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_URI} !(.*)/$
RewriteCond %{REQUEST_URI} !url/to/not/rewrite
RewriteRule ^(.*)$ /$1/ [L,R=301]

Redirecionamentos http e https e configuração HSTS #

Redirecionamento genérico para https: #

# Enable Rewrite engine
RewriteEngine on

# Check if URL does not contain https
RewriteCond %{HTTPS} off [NC]
# If condition is true, redirect to https
RewriteRule (.*) https://%{SERVER_NAME}/$1 [R=301,L]

Redirecionamento genérico para http: #

# Enable Rewrite engine
RewriteEngine on

# Check if URL does contain https
RewriteCond %{HTTPS} on [NC]
# If condition is true, redirect to http
RewriteRule (.*) http://%{SERVER_NAME}/$1 [R=301,L]

Forçando conexão HTTPS (HSTS): #

<IfModule mod_headers.c>
    Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"
</IfModule>

onde, a opção includeSubDomains pode ser removida, se HSTS deve ser aplicado apenas ao domínio base, ou ao domínio com a configuração acima.
