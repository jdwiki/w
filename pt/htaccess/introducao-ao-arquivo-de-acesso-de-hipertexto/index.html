Configurando .htaccess #

Arquivos .htaccess (ou “arquivos de configuração distribuídos”) fornecem uma maneira para fazer alterações de configuração por diretório. Um arquivo, contendo uma ou mais diretivas de configuração, é colocado em um diretório de documentos específico, e as diretivas se aplicam a esse diretório e todos os seus subdiretórios.

Um arquivo .htaccess controla como o Apache interage com seu site. Ele é usado para alterar as solicitações e modificar o comportamento padrão sem a necessidade de alterar os arquivos de configuração do servidor núcleo.

Configurar o .htaccess é tão simples quanto abrir um bloco de notas e salvá-lo como .htaccess. Geralmente, este arquivo será colocado no diretório raiz dos arquivos do seu site, mas você pode usá-lo em vários diretórios diferentes. Isso é especialmente útil se você estiver procurando proteger diretórios específicos com senha.

Habilitando .htaccess #

Às vezes até mesmo um único erro em seu arquivo httpd.conf ou .htacces resultará em um colapso temporário do servidor e os usuários verão página 500 - Erro interno do servidor. Portanto, certifique-se de sempre fazer um backup de seus arquivos httpd.conf e .htaccess antes de fazer uma alteração.

<Directory "/var/www">
    AllowOverride All
</Directory>

Arquivos .htaccess são normalmente habilitados por padrão. Isso é controlado pela diretiva AllowOverride no arquivo httpd.conf. Esta diretiva só pode ser colocada dentro de uma seção <Directory>.

Além de All, existem vários outros valores que limitam a configuração de apenas determinados contextos. Alguns deles são:

  • Nenhum - Desabilite completamente .htaccess.

  • AuthConfig - Diretivas de autorização, como as que lidam com autenticação básica.

  • FileInfo - Diretivas que lidam com a configuração de cabeçalhos, documentos de erro, cookies, reescrita de URL e muito mais.

  • Indexes - Personalizações de listagem de diretório padrão.

  • Limite - Controle o acesso às páginas de várias maneiras.

  • Opções - Acesso semelhante a Índices, mas inclui ainda mais valores, como ExecCGI, FollowSymLinks, Include e muito mais.

    Only allow .htaccess files to override Authorization and Indexes #

    AllowOverride AuthConfig Indexes

Páginas de erro personalizadas #

.htaccess pode ser usado para definir páginas de erro personalizadas que correspondam ao tema do seu site, em vez de ver uma página de erro branca com tecno-babble preta quando os usuários terminam em uma página com um código de resposta do servidor de erro. A página de erro pode ser qualquer arquivo analisável do navegador, incluindo (mas não limitado a) .html, .php, .asp, .txt, .xml.

Exemplos para quase todos os códigos de resposta de erro comuns:

#Client Errors

ErrorDocument 400 /mycool400page.html    # Bad Request
ErrorDocument 401 /mycool401page.html    # Unauthorized
ErrorDocument 402 /mycool402page.html    # Payment Required
ErrorDocument 403 /mycool403page.html    # Forbidden
ErrorDocument 404 /mycool404page.html    # Page Not Found

#Server Errors

ErrorDocument 500 /mycool500page.html    # Internal Server Error
ErrorDocument 501 /mycool501page.html    # Not Implemented
ErrorDocument 502 /mycool502page.html    # Bad Gateway
ErrorDocument 503 /mycool503page.html    # Service Unavailable
ErrorDocument 504 /mycool504page.html    # Gateway Timeout
ErrorDocument 505 /mycool505page.html    # Internal Server Error

É sempre uma boa prática incluir Documentos de Erro para as respostas de erro mais comuns, 400, 403, 404 e 500, pois esses erros podem ocorrer em todos os navegadores.

o erro 500 é um dos erros mais notórios, pois ocorre se algo falhar ao carregar a página para enviar, mais comumente falhas de pré-processamento de html do servidor de coisas como PHP, ASP e outros pré-processadores de html. É uma boa prática durante o teste definir a página 500 para exibir o erro que ocorreu, em vez de uma página de erro 500 não específica.

Para habilitar a página de erro 500 para gravar um erro específico, veja um dos seguintes com base no pré-processador de html que você está usando: php asp

Configurando o fuso horário do servidor #

Existem muitos fusos horários ao redor do mundo, é importante certificar-se de que seu servidor esteja configurado para o correto. Isso é feito em .htaccess usando:

SetEnv TZ America/Indianapolis

Alguns exemplos de possíveis outros fusos horários:

America/Los_Angeles
America/Los_Angeles - Pacific Time 
Pacific/Honolulu - Hawaii

Apenas certifique-se de usar SetEnv na frente do fuso horário selecionado.
