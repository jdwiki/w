Capacidades de idioma #

O Jenkins Pipeline DSL é usado como exemplo para tal linguagem:

node {
  git 'https://github.com/joe_user/simple-maven-project-with-tests.git'
  def mvnHome = tool 'M3'
  sh "${mvnHome}/bin/mvn -B -Dmaven.test.failure.ignore verify"
  archiveArtifacts artifacts: '**/target/*.jar', fingerprint: true
  junit '**/target/surefire-reports/TEST-*.xml'
 }

O objetivo desta DSL é definir e executar trabalhos de construção do Jenkins (ou melhores pipelines) em uma linguagem mais natural.

Escrever uma linguagem específica de domínio no Groovy beneficia-se dos principais recursos do Groovy, como:

  • Opcional (por exemplo, omitir parênteses)
  • Sobrecarga do operador
  • Meta programação (por exemplo, resolvendo propriedades ou métodos ausentes)
  • Fechamentos e estratégias de delegação
  • Personalização do compilador
  • Suporte a scripts e capacidades de integração
