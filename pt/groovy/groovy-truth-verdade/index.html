O Groovy avalia condições em instruções if, while e for da mesma forma que Java faz para condições Java padrão : em Java você deve fornecer uma expressão booleana (uma expressão que é avaliada como booleana) e o resultado é o resultado da avaliação.

Em Groovy , o resultado é o mesmo que em Java para essas condições (nenhum exemplo fornecido, este é o Java padrão).

O outro mecanismo de avaliação de veracidade mostrado por exemplos pode ser resumido como:

  • números: um valor zero é avaliado como falso, diferente de zero como verdadeiro.
  • objetos: uma referência de objeto nula é avaliada como falsa, uma referência não nula como verdadeira.
  • Caractere: um caractere com valor zero é avaliado como falso, caso contrário verdadeiro.
  • String : uma string é avaliada como verdadeira se não for nula e não vazia, falsa se for nula ou vazia (aplica-se a GStrings e CharSequences também).
  • Coleções e mapas (incluindo subclasses List, Map, Set, HashSet …): também leva em consideração o tamanho, avalia como verdadeiro se a coleção não for nulo e não vazio, falso se nulo ou vazio.
  • Enumerations e Iterators são avaliados como true se não forem nulos e são mais alguns elementos (groovy avalia hasMoreElements ou hasNext no objeto), false se for nulo ou sem mais elementos.
  • Matcher : um matcher é avaliado como true se houver pelo menos uma correspondência, false se não for encontrada uma correspondência.
  • Fechamento: um fechamento avalia a avaliação do resultado retornado pelo fechamento.

O método asBoolean pode ser substituído em uma classe definida pelo usuário para fornecer avaliação booleana personalizada.

Avaliação booleana de números #

para números, um valor zero é avaliado como falso, diferente de zero como verdadeiro

    int i = 0
...
    if (i)
        print "some ${i}"
    else
        print "nothing"

vai imprimir “nada”

Avaliação booleana de strings #

uma string (incluindo GStrings) é avaliada como verdadeira se não for nula e não vazia, falsa se for nula ou vazia

def s = ''
...
if (s) 
    println 's is not empty'
else
    println 's is empty'

imprimirá: ’s está vazio'

Avaliação booleana de coleções e mapas #

Collections and Maps é avaliado como verdadeiro se não for nulo e não vazio e falso se for nulo ou vazio

/* an empty map example*/
def userInfo = [:]
if (!userInfo)
    userInfo << ['user': 'Groot', 'species' : 'unknown' ]

adicionará user: 'Groot' , species : 'unknown' como userInfo padrão, pois o mapa userInfo está vazio (observe que o mapa não é nulo aqui)

Com um objeto nulo, o código é ligeiramente diferente, não podemos invocar « em userInfo porque é nulo, temos que fazer uma atribuição (consulte também a avaliação booleana do objeto):

/* an example with a null object (def does not implies Map type) */
def userInfo = null
if (!userInfo)
    userInfo = ['user': 'Groot', 'species' : 'unknown' ]

E com um mapa nulo:

/* The same example with a null Map */
Map<String,String> userInfo = null
if (!userInfo)
    userInfo = ['user': 'Groot', 'species' : 'unknown' ]

Avaliação booleana do objeto #

uma referência de objeto nula é avaliada como falsa, uma referência não nula como verdadeira, mas para strings, coleções, iteradores e enumerações também leva em consideração o tamanho.

def m = null

if (!m)
    println "empty"
else
    println "${m}"

imprimirá “vazio”

def m = [:]

if (!m)
    println "empty"
else
    println "${m}"

O mapa não é nulo, mas vazio, este código imprimirá “vazio”

Depois de fazer

m << ['user' : 'Groot' ]

ele imprimirá o mapa:

[user:Groot]

Sobrescrevendo a avaliação booleana em uma classe definida pelo usuário #

Algumas vezes pode ser útil ter uma definição asBooleana específica em seu próprio programa para algum tipo de objeto.

/** an oversimplified robot controller */
class RunController {
    
    def complexCondition
    int position = 0
    
    def asBoolean() {
        return complexCondition(this);
    }
    def advanceTo(step) {
        position += step
    }
}
def runController = new RunController(complexCondition : { c -> c.position < 10 } )

assert runController
runController.advanceTo(5)
assert runController
runController.advanceTo(5)
// The limit has been reached : the controller evaluates to false
assert !runController

Este código mostra um controlador de robô simplificado que verifica se a posição do robô não excede 10 (com um fechamento para avaliação de condição)

Avaliação do personagem #

um caractere é avaliado como verdadeiro se seu valor não for zero, falso se for zero

assert ! new Character((char)0)
assert ! new Character('\u0000Hello Zero Char'.charAt(0))
assert  new Character('Hello'.charAt(0))

Avaliação de correspondência #

um Matcher é avaliado como true se puder encontrar pelo menos uma correspondência, false se nenhuma correspondência for encontrada

// a match is found => true
assert 'foo' =~ /[a-z]/
// the regexp does not match fully => no match => false
assert !( 'foo' ==~ /[a-z]/ )
// a match is found => true
assert 'foo' =~ /o/
// no match => false
assert !( 'foo' =~ /[A-Z]/ )

Avaliação de encerramento #

A avaliação de um fechamento é a avaliação do resultado do fechamento.

Todas as regras se aplicam: se o encerramento retornar um número nulo, zero ou String, Collection, Map ou Array vazio, ele será avaliado como false caso contrário para verdadeiro.

// Closure return non zero number => true
assert { 42 }()
// closure returns 0 => false
assert ! ( { 0 }())
// closure returns null => false
assert !( { }())
