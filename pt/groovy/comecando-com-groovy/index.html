Instalação ou Configuração #

Existem duas maneiras comuns de instalar o Groovy.

Download

O binário do Groovy pode ser baixado na página download do site do Groovy. Você pode descompactar o arquivo e adicionar o caminho para %GROOVY_HOME%/bin/groovy.bat à variável de ambiente do sistema PATH, onde %GROOVY_HOME% é o diretório onde o Groovy é descompactado.

SDKMAN

A outra opção é usar SDKMAN. Essa opção cresceu rapidamente em popularidade e torna o gerenciamento de várias versões do Groovy muito simples. Ele também suporta outras aplicações na ecosfera “GR8”. Esta opção funciona muito bem de forma nativa em Linux e Mac, mas requer Cygwin no Windows.

Seguindo as instruções na página de download do Groovy, você pode seguir as etapas a seguir para instalar o SDKMAN.

$ curl -s get.sdkman.io | bash

Depois que o SDKMAN estiver instalado, você terá acesso ao comando sdk. Com este comando você pode fazer muitas coisas úteis.

Instale o Groovy

$ sdk instalar groovy

Isso instalará a versão mais recente do Groovy.

Lista as versões do Groovy

$ sdk ls groovy

Isso permite que você execute um comando ls no estilo Linux no software Groovy, listando todas as opções disponíveis. Há um * ao lado de cada versão instalada e um > para indicar suas versões atuais.

Alternar versões do Groovy

$ SDK usa groovy 2.4.7

Isso mudará a versão atual do Groovy para 2.4.7. Se você tiver outras versões instaladas, poderá alternar para qualquer uma delas.

Você pode listar a versão atual do groovy com o comando groovy -version.

posh-gvm

O nome inicial do SDKMAN era GVM e posh-gvm é uma porta do GVM para o Windows Powershell. Portanto, se você desenvolve em uma máquina Windows e não deseja usar o SDKMAN no Cygwin, o posh-gvm é para você. Funciona da mesma forma que SDKMAN, mas em vez de sdk, o comando é gmv. Então

PS C:\Users\You> gmv install groovy

instalará o groovy através do posh-gvm em sua máquina Windows.

O exemplo a seguir ilustra o Hello World mais simples em groovy usando script, coloque o seguinte trecho de código em um arquivo, digamos helloWorld.groovy

Resultado: Olá Mundo!

Usando Groovy em um projeto Java #

Groovy tem acesso a todas as classes Java, na verdade as classes Groovy SÃO classes Java e podem ser executadas diretamente pela JVM. Se você estiver trabalhando em um projeto Java, usar Groovy como uma linguagem de script simples para interagir com seu código Java é um acéfalo.

Para tornar as coisas ainda melhores, quase qualquer classe Java pode ser renomeada para .groovy e compilada e executada e funcionará exatamente como funcionou, groovy está perto de ser um superconjunto de Java, este é um objetivo declarado do groovy.

Groovy tem um REPL. O groovysh vem com o Groovy e pode ser usado para instanciar e testar rapidamente uma classe Java se seu caminho de classe estiver configurado corretamente. Por exemplo, se seu classpath apontasse para o diretório “classes/bin” do eclipse, você poderia salvar seu arquivo no eclipse, pular para groovysh e instanciar a classe para testá-lo.

As razões para usar o Groovy para fazer isso em vez de apenas Java são: O carregador de classes é ÓTIMO para pegar novas classes à medida que elas são compiladas. Você geralmente não precisa sair/reiniciar o groovysh enquanto desenvolve.

A sintaxe é TERSE. Isso não é ótimo para código sustentável, mas para scripts e testes pode cortar seu código significativamente. Uma das grandes coisas que ele faz é eliminar as exceções verificadas (ou, mais precisamente, transformar todas as exceções verificadas em exceções não verificadas). Isso transforma o código assim (Imprima olá após um segundo):

class JavaClass {
    public static void main(String[] args) {
        try {
            Thread.sleep(1000);
        } catch(InterruptedException e) {
            // You shouldn't leave an empty catch block, but who cares if this was interrupted???
        }
        System.out.println("Hello!");
    }
}

no Groovy:

Thread.sleep(1000)
print "Hello!"

O Groovy também possui uma sintaxe de inicialização muito rígida. Isso permite que você especifique os dados exatamente como você gosta, sem pensar nisso:

Em Java para inicializar um mapa você provavelmente deveria fazer algo assim:

String[] init = { "1:Bill", "2:Doug", "3:Bev" };
// Note the rest of this can be put in a function and reused or maybe found in a library, but I always seem to have to write this function!
Map m = new HashMap<Integer, String>();
for(String pair : int) {
    String[] split = pair.split(":");
    m.put(new Integer(split[0]), split[1])
}

Isso não é ruim, mas é outra coisa para manter. No groovy você usaria apenas:

Map map = { 1 : "Bill", 2 : "Doug", 3 : "Bev" }

E você está feito. A sintaxe da lista é igualmente fácil.

A outra grande vantagem é a sintaxe de fechamento do groovy. É incrivelmente conciso e divertido, um pouco mais difícil de manter, mas para scripts isso não é uma prioridade. Como exemplo, aqui está um código groovy para encontrar todos os arquivos .txt que contêm a palavra Hello no diretório atual:

println new File('.').files.findAll{ it.name.endsWith('.txt') && it.text.contains('Hello') }.collect{ it.name }

Este exemplo usa alguns truques “Groovy”:

  • .files refere-se ao método getFiles() - groovy pode alternar entre getter/setter e sintaxe de propriedade à vontade

  • it. refere-se ao elemento atual de uma iteração. { it } é um atalho para { it -> it }, ex. :

    [1, 2, 3].collect{ it ^ 2 } == [1, 4, 9]

  • it.text (onde it é um arquivo) usa um método que o groovy adiciona ao File para recuperar todo o texto do arquivo. Isso é incrivelmente útil em scripts.

Olá mundo Shebang (linux) #

Dado um arquivo hello.groovy com conteúdo:

#!/usr/bin/env groovy
println "Hello world"

Pode ser executado a partir da linha de comando se tiver permissão de execução como

$ ./hello.groovy

Usando inject() na lista para criar uma string CSV #

No Groovy, o método inject() é um dos métodos cumulativos que nos permite adicionar (ou injetar) novas funcionalidades em qualquer objeto que implemente o método inject(). No caso de uma coleção, podemos aplicar um encerramento a uma coleção de objetos de maneira uniforme e agrupar os resultados em um único valor. O primeiro parâmetro para o método inject() é o valor inicial da acumulação e o segundo parâmetro é o fechamento.

Neste exemplo, tomaremos uma Lista de Strings como parâmetro e exibiremos os valores dessas strings delimitadas por vírgulas. Usei essa funcionalidade para anexar uma lista de valores a uma string de consulta REST e, se você modificá-la um pouco, usei-a para incluir valores em uma instrução SQL como parte de uma cláusula IN. Aqui está o código para fazer isso:

public String convertToCSV( List<String> list ) {
    if (list == null) {
        return ""
    }
    return list.inject( '' ) { result, item ->
        result + ( result && item ? ',' : '' ) + ( item ? "${item.trim()}" : '' )
    }
}

assert convertToCSV( null ) == ""
assert convertToCSV( ["aaa", "bbb  ", null, "  ccc  "] ) == "aaa,bbb,ccc"

Neste exemplo, o primeiro parâmetro para o método inject() é uma string de comprimento zero, o que significa que ao processar o primeiro elemento da lista, result também é uma string de comprimento zero. Isso resolve para false na primeira avaliação ternária e é por isso que não recebemos uma vírgula no início da string. A cada iteração consecutiva através dos elementos da lista, result torna-se a concatenação de si mesmo, uma vírgula e depois o próximo item até chegarmos ao último item da lista.

A vantagem dessa abordagem é que você não precisa de uma variável fora de uma construção de loop para manter o resultado String concatenado. A implicação é que isso pode levar a efeitos colaterais em seu código. Com a abordagem inject(), esse comportamento é injetado e a coleção agrupa o resultado das chamadas para o encerramento para você. A desvantagem dessa abordagem pode ser a legibilidade. Mas com alguma experiência, fica mais fácil de ler e entender, e espero que este exemplo ajude você a atingir esse objetivo.
