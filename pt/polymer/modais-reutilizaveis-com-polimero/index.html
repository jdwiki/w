Modais #

Então você quer adicionar material design ao seu portfólio de negócios ou carreira.hein? Você simplesmente não resiste em usar um Modal? que aparece com um clique nos cartões nítidos que você pretende criar, para cada um de seus projetos/produtos!

Digamos que, para cada projeto que você fez, ou para cada produto que você projetou, você precisa de um cartão de material. Cada cartão deve ter

  • Um ícone, que exibe uma caixa de informações, que lista todos os recursos do
  • seu produto ou projeto, em detalhes. Um ícone “compartilhar isso”, que compartilha
  • seu produto ou projeto nas mídias sociais Um ícone “fork this”, que abre a página do github para seu produto/projeto

Com html nativo, e supondo que você use bootstrap ou qualquer outro layout de caixa flexível, você precisaria

  • Escreva um div wrapper para cada novo projeto com uma linha de classe
  • Enrole sua imagem de herói para cada projeto em uma coluna de 12 cols de largura
  • Escrever outra linha
  • Enrole cada link (info | share | fork) em uma coluna de 4 cols de largura própria
  • Injete essas duas linhas em um recipiente

Para iniciantes. Depois de ter feito o acima para todos os seus projetos, você precisa trabalhar na criação do pop-up necessário para cada projeto.

  • Você baixa toneladas de javascript e css (personalizando o bootstrap para funcionalidade modal)
  • Você escreve um Modal com um id único, para diferenciar as informações de cada projeto (modal-P1 para o Projeto 1, modal-P2 para o Projeto 2 e assim por diante)
  • Você então compõe o html necessário, para exibir as informações de cada projeto, em seu Modal correspondente.

Barra de ferramentas reutilizável com Modal | Compartilhar | Ícones de garfo - Codificando o elemento #

Até agora, definimos como é fácil escrever um elemento personalizado que esconda o html confuso por trás dele e dê ao usuário uma marcação fácil de escrever e breve.

Hora de codificá-lo!

Nosso elemento personalizado, para exibir a barra abaixo da imagem do herói, deve

  • Aceitar um Link a ser compartilhado
  • Aceitar um Link para o Repo a ser bifurcado
  • Aceite um modalId para diferenciar entre outros Modals.
  • Aceitar um Título para o Modal

Registre o elemento personalizado da barra de ferramentas com Modal | Compartilhar | Ícones de garfo

Vamos registrar nosso elemento customizado, que aceita a lista de propriedades acima.

<script>
Polymer({
    is:"tool-bar",
    properties:{
        link2Share:{
            type:String,
            value:""
        },
        link2Fork:{
            type:String,
            value:""
        },
        modalId:{
            type:String,
            value:""
        },
       title:{
            type:String,
            value:"myModal"
        }
    }
});
</script>

Muito agradável! Em seguida, adicione o Html que precisamos.

Nota: Uso e gramática do Iron Icon #

Se precisarmos usar o ícone de informações para nosso Popup Modal, escrevemos a marcação assim:

<iron-icon icon="icons:info"></iron-icon>

Se nós deseja o ícone de compartilhamento de rede social e, em seguida, o atributo de ícone acima, torna-se

ícone=”social:compartilhar”

Então o significado gramatical, eu preciso do iconset de “SOCIAL” e eu quero o ícone SHARE desse conjunto.

Você pode, alternativamente, usar font-awesome , nesse caso, você pode usar,

escolha o que você considera mais adequado.

Uma vez que as inclusões são feitas, escrevemos o template para nosso elemento customizado. Usaremos binders unidirecionais para dados na marcação do nosso elemento personalizado. Ligamos o que for passado como atributos pelo usuário, às suas propriedades correspondentes do nosso elemento, enquanto embarcamos na chamada de registro do nosso elemento.

Portanto, o blueprint html para nosso elemento personalizado é:

<dom-module id="tool-bar">
 
    <template id="tool-box">
 
        <ul class="flex-wrap toolbar">
            <li>
                <iron-icon icon="icons:info" id="anchor-for-[[modalId]]" onclick="clickHandler(event)"></iron-icon>
            </li>
            <li>
                <a href="https://plus.google.com/share?url=[[link2Share]]" target="_blank" ><iron-icon icon="social:share"></iron-icon></a>
            </li>
            <li>
                <a href="[[link2Fork]]" target="_blank"><i class="fa fa-2x fa-code-fork" aria-hidden=true></i></a> 
            </li>
        </ul>
 
    </template>
 
</dom-module>

Legal! Portanto, a barra de ferramentas tem seu modelo ul , que lista os três links

  • ícone de informações - Popup Modal
  • compartilhe este ícone - compartilhe no google
  • Bifurcar este ícone – Bifurcar o repositório

Além disso, vinculamos os atributos que o usuário escreve às respectivas propriedades.

Observação: Tornar o ID modal exclusivo #

Abordamos a exclusividade do ícone de informações, especificando o id atributo assim

id="anchor-for-[[modalId]]" onclick="clickHandler(event)"></iron-icon>

Assim, cada projeto terá um id de âncora único.

ou seja

Se o usuário passar “project-cats” como modal-id, na chamada ao nosso elemento personalizado, o ID da âncora seria “gatos-âncora-para-projeto”

Se o usuário adicionar outro projeto e passar “project-puppy” como o atributo modal-id, o id âncora seria âncora-para-projeto-filhote

e assim por diante.

Mas como podemos garantir que cada ícone de informação seja mapeado para seu próprio Modal?

Ao “ícone de ferro” para info, deve ser atribuído um Atributo “data-dialog”, que também precisa ser exclusivo.

O atributo data-dialog é semelhante ao atributo data-target em o modal bootstrap. Ele precisa ser único para cada novo projeto.

Deixamos isso para o usuário, para manter a exclusividade. Assim, o usuário não pode têm o mesmo atributo modal-id para diferentes chamadas <tool-bar>.

Então, como a nota acima afirma, precisamos mapear o ícone de informações de cada projeto para uma caixa de diálogo de dados.

Vamos adicionar essa funcionalidade.

Usamos o método ready. É muito parecido com o do Jquery

$.pronto();

.É invocado quando o elemento personalizado termina de carregar.

Adicione esta função, após o objeto de propriedades, na chamada do Polymer para registro.

ready: function() {
    document.querySelector("#anchor-for-" + this.modalId).setAttribute('data-dialog', this.modalId);
}

Muito legal! Então agora temos

  • Um elemento personalizado registrado como um elemento Polymer
  • Modal | Compartilhar | Ícones de bifurcação adicionados ao elemento personalizado

Um Modal é anexado ao info iron-icon, com o id “anchor-for-[Whatever is pass as modal-id]”. Isso é cortesia do método pronto do nosso elemento.

Então, até agora, escrevendo

<tool-bar
link2-share="http://www.myportfolio.com/project-neighbourhood-map"
link2-fork="https://github.com/me/project-neighbourhood-map"
modal-id="project-neighbourhood-map">
</tool-bar>

Produz isso:

Mas quando o usuário clica nesse ícone de informações, algo como Isso deve acontecer

e nosso objetivo era envolver esta descrição completa do projeto, dentro da tag <tool-bar> na página de usuários.

Lembra do Shadow DOM? Precisamos ter um container em nosso elemento customizado, que forneça o shadow DOM para o conteúdo do nosso Modal.

Para o container que hospeda a descrição do Projeto, utilizamos o elemento Polymer “paper-dialog”. Paper-dialog, é atribuído como id, o que for passado como modal-id na declaração na página do usuário.

Então, adicione esta marcação, após o ul no DOM do elemento personalizado da barra de ferramentas

<paper-dialog id="[[modalId]]" modal>
    <div class="text-right modal-close">
        <iron-icon icon="icons:power-settings-new" dialog-dismiss></iron-icon>
    </div>
    <h2 class="text-center text-capitalize">[[title]]</h2>
    <paper-dialog-scrollable>
        <div class="container-fluid">
            <div class="row flex-wrap info">
                <content></content>
            </div>
        </div>
    </paper-dialog-scrollable>
</paper-dialog>

Nota: Paper-dialog-scrollable #

Usamos Paper-dialog-scrollable, para envolver a descrição do nosso projeto, então que, quaisquer descrições longas, não transbordem.

A última parte, é adicionar a funcionalidade onclick para quando o modal deve abrir, ao clicar no ícone info.

Quando um clique acontece, como é normal, um evento é acionado, que passamos para nossa função clickHandler assim:

<iron-icon icon="icons:info" id="anchor-for-[[modalId]]" onclick="clickHandler(event)"></iron-icon>

Então, uma vez que a função clickHandler recebe o evento passado para ela,

  • Ele precisa coletar, qual elemento acionou o clique,
  • Ele precisa reunir, o atributo data-dialog desse elemento, para verificar qual modal abrir (Lembra que cada projeto tem seu próprio modal id?)
  • Em seguida, ele precisa chamar o método aberto, exatamente para esse modal

Então, a função fica assim:

Adicione isto, antes da chamada de registro do elemento. Isso não faz parte do elemento, mas tem o escopo dentro do elemento.

function clickHandler(e) {
    var button = e.target;
    while (!button.hasAttribute('data-dialog') && button !== document.body) {
        button = button.parentElement;
    }
 
    if (!button.hasAttribute('data-dialog')) {
        return;
    }
 
    var id = button.getAttribute('data-dialog');
    var dialog = document.getElementById(id);
    if (dialog) {
        dialog.open();
    }
}

Então, com isso, completamos a codificação do elemento personalizado, <tool-bar>.
