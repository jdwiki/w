A - Introdução #

Em poucas palavras, nosso elemento personalizado deve

  • Possui uma funcionalidade de busca integrada, que busca e marca lugares no mapa.
  • Accept Um atributo chamado location-array , que é uma lista de lugares
  • Ter um Listener , para ouvir um evento chamado “google-map-ready“. Este evento é acionado, quando o elemento é carregado.
  • Este ouvinte deve percorrer a matriz de localização e atribuir o próximo elemento em sua matriz de localização como a “consulta de pesquisa” atual
  • Tenha um método chamado cache
  • Este método armazena em cache a latitude e longitude de cada local na matriz de localização, assim que a pesquisa retornar um resultado
  • Tenha um modelo de repetição de Dom, que percorre os resultados em cache e solta um marcador em cada local

Portanto, o objetivo do nosso elemento personalizado significa essencialmente, se você passar uma matriz de locais assim:

<g-map location-array='["Noruega","Suécia"]'></g-map>

O elemento personalizado

  • Não deve apenas marcar a Noruega e a Suécia no mapa, mas,
  • Também deve marcar todas as pesquisas subsequentes no mapa - ou seja, se você pesquisar por Boston, após o mapa ter marcado a Noruega e a Suécia, Boston também deve ter um alfinete no mapa

C - Adicionando a capacidade de pesquisa ao nosso elemento personalizado #

Para pesquisar um local, usamos o poderoso elemento fornecido pelo Polymer, chamado google-map-search .

Tudo que você precisa fazer é passar

  • um objeto de mapa e
  • uma string de consulta para o elemento assim:
<google-map-search map=[[map]] query=[[query]]></google-map-search>

Como passamos o objeto do mapa? De onde tiramos isso?

Simples!

Ao invocar o elemento google-map, vincule o mapa de propriedades do seu elemento personalizado ao mapa de propriedades do elemento .

Então, invocamos o elemento google-map assim:

<google-map api-key="whatever key you generated for google's javascript API" map={{map}}></google-map>

Nota: #

Fazemos uma ligação de dados bidirecional para o mapa acima, ao invocar o google-map polímero.

Ele é representado pelas chaves {{}} em vez de um sentido único encadernação indicada por colchetes[[]].

Quando fazemos uma ligação bidirecional,

  • Sempre que a propriedade do mapa do nosso elemento personalizado g-map muda, o elemento google-map é notificado

e,

  • Sempre que a propriedade do mapa do google-map muda, somos notificados em nosso elemento personalizado g-map.

Portanto, com a vinculação de dados bidirecional em vigor, sempre que o objeto do mapa for alterado no google-map, nosso elemento personalizado será atualizado com as alterações.

e passamos o objeto map como um atributo para o elemento google-map-search, para que , qualquer resultado de pesquisa seja marcado no mapa atual renderizado.

Como está o DOM do nosso elemento personalizado neste momento?

<template id="google-map-custom-element">
 
<google-map-search map=[[map]] query=[[query]]></google-map-search>
<google-map api-key=[[apiKey]] map={{map}}></google-map>
</template>

Nunca adicionamos as propriedades de mapa e consulta para nosso elemento personalizado!

Adicione as propriedades na chamada de registro ao Polymer

Polymer({
 
    is:"g-map",
 
    properties:{
 
        map:{
 
            type: Object
 
        },
 
        query:{
 
            type:String,
 
            value:""
 
        }
    }
 
});

Agora que registramos as propriedades, notamos que

  • Estão obtendo o mapa atualmente renderizado como um objeto - via vinculação de dados e armazenando-o como uma propriedade local também chamada de mapa,
  • Ter uma consulta de propriedade, passada para google-map-search

Aceitando a entrada do usuário como consulta de pesquisa #

Mas quem nos dá a consulta para pesquisar? Errar! agora mesmo? NINGUÉM.

Isso é precisamente o que fazemos a seguir.

Vamos adicionar um formulário de pesquisa, que o usuário que renderiza o mapa usaria para inserir uma consulta. Precisamos de entradas para a caixa de pesquisa e usamos a entrada de ferro do Polymer.

Adicionando um formulário de pesquisa #

Inclua os elementos de ferro necessários no início do arquivo g-map.html

<link rel=import href=../bower_components/iron-input/iron-input.html>
<link rel=import href=../bower_components/iron-icon/iron-icon.html>
<link rel=import href=../bower_components/iron-icons/iron-icons.html>
<link rel=import href=../bower_components/iron-icons/maps-icons.html>

Queremos o formulário de pesquisa, no canto superior esquerdo, então adicionamos uma entrada de ferro com posição absoluta e alguns css para fixá-la lá.

Então, envolva o DOM para nosso elemento personalizado, dentro de um div chamado “map-container“, e adicione um div filho chamado “search_form“.

<template id="google-map-custom-element">
    <div class="map-container">
         <google-map-search map="[[map]]" query="[[query]]"></google-map-search>
         <google-map api-key="[[apiKey]]"></google-map>
         <div class="search_form">
             <iron-icon icon="icons:search" on-tap=__search></iron-icon>
             <input is="iron-input" placeholder="Search Places" type=text name="search" bind-value="{{query}}">
         </div>
     </div>
 </template>

Adicione algum CSS básico para fixar o formulário de pesquisa que escrevemos no canto superior direito do mapa

.search_form{
    position: absolute;
    top:10px;
    right:10px;
}
.search_form iron-icon{
    position: absolute;
    right: 1px;
    top:1px;
    cursor: pointer;
}

E , então, quando atualizamos, temos isso. O formulário de pesquisa no canto superior direito

Tudo o que resta a fazer é fornecer uma entrada para o elemento google-map-search, por meio da propriedade query do nosso elemento personalizado.

Então, vinculamos nossa propriedade “query”, à entrada de ferro no formulário de pesquisa. igual a:

<input is="iron-input" placeholder="Search" type=text name="search" bind-value="{{query}}">

Nota: #

Vinculamos a consulta de propriedade do nosso elemento personalizado à entrada de pesquisa usando,

atributo “bind-value” de um insumo de ferro.

Então, bind-value={{query}} na entrada de ferro acima, garante que qualquer alterações no texto dentro da entrada do ferro, é notificado de volta ao propriedade de consulta.

Com nossa configuração atual,

Para cada alfabeto que um usuário digita na caixa de pesquisa, ocorre uma pesquisa, que deixa o mapa mais lento e consome o limite da nossa chave de API.

Por quê?

Porque, a consulta de propriedade do nosso elemento personalizado, está vinculada ao iron entrada, e cada letra digitada na entrada de ferro, altera o valor de “query“, que então afeta o html em

<google-map-search map="[[map]]" query="[[query]]">

fazendo com que uma busca aconteça

Nossa solução é simples! Basta vincular o atributo “query” do elemento google-map-search a uma propriedade separada.

Digamos que criamos uma propriedade “searchQuery” para nosso elemento personalizado.

Adicione isso, abaixo da propriedade “query” na chamada do Polymer

searchQuery :{
 
    type:String
 
}

Em seguida, altere a ligação na chamada do elemento google-map-search no DOM do nosso elemento personalizado

igual a:

<google-map-search map="[[map]]" query="[[searchQuery]]">

Por fim, observe que adicionamos um ícone de ferro (ícone de pesquisa) ao nosso formulário de pesquisa?

<iron-icon icon="icons:search" on-tap=search></iron-icon>

Vamos adicionar essa função de pesquisa ao nosso elemento personalizado. É chamado ao tocar no ícone de pesquisa.

Em nosso método de pesquisa, atribuímos o que estiver na caixa de pesquisa ao atributo “query” do google-map-search!

Adicione esta função, após o objeto “propriedades” no polímero chamar para registro.

//paste this after, the properties object
 
search: function() {
this.query = this.searchQuery;
}

Muito agradável! Então, agora, uma pesquisa acontece apenas na torneira do ícone de pesquisa.

Exibir os resultados da pesquisa #

Nós temos

  • Renderizou um mapa
  • Fixou um formulário de pesquisa no mapa
  • Escreveu a funcionalidade para pesquisar na torneira do ícone de pesquisa

Agora precisamos exibir os resultados.

Para isso, precisamos vincular os resultados do elemento google-map-search a uma propriedade local.

Então, vamos criar uma propriedade para nosso elemento personalizado g-map, e nomeá-lo results duh!

Na chamada de registro Polymer, adicione os resultados da propriedade do tipo objeto na declaração de propriedades

results:{
 
type:Object
 
}

E vincule a propriedade results de google-map-search à propriedade results local que você definiu acima.

<google-map-search results={{results}} map=[[map]] query=[[query]]></google-map-search>

Feito isso, vamos atualizar a página e pesquisar starbucks no mapa.

Nada aconteceu!

Bem Duh! acabamos de vincular os resultados do google-map-search à propriedade results. Escrevemos alguma coisa para exibi-los no mapa? NÃO!

Nós precisamos.

Então, sabemos que o elemento Polymer google-map-search, retorna uma matriz de marcadores como resultados.

Precisamos percorrer o array e colocar marcadores no mapa.

Então, escrevemos um modelo dom-repeat, dentro do nosso elemento , para exibir os marcadores para cada resultado

igual a:

<google-map api-key=[[apiKey]] map=[[map]]>
 
    <template is="dom-repeat" items="{{results}}" as="marker">
        <google-map-marker latitude="{{marker.latitude}}" longitude="{{marker.longitude}}">
            <h2>{{marker.name}}</h2>
            <span>{{marker.formatted_address}}</span>
        </google-map-marker>
    </template>
 
</google-map>

E agora na atualização, vemos isso

Muito agradável!

Adicionamos com sucesso a funcionalidade de pesquisa ao nosso elemento personalizado.

Tudo o que precisamos fazer é,

escreva isso em nosso index.html

<g-map api-key="Whatever API key you generated"></g-map>

E você recebe um mapa no qual você pode pesquisar!

D - Resultados de pesquisa em cache #

Nosso objetivo, porém, era

  • fazer uma lista de lugares
  • marque todos eles no mapa e,
  • marque qualquer outro local de nossa escolha, usando a caixa de pesquisa

Temos uma caixa de pesquisa, onde o usuário pode pesquisar uma consulta por vez. Para acomodar várias consultas, precisamos armazenar em cache os resultados de cada consulta, antes de exibir os marcadores e fixá-los no mapa

Então, vamos adicionar

  • Uma propriedade diz locais que aceitam uma variedade de locais
  • Um cache de propriedade que armazena os resultados de cada consulta no array de localização passado para nosso elemento personalizado
cache:{
 
type:Object
 
},
 
locations:{
 
type:Object
 
}

Em seguida, escrevemos uma função para armazenar em cache cada resultado de pesquisa, resultante de uma pesquisa para cada lugar na matriz de localização.

É bem simples!

Já estamos reunindo os resultados em uma propriedade chamada “resultados”, que então estamos percorrendo para exibir marcadores.

Tudo o que precisamos fazer agora é enviar cada resultado da pesquisa para um array e nosso cache está pronto!

Precisamos usar um evento para acionar o cache. Esse evento é chamado de “on-google-map-search-result” e é acionado, após o elemento google-map-search finalizar uma pesquisa.

Portanto, “on-google-map-search-result” , chamamos uma função, digamos, “cacher” , para armazenar em cache os resultados da pesquisa atual em uma propriedade chamada cache.

<google-map-search map=[[map]] query=[[searchQuery]] on-google-map-search-results=cacher>

Nota: #

Normalmente, para enviar um item para um array, usamos o comando vanilla método

por exemplo: tenho um novo local para adicionar à minha lista de locais que quero marcar, diga Nova York.

Se o array ao qual eu quero adicioná-lo for chamado de locais, eu escreveria

locais.push('Nova York');

No entanto, no nosso caso, precisamos envie qualquer resultado do google-map-search para um array chamado cache.Then precisamos percorrer o cache com os resultados mais recentes incluídos. o cache existente.

Agora, usamos o modelo dom-repeat para percorrer os resultados e colocar marcadores no mapa.

No entanto, se passarmos um array de localização assim:

<g-map locations=["Iceland","Argentina","London"]></g-map>

Em seguida, a propriedade results do nosso elemento personalizado será substituída por cada item desse array.

Portanto, chamamos o método cacher evento on-google-map-search-results e enviamos o resultado atual para a propriedade cache.

Isso não pode ser um impulso de baunilha como na nota acima.

Precisamos informar ao repetidor dom que nosso cache foi substituído a cada nova pesquisa.

Então, usamos um push especial que o Polymer envia, assim:

cacher: function() {
this.push('cache', this.results);
}

Adicione essa função, após a função de pesquisa que escrevemos anteriormente.

A sintaxe implica que o cache de propriedade deve ser alterado, pela adição do valor atual dos resultados da propriedade e quaisquer repetidores dom usando a propriedade cache para fazer o loop, devem ser notificados da mutação.

Por fim, alteramos o modelo dom-repeat para percorrer o cache, em vez de resultados

Nota: #

cache é um array de arrays.

cada elemento do array de cache, é um array de resultados

Então nosso dom-repeater, será assim:

<template id="resultList" is="dom-repeat" items="[[cache]]">
 
    <template is="dom-repeat" items="[[item]]" as="marker">
 
        <google-map-marker latitude="{{marker.latitude}}" longitude="{{marker.longitude}}">
            <h2>{{marker.name}}</h2>
            <span>{{marker.formatted_address}}</span>
        </google-map-marker>
    </template>
</template>

Muito agradável! Temos o elemento personalizado todo codificado e pronto para ser usado!

<dom-module id="g-map">
    <template id="google-map">
        <div class="map-container">
            <google-map-search map="[[map]]" query="[[query]]" results="{{results}}" on-google-map-search-results=cacher>
            </google-map-search>
            <google-map api-key="[[apiKey]]" map="{{map}}" on-google-map-ready="_onMapResolve" fit-to-markers disable-zoom single-info-window>
                <template id="resultList" is="dom-repeat" items="[[cache]]">
 
                    <template is="dom-repeat" items="[[item]]" as="marker">
 
                        <google-map-marker latitude="{{marker.latitude}}" longitude="{{marker.longitude}}">
                            <h2>{{marker.name}}</h2>
                            <span>{{marker.formatted_address}}</span>
                        </google-map-marker>
                    </template>
                </template>
            </google-map>
            <div class="search_form">
                <p>
                    <iron-icon icon=icons:search on-tap=search></iron-icon>
                    <input is=iron-input placeholder="Search" type=text name=start bind-value={{searchQuery}}>
                </p>
            </div>
        </div>
    </template>
    <script>
        Polymer({
            is: "g-map",
            properties: {
                apiKey: {
                    type: String,
                    value: "AIzaSyBLc_T17p91u6ujSpThe2H3nh_8nG2p6FQ"
                },
                map: {
                    type: Object
                },
                query: {
                    type: String,
                    value: ""
                },
                locations: {
                    type: Array,
                    value: []
                },
                cache: {
                    type: Array,
                    value: []
                },
                results: {
                    type: Array,
                    value: function() {
                            return [];
                        }
                    }
            },
            _onMapResolve: function() {
                var search = document.querySelector("google-map-search");
                this.locations.forEach(function(location) {
                    search.query = location;
                })
            },
            search: function() {
                this.query = this.searchQuery;
            },
            cacher: function() {
                this.push('cache', this.results);
            }
        })
    </script>
</dom-module>
