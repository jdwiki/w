Criando uma tela de página inteira responsiva #

Código inicial para criar e remover uma tela de página inteira que responde a eventos de redimensionamento via javascript.

var canvas;    // Global canvas reference
var ctx;       // Global 2D context reference
// Creates a canvas
function createCanvas () {                
    const canvas = document.createElement("canvas"); 
    canvas.style.position = "absolute"; // Set the style 
    canvas.style.left     = "0px";      // Position in top left
    canvas.style.top      = "0px";
    canvas.style.zIndex   = 1;        
    document.body.appendChild(canvas);  // Add to document
    return canvas;
}
// Resizes canvas. Will create a canvas if it does not exist
function sizeCanvas () {                
    if (canvas === undefined) {         // Check for global canvas reference
        canvas = createCanvas();        // Create a new canvas element
        ctx = canvas.getContext("2d");  // Get the 2D context
    }
    canvas.width  = innerWidth;         // Set the canvas resolution to fill the page
    canvas.height = innerHeight;        
}
// Removes the canvas
function removeCanvas () {
    if (canvas !== undefined) {              // Make sure there is something to remove
        removeEventListener("resize", sizeCanvas); // Remove resize event
        document.body.removeChild(canvas);   // Remove the canvas from the DOM
        ctx = undefined;                     // Dereference the context
        canvas = undefined;                  // Dereference the canvas
     }
}

// Add the resize listener
addEventListener("resize", sizeCanvas); 
// Call sizeCanvas to create and set the canvas resolution
sizeCanvas();
// ctx and canvas are now available for use.

Se você não precisar mais da tela, poderá removê-la chamando removeCanvas()

Coordenadas do mouse após o redimensionamento (ou rolagem) #

Os aplicativos de tela geralmente dependem muito da interação do usuário com o mouse, mas quando a janela é redimensionada, as coordenadas do evento do mouse nas quais a tela depende provavelmente são alteradas porque o redimensionamento faz com que a tela seja deslocada em uma posição diferente em relação à janela. Assim, o design responsivo requer que a posição de deslocamento da tela seja recalculada quando a janela for redimensionada – e também recalculada quando a janela for rolada.

Este código escuta eventos de redimensionamento de janela e recalcula os deslocamentos usados ​​nos manipuladores de eventos do mouse:

// variables holding the current canvas offset position
//    relative to the window
var offsetX,offsetY;

// a function to recalculate the canvas offsets
function reOffset(){
    var BB=canvas.getBoundingClientRect();
    offsetX=BB.left;
    offsetY=BB.top;        
}

// listen for window resizing (and scrolling) events
//     and then recalculate the canvas offsets
window.onscroll=function(e){ reOffset(); }
window.onresize=function(e){ reOffset(); }

// example usage of the offsets in a mouse handler
function handleMouseUp(e){
    // use offsetX & offsetY to get the correct mouse position
    mouseX=parseInt(e.clientX-offsetX);
    mouseY=parseInt(e.clientY-offsetY);
    // ...
}

Animações de tela responsivas sem eventos de redimensionamento. #

Os eventos de redimensionamento da janela podem ser acionados em resposta ao movimento do dispositivo de entrada do usuário. Quando você redimensiona uma tela, ela é automaticamente limpa e você é forçado a renderizar novamente o conteúdo. Para animações, você faz isso a cada quadro através da função de loop principal chamada por requestAnimationFrame, que faz o possível para manter a renderização em sincronia com o hardware de exibição.

O problema com o evento de redimensionamento é que quando o mouse é usado para redimensionar a janela, os eventos podem ser acionados muitas vezes mais rápido do que a taxa padrão de 60 fps do navegador. Quando o evento de redimensionamento sai, o buffer de fundo da tela é apresentado ao DOM fora de sincronia com o dispositivo de exibição, o que pode causar cisalhamento e outros efeitos negativos. Há também muita alocação e liberação de memória desnecessárias que podem impactar ainda mais a animação quando o GC limpa algum tempo depois.

Evento de redimensionamento rejeitado #

Uma maneira comum de lidar com as altas taxas de disparo do evento de redimensionamento é debounce o evento de redimensionamento.

 // Assume canvas is in scope
 addEventListener.("resize", debouncedResize );

 // debounce timeout handle
 var debounceTimeoutHandle;

 // The debounce time in ms (1/1000th second)
 const DEBOUNCE_TIME = 100; 

 // Resize function 
 function debouncedResize () { 
     clearTimeout(debounceTimeoutHandle);  // Clears any pending debounce events

     // Schedule a canvas resize 
     debounceTimeoutHandle = setTimeout(resizeCanvas, DEBOUNCE_TIME);
 }

 // canvas resize function
 function resizeCanvas () { ... resize and redraw ... }

O exemplo acima atrasa o redimensionamento da tela até 100ms após o evento de redimensionamento. Se nesse período outros eventos de redimensionamento forem acionados, o tempo limite de redimensionamento existente será cancelado e um novo será agendado. Isso efetivamente consome a maioria dos eventos de redimensionamento.

Ainda apresenta alguns problemas, o mais notável é o atraso entre o redimensionamento e a visualização da tela redimensionada. Reduzir o tempo de debounce melhora isso, mas o redimensionamento ainda está fora de sincronia com o dispositivo de exibição. Você também tem a renderização do loop principal da animação para uma tela mal ajustada.

Mais código pode reduzir os problemas! Mais código também cria seus próprios novos problemas.

Simples e o melhor redimensionamento #

Tendo tentado muitas maneiras diferentes de suavizar o redimensionamento da tela, desde o absurdamente complexo, até simplesmente ignorar o problema (quem se importa de qualquer maneira?) eu recorri a um amigo de confiança.

K.I.S.S. é algo que a maioria dos programadores deveria estar ciente ((Keep It Ssimples Stupid) O estúpido se refere a mim por não ter pensado disso anos atrás. ) e acontece que a melhor solução é a mais simples de todas.

Basta redimensionar a tela de dentro do loop de animação principal. Ele permanece sincronizado com o dispositivo de exibição, não há renderização desnecessária e o gerenciamento de recursos é o mínimo possível, mantendo a taxa de quadros total. Nem você precisa adicionar um evento de redimensionamento à janela ou qualquer função de redimensionamento adicional.

Você adiciona o redimensionamento onde normalmente limparia a tela verificando se o tamanho da tela corresponde ao tamanho da janela. Se não redimensione.

// Assumes canvas element is in scope as canvas

// Standard main loop function callback from requestAnimationFrame
function mainLoop(time) {

    // Check if the canvas size matches the window size
    if (canvas.width !== innerWidth || canvas.height !== innerHeight) {
        canvas.width = innerWidth;    // resize canvas
        canvas.height = innerHeight;  // also clears the canvas
    } else {
        ctx.clearRect(0, 0, canvas.width, canvas.height); // clear if not resized
    }

    // Animation code as normal.



    requestAnimationFrame(mainLoop);
}
