Desenhando muitas imagens traduzidas, dimensionadas e giradas rapidamente #

Há muitas situações em que você deseja desenhar uma imagem que é girada, dimensionada e traduzida. A rotação deve ocorrer em torno do centro da imagem. Esta é a maneira mais rápida de fazer isso na tela 2D. Estas funções são adequadas para jogos 2D onde a expectativa é renderizar algumas centenas até mais de 1000 imagens a cada 60 segundos. (dependendo do hardware)

// assumes that the canvas context is in ctx and in scope
function drawImageRST(image, x, y, scale, rotation){
    ctx.setTransform(scale, 0, 0, scale, x, y); // set the scale and translation
    ctx.rotate(rotation);                       // add the rotation
    ctx.drawImage(image, -image.width / 2, -image.height / 2); // draw the image offset by half its width and height
}

Uma variante também pode incluir o valor alfa que é útil para sistemas de partículas.

function drawImageRST_Alpha(image, x, y, scale, rotation, alpha){
    ctx.setTransform(scale, 0, 0, scale, x, y); // set the scale and translation
    ctx.rotate(rotation);                       // add the rotation
    ctx.globalAlpha = alpha;
    ctx.drawImage(image, -image.width / 2, -image.height / 2); // draw the image offset by half its width and height
}

É importante observar que ambas as funções deixam o contexto da tela em um estado aleatório. Embora as funções não sejam afetadas, outras renderizações serão. Quando você terminar de renderizar as imagens, talvez seja necessário restaurar a transformação padrão

ctx.setTransform(1, 0, 0, 1, 0, 0); // set the context transform back to the default

Se você usar a versão alfa (segundo exemplo) e depois a versão padrão, você terá que garantir que o estado alfa global seja restaurado

ctx.globalAlpha = 1;

Um exemplo de uso das funções acima para renderizar algumas partículas e algumas imagens

// assume particles to contain an array of particles
for(var i = 0; i < particles.length; i++){
    var p = particles[i];
    drawImageRST_Alpha(p.image, p.x, p.y, p.scale, p.rot, p.alpha);
    // no need to rest the alpha in the loop
}
// you need to reset the alpha as it can be any value 
ctx.globalAlpha = 1;

drawImageRST(myImage, 100, 100, 1, 0.5);  // draw an image at 100,100
// no need to reset the transform 
drawImageRST(myImage, 200, 200, 1, -0.5); // draw an image at 200,200 
ctx.setTransform(1,0,0,1,0,0);            // reset the transform

Gire uma imagem ou caminho em torno de seu ponto central #

Os passos 1-5 abaixo permitem que qualquer imagem ou forma de caminho seja movida para qualquer lugar na tela e girada para qualquer ângulo sem alterar nenhuma das coordenadas de ponto original da imagem/forma de caminho.

  1. Mova a origem da tela [0,0] para o ponto central da forma

    context.translate( shapeCenterX, shapeCenterY );

  2. Gire a tela pelo ângulo desejado (em radianos)

    context.rotate( radianAngle );

  3. Mova a origem da tela de volta para o canto superior esquerdo

     context.translate( -shapeCenterX, -shapeCenterY );

  4. Desenhe a imagem ou forma do caminho usando suas coordenadas originais.

     context.fillRect( shapeX, shapeY, shapeWidth, shapeHeight );

  5. Sempre limpe! Defina o estado de transformação de volta para onde estava antes de #1

  • Etapa#5, Opção#1: Desfaça todas as transformações na ordem inversa

          // undo #3
      context.translate( shapeCenterX, shapeCenterY );
      // undo #2
      context.rotate( -radianAngle );
      // undo #1
      context.translate( -shapeCenterX, shapeCenterY );

    • Step#5, Option#2: If the canvas was in an untransformed state (the default) before beginning step#1, you can undo the effects of steps#1-3 by resetting all transformations to their default state

         // set transformation to the default state (==no transformation applied)
   context.setTransform(1,0,0,1,0,0)

Exemplo de demonstração de código:

// canvas references & canvas styling
var canvas=document.createElement("canvas");
canvas.style.border='1px solid red';
document.body.appendChild(canvas);
canvas.width=378;
canvas.height=256;
var ctx=canvas.getContext("2d");
ctx.fillStyle='green';
ctx.globalAlpha=0.35;        

// define a rectangle to rotate
var rect={ x:100, y:100, width:175, height:50 };

// draw the rectangle unrotated
ctx.fillRect( rect.x, rect.y, rect.width, rect.height );

// draw the rectangle rotated by 45 degrees (==PI/4 radians)
ctx.translate( rect.x+rect.width/2, rect.y+rect.height/2 );
ctx.rotate( Math.PI/4 );
ctx.translate( -rect.x-rect.width/2, -rect.y-rect.height/2 );
ctx.fillRect( rect.x, rect.y, rect.width, rect.height );

Introdução às transformações #

As transformações alteram a posição inicial de um determinado ponto movendo, girando e dimensionando esse ponto.

  • Tradução: move um ponto em uma distanceX e uma distanceY.
  • Rotação: gira um ponto em um “ângulo radiano” em torno de seu ponto de rotação. O ponto de rotação padrão no Html Canvas é a origem superior esquerda [x=0,y=0] do Canvas. Mas você pode reposicionar o ponto de rotação usando translações.
  • Scaling: dimensiona a posição de um ponto por um scalingFactorX e um scalingFactorY a partir de seu ponto de dimensionamento. O ponto de escala padrão no Html Canvas é a origem superior esquerda [x=0,y=0] do Canvas. Mas você pode reposicionar o ponto de escala usando traduções.

Você também pode fazer transformações menos comuns, como cisalhamento (inclinação), definindo diretamente a matriz de transformação da tela usando context.transform.

Traduzir (==mover) um ponto com context.translate(75,25)

Gire um ponto com context.rotate(Math.PI/8)

Escala um ponto com context.scale(2,2)

O Canvas realmente realiza transformações alterando todo o sistema de coordenadas do canvas.

  • context.translate moverá a origem da tela [0,0] do canto superior esquerdo para um novo local.
  • context.rotate irá girar todo o sistema de coordenadas da tela em torno da origem.
  • context.scale dimensionará todo o sistema de coordenadas da tela em torno da origem. Pense nisso como aumentar o tamanho de cada x,y na tela: every x*=scaleX e every y*=scaleY.

As transformações de tela são persistentes. Todos os novos desenhos continuarão a ser transformados até que você redefina a transformação da tela de volta ao seu estado padrão (==totalmente não transformado). Você pode redefinir para o padrão com:

// reset context transformations to the default (untransformed) state
context.setTransform(1,0,0,1,0,0);

Uma Matriz de Transformação para rastrear formas traduzidas, giradas e dimensionadas #

O Canvas permite que você context.translate, context.rotate e context.scale para desenhar sua forma na posição e tamanho que você precisa.

O próprio Canvas usa uma matriz de transformação para rastrear transformações com eficiência.

  • Você pode alterar a matriz do Canvas com context.transform
  • Você pode alterar a matriz do Canvas com comandos individuais de ’traduzir, girar e dimensionar'
  • Você pode substituir completamente a matriz do Canvas com context.setTransform,
  • Mas você não pode ler a matriz de transformação interna do Canvas - é somente gravação.

Por que usar uma matriz de transformação? #

Uma matriz de transformação permite agregar muitas traduções, rotações e escalas individuais em uma única matriz facilmente reaplicada.

Durante animações complexas, você pode aplicar dezenas (ou centenas) de transformações a uma forma. Ao usar uma matriz de transformação, você pode (quase) reaplicar instantaneamente essas dezenas de transformações com uma única linha de código.

Alguns exemplos de uso:

  • Teste se o mouse está dentro de uma forma que você traduziu, girou e dimensionou

    There is a built-in context.isPointInPath that tests if a point (eg the mouse) is inside a path-shape, but this built-in test is very slow compared to testing using a matrix.

Testar com eficiência se o mouse está dentro de uma forma envolve pegar a posição do mouse informada pelo navegador e transformá-la da mesma forma que a forma foi transformada. Em seguida, você pode aplicar o teste de acerto como se a forma não tivesse sido transformada.

  • Redesenhe uma forma que foi amplamente traduzida, girada e dimensionada.

    Instead of reapplying individual transformations with multiple .translate, .rotate, .scale you can apply all the aggregated transformations in a single line of code.

  • Formas de teste de colisão que foram convertidas, giradas e dimensionadas

    You can use geometry & trigonometry to calculate the points that make up transformed shapes, but it’s faster to use a transformation matrix to calculate those points.

Uma Matriz de Transformação “Classe” #

Este código espelha os comandos de transformação nativos context.translate, context.rotate, context.scale. Ao contrário da matriz de tela nativa, esta matriz é legível e reutilizável.

Métodos:

  • translate, rotate, scale espelham os comandos de transformação de contexto e permitem que você alimente transformações na matriz. A matriz contém eficientemente as transformações agregadas.

  • setContextTransform pega um contexto e define a matriz desse contexto igual a esta matriz de transformação. Isso reaplica com eficiência todas as transformações armazenadas nessa matriz ao contexto.

  • resetContextTransform redefine a transformação do contexto para seu estado padrão (== não transformado).

  • getTransformedPoint pega um ponto de coordenada não transformado e o converte em um ponto transformado.

  • getScreenPoint pega um ponto de coordenada transformado e o converte em um ponto não transformado.

  • getMatrix retorna as transformações agregadas na forma de um array de matrizes.

Código:

var TransformationMatrix=( function(){
    // private
    var self;
    var m=[1,0,0,1,0,0];
    var reset=function(){ var m=[1,0,0,1,0,0]; }
    var multiply=function(mat){
        var m0=m[0]*mat[0]+m[2]*mat[1];
        var m1=m[1]*mat[0]+m[3]*mat[1];
        var m2=m[0]*mat[2]+m[2]*mat[3];
        var m3=m[1]*mat[2]+m[3]*mat[3];
        var m4=m[0]*mat[4]+m[2]*mat[5]+m[4];
        var m5=m[1]*mat[4]+m[3]*mat[5]+m[5];
        m=[m0,m1,m2,m3,m4,m5];
    }
    var screenPoint=function(transformedX,transformedY){
        // invert
        var d =1/(m[0]*m[3]-m[1]*m[2]);
        im=[ m[3]*d, -m[1]*d, -m[2]*d, m[0]*d, d*(m[2]*m[5]-m[3]*m[4]), d*(m[1]*m[4]-m[0]*m[5]) ];
        // point
        return({
            x:transformedX*im[0]+transformedY*im[2]+im[4],
            y:transformedX*im[1]+transformedY*im[3]+im[5]
        });
    }
    var transformedPoint=function(screenX,screenY){
        return({
            x:screenX*m[0] + screenY*m[2] + m[4],
            y:screenX*m[1] + screenY*m[3] + m[5]
        });    
    }
    // public
    function TransformationMatrix(){
        self=this;
    }
    // shared methods
    TransformationMatrix.prototype.translate=function(x,y){
        var mat=[ 1, 0, 0, 1, x, y ];
        multiply(mat);
    };
    TransformationMatrix.prototype.rotate=function(rAngle){
        var c = Math.cos(rAngle);
        var s = Math.sin(rAngle);
        var mat=[ c, s, -s, c, 0, 0 ];    
        multiply(mat);
    };
    TransformationMatrix.prototype.scale=function(x,y){
        var mat=[ x, 0, 0, y, 0, 0 ];        
        multiply(mat);
    };
    TransformationMatrix.prototype.skew=function(radianX,radianY){
        var mat=[ 1, Math.tan(radianY), Math.tan(radianX), 1, 0, 0 ];
        multiply(mat);
    };
    TransformationMatrix.prototype.reset=function(){
        reset();
    }
    TransformationMatrix.prototype.setContextTransform=function(ctx){
        ctx.setTransform(m[0],m[1],m[2],m[3],m[4],m[5]);
    }
    TransformationMatrix.prototype.resetContextTransform=function(ctx){
        ctx.setTransform(1,0,0,1,0,0);
    }
    TransformationMatrix.prototype.getTransformedPoint=function(screenX,screenY){
        return(transformedPoint(screenX,screenY));
    }
    TransformationMatrix.prototype.getScreenPoint=function(transformedX,transformedY){
        return(screenPoint(transformedX,transformedY));
    }
    TransformationMatrix.prototype.getMatrix=function(){
        var clone=[m[0],m[1],m[2],m[3],m[4],m[5]];
        return(clone);
    }
    // return public
    return(TransformationMatrix);
})();

Demonstração:

Esta demonstração usa a “Classe” da Matriz de Transformação acima para:

  • Track (==save) matriz de transformação de um retângulo.

  • Redesenhe o retângulo transformado sem usar comandos de transformação de contexto.

  • Teste se o mouse clicou dentro do retângulo transformado.

Código:

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }
    #canvas{border:1px solid red; }
</style>
<script>
window.onload=(function(){

    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");
    var cw=canvas.width;
    var ch=canvas.height;
    function reOffset(){
        var BB=canvas.getBoundingClientRect();
        offsetX=BB.left;
        offsetY=BB.top;        
    }
    var offsetX,offsetY;
    reOffset();
    window.onscroll=function(e){ reOffset(); }
    window.onresize=function(e){ reOffset(); }

    // Transformation Matrix "Class"
    
    var TransformationMatrix=( function(){
        // private
        var self;
        var m=[1,0,0,1,0,0];
        var reset=function(){ var m=[1,0,0,1,0,0]; }
        var multiply=function(mat){
            var m0=m[0]*mat[0]+m[2]*mat[1];
            var m1=m[1]*mat[0]+m[3]*mat[1];
            var m2=m[0]*mat[2]+m[2]*mat[3];
            var m3=m[1]*mat[2]+m[3]*mat[3];
            var m4=m[0]*mat[4]+m[2]*mat[5]+m[4];
            var m5=m[1]*mat[4]+m[3]*mat[5]+m[5];
            m=[m0,m1,m2,m3,m4,m5];
        }
        var screenPoint=function(transformedX,transformedY){
            // invert
            var d =1/(m[0]*m[3]-m[1]*m[2]);
            im=[ m[3]*d, -m[1]*d, -m[2]*d, m[0]*d, d*(m[2]*m[5]-m[3]*m[4]), d*(m[1]*m[4]-m[0]*m[5]) ];
            // point
            return({
                x:transformedX*im[0]+transformedY*im[2]+im[4],
                y:transformedX*im[1]+transformedY*im[3]+im[5]
            });
        }
        var transformedPoint=function(screenX,screenY){
            return({
                x:screenX*m[0] + screenY*m[2] + m[4],
                y:screenX*m[1] + screenY*m[3] + m[5]
            });    
        }
        // public
        function TransformationMatrix(){
            self=this;
        }
        // shared methods
        TransformationMatrix.prototype.translate=function(x,y){
            var mat=[ 1, 0, 0, 1, x, y ];
            multiply(mat);
        };
        TransformationMatrix.prototype.rotate=function(rAngle){
            var c = Math.cos(rAngle);
            var s = Math.sin(rAngle);
            var mat=[ c, s, -s, c, 0, 0 ];    
            multiply(mat);
        };
        TransformationMatrix.prototype.scale=function(x,y){
            var mat=[ x, 0, 0, y, 0, 0 ];        
            multiply(mat);
        };
        TransformationMatrix.prototype.skew=function(radianX,radianY){
            var mat=[ 1, Math.tan(radianY), Math.tan(radianX), 1, 0, 0 ];
            multiply(mat);
        };
        TransformationMatrix.prototype.reset=function(){
            reset();
        }
        TransformationMatrix.prototype.setContextTransform=function(ctx){
            ctx.setTransform(m[0],m[1],m[2],m[3],m[4],m[5]);
        }
        TransformationMatrix.prototype.resetContextTransform=function(ctx){
            ctx.setTransform(1,0,0,1,0,0);
        }
        TransformationMatrix.prototype.getTransformedPoint=function(screenX,screenY){
            return(transformedPoint(screenX,screenY));
        }
        TransformationMatrix.prototype.getScreenPoint=function(transformedX,transformedY){
            return(screenPoint(transformedX,transformedY));
        }
        TransformationMatrix.prototype.getMatrix=function(){
            var clone=[m[0],m[1],m[2],m[3],m[4],m[5]];
            return(clone);
        }
        // return public
        return(TransformationMatrix);
    })();

    // DEMO starts here

    // create a rect and add a transformation matrix
    // to track it's translations, rotations & scalings
    var rect={x:30,y:30,w:50,h:35,matrix:new TransformationMatrix()};

    // draw the untransformed rect in black
    ctx.strokeRect(rect.x, rect.y, rect.w, rect.h);
    // Demo: label
    ctx.font='11px arial';
    ctx.fillText('Untransformed Rect',rect.x,rect.y-10);

    // transform the canvas & draw the transformed rect in red
    ctx.translate(100,0);
    ctx.scale(2,2);
    ctx.rotate(Math.PI/12);
    // draw the transformed rect
    ctx.strokeStyle='red';
    ctx.strokeRect(rect.x, rect.y, rect.w, rect.h);
    ctx.font='6px arial';
    // Demo: label
    ctx.fillText('Same Rect: Translated, rotated & scaled',rect.x,rect.y-6);
    // reset the context to untransformed state
    ctx.setTransform(1,0,0,1,0,0);

    // record the transformations in the matrix
    var m=rect.matrix;
    m.translate(100,0);
    m.scale(2,2);
    m.rotate(Math.PI/12);

    // use the rect's saved transformation matrix to reposition, 
    //     resize & redraw the rect
    ctx.strokeStyle='blue';
    drawTransformedRect(rect);

    // Demo: instructions
    ctx.font='14px arial';
    ctx.fillText('Demo: click inside the blue rect',30,200);

    // redraw a rect based on it's saved transformation matrix
    function drawTransformedRect(r){
        // set the context transformation matrix using the rect's saved matrix
        m.setContextTransform(ctx);
        // draw the rect (no position or size changes needed!)
        ctx.strokeRect( r.x, r.y, r.w, r.h );
        // reset the context transformation to default (==untransformed);
        m.resetContextTransform(ctx);
    }

    // is the point in the transformed rectangle?
    function isPointInTransformedRect(r,transformedX,transformedY){
        var p=r.matrix.getScreenPoint(transformedX,transformedY);
        var x=p.x;
        var y=p.y;
        return(x>r.x && x<r.x+r.w && y>r.y && y<r.y+r.h);
    } 

    // listen for mousedown events
    canvas.onmousedown=handleMouseDown;
    function handleMouseDown(e){
        // tell the browser we're handling this event
        e.preventDefault();
        e.stopPropagation();
        // get mouse position
        mouseX=parseInt(e.clientX-offsetX);
        mouseY=parseInt(e.clientY-offsetY);
        // is the mouse inside the transformed rect?
        if(isPointInTransformedRect(rect,mouseX,mouseY)){
            alert('You clicked in the transformed Rect');
        }
    }

    // Demo: redraw transformed rect without using
    //       context transformation commands
    function drawTransformedRect(r,color){
        var m=r.matrix;
        var tl=m.getTransformedPoint(r.x,r.y);
        var tr=m.getTransformedPoint(r.x+r.w,r.y);
        var br=m.getTransformedPoint(r.x+r.w,r.y+r.h);
        var bl=m.getTransformedPoint(r.x,r.y+r.h);
        ctx.beginPath();
        ctx.moveTo(tl.x,tl.y);
        ctx.lineTo(tr.x,tr.y);
        ctx.lineTo(br.x,br.y);
        ctx.lineTo(bl.x,bl.y);
        ctx.closePath();
        ctx.strokeStyle=color;
        ctx.stroke();
    }

}); // end window.onload
</script>
</head>
