Caminho

Um caminho define um conjunto de linhas e curvas que podem ser visivelmente desenhadas no Canvas.

Um caminho não é desenhado automaticamente no Canvas. Mas as linhas e curvas do caminho podem ser desenhadas no Canvas usando um traço estilizado. E a forma criada pelas linhas e curvas também pode ser preenchida com um preenchimento estilizado.

Os caminhos têm usos além do desenho no Canvas:

  • Teste de acerto se uma coordenada x,y estiver dentro da forma do caminho.
  • Definir uma região de recorte onde apenas os desenhos dentro da região de recorte serão visíveis. Quaisquer desenhos fora da região de recorte não serão desenhados (==transparente) – semelhante ao estouro de CSS.

Os comandos básicos de desenho de caminho são:

  • inícioCaminho
  • mover para
  • linhaPara
  • arco
  • quadráticaCurveTo
  • bezierCurveTo
  • arcTo
  • correto
  • fecharCaminho

Descrição dos comandos básicos de desenho: #

beginCaminho

context.beginPath()

Começa a montar um novo conjunto de comandos de caminho e também descarta qualquer caminho montado anteriormente.

O descarte é um ponto importante e muitas vezes esquecido. Se você não iniciar um novo caminho, todos os comandos de caminho emitidos anteriormente serão redesenhados automaticamente.

Ele também move a “caneta” do desenho para a origem superior esquerda da tela (==coordenada[0,0]).

mover para

context.moveTo(startX, startY)

Move a localização atual da caneta para a coordenada [startX,startY].

Por padrão, todos os desenhos de caminho são conectados juntos. Portanto, o ponto final de uma linha ou curva é o ponto inicial da próxima linha ou curva. Isso pode fazer com que uma linha inesperada seja desenhada conectando dois desenhos adjacentes. O comando context.moveTo basicamente “pega a caneta de desenho” e a coloca em uma nova coordenada para que a linha de conexão automática não seja desenhada.

linhaPara

context.lineTo(endX, endY)

Desenha um segmento de linha da localização atual da caneta para coordenar [endX,endY]

Você pode montar vários comandos .lineTo para desenhar uma polilinha. Por exemplo, você pode montar 3 segmentos de linha para formar um triângulo.

arco

context.arc(centerX, centerY, radius, startingRadianAngle, endingRadianAngle)

Desenha um arco circular dado um ponto central, raio e ângulos inicial e final. Os ângulos são expressos em radianos. Para converter graus em radianos você pode usar esta fórmula: radianos = graus * Math.PI / 180;.

O ângulo 0 está voltado diretamente para a direita a partir do centro do arco. Para desenhar um círculo completo você pode fazer endingAngle = beginAngle + 360 graus (360 graus == Math.PI2): `context.arc(10,10,20,0,Math.PI2);

Por padrão, o arco é desenhado no sentido horário. Um parâmetro opcional [true|false] instrui o arco a ser desenhado no sentido anti-horário: context.arc(10,10,20,0,Math.PI*2,true)

**curva quadráticaPara **

context.quadraticCurveTo(controlX, controlY, endingX, endingY)

Desenha uma curva quadrática começando na localização atual da caneta até uma determinada coordenada final. Outra coordenada de controle determinada determina a forma (curvatura) da curva.

bezierCurveTo

context.bezierCurveTo(control1X, control1Y, control2X, control2Y, endingX, endingY)

Desenha uma curva Bezier cúbica começando na localização atual da caneta até uma determinada coordenada final. Outras 2 coordenadas de controle determinadas determinam a forma (curvatura) da curva.

arcPara

context.arcTo(pointX1, pointY1, pointX2, pointY2, radius);

Desenha um arco circular com um determinado raio. O arco é desenhado no sentido horário dentro da cunha formada pela localização atual da caneta e recebe dois pontos: Ponto1 e Ponto2.

Uma linha conectando a localização atual da caneta e o início do arco é desenhada automaticamente antes do arco.

correto

context.rect(leftX, topY, width, height)

Desenha um retângulo dado um canto superior esquerdo e uma largura e altura.

O context.rect é um comando de desenho exclusivo porque adiciona retângulos desconectados. Esses retângulos desconectados não são conectados automaticamente por linhas.

closePath

context.closePath()

Desenha uma linha da localização atual da caneta de volta à coordenada do caminho inicial.

Por exemplo, se você desenhar 2 linhas formando 2 pernas de um triângulo, closePath “fechará” o triângulo desenhando a terceira perna do triângulo do ponto final da segunda perna de volta ao ponto inicial da primeira perna.

O nome desse comando geralmente faz com que ele seja mal interpretado. context.closePath NÃO é um delimitador final para context.beginPath. Novamente, o comando closePath desenha uma linha – ele não “fecha” um beginPath.

lineTo (um comando de caminho) #

context.lineTo(endX, endY)

Desenha um segmento de linha da localização atual da caneta para coordenar [endX,endY]

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }
    #canvas{border:1px solid red; }
</style>
<script>
window.onload=(function(){

    // get a reference to the canvas element and it's context
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");

    // arguments
    var startX=25;
    var startY=20;
    var endX=125;
    var endY=20;

    // Draw a single line segment drawn using "moveTo" and "lineTo" commands
    ctx.beginPath();
    ctx.moveTo(startX,startY);
    ctx.lineTo(endX,endY);
    ctx.stroke();

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=200 height=150></canvas>
</body>
</html>

Você pode montar vários comandos .lineTo para desenhar uma polilinha. Por exemplo, você pode montar 3 segmentos de linha para formar um triângulo.

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }

</style>
<script>

    // get a reference to the canvas element and it's context
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");

    // arguments
    var topVertexX=50;
    var topVertexY=20;
    var rightVertexX=75;
    var rightVertexY=70;
    var leftVertexX=25;
    var leftVertexY=70;

    // A set of line segments drawn to form a triangle using
    //     "moveTo" and multiple "lineTo" commands
    ctx.beginPath();
    ctx.moveTo(topVertexX,topVertexY);
    ctx.lineTo(rightVertexX,rightVertexY);
    ctx.lineTo(leftVertexX,leftVertexY);
    ctx.lineTo(topVertexX,topVertexY);
    ctx.stroke();

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=200 height=150></canvas>
</body>
</html>

arco (um comando de caminho) #

context.arc(centerX, centerY, radius, startingRadianAngle, endingRadianAngle)

Desenha um arco circular dado um ponto central, raio e ângulos inicial e final. Os ângulos são expressos em radianos. Para converter graus em radianos você pode usar esta fórmula: radianos = graus * Math.PI / 180;.

O ângulo 0 está voltado diretamente para a direita a partir do centro do arco.

Por padrão, o arco é desenhado no sentido horário. Um parâmetro opcional [true|false] instrui o arco a ser desenhado no sentido anti-horário: context.arc(10,10,20,0,Math.PI*2,true)

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }
    #canvas{border:1px solid red; }
</style>
<script>
window.onload=(function(){

    // get a reference to the canvas element and its context
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");

    // arguments
    var centerX=50;
    var centerY=50;
    var radius=30;
    var startingRadianAngle=Math.PI*2*;  // start at 90 degrees == centerY+radius
    var endingRadianAngle=Math.PI*2*.75;  // end at 270 degrees (==PI*2*.75 in radians)

    // A partial circle (i.e. arc) drawn using the "arc" command
    ctx.beginPath();
    ctx.arc(centerX, centerY, radius,  startingRadianAngle, endingRadianAngle);
    ctx.stroke();

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=200 height=150></canvas>
</body>
</html>

Para desenhar um círculo completo você pode fazer endingAngle = beginAngle + 360 graus (360 graus == Math.PI2).

</body>
</html>

closePath (um comando de caminho) #

context.closePath()

Desenha uma linha da localização atual da caneta de volta à coordenada do caminho inicial.

Por exemplo, se você desenhar 2 linhas formando 2 pernas de um triângulo, closePath “fechará” o triângulo desenhando a terceira perna do triângulo do ponto final da segunda perna de volta ao ponto inicial da primeira perna.

Um equívoco explicado!

O nome desse comando geralmente faz com que ele seja mal interpretado.

context.closePath NÃO é um delimitador final para context.beginPath.

Novamente, o comando closePath desenha uma linha – ele não “fecha” um beginPath.

Este exemplo desenha 2 pernas de um triângulo e usa closePath para completar (fechar?!) o triângulo desenhando a terceira perna. O que closePath está realmente fazendo é desenhar uma linha do ponto final da segunda perna de volta ao ponto inicial da primeira perna.

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }
    #canvas{border:1px solid red; }
</style>
<script>
window.onload=(function(){

    // get a reference to the canvas element and it's context
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");

    // arguments
    var topVertexX=50;
    var topVertexY=50;
    var rightVertexX=75;
    var rightVertexY=75;
    var leftVertexX=25;
    var leftVertexY=75;

    // A set of line segments drawn to form a triangle using
    //     "moveTo" and multiple "lineTo" commands
    ctx.beginPath();
    ctx.moveTo(topVertexX,topVertexY);
    ctx.lineTo(rightVertexX,rightVertexY);
    ctx.lineTo(leftVertexX,leftVertexY);

    // closePath draws the 3rd leg of the triangle
    ctx.closePath()

    ctx.stroke();

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=200 height=150></canvas>
</body>
</html>

beginPath (um comando de caminho) #

context.beginPath()

Começa a montar um novo conjunto de comandos de caminho e também descarta qualquer caminho montado anteriormente.

Ele também move a “caneta” do desenho para a origem superior esquerda da tela (==coordenada[0,0]).

Embora opcional, você SEMPRE deve iniciar um caminho com beginPath

O descarte é um ponto importante e muitas vezes esquecido. Se você não iniciar um novo caminho com beginPath, quaisquer comandos de caminho emitidos anteriormente serão redesenhados automaticamente.

Essas 2 demos tentam desenhar um “X” com um traço vermelho e um traço azul.

Esta primeira demonstração usa corretamente o beginPath para iniciar seu segundo traço vermelho. O resultado é que o “X” corretamente tem um traço vermelho e um azul.

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }
    #canvas{border:1px solid red; }
</style>
<script>
window.onload=(function(){

    // get a reference to the canvas element and it's context
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");

    // draw a blue line
    ctx.beginPath();
    ctx.moveTo(30,30);
    ctx.lineTo(100,100);
    ctx.strokeStyle='blue';
    ctx.lineWidth=3;
    ctx.stroke();

    // draw a red line
    ctx.beginPath();        // Important to begin a new path! 
    ctx.moveTo(100,30);
    ctx.lineTo(30,100);
    ctx.strokeStyle='red';
    ctx.lineWidth=3;
    ctx.stroke();

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=200 height=150></canvas>
</body>
</html>

strokeStyle (um atributo de estilo de caminho) #

context.strokeStyle=color

Define a cor que será usada para traçar o contorno do caminho atual.

Estas são opções de cor (devem ser citadas):

  • Um CSS chamado color, por exemplo context.strokeStyle='red'

  • Uma cor hexadecimal, por exemplo context.strokeStyle='#FF0000'

  • Uma cor RGB, por exemplo context.strokeStyle='rgb(red,green,blue)' onde vermelho, verde e azul são inteiros de 0 a 255 indicando a intensidade de cada cor componente.

  • Uma cor HSL, por exemplo context.strokeStyle='hsl(hue,saturation,lightness)' onde matiz é um inteiro 0-360 na roda de cores e saturação e luminosidade são porcentagens (0-100% ) indicando a força de cada componente.

  • Uma cor HSLA, por exemplo context.strokeStyle='hsl(hue,saturation,lightness,alpha)' onde matiz é um inteiro 0-360 na roda de cores e saturação e luminosidade são porcentagens (0- 100%) indicando a força de cada componente e alfa é um valor decimal 0,00-1,00 indicando a opacidade.

Você também pode especificar essas opções de cores (essas opções são objetos criados pelo contexto):

  • Um gradiente linear que é um objeto gradiente linear criado com context.createLinearGradient

  • Um gradiente radial que é um objeto de gradiente radial criado com context.createRadialGradient

  • Um padrão que é um objeto de padrão criado com context.createPattern

<html>
<head>

window.onload=(function(){

    // canvas related variables
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");
    ctx.lineWidth=15;

    // stroke using a CSS color: named, RGB, HSL, etc
    ctx.strokeStyle='red';
    drawLine(50,40,250,40);

    // stroke using a linear gradient
    var gradient = ctx.createLinearGradient(75,75,175,75);
    gradient.addColorStop(0,'red');

shadowColor, shadowBlur, shadowOffsetX, shadowOffsetY (atributos de estilo de caminho) #

shadowColor = color        // CSS color
shadowBlur =  width        // integer blur width
shadowOffsetX = distance   // shadow is moved horizontally by this offset
shadowOffsetY = distance   // shadow is moved vertically by this offset

Este conjunto de atributos adicionará uma sombra ao redor de um caminho.

Ambos os caminhos preenchidos e traçados podem ter uma sombra.

A sombra é mais escura (opaca) no perímetro do caminho e torna-se gradualmente mais clara à medida que se afasta do perímetro do caminho.

  • shadowColor indica qual cor CSS será usada para criar a sombra.
  • shadowBlur é a distância na qual a sombra se estende para fora do caminho.
  • shadowOffsetX é a distância pela qual a sombra é deslocada horizontalmente para longe do caminho. Uma distância positiva move a sombra para a direita, uma distância negativa move a sombra para a esquerda.
  • shadowOffsetY é a distância pela qual a sombra é deslocada verticalmente para longe do caminho. Uma distância positiva move a sombra para baixo, uma distância negativa move a sombra para cima.

Sobre shadowOffsetX e shadowOffsetY

É importante notar que toda a sombra é deslocada em sua totalidade. Isso fará com que parte da sombra se desloque por baixo dos caminhos preenchidos e, portanto, parte da sombra não será visível.

Sobre traços sombreados

Ao sombrear um traçado, tanto a parte interna quanto a externa do traçado são sombreadas. A sombra é mais escura no traço e clareia à medida que a sombra se estende para fora em ambas as direções do traço.

Desativando o sombreamento quando terminar

Depois de desenhar suas sombras, você pode desativar o sombreamento para desenhar mais caminhos. Para desativar o sombreamento, defina o shadowColor como transparente.

context.shadowColor = 'rgba(0,0,0,0)';

Considerações de desempenho

Sombras (como gradientes) requerem cálculos extensos e, portanto, você deve usar sombras com moderação.

Seja especialmente cauteloso ao animar porque desenhar sombras muitas vezes por segundo afetará muito o desempenho. Uma solução alternativa se você precisar animar caminhos sombreados é pré-criar o caminho sombreado em uma segunda “tela de sombra”. O shadow-canvas é um canvas normal que é criado na memória com document.createElement – ele não é adicionado ao DOM (é apenas um canvas de teste). Em seguida, desenhe a tela de sombra na tela principal. Isso é muito mais rápido porque os cálculos de sombra não precisam ser feitos muitas vezes por segundo. Tudo o que você está fazendo é copiar uma tela pré-criada em sua tela visível.

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }
    #canvas{border:1px solid red; }
</style>
<script>
window.onload=(function(){

    // canvas related variables
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");

    // shadowed stroke
    ctx.shadowColor='black';
    ctx.shadowBlur=6;
    ctx.strokeStyle='red';
    ctx.strokeRect(50,50,100,50);
    // darken the shadow by stroking a second time
    ctx.strokeRect(50,50,100,50);

    // shadowed fill
    ctx.shadowColor='black';
    ctx.shadowBlur=10;
    ctx.fillStyle='red';
    ctx.fillRect(225,50,100,50);
    // darken the shadow by stroking a second time
    ctx.fillRect(225,50,100,50);

    // the shadow offset rightward and downward 
    ctx.shadowColor='black';
    ctx.shadowBlur=10;
    ctx.shadowOffsetX=5;
    ctx.shadowOffsetY=5;
    ctx.fillStyle='red';
    ctx.fillRect(50,175,100,50);

    // a wider blur (==extends further from the path)
    ctx.shadowColor='black';
    ctx.shadowBlur=35;
    ctx.fillStyle='red';
    ctx.fillRect(225,175,100,50);

    // always clean up! Turn off shadowing
    ctx.shadowColor='rgba(0,0,0,0)';

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=400 height=300></canvas>
</body>
</html>

createLinearGradient (cria um objeto de estilo de caminho) #

var gradient = createLinearGradient( startX, startY, endX, endY )
gradient.addColorStop(gradientPercentPosition, CssColor)
gradient.addColorStop(gradientPercentPosition, CssColor)
[optionally add more color stops to add to the variety of the gradient]

Cria um gradiente linear reutilizável (objeto).

O objeto pode ser atribuído a qualquer strokeStyle e/ou fillStyle.

Então stroke() ou fill() irá colorir o Path com as cores gradientes do objeto.

A criação de um objeto de gradiente é um processo de 2 etapas:

  1. Crie o próprio objeto gradiente. Durante a criação, você define uma linha na tela onde o gradiente começará e terminará. O objeto gradiente é criado com var gradient = context.createLinearGradient.
  2. Em seguida, adicione 2 (ou mais) cores que compõem o gradiente. Isso é feito adicionando várias paradas de cor ao objeto gradiente com gradient.addColorStop.

Argumentos:

  • startX,startY é a coordenada da tela onde o gradiente começa. No ponto inicial (e antes) a tela é solidamente da cor do gradientPercentPosition mais baixo.

  • endX,endY é a coordenada da tela onde o gradiente termina. No ponto final (e depois) a tela é solidamente da cor do gradientPercentPosition mais alto.

  • gradientPercentPosition é um número flutuante entre 0,00 e 1,00 atribuído a uma parada de cor. É basicamente um waypoint de porcentagem ao longo da linha onde essa parada de cor específica se aplica.

  • O gradiente começa na porcentagem 0,00 que é [startX,startY] na tela.

  • O gradiente termina na porcentagem 1,00 que é [endX,endY] na tela.

  • Nota técnica: O termo “porcentagem” não é tecnicamente correto, pois os valores vão de 0,00 a 1,00 em vez de 0% a 100%.

  • CssColor é uma cor CSS atribuída a essa cor específica.

O objeto gradiente é um objeto que você pode usar (e reutilizar!) para fazer com que seus traçados e preenchimentos se tornem gradientes coloridos.

Nota Lateral: O objeto gradiente não é interno ao elemento Canvas nem é Contexto. É um objeto JavaScript separado e reutilizável que você pode atribuir a qualquer caminho que desejar. Você pode até usar este objeto para colorir um Path em um elemento Canvas diferente (!)

Color stops são pontos de passagem (porcentuais) ao longo da linha de gradiente. Em cada waypoint de parada de cor, o gradiente é totalmente (==opacamente) colorido com sua cor atribuída. Os pontos intermediários ao longo da linha de gradiente entre as paradas de cor são coloridos como gradientes desta cor e da cor anterior.

Dica importante sobre os gradientes do Canvas!

Quando você cria um objeto de gradiente, a tela inteira é “invisivelmente” preenchida com esse gradiente.

Quando você stroke() ou fill() um caminho, o gradiente invisível é revelado, mas apenas revelado sobre o caminho que está sendo traçado ou preenchido.

  1. Se você criar um gradiente linear de vermelho para magenta como este:

     // create a linearGradient
 var gradient=ctx.createLinearGradient(100,0,canvas.width-100,0);
 gradient.addColorStop(0,'red');
 gradient.addColorStop(1,'magenta');
 ctx.fillStyle=gradient;

  2. Em seguida, o Canvas verá “invisivelmente” sua criação de gradiente assim:

  1. Mas até você stroke() ou fill() com o gradiente, você não verá nenhum gradiente no Canvas.

  2. Finalmente, se você traçar ou preencher um caminho usando o gradiente, o gradiente “invisível” se tornará visível na Tela… mas somente onde o caminho for desenhado.

createRadialGradient (cria um objeto de estilo de caminho) #

var gradient = createRadialGradient( 
      centerX1, centerY1, radius1,     // this is the "display' circle
      centerX2, centerY2, radius2      // this is the "light casting" circle
)
gradient.addColorStop(gradientPercentPosition, CssColor)
gradient.addColorStop(gradientPercentPosition, CssColor)
[optionally add more color stops to add to the variety of the gradient]

Cria um gradiente radial reutilizável (objeto). O objeto gradiente é um objeto que você pode usar (e reutilizar!) para fazer com que seus traçados e preenchimentos se tornem gradientes coloridos.

Sobre…

O gradiente radial do Canvas é extremamente diferente dos gradientes radiais tradicionais.

A definição “oficial” (quase indecifrável!) do gradiente radial do Canvas está na parte inferior desta postagem. Não olhe se você tem uma disposição fraca!!

Em termos (quase compreensíveis):

  • O gradiente radial tem 2 círculos: um círculo de “casting” e um círculo de “display”.
  • O círculo de fundição lança luz no círculo de exibição.
  • Essa luz é o gradiente.
  • A forma dessa luz gradiente é determinada pelo tamanho e posição relativos de ambos os círculos.

A criação de um objeto de gradiente é um processo de 2 etapas:

  1. Crie o próprio objeto gradiente. Durante a criação, você define uma linha na tela onde o gradiente começará e terminará. O objeto gradiente é criado com var gradient = context.radialLinearGradient.
  2. Em seguida, adicione 2 (ou mais) cores que compõem o gradiente. Isso é feito adicionando várias paradas de cor ao objeto gradiente com gradient.addColorStop.

Argumentos:

  • centerX1,centerY1,radius1 define um primeiro círculo onde o gradiente será exibido.

  • centerX2,centerY2,radius2 define um segundo círculo que está lançando luz de gradiente no primeiro círculo.

  • gradientPercentPosition é um número flutuante entre 0,00 e 1,00 atribuído a uma parada de cor. É basicamente um waypoint de porcentagem que define onde essa parada de cor específica se aplica ao longo do gradiente.

  • O gradiente começa na porcentagem 0,00.

  • O gradiente termina na porcentagem 1,00.

  • Nota técnica: O termo “porcentagem” não é tecnicamente correto, pois os valores vão de 0,00 a 1,00 em vez de 0% a 100%.

  • CssColor é uma cor CSS atribuída a essa cor específica.

Nota Lateral: O objeto gradiente não é interno ao elemento Canvas nem é Contexto. É um objeto JavaScript separado e reutilizável que você pode atribuir a qualquer caminho que desejar. Você pode até usar este objeto para colorir um Path em um elemento Canvas diferente (!)

Color stops são pontos de passagem (porcentuais) ao longo da linha de gradiente. Em cada waypoint de parada de cor, o gradiente é totalmente (==opacamente) colorido com sua cor atribuída. Os pontos intermediários ao longo da linha de gradiente entre as paradas de cor são coloridos como gradientes desta cor e da cor anterior.

Dica importante sobre os gradientes do Canvas!

Quando você cria um objeto de gradiente, todo o gradiente radial é “invisível” lançado na tela.

Quando você stroke() ou fill() um caminho, o gradiente invisível é revelado, mas apenas revelado sobre o caminho que está sendo traçado ou preenchido.

  1. Se você criar um gradiente radial verde para vermelho como este:

     // create a radialGradient
 var x1=150;
 var y1=150;
 var x2=280;
 var y2=150;
 var r1=100;
 var r2=120;
 var gradient=ctx.createRadialGradient(x1,y1,r1,x2,y2,r2);
 gradient.addColorStop(0,'red');
 gradient.addColorStop(1,'green');
 ctx.fillStyle=gradient;

  2. Em seguida, o Canvas verá “invisivelmente” sua criação de gradiente assim:

  1. Mas até você stroke() ou fill() com o gradiente, você não verá nenhum gradiente no Canvas.

  2. Finalmente, se você traçar ou preencher um caminho usando o gradiente, o gradiente “invisível” se tornará visível na Tela… mas somente onde o caminho for desenhado.

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; padding:10px; }
    #canvas{border:1px solid blue; }
</style>
<script>
window.onload=(function(){

    // canvas related vars
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");
    
    // create a radial gradient
    var x1=150;
    var y1=175;
    var x2=350;
    var y2=175;
    var r1=100;
    var r2=40;
    x2=x1;
    var gradient=ctx.createRadialGradient(x1,y1,r1,x2,y2,r2);
    gradient.addColorStop(0,'red');
    gradient.addColorStop(1,'green');
    ctx.fillStyle=gradient;

    // fill a path with the gradient
    ctx.beginPath();
    ctx.moveTo(150,0);
    ctx.lineTo(300,150);
    ctx.lineTo(150,325);
    ctx.lineTo(0,150);
    ctx.lineTo(150,0);
    ctx.fill();

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=300 height=325></canvas>
</body>
</html>

Os detalhes oficiais assustadores #

Quem decide o que `createRadialGradient faz?

O W3C emite as especificações oficiais recomendadas que os navegadores usam para construir o elemento Html5 Canvas.

A especificação do W3C para createRadialGradient é assim:

O que createRadialGradient cria

createRadialGradient … efetivamente cria um cone, tocado pelo dois círculos definidos na criação do gradiente, com a parte de o cone antes do círculo inicial (0,0) usando a cor do primeiro offset, a parte do cone após o círculo final (1.0) usando o cor do último deslocamento e áreas fora do cone intocadas pelo gradiente (preto transparente).

Como funciona internamente

O método createRadialGradient(x0, y0, r0, x1, y1, r1) leva seis argumentos, os três primeiros representando o círculo inicial com origem (x0, y0) e raio r0, e os três últimos representando o círculo final com origem (x1, y1) e raio r1. Os valores estão no espaço de coordenadas unidades. Se r0 ou r1 for negativo, uma exceção IndexSizeError deve ser lançado. Caso contrário, o método deve retornar um radial CanvasGradient inicializado com os dois círculos especificados.

Os gradientes radiais devem ser renderizados seguindo estas etapas:

  1. Se x0 = x1 e y0 = y1 e r0 = r1, então o gradiente radial não deve pintar nada. Aborte essas etapas.
  2. Seja x(ω) = (x1-x0)ω + x0; Seja y(ω) = (y1-y0)ω + y0; Seja r(ω) = (r1-r0)ω + r0 Seja a cor em ω a cor naquela posição no gradiente (com as cores provenientes da interpolação e extrapolação descrita acima).
  3. Para todos os valores de ω onde r(ω) > 0, começando com o valor de ω mais próximo do infinito positivo e terminando com o valor de ω mais próximo de infinito negativo, desenhe a circunferência do círculo com raio r(ω) na posição (x(ω), y(ω)), com a cor em ω, mas apenas pintando nas partes da tela que ainda não foram pintadas por círculos anteriores nesta etapa para esta renderização do gradiente.

createPattern (cria um objeto de estilo de caminho) #

var pattern = createPattern(imageObject,repeat)

Cria um padrão reutilizável (objeto).

O objeto pode ser atribuído a qualquer strokeStyle e/ou fillStyle.

Então stroke() ou fill() irá pintar o Path com o padrão do objeto.

Argumentos:

  • imageObject é uma imagem que será usada como padrão. A fonte da imagem pode ser:

  • HTMLImageElement — um elemento img ou um novo Image(),

  • HTMLCanvasElement — um elemento de tela,

  • HTMLVideoElement — um elemento de vídeo (capturará o quadro de vídeo atual)

  • ImagemBitmap, *Bolha.

  • repeat determina como o imageObject será repetido na tela (muito parecido com um plano de fundo CSS). Este argumento deve ser delimitado por aspas e os valores válidos são:

  • “repetir” — o padrão preencherá horizontalmente e verticalmente a tela

  • “repeat-x” — o padrão só se repetirá horizontalmente (1 linha horizontal)

  • “repeat-y” — o padrão só se repetirá verticalmente (1 linha vertical)

  • “repetir nenhum” — o padrão aparece apenas uma vez (no canto superior esquerdo)

O objeto de padrão é um objeto que você pode usar (e reutilizar!) para tornar seus traçados e preenchimentos de caminho padronizados.

Nota Lateral: O objeto pattern não é interno ao elemento Canvas nem é Context. É um objeto JavaScript separado e reutilizável que você pode atribuir a qualquer caminho que desejar. Você pode até usar este objeto para aplicar um padrão a um Path em um elemento Canvas diferente (!)

Dica importante sobre os padrões Canvas!

Quando você cria um objeto de padrão, a tela inteira é “invisivelmente” preenchida com esse padrão (sujeito ao argumento repeat).

Quando você stroke() ou fill() um caminho, o padrão invisível é revelado, mas apenas revelado sobre o caminho que está sendo traçado ou preenchido.

  1. Comece com uma imagem que deseja usar como padrão. Importante(!): Certifique-se de que sua imagem foi totalmente carregada (usando patternimage.onload) antes de tentar usá-la para criar seu padrão.

  1. Você cria um padrão como este:

     // create a pattern
 var pattern = ctx.createPattern(patternImage,'repeat');
 ctx.fillStyle=pattern;

  2. Em seguida, o Canvas verá “invisivelmente” sua criação de padrão assim:

  1. Mas até você stroke() ou fill() com o padrão, você não verá nenhum padrão no Canvas.

  2. Finalmente, se você traçar ou preencher um caminho usando o padrão, o padrão “invisível” se tornará visível na Tela… mas somente onde o caminho for desenhado.

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }
    #canvas{border:1px solid red; }
</style>
<script>
window.onload=(function(){

    // canvas related variables
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");

    // fill using a pattern
    var patternImage=new Image();
    // IMPORTANT! 
    // Always use .onload to be sure the image has
    //     fully loaded before using it in .createPattern
    patternImage.onload=function(){
        // create a pattern object
        var pattern = ctx.createPattern(patternImage,'repeat');
        // set the fillstyle to that pattern
        ctx.fillStyle=pattern;
        // fill a rectangle with the pattern
        ctx.fillRect(50,50,150,100);
        // demo only, stroke the rect for clarity
        ctx.strokeRect(50,50,150,100);
    }
    patternImage.src='http://i.stack.imgur.com/K9EZl.png';

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=325 height=250></canvas>
</body>
</html>

traço (um comando de caminho) #

context.stroke()

Faz com que o perímetro do Path seja traçado de acordo com o context.strokeStyle atual e o Path traçado é desenhado visualmente na tela.

Antes de executar context.stroke (ou context.fill) o Caminho existe na memória e ainda não foi desenhado visualmente na tela.

A forma incomum como os traços são desenhados #

Considere este exemplo Path que desenha uma linha preta de 1 pixel de [0,5] ​​para [5,5]:

// draw a 1 pixel black line from [0,5] to [5,5]
context.strokeStyle='black';
context.lineWidth=1;
context.beginPath();
context.moveTo(0,5);
context.lineTo(5,5);
context.stroke();

Pergunta: O que o navegador realmente desenha na tela?

Você provavelmente espera obter 6 pixels pretos em y=5

Mas(!) … O Canvas sempre desenha traços a meio caminho de cada lado do caminho definido!

Portanto, como a linha é definida em y==5.0, o Canvas deseja desenhar a linha entre y==4.5 e y==5.5

Mas, novamente(!)… A tela do computador não pode desenhar meio pixel!

Então, o que deve ser feito com os meio-pixels indesejados (mostrados em azul abaixo)?

A resposta é que o Canvas na verdade ordena que a tela desenhe uma linha de 2 pixels de largura de 4.0 a 6.0. Também colore a linha mais clara do que o ‘preto’ definido. Esse comportamento estranho de desenho é “anti-aliasing” e ajuda o Canvas a evitar traços de desenho que parecem irregulares.

Um truque de ajuste que funciona APENAS para traços exatamente horizontais e verticais

Você pode obter uma linha preta sólida de 1 pixel especificando a linha a ser desenhada no meio pixel:

context.moveTo(0,5.5);
context.lineto(5,5.5);

Código de exemplo usando context.stroke() para desenhar um caminho traçado na tela:

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }
    #canvas{border:1px solid red; }
</style>
<script>
window.onload=(function(){

    // canvas related variables
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");

    ctx.beginPath();
    ctx.moveTo(50,30);
    ctx.lineTo(75,55);
    ctx.lineTo(25,55);
    ctx.lineTo(50,30);
    ctx.lineWidth=2;
    ctx.stroke();

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=100 height=100></canvas>
</body>
</html>

clipe (um comando de caminho) #

context.clip

Limita quaisquer desenhos futuros para serem exibidos apenas dentro do Caminho atual.

Exemplo: Recorte esta imagem em um caminho triangular

<!doctype html>
<html>
<head>
<style>
    body{ background-color:white; }
    #canvas{border:1px solid red; }
</style>
<script>
window.onload=(function(){

    // canvas related variables
    var canvas=document.getElementById("canvas");
    var ctx=canvas.getContext("2d");

    var img=new Image();
    img.onload=start;
    img.src='http://i.stack.imgur.com/1CqWf.jpg'

    function start(){
        // draw a triangle path
        ctx.beginPath();
        ctx.moveTo(75,50);
        ctx.lineTo(125,100);
        ctx.lineTo(25,100);
        ctx.lineTo(75,50);
        
        // clip future drawings to appear only in the triangle
        ctx.clip();
        
        // draw an image
        ctx.drawImage(img,0,0);
    }

}); // end window.onload
</script>
</head>
<body>
    <canvas id="canvas" width=150 height=150></canvas>
</body>
</html>
