Este tópico aborda os vários tipos de mídia e como eles podem ser usados ​​com a tela na interface 2D.

Os tipos de mídia têm categorias genéricas e específicas de formato

Tipos de mídia

  • Animações
  • Vídeos
  • Imagens
  • imagens em alta definição
  • Imagem vetorial

  • PNG

Imagens

Há uma grande variedade de formatos de imagem suportados pelos navegadores, embora nenhum navegador suporte todos eles. Se você tiver formatos de imagem específicos, você deseja usar Wiki Browsers e formatos de imagem suportados fornece uma boa visão geral.

O melhor suporte é para os 3 formatos principais, “jpeg”, “png” e “gif”, com todos os principais navegadores fornecendo suporte.

JPEG

As imagens JPEG são mais adequadas para fotos e imagens semelhantes a fotos. Eles não se adaptam bem a gráficos, diagramas e textos. As imagens JPEG não suportam transparência.

O Canvas pode gerar imagens JPEG via canvas.toDataURL e canvas.toBlob e fornece uma configuração de qualidade. Como o JPEG não suporta transparência, quaisquer pixels transparentes serão misturados com preto para o JPG de saída final. A imagem resultante não será uma cópia perfeita da tela.

JPEG na wikipedia

PNG

PNG Image são as imagens da mais alta qualidade e também podem incluir um canal alfa para pixels transparentes. Os dados da imagem são compactados, mas não produzem artefatos como imagens JPG.

Por causa da compactação sem perdas e do suporte ao canal alfa, os PNGs são usados ​​para jogos, imagens de componentes de interface do usuário, gráficos, diagramas, texto. Ao usá-los para fotos e imagens semelhantes a fotos, o tamanho do arquivo pode ser muito maior que o do JPEG. .

O formato PNG também oferece suporte à animação, embora o suporte ao navegador seja limitado, e o acesso à animação para uso na tela só pode ser feito por meio de APIs e bibliotecas Javascript

A tela pode ser usada para codificar imagens PNG via canvas.toDataURL e canvas.toBlob embora o formato de saída seja limitado a RGBA de 32 bits compactado. O PNG fornecerá uma cópia perfeita em pixels da tela.

PNG na wikipedia

GIF

Os GIFs são usados ​​para animações curtas, mas também podem ser usados ​​para fornecer gráficos, diagramas e imagens de texto de alta qualidade. Os GIFs têm suporte a cores muito limitado, com um máximo de 256 cores por quadro. Com o processamento de imagens do cutelo, as imagens gif podem produzir resultados surpreendentemente bons, especialmente quando animadas. Gifs também fornecem transparência, embora isso seja limitado a ligado ou desligado

Assim como o PNG, as animações GIF não são diretamente acessíveis para uso na tela e você precisará de uma API ou biblioteca Javascript para obter acesso. O GIF não pode ser salvo por meio da tela e exigirá uma API ou biblioteca para fazê-lo.

Carregamento básico e reprodução de um vídeo na tela. #

A tela pode ser usada para exibir vídeo de várias fontes. Este exemplo mostra como carregar um vídeo como um recurso de arquivo, exibi-lo e adicionar um simples clique na alternância de reprodução/pausa da tela.

Esta pergunta auto-respondida do stackoverflow Como exibir um vídeo usando a tag de tela HTML5 mostra o código de exemplo a seguir em ação.

Apenas uma imagem #

Um vídeo é apenas uma imagem no que diz respeito à tela. Você pode desenhá-lo como qualquer imagem. A diferença é que o vídeo pode ser reproduzido e tem som.

Obtenha a tela e a configuração básica #

// It is assumed you know how to add a canvas and correctly size it.
var canvas = document.getElementById("myCanvas"); // get the canvas from the page
var ctx = canvas.getContext("2d");
var videoContainer; // object to hold video and associated info

Criando e carregando o vídeo #

var video = document.createElement("video"); // create a video element
video.src = "urlOffVideo.webm"; 
// the video will now begin to load.
// As some additional info is needed we will place the video in a
// containing object for convenience
video.autoPlay = false; // ensure that the video does not auto play
video.loop = true; // set the video to loop.
videoContainer = {  // we will add properties as needed
     video : video,
     ready : false,   
};

Ao contrário dos elementos de imagens, os vídeos não precisam ser totalmente carregados para serem exibidos na tela. Os vídeos também fornecem uma série de eventos extras que podem ser usados ​​para monitorar o status do vídeo.

Neste caso, queremos saber quando o vídeo está pronto para ser reproduzido. oncanplay significa que o suficiente do vídeo foi carregado para reproduzir parte dele, mas pode não haver o suficiente para reproduzir até o fim.

video.oncanplay = readyToPlayVideo; // set the event to the play function that 
                                  // can be found below

Alternativamente, você pode usar oncanplaythrough, que será acionado quando o vídeo for carregado o suficiente para que possa ser reproduzido até o final.

video.oncanplaythrough = readyToPlayVideo; // set the event to the play function that
                                         // can be found below

Use apenas um dos eventos canPlay e não ambos.

Exibindo #

O vídeo não será reproduzido sozinho na tela. Você precisa desenhá-lo para cada novo quadro. Como é difícil saber a taxa de quadros exata e quando eles ocorrem, a melhor abordagem é exibir o vídeo como se estivesse rodando a 60fps. Se a taxa de quadros for menor, w apenas renderizará o mesmo quadro duas vezes. Se a taxa de quadros for maior, não há nada que possa ser feito para ver os quadros extras, então apenas os ignoramos.

O elemento de vídeo é apenas um elemento de imagem e pode ser desenhado como qualquer imagem, você pode dimensionar, girar, deslocar o vídeo, espelhar, desvanecer, recortar e exibir apenas partes, desenhá-lo duas vezes pela segunda vez com um modo de composição global para adicionar FX como iluminar, tela, etc.

function updateCanvas(){
    ctx.clearRect(0,0,canvas.width,canvas.height); // Though not always needed 
                                                     // you may get bad pixels from 
                                                     // previous videos so clear to be
                                                     // safe
    // only draw if loaded and ready
    if(videoContainer !== undefined && videoContainer.ready){ 
        // find the top left of the video on the canvas
        var scale = videoContainer.scale;
        var vidH = videoContainer.video.videoHeight;
        var vidW = videoContainer.video.videoWidth;
        var top = canvas.height / 2 - (vidH /2 ) * scale;
        var left = canvas.width / 2 - (vidW /2 ) * scale;
        // now just draw the video the correct size
        ctx.drawImage(videoContainer.video, left, top, vidW * scale, vidH * scale);
        if(videoContainer.video.paused){ // if not playing show the paused screen 
            drawPayIcon();
        }
    }
    // all done for display 
    // request the next frame in 1/60th of a second
    requestAnimationFrame(updateCanvas);
}

Controle básico de pausa de reprodução #

Agora temos o vídeo carregado e exibido, tudo o que precisamos é o controle de reprodução. Vamos torná-lo como um clique de alternância de reprodução na tela. Quando o vídeo está sendo reproduzido e o usuário clica, o vídeo é pausado. Quando pausado, o clique retoma a reprodução. Adicionaremos uma função para escurecer o vídeo e desenhar um ícone de reprodução (triângulo)

function drawPayIcon(){
     ctx.fillStyle = "black";  // darken display
     ctx.globalAlpha = 0.5;
     ctx.fillRect(0,0,canvas.width,canvas.height);
     ctx.fillStyle = "#DDD"; // colour of play icon
     ctx.globalAlpha = 0.75; // partly transparent
     ctx.beginPath(); // create the path for the icon
     var size = (canvas.height / 2) * 0.5;  // the size of the icon
     ctx.moveTo(canvas.width/2 + size/2, canvas.height / 2); // start at the pointy end
     ctx.lineTo(canvas.width/2 - size/2, canvas.height / 2 + size);
     ctx.lineTo(canvas.width/2 - size/2, canvas.height / 2 - size);
     ctx.closePath();
     ctx.fill();
     ctx.globalAlpha = 1; // restore alpha
}

Agora o evento de pausa de reprodução #

function playPauseClick(){
     if(videoContainer !== undefined && videoContainer.ready){
          if(videoContainer.video.paused){                                 
                videoContainer.video.play();
          }else{
                videoContainer.video.pause();
          }
     }
}
// register the event
canvas.addEventListener("click",playPauseClick);

Resumo #

Reproduzir um vídeo é muito fácil usando a tela, adicionar efeito em tempo real também é fácil. No entanto, existem algumas limitações nos formatos, como você pode jogar e buscar. MDN HTMLMediaElement é o local para obter a referência completa ao objeto de vídeo.

Uma vez que a imagem foi desenhada na tela você pode usar ctx.getImageData para acessar os pixels que ela contém. Ou você pode usar canvas.toDataURL para tirar uma foto e baixá-la. (Somente se o vídeo for de uma fonte confiável e não manchar a tela).

Observe que se o vídeo tiver som, reproduzi-lo também reproduzirá o som.

Criando uma imagem

Existem várias maneiras de criar uma imagem

  • nova imagem()
  • document.createElement("img")
  • <img src = 'imageUrl' id='myImage'> Como parte do corpo HTML e recuperado com document.getElementById('myImage')

A imagem é um HTMLImageElement

Propriedade Image.src

A imagem src pode ser qualquer URL de imagem válida ou URL de dados codificado. Consulte os Comentários deste tópico para obter mais informações sobre formatos de imagem e suporte.

  • image.src = "http://my.domain.com/images/myImage.jpg"
  • image.src = "data:image/gif;base64,R0lGODlhAQABAIAAAAUEBAAAACwAAAAAAQABAACAkQBADs=" *

*O dataURL é uma imagem gif de 1 por 1 pixel contendo preto

Observações sobre carregamento e erros

A imagem começará a carregar quando sua propriedade src for definida. O carregamento é sincronizado, mas o evento onload não será chamado até que a função ou código seja encerrado/retornado.

Se você obtiver uma imagem da página (por exemplo document.getElementById("myImage")) e seu src estiver definido, ela pode ou não ter sido carregada. Você pode verificar o status da imagem com HTMLImageElement.complete que será true se completo. Isso não significa que a imagem foi carregada, significa que ela

  • carregado
  • Havia um erro
  • a propriedade src não foi definida e é igual à String vazia ""

Se a imagem for de uma fonte não confiável e não puder ser acessada por vários motivos, ela gerará um evento de erro. Quando isso acontecer, a imagem estará em um estado quebrado. Se você tentar desenhá-lo na tela, ele lançará o seguinte erro

Uncaught DOMException: Failed to execute 'drawImage' on 'CanvasRenderingContext2D': The HTMLImageElement provided is in the 'broken' state.

Ao fornecer o evento image.onerror = myImgErrorHandler você pode tomar as medidas apropriadas para evitar erros.

## Desenhando uma imagem svg Para desenhar uma imagem SVG vetorial, a operação não é diferente de uma imagem raster: Você primeiro precisa carregar sua imagem SVG em um elemento HTMLImage, então use o método `drawImage()`. var image = new Image(); image.onload = function(){ ctx.drawImage(this, 0,0); } image.src = "someFile.SVG"; As imagens SVG têm algumas vantagens sobre as raster, pois você não perderá qualidade, seja qual for a escala que você desenhar na tela. Mas cuidado, também pode ser um pouco mais lento do que desenhar uma imagem raster. No entanto, as imagens SVG vêm com mais restrições do que as imagens raster. - **Por motivos de segurança, nenhum conteúdo externo pode ser carregado de uma imagem SVG referenciada em um HTMLImageElement(``)** Nenhuma folha de estilo externa, nenhuma imagem externa referenciada em elementos SVGImage (``), nenhum filtro externo ou elemento vinculado pelo atributo `xlink:href` (``) ou o método de atributo funcIRI (`url()`) etc. Além disso, as folhas de estilo anexadas no documento principal não terão nenhum efeito no documento SVG uma vez referenciadas em um elemento HTMLImage. Finalmente, nenhum script será executado dentro da imagem SVG. **Solução alternativa:** você precisará anexar todos os elementos externos dentro do próprio SVG antes de fazer referência ao elemento HTMLImage.
