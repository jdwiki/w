Corte de imagem usando canvas #

Este exemplo mostra uma função de corte de imagem simples que obtém uma imagem e coordenadas de corte e retorna a imagem cortada.

function cropImage(image, croppingCoords) {
    var cc = croppingCoords;
    var workCan = document.createElement("canvas"); // create a canvas
    workCan.width = Math.floor(cc.width);  // set the canvas resolution to the cropped image size
    workCan.height = Math.floor(cc.height);
    var ctx = workCan.getContext("2d");    // get a 2D rendering interface
    ctx.drawImage(image, -Math.floor(cc.x), -Math.floor(cc.y)); // draw the image offset to place it correctly on the cropped region
    image.src = workCan.toDataURL();       // set the image source to the canvas as a data URL
    return image;
}

Usar

var image = new Image();
image.src = "image URL"; // load the image
image.onload = function () {  // when loaded
    cropImage(
        this, {
        x : this.width / 4,     // crop keeping the center
        y : this.height / 4,
        width : this.width / 2,
        height : this.height / 2,
    });
    document.body.appendChild(this);  // Add the image to the DOM
};

A tela Tained #

Ao adicionar conteúdo de fontes fora do seu domínio ou do sistema de arquivos local, a tela é marcada como contaminada. A tentativa de acessar os dados do pixel ou converter para um dataURL gerará um erro de segurança.

vr image = new Image();
image.src = "file://myLocalImage.png";
image.onload = function(){
    ctx.drawImage(this,0,0);
    ctx.getImageData(0,0,canvas.width,canvas.height);  // throws a security error
}

Este exemplo é apenas um esboço para atrair alguém com uma compreensão detalhada elaborada.

O “context.drawImage” não está exibindo a imagem no Canvas? #

Certifique-se de que seu objeto de imagem esteja totalmente carregado antes de tentar desenhá-lo na tela com context.drawImage. Caso contrário, a imagem não será exibida silenciosamente.

Em JavaScript, as imagens não são carregadas imediatamente. Em vez disso, as imagens são carregadas de forma assíncrona e durante o tempo que levam para carregar o JavaScript continua executando qualquer código que siga image.src. Isso significa que context.drawImage pode ser executado com uma imagem vazia e, portanto, não exibirá nada.

Exemplo certificando-se de que a imagem está totalmente carregada antes de tentar desenhá-la com .drawImage

var img=new Image();
img.onload=start;
img.onerror=function(){alert(img.src+' failed');} 
img.src="someImage.png";
function start(){
    // start() is called AFTER the image is fully loaded regardless 
    //     of start's position in the code
}

Exemplo carregando várias imagens antes de tentar desenhar com qualquer uma delas

Existem mais carregadores de imagem com funções completas, mas este exemplo ilustra como fazer isso

// first image
var img1=new Image();
img1.onload=start;
img1.onerror=function(){alert(img1.src+' failed to load.');};
img1.src="imageOne.png";
// second image
var img2=new Image();
img2.onload=start;
img1.onerror=function(){alert(img2.src+' failed to load.');};
img2.src="imageTwo.png";
//
var imgCount=2;
// start is called every time an image loads
function start(){
    // countdown until all images are loaded
    if(--imgCount>0){return;}
    // All the images are now successfully loaded
    // context.drawImage will successfully draw each one
    context.drawImage(img1,0,0);
    context.drawImage(img2,50,0);
}

Dimensionamento da imagem para caber ou preencher. #

Escalando para caber

Significa que toda a imagem ficará visível, mas pode haver algum espaço vazio nas laterais ou na parte superior e inferior se a imagem não tiver o mesmo aspecto da tela. O exemplo mostra a imagem dimensionada para caber. O azul nas laterais se deve ao fato da imagem não ter o mesmo aspecto da tela.

Escala para preenchimento

Significa que a imagem é dimensionada para que todos os pixels da tela sejam cobertos pela imagem. Se o aspecto da imagem não for o mesmo da tela, algumas partes da imagem serão cortadas. O exemplo mostra a imagem dimensionada para preencher. Observe como a parte superior e inferior da imagem não são mais visíveis.

Exemplo de escala para caber #

var image = new Image();
image.src = "imgURL";
image.onload = function(){
    scaleToFit(this);
}

function scaleToFit(img){
    // get the scale
    var scale = Math.min(canvas.width / img.width, canvas.height / img.height);
    // get the top left position of the image
    var x = (canvas.width / 2) - (img.width / 2) * scale;
    var y = (canvas.height / 2) - (img.height / 2) * scale;
    ctx.drawImage(img, x, y, img.width * scale, img.height * scale);
}

Exemplo de escala para preencher #

var image = new Image();
image.src = "imgURL";
image.onload = function(){
    scaleToFill(this);
}

function scaleToFill(img){
    // get the scale
    var scale = Math.max(canvas.width / img.width, canvas.height / img.height);
    // get the top left position of the image
    var x = (canvas.width / 2) - (img.width / 2) * scale;
    var y = (canvas.height / 2) - (img.height / 2) * scale;
    ctx.drawImage(img, x, y, img.width * scale, img.height * scale);
}

A única diferença entre as duas funções é obter a escala. O ajuste usa a escala de ajuste mínimo e o preenchimento usa a escala de ajuste máximo.
