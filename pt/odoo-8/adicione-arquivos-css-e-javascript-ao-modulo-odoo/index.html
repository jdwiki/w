Adicione arquivos CSS e Javascript ao módulo Odoo

Sintaxe #

  • Nota sobre a sintaxe XML: Como o registro é feito dentro do arquivo XML, você não pode deixar nenhuma tag aberta como faria em um HTML simples, como: <link rel=‘stylesheet’ href="…" ** >**, feche a tag do link, como:
  • <link rel=‘stylesheet’ href="…" />

Parâmetros #

Valores possíveis do parâmetro inherit_id significado
web.assets_backend Usado apenas em páginas internas, NÃO incluído em um site público.
website.assets_frontend Usado apenas em um site público (através do módulo “website”).
web.assets_common Usado em sites públicos e páginas internas.

Se você não tiver certeza sobre qual opção é adequada para você, tente a primeira opção (backend), pois é usada na maioria dos casos e em quase todos os casos, se você não instalou o módulo “site”. Odoo diferencia entre ativos “backend” e “frontend” porque o site público fornecido pelo módulo “website” usa estilo e código JS diferentes das páginas internas destinadas a serem usadas para tarefas de ERP, ou seja, “frontend” está associado ao site público e “backend” está associado a páginas internas para ERP (o significado de “frontend” e “backend” são específicos do Odoo aqui, mas ambos são “frontend” em um sentido mais geral).

Você pode não apenas escolher e usar uma das opções, mas também usar qualquer combinação delas (duas ou todas) no mesmo módulo. Fatore um backend, um frontend e um código JS/CSS comum em arquivos separados para melhor aderir ao DRY e ter código adequado no site público e nas páginas internas.

Não se esqueça de adicionar “web” (ao usar opção 1) ou “website” (ao usar opção 2) à lista de dependências no manifesto __openerp__.py.

Armazene arquivos CSS e JS corretamente no módulo Odoo #

Os arquivos CSS e JS devem residir no diretório ‘static’ no diretório raiz do módulo (o restante da árvore de subdiretórios em ‘static’ é uma convenção opcional):

  • static/src/css/seu_arquivo.css
  • static/src/js/seu_arquivo.js

Em seguida, adicione links para esses arquivos usando uma das 3 maneiras listadas nos exemplos a seguir.

Opção 1: [BACKEND] Adicione arquivos CSS e Javascript para usar em páginas internas #

A maneira do Odoo v8.0 é adicionar o registro correspondente no arquivo XML:

  • ​Adicione o arquivo XML ao manifesto (ou seja, arquivo __openerp__.py.):

    …
    ‘data’ : [ ‘your_file.xml’ ],
    ​…

  • Em seguida, adicione o seguinte registro em 'your_file.xml': <pré> <openerp> <data>
    <template id=“assets_backend” name="your_module_name assets" inherit_id=“web.assets_backend”>
    <xpath expr="." position=“inside”>
    <link rel=‘stylesheet’ href="/your_module_name/static/src/css/your_file.css"/>
    <script type=“text/javascript” src="/your_module_name/static/src/js/your_file.js"></script>
    </xpath>
    </template>
    ….
    ….
    </data>
    </openerp>

Opção 2: [FRONTEND] Adicione arquivos CSS e Javascript para usar em um site público #

Nota: você deve usar esta forma se tiver instalado um módulo “site” e tiver um site público disponível.

  • Adicione o seguinte registro em 'your_file.xml': <pré> <openerp> <data>

          &lt;template <i>id="assets_frontend"</i> name="<strong>your_module_name</strong> assets" <i>inherit_id="website.assets_frontend"</i>&gt;  
          &lt;xpath expr="link[last()]" position="after"&gt;  
              &lt;link rel='stylesheet' href="/<strong>your_module_name</strong>/static/src/css/<strong>your_file.css</strong>"/&gt;  
          &lt;/xpath&gt;  
          &lt;xpath expr="script[last()]" position="after"&gt;  
              &lt;script type="text/javascript" src="/<strong>your_module_name</strong>/static/src/js/<strong>your_file.js</strong>"&gt;&lt;/script&gt;  
          &lt;/xpath&gt;  
      &lt;/template&gt;  

  &lt;/data&gt;

    </openerp>

Opção 3: [COMMON] Adicione arquivos CSS e Javascript para usar em todas as páginas (backend e frontend) #

  • Adicione o seguinte registro em 'your_file.xml': <pré> <openerp> <data>

          &lt;template id="assets_common" name="<strong>your_module_name</strong> assets" inherit_id="web.assets_common"&gt;  
          &lt;xpath expr="." position="inside"&gt;  
              &lt;link rel='stylesheet' href="/<strong>your_module_name</strong>/static/src/css/<strong>your_file.css</strong>"/&gt;  
              &lt;script type="text/javascript" src="/<strong>your_module_name</strong>/static/src/js/<strong>your_file.js</strong>"&gt;&lt;/script&gt;  
          &lt;/xpath&gt;  
      &lt;/template&gt;  

  &lt;/data&gt;

    </openerp>
