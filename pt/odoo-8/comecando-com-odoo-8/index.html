O que é Odoo? #

Odoo (anteriormente conhecido como OpenERP e antes disso, TinyERP) é um conjunto de aplicativos de gerenciamento empresarial de núcleo aberto. Visando empresas de todos os portes, o conjunto de aplicativos cobre todas as necessidades de negócios, desde site/comércio eletrônico até fabricação, estoque e contabilidade, tudo perfeitamente integrado. É a primeira vez que um editor de software consegue alcançar uma cobertura tão funcional. Odoo é o software empresarial mais instalado no mundo. Odoo é usado por mais de 2.000.000 de usuários em todo o mundo, desde empresas muito pequenas (1 usuário) até empresas muito grandes (300.000 usuários).

O código-fonte para a estrutura OpenObject e os módulos principais do ERP (planejamento de recursos empresariais) são selecionados pela Odoo S.A., com sede na Bélgica. Além disso, programação personalizada, suporte e outros serviços são fornecidos por uma comunidade global ativa e uma rede de 500 parceiros oficiais. Os principais componentes do Odoo são o framework OpenObject, cerca de 30 módulos principais (também chamados de módulos oficiais) e mais de 3000 módulos da comunidade

Odoo tem sido usado como um componente de cursos universitários. Um estudo sobre aprendizagem experiencial sugeriu que o OpenERP fornece uma alternativa adequada aos sistemas proprietários para complementar o ensino.

Vários livros foram escritos sobre Odoo, alguns cobrindo áreas específicas, como contabilidade ou desenvolvimento

Odoo recebeu prêmios, incluindo Trends Gazelle e BOSSIE Awards por três anos consecutivos.

Ele usa scripts Python e PostgreSQL como banco de dados. Sua edição da comunidade é complementada com uma edição Enterprise @ USD 240/- por usuário por ano e uma edição online com suporte comercial. O repositório de desenvolvimento está no GitHub.

Em 2013, a Odoo Community Association, sem fins lucrativos, foi formada para garantir a promoção e manutenção contínua das versões e módulos da comunidade Odoo para complementar o trabalho da Odoo S.A. Esta organização tem mais de 150 membros que são uma mistura de indivíduos e organizações.

Configurar #

Odoo pode ser instalado de três maneiras diferentes:

  1. Instaladores empacotados (mais fáceis, menos flexíveis)
  2. Instalação da fonte (leva algum tempo para configurar, muito flexível)
  3. Uma imagem oficial do docker de docker.com

Pacotes oficiais com todos os requisitos de dependência relevantes estão disponíveis em odoo.com.

Janelas

Baixe e execute o instalador.

Observação: No Windows 8, você pode ver um aviso intitulado “O Windows protegeu seu PC”. Clique em Mais informações e depois em Executar mesmo assim. Aceite o prompt do UAC e siga as várias etapas de instalação. Odoo será iniciado automaticamente no final da instalação.

Configuração: #

O arquivo de configuração pode ser encontrado em %PROGRAMFILES%\Odoo 8.0-id\server\openerp-server.conf. (id é o seu nome de usuário do sistema)

O arquivo de configuração pode ser editado para conectar a um Postgresql remoto, editar localizações de arquivos ou definir um dbfilter. Para recarregar o arquivo de configuração, reinicie o serviço Odoo via Serviços ‣ servidor odoo.

Linux

Distribuições baseadas em Debian

Para instalar o Odoo 8.0 na distribuição baseada em Debian, execute os seguintes comandos como root:

# wget -O - https://nightly.odoo.com/odoo.key | apt-key add -
# echo "deb http://nightly.odoo.com/8.0/nightly/deb/ ./" >> /etc/apt/sources.list
# apt-get update && apt-get install odoo

Isso instalará automaticamente todas as dependências, instalará o próprio Odoo como um daemon e o iniciará automaticamente.

Observe que

para imprimir relatórios em PDF, você mesmo deve instalar o wkhtmltopdf: a versão do wkhtmltopdf disponível nos repositórios debian não suporta cabeçalhos e rodapés, portanto não pode ser instalado automaticamente. A versão recomendada é 0.12.1 e está disponível na página de download wkhtmltopdf, na seção de arquivo. Como não há um lançamento oficial para o Debian Jessie, você pode encontrar o pacote em http://nightly.odoo.com/extra/. ou você pode baixá-lo e instalá-lo na página de download do wkhtmltopdf como esta

# wget https://bitbucket.org/wkhtmltopdf/wkhtmltopdf/downloads/{path to correct distro and system architecture}
# sudo dpkg -i {.deb package}
# sudo cp /usr/local/bin/wkhtmlto* /usr/bin/

O arquivo de configuração pode ser encontrado em /etc/odoo/openerp-server.conf

Quando o arquivo de configuração é editado, o Odoo deve ser reiniciado usando o serviço:

$ sudo service odoo restart Reiniciando odoo: ok

Distribuições baseadas em RPM

Com distribuições baseadas em RHEL (RHEL, CentOS, Scientific Linux), o EPEL deve ser adicionado aos repositórios da distribuição para que todas as dependências do Odoo estejam disponíveis. Para CentOS:

$ sudo yum install -y epel-release

Para outras distribuições baseadas em RHEL, consulte a documentação do EPEL.

Abaixo estão as etapas de instalação.

$ sudo yum install -y postgresql-server
$ sudo postgresql-setup initdb
$ sudo systemctl enable postgresql
$ sudo systemctl start postgresql
$ sudo yum-config-manager --add-repo=https://nightly.odoo.com/8.0/nightly/rpm/odoo.repo
$ sudo yum install -y odoo
$ sudo systemctl enable odoo
$ sudo systemctl start odoo

Observe que

Para imprimir relatórios em PDF, você mesmo deve instalar o wkhtmltopdf: a versão do wkhtmltopdf disponível nos repositórios do Fedora/CentOS não suporta cabeçalhos e rodapés, portanto não pode ser instalado automaticamente. Use a versão disponível na página de download wkhtmltopdf. Configuração, semelhante ao debian, pode ser instalado com

wget https://bitbucket.org/wkhtmltopdf/wkhtmltopdf/downloads/{path to correct distro and system architecture}
sudo rpm -i  {.rpm package}
sudo cp /usr/local/bin/wkhtmlto* /usr/bin/

O arquivo de configuração pode ser encontrado em /etc/odoo/openerp-server.conf

Quando o arquivo de configuração é editado, o Odoo deve ser reiniciado via Systemd:

$ sudo systemctl restart odoo

Instalação de origem

Odoo zip pode ser baixado em https://nightly.odoo.com/8.0/nightly/src/odoo_8.0.latest.zip, o arquivo zip precisa ser descompactado para usar seu conteúdo

O Git permite uma atualização mais simples e uma troca mais fácil entre as diferentes versões do Odoo. Também simplifica a manutenção de patches e contribuições não modulares. A principal desvantagem do git é que ele é significativamente maior que um tarball, pois contém todo o histórico do projeto Odoo.

O repositório git é https://github.com/odoo/odoo.git.

Então você pode clonar o repositório com

$ git clone https://github.com/odoo/odoo.git

Instalando dependências

A instalação de origem requer a instalação manual de dependências:

Python 2.7. no Linux e OS X, incluído por padrão

no Windows, use o instalador oficial do Python 2.7.9.

se o Python já estiver instalado, verifique se é 2.7.9, as versões anteriores são menos convenientes e as versões 3.x não são compatíveis com Odoo

configurando o PostgreSQL

Após a instalação, você precisará criar um usuário postgres: por padrão, o único usuário é postgres, e o Odoo proíbe a conexão como postgres.

no Linux, use o pacote da sua distribuição e crie um usuário postgres chamado como seu login:

$ sudo su - postgres -c "createuser -s $USER"

Como o login da função é o mesmo que o seu login unix, os soquetes unix podem ser usados ​​sem uma senha. no OS X, postgres.app é a maneira mais simples de começar, então crie um usuário postgres como no Linux

no Windows, use o PostgreSQL para windows então adicione o diretório bin do PostgreSQL (padrão: C:\Program Files\PostgreSQL\9.4\bin) ao seu PATH

crie um usuário postgres com uma senha usando o pg admin gui: abra o pgAdminIII, clique duas vezes no servidor para criar uma conexão, selecione Edit ‣ New Object ‣ New Login Role, digite o nome de usuário no campo Role Name (por exemplo, odoo), então abra a guia Definição e digite a senha (por exemplo, odoo) e clique em OK.

O usuário e a senha devem ser passados ​​para o Odoo usando as opções -w e -r ou o arquivo de configuração

Dependências do Python listadas no arquivo requirements.txt.

no Linux, as dependências do python podem ser instaláveis ​​com o gerenciador de pacotes do sistema ou usando o pip.

Para bibliotecas usando código nativo (Pillow, lxml, greenlet, gevent, psycopg2, ldap) pode ser necessário instalar ferramentas de desenvolvimento e dependências nativas antes que o pip possa instalar as dependências. Eles estão disponíveis em pacotes -dev ou -devel para Python, Postgres, libxml2, libxslt, libevent, libsasl2 e libldap2. Em seguida, as dependências do Python podem ser instaladas:

$ pip install -r requirements.txt

No OS X, você precisará instalar as Ferramentas de Linha de Comando (xcode-select –install) e depois baixar e instalar um gerenciador de pacotes de sua escolha (homebrew, macports) para instalar dependências não-Python. pip pode ser usado para instalar as dependências do Python como no Linux:

$ pip install -r requirements.txt

no Windows, você precisa instalar algumas das dependências manualmente, ajustar o arquivo requirements.txt e executar o pip para instalar as restantes.

Install psycopg using the installer here http://www.stickpeople.com/projects/python/win-psycopg/

Em seguida, edite o arquivo requirements.txt: remova o psycopg2 como você já o possui. remova os opcionais python-ldap, gevent e psutil porque eles exigem compilação. adicione pypiwin32 porque é necessário no windows.

Em seguida, use pip para instalar as dependências usando o seguinte comando em um prompt cmd.exe (substitua \YourOdooPath pelo caminho real onde você baixou o Odoo):

C:\> cd \YourOdooPath
C:\YourOdooPath> C:\Python27\Scripts\pip.exe install -r requirements.txt

Menos CSS via nodejs

no Linux, use o gerenciador de pacotes da sua distribuição para instalar o nodejs e o npm.

Observe que

No debian wheezy e no Ubuntu 13.10 e antes você precisa instalar o nodejs manualmente:

$ wget -qO- https://deb.nodesource.com/setup | bash -
$ apt-get install -y nodejs

Em versões posteriores do debian (> jessie) e ubuntu (> 14.04) você pode precisar adicionar um link simbólico como pacotes npm chamam node mas o debian chama o nodejs binário

$ apt-get install -y npm
$ sudo ln -s /usr/bin/nodejs /usr/bin/node

Once npm is installed, use it to install less and less-plugin-clean-css:

$ sudo npm install -g less less-plugin-clean-css

on OS X, install nodejs via your preferred package manager (homebrew, macports) then install less and less-plugin-clean-css:

$ sudo npm install -g less less-plugin-clean-css

no Windows, instale nodejs, reinicie (para atualizar o PATH) e instale less e less-plugin-clean-css:

C:\> npm install -g less less-plugin-clean-css

Executando Odoo

Depois que todas as dependências estiverem configuradas, o Odoo pode ser iniciado executando odoo.py.

A configuração pode ser fornecida por meio de argumentos de linha de comando ou por meio de um arquivo de configuração.

As configurações necessárias comuns são:

PostgreSQL host, port, user and password.

Odoo não tem padrões além dos padrões do psycopg2: conecta-se em um soquete UNIX na porta 5432 com o usuário atual e sem senha. Por padrão, isso deve funcionar no Linux e no OS X, mas não funcionará no Windows, pois não suporta soquetes UNIX. Custom addons path beyond the defaults, to load your own modules

No Windows, uma maneira típica de executar odoo seria:

C:\YourOdooPath> python odoo.py -w odoo -r odoo --addons-path=addons,../mymodules --db-filter=mydb$

Onde odoo, odoo são o login e senha do postgresql, ../mymodules um diretório com addons adicionais e mydb o db padrão para servir em localhost:8069

Em sistemas *nix, uma maneira típica de executar odoo seria:

$ ./odoo.py --addons-path=addons,../mymodules --db-filter=mydb$Packaged installers
