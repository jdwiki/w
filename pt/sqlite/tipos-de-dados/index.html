Tipos de data/hora #

SQLite não tem tipo de dados separado para valores de data ou hora.

Cordas ISO8601 #

As palavras-chave internas CURRENT_DATE, CURRENT_TIME e CURRENT_TIMESTAMP retornam strings no formato ISO8601:

> SELECT CURRENT_DATE, CURRENT_TIME, CURRENT_TIMESTAMP;
CURRENT_DATE  CURRENT_TIME  CURRENT_TIMESTAMP  
------------  ------------  -------------------
2016-07-08    12:34:56      2016-07-08 12:34:56

Esses valores também são entendidos por todas as funções de data/hora incorporadas:

> SELECT strftime('%Y', '2016-07-08');
2016

Números do dia juliano #

As funções de data/hora incorporadas interpretam números como dias julianos:

> SELECT datetime(2457578.02425926);
2016-07-08 12:34:56

A função julianday() converte qualquer valor de data/hora suportado em um número de dia juliano:

> SELECT julianday('2016-07-08 12:34:56');
2457578.02425926

timestamps Unix #

As funções de data/hora incorporadas podem interpretar números como carimbos de data e hora Unix com o modificador unixepoch:

> SELECT datetime(0, 'unixepoch');
1970-01-01 00:00:00

A função strftime() pode converter qualquer valor de data/hora suportado em um timestamp Unix:

> SELECT strftime('%s', '2016-07-08 12:34:56');
1467981296

formatos não suportados #

Seria possível armazenar valores de data/hora em qualquer outro formato no banco de dados, mas as funções internas de data/hora não os analisarão e retornarão NULL:

> SELECT time('1:30:00');   -- not two digits

> SELECT datetime('8 Jul 2016');
 

Impondo tipos de coluna #

SQLite usa digitação dinâmica e ignora os tipos de coluna declarados:

> CREATE TABLE Test (
      Col1 INTEGER,
      Col2 VARCHAR(2),       -- length is ignored, too
      Col3 BLOB,
      Col4,                  -- no type required
      Col5 FLUFFY BUNNIES    -- use whatever you want
  );
> INSERT INTO Test VALUES (1, 1, 1, 1, 1);
> INSERT INTO Test VALUES ('xxx', 'xxx', 'xxx', 'xxx', 'xxx');
> SELECT * FROM Test;
1   1   1   1   1         
xxx xxx xxx xxx xxx

(No entanto, os tipos de coluna declarados são usados ​​para afinidade de tipo.)

Para impor tipos, você precisa adicionar uma restrição com a função [typeof()](https://www.wikiod.com/pt/sqlite/tipos-de-dados#Função TIPO DE):

CREATE TABLE Tab (
    Col1 TEXT   CHECK (typeof(Col1) = 'text' AND length(Col1) <= 10),
    [...]
);

(Se tal coluna deve ser NULLable, você deve permitir explicitamente 'null'.)

Função TYPEOF #

sqlite> SELECT TYPEOF(NULL);
null
sqlite> SELECT TYPEOF(42);
integer
sqlite> SELECT TYPEOF(3.141592653589793);
real
sqlite> SELECT TYPEOF('Hello, world!');
text
sqlite> SELECT TYPEOF(X'0123456789ABCDEF');
blob

Usando booleanos #

Para booleanos, o SQLite usa inteiros 0 e 1:

sqlite> SELECT 2 + 2 = 4;
1
sqlite> SELECT 'a' = 'b';
0
sqlite> SELECT typeof('a' = 'b');
integer

> CREATE TABLE Users ( Name, IsAdmin );
> INSERT INTO Users VALUES ('root', 1);
> INSERT INTO Users VALUES ('john', 0);
> SELECT Name FROM Users WHERE IsAdmin;
root
