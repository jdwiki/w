Sintaxe #

  • $variável = ‘valor’; // Atribui variável geral
  • $objeto->propriedade = ‘valor’; // Atribui uma propriedade de objeto
  • ClassName::$propriedade = ‘valor’; // Atribui uma propriedade de classe estática
  • $array[0] = ‘valor’; // Atribui um valor a um índice de um array
  • $array[] = ‘valor’; // Empurra um item no final de um array
  • $array[‘chave’] = ‘valor’; // Atribui um valor de array
  • echo $variável; // Echo (imprime) um valor de variável
  • alguma_função($variável); // Usa a variável como parâmetro da função

  • isset($variável); // Verifica se uma variável está definida ou não
  • vazio($variável); // Verifica se uma variável está vazia ou não

Verificação de tipos #

Algumas das documentações sobre variáveis ​​e tipos mencionam que o PHP não usa digitação estática. Isso está correto, mas o PHP faz alguma verificação de tipo quando se trata de parâmetros de função/método e valores de retorno (especialmente com PHP 7).

Você pode impor a verificação de tipo de parâmetro e valor de retorno usando dica de tipo no PHP 7 da seguinte forma:

<?php

/**
 * Juggle numbers and return true if juggling was
 * a great success.
 */
function numberJuggling(int $a, int $b) : bool
{
    $sum = $a + $b;

    return $sum % 2 === 0;
}

Nota: gettype() do PHP para inteiros e booleanos é integer e boolean respectivamente. Mas para dicas de tipo para tais variáveis ​​você precisa usar int e bool. Caso contrário, o PHP não lhe dará um erro de sintaxe, mas esperará que as classes integer e boolean sejam passadas.

O exemplo acima gera um erro caso um valor não numérico seja fornecido como o parâmetro $a ou $b, e se a função retornar algo diferente de true ou false. O exemplo acima é “loose”, pois você pode dar um valor float para $a ou $b. Se você deseja impor tipos estritos, o que significa que você só pode inserir números inteiros e não floats, adicione o seguinte no início do seu arquivo PHP:

<?php
declare('strict_types=1');

Antes do PHP 7, funções e métodos permitiam dicas de tipo para os seguintes tipos:

  • callable (uma função ou método que pode ser chamado)
  • array (qualquer tipo de array, que pode conter outros arrays também)
  • Interfaces (nome de classe totalmente qualificado ou FQDN)
  • Aulas (FQDN)

Veja também: https://www.wikiod.com/pt/php/saida-do-valor-de-uma-variavel

Acessando uma variável dinamicamente por nome (variáveis ​​variáveis) #

As variáveis ​​podem ser acessadas por meio de nomes de variáveis ​​dinâmicas. O nome de uma variável pode ser armazenado em outra variável, permitindo que ela seja acessada dinamicamente. Tais variáveis ​​são conhecidas como variáveis ​​variáveis.

Para transformar uma variável em uma variável variável, você coloca um $ extra na frente de sua variável.

$variableName = 'foo';
$foo = 'bar';

// The following are all equivalent, and all output "bar":
echo $foo;
echo ${$variableName};
echo $$variableName;

//similarly,
$variableName  = 'foo';
$$variableName = 'bar';

// The following statements will also output 'bar'
echo $foo; 
echo $$variableName; 
echo ${$variableName};

Variáveis ​​variáveis ​​são úteis para mapear chamadas de função/método:

function add($a, $b) {
    return $a + $b;
}

$funcName = 'add';

echo $funcName(1, 2); // outputs 3

Isso se torna particularmente útil em classes PHP:

class myClass {
    public function __construct() {
        $functionName = 'doSomething';
        $this->$functionName('Hello World');
    }

    private function doSomething($string) {
        echo $string; // Outputs "Hello World"
    }
}

É possível, mas não obrigatório, colocar $variableName entre {}:

${$variableName} = $value;

Os exemplos a seguir são equivalentes e produzem “baz”:

$fooBar = 'baz';
$varPrefix = 'foo';

echo $fooBar;               // Outputs "baz"
echo ${$varPrefix . 'Bar'}; // Also outputs "baz"

Usar {} só é obrigatório quando o nome da variável é uma expressão, como esta:

${$variableNamePart1 . $variableNamePart2} = $value;

No entanto, é recomendado sempre usar {}, porque é mais legível.

Embora não seja recomendado fazer isso, é possível encadear esse comportamento:

$$$$$$$$DoNotTryThisAtHomeKids = $value;

É importante notar que o uso excessivo de variáveis ​​variáveis ​​é considerado uma má prática por muitos desenvolvedores. Como eles não são adequados para análise estática por IDEs modernos, grandes bases de código com muitas variáveis ​​variáveis ​​(ou invocações de métodos dinâmicos) podem rapidamente se tornar difíceis de manter.

Diferenças entre PHP5 e PHP7 #

Outra razão para sempre usar {} ou () é que PHP5 e PHP7 têm uma maneira ligeiramente diferente de lidar com variáveis ​​dinâmicas, o que resulta em um resultado diferente em alguns casos.

No PHP7, variáveis ​​dinâmicas, propriedades e métodos agora serão avaliados estritamente da esquerda para a direita, em oposição à mistura de casos especiais no PHP5. Os exemplos abaixo mostram como a ordem de avaliação mudou.

Caso 1: $$foo['bar']['baz'] #

  • Interpretação do PHP5: ${$foo['bar']['baz']}
  • Interpretação do PHP7: ($$foo)['bar']['baz']

Caso 2: $foo->$bar['baz'] #

  • Interpretação do PHP5: $foo->{$bar['baz']}
  • Interpretação do PHP7: ($foo->$bar)['baz']

Caso 3: $foo->$bar['baz']() #

  • Interpretação do PHP5: $foo->{$bar['baz']}()
  • Interpretação do PHP7: ($foo->$bar)['baz']()

Caso 4 : Foo::$bar['baz']() #

  • Interpretação do PHP5: Foo::{$bar['baz']}()
  • Interpretação do PHP7: (Foo::$bar)['baz']()

Tipos de dados #

Existem diferentes tipos de dados para diferentes propósitos. O PHP não possui definições de tipo explícitas, mas o tipo de uma variável é determinado pelo tipo do valor atribuído ou pelo tipo para o qual ela é convertida. Esta é uma breve visão geral sobre os tipos, para uma documentação detalhada e exemplos, veja o tópico de tipos PHP.

Existem os seguintes tipos de dados no PHP: null, boolean, integer, float, string, object, resource e array.

Nulo #

Nulo pode ser atribuído a qualquer variável. Representa uma variável sem valor.

$foo = null;

Isso invalida a variável e seu valor seria indefinido ou nulo se chamado. A variável é limpa da memória e excluída pelo coletor de lixo.

Boleano #

Este é o tipo mais simples com apenas dois valores possíveis.

$foo = true;
$bar = false;

Booleanos podem ser usados ​​para controlar o fluxo de código.

$foo = true;

if ($foo) {
    echo "true";
} else {
    echo "false";
}

Inteiro #

Um inteiro é um número inteiro positivo ou negativo. Pode ser usado com qualquer base numérica. O tamanho de um inteiro depende da plataforma. O PHP não suporta inteiros não assinados.

$foo = -3;  // negative
$foo = 0;   // zero (can also be null or false (as boolean)
$foo = 123; // positive decimal
$bar = 0123; // octal = 83 decimal
$bar = 0xAB; // hexadecimal = 171 decimal
$bar = 0b1010; // binary = 10 decimal
var_dump(0123, 0xAB, 0b1010); // output: int(83) int(171) int(10)

Flutuar #

Números de ponto flutuante, “duplos” ou simplesmente chamados de “floats” são números decimais.

$foo = 1.23;
$foo = 10.0;
$bar = -INF;
$bar = NAN;

Variedade #

Um array é como uma lista de valores. A forma mais simples de uma matriz é indexada por inteiro e ordenada pelo índice, com o primeiro elemento no índice 0.

$foo = array(1, 2, 3); // An array of integers
$bar = ["A", true, 123 => 5]; // Short array syntax, PHP 5.4+

echo $bar[0];    // Returns "A"
echo $bar[1];    // Returns true
echo $bar[123];  // Returns 5
echo $bar[1234]; // Returns null

As matrizes também podem associar uma chave diferente de um índice inteiro a um valor. Em PHP, todos os arrays são arrays associativos nos bastidores, mas quando nos referimos a um ‘array associativo’ distintamente, geralmente queremos dizer um que contém uma ou mais chaves que não são números inteiros.

$array = array();
$array["foo"] = "bar";
$array["baz"] = "quux";
$array[42] = "hello";
echo $array["foo"]; // Outputs "bar"
echo $array["bar"]; // Outputs "quux"
echo $array[42]; // Outputs "hello"

Corda #

Uma string é como um array de caracteres.

$foo = "bar";

Como um array, uma string pode ser indexada para retornar seus caracteres individuais:

$foo = "bar";
echo $foo[0]; // Prints 'b', the first character of the string in $foo.

Objeto #

Um objeto é uma instância de uma classe. Suas variáveis ​​e métodos podem ser acessados ​​com o operador ->.

$foo = new stdClass(); // create new object of class stdClass, which a predefined, empty class
$foo->bar = "baz";
echo $foo->bar; // Outputs "baz"
// Or we can cast an array to an object:
$quux = (object) ["foo" => "bar"];
echo $quux->foo; // This outputs "bar".

Recurso #

Variáveis ​​de recursos contêm identificadores especiais para arquivos abertos, conexões de banco de dados, fluxos, áreas de tela de imagem e similares (como é declarado no [manual](https://secure.php.net/manual/en/language.types.resource .php#language.types.resource.casting)).

$fp = fopen('file.ext', 'r'); // fopen() is the function to open a file on disk as a resource.
var_dump($fp); // output: resource(2) of type (stream)

Para obter o tipo de uma variável como uma string, use a função gettype():

echo gettype(1); // outputs "integer"
echo gettype(true); // "boolean"

Práticas recomendadas de variáveis ​​globais #

Podemos ilustrar este problema com o seguinte pseudo-código

function foo() {
    global $bob;
    $bob->doSomething();
}

Sua primeira pergunta aqui é óbvia

De onde veio o $bob?

Você está confuso? Bom. Você acabou de aprender por que os globais são confusos e considerados uma prática ruim.

Se este fosse um programa real, sua próxima diversão é rastrear todas as instâncias de $bob e esperar que você encontre a correta (isso fica pior se $bob for usado em todos os lugares). Pior, se outra pessoa definir $bob (ou você esqueceu e reutilizou essa variável) seu código pode quebrar (no exemplo de código acima, ter o objeto errado, ou nenhum objeto, causaria um erro fatal).

Já que virtualmente todos os programas PHP fazem uso de código como include('file.php'); seu trabalho de manutenção de código como este se torna exponencialmente mais difícil quanto mais arquivos você adiciona.

Além disso, isso torna a tarefa de testar seus aplicativos muito difícil. Suponha que você use uma variável global para manter sua conexão com o banco de dados:

$dbConnector = new DBConnector(...);

function doSomething() {
    global $dbConnector;
    $dbConnector->execute("...");
}

Para testar esta função, você precisa sobrescrever a variável global $dbConnector, executar os testes e redefini-la para seu valor original, que é muito propenso a erros:

/**
 * @test
 */
function testSomething() {
     global $dbConnector;

     $bkp = $dbConnector; // Make backup
     $dbConnector = Mock::create('DBConnector'); // Override

     assertTrue(foo());

     $dbConnector = $bkp; // Restore
}

Como evitamos Globals?

A melhor maneira de evitar globais é uma filosofia chamada Dependency Injection. É aqui que passamos as ferramentas que precisamos para a função ou classe.

function foo(\Bar $bob) {
    $bob->doSomething();
}

Isso é muito mais fácil de entender e manter. Não há como adivinhar onde $bob foi configurado porque o chamador é responsável por saber disso (está nos passando o que precisamos saber). Melhor ainda, podemos usar declarações de tipo para restringir o que está sendo passado.

Então sabemos que $bob é uma instância da classe Bar, ou uma instância de um filho de Bar, o que significa que sabemos que podemos usar os métodos dessa classe. Combinado com um autoloader padrão (disponível desde o PHP 5.3), agora podemos rastrear onde Bar está definido. O PHP 7.0 ou posterior inclui declarações de tipo expandidas, onde você também pode usar tipos escalares (como int ou string).

Variáveis ​​superglobais

Super globais em PHP são variáveis ​​predefinidas, que estão sempre disponíveis, podem ser acessadas de qualquer escopo ao longo do script.

Não há necessidade de fazer $variável global; para acessá-los dentro de funções/métodos, classes ou arquivos.

Essas variáveis ​​superglobais do PHP estão listadas abaixo:

  • $GLOBAIS
  • $_SERVER
  • $_REQUEST
  • $_POST
  • $_GET
  • $_FILES
  • $_ENV
  • $_COOKIE
  • $_SESSION

Valores padrão de variáveis ​​não inicializadas #

Embora não seja necessário em PHP, é uma prática muito boa inicializar variáveis. As variáveis ​​não inicializadas têm um valor padrão de seu tipo dependendo do contexto em que são usadas:

Não definido e não referenciado

var_dump($unset_var); // outputs NULL

Boleano

echo($unset_bool ? "true\n" : "false\n"); // outputs 'false'

Corda

$unset_str .= 'abc';
var_dump($unset_str); // outputs 'string(3) "abc"'

Inteiro

$unset_int += 25; // 0 + 25 => 25
var_dump($unset_int); // outputs 'int(25)'

Flutuante/duplo

$unset_float += 1.25;
var_dump($unset_float); // outputs 'float(1.25)'

Variedade

$unset_arr[3] = "def";
var_dump($unset_arr); //  outputs array(1) {  [3]=>  string(3) "def" }

Objeto

$unset_obj->foo = 'bar';
var_dump($unset_obj); // Outputs: object(stdClass)#1 (1) {  ["foo"]=>  string(3) "bar" }

Confiar no valor padrão de uma variável não inicializada é problemático no caso de incluir um arquivo em outro que usa o mesmo nome de variável.

Veracidade de valor variável e operador idêntico #

No PHP, os valores das variáveis ​​têm uma “veracidade” associada, de modo que mesmo valores não booleanos equivalem a true ou false. Isso permite que qualquer variável seja usada em um bloco condicional, por exemplo

if ($var == true) { /* explicit version */ }
if ($var) { /* $var == true is implicit */ }

Aqui estão algumas regras fundamentais para diferentes tipos de valores de variáveis:

  • Strings com comprimento diferente de zero equivalem a true, incluindo strings contendo apenas espaços em branco, como ' '.

  • Strings vazias '' equivalem a false.

    $var = ‘’; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

    $var = ’ ‘; $var_is_true = ($var == true); // true $var_is_false = ($var == false); // false

  • Inteiros equivalem a true se forem diferentes de zero, enquanto zero equivale a false.

    $var = -1; $var_is_true = ($var == true); // true $var = 99; $var_is_true = ($var == true); // true $var = 0; $var_is_true = ($var == true); // false

  • null equivale a false

    $var = null; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

  • Empty strings '' e string zero '0' equivalem a false.

    $var = ‘’; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

    $var = ‘0’; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

  • Valores de ponto flutuante equivalem a true se forem diferentes de zero, enquanto valores zero equivalem a false.

    • NAN (PHP’s Not-a-Number) equates to true, i.e. NAN == true is true. This is because NAN is a nonzero floating-point value.
    • Zero-values include both +0 and -0 as defined by IEEE 754. PHP does not distinguish between +0 and -0 in its double-precision floating-point, i.e. floatval('0') == floatval('-0') is true.
      • In fact, floatval('0') === floatval('-0').
      • Additionally, both floatval('0') == false and floatval('-0') == false.

    $var = NAN; $var_is_true = ($var == true); // true $var_is_false = ($var == false); // false

    $var = floatval(’-0’); $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

    $var = floatval(‘0’) == floatval(’-0’); $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

OPERADOR IDÊNTICO

Na Documentação PHP para Operadores de Comparação, existe um Operador Idêntico ===. Este operador pode ser usado para verificar se uma variável é idêntica a um valor de referência:

$var = null;
$var_is_null = $var === null; // true
$var_is_true = $var === true; // false
$var_is_false = $var === false; // false

Tem um operador não idêntico correspondente !==:

$var = null;
$var_is_null = $var !== null; // false
$var_is_true = $var !== true; // true
$var_is_false = $var !== false; // true

O operador idêntico pode ser usado como uma alternativa para funções de linguagem como is_null().

CASO DE USO COM strpos()

A função de linguagem strpos($haystack, $needle) é usada para localizar o índice no qual $needle ocorre em $haystack, ou se ocorre. A função strpos() diferencia maiúsculas de minúsculas; se encontrar sem distinção entre maiúsculas e minúsculas é o que você precisa, você pode usar stripos($haystack, $needle)

A função strpos & stripos também contém o terceiro parâmetro offset (int) que, se especificado, a pesquisa iniciará este número de caracteres contados a partir do início da string. Ao contrário de strrpos e stripos, o deslocamento não pode ser negativo

A função pode retornar:

  • 0 se $needle for encontrado no início de $haystack;
  • um inteiro diferente de zero especificando o índice se $needle for encontrado em algum lugar diferente do início em $haystack;
  • e valor false se $needle não for encontrado em nenhum lugar em $haystack.

Porque ambos 0 e false têm veracidade false em PHP, mas representam situações distintas para strpos(), é importante distinguir entre eles e usar o operador idêntico === para procurar exatamente por false e não apenas um valor que equivale a false.

$idx = substr($haystack, $needle);
if ($idx === false) 
{
    // logic for when $needle not found in $haystack
} 
else
{
    // logic for when $needle found in $haystack
}

Alternativamente, usando o operador não idêntico:

$idx = substr($haystack, $needle);
if ($idx !== false) 
{
    // logic for when $needle found in $haystack
} 
else
{
    // logic for when $needle not found in $haystack
}

Obtendo todas as variáveis ​​definidas #

get_defined_vars() retorna um array com todos os nomes e valores das variáveis ​​definidas no escopo em que a função é chamada. Se você quiser imprimir dados, você pode usar funções padrão para saída de dados legíveis por humanos, como print_r ou var_dump.

var_dump(get_defined_vars());

Nota: Esta função geralmente retorna apenas 4 superglobais: $_GET,$_POST,$_COOKIE,$_FILES. Outras superglobais são retornadas somente se tiverem sido usadas em algum lugar no código. Isso é por causa da diretiva auto_globals_jit que é habilitada por padrão. Quando ativado, as variáveis ​​$_SERVER e $_ENV são criadas quando são usadas pela primeira vez (Just In Time) em vez de quando o script é iniciado. Se essas variáveis ​​não forem usadas em um script, ter essa diretiva ativada resultará em um ganho de desempenho.
