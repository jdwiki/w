Sintaxe #

  • int readfile (string $filename [, bool $use_include_path = false [, resource $context ]] )

Parâmetros #

Parâmetro Descrição
nome do arquivo O nome do arquivo que está sendo lido.
use_include_path Você pode usar o segundo parâmetro opcional e defini-lo como TRUE, se quiser pesquisar o arquivo no include_path também.
contexto Um recurso de fluxo de contexto.

Sintaxe do nome do arquivo #

A maioria dos nomes de arquivos passados ​​para funções neste tópico são:

  1. Cordas na natureza.
  • Os nomes dos arquivos podem ser passados ​​diretamente. Se valores de outros tipos forem passados, eles serão convertidos em string. Isso é especialmente útil com SplFileInfo, que é o valor na iteração de DirectoryIterator.
  1. Relativo ou absoluto.
  • Podem ser absolutos. Em sistemas do tipo Unix, os caminhos absolutos começam com /, por exemplo /home/user/file.txt, enquanto no Windows, os caminhos absolutos começam com a unidade, por exemplo C:/Users/user/file.txt
  • Eles também podem ser relativos, o que depende do valor de getcwd e está sujeito a alterações por chdir.
  1. Aceite os protocolos.
  • Eles podem começar com scheme:// para especificar o wrapper de protocolo para gerenciar. Por exemplo, file_get_contents("http://example.com") recupera o conteúdo de http://example.com.
  1. Compatível com Slash.
  • Enquanto o DIRECTORY_SEPARATOR no Windows é uma barra invertida, e o sistema retorna barras invertidas para caminhos por padrão, o desenvolvedor ainda pode usar / como separador de diretório. Portanto, para compatibilidade, os desenvolvedores podem usar / como separadores de diretório em todos os sistemas, mas esteja ciente de que os valores retornados pelas funções (por exemplo, realpath) podem conter barras invertidas.

Excluindo arquivos e diretórios #

Apagando arquivos #

A função [unlink][unlink] exclui um único arquivo e retorna se a operação foi bem-sucedida.

$filename = '/path/to/file.txt';

if (file_exists($filename)) {
    $success = unlink($filename);
    
    if (!$success) {
         throw new Exception("Cannot delete $filename");
    }
}

Deletando diretórios, com deleção recursiva #

Por outro lado, os diretórios devem ser deletados com rmdir. No entanto, esta função apenas exclui diretórios vazios. Para excluir um diretório com arquivos, exclua primeiro os arquivos nos diretórios. Se o diretório contém subdiretórios, recursion pode ser necessário.

O exemplo a seguir verifica arquivos em um diretório, exclui arquivos/diretórios de membros recursivamente e retorna o número de arquivos (não diretórios) excluídos.

function recurse_delete_dir(string $dir) : int {
    $count = 0;

    // ensure that $dir ends with a slash so that we can concatenate it with the filenames directly
    $dir = rtrim($dir, "/\\") . "/";

    // use dir() to list files
    $list = dir($dir);

    // store the next file name to $file. if $file is false, that's all -- end the loop.
    while(($file = $list->read()) !== false) {
        if($file === "." || $file === "..") continue;
        if(is_file($dir . $file)) {
            unlink($dir . $file);
            $count++;
        } elseif(is_dir($dir . $file)) {
            $count += recurse_delete_dir($dir . $file);
        }
    }

    // finally, safe to delete directory!
    rmdir($dir);

    return $count;
}

Funções de conveniência #

IO direto bruto #

file_get_contents e file_put_contents fornecem a capacidade de ler/escrever de/para um arquivo de/para uma string PHP em uma única chamada.

file_put_contents também pode ser usado com o sinalizador de máscara de bits FILE_APPEND para anexar ao arquivo, em vez de truncar e sobrescrever. Ele pode ser usado junto com a máscara de bits LOCK_EX para adquirir um bloqueio exclusivo para o arquivo durante a escrita. Os sinalizadores de máscara de bits podem ser unidos com o operador OR bit a bit |.

$path = "file.txt";
// reads contents in file.txt to $contents
$contents = file_get_contents($path);
// let's change something... for example, convert the CRLF to LF!
$contents = str_replace("\r\n", "\n", $contents);
// now write it back to file.txt, replacing the original contents
file_put_contents($path, $contents);

FILE_APPEND é útil para anexar arquivos de log, enquanto LOCK_EX ajuda a evitar a condição de corrida de gravação de arquivos de vários processos. Por exemplo, para gravar em um arquivo de log sobre a sessão atual:

file_put_contents("logins.log", "{$_SESSION["username"]} logged in", FILE_APPEND | LOCK_EX);

CSV IO #

fgetcsv($file, $length, $separator)

O fgetcsv analisa a linha do arquivo aberto verificando os campos csv. Ele retorna campos CSV em uma matriz em caso de sucesso ou FALSE em caso de falha.

Por padrão, ele lerá apenas uma linha do arquivo CSV.

$file = fopen("contacts.csv","r");
print_r(fgetcsv($file));    
print_r(fgetcsv($file,5," "));
fclose($file);

contatos.csv

Kai Jim, Refsnes, Stavanger, Norway
Hege, Refsnes, Stavanger, Norway

Resultado:

Array
(
    [0] => Kai Jim
    [1] => Refsnes
    [2] => Stavanger
    [3] => Norway
)
Array
(
    [0] => Hege,
)

Lendo um arquivo para stdout diretamente #

readfile copia um arquivo para o buffer de saída. readfile() não apresentará nenhum problema de memória, mesmo ao enviar arquivos grandes, por conta própria.

$file = 'monkey.gif';

if (file_exists($file)) {
    header('Content-Description: File Transfer');
    header('Content-Type: application/octet-stream');
    header('Content-Disposition: attachment; filename="'.basename($file).'"');
    header('Expires: 0');
    header('Cache-Control: must-revalidate');
    header('Pragma: public');
    header('Content-Length: ' . filesize($file));
    readfile($file);
    exit;
}

Ou de um ponteiro de arquivo #

Alternativamente, para buscar um ponto no arquivo para começar a copiar para stdout, use fpassthru. No exemplo a seguir, os últimos 1024 bytes são copiados para stdout:

$fh = fopen("file.txt", "rb");
fseek($fh, -1024, SEEK_END); 
fpassthru($fh);

Lendo um arquivo em um array #

[file][file] retorna as linhas do arquivo passado em um array. Cada elemento do array corresponde a uma linha no arquivo, com a nova linha ainda anexada.

print_r(file("test.txt"));

test.txt

Welcome to File handling
This is to test file handling

Resultado:

Array 
( 
    [0] => Welcome to File handling 
    [1] => This is to test file handling 
)

Obtendo informações do arquivo #

Verifica se um caminho é um diretório ou um arquivo #

A função [is_dir][is_dir] retorna se o argumento é um diretório, enquanto [is_file][is_file] retorna se o argumento é um arquivo. Use [file_exists][file_exists] para verificar se é.

$dir  = "/this/is/a/directory";
$file = "/this/is/a/file.txt";

echo is_dir($dir) ? "$dir is a directory" : "$dir is not a directory", PHP_EOL,
    is_file($dir) ? "$dir is a file" : "$dir is not a file", PHP_EOL,
    file_exists($dir) ? "$dir exists" : "$dir doesn't exist", PHP_EOL,
    is_dir($file) ? "$file is a directory" : "$file is not a directory", PHP_EOL,
    is_file($file) ? "$file is a file" : "$file is not a file", PHP_EOL,
    file_exists($file) ? "$file exists" : "$file doesn't exist", PHP_EOL;

Isto dá:

/this/is/a/directory is a directory
/this/is/a/directory is not a file
/this/is/a/directory exists
/this/is/a/file.txt is not a directory
/this/is/a/file.txt is a file
/this/is/a/file.txt exists

Verificando o tipo de arquivo #

Use [filetype][filetype] para verificar o tipo de um arquivo, que pode ser:

  • fifo
  • char
  • dir
  • bloquear
  • ligação
  • arquivo
  • soquete
  • desconhecido

Passando o nome do arquivo para o [filetype][filetype] diretamente:

echo filetype("~"); // dir

Observe que filetype retorna false e aciona um E_WARNING se o arquivo não existir.

Verificando a legibilidade e a capacidade de escrita #

Passar o nome do arquivo para as funções [is_writable][is_writable] e [is_readable][is_readable] verifica se o arquivo é gravável ou legível, respectivamente.

As funções retornam false normalmente se o arquivo não existir.

Verificando o tempo de acesso/modificação do arquivo #

Usando [filemtime][filemtime] e [fileatime][fileatime] retorna o timestamp da última modificação ou acesso do arquivo. O valor de retorno é um timestamp Unix – veja https://www.wikiod.com/pt/php/trabalhando-com-datas-e-horas para detalhes.

echo "File was last modified on " . date("Y-m-d", filemtime("file.txt"));
echo "File was last accessed on " . date("Y-m-d", fileatime("file.txt"));

Obtém partes do caminho com fileinfo #

$fileToAnalyze = ('/var/www/image.png');

$filePathParts = pathinfo($fileToAnalyze);

echo '<pre>';
   print_r($filePathParts);
echo '</pre>';

Este exemplo produzirá:

Array
(
    [dirname] => /var/www
    [basename] => image.png
    [extension] => png
    [filename] => image
)

Que pode ser usado como:

$filePathParts['dirname']
$filePathParts['basename']
$filePathParts['extension']
$filePathParts['filename']
Parâmetro Detalhes
$caminho O caminho completo do arquivo a ser analisado
$opção Uma das quatro opções disponíveis [PATHINFO_DIRNAME, PATHINFO_BASENAME, PATHINFO_EXTENSION ou PATHINFO_FILENAME]
  • Se uma opção (o segundo parâmetro) não for passada, uma matriz associativa é retornada, caso contrário, uma string é retornada.
  • Não valida que o arquivo existe.
  • Simplesmente analisa a string em partes. Nenhuma validação é feita no arquivo (sem verificação do tipo mime, etc.)
  • A extensão é simplesmente a última extensão de $path O caminho para o arquivo image.jpg.png seria .png mesmo que tecnicamente seja um arquivo .jpg. Um arquivo sem extensão não retornará um elemento de extensão na matriz.

[tipo de arquivo]: http://php.net/tipo de arquivo [is_dir]: http://php.net/is-dir [is_file]: http://php.net/is-file [file_exists]: http://php.net/file-exists [is_readable]: http://php.net/is-readable [is_writable]: http://php.net/is-writable [filetime]: http://php.net/filetime [filemtime]: http://php.net/filemtime

Minimize o uso de memória ao lidar com arquivos grandes #

Se precisarmos analisar um arquivo grande, por exemplo, um CSV com mais de 10 Mbytes contendo milhões de linhas, alguns usam as funções file ou file_get_contents e acabam atingindo a configuração memory_limit com

Tamanho de memória permitido de XXXXX bytes esgotados

erro. Considere a seguinte fonte (top-1m.csv tem exatamente 1 milhão de linhas e tem cerca de 22 Mbytes de tamanho)

var_dump(memory_get_usage(true));
$arr = file('top-1m.csv');
var_dump(memory_get_usage(true));

Isso produz:

int(262144)
int(210501632)

porque o interpretador precisava manter todas as linhas no array $arr, então consumiu ~200 Mbytes de RAM. Observe que ainda não fizemos nada com o conteúdo do array.

Agora considere o seguinte código:

var_dump(memory_get_usage(true));
$index = 1;
if (($handle = fopen("top-1m.csv", "r")) !== FALSE) {
    while (($row = fgetcsv($handle, 1000, ",")) !== FALSE) {
        file_put_contents('top-1m-reversed.csv',$index . ',' . strrev($row[1]) . PHP_EOL, FILE_APPEND);
        $index++;
    }
    fclose($handle);
}
var_dump(memory_get_usage(true));

quais saídas

int(262144)
int(262144)

então não usamos um único byte extra de memória, mas analisamos todo o CSV e salvamos em outro arquivo invertendo o valor da 2ª coluna. Isso porque fgetcsv lê apenas uma linha e $row é sobrescrito em cada loop.

E/S de arquivo baseado em fluxo #

Abrindo um fluxo #

fopen abre um identificador de fluxo de arquivo, que pode ser usado com várias funções para leitura, escrita, busca e outras funções sobre ele. Este valor é do tipo resource e não pode ser passado para outras threads mantendo sua funcionalidade.

$f = fopen("errors.log", "a"); // Will try to open errors.log for writing

O segundo parâmetro é o modo do fluxo de arquivos:

Modo Descrição
r Abrir em modo somente leitura, começando no início do arquivo
r+ Aberto para leitura e escrita, começando no início do arquivo
w aberto apenas para escrita, começando no início do arquivo. Se o arquivo existir, ele será esvaziado. Se não existir, tentará criá-lo.
w+ aberto para leitura e escrita, começando no início do arquivo. Se o arquivo existir, ele será esvaziado. Se não existir, tentará criá-lo.
a abra um arquivo somente para escrita, começando no final do arquivo. Se o arquivo não existir, ele tentará criá-lo
a+ abra um arquivo para leitura e escrita, começando no final do arquivo. Se o arquivo não existir, ele tentará criá-lo
x crie e abra um arquivo somente para escrita. Se o arquivo existir a chamada fopen falhará
x+ criar e abrir um arquivo para leitura e escrita. Se o arquivo existir a chamada fopen falhará
c abra o arquivo somente para escrita. Se o arquivo não existir, ele tentará criá-lo. Ele começará a gravar no início do arquivo, mas não o esvaziará antes da gravação
c+ abra o arquivo para leitura e escrita. Se o arquivo não existir, ele tentará criá-lo. Ele começará a gravar no início do arquivo, mas não o esvaziará antes da gravação

Adicionar um t atrás do modo (por exemplo, a+b, wt, etc.) no Windows traduzirá os finais de linha "\n" para "\r\n" ao trabalhar com o arquivo. Adicione b atrás do modo se isso não for pretendido, especialmente se for um arquivo binário.

A aplicação PHP deve fechar streams usando fclose quando eles não forem mais usados ​​para evitar o erro Too many open files. Isso é particularmente importante em programas CLI, pois os fluxos são fechados apenas quando o tempo de execução é encerrado - isso significa que em servidores web, pode não ser necessário_ (mas ainda deve deve, como prática para evitar vazamento de recursos) feche os fluxos se você não espera que o processo seja executado por um longo tempo e não abrirá muitos fluxos.

Leitura #

Usando [fread][fread] lerá o número dado de bytes do ponteiro do arquivo, ou até que um EOF seja encontrado.

Lendo linhas #

Usar fgets lerá o arquivo até que um EOL seja alcançado, ou o comprimento dado seja lido.

Tanto [fread][fread] quanto fgets moverão o ponteiro do arquivo durante a leitura.

Lendo tudo o que resta #

Usar stream_get_contents transformará todos os bytes restantes no fluxo em uma string e a retornará.

Ajustando a posição do ponteiro do arquivo #

Inicialmente, após abrir o fluxo, o ponteiro do arquivo está no início do arquivo (ou no final, se o modo a for usado). Usar a função fseek moverá o ponteiro do arquivo para uma nova posição, relativa a um dos três valores:

  • SEEK_SET: Este é o valor padrão; o deslocamento da posição do arquivo será relativo ao início do arquivo.
  • SEEK_CUR: O deslocamento da posição do arquivo será relativo à posição atual.
  • SEEK_END: O deslocamento da posição do arquivo será relativo ao final do arquivo. Passar um deslocamento negativo é o uso mais comum para esse valor; ele moverá a posição do arquivo para o número especificado de bytes antes do final do arquivo.

[rewind][rewind] é um atalho conveniente de fseek($fh, 0, SEEK_SET).

Usar ftell mostrará a posição absoluta do ponteiro do arquivo.

Por exemplo, o script a seguir lê pula os primeiros 10 bytes, lê os próximos 10 bytes, pula 10 bytes, lê os próximos 10 bytes e, em seguida, os últimos 10 bytes em file.txt:

$fh = fopen("file.txt", "rb");
fseek($fh, 10); // start at offset 10
echo fread($fh, 10); // reads 10 bytes
fseek($fh, 10, SEEK_CUR); // skip 10 bytes
echo fread($fh, 10); // read 10 bytes
fseek($fh, -10, SEEK_END); // skip to 10 bytes before EOF
echo fread($fh, 10); // read 10 bytes
fclose($fh);

Escrita #

Usando fwrite grava a string fornecida no arquivo começando no ponteiro do arquivo atual.

fwrite($fh, "Some text here\n");

Movendo e copiando arquivos e diretórios #

Copiando arquivos #

[copy][copy] copia o arquivo de origem no primeiro argumento para o destino no segundo argumento. O destino resolvido precisa estar em um diretório já criado.

if (copy('test.txt', 'dest.txt')) {
    echo 'File has been copied successfully';
} else {
    echo 'Failed to copy file to destination given.'
}

Copiando diretórios, com recursão #

Copiar diretórios é muito parecido com deletar diretórios, exceto que para arquivos [copy][copy] em vez de [unlink][unlink] é usado, enquanto para diretórios, mkdir em vez de rmdir é usado, no início em vez de estar no final da função.

function recurse_delete_dir(string $src, string $dest) : int {
    $count = 0;

    // ensure that $src and $dest end with a slash so that we can concatenate it with the filenames directly
    $src = rtrim($dest, "/\\") . "/";
    $dest = rtrim($dest, "/\\") . "/";

    // use dir() to list files
    $list = dir($src);

    // create $dest if it does not already exist
    @mkdir($dest);

    // store the next file name to $file. if $file is false, that's all -- end the loop.
    while(($file = $list->read()) !== false) {
        if($file === "." || $file === "..") continue;
        if(is_file($src . $file)) {
            copy($src . $file, $dest . $file);
            $count++;
        } elseif(is_dir($src . $file)) {
            $count += recurse_copy_dir($src . $file, $dest . $file);
        }
    }

    return $count;
}

Renomeando/Movendo #

Renomear/mover arquivos e diretórios é muito mais simples. Diretórios inteiros podem ser movidos ou renomeados em uma única chamada, usando a função [rename][rename].

  • rename("~/arquivo.txt", "~/arquivo.html");
  • rename("~/dir", "~/old_dir");
  • rename("~/dir/arquivo.txt", "~/dir2/arquivo.txt");
