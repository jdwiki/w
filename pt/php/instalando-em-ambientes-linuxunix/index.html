Instalação de linha de comando usando o APT para PHP 7 #

Isso só instalará o PHP. Se você deseja servir um arquivo PHP para a web, você também precisa instalar um servidor web como Apache, Nginx, ou usar PHP’s built-in web-server (php versão 5.4+).

Se você estiver em uma versão do Ubuntu abaixo de 16.04 e quiser usar o PHP 7 de qualquer maneira, você pode adicionar Repositório PPA da Ondrej fazendo: sudo add-apt-repository ppa:ondrej/php

Certifique-se de que todos os seus repositórios estejam atualizados:

sudo apt-get update

Após atualizar os repositórios do seu sistema, instale o PHP:

sudo apt-get install php7.0

Vamos testar a instalação verificando a versão do PHP:

php --version

Isso deve produzir algo assim.

Observação: sua saída será um pouco diferente.

PHP 7.0.8-0ubuntu0.16.04.1 (cli) ( NTS )
Copyright (c) 1997-2016 The PHP Group
Zend Engine v3.0.0, Copyright (c) 1998-2016 Zend Technologies
with Zend OPcache v7.0.8-0ubuntu0.16.04.1, Copyright (c) 1999-2016, by Zend Technologies
with Xdebug v2.4.0, Copyright (c) 2002-2016, by Derick Rethans

Agora você tem a capacidade de executar o PHP a partir da linha de comando.

Instalando em distribuições Enterprise Linux (CentOS, Scientific Linux, etc) #

Use o comando yum para gerenciar pacotes em sistemas operacionais baseados em Enterprise Linux:

yum install php

Isso instala uma instalação mínima do PHP, incluindo alguns recursos comuns. Se você precisar de módulos adicionais, precisará instalá-los separadamente. Mais uma vez, você pode usar yum para procurar esses pacotes:

yum search php-*

Saída de exemplo:

php-bcmath.x86_64 : A module for PHP applications for using the bcmath library
php-cli.x86_64 : Command-line interface for PHP
php-common.x86_64 : Common files for PHP
php-dba.x86_64 : A database abstraction layer module for PHP applications
php-devel.x86_64 : Files needed for building PHP extensions
php-embedded.x86_64 : PHP library for embedding in applications
php-enchant.x86_64 : Human Language and Character Encoding Support
php-gd.x86_64 : A module for PHP applications for using the gd graphics library
php-imap.x86_64 : A module for PHP applications that use IMAP

Para instalar a biblioteca gd:

yum install php-gd

As distribuições do Enterprise Linux sempre foram conservadoras com atualizações e normalmente não atualizam além da versão pontual com a qual foram enviadas. Vários repositórios de terceiros fornecem versões atuais do PHP:

  • SIU
  • Remi Colette
  • Webtatic

IUS e Webtatic fornecem pacotes de substituição com nomes diferentes (por exemplo, php56u ou php56w para instalar o PHP 5.6), enquanto o repositório do Remi fornece atualizações no local usando os mesmos nomes dos pacotes do sistema.

A seguir estão as instruções sobre como instalar o PHP 7.0 do repositório da Remi. Este é o exemplo mais simples, pois não é necessário desinstalar os pacotes do sistema.

# download the RPMs; replace 6 with 7 in case of EL 7
wget https://dl.fedoraproject.org/pub/epel/epel-release-latest-6.noarch.rpm
wget http://rpms.remirepo.net/enterprise/remi-release-6.rpm
# install the repository information
rpm -Uvh remi-release-6.rpm epel-release-latest-6.noarch.rpm
# enable the repository
yum-config-manager --enable epel --enable remi --enable remi-safe --enable remi-php70
# install the new version of PHP
# NOTE: if you already have the system package installed, this will update it
yum install php
