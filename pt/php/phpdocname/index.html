Sintaxe #

  • @api
  • @autor [nome] [<endereço de e-mail>]
  • @copyright <descrição>
  • @deprecated [<“Versão Semântica”>][:<“Versão Semântica”>] [<descrição>]
  • @example [URI] [<descrição>]
  • {@example [URI] [:<início>..<fim>]}
  • @inheritDoc
  • @interno
  • {@internal [descrição]}}
  • @license [<identificador SPDX>|URI] [nome]
  • @method [return “Tipo”] [nome]([“Tipo”] [parâmetro], […]) [descrição]
  • @package [nível 1]\[nível 2]\[etc.]
  • @param [“Tipo”] [nome] [<descrição>]
  • @property [“Tipo”] [nome] [<descrição>]
  • @return <“Tipo”> [descrição]
  • @veja [URI | “FQSEN”] [<descrição>]
  • @since [<“Versão Semântica”>] [<descrição>]
  • @throws [“Tipo”] [<descrição>]
  • @todo [description]
  • @uses [arquivo | “FQSEN”] [<descrição>]
  • @var [“Tipo”] [element_name] [<descrição>]
  • @version [“Versão Semântica”] [<descrição>]
  • @filesource - Inclui o arquivo atual nos resultados da análise do phpDocumentor
  • @link [URI] [<description>] - Tag de link ajuda a definir relação ou link entre elementos estruturais.

“PHPDoc” é uma seção de documentação que fornece informações sobre aspectos de um “Elemento Estrutural” — PSR-5

Anotações PHPDoc são comentários que fornecem metadados sobre todos os tipos de estruturas em PHP. Muitos IDEs populares são configurados por padrão para utilizar anotações PHPDoc para fornecer insights de código e identificar possíveis problemas antes que eles surjam.

Embora as anotações do PHPDoc não façam parte do núcleo do PHP, elas atualmente possuem status de rascunho com PHP-FIG como PSR-5.

Todas as anotações do PHPDoc estão contidas em DocBlocks que são demonstrados por uma multilinha com dois asteriscos:

/**
 *
 */

O rascunho completo dos padrões PHP-FIG está disponível no GitHub.

Adicionando metadados a funções #

As anotações de nível de função ajudam os IDEs a identificar valores de retorno ou código potencialmente perigoso

/**
 * Adds two numbers together.
 *
 * @param Int $a First parameter to add
 * @param Int $b Second parameter to add
 * @return Int
 */
function sum($a, $b)
{
    return (int) $a + $b;
}

/**
 * Don't run me! I will always raise an exception.
 *
 * @throws Exception Always
 */
function dangerousCode()
{
    throw new Exception('Ouch, that was dangerous!');
}

/**
 * Old structures should be deprecated so people know not to use them.
 *
 * @deprecated
 */
function oldCode()
{
    mysql_connect(/* ... */);
}

Descrevendo uma variável #

A palavra-chave @var pode ser usada para descrever o tipo e uso de:

  • uma propriedade de classe

  • uma variável local ou global

  • uma classe ou constante global

    class Example { /** @var string This is something that stays the same */ const UNCHANGING = “Untouchable”;

      /** @var string $some_str This is some string */
  public $some_str;

  /**
   * @var array $stuff    This is a collection of stuff
   * @var array $nonsense These are nonsense
   */
  private $stuff, $nonsense;

  ...

    }

O tipo pode ser um dos tipos internos do PHP ou uma classe definida pelo usuário, incluindo namespaces.

O nome da variável deve ser incluído, mas pode ser omitido se o docblock se aplicar a apenas um item.

Adicionando metadados aos arquivos #

Os metadados de nível de arquivo se aplicam a todo o código dentro do arquivo e devem ser colocados na parte superior do arquivo:

<?php

/**
 * @author John Doe (jdoe@example.com)
 * @copyright MIT
 */

Herdando metadados de estruturas pai #

Se uma classe estender outra classe e usar os mesmos metadados, fornecer @inheritDoc é uma maneira simples de usar a mesma documentação. Se várias classes herdarem de uma base, apenas a base precisaria ser alterada para que os filhos fossem afetados.

abstract class FooBase
{
    /**
     * @param Int $a First parameter to add
     * @param Int $b Second parameter to add
     * @return Int
     */
    public function sum($a, $b) {}
}

class ConcreteFoo extends FooBase
{
    /**
     * @inheritDoc
     */
    public function sum($a, $b)
    {
        return $a + $b;
    }
}

Coleções #

PSR-5 propõe uma forma de notação no estilo genérico para coleções.

Sintaxe dos genéricos #

Type[]
Type<Type>
Type<Type[, Type]...>
Type<Type[|Type]...>

Valores em uma coleção PODEM até ser outro array e até mesmo outra coleção.

Type<Type<Type>>
Type<Type<Type[, Type]...>>
Type<Type<Type[|Type]...>>

Exemplos #

<?php

/** 
 * @var ArrayObject<string> $name 
 */
$name = new ArrayObject(['a', 'b']);

/** 
 * @var ArrayObject<int> $name 
 */
$name = new ArrayObject([1, 2]);

/** 
 * @var ArrayObject<stdClass> $name 
 */
$name = new ArrayObject([
    new stdClass(), 
    new stdClass()
]);

/** 
 * @var ArrayObject<string|int|stdClass|bool> $name 
 */
$name = new ArrayObject([
    'a', 
    true, 
    1, 
    'b', 
    new stdClass(), 
    'c', 
    2
]);

/**
 * @var ArrayObject<ArrayObject<int>> $name 
 */
$name = new ArrayObject([
    new ArrayObject([1, 2]), 
    new ArrayObject([1, 2])
]);

/** 
 * @var ArrayObject<int, string> $name 
 */
$name = new ArrayObject([
    1 => 'a', 
    2 => 'b'
]);

/** 
 * @var ArrayObject<string, int> $name 
 */
$name = new ArrayObject([
    'a' => 1, 
    'b' => 2
]);

/** 
 * @var ArrayObject<string, stdClass> $name 
 */
$name = new ArrayObject([
    'a' => new stdClass(), 
    'b' => new stdClass()
]);
