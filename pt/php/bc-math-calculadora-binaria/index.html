BC Math (Calculadora Binária)

  • Sintaxe
  • Parâmetros
  • Comparação entre operações aritméticas BCMath e float
  • Usando bcmath para ler/escrever um binário longo em um sistema de 32 bits

Nesta página

  • Sintaxe
  • Parâmetros
  • Comparação entre operações aritméticas BCMath e float
  • Usando bcmath para ler/escrever um binário longo em um sistema de 32 bits

A calculadora binária pode ser usada para calcular com números de qualquer tamanho e precisão até 2147483647-1 decimais, em formato de string. A calculadora binária é mais precisa que o cálculo float do PHP.

Sintaxe #

  • string bcadd ( string $ left_operand , string $ right_operand [, int $ scale = 0 ] )
  • int bccomp ( string $ left_operand , string $ right_operand [, int $ scale = 0 ] )
  • string bcdiv ( string $ left_operand , string $ right_operand [, int $scale = 0 ] )
  • string bcmod ( string $ left_operand , string $ modulus )
  • string bcmul ( string $ left_operand , string $ right_operand [, int $ scale = 0 ] )
  • string bcpowmod ( string $ left_operand , string $ right_operand , string $ modulus [, int $ scale = 0 ] )
  • bool bcscale ( int $scale )
  • string bcsqrt ( string $ operando [, int $ escala = 0 ] )
  • string bcsub ( string $ left_operand , string $ right_operand [, int $ scale = 0 ] )

Parâmetros #

bcadd Adicione dois números de precisão arbitrários.
left_operando O operando esquerdo, como uma string.
operando_direito O operando direito, como uma string.
escala Um parâmetro opcional para definir o número de dígitos após a casa decimal no resultado.
bcomp Compare dois números de precisão arbitrários.
left_operando O operando esquerdo, como uma string.
operando_direito O operando direito, como uma string.
escala Um parâmetro opcional para definir o número de dígitos após a casa decimal que será usada na comparação.
bcdiv Divida dois números de precisão arbitrários.
left_operando O operando esquerdo, como uma string.
operando_direito O operando direito, como uma string.
escala Um parâmetro opcional para definir o número de dígitos após a casa decimal no resultado.
bcmod Obter módulo de um número de precisão arbitrário.
left_operando O operando esquerdo, como uma string.
módulo O módulo, como uma string.
bcmul Multiplique dois números de precisão arbitrária.
left_operando O operando esquerdo, como uma string.
operando_direito O operando direito, como uma string.
escala Um parâmetro opcional para definir o número de dígitos após a casa decimal no resultado.
bcpow Elevar um número de precisão arbitrário para outro.
left_operando O operando esquerdo, como uma string.
operando_direito O operando direito, como uma string.
escala Um parâmetro opcional para definir o número de dígitos após a casa decimal no resultado.
bcpowmod Elevar um número de precisão arbitrário para outro, reduzido por um módulo especificado.
left_operando O operando esquerdo, como uma string.
operando_direito O operando direito, como uma string.
módulo O módulo, como uma string.
escala Um parâmetro opcional para definir o número de dígitos após a casa decimal no resultado.
bcscale Defina o parâmetro de escala padrão para todas as funções matemáticas bc.
escala O fator de escala.
bcsqrt Obter a raiz quadrada de um número de precisão arbitrário.
operando O operando, como uma string.
escala Um parâmetro opcional para definir o número de dígitos após a casa decimal no resultado.
bcsub Subtraia um número de precisão arbitrário de outro.
left_operando O operando esquerdo, como uma string.
right_operand O operando certo, como uma string.
escala Um parâmetro opcional para definir o número de dígitos após a casa decimal no resultado.

Para todas as funções BC, se o parâmetro scale não estiver definido, o padrão será 0, o que fará com que todas as operações sejam operações inteiras.

Comparação entre operações aritméticas BCMath e float #

bcadd vs float+float #

var_dump('10' + '-9.99');           // float(0.0099999999999998)
var_dump(10 + -9.99);               // float(0.0099999999999998)
var_dump(10.00 + -9.99);            // float(0.0099999999999998)
var_dump(bcadd('10', '-9.99', 20)); // string(22) "0.01000000000000000000"

bcsub vs float-float #

var_dump('10' - '9.99');           // float(0.0099999999999998)
var_dump(10 - 9.99);               // float(0.0099999999999998)
var_dump(10.00 - 9.99);            // float(0.0099999999999998)
var_dump(bcsub('10', '9.99', 20)); // string(22) "0.01000000000000000000"

bcmul vs int*int #

var_dump('5.00' * '2.00');            // float(10)
var_dump(5.00 * 2.00);                // float(10)
var_dump(bcmul('5.0', '2', 20));      // string(4) "10.0"
var_dump(bcmul('5.000', '2.00', 20)); // string(8) "10.00000"
var_dump(bcmul('5', '2', 20));        // string(2) "10"

bcmul vs float*float #

var_dump('1.6767676767' * '1.6767676767');           // float(2.8115498416259)
var_dump(1.6767676767 * 1.6767676767);               // float(2.8115498416259)
var_dump(bcmul('1.6767676767', '1.6767676767', 20)); // string(22) "2.81154984162591572289"

bcdiv vs float/float #

var_dump('10' / '3.01');           // float(3.3222591362126)
var_dump(10 / 3.01);               // float(3.3222591362126)
var_dump(10.00 / 3.01);            // float(3.3222591362126)
var_dump(bcdiv('10', '3.01', 20)); // string(22) "3.32225913621262458471"

Usando bcmath para ler/escrever um binário longo em um sistema de 32 bits #

Em sistemas de 32 bits, inteiros maiores que 0x7FFFFFFF não podem ser armazenados primitivamente, enquanto números inteiros entre 0x0000000080000000 e 0x7FFFFFFFFFFFFFFF podem ser armazenados de forma primitiva em sistemas de 64 bits, mas não sistemas de 32 bits (assinado long long). No entanto, como os sistemas de 64 bits e muitas outras linguagens suportam o armazenamento de inteiros signed long long, às vezes é necessário armazenar esse intervalo de inteiros no valor exato. Existem várias maneiras de fazer isso, como criar uma matriz com dois números ou converter o inteiro em sua forma decimal legível por humanos. Isso tem várias vantagens, como a conveniência de apresentar ao usuário e a capacidade de manipulá-lo diretamente com bcmath.

Os métodos [pack][pack]/[unpack][unpack] podem ser usados ​​para converter entre bytes binários e a forma decimal dos números (ambos do tipo string, mas um é binário e o outro é ASCII) , mas eles sempre tentarão converter a string ASCII em um int de 32 bits em sistemas de 32 bits. O snippet a seguir fornece uma alternativa:

/** Use pack("J") or pack("p") for 64-bit systems */
function writeLong(string $ascii) : string {
    if(bccomp($ascii, "0") === -1) { // if $ascii < 0
        // 18446744073709551616 is equal to (1 << 64)
        // remember to add the quotes, or the number will be parsed as a float literal
        $ascii = bcadd($ascii, "18446744073709551616");
    }

    // "n" is big-endian 16-bit unsigned short. Use "v" for small-endian.
    return pack("n", bcmod(bcdiv($ascii, "281474976710656"), "65536")) .
        pack("n", bcmod(bcdiv($ascii, "4294967296"), "65536")) .
        pack("n", bcdiv($ascii, "65536"), "65536")) .
        pack("n", bcmod($ascii, "65536"));
}

function readLong(string $binary) : string {
    $result = "0";
    $result = bcadd($result, unpack("n", substr($binary, 0, 2)));
    $result = bcmul($result, "65536");
    $result = bcadd($result, unpack("n", substr($binary, 2, 2)));
    $result = bcmul($result, "65536");
    $result = bcadd($result, unpack("n", substr($binary, 4, 2)));
    $result = bcmul($result, "65536");
    $result = bcadd($result, unpack("n", substr($binary, 6, 2)));

    // if $binary is a signed long long
    // 9223372036854775808 is equal to (1 << 63) (note that this expression actually does not work even on 64-bit systems)
    if(bccomp($result, "9223372036854775808") !== -1) { // if $result >= 9223372036854775807
        $result = bcsub($result, "18446744073709551616"); // $result -= (1 << 64)
    }
    return $result;
}
