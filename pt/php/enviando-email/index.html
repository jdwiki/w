E-mail que estou enviando através do meu script nunca chega. O que devo fazer?

  • Verifique se o relatório de erros está ativado para ver os erros.

  • Se você tiver acesso aos arquivos de log de erros do PHP, verifique-os.

  • O comando mail() está configurado corretamente em seu servidor? (Se você estiver em hospedagem compartilhada, não poderá alterar nada aqui.)

  • Se os e-mails estão desaparecendo, inicie uma conta de e-mail com um serviço de e-mail gratuito que tenha uma pasta de spam (ou use uma conta de e-mail que não faça filtragem de spam). Dessa forma, você pode ver se o e-mail não está sendo enviado, ou talvez enviado, mas filtrado como spam.

  • Você verificou o endereço “de:” usado para possíveis e-mails “devolvidos ao remetente”? Você também pode configurar um endereço de devolução separado para e-mails de erro.

O e-mail que estou enviando está sendo filtrado como spam. O que devo fazer?

  • O endereço do remetente (“De”) pertence a um domínio que é executado no servidor de onde você envia o e-mail? Se não, mude isso.

Nunca use endereços de remetentes como xxx@gmail.com. Use reply-to se precisar que as respostas cheguem a um endereço diferente.

  • Seu servidor está em uma lista negra? Esta é uma possibilidade quando você está em hospedagem compartilhada quando os vizinhos se comportam mal. A maioria dos provedores de lista negra, como Spamhaus, tem ferramentas que permitem que você procure o IP do seu servidor. Há também ferramentas de terceiros como MX Toolbox.

  • Algumas instalações do PHP requerem a configuração de um quinto parâmetro para mail() para adicionar um endereço de remetente. Veja se esse pode ser o seu caso.

  • Se tudo mais falhar, considere o uso de e-mail como serviço, como Mailgun, SparkPost, [Amazon SES][amazonses], Mailjet, SendinBlue ou SendGrid—para citar alguns—em vez disso. Todos eles têm APIs que podem ser chamadas usando PHP.

Enviando e-mail - O básico, mais detalhes e um exemplo completo #

Um e-mail típico tem três componentes principais:

  1. Um destinatário (representado como um endereço de e-mail)
  2. Um assunto
  3. Um corpo de mensagem

Enviar e-mail em PHP pode ser tão simples quanto chamar a função interna mail(). mail() leva até cinco parâmetros, mas os três primeiros são tudo o que é necessário para enviar um e-mail (embora os quatro parâmetros sejam comumente usados ​​como será demonstrado abaixo). Os três primeiros parâmetros são:

  1. O endereço de e-mail do destinatário (string)
  2. O assunto do e-mail (string)
  3. O corpo do e-mail (string) (por exemplo, o conteúdo do e-mail)

Um exemplo mínimo seria semelhante ao código a seguir:

mail('recipient@example.com', 'Email Subject', 'This is the email message body');

O exemplo simples acima funciona bem em circunstâncias limitadas, como codificar um alerta de e-mail para um sistema interno. No entanto, é comum colocar os dados passados ​​como parâmetros para mail() em variáveis ​​para tornar o código mais limpo e fácil de gerenciar (por exemplo, construir dinamicamente um email a partir de um formulário enviado).

Além disso, mail() aceita um quarto parâmetro que permite que você tenha cabeçalhos de e-mail adicionais enviados com seu e-mail. Esses cabeçalhos podem permitir que você defina:

  • o nome De e o endereço de e-mail que o usuário verá

  • o endereço de e-mail Reply-To para o qual a resposta do usuário será enviada

  • cabeçalhos não padronizados adicionais como X-Mailer que podem dizer ao destinatário que este e-mail foi enviado via PHP

    $to = ‘recipient@example.com’; // Could also be $to = $_POST[‘recipient’];
    $subject = ‘Email Subject’; // Could also be $subject = $_POST[‘subject’];
    $message = ‘This is the email message body’; // Could also be $message = $_POST[‘message’];
    $headers = implode("\r\n", [ ‘From: John Conde webmaster@example.com’, ‘Reply-To: webmaster@example.com’, ‘X-Mailer: PHP/’ . PHP_VERSION ]);

O quinto parâmetro opcional pode ser usado para passar sinalizadores adicionais como opções de linha de comando para o programa configurado para ser usado ao enviar mensagens, conforme definido pela configuração sendmail_path. Por exemplo, isso pode ser usado para definir o endereço do remetente do envelope ao usar sendmail/postfix com a opção -f sendmail.

$fifth  = '-fno-reply@example.com';

Embora usar mail() possa ser bastante confiável, não é garantido que um e-mail seja enviado quando mail() for chamado. Para ver se há um erro potencial ao enviar seu e-mail, você deve capturar o valor de retorno de mail(). TRUE será devolvido se o e-mail foi aceito com sucesso para entrega. Caso contrário, você receberá FALSE.

$result = mail($to, $subject, $message, $headers, $fifth);

NOTA: Embora mail() possa retornar TRUE, isso não significa que o e-mail foi enviado ou que o e-mail será recebido pelo destinatário. Indica apenas que o correio foi entregue com sucesso ao sistema de correio do seu sistema.

Se você deseja enviar um e-mail em HTML, não há muito mais trabalho a ser feito. Você precisa:

  1. Adicione o cabeçalho MIME-Version

  2. Adicione o cabeçalho Content-Type

  3. Verifique se o conteúdo do seu e-mail é HTML

    $to = ‘recipient@example.com’;
    $subject = ‘Email Subject’;
    $message = ‘This is the email message body’;
    $headers = implode("\r\n", [ ‘From: John Conde webmaster@example.com’, ‘Reply-To: webmaster@example.com’, ‘MIME-Version: 1.0’, ‘Content-Type: text/html; charset=ISO-8859-1’, ‘X-Mailer: PHP/’ . PHP_VERSION ]);

Aqui está um exemplo completo de como usar a função mail() do PHP

<?php

// Debugging tools. Only turn these on in your development environment.

error_reporting(-1);
ini_set('display_errors', 'On');
set_error_handler("var_dump");

// Special mail settings that can make mail less likely to be considered spam
// and offers logging in case of technical difficulties.

ini_set("mail.log", "/tmp/mail.log");
ini_set("mail.add_x_header", TRUE);

// The components of our email

$to      = 'recipient@example.com';
$subject = 'Email Subject';
$message = 'This is the email message body';
$headers = implode("\r\n", [
    'From: webmaster@example.com',
    'Reply-To: webmaster@example.com',
    'X-Mailer: PHP/' . PHP_VERSION
]);

// Send the email

$result = mail($to, $subject, $message, $headers);

// Check the results and react accordingly

if ($result) {
  
    // Success! Redirect to a thank you page. Use the
    // POST/REDIRECT/GET pattern to prevent form resubmissions
    // when a user refreshes the page.
  
    header('Location: http://example.com/path/to/thank-you.php', true, 303);
    exit;
  
}
else {
  
    // Your mail was not sent. Check your logs to see if
    // the reason was reported there for you.
  
}

Enviando e-mail HTML usando mail() #

<?php
$to      = 'recipent@example.com';
$subject = 'Sending an HTML email using mail() in PHP';
$message = '<html><body><p><b>This paragraph is bold.</b></p><p><i>This text is italic.</i></p></body></html>';

$headers = implode("\r\n", [
    "From: John Conde <webmaster@example.com>",
    "Reply-To: webmaster@example.com",
    "X-Mailer: PHP/" . PHP_VERSION,
    "MIME-Version: 1.0",
    "Content-Type: text/html; charset=UTF-8"
]);

mail($to, $subject, $message, $headers);

Isso não é muito diferente de enviar um e-mail de texto simples. As principais diferenças sendo que o corpo do conteúdo é estruturado como um documento HTML e há dois cabeçalhos adicionais que devem ser incluídos para que o cliente de e-mail saiba como transformar o e-mail em HTML. Eles são:

  • MIME-Versão: 1.0
  • Tipo de conteúdo: texto/html; conjunto de caracteres=UTF-8

Enviando e-mail com um anexo usando mail() #

<?php

$to         = 'recipient@example.com';
$subject    = 'Email Subject';
$message    = 'This is the email message body';

$attachment = '/path/to/your/file.pdf';
$content = file_get_contents($attachment);

/* Attachment content transferred in Base64 encoding
MUST be split into chunks 76 characters in length as
specified by RFC 2045 section 6.8. By default, the
function chunk_split() uses a chunk length of 76 with
a trailing CRLF (\r\n). The 76 character requirement
does not include the carriage return and line feed */
$content = chunk_split(base64_encode($content));

/* Boundaries delimit multipart entities. As stated
in RFC 2046 section 5.1, the boundary MUST NOT occur
in any encapsulated part. Therefore, it should be
unique. As stated in the following section 5.1.1, a
boundary is defined as a line consisting of two hyphens
("--"), a parameter value, optional linear whitespace,
and a terminating CRLF. */
$prefix     = "part_"; // This is an optional prefix
/* Generate a unique boundary parameter value with our
prefix using the uniqid() function. The second parameter
makes the parameter value more unique. */
$boundary   = uniqid($prefix, true);

// headers
$headers    = implode("\r\n", [
    'From: webmaster@example.com',
    'Reply-To: webmaster@example.com',
    'X-Mailer: PHP/' . PHP_VERSION,
    'MIME-Version: 1.0',
    // boundary parameter required, must be enclosed by quotes
    'Content-Type: multipart/mixed; boundary="' . $boundary . '"',
    "Content-Transfer-Encoding: 7bit",
    "This is a MIME encoded message." // message for restricted transports
]);

// message and attachment
$message    = implode("\r\n", [ 
    "--" . $boundary, // header boundary delimiter line
    'Content-Type: text/plain; charset="iso-8859-1"',
    "Content-Transfer-Encoding: 8bit",
    $message,
    "--" . $boundary, // content boundary delimiter line
    'Content-Type: application/octet-stream; name="RenamedFile.pdf"',
    "Content-Transfer-Encoding: base64",
    "Content-Disposition: attachment",
    $content,
    "--" . $boundary . "--" // closing boundary delimiter line
]);

$result = mail($to, $subject, $message, $headers); // send the email

if ($result) {
    // Success! Redirect to a thank you page. Use the
    // POST/REDIRECT/GET pattern to prevent form resubmissions
    // when a user refreshes the page.
  
    header('Location: http://example.com/path/to/thank-you.php', true, 303);
    exit;
}
else {
    // Your mail was not sent. Check your logs to see if
    // the reason was reported there for you.
}

Codificações de Transferência de Conteúdo #

As codificações disponíveis são 7bit, 8bit, binary, quoted-printable, base64, ietf-token e x-token. Dessas codificações, quando um cabeçalho tem um tipo de conteúdo multipart, o Content-Transfer-Encoding não deve ser qualquer outro valor além de 7bit, 8bit ou binary conforme declarado na RFC 2045, seção 6.4.

Nosso exemplo escolhe a codificação de 7 bits, que representa os caracteres US-ASCII, para o cabeçalho de várias partes porque, conforme observado na seção 6 da RFC 2045, alguns protocolos suportam apenas essa codificação. Os dados dentro dos limites podem ser codificados parte por parte (RFC 2046, seção 5.1). Este exemplo faz exatamente isso. A primeira parte, que contém a mensagem de texto/simples, é definida como 8 bits, pois pode ser necessário suportar caracteres adicionais. Nesse caso, o conjunto de caracteres Latin1 (iso-8859-1) está sendo usado. A segunda parte é o anexo e, portanto, é definido como um application/octet-stream codificado em base64. Como a base64 transforma dados arbitrários na faixa de 7 bits, eles podem ser enviados por transportes restritos (RFC 2045, seção 6.2).

Enviando e-mail de texto simples usando PHPMailer #

E-mail de texto básico

<?php

$mail = new PHPMailer();
