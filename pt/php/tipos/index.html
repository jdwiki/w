Comparação de tipos #

Existem dois tipos de comparação: comparação solta com == e comparação estrita com ===. A comparação estrita garante que o tipo e o valor de ambos os lados do operador sejam os mesmos.

// Loose comparisons
var_dump(1 == 1); // true
var_dump(1 == "1"); // true
var_dump(1 == true); // true
var_dump(0 == false); // true

// Strict comparisons
var_dump(1 === 1); // true
var_dump(1 === "1"); // false
var_dump(1 === true); // false
var_dump(0 === false); // false

// Notable exception: NAN — it never is equal to anything
var_dump(NAN == NAN); // false
var_dump(NAN === NAN); // false

Você também pode usar uma comparação forte para verificar se o tipo e o valor não correspondem usando !==.

Um exemplo típico onde o operador == não é suficiente, são funções que podem retornar diferentes tipos, como strpos, que retorna false se a palavra de busca não for encontrada, e a posição correspondente (int) caso contrário:

if(strpos('text', 'searchword') == false)
  // strpos returns false, so == comparison works as expected here, BUT:
if(strpos('text bla', 'text') == false)
  // strpos returns 0 (found match at position 0) and 0==false is true.
  // This is probably not what you expect!
if(strpos('text','text') === false)
  // strpos returns 0, and 0===false is false, so this works as expected.

Boleano #

Boolean é um tipo, com dois valores, denotados como true ou false.

Este código define o valor de $foo como true e $bar como false:

$foo = true;
$bar = false;

true e false não diferenciam maiúsculas de minúsculas, então TRUE e FALSE também podem ser usados, até mesmo FaLsE é possível. Usar letras minúsculas é mais comum e recomendado na maioria dos guias de estilo de código, por exemplo PSR-2.

Booleanos podem ser usados ​​em instruções if como esta:

if ($foo) { //same as evaluating if($foo == true)
    echo "true";
}

Devido ao fato de que o PHP é fracamente tipado, se $foo acima for diferente de true ou false, ele é automaticamente forçado a um valor booleano. Os seguintes valores resultam em false:

  • um valor zero: 0 (inteiro), 0.0 (float) ou '0' (string)
  • uma string vazia '' ou array []
  • null (o conteúdo de uma variável não definida ou atribuída a uma variável)

Qualquer outro valor resulta em true.

Para evitar essa comparação frouxa, você pode impor uma comparação forte usando ===, que compara valor e tipo. Consulte [Comparação de Tipos][3] para obter detalhes.

Para converter um tipo em booleano, você pode usar a conversão (bool) ou (boolean) antes do tipo.

var_dump((bool) "1"); //evaluates to true

ou chame a função [boolval][4]:

var_dump( boolval("1") ); //evaluates to true

Conversão booleana para uma string (observe que false produz uma string vazia):

var_dump( (string) true ); // string(1) "1"
var_dump( (string) false ); // string(0) ""

Conversão booleana para um inteiro:

var_dump( (int) true ); // int(1)
var_dump( (int) false ); // int(0)

Observe que o contrário também é possível:

var_dump((bool) "");        // bool(false)
var_dump((bool) 1);         // bool(true)

Além disso, todos os valores diferentes de zero retornarão verdadeiro:

var_dump((bool) -2);        // bool(true)
var_dump((bool) "foo");     // bool(true)
var_dump((bool) 2.3e5);     // bool(true)
var_dump((bool) array(12)); // bool(true)
var_dump((bool) array());   // bool(false)
var_dump((bool) "false");   // bool(true)

[3]: https://www.wikiod.com/pt/php/tipos#Comparação de tipos [4]: http://php.net/manual/en/function.boolval.php

Flutuar #

$float = 0.123;

Por razões históricas, “double” é retornado por gettype() no caso de um float, e não simplesmente “float”

Floats são números de ponto flutuante, que permitem mais precisão de saída do que números inteiros simples.

Floats e integers podem ser usados ​​juntos devido à conversão livre de tipos de variáveis ​​do PHP:

$sum = 3 + 0.14;

echo $sum; // 3.14

php não mostra float como número float como outras linguagens, por exemplo:

$var = 1;
echo ((float) $var); //returns 1 not 1.0

Aviso #

Precisão de ponto flutuante

(Da página de manual do PHP)

Números de ponto flutuante têm precisão limitada. Embora dependa do sistema, o PHP normalmente fornece um erro relativo máximo devido ao arredondamento na ordem de 1.11e-16. Operações aritméticas não elementares podem gerar erros maiores, e a propagação do erro deve ser considerada quando várias operações são compostas.

Além disso, números racionais que são exatamente representáveis ​​como números de ponto flutuante na base 10, como 0,1 ou 0,7, não têm uma representação exata como números de ponto flutuante na base 2 (binário), que é usado internamente, não importa o tamanho do mantissa. Portanto, eles não podem ser convertidos em suas contrapartes binárias internas sem uma pequena perda de precisão. Isso pode levar a resultados confusos: por exemplo, floor((0.1+0.7)*10) geralmente retornará 7 em vez do esperado 8, pois a representação interna será algo como 7.9999999999999991118….

Portanto, nunca confie resultados de números flutuantes no último dígito e não compare números de ponto flutuante diretamente para igualdade. Se for necessária uma precisão mais alta, as funções matemáticas de precisão arbitrária e as funções gmp estão disponíveis.

Cordas #

Uma string em PHP é uma série de caracteres de byte único (ou seja, não há suporte nativo a Unicode) que podem ser especificados de quatro maneiras:

Aspas simples #

Exibe as coisas quase completamente “como estão”. Variáveis ​​e a maioria das sequências de escape não serão interpretadas. A exceção é que para exibir uma aspa simples literal, pode-se escapá-la com uma barra invertida ', e para exibir uma barra invertida, pode-se escapá-la com outra barra invertida \

$my_string = 'Nothing is parsed, except an escap\'d apostrophe or backslash. $foo\n';
var_dump($my_string);

/*
string(68) "Nothing is parsed, except an escap'd apostrophe or backslash. $foo\n"
*/

Aspas duplas #

Ao contrário de uma string entre aspas simples, nomes de variáveis ​​simples e sequências de escape nas strings serão avaliados. Chaves (como no último exemplo) podem ser usadas para isolar nomes de variáveis ​​complexas.

$variable1 = "Testing!";
$variable2 = [ "Testing?", [ "Failure", "Success" ] ];
$my_string = "Variables and escape characters are parsed:\n\n";
$my_string .= "$variable1\n\n$variable2[0]\n\n";
$my_string .= "There are limits: $variable2[1][0]";
$my_string .= "But we can get around them by wrapping the whole variable in braces: {$variable2[1][1]}";
var_dump($my_string);

/*
string(98) "Variables and escape characters are parsed:

Testing!

Testing?

There are limits: Array[0]"

But we can get around them by wrapping the whole variable in braces: Success

*/

Heredoc #

Em uma string heredoc, nomes de variáveis ​​e sequências de escape são analisados ​​de maneira semelhante a strings com aspas duplas, embora chaves não estejam disponíveis para nomes de variáveis ​​complexas. O início da string é delimitado por <<< identifier, e o final por identifier, onde identifier é qualquer nome PHP válido. O identificador final deve aparecer em uma linha por si só. Nenhum espaço em branco é permitido antes ou depois do identificador, embora, como qualquer linha no PHP, também deva ser encerrado por um ponto e vírgula.

$variable1 = "Including text blocks is easier";
$my_string = <<< EOF
Everything is parsed in the same fashion as a double-quoted string,
but there are advantages. $variable1; database queries and HTML output
can benefit from this formatting.
Once we hit a line containing nothing but the identifier, the string ends.
EOF;
var_dump($my_string);

/*
string(268) "Everything is parsed in the same fashion as a double-quoted string,
but there are advantages. Including text blocks is easier; database queries and HTML output
can benefit from this formatting.
Once we hit a line containing nothing but the identifier, the string ends."
*/

Nowdoc #

Uma string nowdoc é como a versão entre aspas simples de heredoc, embora nem mesmo as sequências de escape mais básicas sejam avaliadas. O identificador no início da string é colocado entre aspas simples.

$my_string = <<< 'EOF'
A similar syntax to heredoc but, similar to single quoted strings,
nothing is parsed (not even escaped apostrophes \' and backslashes \\.)
EOF;
var_dump($my_string);

/*
string(116) "A similar syntax to heredoc but, similar to single quoted strings,
nothing is parsed (not even escaped apostrophes \' and backslashes \\.)"
*/

Chamado #

Callables são qualquer coisa que pode ser chamada como callback. As coisas que podem ser chamadas de “retorno de chamada” são as seguintes:

  • Funções anônimas

  • Funções PHP padrão (nota: não são construções de linguagem)

  • Classes estáticas

  • Classes não estáticas (usando uma sintaxe alternativa)

  • Métodos de Objeto/Classe Específicos

  • Objetos em si, desde que o objeto seja encontrado na chave ‘0’ de um array

    Example Of referencing an object as an array element:

    $obj = new MyClass(); call_user_func([$obj, ‘myCallbackMethod’]);

Callbacks podem ser indicados por callable type hint a partir do PHP 5.4.

$callable = function () {
    return 'value';
};

function call_something(callable $fn) {
    call_user_func($fn);
}

call_something($callable);

Nulo #

PHP representa “sem valor” com a palavra-chave null. É um pouco semelhante ao ponteiro nulo na linguagem C e ao valor NULL no SQL.

Definindo a variável como nula:

$nullvar = null; // directly

function doSomething() {} // this function does not return anything
$nullvar = doSomething(); // so the null is assigned to $nullvar

Verificando se a variável foi definida como nula:

if (is_null($nullvar)) { /* variable is null */ }

if ($nullvar === null) {  /* variable is null */ }

Variável nula vs indefinida #

Se a variável não foi definida ou não foi definida, qualquer teste contra o valor nulo será bem-sucedido, mas também gerará um Aviso: Variável indefinida: nullvar:

$nullvar = null;
unset($nullvar);
if ($nullvar === null) {  /* true but also a Notice is printed */ }
if (is_null($nullvar)) {  /* true but also a Notice is printed */ }

Portanto, valores indefinidos devem ser verificados com isset:

if (!isset($nullvar)) {  /* variable is null or is not even defined */  }

Tipo de fundição #

O PHP geralmente adivinha corretamente o tipo de dados que você pretende usar a partir do contexto em que é usado, mas às vezes é útil forçar manualmente um tipo. Isso pode ser feito prefixando a declaração com o nome do tipo necessário entre parênteses:

$bool = true;
var_dump($bool); // bool(true)

$int = (int) true;
var_dump($int); // int(1)

$string = (string) true;
var_dump($string); // string(1) "1"
$string = (string) false;
var_dump($string); // string(0) ""

$float = (float) true;
var_dump($float); // float(1)

$array = ['x' => 'y'];
var_dump((object) $array); // object(stdClass)#1 (1) { ["x"]=> string(1) "y" } 

$object = new stdClass();
$object->x = 'y';
var_dump((array) $object); // array(1) { ["x"]=> string(1) "y" }

$string = "asdf";
var_dump((unset)$string); // NULL

Mas tenha cuidado: nem todos os tipos de elenco funcionam como se poderia esperar:

// below 3 statements hold for 32-bits systems (PHP_INT_MAX=2147483647)
// an integer value bigger than PHP_INT_MAX is automatically converted to float:
var_dump(       999888777666 ); // float(999888777666)
// forcing to (int) gives overflow:
var_dump((int)  999888777666 ); // int(-838602302)
// but in a string it just returns PHP_INT_MAX
var_dump((int) "999888777666"); // int(2147483647)

var_dump((bool) []);      // bool(false) (empty array)
var_dump((bool) [false]); // bool(true)  (non-empty array)

Recursos #

Um recurso é um tipo especial de variável que faz referência a um recurso externo, como um arquivo, soquete, fluxo, documento ou conexão.

$file = fopen('/etc/passwd', 'r');

echo gettype($file);
# Out: resource

echo $file;
# Out: Resource id #2

Existem diferentes (sub)tipos de recursos. Você pode verificar o tipo de recurso usando get_resource_type():

$file = fopen('/etc/passwd', 'r');
echo get_resource_type($file);
#Out: stream

$sock = fsockopen('www.google.com', 80);
echo get_resource_type($sock);
#Out: stream

Você pode encontrar uma lista completa de tipos de recursos internos aqui.

Tipo de malabarismo #

PHP é uma linguagem de tipagem fraca. Não requer declaração explícita de tipos de dados. O contexto no qual a variável é usada determina seu tipo de dados; a conversão é feita automaticamente:

$a = "2";             // string 
$a = $a + 2;          // integer (4) 
$a = $a + 0.5;        // float (4.5)
$a = 1 + "2 oranges"; // integer (3)

Inteiros #

Os inteiros em PHP podem ser especificados nativamente em base 2 (binário), base 8 (octal), base 10 (decimal) ou base 16 (hexadecimal).

$my_decimal = 42;
$my_binary = 0b101010;
$my_octal = 052;
$my_hexadecimal = 0x2a;

echo ($my_binary + $my_octal) / 2;
// Output is always in decimal: 42

Os inteiros têm 32 ou 64 bits, dependendo da plataforma. A constante PHP_INT_SIZE contém tamanho inteiro em bytes. PHP_INT_MAX e (desde o PHP 7.0) PHP_INT_MIN também estão disponíveis.

printf("Integers are %d bits long" . PHP_EOL, PHP_INT_SIZE * 8);
printf("They go up to %d" . PHP_EOL, PHP_INT_MAX);

Valores inteiros são criados automaticamente conforme necessário a partir de floats, booleanos e strings. Se for necessário um typecast explícito, ele pode ser feito com o cast (int) ou (integer):

$my_numeric_string = "123";
var_dump($my_numeric_string);
// Output: string(3) "123"
$my_integer = (int)$my_numeric_string;
var_dump($my_integer);
// Output: int(123)

O estouro de inteiro será tratado pela conversão para um float:

$too_big_integer = PHP_INT_MAX + 7;
var_dump($too_big_integer);
// Output: float(9.2233720368548E+18)

Não existe operador de divisão inteiro no PHP, mas pode ser simulado usando uma conversão implícita, que sempre ‘arredonda’ apenas descartando a parte flutuante. A partir da versão 7 do PHP, uma função de divisão de inteiros foi adicionada.

$not_an_integer = 25 / 4;
var_dump($not_an_integer);
// Output: float(6.25)
var_dump((int) (25 / 4)); // (see note below)
// Output: int(6)
var_dump(intdiv(25 / 4)); // as of PHP7
// Output: int(6)

(Observe que os parênteses extras em torno de (25 / 4) são necessários porque a conversão (int) tem precedência maior que a divisão)
