Sintaxe #

  • resource curl_init ([ string $ url = NULL ] )
  • bool curl_setopt (recurso $ch, int $option, mixed $value)
  • bool curl_setopt_array (recurso $ch, array $options)
  • misturado curl_exec (recurso $ch)
  • void curl_close (recurso $ch)

Parâmetros #

Parâmetro Detalhes
curl_init – Inicializa uma sessão cURL
URL A url a ser usada na solicitação cURL
curl_setopt – Definir uma opção para uma transferência cURL
ch O identificador cURL (valor de retorno de curl_init())
opção CURLOPT_XXX a ser definido - veja documentação do PHP para a lista de opções e valores aceitáveis ​​
valor O valor a ser definido no identificador cURL para a opção fornecida
curl_exec – Executar uma sessão cURL
ch O identificador cURL (valor de retorno de curl_init())
curl_close – Fechar uma sessão cURL
ch O identificador cURL (valor de retorno de curl_init())

Solicitações POST #

Se você quiser imitar a ação POST do formulário HTML, você pode usar cURL.

// POST data in array
$post = [
    'a' => 'apple',
    'b' => 'banana'
];

// Create a new cURL resource with URL to POST
$ch = curl_init('http://www.example.com');

// We set parameter CURLOPT_RETURNTRANSFER to read output
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

// Let's pass POST data
curl_setopt($ch, CURLOPT_POSTFIELDS, $post);

// We execute our request, and get output in a $response variable
$response = curl_exec($ch);

// Close the connection
curl_close($ch);

Uso básico (solicitações GET) #

cURL é uma ferramenta para transferência de dados com sintaxe de URL. Ele suporta HTTP, FTP, SCP e muitos outros (curl >= 7.19.4). Lembre-se de que você precisa instalar e ativar a extensão cURL para usá-la.

// a little script check is the cURL extension loaded or not
if(!extension_loaded("curl")) {
    die("cURL extension not loaded! Quit Now.");
}

// Actual script start

// create a new cURL resource
// $curl is the handle of the resource
$curl = curl_init();

// set the URL and other options
curl_setopt($curl, CURLOPT_URL, "http://www.example.com");

// execute and pass the result to browser
curl_exec($curl);

// close the cURL resource
curl_close($curl);

Usando cookies #

O cURL pode manter os cookies recebidos em respostas para uso com solicitações subsequentes. Para manipulação simples de cookies de sessão na memória, isso é obtido com uma única linha de código:

curl_setopt($ch, CURLOPT_COOKIEFILE, "");

Nos casos em que você precisa manter os cookies após a destruição do identificador cURL, você pode especificar o arquivo para armazená-los:

curl_setopt($ch, CURLOPT_COOKIEJAR, "/tmp/cookies.txt");

Então, quando você quiser usá-los novamente, passe-os como o arquivo de cookie:

curl_setopt($ch, CURLOPT_COOKIEFILE, "/tmp/cookies.txt");

Lembre-se, porém, que essas duas etapas não são necessárias, a menos que você precise transportar cookies entre diferentes identificadores de cURL. Para a maioria dos casos de uso, configurar CURLOPT_COOKIEFILE para a string vazia é tudo que você precisa.

O manuseio de cookies pode ser usado, por exemplo, para recuperar recursos de um site que requer um login. Este é tipicamente um procedimento de duas etapas. Primeiro, POST para a página de login.

<?php

# create a cURL handle
$ch  = curl_init();

# set the URL (this could also be passed to curl_init() if desired)
curl_setopt($ch, CURLOPT_URL, "https://www.example.com/login.php");

# set the HTTP method to POST
curl_setopt($ch, CURLOPT_POST, true);

# setting this option to an empty string enables cookie handling
# but does not load cookies from a file
curl_setopt($ch, CURLOPT_COOKIEFILE, "");

# set the values to be sent
curl_setopt($ch, CURLOPT_POSTFIELDS, array(
    "username"=>"joe_bloggs",
    "password"=>"$up3r_$3cr3t",
));

# return the response body
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

# send the request
$result = curl_exec($ch);

A segunda etapa (após a verificação de erros padrão) geralmente é uma solicitação GET simples. O importante é reutilizar o identificador cURL existente para a segunda solicitação. Isso garante que os cookies da primeira resposta sejam incluídos automaticamente na segunda solicitação.

# we are not calling curl_init()

# simply change the URL
curl_setopt($ch, CURLOPT_URL, "https://www.example.com/show_me_the_foo.php");

# change the method back to GET
curl_setopt($ch, CURLOPT_HTTPGET, true);

# send the request
$result = curl_exec($ch);

# finished with cURL
curl_close($ch);

# do stuff with $result...

Isso é apenas um exemplo de manipulação de cookies. Na vida real, as coisas costumam ser mais complicadas. Frequentemente, você deve executar um GET inicial da página de login para obter um token de login que precisa ser incluído em seu POST. Outros sites podem bloquear o cliente cURL com base em sua string User-Agent, exigindo que você a altere.

Usando multi_curl para fazer várias solicitações POST #

Às vezes, precisamos fazer muitas solicitações POST para um ou vários endpoints diferentes. Para lidar com este cenário, podemos usar multi_curl.

Primeiramente, criamos quantas solicitações forem necessárias exatamente da mesma forma do exemplo simples e as colocamos em um array.

Usamos o curl_multi_init e adicionamos cada handle a ele.

Neste exemplo, estamos usando 2 endpoints diferentes:

//array of data to POST
$request_contents = array();
//array of URLs
$urls = array();
//array of cURL handles
$chs = array();

//first POST content
$request_contents[] = [
    'a' => 'apple',
    'b' => 'banana'
];
//second POST content
$request_contents[] = [
    'a' => 'fish',
    'b' => 'shrimp'
];
//set the urls
$urls[] = 'http://www.example.com';
$urls[] = 'http://www.example2.com';

//create the array of cURL handles and add to a multi_curl
$mh = curl_multi_init();
foreach ($urls as $key => $url) {
    $chs[$key] = curl_init($url);
    curl_setopt($chs[$key], CURLOPT_RETURNTRANSFER, true);
    curl_setopt($chs[$key], CURLOPT_POST, true);
    curl_setopt($chs[$key], CURLOPT_POSTFIELDS, $request_contents[$key]);

    curl_multi_add_handle($mh, $chs[$key]);
}

Em seguida, usamos curl_multi_exec para enviar as solicitações

//running the requests
$running = null;
do {
  curl_multi_exec($mh, $running);
} while ($running);

//getting the responses
foreach(array_keys($chs) as $key){
    $error = curl_error($chs[$key]);
    $last_effective_URL = curl_getinfo($chs[$key], CURLINFO_EFFECTIVE_URL);
    $time = curl_getinfo($chs[$key], CURLINFO_TOTAL_TIME);
    $response = curl_multi_getcontent($chs[$key]);  // get results
    if (!empty($error)) {
      echo "The request $key return a error: $error" . "\n";
    }
    else {
      echo "The request to '$last_effective_URL' returned '$response' in $time seconds." . "\n";
    }

    curl_multi_remove_handle($mh, $chs[$key]);
}

// close current handler
curl_multi_close($mh);

Um possível retorno para este exemplo poderia ser:

A solicitação para ‘http://www.example.com’ retornou ‘frutas’ em 2 segundos.

A solicitação para ‘http://www.example2.com’ retornou ‘seafood’ em 5 segundos.

Criando e enviando uma solicitação com um método personalizado #

Por padrão, o PHP Curl suporta solicitações GET e POST. Também é possível enviar requisições personalizadas, como DELETE, PUT ou PATCH (ou até métodos não padrão) usando o parâmetro CURLOPT_CUSTOMREQUEST.

$method = 'DELETE'; // Create a DELETE request

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, 1);
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, $method);
$content = curl_exec($ch);
curl_close($ch);

Enviando dados multidimensionais e vários arquivos com CurlFile em uma solicitação #

Digamos que temos um formulário como o abaixo. Queremos enviar os dados para nosso servidor web via AJAX e de lá para um script rodando em um servidor externo.

Portanto, temos entradas normais, um campo de seleção múltipla e uma zona de lançamento de arquivos onde podemos fazer upload de vários arquivos.

Assumindo que a solicitação POST AJAX foi bem-sucedida, obtemos os seguintes dados no site PHP:

// print_r($_POST)

Array
(
    [first_name] => John
    [last_name] => Doe
    [activities] => Array
        (
            [0] => soccer
            [1] => hiking
        )
)

e os arquivos devem ficar assim

// print_r($_FILES)

Array
(
    [upload] => Array
        (
            [name] => Array
                (
                    [0] => my_photo.jpg
                    [1] => my_life.pdf
                )

            [type] => Array
                (
                    [0] => image/jpg
                    [1] => application/pdf
                )

            [tmp_name] => Array
                (
                    [0] => /tmp/phpW5spji
                    [1] => /tmp/phpWgnUeY
                )

            [error] => Array
                (
                    [0] => 0
                    [1] => 0
                )

            [size] => Array
                (
                    [0] => 647548
                    [1] => 643223
                )

        )

)

Até agora tudo bem. Agora queremos enviar esses dados e arquivos para o servidor externo usando cURL com a classe CurlFile

Como o cURL aceita apenas um array simples, mas não multidimensional, temos que nivelar o array $_POST primeiro.

Para fazer isso, você pode usar esta função por exemplo que fornece o seguinte:

// print_r($new_post_array)

Array
(
    [first_name] => John
    [last_name] => Doe
    [activities[0]] => soccer
    [activities[1]] => hiking
)

A próxima etapa é criar objetos CurlFile para os arquivos carregados. Isso é feito pelo seguinte loop:

$files = array();

foreach ($_FILES["upload"]["error"] as $key => $error) {
    if ($error == UPLOAD_ERR_OK) {

        $files["upload[$key]"] = curl_file_create(
            $_FILES['upload']['tmp_name'][$key],
            $_FILES['upload']['type'][$key],
            $_FILES['upload']['name'][$key]
        );
    }
}

curl_file_create é uma função auxiliar da classe CurlFile e cria os objetos CurlFile. Salvamos cada objeto no array $files com chaves chamadas “upload[0]” e “upload[1]” para nossos dois arquivos.

Agora temos que combinar o array de post achatado e o array de arquivos e salvá-lo como $data assim:

$data = $new_post_array + $files;

A última etapa é enviar a solicitação cURL:

$ch = curl_init();

curl_setopt_array($ch, array(
    CURLOPT_POST => 1,
    CURLOPT_URL => "https://api.externalserver.com/upload.php",
    CURLOPT_RETURNTRANSFER => 1,
    CURLINFO_HEADER_OUT => 1,
    CURLOPT_POSTFIELDS => $data
));

$result = curl_exec($ch);

curl_close ($ch);

Como $data agora é um array simples (plano), o cURL envia automaticamente esta solicitação POST com o tipo de conteúdo: multipart/form-data

Em upload.php no servidor externo agora você pode obter os dados e arquivos do post com $_POST e $_FILES como faria normalmente.
