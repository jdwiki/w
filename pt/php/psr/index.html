O PSR (Recomendação de Padrões PHP) é uma série de recomendações reunidas pela FIG (Grupo de Interoperabilidade de Estrutura).

“A ideia por trás do grupo é que os representantes do projeto falem sobre as semelhanças entre nossos projetos e encontrem maneiras de trabalharmos juntos” - FIG FAQ

Os PSRs podem estar nos seguintes estados: Aceito, Revisão, Rascunho ou Descontinuado.

PSR-4: Autoloader #

PSR-4 é uma recomendação aceita que descreve o padrão para classes de carregamento automático por meio de nomes de arquivos. Esta recomendação é recomendada como alternativa ao anterior (e agora obsoleto) PSR-0.

O nome de classe totalmente qualificado deve corresponder ao seguinte requisito:

 \<NamespaceName>(\<SubNamespaceNames>)*\<ClassName>
  • Ele DEVE conter um namespace de fornecedor de nível superior (por exemplo: Alphabet)
  • PODE conter um ou mais sub-namespaces (por exemplo: Google\AdWord)
  • DEVE conter um nome de classe final (por exemplo: KeywordPlanner)

Assim, o nome final da classe seria Alphabet\Google\AdWord\KeywordPlanner. O nome de classe totalmente qualificado também deve ser traduzido em um caminho de arquivo significativo, portanto, Alphabet\Google\AdWord\KeywordPlanner estaria localizado em [path_to_source]/Alphabet/Google/AdWord/KeywordPlanner.php

A partir do PHP 5.3.0, uma função personalizada de autoloader pode ser definida para carregar arquivos com base no caminho e padrão de nome de arquivo que você define.

# Edit your php to include something like:
spl_autoload_register(function ($class) { include 'classes/' . $class . '.class.php';});

Substituindo o local (‘classes/’) e a extensão do nome do arquivo (’.class.php’) por valores que se aplicam à sua estrutura.

Gerenciador de pacotes Composer [suporta PSR-4](https://getcomposer.org/doc/01-basic-usage.md #autoloading) o que significa que, se você seguir o padrão, poderá carregar suas classes em seu projeto automaticamente usando o autoloader do fornecedor do Composer.

# Edit the composer.json file to include
{
    "autoload": {
        "psr-4": {
            "Alphabet\\": "[path_to_source]"
        }
    }
}

Regenerar o arquivo do carregador automático

$ composer dump-autoload

Agora no seu código você pode fazer o seguinte:

<?php

require __DIR__ . '/vendor/autoload.php';
$KeywordPlanner = new Alphabet\Google\AdWord\KeywordPlanner();

PSR-1: Padrão Básico de Codificação #

PSR-1 é uma recomendação aceita e descreve uma recomendação padrão básica de como o código deve ser escrito.

  • Descreve convenções de nomenclatura para classes, métodos e constantes.
  • Torna obrigatória a adoção das recomendações PSR-0 ou PSR-4.
  • Indica quais tags PHP usar: <?php e <?= mas não <?.
  • Especifica qual codificação de arquivo usar (UTF8).
  • Também afirma que os arquivos devem declarar novos símbolos (classes, funções, constantes, etc.) e não causar outros efeitos colaterais, ou executar lógica com efeitos colaterais e não definir símbolos, mas fazer as duas coisas.

PSR-8: Interface Huggable #

PSR-8 é um PSR falso (atualmente em Draft) proposto por Larry Garfield como uma piada de 1º de abril de 2014.

O rascunho descreve como definir uma interface para tornar um objeto ‘Huggable’.

Trecho do contorno do código:

<?php

namespace Psr\Hug;

/**
 * Defines a huggable object.
 *
 * A huggable object expresses mutual affection with another huggable object.
 */
interface Huggable
{

    /**
     * Hugs this object.
     *
     * All hugs are mutual. An object that is hugged MUST in turn hug the other
     * object back by calling hug() on the first parameter. All objects MUST
     * implement a mechanism to prevent an infinite loop of hugging.
     *
     * @param Huggable $h
     *   The object that is hugging this object.
     */
    public function hug(Huggable $h);
}
