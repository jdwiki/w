JSON (JavaScript Object Notation) é uma maneira independente de plataforma e linguagem de serializar objetos em texto simples. Como é frequentemente usado na web e PHP também, existe uma extensão básica para trabalhar com JSON em PHP.

Sintaxe #

  • string json_encode ( $ valor misto [, int $ opções = 0 [, int $ profundidade = 512 ]] )
  • json_decode misto ( string $ json [, bool $ assoc = false [, int $ depth = 512 [, int $ options = 0 ]]] )

Parâmetros #

Parâmetro Detalhes
json_encode -
valor O valor que está sendo codificado. Pode ser qualquer tipo, exceto um recurso. Todos os dados de string devem ser codificados em UTF-8.
options Bitmask consistindo em JSON_HEX_QUOT, JSON_HEX_TAG, JSON_HEX_AMP, JSON_HEX_APOS, JSON_NUMERIC_CHECK, JSON_PRETTY_PRINT, JSON_UNESCAPED_SLASHES, JSON_FORCE_OBJECT, JSON_PRESERVE_ZERO_FRACTION, JSON_UNESCAPED_UNICODE, JSON_PARTIAL_OUTPUT_ON_ERRRATION. O comportamento dessas constantes é descrito na página constantes JSON.
profundidade Defina a profundidade máxima. Deve ser maior que zero.
json_decode -
json A string json sendo decodificada. Esta função só funciona com strings codificadas em UTF-8.
assoc A função deve retornar array associativo em vez de objetos.
options Bitmask de opções de decodificação JSON. Atualmente, apenas JSON_BIGINT_AS_STRING é suportado (o padrão é converter inteiros grandes como floats)
  • json_decode o tratamento de JSON inválido é muito esquisito e é muito difícil determinar com segurança se a decodificação foi bem-sucedida, json_decode retorna null para entrada inválida, mesmo que null também seja um objeto perfeitamente válido para o JSON decodificar. Para evitar esses problemas, você deve sempre chamar json_last_error toda vez que usá-lo.

Decodificando uma string JSON #

A função json_decode() pega uma string codificada em JSON como seu primeiro parâmetro e a analisa em uma variável PHP.

Normalmente, json_decode() retornará um objeto de \stdClass se o item de nível superior no objeto JSON for um dicionário ou um array indexado se o objeto JSON for um array . Ele também retornará valores escalares ou NULL para certos valores escalares, como strings simples, "true", "false" e "null". Ele também retorna NULL em qualquer erro.

// Returns an object (The top level item in the JSON string is a JSON dictionary)
$json_string = '{"name": "Jeff", "age": 20, "active": true, "colors": ["red", "blue"]}';
$object = json_decode($json_string);
printf('Hello %s, You are %s years old.', $object->name, $object->age);
#> Hello Jeff, You are 20 years old.

// Returns an array (The top level item in the JSON string is a JSON array)
$json_string = '["Jeff", 20, true, ["red", "blue"]]';
$array = json_decode($json_string);
printf('Hello %s, You are %s years old.', $array[0], $array[1]);

Use var_dump() para visualizar os tipos e valores de cada propriedade no objeto que decodificamos acima.

// Dump our above $object to view how it was decoded
var_dump($object);

Saída (observe os tipos de variáveis):

class stdClass#2 (4) {
 ["name"] => string(4) "Jeff"
 ["age"] => int(20)
 ["active"] => bool(true)
 ["colors"] =>
   array(2) {
     [0] => string(3) "red"
     [1] => string(4) "blue"
   }
}

Observação: A variável types em JSON foi convertida para seu equivalente PHP.

Para retornar um matriz associativa para objetos JSON em vez de retornar um objeto, passe true como o segundo parâmetro para json_decode().

$json_string = '{"name": "Jeff", "age": 20, "active": true, "colors": ["red", "blue"]}';
$array = json_decode($json_string, true); // Note the second parameter
var_dump($array);

Saída (observe a estrutura associativa da matriz):

array(4) {
  ["name"] => string(4) "Jeff"
  ["age"] => int(20)
  ["active"] => bool(true)
  ["colors"] =>
  array(2) {
    [0] => string(3) "red"
    [1] => string(4) "blue"
  }
}

O segundo parâmetro ($assoc) não tem efeito se a variável a ser retornada não for um objeto.

Nota: Se você usar o parâmetro $assoc, você perderá a distinção entre um array vazio e um objeto vazio. Isso significa que executar json_encode() em sua saída decodificada novamente resultará em uma estrutura JSON diferente.

Se a string JSON tiver uma “profundidade” de mais de 512 elementos (20 elementos em versões anteriores a 5.2.3 ou 128 na versão 5.2.3) em recursão, a função json_decode() retornará NULL. Nas versões 5.3 ou posteriores, esse limite pode ser controlado usando o terceiro parâmetro ($depth), conforme discutido abaixo.

De acordo com o manual:

PHP implementa um superconjunto de JSON conforme especificado no original » RFC 4627 - ele também codificará e decodificará tipos escalares e NULL. A RFC 4627 só oferece suporte a esses valores quando eles estão aninhados dentro de uma matriz ou objeto. Embora esse superconjunto seja consistente com a definição expandida de “texto JSON” no mais recente » RFC 7159 (que visa substituir o RFC 4627) e » ECMA-404, isso pode causar problemas de interoperabilidade com analisadores JSON mais antigos que aderem estritamente à RFC 4627 ao codificar um único valor escalar.

Isso significa que, por exemplo, uma string simples será considerada um objeto JSON válido em PHP:

$json = json_decode('"some string"', true);
var_dump($json, json_last_error_msg());

Resultado:

string(11) "some string"
string(8) "No error"

Mas strings simples, não em uma matriz ou objeto, não fazem parte do padrão RFC 4627. Como resultado, verificadores online como JSLint, JSON Formatter & Validator (no modo RFC 4627) apresentarão um erro.

Há um terceiro parâmetro $depth para a profundidade da recursão (o valor padrão é 512), que significa a quantidade de objetos aninhados dentro do objeto original a ser decodificado.

Há um quarto parâmetro $options. Atualmente aceita apenas um valor, JSON_BIGINT_AS_STRING. O comportamento padrão (que deixa de fora esta opção) é converter grandes inteiros em floats em vez de strings.

Variantes não minúsculas inválidas dos literais verdadeiro, falso e nulo não são mais aceitos como entrada válida.

Então este exemplo:

var_dump(json_decode('tRue'), json_last_error_msg());
var_dump(json_decode('tRUe'), json_last_error_msg());
var_dump(json_decode('tRUE'), json_last_error_msg());
var_dump(json_decode('TRUe'), json_last_error_msg());
var_dump(json_decode('TRUE'), json_last_error_msg());
var_dump(json_decode('true'), json_last_error_msg());

Antes do PHP 5.6:

bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"

E depois:

NULL
string(12) "Syntax error"
NULL
string(12) "Syntax error"
NULL
string(12) "Syntax error"
NULL
string(12) "Syntax error"
NULL
string(12) "Syntax error"
bool(true)
string(8) "No error"

Comportamento semelhante ocorre para false e ​​null.

Observe que json_decode() retornará NULL se a string não puder ser convertida.

$json = "{'name': 'Jeff', 'age': 20 }" ;  // invalid json 

$person = json_decode($json);
echo $person->name;    //  Notice: Trying to get property of non-object: returns null
echo json_last_error();     
#  4 (JSON_ERROR_SYNTAX)
echo json_last_error_msg(); 
#  unexpected character

Não é seguro confiar apenas no valor de retorno sendo NULL para detectar erros. Por exemplo, se a string JSON não contiver nada além de "null", json_decode() retornará null, mesmo que nenhum erro tenha ocorrido.

Codificando uma string JSON #

A função json_encode converterá um array PHP (ou, desde o PHP 5.4, um objeto que implementa a interface JsonSerializable) em uma string codificada em JSON. Ele retorna uma string codificada em JSON em caso de sucesso ou FALSE em caso de falha.

$array = [
    'name' => 'Jeff',
    'age' => 20,
    'active' => true,
    'colors' => ['red', 'blue'],
    'values' => [0=>'foo', 3=>'bar'],
];

Durante a codificação, os tipos de dados PHP string, integer e boolean são convertidos em seu equivalente JSON. Arrays associativos são codificados como objetos JSON e – quando chamados com argumentos padrão – arrays indexados são codificados como arrays JSON. (A menos que as chaves da matriz não sejam uma sequência numérica contínua começando em 0, nesse caso a matriz será codificada como um objeto JSON.)

echo json_encode($array);

Resultado:

{"name":"Jeff","age":20,"active":true,"colors":["red","blue"],"values":{"0":"foo","3":"bar"}}

Argumentos #

Desde o PHP 5.3, o segundo argumento para json_encode é uma máscara de bits que pode ser um ou mais dos seguintes.

Como com qualquer bitmask, eles podem ser combinados com o operador binário OR |.

JSON_FORCE_OBJECT #

Força a criação de um objeto em vez de um array

$array = ['Joel', 23, true, ['red', 'blue']];
echo json_encode($array);
echo json_encode($array, JSON_FORCE_OBJECT);

Resultado:

["Joel",23,true,["red","blue"]]
{"0":"Joel","1":23,"2":true,"3":{"0":"red","1":"blue"}}

JSON_HEX_TAG, JSON_HEX_AMP, JSON_HEX_APOS, [JSON_HEX_QUOT][json-hex -quot] #

Garante as seguintes conversões durante a codificação:

Constante Entrada Saída
JSON_HEX_TAG < \u003C
JSON_HEX_TAG > \u003E
JSON_HEX_AMP & \u0026
JSON_HEX_APOS ' \u0027
JSON_HEX_QUOT " \u0022 
$array = ["tag"=>"<>", "amp"=>"&", "apos"=>"'", "quot"=>"\""];
echo json_encode($array);
echo json_encode($array, JSON_HEX_TAG | JSON_HEX_AMP | JSON_HEX_APOS | JSON_HEX_QUOT);

Resultado:

{"tag":"<>","amp":"&","apos":"'","quot":"\""}
{"tag":"\u003C\u003E","amp":"\u0026","apos":"\u0027","quot":"\u0022"}

JSON_NUMERIC_CHECK #

Garante que as strings numéricas sejam convertidas em números inteiros.

$array = ['23452', 23452];
echo json_encode($array);
echo json_encode($array, JSON_NUMERIC_CHECK);

Resultado:

["23452",23452]    
[23452,23452]

JSON_PRETTY_PRINT #

Torna o JSON facilmente legível

$array = ['a' => 1, 'b' => 2, 'c' => 3, 'd' => 4];
echo json_encode($array);
echo json_encode($array, JSON_PRETTY_PRINT);

Resultado:

{"a":1,"b":2,"c":3,"d":4}
{
    "a": 1,
    "b": 2,
    "c": 3,
    "d": 4
}

JSON_UNESCAPED_SLASHES #

Inclui barras / sem escape na saída

$array = ['filename' => 'example.txt', 'path' => '/full/path/to/file/'];
echo json_encode($array);
echo json_encode($array, JSON_UNESCAPED_SLASHES);

Resultado:

{"filename":"example.txt","path":"\/full\/path\/to\/file"}
{"filename":"example.txt","path":"/full/path/to/file"}

JSON_UNESCAPED_UNICODE #

Inclui caracteres codificados em UTF8 na saída em vez de strings codificadas em \u

$blues = ["english"=>"blue", "norwegian"=>"blå", "german"=>"blau"];
echo json_encode($blues);
echo json_encode($blues, JSON_UNESCAPED_UNICODE);

Resultado:

{"english":"blue","norwegian":"bl\u00e5","german":"blau"}
{"english":"blue","norwegian":"blå","german":"blau"}

JSON_PARTIAL_OUTPUT_ON_ERROR #

Permite que a codificação continue se alguns valores não codificáveis ​​forem encontrados.

$fp = fopen("foo.txt", "r");
$array = ["file"=>$fp, "name"=>"foo.txt"];
echo json_encode($array); // no output
echo json_encode($array, JSON_PARTIAL_OUTPUT_ON_ERROR);

Resultado:

{"file":null,"name":"foo.txt"}

JSON_PRESERVE_ZERO_FRACTION #

Garante que os floats sejam sempre codificados como floats.

$array = [5.0, 5.5];
echo json_encode($array);
echo json_encode($array, JSON_PRESERVE_ZERO_FRACTION);

Resultado:

[5,5.5]
[5.0,5.5]

JSON_UNESCAPED_LINE_TERMINATORS #

Quando usado com JSON_UNESCAPED_UNICODE, reverte para o comportamento de versões mais antigas do PHP e não escapa os caracteres U+2028 LINE SEPARATOR e U+2029 PARAGRAPH SEPARATOR. Embora válidos em JSON, esses caracteres não são válidos em JavaScript, portanto, o comportamento padrão de JSON_UNESCAPED_UNICODE foi alterado na versão 7.1.

$array = ["line"=>"\xe2\x80\xa8", "paragraph"=>"\xe2\x80\xa9"];
echo json_encode($array, JSON_UNESCAPED_UNICODE);
echo json_encode($array, JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_LINE_TERMINATORS);

Resultado:

{"line":"\u2028","paragraph":"\u2029"}
{"line":" ","paragraph":" "}

Depuração de erros JSON #

Quando json_encode ou json_decode falha ao analisar a string fornecida, ele retornará false. O próprio PHP não gerará nenhum erro ou aviso quando isso acontecer, o ônus é do usuário usar as funções json_last_error() e json_last_error_msg() para verificar se ocorreu um erro e agir de acordo em seu aplicativo (depurar, mostrar uma mensagem de erro, etc.).

O exemplo a seguir mostra um erro comum ao trabalhar com JSON, uma falha ao decodificar/codificar uma string JSON (devido à passagem de uma string codificada em UTF-8 incorreta, por exemplo).

// An incorrectly formed JSON string
$jsonString = json_encode("{'Bad JSON':\xB1\x31}");

if (json_last_error() != JSON_ERROR_NONE) {
    printf("JSON Error: %s", json_last_error_msg());
}

#> JSON Error: Malformed UTF-8 characters, possibly incorrectly encoded

json_last_error_msg #

json_last_error_msg() retorna uma mensagem legível do último erro que ocorreu ao tentar codificar/decodificar uma string.

  • Esta função sempre retornará uma string, mesmo que não tenha ocorrido nenhum erro. A string padrão sem erro é Sem erro
  • Ele retornará false se algum outro erro (desconhecido) ocorrer
  • Cuidado ao usar isso em loops, pois json_last_error_msg será substituído em cada iteração.

Você só deve usar esta função para obter a mensagem para exibição, não para testar em declarações de controle.

// Don't do this:
if (json_last_error_msg()){} // always true (it's a string)
if (json_last_error_msg() != "No Error"){} // Bad practice

// Do this: (test the integer against one of the pre-defined constants)
if (json_last_error() != JSON_ERROR_NONE) {
    // Use json_last_error_msg to display the message only, (not test against it)
    printf("JSON Error: %s", json_last_error_msg());
}

Esta função não existe antes do PHP 5.5. Aqui está uma implementação de polyfill:

if (!function_exists('json_last_error_msg')) {
    function json_last_error_msg() {
        static $ERRORS = array(
            JSON_ERROR_NONE => 'No error',
            JSON_ERROR_DEPTH => 'Maximum stack depth exceeded',
            JSON_ERROR_STATE_MISMATCH => 'State mismatch (invalid or malformed JSON)',
            JSON_ERROR_CTRL_CHAR => 'Control character error, possibly incorrectly encoded',
            JSON_ERROR_SYNTAX => 'Syntax error',
            JSON_ERROR_UTF8 => 'Malformed UTF-8 characters, possibly incorrectly encoded'
        );

        $error = json_last_error();
        return isset($ERRORS[$error]) ? $ERRORS[$error] : 'Unknown error';
    }
}

json_last_error #

json_last_error() retorna um inteiro mapeado para uma das constantes predefinidas fornecidas pelo PHP.

Constante Significado
JSON_ERROR_NONE Não ocorreu nenhum erro
JSON_ERROR_DEPTH A profundidade máxima da pilha foi excedida
JSON_ERROR_STATE_MISMATCH JSON inválido ou malformado
JSON_ERROR_CTRL_CHAR Erro de caractere de controle, possivelmente codificado incorretamente
JSON_ERROR_SYNTAX Erro de sintaxe (desde PHP 5.3.3)
JSON_ERROR_UTF8 Caracteres UTF-8 malformados, possivelmente codificados incorretamente (desde PHP 5.5.0)
JSON_ERROR_RECURSION Uma ou mais referências recursivas no valor a ser codificado
JSON_ERROR_INF_OR_NAN Um ou mais valores NAN ou INF no valor a ser codificado
JSON_ERROR_UNSUPPORTED_TYPE Um valor de um tipo que não pode ser codificado foi fornecido

Usando JsonSerializable em um objeto #

Ao construir APIs REST, pode ser necessário reduzir as informações de um objeto a ser passado para o aplicativo cliente. Para isso, este exemplo ilustra como usar a interface JsonSerialiazble.

Neste exemplo, a classe User na verdade estende um objeto de modelo de banco de dados de um ORM hipotético.

class User extends Model implements JsonSerializable {
    public $id;
    public $name;
    public $surname;
    public $username;
    public $password;
    public $email;
    public $date_created;
    public $date_edit;
    public $role;
    public $status;

    public function jsonSerialize() {
        return [
            'name' => $this->name,
            'surname' => $this->surname,
            'username' => $this->username
        ];
    }
}

Adicione a implementação JsonSerializable à classe, fornecendo o método jsonSerialize().

public function jsonSerialize()

Agora, em seu controlador de aplicativo ou script, ao passar o objeto User para json_encode(), você obterá o array codificado em json de retorno do método jsonSerialize() em vez do objeto inteiro.

json_encode($User);

Retornará:

{"name":"John", "surname":"Doe", "username" : "TestJson"}

exemplo de valores de propriedades. #

Isso reduzirá a quantidade de dados retornados de um endpoint RESTful e permitirá excluir propriedades de objetos de uma representação json.

Usando propriedades privadas e protegidas com json_encode()

Para evitar o uso de JsonSerializable, também é possível usar propriedades privadas ou protegidas para ocultar informações de classe da saída json_encode(). A classe então não precisa implementar \JsonSerializable.

A função json_encode() só codificará propriedades públicas de um classe em JSON.

<?php

class User {
    // private properties only within this class
    private $id;
    private $date_created;
    private $date_edit;

    // properties used in extended classes
    protected $password;
    protected $email;
    protected $role;
    protected $status;

    // share these properties with the end user        
    public $name;
    public $surname;
    public $username;

    // jsonSerialize() not needed here
}        

$theUser = new User();

var_dump(json_encode($theUser));

Resultado: #

string(44) "{"name":null,"surname":null,"username":null}"

Cabeçalho json e a resposta retornada #

Adicionando um cabeçalho com tipo de conteúdo como JSON:

<?php
 $result = array('menu1' => 'home', 'menu2' => 'code php', 'menu3' => 'about');

//return the json response :
header('Content-Type: application/json');  // <-- header declaration
echo json_encode($result, true);    // <--- encode
exit();

O cabeçalho está lá para que seu aplicativo possa detectar quais dados foram retornados e como ele deve lidar com eles.
Observe que : o cabeçalho do conteúdo é apenas uma informação sobre o tipo de dados retornados.

Se você estiver usando UTF-8, você pode usar:

header("Content-Type: application/json;charset=utf-8");
