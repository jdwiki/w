Cartilha de carregamento automático

  • Carregamento manual de classes com require

Sintaxe #

  • exigir
  • spl_autoload_require

O carregamento automático, como parte de uma estratégia de estrutura, facilita a quantidade de código clichê que você precisa escrever.

Definição de classe inline, sem necessidade de carregamento #

// zoo.php
class Animal {
    public function eats($food) {
        echo "Yum, $food!";
    }
}

$animal = new Animal();
$animal->eats('meat');

O PHP sabe o que é Animal antes de executar new Animal, porque o PHP lê os arquivos fonte de cima para baixo. Mas e se quiséssemos criar novos animais em muitos lugares, não apenas no arquivo de origem onde está definido? Para fazer isso, precisamos carregar a definição da classe.

Carregamento manual de classes com require #

// Animal.php
class Animal {
    public function eats($food) {
         echo "Yum, $food!";
    }
}

// zoo.php
require 'Animal.php';
$animal = new Animal;
$animal->eats('slop');

// aquarium.php
require 'Animal.php';
$animal = new Animal;
$animal->eats('shrimp');

Aqui temos três arquivos. Um arquivo (“Animal.php”) define a classe. Este arquivo não tem efeitos colaterais além de definir a classe e mantém todo o conhecimento sobre um “Animal” em um só lugar. É facilmente controlado por versão. É facilmente reutilizado.

Dois arquivos consomem o arquivo “Animal.php” manualmente require-ing o arquivo. Novamente, o PHP lê os arquivos fonte de cima para baixo, então o require vai e encontra o arquivo “Animal.php” e disponibiliza a definição de classe Animal antes de chamar new Animal.

Agora imagine que tivéssemos dezenas ou centenas de casos em que queríamos realizar ’novo Animal’. Isso exigiria (trocadilhos) muitas, muitas declarações require que são muito tediosas para codificar.

O carregamento automático substitui o carregamento manual de definição de classe #

// autoload.php
spl_autoload_register(function ($class) {
    require_once "$class.php";
});

// Animal.php
class Animal {
    public function eats($food) {
         echo "Yum, $food!";
    }
}

// zoo.php
require 'autoload.php';
$animal = new Animal;
$animal->eats('slop');

// aquarium.php
require 'autoload.php';
$animal = new Animal;
$animal->eats('shrimp');

Compare isso com os outros exemplos. Observe como require "Animal.php" foi substituído por require "autoload.php". Ainda estamos incluindo um arquivo externo em tempo de execução, mas em vez de incluir uma definição de classe específica, estamos incluindo uma lógica que pode incluir qualquer classe. É um nível de indireção que facilita nosso desenvolvimento. Em vez de escrever um require para cada classe que precisamos, escrevemos um require para todas as classes. Podemos substituir N require por 1 require.

A mágica acontece com spl_autoload_register. Esta função PHP recebe um encerramento e adiciona o encerramento a uma fila de encerramentos. Quando o PHP encontra uma classe para a qual não tem definição, o PHP entrega o nome da classe para cada encerramento na fila. Se a classe existir após chamar um encerramento, o PHP retornará ao seu negócio anterior. Se a classe não existir depois de tentar a fila inteira, o PHP trava com “Class ‘Whatever’ not found”.

Carregamento automático como parte de uma solução de estrutura #

// autoload.php
spl_autoload_register(function ($class) {
    require_once "$class.php";
});

// Animal.php
class Animal {
    public function eats($food) {
         echo "Yum, $food!";
    }
}

// Ruminant.php
class Ruminant extends Animal {
    public function eats($food) {
        if ('grass' === $food) {
            parent::eats($food);
        } else {
            echo "Yuck, $food!";
        }
    }
}

// Cow.php
class Cow extends Ruminant {
}

// pasture.php
require 'autoload.php';
$animal = new Cow;
$animal->eats('grass');

Graças ao nosso autoloader genérico, temos acesso a qualquer classe que siga nossa convenção de nomenclatura do autoloader. Neste exemplo, nossa convenção é simples: a classe desejada deve ter um arquivo no mesmo diretório nomeado para a classe e terminando em “.php”. Observe que o nome da classe corresponde exatamente ao nome do arquivo.

Sem autoload, teríamos que manualmente require classes base. Se construíssemos um zoológico inteiro de animais, teríamos milhares de declarações de exigência que poderiam ser substituídas mais facilmente por um único carregador automático.

Na análise final, o carregamento automático do PHP é um mecanismo para ajudá-lo a escrever menos código mecânico para que você possa se concentrar na solução de problemas de negócios. Tudo o que você precisa fazer é definir uma estratégia que mapeie o nome da classe para o nome do arquivo. Você pode lançar sua própria estratégia de carregamento automático, como feito aqui. Ou você pode usar qualquer um dos padrões que a comunidade PHP adotou: PSR-0 ou PSR-4. Ou você pode usar composer para definir e gerenciar genericamente essas dependências.

Carregamento automático com o Composer #

O Composer gera um arquivo vendor/autoload.php.

Você pode simplesmente incluir este arquivo e obterá o carregamento automático gratuitamente.

requer __DIR__ . '/vendor/autoload.php';

Isso facilita muito o trabalho com dependências de terceiros.

Você também pode adicionar seu próprio código ao Autoloader adicionando uma seção de autoload ao seu composer.json.

{
    "autoload": {
        "psr-4": {"YourApplicationNamespace\\": "src/"}
    }
}

Nesta seção, você define os mapeamentos de carregamento automático. Neste exemplo é um mapeamento PSR-4 de um namespace para um diretório: o diretório /src reside na pasta raiz do seu projeto, no mesmo nível que o diretório /vendor. Um exemplo de nome de arquivo seria src/Foo.php contendo uma classe YourApplicationNamespace\Foo.

Importante: Depois de adicionar novas entradas à seção de carregamento automático, você deve executar novamente o comando dump-autoload para gerar novamente e atualizar o arquivo vendor/autoload.php com o nova informação.

Além do carregamento automático PSR-4, o Composer também suporta o carregamento automático PSR-0, classmap e files. Consulte a referência de carregamento automático para obter mais informações.

Ao incluir o arquivo /vendor/autoload.php, ele retornará uma instância do Composer Autoloader. Você pode armazenar o valor de retorno da chamada de inclusão em uma variável e adicionar mais namespaces. Isso pode ser útil para carregar automaticamente classes em um conjunto de testes, por exemplo.

$loader = require __DIR__ . '/vendor/autoload.php';
$loader->add('Application\\Test\\', __DIR__);
