Superglobais são variáveis ​​internas que estão sempre disponíveis em todos os escopos.

Várias variáveis ​​predefinidas em PHP são “superglobais”, o que significa que estão disponíveis em todos os escopos ao longo de um script. Não há necessidade de fazer global $variable; para acessá-los dentro de funções ou métodos.

Sub-globais explicados #

Introdução #

Simplificando, essas são variáveis ​​que estão disponíveis no escopo todos em seus scripts.

Isso significa que não há necessidade de passá-los como parâmetros em suas funções ou armazená-los fora de um bloco de código para disponibilizá-los em diferentes escopos.

O que é uma superglobal?? #

Se você está pensando que eles são como super-heróis - eles não são.

A partir do PHP versão 7.1.3 existem 9 variáveis ​​superglobais. Eles são os seguintes:

  • $GLOBALS - Referencia todas as variáveis ​​disponíveis no escopo global
  • $_SERVER - Informações do servidor e do ambiente de execução
  • $_GET - Variáveis ​​HTTP GET
  • $_POST - Variáveis ​​HTTP POST
  • $_FILES - Variáveis ​​de upload de arquivo HTTP
  • $_COOKIE - Cookies HTTP
  • $_SESSION - Variáveis ​​de sessão
  • $_REQUEST - Variáveis ​​de solicitação HTTP
  • $_ENV - Variáveis ​​de ambiente

Consulte a documentação.

Hora de algumas explicações sobre esses superheróisglobais.

$GLOBALS #

Um array associativo contendo referências a todas as variáveis ​​que estão atualmente definidas no escopo global do script. Os nomes das variáveis ​​são as chaves do array.

Código

$myGlobal = "global"; // declare variable outside of scope

function test()
{
    $myLocal = "local"; // declare variable inside of scope
    // both variables are printed
    var_dump($myLocal);
    var_dump($GLOBALS["myGlobal"]);
}

test(); // run function
// only $myGlobal is printed since $myLocal is not globally scoped
var_dump($myLocal);
var_dump($myGlobal);

Resultado

string 'local' (length=5)
string 'global' (length=6)
null
string 'global' (length=6)

No exemplo acima $myLocal não é exibido pela segunda vez porque é declarado dentro da função test() e então destruído após o fechamento da função.

Tornando-se global #

Para remediar isso, existem duas opções.

Opção 1: palavra-chave global

function test()
{
    global $myLocal;
    $myLocal = "local";
    var_dump($myLocal);
    var_dump($GLOBALS["myGlobal"]);
}

A palavra-chave global é um prefixo em uma variável que a força a fazer parte do escopo global.

Observe que você não pode atribuir um valor a uma variável na mesma instrução que a palavra-chave global. Por isso, tive que atribuir um valor abaixo. (É possível se você remover novas linhas e espaços, mas não acho legal. global $myLocal; $myLocal = "local").

Opção dois: $GLOBALS array

function test()
{
    $GLOBALS["myLocal"] = "local";
    $myLocal = $GLOBALS["myLocal"];
    var_dump($myLocal);
    var_dump($GLOBALS["myGlobal"]);
}

Neste exemplo eu reatribuí $myLocal o valor de $GLOBAL["myLocal"] já que acho mais fácil escrever um nome de variável em vez do array associativo.

$_SERVER é um array contendo informações como cabeçalhos, caminhos e localizações de script. As entradas neste array são criadas pelo servidor web. Não há garantia de que cada servidor web fornecerá qualquer um destes; servidores podem omitir alguns, ou fornecer outros não listados aqui. Dito isso, um grande número dessas variáveis ​​é contabilizado na especificação CGI/1.1, então você deve poder esperar por elas.

  • HTTP_HOST - O endereço do host. Isso retornaria www.example.com
  • HTTP_USER_AGENT - Conteúdo do agente do usuário. Esta é uma string que contém todas as informações sobre o navegador do cliente, incluindo o sistema operacional.

$_GET #

Uma matriz associativa de variáveis ​​passadas para o script atual por meio dos parâmetros de URL.

$_GET é um array que contém todos os parâmetros de URL; estes são o que está depois do ? na URL.

Usando http://www.example.com/index.php?myVar=myVal como exemplo. Esta informação desta URL pode ser obtida acessando neste formato $_GET["myVar"] e o resultado disso será myVal.

Usando algum código para aqueles que não gostam de ler.

// URL = http://www.example.com/index.php?myVar=myVal
echo $_GET["myVar"] == "myVal" ? "true" : "false"; // returns "true"

O exemplo acima usa o [operador ternário](https://www.wikiod.com/pt/php/operadores#Operador Ternário (?:)).

Isso mostra como você pode acessar o valor da URL usando a superglobal $_GET.

Agora outro exemplo! suspiro

// URL = http://www.example.com/index.php?myVar=myVal&myVar2=myVal2
echo $_GET["myVar"]; // returns "myVal"
echo $_GET["myVar2"]; // returns "myVal2"

É possível enviar múltiplas variáveis ​​através da URL separando-as com um caractere e comercial (&).

Risco de segurança É muito importante não enviar nenhuma informação sensível pela URL, pois ela ficará no histórico do computador e ficará visível para qualquer pessoa que possa acessar esse navegador.

$_POST #

Uma matriz associativa de variáveis ​​passadas para o script atual por meio do método HTTP POST ao usar application/x-www-form-urlencoded ou multipart/form-data como o HTTP Content-Type na solicitação.

Muito semelhante a $_GET em que os dados são enviados de um lugar para outro.

Vou começar indo direto para um exemplo. (Eu omiti o atributo de ação, pois isso enviará as informações para a página em que o formulário está).

<form method="POST">
    <input type="text" name="myVar" value="myVal" />
    <input type="submit" name="submit" value="Submit" />
</form>

Acima está um formulário básico para o qual os dados podem ser enviados. Em um ambiente real, o atributo value não seria definido, o que significa que o formulário ficaria em branco. Isso enviaria qualquer informação inserida pelo usuário.

echo $_POST["myVar"]); // returns "myVal"

Risco de segurança O envio de dados via POST também não é seguro. O uso de HTTPS garantirá que os dados sejam mantidos mais seguros.

$_FILES #

Uma matriz associativa de itens carregados no script atual por meio do método HTTP POST. A estrutura dessa matriz é descrita na seção uploads do método POST.

Vamos começar com um formulário básico.

<form method="POST" enctype="multipart/form-data">
    <input type="file" name="myVar" />
    <input type="submit" name="Submit" />
</form>

Note que eu omiti o atributo action (de novo!). Além disso, adicionei enctype="multipart/form-data", isso é importante para qualquer formulário que esteja lidando com uploads de arquivos.

// ensure there isn't an error
if ($_FILES["myVar"]["error"] == UPLOAD_ERR_OK)
{
    $folderLocation = "myFiles"; // a relative path. (could be "path/to/file" for example)
    
    // if the folder doesn't exist then make it
    if (!file_exists($folderLocation)) mkdir($folderLocation);

    // move the file into the folder
    move_uploaded_file($_FILES["myVar"]["tmp_name"], "$folderLocation/" . basename($_FILES["myVar"]["name"]));
}

Isso é usado para fazer upload de um arquivo. Às vezes você pode querer carregar mais de um arquivo. Existe um atributo para isso, é chamado de múltiplo. Há um atributo para praticamente qualquer coisa. Sinto muito

Abaixo está um exemplo de um formulário enviando vários arquivos.

<form method="POST" enctype="multipart/form-data">
    <input type="file" name="myVar[]" multiple="multiple" />
    <input type="submit" name="Submit" />
</form>

Observe as alterações feitas aqui; existem apenas alguns.

  • O nome input tem colchetes. Isso ocorre porque agora é uma matriz de arquivos e, portanto, estamos dizendo ao formulário para criar uma matriz dos arquivos selecionados. Omitir os colchetes resultará no último arquivo sendo definido como $_FILES["myVar"].

  • O atributo multiple="multiple". Isso apenas informa ao navegador que os usuários podem selecionar mais de um arquivo.

      case UPLOAD_ERR_FORM_SIZE:
          echo "Value: 2; The uploaded file exceeds the MAX_FILE_SIZE directive that was specified in the HTML form.";
          break;
      case UPLOAD_ERR_PARTIAL:
          echo "Value: 3; The uploaded file was only partially uploaded.";
          break;
      case UPLOAD_ERR_NO_FILE:
          echo "Value: 4; No file was uploaded.";
          break;
      case UPLOAD_ERR_NO_TMP_DIR:
          echo "Value: 6; Missing a temporary folder. Introduced in PHP 5.0.3.";
          break;
      case UPLOAD_ERR_CANT_WRITE:
          echo "Value: 7; Failed to write file to disk. Introduced in PHP 5.1.0.";
          break;
      case UPLOAD_ERR_EXTENSION:
          echo "Value: 8; A PHP extension stopped the file upload. PHP does not provide a way to ascertain which extension caused the file upload to stop; examining the list of loaded extensions with phpinfo() may help. Introduced in PHP 5.2.0.";
          break;

      default:
          echo "An unknown error has occured.";
          break;
  }

    }

Este é um exemplo muito simples e não trata de problemas como extensões de arquivo que não são permitidas ou arquivos nomeados com código PHP (como um equivalente PHP de uma injeção SQL). Consulte a [documentação](https://www.wikiod.com/pt/php/seguranca#Fazendo upload de arquivos).

O primeiro processo é verificar se existem arquivos e, em caso afirmativo, definir o número total deles como $total.

Usar o loop for permite uma iteração do array $_FILES e acessar cada item um de cada vez. Se esse arquivo não encontrar um problema, a instrução if será verdadeira e o código do upload de arquivo único será executado. Se for encontrado um problema, o bloco de comutação é executado e um erro é apresentado de acordo com o erro para esse upload específico.

$_COOKIE #

Um array associativo de variáveis ​​passadas para o script atual via HTTP Cookies.

Cookies são variáveis ​​que contêm dados e são armazenadas no computador do cliente.

Ao contrário das superglobais acima mencionadas, os cookies devem ser criados com uma função (e não atribuir um valor). A convenção está abaixo.

setcookie("myVar", "myVal", time() + 3600);

Neste exemplo, um nome é especificado para o cookie (neste exemplo é “myVar”), um valor é fornecido (neste exemplo é “myVal”, mas uma variável pode ser passada para atribuir seu valor ao cookie), e, em seguida, um tempo de expiração é fornecido (neste exemplo, é uma hora, pois 3600 segundos é um minuto).

Apesar da convenção para a criação de um cookie ser diferente, ele é acessado da mesma forma que os demais.

echo $_COOKIE["myVar"]; // returns "myVal"

Para destruir um cookie, setcookie deve ser chamado novamente, mas o tempo de expiração é definido como qualquer tempo no passado. Veja abaixo.

setcookie("myVar", "", time() - 1);
var_dump($_COOKIE["myVar"]); // returns null

Isso irá desabilitar os cookies e removê-los do computador do cliente.

$_SESSION #

Um array associativo contendo variáveis ​​de sessão disponíveis para o script atual. Consulte a documentação das Funções de sessão para obter mais informações sobre como isso é usado.

As sessões são muito parecidas com cookies, exceto que são do lado do servidor.

Para usar sessões você deve incluir session_start() na parte superior de seus scripts para permitir que as sessões sejam utilizadas.

Definir uma variável de sessão é o mesmo que definir qualquer outra variável. Veja exemplo abaixo.

$_SESSION["myVar"] = "myVal";

Ao iniciar uma sessão, um ID aleatório é definido como um cookie e chamado de “PHPSESSID” e conterá o ID da sessão atual. Isso pode ser acessado chamando a função session_id().

É possível destruir variáveis ​​de sessão usando a função unset (de forma que unset($_SESSION["myVar"]) destruiria essa variável). A alternativa é chamar session_destory(). Isso destruirá a sessão inteira, o que significa que todas as variáveis ​​de sessão não existirão mais.

$_REQUEST #

Um array associativo que por padrão contém o conteúdo de $_GET, [$_POST](http:// php.net/manual/en/reserved.variables.post.php) e $_COOKIE.

Como a documentação do PHP afirma, este é apenas um agrupamento de $_GET, $_POST e $_COOKIE tudo em uma variável.

Como é possível que todos os três arrays tenham um índice com o mesmo nome, há uma configuração no arquivo php.ini chamada request_order que pode especificar qual dos três tem precedência. Por exemplo, se foi definido como "GPC", o valor de $_COOKIE será usado, pois é lido da esquerda para a direita, o que significa que o $_REQUEST definirá seu valor como $_GET, então $_POST, e então $_COOKIE e já que $_COOKIE é o último que é o valor que está em $_REQUEST. Consulte esta pergunta.

$_ENV #

Uma matriz associativa de variáveis ​​passadas para o script atual através do método de ambiente.

Essas variáveis ​​são importadas para o namespace global do PHP do ambiente sob o qual o analisador PHP está sendo executado. Muitos são fornecidos pelo shell sob o qual o PHP está sendo executado e diferentes sistemas provavelmente executam diferentes tipos de shell, uma lista definitiva é impossível. Consulte a documentação do seu shell para obter uma lista de variáveis ​​de ambiente definidas.

Outras variáveis ​​de ambiente incluem as variáveis ​​CGI, colocadas lá independentemente de o PHP estar rodando como um módulo de servidor ou processador CGI.

Qualquer coisa armazenada em $_ENV é do ambiente no qual o PHP está sendo executado.

$_ENV só é preenchido se php.ini permitir. Consulte esta resposta para obter mais informações sobre por que $_ENV não é preenchido.
