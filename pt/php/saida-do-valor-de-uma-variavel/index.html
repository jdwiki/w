Para construir um programa PHP dinâmico e interativo, é útil gerar variáveis ​​de saída e seus valores. A linguagem PHP permite vários métodos de saída de valor. Este tópico aborda os métodos padrão de impressão de um valor em PHP e onde esses métodos podem ser usados.

Variáveis ​​em PHP vêm em uma variedade de tipos. Dependendo do caso de uso, você pode querer enviá-los para o navegador como HTML renderizado, enviá-los para depuração ou enviá-los para o terminal (se estiver executando um aplicativo por meio da linha de comando).

Abaixo estão alguns dos métodos mais usados ​​e construções de linguagem para variáveis ​​de saída:

  • [echo][echo] - Emite uma ou mais strings
  • [print][print] - Gera uma string e retorna 1 (sempre)
  • printf - Gera uma string formatada e retorna o comprimento da string de saída
  • sprintf - Formata uma string e retorna a string formatada
  • print_r - Gera ou retorna o conteúdo do argumento como uma string legível por humanos
  • [var_dump][var_dump] - Gera informações de depuração legíveis por humanos sobre o conteúdo do(s) argumento(s), incluindo seu tipo e valor
  • [var_export][var_export] - Gera ou retorna uma string de renderização da variável como código PHP válido, que pode ser usado para recriar o valor.

Nota: Ao tentar gerar um objeto como uma string, o PHP tentará convertê-lo em uma string (chamando [__toString()][to-string] - se o objeto tiver tal método) . Se não estiver disponível, um erro semelhante a Objeto da classe [CLASS] não pôde ser convertido em string será mostrado. Neste caso, você terá que inspecionar o objeto ainda mais, veja: [saída-a-exibição-estruturada-de-matrizes-e-objetos][saída-estruturada].

ecoe e imprima #

[echo][echo] e [print][print] são construções de linguagem, não funções. Isso significa que eles não exigem parênteses em torno do argumento como uma função faz (embora sempre se possa adicionar parênteses em torno de quase qualquer expressão PHP e, portanto, echo("test") também não causará nenhum dano). Eles produzem a representação de string de uma variável, constante ou expressão. Eles não podem ser usados ​​para imprimir arrays ou objetos.

  • Atribua a string Joel à variável $name

    $name = "Joel";

  • Saída do valor de $name usando echo & print

    echo $name;   #> Joel
print $name;  #> Joel

  • Parênteses não são obrigatórios, mas podem ser usados

    echo($name);  #> Joel
print($name); #> Joel

  • Usando vários parâmetros (apenas echo)

    echo $name, "Smith";       #> JoelSmith
echo($name, " ", "Smith"); #> Joel Smith

  • print, diferentemente de echo, é uma expressão (ela retorna 1), e assim pode ser usada em mais lugares:

    print("hey") && print(" ") && print("you"); #> you11

  • O acima é equivalente a:

    print ("hey" && (print (" " && print "you"))); #> you11

Notação abreviada para echo #

Quando fora das tags PHP, uma notação abreviada para echo está disponível por padrão, usando <?= para iniciar a saída e ?> para terminá-la. Por exemplo:

<p><?=$variable?></p>    
<p><?= "This is also PHP" ?></p>

Observe que não há terminação ;. Isso funciona porque a tag PHP de fechamento atua como o terminador para a instrução única. Portanto, é convencional omitir o ponto e vírgula nesta notação abreviada.

Prioridade de print #

Embora o print seja uma construção de linguagem, ele tem prioridade como operador. Ele coloca entre os operadores = += -= *= **= /= .= %= &= e and e deixou a associação. Exemplo:

echo '1' . print '2' + 3; //output 511

Mesmo exemplo com colchetes:

echo '1' . print ('2' + 3); //output 511

Diferenças entre echo e print #

Em suma, existem duas diferenças principais:

  • print só tem um parâmetro, enquanto echo pode ter vários parâmetros.
  • print retorna um valor, então pode ser usado como uma expressão.

Saída de uma visão estruturada de arrays e objetos #

print_r() - Saída de Matrizes e Objetos para depuração #

print_r produzirá um formato legível por humanos de uma matriz ou objeto.

Você pode ter uma variável que é uma matriz ou objeto. Tentar produzi-lo com um echo lançará o erro: Aviso: Conversão de array para string. Você pode usar a função print_r para despejar um formato legível desta variável.

Você pode passar true como segundo parâmetro para retornar o conteúdo como uma string.

$myobject = new stdClass();
$myobject->myvalue = 'Hello World';
$myarray = [ "Hello", "World" ];
$mystring = "Hello World";
$myint = 42;

// Using print_r we can view the data the array holds.
print_r($myobject);
print_r($myarray);
print_r($mystring);
print_r($myint);

Isso gera o seguinte:

stdClass Object
(
    [myvalue] => Hello World
)
Array
(
    [0] => Hello
    [1] => World
)
Hello World
42

Além disso, a saída de print_r pode ser capturada como uma string, em vez de simplesmente ecoada. Por exemplo, o código a seguir despejará a versão formatada de $myarray em uma nova variável:

$formatted_array = print_r($myarray, true);

Observe que se você estiver visualizando a saída do PHP em um navegador e ela for interpretada como HTML, as quebras de linha não serão mostradas e a saída será muito menos legível, a menos que você faça algo como

echo '<pre>' . print_r($myarray, true) . '</pre>';

Abrir o código fonte de uma página também irá formatar sua variável da mesma forma sem o uso da tag <pre>.

Alternativamente, você pode dizer ao navegador que o que você está produzindo é texto simples e não HTML:

header('Content-Type: text/plain; charset=utf-8');
print_r($myarray);

var_dump() - Saída de informações de depuração legíveis por humanos sobre o conteúdo do(s) argumento(s), incluindo seu tipo e valor #

A saída é mais detalhada como comparada para print_r porque também gera o tipo da variável junto com seu valor e outras informações como IDs de objetos, tamanhos de arrays, comprimentos de strings, marcadores de referência, etc.

Você pode usar var_dump para gerar uma versão mais detalhada para depuração.

var_dump($myobject, $myarray, $mystring, $myint);

A saída é mais detalhada:

object(stdClass)#12 (1) {
  ["myvalue"]=>
  string(11) "Hello World"
}
array(2) {
  [0]=>
  string(5) "Hello"
  [1]=>
  string(5) "World"
}
string(11) "Hello World"
int(42)

Observação: Se você estiver usando xDebug em seu ambiente de desenvolvimento, a saída de var_dump será limitada/truncada por padrão. Consulte a documentação oficial para obter mais informações sobre as opções para alterar isso.

var_export() - Saída de código PHP válido #

var_export() despeja uma representação PHP analisável do item.

Você pode passar true como o segundo parâmetro para retornar o conteúdo em uma variável.

var_export($myarray);
var_export($mystring);
var_export($myint);

A saída é um código PHP válido:

array (
  0 => 'Hello',
  1 => 'World',
)
'Hello World'
42

Para colocar o conteúdo em uma variável, você pode fazer isso:

$array_export = var_export($myarray, true);
$string_export = var_export($mystring, true);
$int_export = var_export($myint, 1); // any `Truthy` value

Depois disso, você pode emitir assim:

printf('$myarray = %s; %s', $array_export, PHP_EOL);
printf('$mystring = %s; %s', $string_export, PHP_EOL);
printf('$myint = %s; %s', $int_export, PHP_EOL);

Isso produzirá a seguinte saída:

$myarray = array (
  0 => 'Hello',
  1 => 'World',
);
$mystring = 'Hello World';
$myint = 42;

Concatenação de strings com eco #

Você pode usar concatenação para unir strings “de ponta a ponta” enquanto as produz (com echo ou print por exemplo).

Você pode concatenar variáveis ​​usando um . (ponto/ponto).

// String variable
$name = 'Joel';

// Concatenate multiple strings (3 in this example) into one and echo it once done.
//      1. ↓        2. ↓            3. ↓    - Three Individual string items
echo '<p>Hello ' . $name . ', Nice to see you.</p>';
//               ↑       ↑                  - Concatenation Operators

#> "<p>Hello Joel, Nice to see you.</p>"

Semelhante à concatenação, echo (quando usado sem parênteses) pode ser usado para combinar strings e variáveis ​​(juntamente com outras expressões arbitrárias) usando uma vírgula (,).

$itemCount = 1;

echo 'You have ordered ', $itemCount, ' item', $itemCount === 1 ? '' : 's';
//                      ↑           ↑        ↑                - Note the commas

#> "You have ordered 1 item"

Concatenação de string vs passagem de vários argumentos para echo #

Passar vários argumentos para o comando echo é mais vantajoso do que a concatenação de strings em algumas circunstâncias. Os argumentos são gravados na saída na mesma ordem em que são passados.

echo "The total is: ", $x + $y;

O problema com a concatenação é que o ponto . tem precedência na expressão. Se concatenada, a expressão acima precisa de parênteses extras para o comportamento correto. A precedência do período também afeta os operadores ternários.

echo "The total is: " . ($x + $y);

printf vs sprintf #

printf irá exibir uma string formatada usando espaços reservados

sprintf retornará a string formatada

$name = 'Jeff';

// The `%s` tells PHP to expect a string
//            ↓  `%s` is replaced by  ↓
printf("Hello %s, How's it going?", $name);
#> Hello Jeff, How's it going?

// Instead of outputting it directly, place it into a variable ($greeting)
$greeting = sprintf("Hello %s, How's it going?", $name);
echo $greeting;
#> Hello Jeff, How's it going?

Também é possível formatar um número com essas 2 funções. Isso pode ser usado para formatar um valor decimal usado para representar dinheiro para que sempre tenha 2 dígitos decimais.

$money = 25.2;
printf('%01.2f', $money);
#> 25.20

As duas funções [vprintf][vprintf] e [vsprintf ][vsprintf] operam como printf e sprintf, mas aceitam uma string de formato e um array de valores , em vez de variáveis ​​individuais.

Saída de inteiros grandes #

Em sistemas de 32 bits, inteiros maiores que PHP_INT_MAX são automaticamente convertidos em float. A saída deles como valores inteiros (ou seja, notação não científica) pode ser feita com printf, usando a representação float, conforme ilustrado abaixo:

foreach ([1, 2, 3, 4, 5, 6, 9, 12] as $p) {
    $i = pow(1024, $p);
    printf("pow(1024, %d) > (%7s) %20s %38.0F", $p, gettype($i), $i, $i);
    echo "  ", $i, "\n";
}
// outputs:
pow(1024,  1)  integer                 1024                                   1024  1024
pow(1024,  2)  integer              1048576                                1048576  1048576

Embora isso pareça bom, neste exemplo artificial, os números podem ser representados como um número binário, pois são todos potências de 1024 (e, portanto, 2). Veja por exemplo:

$n = pow(10, 27);
printf("%s %.0F\n", $n, $n);
// 1.0E+27 1000000000000000013287555072
