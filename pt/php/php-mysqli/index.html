A interface mysqli é uma melhoria (significa “extensão de melhoria do MySQL”) da interface mysql, que foi preterida na versão 5.5 e é removido na versão 7.0. A extensão mysqli, ou como às vezes é conhecida, a extensão aprimorada do MySQL, foi desenvolvida para aproveitar os novos recursos encontrados nos sistemas MySQL versões 4.1.3 e mais recentes. A extensão mysqli está incluída nas versões 5 e posteriores do PHP.

Características #

A interface mysqli tem vários benefícios, sendo os principais aprimoramentos sobre a extensão mysql:

  • Interface orientada a objetos
  • Suporte para Declarações Preparadas
  • Suporte para várias declarações
  • Suporte para transações
  • Recursos de depuração aprimorados
  • Suporte a servidor embutido

Ele apresenta uma interface dupla: o estilo procedural mais antigo e uma nova programação orientada a objetos (OOP) estilo. O obsoleto mysql tinha apenas uma interface procedural, então o estilo orientado a objetos é frequentemente preferido. No entanto, o novo estilo também é favorável devido ao poder do OOP.

Alternativas #

Uma alternativa à interface mysqli para acessar bancos de dados é a nova interface PHP Data Objects (PDO). Isso apresenta apenas programação no estilo OOP e pode acessar mais do que apenas bancos de dados do tipo MySQL.

Fechar conexão #

Quando terminarmos de consultar o banco de dados, é recomendável fechar a conexão para liberar recursos.

Estilo orientado a objetos

$conn->close();

Estilo de procedimento

mysqli_close($conn);

Observação: A conexão com o servidor será encerrada assim que a execução do script terminar, a menos que seja encerrada antes chamando explicitamente a função close connection.

Caso de uso: se nosso script tiver uma quantidade razoável de processamento para realizar após buscar o resultado e tiver recuperado o conjunto de resultados completo, definitivamente devemos fechar a conexão. Se não o fizermos, há uma chance do servidor MySQL atingir seu limite de conexão quando o servidor web estiver sob uso intenso.

MySQLi conectar #

Estilo orientado a objetos

Conecte-se ao servidor

$conn = new mysqli("localhost","my_user","my_password");

Defina o banco de dados padrão: $conn->select_db("my_db");

Conectar ao banco de dados

$conn = new mysqli("localhost","my_user","my_password","my_db");

Estilo de procedimento

Conecte-se ao servidor

$conn = mysqli_connect("localhost","my_user","my_password");

Defina o banco de dados padrão: mysqli_select_db($conn, "my_db");

Conectar ao banco de dados

$conn = mysqli_connect("localhost","my_user","my_password","my_db");

Verificar conexão de banco de dados

Estilo orientado a objetos

if ($conn->connect_errno > 0) {
    trigger_error($db->connect_error);
} // else: successfully connected

Estilo de procedimento

if (!$conn) {
   trigger_error(mysqli_connect_error());
} // else: successfully connected

Percorrer os resultados do MySQLi #

O PHP facilita a obtenção de dados de seus resultados e o loop usando uma instrução while. Quando ele não consegue obter a próxima linha, ele retorna false e ​​seu loop termina. Esses exemplos funcionam com

  • mysqli_fetch_assoc - Array associativo com nomes de colunas como chaves
  • mysqli_fetch_object - objeto stdClass com nomes de colunas como variáveis
  • mysqli_fetch_array - Array numérico e associativo (pode usar argumentos para obter um ou outro)
  • mysqli_fetch_row - Matriz numérica

Estilo orientado a objetos

while($row = $result->fetch_assoc()) {
    var_dump($row);
}

Estilo de procedimento

while($row = mysqli_fetch_assoc($result)) {
    var_dump($row);
}

Para obter informações exatas dos resultados, podemos usar:

while ($row = $result->fetch_assoc()) {
    echo 'Name and surname: '.$row['name'].' '.$row['surname'].'<br>';
    echo 'Age: '.$row['age'].'<br>'; // Prints info from 'age' column
}

Declarações preparadas no MySQLi #

Por favor, leia https://www.wikiod.com/pt/php/dop#Evitando injeção de SQL com consultas parametrizadas para uma discussão completa de por que instruções preparadas ajudam a proteger suas instruções SQL contra ataques de SQL Injection

A variável $conn aqui é um objeto MySQLi. Veja exemplo de conexão MySQLi para mais detalhes.

Para ambos os exemplos, assumimos que $sql é

$sql = "SELECT column_1 
    FROM table 
    WHERE column_2 = ? 
        AND column_3 > ?";

O ? representa os valores que forneceremos posteriormente. Observe que não precisamos de cotações para os espaços reservados, independentemente do tipo. Também podemos fornecer marcadores de posição apenas nas partes de dados da consulta, significando SET, VALUES e WHERE. Você não pode usar espaços reservados nas partes SELECT ou FROM.

Estilo orientado a objetos

if ($stmt = $conn->prepare($sql)) {
  $stmt->bind_param("si", $column_2_value, $column_3_value);
  $stmt->execute();

  $stmt->bind_result($column_1);
  $stmt->fetch();
  //Now use variable $column_1 one as if it were any other PHP variable
  $stmt->close();
}

Estilo de procedimento

if ($stmt = mysqli_prepare($conn, $sql)) {
  mysqli_stmt_bind_param($stmt, "si", $column_2_value, $column_3_value);
  mysqli_stmt_execute($stmt);
  // Fetch data here
  mysqli_stmt_close($stmt);
}

O primeiro parâmetro de $stmt->bind_param ou o segundo parâmetro de mysqli_stmt_bind_param é determinado pelo tipo de dados do parâmetro correspondente na consulta SQL:

Parâmetro Tipo de dados do parâmetro vinculado
eu inteiro
d duplo
s seqüência
b bolha

Sua lista de parâmetros precisa estar na ordem fornecida em sua consulta. Neste exemplo, si significa que o primeiro parâmetro (column_2 = ?) é string e o segundo parâmetro (column_3 > ?) é inteiro.

Para recuperar dados, consulte https://www.wikiod.com/pt/php/php-mysqli#Como obter dados de uma declaração preparada

Strings de escape #

O escape de strings é um método mais antigo (e menos seguro) de proteger dados para inserção em uma consulta. Ele funciona usando a função do MySQL mysql_real_escape_string() para processar e limpar os dados (em outras palavras , o PHP não está fazendo o escape). A API MySQLi fornece acesso direto a esta função

$escaped = $conn->real_escape_string($_GET['var']);
// OR
$escaped = mysqli_real_escape_string($conn, $_GET['var']);

Neste ponto, você tem uma string que o MySQL considera segura para uso em uma consulta direta

$sql = 'SELECT * FROM users WHERE username = "' . $escaped . '"';
$result = $conn->query($sql);

Então, por que isso não é tão seguro quanto [declarações preparadas](https://www.wikiod.com/pt/php/php-mysqli#Declarações preparadas no MySQLi)? Existem maneiras de enganar o MySQL para produzir uma string que considere segura. Considere o seguinte exemplo

$id = mysqli_real_escape_string("1 OR 1=1");    
$sql = 'SELECT * FROM table WHERE id = ' . $id;

1 OR 1=1 não representa dados que o MySQL irá escapar, mas ainda representa injeção de SQL. Existem outros exemplos que representam lugares onde retorna dados não seguros. O problema é que a função de escape do MySQL é projetada para tornar os dados compatíveis com a sintaxe SQL. Ele NÃO foi projetado para garantir que MySQL não possa confundir dados do usuário com instruções SQL.

consulta MySQLi #

A função query pega uma string SQL válida e a executa diretamente na conexão do banco de dados $conn

Estilo orientado a objetos

$result = $conn->query("SELECT * FROM `people`");

Estilo de procedimento

$result = mysqli_query($conn, "SELECT * FROM `people`");

CUIDADO

Um problema comum aqui é que as pessoas simplesmente executam a consulta e esperam que ela funcione (ou seja, retornar um objeto mysqli_stmt). Como essa função usa apenas uma string, você está construindo a consulta primeiro. Se houver algum erro no SQL, o compilador MySQL falhará, no ponto em que esta função retornará false.

$result = $conn->query('SELECT * FROM non_existent_table'); // This query will fail
$row = $result->fetch_assoc();

O código acima irá gerar um erro E_FATAL porque $result é false, e não um objeto.

PHP Fatal error: Chamada para uma função de membro fetch_assoc() em um não-objeto

O erro procedural é semelhante, mas não fatal, pois estamos apenas violando as expectativas da função.

$row = mysqli_fetch_assoc($result); // same query as previous

Você receberá a seguinte mensagem do PHP

mysqli_fetch_array() espera que o parâmetro 1 seja mysqli_result, dado booleano

Você pode evitar isso fazendo um teste primeiro

if($result) $row = mysqli_fetch_assoc($result);

Depurando SQL no MySQLi #

Portanto, sua consulta falhou (consulte https://www.wikiod.com/pt/php/php-mysqli#MySQLi conectar para saber como fizemos $conn)

$result = $conn->query('SELECT * FROM non_existent_table'); // This query will fail

Como descobrimos o que aconteceu? $result é false então isso não ajuda. Felizmente o connect $conn pode nos dizer o que o MySQL nos disse sobre a falha

trigger_error($conn->error);

ou processual

trigger_error(mysqli_error($conn));

Você deve receber um erro semelhante a

A tabela ‘my_db.non_exist_table’ não existe

Como obter dados de uma declaração preparada #

Declarações preparadas #

Consulte [Declarações preparadas no MySQLi](https://www.wikiod.com/pt/php/php-mysqli#Declarações preparadas no MySQLi) para saber como preparar e executar uma consulta.

Vinculação de resultados #

Estilo orientado a objetos

$stmt->bind_result($forename);

Estilo de procedimento

mysqli_stmt_bind_result($stmt, $forename);

O problema de usar o bind_result é que ele requer que a instrução especifique as colunas que serão usadas. Isso significa que para o acima funcionar a consulta deve ter se parecido com este SELECT forename FROM users. Para incluir mais colunas, basta adicioná-las como parâmetros à função bind_result (e certifique-se de adicioná-las à consulta SQL).

Em ambos os casos, estamos atribuindo a coluna forename à variável $forename. Essas funções aceitam tantos argumentos quanto as colunas que você deseja atribuir. A atribuição é feita apenas uma vez, pois a função é vinculada por referência.

Podemos então fazer um loop da seguinte forma:

Estilo orientado a objetos

while ($stmt->fetch())
    echo "$forename<br />";

Estilo de procedimento

while (mysqli_stmt_fetch($stmt))
    echo "$forename<br />";

A desvantagem disso é que você precisa atribuir muitas variáveis ​​de uma só vez. Isso dificulta o acompanhamento de consultas grandes. Se você tiver MySQL Native Driver (mysqlnd) instalado, tudo que você precisa fazer é usar [get_result](http://php .net/manual/en/mysqli-stmt.get-result.php).

Estilo orientado a objetos

$result = $stmt->get_result();

Estilo de procedimento

$result = mysqli_stmt_get_result($stmt);

Isso é muito mais fácil de trabalhar porque agora estamos obtendo um objeto mysqli_result. Este é o mesmo objeto que [mysqli_query retorna](https://www.wikiod.com/pt/php/php-mysqli#consulta MySQLi). Isso significa que você pode usar um [loop de resultado regular](https://www.wikiod.com/pt/php/php-mysqli#Faça um loop pelos resultados do MySQLi) para obter seus dados.

E se eu não conseguir instalar o mysqlnd? #

Se for esse o caso, o @Sophivorus cobriu você com [esta resposta incrível] (http://stackoverflow.com/a/30551477/3578036).

Esta função pode executar a tarefa de get_result sem que esteja instalada no servidor. Ele simplesmente percorre os resultados e cria uma matriz associativa

function get_result(\mysqli_stmt $statement)
{
    $result = array();
    $statement->store_result();
    for ($i = 0; $i < $statement->num_rows; $i++)
    {
        $metadata = $statement->result_metadata();
        $params = array();
        while ($field = $metadata->fetch_field())
        {
            $params[] = &$result[$i][$field->name];
        }
        call_user_func_array(array($statement, 'bind_result'), $params);
        $statement->fetch();
    }
    return $result;
}

Podemos então usar a função para obter resultados como este, como se estivéssemos usando mysqli_fetch_assoc()

<?php
$query = $mysqli->prepare("SELECT * FROM users WHERE forename LIKE ?");
$condition = "J%";
$query->bind_param("s", $condition);
$query->execute();
$result = get_result($query);

while ($row = array_shift($result)) {
    echo $row["id"] . ' - ' . $row["forename"] . ' ' . $row["surname"] . '<br>';
}

Ele terá a mesma saída como se você estivesse usando o driver mysqlnd, exceto que não precisa ser instalado. Isso é muito útil se você não conseguir instalar o driver em seu sistema. Basta implementar esta solução.

ID de inserção MySQLi #

Recupere o último ID gerado por uma consulta INSERT em uma tabela com uma coluna [AUTO_INCREMENT][3].

Estilo orientado a objetos

$id = $conn->insert_id;

Estilo de procedimento

$id = mysqli_insert_id($conn);

Retorna zero se não houve consulta anterior na conexão ou se a consulta não atualizou um valor AUTO_INCREMENT.

Insira o id ao atualizar as linhas

Normalmente uma instrução UPDATE não retorna um id de inserção, já que um id AUTO_INCREMENT só é retornado quando uma nova linha é salva (ou inserida). Uma maneira de fazer atualizações para o novo id é usar a sintaxe INSERT ... ON DUPLICATE KEY UPDATE para atualização.

Configuração para exemplos a seguir:

CREATE TABLE iodku (
    id INT AUTO_INCREMENT NOT NULL,
    name VARCHAR(99) NOT NULL,
    misc INT NOT NULL,
    PRIMARY KEY(id),
    UNIQUE(name)
) ENGINE=InnoDB;

INSERT INTO iodku (name, misc)
    VALUES
    ('Leslie', 123),

O caso de IODKU realizando uma “atualização” e LAST_INSERT_ID() recuperando o id relevante:

$sql = "INSERT INTO iodku (name, misc)
    VALUES
    ('Sally', 3333)            -- should update
    ON DUPLICATE KEY UPDATE    -- `name` will trigger "duplicate key"
    id = LAST_INSERT_ID(id),
    misc = VALUES(misc)";
$conn->query($sql);
$id = $conn->insert_id;        -- picking up existing value (2)

O caso em que IODKU executa uma “inserção” e LAST_INSERT_ID() recupera o novo id:

$sql = "INSERT INTO iodku (name, misc)
    VALUES
    ('Dana', 789)            -- Should insert
    ON DUPLICATE KEY UPDATE
    id = LAST_INSERT_ID(id),
    misc = VALUES(misc);
$conn->query($sql);
$id = $conn->insert_id;      -- picking up new value (3)
