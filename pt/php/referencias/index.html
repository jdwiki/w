Sintaxe #

  • $foo = 1; $bar = &$foo; // ambos $foo e $bar apontam para o mesmo valor: 1
  • $var = 1; função calc(&$var) { $var *= 15; } calc($var); echo $var;

Ao atribuir duas variáveis ​​por referência, ambas as variáveis ​​apontam para o mesmo valor. Tome o seguinte exemplo:

$foo = 1;
$bar = &$foo;

$foo não aponta para $bar. $foo e $bar apontam para o mesmo valor de $foo, que é 1. Ilustrar:

$baz = &$bar;
unset($bar);
$baz++;

Se tivéssemos um relacionamento points to, isso seria quebrado agora após o unset(); em vez disso, $foo e $baz ainda apontam para o mesmo valor, que é 2.

Atribuir por referência #

Esta é a primeira fase de referência. Essencialmente, quando você atribui por referência, está permitindo que duas variáveis ​​compartilhem o mesmo valor como tal.

$foo = &$bar;

$foo e $bar são iguais aqui. Eles não apontam um para o outro. Eles apontam para o mesmo lugar (o “valor”).

Você também pode atribuir por referência dentro da construção de linguagem array(). Embora não seja estritamente uma atribuição por referência.

$foo = 'hi';
$bar = array(1, 2);
$array = array(&$foo, &$bar[0]);

Observe, no entanto, que as referências dentro de arrays são potencialmente perigoso. Fazer uma atribuição normal (não por referência) com um referência do lado direito não transforma o lado esquerdo em referência, mas as referências dentro de arrays são preservadas nestes atribuições. Isso também se aplica a chamadas de função onde a matriz é passado por valor.

A atribuição por referência não se limita apenas a variáveis ​​e arrays, elas também estão presentes para funções e todas as associações “pass-by-reference”.

function incrementArray(&$arr) {
    foreach ($arr as &$val) {
        $val++;
    }
}

function &getArray() {
    static $arr = [1, 2, 3];
    return $arr;
}

incrementArray(getArray());
var_dump(getArray()); // prints an array [2, 3, 4]

A atribuição é a chave dentro da definição da função como acima. Você não pode passar uma expressão por referência, apenas um valor/variável. Daí a instanciação de $a em bar().

Retorno por referência #

Ocasionalmente, chega a hora de você retornar implicitamente por referência.

Retornar por referência é útil quando você deseja usar uma função para encontre a qual variável uma referência deve ser vinculada. Não use retorno por referência para aumentar o desempenho. O motor irá otimize isso automaticamente por conta própria. Retornar referências somente quando você tem uma razão técnica válida para fazê-lo.

Extraído da Documentação PHP para Devolução por Referência.

Existem muitas formas diferentes que o retorno por referência pode assumir, incluindo o seguinte exemplo:

function parent(&$var) {
    echo $var;
    $var = "updated";
}

function &child() {
    static $a = "test";
    return $a;
}

parent(child()); // returns "test"
parent(child()); // returns "updated"

O retorno por referência não se limita apenas a referências de função. Você também tem a capacidade de chamar implicitamente a função:

function &myFunction() {
    static $a = 'foo';
    return $a;
}

$bar = &myFunction();
$bar = "updated"
echo myFunction();

Você não pode referir diretamente uma chamada de função, ela deve ser atribuída a uma variável antes de aproveitá-la. Para ver como isso funciona, simplesmente tente echo &myFunction();.

Notas #

  • Você deve especificar uma referência (&) em ambos os lugares em que pretende usá-la. Isso significa que, para sua definição de função (function &myFunction() {...) e na referência de chamada (function callFunction(&$variable) {... ou &myFunction();).
  • Você só pode retornar uma variável por referência. Daí a instanciação de $a no exemplo acima. Isso significa que você não pode retornar uma expressão, caso contrário um erro E_NOTICE PHP será gerado (Aviso: Somente referências de variáveis ​​devem ser retornadas por referência em ......).
  • O retorno por referência tem casos de uso legítimos, mas devo alertar que eles devem ser usados ​​com moderação, somente depois de explorar todas as outras opções potenciais para atingir o mesmo objetivo.

Passar por referência #

Isso permite que você passe uma variável por referência a uma função ou elemento que permite modificar a variável original.

A passagem por referência não se limita apenas a variáveis, o seguinte também pode ser passado por referência:

  • Novas declarações, por ex. foo(new SomeClass)
  • Referências retornadas de funções

Matrizes #

Um uso comum de “passing-by-reference” é modificar valores iniciais dentro de um array sem chegar ao ponto de criar novos arrays ou sujar seu namespace. Passar por referência é tão simples quanto preceder/prefixar a variável com um & => &$myElement.

Abaixo está um exemplo de como aproveitar um elemento de uma matriz e simplesmente adicionar 1 ao seu valor inicial.

$arr = array(1, 2, 3, 4, 5);

foreach($arr as &$num) {
    $num++;
}

Agora, quando você aproveitar qualquer elemento dentro de $arr, o elemento original será atualizado à medida que a referência for aumentada. Você pode verificar isso por:

print_r($arr);

Observação

Você deve tomar nota ao aproveitar a passagem por referência dentro de loops. No final do loop acima, $num ainda mantém uma referência ao último elemento do array. Atribuí-lo ao pós-loop acabará manipulando o último elemento do array! Você pode garantir que isso não aconteça unset()’ post-loop:

$myArray = array(1, 2, 3, 4, 5);

foreach($myArray as &$num) { $num++; } unset($num); O acima irá garantir que você não tenha problemas. Um exemplo de problemas que podem estar relacionados a isso está presente em esta pergunta no StackOverflow.

Funções #

Outro uso comum para passagem por referência é dentro de funções. Modificar a variável original é tão simples quanto:

$var = 5;
// define
function add(&$var) {
    $var++;
}
// call
add($var);

O que pode ser verificado ’ecoando’ a variável original.

echo $var;

Existem várias restrições em torno das funções, conforme observado abaixo nos documentos do PHP:

Observação: não há sinal de referência em uma chamada de função - apenas na função definições. As definições de função sozinhas são suficientes para passar corretamente o argumento por referência. A partir do PHP 5.3.0, você receberá um aviso dizendo que “passar por referência de tempo de chamada” está obsoleto quando você usa & em foo(&$a);. E a partir do PHP 5.4.0, a passagem por referência de tempo de chamada era removido, então usá-lo gerará um erro fatal.
