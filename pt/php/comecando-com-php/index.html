Saída HTML do servidor web #

PHP pode ser usado para adicionar conteúdo a arquivos HTML. Enquanto o HTML é processado diretamente por um navegador da Web, os scripts PHP são executados por um servidor da Web e o HTML resultante é enviado ao navegador.

A marcação HTML a seguir contém uma instrução PHP que adicionará Hello World! à saída:

<!DOCTYPE html>
<html>
    <head>
        <title>PHP!</title>
    </head>
    <body>
        <p><?php echo "Hello world!"; ?></p>
    </body>
</html>

Quando isso é salvo como um script PHP e executado por um servidor web, o seguinte HTML será enviado ao navegador do usuário:

<!DOCTYPE html>
<html>
    <head>
        <title>PHP!</title>
    </head>
    <body>
        <p>Hello world!</p>
    </body>
</html>

echo também tem uma sintaxe de atalho, que permite imprimir imediatamente um valor. Antes do PHP 5.4.0, essa sintaxe curta funcionava apenas com a configuração short_open_tag habilitada.

Por exemplo, considere o seguinte código:

<p><?= "Hello world!" ?></p>

Sua saída é idêntica à saída do seguinte:

<p><?php echo "Hello world!"; ?></p>

Em aplicativos do mundo real, todos os dados de saída do PHP para uma página HTML devem ser escapados adequadamente para evitar ataques XSS (Cross-site scripting) ou corrupção de texto.

Veja também: Strings e PSR-1, que descreve as melhores práticas, incluindo o uso adequado de tags curtas (<?= ... ?>).

Olá Mundo! #

A construção de linguagem mais usada para imprimir a saída em PHP é echo:

echo "Hello, World!\n";

Alternativamente, você também pode usar print:

print "Hello, World!\n";

Ambas as instruções executam a mesma função, com pequenas diferenças:

  • echo tem um retorno void, enquanto print retorna um int com um valor de 1
  • echo pode receber vários argumentos (sem parênteses apenas), enquanto print só recebe um argumento
  • echo é um pouco mais rápido que print

Ambos echo e print são construções de linguagem, não funções. Isso significa que eles não exigem parênteses em torno de seus argumentos. Para consistência cosmética com funções, parênteses podem ser incluídos. Exemplos extensos do uso de echo e print estão disponíveis em outro lugar.

printf no estilo C e funções relacionadas também estão disponíveis, como no exemplo a seguir:

printf("%s\n", "Hello, World!");

Veja Saindo o valor de uma variável para uma introdução abrangente da saída de variáveis ​​em PHP.

Saída não HTML do servidor web #

Em alguns casos, ao trabalhar com um servidor web, pode ser necessário substituir o tipo de conteúdo padrão do servidor web. Pode haver casos em que você precise enviar dados como texto simples, JSON ou XML, por exemplo.

A função [header()][header] pode enviar um cabeçalho HTTP bruto. Você pode adicionar o cabeçalho Content-Type para notificar o navegador sobre o conteúdo que estamos enviando.

Considere o seguinte código, onde definimos Content-Type como text/plain:

header("Content-Type: text/plain");
echo "Hello World";

Isso produzirá um documento de texto simples com o seguinte conteúdo:

Olá mundo

Para produzir conteúdo JSON, use o tipo de conteúdo application/json:

header("Content-Type: application/json");

// Create a PHP data array.
$data = ["response" => "Hello World"];

// json_encode will convert it to a valid JSON string.
echo json_encode($data);

Isso produzirá um documento do tipo application/json com o seguinte conteúdo:

{“response”:“Olá Mundo”}

Observe que a função header() deve ser chamada antes que o PHP produza qualquer saída, ou o servidor web já terá enviado cabeçalhos para a resposta. Então, considere o seguinte código:

// Error: We cannot send any output before the headers
echo "Hello";

// All headers must be sent before ANY PHP output
header("Content-Type: text/plain");
echo "World";

Isso produzirá um aviso:

Aviso: não é possível modificar as informações do cabeçalho - cabeçalhos já enviados por (saída iniciada em /dir/example.php:2) em /dir/example.php na linha 3

Ao usar header(), sua saída precisa ser o primeiro byte enviado do servidor. Por esta razão é importante não ter linhas ou espaços vazios no início do arquivo antes da tag de abertura do PHP <?php. Pela mesma razão, é considerado uma prática recomendada (consulte PSR-2) omitir a tag de fechamento do PHP ?> de arquivos que contêm apenas PHP e de blocos de código PHP no final de um arquivo .

Veja a [seção de buffer de saída][buffer de saída] para saber como ‘capturar’ seu conteúdo em uma variável para saída mais tarde, por exemplo, após a saída de cabeçalhos.

Servidor embutido PHP #

O PHP 5.4+ vem com um servidor de desenvolvimento embutido. Ele pode ser usado para executar aplicativos sem precisar instalar um servidor HTTP de produção, como nginx ou Apache. O servidor interno foi projetado apenas para ser usado para fins de desenvolvimento e teste.

Ele pode ser iniciado usando o sinalizador -S:

php -S <host/ip>:<port>

Exemplo de uso #

  1. Crie um arquivo index.php contendo:

  2. Execute o comando php -S localhost:8080 na linha de comando. Não inclua http://. Isso iniciará um servidor web escutando na porta 8080 usando o diretório atual em que você está como a raiz do documento.

  3. Abra o navegador e navegue até http://localhost:8080. Você deve ver sua página “Hello World”.

Configuração #

Para substituir a raiz do documento padrão (ou seja, o diretório atual), use o sinalizador -t:

php -S <host/ip>:<port> -t <directory>

Por exemplo. se você tem um diretório public/ em seu projeto, você pode servir seu projeto a partir desse diretório usando php -S localhost:8080 -t public/.

Histórico #

CLI PHP #

O PHP também pode ser executado a partir da linha de comando diretamente usando a CLI (Command Line Interface).

CLI é basicamente o mesmo que PHP de servidores web, exceto algumas diferenças em termos de entrada e saída padrão.

Acionamento #

A CLI do PHP permite quatro maneiras de executar o código PHP:

  1. Entrada padrão. Execute o comando php sem nenhum argumento, mas insira o código PHP nele: 
    echo ‘<?php echo “Hello world!”;’ | php
  2. Nome do arquivo como argumento. Execute o comando php com o nome de um arquivo de origem PHP como o primeiro argumento: 
    php hello_world.php
  3. Codifique como argumento. Use a opção -r no comando php, seguida pelo código a ser executado. As tags abertas <?php não são obrigatórias, pois tudo no argumento é considerado como código PHP: 
    php -r ’echo “Hello world!”;’
  4. Escudo interativo. Use a opção -a no comando php para iniciar um shell interativo. Em seguida, digite (ou cole) o código PHP e pressione return: 
    $ php -a
Modo interativo ativado
php > echo “Olá mundo!”;
Olá mundo!

Resultado #

Todas as funções ou controles que produzem saída HTML no servidor web PHP podem ser usados ​​para produzir saída no fluxo stdout (descritor de arquivo 1), e todas as ações que produzem saída em logs de erro no servidor web PHP produzirão saída no fluxo stderr (arquivo descritor 2).

Example.php #

<?php
echo "Stdout 1\n";
trigger_error("Stderr 2\n");
print_r("Stdout 3\n");
fwrite(STDERR, "Stderr 4\n");
throw new RuntimeException("Stderr 5\n");
?>
Stdout 6

Linha de comando do Shell #

$ php Example.php 2>stderr.log >stdout.log;\
> echo STDOUT; cat stdout.log; echo;\
> echo STDERR; cat stderr.log\

Separação de instruções #

Assim como a maioria das outras linguagens de estilo C, cada instrução termina com um ponto e vírgula. Além disso, uma tag de fechamento é usada para encerrar a última linha de código do bloco PHP.

Se a última linha do código PHP terminar com um ponto e vírgula, a tag de fechamento é opcional se não houver código após essa linha final de código. Por exemplo, podemos deixar de fora a tag de fechamento após echo "No error"; no exemplo a seguir:

<?php echo "No error"; // no closing tag is needed as long as there is no code below

No entanto, se houver qualquer outro código seguindo seu bloco de código PHP, a tag de fechamento não é mais opcional:

<?php echo "This will cause an error if you leave out the closing tag"; ?>
<html>
    <body>
    </body>
</html>

Também podemos deixar de fora o ponto e vírgula da última instrução em um bloco de código PHP se esse bloco de código tiver uma tag de fechamento:

<?php echo "I hope this helps! :D";
echo "No error" ?>

Geralmente, é recomendado sempre usar um ponto e vírgula e usar uma tag de fechamento para cada bloco de código PHP, exceto o último bloco de código PHP, se nenhum código seguir esse bloco de código PHP.

Então, seu código deve ficar basicamente assim:

<?php
    echo "Here we use a semicolon!";
    echo "Here as well!";
    echo "Here as well!";
    echo "Here we use a semicolon and a closing tag because more code follows";
?>
<p>Some HTML code goes here</p>
<?php
    echo "Here we use a semicolon!";
    echo "Here as well!";
    echo "Here as well!";
    echo "Here we use a semicolon and a closing tag because more code follows";
?>
<p>Some HTML code goes here</p>
<?php
    echo "Here we use a semicolon!";
    echo "Here as well!";
    echo "Here as well!";
    echo "Here we use a semicolon but leave out the closing tag";

Tags PHP #

Existem três tipos de tags para denotar blocos PHP em um arquivo. O analisador PHP está procurando as tags de abertura e (se houver) de fechamento para delimitar o código a ser interpretado.

Tags padrão #

Essas tags são o método padrão para incorporar código PHP em um arquivo.

<?php
    echo "Hello World";
?>

Etiquetas de eco #

Essas tags estão disponíveis em todas as versões do PHP, e desde o PHP 5.4 estão sempre habilitadas. Nas versões anteriores, as tags de eco só podiam ser habilitadas em conjunto com as tags curtas.

<?= "Hello World" ?>

Tags curtas #

Você pode desabilitar ou habilitar essas tags com a opção short_open_tag.

<?
    echo "Hello World";
?>

Tags curtas:

  • não são permitidos em todos os principais [padrões de codificação] PHP 1
  • são desencorajados na documentação oficial
  • estão desabilitados por padrão na maioria das distribuições
  • interfere nas instruções de processamento do XML embutido
  • não são aceitos em envios de código pela maioria dos projetos de código aberto

Tags ASP #

Ao habilitar a opção asp_tags, tags no estilo ASP podem ser usadas.

<%
    echo "Hello World";
%>

Estes são uma peculiaridade histórica e nunca devem ser usados. Eles foram removidos no PHP 7.0.
