Um operador é algo que recebe um ou mais valores (ou expressões, no jargão da programação) e produz outro valor (para que a própria construção se torne uma expressão).

Os operadores podem ser agrupados de acordo com o número de valores que assumem.

Operadores ‘operam’ ou atuam em um (operadores unários como !$a e ++$a), dois (operadores binários como $a + $b ou $a >> $b) ou três (o único operador ternário é $a ? $b : $c) expressões.

A precedência do operador influencia como os operadores são agrupados (como se houvesse parênteses). A seguir está uma lista de operadores em ordem de precedência (operadores na segunda coluna). Se vários operadores estiverem em uma linha, o agrupamento é determinado pela ordem do código, onde a primeira coluna indica a associatividade (veja exemplos).

Observe que funções e construções de linguagem (por exemplo, print) são sempre avaliadas primeiro, mas qualquer valor de retorno será usado de acordo com as regras de precedência/associatividade acima. Cuidado especial é necessário se os parênteses após uma construção de linguagem forem omitidos. Por exemplo. eco 2 . print 3 + 4; echo’s 721: a parte print avalia 3 + 4, imprime o resultado 7 e retorna 1. Depois disso, 2 é ecoado, concatenado com o valor de retorno de print (1).

Operador de Coalescência Nulo (??) #

Null coalescing é um novo operador introduzido no PHP 7. Este operador retorna seu primeiro operando se estiver definido e não NULL. Caso contrário, retornará seu segundo operando.

O exemplo a seguir:

$name = $_POST['name'] ?? 'nobody';

é equivalente a ambos:

if (isset($_POST['name'])) {
    $name = $_POST['name'];
} else {
    $name = 'nobody';
}

e:

$name = isset($_POST['name']) ? $_POST['name'] : 'nobody';

Este operador também pode ser encadeado (com semântica associativa à direita):

$name = $_GET['name'] ?? $_POST['name'] ?? 'nobody';

que é equivalente a:

if (isset($_GET['name'])) {
    $name = $_GET['name'];
} elseif (isset($_POST['name'])) {
    $name = $_POST['name'];
} else {
    $name = 'nobody';
}

Observação:
Ao usar o operador de coalescência na concatenação de strings, não se esqueça de usar parênteses () 

$firstName = "John";
$lastName = "Doe";
echo $firstName ?? "Unknown" . " " . $lastName ?? "";

Isso produzirá apenas John, e se seu $firstName for nulo e $lastName for Doe, ele produzirá Unknown Doe. Para produzir John Doe, devemos usar parênteses como este.

$firstName = "John";
$lastName = "Doe";
echo ($firstName ?? "Unknown") . " " . ($lastName ?? "");

Isso produzirá apenas John Doe em vez de John.

Operador de nave espacial (<=>) #

O PHP 7 introduz um novo tipo de operador, que pode ser usado para comparar expressões. Este operador retornará -1, 0 ou 1 se a primeira expressão for menor, igual ou maior que a segunda expressão.

// Integers
print (1 <=> 1); // 0
print (1 <=> 2); // -1
print (2 <=> 1); // 1

// Floats
print (1.5 <=> 1.5); // 0
print (1.5 <=> 2.5); // -1
print (2.5 <=> 1.5); // 1
 
// Strings
print ("a" <=> "a"); // 0
print ("a" <=> "b"); // -1
print ("b" <=> "a"); // 1

Os objetos não são comparáveis, e isso resultará em um comportamento indefinido.

Este operador é particularmente útil ao escrever uma função de comparação definida pelo usuário usando usort, uasort ou uksort. Dada uma matriz de objetos a serem classificados por sua propriedade weight, por exemplo, uma função anônima pode usar <=> para retornar o valor esperado pelas funções de classificação.

usort($list, function($a, $b) { return $a->weight <=> $b->weight; });

No PHP 5, isso exigiria uma expressão um pouco mais elaborada.

usort($list, function($a, $b) {
    return $a->weight < $b->weight ? -1 : ($a->weight == $b->weight ? 0 : 1);
});

Operador Ternário (?:) #

O operador ternário pode ser pensado como uma instrução if embutida. É composto por três partes. O operador e dois resultados. A sintaxe é a seguinte:

$value = <operator> ? <true value> : <false value>

Se o operator for avaliado como true, o valor do primeiro bloco será retornado (<true value>), caso contrário o valor do segundo bloco será retornado (<false value>). Como estamos definindo $value para o resultado do nosso operador ternário, ele armazenará o valor retornado.

Exemplo:

$action = empty($_POST['action']) ? 'default' : $_POST['action'];

$action conteria a string 'default' se empty($_POST['action']) for avaliada como verdadeira. Caso contrário, conteria o valor de $_POST['action'].

A expressão (expr1) ? (expr2): (expr3) avalia como expr2 se expr1 avalia como true, e expr3 se expr1 avalia como false.

É possível deixar de fora a parte do meio do operador ternário. Expressão expr1 ?: expr3 retorna expr1 se expr1 for TRUE, e expr3 caso contrário. ?: é muitas vezes referido como operador Elvis.

Isso se comporta como o Null Coalescing operator ??, exceto que ?? requer que o operando esquerdo seja exatamente null enquanto ?: tenta resolver o operando esquerdo em um booleano e verifica se ele resolve para booleano false.

Exemplo:

function setWidth(int $width = 0){
    $_SESSION["width"] = $width ?: getDefaultWidth();
}

Neste exemplo, setWidth aceita um parâmetro de largura, ou padrão 0, para alterar o valor da sessão de largura. Se $width for 0 (se $width não for fornecido), que resolverá para boolean false, o valor de getDefaultWidth() será usado. A função getDefaultWidth() não será chamada se $width não resolver para boolean false.

Consulte https://www.wikiod.com/pt/php/tipos para obter mais informações sobre a conversão de variáveis ​​para booleano.

Operadores de Incremento (++) e Decremento (–) #

As variáveis ​​podem ser incrementadas ou decrementadas em 1 com ++ ou --, respectivamente. Eles podem preceder ou suceder variáveis ​​e variar ligeiramente semanticamente, conforme mostrado abaixo.

$i = 1;
echo $i; // Prints 1

// Pre-increment operator increments $i by one, then returns $i
echo ++$i; // Prints 2

// Pre-decrement operator decrements $i by one, then returns $i
echo --$i; // Prints 1

// Post-increment operator returns $i, then increments $i by one
echo $i++; // Prints 1 (but $i value is now 2)

// Post-decrement operator returns $i, then decrements $i by one
echo $i--; // Prints 2 (but $i value is now 1)

Mais informações sobre operadores de incremento e decremento podem ser encontradas na documentação oficial.

Operador de execução (``) #

O operador de execução do PHP consiste em acentos graves (``) e é usado para executar comandos do shell. A saída do comando será retornada, podendo, portanto, ser armazenada em uma variável.

// List files
$output = `ls`;
echo "<pre>$output</pre>";

Observe que o operador execute e shell_exec() fornecerão o mesmo resultado.

Operadores lógicos (&&/AND e ||/OR) #

Em PHP, existem duas versões de operadores lógicos AND e OR.

Operador Verdadeiro se
$a e $b Ambos $a e $b são verdadeiros
$a && $b Ambos $a e $b são verdadeiros
$a ou $b Ou $a ou $b é verdade
$a || $b Ou $a ou $b é verdade

Observe que os operadores && e || têm maior precedência do que and e or. Veja a tabela abaixo:

Avaliação Resultado de $e Avaliado como
$e = false || verdadeiro Verdadeiro $e = (falso || verdadeiro)
$e = falso ou verdadeiro Falso ($e = falso) ou verdadeiro

Por isso, é mais seguro usar && e || em vez de and e or.

Operadores de string (. e .=) #

Existem apenas dois operadores de string:

  • Concatenação de duas strings (ponto):

    $a = "a";
$b = "b";
$c = $a . $b; // $c => "ab"

  • Concatenar atribuição (ponto=):

    $a = "a";
$a .= "b"; // $a => "ab"

Operadores de objeto e classe #

Membros de objetos ou classes podem ser acessados ​​usando o operador de objeto (->) e o operador de classe (::).

class MyClass {
    public $a = 1;
    public static $b = 2;
    const C = 3;
    public function d() { return 4; }
    public static function e() { return 5; }
}

$object = new MyClass();
var_dump($object->a);   // int(1)
var_dump($object::$b);  // int(2)
var_dump($object::C);   // int(3)
var_dump(MyClass::$b);  // int(2)
var_dump(MyClass::C);   // int(3)
var_dump($object->d()); // int(4)
var_dump($object::d()); // int(4)
var_dump(MyClass::e()); // int(5)
$classname = "MyClass";
var_dump($classname::e()); // also works! int(5)

Observe que após o operador de objeto, o $ não deve ser escrito ($object->a em vez de $object->$a). Para o operador de classe, este não é o caso e o $ é necessário. Para uma constante definida na classe, o $ nunca é usado.

Observe também que var_dump(MyClass::d()); só é permitido se a função d() não fizer referência ao objeto:

class MyClass {
    private $a = 1;
    public function d() {
        return $this->a;
    }
}

$object = new MyClass();
var_dump(MyClass::d());   // Error!

Isso causa um ‘Erro fatal do PHP: Erro não capturado: usando $ this quando não estiver no contexto do objeto’

Esses operadores têm associatividade left, que pode ser usada para ’encadeamento’:

class MyClass {
    private $a = 1;
    
    public function add(int $a) {
        $this->a += $a;
        return $this;
    }
    
    public function get() {
        return $this->a;
    }
}

$object = new MyClass();
var_dump($object->add(4)->get());  // int(5)

Esses operadores têm a maior precedência (eles nem são mencionados no manual), ainda mais que clone. Desta forma:

class MyClass {
    private $a = 0;
    public function add(int $a) {
        $this->a += $a;
        return $this;
    }
    public function get() {
        return $this->a;
    }
}

$o1 = new MyClass();
$o2 = clone $o1->add(2);
var_dump($o1->get()); // int(2)
var_dump($o2->get()); // int(2)

O valor de $o1 é adicionado a antes do objeto ser clonado!

Observe que usar parênteses para influenciar a precedência não funcionou no PHP versão 5 e anterior (funciona no PHP 7):

// using the class MyClass from the previous code
$o1 = new MyClass();
$o2 = (clone $o1)->add(2);  // Error in PHP 5 and before, fine in PHP 7
var_dump($o1->get()); // int(0) in PHP 7
var_dump($o2->get()); // int(2) in PHP 7

Atribuição Combinada (+= etc) #

Os operadores de atribuição combinados são um atalho para uma operação em alguma variável e, posteriormente, atribuir esse novo valor a essa variável.

Aritmética:

$a = 1;   // basic assignment
$a += 2; // read as '$a = $a + 2'; $a now is (1 + 2) => 3
$a -= 1; // $a now is (3 - 1) => 2
$a *= 2; // $a now is (2 * 2) => 4
$a /= 2; // $a now is (16 / 2) => 8
$a %= 5; // $a now is (8 % 5) => 3 (modulus or remainder)

// array +
$arrOne = array(1);
$arrTwo = array(2);
$arrOne += $arrTwo;

Processando vários arrays juntos

$a **= 2; // $a now is (4 ** 2) => 16 (4 raised to the power of 2)

Concatenação e atribuição combinadas de uma string:

$a = "a";
$a .= "b"; // $a => "ab"

Operadores de atribuição bit a bit binários combinados:

$a = 0b00101010;  // $a now is 42
$a &= 0b00001111; // $a now is (00101010 & 00001111) => 00001010 (bitwise and)
$a |= 0b00100010; // $a now is (00001010 | 00100010) => 00101010 (bitwise or)
$a ^= 0b10000010; // $a now is (00101010 ^ 10000010) => 10101000 (bitwise xor)
$a >>= 3;         // $a now is (10101000 >> 3) => 00010101 (shift right by 3)
$a <<= 1;         // $a now is (00010101 << 1) => 00101010 (shift left by 1)

Alterando a precedência do operador (com parênteses) #

A ordem na qual os operadores são avaliados é determinada pela precedência do operador (consulte também a seção Comentários).

Dentro

$a = 2 * 3 + 4;

$a recebe um valor de 10 porque 2 * 3 é avaliado primeiro (a multiplicação tem uma precedência maior que a adição) produzindo um sub-resultado de 6 + 4, que é igual a 10.

A precedência pode ser alterada usando parênteses: em

$a = 2 * (3 + 4);

$a recebe um valor de 14 porque (3 + 4) é avaliado primeiro.

Atribuição Básica (=) #

$a = "some string";

resulta em $a tendo o valor some string.

O resultado de uma expressão de atribuição é o valor que está sendo atribuído. Observe que um único sinal de igual = NÃO é para comparação!

$a = 3;
$b = ($a = 5);

faz o seguinte:

  1. A linha 1 atribui 3 a $a.
  2. A linha 2 atribui 5 a $a. Esta expressão também produz o valor 5.
  3. A linha 2 então atribui o resultado da expressão entre parênteses (5) a $b.

Assim: tanto $a quanto $b agora têm valor 5.

Associação #

Associação esquerda #

Se a precedência de dois operadores for igual, a associatividade determina o agrupamento (veja também a seção Observações):

$a = 5 * 3 % 2; // $a now is (5 * 3) % 2 => (15 % 2) => 1

* e % têm precedência igual e associatividade esquerda. Como a multiplicação ocorre primeiro (à esquerda), ela é agrupada.

$a = 5 % 3 * 2; // $a now is (5 % 3) * 2 => (2 * 2) => 4

Agora, o operador de módulo ocorre primeiro (à esquerda) e, portanto, é agrupado.

Associação certa #

$a = 1;
$b = 1;
$a = $b += 1;

Ambos $a e $b agora têm valor 2 porque $b += 1 é agrupado e então o resultado ($b é 2) é atribuído a $a.

Operadores de comparação #

Igualdade #

Para testes básicos de igualdade, o operador igual == é usado. Para verificações mais abrangentes, use o operador idêntico ===.

O operador idêntico funciona da mesma forma que o operador igual, exigindo que seus operandos tenham o mesmo valor, mas também exige que eles tenham o mesmo tipo de dados.

Por exemplo, o exemplo abaixo exibirá ‘a e b são iguais’, mas não ‘a e b são idênticos’.

$a = 4;
$b = '4';
if ($a == $b) {
    echo 'a and b are equal'; // this will be printed
}
if ($a === $b) {
    echo 'a and b are identical'; // this won't be printed
}

Ao usar o operador igual, strings numéricas são convertidas em inteiros.

Comparação de objetos #

=== compara dois objetos verificando se eles são exatamente a mesma instância. Isso significa que new stdClass() === new stdClass() resolve para false, mesmo que eles sejam criados da mesma maneira (e tenham exatamente os mesmos valores).

== compara dois objetos verificando recursivamente se eles são iguais (deep equals). Isso significa que, para $a == $b, se $a e $b forem:

  1. da mesma classe
  2. ter o mesmo conjunto de propriedades, incluindo propriedades dinâmicas
  3. para cada propriedade $property definida, $a->property == $b->property é verdadeiro (portanto, verificado recursivamente).

Outros operadores comumente usados #

Eles incluem:

  1. Maior que (>)

  2. Menor que (<)

  3. Maior ou igual a (>=)

  4. Menor ou igual a (<=)

  5. Diferente de (!=)

  6. Diferente de (!==)

  7. Maior que: $a > $b, retorna true se o valor de $a for maior que $b, caso contrário, retorna false.

Exemplo:

var_dump(5 > 2); // prints bool(true)
var_dump(2 > 7); // prints bool(false)
  1. Menor que: $a < $b, retorna true se o valor de $a for menor que o de $b, caso contrário, retorna false.

Exemplo:

var_dump(5 < 2); // prints bool(false)
var_dump(1 < 10); // prints bool(true)
  1. Maior que ou igual a: $a >= $b, retorna true se o valor de $a for maior que $b ou igual a $b, caso contrário, retorna falso.

Exemplo:

var_dump(2 >= 2); // prints bool(true)
var_dump(6 >= 1); // prints bool(true)
var_dump(1 >= 7); // prints bool(false)
  1. Menor que ou igual a: $a <= $b, retorna true se o valor de $a for menor que $b ou igual a $b, caso contrário, retorna falso.

Exemplo:

var_dump(5 <= 5); // prints bool(true)
var_dump(5 <= 8); // prints bool(true)
var_dump(9 <= 1); // prints bool(false)

5/6. Não igual/idêntico a: para repetir o exemplo anterior sobre igualdade, o exemplo abaixo exibirá ‘a e b não são idênticos’, mas não ‘a e b não são iguais’.

$a = 4;
$b = '4';
if ($a != $b) {
    echo 'a and b are not equal'; // this won't be printed
}
if ($a !== $b) {
    echo 'a and b are not identical'; // this will be printed
}

instanceof (operador de tipo) #

Para verificar se algum objeto é de uma determinada classe, o operador (binário) instanceof pode ser usado desde a versão 5 do PHP.

O primeiro parâmetro (esquerdo) é o objeto a ser testado. Se esta variável não for um objeto, instanceof sempre retornará false. Se uma expressão constante for usada, um erro será lançado.

O segundo parâmetro (à direita) é a classe com a qual comparar. A classe pode ser fornecida como o próprio nome da classe, uma variável de string contendo o nome da classe (não uma constante de string!) ou um objeto dessa classe.

class MyClass {
}

$o1 = new MyClass();
$o2 = new MyClass();
$name = 'MyClass';

// in the cases below, $a gets boolean value true
$a = $o1 instanceof MyClass;
$a = $o1 instanceof $name;
$a = $o1 instanceof $o2;

// counter examples:
$b = 'b';
$a = $o1 instanceof 'MyClass'; // parse error: constant not allowed
$a = false instanceof MyClass; // fatal error: constant not allowed
$a = $b instanceof MyClass;    // false ($b is not an object)

instanceof também pode ser usado para verificar se um objeto é de alguma classe que estende outra classe ou implementa alguma interface:

interface MyInterface {
}

class MySuperClass implements MyInterface {
}

class MySubClass extends MySuperClass {
}

$o = new MySubClass();

// in the cases below, $a gets boolean value true    
$a = $o instanceof MySubClass;
$a = $o instanceof MySuperClass;
$a = $o instanceof MyInterface;

Para verificar se um objeto é não de alguma classe, o operador not (!) pode ser usado:

class MyClass {
}

class OtherClass {
}

$o = new MyClass();
$a = !$o instanceof OtherClass; // true

Observe que os parênteses em torno de $o instanceof MyClass não são necessários porque instanceof tem precedência mais alta que !, embora possa tornar o código mais legível com parênteses.

Ressalvas #

Se uma classe não existir, as funções de autoload registradas são chamadas para tentar definir a classe (este é um tópico fora do escopo desta parte da Documentação!). Nas versões do PHP anteriores a 5.1.0, o operador instanceof também acionava essas chamadas, definindo assim a classe (e se a classe não pudesse ser definida, ocorreria um erro fatal). Para evitar isso, use uma string:

// only PHP versions before 5.1.0!
class MyClass {
}

$o = new MyClass();
$a = $o instanceof OtherClass; // OtherClass is not defined!
// if OtherClass can be defined in a registered autoloader, it is actually
// loaded and $a gets boolean value false ($o is not a OtherClass)
// if OtherClass can not be defined in a registered autoloader, a fatal
// error occurs.

$name = 'YetAnotherClass';
$a = $o instanceof $name; // YetAnotherClass is not defined!
// $a simply gets boolean value false, YetAnotherClass remains undefined.

A partir da versão 5.1.0 do PHP, os autoloaders registrados não são mais chamados nestas situações.

Versões mais antigas do PHP (antes de 5.0) #

Em versões mais antigas do PHP (antes de 5.0), a função is_a pode ser usada para determinar se um objeto é de alguma classe. Esta função foi preterida no PHP versão 5 e não preterida no PHP versão 5.3.0.

Operadores bit a bit #

Prefixo operadores bit a bit #

Operadores bit a bit são como operadores lógicos, mas executados por bit e não por valor booleano.

// bitwise NOT ~: sets all unset bits and unsets all set bits
printf("%'06b", ~0b110110); // 001001

Operadores Bitmask-bitmask #

Bitwise AND &: um bit é definido apenas se estiver definido em ambos os operandos

printf("%'06b", 0b110101 & 0b011001); // 010001

Bitwise OR |: um bit é definido se estiver definido em um ou ambos os operandos

printf("%'06b", 0b110101 | 0b011001); // 111101

Bitwise XOR ^: um bit é definido se estiver definido em um operando e não definido em outro operando, ou seja, somente se esse bit estiver em estado diferente nos dois operandos

printf("%'06b", 0b110101 ^ 0b011001); // 101100

Exemplo de uso de bitmasks #

Esses operadores podem ser usados ​​para manipular bitmasks. Por exemplo:

file_put_contents("file.log", LOCK_EX | FILE_APPEND);

Aqui, o operador | é usado para combinar as duas bitmasks. Embora + tenha o mesmo efeito, | enfatiza que você está combinando bitmasks, não adicionando dois inteiros escalares normais.

class Foo{
    const OPTION_A = 1;
    const OPTION_B = 2;
    const OPTION_C = 4;
    const OPTION_A = 8;

    private $options = self::OPTION_A | self::OPTION_C;

    public function toggleOption(int $option){
        $this->options ^= $option;
    }

    public function enable(int $option){
        $this->options |= $option; // enable $option regardless of its original state
    }

    public function disable(int $option){
        $this->options &= ~$option; // disable $option regardless of its original state,
                                    // without affecting other bits
    }

    /** returns whether at least one of the options is enabled */
    public function isOneEnabled(int $options) : bool{
        return $this->options & $option !== 0;
        // Use !== rather than >, because 
        // if $options is about a high bit, we may be handling a negative integer
    }

    /** returns whether all of the options are enabled */
    public function areAllEnabled(int $options) : bool{
        return ($this->options & $options) === $options;
        // note the parentheses; beware the operator precedence
    }
}

Este exemplo (assumindo que $option sempre contém apenas um bit) usa:

  • o operador ^ para alternar convenientemente as bitmasks.
  • o operador | para definir um bit negligenciando seu estado original ou outros bits
  • o operador ~ para converter um inteiro com apenas um bit definido em um inteiro com apenas um bit não definido
  • o operador & para desarmar um pouco, usando estas propriedades de &:
  • Como &= com um bit definido não fará nada ((1 & 1) === 1, (0 & 1) === 0), fazer &= com um inteiro com apenas um bit não definido apenas desabilitará esse bit, não afetando outros bits.
  • &= com um bit não definido irá desarmar esse bit ((1 & 0) === 0, (0 & 0) === 0)
  • Usar o operador & com outra máscara de bits filtrará todos os outros bits não definidos nessa máscara de bits.
  • Se a saída tiver algum bit definido, significa que qualquer uma das opções está habilitada.
  • Se a saída tiver todos os bits do bitmask configurados, significa que todas as opções do bitmask estão habilitadas.

Tenha em mente que estes operadores de comparação: (< > <= >= == === != !== <> <=> ) têm maior precedência do que esses operadores bitmask-bitmask: (| ^ &). Como os resultados bit a bit são frequentemente comparados usando esses operadores de comparação, essa é uma armadilha comum a ser observada.

Operadores de deslocamento de bits #

Deslocamento à esquerda bit a bit <<: desloca todos os bits para a esquerda (mais significativos) pelo número de passos dado e descarta os bits que excedem o tamanho int

<< $x é equivalente a desarmar os bits $x mais altos e multiplicar pela $xth potência de 2

printf("%'08b", 0b00001011<< 2); // 00101100

assert(PHP_INT_SIZE === 4); // a 32-bit system
printf("%x, %x", 0x5FFFFFFF << 2, 0x1FFFFFFF << 4); // 7FFFFFFC, FFFFFFFF

Deslocamento à direita bit a bit >>: descarta o deslocamento mais baixo e desloca os bits restantes para a direita (menos significativo)

>> $x é equivalente a dividir pela $xth potência de 2 e descartar a parte não inteira

printf("%x", 0xFFFFFFFF >> 3); // 1FFFFFFF

Exemplo de uso de deslocamento de bits: #

Divisão rápida por 16 (melhor desempenho que /= 16)

$x >>= 4;

Em sistemas de 32 bits, isso descarta todos os bits do inteiro, definindo o valor como 0. Em sistemas de 64 bits, isso desativa os 32 bits mais significativos e mantém os menos

$x = $x << 32 >> 32;

32 bits significativos, equivalente a $x & 0xFFFFFFFF

Nota: Neste exemplo, printf("%'06b") é usado. Ele gera o valor em 6 dígitos binários.
