Lembre-se das seguintes dicas ao decidir como comentar seu código:

  • Você deve sempre escrever seu código como se não existissem comentários, usando nomes de variáveis ​​e funções bem escolhidos.
  • Os comentários servem para comunicar a outros seres humanos, não para repetir o que está escrito no código.
  • Existem vários guias de estilo de comentários php (por exemplo, pear, zend, etc). Descubra qual sua empresa usa e use-a de forma consistente!

Comentários de linha única #

O comentário de linha única começa com “//” ou “#”. Quando encontrado, todo o texto à direita será ignorado pelo interpretador PHP.

// This is a comment

# This is also a comment

echo "Hello World!"; // This is also a comment, beginning where we see "//"

Comentários de várias linhas #

O comentário de várias linhas pode ser usado para comentar grandes blocos de código. Começa com /* e termina com */.

/* This is a multi-line comment.
   It spans multiple lines.
   This is still part of the comment. 
*/
