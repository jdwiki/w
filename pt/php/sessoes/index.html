Observe que chamar session_start() mesmo que a sessão já tenha iniciado resultará em um aviso do PHP.

session_start() Opções #

Começando com PHP Sessions podemos passar um array com opções php.ini baseadas em sessão para a função session_start.

Exemplo

<?php
   if (version_compare(PHP_VERSION, '7.0.0') >= 0) {
       // php >= 7 version
       session_start([
          'cache_limiter' => 'private',
          'read_and_close' => true,
       ]);
   } else {
       // php < 7 version
       session_start();
   }
?>

Este recurso também introduz uma nova configuração php.ini chamada session.lazy_write, cujo padrão é true e significa que os dados da sessão só são reescritos, se forem alterados.

Referência: https://wiki.php.net/rfc/session-lock-ini

Bloqueio de sessão #

Como todos sabemos, o PHP grava os dados da sessão em um arquivo no lado do servidor. Quando uma solicitação é feita para o script php que inicia a sessão via session_start(), o PHP bloqueia este arquivo de sessão resultando em bloquear/aguardar outras solicitações recebidas para o mesmo session_id para serem concluídas, por causa da qual as outras solicitações ficarão presas session_start() até ou a menos que o arquivo de sessão bloqueado não seja liberado

O arquivo de sessão permanece bloqueado até que o script seja concluído ou a sessão seja fechada manualmente. Para evitar esta situação ou seja, para evitar que várias solicitações sejam bloqueadas, podemos iniciar a sessão e fechar a sessão, o que liberará o bloqueio do arquivo de sessão e permitirá continuar as solicitações restantes.

// php < 7.0 
// start session 
session_start();

// write data to session
$_SESSION['id'] = 123; // session file is locked, so other requests are blocked

// close the session, release lock
session_write_close();

Agora se pensará se a sessão for fechada como vamos ler os valores da sessão, embelezar mesmo depois que a sessão for encerrada, a sessão ainda estará disponível. Assim, ainda podemos ler os dados da sessão.

echo $_SESSION['id'];    // will output  123

Em php >= 7.0, podemos ter sessão read_only, sessão read_write e sessão lazy_write, então pode não ser necessário usar session_write_close()

Manipulando dados de sessão #

A variável $_SESSION é um array, e você pode recuperá-la ou manipulá-la como um array normal.

<?php
// Starting the session
session_start();

// Storing the value in session
$_SESSION['id'] = 342;

// conditional usage of session values that may have been set in a previous session
if(!isset($_SESSION["login"])) {
    echo "Please login first";
    exit;
}
// now you can use the login safely
$user = $_SESSION["login"];

// Getting a value from the session data, or with default value, 
//     using the Null Coalescing operator in PHP 7
$name = $_SESSION['name'] ?? 'Anonymous';

Consulte também https://www.wikiod.com/pt/php/manipulando-um-array para obter mais referências sobre como trabalhar em uma matriz.

Observe que, se você armazenar um objeto em uma sessão, ele poderá ser recuperado normalmente apenas se você tiver um carregador automático de classe ou já tiver carregado a classe. Caso contrário, o objeto será do tipo __PHP_Incomplete_Class, o que pode levar a travamentos. Consulte https://www.wikiod.com/pt/php/classes-e-objetos#Espaçamento de nomes e carregamento automático sobre carregamento automático.

Aviso: #

Os dados da sessão podem ser sequestrados. Isso é descrito em: Pro PHP Security: From Application Security Principles to the Implementation of XSS Defense - Chapter 7: Preventing Session Hijacking Portanto, é altamente recomendável nunca armazenar nenhuma informação pessoal em $_SESSION . Isso incluiria os números de cartão de crédito, IDs emitidos pelo governo e senhas; mas também se estenderia a dados menos presumidos, como nomes, e-mails, números de telefone etc., que permitiriam a um hacker se passar por/comprometer um usuário legítimo. Como regra geral, use valores inúteis/não pessoais, como identificadores numéricos, nos dados da sessão.

Destrua uma sessão inteira #

Se você tem uma sessão que deseja destruir, você pode fazer isso com session_destroy()

/*
    Let us assume that our session looks like this:
    Array([firstname] => Jon, [id] => 123)

    We first need to start our session:
*/
session_start();

/*
    We can now remove all the values from the `SESSION` superglobal:
    If you omitted this step all of the global variables stored in the 
    superglobal would still exist even though the session had been destroyed.
*/
$_SESSION = array();

// If it's desired to kill the session, also delete the session cookie.
// Note: This will destroy the session, and not just the session data!
if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(session_name(), '', time() - 42000,
        $params["path"], $params["domain"],
        $params["secure"], $params["httponly"]
    );
}

//Finally we can destroy the session:
session_destroy();

Usar session_destroy() é diferente de usar algo como $_SESSION = array(); que removerá todos os valores armazenados na superglobal SESSION mas não destruirá a versão real armazenada da sessão.

Observação: Usamos $_SESSION = array(); em vez de session_unset() porque o manual estipula:

Use apenas session_unset() para código obsoleto mais antigo que não usa $_SESSION.

Nome da sessão #

Verificando se os cookies de sessão foram criados #

O nome da sessão é o nome do cookie usado para armazenar sessões. Você pode usar isso para detectar se os cookies de uma sessão foram criados para o usuário:

if(isset($_COOKIE[session_name()])) {
    session_start();
}

Observe que esse método geralmente não é útil, a menos que você realmente não queira criar cookies desnecessariamente.

Alterando o nome da sessão #

Você pode atualizar o nome da sessão chamando session_name().

//Set the session name
session_name('newname');
//Start the session
session_start();

Se nenhum argumento for fornecido em session_name(), o nome da sessão atual será retornado.

Deve conter apenas caracteres alfanuméricos; deve ser curto e descritivo (ou seja, para usuários com avisos de cookies ativados). O nome da sessão não pode conter apenas dígitos, pelo menos uma letra deve estar presente. Caso contrário, um novo ID de sessão é gerado a cada vez.

Início de sessão seguro sem erros #

Muitos desenvolvedores têm esse problema quando trabalham em projetos grandes, especialmente se eles trabalham em algum CMS modular em plugins, complementos, componentes etc. se a sessão for iniciada. Se a sessão não existir, eu inicio a sessão segura. Se a sessão existe, nada acontece.

if (version_compare(PHP_VERSION, '7.0.0') >= 0) {
    if(session_status() == PHP_SESSION_NONE) {
        session_start(array(
          'cache_limiter' => 'private',
          'read_and_close' => true,
       ));
    }
}
else if (version_compare(PHP_VERSION, '5.4.0') >= 0)
{
    if (session_status() == PHP_SESSION_NONE) {
        session_start();
    }
}
else
{
    if(session_id() == '') {
        session_start();
    }
}

Isso pode ajudá-lo muito a evitar o erro session_start.
