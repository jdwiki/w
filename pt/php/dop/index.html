A extensão PDO (PHP Data Objects) permite que os desenvolvedores se conectem a vários tipos diferentes de bancos de dados e executem consultas neles de forma uniforme e orientada a objetos. maneiras.

Sintaxe #

  • PDO::LastInsertId()
  • PDO::LastInsertId($columnName) // alguns drivers precisam do nome da coluna

Aviso Não deixe de verificar as exceções ao usar lastInsertId(). Ele pode lançar o seguinte erro:

SQLSTATE IM001: Driver não suporta esta função

Aqui está como você deve verificar corretamente as exceções usando este método:

// Retrieving the last inserted id
$id = null;

try {
    $id = $pdo->lastInsertId(); // return value is an integer    
}
catch( PDOException $e ) {
    echo $e->getMessage();
}

Evitando injeção de SQL com consultas parametrizadas #

A injeção de SQL é um tipo de ataque que permite que um usuário mal-intencionado modifique a consulta SQL, adicionando comandos indesejados a ela. Por exemplo, o código a seguir é vulnerável:

// Do not use this vulnerable code!
$sql = 'SELECT name, email, user_level FROM users WHERE userID = ' . $_GET['user'];
$conn->query($sql);

Isso permite que qualquer usuário deste script modifique nosso banco de dados basicamente à vontade. Por exemplo, considere a seguinte string de consulta:

page.php?user=0;%20TRUNCATE%20TABLE%20users;

Isso faz com que nossa consulta de exemplo fique assim

SELECT name, email, user_level FROM users WHERE userID = 0; TRUNCATE TABLE users;

Embora este seja um exemplo extremo (a maioria dos ataques de injeção de SQL não visa excluir dados, nem a maioria das funções de execução de consulta PHP suporta multi-consulta), este é um exemplo de como um ataque de injeção de SQL pode ser possibilitado pela montagem descuidada de A pergunta. Infelizmente, ataques como esse são muito comuns e altamente eficazes devido a codificadores que não tomam as devidas precauções para proteger seus dados.

Para evitar a ocorrência de injeção de SQL, instruções preparadas são a solução recomendada. Em vez de concatenar os dados do usuário diretamente na consulta, um espaço reservado é usado. Os dados são então enviados separadamente, o que significa que não há chance de o mecanismo SQL confundir os dados do usuário com um conjunto de instruções.

Embora o tópico aqui seja PDO, observe que a extensão PHP MySQLi também [suporta instruções preparadas](https://www.wikiod.com/pt/php/php-mysqli#Declarações preparadas no MySQLi)

O PDO suporta dois tipos de placeholders (placeholders não podem ser usados ​​para nomes de colunas ou tabelas, apenas valores):

  1. Espaços reservados nomeados. Dois pontos (:), seguido por um nome distinto (por exemplo, :user)

    // using named placeholders
$sql = 'SELECT name, email, user_level FROM users WHERE userID = :user';
$prep = $conn->prepare($sql);
$prep->execute(['user' => $_GET['user']]); // associative array
$result = $prep->fetchAll();

  2. Marcadores posicionais SQL tradicionais, representados como ?:

    // using question-mark placeholders
$sql = 'SELECT name, user_level FROM users WHERE userID = ? AND user_level = ?';
$prep = $conn->prepare($sql);
$prep->execute([$_GET['user'], $_GET['user_level']]); // indexed array
$result = $prep->fetchAll();

Se você precisar alterar dinamicamente os nomes das tabelas ou colunas, saiba que isso é por sua conta e risco de segurança e é uma prática ruim. No entanto, isso pode ser feito por concatenação de strings. Uma maneira de melhorar a segurança dessas consultas é definir uma tabela de valores permitidos e comparar o valor que você deseja concatenar com essa tabela.

Esteja ciente de que é importante definir o conjunto de caracteres de conexão apenas por meio de DSN, caso contrário, seu aplicativo poderá estar propenso a uma vulnerabilidade obscura se alguma codificação estranha for usada. Para versões de PDO anteriores a 5.3.6, a configuração do charset através do DSN não está disponível e, portanto, a única opção é definir o atributo PDO::ATTR_EMULATE_PREPARES como false na conexão logo após sua criação.

$conn->setAttribute(PDO::ATTR_EMULATE_PREPARES, false);

Isso faz com que o PDO use as instruções preparadas nativas do SGBD subjacente em vez de apenas emulá-lo.

No entanto, esteja ciente de que o PDO irá silenciosamente fazer fallback para emular instruções que o MySQL não pode preparar nativamente: aquelas que podem ser listadas no manual (source ).

Conexão e recuperação básica do PDO #

Desde o PHP 5.0, PDO está disponível como uma camada de acesso ao banco de dados. É independente de banco de dados e, portanto, o código de exemplo de conexão a seguir deve funcionar para qualquer de seus bancos de dados suportados simplesmente alterando o DSN.

// First, create the database handle

//Using MySQL (connection via local socket):
$dsn = "mysql:host=localhost;dbname=testdb;charset=utf8";

//Using MySQL (connection via network, optionally you can specify the port too):
//$dsn = "mysql:host=127.0.0.1;port=3306;dbname=testdb;charset=utf8";

//Or Postgres
//$dsn = "pgsql:host=localhost;port=5432;dbname=testdb;";

//Or even SQLite
//$dsn = "sqlite:/path/to/database"

$username = "user";
$password = "pass";
$db = new PDO($dsn, $username, $password);

// setup PDO to throw an exception if an invalid query is provided
$db->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

// Next, let's prepare a statement for execution, with a single placeholder
$query = "SELECT * FROM users WHERE class = ?";
$statement = $db->prepare($query);

// Create some parameters to fill the placeholders, and execute the statement
$parameters = [ "221B" ];
$statement->execute($parameters);

// Now, loop through each record as an associative array
while ($row = $statement->fetch(PDO::FETCH_ASSOC)) {
    do_stuff($row);
}

A função prepare cria um objeto PDOStatement a partir da string de consulta. A execução da consulta e a recuperação dos resultados são realizadas neste objeto retornado. Em caso de falha, a função retorna false ou lança uma exceção (dependendo de como a conexão PDO foi configurada).

Transações de banco de dados com PDO #

As transações de banco de dados garantem que um conjunto de alterações de dados só se torne permanente se cada instrução for bem-sucedida. Qualquer consulta ou falha de código durante uma transação pode ser detectada e você tem a opção de reverter as tentativas de alteração.

O PDO fornece métodos simples para iniciar, confirmar e reverter transações.

$pdo = new PDO(
    $dsn, 
    $username, 
    $password, 
    array(PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION)
);

try {
    $statement = $pdo->prepare("UPDATE user SET name = :name");

    $pdo->beginTransaction();

    $statement->execute(["name"=>'Bob']);
    $statement->execute(["name"=>'Joe']);

    $pdo->commit();
} 
catch (\Exception $e) {
    if ($pdo->inTransaction()) {
        $pdo->rollback();
        // If we got here our two data updates are not in the database
    }
    throw $e;
}

Durante uma transação, quaisquer alterações de dados feitas são visíveis apenas para a conexão ativa. As instruções SELECT retornarão as alterações alteradas mesmo que ainda não tenham sido confirmadas no banco de dados.

Observação: Consulte a documentação do fornecedor do banco de dados para obter detalhes sobre o suporte a transações. Alguns sistemas não suportam transações. Alguns suportam transações aninhadas, enquanto outros não.

Exemplo prático usando transações com PDO

Na seção a seguir é demonstrado um exemplo prático do mundo real onde o uso de transações garante a consistência do banco de dados.

Imagine o seguinte cenário, digamos que você esteja construindo um carrinho de compras para um site de comércio eletrônico e decidiu manter os pedidos em duas tabelas de banco de dados. Um chamado orders com os campos order_id, name, address, telephone e created_at. E um segundo chamado orders_products com os campos order_id, product_id e quantity. A primeira tabela contém os metadados do pedido, enquanto a segunda os produtos reais que foram pedidos.

Inserindo um novo pedido no banco de dados

Para inserir um novo pedido no banco de dados, você precisa fazer duas coisas. Primeiro você precisa INSERIR um novo registro dentro da tabela orders que conterá os metadados do pedido (name, address, etc). E então você precisa INSERIR um registro na tabela orders_products, para cada um dos produtos incluídos no pedido.

Você pode fazer isso fazendo algo semelhante ao seguinte:

// Insert the metadata of the order into the database
$preparedStatement = $db->prepare(
    'INSERT INTO `orders` (`name`, `address`, `telephone`, `created_at`)
     VALUES (:name, :address, :telephone, :created_at)'
);

$preparedStatement->execute([
    'name' => $name,
    'address' => $address,
    'telephone' => $telephone,
    'created_at' => time(),
]);

// Get the generated `order_id`
$orderId = $db->lastInsertId();

// Construct the query for inserting the products of the order
$insertProductsQuery = 'INSERT INTO `orders_products` (`order_id`, `product_id`, `quantity`) VALUES';

$count = 0;
foreach ( $products as $productId => $quantity ) {
    $insertProductsQuery .= ' (:order_id' . $count . ', :product_id' . $count . ', :quantity' . $count . ')';
    
    $insertProductsParams['order_id' . $count] = $orderId;
    $insertProductsParams['product_id' . $count] = $productId;
    $insertProductsParams['quantity' . $count] = $quantity;
    
    ++$count;
}

// Insert the products included in the order into the database
$preparedStatement = $db->prepare($insertProductsQuery);
$preparedStatement->execute($insertProductsParams);

Isso funcionará muito bem para inserir um novo pedido no banco de dados, até que algo inesperado aconteça e, por algum motivo, a segunda consulta INSERT falhe. Se isso acontecer, você terminará com um novo pedido dentro da tabela pedidos, que não terá produtos associados a ele. Felizmente, a correção é muito simples, tudo o que você precisa fazer é fazer as consultas na forma de uma única transação de banco de dados.

Inserindo um novo pedido no banco de dados com uma transação

Para iniciar uma transação usando PDO tudo que você precisa fazer é chamar o método beginTransaction antes de executar qualquer consulta ao seu banco de dados. Em seguida, você faz as alterações desejadas em seus dados executando consultas INSERT e/ou UPDATE. E finalmente você chama o método commit do objeto PDO para tornar as alterações permanentes. Até que você chame o método commit, todas as alterações feitas em seus dados até este ponto ainda não são permanentes e podem ser facilmente revertidas simplesmente chamando o método rollback do objeto PDO.

No exemplo a seguir é demonstrado o uso de transações para inserir um novo pedido no banco de dados, garantindo ao mesmo tempo a consistência dos dados. Se uma das duas consultas falhar, todas as alterações serão revertidas.

// In this example we are using MySQL but this applies to any database that has support for transactions
$db = new PDO('mysql:host=' . $host . ';dbname=' . $dbname . ';charset=utf8', $username, $password);    

// Make sure that PDO will throw an exception in case of error to make error handling easier
$db->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

try {
    // From this point and until the transaction is being committed every change to the database can be reverted
    $db->beginTransaction();    
    
    // Insert the metadata of the order into the database
    $preparedStatement = $db->prepare(
        'INSERT INTO `orders` (`order_id`, `name`, `address`, `created_at`)
         VALUES (:name, :address, :telephone, :created_at)'
    );
    
    $preparedStatement->execute([
        'name' => $name,
        'address' => $address,
        'telephone' => $telephone,
        'created_at' => time(),
    ]);
    
    // Get the generated `order_id`
    $orderId = $db->lastInsertId();

    // Construct the query for inserting the products of the order
    $insertProductsQuery = 'INSERT INTO `orders_products` (`order_id`, `product_id`, `quantity`) VALUES';
    
    $count = 0;
    foreach ( $products as $productId => $quantity ) {
        $insertProductsQuery .= ' (:order_id' . $count . ', :product_id' . $count . ', :quantity' . $count . ')';
        
        $insertProductsParams['order_id' . $count] = $orderId;
        $insertProductsParams['product_id' . $count] = $productId;
        $insertProductsParams['quantity' . $count] = $quantity;
        
        ++$count;
    }
    
    // Insert the products included in the order into the database
    $preparedStatement = $db->prepare($insertProductsQuery);
    $preparedStatement->execute($insertProductsParams);
    
    // Make the changes to the database permanent
    $db->commit();
}
catch ( PDOException $e ) { 
    // Failed to insert the order into the database so we rollback any changes
    $db->rollback();
    throw $e;
}

PDO: conectando ao servidor MySQL/MariaDB #

Existem duas maneiras de se conectar a um servidor MySQL/MariaDB, dependendo de sua infraestrutura.

Conexão padrão (TCP/IP) #

$dsn = 'mysql:dbname=demo;host=server;port=3306;charset=utf8';
$connection = new \PDO($dsn, $username, $password);

// throw exceptions, when SQL error is caused
$connection->setAttribute(\PDO::ATTR_ERRMODE, \PDO::ERRMODE_EXCEPTION);
// prevent emulation of prepared statements
$connection->setAttribute(\PDO::ATTR_EMULATE_PREPARES, false);

Como o PDO foi projetado para ser compatível com versões mais antigas do servidor MySQL (que não tinham suporte para instruções preparadas), você deve desabilitar explicitamente a emulação. Caso contrário, você perderá os benefícios adicionais de prevenção de injeção, que geralmente são concedidos usando declarações preparadas.

Outro compromisso de design, que você deve ter em mente, é o comportamento padrão de tratamento de erros. Se não for configurado de outra forma, o PDO não mostrará nenhuma indicação de erros de SQL.

É altamente recomendável configurá-lo para “modo de exceção”, porque isso lhe dá funcionalidade adicional, ao escrever abstrações de persistência (por exemplo: ter uma exceção, ao violar a restrição UNIQUE).

Conexão de soquete #

$dsn = 'mysql:unix_socket=/tmp/mysql.sock;dbname=demo;charset=utf8';
$connection = new \PDO($dsn, $username, $password);

// throw exceptions, when SQL error is caused
$connection->setAttribute(\PDO::ATTR_ERRMODE, \PDO::ERRMODE_EXCEPTION);
// prevent emulation of prepared statements
$connection->setAttribute(\PDO::ATTR_EMULATE_PREPARES, false);

Em sistemas do tipo unix, se o nome do host for 'localhost', a conexão com o servidor será feita através de um soquete de domínio.

PDO: Obtém o número de linhas afetadas por uma consulta #

Começamos com $db, uma instância da classe PDO. Depois de executar uma consulta, muitas vezes queremos determinar o número de linhas que foram afetadas por ela. O método rowCount() do PDOStatement funcionará bem:

$query = $db->query("DELETE FROM table WHERE name = 'John'");
$count = $query->rowCount();

echo "Deleted $count rows named John";

NOTA: Este método só deve ser usado para determinar o número de linhas afetadas pelas instruções INSERT, DELETE e UPDATE. Embora esse método também funcione para instruções SELECT, ele não é consistente em todos os bancos de dados.

PDO::lastInsertId() #

Muitas vezes, você pode encontrar a necessidade de obter o valor de ID incrementado automaticamente para uma linha que acabou de inserir em sua tabela de banco de dados. Você pode conseguir isso com o método lastInsertId().

// 1. Basic connection opening (for MySQL)
$host = 'localhost';
$database = 'foo';
$user = 'root'
$password = '';
$dsn = "mysql:host=$host;dbname=$database;charset=utf8";
$pdo = new PDO($dsn, $user, $password);

// 2. Inserting an entry in the hypothetical table 'foo_user'
$query = "INSERT INTO foo_user(pseudo, email) VALUES ('anonymous', 'anonymous@example.com')";
$query_success = $pdo->query($query);

// 3. Retrieving the last inserted id
$id = $pdo->lastInsertId(); // return value is an integer

No postgresql e oracle, existe a palavra-chave RETURNING, que retorna as colunas especificadas das linhas atualmente inseridas/modificadas. Aqui exemplo para inserir uma entrada:

// 1. Basic connection opening (for PGSQL)
$host = 'localhost';
$database = 'foo';
$user = 'root'
$password = '';
$dsn = "pgsql:host=$host;dbname=$database;charset=utf8";
$pdo = new PDO($dsn, $user, $password);

// 2. Inserting an entry in the hypothetical table 'foo_user'
$query = "INSERT INTO foo_user(pseudo, email) VALUES ('anonymous', 'anonymous@example.com') RETURNING id";
$statement = $pdo->query($query);

// 3. Retrieving the last inserted id
$id = $statement->fetchColumn();  // return the value of the id column of the new row in foo_user
