O driver SQLSRV é uma extensão PHP com suporte da Microsoft que permite acessar bancos de dados Microsoft SQL Server e SQL Azure. É uma alternativa para os drivers MSSQL que foram preteridos a partir do PHP 5.3 e foram removidos do PHP 7.

A extensão SQLSRV pode ser usada nos seguintes sistemas operacionais:

  • Windows Vista Service Pack 2 ou posterior
  • Windows Server 2008 Service Pack 2 ou posterior
  • Windows Server 2008 R2
  • Windows 7

A extensão SQLSRV requer que o Microsoft SQL Server 2012 Native Client seja instalado no mesmo computador que está executando o PHP. Se o Microsoft SQL Server 2012 Native Client ainda não estiver instalado, clique no link apropriado na página de documentação “Requisitos”.

Para baixar os drivers SQLSRV mais recentes, acesse: Download

Uma lista completa de requisitos de sistema para os drivers SQLSRV pode ser encontrada aqui: Requisitos do sistema

Aqueles que usam SQLSRV 3.1+ devem baixar o Microsoft ODBC Driver 11 for SQL Server

Os usuários do PHP7 podem baixar os drivers mais recentes do GitHub

Microsoft® ODBC Driver 13 for SQL Server suporta Microsoft SQL Server 2008, SQL Server 2008 R2, SQL Server 2012, SQL Server 2014, SQL Server 2016 (pré-visualização), Analytics Platform System, Azure SQL Database e Azure SQL Data Warehouse.

Buscando resultados da consulta #

Existem 3 maneiras principais de buscar resultados de uma consulta:

sqlsrv_fetch_array() #

sqlsrv_fetch_array() recupera a próxima linha como um array.

$stmt = sqlsrv_query($conn, $query);

while($row = sqlsrv_fetch_array($stmt)) {
    echo $row[0];
    $var = $row["name"];
    //...
}

sqlsrv_fetch_array() tem um segundo parâmetro opcional para buscar diferentes tipos de array: SQLSRV_FETCH_ASSOC, SQLSRV_FETCH_NUMERIC e SQLSRV_FETCH_BOTH(padrão) podem ser usados; cada um retorna as matrizes associativas, numéricas ou associativas e numéricas, respectivamente.

sqlsrv_fetch_object() #

sqlsrv_fetch_object() recupera a próxima linha como um objeto.

$stmt = sqlsrv_query($conn, $query);

while($obj = sqlsrv_fetch_object($stmt)) {
    echo $obj->field; // Object property names are the names of the fields from the query
    //...
}

sqlsrv_fetch() #

sqlsrv_fetch() torna a próxima linha disponível para leitura.

$stmt = sqlsrv_query($conn, $query);

while(sqlsrv_fetch($stmt) === true) {
    $foo = sqlsrv_get_field($stmt, 0); //gets the first field -
}

Recuperando mensagens de erro #

Quando uma consulta dá errado, é importante buscar a(s) mensagem(ns) de erro retornada(s) pelo driver para identificar a causa do problema. A sintaxe é:

sqlsrv_errors([int $errorsOrWarnings]);

Isso retorna uma matriz com:

Chave Descrição
SQLSTATE O estado em que o driver SQL Server/OBDC está
código O código de erro do SQL Server
mensagem A descrição do erro

É comum usar a função acima assim:

$brokenQuery = "SELECT BadColumnName FROM Table_1";
$stmt = sqlsrv_query($conn, $brokenQuery);

if ($stmt === false) {
    if (($errors = sqlsrv_errors()) != null) {
        foreach ($errors as $error) {
            echo "SQLSTATE: ".$error['SQLSTATE']."<br />";
            echo "code: ".$error['code']."<br />";
            echo "message: ".$error['message']."<br />";
        }
    }
}

Criando uma conexão #

$dbServer = "localhost,1234"; //Name of the server/instance, including optional port number (default is 1433)
$dbName = "db001"; //Name of the database
$dbUser = "user"; //Name of the user
$dbPassword = "password"; //DB Password of that user

$connectionInfo = array(
    "Database" => $dbName, 
    "UID" => $dbUser,
    "PWD" => $dbPassword
);

$conn = sqlsrv_connect($dbServer, $connectionInfo);

SQLSRV também possui um driver PDO. Para conectar usando PDO:

$conn = new PDO("sqlsrv:Server=localhost,1234;Database=db001", $dbUser, $dbPassword);

Fazendo uma consulta simples #

//Create Connection
$conn = sqlsrv_connect($dbServer, $connectionInfo);

$query = "SELECT * FROM [table]"; 
$stmt = sqlsrv_query($conn, $query);

Observação: o uso de colchetes [] é para escapar da palavra tabela pois é uma [palavra reservada](https://msdn.microsoft.com/en-us/library/ms189822 .aspx). Estes funcionam da mesma forma que os backticks ` do no MySQL.

Chamando um procedimento armazenado #

Para chamar um procedimento armazenado no servidor:

$query = "{call [dbo].[myStoredProcedure](?,?,?)}"; //Parameters '?' includes OUT parameters

$params = array(
    array($name, SQLSRV_PARAM_IN),
    array($age, SQLSRV_PARAM_IN),
    array($count, SQLSRV_PARAM_OUT, SQLSRV_PHPTYPE_INT) //$count must already be initialised
);

$result = sqlsrv_query($conn, $query, $params);

Fazendo uma consulta parametrizada #

$conn = sqlsrv_connect($dbServer, $connectionInfo);

$query = "SELECT * FROM [users] WHERE [name] = ? AND [password] = ?";
$params = array("joebloggs", "pa55w0rd");

$stmt = sqlsrv_query($conn, $query, $params);

Se você planeja usar a mesma instrução de consulta mais de uma vez, com parâmetros diferentes, o mesmo pode ser obtido com as funções sqlsrv_prepare() e sqlsrv_execute(), conforme mostrado abaixo:

$cart = array(
    "apple" => 3,
    "banana" => 1,
    "chocolate" => 2
);

$query = "INSERT INTO [order_items]([item], [quantity]) VALUES(?,?)";
$params = array(&$item, &$qty); //Variables as parameters must be passed by reference

$stmt = sqlsrv_prepare($conn, $query, $params);

foreach($cart as $item => $qty){
    if(sqlsrv_execute($stmt) === FALSE) {
        die(print_r(sqlsrv_errors(), true));
    }
}
