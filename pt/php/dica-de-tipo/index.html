Sintaxe #

  • função f(ClassName $param) {}
  • função f(bool $param) {}
  • função f(int $param) {}
  • função f(float $param) {}
  • função f(string $param) {}
  • função f(self $param) {}
  • função f(chamável $param) {}
  • função f(array $param) {}
  • function f(?type_name $param) {}
  • função f() : type_name {}
  • função f() : void {}
  • função f() : ?type_name {}

Dicas de tipo ou declarações de tipo são uma prática de programação defensiva que garante que os parâmetros de uma função sejam de um tipo especificado. Isso é particularmente útil ao digitar dicas para uma interface, pois permite que a função garanta que um parâmetro fornecido terá os mesmos métodos exigidos na interface.

Passar o tipo incorreto para uma função com dica de tipo levará a um erro fatal:

Erro fatal: TypeError não capturado: Argumento X passado para foo() deve ser de o tipo RequiredType, ProvidedType fornecido

Tipos escalares de dicas de tipo, arrays e callables #

Suporte para parâmetros de array de dicas de tipo (e valores de retorno após o PHP 7.1) foi adicionado no PHP 5.1 com a palavra-chave array. Quaisquer matrizes de quaisquer dimensões e tipos, bem como matrizes vazias, são valores válidos.

Suporte para callables de dica de tipo foi adicionado no PHP 5.4. Qualquer valor que is_callable() seja válido para parâmetros e valores de retorno sugeridos como callable, ou seja, objetos Closure, strings de nome de função e array(class_name|object, method_name).

Se ocorrer um erro de digitação no nome da função que não seja is_callable(), uma mensagem de erro menos óbvia será exibida:

Erro fatal: TypeError não capturado: o argumento 1 passado para foo() deve ser do tipo callable, string/array fornecido

function foo(callable $c) {}
foo("count"); // valid
foo("Phar::running"); // valid
foo(["Phar", "running"); // valid
foo([new ReflectionClass("stdClass"), "getName"]); // valid
foo(function() {}); // valid

foo("no_such_function"); // callable expected, string given

Métodos não estáticos também podem ser passados ​​como callables em formato estático, resultando em um aviso de depreciação e erro de nível E_STRICT no PHP 7 e 5, respectivamente.

A visibilidade do método é levada em consideração. Se o contexto do método com o parâmetro callable não tiver acesso ao callable fornecido, terminará como se o método não existisse.

class Foo{
  private static function f(){
    echo "Good" . PHP_EOL;
  }

  public static function r(callable $c){
    $c();
  }
}

function r(callable $c){}

Foo::r(["Foo", "f"]);
r(["Foo", "f"]);

Resultado:

Erro fatal: TypeError não capturado: o argumento 1 passado para r() deve ser chamado, array fornecido

Suporte para tipos escalares de dica de tipo foi adicionado no PHP 7. Isso significa que ganhamos suporte de dica de tipo para booleans, integers, floats e strings.

<?php

function add(int $a, int $b) {
    return $a + $b;
}

var_dump(add(1, 2)); // Outputs "int(3)"

Por padrão, o PHP tentará converter qualquer argumento fornecido para corresponder à sua dica de tipo. Mudar a chamada para add(1.5, 2) dá exatamente a mesma saída, já que o float 1.5 foi convertido para int pelo PHP.

Para parar este comportamento, deve-se adicionar declare(strict_types=1); no topo de cada arquivo fonte PHP que o requeira.

<?php

declare(strict_types=1);

function add(int $a, int $b) {
    return $a + $b;
}

var_dump(add(1.5, 2));

O script acima agora produz um erro fatal:

Fatal error: Uncaught TypeError: Argumento 1 passado para add() deve ser do tipo inteiro, dado float

Uma exceção: tipos especiais #

Algumas funções PHP podem retornar um valor do tipo resource. Como este não é um tipo escalar, mas um tipo especial, não é possível digitá-lo como dica.

Como exemplo, curl_init() retornará um resource, assim como fopen(). Claro, esses dois recursos não são compatíveis entre si. Por causa disso, o PHP 7 sempre lançará o seguinte TypeError ao digitar resource explicitamente:

TypeError: Argumento 1 passado para sample() deve ser uma instância de resource, resource dado

Classes e interfaces de dica de tipo #

A dica de tipo para classes e interfaces foi adicionada no PHP 5.

Dica de tipo de classe #

<?php

class Student
{
    public $name = 'Chris';
}

class School
{
    public $name = 'University of Edinburgh';
}

function enroll(Student $student, School $school)
{
    echo $student->name . ' is being enrolled at ' . $school->name;
}

$student = new Student();
$school = new School();

enroll($student, $school);

O script acima gera:

Chris está matriculado na Universidade de Edimburgo

Dica de tipo de interface #

<?php

interface Enrollable {};
interface Attendable {};

class Chris implements Enrollable
{
    public $name = 'Chris';
}

class UniversityOfEdinburgh implements Attendable
{
    public $name = 'University of Edinburgh';
}

function enroll(Enrollable $enrollee, Attendable $premises)
{
    echo $enrollee->name . ' is being enrolled at ' . $premises->name;
}

$chris = new Chris();
$edinburgh = new UniversityOfEdinburgh();

enroll($chris, $edinburgh);

O exemplo acima produz o mesmo que antes:

Chris está matriculado na Universidade de Edimburgo

Dicas de auto tipo #

A palavra-chave self pode ser usada como uma dica de tipo para indicar que o valor deve ser uma instância da classe que declara o método.

Dicas de tipo anuláveis #

Parâmetros #

A dica de tipo anulável foi adicionada no PHP 7.1 usando o operador ? antes da dica de tipo.

function f(?string $a) {}
function g(string $a) {}

f(null); // valid
g(null); // TypeError: Argument 1 passed to g() must be of the type string, null given

Antes do PHP 7.1, se um parâmetro tivesse uma dica de tipo, ele deveria declarar um valor padrão null para aceitar valores nulos.

function f(string $a = null) {}
function g(string $a) {}

f(null); // valid
g(null); // TypeError: Argument 1 passed to g() must be of the type string, null given

Retorna valores #

No PHP 7.0, funções com tipo de retorno não devem retornar null.

No PHP 7.1, as funções podem declarar uma dica de tipo de retorno anulável. No entanto, a função ainda deve retornar null, não void (instruções de retorno sem/vazia).

function f() : ?string {
    return null;
}

function g() : ?string {}
function h() : ?string {}

f(); // OK
g(); // TypeError: Return value of g() must be of the type string or null, none returned
h(); // TypeError: Return value of h() must be of the type string or null, none returned

Tipo de dica de objetos genéricos #

Como os objetos PHP não herdam de nenhuma classe base (incluindo stdClass), não há suporte para dicas de tipo para um tipo de objeto genérico.

Por exemplo, o abaixo não funcionará.

<?php

function doSomething(object $obj) {
    return $obj;
}

class ClassOne {}
class ClassTwo {}

$classOne= new ClassOne();
$classTwo= new ClassTwo();

doSomething($classOne);
doSomething($classTwo);

E lançará um erro fatal:

Fatal error: Uncaught TypeError: Argumento 1 passado para doSomething() deve ser uma instância de objeto, instância de OperationOne fornecida

Uma solução para isso é declarar uma interface degenerada que não define métodos e fazer com que todos os seus objetos implementem essa interface.

<?php

interface Object {}

function doSomething(Object $obj) {
    return $obj;
}

class ClassOne implements Object {}
class ClassTwo implements Object {}

$classOne = new ClassOne();
$classTwo = new ClassTwo();

doSomething($classOne);
doSomething($classTwo);

Dica de tipo sem retorno (void) #

No PHP 7.1, o tipo de retorno void foi adicionado. Embora o PHP não tenha um valor void real, é geralmente entendido nas linguagens de programação que uma função que não retorna nada está retornando void. Isso não deve ser confundido com o retorno de null, pois null é um valor que pode ser retornado.

function lacks_return(): void {
    // valid
}

Observe que, se você declarar um retorno void, não poderá retornar nenhum valor ou receberá um erro fatal:

function should_return_nothing(): void {
    return null; // Fatal error: A void function must not return a value
}

No entanto, usar return para sair da função é válido:

function returns_nothing(): void {
    return; // valid
}
