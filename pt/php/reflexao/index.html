Acessando variáveis ​​de membro privadas e protegidas #

A reflexão é frequentemente usada como parte do teste de software, como para a criação/instanciação em tempo de execução de objetos simulados. Também é ótimo para inspecionar o estado de um objeto a qualquer momento. Aqui está um exemplo de como usar o Reflection em um teste de unidade para verificar se um membro de classe protegido contém o valor esperado.

Abaixo está uma classe muito básica para um carro. Ele possui uma variável membro protegida que conterá o valor que representa a cor do carro. Como a variável de membro é protegida, não podemos acessá-la diretamente e devemos usar um método getter e setter para recuperar e definir seu valor, respectivamente.

class Car
{
    protected $color
    
    public function setColor($color)
    {
        $this->color = $color;
    }
    
    public function getColor($color)
    {
        return $this->color;
    }
}

Para testar isso, muitos desenvolvedores criarão um objeto Car, definir a cor do carro usando Car::setColor(), recuperar a cor usando Car::getColor() e comparar esse valor com a cor definida:

/**
 * @test
 * @covers     \Car::setColor
 */
public function testSetColor()
{
    $color = 'Red';

    $car = new \Car();
    $car->setColor($color);
    $getColor = $car->getColor();
        
    $this->assertEquals($color, $reflectionColor);
}

Na superfície, isso parece bom. Afinal, tudo que Car::getColor() faz é retornar o valor da variável de membro protegida Car::$color. Mas este teste é falho de duas maneiras:

  1. Exercita Car::getColor() que está fora do escopo deste teste
  2. Depende de Car::getColor() que pode ter um bug que pode fazer o teste ter um falso positivo ou negativo

Vamos ver porque não devemos usar Car::getColor() em nosso teste de unidade e devemos usar Reflection. Digamos que um desenvolvedor receba a tarefa de adicionar “Metálico” a cada cor de carro. Então eles tentam modificar o Car::getColor() para acrescentar “Metallic” à cor do carro:

class Car
{
    protected $color
    
    public function setColor($color)
    {
        $this->color = $color;
    }
    
    public function getColor($color)
    {
        return "Metallic "; $this->color;
    }
}

Você vê o erro? O desenvolvedor usou um ponto e vírgula em vez do operador de concatenação na tentativa de acrescentar “Metálico” à cor do carro. Como resultado, sempre que Car::getColor() for chamado, “Metallic " será retornado independentemente da cor real do carro. Como resultado, nosso teste de unidade Car::setColor() falhará mesmo que Car::setColor() funcione perfeitamente e não tenha sido afetado por essa alteração.

Então, como verificamos se Car::$color contém o valor que estamos configurando via Car::setColor()? Podemos usar Refelection para inspecionar a variável de membro protegida diretamente. Então, como fazemos isso? Podemos usar Refelection para tornar a variável de membro protegida acessível ao nosso código para que ele possa recuperar o valor.

Vamos ver o código primeiro e depois dividi-lo:

/**
 * @test
 * @covers     \Car::setColor
 */
public function testSetColor()
{
    $color = 'Red';

    $car = new \Car();
    $car->setColor($color);
    
    $reflectionOfCar = new \ReflectionObject($car);
    $protectedColor = $reflectionOfForm->getProperty('color');
    $protectedColor->setAccessible(true);
    $reflectionColor = $protectedColor->getValue($car);
    
    $this->assertEquals($color, $reflectionColor);
}

Aqui está como estamos usando o Reflection para obter o valor de Car::$color no código acima:

  1. Criamos um novo ReflectionObject representando nosso objeto Car
  2. Obtemos uma ReflectionProperty para Car::$color (isto “representa” a variável Car::$color)
  3. Tornamos Car::$color acessível
  4. Obtemos o valor de Car::$color

Como você pode ver usando o Reflection, podemos obter o valor de Car::$color sem ter que chamar Car::getColor() ou qualquer outra função de acesso que poderia causar resultados de teste inválidos. Agora nosso teste de unidade para Car::setColor() é seguro e preciso.

Detecção de recursos de classes ou objetos #

A detecção de recursos de classes pode ser feita parcialmente com as funções property_exists e method_exists.

class MyClass {
    public $public_field;
    protected $protected_field;
    private $private_field;
    static $static_field;
    const CONSTANT = 0;
    public function public_function() {}
    protected function protected_function() {}
    private function private_function() {}
    static function static_function() {}
}

// check properties
$check = property_exists('MyClass', 'public_field');    // true
$check = property_exists('MyClass', 'protected_field'); // true
$check = property_exists('MyClass', 'private_field');   // true, as of PHP 5.3.0
$check = property_exists('MyClass', 'static_field');    // true
$check = property_exists('MyClass', 'other_field');     // false

// check methods
$check = method_exists('MyClass', 'public_function');    // true
$check = method_exists('MyClass', 'protected_function');    // true
$check = method_exists('MyClass', 'private_function');    // true
$check = method_exists('MyClass', 'static_function');    // true

// however...
$check = property_exists('MyClass', 'CONSTANT');  // false
$check = property_exists($object, 'CONSTANT');    // false

Com um ReflectionClass, também constantes podem ser detectadas:

$r = new ReflectionClass('MyClass');
$check = $r->hasProperty('public_field');  // true
$check = $r->hasMethod('public_function'); // true
$check = $r->hasConstant('CONSTANT');      // true
// also works for protected, private and/or static members.

Nota: para property_exists e method_exists, também pode ser fornecido um objeto da classe de interesse em vez do nome da classe. Usando reflexão, a classe ReflectionObject deve ser usada em vez de ReflectionClass.

Testando métodos privados/protegidos #

Às vezes é útil testar métodos privados e protegidos, bem como os públicos.

class Car
{
    /**
     * @param mixed $argument
     *
     * @return mixed
     */
    protected function drive($argument)
    {
        return $argument;
    }

    /**
     * @return bool
     */
    private static function stop()
    {
        return true;
    }
}

A maneira mais fácil de testar o método de drive é usar reflexão

class DriveTest
{
    /**
     * @test
     */
    public function testDrive()
    {
        // prepare
        $argument = 1;
        $expected = $argument;
        $car = new \Car();

        $reflection = new ReflectionClass(\Car::class);
        $method = $reflection->getMethod('drive');
        $method->setAccessible(true);

        // invoke logic
        $result = $method->invokeArgs($car, [$argument]);

        // test
        $this->assertEquals($expected, $result);
    }
}

Se o método for estático você passa null no lugar da instância da classe

class StopTest
{
    /**
     * @test
     */
    public function testStop()
    {
        // prepare
        $expected = true;

        $reflection = new ReflectionClass(\Car::class);
        $method = $reflection->getMethod('stop');
        $method->setAccessible(true);

        // invoke logic
        $result = $method->invoke(null);

        // test
        $this->assertEquals($expected, $result);
    }
}
