Noções básicas de texturização #

Uma textura é uma forma de armazenamento de dados que permite acesso conveniente não apenas a entradas de dados específicas, mas também a pontos de amostra que misturam (interpolam) várias entradas juntas.

No OpenGL, as texturas podem ser usadas para muitas coisas, mas o mais comum é mapear uma imagem para um polígono (por exemplo, um triângulo). Para mapear a textura para um triângulo (ou outro polígono) temos que dizer a cada vértice a qual parte da textura ele corresponde. Atribuímos uma coordenada de textura a cada vértice de um polígono e ela será então interpolada entre todos os fragmentos desse polígono. As coordenadas de textura normalmente variam de 0 a 1 nos eixos x e y, conforme mostrado na imagem abaixo:

As coordenadas de textura deste triângulo ficariam assim:

GLfloat texCoords[] = {
    0.0f, 0.0f,  // Lower-left corner  
    1.0f, 0.0f,  // Lower-right corner
    0.5f, 1.0f   // Top-center corner
};

Coloque essas coordenadas no VBO (objeto de buffer de vértice) e crie um novo atributo para o sombreador. Você já deve ter pelo menos um atributo para as posições dos vértices, então crie outro para as coordenadas da textura.

Gerando textura #

A primeira coisa a fazer será gerar um objeto de textura que será referenciado por um ID que será armazenado em um unsigned int texture:

GLuint texture;
glGenTextures(1, &texture);

Depois disso, ele deve ser vinculado para que todos os comandos de textura subsequentes configurem essa textura:

glBindTexture(GL_TEXTURE_2D, texture);

Carregando imagem #

Para carregar uma imagem você pode criar seu próprio carregador de imagem ou você pode usar uma biblioteca de carregamento de imagem como SOIL (Simple OpenGL Image Library) em c++ ou TWL’s PNGDecoder em java.

Um exemplo de carregamento de imagem com SOIL seria:

int width, height;
unsigned char* image = SOIL_load_image("image.png", &width, &height, 0, SOIL_LOAD_RGB);

Agora você pode atribuir esta imagem ao objeto de textura:

glTexImage2D(GL_TEXTURE_2D, 0, GL_RGB, width, height, 0, GL_RGB, GL_UNSIGNED_BYTE, image);

Depois disso, você deve desvincular o objeto de textura:

glBindTexture(GL_TEXTURE_2D, 0);

Parâmetro wrap para coordenadas de textura #

Como visto acima, o canto inferior esquerdo da textura tem as coordenadas UV (st) (0, 0) e o canto superior direito da textura tem as coordenadas (1, 1), mas as coordenadas da textura de uma malha podem estar em qualquer intervalo. Para lidar com isso, deve-se definir como as coordenadas da textura são agrupadas na textura.

O parâmetro wrap para a coordenada de textura pode ser definido com glTextureParameter usando GL_TEXTURE_WRAP_S, GL_TEXTURE_WRAP_T e GL_TEXTURE_WRAP_R .

glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

Os parâmetros possíveis são:

  • GL_CLAMP_TO_EDGE faz com que as coordenadas da textura sejam fixadas no intervalo [1/2N, 1 - 1/2N], onde N é o tamanho da textura na direção.

  • GL_CLAMP_TO_BORDER faz o mesmo que GL_CLAMP_TO_EDGE, mas em casos de fixação, os dados de texel obtidos são substituídos pela cor especificada por GL_TEXTURE_BORDER_COLOR.

  • GL_REPEAT faz com que a parte inteira da coordenada de textura seja ignorada. A textura é ladrilhada.

  • GL_MIRRORED_REPEAT: Se a parte inteira da coordenada de textura for par, então ela será ignorada. Em contraste com, se a parte inteira da coordenada de textura for ímpar, então a coordenada de textura será definida como 1 - frac(s). fract(s) é a parte fracionária da coordenada de textura. Isso faz com que a textura seja espelhada a cada 2 vezes.

  • GL_MIRROR_CLAMP_TO_EDGE faz com que a coordenada do texto seja repetida como para GL_MIRRORED_REPEAT para uma repetição da textura, ponto em que a coordenada será fixada como em GL_CLAMP_TO_EDGE.

Observe que o valor padrão para GL_TEXTURE_WRAP_S, GL_TEXTURE_WRAP_T e GL_TEXTURE_WRAP_R é GL_REPEAT.

Aplicando texturas #

A última coisa a fazer é vincular a textura antes da chamada de desenho:

glBindTexture(GL_TEXTURE_2D, texture);

Textura e Framebuffer #

Você pode anexar uma imagem em uma textura a um framebuffer, para que possa renderizar diretamente nessa textura.

glGenFramebuffers (1, &framebuffer);
glBindFramebuffer (GL_FRAMEBUFFER, framebuffer);
glFramebufferTexture2D(GL_FRAMEBUFFER,
                       GL_COLOR_ATTACHMENT0,
                       GL_TEXTURE_2D,
                       texture,
                       0);

Observação: você não pode ler e escrever na mesma textura na mesma tarefa de renderização, porque chama um comportamento indefinido. Mas você pode usar: glTextureBarrier() entre chamadas de renderização para isso.

Ler dados de textura #

Você pode ler dados de textura com a função glGetTexImage:

char *outBuffer = malloc(buf_size);

glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, texture);

glGetTexImage(GL_TEXTURE_2D,
              0,
              GL_RGBA,
              GL_UNSIGNED_BYTE,
              outBuffer);

Observação: tipo e formato de textura são apenas para exemplo e podem ser diferentes.

Usando texturas em shaders GLSL #

O vertex shader só aceita as coordenadas de textura como um atributo de vértice e encaminha as coordenadas para o fragment shader. Por padrão, também garantirá que o fragmento receba a coordenada interpolada corretamente com base em sua posição em um triângulo:

layout (location = 0) in vec3 position;
layout (location = 1) in vec2 texCoordIn;

out vec2 texCoordOut;

void main()
{
    gl_Position = vec4(position, 1.0f);
    texCoordOut = texCoordIn;
}

O fragment shader então aceita a variável de saída texCoord como uma variável de entrada. Você pode então adicionar uma textura ao sombreador de fragmento declarando um ‘uniforme sampler2D’. Para amostrar um fragmento da textura, usamos uma função interna texture que possui dois parâmetros. A primeira é a textura da qual queremos amostrar e a segunda é a coordenada dessa textura:

in vec2 texCoordOut;

out vec4 color;

uniform sampler2D image;

void main()
{
    color = texture(image, texCoordOut);
}

Observe que image não é o id direto da textura aqui. É o id da unidade de textura que será amostrada. Por sua vez, as texturas não estão vinculadas diretamente aos programas; eles estão vinculados a unidades de textura. Isto é conseguido primeiro tornando a unidade de textura ativa com glActiveTexture, e então chamar glBindTexture afetará esta unidade de textura em particular. No entanto, como a unidade de textura padrão é a unidade de textura 0, os programas que usam uma textura podem ser simplificados omitindo essa chamada.

Usando PBOs #

Se você vincular um buffer a GL_PIXEL_UNPACK_BUFFER, então o parâmetro data em glTexImage2D será um deslocamento nesse buffer.

Isso significa que o glTexImage2D não precisa esperar que todos os dados sejam copiados da memória do aplicativo antes de retornar, reduzindo a sobrecarga no thread principal.

glGenBuffers(1, &pbo);
glBindBuffer(GL_PIXEL_UNPACK_BUFFER, pbo);
glBufferData(GL_PIXEL_UNPACK_BUFFER, width*height*3, NULL, GL_STREAM_DRAW);
void* mappedBuffer = glMapBuffer(GL_PIXEL_UNPACK_BUFFER, GL_WRITE_ONLY);

//write data into the mapped buffer, possibly in another thread.
int width, height;
unsigned char* image = SOIL_load_image("image.png", &width, &height, 0, SOIL_LOAD_RGB);
memcpy(mappedBuffer, image, width*height*3);
SOIL_free_image(image);

// after reading is complete back on the main thread
glBindBuffer(GL_PIXEL_UNPACK_BUFFER, pbo);
glUnmapBuffer(GL_PIXEL_UNPACK_BUFFER);
glTexImage2D(GL_TEXTURE_2D, 0, GL_RGB, width, height, 0, GL_RGB, GL_UNSIGNED_BYTE, 0);

Mas onde os PBOs realmente brilham é quando você precisa ler o resultado de uma renderização na memória do aplicativo. Para ler dados de pixel em um buffer, vincule-o a GL_PIXEL_PACK_BUFFER, então o parâmetro de dados de glGetTexImage será um deslocamento nesse buffer:

glGenBuffers(1, &pbo);
glBindBuffer(GL_PIXEL_PACK_BUFFER, pbo);
glBufferData(GL_PIXEL_PACK_BUFFER, buf_size, NULL, GL_STREAM_COPY);

glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, texture);

glGetTexImage(GL_TEXTURE_2D,
              0,
              GL_RGBA,
              GL_UNSIGNED_BYTE,
              null);
//ensure we don't try and read data before the transfer is complete
GLsync sync = glFenceSync(GL_SYNC_GPU_COMMANDS_COMPLETE, 0);

// then regularly check for completion
GLint result;
glGetSynciv(sync, GL_SYNC_STATUS, sizeof(result), NULL, &result);
if(result == GL_SIGNALED){
    glBindBuffer(GL_PIXEL_PACK_BUFFER, pbo);
    void* mappedBuffer = glMapBuffer(GL_PIXEL_PACK_BUFFER, GL_READ_ONLY);

    //now mapped buffer contains the pixel data

    glUnmapBuffer(GL_PIXEL_PACK_BUFFER);

}
