Implemente uma câmera no OGL 4.0 GLSL 400 #

Se queremos olhar para uma cena como se a tivéssemos fotografado com uma câmera, devemos primeiro definir algumas coisas:

  • A posição a partir da qual a cena é vista, a posição do olho pos.
  • O ponto que olhamos na cena (target). It is also common to define the direction in which we look. Technically we need a line of sight. One straight in space is mathematically defined either by 2 points or by a point and a vector. The first part of the definition is the eye position and the 2nd is either the target or the line of sight vector los.
  • A direção para cima para cima.
  • O campo de visão ‘fov_y. This means the angle between the two straight lines, starting at the eye position and ending at the leftmost point and the rightmost point, which can be seen simultaneously.
  • O tamanho e a proporção da janela de visualização para a qual projetamos nossa imagem vp.
  • O plano próximo perto e o plano distante distante. O plano próximo é a distância da posição do olho ao plano from where the objects become visible to us. The far plane is the distance from the eye position to the plane to which the objects of the scene are visible to us. An explanation of what the near plane and far plane are needed will follow later..

Uma definição desses dados em C++ e em Python pode ser assim:

C++

using TVec3 = std::array<float,3>;
struct Camera
{
    TVec3 pos    {0.0, -8.0, 0.0};
    TVec3 target {0.0, 0.0, 0.0};
    TVec3 up     {0.0, 0.0, 1.0};
    float fov_y  {90.0};
    TSize vp     {800, 600};
    float near   {0.5};
    float far    {100.0};
};

Pitão

class Camera:
    def __init__(self):
        self.pos    = (0, -8, 0)
        self.target = (0, 0, 0)
        self.up     = (0, 0, 1)
        self.fov_y  = 90
        self.vp     = (800, 600)
        self.near   = 0.5
        self.far    = 100.0

Para levar todas essas informações em consideração ao desenhar uma cena, geralmente são usadas uma matriz de projeção e uma matriz de visualização. Para organizar as partes individuais de uma cena na cena, são usadas matrizes de modelo. No entanto, estes são mencionados aqui apenas para fins de completude e não serão tratados aqui.

  • Matriz de projeção: A matriz de projeção descreve o mapeamento de pontos 3D no mundo como eles são vistos de uma câmera pinhole, para pontos 2D da janela de visualização.

  • Ver matriz: A matriz de visualização define a posição do olho e a direção de visualização na cena.

  • Matriz modelo: A matriz do modelo define a localização e o tamanho relativo de um objeto na cena.

Depois de preenchermos as estruturas de dados acima com os dados correspondentes, temos que traduzi-los nas matrizes apropriadas. No modo de compatibilidade OGL, isso pode ser feito com as funções gluLookAt e gluPerspective que definem os uniformes internos gl_ModelViewMatrix, gl_NormalMatrix e gl_ModelViewProjectionMatrix. No OGL 3.1 e no GLSL #versão 150, os uniformes integrados foram removidos, porque toda a pilha de matriz de função fixa ficou obsoleta. Se quisermos usar sombreador de alto nível OGL com GLSL versão 330 ou superior, temos que definir e definir os uniformes da matriz nosso eu (além do uso da palavra-chave GLSL compatibilidade).

Configure a perspectiva - Matriz de projeção #

Um ponto na viewport é visível quando está na AABB nativa (caixa delimitadora alinhada ao eixo) definida pelos pontos (-1.0, -1.0, -1.0) e (1.0, 1.0, 1.0). Isso é chamado de Coordenadas de Dispositivo Normalizadas (NDC). Um ponto com as coordenadas (-1.0, -1.0, z) será pintado no canto inferior esquerdo da janela de visualização e um ponto com as coordenadas (1.0, 1.0, z) será pintada no canto superior direito da viewport. A coordenada Z é mapeada do intervalo (-1,0, 1,0) para o intervalo (0,0, 1,0) e gravada no buffer Z.

Tudo o que podemos ver da cena está dentro de uma pirâmide de 4 lados. O topo da pirâmide é a posição dos olhos. Os 4 lados do a pirâmide é definida pelo campo de visão (fov_y) e a proporção (vp[0]/vp[1]). A matriz de projeção deve mapear os pontos de dentro da pirâmide para o NDC definido pelos pontos (-1.0, -1.0, -1.0) e (1.0, 1.0, 1.0). Neste ponto nossa pirâmide é infinita, não tem fim em profundidade e não podemos mapear um espaço infinito para um finito. Para isso precisamos agora do plano próximo e do plano distante, eles transformam a pirâmide em um tronco cortando o topo e limitando a pirâmide na profundidade. O plano próximo e o plano distante devem ser escolhidos de tal forma que incluam tudo o que deve ser visível da cena.

O mapeamento dos pontos dentro de um tronco para o NDC é pura matemática e geralmente pode ser resolvido. O desenvolvimento das fórmulas foi frequentemente discutido e publicado repetidamente em toda a web. Como você não pode inserir uma fórmula LaTeX em uma documentação do Stack Overflow, isso é dispensado aqui e apenas o código-fonte C++ e Python completo é adicionado. Observe que as coordenadas do olho são definidas no sistema de coordenadas da mão direita, mas o NDC usa o sistema de coordenadas da mão esquerda sistema de coordenadas. A matriz de projeção é calculada a partir do campo de visão fov_y, da proporção vp[0]/vp[1], o plano próximo pertoe o plano distante distante.

C++

using TVec4  = std::array< float, 4 >;
using TMat44 = std::array< TVec4, 4 >;
TMat44 Camera::Perspective( void )
{
    float fn  = far + near;
    float f_n = far - near;
    float r   = (float)vp[0] / vp[1];
    float t   = 1.0f / tan( ToRad( fov_y ) / 2.0f );
    return TMat44{ 
        TVec4{ t / r, 0.0f,  0.0f,                     0.0f },
        TVec4{ 0.0f,  t,     0.0f,                     0.0f },
        TVec4{ 0.0f,  0.0f, -fn / f_n,                -1.0  },
        TVec4{ 0.0f,  0.0f, -2.0f * far * near / f_n,  0.0f } };
}

Pitão

def Perspective(self):
    fn, = self.far + self.near
    f_n = self.far - self.near
    r = self.vp[0] / self.vp[1]
    t = 1 / math.tan( math.radians( self.fov_y ) / 2 )
    return numpy.matrix( [ 
        [ t/r, 0,  0,                               0 ],
        [ 0,   t,  0,                               0 ],
        [ 0,   0, -fn/f_n,                         -1 ],
        [ 0,   0, -2 * self.far * self.near / f_n,  0 ] ] )

Configure o visual da cena - Ver matriz #

No sistema de coordenadas na viewport, o eixo Y aponta para cima (0, 1, 0) e o eixo X aponta para a direita (1, 0, 0). Isso resulta em um eixo Z que aponta para fora da janela de visualização ( (0, 0, -1) = cross( eixo X, eixo Y )).

Na cena, o eixo X aponta para o leste, o eixo Y para o norte e o eixo Z para o topo.

O eixo X da viewport (1, 0, 0) corresponde ao eixo Y da cena (1, 0, 0), o eixo Y da viewport (0, 1, 0 ) corresponde ao eixo Z da cena (0, 0, 1) e o eixo Z da viewport (0, 0, 1 ) corresponde ao eixo Y negado da cena (0, -1, 0).

Cada ponto e cada vetor do sistema de referência da cena deve, portanto, ser convertido primeiro em coordenadas da janela de visualização. Isso pode ser feito por algumas operações de troca e inversão nos vetores escalares.

 x  y  z
--------
 1  0  0  | x' =  x
 0  0  1  | y' =  z
 0 -1  0  | z' = -y

Para configurar uma matriz de visualização, a posição pos, o destino target e o vetor up up devem ser mapeados no sistema de coordenadas da viewport, conforme descrito acima. Isso fornece os 2 pontos p e t e o vetor u, como no trecho de código a seguir. O eixo Z da matriz de visão é a linha de visão inversa, que é calculada por p - t. O eixo Y é o vetor ascendente u. O eixo X é calculado pelo produto vetorial do eixo Y e do eixo Z. Para ortonormalizar a matriz de visualização, o produto cruzado é usado uma segunda vez, para calcular o eixo Y a partir do eixo Z e do eixo X (é claro que a ortogonalização de Gram-Schmidt funcionaria tão bem). No final, todos os 3 eixos devem ser normalizados e a posição do olho pos deve ser definida como a origem da matriz de visualização.

O código abaixo define uma matriz que encapsula exatamente os passos necessários para calcular uma olhada na cena:

  1. Convertendo coordenadas do modelo em coordenadas da viewport.
  2. Gire na direção da visão.
  3. Movimento para a posição dos olhos

C++

template< typename T_VEC >
TVec3 Cross( T_VEC a, T_VEC b )
{
    return { a[1] * b[2] - a[2] * b[1], a[2] * b[0] - a[0] * b[2], a[0] * b[1] - a[1] * b[0] };
}

template< typename T_A, typename T_B >
float Dot( T_A a, T_B b )
{ 
    return a[0]*b[0] + a[1]*b[1] + a[2]*b[2]; 
}

template< typename T_VEC >
void Normalize( T_VEC & v )
{ 
    float len = sqrt( v[0] * v[0] + v[1] * v[1] + v[2] * v[2] ); v[0] /= len; v[1] /= len; v[2] /= len; 
}

TMat44 Camera::LookAt( void )
{ 
    TVec3 mz = { pos[0] - target[0], pos[1] - target[1], pos[2] - target[2] };
    Normalize( mz );
    TVec3 my = { up[0], up[1], up[2] };
    TVec3 mx = Cross( my, mz );
    Normalize( mx );
    my = Cross( mz, mx );

    TMat44 v{
        TVec4{ mx[0], my[0], mz[0], 0.0f },
        TVec4{ mx[1], my[1], mz[1], 0.0f },
        TVec4{ mx[2], my[2], mz[2], 0.0f },
        TVec4{ Dot(mx, pos), Dot(my, pos), Dot(TVec3{-mz[0], -mz[1], -mz[2]}, pos), 1.0f }
    };

    return v;
}

Pitão

def LookAt(self):
    mz = Normalize( (self.pos[0]-self.target[0], self.pos[1]-self.target[1], self.pos[2]-self.target[2]) ) # inverse line of sight
    mx = Normalize( Cross( self.up, mz ) )
    my = Normalize( Cross( mz, mx ) )
    tx = Dot( mx, self.pos )
    ty = Dot( my, self.pos )
    tz = Dot( (-mz[0], -mz[1], -mz[2]), self.pos )   
    return = numpy.matrix( [ 
        [mx[0], my[0], mz[0], 0],
        [mx[1], my[1], mz[1], 0],
        [mx[2], my[2], mz[2], 0],
        [tx, ty, tz, 1] ] )

As matrizes são finalmente escritas em uniformes e usadas no vertex shader para transformar as posições do modelo.

Sombreador de vértice

No sombreador de vértice, uma transformação após a outra é executada.

  1. A matriz modelo traz o objeto (malha) ao seu lugar na cena. (Isso está listado apenas para fins de completude e não foi documentado aqui, pois não tem nada a ver com a visão da cena)

  2. A matriz de visualização define a direção a partir da qual a cena é visualizada. A transformação com a matriz de visualização gira os objetos da cena para que sejam visualizados do ângulo desejado. direção de visão com referência ao sistema de coordenadas da viewport.

  3. A matriz de projeção transforma os objetos de uma visão paralela em uma visão em perspectiva.

    #version 400

    layout (location = 0) in vec3 inPos; layout (location = 1) in vec3 inCol;

    out vec3 vertCol;

    uniform mat4 u_projectionMat44; uniform mat4 u_viewMat44; uniform mat4 u_modelMat44;

    void main() { vertCol = inCol; vec4 modelPos = u_modelMat44 * vec4( inPos, 1.0 ); vec4 viewPos = u_viewMat44 * modelPos; gl_Position = u_projectionMat44 * viewPos; }

import sys

def Cross( a, b ): return ( a[1] * b[2] - a[2] * b[1], a[2] * b[0] - a[0] * b[2], a[0] * b[1] - a[1] * b[0], 0.0 )
def Dot( a, b ): return a[0]*b[0] + a[1]*b[1] + a[2]*b[2]
def Normalize( v ): 
    len = math.sqrt( v[0] * v[0] + v[1] * v[1] + v[2] * v[2] )
    return (v[0] / len, v[1] / len, v[2] / len)

class Camera:
    def __init__(self):
        self.pos    = (0, -8, 0)
        self.target = (0, 0, 0)
        self.up     = (0, 0, 1)
        self.fov_y  = 90
        self.vp     = (800, 600)
        self.near   = 0.5
        self.far    = 100.0
    def Perspective(self):
        fn, f_n = self.far + self.near, self.far - self.near
        r, t = self.vp[0] / self.vp[1], 1 / math.tan( math.radians( self.fov_y ) / 2 )
        return numpy.matrix( [ [t/r,0,0,0], [0,t,0,0], [0,0,-fn/f_n,-1], [0,0,-2*self.far*self.near/f_n,0] ] )
    def LookAt(self):
         mz = Normalize( (self.pos[0]-self.target[0], self.pos[1]-self.target[1], self.pos[2]-self.target[2]) ) # inverse line of sight
    mx = Normalize( Cross( self.up, mz ) )

# shader program object
class ShaderProgram:
    def __init__( self, shaderList, uniformNames ):
        shaderObjs = []
        for sh_info in shaderList: shaderObjs.append( self.CompileShader(sh_info[0], sh_info[1] ) )
        self.LinkProgram( shaderObjs )
        self.__unifomLocation = {}
        for name in uniformNames:
            self.__unifomLocation[name] = glGetUniformLocation( self.__prog, name )
            print( "uniform %-30s at loaction %d" % (name, self.__unifomLocation[name]) )
    def Use(self):
        glUseProgram( self.__prog )
    def SetUniformMat44( self, name, mat ):
        glUniformMatrix4fv( self.__unifomLocation[name], 1, GL_FALSE, mat )
    # read shader program and compile shader
    def CompileShader(self, sourceFileName, shaderStage):
        with open( sourceFileName, 'r' ) as sourceFile:
            sourceCode = sourceFile.read()
        nameMap = { GL_VERTEX_SHADER: 'vertex', GL_FRAGMENT_SHADER: 'fragment' }    
        print( '\n%s shader code:' % nameMap.get( shaderStage, '' ) )
        print( sourceCode )
        shaderObj = glCreateShader( shaderStage )
        glShaderSource( shaderObj, sourceCode )
        glCompileShader( shaderObj )
        result = glGetShaderiv( shaderObj, GL_COMPILE_STATUS )
        if not (result):
            print( glGetShaderInfoLog( shaderObj ) )
            sys.exit()
        return shaderObj
    # linke shader objects to shader program
    def LinkProgram(self, shaderObjs):
        self.__prog = glCreateProgram()
        for shObj in shaderObjs: glAttachShader( self.__prog, shObj )
        glLinkProgram( self.__prog )
        result = glGetProgramiv( self.__prog, GL_LINK_STATUS )
        if not ( result ):
            print( 'link error:' )
            print( glGetProgramInfoLog( self.__prog ) )
            sys.exit()

# vertex array object
class VAObject:
    def __init__( self, dataArrays, tetIndices ):
        self.__obj = glGenVertexArrays( 1 )
        self.__noOfIndices = len( tetIndices )
        self.__indexArr = numpy.array( tetIndices, dtype='uint' )
        noOfBuffers = len( dataArrays )
        buffers = glGenBuffers( noOfBuffers )
        glBindVertexArray( self.__obj )
        for i_buffer in range( 0, noOfBuffers ):
            vertexSize, dataArr = dataArrays[i_buffer]
            glBindBuffer( GL_ARRAY_BUFFER, buffers[i_buffer] )
            glBufferData( GL_ARRAY_BUFFER, numpy.array( dataArr, dtype='float32' ), GL_STATIC_DRAW )
            glEnableVertexAttribArray( i_buffer )
            glVertexAttribPointer( i_buffer, vertexSize, GL_FLOAT, GL_FALSE, 0, None )
    def Draw(self):
        glBindVertexArray( self.__obj )
        glDrawElements( GL_TRIANGLES, self.__noOfIndices, GL_UNSIGNED_INT, self.__indexArr )

class Window:
    def __init__( self, cx, cy ):
        self.__vpsize = ( cx, cy )
        glutInitDisplayMode( GLUT_RGBA | GLUT_DOUBLE | GLUT_ALPHA | GLUT_DEPTH )
        glutInitWindowPosition( 0, 0 )
        glutInitWindowSize( self.__vpsize[0], self.__vpsize[1] )
        self.__id = glutCreateWindow( b'OGL window' ) 
        glutDisplayFunc( self.OnDraw ) 
        glutIdleFunc( self.OnDraw )
    def Run( self ):
        self.__startTime = time()
        glutMainLoop()

    # draw event
    def OnDraw(self):
        self.__vpsize = ( glutGet( GLUT_WINDOW_WIDTH ), glutGet( GLUT_WINDOW_HEIGHT ) )
        currentTime = time()
        # set up camera
        camera = Camera()
        camera.vp = self.__vpsize
        camera.pos = self.EllipticalPosition( 7, 4, self.CalcAng( currentTime, 10 ) )
        
        # set up attributes and shader program
        glEnable( GL_DEPTH_TEST )
        glClear( GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT )
        prog.Use()
        prog.SetUniformMat44( b"u_projectionMat44", camera.Perspective()  )
        prog.SetUniformMat44( b"u_viewMat44", camera.LookAt() )
        
        # draw object
        modelMat = numpy.matrix(numpy.identity(4), copy=False, dtype='float32')
        prog.SetUniformMat44( b"u_modelMat44", modelMat )
        tetVAO.Draw()
    
        glutSwapBuffers()

    def Fract( self, val ): return val - math.trunc(val)
    def CalcAng( self, currentTime, intervall ): return self.Fract( (currentTime - self.__startTime) / intervall ) * 2.0 *     math.pi
    def CalcMove( self, currentTime, intervall, range ):
        pos = self.Fract( (currentTime - self.__startTime) / intervall ) * 2.0
        pos = pos if pos < 1.0 else (2.0-pos)
        return range[0] + (range[1] - range[0]) * pos
    def EllipticalPosition( self, a, b, angRag ):
        a_b = a * a - b * b
        ea = 0 if (a_b <= 0) else math.sqrt( a_b )
        eb = 0 if (a_b >= 0) else math.sqrt( -a_b )
        return ( a * math.sin( angRag ) - ea, b * math.cos( angRag ) - eb, 0 )
