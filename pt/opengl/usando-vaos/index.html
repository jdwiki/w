Usando VAOs

  • Sintaxe

Nesta página

  • Sintaxe
  • Parâmetros
  • Versão 4.3
  • Versão 3.0

O Vertex Array Object armazena como o opengl deve interpretar um conjunto de VBOs.

Em essência, ele permitirá que você evite chamar glVertexAttribPointer toda vez que desejar renderizar uma nova malha.

Se você não quiser lidar com VAOs, você pode simplesmente criar um e vinculá-lo durante a inicialização do programa e fingir que eles não existem.

Sintaxe #

  • void glEnableVertexAttribArray​(GLuint attribIndex);

  • void glDisableVertexAttribArray​(GLuint attribIndex);

  • void glVertexAttribPointer(GLuint attribIndex, tamanho GLint, tipo GLenum, GLboolean normalizado, GLsizei stride, const GLvoid * ponteiro);

  • void glVertexAttribFormat(GLuint attribIndex, tamanho GLint, tipo GLenum, GLboolean normalizado, GLuint relativeoffset);

  • void glVertexAttribBinding(GLuint attribIndex, GLuint bindingIndex);

  • void glBindVertexBuffer(GLuint bindingIndex, GLuint buffer, GLintptr offset, GLintptr stride);

Parâmetros #

parâmetro Detalhes
atribIndex a localização do atributo de vértice para o qual a matriz de vértices alimentará os dados
tamanho o número de componentes a serem extraídos do atributo
tipo O tipo C++ dos dados de atributo no buffer
normalizado se deve mapear tipos inteiros para o intervalo de ponto flutuante [0, 1] (para não assinado) ou [-1, 1] (para assinado)
apontador o deslocamento de byte no buffer para o primeiro byte dos dados do atributo (convertido para void* por motivos legados)
deslocamento o deslocamento de byte base desde o início do buffer até onde os dados da matriz começam
compensação relativa o deslocamento para um atributo específico, relativo ao deslocamento base para o buffer
passo o número de bytes dos dados de um vértice para o próximo
tampão o objeto buffer onde os arrays de vértices são armazenados
índice de ligação o índice ao qual o objeto de buffer de origem será vinculado

A configuração VAO de formato de atributo separado pode interoperar com glVertexAttribPointer (o último é definido em termos do primeiro). Mas você deve ter cuidado ao fazê-lo.

A versão de formato de atributo separado tem equivalentes de acesso de estado direto (DSA) em 4.5. Eles terão os mesmos parâmetros, mas em vez de usar o VAO vinculado, o VAO que está sendo modificado é passado explicitamente. Ao usar o buffer de índice DSA para glDrawElements pode ser definido com glVertexArrayElementBuffer(vao, ebo);

Versão 4.3 #

O OpenGL 4.3 (ou ARB_separate_attrib_format) adiciona uma forma alternativa de especificar os dados de vértice, que cria uma separação entre o formato dos dados vinculados a um atributo e a fonte do objeto de buffer que fornece os dados. Então, em vez de ter um VAO por malha, você pode ter um VAO por formato de vértice.

Cada atributo está associado a um formato de vértice e a um ponto de ligação. O formato do vértice consiste no tipo, contagem de componentes, se está normalizado e o deslocamento relativo desde o início dos dados até esse vértice específico. O ponto de ligação especifica de qual buffer um atributo obtém seus dados. Ao separar os dois, você pode vincular buffers sem especificar novamente nenhum formato de vértice. Você também pode alterar o buffer que fornece dados para vários atributos com uma única chamada de ligação.

//accessible constant declarations
constexpr int vertexBindingPoint = 0;
constexpr int texBindingPoint = 1;// free to choose, must be less than the GL_MAX_VERTEX_ATTRIB_BINDINGS limit

//during initialization
glBindVertexArray(vao);

glVertexAttribFormat(posAttrLoc, 3, GL_FLOAT, false, offsetof(Vertex, pos));
// set the details of a single attribute
glVertexAttribBinding(posAttrLoc, vertexBindingPoint);
// which buffer binding point it is attached to
glEnableVertexAttribArray(posAttrLoc);

glVertexAttribFormat(normalAttrLoc, 3, GL_FLOAT, false, offsetof(Vertex, normal));
glVertexAttribBinding(normalAttrLoc, vertexBindingPoint);
glEnableVertexAttribArray(normalAttrLoc);

glVertexAttribFormat(texAttrLoc, 2, GL_FLOAT, false, offsetof(Texture, tex));
glVertexAttribBinding(texAttrLoc, texBindingPoint);
glEnableVertexAttribArray(texAttrLoc);

Então, durante o sorteio, você mantém o vao vinculado e altera apenas as ligações do buffer.

void drawMesh(Mesh[] mesh){
    glBindVertexArray(vao);

    foreach(mesh in meshes){
        glBindVertexBuffer(vertexBindingPoint, mesh.vbo, mesh.vboOffset, sizeof(Vertex));
        glBindVertexBuffer(texBindingPoint, mesh.texVbo, mesh.texVboOffset, sizeof(Texture));
        // bind the buffers to the binding point

        glBindBuffer(GL_ELEMENT_ARRAY_BUFFER, mesh.ebo);

        glDrawElements(GL_TRIANGLES, mesh.vertexCount, GL_UNSIGNED_INT, mesh.indexOffset);
        //draw
    }
}

Versão 3.0 #

Cada atributo está associado a uma contagem de componentes, tipo, normalizado, deslocamento, passo e VBO. O VBO não é passado explicitamente como um parâmetro, mas é o buffer vinculado a GL_ARRAY_BUFFER no momento da chamada.

void prepareMeshForRender(Mesh mesh){
    glBindVertexArray(mesh.vao);
    glBindBuffer(GL_ARRAY_BUFFER, mesh.vbo);
    
    glVertexAttribPointer (posAttrLoc, 3, GL_FLOAT, false, sizeof(Vertex), mesh.vboOffset + offsetof(Vertex, pos));//will associate mesh.vbo with the posAttrLoc
    glEnableVertexAttribArray(posAttrLoc);

    glVertexAttribPointer (normalAttrLoc, 3, GL_FLOAT, false, sizeof(Vertex), mesh.vboOffset + offsetof(Vertex, normal));
    glEnableVertexAttribArray(normalAttrLoc);

    glBindBuffer(GL_ELEMENT_ARRAY_BUFFER, mesh.ebo); //this binding is also saved.
    glBindVertexArray(0);
}

void drawMesh(Mesh[] meshes){
    foreach(mesh in meshes){        
        glBindVertexArray(mesh.vao);
        glDrawElements(GL_TRIANGLES, mesh.vertexCount, GL_UNSIGNED_INT, mesh.indexOffset);
    }
}
