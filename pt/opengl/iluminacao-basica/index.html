Modelo de iluminação Phong #

NOTA: Este exemplo é WIP, será atualizado com diagramas, imagens, mais exemplos, etc.

O que é Phong?

Phong é um modelo de luz muito básico, mas de aparência real, para superfícies que tem três partes: iluminação ambiente, difusa e especular.

Iluminação ambiente:

A iluminação ambiente é a mais simples das três partes para entender e calcular. A iluminação ambiente é a luz que inunda a cena e ilumina o objeto uniformemente em todas as direções.

As duas variáveis ​​na iluminação ambiente são a força do ambiente e a cor do ambiente. No seu fragment shader, o seguinte funcionará para o ambiente:

in vec3 objColor;

out vec3 finalColor;

uniform vec3 lightColor;

void main() {
   float ambientStrength = 0.3f;
   vec3 ambient = lightColor * ambientStrength;
   finalColor = ambient * objColor;
}

Iluminação Difusa:

A iluminação difusa é um pouco mais complexa que a ambiente. A iluminação difusa é luz direcional, essencialmente significando que os rostos voltados para a fonte de luz serão mais bem iluminados e os rostos apontando para fora serão mais escuros devido à forma como a luz os atinge.

Nota: a iluminação difusa exigirá o uso de normais para cada face que não mostrarei como calcular aqui. Se você quiser aprender a fazer isso, confira a página de matemática 3D.

Para modelar a reflexão da luz em computação gráfica é utilizada uma função de distribuição de refletância bidirecional (BRDF). BRDF é uma função que fornece a relação entre a luz refletida ao longo de uma direção de saída e a luz incidente de uma direção de entrada.

Uma superfície difusa perfeita tem um BRDF que tem o mesmo valor para todas as direções de incidência e saída. Isso reduz substancialmente os cálculos e, portanto, é comumente usado para modelar superfícies difusas, pois é fisicamente plausível, mesmo que não haja materiais difusos puros no mundo real. Essa BRDF é chamada de reflexão lambertiana porque obedece à lei dos cossenos de Lambert.

A reflexão lambertiana é frequentemente usada como modelo para reflexão difusa. Essa técnica faz com que todos os polígonos fechados (como um triângulo dentro de uma malha 3D) reflitam a luz igualmente em todas as direções quando renderizados. O coeficiente de difusão é calculado a partir do ângulo entre o vetor normal e o vetor de luz.

f_Lambertian = max( 0.0, dot( N, L )

onde N é o vetor normal da superfície, e L é o vetor em direção à fonte de luz.

Como funciona #

Em geral O ponto produto de 2 vetores é igual ao coseno do ângulo entre os 2 vetores multiplicado pela magnitude (comprimento) de ambos os vetores.

dot( A, B ) == length( A ) * length( B ) * cos( angle_A_B )

Segue-se que o ponto produto de 2 vetores unitários é igual ao coseno do ângulo entre os 2 vetores, porque o comprimento de um vetor unitário é 1.

uA = normalize( A )
uB = normalize( B )
cos( angle_A_B ) == dot( uA, uB )

Se dermos uma olhada na função cos(x) entre os ângulos -90° e 90°, podemos ver que ela tem um máximo de 1 em um ângulo de 0° e desce até 0 nos ângulos de 90° e -90°.

Esse comportamento é exatamente o que queremos para o modelo de reflexão. Quando o vetor normal da superfície e a direção da fonte de luz estão na mesma direção (o ângulo entre é 0°) então queremos um máximo de reflexão. Em contraste, se os vetores são ortonnormalizados (o ângulo entre é de 90°), então queremos um mínimo de reflexão e queremos um funcionamento suave e contínuo entre as duas bordas de 0° e 90°.

Se a luz for calculada por vértice, a reflexão será calculada para cada canto da primitiva. Entre as primitivas as reflexões são interpoladas de acordo com suas coordenadas baricêntricas. Veja as reflexões resultantes em uma superfície esférica:

Ok, então para começar com nosso sombreador de fragmento, precisaremos de quatro entradas.

  • Normals de vértice (devem estar no buffer e especificados por ponteiros de atributo de vértice)

  • Posição do fragmento (deve ser enviado do vertex shader para o frag shader)

  • Posição da fonte de luz (uniforme)

  • Cor clara (uniforme)

    in vec3 normal; in vec3 fragPos;

    out vec3 finalColor;

    uniform vec3 lightColor; uniform vec3 lightPos; uniform vec3 objColor;

Dentro de main é onde precisamos fazer algumas contas. Todo o conceito de iluminação difusa é baseado no ângulo entre a direção normal e a direção da luz. Quanto maior o ângulo, menos luz há até 90° onde não há luz alguma.

Antes de começarmos a calcular a quantidade de luz, precisamos do vetor de direção da luz. Isso pode ser recuperado simplesmente subtraindo a posição da luz da posição do fragmento que retorna um vetor da posição da luz apontando para a posição do fragmento.

vec3 lightDir = lightPos-fragPos;

Além disso, vá em frente e normalize os vetores normal e lightDir para que eles tenham o mesmo comprimento para trabalhar.

normal  = normalize(normal);
lightDir = normalize(lightDir);

Agora que temos nossos vetores, podemos calcular a diferença entre eles. Para fazer isso, vamos usar a função de produto escalar. Basicamente, isso pega 2 vetores e retorna o cos() do ângulo formado. Isto é perfeito porque a 90 graus produzirá 0 e a 0 graus produzirá 1. Como resultado, quando a luz estiver apontando diretamente para o objeto, ele estará totalmente iluminado e vice-versa.

float diff = dot(normal, lightDir);

Há mais uma coisa que temos que fazer com o número calculado, precisamos ter certeza de que é sempre positivo. Se você pensar sobre isso, um número negativo não faz sentido no contexto porque isso significa que a luz está atrás do rosto. Poderíamos usar uma instrução if, ou podemos usar a função max() que retorna o máximo de duas entradas.

diff = max(diff, 0.0);

Com isso feito, agora estamos prontos para calcular a cor de saída final para o fragmento.

vec3 diffuse = diff * lightColor;
finalColor = diffuse * objColor;
