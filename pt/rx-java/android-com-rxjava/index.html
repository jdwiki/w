O RxAndroid costumava ser uma biblioteca com muitos recursos. Ele foi dividido em muitas bibliotecas diferentes, passando da versão 0.25.0 para 1.x.

Uma lista de bibliotecas que implementam os recursos disponíveis antes do 1.0 é mantida aqui.

RxAndroid - Agendadores Android #

Esta é literalmente a única coisa que você precisa para começar a usar o RxJava no Android.

Inclua RxJava e RxAndroid em suas dependências do gradle:

 // use the last version
compile 'io.reactivex.rxjava2:rxjava:2.1.1'
compile 'io.reactivex.rxjava2:rxandroid:2.0.1'

A principal adição do RxAndroid ao RxJava é um Agendador para o thread principal do Android ou o thread da interface do usuário.

No seu código:

Observable.just("one", "two", "three", "four", "five")
        .subscribeOn(Schedulers.newThread())
        .observeOn(AndroidSchedulers.mainThread())
        .subscribe(
            data -> doStuffOnMainThread(),
            error -> handleErrorOnMainThread()
        )

Ou você pode criar um Agendador para um Looper personalizado:

Looper backgroundLooper = // ...
Observable.just("one", "two", "three", "four", "five")
        .subscribeOn(Schedulers.newThread())
        .observeOn(AndroidSchedulers.from(backgroundLooper))
        .subscribe(
            data -> doStuffOnMainThread(),
            error -> handleErrorOnMainThread()
        )

Para quase todo o resto, você pode consultar a documentação padrão do RxJava.

Componentes do RxLifecycle #

A biblioteca RxLifecycle facilita a vinculação de assinaturas observáveis ​​às atividades do Android e ao ciclo de vida do fragmento.

Lembre-se de que esquecer de cancelar a assinatura de um Observable pode causar vazamentos de memória e manter seu evento de atividade/fragmento vivo após ter sido destruído pelo sistema.

Adicione a biblioteca às dependências:

// use the last version available
compile 'com.trello:rxlifecycle:0.6.1'
compile 'com.trello:rxlifecycle-components:0.6.1'

Em seguida, estende as classes Rx*:

  • RxActivity / support.RxFragmentActivity / support.RxAppCompatActivity
  • RxFragment / support.RxFragment
  • RxDialogFragment / support.RxDialogFragment
  • support.RxAppCompatDialogActivity

Está tudo pronto, quando você se inscrever em um Observable, agora você pode:

someObservable
    .compose(bindToLifecycle())
    .subscribe();

Se você executar isso no método onCreate() da atividade, ele será automaticamente cancelado no método onDestroy().

O mesmo acontece para:

  • onStart() -> onStop()
  • onResume() -> onPause()
  • onAttach() -> onDetach() (apenas fragmento)
  • onViewCreated() -> onDestroyView() (apenas fragmento)

Como alternativa, você pode especificar o evento quando deseja que o cancelamento da assinatura ocorra:

De uma atividade:

someObservable
    .compose(bindUntilEvent(ActivityEvent.DESTROY))
    .subscribe();

De um fragmento:

someObservable
    .compose(bindUntilEvent(FragmentEvent.DESTROY_VIEW))
    .subscribe();

Você também pode obter o ciclo de vida observável usando o método lifecycle() para ouvir eventos de ciclo de vida diretamente.

O RxLifecycle também pode ser usado passando diretamente para ele o ciclo de vida observável:

.compose(RxLifecycleAndroid.bindActivity(lifecycle))

Se você precisa lidar com Single ou Completable, você pode fazer isso apenas adicionando forSingle() ou forCompletable respectivamente após o método bind:

someSingle
    .compose(bindToLifecycle().forSingle())
    .subscribe();

Também pode ser usado com a biblioteca Navi.

Rxpermissões #

Esta biblioteca permite o uso de RxJava com o novo modelo de permissão do Android M.

Adicione a biblioteca às dependências:

RxjavaGenericName

dependencies {
    compile 'com.tbruyelle.rxpermissions:rxpermissions:0.8.0@aar'
}

Rxjava2

dependencies {
    compile 'com.tbruyelle.rxpermissions2:rxpermissions:0.8.1@aar'
}

Uso

Exemplo (com Retrolambda para brevidade, mas não obrigatório):

// Must be done during an initialization phase like onCreate
RxPermissions.getInstance(this)
    .request(Manifest.permission.CAMERA)
    .subscribe(granted -> {
        if (granted) { // Always true pre-M
           // I can control the camera now
        } else {
           // Oups permission denied
        }
    });

Leia mais: https://github.com/tbruyelle/RxPermissions.
