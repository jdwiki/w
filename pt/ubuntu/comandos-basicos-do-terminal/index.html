Como extrair arquivos tar.gz/bz2/tbz :

Se sua extensão de arquivo for .tar.gz (ou .tgz) use este comando

 tar xvzf file.tar.gz

  • x: Isso diz ao tar para extrair os arquivos.

  • v: Esta opção listará todos os arquivos um por um no arquivo. O “v” significa “verboso”.

  • z: A opção z é muito importante e diz ao comando tar para descompacte o arquivo (gzip).

  • f: Esta opção diz ao tar que você dará a ele um nome de arquivo trabalhar com.

** Se sua extensão de arquivo for .tar.bz2 (ou .tbz)**

A principal diferença entre esses dois é que a opção z foi substituída pela opção j.

  • j: Isso irá descompactar um arquivo bzip2.

##!! (lido como bangbang) é um atalho para repetir o último comando digitado no console. É especialmente útil executar o comando anterior com algumas alterações

adduser tom

adduser: Somente o root pode adicionar um usuário ou grupo ao sistema.

Ai sô, e agora? Bem, você pode redigitar o comando com sudo na frente ou tentar

sudo !!

equivalente a sudo “comando anterior inserido”

Isso é especialmente útil se o comando que você acabou de digitar for especialmente longo.

Também pode ser usado para alterar uma parte dos comandos anteriores

cd path/to/wrong/directory

Com

!!:s/wrong/right/

Vai fazer

cd path/to/right/directory

Alterando a senha do usuário atual #

Para alterar a senha do usuário atual basta digitar:

sudo passwd

Ele solicitará que você insira sua senha atual:

[sudo] password for <user>:

E então você será solicitado a inserir uma nova senha:

Enter new UNIX password:

E, finalmente, você será solicitado a reinserir sua senha:

Retype new UNIX password:

Nota: Por padrão, as teclas que você pressiona em um prompt de senha de linha de comando não são exibidas. No entanto, eles ainda estão registrados.

Listar arquivos e pastas #

Para listar arquivos e pastas dentro do diretório atual, usamos o comando ls:

user@host:/$ ls
bin  boot  cdrom  dev  etc  home  initrd.img  lib  lib64  lost+found  
media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var  vmlinuz

ls imprime a estrutura de pastas em visualização simples, codificada por cores por tipo. As cores padrão do Ubuntu para ls são:

azul para diretórios, verde para arquivos executáveis, céu azul para arquivos vinculados, amarelo com fundo preto para dispositivos, rosa para arquivos de imagem e * vermelho* para arquivos compactados.

Outro atalho para ls -la é ll. No entanto, este não é um comando interno. Em vez disso, é um alias comum em sistemas ubuntu, na íntegra é ls -laF. O alias lhe dará a mesma saída que ls -la, mas com uma barra adicional (/) no final de cada pasta, para ajudá-lo com a identificação mais fácil da pasta.

O alias ll pode ser visualizado na íntegra digitando alias ll. Conforme ilustrado abaixo. Se o alias não estiver definido, o comando dará um erro.

vagrant@host ~ ->>
08:05 AM Mon Sep 12$ alias ll
 alias ll='ls -alF'

Adicionando novo usuário #

O comando adduser adiciona um usuário ao sistema. Para adicionar um novo tipo de usuário:

sudo adduser <user_name>

exemplo:

sudo adduser tom

Depois de digitar o comando acima, você será solicitado a inserir detalhes sobre o novo usuário, como nova senha, nome completo do usuário, etc.

Abaixo estão as informações que o usuário deverá preencher para adicionar um novo usuário:

Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
Changing the user information for tom
Enter the new value, or press ENTER for the default

    Home Phone []:

Reinicie o Ubuntu #

Você pode reiniciar o Ubuntu a partir da linha de comando. Abaixo está um exemplo de reiniciar o Ubuntu imediatamente.

 sudo reboot

Você precisa ter o privilégio sudo para usar este comando.

Outros comandos com os mesmos resultados são sudo shutdown -r now e sudo init 6.

Instale um novo software #

APT e APT-GET #

A maneira mais fácil e rápida é com o comando apt-get. Este comando pode ser considerado de nível inferior e “back-end” e suporta outras ferramentas baseadas em APT. Não há carregadores sofisticados, apenas informações básicas de progresso. Esta é a maneira mais rápida de instalar aplicativos.

Uso:

sudo apt-get install dilúvio openssh-server

Este comando instalará dois novos aplicativos: dilúvio e openssh-server. Você pode instalar quantos aplicativos quiser em apenas uma linha de comandos.

A maneira “fantástica” do mesmo processo é wits apt:

sudo apt install dilúvio openssh-server

O resultado é o mesmo, mas a interação com os usuários é diferente do comando anterior. apt é projetado para usuários finais (humanos) e sua saída pode ser alterada entre as versões.

Ambos os comandos irão lidar com dependências automaticamente.

DPKG #

dpkg significa Pacote Debian. Este é um instalador de pacotes básico e de baixo nível para o Debian e outros derivados do Debian. dpkg tem muitas opções, mas estamos interessados ​​em -i, que significa install.

sudo dpkg -i google-chrome-stable.deb

Você pode instalar pacotes .deb com dpkg.

Se você receber um erro ao instalar o pacote .deb, na maioria dos casos você não tem algumas dependências. Para se livrar desse erro e instalar o aplicativo corretamente, execute sudo apt-get -f install sem nenhum outro parâmetro. Isto irá procurar por dependências e instalá-las antes de .deb. A instalação continuará e o aplicativo será instalado.

Lendo um arquivo de texto #

Usando o Ubuntu, você tem diferentes maneiras de ler um arquivo de texto, todas semelhantes, mas úteis em diferentes contextos.

gato

Esta é a maneira mais simples de ler um arquivo de texto; ele simplesmente exibe o conteúdo do arquivo dentro do terminal. Tenha cuidado: se o arquivo for grande, pode levar algum tempo para concluir o processo de impressão! Se você precisar pará-lo, você sempre pode pressionar CTRL+C. Observe que, se você precisar navegar pelo documento, precisará rolar a saída do terminal.

cat file_name.txt

mais

Uma versão melhorada do cat. Se o seu arquivo for maior que a exibição do terminal, você pode simplesmente digitar

more file_name.txt

e você terá uma exibição de texto de rolagem para baixo, na qual você pode mover para baixo pressionando ‘ENTER’.

menos

Este é o comando more com algumas melhorias, e normalmente é uma escolha melhor do que cat para leitura de documentos médios a grandes. Abre arquivo mostrando-os desde o início, permitindo rolar para cima/baixo/direita/esquerda usando as setas.

less file_name.txt

Uma vez que o documento esteja aberto, você pode digitar alguns comandos para habilitar alguns recursos úteis, como:

  • q: fecha imediatamente o arquivo aberto.
  • /word: busca ‘palavra’ dentro do documento. Pressionando n você pode ir para a seguinte ocorrência de ‘palavra’.
  • ENTER: rola para baixo de uma única linha.
  • r: redesenha o conteúdo do arquivo, se estiver mudando durante a leitura.

Esta é a melhor escolha para a leitura de documentos médios a grandes.

rabo

Este software mostra apenas a última parte do arquivo. É útil se você precisar ler apenas algumas linhas no final de um documento muito grande.

tail file_name.txt

O comando acima mostrará as últimas 10 linhas (padrão) do arquivo. Para ler as últimas 3 linhas, precisamos escrever:

tail -3 file_name.txt

Há outro caso de uso em que esse comando é extremamente útil. Imagine ter um documento vazio, que é preenchido enquanto você o assiste; se você quiser ver novas linhas em tempo real enquanto elas são escritas no arquivo sem reabri-lo, basta abrir o arquivo com a opção -f. É muito útil se você estiver assistindo alguns logs, por exemplo.

tail -f file_name.txt

Esta é a melhor escolha para ler documentos em crescimento.

cabeça

Este comando faz a tarefa oposta de tail. Por exemplo, o comando a seguir mostrará as primeiras 15 linhas do arquivo file_name.txt.

head -15 file_name.txt

cauda

Esta é uma alternativa para tail -f filename . Ela segue as mudanças no arquivo conforme elas ocorrem e mostra a saída.

vim

Alguns de nós gostam do vi, outros gostam do vim. Isso não é apenas para ler arquivos, você também pode editá-los! Agora vamos ver apenas alguns recursos relacionados à leitura de documentos. Observe que o vim oferece realce de sintaxe.

vim file_name.txt

Uma vez que o arquivo é aberto, tenha cuidado! Não comece a digitar, ou você vai estragar tudo! Na verdade, mesmo que você possa ver o cursor, você deve pressionar i para começar a digitar e ESC depois de terminar de digitar. A propósito, agora vou mostrar alguns comandos úteis que dizem respeito à leitura (não à escrita):

  • : : você precisa digitar dois pontos antes de inserir cada um dos seguintes comandos!
  • q! : sai do arquivo sem pedir uma confirmação. É o mesmo que q se você não editou o texto.
  • /word : procura por ‘palavra’ dentro do documento.
  • 230 : vai para a linha ‘230’.

Dica: um atalho para inserir dois pontos e digitar wq! para escrever edições no arquivo e sair sem pedir uma confirmação, você pode segurar SHIFT e pressionar duas vezes z.

Esta é a melhor escolha para ler arquivos de código.

Pesquise o nome exato de um pacote para apt-get #

Se você sabe que precisa de um pacote <packet>, mas não sabe o nome exato <packet-exact-name>, em vez de pesquisar com o Google, tente o seguinte:

sudo apt-get update
apt-cache search <packet>

Isso retornará uma lista de nomes de pacotes com uma descrição. Depois de identificar o nome exato do pacote <packet-exact-name>, instale-o normalmente:

sudo apt-get install <packet-exact-name>

Limita a saída a linhas com a string desejada #

Se você está executando um comando que retorna centenas de linhas, mas seu interesse está apenas nas linhas que contém a palavra <word>, você deve definitivamente usar grep! Por exemplo, tente executar:

ifconfig -a

Verifique a saída, que pode ter apenas algumas linhas ou bastante longa, se você tiver um servidor com várias interfaces de rede. Para mostrar (por exemplo) apenas as linhas que contêm ‘HWaddr’, tente usar grep:

ifconfig -a | grep HWaddr

Agora a saída deve ser muito mais curta! De um modo geral, você pode usar grep da seguinte maneira:

<command-with-output> | grep <word>

Compacte arquivos e pastas usando o comando tar #

Use o comando tar (arquivo de fita) para compactar seus arquivos e pastas. É semelhante à criação de arquivos .ZIP no ambiente Windows.

Syntax: tar -zcvf <output tar file> <source file> 

Example: tar -zcvf outputfile.tar.gz source file

Aqui está o que esses interruptores realmente significam:

-c: cria um arquivo.
-z: compacta o arquivo com gzip.
-v: Exibe o progresso no terminal ao criar o arquivo, também conhecido como modo “verbose”. O v é sempre opcional nesses comandos, mas é útil.
-f: Permite especificar o nome do arquivo do arquivo.
