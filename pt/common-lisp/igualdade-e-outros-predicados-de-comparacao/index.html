A diferença entre EQ e EQL #

  1. EQ verifica se dois valores têm o mesmo endereço de memória: em outras palavras, verifica se os dois valores são realmente o mesmo, idêntico objeto. Portanto, pode ser considerado o teste de identidade, e deve ser aplicado somente a estruturas: conses, arrays, estruturas, objetos, normalmente para ver se você está lidando de fato com o mesmo objeto “alcançado” por caminhos diferentes, ou alias através de diferentes variáveis.

  2. EQL verifica se duas estruturas são o mesmo objeto (como EQ) ou se são os mesmos valores não estruturados (ou seja, os mesmos valores numéricos para números do mesmo tipo ou valores de caracteres) . Como inclui o operador EQ e pode ser usado também em valores não estruturados, é o operador mais importante e mais comumente usado, e quase todas as funções primitivas que exigem uma comparação de igualdade, como MEMBER, use por padrão este operador.

Então, é sempre verdade que (EQ X Y) implica (EQL X Y), enquanto o vice-versa não é válido.

Alguns exemplos podem esclarecer a diferença entre os dois operadores:

(eq 'a 'a)
T ;; => since two s-expressions (QUOTE A) are “internalized” as the same symbol by the reader.
(eq (list 'a) (list 'a))
NIL ;; => here two lists are generated as different objects in memory
(let* ((l1 (list 'a))
       (l2 l1))
  (eq l1 l2))
T ;; => here there is only one list which is accessed through two different variables
(eq 1 1)
?? ;; it depends on the implementation: it could be either T or NIL if integers are “boxed”
(eq #\a #\a)
?? ;; it depends on the implementation, like for numbers
(eq 2d0 2d0)
?? ;; => dependes on the implementation, but usually is NIL, since numbers in double 
   ;;    precision are treated as structures in many implementations
(let ((a1 2d0)
      (a2 2d0))
  (eq a1 a2))
?? ;; => also in this case the results depends on the implementation

Vamos tentar os mesmos exemplos com EQL:

(eql 'a 'a)
T ;; => equal because they are the same value, as for EQ
(eql (list 'a) (list 'a))
NIL ;; => different because they different objects in memory, as for EQ
(let* ((l1 (list 'a))
       (l2 l1))
  (eql l1 l2))
T ;; => as above
(eql 1 1)
T ;; they are the same number, even if integers are “boxed”
(eql #\a #\a)
T ;; they are the same character
(eql 2d0 2d0)
T ;; => they are the same number, even if numbers in double precision are treated as
   ;;   structures in many implementations
(let ((a1 2d0)
      (a2 2d0))
  (eql a1 a2))
T ;; => as before
(eql 2 2.0)
NIL;; => since the two values are of a different numeric type

A partir dos exemplos, podemos ver por que o operador EQL deve ser usado para verificar de forma portátil a “mesmidade” de todos os valores, estruturados e não estruturados, e por que, na verdade, muitos especialistas desaconselham o uso de EQ em geral.

Igualdade estrutural com EQUAL, EQUALP, TREE-EQUAL #

Esses três operadores implementam a equivalência estrutural, ou seja, eles verificam se objetos diferentes e complexos possuem estrutura equivalente com componente equivalente.

EQUAL se comporta como EQL para dados não estruturados, enquanto para estruturas construídas por conses (listas e árvores), e os dois tipos especiais de arrays, strings e vetores de bits, ele realiza equivalência estrutural, retornando true em dois estruturas que são isomórficas e cujos componentes elementares são correspondentemente iguais por EQUAL. Por exemplo:

(equal (list 1 (cons 2 3)) (list 1 (cons 2 (+ 2 1))))
T ;; => since the two arguments are both equal to (1 (2 . 3))
(equal "ABC" "ABC")
T ;; => equality on strings
(equal "Abc" "ABC")
NIL ;; => case sensitive equality on strings
(equal '(1 . "ABC") '(1 . "ABC"))
T ;; => equal since it uses EQL on 1 and 1, and EQUAL on "ABC" and "ABC"
(let* ((a (make-array 3 :initial-contents '(1 2 3)))
       (b (make-array 3 :initial-contents '(1 2 3)))
       (c a))
  (values (equal a b)
          (equal a c)))
NIL ;; => the structural equivalence is not used for general arrays
T   ;; => a and c are alias for the same object, so it is like EQL

EQUALP retorna true em todos os casos em que EQUAL é true, mas também usa equivalência estrutural para arrays de qualquer tipo e dimensão, para estruturas e para tabelas de hash (mas não para instâncias de classe!). Além disso, ele usa equivalência que não diferencia maiúsculas de minúsculas para strings.

(equalp "Abc" "ABC")
T ;; => case insensitive equality on strings
(equalp (make-array 3 :initial-contents '(1 2 3))
        (make-array 3 :initial-contents (list 1 2 (+ 2 1))))
T ;; => the structural equivalence is used also for any kind of arrays
(let ((hash1 (make-hash-table))
      (hash2 (make-hash-table)))
      (setf (gethash 'key hash1) 42)
      (setf (gethash 'key hash2) 42)
      (print (equalp hash1 hash2))
      (setf (gethash 'another-key hash1) 84)
      (equalp hash1 hash2))   
T   ;; => after the first two insertions, hash1 and hash2 have the same keys and values
NIL ;; => after the third insertion, hash1 and hash2 have different keys and values
(progn (defstruct s) (equalp (make-s) (make-s)))
T ;; => the two values are structurally equal
(progn (defclass c () ()) (equalp (make-instance 'c) (make-instance 'c)))
NIL ;; => two structurally equivalent class instances returns NIL, it's up to the user to
    ;;    define an equality method for classes

Finalmente, TREE-EQUAL pode ser aplicado a estruturas construídas através de cons e verifica se elas são isomórficas, como EQUAL, mas deixando para o usuário a escolha de qual função usar para comparar as folhas, ou seja, as não- contras (atom) encontrados, que podem ser de qualquer outro tipo de dados (por padrão, o teste usado no atom é EQL). Por exemplo:

(let ((l1 '(1 . ("A" . 2)))
      (l2 '(1 . ("A" . 2))))
  (tree-equal l1 l2 :test #'eql))
NIL ;; => since (eql "A" "A") gives NIL
(let ((l1 '(1 . ("A" . 2)))
      (l2 '(1 . ("A" . 2))))
  (tree-equal l1 l2 :test #'equal))
T ;; since (equal "A" "A") gives T

Operadores de comparação em valores numéricos #

Valores numéricos podem ser comparados com = e outros operadores de comparação numérica (/=, <, <=, >, >=) que ignoram a diferença na representação física dos diferentes tipos de números, e realizar a comparação dos valores matemáticos correspondentes. Por exemplo:

(= 42 42)
T ;; => both number have the sme numeric type and the same value
(= 1 1.0 1d0)
T ;; => all the tree values represent the number 1, while for instance (eql 1 1d0) => NIL
  ;;    since it returns true only if the operands have the same numeric type
(= 0.0 -0.0)
T ;; => again, the value is the same, while (eql 0.0 -0.0) => NIL
(= 3.0 #c(3.0 0.0))
T ;; => a complex number with 0 imaginary part is equal to a real number
(= 0.33333333 11184811/33554432)
T ;; => since a float number is passed to RATIONAL before comparing it to another number
  ;; => and (RATIONAL 0.33333333) => 11184811/33554432 in 32-bit IEEE floats architectures
(= 0.33333333 0.33333334)
T ;; => since the result of RATIONAL on both numbers is equal in 32-bit IEEE floats architectures
(= 0.33333333d0 0.33333334d0)
NIL ;; => since the RATIONAL of the two numbers in double precision is different

A partir desses exemplos, podemos concluir que = é o operador que normalmente deve ser usado para realizar comparação entre valores numéricos, a menos que queiramos ser rigorosos no fato de que dois valores numéricos são iguais somente se eles também tiverem o mesmo tipo numérico, nesse caso deve ser usado EQL.

Operadores de comparação em caracteres e strings #

Common Lisp tem 12 operadores específicos de tipo para comparar dois caracteres, 6 deles diferenciando maiúsculas de minúsculas e os outros insensíveis a maiúsculas e minúsculas. Seus nomes têm um padrão simples para facilitar a lembrança de seu significado:

Sensível a maiúsculas e minúsculas Insensível a maiúsculas e minúsculas
CARACTER= CHAR-EQUAL
CARACTER/= CHAR-NOT-EQUAL
CARACTER< CHAR-LESSP
CARACTER<= CHAR-NOT-GREATERP
CARA> CHAR-GREATERP
CARA>= CHAR-NOT-LESSP

Dois caracteres do mesmo caso estão na mesma ordem que os códigos correspondentes obtidos por CHAR-CODE, enquanto para comparações que não diferenciam maiúsculas de minúsculas a ordem relativa entre quaisquer dois caracteres retirados dos dois intervalos a..z, A. .Z depende da implementação. Exemplos:

(char= #\a #\a)
T ;; => the operands are the same character
(char= #\a #\A)
NIL ;; => case sensitive equality
(CHAR-EQUAL #\a #\A)
T ;; => case insensitive equality
(char> #\b #\a)
T ;; => since in all encodings (CHAR-CODE #\b) is always greater than (CHAR-CODE #\a)
(char-greaterp #\b \#A)
T ;; => since for case insensitive the ordering is such that A=a, B=b, and so on,
  ;;    and furthermore either 9<A or Z<0.
(char> #\b #\A)
?? ;; => the result is implementation dependent

Para strings os operadores específicos são STRING=, STRING-EQUAL, etc. com a palavra STRING em vez de CHAR. Duas strings são iguais se tiverem o mesmo número de caracteres e os caracteres correspondentes forem iguais de acordo com CHAR= ou CHAR-EQUAL se o teste diferenciar maiúsculas de minúsculas ou não.

A ordenação entre as strings é a ordem lexicográfica dos caracteres das duas strings. Quando uma comparação de ordenação é bem-sucedida, o resultado não é T, mas o índice do primeiro caractere em que as duas strings diferem (o que equivale a true, pois todo objeto não NIL é um “booleano generalizado” em Common Lisp) .

Uma coisa importante é que todos os operadores de comparação na string aceitam quatro parâmetros de palavras-chave: start1, end1, start2, end2, que podem ser usados ​​para restringir a comparação a apenas uma sequência contígua de caracteres dentro uma ou ambas as cordas. O índice inicial se omitido é 0, o índice final é omitido é igual ao comprimento da string, e a comparação é realizada na substring começando no caractere com índice :start e terminando com o caractere com índice :end - 1 incluído.

Por fim, observe que uma string, mesmo com um único caractere, não pode ser comparada a um caractere.

Exemplos:

(string= "foo" "foo")
T ;; => both strings have the same lenght and the characters are `CHAR=` in order
(string= "Foo" "foo")
NIL ;; => case sensitive comparison
(string-equal "Foo" "foo")
T ;; => case insensitive comparison
(string= "foobar" "barfoo" :end1 3 :start2 3)
T ;; => the comparison is perform on substrings
(string< "fooarr" "foobar")
3 ;; => the first string is lexicographically less than the second one and 
  ;;   the first character different in the two string has index 3
(string< "foo" "foobar")
3 ;; => the first string is a prefix of the second and the result is its length

Como um caso especial, os operadores de comparação de strings também podem ser aplicados a símbolos, e a comparação é feita no SYMBOL-NAME do símbolo. Por exemplo:

(string= 'a "A")
T ;; since (SYMBOL-NAME 'a) is "A"
(string-equal '|a| 'a)
T ;; since the the symbol names are "a" and "A" respectively

Como nota final, EQL em caracteres é equivalente a CHAR=; EQUAL em strings é equivalente a STRING=, enquanto EQUALP em strings é equivalente a STRING-EQUAL.

Visão geral #

Em Common Lisp existem muitos predicados diferentes para comparar valores. Eles podem ser classificados nas seguintes categorias:

  1. Operadores genéricos de igualdade: EQ, EQL, EQUAL, EQUALP. Eles podem ser usados ​​para valores de qualquer tipo e retornam sempre um valor booleano T ou NIL.
  2. Digite operadores de igualdade específicos: = e = para números, CHAR= CHAR= CHAR-EQUAL CHAR-NOT-EQUAL para caracteres, STRING= STRING= STRING-EQUAL STRING-NOT-EQUAL para strings, TREE-EQUAL para conses.
  3. Operadores de comparação para valores numéricos: <, <=, >, >=. Eles podem ser aplicados a qualquer tipo de número e comparar o valor matemático do número, independentemente do tipo real.
  4. Operadores de comparação para caracteres, como CHAR<, CHAR-LESSP, etc., que comparam caracteres tanto de forma case sensitive quanto case insensitive, de acordo com uma ordem dependente de implementação que preserva a ordenação alfabética natural.
  5. Operadores de comparação para strings, como STRING<, STRING-LESSP, etc., que comparam strings lexicograficamente, seja com distinção entre maiúsculas ou minúsculas, usando os operadores de comparação de caracteres.
