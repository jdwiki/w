Sintaxe #

  • (objeto de citação) -> objeto

Existem alguns objetos (por exemplo, símbolos de palavras-chave) que não precisam ser citados, pois avaliam a si mesmos.

Se os objetos citados forem modificados destrutivamente, as consequências são indefinidas! #

Evite operações destrutivas em objetos citados. Objetos entre aspas são objetos literais. Eles possivelmente estão embutidos no código de alguma forma. Como isso funciona e os efeitos das modificações não são especificados no padrão Common Lisp, mas podem ter consequências indesejadas, como modificar dados compartilhados, tentar modificar dados protegidos contra gravação ou criar efeitos colaterais indesejados.

(excluir 5 '(1 2 3 4 5))

Exemplo de citação simples #

Quote é um operador especial que impede a avaliação de seu argumento. Ele retorna seu argumento, não avaliado.

CL-USER> (quote a)
A

CL-USER> (let ((a 3))
           (quote a))
A

’ é um alias para o operador especial QUOTE #

A notação 'coisa' é igual a (coisa de aspas)`.

O leitor fará a expansão:

> (read-from-string "'a")
(QUOTE A)

A citação é usada para evitar avaliações posteriores. O objeto citado avalia a si mesmo.

> 'a
A

> (eval '+ 1 2)
3

Objetos de cotação e autoavaliação #

Observe que muitos tipos de dados não precisam ser citados, pois eles avaliam a si mesmos. QUOTE é especialmente útil para símbolos e listas, para evitar avaliação como formas Lisp.

Exemplo para outros tipos de dados que não precisam ser citados para evitar avaliação: strings, números, caracteres, objetos CLOS, …

Aqui um exemplo para strings. Os resultados da avaliação são strings, sejam eles citados na fonte ou não.

> (let ((some-string-1 "this is a string")
        (some-string-2 '"this is a string with a quote in the source")
        (some-string-3 (quote "this is another string with a quote in the source")))
    (list some-string-1 some-string-2 some-string-3))

("this is a string"
 "this is a string with a quote in the source"
 "this is another string with a quote in the source")

A citação dos objetos, portanto, é opcional.
