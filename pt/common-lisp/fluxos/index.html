Sintaxe #

  • (read-char & stream opcional eof-error-p eof-value recursive-p) => caractere
  • (caractere de escrita e fluxo opcional) => caractere
  • (read-line &opcional stream eof-error-p eof-value recursive-p) => line, missing-newline-p
  • (linha de escrita e fluxo opcional) => linha

Parâmetros #

Parâmetro Detalhe
fluxo O fluxo para ler ou gravar.
eof-erro-p Caso um erro seja sinalizado se o fim do arquivo for encontrado.
eof-valor Qual valor deve ser retornado se eof for encontrado e eof-error-p for false.
recursivo-p A operação de leitura é chamada recursivamente de READ. Normalmente, isso deve ser deixado como NIL.
personagem O caractere a ser escrito ou o caractere que foi lido.
linha A linha a ser escrita ou a linha que foi lida.

Copiando um arquivo #

Copiar byte por byte de um arquivo

A função a seguir copia um arquivo para outro executando uma cópia exata byte por byte, ignorando o tipo de conteúdo (que pode ser linhas de caracteres em alguma codificação ou dados binários):

(defun byte-copy (infile outfile)
  (with-open-file (instream infile :direction :input :element-type '(unsigned-byte 8)
                            :if-does-not-exist nil)
    (when instream
      (with-open-file (outstream outfile :direction :output :element-type '(unsigned-byte 8)
                                 :if-exists :supersede)
        (loop for byte = (read-byte instream nil)
           while byte
           do (write-byte byte outstream))))))

O tipo (unsigned-byte 8) é o tipo de bytes de 8 bits. As funções read-byte e write-byte funcionam em bytes, em vez de read-char e write-char que funcionam em caracteres. read-byte retorna um byte lido do stream, ou NIL no final do arquivo se o segundo parâmetro opcional for NIL (caso contrário, sinaliza um erro).

Cópia em massa

Uma cópia exata, mais eficiente que a anterior. pode ser feito lendo e gravando os arquivos com grandes blocos de dados a cada vez, em vez de bytes únicos:

(defun bulk-copy (infile outfile)
  (with-open-file (instream infile :direction :input :element-type '(unsigned-byte 8)
                            :if-does-not-exist nil)
    (when instream
      (with-open-file (outstream outfile :direction :output :element-type '(unsigned-byte 8)
                                 :if-exists :supersede)
        (let ((buffer (make-array 8192 :element-type '(unsigned-byte 8))))
          (loop for bytes-read = (read-sequence buffer instream)
             while (plusp bytes-read)
             do (write-sequence buffer outstream :end bytes-read)))))))

read-sequence e write-sequence são usados ​​aqui com um buffer que é um vetor de bytes (eles podem operar em sequências de bytes ou caracteres). read-sequence preenche o array com os bytes lidos a cada vez, e retorna o número de bytes lidos (que pode ser menor que o tamanho do array quando o final do arquivo é atingido). Observe que a matriz é modificada de forma destrutiva a cada iteração.

Cópia exata linha por linha de um arquivo

O exemplo final é uma cópia realizada lendo cada linha de caracteres do arquivo de entrada e gravando-a no arquivo de saída. Observe que, como queremos uma cópia exata, devemos verificar se a última linha do arquivo de entrada é terminada ou não por um caractere(s) de fim de linha. Por esta razão, usamos os dois valores retornados por read-line: uma nova string contendo os caracteres da próxima linha e um valor booleano que é true se a linha for a última do arquivo e não contiver o(s) caractere(s) final(is) de nova linha. Neste caso, write-string é usado em vez de write-line, já que o primeiro não adiciona uma nova linha no final da linha.

(defun line-copy (infile outfile)
  (with-open-file (instream infile :direction :input :if-does-not-exist nil)
    (when instream
      (with-open-file (outstream outfile :direction :output :if-exists :supersede)
        (let (line missing-newline-p)
          (loop
             (multiple-value-setq (line missing-newline-p)
               (read-line instream nil nil))
             (cond (missing-newline-p                          ; we are at the end of file
                    (when line (write-string line outstream))  ; note `write-string`
                    (return))                                  ; exit from simple loop
                   (t (write-line line outstream)))))))))

Observe que este programa é independente de plataforma, pois o(s) caractere(s) de nova linha (variando em diferentes sistemas operacionais) é gerenciado automaticamente pelas funções read-line e write-line.

Criando fluxos de entrada de strings #

A macro WITH-INPUT-FROM-STRING pode ser usada para fazer um stream de uma string.

(with-input-from-string (str "Foobar")
  (loop for i from 0
        for char = (read-char str nil nil)
        while char
        do (format t "~d: ~a~%" i char)))
; 0: F
; 1: o
; 2: o
; 3: b
; 4: a
; 5: r
;=> NIL

O mesmo pode ser feito manualmente usando MAKE-STRING-INPUT-STREAM.

(let ((str (make-string-input-stream "Foobar")))
  (loop for i from 0
        for char = (read-char str nil nil)
        while char
        do (format t "~d: ~a~%" i char)))

Escrevendo saída em uma string #

A macro WITH-OUTPUT-TO-STRING pode ser usada para criar uma saída de string stream e retornar a string resultante no final.

(with-output-to-string (str)
  (write-line "Foobar!" str)
  (write-string "Barfoo!" str))
;=> "Foobar!
;   Barfoo!"

O mesmo pode ser feito manualmente usando MAKE-STRING-OUTPUT-STREAM e GET-OUTPUT-STREAM-STRING .

(let ((str (make-string-output-stream)))
  (write-line "Foobar!" str)
  (write-string "Barfoo!" str)
  (get-output-stream-string str))

Fluxos cinza #

Os fluxos cinza são uma extensão não padrão que permite fluxos definidos pelo usuário. Ele fornece classes e métodos que o usuário pode estender. Você deve verificar seu manual de implementações para ver se ele fornece fluxos Gray.

Para um exemplo simples, um fluxo de entrada de caracteres que retorna caracteres aleatórios pode ser implementado assim:

(defclass random-character-input-stream (fundamental-character-input-stream)
  ((character-table
    :initarg :character-table
    :initform "abcdefghijklmnopqrstuvwxyz
" ; The newline is necessary.
    :accessor character-table))
  (:documentation "A stream of random characters."))

(defmethod stream-read-char ((stream random-character-input-stream))
  (let ((table (character-table stream)))
    (aref table (random (length table)))))

(let ((stream (make-instance 'random-character-input-stream)))
  (dotimes (i 5)
    (print (read-line stream))))
; "gyaexyfjsqdcpciaaftoytsygdeycrrzwivwcfb" 
; "gctnoxpajovjqjbkiqykdflbhfspmexjaaggonhydhayvknwpdydyiabithpt" 
; "nvfxwzczfalosaqw" 
; "sxeiejcovrtesbpmoppfvvjfvx" 
; "hjplqgstbodbalnmxhsvxdox" 
;=> NIL

Lendo arquivo #

Um arquivo pode ser aberto para leitura como um fluxo usando a macro WITH-OPEN-FILE.

(with-open-file (file #P"test.file")
  (loop for i from 0
        for line = (read-line file nil nil)
        while line
        do (format t "~d: ~a~%" i line)))
; 0: Foobar
; 1: Barfoo
; 2: Quuxbar
; 3: Barquux
; 4: Quuxfoo
; 5: Fooquux
;=> T

O mesmo pode ser feito manualmente usando OPEN e [CLOSE](http://www.lispworks.com/ documentação/HyperSpec/Body/f_close.htm).

(let ((file (open #P"test.file"))
      (aborted t))
  (unwind-protect
       (progn
         (loop for i from 0
               for line = (read-line file nil nil)
               while line
               do (format t "~d: ~a~%" i line))
         (setf aborted nil))
    (close file :abort aborted)))

Observe que READ-LINE cria uma nova string para cada linha. Isso pode ser lento. Algumas implementações fornecem uma variante, que pode ler uma linha em um buffer de string. Exemplo: READ-LINE-INTO para Allegro CL.

Gravando em um arquivo #

Um arquivo pode ser aberto para gravação como um fluxo usando a macro WITH-OPEN-FILE.

(with-open-file (file #P"test.file" :direction :output
                                    :if-exists :append
                                    :if-does-not-exist :create)
  (dolist (line '("Foobar" "Barfoo" "Quuxbar"
                  "Barquux" "Quuxfoo" "Fooquux"))
    (write-line line file)))

O mesmo pode ser feito manualmente com OPEN e [CLOSE](http://www.lispworks.com/ documentação/HyperSpec/Body/f_close.htm).

(let ((file (open #P"test.file" :direction :output
                                :if-exists :append
                                :if-does-not-exist :create)))
  (dolist (line '("Foobar" "Barfoo" "Quuxbar"
                  "Barquux" "Quuxfoo" "Fooquux"))
    (write-line line file))
  (close file))

Lendo e gravando arquivos inteiros de e para strings #

A função a seguir lê um arquivo inteiro em uma nova string e a retorna:

(defun read-file (infile)
  (with-open-file (instream infile :direction :input :if-does-not-exist nil)
    (when instream 
      (let ((string (make-string (file-length instream))))
        (read-sequence string instream)
        string))))

O resultado é NIL se o arquivo não existir.

A função a seguir grava uma string em um arquivo. Um parâmetro de palavra-chave é usado para especificar o que fazer se o arquivo já existir (por padrão, causa um erro, os valores admissíveis são os da macro with-open-file).

(defun write-file (string outfile &key (action-if-exists :error))
   (check-type action-if-exists (member nil :error :new-version :rename :rename-and-delete 
                                        :overwrite :append :supersede))
   (with-open-file (outstream outfile :direction :output :if-exists action-if-exists)
     (write-sequence string outstream)))

Neste caso, write-sequence pode ser substituído por write-string.
