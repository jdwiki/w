Lisp é frequentemente usado em contextos educacionais, onde os alunos aprendem a entender e implementar algoritmos recursivos.

O código de produção escrito em Common Lisp ou código portátil tem vários problemas com recursão: Eles não fazem uso de recursos específicos de implementação como otimização de chamada de cauda, muitas vezes tornando necessário evitar a recursão completamente. Nesses casos, implementações:

  • Geralmente tem um limite de profundidade de recursão devido aos limites nos tamanhos das pilhas. Assim, algoritmos recursivos só funcionarão para dados de tamanho limitado.
  • Nem sempre fornece otimização de chamadas finais, especialmente em combinação com operações com escopo dinâmico.
  • Fornece apenas otimização de chamadas de cauda em determinados níveis de otimização.
  • Geralmente não fornece otimização de chamada de cauda.
  • Geralmente não fornece otimização de chamada de cauda em determinadas plataformas. Por exemplo, implementações na JVM podem não fazer isso, pois a própria JVM não suporta otimização de chamada de cauda.

Substituir chamadas de cauda por saltos geralmente torna a depuração mais difícil; Adicionar saltos fará com que os quadros de pilha fiquem indisponíveis em um depurador. Como alternativas, o Common Lisp fornece:

  • Construções de iteração, como DO, DOTIMES, LOOP e outros
  • Funções de ordem superior, como MAP, REDUCE e outras
  • Várias estruturas de controle, incluindo “ir para” de baixo nível

Recursion template 1 single condition single tail recursion #

(defun fn (x)
  (cond (test-condition the-value)
        (t (fn reduced-argument-x))))

Imprime recursivamente os elementos de uma lista #

;;Recursively print the elements of a list
(defun print-list (elements)
    (cond
        ((null elements) '()) ;; Base case: There are no elements that have yet to be printed. Don't do anything and return a null list.
        (t
            ;; Recursive case
            ;; Print the next element.
            (write-line (write-to-string (car elements)))
            ;; Recurse on the rest of the list.
            (print-list (cdr elements))
        )
    )
)

Para testar isso, execute:

(setq test-list '(1 2 3 4))
(print-list test-list)

O resultado será:

1
2
3
4

Calcular o fatorial de um número inteiro #

Um algoritmo fácil de implementar como uma função recursiva é o fatorial.

;;Compute the factorial for any n >= 0. Precondition: n >= 0, n is an integer.
(defun factorial (n)
    (cond
        ((= n 0) 1) ;; Special case, 0! = 1
        ((= n 1) 1) ;; Base case, 1! = 1
        (t
            ;; Recursive case
            ;; Multiply n by the factorial of n - 1.
            (* n (factorial (- n 1)))
        )
    )
)

Calcula o enésimo número de Fibonacci #

;;Find the nth Fibonacci number for any n > 0.
;; Precondition: n > 0, n is an integer. Behavior undefined otherwise.
(defun fibonacci (n)
    (cond
        (                                     ;; Base case.
             ;; The first two Fibonacci numbers (indices 1 and 2) are 1 by definition.
            (<= n 2)                          ;; If n <= 2
            1                                 ;; then return 1.
        )
        (t                                    ;; else
            (+                                ;; return the sum of
                                              ;; the results of calling 
                (fibonacci (- n 1))           ;; fibonacci(n-1) and
                (fibonacci (- n 2))           ;; fibonacci(n-2).
                                              ;; This is the recursive case.
            )
        )
    )
)
