Construindo aplicativo de construção #

Binários Common Lisp autônomos podem ser construídos com buildapp. Antes de podermos usá-lo para gerar binários, precisamos instalá-lo e compilá-lo.

A maneira mais fácil que conheço é usar quicklisp e um Common Lisp (este exemplo usa [sbcl], mas não deve fazer diferença qual um que você tem).

$ sbcl

This is SBCL 1.3.5.nixos, an implementation of ANSI Common Lisp.
More information about SBCL is available at <http://www.sbcl.org/>.

SBCL is free software, provided as is, with absolutely no warranty.
It is mostly in the public domain; some portions are provided under
BSD-style licenses.  See the CREDITS and COPYING files in the
distribution for more information.

* (ql:quickload :buildapp)
To load "buildapp":
  Load 1 ASDF system:
    buildapp
; Loading "buildapp"

(:BUILDAPP)

* (buildapp:build-buildapp)
;; loading system "buildapp"
[undoing binding stack and other enclosing state... done]
[saving current Lisp image into /home/inaimathi/buildapp:
writing 4800 bytes from the read-only space at 0x20000000
writing 3216 bytes from the static space at 0x20100000
writing 47349760 bytes from the dynamic space at 0x1000000000
done]
NIL

* (quit)

$ ls -lh buildapp 
-rwxr-xr-x 1 inaimathi inaimathi 46M Aug 13 20:12 buildapp
$

Depois de ter esse binário construído, você pode usá-lo para construir binários de seus programas Common Lisp. Se você pretende fazer muito isso, você provavelmente também deve colocá-lo em algum lugar em seu PATH para que você possa executá-lo com buildapp de qualquer diretório.

Buildapp Hello Web World #

Um exemplo mais realista envolve um projeto que você está compilando com vários arquivos em disco (em vez de uma opção --eval passada para buildapp) e algumas dependências para puxar.

Como coisas arbitrárias podem acontecer durante a busca e carregamento de sistemas asdf (incluindo o carregamento de outros sistemas potencialmente não relacionados), não é suficiente apenas inspecionar os arquivos asd dos projetos dos quais você depende para descobrir o que você precisa carregar. A abordagem geral é usar quicklisp para carregar o sistema alvo, então chamar ql:write-asdf-manifest-file para escrever um manifesto completo de tudo que está carregado.

Aqui está um sistema de brinquedo construído com hunchetoot para ilustrar como isso pode acontecer na prática:

;;;; buildapp-hello-web-world.asd

(asdf:defsystem #:buildapp-hello-web-world
  :description "An example application to use when getting familiar with buildapp"
  :author "inaimathi <leo.zovic@gmail.com>"
  :license "Expat"
  :depends-on (#:hunchentoot)
  :serial t
  :components ((:file "package")
               (:file "buildapp-hello-web-world"))

;;;; package.lisp

(defpackage #:buildapp-hello-web-world
  (:use #:cl #:hunchentoot))

;;;; buildapp-hello-web-world.lisp

(in-package #:buildapp-hello-web-world)

(define-easy-handler (hello :uri "/") ()
  (setf (hunchentoot:content-type*) "text/plain")
  "Hello Web World!")

(defun main (argv)
  (declare (ignore argv))
  (start (make-instance 'easy-acceptor :port 4242))
  (format t "Press any key to exit...~%")
  (read-char))

;;;; build.lisp
(ql:quickload :buildapp-hello-web-world)
(ql:write-asdf-manifest-file "/tmp/build-hello-web-world.manifest")
(with-open-file (s "/tmp/build-hello-web-world.manifest" :direction :output :if-exists :append)
  (format s "~a~%" (merge-pathnames
            "buildapp-hello-web-world.asd"
            (asdf/system:system-source-directory
             :buildapp-hello-web-world))))

#### build.sh
sbcl --load "build.lisp" --quit

buildapp --manifest-file /tmp/build-hello-web-world.manifest --load-system hunchentoot --load-system buildapp-hello-web-world --output hello-web-world --entry buildapp-hello-web-world:main

Depois de ter esses arquivos salvos em um diretório chamado buildapp-hello-web-world, você pode fazer

$ cd buildapp-hello-web-world/

$ sh build.sh

Isso produz um binário que faz exatamente o que você acha que deveria, dado o acima.

$ ./hello-web-world 
Press any key to exit...

Você deve então ser capaz de abrir outro shell, fazer curl localhost:4242 e ver a resposta em texto simples de Hello Web World! ser impressa.

Construa o aplicativo Hello World #

O binário mais simples possível que você poderia construir

  1. Não tem dependências
  2. Não aceita argumentos de linha de comando
  3. Apenas escreve “Olá, mundo!” para stdout

Depois de construir o buildapp, você pode simplesmente…

$ buildapp --eval '(defun main (argv) (declare (ignore argv)) (write-line "Hello, world!"))' --entry main --output hello-world
[undoing binding stack and other enclosing state... done]
[saving current Lisp image into hello-world:
writing 4800 bytes from the read-only space at 0x20000000
writing 3216 bytes from the static space at 0x20100000
writing 43220992 bytes from the dynamic space at 0x1000000000
done]

$ ./hello-world 
Hello, world!

$
