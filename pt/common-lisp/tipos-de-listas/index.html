Listas de associações #

Listas simples são úteis para representar uma sequência de elementos, mas às vezes é mais útil representar um tipo de chave para mapeamento de valores. Common Lisp fornece várias maneiras de fazer isso, incluindo tabelas de hash genuínas (consulte 18.1 Conceitos de tabela de hash). Existem duas maneiras principais de representar mapeamentos de chave para valor em Common Lisp: listas de propriedades e listas de associação. Este exemplo descreve listas de associações.

Uma lista de associações, ou alist é uma lista “simples” cujos elementos são pares pontilhados em que o car de cada par é a chave e o cdr de cada par é o valor associado. Por exemplo,

(defparameter *ages* (list (cons 'john 34) (cons 'mary 23) (cons 'tim 72)))

pode ser considerado como uma lista de associações que mapeia símbolos que indicam um nome pessoal com um número inteiro que indica a idade. É possível implementar algumas funções de recuperação usando funções de lista simples, como membro. Por exemplo, para recuperar a idade de john, pode-se escrever

(cdr (first (member 'mary *age* :key 'car)))
;=> 23

A função member retorna o final da lista começando com uma célula cons cujo car é mary, ou seja, ((mary . 23) (tim . 72)), first retorna o primeiro elemento dessa lista, que é (mary . 23), e cdr retorna o lado direito desse par, que é 23. Embora essa seja uma maneira de acessar valores em uma lista de associações, o objetivo de uma convenção como listas de associações é abstrair da representação subjacente (uma lista) e fornecer funções de nível superior para trabalhar com a estrutura de dados.

Para listas de associações, a função de recuperação é assoc, que recebe uma chave, uma lista de associações e palavras-chave de teste opcionais (key, test, test-not) e retorna o par para a chave correspondente:

(assoc 'tim *ages*)
;=> (tim . 72)

Como o resultado sempre será uma célula contra se um item estiver presente, se assoc retornar nil, o item não estava na lista:

(assoc 'bob *ages*)
;=> nil

Para atualizar valores em uma lista de associações, setf pode ser usado junto com cdr. Por exemplo, quando o aniversário de john chegar e a idade dele aumentar, uma das seguintes opções poderá ser realizada:

(setf (cdr (assoc 'john *ages*) 35)

(incf (cdr (assoc 'john *ages*)))

incf funciona neste caso porque é baseado em setf.

As listas de associação também podem ser usadas como um tipo de mapa bidirecional, pois os mapeamentos de chave para valor são recuperados com base no valor usando a função assoc invertida, rassoc.

Neste exemplo, a lista de associações foi criada usando list e cons explicitamente, mas as listas de associações também podem ser criadas usando pairlis, que usa uma lista de chaves e dados e cria uma lista de associações com base neles:

(pairlis '(john mary tim) '(23 67 82))
;=> ((john . 23) (mary . 67) (tim . 82))

Um único par de chave e valor pode ser adicionado a uma lista de associação usando acons:

(acons 'john 23 '((mary . 67) (tim . 82)))
;=> ((john . 23) (mary . 67) (tim . 82))

A função assoc pesquisa a lista da esquerda para a direita, o que significa que é possível “mascarar” valores em uma lista de associações sem removê-los de uma lista ou atualizar qualquer estrutura da lista, apenas adicionando novos elementos para o início da lista. A função acons é fornecida para isso:

(defvar *ages* (pairlis '(john mary tim) '(34 23 72)))

(defvar *new-ages* (acons 'mary 29 *ages*))

*new-ages*
;=> ((mary . 29) (john . 34) (mary . 23) (tim . 72))

E agora, uma busca por mary retornará a primeira entrada:

(assoc 'mary *new-ages*)
;=> 29

Listas simples #

Listas simples são o tipo mais simples de lista em Common Lisp. Eles são uma sequência ordenada de elementos. Eles suportam operações básicas como obter o primeiro elemento de uma lista e o resto de uma lista em tempo constante, suportam acesso aleatório em tempo linear.

(list 1 2 3)
;=> (1 2 3)

(first (list 1 2 3))
;=> 1

(rest (list 1 2 3))
;=> (2 3)

Existem muitas funções que operam em listas “simples”, na medida em que se preocupam apenas com os elementos da lista. Isso inclui find, mapcar e muitos outros. (Muitas dessas funções também funcionarão em 17.1 Conceitos de Sequência para algumas dessas funções.

Listas de propriedades #

Listas simples são úteis para representar uma sequência de elementos, mas às vezes é mais útil representar um tipo de chave para mapeamento de valores. Common Lisp fornece várias maneiras de fazer isso, incluindo tabelas de hash genuínas (consulte 18.1 Conceitos de tabela de hash). Existem duas maneiras principais de representar mapeamentos de chave para valor em Common Lisp: listas de propriedades e listas de associação. Este exemplo descreve listas de propriedades.

Uma lista de propriedades, ou plist, é uma lista “simples” na qual valores alternados são interpretados como chaves e seus valores associados. Por exemplo:

(defparameter *ages* (list 'john 34 'mary 23 'tim 72))

pode ser considerado como uma lista de propriedades que mapeia símbolos indicando um nome pessoal com um número inteiro indicando idade. É possível implementar algumas funções de recuperação usando funções de lista simples, como membro. Por exemplo, para recuperar a idade de john, pode-se escrever

(second (member 'mary *age*))
;=> 23

A função member retorna o final da lista começando com mary, ou seja, (mary 23 tim 72), e second retorna o segundo elemento dessa lista, que é 23. Embora essa seja uma maneira de acessar valores em uma lista de propriedades, o objetivo de uma convenção como listas de propriedades é abstrair da representação subjacente (uma lista) e fornecer funções de nível superior para trabalhar com a estrutura de dados.

Para listas de propriedades, a função de recuperação é getf, que recebe a lista de propriedades, uma chave (mais comumente chamada de indicador) e um valor padrão opcional para retornar caso a lista de propriedades o faça não contém um valor para a chave.

(getf *ages* 'tim)
;=> 72

(getf *ages* 'bob -1)
;=> -1

Para atualizar valores em uma lista de propriedades, setf pode ser usado. Por exemplo, quando o aniversário de john chegar e a idade dele aumentar, uma das seguintes opções poderá ser realizada:

(setf (getf *ages* 'john) 35)

(incf (getf *ages* 'john))

incf funciona neste caso porque é baseado em setf.

Para pesquisar várias propriedades em uma lista de propriedades de uma só vez, use get-properties.

A função getf pesquisa a lista da esquerda para a direita, o que significa que é possível “mascarar” valores em uma lista de propriedades sem removê-los de uma lista ou atualizar qualquer estrutura da lista. Por exemplo, usando lista*:

(defvar *ages* '(john 34 mary 23 tim 72))

(defvar *new-ages* (list* 'mary 29 *ages*))

*new-ages*
;=> (mary 29 john 34 mary 23 tim 72)

E agora, uma busca por mary retornará a primeira entrada:

(getf *new-ages* 'mary)
;=> 29
