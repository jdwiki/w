O comando fit pode ajustar uma função definida pelo usuário a um conjunto de pontos de dados (x,y) ou (x,y,z), usando uma implementação dos mínimos quadrados não lineares (NLLS) Algoritmo de Marquardt-Levenberg.

Qualquer variável definida pelo usuário que ocorra no corpo da função pode servir como parâmetro de ajuste, mas o tipo de retorno da função deve ser real.

Sintaxe #

  • fit [xrange][yrange] função “datafile” usando modificador via parameter_file

Parâmetros #

Parâmetros Detalhe
Parâmetros de ajuste a, b, c e qualquer letra que não tenha sido usada anteriormente Use letras para representar parâmetros que serão usados ​​para ajustar uma função. Por exemplo: f(x) = a * exp(b * x) + c, g(x,y) = a*x**2 + b*y**2 + c*x*y
Parâmetros de arquivo start.par Ao invés de usar parâmetros não inicializados (o Marquardt-Levenberg irá inicializar automaticamente para você a=b=c=...=1) você pode colocá-los em um arquivo start.par e eles chame com na seção parameter_file. Ex.: fit f(x) 'data.dat' u 1:2 via 'start.par'. Um exemplo para o arquivo start.par é mostrado abaixo

Breve introdução #

fit é usado para encontrar um conjunto de parâmetros que ‘melhor’ ajusta seus dados à sua função definida pelo usuário. O ajuste é julgado com base na soma das diferenças quadradas ou ‘resíduos’ (SSR) entre os pontos de dados de entrada e os valores da função, avaliados nos mesmos locais. Essa quantidade é frequentemente chamada de ‘qui-quadrado’ (ou seja, a letra grega chi, elevada a 2). O algoritmo tenta minimizar SSR, ou mais precisamente, WSSR, pois os resíduos são ‘ponderados’ pelos erros de dados de entrada (ou 1,0) antes de serem elevados ao quadrado. (Ibidem)

O arquivo fit.log

Após cada etapa de iteração, uma informação detalhada sobre o estado do ajuste é fornecida na tela e no chamado arquivo de log fit.log. Este arquivo nunca será apagado, mas sempre anexado para que o histórico do ajuste não seja perdido.

Ajustando dados com erros #

Pode haver até 12 variáveis ​​independentes, sempre há 1 variável dependente e qualquer número de parâmetros pode ser ajustado. Opcionalmente, estimativas de erro podem ser inseridas para ponderar os pontos de dados. (T. Williams, C. Kelley - gnuplot 5.0, An Interactive Plotting Program)

Se você tem um conjunto de dados e quer ajustar se o comando for bem simples e natural:

fit f(x) "data_set.dat" using 1:2 via par1, par2, par3

onde em vez disso f(x) poderia ser também f(x, y). No caso de você também ter estimativas de erro de dados, basta adicionar o {y | xy | z}errors ({ | } representam as escolhas possíveis) na opção modifier (veja Sintaxe). Por exemplo

fit f(x) "data_set.dat" using 1:2:3 yerrors via par1, par2, par3

onde o {y | xy | A opção z}errors requer respectivamente 1 (y), 2 (xy), 1 (z) coluna que especifica o valor da estimativa de erro.

Ajuste exponencial com xyerrors de um arquivo

As estimativas de erro de dados são usadas para calcular o peso relativo de cada ponto de dados ao determinar a soma ponderada dos resíduos quadrados, WSSR ou qui-quadrado. Eles podem afetar as estimativas dos parâmetros, pois determinam o quanto o desvio de cada ponto de dados da função ajustada tem nos valores finais. Algumas das informações de saída de ajuste, incluindo as estimativas de erro de parâmetro, são mais significativas se tiverem sido fornecidas estimativas de erro de dados precisas.. (Ibidem)

Tomaremos um conjunto de dados de amostra measured.dat, composto por 4 colunas: as coordenadas do eixo x (Temperature (K)), as coordenadas do eixo y (Pressure (kPa) ), as estimativas de erro x (T_err (K)) e as estimativas de erro y (P_err (kPa)).

Agora, basta compor o protótipo da função que a partir da teoria deve aproximar nossos dados. Nesse caso:

Z = 0.001
f(x) = W * exp(x * Z)

onde inicializamos o parâmetro Z porque, caso contrário, avaliar a função exponencial exp(x * Z) resulta em valores enormes, o que leva a (ponto flutuante) Infinity e NaN no algoritmo de ajuste de Marquardt-Levenberg, geralmente você não faria precisa inicializar as variáveis ​​- dê uma olhada aqui, se você quiser saber mais sobre Marquardt-Levenberg.

É hora de ajustar os dados!

fit f(x) "measured.dat" u 1:2:3:4 xyerrors via W, Z

O código abaixo produz o gráfico a seguir.

set term pos col
set out 'PvsT.ps'

set grid
set key center
set xlabel 'T (K)'
set ylabel 'P (kPa)'


Z = 0.001
f(x) = W * exp(x * Z)
fit f(x) "measured.dat" u 1:2:3:4 xyerrors via W, Z

p [305:] 'measured.dat' u 1:2:3:4 ps 1.3 pt 2 t 'Data' w xyerrorbars,\
f(x) t 'Fit'

Plot com ajuste de measured.dat Usar o comando with xyerrorbars exibirá estimativas de erros no x e no y. set grid colocará uma grade tracejada nos tiques principais.

Caso as estimativas de erro não estejam disponíveis ou não sejam importantes, também é possível ajustar os dados sem o {y | xy | z}errors opção de ajuste:

fit f(x) "measured.dat" u 1:2 via W, Z

Neste caso, as xyerrorbars também tiveram que ser evitadas.

Exemplo de arquivo “start.par” #

Se você carregar seus parâmetros de ajuste de um arquivo, você deve declarar nele todos os parâmetros que você vai usar e, quando necessário, inicializá-los.

## Start parameters for the fit of data.dat

m  = -0.0005           
q  = -0.0005           
d  = 1.02              
Tc = 45.0
g_d = 1.0
b  = 0.01002

Ajuste: interpolação linear básica de um conjunto de dados #

O uso básico do ajuste é melhor explicado por um exemplo simples:

f(x) = a + bx + cx**2 ajuste [-234:320][0:200] f(x) ‘measured.dat’ usando 1:2 pule 4 via a,b,c plot ‘measured.dat’ u 1:2, f(x) Os intervalos podem ser especificados para filtrar os dados usados ​​no ajuste. Os pontos de dados fora do intervalo são ignorados. (T. Williams, C. Kelley - gnuplot 5.0, An Interactive Plotting Program)

A interpolação linear (ajustar com uma linha) é a maneira mais simples de ajustar um conjunto de dados. Suponha que você tenha um arquivo de dados em que o crescimento de sua quantidade y seja linear, você pode usar

[…] polinômios lineares para construir novos pontos de dados dentro do intervalo de um conjunto discreto de pontos de dados conhecidos. (da Wikipedia, Linear interpolation)

Exemplo com um polinômio de primeiro grau #

Vamos trabalhar com o seguinte conjunto de dados, chamado house_price.dat, que inclui os metros quadrados de uma casa em uma determinada cidade e seu preço em $ 1000.

## X-Axis: House price (in $1000) - Y-Axis: Square meters (m^2)

245    426.72

Vamos ajustar esses parâmetros com gnuplot O comando em si é bem simples, como você pode notar pela sintaxe, basta definir o seu protótipo de ajuste, e então usar o comando fit para obter o resultado:

## m, q will be our fitting parameters
f(x) = m * x + q       
fit f(x) 'data_set.dat' using 1:2 via m, q

Mas pode ser interessante também usar os parâmetros obtidos no próprio gráfico. O código abaixo vai caber no arquivo house_price.dat e então plotar os parâmetros m e q para obter a melhor aproximação da curva do conjunto de dados. Uma vez que você tenha os parâmetros, você pode calcular o valor y, neste caso o Preço da casa, a partir de qualquer valor x (metros quadrados da casa) apenas substituindo na fórmula

y = m * x + q

o “valor x” apropriado. Vamos comentar o código.

0. Definindo o termo

set term pos col
set out 'house_price_fit.ps'

1. Administração ordinária para embelezar o gráfico

set title 'Linear Regression Example Scatterplot'
set ylabel 'House price (k$ = $1000)'
set xlabel 'Square meters (m^2)'
set style line 1 ps 1.5 pt 7 lc 'red'
set style line 2 lw 1.5 lc 'blue'

set grid
set key bottom center box height 1.4

set xrange [0:450]
set yrange [0:]

2. O ajuste adequado

Para isso, só precisaremos digitar os comandos:

f(x) = m * x + q
fit f(x) 'house_price.dat' via m, q

3. Salvando valores m e q em uma string e plotando

Aqui usamos a função sprintf para preparar o rótulo (encaixotado no objeto retângulo) no qual vamos imprimir o resultado do ajuste. Por fim, traçamos o gráfico inteiro.

mq_value = sprintf("Parameters values\nm = %f k$/m^2\nq = %f k$", m, q)
set object 1 rect from 90,725 to 200, 650 fc rgb "white" 
set label 1 at 100,700 mq_value


p 'house_price.dat' ls 1 t 'House price', f(x) ls 2 t 'Linear regression'
set out

A saída ficará assim.
