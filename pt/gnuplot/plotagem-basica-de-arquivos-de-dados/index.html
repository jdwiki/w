Um dos principais recursos úteis do gnuplot é a possibilidade de plotar arquivos de dados. Plotar um arquivo de dados é realmente simples com gnuplot, na verdade, uma vez que você tenha aberto o software a partir do terminal, você só precisa digitar o comando plot 'file' para obter um gráfico automático.

Antes de tudo, antes de plotar, você deve ter certeza de estar no mesmo diretório onde o arquivo de dados está, caso contrário você receberá um aviso.

Sintaxe #

  • plotar datafile usando column_expression com style

Plotar um único arquivo de dados #

O comando padrão gnuplot plot (também somente p) plota o conjunto de dados com colunas, na forma do arquivo data_set.dat abaixo.

# Prototype of a gnuplot data set
# data_set.dat
# X -   X^2 -    2*X -    Random 
0       0        0        5
1       1        2        15
1.4142  2        2.8284   1
2       4        4        30
3       9        6        26.46
3.1415  9.8696   6.2832   39.11
4       16       8        20
4.5627  20.8182  9.1254   17 
5.0     25.0     10.0     25.50
6       36       12       0.908

Como você pode ver, você pode escrever em seu conjunto de dados em notação de ponto flutuante. Agora está tudo pronto para fazer a plotagem de dados: digitando apenas

plot "data_set.dat"

gnuplot produzirá um gráfico em seu destino output. As configurações padrão usarão as duas primeiras colunas do seu arquivo de dados, respectivamente x e y. Para especificar as colunas a serem plotadas, use o especificador using

plot "data_set.dat" using 2:4

que significa “traçar o arquivo usando a coluna 2 como X e a coluna 4 como Y”. Caso seu conjunto de dados seja um arquivo tridimensional, basta usar splot e adicionar a coluna z

splot "data_set.dat" using 1:2:3

Existem também estilos diferentes (veja a documentação do gnuplot ou Selecionando um estilo de plotagem para mais informações) para pontos de plotagem. Como dito antes, o estilo padrão é point

plot "data_set.dat" using 1:4 with point

que irá plotar o mesmo que se você não digitar with point. Um estilo útil para plotagem de dados é linespoint que é, obviamente, “linhas + pontos”. POR EXEMPLO.:

plot "data_set.dat" using 1:4 with linespoint
# the abbreviated form is completely equivalent:
# p "data_set.dat" u 1:4 w lp

[![linhaspoint][2]][2]

Formas de iteração de plotagem

Caso você tenha mais colunas e queira plotar todas no mesmo gráfico basta passar para a função plot qualquer argumento que preferir, separando-os com um ,:

p "data_set.dat" u 1:2 w lp,\
    "data_set.dat" u 1:3 w lp,\
    "data_set.dat" u 1:4 w lp

De qualquer forma, às vezes pode haver muitas colunas para escrever uma por uma. Nesse caso, o loop de iteração for resulta muito útil:

p for [col = 2:4] "data_set.dat" using 1:col w lp

que dá a saída

[![Trama iterado][3]][3]

Resumidamente a iteração for incrementa a variável no loop, neste caso col, com passos decididos (se não especificado = 1). Por exemplo, for [i = 0:6:2] incrementará i de 0 a 6 em 2 etapas: i = 0, 2, 4, 6. Todos os valores (iniciar, parar e incrementar) são convertidos em valores inteiros.

*Rede

A grade geralmente é útil ao plotar um conjunto de dados. Para adicionar um tipo de grade

set grid

Plotando vários arquivos de dados #

Primeiro método - Concatenação de strings #

O método mais simples para plotar vários arquivos de dados é inserir um loop for dentro do comando plot de gnuplot. Supondo que você tenha arquivos N nomeados sequencialmente, ou seja

file_1.dat
file_2.dat
file_3.dat
...
file_N.dat

Executando o comando

plot for[i = 1:N] "file_".i.".dat"

plotará todos os arquivos entre file_1.dat e file_N.dat no mesmo gráfico.

Exemplo com três arquivos de dados

Tabela de conjuntos de dados

Eixos X Y-Axe arquivo_1.dat Y-Axe arquivo_2.dat Y-Axe arquivo_3.dat
1 1 1 1
2 2 4 2
3 3 9 6
4 4 16 24
5 5 25 120

Comandos

set terminal postscript color noenhanced ##setting the term
set output "multiple_files.ps"

set key center ##legend placement

plot [1:5][1:120] \
    for [i = 1:3] "file_".i.".dat" \
    pointsize 1.3 linecolor i+4 \
    title "file\_".i.".dat" \
    with linespoint

O loop começa com for [i = 1:3] "file_".i.".dat" e executa o comando plot até atingir i = 3. O .i. é o número concatenado.

title "file\_".i.".dat" foi escrito com o \ para fazer com que o símbolo _ no nome dos arquivos apareça como um underscore em vez de um subscript, e o especificador noenhanced é fundamental para obter este resultado.

Segundo método - Usando a função sprintf #

Outro caminho possível a seguir é usar a função sprintf que funciona basicamente o mesmo [que a linguagem C sprintf](https://www.wikiod.com/pt/c/cordas#leitura/gravação de dados formatados em string). A sintaxe correta, da documentação do gnuplot 5.1 é

sprintf("format", x, y, ...)

Um breve exemplo esclarecerá todas as dúvidas.

file_name(n) = sprintf("file_%d.dat", n)
plot for[i = 1:N] file_name(i) title file_name(i)
