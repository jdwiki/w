Kafka oferece ferramentas de linha de comando para gerenciar tópicos, grupos de consumidores, consumir e publicar mensagens e assim por diante.

Importante: Os scripts do console Kafka são diferentes para plataformas baseadas em Unix e Windows. Nos exemplos, pode ser necessário adicionar a extensão de acordo com sua plataforma.

Linux: scripts localizados em bin/ com extensão .sh.

Windows: scripts localizados em bin\windows\ e com extensão .bat.

kafka-console-produtor #

Essa ferramenta permite que você produza mensagens a partir da linha de comando.

Enviar mensagens de string simples para um tópico:

kafka-console-producer --broker-list localhost:9092 --topic test
here is a message
here is another message
^D

(cada nova linha é uma nova mensagem, digite ctrl+D ou ctrl+C para parar)

Envie mensagens com chaves:

kafka-console-producer --broker-list localhost:9092 --topic test-topic \
        --property parse.key=true \
        --property key.separator=,
key 1, message 1
key 2, message 2
null, message 3
^D

Enviar mensagens de um arquivo:

kafka-console-producer --broker-list localhost:9092 --topic test_topic < file.log

kafka-simple-consumer-shell #

Esse consumidor é uma ferramenta de baixo nível que permite consumir mensagens de partições, deslocamentos e réplicas específicos.

Parâmetros úteis:

  • parition: a partição específica para consumir (padrão para todos)
  • offset: o deslocamento inicial. Use -2 para consumir mensagens desde o início, -1 para consumir a partir do final.
  • max-messages: número de mensagens a serem impressas
  • replica: a réplica, padrão para o broker-líder (-1)

Exemplo:

kafka-simple-consumer-shell  \
    --broker-list localhost:9092 \
    --partition 1 \
    --offset 4 \
    --max-messages 3 \
    --topic test-topic

exibe 3 mensagens da partição 1 começando no deslocamento 4 do tópico test-topic.

kafka-tópicos #

Esta ferramenta permite listar, criar, alterar e descrever tópicos.

Lista de tópicos:

kafka-topics  --zookeeper localhost:2181 --list

Crie um tópico:

kafka-topics  --create --zookeeper localhost:2181 --replication-factor 1 --partitions 1 --topic test

cria um tópico com uma partição e sem replicação.

Descreva um tópico:

kafka-topics  --zookeeper localhost:2181 --describe --topic test

Alterar um tópico:

# change configuration
kafka-topics  --zookeeper localhost:2181 --alter --topic test --config max.message.bytes=128000
# add a partition
kafka-topics  --zookeeper localhost:2181 --alter --topic test --partitions 2

(Cuidado: Kafka não suporta a redução do número de partições de um tópico) (veja esta lista de propriedades de configuração)

kafka-console-consumer #

Esta ferramenta permite que você consuma mensagens de um tópico.

para usar a antiga implementação do consumidor, substitua --bootstrap-server por --zookeeper.

Exibir mensagens simples:

kafka-console-consumer --bootstrap-server localhost:9092 --topic test

Consumir mensagens antigas:

Para ver as mensagens mais antigas, você pode usar a opção --from-beginning.

Exibir mensagens de valor-chave:

kafka-console-consumer  --bootstrap-server localhost:9092 --topic test-topic \
    --property print.key=true \
    --property key.separator=,

kafka-consumer-groups #

Essa ferramenta permite listar, descrever ou excluir grupos de consumidores. Consulte este artigo para obter mais informações sobre grupos de consumidores.

se você ainda usa a implementação antiga do consumidor, substitua --bootstrap-server por --zookeeper.

Listar grupos de consumidores:

kafka-consumer-groups  --bootstrap-server localhost:9092 --list
octopus

Descreva um grupo de consumidores:

kafka-consumer-groups  --bootstrap-server localhost:9092 --describe --group octopus
GROUP          TOPIC           PARTITION  CURRENT-OFFSET  LOG-END-OFFSET  LAG       OWNER
octopus        test-topic      0          15              15              0         octopus-1/127.0.0.1
octopus        test-topic      1          14              15              1         octopus-2_/127.0.0.1

Observações: na saída acima,

  • current-offset é o último deslocamento confirmado da instância do consumidor,
  • log-end-offset é o deslocamento mais alto da partição (portanto, a soma desta coluna fornece o número total de mensagens para o tópico)
  • lag é a diferença entre o deslocamento atual do consumidor e o deslocamento mais alto, portanto, quão atrás o consumidor está,
  • owner é o client.id do consumidor (se não for especificado, um padrão será exibido).

Excluir um grupo de consumidores:

a exclusão só está disponível quando os metadados do grupo são armazenados no zookeeper (antiga API do consumidor). Com a nova API do consumidor, o broker lida com tudo, incluindo a exclusão de metadados: o grupo é excluído automaticamente quando o último deslocamento confirmado para o grupo expira.

kafka-consumer-groups --bootstrap-server localhost:9092 --delete --group octopus
