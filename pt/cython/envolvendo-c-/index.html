Envolvendo uma DLL: C++ para Cython para Python #

Isso demonstra um exemplo não trivial de encapsulamento de uma dll C++ com Cython. Ele abrangerá as seguintes etapas principais:

  • Crie uma DLL de exemplo com C++ usando o Visual Studio.
  • Envolva a DLL com Cython para que possa ser chamada em Python.

Supõe-se que você tenha o Cython instalado e possa importá-lo com sucesso em Python.

Para a etapa DLL, também supõe-se que você esteja familiarizado com a criação de uma DLL no Visual Studio.

O exemplo completo inclui a criação dos seguintes arquivos:

  1. complexFunLib.h: arquivo de cabeçalho para a fonte DLL C++
  2. complexFunLib.cpp: arquivo CPP para a fonte DLL C++
  3. ccomplexFunLib.pxd: arquivo “cabeçalho” Cython
  4. complexFunLib.pyx: arquivo Cython “wrapper”
  5. setup.py: arquivo de configuração do Python para criar complexFunLib.pyd com Cython
  6. run.py: Exemplo de arquivo Python que importa a DLL compilada e encapsulada em Cython

** Fonte DLL C++: complexFunLib.h e complexFunLib.cpp** #

Ignore isso se você já tiver uma DLL e um arquivo fonte de cabeçalho. Primeiro, criamos a fonte C++ a partir da qual a DLL será compilada usando o Visual Studio. Nesse caso, queremos fazer cálculos de matriz rápidos com a função exponencial complexa. As duas funções a seguir executam o cálculo k*exp(ee) em arrays k e ee, onde os resultados são armazenados em res. Existem duas funções para acomodar precisão simples e dupla. Observe que essas funções de exemplo usam OpenMP, portanto, certifique-se de que o OpenMP esteja habilitado nas opções do Visual Studio para o projeto.

Arquivo H

// Avoids C++ name mangling with extern "C"
#define EXTERN_DLL_EXPORT extern "C" __declspec(dllexport)  
#include <complex>
#include <stdlib.h>

// Handles 64 bit complex numbers, i.e. two 32 bit (4 byte) floating point numbers
EXTERN_DLL_EXPORT void mp_mlt_exp_c4(std::complex<float>* k, 
                                     std::complex<float>* ee,
                                     int sz, 
                                     std::complex<float>* res, 
                                     int threads);

// Handles 128 bit complex numbers, i.e. two 64 bit (8 byte) floating point numbers
EXTERN_DLL_EXPORT void mp_mlt_exp_c8(std::complex<double>* k,                                       std::complex<double>* ee,
                                     int sz, 
                                     std::complex<double>* res, 
                                     int threads);

Arquivo CPP

#include "stdafx.h"
#include <stdio.h>
#include <omp.h>
#include "complexFunLib.h"

void mp_mlt_exp_c4(std::complex<float>* k,
                   std::complex<float>* ee,
                   int sz,
                   std::complex<float>* res,
                   int threads)
{
    // Use Open MP parallel directive for multiprocessing
    #pragma omp parallel num_threads(threads)
    {
        #pragma omp for
        for (int i = 0; i < sz; i++) res[i] = k[i] * exp(ee[i]);
    }
}

void mp_mlt_exp_c8(std::complex<double>* k,
                   std::complex<double>* ee,
                   int sz, std::complex<double>* res,
                   int threads)
{
    // Use Open MP parallel directive for multiprocessing
    #pragma omp parallel num_threads(threads)
    {
        #pragma omp for
        for (int i = 0; i < sz; i++) res[i] = k[i] * exp(ee[i]);
    }
}

Fonte Cython: ccomplexFunLib.pxd e complexFunLib.pyx #

Em seguida, criamos os arquivos de origem Cython necessários para encapsular a DLL C++. Nesta etapa, fazemos as seguintes suposições:

  • Você instalou o Cython
  • Você possui uma DLL de trabalho, por exemplo o descrito acima

O objetivo final é usar esses arquivos fonte Cython em conjunto com a DLL original para compilar um arquivo .pyd que pode ser importado como um módulo Python e expõe as funções escritas em C++.

Arquivo PXD

Este arquivo corresponde ao arquivo de cabeçalho C++. Na maioria dos casos, você pode copiar e colar o cabeçalho neste arquivo com pequenas alterações específicas do Cython. Neste caso, foram utilizados os tipos específicos de complexos Cython. Observe a adição de c no início de ccomplexFunLib.pxd. Isso não é necessário, mas descobrimos que essa convenção de nomenclatura ajuda a manter a organização.

cdef extern from "complexFunLib.h":
    void mp_mlt_exp_c4(float complex* k, float complex* ee, int sz,
                       float complex* res, int threads);
    void mp_mlt_exp_c8(double complex* k, double complex* ee, int sz,
                       double complex* res, int threads);

Arquivo PYX

Este arquivo corresponde ao arquivo fonte C++ cpp. Neste exemplo, estaremos passando ponteiros para objetos Numpy ndarray para as funções de importação de DLL. Também é possível usar o objeto memoryview do Cython embutido para arrays, mas seu desempenho pode não ser tão bom quanto os objetos ndarray (no entanto, a sintaxe é significativamente mais limpa).

cimport ccomplexFunLib  # Import the pxd "header"
# Note for Numpy imports, the C import most come AFTER the Python import
import numpy as np  # Import the Python Numpy
cimport numpy as np  # Import the C Numpy

# Import some functionality from Python and the C stdlib
from cpython.pycapsule cimport *

# Python wrapper functions.
# Note that types can be delcared in the signature

def mp_exp_c4(np.ndarray[np.complex64_t, ndim=1] k,
              np.ndarray[np.complex64_t, ndim=1] ee,
              int sz,
              np.ndarray[np.complex64_t, ndim=1] res,
              int threads):
    '''
    TODO: Python docstring
    '''
    # Call the imported DLL functions on the parameters.
    # Notice that we are passing a pointer to the first element in each array
    ccomplexFunLib.mp_mlt_exp_c4(&k[0], &ee[0], sz, &res[0], threads)
    
def mp_exp_c8(np.ndarray[np.complex128_t, ndim=1] k,
              np.ndarray[np.complex128_t, ndim=1] ee,
              int sz,
              np.ndarray[np.complex128_t, ndim=1] res,
              int threads):
    '''
    TODO: Python docstring
    '''
    ccomplexFunLib.mp_mlt_exp_c8(&k[0], &ee[0], sz, &res[0], threads)

Fonte Python: setup.py e run.py #

setup.py

Este arquivo é um arquivo Python que executa a compilação Cython. Seu propósito é gerar o arquivo compilado .pyd que pode então ser importado pelos módulos Python. Neste exemplo, mantivemos todos os arquivos necessários (ou seja, complexFunLib.h, complexFunLib.dll, ccomplexFunLib.pxd e complexFunLib.pyx) no mesmo diretório que setup.py.

Uma vez que este arquivo é criado, ele deve ser executado a partir da linha de comando com os parâmetros: build_ext --inplace

Uma vez que este arquivo é executado, ele deve produzir um arquivo .pyd sem gerar nenhum erro. Observe que em alguns casos, se houver um erro, o .pyd pode ser criado, mas é inválido. Certifique-se de que nenhum erro foi gerado na execução de setup.py antes de usar o .pyd gerado.

from distutils.core import setup
from distutils.extension import Extension
from Cython.Distutils import build_ext
import numpy as np

ext_modules = [
    Extension('complexFunLib',
              ['complexFunLib.pyx'],
              # Note here that the C++ language was specified
              # The default language is C
              language="c++",  
              libraries=['complexFunLib'],
              library_dirs=['.'])
    ]

setup(
    name = 'complexFunLib',
    cmdclass = {'build_ext': build_ext},
    ext_modules = ext_modules,
    include_dirs=[np.get_include()]  # This gets all the required Numpy core files
)

run.py

Agora o complexFunLib pode ser importado diretamente para um módulo Python e as funções DLL encapsuladas são chamadas.

import complexFunLib
import numpy as np

# Create arrays of non-trivial complex numbers to be exponentiated,
# i.e. res = k*exp(ee)
k = np.ones(int(2.5e5), dtype='complex64')*1.1234 + np.complex64(1.1234j)
ee = np.ones(int(2.5e5), dtype='complex64')*1.1234 + np.complex64(1.1234j) 
sz = k.size  # Get size integer
res = np.zeros(int(2.5e5), dtype='complex64')  # Create array for results

# Call function
complexFunLib.mp_exp_c4(k, ee, sz, res, 8)  

# Print results
print(res)
