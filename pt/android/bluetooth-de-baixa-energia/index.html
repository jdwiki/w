Esta documentação é um aprimoramento da documentação original e se concentrará na mais recente API Bluetooth LE introduzida no Android 5.0 (API 21). As funções centrais e periféricas serão abordadas, bem como como iniciar as operações de digitalização e publicidade.

Encontrando dispositivos BLE #

As seguintes permissões são necessárias para usar as APIs Bluetooth:

android.permission.BLUETOOTH
android.permission.BLUETOOTH_ADMIN

Se você estiver segmentando dispositivos com Android 6.0 (API nível 23) ou superior e quiser realizar operações de digitalização/publicidade, precisará de uma permissão de localização:

android.permission.ACCESS_FINE_LOCATION

ou

android.permission.ACCESS_COARSE_LOCATION

Observação.- Dispositivos com Android 6.0 (API de nível 23) ou superior também precisam ter os Serviços de localização ativados.

Um objeto BluetoothAdapter é necessário para iniciar as operações de digitalização/publicidade:

BluetoothManager bluetoothManager = (BluetoothManager) context.getSystemService(Context.BLUETOOTH_SERVICE);
bluetoothAdapter = bluetoothManager.getAdapter();

O método startScan (ScanCallback callback) da classe BluetoothLeScanner é a maneira mais básica de iniciar uma operação de varredura. Um objeto ScanCallback é necessário para receber os resultados:

bluetoothAdapter.getBluetoothLeScanner().startScan(new ScanCallback() {
     @Override
     public void onScanResult(int callbackType, ScanResult result) {
     super.onScanResult(callbackType, result);
     Log.i(TAG, "Remote device name: " + result.getDevice().getName());
       }
    });

Conectando-se a um servidor GATT #

Depois de descobrir um objeto BluetoothDevice desejado, você pode se conectar a ele usando seu método connectGatt() que recebe como parâmetros um objeto Context, um booleano que indica se deve se conectar automaticamente ao dispositivo BLE e uma referência BluetoothGattCallback onde os eventos de conexão e os resultados das operações do cliente serão entregues:

 if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
        device.connectGatt(context, false, bluetoothGattCallback, BluetoothDevice.TRANSPORT_AUTO);
    } else {
        device.connectGatt(context, false, bluetoothGattCallback);
    }

Substitua onConnectionStateChange em BluetoothGattCallback para receber eventos de conexão e desconexão:

    BluetoothGattCallback bluetoothGattCallback =
        new BluetoothGattCallback() {
    @Override
    public void onConnectionStateChange(BluetoothGatt gatt, int status,
            int newState) {
        if (newState == BluetoothProfile.STATE_CONNECTED) {
            Log.i(TAG, "Connected to GATT server.");

        } else if (newState == BluetoothProfile.STATE_DISCONNECTED) {
            
            Log.i(TAG, "Disconnected from GATT server.");
        }
    }
   };

Escrita e Leitura a partir de Características #

Uma vez conectado a um Gatt Server, você vai interagir com ele escrevendo e lendo as características do servidor. Para fazer isso, primeiro você deve descobrir quais serviços estão disponíveis neste servidor e quais características estão disponíveis em cada serviço:

 @Override
 public void onConnectionStateChange(BluetoothGatt gatt, int status,
        int newState) {
    if (newState == BluetoothProfile.STATE_CONNECTED) {
        Log.i(TAG, "Connected to GATT server.");
        gatt.discoverServices();

    }
. . .

 @Override
    public void onServicesDiscovered(BluetoothGatt gatt, int status) {
        if (status == BluetoothGatt.GATT_SUCCESS) {
             List<BluetoothGattService> services = gatt.getServices();
                for (BluetoothGattService service : services) {
                    List<BluetoothGattCharacteristic> characteristics = service.getCharacteristics();
                    for (BluetoothGattCharacteristic characteristic : characteristics) { 
                        ///Once you have a characteristic object, you can perform read/write
                        //operations with it         
                    }
                 }
              }
            }

Uma operação de gravação básica é assim:

characteristic.setValue(newValue);
characteristic.setWriteType(BluetoothGattCharacteristic.WRITE_TYPE_DEFAULT);
gatt.writeCharacteristic(characteristic);

Quando o processo de gravação terminar, o método onCharacteristicWrite do seu BluetoothGattCallback será chamado:

    @Override
    public void onCharacteristicWrite(BluetoothGatt gatt, BluetoothGattCharacteristic characteristic, int status) {
        super.onCharacteristicWrite(gatt, characteristic, status);
        Log.d(TAG, "Characteristic " + characteristic.getUuid() + " written);
    }

Uma operação de gravação básica é assim:

gatt.readCharacteristic(characteristic);

Quando o processo de escrita terminar, o método onCharacteristicRead do seu BluetoothGattCallback será chamado:

@Override
public void onCharacteristicRead(BluetoothGatt gatt, BluetoothGattCharacteristic characteristic, int status) {
  super.onCharacteristicRead(gatt, characteristic, status);
  byte[] value = characteristic.getValue();
  }

Assinando notificações do servidor Gatt #

Você pode solicitar ser notificado do Gatt Server quando o valor de uma característica for alterado:

gatt.setCharacteristicNotification(characteristic, true);
BluetoothGattDescriptor descriptor = characteristic.getDescriptor(
    UUID.fromString("00002902-0000-1000-8000-00805f9b34fb");
descriptor.setValue(BluetoothGattDescriptor.ENABLE_NOTIFICATION_VALUE);
mBluetoothGatt.writeDescriptor(descriptor);

Todas as notificações do servidor serão recebidas no método onCharacteristicChanged do seu BluetoothGattCallback:

    @Override
    public void onCharacteristicChanged(BluetoothGatt gatt, BluetoothGattCharacteristic characteristic) {
        super.onCharacteristicChanged(gatt, characteristic);
        byte[] newValue = characteristic.getValue();
}

Publicidade de um dispositivo BLE #

Você pode usar o Bluetooth LE Advertising para transmitir pacotes de dados para todos os dispositivos próximos sem precisar estabelecer uma conexão primeiro. Lembre-se de que há um limite estrito de 31 bytes de dados de anúncio. Anunciar seu dispositivo também é o primeiro passo para permitir que outros usuários se conectem a você.

Como nem todos os dispositivos são compatíveis com o Bluetooth LE Advertising, o primeiro passo é verificar se o seu dispositivo possui todos os requisitos necessários para suportá-lo. Depois, você pode inicializar um objeto BluetoothLeAdvertiser e com ele, você pode iniciar as operações de publicidade:

        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.LOLLIPOP && bluetoothAdapter.isMultipleAdvertisementSupported())

            BluetoothLeAdvertiser advertiser = bluetoothAdapter.getBluetoothLeAdvertiser();

            AdvertiseData.Builder dataBuilder = new AdvertiseData.Builder();
           //Define a service UUID according to your needs

            dataBuilder.setIncludeDeviceName(true);


             AdvertiseSettings.Builder settingsBuilder = new AdvertiseSettings.Builder();
             settingsBuilder.setAdvertiseMode(AdvertiseSettings.ADVERTISE_MODE_LOW_POWER);
             settingsBuilder.setTimeout(0);

             //Use the connectable flag if you intend on opening a Gatt Server
             //to allow remote connections to your device.
             settingsBuilder.setConnectable(true);

            AdvertiseCallback advertiseCallback=new AdvertiseCallback() {
            @Override
            public void onStartSuccess(AdvertiseSettings settingsInEffect) {
                super.onStartSuccess(settingsInEffect);

Usando um servidor Gatt #

Para que seu dispositivo funcione como um periférico, primeiro você precisa abrir um BluetoothGattServer e preenchê-lo com pelo menos um BluetoothGattService e um BluetoothGattCharacteristic:

BluetoothGattServer server=bluetoothManager.openGattServer(context, bluetoothGattServerCallback);

BluetoothGattService service = new BluetoothGattService(SERVICE_UUID, BluetoothGattService.SERVICE_TYPE_PRIMARY);

Este é um exemplo de BluetoothGattCharacteristic com permissões completas de gravação, leitura e notificação. De acordo com suas necessidades, convém ajustar as permissões que você concede a essa característica:

BluetoothGattCharacteristic characteristic = new BluetoothGattCharacteristic(CHARACTERISTIC_UUID,
                    BluetoothGattCharacteristic.PROPERTY_READ | BluetoothGattCharacteristic.PROPERTY_WRITE |
                            BluetoothGattCharacteristic.PROPERTY_NOTIFY,
                    BluetoothGattCharacteristic.PERMISSION_READ | BluetoothGattCharacteristic.PERMISSION_WRITE);

characteristic.addDescriptor(new BluetoothGattDescriptor(UUID.fromString("00002902-0000-1000-8000-00805f9b34fb"), BluetoothGattCharacteristic.PERMISSION_WRITE));

service.addCharacteristic(characteristic);

server.addService(service);

O BluetoothGattServerCallback é responsável por receber todos os eventos relacionados ao seu BluetoothGattServer:

BluetoothGattServerCallback bluetoothGattServerCallback= new BluetoothGattServerCallback() {
                @Override
                public void onConnectionStateChange(BluetoothDevice device, int status, int newState) {
                    super.onConnectionStateChange(device, status, newState);
                }

                @Override
                public void onCharacteristicReadRequest(BluetoothDevice device, int requestId, int offset, BluetoothGattCharacteristic characteristic) {
                    super.onCharacteristicReadRequest(device, requestId, offset, characteristic);
                }

                @Override
                public void onCharacteristicWriteRequest(BluetoothDevice device, int requestId, BluetoothGattCharacteristic characteristic, boolean preparedWrite, boolean responseNeeded, int offset, byte[] value) {
                    super.onCharacteristicWriteRequest(device, requestId, characteristic, preparedWrite, responseNeeded, offset, value);
                }

                @Override
                public void onDescriptorReadRequest(BluetoothDevice device, int requestId, int offset, BluetoothGattDescriptor descriptor) {
                    super.onDescriptorReadRequest(device, requestId, offset, descriptor);
                }

                @Override
                public void onDescriptorWriteRequest(BluetoothDevice device, int requestId, BluetoothGattDescriptor descriptor, boolean preparedWrite, boolean responseNeeded, int offset, byte[] value) {
                    super.onDescriptorWriteRequest(device, requestId, descriptor, preparedWrite, responseNeeded, offset, value);
                }

Sempre que você receber uma solicitação de escrita/leitura de uma característica ou descritor, você deve enviar uma resposta para que a solicitação seja concluída com sucesso:

@Override
 public void onCharacteristicReadRequest(BluetoothDevice device, int requestId, int offset, BluetoothGattCharacteristic characteristic) {
    super.onCharacteristicReadRequest(device, requestId, offset, characteristic);
    server.sendResponse(device, requestId, BluetoothGatt.GATT_SUCCESS, offset, YOUR_RESPONSE);
}
