Criando visualizações personalizadas #

Se você precisar de uma visualização totalmente personalizada, precisará subclassificar View (a superclasse de todas as visualizações do Android) e fornecer seu dimensionamento personalizado (onMeasure(...)) e desenho (onDraw(...) ) métodos:

  1. Crie seu esqueleto de visualização personalizada: isso é basicamente o mesmo para todas as visualizações personalizadas. Aqui criamos o esqueleto para uma visualização personalizada que pode desenhar um smiley, chamado SmileyView:

    public class SmileyView extends View {
    private Paint mCirclePaint;
    private Paint mEyeAndMouthPaint;

    private float mCenterX;
    private float mCenterY;
    private float mRadius;
    private RectF mArcBounds = new RectF();

    public SmileyView(Context context) {
        this(context, null, 0);
    }

    public SmileyView(Context context, AttributeSet attrs) {
        this(context, attrs, 0);
    }

    public SmileyView(Context context, AttributeSet attrs, int defStyleAttr) {
        super(context, attrs, defStyleAttr);
        initPaints();
    }

    private void initPaints() {/* ... */}

    @Override
    protected void onMeasure(int widthMeasureSpec, int heightMeasureSpec) {/* ... */}

    @Override
    protected void onDraw(Canvas canvas) {/* ... */}
}

  2. Inicialize suas pinturas: os objetos Paint são os pincéis de sua tela virtual que definem como seus objetos geométricos são renderizados (por exemplo, cor, estilo de preenchimento e traçado, etc.). Aqui criamos duas Paints, uma tinta amarela preenchida para o círculo e uma tinta preta para os olhos e a boca:

    private void initPaints() {
    mCirclePaint = new Paint(Paint.ANTI_ALIAS_FLAG);
    mCirclePaint.setStyle(Paint.Style.FILL);
    mCirclePaint.setColor(Color.YELLOW);
    mEyeAndMouthPaint = new Paint(Paint.ANTI_ALIAS_FLAG);
    mEyeAndMouthPaint.setStyle(Paint.Style.STROKE);
    mEyeAndMouthPaint.setStrokeWidth(16 * getResources().getDisplayMetrics().density);
    mEyeAndMouthPaint.setStrokeCap(Paint.Cap.ROUND);
    mEyeAndMouthPaint.setColor(Color.BLACK);
}

  3. Implemente seu próprio método onMeasure(...): isso é necessário para que os layouts pai (por exemplo, FrameLayout) possam alinhar adequadamente sua visualização personalizada. Ele fornece um conjunto de measureSpecs que você pode usar para determinar a altura e a largura da sua visualização. Aqui criamos um quadrado certificando-nos de que a altura e a largura são as mesmas:

    @Override
protected void onMeasure(int widthMeasureSpec, int heightMeasureSpec) {
    int w = MeasureSpec.getSize(widthMeasureSpec);
    int h = MeasureSpec.getSize(heightMeasureSpec);

    int size = Math.min(w, h);
    setMeasuredDimension(size, size);
}

Observe que onMeasure(...) deve conter pelo menos uma chamada para setMeasuredDimension(..) ou então sua visualização personalizada irá falhar com um IllegalStateException.

  1. Implemente seu próprio método onSizeChanged(...): isso permite que você capture a altura e a largura atuais de sua visualização personalizada para ajustar adequadamente seu código de renderização. Aqui nós apenas calculamos nosso centro e nosso raio:

    @Override
protected void onSizeChanged(int w, int h, int oldw, int oldh) {
    mCenterX = w / 2f;
    mCenterY = h / 2f;
    mRadius = Math.min(w, h) / 2f;
}

  2. Implemente seu próprio método onDraw(...): é aqui que você implementa a renderização real de sua visualização. Ele fornece um objeto Canvas no qual você pode desenhar (consulte a documentação oficial do Canvas para todos os métodos de desenho disponíveis).

    @Override
protected void onDraw(Canvas canvas) {
    // draw face
    canvas.drawCircle(mCenterX, mCenterY, mRadius, mCirclePaint);
    // draw eyes
    float eyeRadius = mRadius / 5f;
    float eyeOffsetX = mRadius / 3f;
    float eyeOffsetY = mRadius / 3f;
    canvas.drawCircle(mCenterX - eyeOffsetX, mCenterY - eyeOffsetY, eyeRadius, mEyeAndMouthPaint);
    canvas.drawCircle(mCenterX + eyeOffsetX, mCenterY - eyeOffsetY, eyeRadius, mEyeAndMouthPaint);
    // draw mouth
    float mouthInset = mRadius /3f;
    mArcBounds.set(mouthInset, mouthInset, mRadius * 2 - mouthInset, mRadius * 2 - mouthInset);
    canvas.drawArc(mArcBounds, 45f, 90f, false, mEyeAndMouthPaint);
}

  3. Adicione sua visualização personalizada a um layout: a visualização personalizada agora pode ser incluída em qualquer arquivo de layout que você tenha. Aqui nós apenas o envolvemos dentro de um FrameLayout:

    <FrameLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <com.example.app.SmileyView
        android:layout_width="match_parent"
        android:layout_height="match_parent" />
</FrameLayout>

Observe que é recomendável compilar seu projeto após a conclusão do código de exibição. Sem construí-lo, você não poderá ver a visualização em uma tela de visualização no Android Studio.

Adicionando atributos às visualizações #

As visualizações personalizadas também podem ter atributos personalizados que podem ser usados ​​em arquivos de recursos de layout do Android. Para adicionar atributos à sua visualização personalizada, você precisa fazer o seguinte:

  1. Defina o nome e o tipo de seus atributos: isso é feito dentro de res/values/attrs.xml (crie-o se necessário). O arquivo a seguir define um atributo color para a cor do rosto do smiley e um atributo enum para a expressão do smiley:

    <resources>
    <declare-styleable name="SmileyView">
        <attr name="smileyColor" format="color" />
        <attr name="smileyExpression" format="enum">
            <enum name="happy" value="0"/>
            <enum name="sad" value="1"/>
        </attr>
    </declare-styleable>
    <!-- attributes for other views -->
</resources>

  2. Use seus atributos dentro de seu layout: isso pode ser feito dentro de qualquer arquivo de layout que use sua visualização personalizada. O arquivo de layout a seguir cria uma tela com um smiley amarelo feliz:

    <FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_height="match_parent"
    android:layout_width="match_parent">

    <com.example.app.SmileyView
        android:layout_height="56dp"
        android:layout_width="56dp"
        app:smileyColor="#ffff00"
        app:smileyExpression="happy" />
</FrameLayout>

Dica: os atributos personalizados não funcionam com o prefixo tools: no Android Studio 2.1 e anteriores (e possivelmente em versões futuras). Neste exemplo, substituir app:smileyColor por tools:smileyColor resultaria em smileyColor não sendo definido durante o tempo de execução nem em tempo de design.

  1. Leia seus atributos: isso é feito dentro do código-fonte da sua visualização personalizada. O seguinte snippet de SmileyView demonstra como os atributos podem ser extraídos:

    public class SmileyView extends View {
    // ...

    public SmileyView(Context context) {
        this(context, null);
    }

    public SmileyView(Context context, AttributeSet attrs) {
        this(context, attrs, 0);
    }

    public SmileyView(Context context, AttributeSet attrs, int defStyleAttr) {
        super(context, attrs, defStyleAttr);

        TypedArray a = context.obtainStyledAttributes(attrs, R.styleable.SmileyView, defStyleAttr, 0);
        mFaceColor = a.getColor(R.styleable.SmileyView_smileyColor, Color.TRANSPARENT);
        mFaceExpression = a.getInteger(R.styleable.SmileyView_smileyExpression, Expression.HAPPY);
        // Important: always recycle the TypedArray
        a.recycle();

        // initPaints(); ...
    }
}

  2. (Opcional) Adicionar estilo padrão: isso é feito adicionando um estilo com os valores padrão e carregando-o dentro de sua visualização personalizada. O seguinte estilo de smiley padrão representa um amarelo feliz:

    <!-- styles.xml -->
<style name="DefaultSmileyStyle">
    <item name="smileyColor">#ffff00</item>
    <item name="smileyExpression">happy</item>
</style>

    Which gets applied in our SmileyView by adding it as the last parameter of the call to obtainStyledAttributes (see code in step 3):

    TypedArray a = context.obtainStyledAttributes(attrs, R.styleable.SmileyView, defStyleAttr, R.style.DefaultSmileyViewStyle);

    Note that any attribute values set in the inflated layout file (see code in step 2) will override the corresponding values of the default style.

  3. (Opcional) Forneça estilos dentro de temas: isso é feito adicionando um novo atributo de referência de estilo que pode ser usado dentro de seus temas e fornecendo um estilo para esse atributo. Aqui nós simplesmente nomeamos nosso atributo de referência como smileyStyle:

    <!-- attrs.xml -->
<attr name="smileyStyle" format="reference" />

    Which we then provide a style for in our app theme (here we just reuse the default style from step 4):

    <!-- themes.xml -->
<style name="AppTheme" parent="AppBaseTheme">
    <item name="smileyStyle">@style/DefaultSmileyStyle</item>
</style>

Dicas de desempenho do CustomView #

Não aloque novos objetos no onDraw

@Override
protected void onDraw(Canvas canvas) {
    super.onDraw(canvas);
    Paint paint = new Paint(); //Do not allocate here
}

Em vez de desenhar drawables em canvas…

drawable.setBounds(boundsRect);

drawable.draw(canvas);

Use um Bitmap para desenhar mais rápido:

canvas.drawBitmap(bitmap, srcRect, boundsRect, paint);

Não redesenhe a visualização inteira para atualizar apenas uma pequena parte dela. Em vez disso, redesenhe a parte específica da vista.

invalidate(boundToBeRefreshed);

Se sua visualização está fazendo alguma animação contínua, por exemplo, um mostrador de relógio mostrando a cada segundo, pelo menos pare a animação em onStop() da atividade e inicie-a novamente em onStart() da atividade.

Não faça nenhum cálculo dentro do método onDraw de uma view, você deve terminar o desenho antes de chamar invalidate(). Ao usar essa técnica, você pode evitar a queda de quadros em sua visualização.

Rotações

As operações básicas de uma visualização são traduzir, girar, etc… Quase todos os desenvolvedores enfrentaram esse problema quando usam bitmap ou gradientes em sua visualização personalizada. Se a exibição for mostrar uma exibição girada e o bitmap tiver que ser girado nessa exibição personalizada, muitos de nós pensarão que será caro. Muitos pensam que girar um bitmap é muito caro porque, para isso, você precisa traduzir a matriz de pixels do bitmap. Mas a verdade é que não é tão difícil! Em vez de girar o bitmap, basta girar a própria tela!

// Save the canvas state
int save = canvas.save();
// Rotate the canvas by providing the  center point as pivot and angle
canvas.rotate(pivotX, pivotY, angle);
// Draw whatever you want
// Basically whatever you draw here will be drawn as per the angle you rotated the canvas
canvas.drawBitmap(...);
// Now restore your your canvas to its original state
canvas.restore(save);
// Unless canvas is restored to its original state, further draw will also be rotated.

Criando uma visão composta #

Uma visualização composta é um ViewGroup personalizado que é tratado como uma visualização única pelo código do programa ao redor. Tal ViewGroup pode ser realmente útil em um design do tipo DDD, porque pode corresponder a um agregado, neste exemplo, um contato. Ele pode ser reutilizado em todos os lugares em que o contato é exibido.

Isso significa que o código do controlador circundante, uma Activity, Fragment ou Adapter, pode simplesmente passar o objeto de dados para a exibição sem separá-lo em vários widgets de interface do usuário diferentes.

Isso facilita a reutilização de código e contribui para um design melhor de acordo com pricípios SOLID.

O layout XML

Geralmente é aqui que você começa. Você tem um bit de XML existente que você está reutilizando, talvez como um <include/>. Extraia-o em um arquivo XML separado e envolva a tag raiz em um elemento <merge>:

<?xml version="1.0" encoding="utf-8"?>
<merge xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

        <ImageView
            android:id="@+id/photo"
            android:layout_width="48dp"
            android:layout_height="48dp"
            android:layout_alignParentRight="true" />

        <TextView
            android:id="@+id/name"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:layout_toLeftOf="@id/photo" />

        <TextView
            android:id="@+id/phone_number"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:layout_below="@id/name"
            android:layout_toLeftOf="@id/photo" />
</merge>

Este arquivo XML continua funcionando perfeitamente no Editor de Layout no Android Studio. Você pode tratá-lo como qualquer outro layout.

O ViewGroup composto

Depois de ter o arquivo XML, crie o grupo de exibição personalizado.

import android.annotation.TargetApi;
import android.content.Context;
import android.os.Build;
import android.util.AttributeSet;
import android.view.LayoutInflater;
import android.view.View;
import android.widget.RelativeLayout;
import android.widget.ImageView;
import android.widget.TextView;

import myapp.R;

/**
 * A compound view to show contacts.
 *
 * This class can be put into an XML layout or instantiated programmatically, it
 * will work correctly either way.
 */
public class ContactView extends RelativeLayout {

    // This class extends RelativeLayout because that comes with an automatic
    // (MATCH_PARENT, MATCH_PARENT) layout for its child item. You can extend
    // the raw android.view.ViewGroup class if you want more control. See the
    // note in the layout XML why you wouldn't want to extend a complex view
    // such as RelativeLayout.

    // 1. Implement superclass constructors.
    public ContactView(Context context) {
        super(context);
        init(context, null);
    }

    // two extra constructors left out to keep the example shorter

    @TargetApi(Build.VERSION_CODES.LOLLIPOP)
    public ContactView(Context context, AttributeSet attrs, int defStyleAttr, int defStyleRes) {
        super(context, attrs, defStyleAttr, defStyleRes);
        init(context, attrs);
    }

    // 2. Initialize the view by inflating an XML using `this` as parent
    private TextView mName;
    private TextView mPhoneNumber;
    private ImageView mPhoto;

    private void init(Context context, AttributeSet attrs) {
        LayoutInflater.from(context).inflate(R.layout.contact_view, this, true);
        mName = (TextView) findViewById(R.id.name);
        mPhoneNumber = (TextView) findViewById(R.id.phone_number);
        mPhoto = (ImageView) findViewById(R.id.photo);
    }

    // 3. Define a setter that's expressed in your domain model. This is what the example is
    //    all about. All controller code can just invoke this setter instead of fiddling with
    //    lots of strings, visibility options, colors, animations, etc. If you don't use a
    //    custom view, this code will usually end up in a static helper method (bad) or copies 
    //    of this code will be copy-pasted all over the place (worse).
    public void setContact(Contact contact) {
        mName.setText(contact.getName());
        mPhoneNumber.setText(contact.getPhoneNumber());
        if (contact.hasPhoto()) {
            mPhoto.setVisibility(View.VISIBLE);
            mPhoto.setImageBitmap(contact.getPhoto());
        } else {
            mPhoto.setVisibility(View.GONE);
        }
    }
}

O método init(Context, AttributeSet) é onde você leria quaisquer atributos XML personalizados, conforme explicado em [Adicionando atributos a visualizações][aatv].

Com essas peças no lugar, você pode usá-lo em seu aplicativo.

[aatv]:https://www.wikiod.com/pt/android/como-criar-visualizacoes-personalizadas#Adicionando atributos às visualizações Uso em XML

Aqui está um exemplo fragment_contact_info.xml que ilustra como você colocaria um único ContactView no topo de uma lista de mensagens:

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <!-- The compound view becomes like any other view XML element -->
    <myapp.ContactView
        android:id="@+id/contact"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"/>

    <android.support.v7.widget.RecyclerView
        android:id="@+id/message_list"
        android:layout_width="match_parent"
        android:layout_height="0dp"
        android:layout_weight="1"/>

</LinearLayout>

Uso no Código

Aqui está um exemplo RecyclerView.Adapter que mostra uma lista de contatos. Este exemplo ilustra o quanto o código do controlador fica mais limpo quando está completamente livre de manipulação de View.

package myapp;

import android.content.Context;
import android.support.v7.widget.RecyclerView;
import android.view.ViewGroup;

public class ContactsAdapter extends RecyclerView.Adapter<ContactsViewHolder> {

    private final Context context;

    public ContactsAdapter(final Context context) {
        this.context = context;
    }

    @Override
    public ContactsViewHolder onCreateViewHolder(ViewGroup parent, int viewType) {
        ContactView v = new ContactView(context); // <--- this
        return new ContactsViewHolder(v);
    }

    @Override
    public void onBindViewHolder(ContactsViewHolder holder, int position) {
        Contact contact = this.getItem(position);
        holder.setContact(contact);  // <--- this
    }

    static class ContactsViewHolder extends RecyclerView.ViewHolder {

        public ContactsViewHolder(ContactView itemView) {
            super(itemView);
        }

        public void setContact(Contact contact) {
            ((ContactView) itemView).setContact(contact); // <--- this
        }
    }
}

Visualização composta para SVG/VectorDrawable como drawableRight #

O principal motivo para desenvolver esta visão composta é que os dispositivos abaixo de 5.0 não suportam svg em drawable dentro de TextView/EditText. Mais uma vantagem é que podemos definir height e width de drawableRight dentro de EditText. I have separated it from my project and created in separate module.

Nome do módulo: custom_edit_drawable (nome abreviado para prefixo-c_d_e)

Prefixo “c_d_e_” a ser usado para que os recursos do módulo do aplicativo não os substituam por engano. Exemplo: o prefixo “abc” é usado pelo google na biblioteca de suporte.

build.gradle

dependencies {
   compile 'com.android.support:appcompat-v7:25.3.1'

use AppCompat >= 23

<?xml version="1.0" encoding="utf-8"?>
<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"

        android:paddingStart="5dp" />

Código: EditTextWithDrawable.java

public class EditTextWithDrawable extends FrameLayout {
    public AppCompatImageView mDrawableRight;
    public EditText mEditText;

    public EditTextWithDrawable(Context context) {
        super(context);
        init(null);
    }

    public EditTextWithDrawable(Context context, AttributeSet attrs) {
        super(context, attrs);
        init(attrs);
    }

    public EditTextWithDrawable(Context context, AttributeSet attrs, int defStyleAttr) {
        super(context, attrs, defStyleAttr);
        init(attrs);
    }

    @TargetApi(Build.VERSION_CODES.LOLLIPOP)
    public EditTextWithDrawable(Context context, AttributeSet attrs, int defStyleAttr, int defStyleRes) {
        super(context, attrs, defStyleAttr, defStyleRes);
        init(attrs);
    }

    private void init(AttributeSet attrs) {
        if (attrs != null && !isInEditMode()) {
            LayoutInflater inflater = (LayoutInflater) getContext()
                    .getSystemService(Context.LAYOUT_INFLATER_SERVICE);
            inflater.inflate(R.layout.c_e_d_compound_view, this, true);
            mDrawableRight = (AppCompatImageView) ((FrameLayout) getChildAt(0)).getChildAt(1);
            mEditText = (EditText) ((FrameLayout) getChildAt(0)).getChildAt(0);

            TypedArray attributeArray = getContext().obtainStyledAttributes(
                    attrs,
                    R.styleable.EditTextWithDrawable);

            int drawableRes =
                    attributeArray.getResourceId(
                            R.styleable.EditTextWithDrawable_c_e_d_drawableRightSVG, -1);
            if (drawableRes != -1) {
                mDrawableRight.setImageResource(drawableRes);
            }

            mEditText.setHint(attributeArray.getString(
                    R.styleable.EditTextWithDrawable_c_e_d_hint));
            mEditText.setTextColor(attributeArray.getColor(
                    R.styleable.EditTextWithDrawable_c_e_d_textColor, Color.BLACK));
            int textSize = attributeArray.getDimensionPixelSize(R.styleable.EditTextWithDrawable_c_e_d_textSize, 15);
            mEditText.setTextSize(TypedValue.COMPLEX_UNIT_PX, textSize);
            android.view.ViewGroup.LayoutParams layoutParams = mDrawableRight.getLayoutParams();
            layoutParams.width = (textSize * 3) / 2;
            layoutParams.height = (textSize * 3) / 2;
            mDrawableRight.setLayoutParams(layoutParams);

            attributeArray.recycle();
        }
    }
}

Respondendo a eventos de toque #

Muitas visualizações personalizadas precisam aceitar a interação do usuário na forma de eventos de toque. Você pode obter acesso a eventos de toque substituindo onTouchEvent. Há uma série de ações que você pode filtrar. Os principais são

  • ACTION_DOWN: Isso é acionado uma vez quando seu dedo toca a vista pela primeira vez.
  • ACTION_MOVE: Isso é chamado toda vez que seu dedo se move um pouco pela visualização. É chamado muitas vezes.
  • ACTION_UP: Esta é a última ação a ser chamada quando você levanta o dedo da tela.

Você pode adicionar o seguinte método à sua exibição e observar a saída do log quando tocar e mover o dedo pela exibição.

@Override
public boolean onTouchEvent(MotionEvent event) {

    int x = (int) event.getX();
    int y = (int) event.getY();
    int action = event.getAction();

    switch (action) {
        case MotionEvent.ACTION_DOWN:
            Log.i("CustomView", "onTouchEvent: ACTION_DOWN: x = " + x + ", y = " + y);
            break;

        case MotionEvent.ACTION_MOVE:
            Log.i("CustomView", "onTouchEvent: ACTION_MOVE: x = " + x + ", y = " + y);
            break;

        case MotionEvent.ACTION_UP:
            Log.i("CustomView", "onTouchEvent: ACTION_UP: x = " + x + ", y = " + y);
            break;
    }
    return true;
}
