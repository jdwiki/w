  • Quando você optar por implementar o App Indexing, poderá encontrar muitos blogs, documentação por aí que pode confundi-lo, neste caso, sugiro que você se atenha aos documentos oficiais fornecidos pelo Firebase-Google. Mesmo se você quiser usar terceiros para fazer isso, primeiro tente seguir esta documentação, pois isso lhe dará uma ideia clara de como as coisas estão funcionando.

  • O Google levará cerca de 24 horas para indexar seu conteúdo. Então seja paciente. Você pode fazer testes para que tudo esteja bem do seu lado.

  • O primeiro exemplo permite que você suporte a URL HTTP do seu site para redirecionar em seu aplicativo. Isso funcionará como, se você pesquisou uma consulta na pesquisa do google, os resultados mostram uma URL do seu site, cujos links de aplicativos estão presentes em seu aplicativo que já está instalado. Ao clicar nessa URL, você será redirecionado diretamente na tela do seu aplicativo correspondente a esse resultado da pesquisa. É isso que eu descobri para isso.

  • A adição de API AppIndexing indexa seu conteúdo e é usado em autocompletar na barra de pesquisa do Google. Vamos dar um exemplo do aplicativo inShorts para cada página, há um título e uma pequena descrição. Depois de ler 2 ou 3 títulos, feche o aplicativo e vá para o google searchBar.

Tente inserir o título que você acabou de ler, você receberá uma sugestão de página do aplicativo com esse título como título. Isso é diferente das sugestões de aplicativos que você obtém ao pesquisar por aplicativos. Isso acontece porque você escreveu o código da API AppIndexing para esta página em particular e o título é o mesmo que você inicializou em onCreate().

URLs HTTP compatíveis #

Etapa 1 :- Permitir que o Google rastreie seu conteúdo. Edite o arquivo robot.txt do servidor. Você pode controlar o rastreamento do Google para seu conteúdo editando este arquivo, consulte este link para obter mais detalhes.

Etapa 2 :- Associe seu aplicativo ao seu site.Inclua assetlinks.json Você o carrega para o diretório .well-known do seu servidor web.Conteúdo de seus assetlinks.json são

       [{
"relation": ["delegate_permission/common.handle_all_urls"],
"target" : 
{ "namespace": "android_app",
"package_name": "<your_package_name>",
"sha256_cert_fingerprints": ["<hash_of_app_certificate>"] }

}]

Passo 3 :- Inclua links de aplicativos em seu arquivo de manifesto para redirecionar URLs para seu aplicativo, como abaixo,

<activity
       android:name=".activity.SampleActivity"
       android:label="@string/app_name"
       android:windowSoftInputMode="adjustResize|stateAlwaysHidden">
       <intent-filter>
             <action android:name="android.intent.action.VIEW" />
             <category android:name="android.intent.category.DEFAULT" />
             <category android:name="android.intent.category.BROWSABLE" />
       <data
             android:host="example.live"
             android:pathPrefix="/vod"
             android:scheme="https"/>
       <data
             android:host="example.live"
             android:pathPrefix="/vod"
             android:scheme="http"/>
     </intent-filter>
   </activity>

Consulte isso se quiser aprender sobre cada tag aqui.

*Ação> Especifique a ação de intent ACTION_VIEW para que o filtro de intents possa ser acessado a partir da Pesquisa Google.

*dados> Adicione uma ou mais tags , em que cada tag representa um formato de URI que resolve a atividade. No mínimo, a tag deve incluir o atributo android:scheme. Você pode adicionar atributos adicionais para refinar ainda mais o tipo de URI que a atividade aceita. Por exemplo, você pode ter várias atividades que aceitam URIs semelhantes, mas que diferem simplesmente com base no nome do caminho. Nesse caso, use o atributo android:path ou suas variantes (pathPattern ou pathPrefix) para diferenciar qual atividade o sistema deve abrir para diferentes caminhos de URI.

*categoria> Inclua a categoria BROWSABLE. A categoria BROWSABLE é necessária para que o filtro de intents possa ser acessado em um navegador da web. Sem ele, clicar em um link em um navegador não pode resolver seu aplicativo. A categoria DEFAULT é opcional, mas recomendada. Sem essa categoria, a atividade pode ser iniciada apenas com uma intenção explícita, usando o nome do componente do aplicativo.

   if (getAction() != null) {

       mClient.disconnect();
    }

}

 public Action getAction() {
   Thing object = new Thing.Builder()
       .setName(mTitle)
       .setDescription(mDescription)
       .setUrl(mUrl)
       .build();

 return new Action.Builder(Action.TYPE_WATCH)
       .setObject(object)
       .setActionStatus(Action.STATUS_TYPE_COMPLETED)
       .build();
}
