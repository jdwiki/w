Parâmetros #

Parâmetro Detalhes
Parâmetros o tipo dos parâmetros enviados para a tarefa na execução.
Progresso o tipo das unidades de progresso publicadas durante a computação em segundo plano
Resultado o tipo do resultado da computação em segundo plano.

Uso básico #

No Android Activities e Services, a maioria dos retornos de chamada são executados no thread principal. Isso simplifica a atualização da interface do usuário, mas a execução de tarefas pesadas de processador ou E/S no thread principal pode fazer com que sua interface do usuário pause e não responda (documentação oficial sobre o que acontece).

Você pode remediar isso colocando essas tarefas mais pesadas em um thread em segundo plano.

Uma maneira de fazer isso é usar um AsyncTask, que fornece uma estrutura para facilitar o uso de um Thread em segundo plano e também executar tarefas de Thread de interface do usuário antes, durante e depois que o Thread em segundo plano tiver concluído seu trabalho.

Métodos que podem ser substituídos ao estender AsyncTask:

  • onPreExecute() : invocado no thread da interface do usuário antes que a tarefa seja executada
  • doInBackground(): invocado no thread de fundo imediatamente após onPreExecute() terminar a execução.
  • onProgressUpdate(): invocado no encadeamento da interface do usuário após uma chamada para publishProgress(Progress...).
  • onPostExecute(): invocado no encadeamento da interface do usuário após a conclusão da computação em segundo plano

Exemplo #

public class MyCustomAsyncTask extends AsyncTask<File, Void, String> {

    
    @Override
    protected void onPreExecute(){
        // This runs on the UI thread before the background thread executes.
        super.onPreExecute();
        // Do pre-thread tasks such as initializing variables. 
        Log.v("myBackgroundTask", "Starting Background Task");  
    }

    @Override
    protected String doInBackground(File... params) {
        // Disk-intensive work. This runs on a background thread.
        // Search through a file for the first line that contains "Hello", and return
        // that line.
        try (Scanner scanner = new Scanner(params[0])) {
            while (scanner.hasNextLine()) {
                final String line = scanner.nextLine();
                publishProgress(); // tell the UI thread we made progress

                if (line.contains("Hello")) {
                    return line;
                }
            }
            return null;
        }
    }

    @Override
    protected void onProgressUpdate(Void...p) {
        // Runs on the UI thread after publishProgress is invoked
        Log.v("Read another line!")
    }        

    @Override
    protected void onPostExecute(String s) {
        // This runs on the UI thread after complete execution of the doInBackground() method
        // This function receives result(String s) returned from the doInBackground() method.
        // Update UI with the found string.
        TextView view = (TextView) findViewById(R.id.found_string);
        if (s != null) {
            view.setText(s);
        } else {
            view.setText("Match not found.");
        }
    }

}

Uso: #

MyCustomAsyncTask asyncTask = new MyCustomAsyncTask<File, Void, String>();
// Run the task with a user supplied filename.

ou simplesmente:

Observação #

Ao definir um AsyncTask podemos passar três tipos entre colchetes < >. Definido como <Params, Progress, Result> (consulte seção de parâmetros)

No exemplo anterior, usamos os tipos <File, Void, String>:

AsyncTask<File, Void, String>
// Params has type File
// Progress has unused type
// Result has type String

Void é usado quando você deseja marcar um tipo como não utilizado.

Observe que você não pode passar tipos primitivos (ou seja, int, float e 6 outros) como parâmetros. Nesses casos, você deve passar suas classes wrapper, por exemplo. Integer em vez de int, ou Float em vez de float.

O ciclo de vida AsyncTask e Activity

AsyncTasks não segue o ciclo de vida das instâncias de atividade. Se você iniciar uma AsyncTask dentro de uma Activity e girar o dispositivo, a Activity será destruída e uma nova instância será criada. Mas o AsyncTask não morrerá. Continuará a viver até que se complete.

Solução: AsyncTaskLoader

Uma subclasse de Loaders é o AsyncTaskLoader. Essa classe executa a mesma função que o AsyncTask, mas muito melhor. Ele pode lidar com alterações de configuração de atividade com mais facilidade e se comporta dentro dos ciclos de vida de fragmentos e atividades. O legal é que o AsyncTaskLoader pode ser usado em qualquer situação em que o AsyncTask estiver sendo usado. Sempre que os dados precisam ser carregados na memória para a atividade/fragmento manipular, o AsyncTaskLoader pode fazer o trabalho melhor.

Passe Activity como WeakReference para evitar vazamentos de memória #

É comum que um AsyncTask exija uma referência à Activity que o chamou.

Se AsyncTask for uma classe interna da Activity, você poderá fazer referência a ela e a quaisquer variáveis/métodos de membro diretamente.

Se, no entanto, o AsyncTask não for uma classe interna da Activity, você precisará passar uma referência de Activity para o AsyncTask. Ao fazer isso, um possível problema que pode ocorrer é que o AsyncTask manterá a referência da Activity até que o AsyncTask tenha concluído seu trabalho em seu thread em segundo plano. Se a atividade for concluída ou eliminada antes que o trabalho de thread em segundo plano do AsyncTask seja concluído, o AsyncTask ainda terá sua referência à atividade e, portanto, não poderá ser coletado como lixo.

Como resultado, isso causará um vazamento de memória.

Para evitar que isso aconteça, use um WeakReference na AsyncTask em vez de ter uma referência direta à Activity.

Aqui está um exemplo de AsyncTask que utiliza um WeakReference:

private class MyAsyncTask extends AsyncTask<String, Void, Void> {

    private WeakReference<Activity> mActivity;

    public MyAsyncTask(Activity activity) {
        mActivity = new WeakReference<Activity>(activity);
    }

    @Override
    protected void onPreExecute() {
        final Activity activity = mActivity.get();
        if (activity != null) {
            ....
        }
    }

    @Override
    protected Void doInBackground(String... params) {
        //Do something
        String param1 = params[0];
        String param2 = params[1];
        return null;
    }

    @Override
    protected void onPostExecute(Void result) {
        final Activity activity = mActivity.get();
        if (activity != null) {
            activity.updateUI();
        }
    }
}

Chamando o AsyncTask de uma atividade:

new MyAsyncTask(this).execute("param1", "param2");

Chamando a AsyncTask de um Fragmento:

new MyAsyncTask(getActivity()).execute("param1", "param2");

Baixe a imagem usando AsyncTask no Android #

Este tutorial explica como baixar o Image usando o AsyncTask no Android. O exemplo abaixo faz o download da imagem enquanto mostra a barra de progresso durante o download.

Compreendendo o Android AsyncTask

A tarefa assíncrona permite que você implemente o MultiThreading sem sujar as mãos nos threads. AsyncTask permite o uso adequado e fácil do thread de interface do usuário. Ele permite realizar operações em segundo plano e passar os resultados no thread da interface do usuário. Se você estiver fazendo algo isolado relacionado à interface do usuário, por exemplo, baixando dados para apresentar em uma lista, vá em frente e use AsyncTask.
  • AsyncTasks should ideally be used for short operations (a few seconds at the most.)
  • An asynchronous task is defined by 3 generic types, called Params, Progress and Result, and 4 steps, called onPreExecute(), doInBackground(), onProgressUpdate() and onPostExecute().
  • In onPreExecute() you can define code, which need to be executed before background processing starts.
  • doInBackground have code which needs to be executed in background, here in doInBackground() we can send results to multiple times to event thread by publishProgress() method, to notify background processing has been completed we can return results simply.
  • onProgressUpdate() method receives progress updates from doInBackground() method, which is published via publishProgress() method, and this method can use this progress update to update event thread
  • onPostExecute() method handles results returned by doInBackground() method.
  • The generic types used are
    • Params, the type of the parameters sent to the task upon execution
    • Progress, the type of the progress units published during the background computation.
    • Result, the type of the result of the background computation.
  • If an async task not using any types, then it can be marked as Void type.
  • An running async task can be cancelled by calling cancel(boolean) method.

package com.javatechig.droid;

import java.io.InputStream;
import org.apache.http.HttpEntity;
import org.apache.http.HttpResponse;
import org.apache.http.HttpStatus;
import org.apache.http.client.methods.HttpGet;
import org.apache.http.impl.client.DefaultHttpClient;
import android.app.Activity;
import android.app.ProgressDialog;
import android.graphics.Bitmap;
import android.graphics.BitmapFactory;
import android.os.AsyncTask;
import android.os.Bundle;
import android.util.Log;
import android.view.View;
import android.view.View.OnClickListener;
import android.widget.Button;
import android.widget.ImageView;

public class ImageDownladerActivity extends Activity {

    private ImageView downloadedImg;
    private ProgressDialog simpleWaitDialog;
    private String downloadUrl = "http://www.9ori.com/store/media/images/8ab579a656.jpg";

    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.asynch);
        Button imageDownloaderBtn = (Button) findViewById(R.id.downloadButton);

        downloadedImg = (ImageView) findViewById(R.id.imageView);

        imageDownloaderBtn.setOnClickListener(new OnClickListener() {

            @Override
            public void onClick(View v) {
                // TODO Auto-generated method stub
                new ImageDownloader().execute(downloadUrl);
            }

        });
    }

    private class ImageDownloader extends AsyncTask {

        @Override
        protected Bitmap doInBackground(String... param) {
            // TODO Auto-generated method stub
            return downloadBitmap(param[0]);
        }

        @Override
        protected void onPreExecute() {
            Log.i("Async-Example", "onPreExecute Called");
            simpleWaitDialog = ProgressDialog.show(ImageDownladerActivity.this,
                    "Wait", "Downloading Image");

        }

        @Override
        protected void onPostExecute(Bitmap result) {
            Log.i("Async-Example", "onPostExecute Called");
            downloadedImg.setImageBitmap(result);
            simpleWaitDialog.dismiss();

        }

        private Bitmap downloadBitmap(String url) {
            // initilize the default HTTP client object
            final DefaultHttpClient client = new DefaultHttpClient();

            //forming a HttpGet request 
            final HttpGet getRequest = new HttpGet(url);
            try {

                HttpResponse response = client.execute(getRequest);

                //check 200 OK for success
                final int statusCode = response.getStatusLine().getStatusCode();

                if (statusCode != HttpStatus.SC_OK) {
                    Log.w("ImageDownloader", "Error " + statusCode + 
                            " while retrieving bitmap from " + url);
                    return null;

                }

                final HttpEntity entity = response.getEntity();
                if (entity != null) {
                    InputStream inputStream = null;
                    try {
                        // getting contents from the stream 
                        inputStream = entity.getContent();

                        // decoding stream data back into image Bitmap that android understands
                        final Bitmap bitmap = BitmapFactory.decodeStream(inputStream);

                        return bitmap;
                    } finally {
                        if (inputStream != null) {
                            inputStream.close();
                        }
                        entity.consumeContent();
                    }
                }
            } catch (Exception e) {
                // You Could provide a more explicit error message for IOException
                getRequest.abort();
                Log.e("ImageDownloader", "Something went wrong while" +
                        " retrieving bitmap from " + url + e.toString());
            } 

            return null;
        }
    }
}

Cancelando AsyncTask #

YourAsyncTask task = new YourAsyncTask();
task.execute();
task.cancel();

Isso não interrompe sua tarefa se estiver em andamento, apenas define o sinalizador cancelado que pode ser verificado verificando o valor de retorno de isCancelled() (supondo que seu código esteja em execução) fazendo isso:

class YourAsyncTask extends AsyncTask<Void, Void, Void> {
    @Override
    protected Void doInBackground(Void... params) {
        while(!isCancelled()) {
            ... doing long task stuff
            //Do something, you need, upload part of file, for example
            if (isCancelled()) {    
                return null; // Task was detected as canceled
            }
            if (yourTaskCompleted) {
                return null;
            }
        }
    }
}

Observação #

Se uma AsyncTask for cancelada enquanto doInBackground(Params... params) ainda estiver em execução, então o método onPostExecute(Result result) NÃO será chamado após o retorno de doInBackground(Params... params). Em vez disso, o AsyncTask chamará o onCancelled(Result result) para indicar que a tarefa foi cancelada durante a execução.

AsyncTask: Execução Serial e Execução Paralela de Tarefa #

AsyncTask é uma classe abstrata e não herda a classe Thread. Ele tem um método abstract doInBackground(Params... params), que é substituído para executar a tarefa. Este método é chamado de AsyncTask.call().

Executor fazem parte do pacote java.util.concurrent.

Além disso, AsyncTask contém 2 Executors

THREAD_POOL_EXECUTOR #

Ele usa threads de trabalho para executar as tarefas paralelamente.

public static final Executor THREAD_POOL_EXECUTOR = new ThreadPoolExecutor(CORE_POOL_SIZE, MAXIMUM_POOL_SIZE, KEEP_ALIVE, TimeUnit.SECONDS, sPoolWorkQueue, sThreadFactory);

SERIAL_EXECUTOR #

Ele executa a tarefa em série, ou seja, um por um.

private static class SerialExecutor implements Executor { }

Ambos Executors são estáticos, portanto, apenas um objeto THREAD_POOL_EXECUTOR e um SerialExecutor existem, mas você pode criar vários objetos AsyncTask.

Portanto, se você tentar fazer várias tarefas em segundo plano com o Executor padrão (SerialExecutor), essas tarefas serão enfileiradas e executadas serialmente.

Se você tentar fazer várias tarefas em segundo plano com THREAD_POOL_EXECUTOR, elas serão executadas paralelamente.

Exemplo:

public class MainActivity extends Activity {
    private Button bt;
    private int CountTask = 0;
    private static final String TAG = "AsyncTaskExample";

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        bt = (Button) findViewById(R.id.button);
        bt.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                BackgroundTask backgroundTask = new BackgroundTask ();
                Integer data[] = { ++CountTask, null, null };

                // Task Executed in thread pool ( 1 )
                backgroundTask.executeOnExecutor(AsyncTask.THREAD_POOL_EXECUTOR, data);

                // Task executed Serially ( 2 )
                // Uncomment the below code and comment the above code of Thread
                // pool Executor and check
                // backgroundTask.execute(data);
                Log.d(TAG, "Task = " + (int) CountTask + " Task Queued");

            }
        });

    }

    private class BackgroundTask extends AsyncTask<Integer, Integer, Integer> {
        int taskNumber;

        @Override
        protected Integer doInBackground(Integer... integers) {
            taskNumber = integers[0];

            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                // TODO Auto-generated catch block
                e.printStackTrace();
            }

            Log.d(TAG, "Task = " + taskNumber + " Task Running in Background");

            publishProgress(taskNumber);
            return null;
        }

        @Override
        protected void onPreExecute() {
            super.onPreExecute();
        }

        @Override
        protected void onPostExecute(Integer aLong) {
            super.onPostExecute(aLong);
        }

        @Override
        protected void onProgressUpdate(Integer... values) {
            super.onProgressUpdate(values);
            Log.d(TAG, "Task = " + (int) values[0]
                    + " Task Execution Completed");
        }
    }
}

Progresso da publicação #

Às vezes, precisamos atualizar o progresso do cálculo feito por um AsyncTask. Esse progresso pode ser representado por uma string, um inteiro, etc. Para fazer isso, temos que usar duas funções. Primeiro, precisamos definir a função onProgressUpdate cujo tipo de parâmetro é o mesmo que o segundo parâmetro de tipo de nossa AsyncTask.

class YourAsyncTask extends AsyncTask<URL, Integer, Long> {
    @Override
    protected void onProgressUpdate(Integer... args) {
        setProgressPercent(args[0])
    }
}

Segundo, temos que usar a função publishProgress necessariamente na função doInBackground, e isso é tudo, o método anterior fará todo o trabalho.

protected Long doInBackground(URL... urls) {
     int count = urls.length;
     long totalSize = 0;
     for (int i = 0; i < count; i++) {
         totalSize += Downloader.downloadFile(urls[i]);
         publishProgress((int) ((i / (float) count) * 100));
     }
     return totalSize;
 }

Ordem de execução #

Quando introduzido pela primeira vez, AsyncTasks foram executados em série em um único thread em segundo plano. Começando com DONUT, isso foi alterado para um conjunto de threads permitindo que várias tarefas operem em paralelo. Começando com HONEYCOMB, as tarefas são executadas em um único thread para evitar erros comuns de aplicativos causados ​​pela execução paralela.

Se você realmente quer execução paralela, você pode invocar executeOnExecutor(java.util.concurrent.Executor, Object[]) com THREAD_POOL_EXECUTOR.

SERIAL_EXECUTOR -> Um Executor que executa uma tarefa de cada vez em ordem de série.

THREAD_POOL_EXECUTOR -> Um Executor que pode ser usado para executar tarefas em paralelo.

amostra :

Task task = new Task();
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.HONEYCOMB)
    task.executeOnExecutor(AsyncTask.SERIAL_EXECUTOR, data);
else
    task.execute(data);
