WebView é uma visualização que exibe páginas da Web dentro de seu aplicativo. Por isso, você pode adicionar seu próprio URL.

Por favor, não se esqueça de adicionar permissão no seu arquivo de manifesto do Android

<uses-permission android:name="android.permission.INTERNET" />

Solucionando problemas do WebView imprimindo mensagens do console ou por depuração remota #

Imprimindo mensagens do console do webview para logcat #

Para lidar com mensagens de console da página da web, você pode substituir onConsoleMessage em WebChromeClient:

final class ChromeClient extends WebChromeClient {
    @Override
    public boolean onConsoleMessage(ConsoleMessage msg) {
        Log.d(
            "WebView", 
            String.format("%s %s:%d", msg.message(), msg.lineNumber(), msg.sourceId())
        );
        return true;
    }
}

E defina-o em sua atividade ou fragmento:

webView.setWebChromeClient(new ChromeClient());

Então esta página de amostra:

<html>
<head>
    <script type="text/javascript">
        console.log('test message');
    </script>
</head>
<body>
</body>
</html>

escreverá a ‘mensagem de teste’ do log no logcat:

WebView: mensagem de teste sample.html:4

console.info(), console.warn() e console.error() também são suportados pelo chrome-client.

Depuração remota de dispositivos Android com Chrome #

Você pode depurar remotamente o aplicativo baseado em visualização da Web do seu Chrome desktop.

Ative a depuração USB no seu dispositivo Android #

No seu dispositivo Android, abra Configurações, encontre a seção Opções do desenvolvedor e ative a depuração USB.

Conecte-se e descubra seu dispositivo Android #

Abra a página no chrome a seguir: chrome://inspect/#devices

Na caixa de diálogo Inspecionar dispositivos, selecione seu dispositivo e pressione inspecionar. Uma nova instância do DevTools do Chrome é aberta em sua máquina de desenvolvimento.

Diretrizes e descrições mais detalhadas do DevTools podem ser encontradas em developers.google.com

Comunicação de Javascript para Java (Android) #

Atividade do Android

package com.example.myapp;

import android.os.Bundle;
import android.app.Activity;
import android.webkit.WebView;

public class WebViewActivity extends Activity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
    
        WebView webView = new WebView(this);
        setContentView(webView);

       /* 
        *   Note the label Android, this is used in the Javascript side of things
        *   You can of course change this.
        */
        webView.addJavascriptInterface(new JavascriptHandler(), "Android");
    
        webView.loadUrl("http://example.com");
    }
}

Comunicação de Java para Javascript #

Exemplo Básico

package com.example.myapp;

import android.os.Bundle;
import android.app.Activity;
import android.webkit.WebView;

public class WebViewActivity extends Activity {
    
    private Webview webView;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);

       webView = new WebView(this);
       webView.getSettings().setJavaScriptEnabled(true);

       setContentView(webView);

       webView.loadUrl("http://example.com");

       /*
        * Invoke Javascript function
        */
       webView.loadUrl("javascript:testJsFunction('Hello World!')");
    }

   /**
    * Invoking a Javascript function
    */
   public void doSomething() {
     this.webView.loadUrl("javascript:testAnotherFunction('Hello World Again!')");
   }
}

Diálogos de alerta JavaScript no WebView - Como fazê-los funcionar #

Por padrão, o WebView não implementa diálogos de alerta JavaScript, ou seja. alert() não fará nada. Para fazer você precisa primeiro habilitar o JavaScript (obviamente ..), e então definir um WebChromeClient para lidar com solicitações de diálogos de alerta da página:

webView.setWebChromeClient(new WebChromeClient() {
    //Other methods for your WebChromeClient here, if needed..

    @Override
    public boolean onJsAlert(WebView view, String url, String message, JsResult result) {
        return super.onJsAlert(view, url, message, result);
    }
});

Aqui, sobrescrevemos onJsAlert e, em seguida, chamamos a superimplementação, que nos fornece uma caixa de diálogo padrão do Android. Você também pode usar a mensagem e o URL, por exemplo, se quiser criar uma caixa de diálogo com estilo personalizado ou se quiser registrá-los.

Exemplo de discador aberto #

Se a página da Web contiver um número de telefone, você poderá fazer uma chamada usando o discador do seu telefone. Este código verifica a url que começa com tel: então faça uma intenção de abrir o discador e você pode fazer uma chamada para o número de telefone clicado:

public boolean shouldOverrideUrlLoading(WebView view, String url) {
    if (url.startsWith("tel:")) { 
            Intent intent = new Intent(Intent.ACTION_DIAL,
                    Uri.parse(url)); 
            startActivity(intent); 
    }else if(url.startsWith("http:") || url.startsWith("https:")) {
        view.loadUrl(url);
    }
    return true;
}
