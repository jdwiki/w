Um layout define a estrutura visual de uma interface de usuário, como uma atividade ou widget.

Um layout é declarado em XML, incluindo elementos de tela que aparecerão nele. O código pode ser adicionado ao aplicativo para modificar o estado dos objetos de tela em tempo de execução, incluindo aqueles declarados em XML.

Sintaxe #

  • android:gravity=“top|bottom|left|right|center_vertical|fill_vertical|center_horizontal|fill_horizontal|center|fill|clip_vertical|clip_horizontal|start|end”
  • android:layout_gravity=“top|bottom|left|right|center_vertical|fill_vertical|center_horizontal|fill_horizontal|center|fill|clip_vertical|clip_horizontal|start|end”

LayoutParams e Layout_ Attributes #

Impacto no desempenho do uso de RelativeLayouts perto do topo da hierarquia de visualização #

Conforme explicado em este artigo sobre desempenho no Android, um RelativeLayout requer duas passagens de layout para renderizar corretamente. Para hierarquias de exibição complexas, isso pode ter um impacto significativo no desempenho. Aninhar RelativeLayouts torna este problema ainda pior, porque cada RelativeLayout faz com que o número de passes de layout aumente.

Comportamento de rolagem do CoordinatorLayout #

Um CoordinatorLayout pode ser usado para obter Efeitos de rolagem de design de material ao usar layouts internos que suportam rolagem aninhada, como NestedScrollView ou RecyclerView.

Para este exemplo:

  • app:layout_scrollFlags="scroll|enterAlways" é usado na barra de ferramentas propriedades
  • app:layout_behavior="@string/appbar_scrolling_view_behavior" é usado em as propriedades do ViewPager

LayoutParams #

Cada ViewGroup (por exemplo, LinearLayout, RelativeLayout, CoordinatorLayout, etc.) precisa armazenar informações sobre as propriedades de seus filhos . Sobre a forma como seus filhos estão sendo dispostos no ViewGroup. Essas informações são armazenadas em objetos de uma classe wrapper ViewGroup.LayoutParams.

Para incluir parâmetros específicos para um tipo de layout específico, ViewGroups use subclasses da classe ViewGroup.LayoutParams.

Por exemplo. por

  • LinearLayout é LinearLayout.LayoutParams
  • RelativeLayout é RelativeLayout.LayoutParams
  • CoordinatorLayout é CoordinatorLayout.LayoutParams
  • …

A maioria dos ViewGroups reutiliza a capacidade de definir margins para seus filhos, então eles não subclassificam ViewGroup.LayoutParams diretamente, mas eles subclasse ViewGroup.MarginLayoutParams (que em si é uma subclasse de ViewGroup.LayoutParams).

LayoutParams em xml

Objetos LayoutParams são criados com base no arquivo inflado xml de layout.

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
              android:layout_width="match_parent"
              android:layout_height="match_parent"
              android:orientation="vertical">

    <TextView
        android:layout_width="wrap_content"
        android:layout_height="50dp"
        android:layout_gravity="right"
        android:gravity="bottom"
        android:text="Example text"
        android:textColor="@android:color/holo_green_dark"/>

    <ImageView
        android:layout_width="match_parent"
        android:layout_height="0dp"
        android:layout_weight="1"
        android:background="@android:color/holo_green_dark"
        android:scaleType="centerInside"
        android:src="@drawable/example"/>

</LinearLayout>

Todos os parâmetros que começam com layout_ especificam como o layout incluso deve funcionar. Quando o layout é inflado, esses parâmetros são agrupados em um objeto LayoutParams adequado, que mais tarde será usado pelo Layout para posicionar corretamente uma determinada View dentro do ViewGroup. Outros atributos de uma View são diretamente relacionados à View e são processados ​​pela própria View.

Para TextView:

  • layout_width, layout_height e layout_gravity serão armazenados em um objeto LinearLayout.LayoutParams e usados ​​pelo LinearLayout
  • gravity, text e textColor serão usados ​​pelo próprio TextView

Para ImageView:

  • layout_width, layout_height e layout_weight serão armazenados em um objeto LinearLayout.LayoutParams e usados ​​pelo LinearLayout
  • background, scaleType e src serão usados ​​pelo próprio ImageView

Obtendo o objeto LayoutParams

getLayoutParams é um método View's que permite recuperar um objeto LayoutParams atual.

Como o objeto LayoutParams está diretamente relacionado ao incluso ViewGroup, este método retornará um valor não nulo somente quando View for anexado ao ViewGroup. Você precisa ter em mente que esse objeto pode não estar presente o tempo todo. Especialmente você não deve depender de tê-lo dentro do construtor `View'.

public class ExampleView extends View {
    
    public ExampleView(Context context) {
        super(context);
        setupView(context);
    }

    public ExampleView(Context context, AttributeSet attrs) {
        super(context, attrs);
        setupView(context);
    }

    public ExampleView(Context context, AttributeSet attrs, int defStyle) {
        super(context, attrs, defStyle);
        setupView(context);
    }

    private void setupView(Context context) {
        if (getLayoutParams().height == 50){  // DO NOT DO THIS!
                                              // This might produce NullPointerException
            doSomething();
        }
    }
    
    //...
}

Se você quiser depender de ter o objeto LayoutParams, você deve usar o método onAttachedToWindow.

public class ExampleView extends View {

    public ExampleView(Context context) {
        super(context);
    }

    public ExampleView(Context context, AttributeSet attrs) {
        super(context, attrs);
    }

    public ExampleView(Context context, AttributeSet attrs, int defStyle) {
        super(context, attrs, defStyle);
    }

    @Override
    protected void onAttachedToWindow() {
        super.onAttachedToWindow();
        if (getLayoutParams().height == 50) { // getLayoutParams() will NOT return null here
            doSomething();
        }
    }

    //...
}

Transmitindo o objeto LayoutParams

Você pode precisar usar recursos específicos para um determinado ViewGroup (por exemplo, você pode querer alterar programaticamente as regras de um RelativeLayout). Para isso, você precisará saber como converter corretamente o objeto ViewGroup.LayoutParams.

Isso pode ser um pouco confuso ao obter um objeto LayoutParams para um View filho que na verdade é outro ViewGroup.

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
              android:id="@+id/outer_layout"
              android:layout_width="match_parent"
              android:layout_height="match_parent"
              android:orientation="vertical">

    <FrameLayout
        android:id="@+id/inner_layout"
        android:layout_width="match_parent"
        android:layout_height="50dp"
        android:layout_gravity="right"/>

</LinearLayout>

IMPORTANTE: O tipo de objeto LayoutParams está diretamente relacionado ao tipo do ENCLOSING ViewGroup.

Incorreto elenco:

FrameLayout innerLayout = (FrameLayout)findViewById(R.id.inner_layout);
FrameLayout.LayoutParams par = (FrameLayout.LayoutParams) innerLayout.getLayoutParams();
                                          // INCORRECT! This will produce ClassCastException

Correto elenco:

FrameLayout innerLayout = (FrameLayout)findViewById(R.id.inner_layout);
LinearLayout.LayoutParams par = (LinearLayout.LayoutParams) innerLayout.getLayoutParams();
                                         // CORRECT! the enclosing layout is a LinearLayout

Porcentagem de layouts #

A Biblioteca de suporte de porcentagem fornece PercentFrameLayout e PercentRelativeLayout, dois ViewGroups que fornecem uma maneira fácil de especificar dimensões e margens de visualização em termos de um * porcentagem* do tamanho total.

Você pode usar a Percent Support Library adicionando o seguinte às suas dependências.

compile 'com.android.support:percent:25.3.1'

Se você quiser exibir uma visualização que preencha a tela horizontalmente, mas apenas metade da tela verticalmente, faça o seguinte.

<android.support.percent.PercentFrameLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <FrameLayout
        app:layout_widthPercent="100%"
        app:layout_heightPercent="50%"
        android:background="@android:color/black" />

<android.support.percent.PercentFrameLayout>

Você também pode definir as porcentagens em um arquivo XML separado com código como:

<fraction name="margin_start_percent">25%</fraction>

E faça referência a eles em seus layouts com @fraction/margin_start_percent.

Eles também contêm a capacidade de definir uma proporção personalizada via app:layout_aspectRatio. Isso permite que você defina apenas uma única dimensão, como apenas a largura, e a altura será determinada automaticamente com base na proporção que você definiu, seja 4:3 ou 16:9 ou até mesmo um quadrado 1:1 proporção da tela.

Por exemplo:

 <ImageView
    app:layout_widthPercent="100%"
    app:layout_aspectRatio="178%"
    android:scaleType="centerCrop"
    android:src="@drawable/header_background"/>

Ver Peso #

Um dos atributos mais usados ​​para LinearLayout é o weight de suas visualizações filhas. O peso define quanto espaço uma visualização consumirá em comparação com outras visualizações em um LinearLayout.

O peso é usado quando você deseja dar espaço de tela específico a um componente em comparação com outro.

Propriedades da chave:

  • weightSum é a soma geral dos pesos de todas as visualizações filhas. Se você não especificar o weightSum, o sistema calculará a soma de todos os pesos por conta própria.

  • layout_weight especifica a quantidade de espaço da soma total do peso o widget irá ocupar.

Código:

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/activity_main"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="horizontal"
    android:weightSum="4">

    <EditText
        android:layout_weight="2"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        android:text="Type Your Text Here" />

    <Button
        android:layout_weight="1"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        android:text="Text1" />

    <Button
        android:layout_weight="1"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        android:text="Text1" />

</LinearLayout>

A saída é:

Agora mesmo que o tamanho do dispositivo seja maior, o EditText vai ocupar 2/4 do espaço da tela. Portanto, a aparência do seu aplicativo é consistente em todas as telas.

Observação: Aqui o layout_width é mantido como 0dp pois o espaço do widget é dividido horizontalmente. Se os widgets devem ser alinhados verticalmente, layout_height será definido como 0dp. Isso é feito para aumentar a eficiência do código porque em tempo de execução o sistema não tentará calcular a largura ou a altura respectivamente, pois isso é gerenciado pelo peso. Se você usasse wrap_content o sistema tentaria calcular a largura/altura primeiro antes de aplicar o atributo de peso que causa outro ciclo de cálculo.

Esquema relativo #

RelativeLayout é um ViewGroup que exibe visualizações filhas em posições relativas. Por padrão, todas as visualizações filhas são desenhadas no canto superior esquerdo do layout, portanto, você deve definir a posição de cada visualização usando as várias propriedades de layout disponíveis em RelativeLayout.LayoutParams. O valor para cada propriedade de layout é um booleano para habilitar uma posição de layout em relação ao RelativeLayout pai ou um ID que faz referência a outra exibição no layout em relação à qual a exibição deve ser posicionada.

Exemplo:

<?xml version="1.0" encoding="utf-8"?>
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <ImageView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:id="@+id/imageView"
        android:src="@mipmap/ic_launcher" />

    <EditText
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:id="@+id/editText"
        android:layout_toRightOf="@+id/imageView"
        android:layout_toEndOf="@+id/imageView"
        android:hint="@string/hint" />

</RelativeLayout>

Layout de Grade #

GridLayout, como o nome sugere, é um layout usado para organizar Views em uma grade. Um GridLayout se divide em colunas e linhas. Como você pode ver no exemplo abaixo, a quantidade de colunas e/ou linhas é especificada pelas propriedades columnCount e rowCount. Adicionar visualizações a este layout adicionará a primeira visualização à primeira coluna, a segunda visualização à segunda coluna e a terceira visualização à primeira coluna da segunda linha.

<?xml version="1.0" encoding="utf-8"?>
<GridLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:paddingBottom="@dimen/activity_vertical_margin"
    android:paddingLeft="@dimen/activity_horizontal_margin"
    android:paddingRight="@dimen/activity_horizontal_margin"
    android:paddingTop="@dimen/activity_vertical_margin"
    android:columnCount="2"
    android:rowCount="2">

    <TextView
        android:layout_width="@dimen/fixed"
        android:layout_height="wrap_content"
        android:text="@string/first"
        android:background="@color/colorPrimary"
        android:layout_margin="@dimen/default_margin" />

    <TextView
        android:layout_width="@dimen/fixed"
        android:layout_height="wrap_content"
        android:text="@string/second"
        android:background="@color/colorPrimary"
        android:layout_margin="@dimen/default_margin" />

    <TextView
        android:layout_width="@dimen/fixed"
        android:layout_height="wrap_content"
        android:text="@string/third"
        android:background="@color/colorPrimary"
        android:layout_margin="@dimen/default_margin" />

</GridLayout>

Layout de quadro #

FrameLayout foi projetado para bloquear uma área na tela para exibir um único item. Você pode, no entanto, adicionar vários filhos a um FrameLayout e controlar sua posição dentro do FrameLayout atribuindo gravidade a cada filho, usando o atributo android:layout_gravity.

Geralmente, FrameLayout é usado para manter uma única visão filha. Casos de uso comuns são criar espaços reservados para inflar Fragments em Activity, sobrepor visualizações ou aplicar primeiro plano às visualizações.

Exemplo:

<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
android:layout_width="match_parent"
android:layout_height="match_parent">

    <ImageView
        android:src="@drawable/nougat"
        android:scaleType="fitCenter"
        android:layout_height="match_parent"
        android:layout_width="match_parent"/>

    <TextView
        android:text="FrameLayout Example"
        android:textSize="30sp"
        android:textStyle="bold"
        android:layout_height="match_parent"
        android:layout_width="match_parent"
        android:gravity="center"/>

</FrameLayout>

Layout do Coordenador #

O CoordinatorLayout é um contêiner um pouco semelhante ao FrameLayout, mas com recursos extras, é chamado de FrameLayout superpoderoso na documentação oficial.

Ao anexar um CoordinatorLayout.Behavior a um filho direto de CoordinatorLayout, você poderá interceptar eventos de toque, inserções de janela, medição, layout e rolagem aninhada.

Para usá-lo, primeiro você terá que adicionar uma dependência para a biblioteca de suporte em seu arquivo gradle:

Um caso de uso prático do CoordinatorLayout é criar uma visão com um FloatingActionButton. Neste caso específico, vamos criar um RecyclerView com um SwipeRefreshLayout e um FloatingActionButton em cima disso. Veja como você pode fazer isso:

<?xml version="1.0" encoding="utf-8"?>
<android.support.design.widget.CoordinatorLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:id="@+id/coord_layout"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="horizontal">

    <android.support.v4.widget.SwipeRefreshLayout
        android:id="@+id/swipe_refresh_layout"
        android:layout_width="match_parent"
        android:layout_height="match_parent">

        <android.support.v7.widget.RecyclerView
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:id="@+id/recycler_view"/>

    </android.support.v4.widget.SwipeRefreshLayout>

    <android.support.design.widget.FloatingActionButton
        android:id="@+id/fab"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_margin="16dp"
        android:clickable="true"
        android:color="@color/colorAccent"
        android:src="@mipmap/ic_add_white"
        android:layout_gravity="end|bottom"
        app:layout_anchorGravity="bottom|right|end"/>

</android.support.design.widget.CoordinatorLayout>

Observe como o FloatingActionButton é ancorado ao CoordinatorLayout com app:layout_anchor="@id/coord_layout"
