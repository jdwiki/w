Uma folha inferior é uma folha que desliza para cima a partir da borda inferior da tela.

Folhas inferiores deslize para cima da parte inferior da tela para revelar mais conteúdo. Eles foram adicionados à Biblioteca de Suporte do Android na versão v23.2.0.

BottomSheetBehavior como o Google Maps #

Este exemplo depende da Biblioteca de Suporte 23.4.0.+.

BottomSheetBehavior é caracterizado por:

  1. Duas barras de ferramentas com animações que respondem aos movimentos da folha inferior.
  2. Um FAB que se esconde quando está perto da “barra de ferramentas modal” (aquela que aparece quando você está deslizando para cima).
  3. Uma imagem de fundo atrás da folha inferior com algum tipo de efeito de paralaxe.
  4. Um título (TextView) na barra de ferramentas que aparece quando a folha inferior o alcança.
  5. A barra de status de notificação pode tornar seu plano de fundo transparente ou colorido.
  6. Um comportamento de página inferior personalizado com um estado “âncora”.

Agora vamos verificá-los um por um:

Barras de ferramentas
Quando você abre essa visualização no Google Maps, você pode ver uma barra de ferramentas onde você pode pesquisar, é a única que eu não estou fazendo exatamente como o Google Maps, porque eu queria fazer mais genérico. De qualquer forma, o ToolBar está dentro de um AppBarLayout e ficou oculto quando você começa a arrastar o BottomSheet e aparece novamente quando o BottomSheet atinge o estado COLLAPSED.
Para alcançá-lo, você precisa:

  • crie um Behavior e estenda-o de AppBarLayout.ScrollingViewBehavior
  • substituir os métodos layoutDependsOn e onDependentViewChanged. Fazendo isso, você ouvirá os movimentos do bottomSheet.
  • crie alguns métodos para ocultar e exibir o AppBarLayout/ToolBar com animações.

Foi assim que fiz para a primeira barra de ferramentas ou ActionBar:

@Override
public boolean layoutDependsOn(CoordinatorLayout parent, View child, View dependency) {
    return dependency instanceof NestedScrollView;
}

@Override
public boolean onDependentViewChanged(CoordinatorLayout parent, View child,
                                      View dependency) {

    if (mChild == null) {
        initValues(child, dependency);
        return false;
    }

    float dVerticalScroll = dependency.getY() - mPreviousY;
    mPreviousY = dependency.getY();

    //going up
    if (dVerticalScroll <= 0 && !hidden) {
        dismissAppBar(child);
        return true;
    }

    return false;
}

private void initValues(final View child, View dependency) {

    mChild = child;
    mInitialY = child.getY();

    BottomSheetBehaviorGoogleMapsLike bottomSheetBehavior = BottomSheetBehaviorGoogleMapsLike.from(dependency);
    bottomSheetBehavior.addBottomSheetCallback(new BottomSheetBehaviorGoogleMapsLike.BottomSheetCallback() {
        @Override
        public void onStateChanged(@NonNull View bottomSheet, @BottomSheetBehaviorGoogleMapsLike.State int newState) {
            if (newState == BottomSheetBehaviorGoogleMapsLike.STATE_COLLAPSED ||
                    newState == BottomSheetBehaviorGoogleMapsLike.STATE_HIDDEN)
                showAppBar(child);
        }

        @Override
        public void onSlide(@NonNull View bottomSheet, float slideOffset) {

        }
    });
}

private void dismissAppBar(View child){
    hidden = true;
    AppBarLayout appBarLayout = (AppBarLayout)child;
    mToolbarAnimation = appBarLayout.animate().setDuration(mContext.getResources().getInteger(android.R.integer.config_shortAnimTime));
    mToolbarAnimation.y(-(mChild.getHeight()+25)).start();
}

private void showAppBar(View child) {
    hidden = false;
    AppBarLayout appBarLayout = (AppBarLayout)child;
    mToolbarAnimation = appBarLayout.animate().setDuration(mContext.getResources().getInteger(android.R.integer.config_mediumAnimTime));
    mToolbarAnimation.y(mInitialY).start();
}

O FAB

Este é um comportamento personalizado também, mas se estende de FloatingActionButton.Behavior. Em onDependentViewChanged você deve olhar quando atingir o “offSet” ou apontar para onde deseja escondê-lo. No meu caso eu quero escondê-lo quando estiver perto da segunda barra de ferramentas, então eu cavo no pai FAB (um CoordinatorLayout) procurando o AppBarLayout que contém o ToolBar, então eu uso a posição ToolBar como OffSet:

@Override
public boolean onDependentViewChanged(CoordinatorLayout parent, FloatingActionButton child, View dependency) {

    if (offset == 0)
        setOffsetValue(parent);

    if (dependency.getY() <=0)
        return false;

    if (child.getY() <= (offset + child.getHeight()) && child.getVisibility() == View.VISIBLE)
        child.hide();
    else if (child.getY() > offset && child.getVisibility() != View.VISIBLE)
        child.show();

    return false;
}

A imagem por trás do BottomSheet com efeito de paralaxe:
Como os outros, é um comportamento personalizado, a única coisa “complicada” neste é o pequeno algoritmo que mantém a imagem ancorada no BottomSheet e evita o colapso da imagem como efeito de paralaxe padrão:

@Override
public boolean onDependentViewChanged(CoordinatorLayout parent, View child,
                                      View dependency) {

    if (mYmultiplier == 0) {
        initValues(child, dependency);
        return true;
    }

    float dVerticalScroll = dependency.getY() - mPreviousY;
    mPreviousY = dependency.getY();

    //going up
    if (dVerticalScroll <= 0 && child.getY() <= 0) {
        child.setY(0);
        return true;
    }

    //going down
    if (dVerticalScroll >= 0 && dependency.getY() <= mImageHeight)
        return false;

    child.setY( (int)(child.getY() + (dVerticalScroll * mYmultiplier) ) );

    return true;
}

Agora para o final: O comportamento personalizado do BottomSheet
Para realizar as 3 etapas, primeiro você precisa entender que o BottomSheetBehavior padrão tem 5 estados: STATE_DRAGGING, STATE_SETTLING, STATE_EXPANDED, STATE_COLLAPSED, STATE_HIDDEN e para o comportamento do Google Maps você precisa adicionar um estado intermediário entre recolhido e expandido: STATE_ANCHOR_POINT .
Tentei estender o bottomSheetBehavior padrão sem sucesso, então apenas copiei e colei todo o código e modifiquei o que preciso.
Para alcançar o que estou falando, siga os próximos passos:

  1. Crie uma classe Java e estenda-a de CoordinatorLayout.Behavior<V>

  2. Copie e cole o código do arquivo padrão BottomSheetBehavior para o seu novo arquivo.

  3. Modifique o método clampViewPositionVertical com o seguinte código:

     @Override
 public int clampViewPositionVertical(View child, int top, int dy) {
     return constrain(top, mMinOffset, mHideable ? mParentHeight : mMaxOffset);
 }
 int constrain(int amount, int low, int high) {
     return amount < low ? low : (amount > high ? high : amount);
 }

  4. Adicione um novo estado

    public static final int STATE_ANCHOR_POINT = X;

  5. Modifique os próximos métodos: onLayoutChild, onStopNestedScroll, BottomSheetBehavior<V> from(V view) e setState (opcional)



public boolean onLayoutChild(CoordinatorLayout parent, V child, int layoutDirection) {
    // First let the parent lay it out
    if (mState != STATE_DRAGGING && mState != STATE_SETTLING) {
        if (ViewCompat.getFitsSystemWindows(parent) &&
                !ViewCompat.getFitsSystemWindows(child)) {
            ViewCompat.setFitsSystemWindows(child, true);
        }
        parent.onLayoutChild(child, layoutDirection);
    }
    // Offset the bottom sheet
    mParentHeight = parent.getHeight();
    mMinOffset = Math.max(0, mParentHeight - child.getHeight());
    mMaxOffset = Math.max(mParentHeight - mPeekHeight, mMinOffset);

    //if (mState == STATE_EXPANDED) {
    //    ViewCompat.offsetTopAndBottom(child, mMinOffset);
    //} else if (mHideable && mState == STATE_HIDDEN...
    if (mState == STATE_ANCHOR_POINT) {
        ViewCompat.offsetTopAndBottom(child, mAnchorPoint);
    } else if (mState == STATE_EXPANDED) {
        ViewCompat.offsetTopAndBottom(child, mMinOffset);
    } else if (mHideable && mState == STATE_HIDDEN) {
        ViewCompat.offsetTopAndBottom(child, mParentHeight);
    } else if (mState == STATE_COLLAPSED) {
        ViewCompat.offsetTopAndBottom(child, mMaxOffset);
    }
    if (mViewDragHelper == null) {
        mViewDragHelper = ViewDragHelper.create(parent, mDragCallback);
    }
    mViewRef = new WeakReference<>(child);
    mNestedScrollingChildRef = new WeakReference<>(findScrollingChild(child));
    return true;
}


public void onStopNestedScroll(CoordinatorLayout coordinatorLayout, V child, View target) {
    if (child.getTop() == mMinOffset) {
        setStateInternal(STATE_EXPANDED);
        return;
    }
    if (target != mNestedScrollingChildRef.get() || !mNestedScrolled) {
        return;
    }
    int top;
    int targetState;
    if (mLastNestedScrollDy > 0) {
        //top = mMinOffset;
        //targetState = STATE_EXPANDED;
        int currentTop = child.getTop();
        if (currentTop > mAnchorPoint) {
            top = mAnchorPoint;
            targetState = STATE_ANCHOR_POINT;
        }
        else {
            top = mMinOffset;
            targetState = STATE_EXPANDED;
        }
    } else if (mHideable && shouldHide(child, getYVelocity())) {
        top = mParentHeight;
        targetState = STATE_HIDDEN;
    } else if (mLastNestedScrollDy == 0) {
        int currentTop = child.getTop();
        if (Math.abs(currentTop - mMinOffset) < Math.abs(currentTop - mMaxOffset)) {
            top = mMinOffset;
            targetState = STATE_EXPANDED;
        } else {
            top = mMaxOffset;
            targetState = STATE_COLLAPSED;
        }
    } else {
        //top = mMaxOffset;
        //targetState = STATE_COLLAPSED;
        int currentTop = child.getTop();
        if (currentTop > mAnchorPoint) {
            top = mMaxOffset;
            targetState = STATE_COLLAPSED;
        }
        else {
            top = mAnchorPoint;
            targetState = STATE_ANCHOR_POINT;
        }
    }
    if (mViewDragHelper.smoothSlideViewTo(child, child.getLeft(), top)) {
        setStateInternal(STATE_SETTLING);
        ViewCompat.postOnAnimation(child, new SettleRunnable(child, targetState));
    } else {
        setStateInternal(targetState);
    }
    mNestedScrolled = false;
}

public final void setState(@State int state) {
    if (state == mState) {
        return;
    }
    if (mViewRef == null) {
        // The view is not laid out yet; modify mState and let onLayoutChild handle it later
        /**
         * New behavior (added: state == STATE_ANCHOR_POINT ||)
         */
        if (state == STATE_COLLAPSED || state == STATE_EXPANDED ||
                state == STATE_ANCHOR_POINT ||
                (mHideable && state == STATE_HIDDEN)) {
            mState = state;
        }
        return;
    }
    V child = mViewRef.get();
    if (child == null) {
        return;
    }
    int top;
    if (state == STATE_COLLAPSED) {
        top = mMaxOffset;
    } else if (state == STATE_ANCHOR_POINT) {
        top = mAnchorPoint;
    } else if (state == STATE_EXPANDED) {
        top = mMinOffset;
    } else if (mHideable && state == STATE_HIDDEN) {
        top = mParentHeight;
    } else {
        throw new IllegalArgumentException("Illegal state argument: " + state);
    }
    setStateInternal(STATE_SETTLING);
    if (mViewDragHelper.smoothSlideViewTo(child, child.getLeft(), top)) {
        ViewCompat.postOnAnimation(child, new SettleRunnable(child, state));
    }
}


public static <V extends View> BottomSheetBehaviorGoogleMapsLike<V> from(V view) {
    ViewGroup.LayoutParams params = view.getLayoutParams();
    if (!(params instanceof CoordinatorLayout.LayoutParams)) {
        throw new IllegalArgumentException("The view is not a child of CoordinatorLayout");
    }
    CoordinatorLayout.Behavior behavior = ((CoordinatorLayout.LayoutParams) params)
            .getBehavior();
    if (!(behavior instanceof BottomSheetBehaviorGoogleMapsLike)) {
        throw new IllegalArgumentException(
                "The view is not associated with BottomSheetBehaviorGoogleMapsLike");
    }
    return (BottomSheetBehaviorGoogleMapsLike<V>) behavior;
}

Folhas inferiores modais com BottomSheetDialogFragment #

Você pode realizar uma planilha inferior modal usando um BottomSheetDialogFragment.

O BottomSheetDialogFragment é uma folha inferior modal. Esta é uma versão de DialogFragment que mostra uma folha inferior usando BottomSheetDialog em vez de uma caixa de diálogo flutuante.

Basta definir o fragmento:

public class MyBottomSheetDialogFragment extends BottomSheetDialogFragment {

    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container,
            Bundle savedInstanceState) {
        return inflater.inflate(R.layout.my_fragment_bottom_sheet, container);
    }
}

Em seguida, use este código para mostrar o fragmento:

MyBottomSheetDialogFragment mySheetDialog = new MyBottomSheetDialogFragment();
FragmentManager fm = getSupportFragmentManager();
mySheetDialog.show(fm, "modalSheetDialog");

Este fragmento criará um BottomSheetDialog.

Folhas Inferiores Persistentes #

Você pode obter uma Persistent Bottom Sheet anexando um BottomSheetBehavior a uma View filha de um CoordinatorLayout:

<android.support.design.widget.CoordinatorLayout >

    <!-- .....   -->

    <LinearLayout
       android:id="@+id/bottom_sheet"
       android:elevation="4dp"
       android:minHeight="120dp"
       app:behavior_peekHeight="120dp"
       ...
       app:layout_behavior="android.support.design.widget.BottomSheetBehavior">

           <!-- .....   -->

       </LinearLayout>

</android.support.design.widget.CoordinatorLayout>

Então, no seu código, você pode criar uma referência usando:

 // The View with the BottomSheetBehavior  
 View bottomSheet = coordinatorLayout.findViewById(R.id.bottom_sheet);  
 BottomSheetBehavior mBottomSheetBehavior = BottomSheetBehavior.from(bottomSheet);

Você pode definir o estado do seu BottomSheetBehavior usando o método setState():

mBottomSheetBehavior.setState(BottomSheetBehavior.STATE_EXPANDED);

Você pode usar um destes estados:

  • STATE_COLLAPSED: este estado recolhido é o padrão e mostra apenas uma parte do layout na parte inferior. A altura pode ser controlada com o atributo app:behavior_peekHeight (o padrão é 0)

  • STATE_EXPANDED: o estado totalmente expandido da planilha inferior, onde toda a planilha inferior é visível (se sua altura for menor que o CoordinatorLayout) ou todo o CoordinatorLayout é preenchido

  • STATE_HIDDEN: desativado por padrão (e ativado com o atributo app:behavior_hideable), permitindo que os usuários deslizem para baixo na página inferior para ocultar completamente a página inferior

Se você quiser receber retornos de chamada de mudanças de estado, você pode adicionar um BottomSheetCallback:

mBottomSheetBehavior.setBottomSheetCallback(new BottomSheetCallback() {  
    @Override  
    public void onStateChanged(@NonNull View bottomSheet, int newState) {  
      // React to state change  
    }  
      @Override  
      public void onSlide(@NonNull View bottomSheet, float slideOffset) {  
       // React to dragging events  
   }  
 });

Abra BottomSheet DialogFragment no modo Expandido por padrão. #

BottomSheet DialogFragment abre em STATE_COLLAPSED por padrão. Que pode ser forçado a abrir em STATE_EXPANDED e ocupar toda a tela do dispositivo com a ajuda do seguinte modelo de código.

@NonNull @Override public Dialog onCreateDialog(Bundle savedInstanceState) {

    BottomSheetDialog dialog = (BottomSheetDialog) super.onCreateDialog(savedInstanceState);

    dialog.setOnShowListener(new DialogInterface.OnShowListener() {
        @Override
        public void onShow(DialogInterface dialog) {
            BottomSheetDialog d = (BottomSheetDialog) dialog;

            FrameLayout bottomSheet = (FrameLayout) d.findViewById(android.support.design.R.id.design_bottom_sheet);
            BottomSheetBehavior.from(bottomSheet).setState(BottomSheetBehavior.STATE_EXPANDED);
        }
    });

    // Do something with your dialog like setContentView() or whatever
    return dialog;
}

Embora a animação de diálogo seja um pouco perceptível, mas faz a tarefa de abrir o DialogFragment em tela cheia muito bem.
