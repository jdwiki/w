Adicionando uma barra de ferramentas #

Uma Toolbar é uma generalização de ActionBar para uso em layouts de aplicativos. Enquanto uma ActionBar é tradicionalmente parte de uma decoração de janela opaca de Activity controlada pelo framework, uma Toolbar pode ser colocada em qualquer nível arbitrário de aninhamento dentro de uma hierarquia de visualização. Ele pode ser adicionado executando as seguintes etapas:

  1. Certifique-se de que a seguinte dependência seja adicionada ao arquivo build.gradle do seu módulo (por exemplo, do aplicativo) em dependências:

    compile 'com.android.support:appcompat-v7:25.3.1'

  2. Defina o tema do seu aplicativo para um que não tenha um ActionBar. Para fazer isso, edite seu arquivo styles.xml em res/values e defina um tema Theme.AppCompat. Neste exemplo, estamos usando Theme.AppCompat.NoActionBar como pai do seu AppTheme:

    <style name="AppTheme" parent="Theme.AppCompat.NoActionBar">
    <item name="colorPrimary">@color/primary</item>
    <item name="colorPrimaryDark">@color/primaryDark</item>
    <item name="colorAccent">@color/accent</item>
</style>

Você também pode usar Theme.AppCompat.Light.NoActionBar ou Theme.AppCompat.DayNight.NoActionBar, ou qualquer outro tema que não tenha inerentemente um ActionBar

  1. Adicione a Barra de ferramentas ao layout de sua atividade:

    <android.support.v7.widget.Toolbar
    android:id="@+id/toolbar"
    android:layout_width="match_parent"
    android:layout_height="?attr/actionBarSize"
    android:background="?attr/colorPrimary"
    android:elevation="4dp"/>

Abaixo da Barra de ferramentas você pode adicionar o resto do seu layout.

  1. Em sua Activity, defina a Toolbar como a ActionBar para esta Activity. Desde que você esteja usando a appcompat library e uma AppCompatActivity, você usaria o método setSupportActionBar():

    @Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    final Toolbar toolbar = (Toolbar) findViewById(R.id.toolbar);
    setSupportActionBar(toolbar);

    //...
}

Após realizar os passos acima, você pode usar o método getSupportActionBar() para manipular a Toolbar que está definida como ActionBar.

Por exemplo, você pode definir o título conforme mostrado abaixo:

getSupportActionBar().setTitle("Activity Title");

Por exemplo, você também pode definir a cor do título e do plano de fundo conforme mostrado abaixo:

CharSequence title = "Your App Name";
SpannableString s = new SpannableString(title);
s.setSpan(new ForegroundColorSpan(Color.RED), 0, title.length(), Spannable.SPAN_EXCLUSIVE_EXCLUSIVE);
getSupportActionBar().setTitle(s);
getSupportActionBar().setBackgroundDrawable(new ColorDrawable(Color.argb(128, 0, 0, 0)));

Botões estilizados com Material Design #

A AppCompat Support Library define vários estilos úteis para Buttons, cada um dos quais estende um estilo base Widget.AppCompat.Button que é aplicado a todos os botões por padrão se você estiver usando um AppCompat tema. Esse estilo ajuda a garantir que todos os botões tenham a mesma aparência por padrão, seguindo a especificação do Material Design.

Neste caso, a cor de destaque é rosa.

  1. Botão Simples: @style/Widget.AppCompat.Button

    <Button
    style="@style/Widget.AppCompat.Button"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:layout_margin="16dp" 
    android:text="@string/simple_button"/>

  2. Botão colorido: @style/Widget.AppCompat.Button.Colored O estilo Widget.AppCompat.Button.Colored estende o estilo Widget.AppCompat.Button e aplica automaticamente a cor de destaque selecionada no tema do seu aplicativo.

   <Button
       style="@style/Widget.AppCompat.Button.Colored"
       android:layout_width="match_parent"
       android:layout_height="wrap_content"
       android:layout_margin="16dp" 
       android:text="@string/colored_button"/>

Se você quiser personalizar a cor de fundo sem alterar a cor de destaque em seu tema principal, você pode criar um tema personalizado (estendendo o tema ThemeOverlay) para o seu Button e atribuí-lo ao botão android: theme atributo:

   <Button  
        style="@style/Widget.AppCompat.Button.Colored"  
        android:layout_width="wrap_content"  
        android:layout_height="wrap_content" 
        android:layout_margin="16dp"
        android:theme="@style/MyButtonTheme"/> 

Define the theme in `res/values/themes.xml`:

    <style name="MyButtonTheme" parent="ThemeOverlay.AppCompat.Light"> 
         <item name="colorAccent">@color/my_color</item> 
    </style>
  1. Botão sem borda: @style/Widget.AppCompat.Button.Borderless

   <Button
       style="@style/Widget.AppCompat.Button.Borderless"
       android:layout_width="match_parent"
       android:layout_height="wrap_content"
       android:layout_margin="16dp" 
       android:text="@string/borderless_button"/>
  1. Botão colorido sem borda: @style/Widget.AppCompat.Button.Borderless.Colored

   <Button
       style="@style/Widget.AppCompat.Button.Borderless.Colored"
       android:layout_width="match_parent"
       android:layout_height="wrap_content"
       android:layout_margin="16dp" 
       android:text="@string/borderless_colored_button"/>

Adicionando um FloatingActionButton (FAB) #

No material design, um botão de ação flutuante representa a ação principal em uma atividade. Eles são diferenciados por um ícone circulado flutuando acima da interface do usuário e têm comportamentos de movimento que incluem morphing, lançamento e um ponto de ancoragem de transferência.

Certifique-se de que a seguinte dependência seja adicionada ao arquivo build.gradle do seu aplicativo em dependências:

compile 'com.android.support:design:25.3.1'

Agora adicione o FloatingActionButton ao seu arquivo de layout:

<android.support.design.widget.FloatingActionButton
      android:layout_width="wrap_content"
      android:layout_height="wrap_content"
      android:layout_margin="16dp"
      android:src="@drawable/some_icon"/>

onde o atributo src faz referência ao ícone que deve ser usado para a ação flutuante. O resultado deve ser algo assim (presumindo que sua cor de destaque seja Rosa Material):

Por padrão, a cor de fundo do seu FloatingActionButton será definida para a cor de destaque do seu tema. Além disso, observe que um FloatingActionButton requer uma margem em torno dele para funcionar corretamente. A margem recomendada para a parte inferior é 16dp para telefones e 24dp para tablets.

Aqui estão as propriedades que você pode usar para personalizar ainda mais o FloatingActionButton (assumindo que xmlns:app="http://schemas.android.com/apk/res-auto é declarado como namespace no topo do seu layout):

  • app:fabSize: Pode ser definido como normal ou mini para alternar entre um tamanho normal ou uma versão menor.
  • app:rippleColor: Define a cor do efeito cascata do seu FloatingActionButton. Pode ser um recurso de cor ou uma string hexadecimal.
  • app:elevation: Pode ser uma string, inteiro, booleano, valor de cor, ponto flutuante, valor de dimensão.
  • app:useCompatPadding: Habilita o preenchimento de compatibilidade. Talvez um valor booleano, como true ou false. Defina como true para usar o preenchimento de compatibilidade na api-21 e posterior, a fim de manter uma aparência consistente com os níveis de API mais antigos.

Você pode encontrar mais exemplos sobre a FAB aqui.

RippleDrawable #

O efeito Ripple touch foi introduzido com o material design no Android 5.0 (API de nível 21) e a animação é implementada pela nova classe RippleDrawable.

Drawable que mostra um efeito cascata em resposta a mudanças de estado. A posição de ancoragem da ondulação para um determinado estado pode ser especificada chamando setHotspot(float x, float y) com o identificador de atributo de estado correspondente.

Em geral, o efeito cascata para botões regulares funciona por padrão na API 21 e superior, e para outras visualizações tocáveis, pode ser obtido especificando:

android:background="?android:attr/selectableItemBackground">

para ondulações contidas na visualização ou:

android:background="?android:attr/selectableItemBackgroundBorderless"

para ondulações que se estendem além dos limites da visão.

Por exemplo, na imagem abaixo,

  • B1 é um botão que não tem fundo,
  • B2 é configurado com android:background="android:attr/selectableItemBackground"
  • B3 é configurado com android:background="android:attr/selectableItemBackgroundBorderless"

(Cortesia da imagem: http://blog.csdn.net/a396901990/article/details/40187203 )

Você pode conseguir o mesmo no código usando:

int[] attrs = new int[]{R.attr.selectableItemBackground};
TypedArray typedArray = getActivity().obtainStyledAttributes(attrs);
int backgroundResource = typedArray.getResourceId(0, 0);
myView.setBackgroundResource(backgroundResource);

Ripples também podem ser adicionados a uma visualização usando o atributo android:foreground da mesma forma que acima. Como o nome sugere, caso a ondulação seja adicionada ao primeiro plano, a ondulação aparecerá acima de qualquer visualização à qual for adicionada (por exemplo, ImageView, um LinearLayout contendo várias visualizações, etc).

Se você quiser personalizar o efeito cascata em uma visualização, você precisa criar um novo arquivo XML, dentro do diretório drawable.

Aqui estão alguns exemplos:

Exemplo 1: uma ondulação ilimitada

<ripple xmlns:android="http://schemas.android.com/apk/res/android"
    android:color="#ffff0000" />

Exemplo 2: Ondulação com máscara e cor de fundo

<ripple android:color="#7777777"
    xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:id="@android:id/mask"
        android:drawable="#ffff00" />
    <item android:drawable="@android:color/white"/>
</ripple>

Se houver view com um plano de fundo já especificado com uma shape, corners e quaisquer outras tags, para adicionar uma ondulação a essa visão, use uma mask layer e defina a ondulação como plano de fundo da visão .

Exemplo:

 <?xml version="1.0" encoding="utf-8"?>
    <ripple xmlns:android="http://schemas.android.com/apk/res/android"
        android:color="?android:attr/colorControlHighlight">
        <item android:id="@android:id/mask">
           <shape 
              android:shape="rectangle">
              solid android:color="#000000"/>
               <corners
                 android:radius="25dp"/>
            </shape>
        </item>
        <item android:drawable="@drawable/rounded_corners" />
    </ripple>

Exemplo 3: ondular em cima de um recurso desenhável

<ripple xmlns:android="http://schemas.android.com/apk/res/android"
    android:color="#ff0000ff">
    <item android:drawable="@drawable/my_drawable" />
</ripple>

Uso: Para anexar seu arquivo xml ripple a qualquer visualização, defina-o como plano de fundo da seguinte forma (supondo que seu arquivo ripple seja denominado my_ripple.xml):

<View 
    android:id="@+id/myViewId"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:background="@drawable/my_ripple" />

Seletor:

O drawable ripple também pode ser usado no lugar de seletores de lista de estado de cor se sua versão de destino for v21 ou superior (você também pode colocar o seletor ripple na pasta drawable-v21):

<!-- /drawable/button.xml: -->
<selector xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:state_pressed="true" android:drawable="@drawable/button_pressed"/>
    <item android:drawable="@drawable/button_normal"/>
</selector>

<!--/drawable-v21/button.xml:-->
<?xml version="1.0" encoding="utf-8"?>
<ripple xmlns:android="http://schemas.android.com/apk/res/android"
    android:color="?android:colorControlHighlight">
   <item android:drawable="@drawable/button_normal" />
</ripple>

Nesse caso, a cor do estado padrão de sua visualização seria branca e o estado pressionado mostraria o drawable de ondulação.

Ponto a ser observado: Usar ?android:colorControlHighlight dará à ondulação a mesma cor que as ondulação integradas em seu aplicativo.

Para alterar apenas a cor da ondulação, você pode personalizar a cor android:colorControlHighlight em seu tema da seguinte forma:

<?xml version="1.0" encoding="utf-8"?>
<resources>

  <style name="AppTheme" parent="android:Theme.Material.Light.DarkActionBar">
    <item name="android:colorControlHighlight">@color/your_custom_color</item>
  </style>

</resources>

e depois use este tema em suas atividades, etc. O efeito seria como na imagem abaixo:

Adicionando um TabLayout #

TabLayout fornece um layout horizontal para exibir guias e é comumente usado em conjunto com um ViewPager.

Certifique-se de que a seguinte dependência seja adicionada ao arquivo build.gradle do seu aplicativo em dependências:

compile 'com.android.support:design:25.3.1'

Agora você pode adicionar itens a um TabLayout em seu layout usando a classe TabItem.

Por exemplo:

<android.support.design.widget.TabLayout
    android:layout_height="wrap_content"
    android:layout_width="match_parent"
    android:id="@+id/tabLayout">

    <android.support.design.widget.TabItem
        android:text="@string/tab_text_1"
        android:icon="@drawable/ic_tab_1"/>

    <android.support.design.widget.TabItem
        android:text="@string/tab_text_2"
        android:icon="@drawable/ic_tab_2"/>

</android.support.design.widget.TabLayout>

Adicione um OnTabSelectedListener para ser notificado quando uma guia no TabLayout for selecionada/desmarcada/reselecionada:

TabLayout tabLayout = (TabLayout) findViewById(R.id.tabLayout);
tabLayout.addOnTabSelectedListener(new TabLayout.OnTabSelectedListener() {
    @Override
    public void onTabSelected(TabLayout.Tab tab) {
        int position = tab.getPosition();
        // Switch to view for this tab
    }

    @Override
    public void onTabUnselected(TabLayout.Tab tab) {

    }

    @Override
    public void onTabReselected(TabLayout.Tab tab) {

    }
});

As guias também podem ser adicionadas/removidas do TabLayout programaticamente.

TabLayout.Tab tab = tabLayout.newTab();
tab.setText(R.string.tab_text_1);
tab.setIcon(R.drawable.ic_tab_1);
tabLayout.addTab(tab);

tabLayout.removeTab(tab);
tabLayout.removeTabAt(0);
tabLayout.removeAllTabs();

TabLayout tem dois modos, fixo e rolável.

tabLayout.setTabMode(TabLayout.MODE_FIXED);
tabLayout.setTabMode(TabLayout.MODE_SCROLLABLE);

Estes também podem ser aplicados em XML:

<android.support.design.widget.TabLayout
    android:id="@+id/tabLayout"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    app:tabMode="fixed|scrollable" />

Nota: os modos TabLayout são mutuamente exclusivos, o que significa que apenas um pode estar ativo por vez.

A cor do indicador da guia é a cor de destaque definida para o tema do Material Design. Você pode substituir essa cor definindo um estilo personalizado em styles.xml e, em seguida, aplicando o estilo ao seu TabLayout:

<style name="MyCustomTabLayoutStyle" parent="Widget.Design.TabLayout">
    <item name="tabIndicatorColor">@color/your_color</item>
</style>

Em seguida, você pode aplicar o estilo à exibição usando:

<android.support.design.widget.TabLayout
        android:id="@+id/tabs"
        style="@style/MyCustomTabLayoutStyle"
        android:layout_width="match_parent"
        android:layout_height="wrap_content">
</android.support.design.widget.TabLayout>

Folhas Inferiores na Biblioteca de Suporte de Design #

Folhas inferiores deslize para cima da parte inferior da tela para revelar mais conteúdo. Eles foram adicionados à Biblioteca de Suporte do Android na versão v25.1.0 e suportam acima de todas as versões.

Certifique-se de que a seguinte dependência seja adicionada ao arquivo build.gradle do seu aplicativo em dependências:

 compile 'com.android.support:design:25.3.1'

Folhas Inferiores Persistentes #

Você pode obter uma Persistent Bottom Sheet anexando um BottomSheetBehavior a uma View filha de um CoordinatorLayout:

<android.support.design.widget.CoordinatorLayout >

    <!-- .....   -->

    <LinearLayout
       android:id="@+id/bottom_sheet"
       android:elevation="4dp"
       android:minHeight="120dp"
       app:behavior_peekHeight="120dp"
       ...
       app:layout_behavior="android.support.design.widget.BottomSheetBehavior">

           <!-- .....   -->

       </LinearLayout>

</android.support.design.widget.CoordinatorLayout>

Então, no seu código, você pode criar uma referência usando:

 // The View with the BottomSheetBehavior  
 View bottomSheet = coordinatorLayout.findViewById(R.id.bottom_sheet);  
 BottomSheetBehavior mBottomSheetBehavior = BottomSheetBehavior.from(bottomSheet);

Você pode definir o estado do seu BottomSheetBehavior usando o método setState():

mBottomSheetBehavior.setState(BottomSheetBehavior.STATE_EXPANDED);

Você pode usar um destes estados:

  • STATE_COLLAPSED: este estado recolhido é o padrão e mostra apenas uma parte do layout na parte inferior. A altura pode ser controlada com o atributo app:behavior_peekHeight (o padrão é 0)

  • STATE_EXPANDED: o estado totalmente expandido da planilha inferior, onde toda a planilha inferior é visível (se sua altura for menor que o CoordinatorLayout) ou todo o CoordinatorLayout é preenchido

  • STATE_HIDDEN: desativado por padrão (e ativado com o atributo app:behavior_hideable), permitindo que os usuários deslizem para baixo na página inferior para ocultar completamente a página inferior

Além disso, para abrir ou fechar o BottomSheet ao clicar em uma exibição de sua escolha, digamos, um botão, aqui está como alternar o comportamento da planilha e atualizar a exibição.

mButton = (Button) findViewById(R.id.button_2);
    //On Button click we monitor the state of the sheet
    mButton.setOnClickListener(new View.OnClickListener() {
        @Override
        public void onClick(View view) {
            if (mBottomSheetBehavior.getState() == BottomSheetBehavior.STATE_EXPANDED) {
                //If expanded then collapse it (setting in Peek mode).
                mBottomSheetBehavior.setState(BottomSheetBehavior.STATE_COLLAPSED);
                mButton.setText(R.string.button2_hide);
            } else if (mBottomSheetBehavior.getState() == BottomSheetBehavior.STATE_COLLAPSED) {
                //If Collapsed then hide it completely.
                mBottomSheetBehavior.setState(BottomSheetBehavior.STATE_HIDDEN);
                mButton.setText(R.string.button2);
            } else if (mBottomSheetBehavior.getState() == BottomSheetBehavior.STATE_HIDDEN) {
                //If hidden then Collapse or Expand, as the need be.
                mBottomSheetBehavior.setState(BottomSheetBehavior.STATE_EXPANDED);
                mButton.setText(R.string.button2_peek);
            }
        }
    });

Mas o comportamento do BottomSheet também tem um recurso onde o usuário pode interagir com o swipe UP ou Down com um movimento DRAG. Nesse caso, talvez não possamos atualizar a visualização dependente (como o botão acima) se o estado da planilha tiver sido alterado. Para esse assunto, você gostaria de receber retornos de chamada de alterações de estado, portanto, você pode adicionar um BottomSheetCallback para ouvir os eventos de furto do usuário:

mBottomSheetBehavior.setBottomSheetCallback(new BottomSheetCallback() {  
    @Override  
    public void onStateChanged(@NonNull View bottomSheet, int newState) {  
      // React to state change and notify views of the current state
    }  
      @Override  
      public void onSlide(@NonNull View bottomSheet, float slideOffset) {  
       // React to dragging events and animate views or transparency of dependent views
   }  
 });

E se você quiser que sua folha inferior fique visível apenas nos modos RETIRADO e EXPANDIDO, alterna e nunca OCULTA, use:

mBottomSheetBehavior2.setHideable(false);

Bottom Sheet DialogFragment #

Você também pode exibir um BottomSheetDialogFragment no lugar de um View na folha inferior. Para fazer isso, primeiro você precisa criar uma nova classe que estenda BottomSheetDialogFragment.

Dentro do método setupDialog(), você pode inflar um novo arquivo de layout e recuperar o BottomSheetBehavior da visualização do contêiner em sua Activity. Depois de ter o comportamento, você pode criar e associar um BottomSheetCallback a ele para descartar o Fragmento quando a planilha estiver oculta.

public class BottomSheetDialogFragmentExample extends BottomSheetDialogFragment {
 
    private BottomSheetBehavior.BottomSheetCallback mBottomSheetBehaviorCallback = new BottomSheetBehavior.BottomSheetCallback() {
 
        @Override
        public void onStateChanged(@NonNull View bottomSheet, int newState) {
            if (newState == BottomSheetBehavior.STATE_HIDDEN) {
                dismiss();
            }
 
        }
 
        @Override
        public void onSlide(@NonNull View bottomSheet, float slideOffset) {
        }
    };
 
    @Override
    public void setupDialog(Dialog dialog, int style) {
        super.setupDialog(dialog, style);
        View contentView = View.inflate(getContext(), R.layout.fragment_bottom_sheet, null);
        dialog.setContentView(contentView);
 
        CoordinatorLayout.LayoutParams params = (CoordinatorLayout.LayoutParams) ((View) contentView.getParent()).getLayoutParams();
        CoordinatorLayout.Behavior behavior = params.getBehavior();
 
        if( behavior != null && behavior instanceof BottomSheetBehavior ) {
            ((BottomSheetBehavior) behavior).setBottomSheetCallback(mBottomSheetBehaviorCallback);
        }
    }
}

Finalmente, você pode chamar show() em uma instância do seu Fragment para exibi-lo na planilha inferior.

BottomSheetDialogFragment bottomSheetDialogFragment = new BottomSheetDialogFragmentExample();
bottomSheetDialogFragment.show(getSupportFragmentManager(), bottomSheetDialogFragment.getTag());

Aplicar um tema AppCompat #

A biblioteca de suporte AppCompat fornece temas para criar aplicativos com a especificação do Material Design. Um tema com um pai de Theme.AppCompat também é necessário para que uma Activity estenda AppCompatActivity.

O primeiro passo é personalizar a paleta de cores do seu tema para colorir automaticamente seu aplicativo. No arquivo res/styles.xml do seu aplicativo, você pode definir:

<!-- inherit from the AppCompat theme -->
<style name="AppTheme" parent="Theme.AppCompat">

    <!-- your app branding color for the app bar -->
    <item name="colorPrimary">#2196f3</item>
    
    <!-- darker variant for the status bar and contextual app bars -->
    <item name="colorPrimaryDark">#1976d2</item>

    <!-- theme UI controls like checkboxes and text fields -->
    <item name="colorAccent">#f44336</item>
</style>

Em vez de Theme.AppCompat, que tem um fundo escuro, você também pode usar Theme.AppCompat.Light ou Theme.AppCompat.Light.DarkActionBar.

Você pode personalizar o tema com suas próprias cores. Boas opções estão na Tabela de cores da especificação do design do material e na Paleta do material. As cores “500” são boas escolhas para cores primárias (azul 500 neste exemplo); escolha “700” do mesmo tom para o escuro; e uma sombra de uma tonalidade diferente como a cor de destaque. A cor primária é usada para a barra de ferramentas do seu aplicativo e sua entrada na tela de visão geral (aplicativos recentes), a variante mais escura para tingir a barra de status e a cor de destaque para destacar alguns controles.

Depois de criar este tema, aplique-o ao seu aplicativo no AndroidManifest.xml e também aplique o tema a qualquer atividade específica. Isso é útil para aplicar um tema AppTheme.NoActionBar, que permite implementar configurações de barra de ferramentas não padrão.

<application android:theme="@style/AppTheme" 
    ...>
    <activity 
        android:name=".MainActivity"
        android:theme="@style/AppTheme" />
</application>

Você também pode aplicar temas a Views individuais usando android:theme e um tema ThemeOverlay. Por exemplo, com uma Barra de ferramentas:

<android.support.v7.widget.Toolbar
  android:layout_width="match_parent"
  android:layout_height="wrap_content"
  android:background="?attr/colorPrimary"
  android:theme="@style/ThemeOverlay.AppCompat.Dark.ActionBar" />

ou um Botão:

<Button
    style="@style/Widget.AppCompat.Button.Colored"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:theme="@style/MyButtonTheme"/>

<!-- res/values/themes.xml -->
<style name="MyButtonTheme" parent="ThemeOverlay.AppCompat.Light">
    <item name="colorAccent">@color/my_color</item>
</style>

Adicione uma lanchonete #

Uma das principais características do Material Design é a adição de um Snackbar, que em teoria substitui o Toast anterior. De acordo com a documentação do Android:

Snackbars contêm uma única linha de texto diretamente relacionada ao operação realizada. Eles podem conter uma ação de texto, mas nenhum ícone. Os brindes são usados ​​principalmente para mensagens do sistema. Eles também exibem em na parte inferior da tela, mas não pode ser deslizado para fora da tela.

Toasts ainda podem ser usados ​​no Android para exibir mensagens aos usuários, mas se você decidiu optar pelo uso de material design em seu aplicativo, é recomendável que você realmente use uma lanchonete. Em vez de ser exibido como uma sobreposição na tela, um ‘Snackbar’ aparece na parte inferior.

Aqui está como é feito:

Snackbar snackbar = Snackbar
        .make(coordinatorLayout, "Here is your new Snackbar", Snackbar.LENGTH_LONG);
snackbar.show();

Quanto ao tempo de exibição do Snackbar, temos opções semelhantes às oferecidas por um Toast ou podemos definir uma duração personalizada em milissegundos:

  • LENGTH_SHORT
  • LENGTH_LONG
  • LENGTH_INDEFINITE
  • setDuration() (desde a versão 22.2.1)

Você também pode adicionar recursos dinâmicos ao seu Snackbar, como ActionCallback ou cores personalizadas. No entanto, preste atenção às diretrizes de design oferecidas pelo Android ao personalizar um Snackbar.

Implementar o Snackbar tem uma limitação no entanto. O layout pai da visualização em que você implementará um Snackbar precisa ser um CoordinatorLayout. Isso é para que o pop-up real da parte inferior possa ser feito.

Veja como definir um CoordinatorLayout em seu arquivo xml de layout:

<android.support.design.widget.CoordinatorLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/coordinatorLayout"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">
 
    //any other widgets in your layout go here.
 
</android.support.design.widget.CoordinatorLayout>

O CoordinatorLayout precisa então ser definido no método onCreate da sua Activity, e então usado ao criar o próprio Snackbar.

Para mais informações sobre o Snackbar, consulte a documentação oficial ou o tópico dedicado na documentação.

Como usar TextInputLayout #

Certifique-se de que a seguinte dependência seja adicionada ao arquivo build.gradle do seu aplicativo em dependências:

compile 'com.android.support:design:25.3.1'

Mostre a dica de um EditText como um rótulo flutuante quando um valor for inserido.

<android.support.design.widget.TextInputLayout
    android:layout_width="match_parent"
    android:layout_height="wrap_content">

     <android.support.design.widget.TextInputEditText
         android:layout_width="match_parent"
         android:layout_height="wrap_content"
         android:hint="@string/form_username"/>

</android.support.design.widget.TextInputLayout>

Para exibir o ícone de olho de exibição de senha com TextInputLayout, podemos usar o seguinte código:

<android.support.design.widget.TextInputLayout
    android:id="@+id/input_layout_current_password"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    app:passwordToggleEnabled="true">

    <android.support.design.widget.TextInputEditText

        android:id="@+id/current_password"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:hint="@string/current_password"
        android:inputType="textPassword" />

</android.support.design.widget.TextInputLayout>

onde os parâmetros app:passwordToggleEnabled="true" & android:inputType="textPassword" são obrigatórios.

app deve usar o namespace xmlns:app="http://schemas.android.com/apk/res-auto"

Adicionar uma gaveta de navegação #

Gavetas de navegação são usadas para navegar para destinos de nível superior em um aplicativo.

Certifique-se de ter adicionado a biblioteca de suporte ao design em seu arquivo build.gradle nas dependências:

 dependencies {
    // ...
    compile 'com.android.support:design:25.3.1'
}

Em seguida, adicione o DrawerLayout e o NavigationView em seu arquivo de recurso de layout XML. O DrawerLayout é apenas um contêiner sofisticado que permite que o NavigationView, a gaveta de navegação real, deslize da esquerda ou da direita da tela. Nota: para dispositivos móveis, o tamanho padrão da gaveta é 320dp.

<!-- res/layout/activity_main.xml -->
<android.support.v4.widget.DrawerLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/navigation_drawer_layout"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:fitsSystemWindows="true"
    tools:openDrawer="start">
    <! -- You can use "end" to open drawer from the right side -->

    <android.support.design.widget.CoordinatorLayout
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:fitsSystemWindows="true">

        <android.support.design.widget.AppBarLayout
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:theme="@style/AppTheme.AppBarOverlay">

            <android.support.v7.widget.Toolbar
                android:id="@+id/toolbar"
                android:layout_width="match_parent"
                android:layout_height="?attr/actionBarSize"
                android:background="?attr/colorPrimary"
                app:popupTheme="@style/AppTheme.PopupOverlay" />

        </android.support.design.widget.AppBarLayout>

    </android.support.design.widget.CoordinatorLayout>

    <android.support.design.widget.NavigationView
        android:id="@+id/navigation_drawer"
        android:layout_width="320dp"
        android:layout_height="match_parent"
        android:layout_gravity="start"
        android:fitsSystemWindows="true"
        app:headerLayout="@layout/drawer_header"
        app:menu="@menu/navigation_menu" />

</android.support.v4.widget.DrawerLayout>

Agora, se desejar, crie um arquivo de cabeçalho que servirá como o topo da sua gaveta de navegação. Isso é usado para dar uma aparência muito mais elegante à gaveta.

<!-- res/layout/drawer_header.xml -->
<RelativeLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="190dp">

    <ImageView
        android:id="@+id/header_image"
        android:layout_width="140dp"
        android:layout_height="120dp"
        android:layout_centerInParent="true"
        android:scaleType="centerCrop"
        android:src="@drawable/image" />

    <TextView
        android:id="@+id/header_text_view"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_below="@+id/header_image"
        android:text="User name"
        android:textSize="20sp" />

</RelativeLayout>

Ele é referenciado na tag NavigationView no atributo app:headerLayout="@layout/drawer_header". Este app:headerLayout infla o layout especificado no cabeçalho automaticamente. Isso pode ser feito alternativamente em tempo de execução com:

// Lookup navigation view
NavigationView navigationView = (NavigationView) findViewById(R.id.navigation_drawer);
// Inflate the header view at runtime
View headerLayout = navigationView.inflateHeaderView(R.layout.drawer_header);

Para preencher automaticamente sua gaveta de navegação com itens de navegação compatíveis com o material design, crie um arquivo de menu e adicione itens conforme necessário. Observação: embora os ícones dos itens não sejam obrigatórios, eles são sugeridos na especificação de design do material. Ele é referenciado na tag NavigationView no atributo app:menu="@menu/navigation_menu".

<!-- res/menu/menu_drawer.xml -->
<menu xmlns:android="http://schemas.android.com/apk/res/android">
    <item
        android:id="@+id/nav_item_1"
        android:title="Item #1"
        android:icon="@drawable/ic_nav_1" />
    <item
        android:id="@+id/nav_item_2"
        android:title="Item #2"
        android:icon="@drawable/ic_nav_2" />
    <item
        android:id="@+id/nav_item_3"
        android:title="Item #3"
        android:icon="@drawable/ic_nav_3" />
    <item
        android:id="@+id/nav_item_4"
        android:title="Item #4"
        android:icon="@drawable/ic_nav_4" />
</menu>

Para separar itens em grupos, coloque-os em um <menu> aninhado em outro <item> com um atributo android:title ou envolva-os com a tag <group>.

Agora que o layout está pronto, vá para o código Activity:

  // Find the navigation view
  NavigationView navigationView = (NavigationView) findViewById(R.id.navigation_drawer);
  navigationView.setNavigationItemSelectedListener(new NavigationView.OnNavigationItemSelectedListener() {
      @Override
      public boolean onNavigationItemSelected(MenuItem item) {
          // Get item ID to determine what to do on user click
          int itemId = item.getItemId();
          // Respond to Navigation Drawer selections with a new Intent
          startActivity(new Intent(this, OtherActivity.class));
          return true;
      }
  });

  DrawerLayout drawer = (DrawerLayout) findViewById(R.id.navigation_drawer_layout);
  // Necessary for automatically animated navigation drawer upon open and close
  ActionBarDrawerToggle toggle = new ActionBarDrawerToggle(this, drawer, "Open navigation drawer", "Close navigation drawer");
  // The two Strings are not displayed to the user, but be sure to put them into a separate strings.xml file.
  drawer.addDrawerListener(toggle);
  toogle.syncState();

Agora você pode fazer o que quiser na visualização de cabeçalho do NavigationView

View headerView = navigationView.getHeaderView();
TextView headerTextView = (TextView) headerview.findViewById(R.id.header_text_view);
ImageView headerImageView = (ImageView) headerview.findViewById(R.id.header_image);
// Set navigation header text
headerTextView.setText("User name");
// Set navigation header image
headerImageView.setImageResource(R.drawable.header_image);

A visualização do cabeçalho se comporta como qualquer outra View, então uma vez que você usa findViewById() e adiciona algumas outras Views ao seu arquivo de layout, você pode definir as propriedades de qualquer coisa nele.
