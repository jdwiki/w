Gson é uma biblioteca Java que pode ser usada para converter objetos Java em sua representação JSON. Gson considera esses dois objetivos de design muito importantes.

Recursos Gson:

Forneça métodos toJson() e fromJson() simples para converter objetos Java em JSON e vice-versa

Permitir que objetos não modificáveis ​​pré-existentes sejam convertidos de e para JSON

Amplo suporte de Java Generics

Suporta objetos arbitrariamente complexos (com profundas hierarquias de herança e uso extensivo de tipos genéricos)

Analisando JSON com Gson #

O exemplo mostra a análise de um objeto JSON usando a biblioteca Gson do Google.

Objetos de análise:

class Robot {
    //OPTIONAL - this annotation allows for the key to be different from the field name, and can be omitted if key and field name are same . Also this is good coding practice as it decouple your variable names with server keys name 
    @SerializedName("version") 
    private String version;

    @SerializedName("age")
    private int age;
    
    @SerializedName("robotName")
    private String name;
    
    // optional : Benefit it allows to set default values and retain them, even if key is missing from Json response. Not required for primitive data types. 

    public Robot{
       version = "";
       name = "";
    }


}

Então, onde a análise precisa ocorrer, use o seguinte:

String robotJson = "{
                        \"version\": \"JellyBean\",
                        \"age\": 3,
                        \"robotName\": \"Droid\"
                   }";

Gson gson = new Gson();
Robot robot = gson.fromJson(robotJson, Robot.class);

Analisando uma lista:

Ao recuperar uma lista de objetos JSON, muitas vezes você desejará analisá-los e convertê-los em objetos Java.

A string JSON que tentaremos converter é a seguinte:

{
  "owned_dogs": [
    {
      "name": "Ron",
      "age": 12,
      "breed": "terrier"
    },
    {
      "name": "Bob",
      "age": 4,
      "breed": "bulldog"
    },
    {
      "name": "Johny",
      "age": 3,
      "breed": "golden retriever"
    }
  ]
}

Essa matriz JSON específica contém três objetos. Em nosso código Java, queremos mapear esses objetos para objetos Dog. Um objeto Dog ficaria assim:

private class Dog {
    public String name;
    public int age;

    @SerializedName("breed")
    public String breedName;
}

Para converter a matriz JSON em um Dog[]:

Dog[] arrayOfDogs = gson.fromJson(jsonArrayString, Dog[].class);

Convertendo um Dog[] em uma string JSON:

String jsonArray = gson.toJson(arrayOfDogs, Dog[].class);

Para converter o array JSON em um ArrayList<Dog>, podemos fazer o seguinte:

Type typeListOfDogs = new TypeToken<List<Dog>>(){}.getType();
List<Dog> listOfDogs = gson.fromJson(jsonArrayString, typeListOfDogs);

O objeto Type typeListOfDogs define como seria uma lista de objetos Dog. O GSON pode usar esse objeto de tipo para mapear a matriz JSON para os valores corretos.

Como alternativa, a conversão de um List<Dog> em uma matriz JSON pode ser feita de maneira semelhante.

String jsonArray = gson.toJson(listOfDogs, typeListOfDogs);

Adicionando um conversor personalizado para Gson #

Às vezes você precisa serializar ou desserializar alguns campos em um formato desejado, por exemplo, seu backend pode usar o formato “AAAA-MM-dd HH:mm” para datas e você deseja que seu POJOS use a classe DateTime no Joda Time.

Para converter automaticamente essas strings no objeto DateTimes, você pode usar um conversor personalizado.

/**
 * Gson serialiser/deserialiser for converting Joda {@link DateTime} objects.
 */
public class DateTimeConverter implements JsonSerializer<DateTime>, JsonDeserializer<DateTime> {

    private final DateTimeFormatter dateTimeFormatter;

    @Inject
    public DateTimeConverter() {
        this.dateTimeFormatter = DateTimeFormat.forPattern("YYYY-MM-dd HH:mm");
    }

    @Override
    public JsonElement serialize(DateTime src, Type typeOfSrc, JsonSerializationContext context) {
        return new JsonPrimitive(dateTimeFormatter.print(src));
    }

    @Override
    public DateTime deserialize(JsonElement json, Type typeOfT, JsonDeserializationContext context)
            throws JsonParseException {
        
        if (json.getAsString() == null || json.getAsString().isEmpty()) {
            return null;
        }

        return dateTimeFormatter.parseDateTime(json.getAsString());
    }
}

Para fazer o Gson usar o conversor recém-criado, você precisa atribuí-lo ao criar o objeto Gson:

    DateTimeConverter dateTimeConverter = new DateTimeConverter();
    Gson gson = new GsonBuilder().registerTypeAdapter(DateTime.class, dateTimeConverter)
            .create();

    String s = gson.toJson(DateTime.now());
    // this will show the date in the desired format

Para desserializar a data nesse formato basta definir um campo no formato DateTime:

public class SomePojo {
    private DateTime someDate;    
}

Quando o Gson encontrar um campo do tipo DateTime, ele chamará seu conversor para desserializar o campo.

Analisando uma lista com Gson #

Método 1

Gson gson = new Gson();
String json = "[ \"Adam\", \"John\", \"Mary\" ]";

Type type = new TypeToken<List<String>>(){}.getType();
List<String> members = gson.fromJson(json, type);
Log.v("Members", members.toString());

Isso é útil para a maioria das classes de contêineres genéricas, já que você não pode obter a classe de um tipo parametrizado (ou seja: você não pode chamar List<String>.class).

Método 2

public class StringList extends ArrayList<String> { }

...

List<String> members = gson.fromJson(json, StringList.class);

Como alternativa, você sempre pode subclassificar o tipo desejado e, em seguida, passar essa classe. No entanto, isso nem sempre é uma prática recomendada, pois retornará a você um objeto do tipo StringList;

Analisando JSON para Objeto de Classe Genérico com Gson #

Suponha que tenhamos uma string JSON:

["first","second","third"]

Podemos analisar esta string JSON em um array String:

Gson gson = new Gson();
String jsonArray = "[\"first\",\"second\",\"third\"]";
String[] strings = gson.fromJson(jsonArray, String[].class);

Mas se queremos analisá-lo em um objeto List<String>, devemos usar TypeToken.

Aqui está a amostra:

Gson gson = new Gson();
String jsonArray = "[\"first\",\"second\",\"third\"]";
List<String> stringList = gson.fromJson(jsonArray, new TypeToken<List<String>>() {}.getType());

Suponha que temos duas classes abaixo:

public class Outer<T> {
    public int index;
    public T data;
}

public class Person {
    public String firstName;
    public String lastName;
}

e temos uma string JSON que deve ser analisada para um objeto Outer<Person>.

Este exemplo mostra como analisar essa string JSON para o objeto de classe genérico relacionado:

String json = "......";
Type userType = new TypeToken<Outer<Person>>(){}.getType();
Result<User> userResult = gson.fromJson(json,userType);

Se a string JSON deve ser analisada para um objeto Outer<List<Person>>:

Type userListType = new TypeToken<Outer<List<Person>>>(){}.getType();
Result<List<User>> userListResult = gson.fromJson(json,userListType);

Serialização/desserialização JSON com AutoValue e Gson #

Importar em seu arquivo raiz gradle

classpath 'com.neenbedankt.gradle.plugins:android-apt:1.8'

Importar no arquivo do aplicativo gradle

apt 'com.google.auto.value:auto-value:1.2'  
apt 'com.ryanharter.auto.value:auto-value-gson:0.3.1'  
provided 'com.jakewharton.auto.value:auto-value-annotations:1.2-update1'  
provided 'org.glassfish:javax.annotation:10.0-b28'

Criar objeto com autovalue:

@AutoValue public abstract class SignIn {    
    @SerializedName("signin_token") public abstract String signinToken();
    public abstract String username();

    public static TypeAdapter<SignIn> typeAdapter(Gson gson) {
        return new AutoValue_SignIn.GsonTypeAdapter(gson);
    }

    public static SignIn create(String signin, String username) {
        return new AutoValue_SignIn(signin, username);
    }
}

Crie seu conversor Gson com seu GsonBuilder

Gson gson = new GsonBuilder()
                .registerTypeAdapterFactory(
                    new AutoValueGsonTypeAdapterFactory())
                .create());

Desserializar

String myJsonData = "{
    \"signin_token\": \"mySigninToken\",
    \"username\": \"myUsername\" }";
SignIn signInData = gson.fromJson(myJsonData, Signin.class);

Serializar

Signin myData = SignIn.create("myTokenData", "myUsername");
String myJsonData = gson.toJson(myData);

Usar o Gson é uma ótima maneira de simplificar o código de serialização e desserialização usando objetos POJO. O efeito colateral é que a reflexão é dispendiosa em termos de desempenho. É por isso que usar AutoValue-Gson para gerar CustomTypeAdapter evitará esse custo de reflexão, mantendo-se muito simples de atualizar quando uma alteração de API estiver ocorrendo.

Deserializador JSON personalizado usando Gson #

Imagine que você tenha todas as datas em todas as respostas em algum formato personalizado, por exemplo /Date(1465935152)/ e você deseja aplicar a regra geral para desserializar todas as datas Json para instâncias Date java. Nesse caso, você precisa implementar o Json Deserializer personalizado.

Exemplo de json:

{
  "id": 1,
  "created_on": "Date(1465935152)",
  "updated_on": "Date(1465968945)",
  "name": "Oleksandr"
}

Suponha que temos esta classe abaixo:

class User {
    @SerializedName("id")
    long id;
    @SerializedName("created_on")
    Date createdOn;
    @SerializedName("updated_on")
    Date updatedOn;
    @SerializedName("name")
    String name;
}

Desserializador personalizado:

class DateDeSerializer implements JsonDeserializer<Date> {
    private static final String DATE_PREFIX = "/Date(";
    private static final String DATE_SUFFIX = ")/";

    @Override
    public Date deserialize(JsonElement json, Type typeOfT, JsonDeserializationContext context) throws JsonParseException {
        String dateString = json.getAsString();
        if (dateString.startsWith(DATE_PREFIX) && dateString.endsWith(DATE_SUFFIX)) {
            dateString = dateString.substring(DATE_PREFIX.length(), dateString.length() - DATE_SUFFIX.length());
        } else {
            throw new JsonParseException("Wrong date format: " + dateString);
        }
        return new Date(Long.parseLong(dateString) - TimeZone.getDefault().getRawOffset());
    }
}

E o uso:

Gson gson = new GsonBuilder()
                .registerTypeAdapter(Date.class, new DateDeSerializer())
                .create();
String json = "....";
User user = gson.fromJson(json, User.class);

Serialize e desserialize as strings Jackson JSON com tipos de data

Isso também se aplica ao caso em que você deseja tornar a conversão Gson Date compatível com Jackson, por exemplo.

Jackson geralmente serializa Date em “milissegundos desde a época”, enquanto Gson usa um formato legível como 31 de agosto de 2016 10:26:17 para representar Date. Isso leva a JsonSyntaxExceptions no Gson quando você tenta desserializar uma data no formato Jackson.

Para contornar isso, você pode adicionar um serializador personalizado e um desserializador personalizado:

JsonSerializer<Date> ser = new JsonSerializer<Date>() {
    @Override
    public JsonElement serialize(Date src, Type typeOfSrc, JsonSerializationContext
                context) {
        return src == null ? null : new JsonPrimitive(src.getTime());
    }
};

JsonDeserializer<Date> deser = new JsonDeserializer<Date>() {
    @Override
    public Date deserialize(JsonElement json, Type typeOfT,
                JsonDeserializationContext context) throws JsonParseException {
        return json == null ? null : new Date(json.getAsLong());
    }
};

Gson gson = new GsonBuilder()
                .registerTypeAdapter(Date.class, ser)
                .registerTypeAdapter(Date.class, deser)
                .create();

Usando Gson com herança #

Gson não suporta herança pronta para uso.

Digamos que temos a seguinte hierarquia de classes:

public class BaseClass {
    int a;
 
    public int getInt() {
        return a;
   }
}
 
public class DerivedClass1 extends BaseClass {
     int b;
 
     @Override
     public int getInt() {
         return b;
     }
 }
 
public class DerivedClass2 extends BaseClass {
    int c;
 
    @Override
    public int getInt() {
        return c;
    }
}

E agora queremos serializar uma instância de DerivedClass1 para uma string JSON

DerivedClass1 derivedClass1 = new DerivedClass1();
derivedClass1.b = 5;
derivedClass1.a = 10;
 
Gson gson = new Gson();
String derivedClass1Json = gson.toJson(derivedClass1);

Agora, em outro lugar, recebemos esta string json e queremos desserializá-la - mas em tempo de compilação sabemos apenas que deve ser uma instância de BaseClass:

BaseClass maybeDerivedClass1 = gson.fromJson(derivedClass1Json, BaseClass.class);
System.out.println(maybeDerivedClass1.getInt());

Mas o GSON não sabe que derivedClass1Json era originalmente uma instância de DerivedClass1, então isso imprimirá 10.

Como resolver isso?

Você precisa construir seu próprio JsonDeserializer, que lida com esses casos. A solução não é perfeitamente limpa, mas não consegui encontrar uma melhor.

Primeiro, adicione o seguinte campo à sua classe base

@SerializedName("type")
private String typeName;

E inicialize-o no construtor da classe base

public BaseClass() {
    typeName = getClass().getName();
}

Agora adicione a seguinte classe:

public class JsonDeserializerWithInheritance<T> implements JsonDeserializer<T> {
 
 @Override
 public T deserialize(
     JsonElement json, Type typeOfT, JsonDeserializationContext context)
     throws JsonParseException {
     JsonObject jsonObject = json.getAsJsonObject();
     JsonPrimitive classNamePrimitive = (JsonPrimitive) jsonObject.get("type");
 
     String className = classNamePrimitive.getAsString();
 
     Class<?> clazz;
     try {
     clazz = Class.forName(className);
     } catch (ClassNotFoundException e) {
     throw new JsonParseException(e.getMessage());
     }
     return context.deserialize(jsonObject, clazz);
 }
}

Tudo o que resta a fazer é ligar tudo -

GsonBuilder builder = new GsonBuilder();
 builder
 .registerTypeAdapter(BaseClass.class, new JsonDeserializerWithInheritance<BaseClass>());
 Gson gson = builder.create();

E agora, executando o seguinte código-

 DerivedClass1 derivedClass1 = new DerivedClass1();
 derivedClass1.b = 5;
 derivedClass1.a = 10;
 String derivedClass1Json = gson.toJson(derivedClass1);
 
 BaseClass maybeDerivedClass1 = gson.fromJson(derivedClass1Json, BaseClass.class);
 System.out.println(maybeDerivedClass1.getInt());

Irá imprimir 5.
