Trabalhando com EditTexts #

O EditText é o widget de entrada de texto padrão em aplicativos Android. Se o usuário precisar inserir texto em um aplicativo, essa é a principal maneira de fazer isso.

Editar texto

Existem muitas propriedades importantes que podem ser definidas para personalizar o comportamento de um EditText. Vários deles estão listados abaixo. Confira o guia oficial de campos de texto para obter ainda mais detalhes sobre os campos de entrada.

Uso

Um EditText é adicionado a um layout com todos os comportamentos padrão com o seguinte XML:

<EditText
    android:id="@+id/et_simple"
    android:layout_height="wrap_content"
    android:layout_width="match_parent">
</EditText>

Observe que um EditText é simplesmente uma extensão fina do TextView e herda todas as mesmas propriedades.

Recuperando o Valor

Obtendo o valor do texto inserido em um EditText é o seguinte:

EditText simpleEditText = (EditText) findViewById(R.id.et_simple);
String strValue = simpleEditText.getText().toString();

Mais personalização de entrada

Podemos querer limitar a entrada a uma única linha de texto (evite novas linhas):

<EditText
  android:singleLine="true"
  android:lines="1"
/>

Você pode limitar os caracteres que podem ser inseridos em um campo usando o atributo digits:

<EditText
  android:inputType="number"
  android:digits="01"
/>

Isso restringiria os dígitos inseridos a apenas “0” e “1”. Podemos querer limitar o número total de caracteres com:

<EditText
  android:maxLength="5"
/>

Usando essas propriedades, podemos definir o comportamento de entrada esperado para campos de texto.

Ajuste de cores

Você pode ajustar a cor de fundo de realce do texto selecionado em um EditText com a propriedade android:textColorHighlight:

<EditText
    android:textColorHighlight="#7cff88"
/>

Exibindo dicas de espaço reservado

Você pode querer definir a dica para o controle EditText para solicitar a um usuário uma entrada específica com:

<EditText
    ...
    android:hint="@string/my_hint">
</EditText>

Dicas

Mudando a cor da linha de fundo

Supondo que você esteja usando a biblioteca AppCompat, você pode substituir os estilos colorControlNormal, colorControlActivated e colorControlHighlight:

<style name="Theme.App.Base" parent="Theme.AppCompat.Light.DarkActionBar">
    <item name="colorControlNormal">#d32f2f</item>
    <item name="colorControlActivated">#ff5722</item>
    <item name="colorControlHighlight">#f44336</item>
</style>

Se você não vir esses estilos aplicados em um DialogFragment, há um bug conhecido ao usar o LayoutInflater passado para o método onCreateView().

O problema já foi corrigido na biblioteca AppCompat v23. Consulte este guia sobre como atualizar. Outra solução temporária é usar o inflador de layout da Activity em vez daquele passado para o método onCreateView():

public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
        View view = getActivity().getLayoutInflater().inflate(R.layout.dialog_fragment, container);
  }

Ouvindo a entrada EditText

Confira as notas básicas dos ouvintes de eventos para ver como ouvir as alterações em um EditText e executar uma ação quando essas alterações ocorrerem.

Exibindo comentários de rótulos flutuantes

Tradicionalmente, o EditText oculta a mensagem de dica (explicada acima) depois que o usuário começa a digitar. Além disso, qualquer mensagem de erro de validação precisava ser gerenciada manualmente pelo desenvolvedor.

Com o TextInputLayout você pode configurar um rótulo flutuante para exibir dicas e mensagens de erro. Você pode encontrar mais detalhes aqui.

Personalizando o Tipo de Entrada #

Os campos de texto podem ter diferentes tipos de entrada, como número, data, senha ou endereço de e-mail. O tipo determina que tipo de caracteres são permitidos dentro do campo e pode solicitar que o teclado virtual otimize seu layout para caracteres usados ​​com frequência.

Por padrão, qualquer conteúdo de texto dentro de um controle EditText é exibido como texto simples. Ao definir o atributo inputType, podemos facilitar a entrada de diferentes tipos de informações, como números de telefone e senhas:

<EditText
    ...
    android:inputType="phone">
</EditText>

Os tipos de entrada mais comuns incluem:

|Tipo | Descrição | |——————|—————————— ——————-| |textUri | Texto que será usado como URI | |textoEmailEndereço | Texto que será utilizado como endereço de e-mail | |textoNomePessoa | Texto que é o nome de uma pessoa | |textoSenha | Texto que é uma senha que deve ser ocultada | |número | Um campo somente numérico | |telefone | Para inserir um número de telefone | |data | Para inserir uma data | |tempo | Para inserir um horário | |textMultiLinha | Permitir várias linhas de texto no campo |

O android:inputType também permite especificar certos comportamentos do teclado, como capitalizar todas as palavras novas ou usar recursos como preenchimento automático e sugestões de ortografia. Aqui estão alguns dos valores de tipo de entrada comuns que definem os comportamentos do teclado:

|Tipo | Descrição | |——————|—————————— ——————-| |textCapSentenças | Teclado de texto normal que coloca em maiúscula a primeira letra de cada nova frase| |textCapWords | Teclado de texto normal que capitaliza cada palavra. Bom para títulos ou nomes de pessoas| |textAutoCorreção | Teclado de texto normal que corrige palavras com erros ortográficos comuns|

Você pode definir vários atributos inputType se necessário (separados por ‘|’). Exemplo:

<EditText
    android:id="@+id/postal_address"
    android:layout_width="fill_parent"
    android:layout_height="wrap_content"
    android:hint="@string/postal_address_hint"
    android:inputType="textPostalAddress|
                       textCapWords|
                       textNoSuggestions" />

Você pode ver uma lista de todos os tipos de entrada disponíveis aqui.

Escondendo o SoftKeyboard #

Ocultar o Softkeyboard é um requisito básico geralmente ao trabalhar com EditText. O softkeyboard por padrão só pode ser fechado pressionando o botão Voltar e, portanto, a maioria dos desenvolvedores usa InputMethodManager para forçar o Android a ocultar o teclado virtual chamando hideSoftInputFromWindow e passando o token da janela que contém seu visão focada. O código para fazer o seguinte:

public void hideSoftKeyboard()    
{
        InputMethodManager inputMethodManager = (InputMethodManager) getSystemService(Activity.INPUT_METHOD_SERVICE);
        inputMethodManager.hideSoftInputFromWindow(getCurrentFocus().getWindowToken(), 0);    
}

O código é direto, mas outro grande problema que surge é que a função hide precisa ser chamada quando ocorre algum evento. O que fazer quando você precisa do Softkeyboard oculto ao pressionar qualquer lugar que não seja o EditText? O código a seguir fornece uma função legal que precisa ser chamada em seu método onCreate() apenas uma vez.

public void setupUI(View view) 
{
        String s = "inside";
        //Set up touch listener for non-text box views to hide keyboard.
        if (!(view instanceof EditText)) {

            view.setOnTouchListener(new View.OnTouchListener() {

                public boolean onTouch(View v, MotionEvent event) {
                    hideSoftKeyboard();
                    return false;
                }

            });
        }

        //If a layout container, iterate over children and seed recursion.
        if (view instanceof ViewGroup) {

            for (int i = 0; i < ((ViewGroup) view).getChildCount(); i++) {

                View innerView = ((ViewGroup) view).getChildAt(i);

                setupUI(innerView);
            }
        }    
}

Ícone ou botão dentro do Custom Edit Text e sua ação e clique em ouvintes. #

Este exemplo ajudará a ter o texto Editar com o ícone do lado direito.

Nota: Neste apenas estou usando setCompoundDrawablesWithIntrinsicBounds, Então, se você quiser alterar a posição do ícone, você pode conseguir isso usando setCompoundDrawablesWithIntrinsicBounds em setIcon.

public class MKEditText extends AppCompatEditText {

    public interface IconClickListener {
        public void onClick();
    }

    private IconClickListener mIconClickListener;

    private static final String TAG = MKEditText.class.getSimpleName();

    private final int EXTRA_TOUCH_AREA = 50;
    private Drawable mDrawable;
    private boolean touchDown;

    public MKEditText(Context context, AttributeSet attrs, int defStyle) {
        super(context, attrs, defStyle);
    }

    public MKEditText(Context context) {
        super(context);
    }

    public MKEditText(Context context, AttributeSet attrs) {
        super(context, attrs);
    }

    public void showRightIcon() {
        mDrawable = ContextCompat.getDrawable(getContext(), R.drawable.ic_android_black_24dp);

        setIcon();
    }

    public void setIconClickListener(IconClickListener iconClickListener) {
        mIconClickListener = iconClickListener;
    }

    private void setIcon() {
        Drawable[] drawables = getCompoundDrawables();

        setCompoundDrawablesWithIntrinsicBounds(drawables[0], drawables[1], mDrawable, drawables[3]);

        setInputType(InputType.TYPE_CLASS_TEXT | InputType.TYPE_TEXT_VARIATION_PASSWORD);
        setSelection(getText().length());
    }

    @Override
    public boolean onTouchEvent(MotionEvent event) {
        final int right = getRight();
        final int drawableSize = getCompoundPaddingRight();
        final int x = (int) event.getX();
        switch (event.getAction()) {
            case MotionEvent.ACTION_DOWN:
                if (x + EXTRA_TOUCH_AREA >= right - drawableSize && x <= right + EXTRA_TOUCH_AREA) {
                    touchDown = true;
                    return true;
                }
                break;
            case MotionEvent.ACTION_UP:
                if (x + EXTRA_TOUCH_AREA >= right - drawableSize && x <= right + EXTRA_TOUCH_AREA && touchDown) {
                    touchDown = false;
                    if (mIconClickListener != null) {
                        mIconClickListener.onClick();
                    }
                    return true;
                }
                touchDown = false;
                break;

        }
        return super.onTouchEvent(event);
    }
}

Se você quiser alterar a área de toque, você pode alterar o Valores padrão EXTRA_TOUCH_AREA que dei como 50.

E para Ativar o botão e clicar no ouvinte, você pode ligar da sua Atividade ou Fragmento assim,

MKEditText mkEditText = (MKEditText) findViewById(R.id.password);
mkEditText.showRightIcon();
mkEditText.setIconClickListener(new MKEditText.IconClickListener() {
            @Override
            public void onClick() {
              // You can do action here for the icon.
               
            }
        });
