Realm Mobile Database é uma alternativa ao SQLite. O Realm Mobile Database é muito mais rápido que um ORM e geralmente mais rápido que o SQLite bruto.

Benefícios

Funcionalidade offline, consultas rápidas, segmentação segura, aplicativos multiplataforma, criptografia, arquitetura reativa.

Ao usar Realm, você deve lembrar que não deve passar instâncias RealmObjects, RealmResults e Realm entre threads. Se você precisar de uma consulta em um determinado encadeamento, abra uma instância do Realm nesse encadeamento. No final do thread, você deve fechar o Realm.

Consultas classificadas #

Para classificar uma consulta, em vez de usar findAll(), você deve usar findAllSorted().

RealmResults<SomeObject> results = realm.where(SomeObject.class)
                                            .findAllSorted("sortField", Sort.ASCENDING);

Observação:

sort() retorna um RealmResults completamente novo que é classificado, mas uma atualização para este RealmResults irá redefini-lo. Se você usar sort(), você deve sempre reclassificá-lo em seu RealmChangeListener, remover o RealmChangeListener do RealmResults anterior e adicioná-lo ao novo RealmResults retornado. Usar sort() em um RealmResults retornado por uma consulta assíncrona que ainda não foi carregada falhará.

findAllSorted() sempre retornará os resultados ordenados pelo campo, mesmo que seja atualizado. Recomenda-se usar findAllSorted().

Usando Realm com RxJava #

Para consultas, o Realm fornece o método realmResults.asObservable(). Observar os resultados só é possível em threads de looper (normalmente o thread de interface do usuário).

Para que isso funcione, sua configuração deve conter o seguinte

realmConfiguration = new RealmConfiguration.Builder(context)       //
                          .rxFactory(new RealmObservableFactory()) //
                             //...
                          .build();

Depois, você pode usar seus resultados como um observável.

Observable<RealmResults<SomeObject>> observable = results.asObservable();

Para consultas assíncronas, você deve filtrar os resultados por isLoaded(), para receber um evento somente quando a consulta for executada. Este filter() não é necessário para consultas síncronas (isLoaded() sempre retorna true em consultas de sincronização).

    Subscription subscription = RxTextView.textChanges(editText).switchMap(charSequence -> 
        realm.where(SomeObject.class)
             .contains("searchField", charSequence.toString(), Case.INSENSITIVE)
             .findAllAsync()
             .asObservable())
    .filter(RealmResults::isLoaded) //
    .subscribe(objects -> adapter.updateData(objects));

Para gravações, você deve usar o método executeTransactionAsync() ou abrir uma instância de Realm no thread em segundo plano, executar a transação de forma síncrona e fechar a instância de Realm.

public Subscription loadObjectsFromNetwork() {
    return objectApi.getObjects()
        .subscribeOn(Schedulers.io())
        .subscribe(response -> {
            try(Realm realmInstance = Realm.getDefaultInstance()) {
                realmInstance.executeTransaction(realm -> realm.insertOrUpdate(response.objects));
            }
        });
}

Uso básico #

Configurando uma instância #

Para usar o Realm, primeiro você precisa obter uma instância dele. Cada instância do Realm é mapeada para um arquivo no disco. A maneira mais básica de obter uma instância é a seguinte:

// Create configuration
RealmConfiguration realmConfiguration = new RealmConfiguration.Builder(context).build();

// Obtain realm instance
Realm realm = Realm.getInstance(realmConfiguration);
// or
Realm.setDefaultConfiguration(realmConfiguration);
Realm realm = Realm.getDefaultInstance();

O método Realm.getInstance() cria o arquivo de banco de dados caso não tenha sido criado, caso contrário abre o arquivo. O objeto RealmConfiguration controla todos os aspectos de como um Realm é criado - seja um banco de dados inMemory(), nome do arquivo Realm, se o Realm deve ser limpo se for necessária uma migração, dados iniciais, etc.

Observe que chamadas para Realm.getInstance() são contadas por referência (cada chamada incrementa um contador), e o contador é decrementado quando realm.close() é chamado.

Fechando uma instância #

Em threads em segundo plano, é muito importante fechar a(s) instância(s) do Realm assim que ela não for mais usada (por exemplo, a transação é concluída e a execução da thread termina). A falha em fechar todas as instâncias do Realm no encadeamento em segundo plano resulta na fixação da versão e pode causar um grande aumento no tamanho do arquivo.

Runnable runnable = new Runnable() {
    Realm realm = null;
    try {
        realm = Realm.getDefaultInstance();
        // ...
    } finally {
        if(realm != null) {
            realm.close();
        }
    }
};

new Thread(runnable).start(); // background thread, like `doInBackground()` of AsyncTask

Vale a pena notar que acima do nível 19 da API, você pode substituir este código apenas por este:

try(Realm realm = Realm.getDefaultInstance()) {
    // ...
}

Modelos #

O próximo passo seria criar seus modelos. Aqui uma pergunta pode ser feita, “o que é um modelo?”. Um modelo é uma estrutura que define as propriedades de um objeto que está sendo armazenado no banco de dados. Por exemplo, a seguir, modelamos um livro.

public class Book extends RealmObject {
 
    // Primary key of this entity
    @PrimaryKey
    private long id;
 
    private String title;

    @Index // faster queries
    private String author;
 
    // Standard getters & setter
    public long getId() {
        return id;
    }
 
    public void setId(long id) {
        this.id = id;
    }
 
    public String getTitle() {
        return title;
    }
 
    public void setTitle(String title) {
        this.title = title;
    }
 
    public String getAuthor() {
        return author;
    }
 
    public void setAuthor(String author) {
        this.author = author;
    }
}

Observe que seus modelos devem estender a classe RealmObject. A chave primária também é especificada pela anotação @PrimaryKey. As chaves primárias podem ser nulas, mas apenas um elemento pode ter null como chave primária. Além disso, você pode usar a anotação @Ignore para os campos que não devem ser persistidos no disco:

@Ignore
private String isbn;

Inserindo ou atualizando dados #

Para armazenar um objeto de livro em sua instância de banco de dados Realm, você pode primeiro criar uma instância de seu modelo e depois armazená-la no banco de dados através do método copyToRealm. Para criar ou atualizar você pode usar copyToRealmOrUpdate. (Uma alternativa mais rápida é o recém-adicionado insertOrUpdate()).

// Creating an instance of the model
Book book = new Book();
book.setId(1);
book.setTitle("Walking on air");
book.setAuthor("Taylor Swift")

// Store to the database
realm.executeTransaction(new Realm.Transaction() {
    @Override
    public void execute(Realm realm) {
        realm.insertOrUpdate(book);
    }
});

Observe que todas as alterações nos dados devem ocorrer em uma transação. Outra maneira de criar um objeto é usando o seguinte padrão:

Book book = realm.createObject(Book.class, primaryKey);
...

Consultando o banco de dados #

  • Todos os livros:

    RealmResults<Book> results = realm.where(Book.class).findAll();

  • Todos os livros com id maior que 10:

    RealmResults<Book> results = realm.where(Book.class)
                                  .greaterThan("id", 10)
                                  .findAll();

  • Livros de 'Taylor Swift' ou '%Peter%':

    RealmResults<Book> results = realm.where(Book.class)
                                  .beginGroup()
                                      .equalTo("author", "Taylor Swift")
                                      .or()
                                      .contains("author", "Peter")
                                  .endGroup().findAll();

Apagando um objeto #

Por exemplo, queremos excluir todos os livros de Taylor Swift:

// Start of transaction
realm.executeTransaction(new Realm.Transaction() {
    @Override
    public void execute(Realm realm) {
        // First Step: Query all Taylor Swift books
        RealmResults<Book> results = ...
        
        // Second Step: Delete elements in Realm
        results.deleteAllFromRealm();
    }
});

Lista de primitivos (RealmList<Integer/String/…>) #

O Realm atualmente não suporta o armazenamento de uma lista de primitivos. Está na lista de tarefas (edição #575 do GitHub), mas por enquanto, aqui está uma solução alternativa.

Crie uma nova classe para seu tipo primitivo, isso usa Integer, mas altere-o para o que você deseja armazenar.

public class RealmInteger extends RealmObject {
    private int val;

    public RealmInteger() {
    }

    public RealmInteger(int val) {
        this.val = val;
    }

    // Getters and setters
}

Agora você pode usar isso em seu RealmObject.

public class MainObject extends RealmObject {

    private String name;
    private RealmList<RealmInteger> ints;

    // Getters and setters
}

Se você estiver usando GSON para preencher seu RealmObject, você precisará adicionar um adaptador de tipo personalizado.

Type token = new TypeToken<RealmList<RealmInteger>>(){}.getType();
Gson gson = new GsonBuilder()
        .setExclusionStrategies(new ExclusionStrategy() {
            @Override
            public boolean shouldSkipField(FieldAttributes f) {
                return f.getDeclaringClass().equals(RealmObject.class);
            }

            @Override
            public boolean shouldSkipClass(Class<?> clazz) {
                return false;
            }
        })
        .registerTypeAdapter(token, new TypeAdapter<RealmList<RealmInteger>>() {

            @Override
            public void write(JsonWriter out, RealmList<RealmInteger> value) throws IOException {
                // Empty
            }

            @Override
            public RealmList<RealmInteger> read(JsonReader in) throws IOException {
                RealmList<RealmInteger> list = new RealmList<RealmInteger>();
                in.beginArray();
                while (in.hasNext()) {
                    list.add(new RealmInteger(in.nextInt()));
                }
                in.endArray();
                return list;
            }
        })
        .create();

Adicionando Realm ao seu projeto #

Adicione a seguinte dependência ao seu arquivo build.gradle de nível projeto.

dependencies {
    classpath "io.realm:realm-gradle-plugin:3.1.2"
}

Adicione o seguinte logo no topo do seu arquivo build.gradle de nível app.

apply plugin: 'realm-android'

Conclua uma sincronização gradle e agora você terá o Realm adicionado como uma dependência ao seu projeto!

O Realm requer uma chamada inicial desde 2.0.0 antes de usá-lo. Você pode fazer isso em sua classe Application ou no método onCreate da sua primeira atividade.

Realm.init(this); // added in Realm 2.0.0
Realm.setDefaultConfiguration(new RealmConfiguration.Builder().build());

Modelos de Reino #

Modelos de realm devem estender a classe base RealmObject, eles definem o esquema do banco de dados subjacente.

Os tipos de campo suportados são boolean, byte, short, int, long, float, double, String, Date, byte[], links para outros RealmObjects e RealmList<T estende RealmModel>.

public class Person extends RealmObject {
    @PrimaryKey //primary key is also implicitly an @Index 
                //it is required for `copyToRealmOrUpdate()` to update the object.
    private long id;

    @Index //index makes queries faster on this field
    @Required //prevents `null` value from being inserted
    private String name; 

    private RealmList<Dog> dogs; //->many relationship to Dog

    private Person spouse; //->one relationship to Person

    @Ignore
    private Calendar birthday; //calendars are not supported but can be ignored

    // getters, setters
}

Se você adicionar (ou remover) um novo campo ao seu RealmObject (ou adicionar uma nova classe RealmObject ou excluir uma existente), será necessária uma migração. Você pode definir deleteIfMigrationNeeded() em seu RealmConfiguration.Builder ou definir a migração necessária. A migração também é necessária ao adicionar (ou remover) a anotação @Required, ou @Index ou @PrimaryKey.

Os relacionamentos devem ser definidos manualmente, eles NÃO são automáticos com base nas chaves primárias.

Desde 0.88.0, também é possível usar campos públicos em vez de campos/getters/setters privados nas classes RealmObject.

Também é possível implementar RealmModel em vez de estender RealmObject, se a classe também for anotada com @RealmClass.

@RealmClass
public class Person implements RealmModel {
    // ...
}

Nesse caso, métodos como person.deleteFromRealm() ou person.addChangeListener() são substituídos por RealmObject.deleteFromRealm(person) e RealmObject.addChangeListener(person).

Limitações são que por um RealmObject, apenas RealmObject pode ser estendido, e não há suporte para final, volatile e campos transiente.

É importante que uma classe RealmObject gerenciada só possa ser modificada em uma transação. Um RealmObject gerenciado não pode ser passado entre threads.

Consultas assíncronas #

Cada método de consulta síncrona (como findAll() ou findAllSorted()) tem uma contrapartida assíncrona (findAllAsync() / findAllSortedAsync()).

As consultas assíncronas transferem a avaliação do RealmResults para outro encadeamento. Para receber esses resultados no encadeamento atual, o encadeamento atual deve ser um encadeamento de looper (leia-se: consultas assíncronas normalmente só funcionam no encadeamento da interface do usuário).

RealmChangeListener<RealmResults<SomeObject>> realmChangeListener; // field variable

realmChangeListener = new RealmChangeListener<RealmResults<SomeObject>>() {
    @Override
    public void onChange(RealmResults<SomeObject> element) {
        // asyncResults are now loaded
        adapter.updateData(element);
    }
};

RealmResults<SomeObject> asyncResults = realm.where(SomeObject.class).findAllAsync();
asyncResults.addChangeListener(realmChangeListener);
