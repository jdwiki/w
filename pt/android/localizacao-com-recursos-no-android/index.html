Adicionando tradução ao seu aplicativo Android #

Você tem que criar um arquivo strings.xml diferente para cada novo idioma.

  1. Clique com o botão direito do mouse na pasta res
  2. Escolha Novo → Arquivo de recursos de valores
  3. Selecione uma localidade entre os qualificadores disponíveis
  4. Clique no botão Próximo (»)
  5. Selecione um idioma
  6. Nomeie o arquivo strings.xml

strings.xml

<resources>
    <string name="app_name">Testing Application</string>
    <string name="hello">Hello World</string>
</resources>

strings.xml(oi)

<resources>
    <string name="app_name">परीक्षण आवेदन</string>
    <string name="hello">नमस्ते दुनिया</string>
</resources>

Definindo o idioma programaticamente:

public void setLocale(String locale) // Pass "en","hi", etc.
{
    myLocale = new Locale(locale);
    // Saving selected locale to session - SharedPreferences.
    saveLocale(locale);
    // Changing locale.
    Locale.setDefault(myLocale);
    android.content.res.Configuration config = new android.content.res.Configuration();
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.N) {
        config.setLocale(myLocale);
    } else {
        config.locale = myLocale;
    }
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.JELLY_BEAN_MR1) {
        getBaseContext().createConfigurationContext(config);
    } else {
        getBaseContext().getResources().updateConfiguration(config, getBaseContext().getResources().getDisplayMetrics());
    }
}

A função acima irá alterar os campos de texto que são referenciados em strings.xml. Por exemplo, suponha que você tenha as duas visualizações de texto a seguir:

<TextView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content" 
    android:text="@string/app_name"/>
<TextView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content" 
    android:text="@string/hello"/>

Então, depois de alterar a localidade, as strings de idioma com os ids app_name e hello serão alteradas de acordo.

Tipo de diretórios de recursos na pasta “res” #

Ao localizar diferentes tipos de recursos são necessários, cada um dos quais tem sua própria casa na estrutura do projeto android. A seguir estão os diferentes diretórios que podemos colocar no diretório \res. Os tipos de recursos colocados em cada um desses diretórios são explicados na tabela abaixo:

Diretório Tipo de recurso
animador/ Arquivos XML que definem animações de propriedades.
anime/ Arquivos XML que definem animações de interpolação. (As animações de propriedade também podem ser salvas neste diretório, mas o diretório animator/ é preferível para que as animações de propriedade diferenciem os dois tipos.)
cor/ Arquivos XML que definem uma lista de estado de cores. Consulte Recurso de lista de estados de cores
desenhável/ “Arquivos de bitmap (.png, .9.png, .jpg, .gif) ou arquivos XML que são compilados nos seguintes subtipos de recursos desenháveis: : Arquivos de bitmap - Nine-Patches (bitmaps redimensionáveis) - Listas de estado - Formas - Drawables de animação - Outros drawables - "
mipmap/ Arquivos desenháveis ​​para diferentes densidades de ícones do iniciador. Para obter mais informações sobre como gerenciar ícones do inicializador com mipmap/ pastas, consulte Visão geral de gerenciamento de projetos.
disposição/ Arquivos XML que definem um layout de interface do usuário. Consulte Recurso de layout.
cardápio/ Arquivos XML que definem os menus do aplicativo, como um menu de opções, menu de contexto ou submenu. Consulte Recurso de menu.
cru/ Arquivos arbitrários para salvar em sua forma bruta. Para abrir esses recursos com um InputStream bruto, chame Resources.openRawResource() com o ID do recurso, que é R.raw.filename.
No entanto, se você precisar de acesso a nomes de arquivos originais e hierarquia de arquivos, considere salvar alguns recursos no diretório assets/ (em vez de res/raw/). Os arquivos em assets/ não recebem um ID de recurso, portanto, você pode lê-los apenas usando AssetManager.
valores/ Arquivos XML que contêm valores simples, como strings, números inteiros e cores, bem como estilos e temas
xml/ Arquivos XML arbitrários que podem ser lidos em tempo de execução chamando Resources.getXML(). Vários arquivos de configuração XML devem ser salvos aqui, como uma configuração pesquisável.

Altere a localidade do aplicativo Android programaticamente #

Nos exemplos acima você entende como localizar recursos do aplicativo. O exemplo a seguir explica como alterar a localidade do aplicativo no aplicativo, não no dispositivo. Para alterar apenas a localidade do aplicativo, você pode usar abaixo o locale util.

import android.app.Application;
import android.content.Context;
import android.content.SharedPreferences;
import android.content.res.Configuration;
import android.content.res.Resources;
import android.os.Build;
import android.preference.PreferenceManager;
import android.view.ContextThemeWrapper;

import java.util.Locale;

/**
 * Created by Umesh on 10/10/16.
 */
public class LocaleUtils {

    private static Locale mLocale;

    public static void setLocale(Locale locale){
        mLocale = locale;
        if(mLocale != null){
            Locale.setDefault(mLocale);
        }
    }

    public static void updateConfiguration(ContextThemeWrapper wrapper){
        if(mLocale != null && Build.VERSION.SDK_INT >= Build.VERSION_CODES.JELLY_BEAN_MR1){
            Configuration configuration = new Configuration();
            configuration.setLocale(mLocale);
            wrapper.applyOverrideConfiguration(configuration);
        }
    }

    public static void updateConfiguration(Application application, Configuration configuration){
        if(mLocale != null && Build.VERSION.SDK_INT < Build.VERSION_CODES.JELLY_BEAN_MR1){
            Configuration config = new Configuration(configuration);
            config.locale = mLocale;
            Resources res = application.getBaseContext().getResources();
            res.updateConfiguration(configuration, res.getDisplayMetrics());
        }
    }

    public static void updateConfiguration(Context context, String language, String country){
        Locale locale = new Locale(language,country);
        setLocale(locale);
        if(mLocale != null){
            Resources res = context.getResources();
            Configuration configuration = res.getConfiguration();
            configuration.locale = mLocale;
            res.updateConfiguration(configuration,res.getDisplayMetrics());
        }
    }




    public static String getPrefLangCode(Context context) {
        return PreferenceManager.getDefaultSharedPreferences(context).getString("lang_code","en");
    }

    public static void setPrefLangCode(Context context, String mPrefLangCode) {

        SharedPreferences.Editor editor = PreferenceManager.getDefaultSharedPreferences(context).edit();
        editor.putString("lang_code",mPrefLangCode);
        editor.commit();
    }

    public static  String getPrefCountryCode(Context context) {
        return PreferenceManager.getDefaultSharedPreferences(context).getString("country_code","US");
    }

    public static void setPrefCountryCode(Context context,String mPrefCountryCode) {

        SharedPreferences.Editor editor = PreferenceManager.getDefaultSharedPreferences(context).edit();
        editor.putString("country_code",mPrefCountryCode);
        editor.commit();
    }
}

Inicialize a localidade que o usuário preferiu, da classe Application.

public class LocaleApp extends Application{

    @Override
    public void onCreate() {
        super.onCreate();

        LocaleUtils.setLocale(new Locale(LocaleUtils.getPrefLangCode(this), LocaleUtils.getPrefCountryCode(this)));
        LocaleUtils.updateConfiguration(this, getResources().getConfiguration());
    }
}

Você também precisa criar uma atividade base e estender essa atividade para todas as outras atividades para que você possa alterar a localidade do aplicativo em apenas um local da seguinte maneira:

public abstract class LocalizationActivity extends AppCompatActivity {

    public LocalizationActivity() {
        LocaleUtils.updateConfiguration(this);
    }

    // We only override onCreate
    @Override
    protected void onCreate(@Nullable Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
    }

}

Nota: Sempre inicialize a localidade no construtor.

Agora você pode usar LocalizationActivity da seguinte forma.

public class MainActivity extends LocalizationActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

    }
}

Nota: Quando você altera a localidade do aplicativo programaticamente, precisa reinicie sua atividade para ter o efeito da mudança de localidade Para funcionar corretamente para esta solução, você e usa a localidade das preferências compartilhadas na inicialização do aplicativo, android:name=".LocaleApp" em seu Manifest.xml.

Às vezes, o verificador de lint solicita a criação da compilação de lançamento. Para resolver esse problema, siga as opções abaixo.

Primeiro:

Se você quiser desabilitar a tradução apenas para algumas strings, adicione o seguinte atributo ao string.xml padrão

<string name="developer" translatable="false">Developer Name</string>

Segundo:

Ignore todas as traduções ausentes do arquivo de recurso, adicione o seguinte atributo É o atributo ignore do namespace tools em seu arquivo de strings, da seguinte forma:

<?xml version="1.0" encoding="utf-8"?>
<resources
  xmlns:tools="http://schemas.android.com/tools"
  tools:ignore="MissingTranslation" >
https://www.wikiod.com/pt/android/localizacao-com-recursos-no-android
  <!-- your strings here; no need now for the translatable attribute -->

</resources>

Terceiro:

Outra maneira de desativar a string não traduzível

http://tools.android.com/recent/non-translatablestrings

Se você tiver muitos recursos que não devem ser traduzidos, você pode colocá-los em um arquivo chamado donottranslate.xml e o lint considerará todos eles recursos não traduzíveis.

Quarto:

Você também pode adicionar localidade no arquivo de recurso

<resources
xmlns:tools="http://schemas.android.com/tools"
    tools:locale="en" tools:ignore="MissingTranslation">

Você também pode desabilitar a verificação de tradução ausente para lint de app/build.gradle

lintOptions {
        
        disable 'MissingTranslation'
    }
