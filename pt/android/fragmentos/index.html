Introdução sobre Fragmentos e seu mecanismo de intercomunicação

Sintaxe #

  • void onActivityCreated(Bundle savedInstanceState) // Chamado quando a atividade do fragmento foi criada e a hierarquia de visão deste fragmento instanciada.

  • void onActivityResult(int requestCode, int resultCode, Intent data) // Recebe o resultado de uma chamada anterior para startActivityForResult(Intent, int).

  • void onAttach(Activity activity) // Este método foi descontinuado no nível 23 da API. Use onAttach(Context) em vez disso.

  • void onAttach(Context context) // Chamado quando um fragmento é anexado pela primeira vez ao seu contexto.

  • void onAttachFragment(Fragment childFragment) // Chamado quando um fragmento é anexado como filho deste fragmento.

  • void onConfigurationChanged(Configuration newConfig) // Chamado pelo sistema quando a configuração do dispositivo muda enquanto seu componente está em execução.

  • void onCreate(Bundle savedInstanceState) // Chamado para fazer a criação inicial de um fragmento.

  • View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) // Chamado para que o fragmento instancie sua visualização da interface do usuário.

  • void onDestroy() // Chamado quando o fragmento não está mais em uso.

  • void onDestroyView() // Chamado quando a visão criada anteriormente por onCreateView(LayoutInflater, ViewGroup, Bundle) foi desanexada do fragmento.

  • void onDetach() // Chamado quando o fragmento não está mais anexado à sua atividade.

  • void onInflate(Activity activity, AttributeSet attrs, Bundle savedInstanceState) // Este método foi descontinuado na API de nível 23. Use onInflate(Context, AttributeSet, Bundle) em vez disso.

  • void onInflate(Context context, AttributeSet attrs, Bundle savedInstanceState) // Chamado quando um fragmento está sendo criado como parte de uma inflação de layout de visualização, normalmente ao definir a visualização de conteúdo de uma atividade.

  • void onPause() // Chamado quando o Fragment não é mais retomado.

  • void onResume() // Chamado quando o fragmento está visível para o usuário e em execução ativa.

  • void onSaveInstanceState(Bundle outState) // Chamado para solicitar que o fragmento salve seu estado dinâmico atual, para que posteriormente possa ser reconstruído em uma nova instância de seu processo é reiniciado.

  • void onStart() // Chamado quando o Fragment está visível para o usuário.

  • void onStop() // Chamado quando o Fragment não é mais iniciado.

  • void onViewStateRestored(Bundle savedInstanceState) // Chamado quando todo o estado salvo foi restaurado na hierarquia de exibição do fragmento.

Um Fragment representa um comportamento ou uma parte da interface do usuário em uma Activity. Você pode combinar vários fragmentos em uma única atividade para criar uma interface do usuário de vários painéis e reutilizar um fragmento em várias atividades. Você pode pensar em um fragmento como uma seção modular de uma atividade, que tem seu próprio ciclo de vida, recebe seus próprios eventos de entrada e que você pode adicionar ou remover enquanto a atividade está em execução (como uma “subatividade” que você pode reutilização em diferentes atividades).

Construtor

Todo fragmento deve ter um construtor vazio, para que possa ser instanciado ao restaurar o estado de sua atividade. É altamente recomendável que as subclasses não tenham outros construtores com parâmetros, pois esses construtores não serão chamados quando o fragmento for reinstanciado; em vez disso, os argumentos podem ser fornecidos pelo chamador com setArguments(Bundle) e posteriormente recuperados pelo Fragment com getArguments().

Passar dados de Activity para Fragment usando Bundle #

Todos os fragmentos devem ter um construtor vazio (ou seja, um método construtor sem argumentos de entrada). Portanto, para passar seus dados para o Fragment que está sendo criado, você deve usar o método setArguments(). Este método obtém um bundle, no qual você armazena seus dados, e armazena o Bundle nos argumentos. Subsequentemente, este Bundle pode ser recuperado em onCreate() e onCreateView() call backs do Fragment.

Atividade:

 Bundle bundle = new Bundle();
 String myMessage = "Stack Overflow is cool!";
 bundle.putString("message", myMessage );
 FragmentClass fragInfo = new FragmentClass();
 fragInfo.setArguments(bundle);
 transaction.replace(R.id.fragment_single, fragInfo);
 transaction.commit();

Fragmento:

 @Override
 public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
    String myValue = this.getArguments().getString("message");
    ...
 }

O padrão newInstance() #

Embora seja possível criar um construtor de fragmento com parâmetros, o Android chama internamente o construtor de argumento zero ao recriar fragmentos (por exemplo, se eles estiverem sendo restaurados após serem eliminados por motivos próprios do Android). Por esse motivo, não é aconselhável contar com um construtor que tenha parâmetros.

Para garantir que seus argumentos de fragmento esperados estejam sempre presentes, você pode usar um método estático newInstance() para criar o fragmento e colocar quaisquer parâmetros que desejar em um pacote que estará disponível ao criar uma nova instância.

import android.os.Bundle;
import android.support.v4.app.Fragment;

public class MyFragment extends Fragment
{
  // Our identifier for obtaining the name from arguments
  private static final String NAME_ARG = "name";

  private String mName;

  // Required
  public MyFragment(){}

  // The static constructor.  This is the only way that you should instantiate
  // the fragment yourself
  public static MyFragment newInstance(final String name) {
    final MyFragment myFragment = new MyFragment();
    // The 1 below is an optimization, being the number of arguments that will
    // be added to this bundle.  If you know the number of arguments you will add
    // to the bundle it stops additional allocations of the backing map.  If
    // unsure, you can construct Bundle without any arguments
    final Bundle args = new Bundle(1);

    // This stores the argument as an argument in the bundle.  Note that even if
    // the 'name' parameter is NULL then this will work, so you should consider
    // at this point if the parameter is mandatory and if so check for NULL and
    // throw an appropriate error if so
    args.putString(NAME_ARG, name);

    myFragment.setArguments(args);
    return myFragment;
  }

  @Override
  public void onCreate(final Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    final Bundle arguments = getArguments();
    if (arguments == null || !arguments.containsKey(NAME_ARG)) {
      // Set a default or error as you see fit
    } else {
      mName = arguments.getString(NAME_ARG);
    }
  }
}

Agora, na atividade:

FragmentTransaction ft = getSupportFragmentManager().beginTransaction();
MyFragment mFragment = MyFragment.newInstance("my name");
ft.replace(R.id.placeholder, mFragment);
//R.id.placeholder is where we want to load our fragment
ft.commit();

Esse padrão é uma prática recomendada para garantir que todos os argumentos necessários sejam passados ​​para os fragmentos na criação. Observe que quando o sistema destruir o fragmento e recriá-lo posteriormente, ele restaurará automaticamente seu estado - mas você deve fornecer uma implementação onSaveInstanceState(Bundle).

Navegação entre fragmentos usando backstack e padrão de tecido estático #

Antes de tudo, precisamos adicionar nosso primeiro Fragment no início, devemos fazê-lo no método onCreate() da nossa Activity:

if (null == savedInstanceState) {
    getSupportFragmentManager().beginTransaction()
      .addToBackStack("fragmentA")
      .replace(R.id.container, FragmentA.newInstance(), "fragmentA")
      .commit();
}

Em seguida, precisamos gerenciar nosso backstack. A maneira mais fácil é usar uma função adicionada em nossa atividade que é usada para todas as FragmentTransactions.

public void replaceFragment(Fragment fragment, String tag) {
    //Get current fragment placed in container
    Fragment currentFragment = getSupportFragmentManager().findFragmentById(R.id.container);

    //Prevent adding same fragment on top
    if (currentFragment.getClass() == fragment.getClass()) {
        return;
    }

    //If fragment is already on stack, we can pop back stack to prevent stack infinite growth
    if (getSupportFragmentManager().findFragmentByTag(tag) != null) {
        getSupportFragmentManager().popBackStack(tag, FragmentManager.POP_BACK_STACK_INCLUSIVE);
    }

    //Otherwise, just replace fragment
    getSupportFragmentManager()
            .beginTransaction()
            .addToBackStack(tag)
            .replace(R.id.container, fragment, tag)
            .commit();
}

Finalmente, devemos substituir onBackPressed() para sair do aplicativo ao voltar do último Fragment disponível no backstack.

@Override
public void onBackPressed() {
    int fragmentsInStack = getSupportFragmentManager().getBackStackEntryCount();
    if (fragmentsInStack > 1) { // If we have more than one fragment, pop back stack
        getSupportFragmentManager().popBackStack();
    } else if (fragmentsInStack == 1) { // Finish activity, if only one fragment left, to prevent leaving empty screen
        finish();
    } else {
        super.onBackPressed();
    }
}

Execução em atividade:

replaceFragment(FragmentB.newInstance(), "fragmentB");

Execução fora da atividade (assumindo que MainActivity é nossa atividade):

((MainActivity) getActivity()).replaceFragment(FragmentB.newInstance(), "fragmentB");

Enviando eventos de volta para uma atividade com interface de retorno de chamada #

Se você precisar enviar eventos do fragmento para a atividade, uma das soluções possíveis é definir a interface de retorno de chamada e exigir que a atividade do host a implemente.

Exemplo

Enviar retorno de chamada para uma atividade, quando o botão do fragmento for clicado

Antes de tudo, defina a interface de retorno de chamada: 
public interface SampleCallback {
    void onButtonClicked();
}

A próxima etapa é atribuir esse retorno de chamada no fragmento:

public final class SampleFragment extends Fragment {

    private SampleCallback callback;

    @Override
    public void onAttach(Context context) {
        super.onAttach(context);
        if (context instanceof SampleCallback) {
            callback = (SampleCallback) context;
        } else {
            throw new RuntimeException(context.toString()
                    + " must implement SampleCallback");
        }
    }

    @Override
    public void onDetach() {
        super.onDetach();
        callback = null;
    }

    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
        final View view = inflater.inflate(R.layout.sample, container, false);
        // Add button's click listener
        view.findViewById(R.id.actionButton).setOnClickListener(new View.OnClickListener() {
            public void onClick(View v) {
                callback.onButtonClicked(); // Invoke callback here
            }
        });
        return view;
    }
}

E, finalmente, implemente o callback na atividade:

public final class SampleActivity extends Activity implements SampleCallback {

    // ... Skipped code with settings content view and presenting the fragment

    @Override
    public void onButtonClicked() {
        // Invoked when fragment's button has been clicked
    }
}

Animar a transição entre fragmentos #

Para animar a transição entre fragmentos, ou para animar o processo de mostrar ou ocultar um fragmento, você usa o FragmentManager para criar uma FragmentTransaction.

Para uma única FragmentTransaction, existem duas maneiras diferentes de realizar animações: você pode usar uma animação padrão ou fornecer suas próprias animações personalizadas.

As animações padrão são especificadas chamando FragmentTransaction.setTransition(int transit) e usando uma das constantes predefinidas disponíveis na classe FragmentTransaction. No momento da escrita, essas constantes são:

FragmentTransaction.TRANSIT_NONE
FragmentTransaction.TRANSIT_FRAGMENT_OPEN
FragmentTransaction.TRANSIT_FRAGMENT_CLOSE
FragmentTransaction.TRANSIT_FRAGMENT_FADE

A transação completa pode ser algo assim:

getSupportFragmentManager()
    .beginTransaction()
    .setTransition(FragmentTransaction.TRANSIT_FRAGMENT_FADE)
    .replace(R.id.contents, new MyFragment(), "MyFragmentTag")
    .commit();

As animações personalizadas são especificadas chamando FragmentTransaction.setCustomAnimations(int enter, int exit) ou FragmentTransaction.setCustomAnimations(int enter, int exit, int popEnter, int popExit).

As animações enter e exit serão reproduzidas para FragmentTransactions que não envolvem a remoção de fragmentos da pilha de retorno. As animações popEnter e popExit serão reproduzidas ao retirar um fragmento da pilha de volta.

O código a seguir mostra como você substituiria um fragmento deslizando um fragmento para fora e deslizando o outro em seu lugar.

getSupportFragmentManager()
    .beginTransaction()
    .setCustomAnimations(R.anim.slide_in_left, R.anim.slide_out_right)
    .replace(R.id.contents, new MyFragment(), "MyFragmentTag")
    .commit();

As definições de animação XML usariam a tag objectAnimator. Um exemplo de slide_in_left.xml pode ser algo assim:

<?xml version="1.0" encoding="utf-8"?>
<set>
  <objectAnimator xmlns:android="http://schemas.android.com/apk/res/android"
    android:propertyName="x" 
    android:valueType="floatType"
    android:valueFrom="-1280"
    android:valueTo="0" 
    android:duration="500"/>
</set>

Comunicação entre Fragmentos #

Todas as comunicações entre Fragments devem passar por uma Activity. Fragmentos NÃO PODEM se comunicar uns com os outros sem uma atividade.

Recursos adicionais

  • https://stackoverflow.com/questions/24777985/how-to-implement-onfragmentinteractionlistener
  • Android | Comunicação com outros fragmentos

Neste exemplo, temos um MainActivity que hospeda dois fragmentos, SenderFragment e ReceiverFragment, para enviar e receber uma mensagem (uma String simples neste caso), respectivamente.

Um botão em SenderFragment inicia o processo de envio da mensagem. Um TextView no ReceiverFragment é atualizado quando a mensagem é recebida por ele.

A seguir está o trecho da MainActivity com comentários explicando as importantes linhas de código:

// Our MainActivity implements the interface defined by the SenderFragment. This enables
// communication from the fragment to the activity
public class MainActivity extends AppCompatActivity implements SenderFragment.SendMessageListener {

@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
}

/**
 * This method is called when we click on the button in the SenderFragment
 * @param message The message sent by the SenderFragment
 */
@Override
public void onSendMessage(String message) {
    // Find our ReceiverFragment using the SupportFragmentManager and the fragment's id
    ReceiverFragment receiverFragment = (ReceiverFragment)
            getSupportFragmentManager().findFragmentById(R.id.fragment_receiver);

    // Make sure that such a fragment exists
    if (receiverFragment != null) {
        // Send this message to the ReceiverFragment by calling its public method
        receiverFragment.showMessage(message);
    }
}
}

O SenderFragment expõe uma interface SendMessageListener que ajuda o MainActivity a saber quando o botão no SenderFragment foi clicado.

A seguir está o trecho de código para o SenderFragment explicando as linhas de código importantes:

public class SenderFragment extends Fragment {

private SendMessageListener commander;

/**
 * This interface is created to communicate between the activity and the fragment. Any activity
 * which implements this interface will be able to receive the message that is sent by this
 * fragment.
 */
public interface SendMessageListener {
    void onSendMessage(String message);
}

/**
 * API LEVEL >= 23
 * <p>
 * This method is called when the fragment is attached to the activity. This method here will
 * help us to initialize our reference variable, 'commander' , for our interface
 * 'SendMessageListener'
 *
 * @param context
 */
@Override
public void onAttach(Context context) {
    super.onAttach(context);
    // Try to cast the context to our interface SendMessageListener i.e. check whether the
    // activity implements the SendMessageListener. If not a ClassCastException is thrown.
    try {
        commander = (SendMessageListener) context;
    } catch (ClassCastException e) {
        throw new ClassCastException(context.toString()
                + "must implement the SendMessageListener interface");
    }
}

/**
 * API LEVEL < 23
 * <p>
 * This method is called when the fragment is attached to the activity. This method here will
 * help us to initialize our reference variable, 'commander' , for our interface
 * 'SendMessageListener'
 *
 * @param activity
 */
@Override
public void onAttach(Activity activity) {
    super.onAttach(activity);
    // Try to cast the context to our interface SendMessageListener i.e. check whether the
    // activity implements the SendMessageListener. If not a ClassCastException is thrown.
    try {
        commander = (SendMessageListener) activity;
    } catch (ClassCastException e) {
        throw new ClassCastException(activity.toString()
                + "must implement the SendMessageListener interface");
    }
}

@Nullable
@Override
public View onCreateView(LayoutInflater inflater, @Nullable ViewGroup container,
                         @Nullable Bundle savedInstanceState) {
    // Inflate view for the sender fragment.
    View view = inflater.inflate(R.layout.fragment_receiver, container, false);

    // Initialize button and a click listener on it
    Button send = (Button) view.findViewById(R.id.bSend);
    send.setOnClickListener(new View.OnClickListener() {
        @Override
        public void onClick(View v) {

            // Sanity check whether we were able to properly initialize our interface reference
            if (commander != null) {

                // Call our interface method. This enables us to call the implemented method
                // in the activity, from where we can send the message to the ReceiverFragment.
                commander.onSendMessage("HELLO FROM SENDER FRAGMENT!");
            }
        }
    });

    return view;
}
}

public class ReceiverFragment extends Fragment {

    View view = inflater.inflate(R.layout.fragment_receiver, container, false);

public void showMessage(String message) {
    tvMessage.setText(message);
}
}
