Android: armazenamento interno e externo - Esclarecimento de terminologia #

Os desenvolvedores do Android (principalmente iniciantes) ficaram confusos em relação à terminologia de armazenamento interno e externo. Há muitas perguntas no Stackoverflow sobre o mesmo. Isso se deve principalmente ao fato de que a terminologia de acordo com o Google/documentação oficial do Android é bem diferente da do usuário normal do sistema operacional Android. Por isso, pensei que documentar isso ajudaria.

O que pensamos - Terminologia do usuário (UT)

Armazenamento interno (UT) Armazenamento externo (UT)
memória interna interna do telefone cartão removível Secure Digital(SD) ou armazenamento micro SD
Exemplo: memória interna de 32 GB do Nexus 6P. Exemplo: espaço de armazenamento em cartões SD removíveis fornecidos por fornecedores como samsung, sandisk, strôncio, transcend e outros

Mas, De acordo com a documentação/guia do Android - Terminologia do Google (GT)

Armazenamento interno (GT):

Por padrão, os arquivos salvos no armazenamento interno são privados do seu aplicativo e outros aplicativos não podem acessá-los (nem o usuário).

Armazenamento externo (GT):

Pode ser uma mídia de armazenamento removível (como um cartão SD) ou um armazenamento interno (não removível).

O armazenamento externo (GT) pode ser categorizado em dois tipos:

Armazenamento externo primário Armazenamento externo secundário ou armazenamento removível (GT)
Isso é o mesmo que a memória interna embutida do telefone (ou) Armazenamento interno (UT) Isso é o mesmo que armazenamento em cartão micro SD removível (ou) Armazenamento externo (UT)
Exemplo: memória interna de 32 GB do Nexus 6P. Exemplo: espaço de armazenamento em cartões SD removíveis fornecidos por fornecedores como samsung, sandisk, strôncio, transcend e outros
Este tipo de armazenamento pode ser acessado no Windows PC conectando seu telefone ao PC via cabo USB e selecionando Câmera(PTP) na notificação de opções USB. Este tipo de armazenamento pode ser acessado no Windows PC conectando seu telefone ao PC via cabo USB e selecionando File transfer na notificação de opções USB.

Em poucas palavras,

Armazenamento Externo (GT) = Armazenamento Interno (UT) e Armazenamento Externo (UT)

Armazenamento removível (GT) = Armazenamento externo (UT)

Armazenamento interno (GT) não tem um termo em UT.

Deixe-me explicar claramente,

Armazenamento interno (GT): Por padrão, os arquivos salvos no armazenamento interno são privados do seu aplicativo e outros aplicativos não podem acessá-los. O usuário do seu aplicativo também não pode acessá-los usando o gerenciador de arquivos; mesmo depois de habilitar a opção “mostrar arquivos ocultos” no gerenciador de arquivos. Para acessar arquivos no armazenamento interno (GT), você precisa fazer root no seu telefone Android. Além disso, quando o usuário desinstala seu aplicativo, esses arquivos são removidos/excluídos.

Portanto, armazenamento interno (GT) é NÃO o que pensamos como memória interna de 32/64 GB do Nexus 6P

Geralmente, Local de armazenamento interno (GT) seria: /data/data/your.application.package.appname/someDirectory/

Armazenamento Externo (GT):

Todo dispositivo compatível com Android suporta um “armazenamento externo” compartilhado que você pode usar para salvar arquivos. Os arquivos salvos no armazenamento externo podem ser lidos mundialmente e podem ser modificados pelo usuário quando eles habilitam o armazenamento em massa USB para transferir arquivos em um computador.

Local de armazenamento externo (GT): pode estar em qualquer lugar em seu armazenamento interno (UT) ou em seu armazenamento removível (GT), ou seja, cartão micro SD. Depende do OEM do seu telefone e também da versão do sistema operacional Android.

Para ler ou gravar arquivos no Armazenamento Externo (GT), seu aplicativo deve adquirir as permissões de sistema READ_EXTERNAL_STORAGE ou WRITE_EXTERNAL_STORAGE.

Por exemplo:

<manifest ...>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
    ...
</manifest>

Se você precisar ler e gravar arquivos, precisará solicitar apenas a permissão WRITE_EXTERNAL_STORAGE, porque implicitamente também requer acesso de leitura.

No External Storage(GT), você também pode salvar arquivos que são app-private

Mas,

Quando o usuário desinstala seu aplicativo, esse diretório e todo o seu conteúdo são excluídos.

Quando você precisa salvar arquivos privados do aplicativo no Armazenamento Externo (GT)?

Se você estiver manipulando arquivos que não são destinados ao uso de outros aplicativos (como texturas gráficas ou efeitos sonoros usados ​​apenas pelo seu aplicativo), use um diretório de armazenamento privado no armazenamento externo

A partir do Android 4.4, ler ou gravar arquivos nos diretórios privados do seu aplicativo não requer as permissões READ_EXTERNAL_STORAGE ou WRITE_EXTERNAL_STORAGE. Assim, você pode declarar que a permissão deve ser solicitada apenas nas versões inferiores do Android adicionando o atributo maxSdkVersion:

<manifest ...>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"
                     android:maxSdkVersion="18" />
    ...
</manifest

Métodos para armazenar no armazenamento interno (GT):

Ambos os métodos estão presentes na classe Context

File getDir (String name, int mode)

File getFilesDir ()

Métodos para armazenar no armazenamento externo primário, ou seja, armazenamento interno (UT):

File getExternalStorageDirectory ()

File getExternalFilesDir (String type)

File getExternalStoragePublicDirectory (String type)

No início, todos usavam Environment.getExternalStorageDirectory() , que apontava para a raiz de Primary External Storage. Como resultado, o armazenamento externo primário foi preenchido com conteúdo aleatório.

Mais tarde, esses dois métodos foram adicionados:

  1. Na classe Context, eles adicionaram getExternalFilesDir(), apontando para um diretório específico do aplicativo no armazenamento externo primário. Este diretório e seu conteúdo serão excluídos quando o aplicativo for desinstalado.

  2. Environment.getExternalStoragePublicDirectory() para locais centralizados para armazenar tipos de arquivos conhecidos, como fotos e filmes. Este diretório e seu conteúdo NÃO serão excluídos quando o aplicativo for desinstalado.

Métodos para armazenar em Armazenamento Removível (GT), ou seja, cartão micro SD

Antes do API de nível 19, não havia uma maneira oficial de armazenar no cartão SD. Mas, muitos poderiam fazer isso usando bibliotecas ou APIs não oficiais.

Oficialmente, um método foi introduzido na classe Context na API nível 19 (Android versão 4.4 - Kitkat).

File[] getExternalFilesDirs (String type)

Ele retorna caminhos absolutos para diretórios específicos do aplicativo em todos os dispositivos de armazenamento compartilhados/externos onde o aplicativo pode colocar arquivos persistentes que possui. Esses arquivos são internos ao aplicativo, e normalmente não é visível para o usuário como mídia.

Isso significa que ele retornará caminhos para ambos tipos de armazenamento externo (GT) - memória interna e cartão Micro SD. Geralmente o segundo caminho seria o caminho de armazenamento do cartão micro SD (mas nem sempre). Então você precisa verificar executando o código com este método.

Usando armazenamento externo #

O armazenamento “externo” é outro tipo de armazenamento que podemos usar para salvar arquivos no dispositivo do usuário. Ele tem algumas diferenças importantes do armazenamento “interno”, a saber:

  • Nem sempre está disponível. No caso de uma mídia removível (cartão SD), o usuário pode simplesmente remover o armazenamento.
  • Não é privado. O usuário (e outros aplicativos) têm acesso a esses arquivos.
  • Se o usuário desinstalar o aplicativo, os arquivos salvos no diretório recuperado com getExternalFilesDir() serão removidos.

Para usar o armazenamento externo, precisamos primeiro obter as permissões adequadas. Você precisará usar:

  • android.permission.WRITE_EXTERNAL_STORAGE para leitura e escrita
  • android.permission.READ_EXTERNAL_STORAGE apenas para leitura

Para conceder essas permissões, você precisará identificá-las em seu AndroidManifest.xml como tal

<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />

OBSERVAÇÃO: Como são permissões perigosas se você estiver usando API de nível 23 ou superior, será necessário solicitar as permissões em tempo de execução.

Antes de tentar gravar ou ler do Armazenamento Externo, você deve sempre verificar se a mídia de armazenamento está disponível.

String state = Environment.getExternalStorageState();
if (state.equals(Environment.MEDIA_MOUNTED)) {
    // Available to read and write
}
if (state.equals(Environment.MEDIA_MOUNTED) || 
        state.equals(Environment.MEDIA_MOUNTED_READ_ONLY)) {
    // Available to at least read
}

Ao gravar arquivos no Armazenamento Externo, você deve decidir se o arquivo deve ser reconhecido como Público ou Privado. Embora esses dois tipos de arquivos ainda estejam acessíveis ao usuário e outros aplicativos no dispositivo, há uma distinção importante entre eles.

Os arquivos públicos devem permanecer no dispositivo quando o usuário desinstalar o aplicativo. Um exemplo de arquivo que deve ser salvo como Público seriam as fotos tiradas por meio do seu aplicativo.

Todos os arquivos privados devem ser removidos quando o usuário desinstalar o aplicativo. Esses tipos de arquivos seriam específicos do aplicativo e não seriam úteis para o usuário ou outros aplicativos. Ex. arquivos temporários baixados/usados ​​por seu aplicativo.

Veja como obter acesso ao diretório Documents para arquivos públicos e privados.

Público

// Access your app's directory in the device's Public documents directory
File docs = new File(Environment.getExternalStoragePublicDirectory(
        Environment.DIRECTORY_DOCUMENTS), "YourAppDirectory");
// Make the directory if it does not yet exist
myDocs.mkdirs();

Privado

// Access your app's Private documents directory
File file = new File(context.getExternalFilesDir(Environment.DIRECTORY_DOCUMENTS), 
        "YourAppDirectory");
// Make the directory if it does not yet exist
myDocs.mkdirs();

Por padrão, todos os arquivos salvos no armazenamento interno são privados do seu aplicativo. Eles não podem ser acessados ​​por outros aplicativos, nem pelo usuário em circunstâncias normais. Esses arquivos são excluídos quando o usuário desinstala o aplicativo.

Buscar Diretório de Dispositivos: #

Primeira permissão para adicionar armazenamento para ler/buscar o diretório do dispositivo.

<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />

Criar classe de modelo

//create one directory model class
//to store directory title and type in list

public class DirectoryModel {
    String dirName;
    int dirType; // set 1 or 0, where 0 for directory and 1 for file.

    public int getDirType() {
        return dirType;
    }

    public void setDirType(int dirType) {
        this.dirType = dirType;
    }

    public String getDirName() {
        return dirName;
    }

    public void setDirName(String dirName) {
        this.dirName = dirName;
    }
}

Crie uma lista usando o modelo de diretório para adicionar dados de diretório.

//define list to show directory

List<DirectoryModel> rootDir = new ArrayList<>();

Busque o diretório usando o método a seguir.

//to fetch device directory

private void getDirectory(String currDir) { // pass device root directory
        File f = new File(currDir);
        File[] files = f.listFiles();
        if (files != null) {
            if (files.length > 0) {
                rootDir.clear();
                for (File inFile : files) {
                    if (inFile.isDirectory()) { //return true if it's directory
                        // is directory
                        DirectoryModel dir = new DirectoryModel();
                        dir.setDirName(inFile.toString().replace("/storage/emulated/0", ""));
                        dir.setDirType(0); // set 0 for directory
                        rootDir.add(dir);
                    } else if (inFile.isFile()) { // return true if it's file
                        //is file
                        DirectoryModel dir = new DirectoryModel();
                        dir.setDirName(inFile.toString().replace("/storage/emulated/0", ""));
                        dir.setDirType(1); // set 1 for file
                        rootDir.add(dir);
                    }
                }
            }
            printDirectoryList();
        }
    }

Imprimir lista de diretórios no log.

//print directory list in logs

private void printDirectoryList() {
    for (int i = 0; i < rootDir.size(); i++) {
        Log.e(TAG, "printDirectoryLogs: " + rootDir.get(i).toString());
    }
}

Uso

//to Fetch Directory Call function with root directory.

String rootPath = Environment.getExternalStorageDirectory().toString(); // return ==>  /storage/emulated/0/
