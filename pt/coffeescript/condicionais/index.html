se, se / então, se / senão, a menos, operador ternário #

A instância mais básica de uma construção if avalia uma condição e executa algum código de acordo com o resultado da condição. Se a condição retornar true, o código dentro da condicional será executado.

counter = 10
if counter is 10
  console.log 'This will be executed!'

A construção if pode ser enriquecida com uma instrução else. O código dentro da instrução else será executado sempre que a condição if não for atendida.

counter = 9
if counter is 10
  console.log 'This will not be executed...'
else
  console.log '... but this one will!'

Construções if podem ser encadeadas usando else, sem qualquer limitação de quantos podem ser encadeados. A primeira condicional que retornar true executará seu código e interromperá a verificação: nenhuma condicional abaixo desse ponto será avaliada posteriormente e nenhum bloco de código de dentro dessas condicionais será executado.

if counter is 10
  console.log 'I counted to 10'
else if counter is 9
  console.log 'I counted to 9'
else if counter < 7
  console.log 'Not to 7 yet'
else
  console.log 'I lost count'

A forma oposta de if é unless. Ao contrário de if, unless só será executado se a condicional retornar false.

counter = 10
unless counter is 10
  console.log 'This will not be executed!

As instruções if podem ser colocadas em uma única linha, mas neste caso, a palavra-chave then é necessária.

if counter is 10 then console.log 'Counter is 10'

Uma sintaxe alternativa é semelhante ao Ruby:

console.log 'Counter is 10' if counter is 10

Os dois últimos blocos de código são equivalentes.

O operador ternário é uma compressão de uma construção if/then/else, e pode ser usado ao atribuir valores a variáveis. O valor final atribuído à variável será aquele definido após o then quando a condição if for atendida. Caso contrário, o valor após o else será atribuído.

outcome = if counter is 10 then 'Done counting!' else 'Still counting'

TL; DR: As instruções switch do CoffeeScript usam when para cada caso e else para o caso padrão. Eles usam then para casos de uma linha e vírgulas para vários casos com um único resultado. Eles intencionalmente não permitem fallthrough e, portanto, não precisam de um break explícito (já que está sempre lá implicitamente). Uma instrução switch pode ser usada como uma expressão atribuível e retornável.

As instruções switch do CoffeeScript são um tipo de instrução de controle que permite que você execute ações diferentes com base em um valor. Eles são como instruções if, mas onde uma instrução if geralmente executa uma das duas ações com base em se algo é true ou false, instruções switch executam uma de qualquer número de ações, dependendo do valor de qualquer expressão - uma string, número ou qualquer coisa.

CoffeeScript switch começa com a palavra-chave switch seguida pela expressão para ligar. Então, cada case é representado pela palavra-chave when seguida pelo valor para aquele case.

switch name
  when "Alice"
    # Code here will run when name is Alice
    callAlice()
  when "Bob"
    # Code here will run when name is Bob
    giveBobSandwich()

Há também uma sintaxe abreviada para quando cada caso é uma linha, usando a palavra-chave then em vez de uma nova linha:

livesLeft = 2
switch livesLeft
  when 3 then fullHealth()
  when 2 then healthAt 2
  when 1 then healthAt 1
  when 0 then playerDie()

Você pode misturar e combinar os dois formatos conforme necessário:

livesLeft = 2
switch livesLeft
  when 3 then fullHealth()
  when 2 then healthAt 2
  when 1
    healthAt 1
    alert "Warning! Health low!"
  when 0 then playerDie()

Embora as coisas mais comuns para ativar sejam uma variável (como no exemplo anterior) ou o resultado de uma função, você pode ativar qualquer expressão que escolher:

indexOfAnswer = 0
switch indexOfAnswer + 1
  when 1 then console.log "The answer is the 1st item"
  when 2 then console.log "The answer is the 2nd item"
  when 3 then console.log "The answer is the 3rd item"

Você também pode fazer com que vários casos levem à mesma ação:

switch password
  when "password", "123456", "letmein" then console.log "Wrong!"
  when "openpoppyseed" then console.log "Close, but no cigar."
  when "opensesame" then console.log "You got it!"

Um recurso muito útil é um caso padrão ou catch-all, que só será executado se nenhum dos outros critérios for atendido. CoffeeScript significa isso com a palavra-chave else:

switch password
  when "password", "123456", "letmein" then console.log "Wrong!"
  when "openpoppyseed" then console.log "Close, but no cigar."
  when "opensesame" then console.log "You got it!"
  else console.log "Not even close..."

(Observe que você não precisa da palavra-chave then para o caso else porque não há condição.)

Agora aqui está um exemplo de todos os recursos do switch em ação!

switch day
  when "Mon" then go work
  when "Tue" then go relax
  when "Thu" then go iceFishing
  when "Fri", "Sat"
    if day is bingoDay
      go bingo
      go dancing
  when "Sun" then go church
  else go work

Você também pode ter a condição de um caso como uma expressão:

switch fullName
  when myFullName() then alert "Doppelgänger detected"
  when presidentFirstName + " " + presidentLastName
    alert "Get down Mr. president!"
    callSecretService()
  when "Joey Bonzo" then alert "Joey Bonzo everybody"

As instruções switch do CoffeeScript também têm uma característica única: elas podem retornar valores como uma função. Se você atribuir uma variável a uma instrução switch, ela será atribuída a qualquer coisa que a instrução retornar.

address = switch company
  when "Apple" then "One Infinite Loop"
  when "Google" then "1600 Amphitheatre Parkway"
  when "ACME"
    if isReal
      "31918 Hayman St"
    else
      "Unknown desert location"
  else lookUpAddress company

(Lembre-se de que a última instrução em um bloco é retornada implicitamente. Você também pode usar a palavra-chave return manualmente.)

As instruções switch também podem ser usadas sem uma expressão de controle, transformando-as em uma alternativa mais limpa às cadeias if/else.

score = 76
grade = switch
  when score < 60 then 'F'
  when score < 70 then 'D'
  when score < 80 then 'C'
  when score < 90 then 'B'
  else 'A'

(Isto é funcionalmente equivalente a grade = switch true porque o primeiro caso avaliado como true corresponderá. No entanto, como cada caso implicitamente breaks no final, apenas o primeiro caso a ser correspondido será executado.)
