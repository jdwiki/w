Atualizando os dados: um exemplo básico de seleções de entrada, atualização e saída #

Criar um gráfico que exiba um conjunto de dados estático é relativamente simples. Por exemplo, se tivermos esse array de objetos como os dados:

var data = [
    {title: "A", value: 53},
    {title: "B", value: 12},
    {title: "C", value: 91},
    {title: "D", value: 24},
    {title: "E", value: 59}
];

Podemos criar um gráfico de barras onde cada barra representa uma medida, denominada “título”, e sua largura representa o valor dessa medida. Como esse conjunto de dados não muda, nosso gráfico de barras tem apenas uma seleção “enter”:

var bars = svg.selectAll(".bars")
    .data(data);

bars.enter()
    .append("rect")
    .attr("class", "bars")
    .attr("x", xScale(0))
    .attr("y", function(d){ return yScale(d.title)})
    .attr("width", 0)
    .attr("height", yScale.bandwidth())
    .transition()
    .duration(1000)
    .delay(function(d,i){ return i*200})
    .attr("width", function(d){ return xScale(d.value) - margin.left});

Aqui, estamos definindo a largura de cada barra para 0 e, após a transição, para seu valor final.

Esta seleção de entrada, por si só, é suficiente para criar nosso gráfico, que você pode ver neste violino.

Mas e se meus dados mudarem?

Nesse caso, temos que alterar dinamicamente nosso gráfico. A melhor maneira de fazer isso é criar uma seleção de “entrar”, “atualizar” e “sair”. Mas, antes disso, temos que fazer algumas alterações no código.

Primeiro, vamos mover as partes que mudam dentro de uma função chamada draw():

function draw(){
    //changing parts
};

Essas “partes em mudança” incluem:

  1. As seleções de entrada, atualização e saída;
  2. O domínio de cada escala;
  3. A transição do eixo;

Dentro dessa função draw(), chamamos outra função, que cria nossos dados. Aqui, é apenas uma função que retorna um array de 5 objetos, escolhendo aleatoriamente 5 letras de 10 (ordenando alfabeticamente) e, para cada um, um valor entre 0 e 99:

function getData(){
    var title = "ABCDEFGHIJ".split("");
    var data = [];
    for(var i = 0; i < 5; i++){
        var index = Math.floor(Math.random()*title.length);
        data.push({title: title[index],
            value: Math.floor(Math.random()*100)});
        title.splice(index,1);
    }
    data = data.sort(function(a,b){ return d3.ascending(a.title,b.title)});
    return data;
};

E agora, vamos passar para nossas seleções. Mas antes disso, uma palavra de cautela: para manter o que chamamos de constância do objeto, temos que especificar uma função chave como segundo argumento para selection.data:

var bars = svg.selectAll(".bars")
    .data(data, function(d){ return d.title});

Sem isso, nossas barras não farão uma transição suave e seria difícil acompanhar as mudanças no eixo (você pode ver isso removendo o segundo argumento no violino abaixo).

Então, depois de configurar corretamente nossas var bars, podemos lidar com nossas seleções. Esta é a seleção de saída:

bars.exit()
    .transition()
    .duration(1000)
    .attr("width", 0)
    .remove();

E estas são as seleções de entrada e atualização (no D3 v4.x, a seleção de atualização é mesclada com a seleção de entrada usando merge):

bars.enter()//this is the enter selection
    .append("rect")
    .attr("class", "bars")
    .attr("x", xScale(0) + 1)
    .attr("y", function(d){ return yScale(d.title)})
    .attr("width", 0)
    .attr("height", yScale.bandwidth())
    .attr("fill", function(d){ return color(letters.indexOf(d.title)+1)})
    .merge(bars)//and from now on, both the enter and the update selections
    .transition()
    .duration(1000)
    .delay(1000)
    .attr("y", function(d){ return yScale(d.title)})
    .attr("width", function(d){ return xScale(d.value) - margin.left});

Finalmente, chamamos a função draw() toda vez que o botão é clicado:

d3.select("#myButton").on("click", draw);

E este é o violino mostrando todas essas 3 seleções em ação.

Mesclando seleções #

O padrão de atualização no D3 versão 3

O correto entendimento de como funcionam as seleções “entrar”, “atualizar” e “sair” é fundamental para alterar corretamente o dataviz usando D3.

Desde a versão 3 do D3 (na verdade, desde a versão 2), este trecho pode definir as transições para as seleções “enter” e “update” (demo ao vivo aqui):

var divs = body.selectAll("div")
    .data(data);//binding the data

divs.enter()//enter selection
    .append("div")
    .style("width", "0px");

divs.transition()
    .duration(1000)
    .style("width", function(d) { return d + "px"; })
    .attr("class", "divchart")
    .text(function(d) { return d; });

Mas o que acontece exatamente com o mesmo código se usarmos o D3 versão 4? Você pode ver em esta demonstração ao vivo: nada!

Por quê?

Mudanças no padrão de atualização no D3 versão 4

Vamos verificar o código. Primeiro, temos divs. Esta seleção vincula os dados ao <div>.

var divs = body.selectAll("div")
    .data(data);

Então, temos divs.enter(), que é uma seleção que contém todos os dados com elementos não correspondidos. Esta seleção contém todos os divs na primeira vez que chamamos a função draw, e definimos suas larguras para zero.

divs.enter()
    .append("div")
    .style("width", "0px");

Então temos divs.transition(), e aqui temos o comportamento interessante: No D3 versão 3, divs.transition() faz com que todos os <div> na seleção “enter” mudem para sua largura final . Mas isso não faz sentido! divs não contém a seleção “enter” e não deve modificar nenhum elemento DOM.

Há uma razão pela qual esse comportamento estranho foi introduzido no D3 versão 2 e 3 (fonte aqui), e foi “corrigido” no D3 versão 4. Assim, na demonstração ao vivo acima, nada acontece e isso é esperado! Além disso, se você clicar no botão, todas as seis barras anteriores aparecem, porque agora estão na seleção “atualizar”, não mais na seleção “entrar”.

Para a transição atuando sobre a seleção “enter”, temos que criar variáveis ​​separadas ou, uma abordagem mais fácil, mesclar as seleções:

divs.enter()//enter selection
    .append("div")
    .style("width", "0px")
    .merge(divs)//from now on, enter + update selections
    .transition().duration(1000).style("width", function(d) { return d + "px"; })
    .attr("class", "divchart")
    .text(function(d) { return d; });

Agora, mesclamos as seleções “enter” e “update”. Veja como funciona nesta demonstração ao vivo.
