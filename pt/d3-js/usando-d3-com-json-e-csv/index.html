Sintaxe #

  • d3.tsv(url[[, linha], retorno de chamada])
  • d3.html(url[, retorno de chamada])
  • d3.json(url[, retorno de chamada])
  • d3.text(url[, callback])
  • d3.xml(url[, retorno de chamada])

Carregando dados de arquivos CSV #

Existem várias maneiras de obter os dados que você vinculará aos elementos DOM. O mais simples é ter seus dados em seu script como uma matriz …

var data = [ ... ];

Mas o D3.js nos permite carregar dados de um arquivo externo. Neste exemplo, veremos como carregar e lidar adequadamente com os dados de um arquivo CSV.

Os arquivos CSV são valores separados por vírgulas. Nesse tipo de arquivo, cada linha é um registro de dados, cada registro composto por um ou mais campos, separados por vírgulas. É importante saber que a função que vamos usar, d3.csv, usa a primeira linha do CSV como cabeçalho, ou seja, a linha que contém os nomes dos campos.

Então, considere este CSV, chamado “data.csv”:

city,population,area
New York,3400,210
Melbourne,1200,350
Tokyo,5200,125
Paris,800,70

Para carregar “data.csv”, usamos a função d3.csv. Para facilitar, suponha que “data.csv” esteja no mesmo diretório do nosso script e seu caminho relativo seja simplesmente “data.csv”. Assim, escrevemos:

d3.csv("data.csv", function(data){
    //code dealing with data here
});

Observe que, no retorno de chamada, usamos data como argumento. Essa é uma prática comum em D3, mas você pode usar qualquer outro nome.

O que d3.csv faz com nosso CSV? Ele converte o CSV em uma matriz de objetos. Se, por exemplo, nós console.log nossos dados:

d3.csv("data.csv", function(data){
    console.log(data)
});

Isto é o que vamos ver:

[
    {
        "city": "New York",
        "population": "3400",
        "area": "210"
    },{
        "city": "Melbourne",
        "population": "1200",
        "area": "350"
    },{
        "city": "Tokyo",
        "population": "5200",
        "area": "125"
    },{
        "city": "Paris",
        "population": "800",
        "area": "70"
    }
]

Agora podemos vincular esses dados aos nossos elementos DOM.

Observe que, neste exemplo, population e area são strings. Mas, provavelmente, você quer lidar com eles como números. Você pode alterá-los em uma função dentro do callback (como um forEach), mas em d3.csv você pode usar uma função “acessador”:

d3.csv("data.csv", conversor, function(data){
    //code here
});

function conversor(d){
    d.population = +d.population;
    d.area = +d.area;
    return d;
}

Você também pode usar acessadores em d3.tsv, mas não em d3.json.

Observação: d3.csv é uma função assíncrona, o que significa que o código depois dela será executado imediatamente, mesmo antes do arquivo CSV ser carregado. Portanto, atenção especial para usar seus data dentro do callback.

Um ou dois parâmetros no retorno de chamada — tratamento de erros em d3.request() #

Ao usar d3.request() ou um dos construtores de conveniência (d3.json, d3.csv, d3.tsv, d3 .html e d3.xml) existem muitas fontes de erro. Pode haver problemas com a solicitação emitida ou sua resposta devido a erros de rede, ou a análise pode falhar se o conteúdo não estiver bem formado.

Dentro dos retornos de chamada passados ​​para qualquer um dos métodos mencionados acima, é, portanto, desejável implementar algum tratamento de erros. Para isso os callbacks podem aceitar dois argumentos, sendo o primeiro o erro, se houver, e o segundo os dados. Se ocorrer algum erro durante o carregamento ou análise, as informações sobre o erro serão passadas como o primeiro argumento error com os data sendo null.

d3.json{"some_file.json", function(error, data) {
  if (error) throw error;   // Exceptions handling
  // Further processing if successful
});

Você não é obrigado a fornecer dois parâmetros. Não há problema em usar os métodos de solicitação com um retorno de chamada com apenas um parâmetro. Para lidar com esses tipos de callbacks existe uma função privada fixCallback() em request.js, que ajusta a forma como as informações são passadas para o argumento único do método.

function fixCallback(callback) {
  return function(error, xhr) {
    callback(error == null ? xhr : null);
  };
}

Isso será invocado pelo D3 para todos os retornos de chamada com apenas um parâmetro, que por definição são os dados.

Não importa quantos parâmetros sejam fornecidos ao retorno de chamada do método de solicitação, a regra para o parâmetro data é:

  • se a requisição falhar, data será null
  • se a solicitação for bem-sucedida, data conterá o conteúdo carregado (e analisado)

A única diferença entre a versão de um parâmetro versus a versão de dois parâmetros está na forma como as informações são fornecidas sobre o erro que pode ocorrer. Se o parâmetro error for omitido, o método falhará silenciosamente deixando data como null. Se, por outro lado, o callback é definido para ter dois argumentos, as informações sobre um erro durante o carregamento ou análise serão passadas para o primeiro parâmetro, permitindo que você lide com isso adequadamente.

As quatro chamadas a seguir para d3.json demonstram os cenários possíveis para arquivos existentes/inexistentes versus retornos de chamada de um parâmetro/dois parâmetros:

// FAIL: resource not available or content not parsable
// error contains information about the error
// data will be null because of the error
d3.json("non_existing_file.json", function(error, data) {
  console.log("Fail, 2 parameters: ", error, data);
});

// FAIL: resource not available or content not parsable
// no information about the error
// data will be null because of the error
d3.csv("non_existing_file.json", function(data) {
  console.log("Fail, 1 parameter: ", data);
});

// OK: resource loaded successfully
// error is null
// data contains the JSON loaded from the resource
d3.json("existing_file.json", function(error, data) {
  console.log("OK, 2 parameters: ", error, data);
});

// OK: resource loaded successfully
// no information about the error; this fails silently on error
// data contains the JSON loaded from the resource
d3.json("existing_file.json", function(data) {
  console.log("OK, 1 parameter: ", data);
});
