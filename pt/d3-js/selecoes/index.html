O papel dos espaços reservados nas seleções “enter” #

O que é uma seleção de entrada?

Em D3.js, quando se vincula dados a elementos DOM, três situações são possíveis:

  1. O número de elementos e o número de pontos de dados são os mesmos;
  2. Existem mais elementos do que pontos de dados;
  3. Existem mais pontos de dados do que elementos;

Na situação #3, todos os pontos de dados sem um elemento DOM correspondente pertencem à seleção enter. Assim, no D3.js, as seleções enter são seleções que, após unir elementos aos dados, contém todos os dados que não correspondem a nenhum elemento DOM. Se usarmos uma função append em uma seleção enter, D3 criará novos elementos vinculando esses dados para nós.

Este é um diagrama de Venn que explica as possíveis situações em relação ao número de pontos de dados/número de elementos DOM:

Como podemos ver, a seleção enter é a área azul à esquerda: pontos de dados sem elementos DOM correspondentes.

A estrutura da seleção de entrada

Normalmente, uma seleção enter tem estas 4 etapas:

  1. selectAll: Seleciona elementos no DOM;
  2. data: Conta e analisa os dados;
  3. enter: Comparando a seleção com os dados, cria novos elementos;
  4. append: Anexa os elementos reais no DOM;

Este é um exemplo muito básico (veja os 4 passos no var divs):

var data = [40, 80, 150, 160, 230, 260];

var body = d3.select("body");

var divs = body.selectAll("div")
    .data(data)
    .enter()
    .append("div");

divs.style("width", function(d) { return d + "px"; })
    .attr("class", "divchart")
    .text(function(d) { return d; });

E este é o resultado (jsfiddle aqui):

Observe que, neste caso, usamos selectAll("div") como a primeira linha em nossa variável de seleção “enter”. Temos um conjunto de dados com 6 valores e D3 criou 6 divs para nós.

O papel dos marcadores

Mas suponha que já temos um div em nosso documento, algo como <div>Este é meu gráfico</div> no topo. Nesse caso, quando escrevemos:

body.selectAll("div")

estamos selecionando essa div existente. Portanto, nossa seleção de entrada terá apenas 5 dados sem elementos correspondentes. Por exemplo, neste jsfiddle, onde já existe uma div no HTML (“This is my chart”), este será o resultado:

Não vemos mais o valor “40”: nossa primeira “barra” desapareceu, e a razão para isso é que nossa seleção “enter” agora tem apenas 5 elementos.

O que temos que entender aqui é que na primeira linha da nossa variável de seleção enter, selectAll("div"), esses divs são apenas placeholders. Nós não temos que selecionar todos os divs se estivermos anexando divs, ou todo o circle se estivermos anexando circle. Podemos selecionar coisas diferentes. E, se não planejamos ter uma seleção de “atualização” ou “saída”, podemos selecionar qualquer coisa:

var divs = body.selectAll(".foo")//this class doesn't exist, and never will!
    .data(data)
    .enter()
    .append("div");

Fazendo assim, estamos selecionando todos os “.foo”. Aqui, “foo” é uma classe que não só não existe, mas também nunca foi criada em nenhum outro lugar do código! Mas não importa, isso é apenas um espaço reservado. A lógica é esta:

Se na sua seleção “enter” você selecionar algo que não existe, sua seleção “enter” sempre conterá todos os seus dados.

Agora, selecionando .foo, nossa seleção “enter” tem 6 elementos, mesmo que já tenhamos uma div no documento:

E aqui está o jsfiddle correspondente.

Selecionando null

De longe, a melhor maneira de garantir que você não está selecionando nada é selecionar null. Não só isso, mas esta alternativa é muito mais rápida do que qualquer outra.

Assim, para uma seleção enter, basta fazer:

selection.selectAll(null)
    .data(data)
    .enter()
    .append(element);

Aqui está um violino de demonstração: https://jsfiddle.net/gerardofurtado/th6s160p/

Conclusão

Ao lidar com seleções “enter”, tome cuidado extra para não selecionar algo que já existe. Você pode usar qualquer coisa em seu selectAll, mesmo coisas que não existem e nunca existirão (se você não planeja ter uma seleção de “atualização” ou “saída”).

O código nos exemplos é baseado neste código de Mike Bostock: https://bl.ocks.org/mbostock/7322386

Usando “this” com uma função de seta #

A maioria das funções em D3.js aceita uma função anônima como argumento. Os exemplos comuns são .attr, .style, .text, .on e .data, mas a lista é muito maior que isso.

Nesses casos, a função anônima é avaliada para cada elemento selecionado, na ordem, sendo passado:

  1. O dado atual (d)
  2. O índice atual (i)
  3. O grupo atual (nós)
  4. this como o elemento DOM atual.

O dado, o índice e o grupo atual são passados ​​como argumentos, o famoso primeiro, segundo e terceiro argumento em D3.js (cujos parâmetros são tradicionalmente denominados d, i e p no D3 v3.x). Para usar this, no entanto, não é necessário usar nenhum argumento:

.on("mouseover", function(){
    d3.select(this);
});

O código acima selecionará this quando o mouse estiver sobre o elemento. Verifique se está funcionando neste violino: https://jsfiddle.net/y5fwgopx/

A função de seta

Como uma nova sintaxe do ES6, uma função de seta tem uma sintaxe mais curta quando comparada à expressão de função. No entanto, para um programador D3 que usa this constantemente, há uma armadilha: uma função de seta não cria seu próprio contexto this. Isso significa que, em uma função de seta, this tem seu significado original do contexto de fechamento.

Isso pode ser útil em várias circunstâncias, mas é um problema para um codificador acostumado a usar this em D3. Por exemplo, usando o mesmo exemplo no violino acima, isso não funcionará:

.on("mouseover", ()=>{
    d3.select(this);
});

Se você duvida, aqui está o violino: https://jsfiddle.net/tfxLsv9u/

Bem, isso não é um grande problema: pode-se simplesmente usar uma expressão de função regular e antiquada quando necessário. Mas e se você quiser escrever todo o seu código usando funções de seta? É possível ter um código com funções de seta e ainda usar corretamente this em D3?

O segundo e terceiro argumentos combinados

A resposta é sim, porque this é o mesmo que nodes[i]. A dica está presente em toda a API D3, quando descreve isso:

…com this como o elemento DOM atual (nodes[i])

A explicação é simples: como nodes é o grupo atual de elementos no DOM e i é o índice de cada elemento, nodes[i] refere-se ao próprio elemento atual do DOM. Isto é, ‘isto’.

Portanto, pode-se usar:

.on("mouseover", (d, i, nodes) => {
    d3.select(nodes[i]);
});

E aqui está o violino correspondente: https://jsfiddle.net/2p2ux38s/

Seletores diferentes #

Você pode selecionar elementos com seletores diferentes:

  • por tag: "div"
  • por classe: ".class"
  • por id: "#id"
  • por atributo: "[color=blue]"
  • vários seletores (OR): "div1, div2, class1"
  • vários seletores (AND): "div1 div2 class1"

Seleção e modificações básicas #

Se você estiver familiarizado com a sintaxe do jQuery e do Sizzle, as seleções d3 não devem ser muito diferentes. d3 imita a API de seletores W3C para facilitar a interação com os elementos.

Para um exemplo básico, para selecionar todos os <p> e adicionar uma alteração a cada um deles:

d3.selectAll('p')
  .attr('class','textClass') 
  .style('color', 'white');

Em poucas palavras, isso é relativamente o mesmo que fazer no jQuery

$('p')
  .attr('class','textClass') 
  .css('color, 'white')

Geralmente, você começará com uma única seleção no div do seu contêiner para adicionar um elemento SVG que será atribuído a uma variável (mais comumente chamada de svg).

var svg = d3.select('#divID').append('svg');

A partir daqui podemos chamar svg para fazer nossas sub-seleções de vários objetos (mesmo que eles ainda não existam).

svg.selectAll('path')
