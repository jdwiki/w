d3 é uma biblioteca poderosa para criar gráficos interativos; no entanto, esse poder decorre do fato de os usuários terem que trabalhar em um nível mais baixo do que outras bibliotecas interativas. Consequentemente, muitos dos exemplos para gráficos d3 são projetados para demonstrar como produzir uma coisa específica - por exemplo, bigodes para um gráfico de caixa e bigode - enquanto muitas vezes codifica os parâmetros, tornando o código inflexível. O objetivo desta documentação é demonstrar como criar código mais reutilizável para economizar tempo no futuro.

Gráfico de dispersão #

Este exemplo contém mais de 1.000 linhas de código no total (muito para ser incorporado aqui). Por esse motivo, todo o código está acessível em http://blockbuilder.org/SumNeuron/956772481d4e625eec9a59fdb9fbe5b2 (alternadamente hospedado em https://bl.ocks.org/SumNeuron/956772481d4e625eec9a59fdb9fbe5b2). Observe que o bl.ocks.org usa o iframe, portanto, para ver o redimensionamento, você precisará clicar no botão abrir (canto inferior direito do iframe). Como há muito código, ele foi dividido em vários arquivos e o segmento de código relevante será referenciado tanto pelo nome do arquivo quanto pelo número da linha. Por favor, abra este exemplo enquanto o analisamos.

O que faz um gráfico? #

Existem vários componentes principais que fazem parte de qualquer gráfico completo; ou seja, estes incluem:

  • título
  • machados
  • rótulos de eixos
  • os dados

Existem outros aspectos que podem ser incluídos dependendo do gráfico - por exemplo, uma legenda do gráfico. No entanto, muitos desses elementos podem ser contornados com a dica de ferramenta. Por esse motivo, existem elementos específicos do gráfico interativo - por exemplo, botões para alternar entre os dados.

Como o conteúdo do nosso gráfico será interativo, seria apropriado que o próprio gráfico fosse dinâmico - por exemplo, redimensionar quando o tamanho da janela mudar. O SVG é escalável, então você pode permitir que seu gráfico seja dimensionado mantendo a perspectiva atual. No entanto, dependendo da perspectiva definida, o gráfico pode ficar muito pequeno para ser legível, mesmo se ainda houver espaço suficiente para o gráfico (por exemplo, se a largura for maior que a altura). Portanto, pode ser preferível apenas redesenhar o gráfico no tamanho restante.

Este exemplo abordará como calcular dinamicamente o posicionamento dos botões, título, eixos, rótulos de eixos, bem como lidar com conjuntos de dados de quantidades variantes de dados

Configurar #

Configuração #

Como estamos visando a reutilização de código, devemos criar um arquivo de configuração para conter opções globais para aspectos do nosso gráfico. Um exemplo de tal arquivo de configuração é charts_configuration.json.

Se olharmos para este arquivo, podemos ver que incluí vários elementos que já deveriam ter um uso claro para quando fazemos nosso gráfico:

  • arquivos (armazena a string para onde os dados do nosso gráfico são mantidos)
  • document_state (qual botão está atualmente selecionado para o nosso gráfico)
  • chart_ids (ids html para os gráficos que faremos)
  • svg (opções para o svg, por exemplo, tamanho)
  • plot_attributes
  • título (defina vários atributos de fonte)
  • dica de ferramenta (defina várias propriedades de estilo de dica de ferramenta)
  • eixos (defina vários atributos de fonte)
  • botões (defina vários atributos de fonte e estilo)
  • parcelas
  • dispersão (defina vários aspectos do nosso gráfico de dispersão, por exemplo, raio do ponto)
  • cores (uma paleta de cores específica para usar)

Funções auxiliares #

Além de configurar esses aspectos globais, precisamos definir algumas funções auxiliares. Eles podem ser encontrados em helpers.js

  • ajax_json (carregar arquivos json de forma síncrona ou assíncrona)
  • keys (retorna as chaves do objeto dado - equivalente a d3.keys())
  • parseNumber (uma análise geral de números caso não saibamos que tipo ou número é)
  • typeofNumber (retorna o tipo de número)

index.html #

Por último, devemos configurar nosso arquivo html. Para o propósito deste exemplo, colocaremos nosso gráfico em uma tag section onde o id corresponde ao id fornecido no arquivo de configuração (linha 37). Como as porcentagens só funcionam se puderem ser calculadas a partir do membro pai, também incluímos alguns estilos básicos (linhas 19-35)

Fazendo nosso gráfico de dispersão #

Vamos abrir make_scatter_chart.js. Agora vamos prestar atenção na linha 2, onde muitas das variáveis ​​mais importantes são predefinidas:

  • svg - d3 seleção do svg do gráfico
  • chart_group - seleção d3 do grupo dentro do svg em que os dados serão colocados
  • canvas - aspectos centrais da extração svg por conveniência
  • margens - as margens que precisamos levar em consideração
  • maxi_draw_space os maiores valores x e y nos quais podemos desenhar nossos dados
  • doc_state - o estado atual do documento se estivermos usando botões (neste exemplo estamos)

Você deve ter notado que não incluímos o svg no html. Portanto, antes que possamos fazer qualquer coisa com nosso gráfico, precisamos adicionar o svg ao index.html se ele ainda não existir. Isso é feito no arquivo make_svg.js pela função make_chart_svg. Olhando para make_svg.js vemos que usamos a função auxiliar parseNumber na configuração do gráfico para a largura e altura do svg. Se o número for um float, ele torna a largura e a altura do svg proporcional à largura e altura de sua seção. Se o número for um número inteiro, ele apenas o definirá para esses números inteiros.

As linhas 6 a 11 testam para ver - de fato - se esta é a primeira chamada ou não e define o chart_group (e o estado do documento se for a primeira chamada).

A linha 14 - 15 extrai os dados atualmente selecionados pelo botão clicado; a linha 16 define data_extent. Enquanto d3 tem uma função para extrair a extensão dos dados, é minha preferência armazenar a extensão dos dados nesta variável.

As linhas 27 - 38 contêm a mágica que configura nosso gráfico fazendo as margens, os botões, o título e os eixos. Todos eles são determinados dinamicamente e podem parecer um pouco complexos, portanto, examinaremos cada um deles.

make_margins (em make_margins.js) #

Podemos ver que o objeto margins leva em consideração algum espaço à esquerda, direita, superior e inferior do gráfico (x.left, x.right, y.top, y.bottom respectivamente), o título, os botões e os eixos.

Também vemos que as margens dos eixos são atualizadas na linha 21.

porque nós fazemos isso? Bem, a menos que especifiquemos o número de tiques, o tiquetaque rotula o tamanho do tiquetaque e o tamanho da fonte do rótulo do tiquetaque, não poderíamos calcular o tamanho que os eixos precisam. Mesmo assim, ainda teríamos que estimar o espaço entre os rótulos dos carrapatos e os carrapatos. Portanto, é mais fácil fazer alguns eixos fictícios usando nossos dados, ver o tamanho dos elementos svg correspondentes e retornar o tamanho.

Na verdade, precisamos apenas da largura do eixo y e da altura do eixo x, que é o que está armazenado em axes.y e axes.x.

Com nossas margens padrão definidas, calculamos o max_drawing_space (linhas 29-34 em make_margins.js)

make_buttons (em make_buttons.js) #

A função cria um grupo para todos os botões e, em seguida, um grupo para cada botão, que por sua vez armazena um círculo e um elemento de texto. A linha 37 - 85 calcula a posição dos botões. Ele faz isso verificando se o texto à direita do comprimento de cada botão é maior do que o espaço permitido para desenharmos (linha 75). Nesse caso, ele solta o botão em uma linha e atualiza as margens.

make_title (em make_title.js) #

O make_title é semelhante ao make_buttons, pois ele dividirá automaticamente o título do gráfico em várias linhas e hifenizará, se necessário. É um pouco hacky, pois não tem a sofisticação do esquema de hifenização do TeX, mas funciona suficientemente bem. Se precisarmos de mais linhas do que uma, as margens são atualizadas.

Com os botões, título e margens definidos, podemos fazer nossos eixos.

make_axes (em make_axes.js) #

A lógica de make_axes espelha aquela para calcular o espaço necessário para os eixos fictícios. Aqui, no entanto, ele adiciona transições para mudar entre os eixos.

Finalmente nosso gráfico de dispersão #

Com tudo isso configurado, podemos finalmente fazer nosso gráfico de dispersão. Como nossos conjuntos de dados podem ter um número diferente de pontos, precisamos levar isso em consideração e aproveitar os eventos de entrada e saída do d3 de acordo. Obter o número de pontos já existentes é feito na linha 40. A instrução if na linha 45 - 59 adiciona mais elementos de círculo se tivermos mais dados, ou transiciona os elementos extras para um canto e os remove se houver muitos.

Uma vez que sabemos que temos o número certo de elementos, podemos fazer a transição de todos os elementos restantes para sua posição correta (linha 64)

Por último, adicionamos a dica de ferramenta nas linhas 67 e 68. A função de dica de ferramenta está em make_tooltip.js

Gráfico de caixa e bigode #

Para mostrar o valor de fazer funções generalizadas como as do exemplo anterior (make_title, make_axes, make_buttons, etc), considere este gráfico de caixa e bigode: https://bl.ocks.org/SumNeuron/262e37e2f932cf4b693f241c52a410ff
