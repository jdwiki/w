Sistema de coordenadas #

Em um sistema de coordenadas matemáticas normal, o ponto x=0, y=0 está no canto inferior esquerdo do gráfico. Mas no sistema de coordenadas SVG, este ponto (0,0) está no canto superior esquerdo da ’tela’, é semelhante ao CSS quando você especifica a posição para absoluto/fixo e usa o topo e a esquerda para controlar o ponto exato do elemento.

É essencial ter em mente que à medida que y aumenta em SVG, as formas se movem para baixo.

Digamos que vamos criar um gráfico de dispersão com cada ponto correspondente a um valor x e um valor y. Para dimensionar o valor, precisamos definir o domínio e o intervalo assim:

d3.svg.scale()
  .range([0, height])
  .domain([0,max])

No entanto, se você apenas mantiver as configurações assim, os pontos serão baseados na borda horizontal superior em vez da linha horizontal inferior como esperávamos.

O bom do d3 é que você pode alterar isso facilmente com um simples ajuste na configuração do domínio:

d3.scale.linear()
  .range([height, 0])
  .domain([0, max])

Com o código acima, o ponto zero do domínio corresponde à altura do SVG, que é a linha inferior do gráfico aos olhos do visualizador, enquanto isso, o valor máximo dos dados de origem será correspondente ao ponto zero do SVG sistema de coordenadas, que é o valor máximo para os espectadores.

O elemento #

<rect> representa o retângulo, além das propriedades estéticas como traço e preenchimento, o retângulo deve ser definido por localização e tamanho.

Quanto à localização, ela é determinada pelos atributos x e y. A localização é relativa ao pai do retângulo. E se você não especificar o atributo x ou y, o padrão será 0 em relação ao elemento pai.

Depois de especificar o local, ou melhor, o ‘ponto de partida’ do retângulo, o próximo passo é especificar o tamanho, o que é essencial se você quiser realmente desenhar algo na tela, ou seja, se você não especifique os atributos de tamanho ou o valor é definido como 0, você não verá nada na tela.

Caso: Gráfico de Barras

Continue com o primeiro cenário, os eixos y, mas desta vez, vamos tentar desenhar um gráfico de barras.

Supondo que a configuração da escala y seja a mesma, o eixo y também esteja configurado corretamente, a única diferença entre o gráfico de dispersão e este gráfico de barras é que precisamos especificar a largura e a altura, principalmente a altura. Para ser mais específico, já temos o ‘ponto de partida’, o resto é usar coisas como para a altura:

.attr("height", function(d){
  return (height - yScale(d.value))
})

O elemento #

O SVG atualmente não suporta quebras de linha automáticas ou quebra de linha, é aí que o vem em socorro. O elemento posiciona novas linhas de texto em relação à linha de texto anterior. E usando dx ou dy dentro de cada um desses intervalos, podemos posicionar a palavra em relação à palavra anterior.

Caso: Anotação em Eixos

Por exemplo, quando queremos adicionar uma anotação em y Axe:

svg.append("g")
  .attr("class", "y axis")
  .call(yAxis)
.append("text")
  .attr("transform", "rotate(-90)")
  .attr("y", 6)
  .attr("dy", ".71em")
  .style("text-anchor", "end")
  .text("Temperature (ºF)");

Anexando corretamente um elemento SVG #

Este é um erro relativamente comum: você criou um elemento rect, em um gráfico de barras por exemplo, e deseja adicionar um rótulo de texto (digamos, o valor dessa barra). Então, usando a mesma variável que você usou para anexar o rect e definir suas posições x e y, você anexa seu elemento text. Muito lógica, você pode pensar. Mas isso não vai funcionar.

Como esse erro ocorre?

Vamos ver um exemplo concreto, um código bem básico para criar um gráfico de barras (fiddle here):

var data = [210, 36, 322, 59, 123, 350, 290];

var width = 400, height = 300;

var svg = d3.select("body")
    .append("svg")
    .attr("width", width)
    .attr("height", height);

var bars = svg.selectAll(".myBars")
    .data(data)
    .enter()
    .append("rect");

bars.attr("x", 10)
    .attr("y", function(d,i){ return 10 + i*40})
    .attr("width", function(d){ return d})
    .attr("height", 30);

O que nos dá este resultado:

Mas você deseja adicionar alguns elementos de texto, talvez um valor simples para cada barra. Então, você faz isso:

bars.append("text")
   .attr("x", 10)
   .attr("y", function(d,i){ return 10 + i*40})
   .text(function(d){ return d});

E, voilà: nada acontece! Se você duvida, aqui está o violino.

*“Mas estou vendo a etiqueta!”

Se você inspecionar o SVG criado por este último código, verá isto:

E neste momento muita gente diz: “Mas estou vendo a tag de texto, está anexada!”. Sim, é, mas isso não significa que vai funcionar. Você pode anexar qualquer coisa! Veja isso, por exemplo:

svg.append("crazyTag");

Ele vai te dar este resultado:

<svg>
    <crazyTag></crazyTag>
</svg>

Mas você não espera nenhum resultado só porque a etiqueta está lá, não é?

Anexar elementos SVG da maneira correta

Saiba quais elementos SVG podem conter crianças, lendo as especificações. Em nosso último exemplo, o código não funciona porque os elementos rect não podem conter elementos text. Então, como exibir nossos textos?

Crie outra variável e anexe o text ao SVG:

var texts = svg.selectAll(".myTexts")
    .data(data)
    .enter()
    .append("text");

texts.attr("x", function(d){ return d + 16})
    .attr("y", function(d,i){ return 30 + i*40})
    .text(function(d){ return d});

SVG: a ordem de desenho #

Isso é algo que pode ser frustrante: você faz uma visualização usando D3.js, mas o retângulo que você quer no topo está escondido atrás de outro retângulo, ou a linha que você planejou estar atrás de algum círculo está na verdade sobre ele. Você tenta resolver isso usando o z-index em seu CSS, mas não funciona (no SVG 1.1).

A explicação é simples: em um SVG, a ordem dos elementos define a ordem da “pintura”, e a ordem da pintura define quem vai por cima.

Os elementos em um fragmento de documento SVG têm uma ordem de desenho implícita, com os primeiros elementos no fragmento de documento SVG sendo “pintados” primeiro. Os elementos subsequentes são pintados sobre os elementos pintados anteriormente.

Então, suponha que temos este SVG:

<svg width="400" height=200>
    <circle cy="100" cx="80" r="60" fill="blue"></circle>
    <circle cy="100" cx="160" r="60" fill="yellow"></circle>
    <circle cy="100" cx="240" r="60" fill="red"></circle>
    <circle cy="100" cx="320" r="60" fill="green" z-index="-1"></circle>
</svg>

Ele tem quatro círculos. O círculo azul é o primeiro “pintado”, então ficará abaixo de todos os outros. Depois temos o amarelo, depois o vermelho e finalmente o verde. O verde é o último, e estará no topo.

Mudando a ordem dos elementos SVG com D3

Então, é possível mudar a ordem dos elementos? Posso fazer o círculo vermelho na frente do círculo verde?

Sim. A primeira abordagem que você precisa ter em mente é a ordem das linhas em seu código: desenhe primeiro os elementos do plano de fundo e depois no código os elementos do primeiro plano.

Mas podemos alterar dinamicamente a ordem dos elementos, mesmo depois de pintados. Existem várias funções JavaScript simples que você pode escrever para fazer isso, mas o D3 já tem 2 recursos interessantes, selection.raise() e selection.lower().

De acordo com a API:

selection.raise(): Reinsere cada elemento selecionado, em ordem, como o último filho de seu pai. selection.lower(): Reinsere cada elemento selecionado, em ordem, como o primeiro filho de seu pai.

Então, para mostrar como manipular a ordem dos elementos em nosso SVG anterior, aqui está um código bem pequeno:

d3.selectAll("circle").on("mouseover", function(){
    d3.select(this).raise(); 
});

O que isso faz? Ele seleciona todos os círculos e, quando o usuário passa o mouse sobre um círculo, ele seleciona esse círculo específico e o traz para a frente. Muito simples!
