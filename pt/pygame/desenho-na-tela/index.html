Sintaxe #

  • pygame.draw.rect(Superfície, cor, Rect, largura=0)
  • pygame.draw.polygon(Superfície, cor, lista de pontos, largura=0)
  • pygame.draw.circle(Superfície, cor, posição, raio, largura=0)
  • pygame.draw.ellipse(Superfície, cor, Rect, largura=0)
  • pygame.draw.arc(Superfície, cor, Rect, start_angle, stop_angle, largura=1)
  • pygame.draw.line(Superfície, cor, start_pos, end_pos, largura=1)
  • pygame.draw.lines(Superfície, cor, fechado, lista de pontos, largura=1)
  • pygame.draw.aaline(Surface, color, startpos, endpos, blend=1)
  • pygame.draw.aalines(Superfície, cor, fechado, lista de pontos, mistura=1)

Parâmetros #

Parâmetros Detalhes
Superfície A superfície para desenhar a forma.
cor Uma sequência de 3 ou 4 inteiros representando vermelho, verde e azul (e alfa), cada valor variando entre 0-255.
Rect Uma área retangular onde a forma será desenhada.
largura A largura das linhas. A forma será preenchida se largura=0.
lista de pontos Uma lista de uma quantidade arbitrária de pontos/vértices, em pixels (x, y).
posição A posição do centro do círculo, em pixels (x, y).
raio O raio dos círculos em pixels.
fechado Se true, uma linha entre o último e o primeiro ponto será desenhada, fechando a forma.
mistura=1 Se true, os tons serão misturados com tons de pixel existentes em vez de sobrescrevê-los.
ângulo_inicial O ângulo inicial do arco, em radianos.
stop_angle O ângulo final do arco, em radianos.
start_pos A posição inicial da linha, em pixels.
end_pos A posição final da linha, em pixels

Desenhando com o módulo draw #

Pygame tem um módulo, pygame.draw, que contém funções que podem desenhar formas diretamente em uma superfície.

Função Descrição
pygame.draw.rect desenhe uma forma de retângulo
pygame.draw.polygon desenhe uma forma com qualquer número de lados
pygame.draw.circle desenhe um círculo em torno de um ponto
pygame.draw.elipse desenhe uma forma redonda dentro de um retângulo
pygame.draw.arc desenhe uma seção parcial de uma elipse
pygame.draw.line desenhe um segmento de linha reta
pygame.draw.lines desenhar vários segmentos de linha contíguos
pygame.draw.aaline desenhar linhas finas de suavização
pygame.draw.aalines desenhe uma sequência conectada de linhas suavizadas

Como usar o módulo #

Para usar o módulo, primeiro você precisa importar e inicializar o pygame corretamente e definir um modo para a exibição. É conveniente definir as constantes de cor com antecedência, tornando seu código mais legível e mais bonito. Todas as funções usam uma superfície para desenhar, uma cor e um argumento de posição que é um pygame Rect ou uma sequência inteira/float de 2 elementos (o pygame.draw.circle só aceita números inteiros por motivos indefinidos).

Exemplo #

O código abaixo irá mostrar todas as diferentes funções, como elas são usadas e como elas se parecem. Vamos inicializar o pygame e definir algumas constantes antes dos exemplos.

import pygame
from math import pi
pygame.init()

screen = pygame.display.set_mode((100, 100))
WHITE = pygame.Color(255, 255, 255)
RED = pygame.Color(255, 0, 0)

A cor preta é a cor padrão da superfície e representa a parte da superfície que não foi desenhada. Os parâmetros de cada função são explicados abaixo em Parâmetros .

Reto #

size = (50, 50)

rect_border = pygame.Surface(size)  # Create a Surface to draw on.
pygame.draw.rect(rect_border, RED, rect_border.get_rect(), 10)  # Draw on it.

rect_filled = pygame.Surface(size)
pygame.draw.rect(rect_filled, RED, rect_filled.get_rect())

Polígono #

size = (50, 50)
points = [(25, 0), (50, 25), (25, 50), (0, 25)]  # The corner points of the polygon.

polygon = pygame.Surface(size)
pygame.draw.polygon(polygon, RED, points, 10)

polygon_filled = pygame.Surface(size)
pygame.draw.polygon(polygon_filled, RED, points)

Círculo #

size = (50, 50)
radius = 25

circle = pygame.Surface(size)
pygame.draw.circle(circle, RED, (radius, radius), radius, 10)  # Position is the center of the circle.

circle_filled = pygame.Surface(size)
pygame.draw.circle(circle_filled, RED, (radius, radius), radius)

Os buracos são uma consequência infeliz do algoritmo de desenho do pygame.

Elipse #

size = (50, 25)  # Minimize it's height so it doesn't look like a circle.

ellipse = pygame.Surface(size)  
pygame.draw.ellipse(ellipse, RED, ellipse.get_rect(), 5)

ellipse_filled = pygame.Surface(size)
pygame.draw.ellipse(ellipse_filled, RED, ellipse.get_rect())

Os buracos são uma consequência infeliz do algoritmo de desenho do pygame.

Arco #

size = (50, 50)

arc = pygame.Surface(size)
pygame.draw.arc(arc, RED, arc.get_rect(), 0, pi)  # 0 to pi is 180° creating a half circle.

Linha #

size = (50, 50)

line = pygame.Surface(size)
pygame.draw.line(line, RED, (0, 0), (50, 50))  # Start at topleft and ends at bottomright.

Linhas #

size = (50, 50)
points = [(25, 0), (50, 25), (25, 50), (0, 25)]

lines = pygame.Surface(size)
pygame.draw.lines(lines, RED, False, points)

lines_closed = pygame.Surface(size)
pygame.draw.lines(lines_closed, RED, True, points)

Linha suavizada #

size = (50, 50)

antialiased_line = pygame.Surface(size)
pygame.draw.aaline(antialiased_line, RED, (0, 0), (50, 50))

Linhas suavizadas #

size = (50, 50)
points = [(25, 0), (50, 25), (25, 50), (0, 25)]

antialiased_lines = pygame.Surface(size)
pygame.draw.aalines(antialiased_lines, RED, False, points)

antialiased_lines_closed = pygame.Surface(size)
pygame.draw.aalines(antialiased_lines_closed, RED, True, points)

Superfícies #

No pygame você normalmente usa Superfícies para representar a aparência dos objetos e Retângulos para representar suas posições. Uma superfície é como uma folha de papel em branco que contém cores ou imagens. Há duas maneiras de criar uma superfície: em branco do zero ou carregando uma imagem.

Criar uma superfície #

Para criar uma Superfície você precisa no mínimo de seu tamanho, que é uma sequência inteira de 2 elementos de largura e altura, representando o tamanho em pixels.

Você também pode passar argumentos adicionais ao criar uma superfície para controlar a profundidade de bits, máscaras e recursos adicionais como alfa por pixel e/ou criar a imagem na memória de vídeo. Isso está fora do escopo deste exemplo.

size = width, height = (32, 32)
empty_surface = pygame.Surface(size)

Você pode usar o módulo pygame.draw para desenhar formas na superfície ou preenchê-la com uma cor chamando o método Surface fill(color). O argumento color é uma sequência inteira de 3 ou 4 elementos ou um objeto pygame.Color.

Carregar uma imagem #

Na maioria das vezes você gostaria de usar suas próprias imagens em um jogo (chamados sprites). Criar um Surface com a sua imagem é tão fácil quanto:

my_image = pygame.image.load(path_to_image)

O caminho para a imagem pode ser relativo ou absoluto. Para melhorar o desempenho, geralmente é aconselhável converter sua imagem para o mesmo formato de pixel da tela. Isso pode ser feito chamando o método Surface convert(), assim:

my_image = pygame.image.load(path_to_image).convert()

Se sua imagem contiver transparência (valores alfa), basta chamar o método convert_alpha():

my_image = pygame.image.load(path_to_image).convert_alpha()

Queimando #

As superfícies precisam ser iluminadas na tela para poder exibi-las. Blitting significa essencialmente copiar pixels de uma superfície para outra (a tela também é uma superfície). Você também precisa passar a posição da superfície, que deve ser uma sequência inteira de 2 elementos ou um objeto Rect. O canto superior esquerdo da Superfície será colocado na posição.

screen.blit(my_image, (0, 0))
pygame.display.update()  # or pygame.display.flip()

É possível blit para outras superfícies além da tela. Para exibir o que foi exibido na tela, você precisa chamar pygame.display.update() ou pygame.display.flip().

Transparência #

Existem tipos 3 de transparência suportados no pygame: colorkeys, alfas de superfície e alfas por pixel.

Teclas de cores #

Torna uma cor totalmente transparente, ou mais precisamente, fazendo com que uma cor simplesmente não fique manchada. Se você tiver uma imagem com um retângulo preto dentro, você pode definir um colorkey para evitar que a cor preta fique manchada.

BLACK = (0, 0, 0)
my_image.set_colorkey(BLACK)  # Black colors will not be blit.

Uma superfície só pode ter uma chave de cor. Definir outra colorkey substituirá a anterior. As teclas de cores não podem ter valores alfa diferentes, apenas podem tornar uma cor não visível.

Alfas de superfície #

Torna toda a superfície transparente por um valor alfa. Com este método, você pode ter diferentes valores de alfa, mas afetará toda a superfície.

my_image.set_alpha(100)  # 0 is fully transparent and 255 fully opaque.

Alfa por pixel #

Torna cada pixel na superfície transparente por um valor alfa individual. Isso lhe dá mais liberdade e flexibilidade, mas também é o método mais lento. Este método também requer que a superfície seja criada como uma superfície alfa por pixel, e os argumentos de cor precisam conter um quarto inteiro alfa.

size = width, height = (32, 32)
my_image = pygame.Surface(size, pygame.SRCALPHA)  # Creates an empty per-pixel alpha Surface.

A superfície agora desenhará transparência se a cor contiver o quarto valor alfa.

BLUE = (0, 0, 255, 255)
pygame.draw.rect(my_image, BLUE, my_image.get_rect(), 10)

Ao contrário das outras superfícies, esta cor padrão da superfície não será preta, mas transparente. É por isso que o retângulo preto no meio desaparece.

Combine colorkey e Surface alfa #

Colorkeys e alfas de superfície podem ser combinados, mas alfa por pixel não. Isso pode ser útil se você não quiser o desempenho mais lento de um Surface por pixel.

purple_image.set_colorkey(BLACK)
purple_image.set_alpha(50)

Código completo #

Copie isso em um arquivo vazio e execute-o. Pressione as teclas 1, 2, 3 ou 4 para que as imagens apareçam. Pressione 2, 3 ou 4 várias vezes para torná-los mais opacos.

import pygame
pygame.init()

WHITE = (255, 255, 255)
BLACK = (0, 0, 0)
RED = (255, 0, 0)
GREEN = (0, 255, 0)
BLUE = (0, 0, 255, 50)  # This color contains an extra integer. It's the alpha value.
PURPLE = (255, 0, 255)

screen = pygame.display.set_mode((200, 325))
screen.fill(WHITE)  # Make the background white. Remember that the screen is a Surface!
clock = pygame.time.Clock()

size = (50, 50)
red_image = pygame.Surface(size)
green_image = pygame.Surface(size)
blue_image = pygame.Surface(size, pygame.SRCALPHA)  # Contains a flag telling pygame that the Surface is per-pixel alpha
purple_image = pygame.Surface(size)

red_image.set_colorkey(BLACK)
green_image.set_alpha(50)
# For the 'blue_image' it's the alpha value of the color that's been drawn to each pixel that determines transparency.
purple_image.set_colorkey(BLACK)
purple_image.set_alpha(50)

pygame.draw.rect(red_image, RED, red_image.get_rect(), 10)
pygame.draw.rect(green_image, GREEN, green_image.get_rect(), 10)
pygame.draw.rect(blue_image, BLUE, blue_image.get_rect(), 10)
pygame.draw.rect(purple_image, PURPLE, purple_image.get_rect(), 10)

while True:
    clock.tick(60)

    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            quit()
        elif event.type == pygame.KEYDOWN:
            if event.key == pygame.K_1:
                screen.blit(red_image, (75, 25))
            elif event.key == pygame.K_2:
                screen.blit(green_image, (75, 100))
            elif event.key == pygame.K_3:
                screen.blit(blue_image, (75, 175))
            elif event.key == pygame.K_4:
                screen.blit(purple_image, (75, 250))

    pygame.display.update()
