Um simples ‘jogo’ #

Importar e inicializar #

Cada módulo precisa ser importado e o pygame não é exceção. Embora precisemos chamar a função pygame.init() para que todos os módulos importados no pygame sejam inicializados corretamente. Se esquecermos disso, alguns módulos não funcionarão. A função também retorna uma tupla de todas as inicializações bem-sucedidas e com falha (não gerará um erro se um módulo falhar ao inicializar).

import pygame
successes, failures = pygame.init()
print("{0} successes and {1} failures".format(successes, failures))

Crie necessidades #

Também precisamos criar um display. O Pygame já criou uma tela (oculta), então tudo o que precisamos fazer é definir o modo da tela (neste exemplo, definimos apenas a resolução). Também é uma boa ideia criar um relógio para garantir que nosso programa seja atualizado em uma velocidade fixa (caso contrário, ele funcionaria em velocidades diferentes, dependendo da velocidade do computador).

screen = pygame.display.set_mode((720, 480))  # Notice the tuple! It's not 2 arguments.
clock = pygame.time.Clock()
FPS = 60  # This variable will define how many frames we update per second.

Para facilitar a leitura posteriormente em nosso código, criaremos duas constantes de cor, que representam uma tupla de Vermelho, Verde e Azul (RGB). Os valores vão de 0 (sem luz) a 255 (luz total).

BLACK = (0, 0, 0)
WHITE = (255, 255, 255)

No pygame geralmente usamos um Surface para representar a aparência de um objeto e um Rect (retângulo) para representar a posição de um objeto. Uma Superfície é como uma folha de papel em branco que contém cores ou imagens. Se você estiver criando uma classe, você deve nomear os atributos image e rect, pois muitas funções irão procurar e usar esses atributos. Essas classes se beneficiariam herdando a classe pygame.sprite.Sprite por razões que você pode ler aqui.

rect = pygame.Rect((0, 0), (32, 32))  # First tuple is position, second is size.
image = pygame.Surface((32, 32))  # The tuple represent size.
image.fill(WHITE)  # We fill our surface with a nice white color (by default black).

O loop do jogo #

Agora temos tudo definido para o nosso loop de jogo. Este é um loop que será executado por todo o jogo, onde tratamos de eventos e atualizamos a tela e as posições de nossos objetos.

Primeiro vamos garantir que nosso loop seja executado em um determinado FPS. Definimos o FPS e criamos nosso relógio no início do programa. O código a seguir garantirá que nosso programa durma tempo suficiente para fazer nosso loop repetir a quantidade que definimos como nosso FPS. Neste exemplo, 60 vezes por segundo.

clock.tick(FPS)

Então vamos lidar com eventos. Um evento é basicamente uma ação do usuário, como mover o mouse ou pressionar uma tecla. O Pygame registrará todos esses eventos em uma fila que obtemos chamando pygame.event.get(). Podemos iterar sobre isso e verificar se há um evento que gostaríamos de manipular. Os eventos têm um atributo type que podemos verificar em relação às constantes no módulo pygame para determinar que tipo de evento é.

for event in pygame.event.get():
    if event.type == pygame.QUIT:  # The user pressed the close button in the top corner of the window.
        quit()
        # Close the program. Other methods like 'raise SystemExit' or 'sys.exit()'.
        # Calling 'pygame.quit()' won't close the program! It will just uninitialize the modules.

Também podemos verificar if event.type == pygame.KEYDOWN para ver se o usuário pressionou uma tecla. Nesse caso o evento possui um atributo key que podemos verificar para ver qual chave ele representa.

for event in pygame.event.get():
    if event.type == pygame.QUIT:
        quit()
    elif event.type == pygame.KEYDOWN:
        if event.key == pygame.K_w:
            rect.move_ip(0, -2)  # Changes the rect's position.
        elif event.key == pygame.K_s:
            rect.move_ip(0, 2)
        elif event.key == pygame.K_a:
            rect.move_ip(-2, 0)
        elif event.key == pygame.K_d:
            rect.move_ip(2, 0)

Agora precisamos exibir nossa imagem. Primeiro, podemos querer limpar nossa tela da renderização anterior. Fazemos isso preenchendo toda a tela com preto (remova o código para ver por que queremos limpá-lo). Então precisamos blit nossa imagem para a tela. Blitting significa essencialmente copiar a imagem para outra superfície (no nosso caso, a tela). Por último, viramos ou atualizamos a tela.

Quando estamos blitting, não estamos exibindo nada para o usuário. Imagine-o como o computador de um lado e o usuário do outro. O computador desenha (blits) em seu lado da tela, vira na direção do usuário e depois repete.

screen.fill(BLACK)
screen.blit(image, rect)
pygame.display.update()  # Or 'pygame.display.flip()'.

Agora temos um jogo básico! Bastante chato, sim, mas o essencial está lá! Combine isso com seu conhecimento atual de Python e você poderá criar algo incrível.

Código completo #

import pygame
successes, failures = pygame.init()
print("{0} successes and {1} failures".format(successes, failures))


screen = pygame.display.set_mode((720, 480))
clock = pygame.time.Clock()
FPS = 60  # Frames per second.

BLACK = (0, 0, 0)
WHITE = (255, 255, 255)
# RED = (255, 0, 0), GREEN = (0, 255, 0), BLUE = (0, 0, 255).

rect = pygame.Rect((0, 0), (32, 32))
image = pygame.Surface((32, 32))
image .fill(WHITE)  

while True:
    clock.tick(FPS)

    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            quit()
        elif event.type == pygame.KEYDOWN:
            if event.key == pygame.K_w:
                rect.move_ip(0, -2)
            elif event.key == pygame.K_s:
                rect.move_ip(0, 2)
            elif event.key == pygame.K_a:
                rect.move_ip(-2, 0)
            elif event.key == pygame.K_d:
                rect.move_ip(2, 0)

    screen.fill(BLACK)
    screen.blit(image, rect)
    pygame.display.update()  # Or pygame.display.flip()

Mecânica de jogo ligeiramente melhorada #

Observe que o programa verifica quando pressionamos a tecla e não quando a pressionamos. Para corrigir isso, poderíamos introduzir uma variável velocity. Podemos criar uma classe de jogador para mantê-lo mais organizado. Para evitar o movimento dependente do quadro (se mudarmos o FPS para 30, os objetos se moverão na metade da velocidade), introduzimos o movimento dependente do tempo passando o tempo entre os tiques para nossos objetos móveis.

import pygame

successes, failures = pygame.init()
print("Initializing pygame: {0} successes and {1} failures.".format(successes, failures))

screen = pygame.display.set_mode((720, 480))
clock = pygame.time.Clock()
FPS = 60

BLACK = (0, 0, 0)
WHITE = (255, 255, 255)


class Player(pygame.sprite.Sprite):
    def __init__(self):
        super().__init__()
        self.image = pygame.Surface((32, 32))
        self.image.fill(WHITE)
        self.rect = self.image.get_rect()  # Get rect of some size as 'image'.
        self.velocity = [0, 0]

    def update(self):
        self.rect.move_ip(*self.velocity)


player = Player()
running = True
while running:
    dt = clock.tick(FPS) / 1000  # Returns milliseconds between each call to 'tick'. The convert time to seconds.
    screen.fill(BLACK)  # Fill the screen with background color.

    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
        elif event.type == pygame.KEYDOWN:
            if event.key == pygame.K_w:
                player.velocity[1] = -200 * dt  # 200 pixels per second
            elif event.key == pygame.K_s:
                player.velocity[1] = 200 * dt
            elif event.key == pygame.K_a:
                player.velocity[0] = -200 * dt
            elif event.key == pygame.K_d:
                player.velocity[0] = 200 * dt
        elif event.type == pygame.KEYUP:
            if event.key == pygame.K_w or event.key == pygame.K_s:
                player.velocity[1] = 0
            elif event.key == pygame.K_a or event.key == pygame.K_d:
                player.velocity[0] = 0

    player.update()

    screen.blit(player.image, player.rect)
    pygame.display.update()  # Or pygame.display.flip()

print("Exited the game loop. Game will quit...")
quit()  # Not actually necessary since the script will exit anyway.

Instalando o pygame #

Nas janelas #

  1. Navegue até http://www.lfd.uci.edu/~gohlke/pythonlibs/#pygame – um site não oficial que fornece binários do Windows de pacotes python de código aberto para o [CPython] oficial (https://en.wikipedia .org/wiki/CPython) distribuído por Christoph Gohlke.

  2. Baixe o arquivo pygame .whl apropriado de acordo com a versão do python instalada. (O arquivo é nomeado algo como pygame -<pygame version> - <python version>- win32.whl)

  3. Executar

     pip install your-pygame-package.whl

dentro do seu terminal, bash ou console.
Nota: se pip não for encontrado em PATH tente executar python -m pip install your-pygame-package.whl

  1. Verifique se você pode importar pygame como um módulo python

    import pygame

Se você não receber um erro, você instalou corretamente o pygame no seu computador :)

No linux #

  1. Abra seu terminal e execute

    sudo apt-get install python-pygame

Nota: Isso instalará o pygame para python2 2) Tente importar o pygame para dentro

   import pygame

Se você não receber um erro, você instalou corretamente o pygame em seu sistema linux :)

No macOS #

Existem duas maneiras de instalá-lo no mac:

Método 1

Vá para a página de downloads do Pygame e baixe o instalador do mac. Execute-o e ele deve instalar o Pygame no seu Mac.

Método 2

Instale Homebrew:

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Em seguida, use o Homebrew para instalar o Python 2.7.12 e o Pygame:

brew install python; brew install homebrew/python/pygame

Agora execute o Python em seu terminal e tente import pygame. Se não disser nada, foi instalado com sucesso.

Importando pygame e desenhando em um display #

Começando #

Você deve fazer o seguinte para começar a usar o Pygame:

import pygame

Isso abre uma janela de tamanho 640.480 e a armazena em uma variável chamada tela.

Configurando um nome de janela #

A configuração de um nome para a janela pygame requer a seguinte sintaxe:

pygame.display.set_caption('Name')

Sobre a tela #

  • O ponto (0,0) está no canto superior esquerdo da tela.
  • As coordenadas x aumentam da esquerda para a direita, as coordenadas y aumentam de cima para baixo. Ou seja, as coordenadas do lado direito no plano cartesiano são positivas e o lado esquerdo é negativo. No entanto, as coordenadas do lado superior no plano cartesiano são negativas no topo e positivo na parte inferior.(Nota: Isso é considerado se os pontos forem retirados da origem.)

Atualizando a tela #

As alterações que você faz na tela, por exemplo, preenchê-lo com cor, ou desenhar nele, não apareça imediatamente! Então, como fazê-lo? Você tem que chamar esta função:

pygame.display.update()

Cores #

A coloração no pygame funciona no modo RGB. O código para colorir é:

color_Name = (r,g,b)
  • R significa vermelho.
  • G significa verde
  • B significa azul.
  • Todos os três devem ser inteiros entre 0 e 255, sendo 255 o mais claro e 0 o mais escuro

Desenho #

  1. Para desenhar linhas

     pygame.draw.lines(screen, color, closed, pointlist, thickness)

  2. Para desenhar um retângulo

     pygame.draw.rect(screen, color, (x,y,width,height), thickness)

  3. Para desenhar um círculo

    pygame.draw.circle(screen, color, (x,y), radius, thickness)

Configurando tudo em um loop #

Para fazer um loop, use o seguinte código:

running = True
while running:
  for event in pygame.event.get():
    if event.type == pygame.QUIT:
      running = False
      pygame.quit()

Desenhando um retângulo na janela do pygame (código) #

import pygame
background_colour = (255,255,255) # White color
(width, height) = (300, 200) # Screen size
color=(0,0,0) #For retangle
screen = pygame.display.set_mode((width, height)) #Setting Screen
pygame.display.set_caption('Drawing') #Window Name
screen.fill(background_colour)#Fills white to screen
pygame.draw.rect(screen, color, (100,50,30,40), 1) #Drawing the rectangle
pygame.display.update()

#Loop
running = True
while running:
  for event in pygame.event.get():
    if event.type == pygame.QUIT:
      running = False
      pygame.quit()
