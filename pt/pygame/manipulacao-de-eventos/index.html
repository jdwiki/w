Ciclo de eventos #

O Pygame registrará todos os eventos do usuário em uma fila de eventos que pode ser recebida com o código pygame.event.get(). Cada elemento nesta fila é um objeto Event e todos eles terão o atributo type, que é um número inteiro que representa que tipo de evento é. No módulo pygame existem constantes inteiras predefinidas que representam o tipo. Exceto por este atributo, os eventos possuem atributos diferentes.

Nome constante Atributos
SAIR nenhum
EVENTO ATIVO ganho, estado
KEYDOWN unicode, chave, mod
TECLADO chave, mod
MOUSEMOÇÃO pos, rel, botões
BOTÃO DO MOUSE pos, botão
BOTÃO DO MOUSE pos, botão
JOYAXISMOTION alegria, eixo, valor
JOYBALLMOTION alegria, bola, rel
JOYHATMOTION alegria, chapéu, valor
JOYBUTTONUP alegria, botão
JOYBUTTONDOWN alegria, botão
REDIMENSIONAR VÍDEO tamanho, w, h
VIDEOEXPOSIÇÃO nenhum
USEREVENT código

Exemplo #

Para lidar com nossos eventos, simplesmente percorremos a fila, verificamos qual é o tipo (com a ajuda das constantes predefinidas no módulo pygame) e, em seguida, executamos alguma ação. Este código verificará se o usuário pressionou o botão fechar no canto superior da tela e, em caso afirmativo, encerrará o programa.

for event in pygame.event.get():
    if event.type == pygame.QUIT:
        # Close the program any way you want, or troll users who want to close your program.
        raise SystemExit

ATENÇÃO: Você precisa chamar a fila de eventos regularmente ao usar o pygame! Além de buscar os eventos disponíveis, chamar a fila de eventos também é a maneira do pygame interagir com o sistema operacional internamente. Se a fila de eventos não for chamada regularmente, seu sistema operacional assumirá que seu programa não funciona mais corretamente e possivelmente fará parecer que o programa travou (no Windows a janela fica branca). Se você não quiser fazer nada com os eventos, você deve chamar pygame.event.pump() a cada loop de jogo para fazer o pygame processar os eventos internamente.

Eventos de teclado #

Existem dois tipos de eventos chave no pygame: KEYDOWN e KEYUP. Esses eventos têm um atributo key que é um número inteiro que representa uma tecla no teclado. O módulo pygame possui constantes inteiras predefinidas representando todas as chaves comuns. As constantes são nomeadas com um K maiúsculo, um sublinhado e o nome da chave. Por exemplo, <- chama-se K_BACKSPACE, a chama-se K_a e F4 chama-se K_F4.

Exemplo #

Este código verificará se o usuário pressionou w, a, s ou d.

for event in pygame.event.get():
    if event.type == pygame.QUIT:  # Usually wise to be able to close your program.
        raise SystemExit
    elif event.type == pygame.KEYDOWN:
        if event.key == pygame.K_w:
            print("Player moved up!")
        elif event.key == pygame.K_a:
            print("Player moved left!")
        elif event.key == pygame.K_s:
            print("Player moved down!")
        elif event.key == pygame.K_d:
            print("Player moved right!")

Modificadores #

Não há constante inteira para letras maiúsculas. Em vez disso, os eventos de tecla têm outro atributo chamado mod, que são os modificadores (shift, ctrl, alt etc.) sendo pressionados simultaneamente como a chave. O atributo mod é um inteiro que representa o modificador que está sendo pressionado. O valor inteiro de cada modificador é armazenado no módulo pygame sob o nome de KMOD_ e seu nome. Por exemplo, Deslocamento à esquerda é denominado KMOD_LSHIFT, Tab é denominado KMOD_TAB e Ctrl é denominado KMOD_CTRL.

Exemplo #

Este código verificará se o usuário pressionou a, Deslocamento à esquerda + a ou Caps + a< /kbd>.

for event in pygame.event.get():
    if event.type == pygame.QUIT:  # It's still wise to be able to close your program.
        raise SystemExit
    elif event.type == pygame.KEYDOWN:
        if event.key == pygame.K_a:
            if event.mod == 0:  # No modifier.
                print("You pressed 'a'")
            elif event.mod == pygame.KMOD_LSHIFT or event.mod == pygame.KMOD_CAPS:
                print("You pressed 'A'")
            else:
                print("You pressed 'a' with another modifier than right shift or caps.")

Eventos do mouse #

Existem três tipos de eventos de mouse no pygame MOUSEMOTION, MOUSEBUTTONDOWN e MOUSEBUTTONUP. O Pygame registrará esses eventos quando um modo de exibição for definido.

MOUSEMOTION é recebido quando o usuário move o mouse na tela. Possui os atributos buttons, pos e rel.

  • buttons é uma tupla que representa se os botões do mouse (left, mouse-wheel, right) estão sendo pressionados ou não.
  • pos é a posição absoluta (x, y) do cursor em pixels.
  • rel é a posição relativa à posição anterior (rel_x, rel_y) em pixels.

MOUSEBUTTONDOWN e MOUSEBUTTONUP são recebidos quando o usuário pressiona ou solta um botão do mouse. Eles têm os atributos button e pos.

  • button é um número inteiro que representa o botão que está sendo pressionado. 1 para o botão esquerdo, 2 para a roda do mouse e 3 para o botão direito.
  • pos é a posição absoluta do mouse (x, y) quando o usuário pressionou o botão do mouse.

Exemplo #

Aqui está um pequeno exemplo usando alguns dos atributos de cada evento de mouse:

for event in pygame.event.get():
    if event.type == pygame.QUIT:  # Close your program if the user wants to quit.
        raise SystemExit
    elif event.type == pygame.MOUSEMOTION:
        if event.rel[0] > 0:  # 'rel' is a tuple (x, y). 'rel[0]' is the x-value.
            print("You're moving the mouse to the right")
        elif event.rel[1] > 0:  # pygame start y=0 at the top of the display, so higher y-values are further down.
            print("You're moving the mouse down")
    elif event.type == pygame.MOUSEBUTTONDOWN:
        if event.button == 1:
            print("You pressed the left mouse button")
        elif event.button == 3:
            print("You pressed the right mouse button")
    elif event.type == pygame.MOUSEBUTTONUP:
        print("You released the mouse button")

Como não há constantes predefinidas para o atributo do botão do mouse no módulo pygame, aqui estão os valores para cada um:

Botão Valor
Botão esquerdo do mouse 1
Botão de roda do mouse 2
Botão direito do mouse 3
Rolagem da roda do mouse para cima 4
Roda do mouse rolar para baixo 5

Rolar o botão do mouse irá gerar eventos pygame.MOUSEBUTTONDOWN e pygame.MOUSEBUTTONUP.

Verificação de estado #

É possível chamar funções do módulo pygame.key e pygame.mouse para receber o estado da chave e do mouse. No entanto, não é a maneira recomendada de processar eventos no pygame, pois existem algumas falhas:

  • Você receberá os estados quando a função for chamada, o que significa que você pode perder eventos entre as chamadas se o usuário estiver pressionando os botões rapidamente.

  • Você não pode determinar a ordem dos eventos.

  • Você ainda precisa chamar uma das funções de evento do pygame para que o pygame interaja internamente com o sistema operacional, caso contrário ele avisará que o programa parou de responder. As funções que você pode chamar são:

  • pygame.event.get() para obter todos os eventos ou tipos de eventos (passando os tipos como um argumento) da fila.

  • pygame.event.poll() para obter um único evento da fila.

  • pygame.event.wait() para esperar por um único evento da fila.

  • pygame.event.clear() para limpar todos os eventos na fila.

  • pygame.event.pump() para permitir que pygame manipule ações internas (é chamado implicitamente pelas funções acima).

Eventos de teclado #

O módulo de chave tem uma função pygame.key.get_pressed() que retorna uma lista do estado de todas as chaves. A lista contém 0 para todas as teclas que não foram pressionadas e 1 para todas as teclas que foram pressionadas. Seu índice na lista é definido por constantes no módulo pygame, todas prefixadas com K_ e o nome da chave.

pygame.event.pump()  # Allow pygame to handle internal actions.
key = pygame.key.get_pressed()
if key[pygame.K_a]:
    print("You pressed 'a'")
if key[pygame.K_F1]:
    print("You pressed 'F1'")
if key[pygame.K_LSHIFT]:
    print("You pressed 'left shift'")
if key[pygame.K_q]:  # Press 'q' to exit the program
    quit()

Se você quiser verificar se há um único pressionamento de tecla em vez de se a tecla estiver pressionada, você pode armazenar o estado anterior de todas as teclas em uma variável temporária e verificar se o valor muda:

pygame.event.pump()  # Allow pygame to handle internal actions.
key = pygame.key.get_pressed()
if key[pygame.K_q] and not previous_key[pygame.K_q]:
    print("You pressed 'q'")
if key[pygame.K_p] and not previous_key[pygame.K_p]:
    print("You pressed 'p'")
previous_key = key

A instrução é avaliada como verdadeira somente quando a tecla atual é pressionada e a tecla anterior não é pressionada. Para verificar se o usuário liberou a chave, basta trocar a palavra-chave not (if not key[pygame.K_q] e previous_key[pygame.K_q]). Para que isso funcione corretamente, você deve definir a variável previous_key = pygame.key.get_pressed() antes do loop do jogo, caso contrário você receberá um NameError.

Eventos do mouse #

O módulo mouse possui funções que nos permitem verificar e definir a posição do mouse, bem como verificar os botões pressionados. A função pygame.mouse.get_pressed() retorna uma tupla tupla representando se os botões do mouse (esquerdo, roda do mouse, direito) estão sendo pressionados ou não.

pygame.event.pump()  # Allow pygame to handle internal actions.
mouse_pos = pygame.mouse.get_pos()
mouse_buttons = pygame.mouse.get_pressed()
if mouse_pos[0] > 100:
    pygame.mouse.set_pos(10, mouse_pos[1])  # Reset the mouse's x-position to 10.
    print("YOU SHALL NOT PASS!")
if mouse_buttons[2]:
    print("I'm right, right?")
if mouse_buttons[0]:  # Press left mouse button to exit.
    print("Program left")
    quit()
