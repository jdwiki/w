Configuração para Mac #

Instalando o gerenciador de pacotes Homebrew brew

Cole isso em um prompt do Terminal.

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Instalando o Xcode IDE

Faça o download usando o link abaixo ou encontre-o na Mac App Store

https://developer.apple.com/download/

OBSERVAÇÃO: se você tiver Xcode-beta.app instalado junto com a versão de produção do Xcode.app, certifique-se de estar usando a versão de produção da ferramenta xcodebuild. Você pode configurá-lo com:

sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer/

Instalando o ambiente Android

  • Vá vai

*Se você instalou o XCode, o Git já está instalado, caso contrário, execute o seguinte

   brew install git

  • Último JDK

  • Android Studio

    Choose a Custom installation

    Choose both Performance and Android Virtual Device

    After installation, choose Configure -> SDK Manager from the Android Studio welcome window.

    In the SDK Platforms window, choose Show Package Details and under Android 6.0 (Marshmallow), make sure that Google APIs, Intel x86 Atom System Image, Intel x86 Atom_64 System Image, and Google APIs Intel x86 Atom_64 System Image are checked.

    In the SDK Tools window, choose Show Package Details and under Android SDK Build Tools, make sure that Android SDK Build-Tools 23.0.1 is selected.

  • Variável de ambiente ANDROID_HOME

    Ensure the ANDROID_HOME environment variable points to your existing Android SDK. To do that, add this to your ~/.bashrc, ~/.bash_profile (or whatever your shell uses) and re-open your terminal:

    If you installed the SDK without Android Studio, then it may be something like: /usr/local/opt/android-sdk

    export ANDROID_HOME=~/Library/Android/sdk

Dependências para Mac

Você precisará do Xcode para iOS e do Android Studio para Android, node.js, das ferramentas de linha de comando React Native e do Watchman.

Recomendamos instalar o node e o watchman via Homebrew.

brew install node
brew install watchman

Watchman é uma ferramenta do Facebook para observar mudanças no sistema de arquivos. É altamente recomendável que você o instale para obter um melhor desempenho. É opcional.

O Node vem com npm, que permite instalar a interface de linha de comando React Native.

npm install -g react-native-cli

Se você receber um erro de permissão, tente com sudo:

sudo npm install -g react-native-cli.

Para iOS, a maneira mais fácil de instalar o Xcode é através da Mac App Store. E para Android baixe e instale o Android Studio.

Se você planeja fazer alterações no código Java, recomendamos o Gradle Daemon, que acelera a compilação.

Testando sua instalação do React Native

Use as ferramentas de linha de comando React Native para gerar um novo projeto React Native chamado “AwesomeProject” e execute react-native run-ios dentro da pasta recém-criada.

react-native init AwesomeProject
cd AwesomeProject
react-native run-ios

Você deve ver seu novo aplicativo em execução no iOS Simulator em breve. react-native run-ios é apenas uma maneira de executar seu aplicativo - você também pode executá-lo diretamente de dentro do Xcode ou Nuclide.

Modificando seu aplicativo

Agora que você executou o aplicativo com sucesso, vamos modificá-lo.

  • Abra index.ios.js ou index.android.js no editor de texto de sua escolha e edite algumas linhas.
  • Pressione Command⌘ + R no seu simulador iOS para recarregar o aplicativo e ver o seu mudança! É isso!

Parabéns! Você executou e modificou com sucesso seu primeiro aplicativo React Native.

Configuração para Linux (Ubuntu) #

1) Configurar Node.JS

Inicie o terminal e execute os seguintes comandos para instalar o nodeJS: #

curl -sL https://deb.nodesource.com/setup_5.x | sudo -E bash -

sudo apt-get install nodejs

Se o comando node não estiver disponível #

sudo ln -s /usr/bin/nodejs /usr/bin/node

Instalações alternativas do NodeJS: #

curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -
sudo apt-get install -y nodejs

ou

curl -sL https://deb.nodesource.com/setup_7.x | sudo -E bash -
sudo apt-get install -y nodejs

verifique se você tem a versão atual #

node -v

Execute o npm para instalar o react-native #

sudo npm install -g react-native-cli

2) Configurar Java

sudo apt-get install lib32stdc++6 lib32z1 openjdk-7-jdk

3) Configure o Android Studio:

Android SDK ou Android Studio #

http://developer.android.com/sdk/index.html

Android SDK e ENV #

export ANDROID_HOME=/YOUR/LOCAL/ANDROID/SDK
export PATH=$PATH:$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools

4) Configure o emulador:

No terminal execute o comando

android

Selecione “Plataformas SDK” no SDK Manager e você verá uma marca de seleção azul ao lado de “Android 7.0 (Nougat)”. Caso não seja, clique na caixa de seleção e depois em “Aplicar”.

5) Inicie um projeto

Exemplo de inicialização do aplicativo #

react-native init ReactNativeDemo && cd ReactNativeDemo

Obs: Sempre verifique se a versão em android/app/build.gradle é a mesma das Build Tools baixadas no seu Android SDK #

android {
    compileSdkVersion XX
    buildToolsVersion "XX.X.X"
...

6) Execute o projeto

Abra o Android AVD para configurar um android virtual. Execute a linha de comando: #

android avd

Siga as instruções para criar um dispositivo virtual e iniciá-lo

Abra outro terminal e execute as linhas de comando:

react-native run-android
react-native start

Configuração para Windows #

Observação: você não pode desenvolver aplicativos nativos de reação para iOS no Windows, apenas aplicativos Android nativos de reação.

Os documentos oficiais de configuração para react-native no Windows podem ser encontrados aqui. Se você precisar de mais detalhes, há um guia granular aqui.

Ferramentas/Ambiente

  • Windows 10
  • ferramenta de linha de comando (por exemplo, Powershell ou linha de comando do Windows)
  • Chocolatey (etapas para configuração via PowerShell)
  • O JDK (versão 8)
  • Estúdio Android
  • Uma máquina Intel com tecnologia de virtualização habilitada para HAXM (opcional, necessária apenas se você quiser usar um emulador)

1) Configure sua máquina para reagir ao desenvolvimento nativo

Inicie a linha de comando como administrador execute os seguintes comandos:

choco install nodejs.install
choco install python2

Reinicie a linha de comando como administrador para poder executar o npm

npm install -g react-native-cli

Depois de executar o último comando, copie o diretório em que o react-native foi instalado. Você precisará disso para a Etapa 4. Eu tentei isso em dois computadores em um caso: C:\Program Files (x86)\Nodist\v- x64\6.2.2. No outro era: C:\Users\admin\AppData\Roaming\npm

2) Defina suas variáveis ​​de ambiente

Um guia passo a passo com imagens pode ser encontrado aqui para esta seção.

Abra a janela Variáveis ​​de Ambiente navegando para:

[Clique com o botão direito] menu “Iniciar” -> Sistema -> Configurações avançadas do sistema -> Variáveis ​​de ambiente

Na seção inferior, encontre a variável de sistema “Path” e adicione o local em que o react-native foi instalado na etapa 1.

Se você não adicionou uma variável de ambiente ANDROID_HOME você terá que fazer isso aqui também. Ainda na janela “Variáveis ​​de ambiente”, adicione uma nova variável de sistema com o nome “ANDROID_HOME” e valor como o caminho para o seu SDK do Android.

Em seguida, reinicie a linha de comando como administrador para poder executar comandos react-native nela.

3) Crie seu projeto Na linha de comando, navegue até a pasta em que deseja colocar seu projeto e execute o seguinte comando:

react-native init ProjectName

4) Execute seu projeto Inicie um emulador do android studio Navegue até o diretório raiz do seu projeto na linha de comando e execute-o:

cd ProjectName
react-native run-android

Você pode ter problemas de dependência. Por exemplo, pode haver um erro que você não tem a versão correta das ferramentas de compilação. Para corrigir isso, você terá que abrir o gerenciador de SDK no Android Studio e baixar as ferramentas de compilação de lá.

Parabéns!

Para atualizar a interface do usuário, você pode pressionar a tecla r duas vezes enquanto estiver no emulador e executando o aplicativo. Para ver as opções do desenvolvedor, você pode pressionar ctrl + m.
