Sintaxe #

  • buscar(url, opções)[.then(…)[.catch(…)]]

  • A API Fetch é a API mais comumente usada para solicitações HTTP. É moderno, flexível e usa promessas.

  • A API XMLHttpRequest também é usada para solicitações HTTP e está incluída principalmente para que os desenvolvedores possam usar suas bibliotecas existentes favoritas, como ApiSauce.

  • A API Websocket pode ser usada para dados “ao vivo” em cenários de tempo real, como em aplicativos de bate-papo.

Usando Promises com a API de busca e Redux #

Redux é a biblioteca de gerenciamento de estado mais comum usada com React-Native. O exemplo a seguir demonstra como usar a API de busca e enviar alterações para o redutor de estado de seus aplicativos usando redux-thunk.

export const fetchRecipes = (action) => {
  return (dispatch, getState) => {
    fetch('/recipes', {
        method: 'POST',
        headers: {
          'Accept': 'application/json',
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          recipeName,
          instructions,
          ingredients
        })
    })
    .then((res) => {
      // If response was successful parse the json and dispatch an update
      if (res.ok) {
        res.json().then((recipe) => {
          dispatch({
            type: 'UPDATE_RECIPE',
            recipe
          });
        });
      } else {
        // response wasn't successful so dispatch an error
        res.json().then((err) => {
          dispatch({
            type: 'ERROR_RECIPE',
            message: err.reason,
            status: err.status
          });
        });
      }
    })
    .catch((err) => {
      // Runs if there is a general JavaScript error.
      dispatch(error('There was a problem with the request.'));
    });
  };
};

HTTP com a API de busca #

Deve-se notar que Fetch não suporta retornos de chamada de progresso. Veja: https://github.com/github/fetch/issues/89.

A alternativa é usar XMLHttpRequest https://developer.mozilla.org/en-US/docs/Web/Events/progress.

fetch('https://mywebsite.com/mydata.json').then(json => console.log(json));

fetch('/login', {
  method: 'POST',
  body: form,
  mode: 'cors',
  cache: 'default',
}).then(session => onLogin(session), failure => console.error(failure));

Mais detalhes sobre a busca podem ser encontrados em MDN

Rede com XMLHttpRequest #

var request = new XMLHttpRequest();
request.onreadystatechange = (e) => {
  if (request.readyState !== 4) {
    return;
  }

  if (request.status === 200) {
    console.log('success', request.responseText);
  } else {
    console.warn('error');
  }
};

request.open('GET', 'https://mywebsite.com/endpoint/');
request.send();

WebSockets #

var ws = new WebSocket('ws://host.com/path');

ws.onopen = () => {
  // connection opened

  ws.send('something'); // send a message
};

ws.onmessage = (e) => {
  // a message was received
  console.log(e.data);
};

ws.onerror = (e) => {
  // an error occurred
  console.log(e.message);
};

ws.onclose = (e) => {
  // connection closed
  console.log(e.code, e.reason);
};

Http com axios #

Configurar

Para solicitação da web, você também pode usar a biblioteca axios.

É fácil de configurar. Para isso você pode criar o arquivo axios.js por exemplo:

import * as axios from 'axios';

var instance = axios.create();
instance.defaults.baseURL = serverURL;
instance.defaults.timeout = 20000;]
//...
//and other options

export { instance as default };

e, em seguida, use-o em qualquer arquivo que desejar.

Solicitações de

Para evitar o uso do padrão ‘canivete suíço’ para cada serviço em seu back-end, você pode criar um arquivo separado com métodos para isso dentro da pasta para funcionalidade de integração:

import axios from '../axios';
import { 
    errorHandling
} from '../common';

const UserService = {
        getCallToAction() {
        return axios.get('api/user/dosomething').then(response => response.data)
            .catch(errorHandling);
    },
}
export default UserService;

Teste

Existe uma biblioteca especial para testar axios: axios-mock-adapter.

Com esta lib você pode definir para axios qualquer resposta que desejar para testá-la. Você também pode configurar alguns erros especiais para seus métodos axois’es. Você pode adicioná-lo ao seu arquivo axios.js criado na etapa anterior:

import MockAdapter from 'axios-mock-adapter';

var mock = new MockAdapter(instance);
mock.onAny().reply(500);

por exemplo.

Loja Redux

Às vezes, você precisa adicionar tokens de autorização de cabeçalhos, que provavelmente armazena em sua loja redux.

Nesse caso, você precisará de outro arquivo, interceptors.js com esta função:

export function getAuthToken(storeContainer) {
    return config => {
        let store = storeContainer.getState();
        config.headers['Authorization'] = store.user.accessToken;
        return config;
    };
}

Em seguida, no construtor do seu componente raiz, você pode adicionar isso:

axios.interceptors.request.use(getAuthToken(this.state.store));

e então todas as suas solicitações serão seguidas com seu token de autorização.

Como você pode ver axios é uma biblioteca muito simples, configurável e útil para aplicações baseadas em react-native.

Web Socket com Socket.io #

Instale o socket.io-client

npm i socket.io-client --save

Módulo de importação

import SocketIOClient from 'socket.io-client/dist/socket.io.js'

Inicialize no seu construtor

constructor(props){
    super(props);
    this.socket = SocketIOClient('http://server:3000');
  }

Agora, para usar sua conexão de soquete corretamente, você deve vincular suas funções no construtor também. Vamos supor que temos que construir uma aplicação simples, que enviará um ping para um servidor via socket a cada 5 segundos (considere isso como ping), e então a aplicação receberá uma resposta do servidor. Para fazer isso, vamos primeiro criar essas duas funções:

_sendPing(){
    //emit a dong message to socket server
    socket.emit('ding');
}

_getReply(data){
    //get reply from socket server, log it to console
    console.log('Reply from server:' + data);
}

Agora, precisamos vincular essas duas funções em nosso construtor:

constructor(props){
    super(props);
    this.socket = SocketIOClient('http://server:3000');

    //bind the functions
    this._sendPing = this._sendPing.bind(this);
    this._getReply = this._getReply.bind(this);
}

Depois disso, também precisamos vincular a função _getReply ao soquete para receber a mensagem do servidor de soquete. Para fazer isso, precisamos anexar nossa função _getReply ao objeto socket. Adicione a seguinte linha ao nosso construtor:

this.socket.on('dong', this._getReply);

Agora, sempre que o servidor de soquete emitir com o ‘dong’, seu aplicativo poderá recebê-lo.
