Exemplo Android #

Uma das dificuldades centrais de escrever um aplicativo Android usando o Dagger é que muitas classes do framework Android são instanciadas pelo próprio sistema operacional, como Activity e Fragment, mas o Dagger funciona melhor se puder criar todos os objetos injetados. Em vez disso, você precisa executar a injeção de membros em um método de ciclo de vida. A partir da versão 2.10, o punhal permite o uso de dagger.android, o que simplifica o uso do punhal com componentes do Android.

Injetando objetos de atividade

  1. Instale o AndroidInjectionModule no componente do aplicativo para garantir que todas as ligações necessárias para esses tipos básicos estejam disponíveis.

     @Component(modules = {AndroidInjectionModule.class})
 public interface AppComponent {}

  2. Comece escrevendo um @Subcomponent que implementa [AndroidInjector][AndroidInjector], com um @Subcomponent.Builder que estende [AndroidInjector.Builder][AndroidInjector.Builder]:

     @Subcomponent
 public interface MainActivityComponent extends AndroidInjector<MainActivity> {
     @Subcomponent.Builder
     abstract class Builder extends AndroidInjector.Builder<MainActivity> {}
 }

  3. Após definir o subcomponente, adicione-o à sua hierarquia de componentes definindo um módulo que vincule o construtor do subcomponente e adicionando-o ao componente que injeta seu Application:

     @Module(subcomponents = MainActivityComponent.class)
 public abstract class MainActivityModule {

     @Binds @IntoMap @ActivityKey(MainActivity.class)
     abstract AndroidInjector.Factory<? extends Activity>
     bindMainActivityInjectorFactory(MainActivityComponent.Builder builder);
 }

  4. Em seguida, faça seu Application implementar HasDispatchingActivityInjector e @Inject um DispatchingAndroidInjector<Activity> para retornar do método activityInjector():

     public class MyApp extends Application implements HasDispatchingActivityInjector     {

     @Inject
     DispatchingAndroidInjector<Activity> dispatchingActivityInjector;

     @Override
     public void onCreate() {
        super.onCreate();
        DaggerAppComponent.create().inject(this);
     }

     @Override
     public DispatchingAndroidInjector<Activity> activityInjector() {
         return dispatchingActivityInjector;
     }
 }

  5. Finalmente, em seu método Activity.onCreate(), chame AndroidInjection.inject(this) antes de chamar super.onCreate();:

     public class MainActivity extends Activity {
     public void onCreate(Bundle savedInstanceState) {
         AndroidInjection.inject(this);
         super.onCreate(savedInstanceState);
     }
 }

Este exemplo foi baseado em documentação oficial da adaga. Exemplo de trabalho pode ser encontrado em github

Descrição e configuração #

O que é Adaga 2?

O site se descreve como:

Dagger é uma estrutura de injeção de dependência em tempo de compilação totalmente estática

A biblioteca facilita a modelagem de gráficos de dependência, bem como a reutilização de objetos. Como a reflexão é usada apenas em tempo de compilação como parte do processamento de anotações, o Dagger 2 melhorou a velocidade para injeção de dependência.

Configurar

1- Adicione suporte para processamento de anotações:

Android

Script build.gradle de nível superior:

 repositories {
    mavenCentral()
  }
  dependencies {
    classpath 'com.neenbedankt.gradle.plugins:android-apt:1.8'
  }

Script build.gradle no nível do módulo:

apply plugin: 'com.neenbedankt.android-apt'

Java

plugins {
  id "net.ltgt.apt" version "0.5"
}

2- Adicione as dependências do punhal 2

 dependencies {
      compile 'com.google.dagger🗡2.x'
      apt 'com.google.dagger:dagger-compiler:2.x'
    }

Aprenda Dagger2 com um exemplo simples #

Eu li e assisti muitos tutoriais diferentes do Dagger2, mas a maioria deles são muito longos ou difíceis de entender, então decidi escrever um novo tutorial simples e curto para o Dagger2, espero que gostem.

Por que precisamos disso? #

  • Simplifica o acesso a instâncias compartilhadas: Fornece uma maneira simples de obter referências a instâncias compartilhadas, por exemplo, uma vez que declaramos no Dagger nossas instâncias singleton, como SharedPrefrences, podemos declarar campos com uma simples anotação @Inject.
  • Testes unitários e de integração mais fáceis: podemos trocar facilmente os módulos que fazem as respostas da rede e simular esse comportamento.

Vamos começar com um exemplo simples #

A fonte completa do exemplo está disponível na minha conta do GitHub.

Adicionar dependências do Dagger2 #

Antes de tudo, precisamos adicionar as dependências do Dagger2. Coloque o código abaixo no seu arquivo build.gradle no nível do módulo.

compile "com.google.dagger🗡$dagger_version"
compile "com.google.dagger:dagger-android:$dagger_version"
compile "com.google.dagger:dagger-android-support:$dagger_version"
annotationProcessor "com.google.dagger:dagger-compiler:$dagger_version"

Se você estiver recebendo um erro como Error:Conflict with dependency ‘com.google.code.findbugs:jsr305’ no projeto ‘:app’, adicione o seguinte ao seu arquivo app/build.gradle principal.

configurations.all {
   resolutionStrategy.force 'com.google.code.findbugs:jsr305:3.0.1'
}

Duas classes simples #

Temos duas classes (Vehicle e Motor), a classe Vehicle precisa da classe Motor para funcionar e MainActivity precisa da classe Vehicle. Usaremos o Dagger2 para fornecer essas instâncias.

class Vehicle {
   private Motor motor;

  @Inject
  Vehicle(Motor motor) {
     this.motor = motor;
  }

  void increaseSpeed(int value) {
     motor.accelerate(value);
  }

  void decreaseSpeed(int value) {
     motor.decelerate(value);
  }

  void stop() {
     motor.brake();
  }

  int getSpeed() {
     return motor.getRpm();
  }
}

Classe motora:

class Motor {
  private int rpm;

  Motor() {
    this.rpm = 0;
  }

  int getRpm() {
    return rpm;
  }

  void accelerate(int value) {
    rpm += value;
  }

  void decelerate(int value) {
    rpm -= value;
  }

  void brake() {
    rpm = 0;
  }
}

Classe do módulo #

A classe Module é responsável por fornecer objetos que podem ser injetados. Neste exemplo, queremos injetar a classe Motor na classe Vehicle e injetar a classe Vehicle na MainActivity, portanto, devemos criar MyModule para fornecer essas instâncias.

@Module
class MyModule {

  @Provides
  @Singleton
  Motor provideMotor() {
    return new Motor();
  }

  @Provides
  @Singleton
  Vehicle provideVehicle() {
    return new Vehicle(new Motor());
  }
}

@Provide annotation: o objeto retornado deste método está disponível para injeção de dependência.

@Interface de componentes #

Dagger2 precisa de interface de componentes para saber como deve criar instâncias de nossas classes.

@Singleton
@Component(modules = {MyModule.class})
interface MyComponent {
  Vehicle provideVehicle();

  void inject(MainActivity main);
}

@Component interface: conexão entre o provedor do objeto e os objetos que expressam uma dependência.

Injetar dependência no construtor #

Ao adicionar a anotação @Inject, dagger2 pode criar automaticamente uma instância desse objeto como nosso exemplo de objeto Motor na classe Vehicle.

Injetar dependência em MainClass #

O Dagger2 pode injetar dependências automaticamente nos construtores, mas os componentes do Android (atividades, fragmentos, etc.)

public class MainActivity extends AppCompatActivity {
  @Inject
  Vehicle vehicle;

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    MyComponent component = DaggerMyComponent.builder().build();
    component.inject(this);
  }
}

É isso, espero que gostem.

Exemplo básico #

Defina um módulo (o modelo de dependências e seu gráfico):

@Module
public class CoffeeModule{

    @Provides
    public CoffeeMaker provideCoffeeMaker(){
         return new CoffeeMaker();
    }

    @Provides
    public Coffee provideCoffee(CoffeeMaker coffeeMaker){
        return new Coffee(coffeeMaker);
    }

}

Defina um componente:

@Component(
    modules={
        CoffeeModule.class
    }
)
interface CoffeeComponent {

        DeveloperActivity inject(DeveloperActivity developerActivity);
}

Injete as dependências:

class DeveloperActivity extends ...{

    @Inject
    Coffee myCoffee;

    @Override
    protected void onCreate(Bundle savedInstanceState) {           
        super.onCreate(savedInstanceState);

        DaggerCoffeeComponent.builder()
                .coffeeModule(new CoffeeModule())
                .build()
                .inject();
       
    }

}
