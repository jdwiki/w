Colocar fiapos em seu aplicativo iônico antes de executá-lo tem grandes vantagens. Ele analisará o código em busca de possíveis erros e economizará muito tempo.

O que é linting e como instalar os pacotes necessários? #

“Linting é o processo de execução de um programa que analisará o código em busca de possíveis erros.” - veja O que é “Linting”?

Seu aplicativo iônico vem com um arquivo package.json. Vá para a raiz do seu aplicativo em uma linha de comando/terminal e instale os seguintes pacotes:

npm install jshint --save-dev
npm install jshint-stylish --save-dev
npm install gulp-jshint --save-dev

Adicione uma tarefa gulp #

Na raiz do seu aplicativo iônico, há um arquivo gulpfile.js. Abra-o em um editor e cole a seguinte tarefa gulp:

gulp.task('lint', function() {
    return gulp.src(['./www/js/**/*.js']) 
        .pipe(jshint('.jshintrc'))
        .pipe(jshint.reporter('jshint-stylish')) 
        .pipe(jshint.reporter('fail'))
});

Isso procura uma pasta chamada ‘js’ dentro da pasta ‘www’. Se você tiver outras pastas contendo arquivos JavaScript, adicione-as também. Por exemplo, vamos também adicionar uma pasta chamada ‘views’:

gulp.task('lint', function() {
    return gulp.src(['./www/js/**/*.js','./www/views/**/*.js'])
        .pipe(jshint('.jshintrc'))
        .pipe(jshint.reporter('jshint-stylish')) 
        .pipe(jshint.reporter('fail'))
});

Explicações:

1) /**/*.js - This syntax means to look at all the js files in the subfolders too
2) .jshintrc - This is a configuration file that we will create in the next example.

Criar arquivo .jshintrc (opcional) #

Crie um arquivo chamado ‘.jshintrc’ na raiz do seu aplicativo, onde package.json está.

*Observação no Windows: crie um arquivo chamado “jshintrc.txt”. Em seguida, renomeie-o para “.jshintrc”. (observe o ponto no final).

Este é um arquivo de configuração. Ele pode, por exemplo, dizer ao jshint para ignorar certas variáveis ​​e muitas outras coisas. Aqui é minha:

{
    "predef": [ 
        "window",
        "console",
        "cordova",
        "device",
        "alert",
        "document",
        "debug",
        "setServiceVars",
        "StatusBar",
        "config"
    ],
    "globals": {
        "angular"        : false,
        "myApp"            : false,
        "myControllers"    : false,
        "myDirectives"    : false,
        "localStorage"     : false,
        "navigator"         : false, 
        "emit"             : false, 
        "atob"             : false,  
        "moment"        : false,
        "btoa"            : false
    },
    "node"          : true
}

Adicionar Makefile #

  1. Crie um arquivo chamado: “Makefile” (sem extensão) na raiz do seu aplicativo

  2. Abra-o em um editor de texto e adicione isto:

    android: gulp lint gulp sass ionic run android –device

    ios: gulp lint gulp sass ionic build ios

Isso fará com que seu aplicativo seja solto e, se isso for aprovado, compilará o sass e criará seu aplicativo.

Uso: para executar seu aplicativo, em vez do “ionic run android –device” normal, execute estes comandos:

Android: make android
iOS    : make ios
