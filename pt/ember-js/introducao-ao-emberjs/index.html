Instalação ou configuração #

Começar com o Ember é fácil. Projetos Ember são criados e gerenciados através de nossa ferramenta de construção de linha de comando Ember CLI. Esta ferramenta fornece:

  • Gerenciamento de ativos de aplicativos modernos (incluindo concatenação, minificação e controle de versão).
  • Geradores para ajudar a criar componentes, rotas e muito mais.
  • Um layout de projeto convencional, facilitando a abordagem dos aplicativos Ember existentes.
  • Suporte para JavaScript ES2015/ES6 através do projeto Babel. Isso inclui suporte para módulos JavaScript, que são usados ​​em todo este guia.
  • Um equipamento de teste QUnit completo.
  • A capacidade de consumir um ecossistema crescente de Ember Addons.

Dependências #

Node.js e npm #

O Ember CLI é construído com JavaScript e espera o Node.js tempo de execução. Também requer dependências buscadas via npm. npm é empacotado com Node.js, portanto, se o seu computador tiver Node.js instalado você está pronto para ir.

Ember requer Node.js 0.12 ou superior e npm 2.7 ou superior. Se você não tiver certeza se tem o Node.js ou a versão correta, execute-o em seu linha de comando:

node --version
npm --version

Se você receber um erro “comando não encontrado” ou uma versão desatualizada do Node:

  • Usuários de Windows ou Mac podem baixar e executar este instalador do Node.js.
  • Os usuários de Mac geralmente preferem instalar o Node usando o Homebrew. Depois instalando o Homebrew, execute brew install node para instalar o Node.js.
  • Os usuários do Linux podem usar este guia para instalação do Node.js no Linux.

Se você obtiver uma versão desatualizada do npm, execute npm install -g npm.

Git #

O Ember requer o Git para gerenciar muitas de suas dependências. Git vem com Mac OS X e a maioria das distribuições Linux. Os usuários do Windows podem baixe e execute este instalador do Git.

Vigilante (opcional) #

No Mac e no Linux, você pode melhorar o desempenho de visualização de arquivos instalando o Watchman.

PhantomJS (opcional) #

Você pode executar seus testes a partir da linha de comando com PhantomJS, sem o necessidade de um navegador para ser aberto. Consulte as instruções de download do PhantomJS.

Instalação #

Instale o Ember usando o npm:

npm install -g ember-cli

Para verificar se sua instalação foi bem-sucedida, execute:

ember -v

Se um número de versão for mostrado, você está pronto para começar.

Atribuir portas localhost (especialmente permissões/problemas de disponibilidade, executando vários sites ember simultaneamente) #

Ocasionalmente, é útil atribuir uma ou mais portas manualmente em vez de usar os padrões. Isso pode resolver problemas de disponibilidade/permissões de porta ou acomodar a execução de mais de uma instância ember por vez.

Para que o ember-cli tente identificar e atribuir uma porta disponível, use:

ember serve --port 0

Por ember help: “Passe 0 para escolher automaticamente uma porta disponível”. (Em um terminal, digite ember help).

Para executar mais de um site ember ao mesmo tempo, cada um precisa de seu próprio servidor e portas de recarga ao vivo. Uma abordagem simples: em janelas de Terminal separadas, navegue para cada instância e use o seguinte para iniciá-las com suas próprias portas:

ember serve --port 0 --live-reload-port 0

Se você receber um erro sobre disponibilidade ou permissão em qualquer um desses casos, digite o seguinte script python no prompt do Terminal para identificar uma porta disponível:

python -c 'import socket; s=socket.socket(); s.bind(("", 0)); print(s.getsockname()[1]); s.close()'

Use os resultados para especificar as portas que agora você sabe que estão disponíveis:

ember serve --port <known_port_1> --live-reload-port <known_port_2>

Criando aplicativo #

O Ember CLI permite que você use uma das duas opções para gerar um novo aplicativo:

  1. Crie uma pasta e execute ember init (gera a estrutura do aplicativo e configura o git e faz seu primeiro commit)
  2. Execute ember new <app name> (cria uma pasta com o nome especificado, entra nela e executa ember init)

Quando o processo de geração estiver concluído, inicialize um servidor de recarga ao vivo na pasta do aplicativo executando:

ember server

ou ember s para abreviar. *Ta-da, agora você tem um aplicativo Ember em execução! Documentos oficiais

Criando seu primeiro modelo

Vamos criar um novo template usando o comando ember generate.

ember generate template application

O modelo application está sempre na tela quando um usuário está visitando seu aplicativo. No editor de sua escolha, abra app/templates/application.hbs e adicione o seguinte código:

<h2>My first Ember application</h2>

{{outlet}}

Agora você deve ver o texto recém-adicionado na página de boas-vindas do seu aplicativo. Observe também que o Ember detecta automaticamente o novo arquivo e recarrega a página para você. Legal, certo?

Implantar aplicativo #

Para implantar um aplicativo Ember, basta transferir a saída da compilação ember para um servidor web. Isso pode ser feito com ferramentas padrão de transferência de arquivos Unix, como rsync ou scp. Há também serviços que permitem que você implante facilmente.

ember build
scp -r dist/* myserver.com:/var/www/public/

normalmente usaríamos ember build --environment=production que faz mais para deixar seu código pronto para produção (gzip e minify code).
