Parâmetro Detalhes

$addFieldToFilter($field, $condition = null) { mixed } A definição do filtro que usaremos.

Argumentos de comparação de filtros

O Magento também oferece uma maneira flexível de filtrar usando operadores de comparação. Aqui está uma lista de operadores válidos e sua sintaxe:

Todos os argumentos de comparação podem ser passados ​​para o segundo parâmetro dos métodos addFieldToFielter() ou addAttributeToFilter().

$collection_of_products->addAttributeToFilter('visible',array("eq"=>1));
Comparação Matriz de argumentos Fragmento SQL resultante
Equals array(“eq”=>$var) WHERE (`my_field` = $var)
Diferente array(“neq”=>$var) WHERE (`my_field` != $var)
Like array(“like”=>$var) WHERE (`my_field` LIKE $var)
Not Like array(“nlike”=>$var) WHERE (`my_field` NOT LIKE $var)
É array(“é”=>$var) ONDE (`meu_campo` É $var)
In array(“in”=>$var) WHERE (`my_field` IN($var))
Not In array(“nin”=>$var) WHERE (`my_field` NOT IN($var))
Null array(“null”=>true) WHERE (`my_field` IS NULL)

Filtrando coleções #

Magento tem um poderoso conjunto de métodos para filtrar coleções. Como existem dois tipos de objetos que podem estar contidos em coleções, devemos primeiro determinar com que tipo de dados estamos trabalhando antes de poder filtrá-los. Magento implementa um modelo de dados EAV para entidades como produtos e categorias. Há um conjunto diferente de métodos para usar se estivermos filtrando uma coleção de objetos EAV.

No Magento, os pedidos não são armazenados como objetos EAV. Isso torna a coleção de pedidos um bom exemplo para filtrar uma coleção básica.

$collection_of_orders = Mage::getModel('sales/order')->getCollection();
$collection_of_orders->addFieldToFilter('status','processing');

Se observarmos a coleção de produtos, podemos ver que os produtos estão armazenados em um modelo de dados EAV. Também podemos filtrar facilmente por atributos EAV.

$collection_of_products = Mage::getModel('catalog/product')->getCollection();
$collection_of_products->addAttributeToFilter('visible',1);

Manipulando ANDs e ORs em Filtros #

Quando consultamos nossos dados, geralmente precisamos de mais de um filtro para obter o conjunto de dados exato que estamos procurando. No SQL, lidamos com isso com cláusulas AND e OR. Podemos conseguir a mesma coisa com coleções.

Para adicionar uma cláusula AND à sua consulta, basta adicionar outra chamada de método. Isso anexará o segundo filtro à instrução WHERE original, unindo-o com um AND.

Mage::getModel('catalog/product')->getCollection()
        ->addFieldToFilter('sku',array('like'=>'a%'))
        ->addFieldToFilter('sku',array('like'=>'%b'));

A cláusula WHERE resultante ficará assim:

WHERE (e.sku like 'a%') AND (e.sku like '%b')

Agora vamos dizer que queremos todos os skus que começam com ‘a’ OU terminam com ‘b’. Como adicionamos uma cláusula OR? Graças às coleções do Magento, é bastante simples. Adicionamos o filtro como um segundo elemento na matriz de filtros.

Mage::getModel('catalog/product')->getCollection()
        ->addFieldToFilter('sku', array(
            array('like'=>'a%'), 
            array('like'=>'%b')
        ));

Agora, a cláusula WHERE resultante ficará assim:

WHERE (((e.sku like 'a%') or (e.sku like '%b')))
