  • Criando um módulo do zero (Hello World)

Os módulos existem para serem estendidos. Você não pode alterar os arquivos the app/code/ sem proibir quaisquer atualizações futuras. Em vez disso, adicionamos um módulo ao diretório app/code/local (o diretório local pode estar faltando, se estiver, ele precisa ser criado manualmente. Isso é comum em versões posteriores do Magento) para adicionar funcionalidade local personalizada.

Todos os arquivos de configuração do módulo começam com uma tag <config>. O novo módulo é declarado dentro da tag <modules>. Estaremos chamando um módulo chamado YOUR_COMPANY_HelloWorld, portanto as tags <YOUR_COMPANY_HelloWorld> são usadas. Aqui definimos a versão (primeiro = 0.0.1)

Uma lista de eventos XML pode ser encontrada em: http://www.magentocommerce.com/wiki/5_-_modules_and_development/reference/module_config.xml

Se você estiver tendo algum problema, confira: http://coding.smashingmagazine.com/2012/11/30/introducing-magento-layout/

Criando um módulo do zero (Hello World) #

O desenvolvimento de módulos personalizados Magento é uma parte essencial de qualquer desenvolvimento Magento ou projeto Magento, porque em qualquer estágio você pode querer integrar sua própria funcionalidade/módulo em seu projeto Magento existente.

A primeira coisa que os desenvolvedores devem desabilitar é o cache do sistema. Caso contrário, o desenvolvimento se tornará uma dor, pois qualquer alteração exigirá uma lavagem. No painel de administração do magento: navegue até Sistema > Gerenciamento de cache > Selecionar tudo > Ações: Desativar. Use o seguinte guia para criar um novo módulo:

  • Crie uma pasta em app/code/local/ - convenções de nomenclatura normalmente pegue o nome da sua empresa, por exemplo app/code/local/<YOUR_COMPANY>.

  • Agora temos um local personalizado para nossos módulos. Crie outro diretório, chame-o de algo relacionado ao tipo de módulo que você deseja criar, por exemplo app/code/local/<YOUR_COMPANY>/HelloWorld/ - “HelloWorld” é como chamarei este Módulo.

  • Este diretório precisa de um config.xml para que o Magento o reconheça como um novo módulo. Crie outra pasta chamada etc. Seguido por um arquivo xml chamado config.xml. O diretório deve se parecer com app/code/local/<YOUR_COMPANY/HelloWorld/etc/config.xml Esta é a estrutura do arquivo xml:

     <?xml version="1.0" encoding="UTF-8" ?>
 <config>
    <modules>
       <YOUR_COMPANY_HelloWorld>
          <version>0.0.1</version>
       </YOUR_COMPANY_HelloWorld>
    </modules>
 </config>

  • Em seguida, os módulos precisam ser declarados no Magento. Prossiga para app/etc/modules. Crie outro documento XML e dê a ele os nomes escolhidos de suas tags: YOUR_COMPANY_HelloWorld no meu caso. Neste documento escreva:

     <?xml version="1.0" encoding="UTF-8"?>
 <config>
    <modules>
       <YOUR_COMPANY_HelloWorld>   
          <!-- Whether our module is active: true or false -->
          <active>true</active>
          <!-- Which code pool to use: core, community or local -->
          <codePool>local</codePool>
       </YOUR_COMPANY_HelloWorld>
    </modules>
 </config>

  • Novamente, as tags de configuração e módulo são usadas para declarar um novo módulo para o Magento. Ativo é o valor padrão que pode ser acessado no Painel de administração em Sistema > Configuração > Avançado. codePool diz ao Magento em qual diretório procurar. local no nosso caso

  • Este módulo já foi configurado, assim o Modelo da nossa estrutura MVC. você deve ser capaz de ver seu novo módulo no Painel Administrativo em Sistema > Configuração > Avançado. No entanto, ele não faz nada ainda! Você precisará voltar ao nosso arquivo config.xml e definir os elementos XML.

  • Seguindo com o tutorial; Usaremos alguns desses Elementos XML para criar classes e manipular todas as páginas no frontend do nosso site. De volta ao arquivo config.xml escreva o seguinte sob a tag </modules>:

     <global>
    <!-- adding a new block definition -->
    <blocks>
        <!-- A unique short name for our block files -->
        <helloworld>
           <!-- the location of our modules block -->
           <class>YOUR_COMPANY_HelloWorld_Block</class>
        </helloworld>
    </blocks>
 </global>
 <!-- We are making changes to the frontend -->
 <frontend>
    <!-- We are making changes to the layout of the front end -->
    <layout>
       <!-- we are adding a new update file -->
       <updates>
          <!-- Creating the name of our update and linking it the module -->
          <helloworld module="YOUR_COMPANY_HelloWorld">
              <!-- The name of the layout file we are adding -->
              <file>helloworld.xml</file>
          </helloworld>
      </updates>
   </layout>
 </frontend>

  • Como você pode ver, estamos constantemente estendendo em vez de manipular os arquivos principais. A tag helloworld é minúscula porque apontará para um Handle e, para dar continuidade, a nomearemos o mais próximo possível. Em seguida, vinculamos isso ao módulo YOUR_COMPANY_HelloWorld.

  • Estamos mudando o layout. Portanto, precisamos criar este Handle no diretório de layout. vá para app/design/frontend/base/default/layout. Dissemos ao módulo para procurar o arquivo helloworld.xml. Portanto, devemos criá-lo neste diretório. O que você está esperando. Faça isso!! e preencha com:

    <?xml version="1.0" encoding="UTF-8"?>
<!-- All Layout files begin with this code -->
<layout>
   <!-- this is the layout handle. Default is handled on all pages. We want this module to execute on all pages -->
   <default>
      <!-- This is the block we want to bolt onto the existing before_body_end block -->
      <reference name="before_body_end">
          <!-- We define our new block and template to be added to before_body_end -->
          <block name="helloworld_footer" template="helloworld/footer.phtml" type="helloworld/footer"/>
      </reference>
   </default>
</layout>

  • Agora, aqueles de vocês que têm um pouco de experiência em Magento, ou leram outros tutoriais de Magento mais notáveis, podem estar ofegantes com o fato de estarmos fazendo alterações na base/padrão, pois é onde os arquivos principais do Magento estão localizados. No entanto, não estamos modificando nenhum arquivo aqui, estamos criando novos e, além disso, estamos prefixando nosso nome de arquivo com “helloworld”, portanto, há muito pouca chance de entrar em conflito com outros módulos ou causar problemas com a atualização do Magento no futuro. Dias felizes!

  • Como queremos afetar todas as páginas, usamos a tag padrão e a referenciamos ao bloco estrutural before_body_end. Isso fará o papel da ação e acionará a seção View da nossa estrutura MVC.

  • Agora entendemos que estamos entrando no bloco before_body_end. e vinculá-lo ao nosso bloco personalizado. Isso é chamado de Referência e é um gancho. No momento, não o estamos conectando a nada existente, portanto, devemos criar os arquivos necessários.

  • Em helloworld.xml declaramos no template um footer.phtml. Prossiga para app/design/frontend/base/default/template e crie um diretório helloworld.

  • Dentro deste diretório crie o arquivo footer.phtml e preencha com HTML, este tutorial simplesmente escreve isso para exibir algumas funcionalidades do PHP vinculadas ao nosso arquivo PHTML:

     <p>Hello Everyone! Todays date is <?php echo $this->getDate() ?></p>

  • Agora precisamos criar nosso próprio objeto de bloco para acoplar o modelo com nossa funcionalidade de bloco. Crie o diretório app/code/local/YOUR_COMPANY/HelloWorld/Block/ e crie o arquivo Footer.php dentro dele. Isso foi referenciado em nosso zeta_layout.xml no tipo “helloworld/footer”. Preencha este arquivo com:

     <?php
 class YOUR_COMPANY_HelloWorld_Block_Footer extends Mage_Core_Block_Template {
    public function getDate() {
       $date = date('Y-m-d');
       return urlencode($date);
    }
 }
 ?>

  • Esta é a funcionalidade que irá preencher nossa chamada getDate() que chamamos de nosso arquivo .phtml. Estendemos o Mage_Core_Block_Template.

  • Esta funcionalidade está agora completa. Teste isso indo para sua página inicial, onde você deve ver seu módulo no rodapé de cada página!
