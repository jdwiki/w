Entendendo os tipos de produtos

  • Mage_Catalog_Model_Product_Type

Nesta página

  • Mage_Catalog_Model_Product_Type
  • Descreva os tipos de produtos padrão (simples, configuráveis, agrupados)

Existem seis tipos de produtos diferentes integrados ao Magento.

  • Simples

Uma única unidade de estoque

  • Configurável

Primeiro dos produtos compostos. Permita que os clientes configurem seus produtos e adicionem um único produto simples à cesta.

  • Agrupado

O segundo produto composto, um produto agrupado, relaciona produtos simples e oferece ao cliente a possibilidade de escolher as quantidades de cada item.

  • Pacote

O terceiro tipo de produto composto , um pacote relaciona produtos simples para serem adquiridos como um único item.

  • Virtual

Nenhum item físico necessário para entrega, por exemplo. Serviços

  • Para download

Um produto digital em vez de físico. A maioria dos tipos de produtos são implementados como parte do módulo Mage_Catalog, exceto Mage_Bundle e Mage_Downloadable.

Os produtos agrupados, em pacote e configuráveis ​​implementam um relacionamento pai-filho em que vários outros produtos (por padrão, simples, virtuais ou para download) são atribuídos a um produto principal. Isso lida com os dados do produto para toda a coleção (por exemplo, grupo, pacote ou nome do produto configurável, preço e status).

Os produtos para download e pacote têm tabelas extras no banco de dados, enquanto o restante é compartilhado entre todos os outros tipos de produtos. Os produtos configuráveis ​​têm uma tabela extra para vincular a produtos filhos, catalog_product_super_link.

Tipo de produto personalizado

Para criar um tipo de produto que estenda um dos tipos de produto integrados, o modelo de tipo de produto correspondente deve ser estendido. Caso contrário, o novo tipo de produto deve estender a classe Mage_Catalog_Model_Product_Type_Abstract.

Uma entrada no config.xml do módulo também é necessária:

<{name}> Produtos mais complicados podem exigir outras áreas personalizadas, como modelo de preço e recuperador de dados de índice.

Cálculo de Preço

Ao lidar com um único produto, o preço é sempre calculado em tempo real. O atributo preço EAV é carregado com o produto e o preço final é calculado pelo modelo de preço, Mage_Catalog_Model_Product_Type_Price.

Alguns tipos de produtos lidam com isso de maneira diferente. Nesse caso, eles estendem essa classe e implementam sua própria lógica. Por exemplo, o produto configurável substitui getFinalPrice() e adiciona lógica adicional. Esse modelo personalizado pode ser especificado em config.xml com uma tag <price_model>.

As coleções de produtos, no entanto, utilizam o índice de preços para recuperar os preços pré-calculados, eliminando a necessidade de calculá-lo para cada produto.

O preço final pode ser ajustado pelos observadores do evento catalog_product_get_final_price. Por padrão, apenas o módulo Mage_CatalogRule observa esse evento.

Outro método para substituir o preço do produto é simplesmente defini-lo no produto. Se o preço for definido, o produto não o recalculará.

O preço do nível do produto é separado do preço normal (embora levado em consideração no cálculo do preço). Ele é implementado como uma lista com um grupo de clientes e qualificadores de quantidade mínima para cada nível. Os preços dos níveis são exibidos em uma tabela, usando o modelo catalog/product/view/tierprices.phtml.

As opções de produtos personalizados são processadas ao calcular o preço final. Cada opção tem seu próprio preço definido, que é adicionado ao preço final.

Os preços de grupo, nível e especiais são considerados ao mesmo tempo ($priceModel->getBasePrice()) e o menor dos três (ou quatro se você incluir o preço normal) é escolhido como o preço base do produto.

Imposto

O módulo Mage_Tax usa a classe de imposto de um produto e se o preço do produto inclui ou não impostos para identificar a taxa correta a ser aplicada.

Os seguintes fatores são usados ​​para calcular o imposto sobre os produtos:

  • Classe fiscal do produto
  • O valor do imposto já incluído
  • Endereços de cobrança e envio
  • Classe fiscal do cliente
  • Configurações da loja

Navegação em camadas

As classes responsáveis ​​por renderizar a navegação em camadas são:

  • Mage_Catalog_Block_Layer_View

  • Lida com os filtros e opções

  • Mage_Catalog_Block_Layer_State

-Controla o que está sendo filtrado atualmente

Para implementar a navegação em camadas em atributos com modelos de origem personalizados, o método Mage_Catalog_Model_Layer_Filter_Abstract::apply() precisaria ser substituído para ditar como a coleção de produtos deve ser filtrada.

A navegação em camadas é renderizada pelos blocos Mage_Catalog_Block_Layer_View e Mage_Catalog_Block_Layer_State, que usam blocos de filtro para filtros individuais.

A Navegação em Camadas usa a tabela de índice para a maioria dos filtros, por exemplo, preço, índice de atributo do produto, índice de produto decimal.

Categorias

Categorias no banco de dados

A hierarquia de categorias é gerenciada armazenando o ID pai de uma categoria. A hierarquia completa é mostrada na coluna de caminho (IDs separados por barras). Existe uma categoria especial com parent_id de 0. Esta é a verdadeira categoria raiz e cada uma das outras categorias raiz definidas no Magento usam isso como um pai compartilhado.

Para ler e gerenciar uma árvore de categorias do banco de dados, duas classes diferentes são usadas dependendo se o catálogo simples está ativado, Mage_Catalog_Model_Resource_Category_Tree e Mage_Catalog_Model_Resource_Category_Flat.

A vantagem das categorias planas é que é mais rápido consultar. No entanto, ele precisa ser reconstruído a partir das tabelas EAV sempre que houver uma alteração.

getChildren()

retorna uma string separada por vírgulas de IDs filhos imediatos

getAllChildren()

retorna uma string ou array de todos os IDs filhos

getChildrenCategories()

retorna uma coleção de categorias filhas imediatas N.B. Se o catálogo simples estiver habilitado, as únicas categorias filhas retornadas serão aquelas com include_in_menu = 1. Em ambos os casos, somente as categorias ativas são retornadas.

Regras de preços de catálogo

As regras de preço de catálogo aplicam descontos a produtos com base na data, produto, site e grupo de clientes.

Quando getFinalPrice() é chamado em um produto, o evento catalog_product_get_final_price é acionado. Isso é observado por Mage_CatalogRule_Model_Observer, que procurará qualquer regra de preço de catálogo que se aplique ao produto. Se aplicável, ele analisa a tabela de preços do banco de dados e grava o preço de volta no modelo do produto como um campo de dados Varien final_price.

Dentro do banco de dados, a tabela de regras do catálogo descreve as regras, suas condições e suas ações. catalogrule_product contém os produtos correspondentes e algumas informações de regras. Enquanto isso, catalogrule_product_price contém o preço após a aplicação da regra.

Indexação e tabelas planas

As tabelas de catálogo simples são gerenciadas por indexadores de catálogo. Se a reconstrução automática dos índices estiver habilitada, os indexadores de catálogo serão reconstruídos sempre que um produto, categoria ou qualquer entidade relacionada for atualizada. O método _afterSave() chama o processo do indexador. Caso contrário, eles precisam ser reindexados manualmente por meio do administrador.

O tipo de produto afeta o índice de preços e o índice de ações onde os produtos podem definir seus próprios indexadores personalizados (em config.xml) para manipular seus dados para esses índices.

O módulo Mage_Index fornece a estrutura com a qual os índices personalizados podem ser criados para ajudar a otimizar o desempenho do site. A classe Mage_Index_Model_Indexer_Abstract deve ser estendida para criar um novo índice, implementando os métodos _registerEvent() e _processEvent(). Não esquecendo de registrá-lo no config.xml:

<global>
    <index>
        <indexer>
            <{name}>{model}</{name}>
        </indexer>
    </index>
</global>

/**
 * Product type model
 *
 * @category    Mage
 * @package     Mage_Catalog
 * @author      Magento Core Team 
 */
class Mage_Catalog_Model_Product_Type
{
    /**
     * Available product types
     */
    const TYPE_SIMPLE       = 'simple';
    const TYPE_BUNDLE       = 'bundle';
    const TYPE_CONFIGURABLE = 'configurable';
    const TYPE_GROUPED      = 'grouped';
    const TYPE_VIRTUAL      = 'virtual';

    const DEFAULT_TYPE      = 'simple';
    const DEFAULT_TYPE_MODEL    = 'catalog/product_type_simple';
    const DEFAULT_PRICE_MODEL   = 'catalog/product_type_price';

    static protected $_types;
    static protected $_compositeTypes;
    static protected $_priceModels;
    static protected $_typesPriority;

    /**
     * Product type instance factory
     *
     * @param   Mage_Catalog_Model_Product $product
     * @param   bool $singleton
     * @return  Mage_Catalog_Model_Product_Type_Abstract
     */
    public static function factory($product, $singleton = false)
    {
        $types = self::getTypes();
        $typeId = $product->getTypeId();

        if (!empty($types[$typeId]['model'])) {
            $typeModelName = $types[$typeId]['model'];
        } else {
            $typeModelName = self::DEFAULT_TYPE_MODEL;
            $typeId = self::DEFAULT_TYPE;
        }

        if ($singleton === true) {
            $typeModel = Mage::getSingleton($typeModelName);
        }
        else {
            $typeModel = Mage::getModel($typeModelName);
            $typeModel->setProduct($product);
        }
        $typeModel->setConfig($types[$typeId]);
        return $typeModel;
    }

    /**
     * Product type price model factory
     *
     * @param   string $productType
     * @return  Mage_Catalog_Model_Product_Type_Price
     */
    public static function priceFactory($productType)
    {
        if (isset(self::$_priceModels[$productType])) {
            return self::$_priceModels[$productType];
        }

        $types = self::getTypes();

        if (!empty($types[$productType]['price_model'])) {
            $priceModelName = $types[$productType]['price_model'];
        } else {
            $priceModelName = self::DEFAULT_PRICE_MODEL;
        }

        self::$_priceModels[$productType] = Mage::getModel($priceModelName);
        return self::$_priceModels[$productType];
    }

    static public function getOptionArray()
    {
        $options = array();
        foreach(self::getTypes() as $typeId=>$type) {
            $options[$typeId] = Mage::helper('catalog')->__($type['label']);
        }

        return $options;
    }

    static public function getAllOption()
    {
        $options = self::getOptionArray();
        array_unshift($options, array('value'=>'', 'label'=>''));
        return $options;
    }

    static public function getAllOptions()
    {
        $res = array();
        $res[] = array('value'=>'', 'label'=>'');
        foreach (self::getOptionArray() as $index => $value) {
            $res[] = array(
               'value' => $index,
               'label' => $value
            );
        }
        return $res;
    }

    static public function getOptions()
    {
        $res = array();
        foreach (self::getOptionArray() as $index => $value) {
            $res[] = array(
               'value' => $index,
               'label' => $value
            );
        }
        return $res;
    }

    static public function getOptionText($optionId)
    {
        $options = self::getOptionArray();
        return isset($options[$optionId]) ? $options[$optionId] : null;
    }

    static public function getTypes()
    {
        if (is_null(self::$_types)) {
            $productTypes = Mage::getConfig()->getNode('global/catalog/product/type')->asArray();
            foreach ($productTypes as $productKey => $productConfig) {
                $moduleName = 'catalog';
                if (isset($productConfig['@']['module'])) {
                    $moduleName = $productConfig['@']['module'];
                }
                $translatedLabel = Mage::helper($moduleName)->__($productConfig['label']);
                $productTypes[$productKey]['label'] = $translatedLabel;
            }
            self::$_types = $productTypes;
        }

        return self::$_types;
    }

    /**
     * Return composite product type Ids
     *
     * @return array
     */
    static public function getCompositeTypes()
    {
        if (is_null(self::$_compositeTypes)) {
            self::$_compositeTypes = array();
            $types = self::getTypes();
            foreach ($types as $typeId=>$typeInfo) {
                if (array_key_exists('composite', $typeInfo) && $typeInfo['composite']) {
                    self::$_compositeTypes[] = $typeId;
                }
            }
        }
        return self::$_compositeTypes;
    }

    /**
     * Return product types by type indexing priority
     *
     * @return array
     */
    public static function getTypesByPriority()
    {
        if (is_null(self::$_typesPriority)) {
            self::$_typesPriority = array();
            $a = array();
            $b = array();

            $types = self::getTypes();
            foreach ($types as $typeId => $typeInfo) {
                $priority = isset($typeInfo['index_priority']) ? abs(intval($typeInfo['index_priority'])) : 0;
                if (!empty($typeInfo['composite'])) {
                    $b[$typeId] = $priority;
                } else {
                    $a[$typeId] = $priority;
                }
            }

            asort($a, SORT_NUMERIC);
            asort($b, SORT_NUMERIC);

            foreach (array_keys($a) as $typeId) {
                self::$_typesPriority[$typeId] = $types[$typeId];
            }
            foreach (array_keys($b) as $typeId) {
                self::$_typesPriority[$typeId] = $types[$typeId];
            }
        }
        return self::$_typesPriority;
    }
}

Descreva os tipos de produtos padrão (simples, configuráveis, agrupados) #

Simples

O tipo Produtos Simples deve ser usado para que geralmente tenha uma configuração única (tamanho único). Isso pode incluir itens como:

  • Uma caixa de giz de cera, pequena (24 cores)
  • Uma caixa de giz de cera, grande (64 cores)
  • Monitor de computador HD SuperHighTech de 26"
  • Figura de Ação Barrack Obama (6”)

Agrupado

Os produtos agrupados permitem que você crie um novo produto usando um ou mais produtos existentes em sua loja. Por exemplo, vamos supor que você já tenha um “Barrack Obama Action Figure” e um “George W Bush Action Figure” em sua loja e queira vendê-los como um pacote. Você simplesmente criaria um novo produto agrupado (vamos chamá-lo de “Obama + Bush (obtenha os dois e gaste o dobro!)”, e adicione as duas figuras de ação ao grupo por meio da guia “Produtos associados”.

Nota: Infelizmente, você não pode definir um preço especial de “grupo” diretamente na página do produto. Para oferecer um desconto para a compra de itens juntos, você precisará criar uma nova Regra de preço do carrinho de compras.

Configurável

Produto configurável : Este produto permite que seus clientes selecionem a variante que desejam escolhendo opções. Por exemplo, você pode vender camisetas em duas cores e três tamanhos. Você criaria seis produtos simples como produtos individuais (cada um com seus próprios SKUs) e, em seguida, adicionaria esses seis a um produto configurável onde os clientes podem escolher o tamanho e a cor e adicioná-los ao carrinho. Funcionalidade muito semelhante é possível usando Opções personalizadas para produtos Simples. A diferença entre um produto configurável e um produto que inclui opções personalizadas é que o inventário não é verificado ou atualizado para opções individuais durante a compra das opções personalizadas.

Virtual

Produtos Virtuais são aqueles que não possuem contraparte física ou digital. Eles não enviam, nem têm um link para download. Este tipo de produto pode ser usado para serviços como:

  • Limpeza de casa
  • Assinatura do boletim informativo de 1 ano

Observação: se estiver usando Produtos Virtuais para “assinaturas”, é importante observar que não há uma maneira integrada de gerenciar assinaturas de renovação automática. Todas as compras feitas no Magento, independentemente do Tipo de Produto, são compras únicas.

Pacote

Esse tipo de produto também é conhecido como “kit” em outros softwares de comércio eletrônico. Este tipo de produto é ideal para circunstâncias em que o usuário precisa selecionar várias opções configuráveis, mas pelo menos uma opção. Isso pode incluir produtos como:

  • Sistemas de computador personalizáveis
  • Smokings/ternos personalizáveis
  • Clique aqui para um tutorial em vídeo sobre o uso de pacotes

Baixável

Os produtos para download são semelhantes aos produtos virtuais, exceto que incluem a capacidade de adicionar um ou mais arquivos digitais para download. Os arquivos podem ser carregados por meio da interface Admin ou carregados diretamente para o servidor via FTP e, em seguida, adicionados por URL. Quando um cliente compra um produto para download, o Magento gerará um link seguro e criptografado (para que os clientes não possam ver a localização real do arquivo) para que o cliente baixe seu arquivo.

Esta categoria pode incluir produtos como:

  • Música/MP3
  • Softwares de Computador

Observação: Se você tiver o SSL habilitado para seu site, os downloads podem falhar em todas as versões do IE, pois o IE contém um bug que impede o download em conexões seguras se o cabeçalho sem cache estiver definido. Isso pode ser facilmente corrigido em um arquivo htaccess removendo os cabeçalhos no-cache e no-store, ou forçando os links de download a não serem seguros.
