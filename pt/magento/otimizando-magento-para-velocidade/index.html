Otimizando o Magento Alterando o arquivo .htaccess #

Magento é um aplicativo de comércio eletrônico muito popular. Ele oferece uma grande personalização e habilidades desde a instalação inicial. Aqui estão algumas sugestões para otimizar uma instalação do Magento.

Ativando a compactação de saída

Em seu arquivo .htaccess para Magento você encontrará uma seção de texto começando com a linha,

<IfModule mod_deflate.c> and ending at </IfModule>

Esta seção de código pode ser usada para ativar o módulo mod_deflate do Apache, que fornece compactação para texto, css e javascript. Você vai querer descomentar (remover o símbolo #) várias linhas para que fique assim:

###########################################

habilitar compressão de arquivos servidos pelo apache #

http://developer.yahoo.com/performance/rules.html#gzip #

# Insert filter on all content
SetOutputFilter DEFLATE
# Insert filter on selected content types only
AddOutputFilterByType DEFLATE text/html text/plain text/xml text/css text/javascript

# Netscape 4.x has some problems...
BrowserMatch ^Mozilla/4 gzip-only-text/html

# Netscape 4.06-4.08 have some more problems
BrowserMatch ^Mozilla/4\.0[678] no-gzip

# MSIE masquerades as Netscape, but it is fine
BrowserMatch \bMSIE !no-gzip !gzip-only-text/html

# Don't compress images
SetEnvIfNoCase Request_URI \.(?:gif|jpe?g|png)$ no-gzip dont-vary

# Make sure proxies don't deliver the wrong content
Header append Vary User-Agent env=!dont-vary
</IfModule>

Habilitando cabeçalhos de expiração #

Os visitantes de primeira vez em qualquer página da Web precisam fazer várias solicitações HTTP. Ao usar o cabeçalho “Expires”, você torna os componentes das solicitações em cache. Isso evita solicitações HTTP desnecessárias em visualizações de página subsequentes.

Você quer encontrar a área do arquivo .htaccess que começa com <IfModulemod_expires.c> e termina com o primeiro que você vê depois dele, e fazer com que fique assim:

<IfModule mod_expires.c>

############################################
## Add default Expires header
## http://developer.yahoo.com/performance/rules.html#expires
    ExpiresActive On
    ExpiresDefault "access plus 1 year"

</IfModule>

Configurações de administrador #

Mesclar arquivos JS e CSS

Esse ajuste específico reduzirá o número de solicitações HTTP em seu site de comércio eletrônico. [box type=”alert” border=”full”]Observação: às vezes, isso pode interromper alguns aplicativos. Depois de executar as etapas a seguir, verifique se o site ainda funciona como antes de ativar esse recurso.[/box]

  1. Faça login na sua área de administração e vá em – Sistema > Configuração > Desenvolvedor
  2. Em “Configurações JavaScript”, altere “Mesclar arquivos JavaScript” para sim.
  3. Em “Configurações CSS”, altere “Mesclar arquivos CSS” para sim.
  4. Finalmente você vai querer limpar o cache do Magento.

Ativar catálogos simples #

O modelo que Magento usa para armazenar dados de clientes e produtos resulta em consultas SQL mais longas que a média e mais leituras. Habilitar a opção Catálogo Simples para Categorias e Produtos mesclará os dados do produto em uma tabela, melhorando assim o desempenho.

Acesse sua área de administração e vá em – Sistema > Configuração > Catálogo Em “Frontend”, altere “Usar categoria de catálogo simples” para sim. Em “Frontend”, altere “Usar produto de catálogo simples” para sim - isso é opcional. Em seguida, você desejará limpar o cache do Magento. Finalmente, você precisará reindexar as tabelas. Ativar compilação

[box type=”alert” border=”full”]Observação: às vezes, isso pode interromper alguns aplicativos. Depois de executar as etapas a seguir, verifique se o site ainda funciona como antes de ativar esse recurso.[/box]

  1. Faça login na sua área de administração e vá em – Sistema > Ferramentas > Compilation

  2. Em seguida, basta clicar no botão Executar processo de compilação

  3. Após a execução da compilação, ela deve ser habilitada automaticamente

Ativar cache do sistema #

  1. Faça login na sua área de administração e vá em – Sistema > Cache Management
  2. Em seguida, clique no link Selecionar tudo
  3. Por fim, verifique se as Ações estão definidas como Ativar e clique em enviar

Desativar registro de erros

Faça login na sua área de administração e vá em – Sistema > Configuração > Desenvolvedor Na seção Configurações de log, certifique-se de que Ativado esteja definido como Não Dicas de manutenção do banco de dados

Existem várias tabelas usadas pelo Magento para registro. Embora o registro seja muito importante para saber o que está acontecendo e o que está acontecendo com sua loja, os registros podem se tornar grandes muito rapidamente, portanto, a manutenção regular pode ser de grande ajuda.

Aqui estão as tabelas para registro:

log_customer
log_visitor
log_visitor_info
log_url
log_url_info
log_quote
report_viewed_product_index
report_compared_product_index
report_event
catalog_compare_item
