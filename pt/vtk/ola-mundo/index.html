Exemplo de Olá Mundo #

#include <vtkAutoInit.h>

VTK_MODULE_INIT(vtkRenderingOpenGL2);
VTK_MODULE_INIT(vtkRenderingFreeType);
VTK_MODULE_INIT(vtkInteractionStyle);

#include <vtkSmartPointer.h>
#include <vtkTextActor.h>
#include <vtkRenderer.h>
#include <vtkRenderWindow.h>
#include <vtkRenderWindowInteractor.h>

int main(int /*argc*/, char ** /*argv*/)
{
    auto textActor = vtkSmartPointer<vtkTextActor>::New();
    textActor->SetInput("Hello World");

    auto renderer = vtkSmartPointer<vtkRenderer>::New();
    renderer->AddActor(textActor);
    renderer->ResetCamera();

    auto interactor = vtkSmartPointer<vtkRenderWindowInteractor>::New();
    
    auto renderWindow = vtkSmartPointer<vtkRenderWindow>::New();
    renderWindow->AddRenderer(renderer);
    renderWindow->SetInteractor(interactor);
    
    interactor->Start();

    return 0;
}

Discriminação: #

#include <vtkAutoInit.h>

VTK_MODULE_INIT(vtkRenderingOpenGL2);
VTK_MODULE_INIT(vtkRenderingFreeType);
VTK_MODULE_INIT(vtkInteractionStyle);

O design VTK usa um padrão de design método de fábrica para criar novas instâncias de [vtkObject](http://www.vtk.org/doc/nightly/html/ classvtkObject.html) usando o método <ClassName>::New(). Isso permite que uma implementação específica da plataforma seja selecionada durante o tempo de execução para satisfazer uma interface necessária.

Para que esse mecanismo funcione, as classes de fábrica precisam se “registrar” para que possam ser selecionadas pela infraestrutura vtk. Detalhes sobre este tópico estão disponíveis aqui.

VTK_MODULE_INIT é uma macro usada para inicializar automaticamente os módulos/biblioteca(s) necessários(vtkRenderingOpenGL2, vtkRenderingFreeType, vtkInteractionStyle neste exemplo). Falha ao inicializar os módulos resultará em chamadas <ClassName>::New() para retornar NULL e, portanto, erros de tempo de execução.

#include <vtkSmartPointer.h>
#include <vtkTextActor.h>
#include <vtkRenderer.h>
#include <vtkRenderWindow.h>
#include <vtkRenderWindowInteractor.h>

O papel vtkSmartPointer é semelhante ao de um std::unique_ptr, pois gerencia a contagem de referência que controla o tempo de vida de instâncias de classes derivadas de vtkObject.

vtkTextActor é uma classe simples que pode ser usada para exibir strings na tela.

vtkRenderer é uma classe responsável por gerenciar o conteúdo de uma cena. Especificamente, ele gerencia a coleta de

  • Atores 2D derivados de vtkActor2D
  • Atores 3D derivados de vtkProp3D
  • Volumes: vtkVolume
  • Câmera: vtkCamera
  • Luzes: vtkLight

vtkRenderWindow é uma classe que fornece interface independente de plataforma para

  • Gerenciando uma coleção de renderizadores.
  • manipular a entrada do usuário e encaminhá-la para vtkRenderWindowInteractor para processamento adicional

vtkRenderWindowInteractor é uma classe responsável por mapear os eventos de entrada do usuário (mouse/keyboard/timing) para uma ação correspondente. Internamente, ele usa um vtkInteractorStyle para fornecer diferentes comportamentos de mapeamento.

auto textActor = vtkSmartPointer<vtkTextActor>::New();
textActor->SetInput("Hello World");

Crie um ator de texto e defina a string a ser exibida

auto renderer = vtkSmartPointer<vtkRenderer>::New();
renderer->AddActor(textActor);
renderer->ResetCamera();
  • Crie um renderizador
  • Adicione o ator de texto a ele
  • Redefine a posição da câmera para garantir que o ator esteja visível na tela.
auto interactor = vtkSmartPointer<vtkRenderWindowInteractor>::New();

auto renderWindow = vtkSmartPointer<vtkRenderWindow>::New();
renderWindow->AddRenderer(renderer);
renderWindow->SetInteractor(interactor);

Crie uma janela para renderizar, adicione o renderizador a ela e defina o interator. A função de fábrica escolherá automaticamente uma implementação adequada com base nas classes de fábrica disponíveis/registradas

interactor->Start();

Esta é uma chamada de bloqueio que retorna apenas quando o usuário solicita uma saída (tecla q) ou fecha a janela. Executa um loop de mensagens e despacha as mensagens.
