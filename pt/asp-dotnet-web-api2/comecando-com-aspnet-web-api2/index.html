O que e por que Asp.Net Web API2? #

Oque e porque ?

A Web API2 do Asp.Net é a versão mais recente da Web API. É uma maneira fácil de implementar um serviço web RESTful usando todas as vantagens que o framework Asp.Net oferece. Uma vez que você entenda os princípios básicos do REST, então uma API da Web Asp.net2 será muito fácil de implementar. O Web API2 é construído no modelo de pipeline modular e conectável do Asp.Net. Isso significa que quando um servidor que hospeda uma API2 da Web recebe uma solicitação, ele passa primeiro pelo pipeline de solicitação do Asp.Nets. Isso permite que você adicione facilmente seus próprios módulos se achar que os recursos padrão não são suficientes para suas necessidades. Com os anúncios recentes no ASP.net vNext, isso também significa que você pode potencialmente hospedar sua API2 da Web fora do Windows Server, o que abre toda uma gama de casos de uso. Veja aqui para detalhes.

Como funciona?

Web API2 usa os conceitos de Controller e Action do MVC. Os recursos são mapeados diretamente para os controladores; você normalmente teria um controlador diferente para cada uma de suas principais entidades de dados (Produto, Pessoa, Pedido etc). Web API2 usa o mecanismo de roteamento Asp.Net para mapear URLs para controladores. Normalmente, as APIs são mantidas em uma rota /api/ que ajuda a distinguir os controladores de API de outros não API no mesmo site.

As ações são usadas para mapear para verbos HTTP específicos, por exemplo, você normalmente teria uma ação GET que retorna todas as entidades. Esta ação responderia a /api/Products (onde ‘products’ é seu controlador) e seria algo assim:

 public IEnumerable<string> Get()
 {
     return new string[] { "value1", "value2" };
 }

Você também pode ter uma ação GET que aceita um ID específico e retorna uma entidade específica. Ele responderia a /api/Products/81 e seria algo assim:

public string Get(int id)
 {
    return "value";
 }

Há muitos grandes benefícios ocultos em usar a API da Web que você pode não perceber, mas na verdade economiza muito trabalho.

Web API2 faz parte do ‘One Asp.Net’

Web API2 faz parte da família ‘One Asp.Net’, o que significa que ele suporta nativamente todos os ótimos recursos compartilhados que você pode usar atualmente com MVC ou formulários da web, isso inclui (estes são apenas alguns exemplos):

  • Estrutura de entidade
  • Autorização e identidade
  • Andaimes
  • Roteamento

Serialização e vinculação de modelo

Web API2 é configurada por padrão para fornecer respostas em XML ou JSON (JSON é o padrão). No entanto, como desenvolvedor, você não precisa fazer nenhuma conversão ou análise – você simplesmente retorna um objeto fortemente tipado e a Web API2 irá convertê-lo para XML ou JSON e devolvê-lo ao cliente chamador, este é um processo chamado de Negociação de Conteúdo. Este é um exemplo de uma ação GET que retorna um objeto Product fortemente tipado.

 public Product GetProduct(int id)
 {
    var product = _products.FirstOrDefault(p => p.ID == id);
    if (product == null)
    {
       throw new HttpResponseException(HttpStatusCode.NotFound);
    }
    return Request.CreateResponse(HttpStatusCode.OK, product);
 }

Isso também funciona para solicitações de entrada usando um recurso chamado Validação de Modelo. Com a validação de modelo, a API2 da Web é capaz de validar e analisar os dados do corpo da resposta de entrada para um objeto fortemente tipado para você trabalhar em seu código. Este é um exemplo de vinculação de modelo:

 public HttpResponseMessage Post(Product product)
 {
     if (ModelState.IsValid)
     {
         // Do something with the product (not shown).
 
         return new HttpResponseMessage(HttpStatusCode.OK);
     }
     else
     {
         return Request.CreateErrorResponse(HttpStatusCode.BadRequest, ModelState);
     }
 }

Olá Web API #

Web Api 2 - Exemplo Hello World

Vamos criar um novo aplicativo simples da Web Api que retornará para nós Json com mensagem e nome de usuário. Vamos começar! Primeiro, crie um novo projeto de API da Web usando o Visual Studio e selecione Modelo Vazio. Certifique-se de verificar a pasta “Web Api”:

NOTA Eu não escolhi o modelo “Web Api” porque ele adiciona referência ao ASP.NET MVC para fornecer a página de ajuda da API. E em uma aplicação tão básica, realmente não precisamos disso.

Adicionando um modelo

Um modelo é uma classe C# que representa alguns dados em nosso aplicativo. ASP.NET Web API é capaz de serializar automaticamente o modelo para JSON, XML ou algum outro formato (depende da configuração).

Em nosso aplicativo, criaremos apenas um modelo, mas os aplicativos do mundo real geralmente têm muitos deles.

No Solution Explorer, clique com o botão direito do mouse na pasta Modelos. Em seguida, selecione Adicionar e, em seguida, Classe. Nomeie a classe “HelloMessage”. Nosso modelo precisa de duas propriedades: MessageText e UserName:

namespace WebApiHelloWorld.Models
{
    public class HelloMessage
    {
        public string MessageText { get; set; }
        public string UserName { get; set; }
    }
}

Adicionando um controlador

Um controlador lida com solicitações HTTP. Nosso aplicativo precisa apenas de um controlador que retorne Json com mensagem Hello e nome de usuário (que passaremos na URL). No Solution Explorer, clique com o botão direito do mouse na pasta Controllers. Em seguida, selecione Add e selecione Controller. Na janela aberta, selecione Web API Controller - Empty e clique em Add.

Defina o nome do controlador como “HelloController”. Próximo código de edição do controlador criado. Precisamos adicionar o método que retorna a mensagem Hello.

using System.Web.Http;
using WebApiHelloWorld.Models;

namespace WebApiHelloWorld.Controllers
{
    public class HelloController : ApiController
    {
        public HelloMessage GetMessage(string name)
        {
            HelloMessage message = new HelloMessage
            {
                MessageText = "Hello my Dear!",
                UserName = name
            };

            return message;
        }
    }
}

NOTA Certifique-se de adicionar using WebApiHelloWorld.Models. Sem ele, seu controlador não encontrará a classe HelloMessage.

Terminar

Isso é tudo! Agora você só precisa construir e iniciar seu aplicativo. Basta pressionar Ctrl + F5 ou apenas F5 (para iniciar sem depurar). O Visual Studio iniciará o navegador da Web. Você precisa ligar para o seu controlador. Para fazer isso, adicione no final da URL “/api/hello?name=John”. O resultado deve ser:

{
    "MessageText": "Hello my Dear!",
    "UserName": "John"
}
