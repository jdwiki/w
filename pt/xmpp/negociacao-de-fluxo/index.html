As conexões XMPP compreendem dois fluxos XML: um para entrada e outro para saída. Esses fluxos geralmente são enviados pela mesma conexão TCP (embora às vezes várias conexões possam ser usadas, especialmente para conexões servidor a servidor) e compartilham certos recursos para os quais a negociação é necessária (por exemplo, autenticação com SASL).

Fechando um fluxo #

Um fluxo é fechado enviando uma tag de fechamento </stream>. Depois que a tag de fluxo de fechamento é enviada, nenhum dado deve ser enviado no fluxo (mesmo em resposta aos dados recebidos da outra parte). Antes de fechar a conexão, a entidade remetente deve esperar por uma tag </stream> de resposta para dar à outra parte tempo para enviar quaisquer dados pendentes e deve expirar (e encerrar a(s) conexão(ões) TCP subjacentes) se um fluxo de fechamento tag não é recebida dentro de um período de tempo escolhido.

</stream:stream>

Se o fluxo for criptografado com TLS, as partes devem encerrar o TLS de forma limpa enviando um alerta TLS close_notify e recebendo um em resposta. Sua biblioteca TLS provavelmente faz isso para você.

Iniciando um fluxo #

Uma vez que uma conexão TCP é estabelecida, o cabeçalho do fluxo inicial é enviado pela entidade iniciadora. Da mesma forma, sempre que uma reinicialização de fluxo for necessária (por exemplo, após negociar uma camada de segurança como TLS), um cabeçalho de fluxo também deve ser enviado:

<?xml version='1.0'?>
<stream:stream
    from='juliet@im.example.com'
    to='im.example.com'
    version='1.0'
    xml:lang='en'
    xmlns='jabber:client'
    xmlns:stream='http://etherx.jabber.org/streams'>

O cabeçalho XML é opcional, mas se existir, não deve especificar nada além do XML versão 1.0 com codificação UTF-8.

Em resposta, a entidade receptora enviará sua própria tag de stream de abertura contendo um ID de sessão exclusivo:

<?xml version='1.0'?>
<stream:stream
    from='im.example.com'
    id='++TR84Sm6A3hnt3Q065SnAbbk3Y='
    to='juliet@im.example.com'
    version='1.0'
    xml:lang='en'
    xmlns='jabber:client'
    xmlns:stream='http://etherx.jabber.org/streams'>

Feche a conexão XMPP usando a biblioteca agsxmpp #

  public class ConnectionManager
    { 
         private XmppClientConnection _xmppClientConnection = null;

         public ConnectionManager()
                {
                    if (_xmppClientConnection == null)
                    {
                        _xmppClientConnection = new XmppClientConnection();
                    }
                }
         public void CloseXmppConnection()
                {
                    try
                    {
                        if (_xmppClientConnection != null)
                        {
                                //Close xmpp Client Connection
                             _xmppClientConnection.Close();
                        }
                        
                      
                    }
                    catch (Exception ex)
                    {
                    }
                }
    }
