Sintaxe #

  • [ localpart “@” ] domainpart [ “/” resourcepart ]

Parâmetros #

Parte Uso comum
Localparte Identifica uma entidade XMPP (opcional)
Parte do domínio Identifica o serviço XMPP
Resourcepart Identifica uma sessão de uma entidade XMPP (opcional)

Os endereços XMPP, mais comumente conhecidos como JIDs (Jabber Identifiers) são definidos em RFC 7622 e atuam como endereços na rede XMPP. Eles se parecem com um endereço de e-mail, mas às vezes têm um “resourcepart” opcional no final que identifica um determinado cliente conectado como a conta representada pelo restante do endereço (já que o XMPP pode ter vários clientes conectados por conta). Um exemplo de um endereço XMPP com o resourcepart (um cliente) xyz é:

romeo@example.net/xyz

Tipos de JID #

Um JID consiste em três partes: localpart@domainpart/resourcepart.

JIDs completos (sempre tem uma parte de recurso)

romeo@example.org/orchard
example.org/da863ab

JIDs simples (sempre sem parte do recurso)

 romeo@example.org
 example.org

Validando um JID (genérico) #

Ao contrário dos emails, os JIDs foram definidos com a Internacionalização (i18n) em mente usando o Preparação, Aplicação e Comparação da Estrutura de Strings Internacionalizadas (PRECIS). PRECIS (definido em RFC 7564), é uma estrutura para comparar strings com segurança em vários contextos. Por exemplo, imagine que você registrou o apelido “Richard IV” (letras maiúsculas latinas I, Vee) em um bate-papo em grupo: Usando PRECIS, o aplicativo de bate-papo pode garantir que ninguém mais apareça e registre o apelido “Richard Ⅳ” (Unicode Roman Numeral 4) e o usa para se passar por você.

A etapa Validations deve realizar o seguinte:

  • Verifique se o localpart tem menos de 1024 bytes (bytes, não glifos)
  • Verifique se a parte local não contém nenhum de "&'/:<>@
  • Verifique se o resourcepart tem menos de 1024 bytes
  • Verifique se a parte do domínio é maior que zero bytes e menor que 1024 bytes (e possivelmente valide se as partes individuais do domínio se encaixam nos requisitos de DNS)
  • Se o domínio for um endereço IPv6 válido, certifique-se de que ele usa notação entre colchetes (por exemplo, [::1] em vez de ::1)

Dividindo um JID (Go) #

O pacote mellium.im/xmpp/jid implementa operações em JIDs. Para dividir uma string JID em suas partes componentes, a função SplitString pode ser usada:

lp, dp, rp, err := SplitString("romeo@example.net")

Nenhuma validação é realizada pela função e as peças não têm garantia de validade.

Para dividir manualmente uma string sem depender do pacote jid, o código subjacente se parece com isso:

// SplitString splits out the localpart, domainpart, and resourcepart from a
// string representation of a JID. The parts are not guaranteed to be valid, and
// each part must be 1023 bytes or less.
func SplitString(s string) (localpart, domainpart, resourcepart string, err error) {

    // RFC 7622 §3.1.  Fundamentals:
    //
    //    Implementation Note: When dividing a JID into its component parts,
    //    an implementation needs to match the separator characters '@' and
    //    '/' before applying any transformation algorithms, which might
    //    decompose certain Unicode code points to the separator characters.
    //
    // so let's do that now. First we'll parse the domainpart using the rules
    // defined in §3.2:
    //
    //    The domainpart of a JID is the portion that remains once the
    //    following parsing steps are taken:
    //
    //    1.  Remove any portion from the first '/' character to the end of the
    //        string (if there is a '/' character present).
    sep := strings.Index(s, "/")

    if sep == -1 {
        sep = len(s)
        resourcepart = ""
    } else {
        // If the resource part exists, make sure it isn't empty.
        if sep == len(s)-1 {
            err = errors.New("The resourcepart must be larger than 0 bytes")
            return
        }
        resourcepart = s[sep+1:]
        s = s[:sep]
    }

    //    2.  Remove any portion from the beginning of the string to the first
    //        '@' character (if there is an '@' character present).

    sep = strings.Index(s, "@")

    switch sep {
    case -1:
        // There is no @ sign, and therefore no localpart.
        localpart = ""
        domainpart = s
    case 0:
        // The JID starts with an @ sign (invalid empty localpart)
        err = errors.New("The localpart must be larger than 0 bytes")
        return
    default:
        domainpart = s[sep+1:]
        localpart = s[:sep]
    }

    // We'll throw out any trailing dots on domainparts, since they're ignored:
    //
    //    If the domainpart includes a final character considered to be a label
    //    separator (dot) by [RFC1034], this character MUST be stripped from
    //    the domainpart before the JID of which it is a part is used for the
    //    purpose of routing an XML stanza, comparing against another JID, or
    //    constructing an XMPP URI or IRI [RFC5122].  In particular, such a
    //    character MUST be stripped before any other canonicalization steps
    //    are taken.

    domainpart = strings.TrimSuffix(domainpart, ".")

    return
}

Dividindo um JID (Rust) #

Em Rust, a caixa xmpp-addr (docs) pode ser usada para manipular JIDs. Para dividir um JID em suas partes componentes (sem validar que essas partes são válidas), a função Jid::split pode ser usada:

let (lp, dp, rp) = Jid::split("feste@example.net")?;
assert_eq!(lp, Some("feste"));
assert_eq!(dp, "example.net");
assert_eq!(rp, None);
