Como obter um token OAuth usando o fluxo de código de autorização

  • Troque o código pelo token OAuth

Envie o usuário para o endpoint de autorização para obter o código de autorização #

Você primeiro enviará o usuário para o endpoint de autorização do Twitch. Essa URL é composta por uma URL de autorização básica (https://api.twitch.tv/kraken/oauth2/authorize) e parâmetros de string de consulta que definem o que você está solicitando. Os parâmetros necessários são response_type, client_id, redirect_uri e scope.

Para o fluxo de código de autorização, o parâmetro response_type é sempre definido como code. Isso significa que você está solicitando um código de autorização da API do Twitch.

O redirect_uri é para onde o usuário será redirecionado após aprovar os escopos que sua aplicação solicitou. Isso deve corresponder ao que você registrou na sua conta do Twitch página de conexões.

O client_id é um identificador exclusivo para seu aplicativo. Você também pode encontrar seu ID de cliente na página Conexões.

O scope define o que você tem acesso em nome do usuário. Você deve solicitar apenas o mínimo necessário para que seu aplicativo funcione. Você pode encontrar a lista de escopos no Twitch API GitHub.

O parâmetro state também é suportado para ajudar a proteger contra ataques de script entre sites. O parâmetro state será incluído no redirect_uri quando o usuário autorizar sua aplicação.

  https://api.twitch.tv/kraken/oauth2/authorize
    ?response_type=code
    &client_id=[your client ID]
    &redirect_uri=[your registered redirect URI]
    &scope=[space separated list of scopes]
    &state=[your provided unique token]

Obtenha o código de autorização da string de consulta #

Quando o usuário for para o endpoint de autorização, ele será solicitado a conceder permissão ao seu aplicativo para os escopos que você solicitou. Eles podem recusar isso, então você deve levar isso em consideração em seu código. Depois que eles permitirem o acesso ao seu aplicativo, o usuário será redirecionado para a URL que você especificou em redirect_uri. A string de consulta agora terá um parâmetro code, que é o código de autorização que você pode trocar por um token OAuth.

<?php
  $authCode = $_GET['code'];
?>

Troque o código pelo token OAuth #

Agora que você tem um código de autorização, você pode fazer um POST no terminal do token (https://api.twitch.tv/kraken/oauth2/token) para obter um token OAuth. Você receberá um token de acesso codificado em JSON, um token de atualização e uma lista dos escopos aprovados pelo usuário. Agora você pode usar esse token para fazer solicitações autenticadas em nome do usuário.

<?php
  $authCode = $_GET['code'];

  $parameterValues = array(
    'client_id' => '...',
    'client_secret' => '...',
    'grant_type' => 'authorization_code',
    'redirect_uri' => 'http://localhost/',
    'code' => $authCode
  );

  $postValues = http_build_query($parameterValues, '', '&');

  $ch = curl_init();
    
  curl_setopt_array($ch, array(
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_URL => 'https://api.twitch.tv/kraken/oauth2/token',
    CURLOPT_POST => 1,
    CURLOPT_POSTFIELDS => $postValues
  ));
            
  $response = curl_exec($ch);
  curl_close($ch);

  echo $response;
?>
