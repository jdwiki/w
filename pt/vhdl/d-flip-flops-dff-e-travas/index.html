D-Flip-Flops (DFF) e travas são elementos de memória. Um DFF amostra sua entrada em uma ou outra borda de seu clock (não em ambas) enquanto uma trava é transparente em um nível de sua habilitação e memorizada no outro. A figura a seguir ilustra a diferença:

A modelagem de DFFs ou travas em VHDL é fácil, mas existem alguns aspectos importantes que devem ser levados em consideração:

  • As diferenças entre os modelos VHDL de DFFs e travas.
  • Como descrever as bordas de um sinal.
  • Como descrever o set ou reset síncrono ou assíncrono.

D-Flip-Flops (DFF) #

Em todos os exemplos:

  • clk é o relógio,
  • d é a entrada,
  • q é a saída,
  • srst é um reset ativo síncrono alto,
  • srstn é um reset síncrono baixo ativo,
  • arst é um reset ativo assíncrono alto,
  • arstn é um reset ativo assíncrono baixo,
  • sset é um conjunto ativo síncrono alto,
  • ssetn é um conjunto síncrono baixo ativo,
  • aset é um conjunto ativo alto assíncrono,
  • assetn é um conjunto ativo assíncrono baixo

Todos os sinais são do tipo ieee.std_logic_1164.std_ulogic. A sintaxe utilizada é aquela que leva a resultados de síntese corretos com todos os sintetizadores lógicos. Por favor, veja o exemplo Clock edge detect para uma discussão sobre sintaxe alternativa.

Relógio de borda ascendente #

process(clk)
begin
  if rising_edge(clk) then
    q <= d;
  end if;
end process;

Relógio de borda descendente #

process(clk)
begin
  if falling_edge(clk) then
    q <= d;
  end if;
end process;

Relógio de borda ascendente, reset ativo síncrono alto #

process(clk)
begin
  if rising_edge(clk) then
    if srst = '1' then
      q <= '0';
    else
      q <= d;
    end if;
  end if;
end process;

Relógio de borda ascendente, reset alto ativo assíncrono #

process(clk, arst)
begin
  if arst = '1' then
    q <= '0';
  elsif rising_edge(clk) then
    q <= d;
  end if;
end process;

Relógio de borda descendente, reset baixo ativo assíncrono, ajuste alto ativo síncrono #

process(clk, arstn)
begin
  if arstn = '0' then
    q <= '0';
  elsif falling_edge(clk) then
    if sset = '1' then
      q <= '1';
    else
      q <= d;
    end if;
  end if;
end process;

Relógio de borda ascendente, reset alto ativo assíncrono, ajuste baixo ativo assíncrono #

Nota: set tem prioridade maior que reset

process(clk, arst, asetn)
begin
  if asetn = '0' then
    q <= '1';
  elsif arst = '1' then
    q <= '0';
  elsif rising_edge(clk) then
    q <= d;
  end if;
end process;

Travas #

Em todos os exemplos:

  • en é o sinal de habilitação,
  • d é a entrada,
  • q é a saída,
  • srst é um reset ativo síncrono alto,
  • srstn é um reset síncrono baixo ativo,
  • arst é um reset ativo assíncrono alto,
  • arstn é um reset ativo assíncrono baixo,
  • sset é um conjunto ativo síncrono alto,
  • ssetn é um conjunto síncrono baixo ativo,
  • aset é um conjunto ativo alto assíncrono,
  • assetn é um conjunto ativo assíncrono baixo

Todos os sinais são do tipo ieee.std_logic_1164.std_ulogic. A sintaxe utilizada é aquela que leva a resultados de síntese corretos com todos os sintetizadores lógicos. Por favor, veja o exemplo Clock edge detect para uma discussão sobre sintaxe alternativa.

Ativação alta ativa #

process(en, d)
begin
  if en = '1' then
    q <= d;
  end if;
end process;

Ativação baixa ativa #

process(en, d)
begin
  if en = '0' then
    q <= d;
  end if;
end process;

Ativação alta ativa, reinicialização alta ativa síncrona #

process(en, d)
begin
  if en = '1' then
    if srst = '1' then
      q <= '0';
    else
      q <= d;
    end if;
  end if;
end process;

Habilitação alta ativa, reset alto ativo assíncrono #

process(en, d, arst)
begin
  if arst = '1' then
    q <= '0';
  elsif en = '1' then
    q <= d;
  end if;
end process;

Ativação baixa ativa, reset baixo ativo assíncrono, ajuste alto ativo síncrono #

process(en, d, arstn)
begin
  if arstn = '0' then
    q <= '0';
  elsif en = '0' then
    if sset = '1' then
      q <= '1';
    else
      q <= d;
    end if;
  end if;
end process;

Ativação alta ativa, reset alto ativo assíncrono, ajuste baixo ativo assíncrono #

Nota: set tem prioridade maior que reset

process(en, d, arst, asetn)
begin
  if asetn = '0' then
    q <= '1';
  elsif arst = '1' then
    q <= '0';
  elsif en = '1' then
    q <= d;
  end if;
end process;

Detecção de borda de clock #

O conto #

Com VHDL 2008 e se o tipo do relógio for bit, boolean, ieee.std_logic_1164.std_ulogic ou ieee.std_logic_1164.std_logic, uma detecção de borda de clock pode ser codificada para borda de subida

  • se crescente_edge(relógio) então
  • if clock'event e clock = '1' então -- digite bit, std_ulogic ou std_logic
  • if clock'event e clock then -- digite boolean

e para borda de queda

  • se Falling_edge(clock) then
  • if clock'event e clock = '0' então -- digite bit, std_ulogic ou std_logic
  • if clock'event e não clock then -- digite boolean

Isso se comportará como esperado, tanto para simulação quanto para síntese.

Nota: a definição de uma borda de subida em um sinal do tipo std_ulogic é um pouco mais complexa do que o simples if clock'event and clock = '1' then. A função padrão rising_edge, por exemplo, tem uma definição diferente. Mesmo que provavelmente não faça diferença para a síntese, poderia fazer uma para a simulação.

O uso das funções padrão rising_edge e falling_edge é fortemente encorajado. Com versões anteriores do VHDL, o uso dessas funções pode exigir declarar explicitamente o uso de pacotes padrão (por exemplo, ieee.numeric_bit para o tipo bit) ou até mesmo defini-los em um pacote personalizado.

Nota: não use as funções padrão rising_edge e falling_edge para detectar bordas de sinais que não sejam de clock. Alguns sintetizadores podem concluir que o sinal é um relógio. Dica: detectar uma borda em um sinal que não seja de clock pode ser feito com frequência por amostragem do sinal em um registrador de deslocamento e comparando os valores amostrados em diferentes estágios do registrador de deslocamento.

A longa história #

Descrever adequadamente a detecção das bordas de um sinal de clock é essencial ao modelar D-Flip-Flops (DFF). Uma aresta é, por definição, uma transição de um valor particular para outro. Por exemplo, podemos definir a borda de subida de um sinal do tipo bit (o tipo padrão VHDL enumerado que recebe dois valores: '0' e '1') como a transição de '0' para '1'. Para o tipo boolean podemos defini-lo como uma transição de false para true.

Freqüentemente, tipos mais complexos são usados. O tipo ieee.std_logic_1164.std_ulogic, por exemplo, também é um tipo enumerado, assim como bit ou boolean, mas tem 9 valores em vez de 2:

Valor Significado
'U' Não inicializado
'X' Forçando desconhecido
'0' Forçando baixo nível
'1' Forçando alto nível
'Z' Alta impedância
'W' Fraco desconhecido
'L' Baixo nível fraco
'H' Alto nível fraco
'-' Não me importo

Definir uma borda ascendente em tal tipo é um pouco mais complexo do que para bit ou boolean. Podemos, por exemplo, decidir que é uma transição de '0' para '1'. Mas também podemos decidir que é uma transição de '0' ou 'L' para '1' ou 'H'.

Nota: é esta segunda definição que o padrão usa para a função rising_edge(signal s: std_ulogic) definida em ieee.std_logic_1164.

Ao discutir as várias maneiras de detectar bordas, é importante considerar o tipo de sinal. Também é importante levar em consideração o objetivo da modelagem: apenas simulação ou síntese lógica? Vamos ilustrar isso com alguns exemplos:

DFF de borda ascendente com bit de tipo #

signal clock, d, q: bit;
...
P1: process(clock)
begin
  if clock = '1' then
    q <= d;
  end if;
end process P1;

Tecnicamente, em um ponto de vista semântico de simulação pura, o processo P1 modela um DFF acionado por borda de subida. De fato, a atribuição q <= d é executada se e somente se:

  • clock alterado (isto é o que a lista de sensibilidade expressa) and
  • o valor atual de clock é '1'.

Como clock é do tipo bit e tipo bit possui apenas valores '0' e '1', isso é exatamente o que definimos como uma borda ascendente de um sinal do tipo bit. Qualquer simulador irá lidar com este modelo como esperamos.

Nota: Para sintetizadores lógicos, as coisas são um pouco mais complexas, como veremos mais adiante.

DFF de borda ascendente com reset alto ativo assíncrono e bit de tipo #

Para adicionar um alto reset ativo assíncrono ao nosso DFF, pode-se tentar algo como:

signal clock, reset, d, q: bit;
...
P2_BOGUS: process(clock, reset)
begin
  if reset = '1' then
    q <= '0';
  elsif clock = '1' then
    q <= d;
  end if;
end process P2_BOGUS;

Mas isso não funciona. A condição para que a atribuição q <= d seja executada deve ser: a borda ascendente de clock enquanto reset = '0'. Mas o que modelamos é:

  • clock ou reset ou ambos alterados and
  • redefinir = '0' e
  • relógio = '1'

O que não é o mesmo: se reset mudar de '1' para '0' enquanto clock = '1' a atribuição será executada enquanto não for uma borda ascendente de clock.

Na verdade, não há como modelar isso em VHDL sem a ajuda de um atributo de sinal:

P2_OK: process(clock, reset)
begin
  if reset = '1' then
    q <= '0';
  elsif clock = '1' and clock'event then
    q <= d;
  end if;
end process P2_OK;

O clock'event é o atributo de sinal event aplicado ao sinal clock. Ele é avaliado como um boolean e é true se e somente se o sinal clock mudou durante a fase de atualização do sinal que precedeu a fase de execução atual. Graças a isso, o processo P2_OK agora modela perfeitamente o que queremos em simulação (e síntese).

Semântica de síntese #

Muitos sintetizadores lógicos identificam detecções de borda de sinal com base em padrões sintáticos, não na semântica do modelo VHDL. Em outras palavras, eles consideram a aparência do código VHDL, não o comportamento que ele modela. Um dos padrões que todos reconhecem é:

if clock = '1' and clock'event then

Então, mesmo no exemplo do processo P1 devemos usá-lo se quisermos que nosso modelo seja sintetizado por todos os sintetizadores lógicos:

signal clock, d, q: bit;
...
P1_OK: process(clock)
begin
  if clock = '1' and clock'event then
    q <= d;
  end if;
end process P1_OK;

A parte and clock'event da condição é completamente redundante com a lista de sensibilidade, mas como alguns sintetizadores precisam dela…

DFF de borda ascendente com reset alto ativo assíncrono e digite std_ulogic #

Neste caso, expressar a borda de subida do clock e a condição de reset pode se tornar complicado. Se mantivermos a definição de borda de subida que propusemos acima e se considerarmos que o reset está ativo se for '1' ou 'H', o modelo se torna:

library ieee;
use ieee.std_logic_1164.all;
...
signal clock, reset, d, q: std_ulogic;
...
P4: process(clock, reset)
begin
  if reset = '1' or reset = 'H' then
    q <= '0';
  elsif clock'event and
        (clock'last_value = '0' or clock'last_value = 'L') and
        (clock = '1' or clock = 'H') then
    q <= d;
  end if;
end process P4;

Nota: 'last_value é outro atributo de sinal que retorna o valor que o sinal tinha antes da última mudança de valor.

Funções auxiliares #

O padrão VHDL 2008 oferece várias funções auxiliares para simplificar a detecção de bordas de sinal, especialmente com tipos enumerados multivalorados como std_ulogic. O pacote std.standard define as funções rising_edge e falling_edge nos tipos bit e boolean e o pacote ieee.std_logic_1164 as define nos tipos std_ulogic e std_logic.

Nota: com versões anteriores do VHDL o uso dessas funções pode exigir declarar explicitamente o uso de pacotes padrão (por exemplo, ieee.numeric_bit para tipo bit) ou mesmo defini-los em um pacote de usuário.

Vamos revisitar os exemplos anteriores e usar as funções auxiliares:

signal clock, d, q: bit;
...
P1_OK_NEW: process(clock)
begin
  if rising_edge(clock) then
    q <= d;
  end if;
end process P1_OK_NEW;

signal clock, d, q: bit;
...
P2_OK_NEW: process(clock, reset)
begin
  if reset = '1' then
    q <= '0';
  elsif rising_edge(clock) then
    q <= d;
  end if;
end process P2_OK_NEW;

library ieee;
use ieee.std_logic_1164.all;
...
signal clock, reset, d, q: std_ulogic;
...
P4_NEW: process(clock, reset)
begin
  if reset = '1' then
    q <= '0';
  elsif rising_edge(clock) then
    q <= d;
  end if;
end process P4_NEW;

Nota: neste último exemplo também simplificamos o teste no reset. Os resets flutuantes, de alta impedância, são bastante raros e, na maioria dos casos, esta versão simplificada funciona para simulação e síntese.
