FS - Separador de Campo #

A variável FS é usada para definir o separador de campo de entrada. Em awk, espaço e tab atuam como separadores de campo padrão. O valor do campo correspondente pode ser acessado através de $1, $2, $3… e assim por diante.

awk -F'=' '{print $1}' file

  • -F - opção de linha de comando para definir o separador de campo de entrada.

    awk ‘BEGIN { FS="=" } { print $1 }’ file

OFS - Separador de Campo de Saída #

Esta variável é usada para definir o separador de campo de saída que é um espaço por padrão.

awk -F'=' 'BEGIN { OFS=":" } { print $1 }' file

Exemplo:

$ cat file.csv 
col1,col2,col3,col4
col1,col2,col3
col1,col2
col1
col1,col2,col3,col4,col5

$ awk -F',' 'BEGIN { OFS="|" } { $1=$1 } 1' file.csv
col1|col2|col3|col4
col1|col2|col3
col1|col2
col1
col1|col2|col3|col4|col5

Atribuir $1 a $1 em $1=$1 modifica um campo ($1 neste caso) e isso resulta em awk reconstruindo o registro $0. A reconstrução do registro substitui os delimitadores FS por OFS.

FNR - O número do registro atual sendo processado #

FNR contém o número da linha do arquivo de entrada que está sendo processada. Neste exemplo, você verá o awk iniciando em 1 novamente ao iniciar o processamento do segundo arquivo.

Exemplo com um arquivo

$ cat file1
AAAA
BBBB
CCCC
$ awk '{ print FNR }' file1
1
2
3

Exemplo com dois arquivos

$ cat file1
AAAA
BBBB
CCCC
$ cat file2
WWWW
XXXX
YYYY
ZZZZ
$ awk '{ print FNR, FILENAME, $0 }' file1 file2
1 file1 AAAA
2 file1 BBBB
3 file1 CCCC
1 file2 WWWW
2 file2 XXXX
3 file2 YYYY
4 file2 ZZZZ

Exemplo estendido com dois arquivos

FNR pode ser usado para detectar se o awk está processando o primeiro arquivo já que NR==FNR é verdadeiro apenas para o primeiro arquivo. Por exemplo, se quisermos juntar registros dos arquivos file1 e file2 em seus FNR:

$ awk 'NR==FNR { a[FNR]=$0; next } (FNR in a) { print FNR, a[FNR], $1 }' file1 file2
1 AAAA WWWW
2 BBBB XXXX
3 CCCC YYYY

O registro ZZZZ de file2 está ausente, pois FNR tem valor máximo diferente para file1 e file2 e não há junção para diferentes FNRs.

FS - Separador de Campo #

Usado pelo awk para dividir cada registro em vários campos:

echo "a-b-c
d-e-f" | awk 'BEGIN {FS="-"} {print $2}'

vai resultar em:

b
e

A variável FS também pode ser definida usando a opção -F:

echo "a-b-c
d-e-f" | awk -F '-' '{print $2}'

Por padrão, os campos são separados por espaços em branco (espaços e tabulações) e vários espaços e tabulações contam como um único separador.

RS - Separador de Registro #

Usado pelo awk para dividir a entrada em vários registros. Por exemplo:

echo "a b c|d e f" | awk 'BEGIN {RS="|"} {print $0}'

produz:

a b c
d e f

Por padrão, o separador de registro é o caractere de nova linha.

De forma similar: echo “a b c|d e f” | awk ‘BEGIN {RS="|"} {print $2}’

produz:

b
e

OFS - Separador de Campo de Saída #

Usado pelo awk para separar a saída de campos pela instrução print. Por exemplo:

echo "a b c
d e f" | awk 'BEGIN {OFS="-"} {print $2, $3}'

produz:

b-c
e-f

O valor padrão é , uma string que consiste em um único espaço.

ORS - Separador de Registro de Saída #

Usado pelo awk para separar registros e é gerado no final de cada comando print. Por exemplo:

echo "a b c
d e f" | awk 'BEGIN {ORS="|"} {print $2, $3}'

produz:

b c|e f

O valor padrão é \n (caractere de nova linha).

ARGV, ARGC - Array of Command Line Arguments #

Os argumentos de linha de comando passados ​​para o awk são armazenados no array interno ARGV dos elementos ARGC. O primeiro elemento do array é o nome do programa. Por exemplo:

awk 'BEGIN {
   for (i = 0; i < ARGC; ++i) {
      printf "ARGV[%d]=\"%s\"\n", i, ARGV[i]
   }
}' arg1 arg2 arg3

produz:

ARGV[0]="awk"
ARGV[1]="arg1"
ARGV[2]="arg2"
ARGV[3]="arg3"

RS - Separador de Registro de Entrada #

NF - Número de Campos #

NR - Número total de registros #

Fornecerá o número total de registros processados ​​na instância awk atual.

cat > file1
suicidesquad
harley quinn
joker
deadshot

cat > file2
avengers
ironman
captainamerica
hulk

awk '{print NR}' file1 file2
1
2
3
4
5
6
7
8

Um total de 8 registros foram processados ​​na instância.

FNR - Número de registros no arquivo #

Fornece o número total de registros processados ​​pela instância awk em relação aos arquivos que o awk está processando

cat > file1
suicidesquad
harley quinn
joker
deadshot

cat > file2
avengers
ironman
captainamerica
hulk

awk '{print FNR}' file1 file2
1
2
3
4
1
2
3
4

Cada arquivo tinha 4 linhas cada, então sempre que awk encontrava um EOF FNR era redefinido para 0.

NF - Número de Campos #

Fornece o número de colunas ou campos em cada registro (o registro corresponde a cada linha). Cada linha é demarcada por RS, que é o padrão para nova linha.

cat > file1
Harley Quinn Loves Joker
Batman Loves Wonder Woman
Superman is not dead
Why is everything I type four fielded!?

awk '{print NF}' file1
4
4
4
7

FS (em algum lugar lá em cima) padroniza para tab ou espaço. Então Harley, Quinn, Loves, Joker são considerados colunas. O caso vale para as próximas duas linhas, mas a última linha tem 7 palavras separadas por espaço, o que significa 7 colunas.
