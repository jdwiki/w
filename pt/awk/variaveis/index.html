Atribuição de variável de linha de comando #

Para atribuir variáveis ​​a partir da linha de comando, -v pode ser usado:

$ awk -v myvar="hello" 'BEGIN {print myvar}'
hello

Observe que não há espaços ao redor do sinal de igual.

Isso permite usar variáveis ​​de shell:

$ shell_var="hello"
$ awk -v myvar="$shell_var" 'BEGIN {print myvar}'
hello

Além disso, isso permite definir variáveis ​​internas que controlam o awk:

Veja um exemplo com FS (separador de campo):

$ cat file
1,2;3,4
$ awk -v FS="," '{print $2}' file
2;3
$ awk -v FS=";" '{print $2}' file
3,4

Ou com OFS (separador de campo de saída):

$ echo "2 3" | awk -v OFS="--" '{print $1, $2}'
2--3
$ echo "2 3" | awk -v OFS="+" '{print $1, $2}'
2+3

Passando parâmetros para um programa usando a opção -v #

A opção -v seguida por uma atribuição da forma variable=value pode ser usada para passar parâmetros para um programa awk. Isso é ilustrado pelo programa punição abaixo, cujo trabalho é escrever contar vezes a frase “Não vou falar na aula”. na saída padrão. O exemplo a seguir usa o valor 100, que é muito popular entre os professores:

awk -v count=100 'BEGIN {
  for(i = 1; i <= count; ++i) {
    print("I shall not talk in class.")
  }
  exit
}'

É possível passar vários parâmetros com o uso repetido do sinalizador -v:

awk -v count=100 -v "sentence=I shall not talk in class." 'BEGIN {
  for(i = 1; i <= count; ++i) {
    print(sentence)
  }
  exit
}'

Não há suporte interno para parâmetros de matriz ou lista, eles devem ser tratados manualmente. Uma abordagem clássica para passar um parâmetro de lista é concatenar a lista usando um delimitador, escolhas populares são :, | ou ,. A função split permite então recuperar a lista como um array awk:

awk -v 'serialised_list=a🅱️c:d:e:f' 'BEGIN {
  list_sz = split(serialised_list, list, ":")
  for(i = 1; i <= list_sz; ++i) {
    printf("list: %d: %s\n", i, list[i])
  }
  exit
}'

A saída deste programa awk é

list: 1: a
list: 2: b
list: 3: c
list: 4: d
list: 5: e
list: 6: f

Às vezes, é mais conveniente recuperar itens de lista como chaves de um array awk, pois isso permite uma fácil verificação de associação. Por exemplo, o programa a seguir imprime cada linha cuja primeira palavra não pertence a uma lista fixa de exceções:

awk -v 'serialised_exception_list=apple🍐cherry' 'BEGIN {
  _list_sz = split(serialised_exception_list, _list, ":")
  for(i = 1; i <= _list_sz; ++i) {
    exception[_list[i]]
  }
}

! ($1 in exception) { print }' <<EOF
apple Apples are yummy, I like them.
pineapple Do you like pineapple?
EOF

A saída deste programa é

pineapple Do you like pineapple?

Como exemplo final, mostramos como envolver o programa punishment em um shell script, pois isso ilustra como um shell script transmite parâmetros para um script auxiliar awk:

#!/bin/sh

usage()
{
   cat <<EOF
Usage: punishment [-c COUNT][-s SENTENCE]
 Prepare your punishments for you
EOF
}

punishment_count='100'
punishment_sentence='I shall not talk in class.'
while getopts "c:hs:" OPTION; do
  case "${OPTION}" in
    c) punishment_count="${OPTARG}";;
    s) punishment_sentence="${OPTARG}";;
    h) usage; exit 0;;
    *) usage; exit 64;;
  esac
done

awk -v "count=${punishment_count}" -v "sentence=${punishment_sentence}" 'BEGIN {
  for(i = 1; i <= count; ++i) {
    print(sentence)
  }
  exit
}'

Variáveis ​​locais #

A linguagem awk não suporta diretamente variáveis ​​locais para funções. No entanto, é fácil emulá-los adicionando argumentos extras às funções. É tradicional prefixar essas variáveis ​​com um _ para indicar que elas não são parâmetros reais.

Ilustramos essa técnica com a definição de uma função single_quote que adiciona aspas simples em torno de uma string:

# single_quote(TEXT)
#  Return a string made of TEXT surrounded by single quotes

function single_quote(text, _quote) {
  _quote = sprintf("%c", 39)
  return sprintf("%s%s%s", _quote, text, _quote);
}

A abordagem mais simples de usar sprintf("'%s'", text) leva a problemas práticos porque scripts awk geralmente são passados ​​como argumentos entre aspas simples para o programa awk.

Argumentos de atribuição #

Os argumentos de atribuição aparecem no final de uma chamada awk, na mesma área que as variáveis ​​de arquivo, tanto as atribuições -v quanto as atribuições de argumento devem corresponder à seguinte expressão regular. (assumindo uma localidade POSIX)

^[[:alpha:]_][[:alnum:]_]*=

O exemplo a seguir assume um arquivo file contendo o seguinte: 1 2 3 (separado por espaços em branco)

$ awk '{$1=$1}1' file OFS=, file OFS=- file
1 2 3
1,2,3
1-2-3
