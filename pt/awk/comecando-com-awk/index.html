AWK por exemplos #

AWK é uma linguagem de manipulação de strings, amplamente utilizada em sistemas UNIX. A ideia por trás do AWK era criar uma linguagem versátil para usar ao trabalhar em arquivos, o que não era muito complexo para entender.

O AWK possui algumas outras variantes, mas o conceito principal é o mesmo, apenas com recursos adicionais. Essas outras variantes são NAWK e GAWK. O GAWK contém todos os recursos de ambos, enquanto o NAWK está um passo acima do AWK, se preferir.

A maneira mais simples de pensar no AWK é considerar que ele possui 2 partes principais. O padrão e a ação.

Provavelmente o exemplo mais básico de AWK: (Veja também: Hello World)

BEGIN {print "START"}
      {print        }
END   {print "STOP" }

Aqui, as palavras-chave BEGIN e END são o padrão, enquanto a ação está dentro do {}. Este exemplo seria inútil, mas seriam necessárias apenas pequenas alterações para torná-lo uma função útil.

BEGIN {print "File\tAuthor"}
      {print $8, "\t", $3}
END {print " - DONE - "}

Aqui, \t representa um caractere Tab e é usado para igualar os limites da linha de saída. $8 e $3 são semelhantes ao uso que é visto em Shell Scripts, mas em vez de usar o 3º e 8º argumentos, ele usa a 3ª e 8ª coluna da linha de entrada.

Portanto, este exemplo imprimiria: Autor do arquivo na linha superior, enquanto a segunda linha está relacionada aos caminhos do arquivo. $8 é o nome do arquivo, $3 é o proprietário (ao olhar para o caminho do diretório, isso ficará mais claro). Finalmente, a linha inferior seria impressa, como seria de esperar - CONCLUÍDO -

O crédito para o exemplo acima vai para http://www.grymoire.com/Unix/Awk.html

Teoria mínima #

Geral awk one-liner:

awk <awk program> <file>

ou:

<shell-command> | awk <awk program>

<shell-command> e <file> são endereçados como awk input.

<awk program> é um código seguindo este modelo (aspas simples, não duplas):

'BEGIN   {<init actions>};
 <cond1> {<program actions>};
 <cond2> {<program actions>};
 ...
 END  {<final actions>}'

Onde:

  • A condição <condX> é mais frequentemente uma expressão regular /re/, para ser combinada com as linhas de entrada do awk;
  • <* actions> são sequências de statements, semelhantes aos comandos do shell, equipados com construções semelhantes a C.

`` é processado de acordo com as seguintes regras:
  1. BEGIN ... e END ... são opcionais e executados antes ou depois de processar as linhas de entrada do awk.
  2. Para cada linha na entrada awk, se a condição <condN> for meat, então o bloco <program actions> relacionado é executado.
  3. O padrão {<program actions>} é {print $0}.

Conditions pode ser combinado com operadores lógicos padrão:

    /gold/ || /USA/ && !/1986/

onde && tem precedência sobre ||;

A instrução print. A instrução print item1 item2 imprime itens em STDOUT. Os itens podem ser variáveis ​​(X, $0), strings (“hello”) ou números. item1, item2 são agrupados com o valor da variável OFS; item1 item2 são apenas apoxados! Use item1 " " item2 para espaços ou printf para mais recursos.

Variáveis não precisam de $, ou seja: print myVar; As seguintes variáveis ​​especiais estão embutidas no awk:

  • FS: atua como separador de campo para dividir as linhas de entrada do awk nos campos. Eu posso ser um único caractere, FS="c"; uma string nula, FS="" (então cada caractere individual se torna um campo separado); uma expressão regular sem barras, FS="re"; FS=" " significa execuções de espaços e tabulações e é o valor padrão.
  • NF: o número de campos a serem lidos;
  • $1, $2, …: 1º campo, 2º campo. etc. da linha de entrada atual,
  • $0: linha de entrada atual;
  • NR: número da linha de venda atual.
  • OFS: string para agrupar campos quando impressos.
  • ORS: separador de registro de saída, por padrão uma nova linha.
  • RS: Separador de linha de entrada (registro). Padrões para nova linha. Defina como FS.
  • IGNORECASE: afeta FS e RS quando são expressões regulares;

Exemplos de condição #

awk 'NR % 6'            # prints all lines except those divisible by 6
awk 'NR > 5'            # prints from line 6 onwards (like tail -n +6, or sed '1,5d')
awk '$2 == "foo"'       # prints lines where the second field is "foo"
awk '$2 ~ /re/'         # prints lines where the 2nd field mateches the regex /re/
awk 'NF >= 6'           # prints lines with 6 or more fields
awk '/foo/ && !/bar/'   # prints lines that match /foo/ but not /bar/
awk '/foo/ || /bar/'    # prints lines that match /foo/ or /bar/ (like grep -e 'foo' -e 'bar')
awk '/foo/,/bar/'       # prints from line matching /foo/ to line matching /bar/, inclusive
awk 'NF'                # prints only nonempty lines (or: removes empty lines, where NF==0)
awk 'NF--'              # removes last field and prints the line

Ao adicionar uma ação {...} pode-se imprimir um campo específico, em vez da linha inteira, por exemplo:

awk '$2 ~ /re/{print $3 " " $4}'

imprime o terceiro e o quarto campo de linhas onde o segundo campo corresponde à regex /re/.

match(s, r [, arr]) retorna a posição em s onde a regex r ocorre e define os valores de RSTART e RLENGTH. Se o argumento arr for fornecido, ele retorna o array arr onde os elementos são definidos para a subexpressão entre parênteses correspondente. As correspondências de elemento 0 de arr são definidas para a correspondência de regex inteira. Também as expressões arr[n, "start"] e arr[n, "length"] fornecem a posição inicial e o comprimento de cada substring correspondente.

Declarações #

Uma declaração simples geralmente é qualquer uma das seguintes:

variable = expression 
print [ expression-list ] 
printf format [ , expression-list ] 
next # skip remaining patterns on this input line
exit # skip the rest of the input

Se stat1 e stat2 forem declarações, o seguinte também será:

{stat}

{stat1;  stat2}

{stat1 
stat2}

if ( conditional ) statement [ else statement ]

As seguintes construções padrão do tipo C são declarações:

if ( conditional ) statement [ else statement ]
while ( conditional ) statement
for ( expression ; conditional ; expression ) statement
break    # usual C meaning 
continue # usual C meaning

Olá Mundo #

O exemplo Hello world é tão simples quanto:

awk 'BEGIN {print "Hello world"}'

O programa awk mais básico consiste em um valor verdadeiro (normalmente 1) e faz awk ecoar sua entrada:

$ date | awk '1'
Mon Jul 25 11:12:05 CEST 2016

Como “hello world” também é um valor verdadeiro, você também pode dizer:

$ date | awk '"hello world"'
Mon Jul 25 11:12:05 CEST 2016

No entanto, sua intenção fica muito mais clara se você escrever

$ date | awk '{print}'
Mon Jul 25 11:12:05 CEST 2016

em vez de.

Como executar programas AWK #

Se o programa for curto, você pode incluí-lo no comando que executa o awk:

awk -F: '{print $1, $2}' /etc/passwd

Neste exemplo, usando a opção de linha de comando -F:, aconselhamos o awk a usar : como delimitador de campos de entrada. é igualzinho

awk 'BEGIN{FS=":"}{print $1,$2}' file

Alternativamente, podemos salvar todo o código awk em um arquivo awk e chamar este programa awk assim:

awk -f 'program.awk' input-file1 input-file2 ...

program.awk pode ser qualquer programa multilinha, ou seja:

# file print_fields.awk
BEGIN {print "this is a header"; FS=":"}
{print $1, $2}
END {print "that was it"}

E então execute-o com:

awk -f print_fields.awk /etc/passwd   #-f advises awk which program file to load

Ou Mais geralmente:

awk -f program-file input-file1 input-file2 ...

A vantagem de ter o programa em um arquivo separado é que você pode escrever o programa com a identificação correta para fazer sentido, pode incluir comentários com # , etc
