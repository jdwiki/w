O projeto Spring Data permite que os programadores de aplicativos trabalhem com armazenamentos de dados usando uma interface consistente que faz uso de uma abstração chamada Repositório. Um “Repositório” Spring Data é modelado após o padrão de repositório popularizado por [design orientado por domínio](https://en.wikipedia.org/wiki /Design_controlado por domínio). O Spring Data fornece uma interface Java central chamada Repository que os subprojetos podem estender para fornecer recursos específicos para armazenamentos de dados.

Além da interface Repository, o Spring Data também fornece mais duas interfaces principais - CrudRepository que define o contrato para a funcionalidade básica CRUD (criar, ler, atualizar e excluir); e PagingAndSortingRepository que estende CrudRepository definindo um contrato para paginação e classificação.

Essas três interfaces principais (Repository, CrudRepository e PagingAndSortingRepository) garantem que:

  1. Os programadores de aplicativos podem acessar armazenamentos de dados (como bancos de dados relacionais, bancos de dados NoSQL baseados em documentos, bancos de dados de gráficos, etc.) de maneira consistente.
  2. É fácil alternar o armazenamento subjacente para uma entidade de domínio (consulte [design orientado por domínio](https://en.wikipedia.org/wiki/design_guiado por domínio)) sem ter que alterar também a forma de qual o aplicativo interage com o armazenamento de dados.
  3. Implementações específicas podem fornecer recursos específicos para armazenamentos de dados.

Criando um repositório para uma entidade gerenciada por JPA #

Classe Entidade

@Entity
@Table(name = "USER")
public class User {
    @Id
    @Column(name = "ID")
    private Long id;
    
    @Column(name = "USERNAME")
    private String username;
    
    @ManyToOne
    @JoinColumn("ORGANIZATION_ID")
    private Organization organization;
}

Interface do Repositório

@Repository
public interface UserRepository extends CrudRepository<User, Long> {
    public User findByUsername(String username);
}

A declaração do método na interface irá gerar a seguinte consulta jpql:

select u from User u where u.username = :username

alternativamente, podemos definir uma consulta personalizada:

@Query("select u from User u where u.username = :username")
public User findByUsername(@Param("username") String username)

podemos facilmente adicionar ordenação à declaração do método:

public interface UserRepository extends PagingAndSortingRepository<User, Long> {
    public User findByUsernameOrderByUsernameAsc(String username);
}

também podemos usar o suporte de paginação embutido:

public Page<User> findByOrganizationPaged(Organization organization, Pageable pageable);

a camada de serviço (ou quem chamar esse método) passará um PageRequest para o método:

public Page<User> getByOrganizationPagedOrderByUsername(Organization organization, int page, int size, String direction){
    return userRepository.findByOrganizationPaged(organization, new PageRequest(page, size, Direction.valueOf(direction),
                "username")
}

Encontrando todas as instâncias de uma classe de entidade #

Todas as instâncias (objetos) de uma classe de entidade podem ser carregadas da tabela de banco de dados subjacente da seguinte forma (semelhante a recuperar todas as linhas da tabela):

Iterable<Foo> foos = fooRepository.findAll();

O método findAll é fornecido pela interface CrudRepository. Ele retorna um Iterable em vez de um tipo mais concreto como List ou Set porque [algumas implementações da interface podem não conseguir retornar um tipo Collection](https://jira.spring.io/browse /DATACMNS-21) e, portanto, usar um tipo Collection para o valor retornado resultará em perda de funcionalidade para eles.

Invocar o método findAll resulta na consulta JPA select foo from Foo foo sendo executada no banco de dados subjacente.

Encontrando uma instância particular de uma classe de entidade pelo identificador #

Uma instância específica de uma classe de entidade pode ser carregada da seguinte forma:

Foo foo = fooRepository.findOne(id);

O método findOne é fornecido pela interface CrudRepository. Ele espera um identificador que identifique exclusivamente uma instância de entidade (por exemplo, uma chave primária em uma tabela de banco de dados). O tipo Java para o parâmetro id deve corresponder ao tipo atribuído ao atributo de entidade anotado com a anotação JPA @Id.

Invocar o método findOne resulta na consulta JPA select foo from Foo foo where foo.[primary-key- column] = :id sendo executado no banco de dados subjacente.

Encontrando todas as instâncias de uma classe de entidade com um atributo que corresponde a um valor especificado #

Todas as instâncias de uma classe de entidade com um dos atributos de classe correspondentes a um valor especificado podem ser recuperadas da seguinte forma:

public interface FooRepository extends CrudRepository<Foo, Long> {
  List<Foo> findAllByName(String name);
}

Invocar o método findAllByName resulta na consulta JPA select foo from Foo foo where foo.name = :name será executado no banco de dados subjacente.

Pontos a observar

  1. name deve ser um atributo na classe de entidade Foo.
  2. O nome do método deve começar com find, get ou read. Outras palavras-chave como select não funcionarão.
  3. Não há garantia quanto à ordem de devolução dos resultados.
