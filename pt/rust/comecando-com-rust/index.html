Uso avançado de println! #

println! (e seu irmão, [print!](https://doc.rust-lang. org/std/macro.print!.html)) fornece um mecanismo conveniente para produzir e imprimir texto que contém dados dinâmicos, semelhante à família de funções printf encontrada em muitas outras linguagens. Seu primeiro argumento é uma string de formato, que determina como os outros argumentos devem ser impressos como texto. A string de formato pode conter espaços reservados (incluídos em {}) para especificar que uma substituição deve ocorrer:

// No substitution -- the simplest kind of format string
println!("Hello World");
// Output: Hello World

// The first {} is substituted with a textual representation of
// the first argument following the format string. The second {}
// is substituted with the second argument, and so on.
println!("{} {} {}", "Hello", true, 42);
// Output: Hello true 42

Neste ponto, você deve estar se perguntando: Como o println! soube imprimir o valor booleano true como a string “true”? {} é realmente uma instrução para o formatador de que o valor deve ser convertido em texto usando o atributo Display . Esta característica é implementada para a maioria dos tipos primitivos de Rust (strings, números, booleanos, etc.) e destina-se a “saída voltada para o usuário”. Assim, o número 42 será impresso em decimal como 42, e não, digamos, em binário, que é como é armazenado internamente.

Como imprimimos tipos, então, que não implementam Display, exemplos sendo Slices ([i32]), vetores (Vec<i32>) ou opções (Option<&str>)? Não há uma representação textual clara para o usuário (ou seja, uma que você poderia inserir trivialmente em uma frase). Para facilitar a impressão de tais valores, o Rust também possui o atributo Debug e o correspondente {:?} espaço reservado. Da documentação: “Debug deve formatar a saída em um contexto de depuração voltado para o programador.” Vejamos alguns exemplos:

println!("{:?}", vec!["a", "b", "c"]);
// Output: ["a", "b", "c"]

println!("{:?}", Some("fantastic"));
// Output: Some("fantastic")

println!("{:?}", "Hello");
// Output: "Hello"
// Notice the quotation marks around "Hello" that indicate
// that a string was printed.

O Debug também possui um mecanismo de impressão bonito embutido, que você pode habilitar usando o modificador # após os dois pontos:

println!("{:#?}", vec![Some("Hello"), None, Some("World")]);
// Output: [
//    Some(
//        "Hello"
//    ),
//    None,
//    Some(
//        "World"
//    )
// ]

As strings de formato permitem que você expresse razoavelmente substituições complexas:

// You can specify the position of arguments using numerical indexes.
println!("{1} {0}", "World", "Hello");
// Output: Hello World

// You can use named arguments with format
println!("{greeting} {who}!", greeting="Hello", who="World");
// Output: Hello World

// You can mix Debug and Display prints:
println!("{greeting} {1:?}, {0}", "and welcome", Some(42), greeting="Hello");
// Output: Hello Some(42), and welcome

println! e amigos também irão avisá-lo se você estiver tentando fazer algo que não vai funcionar, ao invés de travar em tempo de execução:

// This does not compile, since we don't use the second argument.
println!("{}", "Hello World", "ignored");

// This does not compile, since we don't give the second argument.
println!("{} {}", "Hello");

// This does not compile, since Option type does not implement Display
println!("{}", Some(42));

Em sua essência, as macros de impressão Rust são simplesmente wrappers em torno da macro format!, que permite construir uma string juntando juntos representações textuais de diferentes valores de dados. Assim, para todos os exemplos acima, você pode substituir println! por format! para armazenar a string formatada em vez de imprimi-la:

let x: String = format!("{} {}", "Hello", 42);
assert_eq!(x, "Hello 42");

Saída do console sem macros #

// use Write trait that contains write() function
use std::io::Write;

fn main() {
    std::io::stdout().write(b"Hello, world!\n").unwrap();
}

  • O traço std::io::Write é projetado para objetos que aceitam fluxos de bytes. Neste caso, um handle para a saída padrão é adquirido com std::io::stdout().

  • Write::write() aceita uma fatia de byte (&[u8]), que é criada com um literal de cadeia de bytes (b"<string>"). Write::write() retorna um Result<usize, IoError>, que contém o número de bytes gravados (em caso de sucesso) ou um valor de erro (em caso de falha).

  • A chamada para Result::unwrap() indica que a chamada deve ser bem-sucedida (Result<usize, IoError> -> usize), e o valor é descartado.

Exemplo mínimo #

Para escrever o programa tradicional Hello World em Rust, crie um arquivo de texto chamado hello.rs contendo o seguinte código-fonte:

fn main() {
    println!("Hello World!");
}

Isso define uma nova função chamada main, que não recebe parâmetros e não retorna dados. É aqui que seu programa inicia a execução quando executado. Dentro dele, você tem um println!, que é uma macro que imprime texto no console.

Para gerar um aplicativo binário, invoque o compilador Rust passando o nome do arquivo de origem:

$ rustc hello.rs

O executável resultante terá o mesmo nome do módulo fonte principal, portanto, para executar o programa em um sistema Linux ou MacOS, execute:

$ ./hello
Hello World!

Em um sistema Windows, execute:

C:\Rust> hello.exe
Hello World!

Começando #

Instalando #

Antes que você possa fazer qualquer coisa usando a linguagem de programação Rust, você precisará adquiri-la—para Windows ou usando seu terminal em sistemas tipo Unix, onde $ simboliza a entrada no terminal:

$ curl https://sh.rustup.rs -sSf | sh

Isso recuperará os arquivos necessários e configurará a versão mais recente do Rust para você, independentemente do sistema em que você estiver. Para mais informações, veja página do projeto.

_Nota: Algumas distribuições Linux (por exemplo, Arch Linux) fornecem rustup como um pacote, que pode ser instalado em seu lugar. E embora muitos sistemas Unix-like forneçam rustc e cargo como pacotes separados, ainda é recomendado usar rustup já que torna muito mais fácil gerenciar vários canais de lançamento e fazer compilação cruzada. _

Compilador Rust #

Agora podemos verificar se o Rust foi de fato instalado com sucesso em nossos computadores executando o seguinte comando em nosso terminal—se no UNIX—ou no prompt de comando—se no Windows:

$ rustc --version

Se este comando for bem sucedido, a versão do compilador de Rust instalada em nossos computadores será exibida diante de nossos olhos.

Carga #

Com Rust vem Cargo, que é uma ferramenta de construção usada para gerenciar seus pacotes e projetos Rust. Para garantir que isso também esteja presente em seu computador, execute o seguinte dentro do console - console referindo-se ao terminal ou ao prompt de comando, dependendo do sistema em que você está:

$ cargo --version

Assim como o comando equivalente para o compilador Rust, isso retornará e exibirá a versão atual do Cargo.

Para criar seu primeiro projeto Cargo, você pode acessar [Cargo][2].

Alternativamente, você pode compilar programas diretamente usando rustc como mostrado em [Exemplo mínimo][3].
