Cargo é o gerenciador de pacotes do Rust, usado para gerenciar caixas (termo do Rust para bibliotecas/pacotes). Cargo busca predominantemente pacotes de crates.io e pode gerenciar árvores de dependência complexas com requisitos de versão específicos (usando versionamento semântico). O Cargo também pode ajudar a construir, executar e gerenciar projetos Rust com cargo build, cargo run e cargo test (entre outros comandos úteis).

Sintaxe #

  • carga novo crate_name [–bin]
  • inicialização de carga [–bin]
  • construção de carga [–release]
  • corrida de carga [–release] *verificação de carga
  • teste de carga *banco de carga
  • atualização de carga
  • pacote de carga
  • publicação de carga
  • carga [un]instalar binary_crate_name
  • pesquisa de carga crate_name
  • versão de carga
  • api_key de login de carga
  • No momento, o subcomando cargo bench requer que a versão noturna do compilador funcione efetivamente.

Criar novo projeto #

Biblioteca #

cargo new my-library

Isso cria um novo diretório chamado my-library contendo o arquivo de configuração de carga e um diretório de origem contendo um único arquivo de origem Rust:

my-library/Cargo.toml
my-library/src/lib.rs

Esses dois arquivos já conterão o esqueleto básico de uma biblioteca, de modo que você pode fazer um teste de carga (de dentro do diretório minha-biblioteca) imediatamente para verificar se tudo funciona.

Binário #

cargo new my-binary --bin

Isso cria um novo diretório chamado my-binary com uma estrutura semelhante a uma biblioteca:

my-binary/Cargo.toml
my-binary/src/main.rs

Desta vez, cargo terá configurado um binário simples Hello World que podemos executar imediatamente com cargo run.

Você também pode criar o novo projeto no diretório atual com o subcomando init:

cargo init --bin

Assim como acima, remova o sinalizador --bin para criar um novo projeto de biblioteca. O nome da pasta atual é usado automaticamente como nome da caixa.

Construir projeto #

Depurar #

cargo build

Liberar #

Construir com o sinalizador --release permite certas otimizações do compilador que não são feitas ao construir uma compilação de depuração. Isso faz com que o código seja executado mais rápido, mas também torna o tempo de compilação um pouco mais longo. Para um desempenho ideal, esse comando deve ser usado quando uma compilação de lançamento estiver pronta.

cargo build --release

Executando testes #

Uso Básico #

cargo test

Mostra a saída do programa #

cargo test -- --nocapture

Executar exemplo específico #

cargo test test_name

Publicando uma caixa #

Para publicar uma caixa em crates.io, você deve fazer login com Cargo (consulte ‘Conectando Cargo a uma conta Crates.io’).

Você pode empacotar e publicar sua caixa com os seguintes comandos:

cargo package
cargo publish

Quaisquer erros em seu arquivo Cargo.toml serão destacados durante este processo. Você deve certificar-se de atualizar sua versão e garantir que seu arquivo .gitignore ou Cargo.toml exclua quaisquer arquivos indesejados.

Conectando Carga a uma Conta Crates.io #

As contas no crates.io são criadas fazendo login no GitHub; você não pode se inscrever com nenhum outro método.

Para conectar sua conta do GitHub ao crates.io, clique no botão ‘Login with GitHub’ na barra de menu superior e autorize o crates.io a acessar sua conta. Isso fará com que você faça login no crates.io, supondo que tudo tenha corrido bem.

Você deve então encontrar sua chave de API, que pode ser encontrada clicando em seu avatar, indo em ‘Configurações da conta’ e copiando a linha que se parece com esta:

cargo login abcdefghijklmnopqrstuvwxyz1234567890rust

Isso deve ser colado em seu terminal/linha de comando e deve autenticá-lo com sua instalação local do cargo.

Tenha cuidado com sua chave de API - ela deve ser mantida em segredo, como uma senha, caso contrário suas caixas podem ser invadidas!

Programa Olá mundo #

Esta é uma sessão de shell mostrando como criar um programa “Hello world” e executá-lo com Cargo:

$ cargo new hello --bin
$ cd hello
$ cargo run
   Compiling hello v0.1.0 (file:///home/rust/hello)
     Running `target/debug/hello`
Hello, world!

Depois de fazer isso, você pode editar o programa abrindo src/main.rs em um editor de texto.
