Pânicos e Descontrações

  • Tente não entrar em pânico

Nesta página

  • Tente não entrar em pânico

Quando os programas Rust atingem um estado em que ocorreu um erro crítico, a macro panic! pode ser chamada para sair rapidamente (geralmente comparada, mas sutilmente diferente, a uma exceção em outras linguagens). O tratamento adequado de erros deve envolver tipos Result, embora esta seção discuta apenas panic! e seus conceitos.

Pânicos nem sempre causam vazamentos de memória ou outros vazamentos de recursos. Na verdade, panics normalmente preservam invariantes RAII, executando os destruidores (implementações Drop) de structs à medida que a pilha se desenrola. No entanto, se houver um segundo pânico durante esse processo, o programa simplesmente aborta; nesse ponto, as garantias invariantes RAII são nulas.

Tente não entrar em pânico #

No Rust, existem dois métodos principais para indicar que algo deu errado em um programa: Uma função que retorna um ([potencialmente personalizado](https://www.wikiod.com/pt/rust/manipulacao-de-erros custom-error-types#t=201609201438235945124)) Err(E), do [Result<T, E>](https://www.wikiod.com/pt/rust/manipulacao-de-erros /basic-error-handling-with-the-result-type#t=201609201438235945124) e um panic!.

Entrar em pânico não é uma alternativa para exceções, que são comumente encontradas em outros idiomas. Em Rust, um pânico é indicar que algo deu muito errado e que não pode continuar. Aqui está um exemplo da fonte Vec para push:

pub fn push(&mut self, value: T) {
    // This will panic or abort if we would allocate > isize::MAX bytes
    // or if the length increment would overflow for zero-sized types.
    if self.len == self.buf.cap() {
        self.buf.double();
    }
    ...
}

Se ficarmos sem memória, não há muito mais o que Rust possa fazer, então ele entrará em pânico (o comportamento padrão) ou abortará (o que precisa ser definido com um sinalizador de compilação).

O pânico desenrolará a pilha, executando destruidores e garantindo que a memória seja limpa. Abort não faz isso e depende do sistema operacional para limpá-lo corretamente.

Tente executar o seguinte programa normalmente e com

[profile.dev]
panic = "abort"

no seu Cargo.toml.

// main.rs
struct Foo(i32);
impl Drop for Foo {
    fn drop(&mut self) {
        println!("Dropping {:?}!", self.0);
    }
}
fn main() {
    let foo = Foo(1);
    panic!("Aaaaaaahhhhh!");
}
