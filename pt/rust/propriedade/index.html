Propriedade é um dos conceitos mais importantes em Rust, e é algo que não está presente na maioria das outras linguagens. A ideia de que um valor pode ser possuído por uma variável específica é muitas vezes muito difícil de entender, especialmente em linguagens onde a cópia é implícita, mas esta seção revisará as diferentes ideias em torno da propriedade.

Sintaxe #

  • let x: &T = … // x é uma referência imutável

  • let x: &mut T = … // x é uma referência exclusiva e mutável

  • deixe _ = &mut foo; // empresta foo mutável (ou seja, exclusivamente)

  • deixe _ = &foo; // empresta foo imutavelmente

  • deixe _ = foo; // move foo (requer propriedade)

  • Em versões muito mais antigas do Rust (antes de 1.0; maio de 2015), uma variável de propriedade tinha um tipo começando com ~. Você pode ver isso em exemplos muito antigos.

Propriedade e empréstimo #

Todos os valores em Rust têm exatamente um proprietário. O proprietário é responsável por descartar esse valor quando sai do escopo e é o único que pode mover a propriedade do valor. O proprietário de um valor pode fornecer referências a ele, permitindo que outros pedaços de código emprestem esse valor. A qualquer momento, pode haver qualquer número de referências imutáveis ​​a um valor:

let owned = String::from("hello");
// since we own the value, we may let other variables borrow it
let immutable_borrow1 = &owned;
// as all current borrows are immutable, we can allow many of them
let immutable_borrow2 = &owned;
// in fact, if we have an immutable reference, we are also free to
// duplicate that reference, since we maintain the invariant that
// there are only immutable references
let immutable_borrow3 = &*immutable_borrow2;

ou uma única referência mutável a ele (ERROR denota um erro em tempo de compilação):

// for us to borrow a value mutably, it must be mutable
let mut owned = String::from("hello");
// we can borrow owned mutably
let mutable_borrow = &mut owned;
// but note that we cannot borrow owned *again*
let mutable_borrow2 = &mut owned; // ERROR, already borrowed
// nor can we cannot borrow owned immutably
// since a mutable borrow is exclusive.
let immutable_borrow = &owned; // ERROR, already borrowed

Se houver referências pendentes (mutáveis ​​ou imutáveis) a um valor, esse valor não pode ser movido (ou seja, sua propriedade doada). Teríamos que ter certeza de que todas as referências foram descartadas primeiro para poder mover um valor:

let foo = owned; // ERROR, outstanding references to owned
let owned = String::from("hello");
{
    let borrow = &owned;
    // ...
} // the scope ends the borrow
let foo = owned; // OK, owned and not borrowed

Empréstimos e vidas #

Todos os valores em Rust têm uma vida. O tempo de vida de um valor abrange o segmento de código desde o valor é introduzido até onde ele é movido ou o final do escopo contido

{
    let x = String::from("hello"); //             +
    // ...                                        :
    let y = String::from("hello"); //      +      |
    // ...                                 :      :
    foo(x) // x is moved                   |      = x's lifetime
    // ...                                 :
} //                                       = y's lifetime

Sempre que você empresta um valor, a referência resultante tem um tempo de vida que está vinculado ao tempo de vida do valor que está sendo emprestado:

{
    let x = String::from("hello");
    let y = String::from("world");
    // when we borrow y here, the lifetime of the reference
    // stored in foo is equal to the lifetime of y
    // (i.e., between let y = above, to the end of the scope below)
    let foo = &y;
    // similarly, this reference to x is bound to the lifetime
    // of x --- bar cannot, for example, spawn a thread that uses
    // the reference beyond where x is moved below.
    bar(&x);
}

Propriedade e chamadas de função #

A maioria das questões sobre propriedade surgem ao escrever funções. Quando você especifica os tipos de argumentos de uma função, você pode escolher como esse valor é passado. Se você só precisa de acesso somente leitura, você pode usar uma referência imutável:

fn foo(x: &String) {
    // foo is only authorized to read x's contents, and to create
    // additional immutable references to it if it so desires.
    let y = *x; // ERROR, cannot move when not owned
    x.push_str("foo"); // ERROR, cannot mutate with immutable reference
    println!("{}", x.len()); // reading OK
    foo(x); // forwarding reference OK
}

Se foo precisar modificar o argumento, ele deve receber uma referência exclusiva e mutável:

fn foo(x: &mut String) {
    // foo is still not responsible for dropping x before returning,
    // nor is it allowed to. however, foo may modify the String.
    let x2 = *x; // ERROR, cannot move when not owned
    x.push_str("foo"); // mutating OK
    drop(*x); // ERROR, cannot drop value when not owned
    println!("{}", x.len()); // reading OK
}

Se você não especificar & ou &mut, você está dizendo que a função se apropriará de um argumento. Isso significa que foo agora também é responsável por descartar x.

fn foo(x: String) {
    // foo may do whatever it wishes with x, since no-one else has
    // access to it. once the function terminates, x will be dropped,
    // unless it is moved away when calling another function.
    let mut x2 = x; // moving OK
    x2.push_str("foo"); // mutating OK
    let _ = &mut x2; // mutable borrow OK
    let _ = &x2; // immutable borrow OK (note that &mut above is dropped)
    println!("{}", x2.len()); // reading OK
    drop(x2); // dropping OK
}

Propriedade e o traço de cópia #

Alguns tipos de Rust implementam a característica Copy. Tipos que são ‘Copiar’ podem ser movidos sem possuir o valor em questão. Isso ocorre porque o conteúdo do valor pode simplesmente ser copiado byte a byte na memória para produzir um valor novo e idêntico. A maioria das primitivas em Rust (bool, usize, f64, etc.) são Copy.

let x: isize = 42;
let xr = &x;
let y = *xr; // OK, because isize is Copy
// both x and y are owned here

Notavelmente, Vec e String são não Copy:

let x = Vec::new();
let xr = &x;
let y = *xr; // ERROR, cannot move out of borrowed content
