Sintaxe #

  • mod modname; // Procure o módulo em modname.rs ou modname/mod.rs no mesmo diretório
  • mod modname { block }

O atributo #[caminho] #

O atributo #[path] do Rust pode ser usado para especificar o caminho para procurar um módulo específico se ele não estiver no local padrão. Isso é normalmente desencorajado, no entanto, porque torna a hierarquia do módulo frágil e torna fácil quebrar a compilação movendo um arquivo em um diretório completamente diferente.

#[path="../path/to/module.rs"]
mod module;

Árvore de módulos #

Arquivos:

- example.rs (root of our modules tree, generally named lib.rs or main.rs when using Cargo)
- first.rs
- second/
  - mod.rs
  - sub.rs

Módulos:

- example        -> example
  - first        -> example::first
  - second       -> example::second
    - sub        -> example::second::sub
  - third        -> example::third

exemplo.rs

pub mod first;
pub mod second;
pub mod third {
    ...
}

O módulo second deve ser declarado no arquivo example.rs pois seu pai é example e não, por exemplo, first e, portanto, não pode ser declarado no first.rs ou outro arquivo em o mesmo nível de diretório

segundo/mod.rs

pub mod sub;

Nomes no código vs nomes em use #

A sintaxe de dois-pontos dos nomes na instrução use se parece com os nomes usados ​​em outras partes do código, mas o significado desses caminhos é diferente.

Nomes na instrução use por padrão são interpretados como absolutos, começando na raiz do engradado. Os nomes em outras partes do código são relativos ao módulo atual.

A declaração:

use std::fs::File;

tem o mesmo significado no arquivo principal do engradado assim como nos módulos. Por outro lado, um nome de função como std::fs::File::open() fará referência à biblioteca padrão do Rust apenas no arquivo principal da grade, porque os nomes no código são interpretados em relação ao atual módulo.

fn main() {
    std::fs::File::open("example"); // OK
}

mod my_module {
   fn my_fn() {
       // Error! It means my_module::std::fs::File::open()
       std::fs::File::open("example"); 

       // OK. `::` prefix makes it absolute 
       ::std::fs::File::open("example"); 

       // OK. `super::` reaches out to the parent module, where `std` is present
       super::std::fs::File::open("example"); 
   } 
}

Para fazer com que os nomes std::… se comportem em todos os lugares da mesma forma que na raiz da caixa, você pode adicionar:

use std;

Por outro lado, você pode tornar os caminhos use relativos prefixando-os com palavras-chave self ou super:

 use self::my_module::my_fn;

Acessando o Módulo Pai #

Às vezes, pode ser útil importar funções e structs relativamente sem ter que usar algo com seu caminho absoluto em seu projeto. Para conseguir isso, você pode usar o módulo super, assim:

fn x() -> u8 {
    5
}

mod example {
    use super::x;

    fn foo() {
        println!("{}", x());
    }
}

Você pode usar super várias vezes para alcançar o ‘avô’ do seu módulo atual, mas deve ter cuidado ao introduzir problemas de legibilidade se usar super muitas vezes em uma importação.

Exportações e Visibilidade #

Organização básica do código #

Vamos ver como podemos organizar o código, quando a base de código estiver ficando maior.

01. Funções

fn main() {
  greet();
}

fn greet() {
  println!("Hello, world!");
}

02. Módulos - No mesmo arquivo

fn main() {
  greet::hello();
}

mod greet {
  // By default, everything inside a module is private
  pub fn hello() { // So function has to be public to access from outside
    println!("Hello, world!");
  }
}

03. Módulos - Em um arquivo diferente no mesmo diretório

Ao mover algum código para um novo arquivo, não há necessidade de envolver o código em uma declaração mod. O próprio arquivo atua como um módulo.

// ↳ main.rs
mod greet; // import greet module

fn main() {
  greet::hello();
}

// ↳ greet.rs
pub fn hello() { // function has to be public to access from outside
  println!("Hello, world!");
}

Ao mover algum código para um novo arquivo, se esse código foi encapsulado de uma declaração mod, que será um submódulo do arquivo.

// ↳ main.rs
mod greet;

fn main() {
  greet::hello::greet();
}

// ↳ greet.rs
pub mod hello { // module has to be public to access from outside
  pub fn greet() { // function has to be public to access from outside
    println!("Hello, world!");
  }
}

04. Módulos - Em um arquivo diferente em um diretório diferente

Ao mover algum código para um novo arquivo em um diretório diferente, o próprio diretório atua como um módulo. E mod.rs na raiz do módulo é o ponto de entrada para o módulo de diretório. Todos os outros arquivos nesse diretório atuam como um submódulo desse diretório.

// ↳ main.rs
mod greet;

fn main() {
  greet::hello();
}

// ↳ greet/mod.rs
pub fn hello() {
  println!("Hello, world!");
}

Quando você tem vários arquivos na raiz do módulo,

// ↳ main.rs
mod greet;

fn main() {
  greet::hello_greet()
}

// ↳ greet/mod.rs
mod hello;

pub fn hello_greet() {
  hello::greet()
}

// ↳ greet/hello.rs
pub fn greet() {
  println!("Hello, world!");
}

05. Módulos - Com self

fn main() {
  greet::call_hello();
}

mod greet {
  pub fn call_hello() { 
    self::hello();
  }

  fn hello() {
    println!("Hello, world!");
  }
}

06. Módulos - Com super

  1. Quando você deseja acessar uma função root de dentro de um módulo,
fn main() {
  dash::call_hello();
}

fn hello() {
  println!("Hello, world!");
}

mod dash {
  pub fn call_hello() {
    super::hello();
  }
}
  1. Quando você deseja acessar uma função no módulo externo/pai de dentro de um módulo aninhado,
fn main() {
  outer::inner::call_hello();
}

mod outer {

  pub fn hello() {
    println!("Hello, world!");
  }

  mod inner {
    pub fn call_hello() {
      super::hello();
    }
  }

}

07. Módulos - Com use

  1. Quando você deseja vincular o caminho completo a um novo nome,
use greet::hello::greet as greet_hello;

fn main() {
  greet_hello();
}

mod greet {
  pub mod hello {
    pub fn greet() {
      println!("Hello, world!");
    }
  }
}
  1. Quando você deseja usar o conteúdo do nível do escopo da caixa
fn main() {
  user::hello();
}

mod greet {
  pub mod hello {
    pub fn greet() {
      println!("Hello, world!");
    }
  }
}

mod user {
  use greet::hello::greet as call_hello;

  pub fn hello() {
    call_hello();
  }
}
