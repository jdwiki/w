As operações de agregações processam registros de dados e retornam resultados computados. As operações de agregação agrupam valores de vários documentos e podem executar várias operações nos dados agrupados para retornar um único resultado. O MongoDB fornece três maneiras de realizar a agregação: o pipeline de agregação, a função map-reduce e métodos de agregação de propósito único.

Do manual do Mongo https://docs.mongodb.com/manual/aggregation/

Sintaxe #

  • db.collection.aggregate(pipeline, opções)

Parâmetros #

Parâmetro Detalhes
tubulação array(Uma sequência de operações ou estágios de agregação de dados)
opções document(opcional, disponível apenas se o pipeline estiver presente como um array)

A estrutura de agregação no MongoDB é usada para obter a funcionalidade comum GROUP BY do SQL.

Considere as seguintes inserções na coleção chamada transactions para cada exemplo.

> db.transactions.insert({ cr_dr : "D", amount : 100, fee : 2});
> db.transactions.insert({ cr_dr : "C", amount : 100, fee : 2});
> db.transactions.insert({ cr_dr : "C", amount : 10,  fee : 2});
> db.transactions.insert({ cr_dr : "D", amount : 100, fee : 4});
> db.transactions.insert({ cr_dr : "D", amount : 10,  fee : 2});
> db.transactions.insert({ cr_dr : "C", amount : 10,  fee : 4});
> db.transactions.insert({ cr_dr : "D", amount : 100, fee : 2});

Contar #

Como você obtém o número de transações de débito e crédito? Uma maneira de fazer isso é usando a função count() como abaixo.

> db.transactions.count({cr_dr : "D"});

ou

> db.transactions.find({cr_dr : "D"}).length();

Mas e se você não souber os possíveis valores de cr_dr antecipadamente. Aqui a estrutura de agregação vem para jogar. Consulte a consulta agregada abaixo.

> db.transactions.aggregate( 
      [
          {
              $group : {
                  _id : '$cr_dr', // group by type of transaction
                // Add 1 for each document to the count for this type of transaction
                  count : {$sum : 1}
              }
          }
      ]
  );

E o resultado é

{
    "_id" : "C",
    "count" : 3
}
{
    "_id" : "D",
    "count" : 5
}

Soma #

Como obter a soma de ‘quantia’? Veja a consulta agregada abaixo.

> db.transactions.aggregate( 
     [
         {
             $group : {
                 _id : '$cr_dr',
                 count : {$sum : 1},    //counts the number
                 totalAmount : {$sum : '$amount'}    //sums the amount
             }
         }
     ] 
 );

E o resultado é

{
    "_id" : "C",
    "count" : 3.0,
    "totalAmount" : 120.0
}
{
    "_id" : "D",
    "count" : 5.0,
    "totalAmount" : 410.0
}

Outra versão que soma quantia e taxa.

> db.transactions.aggregate(
      [
          {
              $group : {
                  _id : '$cr_dr',
                  count : {$sum : 1},
                  totalAmount : {$sum : { $sum : ['$amount', '$fee']}}
              }
          }
      ] 
  );

E o resultado é

{
    "_id" : "C",
    "count" : 3.0,
    "totalAmount" : 128.0
}    
{
    "_id" : "D",
    "count" : 5.0,
    "totalAmount" : 422.0
}

Média #

Como obter o valor médio das transações de débito e crédito?

> db.transactions.aggregate(
    [
        {
            $group : {
                _id : '$cr_dr', // group by type of transaction (debit or credit)
                count : {$sum : 1},    // number of transaction for each type
                totalAmount : {$sum : { $sum : ['$amount', '$fee']}},    // sum
                averageAmount : {$avg : { $sum : ['$amount', '$fee']}}   // average
            }
        }
    ] 
)

O resultado é

{
    "_id" : "C", // Amounts for credit transactions
    "count" : 3.0,
    "totalAmount" : 128.0,
    "averageAmount" : 40.0
}
{
    "_id" : "D", // Amounts for debit transactions
    "count" : 5.0,
    "totalAmount" : 422.0,
    "averageAmount" : 82.0
}

Operações com arrays. #

Quando você deseja trabalhar com as entradas de dados em matrizes, primeiro precisa desenrolar a matriz. A operação de desenrolamento cria um documento para cada entrada na matriz. Quando você tem muitos documentos com grandes arrays, você verá uma explosão no número de documentos.

{ "_id" : 1, "item" : "myItem1", sizes: [ "S", "M", "L"] }
{ "_id" : 2, "item" : "myItem2", sizes: [ "XS", "M", "XL"] }
    
db.inventory.aggregate( [ { $unwind : "$sizes" }] )

Um aviso importante é que quando um documento não contém o array ele será perdido. A partir do mongo 3.2 e superior, há uma opção de desenrolamento “preserveNullAndEmptyArrays” adicionada. Esta opção garante que o documento seja preservado quando a matriz estiver ausente.

{ "_id" : 1, "item" : "myItem1", sizes: [ "S", "M", "L"] }
{ "_id" : 2, "item" : "myItem2", sizes: [ "XS", "M", "XL"] }
{ "_id" : 3, "item" : "myItem3" }
    
db.inventory.aggregate( [ { $unwind : { path: "$sizes", includeArrayIndex: "arrayIndex" } }] )

Combine #

Como escrever uma consulta para obter todos os departamentos onde a idade média dos funcionários que ganham menos ou $ 70.000 é maior ou igual a 35?

Para isso, precisamos escrever uma consulta para corresponder aos funcionários que têm um salário menor ou igual a $ 70.000. Em seguida, adicione o estágio agregado para agrupar os funcionários por departamento. Em seguida, adicione um acumulador com um campo chamado e.g. average_age para encontrar a idade média por departamento usando o acumulador $avg e abaixo dos agregados $match e $group existentes, adicione outro agregado $match para que possamos recuperar apenas resultados com um average_age maior ou igual a 35.

db.employees.aggregate([
  {"$match": {"salary": {"$lte": 70000}}},
  {"$group": {"_id": "$dept",
              "average_age": {"$avg": "$age"}
             }
  },
  {"$match": {"average_age": {"$gte": 35}}}
])

O resultado é:

{
  "_id": "IT",
  "average_age": 31
}
{
  "_id": "Customer Service",
  "average_age": 34.5
}
{
  "_id": "Finance",
  "average_age": 32.5
}

Remova documentos que possuem um campo duplicado em uma coleção (desduplicação) #

Observe que a opção allowDiskUse: true é opcional, mas ajudará a mitigar problemas de falta de memória, pois essa agregação pode ser uma operação intensiva de memória se o tamanho da coleção for grande - portanto, recomendo sempre usá-la.

var duplicates = [];

db.transactions.aggregate([
 { $group: { 
   _id: { cr_dr: "$cr_dr"},
   dups: { "$addToSet": "$_id" }, 
   count: { "$sum": 1 } 
 }
}, 
{ $match: { 
  count: { "$gt": 1 }   
}}
],allowDiskUse: true}
)              
.result          
.forEach(function(doc) {
  doc.dups.shift();   
  doc.dups.forEach( function(dupId){ 
    duplicates.push(dupId); 
  }
)    
})
// printjson(duplicates);     

// Remove all duplicates in one go    
db.transactions.remove({_id:{$in:duplicates}})
