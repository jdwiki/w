Sintaxe #

  • insert(documento ou array de documentos)
  • inserirUm( ‘documento’, { writeConcern: ‘document’ } )
  • inserirMuitos( { [ documento 1 , documento 2, … ] }, { writeConcern: document, ordered: boolean } )
  • find(consulta, projeção)
  • findOne(consulta, projeção)
  • atualização(consulta, atualização)
  • atualizarUm( consulta, atualizar, { upsert: boolean, writeConcern: document } )
  • atualizarMuitos( consulta, atualizar, { upsert: boolean, writeConcern: document } )
  • substituirUm( consulta, substituição, { upsert: boolean, writeConcern: document } )
  • remove(consulta, justOne)
  • findAndModify(query, sort, update, options[opcional])

Atualizar e excluir um documento deve ser feito com cuidado. Uma vez que a operação pode afetar vários documentos.

Crio #

db.people.insert({name: 'Tom', age: 28});

Ou

db.people.save({name: 'Tom', age: 28});

A diferença com save é que se o documento passado contiver um campo _id, se um documento já existir com esse _id ele será atualizado ao invés de ser adicionado como novo.

Dois novos métodos para inserir documentos em uma coleção, no MongoDB 3.2.x:-

Use insertOne para inserir apenas um registro: -

db.people.insertOne({name: 'Tom', age: 28});

Use insertMany para inserir vários registros: -

db.people.insertMany([{name: 'Tom', age: 28},{name: 'John', age: 25}, {name: 'Kathy', age: 23}])

Observe que insert é destacado como obsoleto em todos os drivers de idioma oficial desde a versão 3.0. A diferença completa é que os métodos do shell na verdade ficaram atrás dos outros drivers na implementação do método. A mesma coisa se aplica a todos os outros métodos CRUD

Atualizar #

Atualize o objeto inteiro:

db.people.update({name: 'Tom'}, {age: 29, name: 'Tom'})

// New in MongoDB 3.2
db.people.updateOne({name: 'Tom'},{age: 29, name: 'Tom'}) //Will replace only first matching document.

db.people.updateMany({name: 'Tom'},{age: 29, name: 'Tom'}) //Will replace all matching documents.

Ou apenas atualize um único campo de um documento. Neste caso idade:

db.people.update({name: 'Tom'}, {$set: {age: 29}})

Você também pode atualizar vários documentos simultaneamente adicionando um terceiro parâmetro. Esta consulta atualizará todos os documentos cujo nome seja igual a Tom:

db.people.update({name: 'Tom'}, {$set: {age: 29}}, {multi: true})

// New in MongoDB 3.2
db.people.updateOne({name: 'Tom'},{$set:{age: 30}) //Will update only first matching document.

db.people.updateMany({name: 'Tom'},{$set:{age: 30}}) //Will update all matching documents.

Se um novo campo estiver chegando para atualização, esse campo será adicionado ao documento.

db.people.updateMany({name: 'Tom'},{$set:{age: 30, salary:50000}})// Document will have `salary` field as well.

Se for necessário substituir um documento,

db.collection.replaceOne({name:'Tom'}, {name:'Lakmal',age:25,address:'Sri Lanka'})

pode ser usado.

Observação: Os campos usados ​​para identificar o objeto serão salvos no documento atualizado. Campos que não estiverem definidos na seção de atualização serão removidos do documento.

Excluir #

Exclui todos os documentos que correspondem ao parâmetro de consulta:

// New in MongoDB 3.2
db.people.deleteMany({name: 'Tom'})

// All versions
db.people.remove({name: 'Tom'})

Ou apenas um

// New in MongoDB 3.2
db.people.deleteOne({name: 'Tom'})

// All versions
db.people.remove({name: 'Tom'}, true)

O método remove() do MongoDB. Se você executar este comando sem nenhum argumento ou sem argumento vazio, ele removerá todos os documentos da coleção.

db.people.remove();

ou

db.people.remove({});

Ler #

Consulte todos os documentos na coleção people que tenham um campo name com um valor de 'Tom':

db.people.find({name: 'Tom'})

Ou apenas o primeiro:

db.people.findOne({name: 'Tom'})

Você também pode especificar quais campos devem ser retornados passando um parâmetro de seleção de campo. O seguinte excluirá o campo _id e incluirá apenas o campo age:

db.people.find({name: 'Tom'}, {_id: 0, age: 1})

Nota: por padrão, o campo _id será retornado, mesmo que você não o solicite. Se você não quiser obter o _id de volta, você pode simplesmente seguir o exemplo anterior e pedir que o _id seja excluído especificando _id: 0 (ou _id: false). encontrar sub-registro como objeto de endereço contém país, cidade, etc.

db.people.find({'address.country': 'US'})

& especifique o campo também se necessário

db.people.find({'address.country': 'US'}, {'name': true, 'address.city': true})Remember that the result has a `.pretty()` method that pretty-prints resulting JSON:

db.people.find().pretty()

Mais operadores de atualização #

Você pode usar outros operadores além de $set ao atualizar um documento. O operador $push permite que você insira um valor em um array, neste caso adicionaremos um novo apelido ao array nicknames.

db.people.update({name: 'Tom'}, {$push: {nicknames: 'Tommy'}})
// This adds the string 'Tommy' into the nicknames array in Tom's document.

O operador $pull é o oposto de $push, você pode puxar itens específicos de arrays.

db.people.update({name: 'Tom'}, {$pull: {nicknames: 'Tommy'}})
// This removes the string 'Tommy' from the nicknames array in Tom's document.

O operador $pop permite remover o primeiro ou o último valor de um array. Digamos que o documento de Tom tenha uma propriedade chamada brothers que tenha o valor ['Marie', 'Bob', 'Kevin', 'Alex'].

db.people.update({name: 'Tom'}, {$pop: {siblings: -1}})
// This will remove the first value from the siblings array, which is 'Marie' in this case.


db.people.update({name: 'Tom'}, {$pop: {siblings: 1}})
// This will remove the last value from the siblings array, which is 'Alex' in this case.

Parâmetro “multi” ao atualizar vários documentos #

Para atualizar vários documentos em uma coleção, defina a opção multi como true.

db.collection.update(
   query,
   update,
   {
     upsert: boolean,
     multi: boolean,
     writeConcern: document
   }
)

multi é opcional. Se definido como true, atualiza vários documentos que atendem aos critérios de consulta. Se definido como false, atualiza um documento. O valor padrão é falso.

db.mycol.find() { “_id”: ObjectId(598354878df45ec5), “title”:“Visão geral do MongoDB”} { “_id”: ObjectId(59835487adf45ec6), “title”:“Visão geral do NoSQL”} { “_id”: ObjectId(59835487adf45ec7), “title”:“Visão geral dos pontos de tutoriais”}

db.mycol.update({’title’:‘MongoDB Overview’}, {$set:{’title’:‘Novo Tutorial MongoDB’}},{multi:true})

Atualização de documentos incorporados. #

Para o seguinte esquema:

{name: 'Tom', age: 28, marks: [50, 60, 70]}

Atualize as marcas de Tom para 55, onde as marcas são 50 (use o operador posicional $):

db.people.update({name: "Tom", marks: 50}, {"$set": {"marks.$": 55}})

Para o seguinte esquema:

{name: 'Tom', age: 28, marks: [{subject: "English", marks: 90},{subject: "Maths", marks: 100}, {subject: "Computes", marks: 20}]}

Atualize as marcas em inglês de Tom para 85:

db.people.update({name: "Tom", "marks.subject": "English"},{"$set":{"marks.$.marks": 85}})

Explicando o exemplo acima:

Usando {name: “Tom”, “marks.subject”: “English”} você obterá a posição do objeto no array de marcas, onde o assunto é inglês. Em “marks.$.marks”, $ é usado para atualizar nessa posição do array de marcas

Atualizar valores em uma matriz

O operador posicional $ identifica um elemento em uma matriz para atualizar sem especificar explicitamente a posição do elemento na matriz.

Considere uma coleção de alunos com os seguintes documentos:

{ "_id" : 1, "grades" : [ 80, 85, 90 ] }
{ "_id" : 2, "grades" : [ 88, 90, 92 ] }
{ "_id" : 3, "grades" : [ 85, 100, 90 ] }

Para atualizar de 80 para 82 no array grades no primeiro documento, use o operador posicional $ se você não souber a posição do elemento no array:

db.students.update(
   { _id: 1, grades: 80 },
   { $set: { "grades.$" : 82 } }
)
