Vamos lançar um único thread CUDA para dizer olá #

Este simples programa CUDA demonstra como escrever uma função que será executada na GPU (também conhecida como “dispositivo”). A CPU, ou “host”, cria threads CUDA chamando funções especiais chamadas “kernels”. Os programas CUDA são programas C++ com sintaxe adicional.

Para ver como funciona, coloque o seguinte código em um arquivo chamado hello.cu:

#include <stdio.h>

// __global__ functions, or "kernels", execute on the device
__global__ void hello_kernel(void)
{
  printf("Hello, world from the device!\n");
}

int main(void)
{
  // greet from the host
  printf("Hello, world from the host!\n");

  // launch a kernel with a single thread to greet from the device
  hello_kernel<<<1,1>>>();

  // wait for the device to finish so that we see the message
  cudaDeviceSynchronize();

  return 0;
}

(Observe que, para usar a função printf no dispositivo, você precisa de um dispositivo com capacidade de computação de pelo menos 2.0. Consulte a [visão geral das versões](https://www.wikiod.com/pt/cuda/comecando-com-cuda /introduction-to-cuda#t=20160918151348344166&a=versions) para obter detalhes.)

Agora vamos compilar o programa usando o compilador NVIDIA e executá-lo:

$ nvcc hello.cu -o hello
$ ./hello
Hello, world from the host!
Hello, world from the device!

Algumas informações adicionais sobre o exemplo acima:

  • nvcc significa “NVIDIA CUDA Compiler”. Ele separa o código-fonte em componentes de host e dispositivo.
  • __global__ é uma palavra-chave CUDA usada em declarações de função indicando que a função é executada no dispositivo GPU e é chamada do host.
  • Colchetes angulares triplos (<<<,>>>) marcam uma chamada do código do host para o código do dispositivo (também chamado de “lançamento do kernel”). Os números dentro desses colchetes triplos indicam o número de vezes para executar em paralelo e o número de threads.

Pré-requisitos #

Para começar a programar com CUDA, baixe e instale o CUDA Toolkit and developer driver. O kit de ferramentas inclui nvcc, o compilador NVIDIA CUDA e outros softwares necessários para desenvolver aplicativos CUDA. O driver garante que os programas GPU sejam executados corretamente em hardware compatível com CUDA, que você também precisará.

Você pode confirmar que o CUDA Toolkit está instalado corretamente em sua máquina executando nvcc --version a partir de uma linha de comando. Por exemplo, em uma máquina Linux,

$ nvcc --version
nvcc: NVIDIA (R) Cuda compiler driver
Copyright (c) 2005-2016 NVIDIA Corporation
Built on Tue_Jul_12_18:28:38_CDT_2016
Cuda compilation tools, release 8.0, V8.0.32

emite as informações do compilador. Se o comando anterior não foi bem sucedido, então o CUDA Toolkit provavelmente não está instalado, ou o caminho para nvcc (C:\CUDA\bin em máquinas Windows, /usr/local/cuda/bin em sistemas operacionais POSIX ) não faz parte da sua variável de ambiente PATH.

Além disso, você também precisará de um compilador de host que funcione com nvcc para compilar e construir programas CUDA. No Windows, este é o cl.exe, o compilador da Microsoft, que vem com o Microsoft Visual Studio. Em sistemas operacionais POSIX, outros compiladores estão disponíveis, incluindo gcc ou g++. O [Guia de início rápido] oficial do CUDA 1 pode informar quais versões do compilador são suportadas em sua plataforma específica.

Para garantir que tudo esteja configurado corretamente, vamos compilar e executar um programa CUDA trivial para garantir que todas as ferramentas funcionem juntas corretamente.

__global__ void foo() {}

int main()
{
  foo<<<1,1>>>();

  cudaDeviceSynchronize();
  printf("CUDA error: %s\n", cudaGetErrorString(cudaGetLastError()));

  return 0;
}

Para compilar este programa, copie-o para um arquivo chamado test.cu e compile-o a partir da linha de comando. Por exemplo, em um sistema Linux, o seguinte deve funcionar:

$ nvcc test.cu -o test
$ ./test
CUDA error: no error

Se o programa for bem-sucedido sem erros, vamos começar a codificar!

Compilando e executando os programas de exemplo #

O guia de instalação da NVIDIA termina com a execução dos programas de amostra para verificar a instalação do CUDA Toolkit, mas não informa explicitamente como. Primeiro verifique todos os pré-requisitos. Verifique o diretório CUDA padrão para os programas de amostra. Se não estiver presente, pode ser baixado do site oficial da CUDA. Navegue até o diretório onde os exemplos estão presentes.

$ cd /path/to/samples/
$ ls

Você deve ver uma saída semelhante a:

0_Simple     2_Graphics  4_Finance      6_Advanced       bin     EULA.txt
1_Utilities  3_Imaging   5_Simulations  7_CUDALibraries  common  Makefile

Certifique-se de que o Makefile esteja presente neste diretório. O comando make em sistemas baseados em UNIX construirá todos os programas de amostra. Alternativamente, navegue até um subdiretório onde outro Makefile esteja presente e execute o comando make de lá para construir apenas essa amostra.

Soma dois arrays com CUDA #

Este exemplo ilustra como criar um programa simples que irá somar dois arrays int com CUDA.

Um programa CUDA é heterogêneo e consiste em partes executadas tanto na CPU quanto na GPU.

As partes principais de um programa que utiliza CUDA são semelhantes aos programas de CPU e consistem em

  • Alocação de memória para dados que serão usados ​​na GPU
  • Cópia de dados da memória do host para a memória das GPUs
  • Invocando a função do kernel para processar dados
  • Copie o resultado para a memória da CPU

Para alocar a memória dos dispositivos usamos a função cudaMalloc. Para copiar dados entre o dispositivo e o host, a função cudaMemcpy pode ser usada. O último argumento de cudaMemcpy especifica a direção da operação de cópia. Existem 5 tipos possíveis:

  • cudaMemcpyHostToHost - Host -> Host
  • cudaMemcpyHostToDevice - Host -> Dispositivo
  • cudaMemcpyDeviceToHost - Dispositivo -> Host
  • cudaMemcpyDeviceToDevice - Dispositivo -> Dispositivo
  • cudaMemcpyDefault - Espaço de endereço virtual unificado baseado em padrão

Em seguida, a função do kernel é invocada. A informação entre as divisas triplas é a configuração de execução, que determina quantos threads de dispositivo executam o kernel em paralelo. O primeiro número (2 no exemplo) especifica o número de blocos e o segundo ((tamanho + 1) / 2 no exemplo) - número de threads em um bloco. Observe que neste exemplo adicionamos 1 ao tamanho, de modo que solicitamos um thread extra em vez de ter um thread responsável por dois elementos.

Como a invocação do kernel é uma função assíncrona, cudaDeviceSynchronize é chamada para aguardar até que a execução seja concluída. Os arrays de resultados são copiados para a memória do host e toda a memória alocada no dispositivo é liberada com cudaFree.

Para definir a função como especificador de declaração do kernel __global__ é usado. Esta função será invocada por cada thread. Se quisermos que cada thread processe um elemento da matriz resultante, precisamos de um meio de distinguir e identificar cada thread. CUDA define as variáveis ​​blockDim, blockIdx e threadIdx. A variável predefinida blockDim contém as dimensões de cada bloco de thread conforme especificado no segundo parâmetro de configuração de execução para a inicialização do kernel. As variáveis ​​predefinidas threadIdx e blockIdx contêm o índice do thread dentro de seu bloco de thread e o bloco de thread dentro da grade, respectivamente. Observe que, como potencialmente solicitamos um thread a mais do que elementos nos arrays, precisamos passar size para garantir que não acessemos além do final do array.

#include "cuda_runtime.h"
#include "device_launch_parameters.h"

#include <stdio.h>

__global__ void addKernel(int* c, const int* a, const int* b, int size) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < size) {
        c[i] = a[i] + b[i];
    }
}

// Helper function for using CUDA to add vectors in parallel.
void addWithCuda(int* c, const int* a, const int* b, int size) {
    int* dev_a = nullptr;
    int* dev_b = nullptr;
    int* dev_c = nullptr;

    // Allocate GPU buffers for three vectors (two input, one output)
    cudaMalloc((void**)&dev_c, size * sizeof(int));
    cudaMalloc((void**)&dev_a, size * sizeof(int));
    cudaMalloc((void**)&dev_b, size * sizeof(int));

    // Copy input vectors from host memory to GPU buffers.
    cudaMemcpy(dev_a, a, size * sizeof(int), cudaMemcpyHostToDevice);
    cudaMemcpy(dev_b, b, size * sizeof(int), cudaMemcpyHostToDevice);

    // Launch a kernel on the GPU with one thread for each element.
    // 2 is number of computational blocks and (size + 1) / 2 is a number of threads in a block
    addKernel<<<2, (size + 1) / 2>>>(dev_c, dev_a, dev_b, size);
    
    // cudaDeviceSynchronize waits for the kernel to finish, and returns
    // any errors encountered during the launch.
    cudaDeviceSynchronize();

    // Copy output vector from GPU buffer to host memory.
    cudaMemcpy(c, dev_c, size * sizeof(int), cudaMemcpyDeviceToHost);

    cudaFree(dev_c);
    cudaFree(dev_a);
    cudaFree(dev_b);
}

int main(int argc, char** argv) {
    const int arraySize = 5;
    const int a[arraySize] = {  1,  2,  3,  4,  5 };
    const int b[arraySize] = { 10, 20, 30, 40, 50 };
    int c[arraySize] = { 0 };

    addWithCuda(c, a, b, arraySize);

    printf("{1, 2, 3, 4, 5} + {10, 20, 30, 40, 50} = {%d, %d, %d, %d, %d}\n", c[0], c[1], c[2], c[3], c[4]);

    cudaDeviceReset();

    return 0;
}
