O algoritmo de redução paralela normalmente se refere a um algoritmo que combina uma matriz de elementos, produzindo um único resultado. Os problemas típicos que se enquadram nesta categoria são:

  • somando todos os elementos em uma matriz
  • encontrar um máximo em uma matriz

Em geral, a redução paralela pode ser aplicada para qualquer binário operador associativo, ou seja, (A*B)*C = A*(B*C). Com tal operador *, o algoritmo de redução paralela agrupa repetidamente os argumentos do array em pares. Cada par é calculado em paralelo com os outros, reduzindo pela metade o tamanho geral do array em uma etapa. O processo é repetido até que apenas um único elemento exista.

Se o operador for comutativo (ou seja, A*B = B*A) além de ser associativo, o algoritmo pode emparelhar em um padrão diferente. Do ponto de vista teórico não faz diferença, mas na prática dá um melhor padrão de acesso à memória:

Nem todos os operadores associativos são comutativos - tome a multiplicação de matrizes, por exemplo.

Redução paralela de bloco único para operador comutativo #

A abordagem mais simples para redução paralela em CUDA é atribuir um único bloco para realizar a tarefa:

static const int arraySize = 10000;
static const int blockSize = 1024;

__global__ void sumCommSingleBlock(const int *a, int *out) {
    int idx = threadIdx.x;
    int sum = 0;
    for (int i = idx; i < arraySize; i += blockSize)
        sum += a[i];
    __shared__ int r[blockSize];
    r[idx] = sum;
    __syncthreads();
    for (int size = blockSize/2; size>0; size/=2) { //uniform
        if (idx<size)
            r[idx] += r[idx+size];
        __syncthreads();
    }
    if (idx == 0)
        *out = r[0];
}

...

sumCommSingleBlock<<<1, blockSize>>>(dev_a, dev_out);

Isso é mais viável quando o tamanho dos dados não é muito grande (em torno de alguns milhares de elementos). Isso geralmente acontece quando a redução faz parte de algum programa CUDA maior. Se a entrada corresponder a blockSize desde o início, o primeiro loop for pode ser completamente removido.

Observe que na primeira etapa, quando há mais elementos do que threads, adicionamos as coisas de forma totalmente independente. Somente quando o problema é reduzido para blockSize, a redução paralela real é acionada. O mesmo código pode ser aplicado a qualquer outro operador comutativo, associativo, como multiplicação, mínimo, máximo, etc.

Observe que o algoritmo pode ser feito mais rápido, por exemplo, usando uma redução paralela em nível de dobra.

Redução paralela de bloco único para operador não comutativo #

Fazer a redução paralela para um operador não comutativo é um pouco mais complicado, em comparação com a versão comutativa. No exemplo, ainda usamos uma adição sobre inteiros para simplificar. Poderia ser substituído, por exemplo, pela multiplicação de matrizes que realmente não é comutativa. Observe que, ao fazer isso, 0 deve ser substituído por um elemento neutro da multiplicação - ou seja, uma matriz identidade.

static const int arraySize = 1000000;
static const int blockSize = 1024;

__global__ void sumNoncommSingleBlock(const int *gArr, int *out) {
    int thIdx = threadIdx.x;
    __shared__ int shArr[blockSize*2];
    __shared__ int offset;
    shArr[thIdx] = thIdx<arraySize ? gArr[thIdx] : 0;
    if (thIdx == 0)
        offset = blockSize;
    __syncthreads();
    while (offset < arraySize) { //uniform
        shArr[thIdx + blockSize] = thIdx+offset<arraySize ? gArr[thIdx+offset] : 0;
        __syncthreads();
        if (thIdx == 0)
            offset += blockSize;
        int sum = shArr[2*thIdx] + shArr[2*thIdx+1];
        __syncthreads();
        shArr[thIdx] = sum;
    }
    __syncthreads();
    for (int stride = 1; stride<blockSize; stride*=2) { //uniform
        int arrIdx = thIdx*stride*2;
        if (arrIdx+stride<blockSize)
            shArr[arrIdx] += shArr[arrIdx+stride];
        __syncthreads();
    }   
    if (thIdx == 0)
        *out = shArr[0];
}

...

sumNoncommSingleBlock<<<1, blockSize>>>(dev_a, dev_out);

No primeiro while o loop é executado enquanto houver mais elementos de entrada do que threads. Em cada iteração, uma única redução é executada e o resultado é compactado na primeira metade do array shArr. A segunda metade é então preenchida com novos dados.

Uma vez que todos os dados são carregados de gArr, o segundo loop é executado. Agora, não comprimimos mais o resultado (o que custa um __syncthreads() extra). Em cada passo o thread n acessa o elemento ativo 2*n-th e adiciona-o com o elemento 2*n+1-th:

Existem muitas maneiras de otimizar ainda mais este exemplo simples, por exemplo. através da redução do nível warp e removendo conflitos de banco de memória compartilhada.

Redução paralela multibloco para operador comutativo #

A abordagem de vários blocos para redução paralela em CUDA representa um desafio adicional, em comparação com a abordagem de bloco único, porque os blocos são limitados na comunicação. A ideia é permitir que cada bloco calcule uma parte da matriz de entrada e, em seguida, tenha um bloco final para mesclar todos os resultados parciais. Para fazer isso, pode-se iniciar dois kernels, criando implicitamente um ponto de sincronização em toda a grade.

static const int wholeArraySize = 100000000;
static const int blockSize = 1024;
static const int gridSize = 24; //this number is hardware-dependent; usually #SM*2 is a good number.

__global__ void sumCommMultiBlock(const int *gArr, int arraySize, int *gOut) {
    int thIdx = threadIdx.x;
    int gthIdx = thIdx + blockIdx.x*blockSize;
    const int gridSize = blockSize*gridDim.x;
    int sum = 0;
    for (int i = gthIdx; i < arraySize; i += gridSize)
        sum += gArr[i];
    __shared__ int shArr[blockSize];
    shArr[thIdx] = sum;
    __syncthreads();
    for (int size = blockSize/2; size>0; size/=2) { //uniform
        if (thIdx<size)
            shArr[thIdx] += shArr[thIdx+size];
        __syncthreads();
    }
    if (thIdx == 0)
        gOut[blockIdx.x] = shArr[0];
}

__host__ int sumArray(int* arr) {
    int* dev_arr;
    cudaMalloc((void**)&dev_arr, wholeArraySize * sizeof(int));
    cudaMemcpy(dev_arr, arr, wholeArraySize * sizeof(int), cudaMemcpyHostToDevice);

    int out;
    int* dev_out;
    cudaMalloc((void**)&dev_out, sizeof(int)*gridSize);
    
    sumCommMultiBlock<<<gridSize, blockSize>>>(dev_arr, wholeArraySize, dev_out);
    //dev_out now holds the partial result
    sumCommMultiBlock<<<1, blockSize>>>(dev_out, gridSize, dev_out);
    //dev_out[0] now holds the final result
    cudaDeviceSynchronize();
    
    cudaMemcpy(&out, dev_out, sizeof(int), cudaMemcpyDeviceToHost);
    cudaFree(dev_arr);
    cudaFree(dev_out);
    return out;
}

O ideal é lançar blocos suficientes para saturar todos os multiprocessadores na GPU com ocupação total. Exceder esse número - em particular, lançar tantos threads quantos os elementos da matriz - é contraproducente. Isso não aumenta mais o poder de computação bruto, mas evita o uso do primeiro loop muito eficiente.

Também é possível obter o mesmo resultado usando um único kernel, com a ajuda de last-block guard:

static const int wholeArraySize = 100000000;
static const int blockSize = 1024;
static const int gridSize = 24;

__device__ bool lastBlock(int* counter) {
    __threadfence(); //ensure that partial result is visible by all blocks
    int last = 0;
    if (threadIdx.x == 0)
        last = atomicAdd(counter, 1);
    return __syncthreads_or(last == gridDim.x-1);
}    

__global__ void sumCommMultiBlock(const int *gArr, int arraySize, int *gOut, int* lastBlockCounter) {
    int thIdx = threadIdx.x;
    int gthIdx = thIdx + blockIdx.x*blockSize;
    const int gridSize = blockSize*gridDim.x;
    int sum = 0;
    for (int i = gthIdx; i < arraySize; i += gridSize)
        sum += gArr[i];
    __shared__ int shArr[blockSize];
    shArr[thIdx] = sum;
    __syncthreads();
    for (int size = blockSize/2; size>0; size/=2) { //uniform
        if (thIdx<size)
            shArr[thIdx] += shArr[thIdx+size];
        __syncthreads();
    }
    if (thIdx == 0)
        gOut[blockIdx.x] = shArr[0];
    if (lastBlock(lastBlockCounter)) {
        shArr[thIdx] = thIdx<gridSize ? gOut[thIdx] : 0;
        __syncthreads();
        for (int size = blockSize/2; size>0; size/=2) { //uniform
            if (thIdx<size)
                shArr[thIdx] += shArr[thIdx+size];
            __syncthreads();
        }
        if (thIdx == 0)
            gOut[0] = shArr[0];            
    }
}

__host__ int sumArray(int* arr) {
    int* dev_arr;
    cudaMalloc((void**)&dev_arr, wholeArraySize * sizeof(int));
    cudaMemcpy(dev_arr, arr, wholeArraySize * sizeof(int), cudaMemcpyHostToDevice);

    int out;
    int* dev_out;
    cudaMalloc((void**)&dev_out, sizeof(int)*gridSize);
    
    int* dev_lastBlockCounter;
    cudaMalloc((void**)&dev_lastBlockCounter, sizeof(int));
    cudaMemset(dev_lastBlockCounter, 0, sizeof(int));
    
    sumCommMultiBlock<<<gridSize, blockSize>>>(dev_arr, wholeArraySize, dev_out, dev_lastBlockCounter);
    cudaDeviceSynchronize();
    
    cudaMemcpy(&out, dev_out, sizeof(int), cudaMemcpyDeviceToHost);
    cudaFree(dev_arr);
    cudaFree(dev_out);
    return out;
}

Observe que o kernel pode ser feito mais rápido, por exemplo, usando uma redução paralela de nível warp.

Redução paralela multibloco para operador não comutativo #

A abordagem de vários blocos para redução paralela é muito semelhante à abordagem de bloco único. A matriz de entrada global deve ser dividida em seções, cada uma reduzida por um único bloco. Quando um resultado parcial de cada bloco é obtido, um bloco final reduz estes para obter o resultado final.

  • sumNoncommSingleBlock é explicado mais detalhadamente no exemplo de redução de bloco único.

  • lastBlock aceita apenas o último bloco que o alcança. Se você quiser evitar isso, você pode dividir o kernel em duas chamadas separadas.

    static const int wholeArraySize = 100000000; static const int blockSize = 1024; static const int gridSize = 24; //this number is hardware-dependent; usually #SM*2 is a good number.

    device bool lastBlock(int* counter) { __threadfence(); //ensure that partial result is visible by all blocks int last = 0; if (threadIdx.x == 0) last = atomicAdd(counter, 1); return __syncthreads_or(last == gridDim.x-1); }

    device void sumNoncommSingleBlock(const int* gArr, int arraySize, int* out) { int thIdx = threadIdx.x; shared int shArr[blockSize2]; shared int offset; shArr[thIdx] = thIdx<arraySize ? gArr[thIdx] : 0; if (thIdx == 0) offset = blockSize; __syncthreads(); while (offset < arraySize) { //uniform shArr[thIdx + blockSize] = thIdx+offset<arraySize ? gArr[thIdx+offset] : 0; __syncthreads(); if (thIdx == 0) offset += blockSize; int sum = shArr[2thIdx] + shArr[2thIdx+1]; __syncthreads(); shArr[thIdx] = sum; } __syncthreads(); for (int stride = 1; stride<blockSize; stride=2) { //uniform int arrIdx = thIdxstride2; if (arrIdx+stride<blockSize) shArr[arrIdx] += shArr[arrIdx+stride]; __syncthreads(); }
    if (thIdx == 0) *out = shArr[0]; }

    global void sumNoncommMultiBlock(const int* gArr, int* out, int* lastBlockCounter) { int arraySizePerBlock = wholeArraySize/gridSize; const int* gArrForBlock = gArr+blockIdx.xarraySizePerBlock; int arraySize = arraySizePerBlock; if (blockIdx.x == gridSize-1) arraySize = wholeArraySize - blockIdx.xarraySizePerBlock; sumNoncommSingleBlock(gArrForBlock, arraySize, &out[blockIdx.x]); if (lastBlock(lastBlockCounter)) sumNoncommSingleBlock(out, gridSize, out);
    }

Exceder esse número - em particular, lançar tantos threads quantos os elementos da matriz - é contraproducente. Redução paralela de dobra única para operador comutativo #

Às vezes, a redução deve ser realizada em uma escala muito pequena, como parte de um kernel CUDA maior. Suponha, por exemplo, que os dados de entrada tenham exatamente 32 elementos - o número de threads em um warp. Nesse cenário, uma única dobra pode ser atribuída para realizar a redução. Dado que o warp é executado em perfeita sincronia, muitas instruções __syncthreads() podem ser removidas - quando comparadas a uma redução em nível de bloco.

static const int warpSize = 32;

__device__ int sumCommSingleWarp(volatile int* shArr) {
    int idx = threadIdx.x % warpSize; //the lane index in the warp
    if (idx<16) shArr[idx] += shArr[idx+16];
    if (idx<8) shArr[idx] += shArr[idx+8];
    if (idx<4) shArr[idx] += shArr[idx+4];
    if (idx<2) shArr[idx] += shArr[idx+2];
    if (idx==0) shArr[idx] += shArr[idx+1];
    return shArr[0];
}

shArr é preferencialmente um array em memória compartilhada. O valor deve ser o mesmo para todos os threads no warp. Se sumCommSingleWarp for chamado por vários warps, shArr deve ser diferente entre warps (o mesmo dentro de cada warp).

O argumento shArr é marcado como volátil para garantir que as operações no array sejam realmente executadas onde indicado. Caso contrário, a atribuição repetitiva para shArr[idx] pode ser otimizada como uma atribuição para um registrador, com apenas a atribuição final sendo um armazenamento real para shArr. Quando isso acontece, as atribuições imediatas não são visíveis para outras threads, gerando resultados incorretos. Observe que você pode passar um array normal não volátil como um argumento de um volátil, da mesma forma que quando você passa non-const como um parâmetro const.

Se não se importar com o conteúdo de shArr[1..31] após a redução, pode-se simplificar ainda mais o código:

static const int warpSize = 32;

__device__ int sumCommSingleWarp(volatile int* shArr) {
    int idx = threadIdx.x % warpSize; //the lane index in the warp
    if (idx<16) {
      shArr[idx] += shArr[idx+16];
      shArr[idx] += shArr[idx+8];
      shArr[idx] += shArr[idx+4];
      shArr[idx] += shArr[idx+2];
      shArr[idx] += shArr[idx+1];
    }
    return shArr[0];
}

Nesta configuração removemos muitas condições if. Os threads extras realizam algumas adições desnecessárias, mas não nos importamos mais com o conteúdo que eles produzem. Como os warps são executados no modo SIMD, não economizamos tempo fazendo com que esses threads não façam nada. Por outro lado, avaliar as condições leva um tempo relativamente grande, já que o corpo dessas instruções if é muito pequeno. A instrução inicial if também pode ser removida se shArr[32..47] for preenchido com 0.

A redução de nível de dobra também pode ser usada para aumentar a redução de nível de bloco:

__global__ void sumCommSingleBlockWithWarps(const int *a, int *out) {
    int idx = threadIdx.x;
    int sum = 0;
    for (int i = idx; i < arraySize; i += blockSize)
        sum += a[i];
    __shared__ int r[blockSize];
    r[idx] = sum;
    sumCommSingleWarp(&r[idx & ~(warpSize-1)]);
    __syncthreads();
    if (idx<warpSize) { //first warp only
        r[idx] = idx*warpSize<blockSize ? r[idx*warpSize] : 0;
        sumCommSingleWarp(r);
        if (idx == 0)
            *out = r[0];
    }
}

O argumento &r[idx & ~(warpSize-1)] é basicamente r + warpIdx*32. Isso efetivamente divide o array r em pedaços de 32 elementos, e cada pedaço é atribuído a um warp separado.

Redução paralela de dobra única para operador não comutativo #

Às vezes, a redução deve ser realizada em uma escala muito pequena, como parte de um kernel CUDA maior. Suponha, por exemplo, que os dados de entrada tenham exatamente 32 elementos - o número de threads em um warp. Nesse cenário, uma única dobra pode ser atribuída para realizar a redução. Dado que o warp é executado em perfeita sincronia, muitas instruções __syncthreads() podem ser removidas - quando comparadas a uma redução em nível de bloco.

static const int warpSize = 32;

__device__ int sumNoncommSingleWarp(volatile int* shArr) {
    int idx = threadIdx.x % warpSize; //the lane index in the warp
    if (idx%2 == 0) shArr[idx] += shArr[idx+1];
    if (idx%4 == 0) shArr[idx] += shArr[idx+2];
    if (idx%8 == 0) shArr[idx] += shArr[idx+4];
    if (idx%16 == 0) shArr[idx] += shArr[idx+8];
    if (idx == 0) shArr[idx] += shArr[idx+16];
    return shArr[0];
}

shArr é preferencialmente um array em memória compartilhada. O valor deve ser o mesmo para todos os threads no warp. Se sumCommSingleWarp for chamado por vários warps, shArr deve ser diferente entre warps (o mesmo dentro de cada warp).

O argumento shArr é marcado como volátil para garantir que as operações no array sejam realmente executadas onde indicado. Caso contrário, a atribuição repetitiva para shArr[idx] pode ser otimizada como uma atribuição para um registrador, com apenas a atribuição final sendo um armazenamento real para shArr. Quando isso acontece, as atribuições imediatas não são visíveis para outras threads, gerando resultados incorretos. Observe que você pode passar um array normal não volátil como um argumento de um volátil, da mesma forma que quando você passa non-const como um parâmetro const.

Se alguém não se importa com o conteúdo final de shArr[1..31] e pode preencher shArr[32..47] com zeros, pode-se simplificar o código acima:

static const int warpSize = 32;

__device__ int sumNoncommSingleWarpPadded(volatile int* shArr) {
    //shArr[32..47] == 0
    int idx = threadIdx.x % warpSize; //the lane index in the warp
    shArr[idx] += shArr[idx+1];
    shArr[idx] += shArr[idx+2];
    shArr[idx] += shArr[idx+4];
    shArr[idx] += shArr[idx+8];
    shArr[idx] += shArr[idx+16];
    return shArr[0];
}

Nesta configuração removemos todas as condições if, que constituem cerca da metade das instruções. As threads extras realizam algumas adições desnecessárias, armazenando o resultado em células de shArr que, em última análise, não têm impacto no resultado final. Como os warps são executados no modo SIMD, não economizamos tempo fazendo com que esses threads não façam nada.

Redução paralela de dobra única usando apenas registradores #

Normalmente, a redução é realizada no array global ou compartilhado. No entanto, quando a redução é realizada em uma escala muito pequena, como parte de um kernel CUDA maior, ela pode ser realizada com um único warp. Quando isso acontece, em arquiteturas Keppler ou superiores (CC>=3.0), é possível usar funções warp-shuffle para evitar o uso de memória compartilhada.

Suponha, por exemplo, que cada thread em um warp mantenha um único valor de dados de entrada. Todas as threads juntas possuem 32 elementos, que precisamos somar (ou realizar outra operação associativa)

__device__ int sumSingleWarpReg(int value) {
    value += __shfl_down(value, 1);
    value += __shfl_down(value, 2);
    value += __shfl_down(value, 4);
    value += __shfl_down(value, 8);
    value += __shfl_down(value, 16);
    return __shfl(value,0);
}

Esta versão funciona para operadores comutativos e não comutativos.
