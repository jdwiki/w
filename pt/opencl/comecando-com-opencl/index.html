O que é OpenCL? #

OpenCL é a abreviação de Open Ccomputing Language. OpenCL é um Framework para programação paralela em plataformas heterogêneas, chamadas de dispositivos de computação, que vão desde CPUs sobre GPUs até plataformas mais especiais como FPGAs. OpenCL fornece uma interface padrão para computação paralela nesses dispositivos de computação, mas também paralelismo entre dispositivos. Ele especifica uma linguagem de programação, baseada em C99, e requisitos mínimos de funções básicas implementadas em dispositivos compatíveis com OpenCL. Além disso, OpenCL descreve um modelo abstrato de computação e memória, sendo o mais geral possível para tornar simples a reutilização de código entre diferentes plataformas.

Pré-requisitos #

Se você tem uma CPU moderna ou placa gráfica (GPU) dentro de sua máquina, é provável que você tenha tudo pronto para os primeiros passos no OpenCL. Descobrir se o seu processador é compatível com OpenCL geralmente pode ser feito através da página inicial do fabricante, um bom começo é a documentação oficial em

https://www.khronos.org/conformance/adopters/conformant-products#opencl

O que é OpenCL? #

Open Computing Language (OpenCL) é uma estrutura para escrever programas que são executados em CPUs, GPUs e outros processadores e aceleradores paralelos.

OpenCL especifica uma linguagem de programação (baseada em C) que fornece acesso à memória no chip nomeada, um modelo para executar tarefas em paralelo e a capacidade de sincronizar essas tarefas.

Implementação C# do OpenCL 1.2: número de plataformas para um sistema AMD em janelas de 64 bits #

OpenCL é uma API de baixo nível, portanto, deve ser implementada primeiro no “espaço C”. Para isso, é necessário baixar os arquivos de cabeçalho do site da Khronos. Meu hardware é AMD e compatível com a versão 1.2, baixando

opencl.h 
cl_platform.h 
cl.h 
cl_ext.h 
cl_egl.h 
cl_dx9_media_sharing.h 
cl_d3d10.h 
cl_d3d11.h 
cl_gl.h 
cl_gl_ext.h 
cl.hpp

de esta página

deve ser suficiente para ligações C++, então, depois de adicionar esses arquivos ao seu projeto e definir os locais apropriados dos arquivos binários (e da biblioteca) (

$(AMDAPPSDKROOT)\lib\x86_64 para biblioteca amd de 64 bits (as bibliotecas do SDK do aplicativo amd são preferidas)

,

C:\Windows\SysWOW64 para opencl.dll de 64 bits (arquivo .so se o ICD for de um sistema Linux)

por exemplo, mas diferente para Intel-Nvidia), você pode começar a consultar uma lista de plataformas (amd,intel,xilinx,nvidia) depois de instalar os drivers adequados (como crimson for amd). Drivers são para executar o aplicativo opencl (usando ICD), bibliotecas e arquivos de cabeçalho são para desenvolvimento em resumo.

Para consultar plataformas:

#define __CL_ENABLE_EXCEPTIONS
#include "stdafx.h"
#include <vector>
#include <CL/cl.hpp>

extern "C"
    {
       // when this class is created, it contains a list of platforms in "platforms" field.
       class OpenClPlatformList
       {
           public:
               std::vector<cl::Platform> platforms;
               int platformNum;
               OpenClPlatformList()
               {
                   platforms= std::vector< cl::Platform>();
                   cl::Platform::get(&platforms);
                   platformNum= platforms.size();
               }
        };


        // this is seen from C# when imported. Creates an object in memory.
        __declspec(dllexport)
            OpenClPlatformList * createPlatformList()
        {
            return new OpenClPlatformList();
        }

        __declspec(dllexport)
            int platformNumber(OpenClPlatformList * hList)
        {
            return hList->platformNum;
        }


        __declspec(dllexport)
            void deletePlatformList(OpenClPlatformList * p)
        {
            if (p != NULL)
                delete p;
            p = NULL;
        }


    }

pode ser construído em uma dll (como OCLImplementation.dll)

e para usá-lo do lado do C#,

using System;
using System.Collections.Generic;
using System.Runtime.InteropServices;


namespace WrapperCSharp
{
    public class WrapperCSharp
    {
        [DllImport("OCLImplementation", CallingConvention = CallingConvention.Cdecl)]
        private static extern IntPtr createPlatformList();

        [DllImport("OCLImplementation", CallingConvention = CallingConvention.Cdecl)]
        private static extern int platformNumber(IntPtr hList);

        [DllImport("OCLImplementation", CallingConvention = CallingConvention.Cdecl)]
        private static extern void deletePlatformList(IntPtr hList);
    }
}

é claro que a dll deve ser vista pelo projeto C#, basta colocá-la perto do executável do projeto para resolver.

Agora, se o computador de amostra tiver pelo menos uma plataforma compatível com opencl,

IntPtr platformList = createPlatformList(); // just an address in C-space
int totalPlatforms = platformNumber(platformList); // AMD+NVIDIA systems should have "2"
deletePlatformList(platformList); //

A variável totalPlatforms deve ter pelo menos o valor “1”. Em seguida, você pode usar a variável de plataformas no C-space usando funções adicionais para iterar em todas as plataformas para consultar todos os dispositivos, como CPU, GPU e aceleradores de propósito especial, como phi ou alguns fpga.

Não se escreve simplesmente todos esses wrappers C++ para C# para projetos de tempo crítico. Existem muitos wrappers escritos para C#, Java e outras linguagens. Para java, existe “Aparapi” que é a api do conversor “java bytecode to opencl-c” que leva o que você escreve puramente em java para uma versão gpu-parallel em tempo real, por isso é um pouco portátil.

OpenCL e C# #

Para C# existem muitos wrappers que oferecem uma interface para comunicação com OpenCL.

  • OpenCL.NET: Este é um dos wrappers de nível mais baixo que existem. Ele oferece uma implementação completa da API OpenCL para C# sem adicionar nenhuma abstração. Assim, os exemplos C\C++ são facilmente portados para esta biblioteca. A única página do projeto está atualmente no codeplex, que é encerrada em 15.12.2017, mas o pacote está disponível no NuGet

https://openclnet.codeplex.com/

  • NOpenCL: Esta biblioteca oferece uma interface abstrata entre C# e OpenCL.

O objetivo de curto prazo é fornecer uma camada abstrata fácil de usar que forneça acesso a todos os recursos do OpenCL sem sacrificar o desempenho.

https://github.com/tunnelvisionlabs/NOpenCL

  • Clou:

Cloo é uma biblioteca gerenciada de código aberto, fácil de usar, que permite que aplicativos .NET/Mono aproveitem ao máximo a estrutura OpenCL.

https://sourceforge.net/projects/cloo/
