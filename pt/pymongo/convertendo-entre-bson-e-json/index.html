Em muitos aplicativos, os registros do MongoDB precisam ser serializados no formato JSON. Se seus registros tiverem campos do tipo date, datetime, objectId, binary, code, etc. você encontrará exceções TypeError: not JSON serializable ao usar json.dumps. Este tópico mostra como superar isso.

Usando json_util #

json_util fornece dois métodos auxiliares, dumps e loads, que envolvem os métodos json nativos e fornecem conversão BSON explícita de e para json.

Uso simples #

from bson.json_util import loads, dumps
record = db.movies.find_one()
json_str = dumps(record)
record2 = loads(json_str)

se record for:

{ 
    "_id" : ObjectId("5692a15524de1e0ce2dfcfa3"), 
    "title" : "Toy Story 4", 
    "released" : ISODate("2010-06-18T04:00:00Z") 
}

então json_str se torna:

{
    "_id": {"$oid": "5692a15524de1e0ce2dfcfa3"},
    "title" : "Toy Story 4", 
    "released": {"$date": 1276833600000}
}

JSONOptions #

É possível customizar o comportamento de dumps através de um objeto JSONOptions. Dois conjuntos de opções já estão disponíveis: DEFAULT_JSON_OPTIONS e STRICT_JSON_OPTIONS.

>>> bson.json_util.DEFAULT_JSON_OPTIONS
    JSONOptions(strict_number_long=False, datetime_representation=0,
     strict_uuid=False, document_class=dict, tz_aware=True, 
     uuid_representation=PYTHON_LEGACY, unicode_decode_error_handler='strict',
     tzinfo=<bson.tz_util.FixedOffset object at 0x7fc168a773d0>)

Para usar diferentes opções, você pode:

  1. modifique o objeto DEFAULT_JSON_OPTIONS. Neste caso, as opções serão usadas para todas as chamadas subsequentes para dumps:

     from bson.json_util import DEFAULT_JSON_OPTIONS
 DEFAULT_JSON_OPTIONS.datetime_representation = 2
 dumps(record)

  2. especifique um JSONOptions em uma chamada para dumps usando o parâmetro json_options:

     # using strict
 dumps(record, json_options=bson.json_util.STRICT_JSON_OPTIONS)

 # using a custom set of options
 from bson.json_util import JSONOptions
 options = JSONOptions() # options is a copy of DEFAULT_JSON_OPTIONS
 options.datetime_representation=2
 dumps(record, json_options=options)

Os parâmetros de JSONOptions são:

  • strict_number_long: Se true, os objetos Int64 são codificados para o modo Strict do MongoDB Extended JSON tipo NumberLong, ou seja, {"$numberLong": "<number>" }. Caso contrário, eles serão codificados como um int. Padrões para Falso.
  • datetime_representation: A representação a ser usada ao codificar instâncias de datetime.datetime. 0 => {"$date": <dateAsMilliseconds>}, 1 => {"$date": {"$numberLong": "<dateAsMilliseconds>"}}, 2 => {"$date" : "<ISO-8601>"}
  • strict_uuid: Se true, o objeto uuid.UUID é codificado para o tipo de modo Strict do MongoDB Extended JSON, Binary. Caso contrário, será codificado como {"$uuid": "<hex>" }. Padrões para Falso.
  • document_class: Documentos BSON retornados por cargas() serão decodificados para uma instância desta classe. Deve ser uma subclasse de collections.MutableMapping. Padrões para ditar.
  • uuid_representation: A representação BSON a ser usada ao codificar e decodificar instâncias de uuid.UUID. O padrão é PYTHON_LEGACY.
  • tz_aware: Se true, o tipo de modo Strict do MongoDB Extended JSON do tipo Date será decodificado para instâncias com reconhecimento de fuso horário de datetime.datetime. Caso contrário, eles serão ingênuos. O padrão é Verdadeiro.
  • tzinfo: Uma subclasse datetime.tzinfo que especifica o fuso horário a partir do qual os objetos datetime devem ser decodificados. Padrões para utc.

Usando o módulo json com manipuladores personalizados #

Se tudo que você precisa é serializar os resultados do mongo em json, é possível usar o módulo json, desde que você defina manipuladores personalizados para lidar com tipos de campos não serializáveis. Uma vantagem é que você tem total poder sobre como codificar campos específicos, como a representação de data e hora.

Aqui está um manipulador que codifica datas usando a representação iso e o id como uma string hexadecimal:

import pymongo
import json 
import datetime
import bson.objectid

def my_handler(x):
    if isinstance(x, datetime.datetime):
        return x.isoformat()
    elif isinstance(x, bson.objectid.ObjectId):
        return str(x)
    else:
        raise TypeError(x)

db = pymongo.MongoClient().samples
record = db.movies.find_one()
# {u'_id': ObjectId('5692a15524de1e0ce2dfcfa3'), u'title': u'Toy Story 4',
#   u'released': datetime.datetime(2010, 6, 18, 4, 0),}

json_record = json.dumps(record, default=my_handler)
# '{"_id": "5692a15524de1e0ce2dfcfa3", "title": "Toy Story 4", 
#    "released": "2010-06-18T04:00:00"}'

Usando python-bsonjs #

python-bsonjs não depende do PyMongo e pode oferecer uma boa melhoria de desempenho em relação ao json_util:

bsonjs é aproximadamente 10-15x mais rápido que o json_util do PyMongo na decodificação de BSON para JSON e na codificação de JSON para BSON.

Observe que:

  • para usar o bsonjs de forma eficaz, é recomendado trabalhar diretamente com RawBSONDocument
  • as datas são codificadas usando a representação LEGACY, ou seja, {"$date": <dateAsMilliseconds>}. Atualmente, não há opções para alterar isso.

Instalação #

pip install python-bsonjs

Uso #

Para aproveitar ao máximo o bsonjs, configure o banco de dados para usar a classe RawBSONDocument. Em seguida, use dumps para converter bytes brutos de bson em json e loads para converter bytes brutos de json em bson:

import pymongo
import bsonjs
from pymongo import MongoClient
from bson.raw_bson import RawBSONDocument

# configure mongo to use the RawBSONDocument representation
db = pymongo.MongoClient(document_class=RawBSONDocument).samples
# convert json to a bson record
json_record = '{"_id": "some id", "title": "Awesome Movie"}' 
raw_bson = bsonjs.loads(json_record)
bson_record = RawBSONDocument(raw_bson)
# insert the record
result = db.movies.insert_one(bson_record)
print(result.acknowledged)

# find some record
bson_record2 = db.movies.find_one()
# convert the record to json
json_record2 = bsonjs.dumps(bson_record2.raw)
print(json_record2)
