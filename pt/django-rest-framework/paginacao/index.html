Paginação é a divisão de grandes conjuntos de dados em páginas separadas e autônomas.

No framework django rest, a paginação permite que o usuário modifique a quantidade de dados em cada página e o estilo pelo qual a divisão é aplicada.

[Introdutório] Configurar estilo de paginação globalmente #

Para definir o estilo de paginação para todo o projeto, você precisa definir o DEFAULT_PAGINATION_CLASS e PAGE_SIZE nas configurações do projeto.

Para fazer isso, vá para settings.py e na variável REST_FRAMEWORK, adicione o seguinte:

REST_FRAMEWORK = {
    'DEFAULT_PAGINATION_CLASS': 
        'rest_framework.pagination.DESIRED_PAGINATION_STYLE',
    'PAGE_SIZE': 100
}

No lugar de DESIRED_PAGINATION_STYLE deve ser colocado um dos seguintes:

  • PageNumberPagination: Aceita um único número page nos parâmetros de consulta da solicitação.

     http://your_api_url/a_table/?page=2

  • LimitOffsetPagination: Aceita um parâmetro limit, que indica o número máximo de itens que serão retornados e um parâmetro offset que indica a posição inicial da consulta em relação ao conjunto de dados. PAGE_SIZE não precisa ser definido para este estilo.

     http://your_api_url/a_table/?limit=50&offset=100

  • CursorPagination: A paginação baseada em cursor é mais complexa do que os estilos acima. Requer que o conjunto de dados apresente uma ordenação fixa e não permita que o cliente navegue em posições arbitrárias do conjunto de dados.

  • Estilos de paginação personalizados podem ser definidos no lugar do acima.

[Avançado] Paginação em visualizações/conjuntos de visualizações não genéricos #

Este é um assunto avançado, não tente sem antes entender os outros exemplos desta página.

Conforme declarado no oficial Django Rest Framework na paginação:

A paginação só é executada automaticamente se você estiver usando as visualizações ou conjuntos de visualizações genéricos. Se você estiver usando um APIView normal, precisará chamar a API de paginação por conta própria para garantir o retorno de uma resposta paginada. Consulte o código-fonte das classes mixins.ListModelMixin e generics.GenericAPIView para obter um exemplo.

Mas e se quisermos usar a paginação em uma view/viewset não genérica?

Bem, vamos descer a toca do coelho:

  1. A primeira parada é o repositório oficial do Django Rest Framework e especificamente o django-rest-framework/rest_framework/generics.py. A linha específica para a qual este link está apontando nos mostra como os desenvolvedores do framework lidam com a paginação em seus genéricos.
    Isso é exatamente o que vamos usar para a nossa visão também!

  2. Vamos supor que temos uma configuração de paginação global como a mostrada no [exemplo introdutório desta página][3] e vamos supor também que temos um APIView ao qual queremos aplicar a paginação.

  3. Em seguida, em views.py:

     from django.conf import settings
 from rest_framework.views import APIView

 class MyView(APIView):
     queryset = OurModel.objects.all()
     serializer_class = OurModelSerializer
     pagination_class = settings.DEFAULT_PAGINATION_CLASS # cool trick right? :)

     # We need to override get method to achieve pagination
     def get(self, request):
         ...
         page = self.paginate_queryset(self.queryset)
         if page is not None:
             serializer = self.serializer_class(page, many=True)
             return self.get_paginated_response(serializer.data)

         ... Do other stuff needed (out of scope of pagination)

     # Now add the pagination handlers taken from 
     #  django-rest-framework/rest_framework/generics.py

     @property
     def paginator(self):
         """
         The paginator instance associated with the view, or `None`.
         """
          if not hasattr(self, '_paginator'):
              if self.pagination_class is None:
                  self._paginator = None
              else:
                  self._paginator = self.pagination_class()
          return self._paginator

      def paginate_queryset(self, queryset):
          """
          Return a single page of results, or `None` if pagination is disabled.
          """
          if self.paginator is None:
              return None
          return self.paginator.paginate_queryset(queryset, self.request, view=self)

      def get_paginated_response(self, data):
          """
          Return a paginated style `Response` object for the given output data.
          """
          assert self.paginator is not None
          return self.paginator.get_paginated_response(data)

Agora temos um APIView que trata da paginação!

[Intermediário] Substituir estilo de paginação e configurar paginação por classe #

Substituir estilo de paginação:

Cada estilo de paginação disponível pode ser substituído criando uma nova classe que herda de um dos estilos disponíveis e altera seus parâmetros:

class MyPagination(PageNumberPagination):
    page_size = 20
    page_size_query_param = 'page_size' 
    max_page_size = 200
    last_page_strings = ('the_end',)

Esses parâmetros (conforme listados na documentação oficial da paginação) são:

PageNumberPagination

  • page_size: Um valor numérico que indica o tamanho da página. Se definido, isso substitui a configuração PAGE_SIZE. O padrão é o mesmo valor da chave de configurações PAGE_SIZE.
  • page_query_param: Um valor de string indicando o nome do parâmetro de consulta a ser usado para o controle de paginação.
  • page_size_query_param: Se definido, este é um valor de string que indica o nome de um parâmetro de consulta que permite ao cliente definir o tamanho da página por solicitação. O padrão é “Nenhum”, indicando que o cliente não pode controlar o tamanho da página solicitada.
  • max_page_size: Se definido, este é um valor numérico que indica o tamanho máximo permitido da página solicitada. Este atributo só é válido se page_size_query_param também estiver definido.
  • last_page_strings: Uma lista ou tupla de valores de string indicando valores que podem ser usados ​​com o page_query_param para solicitar a página final no conjunto. O padrão é ('último',)
  • template: O nome de um modelo a ser usado ao renderizar controles de paginação na API navegável. Pode ser substituído para modificar o estilo de renderização ou definido como “Nenhum” para desabilitar completamente os controles de paginação HTML. O padrão é "rest_framework/pagination/numbers.html".

LimitOffsetPagination

  • default_limit: Um valor numérico que indica o limite a ser usado se um não for fornecido pelo cliente em um parâmetro de consulta. O padrão é o mesmo valor da chave de configurações PAGE_SIZE.
  • limit_query_param: Um valor de string indicando o nome do parâmetro de consulta “limit”. O padrão é 'limite'.
  • offset_query_param: Um valor de string indicando o nome do parâmetro de consulta “offset”. O padrão é 'offset'.
  • max_limit: Se definido, este é um valor numérico que indica o limite máximo permitido que pode ser solicitado pelo cliente. O padrão é “Nenhum”.
  • template: O mesmo que PageNumberPagination.

CursorPaginação

  • page_size: O mesmo que PageNumberPagination.
  • cursor_query_param: Um valor de string indicando o nome do parâmetro de consulta “cursor”. O padrão é 'cursor'.
  • ordering: Deve ser uma string, ou lista de strings, indicando o campo no qual a paginação baseada em cursor será aplicada. Por exemplo: ordering = 'slug'. O padrão é -created. Esse valor também pode ser substituído usando OrderingFilter na exibição.
  • template: O mesmo que PageNumberPagination.

Configurar paginação por turma:

Além da capacidade de configurar o estilo de paginação globalmente, uma configuração por classe está disponível:

class MyViewSet(viewsets.GenericViewSet):
    pagination_class = LimitOffsetPagination

Agora, apenas MyViewSet tem uma paginação LimitOffsetPagination.

Um estilo de paginação personalizado pode ser usado da mesma maneira:

class MyViewSet(viewsets.GenericViewSet):
    pagination_class = MyPagination

[Intermediário] Exemplo de uso complexo #

Vamos supor que temos uma API complexa, com muitas visualizações genéricas e alguns conjuntos de visualizações genéricos. Queremos habilitar PageNumberPagination para todas as visualizações, exceto uma (seja visão genérica ou conjunto de visualizações, não faz diferença) para a qual queremos um caso personalizado de LimitOffsetPagination.

Para conseguir isso precisamos:

  1. Em settings.py colocaremos nossa paginação padrão para habilitá-la para cada view/viewset genérico e definiremos PAGE_SIZE para 50 itens:

     REST_FRAMEWORK = {
     'DEFAULT_PAGINATION_CLASS': 
         'rest_framework.pagination.PageNumberPagination',
     'PAGE_SIZE': 50
 }

  2. Agora em nosso views.py (ou em outro .py ex: paginations.py), precisamos substituir o LimitOffsetPagination:

     from rest_framework.pagination import LimitOffsetPagination

 class MyOffsetPagination(LimitOffsetPagination):
     default_limit = 20
     max_limit = 1000

    A custom LimitOffsetPagination with PAGE_ZISE of 20 items and maximum limit of 1000 items.

  3. Em nosso views.py, precisamos definir a pagination_class de nossa view especial:

     imports ...

 # ===================================
 #    PageNumberPagination classes
 # ===================================

 class FirstView(generics.ListAPIView):
     ...

 class FirstViewSet(viewsets.GenericViewSet):
     ...

 ...

 # ===================================
 #     Our custom Pagination class
 # ===================================

 class IAmSpecialView(generics.ListAPIView):
     pagination_class = MyOffsetPagination
     ...

Agora, cada view/viewset genérico no aplicativo tem PageNumberPagination, exceto a classe IAmSpecial, que é de fato especial e tem seu próprio LimitOffsetPagination personalizado.

[Intermediário] Paginação em uma visão baseada em função #

Vimos nesses exemplos (ex_1, ex_2) como usar e substituir as classes de paginação em qualquer visualização genérica de base de classe.
O que acontece quando queremos usar a paginação em uma visualização baseada em função?

Vamos supor também que queremos criar uma visão baseada em função para MyModel com PageNumberPagination, respondendo apenas a uma solicitação GET. Então:

from rest_framework.pagination import PageNumberPagination


@api_view(['GET',])
def my_function_based_list_view(request):
    paginator = PageNumberPagination()
    query_set = MyModel.objects.all()
    context = paginator.paginate_queryset(query_set, request)
    serializer = MyModelSerializer(context, many=True)
    return paginator.get_paginated_response(serializer.data)
Também podemos fazer o acima para uma paginação personalizada alterando esta linha: 
paginator = PageNumberPagination()

para isso

paginator = MyCustomPagination()

desde que tenhamos definido MyCustomPagination para substituir alguma paginação padrão
