Configurando a autenticação globalmente #

A autenticação no Django REST Framework pode ser configurada globalmente como uma subchave da variável REST_FRAMEWORK em settings.py, assim como o restante das configurações padrão do framework.

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework.authentication.BasicAuthentication',
        'rest_framework.authentication.SessionAuthentication',
    )
}

Configurando a autenticação para um APIView específico #

A autenticação pode ser definida para um endpoint APIView específico, usando a variável authentication_classes:

from rest_framework.authentication import SessionAuthentication, BasicAuthentication
from rest_framework.permissions import IsAuthenticated
from rest_framework.response import Response
from rest_framework.views import APIView

class ExampleView(APIView):
    authentication_classes = (SessionAuthentication, BasicAuthentication)
    permission_classes = (IsAuthenticated,)

    def get(self, request):
        content = {
            'user': unicode(request.user),  # `django.contrib.auth.User` instance.
            'auth': unicode(request.auth),  # None
        }
        return Response(content)

Usando autenticação básica baseada em token #

O Django REST Framework fornece um mecanismo básico de autenticação baseado em token que precisa ser configurado como um aplicativo no Django antes de ser usado, para que os tokens sejam criados no banco de dados e seu ciclo de vida seja tratado.

Adicione autenticação baseada em token ao settings.py #

INSTALLED_APPS = (
    ...
    'rest_framework.authtoken'
)

#Executa a migração do banco de dados

./manage.py migrate

Crie tokens para seus usuários #

De alguma forma, você terá que criar um token e devolvê-lo:

def some_api(request):
    token = Token.objects.create(user=request.user)
    return Response({'token': token.key})

Já existe um endpoint de API no aplicativo de token, para que você possa simplesmente adicionar o seguinte ao seu urls.py:

##urls.py

from rest_framework.authtoken import views
urlpatterns += [
    url(r'^auth-token/', views.obtain_auth_token)
]

Clientes agora podem autenticar #

Usando um cabeçalho Authorization como:

Authorization: Token 123456789

Prefixado por um “Token” literal e o próprio token após o espaço em branco.

O literal pode ser alterado criando uma subclasse de TokenAuthentication e alterando a variável de classe keyword.

Se autenticado, request.auth conterá a instância rest_framework.authtoken.models.BasicToken.

Configurando a autenticação OAuth2 #

OAuth não é tratado pelo Django REST Framework, mas existem alguns módulos pip que implementam um cliente OAuth. A documentação do REST Framework sugere um dos seguintes módulos:

  • Django OAuth Toolkit
  • Django REST Framework OAuth

Django OAuth Toolkit #

pip install django-oauth-toolkit

configurações.py #

INSTALLED_APPS = (
    ...
    'oauth2_provider',
)

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'oauth2_provider.ext.rest_framework.OAuth2Authentication',
    )
}

##urls.py

urlpatterns = patterns(
    ...
    url(r'^o/', include('oauth2_provider.urls', namespace='oauth2_provider')),
)

Django REST Framework OAuth #

pip install djangorestframework-oauth django-oauth2-provider

configurações.py #

INSTALLED_APPS = (
    ...
    'provider',
    'provider.oauth2',
)

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework.authentication.OAuth2Authentication',
    )
}

##urls.py

urlpatterns = patterns(
    ...
    url(r'^oauth2/', include('provider.oauth2.urls', namespace='oauth2')),
)

Administrador #

Vá para o painel de administração e crie um novo Provider.Client para ter um client_id e um client_secret.

Usando melhor autorização baseada em token com vários tokens de cliente #

O pacote mais interessante para gerenciar tokens reais é o django-rest-knox que suporta vários tokens por usuário (e cancelando cada token independentemente), bem como tendo suporte para expiração de token e vários outros mecanismos de segurança.

django-rest-knox depende da criptografia. Você pode encontrar mais informações sobre como instalá-lo em: http://james1345.github.io/django-rest-knox/installation/

#Instalando o knox

pip install django-rest-knox

configurações.py #

INSTALLED_APPS = (
  ...
  'rest_framework',
  'knox',
  ...
)

Aplicar migrações:

./manage.py migrate
