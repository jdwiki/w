Exemplos de filtragem, dos mais simples aos mais complexos #

Filtragem Simples de Baunilha #

Para filtrar qualquer visualização, substitua seu método get_queryset para retornar um conjunto de consultas filtradas

class HREmployees(generics.ListAPIView):
    def get_queryset(self):
        return Employee.objects.filter(department="Human Resources")

Todas as funções da API usarão o conjunto de consultas filtradas para operações. Por exemplo, a listagem da visualização acima conterá apenas funcionários cujo departamento seja Recursos Humanos.

Acessando parâmetros de consulta em get_queryset #

A filtragem com base em parâmetros de solicitação é fácil.

self.request, self.args e self.kwargs estão disponíveis e apontam para a solicitação atual e seus parâmetros para filtragem

def DepartmentEmployees(generics.ListAPIView):
    def get_queryset(self):
        return Employee.objects.filter(department=self.kwargs["department"])

Deixar os parâmetros da API decidirem o que filtrar #

Se você quiser mais flexibilidade e permitir que a chamada da API passe parâmetros para filtrar a visualização, você pode plugar back-ends de filtro como o Django Request Framework (instalado via pip)

from rest_framework import filters  

class Employees(generics.ListAPIView):
    queryset=Employee.objects.all()
    filter_backends = (filters.DjangoFilterBackend,)
    filter_fields = ("department", "role",)

Agora você pode fazer uma chamada de API /api/employees?department=Human Resources e obterá uma lista de funcionários que pertencem apenas ao departamento de RH, ou /api/employees?role=manager&department=Human Resources para obter apenas gerentes no departamento de RH.

Você pode combinar a filtragem do conjunto de consultas com o Django Filter Backend, sem problemas. Os filtros funcionarão no conjunto de consultas filtradas retornado por get_queryset

from rest_framework import filters  

class HREmployees(generics.ListAPIView):
    filter_backends = (filters.DjangoFilterBackend,)
    filter_fields = ("department", "role",)

    def get_queryset(self):
        return Employee.objects.filter(department="Human Resources")

Conjuntos de Filtros #

Até agora, você pode se virar com correspondências de tipo simples nos casos acima.

Mas e se você quiser algo mais complexo, como uma lista de funcionários de RH com idade entre 25 e 32 anos?

Resposta ao problema: Conjuntos de filtros

Conjuntos de filtros são classes que definem como filtrar vários campos do modelo.

Defina-os assim

class EmployeeFilter(django_filters.rest_framework.FilterSet):
    min_age = filters.django_filters.NumberFilter(name="age", lookup_expr='gte')
    max_age = filters.django_filters.NumberFilter(name="price", lookup_expr='lte')     

    class Meta:
        model = Employee
        fields = ['age', 'department']

name aponta para o campo que você deseja filtrar

lookup_expr basicamente se refere aos mesmos nomes que você usa ao filtrar conjuntos de consultas, por exemplo, você pode fazer uma correspondência “começa com” usando lookup_expr="startswith" que é equivalente a Employee.objects.filter(department__startswith="Human ")

Em seguida, use-os em suas classes de visualização usando filter_class em vez de filter_fields

class Employees(generics.ListAPIView):
    queryset=Employee.objects.all()
    filter_backends = (filters.DjangoFilterBackend,)
    filter_class = EmployeeFilter

Agora você pode fazer /api/employees?department=Human Resources&min_age=25&max_age=32

Correspondências e relacionamentos não exatos #

As classes e expressões de filtro são muito semelhantes à forma como você especifica a filtragem em conjuntos de consultas

Você pode usar a notação “__” para filtrar campos em relacionamentos, por exemplo, se o departamento era uma chave estrangeira do funcionário, você pode adicionar

filter_fields=("department__name",)

e então você pode fazer /api/employees?department__name=Human Resources

Ou mais elegantemente, você pode criar um conjunto de filtros, adicionar uma variável de filtro chamada dept e definir seu nome como department__name, permitindo que você faça /api/employees?dept=Human Resources
