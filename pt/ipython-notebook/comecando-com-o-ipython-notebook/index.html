Instalação ou configuração #

Instruções detalhadas sobre como configurar ou instalar o ipython-notebook:

Observação: iPython Notebook agora não é mais suportado, pois todas as funcionalidades foram movidas para o projeto Jupyter.

Pré-requisito #

Para instalar o Jupyter ou iPython Notebook você deve ter o python instalado pelo menos na versão 2.7.3+ ou 3.3+ - o python pode ser instalado a partir do site principal do python, da sua distribuição Linux/OS-X ou como um dos os pacotes empacotados como Anaconda recomendado.

Instalando o Notebook Jupyter #

Para instalar o Jupyter Notebook:

  • No Anaconda: Verifique se você está executando uma versão recente do Anaconda para Python 3 e já possui o Jupyter e seu Notebook instalado, basta executar com jupyter notebook
  • Usando pip Usuários Linux e OS-X podem precisar prefixar todos os comandos a seguir com sudo:
  • Atualize pip com: python -m pip install --upgrade pip
  • Instale o Jupyter com: pip install --upgrade jupyter
  • Execute com: jupyter notebook para iniciar o servidor e você deverá ver a interface “home” no seu navegador:

Para obter instruções sobre como habilitar kernals adicionais, consulte o site Jupyter.

Nota: Existe atualmente, 2016, uma nova, próxima geração, interface de usuário para Jupyter em desenvolvimento ativo chamada Jupyter Lab que vale a pena assistir:

Sistemas legados #

Em sistemas legados iPython, pode ser possível instalar o notebook iPython em um sistema python com o pip instalado, use o comando:

pip install ipython[notebook]

Toda vez que você executar o ipython notebook, você receberá um aviso:

[TerminalIPythonApp] WARNING | Subcommand `ipython notebook` is deprecated and will be removed in future versions.
[TerminalIPythonApp] WARNING | You likely want to use `jupyter notebook` in the future

e descobrir que você está executando o Jupyter.

Nota sobre a importação de bibliotecas

Em alguns sistemas, principalmente OS-X ou quando executado a partir de um ambiente virtual, o notebook pode não reconhecer algumas das bibliotecas instaladas no sistema subjacente, isso é devido ao python do sistema e que está sendo usado para rodar o notebook sendo instalações diferentes. A adição das bibliotecas pode ser feita independentemente no ambiente de notebooks pelo seguinte comando de dentro do ambiente iPython:

!pip install [desired library]

Ou com o seguinte código:

import pip
pip.main(['install', 'libary-name'])  # '-U' can be added after install to update existing packages
