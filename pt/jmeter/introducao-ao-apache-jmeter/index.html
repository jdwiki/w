Instalação ou Configuração #

  1. Baixe um arquivo distribuído da seção Binaries do JMeter da página Download Apache JMeter.
  2. Dependendo da versão baixada, verifique requisitos mínimos da versão Java e instale o Java, se necessário. Certifique-se de que a variável de ambiente JAVA_HOME esteja configurada e aponte para uma versão correta.
  3. Extraia o arquivo de distribuição no diretório de sua escolha.
  4. Abra a interface do usuário do JMeter:

  • No Windows: navegue até o diretório <jmeter_location>\bin e execute jmeterw.bat ou jmeter.bat

  • No Linux/Mac: navegue até o diretório <jmeter_location>/bin e execute jmeter ou ‘jmeter.sh`.

     For example:

   cd /Users/me/apache-jmeter/bin
   ./jmeter 

 **Note**: if the above command fails with `Permission denied` error, set execute permission on `jmeter` file:

   cd /Users/me/apache-jmeter/bin
   chmod u+x ./jmeter

Se você conseguir ver a interface do usuário do JMeter, a configuração básica foi bem-sucedida.

Visão geral dos componentes Apache JMeter em alto nível #

O Apache JMeter segregou todos os componentes nos seguintes grupos com base em sua funcionalidade:

  1. Plano de Teste: Ponto de partida para a criação de scripts. O JMeter salva o Plano de Teste no formato .jmx. Você adiciona componentes ao Plano de Testes clicando com o botão direito do mouse no Painel de Testes e navegando até o componente que deseja adicionar.
  2. Workbench: É um local temporário para iniciar o script. Junto com todos os componentes disponíveis no Plano de Teste, você obtém o HTTP(s) Test Script Recorder para gravar as ações do navegador. Os scripts podem ser salvos no Workbench desde que você marque a caixa de seleção “Save Workbench”, caso contrário, eles não são.
  3. Threads (Users): você pode definir um número de usuários (virtuais) para executar, tempo de aceleração e contagem de loops. você também pode definir no Plano de Teste se os Grupos de Encadeamento precisam ser executados em sequência ou em paralelo no caso de vários Grupos de Encadeamento. alguns exemplos são Thread Group, setUp Thread Group e tearDown Thread Group
  4. Controlador Lógico: Permite definir o fluxo de execução e agrupamento dos amostradores. um dos exemplos úteis é o Transaction Controller, onde você combina todos os samplers da página Login (todos os recursos, incluindo imagens, arquivos .css e .js) para que o tempo de resposta combinado possa ser recuperado.
  5. Sampler: O Sampler é o núcleo do JMeter. Ele fornece componentes para simular solicitações de vários protocolos como HTTP, JDBC, FTP, SMTP etc. Por exemplo, o amostrador HTTP permite simular um pacote HTTP (de GET, POST ou quaisquer métodos suportados). Protocolos de fluxo principal são suportados, para outros você pode usar plugins gratuitos ou comerciais.
  6. Config Element: Elementos de configuração podem ser usados ​​para configurar padrões e variáveis ​​para uso posterior por samplers. Observe que esses elementos geralmente são processados ​​no início do escopo em que são encontrados, ou seja, antes de qualquer amostrador no mesmo escopo. CSV Dataset Config permite que você forneça dados de teste como nomes de usuário, senhas do cenário de Login a partir de um arquivo. O elemento de configuração User Defined variables permite que você defina variáveis ​​que podem ser usadas em todo o Plano de Teste, mas onde cada Thread tem sua própria cópia.
  7. Timer: Por padrão, um thread JMeter executa samplers em sequência sem pausa. Os componentes apresentados aqui fornecem a funcionalidade de introduzir o User Think Time em várias formas entre os samplers. alguns exemplos são Constant Timer, Constant Throughput Timer.
  8. Pre Processors: permitem que você execute operações/ações antes que o sampler seja executado. JSR223 Pre Processor com Apache Groovy (semelhante ao estilo de codificação java) permite que você faça alterações no sampler antes de enviá-lo.
  9. Post Processors: permite que você execute operações/ações após o sampler ser executado. alguns exemplos úteis são a recuperação de valor dinâmico, como Session ID, usando o pós-processador Regular Expression Extractor para qualquer tipo de texto, CSS/JQuery Extractor para HTML, JSON Extractor para JSON, XPath Extractor para XML.
  10. Assertions: como o nome sugere, você pode afirmar a resposta dos samplers de diferentes maneiras, como pesquisar algum texto, o tamanho da resposta e a duração para receber a resposta, etc. Por exemplo, você pode usar Response Assertion para procurar algum texto na resposta. Se a Asserção falhar, o JMeter marca o amostrador, ao qual a Asserção é aplicada, como Falha.
  11. Ouvintes: Os ouvintes permitem que você salve os resultados do teste, veja a execução do teste, etc. por exemplo, usando ‘Exibir árvore de resultados’, você pode ver a solicitação/resposta dos amostradores e se eles foram marcados como PASSA (cor verde)/FALHA ( cor vermelha) por JMeter. usando o Relatório Agregado, você pode salvar os resultados do teste no formato CSV. Observação importante é que você usa ouvintes antes da execução do teste (para depuração do script de teste) ou após a execução do teste (para exibir os resultados em gráficos ou resumo), mas não durante a execução. devemos remover os ouvintes durante o teste, pois consome muitos recursos do sistema. Assim, executamos o teste no modo não-GUI e salvamos os resultados usando a opção -l nos formatos .csv/.jtl. Poste o teste, você pode carregar esses arquivos salvos em qualquer um dos listeners no JMeter para visualizar gráficos/resumo.
