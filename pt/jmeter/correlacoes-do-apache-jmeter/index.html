No teste de desempenho do JMeter, Correlações significa a capacidade de buscar dados dinâmicos da resposta do servidor e postá-los nas solicitações subsequentes. Esse recurso é fundamental para muitos aspectos do teste, como aplicativos protegidos baseados em token.

Correlação usando o Extrator de Expressão Regular no Apache JMeter #

Se você precisar extrair informações de uma resposta de texto, a maneira mais fácil é usar Expressões Regulares. O padrão de correspondência é muito semelhante ao usado em Perl. Vamos supor que queremos testar um fluxo de trabalho de compra de passagens aéreas. O primeiro passo é submeter a operação de compra. A próxima etapa é garantir que possamos verificar todos os detalhes usando o ID da compra, que deve ser retornado na primeira solicitação. Vamos imaginar que a primeira requisição retorne uma página html com esse tipo de ID que precisamos extrair:

<div class="container">
  <div class="container hero-unit">
    <h1>Thank you for you purchse today!</h1>
       <table class="table">
           <tr>
             <td>Id</td>
             <td>Your purchase id is 1484697832391</td>
           </tr>
           <tr>
             <td>Status</td>
             <td>Pending</td>
           </tr>
           <tr>
             <td>Amount</td>
             <td>120 USD</td>
            </tr>
         </table>
    </div>
</div>

Esse tipo de situação é o melhor candidato para usar o extrator JMeter Regular Expression. Expressão Regular é uma string de texto especial para descrever um padrão de pesquisa. Existem muitos recursos online que ajudam a escrever e testar Expressões Regulares. Um deles é https://regex101.com/.

Para usar este componente, abra o menu JMeter e: Adicionar -> Pós-processadores -> Extrator de Expressão Regular

O Extrator de Expressão Regular contém estes campos:

  • Nome de referência - o nome da variável que pode ser usada após a extração
  • Expressão Regular - uma sequência de símbolos e caracteres que expressam uma string (padrão) que será pesquisada no texto
  • Template - contém referências aos grupos. Como uma regex pode ter mais de um grupo, ela permite especificar qual valor de grupo extrair, especificando o número do grupo como $1$ ou $2$ ou $1$$2$ (extrair os dois grupos)
  • Número de correspondência - especifica qual correspondência será usada (o valor 0 corresponde a valores aleatórios / qualquer número positivo N significa selecionar a enésima correspondência / valor negativo precisa ser usado com o ForEach Controller)
  • Padrão - o valor padrão que será armazenado na variável caso nenhuma correspondência seja encontrada, é armazenado na variável.

A caixa de seleção “Aplicar a” lida com amostras que fazem solicitações de recursos incorporados. Este parâmetro define se a Expressão Regular será aplicada aos resultados da amostra principal ou a todas as solicitações, incluindo recursos incorporados. Existem várias opções para este parâmetro:

  • Amostra principal e subamostras
  • Apenas amostra principal
  • Apenas subamostras
  • Variável JMeter - a asserção é aplicada ao conteúdo da variável nomeada, que pode ser preenchida por outra solicitação

A caixa de seleção “Campo a ser verificado” permite escolher em qual campo a Expressão Regular deve ser aplicada. Quase todos os parâmetros são autodescritivos:

  • Corpo - o corpo da resposta, por exemplo o conteúdo de uma página da web (excluindo cabeçalhos)
  • Corpo (sem escape) - o corpo da resposta, com todos os códigos de escape HTML substituídos. Observe que os escapes HTML são processados ​​sem levar em conta o contexto, portanto, algumas substituições incorretas podem ser feitas (*esta opção afeta muito o desempenho)
  • Body - Body as a Document - o texto extraído de vários tipos de documentos via Apache Tika (*também pode afetar o desempenho)
  • Corpo - Cabeçalhos de solicitação - podem não estar presentes para amostras não HTTP
  • Corpo - Cabeçalhos de resposta - podem não estar presentes para amostras não HTTP
  • Corpo - URL
  • Código de resposta - ex. 200
  • Corpo - Mensagem de Resposta - ex. OK

Depois que a expressão é extraída, ela pode ser usada em solicitações subsequentes usando a variável ${purchaseId}.

Correlação usando o extrator XPath no JMeter #

XPath pode ser usado para navegar pelos elementos e atributos em um documento XML. Pode ser útil quando os dados da resposta não podem ser extraídos usando o Extrator de Expressão Regular. Por exemplo, no caso de um cenário onde você precisa extrair dados de tags semelhantes com os mesmos atributos, mas com valores diferentes. O XPath Extractor é semelhante ao CSS/JQuery Extractor, mas o XPath Extractor deve ser usado para conteúdo XML, enquanto o CSS/JQuery Extractor deve ser usado para conteúdo HTML. Vamos supor que na resposta temos uma tabela com valores diferentes onde precisamos extrair valor da segunda linha da tabela.

<div id="weeklyPrices">
    <tr>
        <td>$56.00</td>
        <td>$56.00</td>
        <td>$56.00</td>
        <td>$56.00</td>
        <td>$60.00</td>
        <td>$70.00</td>
        <td>$70.00</td>
    </tr>
</div>

Olhando para o futuro, o XPath certo para esse caso será: //div[@id=‘weeklyPrices’]/tr/td1

Para usar este componente, abra o menu JMeter e: Adicionar -> Pós-processadores -> Extrator XPath

O XPath Extractor contém vários elementos de configuração comuns que são mencionados em ‘Correlation using Regular Expression Extractor’. Isso inclui Nome, Aplicar a, Nome de referência, Nº de correspondência (desde JMeter 3.2) e Valor padrão.

Existem muitos recursos da web com folhas de dicas e editores online para criar e testar seu xpath criado (como [este aqui] 3). Mas com base nos exemplos abaixo, podemos encontrar a maneira de criar os localizadores xpath mais comuns.

Se você deseja analisar HTML em XHTML, precisamos marcar a opção “Use Tidy”. Depois de decidir sobre o status “Use Tidy”, também existem opções adicionais:

Se ‘Usar arrumado’ estiver marcado:

  • Silencioso - define a bandeira Tidy Quiet
  • Reportar erros - se ocorrer um erro Tidy, defina a Asserção de acordo
  • Mostrar avisos - define a opção de avisos de exibição organizada

Se ‘Usar arrumado’ estiver desmarcado:

  • Use Namespaces - se marcado, o analisador XML usará o namespace resolução
  • Validar XML - verificar o documento em relação ao esquema especificado
  • Ignore Whitespace - ignore Element Whitespace
  • Buscar DTDs Externos - se selecionado, os DTDs externos são buscados

‘Retornar fragmento XPath inteiro em vez de conteúdo de texto’ é auto-descritivo e deve ser usado se você quiser retornar não apenas o valor xpath, mas também o valor dentro de seu localizador xpath. Pode ser útil para necessidades de depuração.

Também vale a pena mencionar que há uma lista de plugins de navegador muito convenientes para testar localizadores XPath. Para o Firefox, você pode usar o plug-in ‘Firebug’ enquanto para o Chrome o ‘XPath Helper’ é a ferramenta mais conveniente.

Correlação usando o CSS/JQuery Extractor no JMeter #

O extrator CSS/JQuery permite extrair valores de uma resposta do servidor usando uma sintaxe de seletor CSS/JQuery, que poderia ser difícil de escrever usando Expressão Regular. Como um pós-processador, esse elemento deve ser executado para extrair os nós solicitados, texto ou valores de atributo de um amostrador de solicitação e armazenar o resultado na variável fornecida. Este componente é muito semelhante ao XPath Extractor. A escolha entre CSS, JQuery ou XPath geralmente depende da preferência do usuário, mas vale ressaltar que XPath ou JQuery podem percorrer o DOM para baixo e também para cima, enquanto o CSS não pode subir o DOM. Vamos supor que queremos extrair todos os tópicos da documentação do Stack Overflow relacionados ao Java. Você pode usar o plug-in Firebug para testar seus seletores CSS/JQuery no Firefox, ou o CSS Selector Tester no Chrome.

Para usar este componente, abra o menu JMeter e: Adicionar -> Pós-processadores -> CSS/JQuery Extractor

Quase todos os campos desse extrator são semelhantes aos campos do extrator de Expressão Regular, portanto, você pode obter a descrição desse exemplo. Uma diferença, no entanto, é o campo “Implementação do extrator CSS/JQuery”. Desde o JMeter 2.9 você pode usar o extrator CSS/JQuery baseado em duas implementações diferentes: a implementação jsoup (descrição detalhada de sua sintaxe aqui) ou a JODD Lagarto (sintaxe detalhada pode ser encontrado aqui). Ambas as implementações são quase as mesmas e têm apenas pequenas diferenças de sintaxe. A escolha entre eles é baseada na preferência do usuário.

Com base na configuração mencionada acima, podemos extrair todos os tópicos da página solicitada e verificar os resultados extraídos usando o “Debug Sampler” e o listener “View Results Tree”.

Correlação usando o extrator JSON #

JSON é um formato de dados comumente usado em aplicativos baseados na web. O JMeter JSON Extractor fornece uma maneira de usar expressões de caminho JSON para extrair valores de respostas baseadas em JSON no JMeter. Este pós-processador deve ser colocado como filho do Amostrador HTTP ou para qualquer outro amostrador que tenha respostas.

Para usar este componente, abra o menu JMeter e: Adicionar -> Pós-processadores -> Extrator JSON.

O Extrator JSON é muito semelhante ao Extrator de Expressão Regular. Quase todos os campos principais são mencionados nesse exemplo. Existe apenas um parâmetro JSON Extractor específico: ‘Compute concatenation var’. Caso sejam encontrados muitos resultados, este extrator irá concatená-los usando o separador ‘,’ e armazenando-o em uma var chamada <nome da variável>_ALL.

Correlação automatizada usando o ‘SmartJMX’ do BlazeMeter #

Ao escrever manualmente seus scripts de desempenho, você mesmo precisa lidar com a correlação. Mas há outra opção para criar seus scripts - gravação de scripts de automação. Por um lado, a abordagem manual ajuda a escrever scripts estruturados e você pode adicionar todos os extratores necessários ao mesmo tempo. Por outro lado, esta abordagem é muito demorada.

A gravação de scripts de automação é muito fácil e permite que você faça o mesmo trabalho, só que muito mais rápido. Mas se você usar formas de gravação comuns, os scripts serão muito desestruturados e geralmente exigirão a adição de parametrização adicional. O recurso “Smart JMX” no gravador Blazemeter combina as vantagens de ambas as formas. Ele pode ser encontrado neste link: https://a.blazemeter.com/app/recorder/index.html

Após o registro, vá para a seção “Gravador”.

Para iniciar a gravação do script, primeiro você precisa configurar o proxy do seu navegador (abordado aqui), mas desta vez você deve obter um host proxy e uma porta fornecida pelo gravador BlazeMeter.

Quando o navegador estiver configurado, você poderá prosseguir com a gravação do script pressionando o botão vermelho na parte inferior. Agora você pode acessar o aplicativo em teste e executar fluxos de trabalho do usuário para gravação.

Após a gravação do script, você pode exportar os resultados para um arquivo JMX “SMART”. Um arquivo jmx exportado contém uma lista de opções que permitem configurar seu script e parametrizar, sem esforços adicionais. Uma dessas melhorias é que o JMX “SMART” encontra automaticamente candidatos de correlação, substitui-o pelo extrator apropriado e fornece uma maneira fácil de parametrização adicional.
