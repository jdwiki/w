A gravação de cenários de teste é uma das maneiras mais convenientes de criar scripts de teste. Isso ocorre porque as gravações de teste permitem imitar fluxos de trabalho de usuário realistas, em vez de ter que criar manualmente um script de teste. As gravações capturam todas as solicitações do navegador para o aplicativo da web e, em seguida, criam automaticamente um arquivo jmx que pode ser executado em testes de desempenho. Ao usar a funcionalidade de gravação/reprodução do JMeter ou ferramentas de terceiros como BlazeMeter e BadBoy, os testadores podem tornar seu trabalho 3 vezes mais rápido.

Gravação de script com o JMeter Proxy Recorder #

O JMeter também permite configurar manualmente sua área de trabalho. É mais complexo, mas você pode ajustar os scripts às suas necessidades exatas.

Configure o proxy do seu navegador

  1. Configure seu navegador, conforme descrito no capítulo 1.

Configurar JMeter

A ramificação “WorkBench” pode ser usada como um espaço de trabalho temporário para a criação de scripts. Tenha em mente que as entradas adicionadas a esta seção não serão salvas com o plano de teste. Portanto, se você quiser reutilizar a mesma configuração de gravação no futuro, precisará copiá-la e colá-la na seção “Plano de teste”.

  1. Adicione “Controlador de Gravação” ao “WorkBench”: clique com o botão direito do mouse em “WorkBench” -> “Adicionar” -> “Controlador Lógico” -> “Controlador de Gravação”

  1. Adicione “HTTP(S) Test Script Recorder” ao mesmo “WorkBench”: Clique com o botão direito em “WorkBench” -> “Add” -> “Non-Test Elements” -> “HTTP(S) Test Script Recorder”

  1. Na página de configuração “HTTP(S) Test Script Recorder” em “Global Settings: Port”, você precisa colocar a mesma porta especificada na configuração de proxy do seu navegador, por exemplo 8080.

  2. Se você deseja excluir solicitações para recursos específicos, pode usar a seção “Padrões de URL a serem excluídos”. Isso pode ser útil se você quiser incluir apenas os tipos de conteúdo que deseja solicitar (por exemplo, *.html, *.php, etc) ou excluir os tipos de conteúdo que não deseja solicitar (por exemplo, *.jpg, * .png, *.js, etc).

Quando usaríamos isso? Por exemplo, ao gravar um script que chama um aplicativo de terceiros ou ao testar um script do lado do servidor, talvez você não queira fazer download de ativos, pois eles podem sobrecarregar seus testes e consumir largura de banda, ou talvez você queira gravar determinadas solicitações de um determinado caminho.

Os padrões de exclusão mais comuns são: “..png“, “..jpg“, “..gif“, “..css”, “..js”. Você também pode combinar diferentes padrões juntos. Este padrão combinado deve se livrar de todas as solicitações redundantes que podem distraí-lo de outras importantes: “..(bmp|css|js|gif|ico|jpe?g|png|swf|woff)”

  1. Por outro lado, você pode querer imitar o comportamento completo do navegador e incluir o carregamento de todos os recursos. Nesse caso, não é necessário excluir padrões de URL. Tenha em mente que o navegador baixa todos os recursos incorporados da página solicitada e que possui um mecanismo de cache, o que pode afetar o desempenho do resultado.

Nesse caso, é recomendável baixar todos os recursos incorporados no script: Clique com o botão direito do mouse em “Plano de teste” -> “Adicionar” -> “Elemento de configuração” -> “Padrões de solicitação HTTP” -> “Avançado” -> marque a caixa de seleção “Recuperar todos os recursos incorporados”.

  1. Para que o JMeter se comporte mais como um navegador real, é recomendado adicionar o “HTTP Cache Manager”, que permite simular a funcionalidade de cache do navegador em seus testes de desempenho. Clique com o botão direito do mouse em “Plano de teste” -> “Adicionar” -> “Config Element” -> “HTTP Cache Manager”.

  2. Agora clique no botão ‘Iniciar’, que está na parte inferior da página “HTTP(S) Test Script Recorder”, e percorra o fluxo de trabalho do aplicativo da Web que você deseja testar. Quando você voltar ao JMeter, deverá ver todas as solicitações capturadas do seu navegador no “Controlador de Gravação”.

Gravação de script com o recurso de modelo JMeter #

Na versão 2.10, JMeter introduziu um mecanismo que economiza seu tempo na criação de scripts - JMeter Templates. Esses modelos são esqueletos que podem ser reutilizados como base para seus novos scripts.

O JMeter já possui vários templates disponíveis com descrições detalhadas, e você também pode adicionar os seus próprios. Os modelos têm toda a configuração e elementos necessários para gravar seus scripts de desempenho do zero.

Veja como usar o recurso de modelo JMeter:

Configurar JMeter

  1. Abra o JMeter

  2. Selecione o modelo para gravação de script: Arquivo -> Modelos… -> Selecionar Modelo -> Gravação -> Criar O JMeter adicionará os elementos relevantes à árvore de teste.

Configure o proxy do seu navegador

Para usar o JMeter Recorder, você precisa configurar seu navegador para enviar todas as solicitações via proxy. Qualquer navegador pode ser usado para essas necessidades, embora possa haver diferenças entre as localizações das configurações dos navegadores, que são específicas do navegador e podem variar de acordo com o sistema operacional.

  1. Para configurar seu navegador:

    Chrome: Menu button -> Settings -> Show advanced settings… -> Network -> Change proxy settings

    Safari: Preferences -> Advanced -> Proxies -> Change Settings…

    Firefox: Menu button -> Preferences -> Advanced -> Network -> Connection -> Settings..

  2. Por exemplo, você pode usar o localhost 127.0.0.1. Altere a porta para a porta no HTTP(S) Script Recorder.

Se você tiver problemas ao concluir esta etapa, verifique se não possui plug-ins de terceiros que possam gerenciar as configurações de proxy do seu navegador. Se você fizer isso, como o Hola VPN, por exemplo, as configurações de proxy não estarão disponíveis no menu do seu navegador.

  1. Clique no botão ‘Iniciar’, que está na parte inferior da página “HTTP(S) Test Script Recorder” e percorra o fluxo de trabalho do aplicativo da Web que deseja testar. Quando você voltar ao JMeter, deverá ver todas as solicitações capturadas do seu navegador.

Gravação de scripts de desempenho para dispositivos móveis #

O JMeter também pode ser usado para gravar testes de desempenho móvel. A gravação de scripts móveis é muito semelhante à gravação de scripts de aplicativos da web.

Configurar JMeter

  1. Configure “Modelos JMeter” conforme especificado no capítulo 1.

Configure seu celular

Após a configuração do JMeter ser preparada, incluindo o elemento JMeter “HTTP(S) Test Script Recording” iniciado em uma porta especificada, você pode configurar seu telefone celular para enviar uma solicitação ao aplicativo da Web que está testando por meio do proxy JMeter.

  1. IOS:

  • Configuração -> Wi-Fi

  • Clique na rede conectada

  • Vá para a seção de configuração “HTTP PROXY”

  • Clique na aba “Manual”

  • Defina o IP do computador em que o aplicativo JMeter está sendo executado em “Servidor”

  • Defina a porta especificada em “HTTP(S) Test Script Recording” em “Porta”

    Android:

  • Configuração -> Wi-Fi

  • Dê um clique longo na rede conectada e clique na opção ‘Modificar rede’

  • Clique na caixa de seleção “Opções avançadas”

  • Defina a opção “Proxy” para “Manual”

  • Defina o “Proxy hostname” como o endereço IP do seu computador e “Proxy Port” conforme especificado na configuração “HTTP(S) Test Script Recording” em “Port”

  • Clique em “Salvar”

  1. Agora você pode começar a executar o aplicativo em seu dispositivo móvel. As solicitações serão registradas no JMeter.

Gravando tráfego HTTPS #

Se seu aplicativo da web usa a criptografia SSL, você precisa capturar o tráfego HTTPS em vez de HTTP. Para registrar o tráfego HTTPS com JMeter, você precisa configurar os certificados SSL.

Configure seu proxy SSL

  1. Certifique-se de que o proxy SSL esteja configurado da mesma forma que o proxy HTTP:

Configurar JMeter 2. Inicie a gravação de script usando o recurso “JMeter Recording Template” conforme explicado no exemplo “Gravação de script com o recurso JMeter Template”.

  1. Após abrir o aplicativo da web, você verá uma mensagem referente a uma conexão não segura. Para prosseguir, você só precisa aceitar o certificado fictício JMeter:
  • Clique em ‘Avançado’
  • Clique em ‘Adicionar exceção…’
  • Desmarque ‘Armazenar permanentemente esta exceção’
  • Clique em ‘Confirmar exceção de segurança’

  1. Se você vir a mensagem “Este site fornece identificação válida e verificada. Não há necessidade de adicionar uma exceção.” mensagem de aviso, você precisa limpar o histórico do navegador do seu aplicativo, incluindo cookies, cache, dados do site offline. Em seguida, prossiga com os mesmos passos novamente.

Essa abordagem também funciona para gravação de scripts móveis, pois o certificado JMeter precisa ser instalado apenas no host que está sendo usado para executar o JMeter.

Gravação de script com a extensão BlazeMeter Chrome #

Até agora, abordamos as formas básicas de registrar cenários de teste. Mas uma das maneiras mais rápidas e fáceis de gravar seus scripts de performance, que também é gratuita, é usar a extensão BlazeMeter Recorder Chrome. Essas gravações podem ser executadas no JMeter ou no BlazeMeter.

A razão pela qual a extensão é tão útil é que ela permite gravar scripts de desempenho do seu navegador sem precisar configurar seu proxy.

Para criar um novo script de desempenho:

  1. Abra o gravador do seu Chrome
  2. Digite um nome de teste no campo superior
  3. Inicie a gravação clicando no botão gravar, em forma de círculo, e execute as ações da web que deseja gravar. Todos os seus pedidos serão capturados. A extensão Blazemeter Chrome também suporta a gravação de tráfego HTTPS.
  4. Após finalizar a gravação, clique no botão Parar, em formato de quadrado. Você também pode pausar sua gravação e depois retomá-la, bem como editá-la, no formato .jmx ou JSON, ou na nuvem.
  5. Exporte sua gravação - para executar o teste no JMeter, exporte para o formato .jmx clicando no botão .jmx. Para executar o teste no BlazeMeter, clique em ‘play’.

Para mais informações veja aqui.

Gravação de script com BadBoy #

Outra ferramenta útil de gravação de terceiros é o BadBoy. No entanto, ele funciona apenas para o sistema operacional Windows.

Para criar um novo script de desempenho:

  1. Instale o BadBoy aqui

  2. Insira o URL em teste na barra de endereço

  3. Pressione o botão de gravação, em forma de círculo vermelho e execute as ações que deseja capturar.

  4. Exporte seu script para JMeter - Arquivo -> Exportar para JMeter

Para obter mais informações, consulte aqui.

Usar um gravador de scripts de desempenho é uma ótima maneira de evitar tarefas rotineiras e ainda obter os melhores scripts de teste. Após a gravação, configure o teste para o número de usuários virtuais que deseja testar, bem como configurações de teste adicionais, execute seu teste e analise os resultados para identificar erros e gargalos e caracterizar tendências que mostram a integridade do seu sistema.
