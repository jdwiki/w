Crie um Banco de Dados SQL do Windows Azure a partir do Portal #

Como tudo no Windows Azure, você precisa ter uma conta do Windows Azure e uma assinatura do Azure. Depois de ter ambos, vá para https://portal.azure.com. A partir daqui, você pode adicionar novos recursos à sua assinatura do Azure.

Clique em Novo no menu à esquerda. Uma nova folha será adicionada à direita do seu menu. Nesse menu, escolha Bancos de dados e mais uma folha será adicionada ao seu menu.

Escolha Banco de Dados SQL, então as lâminas “Novo” e “Bancos de Dados” serão removidas e uma nova lâmina Banco de Dados SQL será adicionada ao seu menu.

A partir daqui, você deve escolher um nome de banco de dados para dar a este novo banco de dados. Você também precisa selecionar a assinatura à qual adicionar esse banco de dados, se tiver acesso a mais de uma assinatura. É aqui que a fatura do seu banco de dados será registrada.

Em seguida, se você já tiver definido Grupos de recursos para sua assinatura, poderá optar por reutilizar um desses grupos de recursos ou criar um novo. Pense nesses grupos de recursos como coleções de recursos no Windows Azure que compartilham um ciclo de vida comum. Todos os recursos deste grupo são todos criados e destruídos no mesmo período de tempo. Esse conceito de grupos de recursos ajudará você a gerenciar seus recursos com mais facilidade no futuro.

Depois de selecionar o grupo de recursos mais aplicável à sua necessidade de negócios, você pode escolher a origem do seu novo banco de dados.

  1. Você pode optar por criar um banco de dados em branco, isso lhe dá mais flexibilidade com seu novo banco de dados.
  2. Você também pode optar por criar seu banco de dados a partir de um banco de dados de amostra. Atualmente, as duas opções de exemplo são o AdventureWorks, o banco de dados de demonstração OLTP (transacional) da Microsoft.
  3. A última opção é escolher criar seu banco de dados a partir de um backup. Essa opção é melhor para aqueles que estão criando várias cópias de um único banco de dados. Com esta opção, você pode optar por restaurar a partir de um backup automático feito em um Banco de Dados SQL do Windows Azure. Isso pode ser útil se você precisar executar uma restauração “parcial”.

Depois de selecionar a opção de origem mais adequada à sua necessidade de negócios, você pode optar por criar esse novo banco de dados em um servidor de banco de dados SQL do Windows Azure existente (se houver um) ou criar um novo servidor. Se você estiver criando um novo servidor, o nome dado a esse novo servidor deverá ser exclusivo em todo o Windows Azure. Você poderá selecionar o alias para este servidor. O nome completo deste servidor será .database.windows.net.

Depois de selecionar a opção de servidor que melhor atende às suas necessidades de negócios, você será solicitado a fornecer um tipo de preço. Se você clicar na seleção do tipo de preço, poderá ver os SLAs e preços atuais para os diferentes níveis de preço.

Aqui estão os conceitos básicos dos níveis de preços. Os bancos de dados de camada B terão o desempenho mais lento e terão as restrições mais severas de tamanho e recursos, mas serão os menos caros. Os bancos de dados de nível P serão as ofertas mais rápidas e cheias de recursos, mas seu preço refletirá isso. O nível S está no meio.

A última opção com a qual você se depara ao criar um novo banco de dados é o agrupamento. Isso controla como os dados serão comparados e classificados. Se você não tiver um bom motivo comercial para alterar a opção padrão, sugiro deixar isso em paz. Depois de fazer todas as suas seleções, clique em Criar e uma tarefa em segundo plano será acionada para criar seu novo banco de dados.

Você receberá um aviso no Portal do Windows Azure assim que seu banco de dados estiver pronto. Para se conectar a esse banco de dados de fora do Windows Azure (como sua estação de trabalho), serão necessárias etapas adicionais. Consulte a documentação do StackOverflow para gerenciar as configurações de firewall do banco de dados SQL do Windows Azure.

Crie um banco de dados SQL do Windows Azure com o PowerShell #

Antes de começar, verifique se você tem o Azure PowerShell mais recente instalado. Depois de instalado, inicie uma sessão do Azure PowerShell em sua máquina. Primeiro, você precisará fazer login e se autenticar no Windows Azure.

Add-AzureRmAccount

Você receberá uma caixa de diálogo solicitando suas credenciais do Azure. Insira-os e clique em entrar. Em seguida, você precisará identificar a qual assinatura deseja adicionar esse novo Banco de Dados SQL. No PowerShell, é mais fácil identificar sua assinatura pelo identificador global exclusivo (GUID), em vez de pelo nome.

Para encontrar seu GUID de assinatura, acesse https://portal.azure.com e clique no bloco de assinaturas na página inicial do portal.

A folha da página de destino será substituída pela folha Assinaturas. A partir daqui, você pode ver todas as assinaturas às quais tem acesso no contexto atual do Windows Azure Active Directory. Mantenha uma cópia desses GUIDs pronta para futuros scripts do PowerShell. Agora que você tem o GUID da sua assinatura, você pode definir o escopo da sua sessão do PowerShell para essa assinatura.

Set-AzureRmContext -SubscriptionId '<your subscription's GUID>'

Agora, se você não tiver um grupo de recursos existente ao qual adicionar esse novo Banco de Dados SQL, precisará criar um. No mínimo, seu grupo de recursos precisa de um nome e um local. Os locais são os diferentes datacenters que podem hospedar seus recursos do Azure.

Para obter uma lista de data centers do Azure capazes de hospedar seu Banco de Dados SQL, execute o seguinte comando.

(Get-AzureRmLocation | Where-Object { $_.Providers -eq "Microsoft.Sql" }).Location

Observe que todos os locais estão em letras minúsculas. Agora, para criar um novo grupo de recursos, use o comando a seguir.

New-AzureRmResourceGroup 
     -Name '<new_resource_group>' `
     -Location '<data_center>'

Em seguida, você precisa de um servidor para hospedar seu banco de dados SQL. Se você ainda não tem um servidor que deseja usar, crie um agora. Os servidores exigem um nome de grupo de recursos, nome de servidor e local.

New-AzureRmSqlServer `
     -Location '<data_center>' `
     -ResourceGroupName '<new_resource_group>' `
     -ServerName '<sql_server_name>'

Você será solicitado a fornecer um nome de usuário e senha administrativos. Esta será a conta de administrador do sistema ou “sa” do seu novo SQL Server.

Agora que você tem um grupo de recursos e um nome de servidor SQL, está pronto para criar o próprio banco de dados. As duas últimas seleções a serem feitas são edição e nível de serviço.

Para edição, você pode escolher Padrão, Nenhum, Premium, Básico, Padrão, DataWarehouse ou Gratuito. Para o nível de serviço, você tem muito mais opções. Para iniciantes, fique com aqueles para os quais você pode ver preços na calculadora de preços do Azure. Para usuários mais avançados, confira o cmdlet do PowerShellGet-AzureRmSqlServerServiceObjective.

O comando a seguir criará seu Banco de Dados SQL.

New-AzureRmSqlDatabase `
     -DatabaseName '<database_name>' `
     -Edition 'basic' `
     -ResourceGroupName '<new_resource_group>' `
     -RequestedServiceObjectiveName 'basic' `
     -ServerName '<sql_server_name>'
