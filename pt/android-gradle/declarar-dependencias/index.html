Declare dependências para sabores #

As dependências podem ser adicionadas para variações de produtos específicos de maneira semelhante às configurações de compilação.

android {
    ...   
    productFlavors {
        flavor1 {
            //...
        }
        flavor2 {
            //...
        }
    }
}

dependencies {
    flavor1Compile 'com.android.support:appcompat-v7:24.1.1'
    flavor1Compile 'com.google.firebase:firebase-crash:9.4.0'
        
    flavor2Compile 'com.android.support:appcompat-v7:24.1.1'
}

1: https://www.wikiod.com/pt/android-gradle/declarar-dependencias#Declarar dependências para configurações

Como adicionar dependências #

O exemplo abaixo descreve como declarar três tipos diferentes de dependências diretas no arquivo build.gradle do aplicativo/módulo:

   android {...}
    ...
    dependencies {
        // The 'compile' configuration tells Gradle to add the dependency to the
        // compilation classpath and include it in the final package.
    
        // Dependency on the "mylibrary" module from this project
        compile project(":mylibrary")
    
        // Remote binary dependency
        compile 'com.android.support:appcompat-v7:24.1.0'
    
        // Local binary dependency
        compile fileTree(dir: 'libs', include: ['*.jar'])
    }

Como adicionar um repositório #

Para baixar dependências, declare o repositório para que o Gradle possa encontrá-las. Para fazer isso, adicione um repositories { ... } ao build.gradle do aplicativo/módulo no arquivo de nível superior.

repositories {
  // Gradle's Java plugin allows the addition of these two repositories via method calls:
  jcenter()
  mavenCentral()

  maven { url "http://repository.of/dependency" }

  maven { 
      credentials {
          username 'xxx'
          password 'xxx'
      }

  url 'http://my.maven
  }
}

Dependências do módulo #

Em um gradle build de vários projetos, você pode ter uma dependência com outro módulo em seu build.

Exemplo:

  dependencies {
        // Dependency on the "mylibrary" module from this project
        compile project(":mylibrary")
  }

A linha compile project(':mylibrary') declara um módulo de biblioteca local do Android chamado “mylibrary” como uma dependência e exige que o sistema de compilação compile e inclua o módulo local ao compilar seu aplicativo.

Dependências binárias locais #

Você pode ter uma dependência com um único jar ou vários arquivos jar.

Com um único arquivo jar, você pode adicionar:

dependencies {
    compile files('libs/local_dependency.jar')
}

É possível adicionar um diretório de jars para compilar.

dependencies {
        compile fileTree(dir: 'libs', include: ['*.jar'])
}

A linha compile fileTree(dir: 'libs', include: ['*.jar']) diz ao sistema de compilação para incluir quaisquer arquivos JAR dentro do diretório app/libs/ no classpath de compilação e no pacote final do seu aplicativo.

Se você tiver módulos que requerem dependências binárias locais, copie os arquivos JAR para essas dependências em <moduleName>/libs dentro de seu projeto.

Se você precisar adicionar um arquivos aar, leia mais detalhes aqui.

Dependências binárias remotas #

Você pode adicionar dependências remotas no Gradle usando esta estrutura:

compile 'group:name:version'

ou esta sintaxe alternativa:

compile group: 'xxx', name: 'xxxxx', version: 'xxxx'

Por exemplo:

compile 'com.android.support:appcompat-v7:24.1.0'

A linha compile 'com.android.support:appcompat-v7:24.1.0’ declara uma dependência na versão 24.1.0 da Android Support Library.

Declare dependências para configurações #

Dependências podem ser adicionadas para configuração específica como test/androidTest

androidTestCompile 'com.android.support.test.espresso:espresso-core:2.2.1'
testCompile 'junit:junit:3.8.1'

Como alternativa, crie sua própria configuração

configurations {
    myconfig
}

E então baixe a dependência para esta configuração

myconfig group: 'com.mycompany', name: 'my_artifact', version: '1.0.0'

Declare dependências para tipos de compilação #

As dependências podem ser adicionadas para tipos de compilação específicos:

android {
    ...   
    buildTypes {
        release {
            //...
        }

        debug {
            //....
        }
    }
}

dependencies {
    debugCompile 'com.android.support:appcompat-v7:24.1.1'
    releaseCompile 'com.google.firebase:firebase-crash:9.4.0'            
}
