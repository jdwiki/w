Parâmetros #

Parâmetro Detalhes
Estado da resposta 401 se o servidor de origem exigir autenticação, [407](https://tools.ietf.org/html/rfc7235 #section-3.2) se um proxy intermediário requer autenticação
Cabeçalhos de resposta WWW-Authenticate pelo servidor de origem, [Proxy-Authenticate](https://tools.ietf.org/html /rfc7235#section-4.3) por um proxy intermediário
Cabeçalhos de solicitação Authorization para autorização em um servidor de origem, [Proxy-Authorization](https://tools.ietf.org/html /rfc7235#section-4.4) em relação a um proxy intermediário
Esquema de autenticação Basic para autenticação básica, mas outros como Digest e SPNEGO podem ser usados. Consulte o Registro de esquemas de autenticação HTTP.
Reino Um nome do espaço protegido no servidor; um servidor pode ter vários desses espaços, cada um com um nome e mecanismos de autenticação distintos.
Credenciais Para Basic: nome de usuário e senha separados por dois pontos, codificado em base64; por exemplo, username:password codificado em base64 é dXNlcm5hbWU6cGFzc3dvcmQ=

A autenticação básica é definida em RFC2617. Ele pode ser usado para autenticar no servidor de origem após receber um 401 Unauthorized, bem como em um servidor proxy após um 407 (Proxy Authentication Required). Nas credenciais (decodificadas), a senha começa após os primeiros dois pontos. Portanto, o nome de usuário não pode conter dois pontos, mas a senha pode.

Autenticação básica HTTP #

A autenticação básica HTTP fornece um mecanismo direto para autenticação. As credenciais são enviadas em texto simples e, portanto, são inseguras por padrão. A autenticação bem-sucedida procede da seguinte maneira.

O cliente solicita uma página cujo acesso é restrito:

GET /secret

O servidor responde com o código de status 401 Unauthorized e solicita que o cliente autentique:

401 Unauthorized
WWW-Authenticate: Basic realm="Secret Page"

O cliente envia o cabeçalho Authorization. As credenciais são username:password codificadas em base64:

GET /secret
Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=

O servidor aceita as credenciais e responde com o conteúdo da página:

HTTP/1.1 200 OK
