Instalação do Angular usando angular-cli #

Este exemplo é uma configuração rápida do Angular e como gerar um projeto de exemplo rápido.

Pré-requisitos: #

  • Node.js 6.9.0 ou superior.
  • npm v3 ou superior ou yarn.
  • Digitações v1 ou superior.

Abra um terminal e execute os comandos um por um:

npm install -g typos ou yarn global add typos

npm install -g @angular/cli ou yarn global add @angular/cli

O primeiro comando instala a [biblioteca de digitações][typings] globalmente (e adiciona o executável typings ao PATH). O segundo instala @angular/cli globalmente, adicionando o executável ng ao PATH.

Para configurar um novo projeto #

Navegue com o terminal para uma pasta onde você deseja configurar o novo projeto.

Execute os comandos:

ng new PROJECT_NAME
cd PROJECT_NAME
ng serve

É isso, agora você tem um projeto de exemplo simples feito com Angular. Agora você pode navegar até o link exibido no terminal e ver o que está sendo executado.

Para adicionar a um projeto existente #

Navegue até a raiz do seu projeto atual.

Execute o comando:

ng init

Isso adicionará o andaime necessário ao seu projeto. Os arquivos serão criados no diretório atual, portanto, certifique-se de executá-lo em um diretório vazio.

Executando o projeto localmente #

Para ver e interagir com seu aplicativo enquanto ele está sendo executado no navegador, você deve iniciar um servidor de desenvolvimento local hospedando os arquivos do seu projeto.

ng serve

Se o servidor foi iniciado com sucesso, ele deve exibir um endereço no qual o servidor está sendo executado. Geralmente é isso:

http://localhost:4200

Fora da caixa, este servidor de desenvolvimento local está conectado com o Hot Module Reloading, portanto, qualquer alteração no html, typescript ou css fará com que o navegador seja recarregado automaticamente (mas pode ser desativado, se desejado).

Geração de Componentes, Diretivas, Tubos e Serviços #

O comando ng generate <scaffold-type> <name> (ou simplesmente ng g <scaffold-type> <name>) permite que você gere automaticamente componentes Angular:

# The command below will generate a component in the folder you are currently at
ng generate component my-generated-component
# Using the alias (same outcome as above)
ng g component my-generated-component
# You can add --flat if you don't want to create new folder for a component
ng g component my-generated-component --flat
# You can add --spec false if you don't want a test file to be generated (my-generated-component.spec.ts)
ng g component my-generated-component --spec false

Existem vários tipos possíveis de andaimes que o angular-cli pode gerar:

Tipo de andaime Uso
Módulo ng g módulo meu-novo-módulo
Componente ng g componente meu-novo-componente
Directiva ng g diretiva minha-nova diretiva
Tubulação ng g pipe my-new-pipe
Serviço ng g serviço meu-novo-serviço
Classe ng g class minha-nova-classe
Interface ng g interface minha-nova-interface
Enum ng g enum meu-novo-enum

Você também pode substituir o nome do tipo por sua primeira letra. Por exemplo:

ng g m my-new-module para gerar um novo módulo ou ng g c my-new-component para criar um componente.

Construção/Agregação

Quando você terminar de criar seu aplicativo da Web Angular e quiser instalá-lo em um servidor da Web como o Apache Tomcat, tudo o que você precisa fazer é executar o comando de compilação com ou sem o sinalizador de produção definido. A produção minimizará o código e otimizará para uma configuração de produção.

ng build

ou

ng build --prod

Em seguida, procure no diretório raiz do projeto por uma pasta /dist, que contém o arquivo build.

Se desejar os benefícios de um pacote de produção menor, você também pode usar a compilação de modelo Ahead-of-Time, que remove o compilador de modelo da compilação final:

ng build --prod --aot

Teste Unitário

Angular fornece testes de unidade embutidos, e cada item criado por angular-cli gera um teste de unidade básico, que pode ser gasto. Os testes unitários são escritos usando jasmine e executados através do Karma. Para iniciar o teste execute o seguinte comando:

ng test

Este comando executará todos os testes no projeto e os executará novamente toda vez que um arquivo fonte for alterado, seja um teste ou código do aplicativo.

Para obter mais informações, visite: angular-cli github page

Programa Angular “Hello World” #

Pré-requisitos: #

Configurando o ambiente de desenvolvimento

Antes de começarmos, precisamos configurar nosso ambiente.

  • Instale Node.js e npm se ainda não estiverem em sua máquina.

    Verify that you are running at least node 6.9.x and npm 3.x.x by running node -v and npm -v in a terminal/console window. Older versions produce errors, but newer versions are fine.

  • Instale o Angular CLI globalmente usando npm install -g @angular/cli.

Etapa 1: Criando um novo projeto #

Abra uma janela de terminal (ou prompt de comando do Node.js no Windows).

Criamos um novo projeto e aplicativo esqueleto usando o comando:

ng new my-app

Aqui o ng é para Angular. Obtemos uma estrutura de arquivos mais ou menos assim.

Existem muitos arquivos. Não precisamos nos preocupar com todos eles agora.

Etapa 2: Servindo o aplicativo #

Iniciamos nosso aplicativo usando o seguinte comando:

ng serve

Podemos usar um sinalizador -open(ou simplesmente -o) que abrirá automaticamente nosso navegador em http://localhost:4200/

ng serve --open
