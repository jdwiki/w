Instalando o PostgreSQL no Windows #

Embora seja uma boa prática usar um sistema operacional baseado em Unix (ex. Linux ou BSD) como servidor de produção, você pode instalar facilmente o PostgreSQL no Windows (espero que apenas como servidor de desenvolvimento).

Baixe os binários de instalação do Windows do EnterpriseDB: http://www.enterprisedb.com/products-services-training/pgdownload Esta é uma empresa terceirizada iniciada por colaboradores principais do projeto PostgreSQL que otimizaram os binários para Windows.

Selecione a versão estável (não-Beta) mais recente (9.5.3 no momento da redação). Você provavelmente desejará o pacote Win x86-64, mas se estiver executando uma versão de 32 bits do Windows, que é comum em computadores mais antigos, selecione Win x86-32.

Observação: alternar entre as versões Beta e Estável envolverá tarefas complexas, como despejo e restauração. A atualização na versão beta ou estável só precisa de uma reinicialização do serviço.

Você pode verificar se sua versão do Windows é de 32 ou 64 bits acessando Painel de Controle -> Sistema e Segurança -> Sistema -> Tipo de sistema, que dirá “##-bit Operating System”. Este é o caminho para o Windows 7, pode ser um pouco diferente em outras versões do Windows.

No instalador selecione os pacotes que você gostaria de usar. Por exemplo:

  • pgAdmin ( https://www.pgadmin.org ) é uma GUI gratuita para gerenciar seu banco de dados e eu recomendo. Em 9.6 isso será instalado por padrão.
  • PostGIS ( http://postgis.net ) fornece recursos de análise geoespacial em coordenadas GPS, distâncias etc. muito populares entre os desenvolvedores de GIS.
  • O Pacote de Idiomas fornece bibliotecas necessárias para linguagens procedurais oficialmente suportadas PL/Python, PL/Perl e PL/Tcl.
  • Outros pacotes como pgAgent, pgBouncer e Slony são úteis para servidores de produção maiores, apenas verificados conforme necessário.

Todos esses pacotes opcionais podem ser instalados posteriormente através do “Application Stack Builder”.

Nota: Há também outros idiomas não oficialmente suportados, como PL/V8, PL/Lua PL/Java disponíveis.

Abra o pgAdmin e conecte-se ao seu servidor clicando duas vezes em seu nome, ex. “PostgreSQL 9.5 (localhost:5432).

A partir deste ponto você pode seguir guias como o excelente livro PostgreSQL: Up and Running, 2nd Edition ( http://shop.oreilly.com/product/0636920032144.do ).

Opcional: Tipo de inicialização de serviço manual

O PostgreSQL é executado como um serviço em segundo plano, um pouco diferente da maioria dos programas. Isso é comum para bancos de dados e servidores web. Seu tipo de inicialização padrão é Automático, o que significa que ele sempre será executado sem nenhuma entrada sua.

Por que você deseja controlar manualmente o serviço PostgreSQL? Se você estiver usando seu PC como servidor de desenvolvimento algumas vezes e também usá-lo para jogar videogames, por exemplo, o PostegreSQL pode deixar seu sistema um pouco mais lento durante a execução.

Por que você não quer o controle manual? Iniciar e interromper o serviço pode ser um incômodo se você fizer isso com frequência.

Se você não notar nenhuma diferença na velocidade e preferir evitar o incômodo, deixe o Tipo de inicialização como Automático e ignore o restante deste guia. Por outro lado…

Vá para Painel de Controle -> Sistema e Segurança -> Ferramentas Administrativas.

Selecione “Serviços” na lista, clique com o botão direito do mouse em seu ícone e selecione Enviar para -> Área de trabalho para criar um ícone na área de trabalho para um acesso mais conveniente.

Feche a janela Ferramentas Administrativas e inicie os Serviços a partir do ícone da área de trabalho que você acabou de criar.

Role para baixo até ver um serviço com um nome como postgresql-x##-9.# (ex. “postgresql-x64-9.5”).

Clique com o botão direito do mouse no serviço postgres, selecione Propriedades -> Tipo de inicialização -> Manual -> Aplicar -> OK. Você pode alterá-lo de volta para automático com a mesma facilidade.

Se você vir outros serviços relacionados ao PostgreSQL na lista, como “pgbouncer” ou “PostgreSQL Scheduling Agent - pgAgent”, você também pode alterar o tipo de inicialização para Manual porque eles não são muito úteis se o PostgreSQL não estiver em execução. Embora isso signifique mais problemas cada vez que você inicia e para, então é com você. Eles não usam tantos recursos quanto o próprio PostgreSQL e podem não ter nenhum impacto perceptível no desempenho de seus sistemas.

Se o serviço estiver em execução, seu Status dirá Iniciado, caso contrário, não está em execução.

Para iniciá-lo, clique com o botão direito e selecione Iniciar. Um prompt de carregamento será exibido e deverá desaparecer por conta própria logo depois. Se der erro, tente uma segunda vez. Se isso não funcionar, houve algum problema com a instalação, possivelmente porque você alterou alguma configuração no Windows que a maioria das pessoas não altera, portanto, encontrar o problema pode exigir algumas investigações.

Para parar o postgres, clique com o botão direito do mouse no serviço e selecione Parar.

Se você receber um erro ao tentar se conectar ao seu banco de dados, verifique os Serviços para certificar-se de que está em execução.

Para outros detalhes muito específicos sobre a instalação do EDB PostgreSQL, ex. a versão do runtime do python no pacote de idioma oficial de uma versão específica do PostgreSQL, sempre consulte o guia de instalação oficial do EBD , altere a versão no link para a versão principal do seu instalador.

Instale o PostgreSQL da fonte no Linux #

Dependências:

  • Versão GNU Make > 3.80
  • um compilador ISO/ANSI C (por exemplo, gcc)
  • um extrator como tar ou gzip
  • zlib-devel
  • readline-devel ou libedit-devel

Fontes: Link para a fonte mais recente (9.6.3)

Agora você pode extrair os arquivos de origem:

tar -xzvf postgresql-9.6.3.tar.gz

Há um grande número de opções diferentes para a configuração do PostgreSQL:

Link completo para o procedimento de instalação completo

Pequena lista de opções disponíveis:

  • Caminho --prefix=PATH para todos os arquivos
  • Caminho --exec-prefix=PATH para o arquivo architectur-dependet
  • Caminho --bindir=PATH para programas executáveis
  • Caminho --sysconfdir=PATH para arquivos de configuração
  • --with-pgport=NUMBER especifique uma porta para o seu servidor
  • --with-perl adiciona suporte a perl
  • --with-python adiciona suporte a python
  • --with-openssl adiciona suporte a openssl
  • --with-ldap adiciona suporte ao ldap
  • --with-blocksize=BLOCKSIZE define o tamanho da página em KB
    • BLOCKSIZE must a power of 2 and between 1 and 32
  • --with-wal-segsize=SEGSIZE define o tamanho do segmento WAL em MB
    • SEGSIZE must be a power of 2 between 1 and 64

Entre na nova pasta criada e execute o script cofigure com as opções desejadas:

./configure --exec=/usr/local/pgsql

Execute make para criar os objectfiles

Execute make install para instalar o PostgreSQL a partir dos arquivos compilados

Execute make clean para arrumar

Para a extensão mude o diretório cd contrib, execute make e make install

Instalação no GNU+Linux #

Na maioria dos sistemas operacionais GNU+Linux, o PostgreSQL pode ser facilmente instalado usando o gerenciador de pacotes do sistema operacional.

Família Red Hat #

Os repositórios podem ser encontrados aqui: https://yum.postgresql.org/repopackages.php

Baixe o repositório para a máquina local com o comando

yum -y install https://download.postgresql.org/pub/repos/yum/X.X/redhat/rhel-7-x86_64/pgdg-redhatXX-X.X-X.noarch.rpm

Veja os pacotes disponíveis:

yum list available | grep postgres*

Os pacotes necessários são: postgresqlXX postgresqlXX-server postgresqlXX-libs postgresqlXX-contrib

Estes são instalados com o seguinte comando: yum -y install postgresqlXX postgresqlXX-server postgresqlXX-libs postgresqlXX-contrib

Uma vez instalado, você precisará iniciar o serviço de banco de dados como proprietário do serviço (o padrão é postgres). Isso é feito com o comando pg_ctl.

sudo -su postgres
./usr/pgsql-X.X/bin/pg_ctl -D /var/lib/pgsql/X.X/data start

Para acessar o banco de dados na CLI digite psql

Família Debian #

Em sistemas operacionais Debian e derivados, digite:

sudo apt-get install postgresql

Isso instalará o pacote do servidor PostgreSQL, na versão padrão oferecida pelos repositórios de pacotes do sistema operacional.

Se a versão instalada por padrão não for a que você deseja, você pode usar o gerenciador de pacotes para procurar versões específicas que podem ser oferecidas simultaneamente.

Você também pode usar o repositório Yum fornecido pelo projeto PostgreSQL (conhecido como PGDG) para obter uma versão diferente. Isso pode permitir versões ainda não oferecidas pelos repositórios de pacotes do sistema operacional.

Como instalar o PostgreSQL via MacPorts no OSX #

Para instalar o PostgreSQL no OSX, você precisa saber quais versões são suportadas atualmente.

Use este comando para ver quais versões você tem disponíveis.

sudo port list | grep "^postgresql[[:digit:]]\{2\}[[:space:]]"

Você deve obter uma lista que se parece com o seguinte:

postgresql80                   @8.0.26         databases/postgresql80
postgresql81                   @8.1.23         databases/postgresql81
postgresql82                   @8.2.23         databases/postgresql82
postgresql83                   @8.3.23         databases/postgresql83
postgresql84                   @8.4.22         databases/postgresql84
postgresql90                   @9.0.23         databases/postgresql90
postgresql91                   @9.1.22         databases/postgresql91
postgresql92                   @9.2.17         databases/postgresql92
postgresql93                   @9.3.13         databases/postgresql93
postgresql94                   @9.4.8          databases/postgresql94
postgresql95                   @9.5.3          databases/postgresql95
postgresql96                   @9.6beta2       databases/postgresql96

Neste exemplo, a versão mais recente do PostgreSQL que é suportada em 9.6, então vamos instalá-la.

sudo port install postgresql96-server postgresql96

Você verá um log de instalação como este:

--->  Computing dependencies for postgresql96-server
--->  Dependencies to be installed: postgresql96
--->  Fetching archive for postgresql96
--->  Attempting to fetch postgresql96-9.6beta2_0.darwin_15.x86_64.tbz2 from         https://packages.macports.org/postgresql96
--->  Attempting to fetch postgresql96-9.6beta2_0.darwin_15.x86_64.tbz2.rmd160 from https://packages.macports.org/postgresql96
--->  Installing postgresql96 @9.6beta2_0
--->  Activating postgresql96 @9.6beta2_0

To use the postgresql server, install the postgresql96-server port

--->  Cleaning postgresql96
--->  Fetching archive for postgresql96-server
--->  Attempting to fetch postgresql96-server-9.6beta2_0.darwin_15.x86_64.tbz2 from https://packages.macports.org/postgresql96-server
--->  Attempting to fetch postgresql96-server-9.6beta2_0.darwin_15.x86_64.tbz2.rmd160 from https://packages.macports.org/postgresql96-server
--->  Installing postgresql96-server @9.6beta2_0
--->  Activating postgresql96-server @9.6beta2_0

To create a database instance, after install do
 sudo mkdir -p /opt/local/var/db/postgresql96/defaultdb
 sudo chown postgres:postgres /opt/local/var/db/postgresql96/defaultdb
 sudo su postgres -c '/opt/local/lib/postgresql96/bin/initdb -D /opt/local/var/db/postgresql96/defaultdb'

--->  Cleaning postgresql96-server
--->  Computing dependencies for postgresql96
--->  Cleaning postgresql96
--->  Updating database of binaries
--->  Scanning binaries for linking errors
--->  No broken files found.

O log fornece instruções sobre o restante das etapas de instalação, então faremos isso a seguir.

sudo mkdir -p /opt/local/var/db/postgresql96/defaultdb
sudo chown postgres:postgres /opt/local/var/db/postgresql96/defaultdb
sudo su postgres -c '/opt/local/lib/postgresql96/bin/initdb -D /opt/local/var/db/postgresql96/defaultdb'

Agora iniciamos o servidor:

sudo port load -w postgresql96-server

Verifique se podemos nos conectar ao servidor:

su postgres -c psql

Você verá um prompt do postgres:

psql (9.6.1)
Type "help" for help.

postgres=#

Aqui você pode digitar uma consulta para ver se o servidor está em execução.

postgres=#SELECT setting FROM pg_settings WHERE name='data_directory';

E veja a resposta:

                setting
------------------------------------------
/opt/local/var/db/postgresql96/defaultdb
(1 row)
postgres=#

Digite \q para sair:

postgres=#\q

E você estará de volta ao prompt do shell.

Parabéns! Agora você tem uma instância do PostgreSQL em execução no OS/X.

Instale o postgresql com brew no Mac #

O Homebrew se autodenomina ‘o gerenciador de pacotes ausente para macOS’. Ele pode ser usado para construir e instalar aplicativos e bibliotecas. Uma vez instalado, você pode usar o comando brew para instalar o PostgreSQL e suas dependências da seguinte forma:

brew update
brew install postgresql

O Homebrew geralmente instala a versão estável mais recente. Se você precisar de um diferente então brew search postgresql irá listar as versões disponíveis. Se você precisa do PostgreSQL construído com opções particulares então brew info postgresql irá listar quais opções são suportadas. Se você precisar de uma opção de compilação não suportada, talvez seja necessário fazer a compilação por conta própria, mas ainda poderá usar o Homebrew para instalar as dependências comuns.

Inicie o servidor:

brew services start postgresql

Abra o prompt do PostgreSQL

psql

Se o psql reclamar que não há banco de dados correspondente para seu usuário, execute createdb.

Postgres.app para Mac OSX #

Uma ferramenta extremamente simples para instalar o PostgreSQL em um Mac está disponível baixando Postgres.app.

Você pode alterar as preferências para que o PostgreSQL seja executado em segundo plano ou apenas quando o aplicativo estiver em execução.
