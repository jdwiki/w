Funções puras #

As funções puras são independentes e não têm efeitos colaterais. Dado o mesmo conjunto de entradas, uma função pura sempre retornará o mesmo valor de saída.

A seguinte função é pura:

function pure(data) {
    return data.total + 3;
}

No entanto, esta função não é pura, pois modifica uma variável externa:

function impure(data) {
    data.total += 3;
    return data.total;
}

Exemplo:

data = {
    total: 6
};

pure(data);   // outputs: 9
impure(data); // outputs: 9 (but now data.total has changed)
impure(data); // outputs: 12

Funções de ordem superior #

Funções de ordem superior recebem outras funções como argumentos e/ou as retornam como resultados. Eles formam os blocos de construção da programação funcional. A maioria das linguagens funcionais tem alguma forma de função de filtro, por exemplo. Esta é uma função de ordem superior, tendo uma lista e um predicado (função que retorna verdadeiro ou falso) como argumentos.

Funções que não fazem nada disso são frequentemente chamadas de funções de primeira ordem.

function validate(number,predicate) {
    if (predicate) {    // Is Predicate defined
        return predicate(number);
    }
    return false;
}

Aqui “predicado” é uma função que testará alguma condição envolvendo seus argumentos e retornará verdadeiro ou falso.

Um exemplo de chamada para o acima é:

validate(someNumber, function(arg) {
    return arg % 10 == 0;
    }
);

Um requisito comum é adicionar números dentro de um intervalo. Usando funções de ordem superior, podemos estender essa capacidade básica, aplicando uma função de transformação em cada número antes de incluí-lo na soma.

Você deseja adicionar todos os inteiros dentro de um determinado intervalo (usando Scala)

def sumOfInts(a: Int, b: Int): Int = {
  if(a > b) 0
  else a + sumOfInts(a+1, b)
}

Você deseja adicionar quadrados de todos os números inteiros dentro de um determinado intervalo

def square(a: Int): Int = a * a

def sumOfSquares(a: Int, b: Int): Int = {
  if(a > b) 0
  else square(a) + sumOfSquares(a + 1, b)
}

Observe que essas coisas têm uma coisa em comum, que você deseja aplicar uma função em cada argumento e depois adicioná-los.

Vamos criar uma função de ordem superior para fazer as duas coisas:

def sumHOF(f: Int => Int, a: Int, b: Int): Int = {
  if(a > b) 0
  else f(a) + sumHOF(f, a + 1, b)
}

Você pode chamar assim:

def identity(a: Int): Int = a

def square(a: Int): Int = a * a

Observe que sumOfInts e sumOfSquare podem ser definidos como:

def sumOfInts(a: Int, b: Int): Int = sumHOF(identity, a, b)

def sumOfSquares(a: Int, b: Int): Int = sumHOF(square, a, b)

Como você pode ver neste exemplo simples, as funções de ordem superior fornecem soluções mais generalizadas e reduzem a duplicação de código.

Usei Scala By Example - de Martin Odersky como referência.

Escovando #

Currying é o processo de transformar uma função que recebe vários argumentos em uma sequência de funções em que cada uma tem apenas um único parâmetro. Currying está relacionado, mas não o mesmo que, aplicação parcial.

Vamos considerar a seguinte função em JavaScript:

var add = (x, y) => x + y

Podemos usar a definição de currying para reescrever a função add:

var add = x => y => x + y

Esta nova versão usa um único parâmetro, x, e retorna uma função que recebe um único parâmetro, y, que retornará o resultado da adição de x e y.

var add5 = add(5)
var fifteen = add5(10) // fifteen = 15

Outro exemplo é quando temos as seguintes funções que colocam colchetes em torno de strings:

var generalBracket = (prefix, str, suffix) => prefix + str + suffix

Agora, toda vez que usamos generalBracket temos que passar os colchetes:

var bracketedJim = generalBracket("{", "Jim", "}") // "{Jim}"
var doubleBracketedJim = generalBracket("{{", "Jim", "}}") // "{{Jim}}"

Além disso, se passarmos as strings que não são colchetes, nossa função ainda retornará um resultado errado. Vamos corrigir isso:

var generalBracket = (prefix, suffix) => str => prefix + str + suffix
var bracket = generalBracket("{", "}")
var doubleBracket = generalBracket("{{", "}}")

Observe que tanto bracket quanto doubleBracket agora são funções aguardando seu parâmetro final:

var bracketedJim = bracket("Jim") // "{Jim}"
var doubleBracketedJim = doubleBracket("Jim") // "{{Jim}}"

Imutabilidade #

Em linguagens tradicionais orientadas a objetos, x = x + 1 é uma expressão simples e legal. Mas em Programação Funcional, é ilegal.

Variáveis ​​não existem em Programação Funcional. Os valores armazenados ainda são chamados de variáveis ​​apenas por causa do histórico. Na verdade, eles são constantes. Uma vez que x assume um valor, é esse valor para toda a vida.

Então, se uma variável é uma constante, então como podemos alterar seu valor?

A Programação Funcional lida com alterações de valores em um registro fazendo uma cópia do registro com os valores alterados.

Por exemplo, em vez de fazer:

var numbers = [1, 2, 3];
numbers[0] += 1; // numbers = [2, 2, 3];

Você faz:

var numbers = [1, 2, 3];
var newNumbers = numbers.map(function(number) {
    if (numbers.indexOf(number) == 0)
        return number + 1
    return number
});
console.log(newNumbers) // prints [2, 2, 3]

E não há loops na Programação Funcional. Usamos recursão ou funções de ordem superior como map, filter e reduce para evitar loops.

Vamos criar um loop simples em JavaScript:

var acc = 0;
for (var i = 1; i <= 10; ++i)
    acc += i;
console.log(acc); // prints 55

Ainda podemos fazer melhor alterando o tempo de vida do acc de global para local:

function sumRange(start, end, acc) {
    if (start > end)
        return acc;
    return sumRange(start + 1, end, acc + start)
}
console.log(sumRange(1, 10, 0)); // 55

Nenhuma variável ou loop significa código mais simples, seguro e legível (especialmente ao depurar ou testar - você não precisa se preocupar com o valor de x após várias instruções, ele nunca mudará).
