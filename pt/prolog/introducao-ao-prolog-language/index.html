Programação de banco de dados #

Prolog categoriza tudo em:

  • Atoms - Qualquer sequência de caracteres que não comece com letras maiúsculas. Por exemplo - a, b, ok
  • Números - Não há sintaxe especial para números, nenhuma declaração é necessária. Por exemplo, 1, 22, 35,8
  • Variáveis - Uma string que começa com um caractere maiúsculo ou sublinhado (_). Por exemplo, X, Y, Abc, AA
  • Termos Complexos - São feitos a partir de um functor e uma sequência de argumentos. O nome de um termo complexo é sempre um átomo, enquanto os argumentos podem ser átomos ou variáveis. Por exemplo, pai(joão, corça), parente(a), mãe(X,Y).

Um banco de dados lógico contém um conjunto de fatos e regras.

Um termo complexo com apenas átomos como argumentos é chamado de fato, enquanto um termo complexo com variáveis ​​como argumentos é chamado de regra.

Exemplo de fatos em Prolog:

father_child(fred, susan).
mother_child(hillary, joe).

Exemplo de uma regra em Prolog:

child_of(X,Y):-
    father_child(Y,X)
    ;
    mother_child(Y,X).

Observe que o ; aqui é como o operador or em outras linguagens.

Prolog é uma linguagem declarativa e você pode ler esse banco de dados da seguinte forma:

fred é o pai de susan

hillary é a mãe de joe.

Para todos os X e Y, X é filho de Y se Y for pai de X ou Y for mãe de X.

De fato, um conjunto finito de fatos e/ou regras constitui um programa lógico.

O uso de tal programa é demonstrado fazendo consultas. As consultas permitem que você recupere informações de um programa lógico.

Para carregar o banco de dados no interpretador (assumindo que você salvou o banco de dados no diretório em que está executando o interpretador), basta digitar:

?- [nameofdatabase].

substituindo o nameofdatabase pelo nome real do arquivo (observe que aqui excluímos a extensão .pl do nome do arquivo).

Exemplo de consultas no interpretador para o programa acima e os resultados:

?- child_of(susan,fred).
true

?- child_of(joe,hillary).
true

?- child_of(fred,susan).
false

?- child_of(susan,hillary).
false

?- child_of(susan,X).
X = fred

?- child_of(X,Y).
X = susan,
Y = fred ;
X = joe,
Y = hillary.

As perguntas acima e suas respostas podem ser lidas da seguinte forma:

Susan é filha de Fred? - verdadeiro

Joe é filho de Hillary? - verdadeiro

*Fred é filho de Susan? - falso *

*Susan é filha de Hillary? - falso *

de quem a Susan é filha? - Fred

É assim que programamos a lógica no Prolog. Um programa lógico é mais formalmente: um conjunto de axiomas, ou regras, definindo relações (também conhecidas como predicados) entre objetos. Uma maneira alternativa de interpretar o banco de dados acima de uma maneira lógica mais formal é:

A relação father_child mantém entre fred e susan

A relação mother_child mantém entre Hillary e Joe

Para todos os X e Y a relação child_of vale entre X e Y se a relação father_child for válida entre Y e X, ou a relação mother_child for válida entre Y e X.

Olá Mundo #

Hello, World no interpretador interativo #

Para imprimir “Olá, Mundo!” no interpretador Prolog (aqui estamos usando swipl, o shell para SWI Prolog):

$ swipl
<...banner...>
?- write('Hello, World!'), nl.

?- é o prompt do sistema: indica que o sistema está pronto para o usuário inserir uma sequência de objetivos (ou seja, uma consulta) que deve ser encerrada com um . (ponto final).

Aqui a consulta write('Hello World!'), nl tem dois objetivos:

  • write('Hello World!'): 'Hello World!' deve ser exibido e (,)
  • uma nova linha (nl) deve seguir.

write/1 (o /1 é usado para indicar que o predicado recebe um argumento) e nl/0 são predicados embutidos (a definição é fornecida antecipadamente pelo sistema Prolog). Os predicados integrados fornecem recursos que não podem ser obtidos pela definição pura do Prolog ou para evitar que o programador precise defini-los.

A saída:

Olá, Mundo!

sim

termina com “sim” significando que a consulta foi bem-sucedida. Em alguns sistemas, true é impresso em vez de yes.

Olá, Mundo de um arquivo #

Abra um novo arquivo chamado hello_world.pl e insira o seguinte texto:

:- initialization hello_world, halt.

hello_world :-
    write('Hello, World!'), nl.

A diretiva initialization especifica que o objetivo hello_world, halt deve ser chamado quando o arquivo é carregado. halt sai do programa.

Este arquivo pode então ser executado pelo seu executável Prolog. Os sinalizadores exatos dependem do sistema Prolog. Se você estiver usando o SWI Prolog:

$ swipl -q -l hello_world.pl

Isso produzirá a saída Hello, World!. O sinalizador -q suprime o banner que geralmente é exibido quando você chama run swipl. O -l especifica um arquivo para carregar.

Instalação ou Configuração #

SWI-Prolog

Windows e Mac:

  • Faça o download do SWI-Prolog no site oficial
  • Basta instalar seguindo as instruções do instalador.

Linux (PPA):

  • Adicione o PPA ppa:swi-prolog/stable às fontes de software do seu sistema (os desenvolvedores podem escolher por ppa:swi-prolog/devel):

    • Open a terminal (Ctrl+Alt+T) and type: sudo add-apt-repository ppa:swi-prolog/stable

    • Afterwards, update the package information: sudo apt-get update

  • Agora instale o SWI-Prolog através do gerenciador de pacotes: sudo apt-get install swi-prolog

  • Agora você pode iniciar o SWI-Prolog através da linha de comando com o comando swipl

anexo/3 #

append([], Bs, Bs).
append([A|As], Bs, [A|Cs]) :-
    append(As, Bs, Cs).

append/3 é uma das relações Prolog mais conhecidas. Ele define uma relação entre três argumentos e é verdadeiro se o terceiro argumento for uma lista que denota a concatenação das listas especificadas no primeiro e segundo argumentos.

Notavelmente, e como é típico para um bom código Prolog, append/3 pode ser usado em várias direções: Ele pode ser usado para:

  • anexar duas listas total ou parcialmente instanciadas:

      ?- A = [1, 2, 3], B=[4, 5, 6], append(A, B, Y)
  Output:
  A = [1, 2, 3],
  B = [4, 5, 6],
  Y = [1, 2, 3, 4, 5, 6].

  • verifique se a relação é verdadeira para três listas totalmente instanciadas:

      ?- A = [1, 2, 3], B = [4, 5], C = [1, 2, 3, 4, 5, 6], append(A, B, C)
  Output:
  false

  • gerar todas as maneiras possíveis de anexar duas listas a uma determinada lista:

      ?- append(A, B, [1, 2, 3, 4]).
  Output:
  A = [],
  B = [1, 2, 3, 4] ;
  A = [1],
  B = [2, 3, 4] ;
  A = [1, 2],
  B = [3, 4] ;
  A = [1, 2, 3],
  B = [4] ;
  A = [1, 2, 3, 4],
  B = [] ;
  false.

Restrições CLP(FD) #

Restrições CLP(FD) são fornecidas por todas as implementações sérias do Prolog. Eles nos permitem raciocinar sobre inteiros de maneira pura.

?- X #= 1 + 2.
X = 3.

?- 5 #= Y + 2.
Y = 3.
