Listas de Diferenças

  • Uso básico
  • Avalie uma expressão aritmética

Nesta página

  • Uso básico
  • Avalie uma expressão aritmética

Listas de Diferenças no Prolog denota o conceito de conhecer a estrutura de uma lista até certo ponto. O restante da lista pode ser deixado desvinculado até a avaliação completa de um predicado. Uma lista cujo final é desconhecido é chamada de lista aberta, terminada por um buraco. Essa técnica é especialmente útil para validar sintaxes ou gramáticas complexas.

A bem conhecida Definite Clause Grammars (DCG) está usando Listas de Diferenças para operar sob o capô.

Uso básico #

Vamos considerar o predicado sumDif/2, verificado se a estrutura de uma lista corresponde a várias restrições. O primeiro termo representa a lista a ser analisada e o segundo termo outra lista que contém a parte da primeira lista que é desconhecida para nossas restrições.

Para a demonstração, sumDif/2 reconhece uma expressão aritmética para somar n inteiros.

sumDif([X, +|OpenList], Hole) :-
    integer(X),
    sumDif(OpenList, Hole).

Sabemos que o primeiro elemento da lista a validar é um inteiro, aqui ilustrado por X, seguido do símbolo da adição (+). O restante da lista que ainda precisa ser processada posteriormente (OpenList) é deixado sem validação nesse nível. Hole representa a parte da lista que não precisamos validar.

Vamos dar outra definição do predicado sumDif/2 para completar a validação da expressão aritmética:

sumDif([X|Hole], Hole) :-
    integer(X).

Esperamos um inteiro chamado X diretamente no início da lista aberta. Curiosamente, o restante da lista Hole é deixado desconhecido e esse é todo o propósito das Listas de Diferenças: a estrutura da lista é conhecida até certo ponto.

Finalmente, a peça que falta vem quando uma lista é avaliada:

?- sumDif([1,+,2,+,3], []).
true

É quando o predicado é usado que o final da lista é mencionado, aqui [], indica que a lista não contém elementos adicionais.

Avalie uma expressão aritmética #

Vamos definir uma gramática que nos permita fazer adições, multiplicações com o uso de parênteses. Para agregar mais valor a este exemplo, vamos calcular o resultado da expressão aritmética. Resumo da gramática:

expressão → tempos expressão → vezes ‘+’ expressão vezes → elemento vezes → elemento ‘*’ vezes elemento → “inteiro” elemento → ‘(’ expressão ‘)’

Todos os predicados têm aridade 3, pois precisam abrir a lista, o buraco e o valor da expressão aritmética.

expression(Value, OpenList, FinalHole) :-
    times(Value, OpenList, FinalHole).

expression(SumValue, OpenList, FinalHole) :-
    times(Value1, OpenList, ['+'|Hole1]),
    expression(Value2, Hole1, FinalHole),
    plus(Value1, Value2, SumValue).

times(Value, OpenList, FinalHole) :-
    element(Value, OpenList, FinalHole).

times(TimesValue, OpenList, FinalHole) :-
    element(Value1, OpenList, ['*'|Hole1]),
    times(Value2, Hole1, FinalHole),
    TimesValue is Value1 * Value2.

element(Value, [Value|FinalHole], FinalHole) :-
    integer(Value).

element(Value, ['('|OpenList], FinalHole) :-
    expression(Value, OpenList, [')'|FinalHole]).

Para explicar adequadamente o princípio dos buracos e como o valor é calculado, vamos usar a segunda cláusula expression:

expression(SumValue, OpenList, FinalHole) :-
    times(Value1, OpenList, ['+'|Hole1]),
    expression(Value2, Hole1, FinalHole),
    plus(Value1, Value2, SumValue).

A lista aberta é denotada pelo predicado OpenList. O primeiro elemento a validar é o que vem antes do símbolo de adição (+). Quando o primeiro elemento é validado, é seguido diretamente pelo símbolo de adição e pela continuação da lista, chamada Hole1. Sabemos que Hole1 é o próximo elemento a validar e pode ser outra expression, portanto Hole1 é então o termo dado ao predicado expression.

O valor é sempre representado no primeiro termo. Nesta cláusula, é definido pela soma do Value1 (tudo antes do símbolo de adição) e Value2 (tudo após o símbolo de adição).

Finalmente, a expressão an pode ser avaliada.

?- expression(V, [1,+,3,*,'(',5,+,5,')'], []).
V = 31
