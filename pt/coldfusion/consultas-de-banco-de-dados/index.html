Trabalhando com bancos de dados #

Um dos pontos fortes do ColdFusion é a facilidade de trabalhar com bancos de dados. E, claro, as entradas de consulta podem e devem ser parametrizadas.

Implementação de tags

<cfquery name="myQuery" datasource="myDatasource" result="myResult">
    select firstName, lastName
    from users
    where lastName = <cfqueryparam value="Allaire" cfsqltype="cf_sql_varchar">
</cfquery>

Implementação de CFScript

// ColdFusion 9+
var queryService = new query(name="myQuery", datasource="myDatasource");
queryService.addParam(name="lName", value="Allaire", cfsqltype="cf_sql_varchar");
var result = queryService.execute(sql="select firstName, lastName from users where lastName = :lName");
var myQuery = result.getResult();
var myResult = result.getPrefix();

// ColdFusion 11+
var queryParams = {lName = {value="Allaire", cfsqltype="cf_sql_varchar"}};
var queryOptions = {datasource="myDatasource", result="myResult"};
var myQuery = queryExecute("select firstName, lastName from users", queryParams, queryOptions);

Inserir valores é tão fácil quanto:

queryExecute("
    insert into user( firstname, lastname )
    values( :firstname, :lastname )
",{
    firstname: { cfsqltype: "cf_sql_varchar", value: "Dwayne" }
    ,lastname: { cfsqltype: "cf_sql_varchar", value: "Camacho" }
},{
    result: "local.insertResult"
});

return local.insertResult.generated_key;

Exemplo básico #

As conexões de banco de dados são configuradas usando a ferramenta CF Administrator. Consulte Conexões de banco de dados para saber como conectar uma fonte de dados.

Para executar consultas, tudo o que você precisa é da tag <cfquery>. A tag <cfquery> conecta e abre o banco de dados para você, tudo que você precisa fazer é fornecer o nome da fonte de dados.

<cfquery name="Movies" datasource="Entertainment">
    SELECT title
    FROM   Movies
</cfquery>

Para exibir os resultados da consulta:

<cfoutput query="Movies">
    #title#<BR>
</cfoutput>

Autenticação #

Muitas configurações de banco de dados exigem autenticação (na forma de nome de usuário e senha) antes que você possa consultar o banco de dados. Você pode fornecê-los usando os atributos de nome de usuário e senha.

Observação: o nome de usuário e a senha também podem ser configurados na fonte de dados no ColdFusion Administrator. Fornecer esses detalhes em sua consulta substitui o nome de usuário e a senha no ColdFusion Administrator.

<cfquery datasource="Entertainment" username="webuser" password="letmein">
    select *
    from Movies
</cfquery>

Consultas em cache #

Uma consulta em cache é uma consulta que tem seus resultados armazenados na memória do servidor. Os resultados são armazenados quando a consulta é executada pela primeira vez. A partir daí, sempre que essa consulta for solicitada novamente, o ColdFusion recuperará os resultados da memória.

Você pode armazenar em cache uma consulta usando o atributo cachedAfter. Se a consulta foi executada pela última vez após a data fornecida, os dados armazenados em cache serão usados. Caso contrário, a consulta é executada novamente.

<cfquery datasource="Entertainment" cachedAfter="July 20, 2016">
    select *
    from Movies
</cfquery>

Para que o cache seja usado e várias chamadas para o banco de dados sejam evitadas, a consulta atual deve usar a mesma instrução SQL, fonte de dados, nome de consulta, nome de usuário e senha que a consulta em cache usada. Isso inclui espaços em branco na consulta.

Como tal, as seguintes consultas criam caches diferentes, mesmo que os caracteres aparados sejam os mesmos e os resultados da consulta sejam idênticos:

<cfquery datasource="Entertainment" cachedAfter="July 20, 2016">
    select *
    from Movies
    <cfif false>
    where 1 = 1
    </cfif>
    <cfif true>
    where 1 = 1
    </cfif>
</cfquery>

<cfquery datasource="Entertainment" cachedAfter="July 20, 2016">
    select * 
    from Movies
    <cfif true>
    where 1 = 1
    </cfif>
    <cfif false>
    where 1 = 1
    </cfif>
</cfquery>

Limitando o número de registros retornados #

Você pode limitar o número de linhas a serem retornadas usando o atributo maxrows.

<cfquery datasource="Entertainment" maxrows="50">
    select *
    from Movies
</cfquery>

Tempos limite #

Você pode definir um limite de tempo limite usando o atributo timeout. Isso pode ser útil para evitar que as solicitações sejam executadas por muito mais tempo do que deveriam e impactando em todo o aplicativo como resultado.

O atributo timeout define o número máximo de segundos que cada ação de uma consulta pode executar antes de retornar um erro.

<cfquery datasource="Entertainment" timeout="30">
    select *
    from Movies
</cfquery>
