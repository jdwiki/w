Transformação de identidade usando a “regra de identidade” #

Este exemplo mostra a base de quase qualquer transformação XSLT e o padrão de design XSLT mais fundamental. Produzindo como saída um documento XML idêntico ao documento XML de origem.

Documento XML de origem:

<t>Hello, World!</t>

Transformação XSLT:

<xsl:stylesheet version="1.0"  xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
 <xsl:output omit-xml-declaration="yes" indent="yes"/>
 
  <xsl:template match="@*|node()">
    <xsl:copy>
      <xsl:apply-templates select="@*|node()"/>
    </xsl:copy>
  </xsl:template>
</xsl:stylesheet>

Resultado: Ao aplicar essa transformação em qualquer documento XML de origem, a saída é um documento XML idêntico ao documento XML de origem. Nesse caso:

<t>Hello, World!</t>

Observe:

  1. Usar e substituir a regra de identidade é o padrão de projeto XSLT mais fundamental. Isso leva a soluções simples, curtas e elegantes para tarefas fundamentais, como exclusão/inserção/renomeação de elementos e muito mais.

  2. A regra/modelo de identidade é aquela publicada na W3C XSLT 1.0 Specification

Instalação ou Configuração #

Um processador XSLT é necessário para realizar qualquer transformação XSLT. Geralmente pode ser instalado através do gerenciador de pacotes do sistema. Por exemplo. no Debian ele pode ser instalado com:

sudo apt-get install xsltproc

Transformação mínima “Hello World” #

Esta é uma transformação XSLT mínima possível. Ele produz o valor da string do documento XML de origem. O formato de saída é texto.

Documento XML de origem:

<t>Hello, World!</t>

Transformação XSLT:

<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:output method="text"/>
</xsl:stylesheet>

Resultado da aplicação da transformação no documento XML de origem especificado acima:

Hello, World!

Observe:

  1. Nenhuma declaração <xsl:template> é usada.

  2. O tipo de saída desejado é especificado na declaração <xsl:output>, como o valor de seu atributo method.

  3. Quando não há modelos correspondentes, o processador XSLT, seguindo as regras do modelo de processamento XSLT, aplica os modelos internos XSLT padrão e isso resulta na cópia para a saída da concatenação de todos os nós de texto na ordem do documento. Neste caso simples, o documento XML de origem tem apenas um nó de texto com valor de string a string “`Hello, World!”.
