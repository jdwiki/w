Comandos importantes #

Documentando seu trabalho #

Lembre-se do que você fez e retenha saídas longas que não podem ser mantidas no buffer do WinDbg. É sempre bom ter um log disponível para reproduzir as etapas de depuração, por exemplo. para fazer perguntas no Stack Overflow.

Comando Finalidade
.logopen crie um arquivo de log
.logclose feche o arquivo de log
.dump salvar arquivo de despejo de memória (instantâneo da sessão de depuração atual)

Trabalhando com símbolos #

Sem ou com símbolos incorretos, você pode receber informações erradas e ser enganado. Certifique-se de estar familiarizado com esses comandos antes de começar a trabalhar no WinDbg. Veja também Como configurar símbolos no WinDbg.

Comando Finalidade
.symfix definir ou adicionar símbolos ao caminho oficial de símbolos da Microsoft
.sympath definir ou adicionar símbolos próprios ou de terceiros
.recarregar recarregar símbolos
.symopt definir opções de manipulação de símbolos
!sym controle de carregamento do símbolo
x examinar símbolos
ln lista os símbolos mais próximos

Análise de falhas #

Descubra o que aconteceu (em despejos de memória) e como lidar com eventos (em depuração ao vivo).

Comando Finalidade
.exr exibir registro de exceção
.lastevent exibir último evento
sx definir tratamento de exceção
!analisar analisar um travamento ou travamento
!avrf verificador de aplicativos

O ambiente #

Verifique o nome do processo e as informações da versão.

Comando Finalidade
| (tubulação) informações do processo
lm lista de módulos

Threads, pilhas de chamadas, registradores e memória #

Inspecione os detalhes.

Comando Finalidade
~ lista de tópicos
r registros
k pilha de chamadas
d* memória de exibição
e* editar memória
s memória de pesquisa
.formatos converter entre formatos numéricos
? avaliar expressão
u* desmontar
a montar
!endereço informações de memória

Controlando o alvo #

Na depuração ao vivo, assuma o controle da execução.

Comando Finalidade
g ir / continuar
gu subir
p passo único
t trace (passo único e registradores de saída)
bp definir ponto de interrupção
bl lista de pontos de interrupção

Trabalhando com extensões #

As extensões podem fornecer vantagens e aprimoramentos significativos.

Comando Finalidade
.load extensão de carga (caminho completo)
.loadby carrega a extensão relativa ao módulo
.cadeia exibir extensões carregadas
.descarregar descarregar extensão

Parar de depuração #

Comando Finalidade
q encerrar e encerrar o aplicativo
qd desanexar e sair

Anexar e desanexar #

Comando Finalidade
.tlist lista de processos
.anexar anexar ao processo
.criar crie um processo e anexe
.childdbg definir o comportamento de depuração do processo filho
.detach desanexar de um processo
.kill matar um processo
.reiniciar reinicie o processo

Comportamento do WinDbg #

Comando Finalidade
.prefer_dml definir manipulação de linguagem de marcação do depurador
.efmach mudar o bitness

Comandos de usabilidade #

Comando Finalidade
.cmdtree Carrega um arquivo de texto com comandos predefinidos em uma janela separada

Obtendo ajuda #

Comando Finalidade
.hh Exibe o manual de ajuda para comandos do WinDbg

Crie uma janela de comando personalizada no Windbg #

O comando .cmdtree permite abrir um arquivo .txt com comandos predefinidos que você pode simplesmente clicar duas vezes para executar.

Como criar arquivo de comando

Crie o arquivo usando este modelo

windbg ANSI Command Tree 1.0    
title {"Window title"}
body
{"Group Heading"}
 {"Name of command to display"} {"command"}
 {"Name of command to display"} {"command"}
{"Group Heading"}
 {"Name of command to display"} {"command"}

Coisas para cuidar

  1. O formato do template deve ser seguido com precisão para abrir o arquivo no Windbg.
  2. A nova linha é necessária após cada {Group Heading}.
  3. Cada par {Nome do comando a ser exibido} {comando} deve estar em uma linha e deve ser seguido por uma nova linha.

Exemplo de arquivo de comando personalizado

windbg ANSI Command Tree 1.0    
title {"Your title goes here"}
body
{"Basic commands"}
 {"Show CLR Version"} {"lmv m clr"}
 {"Load SOS from CLR"} {".loadby sos clr "}
{"Symbols"}
 {"Load my symbols"} {".sympath+ "c:\DebugSymbols" ; .reload"}

Como abrir a interface do usuário de comando a partir da janela de comando

Execute .cmdtree <path of your .txt file> para abrir a janela. Você verá uma janela como esta

Dê um duplo clique no comando para executar.
