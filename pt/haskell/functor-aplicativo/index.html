Functor Aplicativo

Aplicativo é a classe de tipos f :: * -> * que permite a aplicação de funções levantadas sobre uma estrutura onde a função também está embutida nessa estrutura.

Definição #

class Functor f => Applicative f where
    pure  :: a -> f a
    (<*>) :: f (a -> b) -> f a -> f b

Observe a restrição Functor em f. A função pure retorna seu argumento embutido na estrutura Aplicative. A função infixa <*> (pronunciada “apply”) é muito semelhante a fmap, exceto com a função incorporada na estrutura Aplicative.

Uma instância correta de Aplicativo deve satisfazer as leis de aplicativos, embora elas não sejam impostas pelo compilador:

pure id <*> a = a                              -- identity
pure (.) <*> a <*> b <*> c = a <*> (b <*> c)   -- composition
pure f <*> pure a = pure (f a)                 -- homomorphism
a <*> pure b = pure ($ b) <*> a                -- interchange

Definição alternativa #

Como todo Functor de Aplicativo é um Functor, fmap sempre pode ser usado nele; assim, a essência do Applicative é o emparelhamento dos conteúdos transportados, bem como a capacidade de criá-los:

class Functor f => PairingFunctor f where
  funit :: f ()                  -- create a context, carrying nothing of import
  fpair :: (f a,f b) -> f (a,b)  -- collapse a pair of contexts into a pair-carrying context

Esta classe é isomórfica para Aplicativo.

pure a = const a <$> funit = a <$ funit  

fa <*> fb = (\(a,b) -> a b) <$> fpair (fa, fb) = uncurry ($) <$> fpair (fa, fb)

Por outro lado,

funit = pure ()

fpair (fa, fb) = (,) <$> fa <*> fb

Instâncias comuns do aplicativo #

Pode ser #

Maybe é um functor de aplicativo contendo um valor possivelmente ausente.

instance Applicative Maybe where
    pure = Just
    
    Just f <*> Just x = Just $ f x
    _ <*> _ = Nothing

pure eleva o valor dado para Maybe aplicando Just a ele. A função (<*>) aplica uma função envolvida em um Maybe a um valor em um Maybe. Se a função e o valor estiverem presentes (construídos com Just), a função será aplicada ao valor e o resultado agrupado será retornado. Se algum deles estiver ausente, o cálculo não poderá prosseguir e Nothing será retornado.

Listas #

Uma maneira de as listas se ajustarem à assinatura de tipo <*> :: [a -> b] -> [a] -> [b] é pegar o produto cartesiano das duas listas, emparelhando cada elemento da primeira lista com cada elemento do segundo:

fs <*> xs = [f x | f <- fs, x <- xs]
         -- = do { f <- fs; x <- xs; return (f x) }

pure x = [x]

Isso geralmente é interpretado como emulação de não determinismo, com uma lista de valores representando um valor não determinístico cujos valores possíveis variam nessa lista; portanto, uma combinação de dois valores não determinísticos abrange todas as combinações possíveis dos valores nas duas listas:

ghci> [(+1),(+2)] <*> [3,30,300]
[4,31,301,5,32,302]

Fluxos infinitos e listas zip #

Existe uma classe de Aplicativos que “zipa” suas duas entradas juntas. Um exemplo simples é o de fluxos infinitos:

data Stream a = Stream { headS :: a, tailS :: Stream a }

A instância Aplicative de Stream aplica um fluxo de funções a um fluxo de argumentos ponto a ponto, emparelhando os valores nos dois fluxos por posição. pure retorna um fluxo constante – uma lista infinita de um único valor fixo:

instance Applicative Stream where
    pure x = let s = Stream x s in s
    Stream f fs <*> Stream x xs = Stream (f x) (fs <*> xs)

As listas também admitem uma instância Aplicativa “zippy”, para a qual existe o novo tipo ZipList:

newtype ZipList a = ZipList { getZipList :: [a] }

instance Applicative ZipList where
    ZipList xs <*> ZipList ys = ZipList $ zipWith ($) xs ys

Como o zip corta seu resultado de acordo com a entrada mais curta, a única implementação de pure que satisfaz as leis do Aplicativo é aquela que retorna uma lista infinita:

    pure a = ZipList (repeat a)   -- ZipList (fix (a:)) = ZipList [a,a,a,a,...

Por exemplo:

ghci> getZipList $ ZipList [(+1),(+2)] <*> ZipList [3,30,300]
[4,32]

As duas possibilidades nos lembram o produto externo e interno, semelhante a multiplicar uma matriz de 1 coluna (n x 1) por uma de 1 linha (1 x m) no primeiro caso, obtendo a matriz n x m como resultado (mas achatado); ou multiplicando matrizes de 1 linha e 1 coluna (mas sem somar) no segundo caso.

Funções #

Quando especializado para funções (->) r, as assinaturas de tipo de pure e <*> correspondem às dos combinadores K e S, respectivamente:

pure :: a -> (r -> a)
<*> :: (r -> (a -> b)) -> (r -> a) -> (r -> b)

pure deve ser const, e <*> pega um par de funções e aplica cada uma a um argumento fixo, aplicando os dois resultados:

instance Applicative ((->) r) where
    pure = const
    f <*> g = \x -> f x (g x)

As funções são o protótipo do aplicativo “zippy”. Por exemplo, como fluxos infinitos são isomórficos a (->) Nat, …

-- | Index into a stream
to :: Stream a -> (Nat -> a)
to (Stream x xs) Zero = x
to (Stream x xs) (Suc n) = to xs n

-- | List all the return values of the function in order
from :: (Nat -> a) -> Stream a
from f = from' Zero
    where from' n = Stream (f n) (from' (Suc n))

… a representação de fluxos de uma forma de ordem superior produz a instância Aplicative compacta automaticamente.
