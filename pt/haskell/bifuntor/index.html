Sintaxe #

  • bimap :: (a -> b) -> (c -> d) -> p a c -> p b d
  • primeiro :: (a -> b) -> p a c -> p b c
  • segundo :: (b -> c) -> p a b -> p a c

Um Functor comum é covariante em um parâmetro de tipo único. Por exemplo, se f é um Functor, então dado um f a, e uma função da forma a -> b, pode-se obter um f b (através do uso de fmap).

Um Bifunctor é covariante em dois parâmetros de tipo. Se f for um Bifunctor, então dado um f a b, e duas funções, uma de a -> c, e outra de b -> d, então pode-se obter um f c d (usando bimapa).

first deve ser pensado como um fmap sobre o primeiro parâmetro de tipo, second como um fmap sobre o segundo, e bimap deve ser concebido como mapeando duas funções covariavelmente sobre o primeiro e segundo parâmetros de tipo, respectivamente.

Instâncias comuns de Bifunctor #

Tuplas de dois elementos #

(,) é um exemplo de um tipo que tem uma instância Bifunctor.

instance Bifunctor (,) where
    bimap f g (x, y) = (f x, g y)

bimap pega um par de funções e as aplica aos respectivos componentes da tupla.

bimap (+ 2) (++ "nie") (3, "john") --> (5,"johnnie")
bimap ceiling length (3.5 :: Double, "john" :: String) --> (4,4)

Tanto #

A instância de Either de Bifunctor seleciona uma das duas funções a serem aplicadas dependendo se o valor é Left ou Right.

instance Bifunctor Either where
    bimap f g (Left x) = Left (f x)
    bimap f g (Right y) = Right (g y)

Definição de Bifuntor #

Bifunctor é a classe de tipos com dois parâmetros de tipo (f :: * -> * -> *), os quais podem ser mapeados de forma covariante simultaneamente.

class Bifunctor f where
    bimap :: (a -> c) -> (b -> d) -> f a b -> f c d

bimap pode ser pensado como a aplicação de um par de operações fmap a um tipo de dados.

Uma instância correta de Bifunctor para um tipo f deve satisfazer as leis bifunctor, que são análogas às [leis functor](https://www.wikiod.com/pt/haskell/functor#Definição de Classe de Functor e Leis -functor#t=201611271637470659956):

bimap id id = id  -- identity
bimap (f . g) (h . i) = bimap f h . bimap g i  -- composition

A classe Bifunctor é encontrada no módulo Data.Bifunctor. Para versões do GHC >7.10, este módulo é empacotado com o compilador; para versões anteriores você precisa instalar o pacote bifunctors.

primeiro e segundo #

Se o mapeamento covariante sobre apenas o primeiro argumento, ou apenas o segundo argumento, é desejado, então first ou second deve ser usado (em vez de bimap).

first :: Bifunctor f => (a -> c) -> f a b -> f c b
first f = bimap f id

second :: Bifunctor f => (b -> d) -> f a b -> f a d
second g = bimap id g

Por exemplo,

ghci> second (+ 2) (Right 40)
Right 42
ghci> second (+ 2) (Left "uh oh")
Left "uh oh"
