Mônadas grátis dividem cálculos monádicos em estruturas de dados e interpretadores #

Por exemplo, uma computação envolvendo comandos para ler e escrever no prompt:

Primeiro, descrevemos os “comandos” de nossa computação como um tipo de dados Functor

{-# LANGUAGE DeriveFunctor #-}

data TeletypeF next
    = PrintLine String next
    | ReadLine (String -> next)
    deriving Functor

Então usamos Free para criar a “Free Monad over TeletypeF” e construir algumas operações básicas.

import Control.Monad.Free (Free, liftF, iterM)

type Teletype = Free TeletypeF

printLine :: String -> Teletype ()
printLine str = liftF (PrintLine str ())

readLine :: Teletype String
readLine = liftF (ReadLine id)

Como Free f é uma Monad sempre que f for um Functor, podemos usar os combinadores padrão Monad (incluindo a notação do) para construir cálculos Teletype.

import Control.Monad -- we can use the standard combinators

echo :: Teletype ()
echo = readLine >>= printLine

mockingbird :: Teletype a
mockingbird = forever echo

Finalmente, escrevemos um “intérprete” transformando os valores Teletype a em algo que sabemos como trabalhar como IO a

interpretTeletype :: Teletype a -> IO a
interpretTeletype = foldFree run where
  run :: TeletypeF a -> IO a
  run (PrintLine str x) = putStrLn *> return x
  run (ReadLine f) = fmap f getLine

Que podemos usar para “executar” a computação Teletype a em IO

> interpretTeletype mockingbird
hello
hello
goodbye
goodbye
this will go on forever
this will go on forever

A mônada mais livre #

Existe uma formulação alternativa da mônada livre chamada Mônada Mais Livre (ou Prompt, ou Operacional). A mônada Freer não requer uma instância Functor para seu conjunto de instruções subjacente e tem uma estrutura semelhante a uma lista mais reconhecível do que a mônada livre padrão.

A mônada Freer representa programas como uma seqüência de instruções atômicas pertencentes ao conjunto de instruções i :: * -> *. Cada instrução usa seu parâmetro para declarar seu tipo de retorno. Por exemplo, o conjunto de instruções básicas para a mônada State é o seguinte:

data StateI s a where
    Get :: StateI s s  -- the Get instruction returns a value of type 's'
    Put :: s -> StateI s ()  -- the Put instruction contains an 's' as an argument and returns ()

O sequenciamento dessas instruções ocorre com o construtor :>>=. :>>= pega uma única instrução retornando um a e a anexa ao resto do programa, canalizando seu valor de retorno para a continuação. Em outras palavras, dada uma instrução retornando um a, e uma função para transformar um a em um programa retornando um b, :>>= produzirá um programa retornando um b.

data Freer i a where
    Return :: a -> Freer i a
    (:>>=) :: i a -> (a -> Freer i b) -> Freer i b

Observe que a é quantificado existencialmente no construtor :>>=. A única maneira de um interpretador aprender o que é a é por correspondência de padrões no GADT i.

Aparte: O lema co-Yoneda nos diz que Freer é isomórfico a Free. Lembre-se da definição do functor CoYoneda:

dados CoYoneda i b onde CoYoneda :: i a -> (a -> b) -> CoYoneda i b

Freer i é equivalente a Free (CoYoneda i). Se você pegar os [construtores do Free](https://www.wikiod.com/pt/haskell/monadas-gratis#Mônadas grátis são como pontos fixos) e definir f ~ CoYoneda i, você obtém:

Puro :: a -> Gratuito (CoYoneda i) a Gratuito :: CoYoneda i (Gratuito (CoYoneda i) b) -> Gratuito (CoYoneda i) b ~ i a -> (a -> Gratuito (CoYoneda i) b) -> Gratuito (CoYoneda i) b

do qual podemos recuperar os construtores do Freer i apenas definindo Freer i ~ Free (CoYoneda i).

Como CoYoneda i é um Functor para qualquer i, Freer é uma Monad para qualquer i, mesmo que i não seja um Functor.

instance Monad (Freer i) where
    return = Return
    Return x >>= f = f x
    (i :>>= g) >>= f = i :>>= fmap (>>= f) g  -- using `(->) r`'s instance of Functor, so fmap = (.)

Os intérpretes podem ser construídos para o Freer mapeando instruções para algum manipulador mônada.

foldFreer :: Monad m => (forall x. i x -> m x) -> Freer i a -> m a
foldFreer eta (Return x) = return x
foldFreer eta (i :>>= f) = eta i >>= (foldFreer eta . f)

Por exemplo, podemos interpretar a mônada Freer (StateI s) usando a mônada State s regular como manipulador:

runFreerState :: Freer (StateI s) a -> s -> (a, s)
runFreerState = State.runState . foldFreer toState
    where toState :: StateI s a -> State s a
          toState Get = State.get
          toState (Put x) = State.put x

Mônadas grátis são como pontos fixos #

Compare a definição de Free com a de [Fix](https://www.wikiod.com/pt/haskell/esquemas-de-recursao#Pontos fixos):

data Free f a = Return a
              | Free (f (Free f a))

newtype Fix f = Fix { unFix :: f (Fix f) }

Em particular, compare o tipo do construtor Free com o tipo do construtor Fix. Free sobrepõe um functor como Fix, exceto que Free tem um caso adicional Return a.

Como o foldFree e o iterM funcionam? #

Existem algumas funções para ajudar a derrubar cálculos Free interpretando-os em outra mônada m: iterM :: (Functor f, Monad m) => (f (m a) -> m a) -> (Free f a - > m a) e foldFree :: Monad m => (forall x. f x -> m x) -> (Free f a -> m a). O que eles estão fazendo?

Primeiro vamos ver o que seria necessário para desmontar e interpretar uma função Teletype a em IO manualmente. Podemos ver Free f a como sendo definido

data Free f a 
    = Pure a 
    | Free (f (Free f a))

O caso Pure é fácil:

interpretTeletype :: Teletype a -> IO a
interpretTeletype (Pure x) = return x
interpretTeletype (Free teletypeF) = _

Agora, como interpretar uma computação Teletype que foi construída com o construtor Free? Gostaríamos de chegar a um valor do tipo IO a examinando teletypeF :: TeletypeF (Teletype a). Para começar, vamos escrever uma função runIO :: TeletypeF a -> IO a que mapeia uma única camada da mônada livre para uma ação IO:

runIO :: TeletypeF a -> IO a
runIO (PrintLine msg x) = putStrLn msg *> return x
runIO (ReadLine k) = fmap k getLine

Agora podemos usar runIO para preencher o resto de interpretTeletype. Lembre-se que teletypeF :: TeletypeF (Teletype a) é uma camada do functor TeletypeF que contém o resto da computação Free. Usaremos runIO para interpretar a camada mais externa (então temos runIO teletypeF :: IO (Teletype a)) e então usaremos o combinador >>= da mônada IO para interpretar o Teletype a retornado .

interpretTeletype :: Teletype a -> IO a
interpretTeletype (Pure x) = return x
interpretTeletype (Free teletypeF) = runIO teletypeF >>= interpretTeletype

A definição de foldFree é apenas a de interpretTeletype, exceto que a função runIO foi fatorada. Como resultado, o foldFree funciona independentemente de qualquer functor base em particular e de qualquer mônada alvo.

foldFree :: Monad m => (forall x. f x -> m x) -> Free f a -> m a
foldFree eta (Pure x) = return x
foldFree eta (Free fa) = eta fa >>= foldFree eta

foldFree tem um tipo de rank 2: eta é uma transformação natural. Poderíamos ter dado a foldFree um tipo de Monad m => (f (Free f a) -> m (Free f a)) -> Free f a -> m a, mas isso dá eta a liberdade de inspecionar o Computação livre dentro da camada f. Dar ao foldFree este tipo mais restritivo garante que eta possa processar apenas uma única camada de cada vez.

iterM dá a função de dobrar a capacidade de examinar a subcomputação. O resultado (monádico) da iteração anterior está disponível para a próxima, dentro do parâmetro f. iterM é análogo a um [paramorphism](https://www.wikiod.com/pt/haskell/esquemas-de-recursao#Recursão primitiva) enquanto foldFree é como um [catamorphism]( https://www.wikiod.com/pt/haskell/esquemas-de-recursao#Dobrando uma estrutura uma camada de cada vez).

iterM :: (Monad m, Functor f) => (f (m a) -> m a) -> Free f a -> m a
iterM phi (Pure x) = return x
iterM phi (Free fa) = phi (fmap (iterM phi) fa)
