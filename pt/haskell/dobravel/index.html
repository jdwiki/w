Foldable é a classe de tipos t :: * -> * que admite uma operação folding. Uma dobra agrega os elementos de uma estrutura em uma ordem bem definida, usando uma função de combinação.

Se t for Foldable significa que para qualquer valor t a sabemos como acessar todos os elementos de a de “dentro” de t a em uma ordem linear fixa. Este é o significado de foldMap :: Monoid m => (a -> m) -> (t a -> m): nós “visitamos” cada elemento com uma função de resumo e esmagamos todos os resumos juntos. A ordem do `Monoid’s respeita (mas são invariáveis ​​para diferentes agrupamentos).

Uma instância de Foldable para uma árvore binária #

Para instanciar Foldable você precisa fornecer uma definição para pelo menos foldMap ou foldr.

data Tree a = Leaf
            | Node (Tree a) a (Tree a)

instance Foldable Tree where
    foldMap f Leaf = mempty
    foldMap f (Node l x r) = foldMap f l `mappend` f x `mappend` foldMap f r
    
    foldr f acc Leaf = acc
    foldr f acc (Node l x r) = foldr f (f x (foldr f acc r)) l

Esta implementação executa um percurso em ordem da árvore.

ghci> let myTree = Node (Node Leaf 'a' Leaf) 'b' (Node Leaf 'c' Leaf)

--    +--'b'--+
--    |       |
-- +-'a'-+ +-'c'-+
-- |     | |     |
-- *     * *     *

ghci> toList myTree
"abc"

A extensão DeriveFoldable permite que o GHC gere instâncias Foldable com base na estrutura do tipo. Podemos variar a ordem do percurso escrito por máquina ajustando o layout do construtor Node.

data Inorder a = ILeaf
               | INode (Inorder a) a (Inorder a)  -- as before
               deriving Foldable

data Preorder a = PrLeaf
                | PrNode a (Preorder a) (Preorder a)
                deriving Foldable

data Postorder a = PoLeaf
                 | PoNode (Postorder a) (Postorder a) a
                 deriving Foldable

-- injections from the earlier Tree type
inorder :: Tree a -> Inorder a
inorder Leaf = ILeaf
inorder (Node l x r) = INode (inorder l) x (inorder r)

preorder :: Tree a -> Preorder a
preorder Leaf = PrLeaf
preorder (Node l x r) = PrNode x (preorder l) (preorder r)

postorder :: Tree a -> Postorder a
postorder Leaf = PoLeaf
postorder (Node l x r) = PoNode (postorder l) (postorder r) x

ghci> toList (inorder myTree)
"abc"
ghci> toList (preorder myTree)
"bac"
ghci> toList (postorder myTree)
"acb"

Definição de Dobrável #

class Foldable t where
    {-# MINIMAL foldMap | foldr #-}

    foldMap :: Monoid m => (a -> m) -> t a -> m
    foldMap f = foldr (mappend . f) mempty

    foldr :: (a -> b -> b) -> b -> t a -> b
    foldr f z t = appEndo (foldMap (Endo #. f) t) z

    -- and a number of optional methods

Intuitivamente (embora não tecnicamente), estruturas dobráveis são recipientes de elementos a que permitem o acesso a seus elementos em uma ordem bem definida. A operação foldMap mapeia cada elemento do contêiner para um Monoid e os recolhe usando a estrutura Monoid.

Contando os elementos de uma estrutura dobrável #

length conta as ocorrências dos elementos a em uma estrutura dobrável t a.

ghci> length [7, 2, 9]  -- t ~ []
3
ghci> length (Right 'a')  -- t ~ Either e
1  -- 'Either e a' may contain zero or one 'a'
ghci> length (Left "foo")  -- t ~ Either String
0
ghci> length (3, True)  -- t ~ (,) Int
1  -- '(c, a)' always contains exactly one 'a'

length é definido como sendo equivalente a:

class Foldable t where
    -- ...
    length :: t a -> Int
    length = foldl' (\c _ -> c+1) 0

Observe que este tipo de retorno Int restringe as operações que podem ser realizadas em valores obtidos por chamadas para a função length. fromIntegralé uma função útil que nos permite lidar com este problema.

Dobrando uma estrutura ao contrário #

Qualquer dobra pode ser executada na direção oposta com a ajuda do monóide Dual, que inverte um monóide existente para que a agregação volte.

newtype Dual a = Dual { getDual :: a }

instance Monoid m => Monoid (Dual m) where
    mempty = Dual mempty
    (Dual x) `mappend` (Dual y) = Dual (y `mappend` x)

Quando o monóide subjacente de uma chamada foldMap é invertido com Dual, a dobra é executada para trás; o seguinte tipo Reverse é definido em Data.Functor.Reverse:

newtype Reverse t a = Reverse { getReverse :: t a }

instance Foldable t => Foldable (Reverse t) where
    foldMap f = getDual . foldMap (Dual . f) . getReverse

Podemos usar esse mecanismo para escrever um reverse conciso para listas:

reverse :: [a] -> [a]
reverse = toList . Reverse

Achatando uma estrutura dobrável em uma lista #

toList nivela uma estrutura t a dobrável em uma lista de as.

ghci> toList [7, 2, 9]  -- t ~ []
[7, 2, 9]
ghci> toList (Right 'a')  -- t ~ Either e
"a"
ghci> toList (Left "foo")  -- t ~ Either String
[]
ghci> toList (3, True)  -- t ~ (,) Int
[True]

toList é definido como sendo equivalente a:

class Foldable t where
    -- ...
    toList :: t a -> [a]
    toList = foldr (:) []

Executando um efeito colateral para cada elemento de uma estrutura dobrável #

traverse_ executa uma ação Aplicativo para cada elemento em uma estrutura dobrável. Ignora o resultado da ação, mantendo apenas os efeitos colaterais. (Para uma versão que não descarta resultados, use Traversable.)

-- using the Writer applicative functor (and the Sum monoid)
ghci> runWriter $ traverse_ (\x -> tell (Sum x)) [1,2,3]
((),Sum {getSum = 6})
-- using the IO applicative functor
ghci> traverse_ putStrLn (Right "traversing")
traversing
ghci> traverse_ putStrLn (Left False)
-- nothing printed

for_ é traverse_ com os argumentos invertidos. Assemelha-se a um loop foreach em uma linguagem imperativa.

ghci> let greetings = ["Hello", "Bonjour", "Hola"]
ghci> :{
ghci|     for_ greetings $ \greeting -> do
ghci|         print (greeting ++ " Stack Overflow!")
ghci| :}
"Hello Stack Overflow!"
"Bonjour Stack Overflow!"
"Hola Stack Overflow!"

sequenceA_ recolhe um Foldable cheio de ações Aplicative em uma única ação, ignorando o resultado.

ghci> let actions = [putStrLn "one", putStLn "two"]
ghci> sequenceA_ actions
one
two

traverse_ é definido como sendo equivalente a:

traverse_ :: (Foldable t, Applicative f) => (a -> f b) -> t a -> f ()
traverse_ f = foldr (\x action -> f x *> action) (pure ())

sequenceA_ é definido como:

sequenceA_ :: (Foldable t, Applicative f) -> t (f a) -> f ()
sequenceA_ = traverse_ id

Além disso, quando o Foldable também é um Functor, traverse_ e sequenceA_ têm a seguinte relação:

traverse_ f = sequenceA_ . fmap f

Achatando uma estrutura dobrável em um monóide #

foldMap mapeia cada elemento da estrutura Foldable para um Monoid, e então os combina em um único valor.

foldMap e foldr podem ser definidos em termos um do outro, o que significa que instâncias de Foldable precisam apenas dar uma definição para um deles.

class Foldable t where
    foldMap :: Monoid m => (a -> m) -> t a -> m
    foldMap f = foldr (mappend . f) mempty

Exemplo de uso com [o monóide Product](https://www.wikiod.com/pt/haskell/monoide#Monóides Numéricos):

product :: (Num n, Foldable t) => t n -> n
product = getProduct . foldMap Product

Verificando se uma estrutura dobrável está vazia #

null retorna True se não houver elementos a em uma estrutura dobrável t a, e False se houver um ou mais. Estruturas para as quais null é True têm um [length](https://www.wikiod.com/pt/haskell/dobravel#Contando os elementos de uma estrutura dobrável t=201608081807124789244) de 0.

ghci> null []
True
ghci> null [14, 29]
False
ghci> null Nothing
True
ghci> null (Right 'a')
False
ghci> null ('x', 3)
False

null é definido como sendo equivalente a:

class Foldable t where
    -- ...
    null :: t a -> Bool
    null = foldr (\_ _ -> False) True
