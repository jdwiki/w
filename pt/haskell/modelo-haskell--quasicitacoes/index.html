O que é o modelo Haskell? #

Template Haskell refere-se aos recursos de meta-programação de templates embutidos no GHC Haskell. O artigo que descreve a implementação original pode ser encontrado aqui.

O que são estágios? (Ou, qual é a restrição de estágio?) #

Os estágios referem-se a quando o código é executado. Normalmente, o código é executado apenas em tempo de execução, mas com o Template Haskell, o código pode ser executado em tempo de compilação. O código “normal” é o estágio 0 e o código de tempo de compilação é o estágio 1.

A restrição de estágio refere-se ao fato de que um programa de estágio 0 não pode ser executado no estágio 1 - isso seria equivalente a poder executar qualquer programa regular (não apenas meta-programa) em tempo de compilação.

Por convenção (e para simplificar a implementação), o código no módulo atual é sempre o estágio 0 e o código importado de todos os outros módulos é o estágio 1. Por esse motivo, apenas expressões de outros módulos podem ser emendadas.

Observe que um programa de estágio 1 é uma expressão de estágio 0 do tipo Q Exp, Q Type, etc; mas o inverso não é verdadeiro - nem todo valor (programa de estágio 0) do tipo Q Exp é um programa de estágio 1,

Além disso, como as emendas podem ser aninhadas, os identificadores podem ter estágios maiores que 1. A restrição de estágio pode então ser generalizada - um programa de estágio n não pode ser executado em nenhum estágio m>n. Por exemplo, pode-se ver referências a tais estágios maiores que 1 em certas mensagens de erro:

>:t [| \x -> $x |]

<interactive>:1:10: error:
    * Stage error: `x' is bound at stage 2 but used at stage 1
    * In the untyped splice: $x
      In the Template Haskell quotation [| \ x -> $x |]

Usar o Template Haskell causa erros fora do escopo de identificadores não relacionados? #

Normalmente, todas as declarações em um único módulo Haskell podem ser consideradas como sendo mutuamente recursivas. Em outras palavras, cada declaração de nível superior está no escopo de todas as outras em um único módulo. Quando o Template Haskell está habilitado, as regras de escopo mudam - o módulo é dividido em grupos de código separados por emendas TH, e cada grupo é mutuamente recursivo e cada grupo no escopo de todos os outros grupos.

Sintaxe do Template Haskell e Quasiquotes #

Template Haskell é habilitado pela extensão GHC -XTemplateHaskell. Esta extensão habilita todos os recursos sintáticos detalhados nesta seção. Os detalhes completos sobre o Template Haskell são fornecidos pelo guia do usuário.

Emendas #

  • Uma emenda é uma nova entidade sintática habilitada pelo Template Haskell, escrita como $(...), onde (...) é alguma expressão.

  • Não deve haver espaço entre $ e o primeiro caractere da expressão; e o Template Haskell sobrescreve a análise do operador $ - por exemplo f$g é normalmente analisado como ($) f g enquanto que com o Template Haskell ativado, é analisado como uma emenda.

  • Quando uma emenda aparece no nível superior, o $ pode ser omitido. Nesse caso, a expressão emendada é a linha inteira.

  • Uma emenda representa o código que é executado em tempo de compilação para produzir um Haskell AST, e esse AST é compilado como código Haskell e inserido no programa

  • As emendas podem aparecer no lugar de: expressões, padrões, tipos e declarações de nível superior. O tipo da expressão emendada, em cada caso, respectivamente, é Q Exp, Q Pat, Q Type, Q [Decl]. Observe que as emendas de declaração podem somente aparecer no nível superior, enquanto as outras podem estar dentro de outras expressões, padrões ou tipos, respectivamente.

Citações de expressão (nota: não uma quase citação) #

  • Uma citação de expressão é uma nova entidade sintática escrita como uma das seguintes:

  • [e|..|] ou [|..|] - .. é uma expressão e a cotação tem o tipo Q Exp;

  • [p|..|] - .. é um padrão e a cotação tem o tipo Q Pat;

  • [t|..|] - .. é um tipo e a cotação tem o tipo Q Type;

  • [d|..|] - .. é uma lista de declarações e a citação tem o tipo Q [Dec].

  • Uma cotação de expressão pega um programa em tempo de compilação e produz o AST representado por esse programa.

  • O uso de um valor em uma cotação (por exemplo, \x -> [| x |]) sem emenda corresponde ao açúcar sintático para \x -> [| $(lift x) |], onde lift :: Lift t => t -> Q Exp vem da classe

    class Lift t where
      lift :: t -> Q Exp
      default lift :: Data t => t -> Q Exp

Emendas digitadas e cotações #

  • As emendas tipadas são semelhantes às emendas mencionadas anteriormente (não digitadas) e são escritas como $$(..) onde (..) é uma expressão.

  • Se e tem o tipo Q (TEp a) então $$e tem o tipo a.

  • As citações digitadas assumem a forma [||..||] onde .. é uma expressão do tipo a; a cotação resultante tem o tipo Q (TEp a).

  • Expressões digitadas podem ser convertidas em não tipadas: unType :: TExp a -> Exp.

QuasiCitações #

  • QuasiQuotes generalizam citações de expressão - anteriormente, o analisador usado pela expressão quote é um de um conjunto fixo (e,p,t,d), mas QuasiQuotes permite que um analisador personalizado seja definido e usado para produzir código em tempo de compilação . As quase-citações podem aparecer em todos os mesmos contextos que as citações regulares.

  • Uma quase aspas é escrita como [iden|...|], onde iden é um identificador do tipo Language.Haskell.TH.Quote.QuasiQuoter.

  • Um QuasiQuoter é composto simplesmente por quatro analisadores, um para cada um dos diferentes contextos em que as citações podem aparecer:

    data QuasiQuoter = QuasiQuoter { quoteExp  :: String -> Q Exp,
                                     quotePat  :: String -> Q Pat,
                                     quoteType :: String -> Q Type,
                                     quoteDec  :: String -> Q [Dec] }

Nomes #

  • Identificadores Haskell são representados pelo tipo Language.Haskell.TH.Syntax.Name. Os nomes formam as folhas das árvores sintáticas abstratas que representam os programas Haskell no Template Haskell.

  • Um identificador que está atualmente no escopo pode ser transformado em um nome com: 'e ou 'T. No primeiro caso, e é interpretado no escopo da expressão, enquanto no segundo caso T está no escopo do tipo (lembrando que tipos e construtores de valor podem compartilhar o nome sem ambiguidade em Haskell).

O tipo Q #

O construtor do tipo Q :: * -> * definido em Language.Haskell.TH.Syntax é um tipo abstrato que representa computações que têm acesso ao ambiente de tempo de compilação do módulo no qual a computação é executada. O tipo Q também lida com substituição de variáveis, chamada captura de nome por TH (e discutida aqui.) Todas as emendas têm o tipo Q X para alguns X.

O ambiente de tempo de compilação inclui:

  • identificadores no escopo e informações sobre esses identificadores,
  • tipos de funções
  • tipos e tipos de dados de origem dos construtores
  • especificação completa de declarações de tipo (classes, famílias de tipos)
  • a localização no código-fonte (linha, coluna, módulo, pacote) onde ocorre a emenda
  • fixidez de funções (GHC 7.10)
  • extensões GHC habilitadas (GHC 8.0)

O tipo Q também tem a capacidade de gerar novos nomes, com a função newName :: String -> Q Name. Observe que o nome não está vinculado a nenhum lugar implicitamente, portanto, o usuário deve vinculá-lo e, portanto, garantir que o uso resultante do nome seja bem definido é responsabilidade do usuário.

Q tem instâncias para Functor,Monad,Aplicative e esta é a interface principal para manipular valores Q, juntamente com os combinadores fornecidos em Language.Haskell.TH.Lib, que definem uma função auxiliar para cada construtor do TH ast do formulário:

LitE :: Lit -> Exp
litE :: Lit -> ExpQ

AppE :: Exp -> Exp -> Exp 
appE :: ExpQ -> ExpQ -> ExpQ

Observe que ExpQ, TypeQ, DecsQ e PatQ são sinônimos para os tipos AST que normalmente são armazenados dentro do tipo Q.

A biblioteca TH fornece uma função runQ :: Quasi m => Q a -> m a, e há uma instância Quasi IO, então parece que o tipo Q é apenas um IO sofisticado. No entanto, o uso de runQ :: Q a -> IO a produz uma ação IO que não tem acesso a nenhum ambiente de tempo de compilação - isso só está disponível no tipo Q real. Tais ações IO falharão em tempo de execução se tentar acessar o referido ambiente.

Um curry n-arity #

O conhecido

curry :: ((a,b) -> c) -> a -> b -> c
curry = \f a b -> f (a,b)

função pode ser generalizada para tuplas de aridade arbitrária, por exemplo:

curry3 :: ((a, b, c) -> d) -> a -> b -> c -> d
curry4 :: ((a, b, c, d) -> e) -> a -> b -> c -> d -> e

No entanto, escrever essas funções para tuplas de aridade 2 a (por exemplo) 20 manualmente seria tedioso (e ignorando o fato de que a presença de 20 tuplas em seu programa quase certamente sinaliza problemas de design que devem ser corrigidos com registros).

Podemos usar o Template Haskell para produzir funções curryN para n arbitrário:

{-# LANGUAGE TemplateHaskell #-}
import Control.Monad (replicateM) 
import Language.Haskell.TH (ExpQ, newName, Exp(..), Pat(..))
import Numeric.Natural (Natural) 

curryN :: Natural -> Q Exp

A função curryN recebe um número natural e produz a função curry dessa aridade, como um Haskell AST.

curryN n = do
  f  <- newName "f"
  xs <- replicateM (fromIntegral n) (newName "x")

Primeiro, produzimos variáveis ​​do tipo fresh para cada um dos argumentos da função - uma para a função de entrada e uma para cada um dos argumentos da referida função.

  let args = map VarP (f:xs)

A expressão args representa o padrão f x1 x2 .. xn. Observe que um padrão é uma entidade sintática separada - poderíamos pegar esse mesmo padrão e colocá-lo em um lambda, ou uma ligação de função, ou até mesmo o LHS de uma ligação let (o que seria um erro).

      ntup = TupE (map VarE xs)

A função deve construir a tupla de argumentos a partir da sequência de argumentos, que é o que nós fizemos aqui. Observe a distinção entre variáveis ​​de padrão (VarP) e variáveis ​​de expressão (VarE).

  return $ LamE args (AppE (VarE f) ntup)

Finalmente, o valor que produzimos é o AST \f x1 x2 .. xn -> f (x1, x2, .. , xn).

Também poderíamos ter escrito esta função usando aspas e construtores ’levantados':

...
import Language.Haskell.TH.Lib  

curryN' :: Natural -> ExpQ
curryN' n = do
  f  <- newName "f"
  xs <- replicateM (fromIntegral n) (newName "x")
  lamE (map varP (f:xs)) 
        [| $(varE f) $(tupE (map varE xs)) |]

Observe que as citações devem ser sintaticamente válidas, então [| \ $(map varP (f:xs)) -> .. |] é inválido, porque não há como no Haskell regular declarar uma ’lista’ de padrões - o acima é interpretado como \ var -> .. e espera-se que a expressão emendada tenha o tipo PatQ, ou seja, um único padrão, não uma lista de padrões.

Finalmente, podemos carregar esta função TH no GHCi:

>:set -XTemplateHaskell
>:t $(curryN 5)
$(curryN 5)
  :: ((t1, t2, t3, t4, t5) -> t) -> t1 -> t2 -> t3 -> t4 -> t5 -> t
>$(curryN 5) (\(a,b,c,d,e) -> a+b+c+d+e) 1 2 3 4 5
15
