Adição e multiplicação #

A adição e a multiplicação têm equivalentes neste tipo de álgebra. Eles correspondem aos sindicatos marcados e aos tipos de produto.

data Sum a b = A a | B b
data Prod a b = Prod a b

Podemos ver como o número de habitantes de cada tipo corresponde às operações da álgebra.

De forma equivalente, podemos usar Either e (,) como construtores de tipo para a adição e a multiplicação. Eles são isomórficos aos nossos tipos previamente definidos:

type Sum' a b = Either a b
type Prod' a b = (a,b)

Os resultados esperados de adição e multiplicação são seguidos pela álgebra de tipo até o isomorfismo. Por exemplo, podemos ver um isomorfismo entre 1 + 2, 2 + 1 e 3; como 1 + 2 = 3 = 2 + 1.

data Color = Red | Green | Blue

f :: Sum () Bool -> Color
f (Left ())     = Red
f (Right True)  = Green
f (Right False) = Blue

g :: Color -> Sum () Bool
g Red   = Left ()
g Green = Right True
g Blue  = Right False

f' :: Sum Bool () -> Color
f' (Right ())   = Red
f' (Left True)  = Green
f' (Left False) = Blue

g' :: Color -> Sum Bool ()
g' Red   = Right ()
g' Green = Left True
g' Blue  = Left False

Regras de adição e multiplicação #

As regras comuns de comutatividade, associatividade e distributividade são válidas porque existem isomorfismos triviais entre os seguintes tipos:

-- Commutativity
Sum a b           <=> Sum b a
Prod a b          <=> Prod b a
-- Associativity
Sum (Sum a b) c   <=> Sum a (Sum b c)
Prod (Prod a b) c <=> Prod a (Prod b c)
-- Distributivity
Prod a (Sum b c)  <=> Sum (Prod a b) (Prod a c)

Funções #

As funções podem ser vistas como exponenciais em nossa álgebra. Como podemos ver, se pegarmos um tipo a com n instâncias e um tipo b com m instâncias, o tipo a -> b terá m elevado a n instâncias.

Como exemplo, Bool -> Bool é isomórfico para (Bool,Bool), pois 2*2 = 2².

iso1 :: (Bool -> Bool) -> (Bool,Bool)
iso1 f = (f True,f False)

iso2 :: (Bool,Bool) -> (Bool -> Bool)
iso2 (x,y) = (\p -> if p then x else y)

Números naturais no tipo álgebra #

Podemos traçar uma conexão entre os tipos Haskell e os números naturais. Esta ligação pode ser feita atribuindo a cada tipo o número de habitantes que possui.

Tipos de união finita #

Para tipos finitos, basta ver que podemos atribuir um tipo natural a cada número, com base no número de construtores. Por exemplo:

type Color = Red | Yellow | Green

seria 3. E o tipo Bool seria 2.

type Bool = True | False

Unicidade até isomorfismo #

Vimos que vários tipos corresponderiam a um único número, mas neste caso seriam isomórficos. Isso quer dizer que haveria um par de morfismos f e g, cuja composição seria a identidade, conectando os dois tipos.

f :: a -> b
g :: b -> a

f . g == id == g . f

Neste caso, diríamos que os tipos são isomórficos. Consideraremos dois tipos iguais em nossa álgebra, desde que sejam isomórficos.

Por exemplo, duas representações diferentes do número dois são trivalmente isomórficas:

type Bit  = I    | O
type Bool = True | False

bitValue :: Bit -> Bool
bitValue I = True
bitValue O = False

booleanBit :: Bool -> Bit
booleanBit True  = I
booleanBit False = O

Porque podemos ver bitValue . booleanBit == id == booleanBit . bitValue

Um e Zero #

A representação do número 1 é obviamente um tipo com apenas um construtor. Em Haskell, este tipo é canonicamente o tipo (), chamado Unit. Todos os outros tipos com apenas um construtor são isomórficos a ().

E nossa representação de 0 será um tipo sem construtores. Este é o tipo Void em Haskell, conforme definido em Data.Void. Isso seria equivalente a um tipo desabitado, sem construtores de dados:

data Void

Tipos recursivos #

listas #

As listas podem ser definidas como:

data List a = Nil | Cons a (List a)

Se traduzirmos isso em nossa álgebra de tipos, obtemos

Lista(a) = 1 + a * Lista(a)

Mas agora podemos substituir List(a) novamente nesta expressão várias vezes, para obter:

Lista(a) = 1 + a + a*a + a*a*a + a*a*a*a + …

Isso faz sentido se vemos uma lista como um tipo que pode conter apenas um valor, como em []; ou todo valor do tipo a, como em [x]; ou dois valores do tipo a, como em [x,y]; e assim por diante. A definição teórica de List que devemos obter daí seria:

-- Not working Haskell code!
data List a = Nil
            | One a
            | Two a a
            | Three a a a 
            ...

Árvores #

Podemos fazer a mesma coisa com árvores binárias, por exemplo. Se os definirmos como:

data Tree a = Empty | Node a (Tree a) (Tree a)

Obtemos a expressão:

Árvore(a) = 1 + a * Árvore(a) * Árvore(a)

E se fizermos as mesmas substituições repetidas vezes, obteremos a seguinte sequência:

Árvore(a) = 1 + a + 2 (a*a) + 5 (a*a*a) + 14 (a*a*a*a) + …

Os coeficientes que obtemos aqui correspondem à sequência de números catalães, e o n-ésimo número catalão é precisamente o número de árvores binárias possíveis com n nós.

Derivados #

A derivada de um tipo é o tipo de seu tipo de contextos de um furo. Este é o tipo que obteríamos se fizéssemos uma variável de tipo desaparecer em todos os pontos possíveis e somarmos os resultados.

Como exemplo, podemos pegar o tipo triplo (a,a,a) e derivá-lo, obtendo

data OneHoleContextsOfTriple = (a,a,()) | (a,(),a) | ((),a,a)

Isso é coerente com nossa definição usual de derivação, como:

d/da (a*a*a) = 3*a*a
