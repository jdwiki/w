Gerando dados aleatoriamente para tipos personalizados #

A classe Arbitrary é para tipos que podem ser gerados aleatoriamente pelo QuickCheck.

A implementação mínima de Arbitrary é o método arbitrary, que é executado na mônada Gen para produzir um valor aleatório.

Aqui está uma instância de Arbitrary para o seguinte tipo de dados de listas não vazias.

import Test.QuickCheck.Arbitrary (Arbitrary(..))
import Test.QuickCheck.Gen (oneof)
import Control.Applicative ((<$>), (<*>))

data NonEmpty a = End a | Cons a (NonEmpty a)

instance Arbitrary a => Arbitrary (NonEmpty a) where
    arbitrary = oneof [  -- randomly select one of the cases from the list
        End <$> arbitrary,  -- call a's instance of Arbitrary
        Cons <$>
            arbitrary <*>  -- call a's instance of Arbitrary
            arbitrary  -- recursively call NonEmpty's instance of Arbitrary
        ]

Usando implicação (==>) para verificar propriedades com pré-condições #

prop_evenNumberPlusOneIsOdd :: Integer -> Property
prop_evenNumberPlusOneIsOdd x = even x ==> odd (x + 1)

Se você quiser verificar se uma propriedade é válida, dado que uma pré-condição é válida, você pode usar o operador ==>. Observe que, se for muito improvável que entradas arbitrárias correspondam à pré-condição, o QuickCheck pode desistir mais cedo.

prop_overlySpecific x y = x == 0 ==> x * y == 0

ghci> quickCheck prop_overlySpecific
*** Gave up! Passed only 31 tests.

Declarando uma propriedade #

Em sua forma mais simples, uma property é uma função que retorna um Bool.

prop_reverseDoesNotChangeLength xs = length (reverse xs) == length xs

Uma propriedade declara uma invariante de alto nível de um programa. O executor de teste QuickCheck avaliará a função com 100 entradas aleatórias e verificará se o resultado é sempre True.

Por convenção, funções que são propriedades têm nomes que começam com prop_.

Verificando uma única propriedade #

A função quickCheck testa uma propriedade em 100 entradas aleatórias.

ghci> quickCheck prop_reverseDoesNotChangeLength
+++ OK, passed 100 tests.

Se uma propriedade falhar para alguma entrada, quickCheck imprime um contra-exemplo.

prop_reverseIsAlwaysEmpty xs = reverse xs == []  -- plainly not true for all xs

ghci> quickCheck prop_reverseIsAlwaysEmpty
*** Failed! Falsifiable (after 2 tests):                  
[()]

Verificando todas as propriedades em um arquivo #

quickCheckAll é um auxiliar do Template Haskell que encontra todas as definições no arquivo atual cujo nome começa com prop_ e as testa.

{-# LANGUAGE TemplateHaskell #-}

import Test.QuickCheck (quickCheckAll)
import Data.List (sort)

idempotent :: Eq a => (a -> a) -> a -> Bool
idempotent f x = f (f x) == f x

prop_sortIdempotent = idempotent sort

-- does not begin with prop_, will not be picked up by the test runner
sortDoesNotChangeLength xs = length (sort xs) == length xs


return []
main = $quickCheckAll

Observe que a linha return [] é obrigatória. Torna as definições textualmente acima dessa linha visíveis para o Template Haskell.

$ runhaskell QuickCheckAllExample.hs
=== prop_sortIdempotent from tree.hs:7 ===
+++ OK, passed 100 tests.

Limitando o tamanho dos dados de teste #

Pode ser difícil testar funções com baixa complexidade assintótica usando verificação rápida, pois as entradas aleatórias geralmente não são limitadas por tamanho. Adicionando um limite superior ao tamanho da entrada, ainda podemos testar essas funções caras.

import Data.List(permutations)
import Test.QuickCheck

longRunningFunction :: [a] -> Int
longRunningFunction xs = length (permutations xs)

factorial :: Integral a => a -> a
factorial n = product [1..n]

prop_numberOfPermutations xs =
    longRunningFunction xs == factorial (length xs)

ghci> quickCheckWith (stdArgs { maxSize = 10}) prop_numberOfPermutations

Usando quickCheckWith com uma versão modificada de stdArgs podemos limitar o tamanho das entradas para no máximo 10. Neste caso, como estamos gerando listas, isso significa que geramos listas de até tamanho 10. Nossas permutações A função não demora muito para ser executada para essas listas curtas, mas ainda podemos estar razoavelmente confiantes de que nossa definição está correta.
