Functor é a classe de tipos f :: * -> * que pode ser mapeado de forma covariante. Mapear uma função sobre uma estrutura de dados aplica a função a todos os elementos da estrutura sem alterar a própria estrutura.

Um Functor pode ser pensado como um contêiner para algum valor ou um contexto de computação. Exemplos são Talvez a ou [a]. O artigo Typeclassopedia tem uma boa descrição dos conceitos por trás dos Functors.

Para ser considerado um Functor real, uma instância deve respeitar as 2 seguintes leis:

Identidade #

fmap id == id

Composição #

fmap (f . g) = (fmap f) . (fmap g)

Definição de Classe de Functor e Leis #

class Functor f where
    fmap :: (a -> b) -> f a -> f b

Uma maneira de ver isso é que fmap eleva uma função de valores em uma função de valores em um contexto f.

Uma instância correta de Functor deve satisfazer as leis do functor, embora elas não sejam impostas pelo compilador:

fmap id = id                    -- identity
fmap f . fmap g = fmap (f . g)  -- composition

Existe um alias infixo comumente usado para fmap chamado <$>.

infixl 4 <$>
(<$>) :: Functor f => (a -> b) -> f a -> f b
(<$>) = fmap

Instâncias comuns de Functor #

Pode ser #

Maybe é um Functor contendo um valor possivelmente ausente:

instance Functor Maybe where
    fmap f Nothing = Nothing
    fmap f (Just x) = Just (f x)

A instância de Maybe de Functor aplica uma função a um valor envolvido em um Just. Se a computação falhou anteriormente (portanto, o valor Maybe é um Nothing), então não há valor para aplicar a função, então fmap é um no-op.

> fmap (+ 3) (Just 3)
Just 6
> fmap length (Just "mousetrap")
Just 9
> fmap sqrt Nothing
Nothing

Podemos verificar as leis do functor para esta instância usando raciocínio equacional. Para a lei de identidade,

fmap id Nothing
Nothing  -- definition of fmap
id Nothing  -- definition of id

fmap id (Just x)
Just (id x)  -- definition of fmap
Just x  -- definition of id
id (Just x)  -- definition of id

Para a lei de composição,

(fmap f . fmap g) Nothing
fmap f (fmap g Nothing)  -- definition of (.)
fmap f Nothing  -- definition of fmap
Nothing  -- definition of fmap
fmap (f . g) Nothing  -- because Nothing = fmap f Nothing, for all f

(fmap f . fmap g) (Just x)
fmap f (fmap g (Just x))  -- definition of (.)
fmap f (Just (g x))  -- definition of fmap
Just (f (g x))  -- definition of fmap
Just ((f . g) x)  -- definition of (.)
fmap (f . g) (Just x)  -- definition of fmap

Listas #

A instância de Functor das listas aplica a função a todos os valores na lista em vigor.

instance Functor [] where
    fmap f [] = []
    fmap f (x:xs) = f x : fmap f xs

Alternativamente, isso pode ser escrito como uma compreensão de lista: fmap f xs = [f x | x <- xs].

Este exemplo mostra que fmap generaliza map. map opera apenas em listas, enquanto fmap funciona em um Functor arbitrário.

A lei de identidade pode ser demonstrada por indução:

-- base case
fmap id []
[]  -- definition of fmap
id []  -- definition of id

-- inductive step
fmap id (x:xs)
id x : fmap id xs  -- definition of fmap
x : fmap id xs  -- definition of id
x : id xs  -- by the inductive hypothesis
x : xs  -- definition of id
id (x : xs)  -- definition of id

e da mesma forma, a lei de composição:

-- base case
(fmap f . fmap g) []
fmap f (fmap g [])  -- definition of (.)
fmap f []  -- definition of fmap
[]  -- definition of fmap
fmap (f . g) []  -- because [] = fmap f [], for all f

-- inductive step
(fmap f . fmap g) (x:xs)
fmap f (fmap g (x:xs))  -- definition of (.)
fmap f (g x : fmap g xs)  -- definition of fmap
f (g x) : fmap f (fmap g xs)  -- definition of fmap
(f . g) x : fmap f (fmap g xs)  -- definition of (.)
(f . g) x : fmap (f . g) xs  -- by the inductive hypothesis
fmap (f . g) xs  -- definition of fmap

Funções #

Nem todo Functor se parece com um contêiner. A instância de funções de Functor aplica uma função ao valor de retorno de outra função.

instance Functor ((->) r) where
    fmap f g = \x -> f (g x)

Observe que esta definição é equivalente a fmap = (.). Então fmap generaliza a composição de funções.

Mais uma vez verificando a lei de identidade:

fmap id g
\x -> id (g x)  -- definition of fmap
\x -> g x  -- definition of id
g  -- eta-reduction
id g  -- definition of id

e a lei de composição:

(fmap f . fmap g) h
fmap f (fmap g h)  -- definition of (.)
fmap f (\x -> g (h x))  -- definition of fmap
\y -> f ((\x -> g (h x)) y)  -- definition of fmap
\y -> f (g (h y))  -- beta-reduction
\y -> (f . g) (h y)  -- definition of (.)
fmap (f . g) h  -- definition of fmap

Substituindo todos os elementos de um Functor por um único valor #

O módulo Data.Functor contém dois combinadores, <$ e $>, que ignoram todos os valores contidos em um functor, substituindo todos por um único valor constante.

infixl 4 <$, $>

<$ :: Functor f => a -> f b -> f a
(<$) = fmap . const

$> :: Functor f => f a -> b -> f b
($>) = flip (<$)

void ignora o valor de retorno de um cálculo.

void :: Functor f => f a -> f ()
void = (() <$)

Funtores polinomiais #

Há um conjunto útil de combinadores de tipos para construir grandes Functors a partir de menores. Eles são instrutivos como instâncias de exemplo de Functor, e também são úteis como uma técnica para programação genérica, porque podem ser usados ​​para representar uma grande classe de functors comuns.

O funtor de identidade #

O functor de identidade simplesmente encerra seu argumento. É uma implementação de nível de tipo do combinador I do cálculo SKI.

newtype I a = I a

instance Functor I where
    fmap f (I x) = I (f x)

I pode ser encontrado, sob o nome de Identity, no [módulo Data.Functor.Identity](https://hackage.haskell.org/package/base-4.9.0.0/docs/Data- Functor-Identity.html).

O funtor constante #

O functor constante ignora seu segundo argumento, contendo apenas um valor constante. É um análogo de nível de tipo de const, o combinador K do cálculo SKI.

newtype K c a = K c

Observe que K c a não contém nenhum valor a; K() é isomórfico para Proxy. Isso significa que a implementação do fmap do K não faz nenhum mapeamento!

instance Functor (K c) where
    fmap _ (K c) = K c

K também é conhecido como Const, de Data.Functor.Const .

Os functores restantes neste exemplo combinam functores menores em maiores.

Produtos Functor #

O produto functor pega um par de functores e os empacota. É análogo a uma tupla, exceto que enquanto (,) :: * -> * -> * opera em types *, (:*:) :: (* -> *) -> (* -> *) -> (* -> *) opera em functors * -> *.

infixl 7 :*:
data (f :*: g) a = f a :*: g a

instance (Functor f, Functor g) => Functor (f :*: g) where
    fmap f (fx :*: gy) = fmap f fx :*: fmap f gy

Esse tipo pode ser encontrado, sob o nome Product, no [módulo Data.Functor.Product](https://hackage.haskell.org/package/base-4.9.0.0/docs/Data-Functor- Produto.html).

Coprodutos funcionais #

Assim como :*: é análogo a (,), :+: é o análogo em nível de função de Either.

infixl 6 :+:
data (f :+: g) a = InL (f a) | InR (g a)

instance (Functor f, Functor g) => Functor (f :+: g) where
    fmap f (InL fx) = InL (fmap f fx)
    fmap f (InR gy) = InR (fmap f gy)

:+: pode ser encontrado sob o nome Sum, no [módulo Data.Functor.Sum](https://hackage.haskell.org/package/base-4.9.0.0/docs/Data- Functor-Sum.html).

Composição do functor #

Finalmente, :.: funciona como um nível de tipo (.), pegando a saída de um functor e canalizando-a na entrada de outro.

infixr 9 :.:
newtype (f :.: g) a = Cmp (f (g a))

instance (Functor f, Functor g) => Functor (f :.: g) where
    fmap f (Cmp fgx) = Cmp (fmap (fmap f) fgx)

O tipo Compose pode ser encontrado em Data.Functor.Compose

Functores polinomiais para programação genérica #

I, K, :*:, :+: e :.: podem ser pensados ​​como um kit de blocos de construção para uma certa classe de tipos de dados simples. O kit se torna especialmente poderoso quando você o combina com [pontos fixos](https://www.wikiod.com/pt/haskell/esquemas-de-recursao#Pontos fixos) porque os tipos de dados criados com esses combinadores são automaticamente instâncias de Functor. Você usa o kit para construir um tipo de template, marcando pontos recursivos usando I, e então o conecta em Fix para obter um tipo que pode ser usado com o zoo padrão de esquemas de recursão.

| Nome | Como um tipo de dados | Usando o kit functor | |——————|—————————— ————————|————– ———————————–| | Pares de valores | dados par a = par a a | tipo Par = I :*: I | | Grades dois a dois | tipo Grade a = Par (Par a) | tipo Grade = Par :.: Par | | Números naturais | dados Nat = Zero | Sucesso Nat | type Nat = Fix (K() :+: I) | | Listas | lista de dados a = Nil | Contras a (Lista a) | type List a = Fix (K() :+: K a :*: I) | | Árvores binárias | árvore de dados a = Folha | Nó (Árvore a) a (Árvore a) | type Tree a = Fix (K() :+: I :*: K a :*: I) | | Roseiras | data Rose a = Rose a (Lista (Rosa a)) | type Rose a = Fix (K a :*: List :.: I) |

Essa abordagem de “kit” para projetar tipos de dados é a ideia por trás de bibliotecas de programação genérica como generics-sop. A ideia é escrever operações genéricas usando um kit como o apresentado acima e, em seguida, usar uma classe de tipo para converter tipos de dados arbitrários de e para sua representação genérica:

class Generic a where
    type Rep a  -- a generic representation built using a kit
    to :: a -> Rep a
    from :: Rep a -> a

Funtores na Teoria das Categorias #

Um Functor é definido na teoria das categorias como um mapa de preservação de estrutura (um ‘homomorfismo’) entre categorias. Especificamente, (todos) os objetos são mapeados para objetos e (todas) as setas são mapeadas para setas, de modo que as leis de categoria sejam preservadas.

A categoria na qual objetos são tipos Haskell e morfismos são funções Haskell é chamada Hask. Portanto, um functor de Hask para Hask consistiria em um mapeamento de tipos para tipos e um mapeamento de funções para funções.

A relação que este conceito teórico de categoria tem com a construção de programação de Haskell Functor é bastante direta. O mapeamento de tipos para tipos assume a forma de um tipo f :: * -> *, e o mapeamento de funções para funções assume a forma de uma função `fmap :: (a -> b) -> (f a - > fb)». Juntando-os em uma classe,

class Functor (f :: * -> *) where
    fmap :: (a -> b) -> f a -> f b

fmap é uma operação que pega uma função (um tipo de morfismo), :: a -> b, e a mapeia para outra função, :: f a -> f b. Supõe-se (mas é deixado para o programador garantir) que instâncias de Functor são de fato functores matemáticos, preservando a estrutura categórica de Hask:

fmap (id {- :: a -> a -})  ==  id {- :: f a -> f a -}
fmap (h . g)               ==  fmap h . fmap g

fmap eleva uma função :: a -> b em uma subcategoria de Hask de uma forma que preserva tanto a existência de quaisquer setas de identidade, quanto a associatividade da composição.

A classe Functor codifica apenas endofunctors em Hask. Mas em matemática, functores podem mapear entre categorias arbitrárias. Uma codificação mais fiel desse conceito ficaria assim:

class Category c where
    id  :: c i i
    (.) :: c j k -> c i j -> c i k

class (Category c1, Category c2) => CFunctor c1 c2 f where
    cfmap :: c1 a b -> c2 (f a) (f b)

A classe Functor padrão é um caso especial dessa classe em que as categorias de origem e destino são Hask. Por exemplo,

instance Category (->) where        -- Hask
    id    = \x -> x
    f . g = \x -> f (g x)

instance CFunctor (->) (->) [] where
    cfmap = fmap

Funtor de derivação #

A extensão de linguagem DeriveFunctor permite que o GHC gere instâncias de Functor automaticamente.

{-# LANGUAGE DeriveFunctor #-}

data List a = Nil | Cons a (List a) deriving Functor

-- instance Functor List where            -- automatically defined
--   fmap f Nil = Nil
--   fmap f (Cons x xs) = Cons (f x) (fmap f xs)

map :: (a -> b) -> List a -> List b
map = fmap
