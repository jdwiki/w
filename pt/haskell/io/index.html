Tirando o ‘a’ “de” ‘IO a’ #

Uma pergunta comum é “Eu tenho um valor de ‘IO a’, mas quero fazer algo com esse valor de ‘a’: como faço para ter acesso a ele?” Como se pode operar com dados que vêm do mundo exterior (por exemplo, incrementando um número digitado pelo usuário)?

O ponto é que, se você usar uma função pura em dados obtidos de forma impura, o resultado ainda será impuro. Depende do que o usuário fez! Um valor do tipo IO a significa uma “computação de efeito colateral resultando em um valor do tipo a” que pode somente ser executado (a) compondo-o em main e (b) compilando e executando seu programa. Por essa razão, não há como no mundo puro de Haskell “tirar o ‘a’”.

Em vez disso, queremos construir uma nova computação, um novo valor IO, que faz uso do valor a em tempo de execução. Esta é outra maneira de compor valores IO e, novamente, podemos usar a notação do:

-- assuming
myComputation :: IO Int

getMessage :: Int -> String
getMessage int = "My computation resulted in: " ++ show int
 
newComputation :: IO ()
newComputation = do
  int <- myComputation       -- we "bind" the result of myComputation to a name, 'int'
  putStrLn $ getMessage int   -- 'int' holds a value of type Int

Aqui estamos usando uma função pura (getMessage) para transformar um Int em uma String, mas estamos usando a notação do para fazê-la ser aplicada ao resultado de uma computação IO myComputation quando (depois) que a computação é executada. O resultado é uma computação ‘IO’ maior, ’newComputation’. Essa técnica de usar funções puras em um contexto impuro é chamada de lifting.

IO define a ação main do seu programa #

Para tornar um programa Haskell executável você deve fornecer um arquivo com uma função main do tipo IO()

main :: IO ()
main = putStrLn "Hello world!"

Quando o Haskell é compilado, ele examina os dados IO aqui e os transforma em um executável. Quando executarmos este programa, ele imprimirá Hello world!.

Se você tiver valores do tipo IO a diferentes de main eles não farão nada.

other :: IO ()
other = putStrLn "I won't get printed"

main :: IO ()
main = putStrLn "Hello world!"

Compilar este programa e executá-lo terá o mesmo efeito do último exemplo. O código em other é ignorado.

Para fazer com que o código em other tenha efeitos de tempo de execução, você deve compor em main. Nenhum valor IO não eventualmente composto em main terá qualquer efeito de tempo de execução. Para compor dois valores IO sequencialmente você pode usar a notação do:

other :: IO ()
other = putStrLn "I will get printed... but only at the point where I'm composed into main"

main :: IO ()
main = do 
  putStrLn "Hello world!"
  other

Quando você compila e executa este programa, ele gera

Hello world!
I will get printed... but only at the point where I'm composed into main

Observe que a ordem das operações é descrita por como other foi composto em main e não pela ordem de definição.

Verificando as condições de fim de arquivo #

Um pouco contra-intuitivo para a forma como a maioria das bibliotecas de E/S padrão de outras linguagens fazem isso, o isEOF de Haskell não exige que você execute uma operação de leitura antes de verificar uma condição EOF; o tempo de execução fará isso por você.

import System.IO( isEOF )


eofTest :: Int -> IO ()
eofTest line = do
    end <- isEOF
    if end then
        putStrLn $ "End-of-file reached at line " ++ show line ++ "."
    else do
        getLine
        eofTest $ line + 1


main :: IO ()
main =
    eofTest 1

Entrada:

Line #1.
Line #2.
Line #3.

Resultado:

End-of-file reached at line 4.

Lendo todo o conteúdo da entrada padrão em uma string #

main = do
    input <- getContents
    putStr input

Entrada:

This is an example sentence.
And this one is, too!

Resultado:

This is an example sentence.
And this one is, too!

Nota: Este programa na verdade imprimirá partes da saída antes que toda a entrada tenha sido totalmente lida. Isso significa que, se, por exemplo, você usar getContents em um arquivo de 50MiB, a avaliação preguiçosa e o coletor de lixo do Haskell garantirão que apenas as partes do arquivo que são atualmente necessárias (leia-se: indispensável para execução posterior) serão carregadas na memória. Assim, o arquivo de 50MiB não será carregado na memória de uma só vez.

Função e Objetivo do IO #

Haskell é uma linguagem pura, o que significa que as expressões não podem ter efeitos colaterais. Um efeito colateral é qualquer coisa que a expressão ou função faça além de produzir um valor, por exemplo, modificar um contador global ou imprimir na saída padrão.

Em Haskell, computações de efeito colateral (especificamente, aquelas que podem afetar o mundo real) são modeladas usando ‘IO’. Estritamente falando, IO é um construtor de tipo, pegando um tipo e produzindo um tipo. Por exemplo, IO Int é o tipo de computação de E/S que produz um valor Int. O tipo IO é abstract, e a interface fornecida para IO garante que certos valores ilegais (ou seja, funções com tipos sem sentido) não possam existir, garantindo que todas as funções internas que executam IO tenham um tipo de retorno entre IO.

Quando um programa Haskell é executado, a computação representada pelo valor Haskell chamado main, cujo tipo pode ser IO x para qualquer tipo x, é executada.

Manipulando valores de IO #

Existem muitas funções na biblioteca padrão que fornecem ações típicas de ‘IO’ que uma linguagem de programação de uso geral deve executar, como ler e gravar em handles de arquivo. As ações gerais IO são criadas e combinadas principalmente com duas funções:

 (>>=) :: IO a -> (a -> IO b) -> IO b

Esta função (normalmente chamada de bind) recebe uma ação IO e uma função que retorna uma ação IO, e produz a ação IO que é o resultado da aplicação da função ao valor produzido pelo primeiro IO ` ação.

 return :: a -> IO a

Essa função recebe qualquer valor (ou seja, um valor puro) e retorna o cálculo de IO que não faz IO e produz o valor fornecido. Em outras palavras, é uma ação de E/S sem operação.

Existem funções gerais adicionais que são frequentemente usadas, mas todas podem ser escritas em termos das duas acima. Por exemplo, (>>) :: IO a -> IO b -> IO b é semelhante a (>>=) mas o resultado da primeira ação é ignorado.

Um programa simples cumprimentando o usuário usando estas funções:

 main :: IO ()
 main =
   putStrLn "What is your name?" >>
   getLine >>= \name ->
   putStrLn ("Hello " ++ name ++ "!")

Este programa também usa putStrLn :: String -> IO() e getLine :: IO String.

Nota: os tipos de certas funções acima são realmente mais gerais do que os tipos fornecidos (nomeadamente >>=, >> e return).

Semântica de E/S #

O tipo IO em Haskell tem uma semântica muito semelhante à das linguagens de programação imperativas. Por exemplo, quando se escreve s1 ; s2 em uma linguagem imperativa para indicar a execução da instrução s1, então a instrução s2, pode-se escrever s1 >> s2 para modelar a mesma coisa em Haskell.

No entanto, a semântica de IO diverge ligeiramente do que seria esperado de um background imperativo. A função return não interrompe o fluxo de controle - ela não tem efeito no programa se outra ação IO for executada em seqüência. Por exemplo, return() >> putStrLn "boom" imprime corretamente “boom” na saída padrão.

A semântica formal de IO pode ser dada em termos de igualdades simples envolvendo as funções da seção anterior:

 return x >>= f ≡ f x, ∀ f x
 y >>= return ≡ return y, ∀ y
 (m >>= f) >>= g ≡ m >>= (\x -> (f x >>= g)), ∀ m f g

Essas leis são normalmente chamadas de identidade à esquerda, identidade à direita e composição, respectivamente. Eles podem ser declarados mais naturalmente em termos da função

 (>=>) :: (a -> IO b) -> (b -> IO c) -> a -> IO c
 (f >=> g) x = (f x) >>= g

do seguinte modo:

 return >=> f ≡ f, ∀ f
 f >=> return ≡ f, ∀ f
 (f >=> g) >=> h ≡ f >=> (g >=> h), ∀ f g h

IO preguiçoso #

As funções que executam cálculos de E/S são geralmente rígidas, o que significa que todas as ações anteriores em uma sequência de ações devem ser concluídas antes que a próxima ação seja iniciada. Normalmente, isso é um comportamento útil e esperado - putStrLn "X" >> putStrLn "Y" deve imprimir “XY”. No entanto, certas funções de biblioteca executam E/S lentamente, o que significa que as ações de E/S necessárias para produzir o valor são executadas apenas quando o valor é realmente consumido. Exemplos de tais funções são getContents e readFile. Lazy I/O pode reduzir drasticamente o desempenho de um programa Haskell, portanto, ao usar funções de biblioteca, deve-se tomar cuidado para observar quais funções são preguiçosas.

IO e notação do #

Haskell fornece um método mais simples de combinar diferentes valores de IO em valores de IO maiores. Esta sintaxe especial é conhecida como notação do* e é simplesmente açúcar sintático para uso das funções >>=, >> e return.

O programa na seção anterior pode ser escrito de duas maneiras diferentes usando a notação do, sendo a primeira sensível ao layout e a segunda insensível ao layout:

 main = do
   putStrLn "What is your name?"
   name <- getLine
   putStrLn ("Hello " ++ name ++ "!")


 main = do {
   putStrLn "What is your name?" ;
   name <- getLine ;
   putStrLn ("Hello " ++ name ++ "!")
   }

Todos os três programas são exatamente equivalentes.

*Observe que a notação do também é aplicável a uma classe mais ampla de construtores de tipos chamados monas.

Lendo uma linha da entrada padrão #

Lendo palavras de um arquivo inteiro #

Em Haskell, geralmente faz sentido não se preocupar com manipuladores de arquivos, mas simplesmente ler ou gravar um arquivo inteiro direto do disco para a memória† e fazer todo o particionamento/processamento do texto com o estrutura de dados de string pura. Isso evita misturar IO e lógica de programa, o que pode ajudar muito a evitar bugs.

-- | The interesting part of the program, which actually processes data
--   but doesn't do any IO!
reverseWords :: String -> [String]
reverseWords = reverse . words

-- | A simple wrapper that only fetches the data from disk, lets
--   'reverseWords' do its job, and puts the result to stdout.
main :: IO ()
main = do
   content <- readFile "loremipsum.txt"
   mapM_ putStrLn $ reverseWords content

Se loremipsum.txt contiver

Lorem ipsum dolor sit amet,
consectetur adipiscing elit

então o programa irá produzir

elit
adipiscing
consectetur
amet,
sit
dolor
ipsum
Lorem

Aqui, mapM_ passou pela lista de todas as palavras no arquivo e imprimiu cada um deles em uma linha separada com putStrLn.

†Se você acha que isso é um desperdício de memória, você tem razão. Na verdade, a preguiça de Haskell muitas vezes pode evitar que o arquivo inteiro precise residir na memória simultaneamente… mas cuidado, esse tipo de IO preguiçoso causa seu próprio conjunto de problemas. Para aplicativos de desempenho crítico, geralmente faz sentido forçar a leitura de todo o arquivo de uma vez, estritamente; você pode fazer isso com [a versão Data.Text do readFile.](http://hackage.haskell.org/package/text-1.2.2.1/docs/Data-Text-IO.html#v: readFile)

Escrevendo para stdout #

De acordo com a Especificação de linguagem Haskell 2010, a seguir estão as funções de E/S padrão disponíveis no Prelude, portanto, nenhuma importação é necessária para usá-las.

putChar :: Char -> IO() - escreve um char para stdout #

Prelude> putChar 'a'
aPrelude>  -- Note, no new line

putStr :: String -> IO () - escreve uma String para stdout #

Prelude> putStr "This is a string!"
This is a string!Prelude>  -- Note, no new line

putStrLn :: String -> IO () - escreve uma String para stdout e adiciona uma nova linha #

Prelude> putStrLn "Hi there, this is another String!"
Hi there, this is another String!

print :: Show a => a -> IO() - escreve a uma instância de Show para stdout #

Prelude> print "hi"
"hi"
Prelude> print 1
1
Prelude> print 'a'
'a'
Prelude> print (Just 'a')  -- Maybe is an instance of the `Show` type class
Just 'a'
Prelude> print Nothing
Nothing

Lembre-se de que você pode instanciar Show para seus próprios tipos usando deriving:

-- In ex.hs
data Person = Person { name :: String } deriving Show
main = print (Person "Alex")  -- Person is an instance of `Show`, thanks to `deriving`

Carregando e executando no GHCi:

Prelude> :load ex.hs
[1 of 1] Compiling ex             ( ex.hs, interpreted )
Ok, modules loaded: ex.
*Main> main  -- from ex.hs
Person {name = "Alex"}
*Main>

Analisando e construindo um objeto da entrada padrão #

readFloat :: IO Float
readFloat =
    fmap read getLine


main :: IO ()
main = do
    putStr "Type the first number: "
    first <- readFloat

    putStr "Type the second number: "
    second <- readFloat

    putStrLn $ show first ++ " + " ++ show second ++ " = " ++ show ( first + second )

Entrada:

Type the first number: 9.5
Type the second number: -2.02

Resultado:

9.5 + -2.02 = 7.48

Lendo de identificadores de arquivo #

Como em várias outras partes da biblioteca de E/S, funções que usam implicitamente um fluxo padrão têm uma contrapartida em System.IO que executa o mesmo trabalho, mas com um parâmetro extra à esquerda, do tipo Handle, que representa o fluxo que está sendo tratado. Por exemplo, getLine :: IO String tem uma contrapartida hGetLine :: Handle -> IO String.

import System.IO( Handle, FilePath, IOMode( ReadMode ), 
                  openFile, hGetLine, hPutStr, hClose, hIsEOF, stderr )

import Control.Monad( when )


dumpFile :: Handle -> FilePath -> Integer -> IO ()

dumpFile handle filename lineNumber = do      -- show file contents line by line
    end <- hIsEOF handle
    when ( not end ) $ do
        line <- hGetLine handle
        putStrLn $ filename ++ ":" ++ show lineNumber ++ ": " ++ line
        dumpFile handle filename $ lineNumber + 1


main :: IO ()

main = do
    hPutStr stderr "Type a filename: "
    filename <- getLine
    handle <- openFile filename ReadMode
    dumpFile handle filename 1
    hClose handle

This is an example.

This is another example.

Entrada:

Type a filename: example.txt

Resultado:

example.txt:1: This is an example.
example.txt:2: Hello, world!
example.txt:3: This is another example

Lendo de stdin #

De acordo com a Haskell 2010 Language Specification, a seguir estão as funções de E/S padrão disponíveis no Prelude, portanto, nenhuma importação é necessária para usá-las.

getChar :: IO Char - lê um Char de stdin #

-- MyChar.hs
main = do
  myChar <- getChar
  print myChar

-- In your shell

runhaskell MyChar.hs
a -- you enter a and press enter
'a'  -- the program prints 'a'

getLine :: IO String - lê uma String de stdin, sem o caractere de nova linha #

Prelude> getLine
Hello there!  -- user enters some text and presses enter
"Hello there!"

read :: Read a => String -> a - converte uma String em um valor #

Prelude> read "1" :: Int
1
Prelude> read "1" :: Float
1.0
Prelude> read "True" :: Bool
True

Outras funções menos comuns são:

  • getContents :: IO String - retorna todas as entradas do usuário como uma única string, que é lida lentamente conforme necessário
  • interact :: (String -> String) -> IO () - recebe uma função do tipo String->String como argumento. A entrada inteira do dispositivo de entrada padrão é passada para esta função como seu argumento
