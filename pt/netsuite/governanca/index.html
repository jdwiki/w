Governança #

“Governança” é o nome dado ao sistema do NetSuite para detectar e interromper scripts de longa duração, descontrolados ou que consomem muitos recursos.

Cada tipo de script tem limites de governança que não podem ser excedidos e há quatro tipos de limites de governança em vigor para cada tipo de script.

  • Limite de uso da API
  • Limite de contagem de instruções
  • Limite de tempo limite
  • Limite de uso de memória

Se um script exceder seu limite de governança em qualquer uma dessas quatro áreas, o NetSuite criará uma exceção incapturável e encerrará o script imediatamente.

Limite de uso da API #

O NetSuite limita o uso da API de seus scripts com um sistema baseado em “unidades de uso”. Algumas chamadas de API do NetSuite, principalmente aquelas que executam uma ação de leitura ou gravação no banco de dados, custam um número específico de unidades cada vez que são chamadas. Cada tipo de script tem então um número máximo de unidades que podem ser usadas durante cada execução do script.

Se um script exceder o limite de uso da API, o NetSuite encerrará o script lançando um erro SSS_USAGE_LIMIT_EXCEEDED.

Abaixo estão alguns exemplos de custos unitários para operações comuns. Para obter uma lista completa dos custos de governança, consulte o artigo intitulado “API Governance” na Ajuda do NetSuite.

Operação Custo Unitário
Carregando uma pesquisa salva 5
Recuperando resultados de pesquisa 10
Agendando uma tarefa 10
Solicitando um URL 10
Enviando um e-mail 10
Criando um registro personalizado 2
Criando um registro de funcionário 5
Criando um registro de Pedido de Vendas 10
Salvando um registro personalizado 4
Salvando um registro de contato 10
Salvando um registro de pedido de compra 20

Diferentes operações usam diferentes quantidades de unidades e certas operações custam uma quantia diferente com base no tipo de registro que está sendo usado. Quanto maior o número de unidades que uma função custa, normalmente mais tempo levará para ser executado.

As transações são os maiores tipos de registro, portanto, trabalhar com elas custa a maior quantidade de unidades. Por outro lado, os registros personalizados são muito leves e, portanto, não custam muitas unidades. Os registros padrão do NetSuite que não são Transações, como Clientes, Funcionários ou Contatos, ficam entre os dois em termos de custo.

Estes são os limites de uso por tipo de script:

Tipo de script Limite de uso
Cliente 1.000
Evento do usuário 1.000
Suite 1.000
Portlet 1.000
Ação do fluxo de trabalho 1.000
RESTlet 5.000
Programado 10.000
Mapear/Reduzir 10.000
Instalação do pacote 10.000
Atualização em massa 10.000 por registro

Limites de tempo limite e contagem de instruções #

O NetSuite também usa o sistema de governança para detectar e interromper scripts descontrolados usando um mecanismo de tempo limite e um contador de instruções.

Se um script demorar muito para ser executado, o NetSuite o interromperá lançando um erro SSS_TIME_LIMIT_EXCEEDED.

Além disso, scripts descontrolados podem ser detectados e interrompidos com base em sua “Contagem de instruções”. Se os limites de contagem de instruções definidos forem excedidos, o NetSuite interromperá o script lançando um erro SSS_INSTRUCTION_COUNT_EXCEEDED.

Infelizmente, não existe documentação de ajuda que defina:

  • o tempo limite para cada tipo de script
  • os limites de contagem de instruções para cada tipo de script
  • o que constitui uma única “instrução”

É simplesmente importante saber que, se você encontrar o erro SSS_TIME_LIMIT_EXCEEDED ou o erro SSS_INSTRUCTION_COUNT_EXCEEDED em um de seus scripts, o processamento está demorando muito. Concentre sua investigação em suas estruturas de loop para determinar onde as otimizações podem ser feitas.

Limite de uso de memória #

Se seu script exceder o limite de uso de memória, o NetSuite encerrará seu script lançando um erro SSS_MEMORY_USAGE_EXCEEDED.

Cada variável declarada, cada função definida, cada objeto armazenado contribui para o uso de memória do seu script.

Tanto o Scheduled Script quanto o Map/Reduce Script têm limites de memória de 50 MB documentados. Há também um limite documentado de 10MB para o tamanho de qualquer String passada ou retornada de um RESTlet. Não há outra documentação sobre os limites específicos para um determinado script.

Quantas unidades me restam? #

No SuiteScript 1.0, use nlobjContext.getRemainingUsage() para recuperar as unidades restantes. Uma referência nlobjContext é recuperada usando a função global nlapiGetContext.

// 1.0
var context = nlapiGetContext();
nlapiLogExecution("DEBUG", "Governance Monitoring", "Remaining Usage = " + context.getRemainingUsage());

nlapiSearchRecord("transaction"); // uses 10 units
nlapiLogExecution("DEBUG", "Governance Monitoring", "Remaining Usage = " + context.getRemainingUsage());

No SuiteScript 2.0, use o método getRemainingUsage do objeto Script do módulo N/runtime.

// 2.0
require(["N/log", "N/runtime", "N/search"], function (log, runtime, s) {
    var script = runtime.getCurrentScript();
    log.debug({
        "title": "Governance Monitoring",
        "details": "Remaining Usage = " + script.getRemainingUsage()
    });

    s.load({"id":"customsearch_mysearch"}); // uses 5 units
    log.debug({
        "title": "Governance Monitoring",
        "details": "Remaining Usage = " + script.getRemainingUsage()
    });
});
