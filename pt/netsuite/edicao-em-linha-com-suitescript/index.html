Edição em linha com SuiteScript

A edição em linha permite que os usuários modifiquem e atualizem rapidamente os dados de um registro específico sem precisar carregar o registro inteiro em uma página, editar o formulário e salvar o registro.

Os desenvolvedores do NetSuite têm uma funcionalidade correspondente chamada submitFields. A funcionalidade submitFields é fornecida pela função global nlapiSubmitField no SuiteScript 1.0 e o método N/record#submitFields no SuiteScript 2.0.

Sintaxe #

  • nlapiSubmitField(recordType, recordId, fieldId, fieldValue);
  • nlapiSubmitField(recordType, recordId, fieldIds, fieldValues);
  • nlapiSubmitField(recordType, recordId, fieldId, fieldValue, doSourcing);

Parâmetros #

Parâmetro Detalhes
tipoderegistro String - O ID interno do tipo de registro que está sendo atualizado
recordId String ou Number - O ID interno do registro que está sendo atualizado
fieldIds String ou String[] - O(s) ID(s) interno(s) do(s) campo(s) sendo atualizado(s)
fieldValues ​​ any ou any[] - Os valores correspondentes a serem definidos nos campos fornecidos
doSourcing Boolean - Se os valores dependentes devem ser originados no envio do registro. O padrão é falso

A funcionalidade submitFields é um recurso complementar da funcionalidade lookupFields.

Desempenho e Limitações #

submitFields executa significativamente mais rápido e usa menos governança do que fazer as mesmas alterações carregando e enviando o registro completo.

Vários campos podem ser atualizados de uma só vez pelo mesmo custo da atualização de um único campo. Atualizar mais campos com submitFields não incorre em um custo de governança mais alto.

No entanto, você deve estar ciente de que apenas determinados campos em cada tipo de registro são editáveis ​​em linha, e a economia de desempenho somente se aplica a esses campos editáveis ​​em linha. Se você usar a função submitFields em qualquer campo não editável em linha, o campo será atualizado corretamente, mas nos bastidores, o NetSuite realmente carregará e enviará o registro, levando mais tempo e usando mais governança. Você pode determinar se um campo é editável em linha consultando a coluna “nlapiSubmitField” no Records Browser.

A funcionalidade submitFields também é limitada aos campos body de um registro. Se você precisar modificar os dados da sublista, será necessário carregar o registro para fazer suas alterações e, em seguida, enviar o registro.

[1.0] Envie um único campo #

/**
 * A SuiteScript 1.0 example of using nlapiSubmitField to update a single field on a related record
 */

// From a Sales Order, get the Customer ID
var customerId = nlapiGetFieldValue("entity");

// Set a comment on the Customer record
nlapiSubmitField("customer", customerId, "comments", "This is a comment added by inline editing with SuiteScript.");

[1.0] Enviar vários campos #

/**
 * A SuiteScript 1.0 example of using nlapiSubmitField to update multiple fields on a related record
 */

// From a Sales Order, get the Customer ID
var customerId = nlapiGetFieldValue("entity");

// Set a Comment and update the Budget Approved field on the Customer record
nlapiSubmitField("customer", customerId,
    ["comments", "isbudgetapproved"],
    ["The budget has been approved.", "T"]);

[2.0] Envie um único campo #

/**
 * A SuiteScript 2.0 example of using N/record#submitFields to update a single field on a related record
 */

require(["N/record", "N/currentRecord"], function (r, cr) {

    // From a Sales Order, get the Customer ID
    var customerId = cr.get().getValue({"fieldId": "entity"});

    // Set a Comment on the Customer record
    r.submitFields({
        "type": r.Type.CUSTOMER,
        "id": customerId,
        "values": {
            "comments": "This is a comment added by inline editing with SuiteScript."
        }
    });
});

[2.0] Enviar vários campos #

/**
 * A SuiteScript 2.0 example of using N/record#submitFields to update multiple fields on a related record
 */

require(["N/record", "N/currentRecord"], function (r, cr) {

    // From a Sales Order, get the Customer ID
    var customerId = cr.get().getValue({"fieldId": "entity"});

    // Set a Comment and check the Budget Approved box on the Customer record
    r.submitFields({
        "type": r.Type.CUSTOMER,
        "id": customerId,
        "values": {
            "comments": "The budget has been approved.",
            "isbudgetapproved": true
        }
    });
});
