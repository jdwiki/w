As colunas de fórmula em pesquisas salvas podem explorar muitos recursos do Oracle SQL e HTML. Os exemplos mostram como esses recursos podem ser usados, bem como as armadilhas a serem evitadas.

Instrução Oracle SQL CASE em uma fórmula Netsuite #

Usando uma instrução CASE, exiba condicionalmente uma expressão na coluna com base nos valores encontrados em outra coluna, também conhecida como “meu reino para um OR”. No exemplo, o resultado é obtido quando o status da transação é Pending Fulfillment ou Partially Fullfilled:

CASE DECODE( {status}, 'Pendente Processamento', 1, 'Parcialmente Cumprido', 1, 0 ) WHEN 1 THEN expressão-1 FIM

Analisando um nome de registro hierárquico usando uma expressão regular #

Usando uma expressão regular, analise um nome de registro que pode ser hierárquico. A expressão procura os dois pontos finais no nome. Ele retorna o que segue os dois pontos, ou o nome inteiro se nenhum:

regexp_substr({name}, '[^:]*$')

Construa uma string complexa concatenando vários campos #

O exemplo cria uma string a partir do nome do registro pai, do nome desse registro e do memorando desse registro.

{criado de} || ' ' || {nome} || ' ' || {memorando}

Personalize o CSS (folha de estilo) para uma coluna inserindo um elemento DIV #

'<div style="font-size:11pt">' || expressão || '</div>'

Proteja as fórmulas de string contra ataques de corrupção e injeção #

Em um campo de fórmula de string, considere que alguns valores podem conter substrings que parecem HTML para o navegador. A menos que isso seja intencional, é importante proteger os valores da corrupção. Isso é útil para evitar ataques de injeção: impede que alguém insira HTML em um campo de comentário em um pedido da Web que posteriormente é interpretado na mesa do representante de atendimento ao cliente.

htf.escape_sc( expressão )

Proteja os valores dos campos contra corrupção ao passar por uma URL #

utl_url.escape( expressão )

Teste o valor de mainline em uma instrução SQL CASE #

Em uma fórmula de pesquisa salva, os valores possíveis de mainline são projetados para serem úteis em um contexto HTML. Quando mainline for true, o valor de {mainline} é a string de 1 caractere * (asterisco). Quando mainline é falso, o valor de {mainline} é a string de 6 caracteres &nbsp; (espaço sem quebra, HTML codificado como uma referência de entidade de caractere). Esses valores de string podem ser comparados com literais de string em um contexto SQL.

CASO WHEN {mainline} = '*' THEN expressão-quando-verdadeira QUANDO {principal} = '&nbsp;' THEN expressão-quando-falsa FIM

Exemplo complexo do mundo real #

O exemplo a seguir combina várias das técnicas abordadas aqui. Ele coloca um hiperlink em uma coluna de formatação personalizada que, quando clicada, abre o registro do pedido de venda associado a uma linha. O hiperlink foi projetado para abrir o registro em uma nova janela ou guia quando clicado e para exibir uma dica de ferramenta ao passar o mouse. O campo internalid usado na URL é protegido contra codificação de URL. O nome do cliente, quando disponível, é exibido na mesma coluna, protegido da codificação HTML.

'<div style="font-size:11pt">'
 ||
CASE {mainline}
WHEN '*' THEN '<br>' || htf.escape_sc( regexp_substr( {name} , '[^:]*$' ) ) || '<br>'
END
 ||
'<a alt="" title="Open the order associated with this line." '
 ||
'href="javascript:void(0);" onClick="window.open('''
 ||
'https://system.na1.netsuite.com/app/accounting/transactions/transaction.nl?id='
 ||
utl_url.escape( {internalid} )
 ||
''' , ''_blank'' )">'
 ||
{number}
 ||
'</a>'
 ||
'</div>'

Contar registros com e sem valor fornecido em um campo (contar valores ausentes e não ausentes) #

Usando a função NVL2() do Oracle SQL, você pode criar uma coluna de exibição que contém um valor se um campo contém dados e outro valor se um campo não contém dados. Por exemplo, em uma pesquisa de Entidade, transforme a presença de um endereço de e-mail principal em uma coluna de exibição de texto:

NVL2({email}, 'SIM', 'NÃO')

Isso permite contar registros subtotalizados pela presença ou ausência de um endereço de e-mail:

Field: Internal ID
Summary Type: Count

Field: Formula (Text)
Summary Type: Group
Formula: NVL2( {email} , 'YES' , 'NO' )
