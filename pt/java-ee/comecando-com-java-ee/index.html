O que é Java EE? #

Java EE significa Java Enterprise Edition. Java EE estende o Java SE (que significa Java Standard Edition). Java EE é um conjunto de tecnologias e especificações relacionadas que são orientadas para o desenvolvimento de aplicativos corporativos de grande escala. Java EE é desenvolvido em um processo orientado pela comunidade. Até agora, as seguintes versões do Java EE foram lançadas:

  • J2EE 1.2 (12 de dezembro de 1999)
  • J2EE 1.3 (24 de setembro de 2001)
  • J2EE 1.4 (11 de novembro de 2003)
  • Java EE 5 (11 de maio de 2006)
  • Java EE 6 (10 de dezembro de 2009)
  • Java EE 7 (5 de abril de 2013)

E espera-se que o Java EE 8 seja lançado no primeiro semestre de 2017.

Um conceito chave do Java EE é que cada versão do Java EE é composta por um conjunto de tecnologias específicas. Essas tecnologias tratam de JSRs (Java Specification Requests) específicos. Para que um programador use essas tecnologias, ele precisa baixar uma implementação das especificações da tecnologia Java EE. A Comunidade Java fornece uma implementação de referência para cada tecnologia, mas outras tecnologias compatíveis com Java EE são desenvolvidas e também podem ser usadas. A comunidade disponibiliza um conjunto de testes, nomeadamente o Java Compatibility Kit (JCK) que pode ser utilizado pelos programadores de uma implementação JSR para verificar se esta é compatível ou não com o JSR. Instalação #

Em primeiro lugar, você não pode “instalar” o Java EE. O Java EE consiste em várias especificações. No entanto, você pode instalar implementações dessas especificações.

Dependendo de suas necessidades, há muitas possibilidades. Para instalar a maioria (ou todas) as especificações, você pode escolher um servidor de aplicativos compatível com Java EE 7. Dependendo de suas necessidades, você pode escolher entre servidores de aplicativos que implementam o perfil da Web ou servidores de aplicativos que implementam o perfil completo. Para obter uma lista de servidores de aplicativos compatíveis com Java EE7, consulte Compatibilidade com Java EE.

Instalando o Servidor Payara Completo #

Pré-requisitos

  • JDK 1.7 ou posterior instalado. Você pode encontrar os JDKs da Oracle aqui.

Passos

  • Baixar Payara Server Full.
  • Descompacte o Servidor Payara em algum local do seu computador. Usaremos C:\payara41 como INSTALL_DIR para usuários de Windows e /payara41 para usuários de Linux/Mac.

Iniciando/parando o Payara no prompt de comando

  • Windows: Abra um prompt de comando e execute o seguinte comando para iniciar/parar o Payara:

    "C:\payara41\bin\asadmin" start-domain

    "C:\payara41\bin\asadmin" stop-domain

  • Linux/Max: Abra um terminal e execute o seguinte comando para iniciar/parar o Payara:

    /payara41/bin/asadmin start-domain

    /payara41/bin/asadmin stop-domain

Iniciando Payara do Netbeans

  • Adicione o servidor Payara ao Netbeans (veja o capítulo anterior)
  • Vá para a guia ‘Serviços’ (Windows -> Serviços).
  • Expanda o item ‘Servidores’.
  • Clique com o botão direito do mouse no servidor Payara e selecione ‘Iniciar’.
  • (Opcional) Clique com o botão direito do mouse no servidor Payara e selecione ‘Exibir console de administração de domínio’ para acessar o console.

Para verificar se você está executando o Application Server, abra um navegador e vá para http://localhost:4848 para ver o Payara Server Console.

Voilá! Agora é hora de construir seu primeiro aplicativo usando JavaEE e implementá-lo em seu servidor!

Construindo meu primeiro aplicativo JavaEE (Hello World) #

Vamos entender uma coisa. JavaEE consiste em uma série de especificações. Ao instalar um servidor de aplicativos (Payara, por exemplo), você instala todas as especificações de uma vez. Por exemplo, há uma especificação para um ORM chamado JPA (Java Persistence API), uma especificação para construir aplicativos Web baseados em componentes chamados JSF (Java Server Faces), uma especificação para construir serviços web REST e clientes chamados JAX-RS.

Então, como você pode imaginar, não há apenas uma maneira de construir um aplicativo Hello World simples em JavaEE.

Em segundo lugar, a especificação JavaEE possui uma estrutura específica de pastas que se parece com isso (simplificada):

/projectname/src/main/java
/projectname/src/main/resources
/projectname/src/main/resources/META-INF
/projectname/src/main/webapp
/projectname/src/main/webapp/WEB-INF

Dentro do /projectname/src/main/java colocamos todas as classes java que precisamos.

Dentro do /projectname/src/main/webapp colocamos arquivos html, arquivos css, arquivos javascript, etc.

Dentro do /projectname/src/main/webapp/WEB-INF vão alguns arquivos de configuração opcionais, como web.xml e beans.xml.

Para simplificar, usaremos o NetBeans IDE (é gratuito) para construir nosso primeiro aplicativo JavaEE. Você pode encontrá-lo aqui. Escolha a versão JavaEE e instale-a.

Criar novo projeto #

  • Abra o NetBeans IDE
  • Vá para File > New Project > Maven > Web Application e clique em Next.
  • Altere nome do projeto para HelloJavaEE, clique em Avançar e em Concluir.

Limpe e construa o projeto #

  • Vá em Executar > Limpar e Construir Projeto (HelloJavaEE).

Implante o arquivo WAR #

  • Em um navegador, vá para http://localhost:4848 (siga as instruções para instalar o servidor payara).
  • Vá para Applications > Click Deploy, clique em Select File e escolha seu arquivo war (HelloJavaEE-1.0-SNAPSHOT.war) em ../NetBeansProjects/HelloJavaEE/target.

Implante o arquivo WAR diretamente do Netbeans #

  • Instale o Payara (veja o próximo capítulo).
  • No Netbeans, vá para a aba ‘Serviços’ (Janela-> Serviços caso não veja).
  • Clique com o botão direito em Servidores e selecione ‘Adicionar Servidor…’
  • Selecione ‘Servidor GlassFish’, dê um nome e clique em próximo.
  • Aponte para sua instalação local do Payara, selecione ‘Local Domain’ e clique em próximo.
  • Deixe as configurações de localização do domínio como estão (Domínio: domínio1, Host: localhost, Porta DAS: 4848, Porta HTTP: 8080).
  • Vá para a guia ‘Projetos’ (Windows -> Projetos).
  • Clique com o botão direito do mouse em seu projeto e selecione ‘Propriedades’.
  • No painel esquerdo, vá para ‘Executar’ e selecione o servidor que você acabou de adicionar.
  • (Opcional) Altere o caminho de contexto. Se você definir o caminho de contexto para ‘/MyFirstApplication’, a URL padrão será ‘http://localhost:8080/MyFirstApplication’.

Ver resultados #

  • Em um navegador, acesse http://localhost:8080/HelloJavaEE-1.0-SNAPSHOT

Voilá! Esse é o seu primeiro aplicativo usando a tecnologia JavaEE. Agora você deve começar a criar outros aplicativos “Hello World” usando especificações diferentes como JPA, EJB, JAX-RS, JavaBatch, etc…
