Configurando o Dojo com dojoConfig

  • Configuração do carregador
  • Carregar DojoConfig

Nesta página

  • Configuração do carregador
  • Carregar DojoConfig

O objeto dojoConfig (anteriormente djConfig) permite definir opções e comportamento padrão para vários aspectos do kit de ferramentas dojo. Os exemplos explicarão o que é possível e como você pode colocar dojoConfig para usar em seu código.

Observe que dojoConfig é definido em um bloco de script antes de o dojo.js ser carregado. Isso é de suma importância — se revertido, as propriedades de configuração serão ignoradas.

Configuração do carregador #

O Dojo recebeu um novo carregador no Dojo 1.7 para acomodar o novo formato de módulo AMD do kit de ferramentas. Este novo carregador adicionou algumas novas opções de configuração que são cruciais para definir pacotes, mapas e muito mais. Para obter detalhes sobre o carregador, consulte o tutorial Uso avançado da AMD. Os parâmetros importantes de configuração do carregador incluem:

baseUrl: A URL base anexada a um identificador de módulo quando convertendo-o em um caminho ou URL.

baseUrl: "/js"

pacotes: Uma matriz de objetos que fornecem o nome do pacote e local:

packages: [{
    name: "myapp",
    location: "/js/myapp"
}]

map: Permite mapear caminhos em identificadores de módulo para diferentes caminhos:

map: {
    dijit16: {
        dojo: "dojo16"
    }
}

caminhos: um mapa de fragmentos de ID de módulo para caminhos de arquivo:

var dojoConfig = {
    packages: [
        "package1",
        "package2"
    ],
    paths: {
        package1: "../lib/package1",
        package2: "/js/package2"
    }
};

    // ...is equivalent to:
var dojoConfig = {
    packages: [
        { name: "package1", location: "../lib/package1" },
        { name: "package2", location: "/js/package2" }
    ]
};

async: Define se o núcleo Dojo deve ser carregado de forma assíncrona. Valores pode ser true, false ou legacyAsync, o que coloca o carregador permanentemente no modo legado entre domínios.

async: true

parseOnLoad: Se true, analisa a página com dojo/parser quando o DOM e todas as dependências iniciais (incluindo aquelas no dojoConfig.deps array) foram carregados.

parseOnLoad: true

It is recommended that parseOnLoad be left at false (it defaults to false, so you can simply omit this property), and that developers explicitly require dojo/parser and call parser.parse().

deps: uma matriz de caminhos de recursos que devem ser carregados imediatamente uma vez O Dojo carregou:

deps: ["dojo/parser"]

callback: O callback a ser executado assim que os deps forem recuperados:

callback: function(parser) {
    // Use the resources provided here
}

waitSeconds: Quantidade de tempo a esperar antes de sinalizar o tempo limite de carregamento para um módulo; padrão para 0 (esperar para sempre):

waitSeconds: 5

cacheBust: Se true, anexa a hora como uma querystring a cada módulo URL para evitar o cache do módulo:

cacheBust: true

Agora vamos criar uma demonstração simples que coloca os parâmetros básicos em uso. Um cenário muito comum é usar o Dojo Toolkit da CDN com módulos locais.

Digamos que usamos o Google CDN com módulos no espaço /documentation/tutorials/1.10/dojo_config/demo:

<!-- Configure Dojo first -->
<script>
    dojoConfig = {
        has: {
            "dojo-firebug": true,
            "dojo-debug-messages": true
        },
        // Don't attempt to parse the page for widgets
        parseOnLoad: false,
        packages: [
            // Any references to a "demo" resource should load modules locally, *not* from CDN
            {
                name: "demo",
                location: "/documentation/tutorials/1.10/dojo_config/demo"
            }
        ],
        // Timeout after 10 seconds
        waitSeconds: 10,
        map: {
            // Instead of having to type "dojo/domReady!", we just want "ready!" instead
            "*": {
                ready: "dojo/domReady"
            }
        },
        // Get "fresh" resources
        cacheBust: true
    };
</script>

<!-- Load Dojo, Dijit, and DojoX resources from Google CDN -->
<script src="//ajax.googleapis.com/ajax/libs/dojo/1.10.4/dojo/dojo.js"></script>

<!-- Load a "demo" module -->

<script>
    require(["demo/AuthoredDialog", "dojo/parser", "ready!"], function(AuthoredDialog, parser) {
        // Parse the page
        parser.parse();

        // Do something with demo/AuthoredDialog...
    });
</script>

Usando a configuração de pacotes, fizemos todas as referências a demo/* apontarem para nosso diretório local /documentation/tutorials/1.10/dojo_config/demo/, enquanto permitimos qualquer referência a dojo, dijit, e dojox vindo do Google CDN. Se o pacote de demonstração não tivesse sido definido, a solicitação para demo/AuthoredDialog teria ido para //ajax.googleapis.com/ajax/libs/dojo/1.10.4/dojo/demo/AuthoredDialog.js. Também usamos alias, associando ready com dojo/domReady.

Carregar DojoConfig #

No exemplo abaixo, estamos criando um objeto javascript global dojoConfig que conterá todos os valores de configuração.

Observação: dojoConfig é definido em um bloco de script antes de o dojo.js ser carregado. Isso é de suma importância — se revertido, as propriedades de configuração serão ignoradas.

<script>
    dojoConfig= {
        has: {
            "dojo-firebug": true
        },
        parseOnLoad: false,
        foo: "bar",
        async: true
    };
</script>
<script src="//ajax.googleapis.com/ajax/libs/dojo/1.10.4/dojo/dojo.js"></script>

<script>
// Require the registry, parser, Dialog, and wait for domReady
require(["dijit/registry", "dojo/parser", "dojo/json", "dojo/_base/config", "dijit/Dialog", "dojo/domReady!"]
, function(registry, parser, JSON, config) {
    // Explicitly parse the page
    parser.parse();
    // Find the dialog
    var dialog = registry.byId("dialog");
    // Set the content equal to what dojo.config is
    dialog.set("content", "<pre>" + JSON.stringify(config, null, "\t") + "```");
    // Show the dialog
    dialog.show();
});
</script>

<!-- and later in the page -->
<div id="dialog" data-dojo-type="dijit/Dialog" data-dojo-props="title: 'dojoConfig / dojo/_base/config'"></div>
