Instalação ou Configuração #

Pré-requisitos

Este artigo demonstrou a instalação do oozie-4.3.0 no Hadoop 2.7.3

  1. Java 1.7+
  2. Hadoop 2.x (aqui, 2.7.3)
  3. Especialista3+
  4. Caixa Unix

Etapa 1: arquivo Dist

Obtenha o arquivo oozie tar.gz de http://www-eu.apache.org/dist/oozie/4.3.0/ e extraia-o

cd $HOME
tar -xvf oozie-4.3.0.tar.gz

Passo 2: Construir Oozie

cd $HOME/oozie-4.3.0/bin
./mkdistro.sh -DskipTests

Etapa 3: Instalação do Servidor

Copie os binários construídos para o diretório inicial como ‘oozie’

cd $HOME
cp -R $HOME/oozie-4.3.0/distro/target/oozie-4.3.0-distro/oozie-4.3.0 .

Passo 3.1: libext Crie o diretório libext dentro do diretório oozie

 cd $HOME/oozie
 mkdir libext

Observação: Biblioteca ExtJS (2.2+) (opcional, para habilitar o Oozie webconsole) Mas, a biblioteca ExtJS não é fornecida com o Oozie porque usa uma licença diferente :( Agora você precisa colocar os jars do hadoop dentro do diretório libext, senão ele lançará abaixo do erro no arquivo oozie.log

WARN ActionStartXCommand:523 - SERVER[data01.teg.io] USER[hadoop] GRUPO[-] TOKEN[] APP[map-reduce-wf] TRABALHO[0000000-161215143751620-oozie-hado-W] ACTION[0000000-161215143751620-oozie-hado-W@mr-node] Erro ao iniciar ação [mr-nó]. ErrorType [TRANSIENT], ErrorCode [JA009], Mensagem [JA009: Não é possível inicializar o cluster. Por favor, verifique sua configuração para mapreduce.framework.name e os endereços de servidor correspondentes.]

Então, vamos colocar abaixo os jars dentro do diretório libext

cp $HADOOP_HOME/share/hadoop/common/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/common/lib/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/hdfs/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/hdfs/lib/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/mapreduce/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/mapreduce/lib/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/yarn/*.jar oozie/libext/
cp $HADOOP_HOME/share/hadoop/yarn/lib/*.jar oozie/libext/

Etapa 3.2: Personificação de Oozie Para evitar o erro de representação no oozie, modifique o core-site.xml como abaixo

<!-- OOZIE -->
  <property>
    <name>hadoop.proxyuser.[OOZIE_SERVER_USER].hosts</name>
    <value>[OOZIE_SERVER_HOSTNAME]</value>
  </property>
  <property>
    <name>hadoop.proxyuser.[OOZIE_SERVER_USER].groups</name>
    <value>[USER_GROUPS_THAT_ALLOW_IMPERSONATION]</value>
  </property>

Supondo que meu usuário oozie seja huser e host seja localhost e grupo seja hadoop

<!-- OOZIE -->
  <property>
    <name>hadoop.proxyuser.huser.hosts</name>
    <value>localhost</value>
  </property>
  <property>
    <name>hadoop.proxyuser.huser.groups</name>
    <value>hadoop</value>
  </property>

Nota: Você pode usar * em todos os valores, em caso de confusão

Passo 3.3: Prepare a guerra

cd $HOME/oozie/bin
./oozie-setup.sh prepare-war

Isso criará o arquivo oozie.war dentro do diretório oozie. Se esta guerra for usada ainda mais, você poderá enfrentar este erro:

ERRO ActionStartXCommand:517 - SERVER[data01.teg.io] USER[hadoop] GRUPO[-] TOKEN[] APP[map-reduce-wf] TRABALHO[0000000-161220104605103-oozie-hado-W] AÇÃO [0000000-161220104605103-oozie-hado-W@mr-node] Erro, java.lang.NoSuchFieldError: HADOOP_CLASSPATH

Por quê? porque, a compilação oozie produziu jars do Hadoop 2.6.0 mesmo ao especificar o Hadoop 2.7.3 com a opção “-Dhadoop.version=2.7.3”.

Então, para evitar esse erro, copie o arquivo oozie.war para um diretório diferente

mkdir $HOME/oozie_war_dir
cp $HOME/oozie/oozie.war $HOME/oozie_war_dir
cd $HOME/oozie_war_dir
jar -xvf oozie.war
rm -f oozie.war/WEB-INF/lib/hadoop-*.jar
rm -f oozie.war/WEB-INF/lib/hive-*.jar
rm oozie.war
jar -cvf oozie.war ./*
cp oozie.war $HOME/oozie/

Em seguida, regenere os binários oozie.war para oozie com um prepare-war

cd $HOME/oozie/bin
./oozie-setup.sh prepare-war

Etapa 3.4: Criar sharelib no HDFS

cd $HOME/oozie/bin
./oozie-setup.sh sharelib create -fs hdfs://localhost:9000

Agora, esta configuração do sharelib pode fornecer o erro abaixo:

org.apache.oozie.service.ServiceException: E0104: Não foi possível totalmente inicializar serviço [org.apache.oozie.service.ShareLibService], não capaz de armazenar em cache sharelib. Um administrador precisa instalar o sharelib com oozie-setup.sh e emita o comando CLI ‘oozie admin’ para atualizar o sharelib

Para evitar isso, modifique oozie-site.xml como abaixo

cd $HOME/oozie
vi conf/oozie-site.xml

Adicionar propriedade abaixo

<property>
    <name>oozie.service.HadoopAccessorService.hadoop.configurations</name> 
    <value>*=/usr/local/hadoop/etc/hadoop/</value>
</property>

O valor deve ser seu $HADOOP_HOME/etc/hadoop, onde todos os arquivos de configuração do hadoop estão presentes.

Etapa 3.5: Criar banco de dados Oozie

cd $HOME/oozie
./bin/ooziedb.sh create -sqlfile oozie.sql -run

Passo 3.6: Iniciar Daemon

Para iniciar o Oozie como um daemon, use o seguinte comando:

./bin/oozied.sh start

Parar

./bin/oozied.sh stop

verifique os logs quanto a erros, se houver

cd $HOME/oozie/logs
tail -100f oozie.log

Use o seguinte comando para verificar o status do Oozie na linha de comando:

$ ./bin/oozie admin -oozie http://localhost:11000/oozie -status
System mode: NORMAL

Etapa 4: Instalação do cliente

$ cd
$ cp oozie/oozie-client-4.3.0.tar.gz .
$ tar -xvf oozie-client-4.3.0.tar.gz
$ mv oozie-client-3.3.2 oozie-client
$ cd bin

Adicione $HOME/oozie-client/bin à variável PATH no arquivo .bashrc e reinicie seu terminal ou faça

source $HOME/.bashrc

Para mais detalhes sobre a configuração, você pode consultar este URL https://oozie.apache.org/docs/4.3.0/DG_QuickStart.html

Agora você pode enviar trabalhos do hadoop para o oozie em seu terminal.

Para executar um exemplo, você pode seguir este URL e configurar seu primeiro exemplo para ser executado https://oozie.apache.org/docs/4.3.0/DG_Examples.html

Você pode enfrentar o erro abaixo ao executar o exemplo de redução de mapa no URL acima

java.io.IOException: java.net.ConnectException: Chamada de localhost.localdomain/127.0.0.1 a 0.0.0.0:10020 falhou na conexão exceção: java.net.ConnectException: Conexão recusada; Para mais detalhes veja: http://wiki.apache.org/hadoop/ConnectionRefused

Solução: Inicie mr-jobhistory-server.sh

cd $HADOOP_HOME/sbin
./mr-jobhistory-server.sh start historyserver

Outro ponto a ser observado sobre a modificação do arquivo job.properties é:

nameNode=hdfs://localhost:9000
jobTracker=localhost:8032

no seu caso, isso pode ser diferente, pois estou usando o apache hadoop, você pode estar usando cloudera/hdp/anything

To run spark job, I have tried running in local[*], yarn-client and
yarn-cluster as master, but succeeded in local[*] only
