Uma explicação detalhada é fornecida no trabalho de coordenador acionado por dados oozie com exemplo.

O coordenador é executado periodicamente desde o horário de início até o horário de término. Começando na hora de início, o trabalho do coordenador verifica se os dados de entrada estão disponíveis. Quando os dados de entrada ficam disponíveis, um fluxo de trabalho é iniciado para processar os dados de entrada que, ao serem concluídos, produzem os dados de saída necessários. Este processo é repetido a cada tick de frequência até o tempo final do coordenador.

    <done-flag>_SUCCESS</done_flag>

O trecho acima em Coordinator.xml para o conjunto de dados de entrada sinaliza a presença de dados de entrada. Isso significa que a ação do coordenador estará no estado WAITING até que o arquivo _SUCCESS esteja presente no diretório de entrada fornecido. Quando estiver presente, o fluxo de trabalho iniciará a execução.

amostra de coordenador oozie #

O trabalho de coordenador abaixo acionará a ação do coordenador uma vez por dia que executa um fluxo de trabalho. O fluxo de trabalho tem um script de shell que move a entrada para a saída.

<coordinator-app name="log_process_coordinator" frequency="${coord:days(1)}" start="2017-04-29T06:00Z" end="2018-04-29T23:25Z" timezone="UTC"
             xmlns="uri:oozie:coordinator:0.2">
<datasets>
    <dataset name="input_dataset" frequency="${coord:days(1)}" initial-instance="2017-04-29T06:00Z" timezone="GMT">
        <uri-template>${nameNode}/mypath/coord_job_example/input/${YEAR}${MONTH}${DAY}</uri-template>
        <done-flag>_SUCCESS</done-flag>
    </dataset>
    <dataset name="output_dataset" frequency="${coord:days(1)}" initial-instance="2017-04-29T06:00Z" timezone="GMT">
        <uri-template>${nameNode}/mypath/coord_job_example/output/${YEAR}${MONTH}${DAY}</uri-template>
        <done-flag>_SUCCESS</done-flag>
    </dataset>
</datasets>
<input-events>
    <data-in name="input_event" dataset="input_dataset">
        <instance>${coord:current(0)}</instance>
    </data-in>
</input-events>                
 <output-events>
    <data-out name="output_event" dataset="output_dataset">
        <instance>${coord:current(0)}</instance>
    </data-out>
</output-events>
 <action>
    <workflow>
        <app-path>${workflowAppUri}</app-path>
        <configuration>
            <property>
                <name>jobTracker</name>
                <value>${jobTracker}</value>
            </property>
            <property>
                <name>nameNode</name>
                <value>${nameNode}</value>
            </property>
            <property>
                <name>pool.name</name>
                <value>${poolName}</value>
            </property>
            <property>
                <name>inputDir</name>
                <value>${coord:dataIn('input_event')}</value>
            </property>
             <property>
                <name>outputDir</name>
                <value>${coord:dataOut('output_event')}</value>
            </property>
        </configuration>
    </workflow>
</action>

amostra de fluxo de trabalho oozie #

<workflow-app xmlns="uri:oozie:workflow:0.4" name="shell-wf">
<start to="shell-node"/>
<action name="shell-node">
 <shell xmlns="uri:oozie:shell-action:0.2">
<job-tracker>${jobTracker}</job-tracker>
<name-node>${nameNode}</name-node>
<configuration>
 <property>
   <name>mapred.job.queue.name</name>
   <value>${poolName}</value>
 </property>
</configuration>
<exec>${myscript}</exec>
<argument>${inputDir}</argument>
<argument>${outputDir}</argument>
<file>${myscriptPath}</file>
<capture-output/>
</shell>
<ok to="end"/>
<error to="fail"/>
</action>
 <kill name="fail">
  <message>Shell action failed, error message[${wf:errorMessage(wf:lastErrorNode())}]
  </message>
</kill>
 <end name="end"/>
</workflow-app>

job.amostra de propriedades #

nameNode=hdfs://namenode:port
start=2016-04-12T06:00Z
end=2017-02-26T23:25Z
jobTracker=yourjobtracker
poolName=yourpool
oozie.coord.application.path=${nameNode}/hdfs_path/coord_job_example/coord
workflowAppUri=${oozie.coord.application.path}
myscript=myscript.sh
myscriptPath=${oozie.coord.application.path}/myscript.sh

exemplo de script de shell #

inputDir=${1}
outputDir=${2}
hadoop fs -mkdir -p ${outputDir}
hadoop fs -cp ${inputDir}/* ${outputDir}/

enviando o trabalho do coordenador #

Copie o script, Coordinator.xml e workflow.xml no HDFS. coordenador.xml deve estar presente no diretório especificado por oozie.coord.application.path em job.properties. workflow.xml deve estar presente no diretório especificado por workflowAppUri. Quando tudo estiver no lugar, execute o comando abaixo do shell

oozie job -oozie <oozie_url>/oozie/ -config job.properties
