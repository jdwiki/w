Criando um pacote NuGet #

Para criar um pacote NuGet de um projeto, execute este comando em um diretório que contém project.json:

dotnet pack

O arquivo .nupkg resultante será nomeado e versionado de acordo com as propriedades em project.json. Se houver vários frameworks direcionados no arquivo de projeto, o pacote oferecerá suporte a todos eles.

Publicar e executar um projeto .NET Core #

Acesse o diretório project.json e publique:

dotnet publish

Ele imprimirá o diretório de saída da operação, entrará no diretório e executará o projeto publicado:

dotnet <project output>.dll

A pasta padrão será: <raiz do projeto>/bin/<configuração>/<estrutura de destino>/publish

Por exemplo: example/bin/Debug/netcoreapp1.0/publish

Se você criou o projeto anteriormente, pode publicar usando:

dotnet --no-build publish

Importante: certifique-se de publicar o projeto do mesmo usuário que restaurou os pacotes ou você pode publicá-lo sem as bibliotecas necessárias.

Você pode especificar a configuração com a opção -c <Configuration>. Para publicar no modo Release, use dotnet publish -c Release.

Criar projeto de console .NET Core “Hello World” #

Crie um novo project.json e um exemplo de Program.cs:

dotnet new

Restaure os pacotes necessários:

dotnet restore

Compile e execute o exemplo:

dotnet run

Andaimes de outros tipos de projeto #

Usar dotnet new criará um novo aplicativo de console. Para estruturar outros tipos de projetos, use o sinalizador -t ou --type:

dotnet new -t web
dotnet restore
dotnet run

Os modelos disponíveis variam de acordo com o idioma.

Modelos C# #

  • console (padrão) - Um aplicativo de console.
  • web - Um aplicativo ASP.NET Core.
  • lib - Uma biblioteca de classes.
  • xunittest - Um projeto de teste xUnit.

F# Modelos #

  • console (padrão) - Um aplicativo de console.
  • lib - Uma biblioteca de classes.

Projetos de andaimes em outros idiomas #

Por padrão, dotnet new cria projetos C#. Você pode usar o sinalizador -l ou --lang para desenvolver projetos em outros idiomas:

dotnet new -l f#
dotnet restore
dotnet run

Atualmente, dotnet new suporta C# e F#.

Executando testes automatizados #

A execução do dotnet test de dentro de uma pasta que contém um projeto de teste iniciará o executor de testes. O executor de teste descobrirá e executará os testes no projeto.

Para ser compatível com dotnet test, o arquivo project.json deve conter uma propriedade testRunner e uma dependência de um pacote de execução de teste compatível:

{
  "dependencies": {
    "dotnet-test-xunit": "2.2.0-preview2-build1029",
    "Microsoft.NETCore.App": {
      "type": "platform",
      "version": "1.0.0"
    },
    "xunit": "2.1.0"
  },
  "frameworks": {
    "netcoreapp1.0": {
      "imports": [ "dotnet", "portable-net45+win8" ]
    }
  },
  "testRunner": "xunit"
}
