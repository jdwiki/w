Esta seção discute os tipos de dados que o SQL Server pode usar, incluindo seu intervalo de dados, comprimento e limitações (se houver).

Números exatos #

Há duas classes básicas de tipos de dados numéricos exatos - Integer e Precisão fixa e escala.

Tipos de dados inteiros

  • pedaço
  • tinyint
  • smallint
  • int
  • bigint

Inteiros são valores numéricos que nunca contêm uma parte fracionária e sempre usam uma quantidade fixa de armazenamento. O intervalo e os tamanhos de armazenamento dos tipos de dados inteiros são mostrados nesta tabela:

Tipo de dados Gama Armazenamento
pouco 0 ou 1 1 bit **
tinyint 0 a 255 1 byte
smallint -2^15 (-32.768) a 2^15-1 (32.767) 2 bytes
int -2^31 (-2.147.483.648) a 2^31-1 (2.147.483.647) 4 bytes
bigint -2^63 (-9.223.372.036.854.775.808) a 2^63-1 (9.223.372.036.854.775.807) 8 bytes

Tipos de dados de precisão e escala fixos

  • numérico
  • decimal
  • dinheiro pequeno
  • dinheiro

Esses tipos de dados são úteis para representar números exatamente. Contanto que os valores possam caber dentro do intervalo dos valores armazenáveis ​​no tipo de dados, o valor não terá problemas de arredondamento. Isso é útil para quaisquer cálculos financeiros, onde os erros de arredondamento resultarão em insanidade clínica para os contadores.

Observe que decimal e numérico são sinônimos para o mesmo tipo de dados.

Tipo de dados Gama Armazenamento
Decimal [(p [, s])] ou Numérico [(p [, s])] -10^38 + 1 a 10^38 - 1 Consulte a tabela Precisão
Ao definir um tipo de dados decimal ou numérico, pode ser necessário especificar a Precisão [p] e Escala [s].

Precisão é o número de dígitos que podem ser armazenados. Por exemplo, se você precisasse armazenar valores entre 1 e 999, precisaria de uma Precisão de 3 (para manter os três dígitos em 100). Se você não especificar uma precisão, a precisão padrão será 18.

Escala é o número de dígitos após o ponto decimal. Se você precisasse armazenar um número entre 0,00 e 999,99, precisaria especificar uma Precisão de 5 (cinco dígitos) e uma Escala de 2 (dois dígitos após o ponto decimal). Você deve especificar uma precisão para especificar uma escala. A escala padrão é zero.

A Precisão de um tipo de dados decimal ou numérico define o número de bytes necessários para armazenar o valor, conforme mostrado abaixo:

Tabela de precisão

Precisão Bytes de armazenamento
1 - 9 5
10-19 9
20-28 13
29-38 17

Tipos de dados fixos monetários

Esses tipos de dados destinam-se especificamente a dados contábeis e outros dados monetários. Esses tipos têm uma escala fixa de 4 - você sempre verá quatro dígitos após a casa decimal. Para a maioria dos sistemas que trabalham com a maioria das moedas, usar um valor numérico com uma Escala de 2 será suficiente. Observe que nenhuma informação sobre o tipo de moeda representada é armazenada com o valor.

Tipo de dados Gama Armazenamento
dinheiro -922.337.203.685.477,5808 a 922.337.203.685.477,5807 8 bytes
dinheiro pequeno -214.748,3648 a 214.748,3647 4 bytes

Numéricos Aproximados #

  • flutuar [(n)]
  • real

Esses tipos de dados são usados ​​para armazenar números de ponto flutuante. Como esses tipos destinam-se a conter apenas valores numéricos aproximados, eles não devem ser usados ​​em casos em que qualquer erro de arredondamento seja inaceitável. No entanto, se você precisar lidar com números muito grandes ou com um número indeterminado de dígitos após a casa decimal, essa pode ser sua melhor opção.

| Tipo de dados | Gama | Tamanho | |————————-|———————– —————————-|———– —————-| | flutuar | -1,79E+308 a -2,23E-308, 0 e 2,23E-308 a 1,79E+308 | depende de n na tabela abaixo | reais | -3,40E + 38 a -1,18E - 38, 0 e 1,18E - 38 a 3,40E + 38 | 4 Bytes |

n tabela de valores para números float. Se nenhum valor for especificado na declaração do float, o valor padrão de 53 será usado. Observe que float(24) é o equivalente a um valor real.

valor n Precisão Tamanho
1-24 7 dígitos 4 bytes
25-53 15 dígitos 8 bytes
