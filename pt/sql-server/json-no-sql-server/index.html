Sintaxe #

  • JSON_VALUE(expression , path) – extrai um valor escalar de uma string JSON.
  • JSON_QUERY( expressão [ , caminho ] ) – Extrai um objeto ou uma matriz de uma string JSON.
  • OPENJSON( jsonExpression [ , path ] ) – função de valor de tabela que analisa o texto JSON e retorna objetos e propriedades em JSON como linhas e colunas.
  • ISJSON( expression ) – Testa se uma string contém JSON válido.
  • JSON_MODIFY( expression , path , newValue ) – Atualiza o valor de uma propriedade em uma string JSON e retorna a string JSON atualizada.

Parâmetros #

Parâmetros Detalhes
expressão Normalmente, o nome de uma variável ou coluna que contém texto JSON.
caminho Uma expressão de caminho JSON que especifica a propriedade a ser atualizada. path tem a seguinte sintaxe: [append] [ lax | estrito ] $.<caminho json>
jsonExpressão É uma expressão de caractere Unicode que contém o texto JSON.

A função OPENJSON está disponível apenas no nível de compatibilidade 130. Se o nível de compatibilidade do banco de dados for inferior a 130, o SQL Server não poderá localizar e executar a função OPENJSON. Atualmente, todos os bancos de dados SQL do Azure são definidos como 120 por padrão. Você pode alterar o nível de compatibilidade de um banco de dados usando o seguinte comando:

ALTER DATABASE <Database-Name-Here> SET COMPATIBILITY_LEVEL = 130

Formate os resultados da consulta como JSON com FOR JSON #

Dados da tabela de entrada (tabela Pessoas)

ID Nome Idade
1 João 23
2 Jane 31

Consulta

SELECT Id, Name, Age
FROM People
FOR JSON PATH

Resultado

[
    {"Id":1,"Name":"John","Age":23},
    {"Id":2,"Name":"Jane","Age":31}
]

Analisar texto JSON #

As funções JSON_VALUE e JSON_QUERY analisam o texto JSON e retornam valores escalares ou objetos/matrizes no caminho no texto JSON.

DECLARE @json NVARCHAR(100) = '{"id": 1, "user":{"name":"John"}, "skills":["C#","SQL"]}'

SELECT
    JSON_VALUE(@json, '$.id') AS Id,
    JSON_VALUE(@json, '$.user.name') AS Name,
    JSON_QUERY(@json, '$.user') AS UserObject,
    JSON_QUERY(@json, '$.skills') AS Skills,
    JSON_VALUE(@json, '$.skills[0]') AS Skill0

Resultado

ID Nome UserObject Habilidades Habilidade0
1 João {“name”:“João”} [“C#”,“SQL”] C#

Junte-se a entidades JSON pai e filho usando CROSS APPLY OPENJSON #

Junte objetos pai com suas entidades filho, por exemplo, queremos uma tabela relacional de cada pessoa e seus hobbies

DECLARE @json nvarchar(1000) =
N'[
    {
        "id":1,
        "user":{"name":"John"},
        "hobbies":[
            {"name": "Reading"},
            {"name": "Surfing"}
        ]
    },
    {
        "id":2,
        "user":{"name":"Jane"},
        "hobbies":[
            {"name": "Programming"},
            {"name": "Running"}
        ]
    }
 ]'

Consulta

SELECT 
    JSON_VALUE(person.value, '$.id') as Id,
    JSON_VALUE(person.value, '$.user.name') as PersonName,
    JSON_VALUE(hobbies.value, '$.name') as Hobby
FROM OPENJSON (@json) as person
    CROSS APPLY OPENJSON(person.value, '$.hobbies') as hobbies

Alternativamente, esta consulta pode ser escrita usando a cláusula WITH.

SELECT 
    Id, person.PersonName, Hobby
FROM OPENJSON (@json)
WITH(
    Id int '$.id',
    PersonName nvarchar(100) '$.user.name',
    Hobbies nvarchar(max) '$.hobbies' AS JSON
) as person
CROSS APPLY OPENJSON(Hobbies)
WITH(
    Hobby nvarchar(100) '$.name'
)

Resultado

Id PessoaNome Hobby
1 João Leitura
1 João Surfe
2 Jane Programação
2 Jane Correndo

Índice em propriedades JSON usando colunas computadas #

Ao armazenar documentos JSON no SQL Server, precisamos filtrar e classificar com eficiência os resultados da consulta nas propriedades dos documentos JSON.

CREATE TABLE JsonTable
(
    id int identity primary key,
    jsonInfo nvarchar(max),
    CONSTRAINT [Content should be formatted as JSON]
    CHECK (ISJSON(jsonInfo)>0)
)

INSERT INTO JsonTable
VALUES(N'{"Name":"John","Age":23}'),
(N'{"Name":"Jane","Age":31}'),
(N'{"Name":"Bob","Age":37}'),
(N'{"Name":"Adam","Age":65}')
GO

Dada a tabela acima Se quisermos encontrar a linha com o nome = ‘Adam’, executaríamos a seguinte consulta.

SELECT * 
FROM JsonTable Where 
JSON_VALUE(jsonInfo, '$.Name') = 'Adam'

No entanto, isso exigirá que o SQL Server execute uma tabela completa, o que em uma tabela grande não é eficiente.

Para acelerar isso, gostaríamos de adicionar um índice, mas não podemos referenciar diretamente as propriedades no documento JSON. A solução é adicionar uma coluna computada no caminho JSON $.Name e, em seguida, adicionar um índice na coluna computada.

ALTER TABLE JsonTable
ADD vName as JSON_VALUE(jsonInfo, '$.Name')

CREATE INDEX idx_name
ON JsonTable(vName)

Agora, quando executamos a mesma consulta, em vez de uma varredura de tabela completa, o SQL Server usa um índice para buscar no índice não clusterizado e encontrar as linhas que atendem às condições especificadas.

Nota: Para o SQL Server usar o índice, você deve criar a coluna computada com a mesma expressão que planeja usar em suas consultas - neste exemplo JSON_VALUE(jsonInfo, '$.Name'), mas você também pode usar o nome da coluna computada vName

Formate uma linha da tabela como um único objeto JSON usando FOR JSON #

WITHOUT_ARRAY_WRAPPER na cláusula FOR JSON removerá colchetes de matriz da saída JSON. Isso é útil se você estiver retornando uma única linha na consulta.

Nota: esta opção produzirá uma saída JSON inválida se mais de uma linha for retornada.

Dados da tabela de entrada (tabela Pessoas)

ID Nome Idade
1 João 23
2 Jane 31

Consulta

SELECT Id, Name, Age
FROM People
WHERE Id = 1
FOR JSON PATH, WITHOUT_ARRAY_WRAPPER

Resultado

{"Id":1,"Name":"John","Age":23}

Analisar texto JSON usando a função OPENJSON #

A função OPENJSON analisa o texto JSON e retorna várias saídas. Os valores que devem ser retornados são especificados usando os caminhos definidos na cláusula WITH. Se um caminho não for especificado para alguma coluna, o nome da coluna será usado como caminho. Esta função converte valores retornados para os tipos SQL definidos na cláusula WITH. A opção AS JSON deve ser especificada na definição da coluna se algum objeto/matriz deve ser retornado.

DECLARE @json NVARCHAR(100) = '{"id": 1, "user":{"name":"John"}, "skills":["C#","SQL"]}'

SELECT * 
FROM OPENJSON (@json)
    WITH(Id int '$.id',
        Name nvarchar(100) '$.user.name',
        UserObject nvarchar(max) '$.user' AS JSON,
        Skills nvarchar(max) '$.skills' AS JSON,
        Skill0 nvarchar(20) '$.skills[0]')

Resultado

ID Nome UserObject Habilidades Habilidade0
1 João {“name”:“João”} [“C#”,“SQL”] C#
