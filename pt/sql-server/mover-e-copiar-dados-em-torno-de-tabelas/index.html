Mover e copiar dados em torno de tabelas

  • Copia dados de uma tabela para outra
  • Copie dados em uma tabela, criando essa tabela em tempo real
  • Mova os dados para uma tabela (assumindo o método de chaves exclusivas)

Nesta página

  • Copia dados de uma tabela para outra
  • Copie dados em uma tabela, criando essa tabela em tempo real
  • Mova os dados para uma tabela (assumindo o método de chaves exclusivas)

Copia dados de uma tabela para outra #

Este código seleciona dados de uma tabela e os exibe na ferramenta de consulta (geralmente SSMS)

SELECT Column1, Column2, Column3 FROM MySourceTable;

Este código insere esses dados em uma tabela:

INSERT INTO MyTargetTable (Column1, Column2, Column3)
SELECT Column1, Column2, Column3 FROM MySourceTable;

Copie dados em uma tabela, criando essa tabela em tempo real #

Este código seleciona dados de uma tabela:

SELECT Column1, Column2, Column3 FROM MySourceTable;

Este código cria uma nova tabela chamada MyNewTable e coloca esses dados nela

SELECT Column1, Column2, Column3 
INTO MyNewTable
FROM MySourceTable;

Mova os dados para uma tabela (assumindo o método de chaves exclusivas) #

Para mover dados, você primeiro os insere no destino e, em seguida, exclui o que você inseriu da tabela de origem. Esta não é uma operação SQL normal, mas pode ser esclarecedora

O que você inseriu? Normalmente, em bancos de dados, você precisa ter uma ou mais colunas que possam ser usadas para identificar exclusivamente as linhas, portanto, assumiremos isso e faremos uso disso.

Esta instrução seleciona algumas linhas

SELECT Key1, Key2, Column3, Column4 FROM MyTable;

Primeiro, inserimos isso em nossa tabela de destino:

INSERT INTO TargetTable (Key1, Key2, Column3, Column4)
SELECT Key1, Key2, Column3, Column4 FROM MyTable;

Agora assumindo que os registros em ambas as tabelas são únicos em Key1,Key2, podemos usar isso para localizar e excluir dados da tabela de origem

DELETE MyTable
WHERE EXISTS (
    SELECT * FROM TargetTable 
    WHERE TargetTable.Key1 = SourceTable.Key1
    AND TargetTable.Key2 = SourceTable.Key2
);

Isso só funcionará corretamente se Key1, Key2 forem únicos em ambas as tabelas

Por último, não queremos o trabalho pela metade. Se encerrarmos isso em uma transação, todos os dados serão movidos ou nada acontecerá. Isso garante que não inserimos os dados e não conseguimos excluir os dados da fonte.

BEGIN TRAN;

INSERT INTO TargetTable (Key1, Key2, Column3, Column4)
SELECT Key1, Key2, Column3, Column4 FROM MyTable;


DELETE MyTable
WHERE EXISTS (
    SELECT * FROM TargetTable 
    WHERE TargetTable.Key1 = SourceTable.Key1
    AND TargetTable.Key2 = SourceTable.Key2
);

COMMIT TRAN;
