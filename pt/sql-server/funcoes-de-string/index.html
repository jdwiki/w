Nome da cotação #

Retorna uma string Unicode cercada por delimitadores para torná-la um identificador delimitado válido do SQL Server.

Parâmetros:

  1. cadeia de caracteres. Uma string de dados Unicode, de até 128 caracteres (sysname). Se uma string de entrada tiver mais de 128 caracteres, a função retornará null.

  2. personagem de citação. Opcional. Um único caractere para usar como delimitador. Pode ser uma aspa simples (' ou ``), um colchete esquerdo ou direito ({,[,(,< ou >,),], }) ou aspas duplas ("). Qualquer outro valor retornará nulo. O valor padrão é colchetes.

    SELECT QUOTENAME(‘what’’s my name?’) – Returns [what’s my name?]

    SELECT QUOTENAME(‘what’’s my name?’, ‘[’) – Returns [what’s my name?] SELECT QUOTENAME(‘what’’s my name?’, ‘]’) – Returns [what’s my name?]

    SELECT QUOTENAME(‘what’’s my name?’, ‘’’’) – Returns ‘what’’s my name?’

    SELECT QUOTENAME(‘what’’s my name?’, ‘"’) – Returns “what’s my name?”

    SELECT QUOTENAME(‘what’’s my name?’, ‘)’) – Returns (what’s my name?) SELECT QUOTENAME(‘what’’s my name?’, ‘(’) – Returns (what’s my name?)

    SELECT QUOTENAME(‘what’’s my name?’, ‘<’) – Returns <what’s my name?> SELECT QUOTENAME(‘what’’s my name?’, ‘>’) – Returns <what’s my name?>

    SELECT QUOTENAME(‘what’’s my name?’, ‘{’) – Returns {what’s my name?} SELECT QUOTENAME(‘what’’s my name?’, ‘}’) – Returns {what’s my name?}

    SELECT QUOTENAME(‘what’’s my name?’, ‘') -- Returns what’s my name?`

Substituir #

Retorna uma string (varchar ou nvarchar) onde todas as ocorrências de uma substring especificada são substituídas por outra substring.

Parâmetros:

  1. expressão de string. Esta é a string que seria pesquisada. Pode ser um tipo de dados de caractere ou binário.

  2. padrão. Esta é a substring que seria substituída. Pode ser um tipo de dados de caractere ou binário. O argumento padrão não pode ser uma string vazia.

  3. substituição. Esta é a substring que substituiria a substring padrão. Pode ser um caractere ou dados binários.

    SELECT REPLACE(‘This is my string’, ‘is’, ‘XX’) – Returns ‘ThXX XX my string’.

Notas:

  • Se a expressão de string não for do tipo varchar(max) ou nvarchar(max), a função replace trunca o valor de retorno em 8.000 caracteres.
  • O tipo de dados de retorno depende dos tipos de dados de entrada - retorna nvarchar se um dos valores de entrada for nvarchar, ou varchar caso contrário.
  • Retorna NULL se algum dos parâmetros de entrada for NULL

Substring #

Retorna uma substring que começa com o char que está no índice inicial especificado e o comprimento máximo especificado.

Parâmetros:

  1. Expressão de caracteres. A expressão de caractere pode ser de qualquer tipo de dados que pode ser convertido implicitamente em varchar ou nvarchar, exceto para text ou ntext.

  2. Inicie o índice. Um número (int ou bigint) que especifica o índice inicial da substring solicitada. (Nota: as strings no sql server são o índice de base 1, o que significa que o primeiro caractere da string é o índice 1). Este número pode ser inferior a 1. Neste caso, se a soma do índice inicial e comprimento máximo for superior a 0, o texto de retorno será um texto que começa no primeiro caractere da expressão de caracteres e com o comprimento de (índice inicial + comprimento máximo - 1). Se for menor que 0, uma string vazia será retornada.

  3. Comprimento máximo. Um número inteiro entre 0 e o valor máximo de bigint (9.223.372.036.854.775.807). Se o parâmetro de comprimento máximo for negativo, um erro será gerado.

    SELECT SUBSTRING(‘This is my string’, 6, 5) – returns ‘is my’

Se o comprimento máximo + índice inicial for maior que o número de caracteres na string, a string inteira será retornada.

SELECT SUBSTRING('Hello World',1,100) -- returns 'Hello World'

Se o índice inicial for maior que o número de caracteres na string, uma string vazia será retornada.

SELECT SUBSTRING('Hello World',15,10) -- returns ''

String_Split #

Divide uma expressão de string usando um separador de caracteres. Observe que STRING_SPLIT() é uma função com valor de tabela e, portanto, deve ser usada dentro da cláusula FROM.

Parâmetros:

  1. corda. Qualquer expressão de tipo de caractere (char, nchar, varchar ou nvarchar)
  2. separador. Uma expressão de caractere único de qualquer tipo (char(1), nchar(1), varchar(1) ou nvarchar(1)).

Retorna uma tabela de coluna única em que cada linha contém um fragmento da string. O nome das colunas é value, e o tipo de dados é nvarchar se algum dos parâmetros for nchar ou nvarchar, caso contrário, varchar.

O exemplo a seguir divide uma string usando espaço como separador:

SELECT value FROM STRING_SPLIT('Lorem ipsum dolor sit amet.', ' ');

Resultado:

value
-----
Lorem
ipsum
dolor
sit
amet.

Observações:

A função STRING_SPLIT está disponível apenas no nível de compatibilidade 130. Se o nível de compatibilidade do banco de dados for inferior a 130, o SQL Server não poderá localizar e executar a função STRING_SPLIT. Você pode alterar o nível de compatibilidade de um banco de dados usando o seguinte comando:

ALTER DATABASE [database_name] SET COMPATIBILITY_LEVEL = 130

Versões mais antigas do sql server não possuem uma função de string dividida embutida. Existem muitas funções definidas pelo usuário que tratam do problema de dividir uma string. Você pode ler o artigo de Aaron Bertrand Split strings the right way – or the next best way para uma comparação abrangente de alguns deles .

Deixei #

Retorna uma substring começando com o caractere mais à esquerda de uma string e até o comprimento máximo especificado.

Parâmetros:

  1. expressão de caracteres. A expressão de caractere pode ser de qualquer tipo de dados que pode ser convertido implicitamente em varchar ou nvarchar, exceto para text ou ntext

  2. comprimento máximo. Um número inteiro entre 0 e o valor máximo de bigint (9.223.372.036.854.775.807). Se o parâmetro de comprimento máximo for negativo, um erro será gerado.

    SELECT LEFT(‘This is my string’, 4) – result: ‘This’

Se o comprimento máximo for maior que o número de caracteres na string, a string inteira será retornada.

SELECT LEFT('This is my string', 50) -- result: 'This is my string'

Certo #

Retorna uma substring que é a parte mais à direita da string, com o comprimento máximo especificado.

Parâmetros:

  1. expressão de caracteres. A expressão de caractere pode ser de qualquer tipo de dados que pode ser convertido implicitamente em varchar ou nvarchar, exceto para text ou ntext

  2. comprimento máximo. Um número inteiro entre 0 e o valor máximo de bigint (9.223.372.036.854.775.807). Se o parâmetro de comprimento máximo for negativo, um erro será gerado.

    SELECT RIGHT(‘This is my string’, 6) – returns ‘string’

Se o comprimento máximo for maior que o número de caracteres na string, a string inteira será retornada.

SELECT RIGHT('This is my string', 50) -- returns 'This is my string'

Soundex #

Retorna um código de quatro caracteres (varchar) para avaliar a similaridade fonética de duas strings.

Parâmetros:

  1. expressão de caracteres. Uma expressão alfanumérica de dados de caracteres.

A função soundex cria um código de quatro caracteres que se baseia em como a expressão do caractere soaria quando falada. o primeiro caractere é a versão maiúscula do primeiro caractere do parâmetro, os 3 caracteres restantes são números que representam as letras na expressão (exceto a, e, i, o, u, h, w e y que são ignorados) .

SELECT SOUNDEX ('Smith') -- Returns 'S530'

SELECT SOUNDEX ('Smythe') -- Returns 'S530'

Formato #

Retorna um valor NVARCHAR formatado com o formato e a cultura especificados (se especificado). Isso é usado principalmente para converter tipos de data e hora em strings.

Parâmetros:

  1. ‘valor’. Uma expressão de um tipo de dados com suporte para formatar. tipos válidos estão listados abaixo.
  2. ‘formato’. Um padrão de formato NVARCHAR. Consulte a documentação oficial da Microsoft para strings de formato padrão e personalizado.
  3. ‘cultura’. Opcional. argumento nvarchar especificando uma cultura. O valor padrão é a cultura da sessão atual.

ENCONTRO

Usando strings de formato padrão:

DECLARE @d DATETIME = '2016-07-31';  

SELECT 
    FORMAT ( @d, 'd', 'en-US' ) AS 'US English Result' -- Returns '7/31/2016'
   ,FORMAT ( @d, 'd', 'en-gb' ) AS 'Great Britain English Result' -- Returns '31/07/2016'
   ,FORMAT ( @d, 'd', 'de-de' ) AS 'German Result' -- Returns '31.07.2016'
   ,FORMAT ( @d, 'd', 'zh-cn' ) AS 'Simplified Chinese (PRC) Result' -- Returns '2016/7/31'
   ,FORMAT ( @d, 'D', 'en-US' ) AS 'US English Result' -- Returns 'Sunday, July 31, 2016'
   ,FORMAT ( @d, 'D', 'en-gb' ) AS 'Great Britain English Result' -- Returns '31 July 2016'
   ,FORMAT ( @d, 'D', 'de-de' ) AS 'German Result' -- Returns 'Sonntag, 31. Juli 2016'

Usando strings de formato personalizado:

SELECT FORMAT( @d, 'dd/MM/yyyy', 'en-US' ) AS 'DateTime Result' -- Returns '31/07/2016'
      ,FORMAT(123456789,'###-##-####') AS 'Custom Number Result' -- Returns '123-45-6789',
      ,FORMAT( @d,'dddd, MMMM dd, yyyy hh:mm:ss tt','en-US') AS 'US' -- Returns 'Sunday, July 31, 2016 12:00:00 AM'
      ,FORMAT( @d,'dddd, MMMM dd, yyyy hh:mm:ss tt','hi-IN') AS 'Hindi' -- Returns रविवार, जुलाई 31, 2016 12:00:00 पूर्वाह्न
      ,FORMAT ( @d, 'dddd', 'en-US' )  AS 'US' -- Returns 'Sunday'
      ,FORMAT ( @d, 'dddd', 'hi-IN' )  AS 'Hindi' -- Returns 'रविवार'

FORMAT também pode ser usado para formatar CURRENCY,PERCENTAGE e NUMBERS.

MOEDA

DECLARE @Price1 INT = 40
SELECT FORMAT(@Price1,'c','en-US') AS 'CURRENCY IN US Culture' -- Returns '$40.00'      
       ,FORMAT(@Price1,'c','de-DE') AS 'CURRENCY IN GERMAN Culture' -- Returns '40,00 €'

Podemos especificar o número de dígitos após o decimal.

DECLARE @Price DECIMAL(5,3) = 40.356
SELECT FORMAT( @Price, 'C') AS 'Default', -- Returns '$40.36'
       FORMAT( @Price, 'C0') AS 'With 0 Decimal', -- Returns '$40'
       FORMAT( @Price, 'C1') AS 'With 1 Decimal', -- Returns '$40.4'
       FORMAT( @Price, 'C2') AS 'With 2 Decimal', -- Returns '$40.36'

Anotações importantes:

  • FORMAT retorna NULL para erros que não sejam uma cultura inválida. Por exemplo, NULL é retornado se o valor especificado no formato não for válido.
  • FORMAT depende da presença do .NET Framework Common Language Runtime (CLR).
  • FORMAT depende de regras de formatação CLR que determinam que os dois pontos e pontos devem ser escapados. Portanto, quando a string de formato (segundo parâmetro) contém dois pontos ou ponto, os dois pontos ou ponto devem ser escapados com barra invertida quando um valor de entrada (primeiro parâmetro) for do tipo de dados time.

ASCII #

Retorna um valor int que representa o código ASCII do caractere mais à esquerda de uma string.

SELECT ASCII('t') -- Returns 116
SELECT ASCII('T') -- Returns 84
SELECT ASCII('This') -- Returns 84

Se a string for Unicode e o caractere mais à esquerda não for ASCII, mas representável no agrupamento atual, um valor maior que 127 poderá ser retornado:

SELECT ASCII(N'ï') -- returns 239 when `SERVERPROPERTY('COLLATION') = 'SQL_Latin1_General_CP1_CI_AS'`

Se a string for Unicode e o caractere mais à esquerda não puder ser representado na ordenação atual, o valor int de 63 será retornado: (que representa o ponto de interrogação em ASCII):

SELECT ASCII(N'߷') -- returns 63
SELECT ASCII(nchar(2039)) -- returns 63

Caracteres #

Retorna um char representado por um código int ASCII.

SELECT CHAR(116) -- Returns 't'
SELECT CHAR(84)  -- Returns 'T'

Isto pode ser usado para introduzir nova linha/alimentação de linha CHAR(10), retornos de carro CHAR(13), etc. Veja AsciiTable.com para referência.

Se o valor do argumento não estiver entre 0 e 255, a função CHAR retornará NULL. O tipo de dados de retorno da função CHAR é char(1)

Concatenar #

Retorna uma string que é o resultado de duas ou mais strings unidas. CONCAT aceita dois ou mais argumentos.

SELECT CONCAT('This', ' is', ' my', ' string') -- returns 'This is my string'

Nota: Ao contrário da concatenação de strings usando o operador de concatenação de strings (+), ao passar um valor nulo para a função concat, ele o converterá implicitamente em uma string vazia:

SELECT CONCAT('This', NULL, ' is', ' my', ' string'), -- returns 'This is my string'
       'This' + NULL + ' is' + ' my' + ' string' -- returns NULL.

Também os argumentos de um tipo não string serão convertidos implicitamente em uma string:

SELECT CONCAT('This', ' is my ', 3, 'rd string') -- returns 'This is my 3rd string'

As variáveis ​​do tipo não string também serão convertidas para o formato string, não há necessidade de converter manualmente ou convertê-lo em string:

DECLARE @Age INT=23;
SELECT CONCAT('Ram is ', @Age,' years old');  -- returns 'Ram is 23 years old'

Versões mais antigas não suportam a função CONCAT e devem usar o operador de concatenação de strings (+). Tipos não string devem ser convertidos em tipos string para concatená-los dessa maneira.

SELECT 'This is the number ' + CAST(42 AS VARCHAR(5)) --returns 'This is the number 42'

Parâmetros:

Parâmetros:

  1. expressão de caracteres. Qualquer expressão de caractere ou dados binários que podem ser convertidos implicitamente em varcher, exceto text, ntext e image.

PatIndex #

Retorna a posição inicial da primeira ocorrência de um padrão especificado na expressão especificada.

Parâmetros:

  1. padrão. Uma expressão de caractere contém a sequência a ser encontrada. Limitado a Um comprimento máximo de 8000 caracteres. Curingas (%, _) podem ser usados ​​no padrão. Se o padrão não começar com um curinga, ele poderá corresponder apenas ao que estiver no início da expressão. Se não terminar com um curinga, pode corresponder apenas ao que estiver no final da expressão.

  2. expressão. Qualquer tipo de dados de string.

    SELECT PATINDEX(’%ter%’, ‘interesting’) – Returns 3.

    SELECT PATINDEX(’%t_r%t%’, ‘interesting’) – Returns 3.

    SELECT PATINDEX(’ter%’, ‘interesting’) – Returns 0, since ’ter’ is not at the start.

    SELECT PATINDEX(‘inter%’, ‘interesting’) – Returns 1.

    SELECT PATINDEX(’%ing’, ‘interesting’) – Returns 9.

Retorna uma string (varchar) de espaços repetidos.

Parâmetros:

  1. expressão inteira. Qualquer expressão inteira, até 8000. Se negativo, null é retornado. se 0, uma string vazia é retornada. (Para retornar uma string com mais de 8.000 espaços, use Replicar.

    SELECT SPACE(-1) – Returns NULL SELECT SPACE(0) – Returns an empty string SELECT SPACE(3) – Returns ’ ’ (a string containing 3 spaces)

Diferença #

Retorna um valor inteiro (int) que indica a diferença entre os valores soundex de duas expressões de caracteres.

Parâmetros:

  1. expressão de caractere 1.
  2. expressão de caracteres 2.

Ambos os parâmetros são expressões alfanuméricas de dados de caracteres.

O inteiro retornado é o número de caracteres nos valores soundex dos parâmetros que são os mesmos, então 4 significa que as expressões são muito semelhantes e 0 significa que elas são muito diferentes.

SELECT  SOUNDEX('Green'),  -- G650
        SOUNDEX('Greene'),  -- G650
        DIFFERENCE('Green','Greene') -- Returns 4
        
SELECT  SOUNDEX('Blotchet-Halls'),  -- B432
        SOUNDEX('Greene'),  -- G650
        DIFFERENCE('Blotchet-Halls', 'Greene') -- Returns 0

Len #

Retorna o número de caracteres de uma string. Nota: a função LEN ignora espaços à direita:

SELECT LEN('My string'), -- returns 9
       LEN('My string   '), -- returns 9
       LEN('   My string') -- returns 12

Se o comprimento incluindo os espaços à direita for desejado, existem várias técnicas para conseguir isso, embora cada uma tenha suas desvantagens. Uma técnica é anexar um único caractere à string e, em seguida, usar o LEN menos um:

DECLARE @str varchar(100) = 'My string   '
SELECT LEN(@str + 'x') - 1 -- returns 12

A desvantagem disso é que, se o tipo da variável ou coluna de string tiver o comprimento máximo, o acréscimo do caractere extra será descartado e o comprimento resultante ainda não contará os espaços à direita. Para resolver isso, a versão modificada a seguir resolve o problema e fornece os resultados corretos em todos os casos à custa de uma pequena quantidade de tempo de execução adicional e por causa disso (resultados corretos, inclusive com pares substitutos e velocidade de execução razoável) parece ser a melhor técnica a ser usada:

SELECT LEN(CONVERT(NVARCHAR(MAX), @str) + 'x') - 1

Outra técnica é usar a função DATALENGTH.

DECLARE @str varchar(100) = 'My string   '
SELECT DATALENGTH(@str) -- returns 12

É importante notar que DATALENGTH retorna o comprimento em bytes da string na memória. Isso será diferente para varchar vs. nvarchar.

DECLARE @str nvarchar(100) = 'My string   '
SELECT DATALENGTH(@str) -- returns 24

Você pode ajustar isso dividindo o comprimento de dados da string pelo comprimento de dados de um único caractere (que deve ser do mesmo tipo). O exemplo abaixo faz isso e também trata do caso em que a string de destino está vazia, evitando assim uma divisão por zero.

DECLARE @str nvarchar(100) = 'My string   '
SELECT DATALENGTH(@str) / DATALENGTH(LEFT(LEFT(@str, 1) + 'x', 1)) -- returns 12

Mesmo isso, porém, tem um problema no SQL Server 2012 e superior. Ele produzirá resultados incorretos quando a string contiver pares substitutos (alguns caracteres podem ocupar mais bytes do que outros caracteres na mesma string).

Outra técnica é usar REPLACE para converter espaços em um caractere que não seja espaço e obter o LEN do resultado. Isso dá resultados corretos em todos os casos, mas tem uma velocidade de execução muito baixa com strings longas.

Replicar #

Repete um valor de string um número especificado de vezes.

Parâmetros:

  1. expressão de string. A expressão de string pode ser uma string de caracteres ou dados binários.

  2. expressão inteira. Qualquer tipo inteiro, incluindo bigint. Se negativo, null é retornado. Se 0, uma string vazia é retornada.

    SELECT REPLICATE(‘a’, -1) – Returns NULL

    SELECT REPLICATE(‘a’, 0) – Returns '’

    SELECT REPLICATE(‘a’, 5) – Returns ‘aaaaa’

    SELECT REPLICATE(‘Abc’, 3) – Returns ‘AbcAbcAbc’

Observação: Se a expressão de string não for do tipo varchar(max) ou nvarchar(max), o valor de retorno não excederá 8000 caracteres. A replicação será interrompida antes de adicionar a string que fará com que o valor de retorno exceda esse limite:

SELECT LEN(REPLICATE('a b c d e f g h i j k l', 350)) -- Returns 7981

SELECT LEN(REPLICATE(cast('a b c d e f g h i j k l' as varchar(max)), 350)) -- Returns 8050

Rua #

Retorna dados de caracteres (varchar) convertidos de dados numéricos.

Parâmetros:

  1. expressão flutuante. Um tipo de dados numérico aproximado com um ponto decimal.

  2. comprimento. opcional. O comprimento total da expressão de string que retornaria, incluindo dígitos, ponto decimal e espaços à esquerda (se necessário). O valor padrão é 10.

  3. decimal. opcional. O número de dígitos à direita do ponto decimal. Se for maior que 16, o resultado será truncado para dezesseis casas à direita do ponto decimal.

    SELECT STR(1.2) – Returns ’ 1’

    SELECT STR(1.2, 3) – Returns ’ 1’

    SELECT STR(1.2, 3, 2) – Returns ‘1.2’

    SELECT STR(1.2, 5, 2) – Returns ’ 1.20’

    SELECT STR(1.2, 5, 5) – Returns ‘1.200’

    SELECT STR(1, 5, 2) – Returns ’ 1.00’

    SELECT STR(1) – Returns ’ 1’

Índice de caracteres #

Retorna o índice inicial de uma primeira ocorrência de expressão de string dentro de outra expressão de string.

Lista de parâmetros:

  1. String para encontrar (até 8.000 caracteres)
  2. String para pesquisar (qualquer tipo e comprimento de dados de caractere válido, incluindo binário)
  3. Índice (opcional) para iniciar. Um número do tipo int ou big int. Se omitido ou menor que 1, a pesquisa começa no início da string.

Se a string a ser pesquisada for varchar(max), nvarchar(max) ou varbinary(max), a função CHARINDEX retornará um valor bigint. Caso contrário, ele retornará um int.

SELECT CHARINDEX('is', 'this is my string') -- returns 3
SELECT CHARINDEX('is', 'this is my string', 4) -- returns 6
SELECT CHARINDEX(' is', 'this is my string') -- returns 5
