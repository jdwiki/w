Habilita o armazenamento de consultas no banco de dados #

O armazenamento de consultas pode ser habilitado no banco de dados usando o seguinte comando:

ALTER DATABASE tpch SET QUERY_STORE = ON

O SQL Server/Azure SQL Database coletará informações sobre consultas executadas e fornecerá informações nas exibições sys.query_store:

  • sys.query_store_query
  • sys.query_store_query_text
  • sys.query_store_plan
  • sys.query_store_runtime_stats
  • sys.query_store_runtime_stats_interval
  • sys.database_query_store_options
  • sys.query_context_settings

Obtenha estatísticas de execução para consultas/planos SQL #

A consulta a seguir retornará informações sobre consultas, seus planos e estatísticas médias referentes à duração, tempo de CPU, leituras de io físicas e lógicas.

SELECT Txt.query_text_id, Txt.query_sql_text, Pl.plan_id,
        avg_duration, avg_cpu_time, 
        avg_physical_io_reads, avg_logical_io_reads
FROM sys.query_store_plan AS Pl  
JOIN sys.query_store_query AS Qry  
    ON Pl.query_id = Qry.query_id  
JOIN sys.query_store_query_text AS Txt  
    ON Qry.query_text_id = Txt.query_text_id
JOIN sys.query_store_runtime_stats Stats
    ON Pl.plan_id = Stats.plan_id

Remover dados do repositório de consultas #

Se você deseja remover alguma consulta ou plano de consulta do repositório de consultas, você pode usar os seguintes comandos:

EXEC sp_query_store_remove_query 4;
EXEC sp_query_store_remove_plan 3;

Os parâmetros para esses procedimentos armazenados são a ID de consulta/plano recuperada das exibições do sistema.

Você também pode remover as estatísticas de execução de um plano específico sem remover o plano da loja:

EXEC sp_query_store_reset_exec_stats 3;

Parâmetro fornecido para este id de plano de procedimento.

Forçando plano para consulta #

O otimizador de consultas SQL escolherá o plano possível de baes que ele possa encontrar para alguma consulta. Se você encontrar algum plano que funcione de maneira ideal para alguma consulta, poderá forçar o QO a sempre usar esse plano usando o seguinte procedimento armazenado:

EXEC sp_query_store_unforce_plan @query_id, @plan_id

A partir deste ponto, o QO sempre usará o plano fornecido para a consulta.

Se você deseja remover essa associação, pode usar o seguinte procedimento armazenado:

EXEC sp_query_store_force_plan @query_id, @plan_id

A partir deste ponto, o QO tentará novamente encontrar o melhor plano.
