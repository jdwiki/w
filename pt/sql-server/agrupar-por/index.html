Agrupamento Simples #

Tabela de pedidos

ID do cliente ProductId Quantidade Preço
1 2 5 100
1 3 2 200
1 4 1 500
2 1 4 50
3 5 6 700

Ao agrupar por uma coluna específica, somente valores exclusivos dessa coluna são retornados.

SELECT customerId
FROM orders
GROUP BY customerId;

Valor de retorno:

customerId
1
2
3

Funções agregadas como count() se aplicam a cada grupo e não à tabela completa:

SELECT customerId, 
       COUNT(productId) as numberOfProducts,
       sum(price) as totalPrice
FROM orders
GROUP BY customerId;

Valor de retorno:

customerId númeroDeProdutos preço total
1 3 800
2 1 50
3 1 700

GROUP BY várias colunas #

Pode-se querer GROUP BY mais de uma coluna

declare @temp table(age int, name varchar(15))

insert into @temp
select 18, 'matt' union all
select 21, 'matt' union all
select 21, 'matt' union all
select 18, 'luke' union all
select 18, 'luke' union all
select 21, 'luke' union all
select 18, 'luke' union all
select 21, 'luke'

SELECT Age, Name, count(1) count
FROM @temp 
GROUP BY Age, Name

agrupará por idade e nome e produzirá:

Idade Nome contar
18 Lucas 3
21 luco 2
18 mate 1
21 mate 2

GROUP BY com ROLLUP e CUBE #

O operador ROLLUP é útil na geração de relatórios que contêm subtotais e totais.

  • CUBE gera um conjunto de resultados que mostra agregados para todas as combinações de valores nas colunas selecionadas.

  • ROLLUP gera um conjunto de resultados que mostra agregações para uma hierarquia de valores nas colunas selecionadas.

    Item Color Quantity
    Table Blue 124
    Table Red 223
    Chair Blue 101
    Chair Red 210

    SELECT CASE WHEN (GROUPING(Item) = 1) THEN ‘ALL’ ELSE ISNULL(Item, ‘UNKNOWN’) END AS Item, CASE WHEN (GROUPING(Color) = 1) THEN ‘ALL’ ELSE ISNULL(Color, ‘UNKNOWN’) END AS Color, SUM(Quantity) AS QtySum FROM Inventory GROUP BY Item, Color WITH ROLLUP

    Item Color QtySum

    Chair Blue 101.00
    Chair Red 210.00
    Chair ALL 311.00
    Table Blue 124.00
    Table Red 223.00
    Table ALL 347.00
    ALL ALL 658.00

(7 linhas afetadas)

Se a palavra-chave ROLLUP na consulta for alterada para CUBE, o conjunto de resultados CUBE será o mesmo, exceto que essas duas linhas adicionais serão retornadas no final:

ALL                  Blue                 225.00                     
ALL                  Red                  433.00

https://technet.microsoft.com/en-us/library/ms189305(v=sql.90).aspx

Agrupar por com várias tabelas, várias colunas #

Group by é frequentemente usado com a instrução join. Vamos supor que temos duas tabelas. A primeira é a tabela de alunos:

ID Nome Completo Idade
1 Matt Jones 20
2 Frank Blue 21
3 Antônio Anjo 18

A segunda tabela é a tabela de disciplinas que cada aluno pode fazer:

Subject_Id Assunto
1 Matemática
2 P.E.
3 Física

E porque um aluno pode cursar muitas matérias e uma matéria pode ser cursada por muitos alunos (portanto, relação N:N), precisamos ter uma terceira tabela “limitante”. Vamos chamar a tabela Students_subjects:

Subject_Id Aluno_Id
1 1
2 2
2 1
3 2
1 3
1 1

Agora vamos dizer que queremos saber o número de disciplinas que cada aluno está cursando. Aqui a instrução autônoma GROUP BY não é suficiente, pois a informação não está disponível através de uma única tabela. Portanto, precisamos usar GROUP BY com a instrução JOIN:

Select Students.FullName, COUNT(Subject Id) as SubjectNumber FROM Students_Subjects
LEFT JOIN Students
ON Students_Subjects.Student_id = Students.Id
GROUP BY Students.FullName

O resultado da consulta fornecida é o seguinte:

FullName AssuntoNúmero
Matt Jones 3
Frank Blue 2
Antônio Anjo 1

Para um exemplo ainda mais complexo de uso GROUP BY, digamos que o aluno possa atribuir o mesmo assunto ao seu nome mais de uma vez (como mostrado na tabela Estudantes_Assuntos). Nesse cenário, podemos contar o número de vezes que cada disciplina foi atribuída a um aluno por AGRUPAMENTO por mais de uma coluna:

SELECT Students.FullName, Subjects.Subject,
COUNT(Students_subjects.Subject_id) AS NumberOfOrders
FROM ((Students_Subjects
INNER JOIN Students
ON Students_Subjcets.Student_id=Students.Id)
INNER JOIN Subjects
ON Students_Subjects.Subject_id=Subjects.Subject_id)
GROUP BY Fullname,Subject

Essa consulta fornece o seguinte resultado:

NomeCompleto Assunto AssuntoNúmero
Matt Jones Matemática 2
Matt Jones P.E 1
Frank Blue P.E 1
Frank Blue Física 1
Antônio Anjo Matemática 1

TENDO #

Como a cláusula WHERE é avaliada antes de GROUP BY, você não pode usar WHERE para reduzir os resultados do agrupamento (normalmente uma função de agregação, como COUNT(*)). Para atender a essa necessidade, a cláusula HAVING pode ser usada.

Por exemplo, usando os seguintes dados:

DECLARE @orders TABLE(OrderID INT, Name NVARCHAR(100))

INSERT INTO @orders VALUES
( 1, 'Matt' ),
( 2, 'John' ),
( 3, 'Matt' ),
( 4, 'Luke' ),
( 5, 'John' ),
( 6, 'Luke' ),
( 7, 'John' ),
( 8, 'John' ),
( 9, 'Luke' ),
( 10, 'John' ),
( 11, 'Luke' )

Se quisermos obter o número de pedidos que cada pessoa fez, usaríamos

SELECT Name, COUNT(*) AS 'Orders'
FROM @orders
GROUP BY Name

e pegue

Nome Encomendas
Matt 2
João 5
Lucas 4

No entanto, se quisermos limitar isso a indivíduos que fizeram mais de dois pedidos, podemos adicionar uma cláusula HAVING.

SELECT Name, COUNT(*) AS 'Orders'
FROM @orders
GROUP BY Name
HAVING COUNT(*) > 2

vai render

Nome Encomendas
João 5
Lucas 4

Observe que, assim como GROUP BY, as colunas colocadas em HAVING devem corresponder exatamente às suas contrapartes na instrução SELECT. Se no exemplo acima tivéssemos dito

SELECT Name, COUNT(DISTINCT OrderID)

nossa cláusula HAVING teria que dizer

HAVING COUNT(DISTINCT OrderID) > 2
