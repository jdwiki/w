Comandos de manutenção DBCC #

Os comandos DBCC permitem ao usuário manter espaço no banco de dados, limpar caches, reduzir bancos de dados e tabelas.

Exemplos são:

DBCC DROPCLEANBUFFERS

Remove todos os buffers limpos do conjunto de buffers e objetos columnstore do conjunto de objetos columnstore.

DBCC FREEPROCCACHE
-- or
DBCC FREEPROCCACHE (0x060006001ECA270EC0215D05000000000000000000000000);

Remove todas as consultas SQL no cache do plano. Cada novo plano será recompilado: Você pode especificar o identificador de plano, identificador de consulta para limpar planos para o plano de consulta específico ou instrução SQL.

DBCC FREESYSTEMCACHE ('ALL', myresourcepool); 
-- or
DBCC FREESYSTEMCACHE;

Limpa todas as entradas em cache criadas pelo sistema. Ele pode limpar as entradas o=em todos ou alguns pools de recursos especificados (myresourcepool no exemplo acima)

DBCC FLUSHAUTHCACHE

Esvazia o cache de autenticação do banco de dados que contém informações sobre logins e regras de firewall.

DBCC SHRINKDATABASE (MyDB [, 10]);

Reduz o banco de dados MyDB para 10%. O segundo parâmetro é opcional. Você pode usar o ID do banco de dados em vez do nome.

DBCC SHRINKFILE (DataFile1, 7);

Reduz o arquivo de dados chamado DataFile1 no banco de dados atual. O tamanho do destino é 7 MB (o parâmetro tis é opcional).

DBCC CLEANTABLE (AdventureWorks2012,'Production.Document', 0)

Recupera um espaço da tabela especificada

Declarações de validação DBCC #

Os comandos DBCC permitem que o usuário valide o estado do banco de dados.

ALTER TABLE Table1 WITH NOCHECK ADD CONSTRAINT chkTab1 CHECK (Col1 > 100);  
GO  
DBCC CHECKCONSTRAINTS(Table1);  
--OR
DBCC CHECKCONSTRAINTS ('Table1.chkTable1');

A restrição de verificação é adicionada com opções nocheck, portanto, não será verificada nos dados existentes. O DBCC acionará a verificação de restrição.

Os seguintes comandos DBCC verificam a integridade do banco de dados, tabela ou catálogo:

DBCC CHECKTABLE tablename1 | tableid
DBCC CHECKDB databasename1 | dbid
DBCC CHECKFILEGROUP filegroup_name | filegroup_id | 0
DBCC CHECKCATALOG databasename1 | database_id1 | 0

Declarações informativas DBCC #

Os comandos DBCC podem mostrar informações sobre objetos de banco de dados.

DBCC PROCCACHE

Exibe informações em formato de tabela sobre o cache do procedimento.

DBCC OUTPUTBUFFER ( session_id [ , request_id ])

Retorna o buffer de saída atual em formato hexadecimal e ASCII para o session_id especificado (e request_id opcional).

DBCC INPUTBUFFER ( session_id [ , request_id ])

Exibe a última instrução enviada de um cliente para uma instância do Microsoft SQL Server.

DBCC SHOW_STATISTICS ( table_or_indexed_view_name , column_statistic_or_index_name)

Comandos de rastreamento DBCC #

Os sinalizadores de rastreamento no SQL Server são usados ​​para modificar o comportamento do SQL Server, ativar/desativar alguns recursos. Os comandos DBCC podem controlar sinalizadores de rastreamento:

O exemplo a seguir ativa o sinalizador de rastreamento 3205 globalmente e 3206 para a sessão atual:

DBCC TRACEON (3205, -1); 
DBCC TRACEON (3206);

O exemplo a seguir desativa o sinalizador de rastreamento 3205 globalmente e 3206 para a sessão atual:

DBCC TRACEON (3205, -1); 
DBCC TRACEON (3206);

O exemplo a seguir exibe o status dos sinalizadores de rastreamento 2528 e 3205:

DBCC TRACESTATUS (2528, 3205);

Declaração DBCC #

As instruções DBCC atuam como comandos do console de banco de dados para SQL Server. Para obter as informações de sintaxe para o comando DBCC especificado, use a instrução DBCC HELP (…).

O exemplo a seguir retorna todas as instruções DBCC para as quais a Ajuda está disponível:

DBCC HELP ('?');

O exemplo a seguir retorna opções para a instrução DBCC CHECKDB:

DBCC HELP ('CHECKDB');
