Formatação de data e hora usando FORMAT #

Você pode utilizar a nova função: FORMAT().

Usando isso, você pode transformar seus campos DATETIME em seu próprio formato VARCHAR personalizado.

Exemplo

DECLARE @Date DATETIME = '2016-09-05 00:01:02.333'

SELECT FORMAT(@Date, N'dddd, MMMM dd, yyyy hh:mm:ss tt')

Segunda-feira, 05 de setembro de 2016 12:01:02 AM

DATEADD para adicionar e subtrair períodos de tempo #

Sintaxe geral:

DATEADD (datepart , number , datetime_expr)

Para adicionar uma medida de tempo, o number deve ser positivo. Para subtrair uma medida de tempo, o number deve ser negativo.

Exemplos

DECLARE @now DATETIME2 = GETDATE();
SELECT @now;                        --2016-07-21 14:39:46.4170000
SELECT DATEADD(YEAR, 1, @now)       --2017-07-21 14:39:46.4170000
SELECT DATEADD(QUARTER, 1, @now)    --2016-10-21 14:39:46.4170000
SELECT DATEADD(WEEK, 1, @now)       --2016-07-28 14:39:46.4170000
SELECT DATEADD(DAY, 1, @now)        --2016-07-22 14:39:46.4170000
SELECT DATEADD(HOUR, 1, @now)       --2016-07-21 15:39:46.4170000
SELECT DATEADD(MINUTE, 1, @now)     --2016-07-21 14:40:46.4170000
SELECT DATEADD(SECOND, 1, @now)     --2016-07-21 14:39:47.4170000
SELECT DATEADD(MILLISECOND, 1, @now)--2016-07-21 14:39:46.4180000

NOTA: DATEADD também aceita abreviações no parâmetro datepart. O uso dessas abreviações geralmente é desencorajado, pois podem ser confusos (m vs mi, ww vs w, etc.).

Obtém o DateTime atual #

As funções internas GETDATE e GETUTCDATE retornam a data e a hora atuais sem um deslocamento de fuso horário.

O valor de retorno de ambas as funções é baseado no sistema operacional do computador no qual a instância do SQL Server está sendo executada.

O valor de retorno de GETDATE representa a hora atual no mesmo fuso horário do sistema operacional. O valor de retorno de GETUTCDATE representa a hora UTC atual.

Qualquer função pode ser incluída na cláusula SELECT de uma consulta ou como parte da expressão booleana na cláusula WHERE.

Exemplos:

-- example query that selects the current time in both the server time zone and UTC
SELECT GETDATE() as SystemDateTime, GETUTCDATE() as UTCDateTime

-- example query records with EventDate in the past.
SELECT * FROM MyEvents WHERE EventDate < GETDATE()

Existem algumas outras funções internas que retornam diferentes variações da data e hora atual:

SELECT 
    GETDATE(),          --2016-07-21 14:27:37.447
    GETUTCDATE(),       --2016-07-21 18:27:37.447
    CURRENT_TIMESTAMP,  --2016-07-21 14:27:37.447
    SYSDATETIME(),      --2016-07-21 14:27:37.4485768
    SYSDATETIMEOFFSET(),--2016-07-21 14:27:37.4485768 -04:00
    SYSUTCDATETIME()    --2016-07-21 18:27:37.4485768

Obtendo o último dia de um mês #

Usando as funções DATEADD e DATEDIFF, é possível retornar a última data de um mês.

SELECT DATEADD(d, -1, DATEADD(m, DATEDIFF(m, 0, '2016-09-23') + 1, 0))
-- 2016-09-30 00:00:00.000

A função EOMONTH fornece uma maneira mais concisa de retornar a última data de um mês e possui um parâmetro opcional para compensar o mês.

SELECT EOMONTH('2016-07-21')        --2016-07-31
SELECT EOMONTH('2016-07-21', 4)     --2016-11-30
SELECT EOMONTH('2016-07-21', -5)    --2016-02-29

Criar função para calcular a idade de uma pessoa em uma data específica #

Esta função terá 2 parâmetros de data e hora, o DOB ​​e uma data para verificar a idade em

  CREATE FUNCTION [dbo].[Calc_Age]
    (
    @DOB datetime , @calcDate datetime 
    )
    RETURNS int
    AS
   BEGIN
declare @age int

IF (@calcDate < @DOB  )
RETURN -1

-- If a DOB is supplied after the comparison date, then return -1
SELECT @age = YEAR(@calcDate) - YEAR(@DOB) +
  CASE WHEN DATEADD(year,YEAR(@calcDate) - YEAR(@DOB)
  ,@DOB) > @calcDate THEN -1 ELSE 0 END
    

RETURN @age
    

END

por exemplo, para verificar a idade hoje de alguém nascido em 01/01/2000

SELECT  dbo.Calc_Age('2000-01-01',Getdate())

DATEDIFF para calcular diferenças de período de tempo #

Sintaxe geral:

DATEDIFF (datepart, datetime_expr1, datetime_expr2)

Ele retornará um número positivo se datetime_expr estiver no passado em relação a datetime_expr2, e um número negativo caso contrário.

Exemplos

DECLARE @now DATETIME2 = GETDATE();
DECLARE @oneYearAgo DATETIME2 = DATEADD(YEAR, -1, @now);
SELECT @now                                    --2016-07-21 14:49:50.9800000
SELECT @oneYearAgo                             --2015-07-21 14:49:50.9800000
SELECT DATEDIFF(YEAR, @oneYearAgo, @now)       --1
SELECT DATEDIFF(QUARTER, @oneYearAgo, @now)    --4
SELECT DATEDIFF(WEEK, @oneYearAgo, @now)       --52
SELECT DATEDIFF(DAY, @oneYearAgo, @now)        --366
SELECT DATEDIFF(HOUR, @oneYearAgo, @now)       --8784
SELECT DATEDIFF(MINUTE, @oneYearAgo, @now)     --527040
SELECT DATEDIFF(SECOND, @oneYearAgo, @now)     --31622400

NOTA: DATEDIFF também aceita abreviações no parâmetro datepart. O uso dessas abreviações geralmente é desencorajado, pois podem ser confusos (m vs mi, ww vs w, etc.).

DATEDIFF também pode ser usado para determinar o deslocamento entre UTC e a hora local do SQL Server. A instrução a seguir pode ser usada para calcular o deslocamento entre o UTC e a hora local (incluindo o fuso horário).

select  DATEDIFF(hh, getutcdate(), getdate()) as 'CentralTimeOffset'

DATEPART & DATENAME #

DATEPART retorna o datepart especificado da expressão de data e hora especificada como um valor numérico.

DATENAME retorna uma string de caracteres que representa a datepart especificada da data especificada. Na prática, DATENAME é útil principalmente para obter o nome do mês ou o dia da semana.

Existem também algumas funções abreviadas para obter o ano, mês ou dia de uma expressão datetime, que se comportam como DATEPART com suas respectivas unidades datepart.

Sintaxe:

DATEPART ( datepart , datetime_expr )
DATENAME ( datepart , datetime_expr )
DAY ( datetime_expr )
MONTH ( datetime_expr )
YEAR ( datetime_expr )

Exemplos:

DECLARE @now DATETIME2 = GETDATE();
SELECT @now                       --2016-07-21 15:05:33.8370000
SELECT DATEPART(YEAR, @now)       --2016
SELECT DATEPART(QUARTER, @now)    --3
SELECT DATEPART(WEEK, @now)       --30
SELECT DATEPART(HOUR, @now)       --15
SELECT DATEPART(MINUTE, @now)     --5
SELECT DATEPART(SECOND, @now)     --33
-- Differences between DATEPART and DATENAME:
SELECT DATEPART(MONTH, @now)      --7
SELECT DATENAME(MONTH, @now)      --July
SELECT DATEPART(WEEKDAY, @now)    --5
SELECT DATENAME(WEEKDAY, @now)    --Thursday
--shorthand functions
SELECT DAY(@now)    --21
SELECT MONTH(@now)  --7
SELECT YEAR(@now)   --2016

NOTA: DATEPART e DATENAME também aceitam abreviações no parâmetro datepart. O uso dessas abreviações geralmente é desencorajado, pois podem ser confusos (m vs mi, ww vs w, etc.).

Retorna apenas a Data de um DateTime #

Há muitas maneiras de retornar uma data de um objeto DateTime

  1. SELECT CONVERT(Data, GETDATE())
  2. SELECT DATEADD(dd, 0, DATEDIFF(dd, 0, GETDATE())) retorna 21/07/2016 00:00:00.000
  3. SELECT CAST(GETDATE() AS DATE)
  4. SELECT CONVERT(CHAR(10),GETDATE(),111)
  5. SELECT FORMAT(GETDATE(), 'aaaa-MM-dd')

Observe que as opções 4 e 5 retornam uma string, não uma data.

OBJETO DE DATA DE PLATAFORMA CRUZADA #

No Transact SQL, você pode definir um objeto como Date (ou DateTime) usando a função [DATEFROMPARTS][1] (ou [DATETIMEFROMPARTS][1]) como a seguir:

 DECLARE @myDate DATE=DATEFROMPARTS(1988,11,28) 
 DECLARE @someMoment DATETIME=DATEFROMPARTS(1988,11,28,10,30,50,123)

Os parâmetros que você fornece são Ano, Mês, Dia para a função DATEFROMPARTS e, para a função DATETIMEFROMPARTS você precisará fornecer ano, mês, dia, hora, minutos, segundos e milissegundos.

Esses métodos são úteis e valem a pena serem usados ​​porque usar a string simples para construir uma data (ou data/hora) pode falhar dependendo da região da máquina host, local ou configurações de formato de data.
