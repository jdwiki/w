No SQL Server, um procedimento é um programa armazenado para o qual você pode passar parâmetros. Ele não retorna um valor como uma função faz. No entanto, ele pode retornar um status de sucesso/falha para o procedimento que o chamou.

Sintaxe #

  • CRIAR { PROCEDIMENTO | PROC } [schema_name.]procedure_name
  • [ @parameter [type_schema_name.] tipo de dados
  • [ VARYING ] [ = default ] [ OUT | SAÍDA | SOMENTE LEITURA ]
  • , @parameter [type_schema_name.] tipo de dados
  • [ VARYING ] [ = default ] [ OUT | SAÍDA | SOMENTE LEITURA ] ]
  • [ COM { CRIPTOGRAFIA | RECOMPILAR | Cláusula EXECUTE AS } ]
  • [ PARA REPLICAÇÃO ]
  • COMO
  • COMEÇAR
  • [declaração_seção]
  • executável_seção
  • FIM;

Criando e executando um procedimento armazenado básico #

Usando a tabela Authors no Library Database

CREATE PROCEDURE GetName
(
    @input_id INT = NULL,      --Input parameter,  id of the person, NULL default
    @name VARCHAR(128) = NULL  --Input parameter, name of the person, NULL default
) 
AS
BEGIN 
    SELECT Name + ' is from ' + Country 
    FROM Authors 
    WHERE Id = @input_id OR Name = @name
END 
GO

Você pode executar um procedimento com algumas sintaxes diferentes. Primeiro, você pode usar EXECUTE ou EXEC

EXECUTE GetName @id = 1
EXEC Getname @name = 'Ernest Hemingway'

Além disso, você pode omitir o comando EXEC. Além disso, você não precisa especificar qual parâmetro está passando, pois passa todos os parâmetros.

GetName NULL, 'Ernest Hemingway'

Quando você deseja especificar os parâmetros de entrada em uma ordem diferente de como eles são declarados no procedimento, você pode especificar o nome do parâmetro e atribuir valores. Por exemplo

 CREATE PROCEDURE dbo.sProcTemp 
 (
    @Param1 INT,
    @Param2 INT
)
AS
BEGIN

    SELECT
        Param1 = @Param1,
        Param2 = @Param2

END

a ordem normal para executar este procedimento é especificar o valor para @Param1 primeiro e depois @Param2 depois. Então vai ficar mais ou menos assim

  EXEC dbo.sProcTemp @Param1 = 0,@Param2=1

Mas também é possível que você possa usar o seguinte

  EXEC dbo.sProcTemp @Param2 = 0,@Param1=1

neste, você está especificando o valor para @param2 primeiro e @Param1 segundo. O que significa que você não precisa manter o mesmo pedido declarado no procedimento, mas pode fazer qualquer pedido que desejar. mas você precisará especificar para qual parâmetro você está definindo o valor

Acesse o procedimento armazenado de qualquer banco de dados

E também você pode criar um procedimento com um prefixo sp_ esses procedimentos, como todos os procedimentos armazenados do sistema, podem ser executados sem especificar o banco de dados devido ao comportamento padrão do SQL Server. Quando você executa um procedimento armazenado que começa com “sp_”, o SQL Server procura primeiro o procedimento no banco de dados mestre. Se o procedimento não for encontrado no mestre, ele procura no banco de dados ativo. Se você tiver um procedimento armazenado que deseja acessar de todos os bancos de dados, crie-o no master e use um nome que inclua o prefixo “sp_”.

Use Master

CREATE PROCEDURE sp_GetName
(
    @input_id INT = NULL,      --Input parameter,  id of the person, NULL default
    @name VARCHAR(128) = NULL  --Input parameter, name of the person, NULL default
) 
AS
BEGIN 
    SELECT Name + ' is from ' + Country 
    FROM Authors 
    WHERE Id = @input_id OR Name = @name
END 
GO

Stored Procedure com If…Else e Insert Into operation #

Crie uma tabela de exemplo Employee:

CREATE TABLE Employee
(
    Id INT,
    EmpName VARCHAR(25),
    EmpGender VARCHAR(6),
    EmpDeptId INT
)

Cria procedimento armazenado que verifica se os valores passados ​​no procedimento armazenado não são nulos ou não vazios e executa a operação de inserção na tabela Employee.

CREATE PROCEDURE spSetEmployeeDetails
(
    @ID int,
    @Name VARCHAR(25),
    @Gender VARCHAR(6),
    @DeptId INT
)
AS
BEGIN
    IF (
        (@ID IS NOT NULL AND LEN(@ID) !=0) 
        AND (@Name IS NOT NULL AND LEN(@Name) !=0)
        AND (@Gender IS NOT NULL AND LEN(@Gender) !=0)
        AND (@DeptId IS NOT NULL AND LEN(@DeptId) !=0)
    )
    BEGIN
        INSERT INTO Employee
        (
            Id,
            EmpName,
            EmpGender,
            EmpDeptId
        )
        VALUES
        (
            @ID,
            @Name,
            @Gender,
            @DeptId
        )
    END
ELSE
    PRINT 'Incorrect Parameters'
END
GO

Execute o procedimento armazenado

DECLARE @ID INT,
    @Name VARCHAR(25),
    @Gender VARCHAR(6),
    @DeptId INT

EXECUTE spSetEmployeeDetails
    @ID = 1,
    @Name = 'Subin Nepal',
    @Gender = 'Male',
    @DeptId = 182666

SQL dinâmico em procedimento armazenado #

O SQL dinâmico nos permite gerar e executar instruções SQL em tempo de execução. O SQL dinâmico é necessário quando nossas instruções SQL contêm identificadores que podem mudar em diferentes tempos de compilação.

Exemplo simples de SQL dinâmico:

CREATE PROC sp_dynamicSQL
@table_name      NVARCHAR(20),
@col_name        NVARCHAR(20), 
@col_value       NVARCHAR(20) 
AS
BEGIN
DECLARE  @Query  NVARCHAR(max)
SET      @Query = 'SELECT * FROM ' + @table_name
SET      @Query = @Query + ' WHERE ' + @col_name + ' = ' + ''''+@col_value+''''
EXEC     (@Query)
END

Na consulta sql acima, podemos ver que podemos usar a consulta acima definindo valores em @table_name, @col_name e @col_value em tempo de execução. A consulta é gerada em tempo de execução e executada. Esta é uma técnica na qual podemos criar scripts inteiros como string em uma variável e executá-la. Podemos criar consultas mais complexas usando SQL dinâmico e conceito de concatenação. Este conceito é muito poderoso quando você deseja criar um script que pode ser usado sob diversas condições.

Executando procedimento armazenado

DECLARE @table_name     NVARCHAR(20) = 'ITCompanyInNepal',
        @col_name       NVARCHAR(20) = 'Headquarter',
        @col_value      NVARCHAR(20) = 'USA'
    
EXEC    sp_dynamicSQL   @table_name,    
                        @col_name,
                        @col_value

STORED PROCEDURE com parâmetros OUT #

Os procedimentos armazenados podem retornar valores usando a palavra-chave OUTPUT em sua lista de parâmetros.

Criando um procedimento armazenado com um único parâmetro de saída #

CREATE PROCEDURE SprocWithOutParams
(
    @InParam VARCHAR(30),
    @OutParam VARCHAR(30) OUTPUT
)
AS
BEGIN
    SELECT @OutParam = @InParam + ' must come out'   
    RETURN
END   
GO

Executando o procedimento armazenado #

DECLARE @OutParam VARCHAR(30)    
EXECUTE SprocWithOutParams 'what goes in', @OutParam OUTPUT   
PRINT @OutParam

Criando um procedimento armazenado com vários parâmetros de saída #

CREATE PROCEDURE SprocWithOutParams2
(
    @InParam VARCHAR(30),
    @OutParam VARCHAR(30) OUTPUT,
    @OutParam2 VARCHAR(30) OUTPUT
)
AS
BEGIN
    SELECT @OutParam = @InParam +' must come out'   
    SELECT @OutParam2 = @InParam +' must come out'
    RETURN
END    
GO

Executando o procedimento armazenado #

DECLARE @OutParam VARCHAR(30)    
DECLARE @OutParam2 VARCHAR(30)  
EXECUTE SprocWithOutParams2 'what goes in', @OutParam OUTPUT, @OutParam2 OUTPUT   
PRINT @OutParam 
PRINT @OutParam2

Loop Simples #

Primeiro vamos colocar alguns dados em uma tabela temporária chamada #systables e adicionar um número de linha incrementado para que possamos consultar um registro por vez

select
    o.name,
    row_number() over (order by o.name) as rn
into
    #systables
from
    sys.objects as o
where
    o.type = 'S'

Em seguida, declaramos algumas variáveis ​​para controlar o loop e armazenar o nome da tabela neste exemplo

declare
    @rn int = 1,
    @maxRn int = (
                    select
                        max(rn)
                    from
                        #systables as s
                    )
declare    @tablename sys name

Agora podemos fazer um loop usando um while simples. Incrementamos @rn na instrução select, mas isso também pode ter sido uma instrução separada para ex set @rn = @rn + 1, isso dependerá de seus requisitos. Também usamos o valor de @rn antes de ser incrementado para selecionar um único registro de #systables. Por último, imprimimos o nome da tabela.

while @rn <= @maxRn
    begin

        select
            @tablename = name,
            @rn = @rn + 1
        from
            #systables as s
        where
            s.rn = @rn

        print @tablename
    end

Loop Simples #

CREATE PROCEDURE SprocWithSimpleLoop
(
    @SayThis VARCHAR(30),
    @ThisManyTimes INT
)
AS
BEGIN
    WHILE @ThisManyTimes > 0
    BEGIN
        PRINT @SayThis;
        SET @ThisManyTimes = @ThisManyTimes - 1;
    END
    
    RETURN;
END    
GO
