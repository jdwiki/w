Quando estiver pronto para distribuição, seu aplicativo electron pode ser empacotado em um arquivo executável.

Os aplicativos Electron podem ser empacotados para serem executados no Windows (32/64 bits), OSX (macOS) e Linux (x86/x86_64).

Para empacotar seu código, use o pacote npm ’electron-packager\

https://github.com/electron-userland/electron-packager

Sintaxe #

  • $ pacote de elétrons
  • sourcedir
  • nome do aplicativo
  • –platform=plataforma
  • –arch=arch
  • [sinalizadores opcionais…]

Parâmetros #

Parâmetro Detalhes
sourcedir O diretório de seus arquivos de aplicativos eletrônicos
nome do aplicativo O nome do seu aplicativo
plataforma A plataforma para a qual você deseja compilar seu código. Omitir isso irá compilar para o sistema operacional host
arco A arquitetura do sistema para a qual você deseja compilar seu código. Omitir isso irá compilar para o arco do host

Instalando o empacotador de elétrons #

# for use in npm scripts
npm install electron-packager --save-dev

# for use from cli
npm install electron-packager -g

Empacotamento do script #

var packager = require('electron-packager');

packager({
    dir: '/',
}, function(err, path){
    if(err) throw err;
    // Application has been packaged
});

Fazendo scripts npm para automatizar o empacotamento do Electron #

Uma maneira conveniente de empacotar seu aplicativo é escrever os scripts em seu arquivo packages.json e executá-los com o comando npm run

{
    "name": "AppName",
    "productName": "AppName",
    "version": "0.1.1",
    "main": "main.js",
    "devDependencies": {
        "electron": "^1.6.6",
        "electron-packager": "^8.7.0"
    },
    "scripts": {
        "package-mac": "electron-packager . --overwrite --platform=darwin --arch=x64 --icon=images/icon.png --prune=true --out=release-builds",
        "package-win": "electron-packager . --overwrite --platform=win32 --arch=ia32 --icon=images/icon.png --prune=true --out=release-builds",
        "package-linux" : "electron-packager . --overwrite --platform=linux --arch=x64 --icon=images/icon.png --prune=true --out=release-builds"
    }
}

E para executá-los, basta escrever:

npm run package-mac
npm run package-win
npm run package-linux

Um detalhamento dos sinalizadores de comando é:

electron-packager .     // this runs the packager in the current folder
--overwrite             // overwrite any previous build
--platform=darwin       // platform for which the binaries should be created
--arch=x64              // the OS architecture
--icon=images/icon.png  // the icon for the app executable
--prune=true            // this does not copy your dev-dependencies that appear in your packages.json
--out=release-builds    // the name of the folder were the binaries will be outputed

Antes, a execução dos scripts alterava as devDependencies para dependências, pois o electron-packager não pode agrupar os pacotes nas devDependencies no aplicativo. Em packager.json, altere a palavra (se estiver lá ou se os pacotes estiverem instalados usando –save-dev no npm install) devDependencies para apenas dependências.

Empacotamento da CLI #

electron-packager C:/my-app MyApp
← instalador de elétrons
