As fontes SVG não são mais suportadas diretamente pelos navegadores. Ainda assim, eles são muito convenientes para gerar fontes programaticamente, como fontes de símbolos ou fontes de código de barras.

Existem muitas ferramentas por aí que permitem converter fontes svg para qualquer outro formato de fonte.

Aqui está uma lista de ferramentas que você pode usar com fontes SVG.

Conversores #

  • https://github.com/fontello/svg2ttf

uma fonte simples #

Um exemplo simples de uma fonte svg. Algumas coisas a serem observadas aqui:

  • o sistema de coordenadas dos glifos está em oposição ao sistema de coordenadas usual em svg. o eixo y aponta para cima. O ponto 0,0 está no canto inferior direito.

  • Todos os caminhos em uma fonte devem ser desenhados no sentido anti-horário.

  • Na maioria das ferramentas, apenas o atributo d do elemento glifo é suportado. Elementos filho não funcionarão, embora sejam tecnicamente permitidos.

Se você tiver glifos mais largos ou mais estreitos, basta alterar o horiz-adv-x no próprio elemento de glifo.

<glyph unicode="a" horiz-adv-x="512" d="M0 0 H1000 L500 1000z M200 200 L500 800 L800 200z" />

escolha de fonte #

a propriedade unicode é usada para seleção de glifo posterior. Você pode usar letras simples ou pontos de código unicode, bem como ligaduras (combinação de letras ou pontos de código unicode)

  • unicode="abc"
  • unicode="&#97;&#98;"
  • unicode="ab&#97;&#98;"
  • unicode="a"
  • unicode="&#98;"

os glifos são sempre selecionados pela primeira correspondência, portanto, coloque todas as ligaduras antes de qualquer caractere único.

Os pontos de código unicode podem ser escritos em notação decimal &#123; ou hexadecimal &#x1f.

subida, descida e linha de base #

a propriedade units-per-em é uma das propriedades de fonte mais importantes. É usado para dar significado a qualquer valor de qualquer outra propriedade.

a especificação da fonte CSS2 tem uma boa definição do em sqare:

Certos valores, como métricas de largura, são expressos em unidades relativas a um quadrado abstrato cuja altura é a distância pretendida entre linhas de texto no mesmo tamanho de texto

a linha de base padrão é 0 no quadrado em. para cálculo de altura de linha e propósitos de alinhamento, as duas propriedades de subida e descida são da maior importância.

A subida é a distância máxima da linha de base até o ponto mais alto do seu maior glifo. Normalmente isso é 1em, então o valor que você deu para unidades por em.

A descida é a distância máxima da linha de base até o ponto mais baixo em qualquer glifo de sua fonte.

Aqui está uma fonte com glifos renderizando uma linha no ponto mais baixo e mais alto, bem como na linha de base.

<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 1000 1000">
  <font id = "myFont"
        horiz-adv-x   = "1000"
        vert-origin-x = "0"
        vert-origin-y = "0" >
    <font-face font-family  = "myFont"
               font-weight  = "normal"
               units-per-em = "1000"
               descent="500"
               ascent="1000">
      <font-face-src>
        <font-face-name name="myFont"/>
      </font-face-src>
    </font-face>`
    <glyph unicode = "a" d = "M0 900h1000v100h-1000z" />
    <glyph unicode = "b" d = "M0 0h1000v100h-1000z" />
    <glyph unicode = "c" d = "M0 -500h1000v100h-1000z" />
  </font>
</svg>

A subida e a descida são usadas para determinar a altura da linha. Unidades por em e linha de base são usadas para determinar a posição e o tamanho em relação a outras fontes usadas.
