Loops considerados como uma das estruturas de controle importantes em qualquer linguagem de programação. Existem diferentes maneiras pelas quais podemos alcançar a cobertura de loop.

Esses métodos diferem com base no tipo de loop.

Teste de loop único #

Esses são loops em que seu corpo de loop não contém outros loops (o loop mais interno no caso de aninhado).

Para ter cobertura de loop, os testadores devem realizar os testes abaixo.

Teste 1: Projete um teste no qual o corpo do loop não deve ser executado (ou seja, zero iterações)

Teste 2: Projete um teste no qual a variável de controle de loop seja negativa (número negativo de iterações)

Teste 3: Projete um teste no qual o loop itera apenas uma vez

Teste 4: Projete um teste no qual o loop itera duas vezes

Teste 5: Projete um teste no qual o loop itera certo número de vezes, digamos m onde m < número máximo de iterações possíveis

Teste 6: Projete um teste no qual o loop itera um a menos que o número máximo de iterações

Teste 7: Projete um teste no qual o loop itera o número máximo de iterações

Teste 8: Projete um teste no qual o loop itera uma a mais do que o número máximo de iterações

Considere o exemplo de código abaixo que aplica todas as condições especificadas.

Teste de loops aninhados #

Um loop aninhado é um loop dentro de um loop.

O loop externo muda somente após o loop interno ser completamente finalizado/interrompido.

Nesse caso, os casos de teste devem ser projetados de tal forma que

Comece no loop mais interno. Defina todos os loops externos para seus valores mínimos. Execute o teste de loop simples no loop mais interno (Test3 / Test4 / Test5 / Test6 / Test7). Continue até que todos os loops testados

Teste de loops concatenados #

Dois loops são concatenados se for possível alcançar um depois de sair do outro no mesmo caminho da entrada à saída. Às vezes, esses dois loops são independentes um do outro. Nesses casos, podemos aplicar as técnicas de projeto especificadas como parte do teste de loop único.

Mas se os valores de iteração em um loop estão direta ou indiretamente relacionados aos valores de iteração de outro loop e podem ocorrer no mesmo caminho, podemos considerá-los como loops aninhados.
