Como criar projeto de teste de unidade e testes de unidade e como executar os testes de unidade e ferramenta de cobertura de código.

Neste guia, a estrutura padrão do MSTest será usada e a ferramenta padrão de análise de cobertura de código que está disponível no Visual Studio.

O guia foi escrito para o Visual Studio 2015, então é possível que algumas coisas sejam diferentes em outras versões.

Criando um projeto de teste de unidade #

  • Abra o projeto C#
  • Clique com o botão direito do mouse na solução -> Adicionar -> Novo projeto…
  • (Figura 1)

  • Vá para Instalado -> Visual C# -> Teste
  • Clique em Projeto de teste unitário
  • Dê um nome e clique em OK
  • (Figura 2)

  • O projeto de teste de unidade é adicionado à solução
  • (Figura 3)

Adicionando a referência ao aplicativo que você deseja testar

  • No projeto de teste de unidade, adicione uma referência ao projeto que deseja testar
  • Clique com o botão direito do mouse em Referências -> Adicionar referência…
  • (Figura 3)

  • Selecione o projeto que deseja testar
  • Vá para Projetos -> Solução
  • Marque a caixa de seleção do projeto que deseja testar -> clique em OK
  • (Figura 4)

Dois métodos para criar testes de unidade #

Método 1 #

  • Vá para sua classe de teste de unidade no projeto de teste de unidade

  • Escreva um teste de unidade

    [Testclass] public class UnitTest1 { [TestMethod] public void TestMethod1() { //Arrange ApplicationToTest.Calc ClassCalc = new ApplicationToTest.Calc(); int expectedResult = 5;

          //Act
      int result = ClassCalc.Sum(2,3);

      //Assert
      Assert.AreEqual(expectedResult, result);
  }

    }

Método 2 #

  • Vá para o método que você deseja testar
  • Clique com o botão direito do mouse no método -> Criar Testes Unitários
  • (Figura 4)

  • Definir estrutura de teste para MSTest
  • Defina Test Project para o nome do seu projeto de teste de unidade
  • Defina o arquivo de saída para o nome da classe dos testes de unidade
  • Defina o código para o método de teste para uma das opções listadas que você preferir
  • As outras opções podem ser editadas, mas não é necessário

(Dica: Se você ainda não fez um projeto de teste de unidade, você ainda pode usar esta opção. Basta definir Projeto de teste para e Arquivo de saída para . Isso criará o projeto de teste de unidade e ele adicionará a referência do projeto ao projeto de teste de unidade)

  • (Figura 5)

  • Como você vê abaixo ele cria a base do teste unitário para você preencher
  • (Figura 6)

Executando testes de unidade no Visual Studio #

  • Para ver seus testes de unidade, vá para Test -> Windows -> Test Explorer
  • (Figura 1)

  • Isso abrirá uma visão geral de todos os testes no aplicativo
  • (Figura 2)

  • Na figura acima você pode ver que o exemplo tem um teste unitário e ainda não foi executado
  • Você pode clicar duas vezes em um teste para ir para o código onde o teste de unidade está definido
  • Você pode executar um ou vários testes com o Run All ou Run…
  • Você também pode executar testes e alterar as configurações no menu Teste (Figura 1)

Executando análise de cobertura de código no Visual Studio #

  • Para ver seus testes de unidade, vá para Test -> Windows -> Code Coverage Results
  • (Figura 1)

  • Abrirá a seguinte janela
  • (Figura 2)

  • A janela agora está vazia
  • Vá para o menu Teste -> Analisar Cobertura de Código
  • (Figura 3)

  • Os testes agora também serão executados (Veja os resultados no Test Explorer)
  • Os resultados serão mostrados em uma tabela com você pode ver quais classes e métodos são cobertos com testes de unidade e quais não são
  • (Figura 4)
