Usando arquivos estáticos #

Os aplicativos da Web geralmente exigem arquivos estáticos, como arquivos CSS ou JavaScript. Para usar arquivos estáticos em uma aplicação Flask, crie uma pasta chamada static em seu pacote ou próximo ao seu módulo e ela estará disponível em /static na aplicação.

Um exemplo de estrutura de projeto para usar modelos é o seguinte:

MyApplication/
    /static/
        /style.css
        /script.js
    /templates/
        /index.html
    /app.py

app.py é um exemplo básico de Flask com renderização de modelo.

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def index():
    return render_template('index.html')

Para usar o arquivo CSS e JavaScript estático no modelo index.html, precisamos usar o nome de terminal ’estático’ especial:

{{url_for('static', filename = 'style.css')}}

Portanto, index.html pode conter:

<html>
    <head>
        <title>Static File</title>
        <link href="{{url_for('static', filename = 'style.css')}}" rel="stylesheet">
        <script src="{{url_for('static', filename = 'script.js')}}"></script>
    </head>
    <body>
        <h3>Hello World!</h3>
    </body>
</html>

Depois de executar o app.py, veremos a página da Web em http://localhost:5000/.

Arquivos estáticos em produção (servidos pelo servidor web frontend) #

O servidor web integrado do Flask é capaz de servir ativos estáticos, e isso funciona bem para desenvolvimento. No entanto, para implantações de produção que estão usando algo como uWSGI ou Gunicorn para servir o aplicativo Flask, a tarefa de servir arquivos estáticos normalmente é descarregada para o servidor web front-end (Nginx, Apache, etc.). Essa é uma tarefa pequena/fácil com aplicativos menores, especialmente quando todos os ativos estáticos estão em uma pasta; no entanto, para aplicativos maiores e/ou aqueles que estão usando plug-ins do Flask que fornecem ativos estáticos, pode ser difícil lembrar os locais de todos esses arquivos e copiá-los/coletá-los manualmente em um diretório. Este documento mostra como usar o plug-in Flask-Collect para simplificar essa tarefa.

Observe que o foco desta documentação está na coleção de ativos estáticos. Para ilustrar essa funcionalidade, este exemplo usa o plug-in Flask-Bootstrap, que fornece ativos estáticos. Ele também usa o plug-in Flask-Script, que é usado para simplificar o processo de criação de tarefas de linha de comando. Nenhum desses plugins é crítico para este documento, eles são usados ​​aqui apenas para demonstrar a funcionalidade. Se você optar por não usar o Flask-Script, convém revisar os documentos do Flask-Collect para formas alternativas de chamar o comando collect.

Observe também que a configuração do seu servidor web front-end para atender a esses ativos estáticos está fora do escopo deste documento. Confira alguns exemplos usando [Nginx](https://www.digitalocean.com/community/tutorials/ how-to-deploy-flask-web-applications-using-uwsgi-behind-nginx-on-centos-6-4) e [Apache](http://stackoverflow.com/questions/24739925/serving-static-files -through-apache) para mais informações. Basta dizer que você fará alias de URLs que começam com “/static” para o diretório centralizado que o Flask-Collect criará para você neste exemplo.

O aplicativo está estruturado da seguinte forma:

/manage.py - The app management script, used to run the app, and to collect static assets
/app/ - this folder contains the files that are specific to our app
    | - __init__.py - Contains the create_app function
    | - static/ - this folder contains the static files for our app.
        | css/styles.css - custom styles for our app (we will leave this file empty)
        | js/main.js - custom js for our app (we will leave this file empty)
    | - templates/index.html - a simple page that extends the Flask-Bootstrap template

  1. Primeiro, crie seu ambiente virtual e instale os pacotes necessários: (your-virtualenv) $ pip install flask-script flask-bootstrap flask-collect

  2. Estabeleça a estrutura de arquivos descrita acima:

    $ touch manage.py; mkdir -p app/{static/{css,js},templates}; touch app/{init.py,static/{css/styles.css,js/main.js}}

  3. Estabeleça o conteúdo dos arquivos manage.py, app/__init__.py e app/templates/index.html:

    manage.py #

    #!/usr/bin/env python import os from flask_script import Manager, Server from flask import current_app from flask_collect import Collect from app import create_app

    class Config(object): # CRITICAL CONFIG VALUE: This tells Flask-Collect where to put our static files! # Standard practice is to use a folder named “static” that resides in the top-level of the project directory. # You are not bound to this location, however; you may use basically any directory that you wish. COLLECT_STATIC_ROOT = os.path.dirname(file) + ‘/static’ COLLECT_STORAGE = ‘flask_collect.storage.file’

    app = create_app(Config)

    manager = Manager(app) manager.add_command(‘runserver’, Server(host=‘127.0.0.1’, port=5000))

    collect = Collect() collect.init_app(app)

    @manager.command def collect(): “““Collect static from blueprints. Workaround for issue: https://github.com/klen/Flask-Collect/issues/22"”” return current_app.extensions[‘collect’].collect()

    if name == “main”: manager.run()

# app/__init__.py
from flask import Flask, render_template
from flask_collect import Collect
from flask_bootstrap import Bootstrap

def create_app(config):
  app = Flask(__name__)
  app.config.from_object(config)

  Bootstrap(app)
  Collect(app)

  @app.route('/')
  def home():
    return render_template('index.html')

  return app

# app/templates/index.html
{% extends "bootstrap/base.html" %}
{% block title %}This is an example page{% endblock %}

{% block navbar %}
<div class="navbar navbar-fixed-top">
  <!-- ... -->
</div>
{% endblock %}

{% block content %}
  <h1>Hello, Bootstrap</h1>
{% endblock %}

  1. Com esses arquivos no lugar, agora você pode usar o script de gerenciamento para executar o aplicativo:

    $ ./manage.py runserver # visit http://localhost:5000 to verify that the app works correctly.

  2. Agora, colete seus ativos estáticos pela primeira vez. Antes de fazer isso, vale a pena notar novamente que você NÃO deve ter uma pasta static/ no nível superior do seu aplicativo; é aqui que o Flask-Collect colocará todos os arquivos estáticos que serão coletados do seu aplicativo e os vários plugins que você pode estar usando. Se você tem uma pasta static/ no nível superior do seu aplicativo, você deve excluí-la completamente antes de continuar, pois começar com uma ardósia limpa é uma parte crítica de testemunhar/entender o que o Flask-Collect faz. Observe que esta instrução não se aplica ao uso diário, é simplesmente para ilustrar o fato de que o Flask-Collect criará esse diretório para você e, em seguida, colocará vários arquivos nele.

Com isso dito, você pode executar o seguinte comando para coletar seus ativos estáticos:

$ ./manage.py collect

Depois de fazer isso, você verá que o Flask-Collect criou esta pasta static/ de nível superior e contém os seguintes arquivos:

$ find ./static -type f # execute this from the top-level directory of your app, same dir that contains the manage.py script
static/bootstrap/css/bootstrap-theme.css
static/bootstrap/css/bootstrap-theme.css.map
static/bootstrap/css/bootstrap-theme.min.css
static/bootstrap/css/bootstrap.css
static/bootstrap/css/bootstrap.css.map
static/bootstrap/css/bootstrap.min.css
static/bootstrap/fonts/glyphicons-halflings-regular.eot
static/bootstrap/fonts/glyphicons-halflings-regular.svg
static/bootstrap/fonts/glyphicons-halflings-regular.ttf
static/bootstrap/fonts/glyphicons-halflings-regular.woff
static/bootstrap/fonts/glyphicons-halflings-regular.woff2
static/bootstrap/jquery.js
static/bootstrap/jquery.min.js
static/bootstrap/jquery.min.map
static/bootstrap/js/bootstrap.js
static/bootstrap/js/bootstrap.min.js
static/bootstrap/js/npm.js
static/css/styles.css
static/js/main.js

E é isso: use o comando collect sempre que você fizer edições no CSS ou JavaScript do seu aplicativo, ou quando você atualizar um plug-in do Flask que fornece ativos estáticos (como Flask-Bootstrap neste exemplo).
