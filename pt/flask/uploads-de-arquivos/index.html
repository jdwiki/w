Sintaxe #

  • request.files[’name’] # único arquivo necessário
  • request.files.get(’name’) # Nenhum se não for postado
  • request.files.getlist(’name’) # lista de zero ou mais arquivos postados
  • CombinedMultiDict((request.files, request.form)) # combina dados de formulário e arquivo

Fazendo upload de arquivos #

Formulário HTML #

  • Use um file type input e o navegador fornecerá um campo que permite ao usuário selecionar um arquivo para upload.
  • Somente formulários com o método post podem enviar dados de arquivos.
  • Certifique-se de definir o atributo enctype=multipart/form-data do formulário. Caso contrário, o nome do arquivo será enviado, mas não os dados do arquivo.
  • Use o atributo multiple na entrada para permitir a seleção de vários arquivos para um único campo.
<form method=post enctype=multipart/form-data>
    <!-- single file for the "profile" field -->
    <input type=file name=profile>
    <!-- multiple files for the "charts" field -->
    <input type=file multiple name=charts>
    <input type=submit>
</form>

Solicitações Python #

Requests é uma poderosa biblioteca Python para fazer solicitações HTTP. Você pode usá-lo (ou outras ferramentas) para postar arquivos sem um navegador.

  • Abra os arquivos para ler em modo binário.
  • Existem várias estruturas de dados que os files usam. Isso demonstra uma lista de tuplas (name, data), que permite vários arquivos como o formulário acima.
import requests

with open('profile.txt', 'rb') as f1, open('chart1.csv', 'rb') as f2, open('chart2.csv', 'rb') as f3:
    files = [
        ('profile', f1),
        ('charts', f2),
        ('charts', f3)
    ]
    requests.post('http://localhost:5000/upload', files=files)

Esta não pretende ser uma lista exaustiva. Para obter exemplos usando sua ferramenta favorita ou cenários mais complexos, consulte os documentos dessa ferramenta.

Passando dados para WTForms e Flask-WTF #

WTForms fornece um FileField para renderizar uma entrada de tipo de arquivo. Ele não faz nada de especial com os dados enviados. No entanto, como o Flask divide os dados do formulário (request.form) e os dados do arquivo (request.files), você precisa ter certeza de passar os dados corretos ao criar o formulário. Você pode usar um CombinedMultiDict para combinar os dois em uma única estrutura que o WTForms entende.

form = ProfileForm(CombinedMultiDict((request.files, request.form)))

Se você estiver usando Flask-WTF, uma extensão para integrar Flask e WTForms, a transmissão dos dados corretos será tratada automaticamente.

Devido a um bug no WTForms, apenas um arquivo estará presente para cada campo, mesmo que vários tenham sido carregados. Consulte este problema para obter mais detalhes. Será corrigido em 3.0.

Salvar uploads no servidor #

Os arquivos enviados estão disponíveis em request.files, um MultiDict mapeando nomes de campos para objetos de arquivo. Use getlist — em vez de [] ou get — se vários arquivos foram carregados com o mesmo nome de campo.

request.files['profile']  # single file (even if multiple were sent)
request.files.getlist('charts')  # list of files (even if one was sent)

Os objetos em request.files possuem um método save que salva o arquivo localmente. Crie um diretório comum para salvar os arquivos.

O atributo filename é o nome com o qual o arquivo foi carregado. Isso pode ser definido arbitrariamente pelo cliente, então passe-o pelo método secure_filename para gerar um nome válido e seguro para salvar. Isso não garante que o nome seja único, portanto, os arquivos existentes serão substituídos, a menos que você faça um trabalho extra para detectá-lo.

import os
from flask import render_template, request, redirect, url_for
from werkzeug import secure_filename

# Create a directory in a known location to save files to.
uploads_dir = os.path.join(app.instance_path, 'uploads')
os.makedirs(uploads_dir, exists_ok=True)

@app.route('/upload', methods=['GET', 'POST'])
def upload():
    if request.method == 'POST':
        # save the single "profile" file
        profile = request.files['profile']
        profile.save(os.path.join(uploads_dir, secure_filename(profile.filename)))

        # save each "charts" file
        for file in request.files.getlist('charts'):
            file.save(os.path.join(uploads_dir, secure_filename(file.name)))

        return redirect(url_for('upload'))

    return render_template('upload.html')

PARSE CSV UPLOAD COMO LISTA DE DICIONÁRIOS EM FLASK SEM SALVAR #

Os desenvolvedores geralmente precisam criar sites que permitem que os usuários carreguem um arquivo CSV. Normalmente, não há motivo para salvar o arquivo CSV real, pois os dados serão processados ​​e/ou armazenados em um banco de dados após o upload. No entanto, muitos, se não a maioria, métodos PYTHON de análise de dados CSV exigem que os dados sejam lidos como um arquivo. Isso pode causar um pouco de dor de cabeça se você estiver usando o FLASK para desenvolvimento web.

Suponha que nosso CSV tenha uma linha de cabeçalho e se pareça com o seguinte:

h1,h2,h3
'yellow','orange','blue'
'green','white','black'
'orange','pink','purple'

Agora, suponha que o formulário html para fazer upload de um arquivo seja o seguinte:

<form action="upload.html" method="post" enctype="multipart/form-data">
    <input type="file" name="fileupload" id="fileToUpload">
    <input type="submit" value="Upload File" name="submit">
</form>

Como ninguém quer reinventar a roda, você decide IMPORTAR csv em seu script FLASK. Não há garantia de que as pessoas farão upload do arquivo csv com as colunas na ordem correta. Se o arquivo csv tiver uma linha de cabeçalho, com a ajuda do método csv.DictReader você poderá ler o arquivo CSV como uma lista de dicionários, codificados pelas entradas na linha de cabeçalho. No entanto, csv.DictReader precisa de um arquivo e não aceita strings diretamente. Você pode achar que precisa usar os métodos FLASK para primeiro salvar o arquivo carregado, obter o novo nome e local do arquivo, abri-lo usando csv.DictReader e, em seguida, excluir o arquivo. Parece um pouco de desperdício.

Felizmente, podemos obter o conteúdo do arquivo como uma string e depois dividir a string por linhas terminadas. O método csv csv.DictReader aceitará isso como substituto de um arquivo. O código a seguir demonstra como isso pode ser feito sem salvar temporariamente o arquivo.

@application.route('upload.html',methods = ['POST'])
def upload_route_summary():
    if request.method == 'POST':

        # Create variable for uploaded file
        f = request.files['fileupload']  

        #store the file contents as a string
        fstring = f.read()
        
        #create list of dictionaries keyed by header row
        csv_dicts = [{k: v for k, v in row.items()} for row in csv.DictReader(fstring.splitlines(), skipinitialspace=True)]

        #do something list of dictionaries
    return "success"

A variável csv_dicts agora é a seguinte lista de dicionários:

   csv_dicts = 
    [
        {'h1':'yellow','h2':'orange','h3':'blue'},
        {'h1':'green','h2':'white','h3':'black'},
        {'h1':'orange','h2':'pink','h3':'purple'}
    ]

Caso você seja novo no PYTHON, você pode acessar dados como os seguintes:

csv_dicts[1]['h2'] = 'white'
csv_dicts[0]['h3'] = 'blue'

Outras soluções envolvem importar o módulo io e usar o método io.Stream. Eu sinto que esta é uma abordagem mais direta. Acredito que o código seja um pouco mais fácil de seguir do que usar o método io. Essa abordagem é específica para o exemplo de análise de um arquivo CSV carregado.
