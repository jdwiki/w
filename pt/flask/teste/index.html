Testando uma API JSON implementada no Flask #

Este exemplo pressupõe que você saiba como [testar um aplicativo Flask usando pytest](https://www.wikiod.com/pt/flask/teste#Testando nosso aplicativo Hello World)

Abaixo está uma API que recebe uma entrada JSON com valores inteiros a e b, por exemplo {"a": 1, "b": 2}, soma-os e retorna a soma a + b em uma resposta JSON, por exemplo {"soma": 3}.

# hello_add.py
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/add', methods=['POST'])
def add():
    data = request.get_json()
    return jsonify({'sum': data['a'] + data['b']})

Testando esta API com pytest #

Podemos testá-lo com pytest

# test_hello_add.py
from hello_add import app
from flask import json

def test_add():        
    response = app.test_client().post(
        '/add',
        data=json.dumps({'a': 1, 'b': 2}),
        content_type='application/json',
    )

    data = json.loads(response.get_data(as_text=True))

    assert response.status_code == 200
    assert data['sum'] == 3

Agora execute o teste com o comando py.test.

Testando nosso aplicativo Hello World #

Introdução #

Neste exemplo minimalista, usando pytest vamos testar que de fato nosso aplicativo Hello World retorna “Hello, World!” com um código de status HTTP OK de 200, quando atingido com uma solicitação GET na URL /

Primeiro vamos instalar pytest em nosso virtualenv

pip install pytest

E apenas para referência, este nosso aplicativo hello world:

# hello.py
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello, World!'

Definindo o teste #

Junto com nosso hello.py, definimos um módulo de teste chamado test_hello.py que será descoberto por py.test

# test_hello.py
from hello import app

def test_hello():
    response = app.test_client().get('/')

    assert response.status_code == 200
    assert response.data == b'Hello, World!'

Apenas para revisar, neste momento nossa estrutura de projeto obtida com o comando tree é:

.
├── hello.py
└── test_hello.py

Executando o teste #

Agora podemos executar este teste com o comando py.test que descobrirá automaticamente nosso test_hello.py e a função de teste dentro dele

$ py.test

Você deve ver alguma saída e uma indicação de que 1 teste passou, por exemplo

=== test session starts ===
collected 1 items 
test_hello.py .
=== 1 passed in 0.13 seconds ===

Acessando e manipulando variáveis ​​de sessão em seus testes usando Flask-Testing #

A maioria dos aplicativos da Web usa o objeto de sessão para armazenar algumas informações importantes. Esses exemplos mostram como você pode testar esse aplicativo usando o Flask-Testing. O exemplo completo de trabalho também está disponível no github.

Então, primeiro instale o Flask-Testing no seu virtualenv

pip install flask_testing

Para poder usar o objeto de sessão, você deve definir a chave secreta

app.secret_key = 'my-seCret_KEy'

Vamos imaginar que você tenha em sua aplicação uma função que precise armazenar alguns dados em variáveis ​​de sessão como esta

@app.route('/getSessionVar', methods=['GET', 'POST'])
def getSessionVariable():
  if 'GET' == request.method:
    session['sessionVar'] = 'hello'
  elif 'POST' == request.method:
    session['sessionVar'] = 'hi'
  else:
    session['sessionVar'] = 'error'

  return 'ok'

Para testar esta função, você pode importar flask_testing e deixar sua classe de teste herdar flask_testing.TestCase. Importe também todas as bibliotecas necessárias

import flask
import unittest
import flask_testing
from myapp.run import app
    
class TestMyApp(flask_testing.TestCase):

Muito importante antes de começar o teste é implementar a função create_app caso contrário haverá exceção.

  def create_app(self):
    return app

Para testar se seu aplicativo está funcionando como desejado, você tem algumas possibilidades. Se você quiser apenas garantir que sua função esteja definindo valores específicos para uma variável de sessão, basta manter o contexto e acessar flask.session

def testSession1(self):
    with app.test_client() as lTestClient:
      lResp= lTestClient.get('/getSessionVar')
      self.assertEqual(lResp.status_code, 200)
      self.assertEqual(flask.session['sessionVar'], 'hello')

Mais um truque útil é diferenciar entre os métodos GET e POST como na próxima função de teste

def testSession2(self):
    with app.test_client() as lTestClient:
      lResp= lTestClient.post('/getSessionVar')
      self.assertEqual(lResp.status_code, 200)
      self.assertEqual(flask.session['sessionVar'], 'hi')

Agora imagine que sua função espera que uma variável de sessão seja definida e reaja diferente em valores específicos como este

@app.route('/changeSessionVar')
def changeSessionVariable():
  if session['existingSessionVar'] != 'hello':
    raise Exception('unexpected session value of existingSessionVar!')

  session['existingSessionVar'] = 'hello world'
  return 'ok'

Para testar esta função, você deve usar a chamada transação de sessão e abrir a sessão no contexto do cliente de teste. Esta função está disponível desde Flask 0.8

def testSession3(self):
    with app.test_client() as lTestClient:
      #keep the session
      with lTestClient.session_transaction() as lSess:
        lSess['existingSessionVar'] = 'hello'

      #here the session is stored
      lResp = lTestClient.get('/changeSessionVar')
      self.assertEqual(lResp.status_code, 200)
      self.assertEqual(flask.session['existingSessionVar'], 'hello world')

A execução dos testes é como de costume para o unittest

if __name__ == "__main__":
    unittest.main()

E na linha de comando

python tests/test_myapp.py

Outra boa maneira de executar seus testes é usar o unittest Discovery assim:

python -m unittest discover -s tests
