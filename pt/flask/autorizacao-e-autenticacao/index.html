Usando extensão flask-login #

Uma das maneiras mais simples de implementar um sistema de autorização é usar a extensão flask-login. O site do projeto contém um início rápido detalhado e bem escrito, cuja versão mais curta está disponível neste exemplo.

Ideia geral #

A extensão expõe um conjunto de funções usadas para:

  • logar usuários
  • desconectando usuários
  • verificar se um usuário está logado ou não e descobrir qual usuário é esse

O que não faz e o que você tem que fazer por conta própria:

  • não fornece uma maneira de armazenar os usuários, por exemplo, no banco de dados
  • não fornece uma maneira de verificar as credenciais do usuário, por exemplo, nome de usuário e senha

Abaixo, há um conjunto mínimo de etapas necessárias para que tudo funcione.

Recomendo colocar todo o código relacionado à autenticação em um módulo ou pacote separado, por exemplo auth.py. Dessa forma, você pode criar as classes, objetos ou funções personalizadas necessários separadamente.

Crie um LoginManager #

A extensão usa uma classe LoginManager que deve ser registrada em seu objeto de aplicação Flask.

from flask_login import LoginManager
login_manager = LoginManager()
login_manager.init_app(app) # app is a Flask object

Como mencionado anteriormente, o LoginManager pode, por exemplo, ser uma variável global em um arquivo ou pacote separado. Então ele pode ser importado no arquivo em que o objeto Flask ​​é criado ou na sua função de fábrica de aplicativos e inicializado.

Especifique um retorno de chamada usado para carregar usuários #

Normalmente, os usuários serão carregados de um banco de dados. O retorno de chamada deve retornar um objeto que represente um usuário correspondente ao ID fornecido. Deve retornar None se o ID não for válido.

@login_manager.user_loader
def load_user(user_id):
    return User.get(user_id) # Fetch the user from the database

Isso pode ser feito diretamente abaixo da criação do seu LoginManager.

Uma classe representando seu usuário #

Como mencionado, o callback user_loader deve retornar um objeto que representa um usuário. O que isso significa exatamente? Esse objeto pode, por exemplo, ser um wrapper em torno de objetos de usuário armazenados em seu banco de dados ou simplesmente diretamente um modelo de seu banco de dados. Esse objeto deve implementar os seguintes métodos e propriedades. Isso significa que, se o retorno de chamada retornar seu modelo de banco de dados, você precisará garantir que as propriedades e métodos mencionados sejam adicionados ao seu modelo.

  • is_autenticado

    This property should return True if the user is authenticated, i.e. they have provided valid credentials. You will want to ensure that the objects which represent your users returned by the user_loader callback return True for that method.

  • is_ativo

    This property should return True if this is an active user - in addition to being authenticated, they also have activated their account, not been suspended, or any condition your application has for rejecting an account. Inactive accounts may not log in. If you don’t have such a mechanism present return True from this method.

  • é_anônimo

    This property should return True if this is an anonymous user. That means that your user object returned by the user_loader callback should return True.

  • get_id()

    This method must return a unicode that uniquely identifies this user, and can be used to load the user from the user_loader callback. Note that this must be a unicode - if the ID is natively an int or some other type, you will need to convert it to unicode. If the user_loader callback returns objects from the database this method will most likely return the database ID of this particular user. The same ID should of course cause the user_loader callback to return the same user later on.

Se você quiser tornar as coisas mais fáceis para você (**é de fato recomendado) você pode herdar de UserMixin no objeto retornado pelo callback user_loader (presumivelmente um modelo de banco de dados). Você pode ver como esses métodos e propriedades são implementados por padrão neste mixin aqui.

Logando os usuários #

A extensão deixa a validação do nome de usuário e senha inseridos pelo usuário para você. Na verdade, a extensão não se importa se você usa uma combinação de nome de usuário e senha ou outro mecanismo. Este é um exemplo de login de usuários usando nome de usuário e senha.

@app.route('/login', methods=['GET', 'POST'])
def login():
    # Here we use a class of some kind to represent and validate our
    # client-side form data. For example, WTForms is a library that will
    # handle this for us, and we use a custom LoginForm to validate.
    form = LoginForm()
    if form.validate_on_submit():
        # Login and validate the user.
        # user should be an instance of your `User` class
        login_user(user)

        flask.flash('Logged in successfully.')

        next = flask.request.args.get('next')
        # is_safe_url should check if the url is safe for redirects.
        # See http://flask.pocoo.org/snippets/62/ for an example.
        if not is_safe_url(next):
            return flask.abort(400)

        return flask.redirect(next or flask.url_for('index'))
    return flask.render_template('login.html', form=form)

Em geral, o login de usuários é feito chamando login_user e passando uma instância de um objeto que representa seu usuário mencionado anteriormente para ele. Como mostrado, isso geralmente acontecerá depois de recuperar o usuário do banco de dados e validar suas credenciais, no entanto, o objeto de usuário aparece magicamente neste exemplo.

Eu loguei um usuário, e agora? #

O objeto retornado pelo callback user_loader pode ser acessado de várias maneiras.

  • Nos modelos:

    The extension automatically injects it under the name current_user using a template context processor. To disable that behaviour and use your custom processor set add_context_processor=False in your LoginManager constructor.

      {% if current_user.is_authenticated %}
    Hi {{ current_user.name }}!
  {% endif %}

  • Em código Python:

    The extension provides a request-bound object called current_user.

      from flask_login import current_user    

  @app.route("/hello")
  def hello():
      # Assuming that there is a name property on your user object
      # returned by the callback
      if current_user.is_authenticated:
          return 'Hello %s!' % current_user.name 
      else:
          return 'You are not logged in!'

  • Limitando o acesso rapidamente usando um decorador Um decorador login_required pode ser usado para limitar o acesso rapidamente.

      from flask_login import login_required

  @app.route("/settings")
  @login_required
  def settings():
      pass

Desconectando usuários #

Os usuários podem ser desconectados chamando logout_user(). Parece que é seguro fazer isso mesmo se o usuário não estiver logado, então o decorador @login_required pode ser omitido.

@app.route("/logout")
@login_required
def logout():
    logout_user()
    return redirect(somewhere)

O que acontece se um usuário não estiver logado e eu acessar o objeto current_user? #

Por padrão, um AnonymousUserMixin é retornado:

  • is_active e is_authenticated são False
  • is_anonymous é True
  • get_id() retorna Nenhum

Para usar um objeto diferente para usuários anônimos, forneça um callable (uma classe ou função de fábrica) que cria usuários anônimos em seu LoginManager com:

login_manager.anonymous_user = MyAnonymousUser

Qual o proximo? #

Isso conclui a introdução básica à extensão. Para saber mais sobre configuração e opções adicionais, é altamente recomendável ler o guia oficial.

Tempo limite da sessão de login #

É uma boa prática expirar a sessão logada após um horário específico, você pode conseguir isso com o Flask-Login.

from flask import Flask, session
from datetime import timedelta
from flask_login import LoginManager, login_require, login_user, logout_user

# Create Flask application

app = Flask(__name__) 

# Define Flask-login configuration 

login_mgr = LoginManager(app)
login_mgr.login_view = 'login'
login_mgr.refresh_view = 'relogin'
login_mgr.needs_refresh_message = (u"Session timedout, please re-login")
login_mgr.needs_refresh_message_category = "info"


@app.before_request
def before_request():
    session.permanent = True
    app.permanent_session_lifetime = timedelta(minutes=5)

A vida útil da sessão padrão é de 31 dias, o usuário precisa especificar a visualização de atualização de login em caso de tempo limite.

app.permanent_session_lifetime = timedelta(minutes=5)

A linha acima forçará o usuário a fazer login novamente a cada 5 minutos.
