Esses “hacks” podem ser usados ​​para atingir um navegador/cliente específico. Isso pode ser usado para contornar as diferenças de renderização do navegador aplicando estilos em um dos wrappers listados acima.

Adicionando suporte a bloco em linha para IE6 e IE7 #

display: inline-block;

A propriedade display com o valor de inline-block não é suportada pelo Internet Explorer 6 e 7. Uma solução alternativa para isso é:

zoom: 1;
*display: inline;

A propriedade zoom aciona o recurso hasLayout dos elementos e está disponível apenas no Internet Explorer. O *display garante que a propriedade inválida seja executada apenas nos navegadores afetados. Outros navegadores simplesmente ignorarão a regra.

Modo de alto contraste no Internet Explorer 10 e superior #

No Internet Explorer 10+ e Edge, a Microsoft fornece o seletor de mídia -ms-high-contrast para expor a configuração “Alto Contraste” do navegador, que permite ao programador ajustar os estilos do site de acordo.

O seletor -ms-high-contrast tem 3 estados: active, black-on-white e white-on-black. No IE10+, ele também tinha um estado none, mas isso não é mais suportado no Edge daqui para frente.

Exemplos #

@media screen and (-ms-high-contrast: active), (-ms-high-contrast: black-on-white) {
   .header{
      background: #fff;
      color: #000;
   }
}

Isso mudará o fundo do cabeçalho para branco e a cor do texto para preto quando o modo de alto contraste estiver ativo e estiver no modo preto sobre branco.

@media screen and (-ms-high-contrast: white-on-black) {
   .header{
      background: #000;
      color: #fff;
   }
}

Semelhante ao primeiro exemplo, mas isso seleciona especificamente apenas o estado ‘branco sobre preto’ e inverte as cores do cabeçalho para um fundo preto com texto branco.

Documentação da Microsoft em -ms-high-contrast

Somente Internet Explorer 6 e Internet Explorer 7 #

Para direcionar o Internet Explorer 6 e o ​​Internet Explorer 7, inicie suas propriedades com *:

.hide-on-ie6-and-ie7 {
    *display : none; // This line is processed only on IE6 and IE7
}

Prefixos não alfanuméricos (além de hífens e sublinhados) são ignorados no IE6 e IE7, então este hack funciona para qualquer par propriedade:valor não prefixado.

Somente Internet Explorer 8 #

Para direcionar o Internet Explorer 8, envolva seus seletores dentro de @media \0 screen { }:

@media \0 screen {
    .hide-on-ie8 {
        display : none;
    }
}

Tudo entre @media \0 screen { } é processado apenas por I
