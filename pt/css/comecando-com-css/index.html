Folha de estilo externa #

Uma folha de estilo CSS externa pode ser aplicada a qualquer número de documentos HTML colocando um elemento <link> em cada documento HTML.

O atributo rel da tag <link> deve ser definido como "stylesheet", e o atributo href para o caminho relativo ou absoluto para a folha de estilo. Embora o uso de caminhos de URL relativos seja geralmente considerado uma boa prática, os caminhos absolutos também podem ser usados. Em HTML5 o atributo type pode ser omitido.

É recomendado que a tag <link> seja colocada na tag <head> do arquivo HTML para que os estilos sejam carregados antes dos elementos que eles estilizam. Caso contrário, os usuários verão um flash de conteúdo sem estilo.

Exemplo #

hello-world.html

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8" />
        <link rel="stylesheet" type="text/css" href="style.css">
    </head>
    <body>
        <h1>Hello world!</h1>
        <p>I ♥ CSS</p>
    </body>
</html>

estilo.css

h1 {
    color: green;
    text-decoration: underline;
}
p {
    font-size: 25px;
    font-family: 'Trebuchet MS', sans-serif;
}

Certifique-se de incluir o caminho correto para seu arquivo CSS no href. Se o arquivo CSS estiver na mesma pasta que seu arquivo HTML, nenhum caminho será necessário (como o exemplo acima), mas se estiver salvo em uma pasta, especifique-o assim href="foldername/style.css".

<link rel="stylesheet" type="text/css" href="foldername/style.css">

Folhas de estilo externas são consideradas a melhor maneira de lidar com seu CSS. Há uma razão muito simples para isso: quando você gerencia um site de, digamos, 100 páginas, todas controladas por uma única folha de estilo, e deseja alterar as cores do link de azul para verde, é muito mais fácil fazer a alteração em seu arquivo CSS e deixe as alterações “em cascata” em todas as 100 páginas do que entrar em 100 páginas separadas e fazer a mesma alteração 100 vezes. Novamente, se você quiser mudar completamente a aparência do seu site, você só precisa atualizar este arquivo.

Você pode carregar quantos arquivos CSS em sua página HTML forem necessários.

<link rel="stylesheet" type="text/css" href="main.css">
<link rel="stylesheet" type="text/css" href="override.css">

As regras CSS são aplicadas com algumas regras básicas, e a ordem importa. Por exemplo, se você tiver um arquivo main.css com algum código nele:

p.green { color: #00FF00; }

Todos os seus parágrafos com a classe ‘verde’ serão escritos em verde claro, mas você pode substituir isso por outro arquivo .css apenas incluindo-o depois de main.css. Você pode ter override.css com o seguinte código seguindo main.css, por exemplo:

p.green { color: #006600; }

Agora todos os seus parágrafos com a classe ‘verde’ serão escritos em verde mais escuro ao invés de verde claro.

Outros princípios se aplicam, como a regra ‘importante’, especificidade e herança.

Quando alguém visita seu site pela primeira vez, o navegador faz o download do HTML da página atual mais o arquivo CSS vinculado. Então, quando eles navegam para outra página, seu navegador só precisa baixar o HTML dessa página; o arquivo CSS é armazenado em cache, portanto, não precisa ser baixado novamente. Como os navegadores armazenam em cache a folha de estilo externa, suas páginas são carregadas mais rapidamente.

Estilos internos #

CSS dentro de tags <style></style> dentro de um documento HTML funciona como uma folha de estilo externa, exceto que ele fica no documento HTML que ele estiliza em vez de em um arquivo separado e, portanto, só pode ser aplicado ao documento em que vive. Observe que este elemento deve estar dentro do elemento <head> para validação HTML (embora funcione em todos os navegadores atuais se colocado em body).

<head>
    <style>
        h1 {
            color: green;
            text-decoration: underline;
        }
        p {
            font-size: 25px;
            font-family: 'Trebuchet MS', sans-serif;
        }
    </style>
</head>
<body>
    <h1>Hello world!</h1>
    <p>I ♥ CSS</p>
</body>

Regra CSS @import (uma das regras CSS) #

A regra @import CSS é usada para importar regras de estilo de outras folhas de estilo. Essas regras devem preceder todos os outros tipos de regras, exceto regras @charset; como não é uma instrução aninhada, @import não pode ser usado dentro do grupo condicional at-rules. @import.

Como usar @import #

Você pode usar a regra @import das seguintes maneiras:

UMA. Com etiqueta de estilo interna

   <style>
    @import url('/css/styles.css');
   </style>

B. Com folha de estilo externa

A linha a seguir importa um arquivo CSS chamado additional-styles.css no diretório raiz para o arquivo CSS no qual ele aparece:

@import '/additional-styles.css';

Importar CSS externo também é possível. Um caso de uso comum são os arquivos de fonte.

@import 'https://fonts.googleapis.com/css?family=Lato';

Um segundo argumento opcional para a regra @import é uma lista de consultas de mídia:

@import '/print-styles.css' print;
@import url('landscape.css') screen and (orientation:landscape);

Estilos embutidos #

Use estilos embutidos para aplicar estilo a um elemento específico. Observe que isso não é o ideal. Colocar regras de estilo em uma tag <style> ou arquivo CSS externo é recomendado para manter uma distinção entre conteúdo e apresentação.

Os estilos embutidos substituem qualquer CSS em uma tag <style> ou folha de estilo externa. Embora isso possa ser útil em algumas circunstâncias, esse fato geralmente reduz a capacidade de manutenção de um projeto.

Os estilos no exemplo a seguir se aplicam diretamente aos elementos aos quais estão anexados.

<h1 style="color: green; text-decoration: underline;">Hello world!</h1>
<p style="font-size: 25px; font-family: 'Trebuchet MS';">I ♥ CSS</p>

Os estilos inline geralmente são a maneira mais segura de garantir a compatibilidade de renderização em vários clientes de e-mail, programas e dispositivos, mas pode ser demorado para escrever e um pouco desafiador para gerenciar.

Alterando CSS com JavaScript #

JavaScript puro #

É possível adicionar, remover ou alterar valores de propriedade CSS com JavaScript através da propriedade style de um elemento.

var el = document.getElementById("element");
el.style.opacity = 0.5;
el.style.fontFamily = 'sans-serif';

Observe que as propriedades de estilo são nomeadas em estilo camel minúsculo. No exemplo você vê que a propriedade CSS font-family se torna fontFamily em javascript.

Como alternativa para trabalhar diretamente em elementos, você pode criar um elemento <style> ou <link> em JavaScript e anexá-lo ao <body> ou `` do documento HTML.

##jQuery ##

Modificar propriedades CSS com jQuery é ainda mais simples.

$('#element').css('margin', '5px');

Se você precisar alterar mais de uma regra de estilo:

$('#element').css({
    margin: "5px",
    padding: "10px",
    color: "black"
});

jQuery inclui duas maneiras de alterar as regras CSS que contêm hífens (ou seja, font-size). Você pode colocá-los entre aspas ou colocar em maiúsculas o nome da regra de estilo.

$('.example-class').css({
    "background-color": "blue",
    fontSize: "10px"
});

Estilizando listas com CSS #

Existem três propriedades diferentes para estilizar itens de lista: list-style-type, list-style-image e list-style-position, que devem ser declarados nessa ordem. Os valores padrão são disco, externo e nenhum, respectivamente. Cada propriedade pode ser declarada separadamente ou usando a propriedade abreviada list-style.

list-style-type define a forma ou tipo de marcador usado para cada item da lista.

Alguns dos valores aceitáveis ​​para list-style-type:

  • disco
  • círculo
  • quadrado
  • decimal
  • baixo-romano
  • alto-romano
  • Nenhum

(Para obter uma lista completa, consulte o wiki de especificações do W3C)

Para usar marcadores quadrados para cada item de lista, por exemplo, você usaria o seguinte par de valor de propriedade:

li {
    list-style-type: square;
}

A propriedade list-style-image determina se o ícone do item da lista é definido com uma imagem e aceita um valor de none ou um URL que aponta para uma imagem.

li {
    list-style-image: url(images/bullet.png);
}

A propriedade list-style-position define onde posicionar o marcador de item da lista e aceita um dos dois valores: “dentro” ou “fora”.

li {
  list-style-position: inside;
}
