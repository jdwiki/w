Sintaxe #

  • transição: <propriedade> <duração> <função de temporização> <atraso>;
  • @keyframes <identificador>
  • [ [ de | para | <porcentagem> ] [, de | para | <porcentagem> ]* bloco ]*

Parâmetros #

Transição  
Parâmetro Detalhes
propriedade Ou a propriedade CSS para transição, ou all, que especifica todas as propriedades passíveis de transição.
duração Tempo de transição, em segundos ou milissegundos.
função de temporização Especifica uma função para definir como os valores intermediários das propriedades são calculados. Os valores comuns são ease, linear e step-end. Confira a folha de dicas da função de facilitação para saber mais.
atraso Quantidade de tempo, em segundos ou milissegundos, para esperar antes de reproduzir a animação.
   
@keyframes  
[ de | para | <porcentagem>] Você pode especificar um tempo definido com um valor percentual ou dois valores percentuais, ou seja, 10%, 20%, para um período de tempo em que os atributos definidos do quadro-chave são definidos.
bloquear Qualquer quantidade de atributos CSS para o quadro-chave.

Animações com quadros-chave #

Para animações CSS de vários estágios, você pode criar CSS @keyframes. Os quadros-chave permitem definir vários pontos de animação, chamados de quadro-chave, para definir animações mais complexas.

Exemplo básico #

Neste exemplo, faremos uma animação de fundo básica que alterna entre todas as cores.

@keyframes rainbow-background {
         0%     { background-color: #ff0000; }
     8.333%     { background-color: #ff8000; }
    16.667%     { background-color: #ffff00; }
    25.000%     { background-color: #80ff00; }
    33.333%     { background-color: #00ff00; }
    41.667%     { background-color: #00ff80; }
    50.000%     { background-color: #00ffff; }
    58.333%     { background-color: #0080ff; }
    66.667%     { background-color: #0000ff; }
    75.000%     { background-color: #8000ff; }
    83.333%     { background-color: #ff00ff; }
    91.667%     { background-color: #ff0080; }
    100.00%     { background-color: #ff0000; }
}

.RainbowBackground {
    animation: rainbow-background 5s infinite;
}

Ver resultado

Há algumas coisas diferentes a serem observadas aqui. Primeiro, a sintaxe real @keyframes.

@keyframes rainbow-background{

Isso define o nome da animação como rainbow-background.

0%     { background-color: #ff0000; }

Esta é a definição de um quadro-chave dentro da animação. A primeira parte, o 0% no caso, define onde o quadro-chave está durante a animação. O 0% implica que é 0% do tempo total de animação desde o início.

A animação fará a transição automática entre os quadros-chave. Portanto, ao definir a próxima cor de fundo em 8,333%, a animação levará 8,333% do tempo para a transição entre esses quadros-chave.

.RainbowBackground {
    animation: rainbow-background 5s infinite;
}

Este código anexa nossa animação a todos os elementos que possuem a classe .RainbowBackground.

A propriedade de animação real usa os seguintes argumentos.

  • animation-name: O nome da nossa animação. Neste caso, fundo do arco-íris
  • animação-duração: quanto tempo a animação levará, neste caso 5 segundos.
  • animation-iteration-count (opcional): o número de vezes que a animação será repetida. Nesse caso, a animação continuará indefinidamente. Por padrão, a animação será reproduzida uma vez.
  • animation-delay (opcional): especifica quanto tempo esperar antes do início da animação. O padrão é 0 segundos e pode assumir valores negativos. Por exemplo, -2s iniciaria a animação 2 segundos em seu loop.
  • função de tempo de animação (opcional): especifica a curva de velocidade da animação. O padrão é ease, onde a animação começa lenta, fica mais rápida e termina lenta.

Neste exemplo em particular, os quadros-chave 0% e 100% especificam { background-color: #ff0000; }. Sempre que dois ou mais quadros-chave compartilham um estado, pode-se especificá-los em uma única instrução. Nesse caso, as duas linhas 0% e 100% podem ser substituídas por esta única linha:

0%, 100%     { background-color: #ff0000; }

Compatibilidade entre navegadores #

Para navegadores mais antigos baseados em WebKit, você precisará usar o prefixo do fornecedor na declaração @keyframes e na propriedade animation, assim:

@-webkit-keyframes{}

-webkit-animation: ...

Animações com a propriedade de transição #

Útil para animações simples, a propriedade CSS transition permite que propriedades CSS baseadas em números sejam animadas entre estados.

Exemplo #

.Example{
    height: 100px;
    background: #fff;
}

.Example:hover{
    height: 120px;
    background: #ff0000;
}

Ver resultado

Por padrão, passar o mouse sobre um elemento com a classe .Example faria imediatamente com que a altura do elemento saltasse para 120px e sua cor de fundo para vermelho (#ff0000).

Ao adicionar a propriedade transition, podemos fazer com que essas mudanças ocorram ao longo do tempo:

.Example{
    ...
    transition: all 400ms ease;
}

Ver resultado

O valor all aplica a transição a todas as propriedades compatíveis (baseadas em números). Qualquer nome de propriedade compatível (como height ou top) pode ser substituído por esta palavra-chave.

400ms especifica a quantidade de tempo que a transição leva. Nesse caso, a mudança de altura do elemento levará 400 milissegundos para ser concluída.

Finalmente, o valor ease é a função de animação, que determina como a animação é reproduzida. facilidade significa começar devagar, acelerar e terminar devagar novamente. Outros valores são linear, ease-out e ease-in.

Compatibilidade entre navegadores #

A propriedade transition geralmente é bem suportada em todos os principais navegadores, exceto o IE 9. Para versões anteriores do Firefox e navegadores baseados em Webkit, use prefixos de fornecedor como:

.Example{
    transition:         all 400ms ease;
    -moz-transition:    all 400ms ease;
    -webkit-transition: all 400ms ease;
}

Observação: A propriedade transition pode animar alterações entre quaisquer dois valores numéricos, independentemente da unidade. Ele também pode fazer a transição entre unidades, como 100px para 50vh. No entanto, ele não pode fazer a transição entre um número e um valor padrão ou automático, como a transição da altura de um elemento de 100px para auto.

Exemplos de sintaxe #

Nosso primeiro exemplo de sintaxe mostra a propriedade abreviada de animação usando todas as propriedades/parâmetros disponíveis:

  animation: 3s         ease-in           1s      2                 reverse     both        paused       slidein;
  /*         duration | timing-function | delay | iteration-count | direction | fill-mode | play-state | name   */

Nosso segundo exemplo é um pouco mais simples e mostra que algumas propriedades podem ser omitidas:

  animation: 3s         linear            1s      slidein;
  /*         duration | timing-function | delay | name   */

Nosso terceiro exemplo mostra a declaração mais mínima. Observe que o nome da animação e a duração da animação devem ser declarados:

  animation: 3s         slidein;
  /*         duration | name */

Também vale a pena mencionar que ao usar a abreviação de animação a ordem das propriedades faz a diferença. Obviamente o navegador pode confundir sua duração com seu atraso.

Se a brevidade não é sua coisa, você também pode pular a propriedade abreviada e escrever cada propriedade individualmente:

animation-duration: 3s;
animation-timing-function: ease-in;
animation-delay: 1s;
animation-iteration-count: 2;
animation-direction: reverse;
animation-fill-mode: both;
animation-play-state: paused;
animation-name: slidein;

Aumentando o desempenho da animação usando o atributo will-change #

Ao criar animações e outras ações com uso intenso de GPU, é importante entender o atributo will-change.

Ambos os quadros-chave CSS e a propriedade transition usam aceleração de GPU. O desempenho é aumentado transferindo os cálculos para a GPU do dispositivo. Isso é feito criando camadas de pintura (partes da página que são renderizadas individualmente) que são transferidas para a GPU a ser calculada. A propriedade will-change informa ao navegador o que será animado, permitindo que o navegador crie áreas de pintura menores, aumentando assim o desempenho.

A propriedade will-change aceita uma lista de propriedades separadas por vírgulas para serem animadas. Por exemplo, se você planeja transformar um objeto e alterar sua opacidade, especifique:

.Example{
    ...
    will-change: transform, opacity;
}

Observação: Use will-change com moderação. Definir will-change para cada elemento em uma página pode causar problemas de desempenho, pois o navegador pode tentar criar camadas de pintura para cada elemento, aumentando significativamente a quantidade de processamento feito pela GPU.
