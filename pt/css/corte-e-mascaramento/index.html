Sintaxe #

  • Recorte
  • caminho do clipe: <fonte do clipe> | [ <forma básica> || <caixa de geometria de clipe> ] | Nenhum
  • Mascaramento
  • imagem-máscara: [ nenhum | <referência-máscara> ]#
  • modo de máscara: [ <modo de máscara> ]#
  • repetição de máscara: [ <estilo de repetição ]#
  • posição da máscara: [ <posição> ]#
  • clipe de máscara: [ <caixa de geometria> | nenhum clipe ]#
  • mask-origin: [ <geometry-box> ]#
  • tamanho da máscara: [ <bg-size> ]#
  • mask-composite: [ <compositing-operator> ]#
  • máscara: [ <referência de máscara> <modo de máscara>? || <posição> [ / <tamanho bg> ]? || <repetir-estilo> || <caixa de geometria> || [ <caixa de geometria> | sem clipe ] || <operador de composição> ]#

Parâmetros #

Parâmetro Detalhes
fonte do clipe Um URL que pode apontar para um elemento SVG embutido (ou) um elemento SVG em um arquivo externo que contém a definição do caminho do clipe.
forma básica Refere-se a um entre inset(), circle(), ellipse() ou polygon(). Usando uma dessas funções, o caminho de recorte é definido. Essas funções de forma funcionam exatamente da mesma maneira que em Shapes for Floats
clip-geometria-caixa Isso pode ter um entre content-box, padding-box, border-box, margin-box, fill-box, stroke-box, view-box como valores. Quando isso é fornecido sem nenhum valor para <forma básica>, as bordas da caixa correspondente são usadas como o caminho para o recorte. Quando usado com uma <forma básica>, atua como a caixa de referência para a forma.
máscara-referência Pode ser none ou uma imagem ou um URL de referência para uma fonte de imagem de máscara.
estilo de repetição Isso especifica como a máscara deve ser repetida ou lado a lado nos eixos X e Y. Os valores suportados são repeat-x, repeat-y, repeat, space, round, no-repeat.
modo máscara Pode ser alpha ou luminance ou auto e indica se a máscara deve ser tratada como uma máscara alfa ou uma máscara de luminância. Se nenhum valor for fornecido e a referência de máscara for uma imagem direta, ela será considerada como máscara alfa (ou) se a referência de máscara for um URL, ela será considerada como máscara de luminância.
posição Isso especifica a posição de cada camada de máscara e é semelhante em comportamento à propriedade background-position. O valor pode ser fornecido em sintaxe de 1 valor (como top, 10%) ou em sintaxe de 2 valores (como top right, 50% 50%).
caixa de geometria Isso especifica a caixa na qual a máscara deve ser recortada (área de pintura da máscara) ou a caixa que deve ser usada como referência para a origem da máscara (área de posicionamento da máscara) dependendo da propriedade. A lista de valores possíveis são content-box, padding-box, border-box, margin-box, fill-box, stroke-box, view-box. A explicação detalhada de como cada um desses valores funciona está disponível em W3C Spec.
tamanho bg Isso representa o tamanho de cada camada de imagem de máscara e tem a mesma sintaxe que background-size. O valor pode ser comprimento ou porcentagem ou auto ou cobrir ou conter. Comprimento, porcentagem e auto podem ser fornecidos como um valor único ou como um para cada eixo.
operador de composição Isso pode ser qualquer um entre add, subtract, exclude, multiply por camada e define o tipo de operação de composição que deve ser usada para esta camada com aquelas abaixo dela. Uma explicação detalhada sobre cada valor está disponível nas Especificações do W3C.

CSS Clipping and Masking são conceitos muito novos e, portanto, o suporte do navegador para essas propriedades é bastante baixo.

Máscaras: #

No momento em que escrevo (julho de 16), Chrome, Safari e Opera suportam essas propriedades com o prefixo -webkit-.

O Firefox não requer prefixos, mas suporta máscaras apenas quando usado com elementos SVG mask. Para elementos mask SVG embutidos, a sintaxe é mask: url(#msk) enquanto para usar elementos mask em um arquivo SVG externo a sintaxe é mask: url('yourfilepath/yourfilename.svg#msk') . #msk em ambos os casos refere-se ao id do elemento mask que está sendo referido. Conforme indicado em esta resposta, no momento o Firefox não suporta nenhum parâmetro além de mask-reference em a propriedade mask.

O Internet Explorer (e o Edge) ainda não oferece suporte para essa propriedade.

A propriedade mask-mode atualmente não é suportada por nenhum navegador com ou sem prefixos.

Caminho do clipe: #

Como no momento em que escrevi (Jul ‘16), Chrome, Safari e Opera suportam clip-path quando o caminho é criado usando formas básicas (como círculo, polígono) ou a sintaxe url(#clipper) com SVG em linha. Eles não suportam recorte com base em formas que fazem parte de arquivos SVG externos. Além disso, eles exigem que o prefixo -webkit esteja presente.

Firefox suporta apenas a sintaxe url() para clip-path enquanto o Internet Explorer (e Edge) não oferecem suporte.

Máscara simples que desbota uma imagem de sólida para transparente #

CSS #

div {
  height: 200px;
  width: 200px;
  background: url(http://lorempixel.com/200/200/nature/1);
  mask-image: linear-gradient(to right, white, transparent); 
}

HTML #

<div></div>

No exemplo acima há um elemento com uma imagem como plano de fundo. A máscara aplicada na imagem (usando CSS) faz com que pareça que está desaparecendo da esquerda para a direita.

O mascaramento é obtido usando um gradiente linear que vai de branco (à esquerda) a transparente (à direita) como máscara. Como é uma máscara alfa, a imagem torna-se transparente onde a máscara é transparente.

Recorte (Círculo) #

CSS: #

div{
  width: 200px;
  height: 200px;
  background: teal;
  clip-path: circle(30% at 50% 50%); /* refer remarks before usage */
}

HTML #

<div></div>

Este exemplo mostra como recortar um div em um círculo. O elemento é recortado em um círculo cujo raio é de 30% com base nas dimensões da caixa de referência com seu ponto central no centro da caixa de referência. Aqui, uma vez que nenhuma <clip-geometry-box> (em outras palavras, caixa de referência) é fornecida, a border-box do elemento será usada como caixa de referência.

A forma do círculo precisa ter um raio e um centro com coordenadas (x,y):

circle(radius at x y)

Recorte (Polígono) #

CSS: #

div{
  width:200px;
  height:200px;
  background:teal;
  clip-path: polygon(0 0, 0 100%, 100% 50%); /* refer remarks before usage */
}

HTML: #

<div></div>

No exemplo acima, um caminho de recorte poligonal é usado para recortar o elemento quadrado (200 x 200) em uma forma de triângulo. A forma de saída é um triângulo porque o caminho começa em (ou seja, as primeiras coordenadas estão em) 0 0 - que é o canto superior esquerdo da caixa, depois vai para 0 100% - que é o canto inferior esquerdo canto da caixa e, finalmente, para 100% 50% que nada mais é do que o ponto central direito da caixa. Esses caminhos se fecham automaticamente (ou seja, o ponto inicial será o ponto final) e, portanto, a forma final é a de um triângulo.

Isso também pode ser usado em um elemento com uma imagem ou um gradiente como plano de fundo.

Usando máscaras para cortar um buraco no meio de uma imagem #

CSS #

div {
  width: 200px;
  height: 200px;
  background: url(http://lorempixel.com/200/200/abstract/6);
  mask-image: radial-gradient(circle farthest-side at center, transparent 49%, white 50%); /* check remarks before using */
}

HTML #

No exemplo acima, um círculo transparente é criado no centro usando radial-gradient e isso é usado como uma máscara para produzir o efeito de um círculo sendo cortado do centro de uma imagem.

Usando máscaras para criar imagens com formas irregulares #

CSS #

div { /* check remarks before usage */
  height: 200px;
  width: 400px;
  background-image: url(http://lorempixel.com/400/200/nature/4);
  mask-image: linear-gradient(to top right, transparent 49.5%, white 50.5%), linear-gradient(to top left, transparent 49.5%, white 50.5%), linear-gradient(white, white);
  mask-size: 75% 25%, 25% 25%, 100% 75%;
  mask-position: bottom left, bottom right, top left;
  mask-repeat: no-repeat;
}

HTML #

<div></div>

No exemplo acima, três imagens linear-gradient (que quando colocadas em suas posições apropriadas cobririam 100% x 100% do tamanho do contêiner) são usadas como máscaras para produzir um corte triangular transparente na parte inferior da imagem.
