A especificidade do CSS pretende promover a concisão do código, permitindo que um autor defina algumas regras gerais de formatação para um amplo conjunto de elementos e, em seguida, sobrescreva-as para um determinado subconjunto.

Calculando a especificidade do seletor #

Cada seletor de CSS individual tem seu próprio valor de especificidade. Cada seletor em uma sequência aumenta a especificidade geral da sequência. Os seletores se enquadram em um dos três grupos de especificidade diferentes: A, B e c. Quando várias sequências de seletores selecionam um determinado elemento, o navegador usa os estilos aplicados pela sequência com a maior especificidade geral.

Grupo Composto por Exemplos
A seletores de identificação #foo
B seletores de classe
seletores de atributo
pseudo-classes		 .bar
[title], [colspan="2"]
:hover, :nth-child(2)
c seletores de tipo
pseudo-elementos		 div, li
::antes, ::primeira letra

O grupo A é o mais específico, seguido pelo grupo B e, finalmente, o grupo c.

O seletor universal (*) e os combinadores (como > e ~) não têm especificidade.

Exemplo 1: Especificidade de várias sequências de seletores #

#foo #baz {}      /* a=2, b=0, c=0 */

#foo.bar {}       /* a=1, b=1, c=0 */

#foo {}           /* a=1, b=0, c=0 */

.bar:hover {}     /* a=0, b=2, c=0 */

div.bar {}        /* a=0, b=1, c=1 */

:hover {}         /* a=0, b=1, c=0 */

[title] {}        /* a=0, b=1, c=0 */

.bar {}           /* a=0, b=1, c=0 */

div ul + li {}    /* a=0, b=0, c=3 */

p::after {}       /* a=0, b=0, c=2 */

*::before {}      /* a=0, b=0, c=1 */

::before {}       /* a=0, b=0, c=1 */

div {}            /* a=0, b=0, c=1 */

* {}              /* a=0, b=0, c=0 */

Exemplo 2: Como a especificidade é usada pelo navegador #

Imagine a seguinte implementação de CSS:

#foo {
  color: blue;
}

.bar {
  color: red;
  background: black;
}

Aqui temos um seletor de ID que declara color como blue, e um seletor de classe que declara color como red e background como black.

Um elemento com um ID de #foo e uma classe de .bar será selecionado por ambas as declarações. Os seletores de ID têm uma especificidade de Grupo A e os seletores de classe têm uma especificidade de Grupo B. Um seletor de ID supera qualquer número de seletores de classe. Por isso, color:blue; do seletor #foo e background:black; do seletor .bar serão aplicados ao elemento. A maior especificidade do seletor de ID fará com que o navegador ignore a declaração de cor do seletor .bar.

Agora imagine uma implementação CSS diferente:

.bar {
  color: red;
  background: black;
}

.baz {
  background: white;
}

Aqui temos dois seletores de classe; um dos quais declara color como vermelho e fundo como preto, e o outro declara fundo como branco.

Um elemento com as classes .bar e .baz será afetado por ambas as declarações, porém o problema que temos agora é que tanto .bar quanto .baz têm uma especificidade de Grupo B idêntica . A natureza em cascata do CSS resolve isso para nós: como .baz é definido depois de .bar, nosso elemento termina com o red color de .bar mas o white background de .baz.

Exemplo 3: Como manipular a especificidade #

O último trecho do Exemplo 2 acima pode ser manipulado para garantir que a declaração color do nosso seletor de classe .bar seja usada em vez do seletor de classe .baz.

.bar {}        /* a=0, b=1, c=0 */
.baz {}        /* a=0, b=1, c=0 */

A maneira mais comum de conseguir isso seria descobrir quais outros seletores podem ser aplicados à seqüência de seletores .bar. Por exemplo, se a classe .bar fosse aplicada apenas a elementos span, poderíamos modificar o seletor .bar para span.bar. Isso daria a ele uma nova especificidade de Grupo C, que substituiria a falta do seletor .baz:

span.bar {}    /* a=0, b=1, c=1 */
.baz {}        /* a=0, b=1, c=0 */

No entanto, nem sempre é possível encontrar outro seletor comum que seja compartilhado entre qualquer elemento que use a classe .bar. Por causa disso, o CSS nos permite duplicar seletores para aumentar a especificidade. Em vez de apenas .bar, podemos usar .bar.bar (Veja A gramática dos seletores, Recomendação W3C) . Isso ainda seleciona qualquer elemento com uma classe .bar, mas agora tem o dobro da especificidade do Grupo B:

.bar.bar {}    /* a=0, b=2, c=0 */
.baz {}        /* a=0, b=1, c=0 */

!important e declarações de estilo embutido #

O sinalizador !important em uma declaração de estilo e estilos declarados pelo atributo style do HTML são considerados como tendo uma especificidade maior do que qualquer seletor. Se eles existirem, a declaração de estilo que eles afetarão anulará outras declarações, independentemente de sua especificidade. Isto é, a menos que você tenha mais de uma declaração que contenha um sinalizador !important para a mesma propriedade que se aplica ao mesmo elemento. Em seguida, as regras de especificidade normal serão aplicadas a essas propriedades em referência umas às outras.

Porque eles substituem completamente a especificidade, o uso de !important é desaprovado na maioria dos casos de uso. Deve-se usá-lo o mínimo possível. Para manter o código CSS eficiente e sustentável a longo prazo, é quase sempre melhor aumentar a especificidade do seletor ao redor do que usar !important.

Uma daquelas raras exceções onde !important não é desaprovado, é quando implementamos classes auxiliares genéricas como .hidden ou .background-yellow que devem sempre substituir uma ou mais propriedades onde quer que sejam encontradas. E mesmo assim, você precisa saber o que está fazendo. A última coisa que você quer, ao escrever CSS sustentável, é ter sinalizadores !important em todo o seu CSS.

Uma nota final #

Um equívoco comum sobre a especificidade do CSS é que os valores do Grupo A, B e c devem ser combinados entre si (a=1, b=5, c=1 => 151). Este não é o caso. Se este fosse o caso, ter 20 de um seletor de Grupo B ou c seria suficiente para substituir um único seletor de Grupo A ou B, respectivamente. Os três grupos devem ser considerados como níveis individuais de especificidade. A especificidade não pode ser representada por um único valor.

Ao criar sua folha de estilo CSS, você deve manter a menor especificidade possível. Se você precisar aumentar a especificidade um pouco mais para substituir outro método, aumente-a, mas o mais baixo possível para torná-la mais alta. Você não precisa ter um seletor como este:

body.page header.container nav div#main-nav li a {}

Isso torna as mudanças futuras mais difíceis e polui essa página css.

A !declaração importante #

A declaração !important é usada para substituir a especificidade usual em uma folha de estilo, dando uma prioridade mais alta a uma regra. Seu uso é: propriedade : valor !importante;

#mydiv {
    font-weight: bold !important;    /* This property won't be overridden
                                       by the rule below */
}

#outerdiv #mydiv {
    font-weight: normal;            /* #mydiv font-weight won't be set to normal
                                    even if it has a higher specificity because
                                    of the !important declaration above */
}

Evitar o uso de !important é fortemente recomendado (a menos que seja absolutamente necessário), porque irá perturbar o fluxo natural das regras CSS que podem trazer incerteza em sua folha de estilo. Também é importante notar que quando múltiplas declarações !important são aplicadas à mesma regra em um determinado elemento, aquela com maior especificidade será a ona aplicada.

Aqui estão alguns exemplos onde o uso da declaração !important pode ser justificado:

  • Se suas regras não devem ser substituídas por nenhum estilo embutido do elemento que está escrito dentro do atributo style do elemento html.
  • Para dar ao usuário mais controle sobre a acessibilidade da web, como aumentar ou diminuir o tamanho da fonte, substituindo o estilo do autor usando !important.
  • Para testar e depurar usando o elemento inspecionar.

Veja também:

  • W3C - 6 Atribuindo valores de propriedade, cascata e herança – 6.4.2 !regras importantes

Cascata #

Cascading e especificidade são usados ​​juntos para determinar o valor final de uma propriedade de estilo CSS. Eles também definem os mecanismos para resolver conflitos em conjuntos de regras CSS.

Ordem de carregamento CSS #

Os estilos são lidos das seguintes fontes, nesta ordem:

  1. Folha de estilo do User Agent (os estilos fornecidos pelo fornecedor do navegador)
  2. Folha de estilo do usuário (o estilo adicional que um usuário definiu em seu navegador)
  3. Folha de estilo do autor (Autor aqui significa o criador da página/site)
  • Talvez um ou mais arquivos .css
  • No elemento <style> do documento HTML
  1. Estilos embutidos (no atributo style em um elemento HTML)

O navegador procurará o(s) estilo(s) correspondente(s) ao renderizar um elemento.

Como os conflitos são resolvidos? #

Quando apenas um conjunto de regras CSS está tentando definir um estilo para um elemento, não há conflito e esse conjunto de regras é usado.

Quando vários conjuntos de regras são encontrados com configurações conflitantes, primeiro as regras de especificidade e, em seguida, as regras de cascata são usadas para determinar qual estilo usar.

Exemplo 1 - Regras de especificidade #

.mystyle { color: blue; }  /* specificity: 0, 0, 1, 0 */
div { color: red; }        /* specificity: 0, 0, 0, 1 */

<div class="mystyle">Hello World</div>

Qual será a cor do texto? (passe o mouse para ver a resposta)

! azul

Primeiro as regras de especificidade são aplicadas, e aquele com a maior especificidade “vence”.

Exemplo 2 - Regras de cascata com seletores idênticos #

Arquivo CSS externo

.class {
  background: #FFF;
}

Css interno (em arquivo HTML)

<style>
.class {
  background: #000;
}
<style>

Neste caso, onde você tem seletores idênticos, a cascata entra em ação e determina que o último carregado “vence”.

Exemplo 3 - Regras de cascata após regras de especificidade #

body > .mystyle { background-color: blue; }   /* specificity: 0, 0, 1, 1 */
.otherstyle > div { background-color: red; }  /* specificity: 0, 0, 1, 1 */

<body class="otherstyle">
  <div class="mystyle">Hello World</div>
</body>

Qual será a cor do fundo?

!vermelho

Depois de aplicar as regras de especificidade, ainda há um conflito entre azul e vermelho, portanto, as regras em cascata são aplicadas sobre as regras de especificidade. A cascata analisa a ordem de carregamento das regras, seja dentro do mesmo arquivo .css ou na coleção de fontes de estilo. O último carregado substitui os anteriores. Neste caso, a regra .otherstyle > div “ganha”.

Uma nota final #

  • A especificidade do seletor sempre tem precedência.
  • Enlaces de quebra de ordem de folha de estilo.
  • Estilos inline superam tudo.

Exemplo de especificidade mais complexa #

div {
    font-size: 7px;
    border: 3px dotted pink;
    background-color: yellow;
    color: purple;
}

body.mystyle > div.myotherstyle {
    font-size: 11px;
    background-color: green;
}

#elmnt1 {
    font-size: 24px;
    border-color: red;
}

.mystyle .myotherstyle {
    font-size: 16px;
    background-color: black;
    color: red;
}

<body class="mystyle">
    <div id="elmnt1" class="myotherstyle">
        Hello, world!
    </div>
</body>

Quais bordas, cores e tamanhos de fonte o texto terá?

tamanho da fonte:

! font-size: 24;, uma vez que o conjunto de regras #elmnt1 tem a maior especificidade para o <div> em questão, todas as propriedades aqui são definidas.

fronteira:

! borda: 3px pontilhado vermelho;. A cor da borda red é retirada do conjunto de regras #elmnt1, pois tem a maior especificidade. As outras propriedades da borda, espessura da borda e estilo da borda são do conjunto de regras div.

cor de fundo:

! cor de fundo: verde;. A background-color é definida nos conjuntos de regras div, body.mystyle > div.myotherstyle e .mystyle .myotherstyle. As especificidades são (0, 0, 1) vs. (0, 2, 2) vs. (0, 2, 0), então o do meio “ganha”.

cor:

! cor: vermelho;. A cor é definida nos conjuntos de regras div e .mystyle .myotherstyle. Este último tem a maior especificidade de (0, 2, 0) e “ganha”.
