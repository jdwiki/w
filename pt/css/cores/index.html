Sintaxe #

  • cor: #rgb
  • cor: #rrggbb
  • cor: rgb[a](<red>, <green>, <blue>[, <alpha>])
  • cor: hsl[a](<hue>, <saturation%>, <lightness%>[, <alpha>])
  • color: [colorkeyword](https://www.wikiod.com/pt/css/cores#Palavras-chave de cor) /* verde, azul, amarelo, laranja, vermelho, ..etc */

cor atual #

currentColor retorna o valor de cor calculado do elemento atual.

Use no mesmo elemento #

Aqui currentColor é avaliado como vermelho já que a propriedade color está definida como red:

div {
   color: red;     
   border: 5px solid currentColor;
   box-shadow: 0 0 5px currentColor;
}

Nesse caso, especificar currentColor para a borda provavelmente é redundante, pois sua omissão deve produzir resultados idênticos. Use currentColor apenas dentro da propriedade border dentro do mesmo elemento se ele for substituído de outra forma devido a um [mais específico](https://www.wikiod.com/pt/css/cascata-e-especificidade#Calculando a especificidade do seletor -specificity#t=201607221033325129018) seletor.

Como é a cor calculada, a borda será verde no exemplo a seguir devido à segunda regra substituir a primeira:

div {
   color: blue;
   border: 3px solid currentColor; 
   color: green;
}

Herdado do elemento pai #

A cor do pai é herdada, aqui currentColor é avaliado como ‘blue’, tornando a cor da borda do elemento filho azul.

.parent-class {
    color: blue;
}

.parent-class .child-class {
    border-color: currentColor;
}

currentColor também pode ser usado por outras regras que normalmente não herdariam da propriedade color, como background-color. O exemplo abaixo mostra os filhos usando a cor definida no pai como plano de fundo:

.parent-class {
    color: blue;
}

.parent-class .child-class {
    background-color: currentColor;
}

Palavras-chave de cores #

A maioria dos navegadores suporta o uso de palavras-chave de cores para especificar uma cor. Por exemplo, para definir a cor de um elemento como azul, use a palavra-chave blue:

.some-class {
    color: blue;
}

Palavras-chave CSS não diferenciam maiúsculas de minúsculas — blue, Blue e BLUE resultarão em #0000FF.

Valor hexadecimal #

Fundo #

As cores CSS também podem ser representadas como um trio hexadecimal, onde os membros representam os componentes vermelho, verde e azul de uma cor. Cada um desses valores representa um número no intervalo de 00 a FF, ou 0 a 255 em notação decimal. Valores hexadecimais maiúsculos e/ou minúsculos podem ser usados ​​(ou seja, #3fc = #3FC = #33ffCC). O navegador interpreta #369 como #336699. Se isso não é o que você pretendia, mas queria #306090, você precisa especificar isso explicitamente.

O número total de cores que podem ser representadas com notação hexadecimal é 256 ^ 3 ou 16.777.216.

Sintaxe #

color: #rrggbb;
color: #rgb
Valor Descrição
rr 00 - FF para a quantidade de vermelho
gg 00 - FF para a quantidade de verde
bb 00 - FF para a quantidade de azul 
.some-class {
    /* This is equivalent to using the color keyword 'blue' */
    color: #0000FF;
}

.also-blue {
    /* If you want to specify each range value with a single number, you can!
       This is equivalent to '#0000FF' (and 'blue') */
    color: #00F;
}

Notação hexadecimal é usada para especificar valores de cor no formato de cor RGB, de acordo com os [valores numéricos de cores do W3C](https://www.w3. org/TR/css3-color/#numerical).

Existem muitas ferramentas disponíveis na Internet para pesquisar valores de cores hexadecimais (ou simplesmente hexadecimais).

Pesquise por “paleta de cores hexadecimais” ou “seletor de cores hexadecimais” com seu navegador favorito para encontrar várias opções!

Os valores hexadecimais sempre começam com um sinal de sustenido (#), têm até seis “dígitos” e não diferenciam maiúsculas de minúsculas: ou seja, eles não se importam com a capitalização. #FFC125 e #ffc125 são da mesma cor.

Notação rgb() #

RGB é um modelo de cores aditivas que representa cores como misturas de luz vermelha, verde e azul. Em essência, a representação RGB é o equivalente decimal da Notação Hexadecimal. Em hexadecimal cada número varia de 00-FF que é equivalente a 0-255 em decimal e 0%-100% em porcentagens.

.some-class {
    /* Scalar RGB, equivalent to 'blue'*/
    color: rgb(0, 0, 255);
}

.also-blue {
    /* Percentile RGB values*/
    color: rgb(0%, 0%, 100%);
}

Sintaxe #

rgb(<red>, <green>, <blue>)
Valor Descrição
<vermelho> um inteiro de 0 a 255 ou porcentagem de 0 a 100%
<verde> um inteiro de 0 a 255 ou porcentagem de 0 a 100%
<azul> um inteiro de 0 a 255 ou porcentagem de 0 a 100%

Notação rgba() #

Semelhante à notação [rgb()](https://www.wikiod.com/pt/css/cores#Notação rgb()), mas com um valor alfa (opacidade) adicional.

.red {
    /* Opaque red */
    color: rgba(255, 0, 0, 1);
}

.red-50p {
    /* Half-translucent red. */
    color: rgba(255, 0, 0, .5);
}

Sintaxe #

rgba(<red>, <green>, <blue>, <alpha>);
Valor Descrição
<vermelho> um inteiro de 0 a 255 ou porcentagem de 0 a 100%
<verde> um inteiro de 0 a 255 ou porcentagem de 0 a 100%
<azul> um inteiro de 0 a 255 ou porcentagem de 0 a 100%
<alfa> um número de 0 a 1, onde 0,0 é totalmente transparente e 1,0 é totalmente opaco

Notação hsl() #

HSL significa matiz (“qual cor”), saturação (“quantidade de cor”) e claridade (“quantidade de branco”).

A matiz é representada como um ângulo de 0° a 360° (sem unidades), enquanto a saturação e a luminosidade são representadas como porcentagens.

p {
    color: hsl(240, 100%, 50%); /* Blue */
}

Sintaxe #

color: hsl(<hue>, <saturation>%, <lightness>%);
Valor Descrição
<matiz> especificado em graus ao redor da roda de cores (sem unidades), onde 0° é vermelho, 60° é amarelo, 120° é verde, 180° é ciano, 240° é azul, 300° é magenta e 360° é vermelho
<saturação> especificado em porcentagem em que 0% é totalmente dessaturado (escala de cinza) e 100% é totalmente saturado (com cores vivas)
<leveza> especificado em porcentagem em que 0% é totalmente preto e 100% é totalmente branco

Notas #

  • Uma saturação de 0% sempre produz uma cor em tons de cinza; alterar a tonalidade não tem efeito.
  • Uma luminosidade de 0% sempre produz preto e 100% sempre produz branco; alterar o matiz ou a saturação não tem efeito.

Notação hsla() #

Semelhante à notação [hsl()](https://www.wikiod.com/pt/css/cores#hsl() Notação), mas com um valor alfa (opacidade) adicionado.

hsla(240, 100%, 50%, 0)     /* transparent */
hsla(240, 100%, 50%, 0.5)   /* half-translucent blue */
hsla(240, 100%, 50%, 1)     /* fully opaque blue */

Sintaxe #

hsla(<hue>, <saturation>%, <lightness>%, <alpha>);
Valor Descrição
<matiz> especificado em graus ao redor da roda de cores (sem unidades), onde 0° é vermelho, 60° é amarelo, 120° é verde, 180° é ciano, 240° é azul, 300° é magenta e 360° é vermelho
<saturação> porcentagem em que 0% é totalmente dessaturado (escala de cinza) e 100% é totalmente saturado (com cores vivas)
<leveza> percentual em que 0% é totalmente preto e 100% é totalmente branco
<alfa> um número de 0 a 1 onde 0 é totalmente transparente e 1 é totalmente opaco
