Sintaxe #

  • @media [não|somente] tipo de mídia e (recurso de mídia) { /* regras CSS a serem aplicadas */ }

Parâmetros #

Parâmetro Detalhes
tipo de mídia (Opcional) Este é o tipo de mídia. Pode ser qualquer coisa no intervalo de all para screen.
não (Opcional) Não aplica o CSS para esse tipo de mídia específico e se aplica a todo o resto.
recurso de mídia Lógica para identificar caso de uso para CSS. Opções descritas abaixo.
Recurso de mídia Detalhes
proporção de aspecto Descreve a proporção da área de exibição de destino do dispositivo de saída.
cor Indica o número de bits por componente de cor do dispositivo de saída. Se o dispositivo não for um dispositivo colorido, esse valor será zero.
índice de cores Indica o número de entradas na tabela de pesquisa de cores para o dispositivo de saída.
grade Determina se o dispositivo de saída é um dispositivo de grade ou um dispositivo de bitmap.
altura O recurso de mídia de altura descreve a altura da superfície de renderização do dispositivo de saída.
max-width O CSS não será aplicado em uma largura de tela maior que a especificada.
min-largura O CSS não será aplicado em uma largura de tela menor que a especificada.
max-height O CSS não será aplicado em uma altura de tela maior que a especificada.
min-altura O CSS não será aplicado em uma altura de tela menor que a especificada.
monocromático Indica o número de bits por pixel em um dispositivo monocromático (escala de cinza).
orientação CSS só será exibido se o dispositivo estiver usando a orientação especificada. Veja as observações para mais detalhes.
resolução Indica a resolução (densidade de pixels) do dispositivo de saída.
digitalizar Descreve o processo de digitalização de dispositivos de saída de televisão.
largura O recurso de mídia de largura descreve a largura da superfície de renderização do dispositivo de saída (como a largura da janela do documento ou a largura da caixa de página em uma impressora).
Recursos obsoletos Detalhes
proporção de aspecto do dispositivo Descontinuado CSS será exibido apenas em dispositivos cuja proporção altura/largura corresponda à proporção especificada. Este é um recurso obsoleto e não é garantido que funcione.
max-device-width Descontinuado O mesmo que max-width, mas mede a largura física da tela, em vez da largura de exibição do navegador.
min-device-width Descontinuado O mesmo que min-width, mas mede a largura física da tela, em vez da largura de exibição do navegador.
max-dispositivo-altura Descontinuado O mesmo que max-height, mas mede a largura física da tela, em vez da largura de exibição do navegador.
min-dispositivo-altura Descontinuado O mesmo que min-height, mas mede a largura física da tela, em vez da largura de exibição do navegador.

As consultas de mídia são suportadas em todos os navegadores modernos, incluindo Chrome, Firefox, Opera e Internet Explorer 9 e superior.

É importante observar que o recurso de mídia orientação não se limita a dispositivos móveis. É baseado na largura e na altura da janela de visualização (não na janela ou nos dispositivos).

Modo Paisagem é quando a largura da janela de visualização é maior que a altura da janela de visualização.

Modo Retrato é quando a altura da janela de visualização é maior que a largura da janela de visualização.

Isso geralmente se traduz em um monitor de desktop no modo paisagem, mas às vezes pode ser retrato.

Na maioria dos casos, os dispositivos móveis relatam sua resolução e não seu tamanho real de pixel, que pode diferir devido à densidade de pixel. Para forçá-los a informar seu tamanho real de pixel, adicione o seguinte dentro de sua tag head:

<meta name="viewport" content="width=device-width, initial-scale=1">

Exemplo básico #

@media screen and (min-width: 720px) {
    body {
        background-color: skyblue;
    }
}

A consulta de mídia acima especifica duas condições:

  1. A página deve ser visualizada em uma tela normal (não uma página impressa, projetor, etc).
  2. A largura da porta de visualização do usuário deve ser de pelo menos 720 pixels.

Se essas condições forem atendidas, os estilos dentro da consulta de mídia estarão ativos e a cor de fundo da página será azul celeste.

As consultas de mídia são aplicadas dinamicamente. Se no carregamento da página as condições especificadas na consulta de mídia forem atendidas, o CSS será aplicado, mas será imediatamente desabilitado caso as condições deixem de ser atendidas. Por outro lado, se as condições não forem atendidas inicialmente, o CSS não será aplicado até que as condições especificadas sejam atendidas.

Em nosso exemplo, se a largura da porta de visualização do usuário for inicialmente maior que 720 pixels, mas o usuário diminuir a largura do navegador, a cor de fundo deixará de ser azul celeste assim que o usuário redimensionar a porta de visualização para menos de 720 pixels em largura.

tipo de mídia #

As consultas de mídia têm um parâmetro opcional mediatype. Este parâmetro é colocado diretamente após a declaração @media (@media mediatype), por exemplo:

@media print {
    html {
        background-color: white;
    }
}

O código CSS acima dará ao elemento DOM HTML uma cor de fundo branca ao ser impresso.

O parâmetro mediatype tem um prefixo opcional not ou only que aplicará os estilos a tudo, exceto ao tipo de mídia especificado ou apenas ao tipo de mídia especificado, respectivamente. Por exemplo, o exemplo de código a seguir aplicará o estilo a todos os tipos de mídia, exceto print.

@media not print {
    html {
        background-color: green;
    }
}

E da mesma forma, para mostrar apenas na tela, isso pode ser usado:

@media only screen {
    .fadeInEffects {
        display: block;
    }
}

A lista de mediatype pode ser melhor compreendida com a seguinte tabela:

Tipo de mídia Descrição
todos Aplicar a todos os dispositivos
tela Computadores padrão
imprimir Impressoras em geral. Usado para estilizar versões impressas de sites
portátil PDA’s, telemóveis e dispositivos portáteis com ecrã pequeno
projeção Para apresentação projetada, por exemplo, projetores
aural Sistemas de Fala
braile Dispositivos tácteis em Braille
em relevo Impressoras braille paginadas
tv Aparelhos tipo televisão
tty Dispositivos com uma grade de caracteres de passo fixo. Terminais, portáteis.

Terminologia e estrutura #

Consultas de mídia permitem aplicar regras CSS com base no tipo de dispositivo/mídia (por exemplo, tela, impresso ou portátil) chamado tipo de mídia. Os aspectos adicionais do dispositivo são descritos com recursos de mídia como a disponibilidade de cores ou dimensões da janela de visualização.

Estrutura geral de uma consulta de mídia #

@media [...] {
    /* One or more CSS rules to apply when the query is satisfied */
}

Uma consulta de mídia contendo um tipo de mídia #

@media print {
    /* One or more CSS rules to apply when the query is satisfied */
}

Uma consulta de mídia contendo um tipo de mídia e um recurso de mídia #

@media screen and (max-width: 600px) {
    /* One or more CSS rules to apply when the query is satisfied */
}

Uma consulta de mídia contendo um recurso de mídia (e um tipo de mídia implícito de “todos”) #

@media (orientation: portrait) {        
    /* One or more CSS rules to apply when the query is satisfied */
}

Largura x janela de visualização #

Quando estamos usando “largura” com consultas de mídia, é importante definir a metatag corretamente. A metatag básica se parece com isso e precisa ser colocada dentro da tag <head>.

<meta name="viewport" content="width=device-width,initial-scale=1">

Por que isso é importante?

Com base na definição do MDN, “largura” é

O recurso de mídia de largura descreve a largura da superfície de renderização do dispositivo de saída (como a largura da janela do documento ou a largura da caixa de página em uma impressora).

O que isso significa?

A porta de visualização é a largura do próprio dispositivo. Se a resolução da sua tela diz que a resolução é 1280 x 720, a largura da sua porta de visualização é “1280px”.

Mais frequentemente, muitos dispositivos alocam diferentes quantidades de pixels para exibir um pixel. Por exemplo, o iPhone 6 Plus tem resolução de 1242 x 2208. Mas a largura real da janela de visualização e a altura da janela de visualização são 414 x 736. Isso significa que 3 pixels são usados ​​para criar 1 pixel.

Mas se você não definiu a tag meta corretamente, ela tentará mostrar sua página da web com sua resolução nativa, o que resulta em uma visualização com zoom reduzido (textos e imagens menores).

Consultas de mídia para telas de retina e não retina #

Embora isso funcione apenas para navegadores baseados em WebKit, isso é útil:

/* ----------- Non-Retina Screens ----------- */
@media screen 
  and (min-width: 1200px) 
  and (max-width: 1600px) 
  and (-webkit-min-device-pixel-ratio: 1) { 
}

/* ----------- Retina Screens ----------- */
@media screen 
  and (min-width: 1200px) 
  and (max-width: 1600px) 
  and (-webkit-min-device-pixel-ratio: 2)
  and (min-resolution: 192dpi) { 
}

Informações básicas

Existem dois tipos de pixels na tela. Um são os pixels lógicos e o outro são os pixels físicos. Principalmente, os pixels físicos sempre permanecem os mesmos, porque é o mesmo para todos os dispositivos de exibição. Os pixels lógicos mudam com base na resolução dos dispositivos para exibir pixels de maior qualidade. A proporção de pixels do dispositivo é a proporção entre pixels físicos e pixels lógicos. Por exemplo, o MacBook Pro Retina, iPhone 4 e superior relatam uma proporção de pixels do dispositivo de 2, porque a resolução linear física é o dobro da resolução lógica.

A razão pela qual isso funciona apenas com navegadores baseados em WebKit é por causa de:

  • O prefixo do fornecedor -webkit- antes da regra.
  • Isso não foi implementado em mecanismos que não sejam WebKit e Blink.

Como usar consultas de mídia para segmentar diferentes tamanhos de tela #

Muitas vezes, o web design responsivo envolve media queries, que são blocos CSS que só são executados se uma condição for satisfeita. Isso é útil para web design responsivo porque você pode usar consultas de mídia para especificar diferentes estilos CSS para a versão móvel do seu site versus a versão para desktop.

@media only screen and (min-width: 300px) and (max-width: 767px) {
    .site-title {
        font-size: 80%;
    }

    /* Styles in this block are only applied if the screen size is atleast 300px wide, but no more than 767px */
}

@media only screen and (min-width: 768px) and (max-width: 1023px) {
    .site-title {
        font-size: 90%;
    }

    /* Styles in this block are only applied if the screen size is atleast 768px wide, but no more than 1023px */
}

@media only screen and (min-width: 1024px) {
    .site-title {
        font-size: 120%;
    }

    /* Styles in this block are only applied if the screen size is over 1024px wide. */
}

Use na tag do link #

<link rel="stylesheet" media="min-width: 600px" href="example.css" />

Esta folha de estilo ainda é baixada, mas é aplicada apenas em dispositivos com largura de tela maior que 600px.

Consultas de mídia e IE8 #

Consultas de mídia não são compatíveis com o IE8 e versões anteriores.

Uma solução baseada em Javascript #

Para adicionar suporte ao IE8, você pode usar uma das várias soluções JS. Por exemplo, Respond pode ser adicionado para adicionar suporte a consulta de mídia apenas para IE8 com o seguinte código:

<!--[if lt IE 9]>
<script 
   src="respond.min.js">
</script>
<![endif]-->

CSS Mediaqueries é outra biblioteca que faz a mesma coisa. O código para adicionar essa biblioteca ao seu HTML seria idêntico:

<!--[if lt IE 9]>
<script 
   src="css3-mediaqueries.js">
</script>
<![endif]-->

A alternativa #

Se você não gosta de uma solução baseada em JS, você também deve considerar adicionar uma folha de estilo somente IE<9 onde você ajusta seu estilo específico para IE<9. Para isso, você deve adicionar o seguinte HTML ao seu código:

<!--[if lt IE 9]>
<link rel="stylesheet" type="text/css" media="all" href="style-ielt9.css"/>
<![endif]-->

Observação :

Tecnicamente, é mais uma alternativa: usar CSS hacks para direcionar o IE<9. Ele tem o mesmo impacto que uma folha de estilo somente do IE<9, mas você não precisa de uma folha de estilo separada para isso. Eu não recomendo essa opção, no entanto, pois eles produzem código CSS inválido (que é apenas uma das várias razões pelas quais o uso de hacks CSS é geralmente desaprovado hoje).
