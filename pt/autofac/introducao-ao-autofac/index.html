Instalando o Autofac #

Para usar o Autofac em seu projeto, tudo o que você precisa fazer é instalar o Autofac do NuGet Package Manager. Abra a solução na qual deseja usar o Autofac e selecione Manager NuGet Packages for Solution... acessando:

Tools -> NuGet Package Manager -> Manager NuGet Packages for Solution...

Na guia NuGet-Solution, digite “Autofac” na caixa de pesquisa. Verifique se você está na seção “Procurar”. Instale a primeira opção conforme mostrado na imagem abaixo (observe as regiões marcadas na imagem):

A instalação por meio do NuGet adicionará automaticamente o Autofac nas Referências dos projetos que foram selecionados durante a instalação.

Dê uma olhada na documentação oficial.

Configurando o Autofac #

Este exemplo mostrará como começar com Inverion of Control usando Autofac com um projeto relativamente simples, seguindo de perto a documentação oficial de introdução.

  1. Crie um aplicativo de console em Arquivo -> Novo -> Projeto -> Aplicativo de console

  2. Instale o Autofac para este projeto. Você pode dar uma olhada aqui [Instalando o Autofac][3]

  3. Adicione 2 interfaces e 2 classes, com os seguintes nomes:

      Interfaces  |  Classes
--------------------------
IOutput       | ConsoleOutput (implementing IOutput)
IDateWriter   | TodayWriter (implementing IDateWriter)

Para simplificar, as instruções using e os namespaces não são mostrados.

IOuput.cs #

public interface IOutput
{
    void Write(string content);
}

ConsoleOutput.cs #

public class ConsoleOutput : IOutput
{
    public void Write(string content)
    {
        Console.WriteLine(content);
    }
}

IDateWriter.cs #

public interface IDateWriter
{
    void WriteDate();
}

TodayWriter.cs #

public class TodayWriter : IDateWriter
{
    private IOutput _output;

    public TodayWriter(IOutput output)
    {
        _output = output;
    }

    public void WriteDate()
    {
        _output.Write(DateTime.Today.ToShortDateString());
    }
}

Até agora, o código tem sido simples. Vamos para a parte onde ocorre a injeção automática de dependências, que obviamente está sendo feita pela Autofac!

Substitua a classe Program no arquivo Program.cs por este código (a classe Program é criada automaticamente pelo Visual Studio na criação do projeto. Se não existir, vá em frente e crie uma):

class Program
{
    private static IContainer Container { get; set; }

    static void Main(string[] args)
    {
        var builder = new ContainerBuilder();
        builder.RegisterType<ConsoleOutput>().As<IOutput>();
        builder.RegisterType<TodayWriter>().As<IDateWriter>();
        Container = builder.Build();

        WriteDate();
    }

    public static void WriteDate()
    {
        using (var scope = Container.BeginLifetimeScope())
        {
            var writer = scope.Resolve<IDateWriter>();
            writer.WriteDate();
        }
    }
}

Quando executado, a saída deve ser a data atual no console. Você usou com sucesso o Autofac em seu projeto para injetar dependências automaticamente.

Aqui está o que está acontecendo sob o capô:

  1. Na inicialização do aplicativo, estamos criando um ContainerBuilder e registrando nossos Componentes com ele. Um componente em termos simples é um tipo .NET que implementa uma interface e, portanto, expõe alguns serviços. Leia [Serviços vs. Componentes][4].
  2. Em seguida, registramos nossos componentes (classes) com os serviços (interfaces) que eles expõem. Quando registrado, o Autofac sabe qual instância de uma classe criar quando uma interface deve ser resolvida.
  3. Finalmente, quando executamos o programa:
  • O método WriteDate() (em Main()) pede ao Autofac um IDateWriter.
  • Autofac vê que IDateWriter mapeia para TodayWriter então começa a criar um TodayWriter.
  • Autofac vê que o TodayWriter precisa de um IOutput em seu construtor.
  • Autofac vê que IOutput mapeia para ConsoleOutput então cria uma nova instância ConsoleOutput.
  • Autofac usa a nova instância ConsoleOutput para concluir a construção do TodayWriter.
  • Autofac retorna o TodayWriter totalmente construído para WriteDate() consumir.
