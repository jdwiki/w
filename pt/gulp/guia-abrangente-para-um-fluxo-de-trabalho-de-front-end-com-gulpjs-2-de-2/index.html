Configurando a sincronização do navegador e configurando observadores para estilo e script #

NOTA #

Esta página ilustra o uso de plugins gulp como sincronização de navegador, observe o gole e execute a sequência e continue discutindo gulp-workflow-automation de onde paramos Gulpjs-workflow-automation-1 de 2. Caso você tenha chegado aqui, considere passando por esse post primeiro.

  • Tarefa Padrão
  • Tarefa de cão de guarda - para construir continuamente seus ativos prontos para implantação em tempo real, sempre que qualquer imagem | JS | CSS muda ao longo do desenvolvimento.

Vamos começar com a sincronização do navegador.

Tarefa de cão de guarda Gulp #

Vamos começar com a tarefa de cão de guarda.

O objetivo é observar as mudanças que você faz durante o desenvolvimento. Qualquer alteração deve acionar a tarefa gulp correspondente.

Além disso, precisamos de uma funcionalidade que sincronize suas alterações no navegador.

Sincronização do navegador

Então, precisamos instalar o Browser Sync.

bash $ npm install browser-sync --save-dev

Com essa premissa, vamos abrir nosso gulpfile.js e adicionar a funcionalidade watch. Vamos exigir a sincronização do navegador e definir algumas variáveis ​​para usar sua funcionalidade.

Na parte superior do gulpfile , adicione o trecho abaixo. Coloque-o logo abaixo das declarações de compactação de imagem.

igual a:

//Browser-sync

var sync = require('browser-sync').create();
var reload = sync.reload;

Ter a sincronização do navegador sincronizando seu desenvolvimento com o navegador é uma configuração simples. Vamos criar uma tarefa chamada watchdog.

igual a:

$.task('watchdog', function() {


})

Agora, se navegarmos pelas opções de sincronização do navegador aqui e procurarmos a configuração do servidor, podemos ver como é fácil.

Nós só precisamos colocar o abaixo dentro de nossa tarefa watchdog

Snippet - 1 - dentro do clichê do watchdog #

/*
 Initiate Browser sync
 @documentation - https://www.browsersync.io/docs/options/
 */
 sync.init({
  server: {
    baseDir: "./"
 },
 port: 8000 //change it as required
 });

Insira o acima dentro do seu clichê de watchdog acima.

O próximo trecho é definir um observador para estilos, com o objetivo de reprocessar arquivos css alterados ou novos e acionar um recarregamento do navegador automaticamente.

snippet - 2 - dentro do clichê do watchdog #

$.watch(['css/**/*', 'fonts/google/**/*.css'], reload).on('change', function(event) {
console.log(event.type + ':' + event.path)
if (event.type === 'deleted') {
uncache('styles', event.path);
$$.remember.forget('auto-prefixed-stylesheets', event.path);
}
sequence('optimizeStyles')
});

Insira o acima dentro do seu clichê de watchdog acima.

Então, estamos monitorando " [fonts/google/**/*.css , /**/*.css ]" ou seja,

todos os arquivos css em css/ todos os arquivos css em fonts/google/ Quando algo muda, ou um novo arquivo é adicionado, ele aciona o método reload, que é definido na parte superior do nosso gulpfile, na declaração do browsersync.

Nota: Você pode notar que temos um manipulador de eventos .on anexado ao observador.

$.watch(['css/**/*', 'fonts/google/**/*.css'], reload).on('change', function(event)

Basicamente, qualquer coisa CUD(Create| Update | Delete) aciona a função de recarregamento e passa um objeto de evento como um parâmetro para o função de retorno de chamada.

O retorno de chamada é uma função vital, onde podemos realizar operações como desarmar o cache na exclusão de ativos. Agora, o objeto de evento possui parâmetros como

  • caminho
  • tipo - Criar/Atualizar/Excluir

Se um ativo for excluído, precisamos ter certeza de que os caches que construímos em nossas funções de minificação anteriores, via gulp-cached e gulp-remember , precisam de atualização.

estamos lidando com isso no snippet abaixo, que está dentro do callback on change.

if (event.type === 'deleted') { 
uncache('styles', event.path);
$$.remember.forget('auto-prefixed-stylesheets', event.path);  
}

Observação #

$ - >alias para gulp

$$ - > alias para gulp-load-plugins

você também pode notar que eu tenho uma sequence('optimizeStyles'); depois que eu escrevi a invocação uncache

O método de sequência, garante, o método síncrono é executado em um javascript assíncrono por padrão.

instalar é simples

FAZ

bash $ npm install run-sequence

então , declare-o em seu gulpfile logo abaixo da declaração de sincronização do navegador.

var sequence = require('run-sequence');

Então, com esse entendimento, o observador de scripts é fácil de escrever. apenas globos diferentes!

Então, adicione este trecho abaixo do observador de estilo dentro do clichê do watchdog.

Snippet - 3 - dentro do clichê da tarefa do Watchdog #

/*
on addition or change or deletion of a file in the watched directories
the change event is triggered. An event object with properties like
path,
event-type
is available for perusal passed to the callback

*/
$.watch('js/**/*', reload).on('change', function(event) {
console.log(event.type + ':' + event.path)
if (event.type === 'deleted') {
uncache('scripts', event.path);
$$.remember.forget('linted-scripts', event.path);
}
sequence('optimizeScripts');
});

/*
Deletes a cache entry
*/
var uncache = function(cacheName, cacheKey) {
        var cache = $$.cached;
        if (cache.caches[cacheName] && cache.caches[cacheName][cacheKey])
            return delete cache.caches[cacheName][cacheKey];
        return false;
    }

Definindo uma tarefa padrão #

Então agora, vamos terminar o código gulpfile , definindo uma tarefa padrão.

a tarefa padrão é aquela que é executada, quando você apenas diz

gulp

em um prompt de comando na raiz do seu projeto.

$.task('default', ['generateResponsiveImages'], function() {
 $.start('watchdog');
 console.log('Starting Incremental Build');
});
