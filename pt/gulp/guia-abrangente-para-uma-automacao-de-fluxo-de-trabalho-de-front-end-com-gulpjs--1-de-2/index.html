Carregando todos os plugins do Package.JSON #

Supondo que você tenha uma noção de como instalar o gulp, vamos nos aprofundar na exigência de todas as dependências gulp- de package.json na pasta raiz de seus projetos.

Caso você ainda não tenha um arquivo gulp, por favor crie um arquivo vazio com o nome

gulpfile.js

Primeiro, exigimos gulp. igual a:

var $ = require('gulp');

Em seguida, vamos carregar todos os nossos plugins, em nosso gulpfile, pelo trecho abaixo

Observação #

Leia os comentários em todos os snippets que incluiremos nesta leitura, eles fornecem mais informações sobre todas as funcionalidades

/*
require gulp-load-plugins, and call it
gulp-load-plugins will require individually,
all the plugins installed in package.json at the root directory
these required plugins, are lazy loaded
that is, gulp-load-plugins will only require them when they are referenced in this file
plugins are available for perusal, via camelcased names, minus gulp
eg: gulp-clean-css is accessed by $$.cleanCss
*/
var $$ = require('gulp-load-plugins')();

NOTA #

Ao longo dos muitos exemplos que teremos no artigo, alias

  1. engula como $ e
  2. gulp-load-plugins como $$

puramente para facilitar a digitação.

Instalando plugins para imagens responsivas|Css Minification|Js minification #

Nosso objetivo, é

  1. Faça com que suas imagens se ajustem às larguras e dimensionem adequadamente, gerando uma bateria de imagens de larguras variadas e minimizando-as
  2. Lint seu Javascript
  3. Minimize seus ativos - JS/CSS/HTML, permitindo que você hospede um código mais leve que o mais leve
  4. Observe os arquivos css/js/image para alterações e reconstrua as otimizações
  5. Sincronizar suas alterações durante o desenvolvimento em um navegador que atende ao seu site.

Precisamos de vários plugins, então vamos instalar todos eles. Por favor, execute tudo isso na pasta raiz do projeto.

bash $ npm install --save-dev imagemin-pngquant

Anatomia de uma função gulp #

[Function <name>] (glob) {

$.src(glob)

.pipe([plugin 1])
.pipe([plugin 2])
.
.
.
.pipe([plugin n])
.pipe( $.dest(<destination-name>)

}

Observação #

pipe é um método que transmite todos os arquivos correspondentes à entrada glob , para nossos plugins ( minifyhtml neste caso).

É simples imaginar assim:

$.src é o que constrói o fluxo e canaliza cada indivíduo arquivo que corresponde ao glob para baixo para cada plug-in no pipeline.Each plugin para o qual o arquivo é passado, modifica seu conteúdo apenas na memória até que $.dest seja alcançado, que então atualiza/cria arquivos transmitidos por $.src

Onde ,

$ - > gole #

$$ - > gulp-load-plugins #

Otimização e Minificação de Ativos #

Portanto, antes de escrever as funções do otimizador, precisamos instalar alguns plugins de cache.

bash $ npm install --save-dev gulp-cached
bash $ npm install --save-dev gulp-remember

Você pode se perguntar por que dois caches eh!. gulp-cached , passa apenas conteúdo modificado ou novo no pipeline para outros plugins. Portanto, como queremos que os arquivos sem alterações sejam usados ​​para concatenar em um único arquivo por ativo ( css | js ), precisamos de gulp-remember além de gulp-cached

Primeiro, usamos gulp-cached para criar uma lista de arquivos que foram alterados

Em segundo lugar, precisamos gulp-remember para manter um controle de todos os arquivos que são passados ​​por essa lista na memória.

Primeira execução: Nenhum arquivo é armazenado em cache, então o gulp-cached passará todos para gulp-remember

Execuções subsequentes: Somente arquivos modificados ou novos são canalizados por gulp-cached. Como o conteúdo do arquivo atual foi alterado, o gulp-remember atualiza seu cache.

Legal, vamos escrever nosso primeiro otimizador

Otimizador de estilo #


// Goal

/*

1. cache existing files on the first run
2. each file , 
       a. Is autoprefixed with cross browser compatible prefixes for any css property ( justify-content for e.g)
       b. Is concatenated into app.css
3. app.css is minified
4. On subsequent runs , the above process is implemented on file modifications/additions only

*/

/*
*@src - input a glob pattern - a string eg 'images/**/*' or '**/*.css' or, an array eg ['glob1','glob2']
*/
var optimizeStyles = function(src) {

return $.src(src).
pipe($$.cached('styles')).
pipe($$.autoprefixer({
browsers: ['last 2 versions']
})).
pipe($$.remember('auto-prefixed-stylesheets')).
pipe($$.concat('app.css')).
pipe($.dest('build/css')).
pipe($$.cleanCss()).
pipe($$.rename({
suffix: '.min'
})).
pipe($.dest('build/css'))
}

Observação #

$ - > gole

$$ - > gulp-load-plugins

$.src - > cria fluxos de arquivos correspondentes ao glob passado como src

$.dest - > salva o arquivo manipulado no caminho especificado

Otimizador de script #


// Goal

/*

1. cache existing files on the first run
2. each file , 
       a. Is linted with jshint 
       b. Is concatenated into app.js
3. app.js is minified
4. On subsequent runs , the above process is implemented on file modifications/additions only

*/

/*
*@src - input a glob pattern - a string eg 'js/**/*' or '**/*.js' or, an array eg ['glob1','glob2']
*/

var optimizeScripts = function(src) {

    return $.src(src).
    pipe($$.cached('scripts')).
    pipe($$.jshint()).
    pipe($$.jshint.reporter('default')).
    pipe($$.jshint.reporter('fail')).
    pipe($$.remember('linted-scripts')).
    pipe($$.concat('app.js')).
    pipe($.dest('build/js')).
    pipe($$.uglify()).
    pipe($$.rename({
        suffix: '.min'
    })).
    pipe($.dest('build/js'))


}

Observação #

$ - > gole

$$ - > gulp-load-plugins

$.src - > cria fluxos de arquivos correspondentes ao glob passado como src

$.dest - > salva o arquivo manipulado no caminho especificado

Gerar imagens responsivas #

Passemos agora ao processamento de imagens. Então, o objetivo é ter uma variedade de tamanhos para cada imagem que você vai servir.

Por quê?

Bem , para entender por que precisamos de uma bateria de imagens com larguras variadas , precisamos refletir sobre o fato de que provavelmente existem zilhões de dispositivos com resoluções variadas. Precisamos de uma imagem para dimensionar sem muita pixelização. Ao mesmo tempo, precisamos melhorar os tempos de carregamento das páginas, baixando apenas o uma imagem , que se ajusta à largura pela qual está contida , e também está com a menor dimensão possível, para isso. Existem blogs acadêmicos como a que Eric Portis escreveu , que destaca a ineficácia de apenas consultas de mídia e serve como um guia abrangente para entender conceitos como srcsets e tamanhos.

Você pode consultar o épico escrito de Eric Portis aqui

Agora, nossa função , precisa pegar um glob , e uma largura como entradas, e fazer sua mágica e enviar o arquivo que cada execução gera , para um destino e minimizar a imagem responsável.

Instalamos dois plugins de compressão de imagem no nosso primeiro exemplo

Como esses plugins NÃO começam com um prefixo “gulp-”, precisamos carregá-los manualmente em nosso arquivo gulp.

SO Vamos exigi-los manualmente, após a declaração gulp-load-plugins no topo do nosso gulpfile.

igual a:

var compressJpg = require('imagemin-jpeg-recompress');
var pngquant = require('imagemin-pngquant');

Vale a pena notar que o gulp-responsivo vem com o processador de imagem , que é melhor que imagemagick BY FAAAAR!. Afiado é o que é usado por gulp-responsive , para cortar suas imagens conforme desejado larguras.

você pode olhar para opções de configuração de resposta gulp, para um lista completa de parâmetros de configuração. eu só usei

  • largura - para cortar nossas imagens para uma largura w, passada como parâmetro
  • renomear - para adicionar um sufixo ao nome da imagem, para que permaneça único

na minha função de configuração abaixo. então nossa função , irá cortar a imagem para a largura passada como entrada , para todas as imagens correspondentes decifradas através da entrada glob. então, cada imagem é compactada usando jpeg-recompress ou pngquant e salvo dentro de build/images.

Com isso em mente, nossa função ficaria assim:

/*
@generateResponsiveImages
*@Description:takes in a src of globs, to stream matching image files , a width,
*to resize the matching image to, and a dest to write the resized and minified files to
*@src - input a glob pattern - a string eg 'images/** /*' or 'images/*' or, an array
eg ['glob1','glob2']
*@return returns a stream
*/
var generateResponsiveImages = function(src, width, dest) {

    //declare a default destination
    if (!dest)
        dest = 'build/images';
    //case 1: src glob -  images/**/*
    // the base is the directory immediately preceeding the glob - images/ in this case
    //case 2: images/fixed/flourish.png : the base here is images/fixed/ - we are overriding
    // that by setting base to images.This is done so that, the path beginning after images/
    // - is the path under our destination - without us setting the base, dest would be,
    //build/images/flourish.png.But with us setting the base, the destination would be
    // build/images/fixed/flourish.png
    return $.src(src, {
        base: 'images'
    })

    //generate resized images according to the width passed
    .pipe($$.responsive({
            //match all pngs within the src stream
            '**/*.png': [{
                width: width,
                rename: {
                    suffix: '-' + width
                },
                withoutEnlargement: false,
            }],
            //match all jpgs within the src stream
            '**/*.jpg': [{
                width: width,
                rename: {
                    suffix: '-' + width
                },
                progressive: true,
                withoutEnlargement: false,
            }]
        }, {

            errorOnEnlargement: false,
            errorOnUnusedConfig: false,
            errorOnUnusedImage: false

        }))
        //once the file is resized to width, minify it using the plugins available per format
        .pipe($$.if('*.jpg', compressJpg({
            min: 30,
            max: 90,
            target: 0.5
        })()))
        //use file based cache gulp-cache and it will minify only changed or new files
        //if it is not a new file and if the contents havent changed, the file is served from cache
        .pipe($$.cache($$.imagemin({
            verbose: true
        })))


    //write to destination - dest + path from base
    .pipe($.dest(dest));
}

Observação #

$ - > gole

$$ - > gulp-load-plugins

$.src - > cria fluxos de arquivos correspondentes ao glob passado como src

$.dest - > salva o arquivo manipulado no caminho especificado

Minificador de HTML #

*
 *@minifyHtml
 *Description:takes in a glob src, and minifies all '.html' files matched by the glob
 *@src - input a glob pattern - a string eg '/**/*.html /*' or '*.html' or, an array eg ['glob1','glob2']
 *@dest=file.base means, the modified html file will be in the same directory as the src file being minified
 *often means, the process is just a modification on the existing src file
 *@return returns a stream
 */
var minifyHtml = function(src) {
    return $.src(src)
        .pipe($$.minifyHtml())
        .pipe($.dest(function(file) {
            //file is provided to a dest callback -
            // Refer here https://github.com/gulpjs/gulp/blob/master/docs/API.md#path
            return file.base;
        }));
}

Anatomia de uma tarefa de gole #

A anatomia de uma definição de tarefa é assim:

$.task(<taskname> , [dependencies] , <body>);

dependencies , is an array of tasks that HAVE to finish before the current task you are defining , runs. More like forcing a synchronous execution instead of the default Asynchronous functionality.

Adding Gulp Tasks #

So, we now have

  • A function defined Above to optimise Styles
  • A function defined Above to optimise scripts
  • A function defined Above to optimise HTML
  • A function to generate multiple images per image Above

All we need to do now, is to invoke them when needed.

Let us write our tasks according to the syntax we defined earlier

/*
* $.task('name','dependency array',function)
resulta na construção de um objeto de tarefa como abaixo
tarefa:{
'nome nome,
'dep':[array de dependências],
'fn':função
}
*/


//*@return retorna um fluxo para notificar sobre a conclusão da tarefa
$.task('optimizeHtml', function() {
    var src = ['**/*.html', '!{node_modules}/**/*.html'];
    return minifyHtml(src);
});

//*@return retorna um fluxo para notificar sobre a conclusão da tarefa
$.task('optimizeScripts', function() {
    var src = ['js/**/*.js'];
    return optimizeScripts(src);
});

//*@return retorna um fluxo para notificar sobre a conclusão da tarefa
$.task('optimizeStyles', function() {
    var src = ['css/**/*.css', 'fonts/google/**/*.css'];
    return optimizeStyles(src);
});

//Recebe um retorno de chamada para garantir a notificação do mecanismo gulp, que a tarefa foi concluída
//necessário pois você não está retornando um stream nesta tarefa
$.task('generateResponsiveImages', function(callback) {
    var src = ['images/**/*.{jpg,png}'];
    for (var i = widths.length - 1; i >= 0; i--) {
        generateResponsiveImages(src, widths[i]);
    }
    callback();

});
