Compilar e compilar opencv 3.1.0-dev para Python2 no Windows usando CMake e Visual Studio

  • Lendo imagem e convertendo em escala de cinza

Nesta página

  • Lendo imagem e convertendo em escala de cinza

Construir e compilar o opencv 3.1.0-dev para obter acesso a módulos não gratuitos pode ser uma dor de cabeça para algumas pessoas, especialmente em máquinas Windows. Ao contrário do Ubuntu, configurar o opencv para Windows leva algum tempo e requer que algumas dependências pf sejam instaladas antes de compilar e compilar.

Os programas que você deve baixar e instalar antes de prosseguir em qualquer etapa são:

  1. Python 2.7.x ou Python 3.x.x
  2. CMake

Se você for baixar o Python para Win32, também deverá baixar o CMake para Win32, mesmo se estiver usando uma máquina de 64 bits.

Recomenda-se baixar os programas de 32 bits porque algumas bibliotecas Python são suportadas apenas para máquinas de 32 bits, portanto, para evitar problemas, basta instalar tudo na versão de 32 bits.

  1. Comunidade do Visual Studio 2013

  2. Numpy para Python2.7 Win32

Após instalar todas as dependências acima, reinicie seu PC e então você estará pronto para continuar para a próxima etapa.

Etapa 2:

Se você não é o tipo de pessoa que prefere ler, pode assistir a este tutorial. O tutorial leva você daqui até o final desta documentação.

Você precisará obter opencv e opencv_contrib do github. Você pode encontrar ambos em:

  1. opencv
  2. opencv_contrib

Crie um diretório chamado opencv-3.1.0 onde neste diretório você fará outros dois diretórios um para o build e outro para os sources. Você colocará os dois arquivos zip baixados no arquivo de origem após a extração.

Por exemplo, seu diretório opencv-3.1.0 está localizado na unidade C, então você terá três caminhos:

  1. C:\opencv-3.1.0
  2. C:\opencv-3.1.0\build
  3. C:\opencv-3.1.0\sources

O terceiro diretório incluirá dois caminhos:

  1. C:\opencv-3.1.0\sources\opencv
  2. C:\opencv-3.1.0\sources\opencv_contrib

Agora é feito com preparação. Vamos fazer algumas coisas úteis.

Etapa 3:

Execute o CMake como administrador. Uma janela como esta aparecerá e você terá que fornecer dois diretórios um para os fontes e outro para onde o opencv será compilado. A imagem abaixo pode ajudá-lo melhor do que palavras.

Em seguida clique em configurar e você será promovido a fornecer os geradores; ou seja, compiladores; para opencv. Você deve fornecer o cl.exe localizado no Microsoft Visual Studio 2013. Clique em especificar geradores nativos e uma janela pop-up como a seguinte aparecerá,

Os caminhos serão mais ou menos assim: C:/Program Files (x86)/Microsoft Visual Studio 12.0/VC/bin/cl.exe. Forneça seu caminho para os campos C e C++. Clique em concluir e aguarde até que a configuração seja concluída. Você deve obter zero erros se estiver seguindo todas as etapas anteriores corretamente.

Depois que o CMake terminar de configurar, você verá novos itens aparecendo na janela do CMake destacados em vermelho. Será algo como:

Verifique as compilações que você precisa clicando na pequena caixa quadrada. Procure a linha OPENCV_EXTRA_MODULES_PATH e forneça o diretório de módulos dentro do opencv_contrib dentro do diretório de fontes.

Depois de terminar tudo o que você precisa e fornecer o caminho para os módulos extras, pressione configurar novamente para atualizar. As linhas anteriormente destacadas não serão mais destacadas e os novos campos serão destacados em vermelho.

Marque também as caixas para o que você precisa construir.

Certifique-se de que BUILD_opencv_contrib_world e BUILD_opencv_world estejam ambos desmarcados. Provavelmente há um bug em que ocorre um erro quando qualquer um dos últimos é verificado.

Ao final desta etapa, clique em Gerar e você terminará com o CMake e poderá fechá-lo. Se não houver erros, você receberá uma mensagem no final do painel inferior dizendo Geração concluída.

Etapa 4:

Abra o diretório de compilação localizado em opencv-3.1.0 e você encontrará um monte de novas pastas e arquivos dentro dele. Era uma pasta vazia no início deste processo.

Você só irá lidar com o arquivo OpenCV.sln e não fará nada com os demais arquivos. Abra este arquivo com a versão usada durante a compilação no CMake na etapa anterior. Tem que ser Visual Microsoft 2013.

Ao abrir o arquivo .sln, seja paciente, pois leva algum tempo para preparar tudo para a construção. Quando Pronto estiver estável (sem alterações), você poderá começar a construir seus alvos. Comece a construir conforme numerado na imagem abaixo. Certifique-se também de que a Configuração da solução seja Release e ​​não Debug.

Etapa 5:

Quando a construção estiver concluída, você precisará copiar e colar alguns arquivos do diretório de construção no diretório Python27.

Procure pelo arquivo cv2.pyd e copie-o para o diretório site-packages em Python27. O cv2.pyd deve estar presente em C:\opencv-3.1.0\build\lib\Release. Depois disso, copie somente os arquivos .dll dentro de C:\opencv-3.1.0\build\bin\Release para o diretório pai de Python27 neste local C:\Python27 .

No final desta etapa, reinicie seu PC.

Verificação:

Abra o IDLE e dentro do tipo de shell Python:

>>> import cv2
>>> print cv2.__version__
3.1.0-dev

Lendo imagem e convertendo em escala de cinza #

import cv2
import numpy as np

img = cv2.imread('<your_image>')
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

cv2.imshow('image', img)
cv2.imshow('gray', gray)

cv2.waitKey(0)
cv2.destroyAllWindows()
