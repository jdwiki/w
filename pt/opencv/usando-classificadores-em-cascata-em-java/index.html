Sintaxe #

  • CascadeClassifier cascade = new CascadeClassifier(“cascade.xml”); // Cria um classificador em cascata a partir de cascade.xml
  • Mat image = Imgcodecs.imread(“image.png”); // Converte image.png em um objeto Mat (Matrix)
  • Detecções de MatOfRect = new MatOfRect(); // Cria um arquivo MatOfRect (Matriz de Retângulos) vazio, usado como saída para nossas classes de detecção
  • detects.toArray(); // Retorna um array de objetos Rect que podem ser iterados
  • Imgproc.rectangle(imagem, new Point(rect.x, rect.y), new Point(rect.x + rect.width, rect.y + rect.height), new Scalar(0, 255, 0)); // Desenha um retângulo contornado em verde das localizações x e y do primeiro Point para a localização x e y do segundo Point na “imagem” do objeto Mat. “rect” é um objeto Rect, normalmente fornecido por detects.toArray(). Usa a classe Point do OpenCV.
  • Imgcodecs.imwrite(“saída.png”, imagem); // Escreve o objeto Mat modificado “image” no “output.png”
  • CascadeClassifier.detectMultiScale(imagem, detecções); // Detecta qualquer objeto no objeto Mat “image” e gera as detecções no objeto MatOfRect “detections”
  • CascadeClassifier.detectMultiScale(imagem, detecções, scaleFactor, minNeighbors, flags, minSize, maxSize); // Executa uma detecção com parâmetros adicionais. Veja os detalhes abaixo.
  • Imgproc.ellipse(image, center, axes, 0, 0, 360, new Scalar(255, 0, 255), thickness, lineType, 0); // Desenha uma elipse na imagem no ponto center. Usa a classe Point do OpenCV.

Parâmetros #

Parâmetro Detalhes
fator de escala Quanto o tamanho da imagem é reduzido em cada escala de imagem. Padrão = 1.1
minVizinhos Quantos vizinhos um retângulo candidato deve ter antes de selecioná-lo como um objeto detectado. Padrão = 4
bandeiras Bandeiras de legado. Na maioria dos casos, isso deve ser definido como ‘0’. Padrão = 0
minTamanho Tamanho mínimo que um retângulo candidato pode ter. Isso usa a classe Size do OpenCV. Pode ser usado para diminuir o tempo de detecção e o uso da CPU, bem como para reduzir falsos positivos.
maxSize Tamanho máximo que um retângulo candidato pode ter. Isso usa a classe Size do OpenCV. Pode ser usado para diminuir o tempo de detecção e o uso da CPU, bem como para reduzir falsos positivos.
eixos Usa a classe Size do OpenCV. Define a largura e a altura da elipse.
espessura Espessura da linha, em pixels.
tipodelinha Tem vários parâmetros. 0 é a linha sólida, 8 é para uma linha com 8 conexões, 4 é para uma linha com 4 conexões e CV_AA é para uma linha com suavização de serrilhado. Padrão = 8

Obtendo uma imagem estática, detectando itens nela e gerando os resultados. #

Observe que este exemplo usa OpenCV 3.1.

import org.opencv.core.Mat;
import org.opencv.core.MatOfRect;
import org.opencv.core.Point;
import org.opencv.core.Rect;
import org.opencv.core.Scalar;
import org.opencv.imgcodecs.Imgcodecs;
import org.opencv.imgproc.Imgproc;
import org.opencv.objdetect.CascadeClassifier;

public class Classifier {
    private CascadeClassifier diceCascade = new
        CascadeClassifier("res/newMethod/diceCascade.xml");
    private Mat image;
    private String loc = "path/to/image.png";
    private String output = "path/to/output.png";

    public void detImg() {
    
        Mat image = Imgcodecs.imread(loc); // Reads the image
    
        MatOfRect diceDetections = new MatOfRect(); // Output container
        diceCascade.detectMultiScale(image, diceDetections); // Performs the detection
    
        // Draw a bounding box around each detection.
        for (Rect rect : diceDetections.toArray()) {
            Imgproc.rectangle(image, new Point(rect.x, rect.y),
                new Point(rect.x + rect.width, rect.y + rect.height),
                new Scalar(0, 255, 0));
        }
    
        // Save the visualized detection.
        Imgcodecs.imwrite(output, image);
    
    }
}

O Rect[] retornado por diceDetections.toArray() pode ser iterado. Cada Rect dentro do array terá quatro propriedades principais: x, y, width e height. x e y definem a posição superior esquerda do retângulo, e width e height retornam um int da largura e altura do retângulo. Isso é usado ao desenhar retângulos em imagens. Os parâmetros mínimos obrigatórios da função Imgproc.rectangle são os seguintes:

Imgproc.rectangle(Mat image, Point start, Point end, Scalar color);

Ambos Point são usados ​​para as posições do canto superior esquerdo e do canto inferior direito. Essas posições são ambas absolutas para a imagem fornecida como o primeiro parâmetro, não entre si. Assim, você deve adicionar a posição x ou y do retângulo além da width ou height para definir corretamente o ponto end.

Observe que a classe Point usada nesses parâmetros não é a classe Point da biblioteca padrão Java. Você deve importar a classe Point do OpenCV!

Detectando imagens de um dispositivo de vídeo #

Este exemplo apresenta a classe VideoCapture, onde a usamos para tirar uma imagem de uma webcam e salvá-la em uma imagem.

import org.opencv.core.Mat;
import org.opencv.core.MatOfRect;
import org.opencv.core.Point;
import org.opencv.core.Rect;
import org.opencv.core.Scalar;
import org.opencv.imgcodecs.Imgcodecs;
import org.opencv.imgproc.Imgproc;
import org.opencv.objdetect.CascadeClassifier;
import org.opencv.videoio.VideoCapture;

public class Classifier {
    private CascadeClassifier diceCascade = new
        CascadeClassifier("res/newMethod/diceCascade.xml");
    private Mat image;
    private String loc = "path/to/image.png";
    private String output = "path/to/output.png";
    private VideoCapture vc = new VideoCapture();

    public void detImg() {
        vc.open(0); // Opens the video stream

        Mat image = new Mat(); // Creates an empty matrix
        vc.read(image); // Reads the image from the video stream and
            writes it to the image matrix.
    
        MatOfRect diceDetections = new MatOfRect(); // Output container
        diceCascade.detectMultiScale(image, diceDetections); // Performs the detection
    
        // Draw a bounding box around each detection.
        for (Rect rect : diceDetections.toArray()) {
            Imgproc.rectangle(image, new Point(rect.x, rect.y),
                new Point(rect.x + rect.width, rect.y + rect.height),
                new Scalar(0, 255, 0));
        }
    
        // Save the visualized detection.
        Imgcodecs.imwrite(output, image);

        vc.release(); // Closes the stream.
    
    }
}

Convertendo um objeto Mat em um objeto BufferedImage #

Este exemplo de Daniel Baggio foi tirado diretamente de esta resposta do StackExchange, mas foi republicado para visibilidade.

Esta classe pega um objeto Mat e retorna o objeto BufferedImage usado pelas bibliotecas javax.swing. Isso pode ser usado por um objeto Graphics para desenhar a imagem.

private BufferedImage toBufferedImage(Mat m) {
    if (!m.empty()) {
        int type = BufferedImage.TYPE_BYTE_GRAY;
        if (m.channels() > 1) {
            type = BufferedImage.TYPE_3BYTE_BGR;
        }
        int bufferSize = m.channels() * m.cols() * m.rows();
        byte[] b = new byte[bufferSize];
        m.get(0, 0, b); // get all the pixels
        BufferedImage image = new BufferedImage(m.cols(), m.rows(), type);
        final byte[] targetPixels = ((DataBufferByte) image.getRaster().getDataBuffer()).getData();
        System.arraycopy(b, 0, targetPixels, 0, b.length);
        return image;
    }
    
    return null;
}

Detecções dentro de Detecções #

Este exemplo usa Dice e as manchas pretas nos dados (os pips) como nosso objeto. Como o exemplo é bastante longo, primeiro explicar alguns conceitos-chave é fundamental para entender o exemplo.

Entendendo o primeiro exemplo, “Obter uma imagem estática, detectar itens nela e gerar os resultados”. é fundamental para entender este exemplo, especialmente como o OpenCV desenha retângulos.

Dê uma olhada na imagem a seguir:

Usaremos o método de subimagem, onde usamos uma área detectada como base para aplicar mais detecções. Isso só é possível se um objeto estiver sempre dentro de outro objeto que possamos detectar, como nossos pips em nossos dados. Este método tem vários benefícios:

  • Em vez de escanear a imagem inteira, só precisamos escanear a área onde sabemos que o objeto estará.
  • Remove qualquer chance de falsos positivos fora da área de detecção.

Fazemos isso aplicando primeiro uma varredura de classificador em cascata sobre a imagem inteira para nos dar um objeto MatOfRect contendo nossos objetos grandes (dados, neste caso). Em seguida, iteramos sobre o array Rect[] fornecido pela função toArray() do objeto MatOfRect. Este objeto Rect é usado na criação de um objeto Mat temporário que é “cortado” para as propriedades do objeto Rect (x, y, largura, altura) da imagem original, onde podemos então realizar detecções em o objeto temporário Mat. Em outras palavras, dizemos ao classificador para realizar detecções apenas nas partes dos dados da imagem e especificamos a posição de cada dado usando os objetos Rect que obtivemos ao realizar uma detecção na imagem inteira.

No entanto, os objetos Rect (pips) têm suas propriedades relativas aos seus dados, e não à imagem em si. Para resolver este problema, quando queremos desenhar retângulos na imagem real mostrando as localizações dos pips, adicionamos dice.x e dice.y ao início do Point.

import org.opencv.core.Mat;
import org.opencv.core.MatOfRect;
import org.opencv.core.Point;
import org.opencv.core.Rect;
import org.opencv.core.Scalar;
import org.opencv.core.Size;
import org.opencv.imgcodecs.Imgcodecs;
import org.opencv.imgproc.Imgproc;
import org.opencv.objdetect.CascadeClassifier;
import org.opencv.videoio.VideoCapture;

public class Classifier {

    private CascadeClassifier diceCascade =
        new CascadeClassifier("res/newMethod/diceCascade.xml");
    private CascadeClassifier pipCascade =
        new CascadeClassifier("res/newMethod/pipCascade6.xml");
    private VideoCapture vc = new VideoCapture();
    private Mat image;

    public void openVC(int index) {
        vc.open(index);
    }

    public void closeVC() {
        vc.close();
    }

    public Mat getNextImage() {
        image = new Mat();
        vc.read(image); // Sets the matrix to the current livestream frame.
        
        MatOfRect diceDetections = new MatOfRect(); // Output container
    
        // See syntax for explainations on addition parameters
        diceCascade.detectMultiScale(image, diceDetections, 1.1, 4, 0, new Size(20, 20),
            new Size(38, 38));
        
        // Iterates for every Dice ROI
        for (int i = 0; i < diceDetections.toArray().length; i++) {
            Rect diceRect = diceDetections.toArray()[i];
            
            // Draws rectangles around our detected ROI
            Point startingPoint = new Point(diceRect.x, diceRect.y);
            Point endingPoint = new Point(diceRect.x + diceRect.width,
                diceRect.y + diceRect.height);
            Imgproc.rectangle(image, startingPoint, endingPoint, new Scalar(255, 255, 0));
            
            MatOfRect pipDetections = new MatOfRect();
            
            pipCascade.detectMultiScale(image.submat(diceRect), pipDetections, 1.01, 4, 0,
                new Size(2, 2), new Size(10, 10));
            
            // Gets the number of detected pips and draws a cricle around the ROI
            for (int y = 0; y < pipDetections.toArray().length; y++) {
                // Provides the relative position of the pips to the dice ROI
                Rect pipRect = pipDetections.toArray()[y];
                
                // See syntax explaination
                // Draws a circle around our pips
                Point center = new Point(diceRect.x + pipRect.x + pipRect.width / 2,
                    diceRect.y + pipRect.y + pipRect.height / 2);
                Imgproc.ellipse(image, center, new Size(pipRect.width / 2, pipRect.height / 2),
                     0, 0, 360, new Scalar(255, 0, 255), 1, 0, 0);
            }
        }
        
        return image;
    }
}

A função getNextImage() retorna um objeto Mat, que em conjunto com os outros exemplos postados, pode ser chamado constantemente e pode ser convertido em um BufferImage, para fornecer uma transmissão ao vivo exibindo as detecções.
