Inicialização assíncrona #

Usar a inicialização assíncrona é uma maneira recomendada para o desenvolvimento de aplicativos. Ele usa o OpenCV Manager para acessar bibliotecas OpenCV instaladas externamente no sistema de destino.

Snippet de código que implementa a inicialização assíncrona:

public class MainActivity extends Activity implements CvCameraViewListener2 {

    private BaseLoaderCallback mLoaderCallback = new BaseLoaderCallback(this) {
        @Override
        public void onManagerConnected(int status) {
            switch(status) {
                case LoaderCallbackInterface.SUCCESS:
                    Log.i(TAG,"OpenCV Manager Connected");
                    //from now onwards, you can use OpenCV API
                    Mat m = new Mat(5, 10, CvType.CV_8UC1, new Scalar(0));
                    break;
                case LoaderCallbackInterface.INIT_FAILED:
                    Log.i(TAG,"Init Failed");
                    break;
                case LoaderCallbackInterface.INSTALL_CANCELED:
                    Log.i(TAG,"Install Cancelled");
                    break;
                case LoaderCallbackInterface.INCOMPATIBLE_MANAGER_VERSION:
                    Log.i(TAG,"Incompatible Version");
                    break;
                case LoaderCallbackInterface.MARKET_ERROR:
                    Log.i(TAG,"Market Error");
                    break;
                default:
                    Log.i(TAG,"OpenCV Manager Install");
                    super.onManagerConnected(status);
                    break;
            }
        }
    };

    @Override
    public void onResume() {
        super.onResume();
        OpenCVLoader.initAsync(OpenCVLoader.OPENCV_VERSION_3_1_0, this, mLoaderCallback);
    }

    ...
}

Neste caso, nosso aplicativo funciona com o OpenCV Manager de forma assíncrona. O retorno de chamada OnManagerConnected será chamado no thread da interface do usuário, quando a inicialização terminar.

Observe que não é permitido usar chamadas OpenCV ou carregar Bibliotecas nativas dependentes de OpenCV antes de invocar este retorno de chamada. Carregue seu bibliotecas nativas próprias que dependem do OpenCV após o sucesso do OpenCV inicialização.

A implementação padrão BaseLoaderCallback trata o contexto do aplicativo como Activity e chama o método Activity.finish() para sair em caso de falha na inicialização. Para substituir esse comportamento, você precisa substituir o método finish() da classe BaseLoaderCallback e implementar seu próprio método de finalização.

Gerenciador OpenCV #

OpenCV Manager é um serviço Android direcionado para gerenciar binários da biblioteca OpenCV em dispositivos de usuários finais. Permite compartilhar as bibliotecas dinâmicas do OpenCV entre aplicativos no mesmo dispositivo.

O Gerente oferece os seguintes benefícios:

  • Menos uso de memória (cerca de 40MB). Todos os aplicativos usam os mesmos binários do serviço e não mantêm bibliotecas nativas dentro de si.
  • Otimizações específicas de hardware para todas as plataformas suportadas.
  • Fonte confiável da biblioteca OpenCV. Todos os pacotes com OpenCV são publicados no mercado Google Play.
  • Atualizações regulares e correções de bugs.

A única desvantagem é que o usuário é solicitado a baixar um aplicativo extra, então a experiência do usuário diminui um pouco.

Mais informações: Android OpenCV Manager

Atualizado em 18/10/16: Há um bug na versão do OpenCV Manager distribuída na Play Store (atualizado em 21/09/15). Afeta apenas a versão OpenCV 3.1.0. Quando você executa alguns Funções OpenCV você recebe um erro SIGSEGV. A versão distribuída com Android SDK funciona bem (OpenCV-android-sdk/apk/OpenCV_3.1.0_Manager_3.10_{platform}.apk). Pode ser baixado do site OpenCV. Mais informações: Problema #6247.

Inicialização estática #

De acordo com essa abordagem, todos os binários do OpenCV são incluídos em seu pacote de aplicativos. Ele é projetado principalmente para fins de desenvolvimento e depuração. Essa abordagem é obsoleta para o código de produção. Recomenda-se a inicialização assíncrona.

Se o seu projeto de aplicação não tiver uma parte JNI, basta copiar as bibliotecas nativas do OpenCV correspondentes de OpenCV-3.1.0-android-sdk/sdk/native/libs para o diretório do seu projeto para a pasta app/src/main/ jniLibs.

No caso do projeto de aplicativo com uma parte JNI, em vez de copiar bibliotecas manuais, você precisa modificar seu arquivo Android.mk: adicione as duas linhas de código a seguir após o "include $(CLEAR_VARS)" e antes de "include path_to_OpenCV-3.1.0-android-sdk/sdk/native/jni/OpenCV.mk":

OPENCV_CAMERA_MODULES:=on
OPENCV_INSTALL_MODULES:=on

O resultado deve se parecer com o seguinte:

include $(CLEAR_VARS)
# OpenCV
OPENCV_CAMERA_MODULES:=on
OPENCV_INSTALL_MODULES:=on
include ../../sdk/native/jni/OpenCV.mk

Depois disso, as bibliotecas OpenCV serão copiadas para a pasta jniLibs da sua aplicação durante a compilação JNI.

A última etapa para habilitar o OpenCV em seu aplicativo é o código de inicialização Java antes de chamar a API do OpenCV. Isso pode ser feito, por exemplo, na seção estática da classe Activity:

static {
    if (!OpenCVLoader.initDebug()) {
        // Handle initialization error
    }
}

Se o seu aplicativo inclui outras bibliotecas nativas dependentes do OpenCV, você deve carregá-las após a inicialização do OpenCV:

static {
    if (!OpenCVLoader.initDebug()) {
        // Handle initialization error
    } else {
        System.loadLibrary("my_jni_lib1");
        System.loadLibrary("my_jni_lib2");
    }
}

Observação: o método initDebug() está obsoleto para código de produção. Ele é projetado apenas para fins experimentais e de desenvolvimento local. Se você deseja publicar seu aplicativo, use a abordagem com inicialização assíncrona.
