Sintaxe #

  • DataFrame.merge(right, how=‘inner’, on=Nenhum, left_on=Nenhum, right_on=Nenhum, left_index=False, right_index=False, sort=False, suffixes=(’_x’, ’ _y’), copy=True, Indicator=False)

  • Mesclar objetos DataFrame executando uma operação de junção no estilo de banco de dados por colunas ou índices.

  • Se unir colunas em colunas, os índices DataFrame serão ignorados. Caso contrário, se unir índices em índices ou índices em uma coluna ou colunas, o índice será passado.

Parâmetros #

Parâmetros Explicação
direita DataFrame
como {’esquerda’, ‘direita’, ’externa’, ‘interna’}, padrão ‘interna’
esquerda_on rótulo ou lista, ou tipo array. Nomes de campos para unir no DataFrame esquerdo. Pode ser um vetor ou lista de vetores do tamanho do DataFrame para usar um vetor específico como a chave de junção em vez de colunas
direita_on rótulo ou lista, ou tipo array. Nomes de campos para unir no DataFrame direito ou vetor/lista de vetores por left_on docs
left_index booleano, padrão False. Use o índice do DataFrame esquerdo como a(s) chave(s) de junção. Se for um MultiIndex, o número de chaves no outro DataFrame (o índice ou o número de colunas) deve corresponder ao número de níveis
índice_direito booleano, padrão False. Use o índice do DataFrame certo como a chave de junção. Mesmas advertências que left_index
classificar booleano, Fals padrão. Classifique as chaves de junção lexicograficamente no resultado DataFrame
sufixos Sequência de 2 comprimentos (tupla, lista, …). Sufixo a ser aplicado a nomes de colunas sobrepostos no lado esquerdo e direito, respectivamente
copiar booleano, padrão Verdadeiro. Se False, não copie dados desnecessariamente
indicador booleano ou string, padrão False. Se True, adiciona uma coluna ao DataFrame de saída chamada “_merge” com informações sobre a origem de cada linha. Se for string, a coluna com informações sobre a origem de cada linha será adicionada ao DataFrame de saída, e a coluna será nomeada como valor da string. A coluna de informações é do tipo Categórico e assume um valor de “left_only” para observações cuja chave de mesclagem aparece apenas no DataFrame ’left’, “right_only” para observações cuja chave de mesclagem aparece apenas no DataFrame ‘right’ e “ambos” se o a chave de mesclagem da observação é encontrada em ambos.

Mesclando dois DataFrames #

In [1]: df1 = pd.DataFrame({'x': [1, 2, 3], 'y': ['a', 'b', 'c']})

In [2]: df2 = pd.DataFrame({'y': ['b', 'c', 'd'], 'z': [4, 5, 6]})

In [3]: df1
Out[3]: 
   x  y
0  1  a
1  2  b
2  3  c   

In [4]: df2
Out[4]: 
   y  z
0  b  4
1  c  5
2  d  6

Junção interna: #

Usa a interseção de chaves de dois DataFrames.

In [5]: df1.merge(df2) # by default, it does an inner join on the common column(s)
Out[5]: 
   x  y  z
0  2  b  4
1  3  c  5

Como alternativa, especifique a interseção de chaves de dois Dataframes.

In [5]: merged_inner = pd.merge(left=df1, right=df2, left_on='y', right_on='y')
Out[5]: 
   x  y  z
0  2  b  4
1  3  c  5

Junção externa: #

Usa a união das chaves de dois DataFrames.

In [6]: df1.merge(df2, how='outer')
Out[6]: 
     x  y    z
0  1.0  a  NaN
1  2.0  b  4.0
2  3.0  c  5.0
3  NaN  d  6.0

Associação à esquerda: #

Usa apenas chaves do DataFrame esquerdo.

In [7]: df1.merge(df2, how='left')
Out[7]: 
   x  y    z
0  1  a  NaN
1  2  b  4.0
2  3  c  5.0

União direita #

Usa apenas chaves do DataFrame direito.

In [8]: df1.merge(df2, how='right')
Out[8]: 
     x  y  z
0  2.0  b  4
1  3.0  c  5
2  NaN  d  6

Qual é a diferença entre unir e mesclar #

Considere os dataframes left e right

left = pd.DataFrame([['a', 1], ['b', 2]], list('XY'), list('AB'))
left

   A  B
X  a  1
Y  b  2

right = pd.DataFrame([['a', 3], ['b', 4]], list('XY'), list('AC'))
right

   A  C
X  a  3
Y  b  4

participar Pense em join como querer combinar com dataframes com base em seus respectivos índices. Se houver colunas sobrepostas, join desejará que você adicione um sufixo ao nome da coluna sobreposta do dataframe esquerdo. Nossos dois dataframes têm um nome de coluna sobreposto A.

left.join(right, lsuffix='_')

  A_  B  A  C
X  a  1  a  3
Y  b  2  b  4

Observe que o índice é preservado e temos 4 colunas. 2 colunas da esquerda e 2 da direita.

Se os índices não estiverem alinhados

left.join(right.reset_index(), lsuffix='_', how='outer')

    A_    B index    A    C
0  NaN  NaN     X    a  3.0
1  NaN  NaN     Y    b  4.0
X    a  1.0   NaN  NaN  NaN
Y    b  2.0   NaN  NaN  NaN

Eu usei uma junção externa para ilustrar melhor o ponto. Se os índices não se alinharem, o resultado será a união dos índices.

Podemos dizer ao join para usar uma coluna específica no dataframe esquerdo para usar como chave de junção, mas ainda usará o índice da direita.

left.reset_index().join(right, on='index', lsuffix='_')

  index A_  B  A  C
0     X  a  1  a  3
1     Y  b  2  b  4

mesclar Pense em merge como alinhamento em colunas. Por padrão, merge procurará por colunas sobrepostas para mesclar. merge oferece melhor controle sobre as chaves de mesclagem, permitindo ao usuário especificar um subconjunto das colunas sobrepostas para usar com o parâmetro on, ou permitir separadamente a especificação de quais colunas à esquerda e quais colunas à direita devem ser mescladas .

merge retornará um dataframe combinado no qual o índice será destruído.

Este exemplo simples encontra a coluna sobreposta como 'A' e combina com base nela.

left.merge(right)

   A  B  C
0  a  1  3
1  b  2  4

Observe que o índice é [0, 1] e não mais ['X', 'Y']

Você pode especificar explicitamente que está mesclando o índice com o parâmetro left_index ou right_index

left.merge(right, left_index=True, right_index=True, suffixes=['_', ''])

  A_  B  A  C
X  a  1  a  3
Y  b  2  b  4

E isso se parece exatamente com o exemplo join acima.

Mesclar #

Por exemplo, duas tabelas são fornecidas,

T1

id    x        y
8    42        1.9
9    30        1.9

T2

id    signal
8    55
8    56    
8    59
9    57
9    58    
9    60

O objetivo é obter a nova mesa T3:

id    x        y        s1        s2        s3
8    42        1.9        55        56        58
9    30        1.9        57        58        60

Que é criar as colunas s1, s2 e s3, cada uma correspondendo a uma linha (o número de linhas por id é sempre fixo e igual a 3)

Aplicando join (que recebe um argumento opcional on que pode ser uma coluna ou vários nomes de coluna, que especifica que o DataFrame passado deve ser alinhado nessa coluna no DataFrame). Então a solução pode ser como mostrado abaixo:

df = df1.merge(df2.groupby(‘id’)[‘signal’].apply(lambda x: x.reset_index(drop=True)).unstack().reset_index())

df
Out[63]: 
   id   x    y   0   1   2
0   8  42  1.9  55  56  59
1   9  30  1.9  57  58  60

Se eu os separar:

df2t = df2.groupby('id')['signal'].apply(lambda x: x.reset_index(drop=True)).unstack().reset_index()

df2t
Out[59]: 
   id   0   1   2
0   8  55  56  59
1   9  57  58  60

df = df1.merge(df2t)

df
Out[61]: 
   id   x    y   0   1   2
0   8  42  1.9  55  56  59
1   9  30  1.9  57  58  60

Mesclando/concatenando/juntando vários quadros de dados (horizontal e verticalmente) #

gerar quadros de dados de amostra:

In [57]: df3 = pd.DataFrame({'col1':[211,212,213], 'col2': [221,222,223]})

In [58]: df1 = pd.DataFrame({'col1':[11,12,13], 'col2': [21,22,23]})

In [59]: df2 = pd.DataFrame({'col1':[111,112,113], 'col2': [121,122,123]})

In [60]: df3 = pd.DataFrame({'col1':[211,212,213], 'col2': [221,222,223]})

In [61]: df1
Out[61]:
   col1  col2
0    11    21
1    12    22
2    13    23

In [62]: df2
Out[62]:
   col1  col2
0   111   121
1   112   122
2   113   123

In [63]: df3
Out[63]:
   col1  col2
0   211   221
1   212   222
2   213   223

mesclar/juntar/concatenar quadros de dados [df1, df2, df3] verticalmente - adicionar linhas

In [64]: pd.concat([df1,df2,df3], ignore_index=True)
Out[64]:
   col1  col2
0    11    21
1    12    22
2    13    23
3   111   121
4   112   122
5   113   123
6   211   221
7   212   222
8   213   223

mesclar/juntar/concatenar quadros de dados horizontalmente (alinhando por índice):

In [65]: pd.concat([df1,df2,df3], axis=1)
Out[65]:
   col1  col2  col1  col2  col1  col2
0    11    21   111   121   211   221
1    12    22   112   122   212   222
2    13    23   113   123   213   223

Mesclar, Unir e Concatenar #

Os nomes das chaves de mesclagem são os mesmos

pd.merge(df1, df2, on='key')

Os nomes das chaves de mesclagem são diferentes

pd.merge(df1, df2, left_on='l_key', right_on='r_key')

Diferentes tipos de adesão

pd.merge(df1, df2, on='key', how='left')

Mesclando em várias chaves

pd.merge(df1, df2, on=['key1', 'key2'])

Tratamento de colunas sobrepostas

pd.merge(df1, df2, on='key', suffixes=('_left', '_right'))

Usando índice de linha em vez de mesclar chaves

pd.merge(df1, df2, right_index=True, left_index=True)

Evite o uso da sintaxe .join, pois dá exceção para colunas sobrepostas

Mesclando o índice de dataframe esquerdo e a coluna de dataframe direita

pd.merge(df1, df2, right_index=True, left_on='l_key')

Concate dataframes

Colado verticalmente

pd.concat([df1, df2, df3], axis=0)

Colado horizontalmente

pd.concat([df1, df2, df3], axis=1)
