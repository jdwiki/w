Um ponteiro é um endereço que se refere a um local na memória. Eles são comumente usados ​​para permitir que funções ou estruturas de dados conheçam e modifiquem a memória sem ter que copiar a memória mencionada. Os ponteiros podem ser usados ​​com tipos primitivos (nativos) ou definidos pelo usuário.

Os ponteiros usam os operadores “dereference” * , “address of” & e “arrow” ->. Os operadores ‘*’ e ‘->’ são usados ​​para acessar a memória apontada, e o operador & é usado para obter um endereço na memória.

Sintaxe #

  • <Tipo de dados> *<Nome da variável>;
  • <Tipo de dados> *<Nome da variável> = &<Nome da variável do mesmo tipo de dados>;
  • <Tipo de dados> *<Nome da variável> = <Valor do mesmo tipo de dados>;
  • int *foo; //Um ponteiro que aponta para um valor inteiro
  • int *bar = &myIntVar;
  • longo *bar[2];
  • long *bar[] = {&myLongVar1, &myLongVar2}; //Igual a: long *bar[2]

Esteja ciente dos problemas ao declarar vários ponteiros na mesma linha.

int* a, b, c; //Only a is a pointer, the others are regular ints.

int* a, *b, *c; //These are three pointers!

int *foo[2]; //Both *foo[0] and *foo[1] are pointers.

Noções básicas de ponteiro #

Observação: em todos os itens a seguir, a existência da constante C++ 11 nullptr é assumida. Para versões anteriores, substitua nullptr por NULL, a constante que costumava desempenhar um papel semelhante.

Criando uma variável de ponteiro #

Uma variável de ponteiro pode ser criada usando a sintaxe * específica, por exemplo. int *pointer_to_int;. Quando uma variável é do tipo de ponteiro (int *), ela contém apenas um endereço de memória. O endereço de memória é o local no qual os dados do tipo subjacente (int) são armazenados.

A diferença fica clara ao comparar o tamanho de uma variável com o tamanho de um ponteiro para o mesmo tipo:

// Declare a struct type `big_struct` that contains
// three long long ints.
typedef struct {
    long long int foo1;
    long long int foo2;
    long long int foo3;
} big_struct;

// Create a variable `bar` of type `big_struct`
big_struct bar;
// Create a variable `p_bar` of type `pointer to big_struct`.
// Initialize it to `nullptr` (a null pointer).
big_struct *p_bar0 = nullptr;

// Print the size of `bar`
std::cout << "sizeof(bar) = " << sizeof(bar) << std::endl;
// Print the size of `p_bar`.
std::cout << "sizeof(p_bar0) = " << sizeof(p_bar0) << std::endl;

/* Produces:
    sizeof(bar) = 24
    sizeof(p_bar0) = 8
*/

Pegando o endereço de outra variável #

Ponteiros podem ser atribuídos entre si como variáveis ​​normais; nesse caso, é o endereço de memória que é copiado de um ponteiro para outro, não os dados reais para os quais um ponteiro aponta. Além disso, eles podem assumir o valor nullptr que representa um local de memória nulo. Um ponteiro igual a nullptr contém um local de memória inválido e, portanto, não se refere a dados válidos.

Você pode obter o endereço de memória de uma variável de um determinado tipo prefixando a variável com o operador endereço de &. O valor retornado por & é um ponteiro para o tipo subjacente que contém o endereço de memória da variável (que é um dado válido desde que a variável não saia do escopo).

// Copy `p_bar0` into `p_bar_1`.
big_struct *p_bar1 = p_bar0;

// Take the address of `bar` into `p_bar_2`
big_struct *p_bar2 = &bar;

// p_bar1 is now nullptr, p_bar2 is &bar.

p_bar0 = p_bar2;

// p_bar0 is now &bar.

p_bar2 = nullptr;

// p_bar0 == &bar
// p_bar1 == nullptr
// p_bar2 == nullptr

Ao contrário das referências:

  • a atribuição de dois ponteiros não substitui a memória à qual o ponteiro atribuído se refere;
  • ponteiros podem ser nulos.
  • o operador endereço de é obrigatório explicitamente.

Acessando o conteúdo de um ponteiro #

Assim como pegar um endereço requer &, acessar o conteúdo requer o uso do operador de desreferência *, como prefixo. Quando um ponteiro é desreferenciado, ele se torna uma variável do tipo subjacente (na verdade, uma referência a ele). Ele pode então ser lido e modificado, se não for const.

(*p_bar0).foo1 = 5;

// `p_bar0` points to `bar`. This prints 5.
std::cout << "bar.foo1 = " << bar.foo1 << std::endl;

// Assign the value pointed to by `p_bar0` to `baz`.
big_struct baz;
baz = *p_bar0;

// Now `baz` contains a copy of the data pointed to by `p_bar0`.
// Indeed, it contains a copy of `bar`.

// Prints 5 as well
std::cout << "baz.foo1 = " << baz.foo1 << std::endl;

A combinação de * e o operador . é abreviada por ->:

std::cout << "bar.foo1 = " << (*p_bar0).foo1 << std::endl; // Prints 5
std::cout << "bar.foo1 = " <<  p_bar0->foo1  << std::endl; // Prints 5

Desreferenciando ponteiros inválidos #

Ao desreferenciar um ponteiro, verifique se ele aponta para dados válidos. Desreferenciar um ponteiro inválido (ou um ponteiro nulo) pode levar à violação de acesso à memória ou à leitura ou gravação de dados inúteis.

big_struct *never_do_this() {
   // This is a local variable. Outside `never_do_this` it doesn't exist.
   big_struct retval;
   retval.foo1 = 11;
   // This returns the address of `retval`.
   return &retval;
   // `retval` is destroyed and any code using the value returned
   // by `never_do_this` has a pointer to a memory location that
   // contains garbage data (or is inaccessible).
}

Nesse cenário, g++ e clang++ emitem corretamente os avisos:

(Clang) warning: address of stack memory associated with local variable 'retval' returned [-Wreturn-stack-address]
(Gcc)   warning: address of local variable ‘retval’ returned [-Wreturn-local-addr]

Portanto, cuidado deve ser tomado quando os ponteiros são argumentos de funções, pois eles podem ser nulos:

void naive_code(big_struct *ptr_big_struct) {
    // ... some code which doesn't check if `ptr_big_struct` is valid.
    ptr_big_struct->foo1 = 12;
}

// Segmentation fault.
naive_code(nullptr);

Operações de ponteiro #

Existem dois operadores para ponteiros: Operador Address-of (&): Retorna o endereço de memória de seu operando. Operador Contents-of (Dereference)(*): Retorna o valor da variável localizada no endereço especificado por seu operador.

int var = 20;
int *ptr;
ptr = &var;

cout << var << endl;
//Outputs 20 (The value of var)

cout << ptr << endl;
//Outputs 0x234f119 (var's memory location)

cout << *ptr << endl;
//Outputs 20(The value of the variable stored in the pointer ptr

O asterisco (*) é usado para declarar um ponteiro com o simples propósito de indicar que é um ponteiro. Não confunda isso com o operador dereference, que é usado para obter o valor localizado no endereço especificado. São simplesmente duas coisas diferentes representadas com o mesmo sinal.

Aritmética de ponteiro #

Incrementar / Decrementar #

Um ponteiro pode ser incrementado ou decrementado (prefixo e postfix). Incrementar um ponteiro avança o valor do ponteiro para o elemento na matriz um elemento além do elemento apontado no momento. Decrementar um ponteiro move-o para o elemento anterior na matriz.

A aritmética de ponteiro não é permitida se o tipo para o qual o ponteiro aponta não estiver completo. void é sempre um tipo incompleto.

char* str = new char[10]; // str = 0x010
++str;                    // str = 0x011  in this case sizeof(char) = 1 byte

int* arr = new int[10];   // arr = 0x00100
++arr;                    // arr = 0x00104 if sizeof(int) = 4 bytes

void* ptr = (void*)new char[10];
++ptr;    // void is incomplete.

Se um ponteiro para o elemento final for incrementado, o ponteiro aponta para um elemento após o final da matriz. Tal ponteiro não pode ser desreferenciado, mas pode ser decrementado.

Incrementar um ponteiro para o elemento one-past-the-end na matriz ou decrementar um ponteiro para o primeiro elemento em uma matriz produz um comportamento indefinido.

Um ponteiro para um objeto não-array pode ser tratado, para fins de aritmética de ponteiros, como se fosse um array de tamanho 1.

Adição subtração #

Valores inteiros podem ser adicionados a ponteiros; eles agem como incrementadores, mas por um número específico em vez de 1. Valores inteiros também podem ser subtraídos de ponteiros, atuando como decremento de ponteiro. Assim como no incremento/decremento, o ponteiro deve apontar para um tipo completo.

char* str = new char[10];  // str = 0x010
str += 2;                  // str = 0x010 + 2 * sizeof(char) = 0x012

int* arr = new int[10];    // arr = 0x100
arr += 2;                  // arr = 0x100 + 2 * sizeof(int) = 0x108, assuming sizeof(int) == 4.

Diferença de ponteiro #

A diferença entre dois ponteiros para o mesmo tipo pode ser calculada. Os dois ponteiros devem estar dentro do mesmo objeto de matriz; resultados de comportamento de outra forma indefinidos.

Dados dois ponteiros P e Q no mesmo array, se P for o ith elemento no array, e Q for o jth elemento, então P - Q deve ser i - j. O tipo do resultado é std::ptrdiff_t, de <cstddef>.

char* start = new char[10];  // str = 0x010
char* test = &start[5];
std::ptrdiff_t diff = test - start; //Equal to 5.
std::ptrdiff_t diff = start - test; //Equal to -5; ptrdiff_t is signed.
