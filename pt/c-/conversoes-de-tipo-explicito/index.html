Uma expressão pode ser explicitamente convertida ou cast para digitar T usando dynamic_cast<T>, static_cast<T>, reinterpret_cast<T> ou const_cast<T>, dependendo do que tipo de elenco é pretendido.

C++ também suporta notação de conversão de estilo de função, T(expr), e notação de conversão de estilo C, (T)expr.

Sintaxe #

  • especificador de tipo simples ( )
  • especificador de tipo simples ( lista de expressões )
  • especificador de tipo simples lista de inicialização entre chaves
  • tipo-especificador de nome ( )
  • especificador de nome de tipo ( lista de expressões )
  • especificador de nome de tipo lista de inicialização entre chaves
  • dynamic_cast < type-id > ( expression )
  • static_cast < type-id > ( expression )
  • reinterpret_cast < type-id > ( expression )
  • const_cast < type-id > ( expression )
  • ( type-id ) cast-expression

Todas as seis notações de elenco têm uma coisa em comum:

  • A conversão para um tipo de referência lvalue, como em dynamic_cast<Derived&>(base), produz um lvalue. Portanto, quando você deseja fazer algo com o mesmo objeto, mas tratá-lo como um tipo diferente, você deve converter para um tipo de referência lvalue.
  • A conversão para um tipo de referência rvalue, como em static_cast<string&&>(s), produz um rvalue.
  • A conversão para um tipo sem referência, como em (int)x, produz um prvalue, que pode ser considerado uma cópia do valor que está sendo convertido, mas com um tipo diferente do original.

A palavra-chave reinterpret_cast é responsável por realizar dois tipos diferentes de conversões “inseguras”:

  • As conversões “type punning”, que podem ser usadas para acessar a memória de um tipo como se fosse de um tipo diferente.
  • Conversões entre tipos inteiros e tipos de ponteiro, em qualquer direção.

A palavra-chave static_cast pode realizar várias conversões diferentes:

  • Base para derivada conversões

  • Qualquer conversão que pode ser feita por uma inicialização direta, incluindo conversões implícitas e conversões que chamam um construtor explícito ou uma função de conversão. Consulte aqui e [aqui](https://www.wikiod.com/pt/docs/ c%2b%2b/3090/explicit-type-conversions/18731/conversion-by-explicit-constructor-or-explicit-conversion-function) para obter mais detalhes.

  • Para void, que descarta o valor da expressão.

    // on some compilers, suppresses warning about x being unused
static_cast<void>(x);

  • Entre tipos aritméticos e de enumeração e entre diferentes tipos de enumeração. Consulte conversões enum

  • Do ponteiro para o membro da classe derivada, para o ponteiro para o membro da classe base. Os tipos apontados devem corresponder. Consulte [conversão derivada para base de ponteiros para membros](https://www.wikiod.com/pt/docs/c%2b%2b/3090/explicit-type-conversions/18752/derived-to-base-conversion-for-pointers- para membros)

  • void* para T*.

  • De um lvalue para um xvalue, como em std::move. Consulte mover semântica.

Afastando a constância #

Um ponteiro para um objeto const pode ser convertido em um ponteiro para um objeto não const usando a [palavra-chave] const_cast (https://www.wikiod.com/pt/docs/c%2b%2b/4891/keywords). Aqui usamos const_cast para chamar uma função que não é const-correta. Ele só aceita um argumento char* não const, mesmo que nunca escreva através do ponteiro:

void bad_strlen(char*);
const char* s = "hello, world!";
bad_strlen(s);                    // compile error
bad_strlen(const_cast<char*>(s)); // OK, but it's better to make bad_strlen accept const char*

const_cast para o tipo de referência pode ser usado para converter um lvalue qualificado para const em um valor não qualificado para const.

const_cast é perigoso porque torna impossível para o sistema de tipos C++ impedir que você tente modificar um objeto const. Fazer isso resulta em comportamento indefinido.

const int x = 123;
int& mutable_x = const_cast<int&>(x);
mutable_x = 456; // may compile, but produces *undefined behavior*

Fundição em estilo C #

A conversão do estilo C pode ser considerada a conversão de ‘Melhor esforço’ e é nomeada assim, pois é a única conversão que pode ser usada em C. A sintaxe para essa conversão é (NewType)variable.

Sempre que essa conversão é usada, ela usa uma das seguintes conversões de c++ (em ordem):

  • const_cast<NewType>(variável)
  • static_cast<NewType>(variável)
  • const_cast<NewType>(static_cast<const NewType>(variável))
  • reinterpret_cast<const NewType>(variável)
  • const_cast<NewType>(reinterpret_cast<const NewType>(variável))

A conversão funcional é muito semelhante, embora com algumas restrições como resultado de sua sintaxe: NewType(expression). Como resultado, apenas tipos sem espaços podem ser convertidos.

É melhor usar a nova conversão de c++, porque é mais legível e pode ser detectado facilmente em qualquer lugar dentro de um código-fonte C++ e os erros serão detectados em tempo de compilação, em vez de em tempo de execução.

Como essa conversão pode resultar em reinterpret_cast não intencional, muitas vezes é considerada perigosa.

Base para conversão derivada #

Um ponteiro para uma classe base pode ser convertido em um ponteiro para uma classe derivada usando static_cast. static_cast não faz nenhuma verificação em tempo de execução e pode levar a um comportamento indefinido quando o ponteiro não aponta para o tipo desejado.

struct Base {};
struct Derived : Base {};
Derived d;
Base* p1 = &d;
Derived* p2 = p1;                        // error; cast required
Derived* p3 = static_cast<Derived*>(p1); // OK; p2 now points to Derived object
Base b;
Base* p4 = &b;
Derived* p5 = static_cast<Derived*>(p4); // undefined behaviour since p4 does not
                                         // point to a Derived object

Da mesma forma, uma referência à classe base pode ser convertida em uma referência à classe derivada usando static_cast.

struct Base {};
struct Derived : Base {};
Derived d;
Base& r1 = d;
Derived& r2 = r1;                        // error; cast required
Derived& r3 = static_cast<Derived&>(r1); // OK; r3 now refers to Derived object

Se o tipo de origem for polimórfico, dynamic_cast pode ser usado para realizar uma conversão de base para derivada. Ele executa uma verificação em tempo de execução e a falha é recuperável em vez de produzir um comportamento indefinido. No caso do ponteiro, um ponteiro nulo é retornado em caso de falha. No caso de referência, uma exceção é lançada em caso de falha do tipo std::bad_cast (ou uma classe derivada de std::bad_cast).

struct Base { virtual ~Base(); }; // Base is polymorphic
struct Derived : Base {};
Base* b1 = new Derived;
Derived* d1 = dynamic_cast<Derived*>(b1); // OK; d1 points to Derived object
Base* b2 = new Base;
Derived* d2 = dynamic_cast<Derived*>(b2); // d2 is a null pointer

Conversão entre ponteiro e inteiro #

Um ponteiro de objeto (incluindo void*) ou ponteiro de função pode ser convertido para um tipo inteiro usando reinterpret_cast. Isso só será compilado se o tipo de destino for longo o suficiente. O resultado é definido pela implementação e normalmente fornece o endereço numérico do byte na memória para o qual o ponteiro aponta.

Normalmente, long ou unsigned long é longo o suficiente para conter qualquer valor de ponteiro, mas isso não é garantido pelo padrão.

Se os tipos std::intptr_t e std::uintptr_t existirem, é garantido que eles sejam longos o suficiente para conter um void* (e, portanto, qualquer ponteiro para o tipo de objeto). No entanto, não há garantia de que sejam longos o suficiente para conter um ponteiro de função.

Da mesma forma, reinterpret_cast pode ser usado para converter um tipo inteiro em um tipo de ponteiro. Novamente, o resultado é definido pela implementação, mas um valor de ponteiro é garantido como inalterado por uma viagem de ida e volta por meio de um tipo inteiro. O padrão não garante que o valor zero seja convertido em um ponteiro nulo.

void register_callback(void (*fp)(void*), void* arg); // probably a C API
void my_callback(void* x) {
    std::cout << "the value is: " << reinterpret_cast<long>(x); // will probably compile
}
long x;
std::cin >> x;
register_callback(my_callback,
                  reinterpret_cast<void*>(x)); // hopefully this doesn't lose information...

Conversão de trocadilhos #

Um ponteiro (referência resp.) para um tipo de objeto pode ser convertido em um ponteiro (referência resp.) para qualquer outro tipo de objeto usando reinterpret_cast. Isso não chama nenhum construtor ou função de conversão.

int x = 42;
char* p = static_cast<char*>(&x);      // error: static_cast cannot perform this conversion
char* p = reinterpret_cast<char*>(&x); // OK
*p = 'z';                              // maybe this modifies x (see below)

O resultado de reinterpret_cast representa o mesmo endereço que o operando, desde que o endereço esteja devidamente alinhado para o tipo de destino. Caso contrário, o resultado não é especificado.

int x = 42;
char& r = reinterpret_cast<char&>(x);
const void* px = &x;
const void* pr = &r;
assert(px == pr); // should never fire

O resultado de reinterpret_cast não é especificado, exceto que um ponteiro (referência resp.) sobreviverá a uma viagem de ida e volta do tipo de origem para o tipo de destino e vice-versa, desde que o requisito de alinhamento do tipo de destino não seja mais rigoroso que o da origem modelo.

int x = 123;
unsigned int& r1 = reinterpret_cast<unsigned int&>(x);
int& r2 = reinterpret_cast<int&>(r1);
r2 = 456; // sets x to 456

Na maioria das implementações, reinterpret_cast não altera o endereço, mas esse requisito não foi padronizado até o C++11.

reinterpret_cast também pode ser usado para converter de um tipo de ponteiro para membro de dados para outro, ou de um tipo de ponteiro para função de membro para outro.

O uso de reinterpret_cast é considerado perigoso porque ler ou escrever através de um ponteiro ou referência obtida usando reinterpret_cast pode desencadear um comportamento indefinido quando os tipos de origem e destino não estão relacionados.

Conversão por construtor explícito ou função de conversão explícita #

Uma conversão que envolve chamar um construtor ou função de conversão explícito não pode ser feita implicitamente. Podemos solicitar que a conversão seja feita explicitamente usando static_cast. O significado é o mesmo de uma inicialização direta, exceto que o resultado é temporário.

class C {
    std::unique_ptr<int> p;
  public:
    explicit C(int* p) : p(p) {}
};
void f(C c);
void g(int* p) {
    f(p);                 // error: C::C(int*) is explicit
    f(static_cast<C>(p)); // ok
    f(C(p));              // equivalent to previous line
    C c(p); f(c);         // error: C is not copyable
}

Conversão implícita #

static_cast pode realizar qualquer conversão implícita. Este uso de static_cast pode ocasionalmente ser útil, como nos exemplos a seguir:

  • Ao passar argumentos para reticências, o tipo de argumento “esperado” não é conhecido estaticamente, portanto, nenhuma conversão implícita ocorrerá.

    const double x = 3.14;
printf("%d\n", static_cast<int>(x)); // prints 3
// printf("%d\n", x); // undefined behaviour; printf is expecting an int here
// alternative:
// const int y = x; printf("%d\n", y);

Sem a conversão de tipo explícita, um objeto double seria passado para as reticências e ocorreria um comportamento indefinido.

  • Um operador de atribuição de classe derivada pode chamar um operador de atribuição de classe base da seguinte forma:

    struct Base { /* ... */ };
struct Derived : Base {
    Derived& operator=(const Derived& other) {
        static_cast<Base&>(*this) = other;
        // alternative:
        // Base& this_base_ref = *this; this_base_ref = other;
    }
};

Conversões de enumeração #

static_cast pode converter de um tipo inteiro ou de ponto flutuante para um tipo de enumeração (com ou sem escopo) e vice-versa. Ele também pode converter entre tipos de enumeração.

  • A conversão de um tipo de enumeração sem escopo para um tipo aritmético é uma conversão implícita; é possível, mas não necessário, usar static_cast.
  • Quando um tipo de enumeração com escopo é convertido em um tipo aritmético:
  • Se o valor do enum puder ser representado exatamente no tipo de destino, o resultado será esse valor.
  • Caso contrário, se o tipo de destino for um tipo inteiro, o resultado não será especificado.
  • Caso contrário, se o tipo de destino for um tipo de ponto flutuante, o resultado será o mesmo da conversão para o tipo subjacente e depois para o tipo de ponto flutuante.

Exemplo:

  enum class Format {
      TEXT = 0,
      PDF = 1000,
      OTHER = 2000,
  };
  Format f = Format::PDF;
  int a = f;                         // error
  int b = static_cast<int>(f);       // ok; b is 1000
  char c = static_cast<char>(f);     // unspecified, if 1000 doesn't fit into char
  double d = static_cast<double>(f); // d is 1000.0... probably
  • Quando um tipo inteiro ou de enumeração é convertido em um tipo de enumeração:
  • Se o valor original estiver dentro do intervalo da enumeração de destino, o resultado será esse valor. Observe que esse valor pode ser diferente para todos os enumeradores.
  • Caso contrário, o resultado não é especificado (<= C++14) ou indefinido (>= C++17).

Exemplo:

  enum Scale {
      SINGLE = 1,
      DOUBLE = 2,
      QUAD = 4
  };
  Scale s1 = 1;                     // error
  Scale s2 = static_cast<Scale>(2); // s2 is DOUBLE
  Scale s3 = static_cast<Scale>(3); // s3 has value 3, and is not equal to any enumerator
  Scale s9 = static_cast<Scale>(9); // unspecified value in C++14; UB in C++17

  • Quando um tipo de ponto flutuante é convertido em um tipo de enumeração, o resultado é o mesmo que converter para o tipo subjacente da enumeração e, em seguida, para o tipo enumeração.

    enum Direction {
    UP = 0,
    LEFT = 1,
    DOWN = 2,
    RIGHT = 3,
};
Direction d = static_cast<Direction>(3.14); // d is RIGHT

Derivado da conversão de base para ponteiros para membros #

Um ponteiro para membro da classe derivada pode ser convertido em um ponteiro para membro da classe base usando static_cast. Os tipos apontados devem corresponder.

Se o operando for um ponteiro nulo para o valor do membro, o resultado também será um ponteiro nulo para o valor do membro.

Caso contrário, a conversão só é válida se o membro apontado pelo operando realmente existir na classe de destino, ou se a classe de destino for uma classe base ou derivada da classe que contém o membro apontado pelo operando. static_cast não verifica a validade. Se a conversão não for válida, o comportamento será indefinido.

struct A {};
struct B { int x; };
struct C : A, B { int y; double z; };
int B::*p1 = &B::x;
int C::*p2 = p1;                              // ok; implicit conversion
int B::*p3 = p2;                              // error
int B::*p4 = static_cast<int B::*>(p2);       // ok; p4 is equal to p1
int A::*p5 = static_cast<int A::*>(p2);       // undefined; p2 points to x, which is a member
                                              // of the unrelated class B
double C::*p6 = &C::z;
double A::*p7 = static_cast<double A::*>(p6); // ok, even though A doesn't contain z
int A::*p8 = static_cast<int A::*>(p6);       // error: types don't match

nulo* para T* #

Em C++, void* não pode ser convertido implicitamente em T* onde T é um tipo de objeto. Em vez disso, static_cast deve ser usado para realizar a conversão explicitamente. Se o operando realmente aponta para um objeto T, o resultado aponta para esse objeto. Caso contrário, o resultado não é especificado.

Mesmo que o operando não aponte para um objeto T, desde que o operando aponte para um byte cujo endereço esteja devidamente alinhado para o tipo T, o resultado da conversão aponta para o mesmo byte.

// allocating an array of 100 ints, the hard way
int* a = malloc(100*sizeof(*a));                    // error; malloc returns void*
int* a = static_cast<int*>(malloc(100*sizeof(*a))); // ok
// int* a = new int[100];                           // no cast needed
// std::vector<int> a(100);                         // better

const char c = '!';
const void* p1 = &c;
const char* p2 = p1;                           // error
const char* p3 = static_cast<const char*>(p1); // ok; p3 points to c
const int* p4 = static_cast<const int*>(p1);   // unspecified in C++03;
                                               // possibly unspecified in C++11 if
                                               // alignof(int) > alignof(char)
char* p5 = static_cast<char*>(p1);             // error: casting away constness
