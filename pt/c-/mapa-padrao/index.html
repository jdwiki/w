  • Para usar qualquer um dos std::map ou std::multimap o arquivo de cabeçalho <map> deve ser incluído.

  • std::map e std::multimap ambos mantêm seus elementos ordenados de acordo com a ordem crescente de chaves. No caso de std::multimap, não ocorre ordenação para os valores da mesma chave.

  • A diferença básica entre std::map e std::multimap é que o std::map não permite valores duplicados para a mesma chave onde std::multimap permite.

  • Os mapas são implementados como árvores de busca binária. Então search(), insert(), erase() leva Θ(log n) tempo em média. Para operação em tempo constante, use std::unordered_map.

  • As funções size() e empty() têm Θ(1) complexidade de tempo, o número de nós é armazenado em cache para evitar percorrer a árvore cada vez que essas funções são chamadas.

Acessando elementos #

Um std::map recebe pares (chave, valor) como entrada.

Considere o seguinte exemplo de inicialização de std::map:

std::map < std::string, int > ranking { std::make_pair("stackoverflow", 2), 
                                        std::make_pair("docs-beta", 1) };

Em um std::map , os elementos podem ser inseridos da seguinte forma:

ranking["stackoverflow"]=2;
ranking["docs-beta"]=1;

No exemplo acima, se a chave stackoverflow já estiver presente, seu valor será atualizado para 2. Se ainda não estiver presente, uma nova entrada será criada.

Em um std::map, os elementos podem ser acessados ​​diretamente fornecendo a chave como um índice:

std::cout << ranking[ "stackoverflow" ] << std::endl;

Observe que usar o operator[] no mapa realmente inserirá um novo valor com a chave consultada no mapa. Isso significa que você não pode usá-lo em um const std::map, mesmo que a chave já esteja armazenada no mapa. Para evitar essa inserção, verifique se o elemento existe (por exemplo, usando find()) ou use at() conforme descrito abaixo.

Elementos de um std::map podem ser acessados ​​com at():

std::cout << ranking.at("stackoverflow") << std::endl;

Observe que at() lançará uma exceção std::out_of_range se o contêiner não contiver o elemento solicitado.

Em ambos os contêineres std::map e std::multimap, os elementos podem ser acessados ​​usando iteradores:

// Example using begin()
std::multimap < int, std::string > mmp { std::make_pair(2, "stackoverflow"),
                                         std::make_pair(1, "docs-beta"),
                                         std::make_pair(2, "stackexchange")  };
auto it = mmp.begin();
std::cout << it->first << " : " << it->second << std::endl; // Output: "1 : docs-beta"
it++;
std::cout << it->first << " : " << it->second << std::endl; // Output: "2 : stackoverflow"
it++;
std::cout << it->first << " : " << it->second << std::endl; // Output: "2 : stackexchange"

// Example using rbegin()
std::map < int, std::string > mp {  std::make_pair(2, "stackoverflow"),
                                    std::make_pair(1, "docs-beta"),
                                    std::make_pair(2, "stackexchange")  };
auto it2 = mp.rbegin();
std::cout << it2->first << " : " << it2->second << std::endl; // Output: "2 : stackoverflow"
it2++;
std::cout << it2->first << " : " << it2->second << std::endl; // Output: "1 : docs-beta"

Inserindo elementos #

Um elemento pode ser inserido em um std::map somente se sua chave ainda não estiver presente no mapa. Dado por exemplo:

std::map< std::string, size_t > fruits_count;

  • Um par chave-valor é inserido em um std::map através da função membro insert(). Requer um pair como argumento:

    fruits_count.insert({"grapes", 20});
fruits_count.insert(make_pair("orange", 30));
fruits_count.insert(pair<std::string, size_t>("banana", 40));
fruits_count.insert(map<std::string, size_t>::value_type("cherry", 50));

A função insert() retorna um pair consistindo em um iterador e um valor bool:

- If the insertion was successful, the iterator points to the newly inserted element, and the `bool` value is `true`.
- If there was already an element with the same `key`, the insertion fails. When that happens, the iterator points to the element causing the conflict, and the `bool` is value is `false`.

O seguinte método pode ser usado para combinar a operação de inserção e pesquisa:

  auto success = fruits_count.insert({"grapes", 20});
  if (!success.second) {           // we already have 'grapes' in the map
      success.first->second += 20; // access the iterator to update the value
  }

  • Por conveniência, o contêiner std::map fornece o operador subscript para acessar elementos no mapa e inserir novos se eles não existirem:

    fruits_count["apple"] = 10;

Embora mais simples, impede que o usuário verifique se o elemento já existe. Se um elemento estiver faltando, std:🗺:operator[] implicitamente o cria, inicializando-o com o construtor padrão antes de sobrescrevê-lo com o valor fornecido.

  • insert() pode ser usado para adicionar vários elementos de uma vez usando uma lista de pares entre colchetes. Esta versão de insert() retorna void:

     fruits_count.insert({{"apricot", 1}, {"jackfruit", 1}, {"lime", 1}, {"mango", 7}});

  • insert() também pode ser usado para adicionar elementos usando iteradores que denotam o início e o fim dos valores value_type:

    std::map< std::string, size_t > fruit_list{ {"lemon", 0}, {"olive", 0}, {"plum", 0}};
fruits_count.insert(fruit_list.begin(), fruit_list.end());

Exemplo:

std::map<std::string, size_t> fruits_count;
std::string fruit;
while(std::cin >> fruit){
    // insert an element with 'fruit' as key and '1' as value
    // (if the key is already stored in fruits_count, insert does nothing)
    auto ret = fruits_count.insert({fruit, 1});
    if(!ret.second){            // 'fruit' is already in the map 
        ++ret.first->second;    // increment the counter
    }
}

A complexidade de tempo para uma operação de inserção é O(log n) porque std::map são implementados como árvores.

Um pair pode ser construído explicitamente usando make_pair() e emplace():

std::map< std::string , int > runs;
runs.emplace("Babe Ruth", 714);
runs.insert(make_pair("Barry Bonds", 762));

Se soubermos onde o novo elemento será inserido, podemos usar emplace_hint() para especificar um iterador hint. Se o novo elemento puder ser inserido antes de hint, então a inserção poderá ser feita em tempo constante. Caso contrário, ele se comporta da mesma maneira que emplace():

std::map< std::string , int > runs;
auto it = runs.emplace("Barry Bonds", 762); // get iterator to the inserted element
// the next element will be before "Barry Bonds", so it is inserted before 'it'
runs.emplace_hint(it, "Babe Ruth", 714);

Pesquisando em std::map ou em std::multimap #

Existem várias maneiras de pesquisar uma chave em std::map ou em std::multimap.

  • Para obter o iterador da primeira ocorrência de uma chave, a função find() pode ser usada. Ele retorna end() se a chave não existir.

      std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
  auto it = mmp.find(6);
  if(it!=mmp.end())
      std::cout << it->first << ", " << it->second << std::endl; //prints: 6, 5
  else
      std::cout << "Value does not exist!" << std::endl;

  it = mmp.find(66);
  if(it!=mmp.end())
      std::cout << it->first << ", " << it->second << std::endl; 
  else
      std::cout << "Value does not exist!" << std::endl; // This line would be executed.

  • Outra maneira de descobrir se uma entrada existe em std::map ou em std::multimap é usando a função count(), que conta quantos valores estão associados a uma determinada chave. Como std::map associa apenas um valor a cada chave, sua função count() só pode retornar 0 (se a chave não estiver presente) ou 1 (se estiver). Para std::multimap, count() pode retornar valores maiores que 1, pois pode haver vários valores associados à mesma chave.

      std::map< int , int > mp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
  if(mp.count(3) > 0) // 3 exists as a key in map
      std::cout << "The key exists!" << std::endl; // This line would be executed.
  else
      std::cout << "The key does not exist!" << std::endl;

Se você só se importa se algum elemento existe, find é estritamente melhor: ele documenta sua intenção e, para multimaps, pode parar assim que o primeiro elemento correspondente for encontrado.

  • No caso de std::multimap, pode haver vários elementos com a mesma chave. Para obter esse intervalo, é usada a função equal_range() que retorna std::pair com limite inferior do iterador (inclusive) e limite superior (exclusivo), respectivamente. Se a chave não existir, ambos os iteradores apontariam para end().

      auto eqr = mmp.equal_range(6);
  auto st = eqr.first, en = eqr.second;
  for(auto it = st; it != en; ++it){
      std::cout << it->first << ", " << it->second << std::endl; 
  }
      // prints: 6, 5
      //         6, 7

Inicializando um std::map ou std::multimap #

std::map e std::multimap ambos podem ser inicializados fornecendo pares chave-valor separados por vírgula. Os pares chave-valor podem ser fornecidos por {key, value} ou podem ser criados explicitamente por std::make_pair(key, value). Como std::map não permite chaves duplicadas e o operador de vírgula funciona da direita para a esquerda, o par à direita seria substituído pelo par com a mesma chave à esquerda.

std::multimap < int, std::string > mmp { std::make_pair(2, "stackoverflow"),
                                     std::make_pair(1, "docs-beta"),
                                     std::make_pair(2, "stackexchange")  };
// 1 docs-beta
// 2 stackoverflow
// 2 stackexchange

std::map < int, std::string > mp {  std::make_pair(2, "stackoverflow"),
                                std::make_pair(1, "docs-beta"),
                                std::make_pair(2, "stackexchange")  }; 
// 1 docs-beta
// 2 stackoverflow

Ambos podem ser inicializados com iterador.

// From std::map or std::multimap iterator
std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {6, 8}, {3, 4}, 
                               {6, 7} };
                       // {1, 2}, {3, 4}, {3, 4}, {6, 5}, {6, 8}, {6, 7}, {8, 9}
auto it = mmp.begin();
std::advance(it,3); //moved cursor on first {6, 5}
std::map< int, int > mp(it, mmp.end()); // {6, 5}, {8, 9}

//From std::pair array
std::pair< int, int > arr[10];
arr[0] = {1, 3};
arr[1] = {1, 5};
arr[2] = {2, 5};
arr[3] = {0, 1};
std::map< int, int > mp(arr,arr+4); //{0 , 1}, {1, 3}, {2, 5}

//From std::vector of std::pair
std::vector< std::pair<int, int> > v{ {1, 5}, {5, 1}, {3, 6}, {3, 2} };
std::multimap< int, int > mp(v.begin(), v.end()); 
                        // {1, 5}, {3, 6}, {3, 2}, {5, 1}

Verificando o número de elementos #

O container std::map tem uma função membro empty(), que retorna true ou false, dependendo se o mapa está vazio ou não. A função membro size() retorna o número de elementos armazenados em um container std::map:

std::map<std::string , int> rank {{"facebook.com", 1} ,{"google.com", 2}, {"youtube.com", 3}};
if(!rank.empty()){
    std::cout << "Number of elements in the rank map: " << rank.size() << std::endl;
}
else{
    std::cout << "The rank map is empty" << std::endl;
}

Tipos de mapas #

Mapa Regular #

Um mapa é um contêiner associativo, contendo pares de valores-chave.

#include <string>
#include <map>
std::map<std::string, size_t> fruits_count;

No exemplo acima, std::string é o tipo key e size_t é um valor.

A chave atua como um índice no mapa. Cada chave deve ser exclusiva e deve ser ordenada.

  • Se você precisar de vários elementos com a mesma chave, considere usar multimap (explicado abaixo)

  • Se o seu tipo de valor não especificar nenhuma ordenação ou se você quiser substituir a ordenação padrão, forneça uma:

    #include <string>
#include <map>
#include <cstring>
struct StrLess {
    bool operator()(const std::string& a, const std::string& b) {
        return strncmp(a.c_str(), b.c_str(), 8)<0;
               //compare only up to 8 first characters
    }
}
std::map<std::string, size_t, StrLess> fruits_count2;

    If StrLess comparator returns false for two keys, they are considered the same even if their actual contents differ.

Mapa múltiplo #

Multimap permite que vários pares de chave-valor com a mesma chave sejam armazenados no mapa. Caso contrário, sua interface e criação são muito semelhantes ao mapa regular.

 #include <string>
 #include <map>
 std::multimap<std::string, size_t> fruits_count;
 std::multimap<std::string, size_t, StrLess> fruits_count2;

Mapa de Hash (Mapa Não Ordenado) #

Um mapa de hash armazena pares de valores-chave semelhantes a um mapa regular. No entanto, não ordena os elementos em relação à chave. Em vez disso, um valor hash para a chave é usado para acessar rapidamente os pares de valores-chave necessários.

#include <string>
#include <unordered_map>
std::unordered_map<std::string, size_t> fruits_count;

Mapas não ordenados geralmente são mais rápidos, mas os elementos não são armazenados em nenhuma ordem previsível. Por exemplo, iterar sobre todos os elementos em um unordered_map fornece os elementos em uma ordem aparentemente aleatória.

Excluindo elementos #

Removendo todos os elementos:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
mmp.clear(); //empty multimap

Removendo o elemento de algum lugar com a ajuda do iterador:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
                            // {1, 2}, {3, 4}, {3, 4}, {6, 5}, {6, 7}, {8, 9}
auto it = mmp.begin();
std::advance(it,3); // moved cursor on first {6, 5}
mmp.erase(it); // {1, 2}, {3, 4}, {3, 4}, {6, 7}, {8, 9}

Removendo todos os elementos em um intervalo:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
                            // {1, 2}, {3, 4}, {3, 4}, {6, 5}, {6, 7}, {8, 9}
auto it = mmp.begin();
auto it2 = it;
it++; //moved first cursor on first {3, 4}
std::advance(it2,3);  //moved second cursor on first {6, 5}
mmp.erase(it,it2); // {1, 2}, {6, 5}, {6, 7}, {8, 9}

Removendo todos os elementos com um valor fornecido como chave:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
                            // {1, 2}, {3, 4}, {3, 4}, {6, 5}, {6, 7}, {8, 9}
mmp.erase(6); // {1, 2}, {3, 4}, {3, 4}, {8, 9}

Removendo elementos que satisfazem um predicado pred:

std::map<int,int> m;
auto it = m.begin();
while (it != m.end())
{
   if (pred(*it))
       it = m.erase(it);
   else
       ++it;
}

Iterando sobre std::map ou std::multimap #

std::map ou std::multimap podem ser percorridos das seguintes maneiras:

std::multimap< int , int > mmp{ {1, 2}, {3, 4}, {6, 5}, {8, 9}, {3, 4}, {6, 7} };
                               
//Range based loop - since C++11
for(const auto &x: mmp) 
    std::cout<< x.first <<":"<< x.second << std::endl;

//Forward iterator for loop: it would loop through first element to last element
//it will be a std::map< int, int >::iterator
for (auto it = mmp.begin(); it != mmp.end(); ++it)
std::cout<< it->first <<":"<< it->second << std::endl; //Do something with iterator

//Backward iterator for loop: it would loop through last element to first element
//it will be a std::map< int, int >::reverse_iterator
for (auto it = mmp.rbegin(); it != mmp.rend(); ++it)
std::cout<< it->first <<" "<< it->second << std::endl; //Do something with iterator

Ao iterar sobre um std::map ou um std::multimap, o uso de auto é preferível para evitar conversões implícitas inúteis (veja esta resposta SO para mais detalhes).

Criando std::map com tipos definidos pelo usuário como chave #

Para poder usar uma classe como chave em um mapa, tudo o que é necessário da chave é que ela seja copiável e atribuível. A ordenação dentro do mapa é definida pelo terceiro argumento para o template (e o argumento para o construtor, se usado). este padrão para std::less<KeyType>, que é padronizado para o operador <, mas não há necessidade de usar os padrões. Basta escrever uma comparação operador (de preferência como um objeto funcional):

struct CmpMyType
{
    bool operator()( MyType const& lhs, MyType const& rhs ) const
    {
        //  ...
    }
};

Em C++, o predicado “compare” deve ser uma ordenação fraca estrita. Em particular, compare(X,X) deve retornar false para qualquer X. ou seja, se CmpMyType()(a, b) retornar true, então CmpMyType()(b, a) deve retornar false, e se ambos retornarem false, os elementos são considerados iguais (membros da mesma classe de equivalência).

Ordenação Fraca Estrita #

Este é um termo matemático para definir uma relação entre dois objetos. Sua definição é:

Dois objetos x e y são equivalentes se f(x, y) e f(y, x) forem falsos. Observe que um objeto é sempre (pela invariante de irreflexividade) equivalente a si mesmo.

Em termos de C++ isso significa que se você tiver dois objetos de um determinado tipo, você deve retornar os seguintes valores quando comparado com o operador <.

X    a;
X    b;

Condition:                  Test:     Result
a is equivalent to b:       a < b     false
a is equivalent to b        b < a     false

a is less than b            a < b     true
a is less than b            b < a     false

b is less than a            a < b     false
b is less than a            b < a     true

Como você define equivalente/menos é totalmente dependente do tipo do seu objeto.
