Olá Mundo #

Este programa imprime Hello World! no fluxo de saída padrão:

#include <iostream>

int main()
{
    std::cout << "Hello World!" << std::endl;
}

Veja ao vivo no Coliru.

Análise #

Vamos examinar cada parte deste código em detalhes:

  • #include <iostream> é uma diretiva de pré-processador que inclui o conteúdo do arquivo de cabeçalho C++ padrão iostream.

    iostream is a standard library header file that contains definitions of the standard input and output streams. These definitions are included in the std namespace, explained below.

    The standard input/output (I/O) streams provide ways for programs to get input from and output to an external system – usually the terminal.

  • int main() { ... } define uma nova função chamada main. Por convenção, a função main é chamada na execução do programa. Deve haver apenas uma função main em um programa C++, e deve sempre retornar um número do tipo int.

    Here, the int is what is called the function’s return type. O valor retornado pela função main é um código de saída.

    By convention, a program exit code of 0 or EXIT_SUCCESS is interpreted as success by a system that executes the program. Any other return code is associated with an error.

    If no return statement is present, the main function (and thus, the program itself) returns 0 by default. In this example, we don’t need to explicitly write return 0;.

    All other functions, except those that return the void type, must explicitly return a value according to their return type, or else must not return at all.

  • std::cout << "Olá Mundo!" << std::endl; imprime “Hello World!” para o fluxo de saída padrão:

    • std is a namespace, and :: is the scope resolution operator that allows look-ups for objects by name within a namespace.

      There are many namespaces. Here, we use :: to show we want to use cout from the std namespace. For more information refer to Scope Resolution Operator - Microsoft Documentation.

    • std::cout is the standard output stream object, defined in iostream, and it prints to the standard output (stdout).

    • << is, in this context, the stream insertion operator, so called because it inserts an object into the stream object.

      The standard library defines the << operator to perform data insertion for certain data types into output streams. stream << content inserts content into the stream and returns the same, but updated stream. This allows stream insertions to be chained: std::cout << "Foo" << " Bar"; prints “FooBar” to the console.

    • "Hello World!" is a character string literal, or a “text literal.” The stream insertion operator for character string literals is defined in file iostream.

    • std::endl is a special I/O stream manipulator object, also defined in file iostream. Inserting a manipulator into a stream changes the state of the stream.

      The stream manipulator std::endl does two things: first it inserts the end-of-line character and then it flushes the stream buffer to force the text to show up on the console. This ensures that the data inserted into the stream actually appear on your console. (Stream data is usually stored in a buffer and then “flushed” in batches unless you force a flush immediately.)

      An alternate method that avoids the flush is:

      std::cout << "Hello World!\n";

      where \n is the character escape sequence for the newline character.

    • The semicolon (;) notifies the compiler that a statement has ended. All C++ statements and class definitions require an ending/terminating semicolon.

Comentários #

Um comentário é uma maneira de colocar texto arbitrário dentro do código-fonte sem que o compilador C++ o interprete com qualquer significado funcional. Os comentários são usados ​​para fornecer informações sobre o design ou método de um programa.

Existem dois tipos de comentários em C++:

Comentários de linha única #

A sequência de barra dupla // marcará todo o texto até uma nova linha como um comentário:

int main()
{
   // This is a single-line comment.
   int a;  // this also is a single-line comment
   int i;  // this is another single-line comment
}

Comentários em estilo C/bloco #

A sequência /* é usada para declarar o início do bloco de comentários e a sequência */ é usada para declarar o fim do comentário. Todo o texto entre as sequências inicial e final é interpretado como um comentário, mesmo que o texto seja uma sintaxe C++ válida. Às vezes, esses comentários são chamados de “estilo C”, pois essa sintaxe de comentário é herdada da linguagem predecessora de C++, C:

int main()
{
   /*
    *  This is a block comment.
    */
   int a;
}

Em qualquer comentário de bloco, você pode escrever o que quiser. Quando o compilador encontra o símbolo */, ele encerra o comentário do bloco:

int main()
{
   /* A block comment with the symbol /*
      Note that the compiler is not affected by the second /*
      however, once the end-block-comment symbol is reached,
      the comment ends.
   */
   int a;
}

O exemplo acima é um código C++ (e C) válido. No entanto, ter /* adicional dentro de um comentário de bloco pode resultar em um aviso em alguns compiladores.

Comentários em bloco também podem começar e terminar dentro de uma única linha. Por exemplo:

void SomeFunction(/* argument 1 */ int a, /* argument 2 */ int b);

Importância dos Comentários #

Como em todas as linguagens de programação, os comentários oferecem vários benefícios:

  • Documentação explícita do código para facilitar a leitura/manutenção
  • Explicação do propósito e funcionalidade do código
  • Detalhes sobre a história ou raciocínio por trás do código
  • Colocação de direitos autorais/licenças, notas de projeto, agradecimentos especiais, créditos de contribuidor, etc. diretamente no código fonte.

No entanto, os comentários também têm suas desvantagens:

  • Eles devem ser mantidos para refletir quaisquer alterações no código
  • Comentários excessivos tendem a tornar o código menos legível

A necessidade de comentários pode ser reduzida escrevendo um código claro e autodocumentado. Um exemplo simples é o uso de nomes explicativos para variáveis, funções e tipos. A fatoração de tarefas logicamente relacionadas em funções discretas anda de mãos dadas com isso.

Marcadores de comentários usados ​​para desabilitar o código #

Durante o desenvolvimento, os comentários também podem ser usados ​​para desabilitar rapidamente partes do código sem excluí-lo. Isso geralmente é útil para fins de teste ou depuração, mas não é um bom estilo para nada além de edições temporárias. Isso é muitas vezes referido como “comentando”.

Da mesma forma, manter versões antigas de um pedaço de código em um comentário para fins de referência é desaprovado, pois desordena os arquivos enquanto oferece pouco valor em comparação com a exploração do histórico do código por meio de um sistema de versão.

O processo de compilação C++ padrão #

O código de programa executável em C++ geralmente é produzido por um compilador.

Um compilador é um programa que traduz o código de uma linguagem de programação para outra forma que é (mais) executável diretamente para um computador. O uso de um compilador para traduzir o código é chamado de compilação.

C++ herda a forma de seu processo de compilação de sua linguagem “pai”, C. Abaixo está uma lista que mostra as quatro principais etapas de compilação em C++:

  1. O pré-processador C++ copia o conteúdo de qualquer arquivo de cabeçalho incluído no arquivo de código-fonte, gera código de macro e substitui constantes simbólicas definidas usando #define por seus valores.
  2. O arquivo de código-fonte expandido produzido pelo pré-processador C++ é compilado em linguagem assembly apropriada para a plataforma.
  3. O código montador gerado pelo compilador é montado em código objeto apropriado para a plataforma.
  4. O arquivo de código objeto gerado pelo montador é vinculado aos arquivos de código objeto para quaisquer funções de biblioteca usadas para produzir um arquivo executável.
  • Nota: alguns códigos compilados são vinculados, mas não para criar um programa final. Normalmente, esse código “vinculado” também pode ser empacotado em um formato que pode ser usado por outros programas. Esse “pacote de código empacotado e utilizável” é o que os programadores de C++ chamam de biblioteca.

Muitos compiladores C++ também podem mesclar ou desfazer a mesclagem de certas partes do processo de compilação para facilitar ou para análise adicional. Muitos programadores de C++ usarão ferramentas diferentes, mas todas as ferramentas geralmente seguirão esse processo generalizado quando estiverem envolvidas na produção de um programa.

Função #

Uma função é uma unidade de código que representa uma sequência de instruções.

As funções podem aceitar argumentos ou valores e retornar um único valor (ou não). Para usar uma função, uma chamada de função é usada em valores de argumento e o uso da própria chamada de função é substituído por seu valor de retorno.

Cada função tem uma assinatura de tipo – os tipos de seus argumentos e o tipo de seu tipo de retorno.

As funções são inspiradas nos conceitos do procedimento e da função matemática.

  • Nota: As funções C++ são essencialmente procedimentos e não seguem a definição exata ou as regras das funções matemáticas.

As funções geralmente são destinadas a executar uma tarefa específica. e pode ser chamado de outras partes de um programa. Uma função deve ser declarada e definida antes de ser chamada em outro lugar em um programa.

  • Nota: as definições de funções populares podem estar ocultas em outros arquivos incluídos (geralmente por conveniência e reutilização em muitos arquivos). Este é um uso comum de arquivos de cabeçalho.

Declaração de função #

Uma declaração de função declara a existência de uma função com seu nome e assinatura de tipo para o compilador. A sintaxe é a seguinte:

int add2(int i); // The function is of the type (int) -> (int)

No exemplo acima, a função int add2(int i) declara o seguinte para o compilador:

  • O tipo de retorno é int.
  • O nome da função é add2.
  • O número de argumentos para a função é 1:
  • O primeiro argumento é do tipo int.
  • O primeiro argumento será referido no conteúdo da função pelo nome i.

O nome do argumento é opcional; a declaração para a função também pode ser a seguinte:

int add2(int); // Omitting the function arguments' name is also permitted.

De acordo com a regra de uma definição, uma função com uma determinada assinatura de tipo só pode ser declarada ou definida uma vez em uma base de código C++ inteira visível para o compilador C++. Em outras palavras, funções com uma assinatura de tipo específica não podem ser redefinidas – elas devem ser definidas apenas uma vez. Assim, o seguinte não é C++ válido:

int add2(int i);  // The compiler will note that add2 is a function (int) -> int
int add2(int j);  // As add2 already has a definition of (int) -> int, the compiler
                  // will regard this as an error.

Se uma função não retornar nada, seu tipo de retorno será escrito como void. Se não receber parâmetros, a lista de parâmetros deve estar vazia.

void do_something(); // The function takes no parameters, and does not return anything.
                     // Note that it can still affect variables it has access to.

Chamada de Função #

Uma função pode ser chamada após ser declarada. Por exemplo, o programa a seguir chama add2 com o valor de 2 dentro da função de main:

#include <iostream>

int add2(int i);    // Declaration of add2

// Note: add2 is still missing a DEFINITION.
// Even though it doesn't appear directly in code,
// add2's definition may be LINKED in from another object file.

int main()
{
    std::cout << add2(2) << "\n";  // add2(2) will be evaluated at this point,
                                   // and the result is printed.
    return 0;  
}

Aqui, add2(2) é a sintaxe para uma chamada de função.

Definição de função #

Uma definição de função* é semelhante a uma declaração, exceto que também contém o código que é executado quando a função é chamada dentro de seu corpo.

Um exemplo de definição de função para add2 pode ser:

int add2(int i)       // Data that is passed into (int i) will be referred to by the name i
{                     // while in the function's curly brackets or "scope."
                    
    int j = i + 2;    // Definition of a variable j as the value of i+2.
    return j;         // Returning or, in essence, substitution of j for a function call to
                      // add2.
}

Sobrecarga de funções #

Você pode criar várias funções com o mesmo nome, mas com parâmetros diferentes.

int add2(int i)           // Code contained in this definition will be evaluated
{                         // when add2() is called with one parameter.
    int j = i + 2;
    return j;
}

int add2(int i, int j)    // However, when add2() is called with two parameters, the
{                         // code from the initial declaration will be overloaded,
    int k = i + j + 2 ;   // and the code in this declaration will be evaluated
    return k;             // instead.
}

Ambas as funções são chamadas pelo mesmo nome add2, mas a função real que é chamada depende diretamente da quantidade e do tipo dos parâmetros na chamada. Na maioria dos casos, o compilador C++ pode calcular qual função chamar. Em alguns casos, o tipo deve ser declarado explicitamente.

Parâmetros padrão #

Valores padrão para parâmetros de função só podem ser especificados em declarações de função.

int multiply(int a, int b = 7); // b has default value of 7.
int multiply(int a, int b)
{
    return a * b;               // If multiply() is called with one parameter, the
}                               // value will be multiplied by the default, 7.

Neste exemplo, multiply() pode ser chamado com um ou dois parâmetros. Se apenas um parâmetro for fornecido, b terá o valor padrão de 7. Os argumentos padrão devem ser colocados nos últimos argumentos da função. Por exemplo:

int multiply(int a = 10, int b = 20); // This is legal 
int multiply(int a = 10, int b);      // This is illegal since int a is in the former

Chamadas de Função Especial - Operadores #

Existem chamadas de funções especiais em C++ que possuem sintaxe diferente de name_of_function(value1, value2, value3). O exemplo mais comum é o dos operadores.

Certas seqüências de caracteres especiais que serão reduzidas a chamadas de função pelo compilador, como !, +, -, *, % e << e muito mais. Esses caracteres especiais são normalmente associados ao uso de não programação ou são usados ​​para estética (por exemplo, o caractere + é comumente reconhecido como o símbolo de adição tanto na programação C++ quanto na matemática elementar).

C++ trata essas seqüências de caracteres com uma sintaxe especial; mas, em essência, cada ocorrência de um operador é reduzida a uma chamada de função. Por exemplo, a seguinte expressão C++:

3+3

é equivalente à seguinte chamada de função:

operator+(3, 3)

Todos os nomes de funções de operador começam com operator.

Enquanto no predecessor imediato de C++, C, nomes de funções de operador não podem ser atribuídos a significados diferentes fornecendo definições adicionais com assinaturas de tipo diferentes, em C++ isso é válido. “Ocultar” definições de função adicionais em um nome de função exclusivo é chamado de sobrecarga de operador em C++ e é uma convenção relativamente comum, mas não universal, em C++.

Visibilidade de protótipos e declarações de funções #

Em C++, o código deve ser declarado ou definido antes do uso. Por exemplo, o seguinte produz um erro de tempo de compilação:

int main()
{
  foo(2); // error: foo is called, but has not yet been declared
}

void foo(int x) // this later definition is not known in main
{
}

Existem duas maneiras de resolver isso: colocando a definição ou declaração de foo() antes de seu uso em main(). Aqui está um exemplo:

void foo(int x) {}  //Declare the foo function and body first

int main()
{
  foo(2); // OK: foo is completely defined beforehand, so it can be called here.
}

No entanto, também é possível “declarar adiante” a função colocando apenas uma declaração de “protótipo” antes de seu uso e, em seguida, definindo o corpo da função posteriormente:

void foo(int);  // Prototype declaration of foo, seen by main
                // Must specify return type, name, and argument list types
int main()
{
  foo(2); // OK: foo is known, called even though its body is not yet defined
}

void foo(int x) //Must match the prototype
{
    // Define body of foo here
}

O protótipo deve especificar o tipo de retorno (void), o nome da função (foo) e os tipos de variáveis ​​da lista de argumentos (int), mas os nomes dos argumentos NÃO são obrigatórios .

Uma maneira comum de integrar isso na organização dos arquivos de origem é criar um arquivo de cabeçalho contendo todas as declarações de protótipo:

// foo.h
void foo(int); // prototype declaration

e, em seguida, forneça a definição completa em outro lugar:

// foo.cpp --> foo.o
#include "foo.h" // foo's prototype declaration is "hidden" in here
void foo(int x) { } // foo's body definition

e então, uma vez compilado, vincule o arquivo objeto correspondente foo.o ao arquivo objeto compilado onde ele é usado na fase de ligação, main.o:

// main.cpp --> main.o
#include "foo.h" // foo's prototype declaration is "hidden" in here
int main() { foo(2); } // foo is valid to call because its prototype declaration was beforehand.
// the prototype and body definitions of foo are linked through the object files

Um erro de “símbolo externo não resolvido” ocorre quando a função prototype e call existem, mas a função body não está definida. Isso pode ser mais complicado de resolver, pois o compilador não relatará o erro até o estágio final de vinculação e não sabe para qual linha pular no código para mostrar o erro.

Pré-processador #

O pré-processador é uma parte importante do compilador.

Ele edita o código-fonte, cortando alguns bits, alterando outros e adicionando outras coisas.

Nos arquivos de origem, podemos incluir diretivas de pré-processador. Essas diretivas instruem o pré-processador a executar ações específicas. Uma diretiva começa com um # em uma nova linha. Exemplo:

#define ZERO 0

A primeira diretiva de pré-processador que você encontrará é provavelmente a

#include <something>

diretiva. O que ele faz é pegar tudo de ‘algo’ e inseri-lo em seu arquivo onde a diretiva estava. O programa hello world começa com a linha

#include <iostream>

Esta linha adiciona as funções e objetos que permitem usar a entrada e saída padrão.

A linguagem C, que também usa o pré-processador, não possui tantos arquivos de cabeçalho quanto a linguagem C++, mas em C++ você pode usar todos os arquivos de cabeçalho C.

A próxima directiva importante é provavelmente a

#define something something_else

diretiva. Isso diz ao pré-processador que, à medida que avança no arquivo, ele deve substituir todas as ocorrências de algo por algo_outro. Ele também pode tornar as coisas semelhantes a funções, mas isso provavelmente conta como C++ avançado.

O something_else não é necessário, mas se você definir something como nada, então fora das diretivas do pré-processador, todas as ocorrências de something desaparecerão.

Isso realmente é útil, por causa das diretivas #if,#else e ​​#ifdef. O formato para estes seria o seguinte:

#if something==true
//code
#else
//more code
#endif

#ifdef thing_that_you_want_to_know_if_is_defined
//code
#endif

Essas diretivas inserem o código que está no bit verdadeiro e excluem os bits falsos. isso pode ser usado para ter bits de código que são incluídos apenas em determinados sistemas operacionais, sem ter que reescrever todo o código.
