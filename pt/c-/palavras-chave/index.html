As palavras-chave têm um significado fixo definido pelo padrão C++ e não podem ser usadas como identificadores. É ilegal redefinir palavras-chave usando o pré-processador em qualquer unidade de tradução que inclua um cabeçalho de biblioteca padrão. No entanto, as palavras-chave perdem seu significado especial dentro dos atributos.

Sintaxe #

  • asm (string-literal);
  • noexcept(expression) // significa 1
  • noexcept(constant-expression) // significando 2
  • noexcept // significa 2
  • sizeof expressão unária
  • sizeof(type-id)
  • sizeof…(identifier) // desde C++11
  • typename nested-name-specifier identifier // significa 1
  • typename nested-name-specifier template(opt) simple-template-id // significa 1
  • typename identifier(opt) // significa 2
  • typename… identifier(opt) // significando 2; desde C++11
  • typename identifier(opt) = type-id // significa 2
  • template <template-parameter-list> typename …(opt) identifier(opt) // significando 3
  • template <template-parameter-list> typename identifier(opt) = id-expression // significando 3

Os tokens final e override não são palavras-chave. Eles podem ser usados ​​como identificadores e têm significado especial apenas em determinados contextos.

Os tokens and, and_eq, bitand, bitor, compl, not, not_eq, or, or_eq, xor e xor_eq são grafias alternativas de &&, &=, &, |, ~, !, !=, ||, |=, ^ e ^=, respectivamente. O padrão não as trata como palavras-chave, mas são palavras-chave para todos os efeitos, pois é impossível redefini-las ou usá-las para significar algo diferente dos operadores que elas representam.

asm #

A palavra-chave asm recebe um único operando, que deve ser uma string literal. Ele tem um significado definido pela implementação, mas normalmente é passado para o montador da implementação, com a saída do montador sendo incorporada à unidade de tradução.

A instrução asm é uma definição, não uma expressão, então ela pode aparecer tanto no escopo do bloco quanto no escopo do namespace (incluindo escopo global). No entanto, como o assembly embutido não pode ser restringido pelas regras da linguagem C++, asm pode não aparecer dentro de uma função constexpr.

Exemplo:

[[noreturn]] void halt_system() {
    asm("hlt");
}

explícito #

  1. Quando aplicado a um construtor de argumento único, impede que esse construtor seja usado para realizar conversões implícitas.

    class MyVector {
  public:
    explicit MyVector(uint64_t size);
};
MyVector v1(100);  // ok
uint64_t len1 = 100;
MyVector v2{len1}; // ok, len1 is uint64_t
int len2 = 100;
MyVector v3{len2}; // ill-formed, implicit conversion from int to uint64_t

Como o C++11 introduziu as listas de inicializadores, no C++11 e posteriores, explicit pode ser aplicado a um construtor com qualquer número de argumentos, com o mesmo significado que no caso de argumento único.

   struct S {
       explicit S(int x, int y);
   };
   S f() {
       return {12, 34};  // ill-formed
       return S{12, 34}; // ok
   }

  1. Quando aplicado a uma função de conversão, impede que essa função de conversão seja usada para realizar conversões implícitas.

    class C {
    const int x;
  public:
    C(int x) : x(x) {}
    explicit operator int() { return x; }
};
C c(42);
int x = c;                   // ill-formed
int y = static_cast<int>(c); // ok; explicit conversion

exceto #

  1. Um operador unário que determina se a avaliação de seu operando pode propagar uma exceção. Observe que os corpos das funções chamadas não são examinados, então noexcept pode gerar falsos negativos. O operando não é avaliado.

    #include <iostream>
#include <stdexcept>
void foo() { throw std::runtime_error("oops"); }
void bar() {}
struct S {};
int main() {
    std::cout << noexcept(foo()) << '\n'; // prints 0
    std::cout << noexcept(bar()) << '\n'; // prints 0
    std::cout << noexcept(1 + 1) << '\n'; // prints 1
    std::cout << noexcept(S()) << '\n';   // prints 1
}

Neste exemplo, mesmo que bar() nunca possa lançar uma exceção, noexcept(bar()) ainda é falso porque o fato de que bar() não pode propagar uma exceção não foi explicitamente especificado.

  1. Ao declarar uma função, especifica se a função pode ou não propagar uma exceção. Sozinho, ele declara que a função não pode propagar uma exceção. Com um argumento entre parênteses, ele declara que a função pode ou não propagar uma exceção dependendo do valor de verdade do argumento.

    void f1() { throw std::runtime_error("oops"); }
void f2() noexcept(false) { throw std::runtime_error("oops"); }
void f3() {}
void f4() noexcept {}
void f5() noexcept(true) {}
void f6() noexcept {
    try {
        f1();
    } catch (const std::runtime_error&) {}
}

Neste exemplo, declaramos que f4, f5 e f6 não podem propagar exceções. (Embora uma exceção possa ser lançada durante a execução de f6, ela é capturada e não tem permissão para se propagar para fora da função.) Declaramos que f2 pode propagar uma exceção. Quando o especificador noexcept é omitido, ele é equivalente a noexcept(false), então declaramos implicitamente que f1 e f3 podem propagar exceções, mesmo que exceções não possam ser lançadas durante a execução de f3 .

Se uma função é noexcept ou não é parte do tipo da função: ou seja, no exemplo acima, f1, f2 e f3 têm tipos diferentes de f4, f5 e f6. Portanto, noexcept também é significativo em ponteiros de função, argumentos de modelo e assim por diante.

void g1() {}
void g2() noexcept {}
void (*p1)() noexcept = &g1; // ill-formed, since g1 is not noexcept
void (*p2)() noexcept = &g2; // ok; types match
void (*p3)() = &g1;          // ok; types match
void (*p4)() = &g2;          // ok; implicit conversion

Digite o nome #

  1. Quando seguido por um nome qualificado, typename especifica que é o nome de um tipo. Isso geralmente é necessário em modelos, em particular, quando o especificador de nome aninhado é um tipo dependente diferente da instanciação atual. Neste exemplo, std::decay<T> depende do parâmetro do template T, então para nomear o tipo aninhado type, precisamos prefixar todo o nome qualificado com typename. Para obter mais detalhes, consulte [Onde e por que devo colocar as palavras-chave “modelo” e “nome do tipo”?](http://stackoverflow.com/questions/610245/where-and-why-do-i-have- para-colocar-o-modelo-e-nome-tipo-palavras-chave)

    template <class T>
auto decay_copy(T&& r) -> typename std::decay<T>::type;

  2. Introduz um parâmetro de tipo na declaração de um template. Neste contexto, é intercambiável com class.

    template <typename T>
const T& min(const T& x, const T& y) {
    return b < a ? b : a;
}

  1. typename também pode ser usado ao declarar um template template parameter, precedendo o nome do parâmetro, assim como class.

    template <template <class T> typename U>
void f() {
    U<int>::do_it();
    U<double>::do_it();
}

tamanho de #

Um operador unário que fornece o tamanho em bytes de seu operando, que pode ser uma expressão ou um tipo. Se o operando for uma expressão, ele não será avaliado. O tamanho é uma expressão constante do tipo std::size_t.

Se o operando for um tipo, ele deve ser colocado entre parênteses.

  • É ilegal aplicar sizeof a um tipo de função.
  • É ilegal aplicar sizeof a um tipo incompleto, incluindo void.
  • Se sizeof for aplicado a um tipo de referência T& ou T&&, é equivalente a sizeof(T).
  • Quando sizeof é aplicado a um tipo de classe, ele produz o número de bytes em um objeto completo desse tipo, incluindo quaisquer bytes de preenchimento no meio ou no final. Portanto, uma expressão sizeof nunca pode ter o valor 0. Consulte layout of object types para mais detalhes.
  • Os tipos char, signed char e unsigned char têm um tamanho de 1. Por outro lado, um byte é definido como a quantidade de memória necessária para armazenar um objeto char. Não significa necessariamente 8 bits, pois alguns sistemas possuem objetos char maiores que 8 bits.

Se expr for uma expressão, sizeof(expr) é equivalente a sizeof(T) onde T é o tipo de expr.

int a[100];
std::cout << "The number of bytes in `a` is: " << sizeof a;
memset(a, 0, sizeof a); // zeroes out the array

O operador sizeof... produz o número de elementos em um pacote de parâmetros.

template <class... T>
void f(T&&...) {
    std::cout << "f was called with " << sizeof...(T) << " arguments\n";
}

este ponteiro

  1. O ponteiro this é um ponteiro acessível apenas dentro das funções de membro não estático de uma classe, estrutura ou tipo de união. Ele aponta para o objeto para o qual a função membro é chamada. As funções de membro estáticas não têm um ponteiro this.

     this->member-identifier

O ponteiro this de um objeto não faz parte do próprio objeto; ele não é refletido no resultado de uma instrução sizeof no objeto. Em vez disso, quando uma função membro não estática é chamada para um objeto, o endereço do objeto é passado pelo compilador como um argumento oculto para a função. Por exemplo, a seguinte chamada de função:

myDate.setMonth( 3 );  

can be interpreted this way:


setMonth( &myDate, 3 );  

The object's address is available from within the member function as the this pointer. Most uses of this are implicit. It is legal, though unnecessary, to explicitly use this when referring to members of the class. For example:


void Date::setMonth( int mn )  
{  
   month = mn;            // These three statements  
   this->month = mn;      // are equivalent  
   (*this).month = mn;  
}  

The expression *this is commonly used to return the current object from a member function:


return *this;  

The this pointer is also used to guard against self-reference:


if (&Object != this) {  
// do not execute in cases of self-reference

** Instruções try, throw e catch (C++)**

  1. Para implementar o tratamento de exceções em C++, use as expressões try, throw e catch.

  2. Primeiro, use um bloco try para incluir uma ou mais instruções que possam lançar uma exceção.

  3. Uma expressão throw sinaliza que uma condição excepcional – geralmente, um erro – ocorreu em um bloco try. Você pode usar um objeto de qualquer tipo como o operando de uma expressão throw. Normalmente, esse objeto é usado para comunicar informações sobre o erro. Na maioria dos casos, recomendamos que você use a classe std::exception ou uma das classes derivadas definidas na biblioteca padrão. Se um deles não for apropriado, recomendamos derivar sua própria classe de exceção de std::exception.

  4. Para lidar com exceções que podem ser lançadas, implemente um ou mais blocos catch imediatamente após um bloco try. Cada bloco catch especifica o tipo de exceção que ele pode manipular.

     MyData md;

    try {
    // Code that could throw an exception
    md = GetNetworkResource();
    }
    catch (const networkIOException& e) {
    // Code that executes when an exception of type
    // networkIOException is thrown in the try block
    // …
    // Log error message in the exception object
    cerr « e.what();
    }
    catch (const myDataFormatException& e) {
    // Code that handles another exception type
    // …
    cerr « e.what();
    }

    // The following syntax shows a throw expression
    MyData GetNetworkResource()
    {
    // …
    if (IOSuccess == false)
    throw networkIOException(“Unable to connect”);
    // …
    if (readError)
    throw myDataFormatException(“Format error”);
    // …
    }

O código após a cláusula try é a seção protegida do código. A expressão throw lança – ou seja, gera – uma exceção. O bloco de código após a cláusula catch é o manipulador de exceção. Este é o manipulador que captura a exceção lançada se os tipos nas expressões throw e catch forem compatíveis.

    try {  
   throw CSomeOtherException();  
}  
catch(...) {  
   // Catch all exceptions – dangerous!!!  
   // Respond (perhaps only partially) to the exception, then  
   // re-throw to pass the exception to some other handler  
   // ...  
   throw;  
}

amigo (C++)

  1. Em algumas circunstâncias, é mais conveniente conceder acesso em nível de membro a funções que não são membros de uma classe ou a todos os membros de uma classe separada. Somente o implementador da classe pode declarar quem são seus amigos. Uma função ou classe não pode se declarar como amiga de nenhuma classe. Em uma definição de classe, use a palavra-chave friend e o nome de uma função não membro ou outra classe para conceder acesso aos membros privados e protegidos de sua classe. Em uma definição de modelo, um parâmetro de tipo pode ser declarado como amigo.

  2. Se você declarar uma função de amigo que não foi declarada anteriormente, essa função será exportada para o escopo não-classe delimitador.

     class friend F  
 friend F;
 class ForwardDeclared;// Class name is known.  
 class HasFriends  
 {  
    friend int ForwardDeclared::IsAFriend();// C2039 error expected  
 };

funções de amigo

  1. Uma função de amigo é uma função que não é membro de uma classe, mas tem acesso aos membros privados e protegidos da classe. As funções de amigo não são consideradas membros de classe; são funções externas normais que recebem privilégios de acesso especiais.

  2. Amigos não estão no escopo da classe e não são chamados usando os operadores de seleção de membros (. e –>), a menos que sejam membros de outra classe.

  3. Uma função de amigo é declarada pela classe que está concedendo acesso. A declaração de amigo pode ser colocada em qualquer lugar na declaração de classe. Ele não é afetado pelas palavras-chave de controle de acesso.

     #include <iostream>  

 using namespace std;  
 class Point  
 {  
     friend void ChangePrivate( Point & );  
 public:  
     Point( void ) : m_i(0) {}  
     void PrintPrivate( void ){cout << m_i << endl; }  

 private:  
 int m_i;  
 };  

 void ChangePrivate ( Point &i ) { i.m_i++; }  

 int main()  
 {  
    Point sPoint;  
    sPoint.PrintPrivate();  
    ChangePrivate(sPoint);  
    sPoint.PrintPrivate();  
     // Output: 0  
            1  
 }
