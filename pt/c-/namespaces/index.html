Usado para evitar colisões de nomes ao usar várias bibliotecas, um namespace é um prefixo declarativo para funções, classes, tipos etc.

Sintaxe #

  • namespace identifier(opt) { declaration-seq }
  • namespace embutido identifier(opt) { declaration-seq } /* desde C++11 */
  • inline(opt) namespace attribute-specifier-seq identifier(opt) { declaration-seq } /* desde C++17 */
  • namespace inclusing-namespace-specifier :: identifier { declaration-seq } /* desde C++ 17 */
  • namespace identificador = qualified-namespace-specifier;
  • usando namespace nested-name-specifier(opt) namespace-name;
  • attribute-specifier-seq usando namespace nested-name-specifier(opt) namespace-name; /* desde C++11 */

A palavra-chave namespace tem três significados diferentes, dependendo do contexto:

  1. Quando seguido por um nome opcional e uma sequência de declarações entre chaves, ele define um novo namespace ou estende um namespace existente namespace com essas declarações. Se o nome for omitido, o namespace será um namespace sem nome.

  2. Quando seguido por um nome e um sinal de igual, ele declara um alias de namespace.

  3. Quando precedido por using e seguido por um nome de namespace, ele forma uma [* diretiva using *](https://www.wikiod.com/pt/docs/c%2b%2b/495/namespaces/1624/using-namespaces #t=201608060458446751916), que permite que nomes no namespace fornecido sejam encontrados por pesquisa de nome não qualificado (mas não redeclara esses nomes no escopo atual). Uma diretiva de uso não pode ocorrer no escopo da classe.

usar namespace std; é desencorajado. Por quê? Porque namespace std é enorme! Isso significa que há uma grande chance de que os nomes colidam:

//Really bad!
using namespace std;

//Calculates p^e and outputs it to std::cout
void pow(double p, double e) { /*...*/ }

//Calls pow
pow(5.0, 2.0); //Error! There is already a pow function in namespace std with the same signature,
               //so the call is ambiguous

O que são namespaces? #

Um namespace C++ é uma coleção de entidades C++ (funções, classes, variáveis), cujos nomes são prefixados pelo nome do namespace. Ao escrever código em um namespace, as entidades nomeadas pertencentes a esse namespace não precisam ser prefixadas com o nome do namespace, mas as entidades fora dele devem usar o nome totalmente qualificado. O nome totalmente qualificado tem o formato <namespace>::<entity>. Exemplo:

namespace Example
{
  const int test = 5;

  const int test2 = test + 12; //Works within `Example` namespace
}

const int test3 = test + 3; //Fails; `test` not found outside of namespace.

const int test3 = Example::test + 3; //Works; fully qualified name used.

Os namespaces são úteis para agrupar definições relacionadas. Tomemos a analogia de um shopping center. Geralmente um shopping é dividido em várias lojas, cada loja vendendo itens de uma categoria específica. Uma loja pode vender eletrônicos, enquanto outra loja pode vender sapatos. Essas separações lógicas nos tipos de loja ajudam os compradores a encontrar os itens que estão procurando. Os namespaces ajudam programadores de C++, como compradores, a encontrar as funções, classes e variáveis ​​que estão procurando organizando-as de maneira lógica. Exemplo:

namespace Electronics
{
    int TotalStock;
    class Headphones
    {
        // Description of a Headphone (color, brand, model number, etc.)
    };
    class Television
    {
        // Description of a Television (color, brand, model number, etc.)
    };
}

namespace Shoes
{
    int TotalStock;
    class Sandal
    {
        // Description of a Sandal (color, brand, model number, etc.)
    };
    class Slipper
    {
        // Description of a Slipper (color, brand, model number, etc.)
    };
}

Há um único namespace predefinido, que é o namespace global que não tem nome, mas pode ser denotado por ::. Exemplo:

void bar() {
    // defined in global namespace
}
namespace foo {
    void bar() {
        // defined in namespace foo
    }
    void barbar() {
        bar();   // calls foo::bar()
        ::bar(); // calls bar() defined in global namespace
    }
}

Pesquisa dependente de argumento #

Ao chamar uma função sem um qualificador de namespace explícito, o compilador pode optar por chamar uma função em um namespace se um dos tipos de parâmetro para essa função também estiver nesse namespace. Isso é chamado de “Pesquisa Dependente de Argumento” ou ADL:

namespace Test
{
  int call(int i);

  class SomeClass {...};

  int call_too(const SomeClass &data);
}

call(5); //Fails. Not a qualified function name.

Test::SomeClass data;

call_too(data); //Succeeds

call falha porque nenhum de seus tipos de parâmetro vem do namespace Test. call_too funciona porque SomeClass é membro de Test e, portanto, se qualifica para regras ADL.

Quando a ADL não ocorre #

ADL não ocorre se a pesquisa normal não qualificada encontrar um membro de classe, uma função que foi declarada no escopo do bloco ou algo que não seja do tipo de função. Por exemplo:

void foo();
namespace N {
    struct X {};
    void foo(X ) { std::cout << '1'; }
    void qux(X ) { std::cout << '2'; }
}

struct C {
    void foo() {}
    void bar() {
        foo(N::X{}); // error: ADL is disabled and C::foo() takes no arguments
    }
};

void bar() {
    extern void foo(); // redeclares ::foo
    foo(N::X{});       // error: ADL is disabled and ::foo() doesn't take any arguments
}

int qux;

void baz() {
    qux(N::X{}); // error: variable declaration disables ADL for "qux"
}

Estendendo namespaces #

Um recurso útil de namespaces é que você pode expandi-los (adicionar membros a ele).

namespace Foo
{
    void bar() {}
}

//some other stuff

namespace Foo
{
    void bar2() {}
}

Usando diretiva #

A palavra-chave ‘using’ tem três variações. Combinado com a palavra-chave ’namespace’, você escreve uma ‘diretiva de uso’:

Se você não quiser escrever Foo:: na frente de todas as coisas no namespace Foo, você pode usar using namespace Foo; para importar cada coisa de Foo.

namespace Foo
{
    void bar() {}
    void baz() {}
}

//Have to use Foo::bar()
Foo::bar();

//Import Foo
using namespace Foo;
bar(); //OK
baz(); //OK

Também é possível importar entidades selecionadas em um namespace em vez de todo o namespace:

using Foo::bar;
bar(); //OK, was specifically imported
baz(); // Not OK, was not imported

Uma palavra de cautela: usar namespaces em arquivos de cabeçalho é visto como um estilo ruim na maioria dos casos. Se isso for feito, o namespace é importado em todos os arquivos que incluem o cabeçalho. Como não há como “des-usar” um namespace, isso pode levar à poluição do namespace (mais símbolos ou símbolos inesperados no namespace global) ou, pior, a conflitos. Veja este exemplo para uma ilustração do problema:

/***** foo.h *****/
namespace Foo
{
    class C;
}

/***** bar.h *****/
namespace Bar
{
    class C;
}

/***** baz.h *****/
#include "foo.h"
using namespace Foo;

/***** main.cpp *****/
#include "bar.h"
#include "baz.h"

using namespace Bar;
C c; // error: Ambiguity between Bar::C and Foo::C

Uma diretiva de uso não pode ocorrer no escopo da classe.

Criando namespaces #

Criar um namespace é muito fácil:

//Creates namespace foo
namespace Foo
{
    //Declares function bar in namespace foo
    void bar() {}
}

Para chamar bar, você deve especificar primeiro o namespace, seguido pelo operador de resolução de escopo :::

Foo::bar();

É permitido criar um namespace em outro, por exemplo:

namespace A
{
    namespace B
    {
        namespace C
        {
            void bar() {}
        }
    }
}

O código acima pode ser simplificado para o seguinte:

namespace A::B::C
{
    void bar() {}
}

Espaços de nomes sem nome/anônimos #

Um namespace sem nome pode ser usado para garantir que os nomes tenham ligação interna (só pode ser referenciado pela unidade de tradução atual). Esse namespace é definido da mesma forma que qualquer outro namespace, mas sem o nome:

namespace {
    int foo = 42;
}

foo só é visível na unidade de tradução em que aparece.

Recomenda-se nunca usar namespaces sem nome em arquivos de cabeçalho, pois isso fornece uma versão do conteúdo para cada unidade de tradução em que está incluído. Isso é especialmente importante se você definir globais não const.

// foo.h
namespace {
    std::string globalString;
}

// 1.cpp
#include "foo.h" //< Generates unnamed_namespace{1.cpp}::globalString ...

globalString = "Initialize";

// 2.cpp
#include "foo.h" //< Generates unnamed_namespace{2.cpp}::globalString ...

std::cout << globalString; //< Will always print the empty string

Espaços de nomes aninhados compactos #

namespace a {
  namespace b {
    template<class T>
    struct qualifies : std::false_type {};
  }
}

namespace other {
  struct bob {};
}

namespace a::b {
  template<>
  struct qualifies<::other::bob> : std::true_type {};
}

Você pode inserir os namespaces a e b em uma única etapa com o namespace a::b começando em C++17.

Espaço de nomes embutido #

namespace inline inclui o conteúdo do namespace embutido no namespace delimitador, então

namespace Outer
{
    inline namespace Inner
    {
        void foo();
    }
}

é principalmente equivalente a

namespace Outer
{

    namespace Inner
    {
        void foo();
    }

    using Inner::foo;
}

mas o elemento de Outer::Inner:: e aqueles associados a Outer:: são idênticos.

Então o seguinte é equivalente

Outer::foo();
Outer::Inner::foo();

A alternativa using namespace Inner; não seria equivalente para algumas partes complicadas como especialização de template:

Por

#include <outer.h> // See below

class MyCustomType;
namespace Outer
{
    template <>
    void foo<MyCustomType>() { std::cout << "Specialization"; }
}

  • O namespace inline permite a especialização de Outer::foo

    // outer.h
// include guard omitted for simplification

namespace Outer
{
    inline namespace Inner
    {
        template <typename T>
        void foo() { std::cout << "Generic"; }
    }
}

  • Considerando que o using namespace não permite a especialização de Outer::foo

    // outer.h
// include guard omitted for simplification

namespace Outer
{
    namespace Inner
    {
        template <typename T>
        void foo() { std::cout << "Generic"; }
    }
    using namespace Inner;
    // Specialization of `Outer::foo` is not possible
    // it should be `Outer::Inner::foo`.
}

O namespace embutido é uma maneira de permitir que várias versões coabitem e padronizando para o inline

namespace MyNamespace
{
    // Inline the last version
    inline namespace Version2
    {
        void foo(); // New version
        void bar();
    }

    namespace Version1 // The old one
    {
        void foo();
    }

}

E com o uso

MyNamespace::Version1::foo(); // old version
MyNamespace::Version2::foo(); // new version
MyNamespace::foo();           // default version : MyNamespace::Version1::foo();

Alias ​​de namespace #

Um namespace pode receber um alias (ou seja, outro nome para o mesmo namespace) usando a sintaxe namespace identifier =. Os membros do namespace com alias podem ser acessados ​​qualificando-os com o nome do alias. No exemplo a seguir, o namespace aninhado AReallyLongName::AnotherReallyLongName é inconveniente de digitar, então a função qux declara localmente um alias N. Os membros desse namespace podem ser acessados ​​simplesmente usando N::.

namespace AReallyLongName {
    namespace AnotherReallyLongName {
        int foo();
        int bar();
        void baz(int x, int y);
    }
}
void qux() {
    namespace N = AReallyLongName::AnotherReallyLongName;
    N::baz(N::foo(), N::bar());
}
