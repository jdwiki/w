Cópia profunda e suporte para movimentação #

Se um tipo deseja ter semântica de valor e precisa armazenar objetos que são alocados dinamicamente, nas operações de cópia, o tipo precisará alocar novas cópias desses objetos. Ele também deve fazer isso para atribuição de cópia.

Esse tipo de cópia é chamado de “cópia profunda”. Ele efetivamente pega o que de outra forma seria semântica de referência e o transforma em semântica de valor:

struct Inner {int i;};

const int NUM_INNER = 5;
class Value
{
private:
  Inner *array_; //Normally has reference semantics.

public:
  Value() : array_(new Inner[NUM_INNER]){}

  ~Value() {delete[] array_;}

  Value(const Value &val) : array_(new Inner[NUM_INNER])
  {
    for(int i = 0; i < NUM_INNER; ++i)
      array_[i] = val.array_[i];
  }

  Value &operator=(const Value &val)
  {
    for(int i = 0; i < NUM_INNER; ++i)
      array_[i] = val.array_[i];
    return *this;
  }
};

A semântica de movimentação permite que um tipo como Value evite copiar verdadeiramente seus dados referenciados. Se o usuário usar o valor de uma forma que provoque um movimento, o objeto “copiado” pode ficar vazio dos dados referenciados:

struct Inner {int i;};

constexpr auto NUM_INNER = 5;
class Value
{
private:
  Inner *array_; //Normally has reference semantics.

public:
  Value() : array_(new Inner[NUM_INNER]){}

  //OK to delete even if nullptr
  ~Value() {delete[] array_;}

  Value(const Value &val) : array_(new Inner[NUM_INNER])
  {
    for(int i = 0; i < NUM_INNER; ++i)
      array_[i] = val.array_[i];
  }

  Value &operator=(const Value &val)
  {
    for(int i = 0; i < NUM_INNER; ++i)
      array_[i] = val.array_[i];
    return *this;
  }

  //Movement means no memory allocation.
  //Cannot throw exceptions.
  Value(Value &&val) noexcept : array_(val.array_)
  {
    //We've stolen the old value.
    val.array_ = nullptr;
  }

  //Cannot throw exceptions.
  Value &operator=(Value &&val) noexcept
  {
    //Clever trick. Since `val` is going to be destroyed soon anyway,
    //we swap his data with ours. His destructor will destroy our data.
    std::swap(array_, val.array_);
  }
};

De fato, podemos até tornar esse tipo não copiável, se quisermos proibir cópias profundas enquanto ainda permitimos que o objeto seja movido.

struct Inner {int i;};

constexpr auto NUM_INNER = 5;
class Value
{
private:
  Inner *array_; //Normally has reference semantics.

public:
  Value() : array_(new Inner[NUM_INNER]){}

  //OK to delete even if nullptr
  ~Value() {delete[] array_;}

  Value(const Value &val) = delete;
  Value &operator=(const Value &val) = delete;

  //Movement means no memory allocation.
  //Cannot throw exceptions.
  Value(Value &&val) noexcept : array_(val.array_)
  {
    //We've stolen the old value.
    val.array_ = nullptr;
  }

  //Cannot throw exceptions.
  Value &operator=(Value &&val) noexcept
  {
    //Clever trick. Since `val` is going to be destroyed soon anyway,
    //we swap his data with ours. His destructor will destroy our data.
    std::swap(array_, val.array_);
  }
};

Podemos até aplicar a Regra do Zero, através do uso de unique_ptr:

struct Inner {int i;};

constexpr auto NUM_INNER = 5;
class Value
{
private:
  unique_ptr<Inner []>array_; //Move-only type.

public:
  Value() : array_(new Inner[NUM_INNER]){}

  //No need to explicitly delete. Or even declare.
  ~Value() = default; {delete[] array_;}

  //No need to explicitly delete. Or even declare.
  Value(const Value &val) = default;
  Value &operator=(const Value &val) = default;

  //Will perform an element-wise move.
  Value(Value &&val) noexcept = default;

  //Will perform an element-wise move.
  Value &operator=(Value &&val) noexcept = default;
};

Definições #

Um tipo tem semântica de valor se o estado observável do objeto for funcionalmente distinto de todos os outros objetos desse tipo. Isso significa que, se você copiar um objeto, terá um novo objeto e as modificações do novo objeto não serão visíveis de forma alguma a partir do objeto antigo.

A maioria dos tipos básicos de C++ tem semântica de valor:

int i = 5;
int j = i; //Copied
j += 20;
std::cout << i; //Prints 5; i is unaffected by changes to j.

A maioria dos tipos definidos pela biblioteca padrão também tem semântica de valor:

std::vector<int> v1(5, 12); //array of 5 values, 12 in each.
std::vector<int> v2 = v1; //Copies the vector.
v2[3] = 6; v2[4] = 9;
std::cout << v1[3] << " " << v1[4]; //Writes "12 12", since v1 is unchanged.

Diz-se que um tipo tem semântica de referência se uma instância desse tipo puder compartilhar seu estado observável com outro objeto (externo a ele), de modo que a manipulação de um objeto fará com que o estado mude dentro de outro objeto.

Os ponteiros C++ têm semântica de valor em relação ao objeto para o qual apontam, mas têm semântica de referência em relação ao estado do objeto para o qual apontam:

int *pi = new int(4);
int *pi2 = pi;
pi = new int(16);
assert(pi2 != pi); //Will always pass.

int *pj = pi;
*pj += 5;
std::cout << *pi; //Writes 9, since `pi` and `pj` reference the same object.

As referências C++ também têm semântica de referência.
