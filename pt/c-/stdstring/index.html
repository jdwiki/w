Strings são objetos que representam sequências de caracteres. A classe padrão string fornece uma alternativa simples, segura e versátil ao uso de arrays explícitos de chars ao lidar com texto e outras seqüências de caracteres. A classe C++ string faz parte do namespace std e foi padronizada em 1998.

Sintaxe #

  • // Declaração de string vazia

    std::string s;

  • // Construindo a partir de const char* (c-string)

std::string s(“Olá”);

std::string s = “Olá”;

  • // Construindo usando o construtor de cópia

std::string s1(“Olá”);

std::string s2(s1);

  • // Construindo a partir de substring

std::string s1(“Olá”);

std::string s2(s1, 0, 4); // Copia 4 caracteres da posição 0 de s1 para s2

  • // Construindo a partir de um buffer de caracteres

std::string s1(“Olá Mundo”); std::string s2(s1, 5); // Copia os primeiros 5 caracteres de s1 para s2

  • // Construir usando o construtor de preenchimento (somente char)

std::string s(5, ‘a’); // s contém aaaaa

  • // Construir usando construtor de intervalo e iterador

std::string s1(“Olá Mundo”);

std::string s2(s1.begin(), s1.begin()+5); // Copia os primeiros 5 caracteres de s1 para s2

Antes de usar std::string, você deve incluir o cabeçalho string, pois inclui funções/operadores/sobrecargas que outros cabeçalhos (por exemplo, iostream) não incluem.

Usar o construtor const char* com um nullptr leva a um comportamento indefinido.

std::string oops(nullptr);
std::cout << oops << "\n";

O método at lança uma exceção std::out_of_range se index >= size().

O comportamento de operator[] é um pouco mais complicado, em todos os casos tem comportamento indefinido se index > size(), mas quando index == size():

  1. Em uma string não const, o comportamento é undefined;
  2. Em uma string const, uma referência a um caractere com valor CharT() (o caractere null) é retornada.
  1. Uma referência a um caractere com valor CharT() (o caractere null) é retornada.
  2. Modificar esta referência é comportamento indefinido.

Desde C++14, em vez de usar "foo", é recomendado usar "foo"s, pois s é um [sufixo literal definido pelo usuário](https://www.wikiod.com/pt/docs/ /c%2b%2b/2745/user-defined-literals), que converte o const char* "foo" para std::string "foo".

Nota: você tem que usar o namespace std::string_literals ou std::literals para obter o literal s.

Tokenizar #

Listado do menos caro ao mais caro em tempo de execução:

  1. str::strtok é o método de tokenização fornecido padrão mais barato, também permite que o delimitador seja modificado entre tokens, mas incorre em 3 dificuldades com o C++ moderno:
  • std::strtok não pode ser usado em várias strings ao mesmo tempo (embora algumas implementações se estendam para suportar isso, como: [strtok_s](https://msdn.microsoft.com/en- us/library/ftsafwz3.aspx))
  • Pela mesma razão, std::strtok não pode ser usado em vários threads simultaneamente (isso pode, no entanto, ser definido pela implementação, por exemplo: [A implementação do Visual Studio é thread-safe](https://msdn.microsoft.com/en- us/library/2c8d19sb.aspx#Anchor_3))
  • Chamar std::strtok modifica o std::string em que está operando, então não pode ser usado em const strings, const char*s ou strings literais, para tokenizar qualquer um deles com std::strtok ou para operar em um std::string cujo conteúdo precisa ser preservado, a entrada teria que ser copiada, então a cópia poderia ser operada

Geralmente, o custo de qualquer uma dessas opções estará oculto no custo de alocação dos tokens, mas se o algoritmo mais barato for necessário e as dificuldades do std::strtok não forem superáveis, considere uma [solução feita à mão](http:// stackoverflow.com/a/38595708/2642059).

// String to tokenize
std::string str{ "The quick brown fox" };
// Vector to store tokens
vector<std::string> tokens;

for (auto i = strtok(&str[0], " "); i != NULL; i = strtok(NULL, " "))
    tokens.push_back(i);

Exemplo ao vivo

  1. O std::istream_iterator usa o operador de extração do stream de forma iterativa. Se a entrada std::string for delimitada por espaços em branco, isso é capaz de expandir a opção std::strtok eliminando suas dificuldades, permitindo tokenização inline suportando assim a geração de um const vector<string>, e adicionando suporte para vários caracteres de espaço em branco de delimitação:

    // String to tokenize const std::string str(“The quick \tbrown \nfox”); std::istringstream is(str); // Vector to store tokens const std::vectorstd::string tokens = std::vectorstd::string( std::istream_iteratorstd::string(is), std::istream_iteratorstd::string());

Exemplo ao vivo

  1. O std::regex_token_iterator usa um std::regex para tokenizar iterativamente. Ele fornece uma definição de delimitador mais flexível. Por exemplo, vírgulas não delimitadas e espaços em branco:
// String to tokenize
const std::string str{ "The ,qu\\,ick ,\tbrown, fox" };
const std::regex re{ "\\s*((?:[^\\\\,]|\\\\.)*?)\\s*(?:,|$)" };
// Vector to store tokens
const std::vector<std::string> tokens{ 
    std::sregex_token_iterator(str.begin(), str.end(), re, 1), 
    std::sregex_token_iterator() 
};

Exemplo ao vivo

Veja o regex_token_iterator Exemplo para mais detalhes.

Conversão para (const) char* #

Para obter acesso const char* aos dados de uma std::string você pode usar a função membro c_str() da string. Tenha em mente que o ponteiro só é válido enquanto o objeto std::string estiver dentro do escopo e permanecer inalterado, o que significa que apenas métodos const podem ser chamados no objeto.

A função membro data() pode ser usada para obter um char* modificável, que pode ser usado para manipular os dados do objeto std::string.

Um char* modificável também pode ser obtido tomando o endereço do primeiro caractere: &s[0]. No C++ 11, isso garante uma string bem formada e terminada em nulo. Observe que &s[0] é bem formado mesmo se s estiver vazio, enquanto &s.front() é indefinido se s estiver vazio.

std::string str("This is a string.");
const char* cstr = str.c_str(); // cstr points to: "This is a string.\0"
const char* data = str.data();  // data points to: "This is a string.\0"

std::string str("This is a string.");

// Copy the contents of str to untie lifetime from the std::string object
std::unique_ptr<char []> cstr = std::make_unique<char[]>(str.size() + 1);

// Alternative to the line above (no exception safety):
// char* cstr_unsafe = new char[str.size() + 1];

std::copy(str.data(), str.data() + str.size(), cstr);
cstr[str.size()] = '\0'; // A null-terminator needs to be added

// delete[] cstr_unsafe;
std::cout << cstr.get();

Usando a classe std::string_view #

C++17 introduz std::string_view, que é simplesmente um intervalo não proprietário de const chars, implementável como um par de ponteiros ou um ponteiro e um comprimento. É um tipo de parâmetro superior para funções que requerem dados de string não modificáveis. Antes do C++17, havia três opções para isso:

void foo(std::string const& s);      // pre-C++17, single argument, could incur
                                     // allocation if caller's data was not in a string
                                     // (e.g. string literal or vector<char> )

void foo(const char* s, size_t len); // pre-C++17, two arguments, have to pass them
                                     // both everywhere

void foo(const char* s);             // pre-C++17, single argument, but need to call
                                     // strlen()

template <class StringT>
void foo(StringT const& s);          // pre-C++17, caller can pass arbitrary char data
                                     // provider, but now foo() has to live in a header

Tudo isso pode ser substituído por:

void foo(std::string_view s);        // post-C++17, single argument, tighter coupling
                                     // zero copies regardless of how caller is storing
                                     // the data

Observe que std::string_view não pode modificar seus dados subjacentes.

string_view é útil quando você quer evitar cópias desnecessárias.

Ele oferece um subconjunto útil da funcionalidade que std::string faz, embora algumas das funções se comportem de forma diferente:

std::string str = "lllloooonnnngggg sssstttrrriiinnnggg"; //A really long string

//Bad way - 'string::substr' returns a new string (expensive if the string is long)
std::cout << str.substr(15, 10) << '\n';

//Good way - No copies are created!
std::string_view view = str;

// string_view::substr returns a new string_view
std::cout << view.substr(15, 10) << '\n';

Conversão para std::wstring #

Em C++, sequências de caracteres são representadas pela especialização da classe std::basic_string com um tipo de caractere nativo. As duas principais coleções definidas pela biblioteca padrão são std::string e std::wstring:

  • std::string é construído com elementos do tipo char

  • std::wstring é construído com elementos do tipo wchar_t

Para converter entre os dois tipos, use wstring_convert:

#include <string>
#include <codecvt>
#include <locale>

std::string input_str = "this is a -string-, which is a sequence based on the -char- type.";
std::wstring input_wstr = L"this is a -wide- string, which is based on the -wchar_t- type.";

// conversion
std::wstring str_turned_to_wstr = std::wstring_convert<std::codecvt_utf8<wchar_t>>().from_bytes(input_str);

std::string wstr_turned_to_str = std::wstring_convert<std::codecvt_utf8<wchar_t>>().to_bytes(input_wstr);

Para melhorar a usabilidade e/ou legibilidade, você pode definir funções para realizar a conversão:

#include <string>
#include <codecvt>
#include <locale>

using convert_t = std::codecvt_utf8<wchar_t>;
std::wstring_convert<convert_t, wchar_t> strconverter;

std::string to_string(std::wstring wstr)
{
    return strconverter.to_bytes(wstr);
}

std::wstring to_wstring(std::string str)
{
    return strconverter.from_bytes(str);
}

Exemplo de uso:

std::wstring a_wide_string = to_wstring("Hello World!");

Isso é certamente mais legível do que std::wstring_convert<std::codecvt_utf8<wchar_t>>().from_bytes("Hello World!").

Observe que char e wchar_t não implicam em codificação e não dão indicação de tamanho em bytes. Por exemplo, wchar_t é comumente implementado como um tipo de dados de 2 bytes e normalmente contém dados codificados em UTF-16 no Windows (ou UCS-2 em versões anteriores ao Windows 2000) e como um tipo de dados de 4 bytes codificados usando UTF- 32 no Linux. Isso contrasta com os tipos mais recentes char16_t e char32_t, que foram introduzidos no C++11 e são garantidos como grandes o suficiente para conter qualquer “caractere” UTF16 ou UTF32 (ou mais precisamente, ponto de código) respectivamente.

Aparar caracteres no início/fim #

Este exemplo requer os cabeçalhos <algorithm>, <locale> e <utility>.

aparar uma sequência ou string significa remover todos os elementos (ou caracteres) iniciais e finais que correspondem a um determinado predicado. Primeiro aparamos os elementos à direita, porque não envolve a movimentação de nenhum elemento e, em seguida, aparamos os elementos à esquerda. Observe que as generalizações abaixo funcionam para todos os tipos de std::basic_string (por exemplo, std::string e std::wstring), e acidentalmente também para contêineres de sequência (por exemplo, std::vector e std ::lista).

template <typename Sequence, // any basic_string, vector, list etc.
          typename Pred>     // a predicate on the element (character) type
Sequence& trim(Sequence& seq, Pred pred) {
    return trim_start(trim_end(seq, pred), pred);
}

Aparar os elementos à direita envolve encontrar o último elemento que não corresponde ao predicado e apagar a partir daí:

template <typename Sequence, typename Pred>
Sequence& trim_end(Sequence& seq, Pred pred) {
    auto last = std::find_if_not(seq.rbegin(),
                                 seq.rend(),
                                 pred);
    seq.erase(last.base(), seq.end());
    return seq;
}

Aparar os elementos principais envolve encontrar o primeiro elemento que não corresponde ao predicado e apagar até lá:

template <typename Sequence, typename Pred>
Sequence& trim_start(Sequence& seq, Pred pred) {
    auto first = std::find_if_not(seq.begin(),
                                  seq.end(),
                                  pred);
    seq.erase(seq.begin(), first);
    return seq;
}

Para especializar o acima para cortar espaços em branco em um std::string, podemos usar a função std::isspace() como um predicado:

std::string& trim(std::string& str, const std::locale& loc = std::locale()) {
    return trim(str, [&loc](const char c){ return std::isspace(c, loc); });
}

std::string& trim_start(std::string& str, const std::locale& loc = std::locale()) {
    return trim_start(str, [&loc](const char c){ return std::isspace(c, loc); });
}

std::string& trim_end(std::string& str, const std::locale& loc = std::locale()) {
    return trim_end(str, [&loc](const char c){ return std::isspace(c, loc); });
}

Da mesma forma, podemos usar a função std::iswspace() para std::wstring etc.

Se você deseja criar uma nova sequência que seja uma cópia cortada, você pode usar uma função separada:

template <typename Sequence, typename Pred>
Sequence trim_copy(Sequence seq, Pred pred) { // NOTE: passing seq by value
    trim(seq, pred);
    return seq;
}

Comparação lexicográfica #

Dois std::strings podem ser comparados lexicograficamente usando os operadores ==, !=, <, <=, > e >=:

std::string str1 = "Foo";
std::string str2 = "Bar";

assert(!(str1 < str2));
assert(str > str2);
assert(!(str1 <= str2));
assert(str1 >= str2);
assert(!(str1 == str2));
assert(str1 != str2);

Todas essas funções usam o método std::string::compare() subjacente para realizar a comparação e retornam por conveniência valores booleanos. A operação dessas funções pode ser interpretada da seguinte forma, independentemente da implementação real:

  • operador==:

    If str1.length() == str2.length() and each character pair matches, then returns true, otherwise returns false.

  • operador!=:

    If str1.length() != str2.length() or one character pair doesn’t match, returns true, otherwise it returns false.

  • operador< ou operador>:

    Finds the first different character pair, compares them then returns the boolean result.

  • operador<= ou operador>=:

    Finds the first different character pair, compares them then returns the boolean result.

Nota: O termo par de caracteres significa os caracteres correspondentes em ambas as strings das mesmas posições. Para melhor compreensão, se duas strings de exemplo são str1 e str2, e seus comprimentos são n e m respectivamente, então os pares de caracteres de ambas as strings significam cada str1[i] e str2[i] pares onde i = 0, 1, 2, …, max(n,m). Se para algum i onde o caractere correspondente não existir, ou seja, quando i for maior ou igual a n ou m, será considerado o menor valor.

Aqui está um exemplo de uso de <:

std::string str1 = "Barr";
std::string str2 = "Bar";

assert(str2 < str1);

Os passos são os seguintes:

  1. Compare os primeiros caracteres, 'B' == 'B' - siga em frente.
  2. Compare os segundos caracteres, 'a' == 'a' - siga em frente.
  3. Compare os terceiros caracteres, 'r' == 'r' - siga em frente.
  4. O intervalo str2 está esgotado, enquanto o intervalo str1 ainda possui caracteres. Assim, str2 < str1.

Substituição de strings #

Substitui por posição #

Para substituir uma parte de um std::string você pode usar o método replace de std::string.

replace tem muitas sobrecargas úteis:

//Define string
std::string str = "Hello foo, bar and world!";
std::string alternate = "Hello foobar";

//1)
str.replace(6, 3, "bar"); //"Hello bar, bar and world!"

//2)
str.replace(str.begin() + 6, str.end(), "nobody!"); //"Hello nobody!"

//3)
str.replace(19, 5, alternate, 6, 6); //"Hello foo, bar and foobar!"

//4)
str.replace(19, 5, alternate, 6); //"Hello foo, bar and foobar!"

//5)
str.replace(str.begin(), str.begin() + 5, str.begin() + 6, str.begin() + 9);
//"foo foo, bar and world!"

//6)
str.replace(0, 5, 3, 'z'); //"zzz foo, bar and world!"

//7)
str.replace(str.begin() + 6, str.begin() + 9, 3, 'x'); //"Hello xxx, bar and world!"

//8)
str.replace(str.begin(), str.begin() + 5, { 'x', 'y', 'z' }); //"xyz foo, bar and world!"

Substitui ocorrências de uma string por outra string #

Substitua apenas a primeira ocorrência de replace por with em str:

std::string replaceString(std::string str,
                          const std::string& replace,
                          const std::string& with){
    std::size_t pos = str.find(replace);
    if (pos != std::string::npos)
        str.replace(pos, replace.length(), with);
    return str;
}

Substitua todas as ocorrências de replace por with em str:

std::string replaceStringAll(std::string str,
                             const std::string& replace,
                             const std::string& with) {
    if(!replace.empty()) {
        std::size_t pos = 0;
        while ((pos = str.find(replace, pos)) != std::string::npos) {
            str.replace(pos, replace.length(), with);
            pos += with.length();
        }
    }
    return str;
}

Convertendo para std::string #

std::ostringstream pode ser usado para converter qualquer tipo de stream em uma representação de string, inserindo o objeto em um objeto std::ostringstream (com o operador de inserção de stream <<) e então convertendo todo o std::ostringstream para um std::string.

Para int por exemplo:

#include <sstream>

int main()
{
    int val = 4;
    std::ostringstream str;
    str << val;
    std::string converted = str.str();
    return 0;
}

Escrevendo sua própria função de conversão, o simples:

template<class T>
std::string toString(const T& x)
{
  std::ostringstream ss;
  ss << x;
  return ss.str();
}

funciona, mas não é adequado para código crítico de desempenho.

As classes definidas pelo usuário podem implementar o operador de inserção de fluxo, se desejado:

std::ostream operator<<( std::ostream& out, const A& a )
{
    // write a string representation of a to out
    return out; 
}

Além de streams, desde o C++ 11 você também pode usar a função std::to_string (e std::to_wstring) que é sobrecarregada para todos os tipos fundamentais e retorna o representação de string de seu parâmetro.

std::string s = to_string(0x12f3);  // after this the string s contains "4851"

Divisão #

Use std::string::substr para dividir uma string. Existem duas variantes dessa função de membro.

A primeira toma uma posição inicial a partir da qual a substring retornada deve começar. A posição inicial deve ser válida no intervalo (0, str.length()]:

std::string str = "Hello foo, bar and world!";
std::string newstr = str.substr(11); // "bar and world!"

O segundo assume uma posição inicial e um length total da nova substring. Independentemente do comprimento, a substring nunca passará do final da string de origem:

std::string str = "Hello foo, bar and world!";
std::string newstr = str.substr(15, 3); // "and"

Observe que você também pode chamar substr sem argumentos, neste caso uma cópia exata da string é retornada

std::string str = "Hello foo, bar and world!";
std::string newstr = str.substr(); // "Hello foo, bar and world!"

Acessando um personagem #

Existem várias maneiras de extrair caracteres de um std::string e cada uma é sutilmente diferente.

std::string str("Hello world!");

operador[](n) #

Retorna uma referência ao caractere no índice n.

std::string::operator[] não é verificado por limites e não lança uma exceção. O chamador é responsável por afirmar que o índice está dentro do intervalo da string:

char c = str[6]; // 'w'

em(n) #

Retorna uma referência ao caractere no índice n.

std::string::at is limites verificados e lançará std::out_of_range se o índice não estiver dentro do intervalo da string:

char c = str.at(7); // 'o'

Observação: ambos os exemplos resultarão em comportamento indefinido se a string estiver vazia.

frente() #

Retorna uma referência ao primeiro caractere:

char c = str.front(); // 'H'

de volta() #

Retorna uma referência ao último caractere:

char c = str.back(); // '!'

Percorrendo cada caractere #

std::string suporta iteradores, e assim você pode usar um loop baseado em intervalo para iterar através de cada caractere:

std::string str = "Hello World!";
for (auto c : str)
    std::cout << c;

Você pode usar um loop for “tradicional” para percorrer todos os caracteres:

std::string str = "Hello World!";
for (std::size_t i = 0; i < str.length(); ++i)
    std::cout << str[i];

Verificando se uma string é um prefixo de outra #

Em C++14, isso é feito facilmente por std::mismatch que retorna o primeiro par incompatível de dois intervalos:

std::string prefix = "foo";
std::string string = "foobar";

bool isPrefix = std::mismatch(prefix.begin(), prefix.end(),
    string.begin(), string.end()).first == prefix.end();

Observe que uma versão de intervalo e meio de mismatch() existia antes do C++ 14, mas isso não é seguro no caso de a segunda string ser a mais curta das duas.

Ainda podemos usar a versão range-and-a-half de std::mismatch(), mas precisamos primeiro verificar se a primeira string é no máximo tão grande quanto a segunda:

bool isPrefix = prefix.size() <= string.size() &&
    std::mismatch(prefix.begin(), prefix.end(),
        string.begin(), string.end()).first == prefix.end();

Com std::string_view, podemos escrever a comparação direta que queremos sem ter que nos preocupar com sobrecarga de alocação ou fazer cópias:

bool isPrefix(std::string_view prefix, std::string_view full)
{
    return prefix == full.substr(0, prefix.size());
}

Concatenação #

Você pode concatenar std::strings usando os operadores sobrecarregados + e +=. Usando o operador +:

std::string hello = "Hello";
std::string world = "world";
std::string helloworld = hello + world; // "Helloworld"

Usando o operador +=:

std::string hello = "Hello";
std::string world = "world";
hello += world; // "Helloworld"

Você também pode anexar strings C, incluindo literais de string:

std::string hello = "Hello";
std::string world = "world";
const char *comma = ", ";
std::string newhelloworld = hello + comma + world + "!"; // "Hello, world!"

Você também pode usar push_back() para empurrar chars individuais:

std::string s = "a, b, ";
s.push_back('c'); // "a, b, c"

Há também append(), que é muito parecido com +=:

std::string app = "test and ";
app.append("test"); // "test and test"

Conversão para inteiros/tipos de ponto flutuante #

Um std::string contendo um número pode ser convertido em um tipo inteiro, ou um tipo de ponto flutuante, usando funções de conversão.

Observe que todas essas funções param de analisar a string de entrada assim que encontram um caractere não numérico, então "123abc" será convertido em 123.

A família de funções std::ato* converte strings de estilo C (arrays de caracteres) em tipos inteiros ou de ponto flutuante:

std::string ten = "10";

double num1 = std::atof(ten.c_str());
int num2 = std::atoi(ten.c_str());
long num3 = std::atol(ten.c_str());

long long num4 = std::atoll(ten.c_str());

No entanto, o uso dessas funções é desencorajado porque elas retornam 0 se falharem ao analisar a string. Isso é ruim porque 0 também pode ser um resultado válido, se, por exemplo, a string de entrada for “0”, então é impossível determinar se a conversão realmente falhou.

A nova família de funções std::sto* converte std::strings para tipos inteiros ou de ponto flutuante e lança exceções se não puderem analisar sua entrada. Você deve usar estas funções se possível:

std::string ten = "10";

int num1 = std::stoi(ten);
long num2 = std::stol(ten);
long long num3 = std::stoll(ten);

float num4 = std::stof(ten);
double num5 = std::stod(ten);
long double num6 = std::stold(ten);

Além disso, essas funções também lidam com strings octais e hexadecimais ao contrário da família std::ato*. O segundo parâmetro é um ponteiro para o primeiro caractere não convertido na string de entrada (não ilustrado aqui) e o terceiro parâmetro é a base a ser usada. 0 é a detecção automática de octal (começando com 0) e hexadecimal (começando com 0x ou 0X), e qualquer outro valor é a base a ser usada

std::string ten = "10";
std::string ten_octal = "12";
std::string ten_hex = "0xA";

int num1 = std::stoi(ten, 0, 2); // Returns 2
int num2 = std::stoi(ten_octal, 0, 8); // Returns 10
long num3 = std::stol(ten_hex, 0, 16);  // Returns 10
long num4 = std::stol(ten_hex);  // Returns 0
long num5 = std::stol(ten_hex, 0, 0); // Returns 10 as it detects the leading 0x

Convertendo entre codificações de caracteres #

A conversão entre codificações é fácil com C++ 11 e a maioria dos compiladores é capaz de lidar com isso de maneira multiplataforma por meio dos cabeçalhos <codecvt> e <locale>.

#include <iostream>
#include <codecvt>
#include <locale>
#include <string>
using namespace std;

int main() {
    // converts between wstring and utf8 string
    wstring_convert<codecvt_utf8_utf16<wchar_t>> wchar_to_utf8;
    // converts between u16string and utf8 string
    wstring_convert<codecvt_utf8_utf16<char16_t>, char16_t> utf16_to_utf8;
    
    wstring wstr = L"foobar";
    string utf8str = wchar_to_utf8.to_bytes(wstr);
    wstring wstr2 = wchar_to_utf8.from_bytes(utf8str);
    
    wcout << wstr << endl;
    cout << utf8str << endl;
    wcout << wstr2 << endl;
    
    u16string u16str = u"foobar";
    string utf8str2 = utf16_to_utf8.to_bytes(u16str);
    u16string u16str2 = utf16_to_utf8.from_bytes(utf8str2);
    
    return 0;
}

Lembre-se de que o Visual Studio 2015 fornece suporte para essas conversões, mas um bug em sua implementação de biblioteca requer o uso de um modelo diferente para wstring_convert ao lidar com char16_t:

using utf16_char = unsigned short;
wstring_convert<codecvt_utf8_utf16<utf16_char>, utf16_char> conv_utf8_utf16;

void strings::utf16_to_utf8(const std::u16string& utf16, std::string& utf8)
{
  std::basic_string<utf16_char> tmp;
  tmp.resize(utf16.length());
  std::copy(utf16.begin(), utf16.end(), tmp.begin());
  utf8 = conv_utf8_utf16.to_bytes(tmp);
}
void strings::utf8_to_utf16(const std::string& utf8, std::u16string& utf16)
{ 
  std::basic_string<utf16_char> tmp = conv_utf8_utf16.from_bytes(utf8);
  utf16.clear();
  utf16.resize(tmp.length());
  std::copy(tmp.begin(), tmp.end(), utf16.begin());
}

Encontrando caractere(s) em uma string #

Para encontrar um caractere ou outra string, você pode usar std::string::find. Ele retorna a posição do primeiro caractere da primeira correspondência. Se nenhuma correspondência for encontrada, a função retornará std::string::npos

std::string str = "Curiosity killed the cat";
auto it = str.find("cat");

if (it != std::string::npos)
    std::cout << "Found at position: " << it << '\n';
else
    std::cout << "Not found!\n";

Encontrado na posição: 21

As oportunidades de pesquisa são expandidas ainda mais pelas seguintes funções:

find_first_of     // Find first occurrence of characters 
find_first_not_of // Find first absence of characters 
find_last_of      // Find last occurrence of characters 
find_last_not_of  // Find last absence of characters

Essas funções podem permitir que você pesquise caracteres no final da string, bem como encontre o caso negativo (ou seja, caracteres que não estão na string). Aqui está um exemplo:

std::string str = "dog dog cat cat";
std::cout << "Found at position: " << str.find_last_of("gzx") << '\n';

Encontrado na posição: 6

Observação: Esteja ciente de que as funções acima não pesquisam substrings, mas sim caracteres contidos na string de pesquisa. Neste caso, a última ocorrência de 'g' foi encontrada na posição 6 (os outros caracteres não foram encontrados).
