Classes, funções e variáveis ​​(desde C++14) podem ser modeladas. Um modelo é um pedaço de código com alguns parâmetros livres que se tornarão uma classe, função ou variável concreta quando todos os parâmetros forem especificados. Os parâmetros podem ser tipos, valores ou modelos próprios. Um modelo bem conhecido é std::vector, que se torna um tipo de contêiner concreto quando o tipo de elemento é especificado, por exemplo, std::vector<int>.

Sintaxe #

  • template < template-parameter-list > declaration
  • export template < template-parameter-list > declaration /* até C++11 */
  • modelo <> declaração
  • modelo declaração
  • modelo externo declaração /* desde C++11 */
  • template < template-parameter-list > class …(opt) identifier(opt)
  • template < template-parameter-list > class identifier(opt) = id-expression
  • template < template-parameter-list > typename …(opt) identifier(opt) /* desde C++ 17 */
  • template < template-parameter-list > typename identifier(opt) = id-expression /* desde C++17 */
  • expressão pós-fixada . modelo expressão de id
  • postfix-expression -> template id-expression
  • nested-name-specifier template simple-template-id ::

A palavra template é uma palavra-chave com cinco significados diferentes na linguagem C++, dependendo do contexto.

  1. Quando seguido por uma lista de parâmetros de modelo entre <>, ele declara um modelo como um [modelo de classe](https://www.wikiod.com/pt/docs/c%2b%2b/460/templates/3999 /basic-class-template), um modelo de função ou uma [especialização parcial](http: //stackoverflow.com/documentation/c%2b%2b/460/templates/6253/partial-template-specialization) de um modelo existente.

    template <class T>
void increment(T& x) { ++x; }

  2. Quando seguido por um vazio <>, ele declara uma [especialização explícita (completa)](https://www.wikiod.com/pt/docs/c%2b%2b/460/templates/5054/template- especialização).

    template <class T>
void print(T x);

template <> // <-- keyword used in this sense here
void print(const char* s) {
    // output the content of the string
    printf("%s\n", s);
}

  3. Quando seguido por uma declaração sem <>, forma uma instanciação explícita declaração ou definição .

    template <class T>
std::set<T> make_singleton(T x) { return std::set<T>(x); }

template std::set<int> make_singleton(int x); // <-- keyword used in this sense here

  4. Dentro de uma lista de parâmetros de modelo, ele apresenta um parâmetro de modelo de modelo.

    template <class T, template <class U> class Alloc>
//                 ^^^^^^^^ keyword used in this sense here
class List {
    struct Node {
        T value;
        Node* next;
    };
    Alloc<Node> allocator;
    Node* allocate_node() {
        return allocator.allocate(sizeof(T));
    }
    // ...
};

  5. Após o operador de resolução de escopo :: e os operadores de acesso de membro de classe . e ->, ele especifica que o nome a seguir é um modelo.

    struct Allocator {
    template <class T>
    T* allocate();
};

template <class T, class Alloc>
class List {
    struct Node {
        T value;
        Node* next;
    }
    Alloc allocator;
    Node* allocate_node() {
        // return allocator.allocate<Node>();       // error: < and > are interpreted as
                                                    // comparison operators
        return allocator.template allocate<Node>(); // ok; allocate is a template
        //               ^^^^^^^^ keyword used in this sense here
    }
};

Antes do C++11, um modelo podia ser declarado com a [palavra-chave] export (https://www.wikiod.com/pt/docs/c%2b%2b/4891/keywords), transformando-o em um modelo exportado. A definição de um modelo exportado não precisa estar presente em todas as unidades de tradução nas quais o modelo é instanciado. Por exemplo, o seguinte deveria funcionar:

foo.h:

#ifndef FOO_H
#define FOO_H
export template <class T> T identity(T x);
#endif

foo.cpp:

#include "foo.h"
template <class T> T identity(T x) { return x; }

main.cpp:

#include "foo.h"
int main() {
    const int x = identity(42); // x is 42
}

Devido à dificuldade de implementação, a palavra-chave export não era suportada pela maioria dos principais compiladores. Foi removido em C++11; agora, é ilegal usar a palavra-chave export. Em vez disso, geralmente é necessário definir modelos nos cabeçalhos (em contraste com as funções não-modelo, que geralmente não são definidas nos cabeçalhos). Consulte http://stackoverflow.com/questions/495021/why-can-templates-only-be-implemented-in-the-header-file

Modelos de função #

A modelagem também pode ser aplicada a funções (assim como às estruturas mais tradicionais) com o mesmo efeito.

// 'T' stands for the unknown type
// Both of our arguments will be of the same type.
template<typename T>
void printSum(T add1, T add2)
{
    std::cout << (add1 + add2) << std::endl;
}

Isso pode ser usado da mesma maneira que os modelos de estrutura.

printSum<int>(4, 5);
printSum<float>(4.5f, 8.9f);

Em ambos os casos, o argumento template é usado para substituir os tipos dos parâmetros; o resultado funciona exatamente como uma função C++ normal (se os parâmetros não corresponderem ao tipo de modelo, o compilador aplicará as conversões padrão).

Uma propriedade adicional das funções de modelo (diferente das classes de modelo) é que o compilador pode inferir os parâmetros do modelo com base nos parâmetros passados ​​para a função.

printSum(4, 5);     // Both parameters are int.
                    // This allows the compiler deduce that the type
                    // T is also int.

printSum(5.0, 4);   // In this case the parameters are two different types.
                    // The compiler is unable to deduce the type of T
                    // because there are contradictions. As a result
                    // this is a compile time error.

Esse recurso nos permite simplificar o código quando combinamos estruturas e funções de modelo. Existe um padrão comum na biblioteca padrão que nos permite fazer a estrutura do modelo X usando uma função auxiliar make_X().

// The make_X pattern looks like this.
// 1) A template structure with 1 or more template types.
template<typename T1, typename T2>
struct MyPair
{
    T1      first;
    T2      second;
};
// 2) A make function that has a parameter type for
//    each template parameter in the template structure.
template<typename T1, typename T2>
MyPair<T1, T2> make_MyPair(T1 t1, T2 t2)
{
    return MyPair<T1, T2>{t1, t2};
}

Como isso ajuda?

auto val1 = MyPair<int, float>{5, 8.7};     // Create object explicitly defining the types
auto val2 = make_MyPair(5, 8.7);            // Create object using the types of the paramters.
                                            // In this code both val1 and val2 are the same
                                            // type.

Nota: Isso não foi projetado para encurtar o código. Isso foi projetado para tornar o código mais robusto. Ele permite que os tipos sejam alterados alterando o código em um único local, em vez de em vários locais.

Modelo Básico de Aula #

A ideia básica de um modelo de classe é que o parâmetro do modelo seja substituído por um tipo em tempo de compilação. O resultado é que a mesma classe pode ser reutilizada para vários tipos. O usuário especifica qual tipo será usado quando uma variável da classe for declarada. Três exemplos disso são mostrados em main():

#include <iostream>
using std::cout;

template <typename T>         // A simple class to hold one number of any type
class Number {
public:
    void setNum(T n);         // Sets the class field to the given number
    T plus1() const;          // returns class field's "follower"
private:
    T num;                    // Class field
};

template <typename T>         // Set the class field to the given number
void Number<T>::setNum(T n) {
    num = n;
}

template <typename T>         // returns class field's "follower"
T Number<T>::plus1() const {
    return num + 1;
}

int main() {
    Number<int> anInt;        // Test with an integer (int replaces T in the class)
    anInt.setNum(1);
    cout << "My integer + 1 is " << anInt.plus1() << "\n";     // Prints 2

    Number<double> aDouble;   // Test with a double
    aDouble.setNum(3.1415926535897);
    cout << "My double + 1 is " << aDouble.plus1() << "\n";    // Prints 4.14159

    Number<float> aFloat;     // Test with a float
    aFloat.setNum(1.4);
    cout << "My float + 1 is " << aFloat.plus1() << "\n";      // Prints 2.4

    return 0;  // Successful completion
}

Estruturas de dados de modelo variável #

Muitas vezes é útil definir classes ou estruturas que possuem um número variável e tipo de membros de dados que são definidos em tempo de compilação. O exemplo canônico é std::tuple, mas às vezes é necessário definir suas próprias estruturas personalizadas. Aqui está um exemplo que define a estrutura usando composição (em vez de herança como com std::tuple. Comece com a definição geral (vazia), que também serve como o caso base para o término da recrudescência na especialização posterior:

template<typename ... T>
struct DataStructure {};

Isso já nos permite definir uma estrutura vazia, DataStructure<> data, embora isso ainda não seja muito útil.

Em seguida vem a especialização recursiva de caso:

template<typename T, typename ... Rest>
struct DataStructure<T, Rest ...>
{
    DataStructure(const T& first, const Rest& ... rest)
        : first(first)
        , rest(rest...)
    {}
    
    T first;                                
    DataStructure<Rest ... > rest;
};

Isso agora é suficiente para criarmos estruturas de dados arbitrárias, como DataStructure<int, float, std::string> data(1, 2.1, "hello").

Então o que está acontecendo? Primeiro, note que esta é uma especialização cujo requisito é que pelo menos um parâmetro de template variável (nomeadamente T acima) exista, embora não se preocupe com a composição específica do pacote Rest. Saber que T existe permite a definição de seu membro de dados, first. O restante dos dados é empacotado recursivamente como DataStructure<Rest ... > rest. O construtor inicia ambos os membros, incluindo uma chamada de construtor recursiva para o membro rest.

Para entender melhor, podemos trabalhar com um exemplo: suponha que você tenha uma declaração DataStructure<int, float> data. A declaração primeiro corresponde à especialização, produzindo uma estrutura com membros de dados int first e DataStructure<float> rest. A definição rest novamente corresponde a esta especialização, criando seus próprios membros float first e DataStructure<> rest. Finalmente este último rest combina com a definição do caso base, produzindo uma estrutura vazia.

Você pode visualizar isso da seguinte forma:

DataStructure<int, float>
   -> int first
   -> DataStructure<float> rest
         -> float first
         -> DataStructure<> rest
              -> (empty)

Agora temos a estrutura de dados, mas ainda não é muito útil, pois não podemos acessar facilmente os elementos de dados individuais (por exemplo, para acessar o último membro de DataStructure<int, float, std::string> data teríamos que usar data.rest.rest.first, que não é exatamente amigável). Então adicionamos um método get a ele (necessário apenas na especialização, pois a estrutura do caso base não possui dados para get):

template<typename T, typename ... Rest>
struct DataStructure<T, Rest ...>
{
    ...
    template<size_t idx>
    auto get()
    {
        return GetHelper<idx, DataStructure<T,Rest...>>::get(*this);
    }
    ...
};

Como você pode ver, esta função de membro get é modelada - desta vez no índice do membro que é necessário (portanto, o uso pode ser coisas como data.get<1>(), semelhante a std::tuple ). O trabalho real é feito por uma função estática em uma classe auxiliar, GetHelper. A razão pela qual não podemos definir a funcionalidade necessária diretamente no get do DataStructure é porque (como veremos em breve) precisaríamos nos especializar em idx - mas não é possível especializar um membro do modelo função sem especializar o modelo de classe que o contém. Observe também que o uso de um auto no estilo C++ 14 aqui torna nossas vidas significativamente mais simples, caso contrário, precisaríamos de uma expressão bastante complicada para o tipo de retorno.

Então vamos para a classe auxiliar. Desta vez, precisaremos de uma declaração de encaminhamento vazia e duas especializações. Primeiro a declaração:

template<size_t idx, typename T>
struct GetHelper;

Agora o caso base (quando idx==0). Neste caso, apenas retornamos o membro first:

template<typename T, typename ... Rest>
struct GetHelper<0, DataStructure<T, Rest ... >>
{
    static T get(DataStructure<T, Rest...>& data)
    {
        return data.first;
    }
};

No caso recursivo, decrementamos idx e invocamos o GetHelper para o membro rest:

template<size_t idx, typename T, typename ... Rest>
struct GetHelper<idx, DataStructure<T, Rest ... >>
{
    static auto get(DataStructure<T, Rest...>& data)
    {
        return GetHelper<idx-1, DataStructure<Rest ...>>::get(data.rest);
    }
};

Para trabalhar com um exemplo, suponha que temos DataStructure<int, float> data e precisamos de data.get<1>(). Isso invoca GetHelper<1, DataStructure<int, float>>::get(data) (a segunda especialização), que por sua vez invoca GetHelper<0, DataStructure<float>>::get(data.rest) , que finalmente retorna (pela primeira especialização, pois agora idx é 0) data.rest.first.

Então é isso! Aqui está todo o código de funcionamento, com alguns exemplos de uso na função main:

#include <iostream>

template<size_t idx, typename T>
struct GetHelper;

template<typename ... T>
struct DataStructure
{
};

template<typename T, typename ... Rest>
struct DataStructure<T, Rest ...>
{
    DataStructure(const T& first, const Rest& ... rest)
        : first(first)
        , rest(rest...)
    {}
    
    T first;
    DataStructure<Rest ... > rest;
    
    template<size_t idx>
    auto get()
    {
        return GetHelper<idx, DataStructure<T,Rest...>>::get(*this);
    }
};

template<typename T, typename ... Rest>
struct GetHelper<0, DataStructure<T, Rest ... >>
{
    static T get(DataStructure<T, Rest...>& data)
    {
        return data.first;
    }
};

template<size_t idx, typename T, typename ... Rest>
struct GetHelper<idx, DataStructure<T, Rest ... >>
{
    static auto get(DataStructure<T, Rest...>& data)
    {
        return GetHelper<idx-1, DataStructure<Rest ...>>::get(data.rest);
    }
};

int main()
{
    DataStructure<int, float, std::string> data(1, 2.1, "Hello");
        
    std::cout << data.get<0>() << std::endl;
    std::cout << data.get<1>() << std::endl;
    std::cout << data.get<2>() << std::endl;
    
    return 0;
}

Encaminhamento de argumentos #

O modelo pode aceitar referências lvalue e rvalue usando referência de encaminhamento:

template <typename T>
void f(T &&t);

Neste caso, o tipo real de t será deduzido dependendo do contexto:

struct X { };

X x;
f(x); // calls f<X&>(x)
f(X()); // calls f<X>(x)

No primeiro caso, o tipo T é deduzido como referência a X (X&), e o tipo de t é lvalor referência a X, enquanto no segundo caso o tipo de T é deduzido como X e o tipo de t como rvalue referência a X (X&&).

Nota: Vale a pena notar que no primeiro caso, decltype(t) é o mesmo que T, mas não no segundo.

Para encaminhar perfeitamente t para outra função, seja uma referência lvalue ou rvalue, deve-se usar std::forward:

template <typename T>
void f(T &&t) {
    g(std::forward<T>(t));
}

Referências de encaminhamento podem ser usadas com modelos variáveis:

template <typename... Args>
void f(Args&&... args) {
    g(std::forward<Args>(args)...);
}

Observação: Referências de encaminhamento só podem ser usadas para parâmetros de modelo, por exemplo, no código a seguir, v é uma referência de rvalue, não uma referência de encaminhamento:

#include <vector>

template <typename T>
void f(std::vector<T> &&v);

Especialização de modelos #

Você pode definir a implementação para instanciações específicas de uma classe/método de modelo.

Por exemplo, se você tiver:

template <typename T>
T sqrt(T t) { /* Some generic implementation */ }

Você pode então escrever:

template<>
int sqrt<int>(int i) { /* Highly optimized integer implementation */ }

Então um usuário que escreve sqrt(4.0) obterá a implementação genérica enquanto que sqrt(4) obterá a implementação especializada.

Especialização parcial de modelos #

Ao contrário de uma especialização completa de modelo, a especialização parcial de modelo permite introduzir um modelo com alguns dos argumentos do modelo existente fixo. A especialização parcial de modelo está disponível apenas para classes/estruturas de modelo:

// Common case:
template<typename T, typename U>
struct S {
    T t_val;
    U u_val;
};

// Special case when the first template argument is fixed to int
template<typename V>
struct S<int, V> {
    double another_value;
    int foo(double arg) {// Do something}
};

Conforme mostrado acima, as especializações parciais de modelo podem introduzir conjuntos de dados e membros de função completamente diferentes.

Quando um modelo parcialmente especializado é instanciado, a especialização mais adequada é selecionada. Por exemplo, vamos definir um modelo e duas especializações parciais:

template<typename T, typename U, typename V>
struct S {
    static void foo() {
        std::cout << "General case\n";
    }
};

template<typename U, typename V>
struct S<int, U, V> {
    static void foo() {
        std::cout << "T = int\n";
    }
};

template<typename V>
struct S<int, double, V> {
    static void foo() {
        std::cout << "T = int, U = double\n";
    }
};

Agora as seguintes chamadas:

S<std::string, int, double>::foo();
S<int, float, std::string>::foo();
S<int, double, std::string>::foo();

vai imprimir

General case
T = int
T = int, U = double

Os modelos de função só podem ser totalmente especializados:

template<typename T, typename U>
void foo(T t, U u) {
    std::cout << "General case: " << t << " " << u << std::endl;
}

// OK.
template<>
void foo<int, int>(int a1, int a2) {
    std::cout << "Two ints: " << a1 << " " << a2 << std::endl;
}

void invoke_foo() {
    foo(1, 2.1); // Prints "General case: 1 2.1"
    foo(1,2);    // Prints "Two ints: 1 2"
}

// Compilation error: partial function specialization is not allowed.
template<typename U>
void foo<std::string, U>(std::string t, U u) {
    std::cout << "General case: " << t << " " << u << std::endl;
}

Modelo de alias #

Exemplo básico:

template<typename T> using pointer = T*;

Esta definição torna pointer<T> um alias de T*. Por exemplo:

pointer<int> p = new int; // equivalent to: int* p = new int;

Os modelos de alias não podem ser especializados. No entanto, essa funcionalidade pode ser obtida indiretamente fazendo com que eles se refiram a um tipo aninhado em uma estrutura:

template<typename T>
 struct nonconst_pointer_helper { typedef T* type; };

template<typename T>
 struct nonconst_pointer_helper<T const> { typedef T* type; };

template<typename T> using nonconst_pointer = nonconst_pointer_helper<T>::type;

Parâmetros do modelo de modelo #

Às vezes, gostaríamos de passar para o modelo um tipo de modelo sem fixar seus valores. É para isso que os parâmetros do modelo de modelo são criados. Exemplos de parâmetros de modelo de modelo muito simples:

template <class T>
struct Tag1 { };

template <class T>
struct Tag2 { };

template <template <class> class Tag>
struct IntTag {
   typedef Tag<int> type;
};

int main() {
   IntTag<Tag1>::type t;
}

#include <vector>
#include <iostream>

template <class T, template <class...> class C, class U>
C<T> cast_all(const C<U> &c) {
   C<T> result(c.begin(), c.end());
   return result;
}

int main() {
   std::vector<float> vf = {1.2, 2.6, 3.7};
   auto vi = cast_all<int>(vf);
   for(auto &&i: vi) {
      std::cout << i << std::endl;
   }
}

Declarando argumentos de modelo sem tipo com auto #

Antes do C++17, ao escrever um parâmetro sem tipo de modelo, você precisava especificar seu tipo primeiro. Então, um padrão comum tornou-se escrever algo como:

template <class T, T N>
struct integral_constant {
    using type = T;
    static constexpr T value = N;
};

using five = integral_constant<int, 5>;

Mas para expressões complicadas, usar algo assim envolve escrever decltype(expr), expr ao instanciar templates. A solução é simplificar este idioma e simplesmente permitir auto:

template <auto N>
struct integral_constant {
    using type = decltype(N); 
    static constexpr type value = N;
};

using five = integral_constant<5>;

Excluídor personalizado vazio para unique_ptr #

Um bom exemplo motivador pode vir de tentar combinar a otimização de base vazia com um apagador personalizado para unique_ptr. Diferentes deletores de API C têm diferentes tipos de retorno, mas não nos importamos - só queremos que algo funcione para qualquer função:

template <auto DeleteFn>
struct FunctionDeleter {
    template <class T>
    void operator()(T* ptr) const {
        DeleteFn(ptr);
    }
};

template <T, auto DeleteFn>
using unique_ptr_deleter = std::unique_ptr<T, FunctionDeleter<DeleteFn>>;

E agora você pode simplesmente usar qualquer ponteiro de função que possa receber um argumento do tipo T como um parâmetro sem tipo de modelo, independentemente do tipo de retorno, e obter uma sobrecarga sem tamanho unique_ptr:

unique_ptr_deleter<std::FILE, std::fclose> p;

Parâmetro de modelo sem tipo #

Além de tipos como um parâmetro de modelo, podemos declarar valores de expressões constantes que atendem a um dos seguintes critérios:

  • tipo integral ou enumeração,
  • ponteiro para objeto ou ponteiro para função,
  • lvalue referência ao objeto ou lvalue referência à função,
  • ponteiro para membro,
  • std::nullptr_t.

Como todos os parâmetros de modelo, os parâmetros de modelo que não são de tipo podem ser especificados explicitamente, padronizados ou derivados implicitamente por meio da Dedução de argumento de modelo.

Exemplo de uso de parâmetro de modelo sem tipo:

#include <iostream>

template<typename T, std::size_t size>
std::size_t size_of(T (&anArray)[size])  // Pass array by reference. Requires.
{                                        // an exact size. We allow all sizes
    return size;                         // by using a template "size".
}

int main()
{
    char anArrayOfChar[15];
    std::cout << "anArrayOfChar: " << size_of(anArrayOfChar) << "\n";

    int  anArrayOfData[] = {1,2,3,4,5,6,7,8,9};
    std::cout << "anArrayOfData: " << size_of(anArrayOfData) << "\n";
}

Exemplo de especificação explícita de parâmetros de modelo de tipo e não tipo:

#include <array>
int main ()
{
    std::array<int, 5> foo; // int is a type parameter, 5 is non-type
}

Parâmetros de modelo não-tipo são uma das maneiras de obter o modelo recorrência e permite fazer Metaprogramação.

Instanciação explícita #

Uma definição de instanciação explícita cria e declara uma classe, função ou variável concreta a partir de um modelo, sem usá-la ainda. Uma instanciação explícita pode ser referenciada a partir de outras unidades de tradução. Isso pode ser usado para evitar a definição de um modelo em um arquivo de cabeçalho, se ele for instanciado apenas com um conjunto finito de argumentos. Por exemplo:

// print_string.h
template <class T>
void print_string(const T* str);

// print_string.cpp
#include "print_string.h"
template void print_string(const char*);
template void print_string(const wchar_t*);

Como print_string<char> e print_string<wchar_t> são explicitamente instanciados em print_string.cpp, o vinculador poderá encontrá-los mesmo que o modelo print_string não esteja definido no cabeçalho. Se essas declarações de instanciação explícitas não estivessem presentes, provavelmente ocorreria um erro de vinculador. Consulte http://stackoverflow.com/questions/495021/why-can-templates-only-be-implemented-in-the-header-file

Se uma definição de instanciação explícita for precedida pela [palavra-chave] extern (https://www.wikiod.com/pt/docs/c%2b%2b/4891/keywords), ela se tornará uma declaração de instanciação explícita. A presença de uma declaração de instanciação explícita para uma determinada especialização impede a instanciação implícita da especialização dada dentro da unidade de tradução atual. Em vez disso, uma referência a essa especialização que causaria uma instanciação implícita pode se referir a uma definição de instanciação explícita na mesma ou em outra TU.

foo.h

#ifndef FOO_H
#define FOO_H
template <class T> void foo(T x) {
    // complicated implementation
}
#endif

foo.cpp

#include "foo.h"
// explicit instantiation definitions for common cases
template void foo(int);
template void foo(double);

main.cpp

#include "foo.h"
// we already know foo.cpp has explicit instantiation definitions for these
extern template void foo(double);
int main() {
    foo(42);   // instantiates foo<int> here;
               // wasteful since foo.cpp provides an explicit instantiation already!
    foo(3.14); // does not instantiate foo<double> here;
               // uses instantiation of foo<double> in foo.cpp instead
}

Valor de parâmetro de modelo padrão #

Assim como no caso dos argumentos da função, os parâmetros do modelo podem ter seus valores padrão. Todos os parâmetros do modelo com um valor padrão devem ser declarados no final da lista de parâmetros do modelo. A ideia básica é que os parâmetros do template com valor padrão podem ser omitidos durante a instanciação do template.

Exemplo simples de uso de valor de parâmetro de modelo padrão:

template <class T, size_t N = 10>
struct my_array {
    T arr[N];
};

int main() {
    /* Default parameter is ignored, N = 5 */
    my_array<int, 5> a;

    /* Print the length of a.arr: 5 */
    std::cout << sizeof(a.arr) / sizeof(int) << std::endl;

    /* Last parameter is omitted, N = 10 */
    my_array<int> b;

    /* Print the length of a.arr: 10 */
    std::cout << sizeof(b.arr) / sizeof(int) << std::endl;
}
