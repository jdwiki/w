Em C++, Metaprogramação refere-se ao uso de macros ou templates para gerar código em tempo de compilação.

Em geral, as macros são desaprovadas nessa função e os modelos são preferidos, embora não sejam tão genéricos.

A metaprogramação de templates frequentemente faz uso de computações em tempo de compilação, seja por meio de templates ou funções constexpr, para atingir seus objetivos de geração de código, porém computações em tempo de compilação não são metaprogramação per se.

Metaprogramação (ou mais especificamente, Metaprogramação de Modelo) é a prática de usar templates para criar constantes, funções ou estruturas de dados em tempo de compilação. Isso permite que os cálculos sejam executados uma vez em tempo de compilação, em vez de em cada tempo de execução.

Calculando fatoriais #

Os fatoriais podem ser calculados em tempo de compilação usando técnicas de metaprogramação de modelo.

#include <iostream>

template<unsigned int n>
struct factorial
{
    enum
    {
        value = n * factorial<n - 1>::value
    };
};

template<>
struct factorial<0>
{
    enum { value = 1 };
};

int main()
{
    std::cout << factorial<7>::value << std::endl;    // prints "5040"
}

factorial é uma struct, mas na metaprogramação de template é tratado como uma metafunção de template. Por convenção, as metafunções de template são avaliadas verificando um membro específico, seja ::type para metafunções que resultam em tipos, ou ::value para metafunções que geram valores.

No código acima, avaliamos a metafunção factorial instanciando o template com os parâmetros que queremos passar, e usando ::value para obter o resultado da avaliação.

A própria metafunção depende de instanciar recursivamente a mesma metafunção com valores menores. A especialização factorial<0> representa a condição final. A metaprogramação de template tem a maioria das restrições de uma linguagem de programação funcional, então a recursão é a construção primária de “looping”.

Como as metafunções de modelo são executadas em tempo de compilação, seus resultados podem ser usados ​​em contextos que exigem valores em tempo de compilação. Por exemplo:

int my_array[factorial<5>::value];

Arrays automáticos devem ter um tamanho definido em tempo de compilação. E o resultado de uma metafunção é uma constante de tempo de compilação, então ela pode ser usada aqui.

Limitação: A maioria dos compiladores não permite profundidade de recursão além de um limite. Por exemplo, o compilador g++ por padrão limita a profundidade de recursão a 256 níveis. No caso de g++, o programador pode definir a profundidade de recursão usando a opção -fttemplate-depth-X.

Desde o C++11, o template std::integral_constant pode ser usado para este tipo de cálculo de template:

#include <iostream>
#include <type_traits>

template<long long n>
struct factorial :
  std::integral_constant<long long, n * factorial<n - 1>::value> {};

template<>
struct factorial<0> :
  std::integral_constant<long long, 1> {};

int main()
{
    std::cout << factorial<7>::value << std::endl;    // prints "5040"
}

Além disso, as funções constexpr tornam-se uma alternativa mais limpa.

#include <iostream>

constexpr long long factorial(long long n)
{
  return (n == 0) ? 1 : n * factorial(n - 1);
}

int main()
{
  char test[factorial(3)];
  std::cout << factorial(7) << '\n';
}

O corpo de factorial() é escrito como uma única instrução porque em C++11 as funções constexpr só podem usar um subconjunto bastante limitado da linguagem.

Desde C++14, muitas restrições para funções constexpr foram eliminadas e agora podem ser escritas de forma muito mais conveniente:

constexpr long long factorial(long long n)
{
  if (n == 0)
    return 1;
  else
    return n * factorial(n - 1);
}

Ou ainda:

constexpr long long factorial(int n)
{
  long long result = 1;
  for (int i = 1; i <= n; ++i) {
    result *= i;
  }
  return result;
}

Desde c ++ 17, pode-se usar a expressão de dobra para calcular fatorial:

#include <iostream>
#include <utility>

template <class T, T N, class I = std::make_integer_sequence<T, N>>
struct factorial;

template <class T, T N, T... Is>
struct factorial<T,N,std::index_sequence<T, Is...>> {
   static constexpr T value = (static_cast<T>(1) * ... * (Is + 1));
};

int main() {
   std::cout << factorial<int, 5>::value << std::endl;
}

Iterando sobre um pacote de parâmetros #

Frequentemente, precisamos realizar uma operação em cada elemento em um pacote de parâmetros de template variadic. Há muitas maneiras de fazer isso, e as soluções ficam mais fáceis de ler e escrever com C++17. Suponha que simplesmente queremos imprimir todos os elementos de um pacote. A solução mais simples é recorrer:

void print_all(std::ostream& os) {
    // base case
}

template <class T, class... Ts>
void print_all(std::ostream& os, T const& first, Ts const&... rest) {
    os << first;
    
    print_all(os, rest...);
}

Em vez disso, poderíamos usar o truque do expansor, para realizar todo o streaming em uma única função. Isso tem a vantagem de não precisar de uma segunda sobrecarga, mas tem a desvantagem de ser menos legível:

template <class... Ts>
void print_all(std::ostream& os, Ts const&... args) {
    using expander = int[];
    (void)expander{0,
        (void(os << args), 0)...
    };
}

Para obter uma explicação de como isso funciona, consulte excelente resposta de T.C.

Com o C++17, temos duas novas ferramentas poderosas em nosso arsenal para resolver esse problema. A primeira é uma expressão de dobra:

template <class... Ts>
void print_all(std::ostream& os, Ts const&... args) {
    ((os << args), ...);
}

E o segundo é if constexpr, que nos permite escrever nossa solução recursiva original em uma única função:

template <class T, class... Ts>
void print_all(std::ostream& os, T const& first, Ts const&... rest) {
    os << first;

    if constexpr (sizeof...(rest) > 0) {        
        // this line will only be instantiated if there are further
        // arguments. if rest... is empty, there will be no call to
        // print_all(os). 
        print_all(os, rest...);
    }
}

Iterando com std::integer_sequence #

Desde o C++14, o padrão fornece o modelo de classe

template <class T, T... Ints>
class integer_sequence;

template <std::size_t... Ints>
using index_sequence = std::integer_sequence<std::size_t, Ints...>;

e uma metafunção geradora para ele:

template <class T, T N>
using make_integer_sequence = std::integer_sequence<T, /* a sequence 0, 1, 2, ..., N-1 */ >;

template<std::size_t N>
using make_index_sequence = make_integer_sequence<std::size_t, N>;

Embora isso seja padrão em C++14, isso pode ser implementado usando ferramentas C++11.

Podemos usar esta ferramenta para chamar uma função com um std::tuple de argumentos (padronizado em C++17 como std::apply):

namespace detail {
    template <class F, class Tuple, std::size_t... Is>
    decltype(auto) apply_impl(F&& f, Tuple&& tpl, std::index_sequence<Is...> ) {
        return std::forward<F>(f)(std::get<Is>(std::forward<Tuple>(tpl))...);
    }
}

template <class F, class Tuple>
decltype(auto) apply(F&& f, Tuple&& tpl) {
    return detail::apply_impl(std::forward<F>(f),
        std::forward<Tuple>(tpl),
        std::make_index_sequence<std::tuple_size<std::decay_t<Tuple>>::value>{});
}


// this will print 3
int f(int, char, double);

auto some_args = std::make_tuple(42, 'x', 3.14);
int r = apply(f, some_args); // calls f(42, 'x', 3.14)

Envio de tags #

Uma maneira simples de selecionar entre funções em tempo de compilação é despachar uma função para um par sobrecarregado de funções que recebem uma tag como um (geralmente o último) argumento. Por exemplo, para implementar std::advance(), podemos despachar na categoria do iterador:

namespace details {
    template <class RAIter, class Distance>
    void advance(RAIter& it, Distance n, std::random_access_iterator_tag) {
        it += n;
    }

    template <class BidirIter, class Distance>
    void advance(BidirIter& it, Distance n, std::bidirectional_iterator_tag) {
        if (n > 0) {
            while (n--) ++it;
        }
        else {
            while (n++) --it;
        }
    }

    template <class InputIter, class Distance>
    void advance(InputIter& it, Distance n, std::input_iterator_tag) {
        while (n--) {
            ++it;
        }
    }    
}

template <class Iter, class Distance>
void advance(Iter& it, Distance n) {
    details::advance(it, n, 
            typename std::iterator_traits<Iter>::iterator_category{} );
}

Os argumentos std::XY_iterator_tag das funções sobrecarregadas details::advance são parâmetros de função não utilizados. A implementação real não importa (na verdade, está completamente vazia). Seu único propósito é permitir que o compilador selecione uma sobrecarga com base em qual classe de tag details::advance é chamada.

Neste exemplo, advance usa a metafunção iterator_traits<T>::iterator_category que retorna uma das classes iterator_tag, dependendo do tipo real de Iter. Um objeto construído por padrão do iterator_category<Iter>::type permite que o compilador selecione uma das diferentes sobrecargas de details::advance. (Este parâmetro de função provavelmente será completamente otimizado, pois é um objeto construído por padrão de um struct vazio e nunca usado.)

O despacho de tags pode fornecer um código muito mais fácil de ler do que os equivalentes usando SFINAE e enable_if.

Observação: embora o if constexpr do C++17 possa simplificar a implementação do advance em particular, ele não é adequado para implementações abertas, diferentemente do envio de tags.

Detectar se a expressão é válida #

É possível detectar se um operador ou função pode ser chamado em um tipo. Para testar se uma classe tem uma sobrecarga de std::hash, pode-se fazer isso:

#include <functional> // for std::hash
#include <type_traits> // for std::false_type and std::true_type
#include <utility> // for std::declval

template<class, class = void>
struct has_hash
    : std::false_type
{};

template<class T>
struct has_hash<T, decltype(std::hash<T>()(std::declval<T>()), void())>
    : std::true_type
{};

Desde C++17, std::void_t pode ser usado para simplificar este tipo de construção

#include <functional> // for std::hash
#include <type_traits> // for std::false_type, std::true_type, std::void_t
#include <utility> // for std::declval

template<class, class = std::void_t<> >
struct has_hash
    : std::false_type
{};

template<class T>
struct has_hash<T, std::void_t< decltype(std::hash<T>()(std::declval<T>())) > >
    : std::true_type
{};

onde std::void_t é definido como:

template< class... > using void_t = void;

Para detectar se um operador, como operator<, está definido, a sintaxe é quase a mesma:

template<class, class = void>
struct has_less_than
    : std::false_type
{};

template<class T>
struct has_less_than<T, decltype(std::declval<T>() < std::declval<T>(), void())>
    : std::true_type
{};

Eles podem ser usados ​​para usar um std::unordered_map<T> se T tiver uma sobrecarga para std::hash, mas, caso contrário, tente usar um std::map<T>:

template <class K, class V>
using hash_invariant_map = std::conditional_t<
    has_hash<K>::value,
    std::unordered_map<K, V>,
    std::map<K,V>>;

Se-então-senão #

O tipo std::conditional no cabeçalho da biblioteca padrão <type_traits> pode selecionar um tipo ou outro, com base em um valor booleano de tempo de compilação:

template<typename T>
struct ValueOrPointer
{
    typename std::conditional<(sizeof(T) > sizeof(void*)), T*, T>::type vop;
};

Esta estrutura contém um ponteiro para T se T for maior que o tamanho de um ponteiro, ou T se for menor ou igual ao tamanho de um ponteiro. Portanto, sizeof(ValueOrPointer) sempre será <= sizeof(void*).

Calculando o poder com C++11 (e superior) #

Com C++ 11 e cálculos superiores em tempo de compilação pode ser muito mais fácil. Por exemplo, calcular a potência de um determinado número em tempo de compilação será o seguinte:

template <typename T>
constexpr T calculatePower(T value, unsigned power) {
    return power == 0 ? 1 : value * calculatePower(value, power-1);
}

A palavra-chave constexpr é responsável por calcular a função em tempo de compilação, então e somente então, quando todos os requisitos para isso forem atendidos (veja mais na referência da palavra-chave constexpr) por exemplo, todos os argumentos devem ser conhecidos em tempo de compilação.

Nota: Em C++11, a função constexpr deve compor apenas a partir de uma instrução de retorno.

Vantagens: Comparando com a forma padrão de cálculo de tempo de compilação, este método também é útil para cálculos de tempo de execução. Isso significa que se os argumentos da função não são conhecidos no momento da compilação (por exemplo, valor e poder são dados como entrada via usuário), então a função é executada em tempo de compilação, então não há necessidade de duplicar um código (como nós seria forçado em padrões mais antigos de C++).

Por exemplo.

void useExample() {
    constexpr int compileTimeCalculated = calculatePower(3, 3); // computes at compile time,
                               // as both arguments are known at compilation time
                               // and used for a constant expression.
    int value;
    std::cin >> value;
    int runtimeCalculated = calculatePower(value, 3);  // runtime calculated,
                                    // because value is known only at runtime.
}

Outra maneira de calcular o poder em tempo de compilação pode usar a expressão de dobra da seguinte maneira:

#include <iostream>
#include <utility>

template <class T, T V, T N, class I = std::make_integer_sequence<T, N>>
struct power;

template <class T, T V, T N, T... Is>
struct power<T, V, N, std::integer_sequence<T, Is...>> {
   static constexpr T value = (static_cast<T>(1) * ... * (V * static_cast<bool>(Is + 1)));
};

int main() {
   std::cout << power<int, 4, 2>::value << std::endl;
}

Distinção manual de tipos quando dado qualquer tipo T #

Ao implementar SFINAE usando std::enable_if , muitas vezes é útil ter acesso ao auxiliar templates que determina se um determinado tipo T corresponde a um conjunto de critérios.

Para nos ajudar com isso, o padrão já fornece dois tipos analógicos para true e false que são std::true_type e std::false_type.

O exemplo a seguir mostra como detectar se um tipo T é um ponteiro ou não, o template is_pointer imita o comportamento do auxiliar std::is_pointer padrão:

template <typename T>
struct is_pointer_: std::false_type {};

template <typename T>
struct is_pointer_<T*>: std::true_type {};

template <typename T>
struct is_pointer: is_pointer_<typename std::remove_cv<T>::type> { }

Existem três etapas no código acima (às vezes você só precisa de duas):

  1. A primeira declaração de is_pointer_ é o case padrão, e herda de std::false_type. O caso default deve sempre herdar de std::false_type já que é análogo a uma condição “false”.

  2. A segunda declaração especializa o template is_pointer_ para o ponteiro T* sem se importar com o que T é realmente. Esta versão herda de std::true_type.

  3. A terceira declaração (a real) simplesmente remove qualquer informação desnecessária de T (neste caso removemos os qualificadores const e volatile) e então retorna para uma das duas declarações anteriores.

Como is_pointer<T> é uma classe, para acessar seu valor você precisa:

  • Use ::valor, por exemplo is_pointer<int>::value – value é um membro de classe estático do tipo bool herdado de std::true_type ou std::false_type;
  • Construir um objeto deste tipo, por exemplo. is_pointer<int>{} – Isso funciona porque std::is_pointer herda seu construtor padrão de std::true_type ou std::false_type (que tem construtores constexpr) e ambos std::true_type e std::false_type tem operadores de conversão constexpr para bool.

É um bom hábito fornecer “modelos auxiliares auxiliares” que permitem acessar diretamente o valor:

template <typename T>
constexpr bool is_pointer_v = is_pointer<T>::value;

No C++ 17 e acima, a maioria dos templates auxiliares já fornecem uma versão _v, por exemplo:

template< class T > constexpr bool is_pointer_v = is_pointer<T>::value;
template< class T > constexpr bool is_reference_v = is_reference<T>::value;

Genérico Min/Max com contagem variável de argumentos #

É possível escrever uma função genérica (por exemplo min) que aceita vários tipos numéricos e contagem arbitrária de argumentos pela meta-programação de template. Esta função declara um min para dois argumentos e recursivamente para mais.

template <typename T1, typename T2>
auto min(const T1 &a, const T2 &b) 
-> typename std::common_type<const T1&, const T2&>::type
{
    return a < b ? a : b;
}

template <typename T1, typename T2, typename ... Args>
auto min(const T1 &a, const T2 &b, const Args& ... args)
-> typename std::common_type<const T1&, const T2&, const Args& ...>::type
{
    return min(min(a, b), args...);
}

auto minimum = min(4, 5.8f, 3, 1.8, 3, 1.1, 9);
