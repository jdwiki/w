Visão geral #

Iteradores são posições #

Iteradores são um meio de navegar e operar em uma sequência de elementos e são uma extensão generalizada de ponteiros. Conceitualmente, é importante lembrar que os iteradores são posições, não elementos. Por exemplo, tome a seguinte sequência:

A B C

A sequência contém três elementos e quatro posições

+---+---+---+---+
| A | B | C |   |
+---+---+---+---+

Elementos são coisas dentro de uma sequência. Posições são lugares onde operações significativas podem acontecer com a sequência. Por exemplo, insere-se em uma posição, antes ou depois elemento A, não em um elemento. Até mesmo a exclusão de um elemento (erase(A)) é feita primeiro encontrando sua posição e, em seguida, excluindo-a.

De Iteradores a Valores #

Para converter de uma posição para um valor, um iterador é desreferenciado:

auto my_iterator = my_vector.begin(); // position
auto my_value = *my_iterator; // value

Pode-se pensar em um iterador como referência ao valor ao qual ele se refere na sequência. Isso é especialmente útil para entender por que você nunca deve desreferenciar o iterador end() em uma sequência:

+---+---+---+---+
| A | B | C |   |
+---+---+---+---+
  ↑           ↑
  |           +-- An iterator here has no value. Do not dereference it!
  +-------------- An iterator here dereferences to the value A.

Em todas as sequências e contêineres encontrados na biblioteca padrão C++, begin() retornará um iterador para a primeira posição, e end() retornará um iterador para um após a última posição (não a última posição!) . Consequentemente, os nomes desses iteradores em algoritmos são muitas vezes rotulados como first e last:

+---+---+---+---+
| A | B | C |   |
+---+---+---+---+
  ↑           ↑
  |           |
  +- first    +- last

Também é possível obter um iterador para qualquer sequência, pois mesmo uma sequência vazia contém pelo menos uma posição:

+---+
|   |
+---+

Em uma sequência vazia, begin() e end() estarão na mesma posição, e neither pode ser desreferenciado:

+---+
|   |
+---+
  ↑
  |
  +- empty_sequence.begin()
  |
  +- empty_sequence.end()

A visualização alternativa de iteradores é que eles marcam as posições entre elementos:

+---+---+---+
| A | B | C |
+---+---+---+
↑   ^   ^   ↑
|           |
+- first    +- last

e desreferenciar um iterador retorna uma referência ao elemento que vem após o iterador. Algumas situações em que essa visualização é particularmente útil são:

  • As operações insert irão inserir elementos na posição indicada pelo iterador,
  • As operações erase retornarão um iterador correspondente à mesma posição que a passada,
  • um iterador e seu correspondente iterador reverso estão localizados na mesma posição entre os elementos

Iteradores inválidos #

Um iterador se torna invalidado se (digamos, no decorrer de uma operação) sua posição não for mais parte de uma sequência. Um iterador invalidado não pode ser desreferenciado até que seja reatribuído a uma posição válida. Por exemplo:

std::vector<int>::iterator first;
{
    std::vector<int> foo;
    first = foo.begin(); // first is now valid
} // foo falls out of scope and is destroyed
// At this point first is now invalid

Os muitos algoritmos e funções de membro de sequência na biblioteca padrão C++ têm regras que controlam quando os iteradores são invalidados. Cada algoritmo é diferente na maneira como tratam (e invalidam) os iteradores.

Navegando com Iteradores #

Como sabemos, os iteradores são para navegar em sequências. Para fazer isso, um iterador deve migrar sua posição ao longo da sequência. Os iteradores podem avançar na sequência e alguns podem avançar para trás:

auto first = my_vector.begin();
++first;                                             // advance the iterator 1 position
std::advance(first, 1);                              // advance the iterator 1 position
first = std::next(first);                            // returns iterator to the next element
std::advance(first, -1);                             // advance the iterator 1 position backwards
first = std::next(first, 20);                        // returns iterator to the element 20 position forward
first = std::prev(first, 5);                         // returns iterator to the element 5 position backward
auto dist = std::distance(my_vector.begin(), first); // returns distance between two iterators.

Observe que o segundo argumento de std::distance deve ser alcançável a partir do primeiro (ou, em outras palavras, first deve ser menor ou igual a second).

Mesmo que você possa executar operadores aritméticos com iteradores, nem todas as operações são definidas para todos os tipos de iteradores. a = b + 3; funcionaria para iteradores de acesso aleatório, mas não funcionaria para iteradores de encaminhamento ou bidirecionais, que ainda podem ser avançados em 3 posições com algo como b = a; ++b; ++b; ++b;. Portanto, é recomendável usar funções especiais caso você não tenha certeza do tipo de iterador (por exemplo, em uma função de modelo que aceita iterador).

Conceitos do Iterador #

O padrão C++ descreve vários conceitos de iterador diferentes. Estes são agrupados de acordo com a forma como se comportam nas sequências a que se referem. Se você conhece o conceito de um iterador models (comporta-se como), você pode ter certeza do comportamento desse iterador independentemente da sequência à qual ele pertence. Eles são frequentemente descritos em ordem do mais para o menos restritivo (porque o próximo conceito de iterador é um passo melhor que seu antecessor):

  • Iteradores de entrada: podem ser desreferenciados somente uma vez por posição. Só pode avançar, e apenas uma posição de cada vez.
  • Forward Iterators: Um iterador de entrada que pode ser desreferenciado várias vezes.
  • Iteradores bidirecionais: Um iterador de avanço que também pode avançar backwards uma posição por vez.
  • Iteradores de acesso aleatório: Um iterador bidirecional que pode avançar ou retroceder qualquer número de posições ao mesmo tempo.
  • Iteradores Contíguos (desde C++17): Um iterador de acesso aleatório que garante que os dados subjacentes sejam contíguos na memória.

Os algoritmos podem variar dependendo do conceito modelado pelos iteradores que recebem. Por exemplo, embora random_shuffle possa ser implementado para iteradores de encaminhamento, uma variante mais eficiente que requer iteradores de acesso aleatório pode ser fornecida.

Características do iterador #

As características do iterador fornecem uma interface uniforme para as propriedades dos iteradores. Eles permitem que você recupere valor, diferença, ponteiro, tipos de referência e também categoria do iterador:

template<class Iter>
Iter find(Iter first, Iter last, typename std::iterator_traits<Iter>::value_type val)  {
    while (first != last) {
        if (*first == val)
            return first;
        ++first;
    }
    return last;
}

A categoria do iterador pode ser usada para especializar algoritmos:

template<class BidirIt>
void test(BidirIt a, std::bidirectional_iterator_tag)  {
    std::cout << "Bidirectional iterator is used" << std::endl;
}
 
template<class ForwIt>
void test(ForwIt a, std::forward_iterator_tag)  {
    std::cout << "Forward iterator is used" << std::endl;
}
 
template<class Iter>
void test(Iter a)  {
    test(a, typename std::iterator_traits<Iter>::iterator_category());
}

Categorias de iteradores são basicamente conceitos de iteradores, exceto que Iteradores Contíguos não têm sua própria tag, já que foi encontrado para quebrar o código.

Iterador de vetor #

begin retorna um iterator para o primeiro elemento no contêiner de sequência.

end retorna um iterator para o primeiro elemento após o final.

Se o objeto vetorial for const, tanto begin quanto end retornarão um const_iterator. Se você quiser que um const_iterator seja retornado mesmo que seu vetor não seja const, você pode usar cbegin e cend.

Exemplo:

#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = { 1, 2, 3, 4, 5 };  //intialize vector using an initializer_list

    for (std::vector<int>::iterator it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << " ";
    }

    return 0;
}

Resultado:

1 2 3 4 5

Iterador de mapa #

Um iterador para o primeiro elemento no contêiner.

Se um objeto de mapa for qualificado como const, a função retornará um const_iterator. Caso contrário, ele retorna um iterador.

// Create a map and insert some values
std::map<char,int> mymap;
mymap['b'] = 100;
mymap['a'] = 200;
mymap['c'] = 300;

// Iterate over all tuples
for (std::map<char,int>::iterator it = mymap.begin(); it != mymap.end(); ++it)
    std::cout << it->first << " => " << it->second << '\n';

Resultado:

a => 200 b => 100 c => 300

Iteradores reversos #

Se quisermos iterar para trás em uma lista ou vetor, podemos usar um reverse_iterator. Um iterador reverso é feito de um iterador bidirecional ou de acesso aleatório que ele mantém como um membro que pode ser acessado através de base().

Para iterar para trás, use rbegin() e rend() como os iteradores para o final da coleção e o início da coleção, respectivamente.

Por exemplo, para iterar para trás, use:

std::vector<int> v{1, 2, 3, 4, 5};
for (std::vector<int>::reverse_iterator it = v.rbegin(); it != v.rend(); ++it)
{
    cout << *it;
} // prints 54321

Um iterador reverso pode ser convertido em um iterador direto através da função de membro base(). A relação é que o iterador reverso referencia um elemento após o iterador base():

std::vector<int>::reverse_iterator r = v.rbegin();
std::vector<int>::iterator i = r.base();
assert(&*r == &*(i-1)); // always true if r, (i-1) are dereferenceable
                        // and are not proxy iterators

 |   | 1 | 2 | 3 | 4 | 5 |   |

   ↑   ↑               ↑   ↑
   |   |               |   |
rend() |         rbegin()  end()
       |                   rbegin().base()
     begin()
     rend().base()

Na visualização em que os iteradores marcam posições entre os elementos, o relacionamento é mais simples:

Iteradores de fluxo #

Os iteradores de fluxo são úteis quando precisamos ler uma sequência ou imprimir dados formatados de um contêiner:

// Data stream. Any number of various whitespace characters will be OK.
std::istringstream istr("1\t 2     3 4");
std::vector<int> v;

// Constructing stream iterators and copying data from stream into vector.
std::copy(
    // Iterator which will read stream data as integers.
    std::istream_iterator<int>(istr),
    // Default constructor produces end-of-stream iterator.
    std::istream_iterator<int>(),
    std::back_inserter(v));

// Print vector contents.
std::copy(v.begin(), v.end(),
    //Will print values to standard output as integers delimeted by " -- ".
    std::ostream_iterator<int>(std::cout, " -- "));

O programa de exemplo imprimirá 1 -- 2 -- 3 -- 4 -- na saída padrão.

C Iteradores (Ponteiros) #

// This creates an array with 5 values.
const int array[] = { 1, 2, 3, 4, 5 };

#ifdef BEFORE_CPP11

// You can use `sizeof` to determine how many elements are in an array.
const int* first = array;
const int* afterLast = first + sizeof(array) / sizeof(array[0]);

// Then you can iterate over the array by incrementing a pointer until
// it reaches past the end of our array.
for (const int* i = first; i < afterLast; ++i) {
    std::cout << *i << std::endl;
}

#else

// With C++11, you can let the STL compute the start and end iterators:
for (auto i = std::begin(array); i != std::end(array); ++i) {
    std::cout << *i << std::endl;
}

#endif

Este código produziria os números de 1 a 5, um em cada linha assim:

1
2
3
4
5

Decompondo #

const int array[] = { 1, 2, 3, 4, 5 };

Esta linha cria um novo array inteiro com 5 valores. Os arrays C são apenas ponteiros para a memória onde cada valor é armazenado em um bloco contíguo.

const int* first = array;
const int* afterLast = first + sizeof(array) / sizeof(array[0]);

Essas linhas criam dois ponteiros. O primeiro ponteiro recebe o valor do ponteiro do array, que é o endereço do primeiro elemento do array. O operador sizeof quando usado em um array C retorna o tamanho do array em bytes. Dividido pelo tamanho de um elemento, isso dá o número de elementos na matriz. Podemos usar isso para encontrar o endereço do bloco depois do array.

for (const int* i = first; i < afterLast; ++i) {

Aqui criamos um ponteiro que usaremos como um iterador. Ele é inicializado com o endereço do primeiro elemento sobre o qual queremos iterar, e continuaremos a iterar enquanto i for menor que afterLast, o que significa que i esteja apontando para um endereço dentro matriz.

    std::cout << *i << std::endl;

Finalmente, dentro do loop, podemos acessar o valor para o qual nosso iterador i está apontando, desreferenciando-o. Aqui o operador de desreferência * retorna o valor no endereço em i.

Escreva seu próprio iterador baseado em gerador #

Um padrão comum em outras linguagens é ter uma função que produz um “fluxo” de objetos e poder usar o código de loop para fazer um loop sobre ele.

Podemos modelar isso em C++ como

template<class T>
struct generator_iterator {
  using difference_type=std::ptrdiff_t;
  using value_type=T;
  using pointer=T*;
  using reference=T;
  using iterator_category=std::input_iterator_tag;
  std::optional<T> state;
  std::function< std::optional<T>() > operation;
  // we store the current element in "state" if we have one:
  T operator*() const {
    return *state;
  }
  // to advance, we invoke our operation.  If it returns a nullopt
  // we have reached the end:
  generator_iterator& operator++() {
    state = operation();
    return *this;        
  }
  generator_iterator operator++(int) {
    auto r = *this;
    ++(*this);
    return r;
  }
  // generator iterators are only equal if they are both in the "end" state:
  friend bool operator==( generator_iterator const& lhs, generator_iterator const& rhs ) {
    if (!lhs.state && !rhs.state) return true;
    return false;
  }
  friend bool operator!=( generator_iterator const& lhs, generator_iterator const& rhs ) {
    return !(lhs==rhs);
  }
  // We implicitly construct from a std::function with the right signature:
  generator_iterator( std::function< std::optional<T>() > f ):operation(std::move(f))
  {
    if (operation)
      state = operation();
  }
  // default all special member functions:
  generator_iterator( generator_iterator && ) =default;
  generator_iterator( generator_iterator const& ) =default;
  generator_iterator& operator=( generator_iterator && ) =default;
  generator_iterator& operator=( generator_iterator const& ) =default;
  generator_iterator() =default;
};

exemplo ao vivo.

Armazenamos o elemento gerado antecipadamente para que possamos detectar mais facilmente se já estamos no final.

Como a função de um iterador de gerador final nunca é usada, podemos criar um intervalo de iteradores de gerador apenas copiando o std::function uma vez. Um iterador de gerador construído padrão compara-se igual a si mesmo e a todos os outros iteradores de gerador final.
