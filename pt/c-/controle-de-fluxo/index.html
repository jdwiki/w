Confira o tópico de loops para os diferentes tipos de loops.

caso #

Introduz um rótulo de caso de uma instrução switch. O operando deve ser uma expressão constante e corresponder à condição de alternância no tipo. Quando a instrução switch for executada, ela pulará para o rótulo case com operando igual à condição, se houver.

char c = getchar();
bool confirmed;
switch (c) {
  case 'y':
    confirmed = true;
    break;
  case 'n':
    confirmed = false;
    break;
  default:
    std::cout << "invalid response!\n";
    abort();
}

trocar #

De acordo com o padrão C++,

A instrução switch faz com que o controle seja transferido para uma das várias instruções, dependendo do valor de uma condição.

A palavra-chave switch é seguida por uma condição entre parênteses e um bloco, que pode conter rótulos case e ​​um rótulo opcional default. Quando a instrução switch é executada, o controle será transferido para um rótulo case com um valor correspondente ao da condição, se houver, ou para o rótulo default, se houver.

A condição deve ser uma expressão ou uma declaração, que tenha um tipo inteiro ou de enumeração, ou um tipo de classe com uma função de conversão para um tipo inteiro ou de enumeração.

char c = getchar();
bool confirmed;
switch (c) {
  case 'y':
    confirmed = true;
    break;
  case 'n':
    confirmed = false;
    break;
  default:
    std::cout << "invalid response!\n";
    abort();
}

truque #

A palavra-chave catch introduz um manipulador de exceção, ou seja, um bloco para o qual o controle será transferido quando uma exceção de tipo compatível for lançada. A palavra-chave catch é seguida por uma declaração de exceção entre parênteses, que é semelhante em forma a uma declaração de parâmetro de função: o nome do parâmetro pode ser omitido e as reticências ... são permitidas, o que corresponde a qualquer tipo. O manipulador de exceção só tratará a exceção se sua declaração for compatível com o tipo da exceção. Para obter mais detalhes, consulte capturar exceções.

try {
    std::vector<int> v(N);
    // do something
} catch (const std::bad_alloc&) {
    std::cout << "failed to allocate memory for vector!" << std::endl;
} catch (const std::runtime_error& e) {
    std::cout << "runtime error: " << e.what() << std::endl;
} catch (...) {
    std::cout << "unexpected exception!" << std::endl;
    throw;
}

predefinição #

Em uma instrução switch, introduz um rótulo para o qual será saltado se o valor da condição não for igual a nenhum dos valores dos rótulos de caso.

char c = getchar();
bool confirmed;
switch (c) {
  case 'y':
    confirmed = true;
    break;
  case 'n':
    confirmed = false;
    break;
  default:
    std::cout << "invalid response!\n";
    abort();
}

Define um construtor padrão, construtor de cópia, construtor de movimentação, destruidor, operador de atribuição de cópia ou operador de atribuição de movimentação para ter seu comportamento padrão.

class Base {
    // ...
    // we want to be able to delete derived classes through Base*,
    // but have the usual behaviour for Base's destructor.
    virtual ~Base() = default;
};

E se #

Introduz uma instrução if. A palavra-chave if deve ser seguida por uma condição entre parênteses, que pode ser uma expressão ou uma declaração. Se a condição for verdadeira, a subinstrução após a condição será executada.

int x;
std::cout << "Please enter a positive number." << std::endl;
std::cin >> x;
if (x <= 0) {
    std::cout << "You didn't enter a positive number!" << std::endl;
    abort();
}

senão #

A primeira subinstrução de uma instrução if pode ser seguida pela palavra-chave else. A subinstrução após a palavra-chave else será executada quando a condição for falsa (ou seja, quando a primeira subinstrução não for executada).

int x;
std::cin >> x;
if (x%2 == 0) {
    std::cout << "The number is even\n";
} else {
    std::cout << "The number is odd\n";
}

Vá para #

Salta para uma instrução rotulada, que deve estar localizada na função atual.

bool f(int arg) {
    bool result = false;
    hWidget widget = get_widget(arg);
    if (!g()) {
        // we can't continue, but must do cleanup still
        goto end;
    }
    // ...
    result = true;
  end:
    release_widget(widget);
    return result;
}

Retorna #

Retorna o controle de uma função para seu chamador.

Se return tiver um operando, o operando será convertido para o tipo de retorno da função e o valor convertido será retornado ao chamador.

int f() {
    return 42;
}
int x = f(); // x is 42
int g() {
    return 3.14;
}
int y = g(); // y is 3

Se return não tiver um operando, a função deve ter o tipo de retorno void. Como um caso especial, uma função de retorno void também pode retornar uma expressão se a expressão tiver o tipo void.

void f(int x) {
    if (x < 0) return;
    std::cout << sqrt(x);
}
int g() { return 42; }
void h() {
    return f(); // calls f, then returns
    return g(); // ill-formed
}

Quando main retorna, std::exit é chamado implicitamente com o valor de retorno, e o valor é então retornado ao ambiente de execução. (Entretanto, retornar de main destrói variáveis ​​locais automáticas, enquanto chamar std::exit diretamente não.)

int main(int argc, char** argv) {
    if (argc < 2) {
        std::cout << "Missing argument\n";
        return EXIT_FAILURE; // equivalent to: exit(EXIT_FAILURE);
    }
}

lançar #

  1. Quando throw ocorre em uma expressão com um operando, seu efeito é lançar uma exceção, que é uma cópia de o operando.

    void print_asterisks(int count) {
    if (count < 0) {
        throw std::invalid_argument("count cannot be negative!");
    }
    while (count--) { putchar('*'); }
}

  2. Quando throw ocorre em uma expressão sem um operando, seu efeito é [relançar a exceção atual](https://www.wikiod.com/pt/docs/c%2b%2b/1354/exceptions/5574/rethrow-propagate -exceção). Se não houver nenhuma exceção atual, std::terminate é chamado.

    try {
    // something risky
} catch (const std::bad_alloc&) {
    std::cerr << "out of memory" << std::endl;
} catch (...) {
    std::cerr << "unexpected exception" << std::endl;
    // hope the caller knows how to handle this exception
    throw;
}

  3. Quando throw ocorre em um declarador de função, ele introduz uma especificação de exceção dinâmica, que lista os tipos de exceções que a função pode propagar.

    // this function might propagate a std::runtime_error,
// but not, say, a std::logic_error
void risky() throw(std::runtime_error);
// this function can't propagate any exceptions
void safe() throw();

As especificações de exceção dinâmicas estão obsoletas a partir do C++11.

Observe que os dois primeiros usos de throw listados acima constituem expressões em vez de declarações. (O tipo de uma expressão throw é void.) Isso torna possível aninhá-los dentro de expressões, assim:

unsigned int predecessor(unsigned int x) {
    return (x > 0) ? (x - 1) : (throw std::invalid_argument("0 has no predecessor"));
}

tentar #

A palavra-chave try é seguida por um bloco, ou por uma lista de inicializadores de construtor e depois por um bloco (veja [aqui](https://www.wikiod.com/pt/docs/c%2b%2b/1354/exceptions/6692/function -try-blocks-in-constructor)). O bloco try é seguido por um ou mais blocos de captura. Se uma exceção se propagar para fora do bloco try, cada um dos blocos catch correspondentes após o bloco try terá a oportunidade de tratar a exceção , se os tipos corresponderem.

std::vector<int> v(N);     // if an exception is thrown here,
                           // it will not be caught by the following catch block
try {
    std::vector<int> v(N); // if an exception is thrown here,
                           // it will be caught by the following catch block
    // do something with v
} catch (const std::bad_alloc&) {
    // handle bad_alloc exceptions from the try block
}
