Sintaxe #

  • [default-capture, capture-list] (argument-list) mutável throw-specification attributes -> return-type { lambda-body } // Ordem dos especificadores lambda e atributos.
  • [capture-list] (argument-list) { lambda-body } // Definição lambda comum.
  • [=] (argument-list) { lambda-body } // Captura todas as variáveis ​​locais necessárias por valor.
  • [&] (argument-list) { lambda-body } // Captura todas as variáveis ​​locais necessárias por referência.
  • [capture-list] { lambda-body } // A lista de argumentos e os especificadores podem ser omitidos.

Parâmetros #

Parâmetro Detalhes
captura padrão Especifica como todas as variáveis ​​não listadas são capturadas. Pode ser = (capturar por valor) ou & (capturar por referência). Se omitidas, as variáveis ​​não listadas ficam inacessíveis dentro do lambda-body. A captura padrão deve preceder a lista de capturas.
lista de capturas Especifica como as variáveis ​​locais são disponibilizadas no lambda-body. Variáveis ​​sem prefixo são capturadas por valor. Variáveis ​​prefixadas com & são capturadas por referência. Dentro de um método de classe, this pode ser usado para tornar todos os seus membros acessíveis por referência. Variáveis ​​não listadas são inacessíveis, a menos que a lista seja precedida por uma captura padrão.
lista de argumentos Especifica os argumentos da função lambda.
mutável (opcional) Normalmente as variáveis ​​capturadas por valor são const. Especificar mutable torna-os não-const. As alterações nessas variáveis ​​são mantidas entre as chamadas.
especificação de lançamento (opcional) Especifica o comportamento de lançamento de exceção da função lambda. Por exemplo: noexcept ou throw(std::exception).
atributos (opcional) Quaisquer atributos para a função lambda. Por exemplo, se o lambda-body sempre lança uma exceção, então [[noreturn]] pode ser usado.
-> tipo de retorno (opcional) Especifica o tipo de retorno da função lambda. Necessário quando o tipo de retorno não pode ser determinado pelo compilador.
corpo lambda Um bloco de código contendo a implementação da função lambda.

C++17 (o rascunho atual) introduz lambdas constexpr, basicamente lambdas que podem ser avaliados em tempo de compilação. Um lambda é automaticamente constexpr se satisfizer os requisitos constexpr, mas você também pode especificá-lo usando a palavra-chave constexpr:

//Explicitly define this lambdas as constexpr
[]() constexpr {
    //Do stuff
}

O que é uma expressão lambda? #

Uma expressão lambda fornece uma maneira concisa de criar objetos de função simples. Uma expressão lambda é um prvalue cujo objeto de resultado é chamado objeto de fechamento, que se comporta como um objeto de função.

O nome ’expressão lambda’ se origina de lambda calculus, que é um formalismo matemático inventado na década de 1930 por Alonzo Church para investigar questões sobre lógica e computabilidade. O cálculo lambda formou a base do LISP, uma linguagem de programação funcional. Em comparação com o cálculo lambda e o LISP, as expressões lambda C++ compartilham as propriedades de não terem nome e de capturar variáveis ​​do contexto circundante, mas não têm a capacidade de operar e retornar funções.

Uma expressão lambda é frequentemente usada como um argumento para funções que recebem um objeto que pode ser chamado. Isso pode ser mais simples do que criar uma função nomeada, que só seria usada quando passada como argumento. Nesses casos, as expressões lambda são geralmente preferidas porque permitem definir os objetos de função inline.

Um lambda consiste normalmente em três partes: uma lista de captura [], uma lista de parâmetros opcional () e um corpo {}, todos os quais podem estar vazios:

[](){}                // An empty lambda, which does and returns nothing

Lista de captura

[] é a lista de capturas. Por padrão, as variáveis ​​do escopo delimitador não podem ser acessadas por um lambda. Capturar uma variável a torna acessível dentro do lambda, seja como uma cópia ou como uma referência. As variáveis ​​capturadas tornam-se parte do lambda; ao contrário dos argumentos de função, eles não precisam ser passados ​​ao chamar o lambda.

int a = 0;                       // Define an integer variable
auto f = []()   { return a*9; }; // Error: 'a' cannot be accessed
auto f = [a]()  { return a*9; }; // OK, 'a' is "captured" by value
auto f = [&a]() { return a++; }; // OK, 'a' is "captured" by reference
                                 //      Note: It is the responsibility of the programmer
                                 //      to ensure that a is not destroyed before the
                                 //      lambda is called.
auto b = f();                    // Call the lambda function. a is taken from the capture list and not passed here.

Lista de parâmetros

() é a lista de parâmetros, que é quase a mesma das funções normais. Se o lambda não receber argumentos, esses parênteses podem ser omitidos (exceto se você precisar declarar o lambda mutable). Esses dois lambdas são equivalentes:

auto call_foo  = [x](){ x.foo(); };
auto call_foo2 = [x]{ x.foo(); };

A lista de parâmetros pode usar o tipo de espaço reservado auto em vez de tipos reais. Ao fazer isso, esse argumento se comporta como um parâmetro de modelo de um modelo de função. Os lambdas a seguir são equivalentes quando você deseja classificar um vetor em código genérico:

auto sort_cpp11 = [](std::vector<T>::const_reference lhs, std::vector<T>::const_reference rhs) { return lhs < rhs; }; 
auto sort_cpp14 = [](const auto &lhs, const auto &rhs) { return lhs < rhs; };

Corpo de função

{} é o body, que é o mesmo que em funções regulares.

Chamando um lambda

O objeto de resultado de uma expressão lambda é um closure, que pode ser chamado usando o operator() (como acontece com outros objetos de função):

int multiplier = 5;
auto timesFive = [multiplier](int a) { return a * multiplier; }; 
std::out << timesFive(2); // Prints 10

multiplier = 15;
std::out << timesFive(2); // Still prints 2*5 == 10

Tipo de devolução

Por padrão, o tipo de retorno de uma expressão lambda é deduzido.

[](){ return true; };

Neste caso, o tipo de retorno é bool.

Você também pode especificar manualmente o tipo de retorno usando a seguinte sintaxe:

[]() -> bool { return true; };

Lambda Mutável

Objetos capturados por valor no lambda são imutáveis ​​por padrão. Isso ocorre porque o operator() do objeto de fechamento gerado é const por padrão.

auto func = [c = 0](){++c; std::cout << c;};  // fails to compile because ++c
                                              // tries to mutate the state of
                                              // the lambda.

A modificação pode ser permitida usando a palavra-chave mutable, que torna o operator() do objeto mais próximo não-const:

auto func = [c = 0]() mutable {++c; std::cout << c;};

Se usado junto com o tipo de retorno, mutable vem antes dele.

auto func = [c = 0]() mutable -> int {++c; std::cout << c; return c;};

Um exemplo para ilustrar a utilidade dos lambdas

Antes do C++11:

// Generic functor used for comparison
struct islessthan
{
    islessthan(int threshold) : _threshold(threshold) {}

    bool operator()(int value) const
    {
        return value < _threshold;
    }
private:
    int _threshold;
};

// Declare a vector
const int arr[] = { 1, 2, 3, 4, 5 };
std::vector<int> vec(arr, arr+5);

// Find a number that's less than a given input (assume this would have been function input)
int threshold = 10;
std::vector<int>::iterator it = std::find_if(vec.begin(), vec.end(), islessthan(threshold));

Desde C++11:

// Declare a vector
std::vector<int> vec{ 1, 2, 3, 4, 5 };

// Find a number that's less than a given input (assume this would have been function input)
int threshold = 10;
auto it = std::find_if(vec.begin(), vec.end(), [threshold](int value) { return value < threshold; });

Especificando o tipo de retorno #

Para lambdas com uma única instrução de retorno ou várias instruções de retorno cujas expressões são do mesmo tipo, o compilador pode deduzir o tipo de retorno:

// Returns bool, because "value > 10" is a comparison which yields a Boolean result
auto l = [](int value) {
    return value > 10;
}

Para lambdas com várias instruções de retorno de tipos diferentes, o compilador não pode deduzir o tipo de retorno:

// error: return types must match if lambda has unspecified return type
auto l = [](int value) {
    if (value < 10) {
        return 1;
    } else {
        return 1.5;
    }
};

Nesse caso, você deve especificar o tipo de retorno explicitamente:

// The return type is specified explicitly as 'double'
auto l = [](int value) -> double {
    if (value < 10) {
        return 1;
    } else {
        return 1.5;
    }
};

As regras para isso correspondem às regras para dedução do tipo auto. Lambdas sem tipos de retorno explicitamente especificados nunca retornam referências, portanto, se um tipo de referência for desejado, ele também deverá ser especificado explicitamente:

auto copy = [](X& x) { return x; };       // 'copy' returns an X, so copies its input
auto ref  = [](X& x) -> X& { return x; }; // 'ref' returns an X&, no copy

Captura por valor #

Se você especificar o nome da variável na lista de captura, o lambda a capturará por valor. Isso significa que o tipo de encerramento gerado para o lambda armazena uma cópia da variável. Isso também requer que o tipo da variável seja copy-constructible:

int a = 0;

[a]() {
    return a;   // Ok, 'a' is captured by value
};

auto p = std::unique_ptr<T>(...);

[p]() {         // Compile error; `unique_ptr` is not copy-constructible
    return p->createWidget(); 
};

A partir do C++14, é possível inicializar as variáveis ​​na hora. Isso permite que apenas tipos de movimento sejam capturados no lambda.

auto p = std::make_unique<T>(...);

[p = std::move(p)]() {
    return p->createWidget(); 
};

Mesmo que um lambda capture variáveis ​​por valor quando elas são fornecidas por seu nome, tais variáveis ​​não podem ser modificadas dentro do corpo lambda por padrão. Isso ocorre porque o tipo closure coloca o corpo lambda em uma declaração de operator() const.

O const se aplica a acessos a variáveis ​​de membro do tipo closure, e variáveis ​​capturadas que são membros da closure (todas as aparências ao contrário):

int a = 0;

[a]() {
    a = 2;      // Illegal, 'a' is accessed via `const`

    decltype(a) a1 = 1; 
    a1 = 2; // valid: variable 'a1' is not const
};

Para remover o const, você deve especificar a palavra-chave mutable no lambda:

int a = 0;

[a]() mutable {
    a = 2;      // OK, 'a' can be modified
    return a;
};

Como a foi capturado por valor, quaisquer modificações feitas chamando o lambda não afetarão a. O valor de a foi copiado no lambda quando foi construído, então a cópia do lambda de a é separada da variável externa a.

int a = 5 ; 
auto plus5Val = [a] (void) { return a + 5 ; } ; 
auto plus5Ref = [&a] (void) {return a + 5 ; } ; 

a = 7 ; 
std::cout << a << ", value " << plus5Val() << ", reference " << plus5Ref() ;
// The result will be "7, value 10, reference 12"

Lambdas recursivas #

Digamos que desejamos escrever o gcd() de Euclides como um lambda. Em função, fica:

int gcd(int a, int b) {
    return b == 0 ? a : gcd(b, a%b);
}

Mas um lambda não pode ser recursivo, não tem como invocar a si mesmo. Um lambda não tem nome e usar this no corpo de um lambda se refere a um this capturado (assumindo que o lambda é criado no corpo de uma função de membro, caso contrário, é um erro). Então, como resolvemos esse problema?

Use std::function #

Podemos fazer com que um lambda capture uma referência a um std::function ainda não construído:

std::function<int(int, int)> gcd = [&](int a, int b){
    return b == 0 ? a : gcd(b, a%b);
};

Isso funciona, mas deve ser usado com moderação. É lento (estamos usando type erasure agora em vez de uma chamada de função direta), é frágil (copiar gcd ou retornar gcd será interrompido, pois o lambda se refere ao objeto original) e não funcionará com genéricos lambdas.

Usando dois ponteiros inteligentes: #

auto gcd_self = std::make_shared<std::unique_ptr< std::function<int(int, int)> >>();
*gcd_self = std::make_unique<std::function<int(int, int)>>(
  [gcd_self](int a, int b){
    return b == 0 ? a : (**gcd_self)(b, a%b);
  };
};

Isso adiciona muita indireção (que é sobrecarga), mas pode ser copiado/retornado e todas as cópias compartilham o estado. Ele permite que você retorne o lambda e, de outra forma, é menos frágil do que a solução acima.

Use um combinador Y #

Com a ajuda de uma estrutura de utilidade curta, podemos resolver todos esses problemas:

template <class F>
struct y_combinator {
    F f; // the lambda will be stored here

    // a forwarding operator():
    template <class... Args>
    decltype(auto) operator()(Args&&... args) const {
        // we pass ourselves to f, then the arguments.
        // the lambda should take the first argument as `auto&& recurse` or similar.
        return f(*this, std::forward<Args>(args)...);
    }
};
// helper function that deduces the type of the lambda:
template <class F>
y_combinator<std::decay_t<F>> make_y_combinator(F&& f) {
    return {std::forward<F>(f)};
}
// (Be aware that in C++17 we can do better than a `make_` function)

podemos implementar nosso gcd como:

auto gcd = make_y_combinator(
  [](auto&& gcd, int a, int b){
    return b == 0 ? a : gcd(b, a%b);
  }
);

O y_combinator é um conceito do cálculo lambda que permite que você tenha recursão sem poder se nomear até que você seja definido. Este é exatamente o problema que os lambdas têm.

Você cria um lambda que usa “recurse” como seu primeiro argumento. Quando você deseja recursivamente, você passa os argumentos para recursivamente.

O y_combinator então retorna um objeto de função que chama essa função com seus argumentos, mas com um objeto “recurse” adequado (ou seja, o próprio y_combinator) como seu primeiro argumento. Ele encaminha o resto dos argumentos com os quais você chama o y_combinator para o lambda também.

Resumidamente:

auto foo = make_y_combinator( [&](auto&& recurse, some arguments) {
  // write body that processes some arguments
  // when you want to recurse, call recurse(some other arguments)
});

e você tem recursão em um lambda sem restrições sérias ou sobrecarga significativa.

Captura padrão #

Por padrão, as variáveis ​​locais que não são especificadas explicitamente na lista de captura não podem ser acessadas de dentro do corpo lambda. No entanto, é possível capturar implicitamente variáveis ​​nomeadas pelo corpo lambda:

int a = 1;
int b = 2;

// Default capture by value
[=]() { return a + b; }; // OK; a and b are captured by value

// Default capture by reference
[&]() { return a + b; }; // OK; a and b are captured by reference

A captura explícita ainda pode ser feita junto com a captura padrão implícita. A definição de captura explícita substituirá a captura padrão:

int a = 0;
int b = 1;

[=, &b]() {
    a = 2; // Illegal; 'a' is capture by value, and lambda is not 'mutable'
    b = 2; // OK; 'b' is captured by reference
};

Captura por referência #

Se você preceder o nome de uma variável local com um &, a variável será capturada por referência. Conceitualmente, isso significa que o tipo de fechamento do lambda terá uma variável de referência, inicializada como uma referência à variável correspondente de fora do escopo do lambda. Qualquer uso da variável no corpo lambda fará referência à variável original:

// Declare variable 'a'
int a = 0;

// Declare a lambda which captures 'a' by reference
auto set = [&a]() {
    a = 1;
};

set();
assert(a == 1);

A palavra-chave mutable não é necessária, porque a em si não é const.

Claro, capturar por referência significa que o lambda não deve escapar do escopo das variáveis ​​que ele captura. Portanto, você pode chamar funções que recebem uma função, mas não deve chamar uma função que armazenará o lambda além do escopo de suas referências. E você não deve retornar o lambda.

Lambdas genéricos #

As funções Lambda podem receber argumentos de tipos arbitrários. Isso permite que um lambda seja mais genérico:

auto twice = [](auto x){ return x+x; };

int i = twice(2); // i == 4
std::string s = twice("hello"); // s == "hellohello"

Isso é implementado em C++ fazendo com que o operator() do tipo closure sobrecarregue uma função de modelo. O tipo a seguir tem comportamento equivalente ao encerramento lambda acima:

struct _unique_lambda_type
{
  template<typename T>
  auto operator() (T x) const {return x + x;}
};

Nem todos os parâmetros em um lambda genérico precisam ser genéricos:

[](auto x, int y) {return x + y;}

Aqui, x é deduzido com base no primeiro argumento da função, enquanto y sempre será int.

Lambdas genéricos também podem receber argumentos por referência, usando as regras usuais para auto e &. Se um parâmetro genérico for tomado como auto&&, esta é uma referência de encaminhamento para o argumento passado e não uma rvalue referência:

auto lamb1 = [](int &&x) {return x + 5;};
auto lamb2 = [](auto &&x) {return x + 5;};
int x = 10;
lamb1(x); // Illegal; must use `std::move(x)` for `int&&` parameters.
lamb2(x); // Legal; the type of `x` is deduced as `int&`.

As funções lambda podem ser variáveis ​​e encaminhar perfeitamente seus argumentos:

auto lam = [](auto&&... args){return f(std::forward<decltype(args)>(args)...);};

ou:

auto lam = [](auto&&... args){return f(decltype(args)(args)...);};

que só funciona “corretamente” com variáveis ​​do tipo auto&&.

Um forte motivo para usar lambdas genéricos é para visitar a sintaxe.

boost::variant<int, double> value;
apply_visitor(value, [&](auto&& e){
  std::cout << e;
});

Aqui estamos visitando de maneira polimórfica; mas em outros contextos, os nomes do tipo que estamos passando não são interessantes:

mutex_wrapped<std::ostream&> os = std::cout;
os.write([&](auto&& os){
  os << "hello world\n";
});

Repetir o tipo de std::ostream& é ruído aqui; seria como ter que mencionar o tipo de uma variável toda vez que você a usa. Aqui estamos criando um visitante, mas não polimórfico; auto é usado pela mesma razão que você pode usar auto em um loop for(:).

Classe lambdas e captura disso #

Uma expressão lambda avaliada na função de membro de uma classe é implicitamente uma amiga dessa classe:

class Foo
{
private:
    int i;
    
public:
    Foo(int val) : i(val) {}
    
    // definition of a member function
    void Test()
    {
        auto lamb = [](Foo &foo, int val)
        {
            // modification of a private member variable
            foo.i = val;
        };
        
        // lamb is allowed to access a private member, because it is a friend of Foo
        lamb(*this, 30);
    }
};

Tal lambda não é apenas um amigo dessa classe, ele tem o mesmo acesso que a classe em que foi declarado.

Lambdas pode capturar o ponteiro this que representa a instância do objeto em que a função externa foi chamada. Isso é feito adicionando this à lista de captura:

class Foo
{
private:
    int i;
    
public:
    Foo(int val) : i(val) {}
    
    void Test()
    {
        // capture the this pointer by value
        auto lamb = [this](int val)
        {
            i = val;
        };
        
        lamb(30);
    }
};

Quando this é capturado, o lambda pode usar nomes de membros de sua classe de conteúdo como se estivesse em sua classe de conteúdo. Portanto, um this-> implícito é aplicado a esses membros.

Esteja ciente de que this é capturado pelo valor, mas não pelo valor do tipo. Ele é capturado pelo valor de this, que é um ponteiro. Como tal, o lambda não possui this. Se o lambda fora viver o tempo de vida do objeto que o criou, o lambda pode se tornar inválido.

Isso também significa que o lambda pode modificar this sem ser declarado mutable. É o ponteiro que é const, não o objeto que está sendo apontado. Isto é, a menos que a função de membro externa fosse em si uma função const.

Além disso, esteja ciente de que as cláusulas de captura padrão, tanto [=] quanto [&], também capturam this implicitamente. E ambos o capturam pelo valor do ponteiro. De fato, é um erro especificar this na lista de captura quando um padrão é fornecido.

Lambdas pode capturar uma cópia do objeto this, criado no momento em que o lambda é criado. Isso é feito adicionando *this à lista de captura:

class Foo
{
private:
    int i;
    
public:
    Foo(int val) : i(val) {}
    
    void Test()
    {
        // capture a copy of the object given by the this pointer
        auto lamb = [*this](int val) mutable
        {
            i = val;
        };
        
        lamb(30); // does not change this->i
    }
};

Captura generalizada #

Lambdas pode capturar expressões, em vez de apenas variáveis. Isso permite que lambdas armazenem tipos somente de movimentação:

auto p = std::make_unique<T>(...);

auto lamb = [p = std::move(p)]() //Overrides capture-by-value of `p`.
{
  p->SomeFunc();
};

Isso move a variável externa p para a variável de captura lambda, também chamada de p. lamb agora possui a memória alocada por make_unique. Como a closure contém um tipo que não é copiável, isso significa que lamb não é copiável. Mas pode ser movido:

auto lamb_copy = lamb; //Illegal
auto lamb_move = std::move(lamb); //legal.

Agora lamb_move possui a memória.

Observe que std::function<> requer que os valores armazenados sejam copiáveis. Você pode escrever seu próprio [move-only-requiring std::function](https://www.wikiod.com/pt/docs/c%2b%2b/2872/type-erasure/18042/a-move-only-stdfunction #t=201608061624094409548), ou você pode simplesmente colocar o lambda em um wrapper shared_ptr:

auto shared_lambda = [](auto&& f){
  return [spf = std::make_shared<std::decay_t<decltype(f)>>(decltype(f)(f))]
  (auto&&...args)->decltype(auto) {
    return (*spf)(decltype(args)(args)...);
  };
};
auto lamb_shared = shared_lambda(std::move(lamb_move));

pega nosso lambda só de movimento e coloca seu estado em um ponteiro compartilhado, em seguida, retorna um lambda que pode ser copiado e, em seguida, armazenado em um std::function ou similar.

A captura generalizada usa a dedução de tipo auto para o tipo da variável. Ele declarará essas capturas como valores por padrão, mas elas também podem ser referências:

int a = 0;

auto lamb = [&v = a](int add) //Note that `a` and `v` have different names
{
  v += add; //Modifies `a`
};

lamb(20); //`a` becomes 20.

A captura generalizada não precisa capturar uma variável externa. Ele pode capturar uma expressão arbitrária:

auto lamb = [p = std::make_unique<T>(...)]()
{
    p->SomeFunc();
}

Isso é útil para fornecer valores arbitrários de lambdas que eles podem conter e modificar potencialmente, sem precisar declará-los externamente ao lambda. Claro, isso só é útil se você não pretende acessar essas variáveis ​​depois que o lambda tiver concluído seu trabalho.

Conversão para ponteiro de função #

Se a lista de captura de um lambda estiver vazia, o lambda terá uma conversão implícita para um ponteiro de função que recebe os mesmos argumentos e retorna o mesmo tipo de retorno:

auto sorter = [](int lhs, int rhs) -> bool {return lhs < rhs;};

using func_ptr = bool(*)(int, int);
func_ptr sorter_func = sorter; // implicit conversion

Essa conversão também pode ser aplicada usando o operador unário mais:

func_ptr sorter_func2 = +sorter; // enforce implicit conversion

Chamar esse ponteiro de função se comporta exatamente como chamar operator() no lambda. Este ponteiro de função não depende de forma alguma da existência do encerramento do lambda de origem. Portanto, pode sobreviver ao fechamento lambda.

Esse recurso é útil principalmente para usar lambdas com APIs que lidam com ponteiros de função, em vez de objetos de função C++.

A conversão para um ponteiro de função também é possível para lambdas genéricos com uma lista de captura vazia. Se necessário, a dedução do argumento do modelo será usada para selecionar a especialização correta.

auto sorter = [](auto lhs, auto rhs) { return lhs < rhs; };
using func_ptr = bool(*)(int, int);
func_ptr sorter_func = sorter;  // deduces int, int
// note however that the following is ambiguous
// func_ptr sorter_func2 = +sorter;

Usando lambdas para descompactar o pacote de parâmetros embutido #

A descompactação do pacote de parâmetros tradicionalmente requer escrever uma função auxiliar para cada vez que você deseja fazê-lo.

Neste exemplo de brinquedo:

template<std::size_t...Is>
void print_indexes( std::index_sequence<Is...> ) {
  using discard=int[];
  (void)discard{0,((void)(
    std::cout << Is << '\n' // here Is is a compile-time constant.
  ),0)...};
}
template<std::size_t I>
void print_indexes_upto() {
  return print_indexes( std::make_index_sequence<I>{} );
}

O print_indexes_upto deseja criar e descompactar um pacote de parâmetros de índices. Para fazer isso, ele deve chamar uma função auxiliar. Toda vez que você deseja descompactar um pacote de parâmetros que criou, acaba tendo que criar uma função auxiliar personalizada para fazer isso.

Isso pode ser evitado com lambdas.

Você pode descompactar pacotes de parâmetros em um conjunto de invocações de um lambda, assim:

template<std::size_t I>
using index_t = std::integral_constant<std::size_t, I>;
template<std::size_t I>
constexpr index_t<I> index{};

template<class=void, std::size_t...Is>
auto index_over( std::index_sequence<Is...> ) {
  return [](auto&& f){
    using discard=int[];
    (void)discard{0,(void(
      f( index<Is> )
    ),0)...};
  };
}

template<std::size_t N>
auto index_over(index_t<N> = {}) {
  return index_over( std::make_index_sequence<N>{} );
}

Com expressões de dobra, index_over() pode ser simplificado para:

template<class=void, std::size_t...Is>
auto index_over( std::index_sequence<Is...> ) {
  return [](auto&& f){
    ((void)(f(index<Is>)), ...);
  };
}

Depois de fazer isso, você pode usar isso para substituir a necessidade de descompactar manualmente os pacotes de parâmetros por uma segunda sobrecarga em outro código, permitindo descompactar os pacotes de parâmetros “inline”:

template<class Tup, class F>
void for_each_tuple_element(Tup&& tup, F&& f) {
  using T = std::remove_reference_t<Tup>;
  using std::tuple_size;
  auto from_zero_to_N = index_over< tuple_size<T>{} >();

  from_zero_to_N(
    [&](auto i){
      using std::get;
      f( get<i>( std::forward<Tup>(tup) ) );
    }
  );
}

O auto i passado para o lambda pelo index_over é um std::integral_constant<std::size_t, ???>. Isso tem uma conversão constexpr para std::size_t que não depende do estado de this, então podemos usá-lo como uma constante de tempo de compilação, como quando passamos para std::get <i> acima.

Para voltar ao exemplo do brinquedo no topo, reescreva-o como:

template<std::size_t I>
void print_indexes_upto() {
  index_over(index<I>)([](auto i){
    std::cout << i << '\n'; // here i is a compile-time constant
  });
}

que é muito mais curto e mantém a lógica no código que o utiliza.

Portando funções lambda para C++03 usando functors #

As funções lambda em C++ são açúcar sintático que fornecem uma sintaxe muito concisa para escrever functors. Como tal, uma funcionalidade equivalente pode ser obtida em C++03 (embora muito mais detalhada) convertendo a função lambda em um functor:

// Some dummy types:
struct T1 {int dummy;};
struct T2 {int dummy;};
struct R {int dummy;};

// Code using a lambda function (requires C++11)
R use_lambda(T1 val, T2 ref) {
  // Use auto because the type of the lambda is unknown.
  auto lambda = [val, &ref](int arg1, int arg2) -> R {
    /* lambda-body */
    return R();
  };
  return lambda(12, 27);
}

// The functor class (valid C++03)
// Similar to what the compiler generates for the lambda function.
class Functor {
  // Capture list.
  T1 val;
  T2& ref;

public:
  // Constructor
  inline Functor(T1 val, T2& ref) : val(val), ref(ref) {}

  // Functor body
  R operator()(int arg1, int arg2) const {
    /* lambda-body */
    return R();
  }
};

// Equivalent to use_lambda, but uses a functor (valid C++03).
R use_functor(T1 val, T2 ref) {
  Functor functor(val, ref);
  return functor(12, 27);
}

// Make this a self-contained example.
int main() {
  T1 t1;
  T2 t2;
  use_functor(t1,t2);
  use_lambda(t1,t2);
  return 0;
}

Se a função lambda for mutável, torne o operador de chamada do functor não const, ou seja:

R operator()(int arg1, int arg2) /*non-const*/ {
  /* lambda-body */
  return R();
}
