Usando opcionais para representar a ausência de um valor #

Antes do C++17, ter ponteiros com um valor de nullptr geralmente representava a ausência de um valor. Esta é uma boa solução para objetos grandes que foram alocados dinamicamente e já são gerenciados por ponteiros. No entanto, esta solução não funciona bem para tipos pequenos ou primitivos como int, que raramente são alocados dinamicamente ou gerenciados por ponteiros. std::opcional fornece uma solução viável para este problema comum.

Neste exemplo, struct Person é definido. É possível que uma pessoa tenha um animal de estimação, mas não é necessário. Portanto, o membro pet de Person é declarado com um wrapper std::optional.

#include <iostream>
#include <optional>
#include <string>

struct Animal {
    std::string name;
};

struct Person {
    std::string name;
    std::optional<Animal> pet;
};

int main() {
    Person person;
    person.name = "John";

    if (person.pet) {
        std::cout << person.name << "'s pet's name is " <<
            person.pet->name << std::endl;
    }
    else {
        std::cout << person.name << " is alone." << std::endl;
    }
}

opcional como valor de retorno #

std::optional<float> divide(float a, float b) {
  if (b!=0.f) return a/b;
  return {};
}

Aqui nós retornamos a fração a/b, mas se ela não estiver definida (seria infinito) nós retornamos o opcional vazio.

Um caso mais complexo:

template<class Range, class Pred>
auto find_if( Range&& r, Pred&& p ) {
  using std::begin; using std::end;
  auto b = begin(r), e = end(r);
  auto r = std::find_if(b, e , p );
  using iterator = decltype(r);
  if (r==e)
    return std::optional<iterator>();
  return std::optional<iterator>(r);
}
template<class Range, class T>
auto find( Range&& r, T const& t ) {
  return find_if( std::forward<Range>(r), [&t](auto&& x){return x==t;} );
}

find( some_range, 7 ) procura no container ou intervalo some_range por algo igual ao número 7. find_if faz isso com um predicado.

Ele retorna um opcional vazio se não for encontrado, ou um opcional contendo um iterador para o elemento, se for.

Isso permite que você faça:

if (find( vec, 7 )) {
  // code
}

ou mesmo

if (auto oit = find( vec, 7 )) {
  vec.erase(*oit);
}

sem ter que mexer com iteradores e testes de início/fim.

Introdução #

Os opcionais (também conhecidos como tipos Maybe) são usados ​​para representar um tipo cujo conteúdo pode ou não estar presente. Eles são implementados em C++17 como a classe std::optional. Por exemplo, um objeto do tipo std::optional<int> pode conter algum valor do tipo int, ou pode não conter nenhum valor.

Os opcionais são comumente usados ​​para representar um valor que pode não existir ou como um tipo de retorno de uma função que pode falhar ao retornar um resultado significativo.

Outras abordagens para opcional #

Existem muitas outras abordagens para resolver o problema que std::optional resolve, mas nenhuma delas é completamente completa: usando um ponteiro, usando um sentinela ou usando um pair<bool, T>.

Opcional vs Ponteiro #

Em alguns casos, podemos fornecer um ponteiro para um objeto existente ou nullptr para indicar falha. Mas isso é limitado aos casos em que já existem objetos - opcional, como tipo de valor, também pode ser usado para retornar novos objetos sem recorrer à alocação de memória.

Opcional vs Sentinela #

Um idioma comum é usar um valor especial para indicar que o valor não tem sentido. Pode ser 0 ou -1 para tipos integrais ou nullptr para ponteiros. No entanto, isso reduz o espaço de valores válidos (você não pode diferenciar entre um 0 válido e um 0 sem sentido) e muitos tipos não têm uma escolha natural para o valor sentinela.

Opcional vs std::pair<bool, T> #

Outro idioma comum é fornecer um par, onde um dos elementos é um bool indicando se o valor é significativo ou não.

Isso depende do tipo de valor ser construtível por padrão no caso de erro, o que não é possível para alguns tipos e possível, mas indesejável para outros. Um opcional<T>, no caso de erro, não precisa construir nada.

valor_ou #

Usando opcionais para representar a falha de uma função #

Antes do C++17, uma função normalmente representava falha de várias maneiras:

  • Um ponteiro nulo foi retornado.
    • e.g. Calling a function Delegate *App::get_delegate() on an App instance that did not have a delegate would return nullptr.
    • This is a good solution for objects that have been dynamically allocated or are large and managed by pointers, but isn’t a good solution for small objects that are typically stack-allocated and passed by copying.
  • Um valor específico do tipo de retorno foi reservado para indicar falha.
    • e.g. Calling a function unsigned shortest_path_distance(Vertex a, Vertex b) on two vertices that are not connected may return zero to indicate this fact.
  • O valor foi emparelhado com um bool para indicar que o valor retornado era significativo.
    • e.g. Calling a function std::pair<int, bool> parse(const std::string &str) with a string argument that is not an integer would return a pair with an undefined int and a bool set to false.

Neste exemplo, John recebe dois animais de estimação, Fofo e Bola de Pele. A função Person::pet_with_name() é então chamada para recuperar os bigodes de estimação de John. Como John não tem um animal de estimação chamado Whiskers, a função falha e std::nullopt é retornado.

#include <iostream>
#include <optional>
#include <string>
#include <vector>

struct Animal {
    std::string name;
};

struct Person {
    std::string name;
    std::vector<Animal> pets;
    
    std::optional<Animal> pet_with_name(const std::string &name) {
        for (const Animal &pet : pets) {
            if (pet.name == name) {
                return pet;
            }
        }
        return std::nullopt;
    }
};

int main() {
    Person john;
    john.name = "John";
    
    Animal fluffy;
    fluffy.name = "Fluffy";
    john.pets.push_back(fluffy);
    
    Animal furball;
    furball.name = "Furball";
    john.pets.push_back(furball);
    
    std::optional<Animal> whiskers = john.pet_with_name("Whiskers");
    if (whiskers) {
        std::cout << "John has a pet named Whiskers." << std::endl;
    }
    else {
        std::cout << "Whiskers must not belong to John." << std::endl;
    }
}
