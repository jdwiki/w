Ambiguidades podem ocorrer quando um tipo pode ser convertido implicitamente em mais de um tipo e não há função correspondente para esse tipo específico.

Por exemplo:

void foo(double, double);
void foo(long, long);

//Call foo with 2 ints
foo(1, 2); //Function call is ambiguous - int can be converted into a double/long at the same time

O que é sobrecarga de funções? #

Sobrecarga de função é ter várias funções declaradas no mesmo escopo com exatamente o mesmo nome no mesmo local (conhecido como escopo) diferindo apenas em sua assinatura, ou seja, os argumentos que aceitam.

Suponha que você esteja escrevendo uma série de funções para capacidades de impressão generalizadas, começando com std::string:

void print(const std::string &str)
{
    std::cout << "This is a string: " << str << std::endl;
}

Isso funciona bem, mas suponha que você queira uma função que também aceite um int e imprima isso também. Você poderia escrever:

void print_int(int num)
{
    std::cout << "This is an int:  " << num << std::endl;
}

Mas como as duas funções aceitam parâmetros diferentes, você pode simplesmente escrever:

void print(int num)
{
    std::cout << "This is an int: " << num << std::endl;
}

Agora você tem 2 funções, ambas chamadas print, mas com assinaturas diferentes. Um aceita std::string, o outro um int. Agora você pode chamá-los sem se preocupar com nomes diferentes:

print("Hello world!"); //prints "This is a string: Hello world!"
print(1337);           //prints "This is an int: 1337"

Ao invés de:

print("Hello world!");
print_int(1337);

Quando você tem funções sobrecarregadas, o compilador infere qual das funções chamar a partir dos parâmetros que você fornece. Deve-se tomar cuidado ao escrever sobrecargas de função. Por exemplo, com conversões de tipo implícitas:

void print(int num)
{
    std::cout << "This is an int: " << num << std::endl;
}
void print(double num)
{
    std::cout << "This is a double: " << num << std::endl;
}

Agora não está imediatamente claro qual sobrecarga de print é chamada quando você escreve:

print(5);

E você pode precisar dar algumas pistas ao seu compilador, como:

print(static_cast<double>(5));
print(static_cast<int>(5));
print(5.0);

Alguns cuidados também precisam ser tomados ao escrever sobrecargas que aceitam parâmetros opcionais:

// WRONG CODE
void print(int num1, int num2 = 0)    //num2 defaults to 0 if not included
{ 
    std::cout << "These are ints: << num1 << " and " << num2 << std::endl;
}
void print(int num)
{
    std::cout << "This is an int: " << num << std::endl;
}

Como não há como o compilador saber se uma chamada como print(17) é para a primeira ou segunda função por causa do segundo parâmetro opcional, isso falhará ao compilar.

Tipo de retorno na sobrecarga de funções #

Observe que você não pode sobrecarregar uma função com base em seu tipo de retorno. Por exemplo:

// WRONG CODE
std::string getValue()
{
  return "hello";
}

int getValue()
{
  return 0;
}

int x = getValue();

Isso causará um erro de compilação, pois o compilador não poderá descobrir qual versão de getValue chamar, mesmo que o tipo de retorno seja atribuído a um int.

Sobrecarga do qualificador de cv da função de membro #

As funções dentro de uma classe podem ser sobrecarregadas quando são acessadas por meio de uma referência qualificada de cv para essa classe; isso é mais comumente usado para sobrecarregar para const, mas também pode ser usado para sobrecarregar para volatile e const volátil. Isso ocorre porque todas as funções de membro não estáticas usam this como um parâmetro oculto, ao qual os qualificadores cv são aplicados. Isso é mais comumente usado para sobrecarregar para const, mas também pode ser usado para volatile e const volátil.

Isso é necessário porque uma função-membro só pode ser chamada se for pelo menos tão qualificada como cv quanto a instância em que é chamada. Enquanto uma instância não-const pode chamar membros const e não-const, uma instância const só pode chamar membros const. Isso permite que uma função tenha um comportamento diferente dependendo dos qualificadores cv da instância de chamada e permite que o programador não permita funções para um qualificador(es) de cv indesejado(s) não fornecendo uma versão com esse(s) qualificador(es).

Uma classe com algum método print básico pode ser sobrecarregada com const assim:

#include <iostream>

class Integer
{
    public:
        Integer(int i_): i{i_}{}

        void print()
        {
            std::cout << "int: " << i << std::endl;
        }

        void print() const
        {
            std::cout << "const int: " << i << std::endl;
        }

    protected:
        int i;
};

int main()
{
    Integer i{5};
    const Integer &ic = i;
    
    i.print(); // prints "int: 5"
    ic.print(); // prints "const int: 5"
}

Este é um princípio chave da correção const: Ao marcar as funções-membro como const, elas podem ser chamadas em instâncias const, o que por sua vez permite que as funções tomem instâncias como ponteiros/referências const se não não precisa modificá-los. Isso permite que o código especifique se modifica o estado tomando parâmetros não modificados como const e parâmetros modificados sem qualificadores cv, tornando o código mais seguro e mais legível.

class ConstCorrect 
{
  public:
    void good_func() const 
    {
        std::cout << "I care not whether the instance is const." << std::endl;
    }

    void bad_func() 
    {
        std::cout << "I can only be called on non-const, non-volatile instances." << std::endl;
    }
};

void i_change_no_state(const ConstCorrect& cc) 
{
    std::cout << "I can take either a const or a non-const ConstCorrect." << std::endl;
    cc.good_func(); // Good.  Can be called from const or non-const instance.
    cc.bad_func();  // Error.  Can only be called from non-const instance.
}

void const_incorrect_func(ConstCorrect& cc) 
{
    cc.good_func(); // Good.  Can be called from const or non-const instance.
    cc.bad_func();  // Good.  Can only be called from non-const instance.
}

Um uso comum disso é declarar os acessadores como const e os modificadores como não const.

Nenhum membro de classe pode ser modificado dentro de uma função de membro const. Se houver algum membro que você realmente precise modificar, como bloquear um std::mutex, você pode declará-lo como mutable:

class Integer
{
    public:
        Integer(int i_): i{i_}{}

        int get() const
        {
            std::lock_guard<std::mutex> lock{mut};
            return i;
        }

        void set(int i_)
        {
            std::lock_guard<std::mutex> lock{mut};
            i = i_;
        }

    protected:
        int i;
        mutable std::mutex mut;
};
