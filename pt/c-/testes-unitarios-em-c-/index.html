Testes Unitários em C++

  • Teste do Google
  • Truque

Nesta página

  • Teste do Google
  • Truque

O teste de unidade é um nível de teste de software que valida o comportamento e a correção das unidades de código.

Em C++, “unidades de código” geralmente se referem a classes, funções ou grupos de ambos. O teste de unidade geralmente é realizado usando “estruturas de teste” especializadas ou “bibliotecas de teste” que geralmente usam sintaxe ou padrões de uso não triviais.

Este tópico revisará diferentes estratégias e bibliotecas ou estruturas de teste de unidade.

Teste do Google #

O Google Test é uma estrutura de teste C++ mantida pelo Google. Requer a criação da biblioteca gtest e a vinculação dela à sua estrutura de teste ao criar um arquivo de caso de teste.

Exemplo Mínimo #

// main.cpp

#include <gtest/gtest.h>
#include <iostream>

// Google Test test cases are created using a C++ preprocessor macro
// Here, a "test suite" name and a specific "test name" are provided.
TEST(module_name, test_name) {
    std::cout << "Hello world!" << std::endl;
    // Google Test will also provide macros for assertions.
    ASSERT_EQ(1+1, 2);
}

// Google Test can be run manually from the main() function
// or, it can be linked to the gtest_main library for an already
// set-up main() function primed to accept Google Test test cases.
int main(int argc, char** argv) {
    ::testing::InitGoogleTest(&argc, argv);

    return RUN_ALL_TESTS();
}

// Build command: g++ main.cpp -lgtest

Truque #

Catch é uma biblioteca apenas de cabeçalho que permite usar TDD e BDD estilo de teste de unidade.

O trecho a seguir é da página de documentação do Catch em este link:

<pré><código> SCENARIO( “os vetores podem ser dimensionados e redimensionados”, “[vetor]” ) { GIVEN( “A vector with some items” ) { std::vector v( 5 );

    REQUIRE( v.size() == 5 );
    REQUIRE( v.capacity() >= 5 );
    
    WHEN( "the size is increased" ) {
        v.resize( 10 );
        
        THEN( "the size and capacity change" ) {
            REQUIRE( v.size() == 10 );
            REQUIRE( v.capacity() >= 10 );
        }
    }
    WHEN( "the size is reduced" ) {
        v.resize( 0 );
        
        THEN( "the size changes but not capacity" ) {
            REQUIRE( v.size() == 0 );
            REQUIRE( v.capacity() >= 5 );
        }
    }
    WHEN( "more capacity is reserved" ) {
        v.reserve( 10 );
        
        THEN( "the capacity changes but not the size" ) {
            REQUIRE( v.size() == 5 );
            REQUIRE( v.capacity() >= 10 );
        }
    }
    WHEN( "less capacity is reserved" ) {
        v.reserve( 0 );
        
        THEN( "neither size nor capacity are changed" ) {
            REQUIRE( v.size() == 5 );
            REQUIRE( v.capacity() >= 5 );
        }
    }
}

}

Convenientemente, esses testes serão relatados da seguinte forma quando executados:

<pré> Cenário: os vetores podem ser dimensionados e redimensionados Given: A vector with some items When: more capacity is reserved Then: the capacity changes but not the size
