Arquivos de cabeçalho

  • Exemplo básico
  • Modelos em arquivos de cabeçalho

Nesta página

  • Exemplo básico
  • Modelos em arquivos de cabeçalho

Em C++, como em C, o compilador C++ e o processo de compilação fazem uso do pré-processador C. Conforme especificado pelo manual do GNU C Preprocessor, um arquivo de cabeçalho é definido como o seguinte:

Um arquivo de cabeçalho é um arquivo contendo declarações C e macro definições (veja Macros) a serem compartilhadas entre vários arquivos de origem. Você solicita o uso de um arquivo de cabeçalho em seu programa incluindo-o, com a diretiva de pré-processamento C ‘#include’.

Os arquivos de cabeçalho servem a dois propósitos.

  • Os arquivos de cabeçalho do sistema declaram as interfaces para partes do sistema operacional. Você os inclui em seu programa para fornecer o definições e declarações que você precisa para invocar chamadas de sistema e bibliotecas.
  • Seus próprios arquivos de cabeçalho contêm declarações para interfaces entre os arquivos de origem do seu programa. Cada vez que você tem um grupo de declarações relacionadas e definições de macro, todas ou a maioria delas necessário em vários arquivos de origem diferentes, é uma boa ideia criar um arquivo de cabeçalho para eles.

No entanto, para o próprio pré-processador C, um arquivo de cabeçalho não é diferente de um arquivo de origem.

O esquema de organização de arquivo de cabeçalho/fonte é simplesmente uma convenção padrão e fortemente mantida definida por vários projetos de software para fornecer separação entre interface e implementação.

Embora não seja formalmente imposto pelo próprio padrão C++, seguir a convenção de arquivo de cabeçalho/fonte é altamente recomendado e, na prática, já é quase onipresente.

Observe que os arquivos de cabeçalho podem ser substituídos como uma convenção de estrutura de arquivos do projeto pelo próximo recurso de módulos, que ainda deve ser considerado para inclusão em um futuro padrão C++ no momento da redação (por exemplo, C++ 20).

Exemplo básico #

O exemplo a seguir conterá um bloco de código que deve ser dividido em vários arquivos de origem, conforme indicado pelos comentários // filename.

Arquivos Fonte #

// my_function.h

/* Note how this header contains only a declaration of a function.
 * Header functions usually do not define implementations for declarations
 * unless code must be further processed at compile time, as in templates.
 */

/* Also, usually header files include preprocessor guards so that every header
 * is never included twice.
 *
 * The guard is implemented by checking if a header-file unique preprocessor
 * token is defined, and only including the header if it hasn't been included
 * once before.
 */
#ifndef MY_FUNCTION_H
#define MY_FUNCTION_H

// global_value and my_function() will be
// recognized as the same constructs if this header is included by different files.
const int global_value = 42;
int my_function();

#endif // MY_FUNCTION_H

// my_function.cpp

/* Note how the corresponding source file for the header includes the interface  
 * defined in the header so that the compiler is aware of what the source file is 
 * implementing.
 *
 * In this case, the source file requires knowledge of the global constant
 * global_value only defined in my_function.h. Without inclusion of the header
 * file, this source file would not compile.
 */
#include "my_function.h" // or #include "my_function.hpp"
int my_function() {
  return global_value; // return 42;
}

Os arquivos de cabeçalho são incluídos por outros arquivos de origem que desejam usar a funcionalidade definida pela interface de cabeçalho, mas não exigem conhecimento de sua implementação (reduzindo assim o acoplamento de código). O programa a seguir faz uso do cabeçalho my_function.h conforme definido acima:

// main.cpp

#include <iostream>       // A C++ Standard Library header.
#include "my_function.h"  // A personal header

int main(int argc, char** argv) {
  std::cout << my_function() << std::endl;
  return 0;
}

O processo de compilação #

Como os arquivos de cabeçalho geralmente fazem parte de um fluxo de trabalho do processo de compilação, um processo de compilação típico que usa a convenção de arquivo de cabeçalho/fonte geralmente fará o seguinte.

Supondo que o arquivo de cabeçalho e o arquivo de código-fonte já estejam no mesmo diretório, um programador executaria os seguintes comandos:

g++ -c my_function.cpp       # Compiles the source file my_function.cpp
                             # --> object file my_function.o

g++ main.cpp my_function.o   # Links the object file containing the 
                             # implementation of int my_function()
                             # to the compiled, object version of main.cpp
                             # and then produces the final executable a.out

Alternativamente, se alguém deseja compilar o main.cpp para um arquivo objeto primeiro, e em seguida, vincule apenas os arquivos de objeto como a etapa final:

g++ -c my_function.cpp
g++ -c main.cpp

g++ main.o my_function.o

Modelos em arquivos de cabeçalho #

Os modelos exigem geração de código em tempo de compilação: uma função de modelo, por exemplo, será efetivamente transformada em várias funções distintas assim que uma função de modelo for parametrizada pelo uso no código-fonte.

Isso significa que a função de modelo, função de membro e definições de classe não podem ser delegadas a um arquivo de código-fonte separado, pois qualquer código que usará qualquer construção de modelo requer conhecimento de sua definição para geralmente gerar qualquer código derivado.

Assim, o código modelo, se colocado em cabeçalhos, também deve conter sua definição. Um exemplo disso está abaixo:

// templated_function.h

template <typename T>
T* null_T_pointer() {
  T* type_point = NULL; // or, alternatively, nullptr instead of NULL
                        // for C++11 or later
  return type_point;
}
