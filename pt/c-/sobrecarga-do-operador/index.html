Em C++, é possível definir operadores como + e -> para tipos definidos pelo usuário. Por exemplo, o cabeçalho <string> define um operador + para concatenar strings. Isso é feito definindo uma função de operador usando o operador palavra-chave.

Os operadores para tipos internos não podem ser alterados, os operadores só podem ser sobrecarregados para tipos definidos pelo usuário. Ou seja, pelo menos um dos operandos deve ser de um tipo definido pelo usuário.

Os seguintes operadores não podem ser sobrecarregados:

  • O acesso de membro ou operador “ponto” .
  • O ponteiro para o operador de acesso de membro .*
  • O operador de resolução de escopo, ::
  • O operador condicional ternário, ?:
  • dynamic_cast, static_cast, reinterpret_cast, const_cast, typeid, sizeof, alignof e noexcept
  • As diretivas de pré-processamento, # e ##, que são executadas antes que qualquer informação de tipo esteja disponível.

Existem alguns operadores que você não (99,98% do tempo) deve sobrecarregar:

  • && e || (prefira, em vez disso, usar a conversão implícita para bool)
  • ,
  • O operador address-of (unário &)

Por quê? Porque eles sobrecarregam os operadores que outro programador nunca poderia esperar, resultando em um comportamento diferente do previsto.

Por exemplo, o usuário definiu sobrecargas && e || desses operadores perdem sua avaliação de curto-circuito e perdem suas propriedades especiais de sequenciamento (C++17), o problema de sequenciamento também se aplica a sobrecargas do operador ,.

Operador de subscrito de matriz #

Você pode até sobrecarregar o operador de subscrito de array [].

Você deve sempre (99,98% do tempo) implementar 2 versões, uma versão const e uma não-const, porque se o objeto for const, ele não poderá ser modificado o objeto retornado por [].

Os argumentos são passados ​​por const& em vez de por valor porque passar por referência é mais rápido que por valor, e const para que o operador não altere o índice acidentalmente.

Os operadores retornam por referência, pois por design você pode modificar o objeto [] return, ou seja:

std::vector<int> v{ 1 };
v[0] = 2; //Changes value of 1 to 2
          //wouldn't be possible if not returned by reference

Você pode somente sobrecarregar dentro de uma class/struct:

//I is the index type, normally an int
T& operator[](const I& index)
{
    //Do something
    //return something
}

//I is the index type, normally an int
const T& operator[](const I& index) const
{
    //Do something
    //return something
}

Vários operadores de subscrito, [][]..., podem ser obtidos por meio de objetos proxy. O exemplo a seguir de uma classe simples de matriz de linha principal demonstra isso:

template<class T>
class matrix {
    // class enabling [][] overload to access matrix elements
    template <class C>
    class proxy_row_vector {
        using reference = decltype(std::declval<C>()[0]);
        using const_reference = decltype(std::declval<C const>()[0]);
    public:
        proxy_row_vector(C& _vec, std::size_t _r_ind, std::size_t _cols)
            : vec(_vec), row_index(_r_ind), cols(_cols) {}
        const_reference operator[](std::size_t _col_index) const {
            return vec[row_index*cols + _col_index];
        }
        reference operator[](std::size_t _col_index) {
            return vec[row_index*cols + _col_index];
        }
    private:
        C& vec;
        std::size_t row_index; // row index to access
        std::size_t cols; // number of columns in matrix
    };

    using const_proxy = proxy_row_vector<const std::vector<T>>;
    using proxy = proxy_row_vector<std::vector<T>>;
public:
    matrix() : mtx(), rows(0), cols(0) {}
    matrix(std::size_t _rows, std::size_t _cols)
        : mtx(_rows*_cols), rows(_rows), cols(_cols) {}

    // call operator[] followed by another [] call to access matrix elements
    const_proxy operator[](std::size_t _row_index) const {
        return const_proxy(mtx, _row_index, cols);
    }

    proxy operator[](std::size_t _row_index) {
        return proxy(mtx, _row_index, cols);
    }
private:
    std::vector<T> mtx;
    std::size_t rows;
    std::size_t cols;
};

Operadores aritméticos #

Você pode sobrecarregar todos os operadores aritméticos básicos:

  • + e +=
  • - e -=
  • * e *=
  • / e /=
  • & e &=
  • | e |=
  • ^ e ^=
  • >> e >>=
  • << e <<=

A sobrecarga para todos os operadores é a mesma. Role para baixo para obter uma explicação

Sobrecarregando fora de class/struct:

//operator+ should be implemented in terms of operator+=
T operator+(T lhs, const T& rhs)
{
    lhs += rhs;
    return lhs;
}

T& operator+=(T& lhs, const T& rhs)
{
    //Perform addition
    return lhs;
}

Sobrecarregando dentro de class/struct:

//operator+ should be implemented in terms of operator+=
T operator+(const T& rhs)
{
    *this += rhs;
    return *this;
}

T& operator+=(const T& rhs)
{
    //Perform addition
    return *this;
}

Nota: operator+ deve retornar por um valor não const, pois retornar uma referência não faria sentido (ele retorna um objeto novo) nem retornar um valor const (você geralmente não deve retornar por const). O primeiro argumento é passado por valor, por quê? Porque

  1. Você não pode modificar o objeto original (Object foobar = foo + bar; não deve modificar foo afinal, não faria sentido)
  2. Você não pode torná-lo const, porque você terá que ser capaz de modificar o objeto (porque operator+ é implementado em termos de operator+=, que modifica o objeto)

Passar por const& seria uma opção, mas então você terá que fazer uma cópia temporária do objeto passado. Ao passar por valor, o compilador faz isso para você.

operator+= retorna uma referência ao próprio, porque então é possível encadeá-los (mas não use a mesma variável, isso seria um comportamento indefinido devido a pontos de sequência).

O primeiro argumento é uma referência (queremos modificá-lo), mas não const, porque então você não poderá modificá-lo. O segundo argumento não deve ser modificado e, portanto, por motivos de desempenho, é passado por const& (passar por referência const é mais rápido que por valor).

Operadores de conversão #

Você pode sobrecarregar os operadores de tipo, para que seu tipo possa ser convertido implicitamente no tipo especificado.

O operador de conversão deve ser definido em uma class/struct:

operator T() const { /* return something */ }

Observação: o operador é const para permitir que objetos const sejam convertidos.

Exemplo:

struct Text
{
    std::string text;

    // Now Text can be implicitly converted into a const char*
    /*explicit*/ operator const char*() const { return text.data(); }
    // ^^^^^^^
    // to disable implicit conversion
};

Text t;
t.text = "Hello world!";

//Ok
const char* copyoftext = t;

Números complexos revisitados #

O código abaixo implementa um tipo de número complexo muito simples para o qual o campo subjacente é promovido automaticamente, seguindo as regras de promoção de tipo da linguagem, sob aplicação dos quatro operadores básicos (+, -, * e /) com um membro de um campo diferente (seja outro complex<T> ou algum tipo escalar).

Este pretende ser um exemplo holístico que abrange a sobrecarga do operador juntamente com o uso básico de modelos.

#include <type_traits>

namespace not_std{

using std::decay_t;

//----------------------------------------------------------------
// complex< value_t >
//----------------------------------------------------------------

template<typename value_t>
struct complex
{
    value_t x;
    value_t y;

    complex &operator += (const value_t &x)
    {
        this->x += x;
        return *this;
    }
    complex &operator += (const complex &other)
    {
        this->x += other.x;
        this->y += other.y;
        return *this;
    }

    complex &operator -= (const value_t &x)
    {
        this->x -= x;
        return *this;
    }
    complex &operator -= (const complex &other)
    {
        this->x -= other.x;
        this->y -= other.y;
        return *this;
    }

    complex &operator *= (const value_t &s)
    {
        this->x *= s;
        this->y *= s;
        return *this;
    }
    complex &operator *= (const complex &other)
    {
        (*this) = (*this) * other;
        return *this;
    }

    complex &operator /= (const value_t &s)
    {
        this->x /= s;
        this->y /= s;
        return *this;
    }
    complex &operator /= (const complex &other)
    {
        (*this) = (*this) / other;
        return *this;
    }

    complex(const value_t &x, const value_t &y)
    : x{x}
    , y{y}
    {}

    template<typename other_value_t>
    explicit complex(const complex<other_value_t> &other)
    : x{static_cast<const value_t &>(other.x)}
    , y{static_cast<const value_t &>(other.y)}
    {}

    complex &operator = (const complex &) = default;
    complex &operator = (complex &&) = default;
    complex(const complex &) = default;
    complex(complex &&) = default;
    complex() = default;
};

// Absolute value squared
template<typename value_t>
value_t absqr(const complex<value_t> &z)
{ return z.x*z.x + z.y*z.y; }

{
    return {
        a.x*b.x - a.y*b.y,
        a.x*b.y + a.y*b.x
        };

{ return {a.x * b, a.y * b}; }

// operator /
//----------------------------------------------------------------

template<typename left_t, typename right_t>
auto operator / (const complex<left_t> &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a.x / b.x)>>
{
    const auto r = absqr(b);
    return {
        ( a.x*b.x + a.y*b.y) / r,
        (-a.x*b.y + a.y*b.x) / r
        };
}

template<typename left_t, typename right_t>
auto operator / (const left_t &a, const complex<right_t> &b)
-> complex<decay_t<decltype(a / b.x)>>
{
    const auto s = a/absqr(b);
    return {
         b.x * s,
        -b.y * s
        };
}

template<typename left_t, typename right_t>
auto operator / (const complex<left_t> &a, const right_t &b)
-> complex<decay_t<decltype(a.x / b)>>
{ return {a.x / b, a.y / b}; }

Operadores unários #

Você pode sobrecarregar os 2 operadores unários:

  • ++foo e foo++
  • --foo e foo--

A sobrecarga é a mesma para ambos os tipos (++ e --). Role para baixo para obter uma explicação

Sobrecarregando fora de class/struct:

//Prefix operator ++foo
T& operator++(T& lhs)
{
    //Perform addition
    return lhs;
}

//Postfix operator foo++ (int argument is used to separate pre- and postfix) 
//Should be implemented in terms of ++foo (prefix operator)
T operator++(T& lhs, int)
{
    T t(lhs);
    ++lhs;
    return t;
}

Sobrecarregando dentro de class/struct:

//Prefix operator ++foo
T& operator++()
{
    //Perform addition
    return *this;
}

//Postfix operator foo++ (int argument is used to separate pre- and postfix) 
//Should be implemented in terms of ++foo (prefix operator)
T operator++(int)
{
    T t(*this);
    ++(*this);
    return t;
}

Nota: O operador prefix retorna uma referência a si mesmo, para que você possa continuar as operações nele. O primeiro argumento é uma referência, já que o operador prefix altera o objeto, essa também é a razão pela qual não é const (você não poderia modificá-lo de outra forma).

O operador postfix retorna por valor um temporário (o valor anterior), e portanto não pode ser uma referência, pois seria uma referência a um temporário, que seria um valor lixo no final da função, pois a variável temporária se apaga de escopo). Também não pode ser const, porque você deve poder modificá-lo diretamente.

O primeiro argumento é uma referência não const ao objeto “calling”, porque se fosse const, você não poderia modificá-lo, e se não fosse uma referência, você não mudaria o valor originário.

É por causa da cópia necessária em sobrecargas de operadores postfix que é melhor criar o hábito de usar prefix ++ em vez de postfix ++ em loops for. Da perspectiva do loop for, eles geralmente são funcionalmente equivalentes, mas pode haver uma pequena vantagem de desempenho em usar o prefixo ++, especialmente com classes “fat” com muitos membros para copiar. Exemplo de uso do prefixo ++ em um loop for:

for (list<string>::const_iterator it = tokens.begin();
     it != tokens.end();
     ++it) { // Don't use it++
    ...
}

Operadores de comparação #

Você pode sobrecarregar todos os operadores de comparação:

  • == e !=
  • > e <
  • >= e <=

A maneira recomendada de sobrecarregar todos esses operadores é implementar apenas 2 operadores (== e <) e usá-los para definir o restante. Role para baixo para obter uma explicação

Sobrecarregando fora de class/struct:

//Only implement those 2
bool operator==(const T& lhs, const T& rhs) { /* Compare */ }
bool operator<(const T& lhs, const T& rhs) { /* Compare */ }

//Now you can define the rest
bool operator!=(const T& lhs, const T& rhs) { return !(lhs == rhs); }
bool operator>(const T& lhs, const T& rhs) { return rhs < lhs; }
bool operator<=(const T& lhs, const T& rhs) { return !(lhs > rhs); }
bool operator>=(const T& lhs, const T& rhs) { return !(lhs < rhs); }

Sobrecarregando dentro de class/struct:

//Note that the functions are const, because if they are not const, you wouldn't be able
//to call them if the object is const

//Only implement those 2
bool operator==(const T& rhs) const { /* Compare */ }
bool operator<(const T& rhs) const { /* Compare */ }

//Now you can define the rest
bool operator!=(const T& rhs) const { return !(*this == rhs); }
bool operator>(const T& rhs) const { return rhs < *this; }
bool operator<=(const T& rhs) const { return !(*this > rhs); }
bool operator>=(const T& rhs) const { return !(*this < rhs); }

Os operadores obviamente retornam um bool, indicando true ou false para a operação correspondente.

Todos os operadores recebem seus argumentos por const&, porque a única coisa que os operadores fazem é comparar, então eles não devem modificar os objetos. Passar por & (referência) é mais rápido que por valor, e para garantir que os operadores não o modifiquem, é uma referência const.

Observe que os operadores dentro de class/struct são definidos como const, a razão para isso é que sem as funções serem const, comparar objetos const não seria possível, pois o compilador não saiba que os operadores não modificam nada.

Operador de atribuição #

O operador de atribuição é um dos operadores mais importantes porque permite alterar o status de uma variável.

Se você não sobrecarregar o operador de atribuição para sua class/struct, ele é gerado automaticamente pelo compilador: o operador de atribuição gerado automaticamente executa uma “atribuição de membro”, ou seja, invocando operadores de atribuição em todos os membros, de modo que um objeto é copiado para o outro, um membro por vez. O operador de atribuição deve ser sobrecarregado quando a atribuição simples de membro não for adequada para sua class/struct, por exemplo, se você precisar executar uma cópia profunda de um objeto.

Sobrecarregar o operador de atribuição = é fácil, mas você deve seguir alguns passos simples.

  1. Teste para autoatribuição. Essa verificação é importante por dois motivos:

    • a self-assignment is a needless copy, so it does not make sense to perform it;
    • the next step will not work in the case of a self-assignment.

  2. Limpe os dados antigos. Os dados antigos devem ser substituídos por novos. Agora, você pode entender o segundo motivo da etapa anterior: se o conteúdo do objeto foi destruído, uma auto-atribuição não conseguirá realizar a cópia.

  3. Copiar todos os membros. Se você sobrecarregar o operador de atribuição para sua class ou struct, ele não será gerado automaticamente pelo compilador, então você precisará se encarregar de copiar todos os membros do outro objeto.

  4. Retornar *este. O operador retorna sozinho por referência, pois permite encadeamento (ou seja, int b = (a = 6) + 4; //b == 10).

    //T is some type T& operator=(const T& other) { //Do something (like copying values) return *this; }

Observação: other é passado por const&, porque o objeto que está sendo atribuído não deve ser alterado, e a passagem por referência é mais rápida do que por valor, e para garantir que operator= não o modifique acidentalmente, é const.

O operador de atribuição pode somente ser sobrecarregado na class/struct, porque o valor à esquerda de = é sempre a própria class/struct. Defini-la como uma função livre não tem essa garantia e não é permitida por causa disso.

Quando você declara no class/struct, o valor da esquerda é implicitamente o próprio class/struct, então não há problema com isso.

Operadores nomeados #

Você pode estender C++ com operadores nomeados que são “citados” por operadores C++ padrão.

Primeiro começamos com uma biblioteca de doze linhas:

namespace named_operator {
  template<class D>struct make_operator{constexpr make_operator(){}};

  template<class T, char, class O> struct half_apply { T&& lhs; };

  template<class Lhs, class Op>
  half_apply<Lhs, '*', Op> operator*( Lhs&& lhs, make_operator<Op> ) {
    return {std::forward<Lhs>(lhs)};
  }

  template<class Lhs, class Op, class Rhs>
  auto operator*( half_apply<Lhs, '*', Op>&& lhs, Rhs&& rhs )
  -> decltype( named_invoke( std::forward<Lhs>(lhs.lhs), Op{}, std::forward<Rhs>(rhs) ) )
  {
    return named_invoke( std::forward<Lhs>(lhs.lhs), Op{}, std::forward<Rhs>(rhs) );
  }
}

isso não faz nada ainda.

Primeiro, anexando vetores

namespace my_ns {
  struct append_t : named_operator::make_operator<append_t> {};
  constexpr append_t append{};
  
  template<class T, class A0, class A1>
  std::vector<T, A0> named_invoke( std::vector<T, A0> lhs, append_t, std::vector<T, A1> const& rhs ) {
      lhs.insert( lhs.end(), rhs.begin(), rhs.end() );
      return std::move(lhs);
  }
}
using my_ns::append;

std::vector<int> a {1,2,3};
std::vector<int> b {4,5,6};

auto c = a *append* b;

O núcleo aqui é que definimos um objeto append do tipo append_t:named_operator::make_operator<append_t>.

Em seguida, sobrecarregamos named_invoke( lhs, append_t, rhs ) para os tipos que queremos à direita e à esquerda.

A biblioteca sobrecarrega lhs*append_t, retornando um objeto temporário half_apply. Ele também sobrecarrega half_apply*rhs para chamar named_invoke( lhs, append_t, rhs ).

Nós simplesmente temos que criar o token append_t apropriado e fazer um named_invoke compatível com ADL da assinatura apropriada, e tudo se conecta e funciona.

Para um exemplo mais complexo, suponha que você queira multiplicar elementos de elementos de um std::array:

template<class=void, std::size_t...Is>
auto indexer( std::index_sequence<Is...> ) {
  return [](auto&& f) {
    return f( std::integral_constant<std::size_t, Is>{}... );
  };
}
template<std::size_t N>
auto indexer() { return indexer( std::make_index_sequence<N>{} ); }

namespace my_ns {
  struct e_times_t : named_operator::make_operator<e_times_t> {};
  constexpr e_times_t e_times{};

  template<class L, class R, std::size_t N,
    class Out=std::decay_t<decltype( std::declval<L const&>()*std::declval<R const&>() )>
  >
  std::array<Out, N> named_invoke( std::array<L, N> const& lhs, e_times_t, std::array<R, N> const& rhs ) {
    using result_type = std::array<Out, N>;
    auto index_over_N = indexer<N>();
    return index_over_N([&](auto...is)->result_type {
      return {{
        (lhs[is] * rhs[is])...
      }};
    });
  }
}

Operador NOT bit a bit #

Sobrecarregar o NOT bit a bit (~) é bastante simples. Role para baixo para obter uma explicação

Sobrecarregando fora de class/struct:

T operator~(T lhs)
{
    //Do operation
    return lhs;
}

Sobrecarregando dentro de class/struct:

T operator~()
{
    T t(*this);
    //Do operation
    return t;
}

Nota: operator~ retorna por valor, pois tem que retornar um novo valor (o valor modificado), e não uma referência ao valor (seria uma referência ao objeto temporário, que teria um valor lixo como assim que o operador terminar). Não const porque o código de chamada deve poder modificá-lo posteriormente (ou seja, int a = ~a + 1; deve ser possível).

Dentro da class/struct você tem que fazer um objeto temporário, pois não pode modificar this, pois modificaria o objeto original, o que não deveria ser o caso.

Operadores de deslocamento de bits para E/S #

Os operadores << e >> são comumente usados ​​como operadores “write” e “read”:

  • std::ostream sobrecarrega << para escrever variáveis ​​no stream subjacente (exemplo: std::cout)
  • std::istream sobrecarrega >> para ler do fluxo subjacente para uma variável (exemplo: std::cin)

A maneira como eles fazem isso é semelhante se você quiser sobrecarregá-los “normalmente” fora da class/struct, exceto que especificar os argumentos não são do mesmo tipo:

  • O tipo de retorno é o fluxo do qual você deseja sobrecarregar (por exemplo, std::ostream) passado por referência, para permitir o encadeamento (Encadeamento: std::cout << a << b;). Exemplo: std::ostream&
  • lhs seria o mesmo que o tipo de retorno
  • rhs é o tipo do qual você deseja permitir a sobrecarga (ou seja, T), passado por const& em vez de valor por motivo de desempenho (rhs não deve ser alterado de qualquer maneira). Exemplo: const Vetor&.

Exemplo:

//Overload std::ostream operator<< to allow output from Vector's
std::ostream& operator<<(std::ostream& lhs, const Vector& rhs)
{
    lhs << "x: " << rhs.x << " y: " << rhs.y << " z: " << rhs.z << '\n';
    return lhs;
}
