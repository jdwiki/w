A E/S de arquivo C++ é feita por meio de streams. As principais abstrações são:

std::istream para leitura de texto.

std::ostream para escrever texto.

std::streambuf para ler ou escrever caracteres.

Entrada formatada usa operador>>.

Saída formatada usa operator<<.

Streams usam std::locale, por exemplo, para detalhes da formatação e para tradução entre codificações externas e a codificação interna.

Mais sobre fluxos: https://www.wikiod.com/pt/docs/c%2b%2b/7660/iostream-library

Gravando em um arquivo #

Existem várias maneiras de gravar em um arquivo. A maneira mais fácil é usar um fluxo de arquivo de saída (ofstream) junto com o operador de inserção de fluxo (<<):

std::ofstream os("foo.txt");
if(os.is_open()){
    os << "Hello World!";
}

Em vez de <<, você também pode usar a função membro do fluxo do arquivo de saída write():

std::ofstream os("foo.txt");
if(os.is_open()){
    char data[] = "Foo";

    // Writes 3 characters from data -> "Foo".
    os.write(data, 3);
}

Depois de gravar em um fluxo, você deve sempre verificar se o sinalizador de estado de erro badbit foi definido, pois indica se a operação falhou ou não. Isso pode ser feito chamando a função membro do stream do arquivo de saída bad():

os << "Hello Badbit!"; // This operation might fail for any reason.
if (os.bad())
    // Failed to write!

Abrindo um arquivo #

A abertura de um arquivo é feita da mesma forma para todos os 3 fluxos de arquivos (ifstream, ofstream e fstream).

Você pode abrir o arquivo diretamente no construtor:

std::ifstream ifs("foo.txt");  // ifstream: Opens file "foo.txt" for reading only.

std::ofstream ofs("foo.txt");  // ofstream: Opens file "foo.txt" for writing only.

std::fstream iofs("foo.txt");  // fstream:  Opens file "foo.txt" for reading and writing.

Alternativamente, você pode usar a função membro do fluxo de arquivos open():

std::ifstream ifs;
ifs.open("bar.txt");           // ifstream: Opens file "bar.txt" for reading only.

std::ofstream ofs;
ofs.open("bar.txt");           // ofstream: Opens file "bar.txt" for writing only.

std::fstream iofs;
iofs.open("bar.txt");          // fstream:  Opens file "bar.txt" for reading and writing.

Você deve sempre verificar se um arquivo foi aberto com sucesso (mesmo durante a gravação). As falhas podem incluir: o arquivo não existe, o arquivo não tem os direitos de acesso corretos, o arquivo já está em uso, ocorreram erros de disco, unidade desconectada … A verificação pode ser feita da seguinte forma:

// Try to read the file 'foo.txt'.
std::ifstream ifs("fooo.txt");  // Note the typo; the file can't be opened.

// Check if the file has been opened successfully.
if (!ifs.is_open()) {
    // The file hasn't been opened; take appropriate actions here.
    throw CustomException(ifs, "File could not be opened");
}

Quando o caminho do arquivo contém barras invertidas (por exemplo, no sistema Windows), você deve escapá-las corretamente:

// Open the file 'c:\folder\foo.txt' on Windows.
std::ifstream ifs("c:\\folder\\foo.txt"); // using escaped backslashes

ou use literal bruto:

// Open the file 'c:\folder\foo.txt' on Windows.
std::ifstream ifs(R"(c:\folder\foo.txt)"); // using raw literal

ou use barras em vez disso:

// Open the file 'c:\folder\foo.txt' on Windows.
std::ifstream ifs("c:/folder/foo.txt");

Se você deseja abrir um arquivo com caracteres não ASCII no caminho no Windows atualmente, você pode usar o argumento de caminho de caractere largo não padrão:

// Open the file 'пример\foo.txt' on Windows.
std::ifstream ifs(LR"(пример\foo.txt)"); // using wide characters with raw literal

Lendo de um arquivo #

Existem várias maneiras de ler dados de um arquivo.

Se você souber como os dados são formatados, poderá usar o operador de extração de fluxo (>>). Vamos supor que você tenha um arquivo chamado foo.txt que contém os seguintes dados:

John Doe 25 4 6 1987
Jane Doe 15 5 24 1976

Em seguida, você pode usar o seguinte código para ler esses dados do arquivo:

// Define variables.
std::ifstream is("foo.txt");
std::string firstname, lastname;
int age, bmonth, bday, byear;

// Extract firstname, lastname, age, bday month, bday day, and bday year in that order.
// Note: '>>' returns false if it reached EOF (end of file) or if the input data doesn't
// correspond to the type of the input variable (for example, the string "foo" can't be
// extracted into an 'int' variable).
while (is >> firstname >> lastname >> age >> bmonth >> bday >> byear)
    // Process the data that has been read.

O operador de extração de fluxo >> extrai todos os caracteres e para se encontrar um caractere que não pode ser armazenado ou se for um caractere especial:

  • Para tipos de string, o operador para em um espaço em branco ( ) ou em uma nova linha (\n).
  • Para números, o operador para em um caractere não numérico.

Isso significa que a seguinte versão do arquivo foo.txt também será lida com sucesso pelo código anterior:

John 
Doe 25
4 6 1987


Jane
Doe 
15 5
24
1976

O operador de extração de fluxo >> sempre retorna o fluxo fornecido a ele. Portanto, vários operadores podem ser encadeados para ler os dados consecutivamente. No entanto, um fluxo também pode ser usado como uma expressão booleana (como mostrado no loop while no código anterior). Isso ocorre porque as classes de fluxo possuem um operador de conversão para o tipo bool. Este operador bool() retornará true desde que o fluxo não tenha erros. Se um fluxo entrar em um estado de erro (por exemplo, porque nenhum dado pode ser extraído), então o operador bool() retornará false. Portanto, o loop while no código anterior será encerrado após o arquivo de entrada ter sido lido até o fim.

Se você deseja ler um arquivo inteiro como uma string, você pode usar o seguinte código:

// Opens 'foo.txt'.
std::ifstream is("foo.txt");
std::string whole_file;

// Sets position to the end of the file.
is.seekg(0, std::ios::end);

// Reserves memory for the file.
whole_file.reserve(is.tellg());

// Sets position to the start of the file.
is.seekg(0, std::ios::beg);

// Sets contents of 'whole_file' to all characters in the file.
whole_file.assign(std::istreambuf_iterator<char>(is),
  std::istreambuf_iterator<char>());

Este código reserva espaço para a string para reduzir alocações de memória desnecessárias.

Se você quiser ler um arquivo linha por linha, você pode usar a função getline():

std::ifstream is("foo.txt");   

// The function getline returns false if there are no more lines.
for (std::string str; std::getline(is, str);) {
    // Process the line that has been read.
}

Se você quiser ler um número fixo de caracteres, você pode usar a função membro do stream read():

std::ifstream is("foo.txt");
char str[4];

// Read 4 characters from the file.
is.read(str, 4);

Após executar um comando de leitura, você deve sempre verificar se o flag de estado de erro failbit foi definido, pois indica se a operação falhou ou não. Isso pode ser feito chamando a função membro do stream de arquivos fail():

is.read(str, 4); // This operation might fail for any reason.

if (is.fail())
    // Failed to read!

Modos de abertura #

Ao criar um fluxo de arquivos, você pode especificar um modo de abertura. Um modo de abertura é basicamente uma configuração para controlar como o fluxo abre o arquivo.

(Todos os modos podem ser encontrados no namespace std::ios.)

Um modo de abertura pode ser fornecido como segundo parâmetro para o construtor de um fluxo de arquivo ou para sua função membro open():

std::ofstream os("foo.txt", std::ios::out | std::ios::trunc);

std::ifstream is;
is.open("foo.txt", std::ios::in | std::ios::binary);

Deve-se notar que você deve definir ios::in ou ios::out se quiser definir outros sinalizadores, pois eles não são definidos implicitamente pelos membros do iostream, embora tenham um valor padrão correto.

Se você não especificar um modo de abertura, os seguintes modos padrão serão usados:

  • ifstream - in
  • offstream - out
  • fstream - in e out

Os modos de abertura de arquivo que você pode especificar por design são:

Modo Significado Para Descrição
aplicativo anexar Saída Acrescenta dados no final do arquivo.
binário binário Entrada/Saída A entrada e a saída são feitas em binário.
em entrada Entrada Abre o arquivo para leitura.
fora saída Saída Abre o arquivo para escrita.
truncamento truncar Entrada/Saída Remove o conteúdo do arquivo ao abrir.
comeu no final Entrada Vai para o final do arquivo ao abrir.
Nota: Definir o modo binário permite que os dados sejam lidos/escritos exatamente como estão; não configurá-lo habilita a tradução do caractere de nova linha '\n' para/de uma sequência de fim de linha específica da plataforma.

Por exemplo, no Windows, a sequência de fim de linha é CRLF ("\r\n"). Escreva: "\n" => "\r\n" Leia: "\r\n" => "\n"

Lendo um arquivo ASCII em um std::string #

std::ifstream f("file.txt");

if (f)
{
  std::stringstream buffer;
  buffer << f.rdbuf();
  f.close();

  // The content of "file.txt" is available in the string `buffer.str()`
}

O método rdbuf() retorna um ponteiro para um streambuf que pode ser enviado para o buffer através da função de membro stringstream::operator<< .

Outra possibilidade (popularizada em Effective STL por Scott Meyers) é:

std::ifstream f("file.txt");

if (f)
{
  std::string str((std::istreambuf_iterator<char>(f)),
                  std::istreambuf_iterator<char>());

  // Operations on `str`...
}

Isso é bom porque requer pouco código (e permite ler um arquivo diretamente em qualquer contêiner STL, não apenas strings), mas pode ser lento para arquivos grandes.

NOTA: os parênteses extras em torno do primeiro argumento para o construtor de strings são essenciais para evitar o problema de análise mais irritante.

Por último mas não menos importante:

std::ifstream f("file.txt");

if (f)
{
  f.seekg(0, std::ios::end);
  const auto size = f.tellg();

  std::string str(size, ' ');
  f.seekg(0);
  f.read(&str[0], size); 
  f.close();

  // Operations on `str`...
}

que é provavelmente a opção mais rápida (entre as três propostas).

Gravando arquivos com configurações de localidade não padrão #

Se você precisar escrever um arquivo usando configurações de localidade diferentes para o padrão, você pode usar std::locale e std: :basic_ios::imbue() para fazer isso para um fluxo de arquivo específico:

Orientação para uso:

  • Você deve sempre aplicar um local a um stream antes de abrir o arquivo.
  • Uma vez que o fluxo tenha sido imbuído, você não deve alterar a localidade.

Motivos para restrições: Imbuir um fluxo de arquivo com uma localidade tem um comportamento indefinido se a localidade atual não for independente de estado ou não apontar para o início do arquivo.

Os fluxos UTF-8 (e outros) não são independentes de estado. Além disso, um fluxo de arquivo com uma localidade UTF-8 pode tentar ler o marcador de BOM do arquivo quando ele for aberto; portanto, apenas abrir o arquivo pode ler caracteres do arquivo e não estará no início.

#include <iostream>
#include <fstream>
#include <locale>

int main()
{
  std::cout << "User-preferred locale setting is "
            << std::locale("").name().c_str() << std::endl;

  // Write a floating-point value using the user's preferred locale.
  std::ofstream ofs1;
  ofs1.imbue(std::locale(""));
  ofs1.open("file1.txt");
  ofs1 << 78123.456 << std::endl;

  // Use a specific locale (names are system-dependent)
  std::ofstream ofs2;
  ofs2.imbue(std::locale("en_US.UTF-8"));
  ofs2.open("file2.txt");
  ofs2 << 78123.456 << std::endl;

  // Switch to the classic "C" locale
  std::ofstream ofs3;
  ofs3.imbue(std::locale::classic());
  ofs3.open("file3.txt");
  ofs3 << 78123.456 << std::endl;
}

Alternar explicitamente para a localidade clássica “C” é útil se o seu programa usar uma localidade padrão diferente e você quiser garantir um padrão fixo para leitura e gravação de arquivos. Com uma localidade preferencial “C”, o exemplo grava

78,123.456
78,123.456
78123.456

Se, por exemplo, a localidade preferencial for o alemão e, portanto, usar um formato de número diferente, o exemplo grava

78 123,456
78,123.456
78123.456

(observe a vírgula decimal na primeira linha).

Liberando um fluxo #

Os fluxos de arquivos são armazenados em buffer por padrão, assim como muitos outros tipos de fluxos. Isso significa que as gravações no fluxo podem não fazer com que o arquivo subjacente seja alterado imediatamente. Para forçar todas as gravações em buffer a ocorrerem imediatamente, você pode descarregar o fluxo. Você pode fazer isso diretamente invocando o método flush() ou através do manipulador de fluxo std::flush:

std::ofstream os("foo.txt");
os << "Hello World!" << std::flush;

char data[3] = "Foo";
os.write(data, 3);
os.flush();

Existe um manipulador de fluxo std::endl que combina a escrita de uma nova linha com a liberação do fluxo:

// Both following lines do the same thing
os << "Hello World!\n" << std::flush;
os << "Hello world!" << std::endl;

O armazenamento em buffer pode melhorar o desempenho da gravação em um fluxo. Portanto, os aplicativos que escrevem muito devem evitar a descarga desnecessariamente. Ao contrário, se a E/S for feita com pouca frequência, os aplicativos devem considerar a liberação frequente para evitar que os dados fiquem presos no objeto de fluxo.

Lendo um arquivo em um container #

No exemplo abaixo usamos std::string e operator>> para ler os itens do arquivo.

    std::ifstream file("file3.txt");

    std::vector<std::string>  v;

    std::string s;
    while(file >> s) // keep reading until we run out
    {
        v.push_back(s);
    }

No exemplo acima, estamos simplesmente iterando pelo arquivo lendo um “item” de cada vez usando operator>>. Este mesmo efeito pode ser obtido usando o std::istream_iterator que é um iterador de entrada que lê um “item” por vez do fluxo. Além disso, a maioria dos contêineres pode ser construída usando dois iteradores para que possamos simplificar o código acima para:

    std::ifstream file("file3.txt");

    std::vector<std::string>  v(std::istream_iterator<std::string>{file},
                                std::istream_iterator<std::string>{});

Podemos estender isso para ler qualquer tipo de objeto que quisermos simplesmente especificando o objeto que queremos ler como parâmetro do template para o std::istream_iterator. Assim, podemos simplesmente estender o acima para ler linhas (em vez de palavras) como esta:

// Unfortunately there is  no built in type that reads line using >>
// So here we build a simple helper class to do it. That will convert
// back to a string when used in string context.
struct Line
{
    // Store data here
    std::string data;
    // Convert object to string
    operator std::string const&() const {return data;}
    // Read a line from a stream.
    friend std::istream& operator>>(std::istream& stream, Line& line)
    {
        return std::getline(stream, line.data);
    }
};


    std::ifstream file("file3.txt");

    // Read the lines of a file into a container.
    std::vector<std::string>  v(std::istream_iterator<Line>{file},
                                std::istream_iterator<Line>{});

Verificando o final do arquivo dentro de uma condição de loop, má prática? #

eof retorna true somente após ler o final do arquivo. NÃO indica que a próxima leitura será o fim do fluxo.

while (!f.eof())
{
  // Everything is OK

  f >> buffer;

  // What if *only* now the eof / fail bit is set?

  /* Use `buffer` */
}

Você poderia escrever corretamente:

while (!f.eof()) 
{  
  f >> buffer >> std::ws;

  if (f.fail())
    break;

  /* Use `buffer` */
}

mas

while (f >> buffer)
{
  /* Use `buffer` */
}

é mais simples e menos propenso a erros.

Outras referências:

  • std::ws: descarta os espaços em branco iniciais de um fluxo de entrada
  • std::basic_ios::fail: retorna true se ocorreu um erro no stream associado

Fechando um arquivo #

Fechar explicitamente um arquivo raramente é necessário em C++, pois um fluxo de arquivo fechará automaticamente seu arquivo associado em seu destruidor. No entanto, você deve tentar limitar o tempo de vida de um objeto de fluxo de arquivo, para que ele não mantenha o identificador de arquivo aberto por mais tempo do que o necessário. Por exemplo, isso pode ser feito colocando todas as operações de arquivo em um escopo próprio ({}):

std::string const prepared_data = prepare_data();
{
    // Open a file for writing.
    std::ofstream output("foo.txt");

    // Write data.
    output << prepared_data;
}  // The ofstream will go out of scope here.
   // Its destructor will take care of closing the file properly.

Chamar close() explicitamente só é necessário se você quiser reutilizar o mesmo objeto fstream posteriormente, mas não quiser manter o arquivo aberto entre:

// Open the file "foo.txt" for the first time.
std::ofstream output("foo.txt");

// Get some data to write from somewhere.
std::string const prepared_data = prepare_data();

// Write data to the file "foo.txt".
output << prepared_data;

// Close the file "foo.txt".
output.close();

// Preparing data might take a long time. Therefore, we don't open the output file stream
// before we actually can write some data to it.
std::string const more_prepared_data = prepare_complex_data();

// Open the file "foo.txt" for the second time once we are ready for writing.
output.open("foo.txt");

// Write the data to the file "foo.txt".
output << more_prepared_data;

// Close the file "foo.txt" once again.
output.close();

Lendo um struct de um arquivo de texto formatado. #

struct info_type
{
    std::string name;
    int age;
    float height;
    
    // we define an overload of operator>> as a friend function which
    // gives in privileged access to private data members 
    friend std::istream& operator>>(std::istream& is, info_type& info)
    {
        // skip whitespace
        is >> std::ws;
        std::getline(is, info.name);
        is >> info.age;
        is >> info.height;
        return is;
    }
};

void func4()
{
    auto file = std::ifstream("file4.txt");

    std::vector<info_type> v;

    for(info_type info; file >> info;) // keep reading until we run out
    {
        // we only get here if the read succeeded
        v.push_back(info);
    }

    for(auto const& info: v)
    {
        std::cout << "  name: " << info.name << '\n';
        std::cout << "   age: " << info.age << " years" << '\n';
        std::cout << "height: " << info.height << "lbs" << '\n';
        std::cout << '\n';
    }
}

Copiando um arquivo #

std::ifstream  src("source_filename", std::ios::binary);
std::ofstream  dst("dest_filename",   std::ios::binary);
dst << src.rdbuf();

Com o C++17, a maneira padrão de copiar um arquivo é incluir o cabeçalho <filesystem> e usar copy_file:

std::fileystem::copy_file("source_filename", "dest_filename");

A biblioteca do sistema de arquivos foi originalmente desenvolvida como boost.filesystem e finalmente mesclada ao ISO C++ a partir do C++17.
