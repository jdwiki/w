Em 1998, houve a primeira publicação do padrão tornando C++ uma linguagem padronizada internamente. A partir desse momento, C++ evoluiu resultando em diferentes dialetos de C++. Nesta página, você pode encontrar uma visão geral de todos os diferentes padrões e suas alterações em comparação com a versão anterior. Os detalhes sobre como usar esses recursos estão descritos em páginas mais especializadas.

Quando o C++ é mencionado, muitas vezes é feita uma referência ao “Padrão”. Mas o que é exatamente esse padrão?

C++ tem uma longa história. Iniciado como um pequeno projeto de Bjarne Stroustrup dentro da Bell Labs, no início dos anos 90 tornou-se bastante popular. Várias empresas estavam criando compiladores C++ para que os usuários pudessem executar seus compiladores C++ em uma ampla variedade de computadores. Mas, para facilitar isso, todos esses compiladores concorrentes devem compartilhar uma única definição da linguagem.

Nesse ponto, a linguagem C havia sido padronizada com sucesso. Isso significa que uma descrição formal da linguagem foi escrita. Este foi submetido ao American National Standards Institute (ANSI), que abriu o documento para revisão e posteriormente o publicou em 1989. Um ano depois, a International Organization for Standards (por ter siglas diferentes em diferentes idiomas, eles escolheram uma forma , ISO, derivado do grego isos, que significa igual.) adotou o padrão americano como um padrão internacional.

Para C++, ficou claro desde o início que havia um interesse internacional. Um grupo de trabalho dentro da ISO foi iniciado (conhecido como WG21, dentro do Subcomitê 22). Esse grupo de trabalho elaborou um primeiro padrão por volta de 1995. Mas, como nós programadores sabemos, não há nada mais perigoso para uma entrega planejada do que recursos de última hora, e isso também aconteceu com C++. Em 1995, uma nova biblioteca legal chamada STL surgiu, e as pessoas que trabalhavam no WG21 decidiram adicionar uma versão simplificada ao padrão de rascunho C++. Naturalmente, isso fez com que os prazos fossem perdidos e apenas 3 anos depois o documento se tornasse definitivo. A ISO é uma organização muito formal, então o padrão C++ foi batizado com o nome pouco comercializável de ISO/IEC 14882. Como os padrões podem ser atualizados, essa versão exata ficou conhecida como 14882:1998.

E, de fato, houve uma demanda para atualizar o Padrão. O Padrão é um documento muito denso, que visa descrever exatamente como os compiladores C++ devem funcionar. Mesmo uma pequena ambiguidade pode valer a pena ser corrigida, então em 2003 uma atualização foi lançada como 14882:2003. No entanto, isso não adicionou nenhum recurso ao C++; os novos recursos foram agendados para a segunda atualização.

Informalmente, esta segunda atualização era conhecida como C++0x, porque não se sabia se isso levaria até 2008 ou 2009. Bem - essa versão também teve um pequeno atraso, e é por isso que se tornou 14882:2011.

Felizmente, o GT21 decidiu não deixar que isso acontecesse novamente. C++11 foi bem recebido e levou a um interesse renovado em C++. Então, para manter o ritmo, a terceira atualização passou de planejamento para publicação em 3 anos, para se tornar 14882:2014.

O trabalho também não parou por aí. O padrão C++17 foi proposto e o trabalho para C++20 foi iniciado.

Rascunhos de trabalho atuais #

Todos os padrões ISO publicados estão disponíveis para venda na loja ISO ( http://www.iso.org ). Os rascunhos de trabalho dos padrões C++ estão disponíveis publicamente gratuitamente.

As diferentes versões do padrão:

  • Próximos (às vezes chamados de C++20 ou C++2a): Rascunho de trabalho atual (versão HTML)
  • Proposta (às vezes chamada de C++17 ou C++1z): [rascunho de trabalho N4659 de março de 2017](http://www.open-std.org/jtc1/sc22/wg21/docs/papers/2017/n4659 .pdf).
  • C++14 (às vezes chamado de C++1y): rascunho de trabalho de novembro de 2014 N4296
  • C++11 (às vezes chamado de C++0x): rascunho de trabalho de fevereiro de 2011 N3242
  • C++03
  • C++98

C++17 #

O padrão C++17 é completo e foi proposto para padronização. Em compiladores com suporte experimental para esses recursos, geralmente é chamado de C++1z.

Extensões de idioma ================================================== ================

  • Fold Expressions
  • declarando argumentos de modelo que não são de tipo com auto
  • Elisão de cópia garantida
  • Dedução de parâmetro de modelo para construtores
  • Associações estruturadas
  • Espaços de nomes aninhados compactos
  • Novos atributos: [[fallthrough]], [[nodiscard]], [[maybe_unused]]
  • Mensagem padrão para static_assert
  • Inicializadores em if e switch
  • Variáveis ​​em linha
  • if constexpr
  • Garantias de avaliação de ordem de expressão
  • Alocação dinâmica de memória para dados super-alinhados

Extensões de biblioteca ================================================== =============

  • std::opcional
  • std::variant
  • std::string_view
  • merge() e extract() para contêineres associativos
  • Uma biblioteca de sistema de arquivos com [o cabeçalho <filesystem>](http:/ /en.cppreference.com/w/cpp/filesystem).
  • Versões paralelas da maioria dos algoritmos padrão (no [cabeçalho <algorithm>](http://en.cppreference.com/ c/cpp/algoritmo)).
  • Adição de funções especiais matemáticas no [cabeçalho <cmath>](http://en.cppreference.com/w/cpp/ numeric/special_math).
  • Movendo nós entre map<>, unordered_map<>, set<> e unordered_set<>

Abaixo, você pode encontrar uma visão geral das alterações, pois elas foram agrupadas em as perguntas frequentes do isocpp com links para documentação mais detalhada.

  • Literais binários
  • Dedução generalizada do tipo de retorno
  • tipo decl(automático)
  • Capturas lambda generalizadas
  • Lambdas genéricos
  • Modelos variáveis
  • constexpr estendido
  • O atributo [[obsoleto]]
  • Separadores de dígitos

Extensões de biblioteca ================================================== =============

  • Bloqueio compartilhado
  • Literais definidos pelo usuário para tipos std::

  • Abordando tuplas por tipo (por exemplo, get<string>(t))
  • Functores de operador transparente (por exemplo, maior<>(x))
  • std::quoted

Descontinuado/Removido ================================================== =============

  • std::gets foi preterido no C++11 e removido do C++14
  • std::random_shuffle está obsoleto

C++98 #

C++98 é a primeira versão padronizada de C++. Como foi desenvolvido como uma extensão para C, muitos dos recursos que diferenciam C++ de C são adicionados.

Extensões de idioma (em relação a C89/C90) #

  • Classes, classes derivadas, funções-membro virtuais, funções-membro const
  • Sobrecarga de função, sobrecarga de operador
  • Comentários de linha única (Foi introduzido na linguagem C com o padrão C99)
  • Referências
  • novo e excluir
  • tipo booleano (foi introduzido na linguagem C com o padrão C99)
  • modelos
  • espaços de nomes
  • exceções
  • elencos específicos

Extensões de biblioteca #

  • A Biblioteca de Modelos Padrão

##C++03 O padrão C++03 aborda principalmente relatórios de defeitos do padrão C++98. Além desses defeitos, ele adiciona apenas um novo recurso.

Extensões de idioma #

  • inicialização do valor

C++20 #

C++20 é o próximo padrão de C++, atualmente em desenvolvimento, baseado no padrão C++17. Seu progresso pode ser acompanhado no site oficial ISO cpp.

Os recursos a seguir são simplesmente o que foi aceito para o próximo lançamento do padrão C++, direcionado para 2020.

Nenhuma extensão de idioma foi aceita por enquanto.
