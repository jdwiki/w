Sintaxe #

  • add_action( $tag, $function_to_add )
  • add_action( $tag, $function_to_add, $priority)
  • add_action( $tag, $function_to_add, $priority, $accepted_args)

Parâmetros #

Parâmetro Detalhes
$tag (string) (Obrigatório) O nome da ação à qual o $function_to_add está vinculado
$function_to_add (callable) (Obrigatório) A função que deve ser chamada quando a ação indicada por $tag for executada
$prioridade (int) (Opcional) Valor padrão: 10 Usado para especificar a ordem na qual as funções associadas a uma determinada ação são executadas. Os números mais baixos correspondem à execução anterior e as funções com a mesma prioridade são executadas na ordem em que foram adicionadas à ação.
$accepted_args (int) (Opcional) Valor padrão: 1 O número de argumentos que a função aceita.

A função add_action() cria um Action Hook, associando uma função PHP com um determinado action “tag” ou nome. Quando a ação é “disparada” por uma chamada para do_action() (ou do_action_ref_array()) com uma tag específica, todas as funções “vinculadas” a essa tag serão executadas.

Na maioria dos casos, esta função deve ser usada em um arquivo functions.php de um tema ou um arquivo de plugin - ou outro arquivo fonte carregado por qualquer um.

Esta função faz parte da API de plug-in

Gancho de Ação Básico #

A aplicação mais básica de add_action() é adicionar código personalizado a ser executado em um determinado local no código-fonte de uma instalação do WordPress - usando ações fornecidas pelo Core do WordPress ou criadas por terceiros. código do partido, como plugins e temas.

Para adicionar conteúdo à seção <head></head> do site - digamos, para adicionar um elemento meta <link> para indicar onde as informações de direitos autorais do site podem ser encontradas - add_action() pode ser usado para anexar uma função que imprima a marcação apropriada para a ação 'wp_head' (que “dispara” quando o WordPress cria a seção <head>):

function add_copyright_meta_link() {
  echo( '<link rel="copyright" href="' . get_home_url() . '/copyright">' );
}

add_action( 'wp_head', 'add_copyright_meta_link' );

Prioridade do Gancho de Ação #

Qualquer número de funções pode ser “vinculado” a qualquer ação. Em alguns casos é importante que uma função hooked execute antes ou depois de outras, que é onde o terceiro parâmetro para add_action(), $priority entra em jogo.

Se o argumento $priority for omitido, a função será anexada com a prioridade padrão de 10. Quando a ação é “disparada”, as funções “hooked” serão chamadas começando com aquelas adicionadas com o menor $priority, e progredindo para as funções com o maior $priority. Quaisquer funções hooked que compartilhem a mesma prioridade serão chamadas na ordem em que foram adicionadas (a ordem em que suas respectivas chamadas add_action() foram executadas).

Por exemplo, digamos que um plug-in de terceiros esteja usando uma função vinculada à ação 'template_redirect' para encaminhar os visitantes da página daily-deal para um link de afiliado de um site de comércio eletrônico externo, mas você’ gostaria que o redirecionamento ocorresse apenas para usuários logados. Você precisaria usar seu próprio gancho 'template_redirect' para enviar visitantes desconectados para a página de login. Depois de determinar que o plug-in de terceiros anexa sua função com o padrão $piority de 10, você pode vincular sua função com uma prioridade de 9 para garantir que sua verificação de login ocorra primeiro:

function redirect_deal_visitors_to_login() {
  if( is_page( 'daily-deal' ) && !user_is_logged_in() ) {
    wp_redirect( wp_login_url() );
    exit();
  }
}

add_action( 'template_redirect', 'redirect_deal_visitors_to_login', 9 );

Conectando métodos de classe e objeto a ações #

PHP Classes são ferramentas poderosas para melhorar a organização do código e minimizar colisões de nomenclatura. Em algum ponto ou outro, surge inevitavelmente a questão de como criar um gancho de ação para um método de classe.

O argumento $function_to_add é frequentemente mostrado como uma string contendo o nome da função, entretanto o tipo de dados do argumento é na verdade um “[callable](http://php.net/manual/en/language.types.callable .php)”, que para nossos propósitos pode ser resumido como “uma referência a uma função ou método”.

Existem vários formatos que podem ser chamados que podem ser usados ​​para referenciar métodos em classes e objetos. Em todos os casos, no entanto, o método referenciado deve ser publicamente visível. Um método é público quando é prefixado com a palavra-chave public, ou nenhuma palavra-chave de visibilidade (nesse caso, o método assume o padrão public).

Ganchos de ação de método de objeto #

Os métodos de objeto são executados em uma instância específica de uma classe.

class My_Class {
  // Constructor
  function My_Class() {
    // (Instantiation logic)
  }

  // Initialization function
  public function initialize() {
    // (Initialization logic)
  }
}

Depois de instanciar a classe acima da seguinte forma,

$my_class_instance = new My_Class();

o método initialize() normalmente seria invocado no objeto chamando $my_class_instance->initialize();. Conectar o método à ação 'init' do WordPress é feito passando um array contendo uma referência à instância e uma string contendo o nome do método do objeto:

add_action( 'init', [ $my_class_instance, 'initialize' ] );

Se add_action() for chamado dentro de um método de objeto, a pseudo-variável $this também pode ser usada:

class My_Class {
  // Constructor
  function My_Class() {
    // (Instantiation logic)
    add_action( 'init', [ $this, 'initialize' ] );
  }

  // Initialization function
  public function initialize() {
    // (Initialization logic)
  }
}

Ganchos de ação de método de classe #

Os métodos de classe são executados estaticamente em uma classe em vez de em qualquer instância específica. Dada a seguinte classe,

class My_Class {   
  // Initialization function
  public static function initialize() {
    // (Initialization logic)
  }
}

o método initialize() normalmente seria invocado usando o operador de resolução de escopo ::, ou seja, My_Class::initialize();. Conectar um método de classe estática a um WordPress pode ser feito de duas maneiras diferentes:

  • Usando um array composto por uma string contendo o nome da classe e uma string contendo o nome do método:

     add_action( 'init', [ 'My_Class', 'initialize' ] );

  • Passando uma string contendo uma referência completa ao método, incluindo o operador :::

     add_action( 'init', 'My_Class::initialize' );

  • Se add_action() for chamado dentro de um método de classe estático, a palavra-chave self ou a constante mágica __CLASS__ pode ser usada no lugar do nome da classe. Observe que isso geralmente é desaconselhável, pois os valores desses itens se tornam um pouco contra-intuitivos no caso de herança de classe.

     class My_Class {
   // Setup function
   public static function setup_actions() {
     add_action( 'init', 'self::initialize' );
   }

   // Initialization function
   public static function initialize() {
     // (Initialization logic)
   }
 }
