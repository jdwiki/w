Tema Wordpress e desenvolvimento de tema infantil

  • Desenvolvendo seu próprio tema

  • Desenvolvendo seu próprio tema

Wordpress é um CMS amplamente utilizado para criar sites de informação simples, mas também para criar sites mais sofisticados e até pequenas lojas virtuais.

Wordpress faz uso de temas. Esses temas são usados ​​para criar o layout e a funcionalidade de conteúdo de um site Wordpress. Os temas podem ser encontrados em toda a internet.

Cada tema tem sua própria funcionalidade e layout, mas às vezes é difícil encontrar o tema certo para um site. Felizmente, também podemos criar nosso próprio tema.

Desenvolvendo seu próprio tema #

Um tema wordpress consiste em dois tipos de arquivos. Os arquivos básicos que cada tema possui e os arquivos que definem o layout e a funcionalidade do tema. Este segundo grupo vou chamar de arquivos específicos do tema.

Os arquivos de temas básicos
Os arquivos de temas básicos são os arquivos usados ​​para configurar e registrar um tema. Na lista abaixo, descreverei brevemente cada arquivo e seu uso. Mais tarde, adicionarei os arquivos de exemplo mais básicos necessários para configurar seu próprio tema wordpress.

  • functions.php: O arquivo functions.php é usado para registrar todas as funções, barras laterais, scripts e inclusões do tema. Neste arquivo você pode, por exemplo, incluir arquivos CSS, arquivos JS, etc.
  • Header and footer: Os arquivos de cabeçalho e rodapé (header.php e footer.php) são os arquivos que são usados ​​para chamar o cabeçalho e o rodapé. O arquivo de cabeçalho e rodapé, por exemplo, contém o link para o sistema de back-end wordpress.
  • index.php: O arquivo index.php é o arquivo que cria o modelo de página padrão. Neste arquivo você pode ver, editar e remover partes deste layout de modelo padrão.
  • single.php: O arquivo single.php é o arquivo que cria a página de template de posts únicos. Assim como o modelo padrão para as páginas, mas agora para as páginas de postagem única.
  • format.php O arquivo format.php é o arquivo que constrói o modelo de texto de conteúdo de uma página. Portanto, se você tiver uma página inicial e a editar a partir do back-end, adicionando um texto. Este arquivo cria a marcação padrão deste texto.
  • 404.php O arquivo 404.php cria o modelo 404. Este arquivo consiste no layout básico desta página.
  • archive.php O arquivo archive.php cria o layout da página de arquivo.
  • style.css O arquivo de folha de estilo básico.

Portanto, nesta lista você pode ver todos os arquivos necessários para a configuração do seu próprio tema Wordpress. Agora vamos dar uma olhada em alguns arquivos que você pode criar se quiser, mas são arquivos não obrigatórios para um tema wordpress. Esses arquivos são principalmente arquivos de modelo e outras extensões funcionais.

Modelos de página personalizados
page-<your own name>.php: Em um tema Wordpress você pode criar vários templates de página. criando novos arquivos de modelo de página. Um arquivo de modelo de página padrão consiste nos seguintes atributos de nome. page nome do template e .php Se por exemplo você gostaria de criar um novo template de página para a sua página de blog você poderia chamá-lo de page-blog.php Wordpress automaticamente lê o arquivo e adiciona o arquivo para o menu Escolher modelo. Certifique-se de incluir pelo menos as funções get_header() e get_footer(). Certifique-se também de nomear seu modelo em um comentário na parte superior do arquivo, adicionando o exemplo a seguir.

<?php
    /*
     * Template Name: Homepage Template
     */
    get_header();
?>

Modelos personalizados de página de postagem única
single-<your own name>.php: Em um tema Wordpress, assim como o modelo de página descrito acima, você também pode criar seus próprios modelos de página de posts únicos. Assim como o modelo de página, o arquivo consiste em três partes single para declarar que é uma única página de postagem <seu nome do modelo> e a extensão do arquivo .php. Assim como os requisitos mínimos do template de página para garantir que o Wordpress leia o novo template estão adicionando as funções get_header() e get_footer(). E, claro, também adicionando o nome do seu modelo, como no exemplo abaixo

<?php

/*
* Template Name: Post Portfolio
* Template Post Type: post, page
*/

?>

Indicamos também o Template post type: que representa o tipo de template que é, neste caso post e página.

Modelos de texto de postagem personalizados
format -<your own name>.php: Em um tema Wordpress, você também pode criar modelos de saída de postagem. Esses modelos de formato são o layout e o conteúdo de uma postagem. Por exemplo, se em alguns casos você quiser que a postagem mostre apenas o conteúdo ou o título da postagem, você pode usar esses modelos para criar esse tipo de ajuste. Como esses tipos de templates estão apenas formatando o conteúdo de back-ends do post que foi criado por um usuário, não precisamos incluir get_header() e get_footer(), pois eles já estão definidos nos templates das páginas. Certifique-se de que seu modelo seja capaz de reconhecer uma postagem usando o exemplo básico a seguir.

<div>
    <article id="post-<?php the_ID(); ?>" <?php post_class(); ?>>
    </article>
</div>

Então agora que sabemos algo sobre os arquivos básicos e alguns dos muitos arquivos específicos de templates, é hora de começar a falar sobre barras laterais e widgets. No futuro, isso será adicionado junto com um tutorial passo a passo sobre como criar um tema Wordpress muito próprio.
