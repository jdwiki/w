Sites WordPress são frequentemente invadidos. Este tópico é para técnicas e práticas que aumentam a segurança de sua instalação do WordPress além do que é alcançado em uma instalação básica.

Além deste tópico, outro bom lugar para ler sobre como proteger uma instalação do WordPress é a página Hardening WordPress Codex.

Move wp-config.php #

As informações mais confidenciais de uma instalação do WordPress são armazenadas no arquivo wp-config.php. Se um hacker tiver acesso a esse arquivo, ele terá controle total do seu site.

Por padrão, o wp-config.php é armazenado na pasta de instalação do WordPress. Para dificultar o roubo desse arquivo, você pode movê-lo para fora da pasta acessível pela Web. Se você movê-lo apenas uma pasta acima, o WordPress o encontrará automaticamente. Se você mover o wp-config.php para um local diferente, crie um arquivo vazio chamado wp-config.php na pasta de instalação do WordPress. Em seguida, adicione o seguinte:

define('ABSPATH', dirname(__FILE__) . '/');
// '../../wp-config.php' defines location two folders above installation folder.
// Substitute with actual location of wp-config.php file as necessary.
require_once(ABSPATH . '../../wp-config.php');

Você pode precisar tornar o php executável na pasta em que você colocou o wp-config.php. Você deve tornar o php executável no menor número de pastas possível. Um bom sistema coloca a instalação do WordPress em /path/to/wordpress/install/ e a configuração em /path/to/wordpress/config. Você deve certificar-se de que a pasta de configuração não é acessível pela web e não colocar nenhuma outra informação confidencial em /path/to/ ou superior na hierarquia de pastas. Nesse caso, você escreveria uma linha semelhante à seguinte em seu php.ini:

open_basedir = "/path/to/wordpress/install/;/path/to/wordpress/config"

Esta técnica é controversa e algumas pessoas não acham que aumenta a segurança. Uma extensa discussão sobre o tópico pode ser lida em esta pergunta do WordPress StackExchange.

Desativar o Editor de Arquivos #

O editor de arquivos que acompanha o WordPress é um risco de segurança. Se um invasor obtiver acesso de administrador ao seu site WordPress, ele poderá inserir facilmente código malicioso em arquivos de temas e plugins. Também é um risco com clientes que não sabem o que estão fazendo. Uma vez que dois pontos extraviados no editor de arquivos podem quebrar um site e torná-lo inacessível a partir do navegador.

No arquivo wp-config.php do WordPress, desative o editor de arquivos adicionando a seguinte linha de código.

define( 'DISALLOW_FILE_EDIT', true );

Essa linha terá o efeito desejado quando adicionada ao arquivo functions.php do seu tema também, mas é melhor adicionar a wp-config.php.

Se você estiver usando WordPress CLI para instalar o WordPress, você pode usar o seguinte comando para criar um arquivo wp-config.php com edição de arquivo desabilitada.

/* declare variables beforehand or substitute strings in */
wp core config --dbname="$MYSQL_DBNAME" --dbuser="$MYSQL_USERNAME" --dbpass="$MYSQL_PASS" --dbprefix="$WP_DBPREFIX"_ --locale=en_AU --extra-php <<PHP
define( 'DISALLOW_FILE_EDIT', true );
PHP

Este método é útil se você instalar o WordPress com um script.

Defina um prefixo personalizado para tabelas do WordPress #

Quando você instala o WordPress em seu servidor, o script de instalação colocará um prefixo na frente de todos os nomes de tabela MySQL do WordPress. Esse prefixo é definido como ‘wp_’ por padrão. A tabela de posts do WordPress será chamada de wp_posts por exemplo. Ao alterar o prefixo da tabela você pode criar alguma segurança por obscuridade. Dessa forma, quando um hacker tentar ataques de injeção de SQL, ele terá que adivinhar o prefixo da sua tabela em vez de apenas usar ‘wp_’. Você pode definir esse prefixo para ser o que quiser.

Definir prefixo na nova instalação do WordPress

Se estiver usando a famosa instalação de 5 minutos Altere o prefixo no campo durante a instalação.

Se instalar via WordPress CLI, use o seguinte comando:

// set other variables above, or substitute your strings in.
WP_DBPREFIX=foo
wp core config --dbname="$MYSQL_DBNAME" --dbuser="$MYSQL_USERNAME" --dbpass="$MYSQL_PASS" --dbprefix="$WP_DBPREFIX"_ --locale=en_AU

Alterar prefixo na instalação existente

Alterar o prefixo é um pouco mais difícil. Primeiramente use um programa FTP como o FileZilla para editar o arquivo wp-config.php. Altere a entrada $table_prefix = 'wp_'; para $table_prefix = 'foo_'; substituindo ‘foo’ pelo prefixo desejado.

Em seguida, precisaremos editar o banco de dados. Se você tiver acesso ao phpMyAdmin, faça o login e faça o seguinte:

  • Selecione o banco de dados do WordPress
  • Selecione todas as tabelas e, no menu suspenso, selecione substituir prefixo da tabela.
  • Em “De” digite ‘wp_’. Em “Para” digite seu prefixo, ‘foo_’ neste exemplo e pressione “Enviar”.
  • As tabelas agora devem ficar assim:

Se você não puder usar o phpMyAdmin, use o seguinte comando MySQL:

RENAME table `wp_comments` TO `foo_comments`

Você precisará executar esse comando para cada tabela, substituindo ‘comentários’ pelos outros nomes de tabela.

Em seguida, precisamos alterar algumas entradas em algumas tabelas. Execute esta consulta na tabela ‘foo_options’

SELECT * FROM  foo_options WHERE option_name LIKE '%user_roles%'

Uma entrada com option_name de ‘wp_user_roles’ deve aparecer. Nessa entrada, altere a entrada ‘option_name’ de wp_user_roles para foo_user_roles.

Em seguida, abra a tabela ‘foo_usermeta’ e encontre todas as entradas com ‘wp_’ na frente. e mude para ‘foo_’. O número de entradas que você deve alterar dependerá de quantos usuários você tem.

Isso deve ser tudo o que você precisa para alterar o prefixo em uma instalação existente
