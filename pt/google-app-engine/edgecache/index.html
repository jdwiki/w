EdgeCache

  • Ativando o EdgeCache

Nesta página

  • Ativando o EdgeCache

Detalhes

  • Quando o Edge Cache está ativado e funcionando, o App Engine envia um cabeçalho age cujo valor é o tempo (em segundos) desde que a resposta foi armazenada em cache. Por exemplo, se a resposta foi armazenada em cache por dois minutos até agora, a resposta incluirá um cabeçalho de age: 120. Se nenhum cabeçalho age estiver sendo enviado, é um bom sinal de que você ainda não conseguiu que o Edge Cache funcione. Além disso, quando a resposta não inclui um cabeçalho age, isso significa que a solicitação perdeu o Edge Cache.
  • O Edge Cache não funciona no ambiente de desenvolvimento local.
  • Existem diferentes caches para diferentes regiões. Existem caches separados para regiões separadas do mundo. Por exemplo, uma solicitação da Europa pode atingir o cache, enquanto uma solicitação exatamente ao mesmo tempo da Austrália pode não atingir o Edge Cache.
  • Existem vários caches mesmo na mesma região. Por exemplo, duas solicitações sequenciais do mesmo cliente podem receber respostas com valores diferentes para o cabeçalho age.
  • O Google despeja o conteúdo do Edge Cache antes que a idade máxima seja atingida. Isso é especialmente verdadeiro se o recurso não tiver sido solicitado por mais de 5 minutos.
  • O Google envia automaticamente um cabeçalho Vary com o valor Accept-Encoding com algumas respostas. Consulte esta documentação.
  • O App Engine não permitirá que você defina o cabeçalho Cache-Control como público se você também tiver um cabeçalho Set-Cookie presente. O App Engine até alterará o valor do cabeçalho Cache-Control de public para private se houver um cabeçalho Set-Cookie. Consulte aqui.

Como o EdgeCache funciona

EdgeCache é um cache de proxy reverso que armazena dados por um determinado período de tempo e os retorna rapidamente ao ver a mesma solicitação, desde que o cache ainda seja válido.

Aqui está um diagrama de este vídeo da equipe do App Engine sobre como o EdgeCache funciona:

Os data centers do “Google Front End” estão localizados em todo o mundo e podem armazenar dados em cache para serem retornados rapidamente mediante solicitação, sem precisar executar nenhum código do seu App Engine (que é executado no “App Engine Front End”).

Consulte esta resposta do StackOverflow para obter mais informações sobre proxies reversos em geral.

Ativando o EdgeCache #

  • Para habilitar o EdgeCache, defina os seguintes cabeçalhos de resposta HTTP (Não se desvie desse formato exato):
    • Cache-Control header to public, max-age=X where:
      • X = the number of seconds that you want the response to be cached for
      • X > 60 seconds
      • X < 365*24*60*60
    • Set the Pragma header to Public.
