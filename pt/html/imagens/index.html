Sintaxe #

  • <img src="" alt="">

Parâmetros #

Parâmetros Detalhes
src Especifica o URL da imagem
srcset Imagens para usar em diferentes situações (por exemplo, telas de alta resolução, monitores pequenos, etc.)
tamanhos Tamanhos de imagem entre pontos de interrupção
crossorigin Como o elemento trata solicitações de origem cruzada
mapa de uso Nome do mapa de imagem a ser usado
ismap Se a imagem é um mapa de imagem do lado do servidor
alt Texto alternativo que deve ser exibido se por algum motivo a imagem não puder ser exibida
largura Especifica a largura da imagem (opcional)
altura Especifica a altura da imagem (opcional)

Criando uma imagem #

Para adicionar uma imagem a uma página, use a tag de imagem.

Tags de imagem (img) não possuem tags de fechamento. Os dois principais atributos que você dá à tag img são src, a fonte da imagem e alt, que é um texto alternativo que descreve a imagem.

<img src="images/hello.png" alt="Hello World">

Você também pode obter imagens de um URL da web:

<img src="https://i.stack.imgur.com/ALgZi.jpg?s=48&g=1" alt="StackOverflow user Caleb Kleveter">

Nota: As imagens não são tecnicamente inseridas em uma página HTML, as imagens são vinculadas a páginas HTML. A tag <img> cria um espaço de retenção para a imagem referenciada.

Também é possível incorporar imagens diretamente dentro da página usando base64:

<img src="data:image/png;base64,iVBOR..." alt="Hello World">

Dica: Para vincular uma imagem a outro documento, simplesmente aninhe a tag <img> dentro das tags <a>.

Escolhendo o texto alternativo #

O texto alternativo é usado por leitores de tela para usuários com deficiência visual e por mecanismos de pesquisa. Portanto, é importante escrever um bom texto alternativo para suas imagens.

O texto deve parecer correto mesmo se você substituir a imagem por seu atributo alt. Por exemplo:

<!-- Incorrect -->
<img src="anonymous.png" alt="Anonymous user avatar"/> An anonymous user wrote:
<blockquote>Lorem ipsum dolor sed.</blockquote>
<a href="https://google.com/"><img src="edit.png" alt="Edit icon"/></a> /
<a href="https://google.com/"><img src="delete.png" alt="Delete icon"/></a>

Sem as imagens, ficaria assim:

Avatar de usuário anônimo Um usuário anônimo escreveu:

É um saco.
ícone Editar / Ícone de exclusão

Para corrigir isso:

  • Remova o texto alternativo do avatar. Esta imagem adiciona informações para usuários com visão (um ícone facilmente identificável para mostrar que o usuário é anônimo), mas essas informações já estão disponíveis no texto.1
  • Remova o “ícone” do texto alternativo dos ícones. Saber que isso seria um ícone se estivesse lá não ajuda a transmitir seu real propósito.
<!-- Correct -->
<img src="anonymous.png" alt=""/> An anonymous user wrote:
<blockquote>Lorem ipsum dolor sed.</blockquote>
<a href="https://google.com/"><img src="edit.png" alt="Edit"/></a> /
<a href="https://google.com/"><img src="delete.png" alt="Delete"/></a>

Um usuário anônimo escreveu:

É um saco.
Editar / Excluir

Notas de rodapé

1 Há uma diferença semântica entre incluir um atributo alt vazio e excluí-lo completamente. Um atributo alt vazio indica que a imagem não é uma parte chave do conteúdo (como é verdade neste caso - é apenas uma imagem aditiva que não é necessária para entender o resto) e, portanto, pode ser omitida da renderização. No entanto, a falta de um atributo alt indica que a imagem é uma parte fundamental do conteúdo e que simplesmente não há equivalente textual disponível para renderização.

Imagem responsiva usando o atributo srcset #

Usando srcset com tamanhos #

<img sizes="(min-width: 1200px) 580px,
      (min-width: 640px) 48vw,
      98vw"
    srcset="img/hello-300.jpg 300w,
      img/hello-600.jpg 600w,
      img/hello-900.jpg 900w,
      img/hello-1200.jpg 1200w"
    src="img/hello-900.jpg" alt="hello">

sizes são como consultas de mídia, descrevendo quanto espaço a imagem ocupa na janela de visualização.

  • se a janela de visualização for maior que 1200px, a imagem terá exatamente 580px (por exemplo, nosso conteúdo está centralizado em um contêiner com largura máxima de 1200px. A imagem ocupa metade dela menos as margens).
  • se a viewport estiver entre 640px e 1200px, a imagem ocupa 48% da viewport (por exemplo, a imagem é dimensionada com nossa página e ocupa metade da largura da viewport menos as margens).
  • se a viewport for de qualquer outro tamanho , no nosso caso menor que 640px, a imagem ocupa 98% da viewport (por exemplo, a imagem é dimensionada com nossa página e ocupa a largura total da viewport menos as margens). A condição de mídia deve ser omitida para o último item.

srcset está apenas dizendo ao navegador quais imagens temos disponíveis e quais são seus tamanhos.

  • img/hello-300.jpg tem 300px de largura,
  • img/hello-600.jpg tem 600px de largura,
  • img/hello-900.jpg tem 900px de largura,
  • img/hello-1200.jpg tem 1200px de largura

src é sempre uma fonte de imagem obrigatória. No caso de usar com srcset, src servirá a imagem de fallback caso o navegador não suporte srcset.

Usando srcset sem tamanhos #

<img src="img/hello-300.jpg" alt="hello"
     srcset="img/hello-300.jpg 1x, 
       img/hello-600.jpg 2x,
       img/hello-1200.jpg 3x">

srcset fornece uma lista de imagens disponíveis, com o descritor de proporção dispositivo-pixel x.

  • se a proporção dispositivo-pixel for 1, use img/hello-300.jpg
  • se a proporção dispositivo-pixel for 2, use img/hello-600.jpg
  • se a proporção dispositivo-pixel for 3, use img/hello-1200.jpg

src é sempre uma fonte de imagem obrigatória. No caso de usar com srcset, src servirá a imagem de fallback caso o navegador não suporte srcset.

Imagem responsiva usando elemento de imagem #

Código

<picture>
  <source media="(min-width: 600px)" srcset="large_image.jpg">
  <source media="(min-width: 450px)" srcset="small_image.jpg">
  <img src="default_image.jpg" style="width:auto;">
</picture>

Uso

Para exibir imagens diferentes em larguras de tela diferentes, você deve incluir todas as imagens usando a tag de origem em uma tag de imagem, conforme mostrado no exemplo acima.

Resultado

  • Em telas com largura de tela >600px, mostra large_image.jpg
  • Em telas com largura de tela >450px, mostra small_image.jpg
  • Em telas com outra largura de tela, mostra default_image.jpg

Largura e altura da imagem #

Observação: os atributos largura e altura não são obsoletos em imagens e nunca foram. Seu uso foi feito muito mais estrito embora.

As dimensões de uma imagem podem ser especificadas usando os atributos width e height da tag de imagem:

<img src="images/200x200-img.png" width="200" height="200" alt="A 200x200 image">

Ao especificar a largura e a altura de uma imagem, sua estrutura dá ao navegador uma dica de como a página deve ser disposta, mesmo que você esteja apenas especificando o tamanho real da imagem. Se as dimensões da imagem não forem especificadas, o navegador precisará recalcular o layout da página depois que a imagem for carregada, o que pode fazer com que a página “salte” durante o carregamento.

Você pode dar à imagem uma largura e altura no número de pixels CSS ou em uma porcentagem das dimensões reais da imagem.

Estes exemplos são todos válidos:

<img src="images/50x50-img.png" width="50" height="50" alt="A 50x50 image">
<img src="images/50x50-img.png" width="200" height="200" alt="A 200x200 image">
<img src="images/200x200-img.png" width="50" height="50" alt="A 50x50 image">
<img src="images/200x200-img.png" width="50%" height="50%" alt="A 100x100 image">

A largura e a altura da imagem devem ser especificadas em pixels CSS; um valor percentual não é mais um valor válido. Além disso, se ambos os atributos forem especificados, eles devem caber em uma das três fórmulas que mantêm a proporção . Embora válido, você não deve usar os atributos largura e altura para esticar uma imagem para um tamanho maior.

Estes exemplos são válidos:

<img src="images/50x50-img.png" width="50" height="50" alt="A 50x50 image">
<img src="images/200x200-img.png" width="50" height="50" alt="A 50x50 image">
<img src="images/50x50-img.png" width="0" height="0" alt="A hidden tracking image">

Este exemplo não é recomendado:

<img src="images/50x50-img.png" width="200" height="200" alt="A 200x200 image">

Estes exemplos são inválidos:

<img src="images/200x200-img.png" width="50%" height="50%" alt="A 100x100 image">
<img src="images/200x200-img.png" width="1" height="200" alt="A 1x200 image">
