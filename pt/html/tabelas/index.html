O elemento HTML <table> permite que os autores da web exibam dados tabulares (como texto, imagens, links, outras tabelas, etc.) em uma tabela bidimensional com linhas e colunas de células.

Sintaxe #

  • <table></table>
  • <thead></thead>
  • <tbody></tbody>
  • <tfoot></tfoot>
  • <tr></tr>
  • <th></th>
  • <td></td>

Os vários elementos de tabela e seus atributos de conteúdo juntos definem o modelo de tabela. O elemento <table> é o elemento contêiner para modelos de tabela/dados tabulares. As tabelas têm linhas, colunas e células dadas por seus descendentes. As linhas e colunas formam uma grade; as células de uma tabela devem cobrir completamente essa grade sem sobreposição. A lista abaixo descreve os vários elementos no modelo de tabela:

  • <table> - O elemento container para modelos de tabela/dados tabulares. <table> representa dados com mais de uma dimensão na forma de uma tabela.
  • <caption> - Legenda ou título da tabela (como uma figcaption para uma figure)
  • <col> - Uma coluna (um elemento sem conteúdo)
  • <colgroup> - Um agrupamento de colunas
  • <thead> - Cabeçalho da tabela (apenas um)
  • <tbody> - Corpo/conteúdo da tabela (vários estão corretos)
  • <tfoot> - Rodapé da tabela (apenas um)
  • <tr> - Linha da tabela
  • <th> - Célula do cabeçalho da tabela
  • <td> - Célula de dados da tabela

Semanticamente, as tabelas são destinadas a armazenar dados tabulares. Você pode pensar nisso como uma maneira de exibir e descrever dados que fariam sentido em uma planilha (colunas e linhas).

O uso de tabelas para layout não é recomendado. Em vez disso, use regras CSS para layout e formatação, incluindo display: table.

Uma exceção notável normalmente exibida na indústria em relação ao uso do layout <table> é em relação ao e-mail HTML: alguns clientes de e-mail, incluindo o Outlook, foram revertidos para mecanismos de renderização mais antigos depois que a Microsoft perdeu seu caso de monopólio contra a UE. Para que a Microsoft faça o IE não fazer parte do sistema operacional, eles simplesmente reverteram o mecanismo de renderização do Outlook para uma versão anterior do Trident. Essa reversão simplesmente não suporta tecnologias modernas da web, portanto, usar layouts baseados em <tabela> para e-mail HTML é a única maneira de garantir a compatibilidade entre navegadores/plataformas/clientes.

Tabela Simples #

<table>
  <tr>
    <th>Heading 1/Column 1</th>
    <th>Heading 2/Column 2</th>
  </tr>
  <tr>
    <td>Row 1 Data Column 1</td>
    <td>Row 1 Data Column 2</td>
  </tr>
  <tr>
    <td>Row 2 Data Column 1</td>
    <td>Row 2 Data Column 2</td>
  </tr>
</table>

Isso renderizará uma <table> consistindo em três linhas totais (<tr>): uma linha de células de cabeçalho (<th>) e duas linhas de células de conteúdo (<td>). Os elementos <th> são cabeçalhos tabulares e os elementos <td> são dados tabulares. Você pode colocar o que quiser dentro de um <td> ou <th>.

Heading 1/Column 1 Heading 2/Column 2 Row 1 Data Column 1 Row 1 Data Column 2 Row 2 Data Column 1 Row 2 Data Column 2

Abrangendo colunas ou linhas #

As células da tabela podem abranger várias colunas ou linhas usando os atributos colspan e rowspan. Esses atributos podem ser aplicados aos elementos <th> e <td>.

<table>
    <tr>
        <td>row 1 col 1</td>
        <td>row 1 col 2</td>
        <td>row 1 col 3</td>
    </tr>
    <tr>
        <td colspan="3">This second row spans all three columns</td>
    </tr>
    <tr>
        <td rowspan="2">This cell spans two rows</td>
        <td>row 3 col 2</td>
        <td>row 3 col 3</td>
    </tr>
    <tr>
        <td>row 4 col 2</td>
        <td>row 4 col 3</td>
    </tr>

</table>

Vai resultar em

Observe que você não deve criar uma tabela em que as linhas e as colunas se sobreponham, pois isso é um HTML inválido e o resultado é tratado de maneira diferente por diferentes navegadores da Web.

rowspan = Um inteiro não negativo que especifica o número de linhas abrangidas por uma célula. O valor padrão deste atributo é um (1). Um valor zero (0) significa que a célula se estenderá da linha atual até a última linha da tabela (<thead>, <tbody> ou <tfoot>).

colspan = Um inteiro não negativo que especifica o número de colunas abrangidas pela célula atual. O valor padrão deste atributo é um (1). Um valor zero (0) significa que a célula se estenderá da coluna atual até a última coluna do grupo de colunas <colgroup> no qual a célula está definida.

Grupos de colunas #

Às vezes, você pode querer aplicar estilo a uma coluna ou grupo de colunas. Ou para fins semânticos, você pode querer agrupar colunas. Para fazer isso, use os elementos <colgroup> e <col>.

A tag opcional <colgroup> permite agrupar colunas. Os elementos <colgroup> devem ser elementos filhos de uma <table> e devem vir após qualquer elemento <caption> e antes de qualquer conteúdo da tabela (por exemplo, <tr>, <thead>, < tbody>, etc.).

<table>
    <colgroup span="2"></colgroup>
    <colgroup span="2"></colgroup>
    ...
</table>

A tag opcional <col> permite que você faça referência a colunas individuais ou a um intervalo de colunas sem aplicar um agrupamento lógico. Os elementos <col> são opcionais, mas se presentes, eles devem estar dentro de um elemento <colgroup>.

<table>
    <colgroup>
        <col id="MySpecialColumn" />
        <col />
    </colgroup>
    <colgroup>
        <col class="CoolColumn" />
        <col class="NeatColumn" span="2" />
    </colgroup>
    ...
</table>

Os seguintes estilos CSS podem ser aplicados aos elementos <colgroup> e <col>:

  • fronteira

  • fundo

  • largura

  • visibilidade

  • display (como em display: none)

    • display: none; will actually remove the columns from the display, causing the table to render as if those cells don’t exist

Para obter mais informações, consulte Dados tabulares HTML5.

Tabela com thead, tbody, tfoot e caption #

O HTML também fornece as tabelas com os elementos <thead>, <tbody>, <tfoot> e <caption>. Esses elementos adicionais são úteis para adicionar valor semântico às suas tabelas e fornecer um local para estilos CSS separados.

Ao imprimir uma tabela que não cabe em uma página (papel), a maioria dos navegadores repete o conteúdo de <thead> em todas as páginas.

Há uma ordem específica que deve ser respeitada, e devemos estar cientes de que nem todos os elementos se encaixam como seria de esperar. O exemplo a seguir demonstra como nossos 4 elementos devem ser colocados.

<table>
 <caption>Table Title</caption> <!--| caption is the first child of table |-->
  <thead> <!--======================| thead is after caption |-->
    <tr>
      <th>Header content 1</th> 
      <th>Header content 2</th>
    </tr>
  </thead>

  <tbody> <!--======================| tbody is after thead |-->
    <tr>
      <td>Body content 1</td>
      <td>Body content 2</td>
    </tr>
  </tbody>
  
  <tfoot><!--| tfoot can be placed before or after tbody, but not in a group of tbody. |-->         <!--| Regardless where tfoot is in markup, it's rendered at the bottom. |-->                                                                                                                                                                                                                                                                                               
    <tr>
      <td>Footer content 1</td>
      <td>Footer content 2</td>
    </tr>
  </tfoot>

</table>

Os resultados do exemplo a seguir são demonstrados duas vezes - a primeira tabela não tem estilos, a segunda tabela tem algumas propriedades CSS aplicadas: background-color, color e border*. Os estilos são fornecidos como um guia visual e não é um aspecto essencial do tópico em questão.

Elemento Estilos aplicáveis
<título> Texto amarelo sobre fundo preto.
<thead> Texto em negrito sobre fundo roxo.
<tbody> Texto sobre fundo azul.
<tfoot> Texto sobre fundo verde.
<th> Bordas laranja.
<td> Fronteiras vermelhas.

Escopo do título #

Os elementos th são muito comumente usados ​​para indicar cabeçalhos para linhas e colunas da tabela, assim:

<table>
    <thead>
        <tr>
            <td></td>
            <th>Column Heading 1</th>
            <th>Column Heading 2</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th>Row Heading 1</th>
            <td></td>
            <td></td>
        </tr>
        <tr>
            <th>Row Heading 2</th>
            <td></td>
            <td></td>
        </tr>
    </tbody>
</table>

Isso pode ser melhorado para acessibilidade pelo uso do atributo scope. O exemplo acima seria alterado da seguinte forma:

<table>
    <thead>
        <tr>
            <td></td>
            <th scope="col">Column Heading 1</th>
            <th scope="col">Column Heading 2</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th scope="row">Row Heading 1</th>
            <td></td>
            <td></td>
        </tr>
        <tr>
            <th scope="row">Row Heading 1</th>
            <td></td>
            <td></td>
        </tr>
    </tbody>
</table>

scope é conhecido como um atributo enumerado, o que significa que pode ter um valor de um conjunto específico de valores possíveis. Este conjunto inclui:

  • colo
  • linha
  • colegrupo
  • grupo de linhas
