Embora muitos scripts, ícones e folhas de estilo possam ser escritos diretamente na marcação HTML, é uma prática recomendada e mais eficiente incluir esses recursos em seu próprio arquivo e vinculá-los ao seu documento. Este tópico abrange a vinculação de recursos externos, como folhas de estilo e scripts, em um documento HTML.

Sintaxe #

  • <link rel="link-relation" type="mime-type" href="url">
  • <script src="path-to-script"></script>

Parâmetros #

Atributo Detalhes
charset Especifica a codificação de caracteres do documento vinculado
crossorigin Especifica como o elemento trata solicitações de origem cruzada
href Especifica a localização do documento vinculado
hreflang Especifica o idioma do texto no documento vinculado
mídia Especifica em qual dispositivo o documento vinculado será exibido, geralmente usado com a seleção de folhas de estilo com base no dispositivo em questão
rel Requeridos. Especifica o relacionamento entre o documento atual e o documento vinculado
revolução Especifica o relacionamento entre o documento vinculado e o documento atual
tamanhos Especifica o tamanho do recurso vinculado. Somente quando rel="icon"
alvo Especifica onde o documento vinculado deve ser carregado
tipo Especifica o tipo de mídia do documento vinculado
integridade Especifica um hash codificado em base64 (sha256, sha384 ou sha512) do recurso vinculado, permitindo que o navegador verifique sua legitimidade.

##Javascript

Síncrono #

<script src="path/to.js"></script>

A prática padrão é colocar as tags JavaScript <script> antes da tag de fechamento </body>. Carregar seus scripts por último permite que os visuais do seu site apareçam mais rapidamente e desencoraja seu JavaScript de tentar interagir com elementos que ainda não foram carregados.

Assíncrono #

<script src="path/to.js" async></script>

Outra alternativa, quando o código Javascript que está sendo carregado não for necessário para a inicialização da página, ele pode ser carregado de forma assíncrona, acelerando o carregamento da página. Usando async o navegador irá carregar o conteúdo do script em paralelo e, uma vez que o download esteja completo, irá interromper a análise do HTML para analisar o arquivo Javascript.

Diferido #

<script src="path/to.js" defer></script>

Os scripts adiados são como scripts assíncronos, com a exceção de que a análise só será executada quando o HTML for totalmente analisado. Scripts adiados são garantidos para serem carregados na ordem de declaração, da mesma forma que scripts síncronos.

<noscript> #

<noscript>JavaScript disabled</noscript>

O elemento <noscript> define o conteúdo a ser exibido se o usuário tiver os scripts desabilitados ou se o navegador não suportar o uso de scripts. A tag <noscript> pode ser colocada no <head> ou no <body>.

Folha de estilo CSS externa #

<link rel="stylesheet" href="path/to.css" type="text/css">

A prática padrão é colocar tags CSS <link> dentro da tag <head> na parte superior do seu HTML. Dessa forma, o CSS será carregado primeiro e será aplicado à sua página à medida que estiver carregando, em vez de mostrar HTML sem estilo até que o CSS seja carregado. O atributo type não é necessário em HTML5, porque HTML5 geralmente suporta CSS.

 <link rel="stylesheet" href="path/to.css" type="text/css">

e

 <link rel="stylesheet" href="path/to.css">

… faça a mesma coisa em HTML5.

Outra prática, embora menos comum, é usar uma instrução @import dentro do CSS direto. Assim:

<style type="text/css">
    @import("path/to.css")
</style>

<style>
    @import("path/to.css")
</style>

Favicon #

<link rel="icon" type="image/png" href="/favicon.png">
<link rel="shortcut icon" type="image/x-icon" href="/favicon.ico">

Use o tipo mime image/png para arquivos PNG e image/x-icon para arquivos de ícone (*.ico). Para saber a diferença, veja esta pergunta SO.

Um arquivo chamado favicon.ico na raiz do seu site normalmente será carregado e aplicado automaticamente, sem a necessidade de uma tag <link>. Se esse arquivo mudar, os navegadores podem ser lentos e teimosos em atualizar seu cache.

CSS alternativo #

<link rel="alternate stylesheet" href="path/to/style.css" title="yourTitle">

Alguns navegadores permitem a aplicação de folhas de estilo alternativas se forem oferecidas. Por padrão, eles não serão aplicados, mas geralmente podem ser alterados através das configurações do navegador:

O Firefox permite que o usuário selecione a folha de estilo usando o submenu Exibir > Estilo de página, o Internet Explorer também suporta esse recurso (começando com o IE 8), também acessado em Exibir > Estilo de página (pelo menos a partir do IE 11), mas o Chrome requer uma extensão para usar o recurso (a partir da versão 48). A página da Web também pode fornecer sua própria interface de usuário para permitir que o usuário alterne estilos.

(Fonte: os MDN Docs)

Dica de recurso: pré-busca de dns, pré-busca, pré-renderização #

Pré-conexão #

O relacionamento preconnect é semelhante ao dns-prefetch, pois resolverá o DNS. No entanto, ele também fará o handshake TCP e a negociação TLS opcional. Este é um recurso experimental.

<link rel="preconnect" href="URL">

Pré-busca de DNS #

Informa os navegadores para resolver o DNS para um URL, para que todos os ativos desse URL sejam carregados mais rapidamente.

<link rel="dns-prefetch" href="URL">

Pré-busca #

Informa aos navegadores que um determinado recurso deve ser pré-buscado para que possa ser carregado mais rapidamente.

<link rel="prefetch" href="URL">

A pré-busca DNS resolve apenas o nome de domínio, enquanto a pré-busca baixa/armazena os recursos especificados.

visualização #

Informa os navegadores para buscar e renderizar a URL em segundo plano, para que possam ser entregues ao usuário instantaneamente à medida que o usuário navega para essa URL. Este é um recurso experimental.

<link rel="prerender" href="URL">

Atributo ‘media’ do link #

<link rel="stylesheet" href="test.css" media="print">

Mídia especifica qual folha de estilo deve ser usada para qual tipo de mídia. Usar o valor print só exibiria essa folha de estilo para páginas impressas.

O valor deste atributo pode ser qualquer um dos [valores mediatype](https://www.wikiod.com/pt/css/consultas-de-midia#tipo de mídia) (semelhante a um CSS media consulta).

Anterior e Próximo #

Quando uma página faz parte de uma série de artigos, por exemplo, pode-se usar prev e next para apontar para as páginas que vêm antes e depois.

<link rel="prev" href="https://www.wikiod.com/pt/java">

<link rel="next" href="https://www.wikiod.com/pt/css">

WebFeed #

Use o atributo rel="alternate" para permitir a descoberta de seus feeds Atom/RSS.

<link rel="alternate" type="application/atom+xml" href="http://example.com/feed.xml" />
<link rel="alternate" type="application/rss+xml" href="http://example.com/feed.xml" />

Consulte os documentos MDN para feeds RSS e [Atomic RSS](https://developer.mozilla.org/ en-US/docs/Web/RSS/Module/Atom).
