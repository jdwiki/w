Meta tags em documentos HTML fornecem informações úteis sobre o documento, incluindo descrição, palavras-chave, autor, datas de modificações e cerca de 90 outros campos. Este tópico aborda o uso e a finalidade dessas tags.

Sintaxe #

  • <meta name="metadata name" content="value">
  • <meta http-equiv="pragma Directive" content="value">
  • <meta charset="etiqueta de codificação">

A meta tag é uma tag HTML usada para definir os metadados do documento HTML. Meta tags precisam ir no elemento head. Uma página pode ter qualquer número de metatags.

A metatag “palavras-chave” normalmente não é usada por robôs. A maioria dos mecanismos de pesquisa determina quais palavras-chave se encaixam no conteúdo das páginas da web. Dito isto, nada diz que você não deve mais incluir a metatag de palavras-chave.

Os metadados de uma página são usados ​​principalmente pelo navegador (como o dimensionamento de um documento) e aranhas de rastreamento da web usadas pelos mecanismos de pesquisa (Google, Yahoo!, Bing).

A especificação fornece vários nomes de metadados padronizados para uso com <meta name> e pragma de metadados padronizados diretivas para uso com <meta http-equiv>. No entanto, muitos serviços na Internet (rastreadores da Web, ferramentas de autoria, serviços de compartilhamento social, etc.) usam o formulário <meta name> como um ponto de extensão genérico para metadados. Alguns deles estão listados na página wiki da especificação.

Informações da página #

nome do aplicativo #

Dando o nome do aplicativo Web que a página representa.

<meta name="application-name" content="OpenStreetMap">

Se não for um aplicativo da Web, a metatag application-name não deve ser usada.

autor #

Defina o autor da página:

<meta name="author" content="Your Name">

Apenas um nome pode ser dado.

descrição #

Defina a descrição da página:

<meta name="description" content="Page Description">

A metatag descrição pode ser usada por vários mecanismos de pesquisa ao indexar sua página da web para fins de pesquisa. Normalmente, a descrição contida na metatag é o breve resumo que aparece sob o título principal da página/site nos resultados do mecanismo de pesquisa. O Google geralmente usa apenas as primeiras 20 a 25 palavras da sua descrição.

gerador #

<meta name="generator" content="HTML Generator 1.42">

Identifica um dos pacotes de software usados ​​para gerar o documento. Apenas para ser usado para páginas onde a marcação é gerada automaticamente.

palavras-chave #

Defina palavras-chave para mecanismos de pesquisa (separadas por vírgulas):

<meta name="keywords" content="Keyword1, Keyword2">

A metatag palavras-chave às vezes é usada pelos mecanismos de pesquisa para conhecer a consulta de pesquisa relevante para sua página da web. Como regra geral, provavelmente é uma boa ideia não adicionar muitas palavras, pois a maioria dos mecanismos de pesquisa que usam essa metatag para indexação indexará apenas as primeiras 20 palavras. Certifique-se de colocar as palavras-chave mais importantes primeiro.

Codificação de Caracteres #

O atributo charset especifica a codificação de caracteres para o documento HTML e precisa ser uma codificação de caracteres válida (os exemplos incluem windows-1252, ISO-8859-2, Shift_JIS e UTF-8). UTF-8 (Unicode) é o mais usado e deve ser usado para qualquer novo projeto.

<meta charset="UTF-8">

<meta charset="ISO-8859-1">

Todos os navegadores sempre reconheceram o formulário <meta charset>, mas se por algum motivo você precisar que sua página seja HTML 4.01 válida, você pode usar o seguinte:

<meta http-equiv="content-type" content="text/html; charset=UTF-8">

<meta http-equiv="content-type" content="text/html; charset=ISO-8859-1">

Consulte também o Padrão de codificação, para visualizar todos os rótulos de codificação de caracteres disponíveis que os navegadores reconhecem.

Robôs #

O atributo robots, suportado por vários motores de busca principais, controla se os spiders dos motores de busca têm permissão para indexar uma página ou não e se devem seguir os links de uma página ou não.

<meta name="robots" content="noindex">

Este exemplo instrui todos os mecanismos de pesquisa a não mostrar a página nos resultados da pesquisa. Outros valores permitidos são:

Valor/Diretiva Significado
todos Padrão. Equivalente a index, follow. Ver nota abaixo.
noindex Não indexe a página de forma alguma.
nofollow Não siga os links nesta página
seguir Os links na página podem ser seguidos. Ver nota abaixo.
nenhum Equivalente a noindex, nofollow.
noarchive Não disponibilize uma versão em cache desta página nos resultados da pesquisa.
nocache Sinônimo de noarchive usado por alguns bots como o Bing.
sem trecho Não mostre um snippet desta página nos resultados da pesquisa.
noodp Não use metadados desta página do projeto Open Directory para títulos ou trechos nos resultados da pesquisa.
notranslate Não ofereça traduções desta página nos resultados da pesquisa.
noimageindex Não indexe imagens nesta página.
indisponível_após [RFC-850 data/hora] Não mostre esta página nos resultados da pesquisa após a data/hora especificada. A data/hora deve ser especificada no formato RFC 850.

Observação: Definir explicitamente index e/ou follow, enquanto valores válidos, não é necessário, pois praticamente todos os mecanismos de pesquisa assumirão que têm permissão para fazê-lo se não forem explicitamente impedidos de fazê-lo. Semelhante à forma como o arquivo robots.txt opera, os mecanismos de pesquisa geralmente procuram apenas coisas que não têm permissão para fazer. Apenas declarar coisas que um mecanismo de busca não tem permissão para fazer também evita a afirmação acidental de opostos (como index, ..., noindex) que nem todos os mecanismos de busca tratarão da mesma maneira.

Mídia social #

Open Graph é um padrão para metadados que estende as informações normais contidas na marcação principal de um site. Isso permite que sites como o Facebook exibam informações mais profundas e ricas sobre um site em um formato estruturado. Essas informações são exibidas automaticamente quando os usuários compartilham links para sites contendo metadados OG no Facebook.

Facebook / gráfico aberto #

<meta property="fb:app_id" content="123456789">
<meta property="og:url" content="https://example.com/page.html">
<meta property="og:type" content="website">
<meta property="og:title" content="Content Title">
<meta property="og:image" content="https://example.com/image.jpg">
<meta property="og:description" content="Description Here">
<meta property="og:site_name" content="Site Name">
<meta property="og:locale" content="en_US">
<meta property="article:author" content="">
<!-- Facebook: https://developers.facebook.com/docs/sharing/webmasters#markup -->
<!-- Open Graph: http://ogp.me/ -->

<meta charset="utf-8">

<!-- The URL of the web version of your article -->

O Twitter usa sua própria marcação para metadados. Esses metadados são usados ​​como informações para controlar como os tweets são exibidos quando contêm um link para o site.

Twitter #

<meta name="twitter:card" content="summary">
<meta name="twitter:site" content="@site_account">
<meta name="twitter:creator" content="@individual_account">
<meta name="twitter:url" content="https://example.com/page.html">
<meta name="twitter:title" content="Content Title">
<meta name="twitter:description" content="Content description less than 200 characters">
<meta name="twitter:image" content="https://example.com/image.jpg">

Controle de layout móvel #

Sites otimizados para celular comuns usam a tag <meta name="viewport"> assim:

<meta name="viewport" content="width=device-width, initial-scale=1">

O elemento viewport fornece ao navegador instruções sobre como controlar as dimensões e o dimensionamento da página com base no dispositivo que você está usando.

No exemplo acima, content="width=device-width significa que o navegador renderizará a largura da página na largura de sua própria tela. Portanto, se essa tela tiver 480px de largura, a janela do navegador será 480px de largura. initial-scale=1 mostra que o zoom inicial (que é 1 neste caso, significa que não faz zoom).

Abaixo estão os atributos que esta tag suporta:

Atributo Descrição
largura A largura da janela de visualização virtual do dispositivo.
Valores1: device-width ou a largura real em pixels, como 480
altura A altura da janela de visualização virtual do dispositivo.
Valores2: device-height ou a largura real em pixels, como 600
escala inicial O zoom inicial quando a página é carregada. 1.0 não amplia.
escala mínima A quantidade mínima que o visitante pode ampliar na página. 1.0 não amplia.
escala máxima A quantidade máxima que o visitante pode ampliar na página. 1.0 não amplia.
escalável pelo usuário Permite que o dispositivo aumente e diminua o zoom. Os valores são sim ou não. Se definido como não, o usuário não poderá ampliar a página da web. O padrão é sim. As configurações do navegador podem ignorar essa regra.

Notas:

1 A propriedade width pode ser especificada em pixels (width=600) ou por device-width (width=device-width) que representa a largura física de a tela do dispositivo.

2 Da mesma forma, a propriedade height pode ser especificada em pixels (height=600) ou por device-height (height=device-height) que representa a altura da tela do dispositivo.

Atualização automática #

Para atualizar a página a cada cinco segundos, adicione este elemento meta no elemento head:

<meta http-equiv="refresh" content="5">

CUIDADO! Embora este seja um comando válido, é recomendado que você não o use devido aos seus efeitos negativos na experiência do usuário. Atualizar a página com muita frequência pode fazer com que ela pare de responder e, muitas vezes, rola para o topo da página. Se alguma informação na página precisar ser atualizada continuamente, há maneiras muito melhores de fazer isso atualizando apenas uma parte da página.

Reconhecimento de número de telefone #

Plataformas móveis como o iOS reconhecem automaticamente os números de telefone e os transformam em links tel:. Embora o recurso seja muito prático, o sistema às vezes detecta códigos ISBN e outros números como números de telefone.

Para que o Safari móvel e alguns outros navegadores móveis baseados em WebKit desativem o reconhecimento e a formatação automáticos de números de telefone, você precisa desta metatag:

<meta name="format-detection" content="telephone=no">

Redirecionamento automático #

Às vezes, sua página da Web precisa de um redirecionamento automático.

Por exemplo, para redirecionar para example.com após 5 segundos:

<meta http-equiv="refresh" content="5;url=https://www.example.com/" />

Esta linha irá enviar você para o site designado (neste caso example.com após 5 segundos.

Se você precisar alterar o atraso de tempo antes de um redirecionamento, basta alterar o número logo antes de seu ;url= alterar o atraso de tempo.

Aplicativo web #

Você pode configurar seu aplicativo da Web ou site para ter um ícone de atalho de aplicativo adicionado à tela inicial de um dispositivo e iniciar o aplicativo no “modo de aplicativo” de tela inteira usando o item de menu “Adicionar à tela inicial” do Chrome para Android .

Abaixo da(s) metatag(s) abrirá o aplicativo da web no modo de tela cheia (sem barra de endereço).

Android Chrome

<meta name="mobile-web-app-capable" content="yes">

IOS

<meta name="apple-mobile-web-app-capable" content="yes">


Você também pode definir a cor da barra de status e da barra de endereço na metatag.

Android Chrome

<meta name="theme-color" content="black">

IOS

<meta name="apple-mobile-web-app-status-bar-style" content="black">
