As tags âncora são comumente usadas para vincular páginas da Web separadas, mas também podem ser usadas para vincular diferentes locais em um único documento, geralmente no índice ou até mesmo iniciar aplicativos externos. Este tópico explica a implementação e aplicação de tags de âncora HTML em várias funções.

Sintaxe #

  • <a href="URL ou âncora">Texto do link</a>

Parâmetros #

Parâmetro Detalhes
href Especifica o endereço de destino. Pode ser uma URL absoluta ou relativa, ou o nome de uma âncora. Um URL absoluto é o URL completo de um site como http://example.com/. Um URL relativo aponta para outro diretório e/ou documento dentro do mesmo site, por exemplo /about-us/ aponta para o diretório “about-us” dentro do diretório raiz (/). Ao apontar para outro diretório sem especificar explicitamente o documento, os servidores web normalmente retornam o documento “index.html” dentro desse diretório.
hreflang Especifica o idioma do recurso vinculado pelo atributo href (que deve estar presente com este). Use valores de idioma de BCP 47 para HTML5 e RFC 1766 para HTML 4.
rel Especifica a relação entre o documento atual e o documento vinculado. Para HTML5, os valores devem ser definidos na especificação ou [registrados no wiki de Microformatos](http ://microformats.org/wiki/existing-rel-values#HTML5_link_type_extensions).
alvo Especifica onde abrir o link, por exemplo. em uma nova guia ou janela. Os valores possíveis são _blank, _self, _parent, _top e framename (obsoleto). Forçar tal comportamento não é recomendado, pois viola o controle do usuário sobre um site.
título Especifica informações extras sobre um link. As informações geralmente são mostradas como um texto de dica de ferramenta quando o cursor se move sobre o link. Este atributo não se restringe a links, pode ser usado em quase todas as tags HTML.
baixar Especifica que o destino será baixado quando um usuário clicar no hiperlink. O valor do atributo será o nome do arquivo baixado. Não há restrições sobre os valores permitidos, e o navegador detectará automaticamente a extensão de arquivo correta e a adicionará ao arquivo (.img, .pdf, etc.). Se o valor for omitido, o nome do arquivo original será usado.

Link para outro site #

Este é o uso básico do elemento <a> (anchor element):

<a href="http://example.com/">Link to example.com</a>

Ele cria um hiperlink para a URL http://example.com/ conforme especificado pelo atributo href (referência de hipertexto), com o texto âncora “Link to example.com”. Seria algo como o seguinte:

Link para example.com

Para indicar que este link leva a um site externo, você pode usar o tipo de link externo:

<a href="http://example.com/" rel="external">example site</a>

Você pode vincular a um site que usa um protocolo diferente de HTTP. Por exemplo, para vincular a um site FTP, você pode fazer,

<a href="ftp://example.com/">This could be a link to a FTP site</a>

Nesse caso, a diferença é que essa marca âncora está solicitando que o navegador do usuário se conecte a example.com usando o protocolo FTP (File Transfer Protocol) em vez do HTTP (Hypertext Transfer Protocol).

Pode ser um link para um site FTP

Link para uma âncora #

As âncoras podem ser usadas para pular para tags específicas em uma página HTML. A tag <a> pode apontar para qualquer elemento que tenha um atributo id. Para saber mais sobre IDs, visite a documentação sobre Classes e IDs. As âncoras são usadas principalmente para pular para uma subseção de uma página e são usadas em conjunto com as tags de cabeçalho.

Suponha que você tenha criado uma página (page1.html) sobre vários tópicos:

<h2>First topic</h2>
<p>Content about the first topic</p>
<h2>Second topic</h2>
<p>Content about the second topic</p>

Depois de ter várias seções, você pode querer criar um Índice na parte superior da página com links rápidos (ou marcadores) para seções específicas.

Se você der um atributo id aos seus tópicos, você pode criar um link para eles

<h2 id="Topic1">First topic</h2>
<p>Content about the first topic</p>
<h2 id="Topic2">Second topic</h2>
<p>Content about the second topic</p>

Agora você pode usar a âncora em seu índice:

<h1>Table of Contents</h1>
    <a href='#Topic1'>Click to jump to the First Topic</a>
    <a href='#Topic2'>Click to jump to the Second Topic</a>

Essas âncoras também são anexadas à página da web em que estão (page1.html). Assim, você pode vincular o site de uma página para outra, referenciando a página e o nome da âncora.

 Remember, you can always <a href="page1.html#Topic1">look back in the First Topic</a> for supporting information.

Link para uma página no mesmo site #

Você pode usar um caminho relativo para vincular a páginas no mesmo site.

<a href="/example">Text Here</a>

O exemplo acima iria para o arquivo example no diretório raiz (/) do servidor.

Se este link estivesse em http://example.com, os dois links a seguir levariam o usuário ao mesmo local

<a href="/page">Text Here</a>
<a href="http://example.com/page">Text Here</a>

Ambos os itens acima iriam para o arquivo page no diretório raiz de example.com.

Link que disca um número #

Se o valor do atributo href começar com tel:, seu dispositivo discará o número quando você clicar nele. Isso funciona em dispositivos móveis ou em computadores/tablets que executam softwares – como Skype ou FaceTime – que podem fazer chamadas telefônicas.

<a href="tel:11234567890">Call us</a>

A maioria dos dispositivos e programas solicitará ao usuário de alguma forma que confirme o número que está prestes a discar.

Abrir link em nova aba/janela #

<a href="example.com" target="_blank">Text Here</a>

O atributo target especifica onde abrir o link. Ao configurá-lo para _blank, você diz ao navegador para abri-lo em uma nova aba ou janela (de acordo com a preferência do usuário).

AVISO DE VULNERABILIDADE DE SEGURANÇA!

Usar target="_blank" dá ao site de abertura acesso parcial ao objeto window.opener via JavaScript, que permite que essa página acesse e altere o window.opener.location de sua página e potencialmente redirecionar os usuários para sites de malware ou phishing.

Sempre que usar isso para páginas que você não controla, adicione rel="noopener" ao seu link para evitar que o objeto window.opener seja enviado com a solicitação.

Atualmente, o Firefox não suporta noopener, então você precisará usar rel="noopener noreferrer" para obter o máximo efeito.

Link que executa JavaScript #

Basta usar o protocolo javascript: para executar o texto como JavaScript em vez de abri-lo como um link normal:

<a href="javascript:myFunction();">Run Code</a>

Você também pode conseguir a mesma coisa usando o atributo onclick:

<a href="#" onclick="myFunction(); return false;">Run Code</a>

O return false; é necessário para evitar que sua página role para o topo quando o link para # for clicado. Certifique-se de incluir todo o código que deseja executar antes dele, pois retornar interromperá a execução de código adicional.

Também digno de nota, você pode incluir um ponto de exclamação ! após a hashtag para evitar que a página role para o topo. Isso funciona porque qualquer slug inválido fará com que o link não role em qualquer lugar na página, porque não conseguiu localizar o elemento referenciado (um elemento com id="!"). Você também pode usar qualquer slug inválido (como #scrollsNowhere) para obter o mesmo efeito. Nesse caso, return false; não é necessário:

<a href="#!" onclick="myFunction();">Run Code</a>

Você deveria usar algo disso?

A resposta é quase certamente não. Executar JavaScript inline com o elemento como este é uma prática bastante ruim. Considere usar soluções JavaScript puras que procuram o elemento na página e vinculam uma função a ele. https://www.wikiod.com/pt/javascript

Considere também se este elemento é realmente um botão em vez de um link. Se sim, você deve usar <button>.

Link que executa o cliente de email #

Uso básico

Se o valor do atributo href começar com mailto: ele tentará abrir um cliente de e-mail ao clicar:

<a href="mailto:example@example.com">Send email</a>

Isso colocará o endereço de e-mail example@example.com como o destinatário do e-mail recém-criado.

Cc e Cco

Você também pode adicionar endereços para destinatários cc ou bcc usando a seguinte sintaxe:

<a href="mailto:example@example.com?cc=john@example.com&bcc=jane@example.com">Send email</a>

Assunto e corpo do texto

Você também pode preencher o assunto e o corpo do novo e-mail:

<a href="mailto:example@example.com?subject=Example+subject&body=Message+text">Send email</a>

Esses valores devem ser [codificados por URL] (https://en.wikipedia.org/wiki/Percent-encoding).

Clicar em um link com mailto: tentará abrir o cliente de e-mail padrão especificado pelo seu sistema operacional ou solicitará que você escolha qual cliente deseja usar. Nem todas as opções especificadas após o endereço do destinatário são compatíveis com todos os clientes de email.
