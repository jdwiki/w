Parâmetros #

Atributo Detalhes
nome Define o nome do elemento, a ser usado com uma tag a para alterar o src do iframe.
largura Define a largura do elemento em pixels.
altura Define a altura do elemento em pixels.
src Especifica a página que será exibida no quadro.
srcdoc Especifica o conteúdo que será exibido no quadro, supondo que o navegador o suporte. O conteúdo deve ser HTML válido.
caixa de areia Quando definido, o conteúdo do iframe é tratado como sendo de uma origem única e recursos como scripts, plugins, formulários e pop-ups serão desabilitados. As restrições podem ser relaxadas seletivamente adicionando uma lista de valores separada por espaços. Consulte a tabela em Comentários para valores possíveis.
allowfullscreen Se deve permitir que o conteúdo do iframe use requestFullscreen()

Um iframe é usado para incorporar outro documento no documento HTML atual.

Você PODE usar iframes para exibir:

  • outras páginas HTML no mesmo domínio;
  • outras páginas HTML em outro domínio (veja abaixo - Política de mesma origem);
  • Documentos PDF (embora o IE possa ter alguns problemas, Esta pergunta SO pode ajudar);

Você DEVE usar um iframe como último recurso, pois tem problemas com bookmarking e navegação, e sempre há opções melhores além de um iframe. Esta pergunta SO deve ajudar você a entender mais sobre os altos e baixos dos iframes.

Política de mesma origem #

Alguns sites não podem ser exibidos usando um iframe porque impõem uma política chamada política de mesma origem. Isso significa que o site no qual o iframe se encontra deve estar no mesmo domínio que aquele a ser exibido.

Esta política também se aplica à manipulação de conteúdo dentro de um iFrame. Se o iFrame estiver acessando conteúdo de um domínio diferente, você não poderá acessar ou manipular o conteúdo dentro de um iFrame.

O elemento iframe em W3C

atributo sandbox #

O atributo sandbox, quando definido, adiciona restrições extras ao iframe. Uma lista de tokens separados por espaços pode ser usada para relaxar essas restrições.

Valor Detalhes
permitir-forms Permite o envio de formulários.
permitir-bloqueio de ponteiro Ativa a API de ponteiro JavaScript.
permitir pop-ups Popups podem ser criados usando window.open ou <a target="_blank"
permitir-mesma-origem O documento iframe usa sua origem real em vez de receber uma única. Se usado com allow-scripts, o documento iframe pode remover todo o sandboxing se for da mesma origem que o documento pai.
permitir scripts Habilita scripts. O documento iframe e o documento pai podem se comunicar usando a API postMessage(). Se usado com allow-same-origin, o documento iframe pode remover todo o sandboxing se for da mesma origem que o documento pai.
permitir navegação superior Permite que o conteúdo do iframe altere a localização do documento de nível superior.

Caixa de areia #

O seguinte incorpora uma página da web não confiável com todas as restrições habilitadas

<iframe sandbox src="http://example.com/"></iframe>

Para permitir que a página execute scripts e envie formulários, adicione allow-scripts e allow-forms ao atributo sandbox.

<iframe sandbox="allow-scripts allow-forms" src="http://example.com/"></iframe>

Se houver conteúdo não confiável (como comentários do usuário) no mesmo domínio da página da Web pai, um iframe poderá ser usado para desabilitar scripts enquanto ainda permite que o documento pai interaja com seu conteúdo usando JavaScript.

<iframe sandbox="allow-same-origin allow-top-navigation" src="http://example.com/untrusted/comments/page2">

O documento pai pode adicionar ouvintes de eventos e redimensionar o IFrame para ajustar seu conteúdo. Isso, junto com allow-top-navigation, pode fazer com que o iframe em área restrita pareça fazer parte do documento pai.

Este sandbox não substitui a entrada de sanitização, mas pode ser usado como parte de uma estratégia de defesa em profundidade.

Esteja ciente também de que esse sandbox pode ser subvertido por um invasor que convença um usuário a visitar a fonte do iframe diretamente. O cabeçalho HTTP Content Security Policy pode ser usado para mitigar esse ataque.

Configurando o tamanho do quadro #

O IFrame pode ser redimensionado usando os atributos width e height, onde os valores são representados em pixels (HTML 4.01 permite valores percentuais, mas HTML 5 só permite valores em pixels CSS).

<iframe src="base.html" width="800" height="600"></iframe>

Noções básicas de um quadro embutido #

O termo “IFrame” significa Inline Frame. Ele pode ser usado para incluir outra página em sua página. Isso produzirá um pequeno quadro que mostra o conteúdo exato do base.html.

<iframe src="base.html"></iframe>

Usando o atributo “srcdoc” #

O atributo srcdoc pode ser usado (ao invés do atributo src) para especificar o conteúdo exato do iframe como um documento HTML completo. Isso produzirá um IFrame com o texto “IFrames são legais!”

<iframe srcdoc="<p>IFrames are cool!</p>"></iframe>

Se o atributo srcdoc não for suportado pelo navegador, o IFrame voltará a usar o atributo src, mas se os atributos src e srcdoc estiverem presentes e suportados pelo navegador, srcdoc tem precedência.

<iframe srcdoc="<p>Iframes are cool!</p>" src="base.html"></iframe>

No exemplo acima, se o navegador não suportar o atributo srcdoc, ele exibirá o conteúdo da página base.html.

Usando âncoras com IFrames #

Normalmente, uma mudança de página da Web dentro de um Iframe é iniciada com o Iframe, por exemplo, clicando em um link dentro do Iframe. No entanto, é possível alterar o conteúdo de um IFrame de fora do IFrame. Você pode usar uma tag âncora cujo atributo href seja definido para a URL desejada e cujo atributo target seja definido para o atributo name do iframe.

<iframe src="webpage.html" name="myIframe"></iframe>
<a href="different_webpage.html" target="myIframe">Change the Iframe content to different_webpage.html</a>
