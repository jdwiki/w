Para agrupar elementos de entrada e enviar dados, o HTML usa um elemento de formulário para encapsular elementos de entrada e envio. Esses formulários tratam do envio dos dados no método especificado para uma página tratada por um servidor ou manipulador. Este tópico explica e demonstra o uso de formulários HTML na coleta e envio de dados de entrada.

Sintaxe #

  • <form method="post|get" action="somePage.php" target="_blank|_self|_parent|_top|framename">

Parâmetros #

Atributo                   Descrição
aceitar-charset Especifica as codificações de caracteres que devem ser usadas para o envio do formulário.
ação Especifica para onde enviar os dados do formulário quando um formulário é enviado.
autocompletar Especifica se um formulário deve ter o preenchimento automático ativado ou desativado.
enctype Especifica como os dados do formulário devem ser codificados ao enviá-los ao servidor (somente para method=“post”).
método Especifica o método HTTP a ser usado ao enviar dados de formulário (POST ou GET).
nome Especifica o nome de um formulário.
novalidar Especifica que o formulário não deve ser validado quando enviado.
alvo Especifica onde exibir a resposta recebida após o envio do formulário.

O elemento <form> representa uma seção que contém elementos associados ao formulário (por exemplo, <button> <fieldset> <input> <label> <output> <select> < textarea>) que envia informações para um servidor. Ambas as tags inicial (<form>) e final (</form>) são necessárias.

Enviando #

O atributo de ação #

O atributo action define a ação a ser executada quando o formulário é enviado, o que geralmente leva a um script que coleta as informações enviadas e trabalha com elas. se você deixar em branco, ele será enviado para o mesmo arquivo

<form action="action.php">

O atributo de método #

O atributo method é usado para definir o método HTTP do formulário que é GET ou POST.

<form action="action.php" method="get">
<form action="action.php" method="post">

O método GET é usado principalmente para get dados, por exemplo, para receber uma postagem por seu ID ou nome, ou para enviar uma consulta de pesquisa. O método GET anexará os dados do formulário à URL especificada no atributo de ação.

www.example.com/action.php?firstname=Mickey&lastname=Mouse

O método POST é usado ao enviar dados para um script. O método POST não anexa os dados do formulário ao URL da ação, mas envia usando o corpo da solicitação.

Para enviar os dados do formulário corretamente, um nome de atributo de nome deve ser especificado. Como exemplo vamos enviar o valor do campo e definir seu nome como lastname:

<input type="text" name="lastname" value="Mouse">

Mais atributos #

<form action="action.php" method="post" target="_blank" accept-charset="UTF-8" 
enctype="application/x-www-form-urlencoded" autocomplete="off" novalidate>

<!-- form elements -->

</form>

Atributo de destino na tag de formulário #

O atributo target especifica um nome ou uma palavra-chave que indica onde exibir a resposta recebida após o envio do formulário.

O atributo target define um nome ou palavra-chave para um contexto de navegação (por exemplo, guia, janela ou quadro embutido).

Da Tag com um atributo de destino:

<form target="_blank">

Valores do atributo

Value Description _blank The response is displayed in a new window or tab _self The response is displayed in the same frame (this is default) _parent The response is displayed in the parent frame _top The response is displayed in the full body of the window framename The response is displayed in a named iframe

Observação: o atributo de destino foi obsoleto no HTML 4.01. O alvo o atributo é suportado em HTML5.

Quadros e conjuntos de quadros não são suportados em HTML5, então o _parent, _top e valores de nome de quadro agora são usados ​​principalmente com iframes.

Fazendo upload de arquivos #

Imagens e arquivos podem ser carregados/enviados para o servidor configurando o atributo enctype da tag form para multipart/form-data. enctype especifica como os dados do formulário seriam codificados durante o envio ao servidor.

Exemplo

<form method="post" enctype="multipart/form-data" action="upload.php"> 
    <input type="file" name="pic" />
    <input type="submit" value="Upload" />
</form>

Agrupando alguns campos de entrada #

Ao projetar um formulário, você pode agrupar alguns campos de entrada em um grupo para ajudar a organizar o layout do formulário. Isso pode ser feito usando a tag

. Aqui está um exemplo para usá-lo.

Para cada fieldset, você pode definir uma legenda para o conjunto usando a tag LEGEND TEXT

Exemplo

<form>
  <fieldset>
     <legend>1st field set:</legend>
     Field one:<br>
     <input type="text"><br>
     Field two:<br>
     <input type="text"><br>
  </fieldset><br>
  <fieldset>
     <legend>2nd field set:</legend>
     Field three:<br>
     <input type="text"><br>
     Field four:<br>
     <input type="text"><br>
  </fieldset><br>
  <input type="submit" value="Submit">
</form>
