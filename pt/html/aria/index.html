ARIA é uma especificação para descrever semanticamente aplicações web ricas. Seguir os padrões ARIA pode aumentar a acessibilidade para aqueles que usam tecnologias assistivas (como um leitor de tela) para acessar seu conteúdo.

role=“apresentação” #

Um elemento cuja semântica de função nativa implícita não será mapeada para a API de acessibilidade.

<div style="float:left;">Some content on the left.</div>
<div style="float:right;">Some content on the right</div>
<div role="presentation" style="clear:both;"></div> <!-- Only used to clear floats -->

role=“alerta” #

Uma mensagem com informações importantes e geralmente sensíveis ao tempo.

<div role="alert" aria-live="assertive">Your session will expire in 60 seconds.</div>

Observe que incluí tanto role="alert" quanto aria-live="assertive" ao mesmo tempo. Esses são atributos sinônimos, mas alguns leitores de tela suportam apenas um ou outro. Ao usar os dois simultaneamente, maximizamos as chances de que a região ativa funcione conforme o esperado.

Fonte - Heydon Pickering ‘Alguns exemplos práticos de ARIA’

role=“alertdialog” #

Um tipo de diálogo que contém uma mensagem de alerta, onde o foco inicial vai para um elemento dentro do diálogo.

<div role="alertdialog">
  <h1>Warning</h1>
  <div role="alert">Your session will expire in 60 seconds.</div>
</div>

role=“aplicativo” #

Uma região declarada como um aplicativo da Web, em oposição a um documento da Web. Neste exemplo, o aplicativo é uma calculadora simples que pode somar dois números.

<div role="application">
  <h1>Calculator</h1>
  <input id="num1" type="text"> + <input id="num2" type="text"> =
  <span id="result"></span>
</div>

role=“artigo” #

Uma seção de uma página que consiste em uma composição que forma uma parte independente de um documento, página ou site.

Definir uma função ARIA e/ou atributo aria-* que corresponda à semântica ARIA implícita padrão é desnecessário e não é recomendado, pois essas propriedades já estão definidas pelo navegador.

<article>
  <h1>My first article</h1>
  <p>Lorem ipsum...</p>
</article>

Você usaria role=article em elementos não semânticos (não recomendado, inválido)

<div role="article">
  <h1>My first article</h1>
  <p>Lorem ipsum...</p>
</div>

Entrada do W3C para role=article

role=“banner” #

Uma região que contém principalmente conteúdo orientado ao site, em vez de conteúdo específico da página.

<div role="banner">
  <h1>My Site</h1>

  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</div>

role=“botão” #

Uma entrada que permite ações acionadas pelo usuário quando clicadas ou pressionadas.

<button role="button">Add</button>

role=“célula” #

Uma célula em um contêiner tabular.

<table>
  <thead>
    <!-- etc -->
  </thead>
  <tbody>
    <td role="cell">95</td>
    <td role="cell">14</td>
    <td role="cell">25</td>
  </tbody>
</table>

role=“checkbox” #

Uma entrada verificável que tem três valores possíveis: verdadeiro, falso ou misto.

<p>
  <input type="checkbox" role="checkbox" aria-checked="false">
  I agree to the terms
</p>

role=“columnheader” #

Uma célula contendo informações de cabeçalho para uma coluna.

<table role="grid">
  <thead>
    <tr>
      <th role="columnheader">Day 1</th>
      <th role="columnheader">Day 2</th>
      <th role="columnheader">Day 3</th>
    </tr>
  </thead>
  <tbody>
    <!-- etc -->
  </tbody>
<table>

role=“combobox” #

Uma apresentação de um select; geralmente semelhante a uma caixa de texto onde os usuários podem digitar antecipadamente para selecionar uma opção ou digitar para inserir um texto arbitrário como um novo item na lista.

<input type="text" role="combobox" aria-expanded="false">

Normalmente, você usaria JavaScript para construir o restante da funcionalidade typeahead ou list select.

role=“complementar” #

Uma seção de suporte do documento, projetada para complementar o conteúdo principal em um nível semelhante na hierarquia DOM, mas permanece significativa quando separada do conteúdo principal.

<div role="complementary">
  <h2>More Articles</h2>

  <ul>
    <!-- etc -->
  </ul>
</div>

role=“contentinfo” #

Uma grande região perceptível que contém informações sobre o documento pai.

<p role="contentinfo">
  Author: Albert Einstein<br>
  Published: August 15, 1940
</p>

role=“definição” #

Definição de um termo ou conceito.

<span role="term" aria-labelledby="def1">Love</span>
<span id="def1" role="definition">an intense feeling of deep affection.</span>

role=“diálogo” #

Uma caixa de diálogo é uma janela de aplicativo projetada para interromper o processamento atual de um aplicativo para solicitar ao usuário que insira informações ou solicite uma resposta.

<div role="dialog">
  <p>Are you sure?</p>
  <button role="button">Yes</button>
  <button role="button">No</button>
</div>

role=“diretório” #

Uma lista de referências a membros de um grupo, como um índice estático.

<ul role="directory">
  <li><a href="/chapter-1">Chapter 1</a></li>
  <li><a href="/chapter-2">Chapter 2</a></li>
  <li><a href="/chapter-3">Chapter 3</a></li>
</ul>

role=“documento” #

Uma região que contém informações relacionadas que são declaradas como conteúdo de documento, em oposição a um aplicativo da web.

<div role="document">
  <h1>The Life of Albert Einstein</h1>
  <p>Lorem ipsum...</p>
</div>

role=“form” #

Uma região de referência que contém uma coleção de itens e objetos que, como um todo, se combinam para criar um formulário.

Usar o elemento HTML semanticamente correto <form> implica semântica ARIA padrão, significando que role=form não é necessário, pois você não deve aplicar uma função contrastante a um elemento que já é semântico, pois adicionar uma função substitui a semântica nativa de um elemento.

Definir uma função ARIA e/ou atributo aria-* que corresponda à semântica ARIA implícita padrão é desnecessário e não é recomendado, pois essas propriedades já estão definidas pelo navegador.

<form action="">
  <fieldset>
    <legend>Login form</legend>
    <div>
      <label for="username">Your username</label>
      <input type="text" id="username" aria-describedby="username-tip" required />
      <div role="tooltip" id="username-tip">Your username is your email address</div>
    </div>
    <div>
      <label for="password">Your password</label>
      <input type="text" id="password" aria-describedby="password-tip" required />
      <div role="tooltip" id="password-tip">Was emailed to you when you signed up</div>
    </div>
  </fieldset>
</form>

Você usaria role=form em elementos não semânticos (não recomendado, inválido)

<div role=form>
  <input type="email" placeholder="Your email address">
  <button>Sign up</button>
</div>

role=“grade” #

Uma grade é um controle interativo que contém células de dados tabulares organizados em linhas e colunas, como uma tabela.

<table role="grid">
  <thead>
    <!-- etc -->
  </thead>
  <tbody>
    <!-- etc -->
  </tbody>
</table>

role=“gridcell” #

Uma célula em uma grade ou treegrid.

<table role="grid">
  <thead>
    <!-- etc -->
  </thead>
  <tbody>
    <tr>
      <td role="gridcell">17</td>
      <td role="gridcell">64</td>
      <td role="gridcell">18</td>
    </tr>
  </tbody>
<table>

role=“grupo” #

Um conjunto de objetos de interface do usuário que não se destinam a ser incluídos em um resumo de página ou índice por tecnologias assistivas.

<div role="group">
  <button role"button">Previous</button>
  <button role"button">Next</button>
</div>

role=“título” #

Um título para uma seção da página.

<h1 role="heading">Introduction</h1>
<p>Lorem ipsum...</p>

role=“img” #

Um contêiner para uma coleção de elementos que formam uma imagem.

<figure role="img">
  <img alt="A cute cat." src="albert.jpg">
  <figcaption>This is my cat, Albert.</figcaption>
<figure>

role=“link” #

Uma referência interativa a um recurso interno ou externo que, quando ativado, faz com que o agente do usuário navegue até esse recurso.

Na maioria dos casos, definir uma função ARIA e/ou atributo aria-* que corresponda à semântica ARIA implícita padrão é desnecessário e não recomendado, pois essas propriedades já estão definidas pelo navegador.

Fonte - https://www.w3.org/TR/html5/dom.html#aria-usage-note

role=“lista” #

Um grupo de itens de lista não interativos.

<ul role="list">
  <li role="listitem">One</li>
  <li role="listitem">Two</li>
  <li role="listitem">Three</li>
</ul>

role=“listbox” #

Um widget que permite ao usuário selecionar um ou mais itens de uma lista de opções.

<ul role="listbox">
  <li>One</li>
  <li>Two</li>
  <li>Three</li>
</ul>

Normalmente, você usaria JavaScript para criar a funcionalidade de seleção múltipla.

role=“listitem” #

Um único item em uma lista ou diretório.

<ul role="list">
  <li role="listitem">One</li>
  <li role="listitem">Two</li>
  <li role="listitem">Three</li>
</ul>

role=“log” #

Um tipo de região ativa onde novas informações são adicionadas em uma ordem significativa e as informações antigas podem desaparecer.

<ul role="log">
  <li>User 1 logged in.</li>
  <li>User 2 logged in.</li>
  <li>User 1 logged out.</li>
</ul>

role=“principal” #

O conteúdo principal de um documento.

<!-- header & nav here -->
<div role="main">
  <p>Lorem ipsum...</p>
</div>
<!-- footer here -->

role=“marquee” #

Um tipo de região ativa onde informações não essenciais mudam com frequência.

<ul role="marquee">
  <li>Dow +0.26%</li>
  <li>Nasdaq +0.54%</li>
  <li>S&amp;P +0.44%</li>
</ul>

role=“matemática” #

Conteúdo que representa uma expressão matemática.

<img role="math" alt="y=mx+b" src="slope.png">

role=“menu” #

Um tipo de widget que oferece uma lista de opções ao usuário.

<ul role="menu">
  <li role="menuitem">New</li>
  <li role="menuitem">Open</li>
  <li role="menuitem">Save</li>
  <li role="menuitem">Close</li>
</ul>

role=“menubar” #

Uma apresentação de menu que geralmente permanece visível e geralmente é apresentada horizontalmente.

<ul role="menubar">
  <li role="menuitem">File</li>
  <li role="menuitem">Edit</li>
  <li role="menuitem">View</li>
  <li role="menuitem">Help</li>
</ul>

role=“menuitem” #

Uma opção em um grupo de opções contidas por um menu ou barra de menus.

<ul role="menubar">
  <li role="menuitem">File</li>

role=“menuitemcheckbox” #

Um item de menu verificável que tem três valores possíveis: verdadeiro, falso ou misto.

<ul role="menu">
  <li role="menuitem">Console</li>
  <li role="menuitem">Layout</li>
  <li role="menuitemcheckbox" aria-checked="true">Word wrap</li>
</ul>

role=“menuitemradio” #

Um item de menu verificável em um grupo de funções de menuitemradio, das quais apenas uma pode ser verificada por vez.

<ul role="menu">
  <li role="menuitemradio" aria-checked="true">Left</li>
  <li role="menuitemradio" aria-checked="false">Center</li>
  <li role="menuitemradio" aria-checked="false">Right</li>
</ul>

role=“navegação” #

Uma coleção de elementos de navegação (geralmente links) para navegar pelo documento ou documentos relacionados.

<ul role="navigation">
  <li><a href="/">Home</a></li>
  <li><a href="/about">About</a></li>
  <li><a href="/contact">Contact</a></li>
</ul>

role=“nota” #

Uma seção cujo conteúdo é parentético ou auxiliar ao conteúdo principal do recurso.

<p>Lorem ipsum...</p>
<p>Lorem ipsum...</p>
<p role="note">Lorem ipsum...</p>

role=“opção” #

Um item selecionável em uma lista de seleção.

<ul role="listbox">
  <li role="option">Option 1</li>
  <li role="option">Option 2</li>
  <li role="option">Option 3</li>
</ul>

role=“progressbar” #

Um elemento que exibe o status de andamento de tarefas que levam muito tempo.

<progress role="progressbar" value="25" max="100">25%</progress>

role=“radio” #

Uma entrada verificável em um grupo de funções de rádio, das quais apenas uma pode ser verificada por vez.

<div role="radiogroup">
  <input role="radio" type="radio" aria-checked="true"> One<br>
  <input role="radio" type="radio" aria-checked="false"> Two<br>
  <input role="radio" type="radio" aria-checked="false"> Three
</div>

role=“região” #

Uma grande seção perceptível de uma página da Web ou documento, que o autor considera importante o suficiente para ser incluída em um resumo de página ou índice, por exemplo, uma área da página contendo estatísticas de eventos esportivos ao vivo.

<div role="region">
  Home team: 4<br>
  Away team: 2
</div>

role=“radiogrupo” #

Um grupo de botões de rádio.

<div role="radiogroup">
  <input role="radio" type="radio" aria-checked="true"> One<br>
  <input role="radio" type="radio" aria-checked="false"> Two<br>
  <input role="radio" type="radio" aria-checked="false"> Three
</div>

role=“linha” #

Uma linha de células em um contêiner tabular.

role=“rowgroup” #

Um grupo que contém um ou mais elementos de linha em uma grade.

<table>
  <thead role="rowgroup">
    <!-- etc -->
  </thead>
  <tbody role="rowgroup">
    <!-- etc -->
  </tbody>
</table>

role=“rowheader” #

Uma célula contendo informações de cabeçalho para uma linha em uma grade.

<table role="grid">
  <thead>
    <!-- etc -->
  </thead>

role=“barra de rolagem” #

Um objeto gráfico que controla a rolagem do conteúdo em uma área de visualização, independentemente de o conteúdo ser totalmente exibido na área de visualização.

<div id="content1">Lorem ipsum...</div>
<div
  role="scrollbar"
  aria-controls="content1"
  aria-orientation="vertical"
  aria-valuemax="100"
  aria-valuemin="0"
  aria-valuenow="25">
    <div class="scrollhandle"></div>
</div>

role=“pesquisar” #

Uma região de referência que contém uma coleção de itens e objetos que, como um todo, se combinam para criar um recurso de pesquisa.

<div role="search">
  <input role="searchbox" type="text">
  <button role="button">Search</button>
</div>

role=“caixa de pesquisa” #

Um tipo de caixa de texto destinada a especificar critérios de pesquisa.

<div role="search">
  <input role="searchbox" type="text">
  <button role="button">Search</button>
</div>

role=“separador” #

Um divisor que separa e distingue seções de conteúdo ou grupos de itens de menu.

<p>Lorem ipsum...</p>
<hr role="separator">
<p>Lorem ipsum...</p>

role=“slider” #

Uma entrada do usuário em que o usuário seleciona um valor dentro de um determinado intervalo.

<div
  role="slider"
  aria-valuemax="100"
  aria-valuemin="0"
  aria-valuenow="25">
    <div class="sliderhandle"></div>
</div>

role=“spinbutton” #

Uma forma de intervalo que espera que o usuário selecione entre opções discretas.

<input
  role="spinbutton"
  aria-valuemax="100"
  aria-valuemin="0"
  aria-valuenow="25"
  type="number"
  value="25">

role=“status” #

Um contêiner cujo conteúdo é uma informação consultiva para o usuário, mas não é importante o suficiente para justificar um alerta, muitas vezes, mas não necessariamente, apresentado como uma barra de status.

<div role="status">Online</div>

role=“switch” #

Um tipo de caixa de seleção que representa valores ativados/desativados, em oposição aos valores marcados/desmarcados.

  <option selected>Off</option>

role=“dica de ferramenta” #

Um pop-up contextual que exibe uma descrição para um elemento.

<span aria-describedby="slopedesc">Slope</span>
<div role="tooltip" id="slopedesc">y=mx+b</div>

Normalmente, a dica de ferramenta estaria oculta. Usando JavaScript, a dica de ferramenta seria exibida após um atraso quando o usuário passasse o mouse sobre o elemento que ela descreve.
