Classes e IDs facilitam a referência de elementos HTML de scripts e folhas de estilo. O atributo class pode ser usado em uma ou mais tags e é usado pelo CSS para estilização. No entanto, os IDs devem se referir a um único elemento, o que significa que o mesmo ID nunca deve ser usado duas vezes. Os IDs geralmente são usados ​​com JavaScript e links de documentos internos e são desencorajados em CSS. Este tópico contém explicações e exemplos úteis sobre o uso adequado de atributos de classe e ID em HTML.

Sintaxe #

  • class=“class1 class2 class3”
  • id=“uniqueid”

Parâmetros #

Parâmetro Detalhes
aula Indica a classe do elemento (não exclusivo)
identificação Indica o ID do elemento (único no mesmo contexto)
  • Ambos class e id são atributos globais e, portanto, podem ser atribuídos a qualquer elemento HTML.
  • Os nomes das classes devem começar com uma letra (A-Z ou a-z) e podem ser seguidos por letras, dígitos, hífens e sublinhados.
  • Em HTML5, os atributos class e id podem ser usados ​​em qualquer elemento. No HTML 4.0.1, eles estavam fora dos limites para <base>, <head>, <html>, <meta>, <param>, <script>, tags <style> e <title>.
  • Um elemento pode ter uma ou mais classes. As classes são separadas por espaços e não podem conter espaços.
  • Um elemento pode ter apenas um ID e deve ser único dentro de seu contexto (ou seja, uma página da web). Os IDs também não podem conter espaços.

Dando uma classe a um elemento #

As classes são identificadores para os elementos aos quais são atribuídas. Use o atributo class para atribuir uma classe a um elemento.

<div class="example-class"></div>

Para atribuir várias classes a um elemento, separe os nomes das classes com espaços.

<div class="class1 class2"></div>

Usando classes em CSS #

As classes podem ser usadas para estilizar certos elementos sem alterar todos os elementos desse tipo. Por exemplo, esses dois elementos span podem ter estilos completamente diferentes:

<span></span>
<span class="special"></span>

Classes com o mesmo nome podem ser dadas a qualquer número de elementos em uma página e todos eles receberão o estilo associado a essa classe. Isso sempre será verdade, a menos que você especifique o elemento dentro do CSS.

Por exemplo, temos dois elementos, ambos com a classe highlight:

<div class="highlight">Lorem ipsum</div>
<span class="highlight">Lorem ipsum</span>

Se nosso CSS for como abaixo, a cor verde será aplicada ao texto em ambos os elementos:

.highlight { color: green; }

No entanto, se queremos apenas segmentar div’s com a classe highlight, podemos adicionar especificidade como abaixo:

div.highlight { color: green; }

No entanto, ao estilizar com CSS, geralmente é recomendado que apenas classes (por exemplo, .highlight) sejam usadas em vez de elementos com classes (por exemplo, div.highlight).

Como com qualquer outro seletor, as classes podem ser aninhadas:

.main .highlight { color: red; } /* Descendant combinator */ 
.footer > .highlight { color: blue; } /* Child combinator */

Você também pode encadear o seletor de classe para selecionar apenas elementos que tenham uma combinação de várias classes. Por exemplo, se este for nosso HTML:

<div class="special left menu">This text will be pink</div>

E queremos colorir esse texto específico de rosa, podemos fazer o seguinte em nosso CSS:

.special.left.menu { color: pink; }

Dando um ID a um elemento #

O atributo ID de um elemento é um identificador que deve ser único em todo o documento. Sua finalidade é identificar exclusivamente o elemento ao vincular (usando uma âncora), scripting ou estilo (com CSS).

<div id="example-id"></div>

Você não deve ter dois elementos com o mesmo ID no mesmo documento, mesmo que os atributos estejam anexados a dois tipos diferentes de elementos. Por exemplo, o código a seguir está incorreto:

<div id="example-id"></div>
<span id="example-id"></span>

Os navegadores farão o possível para renderizar esse código, mas pode ocorrer um comportamento inesperado ao estilizar com CSS ou adicionar funcionalidade com JavaScript.

Para referenciar elementos por seu ID em CSS, prefixe o ID com #.

#example-id { color: green; }

Para pular para um elemento com um ID em uma determinada página, acrescente # com o nome do elemento na URL.

http://example.com/about#example-id

Esse recurso é compatível com a maioria dos navegadores e não requer JavaScript ou CSS adicional para funcionar.

Valores Aceitáveis #

Para um ID #

As únicas restrições sobre o valor de um id são:

  1. deve ser único no documento
  2. não deve conter nenhum caractere de espaço
  3. deve conter pelo menos um caractere

Portanto, o valor pode ser todos os dígitos, apenas um dígito, apenas caracteres de pontuação, incluir caracteres especiais, o que for. Apenas nenhum espaço em branco.

Então são válidos:

<div id="container"> ... </div>
<div id="999"> ... </div>
<div id="#%LV-||"> ... </div>
<div id="____V"> ... </div>
<div id="⌘⌥"> ... </div>
<div id="♥"> ... </div>
<div id="{}"> ... </div>
<div id="©"> ... </div>
<div id="♤₩¤☆€~¥"> ... </div>

Isso é inválido:

<div id=" "> ... </div>

Isso também é inválido, quando incluído no mesmo documento:

<div id="results"> ... </div>
<div id="results"> ... </div>

Um valor id deve começar com uma letra, que pode ser seguida apenas por:

  • letras (A-Z/a-z)
  • dígitos (0-9)
  • hifens ("-")
  • sublinhado ("_")
  • dois pontos (":")
  • pontos ("."")

Referindo-se ao primeiro grupo de exemplos na seção HTML5 acima, apenas um é válido:

<div id="container"> ... </div>

Também são válidos:

<div id="sampletext"> ... </div>
<div id="sample-text"> ... </div>
<div id="sample_text"> ... </div>
<div id="sample:text"> ... </div>
<div id="sample.text"> ... </div>

Novamente, se não começar com uma letra (maiúscula ou minúscula), não é válido.

Para uma aula #

As regras para classes são essencialmente as mesmas de um id. A diferença é que os valores class não precisam ser únicos no documento.

Referindo-se aos exemplos acima, embora isso não seja válido no mesmo documento:

<div id="results"> ... </div>
<div id="results"> ... </div>

Isso está perfeitamente bem:

<div class="results"> ... </div>
<div class="results"> ... </div>

Observação importante: como os valores de ID e classe são tratados fora do HTML #

Tenha em mente que as regras e exemplos acima se aplicam no contexto do HTML.

Usar números, pontuação ou caracteres especiais no valor de um id ou uma class pode causar problemas em outros contextos, como CSS, JavaScript e expressões regulares.

Por exemplo, embora o seguinte id seja válido em HTML5:

<div id="9lions"> ... </div>

… é inválido em CSS:

4.1.3 Caracteres e maiúsculas

Em CSS, identificadores (incluindo nomes de elementos, classes e IDs em seletores) podem conter apenas os caracteres [a-zA-Z0-9] e ISO 10646 caracteres U+00A0 e superiores, mais o hífen (-) e o sublinhado (_); eles não podem começar com um dígito, dois hífens ou um hífen seguido de um dígito. (ênfase adicionada)

Na maioria dos casos, você pode escapar caracteres em contextos onde eles têm restrições ou significado especial.

Problemas relacionados a IDs duplicados #

Ter mais de um elemento com o mesmo ID é um problema difícil de solucionar. O analisador HTML normalmente tentará renderizar a página em qualquer caso. Normalmente não ocorre nenhum erro. Mas o ritmo pode acabar em uma página da web com mau comportamento.

Neste exemplo:

<div id="aDiv">a</div>
<div id="aDiv">b</div>

Seletores CSS ainda funcionam

#aDiv {
    color: red;
}

Mas o JavaScript não consegue lidar com ambos os elementos:

var html = document.getElementById("aDiv").innerHTML;

Neste caso, a variável html carrega apenas o primeiro conteúdo div ("a").
