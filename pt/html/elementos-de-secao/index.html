Elemento de navegação #

O elemento <nav> destina-se principalmente a ser usado para seções que contêm blocos de navegação principais para o site, isso pode incluir links para outras partes da página da web (por exemplo, âncoras para um índice) ou outras páginas inteiramente.

itens embutidos #

O seguinte exibirá um conjunto embutido de hiperlinks.

<nav>
    <a href="https://google.com">Google</a>
    <a href="https://www.yahoo.com">Yahoo!</a>
    <a href="https://www.bing.com">Bing</a>
</nav>

Use itens de lista quando necessário #

Se o conteúdo representar uma lista de itens, use um item de lista para mostrar isso e aprimorar a experiência do usuário.

Observe o role="navigation", mais sobre isso abaixo.

<nav role="navigation">
    <ul>
        <li><a href="https://google.com">Google</a></li>
        <li><a href="https://www.yahoo.com">Yahoo!</a></li>
        <li><a href="https://www.bing.com">Bing</a></li>
    </ul>
</nav>

Evite o uso desnecessário #

Os elementos <footer> podem ter uma lista de links para outras partes do site (FAQ, T&C, etc.). O elemento footer sozinho é suficiente neste caso, você não precisa envolver seus links com um elemento <nav> no <footer>.

<!-- the <nav> is not required in the <footer> -->
<footer>
    <nav>
        <a href="#">...</a>
    </nav>
</footer>

<!-- The footer alone is sufficient -->
<footer>
    <a href="#">...</a>
</footer>

Observações:

  • <main> element descendentes não são permitidos dentro de um <nav>

Adicionar um role="navigation" ARIA role ao elemento <nav> é recomendado para ajudar os agentes de usuário que não suportam HTML5 e também para fornecer mais contexto para aqueles que suportam.

Leitores de tela: (software que permite que usuários cegos ou com deficiência visual naveguem no site)

Agentes de usuário como leitores de tela irão interpretar o elemento <nav> de forma diferente dependendo de seus requisitos.

  • Poderia dar ao elemento <nav> uma prioridade mais alta ao renderizar a página
  • Pode atrasar a renderização do elemento
  • Poderia adaptar a página de uma forma específica às necessidades do usuário exemplo: torne os links de texto dentro dos elementos <nav> maiores para pessoas com deficiência visual.

Clique aqui para ler a Especificação HTML5 oficial para o elemento <nav>

Elemento do artigo #

O elemento <article> contém conteúdo autocontido como artigos, postagens de blog, comentários de usuários ou um widget interativo que pode ser distribuído fora do contexto da página, por exemplo, por RSS.

  • Quando os elementos do artigo são aninhados, o conteúdo do nó do artigo interno deve estar relacionado ao elemento do artigo externo.

Um blog (section) com vários posts (article) e comentários (article) pode ser algo assim.

<section>
    <!-- Each individual blog post is an <article> -->
    <article>
        <header>
            <h1>Blog Post</h1>
            <time datetime="2016-03-13">13th March 2016</time>
        </header>

        <p>The article element represents a self contained article or document.</p>
        <p>The section element represents a grouping of content.</p>

        <section>
            <h2>Comments <small>relating to "Blog Post"</small></h2>

            <!-- Related comment is also a self-contained article -->
            <article id="user-comment-1">
                <p>Excellent!</p>
                <footer><p>...</p><time>...</time></footer>
            </article>
        </section>
    </article>

    <!-- ./repeat: <article> -->

</section>

<!-- Content unrelated to the blog or posts should be outside the section. -->
<footer>
    <p>This content should be unrelated to the blog.</p>
</footer>

Evite o uso desnecessário #

Quando o conteúdo principal da página (excluindo cabeçalhos, rodapés, barras de navegação, etc.) é simplesmente um grupo de elementos. Você pode omitir o <article> em favor do elemento <main>.

<article>
    <p>This doesn't make sense, this article has no real `context`.</p>
</article>

Em vez disso, substitua o artigo por um elemento <main> para indicar que este é o conteúdo main desta página.

<main>
    <p>I'm the main content, I don't need to belong to an article.</p>
</main>

Se você usar outro elemento, certifique-se de especificar a função <main> ARIA para interpretação e renderização corretas em vários dispositivos e navegadores não HTML5.

<section role="main">
    <p>This section is the main content of this page.</p>
</section>

Observações:

  • <main> element descendentes não são permitidos dentro de um <article>

Clique aqui para ler a Especificação HTML5 oficial para o elemento <article>

Elemento principal #

O elemento <main> contém o conteúdo principal de sua página da web. Este conteúdo é exclusivo para a página individual e não deve aparecer em nenhum outro lugar do site. O conteúdo repetido, como cabeçalhos, rodapés, navegação, logotipos, etc., é colocado fora do elemento.

  • O elemento <main> só deve ser usado no máximo uma vez em uma única página.
  • O elemento <main> não deve ser incluído como descendente de um elemento article, aside, footer, header ou nav.

No exemplo a seguir, estamos exibindo uma única postagem no blog (e informações relacionadas, como referências e comentários).

<body>
    <header>
        <nav>...</nav>
    </header>

    <main>
        <h1>Individual Blog Post</h1>
        <p>An introduction for the post.</p>

        <article>
            <h2>References</h2>
            <p>...</p>
        </article>

        <article>
            <h2>Comments</h2> ...
        </article>
    </main>

    <footer>...</footer>
</body>

  • A postagem do blog está contida no elemento <main> para indicar que este é o conteúdo principal desta página (e, portanto, exclusivo em todo o site).

  • As tags <header> e <footer> são irmãos do elemento <main>.

Observações:

A especificação HTML5 reconhece o elemento <main> como um elemento grouping, e não um elemento sectioning.

  • Atributos da função ARIA: main (padrão), presentation

Adicionar um atributo role="main" ARIA role a outros elementos destinados a serem usados ​​como conteúdo principal é recomendado para auxiliar os agentes de usuário que não suportam HTML5 e também para fornecer mais contexto para aqueles que o fazem.

O elemento <main> por padrão tem o papel principal e, portanto, não precisa ser fornecido.

Clique aqui para ler a Especificação HTML5 oficial para o elemento <main>

Elemento de seção #

O elemento <section> representa uma seção genérica para agrupar conteúdo tematicamente. Cada seção, normalmente, deve poder ser identificada com um elemento de título como filho da seção.

  • Você pode usar o elemento <section> dentro de um <article> e vice-versa.
  • Cada seção deve ter um tema (um elemento de cabeçalho identificando esta região)
  • Não use o elemento <section> como um ‘contêiner’ de estilo geral. Se você precisar de um contêiner para aplicar estilo, use um <div>.

No exemplo a seguir, estamos exibindo uma única postagem de blog com vários capítulos, cada capítulo é uma seção (um conjunto de conteúdo agrupado por tema, que pode ser identificado pelos elementos de título em cada seção).

<article>
    <header>
        <h2>Blog Post</h2>
    </header>
    <p>An introduction for the post.</p>
    <section>
        <h3>Chapter 1</h3>
        <p>...</p>
    </section>
    <section>
        <h3>Chapter 2</h3>
        <p>...</p>
    </section>
    <section>
        <h3>Comments</h3> ...
    </section>
</article>

Observações:

Os desenvolvedores devem usar o elemento article quando fizer sentido distribuir o conteúdo do elemento.

Clique aqui para ler a Especificação HTML5 oficial para o elemento <main>

Elemento de cabeçalho #

O elemento <header> representa o conteúdo introdutório para seu conteúdo de corte ancestral mais próximo ou elemento raiz de corte. Um <header> normalmente contém um grupo de ajudas introdutórias ou de navegação.

Observação: o elemento de cabeçalho não está seccionando o conteúdo; não introduz uma nova seção.

Exemplos: #

<header>
  <p>Welcome to...</p>
  <h1>Voidwars!</h1>
</header>

Neste exemplo, o [<article>][1] tem um <header>.

<article>
  <header>
    <h1>Flexbox: The definitive guide</h1>
  </header>
  <p>The guide about Flexbox was supposed to be here, but it turned out Wes wasn’t a Flexbox expert either.</p>
</article>

[Recomendação proposta do W3C][2]

[1]: https://www.wikiod.com/pt/html/elementos-de-secao#Elemento do artigo [2]: https://www.w3.org/TR/html51/sections.html#the-header-element

Elemento de rodapé #

O elemento <footer> contém a parte do rodapé da página.

Aqui está um exemplo para o elemento <footer> que contém a tag de parágrafo p.

<footer>
    <p>All rights reserved</p>
</footer>
