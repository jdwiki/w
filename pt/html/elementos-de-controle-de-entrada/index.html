Um componente-chave dos sistemas interativos da Web, as tags de entrada são elementos HTML projetados para receber uma forma específica de entrada dos usuários. Diferentes tipos de elementos de entrada podem regular os dados inseridos para se adequarem a um formato especificado e fornecer segurança à entrada de senha.

Sintaxe #

  • <tipo de entrada="" nome="" valor="">

Parâmetros #

Parâmetro Detalhes
aula Indica a classe do input
identificação Indica o ID da entrada
tipo Identifica o tipo de controle de entrada a ser exibido. Os valores aceitáveis ​​são hidden, text, tel, url, email, password, date, time, number, range, color, checkbox , radio, file, submit, image, reset e button. O padrão é text se não for especificado, se o valor for inválido ou se o navegador não suportar o tipo especificado.
nome Indica o nome da entrada
desativado Valor booleano que indica que a entrada deve ser desabilitada. Os controles desabilitados não podem ser editados, não são enviados no envio do formulário e não podem receber foco.
verificado Quando o valor do atributo type for radio ou checkbox, a presença deste atributo booleano indica que o controle está selecionado por padrão; caso contrário, é ignorado.
múltiplos HTML5 Indica que vários arquivos ou valores podem ser passados ​​(Aplica-se apenas a entradas do tipo file e email)
marcador de posição HTML5 Uma dica para o usuário sobre o que pode ser inserido no controle . O texto do espaço reservado não deve conter retornos de carro ou feeds de linha
autocompletar HTML5 Indica se o valor do controle pode ser preenchido automaticamente pelo navegador.
somente leitura Valor booleano que indica que a entrada não é editável. Os controles somente leitura ainda são enviados no envio do formulário, mas não recebem foco. HTML5: Este atributo é ignorado quando o valor do atributo type é definido como hidden, range, color, checkbox, radio, file ou button.
necessário HTML5 Indica que um valor deve estar presente ou o elemento deve ser verificado para que o formulário seja enviado
alt Um texto alternativo para imagens, caso não sejam exibidas.
foco automático O elemento <input> deve obter o foco quando a página for carregada.
valor Especifica o valor do elemento <input>.
passo O atributo step especifica os intervalos de números legais. Ele funciona com os seguintes tipos de entrada: number, range, date, datetime-local, month, time e week.

Assim como outros elementos void do HTML5, <input> é de fechamento automático e pode ser escrito como <input />. HTML5 não requer essa barra.

Os seguintes são tipos de entrada válidos em HTML:

  • botão
  • caixa de seleção
  • file
  • hidden
  • image
  • password
  • [radio](https://www.wikiod.com/pt/html/elementos-de-controle-de-entrada#Caixa de seleção e botões de opção)
  • reset
  • submit
  • text (valor padrão)

A seguir estão os tipos de entrada recém-introduzidos como parte do padrão HTML 5. Alguns desses tipos não são suportados por todos os navegadores da web. No caso em que um tipo não é suportado, o elemento de entrada assumirá como padrão o tipo text.

  • color
  • data
  • datetime (obsoleto e obsoleto)
  • datetime-local
  • email
  • mês
  • number
  • intervalo
  • pesquisar
  • tel
  • time
  • url
  • semana

Para verificar quais navegadores suportam quais tipos, você pode acessar caniuse.com.

Caixa de seleção e botões de opção #

Visão geral #

Caixas de seleção e botões de opção são escritos com a tag HTML <input>, e seu comportamento é definido na especificação HTML.

A caixa de seleção ou botão de opção mais simples é um elemento <input> com um atributo type de checkbox ou radio, respectivamente:

<input type="checkbox">
<input type="radio">

Um único elemento de caixa de seleção autônomo é usado para uma única opção binária, como uma pergunta sim ou não. As caixas de seleção são independentes, o que significa que o usuário pode selecionar quantas opções quiser em um grupo de caixas de seleção. Em outras palavras, marcar uma caixa de seleção não desmarca as outras caixas de seleção no grupo de caixas de seleção.

Os botões de opção geralmente vêm em grupos (se não estiverem agrupados com outro botão de opção, você provavelmente quis usar uma caixa de seleção) identificados usando o mesmo atributo name em todos os botões desse grupo. A seleção de botões de opção é mutuamente exclusiva, o que significa que o usuário pode selecionar apenas uma opção de um grupo de botões de opção. Quando um botão de opção é marcado, qualquer outro botão de opção com o mesmo nome que foi marcado anteriormente fica desmarcado.

Exemplo:

<input type="radio" name="color" id="red" value="#F00">
<input type="radio" name="color" id="green" value="#0F0">
<input type="radio" name="color" id="blue" value="#00F">

Quando visualizados, os botões de opção aparecem como um círculo (desmarcado) ou um círculo preenchido (marcado). As caixas de seleção aparecem como um quadrado (desmarcado) ou um quadrado preenchido (marcado). Dependendo do navegador e do sistema operacional, o quadrado às vezes tem cantos arredondados.

Atributos #

caixas de seleção e botões de opção têm vários atributos para controlar seu comportamento:

valor #

Como qualquer outro elemento de entrada, o atributo value especifica o valor da string a ser associado ao botão no caso de envio de formulário. No entanto, caixas de seleção e botões de opção são especiais, pois quando o valor é omitido, o padrão é ‘on’ quando enviado, em vez de enviar um valor em branco. O atributo value não é refletido na aparência do botão.

verificado #

O atributo checked especifica o estado inicial de uma caixa de seleção ou botão de opção. Este é um atributo booleano e pode ser omitido.

Cada uma delas são formas válidas e equivalentes de definir um botão de opção marcado:

<input checked>
<input checked="">
<input checked="checked">
<input checked="ChEcKeD">

A ausência do atributo checked é a única sintaxe válida para um botão desmarcado:

<input type="radio">
<input type="checkbox">

Ao redefinir um <form>, caixas de seleção e botões de opção revertem para o estado de seu atributo checked.

Acessibilidade #

Etiquetas #

Para dar contexto aos botões e mostrar aos usuários para que serve cada botão, cada um deles deve ter um rótulo. Isso pode ser feito usando um elemento <label> para envolver o botão. Além disso, isso torna o rótulo clicável, então você seleciona o botão correspondente.

Exemplo:

<label>
  <input type="radio" name="color" value="#F00">
  Red
</label>

ou com um elemento <label> com um atributo for definido para o atributo id do botão:

<input type="checkbox" name="color" value="#F00" id="red">
<label for="red">Red</label>

Grupos de botões #

Como cada botão de opção afeta os outros no grupo, é comum fornecer um rótulo ou contexto para todo o grupo de botões de opção.

Para fornecer um rótulo para todo o grupo, os botões de opção devem ser incluídos em um elemento <fieldset> com um elemento <legend> dentro dele.

Exemplo:

<fieldset>
  <legend>Theme color:</legend>
  <p>
    <input type="radio" name="color" id="red" value="#F00">
    <label for="red">Red</label>
  </p>
  <p>
    <input type="radio" name="color" id="green" value="#0F0">
    <label for="green">Green</label>
  </p>
  <p>
    <input type="radio" name="color" id="blue" value="#00F">
    <label for="blue">Blue</label>
  </p>
</fieldset>

As caixas de seleção também podem ser agrupadas de maneira semelhante, com um conjunto de campos e uma legenda identificando o grupo de caixas de seleção relacionadas. No entanto, lembre-se de que as caixas de seleção não devem compartilhar o mesmo nome porque não são mutuamente exclusivas. Isso fará com que o formulário envie vários valores para a mesma chave e nem todos os idiomas do lado do servidor lidam com isso da mesma maneira (comportamento indefinido). Cada caixa de seleção deve ter um nome exclusivo ou usar um conjunto de colchetes ([]) para indicar que o formulário deve enviar uma matriz de valores para essa chave. O método escolhido deve depender de como você planeja lidar com os dados do formulário do lado do cliente ou do lado do servidor. Você também deve manter a legenda curta, pois algumas combinações de navegadores e leitores de tela lêem a legenda antes de cada campo de entrada no conjunto de campos.

Validação de entrada #

A validação de entrada HTML é feita automaticamente pelo navegador com base em atributos especiais no elemento de entrada. Ele pode substituir parcial ou completamente a validação de entrada JavaScript. Esse tipo de validação pode ser contornado pelo usuário por meio de solicitações HTTP especialmente criadas, portanto, não substitui a validação de entrada do lado do servidor. A validação ocorre apenas ao tentar enviar o formulário, portanto, todas as entradas restritas devem estar dentro de um formulário para que a validação ocorra (a menos que você esteja usando JavaScript). Lembre-se de que as entradas desabilitadas ou somente leitura não acionarão a validação.

Alguns tipos de entrada mais recentes (como email, url, tel, date e muitos mais) são validados automaticamente e não requerem suas próprias restrições de validação.

Requeridos #

Use o atributo required para indicar que um campo deve ser preenchido para passar na validação.

<input required>

Comprimento mínimo/máximo #

Use os atributos minlength e maxlength para indicar os requisitos de comprimento. A maioria dos navegadores impedirá que o usuário digite mais de max caracteres na caixa, impedindo-o de tornar sua entrada inválida mesmo antes de tentar o envio.

<input minlength="3">
<input maxlength="15">
<input minlength="3" maxlength="15">

Especificando um intervalo #

Use os atributos min e max para restringir o intervalo de números que um usuário pode inserir em uma entrada do tipo number ou range

Marks: <input type="number" size="6" name="marks" min="0" max="100" />
Subject Feedback: <input type="range" size="2" name="feedback" min="1" max="5" />

Corresponder a um padrão #

Para obter mais controle, use o atributo pattern para especificar qualquer expressão regular que deve ser correspondida para passar na validação. Você também pode especificar um título, que é incluído na mensagem de validação se o campo não for aprovado.

<input pattern="\d*" title="Numbers only, please.">

Aqui está a mensagem mostrada no Google Chrome versão 51 ao tentar enviar o formulário com um valor inválido dentro deste campo:

Nem todos os navegadores exibem uma mensagem para padrões inválidos, embora haja suporte total entre os navegadores modernos mais usados.

Verifique o suporte mais recente em CanIUse e implemente de acordo.

Aceitar tipo de arquivo #

Para campos de entrada do tipo file, é possível aceitar apenas determinados tipos de arquivos, como vídeos, imagens, áudios, extensões de arquivo específicas ou determinados tipos de mídia. Por exemplo:

<input type="file" accept="image/*" title="Only images are allowed">

Vários valores podem ser especificados com uma vírgula, por exemplo:

<input type="file" accept="image/*,.rar,application/zip">

Observação: Adicionar o atributo novalidate ao elemento form ou o atributo formnovalidate ao botão de envio impede a validação nos elementos do formulário. Por exemplo:

<form>
    <input type="text" name="name" required>
    <input type="email" name="email" required>
    <input pattern="\d*" name="number" required>

    <input type="submit" value="Publish"> <!-- form will be validated -->
    <input type="submit" value="Save" formnovalidate> <!-- form will NOT be validated -->
</form>

O formulário tem campos obrigatórios para “publicar” o rascunho, mas não para “salvar” o rascunho.

Texto #

O tipo de entrada mais básico e a entrada padrão se nenhum type for especificado. Esse tipo de entrada define um campo de texto de linha única com quebras de linha removidas automaticamente do valor de entrada. Todos os outros caracteres podem ser inseridos neste. Elementos <input> são usados ​​dentro de um elemento <form> para declarar controles de entrada que permitem aos usuários inserir dados.

Sintaxe

<input type="text">

ou (sem especificar um type, usando o atributo default):

<input>

A largura padrão de uma entrada de campo de texto é de 20 caracteres. Isso pode ser alterado especificando um valor para o atributo size assim:

<input type="text" size="50">

O atributo size é distintamente diferente de definir uma largura com CSS. Usar uma largura define um valor específico (em número de pixels, porcentagem do elemento pai, etc.) que a entrada deve ser sempre larga. Usar o size calcula a quantidade de largura a ser alocada com base na fonte que está sendo usada e na largura dos caracteres normalmente.

Observação: O uso do atributo size não limita inerentemente o número de caracteres que podem ser inseridos na caixa, apenas a largura da caixa exibida. Para limitar o comprimento, consulte [Validação de entrada](https://www.wikiod.com/pt/html/elementos-de-controle-de-entrada#Validação de entrada).

Um campo de entrada permite apenas uma linha de texto. Se você precisar de uma entrada de texto de várias linhas para uma quantidade substancial de texto, use um elemento <textarea>.

Cor #

<input type="color" name="favcolor" value="#ff0000">

Em navegadores compatíveis, o elemento de entrada com um atributo de tipo cujo valor é color cria um controle semelhante a um botão, com uma cor igual ao valor do atributo color (o padrão é preto se o valor não for especificado ou for um hexadecimal inválido formato).

Clicar neste botão abre o widget de cores do sistema operacional, que permite ao usuário selecionar uma cor.

Fallback para navegadores que não suportam este tipo de entrada é um input type=text normal.

Senha #

<input type="password" name="password">

O elemento input com um atributo type cujo valor é password cria um campo de texto de linha única semelhante ao input type=text, exceto que o texto não é exibido quando o usuário o insere.

<input type="password" name="password" placeholder="Password">

O texto do espaço reservado é mostrado em texto simples e é substituído automaticamente quando um usuário começa a digitar.

Observação: alguns navegadores e sistemas modificam o comportamento padrão do campo de senha para exibir também o caractere digitado mais recentemente por um curto período, assim:

Arquivo #

<input type="file" name="fileSubmission">

As entradas de arquivo permitem que os usuários selecionem um arquivo de seu sistema de arquivos local para uso com a página atual. Se usados ​​em conjunto com um elemento form, eles podem ser usados ​​para permitir que os usuários façam upload de arquivos para um servidor (para mais informações, consulte Uploading Files).

O exemplo a seguir permite que os usuários usem a entrada file para selecionar um arquivo de seu sistema de arquivos e enviar esse arquivo para um script no servidor chamado upload_file.php.

<form action="upload_file.php" method="post" enctype="multipart/form-data">
    Select file to upload:
    <input type="file" name="fileSubmission" id="fileSubmission">
    <input type="submit" value="Upload your file" name="submit">
</form>

Vários arquivos

Adicionando o atributo multiple o usuário poderá selecionar mais de um arquivo:

<input type="file" name="fileSubmission" id="fileSubmission" multiple>

Aceitar arquivos

O atributo Accept especifica os tipos de arquivos que o usuário pode selecionar. Por exemplo. .png, .gif, .jpeg.

<input type="file" name="fileSubmission" accept="image/x-png,image/gif,image/jpeg" />

1: https://www.wikiod.com/pt/html/formularios#Fazendo upload de arquivos

Enviar #

<input type="submit" value="Submit">

Uma entrada de envio cria um botão que envia o formulário em que está quando clicado.

Você também pode usar o elemento <button> se precisar de um botão de envio que possa ser estilizado mais facilmente ou conter outros elementos:

<button type="submit">
  <img src="submit-icon.jpg" /> Submit
</button>

Redefinir #

<input type="reset" value="Reset">

Uma entrada do tipo reset cria um botão que, quando clicado, redefine todas as entradas no formulário em que está contida para seu estado padrão.

  • O texto em um campo de entrada será redefinido para branco ou seu valor padrão (especificado usando o atributo value).
  • Quaisquer opções em um menu de seleção serão desmarcadas, a menos que tenham o atributo selected.
  • Todas as caixas de seleção e caixas de rádio serão desmarcadas, a menos que tenham o atributo checked.

Nota: Um botão de reset deve estar dentro ou anexado (através do atributo form) a um elemento <form> para ter algum efeito. O botão só irá redefinir os elementos dentro deste formulário.

Botão #

<input type="button" value="Button Text">

Os botões podem ser usados ​​para acionar ações que ocorrem na página, sem enviar o formulário. Você também pode usar o elemento <button> se precisar de um botão que possa ser estilizado mais facilmente ou conter outros elementos:

<button type="button">Button Text</button>

Os botões são normalmente usados ​​com um evento “onclick”:

<input type="button" onclick="alert('hello world!')" value="Click Me">

ou

<button type="button" onclick="alert('hello world!')">Click Me</button>

Atributos #

[nome] #

O nome do botão, que é enviado com os dados do formulário.

[tipo] #

O ’tipo’ do botão.

Os valores possíveis são:

submit : O botão envia os dados do formulário para o servidor. Este é o padrão se o atributo não for especificado ou se o atributo for alterado dinamicamente para um valor vazio ou inválido.

reset : O botão redefine todos os controles para seus valores iniciais.

button : O botão não tem comportamento padrão. Ele pode ter scripts do lado do cliente associados aos eventos do elemento, que são acionados quando os eventos ocorrem.

menu : O botão abre um menu popup definido através de seu elemento

designado.

[valor] #

O valor inicial do botão.

Atributos extras para botões de envio #

Atributo Descrição
formulário Especifica o ID do formulário ao qual o botão pertence.
Se nenhum for especificado, ele pertencerá ao elemento de formulário ancestral (se houver).
formação Especifica para onde enviar os dados do formulário
quando o formulário é enviado usando este botão.
formenctype Especifica como os dados do formulário devem ser codificados
ao enviá-los ao servidor usando este botão.
Só pode ser usado com formmethod="post".
formmethod Especifica o método HTTP a ser usado (POST ou GET)
ao enviar dados de formulário usando este botão.
formnovalidate Especifica que os dados do formulário não devem ser validados no envio.
formtarget Especifica onde exibir a resposta recebida
após enviar o formulário usando este botão.

Escondido #

<input type="hidden" name="inputName" value="inputValue">

Uma entrada oculta não ficará visível para o usuário, mas seu valor será enviado ao servidor quando o formulário for enviado.

Telefone #

<input type="tel" value="+8400000000">

O elemento de entrada com um atributo de tipo cujo valor é tel representa um controle de edição de texto simples de uma linha para inserir um número de telefone.

Número #

<input type="number" value="0" name="quantity">

O elemento Input com um atributo type cujo valor é number representa um controle preciso para definir o valor do elemento para uma string representando um número.

Observe que este campo não garante um número correto. Ele apenas permite todos os símbolos que podem ser usados ​​em qualquer número real, por exemplo, o usuário poderá inserir valores como e1e-,0.

E-mail #

O <input type="email"> é usado para campos de entrada que devem conter um endereço de e-mail.

<form>
  <label>E-mail: <label>
  <input type="email" name="email">
</form>

O endereço de e-mail pode ser validado automaticamente quando enviado, dependendo do suporte do navegador.

Variar #

<input type="range" min="" max="" step="" />

Um controle para inserir um número cujo valor exato não é importante.

Atributo Descrição Valor padrão
min Valor mínimo para intervalo 0
máx. Valor máximo para intervalo 100
passo Valor a ser aumentado em cada incremento. 1

Imagem #

<input type="image" src="img.png" alt="image_name" height="50px" width="50px"/>

Uma imagem. Você deve usar o atributo src para definir a origem da imagem e o atributo alt para definir o texto alternativo. Você pode usar os atributos de altura e largura para definir o tamanho da imagem em pixels.

Semana #

<input type="week" />

Dependendo do suporte do navegador, um controle será exibido para inserir um número de ano-semana e um número de semana sem fuso horário.

Procurar #

A pesquisa de tipo de entrada é usada para pesquisa textual. Ele adicionará o símbolo da lupa ao lado do espaço para texto na maioria dos navegadores

<input type="search" name="googlesearch">

URL #

<input type="url" name="Homepage">

Isso é usado para campos de entrada que devem conter um endereço de URL.

Dependendo do suporte do navegador, o campo url pode ser validado automaticamente quando enviado.

Alguns smartphones reconhecem o tipo url e adicionam “.com” ao teclado para corresponder à entrada de URL.

DataHora-Local #

<input type="datetime-local" />

Dependendo do suporte do navegador, um seletor de data e hora aparecerá na tela para você escolher uma data e hora.

Mês #

<input type="month" />

Dependendo do suporte do navegador, um controle aparecerá para escolher o mês.

Tempo #

<input type="time" />

A entrada time marca este elemento como aceitando uma string representando uma hora. O formato é definido em RFC 3339 e deve ser em tempo parcial, como

19:04:39
08:20:39.04

Atualmente, todas as versões do Edge, Chrome, Opera e Chrome para Android suportam type=“time”. As versões mais recentes do navegador Android, especificamente 4.4 e superior, suportam-no. O Safari para iOS oferece suporte parcial, não suportando os atributos min, max e step.

Encontro #

<input type="date" />

Um seletor de data irá aparecer na tela para você escolher uma data. Isso não é suportado no Firefox ou Internet Explorer.

DataHora (Global) #

O elemento de entrada com um atributo de tipo cujo valor é “datetime” representa um controle para definir o valor do elemento para uma string que representa uma data e hora globais (com informações de fuso horário).

<fieldset>
   <p><label>Meeting time: <input type=datetime name="meeting.start"></label>
</fieldset>

Atributos permitidos:

  • atributos globais
  • nome
  • Desativado
  • Formato
  • modelo
  • autocompletar
  • auto-foco
  • Lista
  • mínimo máximo
  • passo (flutuador)
  • somente leitura
  • valor necessário
