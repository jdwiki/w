Localize um texto após um elemento em BeautifulSoup #

Imagine que você tenha o seguinte HTML:

<div>
    <label>Name:</label>
    John Smith
</div>

E você precisa localizar o texto “John Smith” após o elemento label.

Neste caso, você pode localizar o elemento label por texto e então usar a propriedade .next_sibling:

from bs4 import BeautifulSoup

data = """
<div>
    <label>Name:</label>
    John Smith
</div>
"""

soup = BeautifulSoup(data, "html.parser")

label = soup.find("label", text="Name:")
print(label.next_sibling.strip())

Imprime John Smith.

Localizando comentários #

Para localizar comentários em BeautifulSoup, use o argumento text (ou string nas versões recentes) verificando se o tipo é Comment:

from bs4 import BeautifulSoup
from bs4 import Comment

data = """
<html>
    <body>
        <div>
        <!-- desired text -->
        </div>
    </body>
</html>
"""

soup = BeautifulSoup(data, "html.parser")
comment = soup.find(text=lambda text: isinstance(text, Comment))
print(comment)

Imprime o ’texto desejado'.

Funções de filtro #

BeautifulSoup permite filtrar resultados fornecendo uma função para find_all e funções semelhantes. Isso pode ser útil para filtros complexos, bem como uma ferramenta para reutilização de código.

Uso básico #

Defina uma função que recebe um elemento como seu único argumento. A função deve retornar True se o argumento corresponder.

def has_href(tag):
    '''Returns True for tags with a href attribute'''
    return  bool(tag.get("href"))

soup.find_all(has_href) #find all elements with a href attribute
#equivilent using lambda:
soup.find_all(lambda tag: bool(tag.get("href")))

Fornecendo argumentos adicionais para filtrar funções #

Como a função passada para find_all pode receber apenas um argumento, às vezes é útil criar ‘fábricas de funções’ que produzam funções adequadas para uso em find_all. Isso é útil para tornar suas funções de localização de tags mais flexíveis.

def present_in_href(check_string):
    return lambda tag: tag.get("href") and check_string in tag.get("href")

soup.find_all(present_in_href("/partial/path"))

Usando seletores CSS para localizar elementos no BeautifulSoup #

BeautifulSoup tem um suporte limitado para seletores CSS, mas cobre os mais usados. Use o método select() para encontrar vários elementos e select_one() para encontrar um único elemento.

Exemplo básico:

from bs4 import BeautifulSoup

data = """
<ul>
    <li class="item">item1</li>
    <li class="item">item2</li>
    <li class="item">item3</li>
</ul>
"""

soup = BeautifulSoup(data, "html.parser")

for item in soup.select("li.item"):
    print(item.get_text())

Impressões:

item1
item2
item3

Acessando tags internas e seus atributos da tag inicialmente selecionada #

Vamos supor que você tenha um html depois de selecionar com soup.find('div', class_='base class'):

from bs4 import BeautifulSoup

soup = BeautifulSoup(SomePage, 'lxml')
html = soup.find('div', class_='base class')
print(html)

<div class="base class">
  <div>Sample text 1</div>
  <div>Sample text 2</div>
  <div>
    <a class="ordinary link" href="https://example.com">URL text</a>
  </div>
</div>

<div class="Confusing class"></div>
'''

E se você quiser acessar o href da tag <a>, você pode fazer assim:

a_tag = html.a
link = a_tag['href']
print(link)

https://example.com

Isso é útil quando você não pode selecionar diretamente a tag <a> porque é attrs não fornece identificação exclusiva, existem outras tags “gêmeas” <a> na página analisada. Mas você pode selecionar exclusivamente uma tag pai que contém o <a> necessário.

Coletando elementos opcionais e/ou seus atributos de uma série de páginas #

Vamos considerar a situação quando você analisa o número de páginas e deseja coletar o valor do elemento que é opcional (pode ser apresentado em uma página e pode estar ausente em outra) para uma página específica.

Além disso, o próprio elemento, por exemplo, é o elemento mais comum na página, em outras palavras, nenhum atributo específico pode localizá-lo com exclusividade. Mas você vê que pode selecionar corretamente seu elemento pai e sabe o número do pedido do elemento desejado no respectivo nível de aninhamento.

from bs4 import BeautifulSoup

soup = BeautifulSoup(SomePage, 'lxml')
html = soup.find('div', class_='base class') # Below it refers to html_1 and html_2

O elemento desejado é opcional, então pode haver 2 situações para que html seja:

html_1 = '''
<div class="base class">    # №0
  <div>Sample text 1</div>  # №1
  <div>Sample text 2</div>  # №2  
  <div>!Needed text!</div>  # №3
</div>

<div>Confusing div text</div>  # №4
'''
        
html_2 = '''
<div class="base class">    # №0
  <div>Sample text 1</div>  # №1
  <div>Sample text 2</div>  # №2  
</div>

<div>Confusing div text</div>  # №4
'''

Se você tem html_1 você pode coletar !Texto necessário! da tag №3 desta forma:

wanted tag = html_1.div.find_next_sibling().find_next_sibling() # this gives you whole tag №3

Ele inicialmente recebe №1 div, então 2 vezes muda para o próximo div no mesmo nível de aninhamento para chegar ao №3.

wanted_text = wanted_tag.text # extracting !Needed text!

A utilidade desta abordagem vem quando você obtém html_2 - a abordagem não lhe dará erro, ela dará None:

print(html_2.div.find_next_sibling().find_next_sibling())
None

Usar find_next_sibling() aqui é crucial porque limita a busca de elementos pelo respectivo nível de aninhamento. Se você usar find_next(), a tag №4 será coletada e você não a deseja:

print(html_2.div.find_next().find_next())
<div>Confusing div text</div>

Você também pode explorar find_previous_sibling() e find_previous() que funcionam de maneira oposta.

Todas as funções descritas têm suas variantes miltiple para capturar todas as tags, não apenas a primeira:

find_next_siblings()
find_previous_siblings()
find_all_next()
find_all_previous()
