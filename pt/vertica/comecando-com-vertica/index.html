Instalação ou configuração #

Configuração do sistema operacional (Linux-CentOS x64 ou RedHat x64)

1-Aumente o espaço de troca para um mínimo de 2 Gb

2-Requisitos de espaço e CPU:

-Vertica requer pelo menos 1 GB por CPU.

-a utilização de disco por nó não deve exceder sessenta por cento (60%). O espaço em disco é temporariamente exigido por determinados operadores de execução de consulta, como junções e ordenações de hash, no caso de precisarem transbordar para o disco.

-configure TEMP SPACE separado do espaço em disco de dados.

3-Instale o pré-requisito para o Vertica Cluster

yum install  rsync  python*  telnet ruby*  java* sudo  openssh-server openssh-clients
chkconfig sshd on
service sshd start

4-Edite o arquivo /etc/pam.d/su

vi  /etc/pam.d/su
#add the line
session required pam_limits.so

5-Verifique se o NTP Daemon está em execução

chkconfig --list ntpd
#if is not on use the commands
chkconfig ntpd on
#start ntp service
/etc/init.d/ntpd start

6-Remover aplicativos não essenciais

For optimal performance, Vertica is designed to use all available resources on each host machine. Vertica recommends that you remove or disable all non-essential applications from cluster hosts.

7-Configurando a Rede

Altere o arquivo /etc/hosts. Verifique se o arquivo existe e se contém o endereço de loopback 127.0.0.1

7.1-Configurando hosts de cluster

Certifique-se de que o arquivo /etc/hosts inclua todos os hosts que fazem parte do cluster. Por exemplo, se os hosts forem nomeados como host01, host02, host03 e host04, o arquivo /etc/hosts em cada host terá esta aparência:

# Do not remove the following line, or various programs
# that require network functionality will fail.
127.0.0.1               localhost.localdomain localhost
192.xxx.13.128          host01
192.xxx.13.129          host02
192.xxx.13.130          host03
192.xxx.13.131          host04

Isso deve ser feito em todos os hosts (nós)

7.2- Edite o arquivo /etc/sysconfig/network :

vim /etc/ sysconfig/network
Alter the hostname and set it to the desired name :
HOSTNAME=host01

7.3 Configurando a variável de ambiente HOSTNAME

vim /etc/profile or /etc/bashrc
Add the line
export HOSTNAME=hostname

7.4-Verifique se a resolução do nome do host funciona corretamente Verifique isso com o comando

/bin/hostname
Hostname

Reinicie os hosts (nós) Certifique-se de executar todas essas etapas em todos os hosts (nós) como usuário root.

7.5 Desative o firewall

Firewalls não recomendados para hosts de banco de dados SELinux (Linux com Segurança Aprimorada) Iptables

8 -Baixe e instale o software Vertica no Master Node (onde você executará os trabalhos administrativos do cluster)

Como raiz:

#rpm -ihv  vertica-<version>.x86_64.RHEL5.rpm
After entering the command, a progress indicator appears:
Preparing...   ##################################### [100%]
1:vertica      ##################################### [100%]
Vertica 6.0.xx successfully installed on host hostname.

Normalmente, por padrão, o vertica será instalado no diretório /opt/vertica.

8.1- Execute o script de instalação

No nó mestre, execute o seguinte comando (o nó mestre seria o nó que você mais acessa para trabalho administrativo)

/opt/vertica/sbin/install_vertica -s host_list -r rpm_package -u dba_username

Onde estão as opções: -s host_list lista separada por vírgulas de nomes de host ou endereços IP a serem incluídos no cluster; não inclua caracteres de espaço na lista. Exemplo :

-s host01,host02,host03

ou

-s 192.xxx.233.101,192.xxx.233.102,192.xxx.233.103

-r rpm_package O nome do caminho do pacote Vertica RPM.

Exemplo:

-r “vertica_6.0.x.x86_64.RHEL5.rpm” -u dba_username -este será o nome do usuário como será executado o admintools (somente)

  • Se você omitir o parâmetro, o nome da conta de administrador de banco de dados padrão será dbadmin.

Exemplo de comando completo para cluster de 3 nós :

# /opt/vertica/sbin/install_vertica -s 10.xxx.1.216,10.xxx.1.224,10.xxx.1.225 -r
 /home/user/Downloads/vertica-6.0.0-3.x86_64.RHEL5.rpm -u dbadmin

9 – Crie os diretórios Vertica Data e Catalog em cada nó:

Directory names are totally up to you. Remember that database user must have owner rights over them.

Nota: isso se refere a diretórios extras adicionados após a instalação

mkdir /vertica_db/data
mkdir /vertica_db/data
chown  dbadmin:dbadmin  vertica_db/

10- Adicione o /opt/vertica/bin ao seu caminho de usuário dbadmin para que você possa acessar as ferramentas vertica sem o caminho completo.

11- Acesse o adminTool e você será solicitado a introduzir o arquivo licence.dat fornecido pela Vertica.(isso se aplica apenas para Enterprise Edition).

12- Crie o banco de dados e escolha em quais nós ele irá residir !! Siga as próximas etapas para ver como criar um banco de dados que residirá no Vertica Cluster .
