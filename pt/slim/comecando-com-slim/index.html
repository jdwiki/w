Instalação ou configuração #

A maneira recomendada de instalar o framework Slim é usando o composer.

  1. Crie um diretório vazio

Este diretório conterá todos os arquivos necessários para que nosso aplicativo Slim seja executado. Chamamos esse diretório de diretório raiz para que possamos endereçar todos os outros arquivos e diretórios de aplicativos relativos ao diretório raiz.

mkdir slim-app

  1. Instale o framework Slim e seus requisitos usando o compositor

    cd slim-app composer require slim/slim “^3.0”

De agora em diante, assumimos que este é nosso diretório de trabalho.

Depois que o compositor terminar de baixar os arquivos necessários, deverá haver dois arquivos composer.json e composer.lock e um diretório chamado vendor que inclui os arquivos baixados pelo compositor. Agora estamos prontos para criar nosso aplicativo. Para organizar nossa aplicação, criamos outro diretório:

mkdir public

Chamamos isso de diretório público e vamos dizer ao nosso servidor web para servir nosso aplicativo a partir desse diretório.

  1. Exemplo de aplicativo “Hello World”

Para usar o Slim, crie um index.php no diretório público com o seguinte código:

public/index.php

<?php

include "../vendor/autoload.php";

$app = new \Slim\App();

$app->get('/', function ($request, $response, $args) {
    $response->getBody()->write("Hello world!");
});

$app->run();
  1. Inicie o servidor PHP embutido

Agora podemos usar o servidor PHP embutido para servir nosso aplicativo:

php -S localhost:8080 -t public

e execute o projeto abrindo este endereço em um navegador da web:

http://localhost:8080

Resultado

Olá mundo!

Agora configure o servidor web para que todas as requisições sejam passadas para este arquivo:

Configuração do Apache para URLs limpos (opcional)

Isso não é necessário, mas recomendado para projetos slim para remover index.php na URL da API.

Crie .htaccess na mesma pasta onde seu index.php está localizado. O arquivo deve conter o seguinte código:

RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^ index.php [QSA,L]

Certifique-se de que seu host virtual Apache esteja configurado com a opção AllowOverride para que as regras de reescrita declaradas .htaccess possam realmente ser usadas:

AllowOverride All

Introdução ao Slim Framework #

Instalação ou configuração da estrutura Slim

  • Instalar o Compositor
  • Abra cmd
  • Vá para o diretório raiz da pasta do projeto e execute o seguinte comando.

compositor requer slim/slim “^3.0”

Agora você terá o diretório do fornecedor em seu projeto

Próximo Crie Index.php na pasta raiz e adicione o seguinte código

<?php
use \Psr\Http\Message\ServerRequestInterface as Request;
use \Psr\Http\Message\ResponseInterface as Response;

require 'vendor/autoload.php';

$app = new \Slim\App;
$app->get('/hello/{name}', function (Request $request, Response $response) {
    $name = $request->getAttribute('name');
    $response->getBody()->write("Hello, $name");

    return $response;
});
$app->run();

Em seguida, execute o projeto no host local e tente com o seguinte comando

http://localhost/project-root/index.php/hello/any-thing

Resultado

Olá qualquer coisa

Obter dados Json da tabela de banco de dados (API REST) #

use \Psr\Http\Message\ServerRequestInterface as Request;
use \Psr\Http\Message\ResponseInterface as Response;

require 'vendor/autoload.php';

$app = new \Slim\App;

$app->get('/employee/view', function ($req, $res) {

    $con = new mysqli('localhost','USERNAME','PASSWORD','DATABASE');

    $query = $con->query("SELECT * FROM employee"); 
    while ($row = $query->fetch_assoc()) {
        $data[] = $row;
    }
    return $res->withJson($data); 
});

$app->run();

Exemplo de Olá Mundo #

<?php

include "vendor/autoload.php";

$app = new \Slim\App();

$app->get('/hello', function () {
    echo "Hello, world";
});

$app->run();

Projeto de esqueleto de andaime com composer e @akrabats Slim-Skeleton #

  1. Dependendo se você instalou o composer globalmente ou localmente.

    • Locally: php composer.phar create-project slim/slim-skeleton {my-app-name}
    • Globally: composer create-project slim/slim-skeleton {my-app-name}

  2. Se você estiver executando um servidor web (por exemplo, Apache ou nginx), aponte sua raiz de documento do virtualhost para sua nova pasta pública de aplicativos, por exemplo. {my-app-name}/public.

  3. Certifique-se de que logs/ seja gravável pela configuração do seu servidor web.

Você também pode executar todo o shebang no modo dev com:

  • Localmente: php composer.phar start
  • Globalmente: composer start
