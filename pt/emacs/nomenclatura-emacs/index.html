Elementos da interface do usuário #

A interface de usuário do Emacs usa termos que foram cunhados anteriormente e podem ser perturbadores para usuários acostumados a uma terminologia mais moderna.

Quadro #

No Emacs, o que também é chamado de janela (a área da tela usada por um programa) é chamado de quadro. O Emacs começa usando um frame, embora frames adicionais possam ser criados usando C-x 5.

Janela #

Um frame contém uma ou mais janelas (também chamadas de painéis), cada uma mostrando o conteúdo de um buffer. Cada quadro geralmente começa com apenas uma janela, mas janelas adicionais podem ser criadas dividindo as existentes; horizontalmente usando C-x 2 ou verticalmente com C-x 3. Consulte também https://www.wikiod.com/pt/emacs/teclas-basicas#Múltiplas janelas ou molduras

Amortecedor #

O termo buffer refere-se ao conteúdo exibido em uma janela. Esse conteúdo pode refletir o conteúdo de um arquivo no sistema de arquivos (ou talvez uma versão atualizada que ainda não foi salva no disco), mas geralmente pode ser qualquer tipo de texto.

linha de modo #

Na parte inferior de cada janela há uma linha de modo, que descreve sinteticamente o buffer exibido na janela.

Barra de ferramentas #

De forma semelhante a muitos outros softwares, uma barra de ferramentas pode ser exibida na parte superior de cada quadro. Seu conteúdo pode variar dependendo do tipo de buffer que está sendo editado no momento.

Minibuffer #

Um minibuffer, geralmente exibido na parte inferior de cada frame, permite interagir com o Emacs. Cada vez que um comando solicita a entrada do usuário, ele é solicitado no minibuffer. Por outro lado, as mensagens exibidas para o usuário ver são impressas lá.

Ponto, marca e região #

O Emacs usa os termos ponto, marca e região para fornecer mais precisão sobre o texto selecionado e a posição do cursor. Ao entender esses termos, ele o ajudará a entender e usar outras operações e funções.

O ponto é o local em um buffer onde a edição (i.e. inserção) está ocorrendo no momento e geralmente é indicado por um cursor. A marca é um marcador colocado em qualquer lugar no buffer usando comandos como set-mark-command (C-SPC) ou exchange-point-and-mark ( C-x C-x). A região é a área entre o ponto e a marca, e muitos comandos operam na região para por exemplo excluir, verificar a ortografia, recuar ou compilá-la.

Quando você clica com o mouse em um local em um buffer, está vendo o ponto. Ao selecionar o texto, você está definindo a região (o texto selecionado) e a marca (no início de sua seleção).

Arquivos e buffers #

No Emacs, arquivo tem o mesmo significado que no sistema operacional e é usado para armazenamento permanente de dados. Um buffer é a representação interna de um arquivo que está sendo editado. Arquivos podem ser lidos em buffers usando C-x C-f, e buffers podem ser gravados em arquivos usando C-x C-s (salvar arquivo em seu local atual) ou C-x C-w</ kbd> (gravar o arquivo em um local diferente, solicitando-o - o equivalente a Salvar como).

Matando e puxando #

Matar e arrancar correspondem mais ou menos ao que se costuma chamar de “cortar” e “colar”.

Matando #

killing significa deletar texto e copiá-lo para o kill-ring (que pode ser visto como uma espécie de “área de transferência” na terminologia “cortar e colar”). O kill ring é assim chamado porque armazena vários pedaços de texto killed, que podem ser acessados ​​posteriormente em ordem cíclica.

Existem vários comandos que matam uma palavra (M-d), o resto da linha (C-k), ou blocos de texto maiores (como a região selecionada atualmente: C-w).

Existem outros comandos que salvam texto no kill ring, sem realmente matá-lo (de maneira similar a “copiar” nas terminologias modernas). Por exemplo, M-w, que atua na região selecionada no momento.

Arranque #

As entradas no kill ring podem mais tarde ser puxadas de volta para um buffer. Normalmente, pode-se arrancar o texto eliminado mais recentemente usando por exemplo C-y (que é semelhante à operação “colar” em uma terminologia mais moderna). Mas outros comandos podem acessar e arrancar entradas mais antigas do kill ring.

Modos #

Modo principal #

O Emacs pode adaptar seu comportamento ao tipo específico de texto editado em um buffer. O conjunto de personalizações específicas do Emacs para um determinado tipo de texto é chamado de “modo principal”. Cada buffer tem exatamente um modo principal dependendo de seu tipo de conteúdo.

Modos principais podem alterar o significado de algumas teclas, definir realce de sintaxe ou regras de recuo e instalar novas combinações de teclas (geralmente começando com C-c) para comandos específicos de modo. O Emacs é fornecido com uma ampla variedade de modos principais, divididos em três categorias principais:

  • suporte para texto (por exemplo, linguagens de marcação),
  • suporte para linguagens de programação,
  • aplicativos dentro do emacs (por exemplo, dired, gnus, …). Buffers que usam este último grupo de modos principais geralmente não são associados a arquivos, mas servem como uma interface de usuário.

Modo menor #

Modos menores são recursos opcionais que podem ser ativados e desativados. Modos secundários podem ser habilitados para buffers específicos (modos de buffer local) ou todos os buffers (modos globais). Ao contrário dos modos principais, qualquer número de modos secundários pode ser ativado para um determinado buffer.

O Emacs fornece muitos modos menores. Alguns exemplos incluem:

  • Modo de preenchimento automático para quebrar automaticamente as linhas de texto enquanto você digita.
  • Modo Flyspell para destacar erros de ortografia enquanto você digita.
  • Modo de linha visual para envolver linhas longas para caber na tela.
  • Modo de marca transitória para destacar a região atual.
