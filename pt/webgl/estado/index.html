Texturas #

As unidades de textura são o estado global. Se eles fossem implementados em JavaScript, seriam algo assim

// pseudo code
gl = {
  activeTextureUnit: 0,
  textureUnits: [
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
  ],
};

Você pode escolher qual unidade indexar com gl.activeTexture.

// pseudo code
gl.activeTexture = function(textureUnit) {
  gl.activeTextureUnit = textureUnit - gl.TEXTURE0;
};

Chamar gl.bindTexture liga uma textura à unidade de textura ativa como esta

// pseudo code
gl.bindTexture = function(target, texture) {
  var textureUnit = gl.textureUnits[gl.activeTextureUnit];
  textureUnit[target] = texture;
}

Quando você tem um programa de sombreamento que usa texturas, você precisa dizer a esse programa de sombreamento a quais unidades de textura você vinculou as texturas. Por exemplo, se você tiver um shader como este

uniform sampler2D diffuse;
uniform sampler2D normalMap;
uniform samplerCube environmentMap;

...

Pois você precisa consultar os locais uniformes

var diffuseUniformLocation = gl.getUniformLocation(someProgram, "diffuse");
var normalMapUniformLocation = gl.getUniformLocation(someProgram, "normalMap");
var environmmentMapUniformLocation = gl.getUniformLocation(someProgram,
                                                           "environmentMap");

Então, depois de fazer seu programa de sombreamento o programa atual

gl.useProgram(someProgram);

Você então precisa dizer ao sombreador em quais unidades de textura você colocou/vai colocar as texturas. Por exemplo

var diffuseTextureUnit = 3;
var normalMapTextureUnit = 5;
var environmentMapTextureUnit = 2;

gl.uniform1i(diffuseUniformLocation, diffuseTextureUnit);
gl.uniform1i(normalMapUniformLocation, normalMapTextureUnit);
gl.uniform1i(environmentMapUniformLocation, environmentMapTextureUnit);

Agora você disse ao shader quais unidades você usou/usou. Quais unidades de textura você decide usar depende inteiramente de você.

Para realmente vincular texturas a unidades de textura, você faria algo assim

gl.activeTexture(gl.TEXTURE0 + diffuseTextureUnit);
gl.bindTexture(gl.TEXTURE_2D, diffuseTexture);
gl.activeTexture(gl.TEXTURE0 + normalMapTextureUnit);
gl.bindTexture(gl.TEXTURE_2D, normalMapTexture);
gl.activeTexture(gl.TEXTURE0 + environmentMapTextureUnit);
gl.bindTexture(gl.TEXTURE_CUBE_MAP, environmentMapTexture);

Para exemplos WebGL muito simples que usam apenas 1 textura, é comum nunca chamar gl.activeTexture já que o padrão é a unidade de textura #0. Também é comum não chamar gl.uniform1i porque os uniformes são padronizados para 0, então o programa shader, por padrão, usará a unidade de textura #0 para todas as texturas.

Todas as outras funções de textura também funcionam com os alvos ativos de textura e unidade de textura. Por exemplo gl.texImage2D pode ser algo assim

gl.texImage2D = function(target, level, internalFormat, width, height, 
                         border, format, type, data) {
   var textureUnit = gl.textureUnits[gl.activeTextureUnit];
   var texture = textureUnit[target];

   // Now that we've looked up the texture form the activeTextureUnit and
   // the target we can effect a specific texture
   ...
};

Atributos #

Os atributos são estado global (*). Se eles fossem implementados em JavaScript, seriam algo assim

 // pseudo code
 gl = {
   ARRAY_BUFFER: null,
   vertexArray: {
     attributes: [
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
     ],
     ELEMENT_ARRAY_BUFFER: null,
   },
 }

Como você pode ver acima, existem atributos 8 e eles são de estado global.

Quando você chama gl.enableVertexAttribArray(location) ou gl.disableVertexAttribArray você pode pensar assim

// pseudo code
gl.enableVertexAttribArray = function(location) {
  gl.vertexArray.attributes[location].enable = true;
};

gl.disableVertexAttribArray = function(location) {
  gl.vertexArray.attributes[location].enable = false;
};

Em outras palavras, location se refere diretamente ao índice de um atributo.

Da mesma forma, gl.vertexAttribPointer seria implementado algo assim

// pseudo code
gl.vertexAttribPointer = function(location, size, type, normalize, stride, offset) {
  var attrib = gl.vertexArray.attributes[location];
  attrib.size = size;
  attrib.type = type;
  attrib.normalize = normalize;
  attrib.stride = stride ? stride : sizeof(type) * size;
  attrib.offset = offset;
  attrib.buffer = gl.ARRAY_BUFFER;  // !!!! <-----
};

Observe que o attrib.buffer está definido para o que o gl.ARRAY_BUFFER atual estiver definido. gl.ARRAY_BUFFER é definido chamando gl.bindBuffer(gl.ARRAY_BUFFER, someBuffer).

Então, em seguida, temos shaders de vértice. No vertex shader você declara atributos. Exemplo

attribute vec4 position;
attribute vec2 texcoord;
attribute vec3 normal;

...

void main() {
  ...
}

Quando você vincula um vertex shader com um fragment shader chamando gl.linkProgram(someProgram) WebGL (o driver/GPU/navegador) decide por conta própria qual índice/local usar para cada atributo. Você não tem ideia de quais eles vão escolher. É o navegador/driver/GPU. Então, você deve perguntar qual atributo você usou para position, texcoord e normal?. Você faz isso chamando gl.getAttribLocation

var positionLoc = gl.getAttribLocation(program, "position");
var texcoordLoc = gl.getAttribLocation(program, "texcoord");
var normalLoc = gl.getAttribLocation(program, "normal");

Digamos positionLoc = 5. Isso significa que quando o vertex shader é executado (quando você chama gl.drawArrays ou gl.drawElements) o vertex shader espera que você tenha o atributo de configuração 5 com os corretos type, size, offset, stride , buffer etc.

Observe que ANTES de vincular o programa, você pode escolher os locais chamando gl.bindAttribLoction(program, location, nameOfAttribute). Exemplo:

// Tell `gl.linkProgram` to assign `position` to use attribute #7
gl.bindAttribLocation(program, 7, "position");

estado de atributo completo #

O que falta na descrição acima é que cada atributo também tem um valor padrão. Ele é deixado de fora acima porque é incomum usá-lo.

attributes: [
   { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ?
     value: [0, 0, 0, 1], },
   { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ?
     value: [0, 0, 0, 1], },
   ..

Você pode definir o valor com as várias funções gl.vertexAttribXXX. O value é usado quando enable é false. Quando enable for true, os dados para o atributo são extraídos do buffer atribuído.

Objetos de matriz de vértices #

WebGL tem uma extensão, OES_vertex_array_object

No diagrama acima OES_vertex_array_object permite criar e substituir o vertexArray. Em outras palavras

var vao = ext.createVertexArrayOES();

cria o objeto que você vê anexado a gl.vertexArray no pseudocódigo acima. Chamando ext.bindVertexArrayOES(vao) atribua seu objeto array de vértices criado como o array de vértices atual.

// pseudo code
ext.bindVertexArrayOES = function(vao) {
  gl.vertexArray = vao;
}

Isso permite que você defina todos os atributos e ELEMENT_ARRAY_BUFFER no VAO atual para que quando você quiser desenhar seja uma chamada para ext.bindVertexArrayOES onde como sem a extensão seria até uma chamada para ambos gl.bindBuffer gl.vertexAttribPointer (e possivelmente gl.enableVertexAttribArray) por atributo.

Uniformes #

Os uniformes são por programa estado. Cada programa de sombreamento tem seu próprio estado uniforme e seus próprios locais.
