marcação HTML #

exemplo de entrada:

<html>
  <head>
    <title>the title</title>
    <link href='css/app.css' type='text/css' rel='stylesheet'>
    <script src='js/app.js'></script>
  </head>
  <body>
    <h1>header</h1>
    <div>
      <p>hello!</p>
    </div>
  </body>
</html>

Começando #

O DOM (Document Object Model) é a interface de programação para documentos HTML e XML, ele define a estrutura lógica dos documentos e a forma como um documento é acessado e manipulado.

Os principais implementadores da API DOM são os navegadores da web. As especificações são padronizadas pelos grupos W3C e WHATWG, e o modelo de objeto especifica a lógica modelo para a interface de programação.

A representação da estrutura DOM se assemelha a uma visão em árvore, onde cada nó é um objeto que representa uma parte da marcação, dependendo do tipo, cada elemento também herda funcionalidades específicas e compartilhadas.

O nome “Document Object Model” foi escolhido porque é um “modelo de objeto” no sentido tradicional de design orientado a objetos: documentos são modelados usando objetos, e o modelo engloba não apenas a estrutura de um documento, mas também o comportamento de um documento e os objetos que o compõem. Em outras palavras, tomando o exemplo de diagrama HTML, os nós não representam uma estrutura de dados, eles representam objetos, que têm funções e identidade. Como um modelo de objeto, o Document Object Model identifica:

  • as interfaces e objetos usados ​​para representar e manipular um documento
  • semântica dessas interfaces e objetos - incluindo comportamento e atributos
  • os relacionamentos e colaborações entre essas interfaces e objetos

Recuperando elementos html existentes #

Uma das tarefas mais comuns é recuperar um elemento existente do DOM para manipular. Mais comumente, esses métodos são executados em document, porque é o nó raiz, mas todos esses métodos funcionam em qualquer elemento HTML na árvore. Eles só retornarão filhos do nó em que são executados.

Recuperar por id #

var element = document.getElementById("logo");

element conterá o (apenas) elemento que tem seu atributo id definido como “logo”, ou contém null se tal elemento não existir. Se existirem vários elementos com este id, o documento é inválido e tudo pode acontecer.

Recuperar pelo nome da tag #

var elements = document.getElementsByTagName("a");

elements conterá um live HTMLCollection (um objeto tipo array) de todas as tags de link no documento. Esta coleção está em sincronia com o DOM, portanto, todas as alterações feitas no DOM são refletidas nesta coleção. A coleção fornece acesso aleatório e tem um comprimento.

var element = elements[0];
//Alternative
element = elements.item(0);

element contém o primeiro elemento de link HTML encontrado, ou null se o índice estiver fora dos limites

var length = elements.length;

length é igual ao número de elementos de link HTML atualmente na lista. Este número pode mudar quando o DOM é alterado.

Recuperar por classe #

var elements = document.getElementsByClassName("recipe");

elements conterá um live HTMLCollection (um objeto tipo array) de todos os elementos onde seu atributo class inclui “receita”. Esta coleção está em sincronia com o DOM, portanto, todas as alterações feitas no DOM são refletidas nesta coleção. A coleção fornece acesso aleatório e tem um comprimento.

var element = elements[0];
//Alternative
element = elements.item(0);

element contém o primeiro elemento HTML encontrado com esta classe. Se não houver tais elementos, element terá o valor undefined no primeiro exemplo e null no segundo exemplo.

var length = elements.length;

length é igual ao número de elementos HTML que atualmente possuem a classe “receita”. Este número pode mudar quando o DOM é alterado.

Recuperar por nome #

var elements = document.getElementsByName("zipcode");

elements conterá um live NodeList (um objeto tipo array) de todos os elementos com seu atributo name definido como “zipcode”. Esta coleção está em sincronia com o DOM, portanto, todas as alterações feitas no DOM são refletidas nesta coleção. A coleção fornece acesso aleatório e tem um comprimento.

var element = elements[0];
//Alternative
element = elements.item(0);

element contém o primeiro elemento HTML encontrado com este nome.

var length = elements.length;

length é igual ao número de elementos HTML que atualmente possuem “zipcode” como atributo de nome. Este número pode mudar quando o DOM é alterado.
