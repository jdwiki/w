Parâmetros #

Parâmetro Descrição
tipo String define o nome do evento a ser ouvido.
ouvinte Function é acionado quando o evento ocorre.
opções Boolean para definir a captura, se Object você pode definir as seguintes propriedades nele, observe que a opção de objeto é fracamente suportada.
1. capturar Um booleano que indica que eventos desse tipo serão despachados para o ouvinte registrado antes de serem despachados para qualquer EventTarget abaixo dele na árvore DOM.
2. uma vez Um booleano indicando que o ouvinte deve ser invocado no máximo uma vez após ser adicionado. Se for verdadeiro, o ouvinte será removido automaticamente quando for invocado.
3. passivo Um booleano indicando que o ouvinte nunca chamará preventDefault(). Se isso acontecer, o agente do usuário deve ignorá-lo e gerar um aviso de console.

*

Origem dos eventos

Os eventos não começam na coisa em que você aciona o evento (um botão, por exemplo).

Em vez disso

Ele toca cada elemento em seu caminho e informa a cada elemento que um evento está acontecendo. Os eventos também retornam após chegarem ao seu destino, informando novamente os elementos de sua ocorrência.

*

Capturando e borbulhando

Como aprendemos, os eventos começam no topo da árvore DOM, informam cada nó em seu caminho até seu destino, depois voltam para cima quando atingem seu destino, informando também cada elemento que toca em seu caminho sobre sua ocorrência.

Os eventos que descem na árvore DOM estão na fase de captura, os eventos que sobem na árvore DOM estão na fase de borbulhamento.

Por padrão, os eventos são ouvidos na fase de borbulhamento. Para alterar isso, você pode especificar em qual fase o evento será ouvido, especificando o terceiro parâmetro na função addEventListener. (exemplo de código na seção capture)

Evento borbulhando e capturando #

Eventos disparados em elementos DOM não afetam apenas o elemento que estão segmentando. Qualquer um dos ancestrais do alvo no DOM também pode ter a chance de reagir ao evento. Considere o seguinte documento:

<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8" />
</head>
<body>
  <p id="paragraph">
    <span id="text">Hello World</span>
  </p>
</body>
</html>

Se apenas adicionarmos ouvintes a cada elemento sem nenhuma opção, acionaremos um clique no intervalo…

document.body.addEventListener('click', function(event) {
  console.log("Body clicked!");
});
window.paragraph.addEventListener('click', function(event) {
  console.log("Paragraph clicked!");
});
window.text.addEventListener('click', function(event) {
  console.log("Text clicked!");
});

window.text.click();

…então o evento borbulha em cada ancestral, acionando cada manipulador de clique no caminho:

Text clicked!
Paragraph clicked!
Body clicked!

Se você quiser que um de seus manipuladores pare o evento de acionar mais manipuladores, ele pode chamar o método event.stopPropagation(). Por exemplo, se substituirmos nosso segundo manipulador de eventos por este:

window.paragraph.addEventListener('click', function(event) {
  console.log("Paragraph clicked, and that's it!");
  event.stopPropagation();
});

Veríamos a seguinte saída, com o manipulador click do body nunca acionado:

Text clicked!
Paragraph clicked, and that's it!

Por fim, temos a opção de adicionar ouvintes de eventos que são acionados durante a “captura” em vez de borbulhar. Antes de um evento borbulhar de um elemento por meio de seus ancestrais, ele é primeiro “capturado” até o elemento por meio de seus ancestrais. Um listener de captura é adicionado especificando true ou {capture: true} como o terceiro argumento opcional para addEventListener. Se adicionarmos os seguintes ouvintes ao nosso primeiro exemplo acima:

document.body.addEventListener('click', function(event) {
  console.log("Body click captured!");
}, true);
window.paragraph.addEventListener('click', function(event) {
  console.log("Paragraph click captured!");
}, true);
window.text.addEventListener('click', function(event) {
  console.log("Text click captured!");
}, true);

Teremos a seguinte saída:

Body click captured!
Paragraph click captured!
Text click captured!
Text clicked!
Paragraph clicked!
Body clicked!

Por padrão, os eventos são ouvidos na fase de borbulhamento. Para alterar isso, você pode especificar em qual fase o evento será ouvido, especificando o terceiro parâmetro na função addEventListener. (Para aprender sobre captura e borbulhamento, marque comentários)

element.addEventListener(eventName, eventHandler, useCapture)

useCapture: true significa ouvir o evento quando ele estiver descendo a árvore DOM. false significa ouvir o evento enquanto ele sobe na árvore DOM.

window.addEventListener("click", function(){alert('1: on bubble')}, false);
window.addEventListener("click", function(){alert('2: on capture')}, true);

As caixas de alerta aparecerão nesta ordem:

  • 2: na captura
  • 1: na bolha

Casos de uso do mundo real

O evento de captura será despachado antes do evento de bolha, portanto, você pode garantir que um evento seja ouvido primeiro se você o ouvir em sua fase de captura.

se você estiver ouvindo um evento de clique em um elemento pai e outro em seu filho, poderá ouvir primeiro o filho ou o pai primeiro, dependendo de como alterar o parâmetro useCapture.

no borbulhamento, o evento filho é chamado primeiro, na captura, o pai primeiro

HTML:

<div id="parent">
   <div id="child"></div>
</div>

Javascript:

child.addEventListener('click', function(e) {
   alert('child clicked!');
});

parent.addEventListener('click', function(e) {
   alert('parent clicked!');
}, true);

Definir true para o eventListener pai acionará o ouvinte pai primeiro.

Combinado com e.stopPropagation(), você pode impedir que o evento acione o ouvinte do evento filho / ou o pai. (mais sobre isso no próximo exemplo)

Introdução #

Definição:

Na computação, um evento é uma ação ou ocorrência reconhecida por software que pode ser manipulado pelo software. Os eventos de computador podem ser gerado ou acionado pelo sistema, pelo usuário ou de outras formas. Fonte da Definição

Eventos HTML são “coisas” que acontecem com elementos HTML. JavaScript pode “reagir” a esses eventos. via Ouvintes de eventos. Além disso, eventos personalizados podem ser acionados usando dispatchEvent. Mas esta é apenas uma introdução, então vamos começar!

Ouvinte de evento básico

Para ouvir eventos, você chama target.addEventListener(type, listener);

function loadImage() {
  console.log('image code here!');
}
var myButton = document.querySelector('#my-button');
myButton.addEventListener('click', loadImage);

Isso acionará loadImage toda vez que my-button for clicado.

Os ouvintes de eventos podem ser anexados a qualquer nó na árvore DOM. para ver uma lista completa de todos os eventos acionados nativamente no navegador: acesse aqui link MDN para lista completa de eventos

Objeto de evento #

Para acessar o objeto de evento, inclua um parâmetro event na função de retorno de chamada do ouvinte de eventos:

var foo = document.getElementById("foo");
foo.addEventListener("click", onClick);

function onClick(event) {
  // the `event` parameter is the event object
  // e.g. `event.type` would be "click" in this case
};

e.stopPropagation();

HTML:

<div id="parent">
   <div id="child"></div>
</div>

Javascript:

var parent = document.querySelector('#parent');
var child = document.querySelector('#child');

child.addEventListener('click', function(e) {
   e.stopPropagation();
   alert('child clicked!');
});

parent.addEventListener('click', function(e) {
   alert('parent clicked!');
});

como o filho interrompe a propagação do evento e os eventos são ouvidos durante a fase de borbulhamento, clicar no filho apenas acionará o filho. sem interromper a propagação, ambos os eventos serão acionados.


e.preventDefault();

O método event.preventDefault() impede que a ação padrão de um elemento aconteça.

Por exemplo:

  • Impedir que um botão de envio envie um formulário

  • Impedir que um link siga o URL

    var allAnchorTags = document.querySelector(‘a’);

    allAnchorTags.addEventListener(‘click’, function(e){ e.preventDefault(); console.log(‘anchor tags are useless now! evil laugh’); });

e.target vs e.currentTarget

e.currentTarget Identifica o destino atual para o evento, conforme o evento percorre o DOM. Ele sempre se refere ao elemento que o manipulador de eventos foi anexado a em oposição a event.target que identifica o elemento em em que o evento ocorreu.

em outras palavras

e.target retornará o que aciona o despachador de eventos para acionar

e.currentTarget retornará o que você atribuiu ao seu ouvinte.

HTML:

<body>
   <button id="my-button"></button>
</body>

Javascript:

var body = document.body;
body.addEventListener( 'click', function(e) {
    console.log('e.target', e.target);
    console.log('e.currentTarget', e.currentTarget);
});

se você clicar em meu-botão,

  • e.target será my-button
  • e.currentTarget será body

Removendo ouvintes de eventos #

O método removeEventListener() remove manipuladores de eventos que foram anexados com o método addEventListener():

element.removeEventListener("mousemove", myFunction);

Tudo (nome do evento, função e opções) no removeEventListener deve corresponder ao definido ao adicionar o ouvinte de evento ao elemento.

*

.bind com removeListener

usar .bind na função ao adicionar um ouvinte de evento impedirá que a função seja removida, para remover o eventListener você pode escrever:

function onEvent() {
   console.log(this.name);
}

var bindingOnEvent = onEvent.bind(this);

document.addEventListener('click', bindingOnEvent);

...

document.removeEventListener('click', bindingOnEvent);

ouvir um evento apenas uma vez

Até que a opção once seja amplamente suportada, temos que remover manualmente o ouvinte par assim que o evento for acionado pela primeira vez.

Este pequeno ajudante nos ajudará a conseguir isso:

Object.prototype.listenOnce = Object.prototype.listenOnce ||
  function listenOnce(eventName, eventHandler, options) {
      var target = this;
      target.addEventListener(eventName, function(e) {
          eventHandler(e);
          target.removeEventListener(eventName, eventHandler, options);
      }, options);
  }

var target = document.querySelector('#parent');
target.listenOnce("click", clickFunction, false);

*Não é uma prática recomendada anexar funções ao protótipo Object, portanto, você pode remover a primeira linha desse código e adicionar um destino a ele como primeiro parâmetro.

Acionando eventos personalizados #

A API CustomEvent permite que os desenvolvedores criem eventos personalizados e os acionem em nós DOM, passando dados ao longo do caminho.

event = new CustomEvent(typeArg, customEventInit);

typeArg - DOMString representando o nome do evento.

customEventInit - são parâmetros opcionais (que serão passados ​​como e no exemplo a seguir).

Você pode anexar eventListeners a document ou qualquer elemento HTML.

Uma vez que o evento personalizado foi adicionado e vinculado ao elemento (ou documento), pode-se querer dispará-lo manualmente do javascript.

document.addEventListener("event-name", function(e) {
  console.log(e.detail); // logs custom object passed from the event.
});

var event = new CustomEvent("event-name", { "param-name": "param-value" });
document.dispatchEvent(event);
