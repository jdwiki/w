Matrizes Multidimensionais #

Matrizes Multidimensionais #

Como o nome indica, matrizes multidimensionais são matrizes que contêm mais de uma dimensão, geralmente duas ou três, mas podem ter até 32 dimensões.

Um multi array funciona como uma matriz com vários níveis, tome como exemplo uma comparação entre uma, duas e três dimensões.

One Dimension é sua matriz típica, parece uma lista de elementos.

Dim 1D(3) as Variant

*1D - Visually*
(0)
(1)
(2)

Duas Dimensões se pareceria com um Sudoku Grid ou uma planilha do Excel, ao inicializar o array você definiria quantas linhas e colunas o array teria.

Dim 2D(3,3) as Variant
'this would result in a 3x3 grid 

*2D - Visually*
(0,0) (0,1) (0,2)
(1,0) (1,1) (1,2)
(2,0) (2,1) (2,2)

Três Dimensões começariam a se parecer com o Cubo de Rubik, ao inicializar o array você definiria linhas e colunas e camadas/profundidades que o array teria.

Dim 3D(3,3,2) as Variant
'this would result in a 3x3x3 grid

*3D - Visually*
       1st layer                 2nd layer                  3rd layer
         front                     middle                     back
(0,0,0) (0,0,1) (0,0,2) ¦ (1,0,0) (1,0,1) (1,0,2) ¦ (2,0,0) (2,0,1) (2,0,2)
(0,1,0) (0,1,1) (0,1,2) ¦ (1,1,0) (1,1,1) (1,1,2) ¦ (2,1,0) (2,1,1) (2,1,2)
(0,2,0) (0,2,1) (0,2,2) ¦ (1,2,0) (1,2,1) (1,2,2) ¦ (2,2,0) (2,2,1) (2,2,2)

Dimensões adicionais podem ser pensadas como a multiplicação do 3D, então um 4D(1,3,3,3) seria duas matrizes 3D lado a lado.

Matriz de duas dimensões #

Criando

O exemplo abaixo será uma compilação de uma lista de funcionários, cada funcionário terá um conjunto de informações na lista (Nome, Sobrenome, Endereço, Email, Telefone…), o exemplo estará essencialmente armazenando no array ( funcionário,informações) sendo o (0,0) é o primeiro nome do primeiro funcionário.

Dim Bosses As Variant
'set bosses as Variant, so we can input any data type we want

Bosses = [{"Jonh","Snow","President";"Ygritte","Wild","Vice-President"}]
'initialise a 2D array directly by filling it with information, the redult wil be a array(1,2) size 2x3 = 6 elements

Dim Employees As Variant
'initialize your Employees array as variant
'initialize and ReDim the Employee array so it is a dynamic array instead of a static one, hence treated differently by the VBA Compiler
ReDim Employees(100, 5)
'declaring an 2D array that can store 100 employees with 6 elements of information each, but starts empty
'the array size is 101 x 6 and contains 606 elements

For employee = 0 To UBound(Employees, 1)
'for each employee/row in the array, UBound for 2D arrays, which will get the last element on the array
'needs two parameters 1st the array you which to check and 2nd the dimension, in this case 1 = employee and 2 = information
    For information_e = 0 To UBound(Employees, 2)
    'for each information element/column in the array
        
        Employees(employee, information_e) = InformationNeeded ' InformationNeeded would be the data to fill the array
        'iterating the full array will allow for direct attribution of information into the element coordinates
    Next
Next

Redimensionando

Redimensionar ou ReDim Preserve um Multi-Array como a norma para um array de uma dimensão obteria um erro, em vez disso, as informações precisam ser transferidas para um array temporário com o mesmo tamanho do original mais o número de linhas/colunas para adicionar. No exemplo abaixo veremos como inicializar um Temp Array, transferir as informações do array original, preencher os elementos vazios restantes e substituir o array temp pelo array original.

Dim TempEmp As Variant
'initialise your temp array as variant
ReDim TempEmp(UBound(Employees, 1) + 1, UBound(Employees, 2))
'ReDim/Resize Temp array as a 2D array with size UBound(Employees)+1 = (last element in Employees 1st dimension) + 1,
'the 2nd dimension remains the same as the original array. we effectively add 1 row in the Employee array

'transfer
For emp = LBound(Employees, 1) To UBound(Employees, 1)
    For info = LBound(Employees, 2) To UBound(Employees, 2)
        'to transfer Employees into TempEmp we iterate both arrays and fill TempEmp with the corresponding element value in Employees
        TempEmp(emp, info) = Employees(emp, info)
    
    Next
Next

'fill remaining
'after the transfers the Temp array still has unused elements at the end, being that it was increased
'to fill the remaining elements iterate from the last "row" with values to the last row in the array
'in this case the last row in Temp will be the size of the Employees array rows + 1, as the last row of Employees array is already filled in the TempArray

For emp = UBound(Employees, 1) + 1 To UBound(TempEmp, 1)
    For info = LBound(TempEmp, 2) To UBound(TempEmp, 2)
        
        TempEmp(emp, info) = InformationNeeded & "NewRow"
    
    Next
Next

'erase Employees, attribute Temp array to Employees and erase Temp array
Erase Employees
Employees = TempEmp
Erase TempEmp

Mudando os valores dos elementos

Para alterar/alterar os valores em um determinado elemento pode ser feito simplesmente chamando a coordenada a ser alterada e dando-lhe um novo valor: Employees(0, 0) = "NewValue"

Alternativamente, itere através das condições de uso de coordenadas para combinar os valores correspondentes aos parâmetros necessários:

For emp = 0 To UBound(Employees)
    If Employees(emp, 0) = "Gloria" And Employees(emp, 1) = "Stephan" Then
    'if value found
        Employees(emp, 1) = "Married, Last Name Change"
        Exit For
        'don't iterate through a full array unless necessary
    End If
Next

Leitura

O acesso aos elementos no array pode ser feito com um Nested Loop (iterando cada elemento), Loop e Coordinate (iterando Rows e acessando colunas diretamente), ou acessando diretamente com ambas as coordenadas.

'nested loop, will iterate through all elements
For emp = LBound(Employees, 1) To UBound(Employees, 1)
    For info = LBound(Employees, 2) To UBound(Employees, 2)
        Debug.Print Employees(emp, info)
    Next
Next

'loop and coordinate, iteration through all rows and in each row accessing all columns directly
For emp = LBound(Employees, 1) To UBound(Employees, 1)
    Debug.Print Employees(emp, 0)
    Debug.Print Employees(emp, 1)
    Debug.Print Employees(emp, 2)
    Debug.Print Employees(emp, 3)
    Debug.Print Employees(emp, 4)
    Debug.Print Employees(emp, 5)
Next

'directly accessing element with coordinates
Debug.Print Employees(5, 5)

Lembre-se, é sempre útil manter um mapa de array ao usar arrays multidimensionais, eles podem facilmente se tornar confusos.

Matriz tridimensional #

Para o array 3D, usaremos a mesma premissa do array 2D, com a adição de não apenas armazenar o Funcionário e as Informações, mas também o Edifício em que eles trabalham.

A matriz 3D terá os Funcionários (podem ser pensados ​​como Linhas), as Informações (Colunas) e o Edifício que podem ser pensados ​​como planilhas diferentes em um documento do Excel, elas têm o mesmo tamanho entre elas, mas todas as planilhas têm um conjunto diferente de informações em suas células/elementos. A matriz 3D conterá n número de matrizes 2D.

Criando

Um array 3D precisa de 3 coordenadas para ser inicializado Dim 3Darray(2,5,5) As Variant a primeira coordenada no array será o número de Edifícios/Folhas (diferentes conjuntos de linhas e colunas), a segunda coordenada definirá Linhas e terceiras Colunas. O Dim acima resultará em um array 3D com 108 elementos (3*6*6), efetivamente tendo 3 conjuntos diferentes de arrays 2D.

Dim ThreeDArray As Variant
'initialise your ThreeDArray array as variant
ReDim ThreeDArray(1, 50, 5)
'declaring an 3D array that can store two sets of 51 employees with 6 elements of information each, but starts empty
'the array size is 2 x 51 x 6 and contains 612 elements

For building = 0 To UBound(ThreeDArray, 1)
    'for each building/set in the array
    For employee = 0 To UBound(ThreeDArray, 2)
    'for each employee/row in the array
        For information_e = 0 To UBound(ThreeDArray, 3)
        'for each information element/column in the array
            
            ThreeDArray(building, employee, information_e) = InformationNeeded ' InformationNeeded would be the data to fill the array
        'iterating the full array will allow for direct attribution of information into the element coordinates
        Next
    Next
Next

Redimensionando

Redimensionar um array 3D é semelhante a redimensionar um 2D, primeiro crie um array Temporário com o mesmo tamanho do original adicionando um na coordenada do parâmetro a aumentar, a primeira coordenada aumentará o número de conjuntos no array, o segundo e terceiras coordenadas aumentarão o número de Linhas ou Colunas em cada conjunto.

O exemplo abaixo aumenta o número de Linhas em cada conjunto em um e preenche esses elementos adicionados recentemente com novas informações.

Dim TempEmp As Variant
'initialise your temp array as variant
ReDim TempEmp(UBound(ThreeDArray, 1), UBound(ThreeDArray, 2) + 1, UBound(ThreeDArray, 3))
'ReDim/Resize Temp array as a 3D array with size UBound(ThreeDArray)+1 = (last element in Employees 2nd dimension) + 1,
'the other dimension remains the same as the original array. we effectively add 1 row in the for each set of the 3D array

'transfer
For building = LBound(ThreeDArray, 1) To UBound(ThreeDArray, 1)
    For emp = LBound(ThreeDArray, 2) To UBound(ThreeDArray, 2)
        For info = LBound(ThreeDArray, 3) To UBound(ThreeDArray, 3)
            'to transfer ThreeDArray into TempEmp by iterating all sets in the 3D array and fill TempEmp with the corresponding element value in each set of each row
            TempEmp(building, emp, info) = ThreeDArray(building, emp, info)
        
        Next
    Next
Next

'fill remaining
'to fill the remaining elements we need to iterate from the last "row" with values to the last row in the array in each set, remember that the first empty element is the original array Ubound() plus 1
For building = LBound(TempEmp, 1) To UBound(TempEmp, 1)
    For emp = UBound(ThreeDArray, 2) + 1 To UBound(TempEmp, 2)
        For info = LBound(TempEmp, 3) To UBound(TempEmp, 3)
            
            TempEmp(building, emp, info) = InformationNeeded & "NewRow"
        
        Next
    Next
Next

'erase Employees, attribute Temp array to Employees and erase Temp array
Erase ThreeDArray
ThreeDArray = TempEmp
Erase TempEmp

Mudando os valores e leitura dos elementos

Ler e alterar os elementos no array 3D pode ser feito da mesma forma que fazemos no array 2D, basta ajustar para o nível extra nos loops e coordenadas.

Do
' using Do ... While for early exit
    For building = 0 To UBound(ThreeDArray, 1)
        For emp = 0 To UBound(ThreeDArray, 2)
            If ThreeDArray(building, emp, 0) = "Gloria" And ThreeDArray(building, emp, 1) = "Stephan" Then
            'if value found
                ThreeDArray(building, emp, 1) = "Married, Last Name Change"
                Exit Do
                'don't iterate through all the array unless necessary
            End If
        Next
    Next
Loop While False

'nested loop, will iterate through all elements
For building = LBound(ThreeDArray, 1) To UBound(ThreeDArray, 1)
    For emp = LBound(ThreeDArray, 2) To UBound(ThreeDArray, 2)
        For info = LBound(ThreeDArray, 3) To UBound(ThreeDArray, 3)
            Debug.Print ThreeDArray(building, emp, info)
        Next
    Next
Next

'loop and coordinate, will iterate through all set of rows and ask for the row plus the value we choose for the columns
For building = LBound(ThreeDArray, 1) To UBound(ThreeDArray, 1)
    For emp = LBound(ThreeDArray, 2) To UBound(ThreeDArray, 2)
        Debug.Print ThreeDArray(building, emp, 0)
        Debug.Print ThreeDArray(building, emp, 1)
        Debug.Print ThreeDArray(building, emp, 2)
        Debug.Print ThreeDArray(building, emp, 3)
        Debug.Print ThreeDArray(building, emp, 4)
        Debug.Print ThreeDArray(building, emp, 5)
    Next
Next

'directly accessing element with coordinates
Debug.Print Employees(0, 5, 5)

Matrizes Dinâmicas (Redimensionamento de Matrizes e Manipulação Dinâmica) #

Matrizes dinâmicas #

Adicionar e reduzir variáveis ​​em um array dinamicamente é uma grande vantagem para quando a informação que você está tratando não possui um número definido de variáveis.

Adicionando valores dinamicamente #

Você pode simplesmente redimensionar o Array com a instrução ReDim, isso irá redimensionar o array, mas para reter as informações já armazenadas no array, você precisará da parte Preserve.

No exemplo abaixo, criamos um array e o aumentamos em mais uma variável em cada iteração, preservando os valores que já estão no array.

Dim Dynamic_array As Variant
' first we set Dynamic_array as variant

For n = 1 To 100

    If IsEmpty(Dynamic_array) Then
        'isempty() will check if we need to add the first value to the array or subsequent ones
    
        ReDim Dynamic_array(0)
        'ReDim Dynamic_array(0) will resize the array to one variable only
        Dynamic_array(0) = n

    Else
        ReDim Preserve Dynamic_array(0 To UBound(Dynamic_array) + 1)
        'in the line above we resize the array from variable 0 to the UBound() = last variable, plus one effectivelly increeasing the size of the array by one
        Dynamic_array(UBound(Dynamic_array)) = n
        'attribute a value to the last variable of Dynamic_array
    End If

Next

Removendo valores dinamicamente #

Podemos utilizar a mesma lógica para diminuir o array. No exemplo, o valor “last” será removido do array.

Dim Dynamic_array As Variant
Dynamic_array = Array("first", "middle", "last")
    
ReDim Preserve Dynamic_array(0 To UBound(Dynamic_array) - 1)
' Resize Preserve while dropping the last value

Redefinindo um array e reutilizando dinamicamente #

Também podemos reutilizar os arrays que criamos para não ter muitos na memória, o que tornaria o tempo de execução mais lento. Isso é útil para matrizes de vários tamanhos. Um trecho que você pode usar para reutilizar o array é ReDim o array de volta para (0), atribuir uma variável ao array e aumentar livremente o array novamente.

No trecho abaixo eu construo um array com os valores de 1 a 40, esvazio o array, e recarrego o array com os valores de 40 a 100, tudo isso feito dinamicamente.

Dim Dynamic_array As Variant

For n = 1 To 100

    If IsEmpty(Dynamic_array) Then
        ReDim Dynamic_array(0)
        Dynamic_array(0) = n
    
    ElseIf Dynamic_array(0) = "" Then
        'if first variant is empty ( = "") then give it the value of n
        Dynamic_array(0) = n
    Else
        ReDim Preserve Dynamic_array(0 To UBound(Dynamic_array) + 1)
        Dynamic_array(UBound(Dynamic_array)) = n
    End If
    If n = 40 Then
        ReDim Dynamic_array(0)
        'Resizing the array back to one variable without Preserving,
        'leaving the first value of the array empty
    End If

Next

Matrizes irregulares (matrizes de matrizes) #

Matrizes irregulares NÃO Matrizes multidimensionais #

Arrays de Arrays (Jagged Arrays) não são os mesmos que Multidimensional Arrays se você pensar sobre eles visualmente Multidimensional Arrays se pareceria com Matrizes (Retangular) com um número definido de elementos em suas dimensões (dentro de arrays), enquanto Jagged array seria como um anual calendário com as matrizes internas com um número diferente de elementos, como dias em meses diferentes.

Embora os Jagged Arrays sejam bastante confusos e complicados de usar devido aos seus níveis aninhados e não tenham muita segurança de tipo, eles são muito flexíveis, permitem manipular diferentes tipos de dados com bastante facilidade e não precisam conter elementos vazios.

Criando uma matriz irregular #

No exemplo abaixo vamos inicializar um array irregular contendo dois arrays um para Nomes e outro para Números, e então acessando um elemento de cada

Dim OuterArray() As Variant
Dim Names() As Variant
Dim Numbers() As Variant
'arrays are declared variant so we can access attribute any data type to its elements

Names = Array("Person1", "Person2", "Person3")
Numbers = Array("001", "002", "003")

OuterArray = Array(Names, Numbers)
'Directly giving OuterArray an array containing both Names and Numbers arrays inside

Debug.Print OuterArray(0)(1)
Debug.Print OuterArray(1)(1)
'accessing elements inside the jagged by giving the coordenades of the element

Criando e lendo arrays irregulares dinamicamente #

Também podemos ser mais dinâmicos em nossa aproximação para construir os arrays, imagine que temos uma folha de dados do cliente no Excel e queremos construir um array para gerar os detalhes do cliente.

   Name -   Phone   -  Email  - Customer Number 
Person1 - 153486231 - 1@STACK - 001
Person2 - 153486242 - 2@STACK - 002
Person3 - 153486253 - 3@STACK - 003
Person4 - 153486264 - 4@STACK - 004
Person5 - 153486275 - 5@STACK - 005

Construiremos dinamicamente um array Header e um array Customers, o Header conterá os títulos das colunas e o array Customers conterá as informações de cada cliente/linha como arrays.

Dim Headers As Variant
' headers array with the top section of the customer data sheet
    For c = 1 To 4
        If IsEmpty(Headers) Then
            ReDim Headers(0)
            Headers(0) = Cells(1, c).Value
        Else
            ReDim Preserve Headers(0 To UBound(Headers) + 1)
            Headers(UBound(Headers)) = Cells(1, c).Value
        End If
    Next
    
Dim Customers As Variant
'Customers array will contain arrays of customer values
Dim Customer_Values As Variant
'Customer_Values will be an array of the customer in its elements (Name-Phone-Email-CustNum)
    
    For r = 2 To 6
    'iterate through the customers/rows
        For c = 1 To 4
        'iterate through the values/columns
            
            'build array containing customer values
            If IsEmpty(Customer_Values) Then
                ReDim Customer_Values(0)
                Customer_Values(0) = Cells(r, c).Value
            ElseIf Customer_Values(0) = "" Then
                Customer_Values(0) = Cells(r, c).Value
            Else
                ReDim Preserve Customer_Values(0 To UBound(Customer_Values) + 1)
                Customer_Values(UBound(Customer_Values)) = Cells(r, c).Value
            End If
        Next
        
        'add customer_values array to Customers Array
        If IsEmpty(Customers) Then
            ReDim Customers(0)
            Customers(0) = Customer_Values
        Else
            ReDim Preserve Customers(0 To UBound(Customers) + 1)
            Customers(UBound(Customers)) = Customer_Values
        End If
        
        'reset Custumer_Values to rebuild a new array if needed
        ReDim Customer_Values(0)
    Next

    Dim Main_Array(0 To 1) As Variant
    'main array will contain both the Headers and Customers
    
    Main_Array(0) = Headers
    Main_Array(1) = Customers

Para entender melhor a maneira de construir dinamicamente um array unidimensional, verifique Dynamic Arrays (Array Resizing and Dynamic Handling) na documentação de Arrays.

O Resultado do trecho acima é um Jagged Array com dois arrays um desses arrays com 4 elementos, 2 níveis de recuo, e o outro sendo ele próprio outro Array Jagged contendo 5 arrays de 4 elementos cada e 3 níveis de recuo, veja abaixo a estrutura:

Main_Array(0) - Headers - Array("Name","Phone","Email","Customer Number")
          (1) - Customers(0) - Array("Person1",153486231,"1@STACK",001)
                Customers(1) - Array("Person2",153486242,"2@STACK",002)
                ...
                Customers(4) - Array("Person5",153486275,"5@STACK",005)

Para aceder à informação terá de ter em conta a estrutura do Jagged Array que cria, no exemplo acima pode ver que o Main Array contém um Array de Headers e um Array de Arrays (Customers ), portanto, com diferentes formas de acessar os elementos.

Agora vamos ler as informações do Main Array e imprimir cada uma das informações dos Clientes como Info Type: Info.

For n = 0 To UBound(Main_Array(1))
    'n to iterate from fisrt to last array in Main_Array(1)
    
    For j = 0 To UBound(Main_Array(1)(n))
        'j will iterate from first to last element in each array of Main_Array(1)
        
        Debug.Print Main_Array(0)(j) & ": " & Main_Array(1)(n)(j)
        'print Main_Array(0)(j) which is the header and Main_Array(0)(n)(j) which is the element in the customer array
        'we can call the header with j as the header array has the same structure as the customer array
    Next
Next

LEMBRE-SE de acompanhar a estrutura do seu Jagged Array, no exemplo acima para acessar o Name de um cliente é acessando Main_Array -> Customers -> CustomerNumber -> Name que tem três níveis, para retornar "Person4" você precisará da localização de Customers no Main_Array, então o Location of customer four no array Customers Jagged e, por último, a localização do elemento que você precisa, neste caso Main_Array(1)(3)(0) que é Main_Array(Clientes)(Número do Cliente)(Nome).

Declarando um Array no VBA #

Declarar um array é muito semelhante a declarar uma variável, exceto que você precisa declarar a dimensão do Array logo após seu nome:

Dim myArray(9) As String 'Declaring an array that will contain up to 10 strings

Por padrão, Arrays em VBA são indexados a partir de ZERO, portanto, o número entre parênteses não se refere ao tamanho do array, mas sim ao índice do último elemento

Acessando Elementos #

O acesso a um elemento do Array é feito usando o nome do Array, seguido do índice do elemento, entre parênteses:

myArray(0) = "first element"
myArray(5) = "sixth element"
myArray(9) = "last element"

Indexação de matriz #

Você pode alterar a indexação de Arrays colocando esta linha no topo de um módulo:

Option Base 1

Com esta linha, todos os Arrays declarados no módulo serão indexados de ONE.

Índice específico #

Você também pode declarar cada Array com seu próprio índice usando a palavra-chave To e os limites inferior e superior (= índice):

Dim mySecondArray(1 To 12) As String 'Array of 12 strings indexed from 1 to 12
Dim myThirdArray(13 To 24) As String 'Array of 12 strings indexed from 13 to 24

Declaração dinâmica #

Quando você não sabe o tamanho do seu Array antes de sua declaração, você pode usar a declaração dinâmica e a palavra-chave ReDim:

Dim myDynamicArray() As Strings 'Creates an Array of an unknown number of strings
ReDim myDynamicArray(5) 'This resets the array to 6 elements

Observe que usar a palavra-chave ReDim apagará qualquer conteúdo anterior do seu Array. Para evitar isso, você pode usar a palavra-chave Preserve após ReDim:

Dim myDynamicArray(5) As String
myDynamicArray(0) = "Something I want to keep"

ReDim Preserve myDynamicArray(8) 'Expand the size to up to 9 strings
Debug.Print myDynamicArray(0) ' still prints the element

Uso de Split para criar um array a partir de uma string #

Função Dividida

retorna um array unidimensional baseado em zero contendo um número especificado de substrings.

Sintaxe

Split(expressão [, delimitador [, limite [, comparar]]])

Parte Descrição
expressão Requeridos. Expressão de string contendo substrings e delimitadores. Se expression for uma string de comprimento zero("" ou vbNullString), Split retornará uma matriz vazia que não contém elementos nem dados. Nesse caso, o array retornado terá um LBound de 0 e um UBound de -1.
delimitador Opcional. Caractere de string usado para identificar limites de substring. Se omitido, o caractere de espaço (" “) é considerado o delimitador. Se delimiter for uma seqüência de comprimento zero, uma matriz de elemento único contendo toda a seqüência de expressão será retornada.
limite Opcional. Número de substrings a serem retornadas; -1 indica que todas as substrings são retornadas.
comparar Opcional. Valor numérico que indica o tipo de comparação a ser usado ao avaliar substrings. Consulte a seção Configurações para valores.

Definições

O argumento compare pode ter os seguintes valores:

Constante Valor Descrição
Descrição -1 Executa uma comparação usando a configuração da instrução Option Compare.
vbBinaryCompare 0 Executa uma comparação binária.
vbTextCompare 1 Executa uma comparação textual.
vbDatabaseCompare 2 Apenas Microsoft Access. Executa uma comparação com base nas informações do seu banco de dados.

Exemplo

Neste exemplo é demonstrado como o Split funciona mostrando vários estilos. Os comentários mostrarão o conjunto de resultados para cada uma das diferentes opções de Split realizadas. Finalmente, é demonstrado como fazer um loop sobre o array de strings retornado.

Sub Test
    
    Dim textArray() as String

    textArray = Split("Tech on the Net")
    'Result: {"Tech", "on", "the", "Net"}

    textArray = Split("172.23.56.4", ".")
    'Result: {"172", "23", "56", "4"}

    textArray = Split("A;B;C;D", ";")
    'Result: {"A", "B", "C", "D"}

    textArray = Split("A;B;C;D", ";", 1)
    'Result: {"A;B;C;D"}

    textArray = Split("A;B;C;D", ";", 2)
    'Result: {"A", "B;C;D"}

    textArray = Split("A;B;C;D", ";", 3)
    'Result: {"A", "B", "C;D"}

    textArray = Split("A;B;C;D", ";", 4)
    'Result: {"A", "B", "C", "D"}

    'You can iterate over the created array
    Dim counter As Long

    For counter = LBound(textArray) To UBound(textArray)
        Debug.Print textArray(counter)
    Next
 End Sub

Iterando elementos de um array #

A seguir #

Usar a variável iterator como o número do índice é a maneira mais rápida de iterar os elementos de um array:

Dim items As Variant
items = Array(0, 1, 2, 3)

Dim index As Integer
For index = LBound(items) To UBound(items)
    'assumes value can be implicitly converted to a String:
    Debug.Print items(index) 
Next

Loops aninhados podem ser usados ​​para iterar arrays multidimensionais:

Dim items(0 To 1, 0 To 1) As Integer
items(0, 0) = 0
items(0, 1) = 1
items(1, 0) = 2
items(1, 1) = 3

Dim outer As Integer
Dim inner As Integer
For outer = LBound(items, 1) To UBound(items, 1)
    For inner = LBound(items, 2) To UBound(items, 2)
        'assumes value can be implicitly converted to a String:
        Debug.Print items(outer, inner)
    Next
Next

Para cada…Próximo #

Um loop For Each...Next também pode ser usado para iterar arrays, se o desempenho não importa:

Dim items As Variant
items = Array(0, 1, 2, 3)

Dim item As Variant 'must be variant
For Each item In items
    'assumes value can be implicitly converted to a String:
    Debug.Print item
Next

Um loop For Each irá iterar todas as dimensões de fora para dentro (na mesma ordem em que os elementos são dispostos na memória), então não há necessidade de loops aninhados:

Dim items(0 To 1, 0 To 1) As Integer
items(0, 0) = 0
items(1, 0) = 1
items(0, 1) = 2
items(1, 1) = 3

Dim item As Variant 'must be Variant
For Each item In items
    'assumes value can be implicitly converted to a String:
    Debug.Print item
Next

Observe que os loops For Each são melhor usados ​​para iterar objetos Collection, se o desempenho for importante.
