Copiando matrizes #

Você pode copiar um array VBA em um array do mesmo tipo usando o operador =. As matrizes devem ser do mesmo tipo, caso contrário, o código lançará um erro de compilação “Não é possível atribuir à matriz”.

Dim source(0 to 2) As Long
Dim destinationLong() As Long
Dim destinationDouble() As Double

destinationLong = source     ' copies contents of source into destinationLong
destinationDouble = source   ' does not compile

A matriz de origem pode ser fixa ou dinâmica, mas a matriz de destino deve ser dinâmica. Tentar copiar para uma matriz fixa gerará um erro de compilação “Não é possível atribuir à matriz”. Quaisquer dados preexistentes na matriz de recebimento são perdidos e seus limites e dimensões são alterados para os mesmos da matriz de origem.

Dim source() As Long
ReDim source(0 To 2)

Dim fixed(0 To 2) As Long
Dim dynamic() As Long

fixed = source   ' does not compile
dynamic = source ' does compile

Dim dynamic2() As Long
ReDim dynamic2(0 to 6, 3 to 99) 

dynamic2 = source ' dynamic2 now has dimension (0 to 2)

Uma vez que a cópia é feita, as duas matrizes são separadas na memória, ou seja, as duas variáveis ​​não são referências aos mesmos dados subjacentes, portanto, as alterações feitas em uma matriz não aparecem na outra.

Dim source(0 To 2) As Long
Dim destination() As Long

source(0) = 3
source(1) = 1
source(2) = 4

destination = source
destination(0) = 2

Debug.Print source(0); source(1); source(2)                ' outputs: 3 1 4
Debug.Print destination(0); destination(1); destination(2) ' outputs: 2 1 4

Copiando Matrizes de Objetos #

Com arrays de objetos, as referências a esses objetos são copiadas, não os próprios objetos. Se uma alteração for feita em um objeto em uma matriz, ele também parecerá ter sido alterado na outra matriz - ambos estão referenciando o mesmo objeto. No entanto, definir um elemento para um objeto diferente em uma matriz não o definirá para esse objeto na outra matriz.

Dim source(0 To 2) As Range
Dim destination() As Range

Set source(0) = Range("A1"): source(0).Value = 3
Set source(1) = Range("A2"): source(1).Value = 1
Set source(2) = Range("A3"): source(2).Value = 4

destination = source

Set destination(0) = Range("A4")   'reference changed in destination but not source

destination(0).Value = 2           'affects an object only in destination
destination(1).Value = 5           'affects an object in both source and destination

Debug.Print source(0); source(1); source(2)                  ' outputs 3 5 4
Debug.Print destination(0); destination(1); destination(2)   ' outputs 2 5 4

Variantes contendo uma matriz #

Você também pode copiar uma matriz de e para uma variável variante. Ao copiar de uma variante, ela deve conter uma matriz do mesmo tipo que a matriz receptora, caso contrário, ele gerará um erro de tempo de execução “Tipo incompatível”.

Dim var As Variant
Dim source(0 To 2) As Range
Dim destination() As Range

var = source
destination = var

var = 5
destination = var  ' throws runtime error

Retornando Arrays de Funções #

Uma função em um módulo normal (mas não um módulo Class) pode retornar um array colocando () após o tipo de dados.

Function arrayOfPiDigits() As Long()
    Dim outputArray(0 To 2) As Long
    
    outputArray(0) = 3
    outputArray(1) = 1
    outputArray(2) = 4

    arrayOfPiDigits = outputArray
End Function

O resultado da função pode então ser colocado em uma matriz dinâmica do mesmo tipo ou em uma variante. Os elementos também podem ser acessados ​​diretamente usando um segundo conjunto de colchetes, no entanto, isso chamará a função a cada vez, portanto, é melhor armazenar os resultados em um novo array se você planeja usá-los mais de uma vez

Sub arrayExample()

    Dim destination() As Long
    Dim var As Variant
    
    destination = arrayOfPiDigits()
    var = arrayOfPiDigits
    
    Debug.Print destination(0)          ' outputs 3
    Debug.Print var(1)                  ' outputs 1
    Debug.Print arrayOfPiDigits()(2)    ' outputs 4
    
End Sub

Observe que o que é retornado é na verdade uma cópia do array dentro da função, não uma referência. Portanto, se a função retornar o conteúdo de uma matriz estática, seus dados não poderão ser alterados pelo procedimento de chamada.

Saída de um Array através de um argumento de saída #

Normalmente, é uma boa prática de codificação que os argumentos de um procedimento sejam entradas e saídas por meio do valor de retorno. No entanto, as limitações do VBA às vezes tornam necessário que um procedimento produza dados por meio de um argumento ByRef.

Saída para um array fixo #

Sub threePiDigits(ByRef destination() As Long)
    destination(0) = 3
    destination(1) = 1
    destination(2) = 4
End Sub

Sub printPiDigits()
    Dim digits(0 To 2) As Long
    
    threePiDigits digits
    Debug.Print digits(0); digits(1); digits(2) ' outputs 3 1 4
End Sub

Saída de um Array de um método de classe #

Um argumento de saída também pode ser usado para gerar uma matriz de um método/procedimento em um módulo de classe

' Class Module 'MathConstants'
Sub threePiDigits(ByRef destination() As Long)
    ReDim destination(0 To 2)
    
    destination(0) = 3
    destination(1) = 1
    destination(2) = 4
End Sub

' Standard Code Module
Sub printPiDigits()
    Dim digits() As Long
    Dim mathConsts As New MathConstants
    
    mathConsts.threePiDigits digits
    Debug.Print digits(0); digits(1); digits(2) ' outputs 3 1 4
End Sub

Passando Arrays para Procedimentos #

Arrays podem ser passados ​​para procedimentos colocando () após o nome da variável de array.

Function countElements(ByRef arr() As Double) As Long
    countElements = UBound(arr) - LBound(arr) + 1
End Function

Arrays devem ser passados ​​por referência. Se nenhum mecanismo de passagem for especificado, por exemplo. myFunction(arr()), então o VBA assumirá ByRef por padrão, no entanto, é uma boa prática de codificação torná-lo explícito. Tentando passar uma matriz por valor, por exemplo. myFunction(ByVal arr()) resultará em um erro de compilação “array argument must be ByRef” (ou um erro de compilação “Syntax error” se Auto Syntax Check não estiver marcado nas opções do VBE).

Passar por referência significa que quaisquer alterações no array serão preservadas no procedimento de chamada.

Sub testArrayPassing()
    Dim source(0 To 1) As Long
    source(0) = 3
    source(1) = 1
    
    Debug.Print doubleAndSum(source)  ' outputs 8
    Debug.Print source(0); source(1)  ' outputs 6 2
End Sub

Function doubleAndSum(ByRef arr() As Long)
    arr(0) = arr(0) * 2
    arr(1) = arr(1) * 2
    doubleAndSum = arr(0) + arr(1)
End Function

Se você quiser evitar alterar o array original, tome cuidado para escrever a função para que ela não altere nenhum elemento.

Function doubleAndSum(ByRef arr() As Long)
    doubleAndSum = arr(0) * 2 + arr(1) * 2
End Function

Alternativamente, crie uma cópia de trabalho do array e trabalhe com a cópia.

Function doubleAndSum(ByRef arr() As Long)
    Dim copyOfArr() As Long
    copyOfArr = arr
    
    copyOfArr(0) = copyOfArr(0) * 2
    copyOfArr(1) = copyOfArr(1) * 2
    
    doubleAndSum = copyOfArr(0) + copyOfArr(1)
End Function
