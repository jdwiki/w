Os exemplos mostrados neste tópico usam associação antecipada para maior clareza e exigem uma referência à biblioteca x.x do Microsoft ActiveX Data Object. Eles podem ser convertidos em ligação tardia substituindo as referências fortemente tipadas por Object e substituindo a criação de objetos usando New por CreateObject quando apropriado.

Fazendo uma conexão com uma fonte de dados #

O primeiro passo para acessar uma fonte de dados via ADO é criar um objeto ADO Connection. Isso normalmente é feito usando uma string de conexão para especificar os parâmetros da fonte de dados, embora também seja possível abrir uma conexão DSN passando o DSN, o ID do usuário e a senha para o método .Open.

Observe que um DSN não é necessário para se conectar a uma fonte de dados via ADO - qualquer fonte de dados que tenha um provedor ODBC pode ser conectada com a cadeia de conexão apropriada. Embora strings de conexão específicas para diferentes provedores estejam fora do escopo deste tópico, ConnectionStrings.com é uma excelente referência para encontrar a string apropriada para seu provedor.

Const SomeDSN As String = "DSN=SomeDSN;Uid=UserName;Pwd=MyPassword;"

Public Sub Example()
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    If Not database Is Nothing Then
        '... Do work.
        database.Close          'Make sure to close all database connections.
    End If
End Sub

Public Function OpenDatabaseConnection(ConnString As String) As ADODB.Connection
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = New ADODB.Connection
    
    With database
        .ConnectionString = ConnString
        .ConnectionTimeout = 10             'Value is given in seconds.
        .Open
    End With
     
    OpenDatabaseConnection = database
     
    Exit Function
Handler:
    Debug.Print "Database connection failed. Check your connection string."
End Function

Observe que a senha do banco de dados está incluída na string de conexão no exemplo acima apenas para fins de clareza. As melhores práticas ditariam não armazenar senhas de banco de dados em código. Isso pode ser feito obtendo a senha por meio da entrada do usuário ou usando a autenticação do Windows.

Criando comandos parametrizados #

Sempre que o SQL executado por meio de uma conexão ADO precisar conter a entrada do usuário, é considerado uma prática recomendada parametrizá-lo para minimizar a chance de injeção de SQL. Esse método também é mais legível do que concatenações longas e facilita um código mais robusto e de fácil manutenção (ou seja, usando uma função que retorna uma matriz de Parameter).

Na sintaxe ODBC padrão, os parâmetros recebem ? “espaços reservados” no texto da consulta e, em seguida, os parâmetros são anexados ao Command na mesma ordem em que aparecem na consulta.

Observe que o exemplo abaixo usa a função OpenDatabaseConnection de [Fazendo uma conexão com uma fonte de dados](https://www.wikiod.com/pt/vba/trabalhando-com-ado#Fazendo uma conexão com uma fonte de dados -to-a-data-source) para brevidade.

Public Sub UpdateTheFoos()
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    
    If Not database Is Nothing Then
        Dim update As ADODB.Command
        Set update = New ADODB.Command
        'Build the command to pass to the data source.
        With update
            .ActiveConnection = database
            .CommandText = "UPDATE Table SET Foo = ? WHERE Bar = ?"
            .CommandType = adCmdText
            
            'Create the parameters.
            Dim fooValue As ADODB.Parameter
            Set fooValue = .CreateParameter("FooValue", adNumeric, adParamInput)
            fooValue.Value = 42
            
            Dim condition As ADODB.Parameter
            Set condition = .CreateParameter("Condition", adBSTR, adParamInput)
            condition.Value = "Bar"
            
            'Add the parameters to the Command
            .Parameters.Append fooValue
            .Parameters.Append condition
            .Execute
        End With
    End If
CleanExit:
    If Not database Is Nothing And database.State = adStateOpen Then
        database.Close
    End If
    Exit Sub
Handler:
    Debug.Print "Error " & Err.Number & ": " & Err.Description
    Resume CleanExit
End Sub

Nota: O exemplo acima demonstra uma instrução UPDATE parametrizada, mas qualquer instrução SQL pode receber parâmetros.

Recuperando registros com uma consulta #

As consultas podem ser realizadas de duas maneiras, ambas retornando um objeto ADO Recordset que é uma coleção de linhas retornadas. Observe que ambos os exemplos abaixo usam a função OpenDatabaseConnection de [Fazendo uma conexão com uma fonte de dados](https://www.wikiod.com/pt/vba/trabalhando-com-ado#Fazendo uma conexão com uma fonte de dados -connection-to-a-data-source) para fins de brevidade. Lembre-se de que a sintaxe do SQL passado para a fonte de dados é específica do provedor.

O primeiro método é passar a instrução SQL diretamente para o objeto Connection e é o método mais fácil para executar consultas simples:

Public Sub DisplayDistinctItems()
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    
    If Not database Is Nothing Then
        Dim records As ADODB.Recordset
        Set records = database.Execute("SELECT DISTINCT Item FROM Table")
        'Loop through the returned Recordset.
        Do While Not records.EOF      'EOF is false when there are more records.
            'Individual fields are indexed either by name or 0 based ordinal.
            'Note that this is using the default .Fields member of the Recordset.
            Debug.Print records("Item")
            'Move to the next record.
            records.MoveNext
        Loop
    End If
CleanExit:
    If Not records Is Nothing Then records.Close
    If Not database Is Nothing And database.State = adStateOpen Then
        database.Close
    End If
    Exit Sub
Handler:
    Debug.Print "Error " & Err.Number & ": " & Err.Description
    Resume CleanExit
End Sub

O segundo método é criar um objeto ADO Command para a consulta que você deseja executar. Isso requer um pouco mais de código, mas é necessário para usar consultas parametrizadas:

Public Sub DisplayDistinctItems()
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    
    If Not database Is Nothing Then
        Dim query As ADODB.Command
        Set query = New ADODB.Command
        'Build the command to pass to the data source.
        With query
            .ActiveConnection = database
            .CommandText = "SELECT DISTINCT Item FROM Table"
            .CommandType = adCmdText
        End With
        Dim records As ADODB.Recordset
        'Execute the command to retrieve the recordset.
        Set records = query.Execute()

        Do While Not records.EOF
            Debug.Print records("Item")
            records.MoveNext
        Loop
    End If
CleanExit:
    If Not records Is Nothing Then records.Close
    If Not database Is Nothing And database.State = adStateOpen Then
        database.Close
    End If
    Exit Sub
Handler:
    Debug.Print "Error " & Err.Number & ": " & Err.Description
    Resume CleanExit
End Sub

Observe que os comandos enviados para a fonte de dados são vulneráveis ​​à injeção de SQL, intencionais ou não. Em geral, as consultas não devem ser criadas concatenando qualquer tipo de entrada do usuário. Em vez disso, eles devem ser parametrizados (consulte [Criando comandos parametrizados](https://www.wikiod.com/pt/vba/trabalhando-com-ado#Criando comandos parametrizados)).

Executando funções não escalares #

As conexões ADO podem ser usadas para executar praticamente qualquer função de banco de dados que o provedor oferece suporte via SQL. Neste caso, nem sempre é necessário usar o Recordset retornado pela função Execute, embora possa ser útil para obter atribuições de teclas após instruções INSERT com @@Identity ou comandos SQL semelhantes. Observe que o exemplo abaixo usa a função OpenDatabaseConnection de [Fazendo uma conexão com uma fonte de dados](https://www.wikiod.com/pt/vba/trabalhando-com-ado#Fazendo uma conexão com uma fonte de dados -to-a-data-source) para fins de brevidade.

Public Sub UpdateTheFoos()
    On Error GoTo Handler
    Dim database As ADODB.Connection
    Set database = OpenDatabaseConnection(SomeDSN)
    
    If Not database Is Nothing Then
        Dim update As ADODB.Command
        Set update = New ADODB.Command
        'Build the command to pass to the data source.
        With update
            .ActiveConnection = database
            .CommandText = "UPDATE Table SET Foo = 42 WHERE Bar IS NULL"
            .CommandType = adCmdText
            .Execute        'We don't need the return from the DB, so ignore it.
        End With
    End If
CleanExit:
    If Not database Is Nothing And database.State = adStateOpen Then
        database.Close
    End If
    Exit Sub
Handler:
    Debug.Print "Error " & Err.Number & ": " & Err.Description
    Resume CleanExit
End Sub

Observe que os comandos enviados para a fonte de dados são vulneráveis ​​à injeção de SQL, intencionais ou não. Em geral, as instruções SQL não devem ser criadas concatenando qualquer tipo de entrada do usuário. Em vez disso, eles devem ser parametrizados (consulte [Criando comandos parametrizados](https://www.wikiod.com/pt/vba/trabalhando-com-ado#Criando comandos parametrizados)).
