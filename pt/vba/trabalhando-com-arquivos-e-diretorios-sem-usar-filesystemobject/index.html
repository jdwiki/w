Trabalhando com arquivos e diretórios sem usar FileSystemObject

O Scripting.FileSystemObject é muito mais robusto que os métodos legados neste tópico. Deve ser preferido em quase todos os casos.

Determinando se existem pastas e arquivos #

Arquivos:

Para determinar se um arquivo existe, simplesmente passe o nome do arquivo para a função Dir$ e teste para ver se ele retorna um resultado. Observe que Dir$ suporta curingas, portanto, para testar um arquivo específico, o pathName passado deve ser testado para garantir que não os contenha. O exemplo abaixo gera um erro - se este não for o comportamento desejado, a função pode ser alterada para simplesmente retornar False.

Public Function FileExists(pathName As String) As Boolean
    If InStr(1, pathName, "*") Or InStr(1, pathName, "?") Then
        'Exit Function   'Return False on wild-cards.
        Err.Raise 52    'Raise error on wild-cards.
    End If
    FileExists = Dir$(pathName) <> vbNullString
End Function

Pastas (método Dir$):

A função Dir$() também pode ser usada para determinar se existe uma pasta especificando a passagem de vbDirectory para o parâmetro opcional attributes. Nesse caso, o valor pathName passado deve terminar com um separador de caminho (\), pois a correspondência de nomes de arquivo causará falsos positivos. Lembre-se de que os curingas só são permitidos após o último separador de caminho, portanto, a função de exemplo abaixo lançará um erro de tempo de execução 52 - “Nome ou número de arquivo incorreto” se a entrada contiver um curinga. Se este não for o comportamento desejado, descomente On Error Resume Next na parte superior da função. Lembre-se também que Dir$ suporta caminhos de arquivos relativos (ou seja, ..\Foo\Bar), então os resultados só são garantidos como válidos desde que o diretório de trabalho atual não seja alterado.

Public Function FolderExists(ByVal pathName As String) As Boolean
    'Uncomment the "On Error" line if paths with wild-cards should return False
    'instead of raising an error.
    'On Error Resume Next    
    If pathName = vbNullString Or Right$(pathName, 1) <> "\" Then
        Exit Function
    End If
    FolderExists = Dir$(pathName, vbDirectory) <> vbNullString
End Function

Pastas (método ChDir):

A instrução ChDir também pode ser usada para testar se existe uma pasta. Observe que esse método alterará temporariamente o ambiente em que o VBA está sendo executado, portanto, se isso for considerado, o método Dir$ deve ser usado. Ele tem a vantagem de ser muito menos tolerante com seu parâmetro. Este método também suporta caminhos de arquivos relativos, portanto, tem a mesma ressalva que o método Dir$.

Public Function FolderExists(ByVal pathName As String) As Boolean
    'Cache the current working directory
    Dim cached As String
    cached = CurDir$

    On Error Resume Next
    ChDir pathName
    FolderExists = Err.Number = 0
    On Error GoTo 0
    'Change back to the cached working directory.
    ChDir cached
End Function

Criando e excluindo pastas de arquivos #

OBSERVAÇÃO: para resumir, os exemplos abaixo usam a função FolderExists do exemplo Determinando se pastas e arquivos existem neste tópico.

A instrução MkDir pode ser usada para criar uma nova pasta. Ele aceita caminhos contendo letras de unidade (C:\Foo), nomes UNC (\\Server\Foo), caminhos relativos (..\Foo), ou o diretório de trabalho atual (Foo).

Se a unidade ou o nome UNC for omitido (ou seja, \Foo), a pasta será criada na unidade atual. Esta pode ou não ser a mesma unidade que o diretório de trabalho atual.

Public Sub MakeNewDirectory(ByVal pathName As String)
    'MkDir will fail if the directory already exists.
    If FolderExists(pathName) Then Exit Sub
    'This may still fail due to permissions, etc.
    MkDir pathName
End Sub

A instrução RmDir pode ser usada para excluir pastas existentes. Ele aceita caminhos nos mesmos formatos que MkDir e usa o mesmo relacionamento com o diretório de trabalho atual e a unidade. Observe que a instrução é semelhante ao comando shell rd do Windows, portanto, lançará um erro de tempo de execução 75: “Erro de acesso ao caminho/arquivo” se o diretório de destino não estiver vazio.

Public Sub DeleteDirectory(ByVal pathName As String)
    If Right$(pathName, 1) <> "\" Then
        pathName = pathName & "\"
    End If
    'Rmdir will fail if the directory doesn't exist.
    If Not FolderExists(pathName) Then Exit Sub
    'Rmdir will fail if the directory contains files.
    If Dir$(pathName & "*") <> vbNullString Then Exit Sub
    
    'Rmdir will fail if the directory contains directories.
    Dim subDir As String
    subDir = Dir$(pathName & "*", vbDirectory)
    Do
        If subDir <> "." And subDir <> ".." Then Exit Sub
        subDir = Dir$(, vbDirectory)
    Loop While subDir <> vbNullString
    
    'This may still fail due to permissions, etc.
    RmDir pathName
End Sub
