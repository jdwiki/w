API significa Application Programming Interface

API’s para VBA implicam em um conjunto de métodos que permitem a interação direta com o sistema operacional

Chamadas de sistema podem ser feitas executando procedimentos definidos em arquivos DLL

Arquivos de biblioteca de ambiente operacional comuns (DLLs):

Novos argumentos usados ​​para o sistema 64:

Tipo Artigo Descrição
Qualificador PtrSafe Indica que a instrução Declare é compatível com 64 bits. Este atributo é obrigatório em sistemas de 64 bits
Tipo de dados LongPtr Um tipo de dados variável que é um tipo de dados de 4 bytes em versões de 32 bits e um tipo de dados de 8 bytes em versões de 64 bits do Office 2010. Essa é a maneira recomendada de declarar um ponteiro ou um identificador para um novo código, mas também para código herdado se ele tiver que ser executado na versão de 64 bits do Office 2010. Tipo de dados Longo Longo Este é um tipo de dados de 8 bytes que está disponível apenas em versões de 64 bits do Office 2010. Você pode atribuir valores numéricos, mas não tipos numéricos (para evitar truncamento) Voltar para o início
Conversão Operador CLngPtr Converte uma expressão simples em um tipo de dados LongPtr
Conversão Operador CLngLng Converte uma expressão simples em um tipo de dados LongLong

Declaração e uso da API #

Declarando um procedimento de DLL para trabalhar com diferentes versões do VBA:

Option Explicit

#If Win64 Then

    Private Declare PtrSafe Sub xLib "Kernel32" Alias "Sleep" (ByVal dwMilliseconds As Long)

#ElseIf Win32 Then

    Private Declare Sub apiSleep Lib "Kernel32" Alias "Sleep" (ByVal dwMilliseconds As Long)

#End If

A declaração acima diz ao VBA como chamar a função “Sleep” definida no arquivo Kernel32.dll

Win64 e Win32 são constantes predefinidas usadas para compilação condicional

Constantes predefinidas

Algumas constantes de compilação já estão pré-definidas. Quais deles existem dependerão da quantidade de bits da versão do Office em que você está executando o VBA. Observe que o Vba7 foi introduzido junto com o Office 2010 para oferecer suporte a versões de 64 bits do Office.

Constante 16 bits 32 bits 64 bits
Vba6 Falso Se Vba6 Falso
Vba7 Falso Se Vba7 Verdadeiro
Win16 Verdadeiro Falso Falso
Win32 Falso Verdadeiro Verdadeiro
Win64 Falso Falso Verdadeiro
Mac Falso Se Mac Se Mac

Essas constantes referem-se à versão do Office, não à versão do Windows. Por exemplo Win32 = TRUE no Office de 32 bits, mesmo que o sistema operacional seja uma versão de 64 bits do Windows.

A principal diferença ao declarar APIs está entre as versões de 32 bits e 64 bits do Office que introduziram novos tipos de parâmetros (consulte a seção Comentários para obter mais detalhes)

Notas:

  • As declarações são colocadas na parte superior do módulo e fora de quaisquer Subs ou Funções
  • Procedimentos declarados em módulos padrão são públicos por padrão
  • Para declarar um procedimento privado para um módulo, anteceda a declaração com a palavra-chave Private
  • Os procedimentos de DLL declarados em qualquer outro tipo de módulo são privados para aquele módulo

Exemplo simples para a chamada da API Sleep:

Public Sub TestPause()

    Dim start As Double

    start = Timer

    Sleep 9000      'Pause execution for 9 seconds

    Debug.Print "Paused for " & Format(Timer - start, "#,###.000") & " seconds"

    'Immediate window result: Paused for 9.000 seconds

End Sub

Recomenda-se criar um módulo de API dedicado para fornecer acesso fácil às funções do sistema a partir de wrappers VBA – Subs ou Funções VBA normais que encapsulam os detalhes necessários para a chamada real do sistema, como parâmetros usados ​​em bibliotecas e inicialização desses parâmetros

O módulo pode conter todas as declarações e dependências:

  • Assinaturas de métodos e estruturas de dados necessárias
  • Wrappers que realizam validação de entrada e garantem que todos os parâmetros sejam passados ​​conforme o esperado

Para declarar um procedimento DLL, adicione uma instrução Declare à seção Declarações da janela de código.

Se o procedimento retornar um valor, declare-o como uma Função:

Declare Function publicname Lib "libname" [Alias "alias"] [([[ByVal] variable [As type] [,[ByVal] variable [As type]]...])] As Type

Se um procedimento não retornar um valor, declare-o como Sub:

Declare Sub publicname Lib "libname" [Alias "alias"] [([[ByVal] variable [As type] [,[ByVal] variable [As type]]...])]

O Office 2016 para Mac está em área restrita

Ao contrário de outras versões de aplicativos do Office que oferecem suporte a VBA, os aplicativos do Office 2016 para Mac são colocados em área restrita.

O sandboxing restringe os aplicativos de acessar recursos fora do contêiner do aplicativo. Isso afeta quaisquer suplementos ou macros que envolvam acesso a arquivos ou comunicação entre processos. Você pode minimizar os efeitos do sandboxing usando os novos comandos descritos na seção a seguir. Novos comandos VBA para Office 2016 para Mac

End Function

    Call SetLocalSetting(LOCALE_SDECIMAL, ",")  'to change to ","
    Stop                                        'check your control panel to verify or use the GetLocaleInfo API function
    Call SetLocalSetting(LOCALE_SDECIMAL, ".")  'to back change to "."
End Sub

Private Function SetLocalSetting(LC_CONST As Long, Setting As String) As Boolean
    Call SetLocaleInfo(GetUserDefaultLCID(), LC_CONST, Setting)
End Function
