Uma Collection é um objeto de contêiner que está incluído no tempo de execução do VBA. Não são necessárias referências adicionais para usá-lo. Uma Coleção pode ser usada para armazenar itens de qualquer tipo de dados e permite a recuperação pelo índice ordinal do item ou usando uma chave exclusiva opcional.

Obtendo a contagem de itens de uma coleção #

O número de itens em uma Collection pode ser obtido chamando sua função .Count:

Sintaxe:

.Count()

Amostra de Uso:

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"
        .Add "Two"
        .Add "Three"
        .Add "Four"
    End With
     
    Debug.Print foo.Count   'Prints 4
End Sub

Determinando se uma chave ou item existe em uma coleção #

Chaves #

Ao contrário de um Scripting.Dictionary, uma Collection não tem um método para determinar se uma determinada chave existe ou uma maneira para recuperar as chaves que estão presentes na Collection. O único método para determinar se uma chave está presente é usar o manipulador de erros:

Public Function KeyExistsInCollection(ByVal key As String, _
                                      ByRef container As Collection) As Boolean
    With Err
        If container Is Nothing Then .Raise 91
        On Error Resume Next
        Dim temp As Variant
        temp = container.Item(key)
        On Error GoTo 0

        If .Number = 0 Then
            KeyExistsInCollection = True 
        ElseIf .Number <> 5 Then
            .Raise .Number
        End If
    End With
End Function

Itens #

A única maneira de determinar se um item está contido em uma Coleção é iterar sobre a Coleção até que o item seja localizado. Observe que, como uma Collection pode conter primitivos ou objetos, é necessário algum tratamento extra para evitar erros de tempo de execução durante as comparações:

Public Function ItemExistsInCollection(ByRef target As Variant, _
                                       ByRef container As Collection) As Boolean
    Dim candidate As Variant
    Dim found As Boolean
    
    For Each candidate In container
        Select Case True
            Case IsObject(candidate) And IsObject(target)
                found = candidate Is target
            Case IsObject(candidate), IsObject(target)
                found = False
            Case Else
                found = (candidate = target)
        End Select
        If found Then
            ItemExistsInCollection = True
            Exit Function
        End If
    Next
End Function

Adicionando itens a uma coleção #

Os itens são adicionados a uma Collection chamando seu método .Add:

Sintaxe:

.Add(item, [key], [before, after])
Parâmetro Descrição
item O item a ser armazenado na Coleção. Isso pode ser essencialmente qualquer valor ao qual uma variável possa ser atribuída, incluindo tipos primitivos, arrays, objetos e Nothing.
chave Opcional. Uma String que serve como um identificador exclusivo para recuperar itens da Collection. Se a chave especificada já existir na Collection, resultará em um erro de tempo de execução 457: “Esta chave já está associada a um elemento desta coleção”.
antes Opcional. Uma chave existente (valor String) ou índice (valor numérico) para inserir o item antes na Collection. Se um valor for fornecido, o parâmetro depois deve estar vazio ou um erro de tempo de execução 5: “Chamada ou argumento de procedimento inválido” resultará. Se for passada uma chave String que não existe na Collection, ocorrerá um erro de tempo de execução 5: “Chamada ou argumento de procedimento inválido”. Se for passado um índice numérico que não existe na Collection, ocorrerá um erro de tempo de execução 9: “Subscrito fora do intervalo”.
depois Opcional. Uma chave existente (valor String) ou índice (valor numérico) para inserir o item depois na Collection. Se um valor for fornecido, o parâmetro antes deve estar vazio. Os erros gerados são idênticos ao parâmetro before.

Notas:

  • As chaves não diferenciam maiúsculas de minúsculas. .Add "Bar", "Foo" e .Add "Baz", "foo" resultará em uma colisão de teclas.

  • Se nenhum dos parâmetros opcionais antes ou depois for fornecido, o item será adicionado após o último item na Coleção.

  • As inserções feitas especificando um parâmetro antes ou depois alterarão os índices numéricos dos membros existentes para corresponder à sua nova posição. Isso significa que deve-se tomar cuidado ao fazer inserções em loops usando índices numéricos.

Amostra de Uso:

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"            'No key. This item can only be retrieved by index.
        .Add "Two", "Second"  'Key given. Can be retrieved by key or index.
        .Add "Three", , 1     'Inserted at the start of the collection.
        .Add "Four", , , 1    'Inserted at index 2.
    End With
    
    Dim member As Variant
    For Each member In foo
        Debug.Print member    'Prints "Three, Four, One, Two"
    Next
End Sub

Removendo itens de uma coleção #

Os itens são removidos de uma Collection chamando seu método .Remove:

Sintaxe:

.Remove(index)
Parâmetro Descrição
índice O item a ser removido da Coleção. Se o valor passado for um tipo numérico ou Variant com um subtipo numérico, ele será interpretado como um índice numérico. Se o valor passado for uma String ou Variant contendo uma string, ela será interpretada como a chave. Se for passada uma chave String que não existe na Collection, ocorrerá um erro de tempo de execução 5: “Chamada ou argumento de procedimento inválido”. Se for passado um índice numérico que não existe na Collection, ocorrerá um erro de tempo de execução 9: “Subscrito fora do intervalo”.

Notas:

  • Remover um item de uma Coleção mudará os índices numéricos de todos os itens depois dele na Coleção. Loops For que usam índices numéricos e removem itens devem ser executados para trás (Passo -1) para evitar exceções de subscrito e itens ignorados.
  • Os itens geralmente não devem ser removidos de uma Collection de dentro de um loop For Each, pois pode dar resultados imprevisíveis.

Amostra de Uso:

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"
        .Add "Two", "Second"
        .Add "Three"
        .Add "Four"
    End With
     
    foo.Remove 1            'Removes the first item.
    foo.Remove "Second"     'Removes the item with key "Second".
    foo.Remove foo.Count    'Removes the last item.
    
    Dim member As Variant
    For Each member In foo
        Debug.Print member  'Prints "Three"
    Next
End Sub

Recuperando itens de uma coleção #

Os itens podem ser recuperados de uma Collection chamando a função .Item.

Sintaxe:

.Item(index)
Parâmetro Descrição
índice O item a ser recuperado da Coleção. Se o valor passado for um tipo numérico ou Variant com um subtipo numérico, ele será interpretado como um índice numérico. Se o valor passado for uma String ou Variant contendo uma string, ela será interpretada como a chave. Se for passada uma chave String que não existe na Collection, ocorrerá um erro de tempo de execução 5: “Chamada ou argumento de procedimento inválido”. Se for passado um índice numérico que não existe na Collection, ocorrerá um erro de tempo de execução 9: “Subscrito fora do intervalo”.

Notas:

  • .Item é o membro padrão de Collection. Isso permite flexibilidade na sintaxe, conforme demonstrado no exemplo de uso abaixo.
  • Os índices numéricos são baseados em 1.
  • As chaves não diferenciam maiúsculas de minúsculas. .Item("Foo") e .Item("foo") referem-se à mesma chave.
  • O parâmetro index é não implicitamente convertido em um número de uma String ou vice-versa. É perfeitamente possível que .Item(1) e .Item("1") se refiram a itens diferentes da Coleção.

Uso de amostra (índices):

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"
        .Add "Two"
        .Add "Three"
        .Add "Four"
    End With
     
    Dim index As Long
    For index = 1 To foo.Count
        Debug.Print foo.Item(index) 'Prints One, Two, Three, Four
    Next
End Sub

Amostra de Uso (Chaves):

Public Sub Example()
    Dim keys() As String
    keys = Split("Foo,Bar,Baz", ",")
    Dim values() As String
    values = Split("One,Two,Three", ",")
        
    Dim foo As New Collection
    Dim index As Long
    For index = LBound(values) To UBound(values)
        foo.Add values(index), keys(index)
    Next
     
    Debug.Print foo.Item("Bar") 'Prints "Two"
End Sub

Amostra de Uso (Sintaxe Alternativa):

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One", "Foo"
        .Add "Two", "Bar"
        .Add "Three", "Baz"
    End With
    
    'All lines below print "Two"
    Debug.Print foo.Item("Bar")     'Explicit call syntax.
    Debug.Print foo("Bar")          'Default member call syntax.
    Debug.Print foo!Bar             'Bang syntax.
End Sub

Observe que a sintaxe bang (!) é permitida porque .Item é o membro padrão e pode receber um único argumento String. A utilidade desta sintaxe é questionável.

Limpando todos os itens de uma coleção #

A maneira mais fácil de limpar todos os itens de uma Coleção é simplesmente substituí-la por uma nova Coleção e deixar a antiga sair do escopo:

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"
        .Add "Two"
        .Add "Three"
    End With
     
    Debug.Print foo.Count   'Prints 3
    Set foo = New Collection
    Debug.Print foo.Count   'Prints 0
End Sub

No entanto, se houver várias referências à Collection mantidas, este método fornecerá apenas uma Collection vazia para a variável atribuída.

Public Sub Example()
    Dim foo As New Collection
    Dim bar As Collection
    
    With foo
        .Add "One"
        .Add "Two"
        .Add "Three"
    End With
     
    Set bar = foo
    Set foo = New Collection
    
    Debug.Print foo.Count   'Prints 0
    Debug.Print bar.Count   'Prints 3
End Sub

Nesse caso, a maneira mais fácil de limpar o conteúdo é fazer um loop pelo número de itens na Coleção e remover repetidamente o item mais baixo:

Public Sub ClearCollection(ByRef container As Collection)
    Dim index As Long
    For index = 1 To container.Count
        container.Remove 1
    Next
End Sub
