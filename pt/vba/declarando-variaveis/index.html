Dicas de tipo #

As dicas de tipo são fortemente desencorajadas. Eles existem e estão documentados aqui por motivos históricos e de compatibilidade com versões anteriores. Você deve usar a sintaxe As [DataType] em vez disso.

Public Sub ExampleDeclaration()

        Dim someInteger% '% Equivalent to "As Integer"
        Dim someLong&    '& Equivalent to "As Long"
        Dim someDecimal@ '@ Equivalent to "As Currency"
        Dim someSingle!  '! Equivalent to "As Single"
        Dim someDouble#  '# Equivalent to "As Double"
        Dim someString$  '$ Equivalent to "As String"

        Dim someLongLong^  '^ Equivalent to "As LongLong" in 64-bit VBA hosts
End Sub

As dicas de tipo diminuem significativamente a legibilidade do código e incentivam uma Notação Húngara herdada que também dificulta a legibilidade:

Dim strFile$
Dim iFile%

Em vez disso, declare variáveis ​​mais próximas de seu uso e nomeie as coisas pelo que são usadas, não pelo tipo:

Dim path As String
Dim handle As Integer

As dicas de tipo também podem ser usadas em literais, para impor um tipo específico. Por padrão, um literal numérico menor que 32.768 será interpretado como um literal Integer, mas com uma dica de tipo você pode controlar isso:

Dim foo 'implicit Variant
foo = 42& ' foo is now a Long
foo = 42# ' foo is now a Double
Debug.Print TypeName(42!) ' prints "Single"

Dicas de tipo geralmente não são necessárias em literais, porque elas seriam atribuídas a uma variável declarada com um tipo explícito ou convertidas implicitamente para o tipo apropriado quando passadas como parâmetros. As conversões implícitas podem ser evitadas usando uma das funções de conversão de tipo explícitas:

'Calls procedure DoSomething and passes a literal 42 as a Long using a type hint
DoSomething 42&

'Calls procedure DoSomething and passes a literal 42 explicitly converted to a Long
DoSomething CLng(42)

Funções internas que retornam string #

A maioria das funções integradas que lidam com strings vêm em duas versões: uma versão de tipo flexível que retorna um Variant e uma versão de tipo forte (terminando com $) que retorna um String. A menos que você esteja atribuindo o valor de retorno a uma Variant, você deve preferir a versão que retorna uma String - caso contrário, haverá uma conversão implícita do valor de retorno.

Debug.Print Left(foo, 2)  'Left returns a Variant
Debug.Print Left$(foo, 2) 'Left$ returns a String

Essas funções são:

  • VBA.Conversion.Error -> VBA.Conversion.Error$
  • VBA.Conversion.Hex -> VBA.Conversion.Hex$
  • VBA.Conversion.Oct -> VBA.Conversion.Oct$
  • VBA.Conversion.Str -> VBA.Conversion.Str$
  • VBA.FileSystem.CurDir -> VBA.FileSystem.CurDir$
  • VBA.[_HiddenModule].Input -> VBA.[_HiddenModule].Input$
  • VBA.[_HiddenModule].InputB -> VBA.[_HiddenModule].InputB$
  • VBA.Interaction.Command -> VBA.Interaction.Command$
  • VBA.Interaction.Environ -> VBA.Interaction.Environ$
  • VBA.Strings.Chr -> VBA.Strings.Chr$
  • VBA.Strings.ChrB -> VBA.Strings.ChrB$
  • VBA.Strings.ChrW -> VBA.Strings.ChrW$
  • VBA.Strings.Format -> VBA.Strings.Format$
  • VBA.Strings.LCase -> VBA.Strings.LCase$
  • VBA.Strings.Left -> VBA.Strings.Left$
  • VBA.Strings.LeftB -> VBA.Strings.LeftB$
  • VBA.Strings.LTtrim -> VBA.Strings.LTrim$
  • VBA.Strings.Mid -> VBA.Strings.Mid$
  • VBA.Strings.MidB -> VBA.Strings.MidB$
  • VBA.Strings.Right -> VBA.Strings.Right$
  • VBA.Strings.RightB -> VBA.Strings.RightB$
  • VBA.Strings.RTrim -> VBA.Strings.RTrim$
  • VBA.Strings.Space -> VBA.Strings.Space$
  • VBA.Strings.Str -> VBA.Strings.Str$
  • VBA.Strings.String -> VBA.Strings.String$
  • VBA.Strings.Trim -> VBA.Strings.Trim$
  • VBA.Strings.UCase -> VBA.Strings.UCase$

Observe que estes são aliases de função, não exatamente dicas de tipo. A função Left corresponde à função oculta B_Var_Left, enquanto a versão Left$ corresponde à função oculta B_Str_Left.

Nas primeiras versões do VBA, o sinal $ não é um caractere permitido e o nome da função tinha que ser colocado entre colchetes. No Word Basic, havia muitas, muitas outras funções que retornavam strings que terminavam em $.

Variáveis #

Alcance #

Uma variável pode ser declarada (em nível de visibilidade crescente):

  • A nível de procedimento, utilizando a palavra-chave Dim em qualquer procedimento; uma variável local.
  • A nível de módulo, utilizando a palavra-chave Private em qualquer tipo de módulo; um campo privado.
  • Em nível de instância, usando a palavra-chave Friend em qualquer tipo de módulo de classe; um campo amigo.
  • Em nível de instância, usando a palavra-chave Public em qualquer tipo de módulo de classe; um campo público.
  • Globalmente, usando a palavra-chave Public em um módulo padrão; uma variável global.

As variáveis ​​devem sempre ser declaradas com o menor escopo possível: prefira passar parâmetros para procedimentos, em vez de declarar variáveis ​​globais.

Consulte https://www.wikiod.com/pt/vba/declarando-variaveis#Modificadores de acesso para obter mais informações.

Variáveis ​​locais #

Use a palavra-chave Dim para declarar uma variável local:

Dim identifierName [As Type][, identifierName [As Type], ...]

A parte [As Type] da sintaxe de declaração é opcional. Quando especificado, define o tipo de dados da variável, que determina quanta memória será alocada para essa variável. Isso declara uma variável String:

Dim identifierName As String

Quando um tipo não é especificado, o tipo é implicitamente Variant:

Dim identifierName 'As Variant is implicit

A sintaxe do VBA também suporta a declaração de várias variáveis ​​em uma única instrução:

Dim someString As String, someVariant, someValue As Long

Observe que o [As Type] deve ser especificado para cada variável (exceto as ‘Variant’). Esta é uma armadilha relativamente comum:

Dim integer1, integer2, integer3 As Integer 'Only integer3 is an Integer. 
                                            'The rest are Variant.

Variáveis ​​estáticas #

Variáveis ​​locais também podem ser Static. No VBA a palavra-chave Static é usada para fazer uma variável “lembrar” o valor que ela tinha, da última vez que um procedimento foi chamado:

Private Sub DoSomething()
    Static values As Collection
    If values Is Nothing Then
        Set values = New Collection
        values.Add "foo"
        values.Add "bar"
    End If
    DoSomethingElse values
End Sub

Aqui a coleção values é declarada como local Static; por ser uma variável de objeto, ela é inicializada como Nothing. A condição que segue a declaração verifica se a referência do objeto era Set antes - se for a primeira vez que o procedimento é executado, a coleção é inicializada. DoSomethingElse pode estar adicionando ou removendo itens, e eles ainda estarão na coleção na próxima vez que DoSomething for chamado.

Alternativo

A palavra-chave Static do VBA pode ser facilmente mal interpretada - especialmente por programadores experientes que geralmente trabalham em outras linguagens. Em muitas linguagens, static é usado para fazer um membro de classe (campo, propriedade, método, …) pertencer ao tipo ao invés da instância. O código no contexto static não pode referenciar o código no contexto instance. A palavra-chave Static do VBA significa algo totalmente diferente.

Freqüentemente, um local Static pode ser implementado como uma variável Private em nível de módulo (campo) - no entanto, isso desafia o princípio pelo qual uma variável deve ser declarada com o menor escopo possível; confie em seus instintos, use o que preferir - ambos funcionarão… mas usar Static sem entender o que ele faz pode levar a bugs interessantes.

Dim vs. Privado

A palavra-chave Dim é legal em nível de procedimento e módulo; seu uso no nível do módulo é equivalente ao uso da palavra-chave Private:

Option Explicit
Dim privateField1 As Long 'same as Private privateField2 as Long
Private privateField2 As Long 'same as Dim privateField2 as Long

A palavra-chave Private só é válida no nível do módulo; isso convida a reservar Dim para variáveis ​​locais e declarar variáveis ​​de módulo com Private, especialmente com a palavra-chave contrastante Public que teria que ser usada de qualquer maneira para declarar um membro público. Alternativamente, use Dim everywhere - o que importa é consistência:

“Campos privados”

  • DO use Private para declarar uma variável de nível de módulo.
  • DO use Dim para declarar uma variável local.
  • NÃO use Dim para declarar uma variável de nível de módulo.

“Escurecer em todos os lugares”

  • DO use Dim para declarar qualquer coisa privada/local.
  • NÃO use Private para declarar uma variável de nível de módulo.
  • EVITE declarar campos “Públicos”.*

*Em geral, deve-se evitar declarar campos Público ou Global de qualquer maneira.

Campos #

Uma variável declarada no nível do módulo, na seção de declarações na parte superior do corpo do módulo, é um campo. Um campo Público declarado em um módulo padrão é uma variável global:

Public PublicField As Long

Uma variável com escopo global pode ser acessada de qualquer lugar, incluindo outros projetos VBA que fariam referência ao projeto no qual ela foi declarada.

Para tornar uma variável global/pública, mas visível apenas de dentro do projeto, use o modificador Friend:

Friend FriendField As Long

Isso é especialmente útil em suplementos, onde a intenção é que outros projetos VBA façam referência ao projeto de suplemento e possam consumir a API pública.

Friend FriendField As Long 'public within the project, aka for "friend" code
Public PublicField As Long 'public within and beyond the project

Campos amigos não estão disponíveis em módulos padrão.

Campos de instância #

Uma variável declarada no nível do módulo, na seção de declarações na parte superior do corpo de um módulo de classe (incluindo ThisWorkbook, ThisDocument, Worksheet, UserForm e class modules), é um * campo de instância*: só existe enquanto houver uma instância da classe por perto.

'> Class1
Option Explicit
Public PublicField As Long

'> Module1
Option Explicit
Public Sub DoSomething()
    'Class1.PublicField means nothing here
    With New Class1
        .PublicField = 42
    End With
    'Class1.PublicField means nothing here
End Sub

Campos de encapsulamento #

Os dados da instância geralmente são mantidos como Privado e apelidados de encapsulados. Um campo privado pode ser exposto usando um procedimento Property. Para expor uma variável privada publicamente sem dar acesso de gravação ao chamador, um módulo de classe (ou um módulo padrão) implementa um membro Property Get:

Option Explicit
Private encapsulated As Long

Public Property Get SomeValue() As Long
    SomeValue = encapsulated
End Property

Public Sub DoSomething()
    encapsulated = 42
End Sub

A própria classe pode modificar o valor encapsulado, mas o código de chamada só pode acessar os membros Public (e membros Friend, se o chamador estiver no mesmo projeto).

Para permitir que o chamador modifique:

  • Um valor encapsulado, um módulo expõe um membro Property Let.
  • Uma referência de objeto encapsulada, um módulo expõe um membro Property Set.

Constantes (Const) #

Se você tiver um valor que nunca muda em seu aplicativo, poderá definir uma constante nomeada e usá-la no lugar de um valor literal.

Você pode usar Const apenas em nível de módulo ou procedimento. Isso significa que o contexto de declaração de uma variável deve ser uma classe, estrutura, módulo, procedimento ou bloco e não pode ser um arquivo de origem, namespace ou interface.

Public Const GLOBAL_CONSTANT As String = "Project Version #1.000.000.001"
Private Const MODULE_CONSTANT As String = "Something relevant to this Module"

Public Sub ExampleDeclaration()    

    Const SOME_CONSTANT As String = "Hello World"
    
    Const PI As Double = 3.141592653

End Sub

Embora possa ser considerado uma boa prática especificar tipos Constant, isso não é estritamente necessário. Não especificar o tipo ainda resultará no tipo correto:

Public Const GLOBAL_CONSTANT = "Project Version #1.000.000.001" 'Still a string
Public Sub ExampleDeclaration()

    Const SOME_CONSTANT = "Hello World"           'Still a string
    Const DERIVED_CONSTANT = SOME_CONSTANT        'DERIVED_CONSTANT is also a string
    Const VAR_CONSTANT As Variant = SOME_CONSTANT 'VAR_CONSTANT is Variant/String
    
    Const PI = 3.141592653        'Still a double
    Const DERIVED_PI = PI         'DERIVED_PI is also a double
    Const VAR_PI As Variant = PI  'VAR_PI is Variant/Double
    
End Sub

Observe que isso é específico para Constants e em contraste com variáveis ​​em que não especificar o tipo resulta em um tipo Variant.

Embora seja possível declarar explicitamente uma constante como uma String, não é possível declarar uma constante como uma string usando a sintaxe de string de largura fixa

'This is a valid 5 character string constant
Const FOO As String = "ABCDE"

'This is not valid syntax for a 5 character string constant
Const FOO As String * 5 = "ABCDE"

Declarando Strings de Comprimento Fixo #

Em VBA, Strings podem ser declaradas com um comprimento específico; eles são automaticamente preenchidos ou truncados para manter esse comprimento conforme declarado.

Public Sub TwoTypesOfStrings()

    Dim FixedLengthString As String * 5 ' declares a string of 5 characters
    Dim NormalString As String

    Debug.Print FixedLengthString       ' Prints "     "
    Debug.Print NormalString            ' Prints ""

    FixedLengthString = "123"           ' FixedLengthString now equals "123  "
    NormalString = "456"                ' NormalString now equals "456"

    FixedLengthString = "123456"        ' FixedLengthString now equals "12345"
    NormalString = "456789"             ' NormalString now equals "456789"

End Sub

Declaração Implícita e Explícita #

Se um módulo de código não contém Option Explicit na parte superior do módulo, então o compilador irá automaticamente (ou seja, “implicitamente”) criar variáveis ​​para você quando você as usar. Eles serão padronizados para o tipo de variável Variant.

Public Sub ExampleDeclaration()    

    someVariable = 10                  ' 
    someOtherVariable = "Hello World"
    'Both of these variables are of the Variant type.

End Sub

No código acima, se Option Explicit for especificado, o código interromperá porque está faltando as instruções Dim necessárias para someVariable e someOtherVariable.

Option Explicit

Public Sub ExampleDeclaration()   

    Dim someVariable As Long 
    someVariable = 10

    Dim someOtherVariable As String
    someOtherVariable = "Hello World"

End Sub

É considerado uma prática recomendada usar Option Explicit em módulos de código, para garantir que você declare todas as variáveis.

Consulte https://www.wikiod.com/pt/excel-vba/praticas-recomendadas-de-vba como definir essa opção por padrão.

Modificadores de acesso #

A instrução Dim deve ser reservada para variáveis ​​locais. No nível do módulo, prefira modificadores de acesso explícitos:

  • Private para campos privados, que só podem ser acessados ​​dentro do módulo em que estão declarados.
  • Public para campos públicos e variáveis ​​globais, que podem ser acessadas por qualquer código de chamada.
  • Friend para variáveis ​​públicas dentro do projeto, mas inacessíveis para outros projetos VBA de referência (relevante para add-ins)
  • Global também pode ser usado para campos Public em módulos padrão, mas é ilegal em módulos de classe e é obsoleto de qualquer maneira - prefira o modificador Public. Este modificador também não é legal para procedimentos.

Os modificadores de acesso são aplicáveis ​​a variáveis ​​e procedimentos.

Private ModuleVariable As String
Public GlobalVariable As String

Private Sub ModuleProcedure()

    ModuleVariable = "This can only be done from within the same Module"

End Sub

Public Sub GlobalProcedure()

    GlobalVariable = "This can be done from any Module within this Project"

End Sub

Opção Módulo Privado #

Procedimentos Sub sem parâmetros públicos em módulos padrão são expostos como macros e podem ser anexados a controles e atalhos de teclado no documento host.

Por outro lado, os procedimentos públicos Function em módulos padrão são expostos como funções definidas pelo usuário (UDF’s) no aplicativo host.

Especificar Option Private Module na parte superior de um módulo padrão evita que seus membros sejam expostos como macros e UDFs para o aplicativo host.

Quando usar uma variável estática #

Uma variável estática declarada localmente não é destruída e não perde seu valor quando o procedimento Sub é encerrado. Chamadas subsequentes para o procedimento não requerem reinicialização ou atribuição, embora você possa querer ‘zerar’ qualquer valor lembrado.

Eles são particularmente úteis ao vincular tardiamente um objeto em um sub ‘auxiliar’ que é chamado repetidamente.

Fragmento 1: reutilize um objeto Scripting.Dictionary em muitas planilhas

Option Explicit

Sub main()
    Dim w As Long
    
    For w = 1 To Worksheets.Count
        processDictionary ws:=Worksheets(w)
    Next w
End Sub

Sub processDictionary(ws As Worksheet)
    Dim i As Long, rng As Range
    Static dict As Object
    
    If dict Is Nothing Then
        'initialize and set the dictionary object
        Set dict = CreateObject("Scripting.Dictionary")
        dict.CompareMode = vbTextCompare
    Else
        'remove all pre-existing dictionary entries
        ' this may or may not be desired if a single dictionary of entries
        ' from all worksheets is preferred
        dict.RemoveAll
    End If
    
    With ws
        
        'work with a fresh dictionary object for each worksheet
        ' without constructing/destructing a new object each time
        ' or do not clear the dictionary upon subsequent uses and 
        ' build a dictionary containing entries from all worksheets
    
    End With
End Sub

Snippet 2: crie uma planilha UDF que vincule tardiamente o objeto VBScript.RegExp

Option Explicit

Function numbersOnly(str As String, _
                     Optional delim As String = ", ")
    Dim n As Long, nums() As Variant
    Static rgx As Object, cmat As Object

    'with rgx as static, it only has to be created once
    'this is beneficial when filling a long column with this UDF
    If rgx Is Nothing Then
        Set rgx = CreateObject("VBScript.RegExp")
    Else
        Set cmat = Nothing
    End If
    
    With rgx
        .Global = True
        .MultiLine = True
        .Pattern = "[0-9]{1,999}"
        If .Test(str) Then
            Set cmat = .Execute(str)
            'resize the nums array to accept the matches
            ReDim nums(cmat.Count - 1)
            'populate the nums array with the matches
            For n = LBound(nums) To UBound(nums)
                nums(n) = cmat.Item(n)
            Next n
            'convert the nums array to a delimited string
            numbersOnly = Join(nums, delim)
        Else
            numbersOnly = vbNullString
        End If
    End With
End Function


                    Exemplo de UDF com objeto estático preenchido por meio milhão de linhas

*Tempos decorridos para preencher 500 mil linhas com UDF:
       - com Dim rgx As Object: 148,74 segundos
       - com Static rgx As Object: 26,07 segundos
* Estes devem ser considerados apenas para comparação relativa. Seus próprios resultados variam de acordo com a complexidade e
    escopo das operações realizadas.

Lembre-se de que uma UDF não é calculada uma vez durante a vida útil de uma pasta de trabalho. Mesmo uma UDF não volátil será recalculada sempre que os valores dentro do(s) intervalo(s) a que ela fizer referência estiverem sujeitos a alterações. Cada evento de recálculo subsequente apenas aumenta os benefícios de uma variável declarada estaticamente.

  • Uma variável estática está disponível para o tempo de vida do módulo, não para o procedimento ou função em que foi declarado e atribuído.
  • Variáveis ​​estáticas só podem ser declaradas localmente.
  • Variáveis ​​estáticas mantêm muitas das mesmas propriedades de uma variável de nível de módulo privado, mas com um escopo mais restrito.
