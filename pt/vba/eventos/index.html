Sintaxe #

  • Módulo de origem: [Public] Event [identifier]([argument_list])

  • Módulo Handler: Dim|Private|Public WithEvents [identificador] As [tipo]

  • Um evento só pode ser Público. O modificador é opcional porque os membros do módulo de classe (incluindo eventos) são implicitamente Public por padrão.

  • Uma variável WithEvents pode ser Private ou Public, mas não Friend. O modificador é obrigatório porque WithEvents não é uma palavra-chave que declara uma variável, mas uma palavra-chave modificadora que faz parte da sintaxe de declaração da variável. Portanto, a palavra-chave Dim deve ser usada se um modificador de acesso não estiver presente.

Fontes e manipuladores #

O que são eventos? #

O VBA é orientado a eventos: o código VBA é executado em resposta a eventos gerados pelo aplicativo host ou pelo documento host - entender os eventos é fundamental para entender o VBA.

As APIs geralmente expõem objetos que geram vários eventos em resposta a vários estados. Por exemplo, um objeto Excel.Application gera um evento sempre que uma nova pasta de trabalho é criada, aberta, ativada ou fechada. Ou sempre que uma planilha é calculada. Ou logo antes de um arquivo ser salvo. Ou imediatamente depois. Um botão em um formulário gera um evento Click quando o usuário clica nele, o próprio formulário do usuário gera um evento logo após ser ativado e outro logo antes de ser fechado.

Do ponto de vista da API, os eventos são pontos de extensão: o código do cliente pode optar por implementar o código que trata esses eventos e executar o código personalizado sempre que esses eventos forem acionados: é assim que você pode executar seu código personalizado automaticamente toda vez que a seleção alterações em qualquer planilha - manipulando o evento que é acionado quando a seleção é alterada em qualquer planilha.

Um objeto que expõe eventos é uma fonte de eventos. Um método que trata de um evento é um handler.

Manipuladores #

Módulos de documento VBA (por exemplo, ThisDocument, ThisWorkbook, Sheet1, etc.) e módulos UserForm são módulos de classe que implementam interfaces especiais que expõem vários eventos. Você pode navegar por essas interfaces no menu suspenso do lado esquerdo na parte superior do painel de código:

A lista suspensa do lado direito lista os membros da interface selecionados na lista suspensa do lado esquerdo:

O VBE gera automaticamente um stub do manipulador de eventos quando um item é selecionado na lista do lado direito ou navega até lá se o manipulador existir.

Você pode definir uma variável WithEvents no escopo do módulo em qualquer módulo:

Private WithEvents Foo As Workbook
Private WithEvents Bar As Worksheet

Cada declaração WithEvents fica disponível para seleção no menu suspenso do lado esquerdo. Quando um evento é selecionado no menu suspenso do lado direito, o VBE gera um stub do manipulador de eventos com o nome do objeto WithEvents e o nome do evento, unido com um sublinhado:

Private WithEvents Foo As Workbook
Private WithEvents Bar As Worksheet

Private Sub Foo_Open()

End Sub

Private Sub Bar_SelectionChange(ByVal Target As Range)

End Sub

Apenas tipos que expõem pelo menos um evento podem ser usados ​​com WithEvents, e declarações WithEvents não podem ser atribuídas a uma referência no local com a palavra-chave New. Este código é ilegal:

Private WithEvents Foo As New Workbook 'illegal

A referência do objeto deve ser Set explicitamente; em um módulo de classe, um bom lugar para fazer isso geralmente é no manipulador Class_Initialize, porque então a classe manipula os eventos desse objeto enquanto sua instância existir.

Fontes #

Qualquer módulo de classe (ou módulo de documento ou formulário de usuário) pode ser uma fonte de evento. Use a palavra-chave Event para definir a assinatura do evento, na seção de declarações do módulo:

Public Event SomethingHappened(ByVal something As String)

A assinatura do evento determina como o evento é gerado e qual será a aparência dos manipuladores de eventos.

Os eventos só podem ser gerados dentro da classe em que estão definidos - o código do cliente pode apenas tratá-los. Os eventos são gerados com a palavra-chave RaiseEvent; os argumentos do evento são fornecidos nesse ponto:

Public Sub DoSomething()
    RaiseEvent SomethingHappened("hello")
End Sub

Sem código que trata o evento SomethingHappened, a execução do procedimento DoSomething ainda gerará o evento, mas nada acontecerá. Assumindo que a fonte do evento é o código acima em uma classe chamada Something, este código em ThisWorkbook mostraria uma caixa de mensagem dizendo “hello” sempre que test.DoSomething for chamado:

Private WithEvents test As Something

Private Sub Workbook_Open()
    Set test = New Something
    test.DoSomething
End Sub

Private Sub test_SomethingHappened(ByVal bar As String)
'this procedure runs whenever 'test' raises the 'SomethingHappened' event
    MsgBox bar
End Sub

Passando dados de volta para a fonte do evento #

Usando parâmetros passados ​​por referência #

Um evento pode definir um parâmetro ByRef destinado a ser retornado ao chamador:

Public Event BeforeSomething(ByRef cancel As Boolean)
Public Event AfterSomething()

Public Sub DoSomething()
    Dim cancel As Boolean
    RaiseEvent BeforeSomething(cancel)
    If cancel Then Exit Sub

    'todo: actually do something

    RaiseEvent AfterSomething
End Sub

Se o evento BeforeSomething tiver um manipulador que define seu parâmetro cancel como True, quando a execução retornar do manipulador, cancel será True e AfterSomething nunca será gerado.

Private WithEvents foo As Something

Private Sub foo_BeforeSomething(ByRef cancel As Boolean)
    cancel = MsgBox("Cancel?", vbYesNo) = vbYes
End Sub

Private Sub foo_AfterSomething()
    MsgBox "Didn't cancel!"
End Sub

Assumindo que a referência do objeto foo é atribuída em algum lugar, quando foo.DoSomething é executado, uma caixa de mensagem pergunta se deseja cancelar, e uma segunda caixa de mensagem diz “não cancelou” somente quando Não foi selecionado.

Usando objetos mutáveis #

Você também pode passar uma cópia de um objeto mutável ByVal e permitir que os manipuladores modifiquem as propriedades desse objeto; o chamador pode então ler os valores de propriedade modificados e agir de acordo.

'class module ReturnBoolean
Option Explicit
Private encapsulated As Boolean

Public Property Get ReturnValue() As Boolean
'Attribute ReturnValue.VB_UserMemId = 0
    ReturnValue = encapsulated
End Property

Public Property Let ReturnValue(ByVal value As Boolean)
    encapsulated = value
End Property

Combinado com o tipo Variant, isso pode ser usado para criar maneiras não óbvias de retornar um valor ao chamador:

Public Event SomeEvent(ByVal foo As Variant)

Public Sub DoSomething()
    Dim result As ReturnBoolean
    result = New ReturnBoolean

    RaiseEvent SomeEvent(result)

    If result Then ' If result.ReturnValue Then
        'handler changed the value to True
    Else
        'handler didn't modify the value
    End If
End Sub

O manipulador ficaria assim:

Private Sub source_SomeEvent(ByVal foo As Variant) 'foo is actually a ReturnBoolean object
    foo = True 'True is actually assigned to foo.ReturnValue, the class' default member
End Sub
