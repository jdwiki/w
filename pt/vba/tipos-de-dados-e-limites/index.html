Variante #

Dim Value As Variant    'Explicit
Dim Value               'Implicit

Uma Variant é um tipo de dados COM que é usado para armazenar e trocar valores de tipos arbitrários, e qualquer outro tipo no VBA pode ser atribuído a uma Variant. Variáveis ​​declaradas sem um tipo explícito especificado por As [Type] padrão para Variant.

As variantes são armazenadas na memória como uma estrutura VARIANT que consiste em um descritor de tipo de byte (VARTYPE) seguido por 6 bytes reservados e uma área de dados de 8 bytes. Para tipos numéricos (incluindo Date e Boolean), o valor subjacente é armazenado na própria Variant. Para todos os outros tipos, a área de dados contém um ponteiro para o valor subjacente.

O tipo subjacente de uma Variant pode ser determinado com a função VarType() que retorna o valor numérico armazenado no descritor de tipo, ou a função TypeName() que retorna a representação da string:

Dim Example As Variant
Example = 42
Debug.Print VarType(Example)    'Prints 2 (VT_I2)
Debug.Print TypeName(Example)   'Prints "Integer"
Example = "Some text"
Debug.Print VarType(Example)    'Prints 8 (VT_BSTR)
Debug.Print TypeName(Example)   'Prints "String"

Como as Variantes podem armazenar valores de qualquer tipo, as atribuições de literais sem [dicas de tipo](https://www.wikiod.com/pt/vba/declarando-variaveis#Dicas de tipo) serão convertidas implicitamente para uma Variante do tipo apropriado de acordo com a tabela abaixo. Literais com dicas de tipo serão convertidos em uma Variante do tipo sugerido.

Valor Tipo resultante
Valores de string Cadeia
Números de ponto não flutuante no intervalo inteiro Inteiro
Números de ponto não flutuante em Longo alcance Longo
Números de ponto não flutuante fora de Longo alcance Duplo
Todos os números de ponto flutuante Duplo

Observação: a menos que haja um motivo específico para usar uma Variant (ou seja, um iterador em um loop For Each ou um requisito de API), o tipo geralmente deve ser evitado para tarefas de rotina pelos seguintes motivos:

  • Não são type safe, aumentando a possibilidade de erros de tempo de execução. Por exemplo, uma Variant com um valor Integer se transformará silenciosamente em um Long em vez de transbordar.
  • Eles introduzem sobrecarga de processamento exigindo pelo menos uma desreferência de ponteiro adicional.
  • O requisito de memória para uma Variante é sempre pelo menos 8 bytes maior do que o necessário para armazenar o tipo subjacente.

A função de conversão para uma Variant é CVar().

Boleano #

Dim Value As Boolean

Um Boolean é usado para armazenar valores que podem ser representados como True ou False. Internamente, o tipo de dados é armazenado como um valor de 16 bits com 0 representando False e qualquer outro valor representando True.

Deve-se notar que quando um Boolean é convertido para um tipo numérico, todos os bits são definidos como 1. Isso resulta em uma representação interna de -1 para tipos com sinal e o valor máximo para um tipo sem sinal (Byte).

Dim Example As Boolean
Example = True
Debug.Print CInt(Example)  'Prints -1
Debug.Print CBool(42)      'Prints True
Debug.Print CByte(True)    'Prints 255

A função de conversão para um Boolean é CBool(). Mesmo que seja representado internamente como um número de 16 bits, a conversão para um Boolean de valores fora desse intervalo está protegida contra estouro, embora defina todos os 16 bits como 1:

Dim Example As Boolean
Example = CBool(2 ^ 17)
Debug.Print CInt(Example)   'Prints -1
Debug.Print CByte(Example)  'Prints 255

Corda #

Uma String representa uma sequência de caracteres e vem em dois tipos:

Comprimento variável #

Dim Value As String

Uma String de comprimento variável permite anexação e truncamento e é armazenada na memória como um COM BSTR. Isso consiste em um inteiro sem sinal de 4 bytes que armazena o comprimento da String em bytes seguido pelos próprios dados da string como caracteres largos (2 bytes por caractere) e terminado com 2 bytes nulos. Assim, o comprimento máximo de string que pode ser tratado pelo VBA é de 2.147.483.647 caracteres.

O ponteiro interno para a estrutura (recuperável pela função StrPtr()) aponta para a localização de memória dos dados, não para o prefixo de comprimento. Isso significa que uma String VBA pode receber diretamente funções de API que exigem um ponteiro para uma matriz de caracteres.

Como o comprimento pode mudar, o VBA realoca memória para uma String toda vez que a variável é atribuída a, o que pode impor penalidades de desempenho para procedimentos que as alteram repetidamente.

Comprimento fixo #

Dim Value As String * 1024    'Declares a fixed length string of 1024 characters.

Strings de comprimento fixo são alocados 2 bytes para cada caractere e são armazenados na memória como um array de bytes simples. Uma vez alocado, o comprimento da String é imutável. Eles são não com terminação nula na memória, portanto, uma string que preenche a memória alocada com caracteres não nulos não é adequada para passar para funções da API esperando uma string terminada com nulo.

Strings de comprimento fixo carregam uma limitação de índice de 16 bits herdada, portanto, só podem ter até 65.535 caracteres de comprimento. A tentativa de atribuir um valor maior que o espaço de memória disponível não resultará em um erro de tempo de execução - em vez disso, o valor resultante será simplesmente truncado:

Dim Foobar As String * 5
Foobar = "Foo" & "bar"
Debug.Print Foobar          'Prints "Fooba"

A função de conversão para converter para uma String de qualquer tipo é CStr().

Byte #

Dim Value As Byte

Um Byte é um tipo de dados de 8 bits sem sinal. Ele pode representar números inteiros entre 0 e 255 e tentar armazenar um valor fora desse intervalo resultará em erro de tempo de execução 6: Overflow . Byte é o único tipo não assinado intrínseco disponível no VBA.

A função de conversão para converter em um Byte é CByte(). Para conversões de tipos de ponto flutuante, o resultado é arredondado para o valor inteiro mais próximo com 0,5 de arredondamento para cima.

Byte Arrays e Strings

Strings e arrays de bytes podem ser substituídos uns pelos outros por meio de atribuição simples (sem necessidade de funções de conversão).

Por exemplo:

Sub ByteToStringAndBack()

Dim str As String
str = "Hello, World!"

Dim byt() As Byte
byt = str

Debug.Print byt(0)  ' 72

Dim str2 As String
str2 = byt

Debug.Print str2    ' Hello, World!

End Sub

Para poder codificar caracteres Unicode, cada caractere na string ocupa dois bytes na matriz, com o byte menos significativo primeiro. Por exemplo:

Sub UnicodeExample()

Dim str As String
str = ChrW(&H2123) & "."  ' Versicle character and a dot

Dim byt() As Byte
byt = str

Debug.Print byt(0), byt(1), byt(2), byt(3)  ' Prints: 35,33,46,0

End Sub

Moeda #

Dim Value As Currency

Uma moeda é um tipo de dados de ponto flutuante de 64 bits assinado semelhante a um Double, mas dimensionado em 10.000 para dar maior precisão aos 4 dígitos à direita do ponto decimal. Uma variável Currency pode armazenar valores de -922.337.203.685.477,5808 a 922.337.203.685.477,5807, fornecendo a maior capacidade de qualquer tipo intrínseco em um aplicativo de 32 bits. Como o nome do tipo de dados indica, é considerado uma prática recomendada usar esse tipo de dados ao representar cálculos monetários, pois o dimensionamento ajuda a evitar erros de arredondamento.

A função de conversão para uma moeda é CCur().

Decimal #

Dim Value As Variant
Value = CDec(1.234)

'Set Value to the smallest possible Decimal value
Value = CDec("0.0000000000000000000000000001")

O tipo de dados Decimal está somente disponível como um subtipo de Variant, então você deve declarar qualquer variável que precise conter um Decimal como Variant e depois atribuir um Decimal valor usando a função CDec. A palavra-chave Decimal é uma palavra reservada (o que sugere que o VBA acabaria por adicionar suporte de primeira classe para o tipo), então Decimal não pode ser usado como uma variável ou nome de procedimento.

O tipo Decimal requer 14 bytes de memória (além dos bytes exigidos pela Variant pai) e pode armazenar números com até 28 casas decimais. Para números sem casas decimais, o intervalo de valores permitidos é -79.228.162.514.264.337.593.543.950.335 a +79.228.162.514.264.337.593.543.950.335 inclusive. Para números com o máximo de 28 casas decimais, o intervalo de valores permitidos é de -7,9228162514264337593543950335 a +7,9228162514264337593543950335 inclusive.

Inteiro #

Dim Value As Integer

Um Integer é um tipo de dados de 16 bits assinado. Ele pode armazenar números inteiros no intervalo de -32.768 a 32.767 e tentar armazenar um valor fora desse intervalo resultará no erro de tempo de execução 6: estouro.

Os inteiros são armazenados na memória como valores [little-endian] (https://en.wikipedia.org/wiki/Endianness) com os negativos representados como um [complemento de dois] (https://en.wikipedia.org/wiki /Dois). %27s_complemento).

Observe que, em geral, é melhor usar um Long em vez de um Integer, a menos que o tipo menor é membro de um Type ou é exigido (por uma convenção de chamada de API ou por algum outro motivo) ter 2 bytes. Na maioria dos casos, o VBA trata os inteiros como 32 bits internamente, portanto, geralmente não há vantagem em usar o tipo menor. Além disso, há uma penalidade de desempenho incorrida toda vez que um tipo Integer é usado, pois é silenciosamente convertido como Long.

A função de conversão para converter para um inteiro é CInt(). Para conversões de tipos de ponto flutuante, o resultado é arredondado para o valor inteiro mais próximo com 0,5 de arredondamento para cima.

Grandes #

Dim Value As Long

Um Long é um tipo de dados de 32 bits com sinal. Ele pode armazenar números inteiros no intervalo de -2.147.483.648 a 2.147.483.647 e tentar armazenar um valor fora desse intervalo resultará em erro de tempo de execução 6: estouro.

Longos são armazenados na memória como valores little-endian com negativos representados como um [complemento de dois](https://en.wikipedia.org/wiki/Two %27s_complemento).

Observe que, como um Long corresponde à largura de um ponteiro em um sistema operacional de 32 bits, os Longs são comumente usados ​​para armazenar e passar ponteiros para e de funções da API.

A função de conversão para converter em Long é CLng(). Para conversões de tipos de ponto flutuante, o resultado é arredondado para o valor inteiro mais próximo com 0,5 de arredondamento para cima.

Solteiro #

Dim Value As Single

Um Single é um tipo de dados de ponto flutuante de 32 bits assinado. Ele é armazenado internamente usando uma memória little-endian IEEE 754 disposição. Como tal, não há um intervalo fixo de valores que podem ser representados pelo tipo de dados - o que é limitado é a precisão do valor armazenado. Um Single pode armazenar um valor inteiro valores no intervalo de -16.777.216 a 16.777.216 sem perda de precisão. A precisão dos números de ponto flutuante depende do expoente.

Um Single transbordará se for atribuído um valor maior que aproximadamente 2128. Ele não transbordará com expoentes negativos, embora a precisão utilizável seja questionável antes que o limite superior seja atingido.

Tal como acontece com todos os números de ponto flutuante, deve-se tomar cuidado ao fazer comparações de igualdade. A melhor prática é incluir um valor delta apropriado para a precisão necessária.

A função de conversão para um Single é CSng().

Dobro #

Dim Value As Double

Um Double é um tipo de dados de ponto flutuante de 64 bits assinado. Assim como o Single, ele é armazenado internamente usando um [little-endian](https://en .wikipedia.org/wiki/Endianness) IEEE 754 layout de memória e as mesmas precauções em relação à precisão devem ser tomadas. Um Double pode armazenar valores inteiros no intervalo de -9.007.199.254.740.992 a 9.007.199.254.740.992 sem perda de precisão. A precisão dos números de ponto flutuante depende do expoente.

Um Double transbordará se for atribuído um valor maior que aproximadamente 21024. Ele não transbordará com expoentes negativos, embora a precisão utilizável seja questionável antes que o limite superior seja atingido.

A função de conversão para um Double é CDbl().

Encontro #

Dim Value As Date

Um tipo de data é representado internamente como um tipo de dados de ponto flutuante de 64 bits assinado com o valor à esquerda do decimal representando o número de dias a partir da data de época de 30 de dezembroth de 1899 (embora veja o nota abaixo). O valor à direita do decimal representa a hora como um dia fracionário. Assim, um inteiro Date teria um componente de tempo de 12:00:00AM e x.5 teria um componente de tempo de 12:00:00PM.

Os valores válidos para datas são entre 1º de janeirost 100 e 31 de dezembrost 9999. Como um Double tem um intervalo maior, é possível estourar uma data atribuindo valores fora de esse intervalo.

Como tal, pode ser usado alternadamente com um Double para cálculos de data:

Dim MyDate As Double
MyDate = 0                                  'Epoch date.
Debug.Print Format$(MyDate, "yyyy-mm-dd")   'Prints 1899-12-30.
MyDate = MyDate + 365
Debug.Print Format$(MyDate, "yyyy-mm-dd")   'Prints 1900-12-30.

A função de conversão para uma data é CDate(), que aceita qualquer representação de data/hora de string de tipo numérico. É importante observar que as representações de strings de datas serão convertidas com base na configuração de localidade atual em uso, portanto, conversões diretas devem ser evitadas se o código for portátil.

Longo longo #

Dim Value As LongLong

Um LongLong é um tipo de dados assinado de 64 bits e está disponível apenas em aplicativos de 64 bits. Ele não está disponível em aplicativos de 32 bits executados em sistemas operacionais de 64 bits. Ele pode armazenar valores inteiros no intervalo de -9.223.372.036.854.775.808 a 9.223.372.036.854.775.807 e tentar armazenar um valor fora desse intervalo resultará em erro de tempo de execução 6: estouro.

LongLongs são armazenados na memória como valores little-endian com negativos representados como um [complemento de dois](https://en.wikipedia.org/wiki/Two %27s_complemento).

O tipo de dados LongLong foi introduzido como parte do suporte ao sistema operacional de 64 bits do VBA. Em aplicativos de 64 bits, esse valor pode ser usado para armazenar e passar ponteiros para APIs de 64 bits.

A função de conversão para um LongLong é CLngLng(). Para conversões de tipos de ponto flutuante, o resultado é arredondado para o valor inteiro mais próximo com 0,5 de arredondamento para cima.

LongPtr #

Dim Value As LongPtr

O LongPtr foi introduzido no VBA para suportar plataformas de 64 bits. Em um sistema de 32 bits, ele é tratado como um Long e em sistemas de 64 bits é tratado como a LongLong.

Seu uso principal é fornecer uma maneira portátil de armazenar e passar ponteiros em ambas as arquiteturas (consulte https://www.wikiod.com/pt/vba/compilacao-condicional Changing-code-behavior-at-compile-time #t=20160724185822285365).

Embora seja tratado pelo sistema operacional como um endereço de memória quando usado em chamadas de API, deve-se notar que o VBA o trata como tipo assinado (e, portanto, sujeito a estouro de não assinado para assinado). Por esta razão, qualquer aritmética de ponteiro executada usando LongPtrs não deve usar comparações > ou <. Essa “peculiaridade” também possibilita que a adição de deslocamentos simples apontando para endereços válidos na memória possa causar erros de estouro, portanto, deve-se ter cuidado ao trabalhar com ponteiros no VBA.

A função de conversão para um LongPtr é CLngPtr(). Para conversões de tipos de ponto flutuante, o resultado é arredondado para o valor inteiro mais próximo com 0,5 arredondando para cima (embora, como geralmente é um endereço de memória, usá-lo como destino de atribuição para um cálculo de ponto flutuante seja perigoso na melhor das hipóteses).
