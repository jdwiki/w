Passando Argumentos ByRef ou ByVal

Os modificadores ByRef e ByVal fazem parte da assinatura de um procedimento e indicam como um argumento é passado para um procedimento. No VBA, um parâmetro é passado ByRef a menos que especificado de outra forma (ou seja, ByRef é implícito se ausente).

Observação Em muitas outras linguagens de programação (incluindo VB.NET), os parâmetros são passados ​​implicitamente por valor se nenhum modificador for especificado: considere especificar os modificadores ByRef explicitamente para evitar possíveis confusões.

Passando matrizes #

Matrizes devem ser passadas por referência. Este código compila, mas gera o erro de tempo de execução 424 “Object Required”:

Public Sub Test()
    DoSomething Array(1, 2, 3)
End Sub

Private Sub DoSomething(ByVal foo As Variant)
    foo.Add 42
End Sub

Este código não compila:

Private Sub DoSomething(ByVal foo() As Variant) 'ByVal is illegal for arrays
    foo.Add 42
End Sub

Passando Variáveis ​​Simples ByRef e ByVal #

Passar ByRef ou ByVal indica se o valor real de um argumento é passado para o CalledProcedure pelo CallingProcedure, ou se uma referência (chamada de ponteiro em alguns outros idiomas) é passada para o CalledProcedure.

Se um argumento é passado ByRef, o endereço de memória do argumento é passado para o CalledProcedure e qualquer modificação nesse parâmetro pelo CalledProcedure é feita no valor do CallingProcedure.

Se um argumento for passado ByVal, o valor real, não uma referência à variável, será passado para o CalledProcedure.

Um exemplo simples ilustrará isso claramente:

Sub CalledProcedure(ByRef X As Long, ByVal Y As Long)
    X = 321
    Y = 654
End Sub

Sub CallingProcedure()
    Dim A As Long
    Dim B As Long
    A = 123
    B = 456

    Debug.Print "BEFORE CALL => A: " & CStr(A), "B: " & CStr(B)
    ''Result : BEFORE CALL => A: 123 B: 456

    CalledProcedure X:=A, Y:=B

    Debug.Print "AFTER CALL =  A: " & CStr(A), "B: " & CStr(B)
    ''Result : AFTER CALL => A: 321 B: 456
End Sub

Outro exemplo:

Sub Main()
    Dim IntVarByVal As Integer
    Dim IntVarByRef As Integer
    
    IntVarByVal = 5
    IntVarByRef = 10
    
    SubChangeArguments IntVarByVal, IntVarByRef '5 goes in as a "copy". 10 goes in as a reference
    Debug.Print "IntVarByVal: " & IntVarByVal 'prints 5 (no change made by SubChangeArguments)
    Debug.Print "IntVarByRef: " & IntVarByRef 'prints 99 (the variable was changed in SubChangeArguments)
End Sub

Sub SubChangeArguments(ByVal ParameterByVal As Integer, ByRef ParameterByRef As Integer)
    ParameterByVal = ParameterByVal + 2 ' 5 + 2 = 7 (changed only inside this Sub)
    ParameterByRef = ParameterByRef + 89 ' 10 + 89 = 99 (changes the IntVarByRef itself - in the Main Sub)
End Sub

PorRef #

Modificador padrão #

Se nenhum modificador for especificado para um parâmetro, esse parâmetro será transmitido implicitamente por referência.

Public Sub DoSomething1(foo As Long)
End Sub

Public Sub DoSomething2(ByRef foo As Long)
End Sub

O parâmetro foo é passado ByRef em DoSomething1 e DoSomething2.

Cuidado! Se você está vindo para o VBA com experiência em outras linguagens, este é provavelmente o comportamento exatamente oposto ao que você está acostumado. Em muitas outras linguagens de programação (incluindo VB.NET), o modificador implícito/padrão passa parâmetros por valor.

Passando por referência #

  • Quando um valor é passado ByRef, o procedimento recebe uma referência ao valor.

    Public Sub Test()
    Dim foo As Long
    foo = 42
    DoSomething foo
    Debug.Print foo
End Sub

Private Sub DoSomething(ByRef foo As Long)
    foo = foo * 2
End Sub

    Calling the above Test procedure outputs 84. DoSomething is given foo and receives a reference to the value, and therefore works with the same memory address as the caller.

  • Quando uma referência é passada ByRef, o procedimento recebe uma referência para o ponteiro.

    Public Sub Test()
    Dim foo As Collection
    Set foo = New Collection
    DoSomething foo
    Debug.Print foo.Count
End Sub

Private Sub DoSomething(ByRef foo As Collection)
    foo.Add 42
    Set foo = Nothing
End Sub

    The above code raises [run-time error 91](https://www.wikiod.com/pt/vba/erros-de-tempo-de-execucao-do-vba#Erro em tempo de execução ‘91’: variável de objeto ou com variável de bloco não definida), because the caller is calling the Count member of an object that no longer exists, because DoSomething was given a reference to the object pointer and assigned it to Nothing before returning.

Forçando ByVal no local da chamada #

Usando parênteses no site da chamada, você pode substituir ByRef e forçar um argumento a ser passado ByVal:

Public Sub Test()
    Dim foo As Long
    foo = 42
    DoSomething (foo)
    Debug.Print foo
End Sub

Private Sub DoSomething(ByRef foo As Long)
    foo = foo * 2
End Sub

O código acima gera 42, independentemente de ByRef ser especificado implícita ou explicitamente.

Cuidado! Por causa disso, usar parênteses estranhos em chamadas de procedimento pode facilmente introduzir bugs. Preste atenção ao espaço em branco entre o nome do procedimento e a lista de argumentos:

bar = DoSomething(foo) ‘chamada de função, sem espaço em branco; parênteses fazem parte da lista de argumentos DoSomething (foo) ‘procure call, note whitespace; parênteses NÃO fazem parte da lista de argumentos A chamada do procedimento DoSomething foo ’não força o parâmetro foo a ser ByVal

PorVal #

Passando por valor #

  • Quando um valor é passado ByVal, o procedimento recebe uma cópia do valor.

    Public Sub Test()
    Dim foo As Long
    foo = 42
    DoSomething foo
    Debug.Print foo
End Sub

Private Sub DoSomething(ByVal foo As Long)
    foo = foo * 2
End Sub

    Calling the above Test procedure outputs 42. DoSomething is given foo and receives a copy of the value. The copy is multiplied by 2, and then discarded when the procedure exits; the caller’s copy was never altered.

  • Quando uma referência é passada ByVal, o procedimento recebe uma cópia do ponteiro.

    Public Sub Test()
    Dim foo As Collection
    Set foo = New Collection
    DoSomething foo
    Debug.Print foo.Count
End Sub

Private Sub DoSomething(ByVal foo As Collection)
    foo.Add 42
    Set foo = Nothing
End Sub

    Calling the above Test procedure outputs 1. DoSomething is given foo and receives a copy of the pointer to the Collection object. Because the foo object variable in the Test scope points to the same object, adding an item in DoSomething adds the item to the same object. Because it’s a copy of the pointer, setting its reference to Nothing does not affect the caller’s own copy.
