Acessando o Editor do Visual Basic no Microsoft Office #

Você pode abrir o editor VB em qualquer um dos aplicativos do Microsoft Office pressionando Alt+F11 ou indo para a guia Desenvolvedor e clicando no botão “Visual Basic”. Se você não vir a guia Desenvolvedor na Faixa de Opções, verifique se ela está habilitada.

Por padrão, a guia Desenvolvedor está desabilitada. Para habilitar a guia Desenvolvedor, vá para Arquivo -> Opções, selecione Personalizar Faixa de Opções na lista à esquerda. Na visualização em árvore “Personalizar a faixa de opções” à direita, localize o item da árvore Desenvolvedor e marque a caixa de seleção Desenvolvedor. Clique em Ok para fechar a caixa de diálogo Opções.

A guia Desenvolvedor agora está visível na faixa de opções na qual você pode clicar em “Visual Basic” para abrir o Editor do Visual Basic. Alternativamente, você pode clicar em “Visualizar código” para visualizar diretamente o painel de código do elemento atualmente ativo, por exemplo, Planilha, gráfico, forma.

Você pode usar o VBA para automatizar quase todas as ações que podem ser executadas interativamente (manualmente) e também fornecer funcionalidades que não estão disponíveis no Microsoft Office. O VBA pode criar um documento, adicionar texto a ele, formatá-lo, editá-lo e salvá-lo, tudo sem intervenção humana.

Primeiro Módulo e Olá Mundo #

Para começar a codificar, você deve clicar com o botão direito do mouse no projeto VBA na lista à esquerda e adicionar um novo módulo. Seu primeiro código Hello-World pode ser assim:

Sub HelloWorld()
    MsgBox "Hello, World!"
End Sub

Para testá-lo, pressione o botão Play em sua barra de ferramentas ou simplesmente pressione a tecla F5. Parabéns! Você construiu seu primeiro módulo VBA.

Depuração #

A depuração é uma maneira muito poderosa de examinar mais de perto e corrigir códigos que funcionam incorretamente (ou não funcionam).

Executar código passo a passo #

A primeira coisa que você precisa fazer durante a depuração é parar o código em locais específicos e em seguida, execute-o linha por linha para ver se isso acontece o que é esperado.

  • Breakpoint (F9, Debug - Toggle breakpoint): Você pode adicionar um breakpoint a qualquer linha executada (ex.
  • Você também pode adicionar a palavra-chave Stop a uma linha em branco para que o código pare nesse local em tempo de execução. Isso é útil se, por exemplo, antes de linhas de declaração às quais você não pode adicionar um ponto de interrupção com F9
  • Step into (F8, Debug - Step into): executa apenas uma linha de código, se for uma chamada de uma sub/função definida pelo usuário, então é executada linha por linha.
  • Passar por cima (Shift+F8, Depurar - Passar por cima): executa uma linha de código, não insere subs/funções definidas pelo usuário.
  • Sair (Ctrl+Shift+F8, Depurar - Sair): Sair da sub/função atual (executar o código até o final).
  • Executar para cursor (Ctrl+F8, Debug - Executar para cursor): executa o código até atingir a linha com o cursor.
  • Você pode usar Debug.Print para imprimir linhas para a janela imediata em tempo de execução. Você também pode usar Debug.? como um atalho para Debug.Print

Janela de relógios #

Executar o código linha por linha é apenas o primeiro passo, precisamos saber mais detalhes e uma ferramenta para isso é a janela de observação (Visualizar - Janela de observação), aqui você pode ver os valores das expressões definidas. Para adicionar uma variável à janela de observação:

  • Clique com o botão direito do mouse e selecione “Adicionar relógio”.
  • Clique com o botão direito do mouse na janela de observação, selecione “Adicionar observação”.
  • Vá para Depurar - Adicionar relógio.

Ao adicionar uma nova expressão, você pode escolher se deseja apenas ver seu valor ou também interromper a execução do código quando for verdadeiro ou quando seu valor for alterado.

Janela imediata #

A janela imediata permite que você execute código arbitrário ou imprima itens precedendo-os com a palavra-chave Print ou um único ponto de interrogação “?”

Alguns exemplos:

  • ? ActiveSheet.Name - retorna o nome da planilha ativa
  • Print ActiveSheet.Name - retorna o nome da planilha ativa
  • ? foo - retorna o valor de foo*
  • x = 10 define x como 10*

* Obter/definir valores para variáveis ​​por meio da janela imediata só pode ser feito durante o tempo de execução

Práticas recomendadas de depuração #

Sempre que seu código não funcionar como esperado, a primeira coisa que você deve fazer é lê-lo novamente com atenção, procurando por erros.

Se isso não ajudar, comece a depurá-lo; para procedimentos curtos, pode ser eficiente apenas executá-lo linha por linha, para os mais longos, você provavelmente precisará definir pontos de interrupção ou quebras em expressões observadas, o objetivo aqui é encontrar a linha que não está funcionando conforme o esperado.

Uma vez que você tenha a linha que dá o resultado incorreto, mas o motivo ainda não está claro, tente simplificar as expressões, ou substituir as variáveis ​​por constantes, que podem ajudar a entender se o valor das variáveis ​​está errado.

Se ainda não conseguir resolver, e peça ajuda:

  • Inclua a menor parte possível do seu código para entender seu problema
  • Se o problema não estiver relacionado ao valor das variáveis, substitua-as por constantes. (assim, em vez de Sheets(a*b*c+d^2).Range(addressOfRange) escreva Sheets(4).Range("A2"))
  • Descreva qual linha dá o comportamento errado e o que é (erro, resultado errado…)
