Este artigo mostrará como criar uma classe personalizada completa em VBA. Ele usa o exemplo de um objeto DateRange, porque uma data inicial e final geralmente são passadas juntas para funções.

Adicionando uma propriedade a uma classe #

Um procedimento Property é uma série de instruções que recuperam ou modificam uma propriedade personalizada em um módulo.

Existem três tipos de acessadores de propriedade:

  1. Um procedimento Get que retorna o valor de uma propriedade.
  2. Um procedimento Let que atribui um valor (não-Object) a um objeto.
  3. Um procedimento Set que atribui uma referência Object.

Os acessadores de propriedade geralmente são definidos em pares, usando um Get e um Let/Set para cada propriedade. Uma propriedade com apenas um procedimento Get seria somente leitura, enquanto uma propriedade com apenas um procedimento Let/Set seria somente escrita.

No exemplo a seguir, quatro acessadores de propriedade são definidos para a classe DateRange:

  1. StartDate (leitura/gravação). Valor de data que representa a data anterior em um intervalo. Cada procedimento usa o valor da variável do módulo, mStartDate.

  2. EndDate (leitura/gravação). Valor de data que representa a data posterior em um intervalo. Cada procedimento usa o valor da variável do módulo, mEndDate.

  3. DaysBetween (somente leitura). Valor inteiro calculado que representa o número de dias entre as duas datas. Como existe apenas um procedimento Get, esta propriedade não pode ser modificada diretamente.

  4. RangeToCopy (somente gravação). Um procedimento Set usado para copiar os valores de um objeto DateRange existente.

    Private mStartDate As Date ’ Module variable to hold the starting date Private mEndDate As Date ’ Module variable to hold the ending date

    ’ Return the current value of the starting date Public Property Get StartDate() As Date StartDate = mStartDate End Property

    ’ Set the starting date value. Note that two methods have the name StartDate Public Property Let StartDate(ByVal NewValue As Date) mStartDate = NewValue End Property

    ’ Same thing, but for the ending date Public Property Get EndDate() As Date EndDate = mEndDate End Property

    Public Property Let EndDate(ByVal NewValue As Date) mEndDate = NewValue End Property

    ’ Read-only property that returns the number of days between the two dates Public Property Get DaysBetween() As Integer DaysBetween = DateDiff(“d”, mStartDate, mEndDate) End Function

    ’ Write-only property that passes an object reference of a range to clone Public Property Set RangeToCopy(ByRef ExistingRange As DateRange)

    Me.StartDate = ExistingRange.StartDate Me.EndDate = ExistingRange.EndDate

    End Property

Adicionando funcionalidade a uma classe #

Qualquer Sub, Function ou Property público dentro de um módulo de classe pode ser chamado precedendo a chamada com uma referência de objeto:

Object.Procedure

Em uma classe DateRange, um Sub pode ser usado para adicionar um número de dias à data final:

Public Sub AddDays(ByVal NoDays As Integer)
    mEndDate = mEndDate + NoDays
End Sub

Um Function pode retornar o último dia do próximo final do mês (observe que GetFirstDayOfMonth não seria visível fora da classe porque é privado):

Public Function GetNextMonthEndDate() As Date
    GetNextMonthEndDate = DateAdd("m", 1, GetFirstDayOfMonth())
End Function

Private Function GetFirstDayOfMonth() As Date
    GetFirstDayOfMonth = DateAdd("d", -DatePart("d", mEndDate), mEndDate)
End Function

Os procedimentos podem aceitar argumentos de qualquer tipo, incluindo referências a objetos da classe que está sendo definida.

O exemplo a seguir testa se o objeto DateRange atual tem uma data inicial e uma data final que inclui as datas inicial e final de outro objeto DateRange.

Public Function ContainsRange(ByRef TheRange As DateRange) As Boolean
    ContainsRange = TheRange.StartDate >= Me.StartDate And TheRange.EndDate <= Me.EndDate
End Function

Observe o uso da notação Me como forma de acessar o valor do objeto que executa o código.

Escopo do módulo de classe, instância e reutilização #

Por padrão, um novo módulo de classe é uma classe privada, portanto, está somente disponível para instanciação e uso dentro do VBProject no qual está definido. Você pode declarar, instanciar e usar a classe em qualquer lugar no mesmo projeto:

'Class List has Instancing set to Private
'In any other module in the SAME project, you can use:

Dim items As List
Set items = New List

Mas muitas vezes você escreverá classes que gostaria de usar em outros projetos sem copiar o módulo entre projetos. Se você definir uma classe chamada List em ProjectA e quiser usar essa classe em ProjectB, você precisará executar 4 ações:

  1. Altere a propriedade de instância da classe List em ProjectA na janela Propriedades, de Private para PublicNotCreatable

  2. Crie uma função pública “factory” em ProjectA que cria e retorna uma instância de uma classe List. Normalmente, a função de fábrica inclui argumentos para a inicialização da instância da classe. A função de fábrica é necessária porque a classe pode ser usada pelo ProjectB, mas o ProjectB não pode criar diretamente uma instância da classe do ProjectA.

     Public Function CreateList(ParamArray values() As Variant) As List
     Dim tempList As List
     Dim itemCounter As Long
     Set tempList = New List
     For itemCounter = LBound(values) to UBound(values) 
         tempList.Add values(itemCounter)
     Next itemCounter
     Set CreateList = tempList
 End Function

  3. Em ProjectB adicione uma referência ao ProjectA usando o menu Tools..References....

  4. Em ProjectB, declare uma variável e atribua a ela uma instância de List usando a função de fábrica de ProjectA

     Dim items As ProjectA.List
 Set items = ProjectA.CreateList("foo","bar")

 'Use the items list methods and properties
 items.Add "fizz"
 Debug.Print items.ToString()
 'Destroy the items object
 Set items = Nothing
