Sintaxe #

  • Opção optionName [valor]
  • Opção Explícita
  • Opção Comparar {Texto | Binário | Base de dados}
  • Opção Módulo Privado
  • Opção Base {0 | 1}

Parâmetros #

Opção Detalhe
Explícito Requer declaração de variável no módulo em que está especificado (idealmente todos eles); com esta opção especificada, usar uma variável não declarada (/com erros ortográficos) torna-se um erro de compilação.
Comparar Texto Faz com que as comparações de string do módulo não façam distinção entre maiúsculas e minúsculas, com base na localidade do sistema, priorizando a equivalência alfabética (por exemplo, “a” = “A”).
Comparar Binário Modo de comparação de string padrão. Faz com que as comparações de string do módulo façam distinção entre maiúsculas e minúsculas, comparando strings usando a representação binária/valor numérico de cada caractere (por exemplo, ASCII).
Compare Database (MS-Access apenas) Faz com que as comparações de strings do módulo funcionem da mesma forma que em uma instrução SQL.
Privado Módulo Impede que o membro Public do módulo seja acessado de fora do projeto em que o módulo reside, ocultando efetivamente os procedimentos do aplicativo host (ou seja, não disponível para uso como macros ou funções definidas pelo usuário).
Opção Base 0 Configuração padrão. Define o limite inferior do array implícito para 0 em um módulo. Quando um array é declarado sem um valor limite inferior explícito, 0 será usado.
Opção Base 1 Define o limite inferior do array implícito para 1 em um módulo. Quando um array é declarado sem um valor limite inferior explícito, 1 será usado.

É muito mais fácil controlar os limites de arrays declarando os limites explicitamente ao invés de deixar o compilador retornar a uma declaração Option Base {0|1}. Isso pode ser feito assim:

Dim myStringsA(0 To 5) As String '// This has 6 elements (0 - 5)
Dim myStringsB(1 To 5) As String '// This has 5 elements (1 - 5)
Dim myStringsC(6 To 9) As String '// This has 3 elements (6 - 9)

Opção explícita #

É considerado uma prática recomendada sempre usar Option Explicit no VBA, pois força o desenvolvedor a declarar todas as suas variáveis ​​antes de usar. Isso também traz outros benefícios, como capitalização automática para nomes de variáveis ​​declarados e IntelliSense.

Option Explicit

Sub OptionExplicit()
    Dim a As Integer
    a = 5
    b = 10 '// Causes compile error as 'b' is not declared
End Sub

Definir Requer Declaração de Variável nas Ferramentas ► Opções ► da página de propriedades do Editor do VBE colocará a Opção Explicit na parte superior de cada folha de código recém-criada.

Isso evitará erros de codificação bobos, como erros de ortografia, além de influenciar você a usar o tipo de variável correto na declaração da variável. (Alguns exemplos são fornecidos em <https://www.wikiod.com/pt/excel-vba/praticas-recomendadas-de-vba#SEMPRE use “Opção Explícita”>.)

Base de opções {0 | 1} #

Option Base é usado para declarar o limite inferior padrão de elementos array. Ele é declarado no nível do módulo e é válido apenas para o módulo atual.

Por padrão (e, portanto, se nenhuma Option Base for especificada), a Base é 0. O que significa que o primeiro elemento de qualquer array declarado no módulo tem um índice de 0.

Se Option Base 1 for especificado, o primeiro elemento da matriz terá o índice 1

Exemplo na Base 0: #

Option Base 0

Sub BaseZero()

    Dim myStrings As Variant
    
    ' Create an array out of the Variant, having 3 fruits elements
    myStrings = Array("Apple", "Orange", "Peach")
    
    Debug.Print LBound(myStrings) ' This Prints "0"
    Debug.Print UBound(myStrings) ' This print "2", because we have 3 elements beginning at 0 -> 0,1,2
            
    For i = 0 To UBound(myStrings)
    
        Debug.Print myStrings(i) ' This will print "Apple", then "Orange", then "Peach"
    
    Next i

End Sub

Mesmo Exemplo com Base 1 #

Option Base 1

Sub BaseOne()

    Dim myStrings As Variant
    
    ' Create an array out of the Variant, having 3 fruits elements
    myStrings = Array("Apple", "Orange", "Peach")
    
    Debug.Print LBound(myStrings) ' This Prints "1"
    Debug.Print UBound(myStrings) ' This print "3", because we have 3 elements beginning at 1 -> 1,2,3
            
    For i = 0 To UBound(myStrings)
    
        Debug.Print myStrings(i) ' This triggers an error 9 "Subscript out of range"
    
    Next i

End Sub

O segundo exemplo gerou um Subscrito fora do intervalo (Erro 9) no primeiro estágio do loop porque foi feita uma tentativa de acessar o índice 0 do array, e este índice não existe pois o módulo é declarado com Base 1

O código correto com Base 1 é: #

    For i = 1 To UBound(myStrings)
    
        Debug.Print myStrings(i) ' This will print "Apple", then "Orange", then "Peach"
    
    Next i

Deve-se notar que a função Split sempre cria uma matriz com um índice de elemento baseado em zero, independentemente de qualquer ` Configuração de base de opções. Exemplos de como usar a função Split podem ser encontrados [aqui](https://www.wikiod.com/pt/vba/matrizes#Uso de Split para criar um array a partir de uma string -from-a-string#t=201608041103181263837)

Função Dividir
Retorna um array unidimensional baseado em zero contendo um número especificado de substrings.

No Excel, as propriedades Range.Value e Range.Formula para um intervalo multicelular sempre retornam uma matriz de Variante 2D baseada em 1.

Da mesma forma, no ADO, o método Recordset.GetRows sempre retorna uma matriz 2D baseada em 1.

Uma ‘prática recomendada’ é sempre usar o LBound e [UBound](https://msdn.microsoft.com/ en-us/library/office/gg278658.aspx) para determinar as extensões de uma matriz.

'for single dimensioned array
Debug.Print LBound(arr) & ":" & UBound(arr)
Dim i As Long
For i = LBound(arr) To UBound(arr)
    Debug.Print arr(i)
Next i

'for two dimensioned array
Debug.Print LBound(arr, 1) & ":" & UBound(arr, 1)
Debug.Print LBound(arr, 2) & ":" & UBound(arr, 2)
Dim i As long, j As Long
For i = LBound(arr, 1) To UBound(arr, 1)
    For j = LBound(arr, 2) To UBound(arr, 2)
         Debug.Print arr(i, j)
    Next j
Next i

A Option Base 1 deve estar no topo de cada módulo de código onde uma matriz é criada ou redimensionada se as matrizes forem criadas consistentemente com um limite inferior de 1.

Comparação de opções {Binário | Texto | Base de dados} #

Binário de comparação de opções #

A comparação binária faz todas as verificações de igualdade de strings dentro de um módulo/classe sensível a maiúsculas e minúsculas. Tecnicamente, com esta opção, as comparações de strings são realizadas usando a ordem de classificação das representações binárias de cada caractere.

A < B < E < Z < a < b < e < z

Se nenhuma comparação de opções for especificada em um módulo, Binário será usado por padrão.

Option Compare Binary

Sub CompareBinary()

    Dim foo As String
    Dim bar As String
    
    '// Case sensitive
    foo = "abc"
    bar = "ABC"
    
    Debug.Print (foo = bar) '// Prints "False"
    
    '// Still differentiates accented characters
    foo = "ábc"
    bar = "abc"
    
    Debug.Print (foo = bar) '// Prints "False"
    
    '// "b" (Chr 98) is greater than "a" (Chr 97)
    foo = "a"
    bar = "b"
    
    Debug.Print (bar > foo) '// Prints "True"
    
    '// "b" (Chr 98) is NOT greater than "á" (Chr 225)
    foo = "á"
    bar = "b"
    
    Debug.Print (bar > foo) '// Prints "False"

End Sub

Opção Comparar Texto #

Option Compare Text faz com que todas as comparações de strings dentro de um módulo/classe usem uma comparação insensível a maiúsculas e minúsculas.

(A | a) < (B | b) < (Z | z)

Option Compare Text

Sub CompareText()

    Dim foo As String
    Dim bar As String
    
    '// Case insensitivity
    foo = "abc"
    bar = "ABC"
    
    Debug.Print (foo = bar) '// Prints "True"
    
    '// Still differentiates accented characters
    foo = "ábc"
    bar = "abc"
    
    Debug.Print (foo = bar) '// Prints "False"
    
    '// "b" still comes after "a" or "á"
    foo = "á"
    bar = "b"
    
    Debug.Print (bar > foo) '// Prints "True"

End Sub

Opção Comparar Banco de Dados #

A opção Comparar banco de dados só está disponível no MS Access. Ele define o módulo/classe para usar as configurações atuais do banco de dados para determinar se deve usar o modo Texto ou Binário.

Observação: O uso dessa configuração é desencorajado, a menos que o módulo seja usado para escrever UDFs de acesso personalizadas (funções definidas pelo usuário) que devem tratar comparações de texto da mesma maneira que consultas SQL nesse banco de dados.
