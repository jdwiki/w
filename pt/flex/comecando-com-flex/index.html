Instalação ou Configuração (Apache Flex) #

Existem duas maneiras de configurar o Apache Flex SDK. Você pode usar o Apache Flex SDK Installer fornecido, um aplicativo Adobe AIR que automatiza o processo (no Windows ou OS X/macOS). Ou você pode instalá-lo manualmente, o que obviamente exige maior conforto com sua plataforma, mas oferece mais flexibilidade nos detalhes de instalação.

Instalando o Apache Flex SDK usando o instalador do Apache Flex #

  1. Baixe o binário installer para o seu sistema operacional (Windows, Mac or Linux).
  2. Execute o instalador
  3. Selecione a versão do SDK que deseja baixar. O projeto Apache Flex sugere que você baixe a versão mais recente lançada.
  4. Selecione a versão do Flash Player. O projeto Flex sugere que você o deixe como padrão. Clique em Avançar.
  5. Selecione um diretório no qual deseja instalar o SDK. Clique em Avançar.
  6. Verifique todos os contratos de licença. Clique em Instalar.
  7. Aguarde o instalador baixar e instalar o Apache Flex SDK.

Instalando o Apache Flex SDK manualmente #

NOTA: Este pode ser um processo demorado e complicado.

Prepare seu sistema #

  1. Baixe e instale o Java JDK mais recente
  2. Baixe e instale a versão mais recente do Ant
  3. Baixe e instale a versão mais recente do Git
  4. Baixe e instale o JUnit mais recente

Configura a estrutura de pastas #

  1. Crie a seguinte estrutura de diretórios em seu computador para o Flex SDK, geralmente armazenado no diretório raiz ou Documentos.
/ApacheFlex
/ApacheFlex/dependencies
/ApacheFlex/source

Fonte de download #

  1. Abra um terminal e altere o diretório para:
/ApacheFlex/source
  1. Execute o seguinte comando git para baixar a origem do Flex SDK:
git clone https://git-wip-us.apache.org/repos/asf/flex-sdk.git flex-sdk
  1. Execute o seguinte comando git para baixar a fonte TLF:
git clone https://git-wip-us.apache.org/repos/asf/flex-tlf.git flex-tlf
  1. Mude para o diretório flex-sdk:
/ApacheFlex/source/flex-sdk
  1. Mude para a ramificação ‘develop’ (opcional):
git checkout develop

Edite as propriedades do ambiente #

Para que os scripts de compilação funcionem, você precisa definir as Variáveis ​​de ambiente. O arquivo de propriedades de ambiente contém as variáveis ​​de ambiente que apontam para os locais das dependências que você usará ao longo do processo. Eles são usados ​​pelos vários scripts de compilação para executar e compilar o SDK. Você adicionará variáveis ​​de ambiente adicionais ao longo deste tutorial. Quando você executar o script de compilação posteriormente, o script validará os caminhos fornecidos e avisará se estiverem incorretos.

  1. Crie o arquivo de propriedades do ambiente:

No diretório flex-sdk crie uma cópia do arquivo env-template.properties e nomeie-o como env.properties.

Como você já instalou o Java e o Ant, defina o caminho deles anexando o seguinte no final do arquivo env.properties:

env.JAVA_HOME=/System/Library/Frameworks/JavaVM.framework/Versions/Current/Commands/java

env.ANT_HOME=/Applications/Ant/bin
  1. Em seguida, adicione o caminho ao diretório TLF:
env.TLF_HOME=/ApacheFlex/source/flex-tlf

Adicionar dependências obrigatórias #

SDK do Adobe AIR #

  1. Baixe o Air SDK Flex Edition (Certifique-se de baixar a edição feita especificamente para Flex).
  2. Extraia para:
/ApacheFlex/dependencies/AIRSDK/4.0

onde o nome do diretório final se refere à versão do AIR SDK que você baixou.

  1. Defina a variável de ambiente AIR_HOME para o caminho do AIR SDK que você escolheu na etapa anterior. Por exemplo,
env.ANT_HOME=/ApacheFlex/dependencies/AIRSDK/4.0

Projetor Adobe Flash Player #

  1. Baixe o projetor do Flash Player
  2. Execute o instalador. O programa de instalação do Projector Content Debugger não é instalado em nenhum lugar (pelo menos para mim). Depois de executar o instalador, você deve copiá-lo para o local desejado. Neste caso, copie-o para o diretório /ApacheFlex/dependencies/player. Observação: no OSX, anexe ‘.app/Contents/MacOS/Flash Player Debugger’ ao caminho para o Flash Player.
/ApacheFlex/dependencies/player/Flash Player.app/Contents/MacOS/Flash Player Debugger
  1. Defina a variável de ambiente FLASHPLAYER_DEBUGGER para o caminho de instalação do projetor.
env.FLASHPLAYER_DEBUGGER=/ApacheFlex/dependencies/player/Flash Player.app/Contents/MacOS/Flash Player Debugger

Você também pode usar o plug-in do Flash Player. Geralmente está localizado aqui: /Library/Internet Plug-ins/Flash Player/ O caminho completo seria: /Library/Internet Plug-ins/Flash Player.app/Contents/MacOS/Flash Player Debugger

Adobe Flash Player playerglobal.swc #

  1. Download playerglobal.swc - Está na mesma página que o depurador de conteúdo que você acabou de baixar.
  2. Copie-o para:
/ApacheFlex/dependencies/PlayerGlobal/player/12.0

onde o número corresponde à versão que você baixou. 3. Renomeie o arquivo copiado para ‘playerglobal.swc’ 4. Defina a variável de ambiente PLAYERGLOBAL_HOME para:

env.PLAYERGLOBAL_HOME=/ApacheFlex/Dependencies/PlayerGlobal/player

Kit de ferramentas do Adobe Pixel Bender #

  1. Faça o download do Pixel Bender para Windows ou para Mac
  2. Instale o Pixel Bender. Quando você instala o programa, ele oferece a opção de escolher o local de instalação. Copie este local antes de instalar.
  3. Defina a variável de ambiente PIXELBENDER_HOME para o diretório de instalação.

Preparar Sistema #

Para teste, o arquivo mm.cfg do Adobe Flash Player deve ter as propriedades apropriadas definidas e um arquivo FlashPlayerTrust deve permitir que SWFs locais acessem arquivos locais.

  1. Abra o arquivo mm.cfg em um editor. A localização do arquivo mm.cfg depende do seu sistema operacional. Veja esta página.
  2. Se não existir, inicie manualmente o depurador de conteúdo do Adobe Flash Player Projector e ele deverá criar o mm.cfg nesse momento.
  3. Adicione ou verifique se as seguintes entradas estão no arquivo:
ErrorReportingEnable=1
TraceOutputFileEnable=1
  1. No mesmo diretório que o arquivo mm.cfg está o diretório FlashPlayerTrust.
  2. Abra esse diretório e crie um novo arquivo de texto chamado ApacheFlex.cfg.
  3. Nesse arquivo, adicione o diretório pai do Apache Flex SDK. Por exemplo:
#Path to Apache Flex Source
/ApacheFlex/source
  1. Salve o arquivo.

Defina as propriedades de compilação #

  1. No diretório flex-sdk está o arquivo de propriedades de construção. Este contém os números de versão do software que você baixou.
  2. Abra este arquivo e defina os seguintes valores para a versão do Flash Player e do AIR SDK que você está usando:
# Flash player version for which player global swc to use
playerglobal.version = 12.0

# AIR version number
air.version = 4.0

Esses valores são usados ​​na criação do caminho para o SDK. Então, se você salvou o playerglobal.swc no seguinte diretório: /ApacheFlex/dependencies/PlayerGlobal/player/12.0 o valor do playerglobal.version seria 12.0.

Observação: como esse valor cria o caminho, certifique-se de usar o valor completo, 12.0, em vez de 12, se o caminho o contiver.

Observação: se você estiver usando uma nova versão do Flash Player que ainda não foi mencionada na versão do Flex SDK com a qual está trabalhando, poderá receber a mensagem: Error: configuration variable ‘swf-version’ value contains unknown token ‘playerglobal.swfversion’ Você precisará atualizar o arquivo /ApacheFlex/source/flex-sdk/frameworks/build.xml com um código de verificação de versão atualizado.
