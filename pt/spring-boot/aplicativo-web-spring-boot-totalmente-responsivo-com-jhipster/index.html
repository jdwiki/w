Crie o aplicativo Spring Boot usando jHipster no Mac OS #

O jHipster permite inicializar um aplicativo da Web Spring Boot com um back-end de API REST e um front-end AngularJS e Twitter Bootstrap.

Mais sobre o jHipster aqui: Documentação do jHipster

Instale a bebida:

 /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Veja informações adicionais sobre como instalar o brew aqui: Install Brew

Instalar Gradle

Gradle é um sistema de gerenciamento e compilação de dependências.

 brew install gradle

Instale o Git

Git é uma ferramenta de controle de versão

 brew install git

Instalar NodeJS

O NodeJS dá acesso ao npm, o gerenciador de pacotes do nó que é necessário para instalar outras ferramentas.

 brew install node

Instale Yeoman

Yeoman é um gerador

 npm install -g yo

Instalar Bower

Bower é uma ferramenta de gerenciamento de dependências

 npm install -g bower

Instale o Gulp

Gulp é um executor de tarefas

 npm install -g gulp

Instale jHipster Yeoman Generator

Este é o gerador jHipster

 npm install -g generator-jhipster

Crie um aplicativo

Abra uma janela do Terminal.

Navegue até o diretório raiz onde você manterá seus projetos. Crie um diretório vazio no qual você criará seu aplicativo

 mkdir myapplication

Vá para esse diretório

 cd myapplication/

Para gerar seu aplicativo, digite

 yo jhipster

Você será solicitado com as seguintes perguntas

Que tipo de aplicativo você gostaria de criar?

Seu tipo de aplicativo depende se você deseja usar uma arquitetura de microsserviços ou não. Uma explicação completa sobre microsserviços está disponível aqui, se não tiver certeza, use o padrão “Aplicativo monolítico”.

Escolha Aplicativo monolítico por padrão se não tiver certeza

Qual é o nome do seu pacote Java padrão?

Seu aplicativo Java usará isso como seu pacote raiz.

Que tipo de autenticação você gostaria de usar?

Use o Spring Security básico baseado em sessão por padrão se você não tiver certeza

Que tipo de banco de dados você gostaria de usar?

Qual banco de dados de desenvolvimento você gostaria de usar?

Use H2 por padrão se não tiver certeza

H2, rodando na memória. Esta é a maneira mais fácil de usar o JHipster, mas seus dados serão perdidos quando você reiniciar o servidor.

Deseja usar o cache de 2º nível do Hibernate?

Hibernate é o provedor JPA usado pelo JHipster. Por motivos de desempenho, é altamente recomendável usar um cache e ajustá-lo de acordo com as necessidades do seu aplicativo. Se você optar por fazer isso, poderá usar ehcache (cache local) ou Hazelcast (cache distribuído, para uso em um ambiente clusterizado)

Deseja usar um mecanismo de pesquisa em seu aplicativo? O Elasticsearch será configurado usando o Spring Data Elasticsearch. Você pode encontrar mais informações em nosso guia Elasticsearch.

Escolha não se não tiver certeza

Deseja usar sessões HTTP em cluster?

Por padrão, o JHipster usa uma sessão HTTP apenas para armazenar as informações de autenticação e autorizações do Spring Security. Claro, você pode optar por colocar mais dados em suas sessões HTTP. O uso de sessões HTTP causará problemas se você estiver executando em um cluster, especialmente se não usar um balanceador de carga com “sessões fixas”. Se você deseja replicar suas sessões dentro de seu cluster, escolha esta opção para configurar o Hazelcast.

Escolha não se não tiver certeza

Deseja usar WebSockets? Websockets podem ser habilitados usando Spring Websocket. Também fornecemos uma amostra completa para mostrar como usar a estrutura com eficiência.

Escolha não se não tiver certeza

Você gostaria de usar Maven ou Gradle? Você pode construir seu aplicativo Java gerado com Maven ou Gradle. Maven é mais estável e mais maduro. Gradle é mais flexível, mais fácil de estender e mais hype.

Escolha Gradle se não tiver certeza

Você gostaria de usar o pré-processador de folha de estilo LibSass para seu CSS? Node-sass é uma ótima solução para simplificar o design de CSS. Para ser usado com eficiência, você precisará executar um servidor Gulp, que será configurado automaticamente.

Escolha não se não tiver certeza

Deseja ativar o suporte à tradução com o Angular Translate? Por padrão, o JHipster oferece excelente suporte à internacionalização, tanto no lado do cliente com o Angular Translate quanto no lado do servidor. No entanto, a internacionalização adiciona um pouco de sobrecarga e é um pouco mais complexa de gerenciar, portanto, você pode optar por não instalar esse recurso.

Escolha não se não tiver certeza

Escolha nenhum se não tiver certeza. Você terá acesso ao junit e ao Karma por padrão.
