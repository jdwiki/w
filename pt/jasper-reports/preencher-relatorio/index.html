Parâmetros #

Parâmetros Coluna
jasperPrint A saída do processo de preenchimento que pode ser exportada para o formato desejado
relatórioModelo O arquivo de design compilado .jasper
parâmetros O parâmetro Map, que se definido pode ser referência dentro do relatório por $P{key}
fonte de dados A net.sf.jasperreports.engine.JRDataSource
conexão Uma conexão de banco de dados java.sql.Connection

Preencha o modelo JasperReport usando Java #

Requisitos Comuns #

Todos os relatórios, independentemente de como os dados são apresentados, seguem um caminho para o modelo de relatório e um mapa de parâmetros. As variáveis ​​são usadas em todos os exemplos a seguir:

// Parameters passed into the report.
Map<String, Object> parameters = new HashMap<>();

// Arbitrary parameter passed into the report.
parameters.put("KEY", "Value");

// The compiled report design.
String path = "path/to/template.jasper";

Usar um arquivo .jrxml incorre em uma etapa de compilação extra que não é necessária na maioria das situações. A menos que você tenha escrito um software personalizado para alterar o .jrxml antes da execução do relatório (por exemplo, adicionando ou removendo colunas dinamicamente), use o arquivo .jasper conforme mostrado nos exemplos subsequentes.

Usando uma conexão de banco de dados #

// Establish a database connection.
Connection connection = DriverManager.getConnection(url, username, password); 

// Fill the report, get the JasperPrint that can be exported to desired format.
JasperPrint jasperPrint = JasperFillManager.fillReport(
  path, parameters, connection);

Usando uma fonte de dados personalizada #

// Populate this list of beans as per your requirements.
List<Bean> beans = new ArrayList<>();

// Wrap the beans in a beans in a JRBeanCollectionDataSource.
JRBeanCollectionDataSource datasource = new JRBeanCollectionDataSource(beans);

// Fill the report, get the JasperPrint that can be exported to desired format.
JasperPrint jasperPrint = JasperFillManager.fillReport(
  path, parameters, datasource);

Sem fonte de dados, banda de detalhes não utilizada #

// Fill the report, get the JasperPrint that can be exported to desired format.
JasperPrint jasperPrint = JasperFillManager.fillReport(path, parameters);

Sem uma fonte de dados, o atributo whenNoDataType="AllSectionsNoDetail" no elemento JasperReport deve ser definido, caso contrário será gerado um relatório vazio (em branco).

Com IDE (ambiente de desenvolvimento integrado) #

#JasperSoft Studio

  1. Se a fonte de dados ou conexão de banco de dados for necessária para preencher o relatório, crie seu Adaptador de Dados no Repository Explorer clicando com o botão direito do mouse em “Adaptadores de dados” selecionando “Criar adaptador de dados”

  2. Entre no modo de visualização selecionando a guia Visualizar (nenhum erro de design precisa estar presente)

  3. Selecione a fonte de dados desejada (se nenhuma fonte de dados for necessária, selecione “Um registro vazio”

  4. Defina o parâmetro conforme desejado

  5. Preencha o relatório clicando na seta verde “Executar o relatório”
