Instalação ou Configuração #

O OrientDB está disponível em duas edições:

  • Community Edition é lançado como um projeto de código aberto sob a [licença Apache 2](http://www.apache.org/licenses/LICENSE- 2.0.html). Esta licença permite o uso gratuito irrestrito para projetos de código aberto e comerciais.

  • Enterprise Edition é um software comercial construído sobre o Community Edition. Enterprise é desenvolvido pela mesma equipe que desenvolveu o mecanismo OrientDB. Ele serve como uma extensão do Community Edition, fornecendo recursos Enterprise, como:

    • Non-Stop Backup and Restore
    • Scheduled FULL and Incremental Backups
    • Query Profiler
    • Distributed Clustering configuration
    • Metrics Recording
    • Live Monitoring with configurable Alerts

A Community Edition está disponível como um pacote binário para download ou como código-fonte no GitHub. A licença Enterprise Edition está incluída nas compras de Suporte.

Pré-requisitos

Ambas as edições do OrientDB são executadas em qualquer sistema operacional que implemente a Java Virtual Machine (JVM). Exemplos destes incluem:

  • Linux, todas as distribuições, incluindo ARM (Raspberry Pi, etc.)
  • Mac OS X
  • Microsoft Windows, a partir de 95/NT e posterior
  • Solaris
  • HP-UX
  • IBM AIX

O OrientDB requer Java, versão 1.7 ou superior.

Observação: em contêineres OSGi, o OrientDB usa uma implementação ConcurrentLinkedHashMap fornecida por concurrentlinkedhashmap para criar o cache baseado em LRU. Esta biblioteca usa ativamente o pacote sun.misc que normalmente não é exposto como um pacote de sistema. Para superar esta limitação você deve adicionar a propriedade org.osgi.framework.system.packages.extra com o valor sun.misc à sua lista de propriedades do framework.

Pode ser tão simples quanto passar um argumento para a VM iniciando a plataforma:

$ java -Dorg.osgi.framework.system.packages.extra=sun.misc

Usar Docker #

Se você tiver o Docker instalado em seu computador, esta é a maneira mais fácil de executar o OrientDB. A partir do tipo de linha de comando:

$ docker run -d –-name orientdb -p 2424:2424 -p 2480:2480
   -e ORIENTDB_ROOT_PASSWORD=root orientdb:latest

Onde em vez de “root”, digite a senha do root que você deseja usar.

Instalação Binária #

O OrientDB fornece um pacote binário pré-compilado para instalar o banco de dados em seu sistema. Dependendo do seu sistema operacional, este é um pacote tarado ou compactado que contém todos os arquivos relevantes necessários para executar o OrientDB. Para instalações em desktop, vá para Downloads do OrientDB e selecione o pacote que melhor se adapta ao seu sistema.

Nas instalações do servidor, você pode usar o utilitário wget:

$ wget https://orientdb.com/download.php?file=orientdb-community-2.2.0.tar.gz

Se você usa seu navegador web ou wget, descompacte ou extraia o arquivo baixado em um diretório conveniente para seu uso (por exemplo, /opt/orientdb/ no Linux). Isso cria um diretório chamado orientdb-community-2.2.0 com arquivos e scripts relevantes, que você precisará para executar o OrientDB em seu sistema.

Instalação do código-fonte #

Além de baixar os pacotes binários, você também tem a opção de compilar o OrientDB a partir do código-fonte do Community Edition, disponível no GitHub. Este processo requer que você instale o Git e o Apache Maven em seu sistema.

Para compilar o OrientDB a partir do código-fonte, clone o repositório do Community Edition e execute o Maven (mvn) no diretório recém-criado:

$ git clone https://github.com/orientechnologies/orientdb
$ git checkout develop
$ cd orientdb
$ mvn clean install

É possível pular testes:

$ mvn clean install -DskipTests

A ramificação de desenvolvimento contém código para a próxima versão do OrientDB. Versões estáveis ​​são marcadas no branch master. Para cada versão mantida, o OrientDB tem seu próprio branch hotfix. No momento em que escrevo estas notas, o estado das filiais é:

  • desenvolver: trabalho em andamento para a próxima versão 3.0.x (3.0.x-SNAPSHOT)
  • 2.2.x: hot fix para a próxima versão estável 2.2.x (2.2.x-SNAPSHOT)
  • 2.1.x: hot fix para a próxima versão estável 2.1.x (2.1.x-SNAPSHOT)
  • 2.0.x: hot fix para a próxima versão estável 2.0.x (2.0.x-SNAPSHOT)
  • última tag no master é 2.2.0

O processo de compilação instala todos os jars no repositório local do maven e cria arquivos no módulo distribution dentro do diretório target. No momento da escrita, a construção do branch 2.1.x deu:

$ls -l distribution/target/

O diretório orientdb-community-3.0.0-SNAPSHOT.dir contém a distribuição do OrientDB descompactada. Dê uma olhada em Contribute to OrientDB se você quiser se envolver.

Cada pacote de distribuição contém um banco de dados de exemplo chamado GratefulDeadConcerts. É possível gerar o banco de dados:

Atualizar permissões #

Para sistemas operacionais baseados em Linux, Mac OS X e UNIX, você precisa alterar as permissões em alguns dos arquivos após a compilação da fonte.

$ chmod 755 bin/*.sh
$ chmod -R 777 config

Esses comandos atualizam as permissões de execução em arquivos no diretório config/ e scripts shell em bin/, garantindo que você possa executar os scripts ou programas que você compilou.

Tarefas pós-instalação #

Para usuários de desktop instalando o binário, o OrientDB agora está instalado e pode ser executado através de scripts de shell encontrados no diretório bin do pacote da instalação. Para servidores, há algumas etapas adicionais que você precisa executar para gerenciar o servidor de banco de dados do OrientDB como um serviço. O procedimento para isso varia, dependendo do seu sistema operacional.

  • Instalar como serviço no Unix, Linux e Mac OS X
  • Instalar como serviço no Microsoft Windows

Atualizando #

Quando chega a hora de atualizar para uma versão mais recente do OrientDB, os métodos variam dependendo de como você escolheu instalá-lo em primeiro lugar. Se você instalou a partir de downloads binários, repita o processo de download acima e atualize quaisquer links simbólicos ou atalhos para apontar para o novo diretório.

Para sistemas em que o OrientDB foi construído a partir do código-fonte, baixe o código-fonte mais recente e compile a partir do código-fonte.

$ git pull origin master
$ mvn clean install

Lembre-se de que, ao compilar a partir do código-fonte, você pode alternar entre ramificações para compilar diferentes versões do OrientDB usando o Git. Por exemplo,

$ git checkout 2.2.x
$ mvn clean install

constrói o branch 2.2.x, ao invés do master.

Construindo um único jar executável com OrientDB #

O OrientDB para componentes internos como mecanismos, operadores, fábricas usa Java SPI Service Provider Interface. Isso significa que os jars do OrientDB são enviados com arquivos em META-INF/services que contém a implementação dos componentes. Tenha em mente que ao construir um único jar executável, você deve concatenar o conteúdo dos arquivos com o mesmo nome em diferentes orientdb-*.jar . Se você estiver usando o Maven Shade Plugin, poderá usar o [Service Resource Transformer](https://maven.apache.org/plugins/ maven-shade-plugin/examples/resource-transformers.html#ServicesResourceTransformer) para fazer isso.
