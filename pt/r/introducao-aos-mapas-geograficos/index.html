Criação básica de mapas com map() dos mapas de pacotes #

A função map() do pacote maps fornece um ponto de partida simples para criar mapas com R.

Um mapa-múndi básico pode ser desenhado da seguinte forma:

require(maps)
map()

A cor do contorno pode ser alterada definindo o parâmetro de cor, col, para o nome do caractere ou valor hexadecimal de uma cor:

require(maps)
map(col = "cornflowerblue")

Para preencher massas de terra com a cor em col podemos definir fill = TRUE:

require(maps)
map(fill = TRUE, col = c("cornflowerblue"))

Um vetor de qualquer comprimento pode ser fornecido a col quando fill = TRUE também é definido:

require(maps)
map(fill = TRUE, col = c("cornflowerblue", "limegreen", "hotpink"))

No exemplo acima, as cores de col são atribuídas arbitrariamente a polígonos no mapa representando regiões e as cores são recicladas se houver menos cores do que polígonos.

Também podemos usar codificação de cores para representar uma variável estatística, que opcionalmente pode ser descrita em uma legenda. Um mapa criado como tal é conhecido como “choropleth”.

O exemplo coroplético a seguir define o primeiro argumento de map(), que é database para "county" e "state" para colorir o desemprego usando dados dos conjuntos de dados internos unemp e county .fips ao sobrepor linhas de estado em branco:

require(maps)
if(require(mapproj)) {    # mapproj is used for  projection="polyconic"
  # color US county map by 2009 unemployment rate
  # match counties to map using FIPS county codes
  # Based on J's solution to the "Choropleth Challenge"
  # Code improvements by Hack-R (hack-r.github.io)
  
  # load data
  # unemp includes data for some counties not on the "lower 48 states" county
  # map, such as those in Alaska, Hawaii, Puerto Rico, and some tiny Virginia
  #  cities
  data(unemp)
  data(county.fips)
  
  # define color buckets
  colors = c("paleturquoise", "skyblue", "cornflowerblue", "blueviolet", "hotpink", "darkgrey")
  unemp$colorBuckets <- as.numeric(cut(unemp$unemp, c(0, 2, 4, 6, 8, 10, 100)))
  leg.txt <- c("<2%", "2-4%", "4-6%", "6-8%", "8-10%", ">10%")
  
  # align data with map definitions by (partial) matching state,county
  # names, which include multiple polygons for some counties
  cnty.fips <- county.fips$fips[match(map("county", plot=FALSE)$names,
                                      county.fips$polyname)]
  colorsmatched <- unemp$colorBuckets[match(cnty.fips, unemp$fips)]
  
  # draw map
  par(mar=c(1, 1, 2, 1) + 0.1)
  map("county", col = colors[colorsmatched], fill = TRUE, resolution = 0,
      lty = 0, projection = "polyconic")
  map("state", col = "white", fill = FALSE, add = TRUE, lty = 1, lwd = 0.1,
      projection="polyconic")
  title("unemployment by county, 2009")
  legend("topright", leg.txt, horiz = TRUE, fill = colors, cex=0.6)
}

50 mapas de estado e coropléticos avançados com o Google Viz #

Uma pergunta comum é como justapor (combinar) regiões geográficas fisicamente separadas no mesmo mapa, como no caso de um coropleto descrevendo todos os 50 estados americanos (o continente com o Alasca e o Havaí justapostos).

Criar um mapa atraente de 50 estados é simples ao usar o Google Maps. As interfaces para a API do Google incluem os pacotes googleVis, ggmap e RgoogleMaps.

require(googleVis)

G4 <- gvisGeoChart(CityPopularity, locationvar='City', colorvar='Popularity',
                   options=list(region='US', height=350, 
                                displayMode='markers',
                                colorAxis="{values:[200,400,600,800],
                                 colors:[\'red', \'pink\', \'orange',\'green']}")
) 
plot(G4)

A função gvisGeoChart() requer muito menos codificação para criar um coroplético em comparação com métodos de mapeamento mais antigos, como map() do pacote maps. O parâmetro colorvar permite colorir facilmente uma variável estatística, em um nível especificado pelo parâmetro locationvar. As várias opções passadas para options como uma lista permitem a personalização dos detalhes do mapa, como tamanho (height), forma (markers) e codificação de cores (colorAxis e colors).

Mapas plotados interativos #

O pacote plotly permite vários tipos de gráficos interativos, incluindo mapas. Existem algumas maneiras de criar um mapa em plotly. Forneça você mesmo os dados do mapa (via plot_ly() ou ggplotly()), use os recursos de mapeamento “nativos” do plotly (via plot_geo() ou plot_mapbox()), ou mesmo uma combinação de ambos. Um exemplo de fornecimento do mapa você mesmo seria:

library(plotly)
map_data("county") %>%
    group_by(group) %>%
    plot_ly(x = ~long, y = ~lat) %>%
    add_polygons() %>%
    layout(
        xaxis = list(title = "", showgrid = FALSE, showticklabels = FALSE),
        yaxis = list(title = "", showgrid = FALSE, showticklabels = FALSE)
    )

Para uma combinação de ambas as abordagens, troque plot_ly() por plot_geo() ou plot_mapbox() no exemplo acima. Veja o livro de plotagem para mais exemplos.

O próximo exemplo é uma abordagem “estritamente nativa” que aproveita o atributo layout.geo para definir a estética e o nível de zoom do mapa. Também utiliza o banco de dados world.cities de maps para filtrar as cidades brasileiras e plotá-las em cima do mapa “nativo”.

As principais variáveis: pophé um texto com a cidade e sua população (que é mostrado ao passar o mouse); qé um fator ordenado do quantil da população. ge tem informações para o layout dos mapas. Consulte a documentação do pacote para obter mais informações.

library(maps)
dfb <- world.cities[world.cities$country.etc=="Brazil",]
library(plotly)
dfb$poph <- paste(dfb$name, "Pop", round(dfb$pop/1e6,2), " millions")
dfb$q <- with(dfb, cut(pop, quantile(pop), include.lowest = T))
levels(dfb$q) <- paste(c("1st", "2nd", "3rd", "4th"), "Quantile")
dfb$q <- as.ordered(dfb$q)

ge <- list(
  scope = 'south america',
  showland = TRUE,
  landcolor = toRGB("gray85"),
  subunitwidth = 1,
  countrywidth = 1,
  subunitcolor = toRGB("white"),
  countrycolor = toRGB("white")
)

plot_geo(dfb, lon = ~long, lat = ~lat, text = ~poph,
    marker = ~list(size = sqrt(pop/10000) + 1, line = list(width = 0)),
    color = ~q, locationmode = 'country names') %>%
layout(geo = ge, title = 'Populations<br>(Click legend to toggle)')

Fazendo Mapas HTML Dinâmicos com o Leaflet #

Leaflet é uma biblioteca JavaScript de código aberto para fazer mapas dinâmicos para a web. O RStudio escreveu ligações R para Leaflet, disponíveis através de seu pacote leaflet, construído com [htmlwidgets](http://www.htmlwidgets.org/showcase_leaflet .html). Os mapas de folheto se integram bem aos ecossistemas RMarkdown e Shiny.

A interface é piped, usando uma função leaflet() para inicializar um mapa e funções subsequentes adicionando (ou removendo) camadas do mapa. Muitos tipos de camadas estão disponíveis, desde marcadores com pop-ups até polígonos para criar mapas coropléticos. Variáveis ​​no data.frame passadas para leaflet() são acessadas por meio de cotação ~ no estilo de função.

Para mapear os conjuntos de dados state.name e state.center:

library(leaflet)

data.frame(state.name, state.center) %>% 
    leaflet() %>% 
    addProviderTiles('Stamen.Watercolor') %>% 
    addMarkers(lng = ~x, lat = ~y, 
               popup = ~state.name, 
               clusterOptions = markerClusterOptions())
(Captura de tela; clique para versão dinâmica.)

Mapas de folhetos dinâmicos em aplicativos Shiny #

O pacote Leaflet foi desenvolvido para integrar com Shiny

No ui você chama leafletOutput() e no servidor você chama renderLeaflet()

library(shiny)
library(leaflet)

ui <- fluidPage(
    leafletOutput("my_leaf")
)

server <- function(input, output, session){
    
    output$my_leaf <- renderLeaflet({
        
        leaflet() %>%
            addProviderTiles('Hydda.Full') %>%
            setView(lat = -37.8, lng = 144.8, zoom = 10)
        
    })
    
}

shinyApp(ui, server)

No entanto, entradas reativas que afetam a expressão renderLeaflet farão com que todo o mapa seja redesenhado toda vez que o elemento reativo for atualizado.

Portanto, para modificar um mapa que já está rodando você deve usar a função leafletProxy().

Normalmente você usa leaflet para criar os aspectos estáticos do mapa, e leafletProxy para gerenciar os elementos dinâmicos, por exemplo:

library(shiny)
library(leaflet)

ui <- fluidPage(
    sliderInput(inputId = "slider", 
                label = "values",
                min = 0,
                max = 100,
                value = 0,
                step = 1),
    leafletOutput("my_leaf")
)

server <- function(input, output, session){
    set.seed(123456)
    df <- data.frame(latitude = sample(seq(-38.5, -37.5, by = 0.01), 100),
                     longitude = sample(seq(144.0, 145.0, by = 0.01), 100),
                     value = seq(1,100))
        
    ## create static element
    output$my_leaf <- renderLeaflet({
        
        leaflet() %>%
            addProviderTiles('Hydda.Full') %>%
            setView(lat = -37.8, lng = 144.8, zoom = 8)
        
    })
    
    ## filter data
    df_filtered <- reactive({
        df[df$value >= input$slider, ]
    })
    
    ## respond to the filtered data
    observe({

        leafletProxy(mapId = "my_leaf", data = df_filtered()) %>%
            clearMarkers() %>%   ## clear previous markers
            addMarkers()
    })
    
}
