Sintaxe #

  • lm(formula, data, subset, weights, na.action, method = “qr”, model = TRUE, x = FALSE, y = FALSE, qr = TRUE, singular.ok = TRUE, contrastes = NULL, offset, . ..)

Parâmetros #

Parâmetro Significado
fórmula uma fórmula em notação Wilkinson-Rogers; response ~ ... onde ... contém termos correspondentes a variáveis ​​no ambiente ou no quadro de dados especificado pelo argumento data
dados quadro de dados contendo as variáveis ​​de resposta e de previsão
subconjunto um vetor especificando um subconjunto de observações a ser usado: pode ser expresso como uma declaração lógica em termos das variáveis ​​em data
pesos pesos analíticos (ver secção Pesos acima)
na.ação como lidar com valores ausentes (NA): veja ?na.action
método como fazer o encaixe. As únicas opções são "qr" ou "model.frame" (o último retorna o frame do modelo sem ajustar o modelo, idêntico a especificar model=TRUE)
modelo se deve armazenar a moldura do modelo no objeto ajustado
x se a matriz do modelo deve ser armazenada no objeto ajustado
e se deve armazenar a resposta do modelo no objeto ajustado
qr se deve armazenar a decomposição QR no objeto ajustado
singular.ok se permitir ajustes singulares, modelos com preditores colineares (um subconjunto dos coeficientes será automaticamente definido como NA neste caso
contrastes uma lista de contrastes a serem usados ​​para fatores particulares no modelo; veja o argumento contrasts.arg de ?model.matrix.default. Os contrastes também podem ser definidos com options() (veja o argumento contrasts) ou atribuindo os atributos contrast de um fator (veja ?contrasts)
deslocamento usado para especificar um componente conhecido a priori no modelo. Também pode ser especificado como parte da fórmula. Veja ?model.offset
… argumentos adicionais a serem passados ​​para funções de ajuste de nível inferior (lm.fit() ou lm.wfit())

Regressão linear no conjunto de dados mtcars #

O mtcars [frame de dados] integrado (https://www.wikiod.com/pt/r/quadros-de-dados) contém informações sobre 32 carros, incluindo seu peso, eficiência de combustível (em milhas por galão), velocidade, etc. (Para saber mais sobre o conjunto de dados, use help(mtcars)).

Se estivermos interessados ​​na relação entre eficiência de combustível (mpg) e peso (wt), podemos começar a traçar essas variáveis ​​com:

plot(mpg ~ wt, data = mtcars, col=2)

Os gráficos mostram uma relação (linear)!. Então, se quisermos realizar uma regressão linear para determinar os coeficientes de um modelo linear, usaríamos a função lm:

fit <- lm(mpg ~ wt, data = mtcars)

O ~ aqui significa “explicado por”, então a fórmula mpg ~ wt significa que estamos prevendo mpg conforme explicado por wt. A maneira mais útil de visualizar a saída é com:

summary(fit)

O que dá a saída:

Poderíamos adicionar uma linha ao nosso primeiro gráfico para mostrar o mpg previsto:

abline(fit,col=3,lwd=2)

Também é possível adicionar a equação a esse gráfico. Primeiro, obtenha os coeficientes com coef. Em seguida, usando paste0, recolhemos os coeficientes com as variáveis ​​apropriadas e +/-, para construir a equação. Finalmente, nós o adicionamos ao gráfico usando mtext:

bs <- round(coef(fit), 3) 
lmlab <- paste0("mpg = ", bs[1],
             ifelse(sign(bs[2])==1, " + ", " - "), abs(bs[2]), " wt ")
mtext(lmlab, 3, line=-2)

Ponderação #

Às vezes, queremos que o modelo dê mais peso a alguns pontos de dados ou exemplos do que a outros. Isso é possível especificando o peso para os dados de entrada enquanto aprende o modelo. Geralmente, existem dois tipos de cenários em que podemos usar pesos não uniformes nos exemplos:

  • Pesos analíticos: refletem os diferentes níveis de precisão de diferentes observações. Por exemplo, se analisar dados em que cada observação é a média dos resultados de uma área geográfica, o peso analítico é proporcional ao inverso da variância estimada. Útil ao lidar com médias em dados, fornecendo um peso proporcional ao número de observações. Fonte

  • Pesos de Amostragem (Inverse Probability Weights - IPW): técnica estatística de cálculo de estatísticas padronizada para uma população diferente daquela em que os dados foram coletados. Desenhos de estudo com uma população amostral díspar e população de inferência alvo (população alvo) são comuns na aplicação. Útil ao lidar com dados que possuem valores ausentes. Fonte

A função lm() faz a ponderação analítica. Para pesos de amostragem, o pacote survey é usado para construir um objeto de design de pesquisa e executar svyglm(). Por padrão, o pacote survey usa pesos de amostragem. (OBSERVAÇÃO: lm() e svyglm() com a família gaussian() produzirão as mesmas estimativas pontuais, porque ambos resolvem os coeficientes minimizando os mínimos quadrados ponderados. Eles diferem em como os erros padrão são calculados.)

Usando a função ‘prever’ #

Verificando não linearidade com regressão polinomial #

Às vezes, ao trabalhar com regressão linear, precisamos verificar a não linearidade nos dados. Uma maneira de fazer isso é ajustar um modelo polinomial e verificar se ele se ajusta melhor aos dados do que um modelo linear. Existem outras razões, como as teóricas, que indicam o ajuste de um modelo quadrático ou de ordem superior, pois acredita-se que a relação das variáveis ​​é inerentemente polinomial por natureza.

Vamos ajustar um modelo quadrático para o conjunto de dados mtcars. Para um modelo linear, consulte https://www.wikiod.com/pt/r/modelos-lineares-regressao#Regressão linear no conjunto de dados mtcars

Primeiro fazemos um gráfico de dispersão das variáveis ​​mpg (Miles/gallon), disp (Displacement (cu.in.)) e wt (Weight (1000 lbs)). A relação entre mpg e disp parece não linear.

plot(mtcars[,c("mpg","disp","wt")])

Um ajuste linear mostrará que disp não é significativo.

fit0 = lm(mpg ~ wt+disp, mtcars)
summary(fit0)

#            Estimate Std. Error t value Pr(>|t|)    
#(Intercept) 34.96055    2.16454  16.151 4.91e-16 ***
#wt          -3.35082    1.16413  -2.878  0.00743 ** 
#disp        -0.01773    0.00919  -1.929  0.06362 .  
#---
#Signif. codes:  0 ‘***’ 0.001 ‘**’ 0.01 ‘*’ 0.05 ‘.’ 0.1 ‘ ’ 1
#Residual standard error: 2.917 on 29 degrees of freedom
#Multiple R-squared:  0.7809,    Adjusted R-squared:  0.7658

Então, para obter o resultado de um modelo quadrático, adicionamos I(disp^2). O novo modelo aparece melhor ao olhar para R^2 e todas as variáveis ​​são significativas.

fit1 = lm(mpg ~ wt+disp+I(disp^2), mtcars)

Como temos três variáveis, o modelo ajustado é uma superfície representada por:

mpg = 41,4020-3,4179*wt-0,0824*disp+0,0001277*disp^2

Outra maneira de especificar a regressão polinomial é usar poly com o parâmetro raw=TRUE, caso contrário polinômios ortogonais serão considerados (veja a help(ploy) para mais informações). Obtemos o mesmo resultado usando:

summary(lm(mpg ~ wt+poly(disp, 2, raw=TRUE),mtcars))

Traçando a regressão (base) #

Continuando no exemplo mtcars, aqui está uma maneira simples de produzir um gráfico de sua regressão linear que é potencialmente adequado para publicação.

Primeiro ajuste o modelo linear e

fit <- lm(mpg ~ wt, data = mtcars)

Em seguida, plote as duas variáveis ​​de interesse e adicione a linha de regressão dentro do domínio de definição:

plot(mtcars$wt,mtcars$mpg,pch=18, xlab = 'wt',ylab = 'mpg')
lines(c(min(mtcars$wt),max(mtcars$wt)),
as.numeric(predict(fit, data.frame(wt=c(min(mtcars$wt),max(mtcars$wt))))))

Quase lá! O último passo é adicionar ao gráfico, a equação de regressão, o rsquare bem como o coeficiente de correlação. Isso é feito usando a função vector:

rp = vector('expression',3)
rp[1] = substitute(expression(italic(y) == MYOTHERVALUE3 + MYOTHERVALUE4 %*% x), 
          list(MYOTHERVALUE3 = format(fit$coefficients[1], digits = 2),
                        MYOTHERVALUE4 = format(fit$coefficients[2], digits = 2)))[2]
rp[2] = substitute(expression(italic(R)^2 == MYVALUE), 
             list(MYVALUE = format(summary(fit)$adj.r.squared,dig=3)))[2]
rp[3] = substitute(expression(Pearson-R == MYOTHERVALUE2), 
             list(MYOTHERVALUE2 = format(cor(mtcars$wt,mtcars$mpg), digits = 2)))[2]

legend("topright", legend = rp, bty = 'n')

Avaliação de qualidade #

Depois de construir um modelo de regressão é importante verificar o resultado e decidir se o modelo é apropriado e funciona bem com os dados disponíveis. Isso pode ser feito examinando o gráfico de resíduos, bem como outros gráficos de diagnóstico.

# fit the model
fit <- lm(mpg ~ wt, data = mtcars)
# 
par(mfrow=c(2,1))
# plot model object
plot(fit, which =1:2)

Esses gráficos verificam duas suposições que foram feitas durante a construção do modelo:

  1. Que o valor esperado da variável prevista (neste caso mpg) é dado por uma combinação linear dos preditores (neste caso wt). Esperamos que esta estimativa seja imparcial. Assim, os resíduos devem ser centrados em torno da média para todos os valores dos preditores. Neste caso vemos que os resíduos tendem a ser positivos nas extremidades e negativos no meio, sugerindo uma relação não linear entre as variáveis.
  2. Que a variável real prevista esteja normalmente distribuída em torno de sua estimativa. Assim, os resíduos devem ser normalmente distribuídos. Para dados normalmente distribuídos, os pontos em um gráfico Q-Q normal devem estar na diagonal ou próximos dela. Há uma certa quantidade de inclinação nas extremidades aqui.
