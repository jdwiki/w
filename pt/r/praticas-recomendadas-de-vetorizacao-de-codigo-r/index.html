Por operações de linha #

A chave na vetorização do código R é reduzir ou eliminar “operações por linha” ou despacho de método de funções R.

Isso significa que ao abordar um problema que à primeira vista exige “operações por linha”, como calcular as médias de cada linha, é preciso se perguntar:

  • Quais são as classes dos conjuntos de dados com os quais estou lidando?
  • Existe um código compilado existente que pode fazer isso sem a necessidade de avaliação repetitiva de funções R?
  • Se não, posso fazer essas operações por colunas ao invés de linhas?
  • Finalmente, vale a pena gastar muito tempo desenvolvendo código vetorizado complicado em vez de apenas executar um simples loop apply? Em outras palavras, os dados são grandes/sofisticados o suficiente para que R não consiga lidar com eles com eficiência usando um loop simples?

Deixando de lado o problema de pré-alocação de memória e o crescimento de objetos em loops, vamos nos concentrar neste exemplo em como evitar loops apply, despachar métodos ou reavaliar funções R dentro de loops.

Uma maneira padrão/fácil de calcular a média por linha seria:

apply(mtcars, 1, mean)

Mas podemos fazer melhor? Vamos ver o que aconteceu aqui:

  1. Primeiro, convertemos um data.frame em uma matriz. (Observe que isso acontece dentro da função apply.) Isso é ineficiente e perigoso. uma matriz não pode conter vários tipos de colunas ao mesmo tempo. Portanto, essa conversão provavelmente levará à perda de informações e, às vezes, a resultados enganosos (compare apply(iris, 2, class) com str(iris) ou com sapply(iris, class)).
  2. Em segundo lugar, realizamos uma operação repetidamente, uma vez para cada linha. Ou seja, tivemos que avaliar algumas vezes a função R nrow(mtcars). Neste caso específico, média não é uma função computacionalmente cara, portanto R provavelmente poderia lidar com isso facilmente mesmo para um grande conjunto de dados, mas o que aconteceria se precisássemos calcular o desvio padrão por linha (que envolve uma raiz quadrada cara Operação)? O que nos leva ao próximo ponto:
  3. Nós avaliamos a função R muitas vezes, mas talvez já exista uma versão compilada desta operação?

Na verdade, poderíamos simplesmente fazer:

rowMeans(mtcars)

Isso não envolve operações por linha e, portanto, nenhuma avaliação repetitiva de funções R. No entanto, ainda convertemos um data.frame em uma matriz. Embora o rowMeans tenha um mecanismo de tratamento de erros e não seja executado em um conjunto de dados que ele não pode manipular, ainda tem um custo de eficiência.

rowMeans(iris)
Error in rowMeans(iris) : 'x' must be numeric

Mas ainda assim, podemos fazer melhor? Poderíamos tentar em vez de uma conversão de matriz com tratamento de erros, um método diferente que nos permitirá usar mtcars como um vetor (porque um data.frame é essencialmente uma lista e uma lista é um vetor ).

Reduce(`+`, mtcars)/ncol(mtcars)

Agora, para possível ganho de velocidade, perdemos nomes de colunas e tratamento de erros (incluindo tratamento NA).

Outro exemplo seria calcular a média por grupo, usando a base R poderíamos tentar

aggregate(. ~ cyl, mtcars, mean)

Ainda assim, estamos basicamente avaliando uma função R em um loop, mas o loop agora está oculto em uma função C interna (pouco importa se é um loop C ou R).

Poderíamos evitá-lo? Bem, existe uma função compilada em R chamada rowsum, portanto, poderíamos fazer:

rowsum(mtcars[-2], mtcars$cyl)/table(mtcars$cyl)

Embora tivéssemos que converter para uma matriz primeiro também.

A este ponto podemos questionar se nossa estrutura de dados atual é a mais adequada. Um data.frame é a melhor prática? Ou deve-se apenas mudar para uma estrutura de dados matriz para ganhar eficiência?

As operações por linha ficarão cada vez mais caras (mesmo em matrizes) à medida que começamos a avaliar funções caras a cada vez. Vamos considerar um cálculo de variância por exemplo de linha.

Digamos que temos uma matriz m:

set.seed(100)
m <- matrix(sample(1e2), 10)

Alguém poderia simplesmente fazer:

apply(m, 1, var)
[1]  871.6556  957.5111  699.2111  941.4333 1237.3333  641.8222  539.7889  759.4333  500.4889 1255.6111

Por outro lado, pode-se também vetorizar completamente esta operação seguindo a fórmula de variância

RowVar <- function(x) {
  rowSums((x - rowMeans(x))^2)/(dim(x)[2] - 1)
}
RowVar(m)
[1]  871.6556  957.5111  699.2111  941.4333 1237.3333  641.8222  539.7889  759.4333  500.4889 1255.6111
