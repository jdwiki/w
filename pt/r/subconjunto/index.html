Dado um objeto R, podemos exigir uma análise separada para uma ou mais partes dos dados contidos nele. O processo de obter essas partes dos dados de um determinado objeto é chamado de subconjunto.

Valores ausentes:

Valores ausentes (NAs) usados ​​em subconjuntos com [ return NA desde um índice NA

escolhe um elemento desconhecido e retorna NA no elemento correspondente.

O tipo “padrão” de NA é “lógico” (typeof(NA)), o que significa que, como qualquer vetor “lógico” usado em subconjunto, será reciclado para corresponder ao comprimento do objeto subconjunto . Então x[NA] é equivalente a x[as.logical(NA)] que é equivalente a x[rep_len(as.logical(NA), length(x))] e, consequentemente, retorna um valor ausente (NA) para cada elemento de x. Como um exemplo:

x <- 1:3
x[NA]
## [1] NA NA NA

Ao indexar com “numérico”/“inteiro” NA escolhe um único elemento NA (para cada NA no índice):

x[as.integer(NA)]
## [1] NA

x[c(NA, 1, NA, NA)]
## [1] NA  1 NA NA

Subconjunto fora dos limites:

O operador [, com um argumento passado, permite índices que são > comprimento(x) e retorna NA para vetores atômicos ou NULL para vetores genéricos. Em contraste, com [[ e quando [ é passado mais argumentos (ou seja, subconjunto de objetos fora dos limites com length(dim(x))) > 2) um erro é retornado:

(1:3)[10]
## [1] NA
(1:3)[[10]]
## Error in (1:3)[[10]] : subscript out of bounds
as.matrix(1:3)[10]
## [1] NA
as.matrix(1:3)[, 10]
## Error in as.matrix(1:3)[, 10] : subscript out of bounds
list(1, 2, 3)[10]
## [[1]]
## NULL
list(1, 2, 3)[[10]]
## Error in list(1, 2, 3)[[10]] : subscript out of bounds

O comportamento é o mesmo ao fazer subconjuntos com vetores “caracteres”, que também não são correspondidos no atributo “nomes” do objeto:

c(a = 1, b = 2)["c"]
## <NA> 
##   NA 
list(a = 1, b = 2)["c"]
## <NA>
## NULL

Tópicos de ajuda:

Veja ?Extrair para mais informações.

Quadros de dados #

A divisão de um quadro de dados em um quadro de dados menor pode ser realizada da mesma forma que a divisão de uma lista.

> df3 <- data.frame(x = 1:3, y = c("a", "b", "c"), stringsAsFactors = FALSE)

> df3
##   x y
## 1 1 a
## 2 2 b
## 3 3 c

> df3[1]   # Subset a variable by number
##   x
## 1 1
## 2 2
## 3 3

> df3["x"]   # Subset a variable by name 
##   x
## 1 1
## 2 2
## 3 3

> is.data.frame(df3[1])
## TRUE

> is.list(df3[1])
## TRUE

Subconfigurar um dataframe em um vetor de coluna pode ser feito usando colchetes duplos [[ ]] ou o operador de cifrão $.

> df3[[2]]    # Subset a variable by number using [[ ]]
## [1] "a" "b" "c"

> df3[["y"]]  # Subset a variable by name using [[ ]]
## [1] "a" "b" "c"

> df3$x    # Subset a variable by name using $
## [1] 1 2 3

> typeof(df3$x)
## "integer"

> is.vector(df3$x)
## TRUE

Subconjunto de dados como uma matriz bidimensional pode ser feito usando termos i e j.

> df3[1, 2]    # Subset row and column by number
## [1] "a"

> df3[1, "y"]  # Subset row by number and column by name
## [1] "a"

> df3[2, ]     # Subset entire row by number  
##   x y
## 2 2 b

> df3[ , 1]    # Subset all first variables 
## [1] 1 2 3

> df3[ , 1, drop = FALSE]
##   x
## 1 1
## 2 2
## 3 3

Nota: Subconjunto por j (coluna) sozinho simplifica para o próprio tipo da variável, mas subconjunto por i sozinho retorna um data.frame, pois as diferentes variáveis ​​podem ter tipos e classes diferentes. Definir o parâmetro drop para FALSE mantém o quadro de dados.

> is.vector(df3[, 2])
## TRUE

> is.data.frame(df3[2, ])
## TRUE

> is.data.frame(df3[, 2, drop = FALSE])
## TRUE

Vetores atômicos #

Os vetores atômicos (que excluem listas e expressões, que também são vetores) são subconjuntos usando o operador [:

# create an example vector
v1 <- c("a", "b", "c", "d")

# select the third element
v1[3]
## [1] "c"

O operador [ também pode receber um vetor como argumento. Por exemplo, para selecionar o primeiro e o terceiro elementos:

v1 <- c("a", "b", "c", "d")

v1[c(1, 3)]
## [1] "a" "c"

Algumas vezes podemos exigir a omissão de um determinado valor do vetor. Isso pode ser feito usando um sinal negativo (-) antes do índice desse valor. Por exemplo, para omitir o primeiro valor de v1, use v1[-1]. Isso pode ser estendido para mais de um valor de maneira direta. Por exemplo, v1[-c(1,3)].

> v1[-1]
[1] "b" "c" "d"
> v1[-c(1,3)]
[1] "b" "d"

Em algumas ocasiões, gostaríamos de saber, principalmente, quando o comprimento do vetor é grande, índice de um determinado valor, se existe:

> v1=="c"
[1] FALSE FALSE  TRUE FALSE
> which(v1=="c")
[1] 3

Se o vetor atômico tiver nomes (um atributo names), ele pode ser subconjunto usando um vetor de caracteres de nomes:

v <- 1:3
names(v) <- c("one", "two", "three")

v
##  one   two three 
##    1     2     3 

v["two"]
## two 
##   2

O operador [[ também pode ser usado para indexar vetores atômicos, com diferenças em que ele aceita um vetor de indexação com um comprimento de um e remove quaisquer nomes presentes:

v[[c(1, 2)]]
## Error in v[[c(1, 2)]] : 
##  attempt to select more than one element in vectorIndex

v[["two"]]
## [1] 2

Os vetores também podem ser subconjuntos usando um vetor lógico. Em contraste com o subconjunto com vetores numéricos e de caracteres, o vetor lógico usado para subconjunto deve ser igual ao comprimento do vetor cujos elementos são extraídos, portanto, se um vetor lógico y for usado para subconjunto x, ou seja, x [y], se length(y) < length(x) então y será reciclado para corresponder a length(x):

v[c(TRUE, FALSE, TRUE)]
##  one three 
##    1     3 

v[c(FALSE, TRUE)]  # recycled to 'c(FALSE, TRUE, FALSE)'
## two 
##   2 

v[TRUE]   # recycled to 'c(TRUE, TRUE, TRUE)'
##  one   two three 
##    1     2     3 

v[FALSE]   # handy to discard elements but save the vector's type and basic structure
## named integer(0)

Matrizes #

Para cada dimensão de um objeto, o operador [ recebe um argumento. Os vetores têm uma dimensão e recebem um argumento. Matrizes e quadros de dados têm duas dimensões e recebem dois argumentos, dados como [i, j] onde i é a linha e j é a coluna. A indexação começa em 1.

## a sample matrix
mat <- matrix(1:6, nrow = 2, dimnames = list(c("row1", "row2"), c("col1", "col2", "col3")))

mat
#      col1 col2 col3
# row1    1    3    5
# row2    2    4    6

mat[i,j] é o elemento na linha i-th, coluna j-th da matriz mat. Por exemplo, um valor i de 2 e um valor j de 1 fornece o número na segunda linha e na primeira coluna da matriz. Omitir i ou j retorna todos os valores nessa dimensão.

mat[ , 3]
## row1 row2 
##    5    6 

mat[1, ]
# col1 col2 col3 
#    1    3    5

Quando a matriz tem nomes de linha ou coluna (não obrigatórios), eles podem ser usados ​​para subconjunto:

mat[ , 'col1']
# row1 row2 
#    1    2

Por padrão, o resultado de um subconjunto será simplificado, se possível. Se o subconjunto tiver apenas uma dimensão, como nos exemplos acima, o resultado será um vetor unidimensional em vez de uma matriz bidimensional. Este padrão pode ser substituído com o argumento drop = FALSE para [:

## This selects the first row as a vector
class(mat[1, ])
# [1] "integer"

## Whereas this selects the first row as a 1x3 matrix:
class(mat[1, , drop = F])
# [1] "matrix"

Obviamente, as dimensões não podem ser descartadas se a própria seleção tiver duas dimensões:

mat[1:2, 2:3]  ## A 2x2 matrix
#      col2 col3
# row1    3    5
# row2    4    6

Selecionando entradas de matrizes individuais por suas posições #

Também é possível usar uma matriz Nx2 para selecionar N elementos individuais de uma matriz (como funciona um sistema de coordenadas). Se você deseja extrair, em um vetor, as entradas de uma matriz na categoria (1ª linha, 1ª coluna), (1ª linha, 3ª coluna), (2ª linha, 3ª coluna), (2ª linha, 1ª coluna) isso pode ser feito facilmente criando uma matriz de índice com essas coordenadas e usando isso para subconjunto da matriz:

mat
#      col1 col2 col3
# row1    1    3    5
# row2    2    4    6

ind = rbind(c(1, 1), c(1, 3), c(2, 3), c(2, 1))
ind
#      [,1] [,2]
# [1,]    1    1
# [2,]    1    3
# [3,]    2    3
# [4,]    2    1

mat[ind]
# [1] 1 5 6 2

No exemplo acima, a 1ª coluna da matriz ind refere-se a linhas em mat, a 2ª coluna de ind refere-se a colunas em mat.

Listas #

Uma lista pode ser subconjunto com [:

l1 <- list(c(1, 2, 3), 'two' = c("a", "b", "c"), list(10, 20))
l1
## [[1]]
## [1] 1 2 3
## 
## $two
## [1] "a" "b" "c"
##
## [[3]]
## [[3]][[1]]
## [1] 10
##
## [[3]][[2]]
## [1] 20

l1[1]
## [[1]]
## [1] 1 2 3

l1['two']
## $two
## [1] "a" "b" "c"

l1[[2]]
## [1] "a" "b" "c"

l1[['two']]
## [1] "a" "b" "c"

Observe que o resultado de l1[2] ainda é uma lista, pois o operador [ seleciona elementos de uma lista, retornando uma lista menor. O operador [[ extrai os elementos da lista, retornando um objeto do tipo do elemento da lista.

Os elementos podem ser indexados por número ou uma cadeia de caracteres do nome (se existir). Vários elementos podem ser selecionados com [ passando um vetor de números ou strings de nomes. A indexação com um vetor de length > 1 em [ e [[ retorna uma “lista” com os elementos especificados e um subconjunto recursivo (se disponível), respectivamente:

l1[c(3, 1)]
## [[1]]
## [[1]][[1]]
## [1] 10
## 
## [[1]][[2]]
## [1] 20
## 
## 
## [[2]]
## [1] 1 2 3

Comparado com:

l1[[c(3, 1)]]
## [1] 10

que é equivalente a:

l1[[3]][[1]]
## [1] 10

O operador $ permite selecionar elementos da lista apenas pelo nome, mas diferentemente de [ e [[, não requer aspas. Como um operador infixo, $ só pode ter um único nome:

l1$two
## [1] "a" "b" "c"

Além disso, o operador $ permite correspondência parcial por padrão:

l1$t
## [1] "a" "b" "c"

em contraste com [[ onde é necessário especificar se a correspondência parcial é permitida:

l1[["t"]]
## NULL
l1[["t", exact = FALSE]]
## [1] "a" "b" "c"

Configurando options(warnPartialMatchDollar = TRUE), um “aviso” é dado quando a correspondência parcial acontece com $:

l1$t
## [1] "a" "b" "c"
## Warning message:
## In l1$t : partial match of 't' to 'two'

Outros objetos #

Os operadores [ e [[ são funções primitivas que são genéricas. Isso significa que qualquer object em R (especificamente isTRUE(is.object(x)) –ou seja, tem um atributo “class” explícito) pode ter seu próprio comportamento especificado quando subconjunto; ou seja, tem seus próprios métodos para [ e/ou [[.

Por exemplo, este é o caso dos objetos “data.frame” (is.object(iris)) onde os métodos [.data.frame e [[.data.frame são definidos e são feitos para exibir subconjuntos do tipo “matriz” e do tipo “lista”. Ao forçar um erro ao subdefinir um “data.frame”, vemos que, na verdade, uma função [.data.frame foi chamada quando -apenas- usamos [.

iris[invalidArgument, ]
## Error in `[.data.frame`(iris, invalidArgument, ) : 
##   object 'invalidArgument' not found

Sem mais detalhes sobre o tópico atual, um exemplo[ method:

x = structure(1:5, class = "myClass")
x[c(3, 2, 4)]
## [1] 3 2 4
'[.myClass' = function(x, i) cat(sprintf("We'd expect '%s[%s]' to be returned but this a custom `[` method and should have a `?[.myClass` help page for its behaviour\n", deparse(substitute(x)), deparse(substitute(i))))

x[c(3, 2, 4)]
## We'd expect 'x[c(3, 2, 4)]' to be returned but this a custom `[` method and should have a `?[.myClass` help page for its behaviour
## NULL

Podemos superar o despacho do método [ usando o equivalente não genérico .subset (e .subset2 para [[). Isso é especialmente útil e eficiente ao programar nossas próprias “classes” e queremos evitar soluções alternativas (como unclass(x)) ao calcular nossas “classes” de forma eficiente (evitando o envio de métodos e a cópia de objetos):

.subset(x, c(3, 2, 4))
## [1] 3 2 4

Operações de matriz elementar #

Sejam A e B duas matrizes de mesma dimensão. Os operadores +,-,/,*,^ quando usados ​​com matrizes de mesma dimensão realizam as operações necessárias nos elementos correspondentes das matrizes e retornam uma nova matriz de mesma dimensão. Essas operações são geralmente chamadas de operações elementares.

Operador A operação B Significado
+ A + B Adição de elementos correspondentes de A e B
- A - B Subtrai os elementos de B dos elementos correspondentes de A
/ A/B Divide os elementos de A pelos elementos correspondentes de B
* A * B Multiplica os elementos de A pelos elementos correspondentes de B
^ A^(-1) Por exemplo, dá uma matriz cujos elementos são recíprocos de A

Para multiplicação de matrizes “verdadeira”, como visto em Linear Algebra, use %*%. Por exemplo, a multiplicação de A por B é: A %*% B. Os requisitos dimensionais são que o ncol() de A seja o mesmo que nrow() de B
