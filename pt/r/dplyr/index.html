dplyr é uma iteração do plyr que fornece funções flexíveis baseadas em “verbo” para manipular dados em R. A versão mais recente do dplyr pode ser baixada do CRAN usando

install.package("dplyr")

O objeto chave em dplyr é um tbl, uma representação de uma estrutura de dados tabular. Atualmente dplyr (versão 0.5.0) suporta:

  • quadros de dados
  • tabelas de dados
  • SQLite
  • PostgreSQL/Redshift
  • MySQL/MariaDB
  • BigQuery
  • MonetDB
  • cubos de dados com arrays (implementação parcial)

verbos de tabela única do dplyr #

dplyr introduz uma gramática de manipulação de dados em R. Ele fornece uma interface consistente para trabalhar com dados, não importa onde estejam armazenados: data.frame, [data.table] (https://www.wikiod.com/pt/r/tabela-de-dados) ou um banco de dados. As peças-chave do dplyr são escritas usando Rcpp, o que o torna muito rápido para trabalhar com dados na memória.

A filosofia do dplyr é ter pequenas funções que fazem uma coisa bem. As cinco funções simples (filter, arrange, select, mutate e summarise) podem ser usadas para revelar novas maneiras de descrever dados. Quando combinadas com group_by, essas funções podem ser usadas para calcular estatísticas resumidas de grupo.

Semelhanças de sintaxe #

Todas essas funções têm uma sintaxe semelhante:

  • O primeiro argumento para todas essas funções é sempre um quadro de dados
  • As colunas podem ser referenciadas diretamente usando nomes de variáveis ​​simples (ou seja, sem usar $)
  • Essas funções não modificam os dados originais em si, ou seja, não possuem efeitos colaterais. Portanto, os resultados devem sempre ser salvos em um objeto.

Usaremos o conjunto de dados mtcars integrado para explorar os verbos de tabela única do dplyr . Antes de converter o tipo de mtcars para tbl_df (já que torna a impressão mais limpa), adicionamos os rownames do conjunto de dados como uma coluna usando a função rownames_to_column do [tibble](https://cran.r -project.org/web/packages/tibble/index.html).

library(dplyr) # This documentation was written using version 0.5.0

mtcars_tbl <- as_data_frame(tibble::rownames_to_column(mtcars, "cars"))

# examine the structure of data

filtro #

filter ajuda a subconjunto de linhas que correspondem a determinados critérios. O primeiro argumento é o nome do data.frame e o segundo (e subsequentes) argumentos são os critérios que filtram os dados (esses critérios devem ser avaliados como TRUE ou FALSE)

Subconjunto de todos os carros que têm 4 cilindros - cyl:

filter(mtcars_tbl, cyl == 4)

filter(mtcars_tbl, cyl == 4 | cyl == 6, gear == 5)

filter seleciona linhas com base em critérios, para selecionar linhas por posição, use slice. slice recebe apenas 2 argumentos: o primeiro é um data.frame e o segundo são valores de linha inteiros.

Para selecionar as linhas 6 a 9:

slice(mtcars_tbl, 6:9)

slice(mtcars_tbl, -c(1:5, 10:n()))

Isso resulta na mesma saída que slice(mtcars_tbl, 6:9)

n() representa o número de observações no grupo atual

arranjo #

arrange é usado para classificar os dados por uma(s) variável(is) especificada(s). Assim como o verbo anterior (e todas as outras funções em dplyr), o primeiro argumento é um data.frame, e argumentos consequentes são usados ​​para classificar os dados. Se mais de uma variável for passada, os dados serão classificados primeiro pela primeira variável, depois pela segunda variável e assim por diante.

Para ordenar os dados por cavalo de força - hp

Agregando com o operador %>% (pipe) #

O pipe (%>%) operator pode ser usado em combinação com as funções dplyr. Neste exemplo usamos o conjunto de dados mtcars (veja help("mtcars") para mais informações) para mostrar como sumarizar um quadro de dados e adicionar variáveis ​​aos dados com o resultado da aplicação de uma função.

library(dplyr)
library(magrittr)
df <- mtcars
df$cars <- rownames(df) #just add the cars names to the df
df <- df[,c(ncol(df),1:(ncol(df)-1))] # and place the names in the first column

1. Resumir os dados

Para calcular estatísticas usamos summarize e as funções apropriadas. Neste caso n() é usado para contar o número de casos.

 df %>%
  summarize(count=n(),mean_mpg = mean(mpg, na.rm = TRUE),
            min_weight = min(wt),max_weight = max(wt))

#  count mean_mpg min_weight max_weight

Exemplos de variáveis ​​NSE e string em dpylr #

dplyr usa Non-Standard Evaluation (NSE), e é por isso que normalmente podemos usar os nomes das variáveis ​​sem aspas. No entanto, às vezes durante o pipeline de dados, precisamos obter nossos nomes de variáveis ​​de outras fontes, como uma caixa de seleção Shiny. No caso de funções como select, podemos apenas usar select_ para usar uma variável de string para selecionar

variable1 <- "Sepal.Length"
variable2 <- "Sepal.Width"
iris %>%
select_(variable1, variable2) %>%
head(n=5)
#  Sepal.Length Sepal.Width
#  1          5.1         3.5
#  2          4.9         3.0
#  3          4.7         3.2
#  4          4.6         3.1
#  5          5.0         3.6

Mas se quisermos usar outros recursos, como resumir ou filtrar, precisamos usar a função interp do pacote lazyeval

variable1 <- "Sepal.Length"
variable2 <- "Sepal.Width"
variable3 <- "Species"
iris %>%
select_(variable1, variable2, variable3) %>%
group_by_(variable3) %>%
summarize_(mean1 = lazyeval::interp(~mean(var), var = as.name(variable1)), mean2 = lazyeval::interp(~mean(var), var = as.name(variable2)))
#      Species mean1 mean2
#       <fctr> <dbl> <dbl>
# 1     setosa 5.006 3.428
# 2 versicolor 5.936 2.770
# 3  virginica 6.588 2.974
