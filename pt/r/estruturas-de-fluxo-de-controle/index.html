For loops são um método de controle de fluxo para repetir uma tarefa ou conjunto de tarefas em um domínio. A estrutura central de um loop for é

for ( [index] in [domain]){
  [body]
}

Onde

  1. [index] é um nome que assume exatamente um valor de [domain] em cada iteração do loop.
  2. [domínio] é um vetor de valores sobre os quais iterar.
  3. [body] é o conjunto de instruções a serem aplicadas em cada iteração.

Como um exemplo trivial, considere o uso de um laço for para obter a soma cumulativa de um vetor de valores.

x <- 1:4
cumulative_sum <- 0
for (i in x){
  cumulative_sum <- cumulative_sum + x[i]
}
cumulative_sum

Otimizando a Estrutura de For Loops #

For loops podem ser úteis para conceituar e executar tarefas a serem repetidas. Se não forem construídos com cuidado, no entanto, eles podem ser muito lentos para executar em comparação com o uso preferido da família de funções apply. No entanto, há um punhado de elementos que você pode incluir em sua construção de loop for para otimizar o loop. Em muitos casos, uma boa construção do laço for produzirá eficiência computacional muito próxima à de uma função apply.

Um loop for ‘devidamente construído’ baseia-se na estrutura principal e inclui uma instrução declarando o objeto que capturará cada iteração do loop. Este objeto deve ter uma classe e um comprimento declarados.

[output] <- [vector_of_length]
for ([index] in [length_safe_domain]){
  [output][index] <- [body]
}

Para ilustrar, vamos escrever um loop para elevar ao quadrado cada valor em um vetor numérico (este é um exemplo trivial apenas para ilustração. A maneira ‘correta’ de completar esta tarefa seria x_squared <- x^2).

x <- 1:100
x_squared <- vector("numeric", length = length(x))
for (i in seq_along(x)){
  x_squared[i] <- x[i]^2
}

Novamente, observe que primeiro declaramos um receptáculo para a saída x_squared e demos a classe “numeric” com o mesmo comprimento que x. Além disso, declaramos um “domínio seguro de comprimento” usando a função seq_along. seq_along gera um vetor de índices para um objeto que é adequado para uso em loops for. Embora pareça intuitivo usar for (i in 1:length(x)), se x tiver 0 comprimento, o loop tentará iterar sobre o domínio de 1:0, resultando em um erro (o 0º índice é indefinido em R).

Objetos de receptáculo e domínios de comprimento seguro são manipulados internamente pela família de funções apply e os usuários são encorajados a adotar a abordagem apply no lugar de loops for o máximo possível. No entanto, se construído corretamente, um loop for pode ocasionalmente fornecer maior clareza de código com perda mínima de eficiência.

Vetorizando para Loops #

Loops for muitas vezes podem ser uma ferramenta útil para conceituar as tarefas que precisam ser concluídas em cada iteração. Quando o laço está completamente desenvolvido e conceituado, pode haver vantagens em transformar o laço em uma função.

Neste exemplo, desenvolveremos um loop for para calcular a média de cada coluna no conjunto de dados mtcars (novamente, um exemplo trivial, pois poderia ser realizado através da função colMeans).

column_mean_loop <- vector("numeric", length(mtcars))
for (k in seq_along(mtcars)){
  column_mean_loop[k] <- mean(mtcars[[k]])
}

O loop for pode ser convertido em uma função apply reescrevendo o corpo do loop como uma função.

col_mean_fn <- function(x) mean(x)
column_mean_apply <- vapply(mtcars, col_mean_fn, numeric(1))

E para comparar os resultados:

identical(column_mean_loop, 
          unname(column_mean_apply)) #* vapply added names to the elements
                                     #* remove them for comparison

A vantagem da forma vetorizada é que conseguimos eliminar algumas linhas de código. A mecânica de determinar o comprimento e o tipo do objeto de saída e iterar em um domínio seguro de comprimento são tratadas para nós pela função apply. Além disso, a função apply é um pouco mais rápida que o loop. A diferença de velocidade é muitas vezes insignificante em termos humanos, dependendo do número de iterações e da complexidade do corpo.

Construção ótima de um loop for #

Para ilustrar o efeito da construção good for loop, calcularemos a média de cada coluna de quatro maneiras diferentes:

  1. Usando um loop for mal otimizado
  2. Usando um loop for bem otimizado
  3. Usando uma família de funções *apply
  4. Usando a função colMeans

Cada uma dessas opções será mostrada em código; será mostrada uma comparação do tempo computacional para executar cada opção; e por último será feita uma discussão sobre as diferenças.

Mal otimizado para loop #

column_mean_poor <- NULL
for (i in 1:length(mtcars)){
  column_mean_poor[i] <- mean(mtcars[[i]])
}

Bem otimizado para loop #

column_mean_optimal <- vector("numeric", length(mtcars))
for (i in seq_along(mtcars)){
  column_mean_optimal <- mean(mtcars[[i]])
}

Função vaply #

column_mean_vapply <- vapply(mtcars, mean, numeric(1))

Função colMeans #

column_mean_colMeans <- colMeans(mtcars)

Comparação de eficiência #

Os resultados do benchmarking dessas quatro abordagens são mostrados abaixo (código não exibido)

Unit: microseconds
     expr     min       lq     mean   median       uq     max neval  cld
     poor 240.986 262.0820 287.1125 275.8160 307.2485 442.609   100    d
  optimal 220.313 237.4455 258.8426 247.0735 280.9130 362.469   100   c 
   vapply 107.042 109.7320 124.4715 113.4130 132.6695 202.473   100 a   
 colMeans 155.183 161.6955 180.2067 175.0045 194.2605 259.958   100  b

Observe que o loop for otimizado superou o loop for mal construído. O laço for mal construído está constantemente aumentando o comprimento do objeto de saída e, a cada mudança no comprimento, R está reavaliando a classe do objeto.

Parte dessa sobrecarga é removida pelo loop for otimizado, declarando o tipo de objeto de saída e seu comprimento antes de iniciar o loop.

Neste exemplo, no entanto, o uso de uma função vapply dobra a eficiência computacional, em grande parte porque dissemos ao R que o resultado tinha que ser numérico (se algum resultado não fosse numérico, um erro seria retornado).

O uso da função colMeans é um pouco mais lento do que a função vappy. Esta diferença é atribuída a algumas verificações de erros realizadas em colMeans e principalmente à conversão as.matrix (porque mtcars é um data.frame) que não foram realizadas na função vapply.

Básico para construção de loop #

Neste exemplo vamos calcular o desvio quadrado para cada coluna em um quadro de dados, neste caso o mtcars.

Opção A: índice inteiro

squared_deviance <- vector("list", length(mtcars))
for (i in seq_along(mtcars)){
  squared_deviance[[i]] <- (mtcars[[i]] - mean(mtcars[[i]]))^2
}

squared_deviance é uma lista de 11 elementos, como esperado.

class(squared_deviance)
length(squared_deviance)

Opção B: índice de caracteres

squared_deviance <- vector("list", length(mtcars))
Squared_deviance <- setNames(squared_deviance, names(mtcars))
for (k in names(mtcars)){
  squared_deviance[[k]] <- (mtcars[[k]] - mean(mtcars[[k]]))^2
}

E se quisermos um data.frame como resultado? Bem, existem muitas opções para transformar uma lista em outros objetos. No entanto, e talvez o mais simples neste caso, será armazenar os resultados do for em um data.frame.

squared_deviance <- mtcars #copy the original
squared_deviance[TRUE]<-NA  #replace with NA or do squared_deviance[,]<-NA
for (i in seq_along(mtcars)){
  squared_deviance[[i]] <- (mtcars[[i]] - mean(mtcars[[i]]))^2
}
dim(squared_deviance)
[1] 32 11

O resultado será o mesmo evento embora usemos a opção de caractere (B).

As outras construções de loop: while e repeat #

R fornece duas construções de loop adicionais, while e repeat, que são normalmente usadas em situações em que o número de iterações necessárias é indeterminado.

O laço while #

A forma geral de um loop while é a seguinte,

while (condition) {
    ## do something
    ## in loop body
}

onde condição é avaliada antes de entrar no corpo do loop. Se condition for avaliado como TRUE, o código dentro do corpo do loop é executado, e esse processo se repete até que condition seja avaliado como FALSE (ou uma instrução break seja alcançada; veja abaixo). Ao contrário do loop for, se um loop while usa uma variável para realizar iterações incrementais, a variável deve ser declarada e inicializada antes do tempo e deve ser atualizada dentro do corpo do loop. Por exemplo, os seguintes loops realizam a mesma tarefa:

for (i in 0:4) {
    cat(i, "\n")
}
# 0 
# 1 
# 2 
# 3 
# 4 

i <- 0
while (i < 5) {
    cat(i, "\n")
    i <- i + 1
}
# 0 
# 1 
# 2 
# 3 
# 4

No loop while acima, a linha i <- i + 1 é necessária para evitar um loop infinito.

Além disso, é possível encerrar um loop while com uma chamada para break de dentro do corpo do loop:

iter <- 0
while (TRUE) {
    if (runif(1) < 0.25) {
        break
    } else {
        iter <- iter + 1
    }
}
iter
#[1] 4

Neste exemplo, condition é sempre TRUE, então a única maneira de terminar o loop é com uma chamada para break dentro do corpo. Observe que o valor final de iter dependerá do estado do seu PRNG quando este exemplo for executado e deverá produzir resultados diferentes (essencialmente) cada vez que o código for executado.

O loop repetir #

A construção repeat é essencialmente a mesma que while (TRUE) { ## something }, e tem a seguinte forma:

repeat ({
    ## do something
    ## in loop body
})

Os {} extras não são obrigatórios, mas os () são. Reescrevendo o exemplo anterior usando repeat,

iter <- 0
repeat ({
    if (runif(1) < 0.25) {
        break
    } else {
        iter <- iter + 1
    }
})
iter
#[1] 2

Mais sobre pausa #

É importante notar que break terminará apenas o loop imediatamente delimitador. Ou seja, o seguinte é um loop infinito:

while (TRUE) {
    while (TRUE) {
        cat("inner loop\n")
        break
    }
    cat("outer loop\n")
}

Com um pouco de criatividade, no entanto, é possível romper inteiramente de dentro de um loop aninhado. Como exemplo, considere a seguinte expressão, que, em seu estado atual, fará um loop infinito:

while (TRUE) {
    cat("outer loop body\n")
    while (TRUE) {
        cat("inner loop body\n")
        x <- runif(1)
        if (x < .3) {
            break
        } else {
            cat(sprintf("x is %.5f\n", x))
        }
    }
}

Uma possibilidade é reconhecer que, ao contrário de break, a expressão return tem a capacidade de retornar o controle em vários níveis de loops delimitadores. No entanto, como return só é válido quando usado dentro de uma função, não podemos simplesmente substituir break por return() acima, mas também precisamos envolver a expressão inteira como uma função anônima:

(function() {
    while (TRUE) {
        cat("outer loop body\n")
        while (TRUE) {
            cat("inner loop body\n")
            x <- runif(1)
            if (x < .3) {
                return()
            } else {
                cat(sprintf("x is %.5f\n", x))
            }
        }
    }
})()

Alternativamente, podemos criar uma variável fictícia (exit) antes da expressão e ativá-la via <<- do loop interno quando estivermos prontos para terminar:

exit <- FALSE
while (TRUE) {
    cat("outer loop body\n")
    while (TRUE) {
        cat("inner loop body\n")
        x <- runif(1)
        if (x < .3) {
            exit <<- TRUE
            break
        } else {
            cat(sprintf("x is %.5f\n", x))
        }
    }
    if (exit) break
}
