Expressões regulares (também chamadas de “regex” ou “regexp”) definem padrões que podem ser combinados com uma string. Digite ?regex para a documentação oficial do R e veja os Regex Docs para mais detalhes. A ‘pegadinha’ mais importante que não será aprendida nos tópicos/regex SO é que a maioria das funções R-regex precisam do uso de barras invertidas pareadas para escapar em um parâmetro pattern.

classes de personagens #

  • "[AB]" pode ser A ou B
  • "[[:alpha:]]" pode ser qualquer letra
  • "[[:lower:]]" significa qualquer letra minúscula. Observe que "[a-z]" está próximo, mas não corresponde, por exemplo, ú.
  • "[[:upper:]]" significa qualquer letra maiúscula. Observe que "[A-Z]" está próximo, mas não corresponde, por exemplo, Ú.
  • "[[:digit:]]" significa qualquer dígito: 0, 1, 2, … ou 9 e é equivalente a "[0-9]".

Quantificadores #

+, * e ? aplicam-se normalmente em regex. – + corresponde pelo menos uma vez, * corresponde a 0 ou mais vezes e ? corresponde a 0 ou 1 vez.

Indicadores de início e fim de linha #

Você pode especificar a posição do regex na string:

  • "^..." força a expressão regular a estar no início da string
  • "...$" força a expressão regular a estar no final da string

Diferenças de outros idiomas #

Por favor, note que as expressões regulares em R muitas vezes parecem sempre tão ligeiramente diferentes das expressões regulares usadas em outras linguagens.

  • R requer escapes de barra invertida dupla (porque "\" já implica escape em geral em strings R), então, por exemplo, para capturar espaços em branco na maioria dos mecanismos de expressão regular, basta digitar \s, vs. \\s em R.

  • Caracteres UTF-8 em R devem ser escapados com um U maiúsculo, por exemplo [\U{1F600}] e [\U1F600] correspondem 😀, enquanto em, por exemplo, Ruby, isso seria combinado com um u minúsculo.

Recursos adicionais #

O seguinte site reg101 é um bom lugar para verificar regex online antes de usá-lo R-script.

O wikibook de programação R tem uma página dedicada ao processamento de texto com muitos exemplos usando expressões regulares.

Diferenças entre Perl e POSIX regex #

Existem dois mecanismos ligeiramente diferentes de expressões regulares implementados em R. O padrão é chamado de consistência POSIX; todas as funções regex em R também são equipadas com uma opção para ativar o último tipo: perl = TRUE.

Olhando para frente/para trás #

perl = TRUE permite look-ahead e look-behind em expressões regulares.

  • "(?<=A)B" corresponde a uma aparência da letra B somente se for precedida por A, ou seja, "ABACADABRA" seria correspondido, mas "abacadabra" e "aBacadabra " não faria.

Validar uma data no formato “AAAAMMDD” #

É uma prática comum nomear arquivos usando a data como prefixo no seguinte formato: AAAAMMDD, por exemplo: 20170101_results.csv. Uma data nesse formato de string pode ser verificada usando a seguinte expressão regular:

\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])

A expressão acima considera datas do ano: 0000-9999, meses entre: 01-12 e dias 01-31.

Por exemplo:

> grepl("\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])", "20170101")
[1] TRUE
> grepl("\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])", "20171206")
[1] TRUE
> grepl("\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])", "29991231")
[1] TRUE

Observação: Valida a sintaxe de data, mas podemos ter uma data errada com uma sintaxe válida, por exemplo: 20170229 (2017 não é um ano bissexto).

> grepl("\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])", "20170229")
[1] TRUE

Se você deseja validar uma data, isso pode ser feito através desta função definida pelo usuário:

is.Date <- function(x) {return(!is.na(as.Date(as.character(x), format = '%Y%m%d')))}

Então

> is.Date(c("20170229", "20170101", 20170101))
[1] FALSE  TRUE  TRUE

Validar as abreviaturas postais dos Estados dos EUA #

O seguinte regex inclui 50 estados e também Commonwealth/Territory (consulte www.50states.com):

regex <- "(A[LKSZR])|(C[AOT])|(D[EC])|(F[ML])|(G[AU])|(HI)|(I[DLNA]) |(K[SY])|(LA)|(M[EHDAINSOT])|(N[EVHJMYCD])|(MP)|(O[HKR])|(P[WAR])|(RI)|(S [CD])|(T[NX])|(UT)|(V[TIA])|(W[AVIY])"

Por exemplo:

> test <- c("AL", "AZ", "AR", "AJ", "AS", "DC", "FM", "GU","PW", "FL", "AJ", "AP")
> grepl(us.states.pattern, test)
 [1]  TRUE  TRUE  TRUE FALSE  TRUE  TRUE  TRUE  TRUE  TRUE  TRUE FALSE FALSE
>

Observação:

Se você deseja verificar apenas os 50 estados, recomendamos usar o R-dataset: state.abb de state, por exemplo:

> data(state)
> test %in% state.abb
[1]  TRUE  TRUE  TRUE FALSE FALSE FALSE FALSE FALSE FALSE  TRUE FALSE FALSE

Obtemos TRUE apenas para abreviações de 50 estados: AL, AZ, AR, FL.

Validar números de telefone dos EUA #

A seguinte expressão regular:

us.phones.regex <- "^\\s*(\\+\\s*1(-?|\\s+))*[0-9]{3}\\s*-?\\s*[0-9]{3}\\s*-?\\s*[0-9]{4}$"

Valida um número de telefone na forma de: +1-xxx-xxx-xxxx, incluindo espaços em branco opcionais à esquerda/no início de cada grupo de números, mas não no meio, por exemplo: +1- xxx-xxx-xx xx não é válido. O delimitador - pode ser substituído por espaços em branco: xxx xxx xxx ou sem delimitador: xxxxxxxxxx. O prefixo +1 é opcional.

Vamos verificar:

us.phones.regex <- "^\\s*(\\+\\s*1(-?|\\s+))*[0-9]{3}\\s*-?\\s*[0-9]{3}\\s*-?\\s*[0-9]{4}$"

phones.OK <- c("305-123-4567", "305 123 4567", "+1-786-123-4567", 
    "+1 786 123 4567", "7861234567", "786 - 123   4567", "+ 1 786 - 123   4567")

phones.NOK <- c("124-456-78901", "124-456-789", "124-456-78 90", 
    "124-45 6-7890", "12 4-456-7890")

Casos válidos:

> grepl(us.phones.regex, phones.OK)
[1] TRUE TRUE TRUE TRUE TRUE TRUE TRUE
>

Casos inválidos:

 > grepl(us.phones.regex, phones.NOK)
[1] FALSE FALSE FALSE FALSE FALSE
>

Observação:

  • \\s Corresponde a qualquer caractere de espaço, tabulação ou nova linha

Caracteres de escape em padrões R regex #

Como R e regex compartilham o caractere de escape ,"\", a construção de padrões corretos para grep, sub, gsub ou qualquer outra função que aceite um argumento de padrão geralmente precisará de pares de barras invertidas. Se você construir um vetor de caracteres de três itens no qual um item tem um avanço de linha, outro um caractere de tabulação e outro nenhum, e o desejo é transformar o avanço de linha ou a tabulação em 4 espaços, então uma única barra invertida é necessária para a construção, mas barras invertidas tpareadas para correspondência:

x <- c( "a\nb", "c\td", "e    f")
x  # how it's stored
   #  [1] "a\nb"   "c\td"   "e    f"
cat(x)   # how it will be seen with cat
#a
#b c    d e    f

gsub(patt="\\n|\\t", repl="    ", x)
#[1] "a    b" "c    d" "e    f"

Observe que o argumento padrão (que é opcional se aparecer primeiro e precisar apenas de ortografia parcial) é o único argumento que exige essa duplicação ou emparelhamento. O argumento de substituição não requer a duplicação de caracteres que precisam ser escapados. Se você quisesse que todos os feeds de linha e ocorrências de 4 espaços fossem substituídos por guias, seria:

gsub("\\n|    ", "\t", x)
#[1] "a\tb" "c\td" "e\tf"

Eliminando espaços em branco #

string <- '    some text on line one; 
and then some text on line two     '

Aparar espaços em branco #

“Aparar” espaços em branco normalmente se refere à remoção de espaços em branco iniciais e finais de uma string. Isso pode ser feito usando uma combinação dos exemplos anteriores. gsub é usado para forçar a substituição nas correspondências iniciais e finais.

Antes de R 3.2.0

gsub(pattern = "(^ +| +$)",
     replacement = "",
     x = string)

[1] "some text on line one; \nand then some text on line two"

R 3.2.0 e superior

trimws(x = string)

[1] "some text on line one; \nand then some text on line two"

Removendo o espaço em branco inicial #

Antes de R 3.2.0

sub(pattern = "^ +", 
    replacement = "",
    x = string)

[1] "some text on line one; \nand then some text on line two     "

R 3.2.0 e superior

trimws(x = string,
       which = "left")

[1] "some text on line one; \nand then some text on line two     "

Removendo o espaço em branco à direita #

Antes de R 3.2.0

sub(pattern = " +$",
    replacement = "",
    x = string)

[1] "    some text on line one; \nand then some text on line two"

R 3.2.0 e superior

trimws(x = string,
       which = "right")

[1] "    some text on line one; \nand then some text on line two"

Removendo todos os espaços em branco #

gsub(pattern = "\\s",   
     replacement = "",
     x = string)

[1] "sometextonlineone;andthensometextonlinetwo"

Observe que isso também removerá caracteres de espaço em branco, como tabulações (\t), novas linhas (\r e \n) e espaços.
