Sintaxe #

  1. fator(x = caractere(), níveis, rótulos = níveis, exclusão = NA, ordenado = é.ordenado(x), nmax = NA)
  2. Execute o ?factor ou consulte a documentação online.

Um objeto com classe fator é um vetor com um conjunto particular de características.

  1. Ele é armazenado internamente como um vetor inteiro.
  2. Mantém um atributo levels que mostra a representação de caracteres dos valores.
  3. Sua classe é armazenada como fator

Para ilustrar, vamos gerar um vetor de 1.000 observações a partir de um conjunto de cores.

set.seed(1)
Color <- sample(x = c("Red", "Blue", "Green", "Yellow"), 
                size = 1000, 
                replace = TRUE)
Color <- factor(Color)

Podemos observar cada uma das características de Color listadas acima:

#* 1. It is stored internally as an `integer` vector
typeof(Color)

1 “inteiro”

#* 2. It maintains a `levels` attribute the shows the character representation of the values.
#* 3. Its class is stored as `factor`
attributes(Color)

$níveis 1 “Azul” “Verde” “Vermelho” “Amarelo”

$classe 1 “fator”

A principal vantagem de um objeto de fator é a eficiência no armazenamento de dados. Um inteiro requer menos memória para armazenar do que um caractere. Tal eficiência era altamente desejável quando muitos computadores tinham recursos muito mais limitados do que as máquinas atuais (para uma história mais detalhada das motivações por trás do uso de fatores, veja stringsAsFactors: an Unauthorized Biography). A diferença no uso de memória pode ser vista até mesmo em nosso objeto Color. Como você pode ver, armazenar Color como um caractere requer cerca de 1,7 vezes mais memória que o objeto de fator.

#* Amount of memory required to store Color as a factor.
object.size(Color)

4624 bytes

#* Amount of memory required to store Color as a character
object.size(as.character(Color))

8232 bytes

Mapeando o inteiro para o nível #

Enquanto a computação interna de fatores vê o objeto como um inteiro, a representação desejada para consumo humano é o nível do caractere. Por exemplo,

head(Color)

1 Azul Azul Verde Amarelo Vermelho Amarelo Níveis: Azul Verde Vermelho Amarelo

é mais fácil para a compreensão humana do que

head(as.numeric(Color))

1 1 1 2 4 3 4

Uma ilustração aproximada de como R faz a correspondência da representação de caracteres com o valor inteiro interno é:

head(levels(Color)[as.numeric(Color)])

1 “Azul” “Azul” “Verde” “Amarelo” “Vermelho” “Amarelo”

Compare esses resultados com

head(Color)

1 Azul Azul Verde Amarelo Vermelho Amarelo Níveis: Azul Verde Vermelho Amarelo

Uso moderno de fatores #

Em 2007, o R introduziu um método de hash para caracteres que reduzia a carga de memória dos vetores de caracteres (ref: stringsAsFactors: an Unauthorized Biography). Observe que quando determinamos que os caracteres exigem 1,7 vezes mais espaço de armazenamento do que os fatores, isso foi calculado em uma versão recente do R, o que significa que o uso de memória de vetores de caracteres era ainda mais desgastante antes de 2007.

Devido ao método de hash no R moderno e aos recursos de memória muito maiores em computadores modernos, a questão da eficiência da memória no armazenamento de valores de caracteres foi reduzida a uma preocupação muito pequena. A atitude predominante na comunidade R é uma preferência por vetores de caráter sobre fatores na maioria das situações. As principais causas para o afastamento dos fatores são

  1. O aumento de dados de caracteres não estruturados e/ou pouco controlados
  2. A tendência dos fatores não se comportarem como desejado quando o usuário esquece que está lidando com um fator e não um personagem

No primeiro caso, não faz sentido armazenar texto livre ou campos de resposta abertos como fatores, pois dificilmente haverá algum padrão que permita mais de uma observação por nível. Alternativamente, se a estrutura de dados não for cuidadosamente controlada, é possível obter vários níveis que correspondem à mesma categoria (como “azul”, “Azul” e “AZUL”). Nesses casos, muitos preferem gerenciar essas discrepâncias como caracteres antes de converter em um fator (se a conversão ocorrer).

No segundo caso, se o usuário achar que está trabalhando com um vetor de caracteres, alguns métodos podem não responder como previsto. Esse entendimento básico pode levar à confusão e frustração ao tentar depurar scripts e códigos. Embora, estritamente falando, isso possa ser considerado culpa do usuário, a maioria dos usuários fica feliz em evitar o uso de fatores e evitar essas situações completamente.

Consolidando Níveis de Fatores com uma Lista #

Há momentos em que é desejável consolidar os níveis dos fatores em menos grupos, talvez por causa de dados esparsos em uma das categorias. Também pode ocorrer quando há variações de grafia ou capitalização dos nomes das categorias. Considere como exemplo o fator

set.seed(1)
colorful <- sample(c("red", "Red", "RED", "blue", "Blue", "BLUE", "green", "gren"),
                   size = 20,
                   replace = TRUE)
colorful <- factor(colorful)

Como R é sensível a maiúsculas e minúsculas, uma tabela de frequência desse vetor apareceria como abaixo.

table(colorful)

colorido azul Azul AZUL verde verde gren vermelho Vermelho VERMELHO 3 1 4 2 4 1 3 2

Esta tabela, no entanto, não representa a verdadeira distribuição dos dados, e as categorias podem ser efetivamente reduzidas a três tipos: Azul, Verde e Vermelho. Três exemplos são fornecidos. A primeira ilustra o que parece ser uma solução óbvia, mas na verdade não fornecerá uma solução. O segundo fornece uma solução funcional, mas é detalhado e computacionalmente caro. A terceira não é uma solução óbvia, mas é relativamente compacta e computacionalmente eficiente.

Consolidando níveis usando factor (factor_approach) #

factor(as.character(colorful),
       levels = c("blue", "Blue", "BLUE", "green", "gren", "red", "Red", "RED"),
       labels = c("Blue", "Blue", "Blue", "Green", "Green", "Red", "Red", "Red"))

1 Verde Azul Vermelho Vermelho Azul Vermelho Vermelho Vermelho Azul Vermelho Verde Verde Verde Azul Vermelho Verde [17] Vermelho Verde Verde Vermelho Níveis: Azul Azul Azul Verde Verde Vermelho Vermelho Vermelho Mensagem de aviso: Em levels<-(*tmp*, value = if (nl == nL) as.character(labels) else paste0(labels, : níveis duplicados em fatores estão obsoletos

Observe que há níveis duplicados. Ainda temos três categorias para “Azul”, o que não completa nossa tarefa de consolidar os níveis. Além disso, há um aviso de que os níveis duplicados estão obsoletos, o que significa que esse código pode gerar um erro no futuro.

Consolidando níveis usando ifelse (ifelse_approach) #

factor(ifelse(colorful %in% c("blue", "Blue", "BLUE"),
       "Blue",
       ifelse(colorful %in% c("green", "gren"),
              "Green",
              "Red")))

1 Verde Azul Vermelho Vermelho Azul Vermelho Vermelho Vermelho Azul Vermelho Verde Verde Verde Azul Vermelho Verde [17] Vermelho Verde Verde Vermelho Níveis: Azul Verde Vermelho

Este código gera o resultado desejado, mas requer o uso de instruções ifelse aninhadas. Embora não haja nada de errado com essa abordagem, gerenciar instruções ifelse aninhadas pode ser uma tarefa tediosa e deve ser feita com cuidado.

Níveis de fatores de consolidação com uma lista (list_approach) #

Uma maneira menos óbvia de consolidar níveis é usar uma lista onde o nome de cada elemento é o nome da categoria desejada e o elemento é um vetor de caracteres dos níveis no fator que deve mapear para a categoria desejada. Isso tem a vantagem adicional de trabalhar diretamente no atributo levels do fator, sem ter que atribuir novos objetos.

levels(colorful) <- 
     list("Blue" = c("blue", "Blue", "BLUE"),
          "Green" = c("green", "gren"),
          "Red" = c("red", "Red", "RED"))

1 Verde Azul Vermelho Vermelho Azul Vermelho Vermelho Vermelho Azul Vermelho Verde Verde Verde Azul Vermelho Verde [17] Vermelho Verde Verde Vermelho Níveis: Azul Verde Vermelho

Criação básica de fatores #

Fatores são uma forma de representar variáveis ​​categóricas em R. Um fator é armazenado internamente como um vetor de inteiros. Os elementos exclusivos do vetor de caracteres fornecido são conhecidos como níveis do fator. Por padrão, se os níveis não forem fornecidos pelo usuário, então R irá gerar o conjunto de valores únicos no vetor, ordenar esses valores alfanumericamente e usá-los como níveis.

 charvar <- rep(c("n", "c"), each = 3)
 f <- factor(charvar)
 f
 levels(f)

> f
[1] n n n c c c
Levels: c n
> levels(f)
[1] "c" "n"

Se você deseja alterar a ordem dos níveis, uma opção para especificar os níveis manualmente:

levels(factor(charvar, levels = c("n","c")))

> levels(factor(charvar, levels = c("n","c")))
[1] "n" "c"

Os fatores têm várias propriedades. Por exemplo, os níveis podem receber rótulos:

> f <- factor(charvar, levels=c("n", "c"), labels=c("Newt", "Capybara"))
> f
[1] Newt     Newt     Newt     Capybara Capybara Capybara
Levels: Newt Capybara

Outra propriedade que pode ser atribuída é se o fator é ordenado:

> Weekdays <- factor(c("Monday", "Wednesday", "Thursday", "Tuesday", "Friday", "Sunday", "Saturday"))
> Weekdays
[1] Monday    Wednesday Thursday  Tuesday   Friday    Sunday    Saturday 
Levels: Friday Monday Saturday Sunday Thursday Tuesday Wednesday
> Weekdays <- factor(Weekdays, levels=c("Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday", "Sunday"), ordered=TRUE)
> Weekdays
[1] Monday    Wednesday Thursday  Tuesday   Friday    Sunday    Saturday 
Levels: Monday < Tuesday < Wednesday < Thursday < Friday < Saturday < Sunday

Quando um nível do fator não é mais usado, você pode eliminá-lo usando a função droplevels():

> Weekend <- subset(Weekdays, Weekdays == "Saturday" |  Weekdays == "Sunday")
> Weekend
[1] Sunday   Saturday
Levels: Monday < Tuesday < Wednesday < Thursday < Friday < Saturday < Sunday
> Weekend <- droplevels(Weekend)
> Weekend
[1] Sunday   Saturday
Levels: Saturday < Sunday

Fatores de mudança e reordenação #

Quando os fatores são criados com padrões, os levels são formados por as.character aplicados às entradas e são ordenados alfabeticamente.

charvar <- rep(c("W", "n", "c"), times=c(17,20,14))
f <- factor(charvar)
levels(f)
# [1] "c" "n" "W"

Em algumas situações o tratamento da ordenação padrão de níveis (ordem alfabética/lexical) será aceitável. Por exemplo, se alguém quiser apenas plot as frequências, este será o resultado:

plot(f,col=1:length(levels(f)))

Mas se quisermos uma ordenação diferente de levels, precisamos especificar isso no parâmetro levels ou labels (tendo cuidado para que o significado de “ordem” aqui seja diferente dos fatores ordenados, veja abaixo). Existem muitas alternativas para realizar essa tarefa, dependendo da situação.

1. Redefinir o fator

Quando for possível, podemos recriar o fator usando o parâmetro levels com a ordem que queremos.

ff <- factor(charvar, levels = c("n", "W", "c"))
levels(ff)
# [1] "n" "W" "c"
 
gg <- factor(charvar, levels = c("W", "c", "n"))
levels(gg)
# [1] "W" "c" "n"

Quando os níveis de entrada são diferentes dos níveis de saída desejados, usamos o parâmetro labels que faz com que o parâmetro levels se torne um “filtro” para valores de entrada aceitáveis, mas deixa os valores finais de “níveis” para o vetor de fatores como argumento para labels:

fm <- factor(as.numeric(f),levels = c(2,3,1),
             labels = c("nn", "WW", "cc"))
levels(fm)
# [1] "nn" "WW" "cc"

fm <- factor(LETTERS[1:6], levels = LETTERS[1:4],  # only 'A'-'D' as input
                 labels = letters[1:4])            # but assigned to 'a'-'d'
fm
#[1] a    b    c    d    <NA> <NA>
#Levels: a b c d

2. Use a função relevel

Quando há um level específico que precisa ser o primeiro, podemos usar relevel. Isso acontece, por exemplo, no contexto da análise estatística, quando uma categoria base é necessária para testar hipóteses.

g<-relevel(f, "n") # moves n to be the first level
levels(g)
# [1] "n" "c" "W"

Como pode ser verificado f e g são os mesmos

all.equal(f, g)
# [1] "Attributes: < Component “levels”: 2 string mismatches >"
all.equal(f, g, check.attributes = F)
# [1] TRUE

3. Fatores de reordenação

Há casos em que precisamos reordenar os níveis com base em um número, um resultado parcial, uma estatística computada ou cálculos anteriores. Vamos reordenar com base nas frequências dos níveis

table(g)
# g
#  n  c  W 
# 20 14 17

A função reorder é genérica (veja help(reorder)), mas neste contexto precisa de: x, neste caso o fator; X, um valor numérico do mesmo tamanho que x; e FUN, uma função a ser aplicada ao X e calculada pelo nível do x, que determina a ordem dos níveis, por padrão crescente. O resultado é o mesmo fator com seus níveis reordenados.

g.ord <- reorder(g,rep(1,length(g)), FUN=sum) #increasing
levels(g.ord)
# [1] "c" "W" "n"

Para obter ordem decrescente, consideramos valores negativos (-1)

g.ord.d <- reorder(g,rep(-1,length(g)), FUN=sum)
levels(g.ord.d)
# [1] "n" "W" "c"

Novamente o fator é o mesmo que os outros.

data.frame(f,g,g.ord,g.ord.d)[seq(1,length(g),by=5),] #just same lines
#    f g g.ord g.ord.d
# 1  W W     W       W
# 6  W W     W       W
# 11 W W     W       W
# 16 W W     W       W
# 21 n n     n       n
# 26 n n     n       n
# 31 n n     n       n
# 36 n n     n       n
# 41 c c     c       c
# 46 c c     c       c
# 51 c c     c       c

Quando há uma variável quantitativa relacionada à variável fator, podemos usar outras funções para reordenar os níveis. Vamos pegar os dados iris (help("iris") para mais informações), para reordenar o fator Species usando sua média Sepal.Width.

miris <- iris  #help("iris") # copy the data
with(miris, tapply(Sepal.Width,Species,mean))
#    setosa versicolor  virginica 
#     3.428      2.770      2.974 

miris$Species.o<-with(miris,reorder(Species,-Sepal.Width))
levels(miris$Species.o)
# [1] "setosa"     "virginica"  "versicolor"

O boxplot usual (digamos: with(miris, boxplot(Petal.Width~Species)) mostrará as espécies nesta ordem: setosa, versicolor e virginica. Mas usando o fator ordenado nós obtenha as espécies ordenadas por sua média Sepal.Width:

boxplot(Petal.Width~Species.o, data = miris,
        xlab = "Species", ylab = "Petal Width",
        main = "Iris Data, ordered by mean sepal width", varwidth = TRUE,
        col = 2:4)

Além disso, também é possível alterar os nomes dos levels, combiná-los em grupos ou adicionar novos levels. Para isso usamos a função de mesmo nome levels.

f1<-f
levels(f1)
# [1] "c" "n" "W"
levels(f1) <- c("upper","upper","CAP") #rename and grouping
levels(f1)
# [1] "upper" "CAP" 

f2<-f1
levels(f2) <- c("upper","CAP", "Number") #add Number level, which is empty
levels(f2)
# [1] "upper"  "CAP"    "Number"
f2[length(f2):(length(f2)+5)]<-"Number" # add cases for the new level
table(f2)
# f2
#  upper    CAP Number 
#     33     17      6 

f3<-f1
levels(f3) <- list(G1 = "upper", G2 = "CAP", G3 = "Number") # The same using list
levels(f3)
# [1] "G1" "G2" "G3"
f3[length(f3):(length(f3)+6)]<-"G3" ## add cases for the new level
table(f3)
# f3
# G1 G2 G3 
# 33 17  7

- Fatores ordenados

Finalmente, sabemos que os fatores ordenados são diferentes dos fatores, o primeiro é usado para representar dados ordinais, e o segundo para trabalhar com dados nominais. A princípio, não faz sentido mudar a ordem dos níveis para os fatores ordenados, mas podemos mudar seus rótulos.

ordvar<-rep(c("Low", "Medium", "High"), times=c(7,2,4))

of<-ordered(ordvar,levels=c("Low", "Medium", "High"))
levels(of)
# [1] "Low"    "Medium" "High" 

of1<-of
levels(of1)<- c("LOW", "MEDIUM", "HIGH")
levels(of1)
# [1] "LOW"    "MEDIUM" "HIGH"
is.ordered(of1)
# [1] TRUE
of1
# [1] LOW    LOW    LOW    LOW    LOW    LOW    LOW    MEDIUM MEDIUM HIGH   HIGH   HIGH   HIGH  
# Levels: LOW < MEDIUM < HIGH

Reconstruindo fatores do zero #

Problema #

Fatores são usados ​​para representar variáveis ​​que tomam valores de um conjunto de categorias, conhecidas como Níveis em R. Por exemplo, algum experimento pode ser caracterizado pelo nível de energia de uma bateria, com quatro níveis: vazio, baixo, normal e cheio. Então, para 5 locais de amostragem diferentes, esses níveis puderam ser identificados, nesses termos, da seguinte forma:

completo, completo, normal, vazio, baixo

Normalmente, em bancos de dados ou outras fontes de informação, o tratamento desses dados é feito por índices inteiros arbitrários associados às categorias ou níveis. Se assumirmos que, para o exemplo dado, atribuiríamos os índices da seguinte forma: 1 = vazio, 2 = baixo, 3 = normal, 4 = cheio, então as 5 amostras poderiam ser codificadas como:

4, 4, 3, 1, 2

Pode acontecer que, a partir da sua fonte de informação, por ex. um banco de dados, você tem apenas a lista codificada de inteiros e o catálogo associando cada inteiro a cada palavra-chave de nível. Como um fator de R pode ser reconstruído a partir dessa informação?

Solução #

Vamos simular um vetor de 20 inteiros que representa as amostras, cada uma das quais pode ter um dos quatro valores diferentes:

set.seed(18)
ii <- sample(1:4, 20, replace=T)
ii

1 4 3 4 1 1 3 2 3 2 1 3 4 1 2 4 1 3 1 4 1

O primeiro passo é fazer um fator, da sequência anterior, em que os níveis ou categorias sejam exatamente os números de 1 a 4.

fii <- factor(ii, levels=1:4) # it is necessary to indicate the numeric levels
fii

1 4 3 4 1 1 3 2 3 2 1 3 4 1 2 4 1 3 1 4 1 Níveis: 1 2 3 4

Agora simplesmente, você tem que vestir o fator já criado com as tags index:

levels(fii) <- c("empty", "low", "normal", "full")
fii

1 cheio normal cheio vazio vazio normal baixo normal baixo vazio [11] normal cheio vazio baixo cheio vazio normal vazio cheio vazio Níveis: vazio baixo normal cheio

Fatores #

Fatores são um método para representar variáveis ​​categóricas em R. Dado um vetor x cujos valores podem ser convertidos em caracteres usando as.character(), os argumentos padrão para factor() e as. factor() atribui um inteiro a cada elemento distinto do vetor, bem como um atributo de nível e um atributo de rótulo. Níveis são os valores que x pode assumir e rótulos podem ser o elemento dado ou determinado pelo usuário.

Para exemplificar como os fatores funcionam, criaremos um fator com atributos padrão, depois níveis personalizados e depois níveis e rótulos personalizados.

# standard
factor(c(1,1,2,2,3,3))
[1] 1 1 2 2 3 3
Levels: 1 2 3

Podem surgir instâncias onde o usuário sabe que o número de valores possíveis que um fator pode assumir é maior que os valores atuais no vetor. Para isso, nós mesmos atribuímos os níveis em factor().

factor(c(1,1,2,2,3,3),
         levels = c(1,2,3,4,5))
[1] 1 1 2 2 3 3
Levels: 1 2 3 4 5

Para fins de estilo, o usuário pode querer atribuir rótulos a cada nível. Por padrão, os rótulos são a representação de caracteres dos níveis. Aqui atribuímos rótulos para cada um dos níveis possíveis no fator.

factor(c(1,1,2,2,3,3),
       levels = c(1,2,3,4,5),
       labels = c("Fox","Dog","Cow","Brick","Dolphin"))
[1] Fox Fox Dog Dog Cow Cow
Levels: Fox Dog Cow Brick Dolphin

Normalmente, os fatores só podem ser comparados usando == e != e se os fatores tiverem os mesmos níveis. A seguinte comparação de fatores falha, embora pareçam iguais porque os fatores têm níveis de fatores diferentes.

factor(c(1,1,2,2,3,3),levels = c(1,2,3)) == factor(c(1,1,2,2,3,3),levels = c(1,2,3,4,5))
Error in Ops.factor(factor(c(1, 1, 2, 2, 3, 3), levels = c(1, 2, 3)),  : 
  level sets of factors are different

Isso faz sentido, pois os níveis extras no RHS significam que R não possui informações suficientes sobre cada fator para compará-los de maneira significativa.

Os operadores <, <=, > e >= só podem ser usados ​​para fatores ordenados. Estes podem representar valores categóricos que ainda possuem uma ordem linear. Um fator ordenado pode ser criado fornecendo o argumento ordered = TRUE para a função factor ou apenas usando a função ordered.

x <- factor(1:3, labels = c('low', 'medium', 'high'), ordered = TRUE)
print(x)
[1] low    medium high  
Levels: low < medium < high

y <- ordered(3:1, labels = c('low', 'medium', 'high'))
print(y)
[1] high   medium low   
Levels: low < medium < high

x < y
[1]  TRUE FALSE FALSE

Para obter mais informações, consulte a documentação do fator.
