Uma função na família *apply é uma abstração de um loop for. Comparadas com os loops for, as funções *apply têm as seguintes vantagens:

  1. Requer menos código para escrever.
  2. Não possui contador de iteração.
  3. Não usa variáveis ​​temporárias para armazenar resultados intermediários.

No entanto, loops for são mais gerais e podem nos dar mais controle, permitindo realizar cálculos complexos que nem sempre são triviais de se fazer usando funções *apply.

A relação entre loops for e funções *apply é explicada na documentação para loops for.

Membros da família *apply #

A família de funções *apply contém várias variantes do mesmo princípio que diferem com base principalmente no tipo de saída que retornam.

função Entrada Saída
aplicar matrix, data.frame ou array vetor ou matriz (dependendo do comprimento de cada elemento retornado)
aplicar vetor ou ’lista' vetor ou matriz (dependendo do comprimento de cada elemento retornado)
aplicar vetor ou ’lista' lista
vaplicar vetor ou `lista vetor ou matriz (dependendo do comprimento de cada elemento retornado) da classe designada pelo usuário
mapear vários vetores, ’listas’ ou uma combinação lista

Consulte “Exemplos” para ver como cada uma dessas funções é usada.

Usando funcionais integrados #

Funcionais internos: lapply(), sapply() e mapply() #

R vem com funcionais embutidos, dos quais talvez os mais conhecidos sejam a família de funções de aplicação. Aqui está uma descrição de algumas das funções de aplicação mais comuns:

  • lappy() = recebe uma lista como argumento e aplica a função especificada à lista.
  • sapply() = o mesmo que lappy(), mas tenta simplificar a saída para um vetor ou matriz.
    • vapply() = a variant of sapply() in which the output object’s type must be specified.
  • mapply() = como lappy() mas pode passar vários vetores como entrada para a função especificada. Pode ser simplificado como sapply().
    • Map() is an alias to mapply() with SIMPLIFY = FALSE.

lapply() #

lappy() pode ser usado com duas iterações diferentes:

  • lappy(variável, FUN)

  • lappy(seq_along(variável), FUN)

    Two ways of finding the mean of x #

    set.seed(1) df <- data.frame(x = rnorm(25), y = rnorm(25)) lapply(df, mean) lapply(seq_along(df), function(x) mean(df[[x]))

sapply() #

sapply() tentará resolver sua saída para um vetor ou uma matriz.

# Two examples to show the different outputs of sapply()
sapply(letters, print)  ## produces a vector
x <- list(a = 1:10, beta = exp(-3:3), logic = c(TRUE,FALSE,FALSE,TRUE))
sapply(x, quantile)  ## produces a matrix

mapear() #

mapply() funciona muito como lappy() exceto que pode receber vários vetores como entrada (daí o m para multivariada).

mapply(sum, 1:5, 10:6, 3) # 3 will be "recycled" by mapply

Carregamento de arquivo em massa #

para um grande número de arquivos que podem precisar ser operados em um processo semelhante e com nomes de arquivos bem estruturados.

primeiramente deve ser criado um vetor dos nomes dos arquivos a serem acessados, existem várias opções para isso:

  • Criando o vetor manualmente com paste0()

      files <- paste0("file_", 1:100, ".rds")

  • Usar list.files() com um termo de busca regex para o tipo de arquivo, requer conhecimento de expressões regulares (regex) se outros arquivos do mesmo tipo estiverem no diretório.

      files <- list.files("./", pattern = "\\.rds$", full.names = TRUE)

onde X é um vetor de parte do formato de nomeação de arquivos usado.

lappy produzirá cada resposta como elemento de uma lista.

readRDS é específico para arquivos .rds e mudará dependendo da aplicação do processo.

my_file_list <- lapply(files, readRDS)

Isso não é necessariamente mais rápido do que um loop for do teste, mas permite que todos os arquivos sejam um elemento de uma lista sem atribuí-los explicitamente.

Finalmente, muitas vezes precisamos carregar vários pacotes de uma só vez. Este truque pode ser feito facilmente aplicando library() a todas as bibliotecas que desejamos importar:

lapply(c("jsonlite","stringr","igraph"),library,character.only=TRUE)

Combinando vários data.frames (lappy, mapply) #

Neste exercício, vamos gerar quatro modelos de regressão linear bootstrap e combinar os resumos desses modelos em um único quadro de dados.

library(broom)

#* Create the bootstrap data sets
BootData <- lapply(1:4,
                   function(i) mtcars[sample(1:nrow(mtcars),
                                             size = nrow(mtcars),
                                             replace = TRUE), ])

#* Fit the models
Models <- lapply(BootData,
                 function(BD) lm(mpg ~ qsec + wt + factor(am),
                                 data = BD))

#* Tidy the output into a data.frame
Tidied <- lapply(Models,
                 tidy)

#* Give each element in the Tidied list a name
Tidied <- setNames(Tidied, paste0("Boot", seq_along(Tidied)))

Neste ponto, podemos adotar duas abordagens para inserir os nomes no data.frame.

#* Insert the element name into the summary with `lapply`
#* Requires passing the names attribute to `lapply` and referencing `Tidied` within
#* the applied function.
Described_lapply <- 
 lapply(names(Tidied),
        function(nm) cbind(nm, Tidied[[nm]]))

Combined_lapply <- do.call("rbind", Described_lapply)

#* Insert the element name into the summary with `mapply`
#* Allows us to pass the names and the elements as separate arguments.
Described_mapply <- 
 mapply(
  function(nm, dframe) cbind(nm, dframe),
  names(Tidied),
  Tidied,
  SIMPLIFY = FALSE)

Combined_mapply <- do.call("rbind", Described_mapply)

Se você é fã de pipes estilo magrittr, pode realizar toda a tarefa em uma única cadeia (embora possa não ser prudente fazê-lo se precisar de algum dos objetos intermediários, como os próprios objetos de modelo):

library(magrittr)
library(broom)
Combined <- lapply(1:4,
                   function(i) mtcars[sample(1:nrow(mtcars),
                                             size = nrow(mtcars),
                                             replace = TRUE), ]) %>%
 lapply(function(BD) lm( mpg ~ qsec + wt + factor(am), data = BD)) %>%
 lapply(tidy) %>%
 setNames(paste0("Boot", seq_along(.))) %>%
 mapply(function(nm, dframe) cbind(nm, dframe),
        nm = names(.),
        dframe = .,
        SIMPLIFY = FALSE) %>%
 do.call("rbind", .)

Usando funcionais definidos pelo usuário #

Funcionais definidos pelo usuário #

Os usuários podem criar seus próprios funcionais em vários graus de complexidade. Os exemplos a seguir são de Functionals de Hadley Wickham:

randomise <- function(f) f(runif(1e3))
        
lapply2 <- function(x, f, ...) {
    out <- vector("list", length(x))
    for (i in seq_along(x)) {
        out[[i]] <- f(x[[i]], ...)
    }
    out
}

No primeiro caso, randomise aceita um único argumento f e o chama em uma amostra de variáveis ​​aleatórias uniformes. Para demonstrar a equivalência, chamamos set.seed abaixo:

set.seed(123)
randomise(mean)
#[1] 0.4972778
    
set.seed(123)
mean(runif(1e3))
#[1] 0.4972778


set.seed(123)
randomise(max)
#[1] 0.9994045

set.seed(123)
max(runif(1e3))
#[1] 0.9994045

O segundo exemplo é uma reimplementação de base::lappy, que usa funcionais para aplicar uma operação (f) a cada elemento em uma lista (x). O parâmetro ... permite que o usuário passe argumentos adicionais para f, como a opção na.rm na função mean:

lapply(list(c(1, 3, 5), c(2, NA, 6)), mean)
# [[1]]
# [1] 3
# 
# [[2]]
# [1] NA

lapply2(list(c(1, 3, 5), c(2, NA, 6)), mean)
# [[1]]
# [1] 3
# 
# [[2]]
# [1] NA


lapply(list(c(1, 3, 5), c(2, NA, 6)), mean, na.rm = TRUE)
# [[1]]
# [1] 3
# 
# [[2]]
# [1] 4

lapply2(list(c(1, 3, 5), c(2, NA, 6)), mean, na.rm = TRUE)
# [[1]]
# [1] 3
# 
# [[2]]
# [1] 4

Use funções anônimas com apply #

apply é usado para avaliar uma função (talvez anônima) nas margens de um array ou matriz.

Vamos usar o conjunto de dados iris para ilustrar essa ideia. O conjunto de dados iris tem medidas de 150 flores de 3 espécies. Vamos ver como esse conjunto de dados está estruturado:

> head(iris)

  Sepal.Length Sepal.Width Petal.Length Petal.Width Species
1         5.1          3.5          1.4         0.2  setosa
2         4.9          3.0          1.4         0.2  setosa
3         4.7          3.2          1.3         0.2  setosa
4         4.6          3.1          1.5         0.2  setosa
5         5.0          3.6          1.4         0.2  setosa
6         5.4          3.9          1.7         0.4  setosa

Agora, imagine que você quer saber a média de cada dessas variáveis. Uma maneira de resolver isso pode ser usar um loop for, mas os programadores de R geralmente preferem usar apply (por motivos, consulte Comentários):

> apply(iris[1:4], 2, mean)

Sepal.Length  Sepal.Width Petal.Length  Petal.Width 
    5.843333     3.057333     3.758000     1.199333
  • No primeiro parâmetro, subconjunto iris para incluir apenas as primeiras 4 colunas, porque mean só funciona em dados numéricos.
  • O segundo valor do parâmetro 2 indica que queremos trabalhar apenas nas colunas (o segundo subscrito do array r×c); 1 daria as médias da linha.

Da mesma forma, podemos calcular valores mais significativos:

# standard deviation
apply(iris[1:4], 2, sd)
# variance
apply(iris[1:4], 2, var)

Caveat: R tem algumas funções internas que são melhores para calcular somas e médias de colunas e linhas: [colMeans e rowMeans](http://stat.ethz.ch/R-manual/R-patched/library /base/html/colSums.html).

Agora, vamos fazer uma tarefa diferente e mais significativa: vamos calcular a média somente para aqueles valores maiores que 0,5. Para isso, vamos criar nossa própria função mean.

> our.mean.function <- function(x) { mean(x[x > 0.5]) }
> apply(iris[1:4], 2, our.mean.function)

Sepal.Length  Sepal.Width Petal.Length  Petal.Width 
    5.843333     3.057333     3.758000     1.665347

(Observe a diferença na média de Petal.Width)

Mas, e se não quisermos usar essa função no restante do nosso código? Então, podemos usar uma função anônima e escrever nosso código assim:

apply(iris[1:4], 2, function(x) { mean(x[x > 0.5]) })

Assim, como vimos, podemos usar apply para executar a mesma operação em colunas ou linhas de um conjunto de dados usando apenas uma linha.

Aviso: Como apply retorna tipos de saída muito diferentes dependendo do tamanho dos resultados da função especificada, pode não ser a melhor escolha nos casos em que você não está trabalhando interativamente. Algumas das outras funções da família *apply são um pouco mais previsíveis (veja Comentários).
