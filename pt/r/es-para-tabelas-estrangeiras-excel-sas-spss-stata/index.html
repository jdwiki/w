Importando dados com rio #

Uma maneira muito simples de importar dados de muitos formatos de arquivo comuns é com rio. Este pacote fornece uma função import() que envolve muitas funções de importação de dados comumente usadas, fornecendo assim uma interface padrão. Ele funciona simplesmente passando um nome de arquivo ou URL para import():

import("example.csv")       # comma-separated values
import("example.tsv")       # tab-separated values
import("example.dta")       # Stata
import("example.sav")       # SPSS
import("example.sas7bdat")  # SAS
import("example.xlsx")      # Excel

import() também pode ler diretórios compactados, URLs (HTTP ou HTTPS) e a área de transferência. Uma lista abrangente de todos os formatos de arquivo suportados está disponível no rio package github repository.

É até possível especificar alguns parâmetros adicionais relacionados ao formato de arquivo específico que você está tentando ler, passando-os diretamente dentro da função import():

import("example.csv", format = ",") #for csv file where comma is used as separator
import("example.csv", format = ";") #for csv file where semicolon is used as separator

Ler e gravar arquivos Stata, SPSS e SAS #

Os pacotes foreign e haven podem ser usados ​​para importar e exportar arquivos de uma variedade de outros pacotes estatísticos como Stata, SPSS e SAS e softwares relacionados. Existe uma função read para cada um dos tipos de dados suportados para importar os arquivos.

# loading the packages
library(foreign)
library(haven)
library(readstata13)
library(Hmisc)

Alguns exemplos para os tipos de dados mais comuns:

# reading Stata files with `foreign`
read.dta("path\to\your\data")
# reading Stata files with `haven`
read_dta("path\to\your\data")

O pacote foreign pode ler arquivos stata (.dta) para versões do Stata 7-12. De acordo com a página de desenvolvimento, o read.dta está mais ou menos congelado e não será atualizado para leitura nas versões 13+. Para versões mais recentes do Stata, você pode usar o pacote readstata13 ou haven. Para readstata13, os arquivos são

# reading recent Stata (13+) files with `readstata13`
read.dta13("path\to\your\data")

Para leitura em arquivos SPSS e SAS

# reading SPSS files with `foreign`
read.spss("path\to\your\data.sav", to.data.frame = TRUE)
# reading SPSS files with `haven`
read_spss("path\to\your\data.sav")
read_sav("path\to\your\data.sav")
read_por("path\to\your\data.por")

# reading SAS files with `foreign`
read.ssd("path\to\your\data")
# reading SAS files with `haven`
read_sas("path\to\your\data")
# reading native SAS files with `Hmisc`
sas.get("path\to\your\data")   #requires access to saslib 
# Reading SA XPORT format ( *.XPT ) files
sasxport.get("path\to\your\data.xpt")  # does not require access to SAS executable

O pacote SAScii fornece funções que aceitam o código de importação SAS SET e constroem um arquivo de texto que pode ser processado com read.fwf. Ele provou ser muito robusto para importação de grandes conjuntos de dados divulgados publicamente. O suporte está em https://github.com/ajdamico/SAScii

Para exportar quadros de dados para outros pacotes estatísticos você pode usar as funções de escrita write.foreign(). Isso gravará 2 arquivos, um contendo os dados e outro contendo as instruções que o outro pacote precisa para ler os dados.

# writing to Stata, SPSS or SAS files with `foreign`
write.foreign(dataframe, datafile, codefile,
              package = c("SPSS", "Stata", "SAS"), ...)
write.foreign(dataframe, "path\to\data\file", "path\to\instruction\file", package = "Stata")

# writing to Stata files with `foreign`
write.dta(dataframe, "file", version = 7L,
          convert.dates = TRUE, tz = "GMT",
          convert.factors = c("labels", "string", "numeric", "codes"))

# writing to Stata files with `haven`
write_dta(dataframe, "path\to\your\data")

# writing to Stata files with `readstata13`
save.dta13(dataframe, file, data.label = NULL, time.stamp = TRUE,
  convert.factors = TRUE, convert.dates = TRUE, tz = "GMT",
  add.rownames = FALSE, compress = FALSE, version = 117,
  convert.underscore = FALSE)

# writing to SPSS files with `haven`
write_sav(dataframe, "path\to\your\data")

O arquivo armazenado pelo SPSS também pode ser lido com read.spss desta forma:

 foreign::read.spss('data.sav', to.data.frame=TRUE, use.value.labels=FALSE, 
                     use.missings=TRUE, reencode='UTF-8')
# to.data.frame if TRUE: return a data frame
# use.value.labels if TRUE: convert variables with value labels into R factors with those levels
# use.missings if TRUE: information on user-defined missing values will used to set the corresponding values to NA.
# reencode character strings will be re-encoded to the current locale. The default, NA, means to do so in a UTF-8 locale, only.

Importando arquivos do Excel #

Existem vários pacotes R para ler arquivos do Excel, cada um deles usando diferentes linguagens ou recursos, conforme resumido na tabela a seguir:

pacote R Usos
xlsx Java
XLconnect Java
openxlsx C++
readxl C++
RODBC ODBC
gdata Perl

Para os pacotes que usam Java ou ODBC, é importante saber detalhes sobre seu sistema, pois você pode ter problemas de compatibilidade dependendo da versão do R e do sistema operacional. Por exemplo, se você estiver usando R 64 bits, você também deve ter Java 64 bits para usar xlsx ou XLconnect.

Alguns exemplos de leitura de arquivos do Excel com cada pacote são fornecidos abaixo. Observe que muitos dos pacotes têm nomes de funções iguais ou muito semelhantes. Portanto, é útil declarar o pacote explicitamente, como package::function. O pacote openxlsx requer instalação prévia do RTools.

Lendo arquivos excel com o pacote xlsx #

library(xlsx)

O índice ou nome da planilha é necessário para importar.

xlsx::read.xlsx("Book1.xlsx", sheetIndex=1)

xlsx::read.xlsx("Book1.xlsx", sheetName="Sheet1")

Lendo arquivos do Excel com o pacote XLconnect #

library(XLConnect)
wb <- XLConnect::loadWorkbook("Book1.xlsx")

# Either, if Book1.xlsx has a sheet called "Sheet1":
sheet1 <- XLConnect::readWorksheet(wb, "Sheet1")
# Or, more generally, just get the first sheet in Book1.xlsx:
sheet1 <- XLConnect::readWorksheet(wb, getSheets(wb)[1])

XLConnect importa automaticamente os estilos de célula pré-definidos do Excel incorporados em Book1.xlsx. Isso é útil quando você deseja formatar seu objeto de pasta de trabalho e exportar um documento do Excel perfeitamente formatado. Primeiramente, você precisará criar os formatos de célula desejados em Book1.xlsx e salvá-los, por exemplo, como myHeader, myBody e myPcts. Então, depois de carregar a pasta de trabalho em R (veja acima):

Headerstyle <- XLConnect::getCellStyle(wb, "myHeader")
Bodystyle <- XLConnect::getCellStyle(wb, "myBody")
Pctsstyle <- XLConnect::getCellStyle(wb, "myPcts")

Os estilos de célula agora são salvos em seu ambiente R. Para atribuir os estilos de célula a determinados intervalos de seus dados, você precisa definir o intervalo e, em seguida, atribuir o estilo:

Headerrange <- expand.grid(row = 1, col = 1:8)
Bodyrange <- expand.grid(row = 2:6, col = c(1:5, 8))
Pctrange <- expand.grid(row = 2:6, col = c(6, 7))

XLConnect::setCellStyle(wb, sheet = "sheet1", row = Headerrange$row,
             col = Headerrange$col, cellstyle = Headerstyle)
XLConnect::setCellStyle(wb, sheet = "sheet1", row = Bodyrange$row,
             col = Bodyrange$col, cellstyle = Bodystyle)
XLConnect::setCellStyle(wb, sheet = "sheet1", row = Pctrange$row,
             col = Pctrange$col, cellstyle = Pctsstyle)

Observe que XLConnect é fácil, mas pode se tornar extremamente lento na formatação. Uma opção de formatação muito mais rápida, porém mais complicada, é oferecida pelo openxlsx.

Lendo arquivos excel com o pacote openxlsx #

Arquivos do Excel podem ser importados com o pacote openxlsx

library(openxlsx)

openxlsx::read.xlsx("spreadsheet1.xlsx", colNames=TRUE, rowNames=TRUE)

#colNames: If TRUE, the first row of data will be used as column names.
#rowNames: If TRUE, first column of data will be used as row names.

A planilha, que deve ser lida em R, pode ser selecionada fornecendo sua posição no argumento sheet:

openxlsx::read.xlsx("spreadsheet1.xlsx", sheet = 1)

ou declarando seu nome:

openxlsx::read.xlsx("spreadsheet1.xlsx", sheet = "Sheet1")

Além disso, openxlsx pode detectar colunas de data em uma planilha lida. Para permitir a detecção automática de datas, um argumento detectDates deve ser definido como TRUE:

openxlsx::read.xlsx("spreadsheet1.xlsx", sheet = "Sheet1", detectDates= TRUE)

Lendo arquivos excel com o pacote readxl #

Os arquivos do Excel podem ser importados como um quadro de dados para o R usando o pacote readxl.

library(readxl)

Ele pode ler arquivos .xls e .xlsx.

readxl::read_excel("spreadsheet1.xls")
readxl::read_excel("spreadsheet2.xlsx")

A folha a ser importada pode ser especificada por número ou nome.

readxl::read_excel("spreadsheet.xls", sheet = 1)
readxl::read_excel("spreadsheet.xls", sheet = "summary")

O argumento col_names = TRUE define a primeira linha como os nomes das colunas.

 readxl::read_excel("spreadsheet.xls", sheet = 1, col_names = TRUE)

O argumento col_types pode ser usado para especificar os tipos de coluna nos dados como um vetor.

readxl::read_excel("spreadsheet.xls", sheet = 1, col_names = TRUE,
                   col_types = c("text", "date", "numeric", "numeric"))

Lendo arquivos excel com o pacote RODBC #

Os arquivos do Excel podem ser lidos usando o ODBC Excel Driver que faz interface com o Access Database Engine (ACE) do Windows, anteriormente JET. Com o pacote RODBC, o R pode se conectar a esse driver e consultar diretamente as pastas de trabalho. As planilhas são consideradas para manter os cabeçalhos de coluna na primeira linha com dados em colunas organizadas de tipos semelhantes. OBSERVAÇÃO: Essa abordagem é limitada apenas a máquinas Windows/PC, pois JET/ACE são arquivos .dll instalados e não estão disponíveis em outros sistemas operacionais.

library(RODBC)

xlconn <- odbcDriverConnect('Driver={Microsoft Excel Driver (*.xls, *.xlsx, *.xlsm, *.xlsb)};
                             DBQ=C:\\Path\\To\\Workbook.xlsx')

df <- sqlQuery(xlconn, "SELECT * FROM [SheetName$]")
close(xlconn)

Conectando-se com um mecanismo SQL nesta abordagem, as planilhas do Excel podem ser consultadas de forma semelhante às tabelas do banco de dados, incluindo as operações JOIN e UNION. A sintaxe segue o dialeto JET/ACE SQL. OBSERVAÇÃO: somente instruções DML de acesso a dados, especificamente SELECT, podem ser executadas em pastas de trabalho, consideradas consultas não atualizáveis.

joindf <-  sqlQuery(xlconn, "SELECT t1.*, t2.* FROM [Sheet1$] t1
                             INNER JOIN [Sheet2$] t2
                             ON t1.[ID] = t2.[ID]")

uniondf <-  sqlQuery(xlconn, "SELECT * FROM [Sheet1$]
                              UNION  
                              SELECT * FROM [Sheet2$]")

Até mesmo outras pastas de trabalho podem ser consultadas no mesmo canal ODBC apontando para uma pasta de trabalho atual:

otherwkbkdf <- sqlQuery(xlconn, "SELECT * FROM 
                                 [Excel 12.0 Xml;HDR=Yes;
                                 Database=C:\\Path\\To\\Other\\Workbook.xlsx].[Sheet1$];")

Importação ou exportação de arquivo Feather #

Feather é uma implementação do Apache Arrow projetada para armazenar quadros de dados de maneira agnóstica de linguagem, mantendo metadados ( por exemplo, classes de data), aumentando a interoperabilidade entre Python e R. Ler um arquivo de penas produzirá um tibble, não um data.frame padrão.

library(feather)

path <- "filename.feather"
df <- mtcars

write_feather(df, path)

df2 <- read_feather(path)
