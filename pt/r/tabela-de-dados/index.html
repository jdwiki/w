Data.table é um pacote que estende a funcionalidade dos quadros de dados da base R, melhorando principalmente seu desempenho e sintaxe. Consulte a área de documentos do pacote em https://www.wikiod.com/pt/data-table/primeiros-passos-com-datatable para obter detalhes.

Sintaxe #

  • DT[i, j, por] # DT[where, select|update|do, by]
  • DT[...][...] # chaining
  • ################# Atalhos, funções especiais e símbolos especiais dentro de DT[...]
  • .()
    # in several arguments, replaces list()
  • J() # in i, replaces list()
  • :=
    # in j, a function used to add or modify columns
  • .N # in i, the total number of rows
    # in j, the number of rows in a group
  • .EU # in j, the vector of row numbers in the table (filtered by i)
  • .SD # in j, the current subset of the data
    # selected by the .SDcols argument
  • .GRP # in j, the current index of the subset of the data
  • .POR # in j, the list of by values for the current subset of data
  • V1, V2, … # default names for unnamed columns created in j
  • ################# Junções dentro de DT[...]
  • DT1[DT2, ligado, j] # join two tables
  • eu.* # special prefix on DT2’s columns after the join
  • por=.EACHI # special option available only with a join
  • DT1[!DT2, ligado, j] # anti-join two tables
  • DT1[DT2, ligado, rolo, j] # join two tables, rolling on the last column in on=
  • ################# Remodelagem, empilhamento e divisão
  • derreter(DT, id.vars, medida.vars) # transform to long format
    # for multiple columns, use measure.vars = patterns(…)
  • dcast(DT, fórmula) # transform to wide format
  • rbind(DT1, DT2, …) # stack enumerated data.tables
  • rbindlist(DT_list, idcol) # stack a list of data.tables
  • split(DT, por) # split a data.table into a list
  • ################# Algumas outras funções especializadas para data.tables
  • sobreposições # overlap joins
  • mesclar # another way of joining two tables
  • definir # another way of adding or modifying columns
  • fintersect, fsetdiff, funion, fsetequal, unique, duplicated, anyDuplicated # set-theory operations with rows as elements
  • únicoN # the number of distinct rows
  • rowidv(DT, cols) # row ID (1 to .N) within each group determined by cols
  • rleidv(DT, cols) # group ID (1 to .GRP) within each group determined by runs of cols
  • shift(DT, n, type=c(“lag”, “lead”)) # apply a shift operator to every column
  • setorder, setcolorder, setnames, setkey, setindex, setattr # modify attributes and order by reference

Instalação e suporte #

Para instalar o pacote data.table:

# install from CRAN
install.packages("data.table")       

# or install development version 
install.packages("data.table", type = "source", repos = "http://Rdatatable.github.io/data.table")

# and to revert from devel to CRAN, the current version must first be removed
remove.packages("data.table")
install.packages("data.table")

O site oficial do pacote tem páginas wiki que fornecem ajuda para começar, e listas de apresentações e artigos de toda a web. Antes de fazer uma pergunta – aqui no StackOverflow ou em qualquer outro lugar – leia a página de suporte.

Carregando o pacote #

Muitas das funções nos exemplos acima existem no namespace data.table. Para usá-los, você precisará adicionar uma linha como library(data.table) primeiro ou usar seu caminho completo, como data.table::fread em vez de simplesmente fread. Para ajuda em funções individuais, a sintaxe é help("fread") ou ?fread. Novamente, se o pacote não estiver carregado, use o nome completo como ?data.table::fread.

Criando um data.table #

Um data.table é uma versão aprimorada da classe data.frame da base R. Como tal, seu atributo class() é o vetor "data.table" "data.frame" e funções que funcionam em um data. .frame também funcionará com um data.table. Há muitas maneiras de criar, carregar ou forçar um data.table.

Construir #

Não se esqueça de instalar e ativar o pacote data.table

library(data.table)

Existe um construtor com o mesmo nome:

DT <- data.table(
  x = letters[1:5], 
  y = 1:5, 
  z = (1:5) > 3
)
#    x y     z
# 1: a 1 FALSE
# 2: b 2 FALSE
# 3: c 3 FALSE
# 4: d 4  TRUE
# 5: e 5  TRUE

Ao contrário de data.frame, data.table não forçará strings a fatores:

sapply(DT, class)
#               x           y           z 
#     "character"   "integer"   "logical"

Leia em #

Podemos ler a partir de um arquivo de texto:

dt <- fread("my_file.csv")

Ao contrário de read.csv, fread lerá strings como strings, não como fatores.

Modifica um data.frame #

Para maior eficiência, data.table oferece uma maneira de alterar um data.frame ou lista para criar um data.table no local (sem fazer uma cópia ou alterar sua localização de memória):

# example data.frame
DF <- data.frame(x = letters[1:5], y = 1:5, z = (1:5) > 3)
# modification
setDT(DF)

Observe que não atribuímos o resultado <-, pois o objeto DF foi modificado no local. Os atributos de classe do data.frame serão mantidos:

sapply(DF, class)
#         x         y         z 
#  "factor" "integer" "logical"

Força o objeto para data.table #

Se você tiver um list, data.frame ou data.table, você deve usar a função setDT para converter para um data.table porque ela faz a conversão por referência de fazer uma cópia (o que as.data.table faz). Isso é importante se você estiver trabalhando com grandes conjuntos de dados.

Se você tiver outro objeto R (como uma matriz), você deve usar as.data.table para coagi-lo a um data.table.

mat <- matrix(0, ncol = 10, nrow = 10)

DT <- as.data.table(mat)
# or
DT <- data.table(mat)

Símbolos especiais em data.table #

.SD #

.SD refere-se ao subconjunto do data.table para cada grupo, excluindo todas as colunas usadas em by.

.SD junto com lappy podem ser usados ​​para aplicar qualquer função a várias colunas por grupo em um data.table

Continuaremos usando o mesmo conjunto de dados integrado, mtcars:

mtcars = data.table(mtcars) # Let's not include rownames to keep things simpler

Média de todas as colunas no conjunto de dados por número de cilindros, cil:

mtcars[ , lapply(.SD, mean), by = cyl]

#   cyl      mpg     disp        hp     drat       wt     qsec        vs        am     gear     carb
#1:   6 19.74286 183.3143 122.28571 3.585714 3.117143 17.97714 0.5714286 0.4285714 3.857143 3.428571
#2:   4 26.66364 105.1364  82.63636 4.070909 2.285727 19.13727 0.9090909 0.7272727 4.090909 1.545455
#3:   8 15.10000 353.1000 209.21429 3.229286 3.999214 16.77214 0.0000000 0.1428571 3.285714 3.500000

Além de cyl, existem outras colunas categóricas no conjunto de dados, como vs, am, gear e carb. Não faz sentido tomar a ‘média’ dessas colunas. Então, vamos excluir essas colunas. É aqui que o .SDcols entra em cena.

.SDcols #

.SDcols especifica as colunas do data.table que estão incluídas em .SD.

Média de todas as colunas (colunas contínuas) no conjunto de dados por número de engrenagens gear e número de cilindros, cyl, organizados por gear e cyl:

# All the continuous variables in the dataset
cols_chosen <- c("mpg", "disp", "hp", "drat", "wt", "qsec")

mtcars[order(gear, cyl), lapply(.SD, mean), by = .(gear, cyl), .SDcols = cols_chosen]

#   gear cyl    mpg     disp       hp     drat       wt    qsec
#1:    3   4 21.500 120.1000  97.0000 3.700000 2.465000 20.0100
#2:    3   6 19.750 241.5000 107.5000 2.920000 3.337500 19.8300
#3:    3   8 15.050 357.6167 194.1667 3.120833 4.104083 17.1425
#4:    4   4 26.925 102.6250  76.0000 4.110000 2.378125 19.6125
#5:    4   6 19.750 163.8000 116.5000 3.910000 3.093750 17.6700
#6:    5   4 28.200 107.7000 102.0000 4.100000 1.826500 16.8000
#7:    5   6 19.700 145.0000 175.0000 3.620000 2.770000 15.5000
#8:    5   8 15.400 326.0000 299.5000 3.880000 3.370000 14.5500

Talvez não queiramos calcular a ‘média’ por grupos. Para calcular a média de todos os carros no conjunto de dados, não especificamos a variável by.

mtcars[ , lapply(.SD, mean), .SDcols = cols_chosen] 

#        mpg     disp       hp     drat      wt     qsec
#1: 20.09062 230.7219 146.6875 3.596563 3.21725 17.84875

Observação:

  • Não é necessário definir cols_chosen de antemão. .SDcols pode receber diretamente os nomes das colunas
  • .SDcols também pode receber diretamente um vetor de columnnumbers. No exemplo acima seria mtcars[ , lapply(.SD, média), .SDcols = c(1,3:7)]

.N #

.N é uma abreviação para o número de linhas em um grupo.

iris[, .(count=.N), by=Species]

#      Species count
#1:     setosa    50
#2: versicolor    50
#3:  virginica    50

Adicionando e modificando colunas #

A sintaxe DT[where, select|update|do, by] é usada para trabalhar com colunas de um data.table.

  • A parte “onde” é o argumento i
  • A parte “select|update|do” é o argumento j

Esses dois argumentos geralmente são passados ​​por posição em vez de por nome.

Nossos dados de exemplo abaixo são

mtcars = data.table(mtcars, keep.rownames = TRUE)

Editando colunas inteiras #

Use o operador := dentro de j para atribuir novas colunas:

mtcars[, mpg_sq := mpg^2]

Remova as colunas definindo como NULL:

mtcars[, mpg_sq := NULL]

Adicione várias colunas usando o formato multivariado do operador :=:

mtcars[, `:=`(mpg_sq = mpg^2, wt_sqrt = sqrt(wt))]
# or 
mtcars[, c("mpg_sq", "wt_sqrt") := .(mpg^2, sqrt(wt))]

Se as colunas são dependentes e devem ser definidas em sequência, uma maneira é:

mtcars[, c("mpg_sq", "mpg2_hp") := .(temp1 <- mpg^2, temp1/hp)]

A sintaxe .() é usada quando o lado direito de LHS := RHS é uma lista de colunas.

Para nomes de coluna determinados dinamicamente, use parênteses:

vn = "mpg_sq"
mtcars[, (vn) := mpg^2]

As colunas também podem ser modificadas com set, embora isso raramente seja necessário:

set(mtcars, j = "hp_over_wt", v = mtcars$hp/mtcars$wt)

Editando subconjuntos de colunas #

Use o argumento i para subconjunto de linhas “onde” as edições devem ser feitas:

mtcars[1:3, newvar := "Hello"]
# or
set(mtcars, j = "newvar", i = 1:3, v = "Hello")

Como em um data.frame, podemos subconjuntos usando números de linha ou testes lógicos. Também é possível usar um “join” em i, mas essa tarefa mais complicada é abordada em outro exemplo.

Editando atributos de coluna #

Funções que editam atributos, como levels<- ou names<-, na verdade substituem um objeto por uma cópia modificada. Mesmo se usado apenas em uma coluna em um data.table, todo o objeto é copiado e substituído.

Para modificar um objeto sem cópias, use setnames para alterar os nomes das colunas de um data.table ou data.frame e setattr para alterar um atributo para qualquer objeto.

# Print a message to the console whenever the data.table is copied
tracemem(mtcars)
mtcars[, cyl2 := factor(cyl)]

# Neither of these statements copy the data.table
setnames(mtcars, old = "cyl2", new = "cyl_fac")
setattr(mtcars$cyl_fac, "levels", c("four", "six", "eight"))

# Each of these statements copies the data.table
names(mtcars)[names(mtcars) == "cyl_fac"] <- "cf"
levels(mtcars$cf) <- c("IV", "VI", "VIII")

Esteja ciente de que essas alterações são feitas por referência, portanto, são globais. Alterá-los em um ambiente afeta o objeto em todos os ambientes.

# This function also changes the levels in the global environment
edit_levels <- function(x) setattr(x, "levels", c("low", "med", "high"))
edit_levels(mtcars$cyl_factor)

Escrevendo código compatível com data.frame e data.table #

Diferenças na sintaxe de subconjuntos #

Um data.table é uma das várias estruturas de dados bidimensionais disponíveis em R, além de data.frame, matrix e (2D) array. Todas essas classes usam uma sintaxe muito semelhante, mas não idêntica para subconjunto, o esquema A[rows, cols].

Considere os seguintes dados armazenados em um matrix, um data.frame e um data.table:

ma <- matrix(rnorm(12), nrow=4, dimnames=list(letters[1:4], c('X', 'Y', 'Z')))
df <- as.data.frame(ma)
dt <- as.data.table(ma)

ma[2:3]  #---> returns the 2nd and 3rd items, as if 'ma' were a vector (because it is!)
df[2:3]  #---> returns the 2nd and 3rd columns
dt[2:3]  #---> returns the 2nd and 3rd rows!

Se você quer ter certeza do que será devolvido, é melhor ser explícito.

Para obter linhas específicas, basta adicionar uma vírgula após o intervalo:

ma[2:3, ]  # \
df[2:3, ]  #  }---> returns the 2nd and 3rd rows
dt[2:3, ]  # /

Mas, se você quiser subconjunto colunas, alguns casos são interpretados de forma diferente. Tudo três podem ser subconjuntos da mesma maneira com índices inteiros ou de caracteres não armazenados em um variável.

ma[, 2:3]          #  \
df[, 2:3]          #   \
dt[, 2:3]          #    }---> returns the 2nd and 3rd columns
ma[, c("Y", "Z")]  #   /
df[, c("Y", "Z")]  #  /
dt[, c("Y", "Z")]  # /

No entanto, eles diferem para nomes de variáveis ​​sem aspas

mycols <- 2:3
ma[, mycols]                # \
df[, mycols]                #  }---> returns the 2nd and 3rd columns
dt[, mycols, with = FALSE]  # /

dt[, mycols]                # ---> Raises an error

No último caso, mycols é avaliado como o nome de uma coluna. Porque dt não pode encontrar uma coluna chamada mycols, um erro é gerado.

Nota: Para versões do pacote data.table anteriores à 1.9.8, este comportamento era ligeiramente diferente. Qualquer coisa no índice da coluna teria sido avaliada usando dt como um meio Ambiente. Portanto, tanto dt[, 2:3] quanto dt[, mycols] retornariam o vetor 2:3. Não erro seria gerado para o segundo caso, porque a variável mycols existe em o ambiente dos pais.

Estratégias para manter a compatibilidade com data.frame e data.table #

Há muitas razões para escrever código que é garantido para trabalhar com data.frame e data.table. Talvez você seja forçado a usar data.frame, ou talvez precise compartilhar algum código que você não sabe como será usado. Assim, existem algumas estratégias principais para conseguir isso, por ordem de conveniência:

  1. Use uma sintaxe que se comporte da mesma forma para ambas as classes.
  2. Use uma função comum que faça a mesma coisa que a sintaxe mais curta.
  3. Forçar data.table a se comportar como data.frame (ex.: chamar o método específico print.data.frame).
  4. Trate-os como ’lista’, o que eles são.
  5. Converta a tabela para um data.frame antes de fazer qualquer coisa (má idéia se for uma tabela enorme).
  6. Converta a tabela para data.table, se as dependências não forem uma preocupação.

Subconjunto de linhas. É simples, basta usar o seletor [, ], com a vírgula:

A[1:10, ]
A[A$var > 17, ]  # A[var > 17, ] just works for data.table

Colunas de subconjunto. Se você quiser uma única coluna, use o seletor $ ou [[ ]]:

A$var
colname <- 'var'
A[[colname]]
A[[1]]

Se você deseja uma maneira uniforme de pegar mais de uma coluna, é necessário apelar um pouco:

B <- `[.data.frame`(A, 2:4)

# We can give it a better name
select <- `[.data.frame`
B <- select(A, 2:4)
C <- select(A, c('foo', 'bar'))

Subconjunto de linhas ‘indexadas’. Enquanto data.frame tem row.names, data.table tem seu recurso exclusivo key. A melhor coisa é evitar completamente row.names e aproveitar as otimizações existentes no caso de data.table quando possível.

B <- A[A$var != 0, ]
# or...
B <- with(A, A[var != 0, ])  # data.table will silently index A by var before subsetting

stuff <- c('a', 'c', 'f')
C <- A[match(stuff, A$name), ]  # really worse than: setkey(A); A[stuff, ]

Obtenha uma tabela de 1 coluna, obtenha uma linha como um vetor. Isso é fácil com o que vimos até agora:

B <- select(A, 2)    #---> a table with just the second column
C <- unlist(A[1, ])  #---> the first row as a vector (coerced if necessary)

Configurando chaves em data.table #

Sim, você precisa SETKEY pré 1.9.6

No passado (pré 1.9.6), seu data.table era acelerado definindo colunas como chaves para a tabela, principalmente para tabelas grandes. [Consulte intro vignette page 5 da versão de setembro de 2015, onde a velocidade de pesquisa foi 544 vezes melhor.] Você pode encontrar códigos mais antigos usando essas teclas de configuração com ‘setkey’ ou definindo uma coluna ‘key=’ quando montando a mesa.

library(data.table)
DT <- data.table(
  x = letters[1:5], 
  y = 5:1, 
  z = (1:5) > 3
)

#> DT
#   x y     z
#1: a 5 FALSE
#2: b 4 FALSE
#3: c 3 FALSE
#4: d 2  TRUE
#5: e 1  TRUE

Defina sua chave com o comando setkey. Você pode ter uma chave com várias colunas.

setkey(DT, y)

Verifique a chave da sua tabela em tables()

tables()

> tables()
     NAME NROW NCOL MB COLS  KEY
[1,] DT      5    3  1 x,y,z y  
Total: 1MB

Observe que isso reordenará seus dados.

#> DT
#   x y     z
#1: e 1  TRUE
#2: d 2  TRUE
#3: c 3 FALSE
#4: b 4 FALSE
#5: a 5 FALSE

Agora é desnecessário

Antes da v1.9.6, você precisava definir uma chave para certas operações, especialmente para unir tabelas. Os desenvolvedores do data.table aceleraram e introduziram um recurso "on=" que pode substituir a dependência de chaves. Veja SO responda aqui para uma discussão detalhada.

Em janeiro de 2017, os desenvolvedores escreveram uma vinheta em torno de índices secundários que explica a sintaxe “on” e permite que outras colunas sejam identificadas para indexação rápida.

Criando índices secundários?

De maneira semelhante à chave, você pode setindex(DT, key.col) ou setindexv(DT, "key.col.string"), onde DT é seu data.table. Remova todos os índices com setindex(DT, NULL).

Veja seus índices secundários com indices(DT).

Por que índices secundários?

Isso não classifica a tabela (ao contrário da chave), mas permite indexação rápida usando a sintaxe “on”. Observe que pode haver apenas uma chave, mas você pode usar vários índices secundários, o que evita a necessidade de redigitar e redefinir a tabela. Isso acelerará seu subconjunto ao alterar as colunas nas quais você deseja subconjunto.

Lembre-se, no exemplo acima, y ​​era a chave para a tabela DT:

DT
# x y     z
# 1: e 1  TRUE
# 2: d 2  TRUE
# 3: c 3 FALSE
# 4: b 4 FALSE
# 5: a 5 FALSE

# Let us set x as index 
setindex(DT, x)

# Use indices to see what has been set
indices(DT)
# [1] "x"

# fast subset using index and not keyed column
DT["c", on ="x"]
#x y     z
#1: c 3 FALSE

# old way would have been rekeying DT from y to x, doing subset and 
# perhaps keying back to y (now we save two sorts)
# This is a toy example above but would have been more valuable with big data sets
