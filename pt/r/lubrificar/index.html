Sintaxe #

  • ymd_hms(…, quiet = FALSE, tz = “UTC”, locale = Sys.getlocale(“LC_TIME”))
  • agora(tzone = “”)
  • intervalo(início, fim, tzone = attr(início, “tzone”))
  • duração(num = NULL, unidades = “segundos”, …)
  • period(num = NULL, unidades = “segundo”, …)

Para instalar o pacote do CRAN:

install.packages("lubridate")

Para instalar a versão de desenvolvimento do Github:

library(devtools)
# dev mode allows testing of development packages in a sandbox, without interfering
# with the other packages you have installed.
dev_mode(on=T)
install_github("hadley/lubridate")
dev_mode(on=F)

Para obter vinhetas no pacote lubridate:

vignette("lubridate")

Para obter ajuda sobre alguma função foo:

help(foo)     # help about function foo
?foo          # same thing

# Example
# help("is.period")
# ?is.period

Para obter exemplos para uma função foo:

example("foo")

# Example
# example("interval")

Analisando datas e datas de strings com lubridate #

O pacote lubridate fornece funções convenientes para formatar objetos de data e data e hora a partir de strings de caracteres. As funções são permutações de

Carta Elemento para analisar Equivalente de base R
e ano %y, %Y
m (com ye d) mês %m, %b, %h, %B
d dia %d, %e
h hora %H, %I%p
m (com h e s) minuto %M
s segundos %S

por exemplo. ymd() para analisar uma data com o ano seguido pelo mês seguido pelo dia, por exemplo "2016-07-22", ou ymd_hms() para analisar uma data e hora na ordem ano, mês, dia, horas, minutos, segundos, por exemplo "2016-07-22 13:04:47".

As funções são capazes de reconhecer a maioria dos separadores (como /, - e espaço em branco) sem argumentos adicionais. Eles também trabalham com separadores inconsistentes.

Datas #

As funções de data retornam um objeto da classe Date.

library(lubridate) 

mdy(c(' 07/02/2016 ', '7 / 03 / 2016', ' 7 / 4 / 16 '))
## [1] "2016-07-02" "2016-07-03" "2016-07-04"

ymd(c("20160724","2016/07/23","2016-07-25"))    # inconsistent separators
## [1] "2016-07-24" "2016-07-23" "2016-07-25"

Datetimes #

Funções de utilidade #

Datetimes podem ser analisados ​​usando variantes ymd_hms incluindo ymd_hm e ymd_h. Todas as funções de data e hora podem aceitar um argumento de fuso horário tz semelhante ao de as.POSIXct ou strptime, mas cujo padrão é "UTC" em vez do fuso horário local.

As funções datetime retornam um objeto da classe POSIXct.

x <- c("20160724 130102","2016/07/23 14:02:01","2016-07-25 15:03:00")
ymd_hms(x, tz="EST")
## [1] "2016-07-24 13:01:02 EST" "2016-07-23 14:02:01 EST"
## [3] "2016-07-25 15:03:00 EST"

ymd_hms(x)
## [1] "2016-07-24 13:01:02 UTC" "2016-07-23 14:02:01 UTC"
## [3] "2016-07-25 15:03:00 UTC"

Funções do analisador #

lubridate também inclui três funções para analisar data e hora com uma string de formatação como as.POSIXct ou strptime:

Função Classe de saída Strings de formatação aceitas
parse_date_time POSIXct Flexível. Aceitará estilo strptime com estilo de nome de função de data e hora % ou lubridate, por exemplo, "ymd hms". Aceitará um vetor de pedidos para dados heterogêneos e adivinhará qual é apropriado.
parse_date_time2 POSIXct padrão; if lt = TRUE, POSIXlt Rigoroso. Aceita apenas tokens strptime (com ou sem %) de um conjunto limitado.
fast_strptime POSIXlt padrão; if lt = FALSE, POSIXct Rigoroso. Aceita apenas tokens strptime delimitados por % com delimitadores (-, /, :, etc.) de um conjunto limitado. 
x <- c('2016-07-22 13:04:47', '07/22/2016 1:04:47 pm')

parse_date_time(x, orders = c('mdy Imsp', 'ymd hms'))
## [1] "2016-07-22 13:04:47 UTC" "2016-07-22 13:04:47 UTC"

x <- c('2016-07-22 13:04:47', '2016-07-22 14:47:58')

parse_date_time2(x, orders = 'Ymd HMS')
## [1] "2016-07-22 13:04:47 UTC" "2016-07-22 14:47:58 UTC"

fast_strptime(x, format = '%Y-%m-%d %H:%M:%S')
## [1] "2016-07-22 13:04:47 UTC" "2016-07-22 14:47:58 UTC"

parse_date_time2 e fast_strptime usam um analisador C rápido para eficiência.

Veja ?parse_date_time para formatar tokens.

Diferença entre período e duração #

Ao contrário das durações, os períodos podem ser usados ​​para modelar com precisão as horas do relógio sem saber quando ocorrem eventos como segundos bissextos, dias bissextos e alterações de horário de verão.

start_2012 <- ymd_hms("2012-01-01 12:00:00")
## [1] "2012-01-01 12:00:00 UTC"

# period() considers leap year calculations.
start_2012 + period(1, "years")
## [1] "2013-01-01 12:00:00 UTC"

# Here duration() doesn't consider leap year calculations. 
start_2012 + duration(1)
## [1] "2012-12-31 12:00:00 UTC"

Instantâneos #

Um instante é um momento específico no tempo. Qualquer objeto de data e hora que se refira a um momento de tempo é reconhecido como um instante. Para testar se um objeto é um instante, use is.instant.

library(lubridate)

today_start <- dmy_hms("22.07.2016 12:00:00", tz = "IST") # default tz="UTC"
today_start
## [1] "2016-07-22 12:00:00 IST"
is.instant(today_start)
## [1] TRUE

now_dt <- ymd_hms(now(), tz="IST")
now_dt
## [1] "2016-07-22 13:53:09 IST"
is.instant(now_dt)
## [1] TRUE

is.instant("helloworld")
## [1] FALSE
is.instant(60)
## [1] FALSE

Intervalos, Durações e Períodos #

Intervalos são a maneira mais simples de registrar intervalos de tempo em lubridate. Um intervalo é um intervalo de tempo que ocorre entre dois instantes específicos.

# create interval by substracting two instants
today_start <- ymd_hms("2016-07-22 12-00-00", tz="IST")
today_start
## [1] "2016-07-22 12:00:00 IST"
today_end <- ymd_hms("2016-07-22 23-59-59", tz="IST")
today_end
## [1] "2016-07-22 23:59:59 IST"
span <- today_end - today_start
span
## Time difference of 11.99972 hours
as.interval(span, today_start)
## [1] 2016-07-22 12:00:00 IST--2016-07-22 23:59:59 IST

# create interval using interval() function
span <- interval(today_start, today_end)
[1] 2016-07-22 12:00:00 IST--2016-07-22 23:59:59 IST

Durações medem a quantidade exata de tempo que ocorre entre dois instantes.

duration(60, "seconds")
## [1] "60s"

duration(2, "minutes")
## [1] "120s (~2 minutes)"

Observação: unidades maiores que semanas não são usadas devido à sua variabilidade.

As durações podem ser criadas usando dseconds, dminutes e outras funções auxiliares de duração. Execute ?quick_durations para obter a lista completa.

dseconds(60)
## [1] "60s"

dhours(2)
## [1] "7200s (~2 hours)"

dyears(1)
## [1] "31536000s (~365 days)"

As durações podem ser subtraídas e adicionadas aos instantes para obter novos instantes.

today_start + dhours(5)
## [1] "2016-07-22 17:00:00 IST"

today_start + dhours(5) + dminutes(30) + dseconds(15)
## [1] "2016-07-22 17:30:15 IST"

As durações podem ser criadas a partir de intervalos.

as.duration(span)
[1] "43199s (~12 hours)"

Períodos medem a mudança na hora do relógio que ocorre entre dois instantes.

Períodos podem ser criados usando a função period bem como outras funções auxiliares como seconds, hours, etc. Para obter uma lista completa de funções auxiliares de período, execute ?quick_periods.

period(1, "hour")
## [1] "1H 0M 0S"

hours(1)
## [1] "1H 0M 0S"

period(6, "months")
## [1] "6m 0d 0H 0M 0S"

months(6)
## [1] "6m 0d 0H 0M 0S"

years(1)
## [1] "1y 0m 0d 0H 0M 0S"

A função is.period pode ser usada para verificar se um objeto é um ponto.

is.period(years(1))
## [1] TRUE

is.period(dyears(1))    
## [1] FALSE

Manipulando data e hora no lubridate #

date <- now()
date
## "2016-07-22 03:42:35 IST"

year(date)
## 2016

minute(date)
## 42

wday(date, label = T, abbr = T)
# [1] Fri
# Levels: Sun < Mon < Tues < Wed < Thurs < Fri < Sat

day(date) <- 31
## "2016-07-31 03:42:35 IST"

# If an element is set to a larger value than it supports, the difference
#  will roll over into the next higher element
day(date) <- 32
## "2016-08-01 03:42:35 IST"

Fusos horários #

with_tz retorna uma data e hora como apareceria em um fuso horário diferente.

nyc_time <- now("America/New_York")
nyc_time
## [1] "2016-07-22 05:49:08 EDT"

# corresponding Europe/Moscow time
with_tz(nyc_time, tzone = "Europe/Moscow")
## [1] "2016-07-22 12:49:08 MSK"

force_tz retorna a data-hora que tem a mesma hora do relógio que x no novo fuso horário.

nyc_time <- now("America/New_York")
nyc_time
## [1] "2016-07-22 05:49:08 EDT"

force_tz(nyc_time, tzone = "Europe/Moscow") # only timezone changes
## [1] "2016-07-22 05:49:08 MSK"

Analisando data e hora no lubridate #

Lubridate fornece uma série de funções ymd() para analisar strings de caracteres em datas. As letras y, m e d correspondem aos elementos de ano, mês e dia de uma data-hora.

mdy("07-21-2016")                 # Returns Date

## [1] "2016-07-21"

mdy("07-21-2016", tz = "UTC")     # Returns a vector of class POSIXt

## "2016-07-21 UTC"

dmy("21-07-2016")                 # Returns Date

## [1] "2016-07-21"

dmy(c("21.07.2016", "22.07.2016")) # Returns vector of class Date

## [1] "2016-07-21" "2016-07-22"

now_dt <- ymd_hms(now(), tz="IST")
