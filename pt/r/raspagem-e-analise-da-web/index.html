Raspagem e análise da Web

  • Raspagem básica com rvest
  • Usando rvest quando o login é necessário

Scraping refere-se ao uso de um computador para recuperar o código de uma página da web. Uma vez que o código é obtido, ele deve ser analisado em uma forma útil para uso posterior em R.

O Base R não possui muitas das ferramentas necessárias para esses processos, portanto, a raspagem e a análise geralmente são feitas com pacotes. Alguns pacotes são mais úteis para scraping (RSelenium, httr, curl, RCurl), alguns para análise (XML, xml2), e alguns para ambos (rvest).

Um processo relacionado é a extração de uma API da Web, que, ao contrário de uma página da Web, retorna dados destinados a serem legíveis por máquina. Muitos dos mesmos pacotes são usados ​​para ambos.

Legalidade #

Alguns sites se opõem a serem raspados, seja devido ao aumento da carga do servidor ou a preocupações com a propriedade dos dados. Se um site proíbe a raspagem nos Termos de Uso, a raspagem é ilegal.

Raspagem básica com rvest #

rvest é um pacote para web scraping e análise por Hadley Wickham inspirado na [Beautiful Soup] do Python(https://www.crummy.com/software/BeautifulSoup /). Ele aproveita as ligações do pacote xml2 do Hadley libxml2 para análise de HTML.

Como parte do arrumado, rvest é piped. Ele usa

  • xml2::read_html para extrair o HTML de uma página da web,
  • que pode então ser subconjunto com suas funções html_node e html_nodes usando seletores CSS ou XPath, e
  • analisado para objetos R com funções como html_text e html_table.

Para extrair a tabela de marcos da página da Wikipedia em R, o código ficaria assim

library(rvest)

url <- 'https://en.wikipedia.org/wiki/R_(programming_language)'

        # scrape HTML from website
url %>% read_html() %>% 
    # select HTML tag with class="wikitable"
    html_node(css = '.wikitable') %>% 
    # parse table into data.frame
    html_table() %>%
    # trim for printing
    dplyr::mutate(Description = substr(Description, 1, 70))

Enquanto isso retorna um data.frame, observe que, como é típico para dados raspados, ainda há mais limpeza de dados a ser feita: aqui, formatação de datas, inserção de NAs e assim por diante.

Observe que os dados em um formato retangular menos consistente podem levar loops ou outros ajustes adicionais para analisar com êxito. Se o site faz uso de jQuery ou outros meios para inserir conteúdo, read_html pode ser insuficiente para raspar, e um scraper mais robusto como RSelenium pode ser necessário.

Usando rvest quando o login é necessário #

Um problema comum ao se desfazer de uma web é como inserir um ID de usuário e senha para fazer login em um site.

Neste exemplo que criei para rastrear minhas respostas postadas aqui para estouro de pilha. O fluxo geral é fazer login, ir para uma página da web coletar informações, adicionar um dataframe e depois passar para a próxima página.

library(rvest) 

#Address of the login webpage
login<-"https://stackoverflow.com/users/login?ssrc=head&returnurl=http%3a%2f%2fstackoverflow.com%2f"

#create a web session with the desired login address
pgsession<-html_session(login)
pgform<-html_form(pgsession)[[2]]  #in this case the submit is the 2nd form
filled_form<-set_values(pgform, email="*****", password="*****")
submit_form(pgsession, filled_form)

#pre allocate the final results dataframe.
results<-data.frame()  

#loop through all of the pages with the desired info
for (i in 1:5)
{
  #base address of the pages to extract information from
  url<-"http://stackoverflow.com/users/**********?tab=answers&sort=activity&page="
  url<-paste0(url, i)
  page<-jump_to(pgsession, url)

  #collect info on the question votes and question title
  summary<-html_nodes(page, "div .answer-summary")
  question<-matrix(html_text(html_nodes(summary, "div"), trim=TRUE), ncol=2, byrow = TRUE)

  #find date answered, hyperlink and whether it was accepted
  dateans<-html_node(summary, "span") %>% html_attr("title")
  hyperlink<-html_node(summary, "div a") %>% html_attr("href")
  accepted<-html_node(summary, "div") %>% html_attr("class")

  #create temp results then bind to final results 
  rtemp<-cbind(question, dateans, accepted, hyperlink)
  results<-rbind(results, rtemp)
}

#Dataframe Clean-up
names(results)<-c("Votes", "Answer", "Date", "Accepted", "HyperLink")
results$Votes<-as.integer(as.character(results$Votes))
results$Accepted<-ifelse(results$Accepted=="answer-votes default", 0, 1)

O loop neste caso é limitado a apenas 5 páginas, isso precisa ser alterado para se adequar ao seu aplicativo. Substituí os valores específicos do usuário por ******, espero que isso forneça alguma orientação para seu problema.
