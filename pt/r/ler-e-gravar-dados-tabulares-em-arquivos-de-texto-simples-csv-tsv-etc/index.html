Sintaxe #

  • read.csv(file, header = TRUE, set = “,”, quote = “"”, dec = “.”, fill = TRUE, comment.char = “”, …)

  • read.csv2(file, header = TRUE, set = “;”, quote = “"”, dec = “,”, fill = TRUE, comment.char = “”, …)

  • readr::read_csv(file, col_names = TRUE, col_types = NULL, locale = default_locale(), na = c("", “NA”), comment = “”, trim_ws = TRUE, skip = 0, n_max = - 1, progresso = interativo())

  • data.table::fread(input, sep=“auto”, sep2=“auto”, nrows=-1L, header=“auto”, na.strings=“NA”, stringsAsFactors=FALSE, verbose=getOption(“datatable.verbose”), autostart=1L, skip=0L, select=NULL, drop=NULL, colClasses=NULL, integer64=getOption(“datatable.integer64”), # default: “integer64” dec=if (set!=".") “.” else “,”, col.names, check.names=FALSE, encoding=“unknown”, strip.white=TRUE, showProgress=getOption(“datatable.showProgress”), # padrão: TRUE data.table=getOption(“datatable.fread.datatable”) # padrão: TRUE )

Parâmetros #

Parâmetro Detalhes
arquivo nome do arquivo CSV a ser lido
cabeçalho lógico: o arquivo .csv contém uma linha de cabeçalho com nomes de coluna?
setembro caractere: símbolo que separa as células de cada linha
cotação caractere: símbolo usado para citar strings de caracteres
dezembro caractere: símbolo usado como separador decimal
preencher lógico: quando TRUE, linhas com comprimento desigual são preenchidas com campos em branco.
comentário.char caractere: caractere usado como comentário no arquivo csv. As linhas precedidas por esse caractere são ignoradas.
… argumentos extras a serem passados ​​para read.table

Observe que a exportação para um formato de texto simples sacrifica muitas das informações codificadas nos dados, como classes de variáveis, em prol da ampla portabilidade. Para casos que não exigem tal portabilidade, um formato como .RData ou Feather pode ser mais útil.

A entrada/saída para outros tipos de arquivos é abordada em vários outros tópicos, todos vinculados a partir de Entrada e saída.

Importando arquivos .csv #

Importando usando base R #

Arquivos de valores separados por vírgula (CSVs) podem ser importados usando read.csv, que envolve read.table, mas usa sep = "," para definir o delimitador como uma vírgula.

# get the file path of a CSV included in R's utils package
csv_path <- system.file("misc", "exDIF.csv", package = "utils")

# path will vary based on installation location
csv_path
## [1] "/Library/Frameworks/R.framework/Resources/library/utils/misc/exDIF.csv"

df <- read.csv(csv_path)

df
##    Var1 Var2
## 1  2.70    A
## 2  3.14    B
## 3 10.00    A
## 4 -7.00    A

Uma opção amigável, file.choose, permite navegar pelos diretórios:

df <- read.csv(file.choose())

Notas #

  • Ao contrário do read.table, o padrão read.csv é header = TRUE, e usa a primeira linha como nomes de coluna.
  • Todas essas funções irão converter strings para a classe factor por padrão a menos que as.is = TRUE ou stringsAsFactors = FALSE.
  • A variante read.csv2 tem como padrão sep = ";" e dec = "," para uso em dados de países onde a vírgula é usada como ponto decimal e o ponto e vírgula como separador de campo.

Importando usando pacotes #

A função read_csv do pacote readr oferece desempenho muito mais rápido, uma barra de progresso para arquivos grandes e opções padrão mais populares do que read.csv padrão, incluindo stringsAsFactors = FALSE.

library(readr)

df <- read_csv(csv_path)

df
## # A tibble: 4 x 2
##    Var1  Var2
##   <dbl> <chr>
## 1  2.70     A
## 2  3.14     B
## 3 10.00     A
## 4 -7.00     A

Importando com data.table #

O pacote data.table introduz a função fread. Embora seja semelhante ao read.table, fread é geralmente mais rápido e flexível, adivinhando o delimitador do arquivo automaticamente.

# get the file path of a CSV included in R's utils package
csv_path <- system.file("misc", "exDIF.csv", package = "utils")

# path will vary based on R installation location
csv_path
## [1] "/Library/Frameworks/R.framework/Resources/library/utils/misc/exDIF.csv"

dt <- fread(csv_path)

dt
##     Var1 Var2
## 1:  2.70    A
## 2:  3.14    B
## 3: 10.00    A
## 4: -7.00    A

Onde o argumento input é uma string que representa:

  • o nome do arquivo (e.g. "filename.csv"),
  • um comando shell que atua em um arquivo (por exemplo "grep 'word' filename"), ou
  • a própria entrada (e.g. "input1, input2 \n A, B \n C, D").

fread retorna um objeto da classe data.table que herda da classe data.frame, adequado para uso com o uso de [] do data.table. Para retornar um data.frame comum, defina o parâmetro data.table como FALSE:

df <- fread(csv_path, data.table = FALSE)

class(df)
## [1] "data.frame"

df
##    Var1 Var2
## 1  2.70    A
## 2  3.14    B
## 3 10.00    A
## 4 -7.00    A

Notas #

  • fread não tem todas as mesmas opções que read.table. Um argumento ausente é na.comment, que pode levar a comportamentos indesejados se o arquivo de origem contiver #.
  • fread usa apenas " para o parâmetro quote.
  • fread usa poucas (5) linhas para adivinhar os tipos de variáveis.

Exportando arquivos .csv #

Exportando usando base R #

Os dados podem ser gravados em um arquivo CSV usando write.csv():

write.csv(mtcars, "mtcars.csv")

Parâmetros comumente especificados incluem row.names = FALSE e na = "".

Exportando usando pacotes #

readr::write_csv é significativamente mais rápido que write.csv e não escreve nomes de linhas.

library(readr)

write_csv(mtcars, "mtcars.csv")

Importar vários arquivos csv #

files = list.files(pattern="*.csv")
data_list = lapply(files, read.table, header = TRUE)

Isso lê todos os arquivos e os adiciona a uma lista. Depois, se todos os data.frames tiverem a mesma estrutura, eles podem ser combinados em um big data.frame:

df <- do.call(rbind, data_list)

Importando arquivos de largura fixa #

Arquivos de largura fixa são arquivos de texto em que as colunas não são separadas por nenhum delimitador de caractere, como , ou ;, mas têm um comprimento de caractere fixo (largura). Os dados geralmente são preenchidos com espaços em branco.

Um exemplo:

Column1 Column2   Column3           Column4Column5 
1647    pi        'important'       3.141596.28318
1731    euler     'quite important' 2.718285.43656
1979    answer    'The Answer.'     42     42

Vamos supor que esta tabela de dados exista no arquivo local constants.txt no diretório de trabalho.

Importando com base R# #

df <- read.fwf('constants.txt', widths = c(8,10,18,7,8), header = FALSE, skip = 1)

df
#>     V1     V2                 V3         V4        V5
#> 1 1647     pi         'important'   3.14159   6.28318
#> 2 1731  euler   'quite important'   2.71828   5.43656
#> 3 1979 answer       'The Answer.'   42        42.0000

Observação:

  • Os títulos das colunas não precisam ser separados por um caractere (Column4Column5)
  • O parâmetro widths define a largura de cada coluna
  • Cabeçalhos não separados não são legíveis com read.fwf()

Importando com readr #

library(readr)

df <- read_fwf('constants.txt', 
               fwf_cols(Year = 8, Name = 10, Importance = 18, Value = 7, Doubled = 8), 
               skip = 1)
df
#> # A tibble: 3 x 5
#>    Year    Name        Importance    Value  Doubled
#>    <int>   <chr>           <chr>     <dbl>    <dbl>
#> 1  1647      pi       'important'  3.14159  6.28318
#> 2  1731   euler 'quite important'  2.71828  5.43656
#> 3  1979  answer     'The Answer.' 42.00000 42.00000

Observação:

  • as funções auxiliares fwf_* do readr oferecem formas alternativas de especificar comprimentos de coluna, incluindo adivinhação automática (fwf_empty)
  • readr é mais rápido que a base R
  • Os títulos das colunas não podem ser importados automaticamente do arquivo de dados

Importando arquivos .tsv como matrizes (R básico) #

Muitas pessoas não usam file.path ao criar o caminho para um arquivo. Mas se você estiver trabalhando em máquinas Windows, Mac e Linux, geralmente é uma boa prática usá-lo para criar caminhos em vez de ‘colar’.

FilePath <- file.path(AVariableWithFullProjectPath,"SomeSubfolder","SomeFileName.txt.gz")

Data <- as.matrix(read.table(FilePath, header=FALSE, sep ="\t"))

Geralmente isso é suficiente para a maioria das pessoas.

Às vezes acontece que as dimensões da matriz são tão grandes que o procedimento de alocação de memória deve ser levado em consideração durante a leitura na matriz, o que significa ler na matriz linha por linha.

Veja o exemplo anterior, neste caso FilePath contém um arquivo de dimensão 8970 8970 com 79% das células contendo valores diferentes de zero.

system.time(expr=Data<-as.matrix(read.table(file=FilePath,header=FALSE,sep=" ") ))

system.time diz que 267 segundos foram levados para ler o arquivo.

   user  system elapsed
265.563   1.949 267.563

Da mesma forma, este arquivo pode ser lido linha por linha,

FilePath <- "SomeFile"
connection<- gzfile(FilePath,open="r")
TableList <- list()
Counter <- 1
system.time(expr= while ( length( Vector<-as.matrix(scan(file=connection, sep=" ", nlines=1, quiet=TRUE)) ) > 0 ) {
    TableList[[Counter]]<-Vector
    Counter<-Counter+1
})
   user  system elapsed
165.976   0.060 165.941
close(connection)
system.time(expr=(Data <- do.call(rbind,TableList)))
   user  system elapsed
  0.477   0.088   0.565

Há também o pacote futile.matrix que implementa um método read.matrix, o próprio código se revelará a mesma coisa descrita no exemplo 1.
