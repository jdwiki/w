Sintaxe #

  • @Teste
  • @Test(attribute1 = attributeValue, atrribute2 = attributeValue, etc)

Parâmetros #

Parâmetro Detalhes
alwaysRun Se definido como true, este método de teste sempre será executado mesmo que dependa de um método que falhou.
dataProvider O nome do provedor de dados para este método de teste.
dataProviderClass A classe onde procurar o provedor de dados. Se não for especificado, o provedor de dados será consultado na classe do método de teste atual ou em uma de suas classes base. Se esse atributo for especificado, o método do provedor de dados precisará ser estático na classe especificada.
dependsOnGroups A lista de grupos dos quais este método depende.
dependsOnMethods A lista de métodos dos quais este método depende.
descrição A descrição para este método.
enabled Se os métodos nesta classe/método estão ativados.
expectedExceptions A lista de exceções que um método de teste deve lançar. Se nenhuma exceção ou diferente de uma nesta lista for lançada, este teste será marcado como uma falha.
grupos A lista de grupos a que esta classe/método pertence.
invocationCount O número de vezes que este método deve ser invocado.
invocationTimeOut O número máximo de milissegundos que este teste deve levar para o tempo acumulado de todas as contagens de invocação. Este atributo será ignorado se invocationCount não for especificado.
priority A prioridade para este método de teste. As prioridades mais baixas serão agendadas primeiro.
successPercentage A porcentagem de sucesso esperada deste método
singleThreaded Se definido como true, todos os métodos nesta classe de teste são garantidos para serem executados no mesmo thread, mesmo se os testes estiverem sendo executados com parallel="methods". Este atributo só pode ser usado em nível de classe e será ignorado se usado em nível de método. Observação: este atributo costumava ser chamado de sequencial (agora obsoleto).
timeOut O número máximo de milissegundos que este teste deve levar.
threadPoolSize O tamanho do conjunto de encadeamentos para este método. O método será invocado de vários threads conforme especificado por invocationCount. Nota: este atributo é ignorado se invocationCount não for especificado

Exemplo rápido na anotação @Test #

A anotação @Test pode ser aplicada a qualquer classe ou método. Esta anotação marca uma classe ou um método como parte do teste.

  1. @Test no nível do método - marque o método anotado como método de teste
  2. @Test no nível da turma
    • The effect of a class level @Test annotation is to make all the public methods of the class to become test methods even if they are not annotated.
    • @Test annotation can also be repeated on a method if you want to add certain attributes.

Exemplo de @Test no nível do método:

import org.testng.annotations.Test;

public class TestClass1 {
    public void notTestMethod() {
    }

    @Test
    public void testMethod() {
    }
}

Exemplo de @Test em nível de classe:

import org.testng.annotations.Test;

@Test
public class TestClass2 {
    public void testMethod1() {
    }

    @Test
    public void testMethod2() {
    }
}
