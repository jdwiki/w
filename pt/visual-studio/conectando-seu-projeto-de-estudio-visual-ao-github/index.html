Publicando seu projeto em um repositório do github removendo dados confidenciais #

as etapas neste exemplo usarão a seguinte estrutura de projeto como demonstração

e pretendemos exportá-lo para o Repositório “GHTuts” [Observe que o Repo ainda não existe no github] mas deixe o “SensitiveProject” sem publicar pois contém algumas senhas, chaves, etc.

  1. Antes de tudo, certifique-se de que o plug-in de controle de origem esteja definido como “Git” em “Ferramentas > Opções > Seleção de plug-in”
  2. Se você não conseguir ver a guia “Team Explorer”, visualize-a no visual studio assim

  1. Vá para a pasta de solução local e crie um novo arquivo chamado “.gitignore.txt” [Observação] esta etapa só é importante se você tiver algumas informações confidenciais em seu projeto, caso contrário, deixe o visual studio criá-las para você

  2. Agora abra o arquivo “.gitignore.txt” e cole-o nele, este é um modelo para ignorar arquivos comuns do visual studio (verifique os links abaixo)

  1. Agora adicione sua pasta de projeto sensível ao arquivo “.gitignore.txt” em qualquer linha que não contenha #, então apenas adicione-o no final, e deve ser algo assim

  1. Clique com o botão direito do mouse na solução e escolha “Adicionar solução ao controle de origem…” [Observação] ele pode pedir para você salvar a solução antes de continuar

  2. Agora você tem um git Repo “LOCAL” no seu pc, do qual o VS irá ler, mas sem um github Repo, e você verá um pequeno ícone de cadeado azul ao lado de cada arquivo na solução que foi adicionado ao git e um círculo vermelho no projeto ignorado

    [Note]
    for more information about .gitignore file, check these links

  • https://help.github.com/articles/ignoring-files/
  • https://github.com/github/gitignore

  1. Vá para a guia “Team Explorer” e depois “Sincronizar”

  2. Agora criamos um repositório do vs para o github assim, pressione o botão “Começar”

  3. Agora preencha suas informações no github para o novo Repo e clique em “Publicar”

  4. Agora, quando vamos ao github, vemos que nosso repositório local foi publicado no github sem nosso projeto sensível [Observação] a url do repositório será algo assim https://github.com/<nome do usuário>/<nome do repositório>
