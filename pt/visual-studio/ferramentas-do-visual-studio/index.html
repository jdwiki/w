Lente de código #

A lente de código é uma maneira simples de saber o que acontece com o código. Aqui você pode encontrar uma imagem com o número de referências de um método ou classe.

Se você não conseguir ver a lente do código, consulte esta pergunta: http://stackoverflow.com/questions/33144785/missing-codelens-references-count-in-vs-2015-community-edition

Trechos #

#Introdução

Desde o Visual Studio 2005, você pode criar trechos de código do Intellisense. Isso permite que você gere algum código apenas digitando uma palavra-chave e pressione duas vezes a tecla tab.

Usando o código #

O código XML que você precisa para fazer um trecho de código do Intellisense está abaixo:

<?xml version="1.0" encoding="utf-8"?>

<CodeSnippets xmlns="http://schemas.microsoft.com/VisualStudio/CodeSnippet">
  <CodeSnippet Format="1.0.0"> <!-- format attribute is required -->

    <Header> <!-- 1 -->
      <Title></Title>
      <Author></Author>
      <Shortcut></Shortcut>
      <Description></Description>
      <Keywords>
        <Keyword>abc<Keyword>
        <Keyword>def<Keyword>
      </keywords>
    </Header>

    <Snippet> <!-- 2 -->

      <Imports> <!-- 2.1 -->
        <Import>
          <Namespace>System</Namespace>
        </Import>
      </Imports>

      <Declarations> <!-- 2.2 -->
        <Literal Editable="true/false"> <!-- 2.2.1 -->
          <ID>example</ID>
          <Type>System.String</Type>
          <ToolTip>A tip you can show</ToolTip>
          <Default>default value</Default>
          <Function></Function> <!-- 2.2.2 -->
        </Literal>

        <Object> <!-- 2.2.1 -->
          <ID>example</ID>
          <Type>System.String</Type>
          <ToolTip>A tip you can show</ToolTip>
          <Default>default value</Default>
          <Function></Function> <!-- 2.2.2 -->
        </Object>
      </Declarations>

      <References> <!-- 2.3 -->
        <Reference>
          <Assembly>System.Data.dll</Assembly>
        </Reference>
      </References>

      <Code Language=""> <!-- 2.4 -->
        <![CDATA[
            <!-- your code here if you use literals use dollar chars  -->
        ]]>
      </Code>

    </Snippet>

  </CodeSnippet>
</CodeSnippets>

Na tag do snippet, você tem duas tags obrigatórias chamadas Header e Snippet. Você pode encontrar mais informações nos próximos títulos. O número próximo ao nome corresponde aos números do código acima.

Pode haver zero ou mais elementos CodeSnippets adicionados ao elemento CodeSnippets.

1. Cabeçalho #

Na Header-tag, você pode colocar algumas informações específicas sobre o snippet e o que ele faz. As tags importantes que você pode usar dentro dessa tag são:

Elemento Descrição
Título O título do trecho. Este atributo é obrigatório.
Autor O autor do trecho.
Atalho É o atalho, você pode usar para gerar o código. Observe que isso só pode conter letras e números e deve começar com uma letra.
Observação: Lembre-se também de dar ao snippet um nome e um atalho bons e exclusivos. Caso contrário, isso causará problemas ao importar o snippet para o Visual Studio.
Descrição Fornece mais informações sobre o snippet, se necessário.
HelpUrl Um URL para uma página de ajuda na Internet.
Palavras-chave Agrupa um ou mais elementos de palavra-chave.
Tipos de snippets

Agrupa elementos SnippetType. Este elemento contém um valor de texto e deve ser um dos seguintes valores. Os tipos de snippet são mesclados com uma barra.


  • SurroundsWith: permite que o snippet de código seja colocado em torno de um trecho de código selecionado.
  • Expansão: permite que o snippet de código seja inserido no cursor.
  • Refactoring: especifica que o snippet de código é usado durante a refatoração do Visual C# . A refatoração não pode ser usada em trechos de código personalizados.
Lista de fontes: msdn.microsoft.com

Tabela de origem (mas edições): msdn.microsoft.com

2. Fragmento #

Na tag de snippet, você pode usar três tags diferentes. Isso pode ser:

  • Importações
  • Declarações
  • Código (obrigatório)
  • Referências

Estes são explicados abaixo.

2.1 Importações #

Imports contém os namespaces necessários que você precisa para o código. Use a tag import dentro desta tag e aqui você pode colocar os namespaces necessários, cada um com a tag Namespace.

2.2 Declarações #

Declarations pode ser usado para declarar alguns literais ou objetos em seu código na tag Code. Os filhos são literais e objetos.

2.2.1 Literais e objetos

#

Literais e objetos definem os literais e objetos do snippet de código que você pode editar. Funcionalidade são literais e objetos são os mesmos, mas tem uma restrição de tipo adicional.

O Literal e object-tag podem conter os próximos filhos:

  • ID: O ID do literal (obrigatório)
  • Type: O tipo desse objeto, incluindo namespace e classe (exigido por objetos)
  • ToolTip: dá uma dica
  • Default: Um valor padrão desse objeto (obrigatório)
  • Funções

Nos snippets, existem alguns literais predefinidos. Eles estão listados abaixo:

Literal Detalhes
$fim$ Marca o local para colocar o cursor após a inserção do trecho de código.
$selecionado$ Representa o texto selecionado no documento que deve ser inserido no snippet quando ele é invocado. Exemplo, se você tiver:
A $selected$ é um objeto que eu gosto.
e a palavra foi carro selecionada quando você invocou o modelo, você obteria:
Um carro é um objeto que eu gosto.

2.2.2 Funções

#

Funções na tag Literal ou Object significa que você pode usar uma função para gerar código dependendo de outro elemento. Existem três funções que eu conheço:

Função Descrição Linguagem
GenerateSwitchCases (EnumerationLiteral) Gera uma instrução switch e um conjunto de instruções case para os membros da enumeração especificada pelo parâmetro EnumerationLiteral. O parâmetro EnumerationLiteral deve ser uma referência a um literal de enumeração ou a um tipo de enumeração. Visual C# e Visual J#1
ClassName() Retorna o nome da classe que contém o snippet inserido. Visual C# e Visual J#1
SimpleTypeName(TypeName) Reduz o parâmetro TypeName para sua forma mais simples no contexto em que o snippet foi invocado. Visual C#

1 disponível apenas no Visual Studio 2005.

Tabela de origem: msdn.microsoft.com

Atributos para os Elementos Literal e Objeto

#

As tags Literal e Object podem ter alguns atributos opcionais.

Atributo Descrição Modelo
Editável Especifica se você pode ou não editar o literal após a inserção do trecho de código. O valor padrão deste atributo é true. boleano

Tabela de origem: msdn.microsoft.com

2.3 Referências #

Agrupa elementos de referência que contêm informações sobre referências de assembly para o trecho de código. Isso pode conter os próximos elementos:

  • Assembly: contém o nome do assembly pelo snippet de código (obrigatório)
  • Url: Contém um site que fornece mais informações sobre a montagem

2.4 Código #

Código é o código que você irá gerar entre <![CDATA[ e ]]>. Coloque o ‘ID’ do seu literal entre os caracteres do dólar e o Visual Studio pedirá que você altere esses valores padrão se as declarações forem preenchidas. Aqui, você tem um exemplo para C# e VB para o atalho propfull.

<!-- ... Other code ... -->
<Declarations>
  <Literal>
    <Id>variablename</Id>
    <Default>_myproperty</Default>
  </Literal>

  <Literal>
    <Id>propertytype</Id>
    <Default>int</Default>
  </Literal>

  <Literal>
    <Id>propertyname</Id>
    <Default>myproperty</Default>
  </Literal>
</Declarations>

<Code Language="CSharp">
  <![CDATA[
    private $propertyvalue$ $variablename$;

    public $propertyvalue$ $propertyname$
    {
        get { return $variablename$; }
        set { $Variablename$ = Value; }
    }
  ]]>
</Code>

<!-- ... Other code ... -->

<Declarations>
  <Literal>
    <Id>variablename</Id>
    <Default>_myproperty</Default>
  </Literal>

  <Literal>
    <Id>propertytype</Id>
    <Default>int</Default>
  </Literal>

  <Literal>
    <Id>propertyname</Id>
    <Default>myproperty</Default>
  </Literal> 
</Declarations>

<Code Language="VB">
  <![CDATA[
    Private $variablename$ As $propertyvalue$ 
    
    Public Property $propertyname$ As $propertyvalue$
        Get
            Return $variablename$ 
        End Get

        Set (ByVal value As $propertyvalue$)
            $variablename$ = value
        End Set
    End Property
  ]]>
</Code>

Outros atributos opcionais são:

Atributo Descrição
Delimitador Especifica o delimitador usado para descrever literais e objetos no código. Por padrão, o delimitador é $.
Tipo Especifica o tipo de código que o snippet contém e, portanto, o local no qual um snippet de código deve ser inserido para que o snippet de código seja compilado.

Os valores válidos para a variável kind são:

Valor Descrição
corpo do método Especifica que o trecho de código é um corpo de método e, portanto, deve ser inserido dentro de uma declaração de método.
método decl Especifica que o trecho de código é um método e, portanto, deve ser inserido em uma classe ou módulo.
tipo decl Especifica que o trecho de código é um tipo e, portanto, deve ser inserido em uma classe, módulo ou namespace.
arquivo Especifica que o fragmento é um arquivo de código completo. Esses trechos de código podem ser inseridos sozinhos em um arquivo de código ou dentro de um namespace.
qualquer Especifica que o snippet pode ser inserido em qualquer lugar. Essa tag é usada para snippets de código independentes do contexto, como comentários.

Importar snippet para o Visual Studio #

  1. Salve o código XML e dê a extensão .snippet.

  2. Você pode adicionar o novo trecho criado ao Visual Studio pressionando Control + K, Control + B ou vá para “Ferramentas” → “Gerenciador de trechos de código…”. Isso abre a próxima janela:

  1. Escolha o idioma na caixa de combinação para o idioma em que você criou o snippet. clique em “Import…” e escolha o arquivo que você criou.

  1. Clique em "Concluir". Se o nome do arquivo já foi usado, o Visual Studio vai pedir para substituir o arquivo existente. Você tem três opções:
  • Substituir: substitui o arquivo. Você pode usar essa opção se for editar um snippet antigo.
  • Renomear: Vai renomear o arquivo para um nome exclusivo.
  • Ignorar: cancela a importação. Renomeia o arquivo para um nome exclusivo.

Você também pode adicionar um novo local com todos os snippets que criou clicando no botão “Adicionar…” na primeira janela e selecionar a pasta na “janela de seleção de pasta”. A vantagem agora é que quando um novo snippet válido é adicionado a essa pasta, você pode usá-lo diretamente no Visual Studio.

Observação: teste depois de importar seu snippet para ver se há erros, para que você não tenha problemas ao usar o snippet. Você sempre pode remover ou substituir o snippet se houver um erro.

Substituir ferramentas de mesclagem/comparação #

Chegou a Ferramentas | Opções | Controle de origem | Servidor do Visual Studio Team Foundation

clique em Configurar ferramentas do usuário:

Você pode adicionar substituições separadas para operações ‘Comparar’ e ‘Mesclar’. Clique em Adicionar e selecione a operação que deseja substituir. Você precisa digitar o caminho para a ferramenta que usa e os argumentos exatos que sua ferramenta espera. Por exemplo, para usar o BeyondCompare, adicione os seguintes argumentos " %1 %2 /title1=%6 /title2=%7":

Para Mesclar com o BeyondCompare use os Argumentos “%1 %2 %3 %4 /title1=%6 /title2=%7 /title3=%8 /title4=%9”

Em um post de blog de 2006 o funcionário da MS James Manning pesquisou os argumentos conforme esperado por várias ferramentas: WinDiff, DiffDoc, WinMerge, Beyond Compare, KDiff3, Araxis, Compare It!, SourceGear DiffMerge, TortoiseMerge e Visual SlickEdit. O post é um bom ponto de partida, mas não deixe de conferir a documentação atualizada da sua ferramenta.

É altamente recomendado não usar para ferramentas de mesclagem que são incapazes de mesclagens de 3 vias (por exemplo, WinMerge 2.x).

Estrutura de entidade #

Entity Framework (EF) é um mapeador objeto-relacional que permite que desenvolvedores .NET trabalhem com dados relacionais usando objetos específicos de domínio. Ele elimina a necessidade da maior parte do código de acesso a dados que os desenvolvedores geralmente precisam escrever.

Entity Framework permite que você crie um modelo escrevendo código ou usando caixas e linhas no EF Designer. Ambas as abordagens podem ser usadas para direcionar um banco de dados existente ou criar um novo banco de dados.

Fonte e mais informações: documentação do Entity Framework
