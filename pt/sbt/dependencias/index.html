Adicionar uma dependência de biblioteca gerenciada #

libraryDependency é a SettingKey que lida com dependências de biblioteca ‘gerenciadas’, que são dependências que são baixadas automaticamente, correspondendo às versões fornecidas. Para adicionar uma única dependência:

libraryDependencies += "com.typesafe.slick" %% "slick" % "3.2.0-M1"

A primeira parte, "com.typesafe.slick", indica o pacote da biblioteca. A segunda parte, "slick", é a biblioteca em questão. A parte final, "3.2.0-M1", é a versão. Como a biblioteca é unida por %%, a versão do Scala fornecida pela chave de configuração scalaVersion será utilizada.

Você pode adicionar várias bibliotecas de uma vez usando ++=:

libraryDependencies ++= Seq(
  "com.typesafe.slick" %% "slick" % "3.2.0-M1" % "compile",
  "com.typesafe.slick" %% "slick-hikaricp" % "3.2.0-M1",
  "mysql" % "mysql-connector-java" % "latest.release"
)

Lembre-se da natureza funcional do Scala, permitindo calcular dependências. Apenas lembre-se de retornar um Seq:

libraryDependencies ++= {
  lazy val liftVersion = "3.0-RC3" //Version of a library being used
  lazy val liftEdition = liftVersion.substring(0,3) //Compute a value
  Seq(
    "net.liftweb" %% "lift-webkit" % liftVersion % "compile",  // Use var in Seq
    "net.liftmodules" %% ("ng_" + liftEdition) % "0.9.2" % "compile",  // Use computed var in Seq
  )  // Because this is the last statement, the Seq is returned and appended to libraryDependencies
}

Adicionar um repositório #

Um repositório é um local que o SBT procura por libraryDependencies. Se a compilação reclamar de não encontrar uma dependência, pode estar faltando o repositório correto. Dentro do SBT, os repositórios são listados na SettingKey resolvers:

resolvers += "Flyway" at "https://flywaydb.org/repo"

Isso segue a sintaxe de ‘Nome do repositório’ em ’local da url’.

Fixar biblioteca na versão do projeto do Scala #

Se o seu projeto tem isso:

scalaVersion := 2.11  // Replace '2.11' with the version of Scala your project is running on

Então você pode usar %% para obter automaticamente a versão da biblioteca compilada com a versão do Scala que o projeto está usando:

libraryDependencies += "com.typesafe.slick" %% "slick" % "3.2.0-M1"

Observe que ter as duas linhas acima é equivalente a ter esta linha:

libraryDependencies += "com.typesafe.slick" % "slick_2.11" % "3.2.0-M1"

Fixar biblioteca em uma versão específica do Scala #

Uma biblioteca pode ser ‘fixada’ em uma versão específica do Scala usando o operador % entre o groupId e o artifactId (as duas primeiras strings em uma dependência de biblioteca). Neste exemplo, fixamos a biblioteca com o artifactId de slick na versão Scala 2.10:

 libraryDependencies += "com.typesafe.slick" % "slick_2.10" % "3.2.0-M1"
