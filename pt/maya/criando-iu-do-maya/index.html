Parâmetros #

parâmetro detalhes
e/editar diz ao Maya que você deseja alterar o valor de uma propriedade existente
q/consulta diz ao Maya que você deseja obter o valor de uma propriedade existente

O Maya vem com um kit de ferramentas de interface do usuário bastante completo que inclui janelas, layouts e uma variedade de controles. Isso é implementado usando a estrutura QT em C++, mas exposto a usuários de MEL e Python por meio do conjunto de comandos padrão do Maya.

QT #

Os usuários avançados podem estender a interface do usuário do Maya usando C++ ou Python. As versões do Maya de 2012 a 2016 usam Pyside e QT4; O Maya 2017 usa Pyside2 e QT5. Mais detalhes aqui

Observação: Referências mais antigas na web referem-se ao kit de ferramentas da GUI do Maya como “ELF”; esse ainda é o nome correto, mas raramente é usado.

Funções de retorno de chamada #

Muitos widgets incluem eventos que podem acionar funções de retorno de chamada quando o usuário interage com o widget. Por exemplo, quando um botão é pressionado, uma caixa de seleção marcada ou um menu suspenso escolhido, você pode disparar uma função.

O sinalizador exato associado a esses eventos depende do widget, mas um retorno de chamada típico seria assim:

 def callback_fn(_ignore):
     print "button pressed"

 button = cmds.button(label='press me', command = callback_fn)

Pressionar o botão imprimirá “botão pressionado” na janela do ouvinte. A maioria dos widgets dispara alguns argumentos quando seus callbacks são ativados – o button por exemplo sempre inclui um valor booleano – então você precisa ter certeza de que o manipulador de callback tem a assinatura certa para acompanhar o widget que você está usando. É por isso que callback_fn() recebe um argumento mesmo que não precise dele.

Atribuição de retorno de chamada #

O Maya oferece suporte a duas maneiras diferentes de anexar funções de retorno de chamada:

   # this works, but is not a great idea
   cmds.button(label = 'string reference', command = 'string_name_of_function')
   # use this form whenever possible
   cmds.button(label = 'direct function reference', command = callback_fn)

No primeiro exemplo, o retorno de chamada é atribuído por um valor de string. O Maya encontrará o retorno de chamada no escopo global do Python – que geralmente é difícil de acessar ao escrever um código devidamente organizado. Os retornos de chamada de nome de string também são mais lentos para resolver. O segundo exemplo passa a função Python real para o retorno de chamada – esse formulário é preferido porque é mais rápido e, se você não fornecer uma função válida para a função de retorno de chamada, saberá quando a interface do usuário é criada em vez de quando os widgets da interface do usuário são realmente usados.

Se você quiser passar um valor de argumento para uma função de retorno de chamada, você pode usar um lambda, um closure ou um functools.partial vincular argumentos ao retorno de chamada.

Usando parcial: #

 from functools import partial
 ....
 def callback_fn(myValue, _ignore):  # _ignore swallows the original button argument
     print myValue

 button = cmds.button(label='press me', command = partial(callback_fn, "fooo"))

Usando lambda: #

 def callback_fn(myValue):
     print myValue

 button = cmds.button(label='press me', command = lambda _ignore: callback_fn("fooo"))
 # here the lambda needs to handle the button argument

Usando Fechamentos #

b = cmds.button(label = 'press me')
# by defining this function when `b` exists, we can use it later
# without storing it explicitly
def get_button_label(*_):
    print "label of button", b, " is ", cmds.button(b, q=True, l=True)
cmds.button(b, e=True, c=get_button_label)

Há mais sobre nomes de retorno de chamada de string versus função de retorno de chamada aqui

Exemplo básico de interface do usuário [Python] #

O kit de ferramentas da GUI do Maya cria uma variedade de elementos de interface do usuário de forma simples e imperativa. Existem comandos básicos para criar e editar widgets GUI; os widgets são identificados por um nome de string exclusivo.

Todos os comandos do gui têm a mesma forma básica: você fornece um tipo de comando e o nome da string do objeto no qual deseja trabalhar ou criar, juntamente com sinalizadores que especificam a aparência ou o comportamento do widget. Então, por exemplo, para criar um botão você usaria:

 cmds.button('my_button', label = 'my label')

Isso criará um novo botão gui. Para editar o botão você usaria o mesmo comando com o sinalizador edit (a versão curta é apenas e). Então você pode alterar o rótulo do botão assim:

cmds.button('my_button', e=True, label = 'a different label')

e você pode consultar o valor atual de uma propriedade com o sinalizador query ou q:

cmds.button(`my button`, q=True, label=True)
# 'a different label'

Nomeação de widgets #

Ao criar um novo widget com um comando de interface do usuário, você pode fornecer o nome que deseja que o novo widget obtenha. No entanto, não é garantido: o Maya dará ao botão o nome que você pediu - se você der um caractere que ele não reconhece ou se já houver um widget com o mesmo nome, você poderá receber de volta um nome diferente. É sempre uma boa prática capturar o nome de um novo widget quando ele é criado para evitar surpresas:

 my_button = cmds.button('my_button') 
 cmds.button(my_button, e=True, label = "a new label")

Criando uma janela #

# create a window with a button that closes the window when clicked
window = cmds.window(title='example window')       # create the window
layout = cmds.columnLayout(adjustableColumn=True)  # add a vertical layout

def close_window(*_):
    cmds.deleteUI(window)                          # deletes the window above

button = cmds.button(label= 'press to close", command = close_window)

# show the window
cmds.showWindow(window)

Lambdas e laços #

Lambdas são um atalho útil para conectar comportamentos a elementos GUI.

b = cmds.button("make a cube", command = lambda _: cmds.polyCube())

No entanto, devido à maneira como o Python captura variáveis ​​dentro de lambdas, você pode obter resultados inesperados se vincular comandos usando lambdas dentro de um loop. Por exemplo, isso parece deve produzir botões que criam esferas de tamanhos diferentes:

# warning: doesn't work like it looks!
for n in range(5):
    b = cmds.button("sphere size %i" % n, command = lambda _: cmds.polySphere(radius=n))

Os botões serão rotulados corretamente, mas todos usarão o mesmo raio (4), porque todos os lambdas capturarão esse valor quando o loop for fechado. TLDR: Se você estiver gerando callbacks dentro de um loop, use functools.partial ou outro método para capturar valores - lambdas não funcionam para esta aplicação. Veja aqui para mais detalhes
