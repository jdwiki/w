BITSADMIN #

BITSADMIN foi usado para transferir documentos, baixar sites, baixar arquivos de sites, etc… Este comando é um comando obsoleto e pode ser removido nas próximas atualizações do Windows. Para evitar esse problema, use o novo Powershell BIT cmdlet.

Aqui está um código de exemplo utilizando BITSADMIN.

@echo off
Bitsadmin /create /download Stackoverflow.com
rem download the website StackOverflow.com

DEBUG #

O comando DEBUG foi usado para criar binários/executáveis ​​a partir do arquivo de lote. O comando ainda está disponível nas versões de 32 bits do Windows, mas é capaz de criar binários apenas com instruções de 16 bits, o que o torna inutilizável para máquinas de 64 bits. Agora CERTUTIL é usado para esse propósito que permite decodificar/codificar arquivos binários/mídia dos formatos HEX ou BASE64. Exemplo com um arquivo que cria um arquivo de ícone:

@echo off

del /q /f pointer.jpg >nul 2>nul
certutil -decode "%~f0" pointer.jpg
hh.exe pointer.jpg
exit /b %errorlevel%

ANEXAR #

APPEND era um comando no msdos que permitia usar arquivos de recursos/mídia como se estivessem no mesmo diretório. O comando ainda está disponível em versões de 32 bits do Windows, mas parece que não está funcionando. Em algumas fontes (incluindo microsofts’) é apontado que o comando é substituído por DPATH , mas não é totalmente verdade. Apesar da mensagem de ajuda do DPATH apontar para o comando APPEND, sua sintaxe é a mesma de PATH. Os diretórios listados em DPATH podem ser usados ​​com redirecionamento de entrada ou tipo de comando :

@echo off
dpath %windir%

set /p var=<win.ini
echo using dpath with input redirection:
echo %var%
echo.
echo using dpath with type command:
type win.ini
