Substituição de variável #

Ao contrário de outras linguagens de programação, em um arquivo em lote, uma variável é substituída por seu valor real antes da execução do script em lote. Em outras palavras, a substituição é feita quando o script é lido na memória pelo processador de comandos, não quando o script é executado posteriormente.

Isso permite o uso de variáveis ​​como comandos dentro do script e como parte de outros nomes de variáveis ​​no script, etc. O “script” neste contexto é uma linha - ou bloco - de código, entre colchetes: () .

Mas esse comportamento significa que você não pode alterar o valor de uma variável dentro de um bloco!

SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    ECHO %VAR%
    SET VAR=Goodbye
)

vai imprimir

Hello
Hello

já que (como você vê, ao assistir o script ser executado na janela de comando) ele é avaliado para:

SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    echo Hello
    SET VAR=Goodbye
)

No exemplo acima, o comando ECHO é avaliado como Hello quando o script é lido na memória, então o script irá ecoar Hello para sempre, não importa quantas passagens sejam feitas pelo script.

A maneira de obter o comportamento de variável mais “tradicional” (da variável sendo expandida enquanto o script está em execução) é habilitar a “expansão atrasada”. Isso envolve adicionar esse comando no script antes da instrução de loop (geralmente um loop FOR, em um script em lote) e usar um ponto de exclamação (!) em vez de um sinal de porcentagem (%) no nome da variável:

setlocal enabledelayedexpansion 
SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    echo !VAR!
    SET VAR=Goodbye
)
endlocal

vai imprimir

Hello
Goodbye

A sintaxe %%a em (1,1,2) faz com que o loop seja executado 2 vezes: na primeira ocasião, a variável carrega seu valor inicial de ‘Hello’, mas na segunda passa pelo loop - tendo executado a segunda instrução SET como a última ação na 1ª passagem - isso mudou para o valor revisado ‘Adeus’.

Substituição de variável avançada

Agora, uma técnica avançada. O uso do comando CALL permite que o processador de comandos em lote expanda uma variável localizada na mesma linha do script. Isso pode fornecer expansão multinível, pelo uso repetido de CHAMADA e modificador.

Isso é útil, por exemplo, em um loop FOR. Como no exemplo a seguir, onde temos uma lista numerada de variáveis:

"c:\MyFiles\test1.txt" "c:\MyFiles\test2.txt" "c:\MyFiles\test3.txt"

Podemos conseguir isso usando o seguinte loop FOR:

setlocal enabledelayedexpansion
for %%x in (%*) do (
    set /a "i+=1"
    call set path!i!=%%~!i!
    call echo %%path!i!%%
)
endlocal

Resultado:

c:\MyFiles\test1.txt
c:\MyFiles\test2.txt
c:\MyFiles\test3.txt

Observe que a variável !i! é primeiro expandida para seu valor inicial, 1, então a variável resultante, %1, é expandida para seu valor real de c:\MyFiles\test1.txt. Esta é a expansão dupla da variável i. Na próxima linha, i é novamente expandido duas vezes, pelo uso do comando CALL ECHO junto com o prefixo da variável %%, então impresso na tela (ou seja, exibido na tela).

Em cada passagem sucessiva pelo loop, o número inicial é aumentado em 1 (devido ao código i+=1). Assim, aumenta para ‘2’ na 2ª passagem pelo loop e para ‘3’ na 3ª passagem. Assim, a corda ecoada na tela se altera a cada passagem.

Declaração #

Para criar uma variável simples e atribuí-la a um valor ou string, use o comando SET:

SET var=10

Aqui, o código declara uma nova variável var com um valor de 10. Por padrão, todas as variáveis ​​são armazenadas internamente como strings; isso significa que o valor 10 não é diferente de foo1234 ou Hello, World!

Notas sobre aspas #

As aspas usadas serão incluídas no valor da variável:

SET var="new value"             <-- %var% == '"new value"'

Espaços nas variáveis #

A linguagem de lote considera os espaços como partes aceitáveis ​​de nomes de variáveis. Por exemplo, set var = 10 resultará em uma variável chamada var que contém o valor 10 (observe o espaço extra à direita de var e à esquerda do 10).

Usando aspas para eliminar espaços #

Para evitar espaços, use aspas em toda a atribuição; o nome e o valor da variável. Isso também evita espaços acidentais no final da linha (o caractere ␣ denota um espaço):

SET␣var=my␣new␣value␣           <-- '%var%' == 'my new value '
SET␣"var=my␣new␣value"␣         <-- '%var%' == 'my new value'

Além disso, use aspas ao unir várias instruções com & ou | - alternativamente, coloque o símbolo diretamente após o final do valor da variável:

SET var=val & goto :next        <-- '%var%' == 'val '
SET "var=val" & goto :next      <-- '%var%' == 'val'
SET var=val& goto :next         <-- '%var%' == 'val'

Declare várias variáveis #

Quando várias variáveis ​​são definidas no início do lote, um formulário de definição curto pode ser usado empregando uma string substituição.

@echo off
set "vars=_A=good,_B=,_E=bad,_F=,_G=ugly,_C=,_H=,_I=,_J=,_K=,_L=,_D=6
set "%vars:,=" & set "%"

for /f %%l in ('set _') do echo %%l
exit /b

_A=good
_D=6
_E=bad
_G=ugly

Observe que no exemplo acima, as variáveis ​​são classificadas alfabeticamente nativamente, quando impressas na tela.

Uso #

echo %var%

Este código irá ecoar o valor de var

Se setLocal EnableDelayedExpansion for usado, o seguinte ecoará o valor de var (a expressão padrão %var% não funcionará nesse contexto).

echo !var!

Em arquivos em lote, as variáveis ​​podem ser usadas em qualquer contexto, inclusive como partes de comandos ou partes de outras variáveis. Você não pode chamar uma variável antes de defini-la.

Usando variáveis ​​como comandos:

set var=echo
%var% This will be echoed

Usando variáveis ​​em outras variáveis:

set var=part1
set %var%part2=Hello
echo %part1part2%

Usando uma variável como um array #

É possível criar um conjunto de variáveis ​​que podem agir de forma semelhante a um array (embora não sejam um objeto array real) usando espaços na instrução SET:

@echo off
SET var=A "foo bar" 123
for %%a in (%var%) do (
    echo %%a
)
echo Get the variable directly: %var%

Resultado:

A
"foo bar"
123
Get the variable directly: A "foo bar" 123

Também é possível declarar sua variável usando índices para que você possa recuperar informações específicas. Isso criará várias variáveis, com a ilusão de uma matriz:

@echo off
setlocal enabledelayedexpansion
SET var[0]=A
SET var[1]=foo bar
SET var[2]=123
for %%a in (0,1,2) do (
    echo !var[%%a]!
)
echo Get one of the variables directly: %var[1]%

Resultado:

A
foo bar
123
Get one of the variables directly: foo bar

Observe que no exemplo acima, você não pode fazer referência a var sem informar qual é o índice desejado, porque var não existe por si só. Este exemplo também usa setlocal enabledelayedexpansion em conjunto com os pontos de exclamação em !var[%%a]!. Você pode ver mais informações sobre isso na Documentação do Escopo de Substituição da Variável.

Configurando variáveis ​​de uma entrada #

Usando a chave /p com o comando SET você pode definir variáveis ​​de uma entrada.

Esta entrada pode ser uma entrada de usuário (teclado):

echo Enter your name : 
set /p name=
echo Your name is %name%

Que pode ser simplificado assim:

set /p name=Enter your name :
echo Your name is %name%

Ou você pode obter a entrada de um arquivo:

set /p name=< file.txt

neste caso, você obterá o valor da primeira linha de file.txt

Obtendo o valor de várias linhas em um arquivo:

(
   set /p line1=
   set /p line2=
   set /p line3=

) < file.txt

Operações em Variáveis #

set var=10
set /a var=%var%+10
echo %var%

O valor final de var é 20.

A segunda linha não está funcionando em um bloco de comando usado, por exemplo, em uma condição IF ou em um loop FOR, pois seria necessária uma expansão atrasada em vez da expansão de variável de ambiente padrão.

Aqui está outra maneira melhor de trabalhar também em um bloco de comando:

set var=10
set /A var+=10
echo %var%

O ambiente de prompt de comando é compatível com valores inteiros de 32 bits assinados:

  • adição + e +=
  • subtração - e -=
  • multiplicação * e *=
  • divisão / e /=
  • divisão do módulo % e %=
  • bit a bit AND &
  • bit a bit OU |
  • bit a bit NÃO ~
  • XOR bit a bit ^
  • deslocamento à esquerda bit a bit <<
  • deslocamento à direita bit a bit >>
  • lógico NÃO !
  • unário menos -
  • agrupamento com ( e )

O interpretador de comandos do Windows não oferece suporte a valores inteiros de 64 bits ou valores de ponto flutuante em expressões aritméticas.

Nota: O operador % deve ser escrito em um arquivo batch como %% para ser interpretado como operador.

Em uma janela de prompt de comando, executando a linha de comando set /A Value=8 % 3 atribui o valor 2 à variável de ambiente Value e, adicionalmente, gera 2.

Em um arquivo de lote deve ser escrito set /A Value=8 %% 3 para atribuir o valor 2 à variável de ambiente Value e nada é produzido respectivamente escrito para manipular STDOUT (saída padrão). Uma linha set /A Value=8 % 3 em um arquivo de lote resultaria na mensagem de erro Operador ausente na execução do arquivo em lote.

O ambiente requer a opção /A apenas para operações aritméticas, não para variáveis ​​de string comuns.

Cada string na expressão aritmética após set /A sendo um número ou um operador é automaticamente interpretado como nome de uma variável de ambiente.

Por esta razão referenciar o valor de uma variável com %variable% ou com !variable! não é necessário quando o nome da variável consiste apenas em caracteres de palavra (0-9A-Za-z_) com o primeiro caractere não sendo um dígito que é especialmente útil dentro de um bloco de comando começando com ( e terminando com um ) correspondente.

Os números são convertidos de string para inteiro com a função C/C++ strtol com base sendo zero, o que significa determinação automática de base que pode facilmente resultar em erros inesperados resultados.

Exemplo:

set Divided=11
set Divisor=3

set /A Quotient=Divided / Divisor
set /A Remainder=Divided %% Divisor

echo %Divided% / %Divisor% = %Quotient%
echo %Divided% %% %Divisor% = %Remainder%

set HexValue1=0x14
set HexValue2=0x0A
set /A Result=(HexValue1 + HexValue2) * -3

echo (%HexValue1% + %HexValue2%) * -3 = (20 + 10) * -3 = %Result%

set /A Result%%=7
echo -90 %%= 7 = %Result%

set OctalValue=020
set DecimalValue=12
set /A Result=OctalValue - DecimalValue

echo %OctalValue% - %DecimalValue% = 16 - 12 = %Result%

A saída deste exemplo é:

11 / 3 = 3
11 % 3 = 2
(0x14 + 0x0A) * -3 = (20 + 10) * -3 = -90
-90 %= 7 = -6
020 - 12 = 16 - 12 = 4

As variáveis ​​não definidas na avaliação da expressão aritmética são substituídas pelo valor 0.
