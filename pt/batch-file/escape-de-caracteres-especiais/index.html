Em todas as versões cmd.exe e DOS, alguns caracteres são reservados para uso específico (por exemplo, redirecionamento de comandos). Este tópico falará sobre como usar os caracteres especiais sem problemas.

Escape usando acento circunflexo(^) #

A maioria dos caracteres especiais podem ser escapados usando o acento circunflexo(^). Dê uma olhada no exemplo a seguir.

echo > Hi
echo ^> Hi

Este primeiro comando não produziria > Hi porque > é um caractere especial, o que significa redirecionar a saída para um arquivo. Nesse caso, o arquivo é chamado de “Oi”

No entanto, no segundo comando, > Hi seria gerado sem nenhum problema porque o acento circunflexo (^) diz ao > para parar de funcionar como comando “redirecionar saída para arquivo”, agora > é apenas um caractere normal .

Aqui está uma lista de caracteres especiais que podem ser escapados (retirados e editados da página de Rob van der Woude)

Personagem Resultado escapado Observações
^ ^^
& ^&
< ^<
> ^>
| ^|
\ ^\
! ^^! Necessário apenas quando DelayedExpansion está ativado

Escapando o acento circunflexo #

Os acentos circunflexos podem ser empilhados para escapar de outros acentos circunflexos, considere o exemplo a seguir.

Entrada Saída
^& &
^^^& ^&
^^^^^& ^^&

Nota: Os acentos circunflexos em negrito são escapados.

Problema de segurança #

Um pouco fora do tópico aqui, mas isso é muito importante! Uma fuga de cursor indesejada no final do arquivo pode causar um vazamento de memória!

any-invalid-command-you-like-here ^

Este comando vazaria toda a memória, tornando o sistema completamente inutilizável! Veja aqui para mais informações.

FIND e FINDSTR Caracteres Especiais #

Em find e findstr, existem alguns caracteres especiais que requerem algum cuidado.

ACHAR #

Há apenas um caractere que precisa de escape - " aspas. Para escapar, basta adicionar outra aspas ao lado. Então, " se torna "". Muito simples.

FINDSTR #

Findstr vem com muitos caracteres para escapar, portanto, seja muito cauteloso. Usando \, podemos escapar caracteres especiais. Aqui está uma lista de caracteres especiais para escapar

Personagem Resultado escapado
\ \\
[ \[
] \]
" \"
. \.
* \*
? \?

FOR /F Caracteres especiais #

PARA /F #

Em uma declaração FOR /F, alguns caracteres precisam escapar, aqui uma lista (retirada e editada da página de Rob van der Woude)

Personagem Resultado escapado Observações
' ^' Necessário apenas nos colchetes de FOR /F, a menos que usebackq seja especificado.
` ^` Necessário apenas nos colchetes de FOR /F, quando usebackq é especificado
, ^, ┒
; ^; ┃
= ^= ┣ Deve ser escapado entre colchetes de FOR /F, mesmo se estiver entre aspas duplas
( ^( ┃
) ^) ┙

Caracteres extras especiais #

Aqui está uma lista de outros caracteres especiais, que requerem/podem precisar de escape, mas não mencionados acima.

Personagem Resultado escapado Observações
% %%
[LF] ^[LF] Este truque é mencionado por Mark Stang no grupo de notícias alt.msdos.batch.

Escapando pelo pipeline #

Quando há uma expressão com um pipe, o cmd inicia duas threads em ambos os lados do pipe e a expressão é analisada duas vezes (para cada lado do pipe), então os acentos circunflexos precisam ser duplicados.

No lado esquerdo:

echo ^^^&|more

Do lado direito:

break|echo ^^^&
