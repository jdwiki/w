Confusão entre parênteses #

No site deste, o OP notou um problema.

Causa #

Considere o seguinte trecho de código.

if 1==1 (
    set /a result = 2*(3+4)
)

À primeira vista, você pode pensar que o CMD.exe o processaria assim:

  • A condição é verdadeira, execute o bloco de código
  • Defina o valor do resultado da variável para 14
  • Continuar

No entanto, o processo CMD.exe é assim:

  • A condição é verdadeira, execute o bloco de código
  • Calcule 2*(3+4, o ) após 4 ser processado no final do bloco de código if
  • Um ) aleatório apareceu!

A segunda etapa retornaria o erro “Parênteses não balanceados”.

Solução #

De acordo com o set /? de um CMD.exe alemão, precisaríamos citar operações aritméticas. Aqui está um exemplo.

Anterior Resultado
set /a result=2+5*4 set /a result="2+5*4"

A propósito, de acordo com um CMD.exe em inglês set /?, as aspas são necessárias se operadores lógicos ou de módulo estiverem presentes na expressão (embora essa não seja uma etapa obrigatória).

Caractere de fuga impróprio #

Nesta pergunta do Stack Overflow, o usuário txtechhelp encontrou um problema com o caractere ^ que pode causar um problema de segurança.

Causa #

anyInvaildCommand ^

Nota: Certifique-se de que o acento circunflexo(^) seja o último caractere! Qualquer CR\LF extra não funcionará!

O acento circunflexo procura o próximo caractere a escapar. No entanto, não há mais caracteres disponíveis para escapar, então cmd faz um loop infinito, procurando um caractere para escapar. Neste processo de “loop”, o cmd.exe consumirá a memória do seu computador. E gradualmente comendo toda a memória, deixando o computador de joelhos.

Esse problema pode levar a preocupações de segurança mais sérias, pois é possível inserir o código no computador desbloqueado.

Soluções #

  • Usar a página de código UTF-16 pode resolver esse problema. Apenas UTF-8 ou ASCII causariam o bug.

  • Certifique-se de que haja um ‘CR\LF’ extra no arquivo, ou simplesmente não use acento circunflexo no final do arquivo.

Extra #

Este bug parece ser resolvido no Windows 10.

Extensão do arquivo DEL #

Este bug foi relatado por steve2916 deste tópico do Fórum do Microsoft Windows.

Causa #

Considere uma pasta com esses arquivos.

TestA.doc
TestB.doc
TestC.docx
TestD.docx

Se quisermos remover todos os arquivos .doc neste diretório, normalmente faríamos:

del *.doc

No entanto, este comando também remove os arquivos .docx. O mesmo acontece em extensões de arquivo com esse padrão.

Arquivo A Arquivo B
Qualquer nome.abc OutroNome.abcd

Como podemos ver, desde que a string de extensão do arquivo contenha a string usada no comando del, o arquivo será excluído. Observe que este bug ocorre apenas quando a string de extensão usada no comando del tem pelo menos três caracteres. Por exemplo, del *.do não exclui A.doc ou A.docx.

Solução #

No tópico, o usuário MicroCompsUnltd notou que usar o Powershell resolveria o problema.
