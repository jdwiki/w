Arquivos em lote com mais de 9 argumentos #

Quando mais de 9 argumentos são fornecidos, o comando shift [/n] pode ser usado, onde /n significa iniciar no enésimo argumento, n está entre zero e oito.

Percorrendo os argumentos:

:args
set /a "i+=1"
set arg!i!=%~1
call echo arg!i! = %%arg!i!%%
shift
goto :args

Observe que, no exemplo acima, a variável de expansão atrasada i é usada para atribuir valores de argumento ao array de variáveis. O comando call permite exibir tais valores de variáveis ​​dentro do loop.

Contando argumentos:

for %%i in (%*) do (set /a ArgCount+=1)
echo %ArgCount%

Defina uma variável para o enésimo argumento:

set i=5
call set "path%i%=%%~i"

Argumentos de linha de comando fornecidos a arquivos em lote #

Os argumentos de linha de comando do arquivo em lote são valores de parâmetro enviados ao iniciar o lote. Eles devem ser colocados entre aspas se contiverem espaços. Em um arquivo de lote em execução, os argumentos são usados ​​para vários propósitos, ou seja, redirecionamento para :labels, configuração de variáveis ​​ou execução de comandos.

Os argumentos são referidos no arquivo de lote usando %1, %2, ..., %9.

@echo off
setlocal EnableDelayedExpansion
if not "%1"=="" (
    set "dir=%~1" & set "file=%~2"
    type !dir!\!file! | find /n /i "True" >nul^
     && echo Success! || echo Failure
)
exit /b

C:\Users\UserName> test.bat "C:\Temp\Test Results" "Latest.log"
Success!

Notas:

  • No exemplo acima, as aspas duplas são removidas usando o modificador de argumento %~1.
  • Strings longas são divididas em várias linhas usando ^ , e há um espaço antes do caractere na próxima linha.

Deslocando argumentos entre colchetes #

Vamos ter o seguinte example.bat e chamá-lo com argumentos 1 ,2 e 3:

@echo off

(
    shift
    shift
    echo %1
)

Como a expansão da variável mudará após o contexto dos colchetes ser alcançado, a saída será:

1

Como isso pode ser um problema ao mudar entre colchetes para acessar o argumento, você precisará usar call:

@echo off

(
    shift
    shift
    call echo %%1
)

agora a saída será 3. Como o comando CALL é usado (o que levará a expansão de variáveis ​​adicionais) com esta técnica os argumentos de acesso também podem ser parametrizados:

@echo off

set argument=1

    shift
    shift
    call echo %%%argument%

com expansão atrasada:

@echo off
setlocal enableDelayedExpansion
set argument=1

    shift
    shift
    call echo %%!argument!

a saída será

3
