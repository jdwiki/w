Configurando um repositório Git para Unity #

Ao inicializar um repositório Git para desenvolvimento em Unity, há algumas coisas que precisam ser feitas.

Unity Ignorar Pastas #

Nem tudo deve ser versionado no repositório. Você pode adicionar o modelo abaixo ao seu arquivo .gitignore na raiz do seu repositório. Ou, alternativamente, você pode verificar o código aberto Unity .gitignore no GitHub e, alternativamente, gerar um usando gitignore.io for unity.

# Unity Generated
[Tt]emp/
[Ll]ibrary/
[Oo]bj/

# Unity3D Generated File On Crash Reports
sysinfo.txt

# Visual Studio / MonoDevelop Generated
ExportedObj/
obj/
*.csproj
*.unityproj
*.sln
*.suo
*.tmp
*.user
*.userprefs
*.pidb
*.booproj
*.svd

# OS Generated
desktop.ini
.DS_Store
.DS_Store?
.Spotlight-V100
.Trashes
ehthumbs.db
Thumbs.db

Para saber mais sobre como configurar um arquivo .gitignore, confira aqui.

Configurações do projeto do Unity #

Por padrão, os projetos do Unity não são configurados para dar suporte ao controle de versão corretamente.

  1. (Ignore este passo na v4.5 e acima) Habilite a opção External em Unity → Preferences → Packages → Repository.
  2. Mude para Visible Meta Files em Edit → Project Settings → Editor → Version Control Mode.
  3. Mude para Force Text em Edit → Project Settings → Editor → Asset Serialization Mode.
  4. Salve a cena e o projeto do menu File.

Configuração Adicional #

Um dos poucos grandes aborrecimentos que se tem com o uso do Git com projetos do Unity é que o Git não se importa com diretórios e deixará alegremente diretórios vazios depois de remover arquivos deles. O Unity criará arquivos *.meta para esses diretórios e pode causar um pouco de batalha entre os membros da equipe quando o Git confirma continuar adicionando e removendo esses meta arquivos.

Adicione este hook post-merge do Git à pasta /.git/hooks/ para repositórios com projetos Unity neles. Após qualquer pull/merge do Git, ele verificará quais arquivos foram removidos, verificará se o diretório em que ele existia está vazio e, em caso afirmativo, excluí-lo.

Fusão de cenas e prefabs #

Um problema comum ao trabalhar com o Unity é quando 2 ou mais desenvolvedores estão modificando uma cena ou prefab do Unity (arquivos *.unity). O Git não sabe como mesclá-los corretamente fora da caixa. Felizmente, a equipe do Unity implantou uma ferramenta chamada SmartMerge que torna a mesclagem simples automática. A primeira coisa a fazer é adicionar as seguintes linhas ao seu arquivo .git ou .gitconfig: (Windows: %USERPROFILE%\.gitconfig, Linux/Mac OS X: ~/.gitconfig)

[merge]
tool = unityyamlmerge

[mergetool "unityyamlmerge"]
trustExitCode = false
cmd = '<path to UnityYAMLMerge>' merge -p "$BASE" "$REMOTE" "$LOCAL" "$MERGED"

No Windows, o caminho para UnityYAMLMerge é:

C:\Program Files\Unity\Editor\Data\Tools\UnityYAMLMerge.exe

ou

C:\Program Files (x86)\Unity\Editor\Data\Tools\UnityYAMLMerge.exe

e no MacOSX:

/Applications/Unity/Unity.app/Contents/Tools/UnityYAMLMerge

Feito isso, o mergetool estará disponível quando surgirem conflitos durante o merge/rebase. Não se esqueça de executar o git mergetool manualmente para acionar o UnityYAMLMerge.

Usando o Git Large File Storage (LFS) com Unity #

Prefácio #

O Git pode trabalhar com desenvolvimento de videogame pronto para uso. No entanto, a principal ressalva é que o versionamento de arquivos de mídia grandes (> 5 MB) pode ser um problema a longo prazo, pois seu histórico de commits aumenta - o Git simplesmente não foi originalmente construído para versionar arquivos binários.

A grande notícia é que desde meados de 2015 o GitHub lançou um plugin para Git chamado Git LFS que lida diretamente com esse problema. Agora você pode criar versões de arquivos binários grandes de forma fácil e eficiente!

Por fim, esta documentação é focada nos requisitos específicos e nas informações necessárias para garantir que sua vida no Git funcione bem com o desenvolvimento de videogames. Este guia não abordará como usar o Git em si.

Instalando o Git e o Git-LFS #

Você tem várias opções disponíveis para você como desenvolvedor e a primeira escolha é instalar a linha de comando principal do Git ou deixar que um dos aplicativos populares do Git GUI lide com isso para você.

Opção 1: usar um aplicativo Git GUI #

Esta é realmente uma preferência pessoal aqui, pois existem algumas opções em termos de Git GUI ou se deve usar uma GUI. Você tem vários aplicativos para escolher, aqui estão 3 dos mais populares:

  • Sourcetree (Free)
  • Github Desktop (Grátis)
  • SmartGit (Comercial)

Depois de instalar o aplicativo de sua escolha, pesquise no Google e siga as instruções sobre como garantir que ele esteja configurado para o Git-LFS. Iremos pular esta etapa neste guia, pois ela é específica do aplicativo.

Opção 2: instalar o Git e o Git-LFS #

Isso é bem simples - Install Git. Então. Instale o Git LFS.

Configurando o Git Large File Storage em seu projeto #

Se você estiver usando o plug-in Git LFS para oferecer melhor suporte a arquivos binários, precisará definir alguns tipos de arquivo para serem gerenciados pelo Git LFS. Adicione o seguinte ao seu arquivo .gitattributes na raiz do seu repositório para suportar arquivos binários comuns usados ​​em projetos Unity:

# Image formats:
*.tga filter=lfs diff=lfs merge=lfs -text
*.png filter=lfs diff=lfs merge=lfs -text
*.tif filter=lfs diff=lfs merge=lfs -text
*.jpg filter=lfs diff=lfs merge=lfs -text
*.gif filter=lfs diff=lfs merge=lfs -text
*.psd filter=lfs diff=lfs merge=lfs -text

# Audio formats:
*.mp3 filter=lfs diff=lfs merge=lfs -text
*.wav filter=lfs diff=lfs merge=lfs -text
*.aiff filter=lfs diff=lfs merge=lfs -text

# 3D model formats:
*.fbx filter=lfs diff=lfs merge=lfs -text
*.obj filter=lfs diff=lfs merge=lfs -text

# Unity formats:
*.sbsar filter=lfs diff=lfs merge=lfs -text
*.unity filter=lfs diff=lfs merge=lfs -text

# Other binary formats
*.dll filter=lfs diff=lfs merge=lfs -text
