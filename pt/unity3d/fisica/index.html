Corpos rígidos #

Visão geral #

O componente Rigidbody dá a um GameObject uma presença física na cena, pois é capaz de responder a forças. Você pode aplicar forças diretamente ao GameObject ou permitir que ele reaja a forças externas, como gravidade ou outro corpo rígido que o atinja.

Adicionando um componente Rigidbody #

Você pode adicionar um Rigidbody clicando em Component > Physics > Rigidbody

Movendo um objeto Rigidbody #

Recomenda-se que, se você aplicar um Rigidbody a um GameObject, use forças ou torque para movê-lo em vez de manipular sua transformação. Use os métodos AddForce() ou AddTorque() para isso:

// Add a force to the order of myForce in the forward direction of the Transform.
GetComponent<Rigidbody>().AddForce(transform.forward * myForce);

// Add torque about the Y axis to the order of myTurn.
GetComponent<Rigidbody>().AddTorque(transform.up * torque * myTurn);

Massa #

Você pode alterar a massa de um Rigidbody GameObject para afetar como ele reage com outros Rigidbodies e forças. Uma massa maior significa que o GameObject terá mais influência em outros GameObjects baseados em física e exigirá uma força maior para se mover. Objetos de massa diferente cairão na mesma taxa se tiverem os mesmos valores de arrasto. Para alterar a massa no código:

GetComponent<Rigidbody>().mass = 1000;

Arrastar #

Quanto maior o valor de arrasto, mais um objeto ficará mais lento enquanto se move. Pense nisso como uma força oposta. Para alterar arrastar no código:

GetComponent<Rigidbody>().drag = 10;

isKinematic #

Se você marcar um Rigidbody como Kinematic, ele não poderá ser afetado por outras forças, mas ainda poderá afetar outros GameObjects. Para alterar no código:

GetComponent<Rigidbody>().isKinematic = true;

Restrições #

Também é possível adicionar restrições a cada eixo para congelar a posição ou rotação do corpo rígido no espaço local. O padrão é RigidbodyConstraints.None como mostrado aqui:

Um exemplo de restrições no código:

// Freeze rotation on all axes.
GetComponent<Rigidbody>().constraints = RigidbodyConstraints.FreezeRotation 

// Freeze position on all axes.
GetComponent<Rigidbody>().constraints = RigidbodyConstraints.FreezePosition 

// Freeze rotation and motion an all axes.
GetComponent<Rigidbody>().constraints = RigidbodyConstraints.FreezeAll

Você pode usar o operador OR bit a bit | para combinar várias restrições assim:

// Allow rotation on X and Y axes and motion on Y and Z axes.
GetComponent<Rigidbody>().constraints = RigidbodyConstraints.FreezePositionZ | 
    RigidbodyConstraints.FreezeRotationX;

Colisões #

Se você quiser um GameObject com um Rigidbody para responder a colisões, você também precisará adicionar um colisor a ele. Os tipos de colisores são:

  • Colisor de caixa
  • Colisor de esferas
  • Colisor de cápsulas
  • Colisor de rodas
  • Colisor de malha

Se você aplicar mais de um colisor a um GameObject, nós o chamamos de colisor composto.

Você pode transformar um colisor em um Trigger para usar os métodos OnTriggerEnter(), OnTriggerStay() e OnTriggerExit(). Um colisor de gatilho não reagirá fisicamente a colisões, outros GameObjects simplesmente passarão por ele. Eles são úteis para detectar quando outro GameObject está em uma determinada área ou não, por exemplo, ao coletar um item, podemos querer apenas percorrê-lo, mas detectar quando isso acontece.

Gravidade em Corpo Rígido #

A propriedade useGravity de um RigidBody controla se a gravidade o afeta ou não. Se definido como false, o RigidBody se comportará como se estivesse no espaço sideral (sem que uma força constante seja aplicada a ele em alguma direção).

GetComponent<RigidBody>().useGravity = false;

É muito útil nas situações em que você precisa de todas as outras propriedades do RigidBody, exceto o movimento controlado pela gravidade.

Quando ativado, o RigidBody será afetado por uma força gravitacional, configurada em Physics Settings:

A gravidade é definida em unidades mundiais por segundo ao quadrado e é inserida aqui como um vetor tridimensional: o que significa que com as configurações na imagem de exemplo, todos os RigidBodies com a propriedade useGravity definida como True sofrerão uma força de 9,81 unidades mundiais por segundo por segundo na direção para baixo (como valores Y negativos no sistema de coordenadas do Unity apontam para baixo).
