Inspetor personalizado #

O uso de um inspetor personalizado permite alterar a maneira como um script é desenhado no Inspetor. Às vezes, você deseja adicionar informações extras no inspetor para seu script que não é possível fazer com uma gaveta de propriedades personalizadas.

Abaixo está um exemplo simples de um objeto personalizado que, com o uso de um inspetor personalizado, pode mostrar informações mais úteis.

using UnityEngine;
#if UNITY_EDITOR
using UnityEditor;
#endif

public class InspectorExample : MonoBehaviour {

    public int Level;
    public float BaseDamage;

    public float DamageBonus {
        get {
            return Level / 100f * 50;
        }
    }

    public float ActualDamage {
        get {
            return BaseDamage + DamageBonus;
        }
    }
}

#if UNITY_EDITOR
[CustomEditor( typeof( InspectorExample ) )]
public class CustomInspector : Editor {

    public override void OnInspectorGUI() {
        base.OnInspectorGUI();

        var ie = (InspectorExample)target;

        EditorGUILayout.LabelField( "Damage Bonus", ie.DamageBonus.ToString() );
        EditorGUILayout.LabelField( "Actual Damage", ie.ActualDamage.ToString() );
    }
}
#endif

Primeiro definimos nosso comportamento personalizado com alguns campos

public class InspectorExample : MonoBehaviour {
    public int Level;
    public float BaseDamage;
}

Os campos mostrados acima são desenhados automaticamente (sem inspetor personalizado) quando você está visualizando o script na janela Inspetor.

public float DamageBonus {
    get {
        return Level / 100f * 50;
    }
}

public float ActualDamage {
    get {
        return BaseDamage + DamageBonus;
    }
}

Essas propriedades não são desenhadas automaticamente pelo Unity. Para mostrar essas propriedades na visualização Inspector, temos que usar nosso Custom Inspector.

Primeiro temos que definir nosso inspetor personalizado assim

[CustomEditor( typeof( InspectorExample ) )]
public class CustomInspector : Editor {

O inspetor personalizado deve derivar do Editor e precisa do atributo CustomEditor. O parâmetro do atributo é o tipo de objeto para o qual este inspetor personalizado deve ser usado.

O próximo passo é o método OnInspectorGUI. Este método é chamado sempre que o script é mostrado na janela do inspetor.

public override void OnInspectorGUI() {
    base.OnInspectorGUI();
}

Fazemos uma chamada para base.OnInspectorGUI() para permitir que o Unity manipule os outros campos que estão no script. Se não chamássemos isso, teríamos que fazer mais trabalho nós mesmos.

A seguir estão nossas propriedades personalizadas que queremos mostrar

var ie = (InspectorExample)target;

EditorGUILayout.LabelField( "Damage Bonus", ie.DamageBonus.ToString() );
EditorGUILayout.LabelField( "Actual Damage", ie.ActualDamage.ToString() );

Temos que criar uma variável temporária que mantenha o destino convertido em nosso tipo personalizado (o destino está disponível porque derivamos do Editor).

Em seguida, podemos decidir como desenhar nossas propriedades, neste caso, dois campos de rótulo são suficientes, pois queremos apenas mostrar os valores e não poder editá-los.

Gaveta de propriedade personalizada #

Às vezes, você tem objetos personalizados que contêm dados, mas não derivam do MonoBehaviour. Adicionar esses objetos como um campo em uma classe que é MonoBehaviour não terá efeito visual, a menos que você escreva sua própria gaveta de propriedades personalizadas para o tipo do objeto.

Abaixo está um exemplo simples de um objeto personalizado, adicionado ao MonoBehaviour e uma gaveta de propriedades personalizadas para o objeto personalizado.

public enum Gender {
    Male,
    Female,
    Other
}

// Needs the Serializable attribute otherwise the CustomPropertyDrawer wont be used
[Serializable]
public class UserInfo {
    public string Name;
    public int Age;
    public Gender Gender;
}

// The class that you can attach to a GameObject
public class PropertyDrawerExample : MonoBehaviour {
    public UserInfo UInfo;
}

[CustomPropertyDrawer( typeof( UserInfo ) )]
public class UserInfoDrawer : PropertyDrawer {

    public override float GetPropertyHeight( SerializedProperty property, GUIContent label ) {
        // The 6 comes from extra spacing between the fields (2px each)
        return EditorGUIUtility.singleLineHeight * 4 + 6;
    }

    public override void OnGUI( Rect position, SerializedProperty property, GUIContent label ) {
        EditorGUI.BeginProperty( position, label, property );

        EditorGUI.LabelField( position, label );

        var nameRect = new Rect( position.x, position.y + 18, position.width, 16 );
        var ageRect = new Rect( position.x, position.y + 36, position.width, 16 );
        var genderRect = new Rect( position.x, position.y + 54, position.width, 16 );

        EditorGUI.indentLevel++;

        EditorGUI.PropertyField( nameRect, property.FindPropertyRelative( "Name" ) );
        EditorGUI.PropertyField( ageRect, property.FindPropertyRelative( "Age" ) );
        EditorGUI.PropertyField( genderRect, property.FindPropertyRelative( "Gender" ) );

        EditorGUI.indentLevel--;

        EditorGUI.EndProperty();
    }
}

Primeiro, definimos o objeto personalizado com todos os seus requisitos. Apenas uma classe simples descrevendo um usuário. Essa classe é usada em nossa classe PropertyDrawerExample que podemos adicionar a um GameObject.

public enum Gender {
    Male,
    Female,
    Other
}

[Serializable]
public class UserInfo {
    public string Name;
    public int Age;
    public Gender Gender;
}

public class PropertyDrawerExample : MonoBehaviour {
    public UserInfo UInfo;
}

A classe personalizada precisa do atributo Serializable caso contrário o CustomPropertyDrawer não será usado

Em seguida é o CustomPropertyDrawer

Primeiro temos que definir uma classe que deriva de PropertyDrawer. A definição de classe também precisa do atributo CustomPropertyDrawer. O parâmetro passado é o tipo de objeto para o qual você deseja que esta gaveta seja usada.

[CustomPropertyDrawer( typeof( UserInfo ) )]
public class UserInfoDrawer : PropertyDrawer {

Em seguida, sobrescrevemos a função GetPropertyHeight. Isso nos permite definir uma altura personalizada para nossa propriedade. Neste caso sabemos que nossa propriedade terá quatro partes: etiqueta, nome, idade e gênero. Portanto, usamos EditorGUIUtility.singleLineHeight * 4, adicionamos outros 6 pixels porque queremos espaçar cada campo com dois pixels entre eles.

public override float GetPropertyHeight( SerializedProperty property, GUIContent label ) {
    return EditorGUIUtility.singleLineHeight * 4 + 6;
}

Em seguida é o método OnGUI real. Começamos com EditorGUI.BeginProperty([…]) e terminamos a função com EditorGUI.EndProperty(). Fazemos isso para que, se essa propriedade for parte de um prefab, a lógica de substituição real do prefab funcionará para tudo entre esses dois métodos.

public override void OnGUI( Rect position, SerializedProperty property, GUIContent label ) {
    EditorGUI.BeginProperty( position, label, property );

Depois disso mostramos um rótulo contendo o nome do campo e já definimos os retângulos para nossos campos.

EditorGUI.LabelField( position, label );

var nameRect = new Rect( position.x, position.y + 18, position.width, 16 );
var ageRect = new Rect( position.x, position.y + 36, position.width, 16 );
var genderRect = new Rect( position.x, position.y + 54, position.width, 16 );

Cada campo é espaçado por 16 + 2 pixels e a altura é 16 (que é o mesmo que EditorGUIUtility.singleLineHeight)

Em seguida, recuamos a interface do usuário com uma guia para um layout um pouco melhor, exibimos as propriedades, removemos o recuo da interface e terminamos com EditorGUI.EndProperty.

EditorGUI.indentLevel++;

EditorGUI.PropertyField( nameRect, property.FindPropertyRelative( "Name" ) );
EditorGUI.PropertyField( ageRect, property.FindPropertyRelative( "Age" ) );
EditorGUI.PropertyField( genderRect, property.FindPropertyRelative( "Gender" ) );

EditorGUI.indentLevel--;

EditorGUI.EndProperty();

Exibimos os campos usando EditorGUI.PropertyField que requer um retângulo para a posição e uma SerializedProperty para a propriedade ser exibida. Adquirimos a propriedade chamando FindPropertyRelative("…") na propriedade passada na função OnGUI. Observe que essas propriedades diferenciam maiúsculas de minúsculas e não públicas não podem ser encontradas!

Para este exemplo não estou salvando o retorno das propriedades de property.FindPropertyRelative("…"). Você deve salvá-los em campos privados na classe para evitar chamadas desnecessárias

Itens do menu #

Os itens de menu são uma ótima maneira de adicionar ações personalizadas ao editor. Você pode adicionar itens de menu à barra de menus, tê-los como cliques de contexto em componentes específicos ou até mesmo como cliques de contexto em campos em seus scripts.

Abaixo está um exemplo de como você pode aplicar itens de menu.

public class MenuItemsExample : MonoBehaviour {

    [MenuItem( "Example/DoSomething %#&d" )]
    private static void DoSomething() {
        // Execute some code
    }

    [MenuItem( "Example/DoAnotherThing", true )]
    private static bool DoAnotherThingValidator() {
        return Selection.gameObjects.Length > 0;
    }

    [MenuItem( "Example/DoAnotherThing _PGUP", false )]
    private static void DoAnotherThing() {
        // Execute some code
    }

    [MenuItem( "Example/DoOne %a", false, 1 )]
    private static void DoOne() {
        // Execute some code
    }

    [MenuItem( "Example/DoTwo #b", false, 2 )]
    private static void DoTwo() {
        // Execute some code
    }

    [MenuItem( "Example/DoFurther &c", false, 13 )]
    private static void DoFurther() {
        // Execute some code
    }

    [MenuItem( "CONTEXT/Camera/DoCameraThing" )]
    private static void DoCameraThing( MenuCommand cmd ) {
        // Execute some code
    }

    [ContextMenu( "ContextSomething" )]
    private void ContentSomething() {
        // Execute some code
    }

    [ContextMenuItem( "Reset", "ResetDate" )]
    [ContextMenuItem( "Set to Now", "SetDateToNow" )]
    public string Date = "";

    public void ResetDate() {
        Date = "";
    }

    public void SetDateToNow() {
        Date = DateTime.Now.ToString();
    }
}

Que se parece com isso

Vamos examinar o item de menu básico. Como você pode ver abaixo, você precisa definir uma função estática com um atributo MenuItem, que você passa uma string como título para o item de menu. Você pode colocar seu item de menu em vários níveis de profundidade adicionando um / no nome.

[MenuItem( "Example/DoSomething %#&d" )]
private static void DoSomething() {
    // Execute some code
}

Você não pode ter um item de menu no nível superior. Seus itens de menu precisam estar em um submenu!

Os caracteres especiais no final do nome do MenuItem são para teclas de atalho, não são um requisito.

Existem caracteres especiais que você pode usar para suas teclas de atalho, são eles:

  • % - Ctrl no Windows, Cmd no OS X
  • # - Mudança
  • & - Alt

Isso significa que o atalho %#&d significa ctrl+shift+alt+D no Windows e cmd+shift+alt+D no OS X.

Se você deseja usar um atalho sem nenhuma tecla especial, por exemplo, apenas a tecla ‘D’, você pode preceder o caractere _ (sublinhado) na tecla de atalho que deseja usar.

Existem algumas outras chaves especiais que são suportadas, que são:

  • ESQUERDA, DIREITA, CIMA, BAIXO - para as teclas de seta
  • F1..F12 - para as teclas de função
  • HOME, END, PGUP, PGDN - para as teclas de navegação

As teclas de atalho precisam ser separadas de qualquer outro texto com um espaço

A seguir estão os itens do menu do validador. Os itens de menu do validador permitem que os itens de menu sejam desabilitados (acinzentados, não clicáveis) quando a condição não for atendida. Um exemplo disso pode ser que seu item de menu atue na seleção atual de GameObjects, que você pode verificar no item de menu validador.

[MenuItem( "Example/DoAnotherThing", true )]
private static bool DoAnotherThingValidator() {
    return Selection.gameObjects.Length > 0;
}

[MenuItem( "Example/DoAnotherThing _PGUP", false )]
private static void DoAnotherThing() {
    // Execute some code
}

Para que um item de menu do validador funcione, você precisa criar duas funções estáticas, ambas com o atributo MenuItem e o mesmo nome (a tecla de atalho não importa). A diferença entre eles é que você os está marcando como uma função validadora ou não passando um parâmetro booleano.

Você também pode definir a ordem dos itens de menu adicionando uma prioridade. A prioridade é definida por um inteiro que você passa como o terceiro parâmetro. Quanto menor o número, mais alto na lista, quanto maior o número, mais baixo na lista. Você pode adicionar um separador entre dois itens de menu certificando-se de que haja pelo menos 10 dígitos entre a prioridade dos itens de menu.

[MenuItem( "Example/DoOne %a", false, 1 )]
private static void DoOne() {
    // Execute some code
}

[MenuItem( "Example/DoTwo #b", false, 2 )]
private static void DoTwo() {
    // Execute some code
}

[MenuItem( "Example/DoFurther &c", false, 13 )]
private static void DoFurther() {
    // Execute some code
}

Se você tiver uma lista de menu que tenha uma combinação de itens priorizados e não priorizados, os itens não priorizados serão separados dos itens priorizados.

Em seguida, é adicionar um item de menu ao menu de contexto de um componente já existente. Você precisa iniciar o nome do MenuItem com CONTEXT (sensível a maiúsculas e minúsculas) e fazer com que sua função receba um parâmetro MenuCommand.

O trecho a seguir adicionará um item de menu de contexto ao componente Câmera.

[MenuItem( "CONTEXT/Camera/DoCameraThing" )]
private static void DoCameraThing( MenuCommand cmd ) {
    // Execute some code
}

Que se parece com isso

O parâmetro MenuCommand fornece acesso ao valor do componente e a qualquer dado do usuário enviado com ele.

Você também pode adicionar um item de menu de contexto aos seus próprios componentes usando o atributo ContextMenu. Este atributo recebe apenas um nome, sem validação ou prioridade, e deve fazer parte de um método não estático.

[ContextMenu( "ContextSomething" )]
private void ContentSomething() {
    // Execute some code
}

Que se parece com isso

Você também pode adicionar itens de menu de contexto a campos em seu próprio componente. Esses itens de menu aparecerão quando você clicar em contexto no campo ao qual eles pertencem e podem executar métodos que você definiu nesse componente. Dessa forma, você pode adicionar, por exemplo, valores padrão ou a data atual, conforme mostrado abaixo.

[ContextMenuItem( "Reset", "ResetDate" )]
[ContextMenuItem( "Set to Now", "SetDateToNow" )]
public string Date = "";

public void ResetDate() {
    Date = "";
}

public void SetDateToNow() {
    Date = DateTime.Now.ToString();
}

Janela do Editor #

Por que uma Janela do Editor? #

Como você deve ter visto, você pode fazer muitas coisas em um inspetor personalizado (se você não sabe o que é um inspetor personalizado, veja o exemplo aqui: https://www.wikiod.com/pt/unity3d/estendendo-o-editor -o editor). Mas em um ponto você pode querer implementar um painel de configuração ou uma paleta de ativos personalizada. Nesses casos, você usará uma EditorWindow. A própria interface do usuário do Unity é composta pelo Editor Windows; você pode abri-los (geralmente através da barra superior), tabulá-los, etc.

Crie um EditorWindow básico #

Exemplo Simples #

Criar uma janela de editor personalizada é bastante simples. Tudo que você precisa fazer é estender a classe EditorWindow e usar os métodos Init() e OnGUI(). Aqui está um exemplo simples:

using UnityEngine;
using UnityEditor;

public class CustomWindow : EditorWindow
{
    // Add menu named "Custom Window" to the Window menu
    [MenuItem("Window/Custom Window")]
    static void Init()
    {
        // Get existing open window or if none, make a new one:
        CustomWindow window = (CustomWindow) EditorWindow.GetWindow(typeof(CustomWindow));
        window.Show();
    }

    void OnGUI()
    {
        GUILayout.Label("This is a custom Editor Window", EditorStyles.boldLabel);
    }
}

Os 3 pontos importantes são:

  1. Não se esqueça de estender o EditorWindow
  2. Use o Init() conforme fornecido no exemplo. EditorWindow.GetWindow está verificando se uma CustomWindow já foi criada. Caso contrário, ele criará uma nova instância. Usando isso, você garante que não tenha várias instâncias da sua janela ao mesmo tempo
  3. Use OnGUI() como de costume para exibir informações em sua janela

Aprofundando #

É claro que você provavelmente desejará gerenciar ou modificar alguns recursos usando esta Janela do Editor. Aqui está um exemplo usando a classe Selection (para obter a seleção ativa) e modificando as propriedades do ativo selecionado por meio de SerializedObject e SerializedProperty.

    using System.Linq;
    using UnityEngine;
    using UnityEditor;
    
    public class CustomWindow : EditorWindow
    {
        private AnimationClip _animationClip;
        private SerializedObject _serializedClip;
        private SerializedProperty _events;
    
        private string _text = "Hello World";
    
        // Add menu named "Custom Window" to the Window menu
        [MenuItem("Window/Custom Window")]
        static void Init()
        {
            // Get existing open window or if none, make a new one:
            CustomWindow window = (CustomWindow) EditorWindow.GetWindow(typeof(CustomWindow));
            window.Show();
        }
    
        void OnGUI()
        {
            GUILayout.Label("This is a custom Editor Window", EditorStyles.boldLabel);
    
            // You can use EditorGUI, EditorGUILayout and GUILayout classes to display anything you want
            // A TextField example
            _text = EditorGUILayout.TextField("Text Field", _text);
    
            // Note that you can modify an asset or a gameobject using an EditorWindow. Here is a quick example with an AnimationClip asset
            // The _animationClip, _serializedClip and _events are set in OnSelectionChange()
    
            if (_animationClip == null || _serializedClip == null || _events == null) return;
    
            // We can modify our serializedClip like we would do in a Custom Inspector. For example we can grab its events and display their information
    
            GUILayout.Label(_animationClip.name, EditorStyles.boldLabel);
    
            for (var i = 0; i < _events.arraySize; i++)
            {
                EditorGUILayout.BeginVertical();
    
                EditorGUILayout.LabelField(
                    "Event : " + _events.GetArrayElementAtIndex(i).FindPropertyRelative("functionName").stringValue,
                    EditorStyles.boldLabel);
                EditorGUILayout.PropertyField(_events.GetArrayElementAtIndex(i).FindPropertyRelative("time"), true,
                    GUILayout.ExpandWidth(true));
                EditorGUILayout.PropertyField(_events.GetArrayElementAtIndex(i).FindPropertyRelative("functionName"),
                    true, GUILayout.ExpandWidth(true));
                EditorGUILayout.PropertyField(_events.GetArrayElementAtIndex(i).FindPropertyRelative("floatParameter"),
                    true, GUILayout.ExpandWidth(true));
                EditorGUILayout.PropertyField(_events.GetArrayElementAtIndex(i).FindPropertyRelative("intParameter"),
                    true, GUILayout.ExpandWidth(true));
                EditorGUILayout.PropertyField(
                    _events.GetArrayElementAtIndex(i).FindPropertyRelative("objectReferenceParameter"), true,
                    GUILayout.ExpandWidth(true));
    
                EditorGUILayout.Separator();
                EditorGUILayout.EndVertical();
            }
    
            // Of course we need to Apply the modified properties. We don't our changes won't be saved
            _serializedClip.ApplyModifiedProperties();
        }
    
        /// This Message is triggered when the user selection in the editor changes. That's when we should tell our Window to Repaint() if the user selected another AnimationClip
        private void OnSelectionChange()
        {
            _animationClip =
                Selection.GetFiltered(typeof(AnimationClip), SelectionMode.Assets).FirstOrDefault() as AnimationClip;
            if (_animationClip == null) return;
    
            _serializedClip = new SerializedObject(_animationClip);
            _events = _serializedClip.FindProperty("m_Events");
            Repaint();
        }
    }

Tópicos avançados #

Você pode fazer algumas coisas realmente avançadas no editor, e a classe EditorWindow é perfeita para exibir uma grande quantidade de informações. A maioria dos ativos avançados na Unity Asset Store (como NodeCanvas ou PlayMaker) usa o EditorWindow para exibir visualizações personalizadas.

Desenhando no SceneView #

Uma coisa interessante a se fazer com um EditorWindow é exibir informações diretamente em seu SceneView. Dessa forma, você pode criar um editor de mapa/mundo totalmente personalizado, por exemplo, usando seu EditorWindow personalizado como uma paleta de recursos e ouvindo cliques no SceneView para instanciar novos objetos. Aqui está um exemplo :

using UnityEngine;
using System;
using UnityEditor;

public class CustomWindow : EditorWindow {

    private enum Mode {
        View = 0,
        Paint = 1,
        Erase = 2
    }

    private Mode CurrentMode = Mode.View;

    [MenuItem ("Window/Custom Window")]
    static void Init () {
        // Get existing open window or if none, make a new one:
        CustomWindow window = (CustomWindow)EditorWindow.GetWindow (typeof (CustomWindow));
        window.Show();
    }

    void OnGUI () {
        GUILayout.Label ("This is a custom Editor Window", EditorStyles.boldLabel);
    }

    void OnEnable() {
        SceneView.onSceneGUIDelegate = SceneViewGUI;
        if (SceneView.lastActiveSceneView) SceneView.lastActiveSceneView.Repaint();
    }

    void SceneViewGUI(SceneView sceneView) {
        Handles.BeginGUI();
        // We define the toolbars' rects here
        var ToolBarRect = new Rect((SceneView.lastActiveSceneView.camera.pixelRect.width / 6), 10, (SceneView.lastActiveSceneView.camera.pixelRect.width * 4 / 6) , SceneView.lastActiveSceneView.camera.pixelRect.height / 5);
        GUILayout.BeginArea(ToolBarRect);
        GUILayout.BeginHorizontal();
        GUILayout.FlexibleSpace();
         CurrentMode = (Mode) GUILayout.Toolbar(
            (int) CurrentMode,
            Enum.GetNames(typeof(Mode)),
            GUILayout.Height(ToolBarRect.height));
        GUILayout.FlexibleSpace();
        GUILayout.EndHorizontal();
        GUILayout.EndArea();
        Handles.EndGUI();
    }
}

Isso exibirá a barra de ferramentas diretamente em seu SceneView

Gizmos #

Gizmos são usados ​​para desenhar formas na vista de cena. Você pode usar essas formas para desenhar informações extras sobre seus GameObjects, por exemplo, o tronco que eles têm ou o alcance de detecção.

Abaixo estão dois exemplos de como fazer isso

Exemplo Um #

Este exemplo usa os métodos OnDrawGizmos e OnDrawGizmosSelected (mágico).

public class GizmoExample : MonoBehaviour {

    public float GetDetectionRadius() {
        return 12.5f;
    }

    public float GetFOV() {
        return 25f;
    }

    public float GetMaxRange() {
        return 6.5f;
    }

    public float GetMinRange() {
        return 0;
    }

    public float GetAspect() {
        return 2.5f;
    }

    public void OnDrawGizmos() {
        var gizmoMatrix = Gizmos.matrix;
        var gizmoColor = Gizmos.color;

        Gizmos.matrix = Matrix4x4.TRS( transform.position, transform.rotation, transform.lossyScale );
        Gizmos.color = Color.red;
        Gizmos.DrawFrustum( Vector3.zero, GetFOV(), GetMaxRange(), GetMinRange(), GetAspect() );

        Gizmos.matrix = gizmoMatrix;
        Gizmos.color = gizmoColor;
    }

    public void OnDrawGizmosSelected() {
        Handles.DrawWireDisc( transform.position, Vector3.up, GetDetectionRadius() );
    }
}

Neste exemplo temos dois métodos para desenhar gizmos, um que desenha quando o objeto está ativo (OnDrawGizmos) e outro para quando o objeto é selecionado na hierarquia (OnDrawGizmosSelected).

public void OnDrawGizmos() {
    var gizmoMatrix = Gizmos.matrix;
    var gizmoColor = Gizmos.color;

    Gizmos.matrix = Matrix4x4.TRS( transform.position, transform.rotation, transform.lossyScale );
    Gizmos.color = Color.red;
    Gizmos.DrawFrustum( Vector3.zero, GetFOV(), GetMaxRange(), GetMinRange(), GetAspect() );

    Gizmos.matrix = gizmoMatrix;
    Gizmos.color = gizmoColor;
}

Primeiro, salvamos a matriz e a cor do gizmo porque vamos alterá-la e queremos revertê-la quando terminarmos para não afetar nenhum outro desenho do gizmo.

Em seguida, queremos desenhar o frustum que nosso objeto possui, no entanto, precisamos alterar a matriz dos Gizmos para que ela corresponda à posição, rotação e escala. Também definimos a cor dos Gizmos para vermelho para enfatizar o tronco. Quando isso for feito, podemos chamar Gizmos.DrawFrustum para desenhar o tronco na vista da cena.

Quando terminamos de desenhar o que queremos desenhar, redefinimos a matriz e a cor dos Gizmos.

public void OnDrawGizmosSelected() {
    Handles.DrawWireDisc( transform.position, Vector3.up, GetDetectionRadius() );
}

Também queremos desenhar um intervalo de detecção quando selecionamos nosso GameObject. Isso é feito através da classe Handles já que a classe Gizmos não possui nenhum método para discos.

Usar esta forma de desenhar gizmos resulta na saída mostrada abaixo.

Exemplo dois #

Este exemplo usa o atributo DrawGizmo.

public class GizmoDrawerExample {

    [DrawGizmo( GizmoType.Selected | GizmoType.NonSelected, typeof( GizmoExample ) )]
    public static void DrawGizmo( GizmoExample obj, GizmoType type ) {
        var gizmoMatrix = Gizmos.matrix;
        var gizmoColor = Gizmos.color;

        Gizmos.matrix = Matrix4x4.TRS( obj.transform.position, obj.transform.rotation, obj.transform.lossyScale );
        Gizmos.color = Color.red;
        Gizmos.DrawFrustum( Vector3.zero, obj.GetFOV(), obj.GetMaxRange(), obj.GetMinRange(), obj.GetAspect() );

        Gizmos.matrix = gizmoMatrix;
        Gizmos.color = gizmoColor;

        if ( ( type & GizmoType.Selected ) == GizmoType.Selected ) {
            Handles.DrawWireDisc( obj.transform.position, Vector3.up, obj.GetDetectionRadius() );
        }
    }
}

Dessa forma, você pode separar as chamadas do gizmo do seu script. A maior parte disso usa o mesmo código que o outro exemplo, exceto por duas coisas.

[DrawGizmo( GizmoType.Selected | GizmoType.NonSelected, typeof( GizmoExample ) )]
public static void DrawGizmo( GizmoExample obj, GizmoType type ) {

Você precisa usar o atributo DrawGizmo que usa o enum GizmoType como o primeiro parâmetro e um Type como o segundo parâmetro. O Type deve ser o tipo que você deseja usar para desenhar o gizmo.

O método para desenhar o gizmo precisa ser estático, público ou não público, e pode ser nomeado como você quiser. O primeiro parâmetro é o tipo, que deve corresponder ao tipo passado como segundo parâmetro no atributo, e o segundo parâmetro é o enum GizmoType que descreve o estado atual do seu objeto.

if ( ( type & GizmoType.Selected ) == GizmoType.Selected ) {
    Handles.DrawWireDisc( obj.transform.position, Vector3.up, obj.GetDetectionRadius() );
}

A outra diferença é que para verificar qual é o GizmoType do objeto, você precisa fazer uma verificação AND no parâmetro e no tipo que deseja.
