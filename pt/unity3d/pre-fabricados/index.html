Sintaxe #

  • Public static Object PrefabUtility.InstantiatePrefab(Object target);
  • objeto estático público AssetDatabase.LoadAssetAtPath(string assetPath, Type type);
  • objeto estático público Object.Instantiate(Objeto original);
  • Public static Object Resources.Load(string path);

Introdução #

Prefabs são um tipo de ativo que permite o armazenamento de um GameObject completo com seus componentes, propriedades, componentes anexados e valores de propriedade serializados. Há muitos cenários em que isso é útil, incluindo:

  • Duplicando objetos em uma cena
  • Compartilhando um objeto comum em várias cenas
  • Ser capaz de modificar um prefab uma vez e aplicar as alterações em vários objetos/cenas
  • Criando objetos duplicados com pequenas modificações, enquanto os elementos comuns podem ser editados a partir de uma base pré-fabricada
  • Instanciando GameObjects em tempo de execução

Existe uma regra prática no Unity que diz que “tudo deve ser Prefabs”. Embora isso seja provavelmente um exagero, incentiva a reutilização de código e a construção de GameObjects de maneira reutilizável, o que é eficiente em termos de memória e bom design.

Instanciando prefabs #

Há 2 maneiras de instanciar prefabs: durante tempo de design ou tempo de execução.

Instanciação em tempo de design #

Instanciar prefabs em tempo de design é útil para colocar visualmente várias instâncias do mesmo objeto (por exemplo, colocar árvores ao projetar um nível do seu jogo).

  • Para instanciar visualmente um prefab, arraste-o da visualização do projeto para a hierarquia da cena.

  • Se você estiver escrevendo uma extensão do editor, você também pode instanciar um prefab que chama programaticamente o método PrefabUtility.InstantiatePrefab():

    GameObject gameObject = (GameObject)PrefabUtility.InstantiatePrefab(AssetDatabase.LoadAssetAtPath("Assets/MainCamera.prefab", typeof(GameObject)));

Instanciação em tempo de execução #

Instanciar prefabs em tempo de execução é útil para criar instâncias de um objeto de acordo com alguma lógica (por exemplo, gerando um inimigo a cada 5 segundos).

Para instanciar um prefab, você precisa de uma referência ao objeto prefab. Isso pode ser feito com um campo public GameObject em seu script MonoBehaviour (e definindo seu valor usando o inspetor no Unity Editor):

public class SomeScript : MonoBehaviour {
   public GameObject prefab; 
}

Ou colocando o prefab na pasta Resource e usando Resources.Load:

GameObject prefab = Resources.Load("Assets/Resources/MainCamera");

Depois de ter uma referência ao objeto prefab, você pode instanciá-lo usando a função Instantiate em qualquer lugar do seu código (por exemplo, dentro de um loop para criar vários objetos):

GameObject gameObject = Instantiate<GameObject>(prefab, new Vector3(0,0,0), Quaternion.identity);

Nota: o termo Prefab não existe em tempo de execução.

Inspetor de pré-fabricados #

Se você selecionar um prefab na visualização de hierarquia, notará que seu inspetor difere ligeiramente de um objeto de jogo comum:

vs

Propriedades em negrito significam que seus valores diferem dos valores pré-fabricados. Você pode alterar qualquer propriedade de um prefab instanciado sem afetar os valores originais do prefab. Quando um valor é alterado em uma instância prefab, ele fica em negrito e quaisquer alterações subsequentes do mesmo valor no prefab não serão refletidas na instância alterada.

Você pode reverter para os valores pré-fabricados originais clicando no botão Reverter, que também terá as alterações de valor refletidas na instância. Além disso, para reverter um valor individual, você pode clicar nele com o botão direito e pressionar Reverter valor para pré-fabricado. Para reverter um componente, clique com o botão direito nele e pressione Reverter para Prefab.

Clicar no botão Aplicar substitui os valores da propriedade prefab pelos valores atuais da propriedade do objeto do jogo. Não há botão “Desfazer” ou caixa de diálogo de confirmação, portanto, manuseie este botão com cuidado.

O botão Selecionar destaca o prefab conectado na estrutura de pastas do projeto.

Pré-fabricados aninhados #

Prefabs aninhados não estão disponíveis no Unity no momento. Você pode arrastar um prefab para outro e aplicá-lo, mas quaisquer alterações no prefab filho não serão aplicadas ao aninhado.

Mas há uma solução simples - Você precisa adicionar ao pré-fabricado pai um script simples, que instanciará um filho.

using UnityEngine;

public class ParentPrefab : MonoBehaviour {

    [SerializeField] GameObject childPrefab;
    [SerializeField] Transform childPrefabPositionReference;

    // Use this for initialization
    void Start () {
        print("Hello, I'm a parent prefab!");
        Instantiate(
            childPrefab, 
            childPrefabPositionReference.position, 
            childPrefabPositionReference.rotation, 
            gameObject.transform
        );
    }
}
