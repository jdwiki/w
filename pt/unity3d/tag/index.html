Uma tag é uma string que pode ser aplicada para marcar tipos GameObject. Dessa forma, fica mais fácil identificar objetos GameObject específicos por meio de código.

Uma tag pode ser aplicada a um ou mais objetos do jogo, mas um objeto do jogo sempre terá apenas uma tag. Por padrão, a tag “Untagged” é usada para representar um GameObject que não foi marcado intencionalmente.

Criando e aplicando tags #

As tags são normalmente aplicadas por meio do editor; no entanto, você também pode aplicar tags via script. Qualquer tag personalizada deve ser criada através da janela Tags & Layers antes de ser aplicada a um objeto do jogo.

Configurando Tags no Editor #

Com um ou mais objetos de jogo selecionados, você pode selecionar uma tag no inspetor. Os objetos do jogo sempre carregam uma única tag; por padrão, os objetos do jogo serão marcados como “Untagged”. Você também pode ir para a janela Tags & Layers, selecionando “Add Tag…”; no entanto, é importante notar que isso só leva você para a janela Tags & Layers. Qualquer tag que você criar não se aplicará automaticamente ao objeto do jogo.

Configurando Tags via Script #

Você pode alterar diretamente uma tag de objetos do jogo via código. É importante observar que você deve fornecer uma tag da lista de tags atuais; se você fornecer uma tag que ainda não foi criada, isso resultará em um erro.

Conforme detalhado em outros exemplos, usar uma série de variáveis ​​string estática em vez de escrever manualmente cada tag pode garantir consistência e confiabilidade.

O script a seguir demonstra como podemos alterar uma série de tags de objetos do jogo, usando referências de string estática para garantir consistência. Observe a suposição de que cada string estática representa uma tag que já foi criada na janela Tags & Layers.

using UnityEngine;

public class Tagging : MonoBehaviour
{
    static string tagUntagged = "Untagged";
    static string tagPlayer = "Player";
    static string tagEnemy = "Enemy";

    /// <summary>Represents the player character. This game object should 
    /// be linked up via the inspector.</summary>
    public GameObject player;
    /// <summary>Represents all the enemy characters. All enemies should 
    /// be added to the array via the inspector.</summary>
    public GameObject[] enemy;

    void Start ()
    {
        // We ensure that the game object this script is attached to
        // is left untagged by using the default "Untagged" tag.
        gameObject.tag = tagUntagged;

        // We ensure the player has the player tag.
        player.tag = tagUntagged;

        // We loop through the enemy array to ensure they are all tagged.
        for(int i = 0; i < enemy.Length; i++)
        {
            enemy[i].tag = tagEnemy;
        }
    }
}

Criando tags personalizadas #

Independentemente de você definir tags por meio do Inspector ou por script, as tags devem ser declaradas na janela Tags & Layers antes do uso. Você pode acessar esta janela selecionando “Adicionar Tags…” em um menu suspenso de tags de objetos do jogo. Como alternativa, você pode encontrar a janela em Editar > Configurações do projeto > Tags e camadas.

Basta selecionar o botão +, digitar o nome desejado e selecionar Salvar para criar uma tag. Selecionar o botão - removerá a tag atualmente destacada. Observe que, dessa maneira, a tag será exibida imediatamente como "(Removido)" e será completamente removida quando o projeto for recarregado novamente.

Selecionar a engrenagem/engrenagem no canto superior direito da janela permitirá que você redefina todas as opções personalizadas. Isso removerá imediatamente todas as tags personalizadas, juntamente com qualquer camada personalizada que você possa ter em “Classificar Camadas” e “Camadas”.

Encontrando GameObjects por Tag: #

As tags facilitam particularmente a localização de objetos específicos do jogo. Podemos procurar um único objeto de jogo ou procurar vários.

Encontrando um único GameObject #

Podemos usar a função estática GameObject.FindGameObjectWithTag(string tag) para procurar objetos de jogo individuais. É importante notar que, dessa forma, os objetos do jogo não são consultados em nenhuma ordem específica. Se você procurar uma tag que é usada em vários objetos de jogo na cena, esta função não poderá garantir qual objeto de jogo é retornado. Como tal, é mais apropriado quando sabemos que apenas um objeto de jogo usa tal tag, ou quando não estamos preocupados com a instância exata de GameObject que é retornada.

///<summary>We create a static string to allow us consistency.</summary>
string playerTag = "Player"

///<summary>We can now use the tag to reference our player GameObject.</summary>
GameObject player = GameObject.FindGameObjectWithTag(playerTag);

Encontrando um Array de instâncias de GameObject #

Podemos usar a função estática GameObject.FindGameObjectsWithTag(string tag) para procurar todos os objetos do jogo que usam uma tag específica. Isso é útil quando queremos iterar por meio de um grupo de objetos de jogo específicos. Isso também pode ser útil se quisermos encontrar um único objeto de jogo, mas podemos ter vários objetos de jogo usando a mesma tag. Como não podemos garantir a instância exata retornada por GameObject.FindGameObjectWithTag(string tag), devemos, em vez disso, recuperar uma matriz de todas as instâncias potenciais de GameObject com GameObject.FindGameObjectsWithTag(string tag) e analisar ainda mais a matriz resultante para encontrar a instância que estamos procurando.

///<summary>We create a static string to allow us consistency.</summary>
string enemyTag = "Enemy";

///<summary>We can now use the tag to create an array of all enemy GameObjects.</summary>
GameObject[] enemies = GameObject.FindGameObjectsWithTag(enemyTag );

// We can now freely iterate through our array of enemies
foreach(GameObject enemy in enemies)
{
    // Do something to each enemy (link up a reference, check for damage, etc.)
}

Comparação de tags #

Ao comparar dois GameObjects por Tags, deve-se notar que o seguinte causaria a sobrecarga do Garbage Collector, pois uma string é criada toda vez:

if (go.Tag == "myTag")
{ 
    //Stuff 
}

Ao realizar essas comparações dentro de Update() e outro callback regular do Unity (ou um loop), você deve usar este método livre de alocação de heap:

if (go.CompareTag("myTag")
{ 
    //Stuff 
}

Além disso, é mais fácil manter suas tags em uma classe estática.

public static class Tags
{
    public const string Player = "Player";
    public const string MyCustomTag = "MyCustomTag";
}

Então você pode comparar com segurança

if (go.CompareTag(Tags.MyCustomTag)
{ 
    //Stuff 
}

dessa forma, suas strings de tag são geradas em tempo de compilação e você limita as implicações de erros de ortografia.

Assim como manter as tags em uma classe estática, também é possível armazená-las em uma enumeração:

public enum Tags
{
    Player, Ennemies, MyCustomTag;
}

e então você pode compará-lo usando o método enum toString():

if (go.CompareTag(Tags.MyCustomTag.toString())
{ 
    //Stuff 
}
