SerializeField #

O sistema de serialização do Unity pode ser usado para fazer o seguinte:

  • Pode serializar campos públicos não estáticos (de tipos serializáveis)
  • Pode serializar campos não estáticos não públicos marcados com o atributo [SerializeField]
  • Não é possível serializar campos estáticos
  • Não é possível serializar propriedades estáticas

Seu campo, mesmo que marcado com o atributo SerializeField, só será atribuído se for de um tipo que o Unity possa serializar, que são:

  • Todas as classes herdadas de UnityEngine.Object (por exemplo, GameObject, Component, MonoBehaviour, Texture2D)
  • Todos os tipos de dados básicos como int, string, float, bool
  • Alguns tipos embutidos como Vector2/3/4, Quaternion, Matrix4x4, Color, Rect, LayerMask
  • Arrays de um tipo serializável
  • Lista de um tipo serializável
  • Enums
  • Estruturas

Atributos comuns do inspetor #

[Header( "My variables" )]
public string MyString;

[HideInInspector]
public string MyHiddenString;

[Multiline( 5 )]
public string MyMultilineString;

[TextArea( 2, 8 )]
public string MyTextArea;

[Space( 15 )]
public int MyInt;

[Range( 2.5f, 12.5f )]
public float MyFloat;

[Tooltip( "This is a tip for MyDouble" )]
public double MyDouble;

[SerializeField]
private double myHiddenDouble;

Ao passar o mouse sobre o rótulo de um campo:

[Header( "My variables" )]
public string MyString;

Cabeçalho coloca um rótulo em negrito contendo o texto acima do campo atribuído. Isso geralmente é usado para rotular grupos para destacá-los em relação a outros rótulos.

[HideInInspector]
public string MyHiddenString;

HideInInspector impede que campos públicos sejam mostrados no inspetor. Isso é útil para acessar campos de outras partes do código onde eles não são visíveis ou mutáveis.

[Multiline( 5 )]
public string MyMultilineString;

Multilinha cria uma caixa de texto com um número especificado de linhas. Exceder esse valor não expandirá a caixa nem envolverá o texto.

[TextArea( 2, 8 )]
public string MyTextArea;

TextArea permite texto em estilo de várias linhas com quebra automática de palavras e barras de rolagem se o texto exceder a área alocada.

[Space( 15 )]
public int MyInt;

Espaço força o inspetor a adicionar espaço extra entre os itens anteriores e atuais - útil para distinguir e separar grupos.

[Range( 2.5f, 12.5f )]
public float MyFloat;

Intervalo força um valor numérico entre um mínimo e um máximo. Este atributo também funciona em inteiros e duplos, mesmo que min e max sejam especificados como floats.

[Tooltip( "This is a tip for MyDouble" )]
public double MyDouble;

Dica de ferramenta mostra uma descrição adicional sempre que o rótulo do campo é posicionado sobre o cursor.

[SerializeField]
private double myHiddenDouble;

SerializeField força o Unity a serializar o campo - útil para campos privados.

[CanEditMultipleObjects]

O atributo CanEditMultipleObjects permite que você edite valores de seu componente em vários GameObjects. Sem este componente, você não verá seu componente aparecer normalmente ao selecionar vários GameObjects, mas verá a mensagem “Edição de vários objetos não suportada”

Este atributo é para editores personalizados para suportar multiedição. Editores não personalizados suportam automaticamente a edição múltipla.

[PreferenceItem( "My Prefs" )]

O atributo PreferenceItem permite criar um item extra no menu de preferências do Unity. O método de recebimento precisa ser estático para ser usado.

O atributo DrawGizmo permite desenhar gizmos personalizados para seus componentes. Esses gizmos serão desenhados na Scene View. Você pode decidir quando desenhar o gizmo usando o parâmetro GizmoType no atributo DrawGizmo.

O método de recebimento requer dois parâmetros, o primeiro é o componente para o qual desenhar o gizmo e o segundo é o estado em que o objeto que precisa do gizmo desenhado está.

Atributos do componente #

[DisallowMultipleComponent]
[RequireComponent( typeof( Rigidbody ) )]
public class AttributesExample : MonoBehaviour 
{
    [...]
}

[DisallowMultipleComponent]

O atributo DisallowMultipleComponent impede que os usuários adicionem várias instâncias desse componente a um GameObject.

[RequireComponent( typeof( Rigidbody ) )]

O atributo RequireComponent permite especificar outro componente (ou mais) como requisitos para quando esse componente for adicionado a um GameObject. Quando você adiciona este componente a um GameObject, os componentes necessários serão adicionados automaticamente (se ainda não estiverem presentes) e esses componentes não podem ser removidos até que aquele que os requer seja removido.

public class AttributesExample : MonoBehaviour

        [...]
    }

    [RuntimeInitializeOnLoadMethod( RuntimeInitializeLoadType.BeforeSceneLoad )]
    private static void Foo() 
    {
        [...]
    }

    [RuntimeInitializeOnLoadMethod( RuntimeInitializeLoadType.AfterSceneLoad )]
    private static void Bar() 
    {
        [...]
    }

    void Update() 
    {
        if ( Application.isEditor )
        {
            [...]
        } 
        else
        {
            [...]
        }
    }
}

[ExecuteInEditMode]
public class AttributesExample : MonoBehaviour

O atributo ExecuteInEditMode força o Unity a executar os métodos mágicos deste script mesmo quando o jogo não está sendo executado.

As funções não são chamadas constantemente como no modo de reprodução

  • Update só é chamado quando algo na cena mudou.
  • OnGUI é chamado quando o Game View recebe um Evento.
  • OnRenderObject e as outras funções de callback de renderização são chamadas em cada repintura da Scene View ou Game View.
[RuntimeInitializeOnLoadMethod]
private static void FooBar()

[RuntimeInitializeOnLoadMethod( RuntimeInitializeLoadType.BeforeSceneLoad )]
private static void Foo() 

[RuntimeInitializeOnLoadMethod( RuntimeInitializeLoadType.AfterSceneLoad )]
private static void Bar()

O atributo RuntimeInitializeOnLoadMethod permite que um método de classe de tempo de execução seja chamado quando o jogo carrega o tempo de execução, sem qualquer interação do usuário.

Você pode especificar se deseja que o método seja invocado antes ou depois do carregamento da cena (depois é o padrão). A ordem de execução não é garantida para métodos que usam esse atributo.
