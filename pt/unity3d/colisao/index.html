Colisores #

Colisor de Caixa #

Um Colisor primitivo em forma de cuboide.

Propriedades #

  • É Trigger - Se marcado, o Box Collider ignorará a física e se tornará um Trigger Collider

  • Material - Uma referência, se especificada, ao material físico do Box Collider

  • Centro - A posição central do Box Collider no espaço local

  • Tamanho - O tamanho do Box Collider medido no espaço local

Exemplo #

// Add a Box Collider to the current GameObject.
BoxCollider myBC = BoxCollider)myGameObject.gameObject.AddComponent(typeof(BoxCollider));
 
// Make the Box Collider into a Trigger Collider.
myBC.isTrigger= true;

// Set the center of the Box Collider to the center of the GameObject.
myBC.center = Vector3.zero;

// Make the Box Collider twice as large.
myBC.size = 2;

Colisor de esferas #

Um Collider primitivo em forma de esfera.

Propriedades #

  • Is Trigger - Se marcado, o Sphere Collider ignorará a física e se tornará um Trigger Collider

  • Material - Uma referência, se especificada, ao material físico do Sphere Collider

  • Centro - A posição central do Sphere Collider no espaço local

  • Raio - O raio do Colisor

Exemplo #

// Add a Sphere Collider to the current GameObject.
SphereCollider mySC = SphereCollider)myGameObject.gameObject.AddComponent(typeof(SphereCollider));
 
// Make the Sphere Collider into a Trigger Collider.
mySC.isTrigger= true;

// Set the center of the Sphere Collider to the center of the GameObject.
mySC.center = Vector3.zero;

// Make the Sphere Collider twice as large.
mySC.radius = 2;

Colisor de Cápsulas #

Duas meias esferas unidas por um cilindro.

Propriedades #

  • Is Trigger - Se marcado, o Capsule Collider ignorará a física e se tornará um Trigger Collider

  • Material - Uma referência, se especificada, ao material físico do Capsule Collider

  • Centro - A posição central do Capsule Collider no espaço local

  • Raio - O raio no espaço local

  • Altura - Altura total do Colisor

  • Direção - O eixo de orientação no espaço local

Exemplo #

// Add a Capsule Collider to the current GameObject.
CapsuleCollider myCC = CapsuleCollider)myGameObject.gameObject.AddComponent(typeof(CapsuleCollider));
 
// Make the Capsule Collider into a Trigger Collider.
myCC.isTrigger= true;

// Set the center of the Capsule Collider to the center of the GameObject.
myCC.center = Vector3.zero;

// Make the Sphere Collider twice as tall.
myCC.height= 2;

// Make the Sphere Collider twice as wide.
myCC.radius= 2;

// Set the axis of lengthwise orientation to the X axis.
myCC.direction = 0;

// Set the axis of lengthwise orientation to the Y axis.
myCC.direction = 1;

// Set the axis of lengthwise orientation to the Y axis.
myCC.direction = 2;

Colisor de rodas #

Propriedades #

  • Mass - A massa do Wheel Collider

  • Raio - O raio no espaço local

  • Taxa de amortecimento da roda - Valor de amortecimento para o Wheel Collider

  • Distância de suspensão - Extensão máxima ao longo do eixo Y no espaço local

  • Force app point distance - O ponto onde as forças serão aplicadas,

  • Centro - Centro do Wheel Collider no espaço local

Mola de Suspensão #

  • Mola - a taxa na qual a Roda tenta retornar à Posição Alvo

  • Amortecedor - Um valor maior amortece mais a velocidade e a suspensão se move mais devagar

  • Posição alvo - o padrão é 0,5, em 0 a suspensão está no fundo, em 1 está em extensão total

  • Frito frontal/lateral - como o pneu se comporta ao rolar para frente ou para os lados

Colisor de malha #

Um Collider baseado em um Mesh Asset.

Propriedades #

  • Is Trigger - Se marcado, o Box Collider ignorará a física e se tornará um Trigger Collider

  • Material - Uma referência, se especificada, ao material físico do Box Collider

  • Mesh - Uma referência à malha na qual o Collider é baseado

  • Convexo - Os colisores de malha convexa são limitados a 255 polígonos - se ativado, este colisor pode colidir com outros colisores de malha

Colisor de rodas #

O colisor de rodas dentro da unidade é construído sobre o colisor de rodas PhysX da Nvidia e, portanto, compartilha muitas propriedades semelhantes. Tecnicamente, a unidade é um programa “sem unidades”, mas para que tudo faça sentido, algumas unidades padrão são necessárias.

Propriedades Básicas

  • Massa - o peso da roda em quilogramas, isso é usado para o momento da roda e o momento de interria ao girar.
  • Raio - em metros, o raio do colisor.
  • Wheel Damping Rate - Ajusta a “resposta” das rodas ao torque aplicado.
  • Distância da Suspensão - Distância total do percurso em metros que a roda pode percorrer
  • Force App Point Distance - onde é a força da suspensão aplicada ao corpo rígido pai
  • Centro - A posição central da roda

Configurações de suspensão

  • Mola - Esta é a constante da mola, K , em Newtons/metro na equação:

Força = Constante da Mola * Distância

Um bom ponto de partida para este valor deve ser a massa total do seu veículo, dividida pelo número de rodas, multiplicado por um número entre 50 e 100. Ex. se você tiver um carro de 2.000 kg com 4 rodas, cada roda precisaria suportar 500 kg. Multiplique isso por 75 e sua constante de mola deve ser 37.500 Newtons/metro.

  • Amortecedor - o equivalente a um amortecedor em um carro. Taxas mais altas tornam o suspense “mais rígido” e taxas mais baixas o tornam “mais suave” e mais propenso a oscilar. Eu não sei as unidades ou equação para isso, mas acho que tem a ver com uma equação de frequência na física.

Configurações de atrito lateral

A curva de atrito em unidade tem um valor de escorregamento determinado pelo quanto a roda está escorregando (em m/s) da posição desejada versus a posição real.

  • Deslizamento Extremo - Esta é a quantidade máxima (em m/s) que uma roda pode escorregar antes de perder tração

  • Valor Extremo - Esta é a quantidade máxima de atrito que deve ser aplicada a uma roda.

Os valores para Deslizamento Extremo devem estar entre 0,2 e 2 m/s para carros mais realistas. 2m/s é cerca de 6 pés por segundo ou 5mph, o que é muito escorregadio. Se você achar que seu veículo precisa ter um valor superior a 2 m/s para escorregamento, considere aumentar o atrito máximo (Valor Extremo).

Max Fraction (Valor Extremo) é o coeficiente de atrito na equação:

Força de atrito (em newtons) = Coeficiente de atrito * Força descendente (em newtons)

Isso significa que com um coeficiente de 1, você está aplicando toda a força do carro + suspensão oposta à direção de deslizamento. Em aplicações do mundo real, valores superiores a 1 são raros, mas não impossíveis. Para um pneu em asfalto seco, os valores entre 0,7 e 0,9 são realistas, portanto, o padrão de 1,0 é preferível.

Este valor não deve exceder 2,5, pois um comportamento estranho começará a ocorrer. Por exemplo. você começa a virar à direita, mas como esse valor é muito alto, uma grande força é aplicada na direção oposta à sua direção e você começa a deslizar para a curva em vez de se afastar.

Se você tiver maximizado ambos os valores, deverá começar a aumentar o deslizamento e o valor da assíntota. Asymptote Slip deve estar entre 0,5 e 2 m/s e define o coeficiente de atrito para qualquer valor de deslizamento além do Assymptote slip. Se você achar que seus veículos se comportam bem até quebrar a tração, ponto em que ele age como se estivesse no gelo, você deve aumentar o valor Asymptote. Se você achar que seu veículo não consegue derrapar, você deve diminuir o valor.

Atrito frontal

O atrito para frente é idêntico ao atrito lateral, com a exceção de que isso define quanta tração a roda tem na direção do movimento. Se os valores estiverem muito baixos, seus veículos irão queimar e apenas girar os pneus antes de avançar, lentamente. Se estiver muito alto, seu veículo pode ter uma tendência a tentar fazer um wheely, ou pior, flip.

Notas Adicionais

Não espere poder criar um clone do GTA ou outro clone de corrida simplesmente ajustando esses valores. Na maioria dos jogos de direção, esses valores são constantemente alterados no script para diferentes velocidades, terrenos e valores de curva. Além disso, se você estiver apenas aplicando um torque constante aos colisores das rodas quando uma tecla estiver sendo pressionada, seu jogo não se comportará de forma realista. No mundo real, os carros têm curvas de torque e transmissões para alterar o torque aplicado às rodas.

Para obter melhores resultados, você deve ajustar esses valores até obter um carro que responda razoavelmente bem e, em seguida, fazer alterações no torque da roda, ângulo máximo de giro e valores de atrito no script.

Mais informações sobre colisores de rodas podem ser encontradas na documentação da Nvidia: http://docs.nvidia.com/gameworks/content/gameworkslibrary/physx/guide/Manual/Vehicles.html

Acionar Colliders #

Métodos #

  • OnTriggerEnter()
  • OnTriggerStay()
  • ``OnTriggerExit()''

Você pode transformar um Collider em um Trigger para usar os métodos OnTriggerEnter(), OnTriggerStay() e OnTriggerExit(). Um Trigger Collider não reagirá fisicamente a colisões, outros GameObjects simplesmente passarão por ele. Eles são úteis para detectar quando outro GameObject está em uma determinada área ou não, por exemplo, ao coletar um item, podemos querer apenas percorrê-lo, mas detectar quando isso acontece.
