Sintaxe #

  • private void EventName (remetente do objeto, EventArgs e);

Parâmetros #

Parâmetro Detalhes
remetente do objeto remetente refere-se ao objeto que invocou o evento que disparou o manipulador de eventos. Isso é útil se você tiver muitos objetos usando o mesmo manipulador de eventos.
EventArgs e EventArgs é uma espécie de classe base fictícia. Por si só, é mais ou menos inútil, mas se você derivar dele, poderá adicionar quaisquer dados que precise passar para seus manipuladores de eventos.

Eventos padrão #

O evento padrão para o objeto Page é o evento Load. Da mesma forma, todo controle tem um evento padrão. Por exemplo, o evento padrão para o controle de botão é o evento Click.

O manipulador de eventos padrão pode ser criado no Visual Studio, apenas clicando duas vezes no controle no modo de design. A tabela a seguir mostra alguns dos eventos padrão para controles comuns:

Exemplo Este exemplo inclui uma página simples com um controle de rótulo e um controle de botão. À medida que os eventos de página como Page_Load, Page_Init, Page_PreRender etc. ocorrem, ele envia uma mensagem, que é exibida pelo controle de rótulo. Quando o botão é clicado, o evento Button_Click é gerado e também envia uma mensagem para ser exibida no rótulo.

Crie um novo site e arraste um controle de rótulo e um controle de botão na caixa de ferramentas de controle. Usando a janela de propriedades, defina os IDs dos controles como .lblmessage. e .btnclick. respectivamente. Defina a propriedade Text do controle Button como ‘Click’.

O arquivo de marcação (.aspx):

<%@ Page Language="C#" AutoEventWireup="true" CodeBehind="Default.aspx.cs" 
   Inherits="eventdemo._Default" %>

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" 
   "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

<html xmlns="http://www.w3.org/1999/xhtml" >

   <head runat="server">
      <title>Untitled Page</title>
   </head>
   
   <body>
      <form id="form1" runat="server">
         <div>
            <asp:Label ID="lblmessage" runat="server" >
            
            </asp:Label>
            
            <br />
            <br />
            <br />
            
            <asp:Button ID="btnclick" runat="server" Text="Click" onclick="btnclick_Click" />
         </div>
      </form>
   </body>
   
</html>

Clique duas vezes na visualização de design para mover para o arquivo de código por trás. O evento Page_Load é criado automaticamente sem nenhum código. Anote as seguintes linhas de código autoexplicativas:

using System;
using System.Collections;
using System.Configuration;
using System.Data;
using System.Linq;

using System.Web;
using System.Web.Security;
using System.Web.UI;
using System.Web.UI.HtmlControls;
using System.Web.UI.WebControls;
using System.Web.UI.WebControls.WebParts;

using System.Xml.Linq;

namespace eventdemo {

   public partial class _Default : System.Web.UI.Page {
   
      protected void Page_Load(object sender, EventArgs e) {
         lblmessage.Text += "Page load event handled. <br />";
         
         if (Page.IsPostBack) {
            lblmessage.Text += "Page post back event handled.<br/>";
         }
      }
      
      protected void Page_Init(object sender, EventArgs e) {
         lblmessage.Text += "Page initialization event handled.<br/>";
      }
      
      protected void Page_PreRender(object sender, EventArgs e) {
         lblmessage.Text += "Page prerender event handled. <br/>";
      }
      
      protected void btnclick_Click(object sender, EventArgs e) {
         lblmessage.Text += "Button click event handled. <br/>";
      }
       }
    }

Eventos de aplicativo e sessão #

Os eventos de aplicação mais importantes são:

Application_Start - É gerado quando o aplicativo/site é iniciado.

Application_End - É gerado quando o aplicativo/site é interrompido.

Da mesma forma, os eventos de sessão mais usados ​​são:

Session_Start - É gerado quando um usuário solicita pela primeira vez uma página do aplicativo.

Session_End - É gerado quando a sessão termina.

Eventos de página e controle #

Os eventos comuns de página e controle são:

DataBinding - É gerado quando um controle se vincula a uma fonte de dados.

Disposed - É gerado quando a página ou o controle é liberado.

Erro - É um evento de página, ocorre quando uma exceção não tratada é lançada.

Init - É gerado quando a página ou o controle é inicializado.

Load - É gerado quando a página ou um controle é carregado.

PreRender - É gerado quando a página ou o controle deve ser renderizado.

Descarregar - É gerado quando a página ou controle é descarregado da memória.
