Controles de usuário são contêineres que podem ser preenchidos com marcação HTML e controles de servidor com code-behind da mesma forma que a página ASPX. Elas são tratadas como unidades menores reutilizáveis ​​de uma página, portanto, não podem ser executadas como páginas independentes e não devem conter elementos HTML html, body ou form.

Introdução aos controles do usuário #

Os controles de usuário são feitos para reutilização em páginas ASP.NET, semelhantes às páginas mestras. Em vez de compartilhar o layout da página base, os controles de usuário compartilham um grupo de controles de servidor internos HTML/ASP.NET ou um layout de formulário específico, por exemplo, envio de comentários ou notas de convidados.

Um controle de usuário pode conter controles HTML e controles de servidor ASP.NET, incluindo scripts do lado do cliente.

Os controles de usuário geralmente incluem a diretiva Control em cima de sua definição:

<%@ Control Language="C#" AutoEventWireup="True" CodeFile="UserControl.ascx.cs" %>

Como a página ASPX, os controles de usuário consistem em marcações que podem ser associadas a um arquivo code behind para executar determinados eventos e tarefas, portanto, todas as tags HTML disponíveis na página ASPX podem ser usadas em controles de usuário, exceto <html>, <body> e <form> tags.

Aqui está um exemplo de marcação de controle de usuário simples:

<%-- UserControl.ascx --%>
<%@ Control Language="C#" AutoEventWireup="True" CodeFile="UserControl.ascx.cs" %>
<div>
    <asp:Label ID="Label1" runat="server" />
    <br />
    <asp:Button ID="Button1" runat="server" Text="Click Here" OnClick="Button1_Click" />
</div>

Exemplo de code-behind:

// UserControl.ascx.cs
public partial class UserControl : System.Web.UI.UserControl
{
    protected void Button1_Click(Object sender, EventArgs e)
    {
        Label1.Text = "Hello World!";
    }
}

Antes de um controle de usuário ser inserido na página ASPX, a diretiva Register deve ser declarada no topo da página referenciando o controle de usuário com sua URL de origem, nome da tag e prefixo da tag.

<%@ Register Src="UserControl.ascx" TagName="UserControl" TagPrefix="uc" %>

Depois, você pode colocar o controle do usuário dentro da página ASPX como o controle de servidor interno do ASP.NET:

<uc:UserControl ID="UserControl1" runat="server" />

Criando uma instância de controle de usuário programaticamente #

Se você deseja instanciar uma instância de controle de usuário dentro do código ASPX por trás da página, você precisa escrever a declaração de controle de usuário no evento Page_Load da seguinte forma:

public partial class Default : System.Web.UI.Page
{
    protected void Page_Load(Object sender, EventArgs e)
    {
        Control control1 = LoadControl("UserControl.ascx");
        Page.Controls.Add(control1);
    }
}

Observe que o arquivo ASCX de controle de usuário já deve estar criado ao executar o método LoadControl.

Outra maneira conhecida de declarar controles de usuário programaticamente é usar PlaceHolder:

public partial class Default : System.Web.UI.Page
{
    public PlaceHolder Placeholder1;
    protected void Page_Load(Object sender, EventArgs e)
    {
        Control control1 = LoadControl("UserControl.ascx");
        Placeholder1.Controls.Add(control1);
    }
}

Dependendo de sua necessidade, PlaceHolder coloca controles de usuário em um container armazenando todos os controles de servidor adicionados dinamicamente à página, onde Page.Controls insere diretamente o controle de usuário dentro da página que é mais preferido para renderizar controles literais HTML.

Adicionando propriedades personalizadas para controle de usuário #

Como os controles de servidor internos do ASP.NET padrão, os controles de usuário podem ter propriedades (atributos) em sua marca de definição. Suponha que você queira adicionar efeito de cor no arquivo UserControl.ascx assim:

<uc:UserControl ID="UserControl1" runat="server" Color="blue" />

Neste ponto, atributos/propriedades personalizados para controles de usuário podem ser definidos declarando propriedades dentro do código do controle de usuário:

private String _color;
public String Color
{
    get
    {
        return _color;
    }
    set
    {
        _color = value;
    }
}

Além disso, se você quiser definir o valor padrão em uma propriedade de controle de usuário, atribua o valor padrão dentro do método construtor do controle de usuário.

public UserControl()
{
    _color = "red";
}

Em seguida, a marcação de controle do usuário deve ser modificada para adicionar o atributo de cor conforme o exemplo a seguir:

<%@ Control Language="C#" AutoEventWireup="True" CodeFile="UserControl.ascx.cs" %>
<div>
    <span style="color:<%= Color %>"><asp:Label ID="Label1" runat="server" /></span>
    <br />
    <asp:Button ID="Button1" runat="server" Text="Click Here" OnClick="Button1_Click" />
</div>
