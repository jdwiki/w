Sintaxe #

  • Session[“Session_Key”] = Obj_Value;

HTTP é sem estado. O estado de sessão do ASP.NET é uma estrutura que facilita a manutenção do estado entre as solicitações de página HTTP.

Session difere das variáveis ​​de nível de classe em sua capacidade de permanecer disponível em post-backs e páginas diferentes. Por exemplo, uma variável de sessão criada em Page1.aspx estará disponível se o usuário for redirecionado para Page2.aspx posteriormente, dentro do mesmo aplicativo.

Além disso, em contraste com as variáveis ​​estáticas declaradas no nível da página, as variáveis ​​de sessão são independentes para diferentes usuários. Ou seja, alterar o valor da variável de sessão de um usuário não afetará o valor da mesma variável para outros usuários.

Embora o ViewState possa ser usado para armazenar os dados do usuário temporariamente, ele não permite salvar dados em várias páginas. Além disso, o viewstate faz parte da página e é enviado ao cliente. Como resultado, qualquer informação crítica relacionada ao usuário não pode ser salva no ViewState, e é aí que as variáveis ​​de sessão se tornam úteis.

Usando o objeto Session para armazenar valores #

O objeto System.Web.SessionState.HttpSessionState fornece uma maneira de persistir valores entre solicitações HTTP. No exemplo abaixo, a preferência de um usuário por avisos está sendo salva na sessão. Mais tarde, enquanto atende outra solicitação ao usuário, o aplicativo pode ler essa preferência da sessão e ocultar os avisos.

public partial class Default : System.Web.UI.Page
{
    public void LoadPreferences(object sender, EventArgs args)
    {
        // ... 
        // ... A DB operation that loads the user's preferences.
        // ...

        // Store a value with the key showWarnings
        HttpContext.Current.Session["showWarnings"] = false;
    }

    public void button2Clicked(object sender, EventArgs args)
    {
        // While processing another request, access this value.
        bool showWarnings = (bool)HttpContext.Current.Session["showWarnings"];
        lblWarnings.Visible = false;
    }
}

Observe que as variáveis ​​de sessão não são comuns para todos os usuários (assim como os cookies) e persistem em vários post-backs.

A sessão funciona definindo um cookie que contém um identificador para a sessão do usuário. Por padrão, esse identificador é armazenado na memória do servidor web, juntamente com os valores armazenados nele.

Usando um armazenamento de sessão SQL #

Se você achar que tem vários servidores que precisam compartilhar o estado da sessão, armazená-lo na memória do processo ASP.NET não funcionará. Por exemplo, você pode implantar em um ambiente de web-farm com um balanceador de carga que distribui solicitações de forma round-robin. Nesse ambiente, as solicitações de um único usuário podem ser atendidas por vários servidores.

No arquivo web.config você pode configurar um armazenamento de sessão do SQL Server.

<configuration>
  <system.web>
    <sessionState 
      mode="SQLServer"
      sqlConnectionString="Data Source=localhost;Integrated Security=SSPI"
      cookieless="true"
      timeout="30" />
  </system.web>
</configuration>

Para criar o esquema sql, use a ferramenta aspnet_regsql. [SampleSqlServerName] é o nome do host do servidor SQL. -ssadd diz à ferramenta para criar o banco de dados de estado da sessão. -sstype p diz à ferramenta para criar um novo banco de dados com o nome padrão ASPState.

aspnet_regsql.exe -S [SampleSqlServerName] -U [Username] -P [Password] -ssadd -sstype p

Usando um armazenamento de sessão do Amazon DynamoDB #

Se você não quiser usar o SQL Server, poderá usar o banco de dados nosql Dynamo DB hospedado da Amazon como um armazenamento de sessão.

Você precisará do AWS SDK. Para instalar isso no console do gerenciador de pacotes nuget do Visual Studio, use o seguinte comando

Install-Package AWSSDK

Você pode configurar seu provedor sessionState para usar um provedor personalizado. Você deve especificar a região e as credenciais, um perfil ou uma combinação de acesso do IAM e chave secreta. Por padrão, isso criará uma tabela chamada ASP.NET_SessionState.

<configuration>
  <system.web>
    <sessionState
      timeout="20"
      mode="Custom"
      customProvider="DynamoDBSessionStoreProvider">
      <providers>
        <add name="DynamoDBSessionStoreProvider"
             type="Amazon.SessionProvider.DynamoDBSessionStateStore"
             AWSProfileName="[PROFILE]"
             Region="[REGION]"
             CreateIfNotExist="true"
             />
      </providers>
    </sessionState>
  </system.web>
</configuration>
