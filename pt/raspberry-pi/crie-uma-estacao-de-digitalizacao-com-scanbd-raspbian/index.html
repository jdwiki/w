Introdução e instalação #

O objetivo desta documentação é usar um scanner sem qualquer interface de usuário. Um uso comum é enviar um arquivo PDF digitalizado diretamente para o Google Drive ou Dropbox apenas pressionando os botões do scanner.

scanbd permite acionar ações dos botões do scanner, ele usa sane.

O que torna as coisas um pouco mais complicadas é que ao pesquisar o scanner, o scanbd bloqueia o dispositivo. Como também precisamos escanear a partir de ferramentas locais (como scanimage), precisamos definir um proxy que interromperá o scandb do polling quando necessário. Este é o trabalho do scanbm, que monitora a rede e dá prioridade sobre o scanbd a qualquer acesso local ao scanner.

Assim, precisamos:

  • Configure a configuração local sã para digitalizar apenas da rede (através do scanbm)
  • Configure o scanbd para ter acesso e pesquisar o scanner local

Como raiz:

apt-get update
apt-get install libconfuse-dev libsane-dev libudev-dev libusb-dev xinetd

A versão mais recente do scanbd pode ser encontrada aqui: https://sourceforge.net/projects/scanbd/files/releases/

# Download the latest version of **scanbd**.
wget https://sourceforge.net/projects/scanbd/files/releases/scanbd-1.4.4.tgz/download

# Uncompress
tar xvzf scanbd-1.4.4.tgz && cd 1.4.4

# Configure with the installation path in /etc/
./configure --disable-Werror

# install it
make
make install

# Add a dbus policy to authorize "saned" user to manage scanbd :
cp integration/scanbd_dbus.conf /etc/dbus-1/system.d/

# Add the init script to manage the daemon :
cp integration/scanbd.debian /etc/init.d/scanbd

scanbd agora está instalado, mas ainda não configurado.

Configure o sane para digitalizar da rede #

A configuração local do sane está dentro de /etc/saned.d

/etc/sane.d/dll.conf

# /etc/sane.d/dll.conf - Configuration file for the SANE dynamic backend loader
#
# Backends can also be enabled by configuration snippets under
# /etc/sane.d/dll.d directory -- packages providing backends should drop
# a config file similar to dll.conf in this directory, named after the package.
#

# The next line enables the network backend; comment it out if you don't need
# to use a remote SANE scanner over the network - see sane-net(5) and saned(8)

net

dll.conf contém apenas o backend net.

/etc/sane.d/net.conf :

# This is the net backend config file.

## net backend options
# Timeout for the initial connection to saned. This will prevent the backend
# from blocking for several minutes trying to connect to an unresponsive
# saned host (network outage, host down, ...). Value in seconds.
connect_timeout = 3

## saned hosts
# Each line names a host to attach to.
# If you list "localhost" then your backends can be accessed either
# directly or through the net backend.  Going through the net backend
# may be necessary to access devices that need special privileges.

localhost

Neste ponto, o scanbm ainda não está configurado, nenhum scanner pode ser alcançado pela rede. Precisamos configurar o scanbm junto com o scanbd para testar a configuração.

Configure o scanbd para pesquisar o scanner local #

Identifique o scanner local #

Usando lsusb, identifique o productId (1909 aqui):

pi:# lsusb
pi:# Bus 001 Device 005: ID 04a9:1909 Canon, Inc. CanoScan LiDE 110

Com esse productId, grep o arquivo de configuração correto (depende do modelo do seu scanner, para mim é genesys.conf):

pi:# grep 1909 /etc/sane.d/*conf
pi:# /etc/sane.d/genesys.conf:usb 0x04a9 0x1909

Copie o arquivo dentro da pasta de configuração do scanbd:

cp /etc/sane.d/genesys.conf /usr/local/etc/scanbd/
cp /etc/sane.d/dll.conf /usr/local/etc/scanbd/

Edite /usr/local/etc/scanbd/dll.conf e substitua net pelo backend correto do scanner:

# /etc/sane.d/dll.conf - Configuration file for the SANE dynamic backend loader
#
# Backends can also be enabled by configuration snippets under
# /etc/sane.d/dll.d directory -- packages providing backends should drop
# a config file similar to dll.conf in this directory, named after the package.
#

# The next line enables the network backend; comment it out if you don't need
# to use a remote SANE scanner over the network - see sane-net(5) and saned(8)

genesys

Confirme se o scanner local foi encontrado pelo scanbd #

Agora estamos prontos para testar:

scanbd -d7 -f

Evite ao máximo manipular o env var SANE_CONFIG_DIR. Pode quebrar coisas, o valor padrão já está configurado em /usr/local/etc/scanbd/scanbd.conf e no script init.

A saída deve ser semelhante a:

scanbd: foreground
scanbd: reading config file /usr/local/etc/scanbd/scanbd.conf
scanbd: debug on: level: 7
scanbd: dropping privs to uid saned
scanbd: dropping privs to gid scanner
scanbd: group scanner has member: 
scanbd: saned
scanbd: pi
scanbd: drop privileges to gid: 110
scanbd: Running as effective gid 110
scanbd: drop privileges to uid: 110
scanbd: Running as effective uid 110
scanbd: dbus_init
scanbd: dbus match type='signal',interface='org.freedesktop.Hal.Manager'
scanbd: SANE_CONFIG_DIR=/usr/local/etc/scanbd/sane.d
scanbd: sane version 1.0
scanbd: Scanning for local-only devices
scanbd: found device: genesys:libusb:001:005 Canon LiDE 110 flatbed scanner

O mais importante aqui é o dispositivo encontrado.

E se “dispositivo encontrado” não aparecer? #

Você deve verificar novamente sua configuração, especialmente a parte de identificação desta documentação e limpe qualquer eventual SANE_CONFIG_DIR env var configurado incorretamente digitando: unset SANE_CONFIG_DIR

Configure o xinetd para que o scanbm escute na rede

Crie /etc/xinetd.d/sane-port que contém:

service sane-port
{
        port        = 6566
        socket_type = stream
        wait        = no
        user        = saned
        group       = scanner
        server      = /usr/local/sbin/scanbm
        server_args = scanbm -c /usr/local/etc/scanbd/scanbd.conf
        disable     = no
}

/!\ Verifique novamente o caminho de scanbm com which scanbm

Em seguida, reinicie o xinetd e inicie o scanbd :

service xinetd restart
/etc/init.d/scanbd start

scanimage -L deve agora exibir o scanner.

A maioria dos problemas nesta fase diz respeito às permissões. Garanta que scanimage -L funciona quando você está logado como saned fazendo sudo -u saned -s.

Caso contrário, verifique as permissões USB: ls -al /dev/bus/usb/***

Acione ações dos botões do scanner #

As ações estão localizadas em /usr/local/etc/scanbd/scanbd.conf. Tenho 4 botões que são digitalizar, copiar, e-mail e arquivo.

O arquivo de configuração padrão não inclui todas as ações por padrão, você provavelmente terá que adicionar o bloco manualmente. Você pode ter menos ou mais botões dependendo do modelo do seu scanner.

Para cada ação, definiremos um caminho personalizado para a opção script.

action scan {
    filter = "^scan.*"
    numerical-trigger {
            from-value = 1
            to-value   = 0
    }
    desc   = "Scan to file"
    # script must be an relative path starting from scriptdir (see above), 
    # or an absolute pathname. 
    # It must contain the path to the action script without arguments
    # Absolute path example: script = "/some/path/foo.script 
    script = "/home/pi/scan.sh"
}

Não se esqueça de comentar qualquer outra ação padrão no final do scanbd.conf :

# devices 
# each device can have actions and functions, you can disable not relevant devices
#include(scanner.d/avision.conf)
#include(scanner.d/fujitsu.conf)
#include(scanner.d/hp.conf)
#include(scanner.d/pixma.conf)
#include(scanner.d/snapscan.conf)
#include(scanner.d/canon.conf)

Agora você pode criar seu script personalizado para lidar com cada ação:

Cada linha relativa a /sys/class/leds/led0/trigger serve para controlar o LED para monitorar o que está acontecendo. Você pode fazer o que quiser, cat /sys/class/leds/led0/trigger fornece todos os diferentes padrões de luzes.

/home/pi/scan.sh

#!/bin/bash

# don't forget to create the folder
scan_dir=/home/pi/scanned-files
datetime=`date +%F_%H%M%S`

echo none >/sys/class/leds/led0/trigger

case $SCANBD_ACTION in
  scan)
    filename=file-$datetime
    logger -t "scanbd: $0" "$SCANBD_DEVICE $SCANBD_ACTION - scanning --resolution 150 --mode Color --depth 8 --format=tiff to $scan_dir/$filename.jpg"
    echo timer >/sys/class/leds/led0/trigger
    scanimage -d $SCANBD_DEVICE --resolution 150 --mode Color --depth 8 --format=tiff  --brightness 5 --contrast 20 | convert tiff:- -compress jpeg $scan_dir/$filename.pdf
    echo none >/sys/class/leds/led0/trigger
    logger -t "scanbd: $0" "Finished scanning"
    ;;
    
  email)
    logger -t "scanbd: $0" "Emailing $scan_dir/file-*pdf"
    echo heartbeat >/sys/class/leds/led0/trigger
    # here are the lines to send the file
    echo none >/sys/class/leds/led0/trigger

esac
