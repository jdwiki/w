Configuração do cartão SD #

Comece instalando o SO no cartão MicroSD, seja NOOBS ou Raspbian, ambos fornecidos pela fundação Raspberry Pi, disponível em seu site. NOOBS, que significa New Out Of Box Software, é projetado para iniciantes e é o mais fácil de instalar em seu cartão SD. Você pode seguir as instruções oficiais ou ver abaixo.

Formate o cartão SD com uma única partição de FAT32, usando a ferramenta de gerenciamento de unidade do sistema ou uma ferramenta de terceiros, como o SD Formatter 4.0 da SD Association ou GParted. Baixe o NOOBS arquivo ZIP e descompacte-o. em seguida, copie os arquivos para a partição que você acabou de criar. Certifique-se de ejetá-lo antes de desconectá-lo.

Instalação do Raspberry Pi - Windows #

Pretexto: Estas são instruções detalhadas sobre como configurar um Raspberry Pi com o sistema operacional Raspbian. Essas instruções são um pouco específicas do Windows. Algumas etapas de instalação também podem se aplicar a outros sistemas operacionais, mas lembre-se da primeira.

Conteúdo

  • Requisitos
  • Escolhendo um SO
  • Instalando o SO
  • Configuração básica

Requisitos

  1. Raspberry Pi (versão 1, 2 ou 3 - qualquer modelo)
  2. Fonte de alimentação (adaptador usb de 5V, 2000mAh - recomendado)
  3. Cartão MicroSD (Class 10 recomendado com pelo menos 8 GB - veja a lista de cartões compatíveis aqui: eLinux)
  4. Computador com leitor de cartão SD (ou leitor de cartão SD USB)
  5. Monitor/TV com porta HDMI
  6. Cabo HDMI
  7. Teclado USB
  8. Win32 Disk Imager

Escolhendo um sistema operacional

  • NOOBS Uma coleção de diferentes sistemas operacionais para o Raspberry Pi, contém Raspbian. Além do Raspbian, o NOOBS também inclui Pidora, LibreELEC, OSMC, RISK OS, Arch Linux, Windows 10 IOT Core e algumas outras opções. É importante observar que nem todas essas opções estão disponíveis em todos os modelos do Raspberry Pi (por exemplo, o Windows IOT Core está disponível apenas no Raspberry Pi 2 e mais recente). A instalação de qualquer outro sistema operacional além do Raspbian exigirá uma conexão ethernet no seu Raspberry Pi.
  • Raspbian O sistema operacional oficialmente suportado para o Raspberry Pi, é uma porta do sistema operacional conhecido como Debian. Pré-instalado com software educacional e com uma grande comunidade - este é o sistema operacional recomendado para o Raspberry Pi
  • Windows 10 IOT Core Uma versão simplificada do Windows projetada especificamente para dispositivos IOT (Internet Of Things). É importante notar que esta não é uma versão completa do Windows e não tem coisas que o usuário possa esperar, como um menu Iniciar e a capacidade de executar a maioria dos aplicativos do Windows.

Instalação do sistema operacional

Cartão SD

  1. Conecte seu cartão SD ao seu computador
  2. Baixe o sistema operacional escolhido no formato .img (Pode ser compactado em um arquivo .zip, que deve ser extraído após o download)
  3. Abra o Win32DiskImager
  4. Selecione seu cartão SD
  5. Selecione o arquivo .img do seu SO
  6. Clique em “Escrever”

Raspberry Pi

  1. Insira o cartão SD no Raspberry Pi
  2. Conecte seu teclado USB
  3. Conecte seu Monitor/TV
  4. Conecte um cabo Ethernet (opcional: mas recomendado!)
  5. Conecte o cabo de alimentação USB no Raspberry Pi

Sistema operacional

Se o sistema operacional foi gravado corretamente no cartão SD e o sistema operacional é um sistema operacional ARM válido, ele deve ser instalado automaticamente no Raspberry Pi - com pouca ou nenhuma interação. (No entanto, isso depende muito do tipo de sistema operacional escolhido) Se você escolher NOOBS, uma interface aparecerá e você poderá escolher o que instalar.

Configuração básica

Linha de comando:

O comando sudo raspi-config dá acesso à configuração do Raspberry Pi.

Opções de configuração:

1 Expand Filesystem              Ensures that all of the SD card storage is available to the OS
2 Change User Password           Change password for the default user (pi)
3 Enable Boot to Desktop/Scratch Choose whether to boot into a desktop environment, Scratch, or the command line
4 Internationalisation Options   Set up language and regional settings to match your location
5 Enable Camera                  Enable this Pi to work with the Raspberry Pi camera
6 Add to Rastrack                Add this Pi to the online Raspberry Pi Map (Rastrack)
7 Overclock                      Configure overclocking for your Pi
8 Advanced Options               Configure advanced settings
9 About `raspi-config`           Information about this configuration tool
  • Após uma instalação nova e limpa, é recomendável escolher a primeira opção: Expand Filesystem
  • Se você é europeu e não está familiarizado com o layout do teclado em inglês, você pode ir para Opções de Internacionalização e alterar o idioma do teclado e muito mais.
  • Recomenda-se evitar o menu Overclock como iniciante. Especialmente se você tiver o Raspberry Pi 3. O RPI 3 é conhecido por ficar muito quente, mesmo que não tenha overclock. Se você está pensando em fazer um overclock no RPI 3, por favor, adquira dissipadores de calor ou uma ventoinha para sua placa, para evitar que ela superaqueça.
  • A habilitação do acesso SSH pode ser feita no menu Opções Avançadas, as credenciais serão o nome de usuário e senha padrão.

GUI:

A instalação do Raspbian Jessie inicializa automaticamente em uma interface gráfica do usuário, as opções acima mencionadas são apresentadas no menu Opções no canto superior esquerdo. É recomendado para iniciantes continuar usando a GUI, mas se você comprou um Raspberry Pi para aprender a linha de comando. A opção de inicializar na linha de comando pode ser encontrada no menu Configurações.

Observação Alterar as configurações no Raspberry Pi geralmente solicita uma reinicialização que acontece quando você aceita as alterações feitas.

Informação extra Nome de usuário padrão: pi Senha padrão: framboesa

O desligamento do seu Raspberry Pi é feito puxando o plugue de alimentação USB. (Escolher o desligamento na GUI do Raspbian simplesmente colocará o Raspberry Pi em um estado diminuído, ainda usando energia - mas menos) A reinicialização do Raspberry Pi é feita simplesmente conectando o cabo após desconectá-lo. (A menos que a opção de reinicialização seja escolhida na GUI ou sudo reboot na linha de comando.

Introdução ao Raspberry Pi - Hello World in C #

Pretexto #

Esta é uma introdução ao exemplo do programa Hello World no Raspberry Pi escrito em C. O exemplo a seguir usa a interface de linha de comando e é configurado como um guia passo a passo. Junto com a criação de um programa Hello World, o leitor será apresentado a comandos simples de linha de comando do linux.

Nota: Esta introdução foi escrita para iniciantes.

Olá Mundo - Meu primeiro programa #

Primeiro passo: Criando um diretório que conterá o código-fonte.

  • Localize seu diretório pessoal escrevendo o seguinte comando cd
  • Crie uma nova pasta para o seu código fonte mkdir programas (Dica: o Linux diferencia maiúsculas de minúsculas ao gerenciar nomes de arquivos e diretórios)
  • Mude para o seu novo diretório programas de cd

Segundo passo: Escrevendo seu primeiro programa.

Os sistemas Linux oferecem uma grande variedade de editores de texto, nativamente você encontrará Vim ou Nano. Este exemplo fará uso do editor de texto Nano.

  • Crie seu arquivo de código fonte nano helloworld.c
  • Isso abrirá um editor - sair e salvar criará o arquivo em sua pasta.

O código a seguir é o código-fonte do programa Hello World:

/* My first program */

#include<stdio.h>

int main()
{
   printf("Hello World\n");
}
  • Depois de escrever o código, pressione ctrl + x para sair do editor, pressione y e depois enter para salvar as alterações.
  • Escreva o comando ls para verificar se o arquivo está presente em seu diretório.

Terceiro passo: Compilando seu primeiro programa.

  • Para compilar nosso arquivo de código fonte helloworld.c precisamos usar um compilador, usaremos o GNU Compiler Collection - também conhecido como GCC.
  • O comando a seguir compila o código-fonte para um programa binário executável gcc helloworld.c -o meuprimeiroprograma.bin

O arquivo de código fonte é oferecido como um argumento para o compilador GCC e -o define outro argumento expressando que gostaríamos que o compilador produzisse algo. Neste caso nós queremos que ele produza um arquivo .bin que nós nomeamos.

Existem vários outros argumentos que você pode usar ao compilar com o GCC, um exemplo seria -wall que habilita todos os avisos. Isso fornece informações sobre qualquer erro que o GCC possa encontrar.

Quarto passo: Executando seu primeiro programa.

  • A execução de um programa no Raspberry Pi é feita adicionando ./ na frente do nome do programa que você deseja executar.
  • Podemos executar nosso programa escrevendo ./myfirstprogram.bin

O comando deve executar o programa e produzir Hello World na janela do console.

Login com IPv6 #

Normalmente, lutamos com o processo de login no Raspberry Pi usando ssh ou outras ferramentas semelhantes. Mas podemos tornar o processo mais agradável.

Uma vez que seu Raspberry Pi está conectado a uma rede, ele obtém um endereço IPv4 e um endereço IPv6, que é baseado no endereço MAC da NIC. O bom disso é que o endereço IPv6 não muda mesmo que a rede tenha um serviço DHCP.

Dito isso, vamos descobrir nosso endereço IPv6. Você só precisa fazer login uma vez usando IPv4 ou usando um monitor (HDMI ou VGA). Abra um terminal e digite:

ifconfig

Você verá algo como:

eth0 Link encap: Ethernet HWaddr 00:1C:C0:AE:B5:E6
          inet addr:192.168.0.1  Bcast:192.168.0.255  Mask:255.255.255.0
          inet6 addr: fe80::21c:c0ff:feae:b5e6/64 Scope:Link
          UP BROADCAST RUNNING MULTICAST  MTU:1500  Metric:1
          RX packets:41620 errors:0 dropped:0 overruns:0 frame:0
          TX packets:40231 errors:0 dropped:0 overruns:0 carrier:0
          collisions:0 txqueuelen:1000 
          RX bytes:21601203 (20.6 MiB)  TX bytes:6145876 (5.8 MiB)
          Interrupt:21 Base address:0xe000

Como você pode ver, seu IPv4 seria 192.168.0.1 e seu IPv6 seria fe80::21c:c0ff:feae:b5e6 (observe a linha que começa com inet6 addr).

Então, com esta informação você pode logar usando o comando ssh com a seguinte sintaxe:

ssh -6 pi@fe80::21c:c0ff:feae:b5e6%eth0

Essa abordagem parece mais complicada, mas o endereço IPv6 é definido como o link local e, como é baseado no endereço MAC, a menos que você mesmo o altere, isso sempre funcionará independentemente do seu endereço IPv4.

Configuração de proxy #

Se você estiver atrás de um proxy e precisar se conectar à Internet, poderá usar:

export http_proxy="http://username:password@host:port/"

Para configurar o proxy dentro do apt-get:

cd /etc/apt/apt.conf.d

Crie um arquivo chamado 10proxy:

sudo nano 10proxy

Sem autenticação adicione a seguinte linha:

Acquire::http::Proxy "http://yourproxyaddress:proxyport/";
Acquire::https::Proxy "http://yourproxyaddress:proxyport/";

Com autenticação:

Acquire::http::Proxy "http://username:password@yourproxyaddress:proxyport/";

O / no final era importante. Sem ele não funciona.

Instalação do Arch Linux do Raspberry Pi v2 e v3 — do Mac ou Linux #

Uma das melhores distribuições Linux atualmente para Raspberry Pi (a partir de agora, “RPi”) é Arch Linux. Esta web mostra a instalação para:

  • RPi2. Arquitetura ARMv7 (32 bits).

  • RPi3. Existem duas opções:

    • ARMv7 architecture (32 bits) or
    • AArch architecture (64 bits).

Este tutorial aconselha você a não instalar a opção AArch — “Alguns dos hardwares da placa podem não funcionar ou podem funcionar mal”.
