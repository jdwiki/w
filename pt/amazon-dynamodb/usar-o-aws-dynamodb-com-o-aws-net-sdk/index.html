Usar o AWS DynamoDb com o AWS .NET SDK

Amazon DynamoDB é um serviço de banco de dados NoSQL rápido oferecido pela Amazon Web Services (AWS). O DynamoDB pode ser invocado de aplicativos .NET usando o AWS SDK para .NET. O SDK fornece três modelos diferentes para comunicação com o DynamoDB. Este tópico apresenta as várias APIs em cada modelo.

Os modelos #

O SDK oferece três formas de comunicação com o DynamoDB. Cada um oferece compensações entre controle e facilidade de uso. Consulte a Referência do SDK do AWS .NET para obter detalhes sobre as APIs abaixo.

  • Nível baixo: namespace Amazon.DynamoDBv2 — Este é um wrapper fino sobre as chamadas de serviço do DynamoDB. Ele corresponde a todos os recursos do serviço. Você pode consultar a documentação de serviço para saber mais sobre cada operação individual.

  • Modelo de Documento: namespace Amazon.DynamoDBv2.DocumentModel — Este é um modelo que fornece uma interface mais simples para lidar com dados. As tabelas do DynamoDB são representadas por objetos Table, enquanto linhas individuais de dados são representadas por objetos Document. A conversão de objetos .NET em dados do DynamoDB é automática para tipos básicos.

  • Modelo de persistência de objetos: namespace Amazon.DynamoDBv2.DataModel — Este conjunto de APIs permite armazenar e carregar objetos .NET no DynamoDB. Os objetos devem ser marcados para configurar a tabela de destino e as chaves de hash/intervalo. DynamoDBContext atua em objetos marcados. Ele é usado para armazenar e carregar dados do DynamoDB ou para recuperar objetos .NET de uma consulta ou operação de verificação. Os tipos de dados básicos são convertidos automaticamente em dados do DynamoDB e os conversores permitem que tipos arbitrários sejam armazenados no DynamoDB.

Os três modelos fornecem abordagens diferentes para trabalhar com o serviço. Embora a abordagem de baixo nível exija mais código do lado do cliente — o usuário deve converter tipos .NET, como números e datas, em strings com suporte do DynamoDB — ela fornece acesso a todos os recursos do serviço. Em comparação, a abordagem do Modelo de Persistência de Objetos facilita o uso do serviço — já que o usuário está trabalhando em grande parte com objetos .NET familiares — mas não fornece toda a funcionalidade. Por exemplo, não é possível fazer chamadas Put condicionais com o Object Persistence Model.

Saiba mais sobre como trabalhar na AWS usando o .NET SDK no Guia do desenvolvedor do .NET SDK.

Observação: este tópico foi adaptado com permissão de uma postagem de blog publicada originalmente no blog do AWS .NET SDK.

Exemplo de API de baixo nível #

var client = new AmazonDynamoDBClient();
 
// Store item
client.PutItem(new PutItemRequest
{
    TableName = "Books",
    Item = new Dictionary<string, AttributeValue>
    {
        { "Title", new AttributeValue { S = "Cryptonomicon" } },
        { "Id", new AttributeValue { N = "42" } },
        { "Authors", new AttributeValue {
            SS = new List<string> { "Neal Stephenson" } } },
        { "Price", new AttributeValue { N = "12.95" } }
    }
});
 
// Get item
Dictionary<string, AttributeValue> book = client.GetItem(new GetItemRequest
{
    TableName = "Books",
    Key = new Dictionary<string, AttributeValue>
    {
        { "Id", new AttributeValue { N = "42" } }
    }
}).Item;
 
Console.WriteLine("Id = {0}", book["Id"].S);
Console.WriteLine("Title = {0}", book["Title"].S);
Console.WriteLine("Authors = {0}",
    string.Join(", ", book["Authors"].SS));

Exemplo de modelo de documento #

var client = new AmazonDynamoDBClient();
Table booksTable = Table.LoadTable(client, "Books");
 
// Store item
Document book = new Document();
book["Title"] = "Cryptonomicon";
book["Id"] = 42;
book["Authors"] = new List<string> { "Neal Stephenson" };
book["Price"] = 12.95;
booksTable.PutItem(book);
 
// Get item
book = booksTable.GetItem(42);
Console.WriteLine("Id = {0}", book["Id"]);
Console.WriteLine("Title = {0}", book["Title"]);
Console.WriteLine("Authors = {0}",
    string.Join(", ", book["Authors"].AsListOfString()));

Exemplo de modelo de persistência de objeto #

Este exemplo consiste em duas partes: primeiro, devemos definir nosso tipo Book; segundo, nós o usamos com DynamoDBContext.

[DynamoDBTable("Books")]
class Book
{
    [DynamoDBHashKey]
    public int Id { get; set; }
    public string Title { get; set; }
    public List<string> Authors { get; set; }
    public double Price { get; set; }
}

Agora, use-o com DynamoDBContext.

var client = new AmazonDynamoDBClient();
DynamoDBContext context = new DynamoDBContext(client);
 
// Store item
Book book = new Book
{
    Title = "Cryptonomicon",
    Id = 42,
    Authors = new List<string> { "Neal Stephenson" },
    Price = 12.95
};
context.Save(book);
 
// Get item
book = context.Load<Book>(42);
Console.WriteLine("Id = {0}", book.Id);
Console.WriteLine("Title = {0}", book.Title);
Console.WriteLine("Authors = {0}", string.Join(", ", book.Authors));
