ResolverDados #

Este exemplo mostrará como você pode resolver os dados obtidos de um serviço antes de renderizar a visualização do seu aplicativo.

Usa angular/roteador 3.0.0-beta.2 no momento da escrita

users.service.ts

...
import { Http, Response } from '@angular/http';
import { Observable } from 'rxjs/Rx';
import { User } from './user.ts';



@Injectable()
export class UsersService {

    constructor(public http:Http) {}

    /**
     * Returns all users
     * @returns {Observable<User[]>}
     */
    index():Observable<User[]> {

        return this.http.get('http://mywebsite.com/api/v1/users')
            .map((res:Response) => res.json());
    }

    /**
     * Returns a user by ID
     * @param id
     * @returns {Observable<User>}
     */
    get(id:number|string):Observable<User> {

        return this.http.get('http://mywebsite.com/api/v1/users/' + id)
            .map((res:Response) => res.json());
    }
}

users.resolver.ts

...
import { UsersService } from './users.service.ts';
import { Observable } from 'rxjs/Rx';
import {
    Resolve,
    ActivatedRouteSnapshot,
    RouterStateSnapshot
} from "@angular/router";




@Injectable()
export class UsersResolver implements Resolve<User[] | User> {

    // Inject UsersService into the resolver
    constructor(private service:UsersService) {}

    resolve(route:ActivatedRouteSnapshot, state:RouterStateSnapshot):Observable<User[] | User> {
        // If userId param exists in current URL, return a single user, else return all users
        // Uses brackets notation to access `id` to suppress editor warning, may use dot notation if you create an interface extending ActivatedRoute with an optional id? attribute
        if (route.params['id']) return this.service.get(route.params['id']);
        return this.service.index();
    }
}

users.component.ts

Este é um componente de página com uma lista de todos os usuários. Ele funcionará de forma semelhante para o componente da página de detalhes do usuário, substitua data.users por data.user ou qualquer chave definida em app.routes.ts (veja abaixo)

...
import { ActivatedRoute} from "@angular/router";



@Component(...)
export class UsersComponent {

    users:User[];


    constructor(route: ActivatedRoute) {
        route.data.subscribe(data => {
            // data['Match key defined in RouterConfig, see below']
            this.users = data.users;
        });
    }

    /**
     * It is not required to unsubscribe from the resolver as Angular's HTTP
     * automatically completes the subscription when data is received from the server
     */
}

app.routes.ts

...
import { UsersResolver } from './resolvers/users.resolver';



export const routes:RouterConfig = <RouterConfig>[
    ...
    {
        path: 'user/:id',
        component: UserComponent,
        resolve: {
            // hence data.user in UserComponent
            user: UsersResolver
        }
    },
    {
        path: 'users',
        component: UsersComponent,
        resolve: {
            // hence data.users in UsersComponent, note the pluralisation
            users: UsersResolver
        }
    },
    ...
]
...

app.resolver.ts

Opcionalmente, agrupe vários resolvedores.

IMPORTANTE: Serviços usados ​​no resolvedor devem ser importados primeiro ou você receberá a mensagem ‘Nenhum provedor para ..Erro do solucionador’. Lembre-se de que esses serviços estarão disponíveis globalmente e você não precisará mais declará-los nos providers de nenhum componente. Certifique-se de cancelar a assinatura de qualquer assinatura para evitar vazamento de memória

...
import { UsersService } from './users.service';
import { UsersResolver } from './users.resolver';

export const ROUTE_RESOLVERS = [
    ...,
    UsersService,
    UsersResolver
]

main.browser.ts

Os resolvedores devem ser injetados durante a inicialização.

...
import {bootstrap} from '@angular/platform-browser-dynamic';
import { ROUTE_RESOLVERS } from './app.resolver';

bootstrap(<Type>App, [
    ...
    ...ROUTE_RESOLVERS
])
.catch(err => console.error(err));

Roteamento com crianças #

Ao contrário da documentação original, descobri que essa é a maneira de aninhar corretamente as rotas filhas dentro do arquivo app.routing.ts ou app.module.ts (dependendo da sua preferência). Essa abordagem funciona ao usar WebPack ou SystemJS.

O exemplo abaixo mostra rotas para home, home/counter e home/counter/fetch-data. A primeira e a última rotas são exemplos de redirecionamentos. Por fim no final do exemplo está uma forma adequada de exportar a Rota a ser importada em um arquivo separado. Por ex. app.module.ts

Para explicar melhor, o Angular requer que você tenha uma rota sem caminho na matriz filhos que inclua o componente pai, para representar a rota pai. É um pouco confuso, mas se você pensar em um URL em branco para uma rota filho, seria essencialmente igual ao mesmo URL da rota pai.

import { NgModule } from "@angular/core";
import { RouterModule, Routes } from "@angular/router";

import { HomeComponent } from "./components/home/home.component";
import { FetchDataComponent } from "./components/fetchdata/fetchdata.component";
import { CounterComponent } from "./components/counter/counter.component";

const appRoutes: Routes = [
    {
        path: "",
        redirectTo: "home",
        pathMatch: "full"
    },
    {
        path: "home",            
        children: [
            {
                path: "",
                component: HomeComponent
            },
            {
                path: "counter",                    
                children: [
                    {
                        path: "",
                        component: CounterComponent
                    },
                    {
                        path: "fetch-data",
                        component: FetchDataComponent                            
                    }
                ]
            }
        ]
    },
    {
        path: "**",
        redirectTo: "home"
    }
];

@NgModule({
    imports: [
        RouterModule.forRoot(appRoutes)
    ],
    exports: [
        RouterModule
    ]
})
export class AppRoutingModule { }

Roteamento Básico #

O roteador permite a navegação de uma visualização para outra com base nas interações do usuário com o aplicativo.

A seguir estão as etapas na implementação de roteamento básico em Angular 2 -

Precaução básica: certifique-se de ter a tag

    <base href='/'>

como o primeiro filho sob sua tag head em seu arquivo index.html. Essa tag informa que a pasta do seu aplicativo é a raiz do aplicativo. Angular 2 saberia então organizar seus links.

Primeiro passo é verificar se você está apontando para as dependências de roteamento corretas/mais recentes no package.json -

"dependencies": {
  ......
  "@angular/router": "3.0.0-beta.1",
  ......
}

Segundo passo é definir a rota de acordo com sua definição de classe -

class Route {
  path : string
  pathMatch : 'full'|'prefix'
  component : Type|string
  .........
}

Em um arquivo de rotas (route/routes.ts), importe todos os componentes que você precisa configurar para diferentes caminhos de roteamento. Caminho vazio significa que a visualização é carregada por padrão. “:” no caminho indica o parâmetro dinâmico passado para o componente carregado.

As rotas são disponibilizadas para o aplicativo por meio de injeção de dependência. O método ProviderRouter é chamado com RouterConfig como parâmetro para que possa ser injetado nos componentes para chamar tarefas específicas de roteamento.

import { provideRouter, RouterConfig } from '@angular/router';
import { BarDetailComponent } from '../components/bar-detail.component';
import { DashboardComponent } from '../components/dashboard.component';
import { LoginComponent } from '../components/login.component';
import { SignupComponent } from '../components/signup.component';

export const appRoutes: RouterConfig = [
  { path: '', pathMatch: 'full', redirectTo: 'login' },
  { path: 'dashboard', component: DashboardComponent },
  { path: 'bars/:id', component: BarDetailComponent },
  { path: 'login', component: LoginComponent },
  { path: 'signup',   component: SignupComponent }
];

export const APP_ROUTER_PROVIDER = [provideRouter(appRoutes)];

Terceira etapa é inicializar o provedor de rotas.

No seu main.ts (pode ser qualquer nome. basicamente, deve ser seu arquivo principal definido em systemjs.config)

import { bootstrap } from '@angular/platform-browser-dynamic';
import { AppComponent } from './components/app.component';
import { APP_ROUTER_PROVIDER } from "./routes/routes";

bootstrap(AppComponent, [ APP_ROUTER_PROVIDER ]).catch(err => console.error(err));

A quarta etapa é carregar/exibir os componentes do roteador com base no caminho acessado. A diretiva é usada para informar ao angular onde carregar o componente. Para usar importe o ROUTER_DIRECTIVES.

import { ROUTER_DIRECTIVES } from '@angular/router';

@Component({
  selector: 'demo-app',
  template: `
    ....................................
    <div>
      <router-outlet></router-outlet>
    </div>
    ....................................
  `,
  // Add our router directives we will be using
  directives: [ROUTER_DIRECTIVES]
})

Quinto passo é ligar as outras rotas. Por padrão, RouterOutlet carregará o componente para o qual o caminho vazio é especificado no RouterConfig. A diretiva RouterLink é usada com a marca âncora html para carregar os componentes anexados às rotas. RouterLink gera o atributo href que é usado para gerar links. Por exemplo:

import { Component } from '@angular/core';
import { ROUTER_DIRECTIVES } from '@angular/router';

@Component({
  selector: 'demo-app',
  template: `
    <a [routerLink]="['/login']">Login</a>
    <a [routerLink]="['/signup']">Signup</a>      
    <a [routerLink]="['/dashboard']">Dashboard</a>
    <div>
      <router-outlet></router-outlet>
    </div>
  `,
  // Add our router directives we will be using
  directives: [ROUTER_DIRECTIVES]
})
export class AppComponent { }

Agora, somos bons com roteamento para caminho estático. RouterLink suporta caminho dinâmico também passando parâmetros extras junto com o caminho.

importe { Componente } de ‘@angular/core’; importe { ROUTER_DIRECTIVES } de ‘@angular/router’;

@Component({
  selector: 'demo-app',
  template: `
        <ul>
          <li *ngFor="let bar of bars | async">
            <a [routerLink]="['/bars', bar.id]">
              {{bar.name}}
            </a>
          </li>
        </ul>
    <div>
      <router-outlet></router-outlet>
    </div>
  `,
  // Add our router directives we will be using
  directives: [ROUTER_DIRECTIVES]
})
export class AppComponent { }

RouterLink recebe uma matriz onde o primeiro elemento é o caminho para roteamento e os elementos subsequentes são para os parâmetros de roteamento dinâmico.

Rotas secundárias #

Às vezes, faz sentido aninhar visualizações ou rotas umas dentro das outras. Por exemplo, no painel, você deseja várias subvisualizações, semelhantes a guias, mas implementadas através do sistema de roteamento, para mostrar os projetos, contatos, mensagens e conjuntos dos usuários. Para suportar tais cenários, o roteador nos permite definir rotas filhas.

Primeiro ajustamos nosso RouterConfig de cima e adicionamos as rotas filhas:

import { ProjectsComponent } from '../components/projects.component';
import { MessagesComponent} from '../components/messages.component';

export const appRoutes: RouterConfig = [
  { path: '', pathMatch: 'full', redirectTo: 'login' },
  { path: 'dashboard', component: DashboardComponent,
    children: [
      { path: '', redirectTo: 'projects', pathMatch: 'full' },
      { path: 'projects', component: 'ProjectsComponent' },
      { path: 'messages', component: 'MessagesComponent' }
    ] },
  { path: 'bars/:id', component: BarDetailComponent },
  { path: 'login', component: LoginComponent },
  { path: 'signup',   component: SignupComponent }
];

Agora que temos nossas rotas filhas definidas, temos que garantir que essas rotas filhas possam ser exibidas em nosso DashboardComponent, já que é onde adicionamos as filhas. Anteriormente, aprendemos que os componentes são exibidos em uma tag <router-outlet></router-outlet>. Similarmente, declaramos outro RouterOutlet no DashboardComponent:

import { Component } from '@angular/core';

@Component({
  selector: 'dashboard',
  template: `
    <a [routerLink]="['projects']">Projects</a>
    <a [routerLink]="['messages']">Messages</a>   
    <div>
      <router-outlet></router-outlet>
    </div>
  `
})
export class DashboardComponent { }

Como você pode ver, adicionamos outro RouterOutlet no qual as rotas filhas serão exibidas. Normalmente a rota com um caminho vazio será mostrada, porém, configuramos um redirecionamento para a rota projects, pois queremos que isso seja mostrado imediatamente quando a rota dashboard for carregada. Dito isto, precisamos de uma rota vazia, caso contrário, você receberá um erro como este:

Cannot match any routes: 'dashboard'

Então, adicionando a rota vazia, ou seja, uma rota com um caminho vazio, definimos um ponto de entrada para o roteador.
