Instale o angular2 com angular-cli #

Este exemplo é uma configuração rápida do Angular 2 e como gerar um projeto de exemplo rápido.

Pré-requisitos: #

  • Node.js v4 ou superior.
  • npm v3 ou superior ou yarn.

Abra um terminal e execute os comandos um por um:

npm install -g @angular/cli

ou

yarn global add @angular/cli

dependendo da sua escolha de gerenciador de pacotes.

O comando anterior instala @angular/cli globalmente, adicionando o executável ng ao PATH.

Para configurar um novo projeto #

Navegue com o terminal para uma pasta onde você deseja configurar o novo projeto.

Execute os comandos:

ng new PROJECT_NAME
cd PROJECT_NAME
ng serve

É isso, agora você tem um projeto de exemplo simples feito com Angular 2. Agora você pode navegar até o link exibido no terminal e ver o que ele está executando.

Para adicionar a um projeto existente #

Navegue até a raiz do seu projeto atual.

Execute o comando:

ng init

Isso adicionará o andaime necessário ao seu projeto. Os arquivos serão criados no diretório atual, portanto, certifique-se de executá-lo em um diretório vazio.

Executando o projeto localmente #

Para ver e interagir com seu aplicativo enquanto ele está sendo executado no navegador, você deve iniciar um servidor de desenvolvimento local hospedando os arquivos do seu projeto.

ng serve

Se o servidor foi iniciado com sucesso, ele deve exibir um endereço no qual o servidor está sendo executado. Geralmente é isso:

http://localhost:4200

Fora da caixa, este servidor de desenvolvimento local está conectado com o Hot Module Reloading, portanto, qualquer alteração no html, typescript ou css fará com que o navegador seja recarregado automaticamente (mas pode ser desativado, se desejado).

Geração de Componentes, Diretivas, Tubos e Serviços #

O comando ng generate <scaffold-type> <name> (ou simplesmente ng g <scaffold-type> <name>) permite que você gere automaticamente componentes Angular:

# The command below will generate a component in the folder you are currently at
ng generate component my-generated-component
# Using the alias (same outcome as above)
ng g component my-generated-component

Existem vários tipos possíveis de andaimes que o angular-cli pode gerar:

Tipo de andaime Uso
Módulo ng g módulo meu-novo-módulo
Componente ng g componente meu-novo-componente
Directiva ng g diretiva minha-nova diretiva
Tubulação ng g pipe my-new-pipe
Serviço ng g serviço meu-novo-serviço
Classe ng g class minha-nova-classe
Interface ng g interface minha-nova-interface
Enum ng g enum meu-novo-enum

Você também pode substituir o nome do tipo por sua primeira letra. Por exemplo:

ng g m my-new-module para gerar um novo módulo ou ng g c my-new-component para criar um componente.

Construção/Agregação

Quando você terminar de construir seu aplicativo web Angular 2 e quiser instalá-lo em um servidor web como o Apache Tomcat, tudo o que você precisa fazer é executar o comando build com ou sem o sinalizador de produção definido. A produção minimizará o código e otimizará para uma configuração de produção.

ng build

ou

ng build --prod

Em seguida, procure no diretório raiz do projeto por uma pasta /dist, que contém o arquivo build.

Se desejar os benefícios de um pacote de produção menor, você também pode usar a compilação de modelo Ahead-of-Time, que remove o compilador de modelo da compilação final:

ng build --prod --aot

Teste Unitário

O Angular 2 fornece testes de unidade integrados e cada item criado pelo angular-cli gera um teste de unidade básico, que pode ser expandido. Os testes unitários são escritos usando jasmine e executados através do Karma. Para iniciar o teste execute o seguinte comando:

ng test

Este comando executará todos os testes no projeto e os executará novamente toda vez que um arquivo fonte for alterado, seja um teste ou código do aplicativo.

Para mais informações, visite: angular-cli github page

Começando com Angular 2 sem angular-cli. #

Angular 2.0.0-rc.4

Neste exemplo, criaremos um “Hello World!” app com apenas um componente raiz (AppComponent) para simplificar.

Pré-requisitos:

  • Node.js v5 ou posterior
  • npm v3 ou posterior

Nota: Você pode verificar as versões executando node -v e npm -v no console/terminal.

Passo 1 #

Crie e insira uma nova pasta para o seu projeto. Vamos chamá-lo de angular2-example.

mkdir angular2-example
cd angular2-example

Passo 2 #

Antes de começarmos a escrever o código do nosso aplicativo, adicionaremos os 4 arquivos fornecidos abaixo: package.json, tsconfig.json, typings.json e systemjs.config.js.

Isenção de responsabilidade: Os mesmos arquivos podem ser encontrados no Início Rápido Oficial de 5 Minutos.

package.json - Permite baixar todas as dependências com npm e fornece execução de script simples para facilitar a vida de projetos simples. (Você deve considerar usar algo como Gulp no futuro para automatizar tarefas).

{
  "name": "angular2-example",
  "version": "1.0.0",
  "scripts": {
    "start": "tsc && concurrently \"npm run tsc:w\" \"npm run lite\" ",
    "lite": "lite-server",
    "postinstall": "typings install",
    "tsc": "tsc",
    "tsc:w": "tsc -w",
    "typings": "typings"
  },
  "license": "ISC",
  "dependencies": {
    "@angular/common": "2.0.0-rc.4",
    "@angular/compiler": "2.0.0-rc.4",
    "@angular/core": "2.0.0-rc.4",
    "@angular/forms": "0.2.0",
    "@angular/http": "2.0.0-rc.4",
    "@angular/platform-browser": "2.0.0-rc.4",
    "@angular/platform-browser-dynamic": "2.0.0-rc.4",
    "@angular/router": "3.0.0-beta.1",
    "@angular/router-deprecated": "2.0.0-rc.2",
    "@angular/upgrade": "2.0.0-rc.4",
    "systemjs": "0.19.27",

  },

{

typings.json - Faz com que o TypeScript reconheça as bibliotecas que estamos usando.

{
  "globalDependencies": {
    "core-js": "registry:dt/core-js#0.0.0+20160602141332",
    "jasmine": "registry:dt/jasmine#2.2.0+20160621224255",
    "node": "registry:dt/node#6.0.0+20160621231320"
  }
}

systemjs.config.js - Configura SystemJS (você também pode usar webpack).

/**
 * System configuration for Angular 2 samples
 * Adjust as necessary for your application's needs.
 */
(function(global) {
  // map tells the System loader where to look for things
  var map = {
    'app':                        'app', // 'dist',
    '@angular':                   'node_modules/@angular',
    'angular2-in-memory-web-api': 'node_modules/angular2-in-memory-web-api',
    'rxjs':                       'node_modules/rxjs'
  };
  // packages tells the System loader how to load when no filename and/or no extension
  var packages = {
    'app':                        { main: 'main.js',  defaultExtension: 'js' },
    'rxjs':                       { defaultExtension: 'js' },
    'angular2-in-memory-web-api': { main: 'index.js', defaultExtension: 'js' },
  };
  var ngPackageNames = [
    'common',
    'compiler',
    'core',
    'forms',
    'http',
    'platform-browser',
    'platform-browser-dynamic',
    'router',
    'router-deprecated',
    'upgrade',
  ];
  // Individual files (~300 requests):
  function packIndex(pkgName) {
    packages['@angular/'+pkgName] = { main: 'index.js', defaultExtension: 'js' };
  }
  // Bundled (~40 requests):
  function packUmd(pkgName) {
    packages['@angular/'+pkgName] = { main: '/bundles/' + pkgName + '.umd.js', defaultExtension: 'js' };
  }
  // Most environments should use UMD; some (Karma) need the individual index files
  var setPackageConfig = System.packageWithIndex ? packIndex : packUmd;
  // Add package entries for angular packages
  ngPackageNames.forEach(setPackageConfig);
  var config = {
    map: map,
    packages: packages
  };
  System.config(config);
})(this);

Etapa 3 #

Vamos instalar as dependências digitando

npm install

no console/terminal.

Passo 4

Crie index.html dentro da pasta angular2-example.

<html>
  <head>
    <title>Angular2 example</title>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <!-- 1. Load libraries -->
    <!-- Polyfill(s) for older browsers -->
    <script src="node_modules/core-js/client/shim.min.js"></script>
    <script src="node_modules/zone.js/dist/zone.js"></script>
    <script src="node_modules/reflect-metadata/Reflect.js"></script>
    <script src="node_modules/systemjs/dist/system.src.js"></script>
    <!-- 2. Configure SystemJS -->
    <script src="systemjs.config.js"></script>
    <script>
      System.import('app').catch(function(err){ console.error(err); });
    </script>
  </head>
  <!-- 3. Display the application -->
  <body>
    <my-app></my-app>
  </body>
</html>

Seu aplicativo será renderizado entre as tags my-app.

No entanto, o Angular ainda não sabe o que renderizar. Para dizer isso, vamos definir AppComponent.

Etapa 5 #

Crie uma subpasta chamada app onde podemos definir os componentes e serviços que compõem nosso aplicativo. (Neste caso, ele conterá apenas o código AppComponent e main.ts.)

mkdir app

Etapa 6 #

Crie o arquivo app/app.component.ts

import { Component } from '@angular/core';

@Component({
  selector: 'my-app',
  template: `
    <h1>{{title}}</h1>
    <ul>
        <li *ngFor="let message of messages">
            {{message}}
        </li>
    </ul>
  `
})
export class AppComponent { 
    title = "Angular2 example";
    messages = [
        "Hello World!",
        "Another string",
        "Another one"
    ];
}

O que está acontecendo? Primeiro, estamos importando o decorador @Component que usamos para dar ao Angular a tag HTML e o template para este componente. Então, estamos criando a classe AppComponent com as variáveis ​​title e messages que podemos usar no template.

Agora vamos ver esse modelo:

<h1>{{title}}</h1>
<ul>
    <li *ngFor="let message of messages">
        {{message}}
    </li>
</ul>

Estamos exibindo a variável title em uma tag h1 e depois fazendo uma lista mostrando cada elemento do array messages usando a diretiva *ngFor. Para cada elemento no array, *ngFor cria uma variável message que usamos dentro do elemento li. O resultado será:

<h1>Angular 2 example</h1>
<ul>
    <li>Hello World!</li>
    <li>Another string</li>
    <li>Another one</li>
</ul>

Etapa 7 #

Agora criamos um arquivo main.ts, que será o primeiro arquivo que o Angular examinará.

Crie o arquivo app/main.ts.

import { bootstrap }    from '@angular/platform-browser-dynamic';
import { AppComponent } from './app.component';

bootstrap(AppComponent);

Estamos importando a função bootstrap e a classe AppComponent, então usando bootstrap para dizer ao Angular qual componente usar como root.

Etapa 8 #

É hora de iniciar seu primeiro aplicativo. Modelo

npm start

em seu console/terminal. Isso executará um script preparado de package.json que inicia o lite-server, abre seu aplicativo em uma janela do navegador e executa o transpilador TypeScript no modo de observação (assim, os arquivos .ts serão transpilados e o navegador será atualizado quando você salva as alterações).

E agora? #

Confira o guia oficial do Angular 2 e os outros tópicos na documentação do StackOverflow.

Você também pode editar AppComponent para usar templates externos, estilos ou adicionar/editar variáveis ​​de componentes. Você deve ver suas alterações imediatamente após salvar os arquivos.

Introdução ao Angular 2 com backend node.js/expressjs (exemplo http incluído) #

Vamos criar um simples “Hello World!” app com Angular2 2.4.1 (alteração @NgModule) com um backend node.js (expressjs).

Pré-requisitos #

  • Node.js v4.x.x ou superior
  • npm v3.x.x ou superior ou yarn

Em seguida, execute npm install -g typescript ou yarn global add typescript para instalar o typescript globalmente

Roteiro #

Passo 1 #

Crie uma nova pasta (e o diretório raiz do nosso back-end) para nosso aplicativo. Vamos chamá-lo de Angular2-express.

linha de comando:

mkdir Angular2-express
cd Angular2-express

Passo 2 #

Crie o package.json (para dependências) e app.js (para bootstrapping) para nosso aplicativo node.js.

package.json:

{
  "name": "Angular2-express",
  "version": "1.0.0",
  "description": "",
  "scripts": {
    "start": "node app.js"
  },
  "author": "",
  "license": "ISC",
  "dependencies": {
    "body-parser": "^1.13.3",
    "express": "^4.13.3"
  }
}

app.js:

var express = require('express');
var app = express();
var server = require('http').Server(app);
var bodyParser = require('body-parser');

server.listen(process.env.PORT || 9999, function(){
    console.log("Server connected. Listening on port: " + (process.env.PORT || 9999));
});

app.use(bodyParser.json());
app.use(bodyParser.urlencoded({extended: true}) );

app.use( express.static(__dirname + '/front' ) );

app.get('/test', function(req,res){ //example http request receiver

Passando por aquele proxy de empresa irritante #

Se você está tentando obter um site Angular2 em execução no seu computador de trabalho Windows na XYZ MegaCorp, é provável que esteja tendo problemas para acessar o proxy da empresa.

Existem (pelo menos) dois gerenciadores de pacotes que precisam passar pelo proxy:

  1. NPM
  2. Digitação

Para NPM você precisa adicionar as seguintes linhas ao arquivo .npmrc:

proxy=http://[DOMAIN]%5C[USER]:[PASS]@[PROXY]:[PROXYPORT]/
https-proxy=http://[DOMAIN]%5C[USER]:[PASS]@[PROXY]:[PROXYPORT]/

Para Typings você precisa adicionar as seguintes linhas ao arquivo .typingsrc:

proxy=http://[DOMAIN]%5C[USER]:[PASS]@[PROXY]:[PROXYPORT]/
https-proxy=http://[DOMAIN]%5C[USER]:[PASS]@[PROXY]:[PROXYPORT]/
rejectUnauthorized=false

Esses arquivos provavelmente ainda não existem, então você pode criá-los como arquivos de texto em branco. Eles podem ser adicionados à raiz do projeto (mesmo lugar que package.json ou você pode colocá-los em %HOMEPATH% e eles estarão disponíveis para todos os seus projetos.

A parte que não é óbvia e é a principal razão pela qual as pessoas pensam que as configurações de proxy não estão funcionando é o %5C que é a codificação de URL do \ para separar o domínio e os nomes de usuário. Obrigado a Steve Roberts por isso: http://stackoverflow.com/questions/25660936/using-npm-behind-corporate-proxy-pac

Vamos mergulhar no Angular 4! #

Angular 4 já está disponível! Na verdade, o Angular usa sempre desde o Angular 2, o que exige que o número principal seja aumentado quando as alterações de quebra foram introduzidas. A equipe do Angular adiou recursos que causam mudanças de quebra, que serão lançadas com o Angular 4. O Angular 3 foi ignorado para poder alinhar os números de versão dos módulos principais, porque o Roteador já tinha a versão 3.

De acordo com a equipe Angular, os aplicativos Angular 4 consumirão menos espaço e serão mais rápidos do que antes. Eles separaram o pacote de animação do pacote @angular/core. Se alguém não estiver usando o pacote de animação, espaço extra de código não acabará na produção. A sintaxe de vinculação de modelo agora suporta a sintaxe de estilo if/else. Angular 4 agora é compatível com a versão mais recente do Typescript 2.1 e 2.2. Então, Angular 4 será mais emocionante.

Agora vou mostrar como fazer a configuração do Angular 4 em seu projeto.

Vamos começar a configuração do Angular de três maneiras diferentes:

Você pode usar o Angular-CLI (Command Line Interface) , ele instalará todas as dependências para você.

  • Você pode migrar de Angular 2 para Angular 4.

  • Você pode usar o github e clonar o Angular4-boilerplate. (É o mais fácil.😉)

  • Configuração Angular usando Angular-CLI (interface de linha de comando).

Antes de começar a usar o Angular-CLI, certifique-se de ter o nó instalado em sua máquina. Aqui, estou usando o nó v7.8.0. Agora, abra seu terminal e digite o seguinte comando para Angular-CLI.

npm install -g @angular/cli

ou

yarn global add @angular/cli

dependendo do gerenciador de pacotes que você usa.

Vamos instalar o Angular 4 usando o Angular-CLI.

ng new Angular4-boilerplate

cd Angular4-boilerplate Estamos prontos para o Angular 4. É um método bastante fácil e direto.😌

Todo o código reside na pasta src.

A pasta mocks é para dados simulados que são usados ​​para fins de teste.

A pasta model contém a classe e a interface usadas no componente.

O arquivo package.json é para todas as listas de dependências.

node_modules contém uma lista de pacotes configuráveis.
