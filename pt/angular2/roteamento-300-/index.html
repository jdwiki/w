Existem mais alguns truques que podemos fazer com o roteador (como restringir o acesso), mas eles podem ser abordados em um tutorial separado.

Se você precisar de uma nova rota, basta modificar app.routes.ts e seguir os seguintes passos:

  1. Importe o Componente
  2. Adicione ao array routes. Certifique-se de incluir um novo path e component.

Controlando o acesso de ou para uma rota #

O roteador Angular padrão permite a navegação de e para qualquer rota incondicionalmente. Isso nem sempre é o comportamento desejado.

Em um cenário em que um usuário pode ter permissão condicional para navegar de ou para uma rota, um Route Guard pode ser usado para restringir esse comportamento.

Se o seu cenário se enquadrar em um dos seguintes, considere usar um Route Guard,

  • O usuário precisa ser autenticado para navegar até o componente de destino.
  • O usuário deve estar autorizado a navegar para o componente de destino.
  • O componente requer solicitação assíncrona antes da inicialização.
  • Componente requer entrada do usuário antes de sair dele.

Como funcionam os guardas de rota #

Os Route Guards funcionam retornando um valor booleano para controlar o comportamento da navegação do roteador. Se true for retornado, o roteador continuará com a navegação para o componente de destino. Se false for retornado, o roteador negará a navegação ao componente de destino.

Interfaces de proteção de rota #

O roteador suporta várias interfaces de guarda:

  • CanActivate: ocorre entre a navegação da rota.
  • CanActivateChild: ocorre entre a navegação da rota para uma rota filha.
  • CanDeactivate: ocorre ao sair da rota atual.
  • CanLoad: ocorre entre a navegação de rota para um módulo de recurso carregado de forma assíncrona.
  • Resolve: usado para realizar a recuperação de dados antes da ativação da rota.

Essas interfaces podem ser implementadas em sua guarda para conceder ou remover acesso a determinados processos da navegação.

Guardas de rota síncrona vs. assíncrona #

Os Route Guards permitem operações síncronas e assíncronas para controlar condicionalmente a navegação.

Guarda de rota síncrona #

Um guarda de rota síncrona retorna um booleano, por exemplo, calculando um resultado imediato, para controlar condicionalmente a navegação.

import { Injectable }     from '@angular/core';
import { CanActivate }    from '@angular/router';

@Injectable()
export class SynchronousGuard implements CanActivate {
  canActivate() {
    console.log('SynchronousGuard#canActivate called');
    return true;
  }
}

Proteção de rota assíncrona #

Para um comportamento mais complexo, um protetor de rota pode bloquear a navegação de forma assíncrona. Um protetor de rota assíncrono pode retornar um Observable ou Promise.

Isso é útil para situações como aguardar a entrada do usuário para responder a uma pergunta, esperar para salvar com êxito as alterações no servidor ou esperar para receber dados obtidos de um servidor remoto.

import { Injectable }     from '@angular/core';
import { CanActivate, Router, ActivatedRouteSnapshot, RouterStateSnapshot } from '@angular/router';
import { Observable }     from 'rxjs/Rx';
import { MockAuthenticationService } from './authentication/authentication.service';

@Injectable()
export class AsynchronousGuard implements CanActivate {
    constructor(private router: Router, private auth: MockAuthenticationService) {}

    canActivate(route:ActivatedRouteSnapshot, state:RouterStateSnapshot):Observable<boolean>|boolean {
        this.auth.subscribe((authenticated) => {
            if (authenticated) {
                return true;
            }
            this.router.navigateByUrl('/login');
            return false;
        });
    }
}

Adicionar guarda à configuração de rota #

Usando Resolvedores e Guardas #

Estamos usando um guarda de nível superior em nossa configuração de rota para capturar o usuário atual no carregamento da primeira página e um resolvedor para armazenar o valor do currentUser, que é nosso usuário autenticado do backend.

Uma versão simplificada de nossa implementação tem a seguinte aparência:

Aqui está a nossa rota de nível superior:

export const routes = [
{
  path: 'Dash',
  pathMatch : 'prefix',
  component: DashCmp,
  canActivate: [AuthGuard],
  resolve: {
      currentUser: CurrentUserResolver
  },
  children: [...[
    path: '',
    component: ProfileCmp,
    resolve: {
        currentUser: currentUser
    }
  ]]
  }
];

Aqui está o nosso AuthService

import { Injectable } from '@angular/core';
import { Http, Headers, RequestOptions } from '@angular/http';
import { Observable } from 'rxjs/Rx';
import 'rxjs/add/operator/do';

@Injectable()
export class AuthService {
  constructor(http: Http) {
    this.http = http;

    let headers = new Headers({ 'Content-Type': 'application/json' });
    this.options = new RequestOptions({ headers: headers });
  }
  fetchCurrentUser() {
    return this.http.get('/api/users/me')
      .map(res => res.json())
      .do(val => this.currentUser = val);
 }
}

Aqui está o nosso AuthGuard:

import { Injectable } from '@angular/core';
import { CanActivate } from "@angular/router";
import { Observable } from 'rxjs/Rx';

import { AuthService } from '../services/AuthService';

@Injectable()
export class AuthGuard implements CanActivate {
  constructor(auth: AuthService) {
    this.auth = auth;
  }
  canActivate(route, state) {
    return Observable
      .merge(this.auth.fetchCurrentUser(), Observable.of(true))
      .filter(x => x == true);
  }
}

Aqui está nosso CurrentUserResolver:

import { Injectable } from '@angular/core';
import { Resolve } from "@angular/router";
import { Observable } from 'rxjs/Rx';

import { AuthService } from '../services/AuthService';

@Injectable()
export class CurrentUserResolver implements Resolve {
  constructor(auth: AuthService) {
    this.auth = auth;
  }
  resolve(route, state) {
    return this.auth.currentUser;
  }
}

Use o Guard na inicialização do aplicativo #

Inicialização #

Agora que as rotas estão definidas, precisamos informar nosso aplicativo sobre as rotas. Para fazer isso, inicialize o provedor que exportamos no exemplo anterior.

Encontre sua configuração de bootstrap (deve estar em main.ts, mas sua milhagem pode variar).

//main.ts

import {bootstrap} from '@angular/platform-browser-dynamic';

//Import the App component (root component)
import { App } from './app/app';

//Also import the app routes
import { APP_ROUTES_PROVIDER } from './app/app.routes';

bootstrap(App, [
  APP_ROUTES_PROVIDER,
])
.catch(err => console.error(err));

Configurando a saída do roteador #

Agora que o roteador está configurado e nosso aplicativo sabe como lidar com as rotas, precisamos mostrar os componentes reais que configuramos.

Para fazer isso, configure seu modelo/arquivo HTML para seu componente top-level (app) da seguinte forma:

//app.ts

import {Component} from '@angular/core';
import {Router, ROUTER_DIRECTIVES} from '@angular/router';

@Component({
    selector: 'app',
    templateUrl: 'app.html',
    styleUrls: ['app.css'],
    directives: [
        ROUTER_DIRECTIVES,
    ]
})
export class App {
    constructor() {
    }
}

<!-- app.html -->

<!-- All of your 'views' will go here -->
<router-outlet></router-outlet>

O elemento <router-outlet></router-outlet> mudará o conteúdo de acordo com a rota. Outro bom aspecto sobre este elemento é que ele não precisa ser o único elemento em seu HTML.

Por exemplo: digamos que você queira uma barra de ferramentas em cada página que permaneça constante entre as rotas, semelhante à aparência do Stack Overflow. Você pode aninhar o <router-outlet> em elementos para que apenas algumas partes da página sejam alteradas.

Alterando rotas (usando modelos e diretivas) #

Agora que as rotas estão configuradas, precisamos de alguma maneira de realmente alterar as rotas.

Este exemplo mostrará como alterar rotas usando o modelo, mas é possível alterar rotas no TypeScript.

Aqui está um exemplo (sem vinculação):

<a routerLink="/home">Home</a>

Se o usuário clicar nesse link, ele será roteado para /home. O roteador sabe como lidar com /home, então ele exibirá o componente Home.

Aqui está um exemplo com vinculação de dados:

<a *ngFor="let link of links" [routerLink]="link">{{link}}</a>

O que exigiria que um array chamado links existisse, então adicione isso ao app.ts:

public links[] = [
    'home',
    'login'
]

Isso irá percorrer o array e adicionar um elemento <a> com a diretiva routerLink = o valor do elemento atual no array, criando isto:

 <a routerLink="home">home</a>
 <a routerLink="login">login</a>

Isso é particularmente útil se você tiver muitos links ou talvez os links precisem ser alterados constantemente. Deixamos o Angular lidar com o trabalho ocupado de adicionar links apenas alimentando-o com as informações necessárias.

No momento, links[] é estático, mas é possível alimentá-lo com dados de outra fonte.

Configurando as Rotas #

OBSERVAÇÃO: Este exemplo é baseado na versão 3.0.0.-beta.2 do @angular/router. No momento da redação deste artigo, esta é a versão mais recente do roteador.

Para usar o roteador, defina rotas em um novo arquivo TypeScript como

//app.routes.ts

import {provideRouter} from '@angular/router';

import {Home} from './routes/home/home';
import {Profile} from './routes/profile/profile';

export const routes = [
    {path: '', redirectTo: 'home'},
    {path: 'home', component: Home},
    {path: 'login', component: Login},
];

export const APP_ROUTES_PROVIDER = provideRouter(routes);

Na primeira linha, importamos provideRouter para que possamos informar a nossa aplicação quais são as rotas durante a fase de bootstrap.

Home e Profile são apenas dois componentes como exemplo. Você precisará importar cada Component necessário como uma rota.

Em seguida, exporte a matriz de rotas.

path: O caminho para o componente. VOCÊ NÃO PRECISA USAR ‘/……..’ Angular fará isso automaticamente

component: O componente a ser carregado quando a rota é acessada

redirectTo: Opcional. Se você precisar redirecionar automaticamente um usuário quando ele acessar uma rota específica, forneça isso.

Por fim, exportamos o roteador configurado. provideRouter retornará um provedor que podemos aumentar o rap para que nossa aplicação saiba como lidar com cada rota.
