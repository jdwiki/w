Primeira configuração com exemplo de LED piscando usando SW4STM32 e biblioteca HAL #

(Observação: Existem muitos IDEs, cadeias de ferramentas e bibliotecas que estão prontas para uso com o STM32. A configuração a seguir requer um esforço mínimo para que funcione, mas é apenas uma das muitas. Sinta-se à vontade para explorar outras , não é objetivo deste exemplo forçar alguém a usar as ferramentas que serão usadas aqui.)

Instalação do IDE #

System Workbench for STM32: IDE gratuito no Windows, Linux e OS X. Foi construído por AC6 e está disponível para download após registro no site da Comunidade OpenSTM32.

O próprio IDE é baseado no Eclipse, mas vem com alguns extras para desenvolvimento em STM32 como:

  • Cadeia de ferramentas Ac6 STM32 MCU GCC
  • OpenOCD e GDB (arm-none-eabi-gdb) com configurações de depuração geradas automaticamente dependendo da placa de destino
  • Opções integradas para programar ou apagar chip

Para começar com o STM32 antes de criar sua própria placa, é recomendável experimentar um [Discovery][5], um [Nucleo][5] ou um [Eval board][5], que vem com um SWD integrado ( Serial Wire Debug) programador/depurador chamado ST-Link.

Criando um projeto #

Este exemplo usará um [kit STM32F4 Discovery][6], que possui um microcontrolador STM32F407VG. (Qualquer outra placa também pode ser usada.)

  1. Abra o SW4STM32 e crie um novo projeto C: File → Novo → Projeto C

  2. Dê a ele um nome como “STM32F4_Discovery-Blinky” e, na lista Tipo de projeto, escolha Projeto Executável/Ac6 STM32 MCU. Por padrão, a única cadeia de ferramentas disponível é Ac6 STM32 MCU GCC. Clique em Avançar.

    [![enter image description here][7]][7]

  3. A próxima etapa é Configurações de depuração/liberação, que pode ser ignorada agora clicando em Avançar.

  4. Seleção da placa. Placas existentes podem ser selecionadas como neste exemplo o STM32F4 Discovery ou novas placas personalizadas podem ser adicionadas.

    [![enter image description here][8]][8]

  5. O próximo passo é Configuração do Firmware do Projeto. Escolha entre No firmware, Standard Peripheral Library (SPL) ou Hardware Abstraction Layer (HAL). Questiona-se qual é mais adequado para o desenvolvimento, mas esta questão está fora do escopo neste exemplo. Este exemplo usará a biblioteca HAL, pois é atualmente suportada pela ST Microelectronics. A ferramenta de software adicional disponível para HAL é [STM32CubeMX][9], que é um gerador de código de inicialização. Além disso, vários aplicativos de exemplo estão disponíveis pelos pacotes de software [STM32CubeFx][10] ou STM32CubeLx. Baixe o firmware de destino se estiver faltando e é recomendado selecionar as opções “Adicionar drivers de baixo nível no projeto” e as opções “Como fontes no aplicativo”. Por fim, clique em Concluir.

    [![enter image description here][11]][11]

Aplicação do LED piscante #

Como este projeto foi criado com um STM32F4 Discovery, já existem várias funções prontas para uso na pasta do projeto /STM32F4_Discovery-Blinky/Utilities/STM32F4-Discovery/ que podem ser usadas para interfacear os periféricos do kit Discovery (acelerômetro, áudio, LEDs, botão). Neste exemplo, as funções void BSP_LED_Init(Led_TypeDef Led) e void BSP_LED_Toggle(Led_TypeDef Led) serão usadas a partir do arquivo stm32f4_discovery.c para piscar o LED verde, que é LED4. Para decidir qual LED é qual, use os esquemas do [Discovery kit][12].

[![digite a descrição da imagem aqui][13]][13]

Os nomes reais de pinos e portas já estão ocultos por alguns #define e enum, use Ctrl + Click para rastreá-los.

  1. Dentro do main, chame a função HAL_Init() que reseta todos os periféricos, inicializa a interface Flash e o Systick. (Systick será usado para gerar atraso para piscar.)
  2. O relógio do sistema deve ser configurado. Isso pode ser feito usando o [recurso de configuração do relógio STM32CubeMX][14] ou pelo manual de referência. Neste exemplo, o relógio do sistema é alimentado pelo PLL interno (Phase Locked Loop), que é gerado por um oscilador de cristal externo de 8 MHz (HSE). Os prescalers foram configurados para atingir a frequência máxima disponível, que é de 168 MHz no caso do F4 Discovery.
  3. Inicialização dos periféricos, neste caso um pino GPIO.
  4. Dentro de um loop infinito, chame a alternância de LED e a função HAL_Delay(). HAL_Delay() usa o Systick e gera um atraso em milissegundos.

O código inteiro é o seguinte:

#include "stm32f4xx.h"
#include "stm32f4_discovery.h"
            
void SystemClock_Config(void);

int main(void)
{
    /* Reset of all peripherals, Initializes the Flash interface and the Systick. */
    HAL_Init();

    /* Configure the system clock */
    SystemClock_Config();

    /* Initialize one of the LED GPIO pin */
    BSP_LED_Init(LED4);

    while(1)
    {
        BSP_LED_Toggle(LED4);
        HAL_Delay(1000);       // in miliseconds
    }
}

/**
 * @brief  System Clock Configuration
 *         The system Clock is configured as follow :
 *            System Clock source            = PLL (HSE)
 *            SYSCLK(Hz)                     = 168000000
 *            HCLK(Hz)                       = 168000000
 *            AHB Prescaler                  = 1
 *            APB1 Prescaler                 = 4
 *            APB2 Prescaler                 = 2
 *            HSE Frequency(Hz)              = HSE_VALUE
 *            PLL_M                          = (HSE_VALUE/1000000u)
 *            PLL_N                          = 336
 *            PLL_P                          = 2
 *            PLL_Q                          = 7
 *            VDD(V)                         = 3.3
 *            Main regulator output voltage  = Scale1 mode
 *            Flash Latency(WS)              = 5
 * @param  None
 * @retval None
 */
void SystemClock_Config(void)
{
  RCC_ClkInitTypeDef RCC_ClkInitStruct;
  RCC_OscInitTypeDef RCC_OscInitStruct;

  // Enable Power Control clock
  __PWR_CLK_ENABLE();

  // The voltage scaling allows optimizing the power consumption when the
  // device is clocked below the maximum system frequency, to update the
  // voltage scaling value regarding system frequency refer to product
  // datasheet.
  __HAL_PWR_VOLTAGESCALING_CONFIG(PWR_REGULATOR_VOLTAGE_SCALE1);

  // Enable HSE Oscillator and activate PLL with HSE as source
  RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSE;
  RCC_OscInitStruct.HSEState = RCC_HSE_ON;
  RCC_OscInitStruct.PLL.PLLState = RCC_PLL_ON;
  RCC_OscInitStruct.PLL.PLLSource = RCC_PLLSOURCE_HSE;

  // This assumes the HSE_VALUE is a multiple of 1MHz. If this is not
  // your case, you have to recompute these PLL constants.
  RCC_OscInitStruct.PLL.PLLM = (HSE_VALUE/1000000u);
  RCC_OscInitStruct.PLL.PLLN = 336;
  RCC_OscInitStruct.PLL.PLLP = RCC_PLLP_DIV2;
  RCC_OscInitStruct.PLL.PLLQ = 7;
  HAL_RCC_OscConfig(&RCC_OscInitStruct);

  // Select PLL as system clock source and configure the HCLK, PCLK1 and PCLK2
  // clocks dividers
  RCC_ClkInitStruct.ClockType = (RCC_CLOCKTYPE_SYSCLK | RCC_CLOCKTYPE_HCLK
      | RCC_CLOCKTYPE_PCLK1 | RCC_CLOCKTYPE_PCLK2);
  RCC_ClkInitStruct.SYSCLKSource = RCC_SYSCLKSOURCE_PLLCLK;
  RCC_ClkInitStruct.AHBCLKDivider = RCC_SYSCLK_DIV1;
  RCC_ClkInitStruct.APB1CLKDivider = RCC_HCLK_DIV4;
  RCC_ClkInitStruct.APB2CLKDivider = RCC_HCLK_DIV2;
  HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_5);
}

Construa com o martelo [![digite a descrição da imagem aqui][15]][15], e baixe o aplicativo clicando com o botão direito do mouse na pasta do projeto e selecionando o Destino → Chip de programa… opção.

Outra maneira de baixar é usando debug. Para isso, clique na seta ao lado do ícone do bug [![digite a descrição da imagem aqui][16]][16] na barra de ferramentas e abra o menu Debug Configuration…. Crie uma nova configuração Ac6 STM32 Debugging e se o campo C/C++ Application estiver vazio, preencha o seguinte:

Debug\STM32F4_Discovery-Blinky.elf

Outros parâmetros de depuração, como o arquivo de configuração OpenOCD e as portas Telnet e GDB usadas, são gerados e preenchidos automaticamente pelo framework. Por fim, clique no botão Depurar.
