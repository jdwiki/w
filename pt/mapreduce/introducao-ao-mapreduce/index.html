Instalação ou Configuração #

Mapreduce é uma parte do Hadoop. Portanto, quando o Apache Hadoop (ou qualquer distribuição do Hadoop é instalada), o MR é instalado automaticamente.

MapReduce é a estrutura de processamento de dados sobre HDFS (sistema de arquivos distribuído Hadoop). Os trabalhos de MR podem ser escritos usando Java, python, Scala, R, etc.

O que mapreduce faz e como? #

Mapreduce é um modelo de programação para processar grandes quantidades de dados.

O ‘HPC’ tradicional (High Performance Computing) acelera grandes cálculos em quantidades relativamente grandes de dados criando um conjunto de computadores altamente conectados (usando coisas como rede extremamente rápida e acesso rápido a armazenamento compartilhado, memória compartilhada) para lidar com problemas de computação que geralmente exigem cálculos para ter acesso aos dados uns dos outros. Um exemplo clássico é a previsão do tempo.

O Mapreduce, por outro lado, é excelente em lidar com cálculos relativamente pequenos e independentes em enormes quantidades de dados. Para tornar isso possível, os dados são espalhados por vários computadores (devido à quantidade de dados) e o cálculo desejado é dividido em uma fase que pode ser feita em cada bit de dados de forma independente (a fase de ‘mapa’). Os resultados desses cálculos independentes são então coletados e uma segunda parte dos cálculos é feita para combinar todos esses resultados individuais no resultado final (a fase de ‘redução’).

Exemplo: contagem de votos #

Imagine que você tem uma quantidade muito grande de votos para contar e há um pouco de trabalho para contar cada voto (por exemplo, descobrir pela imagem digitalizada qual caixa foi marcada).

Nesse caso, uma implementação de mapreduce seria:

Etapa 1: ‘Divulgar’ #

Espalhe as imagens a serem processadas pelos computadores disponíveis.

Etapa 2: ‘Mapa’ #

Em cada computador, para cada imagem:

  • receber 1 das imagens copiadas para este computador como entrada
  • descobrir qual caixa foi marcada
  • saída do número (ou código ou nome) do item votado

Observe que o trabalho pode começar assim que um computador fizer com que 1 imagem funcione sobre. Não há necessidade de todos esses computadores interagirem para fazer suas trabalho, por isso não há necessidade de que eles sejam interconectados rapidamente, tenham compartilhado memória ou espaço em disco compartilhado.

Etapa 3: ‘Reunir’ #

Reúna todas essas saídas em 1 computador.

Etapa 4: ‘Reduzir’ #

Conte quantos votos para cada número (ou código ou nome) existem.

Este exemplo muito básico também destaca como outras otimizações geralmente são possíveis. Nesse caso, a etapa de redução em si pode ser feita parcialmente em cada computador e, em seguida, uma redução final pode ser feita em um computador central. Isso reduzirá a quantidade de trabalho em um computador executando a etapa de redução e limitará a quantidade de dados que precisam ser transportados pela rede.

Exemplo: Contagem de votos - otimizado (usando combinador) #

Etapa 1: ‘Divulgar’ #

Igual a antes: Espalhe as imagens a serem processadas pelos computadores disponíveis.

Etapa 2: ‘Mapa’ #

Igual a antes: Em cada computador, para cada imagem:

  • receber 1 das imagens copiadas para este computador como entrada
  • descobrir qual caixa foi marcada
  • saída do número (ou código ou nome) do item votado

Etapa 3: ‘Reunir’ localmente #

Reúna todas as saídas de 1 computador no próprio computador.

Etapa 4: ‘Reduzir’ localmente #

Conte quantos votos de cada número (ou código ou nome) existem nos resultados locais e produza essas contagens.

Etapa 5: ‘Reunir’ globalmente #

Reúna todas as saídas do local reduz em 1 computador.

Etapa 6: ‘Reduzir’ globalmente #

Some as contagens de votos feitas localmente de cada número (ou código ou nome).

Observe que na etapa 3 não é necessário esperar por todos os resultados em qualquer um dos casos abaixo:

  • se isso se tornar demais para os recursos locais do computador, como armazenamento/memória
  • se o custo do trabalho a ser refeito quando um computador quebra for considerado alto, espere todos os resultados locais
  • se a rede está agora livre para transportar resultados intermediários

a coleta local e a redução local podem ser feitas sobre os resultados produzidos até agora no computador local, e isso pode ser feito a qualquer momento.

A etapa de redução local é chamada de etapa de combinação. Esta é uma etapa opcional usada para melhorar o desempenho.
