Botão #

O Botão é provavelmente o controle mais comum não apenas em aplicativos para dispositivos móveis, mas em qualquer aplicativo que tenha uma interface do usuário. O conceito de botão tem muitos propósitos para listar aqui. De um modo geral, porém, você usará um botão para permitir que os usuários iniciem algum tipo de ação ou operação em seu aplicativo. Essa operação pode incluir desde navegação básica em seu aplicativo até o envio de dados para um serviço da Web em algum lugar da Internet.

XAML

<Button
    x:Name="MyButton"
    Text="Click Me!"
    TextColor="Red"
    BorderColor="Blue"
    VerticalOptions="Center"
    HorizontalOptions="Center"
    Clicked="Button_Clicked"/>

Code-Behind XAML

public void Button_Clicked( object sender, EventArgs args ) 
{
    MyButton.Text = "I've been clicked!";
}

Código

var button = new Button( ) 
{
    Text = "Hello, Forms !",
    VerticalOptions = LayoutOptions.CenterAndExpand,
    HorizontalOptions = LayoutOptions.CenterAndExpand,
    TextColor = Color.Red,
    BorderColor = Color.Blue,
};

button.Clicked += ( sender, args ) => 
{
    var b = (Button) sender;
    b.Text = "I've been clicked!";
};

Seletor de Data #

Muitas vezes, em aplicativos móveis, haverá uma razão para lidar com datas. Ao trabalhar com datas, você provavelmente precisará de algum tipo de entrada do usuário para selecionar uma data. Isso pode ocorrer ao trabalhar com um aplicativo de agendamento ou calendário. Nesse caso, é melhor fornecer aos usuários um controle especializado que permita que eles escolham uma data de forma interativa, em vez de exigir que os usuários digitem uma data manualmente. É aqui que o controle DatePicker é realmente útil.

XAML

<DatePicker Date="09/12/2014" Format="d" />

Código

var datePicker = new DatePicker{
Date = DateTime.Now,
Format = "d"
};

Entrada #

A tela de entrada é usada para permitir que os usuários digitem uma única linha de texto. Essa única linha de texto pode ser usada para várias finalidades, incluindo a inserção de notas básicas, credenciais, URLs e muito mais. Esta Visualização é uma Visualização multifuncional, o que significa que, se você precisar digitar texto normal ou quiser ocultar uma senha, tudo será feito por meio desse único controle.

XAML

<Entry Placeholder="Please Enter Some Text Here"
HorizontalOptions="Center"
VerticalOptions="Center"
Keyboard="Email"/>

Código

var entry = new Entry {
Placeholder = "Please Enter Some Text Here",
HorizontalOptions = LayoutOptions.Center,
VerticalOptions = LayoutOptions.Center,
Keyboard = Keyboard.Email
};

Editor #

O Editor é muito semelhante ao Entry, pois permite que os usuários insiram algum texto de formato livre. A diferença é que o Editor permite a entrada de várias linhas, enquanto a Entrada é usada apenas para entrada de linha única. A Entrada também fornece algumas propriedades a mais do que o Editor para permitir maior personalização da Visualização.

XAML

<Editor HorizontalOptions="Fill"
VerticalOptions="Fill"
 Keyboard="Chat"/>

Código

var editor = new Editor {
HorizontalOptions = LayoutOptions.Fill,
VerticalOptions = LayoutOptions.Fill,
Keyboard = Keyboard.Chat
};

Imagem #

As imagens são partes muito importantes de qualquer aplicativo. Eles oferecem a oportunidade de injetar elementos visuais adicionais, bem como a marca em seu aplicativo. Sem falar que as imagens normalmente são mais interessantes de se ver do que texto ou botões. Você pode usar uma imagem como elemento autônomo em seu aplicativo, mas um elemento Image também pode ser adicionado a outros Visualize elementos como um Button.

XAML

<Image Aspect="AspectFit" Source="http://d2g29cya9iq7ip.cloudfront.net/co
ntent/images/company/aboutus-video-bg.png?v=25072014072745"/>

Código

var image = new Image {
Aspect = Aspect.AspectFit,
Source = ImageSource.FromUri(new Uri("http://d2g29cya9iq7ip.cloudfron
t.net/content/images/company/aboutus-video-bg.png?v=25072014072745"))
};

Etiqueta #

Acredite ou não, o Label é uma das classes View mais importantes, porém subestimadas, não apenas no Xamarin.Forms, mas no desenvolvimento de UI em geral. É visto como uma linha bastante chata de texto, mas sem essa linha de texto seria muito difícil transmitir certas ideias ao usuário. Os controles de rótulo podem ser usados ​​para descrever o que o usuário deve inserir em um Editor ou Entrada ao controle. Eles podem descrever uma seção da interface do usuário e contextualizá-la. Eles podem ser usados ​​para mostrar a total em um aplicativo de calculadora. Sim, o Label é realmente o controle mais versátil em sua bolsa de ferramentas que pode nem sempre despertar muita atenção, mas é o primeiro a ser notado se não estiver lá.

XAML

<Label Text="This is some really awesome text in a Label!"
TextColor="Red"
XAlign="Center"
YAlign="Center"/>

Código

var label = new Label {
Text = "This is some really awesome text in a Label!",
TextColor = Color.Red,
XAlign = TextAlignment.Center,
YAlign = TextAlignment.Center
};
