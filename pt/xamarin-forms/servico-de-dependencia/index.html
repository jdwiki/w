Ao usar DependencyService você normalmente precisa de 3 partes:

  • Interface - Isso define as funções que você deseja abstrair.
  • Implementação da plataforma - Uma classe dentro de cada projeto específico da plataforma que implementa a interface definida anteriormente.
  • Registro - Cada classe de implementação específica da plataforma deve ser registrada com o DependencyService através de um atributo de metadados. Isso permite que o DependencyService encontre sua implementação em tempo de execução.

Ao usar o DependencyService, você deve fornecer uma implementação para cada plataforma de destino. Quando uma implementação não é fornecida, o aplicativo falhará em tempo de execução.

Implementação do Android #

A implementação específica do Android é um pouco mais complexa porque força você a herdar de um Java.Lang.Object nativo e força você a implementar a interface IOnInitListener. O Android exige que você forneça um contexto Android válido para muitos dos métodos do SDK que ele expõe. Xamarin.Forms expõe um objeto Forms.Context que fornece um contexto Android que você pode usar nesses casos.

using Android.Speech.Tts;
using Xamarin.Forms;
using System.Collections.Generic;
using DependencyServiceSample.Droid;

public class TextToSpeechAndroid : Java.Lang.Object, ITextToSpeech, TextToSpeech.IOnInitListener
{
    TextToSpeech _speaker;

    public TextToSpeechAndroid () {}

    public void Speak (string whatToSay)
    {
        var ctx = Forms.Context;

        if (_speaker == null) 
        {
            _speaker = new TextToSpeech (ctx, this);
        } 
        else 
        {
            var p = new Dictionary<string,string> ();
            _speaker.Speak (whatToSay, QueueMode.Flush, p);
        }
    }

    #region IOnInitListener implementation
    
    public void OnInit (OperationResult status)
    {
        if (status.Equals (OperationResult.Success)) 
        {
            var p = new Dictionary<string,string> ();
            _speaker.Speak (toSpeak, QueueMode.Flush, p);
        }
    }
    
    #endregion
}

Quando você criou sua classe, você precisa habilitar o DependencyService para descobri-la em tempo de execução. Isso é feito adicionando um atributo [assembly] acima da definição de classe e fora de qualquer definição de namespace.

using Android.Speech.Tts;
using Xamarin.Forms;
using System.Collections.Generic;
using DependencyServiceSample.Droid;


[assembly: Xamarin.Forms.Dependency (typeof (TextToSpeechAndroid))]
namespace DependencyServiceSample.Droid {
    ...

Este atributo registra a classe com o DependencyService para que possa ser usado quando uma instância da interface ITextToSpeech for necessária.

Interface #

A interface define o comportamento que você deseja expor através do DependencyService. Um exemplo de uso de um DependencyService é um serviço Text-To-Speech. Atualmente, não há abstração para esse recurso no Xamarin.Forms, portanto, você precisa criar o seu próprio. Comece definindo uma interface para o comportamento:

public interface ITextToSpeech
{
       void Speak (string whatToSay);
}

Como definimos nossa interface, podemos codificar com base em nosso código compartilhado.

Observação: as classes que implementam a interface precisam ter um construtor sem parâmetros para trabalhar com o DependencyService.

Implementação iOS #

A interface que você definiu precisa ser implementada em todas as plataformas de destino. Para iOS isso é feito através do framework AVFoundation. A seguinte implementação da interface ITextToSpeech lida com a fala de um determinado texto em inglês.

using AVFoundation;

public class TextToSpeechiOS : ITextToSpeech
{
    public TextToSpeechiOS () {}

    public void Speak (string whatToSay)
    {
        var speechSynthesizer = new AVSpeechSynthesizer ();

        var speechUtterance = new AVSpeechUtterance (whatToSay) {
            Rate = AVSpeechUtterance.MaximumSpeechRate/4,
            Voice = AVSpeechSynthesisVoice.FromLanguage ("en-US"),
            Volume = 0.5f,
            PitchMultiplier = 1.0f
        };

        speechSynthesizer.SpeakUtterance (speechUtterance);
    }
}

Quando você criou sua classe, você precisa habilitar o DependencyService para descobri-la em tempo de execução. Isso é feito adicionando um atributo [assembly] acima da definição de classe e fora de qualquer definição de namespace.

using AVFoundation;
using DependencyServiceSample.iOS;

[assembly: Xamarin.Forms.Dependency (typeof (TextToSpeechiOS))]
namespace DependencyServiceSample.iOS {
    public class TextToSpeechiOS : ITextToSpeech
...

Este atributo registra a classe com o DependencyService para que possa ser usado quando uma instância da interface ITextToSpeech for necessária.

Código compartilhado #

Depois de criar e registrar suas classes específicas da plataforma, você pode começar a conectá-las ao seu código compartilhado. A página a seguir contém um botão que aciona a funcionalidade de conversão de texto em fala usando uma frase predefinida. Ele usa DependencyService para recuperar uma implementação específica da plataforma de ITextToSpeech em tempo de execução usando os SDKs nativos.

public MainPage ()
{
    var speakButton = new Button {
        Text = "Talk to me baby!",
        VerticalOptions = LayoutOptions.CenterAndExpand,
        HorizontalOptions = LayoutOptions.CenterAndExpand,
    };
    
    speakButton.Clicked += (sender, e) => {
        DependencyService.Get<ITextToSpeech>().Speak("Xamarin Forms likes eating cake by the ocean.");
    };
    
    Content = speakButton;
}

Ao executar este aplicativo em um dispositivo iOS ou Android e tocar no botão, você ouvirá o aplicativo falar a frase fornecida.
