Se você não quiser que seu código seja interrompido quando nenhuma implementação for encontrada, verifique primeiro o DependencyService se ele tiver uma implementação disponível.

Você pode fazer isso por uma simples verificação se não for null.

var speaker = DependencyService.Get<ITextToSpeech>();

if (speaker != null)
{
    speaker.Speak("Ready for action!");
}

ou, se seu IDE suportar C# 6, com operador condicional nulo:

var speaker = DependencyService.Get<ITextToSpeech>();

speaker?.Speak("Ready for action!");

Se você não fizer isso e nenhuma implementação for encontrada em tempo de execução, seu código gerará uma exceção.

Implementando conversão de texto em fala #

Um bom exemplo de um recurso que solicita código específico da plataforma é quando você deseja implementar a conversão de texto em fala (tts). Este exemplo pressupõe que você esteja trabalhando com código compartilhado em uma biblioteca PCL.

Uma visão geral esquemática de nossa solução seria semelhante à imagem abaixo.

Em nosso código compartilhado definimos uma interface que é registrada com o DependencyService. É aqui que faremos nossos apelos. Defina uma interface como abaixo.

public interface ITextToSpeech
{
    void Speak (string text);
}

Agora em cada plataforma específica, precisamos criar uma implementação dessa interface. Vamos começar com a implementação do iOS.

Implementação iOS #

using AVFoundation;

public class TextToSpeechImplementation : ITextToSpeech
{
    public TextToSpeechImplementation () {}

    public void Speak (string text)
    {
        var speechSynthesizer = new AVSpeechSynthesizer ();

        var speechUtterance = new AVSpeechUtterance (text) {
            Rate = AVSpeechUtterance.MaximumSpeechRate/4,
            Voice = AVSpeechSynthesisVoice.FromLanguage ("en-US"),
            Volume = 0.5f,
            PitchMultiplier = 1.0f
        };

        speechSynthesizer.SpeakUtterance (speechUtterance);
    }
}

No exemplo de código acima, você percebe que existe um código específico para iOS. Como tipos como AVSpeechSynthesizer. Estes não funcionariam em código compartilhado.

Para registrar esta implementação com o DependencyService do Xamarin, adicione este atributo acima da declaração de namespace.

using AVFoundation;
using DependencyServiceSample.iOS;//enables registration outside of namespace

[assembly: Xamarin.Forms.Dependency (typeof (TextToSpeechImplementation))]
namespace DependencyServiceSample.iOS {
    public class TextToSpeechImplementation : ITextToSpeech
//... Rest of code

Agora, quando você faz uma chamada como essa em seu código compartilhado, a implementação correta para a plataforma na qual você está executando seu aplicativo é injetada.

DependencyService.Get<ITextToSpeech>(). Mais sobre isso mais tarde.

Implementação Android #

A implementação do Android deste código ficaria como abaixo.

using Android.Speech.Tts;
using Xamarin.Forms;
using System.Collections.Generic;
using DependencyServiceSample.Droid;

public class TextToSpeechImplementation : Java.Lang.Object, ITextToSpeech, TextToSpeech.IOnInitListener
{
    TextToSpeech speaker;
    string toSpeak;

    public TextToSpeechImplementation () {}

    public void Speak (string text)
    {
        var ctx = Forms.Context; // useful for many Android SDK features
        toSpeak = text;
        if (speaker == null) {
            speaker = new TextToSpeech (ctx, this);
        } else {
            var p = new Dictionary<string,string> ();
            speaker.Speak (toSpeak, QueueMode.Flush, p);
        }
    }

    #region IOnInitListener implementation
    public void OnInit (OperationResult status)
    {
        if (status.Equals (OperationResult.Success)) {
            var p = new Dictionary<string,string> ();
            speaker.Speak (toSpeak, QueueMode.Flush, p);
        }
    }
    #endregion
}

Novamente não se esqueça de registrá-lo com o DependencyService.

using Android.Speech.Tts;
using Xamarin.Forms;
using System.Collections.Generic;
using DependencyServiceSample.Droid;


[assembly: Xamarin.Forms.Dependency (typeof (TextToSpeechImplementation))]
namespace DependencyServiceSample.Droid{
    //... Rest of code

Implementação do Windows Phone #

Finalmente, para Windows Phone este código pode ser usado.

public class TextToSpeechImplementation : ITextToSpeech
{
    public TextToSpeechImplementation() {}

    public async void Speak(string text)
    {
        MediaElement mediaElement = new MediaElement();

        var synth = new Windows.Media.SpeechSynthesis.SpeechSynthesizer();

        SpeechSynthesisStream stream = await synth.SynthesizeTextToStreamAsync("Hello World");

        mediaElement.SetSource(stream, stream.ContentType);
        mediaElement.Play();
        await synth.SynthesizeTextToStreamAsync(text);
    }
}

E mais uma vez não se esqueça de registrá-lo.

using Windows.Media.SpeechSynthesis;
using Windows.UI.Xaml.Controls;
using DependencyServiceSample.WinPhone;//enables registration outside of namespace

[assembly: Xamarin.Forms.Dependency (typeof (TextToSpeechImplementation))]
namespace DependencyServiceSample.WinPhone{
    //... Rest of code

Implementação em código compartilhado #

Agora tudo está no lugar para fazê-lo funcionar! Por fim, em seu código compartilhado, agora você pode chamar essa função usando a interface. Em tempo de execução, será injetada a implementação que corresponde à plataforma atual em que está sendo executada.

Neste código, você verá uma página que pode estar em um projeto Xamarin Forms. Ele cria um botão que invoca o método Speak() usando o DependencyService.

public MainPage ()
{
    var speak = new Button {
        Text = "Hello, Forms !",
        VerticalOptions = LayoutOptions.CenterAndExpand,
        HorizontalOptions = LayoutOptions.CenterAndExpand,
    };
    speak.Clicked += (sender, e) => {
        DependencyService.Get<ITextToSpeech>().Speak("Hello from Xamarin Forms");
    };
    Content = speak;
}

O resultado será que quando o aplicativo for executado e o botão for clicado, o texto fornecido será falado.

Tudo isso sem ter que fazer coisas difíceis como dicas de compilador e tal. Agora você tem uma maneira uniforme de acessar a funcionalidade específica da plataforma por meio de código independente de plataforma.

Obtendo os números de versão do sistema operacional do aplicativo e do dispositivo - Android e iOS - PCL #

O exemplo abaixo coletará o número da versão do SO do dispositivo e a versão do aplicativo (definida nas propriedades de cada projeto) inserida em Nome da versão no Android e Versão no iOS.

Primeiro faça uma interface em seu projeto PCL:

public interface INativeHelper {
    /// <summary>
    /// On iOS, gets the <c>CFBundleVersion</c> number and on Android, gets the <c>PackageInfo</c>'s <c>VersionName</c>, both of which are specified in their respective project properties.
    /// </summary>
    /// <returns><c>string</c>, containing the build number.</returns>
    string GetAppVersion();

    /// <summary>
    /// On iOS, gets the <c>UIDevice.CurrentDevice.SystemVersion</c> number and on Android, gets the <c>Build.VERSION.Release</c>.
    /// </summary>
    /// <returns><c>string</c>, containing the OS version number.</returns>
    string GetOsVersion();
}

Agora implementamos a interface nos projetos Android e iOS.

Android:

[assembly: Dependency(typeof(NativeHelper_Android))]

namespace YourNamespace.Droid{
    public class NativeHelper_Android : INativeHelper {
        
        /// <summary>
        /// See interface summary.
        /// </summary>
        public string GetAppVersion() {
            Context context = Forms.Context;
            return context.PackageManager.GetPackageInfo(context.PackageName, 0).VersionName;
        }

        /// <summary>
        /// See interface summary.
        /// </summary>
        public string GetOsVersion() { return Build.VERSION.Release; }
    }
}

iOS:

[assembly: Dependency(typeof(NativeHelper_iOS))]

namespace YourNamespace.iOS {
    public class NativeHelper_iOS : INativeHelper {
        
        /// <summary>
        /// See interface summary.
        /// </summary>
        public string GetAppVersion() { return Foundation.NSBundle.MainBundle.InfoDictionary[new Foundation.NSString("CFBundleVersion")].ToString(); }

        /// <summary>
        /// See interface summary.
        /// </summary>
        public string GetOsVersion() { return UIDevice.CurrentDevice.SystemVersion; }
    }
}

Agora para usar o código em um método:

public string GetOsAndAppVersion {
    INativeHelper helper = DependencyService.Get<INativeHelper>();

    if(helper != null) {
        string osVersion  = helper.GetOsVersion();
        string appVersion = helper.GetBuildNumber()
    }
}
