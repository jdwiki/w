Layout de página em diferentes dispositivos - CSS #

Se seu aplicativo for ser executado em dispositivos diferentes, ele precisará renderizar em diferentes ViewPorts, com base no tamanho do dispositivo. Você pode lidar com isso de duas maneiras: com regras javascript ou estilos de mídia CSS. Se você estiver usando uma biblioteca MVC ou MVVM, como Angular ou Ember (ou Blaze, nesse caso) e tiver como alvo apenas um único dispositivo ou plataforma de hardware, talvez seja necessário repensar seu modelo MVC, pois diferentes ViewPorts de hardware são apresentado ao seu aplicativo.

// desktop 
@media only screen and (min-width: 960px) {
}

// landscape orientation
@media only screen and (min-width: 768px) {
}

// portrait orientation
@media only screen and (min-width: 480px) {
}

Você precisará descobrir se deseja quebrar os estilos em 768px (modo retrato) ou em 1024 pixels (paisagem). Isso supondo que seu dispositivo móvel de destino seja o iPad, que usa uma proporção de 3:4. Caso contrário, você precisará descobrir as proporções dos dispositivos que deseja segmentar e descobrir os níveis de limite a partir daí.

Janelas de tamanho fixo #

Se você estiver projetando layouts com telas de tamanho fixo para diferentes dispositivos móveis, convém espelhar esse design ao executar seu aplicativo em um desktop. O método a seguir corrige o tamanho da janela FORA do PhoneGap, fornecendo uma janela de tamanho fixo na área de trabalho. Às vezes, é mais fácil gerenciar as expectativas do usuário e o design da interface do usuário limitando as opções!

// create a window of a specific size
var w=window.open('','', 'width=100,height=100');
w.resizeTo(500,500);

// prevent window resize
var size = [window.width,window.height];  //public variable
$(window).resize(function(){
    window.resizeTo(size[0],size[1]);
});

Cache off-line #

Para que tudo isso funcione, você provavelmente precisará de suporte offline, o que significa armazenar em cache os dados do aplicativo e os dados do usuário.

meteor add appcache
meteor add grounddb

Desativar o salto de rolagem #

Em aplicativos de desktop, você pode desabilitar o salto de rolagem, para dar ao seu aplicativo uma sensação mais nativa. Você pode fazer isso com javascript, desativando como o navegador controla o DOM:

// prevent scrolling on the whole page
// this is not meteorish; TODO: translate to meteor-centric code
document.ontouchmove = function(e) {e.preventDefault()};

// prevent scrolling on specific elements
// this is not meteorish; TODO: translate to meteor-centric code
scrollableDiv.ontouchmove = function(e) {e.stopPropagation()};

Como alternativa, você pode usar css e os estilos de estouro e rolagem.

#appBody {
  overflow: hidden;
}

#contentContainer {
  .content-scrollable {
    overflow-y: auto;
    -webkit-overflow-scrolling: touch;
  }
}

O modelo de objeto necessário para que o acima funcione é algo assim:

<div id="appBody">
  <div id="contentContainer">
    <div class="content-scrollable">
      <!-- content -->
    </div>
  </div>
</div>

Multitoque e gestos #

Os dispositivos móveis geralmente não têm teclados, portanto, você precisará adicionar alguns controladores hápticos ao seu aplicativo. Os dois pacotes populares que as pessoas parecem estar usando é FastClick e Hammer. A instalação é fácil.

meteor add fastclick
meteor add hammer:hammer

O FastClick não requer quase nenhuma configuração, enquanto o Hammer requer um pouco de trabalho para conectar. O exemplo conônico do aplicativo Todos se parece com isso:

Template.appBody.onRendered(function() {
  if (Meteor.isCordova) {
    // set up a swipe left / right handler
    this.hammer = new Hammer(this.find('#appBody'));
    this.hammer.on('swipeleft swiperight', function(event) {
      if (event.gesture.direction === 'right') {
        Session.set(MENU_KEY, true);
      } else if (event.gesture.direction === 'left') {
        Session.set(MENU_KEY, false);
      }
    });
  }
});

Pipeline de Arquitetura de Meteor Cordova #

Agora é hora de ver a documentação do Meteor Cordova Phonegap Integration.

Desde que essa documentação foi escrita, XCode e Yosemite foram lançados, o que causou alguns problemas na instalação. Aqui estão as etapas pelas quais tivemos que passar para compilar o Meteor em um dispositivo iOS.

  • Atualize para Yosemite.
  • Excluir XCode (arraste da pasta Aplicativos para a Lixeira)
  • Instale o XCode 6.1 da loja de aplicativos.
  • Concordar com vários termos e condições.
# 5.  clone and rebuild the ios-sim locally
#     (this step will not be needed in future releases)
git clone https://github.com/phonegap/ios-sim.git
cd ios-sim
rake build

# 6.  make sure we can update the .meteor/packages locations
#     (this step will not be needed in future releases)
sudo chmod -R 777 ~/.meteor/packages

# 7.  copy the new build into Meteor locations
#     (this step will not be needed in future releases)
for i in `find ~/.meteor/packages/meteor-tool/ -name ios-sim -type f`; do
  cp -R ./build/Release/ios-sim "$i"
done

# 8.  install the ios platform to your app
cd myapp
meteor list-platforms
meteor add-platform ios
meteor list-platforms

# 9.  and that there aren't dead processes
ps -ax 
kill -9 <pid>
# /Users/abigailwatson/.meteor/packages/meteor-tool/.1.0.35.wql4jh++os.osx.x86_64+web.browser+web.cordova/meteor-tool-os.osx.x86_64/dev_bundle/mongodb/bin/mongod
# tail -f /Users/abigailwatson/Code/Medstar/dart/webapp/.meteor/local/cordova-build/platforms/ios/cordova/console.log

# 10.  make sure there are correct permissions on the application (important!)
sudo chmod -R 777 .meteor/local/

# 11.  run app
meteor run ios

# 12.  if that doesn't work, clear the directory
sudo rm -rf .meteor/local

# 13a.  run meteor again to create the default browser build
meteor

# 13b.  run it a second time so bootstrap and other packages get downloaded into the browser build
ctrl-x
meteor

# 14.  then run the ios version
ctrl-x
meteor run ios

O XCode deve ser iniciado durante o processo. Selecione seu simulador e pressione o botão ‘Play’.

Desenvolvimento IOS #

  • Registre sua conta de desenvolvedor da Apple
  • Registre um ID de aplicativo para seu aplicativo
  • Registre o UUID de seus dispositivos de teste
  • Gerar um perfil de provisionamento de desenvolvimento de aplicativos iOS
  • Gere um CertificateSigningRequest do KeychainAccess
  • Envie CertificateSigningRequest para https://developer.apple.com/account/ios/profile/profileCreate.action
  • Faça o download e clique duas vezes no certificado para importar para o Keychain
  • Vá para XCode > Preferências > Contas e registre sua conta de desenvolvedor da Apple

Teste de dispositivo IOS #

  • Certifique-se de que sua estação de trabalho de desenvolvimento e o iPhone estejam conectados à mesma rede WiFi. Tethering, hotspots e outras redes ad-hoc não funcionarão.
  • Execute sudo meteor run ios-device
  • Implante no seu dispositivo!

Configure seu projeto Cordova (config.xml) #

O Meteor lê um arquivo mobile-config.js na raiz do diretório do seu aplicativo durante a compilação e usa as configurações especificadas para gerar o config.xml do Cordova.

Project_folder
 ├── /.meteor
 └── mobile-config.js

A maioria das configurações pode ser alcançada com mobile-config.js (metadados do aplicativo, preferências, ícones e telas de inicialização, bem como parâmetros de instalação de plugins Cordova).

App.info({
  id: 'com.example.matt.uber',
  name: 'über',
  description: 'Get über power in one button click',
  author: 'Matt Development Group',
  email: 'contact@example.com',
  website: 'http://example.com'
});

// Set up resources such as icons and launch screens.
App.icons({
  'iphone': 'icons/icon-60.png',
  'iphone_2x': 'icons/icon-60@2x.png',
  // ... more screen sizes and platforms ...
});

App.launchScreens({
  'iphone': 'splash/Default~iphone.png',
  'iphone_2x': 'splash/Default@2x~iphone.png',
  // ... more screen sizes and platforms ...
});

// Set PhoneGap/Cordova preferences
App.setPreference('BackgroundColor', '0xff0000ff');
App.setPreference('HideKeyboardFormAccessoryBar', true);
App.setPreference('Orientation', 'default');
App.setPreference('Orientation', 'all', 'ios');

// Pass preferences for a particular PhoneGap/Cordova plugin
App.configurePlugin('com.phonegap.plugins.facebookconnect', {
  APP_ID: '1234567890',
  API_KEY: 'supersecretapikey'
});

Não edite manualmente o arquivo /.meteor/local/cordova-build/config.xml, pois ele será regenerado a cada meteor run ios/android ou meteor build, portanto você perderá todas as suas modificações.

Página de referência: Guia Meteor > Build > Mobile > Configurando seu aplicativo

Detectando o evento deviceready #

Obviamente, a melhor maneira de detectar dispositivos móveis é o hardware notificá-lo diretamente. O Cordova PhoneGap expõe um evento ‘deviceready’, ao qual você pode adicionar um ouvinte de eventos.

 document.addEventListener('deviceready', function(){
  Session.set('deviceready', true);
 }, false);
