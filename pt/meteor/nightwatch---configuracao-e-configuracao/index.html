A Nightwatch fornece testes de aceitação e ponta a ponta para aplicativos Meteor desde a v0.5 dias e gerenciou migrações de PHP para Spark, Blaze e React; e todas as principais plataformas de Integração Contínua. Para obter ajuda adicional, consulte:

Documentação da API Nightwatch Nightwatch.js Grupo do Google

Configuração #

A principal razão pela qual o Nightwatch é tão poderoso, é por causa de seu excelente arquivo de configuração. Ao contrário da maioria das outras estruturas de teste, o Nightwatch é totalmente configurável e personalizável para diferentes ambientes e pilhas de tecnologia.

.meteor/nightwatch.json

O arquivo de configuração a seguir é para Meteor v1.3 e posterior, e suporta dois ambientes… um ambiente default que inicia o navegador chromedriver, e um ambiente phantom que executa os testes em um ambiente headless.

{
  "nightwatch": {
    "version": "0.9.8"
  },
  "src_folders": [
    "./tests/nightwatch/walkthroughs"
  ],
  "custom_commands_path": [
    "./tests/nightwatch/commands"
  ],
  "custom_assertions_path": [
    "./tests/nightwatch/assertions"
  ],
  "output_folder": "./tests/nightwatch/reports",
  "page_objects_path": "./tests/nightwatch/pages",
  "globals_path": "./tests/nightwatch/globals.json",
  "selenium": {
    "start_process": true,
    "server_path": "./node_modules/starrynight/node_modules/selenium-server-standalone-jar/jar/selenium-server-standalone-2.45.0.jar",
    "log_path": "tests/nightwatch/logs",
    "host": "127.0.0.1",
    "port": 4444,
    "cli_args": {
      "webdriver.chrome.driver": "./node_modules/starrynight/node_modules/chromedriver/bin/chromedriver"
    }
  },
  "test_settings": {
    "default": {
      "launch_url": "http://localhost:5000",
      "selenium_host": "127.0.0.1",
      "selenium_port": 4444,
      "pathname": "/wd/hub",
      "silent": true,
      "disable_colors": false,
      "firefox_profile": false,
      "ie_driver": "",
      "screenshots": {
        "enabled": false,
        "path": "./tests/nightwatch/screenshots"
      },
      "desiredCapabilities": {
        "browserName": "chrome",
        "javascriptEnabled": true,
        "acceptSslCerts": true,
        "loggingPrefs": {
          "browser": "ALL"
        }
      },
      "exclude": "./tests/nightwatch/unittests/*",
      "persist_globals": true,
      "detailed_output": false
    },
    "phantom": {
      "desiredCapabilities": {
        "browserName": "phantomjs",
        "javascriptEnabled": true,
        "databaseEnabled": false,
        "locationContextEnabled": false,
        "applicationCacheEnabled": false,
        "browserConnectionEnabled": false,
        "webStorageEnabled": false,
        "acceptSslCerts": true,
        "rotatable": false,
        "nativeEvents": false,
        "phantomjs.binary.path": "./node_modules/starrynight/node_modules/phantomjs-prebuilt/bin/phantomjs"
      }
    },
    "unittests": {
      "selenium": {
        "start_process": false,
        "start_session": false
      },
      "filter": "./tests/nightwatch/unittests/*",
      "exclude": ""
    }
  }
}

Instalação e uso #

Para que o Nightwatch funcione, você precisará de uma cópia local do selenium, que é um servidor de comando e controle que gerencia instâncias automatizadas do navegador. Você também precisará de um navegador da Web que o selenium possa controlar, como chromedriver ou phantomjs.

Adicione as seguintes devDependencies ao seu package.json:

{
  "devDependencies": {
    "nightwatch": "0.9.8",
    "selenium-server-standalone-jar": "2.45.0",
    "chromedriver": "2.19.0",
    "phantomjs-prebuilt": "2.1.12"
  }
}

Em seguida, instale todas as dependências.

cd myapp
meteor npm install

Você deve poder executar o Nightwatch com os seguintes comandos:

nightwatch -c .meteor/nightwatch.json
nightwatch -c .meteor/nightwatch.json --env phantom

Se você ainda não escreveu nenhum teste ou configurou sua estrutura de pastas, poderá receber alguns erros.

Configurando scripts de inicialização #

Na raiz de sua aplicação deve haver um arquivo package.json, onde você pode definir scripts e devDependencies.

{
  "name": "myapp",
  "version": "1.0.0",
  "scripts": {
    "start": "meteor --settings settings-development.json",
    "nightwatch": "nightwatch -c .meteor/nightwatch.json",
    "phantom": "nightwatch -c .meteor/nightwatch.json --env phantom",
  }
}

Você poderá então iniciar o nightwatch com os seguintes comandos:

meteor npm run-script nightwatch
meteor npm run-script phantom

Neste exemplo, seria quase mais fácil simplesmente executar nightwatch -c .meteor/nightwatch.json. No entanto, com comandos mais complexos, com variáveis ​​de ambiente, opções e configurações complexas, isso se torna uma maneira muito útil de especificar scripts de devops para uma equipe.

Estrutura de pastas #

Uma instalação básica do Nightwatch para o Meteor terá os seguintes diretórios e arquivos instalados.

/myapp
/myapp/.meteor/nightwatch.json
/client/main.html
/client/main.js
/client/main.css
/tests
/tests/nightwatch
/tests/nightwatch/assertions
/tests/nightwatch/commands
/tests/nightwatch/data
/tests/nightwatch/logs
/tests/nightwatch/pages
/tests/nightwatch/reports
/tests/nightwatch/screenshots
/tests/nightwatch/walkthroughs
/tests/nightwatch/walkthroughs/critical_path.js
/tests/nightwatch/globals.json

Testes baseados em dados #

Nightwatch aceita um segundo arquivo de configuração globals.json que injeta dados no próprio executor de teste, muito semelhante a como Meteor.settings disponibiliza dados da linha de comando em todo o aplicativo.

globals.json

{
  "default" : {
    "url" : "http://localhost:3000",
    "user": {
      "name": "Jane Doe",
      "username" : "janedoe",
      "password" : "janedoe123",
      "email" : "janedoe@test.org",
      "userId": null
    }
  },
  "circle" : {
    "url" : "http://localhost:3000",
    "user": {
      "name": "Jane Doe",
      "username" : "janedoe",
      "password" : "janedoe123",
      "email" : "janedoe@test.org"
      "userId": null
    }
  },
  "galaxy" : {
    "url" : "http://myapp.meteorapp.com",
    "user": {
      "name": "Jane Doe",
      "username" : "janedoe",
      "password" : "janedoe123",
      "email" : "janedoe@test.org"
      "userId": null
    }
  }
}

Você pode então escrever seus testes que não são codificados com usuários específicos, senhas, entradas de pesquisa, etc.

module.exports = {
  "Login App" : function (client) {
    client
      .url(client.globals.url)
      .login(client.globals.user.email, client.globals.user.password)
      .end();
  }
};
