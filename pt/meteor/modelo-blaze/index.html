Blaze é uma biblioteca poderosa para criar interfaces de usuário escrevendo modelos HTML dinâmicos e reativos. O modelo Blaze permite que loops e lógica condicional sejam usados ​​diretamente na marcação HTML. Esta seção explica e demonstra o uso adequado de modelagem no Meteor.js com o Blaze.

Preencha um modelo de uma chamada de método #

<template name="myTemplate">
  {{#each results}}
    <div><span>{{name}}</span><span>{{age}}</span></div>
  {{/each}}
</template>

Template.myTemplate.onCreated(function() {
  this.results = new ReactiveVar();
  Meteor.call('myMethod', (error, result) => {
    if (error) {
      // do something with the error
    } else {
      // results is an array of {name, age} objects
      this.results.set(result);
    }
  });
});

Template.myTemplate.helpers({
  results() {
    return Template.instance().results.get();
  }
});

Contexto de dados de um modelo #

Sempre que um modelo é chamado, o contexto de dados padrão do modelo é obtido implicitamente do chamador, como no exemplo, o childTemplate ganha o contexto de dados do parentTemplate, ou seja, o modelo do chamador

<template name="parentTemplate">
    {{#with someHelperGettingDataForParentTemplate}}
    <h1>My name is {{firstname}} {{lastname}}</h1>
    //some stuffs here
    {{> childTemplate}}
    {{/with}}
</template>

Na situação acima, quaisquer dados que o auxiliar extraia para o template pai são automaticamente obtidos pelo childTemplate. Por exemplo, o {{firstname}} e o {{lastname}} podem ser acessados ​​do childTemplate, assim como mostrado abaixo.

<template name="childTemplate">
<h2>My name is also {{firstname}} {{lastname}}</h2>
</template>

Podemos até definir explicitamente o contexto de dados do childTemplate passando argumentos para o template como no exemplo abaixo.

<template name="parentTemplate">
    {{#with someHelperGettingDataForParentTemplate}}
    <h1>My name is {{firstname}} {{lastname}}</h1>
    //some stuffs here
    {{> childTemplate childData=someHeplerReturningDataForChild}}
    {{/with}}
</template>

Supondo que o auxiliar someHelperReturningDataForChild retorne um objeto como {profession:“Meteor Developer”,hobby:“stackoverflowing”}, esse objeto específico será o contexto de dados explícito para o childTemplate. Agora no modelo filho podemos fazer algo como

<template name="childTemplate">
    <h2>My profession is {{profession}}</h2>
    <h3>My hobby is {{hobby}}</h3>
</template>

Ajudantes de modelo #

Ajudantes de modelo são uma parte essencial do Blaze e fornecem lógica de negócios e reatividade a um modelo. É importante lembrar que os auxiliares de modelo são na verdade computações reativas que são executadas novamente sempre que suas dependências mudam. Dependendo de suas necessidades, os auxiliares de modelo podem ser definidos globalmente ou com escopo para um modelo específico. Exemplos de cada abordagem de definição do auxiliar de modelo são fornecidos abaixo.

  1. Exemplo de um auxiliar de modelo com escopo para um único modelo.

Primeiro defina seu modelo:

<template name="welcomeMessage">
  <h1>Welcome back {{fullName}}</h1>
</template>

Em seguida, defina o auxiliar de modelo. Isso pressupõe que o contexto de dados do modelo contém uma propriedade firstName e lastName.

Template.welcomeMessage.helpers({
  fullName: function() {
    const instance = Template.instance();
    return instance.data.firstName + ' ' + instance.data.lastName
  },
});
  1. Exemplo de um assistente de modelo global (este assistente pode ser usado em qualquer modelo)

Primeiro cadastre o ajudante:

Template.registerHelper('equals', function(item1, item2) {
  if (!item1 || !item2) {
    return false;
  }

  return item1 === item2;
});

Com o auxiliar equals definido, agora posso usá-lo em qualquer modelo:

<template name="registration">
  {{#if equals currentUser.registrationStatus 'Pending'}}
    <p>Don't forget to complete your registration!<p>
  {{/if}}
</template>
