Começando #

#Instala Meteoro

No OS X e Linux #

Instale a versão oficial mais recente do Meteor do seu terminal:

$ curl https://install.meteor.com/ | sh

No Windows #

Baixe o instalador oficial do Meteor aqui.

Crie seu aplicativo #

Depois de instalar o Meteor, crie um projeto:

$ meteor create myapp

Executá-lo #

Execute-o localmente:

$ cd myapp
$ meteor npm install
$ meteor

Nota: Servidor Meteor rodando em: http://localhost:3000/

Então vá para http://localhost:3000 para ver seu novo aplicativo Meteor.

  • Leia mais sobre como começar com o Meteor no [Meteor Guide].
  • Explore Pacotes Meteor em atmosphere - um gerenciador de pacotes moderno, rápido e bem projetado.

Aplicativos de exemplo #

O Meteor tem vários aplicativos de amostra integrados. Você pode criar um projeto com um deles e aprender como ele foi construído. Para criar um aplicativo de exemplo, instale o Meteor (consulte Introdução) e digite:

meteor create --example <app name>

Por exemplo, para criar um aplicativo todos de amostra, escreva:

meteor create --example todos

Para obter uma lista de todos os aplicativos de amostra, digite:

meteor create --list

Entendendo o progresso da compilação #

Às vezes, as compilações demoram mais do que o esperado. Existem algumas variáveis ​​de ambiente que você pode definir para entender melhor o que está acontecendo durante o processo de compilação.

METEOR_DEBUG_BUILD=1       (logs progress)
METEOR_PROFILE=<n>         (logs time spent)
METEOR_DEBUG_SPRINGBOARD=1 (?)
METEOR_DEBUG_SQL=1         (logs SQLITE calls)
METEOR_PROGRESS_DEBUG=1    (? looks like it might be useful, but seems confusing)

Onde <n> é um número de ms. Qualquer processo que demore mais do que isso será registrado.

Exemplo Linux/OSX #

export METEOR_DEBUG_BUILD=1
export METEOR_PROFILE=100
meteor

Exemplo do Windows #

set METEOR_DEBUG_BUILD=1
set METEOR_PROFILE=100
meteor

Atualizando Projetos Meteor e Pacotes Instalados #

A ferramenta Meteor irá notificá-lo quando uma versão mais recente estiver disponível.

Para atualizar os projetos Meteor para a versão mais recente, execute o seguinte comando dentro de um projeto Meteor:

meteor update

Caso você queira atualizar seu projeto Meteor para uma versão específica do Meteor, execute o seguinte comando dentro do projeto:

meteor update --release <release>

Se você deseja atualizar todos os pacotes não essenciais, execute:

meteor update --packages-only

Você também pode atualizar pacotes específicos passando seus nomes como um argumento de linha de comando para meteor update, por exemplo:

meteor update [packageName packageName2 ...]

Crie aplicativos móveis #

O Meteor usa Cordova para empacotar seu aplicativo em um aplicativo móvel híbrido. Uma vez empacotado, o App pode ser distribuído como Apps nativos (através da Apple App Store, Google Play Store, etc.)

  1. Adicione a(s) plataforma(s) de destino ao seu projeto Meteor:
meteor add-platform android
meteor add-platform ios # Only available with Mac OS

  1. Instalar o Android SDK e/ou Xcode (para iOS, requer Mac OS).

  2. Execute seu projeto (comece com o modo de desenvolvimento):

meteor run android # You may need to configure a default Android emulator first

Para iOS (disponível apenas com Mac OS):

meteor run ios # This will auto start an iOS simulator
  1. Crie seu pacote de aplicativos para distribuição:
meteor build <output_folder> --server <url_app_should_connect_to>

Isso criará a(s) pasta(s) android e/ou ios junto com o pacote do seu servidor.

  • A pasta android contém o arquivo release-unsigned.apk que você precisa para assinar e alinhar o zip.
  • A pasta ios contém o projeto Xcode que você precisa assinar.

Veja também o tópico Meteor Aplicativos Móveis. Página de referência: Guia do Meteoro > Construir > Celular

Gerenciando Pacotes #

Meteor tem seu próprio repositório de pacotes em atmospherejs.com

Você pode adicionar novos pacotes da atmosfera executando:

meteor add [package-author-name:package-name]

Por exemplo:

meteor add kadira:flow-router

Da mesma forma, você pode remover o mesmo pacote:

meteor remove kadira:flow-router

Para ver os pacotes atuais em seu projeto, digite:

meteor list

A lista de pacotes também pode ser encontrada no arquivo ./meteor/packages. Para adicionar um pacote, adicione o nome do pacote neste arquivo e, para remover, exclua-o.

Para adicionar um pacote localmente (por exemplo, pacotes não publicados ou versão editada de pacotes publicados), salve o pacote na pasta packages na raiz.

A partir da versão 1.3, Meteor adicionado suporte para pacotes npm.

Você pode usar o comando npm dentro do diretório do projeto Meteor como faria normalmente sem o Meteor, ou com o comando meteor npm, que usará a versão empacotada do npm.

Verificando a Versão da Ferramenta Meteor e Projetos Meteor #

#Ferramenta Meteoro

Para verificar a versão instalada da ferramenta Meteor, basta executar o seguinte comando fora de qualquer projeto Meteor:

meteor --version

Para obter uma lista de todos os lançamentos oficiais (recomendados) do Meteor, execute:

meteor show METEOR

#Projetos Meteoros

Se você quiser verificar a versão do projeto Meteor, você também pode executar o seguinte comando dentro de um projeto:

meteor --version

ou apenas imprima o conteúdo do arquivo .meteor/release:

cat .meteor/release

Caso você queira verificar a versão dos pacotes que estão atualmente instalados em seu projeto Meteor, imprima o conteúdo do arquivo .meteor/versions:

cat .meteor/versions

Site Meteoro #

Para ver qual versão do Meteor um site baseado em Meteor está rodando, despeje o conteúdo de Meteor.release no console do seu navegador enquanto visita o site:

Meteor.release
