Meteor + React + ReactRouter

Este documento mostrará como usar o ReactRouter com Meteor e React. De zero a um aplicativo funcional, incluindo funções e autenticação.

Vou mostrar cada passo com um exemplo

1- Crie o projeto

2- Adicionar React + ReactRouter

3- Adicionar contas

4- Adicionar pacotes de funções

Crie o projeto #

1- Antes de tudo, instale https://www.meteor.com/install

2- Crie um projeto. (--bare é criar um projeto vazio)

meteor create --bare MyAwesomeProject

3- Crie a estrutura mínima de arquivos (-p para criar diretórios intermediários):

cd MyAwesomeProject

mkdir -p client server imports/api imports/ui/{components,layouts,pages} imports/startup/{client,server}

4- Agora, crie um arquivo HTML em client/main.html

<head>
   <meta charset="utf-8">
   <title>My Awesome Meteor_React_ReactRouter_Roles App</title>
</head>
 
<body>
  Welcome to my Meteor_React_ReactRouter_Roles app
</body>

5- Verifique se está funcionando: (3000 é a porta padrão, então você pode pular o ‘-p 3000’)

corrida de meteoros -p 3000

e abrindo seu navegador em ’localhost:3000'

Observação: #

  • Estou pulando alguns outros arquivos que você precisará criar, para encurtar as coisas. Especificamente, você precisará criar alguns arquivos index.js nos diretórios client , imports/startup/{client,server} e server.

  • Você pode ver um exemplo completo em https://github.com/rafa-lft/Meteor_React_Base. Procure a tag Step1_CreateProject

Adicionar React + ReactRouter #

Se necessário, mude para o diretório do seu projeto cd MyAwesomeProject

1- Adicione react e react-router

meteor npm install --save react-router@3.0.0 react@15.5.4 react-dom@15.5.4

2- Edite client/main.html, e substitua o conteúdo do será:

 <body>
    <div id="react-root"></div>
 </body>

O que quer que o reactRouter decida mostrar, ele o mostrará no elemento ‘#react-root’

3- Crie o arquivo Layouts em imports/ui/layouts/App.jsx

import React, { Component } from 'react';
import PropTypes from 'prop-types';


class App extends Component {
  constructor(props) {
    super(props);
  }

  render() {
    return (
      <div>
        {this.props.children}
      </div>
    );
  }
}

App.propTypes = {
  children: PropTypes.node
};

export default App;

4- Crie o arquivo de Rotas em imports/startup/client/Routes.jsx

import ReactDOM from 'react-dom';
import React, { Component } from 'react';
import { Router, Route, IndexRoute, browserHistory } from 'react-router';

import App from '../../ui/layouts/App.jsx';

import NotFound from '../../ui/pages/NotFound.jsx';
import Index from '../../ui/pages/Index.jsx';


class Routes extends Component {
  constructor(props) {
    super(props);
  }

  render() {
    return (
      <Router history={ browserHistory }>
        <Route path="/" component={ App }>
          <IndexRoute name="index" component={ Index }/>
          <Route path="*" component={ NotFound }/>
        </Route>
      </Router>
    );
  }
}

Routes.propTypes = {};


Meteor.startup(() =>{
  ReactDOM.render(
    <Routes/>,
    document.getElementById('react-root')
  );
});

Observação: #

  • Estou pulando alguns outros arquivos que você precisará criar, para encurtar as coisas. Especificamente, verifique imports/ui/pages{Index.jsx,NotFound.jsx}.

  • Você pode ver um exemplo completo em https://github.com/rafa-lft/Meteor_React_Base. Procure a tag Step2_ReactRouter

Etapa 3- Adicionar contas #

Se necessário, mude para o diretório do seu projeto cd MyAwesomeProject

1- Adicionar pacotes de contas: meteor adiciona contas-base contas-senha react-meteor-data

2- Adicione as rotas às páginas login e signup em imports/startup/Routes.jsx O método render() será o seguinte:

  render() {
    return (
      <Router history={ browserHistory }>
        <Route path="/" component={ App }>
          <IndexRoute name="index" component={ Index }/>
          <Route name="login" path="/login" component={ Login }/>
          <Route name="signup" path="/signup" component={ Signup }/>
          <Route name="users" path="/users" component={ Users }/>
          <Route name="editUser" path="/users/:userId" component={ EditUser }/>
          <Route path="*" component={ NotFound }/>
        </Route>
      </Router>
    );
  }

Observação: #

  • Estou pulando alguns outros arquivos que você vai precisar, para encurtar as coisas. Especificamente, verifique imports/startup/server/index.js imports/ui/layouts/{App,NavBar}.jsx e import/ui/pages/{Login,Signup,Users,EditUser}.jsx

  • Você pode ver um exemplo completo em https://github.com/rafa-lft/Meteor_React_Base. Procure a tag Step3_Accounts

Adicionar funções #

1- Adicionar pacote de funções ( https://github.com/alanning/meteor-roles)

meteor add alanning:roles

2- Crie algumas constantes de papéis. No arquivo imports/api/accounts/roles.js

const ROLES = {
  ROLE1: 'ROLE1',
  ROLE2: 'ROLE2',
  ADMIN: 'ADMIN'
};

export default ROLES;

3- Não mostrarei como adicionar/atualizar funções em um usuário, apenas mencionarei que no lado do servidor, você pode definir funções de usuário por Roles.setUserRoles(user.id, roles); Verifique para obter mais informações em https ://github.com/alanning/meteor-roles e http://alanning.github.io/meteor-roles/classes/Roles.html

4- Supondo que você já tenha configurado todos os arquivos de contas e funções (veja o exemplo completo em https://github.com/rafa-lft/Meteor_React_Base. Procure a tag Step4_roles) agora podemos criar um método que ficará responsável por permitindo (ou não) o acesso às diferentes rotas. Em imports/startup/client/Routes.jsx


class Routes extends Component {
  constructor(props) {
    super(props);
  }

  authenticate(roles, nextState, replace) {
    if (!Meteor.loggingIn() && !Meteor.userId()) {
      replace({
        pathname: '/login',
        state: {nextPathname: nextState.location.pathname}
      });
      return;
    }
    if ('*' === roles) { // allow any logged user
      return;
    }
    let rolesArr = roles;
    if (!_.isArray(roles)) {
      rolesArr = [roles];
    }
    // rolesArr = _.union(rolesArr, [ROLES.ADMIN]);// so ADMIN has access to everything
    if (!Roles.userIsInRole(Meteor.userId(), rolesArr)) {
      replace({
        pathname: '/forbidden',
        state: {nextPathname: nextState.location.pathname}
      });
    }
  }

  render() {
    return (
      <Router history={ browserHistory }>
        <Route path="/" component={ App }>
          <IndexRoute name="index" component={ Index }/>
          <Route name="login" path="/login" component={ Login }/>
          <Route name="signup" path="/signup" component={ Signup }/>

          <Route name="users" path="/users" component={ Users }/>

          <Route name="editUser" path="/users/:userId" component={ EditUser }
                 onEnter={_.partial(this.authenticate, ROLES.ADMIN)} />


          {/* ********************
           Below links are there to show Roles authentication usage.
           Note that you can NOT hide them by
           { Meteor.user() && Roles.userIsInRole(Meteor.user(), ROLES.ROLE1) &&
           <Route name=.....
           }
           as doing so will change the Router component on render(), and ReactRouter will complain with:
           Warning: [react-router] You cannot change <Router routes>; it will be ignored

           Instead, you can/should hide them on the NavBar.jsx component... don't worry: if someone tries to access
           them, they will receive the Forbidden.jsx component
           *************/ }
          <Route name="forAnyOne" path="/for_any_one" component={ ForAnyone }/>

          <Route name="forLoggedOnes" path="/for_logged_ones" component={ ForLoggedOnes }
                 onEnter={_.partial(this.authenticate, '*')} />

          <Route name="forAnyRole" path="/for_any_role" component={ ForAnyRole }
                 onEnter={_.partial(this.authenticate, _.keys(ROLES))}/>

          <Route name="forRole1or2" path="/for_role_1_or_2" component={ ForRole1or2 }
                 onEnter={_.partial(this.authenticate, [ROLES.ROLE1, ROLES.ROLE2])} />

          <Route name="forRole1" path="/for_role1" component={ ForRole1 }
                 onEnter={_.partial(this.authenticate, ROLES.ROLE1)}/>

          <Route name="forRole2" path="/for_role2" component={ ForRole2 }
                 onEnter={_.partial(this.authenticate, ROLES.ROLE2)} />


          <Route name="forbidden" path="/forbidden" component={ Forbidden }/>

          <Route path="*" component={ NotFound }/>
        </Route>
      </Router>
    );
  }
}

Adicionamos um gatilho onEnter a algumas rotas. Para essas rotas, também estamos passando quais Roles têm permissão para entrar. Observe que o retorno de chamada onEnter recebe 2 parâmetros originalmente. Estamos usando o parcial do sublinhado (http://underscorejs.org/#partial), para adicionar outro (funções) O método authenticate (chamado por onEnter) recebe os papéis e:

  • Verifique se o usuário está logado. Caso contrário, redireciona para ‘/login’.
  • Se roles === ‘*’, assumimos que qualquer usuário logado pode entrar, então permitimos
  • Caso contrário, verificamos se o usuário é permitido (Roles.userIsInRole) e se não, redirecionamos para proibido.
  • Opcionalmente, você pode descomentar uma linha, para que o ADMIN tenha acesso a tudo.

O código tem vários exemplos de rotas diferentes que são permitidas para qualquer pessoa (sem retorno de chamada onEnter), para qualquer usuário logado, para qualquer usuário logado com pelo menos 1 função e para funções específicas.

Observe também que o ReactRouter (pelo menos na versão 3) não permite modificar as rotas no Render. Portanto, você não pode ocultar as rotas dentro do Routes.jsx. Por esse motivo, redirecionamos para /forbidden no método authenticate.

5- Um bug comum com ReactRouter e Meteor, está relacionado a atualizações de status do usuário que não são mostradas. Por exemplo, o usuário saiu, mas ainda estamos mostrando seu nome na barra de navegação. Isso acontece porque Meteor.user() mudou, mas não estamos renderizando novamente.

Esse bug pode ser resolvido chamando Meteor.user() no createContainer. Aqui está um exemplo disso, usado em imports/ui/layouts/NavBar.jsx:

export default createContainer((/* {params}*/) =>{
  Meteor.user(); // so we render again in logout or if any change on our User (ie: new roles)
  const loading = !subscription.ready();
  return {subscriptions: [subscription], loading};
}, NavBar);

Observação: #

  • Estou pulando alguns outros arquivos que você vai precisar, para encurtar as coisas. Especificamente, verifique imports/startup/server/index.js imports/ui/layouts/{App,NavBar}.jsx e import/ui/pages/{Login,Signup,Users,EditUser}.jsx

  • Você pode ver um exemplo completo em https://github.com/rafa-lft/Meteor_React_Base. Procure a tag Step4_roles
