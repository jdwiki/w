Criando elementos personalizados em C++ #

QML veio com um rico conjunto de elementos visuais. Usando apenas QML podemos construir aplicações complexas com esses elementos. Também é muito fácil construir seu próprio elemento baseado em um conjunto de itens padrão como Retângulo, Botão, Imagem etc. Além disso, podemos usar itens como Canvas para construir elementos com pintura personalizada. Parece que podemos construir uma variedade de aplicativos apenas em QML, sem tocar nos recursos de C++. E é realmente verdade, mas ainda assim às vezes gostaríamos de tornar nosso aplicativo mais rápido ou queremos estendê-lo com o poder do Qt ou adicionar algumas oportunidades que não estão disponíveis no QML. E certamente existe essa possibilidade em QML. Basicamente o QtQuick usa o Scene Graph para pintar seu conteúdo em um mecanismo de renderização de alto desempenho baseado no OpenGL. Para implementar nosso próprio elemento visual, podemos usar 2 maneiras:

  1. A maneira tradicional para Qt usando QPainter (QQuickPaintedItem).
  2. A maneira comum de QML usando QQuickItem e funcionalidade OpenGL.

É possível que o primeiro método pareça mais fácil, mas vale a pena considerar que também é mais lento que o primeiro, pois o QtQuick pinta o conteúdo do item em uma superfície e depois insira-o no gráfico de cena para que a renderização seja uma operação de duas etapas. Portanto, usar a API de gráfico de cena diretamente é sempre significativamente mais rápido.

Para explorar os dois métodos mais de perto, vamos criar nosso próprio elemento que definitivamente não existe em QML, por exemplo, um triângulo.

Declaração de classe

class QQuickCustomItem : public QQuickItem
{
    Q_OBJECT
    Q_PROPERTY(QColor color READ color WRITE setColor NOTIFY colorChanged)
public:
    QQuickCustomItem(QQuickItem *parent = Q_NULLPTR);

protected:
    QSGNode *updatePaintNode(QSGNode *oldNode, UpdatePaintNodeData *updatePaintNodeData);

    QColor color() const;
    void setColor(const QColor &color);

private:
    QColor m_color;
    bool m_needUpdate;

signals:
    void colorChanged();
};

Adicionamos a macro Q_OBJECT para trabalhar com sinais. Também adicionamos a propriedade customizada para especificar a cor do nosso retângulo. Para fazê-lo funcionar, tudo o que precisamos é reimplementar a função virtual QQuiclItem::updatePaintNode().

Implementação de classe.

Primeiramente definimos um construtor.

QQuickCustomItem::QQuickCustomItem(QQuickItem *parent) :
    QQuickItem(parent),
    m_color(Qt::red),
    m_needUpdate(true)
{
    setFlag(QQuickItem::ItemHasContents);
}

Observe que a chamada da função setFlag() é obrigatória, caso contrário seu objeto não será adicionado ao gráfico de cena. Em seguida, definimos uma função para o paining.

QSGNode *QQuickCustomItem::updatePaintNode(QSGNode *oldNode, QQuickItem::UpdatePaintNodeData *updatePaintNodeData)
{
    Q_UNUSED(updatePaintNodeData)
    QSGGeometryNode *root = static_cast<QSGGeometryNode *>(oldNode);

    if(!root) {
        root = new QSGGeometryNode;
        QSGGeometry *geometry = new QSGGeometry(QSGGeometry::defaultAttributes_Point2D(), 3);
        geometry->setDrawingMode(GL_TRIANGLE_FAN);
        geometry->vertexDataAsPoint2D()[0].set(width() / 2, 0);
        geometry->vertexDataAsPoint2D()[1].set(width(), height());
        geometry->vertexDataAsPoint2D()[2].set(0, height());

        root->setGeometry(geometry);
        root->setFlag(QSGNode::OwnsGeometry);
        root->setFlag(QSGNode::OwnsMaterial);
    }

    if(m_needUpdate) {
        QSGFlatColorMaterial *material = new QSGFlatColorMaterial;
        material->setColor(m_color);
        root->setMaterial(material);
        m_needUpdate = false;
    }

    return root;
}

Na primeira chamada para a função nosso nó ainda não foi criado, então oldNode será NULL. Então, criamos o nó e atribuímos geometria e material a ele. Aqui usamos GL_TRIANGLE_FAN para nossa geometria para pintar um retângulo sólido. Este ponto é o mesmo que no OpenGL. Por exemplo, para desenhar um quadro de triângulo, podemos alterar o código para:

geometry->setDrawingMode(GL_LINE_LOOP);
geometry->setLineWidth(5);

Você pode consultar o manual do OpenGL para verificar outras formas. Então, tudo o que resta é definir setter/getter para nossa propriedade:

QColor QQuickCustomItem::color() const
{
    return m_color;
}

void QQuickCustomItem::setColor(const QColor &color)
{
    if(m_color != color) {
        m_color = color;
        m_needUpdate = true;
        update();
        colorChanged();
    }
}

Agora há apenas um pequeno detalhe para fazê-lo funcionar. Precisamos notificar QtQuick sobre o novo item. Por exemplo, você pode adicionar este código ao seu main.cpp:

qmlRegisterType<QQuickCustomItem>("stackoverflow.qml", 1, 0, "Triangle");

E aqui está nosso arquivo de teste QML:

import QtQuick 2.7
import QtQuick.Window 2.0
import stackoverflow.qml 1.0

Window {
    width: 800
    height: 800
    visible: true

    Rectangle {
        width: 200
        height: 200
        anchors.centerIn: parent
        color: "lightgrey"

        Triangle {
            id: rect
            width: 200
            height: 200
            transformOrigin: Item.Top
            color: "green"
            onColorChanged: console.log("color was changed");
            PropertyAnimation on rotation {
                from: 0
                to: 360
                duration: 5000
                loops: Animation.Infinite
            }
        }
    }
    Timer {
        interval: 1000
        repeat: true
        running: true
        onTriggered: rect.color = Qt.rgba(Math.random(),Math.random(),Math.random(),1);
    }
}

Como você vê, nosso item se comporta como todos os outros itens QML. Agora vamos criar o mesmo item usando QPainter:

Tudo o que precisamos é substituir

QSGNode *updatePaintNode(QSGNode *oldNode, UpdatePaintNodeData *updatePaintNodeData);

com

void paint(QPainter *painter);

e, claro, herdar nossa classe de QQuickPaintedItem em vez de QQuickItem. Aqui está a nossa função de pintura:

void QQuickCustomItem::paint(QPainter *painter)
{
    QPainterPath path;
    path.moveTo(width() / 2, 0);
    path.lineTo(width(), height());
    path.lineTo(0, height());
    path.lineTo(width() / 2, 0);
    painter->fillPath(path, m_color);
}

Todo o resto permanece inalterado.
