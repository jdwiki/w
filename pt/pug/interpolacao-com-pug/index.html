É importante poder usar variáveis ​​do lado do servidor em seu site. O Pug permite interpolar dados gerados pelo seu servidor em HTML, CSS e até código JavaScript.

Sintaxe #

  • res.render(path, variables) // Procura um arquivo pug para renderizar no caminho “path” e passa “variables” para ele
  • #{variable} // Interpola “variable” inline com o código Jade ao redor, após avaliar “variable”
  • !{variable} // Interpola “variable” inline com o código Jade circundante, sem avaliar “variable”.
  • #[element] // Interpola “element” dentro de um elemento HTML Pug existente. A sintaxe dos elementos HTML interpolados é idêntica à dos elementos HTML normais.

Parâmetros #

Parâmetro Detalhes
caminho Usado em res.render. Este é o caminho do arquivo Pug que vamos renderizar. O caminho é obtido da raiz da pasta definida em seu aplicativo Express: app.set("views", "templates/views"). Por exemplo, res.render("index") irá procurar um arquivo Pug em templates/views/index.pug. Os subdiretórios também podem ser especificados; res.render("admin/index") procura um arquivo Pug em templates/views/admin/index.pug.
variáveis ​​ Usado em res.render. Um objeto JavaScript de variáveis ​​a serem acessíveis ao arquivo Pug definido por path (acima). Dentro do arquivo Pug, as chaves do objeto JavaScript acima ficam disponíveis como variáveis. Se variables = {title: "Hello", color: "red"}, poderíamos usar as variáveis ​​title e color. As subpropriedades de objetos aninhados também estão disponíveis.
variável Usado na sintaxe de colchetes #{} ou !{}. O valor de variable será gerado no contexto de seu código Pug circundante. Se um símbolo de libra for anexado ao colchete de abertura, a variável será avaliada antes de ser exibida. Se um ponto de exclamação for anexado à chave de abertura, variable não será avaliado.
elemento Usado em sytax de colchetes #[]. O elemento HTML (na sintaxe Pug, não na sintaxe HTML normal) será avaliado e produzido em linha com o código Pug circundante.

Para obter mais informações sobre a interpolação do PugJS, consulte a documentação oficial da interpolação do PugJS.

Interpolação de variável do lado do servidor #

É possível passar variáveis ​​do seu servidor para o Pug para conteúdo dinâmico ou geração de scripts. Os templates Pug podem acessar variáveis ​​passadas para a função res.render no Express (ou pug.renderFile se você não estiver usando o Express, os argumentos são idênticos).

index.js

let colors = ["Red", "Green", "Blue"];
let langs  = ["HTML", "CSS", "JS"];
let title  = "My Cool Website";

let locals = {
    siteColors: colors,
    siteLangs:  langs,
    title:      title
};
res.render("index", locals);

Dentro do seu arquivo index.pug, você tem acesso à variável locals por meio de suas chaves. Os nomes das variáveis ​​em seu arquivo Pug se tornam siteColors e siteNames.

Para definir a totalidade de um elemento HTML igual a uma variável, use o operador equals = para fazer isso. Se sua variável precisar ser incorporada inline, use a sintaxe de colchetes #{} para fazer isso.

index.pug

doctype html
html
    head
        title= title
    body
        p My favorite color is #{siteColors[0]}.
        p Here's a list of my favorite coding languages
        ul
            each language in siteLangs
                li= language

saída index.pug

<!DOCTYPE html>
<html>
    <head>
        <title>My Cool Website</title>
    </head>
    <body>
        <p>My favorite color is Red.</p>
        <p>Here's a list of my favorite coding languages</p>
        <ul>
            <li>HTML</li>
            <li>CSS</li>
            <li>JS</li>
        </ul>
    </body>
</html>

Interpolação de variável bruta em HTML #

O conteúdo interpolado com a sintaxe de colchetes será avaliado para código, cuja saída é incluída na saída HTML.

title segue o padrão básico para avaliar um template local, mas o código entre #{ e } é avaliado, escapa, e o resultado é armazenado em buffer na saída do template que está sendo renderizado. [Fonte]

Se você precisar incluir sintaxe HTML bruta, use um ponto de exclamação em vez de um símbolo de libra (!{} em vez de #{}).

index.js

let tag = "<div>You can't escape me!</div>";
res.render("index", {
    myTag: tag
});

index.pug

doctype html
html
    head
    body
        !{myTag}

saída index.pug

<!DOCTYPE html>
<html>
    <head></head>
    <body>
        <div>You can't escape me!</div>
    </body>
</html>

Interpolação de valor no código JavaScript #

A interpolação de valores é útil se você precisar passar uma variável do lado do servidor para o JavaScript do lado do cliente (ou outras linguagens que exijam isso).

No caso de variáveis, números, strings e similares, você pode passar esses tipos de variáveis ​​diretamente em seu JavaScript com sintaxe de colchetes mais um ponto de explicação (para que o código dentro dos colchetes não seja avaliado). Código JavaScript que requer algo do seu servidor.

No exemplo abaixo, temos que colocar username entre aspas para que o JavaScript o interprete como uma string; O Pug produzirá o conteúdo da variável como está, então precisamos colocá-lo entre aspas para que seja uma string JavaScript adequada. Isso não é necessário para number, onde o JavaScript interpretará nosso número como pretendemos (como um número).

index.js

let number   = 24;
let username = "John";
res.render("index", {
    number:   number,
    username: username
});

index.pug

html
    head
    script.
        // Sets the username of the current user to be displayed site-wide
        function setUsername(username) {
            // ...
        }
        var number   = #{number};
        var username = "#{username}";
        setUsername(username);
        
    body
        p Welcome to my site!

saída index.pug

<html>
    <head>
        <script>
            // Sets the username of the current user to be displayed site-wide
            function setUsername(username) {
                // ...
            }
            var number   = 24;
            var username = "John";
            setUsername(username);
        </script>
    </head>
    <body>
        <p>Welcome to my site!</p>
    </body>
</html>

Se você precisar interpolar o valor de um objeto JavaScript (por exemplo, todas as informações sobre um usuário), você deve stringificar a saída no Pug para que ela seja tratada como um objeto JavaScript. Também é necessário gerar o conteúdo bruto da variável, em vez da forma avaliada dela. Se você fizesse a saída da variável de escape (var user = #{JSON.stringify(user)}), você receberia uma versão escaped do objeto (onde aspas e apóstrofos são convertidos em &quot;), que não é o que queremos para que JSON.stringify funcione nele.

index.js

var myUser = {
    name:    "Leeroy Jenkins",
    id:      1234567890,
    address: "123 Wilson Way, New York NY, 10165"
};

res.render('index', {
    user: myUser
});

index.pug

doctype html
html
    head
        script.
            window.onload = function () {
                function setUsername(username) {
                    return username;
                }

                var user = !{JSON.stringify(user)};
                document.getElementById("welcome-user").innerHTML = setUsername(user.name);
            };

    body
        div(id="welcome-user")

saída index.pug

<!DOCTYPE html>
<html>
    <head>
        <script>
            window.onload = function() {
                function setUsername(username) {
                    return username;
                }

                var user = {
                    "name": "Leeroy Jenkins",
                    "id": 1234567890,
                    "address": "123 Wilson Way, New York NY, 10165"
                };
                document.getElementById("welcome-user").innerHTML = setUsername(user.name);
        };
        </script>
    </head>
    <body>
        <div id="welcome-user"></div>
    </body>    
</html>

O innerHTML de #welcome-user torna-se igual a Leeroy Jenkins. O conteúdo da variável user é impresso diretamente na fonte HTML

Interpolação de elementos HTML #

Pode ser necessário aninhar tags HTML umas dentro das outras. A interpolação de elementos é feita em uma sintaxe semelhante à interpolação de variáveis; colchetes em vez de chaves são usados ​​aqui. A sintaxe de elementos HTML interpolados é idêntica à implementação de elementos HTML normais.

index.pug

doctype html
html
    head
        title My Awesome Website
    body
        p The server room went #[b boom]!
        p The fire alarm, however, #[u failed to go off...]
        p Not even #[a(href="https://stackoverflow.com/") Stack Overflow] could save them now.

saída index.pug

<!DOCTYPE html>
<html>
    <head>
        <title>My Awesome Website</title>
    </head>
    <body>
        <p>The server room went <b>boom</b>!</p>
        <p>The fire alarm, however, <u>failed to go off...</u></p>
        <p>Not even <a href="https://stackoverflow.com/">Stack Overflow</a> could save them now.</p>
    </body>
</html>
