Para usar o tipo predefinido _Bool e o cabeçalho <stdbool.h>, você deve estar usando as versões C99/C11 de C.

Para evitar avisos do compilador e possivelmente erros, você só deve usar o exemplo typedef/define se estiver usando C89 e versões anteriores da linguagem.

Usando stdbool.h #

O uso do arquivo de cabeçalho do sistema stdbool.h permite que você use bool como um tipo de dados booleano. true é avaliado como 1 e false é avaliado como 0.

#include <stdio.h>
#include <stdbool.h>

int main(void) {
    bool x = true;  /* equivalent to bool x = 1; */
    bool y = false; /* equivalent to bool y = 0; */
    if (x)  /* Functionally equivalent to if (x != 0) or if (x != false) */
    {
        puts("This will print!");
    }
    if (!y) /* Functionally equivalent to if (y == 0) or if (y == false) */
    {
        puts("This will also print!");
    }
}

bool é apenas uma boa grafia para o tipo de dados [_Bool][1]. Ele tem regras especiais quando números ou ponteiros são convertidos para ele.

[1]: https://www.wikiod.com/pt/c/boleano#Usando o tipo intrínseco (interno) _Bool

Usando #define #

C de todas as versões, tratará efetivamente qualquer valor inteiro diferente de 0 como true para operadores de comparação e o valor inteiro 0 como false. Se você não tem _Bool ou bool a partir do C99 disponível, você pode simular um tipo de dados Boolean em C usando macros #define, e você ainda pode encontrar essas coisas no código legado.

#include <stdio.h>

#define bool int
#define true 1
#define false 0

int main(void) {
    bool x = true;  /* Equivalent to int x = 1; */
    bool y = false; /* Equivalent to int y = 0; */
    if (x) /* Functionally equivalent to if (x != 0) or if (x != false) */
    {
        puts("This will print!");
    }
    if (!y) /* Functionally equivalent to if (y == 0) or if (y == false) */
    {
        puts("This will also print!");
    }
}

Não introduza isso em um novo código, pois a definição dessas macros pode colidir com os usos modernos de <stdbool.h>.

Usando o tipo intrínseco (interno) _Bool #

Adicionado na versão C99 padrão C, _Bool também é um tipo de dados C nativo. É capaz de manter os valores 0 (para false) e 1 (para true).

#include <stdio.h>

int main(void) {
    _Bool x = 1; 
    _Bool y = 0;
    if(x) /* Equivalent to if (x == 1) */
    {
        puts("This will print!");
    }
    if (!y) /* Equivalent to if (y == 0) */
    {
        puts("This will also print!");
    }
}

_Bool é um tipo inteiro, mas possui regras especiais para conversões de outros tipos. O resultado é análogo ao uso de outros tipos em expressões [if][1]. Na sequência

_Bool z = X;
  • Se X tem um tipo aritmético (é qualquer tipo de número), z se torna 0 se X == 0. Caso contrário, z se torna 1.
  • Se X tiver um tipo de ponteiro, z se tornará 0 se X for um ponteiro nulo e 1 caso contrário.

Para usar grafias melhores bool, false e ​​true você precisa usar [<stdbool.h>][2].

[1]: https://www.wikiod.com/pt/c/boleano#Inteiros e ponteiros em expressões booleanas. [2]: https://www.wikiod.com/pt/c/boleano#Usando stdbool.h

Inteiros e ponteiros em expressões booleanas. #

Todos os inteiros ou ponteiros podem ser usados ​​em uma expressão que é interpretada como “valor de verdade”.

int main(int argc, char* argv[]) {
  if (argc % 4) {
    puts("arguments number is not divisible by 4");
  } else {
    puts("argument number is divisible by 4");
  }
...

A expressão argc % 4 é avaliada e leva a um dos valores 0, 1, 2 ou 3. O primeiro, 0 é o único valor que é “falso” e traz a execução para a parte else. Todos os outros valores são “true” e vão para a parte if.

double* A = malloc(n*sizeof *A);
if (!A) {
   perror("allocation problems");
   exit(EXIT_FAILURE);
}

Aqui o ponteiro A é avaliado e se for um ponteiro nulo, um erro é detectado e o programa é encerrado.

Muitas pessoas preferem escrever algo como A == NULL, mas se você tiver essas comparações de ponteiros como parte de outras expressões complicadas, as coisas se tornarão rapidamente difíceis de ler.

char const* s = ....;   /* some pointer that we receive */
if (s != NULL && s[0] != '\0' && isalpha(s[0])) {
   printf("this starts well, %c is alphabetic\n", s[0]);
}

Para que isso seja verificado, você teria que escanear um código complicado na expressão e ter certeza sobre a preferência do operador.

char const* s = ....;   /* some pointer that we receive */
if (s && s[0] && isalpha(s[0])) {
   printf("this starts well, %c is alphabetic\n", s[0]);
}

é relativamente fácil de capturar: se o ponteiro for válido, verificamos se o primeiro caractere é diferente de zero e, em seguida, verificamos se é uma letra.

Definindo um tipo bool usando typedef #

Considerando que a maioria dos depuradores não conhece as macros #define, mas pode verificar as constantes enum, pode ser desejável fazer algo assim:

#if __STDC_VERSION__ < 199900L
typedef enum { false, true } bool;
/* Modern C code might expect these to be macros. */
# ifndef bool
#  define bool bool
# endif
# ifndef true
#  define true true
# endif
# ifndef false
#  define false false
# endif
#else
# include <stdbool.h>
#endif

/* Somewhere later in the code ... */
bool b = true;

Isso permite que compiladores para versões históricas de C funcionem, mas permanece compatível se o código for compilado com um compilador C moderno.

Para obter mais informações sobre typedef, consulte https://www.wikiod.com/pt/c/tipo-de-letra para mais informações sobre enum consulte https://www.wikiod.com/pt/c/enumeracoes
