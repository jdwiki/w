Um ponteiro é um tipo de variável que pode armazenar o endereço de outro objeto ou função.

Sintaxe #

  • <Tipo de dados> *<Nome da variável>;
  • int *ptrParaInt;
  • void *ptrToVoid; /* C89+ */
  • struct someStruct *ptrToStruct;
  • int **ptrToPtrToInt;
  • int arr[comprimento]; int *ptrToFirstElem = arr; /* Para <C99 ‘comprimento’ precisa ser uma constante de tempo de compilação, para >=C11 pode precisar ser um. */
  • int *arrayOfPtrsToInt[comprimento]; /* Para <C99 ‘comprimento’ precisa ser uma constante de tempo de compilação, para >=C11 pode precisar ser um. */

A posição do asterisco não afeta o significado da definição:

/* The * operator binds to right and therefore these are all equivalent. */
int *i;
int * i;
int* i;

No entanto, ao definir vários ponteiros de uma só vez, cada um requer seu próprio asterisco:

int *i, *j; /* i and j are both pointers */
int* i, j;  /* i is a pointer, but j is an int not a pointer variable */

Uma matriz de ponteiros também é possível, onde um asterisco é fornecido antes do nome da variável da matriz:

int *foo[2]; /* foo is a array of pointers, can be accessed as *foo[0] and *foo[1] */

Erros comuns #

O uso inadequado de ponteiros é frequentemente uma fonte de bugs que podem incluir bugs de segurança ou falhas de programa, na maioria das vezes devido a falhas de segmentação.

Não verificando falhas de alocação #

A alocação de memória não tem garantia de sucesso e, em vez disso, pode retornar um ponteiro NULL. Usando o valor retornado, sem verificar se a alocação foi bem-sucedida, invoca comportamento indefinido. Isso geralmente leva a uma falha, mas não há garantia de que uma falha acontecerá, portanto, confiar nisso também pode levar a problemas.

Por exemplo, maneira insegura:

struct SomeStruct *s = malloc(sizeof *s);
s->someValue = 0; /* UNSAFE, because s might be a null pointer */

Maneira segura:

struct SomeStruct *s = malloc(sizeof *s);
if (s)
{
    s->someValue = 0; /* This is safe, we have checked that s is valid */
}

Usando números literais em vez de sizeof ao solicitar memória #

Para uma determinada configuração de compilador/máquina, os tipos têm um tamanho conhecido; entretanto, não existe nenhum padrão que defina que o tamanho de um determinado tipo (diferente de char) seja o mesmo para todas as configurações de compilador/máquina. Se o código usa 4 em vez de sizeof(int) para alocação de memória, pode funcionar na máquina original, mas o código não é necessariamente portátil para outras máquinas ou compiladores. Tamanhos fixos para tipos devem ser substituídos por sizeof(that_type) ou sizeof(*var_ptr_to_that_type).

Alocação não portátil:

 int *intPtr = malloc(4*1000);    /* allocating storage for 1000 int */
 long *longPtr = malloc(8*1000);  /* allocating storage for 1000 long */

Alocação portátil:

 int *intPtr = malloc(sizeof(int)*1000);     /* allocating storage for 1000 int */
 long *longPtr = malloc(sizeof(long)*1000);  /* allocating storage for 1000 long */

Ou, melhor ainda:

 int *intPtr = malloc(sizeof(*intPtr)*1000);     /* allocating storage for 1000 int */
 long *longPtr = malloc(sizeof(*longPtr)*1000);  /* allocating storage for 1000 long */

Perdas de memória #

A falha em desalocar memória usando free leva a um acúmulo de memória não reutilizável, que não é mais usada pelo programa; isso é chamado de vazamento de memória. Vazamentos de memória desperdiçam recursos de memória e podem levar a falhas de alocação.

Erros lógicos #

Todas as alocações devem seguir o mesmo padrão:

  1. Alocação usando malloc (ou calloc)
  2. Uso para armazenar dados
  3. Desalocação usando grátis

Falha em aderir a este padrão, como usar memória após uma chamada para free (dangling pointer) ou antes de uma chamada para malloc (wild ponteiro), chamando free duas vezes (“double free”), etc., geralmente causa uma falha de segmentação e resulta em uma falha do programa.

Esses erros podem ser transitórios e difíceis de depurar – por exemplo, a memória liberada geralmente não é recuperada imediatamente pelo sistema operacional e, portanto, os ponteiros pendentes podem persistir por um tempo e parecer funcionar.

Nos sistemas em que funciona, o Valgrind é uma ferramenta inestimável para identificar qual memória vazou e onde foi originalmente alocada.

Criando ponteiros para empilhar variáveis #

Criar um ponteiro não prolonga a vida útil da variável apontada. Por exemplo:

int* myFunction() 
{
    int x = 10;
    return &x;
}

Aqui, x tem duração de armazenamento automática (comumente conhecida como alocação de pilha). Por ser alocado na pilha, seu tempo de vida é apenas enquanto myFunction estiver em execução; após a saída de myFunction, a variável x é destruída. Esta função obtém o endereço de x (usando &x) e o retorna ao chamador, deixando o chamador com um ponteiro para uma variável inexistente. A tentativa de acessar essa variável invocará comportamento indefinido.

A maioria dos compiladores não limpa um quadro de pilha após a saída da função, portanto, desreferenciar o ponteiro retornado geralmente fornece os dados esperados. No entanto, quando outra função é chamada, a memória apontada pode ser substituída e parece que os dados apontados foram corrompidos.

Para resolver isso, malloc o armazenamento para a variável a ser retornada e retorne um ponteiro para o armazenamento recém-criado ou exija que um ponteiro válido seja passado para a função em vez de retornar um, por exemplo:

#include <stdlib.h>
#include <stdio.h>

int *solution1(void) 
{
    int *x = malloc(sizeof *x);
    if (x == NULL) 
    {
        /* Something went wrong */
        return NULL;
    }

    *x = 10;

    return x;
}

void solution2(int *x) 
{
    /* NB: calling this function with an invalid or null pointer 
       causes undefined behaviour. */

    *x = 10;
}

int main(void) 
{
    { 
        /* Use solution1() */

        int *foo = solution1();  
        if (foo == NULL)
        {
            /* Something went wrong */
            return 1;
        }

        printf("The value set by solution1() is %i\n", *foo);
        /* Will output: "The value set by solution1() is 10" */

        free(foo);    /* Tidy up */
    }

    {
        /* Use solution2() */

        int bar;
        solution2(&bar); 

        printf("The value set by solution2() is %i\n", bar);
        /* Will output: "The value set by solution2() is 10" */
    }

    return 0;
}

Incrementando / decrementando e desreferenciando #

Se você escrever *p++ para incrementar o que é apontado por p, você está errado.

O pós-incremento/decremento é executado antes da desreferenciação. Portanto, esta expressão incrementará o próprio ponteiro p e retornará o que foi apontado por p antes de incrementar sem alterá-lo.

Você deve escrever (*p)++ para incrementar o que é apontado por p.

Esta regra também se aplica à pós-decrementação: *p-- decrementará o próprio ponteiro p, não o que é apontado por p.

Introdução #

Um ponteiro é declarado como qualquer outra variável, exceto que um asterisco (*) é colocado entre o tipo e o nome da variável para indicar que é um ponteiro.

int *pointer; /* inside a function, pointer is uninitialized and doesn't point to any valid object yet */

Para declarar duas variáveis ​​de ponteiro do mesmo tipo, na mesma declaração, use o símbolo de asterisco antes de cada identificador. Por exemplo,

int *iptr1, *iptr2;
int *iptr3,  iptr4;  /* iptr3 is a pointer variable, whereas iptr4 is misnamed and is an int */

O operador address-of ou de referência denotado por um e comercial (&) fornece o endereço de uma determinada variável que pode ser colocada em um ponteiro do tipo apropriado.

int value = 1;
pointer = &value;

O operador de indireção ou desreferência indicado por um asterisco (*) obtém o conteúdo de um objeto apontado por um ponteiro.

printf("Value of pointed to integer: %d\n", *pointer);
/* Value of pointed to integer: 1 */

Se o ponteiro apontar para uma estrutura ou tipo de união, você poderá desreferenciá-lo e acessar seus membros diretamente usando o operador ->:

SomeStruct *s = &someObject;
s->someMember = 5; /* Equivalent to (*s).someMember = 5 */

Em C, um ponteiro é um tipo de valor distinto que pode ser reatribuído e, caso contrário, é tratado como uma variável por si só. Por exemplo, o exemplo a seguir imprime o valor do próprio ponteiro (variável).

printf("Value of the pointer itself: %p\n", (void *)pointer);
/* Value of the pointer itself: 0x7ffcd41b06e4 */
/* This address will be different each time the program is executed */

Como um ponteiro é uma variável mutável, é possível que ele não apontar para um objeto válido, seja por ser definido como nulo

pointer = 0;     /* or alternatively */
pointer = NULL;

ou simplesmente contendo um padrão de bits arbitrário que não é válido Morada. Esta última é uma situação muito ruim, porque não pode ser testado antes que o ponteiro seja desreferenciado, há apenas um teste para o caso de um ponteiro ser nulo:

if (!pointer) exit(EXIT_FAILURE);

Um ponteiro só pode ser desreferenciado se apontar para um objeto válido, caso contrário, o comportamento é indefinido. Muitas implementações modernas podem ajudá-lo levantando algum tipo de erro como uma falha de segmentação e terminar a execução, mas outros podem simplesmente deixar seu programa em um Estado inválido.

O valor retornado pelo operador de desreferência é um alias mutável para a variável original, portanto pode ser alterado, modificando a variável original.

*pointer += 1;
printf("Value of pointed to variable after change: %d\n", *pointer);
/* Value of pointed to variable after change: 2 */

Os ponteiros também são reatribuíveis. Isso significa que um ponteiro apontando para um objeto pode ser usado posteriormente para apontar para outro objeto do mesmo tipo.

int value2 = 10;
pointer = &value2;
printf("Value from pointer: %d\n", *pointer);
/* Value from pointer: 10 */

Como qualquer outra variável, os ponteiros têm um tipo específico. Você não pode atribuir o endereço de um int curto a um ponteiro para um int longo, por instância. Tal comportamento é conhecido como troca de tipo e é proibido em C, embora haja algumas exceções.

Embora o ponteiro deva ser de um tipo específico, a memória alocada para cada tipo de ponteiro é igual à memória usada pelo ambiente para armazenar endereços, em vez do tamanho do tipo apontado.

#include <stdio.h>

int main(void) {
    printf("Size of int pointer: %zu\n", sizeof (int*));      /* size 4 bytes */
    printf("Size of int variable: %zu\n", sizeof (int));      /* size 4 bytes */
    printf("Size of char pointer: %zu\n", sizeof (char*));    /* size 4 bytes */
    printf("Size of char variable: %zu\n", sizeof (char));    /* size 1 bytes */
    printf("Size of short pointer: %zu\n", sizeof (short*));  /* size 4 bytes */
    printf("Size of short variable: %zu\n", sizeof (short));  /* size 2 bytes */
    return 0;
}

(NB: se você estiver usando o Microsoft Visual Studio, que não suporta os padrões C99 ou C11, você deve usar [%Iu](https://msdn.microsoft.com/en-us/library/tcxf1dw6. aspx)1 em vez de %zu no exemplo acima.)

Observe que os resultados acima podem variar de ambiente para ambiente em números, mas todos os ambientes mostrariam tamanhos iguais para diferentes tipos de ponteiro.

Ponteiros e Arrays #

Ponteiros e arrays estão intimamente conectados em C. Arrays em C são sempre mantidos em locais contíguos na memória. A aritmética do ponteiro é sempre dimensionada pelo tamanho do item apontado. Portanto, se tivermos uma matriz de três duplos e um ponteiro para a base, *ptr refere-se ao primeiro duplo, *(ptr + 1) ao segundo, *(ptr + 2) ao terceiro . Uma notação mais conveniente é usar a notação de array [].

double point[3] = {0.0, 1.0, 2.0};
double *ptr = point;

/* prints x 0.0, y 1.0 z 2.0 */ 
printf("x %f y %f z %f\n", ptr[0], ptr[1], ptr[2]);

Então, essencialmente, ptr e o nome da matriz são intercambiáveis. Essa regra também significa que um array decai para um ponteiro quando passado para uma sub-rotina.

double point[3] = {0.0, 1.0, 2.0};

printf("length of point is %s\n", length(point));

/* get the distance of a 3D point from the origin */ 
double length(double *pt)
{
   return sqrt(pt[0] * pt[0] + pt[1] * pt[1] + pt[2] * pt[2])
}

Um ponteiro pode apontar para qualquer elemento em uma matriz ou para o elemento além do último elemento. No entanto, é um erro definir um ponteiro para qualquer outro valor, incluindo o elemento antes da matriz. (A razão é que em arquiteturas segmentadas o endereço antes do primeiro elemento pode cruzar um limite de segmento, o compilador garante que isso não aconteça para o último elemento mais um).

Nota de rodapé 1: as informações de formato da Microsoft podem ser encontradas em printf() e [sintaxe de especificação de formato](https: //msdn.microsoft.com/en-us/library/56e442dc.aspx).

Desreferenciando um ponteiro #

int a = 1;
int *a_pointer = &a;

Para desreferenciar a_pointer e alterar o valor de a, usamos a seguinte operação

*a_pointer = 2;

Isso pode ser verificado usando as seguintes instruções de impressão.

printf("%d\n", a); /* Prints 2 */
printf("%d\n", *a_pointer); /* Also prints 2 */

No entanto, seria um erro desreferenciar um ponteiro NULL ou de outra forma inválido. este

int *p1, *p2;

p1 = (int *) 0xbad;
p2 = NULL;

*p1 = 42;
*p2 = *p1 + 1;

geralmente é comportamento indefinido. p1 não pode ser desreferenciado porque aponta para um endereço 0xbad que pode não ser um endereço válido. Quem sabe o que há? Pode ser a memória do sistema operacional ou a memória de outro programa. O único código de tempo como este é usado, está em desenvolvimento embutido, que armazena informações específicas em endereços codificados. p2 não pode ser desreferenciado porque é NULL, o que é inválido.

Desreferenciando um ponteiro para uma estrutura #

Digamos que temos a seguinte estrutura:

struct MY_STRUCT 
{
    int my_int;
    float my_float;
};

Podemos definir MY_STRUCT para omitir a palavra-chave struct para que não tenhamos que digitar struct MY_STRUCT cada vez que a usamos. Isso, no entanto, é opcional.

typedef struct MY_STRUCT MY_STRUCT;

Se tivermos um ponteiro para uma instância dessa estrutura

MY_STRUCT *instance;

Se esta instrução aparecer no escopo do arquivo, instance será inicializado com um ponteiro nulo quando o programa iniciar. Se esta instrução aparece dentro de uma função, seu valor é indefinido. A variável deve ser inicializada para apontar para uma variável MY_STRUCT válida, ou para um espaço alocado dinamicamente, antes que possa ser desreferenciada. Por exemplo:

MY_STRUCT info = { 1, 3.141593F };
MY_STRUCT *instance = &info;

Quando o ponteiro é válido, podemos desreferenciá-lo para acessar seus membros usando uma das duas notações diferentes:

int a = (*instance).my_int;
float b = instance->my_float;

Embora ambos os métodos funcionem, é melhor usar o operador de seta -> em vez da combinação de parênteses, o operador de desreferência * e o operador de ponto . porque é mais fácil de ler e entender, especialmente com usos aninhados.

Outra diferença importante é mostrada abaixo:

MY_STRUCT copy = *instance;
copy.my_int    = 2;

Neste caso, copy contém uma cópia do conteúdo de instance. Alterar my_int de copy não irá alterá-lo em instance.

MY_STRUCT *ref = instance;
ref->my_int    = 2;

Neste caso, ref é uma referência a instance. Alterar my_int usando a referência irá alterá-lo em instance.

É uma prática comum usar ponteiros para estruturas como parâmetros em funções, em vez das próprias estruturas. Usar os structs como parâmetros de função pode fazer com que a pilha estoure se o struct for grande. Usar um ponteiro para uma estrutura usa apenas espaço de pilha suficiente para o ponteiro, mas pode causar efeitos colaterais se a função alterar a estrutura que é passada para a função.

Ponteiros de função #

Ponteiros também podem ser usados ​​para apontar para funções.

Vamos pegar uma função básica:

int my_function(int a, int b)
{
    return 2 * a + 3 * b;
}

Agora, vamos definir um ponteiro do tipo dessa função:

int (*my_pointer)(int, int);

Para criar um, basta usar este modelo:

return_type_of_func (*my_func_pointer)(type_arg1, type_arg2, ...)

Devemos então atribuir este ponteiro à função:

my_pointer = &my_function;

Este ponteiro agora pode ser usado para chamar a função:

/* Calling the pointed function */
int result = (*my_pointer)(4, 2);

...

/* Using the function pointer as an argument to another function */
void another_function(int (*another_pointer)(int, int))
{
    int a = 4;
    int b = 2;
    int result = (*another_pointer)(a, b);

    printf("%d\n", result);
}

Embora esta sintaxe pareça mais natural e coerente com tipos básicos, atribuir e desreferenciar ponteiros de função não requer o uso de operadores & e *. Portanto, o trecho a seguir é igualmente válido:

/* Attribution without the & operator */
my_pointer = my_function;

/* Dereferencing without the * operator */
int result = my_pointer(4, 2);

Para aumentar a legibilidade dos ponteiros de função, typedefs podem ser usados.

typedef void (*Callback)(int a);

void some_function(Callback callback)
{
    int a = 4;
    callback(a);
}

Outro truque de legibilidade é que o padrão C permite simplificar um ponteiro de função em argumentos como acima (mas não na declaração de variável) para algo que se parece com um protótipo de função; assim, o seguinte pode ser usado de forma equivalente para definições e declarações de funções:

void some_function(void callback(int))
{
    int a = 4;
    callback(a);
}

Ponteiros Const #

Ponteiros únicos #

  • Ponteiro para um int

O ponteiro pode apontar para diferentes inteiros e os int’s podem ser alterados através do ponteiro. Este exemplo de código atribui b para apontar para int b e depois altera o valor de b para 100.

  int b;
  int* p;
  p = &b;    /* OK */
  *p = 100;  /* OK */

  • Ponteiro para um const int

    The pointer can point to different integers but the int’s value can’t be changed through the pointer.

    int b;
const int* p;
p = &b;    /* OK */
*p = 100;  /* Compiler Error */

  • Ponteiro const para int

O ponteiro só pode apontar para um int mas o valor do int pode ser alterado através do ponteiro.

  int a, b;
  int* const p = &b; /* OK as initialisation, no assignment */
  *p = 100;  /* OK */
  p = &a;    /* Compiler Error */
  • Ponteiro const para const int

O ponteiro só pode apontar para um int e o int não pode ser alterado através do ponteiro.

  int a, b;
  const int* const p = &b; /* OK as initialisation, no assignment */
  p = &a;   /* Compiler Error */
  *p = 100; /* Compiler Error */

Comportamento polimórfico com ponteiros vazios #

A função de biblioteca padrão qsort() é um bom exemplo de como se pode usar ponteiros void para fazer uma única função operar em uma grande variedade de tipos diferentes.

void qsort (
    void *base,                                 /* Array to be sorted */
    size_t num,                                 /* Number of elements in array */
    size_t size,                                /* Size in bytes of each element */
    int (*compar)(const void *, const void *)); /* Comparison function for two elements */

A matriz a ser classificada é passada como um ponteiro void, portanto, uma matriz de qualquer tipo de elemento pode ser operada. Os próximos dois argumentos dizem ao qsort() quantos elementos ele deve esperar no array, e quão grande, em bytes, cada elemento é.

O último argumento é um ponteiro de função para uma função de comparação que recebe dois ponteiros vazios. Ao fazer o chamador fornecer esta função, qsort() pode efetivamente classificar elementos de qualquer tipo.

Aqui está um exemplo de tal função de comparação, para comparar floats. Note que qualquer função de comparação passada para qsort() precisa ter este tipo de assinatura. A maneira como ele é polimórfico é lançando os argumentos de ponteiro void para ponteiros do tipo de elemento que desejamos comparar.

int compare_floats(const void *a, const void *b)
{
    float fa = *((float *)a);
    float fb = *((float *)b);
    if (fa < fb)
        return -1;
    if (fa > fb)
        return 1;
    return 0;
}

Como sabemos que o qsort usará essa função para comparar floats, convertemos os argumentos de ponteiro void de volta para ponteiros float antes de desreferenciá-los.

Agora, o uso da função polimórfica qsort em um array “array” com comprimento “len” é muito simples:

qsort(array, len, sizeof(array[0]), compare_floats);

A inicialização do ponteiro é uma boa maneira de evitar ponteiros selvagens. A inicialização é simples e não é diferente da inicialização de uma variável.

#include <stddef.h>

int main()
{
    int *p1 = NULL; 
    char *p2 = NULL;
    float *p3 = NULL;

         /* NULL is a macro defined in stddef.h, stdio.h, stdlib.h, and string.h */

    ...
}

Na maioria dos sistemas operacionais, usar inadvertidamente um ponteiro que foi inicializado como NULL geralmente resultará no travamento imediato do programa, facilitando a identificação da causa do problema. O uso de um ponteiro não inicializado geralmente pode causar bugs difíceis de diagnosticar.

Cuidado: #

O resultado da desreferenciação de um ponteiro NULL é indefinido, portanto não causará necessariamente um travamento mesmo que esse seja o comportamento natural do sistema operacional no qual o programa está sendo executado. As otimizações do compilador podem mascarar a falha, fazer com que a falha ocorra antes ou depois do ponto no código-fonte em que ocorreu a desreferência do ponteiro nulo ou fazer com que partes do código que contém a desreferência do ponteiro nulo sejam removidas inesperadamente do programa. As compilações de depuração geralmente não exibem esses comportamentos, mas isso não é garantido pelo padrão da linguagem. Outros comportamentos inesperados e/ou indesejáveis ​​também são permitidos.

Como NULL nunca aponta para uma variável, para memória alocada ou para uma função, é seguro usá-lo como um valor de guarda.

Cuidado: #

Normalmente NULL é definido como (void *)0. Mas isso não implica que o endereço de memória atribuído seja 0x0. Para obter mais esclarecimentos, consulte C-faq for NULL pointers

Observe que você também pode inicializar ponteiros para conter valores diferentes de NULL.

int i1;

int main()
{
   int *p1 = &i1;
   const char *p2 = "A constant string to point to";
   float *p3 = malloc(10 * sizeof(float));
}

Endereço do Operador ( & ) #

Para qualquer objeto (ou seja, variável, matriz, união, estrutura, ponteiro ou função) o operador de endereço unário pode ser usado para acessar o endereço desse objeto.

Suponha que

int i = 1;              
int *p = NULL;

Então uma instrução p = &i;, copia o endereço da variável i para o ponteiro p.

É expresso como p aponta para i.

printf("%d\n", *p); imprime 1, que é o valor de i.

void* ponteiros como argumentos e valores de retorno para funções padrão #

void* é um tipo catch all para ponteiros para tipos de objetos. Um exemplo disso em uso é com a função malloc, que é declarada como

void* malloc(size_t);

O tipo de retorno pointer-to-void significa que é possível atribuir o valor de retorno de malloc a um ponteiro para qualquer outro tipo de objeto:

int* vector = malloc(10 * sizeof *vector);

Geralmente é considerado uma boa prática não converter explicitamente os valores dentro e fora de ponteiros vazios. No caso específico de malloc() isto é porque com uma conversão explícita, o compilador pode assumir, mas não avisar, um tipo de retorno incorreto para malloc(), se você esquecer de incluir stdlib.h. É também um caso de usar o comportamento correto de ponteiros vazios para melhor se adequar ao princípio DRY (não se repita); compare o acima com o seguinte, em que o código a seguir contém vários locais adicionais desnecessários onde um erro de digitação pode causar problemas:

int* vector = (int*)malloc(10 * sizeof int*);

Da mesma forma, funções como

void* memcpy(void *restrict target, void const *restrict source, size_t size);

têm seus argumentos especificados como void * porque o endereço de qualquer objeto, independentemente do tipo, pode ser passado. Aqui também, uma chamada não deve usar um cast

unsigned char buffer[sizeof(int)];
int b = 67;
memcpy(buffer, &b, sizeof buffer);

Mesmo asterisco, significados diferentes #

Premissa #

A coisa mais confusa em torno da sintaxe de ponteiro em C e C++ é que na verdade existem dois significados diferentes que se aplicam quando o símbolo de ponteiro, o asterisco (*), é usado com uma variável.

Exemplo #

Primeiramente, você usa * para declarar uma variável de ponteiro.

int i = 5;
/* 'p' is a pointer to an integer, initialized as NULL */
int *p = NULL;
/* '&i' evaluates into address of 'i', which then assigned to 'p' */
p = &i;
/* 'p' is now holding the address of 'i' */

Quando você não está declarando (ou multiplicando), * é usado para desreferenciar uma variável de ponteiro:

*p = 123;
/* 'p' was pointing to 'i', so this changes value of 'i' to 123 */

Quando você quer que uma variável de ponteiro existente mantenha o endereço de outra variável, você não usa *, mas faz assim:

p = &another_variable;

Uma confusão comum entre os novatos em programação C surge quando eles declaram e inicializam uma variável de ponteiro ao mesmo tempo.

int *p = &i;

Como int i = 5; e int i; i = 5; dão o mesmo resultado, alguns podem pensar que int *p = &i; e int *p; *p = &i; dê o mesmo resultado também. O fato é, não, int *p; *p = &i; tentará deferir um ponteiro não inicializado que resultará em UB. Nunca use * quando você não estiver declarando nem desreferenciando um ponteiro.

Conclusão #

O asterisco (*) tem dois significados distintos em C em relação aos ponteiros, dependendo de onde é usado. Quando usado em uma declaração de variável, o valor do lado direito do lado igual deve ser um valor de ponteiro para um endereço na memória. Quando usado com uma variável declarada, o asterisco irá desreferenciar o valor do ponteiro, seguindo-o até o local apontado na memória, e permitindo que o valor armazenado ali seja atribuído ou recuperado.

Leve embora

É importante prestar atenção nos seus P e Q, por assim dizer, ao lidar com ponteiros. Esteja ciente de quando estiver usando o asterisco e o que significa quando você o usa lá. Negligenciar esse pequeno detalhe pode resultar em um comportamento com erros e/ou indefinidos com o qual você realmente não quer ter que lidar.
