As estruturas fornecem uma maneira de agrupar um conjunto de variáveis ​​relacionadas de diversos tipos em uma única unidade de memória. A estrutura como um todo pode ser referenciada por um único nome ou ponteiro; os membros da estrutura também podem ser acessados ​​individualmente. Estruturas podem ser passadas para funções e retornadas de funções. Eles são definidos usando a palavra-chave struct.

Estruturas de definição de tipo #

Combinar typedef com struct pode tornar o código mais claro. Por exemplo:

typedef struct 
{
    int x, y;
} Point;

ao contrário de:

struct Point 
{
    int x, y;
};

pode ser declarado como:

Point point;

ao invés de:

struct Point point;

Melhor ainda é usar o seguinte

typedef struct Point Point;

struct Point 
{
    int x, y;
};

para ter vantagem de ambas as definições possíveis de ‘ponto’. Tal declaração é mais conveniente se você aprendeu C++ primeiro, onde você pode omitir a palavra-chave struct se o nome não for ambíguo.

Nomes typedef para structs podem estar em conflito com outros identificadores de outras partes do programa. Alguns consideram isso uma desvantagem, mas para a maioria das pessoas ter um struct e outro identificador o mesmo é bastante perturbador. Notorious é, por exemplo, POSIX’ stat

int stat(const char *pathname, struct stat *buf);

onde você vê uma função stat que tem um argumento que é struct stat.

Estruturas typedef’ sem um nome de tag sempre impõem que toda a declaração struct seja visível para o código que a usa. Toda a declaração struct deve então ser colocada em um arquivo de cabeçalho.

Considerar:

#include "bar.h"

struct foo 
{
    bar *aBar;
};

Assim, com um typedefd struct que não tem nome de tag, o arquivo bar.h sempre tem que incluir toda a definição de bar. Se usarmos

typedef struct bar bar;

em bar.h, os detalhes da estrutura bar podem ser ocultados.

Consulte https://www.wikiod.com/pt/c/tipo-de-letra

Membros de matriz flexível #

Declaração de tipo #

Uma estrutura com pelo menos um membro pode conter adicionalmente um único membro de matriz de comprimento não especificado no final da estrutura. Isso é chamado de membro de array flexível:

struct ex1 
{
    size_t foo;
    int flex[];
};

struct ex2_header 
{
    int foo;
    char bar;
};

struct ex2 
{
    struct ex2_header hdr;
    int flex[];
};

/* Merged ex2_header and ex2 structures. */
struct ex3 
{
    int foo;
    char bar;
    int flex[];
};

Efeitos no tamanho e preenchimento #

Um membro de matriz flexível é tratado como não tendo tamanho ao calcular o tamanho de uma estrutura, embora o preenchimento entre esse membro e o membro anterior da estrutura ainda possa existir:

/* Prints "8,8" on my machine, so there is no padding. */
printf("%zu,%zu\n", sizeof(size_t), sizeof(struct ex1));

/* Also prints "8,8" on my machine, so there is no padding in the ex2 structure itself. */
printf("%zu,%zu\n", sizeof(struct ex2_header), sizeof(struct ex2));

/* Prints "5,8" on my machine, so there are 3 bytes of padding. */
printf("%zu,%zu\n", sizeof(int) + sizeof(char), sizeof(struct ex3));

O membro de array flexível é considerado como tendo um tipo de array incompleto, então seu tamanho não pode ser calculado usando sizeof.

Uso #

Você pode declarar e inicializar um objeto com um tipo de estrutura contendo um membro de matriz flexível, mas não deve tentar inicializar o membro de matriz flexível, pois ele é tratado como se não existisse. É proibido tentar fazer isso e resultarão em erros de compilação.

Da mesma forma, você não deve tentar atribuir um valor a qualquer elemento de um membro de matriz flexível ao declarar uma estrutura dessa maneira, pois pode não haver preenchimento suficiente no final da estrutura para permitir quaisquer objetos exigidos pelo membro de matriz flexível. No entanto, o compilador não necessariamente impedirá que você faça isso, portanto, isso pode levar a um comportamento indefinido.

/* invalid: cannot initialize flexible array member */
struct ex1 e1 = {1, {2, 3}};
/* invalid: hdr={foo=1, bar=2} OK, but cannot initialize flexible array member */
struct ex2 e2 = {{1, 2}, {3}};
/* valid: initialize foo=1, bar=2 members */
struct ex3 e3 = {1, 2};

e1.flex[0] = 3; /* undefined behavior, in my case */
e3.flex[0] = 2; /* undefined behavior again */
e2.flex[0] = e3.flex[0]; /* undefined behavior */

Você pode optar por usar malloc, calloc ou realloc para alocar a estrutura com armazenamento extra e depois liberá-lo, o que permite que você use o membro flexível do array como desejar:

/* valid: allocate an object of structure type `ex1` along with an array of 2 ints */
struct ex1 *pe1 = malloc(sizeof(*pe1) + 2 * sizeof(pe1->flex[0]));

/* valid: allocate an object of structure type ex2 along with an array of 4 ints */
struct ex2 *pe2 = malloc(sizeof(struct ex2) + sizeof(int[4]));

/* valid: allocate 5 structure type ex3 objects along with an array of 3 ints per object */
struct ex3 *pe3 = malloc(5 * (sizeof(*pe3) + sizeof(int[3])));

pe1->flex[0] = 3; /* valid */
pe3[0]->flex[0] = pe1->flex[0]; /* valid */

O ‘hack de estrutura’ #

Membros de matriz flexíveis não existiam antes de C99 e são tratados como erros. Uma solução comum é declarar um array de comprimento 1, uma técnica chamada ‘struct hack’:

struct ex1 
{
    size_t foo;
    int flex[1];
};

Isso afetará o tamanho da estrutura, no entanto, ao contrário de um verdadeiro membro de matriz flexível:

/* Prints "8,4,16" on my machine, signifying that there are 4 bytes of padding. */
printf("%d,%d,%d\n", (int)sizeof(size_t), (int)sizeof(int[1]), (int)sizeof(struct ex1));

Para usar o membro flex como um membro de array flexível, você o alocaria com malloc como mostrado acima, exceto que sizeof(*pe1) (ou o equivalente sizeof(struct ex1)) seria substituído com offsetof(struct ex1, flex) ou a expressão mais longa e independente de tipo sizeof(*pe1)-sizeof(pe1->flex). Alternativamente, você pode subtrair 1 do comprimento desejado do array “flexível”, já que ele já está incluído no tamanho da estrutura, assumindo que o comprimento desejado é maior que 0. A mesma lógica pode ser aplicada aos outros exemplos de uso.

Compatibilidade #

Se desejar compatibilidade com compiladores que não suportam membros de array flexíveis, você pode usar uma macro definida como FLEXMEMB_SIZE abaixo:

#if __STDC_VERSION__ < 199901L
#define FLEXMEMB_SIZE 1
#else
#define FLEXMEMB_SIZE /* nothing */
#endif

struct ex1 
{
    size_t foo;
    int flex[FLEXMEMB_SIZE];
};

Ao alocar objetos, você deve usar o formulário offsetof(struct ex1, flex) para se referir ao tamanho da estrutura (excluindo o membro flexível do array), pois é a única expressão que permanecerá consistente entre compiladores que suportam membros flexíveis do array e compiladores isso não:

struct ex1 *pe10 = malloc(offsetof(struct ex1, flex) + n * sizeof(pe10->flex[0]));

A alternativa é usar o pré-processador para subtrair condicionalmente 1 do comprimento especificado. Devido ao aumento do potencial de inconsistência e erro humano geral neste formulário, mudei a lógica para uma função separada:

struct ex1 *ex1_alloc(size_t n)
{
    struct ex1 tmp;
#if __STDC_VERSION__ < 199901L
    if (n != 0)
        n--;
#endif
    return malloc(sizeof(tmp) + n * sizeof(tmp.flex[0]));
}
...

/* allocate an ex1 object with "flex" array of length 3 */
struct ex1 *pe1 = ex1_alloc(3);

Ponteiros para estruturas #

Quando você tem uma variável contendo um struct, você pode acessar seus campos usando o operador ponto (.). No entanto, se você tiver um ponteiro para um struct, isso não funcionará. Você tem que usar o operador de seta (->) para acessar seus campos. Aqui está um exemplo de uma implementação terrivelmente simples (alguns podem dizer “terrível e simples”) de uma pilha que usa ponteiros para structs e demonstra o operador de seta.

#include <stdlib.h>
#include <stdio.h>

/* structs */
struct stack
{
    struct node *top;
    int size;
};

struct node
{
    int data;
    struct node *next;
};

/* function declarations */
int push(int, struct stack*);
int pop(struct stack*);
void destroy(struct stack*);

int main(void)
{
    int result = EXIT_SUCCESS;

    size_t i;

    /* allocate memory for a struct stack and record its pointer */
    struct stack *stack = malloc(sizeof *stack);
    if (NULL == stack)
    {
        perror("malloc() failed");
        return EXIT_FAILURE;
    }

    /* initialize stack */
    stack->top = NULL;
    stack->size = 0;

    /* push 10 ints */
    {
        int data = 0;
        for(i = 0; i < 10; i++)
        {
            printf("Pushing: %d\n", data);
            if (-1 == push(data, stack))
            {
                perror("push() failed");
                result = EXIT_FAILURE;
                break;
            }

            ++data;
        }
    }

    if (EXIT_SUCCESS == result)
    {
        /* pop 5 ints */
        for(i = 0; i < 5; i++)
        {
            printf("Popped: %i\n", pop(stack));
        }
    }

    /* destroy stack */
    destroy(stack);

    return result;
}

/* Push a value onto the stack. */
/* Returns 0 on success and -1 on failure. */
int push(int data, struct stack *stack)
{
    int result = 0;

    /* allocate memory for new node */
    struct node *new_node = malloc(sizeof *new_node);
    if (NULL == new_node)
    {
        result = -1;
    }
    else
    {
        new_node->data = data;
        new_node->next = stack->top;
        stack->top = new_node;
        stack->size++;
    }

    return result;
}

/* Pop a value off of the stack. */
/* Returns the value popped off the stack */
int pop(struct stack *stack)
{
    struct node *top = stack->top;
    int data = top->data;
    stack->top = top->next;
    stack->size--;
    free(top);
    return data;
}

/* destroy the stack */
void destroy(struct stack *stack)
{
    /* free all pointers */
    while(stack->top != NULL)
    {
        pop(stack);
    }
}

Passando estruturas para funções #

Em C, todos os argumentos são passados ​​para funções por valor, incluindo estruturas. Para estruturas pequenas, isso é bom, pois significa que não há sobrecarga de acessar os dados por meio de um ponteiro. No entanto, também torna muito fácil passar acidentalmente uma estrutura enorme, resultando em desempenho ruim, principalmente se o programador estiver acostumado a outras linguagens nas quais os argumentos são passados ​​por referência.

struct coordinates
{
    int x;
    int y;
    int z;
};

// Passing and returning a small struct by value, very fast
struct coordinates move(struct coordinates position, struct coordinates movement)
{
    position.x += movement.x;
    position.y += movement.y;
    position.z += movement.z;
    return position;
}

// A very big struct
struct lotsOfData
{
    int param1;
    char param2[80000];
};

// Passing and returning a large struct by value, very slow!
// Given the large size of the struct this could even cause stack overflow
struct lotsOfData doubleParam1(struct lotsOfData value)
{
    value.param1 *= 2;
    return value;
}

// Passing the large struct by pointer instead, fairly fast
void doubleParam1ByPtr(struct lotsOfData *value)
{
    value->param1 *= 2;
}

Programação baseada em objetos usando structs #

Structs podem ser usados ​​para implementar código de maneira orientada a objetos. Uma struct é semelhante a uma classe, mas está faltando as funções que normalmente também fazem parte de uma classe, podemos adicioná-las como variáveis ​​de membro de ponteiro de função. Para ficar com nosso exemplo de coordenadas:

/* coordinates.h */

typedef struct coordinate_s 
{
    /* Pointers to method functions */
    void (*setx)(coordinate *this, int x);
    void (*sety)(coordinate *this, int y);
    void (*print)(coordinate *this);
    /* Data */
    int x;
    int y;
} coordinate;

/* Constructor */
coordinate *coordinate_create(void);
/* Destructor */
void coordinate_destroy(coordinate *this);

E agora o arquivo C de implementação:

/* coordinates.c */

#include "coordinates.h"
#include <stdio.h>
#include <stdlib.h>

/* Constructor */
coordinate *coordinate_create(void)
{
    coordinate *c = malloc(sizeof(*c));
    if (c != 0)
    {
        c->setx = &coordinate_setx;
        c->sety = &coordinate_sety;
        c->print = &coordinate_print;
        c->x = 0;
        c->y = 0;
    }
    return c;
}

/* Destructor */
void coordinate_destroy(coordinate *this)
{
    if (this != NULL)
    {
        free(this);  
    }  
}

/* Methods */
static void coordinate_setx(coordinate *this, int x)
{
    if (this != NULL)
    {    
        this->x = x;
    }
}

static void coordinate_sety(coordinate *this, int y)
{
    if (this != NULL)
    {
        this->y = y;
    }
}

static void coordinate_print(coordinate *this)
{
    if (this != NULL)
    {
        printf("Coordinate: (%i, %i)\n", this->x, this->y);
    }
    else
    {
        printf("NULL pointer exception!\n");
    }
}

Um exemplo de uso de nossa classe de coordenadas seria:

/* main.c */

#include "coordinates.h"
#include <stddef.h>

int main(void) 
{
    /* Create and initialize pointers to coordinate objects */
    coordinate *c1 = coordinate_create();
    coordinate *c2 = coordinate_create();
    
    /* Now we can use our objects using our methods and passing the object as parameter */
    c1->setx(c1, 1);
    c1->sety(c1, 2);

    c2->setx(c2, 3);
    c2->sety(c2, 4);

    c1->print(c1);
    c2->print(c2);

    /* After using our objects we destroy them using our "destructor" function */
    coordinate_destroy(c1);
    c1 = NULL;
    coordinate_destroy(c2);
    c2 = NULL;

    return 0;
}

Estruturas de dados simples #

Os tipos de dados de estrutura são uma maneira útil de empacotar dados relacionados e fazer com que eles se comportem como uma única variável.

Declarando um struct simples que contém dois membros int:

struct point 
{
    int x;
    int y; 
};

x e y são chamados de members (ou fields) da estrutura point.

Definindo e usando estruturas:

struct point p;    // declare p as a point struct
p.x = 5;           // assign p member variables
p.y = 3;

As estruturas podem ser inicializadas na definição. O acima é equivalente a:

struct point p = {5, 3};

Estruturas também podem ser inicializadas usando [inicializadores designados][1].

O acesso aos campos também é feito usando o operador .

printf("point is (x = %d, y = %d)", p.x, p.y);

[1]: https://www.wikiod.com/pt/c/inicializacao#Usando inicializadores designados
