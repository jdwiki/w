A maioria das variáveis ​​em C tem um tamanho que é um número inteiro de bytes. Campos de bits são parte de uma estrutura que não necessariamente ocupa um número inteiro de bytes; eles podem qualquer número de bits. Vários campos de bits podem ser empacotados em uma única unidade de armazenamento. Eles fazem parte do padrão C, mas há muitos aspectos que são definidos pela implementação. Eles são uma das partes menos portáteis de C.

Sintaxe #

  • identificador do especificador de tipo: tamanho;

Parâmetros #

Parâmetro Descrição
especificador de tipo signed, unsigned, int ou _Bool
identificador O nome deste campo na estrutura
tamanho O número de bits a serem usados ​​para este campo

Os únicos tipos portáteis para campos de bits são signed, unsigned ou _Bool. O tipo int simples pode ser usado, mas o padrão diz (§6.7.2¶5) … para campos de bits, é definido pela implementação se o especificador int designa o mesmo tipo que signed int ou o mesmo tipo que unsigned int.

Outros tipos inteiros podem ser permitidos por uma implementação específica, mas usá-los não é portátil.

Campos de bits #

Um campo de bits simples pode ser usado para descrever coisas que podem ter um número específico de bits envolvidos.

struct encoderPosition {
   unsigned int encoderCounts : 23;
   unsigned int encoderTurns  : 4;
   unsigned int _reserved     : 5;
};

Neste exemplo consideramos um codificador com 23 bits de precisão simples e 4 bits para descrever multi-voltas. Campos de bits são frequentemente usados ​​na interface com hardware que gera dados associados a um número específico de bits. Outro exemplo pode ser a comunicação com um FPGA, onde o FPGA grava dados em sua memória em seções de 32 bits, permitindo leituras de hardware:

struct FPGAInfo {
    union {
        struct bits {
            unsigned int bulb1On  : 1;
            unsigned int bulb2On  : 1;
            unsigned int bulb1Off : 1;
            unsigned int bulb2Off : 1;
            unsigned int jetOn    : 1;
        };
        unsigned int data;
   };
};

Para este exemplo, mostramos uma construção comumente usada para poder acessar os dados em seus bits individuais ou escrever o pacote de dados como um todo (emulando o que o FPGA pode fazer). Poderíamos então acessar os bits assim:

FPGAInfo fInfo;
fInfo.data = 0xFF34F;
if (fInfo.bits.bulb1On) {
    printf("Bulb 1 is on\n");
}

Isso é válido, mas conforme o padrão C99 6.7.2.1, item 10:

A ordem de alocação de campos de bits dentro de uma unidade (de ordem superior para ordem inferior ou ordem inferior para ordem superior) é definido pela implementação.

Você precisa estar ciente de endianness ao definir campos de bits dessa maneira. Como tal, pode ser necessário usar uma diretiva de pré-processador para verificar a endianness da máquina. Segue um exemplo disso:

typedef union {
    struct bits {
#if defined(WIN32) || defined(LITTLE_ENDIAN)
    uint8_t commFailure :1;
    uint8_t hardwareFailure :1;
    uint8_t _reserved :6;
#else
    uint8_t _reserved :6;
    uint8_t hardwareFailure :1;
    uint8_t commFailure :1;
#endif
    };
    uint8_t data;
} hardwareStatus;

Usando campos de bits como pequenos inteiros #

#include <stdio.h>

int main(void)
{
    /* define a small bit-field that can hold values from 0 .. 7 */
    struct
    {
        unsigned int uint3: 3;
    } small;

    /* extract the right 3 bits from a value */
    unsigned int value = 255 - 2; /* Binary 11111101 */
    small.uint3 = value;          /* Binary      101 */
    printf("%d", small.uint3);

    /* This is in effect an infinite loop */
    for (small.uint3 = 0; small.uint3 < 8; small.uint3++)
    {
        printf("%d\n", small.uint3);
    }

    return 0;
}

Alinhamento do campo de bits #

Os campos de bits permitem declarar campos de estrutura menores que a largura do caractere. Campos de bits são implementados com máscara de nível de byte ou nível de palavra. O exemplo a seguir resulta em uma estrutura de 8 bytes.

struct C
{
    short s;            /* 2 bytes */
    char  c;            /* 1 byte */
    int   bit1 : 1;     /* 1 bit */
    int   nib  : 4;     /* 4 bits padded up to boundary of 8 bits. Thus 3 bits are padded */
    int   sept : 7;     /* 7 Bits septet, padded up to boundary of 32 bits. */
};

Os comentários descrevem um layout possível, mas como o padrão diz o alinhamento da unidade de armazenamento endereçável não é especificado, outros layouts também são possíveis.

Um campo de bits sem nome pode ser de qualquer tamanho, mas não pode ser inicializado ou referenciado.

Um campo de bits de largura zero não pode receber um nome e alinha o próximo campo ao limite definido pelo tipo de dados do campo de bits. Isto é conseguido preenchendo bits entre os campos de bits.

O tamanho da estrutura ‘A’ é de 1 byte.

struct A
{
    unsigned char c1 : 3;
    unsigned char c2 : 4;
    unsigned char c3 : 1;
};

Na estrutura B, o primeiro campo de bits sem nome salta 2 bits; o campo de bit de largura zero depois de c2 faz com que c3 comece a partir do limite de char (então 3 bits são saltados entre c2 e c3. Existem 3 bits de preenchimento após c4. Assim, o tamanho do estrutura é de 2 bytes.

struct B
{
    unsigned char c1 : 1;
    unsigned char    : 2;    /* Skips 2 bits in the layout */
    unsigned char c2 : 2;
    unsigned char    : 0;    /* Causes padding up to next container boundary */ 
    unsigned char c3 : 4;
    unsigned char c4 : 1;
};

Quando os campos de bits são úteis? #

Um campo de bits é usado para agrupar muitas variáveis ​​em um objeto, semelhante a uma estrutura. Isso permite o uso reduzido de memória e é especialmente útil em um ambiente incorporado.

 e.g. consider the following variables having the ranges as given below.
 a --> range 0 - 3
 b --> range 0 - 1
 c --> range 0 - 7
 d --> range 0 - 1
 e --> range 0 - 1

Se declararmos essas variáveis ​​separadamente, cada uma deve ser pelo menos um inteiro de 8 bits e o espaço total necessário será de 5 bytes. Além disso, as variáveis ​​não usarão todo o intervalo de um inteiro sem sinal de 8 bits (0-255). Aqui podemos usar campos de bits.

typedef struct {
   unsigned int a:2;
   unsigned int b:1;
   unsigned int c:3;
   unsigned int d:1;
   unsigned int e:1;
} bit_a;

Os campos de bits na estrutura são acessados ​​da mesma forma que qualquer outra estrutura. O programador precisa tomar cuidado para que as variáveis ​​sejam escritas no intervalo. Se estiver fora do intervalo, o comportamento é indefinido.

int main(void)
{
   bit_a bita_var;
   bita_var.a = 2;              // to write into element a
   printf ("%d",bita_var.a);    // to read from element a.
   return 0;
}

Muitas vezes o programador quer zerar o conjunto de campos de bits. Isso pode ser feito elemento por elemento, mas existe um segundo método. Simplesmente crie uma união da estrutura acima com um tipo sem sinal que seja maior ou igual ao tamanho da estrutura. Em seguida, todo o conjunto de campos de bits pode ser zerado zerando esse inteiro sem sinal.

typedef union {
    struct {
       unsigned int a:2;
       unsigned int b:1;
       unsigned int c:3;
       unsigned int d:1;
       unsigned int e:1;
    };
    uint8_t data;
} union_bit;

O uso é o seguinte

int main(void)
{
   union_bit un_bit;
   un_bit.data = 0x00;        // clear the whole bit-field
   un_bit.a = 2;              // write into element a
   printf ("%d",un_bit.a);    // read from element a.
   return 0;
}

Em conclusão, os campos de bits são comumente usados ​​em situações de restrição de memória onde você tem muitas variáveis ​​que podem assumir intervalos limitados.

Não fazer para campos de bits #

  1. Matrizes de campos de bits, ponteiros para campos de bits e funções que retornam campos de bits não são permitidos.
  2. O operador de endereço (&) não pode ser aplicado a membros de campo de bits.
  3. O tipo de dados de um campo de bits deve ser amplo o suficiente para conter o tamanho do campo.
  4. O operador sizeof() não pode ser aplicado a um campo de bits.
  5. Não há como criar um typedef para um campo de bits isoladamente (embora você certamente possa criar um typedef para uma estrutura contendo campos de bits).
typedef struct mybitfield
{
    unsigned char c1 : 20;   /* incorrect, see point 3 */
    unsigned char c2 : 4;    /* correct */
    unsigned char c3 : 1;
    unsigned int x[10]: 5;   /* incorrect, see point 1 */
} A;

int SomeFunction(void)
{
    // Somewhere in the code
    A a = { … };
    printf("Address of a.c2 is %p\n", &a.c2);      /* incorrect, see point 2 */
    printf("Size of a.c2 is %zu\n", sizeof(a.c2)); /* incorrect, see point 4 */
}
