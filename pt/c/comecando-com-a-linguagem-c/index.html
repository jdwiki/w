Olá Mundo #

Para criar um programa C simples que imprima “Hello, World” na tela, use um editor de texto para criar um novo arquivo (por exemplo, hello. c — a extensão do arquivo deve ser .c) contendo o seguinte código fonte:

#olá.c

#include <stdio.h>

int main(void)
{
    puts("Hello, World");
    return 0;
}

Demonstração ao vivo no Coliru

Vejamos este programa simples linha por linha #

#include <stdio.h>

Esta linha diz ao compilador para incluir o conteúdo do arquivo de cabeçalho da biblioteca padrão stdio.h no programa. Os cabeçalhos geralmente são arquivos que contêm declarações de funções, macros e tipos de dados, e você deve incluir o arquivo de cabeçalho antes de usá-los. Esta linha inclui stdio.h para que possa chamar a função puts().

Veja mais sobre cabeçalhos.

int main(void)

Esta linha inicia a definição de uma função. Ela indica o nome da função (main), o tipo e o número de argumentos que ela espera (void, significando nenhum), e o tipo de valor que esta função retorna (int). A execução do programa começa na função main().

{
    …
}

As chaves são usadas em pares para indicar onde um bloco de código começa e termina. Eles podem ser usados ​​de várias maneiras, mas neste caso eles indicam onde a função começa e termina.

    puts("Hello, World");

Esta linha chama a função puts() para enviar texto para a saída padrão (a tela, por padrão), seguida por uma nova linha. A string a ser emitida é incluída entre parênteses.

"Hello, World" é a string que será escrita na tela. Em C, todo valor literal de string deve estar entre aspas duplas "…".

Veja mais sobre strings.

Em programas C, cada instrução precisa ser terminada por um ponto e vírgula (ou seja, ;).

    return 0;

Quando definimos main(), declaramos como uma função que retorna um int, o que significa que precisa retornar um inteiro. Neste exemplo, estamos retornando o valor inteiro 0, que é usado para indicar que o programa foi encerrado com sucesso. Após a instrução return 0;, o processo de execução terminará.

Editando o programa #

Editores de texto simples incluem vim ou gedit no Linux, ou Notepad no Windows. Os editores de plataforma cruzada também incluem Visual Studio Code ou Sublime Text.

O editor deve criar arquivos de texto simples, não RTF ou qualquer outro formato.

Compilando e executando o programa #

Para executar o programa, este arquivo fonte (hello.c) primeiro precisa ser compilado em um arquivo executável (por exemplo, hello no sistema Unix/Linux ou hello.exe no Windows). Isso é feito usando um compilador para a linguagem C.

Veja mais sobre compilação

Compilar usando o GCC #

GCC (GNU Compiler Collection) é um compilador C amplamente utilizado. Para usá-lo, abra um terminal, use a linha de comando para navegar até o local do arquivo de origem e execute:

gcc hello.c -o hello

Se nenhum erro for encontrado no código-fonte (hello.c), o compilador criará um arquivo binário, cujo nome é dado pelo argumento da opção de linha de comando -o (` olá’). Este é o arquivo executável final.

Também podemos usar as opções de aviso -Wall -Wextra -Werror, que ajudam a identificar problemas que podem fazer com que o programa falhe ou produza resultados inesperados. Eles não são necessários para este programa simples, mas esta é uma maneira de adicioná-los:

gcc -Wall -Wextra -Werror -o hello hello.c

Usando o compilador clang #

Para compilar o programa usando clang você pode usar:

clang -Wall -Wextra -Werror -o hello hello.c

Por design, as opções de linha de comando clang são semelhantes às do GCC.

Usando o compilador Microsoft C na linha de comando #

Se estiver usando o compilador cl.exe da Microsoft em um sistema Windows que suporte Visual Studio e se todas as variáveis ​​de ambiente estiverem definidas, este exemplo C pode ser compilado usando o seguinte comando que irá produzir um executável hello.exe dentro do diretório em que o comando é executado (Existem opções de aviso como /W3 para cl, aproximadamente análogo a -Wall etc para GCC ou clang).

cl hello.c

Executando o programa #

Uma vez compilado, o arquivo binário pode ser executado digitando ./hello no terminal. Após a execução, o programa compilado imprimirá Hello, World, seguido por uma nova linha, no prompt de comando.

Original “Olá, Mundo!” em K&R C #

O seguinte é o original “Hello, World!” programa do livro The C Programming Language de Brian Kernighan e Dennis Ritchie (Ritchie foi o desenvolvedor original da linguagem de programação C na Bell Labs), referido como “K&R”:

#include <stdio.h>

main()
{
    printf("hello, world\n");
}

Observe que a linguagem de programação C não era padronizada no momento da escrita da primeira edição deste livro (1978), e que este programa provavelmente não compilará na maioria dos compiladores modernos, a menos que sejam instruídos a aceitar código C90.

Este primeiro exemplo no livro K&R agora é considerado de baixa qualidade, em parte porque não possui um tipo de retorno explícito para main() e em parte porque não possui uma instrução return. A 2ª edição do livro foi escrita para o antigo padrão C89. Em C89, o tipo de main teria como padrão int, mas o exemplo K&R não retorna um valor definido para o ambiente. Nos padrões C99 e posteriores, o tipo de retorno é obrigatório, mas é seguro deixar de fora a instrução return de main (e apenas main), devido a um caso especial introduzido com C99 5.1.2.2.3 — é equivalente a retornar 0, o que indica sucesso.

A forma recomendada e mais portátil de main para sistemas hospedados é int main (void) quando o programa não usa nenhum argumento de linha de comando, ou int main(int argc, char **argv) quando o programa usa use os argumentos da linha de comando.

C90 §5.1.2.2.3 Término do programa

Um retorno da chamada inicial para a função main é equivalente a chamar a função exit com o valor retornado pela função main como seu argumento. Se a função main executa um retorno que não especifica nenhum valor, o status de término retornado ao ambiente host é indefinido.

C90 §6.6.6.4 A declaração de ‘retorno’

Se uma instrução return sem uma expressão for executada, e o valor da chamada de função é usado pelo chamador, o comportamento é indefinido. Alcançar o } que termina uma função é equivalente a executar uma instrução return sem uma expressão.

C99 §5.1.2.2.3 Término do programa

Se o tipo de retorno da função main for um tipo compatível com int, um retorno do chamada inicial para a função main é equivalente a chamar a função exit com o valor retornado pela função main como seu argumento; atingindo o } que termina o A função main retorna um valor de 0. Se o tipo de retorno não for compatível com int, o o status de encerramento retornado ao ambiente host não é especificado.
