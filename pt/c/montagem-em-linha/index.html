Montagem em linha é a prática de adicionar instruções de montagem no meio do código-fonte C. Nenhum padrão ISO C requer suporte de montagem em linha. Como não é necessário, a sintaxe do assembly embutido varia de compilador para compilador. Embora seja normalmente suportado, há muito poucos motivos para usar a montagem em linha e muitos motivos para não fazê-lo.

Prós #

  1. Desempenho Ao escrever as instruções de montagem específicas para uma operação, você pode obter um desempenho melhor do que o código de montagem gerado pelo compilador. Observe que esses ganhos de desempenho são raros. Na maioria dos casos, você pode obter melhores ganhos de desempenho apenas reorganizando seu código C para que o otimizador possa fazer seu trabalho.
  2. Interface de hardware Driver de dispositivo ou código de inicialização do processador pode precisar de algum código de montagem para acessar os registros corretos e garantir que certas operações ocorram em uma ordem específica com um atraso específico entre as operações.

Contras #

  1. Portabilidade do compilador Não é garantido que a sintaxe para montagem em linha seja a mesma de um compilador para outro. Se você estiver escrevendo código com assembly embutido que deve ser suportado por diferentes compiladores, use macros de pré-processador (#ifdef) para verificar qual compilador está sendo usado. Em seguida, escreva uma seção de montagem em linha separada para cada compilador compatível.
  2. Portabilidade do processador Você não pode escrever um assembly inline para um processador x86 e esperar que ele funcione em um processador ARM. A montagem em linha destina-se a ser escrita para um processador específico ou família de processadores. Se você tiver um assembly embutido que deseja ter suporte em processadores diferentes, use macros de pré-processador para verificar para qual processador o código está sendo compilado e para selecionar a seção de código de assembly apropriada.
  3. Mudanças Futuras de Desempenho A montagem em linha pode ser gravada esperando atrasos com base em uma determinada velocidade de clock do processador. Se o programa for compilado para um processador com um clock mais rápido, o código assembly pode não funcionar conforme o esperado.

Suporte básico a asm do gcc #

O suporte básico de montagem com gcc tem a seguinte sintaxe:

asm [ volatile ] ( AssemblerInstructions )

onde AssemblerInstructions é o código de montagem direto para o processador fornecido. A palavra-chave volátil é opcional e não tem efeito, pois o gcc não otimiza o código em uma instrução asm básica. AssemblerInstructions pode conter várias instruções de montagem. Uma instrução asm básica é usada se você tiver uma rotina asm que deve existir fora de uma função C. O exemplo a seguir é do manual do GCC:

 /* Note that this code will not compile with -masm=intel */
 #define DebugBreak() asm("int $3")

Neste exemplo, você pode usar DebugBreak() em outros lugares em seu código e ele executará a instrução de montagem int $3. Observe que mesmo que o gcc não modifique nenhum código em uma instrução asm básica, o otimizador ainda pode mover instruções asm consecutivas. Se você tiver várias instruções de montagem que devem ocorrer em uma ordem específica, inclua-as em uma instrução asm.

Suporte estendido para asm do gcc #

O suporte estendido a asm no gcc tem a seguinte sintaxe:

asm [volatile] ( AssemblerTemplate
                  : OutputOperands
                  [ : InputOperands
                  [ : Clobbers ] ])
 
 asm [volatile] goto ( AssemblerTemplate
                       :
                       : InputOperands
                       : Clobbers
                       : GotoLabels)

onde AssemblerTemplate é o modelo para a instrução do assembler, OutputOperands são quaisquer variáveis ​​C que podem ser modificadas pelo código assembly, InputOperands são quaisquer variáveis ​​C usadas como parâmetros de entrada, Clobbers são uma lista ou registradores que são modificado pelo código assembly, e GotoLabels são quaisquer rótulos de instruções goto que podem ser usados ​​no código assembly.

O formato estendido é usado em funções C e é o uso mais típico de assembly embutido. Abaixo está um exemplo do kernel Linux para troca de bytes de números de 16 bits e 32 bits para um processador ARM:

/* From arch/arm/include/asm/swab.h in Linux kernel version 4.6.4 */
#if __LINUX_ARM_ARCH__ >= 6

static inline __attribute_const__ __u32 __arch_swahb32(__u32 x)
{
    __asm__ ("rev16 %0, %1" : "=r" (x) : "r" (x));
    return x;
}
#define __arch_swahb32 __arch_swahb32
#define __arch_swab16(x) ((__u16)__arch_swahb32(x))

static inline __attribute_const__ __u32 __arch_swab32(__u32 x)
{
    __asm__ ("rev %0, %1" : "=r" (x) : "r" (x));
    return x;
}
#define __arch_swab32 __arch_swab32

#endif

Cada seção asm usa a variável x como seu parâmetro de entrada e saída. A função C então retorna o resultado manipulado.

Com o formato asm estendido, o gcc pode otimizar as instruções de montagem em um bloco asm seguindo as mesmas regras que usa para otimizar o código C. Se você quiser que sua seção asm permaneça intocada, use a palavra-chave volatile para a seção asm.

Montagem em linha do gcc em macros #

Podemos colocar instruções de montagem dentro de uma macro e use a macro como você chamaria uma função.

#define mov(x,y) \
{ \
    __asm__ ("l.cmov %0,%1,%2" : "=r" (x) : "r" (y), "r" (0x0000000F)); \
}

/// some definition and assignment
unsigned char sbox[size][size];
unsigned char sbox[size][size];

///Using
mov(state[0][1], sbox[si][sj]);

O uso de instruções de montagem embutidas no código C pode melhorar o tempo de execução de um programa. Isso é muito útil em situações críticas de tempo, como algoritmos criptográficos, como AES. Por exemplo, para uma operação de deslocamento simples que é necessária no algoritmo AES, podemos substituir uma instrução de montagem direta Rotate Right pelo operador de deslocamento C >>.

Em uma implementação de ‘AES256’, na função ‘AddRoundKey()’ temos algumas declarações como esta:

unsigned int w;          // 32-bit
unsigned char subkey[4]; // 8-bit, 4*8 = 32 

subkey[0] = w >> 24;     // hold 8 bit, MSB, leftmost group of 8-bits 
subkey[1] = w >> 16;     // hold 8 bit, second group of 8-bit from left    
subkey[2] = w >> 8;      // hold 8 bit, second group of 8-bit from right
subkey[3] = w;           // hold 8 bit, LSB, rightmost group of 8-bits

/// subkey <- w

Eles simplesmente atribuem o valor de bit de w ao array subkey.

Podemos alterar três shift + assign e uma expressão C de atribuição com apenas uma operação de montagem Rotate Right.

__asm__ ("l.ror  %0,%1,%2" : "=r" (* (unsigned int *) subkey)  : "r" (w), "r" (0x10));

O resultado final é exatamente o mesmo.
