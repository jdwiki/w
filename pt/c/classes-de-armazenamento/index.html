Uma classe de armazenamento é usada para definir o escopo de uma variável ou função. Conhecendo a classe de armazenamento de uma variável, podemos determinar o tempo de vida dessa variável durante o tempo de execução do programa.

Sintaxe #

  • [auto|registrar|estático|externo] <Tipo de dados> <Nome da variável>[ = <Valor>];

  • [estático _Thread_local|extern _Thread_local|_Thread_local] <Tipo de dados> <Nome da variável>[ = <Valor>]; /* desde >=C11 */

  • Exemplos:

  • typedef int foo;

  • externo int foo[2];

Os especificadores de classe de armazenamento são as palavras-chave que podem aparecer ao lado do tipo de nível superior de uma declaração. O uso dessas palavras-chave afeta a duração do armazenamento e a vinculação do objeto declarado, dependendo se ele é declarado no escopo do arquivo ou no escopo do bloco:

Palavra-chave Duração do armazenamento Ligação Observações
estático Estático Interno Define a ligação interna para objetos no escopo do arquivo; define a duração do armazenamento estático para objetos no escopo do bloco.
externo Estático Externo Implícito e, portanto, redundante para objetos definidos no escopo do arquivo que também possuem um inicializador. Quando usado em uma declaração no escopo do arquivo sem um inicializador, indica que a definição deve ser encontrada em outra unidade de tradução e será resolvida no momento do link.
automático Automático Irrelevante Implícito e, portanto, redundante para objetos declarados no escopo do bloco.
registrar Automático Irrelevante Relevante apenas para objetos com duração de armazenamento automática. Fornece uma dica de que a variável deve ser armazenada em um registro. Uma restrição imposta é que não se pode usar o operador unário & “endereço de” em tal objeto e, portanto, o objeto não pode ser alias.
typedef Irrelevante Irrelevante Não é um especificador de classe de armazenamento na prática, mas funciona como um do ponto de vista sintático. A única diferença é que o identificador declarado é um tipo, em vez de um objeto.
_Thread_local Tópico Interno/externo Introduzido em C11, para representar a duração de armazenamento de thread. Se usado no escopo do bloco, também deve incluir extern ou static.

Cada objeto tem uma duração de armazenamento associada (independentemente do escopo) e ligação (relevante apenas para declarações no escopo do arquivo), mesmo quando essas palavras-chave são omitidas.

A ordenação dos especificadores de classe de armazenamento em relação aos especificadores de tipo de nível superior (int, unsigned, short etc.) e qualificadores de tipo de nível superior (const, volatile) não é aplicada, portanto ambas as declarações são válidas:

int static const unsigned a = 5; /* bad practice */
static const unsigned int b = 5; /* good practice */

No entanto, é considerado uma boa prática colocar os especificadores de classe de armazenamento primeiro, depois quaisquer qualificadores de tipo, então o especificador de tipo (void, char, int, signed long, unsigned long long, ` longo duplo’…).

Nem todos os especificadores de classe de armazenamento são legais em um determinado escopo:

register int x; /* legal at block scope, illegal at file scope */
auto int y; /* same */

static int z; /* legal at both file and block scope */
extern int a; /* same */

extern int b = 5; /* legal and redundant at file scope, illegal at block scope */

/* legal because typedef is treated like a storage class specifier syntactically */
int typedef new_type_name;

Duração do armazenamento

A duração do armazenamento pode ser estática ou automática. Para um objeto declarado, ele é determinado dependendo de seu escopo e dos especificadores da classe de armazenamento.

Duração do armazenamento estático

Variáveis ​​com duração de armazenamento estático vivem durante toda a execução do programa e podem ser declaradas tanto no escopo do arquivo (com ou sem static) quanto no escopo do bloco (colocando static explicitamente). Eles geralmente são alocados e inicializados pelo sistema operacional na inicialização do programa e recuperados quando o processo termina. Na prática, os formatos executáveis ​​têm seções dedicadas para tais variáveis ​​(data, bss e rodata) e essas seções inteiras do arquivo são mapeadas na memória em determinados intervalos.

Duração de armazenamento de thread

Esta duração de armazenamento foi introduzida em C11. Isso não estava disponível nos padrões C anteriores. Alguns compiladores fornecem uma extensão não padrão com semântica semelhante. Por exemplo, gcc suporta o especificador `__thread` que pode ser usado em padrões C anteriores que não tinham `_Thread_local`. Variáveis ​​com duração de armazenamento de thread podem ser declaradas no escopo do arquivo e no escopo do bloco. Se declarado no escopo do bloco, ele também deve usar o especificador de armazenamento `static` ou `extern`. Seu tempo de vida é toda a execução do *thread* em que foi criado. Este é o único especificador de armazenamento que pode aparecer ao lado de outro especificador de armazenamento.

Duração do armazenamento automático

Variáveis ​​com duração de armazenamento automática só podem ser declaradas no escopo do bloco (diretamente dentro de uma função ou dentro de um bloco dessa função). Eles são utilizáveis ​​apenas no período entre a entrada e a saída da função ou bloco. Uma vez que a variável sai do escopo (seja retornando da função ou deixando o bloco), seu armazenamento é desalocado automaticamente. Quaisquer outras referências à mesma variável de ponteiros são inválidas e levam a um comportamento indefinido.

Em implementações típicas, as variáveis ​​automáticas estão localizadas em certos deslocamentos no quadro de pilha de uma função ou em registradores.

Link externo e interno

A vinculação é relevante apenas para objetos (funções e variáveis) declarados no escopo do arquivo e afeta sua visibilidade em diferentes unidades de tradução. Objetos com ligação externa são visíveis em todas as outras unidades de tradução (desde que a declaração apropriada seja incluída). Objetos com ligação interna não são expostos a outras unidades de tradução e só podem ser usados ​​na unidade de tradução onde estão definidos.

automático #

Essa classe de armazenamento indica que um identificador tem duração de armazenamento automática. Isso significa que uma vez que o escopo no qual o identificador foi definido termina, o objeto denotado pelo identificador não é mais válido.

Como todos os objetos, que não vivem no escopo global ou são declarados como static, têm duração de armazenamento automática por padrão quando definidos, esta palavra-chave é principalmente de interesse histórico e não deve ser usada:

int foo(void)
{
    /* An integer with automatic storage duration. */
    auto int i = 3;

    /* Same */
    int j = 5;

    return 0;
} /* The values of i and j are no longer able to be used. */

registro #

Indica ao compilador que o acesso a um objeto deve ser o mais rápido possível. Se o compilador realmente usa a dica é definido pela implementação; ele pode simplesmente tratá-lo como equivalente a auto.

A única propriedade que é definitivamente diferente para todos os objetos que são declarados com register é que eles não podem ter seus endereços computados. Assim register pode ser uma boa ferramenta para garantir certas otimizações:

register size_t size = 467;

é um objeto que nunca pode alias porque nenhum código pode passar seu endereço para outra função onde ele pode ser alterado inesperadamente.

Esta propriedade também implica que um array

register int array[5];

não pode decair em um ponteiro para seu primeiro elemento (ou seja, array se transformando em &array[0]). Isso significa que os elementos de tal array não podem ser acessados ​​e o próprio array não pode ser passado para uma função.

Na verdade, o único uso legal de um array declarado com uma classe de armazenamento register é o operador sizeof; qualquer outro operador exigiria o endereço do primeiro elemento da matriz. Por essa razão, arrays geralmente não devem ser declarados com a palavra-chave register, pois isso os torna inúteis para qualquer coisa além do cálculo do tamanho de todo o array, o que pode ser feito com a mesma facilidade sem a palavra-chave register.

A classe de armazenamento register é mais apropriada para variáveis ​​que são definidas dentro de um bloco e são acessadas com alta frequência. Por exemplo,

/* prints the sum of the first 5 integers*/
/* code assumed to be part of a function body*/ 
{ 
    register int k, sum;
    for(k = 1, sum = 0; k < 6; sum += k, k++);
        printf("\t%d\n",sum);
}

O operador _Alignof também pode ser usado com arrays register.

estático #

A classe de armazenamento static serve a propósitos diferentes, dependendo da localização da declaração no arquivo:

  1. Para confinar o identificador somente àquela unidade de tradução (escopo=arquivo).

    /* No other translation unit can use this variable. */
static int i;

/* Same; static is attached to the function type of f, not the return type int. */
static int f(int n);

  2. Para salvar dados para uso com a próxima chamada de uma função (escopo=bloco):

     void foo()
 {
     static int a = 0; /* has static storage duration and its lifetime is the
                        * entire execution of the program; initialized to 0 on 
                        * first function call */ 
     int b = 0; /* b has block scope and has automatic storage duration and 
                 * only "exists" within function */

     a += 10;
     b += 10; 

     printf("static int a = %d, int b = %d\n", a, b);
 }

 int main(void)
 {
     int i;
     for (i = 0; i < 5; i++)
     {
         foo();
     }

     return 0;
 }

    This code prints:

     static int a = 10, int b = 10
 static int a = 20, int b = 10
 static int a = 30, int b = 10
 static int a = 40, int b = 10
 static int a = 50, int b = 10

As variáveis ​​estáticas mantêm seu valor mesmo quando chamadas de vários threads diferentes.

  1. Usado em parâmetros de função para indicar que um array deve ter um número mínimo constante de elementos e um parâmetro não nulo:

    /* a is expected to have at least 512 elements. */
void printInts(int a[static 512])
{
    size_t i;
    for (i = 0; i < 512; ++i)
        printf("%d\n", a[i]);
}

O número necessário de itens (ou mesmo um ponteiro não nulo) não é necessariamente verificado pelo compilador, e os compiladores não são obrigados a notificá-lo de forma alguma se você não tiver elementos suficientes. Se um programador passar menos de 512 elementos ou um ponteiro nulo, o resultado será um comportamento indefinido. Como é impossível impor isso, cuidado extra deve ser usado ao passar um valor para esse parâmetro para essa função.

typedef #

Define um novo tipo com base em um tipo existente. Sua sintaxe espelha a de uma declaração de variável.

/* Byte can be used wherever `unsigned char` is needed */
typedef unsigned char Byte;

/* Integer is the type used to declare an array consisting of a single int */
typedef int Integer[1];

/* NodeRef is a type used for pointers to a structure type with the tag "node" */
typedef struct node *NodeRef;

/* SigHandler is the function pointer type that gets passed to the signal function. */
typedef void (*SigHandler)(int);

Embora não seja tecnicamente uma classe de armazenamento, um compilador irá tratá-la como uma, já que nenhuma das outras classes de armazenamento é permitida se a palavra-chave typedef for usada.

Os typedefs são importantes e não devem ser substituídos pela macro #define.

typedef int newType; 
newType *ptr;        // ptr is pointer to variable of type 'newType' aka int

No entanto,

#define int newType
newType *ptr;        // Even though macros are exact replacements to words, this doesn't result to a pointer to variable of type 'newType' aka int

externo #

Usado para declarar um objeto ou função que está definido em outro lugar (e que tem vinculação externa). Em geral, é usado para declarar um objeto ou função a ser usado em um módulo que não é aquele em que o objeto ou função correspondente está definido:

/* file1.c */
int foo = 2;  /* Has external linkage since it is declared at file scope. */

/* file2.c */
#include <stdio.h>
int main(void)
{
    /* `extern` keyword refers to external definition of `foo`. */
    extern int foo;
    printf("%d\n", foo);
    return 0;
}

As coisas ficam um pouco mais interessantes com a introdução da palavra-chave inline no C99:

/* Should usually be place in a header file such that all users see the definition */
/* Hints to the compiler that the function `bar` might be inlined */
/* and suppresses the generation of an external symbol, unless stated otherwise. */
inline void bar(int drink)
{
    printf("You ordered drink no.%d\n", drink);
}

/* To be found in just one .c file.
   Creates an external function definition of `bar` for use by other files.
   The compiler is allowed to choose between the inline version and the external
   definition when `bar` is called. Without this line, `bar` would only be an inline
   function, and other files would not be able to call it. */
extern void bar(int);

_Thread_local #

Este foi um novo especificador de armazenamento introduzido no C11 junto com multi-threading. Isso não está disponível em padrões C anteriores.

Indica duração de armazenamento de thread. Uma variável declarada com o especificador de armazenamento _Thread_local denota que o objeto é local para aquela thread e seu tempo de vida é toda a execução da thread na qual foi criado. Também pode aparecer junto com static ou extern.

#include <threads.h>
#include <stdio.h>
#define SIZE 5

int thread_func(void *id)
{
    /* thread local variable i. */
    static _Thread_local int i;

    /* Prints the ID passed from main() and the address of the i.
     * Running this program will print different addresses for i, showing
     * that they are all distinct objects. */
    printf("From thread:[%d], Address of i (thread local): %p\n", *(int*)id, (void*)&i);

    return 0;
}

int main(void)
{
    thrd_t id[SIZE];
    int arr[SIZE] = {1, 2, 3, 4, 5};

    /* create 5 threads. */
    for(int i = 0; i < SIZE; i++) {
        thrd_create(&id[i], thread_func, &arr[i]);
    }

    /* wait for threads to complete. */
    for(int i = 0; i < SIZE; i++) {
        thrd_join(id[i], NULL);
    }
}
