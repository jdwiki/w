Literais para números, caracteres e strings

  • Literais de ponto flutuante
  • Literais inteiros
  • Literais de string
  • Literais de caracteres

Nesta página

  • Literais de ponto flutuante
  • Literais inteiros
  • Literais de string
  • Literais de caracteres

O termo literal é comumente usado para descrever uma sequência de caracteres em um código C que designa um valor constante, como um número (por exemplo, 0) ou uma string (por exemplo, "C"). Estritamente falando, o padrão usa o termo constante para constantes inteiras, constantes flutuantes, constantes de enumeração e constantes de caracteres, reservando o termo ’literal’ para literais de string, mas isso não é de uso comum.

Literais podem ter prefixos ou sufixos (mas não ambos) que são caracteres extras que podem iniciar ou terminar um literal para alterar seu tipo padrão ou sua representação.

Literais de ponto flutuante #

Os literais de ponto flutuante são usados ​​para representar números reais com sinal. Os sufixos a seguir podem ser usados ​​para especificar o tipo de um literal:

Sufixo Tipo Exemplos
nenhum duplo 3.1415926 -3E6
f, F flutuar 3.1415926f 2.1E-6F
l, L longo duplo 3.1415926L 1E126L

Para usar esses sufixos, o literal deve ser um literal de ponto flutuante. Por exemplo, 3f é um erro, já que 3 é um literal inteiro, enquanto 3.f ou 3.0f estão corretos. Para long double, a recomendação é sempre usar L maiúsculo para facilitar a leitura.

Literais inteiros #

Literais inteiros são usados ​​para fornecer valores integrais. Três bases numéricas são suportadas, indicadas por prefixos:

Base Prefixo Exemplo
Decimal Nenhum 5
Octal 0 0345
Hexadecimal 0x ou 0X 0x12AB, 0X12AB, 0x12ab, 0x12Ab

Observe que essa escrita não inclui nenhum sinal, portanto, literais inteiros são sempre positivos. Algo como -1 é tratado como uma expressão que tem um literal inteiro (1) que é negado com um -

O tipo de um literal inteiro decimal é o primeiro tipo de dados que pode ajustar o valor de int e long. Desde C99, long long também é suportado para literais muito grandes.

O tipo de um literal inteiro octal ou hexadecimal é o primeiro tipo de dados que pode ajustar o valor de int, unsigned, long e unsigned long. Desde C99, long long e unsigned long long também são suportados para literais muito grandes.

Usando vários sufixos, o tipo padrão de um literal pode ser alterado.

Sufixo Explicação
L, l int longo
LL, ll (desde C99) long long int
U, u sem assinatura

Os sufixos U e L/LL podem ser combinados em qualquer ordem e caso. É um erro duplicar sufixos (por exemplo, fornecer dois sufixos U) mesmo que tenham casos diferentes.

Literais de string #

Literais de string são usados ​​para especificar matrizes de caracteres. São sequências de caracteres entre aspas duplas (por exemplo, "abcd" e têm o tipo char*).

O prefixo L torna o literal uma ampla matriz de caracteres, do tipo wchar_t*. Por exemplo, L"abcd".

Desde C11, existem outros prefixos de codificação, semelhantes a L:

prefixo tipo básico codificação
nenhum char dependente da plataforma
L wchar_t dependente da plataforma
u8 char UTF-8
u char16_t geralmente UTF-16
U char32_t geralmente UTF-32

Para os dois últimos, ele pode ser consultado com macros de teste de recurso se a codificação for efetivamente a codificação UTF correspondente.

Literais de caracteres #

Literais de caracteres são um tipo especial de literais inteiros usados ​​para representar um caractere. Eles são colocados entre aspas simples, por exemplo 'a' e ter o tipo int. O valor do literal é um valor inteiro de acordo com o conjunto de caracteres da máquina. Eles não permitem sufixos.

O prefixo L antes de um caractere literal o torna um caractere largo do tipo wchar_t. Da mesma forma, os prefixos u e U do C11 o tornam caracteres largos do tipo char16_t e char32_t, respectivamente.

Ao pretender representar determinados caracteres especiais, como um caractere que não é imprimível, são usadas sequências de escape. As sequências de escape usam uma sequência de caracteres que são traduzidos em outro caractere. Todas as seqüências de escape consistem em dois ou mais caracteres, sendo o primeiro uma barra invertida \. Os caracteres imediatamente após a barra invertida determinam em qual literal de caractere a sequência é interpretada.

Sequência de Fuga Personagem Representado
\b Retrocesso
\f Alimentação de formulários
\n Alimentação de linha (nova linha)
\r Retorno de carro
\t Guia horizontal
\v Guia vertical
\\ Barra invertida
\' Aspas simples
\" Aspas duplas
\? Ponto de interrogação
\nnn Valor octal
\xnn… Valor hexadecimal
Sequência de Fuga Personagem Representado
\a Alerta (bip, campainha)
Sequência de Fuga Personagem Representado
\unnn Nome de caractere universal
\Unnnnnnn Nome de caractere universal

Um nome de caractere universal é um ponto de código Unicode. Um nome de caractere universal pode ser mapeado para mais de um caractere. Os dígitos n são interpretados como dígitos hexadecimais. Dependendo da codificação UTF em uso, uma sequência de nome de caractere universal pode resultar em um ponto de código que consiste em vários caracteres, em vez de um único caractere char normal.

Ao usar a sequência de escape de alimentação de linha na E/S do modo de texto, ela é convertida para a sequência de bytes ou byte de nova linha específica do SO.

A seqüência de escape do ponto de interrogação é usada para evitar trigraphs. Por exemplo, ??/ é compilado como o trígrafo representando um caractere de barra invertida '\', mas usar ?\?/ resultaria no string "??/" .

Pode haver um, dois ou três numerais octais n na sequência de escape do valor octal.
