Sintaxe #

  • int main(int argc, char *argv[])

Parâmetros #

Parâmetro Detalhes
argc contagem de argumentos - inicializado com o número de argumentos separados por espaço fornecidos ao programa a partir da linha de comando, bem como o próprio nome do programa.
argv vetor de argumentos - inicializado em uma matriz de ponteiros char (strings) contendo os argumentos (e o nome do programa) que foram fornecidos na linha de comando.

Um programa C executado em um ‘ambiente hospedado’ (o tipo normal — ao contrário de um ‘ambiente autônomo’) deve ter uma função main. Tradicionalmente é definido como:

int main(int argc, char *argv[])

Note que argv também pode ser, e muitas vezes é, definido como char **argv; o comportamento é o mesmo. Além disso, os nomes dos parâmetros podem ser alterados porque são apenas variáveis ​​locais dentro da função, mas argc e argv são convencionais e você deve usar esses nomes.

Para funções main onde o código não usa nenhum argumento, use int main(void).

Ambos os parâmetros são inicializados quando o programa é iniciado:

  • argc é inicializado com o número de argumentos separados por espaços fornecidos ao programa a partir da linha de comando, bem como o próprio nome do programa.
  • argv é um array de ponteiros char (strings) contendo os argumentos (e o nome do programa) que foi dado na linha de comando.
  • alguns sistemas expandem argumentos de linha de comando “no shell”, outros não. No Unix se o usuário digitar myprogram *.txt o programa receberá uma lista de arquivos de texto; no Windows receberá a string “*.txt”.

Nota: Antes de usar argv, você pode precisar verificar o valor de argc. Em teoria, argc poderia ser 0, e se argc for zero, então não há argumentos e argv[0] (equivalente a argv[argc]) é um ponteiro nulo. Seria um sistema incomum com um ambiente hospedado se você se deparasse com esse problema. Da mesma forma, é possível, embora muito incomum, que não haja informações sobre o nome do programa. Nesse caso, argv[0][0] == '\0' — o nome do programa pode estar vazio.

Suponha que iniciamos o programa assim:

./some_program abba banana mamajam

Então argc é igual a 4, e os argumentos da linha de comando:

  • argv[0] aponta para "./some_program" (o nome do programa) se o nome do programa estiver disponível no ambiente host. Caso contrário, uma string vazia "".
  • argv[1] aponta para "abba",
  • argv[2] aponta para "banana",
  • argv[3] aponta para "mamajam",
  • argv[4] contém o valor NULL.

Veja também O que o main() deve retornar em C e C++ para citações completas do padrão.

Imprime os argumentos em um programa e converte em valores inteiros #

O código a seguir imprimirá os argumentos para o programa e o código tentará converter cada argumento em um número (para um longo):

#include <stdlib.h>
#include <stdio.h>
#include <errno.h>
#include <limits.h>

int main(int argc, char* argv[]) {

    for (int i = 1; i < argc; i++) {
        printf("Argument %d is: %s\n", i, argv[i]);

        errno = 0;
        char *p;
        long argument_numValue = strtol(argv[i], &p, 10);

        if (p == argv[i]) {
            fprintf(stderr, "Argument %d is not a number.\n", i);
        }
        else if ((argument_numValue == LONG_MIN || argument_numValue == LONG_MAX) && errno == ERANGE) {
            fprintf(stderr, "Argument %d is out of range.\n", i);
        }
        else {
            printf("Argument %d is a number, and the value is: %ld\n",
                   i, argument_numValue);
        }
    }
    return 0;
}

Imprimindo os argumentos da linha de comando #

Depois de receber os argumentos, você pode imprimi-los da seguinte forma:

int main(int argc, char **argv)
{
    for (int i = 1; i < argc; i++)
    {
        printf("Argument %d: [%s]\n", i, argv[i]); 
    }
}

Notas

  1. O parâmetro argv também pode ser definido como char *argv[].
  2. argv[0] pode conter o próprio nome do programa (dependendo de como o programa foi executado). O primeiro argumento de linha de comando “real” está em argv[1], e esta é a razão pela qual a variável de loop i é inicializada como 1.
  3. Na instrução print, você pode usar *(argv + i) em vez de argv[i] - ele avalia a mesma coisa, mas é mais detalhado.
  4. Os colchetes ao redor do valor do argumento ajudam a identificar o início e o fim. Isso pode ser inestimável se houver espaços em branco à direita, novas linhas, retornos de carro ou outros caracteres estranhos no argumento. Alguma variante deste programa é uma ferramenta útil para depurar scripts de shell onde você precisa entender o que a lista de argumentos realmente contém (embora existam alternativas de shell simples que são quase equivalentes).

Usando ferramentas GNU getopt #

As opções de linha de comando para aplicativos não são tratadas de forma diferente dos argumentos de linha de comando pela linguagem C. São apenas argumentos que, em um ambiente Linux ou Unix, tradicionalmente começam com um traço (-).

Com a glibc em um ambiente Linux ou Unix, você pode usar as ferramentas getopt para definir, validar e analisar facilmente a linha de comando opções do resto de seus argumentos.

Essas ferramentas esperam que suas opções sejam formatadas de acordo com os GNU Coding Standards, que é uma extensão do que o POSIX especifica para o formato de opções de linha de comando.

O exemplo abaixo demonstra como lidar com opções de linha de comando com as ferramentas GNU getopt.

#include <stdio.h>
#include <getopt.h>
#include <string.h>

/* print a description of all supported options */
void usage (FILE *fp, const char *path)
{
    /* take only the last portion of the path */
    const char *basename = strrchr(path, '/');
    basename = basename ? basename + 1 : path;

    fprintf (fp, "usage: %s [OPTION]\n", basename);
    fprintf (fp, "  -h, --help\t\t"
                 "Print this help and exit.\n");
    fprintf (fp, "  -f, --file[=FILENAME]\t"
                 "Write all output to a file (defaults to out.txt).\n");
    fprintf (fp, "  -m, --msg=STRING\t"
                 "Output a particular message rather than 'Hello world'.\n");
}

/* parse command-line options and print message */
int main(int argc, char *argv[])
{
    /* for code brevity this example just uses fixed buffer sizes for strings */
    char filename[256] = { 0 };
    char message[256] = "Hello world";
    FILE *fp;
    int help_flag = 0;
    int opt;

    /* table of all supported options in their long form.
     * fields: name, has_arg, flag, val
     * `has_arg` specifies whether the associated long-form option can (or, in
     * some cases, must) have an argument. the valid values for `has_arg` are
     * `no_argument`, `optional_argument`, and `required_argument`.
     * if `flag` points to a variable, then the variable will be given a value
     * of `val` when the associated long-form option is present at the command
     * line.
     * if `flag` is NULL, then `val` is returned by `getopt_long` (see below)
     * when the associated long-form option is found amongst the command-line
     * arguments.
     */
    struct option longopts[] = {
        { "help", no_argument, &help_flag, 1 },
        { "file", optional_argument, NULL, 'f' },
        { "msg", required_argument, NULL, 'm' },
        { 0 }
    };

    /* infinite loop, to be broken when we are done parsing options */
    while (1) {
        /* getopt_long supports GNU-style full-word "long" options in addition
         * to the single-character "short" options which are supported by
         * getopt.
         * the third argument is a collection of supported short-form options.
         * these do not necessarily have to correlate to the long-form options.
         * one colon after an option indicates that it has an argument, two
         * indicates that the argument is optional. order is unimportant.
         */
        opt = getopt_long (argc, argv, "hf:ⓜ️", longopts, 0);

        if (opt == -1) {
            /* a return value of -1 indicates that there are no more options */
            break;
        }

        switch (opt) {
        case 'h':
            /* the help_flag and value are specified in the longopts table,
             * which means that when the --help option is specified (in its long
             * form), the help_flag variable will be automatically set.
             * however, the parser for short-form options does not support the
             * automatic setting of flags, so we still need this code to set the
             * help_flag manually when the -h option is specified.
             */
            help_flag = 1;
            break;
        case 'f':
            /* optarg is a global variable in getopt.h. it contains the argument
             * for this option. it is null if there was no argument.
             */
            printf ("outarg: '%s'\n", optarg);
            strncpy (filename, optarg ? optarg : "out.txt", sizeof (filename));
            /* strncpy does not fully guarantee null-termination */
            filename[sizeof (filename) - 1] = '\0';
            break;
        case 'm':
            /* since the argument for this option is required, getopt guarantees
             * that aptarg is non-null.
             */
            strncpy (message, optarg, sizeof (message));
            message[sizeof (message) - 1] = '\0';
            break;
        case '?':
            /* a return value of '?' indicates that an option was malformed.
             * this could mean that an unrecognized option was given, or that an
             * option which requires an argument did not include an argument.
             */
            usage (stderr, argv[0]);
            return 1;
        default:
            break;
        }
    }

    if (help_flag) {
        usage (stdout, argv[0]);
        return 0;
    }

    if (filename[0]) {
        fp = fopen (filename, "w");
    } else {
        fp = stdout;
    }

    if (!fp) {
        fprintf(stderr, "Failed to open file.\n");
        return 1;
    }

    fprintf (fp, "%s\n", message);

    fclose (fp);
    return 0;
}

Ele pode ser compilado com gcc:

gcc example.c -o example

Ele suporta três opções de linha de comando (--help, --file e --msg). Todos também têm uma “forma curta” (-h, -f e -m). As opções “file” e “msg” aceitam argumentos. Se você especificar a opção “msg”, seu argumento será obrigatório.

Os argumentos para opções são formatados como:

  • --option=value (para opções de formato longo)
  • -ovalue ou -o"value" (para opções de formato curto)
