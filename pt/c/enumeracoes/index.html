As enumerações consistem na palavra-chave enum e um identificador opcional seguido por uma lista de enumeradores entre chaves.

Um identificador é do tipo int.

A enumerator-list tem pelo menos um elemento enumerator.

Um enumerador pode opcionalmente ser “atribuído” a uma expressão constante do tipo int.

Um enumerador é constante e compatível com um char, um inteiro com sinal ou um inteiro sem sinal. O que for usado é definido pela implementação. Em qualquer caso, o tipo utilizado deve ser capaz de representar todos os valores definidos para a enumeração em questão.

Se nenhuma expressão constante for “atribuída” a um enumerador e for a 1st entrada em uma lista de enumeradores, ela assumirá o valor de 0, caso contrário, obterá o valor da entrada anterior na lista de enumeradores mais 1.

O uso de várias “atribuições” pode fazer com que enumeradores diferentes da mesma enumeração carreguem os mesmos valores.

Enumeração com valor duplicado #

Um valor de enumerações não precisa ser exclusivo:

#include <stdlib.h> /* for EXIT_SUCCESS */
#include <stdio.h> /* for printf() */


enum Dupes
{
   Base, /* Takes 0 */
   One, /* Takes Base + 1 */
   Two, /* Takes One + 1 */
   Negative = -1,
   AnotherZero /* Takes Negative + 1 == 0, sigh */
};

int main(void)
{
  printf("Base = %d\n", Base);
  printf("One = %d\n", One);
  printf("Two = %d\n", Two);
  printf("Negative = %d\n", Negative);
  printf("AnotherZero = %d\n", AnotherZero);

  return EXIT_SUCCESS;
}

A amostra imprime:

Base = 0
One = 1
Two = 2
Negative = -1
AnotherZero = 0

constante de enumeração sem nome do tipo #

Os tipos de enumeração também podem ser declarados sem dar-lhes um nome:

  enum { buffersize = 256, };
  static unsigned char buffer [buffersize] = { 0 };

Isso nos permite definir constantes de tempo de compilação do tipo int que podem, como neste exemplo, ser usadas como comprimento de array.

Enumeração Simples #

Uma enumeração é um tipo de dados definido pelo usuário que consiste em constantes integrais e cada constante integral recebe um nome. A palavra-chave enum é usada para definir o tipo de dados enumerado.

Se você usar enum em vez de int ou string/char*, você aumentará a verificação em tempo de compilação e evitará erros ao passar constantes inválidas, além de documentar quais valores são válidos para uso.

Exemplo 1 #

enum color{ RED, GREEN, BLUE };

void printColor(enum color chosenColor)
{
    const char *color_name = "Invalid color";
    switch (chosenColor)
    {
       case RED:
         color_name = "RED";
         break;
       
       case GREEN:
        color_name = "GREEN";
        break;    

       case BLUE:
        color_name = "BLUE";
        break;
    }
    printf("%s\n", color_name);
}

Com uma função principal definida da seguinte forma (por exemplo):

int main(){
    enum color chosenColor;
    printf("Enter a number between 0 and 2");
    scanf("%d", (int*)&chosenColor);
    printColor(chosenColor);
    return 0;
}

Exemplo 2 #

(Este exemplo usa inicializadores designados que são padronizados desde C99.)

enum week{ MON, TUE, WED, THU, FRI, SAT, SUN };
     
static const char* const dow[] = { 
  [MON] = "Mon", [TUE] = "Tue", [WED] = "Wed", 
  [THU] = "Thu", [FRI] = "Fri", [SAT] = "Sat", [SUN] = "Sun" };
    
void printDayOfWeek(enum week day) 
{ 
   printf("%s\n", dow[day]);
}

O mesmo exemplo usando verificação de intervalo:

enum week{ DOW_INVALID = -1, 
  MON, TUE, WED, THU, FRI, SAT, SUN, 
  DOW_MAX };
     
static const char* const dow[] = { 
  [MON] = "Mon", [TUE] = "Tue", [WED] = "Wed", 
  [THU] = "Thu", [FRI] = "Fri", [SAT] = "Sat", [SUN] = "Sun" };
    
void printDayOfWeek(enum week day) 
{ 
   assert(day > DOW_INVALID && day < DOW_MAX);
   printf("%s\n", dow[day]);
}

Typedef enum #

Existem várias possibilidades e convenções para nomear uma enumeração. A primeira é usar um tag name logo após a palavra-chave enum.

enum color
{ 
    RED, 
    GREEN, 
    BLUE 
};

Essa enumeração deve sempre ser usada com a palavra-chave e a tag assim:

enum color chosenColor = RED;

Se usarmos typedef diretamente ao declarar o enum, podemos omitir o nome da tag e usar o tipo sem a palavra-chave enum:

typedef enum 
{ 
    RED, 
    GREEN, 
    BLUE 
} color;

color chosenColor = RED;

Mas neste último caso não podemos usá-lo como enum color, pois não usamos o nome da tag na definição. Uma convenção comum é usar ambos, de modo que o mesmo nome possa ser usado com ou sem a palavra-chave enum. Isso tem a vantagem particular de ser compatível com C++

enum color                /* as in the first example */
{ 
    RED, 
    GREEN, 
    BLUE 
};
typedef enum color color; /* also a typedef of same identifier */

color chosenColor  = RED;
enum color defaultColor = BLUE;

Função:

void printColor()
{
    if (chosenColor == RED)
    {
        printf("RED\n");
    }
    else if (chosenColor == GREEN)
    {
        printf("GREEN\n");    
    }
    else if (chosenColor == BLUE)
    {
        printf("BLUE\n");
    }
}

Para saber mais sobre typedef, consulte https://www.wikiod.com/pt/docs/c/2681/typedef
