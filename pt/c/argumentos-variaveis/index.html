Argumentos variáveis são usados ​​por funções da família printf (printf, fprintf, etc) e outras para permitir que uma função seja chamada com um número diferente de argumentos a cada vez, daí o nome varargs.

Para implementar funções usando o recurso de argumentos variáveis, use #include <stdarg.h>.

Para chamar funções que recebem um número variável de argumentos, certifique-se de que haja um protótipo completo com as reticências à direita no escopo: void err_exit(const char *format, ...); por exemplo.

Sintaxe #

  • void va_start(va_list ap, last); /* Inicia o processamento de argumentos variádicos; last é o último parâmetro de função antes das reticências (“…”) */
  • type va_arg(va_list ap, type); /* Obtém o próximo argumento variável na lista; certifique-se de passar o tipo promoted correto */
  • void va_end(va_list ap); /* Finaliza o processamento do argumento */
  • void va_copy(va_list dst, va_list src); /* C99 ou posterior: copia a lista de argumentos, ou seja, a posição atual no processamento de argumentos, para outra lista (por exemplo, para passar os argumentos várias vezes) */

Parâmetros #

Parâmetro Detalhes
va_list ap ponteiro de argumento, posição atual na lista de argumentos variádicos
último nome do último argumento de função não variádica, para que o compilador encontre o local correto para iniciar o processamento de argumentos variádicos; não pode ser declarada como uma variável register, uma função ou um tipo de array
tipo promovido tipo do argumento variadic a ser lido (por exemplo, int para um argumento short int)
va_list src ponteiro do argumento atual para copiar
va_list dst nova lista de argumentos a ser preenchida

As funções va_start, va_arg, va_end e va_copy são na verdade macros.

Certifique-se de sempre chamar va_start primeiro, e apenas uma vez, e chamar va_end por último, e apenas uma vez, e em cada ponto de saída da função. Não fazer isso pode funcionar em seu sistema, mas certamente não é portátil e, portanto, convida a bugs.

Tome cuidado para declarar sua função corretamente, ou seja, com um protótipo, e observe as restrições no argumento last não-variável (não register, não uma função ou tipo de array). Não é possível declarar uma função que recebe apenas argumentos variádicos, pois é necessário pelo menos um argumento não variádico para poder iniciar o processamento de argumentos.

Ao chamar va_arg, você deve solicitar o tipo de argumento promoted, ou seja:

  • short é promovido para int (e unsigned short também é promovido para int a menos que sizeof(unsigned short) == sizeof(int), caso em que é promovido para unsigned int) .
  • float é promovido para double.
  • signed char é promovido para int; unsigned char também é promovido para int a menos que sizeof(unsigned char) == sizeof(int), o que raramente é o caso.
  • char geralmente é promovido para int.
  • Tipos C99 como uint8_t ou int16_t são promovidos de forma semelhante.

O processamento de argumentos variádicos históricos (ou seja, K&R) é declarado em <varargs.h>, mas não deve ser usado, pois é obsoleto. O processamento de argumentos variádicos padrão (o descrito aqui e declarado em <stdarg.h>) foi introduzido em C89; a macro va_copy foi introduzida no C99, mas fornecida por muitos compiladores antes disso.

Usando um argumento de contagem explícito para determinar o comprimento do va_list #

Com qualquer função variádica, a função deve saber interpretar a lista de argumentos variáveis. Com as funções printf() ou scanf(), a string de formato diz à função o que esperar.

A técnica mais simples é passar uma contagem explícita dos outros argumentos (que normalmente são todos do mesmo tipo). Isso é demonstrado na função variadic no código abaixo, que calcula a soma de uma série de inteiros, onde pode haver qualquer número de inteiros, mas essa contagem é especificada como um argumento anterior à lista de argumentos variáveis.

#include <stdio.h>
#include <stdarg.h>

/* first arg is the number of following int args to sum. */
int sum(int n, ...) {
    int sum = 0;
    va_list it; /* hold information about the variadic argument list. */

    va_start(it, n); /* start variadic argument processing */
    while (n--)
      sum += va_arg(it, int); /* get and sum the next variadic argument */
    va_end(it); /* end variadic argument processing */

    return sum;
}

int main(void)
{
    printf("%d\n", sum(5, 1, 2, 3, 4, 5)); /* prints 15 */
    printf("%d\n", sum(10, 5, 9, 2, 5, 111, 6666, 42, 1, 43, -6218)); /* prints 666 */
    return 0;
}

Usando valores de terminador para determinar o final de va_list #

Com qualquer função variádica, a função deve saber interpretar a lista de argumentos variáveis. A abordagem “tradicional” (exemplificada por printf) é especificar o número de argumentos antecipadamente. No entanto, isso nem sempre é uma boa ideia:

/* First argument specifies the number of parameters; the remainder are also int */
extern int sum(int n, ...);

/* But it's far from obvious from the code. */
sum(5, 2, 1, 4, 3, 6)

/* What happens if i.e. one argument is removed later on? */
sum(5, 2, 1, 3, 6)  /* Disaster */

Às vezes é mais robusto adicionar um terminador explícito, exemplificado pelo POSIX execlp() função. Aqui está outra função para calcular a soma de uma série de números double:

#include <stdarg.h>
#include <stdio.h>
#include <math.h>

/* Sums args up until the terminator NAN */
double sum (double x, ...) {
  double sum = 0;
  va_list va;

  va_start(va, x);
  for (; !isnan(x); x = va_arg(va, double)) {
    sum += x;
  }
  va_end(va);

  return sum;
}

int main (void) {
  printf("%g\n", sum(5., 2., 1., 4., 3., 6., NAN));
  printf("%g\n", sum(1, 0.5, 0.25, 0.125, 0.0625, 0.03125, NAN));
}

Bons valores de terminador:

  • integer (supostamente positivo ou não negativo) — 0 ou -1
  • tipos de ponto flutuante — NAN
  • tipos de ponteiro — NULL
  • tipos de enumerador — algum valor especial

Implementando funções com uma interface do tipo printf() #

Um uso comum de listas de argumentos de comprimento variável é implementar funções que são um wrapper fino em torno da família de funções printf(). Um exemplo é um conjunto de funções de relatório de erros.

errmsg.h

#ifndef ERRMSG_H_INCLUDED
#define ERRMSG_H_INCLUDED

#include <stdarg.h>
#include <stdnoreturn.h>    // C11

void verrmsg(int errnum, const char *fmt, va_list ap);
noreturn void errmsg(int exitcode, int errnum, const char *fmt, ...);
void warnmsg(int errnum, const char *fmt, ...);

#endif

Este é um exemplo básico; tais pacotes podem ser muito elaborados. Normalmente, os programadores usarão errmsg() ou warnmsg(), que usam verrmsg() internamente. No entanto, se alguém precisar fazer mais, a função verrmsg() exposta será útil. Você pode evitar expô-lo até precisar dele (YAGNI — você não vai precisar dele), mas a necessidade surgirá eventualmente ( você vai precisar - YAGNI).

errmsg.c

Este código só precisa encaminhar os argumentos variáveis ​​para a função vfprintf() para gerar o erro padrão. Ele também relata a mensagem de erro do sistema correspondente ao número do erro do sistema (errno) passado para as funções.

#include "errmsg.h"
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void
verrmsg(int errnum, const char *fmt, va_list ap)
{
    if (fmt)
        vfprintf(stderr, fmt, ap);
    if (errnum != 0)
        fprintf(stderr, ": %s", strerror(errnum));
    putc('\n', stderr);
}

void
errmsg(int exitcode, int errnum, const char *fmt, ...)
{
    va_list ap;
    va_start(ap, fmt);
    verrmsg(errnum, fmt, ap);
    va_end(ap);
    exit(exitcode);
}

void
warnmsg(int errnum, const char *fmt, ...)
{
    va_list ap;
    va_start(ap, fmt);
    verrmsg(errnum, fmt, ap);
    va_end(ap);
}

Usando errmsg.h

Agora você pode usar essas funções da seguinte forma:

#include "errmsg.h"
#include <errno.h>
#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main(int argc, char **argv)
{
    char buffer[BUFSIZ];
    int fd;
    if (argc != 2)
    {
        fprintf(stderr, "Usage: %s filename\n", argv[0]);
        exit(EXIT_FAILURE);
    }
    const char *filename = argv[1];

    if ((fd = open(filename, O_RDONLY)) == -1)
        errmsg(EXIT_FAILURE, errno, "cannot open %s", filename);
    if (read(fd, buffer, sizeof(buffer)) != sizeof(buffer))
        errmsg(EXIT_FAILURE, errno, "cannot read %zu bytes from %s", sizeof(buffer), filename);
    if (close(fd) == -1)
        warnmsg(errno, "cannot close %s", filename);
    /* continue the program */
    return 0;
}

Se quer o open() ou read() chamadas do sistema falham, o erro é gravado no erro padrão e o o programa sai com o código de saída 1. Se o close() falha na chamada do sistema, o erro é meramente impresso como uma mensagem de aviso e o programa continua.

Verificando o uso correto dos formatos printf()

Se você estiver usando o GCC (o GNU C Compiler, que é parte do GNU Coleção do Compilador), ou usando Clang, então você pode ter o compilador verifique se os argumentos que você passa para as funções de mensagem de erro correspondem o que printf() espera. Como nem todos os compiladores suportam a extensão, ela precisa ser compilada condicionalmente, o que é um pouco complicado. No entanto, a proteção que dá vale o esforço.

Primeiro, precisamos saber como detectar que o compilador é GCC ou Clang emulando o GCC. A resposta é que o GCC define __GNUC__ para indicar isso.

Veja função comum atributos para obter informações sobre os atributos — especificamente o formato atributo.

Reescrito errmsg.h

#ifndef ERRMSG_H_INCLUDED
#define ERRMSG_H_INCLUDED

#include <stdarg.h>
#include <stdnoreturn.h>    // C11

#if !defined(PRINTFLIKE)
#if defined(__GNUC__)
#define PRINTFLIKE(n,m) __attribute__((format(printf,n,m)))
#else
#define PRINTFLIKE(n,m) /* If only */
#endif /* __GNUC__ */
#endif /* PRINTFLIKE */

void verrmsg(int errnum, const char *fmt, va_list ap);
void noreturn errmsg(int exitcode, int errnum, const char *fmt, ...)
        PRINTFLIKE(3, 4);
void warnmsg(int errnum, const char *fmt, ...)
        PRINTFLIKE(2, 3);

#endif

Agora, se você cometer um erro como:

errmsg(EXIT_FAILURE, errno, "Failed to open file '%d' for reading", filename);

(onde o %d deve ser %s), então o compilador irá reclamar:

$ gcc -O3 -g -std=c11 -Wall -Wextra -Werror -Wmissing-prototypes -Wstrict-prototypes \
>     -Wold-style-definition -c erruse.c
erruse.c: In function ‘main’:
erruse.c:20:64: error: format ‘%d’ expects argument of type ‘int’, but argument 4 has type ‘const char *’ [-Werror=format=]
         errmsg(EXIT_FAILURE, errno, "Failed to open file '%d' for reading", filename);
                                                           ~^
                                                           %s
cc1: all warnings being treated as errors
$

Usando uma string de formato #

O uso de uma string de formato fornece informações sobre o número esperado e o tipo dos argumentos variádicos subsequentes de forma a evitar a necessidade de um argumento de contagem explícito ou um valor de terminador.

O exemplo abaixo mostra uma função que envolve a função padrão printf(), permitindo apenas o uso de argumentos variáveis ​​do tipo char, int e double (no formato decimal de ponto flutuante). Aqui, como em printf(), o primeiro argumento para a função wrap é a string de formato. À medida que a string de formato é analisada, a função é capaz de determinar se há outro argumento variável esperado e qual deve ser o tipo.

#include <stdio.h>
#include <stdarg.h>

int simple_printf(const char *format, ...)
{
    va_list ap; /* hold information about the variadic argument list. */
    int printed = 0; /* count of printed characters */

    va_start(ap, format); /* start variadic argument processing */

    while (*format != '\0') /* read format string until string terminator */
    {
        int f = 0;

        if (*format == '%')
        {
            ++format;
            switch(*format)
            {
                case 'c' :
                    f = printf("%d", va_arg(ap, int)); /* print next variadic argument, note type promotion from char to int */
                    break;
                case 'd' :
                    f = printf("%d", va_arg(ap, int)); /* print next variadic argument */
                    break;

                case 'f' :
                    f = printf("%f", va_arg(ap, double)); /* print next variadic argument */
                    break;
                default :
                    f = -1; /* invalid format specifier */
                    break;
            }
        }
        else
        {
            f = printf("%c", *format); /* print any other characters */
        }

        if (f < 0) /* check for errors */
        {
            printed = f;
            break;
        }
        else
        {
            printed += f;
        }
        ++format; /* move on to next character in string */
    }

    va_end(ap); /* end variadic argument processing */

    return printed;
}

int main (int argc, char *argv[])
{
    int x = 40;
    int y = 0;

    y = simple_printf("There are %d characters in this sentence", x);
    simple_printf("\n%d were printed\n", y);
}
