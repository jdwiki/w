Devido às falhas do rand(), muitas outras implementações padrão surgiram ao longo dos anos. Entre eles estão:

  • arc4random() (disponível no OS X e BSD)
  • random() (disponível no Linux)
  • drand48() (disponível em POSIX)

Geração básica de números aleatórios #

A função rand() pode ser usada para gerar um valor inteiro pseudo-aleatório entre 0 e RAND_MAX (0 e RAND_MAX incluídos).

srand(int) é usado para semear o gerador de números pseudo-aleatórios. Cada vez que rand() é propagado com a mesma semente, ele deve produzir a mesma sequência de valores. Ele deve ser propagado apenas uma vez antes de chamar rand(). Ele não deve ser propagado repetidamente ou repetido toda vez que você desejar gerar um novo lote de números pseudo-aleatórios.

A prática padrão é usar o resultado de time(NULL) como uma semente. Se o seu gerador de números aleatórios precisar ter uma sequência determinística, você pode propagar o gerador com o mesmo valor em cada início do programa. Isso geralmente não é necessário para o código de lançamento, mas é útil em execuções de depuração para tornar os erros reproduzíveis.

É aconselhável sempre propagar o gerador, caso não seja propagado, ele se comporta como se fosse propagado com srand(1).

#include <stdio.h>
#include <stdlib.h>
#include <time.h>

int main(void) {
    int i;
    srand(time(NULL));
    i = rand();

    printf("Random value between [0, %d]: %d\n", RAND_MAX, i);
    return 0;
}

Saída possível:

Random value between [0, 2147483647]: 823321433

Notas:

O Padrão C não garante a qualidade da sequência aleatória produzida. No passado, algumas implementações de rand() tinham sérios problemas na distribuição e aleatoriedade dos números gerados. O uso de rand() não é recomendado para necessidades sérias de geração de números aleatórios, como criptografia.

Gerador Congruente Permutado #

Aqui está um gerador de números aleatórios autônomo que não depende de rand() ou funções de biblioteca semelhantes.

Por que você iria querer uma coisa dessas? Talvez você não confie no gerador de números aleatórios integrado da sua plataforma ou talvez queira uma fonte reproduzível de aleatoriedade independente de qualquer implementação de biblioteca específica.

Este código é o PCG32 de pcg-random.org, um RNG moderno, rápido e de uso geral com excelentes propriedades estatísticas. Não é criptograficamente seguro, portanto, não o use para criptografia.

#include <stdint.h>

/* *Really* minimal PCG32 code / (c) 2014 M.E. O'Neill / pcg-random.org
 * Licensed under Apache License 2.0 (NO WARRANTY, etc. see website) */

typedef struct { uint64_t state;  uint64_t inc; } pcg32_random_t;

uint32_t pcg32_random_r(pcg32_random_t* rng) {
    uint64_t oldstate = rng->state;
    /* Advance internal state */
    rng->state = oldstate * 6364136223846793005ULL + (rng->inc | 1);
    /* Calculate output function (XSH RR), uses old state for max ILP */
    uint32_t xorshifted = ((oldstate >> 18u) ^ oldstate) >> 27u;
    uint32_t rot = oldstate >> 59u;
    return (xorshifted >> rot) | (xorshifted << ((-rot) & 31));
}

void pcg32_srandom_r(pcg32_random_t* rng, uint64_t initstate, uint64_t initseq) {
    rng->state = 0U;
    rng->inc = (initseq << 1u) | 1u;
    pcg32_random_r(rng);
    rng->state += initstate;
    pcg32_random_r(rng);
}

E aqui está como chamá-lo:

#include <stdio.h>
int main(void) {
    pcg32_random_t rng; /* RNG state */
    int i;

    /* Seed the RNG */
    pcg32_srandom_r(&rng, 42u, 54u);

    /* Print some random 32-bit integers */
    for (i = 0; i < 6; i++)
        printf("0x%08x\n", pcg32_random_r(&rng));
    
    return 0;
}

Restringir a geração a um determinado intervalo #

Normalmente, ao gerar números aleatórios, é útil gerar inteiros dentro de um intervalo ou um valor p entre 0,0 e 1,0. Embora a operação de módulo possa ser usada para reduzir a semente a um inteiro baixo, ela usa os bits baixos, que geralmente passam por um ciclo curto, resultando em um ligeiro desvio da distribuição se N for grande em proporção a RAND_MAX.

A macro

#define uniform() (rand() / (RAND_MAX + 1.0))

produz um valor p em 0,0 a 1,0 - epsilon, então

i = (int)(uniform() * N)

irá definir i para um número aleatório uniforme dentro do intervalo de 0 a N - 1.

Infelizmente, há uma falha técnica, em que RAND_MAX pode ser maior do que uma variável do tipo double pode representar com precisão. Isso significa que RAND_MAX + 1.0 é avaliado como RAND_MAX e a função ocasionalmente retorna a unidade. Isso é improvável no entanto.

Geração Xorshift #

Uma boa e fácil alternativa aos procedimentos falhos rand() é o xorshift, uma classe de geradores de números pseudo-aleatórios descobertos por George Marsaglia. O gerador xorshift está entre os geradores de números aleatórios não criptograficamente seguros mais rápidos. Mais informações e outras implementações de exemplo estão disponíveis na página xorshift Wikipedia

Exemplo de implementação

#include <stdint.h>

/* These state variables must be initialised so that they are not all zero. */
uint32_t w, x, y, z;

uint32_t xorshift128(void) 
{
    uint32_t t = x;
    t ^= t << 11U;
    t ^= t >> 8U;
    x = y; y = z; z = w;
    w ^= w >> 19U;
    w ^= t;
    return w;
}
