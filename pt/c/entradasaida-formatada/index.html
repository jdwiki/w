Especificadores de conversão para impressão #

Especificador de conversão Tipo de argumento Descrição
i, d int imprime decimal
u não assinado imprime decimal
o não assinado imprime octal
x não assinado imprime hexadecimal, minúscula
X não assinado imprime hexadecimal, maiúsculo
f duplo imprime float com uma precisão padrão de 6, se nenhuma precisão for fornecida (minúsculas usadas para números especiais nan e inf ou infinity)
F duplo imprime float com uma precisão padrão de 6, se nenhuma precisão for fornecida (maiúsculas usadas para números especiais NAN e INF ou INFINITY)
e duplo imprime float com uma precisão padrão de 6, se nenhuma precisão for fornecida, usando notação científica usando mantissa/expoente; expoente minúsculo e números especiais
E duplo imprime float com uma precisão padrão de 6, se nenhuma precisão for fornecida, usando notação científica usando mantissa/expoente; expoente maiúsculo e números especiais
g duplo usa f ou e [veja abaixo]
G duplo usa F ou E [veja abaixo]
a duplo imprime hexadecimal, minúscula
A duplo imprime hexadecimal, maiúsculo
c caractere imprime um único caractere
s caractere* imprime cadeia de caracteres até um terminador NUL, ou truncado para o comprimento dado por precisão, se especificado
p nulo* imprime o valor do ponteiro void; um ponteiro não void deve ser explicitamente convertido (“cast”) para void*; ponteiro apenas para objeto, não um ponteiro de função
% s/d imprime o caractere %
n inteiro * write o número de bytes impressos até agora no int apontado.

Observe que os modificadores de comprimento podem ser aplicados a %n (por exemplo, %hhn indica que a após o especificador de conversão n se aplica a um ponteiro para um caractere assinado argument, de acordo com a ISO/IEC 9899:2011 §7.21.6.1 ¶7).

Observe que as conversões de ponto flutuante se aplicam aos tipos float e double por causa das regras de promoção padrão — §6.5.2.2 Chamadas de função, ¶7 A notação de reticências em um declarador de protótipo de função causa conversão de tipo de argumento para parar após o último parâmetro declarado. O argumento padrão as promoções são executadas em argumentos finais.) Assim, funções como printf() só recebem valores double, mesmo se a variável referenciada for do tipo float.

Com os formatos g e G, a escolha entre a notação e e f (ou E e F) é documentada no padrão C e na especificação POSIX para printf() :

O argumento double representando um número de ponto flutuante deve ser convertido no estilo f ou e (ou no estilo F ou E no caso de um especificador de conversão G), dependendo do valor convertido e a precisão. Deixe P igual à precisão se for diferente de zero, 6 se a precisão for omitida ou 1 se a precisão for zero. Então, se uma conversão com estilo E tiver um expoente de X:

  • Se P > X >= -4, a conversão deve ser com estilo f (ou F) e precisão P - (X+1).
  • Caso contrário, a conversão deve ser com estilo e (ou E) e precisão P - 1.

Finalmente, a menos que o sinalizador ‘#’ seja usado, quaisquer zeros à direita devem ser removidos da parte fracionária do resultado e o caractere de ponto decimal deve ser removido se não houver nenhuma parte fracionária restante.

Imprimindo o valor de um ponteiro para um objeto #

Para imprimir o valor de um ponteiro para um objeto (em oposição a um ponteiro de função) use o especificador de conversão p. Ele é definido para imprimir apenas ponteiros void, então para imprimir o valor de um ponteiro não void ele precisa ser explicitamente convertido (“casted*”) para void*.

#include <stdlib.h> /* for EXIT_SUCCESS */
#include <stdio.h>  /* for printf() */

int main(void)
{
  int i;
  int * p = &i;

  printf("The address of i is %p.\n", (void*) p);

  return EXIT_SUCCESS;
}

Usando <inttypes.h> e uintptr_t #

Outra maneira de imprimir ponteiros em C99 ou posterior usa o tipo uintptr_t e as macros de <inttypes.h>:

#include <inttypes.h> /* for uintptr_t and PRIXPTR */
#include <stdio.h>    /* for printf() */

int main(void)
{
  int  i;
  int *p = &i;

  printf("The address of i is 0x%" PRIXPTR ".\n", (uintptr_t)p);

  return 0;
}

Em teoria, pode não haver um tipo inteiro que possa conter qualquer ponteiro convertido em um inteiro (portanto, o tipo uintptr_t pode não existir). Na prática, existe. Ponteiros para funções não precisam ser conversíveis para o tipo uintptr_t — embora mais uma vez eles sejam conversíveis na maioria das vezes.

Se o tipo uintptr_t existe, também existe o tipo intptr_t. No entanto, não está claro por que você deseja tratar endereços como inteiros assinados.

Histórico pré-padrão: #

Antes de C89 durante os tempos de K&R-C não havia tipo void* (nem cabeçalho <stdlib.h>, nem protótipos e, portanto, nenhuma notação int main(void)), então o ponteiro era convertido em long unsigned int e impresso usando o modificador de comprimento/especificador de conversão lx.

O exemplo abaixo é apenas para fins informativos. Hoje em dia este é um código inválido, o que pode muito bem provocar o infame Comportamento Indefinido.

#include <stdio.h> /* optional in pre-standard C - for printf() */

int main()
{
  int  i;
  int *p = &i;

  printf("The address of i is 0x%lx.\n", (long unsigned) p);

  return 0;
}

Imprimindo a diferença dos valores de dois ponteiros para um objeto #

Subtrair os valores de dois ponteiros para um objeto resulta em um inteiro com sinal *1. Portanto, seria impresso usando pelo menos o especificador de conversão d.

Para ter certeza de que existe um tipo grande o suficiente para conter tal “diferença de ponteiro”, já que C99 <stddef.h> define o tipo ptrdiff_t. Para imprimir um ptrdiff_t use o modificador de comprimento t.

#include <stdlib.h> /* for EXIT_SUCCESS */
#include <stdio.h> /* for printf() */
#include <stddef.h> /* for ptrdiff_t */


int main(void)
{
  int a[2];
  int * p1 = &a[0], * p2 = &a[1];
  ptrdiff_t pd = p2 - p1;

  printf("p1 = %p\n", (void*) p1);
  printf("p2 = %p\n", (void*) p2);
  printf("p2 - p1 = %td\n", pd);

  return EXIT_SUCCESS;
}

O resultado pode ficar assim:

p1 = 0x7fff6679f430
p2 = 0x7fff6679f434
p2 - p1 = 1

Observe que o valor resultante da diferença é dimensionado pelo tamanho do tipo para o qual os ponteiros subtraídos apontam, um int aqui. O tamanho de um int para este exemplo é 4.

*1Se os dois ponteiros a serem subtraídos não apontarem para o mesmo objeto, o comportamento é indefinido.

1: https://www.wikiod.com/pt/c/operadores#Aritmética de ponteiro

A função printf() #

Acessada incluindo <stdio.h>, a função printf() é a principal ferramenta usada para imprimir texto no console em C.

printf("Hello world!");
// Hello world!

Matrizes de caracteres normais e não formatadas podem ser impressas sozinhas, colocando-as diretamente entre os parênteses.

printf("%d is the answer to life, the universe, and everything.", 42);
// 42 is the answer to life, the universe, and everything.

int x = 3;
char y = 'Z';
char* z = "Example";
printf("Int: %d, Char: %c, String: %s", x, y, z);
// Int: 3, Char: Z, String: Example

Alternativamente, números inteiros, números de ponto flutuante, caracteres e mais podem ser impressos usando o caractere de escape %, seguido por um caractere ou sequência de caracteres que denotam o formato, conhecido como especificador de formato.

Todos os argumentos adicionais para a função printf() são separados por vírgulas, e esses argumentos devem estar na mesma ordem que os especificadores de formato. Os argumentos adicionais são ignorados, enquanto os argumentos digitados incorretamente ou a falta de argumentos causarão erros ou comportamento indefinido. Cada argumento pode ser um valor literal ou uma variável.

Após a execução bem-sucedida, o número de caracteres impressos é retornado com o tipo int. Caso contrário, uma falha retornará um valor negativo.

Impressão de sinalizadores de formato #

O padrão C (C11 e C99 também) define os seguintes sinalizadores para printf():

Bandeira Conversões Significado
- tudo O resultado da conversão deve ser justificado à esquerda dentro do campo. A conversão é justificada à direita se este sinalizador não for especificado.
+ numérico assinado O resultado de uma conversão com sinal deve sempre começar com um sinal ( ‘+’ ou ‘-’ ). A conversão deve começar com um sinal somente quando um valor negativo for convertido se este sinalizador não for especificado.
<espaço> numérico assinado Se o primeiro caractere de uma conversão assinada não for um sinal ou se uma conversão assinada resultar em nenhum caractere, um <espaço> deve ser prefixado ao resultado. Isso significa que se os sinalizadores <espaço> e ‘+’ aparecerem, o sinalizador <espaço> deve ser ignorado.
# tudo Especifica que o valor deve ser convertido em um formato alternativo. Para a conversão o, deve aumentar a precisão, se e somente se necessário, para forçar o primeiro dígito do resultado a ser um zero (se o valor e a precisão forem 0, um único 0 será impresso). Para especificadores de conversão x ou X, um resultado diferente de zero deve ter o prefixo 0x (ou 0X). Para especificadores de conversão a, A, e, E, f, F, g e G, o resultado deve sempre conter um caractere de base, mesmo se nenhum dígito seguir o caractere raiz. Sem esse sinalizador, um caractere de raiz aparece no resultado dessas conversões somente se um dígito o seguir. Para especificadores de conversão g e G, os zeros à direita não devem ser removidos do resultado como normalmente são. Para outros especificadores de conversão, o comportamento é indefinido.
0 numérico Para especificadores de conversão d, i, o, u, x, X, a, A, e, E, f, F, g e G, zeros à esquerda (após qualquer indicação de sinal ou base) são usados ​​para preencher o campo largura em vez de realizar preenchimento de espaço, exceto ao converter um infinito ou NaN. Se os sinalizadores ‘0’ e ‘-’ aparecerem, o sinalizador ‘0’ será ignorado. Para especificadores de conversão d, i, o, u, x e X, se uma precisão for especificada, o sinalizador ‘0’ deve ser ignorado. ⌦ Se os sinalizadores ‘0’ e <apóstrofo> aparecerem, os caracteres de agrupamento serão inseridos antes do preenchimento zero. Para outras conversões, o comportamento é indefinido. ⌫

Esses sinalizadores também são suportados pela Microsoft com os mesmos significados.

A especificação POSIX para printf() adiciona:

Bandeira Conversões Significado
' i, d, u, f, F, g, G A parte inteira do resultado de uma conversão decimal deve ser formatada com caracteres de agrupamento de milhares. Para outras conversões o comportamento é indefinido. O caractere de agrupamento não monetário é usado.

Modificadores de comprimento #

Os padrões C99 e C11 especificam os seguintes modificadores de comprimento para printf(); seus significados são:

| Modificador | Modifica | Aplica-se a | |———-|———————|—————- ———————————————–| | hh | d, i, o, u, x ou X | char, signed char ou unsigned char | h | d, i, o, u, x ou X | short int ou unsigned short int | l | d, i, o, u, x ou X | long int ou unsigned long int | l | a, A, e, E, f, F, g ou G | double (para compatibilidade com scanf(); indefinido em C90) | ll | d, i, o, u, x ou X | long long int ou unsigned long long int | j | d, i, o, u, x ou X | intmax_t ou uintmax_t | z | d, i, o, u, x ou X | size_t ou o tipo assinado correspondente (ssize_t em POSIX) | t | d, i, o, u, x ou X | ptrdiff_t ou o tipo inteiro não assinado correspondente | L | a, A, e, E, f, F, g ou G | dupla longa

Se um modificador de comprimento aparecer com qualquer especificador de conversão diferente do especificado acima, o comportamento será indefinido.

Microsoft especifica alguns modificadores de comprimento diferentes e explicitamente não oferece suporte a hh, j, z ou t .

| Modificador | Modifica | Aplica-se a | |———-|———————|—————- ———————————————–| | I32 | d, i, o, x ou X | __int32 | I32 | o, u, x ou X | não assinado __int32 | I64 | d, i, o, x ou X | __int64 | I64 | o, u, x ou X | não assinado __int64 | eu | d, i, o, x ou X | ptrdiff_t (ou seja, __int32 em plataformas de 32 bits, __int64 em plataformas de 64 bits) | eu | o, u, x ou X | size_t (ou seja, unsigned __int32 em plataformas de 32 bits, unsigned __int64 em plataformas de 64 bits) | l ou L | a, A, e, E, f, g ou G | long double (No Visual C++, embora long double seja um tipo distinto, ele tem a mesma representação interna que double.) | l ou w | c ou C | Caractere largo com funções printf e wprintf. (Um especificador de tipo lc, lC, wc ou wC é sinônimo de C em funções printf e com c em funções wprintf.) | l ou w | s, S ou Z | String de caracteres largos com funções printf e wprintf. (Um especificador de tipo ls, lS, ws ou wS é sinônimo de S em funções printf e com s em funções wprintf.)

Observe que os especificadores de conversão C, S e Z e os modificadores de comprimento I, I32, I64 e w são extensões da Microsoft. Tratar l como um modificador para long double em vez de double é diferente do padrão, embora seja difícil identificar a diferença, a menos que long double tenha uma representação diferente de double.
