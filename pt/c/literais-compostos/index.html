Sintaxe #

  • (tipo){ lista de inicializadores }

O padrão C diz em C11-§6.5.2.5/3:

Uma expressão postfix que consiste em um nome de tipo entre parênteses seguido por uma lista de inicializadores entre chaves é um literal composto. Ele fornece um objeto sem nome cujo valor é fornecido pela lista de inicializadores.99)

e a nota de rodapé 99 diz:

Observe que isso difere de uma expressão de conversão. Por exemplo, uma conversão especifica uma conversão para tipos escalares ou apenas void, e o resultado de uma expressão de conversão não é um lvalue.

Observe que:

Os literais de string e os literais compostos com tipos qualificados para const não precisam designar objetos distintos.101)

101) Isso permite que implementações compartilhem armazenamento para literais de string e literais compostos constantes com representações iguais ou sobrepostas.

Exemplo é dado no padrão: C11-§6.5.2.5/13:

Como literais de string, literais compostos qualificados para const podem ser colocados em memória somente leitura e podem até ser compartilhados. Por exemplo,

(const char []){“abc”} == “abc” pode render 1 se o armazenamento dos literais for compartilhado.

Definição/inicialização de literais compostos #

Um literal composto é um objeto sem nome que é criado no escopo onde está definido. O conceito foi introduzido pela primeira vez no padrão C99. Um exemplo de literal composto é

Exemplos do padrão C, C11-§6.5.2.5/9: #

int *p = (int [2]){ 2, 4 };

p é inicializado com o endereço do primeiro elemento de um array sem nome de dois ints.

O literal composto é um lvalue. A duração do armazenamento do objeto sem nome é estática (se o literal aparecer no escopo do arquivo) ou automático (se o literal aparecer no escopo do bloco) e, no último caso, o tempo de vida do objeto termina quando o controle sai do bloco delimitador.

nulo f(void) { int p; /…/ p = (int [2]){ p }; /…/ } p é atribuído o endereço do primeiro elemento de um array de dois ints, o primeiro tendo o valor apontado anteriormente por p e o segundo, zero.[…]

Aqui p permanece válido até o final do bloco.

Literal composto com designadores #

(também de C11)

  struct point {
    unsigned x;
    unsigned y;
  };

  extern void drawline(struct point, struct point);

 // used somewhere like this
 drawline((struct point){.x=1, .y=1}, (struct point){.x=3, .y=4});

Uma função fictícia drawline recebe dois argumentos do tipo struct ponto». O primeiro tem valores de coordenadas x == 1ey == 1, enquanto o segundo tem x == 3ey == 4`

Literal composto sem especificar o tamanho do array #

int *p = (int []){ 1, 2, 3};

Neste caso, o tamanho do array não é especificado, então ele será determinado pelo comprimento do inicializador.

Literal composto com comprimento do inicializador menor que o tamanho do array especificado #

int *p = (int [10]){1, 2, 3};

o restante dos elementos do literal composto será inicializado com 0 implicitamente.

Literal composto somente leitura #

Observe que um literal composto é um lvalue e, portanto, seus elementos podem ser modificáveis. Um literal composto somente leitura pode ser especificado usando o qualificador const como (const int[]){1,2}.

Literal composto contendo expressões arbitrárias #

Dentro de uma função, um literal composto, como em qualquer inicialização desde C99, pode ter expressões arbitrárias.

void foo()
{
    int *p;
    int i = 2; j = 5;
    /*...*/
    p = (int [2]){ i+j, i*j };
    /*...*/
}
