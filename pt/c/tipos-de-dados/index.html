Tipos inteiros de largura fixa (desde C99) #

O cabeçalho <stdint.h> fornece várias definições de tipo inteiro de largura fixa. Esses tipos são opcionais e fornecidos apenas se a plataforma tiver um tipo inteiro com a largura correspondente e se o tipo com sinal correspondente tiver uma representação em complemento de dois de valores negativos.

Consulte a seção de comentários para dicas de uso de tipos de largura fixa.

/* commonly used types include */
uint32_t u32 = 32; /* exactly 32-bits wide */

uint8_t u8 = 255;  /* exactly 8-bits wide */

int64_t i64 = -65  /* exactly 64 bit in two's complement representation */

Interpretando Declarações #

Uma peculiaridade sintática distinta de C é que as declarações espelham o uso do objeto declarado como seria em uma expressão normal.

O seguinte conjunto de operadores com precedência e associatividade idênticas são reutilizados em declaradores, a saber:

  • o operador unário * “dereference” que denota um ponteiro;
  • o operador binário [] “assinatura de matriz” que denota uma matriz;
  • o operador (1+n)-ary () “chamada de função” que denota uma função;
  • os parênteses de agrupamento () que substituem a precedência e associatividade do resto dos operadores listados.

Os três operadores acima têm a seguinte precedência e associatividade:

Operador Precedência Relativa Associatividade
[] (assinatura de matriz) 1 Da esquerda para a direita
() (chamada de função) 1 Da esquerda para a direita
* (desreferência) 2 Da direita para a esquerda

Ao interpretar declarações, deve-se começar do identificador para fora e aplicar os operadores adjacentes na ordem correta conforme a tabela acima. Cada aplicação de um operador pode ser substituída pelas seguintes palavras em inglês:

| Expressão | Interpretação | | ————————- | ————————————————– - | | coisa[X] | uma matriz de tamanho ‘X’ de… | | coisa(t1, t2, t3) | uma função pegando t1, t2, t3 e retornando… | | *coisa | um ponteiro para… |

Segue-se que o início da interpretação em inglês sempre começará com o identificador e terminará com o tipo que fica do lado esquerdo da declaração.

Exemplos

char *names[20];

[] tem precedência sobre *, então a interpretação é: names é um array de tamanho 20 de um ponteiro para char.

char (*place)[10];

No caso de usar parênteses para substituir a precedência, o * é aplicado primeiro: place é um ponteiro para um array de tamanho 10 de char.

int fn(long, short);

Não há precedência para se preocupar aqui: fn é uma função que recebe long, short e retorna int.

int *fn(void);

O () é aplicado primeiro: fn é uma função que recebe void e retorna um ponteiro para int.

int (*fp)(void);

Substituindo a precedência de (): fp é um ponteiro para uma função que recebe void e retorna int.

int arr[5][8];

Matrizes multidimensionais não são uma exceção à regra; os operadores [] são aplicados na ordem da esquerda para a direita de acordo com a associatividade na tabela: arr é um array de tamanho 5 de um array de tamanho 8 de int.

int **ptr;

Os dois operadores de desreferência têm precedência igual, portanto, a associatividade entra em vigor. Os operadores são aplicados na ordem da direita para a esquerda: ptr é um ponteiro para um ponteiro para um int.

Declarações múltiplas

A vírgula pode ser usada como separador (*não* agindo como o operador de vírgula) para delimitar várias declarações em uma única instrução. A declaração a seguir contém cinco declarações: 
int fn(void), *ptr, (*fp)(int), arr[10][20], num;

Os objetos declarados no exemplo acima são:

  • fn: uma função que recebe void e retorna int;
  • ptr: um ponteiro para um int;
  • fp: um ponteiro para uma função que recebe int e retorna int;
  • arr: um array de tamanho 10 de um array de tamanho 20 de int;
  • num'': int’'

Interpretação alternativa

Como as declarações espelham o uso, uma declaração também pode ser interpretada em termos dos operadores que podem ser aplicados sobre o objeto e o tipo final resultante dessa expressão. O tipo que fica do lado esquerdo é o resultado final obtido após a aplicação de todos os operadores.

/*
 * Subscripting "arr" and dereferencing it yields a "char" result.
 * Particularly: *arr[5] is of type "char".
 */
char *arr[20];

/*
 * Calling "fn" yields an "int" result.
 * Particularly: fn('b') is of type "int".
 */
int fn(char);

/*
 * Dereferencing "fp" and then calling it yields an "int" result.
 * Particularly: (*fp)() is of type "int".
 */
int (*fp)(void);

/*
 * Subscripting "strings" twice and dereferencing it yields a "char" result.
 * Particularly: *strings[5][15] is of type "char"
 */
char *strings[10][20];

Tipos inteiros e constantes #

Os inteiros assinados podem ser desses tipos (o int depois de short ou long é opcional):

signed char c = 127; /* required to be 1 byte, see remarks for further information. */
signed short int si = 32767; /* required to be at least 16 bits. */
signed int i = 32767; /* required to be at least 16 bits */
signed long int li = 2147483647; /* required to be at least 32 bits. */

signed long long int li = 2147483647; /* required to be at least 64 bits */

Cada um desses tipos inteiros assinados tem uma versão não assinada.

unsigned int i = 65535;
unsigned short = 2767;
unsigned char = 255;

Para todos os tipos, exceto char, a versão signed é assumida se a parte signed ou unsigned for omitida. O tipo char constitui um terceiro tipo de caractere, diferente de signed char e unsigned char e a assinatura (ou não) depende da plataforma.

Diferentes tipos de constantes inteiras (chamadas literais no jargão C) podem ser escritas em diferentes bases e larguras diferentes, com base em seu prefixo ou sufixo.

/* the following variables are initialized to the same value: */
int d = 42;   /* decimal constant (base10) */
int o = 052;  /* octal constant (base8) */
int x = 0xaf; /* hexadecimal constants (base16) */
int X = 0XAf; /* (letters 'a' through 'f' (case insensitive) represent 10 through 15) */

Constantes decimais são sempre assinadas. As constantes hexadecimais começam com 0x ou 0X e as constantes octais começam apenas com um 0. Os dois últimos são signed ou unsigned dependendo se o valor se encaixa no tipo assinado ou não.

/* suffixes to describe width and signedness : */
long int i = 0x32; /* no suffix represent int, or long int */
unsigned int ui = 65535u; /* u or U represent unsigned int, or long int */
long int li = 65536l; /* l or L represent long int */

Sem um sufixo, a constante tem o primeiro tipo que se ajusta ao seu valor, ou seja, uma constante decimal maior que INT_MAX é do tipo long se possível, ou long long caso contrário.

O arquivo de cabeçalho <limits.h> descreve os limites de inteiros como segue. Seus valores definidos pela implementação devem ser iguais ou maiores em magnitude (valor absoluto) aos mostrados abaixo, com o mesmo sinal.

Macro Tipo Valor
CHAR_BIT menor objeto que não é um campo de bits (byte) 8
SCHAR_MIN caractere assinado -127 / -(27 - 1)
SCHAR_MAX caractere assinado +127 / 27 - 1
UCHAR_MAX caracter não assinado 255 / 28 - 1
CHAR_MIN char Veja abaixo
CHAR_MAX char Veja abaixo
SHRT_MIN int curto -32767 / -(215 - 1)
SHRT_MAX int curto +32767 / 215 - 1
USHRT_MAX int curto não assinado 65535 / 216 - 1
INT_MIN int -32767 / -(215 - 1)
INT_MAX int +32767 / 215 - 1
UINT_MAX int não assinado 65535 / 216 - 1
LONG_MIN int longo -2147483647 / -(231 - 1)
LONG_MAX int longo +2147483647 / 231 - 1

Macro Tipo Valor

Se o valor de um objeto do tipo char sign-extends quando usado em uma expressão, o valor de CHAR_MIN deve ser o mesmo de SCHAR_MIN e o valor de CHAR_MAX deve ser o mesmo de CHAR_MAX SCHAR_MAX . Se o valor de um objeto do tipo char não estender sinal quando usado em uma expressão, o valor de CHAR_MIN deve ser 0 e o valor de CHAR_MAX deve ser o mesmo de UCHAR_MAX.

O padrão C99 adicionou um novo cabeçalho, <stdint.h>, que contém definições para inteiros de largura fixa. Veja o exemplo de inteiro de largura fixa para uma explicação mais detalhada.

Constantes de ponto flutuante #

A linguagem C tem três tipos de ponto flutuante reais obrigatórios, float, double e long double.

float f = 0.314f;        /* suffix f or F denotes type float */
double d = 0.314;        /* no suffix denotes double */
long double ld = 0.314l; /* suffix l or L denotes long double */

/* the different parts of a floating point definition are optional */
double x = 1.; /* valid, fractional part is optional */
double y = .1; /* valid, whole-number part is optional */

/* they can also defined in scientific notation */
double sd = 1.2e3; /* decimal fraction 1.2 is scaled by 10^3, that is 1200.0 */

O cabeçalho <float.h> define vários limites para operações de ponto flutuante.

A aritmética de ponto flutuante é definida pela implementação. No entanto, a maioria das plataformas modernas (arm, x86, x86_64, MIPS) usam operações de ponto flutuante IEEE 754.

C também tem três tipos de ponto flutuante complexos opcionais que são derivados do acima.

Literais de String #

Um literal de string em C é uma sequência de caracteres, terminada por um zero literal.

char* str = "hello, world"; /* string literal */

/* string literals can be used to initialize arrays */
char a1[] = "abc"; /* a1 is char[4] holding {'a','b','c','\0'} */
char a2[4] = "abc"; /* same as a1 */
char a3[3] = "abc"; /* a1 is char[3] holding {'a','b','c'}, missing the '\0' */

Literais de string são não modificáveis (e, na verdade, podem ser colocados em memória somente leitura, como .rodata). A tentativa de alterar seus valores resulta em um comportamento indefinido.

char* s = "foobar";
s[0] = 'F'; /* undefined behaviour */

/* it's good practice to denote string literals as such, by using `const` */
char const* s1 = "foobar";
s1[0] = 'F'; /* compiler error! */

Vários literais de string são concatenados em tempo de compilação, o que significa que você pode escrever construções como essas.

/* only two narrow or two wide string literals may be concatenated */
char* s = "Hello, " "World";

/* since C99, more than two can be concatenated */
/* concatenation is implementation defined */
char* s1 = "Hello" ", " "World";

/* common usages are concatenations of format strings */
char* fmt = "%" PRId16; /* PRId16 macro since C99 */

Literais de string, assim como constantes de caracteres, suportam diferentes conjuntos de caracteres.

/* normal string literal, of type char[] */
char* s1 = "abc";

/* wide character string literal, of type wchar_t[] */
wchar_t* s2 = L"abc";

/* UTF-8 string literal, of type char[] */
char* s3 = u8"abc";

/* 16-bit wide string literal, of type char16_t[] */
char16_t* s4 = u"abc";

/* 32-bit wide string literal, of type char32_t[] */
char32_t* s5 = U"abc";
