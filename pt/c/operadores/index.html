Um operador em uma linguagem de programação é um símbolo que diz ao compilador ou intérprete para realizar uma operação matemática, relacional ou lógica específica e produzir um resultado final.

C tem muitos operadores poderosos. Muitos operadores C são operadores binários, o que significa que eles têm dois operandos. Por exemplo, em a/b, / é um operador binário que aceita dois operandos (a, b). Existem alguns operadores unários que recebem um operando (por exemplo: ~, ++), e apenas um operador ternário ? :.

Sintaxe #

  • operador expr1
  • operador expr2
  • expr1 operador expr2 -expr1? expr2: expr3

Os operadores têm uma aridade, uma precedência e uma associatividade.

  • Arity indica o número de operandos. Em C, existem três diferentes aridades de operadores:

  • Unário (1 operando)

  • Binário (2 operandos)

  • Ternário (3 operandos)

  • Precedence indica quais operadores “ligam” primeiro aos seus operandos. Ou seja, qual operador tem prioridade para operar em seus operandos. Por exemplo, a linguagem C obedece à convenção de que multiplicação e divisão têm precedência sobre adição e subtração:

    a * b + c

Dá o mesmo resultado que

  (a * b) + c

Se isso não for o desejado, a precedência pode ser forçada usando parênteses, pois eles têm a precedência mais alta de todos os operadores.

  a * (b + c)

Essa nova expressão produzirá um resultado diferente das duas expressões anteriores.

A linguagem C tem muitos níveis de precedência; Uma tabela é fornecida abaixo de todos os operadores, em ordem decrescente de precedência.

**Precedence Table**

Operators                                                     | Associativity
------                                                        | ------
`()` `[]` `->` `.`                                            | left to right
`!` `~` `++` `--` `+` `-` `*` (dereference) `(type)` `sizeof` | right to left
`*` (multiplication) `/` `%`                                  | left to right
`+` `-`                                                       | left to right
`<<` `>>`                                                     | left to right
`<` `<=` `>` `>=`                                             | left to right
`==` `!=`                                                     | left to right
`&`                                                           | left to right
`^`                                                           | left to right
<code>&#124;</code>                                                          | left to right
`&&`                                                          | left to right
<code>&#124;&#124;</code>                                                        | left to right
`?:`                                                          | right to left
`=` `+=` `-=` `*=` `/=` `%=` `&=` `^=` <code>&#124;=</code> `<<=` `>>=`       | right to left
`,`                                                          | left to right

  • Associativity indica como os operadores de precedência igual são vinculados por padrão e existem dois tipos: Left-to-Right e Right-to-Left. Um exemplo de ligação Left-to-Right é o operador de subtração (-). A expressão

    a - b - c - d

tem três subtrações de precedência idêntica, mas dá o mesmo resultado que

  ((a - b) - c) - d

porque o - mais à esquerda liga primeiro aos seus dois operandos.

Um exemplo de associatividade Right-to-Left são os operadores de desreferenciamento * e pós-incremento ++. Ambos têm precedência igual, portanto, se forem usados ​​em uma expressão como

  * ptr ++

, isto é equivalente a

  * (ptr ++)

porque o operador unário mais à direita (++) liga primeiro ao seu único operando.

Operadores Relacionais #

Os operadores relacionais verificam se uma relação específica entre dois operandos é verdadeira. O resultado é avaliado como 1 (o que significa verdadeiro) ou 0 (o que significa falso). Este resultado é frequentemente usado para afetar o fluxo de controle (via if, while, for), mas também pode ser armazenado em variáveis.

Igual a “==” #

Verifica se os operandos fornecidos são iguais.

1 == 0;         /* evaluates to 0. */
1 == 1;         /* evaluates to 1. */

int x = 5;
int y = 5;
int *xptr = &x, *yptr = &y;
xptr == yptr;   /* evaluates to 0, the operands hold different location addresses. */
*xptr == *yptr; /* evaluates to 1, the operands point at locations that hold the same value. */

Atenção: Este operador não deve ser confundido com o operador de atribuição (=)!

Não é igual a “!=” #

Verifica se os operandos fornecidos não são iguais.

1 != 0;         /* evaluates to 1. */
1 != 1;         /* evaluates to 0. */

int x = 5;
int y = 5;
int *xptr = &x, *yptr = &y;
xptr != yptr;   /* evaluates to 1, the operands hold different location addresses. */
*xptr != *yptr; /* evaluates to 0, the operands point at locations that hold the same value. */

Esse operador efetivamente retorna o resultado oposto ao do operador equals (==).

Não “!” #

Verifique se um objeto é igual a ‘0’.

O ! também pode ser usado diretamente com uma variável da seguinte forma:

!someVal

Isso tem o mesmo efeito que:

someVal == 0

Maior que “>” #

Verifica se o operando esquerdo tem um valor maior que o operando direito

5 > 4      /* evaluates to 1. */
4 > 5      /* evaluates to 0. */
4 > 4      /* evaluates to 0. */

Menor que “<” #

Verifica se o operando esquerdo tem um valor menor que o operando direito

5 < 4      /* evaluates to 0. */
4 < 5      /* evaluates to 1. */
4 < 4      /* evaluates to 0. */

Maior ou igual a “>=” #

Verifica se o operando esquerdo tem valor maior ou igual ao operando direito.

5 >= 4      /* evaluates to 1. */
4 >= 5      /* evaluates to 0. */
4 >= 4      /* evaluates to 1. */

Menor ou igual a “<=” #

Verifica se o operando esquerdo possui valor menor ou igual ao operando direito.

5 <= 4      /* evaluates to 0. */
4 <= 5      /* evaluates to 1. */
4 <= 4      /* evaluates to 1. */

Operador Condicional/Operador Ternário #

Avalia seu primeiro operando e, se o valor resultante não for igual a zero, avalia seu segundo operando. Caso contrário, ele avalia seu terceiro operando, conforme mostrado no exemplo a seguir:

a = b ? c : d;

é equivalente a:

if (b)
    a = c;
else 
    a = d;

Este pseudo-código o representa: condição ? value_if_true : value_if_false. Cada valor pode ser o resultado de uma expressão avaliada.

int x = 5;
int y = 42;
printf("%i, %i\n", 1 ? x : y, 0 ? x : y); /* Outputs "5, 42" */

O operador condicional pode ser aninhado. Por exemplo, o código a seguir determina o maior de três números:

big= a > b ? (a > c ? a : c)
           : (b > c ? b : c);

O exemplo a seguir grava inteiros pares em um arquivo e inteiros ímpares em outro arquivo:

#include<stdio.h>

int main()
{
    FILE *even, *odds;
    int n = 10;
    size_t k = 0;

    even = fopen("even.txt", "w");
    odds = fopen("odds.txt", "w");

    for(k = 1; k < n + 1; k++)
    {
        k%2==0 ? fprintf(even, "\t%5d\n", k)
               : fprintf(odds, "\t%5d\n", k);
    }
    fclose(even);
    fclose(odds);

    return 0;
}

O operador condicional associa da direita para a esquerda. Considere o seguinte:

exp1 ? exp2 : exp3 ? exp4 : exp5

Como a associação é da direita para a esquerda, a expressão acima é avaliado como

exp1 ? exp2 : ( exp3 ? exp4 : exp5 )

Operadores bit a bit #

Os operadores bit a bit podem ser usados ​​para realizar operações em nível de bit em variáveis. Abaixo está uma lista de todos os seis operadores bit a bit suportados em C:

Símbolo Operador
& E bit a bit
| OU inclusivo bit a bit
^ OU exclusivo bit a bit (XOR)
~ bit a bit não (complemento de um)
« deslocamento lógico para a esquerda
>> deslocamento lógico para a direita

O programa a seguir ilustra o uso de todos os operadores bit a bit:

#include <stdio.h>

int main(void)
{
   unsigned int a = 29;    /* 29 = 0001 1101 */  
   unsigned int b = 48;    /* 48 = 0011 0000 */
   int c = 0;           

   c = a & b;              /* 32 = 0001 0000 */ 
   printf("%d & %d = %d\n", a, b, c );

   c = a | b;              /* 61 = 0011 1101 */
   printf("%d | %d = %d\n", a, b, c );

   c = a ^ b;              /* 45 = 0010 1101 */
   printf("%d ^ %d = %d\n", a, b, c );

   c = ~a;                 /* -30 = 1110 0010 */
   printf("~%d = %d\n", a, c );

   c = a << 2;             /* 116 = 0111 0100 */
   printf("%d << 2 = %d\n", a, c );

   c = a >> 2;             /* 7 = 0000 0111 */
   printf("%d >> 2 = %d\n", a, c );

   return 0;
}

Operações bit a bit com tipos assinados devem ser evitadas porque o bit de sinal de tal representação de bit tem um significado particular. Restrições particulares se aplicam aos operadores de turno:

  • O deslocamento à esquerda de 1 bit no bit assinado é errôneo e leva a um comportamento indefinido.

  • O deslocamento para a direita de um valor negativo (com bit de sinal 1) é definido pela implementação e, portanto, não é portátil.

  • Se o valor do operando direito de um operador de deslocamento for negativo ou for maior ou igual à largura do operando esquerdo promovido, o comportamento é indefinido.

Mascaramento:

O mascaramento refere-se ao processo de extrair os bits desejados de (ou transformar os bits desejados em) uma variável usando operações lógicas bit a bit. O operando (uma constante ou variável) que é usado para realizar o mascaramento é chamado de mask.

O mascaramento é usado de muitas maneiras diferentes:

  • Para decidir o padrão de bits de uma variável inteira.
  • Para copiar uma parte de um determinado padrão de bits para uma nova variável, enquanto o restante da nova variável é preenchido com 0s (usando AND bit a bit)
  • Para copiar uma parte de um determinado padrão de bits para uma nova variável, enquanto o restante da nova variável é preenchido com 1s (usando OR bit a bit).
  • Para copiar uma parte de um determinado padrão de bits para uma nova variável, enquanto o restante do padrão de bits original é invertido dentro da nova variável (usando OR exclusivo bit a bit).

A função a seguir usa uma máscara para exibir o padrão de bits de uma variável:

#include <limits.h>
void bit_pattern(int u)
{
    int i, x, word;
    unsigned mask = 1;
    word = CHAR_BIT * sizeof(int);
    mask = mask << (word - 1);    /* shift 1 to the leftmost position */
    for(i = 1; i <= word; i++)
    {
        x = (u & mask) ? 1 : 0;  /* identify the bit */
        printf("%d", x);         /* print bit value */
        mask >>= 1;              /* shift mask to the right by 1 bit */
    }
}

Operadores aritméticos #

Aritmética Básica #

Retorna um valor que é o resultado da aplicação do operando esquerdo ao operando direito, usando a operação matemática associada. Regras matemáticas normais de comutação se aplicam (ou seja, adição e multiplicação são comutativas, subtração, divisão e módulo não são).

Operador de adição #

O operador de adição (+) é usado para somar dois operandos. Exemplo:

#include <stdio.h>

int main(void)
{
    int a = 5;
    int b = 7;

    int c = a + b; /* c now holds the value 12 */

    printf("%d + %d = %d",a,b,c); /* will output "5 + 7 = 12" */

    return 0;
}

Operador de subtração #

O operador de subtração (-) é usado para subtrair o segundo operando do primeiro. Exemplo:

#include <stdio.h>

int main(void)
{
    int a = 10;
    int b = 7;

    int c = a - b; /* c now holds the value 3 */

    printf("%d - %d = %d",a,b,c); /* will output "10 - 7 = 3" */

    return 0;
}

Operador de multiplicação #

O operador de multiplicação (*) é usado para multiplicar ambos os operandos. Exemplo:

#include <stdio.h>

int main(void)
{    
    int a = 5;
    int b = 7;

    int c = a * b; /* c now holds the value 35 */

    printf("%d * %d = %d",a,b,c); /* will output "5 * 7 = 35" */

    return 0;
}

Não confundir com o operador * de desreferenciação.

Operador de Divisão #

O operador de divisão (/) divide o primeiro operando pelo segundo. Se ambos os operandos da divisão forem inteiros, ele retornará um valor inteiro e descartará o resto (use o operador de módulo % para calcular e adquirir o resto).

Se um dos operandos for um valor de ponto flutuante, o resultado será uma aproximação da fração.

Exemplo:

#include <stdio.h>

int main (void)
{
    int a = 19 / 2 ; /* a holds value 9   */
    int b = 18 / 2 ; /* b holds value 9   */
    int c = 255 / 2; /* c holds value 127 */
    int d = 44 / 4 ; /* d holds value 11  */
    double e = 19 / 2.0 ; /* e holds value 9.5   */
    double f = 18.0 / 2 ; /* f holds value 9.0   */
    double g = 255 / 2.0; /* g holds value 127.5 */
    double h = 45.0 / 4 ; /* h holds value 11.25 */

    printf("19 / 2 = %d\n", a);    /* Will output "19 / 2 = 9"    */
    printf("18 / 2 = %d\n", b);    /* Will output "18 / 2 = 9"    */
    printf("255 / 2 = %d\n", c);   /* Will output "255 / 2 = 127" */
    printf("44 / 4 = %d\n", d);    /* Will output "44 / 4 = 11"   */
    printf("19 / 2.0 = %g\n", e);  /* Will output "19 / 2.0 = 9.5"    */
    printf("18.0 / 2 = %g\n", f);  /* Will output "18.0 / 2 = 9"      */
    printf("255 / 2.0 = %g\n", g); /* Will output "255 / 2.0 = 127.5" */
    printf("45.0 / 4 = %g\n", h);  /* Will output "45.0 / 4 = 11.25"  */

    return 0;
}

Operador de módulo #

O operador de módulo (%) recebe somente operandos inteiros e é usado para calcular o resto depois que o primeiro operando é dividido pelo segundo. Exemplo:

#include <stdio.h>

int main (void) {
    int a = 25 % 2;    /* a holds value 1  */
    int b = 24 % 2;    /* b holds value 0  */
    int c = 155 % 5;   /* c holds value 0  */
    int d = 49 % 25;   /* d holds value 24 */

    printf("25 % 2 = %d\n", a);     /* Will output "25 % 2 = 1"    */
    printf("24 % 2 = %d\n", b);     /* Will output "24 % 2 = 0"    */
    printf("155 % 5 = %d\n", c);    /* Will output "155 % 5 = 0"   */
    printf("49 % 25 = %d\n", d);    /* Will output "49 % 25 = 24"  */

    return 0;
}

Operadores de Incremento / Decremento #

Os operadores de incremento (a++) e decremento (a--) são diferentes porque alteram o valor da variável à qual você os aplica sem um operador de atribuição. Você pode usar operadores de incremento e decremento antes ou depois da variável. A colocação do operador altera o tempo de incremento/decremento do valor para antes ou depois de atribuí-lo à variável. Exemplo:

#include <stdio.h>

int main(void)
{
    int a = 1;
    int b = 4;
    int c = 1;
    int d = 4;

    a++;
    printf("a = %d\n",a);    /* Will output "a = 2" */
    b--;
    printf("b = %d\n",b);    /* Will output "b = 3" */

    if (++c > 1) { /* c is incremented by 1 before being compared in the condition */
        printf("This will print\n");    /* This is printed */
    } else {
        printf("This will never print\n");    /* This is not printed */
    }

    if (d-- < 4) {  /* d is decremented after being compared */
        printf("This will never print\n");    /* This is not printed */
    } else {
        printf("This will print\n");    /* This is printed */
    }
}

Como mostra o exemplo para c e d, ambos os operadores têm duas formas, como notação prefixada e notação pós-fixada. Ambos têm o mesmo efeito em incrementar (++) ou decrementar (--) a variável, mas diferem pelo valor que retornam: as operações prefix fazem a operação primeiro e depois retornam o valor, enquanto as operações postfix primeiro determinam o valor que deve ser retornado e, em seguida, faça a operação.

Devido a esse comportamento potencialmente contra-intuitivo, o uso de operadores de incremento/decremento dentro de expressões é controverso.

Operador de vírgula #

Avalia seu operando esquerdo, descarta o valor resultante e, em seguida, avalia seu operando direito e resultado produz o valor de seu operando mais à direita.

int x = 42, y = 42;
printf("%i\n", (x *= 2, y)); /* Outputs "42". */

O operador de vírgula introduz um ponto de sequência entre seus operandos.

Observe que a vírgula usada em chamadas de funções que separam argumentos NÃO é o operador de vírgula, mas é chamado de separador que é diferente do operador de vírgula. Portanto, ele não possui as propriedades do operador de vírgula.

A chamada printf() acima contém o operador de vírgula e o separador.

printf("%i\n", (x *= 2, y)); /* Outputs "42". */
/*           ^        ^ this is a comma operator */
/*           this is a separator */

O operador vírgula é frequentemente usado na seção de inicialização, bem como na seção de atualização de um loop for. Por exemplo:

for(k = 1; k < 10; printf("\%d\\n", k), k += 2);   /*outputs the odd numbers below 9/*

/* outputs sum to first 9 natural numbers */
for(sumk = 1, k = 1; k < 10; k++, sumk += k)
    printf("\%5d\%5d\\n", k, sumk);

Operadores de acesso #

Os operadores de acesso a membros (ponto . e seta ->) são usados ​​para acessar um membro de uma struct.

Membro do objeto #

Avalia no lvalue que denota o objeto que é membro do objeto acessado.

struct MyStruct
{
    int x;
    int y;
};

struct MyStruct myObject;
myObject.x = 42;
myObject.y = 123;

printf(".x = %i, .y = %i\n", myObject.x, myObject.y); /* Outputs ".x = 42, .y = 123". */

Membro do objeto apontado #

Açúcar sintático para desreferenciamento seguido de acesso de membros. Efetivamente, uma expressão da forma x->y é uma abreviação de (*x).y — mas o operador de seta é muito mais claro, especialmente se os ponteiros de estrutura estiverem aninhados.

struct MyStruct
{
    int x;
    int y;
};

struct MyStruct myObject;
struct MyStruct *p = &myObject;

p->x = 42;
p->y = 123;

printf(".x = %i, .y = %i\n", p->x, p->y); /* Outputs ".x = 42, .y = 123". */
printf(".x = %i, .y = %i\n", myObject.x, myObject.y); /* Also outputs ".x = 42, .y = 123". */

Endereço de #

O operador unário & é o endereço do operador. Ele avalia a expressão dada, onde o objeto resultante deve ser um lvalue. Em seguida, ele é avaliado em um objeto cujo tipo é um ponteiro para o tipo do objeto resultante e contém o endereço do objeto resultante.

int x = 3;
int *p = &x;
printf("%p = %p\n", （void *)&x, (void *)p); /* Outputs "A = A", for some implementation-defined A. */

Desreferência #

O operador unário * cancela a referência de um ponteiro. Ele avalia o lvalue resultante da desreferenciação do ponteiro que resulta da avaliação da expressão fornecida.

int x = 42;
int *p = &x;
printf("x = %d, *p = %d\n", x, *p); /* Outputs "x = 42, *p = 42". */

*p = 123;
printf("x = %d, *p = %d\n", x, *p); /* Outputs "x = 123, *p = 123". */

Indexação #

A indexação é açúcar sintático para adição de ponteiro seguido de desreferenciação. Efetivamente, uma expressão da forma a[i] é equivalente a *(a + i) — mas a notação subscrita explícita é preferida.

int arr[] = { 1, 2, 3, 4, 5 };
printf("arr[2] = %i\n", arr[2]); /* Outputs "arr[2] = 3". */

Intercambialidade de indexação #

Adicionar um ponteiro a um inteiro é uma operação comutativa (ou seja, a ordem dos operandos não altera o resultado) então pointer + integer == integer + pointer.

Uma consequência disso é que arr[3] e 3[arr] são equivalentes.

printf("3[arr] = %i\n", 3[arr]); /* Outputs "3[arr] = 4". */

O uso de uma expressão 3[arr] em vez de arr[3] geralmente não é recomendado, pois afeta a legibilidade do código. Ele tende a ser popular em concursos de programação ofuscados.

Comportamento de curto-circuito de operadores lógicos #

Curto-circuito é uma funcionalidade que pula a avaliação de partes de uma condição (se/enquanto/…) quando possível. No caso de uma operação lógica em dois operandos, o primeiro operando é avaliado (para verdadeiro ou falso) e se houver um veredicto (ou seja, o primeiro operando é falso ao usar &&, o primeiro operando é verdadeiro ao usar ||) o segundo operando é Não avaliado.

Exemplo:

#include <stdio.h>
 
int main(void) {
  int a = 20;
  int b = -5;
 
  /* here 'b == -5' is not evaluated,
     since a 'a != 20' is false. */
  if (a != 20 && b == -5) {
    printf("I won't be printed!\n");
  }
   
  return 0;
}

Confira você mesmo:

#include <stdio.h>
 
int print(int i) {
  printf("print function %d\n", i);
  return i;
}
 
int main(void) {
  int a = 20;
 
  /* here 'print(a)' is not called,
     since a 'a != 20' is false. */
  if (a != 20 && print(a)) {
    printf("I won't be printed!\n");
  }

  /* here 'print(a)' is called,
     since a 'a == 20' is true. */
  if (a == 20 && print(a)) {
    printf("I will be printed!\n");
  }

  return 0;
}

Resultado:

$ ./a.out
print function 20
I will be printed!

O curto-circuito é importante quando você deseja evitar avaliar termos que são (computacionalmente) caros. Além disso, pode afetar fortemente o fluxo do seu programa, como neste caso: http://stackoverflow.com/questions/26716255/why-does-this-program-print-forked-4-times

operador sizeof #

Com um tipo como operando #

Avalia o tamanho em bytes, do tipo size_t, de objetos do tipo fornecido. Requer parênteses ao redor do tipo.

printf("%zu\n", sizeof(int)); /* Valid, outputs the size of an int object, which is platform-dependent. */
printf("%zu\n", sizeof int); /* Invalid, types as arguments need to be surrounded by parentheses! */

Com uma expressão como operando #

Avalia o tamanho em bytes, do tipo size_t, de objetos do tipo da expressão dada. A expressão em si não é avaliada. Não são necessários parênteses; no entanto, como a expressão fornecida deve ser unária, é considerado uma prática recomendada sempre usá-las.

char ch = 'a';
printf("%zu\n", sizeof(ch)); /* Valid, will output the size of a char object, which is always 1 for all platforms. */
printf("%zu\n", sizeof ch);  /* Valid, will output the size of a char object, which is always 1 for all platforms. */

Incrementar / Decrementar #

Os operadores de incremento e decremento existem na forma prefix e postfix.

int a = 1;
int b = 1;
int tmp = 0;

tmp = ++a;        /* increments a by one, and returns new value; a == 2, tmp == 2  */
tmp = a++;        /* increments a by one, but returns old value; a == 3, tmp == 2 */
tmp = --b;        /* decrements b by one, and returns new value; b == 0, tmp == 0 */
tmp = b--;        /* decrements b by one, but returns old value; b == -1, tmp == 0 */

Observe que as operações aritméticas não introduzem pontos de sequência, então certas expressões com operadores ++ ou -- podem introduzir [comportamento indefinido]( https://www.wikiod.com/pt/c/comportamento-indefinido#Modificando qualquer objeto mais de uma vez entre dois pontos de sequência).

Operador de elenco #

Executa uma conversão explícita no tipo fornecido a partir do valor resultante da avaliação da expressão fornecida.

int x = 3;
int y = 4;
printf("%f\n", (double)x / y); /* Outputs "0.750000". */

Aqui o valor de x é convertido para um double, a divisão promove o valor de y para double também, e o resultado da divisão, um double é passado para printf para impressão .

Operador de chamada de função #

O primeiro operando deve ser um ponteiro de função (um designador de função também é aceitável porque será convertido em um ponteiro para a função), identificando a função a ser chamada e todos os outros operandos, se houver, são conhecidos coletivamente como argumentos da chamada de função . Avalia no valor de retorno resultante da chamada da função apropriada com os respectivos argumentos.

int myFunction(int x, int y)
{
    return x * 2 + y;
}

int (*fn)(int, int) = &myFunction;
int x = 42;
int y = 123;

printf("(*fn)(%i, %i) = %i\n", x, y, (*fn)(x, y)); /* Outputs "fn(42, 123) = 207". */
printf("fn(%i, %i) = %i\n", x, y, fn(x, y)); /* Another form: you don't need to dereference explicitly */

Operadores de Atribuição #

Atribui o valor do operando à direita ao local de armazenamento nomeado pelo operando à esquerda e retorna o valor.

int x = 5;      /* Variable x holds the value 5. Returns 5. */ 
char y = 'c';   /* Variable y holds the value 99. Returns 99 
                 * (as the character 'c' is represented in the ASCII table with 99).
                 */
float z = 1.5;  /* variable z holds the value 1.5. Returns 1.5. */
char const* s = "foo"; /* Variable s holds the address of the first character of the string 'foo'. */

Várias operações aritméticas têm um operador atribuição composta.

a += b  /* equal to: a = a + b */
a -= b  /* equal to: a = a - b */
a *= b  /* equal to: a = a * b */
a /= b  /* equal to: a = a / b */
a %= b  /* equal to: a = a % b */
a &= b  /* equal to: a = a & b */
a |= b  /* equal to: a = a | b */
a ^= b  /* equal to: a = a ^ b */
a <<= b /* equal to: a = a << b */
a >>= b /* equal to: a = a >> b */

Uma característica importante dessas atribuições compostas é que a expressão do lado esquerdo (a) é avaliada apenas uma vez. Por exemplo, se p for um ponteiro

*p += 27;

desreferencia p apenas uma vez, enquanto o seguinte o faz duas vezes.

*p = *p + 27;

Deve-se notar também que o resultado de uma atribuição como a = b é o que é conhecido como rvalue. Assim, a atribuição realmente tem um valor que pode ser atribuído a outra variável. Isso permite o encadeamento de atribuições para definir várias variáveis ​​em uma única instrução.

Este rvalue pode ser usado nas expressões de controle de instruções if (ou loops ou instruções switch) que guardam algum código no resultado de outra expressão ou chamada de função. Por exemplo:

char *buffer;
if ((buffer = malloc(1024)) != NULL)
{
    /* do something with buffer */
    free(buffer);
}
else
{
    /* report allocation failure */
}

Por causa disso, deve-se tomar cuidado para evitar erros de digitação comuns que podem levar a bugs misteriosos.

int a = 2;
/* ... */
if (a = 1)
    /* Delete all files on my hard drive */

Isso terá resultados desastrosos, pois a = 1 sempre será avaliado como 1 e, portanto, a expressão de controle da instrução if será sempre verdadeira (leia mais sobre essa armadilha comum [aqui](http://stackoverflow .com/documentation/c/2006/common-pitfalls/7886/mistakenly-writing-instead-of-when-comparing#t=201704191400115596587)). O autor quase certamente pretendia usar o operador de igualdade (==) como mostrado abaixo:

int a = 2;
/* ... */
if (a == 1)
    /* Delete all files on my hard drive */

Associatividade do Operador

int a, b = 1, c = 2;
a = b = c;

Isso atribui c a b, que retorna b, que é atribuído a a. Isso acontece porque todos os operadores de atribuição têm associatividade à direita, o que significa que a operação mais à direita na expressão é avaliada primeiro e prossegue da direita para a esquerda.

Operadores lógicos #

E Lógico #

Executa um AND lógico booleano dos dois operandos retornando 1 se ambos os operandos forem diferentes de zero. O operador lógico AND é do tipo int.

0 && 0  /* Returns 0. */
0 && 1  /* Returns 0. */
2 && 0  /* Returns 0. */
2 && 3  /* Returns 1. */

OU lógico #

Executa um OR booleano lógico dos dois operandos retornando 1 se algum dos operandos for diferente de zero. O operador lógico OR é do tipo int.

0 || 0  /* Returns 0. */
0 || 1  /* Returns 1.  */
2 || 0  /* Returns 1.  */
2 || 3  /* Returns 1.  */

NÃO Lógico #

Executa uma negação lógica. O operador lógico NOT é do tipo int. O operador NOT verifica se pelo menos um bit é igual a 1, em caso afirmativo retorna 0. Caso contrário, retorna 1;

!1 /* Returns 0. */
!5 /* Returns 0. */
!0 /* Returns 1. */

Avaliação de curto-circuito #

Existem algumas propriedades cruciais comuns a && e ||:

  • o operando esquerdo (LHS) é totalmente avaliado antes que o operando direito (RHS) seja avaliado,
  • existe um ponto de sequência entre a avaliação do operando da esquerda e do operando da direita,
  • e, mais importante, o operando do lado direito não é avaliado se o resultado do operando do lado esquerdo determinar o resultado geral.

Isso significa que:

  • se o LHS for avaliado como ’true’ (diferente de zero), o RHS de || não será avaliado (porque o resultado de ’true OR qualquer coisa’ é ’true’),
  • se o LHS for avaliado como ‘falso’ (zero), o RHS de ‘&&’ não será avaliado (porque o resultado de ‘falso AND qualquer coisa’ é ‘falso’).

Isso é importante, pois permite que você escreva códigos como:

const char *name_for_value(int value)
{
    static const char *names[] = { "zero", "one", "two", "three", };
    enum { NUM_NAMES = sizeof(names) / sizeof(names[0]) };
    return (value >= 0 && value < NUM_NAMES) ? names[value] : "infinity";
}

Se um valor negativo for passado para a função, o termo value >= 0 será avaliado como falso e o termo value < NUM_NAMES não será avaliado.

Aritmética de ponteiro #

Adição de ponteiro #

Dado um ponteiro e um tipo escalar N, é avaliado como um ponteiro para o Nésimo elemento do tipo apontado que sucede diretamente o objeto apontado na memória.

int arr[] = {1, 2, 3, 4, 5};
printf("*(arr + 3) = %i\n", *(arr + 3)); /* Outputs "4", arr's fourth element. */

Não importa se o ponteiro é usado como valor do operando ou valor escalar. Isso significa que coisas como 3 + arr são válidas. Se arr[k] é o membro k+1 de um array, então arr+k é um ponteiro para arr[k]. Em outras palavras, arr ou arr+0 é um ponteiro para arr[0], arr+1 é um ponteiro para arr[2] e assim por diante. Em geral, *(arr+k) é o mesmo que arr[k].

Ao contrário da aritmética usual, a adição de 1 a um ponteiro para um int adicionará 4 bytes ao valor do endereço atual. Como nomes de arrays são ponteiros constantes, + é o único operador que podemos usar para acessar os membros de um array via notação de ponteiro usando o nome do array. No entanto, definindo um ponteiro para uma matriz, podemos obter mais flexibilidade para processar os dados em uma matriz. Por exemplo, podemos imprimir os membros de um array da seguinte forma:

#include<stdio.h>
static const size_t N = 5
    
int main()
{
    size_t k = 0;
    int arr[] = {1, 2, 3, 4, 5};
    for(k = 0; k < N; k++)
    {
        printf("\n\t%d", *(arr + k));
    }
    return 0;
}

Ao definir um ponteiro para o array, o programa acima é equivalente ao seguinte:

#include<stdio.h>
static const size_t N = 5
    
int main()
{
    size_t k = 0;
    int arr[] = {1, 2, 3, 4, 5};
    int *ptr = arr; /* or int *ptr = &arr[0]; */
    for(k = 0; k < N; k++)
    {
        printf("\n\t%d", ptr[k]);
        /* or   printf("\n\t%d", *(ptr + k)); */
        /* or   printf("\n\t%d", *ptr++); */
    }
    return 0;
}

Veja que os membros do array arr são acessados ​​usando os operadores + e ++. Os outros operadores que podem ser usados ​​com o ponteiro ptr são - e --.

Subtração de ponteiro #

Dados dois ponteiros para o mesmo tipo, é avaliado em um objeto do tipo ptrdiff_t que contém o valor escalar que deve ser adicionado ao segundo ponteiro para obter o valor do primeiro ponteiro.

int arr[] = {1, 2, 3, 4, 5};
int *p = &arr[2];
int *q = &arr[3];
ptrdiff_t diff = q - p;

printf("q - p = %ti\n", diff); /* Outputs "1". */
printf("*(p + (q - p)) = %d\n", *(p + diff)); /* Outputs "4". */

_Alignof #

Consulta o requisito de alinhamento para o tipo especificado. O requisito de alinhamento é uma potência integral positiva de 2 representando o número de bytes entre os quais dois objetos do tipo podem ser alocados. Em C, o requisito de alinhamento é medido em size_t.

O nome do tipo não pode ser um tipo incompleto nem um tipo de função. Se uma matriz for usada como o tipo, o tipo do elemento da matriz será usado.

Este operador é frequentemente acessado através da macro de conveniência alignof de <stdalign.h>.

int main(void)
{
    printf("Alignment of char = %zu\n", alignof(char));
    printf("Alignment of max_align_t = %zu\n", alignof(max_align_t));
    printf("alignof(float[10]) = %zu\n", alignof(float[10]));
    printf("alignof(struct{char c; int n;}) = %zu\n",
            alignof(struct {char c; int n;}));    
}

Saída Possível:

Alignment of char = 1
Alignment of max_align_t = 16
alignof(float[10]) = 4
alignof(struct{char c; int n;}) = 4
