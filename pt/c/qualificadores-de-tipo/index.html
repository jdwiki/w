Os qualificadores de tipo são as palavras-chave que descrevem a semântica adicional sobre um tipo. Eles são parte integrante das assinaturas de tipo. Eles podem aparecer tanto no nível superior de uma declaração (afetando diretamente o identificador) quanto em subníveis (relevantes apenas para ponteiros, afetando os valores apontados):

Palavra-chave Observações
const Impede a mutação do objeto declarado (aparecendo no nível mais alto) ou impede a mutação do valor apontado (aparecendo ao lado de um subtipo de ponteiro).
volátil Informa ao compilador que o objeto declarado (no nível mais alto) ou o valor apontado (em subtipos de ponteiros) pode alterar seu valor como resultado de condições externas, não apenas como resultado do fluxo de controle do programa.
restringir Uma dica de otimização, relevante apenas para ponteiros. Declara a intenção de que durante o tempo de vida do ponteiro, nenhum outro ponteiro será usado para acessar o mesmo objeto apontado.

A ordenação dos qualificadores de tipo em relação aos especificadores de classe de armazenamento (static, extern, auto, register), modificadores de tipo (signed, unsigned, short, long) e tipo especificadores (int, char, double, etc.)

static const volatile unsigned long int a = 5; /* good practice */
unsigned volatile long static int const b = 5; /* bad practice */

Qualificações de nível superior

/* "a" cannot be mutated by the program but can change as a result of external conditions */
const volatile int a = 5;

/* the const applies to array elements, i.e. "a[0]" cannot be mutated */    
const int arr[] = { 1, 2, 3 };

/* for the lifetime of "ptr", no other pointer could point to the same "int" object */
int *restrict ptr;

Qualificações de subtipo de ponteiro

/* "s1" can be mutated, but "*s1" cannot */
const char *s1 = "Hello";

/* neither "s2" (because of top-level const) nor "*s2" can be mutated */
const char *const s2 = "World";

/* "*p" may change its value as a result of external conditions, "**p" and "p" cannot */
char *volatile *p;

/* "q", "*q" and "**q" may change their values as a result of external conditions */
volatile char *volatile *volatile q;

Variáveis ​​não modificáveis ​​(const) #

const int a = 0; /* This variable is "unmodifiable", the compiler
                    should throw an error when this variable is changed */
int b = 0; /* This variable is modifiable */

b += 10; /* Changes the value of 'b' */
a += 10; /* Throws a compiler error */

A qualificação const significa apenas que não temos o direito de alterar os dados. Isso não significa que o valor não pode mudar nas nossas costas.

_Bool doIt(double const* a) {
   double rememberA = *a;
   // do something long and complicated that calls other functions

   return rememberA == *a;
}

Durante a execução das outras chamadas, *a pode ter mudado e, portanto, esta função pode retornar false ou true.

Aviso #

Variáveis ​​com qualificação const ainda podem ser alteradas usando ponteiros:

const int a = 0;

int *a_ptr = (int*)&a; /* This conversion must be explicitly done with a cast */
*a_ptr += 10;          /* This has undefined behavior */

printf("a = %d\n", a); /* May print: "a = 10" */

Mas fazer isso é um erro que leva a um comportamento indefinido. A dificuldade aqui é que isso pode se comportar como esperado em exemplos simples como este, mas depois dar errado quando o código cresce.

Variáveis ​​voláteis #

A palavra-chave volatile informa ao compilador que o valor da variável pode mudar a qualquer momento como resultado de condições externas, não apenas como resultado do fluxo de controle do programa.

O compilador não otimizará nada que tenha a ver com a variável volátil.

volatile int foo; /* Different ways to declare a volatile variable */
int volatile foo;

volatile uint8_t * pReg; /* Pointers to volatile variable */
uint8_t volatile * pReg;

Existem duas razões principais para usar variáveis ​​voláteis:

  • Para fazer interface com hardware que possui registradores de E/S mapeados na memória.
  • Ao usar variáveis ​​que são modificadas fora do fluxo de controle do programa (por exemplo, em uma rotina de serviço de interrupção)

Vejamos este exemplo:

int quit = false;

void main() 
{
    ... 
    while (!quit) {
      // Do something that does not modify the quit variable
    } 
    ...
}

void interrupt_handler(void) 
{
  quit = true;
}

O compilador pode notar que o loop while não modifica a variável quit e converte o loop em um loop infinito while (true). Mesmo que a variável quit esteja definida no manipulador de sinais para SIGINT e SIGTERM, o compilador não sabe disso.

Declarar quit como volatile dirá ao compilador para não otimizar o loop e o problema será resolvido.

O mesmo problema acontece ao acessar o hardware, como vemos neste exemplo:

uint8_t * pReg = (uint8_t *) 0x1717;

// Wait for register to become non-zero 
while (*pReg == 0) { } // Do something else

O comportamento do otimizador é ler o valor da variável uma vez, não há necessidade de reler, pois o valor será sempre o mesmo. Então acabamos com um loop infinito. Para forçar o compilador a fazer o que queremos, modificamos a declaração para:

uint8_t volatile * pReg = (uint8_t volatile *) 0x1717;
