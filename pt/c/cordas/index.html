Em C, uma string não é um tipo intrínseco. Uma string C é a convenção para ter um array unidimensional de caracteres que é terminado por um caractere nulo, por um '\0'.

Isso significa que uma string C com conteúdo "abc" terá quatro caracteres 'a', 'b', 'c' e '\0'.

Veja o exemplo de introdução básica a strings.

1: https://www.wikiod.com/pt/c/cordas#Introdução básica às cordas

Sintaxe #

  • char str1[] = “Olá, mundo!”; /* Modificável */
  • char str2[14] = “Olá, mundo!”; /* Modificável */
  • char* str3 = “Olá, mundo!”; /* Não modificável*/

Tokenização: strtok(), strtok_r() e strtok_s() #

A função strtok quebra uma string em strings menores, ou tokens, usando um conjunto de delimitadores.

#include <stdio.h>
#include <string.h>

int main(void)
{
    int toknum = 0;
    char src[] = "Hello,, world!";
    const char delimiters[] = ", !";
    char *token = strtok(src, delimiters);
    while (token != NULL)
    {
        printf("%d: [%s]\n", ++toknum, token);
        token = strtok(NULL, delimiters);
    }
    /* source is now "Hello\0, world\0\0" */
}

Resultado:

1: [Hello]
2: [world]

A string de delimitadores pode conter um ou mais delimitadores e diferentes strings de delimitadores podem ser usadas com cada chamada para strtok.

Chamadas para strtok para continuar tokenizando a mesma string de origem não devem passar a string de origem novamente, mas sim passar NULL como o primeiro argumento. Se a mesma string de origem for passada, o primeiro token será re-tokenizado. Ou seja, dados os mesmos delimitadores, strtok simplesmente retornaria o primeiro token novamente.

Note que como strtok não aloca nova memória para os tokens, ele modifica a string de origem. Ou seja, no exemplo acima, a string src será manipulada para produzir os tokens que são referenciados pelo ponteiro retornado pelas chamadas para strtok. Isso significa que a string de origem não pode ser const (portanto, não pode ser uma string literal). Isso também significa que a identidade do byte delimitador foi perdida (ou seja, no exemplo, “,” e “!” são efetivamente excluídos da string de origem e você não pode dizer qual caractere delimitador corresponde).

Observe também que vários delimitadores consecutivos na string de origem são tratados como um; no exemplo, a segunda vírgula é ignorada.

strtok não é thread-safe nem reentrante porque usa um buffer estático durante a análise. Isso significa que se uma função chama strtok, nenhuma função que ela chama enquanto está usando strtok também pode usar strtok, e não pode ser chamada por nenhuma função que esteja usando strtok.

Um exemplo que demonstra os problemas causados ​​pelo fato de strtok não ser reentrante é o seguinte:

char src[] = "1.2,3.5,4.2";
char *first = strtok(src, ","); 

do 
{
    char *part;
    /* Nested calls to strtok do not work as desired */
    printf("[%s]\n", first);
    part = strtok(first, ".");
    while (part != NULL)
    {
        printf(" [%s]\n", part);
        part = strtok(NULL, ".");
    }
} while ((first = strtok(NULL, ",")) != NULL);

Resultado:

[1.2]
 [1]
 [2]

A operação esperada é que o loop externo do while crie três tokens consistindo em cada string de número decimal ("1.2", "3.5", "4.2"), para cada um dos quais o strtok chamadas para o loop interno devem dividi-lo em strings de dígitos separados ("1", "2", "3", "5", "4", "2").

No entanto, como o strtok não é reentrante, isso não ocorre. Em vez disso, o primeiro strtok cria corretamente o token “1.2\0”, e o loop interno cria corretamente os tokens "1" e "2". Mas então o strtok no loop externo está no final da string usada pelo loop interno e retorna NULL imediatamente. A segunda e terceira substrings do array src não são analisadas.

As bibliotecas C padrão não contêm uma versão thread-safe ou reentrante, mas algumas outras contêm, como POSIX’ strtok_r. Observe que no MSVC o equivalente strtok, strtok_s é thread-safe.

C11 tem uma parte opcional, Anexo K, que oferece uma versão thread-safe e reentrante chamada strtok_s. Você pode testar o recurso com __STDC_LIB_EXT1__. Esta parte opcional não é amplamente suportada.

A função strtok_s difere da função POSIX strtok_r protegendo contra o armazenamento fora da string que está sendo tokenizada e verificando as restrições de tempo de execução. Em programas escritos corretamente, porém, o strtok_s e o strtok_r se comportam da mesma forma.

Usar strtok_s com o exemplo agora produz a resposta correta, assim:

/* you have to announce that you want to use Annex K */ 
#define __STDC_WANT_LIB_EXT1__ 1
#include <string.h>

#ifndef __STDC_LIB_EXT1__
# error "we need strtok_s from Annex K"
#endif

char src[] = "1.2,3.5,4.2";  
char *next = NULL;
char *first = strtok_s(src, ",", &next);

do 
{
    char *part;
    char *posn;

    printf("[%s]\n", first);
    part = strtok_s(first, ".", &posn);
    while (part != NULL)
    {
        printf(" [%s]\n", part);
        part = strtok_s(NULL, ".", &posn);
    }
} 
while ((first = strtok_s(NULL, ",", &next)) != NULL);

E a saída será:

[1.2]
 [1]
 [2]
[3.5]
 [3]
 [5]
[4.2]
 [4]
 [2]

Literais de string #

Literais de string representam arrays static-duration terminados em nulo de char. Como eles têm duração de armazenamento estático, um literal de string ou um ponteiro para a mesma matriz subjacente pode ser usado com segurança de várias maneiras que um ponteiro para uma matriz automática não pode. Por exemplo, retornar um literal de string de uma função tem um comportamento bem definido:

const char *get_hello() {
    return "Hello, World!";  /* safe */
}

Por razões históricas, os elementos do array correspondentes a uma string literal não são formalmente const. No entanto, qualquer tentativa de modificá-los tem comportamento indefinido. Normalmente, um programa que tenta modificar a matriz correspondente a uma string literal falhará ou funcionará mal.

char *foo = "hello";
foo[0] = 'y';  /* Undefined behavior - BAD! */

Onde um ponteiro aponta para uma string literal – ou onde às vezes pode apontar – é aconselhável declarar o referente do ponteiro const para evitar envolver acidentalmente tal comportamento indefinido.

const char *foo = "hello";
/* GOOD: can't modify the string pointed to by foo */

Por outro lado, um ponteiro para ou para o array subjacente de uma string literal não é inerentemente especial; seu valor pode ser modificado livremente para apontar para outra coisa:

char *foo = "hello";
foo = "World!"; /* OK - we're just changing what foo points to */

Além disso, embora inicializadores para arrays char possam ter a mesma forma que literais de string, o uso de tal inicializador não confere as características de um literal de string no array inicializado. O inicializador simplesmente designa o comprimento e o conteúdo inicial do array. Em particular, os elementos são modificáveis ​​se não forem declarados explicitamente como const:

char foo[] = "hello";
foo[0] = 'y';  /* OK! */

Calcular o comprimento: strlen() #

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(int argc, char **argv) 
{
    /* Exit if no second argument is found. */
    if (argc != 2) 
    {
        puts("Argument missing.");
        return EXIT_FAILURE;
    }

    size_t len = strlen(argv[1]);
    printf("The length of the second argument is %zu.\n", len);

    return EXIT_SUCCESS;
}

Este programa calcula o comprimento de seu segundo argumento de entrada e armazena o resultado em len. Em seguida, imprime esse comprimento no terminal. Por exemplo, quando executado com os parâmetros program_name "Hello, world!", o programa produzirá O comprimento do segundo argumento é 13. porque a string Hello, world! tem 13 caracteres.

strlen conta todos os bytes desde o início da string até, mas não incluindo, o caractere NUL final, '\0'. Como tal, ele só pode ser usado quando a string é garantida para ser terminada em NUL.

Também tenha em mente que se a string contiver algum caractere Unicode, strlen não lhe dirá quantos caracteres estão na string (já que alguns caracteres podem ter vários bytes). Nesses casos, você precisa contar os caracteres (*ou seja, unidades de código) você mesmo. Considere a saída do exemplo a seguir:

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(void) 
{
    char asciiString[50] = "Hello world!";
    char utf8String[50] = "Γειά σου Κόσμε!"; /* "Hello World!" in Greek */

    printf("asciiString has %zu bytes in the array\n", sizeof(asciiString));
    printf("utf8String has %zu bytes in the array\n", sizeof(utf8String));
    printf("\"%s\" is %zu bytes\n", asciiString, strlen(asciiString));
    printf("\"%s\" is %zu bytes\n", utf8String, strlen(utf8String));
}

Resultado:

asciiString has 50 bytes in the array
utf8String has 50 bytes in the array
"Hello world!" is 12 bytes
"Γειά σου Κόσμε!" is 27 bytes

Introdução básica às strings #

Em C, uma string é uma sequência de caracteres terminada por um caractere nulo (’\0’).

Podemos criar strings usando literais de string, que são sequências de caracteres entre aspas duplas; por exemplo, pegue a string literal "hello world". Os literais de string são automaticamente terminados em nulo.

Podemos criar strings usando vários métodos. Por exemplo, podemos declarar um char * e inicializá-lo para apontar para o primeiro caractere de uma string:

char * string = "hello world";

Ao inicializar um char * para uma constante de string como acima, a própria string geralmente é alocada em dados somente leitura; string é um ponteiro para o primeiro elemento do array, que é o caractere 'h'.

Como o literal de string é alocado na memória somente leitura, ele não pode ser modificado1. Qualquer tentativa de modificá-lo levará a comportamento indefinido, então é melhor adicionar const para obter um erro de tempo de compilação como este

char const * string = "hello world";

Tem efeito semelhante2 como

char const string_arr[] = "hello world";

Para criar uma string modificável, você pode declarar uma matriz de caracteres e inicializar seu conteúdo usando um literal de string, assim:

char modifiable_string[] = "hello world";

Isso é equivalente ao seguinte:

char modifiable_string[] = {'h', 'e', 'l', 'l', 'o', ' ', 'w', 'o', 'r', 'l', 'd', '\0'};

Como a segunda versão usa o inicializador entre chaves, a string não é automaticamente terminada em nulo, a menos que um caractere '\0' seja incluído explicitamente na matriz de caracteres geralmente como seu último elemento.

1 Não modificável implica que os caracteres na string literal não podem ser modificados, mas lembre-se que o ponteiro string pode ser modificado (pode apontar para outro lugar ou pode ser incrementado ou decrementado).

2 Ambas as strings têm efeito semelhante no sentido de que os caracteres de ambas as strings não podem ser modificados. Deve-se notar que string é um ponteiro para char e é um valor l modificável para que possa ser incrementado ou apontar para algum outro local enquanto o array string_arr for um valor l não modificável, ele não pode ser modificado.

Copiando strings #

Atribuições de ponteiro não copiam strings #

Você pode usar o operador = para copiar inteiros, mas não pode usar o operador = para copiar strings em C. Strings em C são representadas como arrays de caracteres com um caractere nulo de terminação, portanto, usar o operador = salvará apenas o endereço (ponteiro) de uma string.

#include <stdio.h>

int main(void) {
    int a = 10, b;
    char c[] = "abc", *d;

    b = a; /* Integer is copied */
    a = 20; /* Modifying a leaves b unchanged - b is a 'deep copy' of a */
    printf("%d %d\n", a, b); /* "20 10" will be printed */

    d = c; 
    /* Only copies the address of the string - 
    there is still only one string stored in memory */
    
    c[1] = 'x';
    /* Modifies the original string - d[1] = 'x' will do exactly the same thing */

    printf("%s %s\n", c, d); /* "axc axc" will be printed */

    return 0;
}

O exemplo acima foi compilado porque usamos char *d em vez de char d[3]. Usar o último causaria um erro do compilador. Você não pode atribuir a matrizes em C.

#include <stdio.h>

int main(void) {
    char a[] = "abc";
    char b[8];

    b = a; /* compile error */
    printf("%s\n", b);

    return 0;
}

Copiando strings usando funções padrão #

strcpy() #

Para realmente copiar strings, a função strcpy() está disponível em string.h. Espaço suficiente deve ser alocado para o destino antes de copiar.

#include <stdio.h>
#include <string.h>

int main(void) {
    char a[] = "abc";
    char b[8];

    strcpy(b, a); /* think "b special equals a" */
    printf("%s\n", b); /* "abc" will be printed */

    return 0;
}

snprintf() #

Para evitar saturação de buffer, snprintf() pode ser usado. Não é a melhor solução em termos de desempenho, pois precisa analisar a string do modelo, mas é a única função segura de limite de buffer para copiar strings prontamente disponíveis na biblioteca padrão, que pode ser usada sem etapas extras.

#include <stdio.h>
#include <string.h>

int main(void) {
    char a[] = "012345678901234567890";
    char b[8];

#if 0
    strcpy(b, a); /* causes buffer overrun (undefined behavior), so do not execute this here! */
#endif

    snprintf(b, sizeof(b), "%s", a); /* does not cause buffer overrun */
    printf("%s\n", b); /* "0123456" will be printed */

    return 0;
}

strncat() #

Uma segunda opção, com melhor desempenho, é usar strncat() (uma versão de verificação de buffer overflow de strcat() ) - leva um terceiro argumento que informa o número máximo de bytes a serem copiados:

char dest[32];

dest[0] = '\0';
strncat(dest, source, sizeof(dest) - 1);
    /* copies up to the first (sizeof(dest) - 1) elements of source into dest,
    then puts a \0 on the end of dest */

Observe que esta formulação usa sizeof(dest) - 1; isso é crucial porque strncat() sempre adiciona um byte nulo (bom), mas não conta isso no tamanho da string (uma causa de confusão e substituições de buffer).

Observe também que a alternativa — concatenar após uma string não vazia — é ainda mais complicada. Considerar:

char dst[24] = "Clownfish: ";
char src[] = "Marvin and Nemo";
size_t len = strlen(dst);

strncat(dst, src, sizeof(dst) - len - 1);
printf("%zu: [%s]\n", strlen(dst), dst);

A saída é:

23: [Clownfish: Marvin and N]

Observe, no entanto, que o tamanho especificado como comprimento não era o tamanho da matriz de destino, mas a quantidade de espaço restante nela, sem contar o byte nulo do terminal. Isso pode causar grandes problemas de substituição. Também é um pouco desperdício; para especificar o argumento de comprimento corretamente, você conhece o comprimento dos dados no destino, então você pode especificar o endereço do byte nulo no final do conteúdo existente, salvando strncat() de redigitalizá-lo:

    strcpy(dst, "Clownfish: ");
    assert(len < sizeof(dst) - 1);
    strncat(dst + len, src, sizeof(dst) - len - 1);
    printf("%zu: [%s]\n", strlen(dst), dst);

Isso produz a mesma saída de antes, mas strncat() não precisa varrer o conteúdo existente de dst antes de começar a copiar.

strncpy() #

A última opção é a função strncpy(). Embora você possa pensar que deve vir primeiro, é uma função bastante enganosa que tem duas armadilhas principais:

  1. Se a cópia via strncpy() atingir o limite de buffer, um caractere nulo final não será escrito.
  2. strncpy() sempre preenche completamente o destino, com bytes nulos se necessário.

(Essa implementação peculiar é histórica e foi inicialmente destinada a lidar com nomes de arquivos UNIX)

A única maneira correta de usá-lo é garantir manualmente a terminação nula:

strncpy(b, a, sizeof(b)); /* the third parameter is destination buffer size */
b[sizeof(b)/sizeof(*b) - 1] = '\0'; /* terminate the string */
printf("%s\n", b); /* "0123456" will be printed */

Mesmo assim, se você tiver um buffer grande, torna-se muito ineficiente usar strncpy() por causa do preenchimento nulo adicional.

Iterando sobre os caracteres em uma string #

Se soubermos o comprimento da string, podemos usar um loop for para iterar sobre seus caracteres:

char * string = "hello world"; /* This 11 chars long, excluding the 0-terminator. */
size_t i = 0;
for (; i < 11; i++) {
    printf("%c\n", string[i]);    /* Print each character of the string. */
}

Alternativamente, podemos usar a função padrão strlen() para obter o comprimento de uma string se não soubermos qual é a string:

size_t length = strlen(string);
size_t i = 0; 
for (; i < length; i++) {
    printf("%c\n", string[i]);    /* Print each character of the string. */
}

Finalmente, podemos tirar vantagem do fato de que strings em C são garantidamente terminadas em nulo (o que já fizemos quando passamos para strlen() no exemplo anterior ;-)). Podemos iterar sobre o array independentemente de seu tamanho e parar de iterar quando atingirmos um caractere nulo:

size_t i = 0;
while (string[i] != '\0') {       /* Stop looping when we reach the null-character. */
    printf("%c\n", string[i]);    /* Print each character of the string. */
    i++;
}

Criando Arrays de Strings #

Um array de strings pode significar algumas coisas:

  1. Um array cujos elementos são char *s
  2. Um array cujos elementos são arrays de chars

Podemos criar uma matriz de ponteiros de caracteres assim:

char * string_array[] = {
    "foo",
    "bar",
    "baz"
};

Lembre-se: quando atribuímos literais de string a char *, as próprias strings são alocadas na memória somente leitura. No entanto, o array string_array é alocado na memória de leitura/gravação. Isso significa que podemos modificar os ponteiros no array, mas não podemos modificar as strings para as quais eles apontam.

Em C, o parâmetro para main argv (o array de argumentos de linha de comando passados ​​quando o programa foi executado) é um array de char *: char * argv[].

Também podemos criar arrays de arrays de caracteres. Como strings são arrays de caracteres, um array de strings é simplesmente um array cujos elementos são arrays de caracteres:

char modifiable_string_array_literals[][4] = {
    "foo",
    "bar",
    "baz"
};

Isso é equivalente a:

char modifiable_string_array[][4] = {
    {'f', 'o', 'o', '\0'},
    {'b', 'a', 'r', '\0'},
    {'b', 'a', 'z', '\0'}
};

Observe que especificamos 4 como o tamanho da segunda dimensão do array; cada uma das strings em nosso array tem, na verdade, 4 bytes, pois devemos incluir o caractere de terminação nula.

Converta Strings para Number: atoi(), atof() (perigoso, não os use) #

Atenção: As funções atoi, atol, atoll e atof são inerentemente inseguras, porque: [Se o valor do resultado não puder ser representado, o comportamento é indefinido.](http://www. open-std.org/jtc1/sc22/wg14/www/docs/n1124.pdf) (7.20.1p1)

#include <stdio.h>
#include <stdlib.h>

int main(int argc, char** argv)
{
    int val;
    if (argc < 2)
    {
        printf("Usage: %s <integer>\n", argv[0]);
        return 0;
    }

    val = atoi(argv[1]);

    printf("String value = %s, Int value = %d\n", argv[1], val);

    return 0;
}

Quando a string a ser convertida é um inteiro decimal válido que está no intervalo, a função funciona:

$ ./atoi 100
String value = 100, Int value = 100
$ ./atoi 200
String value = 200, Int value = 200

Para strings que começam com um número, seguido por outra coisa, apenas o número inicial é analisado:

$ ./atoi 0x200
0
$ ./atoi 0123x300
123

Em todos os outros casos, o comportamento é indefinido:

$ ./atoi hello
Formatting the hard disk...

Por causa das ambiguidades acima e deste comportamento indefinido, a família de funções atoi nunca deve ser usada.

  • Para converter para long int, use strtol() em vez de atol().
  • Para converter para double, use strtod() em vez de atof().
  • Para converter para long long int, use strtoll() em vez de atoll().

Gravar dados formatados em string

int sprintf ( char * str, const char * format, ... );

#include <stdio.h>
int main ()

use a função sscanf para analisar dados formatados.

#include <stdio.h>
int main ()
{
  char sentence []="date : 06-06-2012";
  char str [50];
  int year;
  int month;
  int day;
  sscanf (sentence,"%s : %2d-%2d-%4d", str, &day, &month, &year);

  return 0;

Copiar e Concatenar: strcpy(), strcat() #

#include <stdio.h>
#include <string.h>

int main(void)
{
  /* Always ensure that your string is large enough to contain the characters
   * and a terminating NUL character ('\0')!
   */
  char mystring[10];

  /* Copy "foo" into `mystring`, until a NUL character is encountered. */
  strcpy(mystring, "foo");
  printf("%s\n", mystring);

  /* At this point, we used 4 chars of `mystring`, the 3 characters of "foo",
   * and the NUL terminating byte.
   */

  /* Append "bar" to `mystring`. */
  strcat(mystring, "bar");
  printf("%s\n", mystring);

  /* We now use 7 characters of `mystring`: "foo" requires 3, "bar" requires 3
   * and there is a terminating NUL character ('\0') at the end.
   */

  /* Copy "bar" into `mystring`, overwriting the former contents. */
  strcpy(mystring, "bar");
  printf("%s\n", mystring);

  return 0;
}

Saídas:

foo
foobar
bar

Se você anexar ou copiar de uma string existente, certifique-se de que ela seja terminada por NUL!

Literais de string (por exemplo, "foo") sempre serão finalizados com NUL pelo compilador.

Comparação: strcmp(), strncmp(), strcasecmp(), strncasecmp() #

As funções strcase* não são C padrão, mas uma extensão POSIX.

A função strcmp compara lexicograficamente dois arrays de caracteres terminados em nulo. As funções retornam um valor negativo se o primeiro argumento aparecer antes do segundo em ordem lexicográfica, zero se forem iguais ou positivo se o primeiro argumento aparecer após o segundo em ordem lexicográfica.

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs)
{
    int result = strcmp(lhs, rhs); // compute comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "BBB");
    compare("BBB", "CCCCC");
    compare("BBB", "AAAAAA");
    return 0;
}

Saídas:

BBB equals BBB
BBB comes before CCCCC
BBB comes after AAAAAA

Como strcmp, a função strcasecmp também compara lexicograficamente seus argumentos depois de traduzir cada caractere para seu correspondente em minúsculas:

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs)
{
    int result = strcasecmp(lhs, rhs); // compute case-insensitive comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "bBB");
    compare("BBB", "ccCCC");
    compare("BBB", "aaaaaa");
    return 0;
}

Saídas:

BBB equals bBB
BBB comes before ccCCC
BBB comes after aaaaaa

strncmp e strncasecmp comparam no máximo n caracteres:

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs, int n)
{
    int result = strncmp(lhs, rhs, n); // compute comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "Bb", 1);
    compare("BBB", "Bb", 2);
    compare("BBB", "Bb", 3);
    return 0;
}

Saídas:

BBB equals Bb
BBB comes before Bb
BBB comes before Bb

Converta com segurança Strings para Number: strtoX functions #

Desde C99 a biblioteca C tem um conjunto de funções de conversão seguras que interpretam uma string como um número. Seus nomes estão na forma strtoX, onde X é um de l, ul, d, etc. para determinar o tipo de destino da conversão

double strtod(char const* p, char** endptr);
long double strtold(char const* p, char** endptr);

Eles fornecem a verificação de que uma conversão teve um over ou underflow:

double ret = strtod(argv[1], 0); /* attempt conversion */

/* check the conversion result. */
if ((ret == HUGE_VAL || ret == -HUGE_VAL) && errno == ERANGE) 
    return;  /* numeric overflow in in string */
else if (ret == HUGE_VAL && errno == ERANGE) 
    return; /* numeric underflow in in string */

/* At this point we know that everything went fine so ret may be used */

Se a string de fato não contiver nenhum número, esse uso de strtod retornará 0.0.

Se isso não for satisfatório, o parâmetro adicional endptr pode ser usado. É um ponteiro para ponteiro que será apontado para o final do número detectado na string. Se for definido como 0, como acima, ou NULL, ele é simplesmente ignorado.

Este parâmetro endptr indica se houve uma conversão bem-sucedida e, em caso afirmativo, onde o número terminou:

char *check = 0;
double ret = strtod(argv[1], &check); /* attempt conversion */

/* check the conversion result. */
if (argv[1] == check) 
    return; /* No number was detected in string */
else if ((ret == HUGE_VAL || ret == -HUGE_VAL) && errno == ERANGE) 
    return; /* numeric overflow in in string */
else if (ret == HUGE_VAL && errno == ERANGE) 
    return; /* numeric underflow in in string */

/* At this point we know that everything went fine so ret may be used */

Existem funções análogas para converter para os tipos inteiros mais amplos:

long strtol(char const* p, char** endptr, int nbase);
long long strtoll(char const* p, char** endptr, int nbase);
unsigned long strtoul(char const* p, char** endptr, int nbase);
unsigned long long strtoull(char const* p, char** endptr, int nbase);

Estas funções têm um terceiro parâmetro nbase que contém o número base na qual o número está escrito.

long a = strtol("101",   0, 2 ); /* a = 5L */
long b = strtol("101",   0, 8 ); /* b = 65L */
long c = strtol("101",   0, 10); /* c = 101L */
long d = strtol("101",   0, 16); /* d = 257L */
long e = strtol("101",   0, 0 ); /* e = 101L */
long f = strtol("0101",  0, 0 ); /* f = 65L */
long g = strtol("0x101", 0, 0 ); /* g = 257L */

O valor especial 0 para nbase significa que a string é interpretada da mesma forma que os literais numéricos são interpretados em um programa C: um prefixo 0x corresponde a uma representação hexadecimal, caso contrário, um 0 inicial é octal e todos os outros números são vistos como decimais.

Assim, a maneira mais prática de interpretar um argumento de linha de comando como um número seria

int main(int argc, char* argv[] {
    if (argc < 1)
        return EXIT_FAILURE; /* No number given. */

    /* use strtoull because size_t may be wide */
    size_t mySize = strtoull(argv[1], 0, 0);

    /* then check conversion results. */

     ...

    return EXIT_SUCCESS;
}

Isso significa que o programa pode ser chamado com um parâmetro em octal, decimal ou hexadecimal.

strstr #

/* finds the next instance of needle in haystack 
   zbpos: the zero-based position to begin searching from
   haystack: the string to search in
   needle: the string that must be found
   returns the next match of `needle` in `haystack`, or -1 if not found
*/
int findnext(int zbpos, const char *haystack, const char *needle)
{
    char *p; 

    if (((p = strstr(haystack + zbpos, needle)) != NULL)
        return p - haystack;

    return -1;
}

strstr procura no argumento haystack (primeiro) a string apontada por needle. Se encontrado, strstr retorna o endereço da ocorrência. Se não encontrar needle, ele retornará NULL. Usamos zbpos para que não fiquemos encontrando a mesma agulha repetidamente. Para pular a primeira instância, adicionamos um deslocamento de zbpos. Um clone do Bloco de Notas pode chamar findnext assim, para implementar seu diálogo “Find Next”:

/*
    Called when the user clicks "Find Next"
    doc: The text of the document to search
    findwhat: The string to find
*/
void onfindnext(const char *doc, const char *findwhat)
{
    static int i;

    if ((i = findnext(i, doc, findwhat)) != -1)
        /* select the text starting from i and ending at i + strlen(findwhat) */
    else
        /* display a message box saying "end of search" */
}
