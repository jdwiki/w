Visão geral #

O padrão C descreve a sintaxe da linguagem, as funções fornecidas pela biblioteca padrão e o comportamento de processadores C em conformidade (grosseiramente falando, compiladores) e programas C em conformidade. Com relação ao comportamento, o padrão em sua maior parte especifica comportamentos específicos para programas e processadores. Por outro lado, algumas operações têm comportamento indefinido explícito ou implícito – tais operações devem sempre ser evitadas, pois você não pode confiar em nada sobre elas. No meio, há uma variedade de comportamentos definidos para implementação. Esses comportamentos podem variar entre processadores C, tempos de execução e bibliotecas padrão (coletivamente, implementações), mas são consistentes e confiáveis ​​para qualquer implementação, e as implementações em conformidade documentam seu comportamento em cada uma dessas áreas.

Às vezes, é razoável que um programa dependa do comportamento definido pela implementação. Por exemplo, se o programa for de alguma forma específico para um ambiente operacional específico, é improvável que confiar em comportamentos definidos pela implementação gerais para os processadores comuns desse ambiente seja um problema. Alternativamente, pode-se usar diretivas de compilação condicional para selecionar comportamentos definidos pela implementação apropriados para a implementação em uso. Em qualquer caso, é essencial saber quais operações têm comportamento definido de implementação, para evitá-las ou para tomar uma decisão informada sobre se e como usá-las.

O saldo dessas observações constitui uma lista de todos os comportamentos definidos pela implementação e características especificadas no padrão C2011, com referências ao padrão. Muitos deles usam a terminologia do padrão. Alguns outros dependem mais geralmente do contexto do padrão, como os oito estágios de tradução do código-fonte em um programa ou a diferença entre implementações hospedadas e independentes. Alguns que podem ser particularmente surpreendentes ou notáveis ​​são apresentados em negrito. Nem todos os comportamentos descritos são suportados por padrões C anteriores, mas, de modo geral, eles têm um comportamento definido pela implementação em todas as versões do padrão que os suportam.

Representação de inteiros com sinal #

Cada tipo de inteiro com sinal pode ser representado em qualquer um dos três formatos; é definido pela implementação qual é usado. A implementação em uso para qualquer tipo de inteiro com sinal pelo menos tão grande quanto int pode ser determinada em tempo de execução a partir dos dois bits de ordem mais baixa da representação do valor -1 nesse tipo, assim:

enum { sign_magnitude = 1, ones_compl = 2, twos_compl = 3, };
#define SIGN_REP(T) ((T)-1 & (T)3)

switch (SIGN_REP(long)) {
   case sign_magnitude: { /* do something */ break; }
   case ones_compl:     { /* do otherwise */ break; }
   case twos_compl:     { /* do yet else  */ break; }
   case 0:  { _Static_assert(SIGN_REP(long), "bogus sign representation"); }
}

O mesmo padrão se aplica à representação de tipos mais restritos, mas eles não podem ser testados por esta técnica porque os operandos de & estão sujeitos às “conversões aritméticas usuais” antes que o resultado seja computado.
