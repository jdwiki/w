A declaração de identificador referente a um objeto ou função é muitas vezes referida como simplesmente uma declaração de objeto ou função.

Chamando uma função de outro arquivo C #

foo.h

#ifndef FOO_DOT_H    /* This is an "include guard" */
#define FOO_DOT_H    /* prevents the file from being included twice. */
                     /* Including a header file twice causes all kinds */
                     /* of interesting problems.*/

/**
 * This is a function declaration.
 * It tells the compiler that the function exists somewhere.
 */
void foo(int id, char *name);

#endif /* FOO_DOT_H */

foo.c

#include "foo.h"    /* Always include the header file that declares something
                     * in the C file that defines it. This makes sure that the
                     * declaration and definition are always in-sync.  Put this
                     * header first in foo.c to ensure the header is self-contained.
                     */
#include <stdio.h>
                       
/**
 * This is the function definition.
 * It is the actual body of the function which was declared elsewhere.
 */
void foo(int id, char *name)
{
    fprintf(stderr, "foo(%d, \"%s\");\n", id, name);
    /* This will print how foo was called to stderr - standard error.
     * e.g., foo(42, "Hi!") will print `foo(42, "Hi!")`
     */
}

main.c

#include "foo.h"

int main(void)
{
    foo(42, "bar");
    return 0;
}

Compilar e vincular

Primeiro, compilamos tanto foo.c quanto main.c para arquivos de objetos. Aqui usamos o compilador gcc, seu compilador pode ter um nome diferente e precisar de outras opções.

$ gcc -Wall -c foo.c
$ gcc -Wall -c main.c

Agora nós os vinculamos para produzir nosso executável final:

$ gcc -o testprogram foo.o main.o

Usando uma variável global #

Introdução #

Exemplos de declarações são:

int a; /* declaring single identifier of type int */

A declaração acima declara um identificador único chamado a que se refere a algum objeto com o tipo int.

int a1, b1; /* declaring 2 identifiers of type int */

A segunda declaração declara 2 identificadores chamados a1 e b1 que se referem a alguns outros objetos embora com o mesmo tipo int.

Basicamente, a maneira como isso funciona é assim - primeiro você coloca algum type, depois escreve uma única ou várias expressões separadas por vírgula (,) (que não será avaliada neste momento - e que de outra forma devem ser referidos como declaradores neste contexto). Ao escrever tais expressões, você tem permissão para aplicar apenas os operadores de indireção (*), chamada de função (( )) ou subscrito (ou indexação de array - [ ]) em algum identificador (você também não pode usar nenhum operadores). O identificador usado não precisa estar visível no escopo atual. Alguns exemplos:

/* 1 */ int /* 2 */ (*z) /* 3 */ , /* 4 */ *x , /* 5 */ **c /* 6 */ ;
# Descrição
1 O nome do tipo inteiro.
2 Expressão não avaliada aplicando indireção a algum identificador z.
3 Temos uma vírgula indicando que mais uma expressão seguirá na mesma declaração.
4 Expressão não avaliada aplicando indireção a algum outro identificador x.
5 Expressão não avaliada aplicando indireção ao valor da expressão (*c).
6 Fim da declaração.

Observe que nenhum dos identificadores acima era visível antes desta declaração e, portanto, as expressões usadas não seriam válidas antes dela.

Após cada expressão, o identificador usado nela é introduzido no escopo atual. (Se o identificador tiver atribuído uma ligação a ele, ele também pode ser declarado novamente com o mesmo tipo de ligação para que ambos os identificadores se refiram ao mesmo objeto ou função)

Além disso, o sinal do operador igual (=) pode ser usado para inicialização. Se uma expressão não avaliada (declarador) for seguida por = dentro da declaração - dizemos que o identificador que está sendo introduzido também está sendo inicializado. Após o sinal = podemos colocar novamente alguma expressão, mas desta vez ela será avaliada e seu valor será usado como inicial para o objeto declarado.

Exemplos:

int l = 90; /* the same as: */

int l; l = 90; /* if it the declaration of l was in block scope */

int c = 2, b[c]; /* ok, equivalent to: */

int c = 2; int b[c];

Mais tarde em seu código, você tem permissão para escrever exatamente a mesma expressão da parte de declaração do identificador recém-introduzido, dando a você um objeto do tipo especificado no início da declaração, supondo que você atribuiu valores válidos a todos os objetos no caminho. Exemplos:

void f()
{
    int b2; /* you should be able to write later in your code b2 
            which will directly refer to the integer object
            that b2 identifies */
    
    b2 = 2; /* assign a value to b2 */
    
    printf("%d", b2); /*ok - should print 2*/

    int *b3; /* you should be able to write later in your code *b3 */

    b3 = &b2; /* assign valid pointer value to b3 */

    printf("%d", *b3); /* ok - should print 2 */

    int **b4; /* you should be able to write later in your code **b4 */

    b4 = &b3;

    printf("%d", **b4); /* ok - should print 2 */

    void (*p)(); /* you should be able to write later in your code (*p)() */

    p = &f; /* assign a valid pointer value */

    (*p)(); /* ok - calls function f by retrieving the
            pointer value inside p -    p
            and dereferencing it -      *p
            resulting in a function
            which is then called -      (*p)() -

            it is not *p() because else first the () operator is 
            applied to p and then the resulting void object is
            dereferenced which is not what we want here */
}

A declaração de b3 especifica que você pode potencialmente usar o valor b3 como um meio para acessar algum objeto inteiro.

Claro, para aplicar a indireção (*) ao b3, você também deve ter um valor apropriado armazenado nele (veja ponteiros para mais informações). Você também deve primeiro armazenar algum valor em um objeto antes de tentar recuperá-lo (você pode ver mais sobre esse problema aqui). Fizemos tudo isso nos exemplos acima.

int a3(); /* you should be able to call a3 */

Este diz ao compilador que você tentará chamar a3. Neste caso, a3 refere-se a uma função em vez de um objeto. Uma diferença entre objeto e função é que as funções sempre terão algum tipo de ligação. Exemplos:

void f1()
{
    {
        int f2(); /* 1 refers to some function f2 */
    }
    
    {
        int f2(); /* refers to the exact same function f2 as (1) */
    }
}

No exemplo acima, as 2 declarações referem-se à mesma função f2, enquanto que se estivessem declarando objetos então neste contexto (tendo 2 escopos de bloco diferentes), seriam 2 objetos distintos diferentes.

int (*a3)(); /* you should be able to apply indirection to `a3` and then call it */

Agora pode parecer complicado, mas se você conhece a precedência dos operadores, você terá 0 problemas ao ler a declaração acima. Os parênteses são necessários porque o operador * tem menos precedência que o ( ).

No caso de usar o operador subscrito, a expressão resultante não seria realmente válida após a declaração porque o índice usado nela (o valor dentro de [ e ]) será sempre 1 acima do valor máximo permitido para esta objeto/função.

int a4[5]; /* here a4 shouldn't be accessed using the index 5 later on */

Mas deve ser acessível por todos os outros índices inferiores a 5. Exemplos:

a4[0], a4[1]; a4[4];

a4[5] resultará em UB. Mais informações sobre arrays podem ser encontradas aqui.

int (*a5)[5](); /* here a4 could be applied indirection
                indexed up to (but not including) 5
                and called */

Infelizmente para nós, embora sintaticamente possível, a declaração de a5 é proibida pelo padrão atual.

Usando constantes globais #

Definição de tipo #

Typedefs são declarações que têm a palavra-chave typedef na frente e antes do tipo. Por exemplo.:

typedef int (*(*t0)())[5];

(* você pode tecnicamente colocar o typedef após o tipo também - como este int typedef (*(*t0)())[5]; mas isso é desencorajado*)

As declarações acima declaram um identificador para um nome typedef. Você pode usar assim depois:

t0 pf;

Que terá o mesmo efeito que escrever:

int (*(*pf)())[5];

Como você pode ver, o nome typedef “salva” a declaração como um tipo para usar posteriormente em outras declarações. Desta forma, você pode salvar algumas teclas. Além disso, como declaração usando typedef ainda é uma declaração, você não está limitado apenas pelo exemplo acima:

t0 (*pf1);

É o mesmo que:

int (*(**pf1)())[5];

Usando a regra direita-esquerda ou espiral para decifrar a declaração C #

A regra “direita-esquerda” é uma regra completamente regular para decifrar declarações C. Também pode ser útil para criá-los.

Leia os símbolos à medida que os encontra na declaração…

*   as "pointer to"          - always on the left side
[]  as "array of"            - always on the right side
()  as "function returning"  - always on the right side

Como aplicar a regra

PASSO 1

Encontre o identificador. Este é o seu ponto de partida. Então diga a si mesmo, “identificador é”. Você iniciou sua declaração.

PASSO 2

Observe os símbolos à direita do identificador. Se, digamos, você encontrar () lá, então você sabe que esta é a declaração de uma função. Então você teria “identificador é função retornando”. Ou se você encontrar um [] lá, você diria “identifier is array of”. Continue para a direita até ficar sem símbolos OU aperte um parêntese direito ). (Se você acertar um parêntese esquerdo (, esse é o início de um símbolo (), mesmo se houver coisas entre os parênteses. Mais sobre isso abaixo.)

ETAPA 3

Observe os símbolos à esquerda do identificador. Se não for um dos nossos símbolos acima (digamos, algo como “int”), apenas diga. Caso contrário, traduza para o inglês usando a tabela acima. Continue indo para a esquerda até ficar sem símbolos OU aperte um parêntese esquerdo (.

Agora repita os passos 2 e 3 até formar sua declaração.

Aqui estão alguns exemplos:

int *p[];

Primeiro, encontre o identificador:

int *p[];
     ^

“p é”

Agora, mova para a direita até que os símbolos ou o parêntese direito sejam atingidos.

int *p[];
      ^^

“p é uma matriz de”

Não é mais possível mover para a direita (sem símbolos), então mova para a esquerda e encontre:

int *p[];
    ^

“p é uma matriz de ponteiro para”

Continue indo para a esquerda e encontre:

int *p[];
^^^

“p é uma matriz de ponteiro para int”.

(ou “p é um array onde cada elemento é do tipo ponteiro para int”)

Outro exemplo:

int *(*func())();

Encontre o identificador.

int *(*func())();
       ^^^^

“função é”

Mova para a direita.

int *(*func())();
           ^^

“func é função retornando”

Não pode mais mover para a direita por causa do parêntese direito, então mova para a esquerda.

int *(*func())();
      ^

“func é a função que retorna o ponteiro para”

Não pode mais mover para a esquerda por causa do parêntese esquerdo, então continue para a direita.

int *(*func())();
              ^^

“func é a função que retorna o ponteiro para a função que retorna”

Não pode mais mover para a direita porque estamos sem símbolos, então vá para a esquerda.

int *(*func())();
    ^

“func é a função que retorna o ponteiro para a função que retorna o ponteiro para”

E, finalmente, continue indo para a esquerda, porque não há mais nada à direita.

int *(*func())();
^^^

“func é função que retorna ponteiro para função que retorna ponteiro para int”.

Como você pode ver, esta regra pode ser bastante útil. Você também pode usá-lo para verificar a sanidade enquanto cria declarações e dar uma dica sobre onde colocar o próximo símbolo e se os parênteses são necessários.

Algumas declarações parecem muito mais complicadas do que são devido aos tamanhos dos arrays e listas de argumentos em forma de protótipo. Se você vir [3], isso será lido como “array (tamanho 3) de…”. Se você vir (char *,int) que é lido como *“função esperando (char ,int) e retornando…”.

Aqui está um divertido:

int (*(*fun_one)(char *,double))[9][20];

Eu não vou passar por cada uma das etapas para decifrar este.

*“fun_one é ponteiro para função esperando (char ,double) e retornando ponteiro para array (tamanho 9) do array (tamanho 20) de int.”

Como você pode ver, não é tão complicado se você se livrar dos tamanhos dos arrays e listas de argumentos:

int (*(*fun_one)())[][];

Você pode decifrá-lo dessa maneira e, em seguida, inserir os tamanhos de matriz e as listas de argumentos posteriormente.

Algumas palavras finais:

É bem possível fazer declarações ilegais usando esta regra, então é necessário algum conhecimento do que é legal em C. Por exemplo, se o acima tivesse sido:

int *((*fun_one)())[][];

ele teria lido “fun_one é ponteiro para função retornando array de array de ponteiro para int”. Como uma função não pode retornar um array, mas apenas um ponteiro para um array, essa declaração é ilegal.

As combinações ilegais incluem:

[]() - cannot have an array of functions
()() - cannot have a function that returns a function
()[] - cannot have a function that returns an array

Em todos os casos acima, você precisaria de um conjunto de parênteses para vincular um símbolo * à esquerda entre esses símbolos () e [] do lado direito para que a declaração fosse válida.

Aqui estão mais alguns exemplos:

Jurídico

int i;               an int
int *p;              an int pointer (ptr to an int)
int a[];             an array of ints
int f();             a function returning an int
int **pp;            a pointer to an int pointer (ptr to a ptr to an int)
int (*pa)[];         a pointer to an array of ints
int (*pf)();         a pointer to a function returning an int
int *ap[];           an array of int pointers (array of ptrs to ints)
int aa[][];          an array of arrays of ints
int *fp();           a function returning an int pointer
int ***ppp;          a pointer to a pointer to an int pointer
int (**ppa)[];       a pointer to a pointer to an array of ints
int (**ppf)();       a pointer to a pointer to a function returning an int
int *(*pap)[];       a pointer to an array of int pointers
int (*paa)[][];      a pointer to an array of arrays of ints
int *(*pfp)();       a pointer to a function returning an int pointer
int **app[];         an array of pointers to int pointers
int (*apa[])[];      an array of pointers to arrays of ints
int (*apf[])();      an array of pointers to functions returning an int
int *aap[][];        an array of arrays of int pointers
int aaa[][][];       an array of arrays of arrays of int
int **fpp();         a function returning a pointer to an int pointer
int (*fpa())[];      a function returning a pointer to an array of ints
int (*fpf())();      a function returning a pointer to a function returning an int

Ilegal

int af[]();          an array of functions returning an int
int fa()[];          a function returning an array of ints
int ff()();          a function returning a function returning an int
int (*pfa)()[];      a pointer to a function returning an array of ints
int aaf[][]();       an array of arrays of functions returning an int
int (*paf)[]();      a pointer to a an array of functions returning an int
int (*pff)()();      a pointer to a function returning a function returning an int
int *afp[]();        an array of functions returning int pointers
int afa[]()[];       an array of functions returning an array of ints
int aff[]()();       an array of functions returning functions returning an int
int *fap()[];        a function returning an array of int pointers
int faa()[][];       a function returning an array of arrays of ints
int faf()[]();       a function returning an array of functions returning an int
int *ffp()();        a function returning a function returning an int pointer
