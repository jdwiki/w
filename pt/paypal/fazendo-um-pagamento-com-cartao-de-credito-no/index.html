Parâmetros #

Parâmetro Detalhes
cartão_dados Objeto JSON contendo informações de pagamento para transação
credit_card_details Objeto JSON contendo dados de cartão de crédito que são enviados ao PayPal para serem armazenados em cofre
client_id Seu ID de cliente do aplicativo PayPal (credenciais OAuth 2)
paypal Referência do SDK do nó do PayPal
segredo O segredo do seu aplicativo do PayPal (credenciais do OAuth 2)
uuid Referência ao pacote node-uuid

Esta amostra leva o usuário a creditar uma transação simples de cartão de crédito usando os SDKs do PayPal.

Amostra de Nó #

Comece instalando o módulo PayPal Node do NPM

npm install paypal-rest-sdk

Em seu arquivo de aplicativo, adicione as informações de configuração do SDK

var paypal = require('paypal-rest-sdk');

var client_id = 'YOUR CLIENT ID';
var secret = 'YOUR SECRET';

paypal.configure({
    'mode': 'sandbox', //sandbox or live
    'client_id': client_id,
    'client_secret': secret
});

Adicionamos o requisito para o SDK e, em seguida, configuramos variáveis ​​para o ID do cliente e o segredo que foram obtidos ao criar um aplicativo. Em seguida, configuramos nosso aplicativo usando esses detalhes e especificamos o ambiente em que estamos trabalhando (ao vivo ou sandbox).

Em seguida, configuramos o objeto JSON que contém as informações de pagamento do pagador.

var card_data = {
  "intent": "sale",
  "payer": {
    "payment_method": "credit_card",
    "funding_instruments": [{
      "credit_card": {
        "type": "visa",
        "number": "4417119669820331",
        "expire_month": "11",
        "expire_year": "2018",
        "cvv2": "874",
        "first_name": "Joe",
        "last_name": "Shopper",
        "billing_address": {
          "line1": "52 N Main ST",
          "city": "Johnstown",
          "state": "OH",
          "postal_code": "43210",
          "country_code": "US" }}}]},
  "transactions": [{
    "amount": {
      "total": "7.47",
      "currency": "USD",
      "details": {
        "subtotal": "7.41",
        "tax": "0.03",
        "shipping": "0.03"}},
    "description": "This is the payment transaction description." 
}]};

Adicione um intent de sale e um payment_method de credit_card. Em seguida, adicione os detalhes do cartão e endereço do cartão de crédito em funding_instruments, e o valor a ser cobrado em transactions. Vários objetos de transação podem ser colocados aqui.

Por fim, fazemos uma requisição ao payment.create(...), passando nosso objeto card_data, para processar o pagamento.

paypal.payment.create(card_data, function(error, payment){
  if(error){
    console.error(error);
  } else {
    console.log(payment);
  }
});

Efetuar um pagamento com um cartão de crédito protegido (node) #

Neste exemplo, veremos como armazenar um cartão de crédito usando o cofre do PayPal e, em seguida, referenciar esse cartão de crédito armazenado para processar uma transação de cartão de crédito para um usuário.

A razão pela qual gostaríamos de usar o cofre é para que não tenhamos que armazenar informações confidenciais de cartão de crédito em nossos próprios servidores. Simplesmente referenciamos o método de pagamento por meio de um ID de cofre fornecido, o que significa que não precisamos lidar com muitos regulamentos de conformidade com PCI para armazenar os cartões de crédito.

Assim como nos exemplos anteriores, começamos configurando nosso ambiente.

var paypal = require('paypal-rest-sdk'),
    uuid = require('node-uuid');

var client_id = 'YOUR CLIENT ID';
var secret = 'YOUR SECRET';

paypal.configure({
  'mode': 'sandbox', //sandbox or live
  'client_id': client_id,
  'client_secret': secret
});

A única diferença para os exemplos anteriores aqui é que estamos exigindo um novo pacote, node-uuid, que deve ser usado para gerar UUIDs exclusivos para os pagadores ao armazenar o cartão. Você pode instalar esse pacote via:

npm install node-uuid

Em seguida, definimos o objeto JSON do cartão de crédito que será enviado ao cofre do PayPal para armazenamento. Ele contém informações do cartão, bem como um ID de pagador exclusivo que geramos usando node-uuid. Você deve armazenar este payer_id exclusivo em seu próprio banco de dados, pois ele será usado ao criar um pagamento com o cartão protegido.

var create_card_details = {
    "type": "visa",
    "number": "4417119669820331",
    "expire_month": "11",
    "expire_year": "2018",
    "first_name": "John",
    "last_name": "Doe",
    "payer_id": uuid.v4()
};

Por fim, precisamos armazenar o cartão de crédito e processar o pagamento usando esse cartão. Para guardar um cartão de crédito, chamamos credit_card.create(...), passando o objeto credit_card_details que acabamos de criar. Se tudo correr bem, devemos ter um objeto devolvido para nós com detalhes sobre o cartão abobadado. Para o pagamento com esse cartão, precisamos apenas de duas informações: o payer_id que já armazenamos e o ID do cofre, que também deve ser armazenado como referência em nosso próprio banco de dados.

paypal.credit_card.create(create_card_details, function(error, credit_card){
    if(error){
        console.error(error);
    } else {
        var card_data = {
            "intent": "sale",
            "payer": {
                "payment_method": "credit_card",
                "funding_instruments": [{
                    "credit_card_token": {
                        "credit_card_id": credit_card.id,
                        "payer_id": credit_card.payer_id
                    }
                }]
            },
            "transactions": [{
                "amount": {
                    "total": "7.47",
                    "currency": "USD",
                    "details": {
                        "subtotal": "7.41",
                        "tax": "0.03",
                        "shipping": "0.03"
                    }
                },
                "description": "This is the payment transaction description." 
            }]
        };
        
        paypal.payment.create(card_data, function(error, payment){
            if(error){
                console.error(error);
            } else {
                console.log(JSON.stringify(payment));
            }
        });
    }
});

Na seção seguinte ao armazenamento bem-sucedido do cartão de crédito, estamos simplesmente definindo os detalhes do cartão e processando o pagamento, como fizemos com o exemplo anterior de processamento de cartão de crédito. A principal diferença na estrutura do objeto card_data é a seção funding_instruments, que definimos em payer. Em vez de definir as informações do cartão de crédito, usamos o objeto a seguir que contém a referência do ID do cofre e o ID do pagador:

"credit_card_token": {
    "credit_card_id": credit_card.id,
    "payer_id": credit_card.payer_id
}

É assim que usamos um cartão protegido para processar um pagamento.
