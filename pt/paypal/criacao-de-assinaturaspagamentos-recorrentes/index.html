Esses exemplos passam pelo processo de criação de um sistema de assinatura/pagamento recorrente usando o PayPal.

O processo para criar uma assinatura é:

  1. Crie um plano de cobrança. Este é um modelo reutilizável que descreve os detalhes da assinatura.
  2. Ative o plano de cobrança.
  3. Quando você deseja criar uma assinatura para um usuário, você cria um contrato de cobrança usando o ID do plano de cobrança ao qual ele deve se inscrever.
  4. Uma vez criado, você redireciona o usuário para o PayPal para confirmar a assinatura. Uma vez confirmado, o usuário é redirecionado de volta ao site do comerciante.
  5. Por fim, você assina o contrato de cobrança para iniciar a assinatura.

Etapa 2: criar uma assinatura para um usuário usando um contrato de cobrança (amostra de nó) #

A segunda etapa para criar uma assinatura para um usuário é criar e executar um contrato de cobrança com base em um plano de cobrança ativado existente. Este exemplo pressupõe que você já passou e ativou um plano de cobrança no exemplo anterior e tem um ID para esse plano de cobrança para referência no exemplo.

Ao configurar um contrato de cobrança para criar uma assinatura para um usuário, você seguirá 3 etapas, que podem lembrar o processamento de um pagamento do PayPal:

  1. Você cria um contrato de cobrança, referenciando um plano de cobrança subjacente por meio do ID.
  2. Uma vez criado, você redireciona o usuário para o PayPal (se estiver pagando via PayPal) para confirmar a assinatura. Uma vez confirmado, o PayPal redireciona o usuário de volta ao seu site usando o redirecionamento fornecido no plano de cobrança subjacente.
  3. Em seguida, você executa o contrato de cobrança usando um token fornecido por meio do redirecionamento do PayPal.

Este exemplo está configurando um servidor HTTP baseado em Express para mostrar o processo de contrato de cobrança.

Para iniciar o exemplo, primeiro precisamos definir nossa configuração. Adicionamos quatro requisitos, o SDK do PayPal, body-parser para lidar com corpos codificados em JSON, http para nossa integração simples de servidor e express para a estrutura Express. Em seguida, definimos nosso ID de cliente e segredo da criação de um aplicativo, configuramos o SDK para o sandbox e configuramos bodyParser para manipular corpos JSON.

var paypal = require('paypal-rest-sdk'),
    bodyParser = require('body-parser'),
    http = require('http'),
    app = require('express')();

var clientId = 'YOUR APPLICATION CLIENT ID';
var secret = 'YOUR APPLICATION SECRET';

paypal.configure({
  'mode': 'sandbox', //sandbox or live
  'client_id': clientId,
  'client_secret': secret
});

app.use(bodyParser.json());

Nosso primeiro passo no contrato de cobrança é criar uma rota para lidar com a criação de um contrato de cobrança e redirecionar o usuário para o PayPal para confirmar essa assinatura. Estamos supondo que um ID de plano de cobrança seja passado como um parâmetro de string de consulta, como ao carregar o seguinte URL com um ID de plano do exemplo anterior:

http://localhost:3000/createagreement?plan=P-3N543779E9831025ECYGDNVQ

Agora precisamos usar essas informações para criar o contrato de cobrança.

app.get('/createagreement', function(req, res){
    var billingPlan = req.query.plan;
    
    var isoDate = new Date();
    isoDate.setSeconds(isoDate.getSeconds() + 4);
    isoDate.toISOString().slice(0, 19) + 'Z';

    var billingAgreementAttributes = {
        "name": "Standard Membership",
        "description": "Food of the World Club Standard Membership",
        "start_date": isoDate,
        "plan": {
            "id": billingPlan
        },
        "payer": {
            "payment_method": "paypal"
        },
        "shipping_address": {
            "line1": "W 34th St",
            "city": "New York",
            "state": "NY",
            "postal_code": "10001",
            "country_code": "US"
        }
    };

    // Use activated billing plan to create agreement
    paypal.billingAgreement.create(billingAgreementAttributes, function (error, billingAgreement){
        if (error) {
            console.error(error);
            throw error;
        } else {
            //capture HATEOAS links
            var links = {};
            billingAgreement.links.forEach(function(linkObj){
                links[linkObj.rel] = {
                    'href': linkObj.href,
                    'method': linkObj.method
                };
            })

            //if redirect url present, redirect user
            if (links.hasOwnProperty('approval_url')){
                res.redirect(links['approval_url'].href);
            } else {
                console.error('no redirect URI present');
            }
        }
    });
});

Começamos extraindo o ID do plano de cobrança da string de consulta e criamos a data em que o plano deve começar.

A próxima definição de objeto, billingAgreementAttributes, consiste em informações para a assinatura. Ele contém informações legíveis sobre o plano, uma referência ao ID do plano de cobrança, o método de pagamento e detalhes de envio (se necessário para a assinatura).

Em seguida, uma chamada para billingAgreement.create(...) é feita, passando o objeto billingAgreementAttributes que acabamos de criar. Se tudo for bem-sucedido, devemos ter um objeto de contrato de cobrança retornado para nós contendo detalhes sobre nossa assinatura recém-criada. Esse objeto também contém vários links HATEOAS que nos fornecem os próximos passos que podem ser dados neste acordo recém-criado. O que nos interessa aqui é rotulado como approval_url.

Percorremos todos os links fornecidos para colocá-los em um objeto facilmente referenciado. Se approval_url for um desses links, redirecionamos o usuário para esse link, que é o PayPal.

Nesse ponto, o usuário confirma a assinatura no PayPal e é redirecionado de volta para a URL fornecida no plano de cobrança subjacente. Junto com esse URL, o PayPal também passará um token ao longo da string de consulta. Esse token é o que vamos usar para executar (ou iniciar) a assinatura.

Vamos configurar essa funcionalidade na rota a seguir.

app.get('/processagreement', function(req, res){
    var token = req.query.token;
    
    paypal.billingAgreement.execute(token, {}, function (error, billingAgreement) {
        if (error) {
            console.error(error);
            throw error;
        } else {
            console.log(JSON.stringify(billingAgreement));
            res.send('Billing Agreement Created Successfully');
        }
    });
});

Extraímos o token da string de consulta e, em seguida, fazemos uma chamada para billingAgreement.execute, passando esse token. Se tudo der certo, agora temos uma assinatura válida para o usuário. O objeto de devolução contém informações sobre o contrato de faturamento ativo.

Por fim, configuramos nosso servidor HTTP para escutar o tráfego de nossas rotas.

//create server
http.createServer(app).listen(3000, function () {
   console.log('Server started: Listening on port 3000');
});

Etapa 1: criando um modelo de assinatura usando um plano de cobrança (amostra de nó) #

Ao criar uma assinatura para um usuário, primeiro você precisa criar e ativar um plano de cobrança no qual um usuário se inscreveu usando um contrato de cobrança. O processo completo para criar uma assinatura é detalhado nas observações deste tópico.

Neste exemplo, usaremos o PayPal Node SDK. Você pode obtê-lo do NPM usando o seguinte comando:

npm install paypal-rest-sdk

Em nosso arquivo .js, primeiro configuramos nossa configuração do SDK, que inclui adicionar um requisito para o SDK, definir nosso ID de cliente e segredo de [criar nosso aplicativo][2] e, em seguida, configurar o SDK para o ambiente de sandbox.

var paypal = require('paypal-rest-sdk');

var clientId = 'YOUR CLIENT ID';
var secret = 'YOUR SECRET';

paypal.configure({
  'mode': 'sandbox', //sandbox or live
  'client_id': clientId,
  'client_secret': secret
});

Em seguida, precisamos configurar dois objetos JSON. O objeto billingPlanAttribs contém as informações e o detalhamento do pagamento do plano de cobrança que podemos assinar para os usuários, e o objeto billingPlanUpdateAttributes contém valores para definir o plano de cobrança para um estado ativo, permitindo que ele seja usado.

var billingPlanAttribs = {
    "name": "Food of the World Club Membership: Standard",
    "description": "Monthly plan for getting the t-shirt of the month.",
    "type": "fixed",
    "payment_definitions": [{
        "name": "Standard Plan",
        "type": "REGULAR",
        "frequency_interval": "1",
        "frequency": "MONTH",
        "cycles": "11",
        "amount": {
            "currency": "USD",
            "value": "19.99"
        }
    }],
    "merchant_preferences": {
        "setup_fee": {
            "currency": "USD",
            "value": "1"
        },
        "cancel_url": "http://localhost:3000/cancel",
        "return_url": "http://localhost:3000/processagreement",
        "max_fail_attempts": "0",
        "auto_bill_amount": "YES",
        "initial_fail_amount_action": "CONTINUE"
    }
};

var billingPlanUpdateAttributes = [{
    "op": "replace",
    "path": "/",
    "value": {
        "state": "ACTIVE"
    }
}];

Dentro do objeto billingPlanAttribs, existem algumas informações relevantes:

  • nome/descrição/tipo: informações visuais básicas para descrever o plano e o tipo de plano.
  • payment_definitions: Informações sobre como o plano deve funcionar e ser cobrado. Mais detalhes sobre os campos [aqui][3].
  • merchant_preferences: estruturas de taxas adicionais, URLs de redirecionamento e configurações para o plano de assinatura. Mais detalhes sobre os campos [aqui][4].

Com esses objetos no lugar, agora podemos criar e ativar o plano de cobrança.

paypal.billingPlan.create(billingPlanAttribs, function (error, billingPlan){
    if (error){
        console.log(error);
        throw error;
    } else {
        // Activate the plan by changing status to Active
        paypal.billingPlan.update(billingPlan.id, billingPlanUpdateAttributes, function(error, response){
            if (error) {
                console.log(error);
                throw error;
            } else {
                console.log(billingPlan.id);
            }
        });
    }
});

Chamamos billingPlan.create(...), passando o objeto billingPlanAttribs que acabamos de criar. Se isso for bem-sucedido, o objeto de retorno conterá informações sobre o plano de cobrança. Por causa do exemplo, precisamos apenas usar o ID do plano de cobrança para ativar o plano para uso.

Em seguida, chamamos billingPlan.update(...), passando o ID do plano de cobrança e o objeto billingPlanUpdateAttributes que criamos anteriormente. Se isso for bem-sucedido, nosso plano de cobrança estará ativo e pronto para uso.

Para criar uma assinatura para um usuário (ou vários usuários) neste plano, precisaremos referenciar o ID do plano de cobrança (billingPlan.id acima), então armazene-o em um local que possa ser facilmente referenciado.

Na segunda etapa de assinatura, precisamos criar um contrato de cobrança com base no plano que acabamos de criar e executá-lo para começar a processar as assinaturas de um usuário.
