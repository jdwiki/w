Esses exemplos abrangem exemplos práticos de como usar webhooks do PayPal para fornecer monitoramento de eventos para seu aplicativo e pagamentos.

Testando Webhooks do Sandbox com ngrok e Express (Node) #

Neste exemplo, veremos como testar notificações de webhook no sandbox, usando ngrok para fornecer um túnel para nosso ouvinte HTTP Node, executado em localhost, para a Internet. Para este exemplo, usaremos o Node para configurar webhooks de notificação para eventos de pagamento (como um pagamento sendo feito) e, em seguida, configuraremos o servidor para ouvir mensagens HTTP POST recebidas dos eventos de webhook.

Existem alguns passos que vamos seguir aqui para que isso aconteça:

  1. Configure um servidor simples para escutar o tráfego POST de entrada dos webhooks, que será a notificação do PayPal, e comece a escutar no localhost.
  2. Em seguida, use o ngrok para fornecer um túnel do host local para a Internet para que o PayPal possa postar a notificação.
  3. Por fim, inscreva nosso aplicativo (com base nas credenciais fornecidas) para eventos de webhook que queremos rastrear, fornecendo o URI ngrok público da etapa 2.

Criando um ouvinte de Webhooks

A primeira coisa que precisamos fazer é criar o ouvinte. A razão pela qual estamos começando com o listener é porque precisamos do ngrok live URL para fornecer aos webhooks quando os criamos ou atualizamos.

var bodyParser = require('body-parser'),
    http = require('http'),
    app = require('express')();

app.use(bodyParser.json());

app.post('/', function(req, res){
    console.log(JSON.stringify(req.body));
});

//create server
http.createServer(app).listen(3001, function () {
   console.log('Server started: Listening on port 3001');
});

Nosso listener é uma rota simples usando o Express. Ouvimos qualquer tráfego POST de entrada e, em seguida, enviamos o corpo do POST para o console. Podemos usar isso para fazer o que quisermos com o ouvinte quando ele chegar.

Quando criamos o servidor HTTP no final, nós o configuramos para escutar na porta 3001 do host local. Execute esse script agora para começar a escutar o tráfego.

Usando o ngrok para expor o ouvinte à Internet

Com o listener configurado em localhost:3001, nosso próximo trabalho é expor esse script à internet, para que o tráfego possa ser enviado a ele, que é o trabalho do ngrok.

Execute o seguinte comando em uma janela de terminal:

ngrok http 3001

Isso iniciará o processo de fornecimento de um túnel ao vivo para localhost na porta 3001 e fornecerá as seguintes informações após a execução:

Como podemos ver, o endereço ativo que podemos usar para apontar o webhook do PayPal para nosso listener em execução no localhost é http(s)://055b3480.ngrok.io. Isso é tudo que precisamos saber para configurar o ouvinte.

Inscrevendo-se nas notificações

Nosso último passo é criar webhooks para nosso aplicativo, que criará notificações quando ocorrerem determinados eventos com pagamentos, reembolsos, etc em nosso aplicativo. Nós só precisamos criar esses webhooks uma vez para vinculá-los ao aplicativo, para que eles não precisem ser executados toda vez que você quiser usá-los.

Primeiro, configuramos o ambiente do PayPal adicionando o requisito para o SDK do PayPal Node, nosso ID / segredo do cliente para criar um aplicativo e, em seguida, configurando o ambiente para sandbox.

var paypal = require('paypal-rest-sdk');

var clientId = 'YOUR APPLICATION CLIENT ID';
var secret = 'YOUR APPLICATION SECRET';

paypal.configure({
  'mode': 'sandbox', //sandbox or live
  'client_id': clientId,
  'client_secret': secret
});

Em seguida, configuramos a estrutura JSON para nossos webhooks. O webhooks contém duas informações, o url para o qual todos os eventos do webhook devem ser enviados e o event_types que queremos assinar.

No caso desta amostra, o url é definido para nosso URL ativo do ngrok, e os eventos que estamos ouvindo são casos em que os pagamentos são concluídos ou negados.

Para obter uma lista completa de eventos em potencial, consulte https://developer.paypal.com/docs/integration/direct/rest-webhooks-overview/#event-type-support.

Por último, passamos o objeto webhooks para a chamada para criar os webhooks, notification.webhook.create. Se for bem-sucedido, o PayPal enviará notificações para o endpoint que especificamos, que está sendo executado no localhost.

var webhooks = {
    "url": "https://436e4d13.ngrok.io",
    "event_types": [{
        "name": "PAYMENT.SALE.COMPLETED"
    },{
        "name": "PAYMENT.SALE.DENIED"
    }
]};

paypal.notification.webhook.create(webhooks, function (err, webhook) {
    if (err) {
        console.log(err.response);
        throw error;
    } else {
        console.log("Create webhook Response");
        console.log(webhook);
    }
});

Assim que emitirmos um pagamento usando essas credenciais do aplicativo, as informações sobre o estado do pagamento serão enviadas ao terminal que configuramos.

Atualizando um Webhook com um novo URL (amostra de nó) #

Este exemplo mostrará como atualizar um URL de encaminhamento de webhook existente (onde as notificações devem ser postadas). Para executar isso, você deve ter o ID fornecido pelo PayPal quando criou seus webhooks pela primeira vez.

Primeiro, adicione o SDK do PayPal e configure o ambiente (sandbox abaixo).

var paypal = require('paypal-rest-sdk');

var clientId = 'YOUR APPLICATION CLIENT ID';
var secret = 'YOUR APPLICATION SECRET';

paypal.configure({
    'mode': 'sandbox', //sandbox or live
    'client_id': clientId,
    'client_secret': secret
});

Em seguida, configure a estrutura JSON e os detalhes do webhook. Atribua o ID do seu webhook a webhookId primeiro. Em seguida, no webhookUpdate, especifique uma operação de substituição, defina o path como /url para especificar uma atualização desse recurso e forneça a nova URL para substituí-la por value.

var webhookId = "YOUR WEBHOOK ID";
var webhookUpdate = [{
    "op": "replace",
    "path": "/url",
    "value": "https://64fb54a2.ngrok.io"
}];

Por último, chame notification.webhook.replace(...), passando webhookId e webhookUpdate.

paypal.notification.webhook.replace(webhookId, webhookUpdate, function (err, res) { if (err) { console.log(err); throw err; } else { console.log(JSON.stringify(res)); } });

Se tudo der certo, um objeto semelhante ao seguinte deve ser fornecido de volta do PayPal e, no caso desta amostra, exibido no terminal com as informações recém-atualizadas.
