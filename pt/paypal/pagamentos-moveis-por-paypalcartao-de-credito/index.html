Android: Aceitando um pagamento por PayPal/cartão de crédito #

Neste tutorial, vamos aprender como configurar o PayPal Android SDK para processar um pagamento simples via PayPal ou compra com cartão de crédito. Ao final deste exemplo, você deverá ter um simples botão em um aplicativo que, ao ser clicado, encaminhará o usuário ao PayPal para confirmar um pagamento definido, depois retornará o usuário ao aplicativo e registrará a confirmação do pagamento.

O código completo do aplicativo para este exemplo está disponível no PayPal Developer Github Repository.

Vamos começar.

A primeira etapa é obter e adicionar o SDK ao seu projeto. Adicionamos a referência às nossas dependências build.gradle assim:

dependencies {
    compile 'com.paypal.sdk:paypal-android-sdk:2.14.1'
    ...
}

Agora vamos para o nosso arquivo MainActivity.java (ou onde quer que você queira adicionar a integração do botão PayPal), e adicionamos um objeto config para nosso ID de cliente e o ambiente (sandbox) que usaremos.

private static PayPalConfiguration config = new PayPalConfiguration()
    .environment(PayPalConfiguration.ENVIRONMENT_SANDBOX)
    .clientId("YOUR CLIENT ID");

Agora vamos criar um botão em nosso método onCreate(...), que nos permitirá processar um pagamento via PayPal uma vez clicado.

@Override
protected void onCreate(Bundle savedInstanceState){
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    final Button button = (Button) findViewById(R.id.paypal_button);
}

Agora precisamos definir a funcionalidade desse botão. Em seu res > layout > arquivo XML principal, você pode adicionar a seguinte definição para o botão, que definirá o texto e o manipulador onClick para o botão com o ID paypal_button.

<Button android:id="@+id/paypal_button"
    android:layout_height="wrap_content"
    android:layout_width="wrap_content"
    android:text="@string/paypal_button"
    android:onClick="beginPayment" />

Quando clicado, o botão chamará o método beginPayment(...). Podemos então adicionar o texto do botão ao nosso arquivo strings.xml, assim:

<string name="paypal_button">Pay with PayPal</string>

Com o botão no lugar, agora temos que lidar com o clique do botão para iniciar o processamento do pagamento. Adicione o seguinte método beginPayment(...) abaixo do nosso método onCreate(...) anterior.

public void beginPayment(View view){
    Intent serviceConfig = new Intent(this, PayPalService.class);
    serviceConfig.putExtra(PayPalService.EXTRA_PAYPAL_CONFIGURATION, config);
    startService(serviceConfig);

    PayPalPayment payment = new PayPalPayment(new BigDecimal("5.65"), 
        "USD", "My Awesome Item", PayPalPayment.PAYMENT_INTENT_SALE);

    Intent paymentConfig = new Intent(this, PaymentActivity.class);
    paymentConfig.putExtra(PayPalService.EXTRA_PAYPAL_CONFIGURATION, config);
    paymentConfig.putExtra(PaymentActivity.EXTRA_PAYMENT, payment);
    startActivityForResult(paymentConfig, 0);
}

O que estamos fazendo aqui é primeiro configurar a intenção do serviço (serviceConfig), usando o config que definimos anteriormente para nosso ID do cliente e o ambiente sandbox. Em seguida, especificamos o objeto de pagamento que queremos processar. Para este exemplo, estamos definindo um preço, moeda e descrição estáticos. Em sua aplicação final, esses valores devem ser obtidos a partir do que o usuário está tentando comprar na aplicação. Por fim, configuramos o paymentConfig, adicionando os objetos config e payment que havíamos definido anteriormente, e iniciamos a atividade.

Neste ponto, o usuário será apresentado às telas de login e pagamento do PayPal, permitindo que ele selecione se deseja pagar com PayPal ou cartão de crédito (via entrada manual ou card.io, se a câmera estiver disponível). Essa tela ficará mais ou menos assim:

Uma vez feito, precisamos ter um manipulador pronto para quando o PayPal encaminhar o usuário de volta ao aplicativo após a confirmação do pagamento ou cancelamento. Vamos substituir onActivityResult(...) para esse propósito.

@Override
protected void onActivityResult (int requestCode, int resultCode, Intent data){
    if (resultCode == Activity.RESULT_OK){
        PaymentConfirmation confirm = data.getParcelableExtra(
            PaymentActivity.EXTRA_RESULT_CONFIRMATION);
        if (confirm != null){
            try {
                Log.i("sampleapp", confirm.toJSONObject().toString(4));

                // TODO: send 'confirm' to your server for verification

            } catch (JSONException e) {
                Log.e("sampleapp", "no confirmation data: ", e);
            }
        }
    } else if (resultCode == Activity.RESULT_CANCELED) {
        Log.i("sampleapp", "The user canceled.");
    } else if (resultCode == PaymentActivity.RESULT_EXTRAS_INVALID) {
        Log.i("sampleapp", "Invalid payment / config set");
    }
}

Dentro do método onActivityResult(...), estamos verificando se o resultCode que retorna é RESULT_OK (pagamento confirmado pelo usuário), RESULT_CANCELED (pagamento cancelado pelo usuário) ou RESULT_EXTRAS_INVALID (houve um problema de configuração). No caso de uma confirmação válida, obtemos o objeto que é devolvido do pagamento e, neste exemplo, o registramos. O que será devolvido para nós deve ser algo como o seguinte:

{
    "client": {
        "environment": "sandbox",
        "paypal_sdk_version": "2.14.1",
        "platform": "Android",
        "product_name": "PayPal-Android-SDK"
    },
    "response": {
        "create_time": "2016-05-02T15:33:43Z",
        "id": "PAY-0PG63447RB821630KK1TXGTY",
        "intent": "sale",
        "state": "approved"
    },
    "response_type": "payment"
}

Se olharmos sob o objeto response, podemos ver que temos um estado de aprovado, o que significa que o pagamento foi confirmado. Nesse ponto, esse objeto deve ser enviado ao seu servidor para confirmar que um pagamento realmente foi realizado. Para obter mais informações sobre essas etapas, consulte estes documentos.

Nosso último passo é limpar em nosso onDestroy(...).

@Override
public void onDestroy(){
    stopService(new Intent(this, PayPalService.class));
    super.onDestroy();
}

Isso é tudo o que há para isso. Neste exemplo, criamos um botão simples para processar um pagamento com PayPal ou cartão de crédito. A partir deste ponto, há algumas próximas etapas para você expandir este exemplo:

  • Puxando informações de pagamento dinamicamente com base na seleção do produto do usuário no método beginPayment(...).
  • Enviando a confirmação do pagamento para o seu servidor e verificando se o pagamento realmente foi efetuado.
  • Lidar com os casos de usuário de erro e cancelamento dentro do aplicativo.
