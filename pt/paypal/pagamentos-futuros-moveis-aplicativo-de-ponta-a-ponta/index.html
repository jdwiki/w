Este exemplo mostra um exemplo prático de ponta a ponta da criação de um pagamento futuro do PayPal a partir de um dispositivo Android, usando um servidor Node.

Etapa 1 do Android: layout, inicialização e tratamento da resposta do servidor #

O código de amostra completo para este aplicativo (servidor Android + Node) está disponível no repositório do PayPal Developer Github.

A primeira etapa da criação da parte Android do nosso aplicativo é configurar um layout básico e lidar com as respostas que retornam do servidor que configuraremos no Node.

Comece criando um novo objeto PayPalConfiguration para hospedar as informações do seu aplicativo.

private static PayPalConfiguration config = new PayPalConfiguration()
        .environment(PayPalConfiguration.ENVIRONMENT_SANDBOX)
        .clientId("YOUR APPLICATION CLIENT ID")
        .merchantName("My Store")
        .merchantPrivacyPolicyUri(Uri.parse("https://www.example.com/privacy"))
        .merchantUserAgreementUri(Uri.parse("https://www.example.com/legal"));

Em seguida, adicionamos um botão simples ao onCreate(...) para atuar como nosso início de pagamento. Isso é simplesmente para desencadear a ação e deve ser colocado como o processo de iniciação para criar um pagamento futuro para um usuário (por exemplo, quando eles concordam com uma assinatura).

@Override
protected void onCreate(Bundle savedInstanceState){
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    final Button button = (Button) findViewById(R.id.paypal_button);
}

Em res > layout > activity_main.xml adicionamos a definição para o botão com sua ação associada, quando clicado chama beginFuturePayment(...), que definiremos em um minuto.

<Button android:id="@+id/paypal_button"
    android:layout_height="wrap_content"
    android:layout_width="wrap_content"
    android:text="@string/paypal_button"
    android:onClick="beginFuturePayment" />

Em res > valores > strings.xml adicionamos uma referência de string para o botão.

<string name="paypal_button">Process Future Payment</string>

Agora adicionamos o manipulador de botão, para iniciar a chamada para iniciar o processo de pagamento futuro quando o usuário clicar no botão. O que estamos fazendo aqui é iniciar o serviço de pagamento com o objeto de configuração que configuramos no início deste exemplo.

public void beginFuturePayment(View view){
    Intent serviceConfig = new Intent(this, PayPalService.class);
    serviceConfig.putExtra(PayPalService.EXTRA_PAYPAL_CONFIGURATION, config);
    startService(serviceConfig);

    Intent intent = new Intent(this, PayPalFuturePaymentActivity.class);
    intent.putExtra(PayPalService.EXTRA_PAYPAL_CONFIGURATION, config);
    startActivityForResult(intent, 0);
}

Quando essa chamada para fazer um pagamento futuro for iniciada, receberemos algumas informações que precisarão ser enviadas ao nosso servidor. Extraímos essas informações da solicitação de pagamento futuro válida (authCode e metadataId), então executamos a solicitação assíncrona ao servidor para concluir o pagamento futuro (detalhado na etapa 2).

@Override
protected void onActivityResult (int requestCode, int resultCode, Intent data){
    if (resultCode == Activity.RESULT_OK){
        PayPalAuthorization auth = data.getParcelableExtra(PayPalFuturePaymentActivity.EXTRA_RESULT_AUTHORIZATION);
        if (auth != null){
            try{
                //prepare params to be sent to server
                String authCode = auth.getAuthorizationCode();
                String metadataId = PayPalConfiguration.getClientMetadataId(this);
                String [] params = {authCode, metadataId};

                //process async server request for token + payment
                ServerRequest req = new ServerRequest();
                req.execute(params);

            } catch (JSONException e) {
                Log.e("FPSample", "JSON Exception: ", e);
            }
        }
    } else if (resultCode == Activity.RESULT_CANCELED) {
        Log.i("FPSample", "User canceled.");
    } else if (resultCode == PayPalFuturePaymentActivity.RESULT_EXTRAS_INVALID) {
        Log.i("FPSample", "Invalid configuration");
    }
}

Por fim, definimos nosso onDestroy().

@Override
public void onDestroy(){
    stopService(new Intent(this, PayPalService.class));
    super.onDestroy();
}

Etapa 2 do Android: solicitação de servidor assíncrono #

O código de amostra completo para este aplicativo (servidor Android + Node) está disponível no repositório do PayPal Developer Github.

Neste ponto, o botão de pagamentos futuros do PayPal foi clicado, temos um código de autenticação e um ID de metadados do SDK do PayPal e precisamos passá-los para nosso servidor para concluir o processo de pagamento futuro.

No processo em segundo plano abaixo, estamos fazendo algumas coisas:

  • Configuramos o URI para que nosso servidor seja http://10.0.2.2:3000/fpstore, que está atingindo o endpoint /fpstore do nosso servidor rodando em localhost.

  • O objeto JSON que será enviado é então configurado, que contém o código de autenticação e o ID de metadados.

  • A conexão é então feita. No caso de uma solicitação bem-sucedida (intervalo 200 / 201), podemos esperar uma resposta do servidor. Lemos essa resposta e depois a devolvemos.

  • Por último, temos um método onPostExecute(...) configurado para lidar com a string do servidor retornada. No caso deste exemplo, é simplesmente registrado.

    public class ServerRequest extends AsyncTask<String, Void, String> { protected String doInBackground(String[] params){ HttpURLConnection connection = null; try{ //set connection to connect to /fpstore on localhost URL u = new URL(“http://10.0.2.2:3000/fpstore”); connection = (HttpURLConnection) u.openConnection(); connection.setRequestMethod(“POST”);

              //set configuration details
          connection.setRequestProperty("Content-Type", "application/json");
          connection.setRequestProperty("Accept", "application/json");
          connection.setAllowUserInteraction(false);
          connection.setConnectTimeout(10000);
          connection.setReadTimeout(10000);

          //set server post data needed for obtaining access token
          String json = "{\"code\": \"" + params[0] + "\", \"metadataId\": \"" + params[1] + "\"}";
          Log.i("JSON string", json);

          //set content length and config details
          connection.setRequestProperty("Content-length", json.getBytes().length + "");
          connection.setDoInput(true);
          connection.setDoOutput(true);
          connection.setUseCaches(false);

          //send json as request body
          OutputStream outputStream = connection.getOutputStream();
          outputStream.write(json.getBytes("UTF-8"));
          outputStream.close();

          //connect to server
          connection.connect();

          //look for 200/201 status code for received data from server
          int status = connection.getResponseCode();
          switch (status){
              case 200:
              case 201:
                  //read in results sent from the server
                  BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(connection.getInputStream()));
                  StringBuilder sb = new StringBuilder();
                  String line;
                  while ((line = bufferedReader.readLine()) != null){
                      sb.append(line + "\n");
                  }
                  bufferedReader.close();

                  //return received string
                  return sb.toString();
          }

      } catch (MalformedURLException ex) {
          Log.e("HTTP Client Error", ex.toString());
      } catch (IOException ex) {
          Log.e("HTTP Client Error", ex.toString());
      } catch (Exception ex) {
          Log.e("HTTP Client Error", ex.toString());
      } finally {
          if (connection != null) {
              try{
                  connection.disconnect();
              } catch (Exception ex) {
                  Log.e("HTTP Client Error", ex.toString());
              }
          }
      }
      return null;
  }

  protected void onPostExecute(String message){
      //log values sent from the server - processed payment
      Log.i("HTTP Client", "Received Return: " + message);
  }

    }

Etapa 3 do Android: Servidor Node para obter o token de acesso e processar o pagamento #

O código de amostra completo para este aplicativo (servidor Android + Node) está disponível no repositório do PayPal Developer Github.

A partir da etapa 2, uma solicitação assíncrona foi feita ao nosso servidor no endpoint /fpstore, passando o código de autenticação e o ID de metadados. Agora precisamos trocá-los por um token para concluir a solicitação e processar o pagamento futuro.

Primeiro, configuramos nossas variáveis ​​de configuração e objeto.

var bodyParser = require('body-parser'),
    http = require('http'),
    paypal = require('paypal-rest-sdk'),
    app = require('express')();

var client_id = 'YOUR APPLICATION CLIENT ID';
var secret = 'YOUR APPLICATION SECRET';

paypal.configure({
    'mode': 'sandbox',
    'client_id': client_id,
    'client_secret': secret
});

app.use(bodyParser.urlencoded({ extended: false }))
app.use(bodyParser.json());

Agora configuramos uma rota Express que escutará as solicitações POST enviadas para o endpoint /fpstore do nosso código Android.

Estamos fazendo uma série de coisas nesta rota:

  • Capturamos o código de autenticação e o ID de metadados do corpo do POST.

  • Em seguida, fazemos uma solicitação para generateToken(), passando pelo objeto de código. Se for bem-sucedido, obtemos um token que pode ser usado para criar o pagamento.

  • Em seguida, os objetos de configuração são criados para o pagamento futuro a ser feito, e uma solicitação para payment.create(...) é feita, passando os objetos de pagamento futuro e de configuração de pagamento. Isso cria o pagamento futuro.

    app.post(’/fpstore’, function(req, res){ var code = {‘authorization_code’: req.body.code}; var metadata_id = req.body.metadataId;

      //generate token from provided code
  paypal.generateToken(code, function (error, refresh_token) {
      if (error) {
          console.log(error);
          console.log(error.response);
      } else {
          //create future payments config 
          var fp_config = {'client_metadata_id': metadata_id, 'refresh_token': refresh_token};

          //payment details
          var payment_config = {
              "intent": "sale",
              "payer": {
                  "payment_method": "paypal"
              },
              "transactions": [{
                  "amount": {
                      "currency": "USD",
                      "total": "3.50"
                  },
                  "description": "Mesozoic era monster toy"
              }]
          };

          //process future payment
          paypal.payment.create(payment_config, fp_config, function (error, payment) {
              if (error) {
                  console.log(error.response);
                  throw error;
              } else {
                  console.log("Create Payment Response");
                  console.log(payment);

                  //send payment object back to mobile
                  res.send(JSON.stringify(payment));
              }
          });
      }
  });

    });

Por fim, criamos o servidor para escutar na porta 3000.

//create server
http.createServer(app).listen(3000, function () {
   console.log('Server started: Listening on port 3000');
});
