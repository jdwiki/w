Parâmetros #

Parâmetro Detalhes
clientId Seu ID de cliente do aplicativo PayPal (credenciais OAuth 2)
ligações Objeto de referência simples para todos os links HATEOAS de retorno do PayPal
pagamentoId O ID do pagamento devolvido do PayPal para concluir o pagamento
payerId O ID do pagador devolvido do PayPal para concluir o pagamento
paypal Referência do SDK do nó do PayPal
payReq Objeto JSON contendo informações de pagamento para transação
req O objeto de solicitação do pedido do servidor
res O objeto de resposta da solicitação do servidor
segredo O segredo do seu aplicativo do PayPal (credenciais do OAuth 2)

Esses exemplos abordam como processar um pagamento via PayPal, usando os SDKs do PayPal. Estes são exemplos de solicitação simples que descrevem o processo de várias etapas para permitir essa opção de pagamento.

Exemplo de servidor Node Express #

Neste exemplo, vamos configurar uma integração de servidor Express para mostrar como processar um pagamento com o PayPal, usando o PayPal Node SDK. Usaremos uma estrutura JSON estática para os detalhes de pagamento por questões de brevidade.

Existem três etapas gerais que seguiremos ao criar as funções para lidar com o pagamento do PayPal:

  1. Criamos um objeto JSON contendo o pagamento que pretendemos processar através do PayPal. Em seguida, enviamos para o PayPal para obter um link para redirecionar o usuário para confirmar o pagamento.
  2. Em seguida, redirecionamos o usuário para o PayPal para confirmar o pagamento. Uma vez confirmado, o PayPal redireciona o usuário de volta ao nosso aplicativo.
  3. Uma vez retornado ao aplicativo, concluímos o pagamento em nome do usuário.

Dividindo isso como um aplicativo Node simples, começamos obtendo o SDK do PayPal Node do NPM:

npm install paypal-rest-sdk

Em seguida, configuramos a configuração e os pacotes do aplicativo.

var http = require('http'),
    paypal = require('paypal-rest-sdk'),
    bodyParser = require('body-parser'),
    app = require('express')();

var client_id = 'YOUR APPLICATION CLIENT ID';
var secret = 'YOUR APPLICATION SECRET';

//allow parsing of JSON bodies
app.use(bodyParser.json());

//configure for sandbox environment
paypal.configure({
    'mode': 'sandbox', //sandbox or live
    'client_id': client_id,
    'client_secret': secret
});

Exigimos quatro requisitos para este aplicativo:

  1. O pacote HTTP para nosso servidor.
  2. O pacote do SDK do nó do PayPal.
  3. O pacote bodyParser para trabalhar com corpos codificados em JSON.
  4. A estrutura Express para nosso servidor.

As próximas linhas configuram variáveis ​​para o ID do cliente e o segredo que foram obtidos ao criar um aplicativo. Em seguida, configuramos o bodyParser para permitir corpos codificados em JSON, configuramos nosso aplicativo usando os detalhes do aplicativo e especificamos o ambiente em que estamos trabalhando (ao vivo para produção ou sandbox para teste).

Agora vamos criar a rota para criar uma solicitação de pagamento com o PayPal.

app.get('/create', function(req, res){
    //build PayPal payment request
    var payReq = JSON.stringify({
        'intent':'sale',
        'redirect_urls':{
            'return_url':'http://localhost:3000/process',
            'cancel_url':'http://localhost:3000/cancel'
        },
        'payer':{
            'payment_method':'paypal'
        },
        'transactions':[{
            'amount':{
                'total':'7.47',
                'currency':'USD'
            },
            'description':'This is the payment transaction description.'
        }]
    });

    paypal.payment.create(payReq, function(error, payment){
        if(error){
            console.error(error);
        } else {
            //capture HATEOAS links
            var links = {};
            payment.links.forEach(function(linkObj){
                links[linkObj.rel] = {
                    'href': linkObj.href,
                    'method': linkObj.method
                };
            })
        
            //if redirect url present, redirect user
            if (links.hasOwnProperty('approval_url')){
                res.redirect(links['approval_url'].href);
            } else {
                console.error('no redirect URI present');
            }
        }
    });
});

A primeira coisa que fazemos é configurar o objeto JSON de solicitação de pagamento, que contém as informações que precisamos fornecer ao PayPal para criar o pagamento. Definimos o intent para sale, especificamos os URLs de redirecionamento (para onde o PayPal deve encaminhar o usuário após confirmar / cancelar o pagamento), adicionamos um payment_method de paypal para sinalizar que faremos um PayPal pagamento e, em seguida, especifique as informações da transação para o pagador confirmar.

Chamamos então payment.create(...), passando nosso objeto payReq. Isso enviará a solicitação de criação de pagamento para o PayPal. Assim que retornar, e for bem-sucedido, podemos percorrer os links fornecidos HATEOAS no objeto de retorno para extrair a URL para a qual precisamos redirecionar o usuário, que está rotulada em approval_url.

O formato dos links HATEOAS pode causar um código de referência frágil se usado diretamente, portanto, percorremos todos os links fornecidos e os colocamos em um objeto de referência melhor para proteção futura contra alterações. Se o approval_url for encontrado nesse objeto, redirecionamos o usuário.

Neste momento o usuário é redirecionado para o PayPal para confirmar o pagamento. Uma vez feito isso, eles são redirecionados de volta para o return_url que especificamos na função createPayment(...).

Agora temos que fornecer uma rota para lidar com essa devolução, para concluir o pagamento.

app.get('/process', function(req, res){
    var paymentId = req.query.paymentId;
    var payerId = { 'payer_id': req.query.PayerID };

    paypal.payment.execute(paymentId, payerId, function(error, payment){
        if(error){
            console.error(error);
        } else {
            if (payment.state == 'approved'){ 
                res.send('payment completed successfully');
            } else {
                res.send('payment not successful');
            }
        }
    });
});

Quando o usuário retornar ao seu aplicativo, haverá três parâmetros de string de consulta que também serão enviados, o paymentId, PayerID e o token. Só precisamos lidar com os dois primeiros.

Extraímos os parâmetros e colocamos o PayerID em um objeto simples para a necessidade da etapa de execução do pagamento. Em seguida, é feita uma chamada para payment.execute(...), passando esses dois parâmetros, para concluir o pagamento.

Uma vez que a solicitação é feita, vemos se o pagamento foi concluído com sucesso verificando se payment.state está definido como approved. Nesse caso, podemos armazenar o que precisamos do objeto de pagamento que é retornado.

Nossa última etapa é inicializar nosso servidor e ouvir o tráfego que chega às rotas que especificamos.

//create server
http.createServer(app).listen(3000, function () {
   console.log('Server started: Listening on port 3000');
});

Uma vez que o servidor é inicializado, ir para http://localhost:3000/create inicializa o processo de pagamento.
