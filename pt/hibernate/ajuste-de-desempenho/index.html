Use composição em vez de herança #

O Hibernate possui algumas estratégias de herança. O tipo de herança JOINED faz um JOIN entre a entidade filha e a entidade pai.

O problema com esta abordagem é que o Hibernate sempre traz os dados de todas as tabelas envolvidas na herança.

Por exemplo, se você tiver as entidades Bicycle e MountainBike usando o tipo de herança JOINED:

@Entity
@Inheritance(strategy = InheritanceType.JOINED)
public abstract class Bicycle {

}

E:

@Entity
@Inheritance(strategy = InheritanceType.JOINED)
public class MountainBike extends Bicycle {

}

Qualquer consulta JPQL que atingir MountainBike trará os dados Bicycle, criando uma consulta SQL como:

select mb.*, b.* from MountainBike mb JOIN Bicycle b ON b.id = mb.id WHERE ...

Se você tiver outro pai para Bicycle (como Transport, por exemplo), esta consulta acima trará os dados desse pai também, fazendo um JOIN extra.

Como você pode ver, este é um tipo de mapeamento EAGER também. Você não tem a opção de trazer apenas os dados da tabela MountainBike usando esta estratégia de herança.

O melhor para o desempenho é usar composição em vez de herança.

Para fazer isso, você pode mapear a entidade MountainBike para ter um campo bicycle:

@Entity
public class MountainBike {

    @OneToOne(fetchType = FetchType.LAZY)
    private Bicycle bicycle;

}

E Bicicleta:

@Entity
public class Bicycle {

}

Cada consulta agora trará apenas os dados MountainBike por padrão.

Não use o tipo de busca EAGER #

O Hibernate pode usar dois tipos de busca quando você está mapeando o relacionamento entre duas entidades: EAGER e LAZY.

Em geral, o tipo de busca EAGER não é uma boa ideia, porque diz ao JPA para sempre buscar os dados, mesmo quando esses dados não são necessários.

Por exemplo, se você tiver uma entidade Person e o relacionamento com Address assim:

@Entity
public class Person {

  @OneToMany(mappedBy="address", fetch=FetchType.EAGER)
  private List<Address> addresses;

}

Sempre que você consultar uma Person, a lista de Address desta Person será retornada também.

Então, em vez de mapear sua entidade com:

@ManyToMany(mappedBy="address", fetch=FetchType.EAGER)

Usar:

@ManyToMany(mappedBy="address", fetch=FetchType.LAZY)

Outra coisa para prestar atenção são os relacionamentos @OneToOne e @ManyToOne. Ambos são EAGER por padrão. Então, se você está preocupado com o desempenho do seu aplicativo, você precisa definir a busca para esse tipo de relacionamento:

@ManyToOne(fetch=FetchType.LAZY)

E:

@OneToOne(fetch=FetchType.LAZY)
