Primeiro projeto vazio básico #

Criação inicial e download (CakePHP 3.x) #

A maneira mais fácil de criar um novo projeto do CakePHP é através do Composer (se você não conhece o composer, veja aqui para mais informações)

Instalar o Compositor #

Se você precisar instalá-lo e estiver em uma máquina Windows, siga este guia

Se você estiver no Linux/Unix/OSX siga este guia

Crie o primeiro projeto CakePHP #

Abra uma janela de console e navegue até sua instalação do php (no Windows com a instalação padrão do xampp, este é C:\xampp\php)

Para criar um projeto vazio, execute o seguinte comando:

php composer.phar create-project --prefer-dist cakephp/app name_of_your_project

Baking/Model/View/Controllers #

A magia do CakePHP é o cozimento - uma geração automatizada de controlador, modelo e código de visualização com opções básicas de CRUD.

Antes de cozer você precisa ter sua conexão de banco de dados configurada. Para fazer isso você precisa editar o arquivo config/app.php em seu projeto.

'Datasources' => [
'default' => [
    'className' => 'Cake\Database\Connection',
    'driver' => 'Cake\Database\Driver\Mysql',
    'persistent' => false,
    'host' => 'localhost',
    'username' => 'my_app', //in basic xampp: root
    'password' => 'sekret', //in basic xampp: ''
    'database' => 'my_app', //name of the database you want to connect to your project
    'encoding' => 'utf8',
    'timezone' => 'UTC',
    'cacheMetadata' => true,
]

],

Se seu banco de dados estiver conectado corretamente, digite bin/cake bake na pasta raiz do seu projeto em uma janela do console.

Isso deve gerar algo assim:

Welcome to CakePHP v3.1.6 Console
---------------------------------------------------------------
App : src
Path: /var/www/cakephp.dev/src/
PHP: 5.5.8
---------------------------------------------------------------
The following commands can be used to generate skeleton code for your application.

Available bake commands:

- all
- behavior
- cell
- component
- controller
- fixture

- migration_snapshot

By using `cake bake [name]` you can invoke a specific bake task.

Para fins de simplicidade, vamos assar tudo com as configurações padrão. Para fazer isso você entra

cake bake all

Isso produzirá algo nesse sentido:

Welcome to CakePHP v3.2.11 Console
---------------------------------------------------------------

Path: C:\xampp\htdocs\tipping\src\

---------------------------------------------------------------
Bake All

- users
- blogs
Run `cake bake all [name]` to generate skeleton files.

Ao executar cake bake all <modelNameYouWantToBake>, os arquivos de modelo, tabela, controlador, fixture e view são criados. Execute isso para cada nome de modelo possível e você terá um projeto em funcionamento com opções básicas de CRUD.

Agora você pode abrir seu navegador e ver como fica e começar a estender o projeto por sua própria lógica

Introdução Básica ao CakePHP 2.x #

Falaremos sobre a estrutura de diretórios do CakePHP, o que significa cada pasta.

CakePHP tem algumas pastas principais #

  1. app - Contém o código fonte do nosso aplicativo, todo o nosso código está neste diretório.
  2. lib - Este é o núcleo do cakephp liberado, ele contém todo o código base da biblioteca do cakephp. A edição de código dentro deste diretório não é sugerida, pois pode causar erros ao atualizar a biblioteca cakephp.
  3. plugins - Contém o código dos plugins cakephp que serão usados ​​para nossa aplicação.
  4. fornecedores - Este contém código externo, Este código não usará a biblioteca cakephp.
  5. index.php - Este é o arquivo de índice.

Podemos ter várias aplicações hospedadas dentro de um único projeto. ou seja, eles podem usar as mesmas pastas lib, plugin e fornecedores.

Para modificar o código lib, a melhor prática é estendê-los em nossa pasta de aplicativos e realizar as modificações.

A pasta de plugins e fornecedores é compartilhada por todos os aplicativos hospedados no mesmo diretório.

index.php é o arquivo que é chamado primeiro.

Agora devemos pular para a pasta do nosso aplicativo #

Instalação ou Configuração #

Requisitos #

O seguinte guia de configuração é para cakephp 2.8 e superior. Todas as versões do cakephp inferiores a 2.8 não são compatíveis com o php 7

Servidor HTTP. Por exemplo: Apache. Ter mod_rewrite é preferível, mas não é obrigatório.

  • PHP 5.5.9 ou superior (incluindo PHP 7).
  • Extensão PHP mbstring
  • extensão PHP intl

Atenção! Tanto no XAMPP quanto no WAMP, a extensão mbstring está funcionando por padrão. No XAMPP, a extensão intl está incluída, mas você precisa descomentar extension=php_intl.dll no php.ini e reiniciar o servidor através do Painel de Controle do XAMPP. No WAMP, a extensão intl é “ativada” por padrão, mas não funciona. Para fazê-lo funcionar, você deve ir para a pasta php (por padrão) C:\wamp\bin\php\php{versão}, copiar todos os arquivos que se parecem com icu*.dll e colá-los no diretório bin do apache C: \wamp\bin\apache\apache{versão}\bin. Em seguida, reinicie todos os serviços e deve estar OK.

Embora um mecanismo de banco de dados não seja necessário, imaginamos que a maioria dos aplicativos utilizará um. O CakePHP suporta uma variedade de mecanismos de armazenamento de banco de dados:

  • MySQL (5.1.10 ou superior)
  • PostgreSQL
  • Microsoft SQL Server (2008 ou superior)
  • SQLite 3

Estrutura de pastas do CakePHP3 #

Depois de baixar, estes são os arquivos e pastas que você deve ver:

  • A pasta bin contém os executáveis ​​do console do Cake.
  • A pasta config contém os arquivos de configuração que o CakePHP usa. Detalhes de conexão de banco de dados, bootstrapping, arquivos de configuração principal e muito mais devem ser armazenados aqui.
  • A pasta de plugins é onde os plugins que seu aplicativo usa são armazenados.
  • A pasta de logs normalmente contém seus arquivos de log, dependendo de sua configuração de log.
  • A pasta src será onde os arquivos do seu aplicativo serão colocados.
  • A pasta de testes será onde você colocará os casos de teste para seu inscrição.
  • A pasta tmp é onde o CakePHP armazena dados temporários. Os dados reais que ele armazena dependem de como você configurou o CakePHP, mas essa pasta geralmente é usada para armazenar descrições de modelos e, às vezes, informações de sessão.
  • A pasta do fornecedor é onde o CakePHP e outras dependências de aplicativos serão instalados. Assuma um compromisso pessoal de não editar arquivos nesta pasta.
  • O diretório webroot é a raiz do documento público do seu aplicativo. Ele contém todos os arquivos que você deseja que sejam acessíveis publicamente.

Certifique-se de que as pastas tmp e logs existam e estejam gravável, caso contrário o desempenho do seu aplicativo será severamente impactado. No modo de depuração, o CakePHP irá avisá-lo, se não estiver o caso.

Dentro da pasta src #

A pasta src do CakePHP é onde você fará a maior parte do desenvolvimento de sua aplicação.

A pasta Console contém os comandos do console e as tarefas do console para seu aplicativo. Para obter mais informações, consulte Shells, tarefas e ferramentas de console.

A pasta Controller contém os controladores do seu aplicativo e seus componentes.

A pasta Locale armazena arquivos de string para internacionalização.

A pasta Model contém as tabelas, entidades e comportamentos do seu aplicativo.

Visualizar - as classes de apresentação são colocadas aqui: células, auxiliares e arquivos de modelo.

Modelo - os arquivos de apresentação são colocados aqui: elementos, páginas de erro, layouts e arquivos de modelo de visualização.

Requisitos #

1-HTTP Server. For example: Apache. Having mod_rewrite is preferred, but by no means required.
2-PHP 5.5.9 or greater (including PHP 7)
3-mbstring PHP extens ion
4-intl PHP extension

Eu costumo fazer uma instalação do apache e do mysql em um linuxbox. Posso usar windows também, porém não recomendo ;) Então, costumo fazer uma nova entrada no arquivo /etc/hosts para disponibilizar um sitename para cakephp.

127.0.0.1   localhost caketest.local

próximo passo para copiar todos os arquivos cakephp em um subdiretório dentro de /home/myusername/public_html/caketest

app
cake
index.php
plugins
README
vendors
.htaccess

então eu configurei o site para apache (não é necessário),

<VirtualHost *:80> DocumentRoot “/home/myusername/public_html/caketest” ServerName caketest.local
# This should be omitted in the production environment SetEnv APPLICATION_ENV development

<Directory "/home/myusername/public_html/caketest">
Options Indexes MultiViews FollowSymLinks
AllowOverride All
Order allow,deny
Allow from all
</Directory>

reinicie o apache. você também precisa editar os arquivos .htaccess e colocar uma diretiva RewriteBase com o caminho hte para o diretório real, por exemplo.

 RewriteBase /~myusername/caketest

crie um banco de dados, defina a conexão db nos arquivos de configuração do bolo e pronto. você pode apontar seu navegador para http://caketest.local se você não quiser um url do site de teste, você pode pular hosts e criação de vhost apache, mas o url a ser usado deve ser http:/localhost/~myusername/caketest

outra coisa importante é habilitar o módulo userdir no apache, e também verificar se o uso de php está habilitado em userdirs também.
