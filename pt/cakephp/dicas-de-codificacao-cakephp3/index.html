Criando um novo controlador #

namespace App\Controller;

class PostsController extends AppController {

    public function initialize(){
        parent::initialize();
        // code that you want to run before every action
    }
    public function view($id) {
       //Your code here
    }
}

Recuperando dados de postagem equivalentes a $_POST #

Você pode recuperar dados de postagem como Array.

$post_data= $this->request->data;

Você pode recuperar dados de postagem para uma chave específica.

$this->request->data['field'];

Recuperar valor de chave específico

$this->request->data('key_name');

Recuperar valor de chave específico da matriz aninhada

$this->request->data('data.subfield');

a diferença entre a notação de array e o método data() é que data() é seguro contra erros e retorna null se a chave não existir no array

então ao invés de fazer

if(isset($this->request->data['field']) && $this->request->data['field']) { ...}

você pode fazer

if($this->request->data('field')) { ...}

para CakePHP 3.4.x +

obter todos os dados:

$this->request->getData();

obter chave específica:

$this->request->getData('key');

para definir os dados disponíveis para a função getData, você deve fazer algo como:

$this->request = $this->request->withData('some_key_on_the_fly', 'value');
$some_key_on_the_fly = $this->request->getData('some_key_on_the_fly');

útil para atualizar os modelos no controlador com dados estáticos

Adicione o método beforeFilter() no Controller #

O método beforeFilter() é executado antes de qualquer outro método ser executado no controlador.

Primeiro use o namespace Event antes de definir a classe em seu arquivo de controlador.

use Cake\Event\Event;

Em seu controlador, adicione o método beforeFilter() conforme mostrado abaixo.

public function beforeFilter(Event $event) {
    parent::beforeFilter($event);
}

Ou você pode usar o método initialize().

public function initialize(){
    parent::initialize();
}

Passando variáveis ​​para visualização #

Passe cada variável para visualizar de cada vez

$this->set('color', 'pink');
$this->set('color', $color);

Passe várias variáveis ​​para visualizar juntas via função compact()

$color1 = 'pink';
$color2 = 'red';
$this->set(compact('color1', 'color2'));

Passando Variável para Ação do URL com redirecionamento #

Passando Variável no URL como um parâmetro do método

return $this->redirect([
    'controller' => 'users',
    'action' => 'profile',
    $id
]);

O URL deve ser parecido com este http://your_app_url/users/profile/{id}

no arquivo UsersController.php no método profile()

class UsersController extends Controller {
    public function profile($id=null) {
        $userData=$this->Users->get($id);
    }
}

Passando a variável no URL como uma Query String

return $this->redirect([
    'controller' => 'users',
    'action' => 'profile',
    '?'=>['id'=>$id]
]);

O URL deve ficar assim http://your_app_url/users/profile/?id={id}

no arquivo UsersController.php no método profile()

class UsersController extends Controller {
    public function profile() {
        $userData=$this->Users->get($this->request->query('id'));
    }
}

Redirecionando para outra página do Controller #

Redirecionar para dentro do aplicativo (outra ação do controlador específico).

return $this->redirect([
    'controller' => 'myController',
    'action' => 'myAction'
]);

Redirecionar para a página de referência

return $this->redirect($this->referer());

Redirecionar para fora do aplicativo ou URL específico

return $this->redirect("http://stackoverflow.com/users/1793428/haresh-vidja");

Definir ou alterar o layout do aplicativo #

Defina o layout padrão para todo o aplicativo. ou seja, arquivo de layout criado em /src/Template/Layout/admin.ctp

class AppsController extends Controller {

    public function beforeFilter(Event $event) {
        parent::beforeFilter($event);
        $this->viewBuilder()->layout('admin'); // For Version >= 3.1 or
        $this->layout = 'admin'; // for version < 3.1
        
        // your other code should be here
    }
}

Defina o layout padrão para ação específica no aplicativo. ou seja, o aplicativo tem layout diferente na página de login em /src/Template/Layout/login.ctp

class UsersController extends Controller {

    public function login() {

        $this->viewBuilder()->layout('login'); // For Version >= 3.1 or
        $this->layout = 'login'; // for version < 3.1
        
        //your other code should be here
    }
}

Altere o layout para o controlador específico. ou seja, você precisa de layout diferente para todos os métodos de controlador específico

class UsersController estende o controlador {

    public function beforeFilter(Event $event) {
        parent::beforeFilter($event);

        $this->viewBuilder()->layout('user_layout'); // For Version >= 3.1 or
        $this->layout = 'user_layout'; // for version < 3.1
        
        //your other code should be here
    }
}

Definir layout de solicitação do Ajax #

Geralmente no pedido AJAX não há necessidade de carregar CSS, JS. Também omitindo outro código HTML.

Crie o arquivo ajax.ctp em /src/Template/Layout, e o código deve ser

<?php 
    $this->fetch('content');

Defina o layout baseado em AJAX para todo o aplicativo, em AppsController.php

class AppsController estende o Controlador {

public function beforeFilter(Event $event) {
    parent::beforeFilter($event);
    if($this->request->isAjax())
    {
        $this->viewBuilder()->layout('ajax'); // For Version >= 3.1 or
        $this->layout = 'ajax'; // for version < 3.1

    }
    else
    {
        $this->viewBuilder()->layout('admin'); // For Version >= 3.1 or
        $this->layout = 'admin'; // for version < 3.1
    }
    
    // your other code should be here
}

}

Carregar Componentes no CakePHP #

Podemos carregar componentes de duas maneiras.

  1. Ao inicializar ou substituir a propriedade $components no Controlador
  2. Usando o método loadComponent() no método initialize() do Controller.

Caminho-1 Deve ser substituir o componente de carregamento por AppsController.php carregar um ou mais componentes

class UsersController extends AppController {
    public $components = ['RequestHandler','Auth','Flash']; 
}

Caminho 2 Use esta forma quando você precisar carregar o componente dinamicamente para um controlador específico. Carregar um componente

class UsersController extends AppController {
    public function initialize() {
        parent::initialize();
        $this->loadComponent("RequestHandler"); // load specific component
        $this->loadComponent(["RequestHandler","Auth","Flash"]); // load specific component
    }
}

O que é o método initilaize()? #

initialize() é introduzido no CakePHP versão > 3.0

Como uma estrutura de código, parece igual ao método beforeFilter(). mas há muitas diferenças entre beforeFilter() e initialize().

  1. initialize() é sempre chamado depois que o construtor é chamado. mas beforeFilter() não está chamando em caso de método de ação não encontrado em um controlador específico.
  2. O método initialize() é geralmente usado para inicializar algo como adicionar novos componentes e auxiliares. mas em beforeFilter() é geralmente usado para executar alguma parte da lógica global.

Carrega outro modelo no controlador #

Por padrão, o CakePHP carrega o modelo relacionado no controlador. Para carregar outro modelo no controlador, use o método loadModel():

$this->loadModel('Articles');

ou carregue na hora

$table = TableRegistry::get('Articles');
$table->find();

Criando classe de tabela (modelo) #

Como criar uma classe de modelo de usuário

namespace App\Model\Table;
use Cake\ORM\Table;

class UsersTable extends Table {
    public function initialize(array $config) {
        $this->table('users'); //define table name
        $this->displayField('username'); // unique or other special field of users table
        $this->primaryKey('id'); // primary key of users table
        $this->tablePrefix('prefix_'); // if prefix set tablename should be prefix_users

        // your other code here
    }

    // your other methods here
}

Associações de modelo no CakePHP #

Existem 4 tipos de associações (relacionamentos) que podemos definir no CakePHP

class PostsTable extends Table {
    public function initialize(array $config) {

        // table initialization code should be here

        $this->belongsTo('Authors', [
            'className' => 'Authors',
            'foreignKey' => 'author_id',
            'joinType' => 'INNER',
        ]);
        $this->hasMany('Tags');
        $this->hasOne('Categories');
        $this->hasAndBelongsToMany('Topics');
    }
}

No exemplo acima, você pode ver 4 tipos de relacionamentos

Post pertence a Author (One to One), significa que na tabela posts tem uma chave estrangeira author_id que está associada ao id da tabela authors.

Post tem muitas Tags (One to Many), significa que na tabela tags tem uma chave estrangeira post_id que está associada ao id da tabela posts.

Post tem uma Tags (Many to One or One to One), significa que na tabela posts tem uma chave estrangeira category_id que está associada ao id da tabela categories.

Post tem e pertence a Tópicos (Many to Many), esta é uma relação de muitos para muitos entre a tabela posts e topics. para manter muitos para muitos relacionamento deve ser necessário criar uma terceira tabela, tabela, o nome da tabela deve ser posts_categories. Campos desta tabela conforme mencionado abaixo

  1. id (chave primária da tabela)
  2. post_id (chave estrangeira da tabela posts)
  3. topic_id (chave estrangeira da tabela tópicos)
