Atribuição de variável condicional #

O operador ?= é uma extensão que se comporta como =, exceto que a atribuição somente ocorre se a variável ainda não estiver definida.

x = hello
x ?= world
# $(x) will yield "hello"

Variáveis ​​Automáticas #

Dentro do contexto de uma regra individual, Make define automaticamente uma série de variáveis ​​especiais. Essas variáveis ​​podem ter um valor diferente para cada regra em um makefile e são projetadas para simplificar a escrita de regras. Essas variáveis ​​só podem ser usadas na parte da receita de uma regra.

Variável Descrição
$@ Nome do arquivo de destino da regra
$% O nome do membro de destino, se o destino da regra for um arquivo
$< Nome do arquivo do primeiro pré-requisito
$^ Lista de todos os pré-requisitos
$? Lista de todos os pré-requisitos mais recentes que o destino
$* A “haste” de uma regra implícita ou padrão

O exemplo a seguir usa variáveis ​​automáticas para gerar uma regra genérica. Isso instrui o make a construir um arquivo .o a partir de um arquivo .c com o mesmo nome. Como não sabemos o nome específico dos arquivos afetados, usamos $@ como espaço reservado para o nome do arquivo de saída e $^ como espaço reservado para a lista de pré-requisitos (neste caso, a lista de arquivos de entrada ).

%.o: %.c
    cc -Wall $^ -c $@

Referenciando uma variável #

Para usar o valor armazenado em uma variável, use o cifrão seguido do nome da variável entre parênteses ou chaves.

x = hello
y = $(x)
# y now contains the value "hello"
y = ${x}
# parentheses and curly braces are treated exactly the same

Se o nome de uma variável tiver apenas um caractere, os parênteses/chaves podem ser omitidos (por exemplo, $x). Essa prática é usada para variáveis ​​automáticas (veja abaixo), mas não é recomendada para variáveis ​​de uso geral.

Variáveis ​​simplesmente expandidas #

Variáveis ​​simplesmente expandidas se comportam como variáveis ​​de linguagens de programação tradicionais. A expressão do lado direito é avaliada e o resultado é armazenado na variável. Se o lado direito contiver uma referência de variável, essa variável será expandida antes que a atribuição ocorra.

x := hello
y := $(x)
# Both $(x) and $(y) will now yield "hello"
x := world
# $(x) will now yield "world", and $(y) will yield "hello"

Uma forma alternativa é usar a atribuição de dois pontos:

x ::= hello

A atribuição de dois-pontos simples e duplos são equivalentes. O padrão POSIX apenas menciona a forma ::=, portanto, implementações com conformidade estrita com os padrões podem não suportar a versão de dois-pontos.

Variáveis ​​expandidas recursivamente #

Ao definir uma variável expandida recursivamente, o conteúdo do lado direito é armazenado como está. Se uma referência de variável estiver presente, a própria referência será armazenada (não o valor da variável). Make espera para expandir as referências de variáveis ​​até que a variável seja realmente usada.

x = hello
y = $(x)
# Both $(x) and $(y) will now yield "hello"
x = world
# Both $(x) and $(y) will now yield "world"

Neste exemplo, a definição de y é recursiva. A referência a $(x) não é expandida até que $(y) seja expandida. Isso significa que sempre que o valor de x mudar, o valor de y também mudará.

As variáveis ​​expandidas recursivamente são uma ferramenta poderosa, mas facilmente incompreendida. Eles podem ser usados ​​para criar construções que se assemelham a templates ou funções, ou até mesmo para gerar automaticamente partes de um makefile. Eles também podem ser a fonte de problemas difíceis de depurar. Tenha cuidado para usar apenas variáveis ​​expandidas recursivamente quando necessário.

Anexando texto a uma variável existente #

O operador += é uma extensão comum que adiciona o conteúdo especificado ao final da variável, separado por um espaço.

x = hello
x += world

As referências de variáveis ​​no lado direito serão expandidas se e somente se a variável original foi definida como uma variável simplesmente expandida.
