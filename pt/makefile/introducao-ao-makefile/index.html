Makefile básico #

Considere escrever um “olá, mundo!” programa em c. Digamos que nosso código fonte esteja em um arquivo chamado source.c, agora para rodar nosso programa precisamos compilá-lo, normalmente no Linux (usando gcc) precisaríamos digitar $> gcc source.c -o output onde output é o nome do executável a ser gerado. Para um programa básico, isso funciona bem, mas à medida que os programas se tornam mais complexos, nosso comando de compilação também pode se tornar mais complexo. É aqui que entra um Makefile, makefiles nos permitem escrever um conjunto bastante complexo de regras sobre como compilar um programa e então simplesmente compilá-lo digitando make na linha de comando. Por exemplo, aqui está um possível exemplo de Makefile para o exemplo hello wold acima.

Makefile básico

Vamos fazer um Makefile básico e salvá-lo em nosso sistema no mesmo diretório que nosso código-fonte chamado Makefile. Observe que este arquivo precisa ser nomeado Makefile, porém o capitólio M é opcional. Dito isto, é relativamente padrão usar um capitol M.

output: source.c
    gcc source.c -o output

Observe que há exatamente uma guia antes do comando gcc na segunda linha (isso é importante em makefiles). Uma vez que este Makefile é escrito toda vez que o usuário digita make (no mesmo diretório que o Makefile) make irá verificar se source.c foi modificado (verifica o timestamp) se ele foi modificado mais recentemente do que output ele será executado a regra de compilação na linha a seguir.

Variáveis ​​em Makefiles

Dependendo do projeto, você pode querer introduzir algumas variáveis ​​em seu arquivo make. Aqui está um exemplo de Makefile com variáveis ​​presentes.

CFLAGS = -g -Wall

output: source.c
    gcc $< $(CFLAGS) -o $@

Agora vamos explorar o que aconteceu aqui. Na primeira linha declaramos uma variável chamada CFLAGS que contém vários sinalizadores comuns que você pode desejar passar para o compilador, observe que você pode armazenar quantos sinalizadores desejar nesta variável. Então temos a mesma linha de antes dizendo ao make para verificar source.c para ver se ele foi alterado mais recentemente do que o output, se sim, ele executa a regra de compilação. Nossa regra de compilação é basicamente a mesma de antes, mas foi encurtada usando variáveis, a variável $< é incorporada ao make (referida como uma variável automática, consulte https://www.gnu.org/software/make /manual/html_node/Automatic-Variables.html) e sempre representa a fonte, portanto, neste caso, source.c. $(CFLAGS) é nossa variável que definimos antes, mas observe que tivemos que colocar a variável entre parênteses com um $ na frente assim $(someVariable). Esta é a sintaxe para dizer ao Make para expandir a variável para o que você digitou antes. Finalmente, temos o símbolo $@, mais uma vez esta é uma variável incorporada ao make, e simplesmente representa o destino da etapa de compilação, portanto, neste caso, representa saída.

Limpar

Make clean é outro conceito útil para aprender sobre make files. Vamos modificar o Makefile de cima

CFLAGS = -g -Wall
TARGETS = output

output: source.c
    gcc $< $(CFLAGS) -o $@

clean:
    rm $(TARGETS)

Como você pode ver, simplesmente adicionamos mais uma regra ao nosso Makefile e uma variável adicional que contém todos os nossos alvos. Esta é uma regra um tanto comum em makefiles, pois permite que você remova rapidamente todos os binários que você produziu simplesmente digitando $> make clean. Ao digitar make clean, você diz ao programa make para executar a regra de limpeza e, em seguida, make executará o comando rm para excluir todos os seus destinos.

Espero que esta breve visão geral do uso do make ajude você a acelerar seu fluxo de trabalho, Makefiles podem se tornar muito complexos, mas com essas ideias você poderá começar a usar o make e ter uma melhor compreensão do que está acontecendo em outros programadores * Makefiles*. Para obter mais informações sobre o uso de make, um excelente recurso é https://www.gnu.org/software/make/manual/.

Definindo regras #

Começo rápido #

Uma regra descreve quando e como determinados arquivos (os destinos da regra) são criados. Ele também pode servir para atualizar um arquivo de destino se algum dos arquivos necessários para sua criação (os pré-requisitos do destino) for mais recente que o destino.

As regras seguem a sintaxe abaixo: (Observe que os comandos que seguem uma regra são recuados por uma guia)

targets: prerequisites
        <commands>

onde destinos e pré-requisitos são nomes de arquivos ou nomes reservados especiais e comandos (se presentes) são executados por um shell para construir/reconstruir destinos que estão desatualizados.

Para executar uma regra pode-se simplesmente executar o comando make no terminal do mesmo diretório onde o Makefile reside. Executar make sem especificar o destino, executará a primeira regra definida no Makefile. Por convenção, a primeira regra no Makefile geralmente é chamada de all ou default, geralmente listando todos os destinos de compilação válidos como pré-requisitos.

O make só executa a regra se o destino estiver desatualizado, o que significa que ele não existe ou seu tempo de modificação é mais antigo do que qualquer um de seus pré-requisitos. Se a lista de pré-requisitos estiver vazia, a regra só será executada quando for invocada pela primeira vez para construir os destinos. No entanto, quando a regra não cria um arquivo e o destino é uma variável fictícia, a regra sempre será executada.

Regras de padrão #

As regras de padrão são usadas para especificar vários destinos e construir nomes de pré-requisitos a partir de nomes de destino. Eles são mais gerais e mais poderosos em comparação com as regras comuns, pois cada alvo pode ter seus próprios pré-requisitos. Nas regras de padrão, um relacionamento entre um destino e um pré-requisito é construído com base em prefixos, incluindo nomes de caminho e sufixos, ou ambos.

Imagine que queremos construir os alvos foo.o e bar.o, compilando scripts C, foo.c e bar.c, respectivamente. Isso pode ser feito usando as regras comuns abaixo:

foo.o: foo.c
    cc -c $< -o $@

bar.o: bar.c
    cc -c $< -o $@

onde variável automática $< é o nome do primeiro pré-requisito e $@ o nome do destino (Uma lista completa de variáveis ​​automáticas pode ser encontrada [aqui](https://www.gnu.org/ software/make/manual/html_node/Automatic-Variables.html#Automatic-Variables)).

No entanto, como os destinos compartilham o mesmo sufixo, as duas regras acima agora podem ser substituídas pela seguinte regra de padrão:

%.o: %.c
    cc -c $< -o $@

Regras Implícitas #

Regras implícitas dizem ao make como usar métodos habituais para construir certos tipos de arquivos de destino, que são usados ​​com muita frequência. make usa o nome do arquivo de destino para determinar qual regra implícita invocar.

O exemplo de regra padrão que vimos na seção anterior, na verdade não precisa ser declarado em um Makefile já que make tem uma regra implícita para compilação C. Assim, na regra a seguir, os pré-requisitos foo.o e bar.o serão construídos usando a regra implícita para compilação em C, antes de construir o foo.

foo : foo.o bar.o
    cc -o foo foo.o bar.o $(CFLAGS) $(LDFLAGS)

Um catálogo de regras implícitas e as variáveis ​​usadas por elas podem ser encontradas aqui.

regra genérica para gzipar um arquivo #

se um diretório contiver 2 arquivos:

$ ls
makefile
example.txt

e makefile contém o seguinte texto

%.gz: %
    gzip $<

então você pode obter example.txt.gz digitando no shell

$ make -f makefile example.txt.gz

o makefile consiste em apenas uma regra que instrui make como criar um arquivo cujo nome termine com .gz se houver um arquivo com o mesmo nome, mas o sufixo .gz.

makefile Olá Mundo #

C:\makefile:

helloWorld :
[TAB]echo hello world

executar resultados:

C:\>make
echo hello world
hello world

Observação: [TAB] deve ser substituído por uma guia real, o stackoverflow substitui as guias por espaços e os espaços não são usados ​​da mesma forma que as guias em um makefile.
