Vuex é um padrão de gerenciamento de estado + biblioteca para aplicativos Vue.js. Ele serve como um armazenamento centralizado para todos os componentes em um aplicativo, com regras que garantem que o estado só possa ser alterado de maneira previsível. Ele também se integra com a extensão de ferramentas de desenvolvimento oficial do Vue para fornecer recursos avançados, como depuração de viagem no tempo de configuração zero e exportação/importação de instantâneo de estado.

O que é Vuex? #

Vuex é um plugin oficial para Vue.js que oferece um armazenamento de dados centralizado para uso em seu aplicativo. Ele é fortemente influenciado pela arquitetura do aplicativo Flux, que apresenta um fluxo de dados unidirecional que leva a um design e raciocínio mais simples do aplicativo.

Dentro de um aplicativo Vuex, o armazenamento de dados contém todos os estados de aplicativos compartilhados. Esse estado é alterado por mutações que são executadas em resposta a uma ação que invoca um evento de mutação por meio do dispatcher.

Um exemplo do fluxo de dados em um aplicativo Vuex é descrito no diagrama abaixo. Diagrama usado sob a licença MIT, originalmente do Repositório Oficial Vuex GitHub.

Os componentes individuais do aplicativo Vue.js podem acessar o objeto de armazenamento para recuperar dados por meio de getters, que são funções puras que retornam uma cópia somente leitura dos dados desejados.

Os componentes podem ter ações, que são funções que realizam alterações na própria cópia dos dados do componente. Depois, use o dispatcher para enviar um evento de mutação. Este evento é então tratado pelo armazenamento de dados que atualiza o estado conforme necessário.

As alterações são refletidas automaticamente em todo o aplicativo, pois todos os componentes são vinculados de forma reativa à loja por meio de seus getters.

Um exemplo ilustrando o uso do vuex em um projeto vue.

const state = {
    lastClickTime: null
}

const mutations = {
    updateLastClickTime: (state, payload) => {
      state.lastClickTime = payload
  }
}

const getters = {
  getLastClickTime: state => {
    return new Date(state.lastClickTime)
  }
}

const actions = {
    syncUpdateTime: ({ commit }, payload) => {
    commit("updateLastClickTime", payload)
  },
  asyncUpdateTime: ({ commit }, payload) => {
    setTimeout(() => {
      commit("updateLastClickTime", payload)
    }, Math.random() * 5000)
  }
}

const store = new Vuex.Store({
  state,
  getters,
  mutations,
  actions
})

const { mapActions, mapGetters } = Vuex;

// Vue 
const vm = new Vue({
    el: '#container',
  store,
  computed: {
      ...mapGetters([
        'getLastClickTime'
    ])
  },
  methods: {
      ...mapActions([
        'syncUpdateTime',
      'asyncUpdateTime'
    ]),
    updateTimeSyncTest () {
        this.syncUpdateTime(Date.now())
    },
    updateTimeAsyncTest () {
        this.asyncUpdateTime(Date.now())
    }
  }
})

E o modelo HTML para o mesmo:

<div id="container">
  <p>{{ getLastClickTime || "No time selected yet" }}</p>
  <button @click="updateTimeSyncTest">Sync Action test</button>
  <button @click="updateTimeAsyncTest">Async Action test</button>
</div>

  1. Aqui o state contém a propriedade lastClickTime inicializada como nula. Essa configuração de valores padrão é importante para manter as propriedades reativas. Propriedades não mencionadas no estado estarão disponíveis, mas as alterações feitas posteriormente não serão acessíveis usando getters.

  2. O getter usado fornece uma propriedade computada que será atualizada toda vez que uma mutação atualizar o valor da propriedade state.

  3. Somente mutações podem alterar o estado e suas propriedades, ou seja, ele o faz apenas de forma síncrona.

  4. Uma ação pode ser usada no caso de atualizações assíncronas, onde a chamada da API (aqui simulada pelo setTimeout de tempo aleatório) pode ser feita na ação e, após obter a resposta, uma mutação pode ser confirmada, para fazer a alteração para o Estado.

Por que usar Vuex? #

Ao criar aplicativos grandes, como aplicativos de página única (SPAs), que normalmente consistem em muitos componentes reutilizáveis, eles podem se tornar rapidamente difíceis de criar e manter. O compartilhamento de dados e estado entre esses componentes também pode quebrar rapidamente e tornar-se difícil de depurar e manter.

Ao usar um armazenamento de dados de aplicativo centralizado, todo o estado do aplicativo pode ser representado em um só lugar, tornando o aplicativo mais organizado. Por meio do uso de um fluxo de dados unidirecional, de mutações e do escopo de acesso aos dados do componente apenas aos dados necessários, torna-se muito mais simples raciocinar sobre a função do componente e como ele deve afetar o estado do aplicativo.

Os componentes VueJS são entidades separadas e não podem compartilhar dados entre si facilmente. Para compartilhar dados sem vuex, precisamos emitir evento com dados e então ouvir e capturar esse evento com on.

componente 1

this.$emit('eventWithDataObject', dataObject)

componente 2

this.$on('eventWithDataObject', function (dataObject) {
    console.log(dataObject)
})

Com o vuex instalado podemos simplesmente acessar seus dados de qualquer componente sem a necessidade de escutar eventos.

this.$store.state.myData

Também podemos alterar dados de forma síncrona com mutadores, usar ações assíncronas e obter dados com funções getter.

As funções getter podem funcionar como funções computadas globais. Podemos acessá-los a partir de componentes:

this.$store.getters.myGetter

As ações são métodos globais. Podemos despachá-los de componentes:

this.$store.dispatch('myAction', myDataObject)

E as mutações são a única maneira de alterar os dados no vuex. Podemos confirmar as alterações:

this.$store.commit('myMutation', myDataObject)

O código Vuex ficaria assim

state: {
  myData: {
    key: 'val'
  }
},
getters: {
  myGetter: state => {
     return state.myData.key.length
  }
},
actions: {
  myAction ({ commit }, myDataObject) {
    setTimeout(() => {
      commit('myMutation', myDataObject)
    }, 2000)
  }
},
mutations: {
  myMutation (state, myDataObject) {
    state.myData = myDataObject
  }
}

Como instalar o Vuex? #

Na maioria das vezes, você usará o Vuex em aplicativos baseados em componentes maiores, nos quais provavelmente estará usando um empacotador de módulos, como Webpack ou Browserify, em conjunto com o Vueify, se estiver usando arquivos únicos.

Nesse caso, a maneira mais fácil de obter o Vuex é do NPM. Execute o comando abaixo para instalar o Vuex e salve-o nas dependências do seu aplicativo.

 npm install --save vuex

Certifique-se de carregar o link Vuex com sua configuração Vue colocando a seguinte linha após sua instrução require('vue').

Vue.use(require('vuex'))

Vuex também está disponível em CDN; você pode pegar a versão mais recente do cdnjs aqui.

Notificações descartáveis ​​automaticamente #

Este exemplo registrará um módulo vuex dinamicamente para armazenar notificações personalizadas que podem ser dispensadas automaticamente

notificações.js

resolva o armazenamento vuex e defina algumas constantes

//Vuex store previously configured on other side
import _store from 'path/to/store';

//Notification default duration in milliseconds
const defaultDuration = 8000;

//Valid mutation names
const NOTIFICATION_ADDED = 'NOTIFICATION_ADDED';
const NOTIFICATION_DISMISSED = 'NOTIFICATION_DISMISSED';

defina nosso estado inicial do módulo

const state = {
  Notifications: []
}

definir nossos getters de módulo

const getters = {
  //All notifications, we are returning only the raw notification objects
  Notifications: state => state.Notifications.map(n => n.Raw)
}

defina nossas ações do módulo

const actions = {
  //On actions we receive a context object which exposes the 
  //same set of methods/properties on the store instance
  //{commit} is a shorthand for context.commit, this is an 
  //ES2015 feature called argument destructuring
  Add({ commit }, notification) {
    //Get notification duration or use default duration
    let duration = notification.duration || defaultDuration

    //Create a timeout to dismiss notification
    var timeOut = setTimeout(function () {
      //On timeout mutate state to dismiss notification
      commit(NOTIFICATION_DISMISSED, notification);
    }, duration);

    //Mutate state to add new notification, we create a new object 
    //for save original raw notification object and timeout reference
    commit(NOTIFICATION_ADDED, {
      Raw: notification,
      TimeOut: timeOut
    })
  },
  //Here we are using context object directly
  Dismiss(context, notification) {
    //Just pass payload
    context.commit(NOTIFICATION_DISMISSED, notification);
  }
}

definir nossas mutações de módulo

const mutations = {
  //On mutations we receive current state and a payload
  [NOTIFICATION_ADDED](state, notification) {
    state.Notifications.push(notification);
  },
  //remember, current state and payload
  [NOTIFICATION_DISMISSED](state, rawNotification) {
    var i = state.Notifications.map(n => n.Raw).indexOf(rawNotification);
    if (i == -1) {
      return;
    }

    clearTimeout(state.Notifications[i].TimeOut);
    state.Notifications.splice(i, 1);
  }
}

Registre nosso módulo com estado definido, getters, ações e mutação

_store.registerModule('notifications', {
  state,
  getters,
  actions,
  mutations
});

Uso

componentA.vue

Este componente exibe todas as notificações como alertas de bootstrap no canto superior direito da tela, também permite descartar manualmente cada notificação.

<template>
<transition-group name="notification-list" tag="div" class="top-right">
  <div v-for="alert in alerts" v-bind:key="alert" class="notification alert alert-dismissible" v-bind:class="'alert-'+alert.type">
    <button v-on:click="dismiss(alert)" type="button" class="close" aria-label="Close"><span aria-hidden="true">&times;</span></button>
    <div>
      <div>
        <strong>{{alert.title}}</strong>
      </div>
      <div>
        {{alert.text}}
      </div>
    </div>
  </div>
</transition-group>
</template>

<script>
export default {
  name: 'arc-notifications',
  computed: {
    alerts() {
      //Get all notifications from store
      return this.$store.getters.Notifications;
    }
  },
  methods: {
    //Manually dismiss a notification
    dismiss(alert) {
      this.$store.dispatch('Dismiss', alert);
    }
  }
}
</script>
