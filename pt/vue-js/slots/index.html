Importante! Slots após renderização não garantem ordem de posições para slots. Slot, que foi o primeiro, pode ter uma posição diferente após a renderização.

Usando slots nomeados #

Os slots nomeados funcionam de maneira semelhante aos slots únicos, mas permitem distribuir conteúdo para diferentes regiões dentro do modelo de componente filho.

Pegue o componente page do exemplo anterior, mas modifique seu template para que fique da seguinte forma:

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <aside>
            <slot name="sidebar"></slot>
        </aside>
        <main>
            <slot name="content"></slot>
        </main>
    </body>
</html>

Ao usar o componente page, agora podemos determinar onde o conteúdo é colocado através do atributo slot:

<page>
    <p slot="sidebar">This is sidebar content.</p>
    <article slot="content"></article>
</page>

A página resultante será:

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <aside>
            <p>This is sidebar content.</p>
        </aside>
        <main>
            <article></article>
        </main>
    </body>
</html>

Se um slot for definido sem um atributo name, qualquer conteúdo que seja colocado dentro de tags de componentes que não especifiquem um atributo slot será colocado nesse slot.

Veja o exemplo de multiinserção nos documentos oficiais do Vue.js.

Usando slots únicos #

Slots únicos são usados ​​quando um componente filho define apenas um slot dentro de seu template. O componente page acima usa um único slot para distribuir conteúdo.

Um exemplo do modelo do componente page usando um único slot está abaixo:

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <slot>
            This will only be displayed if there is no content
            to be distributed.
        </slot>
    </body>
</html>

Para ilustrar como o slot funciona, podemos configurar uma página da seguinte forma.

<page>
    <p>This content will be displayed within the page component</p>
</page>

O resultado final será:

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <p>This content will be displayed within the page component</p>
    </body>
</html>

Se não colocássemos nada entre as tags page e em vez disso tivéssemos <page></page>, produziríamos o seguinte resultado, pois há conteúdo padrão entre as tags slot no modelo do componente page .

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        This will only be displayed if there is no content
        to be distributed.
    </body>
</html>

O que são slots? #

Os slots oferecem uma maneira conveniente de distribuir conteúdo de um componente pai para um componente filho. Esse conteúdo pode ser qualquer coisa, desde texto, HTML ou até mesmo outros componentes.

Às vezes, pode ser útil pensar em slots como um meio de injetar conteúdo diretamente no modelo de um componente filho.

Os slots são especialmente úteis quando a composição do componente abaixo do componente pai nem sempre é a mesma.

Veja o exemplo a seguir onde temos um componente page. O conteúdo da página pode mudar com base na exibição dessa página, por exemplo. um artigo, postagem de blog ou formulário.

Artigo

<page>
    <article></article>
    <comments></comments>
</page>

Postagem do blog

<page>
    <blog-post></blog-post>
    <comments></comments>
</page>

Forma

<page>
    <form></form>
</page>

Observe como o conteúdo do componente page pode mudar. Se não usássemos slots, isso seria mais difícil, pois a parte interna do modelo seria corrigida.

Lembre-se: “Tudo no modelo pai é compilado no escopo pai; tudo no modelo filho é compilado no escopo filho.”

Usando Slots no Vue JSX com ‘babel-plugin-transform-vue-jsx’ #

Se você estiver usando o VueJS2 e gosta de usar o JSX junto com ele. Neste caso, para usar o slot, a solução com exemplo está abaixo. Temos que usar this.$slots.default É quase como this.props.children no React JS.

Component.js :

export default {
    render(h) {  //eslint-disable-line
        return (
            <li>
                 { this.$slots.default }
            </li>
        );
    }
};

ParentComponent.js

import Component from './Component';

export default {
    render(h) {  //eslint-disable-line
        return (
            <ul>
                 <Component>
                     Hello World
                 </Component>
            </ul>
        );
    }
};
