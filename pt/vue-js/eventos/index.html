Sintaxe de eventos #

Para enviar um evento: vm.$emit('new-message');

Para capturar um evento: vm.$on('new-message');

Para enviar um evento para todos os componentes down: vm.$broadcast('new-message');

Para enviar um evento para todos os componentes up: vm.$dispatch('new-message');

Nota: $broadcast e $dispatch estão obsoletos no Vue2. (consulte recursos do Vue2)

Quando devo usar eventos? #

A figura a seguir ilustra como a comunicação de componentes deve funcionar. A imagem vem de The Progressive Framework slides de [Evan You](https://twitter.com /youyuxi) (Desenvolvedor do VueJS).

Aqui está um exemplo de como funciona:

DEMO

HTML

<script type="x-template" id="message-box">
    <input type="text" v-model="msg" @keyup="$emit('new-message', msg)" />
</script>
 
<message-box :msg="message" @new-message="updateMessage"></message-box>
<div>You typed: {{message}}</div>

JS

var messageBox = {
  template: '#message-box',
  props: ['msg']
};
 
new Vue({
  el: 'body',
  data: {
    message: ''
  },
  methods: {
    updateMessage: function(msg) {
      this.message = msg;
    }
  },
  components: {
    'message-box': messageBox
  }
});

O exemplo acima pode ser melhorado ! #

O exemplo acima mostra como funciona a comunicação do componente. Mas no caso de um componente de entrada personalizado, para sincronizar a variável pai com o valor digitado, devemos usar o v-model.

DEMO Vue1

DEMO Vue2

No Vue1, você deve usar .sync no prop enviado para o componente <message-box>. Isso diz ao VueJS para sincronizar o valor no componente filho com o do pai.

Lembre-se: cada instância de componente tem seu próprio escopo isolado.

HTML Vue1

<script type="x-template" id="message-box">
  <input v-model="value" />
</script>

<div id="app">
  <message-box :value.sync="message"></message-box>
  <div>You typed: {{message}}</div>
</div>

No Vue2, existe um evento especial 'input' que você pode $emit. Usar este evento permite que você coloque um v-model diretamente no componente <message-box>. O exemplo ficará da seguinte forma:

HTML Vue2

<script type="x-template" id="message-box">
  <input :value="value" @input="$emit('input', $event.target.value)" />
</script>

<div id="app">
  <message-box v-model="message"></message-box>
  <div>You typed: {{message}}</div>
</div>

JS Vue 1 e 2

var messageBox = {
  template: '#message-box',
  props: ['value']
};

new Vue({
  el: '#app',
  data: {
    message: ''
  },
  components: {
    'message-box': messageBox
  }
});

Observe a rapidez com que a entrada é atualizada.

Como lidar com a depreciação de $dispatch e $broadcast? (padrão de evento de ônibus) #

Você deve ter percebido que $emit está no escopo do componente que está emitindo o evento. Isso é um problema quando você deseja se comunicar entre componentes distantes um do outro na árvore de componentes.

Nota: No Vue1 você pode usar $dispatch ou $broadcast, mas não no Vue2. A razão é que não escala bem. Existe um padrão bus popular para gerenciar isso:

DEMO

HTML

<script type="x-template" id="sender">
  <button @click="bus.$emit('new-event')">Click me to send an event !</button>
</script>
 
<script type="x-template" id="receiver">
  <div>I received {{numberOfEvents}} event{{numberOfEvents == 1 ? '' : 's'}}</div>
</script>
 
<sender></sender>
<receiver></receiver>

JS

var bus = new Vue();
 
var senderComponent = {
  template: '#sender',
  data() {
    return {
      bus: bus
    }
  }
};
 
var receiverComponent = {
  template: '#receiver',
  data() {
    return {
      numberOfEvents: 0
    }
  },
  ready() {
    var self = this;
 
    bus.$on('new-event', function() {
      ++self.numberOfEvents;
    });
  }
};
 
new Vue({
  el: 'body',
  components: {
    'sender': senderComponent,
    'receiver': receiverComponent
  }
});

Você só precisa entender que qualquer instância Vue() pode $emit e capturar ($on) um evento. Nós apenas declaramos uma instância global Vue chamada bus e então qualquer componente com esta variável pode emitir e capturar eventos dela. Apenas certifique-se de que o componente tenha acesso à variável bus.
