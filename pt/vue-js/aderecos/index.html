camelCase <=> kebab-case #

Ao definir os nomes dos seus props, lembre-se sempre que os nomes dos atributos HTML não diferenciam maiúsculas de minúsculas. Isso significa que se você definir um prop em camel case na sua definição de componente…

Vue.component('child', {
    props: ['myProp'],
    ...
});

…você deve chamá-lo em seu componente HTML como my-prop.

Passando dados de pai para filho com adereços #

No Vue.js, cada instância de componente tem seu próprio escopo isolado, o que significa que se um componente pai tem um componente filho - o componente filho tem seu próprio escopo isolado e o componente pai tem seu próprio escopo isolado.

Para qualquer aplicativo de tamanho médio a grande, seguir as convenções de práticas recomendadas evita muitas dores de cabeça durante a fase de desenvolvimento e depois durante a manutenção. Uma dessas coisas a seguir é que evite fazer referência/mutação de dados pai diretamente do componente filho. Então, como referenciamos os dados pai de dentro de um componente filho?

Quaisquer dados pai necessários em um componente filho devem ser passados ​​para o filho como props do pai.

Caso de Uso: Suponha que tenhamos um banco de dados User com duas tabelas users e addresses com os seguintes campos: Tabela usuários

nome telefone e-mail
John Mclane (1) 234 5678 9012 john@dirhard.com
James Bond (44) 777 0007 0077 bond@mi6.com

Tabela endereços

bloco rua cidade
Torres Nakatomi Broadway Nova York
Casa Mi6 Buckingham Road Londres

e queremos ter três componentes para exibir as informações do usuário correspondentes em qualquer lugar do nosso aplicativo

user-component.js

export default{
    template:`<div class="user-component">
                <label for="name" class="form-control">Name: </label>  
                <input class="form-control input-sm" name="name" v-model="name">
                <contact-details :phone="phone" :email="email"></contact-details>
              </div>`,
    data(){
        return{
            name:'',
            phone:'',
            email:''
        }
    },
}

contact-details.js

import Address from './address';
export default{
    template:`<div class="contact-details-component>
                <h4>Contact Details:</h4>
                <label for="phone" class="form-control">Phone: </label>  
                <input class="form-control input-sm" name="phone" v-model="phone">
                <label for="email" class="form-control">Email: </label>  
                <input class="form-control input-sm" name="email" v-model="email"> 
        
                <h4>Address:</h4>
                <address :address-type="addressType"></address>
                //see camelCase vs kebab-case explanation below
            </div>`,
    props:['phone', 'email'],
    data:(){
        return:{
            addressType:'Office'
        }
    },
    components:{Address}  
}

endereço.js

export default{
    template:`<div class="address-component">
                <h6>{{addressType}}</h6>
                <label for="block" class="form-control">Block: </label>  
                <input class="form-control input-sm" name="block" v-model="block">
                <label for="street" class="form-control">Street: </label>  
                <input class="form-control input-sm" name="street" v-model="street">
                <label for="city" class="form-control">City: </label>  
                <input class="form-control input-sm" name="city" v-model="city">
             </div>`,
    props:{
        addressType:{
            required:true,
            type:String,
            default:'Office'
        },
    data(){
        return{
            block:'',
            street:'',
            city:''
        }
    }
}

main.js

import Vue from 'vue';  
  
Vue.component('user-component', require'./user-component');  
Vue.component('contact-details', require'./contact-details');  

new Vue({
    el:'body'  
});

index.html

...  
<body>
    <user-component></user-component>
        ...
</body>

Estamos exibindo os dados phone e email, que são propriedades do user-component em contact-details que não possui dados de telefone ou e-mail.

Passando dados como adereços

Portanto, dentro do user-component.js na propriedade template, onde incluímos o componente <contact-details>, estamos passando o phone e o ** email** dados de <user-component>(componente pai) para <contact-details>(componente filho) vinculando-os dinamicamente ao props - :phone="phone" e :email="email que é o mesmo que v-bind:phone="phone" e v-bind:email="email"

Adereços - Vinculação dinâmica

Como estamos vinculando dinamicamente as props, qualquer alteração em phone ou email dentro do componente pai, ou seja, <user-component> será imediatamente refletida no componente filho, ou seja, <contact- detalhes>.

Adereços - como literais

No entanto, se tivéssemos passado os valores de phone e email como valores literais de string como phone="(44) 777 0007 0077" email="bond@mi6.com" então não refletiria nenhuma mudança de dados que acontecesse no componente pai.

Encadernação unidirecional

Por padrão, a direção das alterações é de cima para baixo, ou seja, qualquer alteração nas props vinculadas dinamicamente no componente pai será propagada para o componente filho, mas qualquer alteração nos valores da prop em um componente filho não será propagada para o pai.

Por exemplo: se de dentro do <contact-details> alterarmos o email de bond@mi6.com para jamesbond@mi6.com, os dados pai, ou seja, a propriedade de dados do telefone em <user-component> ainda conterá um valor de bond@mi6.com.

No entanto, se alterarmos o valor de email de bond@mi6.com para jamesbond@mi6.co no componente pai (<user-component> em nosso caso de uso), o valor de email no componente filho componente (<contact-details> em nosso caso de uso) mudará para jamesbond@mi6.com automaticamente - a mudança no pai é instantaneamente propagada para o filho.

Encadernação Bidirecional

Se queremos vinculação bidirecional, temos que especificar explicitamente a vinculação bidirecional como :email.sync="email" em vez de :email="email". Agora, se alterarmos o valor de prop no componente filho, a alteração também será refletida no componente pai.

Em um aplicativo de médio a grande porte, alterar o estado pai do estado filho será muito difícil de detectar e acompanhar, especialmente durante a depuração - Tenha cuidado .

Não haverá nenhuma opção .sync disponível no Vue.js 2.0. A vinculação bidirecional para props está sendo preterida no Vue.js 2.0.

Encadernação única

Também é possível definir a vinculação explícita uma vez como :email.once="email, é mais ou menos semelhante a passar um literal, porque quaisquer alterações subsequentes no valor da propriedade pai não serão propagadas para a criança.

EMBARGO Quando Object ou Array é passado como prop, eles são SEMPRE PASSADOS POR REFERÊNCIA, o que significa que independentemente do tipo de vinculação definido explicitamente :email.sync="email" ou :email="email " ou :email.once="email", se email for um objeto ou uma matriz no pai, independentemente do tipo de associação, qualquer alteração no valor prop no componente filho afetará o valor no pai, pois Nós vamos.

Props como Array

No arquivo contact-details.js definimos props:['phone', 'email'] como um array, o que é bom se não queremos um controle refinado com props.

Adereços como Objeto

Se quisermos um controle mais refinado sobre adereços, como

  • se quisermos definir que tipo de valores são aceitáveis ​​como prop
  • qual deve ser um valor padrão para o prop
  • se um valor deve (obrigatório) ser passado para o prop ou é opcional

então precisamos usar a notação de objeto para definir as props, como fizemos em address.js.

Se estamos criando componentes reutilizáveis ​​que também podem ser usados ​​por outros desenvolvedores da equipe, é uma boa prática definir props como objetos para que qualquer pessoa que use o componente tenha uma ideia clara de qual deve ser o tipo de dados e se é obrigatório ou opcional.

Também é referido como validação de adereços. O tipo pode ser qualquer um dos seguintes construtores nativos:

  • Corda
  • Número
  • Boleano
  • Variedade
  • Objeto
  • Função
  • ou um Construtor Personalizado

Alguns exemplos de validação de props retirados de http://vuejs.org/guide/components.html#Props

Vue.component('example', {
   props: {
       // basic type check (`null` means accept any type)
       propA: Number,
       // multiple possible types (1.0.21+)
       propM: [String, Number],
       // a required string
       propB: {
         type: String,
         required: true
       },
       // a number with default value
       propC: {
          type: Number,
          default: 100
       },
       // object/array defaults should be returned from a
       // factory function
       propD: {
          type: Object,
          default: function () {
             return { msg: 'hello' }
         }
       },
       // indicate this prop expects a two-way binding. will
       // raise a warning if binding type does not match.
       propE: {
          twoWay: true
       },
       // custom validator function
       propF: {
          validator: function (value) {
             return value > 10
          }
       },
       // coerce function (new in 1.0.12)
       // cast the value before setting it on the component
       propG: {
          coerce: function (val) {
            return val + '' // cast the value to string
          }
       },
       propH: {
          coerce: function (val) {
            return JSON.parse(val) // cast the value to Object
          }
       }
    }
});

camelCase vs kebab-case

Os atributos HTML não diferenciam maiúsculas de minúsculas, o que significa que ele não pode diferenciar entre addresstype e addressType, portanto, ao usar nomes de props camelCase como atributos, precisamos usar seus equivalentes kebab-case (delimitados por hífen): addressType deve ser escrito como address-type no atributo HTML.

Passando adereços ao usar o Vue JSX #

Temos um componente pai: Importando um componente filho nele, passaremos props por meio de um atributo. Aqui o atributo é ‘src’ e estamos passando o ‘src’ também.

ParentComponent.js #

import ChildComponent from './ChildComponent';
export default {
    render(h, {props}) {
        const src = 'https://cdn-images-1.medium.com/max/800/1*AxRXW2j8qmGJixIYg7n6uw.jpeg';
        return (
           <ChildComponent src={src} />   
        );
    }
};

E um componente filho, onde precisamos passar props. Precisamos especificar quais props estamos passando.

ChildComponent.js: #

export default {
    props: ['src'],
    render(h, {props}) {
        return ( 
            <a href = {props.src} download = "myimage" >
                Click this link
            </a>
        );
    }
};

Adereços Dinâmicos #

Assim como você pode vincular dados de uma visão ao modelo, você também pode vincular props usando a mesma diretiva v-bind para passar informações de componentes pai para filho.

JS #

new Vue({
    el: '#example',
    data: {
        msg: 'hello world'
    }
});

Vue.component('child', {
    props: ['myMessage'],
    template: '<span>{{ myMessage }}</span>
});

HTML #

<div id="example">
    <input v-model="msg" />
    <child v-bind:my-message="msg"></child>
    <!-- Shorthand ... <child :my-message="msg"></child> -->
</div>
