Como usar variáveis ​​dentro de um componente de script

Esta postagem fornece etapas para usar variáveis ​​(variável de usuário, parâmetro de pacote e parâmetro de projeto) no componente de script e visualizar o valor atualizado usando o ponto de interrupção e a janela de observação.

Parâmetros #

Parâmetro Detalhes
UserVar É como uma variável local usada dentro de um pacote. Seu valor pode ser lido e modificado no script task
Resultado É uma variável local que conterá o resultado concatenado. Seu valor pode ser lido e modificado no script task
PacoteVar É um parâmetro de pacote, que pode ser compartilhado entre pacotes. Seu valor é lido somente dentro do componente script
ProjetoParm É o parâmetro do projeto, que fica disponível após a implantação como uma configuração. Seu valor é lido apenas dentro do componente script

Passos para atingir o objetivo. #

Tarefas SSIS necessárias. #

  1. Tarefa de fluxo de dados: como o componente de script está disponível apenas no fluxo de dados.
  2. Componente de Script: Dentro dele usaremos variáveis ​​e brincaremos com seus valores.

Passos #

Existem dois métodos para acessar variáveis ​​dentro do componente de script

Primeiro Método - Usando this.Variables

  1. Crie duas variáveis ​​de usuário Result (String), UserVar (valor de String: UserVar), também crie um parâmetro de pacote PackageVar (valor de String: PackageVariable) e um parâmetro de projeto ProjectParam (valor de String: ProjectParameter).
  2. Arraste e solte uma Tarefa de fluxo de dados da Caixa de ferramentas do SSIS na seção de favoritos.
  3. Clique duas vezes na Tarefa de fluxo de dados que o levará ao Fluxo de dados. Agora, na caixa de ferramentas do SSIS, arraste e solte o Script Component presente na seção Common. Ele será exibido com uma janela com três opções de Origem, Destino e Transformação. Selecione Origem e clique em OK.
  4. Dê um duplo clique no Script component, no lado direito para ReadOnlyVariables clique nos três pontos para abrir uma janela Select Variable. Agora selecione User::UserVar, $Package::PackageVar e $Project::ProjectParm. Clique OK. Da mesma forma, clique nos três pontos correspondentes a ReadWriteVariables e selecione User::Result. Clique OK.
  5. No lado esquerdo do Editor de Transformação de Scripts, selecione Entradas e Saídas, no centro Expanda Saída O -> expanda Colunas de Saída na parte inferior Clique em Adicionar Coluna. Quando usamos o componente script como Source, é necessário ter coluna de saída, por isso criamos esta coluna de saída.
  6. À esquerda clique no Script, na parte inferior direita do editor clique em Edit Scripts, abrirá uma nova janela, nesta janela encontre o método PostExecute() e escreva this.Variables .Result = this.Variables.UserVar + this.Variables.PackageVar + this.Variables.ProjectParm; através desta.Variables estamos acessando a variável. Clique em Ctrl+S para salvar e fechar a janela. Clique OK.
  7. Vá para o Control Flow e clique com o botão direito do mouse em Data Flow Task e selecione Edit Breakpoints. Agora na nova janela Set Breakpoints Selecione Break quando o container receber o evento OnPostExecute. Clique OK.
  8. Agora, no Solution Explorer, clique com o botão direito do mouse no nome do pacote e clique em Executar pacote. Na barra de menu, clique em Debug -> Windows -> Watch -> Watch 1. Agora, na parte inferior, a janela Watch estará visível. Em Nome digite Usuário::Resultado e clique em enter. Em value, os valores concatenados {UserVarPackageVariableProjectParameter} podem ser vistos.

Segundo Método - Usando o Dispenser Variável

Quando o dispensador de variáveis ​​é usado, não é necessário adicionar as variáveis ​​às ReadOnlyVariables e ReadWriteVariables. Você pode usar os seguintes códigos para ler e gravar valores de variáveis: (O código é para ssis 2008)

Escreva

private void WriteVariable(String varName, Object varValue)
{

  IDTSVariables100 vars = null;
  VariableDispenser.LockForWrite(varName);
  VariableDispenser.GetVariables(out vars);
  vars[varName].Value = varValue;
  vars.Unlock();
 
}

Ler

private Object ReadVariable(String varName)
{

  Object varValue;
  IDTSVariables100 vars = null;
  VariableDispenser.LockForRead(varName);
  VariableDispenser.GetVariables(out vars);
  varValue = vars[varName].Value;
  vars.Unlock();
 
  return varValue;

}
