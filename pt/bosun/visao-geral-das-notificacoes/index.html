Sintaxe #

  • notificação nome {
  • email = dev-alerts@example.com, prod-alerts@example.com, …
  • postagem = http://example.com
  • obter = http://example.com
  • próximo = definição de outra notificação
  • tempo limite = 30m
  • runOnActions = false
  • corpo = {“texto”: {{.|json}}}
  • contentType = aplicativo/json
  • imprimir = verdadeiro
  • }

No Bosun, as notificações são usadas para novos incidentes de alerta e quando um alerta é confirmado/fechado/etc. Se você não quiser que os outros eventos acionem uma notificação, adicione runOnActions = false à definição de notificação.

Notificações HTTP GET/POST #

Os incidentes de alerta podem ser enviados para outro sistema usando solicitações HTTP GET ou HTTP POST. Você pode enviar o alerta renderizado diretamente (usando markdown no modelo talvez) ou usar body = ... {{.|json}} ... e contentType para enviar os dados de alerta como parte de um objeto JSON. Outra abordagem é enviar apenas as informações básicas do incidente e, em seguida, fazer com que o sistema de recebimento extraia detalhes adicionais da API do chefe.

notification postjson {
    post = ${env.POSTURL}
    body = {"text": {{.|json}}, apiKey=${env.APIKEY}}
    contentType = application/json
}

O contentType para solicitações HTTP GET/POST é application/x-www-form-urlencoded por padrão.

Notificações SMS com pivo #

Existem dois IDs que você precisará da sua conta pivo. Substitua authid e authtoken neste snippet por esses valores. O valor src também deve ser um número válido atribuído à sua conta. dst pode ser qualquer número que você quiser, ou vários separados por <.

notification sms {
  post = https://authid:authtoken@api.plivo.com/v1/Account/authid/Message/
  body = {"text": {{.|json}}, "dst":"15551234567","src":"15559876543"}
  contentType = application/json
  runOnActions = false
}

Isso enviará um texto para o assunto do alerta para todos os números em dst.

Notificações por e-mail #

Para enviar notificações por e-mail, você precisa adicionar as seguintes configurações ao seu arquivo de configuração:

#Using a company SMTP server (note only one can be define)
smtpHost = mail.example.com:25
emailFrom = bosun@example.com

#Using Gmail with TLS and username/password
smtpHost = smtp.gmail.com:587
emailFrom = myemail@gmail.com
smtpUsername = myemail@gmail.com
smtpPassword = ${env.EMAILPASSWORD}

#Chained notifications will escalate if an incident is not acked before the timeout
notification it {
    email = it-alerts@example.com
    next = oncall
    timeout = 30m
}

#Could set additional escalations here using any notification type (email/get/post)
#or set next = oncall to send another email after the timeout if alert is still not acked
notification oncall {
    email = escalated-alerts@example.com
}

#Multiple email addresses can be specified in one notification definition
#Use runOnActions = false to exclude updates on actions (ack/closed/forget)
notification engineering {
    email = core-alerts@example.com,qa-alerts@example.com,prod-alerts@example.com
    runOnActions = false
}

Visão geral #

Notificações de Bosun são atribuídas a definições de alerta usando warningNotification e critNotification e indicam para onde enviar o modelo de alerta renderizado quando ocorrer um novo incidente. As notificações podem ser enviadas por e-mail ou usar solicitações HTTP GET/POST. Há também uma notificação de impressão que apenas adiciona informações ao arquivo de log do Bosun.

Se você deseja ocultar uma URL, senha ou chave de API de texto simples, você pode usar ${env.VARIABLENAME} para carregar o valor de uma variável de ambiente (geralmente exportada do script de inicialização Bosun). Observe que não há proteções sobre quem pode acessar as variáveis ​​(elas podem ser facilmente exibidas em um modelo), mas isso impede que elas sejam exibidas diretamente na página do Editor de regras ou no arquivo .conf.

notification logfile {
    print = true
}

#print can be added to any notification type to help with debugging
notification email {
    email = sysadmins@example.com
    print = true
}

Notificações SMS com Twilio #

Troque AccountSid, AuthToken, ToPhoneNumber e FromPhoneNumber por suas credenciais/destinatários pretendidos. Você precisa garantir que, se ToPhoneNumber e FromPhoneNumber tiverem + neles, eles serão codificados em URL (ou seja: como %2B)

notification sms {
    post = https://{AccountSid}:{Authtoken}@api.twilio.com/2010-04-01/Accounts/{AccountSid}/Messages.json
    body = Body={{.}}&To={ToPhoneNumber}&From={FromPhoneNumber}
}

De essência: https://gist.github.com/aodj/58535c4c152b6073eaf5

Notificações do PagerDuty #

#Post to pagerduty.com
notification pagerduty {
    post = https://events.pagerduty.com/generic/2010-04-15/create_event.json
    contentType = application/json
    runOnActions = false
    body = `{
     "service_key": "myservicekey",
     "incident_key": {{.|json}},
     "event_type": "trigger",
     "description": {{.|json}},
     "client": "Bosun",
     "client_url": "http://bosun.example.com/"
    }`
}

Alterando a notificação usando a pesquisa #

Em alguns casos, você pode querer alterar qual notificação você usa com base em uma tag nas chaves de alerta. Você pode fazer isso usando o recurso Pesquisa. Nota: Lookup só funciona se você estiver usando OpenTSDB e enviando dados para o Bosun para serem indexados. Para outros back-ends ou dados não indexados, você deve usar lookupSeries.

notification default {
    email = team@example.com
}

notification JSmith{
    email = JSmith@example.com
}

#This will use the JSmith lookup for any alerts where the host tag starts with ny-jsmith
lookup host_base_contact {
    entry host=ny-jsmith* {
        main_contact = JSmith
    }
    entry host=* {
        main_contact = default
    }
}

alert blah {
    ...
    warn = q(...)
    warnNotification = lookup("host_base_contact", "main_contact")
    critNotification = lookup("host_base_contact", "main_contact")
}

Isso também pode ser aplicado a vários alertas usando Macros:

macro host.based.contacts {
    warnNotification = lookup("host_base_contact", "main_contact")
    critNotification = lookup("host_base_contact", "main_contact")
}
