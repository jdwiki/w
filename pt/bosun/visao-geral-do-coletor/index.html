Visão geral do coletor

  • Configuração com arquivo scollector.toml de amostra

Nesta página

  • Configuração com arquivo scollector.toml de amostra

Scollector é um agente de monitoramento que pode ser usado para enviar métricas para Bosun ou qualquer sistema que aceite métricas de estilo OpenTSDB. Ele é modelado após a estrutura de coleta de dados OpenTSDB’s tcollector, mas é escrito em Go e compilado em um único binário. Um dos objetivos do projeto é detectar automaticamente os serviços para que as métricas sejam enviadas com o mínimo ou nenhuma configuração necessária. Você também pode criar coletores externos que geram métricas usando um script ou executável e usar o Scollector para enfileirar e enviar as métricas para o servidor.

Você NÃO é obrigado a usar o Scollector ao usar o Bosun, pois também pode enviar métricas diretamente para a rota /api/put, usar outro agente de monitoramento ou usar um back-end diferente, como Graphite, InfluxDB ou ElasticSearch.

Configuração com arquivo scollector.toml de amostra #

Os binários do Scollector para Windows, Mac e Linux estão disponíveis na página de lançamento do Bosun e podem ser salvos em /opt/scollector/ ou C:\Arquivos de Programas\scollector\. O arquivo de configuração do Scollector usa [TOML v0.2.0](https://github.com/toml-lang/toml/blob/master/versions /en/toml-v0.2.0.md) para especificar várias configurações e padrões para ser nomeado scollector.toml na mesma pasta que o binário. O arquivo de configuração é opcional e necessário apenas se você precisar substituir um valor padrão ou incluir configurações para ativar um coletor específico.

#Where to send metrics. If omitted the default is bosun:80. 
#Config file setting can also be overridden using -h bosunhostname on command line 
Host = "mybosunserver.example.com:8080"

#Optional folder where to find external collector scripts/binaries
ColDir  = 'C:\Program Files\scollector\collectors'

#Number of data points to include in each batch. Default is 500, should be set higher if you are sending a lot of metrics.
BatchSize = 5000

Você pode instalar o Scollector como um serviço ou apenas executá-lo manualmente por meio de:

#Override default configuration file location
scollector -conf /path/to/myconfig.toml

#List all built-in collectors
scollector -l

#-p will print metrics to the screen instead of sending to Bosun. 
#-f "..." will only run specific collectors. Add DisableSelf = true to toml file to exclude scollector.* self metrics
scollector -p -f "c_cpu_windows,c_network_"

Executando o Collector como um serviço #

No Windows, você pode instalar o Scollector como um serviço usando o sinalizador -winsvc=“install”. No Mac e no Linux, você deve criar manualmente um serviço ou script de inicialização. Por exemplo, aqui está um arquivo de unidade systemd básico:

#Scollector unit file saved to /etc/systemd/system/scollector.service
[Unit]
Description=Scollector Service
After=network.target

[Service]
Type=simple
User=root
ExecStart=/opt/scollector/scollector -h mybosunserver.example.com
Restart=on-abort

[Install]
WantedBy=multi-user.target
