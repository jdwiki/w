Parâmetros #

Parâmetro Detalhes
indexRoot O nome raiz do índice a ser atingido, espera-se que o formato seja fmt.Sprintf("%s-%s", index_root, d.Format("2006.01.02"))
keyString Cria grupos (como conjuntos de tags) e também pode filtrar esses grupos. É o formato de "field:regex,field:regex...". O :regex pode ser omitido.
filterString Uma consulta Elastic regexp que pode ser aplicada a qualquer campo. Ele está no mesmo formato que o argumento keystring.
bucketDuração O mesmo formato é uma duração opentsdb e é o tamanho dos buckets retornados (ou seja, contagens a cada 10 minutos)
startDuration defina a janela de tempo a partir de agora - veja a função q() do OpenTSDB para mais detalhes.
endDuração defina a janela de tempo a partir de agora - veja a função q() do OpenTSDB para mais detalhes.

Suspensão #

As funções de consulta do LogStash estão obsoletas e devem ser usadas apenas com a v1.x do ElasticSearch. Se você estiver executando a v2 ou superior do ElasticSearch, consulte as funções do Elastic Query.

Ressalvas #

  • Atualmente, não há escape na string de chave, portanto, se você regex precisar ter uma vírgula ou aspas duplas, estará sem sorte.
  • Os regexs em keystring são aplicados duas vezes. Primeiro como um filtro regexp para elastic, e depois como go regexp para as chaves do resultado. Isso ocorre porque o valor pode ser uma matriz e você obterá grupos que devem ser filtrados. Isso significa que a linguagem regex é a interseção da especificação golang regex e a especificação elastic regex. Elastic usa regex estilo lucene. Isso significa que os regexes estão sempre ancorados (consulte a documentação).
  • Se o tipo do valor do campo no Elastic (também conhecido como mapeamento) for um número, as regexes não atuarão como regex. A única coisa que você pode fazer é uma correspondência exata no número, ou seja, “eventlogid:1234”. Recomenda-se que qualquer coisa que seja um identificador seja armazenada como uma string, pois não são números, mesmo que sejam compostos inteiramente de numerais.
  • Alertas usando essas informações provavelmente desejam definir ignoreUnknown, pois apenas os “grupos” que aparecem no período de tempo estão nos resultados

Contando o número total de documentos nos últimos 5 minutos #

lscount retorna uma contagem em intervalos de tempo de documentos correspondentes no índice LogStash, de acordo com o filtro especificado.

Um uso trivial disso seria verificar quantos documentos no total foram recebidos nos 5 minutos e alertar se estiver abaixo de um determinado limite.

Um alerta Bosun para isso pode ter a seguinte aparência:

alert logstash.docs {
    $notes = This alerts if there hasn't been any logstash documents in the past 5 minutes
    template = logstash.docs
    $count_by_minute = lscount("logstash", "", "", "5m", "5m", "")
    $count_graph = lscount("logstash", "", "", "1m", "60m", "")
    $q = avg($count_by_minute)
    crit = $q < 1
    critNotification = default
}

template logstash.docs {
    body = `{{template "header" .}}
    {{.Graph .Alert.Vars.count_graph }}
    {{template "def" .}}
    {{template "computation" .}}`
    subject = {{.Last.Status}}: Logstash docs per second: {{.Eval .Alert.Vars.q | printf "%.2f"}} in the past 5 minutes
}

Isso tem duas instâncias de lscount:

  • $count_by_minute = lscount(“logstash”, “”, “”, “5m”, “5m”, “”)
    • This counts the number of documents from the last 5 minutes, in a single 5 minute bucket. You will get one data point in the returned seriesSet with the total number of documents from the last 5 minutes, in the latest logstash index
  • $count_graph = lscount(“logstash”, “”, “”, “1m”, “60m”, “”)
    • This counts the number of documents from the last hour, in 1 minute buckets. There will be a total of 60 data points in the seriesSet returned, which in this instance is used in a graph.
