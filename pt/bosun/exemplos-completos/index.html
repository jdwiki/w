Linux Bonding Health #

Definição de modelo #

template linux.bonding {
    subject = {{.Last.Status}}: {{.Eval .Alert.Vars.by_host}} bad bond(s) on {{.Group.host}}
    body = `{{template "header" .}}
    <h2>Bond Status</h2>
    <table>
    <tr><th>Bond</th><th>Slave</th><th>Status</th></tr>
    {{range $r := .EvalAll .Alert.Vars.slave_status}}
        {{if eq $.Group.host .Group.host}}
            <tr>
                <td>{{$r.Group.bond}}</td>
                <td>{{$r.Group.slave}}</td>
                <td {{if lt $r.Value 1.0}} style="color: red;" {{end}}>{{$r.Value}}</td>
            </tr>
        {{end}}
    {{end}}
    </table>
    `
}

Definição de alerta #

alert linux.bonding {
    template = linux.bonding
    macro = host_based
    $notes = This alert triggers when a bond only has a single interface, or the status of a slave in the bond is not up
    $slave_status = max(q("sum:linux.net.bond.slave.is_up{bond=*,host=*,slave=*}", "5m", ""))
    $slave_status_by_bond = sum(t($slave_status, "host,bond"))
    $slave_count = max(q("sum:linux.net.bond.slave.count{bond=*,host=*}", "5m", ""))
    $no_good = $slave_status_by_bond < $slave_count || $slave_count < 2
    $by_host = max(t($no_good, "host"))
    warn = $by_host
}

Certificados SSL expirando #

Esses dados são coletados pelo http_unit e pelo scollector. Ele avisa quando um alerta expira dentro de um determinado período de dias e, em seguida, torna-se crítico se o certificado tiver passado da data de expiração. Isso segue o padrão recomendado de uso de aviso e crítico no Bosun (aviso: algo vai falhar, crit: algo falhou).

Definição do modelo #

template ssl.cert.expiring {
    subject = {{.Last.Status}}: SSL Cert Expiring in {{.Eval .Alert.Vars.daysLeft | printf "%.2f"}} Days for {{.Group.url_host}}
    body = `
    {{ template "header" . }}
    <table>
       <tr>
            <td>Url</td>
            <td>{{.Group.url_host}}</td>
       </tr>
       <tr>
            <td>IP Address Used for Test</td>
            <td>{{.Group.ip}}</td>
       </tr>
       <tr>
            <td>Days Remaining</td>
            <td>{{.Eval .Alert.Vars.daysLeft | printf "%.2f"}}</td>
       </tr>
       <tr>
            <td>Expiration Date</td>
            <td>{{.Last.Time.Add (parseDuration (.Eval .Alert.Vars.hoursLeft | printf "%vh")) }}</td>
       </tr>
    </table>
    `
}

Definição de alerta #

alert ssl.cert.expiring {
    template = ssl.cert.expiring
    ignoreUnknown = true
    $notes = This alert exists to notify of us any SSL certs that will be expiring for hosts monitored by our http unit test cases defined in the scollector configuration file.
    $expireEpoch = last(q("min:hu.cert.expires{host=ny-bosun01,url_host=*,ip=*}", "1h", ""))
    $hoursLeft = ($expireEpoch - epoch()) / d("1h")
    $daysLeft = $hoursLeft / 24
    warn = $daysLeft <= 50
    crit = $daysLeft <= 0
    warnNotification = default
    critNotification = default
}

Explicação do Alerta #

  • q(..) (func doc) consulta OpenTSDB, um dos backends suportados pelo Bosun. In retorna um tipo chamado seriesSet (que é um conjunto de séries temporais, cada uma identificada por tag).
  • last() (func doc) pega o último valor de cada série no seriesSet e retorna um numberSet.
  • A métrica hu.cert.expires. está retornando o timestamp Unix de quando o certificado irá expirar
  • epoch() (func doc) retorna o timestamp unix atual. Portanto, subtrair o timestamp unix atual da época de expiração é o tempo restante.
  • d() (func doc) retorna o número de segundos representado pela string de duração, a string de duração usa as mesmas unidades do OpenTSDB.

Modelo de cabeçalho #

Em modelos Bosun pode fazer referência a outros modelos. Para notificações por e-mail, você pode ter um modelo de cabeçalho para mostrar o que deseja em todos os alertas.

Modelo de cabeçalho #

template header {
    body = `
    <style>
    td, th {
        padding-right: 10px;
    }
    </style>
    <p style="font-weight: bold; text-decoration: underline;">
        <a style="padding-right: 10px;" href="{{.Ack}}">Acknowledge</a>
        <a style="padding-right: 10px;" href="{{.Rule}}">View Alert in Bosun's Rule Editor</a>
        {{if .Group.host}}
            <a style="padding-right: 10px;" href="https://status.stackexchange.com/dashboard/node?node={{.Group.host}}">View {{.Group.host}} in Opserver</a>
            <a href="http://kibana.ds.stackexchange.com/app/kibana?#/discover?_g=(refreshInterval:(display:Off,pause:!f,value:0),time:(from:now-15m,mode:quick,to:now))&_a=(columns:!(_source),index:%5Blogstash-%5DYYYY.MM.DD,interval:auto,query:(query_string:(analyze_wildcard:!t,query:'logsource:{{.Group.host}}')),sort:!('@timestamp',desc))">View {{.Group.host}} in Kibana</a>
        {{end}}
    </p>
    <table>
        <tr>
            <td><strong>Key: </strong></td>
            <td>{{printf "%s%s" .Alert.Name  .Group }}</td>
        </tr>
        <tr>
            <td><strong>Incident: </strong></td>
            <td><a href="{{.Incident}}">#{{.Last.IncidentId}}</a></td>
        </tr>
    </table>
    <br/>
    {{if .Alert.Vars.notes}}
        <p><strong>Notes:</strong> {{html .Alert.Vars.notes}}</p>
    {{end}}
    {{if .Alert.Vars.additionalNotes}}
        <p>
        {{if not .Alert.Vars.notes}}
            <strong>Notes:</strong>
        {{end}}
        {{ html .Alert.Vars.additionalNotes }}</p>
    {{end}}
    `
}

Explicações:

  • <style>...: Embora os blocos de estilo não sejam suportados no e-mail, o bosun processa os blocos de estilo e os insere no html. Portanto, este é o CSS compartilhado para todos os modelos que incluem esse modelo.
  • O link .Ack leva você a uma visualização Bosun onde você pode reconhecer o alerta. O link .Rule leva você ao editor de regras do Bosun definindo o modelo, regra e hora do alerta para que você possa modificar o alerta ou executá-lo em momentos diferentes.
  • {{if .Group.host}}...: .Group é o conjunto de tags do alerta. Portanto, quando a expressão warning ou crit tem tags como host=*, sabemos que o alerta está em referência a um host específico em nosso ambiente. Então, mostramos alguns links para hospedar coisas específicas.
  • O nome e a chave do alerta são incluídos para garantir que pelo menos as informações mais básicas estejam em qualquer alerta
  • .Alert.Vars.notes está incluído, portanto, se em algum alerta alguém definir as variáveis ​​$notes, isso será mostrado no alerta. O incentiva as pessoas a escrever notas explicando o propósito do alerta e como interpretá-lo.
  • .Alert.Vars.additionalNotes existe caso queiramos definir uma macro com notas, e então ter instâncias dessa macro com mais notas adicionadas às notas da macro.
