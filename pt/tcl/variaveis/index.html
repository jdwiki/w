Sintaxe #

  • set varName ?valor? – unset ?-nocomplain? , ?varName varName varName?

  • coloca $varName

  • coloca [set varName]

  • variável varName

  • global varName ?varName varName?

  • Parâmetros entre ?…? como ?varName? representam argumentos opcionais para um comando Tcl.

  • Documentação: global, upvar

Atribuindo valores a variáveis #

O comando set é usado para atribuir valores em Tcl. Quando é chamado com dois argumentos da seguinte maneira,

% set tempVar "This is a string."
This is a string.

ele coloca o segundo argumento (“Isto é uma string”) no espaço de memória referenciado pelo primeiro argumento (tempVar). set sempre retorna o conteúdo da variável nomeada no primeiro argumento. No exemplo acima, set retornaria “Esta é uma string.” sem as aspas.

  • Se value for especificado, o conteúdo da variável varName são iguais a valor.
  • Se varName consistir apenas em caracteres alfanuméricos, e sem parênteses, é um escalar variável.
  • Se varName tem o formato varName(index), é um membro de uma matriz associativa.

Observe que o nome da variável não está restrito ao alfabeto latino, pode consistir em qualquer combinação de caracteres unicode (por exemplo, armênio):

% set տուն house
house
% puts ${տուն}
house

Escopo #

set alpha 1

proc myproc {} {
    puts $alpha
}

myproc

Este código não funciona porque os dois alfas estão em escopos diferentes.

O comando set alpha 1 cria uma variável no escopo global (o que a torna uma variável global).

O comando puts $alpha é executado em um escopo que é criado quando o comando myproc é executado.

Os dois escopos são distintos. Isso significa que quando puts $alpha tenta procurar o nome alpha, ele não encontra nenhuma variável.

Podemos corrigir isso, no entanto:

proc myproc {} {
    global alpha beta
    puts $alpha
}

Neste caso, duas variáveis ​​globais, alpha e beta, são vinculadas a variáveis ​​de alias (com o mesmo nome) no escopo do procedimento. A leitura das variáveis ​​de alias recupera o valor nas variáveis ​​globais e a gravação nelas altera os valores nas variáveis ​​globais.

De maneira mais geral, o comando upvar cria aliases para variáveis ​​de qualquer um dos escopos anteriores. Ele pode ser usado com o escopo global (#0):

proc myproc {} {
    upvar #0 alpha alpha beta b
    puts $alpha
}

Os aliases podem receber o mesmo nome da variável que está vinculada a (alpha) ou outro nome (beta / b).

Se chamarmos myproc do escopo global, esta variante também funciona:

proc myproc {} {
    upvar 1 alpha alpha beta b
    puts $alpha
}

O número do escopo 1 significa “o escopo anterior” ou “o escopo do chamador”.

A menos que você realmente saiba o que está fazendo, #0, 0 e 1 são os únicos escopos que fazem sentido usar com upvar. (upvar 0 cria um alias local para uma variável local, não estritamente uma operação de escopo.)

Algumas outras linguagens definem o escopo por chaves e permitem que o código executado em cada escopo veja todos os nomes nos escopos adjacentes. Em Tcl, um único escopo é criado quando um procedimento é chamado e apenas seus próprios nomes são visíveis. Se um procedimento chamar outro procedimento, seu escopo será empilhado sobre o escopo anterior e assim por diante. Isso significa que, em contraste com as linguagens de estilo C que têm apenas escopo global e escopo local (com subescopos), cada escopo atua como um escopo delimitador (embora não imediatamente visível) para qualquer escopo que ele tenha aberto. Quando um procedimento retorna, seu escopo é destruído.

Documentação: global, upvar

Chamando set com um argumento #

set também pode ser invocado com apenas um argumento. Quando chamado com apenas um argumento, ele retorna o conteúdo desse argumento.

% set x 235
235
% set x
235

Excluindo variável/s #

O comando unset é usado para remover uma ou mais variáveis.

unset ?-nocomplain? ?--? ?name name name name?

  • Cada nome é um nome de variável especificado em qualquer uma das formas aceitáveis ​​para o comando set.

  • Se um nome se refere a um elemento de um array, então esse elemento é removido sem afetar o restante do array.

  • Se um nome consiste em um nome de array sem índice entre parênteses, então o array inteiro é deletado.

  • Se -nocomplain for fornecido como o primeiro argumento, todos os erros possíveis serão suprimidos da saída do comando.

  • A opção – indica o fim das opções, e deve ser usada se você deseja remover uma variável com o mesmo nome de qualquer uma das opções.

    % set x 235 235 % set x 235 % unset x % set x can’t read “x”: no such variable

Imprimindo o valor de uma variável #

Para imprimir o valor de uma variável como,

set tempVar "This is a string."

O argumento na instrução puts é precedido por um sinal $, que diz a Tcl para usar o valor da variável.

% set tempVar "This is a string."
This is a string.
% puts $tempVar
This is a string.

Variáveis ​​de namespace #

O comando variable garante que uma determinada variável de namespace seja criada. Até que um valor seja atribuído a ela, o valor da variável é indefinido:

namespace eval mynamespace {
    variable alpha
    set alpha 0
}

A variável pode ser acessada de fora do namespace (de qualquer lugar, na verdade) anexando o nome do namespace a ela:

set ::mynamespace::alpha

O acesso pode ser simplificado dentro de um procedimento usando o comando variable novamente:

proc ::mynamespace::myproc {} {
    variable alpha
    set alpha
}

Isso cria um alias local para a variável de namespace.

Para um procedimento definido em outro namespace, o nome da variável deve conter o namespace na invocação de variable:

proc myproc {} {
    variable ::mynamespace::alpha
    set alpha
}
