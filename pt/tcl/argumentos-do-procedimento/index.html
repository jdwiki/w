Referências: proc Expansão do argumento (seção 5)

Um procedimento que não aceita argumentos #

proc myproc {} {
    puts "hi"
}
myproc
# => hi

Uma lista de argumentos vazia (o segundo argumento após o nome do procedimento, “myproc”) significa que o procedimento não aceitará argumentos.

Um procedimento que aceita um número variável de argumentos #

### Definition
proc myproc {alpha {beta {}} {gamma green}} {
    puts [list $alpha $beta $gamma]
}

### Use
myproc A
# => A {} green
myproc A B
# => A B green
myproc A B C
# => A B C

Este procedimento aceita um, dois ou três argumentos: aqueles parâmetros cujos nomes são o primeiro item em uma lista de dois itens são opcionais. As variáveis ​​de parâmetro (alpha, beta, gamma) recebem tantos valores de argumentos quantos estiverem disponíveis, atribuídos da esquerda para a direita. Variáveis ​​de parâmetro que não recebem nenhum valor de argumento, em vez disso, obtêm seus valores do segundo item da lista da qual faziam parte.

Observe que os argumentos opcionais devem vir no final da lista de argumentos. Se o argumentoN-1 for opcional, o argumentoN deve ser opcional também. Se em um caso, onde o usuário tem argumentoN mas não argumentoN-1, o valor padrão do argumentoN-1 precisa ser explicitamente mencionado antes do argumentoN, ao chamar o procedimento.

myproc A B C D
# (ERROR) wrong # args: should be "myproc alpha ?beta? ?gamma?"

O procedimento não aceita mais de três argumentos: observe que uma mensagem de erro útil descrevendo a sintaxe do argumento é criada automaticamente.

Um procedimento que aceita um nome/referência a uma variável #

proc myproc {varName alpha beta} {
    upvar 1 $varName var
    set var [expr {$var * $alpha + $beta}]
}
set foo 1
myproc foo 10 5
puts $foo
# => 15

Nesse caso específico, o procedimento recebe o nome de uma variável no escopo atual. Dentro de um procedimento Tcl, tais variáveis ​​não são automaticamente visíveis, mas o comando upvar pode criar um alias para uma variável de outro nível de pilha: 1 significa o nível de pilha do chamador, #0 significa o nível global, etc. Neste caso , o nível de pilha 1 e o nome foo (da variável de parâmetro varName) permitem que upvar encontre essa variável e crie um alias chamado var. Cada operação de leitura ou escrita em var também acontece com foo no nível da pilha do chamador.

Um procedimento que aceita dois argumentos #

proc myproc {alpha beta} {
    ...
    set foo $alpha
    set beta $bar     ;# note: possibly useless invocation
}

myproc 12 34          ;# alpha will be 12, beta will be 34

Se a lista de argumentos consistir em palavras, esses serão os nomes das variáveis ​​locais no procedimento e seus valores iniciais serão iguais aos valores dos argumentos na linha de comando. Os argumentos são passados ​​por valor e o que quer que aconteça com os valores das variáveis ​​dentro do procedimento não influenciará o estado dos dados fora do procedimento.

Um procedimento que aceita qualquer número de argumentos #

proc myproc args { ... }
proc myproc {args} { ... } ;# equivalent

Se o nome do parâmetro especial args for o último item na lista de argumentos, ele receberá uma lista de todos os argumentos naquele ponto na linha de comando. Se não houver nenhum, a lista estará vazia.

Pode haver argumentos, incluindo opcionais, antes de args:

proc myproc {alpha {beta {}} args} { ... }

Este procedimento aceitará um ou mais argumentos. Os dois primeiros, se presentes, serão consumidos por alpha e beta: a lista do restante dos argumentos será atribuída a args.

A sintaxe #

Às vezes, o que você tem é uma lista, mas o comando para o qual você deseja passar os itens da lista exige obter cada item como um argumento separado. Por exemplo: o comando winfo children retorna uma lista de janelas, mas o comando destroy só receberá uma sequência de argumentos de nome de janela.

set alpha [winfo children .]
# => .a .b .c
destroy $alpha
# (no response, no windows are destroyed)

A solução é usar a sintaxe {*}:

destroy {*}[winfo children .]

ou

destroy {*}$alpha

O que a sintaxe {*} faz é pegar o valor a seguir (sem espaço em branco no meio!) e juntar os itens desse valor na linha de comando como se fossem argumentos individuais.

Se o valor a seguir for uma lista vazia, nada será emendado:

puts [list a b {*}{} c d]
# => a b c d

Se houver um ou mais itens, eles serão inseridos:

puts [list a b {*}{1 2 3} c d]
# => a b 1 2 3 c d
