Dicionários em Tcl são values que mantêm um mapeamento de valores arbitrários para outros valores arbitrários. Eles foram introduzidos no Tcl 8.5, embora existam versões limitadas para (o agora não suportado) Tcl 8.4. Os dicionários são sintaticamente iguais às listas com números pares de elementos; o primeiro par de elementos é a primeira chave e valor do dicionário, o segundo par é a segunda tupla.

Desta forma:

fox "quick brown" dogs "lazy"

é um dicionário válido. A mesma chave pode ser várias vezes, mas é exatamente como se o valor da última estivesse no valor da anterior; estes são o mesmo dicionário:

abcd {1 2 3} defg {2 3 4} abcd {3 4 5}

abcd {3 4 5} defg {2 3 4}

O espaço em branco não é importante, assim como as listas.

Um conceito importante com dicionários é a ordem de iteração; os dicionários tentam usar a ordem de inserção da chave como sua ordem de iteração, embora quando você atualiza o valor de uma chave que já existe, você sobrescreve o valor dessa chave. Novas chaves vão no final.

Referências: dict

Anexação de lista a um dicionário aninhado #

Se tivermos este dicionário:

set alpha {alice {items {}} bob {items {}} claudia {items {}} derek {items {}}}

E quiser adicionar “fork” e “peanut” aos itens de Alice, este código não funcionará:

dict lappend alpha alice items fork peanut
dict get $alpha alice
# => items {} items fork peanut

Como seria impossível para o comando saber onde os tokens de chave terminam e os valores a serem anexados à lista começam, o comando é limitado a um token de chave.

A maneira correta de anexar ao dicionário interno é esta:

dict with alpha alice {
    lappend items fork peanut
}
dict get $alpha alice
# => items {fork peanut}

Isso funciona porque o comando dict with nos permite percorrer dicionários aninhados, tantos níveis quanto o número de tokens de chave que fornecemos. Em seguida, ele cria variáveis ​​com os mesmos nomes das chaves desse nível (apenas uma aqui: items). As variáveis ​​são inicializadas com o valor do item correspondente no dicionário. Se alterarmos o valor, esse valor alterado será usado para atualizar o valor do item do dicionário quando o script terminar.

(Observe que as variáveis ​​continuam a existir quando o comando termina.)

Uso básico de um dicionário #

Criando um dicionário:

set mydict [dict create a 1 b 2 c 3 d 4]
dict get $mydict b ; # returns 2
set key c
set myval [dict get $mydict $key]
puts $myval
# remove a value
dict unset mydict b
# set a new value
dict set mydict e 5

As chaves do dicionário podem ser aninhadas.

dict set mycars mustang color green
dict set mycars mustang horsepower 500
dict set mycars prius-c color orange
dict set mycars prius-c horsepower 99
set car [dict get $mycars mustang]
# $car is: color green horsepower 500
dict for {car cardetails} $mycars {
  puts $car
  dict for {key value} $cardetails {
    puts "  $key: $value"
  }
}

O comando dict get pode gerar um erro #

set alpha {a 1 b 2 c 3}
dict get $alpha b
# => 2
dict get $alpha d
# (ERROR) key "d" not known in dictionary

Se dict get for usado para recuperar o valor de uma chave ausente, um erro será gerado. Para evitar o erro, use dict existe:

if {[dict exists $alpha $key]} {
    set result [dict get $alpha $key]
} else {
    # code to deal with missing key
}

Como lidar com uma chave ausente, é claro, depende da situação: uma maneira simples é definir o resultado para um valor “vazio” padrão.

Se o código nunca tentar recuperar outras chaves que estão no dicionário, dict get obviamente não falhará. Mas para chaves arbitrárias, dict get é uma operação que precisa ser protegida. De preferência testando com dict existe, embora a captura de exceção também funcione.

Iterando em um dicionário #

Você pode iterar sobre o conteúdo de um dicionário com dict for, que é semelhante a foreach:

set theDict {abcd {ab cd} bcde {ef gh} cdef {ij kl}}
dict for {theKey theValue} $theDict {
    puts "$theKey -> $theValue"
}

Isso produz esta saída: <pré> abcd -> abcd bcde -> ef gh cdef -> ij kl

Você obteria a mesma saída usando dict keys para listar as chaves e iterar sobre isso:

foreach theKey [dict keys $theDict] {
    set theValue [dict get $theDict $theKey]
    puts "$theKey -> $theValue"
}

Mas dict for é mais eficiente.
