Outro benefício de usar strings de expressão entre colchetes é que o compilador de bytes geralmente pode gerar código mais eficiente (5 - 10x mais rápido) a partir deles.

Os problemas com expressões sem chaves #

É uma boa prática fornecer argumentos de string de expressão como strings entre colchetes. O título “Dupla Substituição” descreve as razões importantes por trás do mesmo.

O comando expr avalia uma string de expressão baseada em operador para calcular um valor. Essa string é construída a partir dos argumentos na invocação.

expr 1 + 2    ; # three arguments
expr "1 + 2"  ; # one argument
expr {1 + 2}  ; # one argument

Essas três invocações são equivalentes e a string de expressão é a mesma.

Os comandos if, for e while usam o mesmo código avaliador para seus argumentos de condição:

if {$x > 0} ...
for ... {$x > 0} ... ...
while {$x > 0} ...

A principal diferença é que a string de expressão de condição deve ser sempre um único argumento.

Como acontece com todos os argumentos em uma invocação de comando em Tcl, o conteúdo pode ou não estar sujeito a substituição, dependendo de como eles são citados / escapados:

set a 1
set b 2
expr $a + $b   ; # expression string is {1 + 2}
expr "$a + $b" ; # expression string is {1 + 2}
expr \$a + \$b ; # expression string is {$a + $b}
expr {$a + $b} ; # expression string is {$a + $b}

Há uma diferença no terceiro e quarto casos, pois as barras invertidas/colchetes impedem a substituição. O resultado ainda é o mesmo, já que o avaliador dentro de expr pode realizar a substituição da variável Tcl e transformar a string em {1 + 2}.

set a 1
set b "+ 2"
expr $a $b   ; # expression string is {1 + 2}
expr "$a $b" ; # expression string is {1 + 2}
expr {$a $b} ; # expression string is {$a $b}: FAIL!

Aqui temos problemas com o argumento entre colchetes: quando o avaliador em expr executa substituições, a expressão string já foi analisada em operadores e operandos, então o que o avaliador vê é uma string consistindo de dois operandos sem operador entre eles. (A mensagem de erro é “operador ausente em _@_ na expressão "$a _@_$b"”.)

Nesse caso, a substituição de variável antes de expr ser chamada evitou um erro. Apertar o argumento impedia a substituição de variável até a avaliação da expressão, o que causava um erro.

Situações como essa podem ocorrer, geralmente quando uma expressão a ser avaliada é passada como uma variável ou parâmetro. Nesses casos, não há outra escolha a não ser deixar o argumento sem chaves para permitir que o avaliador de argumento “descompacte” a string de expressão para entrega em expr.

Na maioria dos outros casos, porém, o uso da expressão não faz mal e, de fato, pode evitar muitos problemas. Alguns exemplos disso:

Dupla substituição

set a {[exec make computer go boom]}
expr $a      ; # expression string is {[exec make computer go boom]}
expr {$a}    ; # expression string is {$a}

A forma sem chaves executará a substituição do comando, que é um comando que destrói o computador de alguma forma (ou criptografa ou formata o disco rígido, ou o que você tiver). A forma entre colchetes realizará uma substituição de variável e, em seguida, tentará (e falhará) fazer algo da string “[exec make computer go boom]”. Desastre evitado.

Loops infinitos

set i 10
while "$i > 0" {puts [incr i -1]}

Este problema afeta tanto for quanto while. Embora pareça que este loop faria uma contagem regressiva até 0 e sairia, o argumento de condição para while é na verdade sempre 10>0 porque foi assim que o argumento foi avaliado quando o comando while foi ativado. Quando o argumento está entre colchetes, ele é passado para o comando while como $i>0, e a variável será substituída uma vez para cada iteração. Use isso em vez disso:

while {$i > 0} {puts [incr i -1]}

Avaliação total

set a 1
if "$a == 0 && [incr a]" {puts abc}

Qual é o valor de a depois de executar este código? Como o operador && só avalia o operando direito se o operando esquerdo for verdadeiro, o valor ainda deve ser 1. Mas, na verdade, é 2. Isso ocorre porque o avaliador de argumento já executou todas as substituições de variáveis ​​e comandos no momento em que a expressão cadeia é avaliada. Use isso em vez disso:

if {$a == 0 && [incr a]} {puts abc}

Vários operadores (os conectivos lógicos || e &&, e o operador condicional ?:) são definidos para não avaliar todos os seus operandos, mas eles só podem funcionar conforme projetado se a string da expressão estiver entre chaves.

Multiplicando uma variável por 17 #

set myVariable [expr { $myVariable * 17 }]

Isso mostra como você pode usar uma expressão simples para atualizar uma variável. O comando expr não atualiza a variável para você; você precisa pegar seu resultado e escrevê-lo na variável com set.

Observe que as novas linhas não são importantes na linguagem pouco compreendida por expr, e adicioná-las pode tornar as expressões mais longas muito mais fáceis de ler.

set myVariable [expr {
    $myVariable * 17
}]

Isso faz exatamente a mesma coisa.

Chamando um comando Tcl de uma expressão #

Às vezes você precisa chamar um comando Tcl da sua expressão. Por exemplo, supondo que você precise do comprimento de uma string nele. Para isso, basta usar uma sequência [...] na expressão:

set halfTheStringLength [expr { [string length $theString] / 2 }]

Você pode chamar qualquer comando Tcl desta forma, mas se você se pegar chamando expr em si, pare! e pense se você realmente precisa dessa chamada extra. Você pode geralmente fazer muito bem colocando a expressão interna entre parênteses.

Erro de palavra simples inválida #

No próprio Tcl, uma string que consiste em uma única palavra não precisa ser citada. Na linguagem de strings de expressão que expr avalia, todos os operandos devem ter um tipo identificável.

Os operandos numéricos são escritos sem qualquer decoração:

expr {455682 / 1.96e4}

Assim são as constantes booleanas:

expr {true && !false}

A sintaxe de substituição da variável Tcl é reconhecida: o operando será definido para o valor da variável:

expr {2 * $alpha}

O mesmo vale para a substituição de comandos:

expr {[llength $alpha] > 0}

Os operandos também podem ser chamadas de funções matemáticas, com uma lista de operandos separados por vírgulas entre parênteses:

expr {sin($alpha)}

Um operando pode ser uma string com aspas duplas ou entre colchetes. Uma string com aspas duplas estará sujeita a substituição, assim como em uma linha de comando.

expr {"abc" < {def}}

Se um operando não for um dos acima, é ilegal. Como não há nenhuma dica que mostre que tipo de palavra é, expr sinaliza um erro de palavra simples.
