Expressão S #

Uma expressão em Scheme é o que vai ser executado. Uma expressão S, como geralmente é chamada, começa com um ( e termina com um ). O primeiro membro da expressão é o que será executado. O seguinte membro da expressão são os parâmetros que serão enviados para a expressão durante a avaliação da expressão.

Por exemplo, adicionando números:

(+ 1 2 3)

Nesse caso, + é um símbolo para uma função add que recebe vários parâmetros. 1, 2 e 3 são enviados para a função +.

S-Expression pode conter S-Expressions como parâmetros, conforme mostrado no exemplo a seguir:

(if (< x y)
  x
  y)

Que pode ser lido como se x fosse menor que y return x senão return y. Neste exemplo, avaliamos a expressão de condição, dependendo do valor resolvido, x ou y serão retornados. Poderia ser avaliado a este

(if #t x y)
x
(if #f x y)
y

Um exemplo menos óbvio para iniciantes é ter uma S-Expression como parte do primeiro membro de uma S-Expression. Dessa forma, podemos alterar o comportamento de um método alterando a função que será chamada sem precisar criar ramificações com os mesmos parâmetros. Aqui está um exemplo rápido de uma expressão que adiciona ou subtrai números se x estiver abaixo de y.

((if (< x y) + -) 
  1 2 3)

Se x estiver abaixo de y, a expressão será avaliada como:

(+ 1 2 3)
6

por outro lado

(- 1 2 3)
-4

Como você pode ver, Scheme permite que o programador construa um pedaço de código complexo enquanto fornece ao programador as ferramentas para evitar a duplicação de código. Em outras linguagens podemos ver o mesmo exemplo escrito assim:

(se (< xy) (+ 1 2 3) (- 1 2 3))

O problema com esse método é que duplicamos muito código enquanto a única coisa que muda é o método que está sendo chamado. Este exemplo é bastante simples, mas com mais condições podemos ver muitas linhas semelhantes duplicadas.

Macro let simples #

As expressões let no esquema são de fato macros. Eles podem ser expressos com lambdas. Um let simples pode ser assim:

(let ((x 1) (y 2))
  (+ x y))

Ele retornará 3 quando o valor da última expressão do corpo let for retornado. Como você pode ver, uma expressão let está realmente executando algo. Se traduzirmos esta parte do código com lambdas, obteremos algo assim:

((lambda (x y) (+ x y)) 1 2)

Aqui podemos ver que estamos chamando o lambda anônimo com 1 e 2 diretamente. Portanto, o resultado neste caso também é 3.

Com isso em mente, entendemos que uma expressão let é composta por 2 partes. Ele tem parâmetros e um corpo como um lambda, mas a diferença é que a expressão let é chamada logo após sua avaliação.

Para explicar como uma expressão let funciona de uma visão abstrata para uma visão concreta, ficaria assim.

(let params body ...)
(let (param1 param2 ...) body ...)
(let ((p1 val1) (p2 val2) ...) body ...)

Os parâmetros são uma lista de pares de (name value) a serem usados ​​no corpo do let.

Por que usar a expressão let?

Let expressões são particularmente úteis para armazenar variáveis ​​em um método como inicializações de variável em linguagens como c. É favorável ao uso de define porque fora da expressão let, as variáveis ​​sumiram… Usar um define é na verdade adicionar uma variável ao ambiente de execução atual. As variáveis ​​que são adicionadas ao ambiente global não podem ser removidas. Deixe a expressão ser segura para usar em qualquer lugar. Ele também pode ser usado para variáveis ​​fantasmas sem tocar nos escopos pai.

Por exemplo:

(let ((x 1))
  (let ((x 2) (y x))
    (display x)
    (display y))
  (display x))

Ele irá imprimir:

2
1
1

Neste caso, x é definido com 1, então fantasma pelo x no segundo let com o valor 2. A variável y é iniciada com o valor x do escopo pai. Depois que a expressão interna let é executada, ela exibe o valor inicial de x com 1. A expressão interna let não alterou o valor do escopo pai.

Sempre que você precisar inicializar variáveis, você deve usar expressões let como esta:

(let (
  (user (get-current-user session))
  (db (get-current-db session))
  (ids (get-active-ids session))
  )
  (mark-task-completed db user ids)
  (change-last-logged db user)
  (db-commit db))

Aqui neste exemplo, as variáveis ​​são inicializadas e usadas várias vezes no bloco de código. E quando a expressão let termina, as variáveis ​​são automaticamente liberadas, pois não são mais necessárias.

Sintaxe pontilhada para pares #

Existe uma sintaxe particular que nos permite escrever a célula cons de uma forma mais compacta do que usando o construtor cons.

Um par pode ser escrito como:

'(1 . 2) == (cons 1 2)

A grande diferença é que podemos criar pares usando aspas. Caso contrário, Scheme criaria uma lista adequada (1 . (2 . '())).

A sintaxe de ponto força a expressão a ter apenas 2 membros. Cada membro pode ser de qualquer tipo, incluindo pares.

'(1 . (2 . (3 . 4)))
> (1 2 3 . 4)

Observe que a lista imprópria deve ser exibida com um ponto no final para mostrar que o cdr do último par da lista não é a lista vazia '().

Essa maneira de mostrar listas às vezes é confusa, pois a expressão a seguir seria expressa de maneira diferente do esperado.

'((1 . 2) . ( 3 . 4))
> ((1 . 2) 3 . 4)

Como a lista geralmente pula o ., o primeiro argumento da lista seria (1 . 2), o segundo argumento seria 3 mas como a lista é imprópria, o último . é mostrado para mostrar que o último elemento da lista não é '(). Mesmo pensando, os dados são mostrados de uma forma diferente, os dados internos são como foram criados.
