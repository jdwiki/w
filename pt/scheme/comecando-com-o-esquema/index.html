Instalando o esquema CHICKEN #

CHICKEN é um interpretador e compilador de Scheme com seu próprio sistema de módulo de extensão chamado “eggs”. É capaz de compilar Scheme para código nativo compilando primeiro Scheme para C.

Instalando #

Debian ou Ubuntu ou outras distribuições derivadas: #

sudo apt-get install chicken-bin

Fedora / RHEL / CentOS: #

sudo yum install chicken-bin

Arch Linux: #

sudo pacman -S chicken

Gentoo: #

sudo emerge -av dev-scheme/chicken

OS X com Homebrew: #

brew install chicken

OpenBSD #

doas pkg_add -vi chicken

Microsoft Windows #

  • Instale o MSYS2

  • Execute o shell MSYS2 MinGW-w64

  • Instale alguns pré-requisitos executando:

    pacman -S mingw-w64-cross-toolchain base-devel mingw-w64-x86_64-gcc winpty wget

  • Baixe o tarball da última versão digitando:

    wget https://code.call-cc.org/releases/current/chicken.tar.gz

  • Extraia o tarball executando tar xvf chicken.tar.gz

  • Entre no diretório extraído, por exemplo, digitando cd chicken-4.11.0

  • Execute make PLATFORM=mingw-msys install

Se você tiver problemas para executar csi, tente executar winpty csi

Usando FRANGO #

Para usar o CHICKEN Scheme REPL, digite csi na linha de comando.

Para compilar um programa Scheme usando CHICKEN, execute csc program.scm, que criará um executável chamado program no diretório atual.

Instalando módulos #

O Chicken Scheme tem muitos módulos que podem ser navegados no índice do ovo. Eggs são módulos de esquema que serão baixados e compilados pelo esquema de frango. Em alguns casos, pode ser necessário instalar dependências externas usando seu gerenciador de pacotes usual.

Você instala os ovos escolhidos com este comando:

sudo chicken-install [name of egg]

Fazendo uso do REPL #

Você pode querer adicionar suporte readline ao seu REPL para fazer a edição de linha em csi se comportar mais como você poderia esperar.

Para fazer isso, execute sudo chicken-install readline e crie um arquivo chamado ~/.csirc com o seguinte conteúdo:

(use readline)
(current-input-port (make-readline-port))
(install-history-file #f "/.csi.history")

Instalando o mit-scheme #

A seguir estão exemplos de como instalar o MIT/GNU Scheme:

Instalação Debian/Ubuntu:

sudo apt-get install mit-scheme

Instalação manual:

Baixe o binário Unix diretamente do Projeto GNU, então siga as instruções da página oficial da web:

# Unpack the tar file
tar xzf mit-scheme.tar.gz

# move into the directory
cd mit-scheme/src

# configure the software
./configure

Por padrão, o software será instalado em /usr/local, nos subdiretórios bin e lib. Se você quiser instalá-lo em outro lugar, por exemplo /opt/mit-scheme, passe a opção --prefix para o script configure, como em ./configure --prefix=/opt/mit-scheme.

O script configure aceita todos os argumentos normais para tais scripts e, adicionalmente, aceita alguns que são específicos do esquema MIT/GNU. Para ver todos os argumentos possíveis e seus significados, execute o comando ./configure --help.

# build
make compile-microcode

# compile
make install # may require super-user permissions (Depending on configuration)

Windows 7:

O executável auto-instalável pode ser encontrado no site oficial.

MIT/GNU Scheme é distribuído como um executável auto-instalável. A instalação do software é simples. Basta executar o arquivo baixado e responder às perguntas do instalador. O instalador permitirá que você escolha o diretório no qual o MIT/GNU Scheme deve ser instalado e o nome da pasta na qual os atalhos devem ser colocados.
