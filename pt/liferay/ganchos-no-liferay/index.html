Gancho JSP #

Hooks JSP são um plugin especial do liferay que permite modificar jsp-s do portlet principal, digamos que você queira modificar o portlet de login para mostrar Welcome in my custom login!.

A estrutura mínima para um Plugin Hook é a seguinte:

[project-name]-hook/
 └── docroot/
     ├── WEB-INF/
     │   ├── src/
     │   ├── lib/
     │   ├── liferay-hook.xml
     │   ├── liferay-plugin-package.properties
     │   └── web.xml
     └── META-INF/
         ├── custom_jsps/
         └── MANIFEST.MF

liferay-hook.xml é o arquivo que distingue o tipo de hook que você está usando, aqui você define dentro da tag hook o parâmetro apropriado para o hook, para o hook JSP:

<?xml version="1.0"?>
<!DOCTYPE hook PUBLIC "-//Liferay//DTD Hook 6.2.0//EN" "http://www.liferay.com/dtd/liferay-hook_6_2_0.dtd">

<hook>
    <custom-jsp-dir>/custom_jsps</custom-jsp-dir>
</hook>

login.jsp é encontrado no Liferay em /docroot/html/portlet/login/login.jsp, para fazer um hook dele precisamos adicionar um jsp com o mesmo nome e caminho em nossa pasta custom_jsps.

Quando o hook for implantado, o Liferay procurará no liferay-hook.xml o valor custom-jsp-dir e substituirá todos os JSPs do portal pelos encontrados neste diretório. Os jsp’s originais são salvos com o nome <nome original>.portal.jsp para serem restaurados em caso de desinstalação do gancho.

Podemos até chamar os JSPs originais no novo JSP modificado se quisermos manter o código tornando-o adaptável a atualizações ou upgrades da versão subjacente da plataforma Liferay. Para fazer isso, em seu JSP customizado, use o seguinte padrão:

<liferay-util:buffer var="contentHtml">
    <liferay-util:include page="/html/{ JSP file’s path }" />
</liferay-util:buffer>

onde { caminho do arquivo JSP } neste caso será portlet/login/login.portal.jsp. Fazer isso é chamado de estendendo o jsp original.

Então podemos adicionar conteúdo a ele com:

<%
contentHtml = StringUtil.add("Stuff I'm adding BEFORE the original content", contentHtml,"\n");    
contentHtml = StringUtil.add(contentHtml,"Stuff I'm adding AFTER the original content","\n");
%>
<%= contentHtml %>

Ganchos de Ação do Struts #

Este tipo de Gancho pode ser usado para substituir o portal principal (por exemplo, c/portal/login) e ações do portlet struts (por exemplo, /login/forgot_password), essas ações para o Portal Liferay são especificadas em um struts-config.xml na pasta WEB-INF. Para substituir uma ação:

  1. no arquivo liferay-hook.xml do seu plugin hook em docroot/WEB-INF, adicione um elemento struts-action dentro do elemento hook.

  2. Dentro do elemento struts-action, adicione struts-action-path que especifica o caminho da ação que você está substituindo e struts-action-impl que especifica sua classe de ação personalizada. Isso se parece com:

    /login/login com.myhook.action.ExampleStrutsPortletAction

  3. Crie uma classe de ação de portlet Struts que estenda BaseStrutsPortletAction. Um exemplo desta classe é:

    public class ExampleStrutsPortletAction extends BaseStrutsPortletAction {

         public void processAction(StrutsPortletAction originalStrutsPortletAction,
             PortletConfig portletConfig, ActionRequest actionRequest,
             ActionResponse actionResponse) throws Exception {

         System.out.println("Custom Struts Action");

         originalStrutsPortletAction.processAction(originalStrutsPortletAction,
                 portletConfig, actionRequest, actionResponse);
     }

 public String render(StrutsPortletAction originalStrutsPortletAction,
         PortletConfig portletConfig, RenderRequest renderRequest,
         RenderResponse renderResponse) throws Exception {

     System.out.println("Custom Struts Action");

     return originalStrutsPortletAction.render(null, portletConfig,
             renderRequest, renderResponse);
 }

    }

Chamar o método que está sendo substituído, como originalStrutsPortletAction.processAction, não é obrigatório, mas uma prática recomendada para manter o comportamento da Ação inalterado em relação ao Portal Liferay. Este tipo de gancho pode ser usado para adicionar novas Ações do Struts também, é o mesmo que modificar uma ação existente, neste caso liferay-hook.xml seria:

<struts-action>
    <struts-action-path>/my/custom/path</struts-action-path>
    <struts-action-impl>
    com.myhook.action.ExampleStrutsAction
    </struts-action-impl>
</struts-action>

Olá usuário “Nome” com ganchos #

Este exemplo mostrará como fazer um simples “Hello User [name]” após o login. O exemplo é baseado em realizar uma ação personalizada usando um gancho

No terminal de linha de comando, navegue até a pasta hooks do SDK de plug-ins. Para criar um projeto de gancho, você deve executar o script de criação. Aqui está o formato a seguir na execução do script:

create.[sh|bat] [project-name] “[Hook Display Name]”

No Linux e Mac OS X, você digitaria um comando semelhante ao deste exemplo:

./create.sh Hello-user “Hello User”

No Windows, você digitaria um comando semelhante ao deste exemplo:

create.bat Olá-usuário “Meu Gancho”

O assistente de novo projeto do Liferay IDE e os scripts de criação geram projetos de gancho na pasta de ganchos do SDK do plug-in. O SDK de plug-ins anexa automaticamente “-hook” ao nome do seu projeto.

Se você criou seu projeto de gancho a partir do Liferay IDE ou da linha de comando, você acaba com a mesma estrutura de projeto (veja antes).

  • Determine o evento no qual você deseja acionar sua ação personalizada. Consulte a documentação do portal.properties para encontrar a correspondência propriedade do evento. Dica: as propriedades do evento possuem .event em seu nome. Existem propriedades de sessão, inicialização, desligamento e eventos do portal em as seguintes seções da documentação do portal.properties: Sessão - Eventos de inicialização - Eventos de desligamento - Portal Eventos

  • Em seu projeto de gancho, crie uma classe Java que estenda o com.liferay.portal.kernel.events.Action class. Substitua o método Action.run(HttpServletRequest, HttpServletResponse).

         import com.liferay.portal.kernel.events.Action;
     import javax.servlet.http.HttpServletRequest;
     import javax.servlet.http.HttpServletResponse;
     import com.liferay.portal.model.User;
     import com.liferay.portal.util.PortalUtil;

     public class HelloUser extends Action {
         public void run(HttpServletRequest req, HttpServletResponse res) {
             User user = PortalUtil.getUser(req);
             System.out.println("Hello User "+user.getScreenName());
         }
     }

Importante: Se sua ação acessar o objeto HttpServletRequest, estender com.liferay.portal.kernel.events.Action; caso contrário, estenda com.liferay.portal.struts.SimpleAction.

  • Crie um arquivo de propriedades, portal.properties, dentro do seu gancho project’s docroot/WEB-INF/src folder. Then add the name of the portal event property that corresponds to the event on which you want to perform your action. Specify your action class’ fully qualified name as the property’s value.

     `login.events.post=HelloUser`

    For example, to perform a class’ action just prior to the portal logging in a user, you’d specify the login.events.pre property with your action class as its value. It could look like this property setting.

Importante: Como as propriedades do portal como login.events.pre aceitam vários valores, você deve anexar seus valores aos valores existentes. Você pode modificar repetidamente as propriedades de ganchos adicionais.

Modifique apenas uma propriedade do portal que aceite um único valor de um plugin de gancho único. Se você modificar o valor de uma propriedade de vários plugins, o Liferay não saberá qual valor usar.

  • Edite seu arquivo docroot/WEB-INF/liferay-hook.xml e adicione seu nome do arquivo de propriedades do portal do gancho como o valor para o elemento <portal-properties>...</portal-properties> dentro do seu elemento <hook>...</hook> do gancho. For example, if your hook’s properties file name is portal.properties, you’d specify this element:

    portal.properties

  • Implante seu gancho, vá para o caminho do gancho e digite ant clean deployvocê verá o .war na pasta dist.

Agora, se você fizer login no liferay, verá no log do servidor uma mensagem como “Hello user Admin”.

Gancho do Ouvinte do Modelo #

Fundo #

Os ganchos do Model Listener são um tipo de plug-in do Liferay que escuta os eventos obtidos em um modelo e executa o código em resposta. Os ganchos do Ouvinte do Modelo são semelhantes aos ganchos de Ação do Struts Personalizados, pois respondem a uma ação realizada no portal. No entanto, enquanto as ações do Struts respondem a uma ação realizada por um usuário, um Model Listener responde (antes ou depois) a um evento envolvendo um modelo Liferay.

Diferenças #

Aqui estão alguns exemplos de Struts Actions v. Model Listeners para comparação.

  • Ação de Struts
    • User Login
    • Account Creation
    • Extend Session
    • Move Folder
  • Modelo de Ouvinte
    • After folder is created
    • When user information is updated
    • After bookmark is removed
    • Before a role association is made

O melhor recurso para aprender a arquitetura do Liferay é através de seu código fonte. Todos os seus arquivos de origem estão localizados no GitHub e visualizando seus JavaDocs. Você pode ver todos os modelos principais do portal no JavaDocs e todas as ações do Struts no GitHub.

Exemplo #

Neste tutorial, vamos desenvolver um Model Listener que envia um e-mail para um usuário depois que sua conta é criada. Para fazer isso, vamos escrever uma classe chamada UserModelListener que estenderá o BaseModelListener do Liferay. Veremos brevemente a criação do gancho e cobriremos as modificações necessárias nos seguintes arquivos de configuração

  • portal.propriedades #

  • liferay-hook.xml #

Começando #

Para começar a desenvolver seu gancho Model Listener, você deve primeiro iniciar seu aplicativo Liferay IDE ou Liferay Developer Studio.

Tanto o Liferay IDE quanto o Liferay Developer Studio são ambientes de desenvolvimento Eclipse personalizados. Eles são surpreendentemente semelhantes e um conjunto de direções deve ser suficiente para ambos os ambientes.

Dentro do seu ambiente de desenvolvimento, execute as etapas a seguir.

  1. No canto superior esquerdo, clique em Arquivo
  2. Passe o mouse sobre Novo
  3. Clique em Projeto de plug-in Liferay

Você vai gerar esta janela.

Por favor, insira as informações como mostrado acima

  • Nome do projeto: User-Model-Listener
  • Selecione Usar local padrão
  • Tipo de construção: Formiga
  • Tipo de plug-in: Gancho

Certifique-se de que seu projeto esteja localizado dentro do diretório Hook do Liferays Plugins SDK. Você precisará selecionar seu SDK e seu Runtime de acordo.

Na perspectiva do Explorador de Pacotes, você verá a seguinte estrutura de diretórios.

Desenvolvimento do Ouvinte #

Agora que você criou seu gancho, você precisará criar sua classe UserModelListener personalizada. Esta classe estenderá a classe BaseModelListener do Liferay.

A classe BaseModelListener do Liferay é uma classe abstrata que implementa a interface ModelListener. Você não deseja implementar a interface ModelListener diretamente, pois isso exigirá que você substitua todos os seus métodos.

Os métodos a seguir são fornecidos a você pela interface ModelListener por meio da classe abstrata BaseModelListener.

  • onAfterAddAssociation
  • onAfterCreate
  • onDepoisRemover
  • onAfterRemoveAssociation
  • onAfterUpdate
  • onBeforeAddAssociation
  • onBeforeCreate
  • onAntesRemover
  • onBeforeRemoveAssociation
  • onBeforeUpdate

Crie sua classe UserModelListener dentro do diretório a seguir. Para criar a classe através da GUI, basta executar os seguintes comandos

  • Clique em Arquivo no canto superior esquerdo

  • Passe o mouse sobre Novo

  • Clique em Classe

    docroot/ WEB-INF/ src/

Insira as informações mostradas abaixo

Dentro de sua classe UserModelListener cole o seguinte código

package com.example.hook;

import com.liferay.mail.service.MailServiceUtil;
import com.liferay.portal.ModelListenerException;
import com.liferay.portal.kernel.mail.MailMessage;
import com.liferay.portal.kernel.util.Validator;
import com.liferay.portal.model.BaseModelListener;
import com.liferay.portal.model.User;


import javax.mail.internet.InternetAddress;

public class UserModelListener extends BaseModelListener<User> {
    private User user = null;

    @Override
    public void onAfterCreate(User user) throws ModelListenerException {
    this.user = user;
    
    if(isValidEmail()) {
        sendEmail("admin@example.com", user.getEmailAddress(), "Welcome!", "Your account is created!");
    } 
    }

    private boolean isValidEmail() {
    return Validator.isNotNull(user.getEmailAddress()) && Validator.isAddress(user.getEmailAddress());
    }
    
    private void sendEmail(String from, String to, String subject, String body) {
    try {
        MailServiceUtil.sendEmail(new MailMessage(new InternetAddress(from), new InternetAddress(to), subject, body, false));
    } catch (Exception e) {
        System.err.print("E-Mail spawned by User Model Listener failed to " + user.getFullName() + " with message " + e.getMessage());
    }
    }
}

Configuração de propriedades #

Para configurar a associação entre nosso ouvinte personalizado e nosso modelo, precisamos fazer alguns ajustes finais. Primeiramente, crie um novo arquivo portal.properties no diretório a seguir.

docroot/
     WEB-INF/
           src/
            + portal.properties

Apenas uma linha precisa ser adicionada ao arquivo.

value.object.listener.com.liferay.portal.model.User = com.example.hook.UserModelListener

Podemos generalizar isso dizendo, para qualquer Model Listener que criamos o valor das propriedades associadas deve estar na forma de

value.object.listener.fully.qualified.model.name = fully.qualified.listener.name

Em outras palavras, se escrevermos uma classe CustomerDLFolderModelListener, empacotada dentro de com.example.code, para o modelo DLFolder teríamos a seguinte propriedade

value.object.listener.com.liferay.portal.model.DLFolder = com.example.code.CustomerDLFolderModelListener

Por fim, localize o arquivo liferay-hook.xml. Na visualização Fonte, escreva o seguinte.

<?xml version="1.0"?>
<!DOCTYPE hook PUBLIC "-//Liferay//DTD Hook 6.2.0//EN" "http://www.liferay.com/dtd/liferay-hook_6_2_0.dtd">

<hook>
    <portal-properties>portal.properties</portal-properties>
</hook>

Explicação #

  1. A linha um é um prólogo opcional que especifica a versão do documento e (em alguns casos) o conjunto de caracteres.
  2. A linha 2 é uma DocType Definition (DTD) formal que define explicitamente quais elementos e atributos são válidos
  3. As linhas 3 e 5 consistem no elemento Hook pai (um dos elementos válidos suportados por este DTD)
  4. A linha 4 substitui e estende o arquivo portal.properties em ${liferay.home}

Para ver quais outros elementos podem ser usados ​​neste arquivo XML, você pode referenciar a URL dentro da DocType Definition. Isso é padrão para todos os arquivos XML e SGML com um DTD. Outro exemplo de um arquivo Liferay XML com um DTD é o service.xml (implementação do ORM do Liferay com base no Hibernate).

Construir e implantar #

Construir e implantar ganchos é um processo simples. O desenvolvimento do Liferay Plugin suporta automação de construção e dependência com

  • Formiga
  • Hera
  • Especialista
  • Grade

Em nosso exemplo, usamos Ant para automação de compilação. O arquivo build.xml contém os comandos de compilação (conhecidos como destinos no Ant). Para construir seu gancho simplesmente execute os seguintes comandos.

  1. Localize seu arquivo build.xml
  2. Em seu IDE, arraste o arquivo build.xml para a perspectiva Ant
  3. Expanda o arquivo e execute o destino all

Com seu Hook construído com sucesso, agora é hora de iniciar seu portal e implementá-lo. Para iniciar seu servidor e implementar o Hook, localize a perspectiva Server.

  1. Clique com o botão direito do mouse em seu servidor e clique em Adicionar ou remover
  2. Localize Usuário-ouvinte-gancho na seleção Disponível
  3. Depois de destacado, clique no botão Adicionar e clique em OK
  4. Clique no botão Play na perspectiva Server
