Sua primeira extensão #

As extensões C são compostas por duas partes gerais:

  1. O próprio Código C.
  2. O arquivo de configuração da extensão.

Para começar com sua primeira extensão coloque o seguinte em um arquivo chamado extconf.rb:

require 'mkmf'

create_makefile('hello_c')

Algumas coisas a destacar:

Primeiro, o nome hello_c é como a saída da sua extensão compilada será nomeada. Será o que você usa em conjunto com require.

Em segundo lugar, o arquivo extconf.rb pode realmente ser nomeado qualquer coisa, é apenas tradicionalmente o que é usado para construir gems que possuem código nativo, o arquivo que realmente irá compilar a extensão é o Makefile gerado ao executar ruby extconf.rb . O Makefile padrão que é gerado compila todos os arquivos .c no diretório atual.

Coloque o seguinte em um arquivo chamado hello.c e execute ruby extconf.rb && make

#include <stdio.h>
#include "ruby.h"

VALUE world(VALUE self) {
  printf("Hello World!\n");
  return Qnil;
}

// The initialization method for this module
void Init_hello_c() {
  VALUE HelloC = rb_define_module("HelloC");
  rb_define_singleton_method(HelloC, "world", world, 0);
}

Um detalhamento do código:

O nome Init_hello_c deve corresponder ao nome definido em seu arquivo extconf.rb, caso contrário, ao carregar dinamicamente a extensão, o Ruby não poderá encontrar o símbolo para inicializar sua extensão.

A chamada para rb_define_module está criando um módulo Ruby chamado HelloC no qual vamos colocar o namespace em nossas funções C.

Finalmente, a chamada para rb_define_singleton_method torna um método de nível de módulo vinculado diretamente ao módulo HelloC que podemos invocar do ruby ​​com HelloC.world.

Depois de compilar a extensão com a chamada para make, podemos executar o código em nossa extensão C.

Acenda um console!

irb(main):001:0> require './hello_c'
=> true
irb(main):002:0> HelloC.world
Hello World!
=> nil

Trabalhando com estruturas C #

Para poder trabalhar com estruturas C como objetos Ruby, você precisa envolvê-los com chamadas para Data_Wrap_Struct e Data_Get_Struct.

Data_Wrap_Struct envolve uma estrutura de dados C em um objeto Ruby. Leva um ponteiro para sua estrutura de dados, juntamente com alguns ponteiros para funções de retorno de chamada, e retorna um VALOR. A macro Data_Get_Struct pega esse VALUE e retorna um ponteiro para sua estrutura de dados C.

Aqui está um exemplo simples:

#include <stdio.h>
#include <ruby.h>

typedef struct example_struct {
  char *name;
} example_struct;

void example_struct_free(example_struct * self) {
  if (self->name != NULL) {
    free(self->name);
  }
  ruby_xfree(self);
}

static VALUE rb_example_struct_alloc(VALUE klass) {
  return Data_Wrap_Struct(klass, NULL, example_struct_free, ruby_xmalloc(sizeof(example_struct)));
}

static VALUE rb_example_struct_init(VALUE self, VALUE name) {
  example_struct* p;

  Check_Type(name, T_STRING);

  Data_Get_Struct(self, example_struct, p);
  p->name = (char *)malloc(RSTRING_LEN(name) + 1);
  memcpy(p->name, StringValuePtr(name), RSTRING_LEN(name) + 1);

  return self;
}

static VALUE rb_example_struct_name(VALUE self) {
  example_struct* p;
  Data_Get_Struct(self, example_struct, p);

  printf("%s\n", p->name);

  return Qnil;
}

void Init_example()
{
  VALUE mExample = rb_define_module("Example");
  VALUE cStruct = rb_define_class_under(mExample, "Struct", rb_cObject);

  rb_define_alloc_func(cStruct, rb_example_struct_alloc);
  rb_define_method(cStruct, "initialize", rb_example_struct_init, 1);
  rb_define_method(cStruct, "name", rb_example_struct_name, 0);
}

E o extconf.rb:

require 'mkmf'

create_makefile('example')

Após compilar a extensão:

irb(main):001:0> require './example'
=> true
irb(main):002:0> test_struct = Example::Struct.new("Test Struct")
=> #<Example::Struct:0x007fc741965068>
irb(main):003:0> test_struct.name
Test Struct
=> nil

Escrevendo C Inline - RubyInLine #

RubyInline é um framework que permite incorporar outras linguagens dentro do seu código Ruby. Ele define o método inline Module#, que retorna um objeto construtor. Você passa ao construtor uma string contendo código escrito em uma linguagem diferente de Ruby, e o construtor a transforma em algo que você pode chamar de Ruby.

Quando recebe o código C ou C++ (as duas linguagens suportadas na instalação padrão do RubyInline), os objetos do construtor gravam uma pequena extensão no disco, compilam e carregam. Você não precisa lidar com a compilação sozinho, mas pode ver o código gerado e as extensões compiladas no subdiretório .ruby_inline do seu diretório inicial.

Incorpore o código C diretamente no seu programa Ruby:

  • RubyInline (disponível como a gem rubyinline) cria uma extensão automaticamente

RubyInline não funcionará de dentro do irb

#!/usr/bin/ruby -w
    # copy.rb
    require 'rubygems'
    require 'inline'

    class Copier
    inline do |builder|
      builder.c <<END
    void copy_file(const char *source, const char *dest)
    {
      FILE *source_f = fopen(source, "r");
      if (!source_f)
      {
        rb_raise(rb_eIOError, "Could not open source : '%s'", source);
      }

      FILE *dest_f = fopen(dest, "w+");
      if (!dest_f)
      {
        rb_raise(rb_eIOError, "Could not open destination : '%s'", dest);
      }

      char buffer[1024];

      int nread = fread(buffer, 1, 1024, source_f);
      while (nread > 0)
      {
        fwrite(buffer, 1, nread, dest_f);
        nread = fread(buffer, 1, 1024, source_f);
      }
    }
    END
     end
    end

A função C copy_file agora existe como um método de instância de Copier:

open('source.txt', 'w') { |f| f << 'Some text.' }
Copier.new.copy_file('source.txt', 'dest.txt')
puts open('dest.txt') { |f| f.read }
