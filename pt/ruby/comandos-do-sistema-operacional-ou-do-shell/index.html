Há muitas maneiras de interagir com o sistema operacional. De dentro do Ruby você pode executar comandos shell/sistema ou subprocessos.

Executivo: O Exec é muito limitado em funcionalidade e, quando executado, sairá do programa Ruby e executará o comando.

O Comando do Sistema: O comando System é executado em um subshell em vez de substituir o processo atual e retorna true ou nill. O comando do sistema é, como backticks, uma operação de bloqueio onde o aplicativo principal espera até que o resultado da operação do sistema seja concluído. Aqui, a operação principal nunca precisa se preocupar em capturar uma exceção gerada pelo processo filho.

A saída da função do sistema sempre será verdadeira ou nula dependendo se o script foi executado sem erros ou não. Portanto, todo erro durante a execução do script não será passado para nossa aplicação. A operação principal nunca precisa se preocupar em capturar uma exceção gerada pelo processo filho. Nesse caso, a saída é nula porque o processo filho gerou uma exceção. Esta é uma operação de bloqueio onde o programa Ruby irá esperar até que a operação do comando seja concluída antes de continuar. A operação do sistema usa fork para bifurcar o processo atual e, em seguida, executar a operação fornecida usando exec.

Os acentos graves (`): O caractere backtick geralmente está localizado sob a tecla Escape no teclado. Backticks é executado em um sub-shell em vez de substituir o processo atual e retorna o resultado do comando. Aqui podemos obter a saída do comando, mas o programa travará quando uma exceção for gerada. Se houver uma exceção no subprocesso, essa exceção será fornecida ao processo principal e o processo principal poderá ser encerrado se a exceção não for tratada. Esta é uma operação de bloqueio onde o programa Ruby irá esperar até que a operação do comando seja concluída antes de continuar. A operação do sistema usa fork para bifurcar o processo atual e, em seguida, executar a operação fornecida usando exec.

** IO.popen: ** IO.popen é executado em um subprocesso. Aqui, a entrada padrão do subprocesso e a saída padrão são conectadas ao objeto IO.

Popen3: Popen3 permite acessar a entrada padrão, saída padrão e erro padrão. A entrada e a saída padrão do subprocesso serão retornadas em objetos de E/S.

$? (o mesmo que $CHILD_STATUS) Pode ser usado com as operações backticks, system() ou %x{} e fornecerá o status do último comando executado pelo sistema. Isso pode ser útil para acessar as propriedades exitstatus e pid.

$?.exitstatus

Maneiras recomendadas de executar código shell em Ruby: #

Open3.popen3 ou Open3.capture3: Na verdade, o Open3 usa apenas o comando spawn do Ruby, mas oferece uma API muito melhor.

Abrir3.popen3

Popen3 é executado em um subprocesso e retorna stdin, stdout, stderr e wait_thr.

require 'open3'
stdin, stdout, stderr, wait_thr = Open3.popen3("sleep 5s && ls")
puts "#{stdout.read} #{stderr.read} #{wait_thr.value.exitstatus}"

ou

require 'open3'
cmd = 'git push heroku master'
Open3.popen3(cmd) do |stdin, stdout, stderr, wait_thr|
  puts "stdout is:" + stdout.read
  puts "stderr is:" + stderr.read
end

emitirá: stdout é: stderr is:fatal: Não é um repositório git (ou qualquer um dos diretórios pai): .git

ou

require 'open3'
cmd = 'ping www.google.com'
Open3.popen3(cmd) do |stdin, stdout, stderr, wait_thr|
  while line = stdout.gets
    puts line
  end
end

emitirá:

Ping em www.google.com [216.58.223.36] com 32 bytes de dados: Resposta de 216.58.223.36: bytes=32 time=16ms TTL=54 Resposta de 216.58.223.36: bytes=32 time=10ms TTL=54 Resposta de 216.58.223.36: bytes=32 time=21ms TTL=54 Resposta de 216.58.223.36: bytes=32 time=29ms TTL=54 Estatísticas de ping para 216.58.223.36: Packets: Sent = 4, Received = 4, Lost = 0 (0% loss),
Tempos aproximados de ida e volta em milissegundos: Minimum = 10ms, Maximum = 29ms, Average = 19ms

Abrir3.captura3:

require 'open3'

stdout, stderr, status = Open3.capture3('my_funky_command', 'and', 'some', 'argumants')
if status.success?
  # command completed successfully, do some more stuff
else
  raise "An error occured"
end

ou

Open3.capture3('/some/binary with some args')

Não recomendado, porém, devido à sobrecarga adicional e ao potencial de injeções de shell.

Se o comando lê de stdin e você deseja alimentá-lo com alguns dados:

Open3.capture3('my_funky_command', stdin_data: 'read from stdin')

Execute o comando com um diretório de trabalho diferente, usando chdir:

Open3.capture3('my_funky_command', chdir: '/some/directory')

Maneiras clássicas de executar código shell em Ruby: #

Executivo:

exec 'echo "hello world"'

ou

exec ('echo "hello world"')

O Comando do Sistema:

system 'echo "hello world"'

Irá produzir “hello world” na janela de comando.

ou

system ('echo "hello world"')

O comando do sistema pode retornar um verdadeiro se o comando foi bem-sucedido ou nulo quando não.

result = system 'echo "hello world"'
puts result  # will return a true in the command window

Os acentos graves (`):

echo "olá mundo" Irá produzir “hello world” na janela de comando.

Você também pode pegar o resultado.

result = `echo "hello world"`  
puts "We always code a " + result
