Argumentos de palavra-chave

  • Usando argumentos de palavras-chave arbitrárias com o operador splat

  • Argumentos de palavra-chave obrigatórios

  • Usando argumentos de palavras-chave arbitrárias com o operador splat
  • Usando argumentos de palavras-chave
  • Argumentos de palavra-chave obrigatórios

Argumentos de palavra-chave foram introduzidos no Ruby 2.0 e aprimorados no Ruby 2.1 com a adição de argumentos de palavra-chave required.

Um método simples com um argumento de palavra-chave se parece com o seguinte:

def say(message: "Hello World")
  puts message
end

say
# => "Hello World"

say message: "Today is Monday"
# => "Today is Monday"

Como lembrete, o mesmo método sem argumento de palavra-chave teria sido:

def say(message = "Hello World")
  puts message
end

say
# => "Hello World"

say "Today is Monday"
# => "Today is Monday"

Você pode simular o argumento de palavra-chave em versões anteriores do Ruby usando um parâmetro Hash. Esta ainda é uma prática muito comum, especialmente em bibliotecas que oferecem compatibilidade com versões anteriores ao Ruby:

def say(options = {})
  message = options.fetch(:message, "Hello World")
  puts 
end

say
# => "Hello World"

say message: "Today is Monday"
# => "Today is Monday"

Usando argumentos de palavras-chave arbitrárias com o operador splat #

Você pode definir um método para aceitar um número arbitrário de argumentos de palavras-chave usando o operador double splat (**):

def say(**args)
  puts args
end

say foo: "1", bar: "2"
# {:foo=>"1", :bar=>"2"}

Os argumentos são capturados em um Hash. Você pode manipular o Hash, por exemplo, para extrair os argumentos desejados.

def say(**args)
  puts args[:message] || "Message not found"
end

say foo: "1", bar: "2", message: "Hello World"
# Hello World

say foo: "1", bar: "2"
# Message not found

Usar um operador splat com argumentos de palavra-chave impedirá a validação de argumento de palavra-chave, o método nunca gerará um ArgumentError no caso de palavra-chave desconhecida.

Quanto ao operador splat padrão, você pode reconverter um Hash em argumentos de palavra-chave para um método:

def say(message: nil, before: "<p>", after: "</p>")
  puts "#{before}#{message}#{after}"
end

args = { message: "Hello World", after: "</p><hr>" }
say(**args)
# <p>Hello World</p><hr>

args = { message: "Hello World", foo: "1" }
say(**args)
# => ArgumentError: unknown keyword: foo

Isso geralmente é usado quando você precisa manipular argumentos de entrada e passá-los para um método subjacente:

def inner(foo:, bar:)
  puts foo, bar
end

def outer(something, foo: nil, bar: nil, baz: nil)
  puts something
  params = {}
  params[:foo] = foo || "Default foo"
  params[:bar] = bar || "Default bar"
  inner(**params)
end

outer "Hello:", foo: "Custom foo"
# Hello:
# Custom foo
# Default bar

Usando argumentos de palavras-chave #

Você define um argumento de palavra-chave em um método especificando o nome na definição do método:

def say(message: "Hello World")
  puts message
end

say
# => "Hello World"

say message: "Today is Monday"
# => "Today is Monday"

Você pode definir vários argumentos de palavra-chave, a ordem de definição é irrelevante:

def say(message: "Hello World", before: "<p>", after: "</p>")
  puts "#{before}#{message}#{after}"
end

say
# => "<p>Hello World</p>"

say message: "Today is Monday"
# => "<p>Today is Monday</p>"

say after: "</p><hr>", message: "Today is Monday"
# => "<p>Today is Monday</p><hr>"

Os argumentos de palavra-chave podem ser misturados com argumentos posicionais:

def say(message, before: "<p>", after: "</p>")
  puts "#{before}#{message}#{after}"
end

say "Hello World", before: "<span>", after: "</span>"
# => "<span>Hello World</span>"

Misturar argumentos de palavras-chave com argumentos posicionais era uma abordagem muito comum antes do Ruby 2.1, porque não era possível definir [argumentos de palavras-chave obrigatórios](https://www.wikiod.com/pt/ruby/argumentos-de-palavra-chave#Argumentos de palavra-chave obrigatórios keyword-arguments#t=201608060918529843522).

Além disso, em Ruby < 2.0, era muito comum adicionar um Hash no final de uma definição de método para usar como argumentos opcionais. A sintaxe é muito semelhante aos argumentos de palavras-chave, a ponto de argumentos opcionais via Hash serem compatíveis com os argumentos de palavras-chave do Ruby 2.

def say(message, options = {})
  before = option.fetch(:before, "<p>")
  after  = option.fetch(:after, "</p>")
  puts "#{before}#{message}#{after}"
end

# The method call is syntactically equivalent to the keyword argument one
say "Hello World", before: "<span>", after: "</span>"
# => "<span>Hello World</span>"

Observe que tentar passar um argumento de palavra-chave não definido resultará em um erro:

def say(message: "Hello World")
  puts message
end

say foo: "Hello"
# => ArgumentError: unknown keyword: foo

Argumentos de palavra-chave obrigatórios #

Argumentos de palavras-chave obrigatórios foram introduzidos no Ruby 2.1, como uma melhoria nos argumentos de palavras-chave.

Para definir um argumento de palavra-chave conforme necessário, simplesmente declare o argumento sem um valor padrão.

def say(message:)
  puts message
end

say
# => ArgumentError: missing keyword: message

say message: "Hello World"
# => "Hello World"

Você também pode misturar argumentos de palavras-chave obrigatórios e não obrigatórios:

def say(before: "<p>", message:, after: "</p>")
  puts "#{before}#{message}#{after}"
end

say
# => ArgumentError: missing keyword: message

say message: "Hello World"
# => "<p>Hello World</p>"

say message: "Hello World", before: "<span>", after: "</span>"
# => "<span>Hello World</span>"
