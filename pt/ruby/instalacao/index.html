Gemas #

Neste exemplo, usaremos ’nokogiri’ como uma gem de exemplo. ’nokogiri’ pode mais tarde ser substituído por qualquer outro nome de gem.

Para trabalhar com gems, usamos uma ferramenta de linha de comando chamada gem seguida por uma opção como install ou update e os nomes das gems que queremos instalar, mas isso não é tudo.

Instale gemas:

$> gem install nokogiri

Mas isso não é a única coisa que precisamos. Também podemos especificar a versão, a fonte da qual instalar ou procurar por gems. Vamos começar com alguns casos de uso básicos (UC) e mais tarde você pode postar uma solicitação de atualização.

Listando todas as gems instaladas:

$> gem list

Desinstalando gemas:

$> gem uninstall nokogiri

Se tivermos mais versões da gem nokogiri, seremos solicitados a especificar qual queremos desinstalar. Obteremos uma lista ordenada e numerada e apenas escrevemos o número.

Atualizando gemas

$> gem update nokogiri

ou se queremos atualizá-los todos

$> gem update

Comman gem tem muito mais usos e opções a serem exploradas. Para mais, consulte a documentação oficial. Se algo não estiver claro poste um pedido e eu vou adicioná-lo.

Instalando Ruby macOS #

Então a boa notícia é que a Apple gentilmente inclui um interpretador Ruby. Infelizmente, tende a não ser uma versão recente:

$ /usr/bin/ruby -v
ruby 2.0.0p648 (2015-12-16 revision 53162) [universal.x86_64-darwin16]

Se você tiver o Homebrew instalado, poderá obter o Ruby mais recente com:

$ brew install ruby

$ /usr/local/bin/ruby -v
ruby 2.4.1p111 (2017-03-22 revision 58053) [x86_64-darwin16]

(É provável que você veja uma versão mais recente se tentar isso.)

Para pegar a versão preparada sem usar o caminho completo, você deve adicionar /usr/local/bin ao início de sua variável de ambiente $PATH:

export PATH=/usr/local/bin:$PATH

Adicionar essa linha ao ~/.bash_profile garante que você obterá esta versão após reiniciar seu sistema:

$ type ruby
ruby is /usr/local/bin/ruby

O Homebrew instalará o gem para instalar Gems. Também é possível [compilar a partir da fonte](https://www.wikiod.com/pt/ruby/instalacao#Linux - Compilando a partir do código-fonte) se você precisar disso. Homebrew também inclui essa opção:

$ brew install ruby --build-from-source

Linux - Compilando a partir da fonte #

`Dessa forma, você obterá o mais novo rubi, mas ele tem suas desvantagens. Fazendo assim, ruby ​​não será gerenciado por nenhum aplicativo.

!! Lembre-se de alterar a versão para que corresponda ao seu !!

  1. você precisa baixar um tarball, encontre um link em um site oficial (https://www.ruby-lang.org/en/downloads/)

  2. Extraia o tarball

  3. Instalar

    $> wget https://cache.ruby-lang.org/pub/ruby/2.3/ruby-2.3.3.tar.gz $> tar -xvzf ruby-2.3.3.tar.gz $> cd ruby-2.3.3 $> ./configure $> make $> sudo make install

Isso instalará o ruby ​​em /usr/local. Se você não estiver satisfeito com este local, você pode passar um argumento para o ./configure --prefix=DIR onde DIR é o diretório no qual você deseja instalar o ruby.

Linux—Instalação usando um gerenciador de pacotes #

Provavelmente a escolha mais fácil, mas cuidado, a versão nem sempre é a mais nova. Basta abrir o terminal e digitar (dependendo da sua distribuição)

no Debian ou Ubuntu usando apt

$> sudo apt install ruby

no CentOS, openSUSE ou Fedora

$> sudo yum install ruby

Você pode usar a opção -y para não ser solicitado a concordar com a instalação, mas na minha opinião é uma boa prática sempre verificar o que o gerenciador de pacotes está tentando instalar.

Windows - Instalação usando o instalador #

Provavelmente a maneira mais fácil de configurar o ruby ​​no Windows é ir para http://rubyinstaller.org/ e de lá baixar um executável que você instalará.

Você não precisa definir quase nada, mas haverá uma janela importante. Ele terá uma caixa de seleção dizendo Adicionar executável ruby ​​ao seu PATH. Confirme se está marcado, caso contrário, verifique ou então você não poderá executar o ruby ​​e terá que definir a variável PATH por conta própria.

Em seguida, basta ir em seguida até que ele seja instalado e pronto.

Linux - solução de problemas de instalação de gem #

Primeira UC no exemplo Gems $> gem install nokogiri pode ter problemas ao instalar gems porque não temos as permissões para isso. Isso pode ser resolvido em mais de uma maneira.

Primeira solução de UC a:

Você pode usar sudo. Isso instalará a gem para todos os usuários. Este método deve ser desaprovado. Isso deve ser usado apenas com a gem que você sabe que será utilizável por todos os usuários. Normalmente na vida real você não quer que algum usuário tenha acesso ao sudo.

$> sudo gem install nokogiri

Primeira solução de UC b

Você pode usar a opção --user-install que instala as gems em sua pasta gem de usuários (geralmente em ~/.gem)

&> gem install nokogiri --user-install

Primeira solução de UC c

Você pode definir GEM_HOME e GEM_PATH que então fará o comando gem install instalar todas as gems em uma pasta que você especificar. Eu posso te dar um exemplo disso (da maneira usual)

  • Primeiro você precisa abrir o .bashrc. Use o nano ou seu editor de texto favorito.

    $> nano ~/.bashrc

  • Então, no final deste arquivo, escreva

    export GEM_HOME=$HOME/.gem export GEM_PATH=$HOME/.gem

  • Agora você precisará reiniciar o terminal ou escrever . ~/.bashrc para recarregar a configuração. Isso permitirá que você use gem isntall nokogiri e instalará essas gems na pasta que você especificou.
