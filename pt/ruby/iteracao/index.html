Cada #

Ruby tem muitos tipos de enumeradores, mas o primeiro e mais simples tipo de enumerador para começar é each. Vamos imprimir even ou odd para cada número entre 1 e 10 para mostrar como each funciona.

Basicamente, existem duas maneiras de passar os chamados blocos. Um bloco é um pedaço de código que está sendo passado que será executado pelo método que é chamado. O método each recebe um block que ele chama para cada elemento da coleção de objetos em que foi chamado.

Existem duas maneiras de passar um bloco para um método:

Método 1: Em linha #

(1..10).each { |i| puts i.even? ? 'even' : 'odd' }

Esta é uma maneira muito compactada e ruby de resolver isso. Vamos quebrar isso peça por peça.

  1. (1..10) é um intervalo de 1 a 10 inclusive. Se quiséssemos que fosse de 1 a 10 exclusivo, escreveríamos (1...10).
  2. .each é um enumerador que enumera sobre cada elemento no objeto em que está atuando. Nesse caso, ele atua em cada número no intervalo.
  3. { |i| coloca i.even? ? 'even' : 'odd' } é o bloco para a instrução each, que pode ser dividida ainda mais.
    1. |i| this means that each element in the range is represented within the block by the identifier i.
    2. puts is an output method in Ruby that has an automatic line break after each time it prints. (We can use print if we don’t want the automatic line break)
    3. i.even? checks if i is even. We could have also used i % 2 == 0; however, it is preferable to use built in methods.
    4. ? "even" : "odd" this is ruby’s ternary operator. The way a ternary operator is constructed is expression ? a : b. This is short for
    if expression
  a
else
  b
end

Para código com mais de uma linha, o block deve ser passado como um multiline block.

Método 2: Multilinha #

(1..10).each do |i|
  if i.even?
    puts 'even'
  else
    puts 'odd'
  end
end

Em um bloco multilinha o do substitui o colchete de abertura e o end substitui o colchete de fechamento do estilo inline.

Ruby também suporta reverse_each. Ele irá iterar a matriz para trás.

@arr = [1,2,3,4]
puts @arr.inspect # output is [1,2,3,4]

print "Reversed array elements["
@arr.reverse_each do |val|
        print " #{val} " # output is 4 3 2 1
end
print "]\n"

Iterando objetos complexos #

Matrizes

Você pode iterar sobre matrizes aninhadas:

[[1, 2], [3, 4]].each { |(a, b)| p "a: #{ a }", "b: #{ b }" }

A seguinte sintaxe também é permitida:

[[1, 2], [3, 4]].each { |a, b| "a: #{ a }", "b: #{ b }" }

Vai produzir:

"a: 1"
"b: 2"
"a: 3"
"b: 4"

Haxe

Você pode iterar sobre pares de valores-chave:

{a: 1, b: 2, c: 3}.each { |pair| p "pair: #{ pair }" }

Vai produzir:

"pair: [:a, 1]"
"pair: [:b, 2]"
"pair: [:c, 3]"

Você pode iterar sobre chaves e valores simultaneamente:

{a: 1, b: 2, c: 3}.each { |(k, v)| p "k: #{ k }", "v: #{ k }" }

Vai produzir:

"k: a"
"v: a"
"k: b"
"v: b"
"k: c"
"v: c"

Para iterador #

Isso itera de 4 a 13 (inclusive).

for i in 4..13
    puts "this is #{i}.th number"
end

Também podemos iterar sobre arrays usando for

names = ['Siva', 'Charan', 'Naresh', 'Manish']

for name in names
    puts name
end

Iteração com índice #

Às vezes, você deseja saber a posição (índice) do elemento atual enquanto itera sobre um enumerador. Para isso, Ruby fornece o método with_index. Pode ser aplicado a todos os enumeradores. Basicamente, adicionando with_index a uma enumeração, você pode enumerar essa enumeração. O índice é passado para um bloco como o segundo argumento.

[2,3,4].map.with_index { |e, i| puts "Element of array number #{i} => #{e}" }
#Element of array number 0 => 2
#Element of array number 1 => 3
#Element of array number 2 => 4
#=> [nil, nil, nil]

with_index tem um argumento opcional – o primeiro índice que é 0 por padrão:

[2,3,4].map.with_index(1) { |e, i| puts "Element of array number #{i} => #{e}" }
#Element of array number 1 => 2
#Element of array number 2 => 3
#Element of array number 3 => 4
#=> [nil, nil, nil]

Existe um método específico each_with_index. A única diferença entre ele e each.with_index é que você não pode passar um argumento para isso, então o primeiro índice é 0 o tempo todo.

[2,3,4].each_with_index { |e, i| puts "Element of array number #{i} => #{e}" }
#Element of array number 0 => 2
#Element of array number 1 => 3
#Element of array number 2 => 4
#=> [2, 3, 4]

Mapa #

Retorna o objeto alterado, mas o objeto original permanece como estava. Por exemplo:

arr = [1, 2, 3]
arr.map { |i| i + 1 } # => [2, 3, 4]
arr # => [1, 2, 3]

map! altera o objeto original:

arr = [1, 2, 3]
arr.map! { |i| i + 1 } # => [2, 3, 4]
arr # => [2, 3, 4]

Nota: você também pode usar collect para fazer a mesma coisa.
