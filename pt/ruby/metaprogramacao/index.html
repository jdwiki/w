Metaprogramação pode ser descrita de duas maneiras:

“Programas de computador que escrevem ou manipulam outros programas (ou a si mesmos) como seus dados, ou que fazem parte do trabalho em tempo de compilação que de outra forma seria feito em tempo de execução”.

Simplificando: Metaprogramação é escrever código que escreve código durante o tempo de execução para facilitar sua vida.

Implementando “com” usando avaliação de instância #

Muitas linguagens apresentam uma instrução with que permite aos programadores omitir o receptor de chamadas de método.

with pode ser facilmente emulado em Ruby usando instance_eval:

def with(object, &block)
  object.instance_eval &block
end

O método with pode ser usado para executar métodos em objetos perfeitamente:

hash = Hash.new

with hash do
  store :key, :value
  has_key? :key       # => true
  values              # => [:value]
end

método send() #

send() é usado para passar mensagem para object. send() é um método de instância da classe Object. O primeiro argumento em send() é a mensagem que você está enviando para o objeto - ou seja, o nome de um método. Pode ser string ou symbol, mas símbolos são os preferidos. Então os argumentos que precisam passar no método, esses serão os argumentos restantes em send().

class Hello
  def hello(*args)
    puts 'Hello ' + args.join(' ')
  end
end
h = Hello.new
h.send :hello, 'gentle', 'readers'   #=> "Hello gentle readers"
# h.send(:hello, 'gentle', 'readers') #=> Here :hello is method and rest are the arguments to method.

Aqui está o exemplo mais descritivo #

class Account
  attr_accessor :name, :email, :notes, :address

  def assign_values(values)
    values.each_key do |k, v|
      # How send method would look a like
      # self.name = value[k]
      self.send("#{k}=", values[k])
    end
  end
end

user_info = {
  name: 'Matt',
  email: 'test@gms.com',
  address: '132 random st.',
  notes: "annoying customer"
}

account = Account.new
If attributes gets increase then we would messup the code
#--------- Bad way --------------
account.name = user_info[:name]
account.address = user_info[:address]
account.email = user_info[:email]
account.notes = user_info[:notes]

# --------- Meta Programing way --------------
account.assign_values(user_info) # With single line we can assign n number of attributes

puts account.inspect

Nota: send() em si não é mais recomendado. Use __send__() que tem o poder de chamar métodos privados, ou (recomendado) public_send()

Definindo métodos dinamicamente #

Com Ruby você pode modificar a estrutura do programa em tempo de execução. Uma maneira de fazer isso é definindo métodos dinamicamente usando o método method_missing.

Digamos que queremos testar se um número é maior que outro número com a sintaxe 777.is_greater_than_123?.

# open Numeric class
class Numeric
  # override `method_missing`
  def method_missing(method_name,*args)
    # test if the method_name matches the syntax we want
    if method_name.to_s.match /^is_greater_than_(\d+)\?$/
      # capture the number in the method_name
      the_other_number = $1.to_i
      # return whether the number is greater than the other number or not
      self > the_other_number
    else
      # if the method_name doesn't match what we want, let the previous definition of `method_missing` handle it
      super
    end
  end
end

Uma coisa importante a ser lembrada ao usar method_missing que também se deve substituir o método respond_to?:

class Numeric
   def respond_to?(method_name, include_all = false) 
     method_name.to_s.match(/^is_greater_than_(\d+)\?$/) || super
   end
end

Esquecer de fazer isso leva a uma situação inconsistente, quando você pode chamar 600.is_greater_than_123 com sucesso, mas 600.respond_to(:is_greater_than_123) retorna false.

Definindo métodos em instâncias #

Em Ruby você pode adicionar métodos a instâncias existentes de qualquer classe. Isso permite adicionar comportamento a uma instância de uma classe sem alterar o comportamento do restante das instâncias dessa classe.

class Example
  def method1(foo)
    puts foo
  end
end

#defines method2 on object exp
exp = Example.new
exp.define_method(:method2) {puts "Method2"}

#with method parameters
exp.define_method(:method3) {|name| puts name}
