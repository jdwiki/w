Sintaxe #

  • q = Fila.novo
  • q.push objeto
  • q « objeto # igual a #push
  • q.pop #=> objeto

Vários trabalhadores em uma pia #

Queremos coletar dados criados por vários Trabalhadores.

Primeiro criamos uma Fila:

sink = Queue.new

Em seguida, 16 trabalhadores gerando um número aleatório e empurrando-o para a pia:

(1..16).to_a.map do
  Thread.new do
    sink << rand(1..100)
  end
end.map(&:join)

E para obter os dados, converta um Queue em um Array:

data = [].tap { |a| a << sink.pop until sink.empty? }

Convertendo uma fila em um array #

q = Queue.new
q << 1
q << 2

a = Array.new
a << q.pop until q.empty?

Ou um um forro:

[].tap { |array| array < queue.pop until queue.empty? }

Uma Origem Vários Trabalhadores #

Queremos processar dados em paralelo.

Vamos preencher a fonte com alguns dados:

source = Queue.new
data = (1..100)
data.each { |e| source << e }

Em seguida, crie alguns trabalhadores para processar dados:

(1..16).to_a.map do
  Thread.new do
    until source.empty?
      item = source.pop
      sleep 0.5
      puts "Processed: #{item}"
    end
  end
end.map(&:join)

Uma fonte - Pipeline of Work - One Sink #

Queremos processar dados em paralelo e empurrá-los para baixo na linha para serem processados ​​por outros trabalhadores.

Como os Workers consomem e produzem dados, temos que criar duas filas:

first_input_source = Queue.new
first_output_sink  = Queue.new
100.times { |i| first_input_source << i }

A primeira onda de trabalhadores lê um item de first_input_source, processa o item e grava os resultados em first_output_sink:

(1..16).to_a.map do
  Thread.new do
    loop do
      item = first_input_source.pop
      first_output_source << item ** 2
      first_output_source << item ** 3
    end
  end
end

A segunda onda de trabalhadores usa first_output_sink como sua fonte de entrada e lê, o processo grava em outro coletor de saída:

second_input_source = first_output_sink
second_output_sink  = Queue.new

(1..32).to_a.map do
  Thread.new do
    loop do
      item = second_input_source.pop
      second_output_sink << item * 2
      second_output_sink << item * 3
    end
  end
end

Agora second_output_sink é o coletor, vamos convertê-lo em um array:

sleep 5 # workaround in place of synchronization
sink = second_output_sink
[].tap { |a| a << sink.pop until sink.empty? }

Empurrando dados para uma fila - #push #

q = Queue.new
q << "any object including another queue"
# or
q.push :data
  • Não há marca d’água alta, as filas podem crescer infinitamente.
  • #push nunca bloqueia

Extraindo dados de uma fila - #pop #

q = Queue.new
q << :data
q.pop #=> :data
  • #pop bloqueará até que haja alguns dados disponíveis.
  • #pop pode ser usado para sincronização.

Sincronização - Depois de um ponto no tempo #

syncer = Queue.new

a = Thread.new do
  syncer.pop
  puts "this happens at end"
end

b = Thread.new do
  puts "this happens first"
  STDOUT.flush
  syncer << :ok
end

[a, b].map(&:join)

Mesclando duas filas #

  • Para evitar bloqueios infinitos, a leitura das filas não deve acontecer no momento em que a mesclagem de threads está acontecendo.
  • Para evitar sincronização ou espera infinita por uma das filas enquanto outra tem dados, a leitura das filas não deve acontecer no mesmo thread.

Vamos começar definindo e preenchendo duas filas:

q1 = Queue.new
q2 = Queue.new
(1..100).each { |e| q1 << e }
(101..200).each { |e| q2 << e }

Devemos criar outra fila e enviar dados de outros threads para ela:

merged = Queue.new

[q1, q2].map do |q|
  Thread.new do
    loop do
      merged << q.pop
    end
  end
end

Se você sabe que pode consumir completamente as duas filas (a velocidade de consumo é maior que a produção, você não ficará sem RAM), existe uma abordagem mais simples:

merged = Queue.new
merged << q1.pop until q1.empty?
merged << q2.pop until q2.empty?
