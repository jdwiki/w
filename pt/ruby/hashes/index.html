Um Hash é uma coleção semelhante a um dicionário de chaves exclusivas e seus valores. Também chamados de arrays associativos, eles são semelhantes aos Arrays, mas onde um Array usa inteiros como seu índice, um Hash permite que você use qualquer tipo de objeto. Você recupera ou cria uma nova entrada em um Hash consultando sua chave.

Sintaxe #

  • { first_name: “Noel”, second_name: “Edmonds” }

  • { :first_name => “Noel”, :second_name => “Edmonds” }

  • { “Primeiro Nome” => “Noel”, “Segundo Nome” => “Edmonds” }

  • { first_key => first_value, second_key => second_value }

Hashes em Ruby mapeiam chaves para valores usando uma tabela de hash.

Qualquer objeto hashable pode ser usado como chaves. No entanto, é muito comum usar um Symbol, pois geralmente é mais eficiente em várias versões do Ruby, devido à alocação reduzida de objetos.

{ key1: "foo", key2: "baz"  }

Criando um hash #

Um hash em Ruby é um objeto que implementa uma tabela de hash, mapeando chaves para valores. Ruby suporta uma sintaxe literal específica para definir hashes usando {}:

my_hash = {}  # an empty hash
grades = { 'Mark' => 15, 'Jimmy' => 10, 'Jack' => 10 }

Um hash também pode ser criado usando o método padrão new:

my_hash = Hash.new  # any empty hash
my_hash = {}        # any empty hash

Hashes podem ter valores de qualquer tipo, incluindo tipos complexos como arrays, objetos e outros hashes:

mapping = { 'Mark' => 15, 'Jimmy' => [3,4], 'Nika' => {'a' => 3, 'b' => 5} }
mapping['Mark']   # => 15
mapping['Jimmy']  # => [3, 4]
mapping['Nika']   # => {"a"=>3, "b"=>5}

As chaves também podem ser de qualquer tipo, incluindo as complexas:

mapping = { 'Mark' => 15, 5 => 10, [1, 2] => 9 }
mapping['Mark']  # => 15
mapping[[1, 2]]  # => 9

Símbolos são comumente usados ​​como chaves de hash, e o Ruby 1.9 introduziu uma nova sintaxe especificamente para encurtar esse processo. Os seguintes hashes são equivalentes:

# Valid on all Ruby versions
grades = { :Mark => 15, :Jimmy => 10, :Jack => 10 }
# Valid in Ruby version 1.9+
grades = { Mark: 15, Jimmy: 10, Jack: 10 }

O seguinte hash (válido em todas as versões do Ruby) é diferente, pois todas as chaves são strings:

grades = { "Mark" => 15, "Jimmy" => 10, "Jack" => 10 }

Embora ambas as versões de sintaxe possam ser misturadas, o seguinte é desencorajado.

mapping = { :length => 45, width: 10 }

Com Ruby 2.2+, existe uma sintaxe alternativa para criar um hash com chaves de símbolos (mais útil se o símbolo contiver espaços):

grades = { "Jimmy Choo": 10, :"Jack Sparrow": 10 }
# => { :"Jimmy Choo" => 10, :"Jack Sparrow" => 10}

Definindo valores padrão #

Por padrão, tentar pesquisar o valor de uma chave que não existe retornará nil. Opcionalmente, você pode especificar algum outro valor a ser retornado (ou uma ação a ser executada) quando o hash for acessado com uma chave inexistente. Embora seja referido como “o valor padrão”, não precisa ser um valor único; pode, por exemplo, ser um valor calculado, como o comprimento da chave.

O valor padrão de um hash pode ser passado para seu construtor:

h = Hash.new(0)

h[:hi] = 1 
puts h[:hi]  # => 1 
puts h[:bye] # => 0 returns default value instead of nil

Um padrão também pode ser especificado em um Hash já construído:

my_hash = { human: 2, animal: 1 }
my_hash.default = 0
my_hash[:plant] # => 0

É importante observar que o valor padrão não é copiado cada vez que uma nova chave é acessada, o que pode levar a resultados surpreendentes quando o valor padrão é um tipo de referência:

# Use an empty array as the default value
authors = Hash.new([])

# Append a book title 
authors[:homer] << 'The Odyssey'

# All new keys map to a reference to the same array:
authors[:plato] # => ['The Odyssey']

Para contornar esse problema, o construtor Hash aceita um bloco que é executado toda vez que uma nova chave é acessada, e o valor retornado é usado como padrão:

authors = Hash.new { [] }

# Note that we're using += instead of <<, see below
authors[:homer] += ['The Odyssey']
authors[:plato] # => []

authors # => {:homer=>["The Odyssey"]}

Observe que acima tivemos que usar += em vez de « porque o valor padrão não é atribuído automaticamente ao hash; usando « teria adicionado ao array, mas autores[:homer] teria permanecido indefinido:

authors[:homer] << 'The Odyssey' # ['The Odyssey']
authors[:homer] # => []
authors # => {}

Para poder atribuir valores padrão no acesso, bem como calcular padrões mais sofisticados, o bloco padrão recebe o hash e a chave:

authors = Hash.new { |hash, key| hash[key] = [] }

authors[:homer] << 'The Odyssey'
authors[:plato] # => []

authors # => {:homer=>["The Odyssey"], :plato=>[]}

Você também pode usar um bloco padrão para realizar uma ação e/ou retornar um valor dependente da chave (ou algum outro dado):

chars = Hash.new { |hash,key| key.length }

chars[:test] # => 4

Você pode até criar hashes mais complexos:

page_views = Hash.new { |hash, key| hash[key] = { count: 0, url: key } }
page_views["http://example.com"][:count] += 1
page_views # => {"http://example.com"=>{:count=>1, :url=>"http://example.com"}}

Para definir o padrão para um Proc em um hash already-existing, use default_proc=:

authors = {}
authors.default_proc = proc { [] }

authors[:homer] += ['The Odyssey']
authors[:plato] # => []

authors # {:homer=>["The Odyssey"]}

Acessando Valores #

Os valores individuais de um hash são lidos e escritos usando os métodos [] e []=:

my_hash = { length: 4, width: 5 }

my_hash[:length] #=> => 4

my_hash[:height] = 9

my_hash #=> {:length => 4, :width => 5, :height => 9 }

Por padrão, acessar uma chave que não foi adicionada ao hash retorna nil, o que significa que é sempre seguro tentar procurar o valor de uma chave:

my_hash = {}

my_hash[:age] # => nil

Hashes também podem conter chaves em strings. Se você tentar acessá-los normalmente, ele retornará apenas um nil, em vez disso, você os acessa por suas chaves de string:

my_hash = { "name" => "user" }

my_hash[:name]    # => nil
my_hash["name"]   # => user

Para situações em que as chaves são esperadas ou obrigatórias, os hashes têm um método fetch que irá gerar uma exceção ao acessar uma chave que não existe:

my_hash = {}

my_hash.fetch(:age) #=> KeyError: key not found: :age

fetch aceita um valor padrão como seu segundo argumento, que é retornado se a chave não tiver sido definida anteriormente:

my_hash =  {}
my_hash.fetch(:age, 45) #=> => 45

fetch também pode aceitar um bloco que é retornado se a chave não tiver sido definida anteriormente:

my_hash = {}
my_hash.fetch(:age) { 21 } #=> 21

my_hash.fetch(:age) do |k|
  puts "Could not find #{k}"
end

#=> Could not find age

Hashes também suportam um método store como um alias para []=:

my_hash = {}

my_hash.store(:age, 45)

my_hash #=> { :age => 45 }

Você também pode obter todos os valores de um hash usando o método values:

my_hash = { length: 4, width: 5 }

my_hash.values #=> [4, 5]

Nota: Isto é apenas para Ruby 2.3+ #dig é útil para Hashs aninhados. Extrai o valor aninhado especificado pela sequência de objetos idx chamando dig em cada etapa, retornando nil se alguma etapa intermediária for nil.

h = { foo: {bar: {baz: 1}}}

h.dig(:foo, :bar, :baz)   # => 1
h.dig(:foo, :zot, :xyz)   # => nil

g = { foo: [10, 11, 12] }
g.dig(:foo, 1)            # => 11

Criando automaticamente um Deep Hash #

Hash tem um valor padrão para chaves que são solicitadas, mas não existem (nil):

a = {}
p a[ :b ] # => nil

Ao criar um novo Hash, pode-se especificar o padrão:

b = Hash.new 'puppy'
p b[ :b ]            # => 'puppy'

Hash.new também recebe um bloco, que permite criar automaticamente hashes aninhados, como o comportamento de autovivificação do Perl ou mkdir -p:

# h is the hash you're creating, and k the key.
#
hash = Hash.new { |h, k| h[k] = Hash.new &h.default_proc }
hash[ :a ][ :b ][ :c ] = 3

p hash # => { a: { b: { c: 3 } } }

Iterando em um Hash #

Um Hash inclui o módulo Enumerable, que fornece vários métodos de iteração, como: Enumerable#each, Enumerable#each_pair, Enumerable#each_key e Enumerable#each_value.

.each e .each_pair iteram sobre cada par de valores-chave:

h = { "first_name" => "John", "last_name" => "Doe" }
h.each do |key, value|
    puts "#{key} = #{value}"
end

# => first_name = John
#    last_name = Doe

.each_key itera apenas sobre as chaves:

h = { "first_name" => "John", "last_name" => "Doe" }
h.each_key do |key|
  puts key
end

# => first_name
#    last_name

.each_value itera apenas sobre os valores:

h = { "first_name" => "John", "last_name" => "Doe" }
h.each_value do |value|
    puts value
end

# => John
#    Doe

.each_with_index itera sobre os elementos e fornece o índice da iteração:

h = { "first_name" => "John", "last_name" => "Doe" }
h.each_with_index do |(key, value), index|
    puts "index: #{index} | key: #{key} | value: #{value}"
end

# => index: 0 | key: first_name | value: John
#    index: 1 | key: last_name | value: Doe

Conversão de e para Arrays #

Hashes podem ser livremente convertidos de e para arrays. Converter um hash de pares chave/valor em uma matriz produzirá uma matriz contendo matrizes aninhadas para par:

{ :a => 1, :b => 2 }.to_a # => [[:a, 1], [:b, 2]]

Na direção oposta, um Hash pode ser criado a partir de um array do mesmo formato:

[[:x, 3], [:y, 4]].to_h # => { :x => 3, :y => 4 }

Da mesma forma, Hashes podem ser inicializados usando Hash[] e uma lista de chaves e valores alternados:

Hash[:a, 1, :b, 2] # => { :a => 1, :b => 2 }

Ou de uma matriz de matrizes com dois valores cada:

Hash[ [[:x, 3], [:y, 4]] ] # => { :x => 3, :y => 4 }

Hashes podem ser convertidos de volta para um Array de chaves e valores alternados usando flatten():

{ :a => 1, :b => 2 }.flatten # => [:a, 1, :b, 2]

A fácil conversão de e para um array permite que Hash funcione bem com muitos métodos Enumerable como collect e zip:

Hash[('a'..'z').collect{ |c| [c, c.upcase] }] # => { 'a' => 'A', 'b' => 'B', ... }

people = ['Alice', 'Bob', 'Eve']
height = [5.7, 6.0, 4.9]
Hash[people.zip(height)] # => { 'Alice' => 5.7, 'Bob' => '6.0', 'Eve' => 4.9 }

Filtrando hashes #

select retorna um novo hash com pares chave-valor para os quais o bloco é avaliado como true.

{ :a => 1, :b => 2, :c => 3 }.select { |k, v| k != :a && v.even? } # => { :b => 2 }

Quando você não precisar da chave ou valor em um bloco de filtro, a convenção é usar um _ nesse local:

{ :a => 1, :b => 2, :c => 3 }.select { |_, v| v.even? } # => { :b => 2 }
{ :a => 1, :b => 2, :c => 3 }.select { |k, _| k == :c } # => { :c => 3 }

reject retorna um novo hash com pares chave-valor para os quais o bloco é avaliado como false:

{ :a => 1, :b => 2, :c => 3 }.reject { |_, v| v.even? } # => { :a => 1, :c => 3 }
{ :a => 1, :b => 2, :c => 3 }.reject { |k, _| k == :b } # => { :a => 1, :c => 3 }

Obtendo todas as chaves ou valores de hash #

{foo: 'bar', biz: 'baz'}.keys   # => [:foo, :biz]
{foo: 'bar', biz: 'baz'}.values # => ["bar", "baz"]
{foo: 'bar', biz: 'baz'}.to_a   # => [[:foo, "bar"], [:biz, "baz"]]
{foo: 'bar', biz: 'baz'}.each   #<Enumerator: {:foo=>"bar", :biz=>"baz"}:each>

Sobrescrevendo a função de hash #

Os hashes Ruby usam os métodos hash e eql? para executar a operação de hash e atribuir objetos armazenados no hash a caixas de hash internas. A implementação padrão de hash em Ruby é a função de hash murmur sobre todos os campos membros do objeto com hash. Para sobrescrever este comportamento é possível sobrescrever os métodos hash e eql?.

Assim como em outras implementações de hash, dois objetos aeb serão codificados para o mesmo bucket se a.hash == b.hash e serão considerados idênticos se a.eql?(b). Assim, ao reimplementar hash e eql? deve-se tomar cuidado para garantir que se a e b forem iguais sob eql? eles devem retornar o mesmo valor hash. Caso contrário, isso pode resultar em entradas duplicadas em um hash. Por outro lado, uma má escolha na implementação do hash pode levar muitos objetos a compartilhar o mesmo hash bucket, destruindo efetivamente o tempo de consulta O(1) e fazendo com que O(n) chame eql? em todos os objetos.

No exemplo abaixo apenas a instância da classe A é armazenada como chave, pois foi adicionada primeiro:

class A
  def initialize(hash_value)
    @hash_value = hash_value
  end
  def hash
    @hash_value # Return the value given externally
  end
  def eql?(b)
    self.hash == b.hash
  end
end

class B < A
end

a = A.new(1)
b = B.new(1)

h = {}
h[a] = 1
h[b] = 2

raise "error" unless h.size == 1
raise "error" unless h.include? b
raise "error" unless h.include? a

Modificando chaves e valores #

Você pode criar um novo hash com as chaves ou valores modificados, na verdade, você também pode adicionar ou excluir chaves, usando [inject](http://ruby-doc.org/core-2.3.1/Enumerable.html#method-i -inject) (AKA, reduce). Por exemplo, para produzir um hash com chaves de string e valores em maiúsculas:

fruit = { name: 'apple', color: 'green', shape: 'round' }
# => {:name=>"apple", :color=>"green", :shape=>"round"}

new_fruit = fruit.inject({}) { |memo, (k,v)| memo[k.to_s] = v.upcase; memo }

# => new_fruit is {"name"=>"APPLE", "color"=>"GREEN", "shape"=>"ROUND"}

Hash é um enumerable, essencialmente uma coleção de pares chave/valor. Portanto, possui métodos como each, map e inject.

Para cada par chave/valor no hash o bloco dado é avaliado, o valor de memo na primeira execução é o valor semente passado para inject, no nosso caso um hash vazio, {}. O valor de memo para avaliações subseqüentes é o valor retornado da avaliação de blocos anteriores, é por isso que modificamos memo definindo uma chave com um valor e retornando memo no final. O valor de retorno da avaliação final dos blocos é o valor de retorno de inject, no nosso caso memo.

Para evitar a necessidade de fornecer o valor final, você pode usar each_with_object:

new_fruit = fruit.each_with_object({}) { |(k,v), memo| memo[k.to_s] = v.upcase }

Ou mesmo mapa:

<!-- if version [gte 1.8] -->
new_fruit = Hash[fruit.map{ |k,v| [k.to_s, v.upcase] }]
<!-- end version if -->

Definir operações em hashes #

  • Intersecção de Hashes

    To get the intersection of two hashes, return the shared keys the values of which are equal:

    hash1 = { :a => 1, :b => 2 }
hash2 = { :b => 2, :c => 3 }
hash1.select { |k, v| (hash2.include?(k) && hash2[k] == v) } # => { :b => 2 }

  • União (fusão) de hashes:

as chaves em um hash são únicas, se uma chave ocorrer em ambos os hashes que devem ser mesclados, aquele do hash que merge é chamado é substituído:

```
hash1 = { :a => 1, :b => 2 }
hash2 = { :b => 4, :c => 3 }

hash1.merge(hash2) # => { :a => 1, :b => 4, :c => 3 }
hash2.merge(hash1) # => { :b => 2, :c => 3, :a => 1 }
```
