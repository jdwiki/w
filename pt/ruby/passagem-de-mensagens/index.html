Introdução #

No Design Orientado a Objetos, os objetos recebem mensagens e respondem a elas. Em Ruby, enviar uma mensagem é chamar um método e o resultado desse método é a resposta.

Em Ruby, a passagem de mensagens é dinâmica. Quando uma mensagem chega ao invés de saber exatamente como respondê-la, Ruby usa um conjunto predefinido de regras para encontrar um método que possa respondê-la. Podemos usar essas regras para interromper e responder à mensagem, enviá-la para outro objeto ou modificá-la entre outras ações.

Cada vez que um objeto recebe uma mensagem o Ruby verifica:

  1. Se este objeto possui uma classe singleton e pode responder a esta mensagem.
  2. Procura a classe deste objeto e depois a cadeia de ancestrais da classe.
  3. Um por um verifica se um método está disponível neste ancestral e sobe na cadeia.

Mensagem passando pela cadeia de herança #

class Example
  def example_method
    :example
  end

  def subexample_method
    :example
  end

  def not_missed_method
    :example
  end

  def method_missing name
    return :example if name == :missing_example_method
    return :example if name == :missing_subexample_method
    return :subexample if name == :not_missed_method
    super
  end
end

class SubExample < Example
  def subexample_method
    :subexample
  end

  def method_missing name
    return :subexample if name == :missing_subexample_method
    return :subexample if name == :not_missed_method
    super
  end
end

s = Subexample.new

Para encontrar um método adequado para SubExample#subexample_method Ruby primeiro olha para a cadeia de ancestrais de SubExample

SubExample.ancestors # => [SubExample, Example, Object, Kernel, BasicObject]

Ele começa em SubExample. Se enviarmos a mensagem subexample_method Ruby escolhe o subexemplo disponível e ignora Example#subexample_method.

s.subexample_method # => :subexample

Após SubExample ele verifica Example. Se enviarmos example_method Ruby verifica se SubExample pode responder a ele ou não e como não pode, Ruby sobe a cadeia e procura por Example.

s.example_method # => :example

Após o Ruby verificar todos os métodos definidos, ele executa method_missing para ver se pode responder ou não. Se enviarmos missing_subexample_method, o Ruby não será capaz de encontrar um método definido em SubExample, então ele sobe para Example. Ele também não pode encontrar um método definido em Example ou qualquer outra classe superior na cadeia. Ruby reinicia e executa method_missing. method_missing de SubExample pode responder a missing_subexample_method.

s.missing_subexample_method # => :subexample

No entanto, se um método é definido, o Ruby usa a versão definida, mesmo que seja superior na cadeia. Por exemplo, se enviarmos not_missed_method mesmo que method_missing de SubExample possa responder a ele, Ruby vai até SubExample porque não tem um método definido com esse nome e procura por Example que tem um.

s.not_missed_method # => :example

Mensagem que passa pela composição do módulo #

Ruby sobe na cadeia de ancestrais de um objeto. Essa cadeia pode conter módulos e classes. As mesmas regras sobre subir na cadeia também se aplicam aos módulos.

class Example
end

module Prepended
  def initialize *args
    return super :default if args.empty?
    super
  end
end

module FirstIncluded
  def foo
    :first
  end
end

module SecondIncluded
  def foo
    :second
  end
end

class SubExample < Example
  prepend Prepended
  include FirstIncluded
  include SecondIncluded

  def initialize data = :subexample
    puts data
  end
end

SubExample.ancestors # => [Prepended, SubExample, SecondIncluded, FirstIncluded, Example, Object, Kernel, BasicObject]

s = SubExample.new # => :default
s.foo # => :second

Interrompendo mensagens #

Existem duas maneiras de interromper mensagens.

  • Use method_missing para interromper qualquer mensagem não definida.
  • Defina um método no meio de uma cadeia para interceptar a mensagem

Após interromper as mensagens, é possível:

  • Responda a eles.
  • Envie-os para outro lugar.
  • Modifique a mensagem ou seu resultado.

Interrompendo via method_missing e respondendo a mensagem:

class Example
  def foo
    @foo
  end

  def method_missing name, data
    return super unless name.to_s =~ /=$/
    name = name.to_s.sub(/=$/, "")
    instance_variable_set "@#{name}", data
  end
end

e = Example.new

e.foo = :foo
e.foo # => :foo

Interceptando mensagem e modificando-a:

class Example
  def initialize title, body
  end
end

class SubExample < Example
end

Agora vamos imaginar que nossos dados são “title:body” e temos que dividi-los antes de chamar Example. Podemos definir initialize em SubExample.

class SubExample < Example
  def initialize raw_data
    processed_data = raw_data.split ":"

    super processed_data[0], processed_data[1]
  end
end

Interceptando mensagem e enviando para outro objeto:

class ObscureLogicProcessor
  def process data
    :ok
  end
end

class NormalLogicProcessor
  def process data
    :not_ok
  end
end

class WrapperProcessor < NormalLogicProcessor
  def process data
    return ObscureLogicProcessor.new.process data if data.obscure?

    super
  end
end
