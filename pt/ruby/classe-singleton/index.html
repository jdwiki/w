Sintaxe #

  • singleton_class = classe « objeto; auto fim

As classes singleton têm apenas uma instância: seu objeto correspondente. Isso pode ser verificado consultando o ObjectSpace do Ruby:

instances = ObjectSpace.each_object object.singleton_class

instances.count            # => 1
instances.include? object  # => true

Usando [<][Module#<], eles também podem ser verificados como subclasses da classe real do objeto:

object.singleton_class < object.class  # => true

Referências:

  • Três contextos implícitos em Ruby

Herança da classe Singleton #

Subclasses também Subclasses Classe Singleton #

class Example
end

Example.singleton_class #=> #<Class:Example>

def Example.foo
  :example
end

class SubExample < Example
end

SubExample.foo #=> :example

SubExample.singleton_class.superclass #=> #<Class:Example>

Estender ou incluir um módulo não estende a classe Singleton #

module ExampleModule
end

def ExampleModule.foo
  :foo
end

class Example
  extend ExampleModule
  include ExampleModule
end

Example.foo #=> NoMethodError: undefined method

Classes Singleton #

Todos os objetos são instâncias de uma classe. No entanto, essa não é toda a verdade. Em Ruby, cada objeto também tem uma classe única escondida.

Isso é o que permite que métodos sejam definidos em objetos individuais. A classe singleton fica entre o próprio objeto e sua classe real, portanto, todos os métodos definidos nela estão disponíveis para esse objeto e apenas para esse objeto.

object = Object.new

def object.exclusive_method
  'Only this object will respond to this method'
end

object.exclusive_method
# => "Only this object will respond to this method"

Object.new.exclusive_method rescue $!
# => #<NoMethodError: undefined method `exclusive_method' for #<Object:0xa17b77c>>

O exemplo acima poderia ter sido escrito usando define_singleton_method :

object.define_singleton_method :exclusive_method do
  "The method is actually defined in the object's singleton class"
end

O que é o mesmo que definir o método em object singleton_class:

# send is used because define_method is private
object.singleton_class.send :define_method, :exclusive_method do
  "Now we're defining an instance method directly on the singleton class"
end

Antes da existência de singleton_class como parte da API principal do Ruby, as classes singleton eram conhecidas como metaclasses e podiam ser acessadas através do seguinte idioma:

class << object
  self  # refers to object's singleton_class
end

Introdução #

Ruby tem três tipos de objetos:

  • Classes e módulos que são instâncias da classe Class ou classe Module.
  • Instâncias de classes.
  • Classes Singleton.

Cada objeto tem uma classe que contém seus métodos:

class Example
end

object = Example.new

object.class  # => Example
Example.class # => Class
Class.class   # => Class

Os próprios objetos não podem conter métodos, apenas sua classe pode. Mas com classes singleton, é possível adicionar métodos a qualquer objeto, incluindo outras classes singleton.

def object.foo
  :foo
end
object.foo #=> :foo

foo é definido na classe singleton de object. Outras instâncias Example não podem responder a foo.

Ruby cria classes singleton sob demanda. Acessá-los ou adicionar métodos a eles força o Ruby a criá-los.

Acessando a classe Singleton #

Existem duas maneiras de obter a classe singleton de um objeto

  • método singleton_class.

  • Reabrindo a classe singleton de um objeto e retornando self.

    object.singleton_class

singleton_class = class << object
  self
end

Acessando variáveis ​​de instância/classe em classes singleton #

As classes singleton compartilham suas variáveis ​​de instância/classe com seu objeto.

class Example
  @@foo = :example
end

def Example.foo
  class_variable_get :@@foo
end

Example.foo #=> :example

class Example
  def initialize
    @foo = 1
  end

  def foo
    @foo
  end
end

e = Example.new

e.instance_eval <<-BLOCK
  def self.increase_foo
    @foo += 1
  end
BLOCK

e.increase_foo
e.foo #=> 2

Os blocos fecham em torno de seu destino de variáveis ​​de instância/classe. Acessar variáveis ​​de instância ou classe usando um bloco em class_eval ou instance_eval não é possível. Passar uma string para class_eval ou usar class_variable_get resolve o problema.

class Foo
  @@foo = :foo
end

class Example
  @@foo = :example 

  Foo.define_singleton_method :foo do
    @@foo
  end
end

Foo.foo #=> :example

Propagação de Mensagem com Classe Singleton #

As instâncias nunca contêm um método, elas apenas carregam dados. No entanto, podemos definir uma classe singleton para qualquer objeto, incluindo uma instância de uma classe.

Quando uma mensagem é passada para um objeto (o método é chamado) o Ruby primeiro verifica se uma classe singleton está definida para aquele objeto e se ele pode responder a essa mensagem, caso contrário, o Ruby verifica a cadeia de ancestrais da classe da instância e caminha sobre ela.

class Example
  def foo
    :example
  end
end

Example.new.foo #=> :example

module PrependedModule
  def foo
    :prepend
  end
end

class Example
  prepend PrependedModule
end

Example.ancestors #=> [Prepended, Example, Object, Kernel, BasicObject]
e = Example.new
e.foo #=> :prepended

def e.foo
  :singleton
end

e.foo #=> :singleton

Reabrindo (monkey patching) Classes Singleton #

Existem três maneiras de reabrir uma classe Singleton

  • Usando class_eval em uma classe singleton.

  • Usando o bloco class <<.

  • Usando def para definir um método diretamente na classe singleton do objeto

    class Example end

    Example.singleton_class.class_eval do def foo :foo end end

    Example.foo #=> :foo

class Example
end

class << Example
  def bar
    :bar
  end
end

Example.bar #=> :bar

class Example
end

def Example.baz
  :baz
end

Example.baz #=> :baz

Cada objeto tem uma classe singleton que você pode acessar

class Example
end
ex1 = Example.new
def ex1.foobar
  :foobar
end
ex1.foobar #=> :foobar

ex2 = Example.new
ex2.foobar #=> NoMethodError
