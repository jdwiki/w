Hierarquia de números #

Ruby inclui várias classes internas para representar números:

Numeric
  Integer
    Fixnum    # 1
    Bignum    # 10000000000000000000
  Float       # 1.0
  Complex     # (1+0i)
  Rational    # Rational(2, 3) == 2/3
  BigDecimal  # not loaded by default

Os mais comuns são:

  • Fixnum para representar, por exemplo, inteiros positivos e negativos
  • Float para representar números de ponto flutuante

BigDecimal é o único não carregado por padrão. Você pode carregá-lo com:

require "bigdecimal"

Note que no Ruby 2.4+, Fixnum e Bignum são unificados; todos inteiros agora são apenas membros da classe Integer. Para compatibilidade com versões anteriores, Fixnum == Bignum == Integer.

Convertendo uma String para Integer #

Você pode usar o método Integer para converter uma String para um Integer:

Integer("123")      # => 123
Integer("0xFF")     # => 255
Integer("0b100")    # => 4
Integer("0555")     # => 365

Você também pode passar um parâmetro base para o método Integer para converter números de uma determinada base

Integer('10', 5)    # => 5
Integer('74', 8)    # => 60
Integer('NUM', 36)  # => 30910

Observe que o método gera um ArgumentError se o parâmetro não puder ser convertido:

Integer("hello")
# raises ArgumentError: invalid value for Integer(): "hello"
Integer("23-hello")
# raises ArgumentError: invalid value for Integer(): "23-hello"

Você também pode usar o método String#to_i. No entanto, este método é um pouco mais permissivo e tem um comportamento diferente de Integer:

"23".to_i         # => 23
"23-hello".to_i   # => 23
"hello".to_i      # => 0

String#to_i aceita um argumento, a base para interpretar o número como:

"10".to_i(2) # => 2
"10".to_i(3) # => 3
"A".to_i(16) # => 10

Criando um inteiro #

0       # creates the Fixnum 0
123     # creates the Fixnum 123
1_000   # creates the Fixnum 1000. You can use _ as separator for readability

Por padrão, a notação é base 10. No entanto, existem algumas outras notações internas para diferentes bases:

0xFF    # Hexadecimal representation of 255, starts with a 0x
0b100   # Binary representation of 4, starts with a 0b
0555    # Octal representation of 365, starts with a 0 and digits

Números de arredondamento #

O método round arredondará um número para cima se o primeiro dígito após sua casa decimal for 5 ou maior e arredondará para baixo se esse dígito for 4 ou menor. Isso recebe um argumento opcional para a precisão que você está procurando.

4.89.round        # => 5
4.25.round        # => 4
3.141526.round(1) # => 3.1
3.141526.round(2) # => 3.14
3.141526.round(4) # => 3.1415

Os números de ponto flutuante também podem ser arredondados para o número inteiro mais alto e menor que o número com o método floor

4.9999999999999.floor # => 4

Eles também podem ser arredondados para o menor inteiro maior que o número usando o método ceil

4.0000000000001.ceil  # => 5

Números pares e ímpares #

O método even? pode ser usado para determinar se um número é par

4.even?      # => true
5.even?      # => false

O método odd? pode ser usado para determinar se um número é ímpar

4.odd?       # => false
5.odd?       # => true

Convertendo um número em uma string #

Fixnum#to_s recebe um argumento base opcional e representa o número fornecido nessa base:

2.to_s(2)   # => "10"
3.to_s(2)   # => "11"
3.to_s(3)   # => "10"
10.to_s(16) # => "a"

Se nenhum argumento for fornecido, ele representa o número na base 10

2.to_s # => "2"
10423.to_s # => "10423"

Números racionais #

Rational representa um número racional como numerador e denominador:

r1 = Rational(2, 3)
r2 = 2.5.to_r
r3 = r1 + r2
r3.numerator   # => 19
r3.denominator # => 6
Rational(2, 4) # => (1/2)

Outras maneiras de criar um Rational

Rational('2/3')  # => (2/3)
Rational(3)      # => (3/1)
Rational(3, -5)  # => (-3/5)
Rational(0.2)    # => (3602879701896397/18014398509481984)
Rational('0.2')  # => (1/5)
0.2.to_r         # => (3602879701896397/18014398509481984)
0.2.rationalize  # => (1/5)
'1/4'.to_r       # => (1/4)

Números complexos #

1i     # => (0+1i)
1.to_c # => (1+0i)
rectangular = Complex(2, 3)  # => (2+3i)
polar       = Complex('1@2') # => (-0.4161468365471424+0.9092974268256817i)

polar.rectangular # => [-0.4161468365471424, 0.9092974268256817]
rectangular.polar # => [3.605551275463989, 0.982793723247329]
rectangular + polar # => (1.5838531634528576+3.909297426825682i)

Dividindo dois números #

Ao dividir dois números preste atenção no tipo que você quer em troca. Observe que dividir dois inteiros invocará a divisão inteira. Se seu objetivo é executar a divisão float, pelo menos um dos parâmetros deve ser do tipo float.

Divisão inteira:

3 / 2 # => 1

Divisão flutuante

3 / 3.0 # => 1.0

16 / 2 / 2    # => 4
16 / 2 / 2.0  # => 4.0
16 / 2.0 / 2  # => 4.0
16.0 / 2 / 2  # => 4.0
