Sintaxe #

  • ‘A string’ // cria uma string via literal entre aspas simples
  • “A string” // cria uma string via literal com aspas duplas
  • String.new(“Uma string”)
  • %q(A string) // sintaxe alternativa para criar strings entre aspas simples
  • %Q(A string) // sintaxe alternativa para criar strings com aspas duplas

Diferença entre literais String com aspas simples e aspas duplas #

A principal diferença é que os literais String com aspas duplas suportam interpolações de strings e o conjunto completo de sequências de escape.

Por exemplo, eles podem incluir expressões Ruby arbitrárias por meio de interpolação:

# Single-quoted strings don't support interpolation
puts 'Now is #{Time.now}'
# Now is #{Time.now}

# Double-quoted strings support interpolation
puts "Now is #{Time.now}"
# Now is 2016-07-21 12:43:04 +0200

Strings com aspas duplas também suportam o conjunto inteiro de sequências de escape incluindo "\n", "\t"…

puts 'Hello\nWorld'
# Hello\nWorld

puts "Hello\nWorld"
# Hello
# World

… enquanto strings entre aspas simples suportam nenhuma seqüências de escape, desnudando o conjunto mínimo necessário para que strings entre aspas simples sejam úteis: Aspas simples e barras invertidas, '\'' e '\\' respectivamente.

Criando uma string #

Ruby fornece várias maneiras de criar um objeto String. A maneira mais comum é usar aspas simples ou duplas para criar um “string literal”:

s1 = 'Hello'
s2 = "Hello"

A principal diferença é que os literais de string com aspas duplas são um pouco mais flexíveis, pois suportam interpolação e algumas sequências de escape de barra invertida.

Há também várias outras maneiras possíveis de criar um literal de string usando delimitadores de string arbitrários. Um delimitador de string arbitrário é um % seguido por um par de delimitadores correspondentes:

%(A string)
%{A string}
%<A string>
%|A string|
%!A string!

Finalmente, você pode usar a sequência %q e %Q, que são equivalentes a ' e "":

puts %q(A string)
# A string
puts %q(Now is #{Time.now})
# Now is #{Time.now}

puts %Q(A string)
# A string
puts %Q(Now is #{Time.now})
# Now is 2016-07-21 12:47:45 +0200

As sequências %q e %Q são úteis quando a string contém aspas simples, aspas duplas ou uma mistura de ambas. Dessa forma, você não precisa escapar do conteúdo:

%Q(<a href="/profile">User's profile<a>)

Você pode usar vários delimitadores diferentes, desde que haja um par correspondente:

%q(A string)
%q{A string}
%q<A string>
%q|A string|
%q!A string!

Manipulação de casos #

"string".upcase     # => "STRING"
"STRING".downcase   # => "string"
"String".swapcase   # => "sTRING"
"string".capitalize # => "String"

Esses quatro métodos não modificam o receptor original. Por exemplo,

str = "Hello"
str.upcase # => "HELLO"
puts str   # => "Hello"

Existem quatro métodos semelhantes que executam as mesmas ações, mas modificam o receptor original.

"string".upcase!     # => "STRING"
"STRING".downcase!   # => "string"
"String".swapcase!   # => "sTRING"
"string".capitalize! # => "String"

Por exemplo,

str = "Hello"
str.upcase!  # => "HELLO"
puts str     # => "HELLO"

Notas:

  • antes do Ruby 2.4 esses métodos não lidam com unicode.

Concatenação de strings #

Concatenar strings com o operador +:

s1 = "Hello"
s2 = " "
s3 = "World"

puts s1 + s2 + s3
# => Hello World

s = s1 + s2 + s3
puts s
# => Hello World

Ou com o operador <<:

s = 'Hello'
s << ' '
s << 'World'
puts s
# => Hello World

Observe que o operador << modifica o objeto do lado esquerdo.

Você também pode multiplicar strings, por exemplo.

"wow" * 3
# => "wowwowwow"

Dividindo uma String #

String#split divide uma String em uma Array, com base em um delimitador.

"alpha,beta".split(",")
# => ["alpha", "beta"]

Um String vazio resulta em um Array vazio:

"".split(",")
# => []

Um delimitador não correspondente resulta em um Array contendo um único item:

"alpha,beta".split(".")
# => ["alpha,beta"]

Você também pode dividir uma string usando expressões regulares:

"alpha, beta,gamma".split(/, ?/)
# => ["alpha", "beta", "gamma"]

O delimitador é opcional, por padrão uma string é dividida em espaço em branco:

"alpha beta".split
# => ["alpha", "beta"]

Posicionando strings #

Em Ruby, as strings podem ser justificadas à esquerda, justificadas à direita ou centralizadas

Para justificar a string à esquerda, use o método ljust. Isso leva em dois parâmetros, um inteiro representando o número de caracteres da nova string e uma string, representando o padrão a ser preenchido.

Se o número inteiro for maior que o comprimento da string original, a nova string será justificada à esquerda com o parâmetro de string opcional ocupando o espaço restante. Se o parâmetro string não for fornecido, a string será preenchida com espaços.

str ="abcd"
str.ljust(4)          => "abcd"
str.ljust(10)         => "abcd      "

Para justificar uma string à direita, use o método rjust. Isso leva em dois parâmetros, um inteiro representando o número de caracteres da nova string e uma string, representando o padrão a ser preenchido.

Se o número inteiro for maior que o comprimento da string original, a nova string será justificada à direita com o parâmetro de string opcional ocupando o espaço restante. Se o parâmetro string não for fornecido, a string será preenchida com espaços.

str = "abcd"
str.rjust(4)          => "abcd"
str.rjust(10)         => "      abcd"

Para centralizar uma string, use o método center. Isso leva em dois parâmetros, um inteiro representando a largura da nova string e uma string, com a qual a string original será preenchida. A corda será alinhada ao centro.

str = "abcd"
str.center(4)          => "abcd"
str.center(10)         => "   abcd   "

Unindo Strings #

Array#join junta um Array em uma String, com base em um delimitador:

["alpha", "beta"].join(",")
# => "alpha,beta"

O delimitador é opcional e o padrão é uma String vazia.

["alpha", "beta"].join
# => "alphabeta"

Um Array vazio resulta em um String vazio, não importa qual delimitador seja usado.

[].join(",")
# => ""

A string começa com #

Para descobrir se uma string começa com um padrão, o método start_with? é útil

str = "zebras are cool"
str.start_with?("zebras")      => true

Você também pode verificar a posição do padrão com index

str = "zebras are cool"
str.index("zebras").zero?      => true

Interpolação de strings #

O delimitador com aspas duplas " e a sequência %Q suporta interpolação de string usando #{ruby_expression}:

puts "Now is #{Time.now}"
# Now is Now is 2016-07-21 12:47:45 +0200

puts %Q(Now is #{Time.now})
# Now is Now is 2016-07-21 12:47:45 +0200

A string termina com #

Para descobrir se uma string termina com um padrão, o método end_with? é útil

str = "I like pineapples"
str.end_with?("pineaaples")      => false

Strings de várias linhas #

A maneira mais fácil de criar uma string de várias linhas é usar várias linhas entre aspas:

address = "Four score and seven years ago our fathers brought forth on this
continent, a new nation, conceived in Liberty, and dedicated to the
proposition that all men are created equal."

O principal problema com essa técnica é que, se a string incluir uma citação, ela quebrará a sintaxe da string. Para contornar o problema, você pode usar um heredoc:

puts <<-RAVEN
  Once upon a midnight dreary, while I pondered, weak and weary, 
  Over many a quaint and curious volume of forgotten lore— 
      While I nodded, nearly napping, suddenly there came a tapping, 
  As of some one gently rapping, rapping at my chamber door. 
  "'Tis some visitor," I muttered, "tapping at my chamber door— 
              Only this and nothing more." 
  RAVEN

Ruby suporta documentos no estilo shell aqui com <<EOT, mas o texto final deve iniciar a linha. Isso estraga o recuo do código, então não há muitas razões para usar esse estilo. Infelizmente, a string terá recuos dependendo de como o próprio código é recuado.

Ruby 2.3 resolve o problema introduzindo <<~ que remove o excesso de espaços à esquerda:

<!-- if version [gte 2.3] -->
def build_email(address)
  return (<<~EMAIL)
    TO: #{address}

    To Whom It May Concern:

    Please stop playing the bagpipes at sunrise!
                     
    Regards,
    Your neighbor               
  EMAIL
end
<!-- end version if -->

Percent Strings também funcionam para criar strings de várias linhas:

%q(
HAMLET        Do you see yonder cloud that's almost in shape of a camel?
POLONIUS        By the mass, and 'tis like a camel, indeed.
HAMLET        Methinks it is like a weasel.
POLONIUS        It is backed like a weasel.
HAMLET        Or like a whale?
POLONIUS        Very like a whale
)

Existem algumas maneiras de evitar interpolação e sequências de escape:

  • Aspas simples em vez de aspas duplas: '\n é um retorno de carro.'

  • Minúsculas q em uma string de porcentagem: %q[#{not-a-variable}]

  • Aspas simples na string do terminal em um heredoc:

    <<-'CODE'
   puts 'Hello world!'
CODE

Strings formatadas #

Ruby pode injetar um array de valores em uma string substituindo qualquer placeholder pelos valores do array fornecido.

"Hello %s, my name is %s!" % ['World', 'br3nt']
# => Hello World, my name is br3nt!

Os espaços reservados são representados por dois %s e os valores são fornecidos pelo array ['Hello', 'br3nt']. O operador % instrui a string a injetar os valores do array.

Substituições de caracteres de string #

O método tr retorna uma cópia de uma string onde os caracteres do primeiro argumento são substituídos pelos caracteres do segundo argumento.

"string".tr('r', 'l') # => "stling"

Para substituir apenas a primeira ocorrência de um padrão por outra expressão, use o método sub

"string ring".sub('r', 'l') # => "stling ring"

Se você quiser substituir todas ocorrências de um padrão com essa expressão use gsub

"string ring".gsub('r','l') # => "stling ling"

Para excluir caracteres, passe uma string vazia para o segundo parâmetro

Você também pode usar expressões regulares em todos esses métodos.

É importante observar que esses métodos retornarão apenas uma nova cópia de uma string e não modificarão a string no local. Para fazer isso, você precisa usar os métodos tr!, sub! e gsub! respectivamente.

Entendendo os dados em uma string #

Em Ruby, uma string é apenas uma sequência de bytes junto com o nome de uma codificação (como UTF-8, US-ASCII, ASCII-8BIT) que especifica como você pode interpretar esses bytes como caracteres.

Strings Ruby podem ser usadas para armazenar texto (basicamente uma sequência de caracteres), caso em que a codificação UTF-8 geralmente é usada.

"abc".bytes  # => [97, 98, 99]
"abc".encoding.name  # => "UTF-8"

Strings Ruby também podem ser usadas para armazenar dados binários (uma sequência de bytes), caso em que a codificação ASCII-8BIT geralmente é usada.

[42].pack("i").encoding  # => "ASCII-8BIT"

É possível que a sequência de bytes em uma string não corresponda à codificação, resultando em erros se você tentar usar a string.

"\xFF \xFF".valid_encoding? # => false
"\xFF \xFF".split(' ')      # ArgumentError: invalid byte sequence in UTF-8

Substituição de strings #

p "This is %s" % "foo"
# => "This is foo"

p "%s %s %s" % ["foo", "bar", "baz"]
# => "foo bar baz"

p "%{foo} == %{foo}" % {:foo => "foo" }
# => "foo == foo"

Veja os documentos String % e Kernel::sprintf para mais detalhes.
