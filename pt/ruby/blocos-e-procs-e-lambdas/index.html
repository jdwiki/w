Sintaxe #

  • Proc.new(block)
  • lambda { |args| código }
  • ->(arg1, arg2) { código }
  • object.to_proc
  • { |arg_único| código }
  • do |arg, (chave, valor)| code fim

Tenha cuidado com a precedência do operador quando tiver uma linha com vários métodos encadeados, como:

str = "abcdefg"
puts str.gsub(/./) do |match|
  rand(2).zero? ? match.upcase : match.downcase
end

Em vez de imprimir algo como abCDeFg, como você esperaria, ele imprime algo como #<Enumerator:0x00000000af42b28> – isso ocorre porque do ... end tem precedência menor que métodos, o que significa que gsub vê apenas o argumento /./, e não o argumento do bloco. Ele retorna um enumerador. O bloco acaba sendo passado para puts, que o ignora e apenas exibe o resultado de gsub(/./).

Para corrigir isso, coloque a chamada gsub entre parênteses ou use { ... } em vez disso.

Lambdas #

# lambda using the arrow syntax
hello_world = -> { 'Hello World!' }
hello_world[]
# 'Hello World!'

# lambda using the arrow syntax accepting 1 argument
hello_world = ->(name) { "Hello #{name}!" }
hello_world['Sven']
# "Hello Sven!"

the_thing = lambda do |magic, ohai, dere|
  puts "magic! #{magic}"
  puts "ohai #{dere}"
  puts "#{ohai} means hello"
end

the_thing.call(1, 2, 3)
# magic! 1
# ohai 3
# 2 means hello

the_thing.call(1, 2)
# ArgumentError: wrong number of arguments (2 for 3)

the_thing[1, 2, 3, 4]
# ArgumentError: wrong number of arguments (4 for 3)

Você também pode usar -> para criar e .() para chamar lambda

the_thing = ->(magic, ohai, dere) {
  puts "magic! #{magic}"
  puts "ohai #{dere}"
  puts "#{ohai} means hello"
}

the_thing.(1, 2, 3)
# => magic! 1
# => ohai 3
# => 2 means hello

Aqui você pode ver que um lambda é quase o mesmo que um proc. No entanto, há várias ressalvas:

  • A aridade dos argumentos de um lambda é reforçada; passar o número errado de argumentos para um lambda, irá gerar um ArgumentError. Eles ainda podem ter parâmetros padrão, parâmetros splat, etc.

  • retornar de dentro de um lambda retorna do lambda, enquanto retornar de um proc retorna fora do escopo delimitador:

    def try_proc
  x = Proc.new {
    return # Return from try_proc
  }
  x.call
  puts "After x.call" # this line is never reached
end

def try_lambda
  y = -> {
    return # return from y
  }
  y.call
  puts "After y.call" # this line is not skipped
end

try_proc # No output
try_lambda # Outputs "After y.call"

Aplicação Parcial e Curry #

Tecnicamente, Ruby não possui funções, mas métodos. No entanto, um método Ruby se comporta de forma quase idêntica a funções em outra linguagem:

def double(n)
  n * 2
end

Este método/função normal recebe um parâmetro n, duplica-o e retorna o valor. Agora vamos definir uma função (ou método) de ordem superior:

def triple(n)
  lambda {3 * n}
end

Em vez de retornar um número, triple retorna um método. Você pode testá-lo usando o Interactive Ruby Shell:

$ irb --simple-prompt
>> def double(n)
>>   n * 2
>> end
=> :double
>> def triple(n)
>>   lambda {3 * n}
>> end
=> :triple
>> double(2)
=> 4
>> triple(2)
=> #<Proc:0x007fd07f07bdc0@(irb):7 (lambda)>

Se você quiser realmente obter o número triplicado, você precisa chamar (ou “reduzir”) o lambda:

triple_two = triple(2)
triple_two.call # => 6

Ou mais concisamente:

triple(2).call

Curry e aplicações parciais #

Isso não é útil em termos de definição de funcionalidades muito básicas, mas é útil se você deseja ter métodos/funções que não são chamados ou reduzidos instantaneamente. Por exemplo, digamos que você queira definir métodos que adicionam um número por um número específico (por exemplo add_one(2) = 3). Se você tivesse que definir uma tonelada disso, você poderia fazer:

def add_one(n)
  n + 1
end 

def add_two(n)
  n + 2
end

No entanto, você também pode fazer isso:

add = -> (a, b) { a + b }
add_one = add.curry.(1)
add_two = add.curry.(2)

Usando cálculo lambda podemos dizer que add é (λa.(λb.(a+b))). Currying é uma forma de aplicar parcialmente add. Então add.curry.(1), é (λa.(λb.(a+b)))(1) que pode ser reduzido para (λb.(1+b)). Aplicação parcial significa que passamos um argumento para add, mas deixamos o outro argumento para ser fornecido posteriormente. A saída é um método especializado.

Mais exemplos úteis de curry #

Digamos que temos uma fórmula geral realmente grande, que se especificarmos certos argumentos para ela, podemos obter fórmulas específicas dela. Considere esta fórmula:

f(x, y, z) = sin(x\*y)*sin(y\*z)*sin(z\*x)

Essa fórmula é feita para trabalhar em três dimensões, mas digamos que queremos apenas essa fórmula em relação a y e z. Digamos também que para ignorar x, queremos definir seu valor para pi/2. Vamos primeiro fazer a fórmula geral:

f = ->(x, y, z) {Math.sin(x*y) * Math.sin(y*z) * Math.sin(z*x)}

Agora, vamos usar currying para obter nossa fórmula yz:

f_yz = f.curry.(Math::PI/2)

Então, para chamar o lambda armazenado em f_yz:

f_xy.call(some_value_x, some_value_y)

Isso é bem simples, mas digamos que queremos obter a fórmula para xz. Como podemos definir y para Math::PI/2 se não for o último argumento? Bem, é um pouco mais complicado:

f_xz = -> (x,z) {f.curry.(x, Math::PI/2, z)}

Nesse caso, precisamos fornecer espaços reservados para o parâmetro que não estamos preenchendo previamente. Para consistência, poderíamos escrever f_xy assim:

f_xy = -> (x,y) {f.curry.(x, y, Math::PI/2)}

Veja como o cálculo lambda funciona para f_yz:

f = (λx.(λy.(λz.(sin(x*y) * sin(y*z) * sin(z*x))))
f_yz = (λx.(λy.(λz.(sin(x*y) * sin(y*z) * sin(z*x)))) (π/2) # Reduce => 
f_yz = (λy.(λz.(sin((π/2)*y) * sin(y*z) * sin(z*(π/2))))

Agora vamos olhar para f_xz

f = (λx.(λy.(λz.(sin(x*y) * sin(y*z) * sin(z*x))))
f_xz = (λx.(λy.(λz.(sin(x*y) * sin(y*z) * sin(z*x)))) (λt.t) (π/2)  # Reduce =>
f_xz = (λt.(λz.(sin(t*(π/2)) * sin((π/2)*z) * sin(z*t))))

Para mais leitura sobre cálculo lambda tente este.

Objetos como argumentos de bloco para métodos #

Colocar um & (e comercial) na frente de um argumento o passará como o bloco do método. Os objetos serão convertidos em um Proc usando o método to_proc.

class Greeter
  def to_proc
    Proc.new do |item|
      puts "Hello, #{item}"
    end
  end
end

greet = Greeter.new

%w(world life).each(&greet)

Este é um padrão comum em Ruby e muitas classes padrão o fornecem.

Por exemplo, [Symbol][symbol]s implementam to_proc enviando-se para o argumento:

# Example implementation
class Symbol
  def to_proc
    Proc.new do |receiver|
      receiver.send self
    end
  end
end

Isso habilita o idioma útil &:symbol, comumente usado com objetos Enumerable:

letter_counts = %w(just some words).map(&:length)  # [4, 4, 5]

Convertendo para Proc #

Objetos que respondem a to_proc podem ser convertidos em procs com o operador & (que também permitirá que eles sejam passados ​​como blocos).

A classe Symbol define #to_proc para tentar chamar o método correspondente no objeto que recebe como parâmetro.

p [ 'rabbit', 'grass' ].map( &:upcase ) # => ["RABBIT", "GRASS"]

Objetos de método também definem #to_proc.

output = method( :p )

[ 'rabbit', 'grass' ].map( &output ) # => "rabbit\ngrass"

Blocos #

Blocos são pedaços de código entre chaves {} (geralmente para blocos de linha única) ou do..end (usado para blocos de várias linhas).

5.times { puts "Hello world" } # recommended style for single line blocks

5.times do
    print "Hello "
    puts "world"
end   # recommended style for multi-line blocks

5.times {
    print "hello "
    puts "world" } # does not throw an error but is not recommended

Nota: chaves têm precedência maior que do..end

Produzindo #

Blocos podem ser usados ​​dentro de métodos e funções usando a palavra yield:

def block_caller
    puts "some code"
    yield
    puts "other code"
end
block_caller { puts "My own block" } # the block is passed as an argument to the method.
#some code
#My own block
#other code

Tenha cuidado, porém, se yield for chamado sem um bloco, ele irá gerar um LocalJumpError. Para este propósito, o ruby ​​fornece outro método chamado block_given? que permite verificar se um bloco foi passado antes de chamar o yield

def block_caller
  puts "some code" 
  if block_given? 
    yield
  else
    puts "default"
  end
  puts "other code"
end
block_caller 
# some code
# default
# other code
block_caller { puts "not defaulted"}
# some code
# not defaulted
# other code

yield pode oferecer argumentos para o bloco também

def yield_n(n)
  p = yield n if block_given?
  p || n 
end
yield_n(12) {|n| n + 7 } 
#=> 19 
yield_n(4) 
#=> 4

Embora este seja um exemplo simples, yield pode ser muito útil para permitir acesso direto a variáveis ​​de instância ou avaliações dentro do contexto de outro objeto. Por exemplo:

class Application
  def configuration
    @configuration ||= Configuration.new
    block_given? ? yield(@configuration) : @configuration
  end
end
class Configuration; end

app = Application.new 
app.configuration do |config| 
  puts config.class.name
end
# Configuration
#=> nil 
app.configuration
#=> #<Configuration:0x2bf1d30>

Como você pode ver, usar yield dessa maneira torna o código mais legível do que chamar continuamente app.configuration.#method_name. Em vez disso, você pode realizar toda a configuração dentro do bloco mantendo o código contido.

Variáveis #

Variáveis ​​para blocos são locais ao bloco (semelhante às variáveis ​​de funções), elas morrem quando o bloco é executado.

my_variable = 8
3.times do |x|
    my_variable = x 
    puts my_variable
end
puts my_variable
#=> 0
# 1
# 2
# 8

Blocos não podem ser salvos, eles morrem uma vez executados. Para salvar blocos você precisa usar procs e lambdas.

Processo #

def call_the_block(&calling); calling.call; end

its_a = proc do |*args|
  puts "It's a..." unless args.empty?
  "beautiful day"
end

puts its_a       #=> "beautiful day"
puts its_a.call  #=> "beautiful day"
puts its_a[1, 2] #=> "It's a..." "beautiful day"

Copiamos o método call_the_block do último exemplo. Aqui, você pode ver que um proc é feito chamando o método proc com um bloco. Você também pode ver que blocos, como métodos, têm retornos implícitos, o que significa que procs (e lambdas) também. Na definição de its_a, você pode ver que os blocos podem receber argumentos splat tanto quanto os normais; eles também são capazes de receber argumentos padrão, mas não consegui pensar em uma maneira de trabalhar com isso. Por último, você pode ver que é possível usar várias sintaxes para chamar um método – seja o método call ou o operador [].
