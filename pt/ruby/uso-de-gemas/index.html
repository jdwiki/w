Instalando gemas de rubi #

Este guia pressupõe que você já tenha o Ruby instalado. Se você estiver usando Ruby < 1.9, você terá que instalar RubyGems manualmente, pois não será incluído nativamente.

Para instalar uma gem ruby, digite o comando:

gem install [gemname]

Se você estiver trabalhando em um projeto com uma lista de dependências de gem, elas serão listadas em um arquivo chamado Gemfile. Para instalar uma nova gem no projeto, adicione a seguinte linha de código no Gemfile:

gem 'gemname'

Este Gemfile é usado pelo Bundler gem para instalar as dependências que seu projeto requer, isso significa que você terá que instalar o Bundler primeiro executando (se ainda não o fez):

gem install bundler

Salve o arquivo e execute o comando:

bundle install

Especificando versões #

O número da versão pode ser especificado no comando live, com o sinalizador -v, como:

gem install gemname -v 3.14

Ao especificar números de versão em um Gemfile, você tem várias opções disponíveis:

  • Nenhuma versão especificada (gem 'gemname') – Irá instalar a versão latest que é compatível com outras gems no Gemfile.
  • Versão exata especificada (gem 'gemname', '3.14') – Apenas tentará instalar a versão 3.14 (e falhará se for incompatível com outras gems no Gemfile).
  • Otimista número mínimo da versão (gem 'gemname', '>=3.14') – Só tentará instalar a versão mais recente que é compatível com outras gems no Gemfile, e falhará se nenhuma versão maior ou igual a 3.14 é compatível. O operador > também pode ser usado.
  • Pessimista número mínimo da versão (gem 'gemname', '~>3.14') – Isto é funcionalmente equivalente a usar gem 'gemname', '>=3.14', '<4'. Em outras palavras, apenas o número após o período final pode aumentar.

Como prática recomendada: você pode usar uma das bibliotecas de gerenciamento de versão do Ruby, como rbenv ou [rvm](https://github.com /rvm/rvm). Por meio dessas bibliotecas, você pode instalar diferentes versões de runtimes e gems Ruby de acordo. Portanto, ao trabalhar em um projeto, isso será especialmente útil porque a maioria dos projetos é codificada em uma versão conhecida do Ruby.

Instalação do Gem do github/filesystem #

Você pode instalar uma gem do github ou do sistema de arquivos. Se a gem foi retirada do git ou de alguma forma já no sistema de arquivos, você pode instalá-la usando

gem install --local path_to_gem/filename.gem

Instalando gem do github. Baixe as fontes do github

mkdir newgem
cd newgem
git clone https://urltogem.git

Construa a gema

gem build GEMNAME.gemspec
gem install gemname-version.gem

Verificando se uma gem necessária está instalada de dentro do código #

Para verificar se uma gem necessária está instalada, de dentro do seu código, você pode usar o seguinte (usando nokogiri como exemplo):

begin
  found_gem = Gem::Specification.find_by_name('nokogiri')
  require 'nokogiri'
  ....
  <the rest of your code>
rescue Gem::LoadError
end

No entanto, isso pode ser estendido a uma função que pode ser usada na configuração da funcionalidade em seu código.

def gem_installed?(gem_name)
  found_gem = false
  begin
    found_gem = Gem::Specification.find_by_name(gem_name)
  rescue Gem::LoadError
     return false
  else
    return true
  end
end

Agora você pode verificar se a gem necessária está instalada e imprimir uma mensagem de erro.

if gem_installed?('nokogiri')
  require 'nokogiri'
else
  printf "nokogiri gem required\n"
  exit 1
end

ou

if gem_installed?('nokogiri')
  require 'nokogiri'
else
  require 'REXML'
end

Usando um Gemfile e um Bundler #

Um Gemfile é a maneira padrão de organizar dependências em seu aplicativo. Um Gemfile básico ficará assim:

source 'https://rubygems.org'

gem 'rack'
gem 'sinatra'
gem 'uglifier'

Você pode especificar as versões da gem que deseja da seguinte forma:

# Match except on point release. Use only 1.5.X
gem 'rack', '~>1.5.2'
# Use a specific version.
gem 'sinatra', '1.4.7'
# Use at least a version or anything greater.
gem 'uglifier', '>= 1.3.0'

Você também pode extrair gems diretamente de um repositório git:

# pull a gem from github
gem 'sinatra', git: 'https://github.com/sinatra/sinatra.git'
# you can specify a sha
gem 'sinatra', git: 'https://github.com/sinatra/sinatra.git', sha: '30d4fb468fd1d6373f82127d845b153f17b54c51'
# you can also specify a branch, though this is often unsafe
gem 'sinatra', git: 'https://github.com/sinatra/sinatra.git', branch: 'master'

Você também pode agrupar gemas dependendo de para que elas são usadas. Por exemplo:

group :development, :test do
    # This gem is only available in dev and test, not production.
    gem 'byebug'
end

Você pode especificar em qual plataforma certas gems devem ser executadas se seu aplicativo precisar ser executado em várias plataformas. Por exemplo:

platform :jruby do
  gem 'activerecord-jdbc-adapter'
  gem 'jdbc-postgres'
end

platform :ruby do
  gem 'pg'
end

Para instalar todas as gems de um Gemfile faça:

gem install bundler
bundle install

Bundler/inline (bundler v1.10 e posterior) #

Às vezes você precisa fazer um script para alguém, mas não tem certeza do que ele tem em sua máquina. Existe tudo que seu script precisa? Não se preocupe. O Bundler tem uma ótima função chamada in line.

Ele fornece um método gemfile e antes que o script seja executado ele baixa e requer todas as gems necessárias. Um pequeno exemplo:

require 'bundler/inline' #require only what you need

#Start the bundler and in it use the syntax you are already familiar with
gemfile(true) do 
  source 'https://rubygems.org'
        gem 'nokogiri', '~> 1.6.8.1'
        gem 'ruby-graphviz'
end
