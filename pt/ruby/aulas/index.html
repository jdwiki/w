Sintaxe #

  • nome da classe
  • #algum código descrevendo o comportamento da classe
  • fim

Nomes de classe em Ruby são Constantes, então a primeira letra deve ser maiúscula.

class Cat # correct
end  

class dog # wrong, throws an error
end

Construtor #

Uma classe pode ter apenas um construtor, que é um método chamado initialize. O método é invocado automaticamente quando uma nova instância da classe é criada.

class Customer
  def initialize(name)
     @name = name.capitalize 
  end
end

sarah = Customer.new('sarah')
sarah.name #=> 'Sarah'

Criando uma classe #

Você pode definir uma nova classe usando a palavra-chave class.

class MyClass
end

Uma vez definido, você pode criar uma nova instância usando o método .new

somevar = MyClass.new
# => #<MyClass:0x007fe2b8aa4a18>

Níveis de acesso #

Ruby tem três níveis de acesso. Eles são públicos, privados e protegidos.

Os métodos que seguem as palavras-chave private ou protected são definidos como tal. Os métodos que vêm antes destes são métodos implicitamente públicos.

Métodos públicos #

Um método público deve descrever o comportamento do objeto que está sendo criado. Esses métodos podem ser chamados de fora do escopo do objeto criado.

class Cat
  def initialize(name)
    @name = name
  end

  def speak
    puts "I'm #{@name} and I'm 2 years old"
  end
  
  ...
end

new_cat = Cat.new("garfield")
#=> <Cat:0x2321868 @name="garfield">
 
new_cat.speak
#=> I'm garfield and I'm 2 years old

Esses métodos são métodos ruby ​​públicos, eles descrevem o comportamento para inicializar um novo gato e o comportamento do método speak.

A palavra-chave public é desnecessária, mas pode ser usada para escapar private ou protected

def MyClass
  def first_public_method
  end

  private

  def private_method
  end

  public

  def second_public_method
  end
end

Métodos Privados #

Métodos privados não são acessíveis de fora do objeto. Eles são usados ​​internamente pelo objeto. Usando o exemplo do gato novamente:

class Cat
  def initialize(name)
    @name = name
  end

  def speak
    age = calculate_cat_age # here we call the private method 
    puts "I'm #{@name} and I'm #{age} years old"
  end

  private
     def calculate_cat_age
       2 * 3 - 4 
     end
end

my_cat = Cat.new("Bilbo")
my_cat.speak #=> I'm Bilbo and I'm 2 years old
my_cat.calculate_cat_age #=> NoMethodError: private method `calculate_cat_age' called for #<Cat:0x2321868 @name="Bilbo">

Como você pode ver no exemplo acima, o objeto Cat recém-criado tem acesso ao método calculate_cat_age internamente. Atribuímos a variável age ao resultado da execução do método privado calculate_cat_age que imprime o nome e a idade do gato no console.

Quando tentamos chamar o método calculate_cat_age de fora do objeto my_cat, recebemos um NoMethodError porque é privado. Pegue?

Métodos protegidos #

Os métodos protegidos são muito semelhantes aos métodos privados. Eles não podem ser acessados ​​fora da instância do objeto da mesma forma que os métodos privados não podem. No entanto, usando o método ruby ​​self, métodos protegidos podem ser chamados dentro do contexto de um objeto do mesmo tipo.

class Cat
  def initialize(name, age)
    @name = name
    @age = age
  end

  def speak
    puts "I'm #{@name} and I'm #{@age} years old"
  end

  # this == method allows us to compare two objects own ages. 
  # if both Cat's have the same age they will be considered equal.
  def ==(other)
     self.own_age == other.own_age
  end

  protected
     def own_age
        self.age
     end
end

cat1 = Cat.new("ricky", 2)
=> #<Cat:0x007fe2b8aa4a18 @name="ricky", @age=2>

cat2 = Cat.new("lucy", 4)
=> #<Cat:0x008gfb7aa6v67 @name="lucy", @age=4>

cat3 = Cat.new("felix", 2)
=> #<Cat:0x009frbaa8V76 @name="felix", @age=2>

Você pode ver que adicionamos um parâmetro age à classe cat e criamos três novos objetos cat com o nome e a idade. Vamos chamar o método protegido own_age para comparar as idades dos nossos objetos cat.

cat1 == cat2
=> false

cat1 == cat3
=> true

Veja isso, conseguimos recuperar a idade de cat1 usando o método protegido self.own_age e compará-lo com a idade de cat2 chamando cat2.own_age dentro de cat1.

Acessando variáveis ​​de instância com getters e setters #

Temos três métodos:

  1. attr_reader: usado para permitir ler a variável fora da classe.

  2. attr_writer: permite modificar a variável fora da classe.

  3. attr_accessor: combina os dois métodos.

    class Cat attr_reader :age # you can read the age but you can never change it attr_writer :name # you can change name but you are not allowed to read attr_accessor :breed # you can both change the breed and read it

    def initialize(name, breed) @name = name @breed = breed @age = 2 end def speak puts “I’m #{@name} and I am a #{@breed} cat” end end

    my_cat = Cat.new(“Banjo”, “birman”)

    reading values: #

    my_cat.age #=> 2 my_cat.breed #=> “birman” my_cat.name #=> Error

    changing values #

    my_cat.age = 3 #=> Error my_cat.breed = “sphynx” my_cat.name = “Bilbo”

    my_cat.speak #=> I’m Bilbo and I am a sphynx cat

Observe que os parâmetros são símbolos. isso funciona criando um método.

class Cat
  attr_accessor :breed
end

É basicamente o mesmo que:

class Cat
  def breed
    @breed
  end
  def breed= value
    @breed = value
  end
end

Tipos de métodos de classe #

As classes têm 3 tipos de métodos: métodos de instância, singleton e de classe.

Métodos de instância #

Estes são métodos que podem ser chamados de uma instância da classe.

class Thing
  def somemethod
    puts "something"
  end
end

foo = Thing.new # create an instance of the class
foo.somemethod # => something

Método de classe #

Esses são métodos estáticos, ou seja, podem ser invocados na classe, e não em uma instanciação dessa classe.

class Thing
  def Thing.hello(name)
    puts "Hello, #{name}!"
  end
end

É equivalente a usar self no lugar do nome da classe. O código a seguir é equivalente ao código acima:

class Thing
  def self.hello(name)
    puts "Hello, #{name}!"
  end
end

Invoque o método escrevendo

Thing.hello("John Doe")  # prints: "Hello, John Doe!"

Métodos Singleton #

Eles estão disponíveis apenas para instâncias específicas da classe, mas não para todas.

# create an empty class
class Thing
end

# two instances of the class
thing1 = Thing.new
thing2 = Thing.new

# create a singleton method
def thing1.makestuff
  puts "I belong to thing one"
end

thing1.makestuff # => prints: I belong to thing one
thing2.makestuff # NoMethodError: undefined method `makestuff' for #<Thing>

Ambos os métodos singleton e class são chamados eigenclasses. Basicamente, o que o ruby ​​faz é criar uma classe anônima que contém tais métodos para que não interfira nas instâncias que são criadas.

Outra maneira de fazer isso é pelo construtor class <<. Por exemplo:

# a class method (same as the above example)
class Thing
  class << self # the anonymous class
    def hello(name)
      puts "Hello, #{name}!"
    end
  end
end

Thing.hello("sarah") # => Hello, sarah!

# singleton method

class Thing
end

thing1 = Thing.new

class << thing1
  def makestuff
    puts "I belong to thing one"
  end
end

thing1.makestuff # => prints: "I belong to thing one"

Criação de classe dinâmica #

As classes podem ser criadas dinamicamente através do uso de Class.new.

# create a new class dynamically
MyClass = Class.new

# instantiate an object of type MyClass
my_class = MyClass.new

No exemplo acima, uma nova classe é criada e atribuída à constante MyClass. Esta classe pode ser instanciada e usada como qualquer outra classe.

O método Class.new aceita uma Class que se tornará a superclasse da classe criada dinamicamente.

# dynamically create a class that subclasses another
Staffy = Class.new(Dog)

# instantiate an object of type Staffy
lucky = Staffy.new
lucky.is_a?(Staffy) # true
lucky.is_a?(Dog)    # true

O método Class.new também aceita um bloco. O contexto do bloco é a classe recém-criada. Isso permite que métodos sejam definidos.

Duck = 
  Class.new do
    def quack
      'Quack!!'
    end
  end

# instantiate an object of type Duck
duck = Duck.new
duck.quack # 'Quack!!'

Novo, alocar e inicializar #

Em muitas linguagens, novas instâncias de uma classe são criadas usando uma palavra-chave especial new. Em Ruby, new também é usado para criar instâncias de uma classe, mas não é uma palavra-chave; em vez disso, é um método estático/classe, não diferente de qualquer outro método estático/classe. A definição é mais ou menos esta:

class MyClass
   def self.new(*args)
     obj = allocate
     obj.initialize(*args) # oversimplied; initialize is actually private
     obj
   end
end

allocate executa a verdadeira ‘mágica’ de criar uma instância não inicializada da classe

Observe também que o valor de retorno de initialize é descartado e obj é retornado. Isso torna imediatamente claro porque você pode codificar seu método de inicialização sem se preocupar em retornar self no final.

O método ’novo’ ’normal’ que todas as classes obtêm de ‘Class’ funciona como acima, mas é possível redefini-lo como quiser, ou definir alternativas que funcionem de forma diferente. Por exemplo:

class MyClass
  def self.extraNew(*args)
    obj = allocate
    obj.pre_initialize(:foo)
    obj.initialize(*args)
    obj.post_initialize(:bar)
    obj
  end
end

Variáveis ​​de classe e instância #

Existem vários tipos de variáveis ​​especiais que uma classe pode usar para compartilhar dados mais facilmente.

Variáveis ​​de instância, precedidas por @. Eles são úteis se você quiser usar a mesma variável em métodos diferentes.

class Person
  def initialize(name, age)
    my_age = age # local variable, will be destroyed at end of constructor
    @name = name # instance variable, is only destroyed when the object is
  end

  def some_method
    puts "My name is #{@name}." # we can use @name with no problem
  end
   
  def another_method
    puts "My age is #{my_age}." # this will not work!
  end
end

mhmd = Person.new("Mark", 23)

mhmd.some_method #=> My name is Mark.
mhmd.another_method #=> throws an error

Variável de classe, precedida por @@. Eles contêm os mesmos valores em todas as instâncias de uma classe.

class Person
  @@persons_created = 0 # class variable, available to all objects of this class
  def initialize(name)
    @name = name

    # modification of class variable persists across all objects of this class
    @@persons_created += 1
  end  
      
  def how_many_persons
    puts "persons created so far: #{@@persons_created}"
  end 
end
    
mark = Person.new("Mark")
mark.how_many_persons #=> persons created so far: 1
helen = Person.new("Helen")

mark.how_many_persons #=> persons created so far: 2
helen.how_many_persons #=> persons created so far: 2
# you could either ask mark or helen

Variáveis ​​Globais, precedidas por $. Eles estão disponíveis em qualquer lugar do programa, portanto, certifique-se de usá-los com sabedoria.

$total_animals = 0

class Cat
  def initialize
    $total_animals += 1
  end
end

class Dog
  def initialize
    $total_animals += 1
  end
end

bob = Cat.new()
puts $total_animals #=> 1
fred = Dog.new()
puts $total_animals #=> 2
