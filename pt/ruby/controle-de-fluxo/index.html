if, elsif, else e fim #

Ruby oferece as expressões esperadas if e else para lógica de ramificação, terminadas pela palavra-chave end:

# Simulate flipping a coin
result = [:heads, :tails].sample

if result == :heads
  puts 'The coin-toss came up "heads"'
else
  puts 'The coin-toss came up "tails"'
end

Em Ruby, instruções if são expressões que avaliam um valor, e o resultado pode ser atribuído a uma variável:

status = if age < 18
           :minor
         else
           :adult
         end

Ruby também oferece operadores ternários no estilo C ([veja aqui para detalhes][1]) que podem ser expressos como:

some_statement ? if_true : if_false

Isso significa que o exemplo acima usando if-else também pode ser escrito como

status = age < 18 ? :minor : :adult

Além disso, Ruby oferece a palavra-chave elsif que aceita uma expressão para habilitar lógica de ramificação adicional:

label = if shirt_size == :s
          'small'
        elsif shirt_size == :m
          'medium'
        elsif shirt_size == :l
          'large'
        else
          'unknown size'
        end

Se nenhuma das condições em uma cadeia if/elsif for verdadeira e não houver nenhuma cláusula else, a expressão será avaliada como nil. Isso pode ser útil dentro da interpolação de strings, já que nil.to_s é a string vazia:

"user#{'s' if @users.size != 1}"

[1]: https://www.wikiod.com/pt/ruby/controle-de-fluxo#Operador ternário

Declaração do caso #

Ruby usa a palavra-chave case para instruções switch.

De acordo com os Documentos Ruby:

As instruções case consistem em uma condição opcional, que está no posição de um argumento para case, e zero ou mais cláusulas when. A primeira cláusula when para corresponder à condição (ou para avaliar Verdade booleana, se a condição for nula) “ganha”, e sua estrofe de código é executado. O valor da instrução case é o valor da cláusula when bem-sucedida, ou nil se não houver tal cláusula.

Uma instrução case pode terminar com uma cláusula else. Cada quando um pode ter vários valores candidatos, separados por vírgulas.

Exemplo:

case x
when 1,2,3
  puts "1, 2, or 3"
when 10
  puts "10"
else
  puts "Some other number"
end

Versão mais curta:

case x
when 1,2,3 then puts "1, 2, or 3"
when 10 then puts "10"
else puts "Some other number"
end

O valor da cláusula case corresponde a cada cláusula when usando o método === (não ==). Portanto, pode ser usado com uma variedade de diferentes tipos de objetos.

Uma instrução case pode ser usada com Ranges:

case 17
when 13..19
  puts "teenager"
end

Uma instrução case pode ser usada com um Regexp:

case "google"
when /oo/
  puts "word contains oo"
end

Uma instrução case pode ser usada com um Proc ou lambda:

case 44
when -> (n) { n.even? or n < 0 }
  puts "even or less than zero"
end

Uma instrução case pode ser usada com Classes:

case x
when Integer
  puts "It's an integer"
when String
  puts "It's a string"
end

Ao implementar o método === você pode criar suas próprias classes de correspondência:

class Empty
  def self.===(object)
    !object or "" == object
  end
end

case ""
when Empty
  puts "name was empty"
else
  puts "name is not empty"
end

Uma instrução case pode ser usada sem um valor para corresponder:

case
when ENV['A'] == 'Y'
  puts 'A'
when ENV['B'] == 'Y'
  puts 'B'
else
  puts 'Neither A nor B'
end

Uma instrução case tem um valor, então você pode usá-la como um argumento de método ou em uma atribuição:

description = case 16
              when 13..19 then "teenager"
              else ""
              end

Valores verdadeiros e falsos #

Em Ruby, existem exatamente dois valores que são considerados “falsos”, e retornarão falsos quando testados como condição para uma expressão if. Eles são:

  • nil
  • booleano falso

Todos outros valores são considerados “verdadeiros”, incluindo:

  • 0 - zero numérico (inteiro ou não)
  • "" - Strings vazias
  • "\n" - Strings contendo apenas espaços em branco
  • [] - Matrizes vazias
  • {} - hashes vazios

Tomemos, por exemplo, o seguinte código:

def check_truthy(var_name, var)
  is_truthy = var ? "truthy" : "falsy"
  puts "#{var_name} is #{is_truthy}"
end

check_truthy("false", false)
check_truthy("nil", nil)
check_truthy("0", 0)
check_truthy("empty string", "")
check_truthy("\\n", "\n")
check_truthy("empty array", [])
check_truthy("empty hash", {})

Saída:

false is falsy
nil is falsy
0 is truthy
empty string is truthy
\n is truthy
empty array is truthy
empty hash is truthy

Em linha se/a menos que #

Um padrão comum é usar um if ou unless embutido ou à direita:

puts "x is less than 5" if x < 5

Isso é conhecido como modificador condicional e é uma maneira prática de adicionar código de guarda simples e retornos antecipados:

def save_to_file(data, filename)
  raise "no filename given" if filename.empty?
  return false unless data.valid?

  File.write(filename, data)
end

Não é possível adicionar uma cláusula else a esses modificadores. Além disso, geralmente não é recomendado usar modificadores condicionais dentro da lógica principal – Para código complexo, deve-se usar if, elsif, else normais.

enquanto, até #

Um loop while executa o bloco enquanto a condição dada é atendida:

i = 0
while i < 5
  puts "Iteration ##{i}"
  i +=1
end

Um loop until executa o bloco enquanto a condicional é falsa:

i = 0
until i == 5
  puts "Iteration ##{i}"
  i +=1
end

Operador de atribuição Or-Equals/Condicional (||=) #

Ruby tem um operador or-equals que permite que um valor seja atribuído a uma variável se e somente se essa variável for avaliada como nil ou false.

 ||= # this is the operator that achieves this.

este operador com os tubos duplos representando ou e o sinal de igual representando a atribuição de um valor. Você pode pensar que representa algo assim:

 x = x || y

este exemplo acima não está correto. O operador or-equals na verdade representa isso:

 x || x = y

Se x for avaliado como nil ou false, então x recebe o valor de y e, caso contrário, permanece inalterado.

Aqui está um caso de uso prático do operador ou igual a. Imagine que você tem uma parte do seu código que deve enviar um e-mail para um usuário. O que você faz se, por algum motivo, não houver e-mail para este usuário. Você pode escrever algo assim:

 if user_email.nil?
    user_email = "error@yourapp.com"
 end

Usando o operador or-equals, podemos cortar todo esse pedaço de código, fornecendo controle e funcionalidade limpos e claros.

 user_email ||= "error@yourapp.com"

Nos casos em que false é um valor válido, deve-se tomar cuidado para não substituí-lo acidentalmente:

has_been_run = false
has_been_run ||= true
#=> true

has_been_run = false
has_been_run = true if has_been_run.nil?
#=> false

Operador de flip-flop #

O operador flip-flop .. é usado entre duas condições em uma instrução condicional:

(1..5).select do |e|
  e if (e == 2) .. (e == 4)
end
# => [2, 3, 4]

A condição é avaliada como false até a primeira parte se tornar true. Em seguida, ele é avaliado como true até a segunda parte se tornar true. Depois disso, ele muda para false novamente.

Este exemplo ilustra o que está sendo selecionado:

[1, 2, 2, 3, 4, 4, 5].select do |e|
  e if (e == 2) .. (e == 4)
end
# => [2, 2, 3, 4]

O operador flip-flop só funciona dentro de ifs (incluindo unless) e operador ternário. Caso contrário, está sendo considerado como o operador de intervalo.

(1..5).select do |e|
  (e == 2) .. (e == 4)
end
# => ArgumentError: bad value for range

Ele pode mudar de false para true e para trás várias vezes:

((1..5).to_a * 2).select do |e|
  e if (e == 2) .. (e == 4)
end
# => [2, 3, 4, 2, 3, 4]

a não ser que #

Uma declaração comum é if !(alguma condição). Ruby oferece a alternativa da declaração unless.

A estrutura é exatamente a mesma de uma instrução if, exceto que a condição é negativa. Além disso, a instrução unless não suporta elsif, mas suporta else:

# Prints not inclusive
unless 'hellow'.include?('all')
  puts 'not inclusive'
end

jogar, pegar #

Ao contrário de muitas outras linguagens de programação, as palavras-chave throw e catch não estão relacionadas ao tratamento de exceções em Ruby.

Em Ruby, throw e catch agem um pouco como rótulos em outras linguagens. Eles são usados ​​para alterar o fluxo de controle, mas não estão relacionados a um conceito de “erro” como são as exceções.

catch(:out) do
  catch(:nested) do
    puts "nested"
  end

  puts "before"
  throw :out
  puts "will not be executed"
end
puts "after"
# prints "nested", "before", "after"

Operador ternário #

Ruby tem um operador ternário (?:), que retorna um de dois valores com base se uma condição for avaliada como verdadeira:

conditional ? value_if_truthy : value_if_falsy

value = true
value ? "true" : "false"
#=> "true"

value = false
value ? "true" : "false"
#=> "false"

é o mesmo que escrever if a then b else c end, embora o ternário seja preferido

Exemplos:

puts (if 1 then 2 else 3 end) # => 2

puts 1 ? 2 : 3                # => 2

x = if 1 then 2 else 3 end
puts x                        # => 2

Controle de loop com break, next e redo #

O fluxo de execução de um bloco Ruby pode ser controlado com as instruções break, next e redo.

quebrar #

A instrução break sairá do bloco imediatamente. Quaisquer instruções restantes no bloco serão ignoradas e a iteração terminará:

actions = %w(run jump swim exit macarena)
index = 0

while index < actions.length
  action = actions[index]

  break if action == "exit"

  index += 1
  puts "Currently doing this action: #{action}"
end

# Currently doing this action: run
# Currently doing this action: jump
# Currently doing this action: swim

próximo #

A instrução next retornará ao topo do bloco imediatamente e prosseguirá com a próxima iteração. Quaisquer instruções restantes no bloco serão ignoradas:

actions = %w(run jump swim rest macarena)
index = 0

while index < actions.length
  action = actions[index]
  index += 1

  next if action == "rest"

  puts "Currently doing this action: #{action}"
end

# Currently doing this action: run
# Currently doing this action: jump
# Currently doing this action: swim
# Currently doing this action: macarena

refazer #

A instrução redo retornará ao topo do bloco imediatamente e tentará novamente a mesma iteração. Quaisquer instruções restantes no bloco serão ignoradas:

actions = %w(run jump swim sleep macarena)
index = 0
repeat_count = 0

while index < actions.length
  action = actions[index]
  puts "Currently doing this action: #{action}"

  if action == "sleep"
    repeat_count += 1
    redo if repeat_count < 3
  end

  index += 1
end

# Currently doing this action: run
# Currently doing this action: jump
# Currently doing this action: swim
# Currently doing this action: sleep
# Currently doing this action: sleep
# Currently doing this action: sleep
# Currently doing this action: macarena

Iteração Enumerable #

Além de loops, essas instruções funcionam com métodos de iteração Enumerable, como each e map:

[1, 2, 3].each do |item|
  next if item.even?
  puts "Item: #{item}"
end

# Item: 1
# Item: 3

Valores do resultado do bloco #

Em ambas as instruções break e next, um valor pode ser fornecido e será usado como um valor de resultado do bloco:

even_value = for value in [1, 2, 3]
  break value if value.even?
end

puts "The first even value is: #{even_value}"

# The first even value is: 2

return vs. next: retorno não local em um bloco #

Considere este trecho quebrado:

def foo
  bar = [1, 2, 3, 4].map do |x|
    return 0 if x.even?
    x
  end
  puts 'baz'
  bar
end
foo # => 0

Pode-se esperar que return produza um valor para o array de resultados de bloco do map. Portanto, o valor de retorno de foo seria [1, 0, 3, 0]. Em vez disso, return retorna um valor do método foo. Observe que baz não é impresso, o que significa que a execução nunca atingiu essa linha.

next com um valor faz o truque. Ele atua como um retorno em nível de bloco.

def foo
  bar = [1, 2, 3, 4].map do |x|
    next 0 if x.even?
    x
  end
  puts 'baz'
  bar
end
foo # baz
    # => [1, 0, 3, 0]

Na ausência de um return, o valor retornado pelo bloco é o valor de sua última expressão.

Fluxo de controle com instruções lógicas #

Embora possa parecer contra-intuitivo, você pode usar operadores lógicos para determinar se uma instrução é executada ou não. Por exemplo:

File.exist?(filename) or STDERR.puts "#{filename} does not exist!"

Isso verificará se o arquivo existe e só imprimirá a mensagem de erro se não existir. A instrução or é preguiçosa, o que significa que ela parará de ser executada assim que tiver certeza se seu valor é verdadeiro ou falso. Assim que o primeiro termo for considerado verdadeiro, não há necessidade de verificar o valor do outro termo. Mas se o primeiro termo for falso, deve verificar o segundo termo.

Um uso comum é definir um valor padrão:

glass = glass or 'full' # Optimist!

Isso define o valor de glass para ‘full’ se ainda não estiver definido. Mais concisamente, você pode usar a versão simbólica de or:

glass ||= 'empty' # Pessimist.

Também é possível executar a segunda instrução somente se a primeira for falsa:

File.exist?(filename) and puts "#{filename} found!"

Novamente, and é preguiçoso, então ele só executará a segunda instrução se necessário para chegar a um valor.

O operador or tem precedência menor do que and. Da mesma forma, || tem precedência menor que &&. As formas de símbolos têm maior precedência do que as formas de palavras. Isso é útil para saber quando você deseja misturar essa técnica com atribuição:

a = 1 and b = 2
#=> a==1
#=> b==2

a = 1 && b = 2; puts a, b
#=> a==2
#=> b==2

Observe que o Ruby Style Guide recomenda:

As palavras-chave and e or são proibidas. A legibilidade mínima adicionada não vale a alta probabilidade de introduzir bugs sutis. Para expressões booleanas, sempre use && e ||. Para controle de fluxo, use if e unless; && e || também são aceitáveis, mas menos claros.

começo, fim #

O bloco begin é uma estrutura de controle que agrupa várias instruções.

begin
  a = 7
  b = 6
  a * b
end

Um bloco begin retornará o valor da última instrução do bloco. O exemplo a seguir retornará 3.

begin
  1
  2
  3
end

O bloco begin é útil para atribuição condicional usando o operador ||= onde várias instruções podem ser necessárias para retornar um resultado.

circumference ||=
  begin
    radius = 7
    tau = Math::PI * 2
    tau * radius
  end

Ele também pode ser combinado com outras estruturas de blocos como rescue, assegurar, while, if, unless, etc. para fornecer maior controle do fluxo do programa.

Blocos Begin não são blocos de código, como { ... } ou do ... end; eles não podem ser passados ​​para funções.
