Funções em Ruby fornecem código organizado e reutilizável para realizar um conjunto de ações. As funções simplificam o processo de codificação, evitam lógica redundante e tornam o código mais fácil de seguir. Este tópico descreve a declaração e utilização de funções, argumentos, parâmetros, declarações de rendimento e escopo em Ruby.

Um método é um bloco de código nomeado, associado a um ou mais objetos e geralmente identificado por uma lista de parâmetros além do nome.

def hello(name)
  "Hello, #{name}"
end

Uma invocação de método especifica o nome do método, o objeto no qual ele deve ser invocado (às vezes chamado de receptor) e zero ou mais valores de argumento que são atribuídos aos parâmetros de método nomeados. O valor da última expressão avaliada no método se torna o valor da expressão de invocação do método.

hello("World")
# => "Hello, World"

Quando o receptor não é explícito, é self.

self
# => main

self.hello("World")
# => "Hello, World"

Conforme explicado no livro Ruby Programming Language, muitas linguagens distinguem entre funções, que não têm objeto associado, e métodos, que são invocados em um objeto receptor. Como Ruby é uma linguagem puramente orientada a objetos, todos os métodos são métodos verdadeiros e estão associados a pelo menos um objeto.

Visão geral dos parâmetros do método #

Tipo Assinatura do Método Exemplo de chamada Atribuições
Robrigatório def fn(a,b,c) fn(2,3,5) a=2, b=3, c=5
Variadico def fn(*rest) fn(2,3,5) rest=[2, 3, 5]
D defeito def fn(a=0,b=1) fn(2,3) a=2, b=3
**Palavra-chave def fn(a:0,b:1) fn(a:2,b:3) a=2, b=3

Esses tipos de argumentos podem ser combinados de praticamente qualquer maneira que você possa imaginar para criar funções variádicas. O número mínimo de argumentos para a função será igual à quantidade de argumentos necessários na assinatura. Os argumentos extras serão atribuídos primeiro aos parâmetros padrão, depois ao parâmetro *rest.

Tipo Assinatura do Método Exemplo de chamada Atribuições
R,D,V,R def fn(a,b=1,*mid,z) fn(2,97) a=2, b=1, mid=[], z=97
fn(2,3,97) a=2, b=3, meio=[], z=97
fn(2,3,5,97) a=2, b=3, mid=[5], z=97
fn(2,3,5,7,97) a=2, b=3, mid=[5,7], z=97
R,K,K def fn(a,g:6,h:7) fn(2) a=2, g=6, h=7
fn(2,h:19) a=2, g=6, h=19
fn(2,g:17,h:19) a=2, g=17, h=19
VK def fn(**ks) fn(a:2,g:17,h:19) ks={a:2, g:17, h:19}
fn(quatro:4,cinco:5) ks={quatro:4, cinco:5}

Cedendo a blocos #

Você pode enviar um bloco para o seu método e ele pode chamar esse bloco várias vezes. Isso pode ser feito enviando um proc/lambda ou algo assim, mas é mais fácil e rápido com yield:

def simple(arg1,arg2)
  puts "First we are here:  #{arg1}"
  yield
  puts "Finally we are here:  #{arg2}"
  yield
end
simple('start','end') { puts "Now we are inside the yield" }

#> First we are here:  start
#> Now we are inside the yield
#> Finally we are here:  end
#> Now we are inside the yield

Observe que o { puts ... } não está dentro dos parênteses, ele vem implicitamente depois. Isso também significa que só podemos ter um bloco yield. Podemos passar argumentos para o yield:

def simple(arg)
  puts "Before yield"
  yield(arg)
  puts "After yield"
end
simple('Dave') { |name| puts "My name is #{name}" }

#> Before yield
#> My name is Dave
#> After yield

Com yield podemos facilmente fazer iteradores ou qualquer função que funcione em outro código:

def countdown(num)
  num.times do |i|
    yield(num-i)
  end
end

contagem regressiva(5) { |i| puts "Chamar número #{i}" }

#> Call number 5
#> Call number 4
#> Call number 3
#> Call number 2
#> Call number 1

Na verdade, é com yield que coisas como foreach, each e times são geralmente implementadas em classes.

Se você quiser descobrir se recebeu um bloqueio ou não, use block_given?:

class Employees
  def names
    ret = []
    @employees.each do |emp|
      if block_given?
        yield(emp.name)
      else
        ret.push(emp.name) 
      end
    end

    ret
  end
end

Este exemplo assume que a classe Employees tem uma lista @employees que pode ser iterada com each para obter objetos que tenham nomes de funcionários usando o método name. Se nos for dado um bloco, então vamos render o nome para o bloco, caso contrário, apenas o empurramos para um array que retornamos.

Parâmetros padrão #

def make_animal_sound(sound = 'Cuack')
    puts sound
end

#

make_animal_sound('Mooo') # Mooo
make_animal_sound         # Cuack

É possível incluir padrões para vários argumentos:

def make_animal_sound(sound = 'Cuack', volume = 11)
    play_sound(sound, volume)
end

make_animal_sound('Mooo') # Spinal Tap cow

No entanto, não é possível [fornecer o segundo](http://stackoverflow.com/questions/695431/in-a-method-that-take-multiple-optional-parameters-how-can-any-but-the- first-be) sem também fornecer o primeiro. Em vez de usar parâmetros posicionais, tente parâmetros de palavras-chave:

def make_animal_sound(sound: 'Cuack', volume: 11)
    play_sound(sound, volume)
end

make_animal_sound(volume: 1) # Duck whisper

Ou um parâmetro de hash que armazena opções:

def make_animal_sound(options = {})
    options[:sound]  ||= 'Cuak'
    options[:volume] ||= 11
    play_sound(sound, volume)
end

make_animal_sound(:sound => 'Mooo')

Os valores de parâmetro padrão podem ser definidos por qualquer expressão ruby. A expressão será executada no contexto do método, então você pode até declarar variáveis ​​locais aqui. Observe que não passará pela revisão de código. Cortesia de caius por apontar isso.

def make_animal_sound( sound = ( raise 'TUU-too-TUU-too...' ) ); p sound; end

make_animal_sound 'blaaaa' # => 'blaaaa'
make_animal_sound          # => TUU-too-TUU-too... (RuntimeError)

Parâmetro(s) opcional(is) (operador splat) #

def welcome_guests(*guests)
    guests.each { |guest| puts "Welcome #{guest}!" }
end

#

welcome_guests('Tom')    # Welcome Tom!
welcome_guests('Rob', 'Sally', 'Lucas') # Welcome Rob!
                                        # Welcome Sally!
                                        # Welcome Lucas!

Observe que welcome_guests(['Rob', 'Sally', 'Lucas']) produzirá Welcome ["Rob", "Sally", "Lucas"]! Em vez disso, se você tiver uma lista, você pode fazer welcome_guests(*['Rob', 'Sally', 'Lucas']) e isso funcionará como welcome_guests('Rob', 'Sally', 'Lucas') .

Definindo um método #

Os métodos são definidos com a palavra-chave def, seguida pelo nome do método e uma lista opcional de nomes dos parâmetros entre parênteses. O código Ruby entre def e end representa o body do método.

def hello(name)
  "Hello, #{name}"
end

Uma invocação de método especifica o nome do método, o objeto no qual ele deve ser invocado (às vezes chamado de receptor) e zero ou mais valores de argumento que são atribuídos aos parâmetros de método nomeados.

hello("World")
# => "Hello, World"

Quando o receptor não é explícito, é self.

Os nomes dos parâmetros podem ser usados ​​como variáveis ​​no corpo do método e os valores desses parâmetros nomeados vêm dos argumentos para uma invocação de método.

hello("World")
# => "Hello, World"
hello("All")
# => "Hello, All"

Parâmetro obrigatório único #

def say_hello_to(name)
    puts "Hello #{name}"
end

#

say_hello_to('Charles')    # Hello Charles

Combinação de parâmetros opcionais padrão obrigatórios #

def my_mix(name,valid=true, *opt)
    puts name
    puts valid
    puts opt
end

Ligue da seguinte forma:

my_mix('me')
# 'me'
# true
# []

my_mix('me', false)
# 'me'
# false
# []

my_mix('me', true, 5, 7) 
# 'me'
# true
# [5,7]

Use uma função como um bloco #

Muitas funções em Ruby aceitam um bloco como argumento. Por exemplo.:

[0, 1, 2].map {|i| i + 1}
 => [1, 2, 3]

Se você já tem uma função que faz o que você quer, você pode transformá-la em um bloco usando &method(:fn):

def inc(num)
   num + 1
end

[0, 1, 2].map &method(:inc)
 => [1, 2, 3]

Argumentos de Tupla #

Um método pode pegar um parâmetro de array e desestrutura-lo imediatamente em variáveis ​​locais nomeadas. Encontrado no blog de Mathias Meyer.

def feed( amount, (animal, food) )

    p "#{amount} #{animal}s chew some #{food}"

end

feed 3, [ 'rabbit', 'grass' ] # => "3 rabbits chew some grass"

Vários parâmetros obrigatórios #

def greet(greeting, name)
    puts "#{greeting} #{name}"
end

#

greet('Hi', 'Sophie')    # Hi Sophie

Definições de método são expressões #

Definir um método em Ruby 2.x retorna um símbolo que representa o nome:

class Example
  puts def hello
  end
end

#=> :hello

Isso permite técnicas interessantes de metaprogramação. Por exemplo, os métodos podem ser encapsulados por outros métodos:

class Class
  def logged(name)
    original_method = instance_method(name)
    define_method(name) do |*args|
      puts "Calling #{name} with #{args.inspect}."
      original_method.bind(self).call(*args)
      puts "Completed #{name}."
    end
  end
end

class Meal
  def initialize
    @food = []
  end
  
  logged def add(item)
    @food << item
  end
end

meal = Meal.new
meal.add "Coffee"
# Calling add with ["Coffee"].
# Completed add.

Capturando argumentos de palavras-chave não declaradas (duplo splat) #

O operador ** funciona de forma semelhante ao operador *, mas se aplica a parâmetros de palavras-chave.

def options(required_key:, optional_key: nil, **other_options)
  other_options
end

options(required_key: 'Done!', foo: 'Foo!', bar: 'Bar!')
#> { :foo => "Foo!", :bar => "Bar!" }

No exemplo acima, se o **other_options não for usado, um erro ArgumentError: unknown keyword: foo, bar seria gerado.

def without_double_splat(required_key:, optional_key: nil)
  # do nothing
end

without_double_splat(required_key: 'Done!', foo: 'Foo!', bar: 'Bar!')
#> ArgumentError: unknown keywords: foo, bar

Isso é útil quando você tem um hash de opções que deseja passar para um método e não deseja filtrar as chaves.

def options(required_key:, optional_key: nil, **other_options)
  other_options
end

my_hash = { required_key: true, foo: 'Foo!', bar: 'Bar!' }

options(my_hash)
#> { :foo => "Foo!", :bar => "Bar!" }

Também é possível descompactar um hash usando o operador **. Isso permite que você forneça a palavra-chave diretamente a um método, além de valores de outros hashes:

my_hash = { foo: 'Foo!', bar: 'Bar!' }

options(required_key: true, **my_hash)
#> { :foo => "Foo!", :bar => "Bar!" }
