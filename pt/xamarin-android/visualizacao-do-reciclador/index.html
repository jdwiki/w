Noções básicas do RecyclerView #

Este é um exemplo de uso do Android Support Library V7 RecyclerView. As bibliotecas de suporte são geralmente recomendadas porque fornecem versões compatíveis com versões anteriores de novos recursos, fornecem elementos de interface do usuário úteis que não estão incluídos na estrutura e fornecem uma variedade de utilitários que os aplicativos podem utilizar.

Para obter o RecyclerView, instalaremos os pacotes Nuget necessários. Primeiro, vamos procurar por v7 recyclerview. Role para baixo até ver Biblioteca de suporte do Xamarin Android - v7 RecyclerView. Selecione-o e clique em Adicionar pacote.

Como alternativa, a Android Support Library V7 RecyclerView está disponível como um componente Xamarin. Para adicionar o componente, clique com o botão direito do mouse em Components dentro do projeto Android no Solution Explorer e clique em Get More Components.

Na janela Component Store que aparece, procure RecyclerView. Na lista de pesquisa, selecione Android Support Library V7 RecyclerView. Em seguida, clique em ‘Adicionar ao aplicativo’. O componente é adicionado ao projeto.

O próximo passo é adicionar o RecyclerView a uma página. Dentro do arquivo axml (layout), podemos adicionar RecyclerView como abaixo.

<android.support.v7.widget.RecyclerView
        android:id="@+id/recyclerView"
        android:scrollbars="vertical"
        android:layout_width="match_parent"
        android:layout_height="match_parent" />

O RecyclerView requer que pelo menos duas classes auxiliares sejam configuradas para implementação padrão básica, a saber: Adapter e ViewHolder. Adapter infla layouts de itens e vincula dados a visualizações que são exibidas em um RecyclerView. ViewHolder pesquisa e armazena referências de visualização. O suporte de visualização também ajuda a detectar cliques de visualização de item.

Aqui está um exemplo básico de classe de adaptador

public class MyAdapter : RecyclerView.Adapter
{
    string [] items;

    public MyAdapter (string [] data)
    {
        items = data;
    }

    // Create new views (invoked by the layout manager)
    public override RecyclerView.ViewHolder OnCreateViewHolder (ViewGroup parent, int viewType)
    {   
        // set the view's size, margins, paddings and layout parameters
        var tv = new TextView (parent.Context);
        tv.SetWidth (200);
        tv.Text = "";

        var vh = new MyViewHolder (tv);
        return vh;
    } 

    // Replace the contents of a view (invoked by the layout manager)
    public override void OnBindViewHolder (RecyclerView.ViewHolder viewHolder, int position)
    {
        var item = items [position];

        // Replace the contents of the view with that element
        var holder = viewHolder as MyViewHolder;
        holder.TextView.Text = items[position];
    }

    public override int ItemCount {
        get {
            return items.Length;
        }
    }
}

No método OnCreateViewHolder primeiro inflamos uma View e criamos uma instância da classe ViewHolder. Esta instância deve ser retornada. Este método é invocado pelo Adapter quando requer uma nova instância do ViewHolder. Este método não será invocado para cada célula. Depois que o RecyclerView tiver células suficientes para preencher a exibição, ele reutilizará as células antigas que são roladas para fora da exibição para outras células.

O retorno de chamada OnBindViewHolder é invocado pelo Adapter para exibir os dados na posição especificada. Este método deve atualizar o conteúdo do itemView para refletir o item na posição especificada.

Como a célula contém apenas um único TextView, podemos ter um ViewHolder simples como abaixo.

public class MyViewHolder : RecyclerView.ViewHolder 
{
    public TextView TextView { get; set; }


    public MyViewHolder (TextView v) : base (v)
    {
        TextView = v;
    }
}

O próximo passo é conectar as coisas em Activity.

RecyclerView mRecyclerView;
MyAdapter mAdapter;
protected override void OnCreate (Bundle bundle)
{
    base.OnCreate (bundle);
    SetContentView (Resource.Layout.Main);
    mRecyclerView = FindViewById<RecyclerView> (Resource.Id.recyclerView);

    // Plug in the linear layout manager:
    var layoutManager = new LinearLayoutManager (this) { Orientation = LinearLayoutManager.Vertical };
    mRecyclerView.SetLayoutManager (layoutManager);
    mRecyclerView.HasFixedSize = true;

    var recyclerViewData = GetData();
    // Plug in my adapter:
    mAdapter = new MyAdapter (recyclerViewData);
    mRecyclerView.SetAdapter (mAdapter);
}

string[] GetData()
{ 
     string[] data;
     .
     .
     .
     return data;
}

A classe LayoutManager é responsável por medir e posicionar visualizações de itens em um RecyclerView, bem como determinar a política de quando reciclar visualizações de itens que não são mais visíveis para o usuário. Antes do RecyclerView, tínhamos que usar ListView para organizar células em um s em uma lista de rolagem vertical e GridView para exibir itens em uma grade bidimensional rolável. Mas agora podemos conseguir ambos com o RecyclerView definindo um LayoutManger diferente. LinearLayoutManager organiza as células como em um ListView e GridLayoutManager organiza as células na forma de grade.

RecyclerView com eventos de clique #

Este exemplo mostra como definir Click EventHandlers em um Xamarin.Android RecyclerView.

No Android Java, a maneira de configurar um ouvinte para um clique é usando um onClickListener para a visualização que será clicada, assim:

ImageView picture = findViewById(R.id.item_picture);
picture.setOnClickListener(new View.OnClickListener() {
    public void onClick(View v) {
        // do stuff
    }
});

No entanto, no Xamarin.Android, a maneira de configurar um ouvinte para um evento Click é adicionando um EventHandler, das seguintes maneiras:

1.

ImageView picture = FindViewById<ImageView>(Resource.Id.item_picture);
picture.Click += delegate {
    // do stuff
};

2.

ImageView picture = FindViewById<ImageView>(Resource.Id.item_picture);
picture.Click += async delegate {
    // await DoAsyncMethod();
    // do async stuff
};

3.

ImageView picture = FindViewById<ImageView>(Resource.Id.item_picture);
picture.Click += Picture_Click;
... // rest of your method

private void Picture_Click(object sender, EventArgs e)
{
    // do stuff
}

Observe que o EventHandler é adicionado, não definido. Se o Click EventHandler for adicionado dentro de um método GetView de um adaptador GridView/ListView, ou um método OnBindViewHolder de um RecyclerView.Adapter, toda vez que a exibição do item for criada, um novo EventHandler será adicionado. Depois de rolar várias vezes, vários EventHandlers serão adicionados e, quando a visualização for clicada, todos eles serão acionados.

Para evitar esse problema, os EventHandlers devem ser cancelados e inscritos posteriormente nos métodos GetView ou OnBindViewHolder. Além disso, eles devem usar o número 3. para definir o EventHandler, caso contrário não será possível cancelar a assinatura dos EventHandlers.

Um exemplo de um RecyclerView.Adapter com eventos Click é mostrado abaixo:

public class ViewHolderPerson : Android.Support.V7.Widget.RecyclerView.ViewHolder
{
    public View Item { get; private set; }
    public ImageView Picture { get; private set; }
    public TextView Name { get; private set; }

    public ViewHolderPerson(View itemView) : base(itemView)
    {
        this.Item = itemView;
        this.Picture = itemView.FindViewById<ImageView>(Resource.Id.Item_Person_Picture);
        this.Name = itemView.FindViewById<TextView>(Resource.Id.Item_Person_Name);
    }
}

public class AdapterPersons : Android.Support.V7.Widget.RecyclerView.Adapter
{
    private Context context;
    private Android.Support.V7.Widget.RecyclerView recyclerView;
    private List<Person> persons;

    public AdapterPersons(Context context, Android.Support.V7.Widget.RecyclerView recyclerView, List<Person> persons)
    {
        this.context = context;
        this.recyclerView = recyclerView;
        this.persons = persons;
    }

    public override int ItemCount => persons.Count;

    public override void OnBindViewHolder(RecyclerView.ViewHolder holder, int position)
    {
        Person person = this.persons[position];
        ((ViewHolderPerson)holder).Name.Text = person.Name;
        ((ViewHolderPerson)holder).Picture.SetImageBitmap(person.Picture);

        // Unsubscribe and subscribe the method, to avoid setting multiple times.
        ((ViewHolderPerson)holder).Item.Click -= Person_Click;
        ((ViewHolderPerson)holder).Item.Click += Person_Click;
    }

    private void Person_Click(object sender, EventArgs e)
    {
        int position = this.recyclerView.GetChildAdapterPosition((View)sender);
        Person personClicked = this.persons[position];
        if(personClicked.Gender == Gender.Female)
        {
            Toast.MakeText(this.context, "The person clicked is a female!", ToastLength.Long).Show();
        }
        else if(personClicked.Gender == Gender.Male)
        {
            Toast.MakeText(this.context, "The person clicked is a male!", ToastLength.Long).Show();
        }
    }

    public override RecyclerView.ViewHolder OnCreateViewHolder(ViewGroup parent, int viewType)
    {
        View itemView = LayoutInflater.From(parent.Context).Inflate(Resource.Layout.item_person, parent, false);
        return new ViewHolderPerson(itemView);
    }

}
