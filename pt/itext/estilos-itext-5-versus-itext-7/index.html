Estilos iText 5 versus iText 7

Criar um documento no qual você tem que alternar entre estilos freqüentemente tende a ser tedioso no iText 5. Você precisa criar muitos objetos Chunk e você sempre tem que fazer uma troca entre aplicar os estilos diretamente a cada novo `` Chunkou criando um método auxiliar que cria oChunk` para você.

O que corrigimos no iText 7:

Agora é possível encadear métodos. Os métodos setFont(), setFontSize(), addStyle() e outros métodos retornam o objeto no qual são invocados. Adicionar um Parágrafo envolvendo estilos diferentes agora pode ser feito em uma linha:

document.add(
    new Paragraph()
        .add("In this example, named ")
        .add(new Text("HelloWorldStyles").addStyle(style))
        .add(", we experiment with some text in ")
        .add(new Text("code style").addStyle(style))
        .add("."));

Usando o objeto Style, agora você também pode aplicar diferentes propriedades (fonte, cor da fonte, cor de fundo, tamanho da fonte,…) de uma só vez com o método addStyle().

Quer saber mais?

Leia Introducing the PdfFont class que é o capítulo 1 do iText 7: Building Blocks tutorial. Obtenha o e-book gratuito!

##HelloWorldStyles.java (iText 5) Neste exemplo do iText 5, precisamos alternar entre estilos diferentes no mesmo documento:

A melhor maneira de fazer isso no iText 5 é criar um método de conveniência que crie um Chunk no estilo que precisa ser usado com frequência; veja o método createBgChunk():

public Chunk createBgChunk(String s, Font font) {
    Chunk chunk = new Chunk(s, font);
    chunk.setBackground(BaseColor.LIGHT_GRAY);
    return chunk;
}

Agora podemos usar este método no código que cria o PDF:

public void createPdf(String dest)
throws DocumentException, IOException {
    Document document = new Document();
    PdfWriter.getInstance(document, new FileOutputStream(dest));
    document.open();
    Font code = new Font(FontFamily.COURIER, 12, Font.NORMAL, BaseColor.RED);
    Paragraph p = new Paragraph("In this example, named ");
    p.add(createBgChunk("HelloWorldStyles", code));
    p.add(", we experiment with some text in ");
    p.add(createBgChunk("code style", code));
    p.add(".");
    document.add(p);
    document.close();
}

Fonte: developers.itextpdf.com

##HelloWorldStyles.java (iText 7) Neste exemplo do iText 7, precisamos alternar entre diferentes estilos no mesmo documento:

A melhor maneira de conseguir isso no iText 7 é criar um objeto Style e aplicar esse Style a um objeto Text:

public void createPdf(String dest) throws IOException {
    PdfDocument pdf = new PdfDocument(new PdfWriter(dest));
    PdfFont code = PdfFontFactory.createFont(FontConstants.COURIER);
    Style style = new Style()
        .setFont(code)
        .setFontSize(12)
        .setFontColor(Color.RED)
        .setBackgroundColor(Color.LIGHT_GRAY);
    try (Document document = new Document(pdf)) {
        document.add(
            new Paragraph()
                .add("In this example, named ")
                .add(new Text("HelloWorldStyles").addStyle(style))
                .add(", we experiment with some text in ")
                .add(new Text("code style").addStyle(style))
                .add("."));
    }
}

Fonte: developers.itextpdf.com e o iText 7: Blocos de construção tutorial.
