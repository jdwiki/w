Tabelas iText 5 versus iText 7

Os nomes das classes do iText 5 PdfPTable e PdfPCell foram escolhidos porque já tínhamos classes chamadas Table e Cell para criar objetos de tabela e célula no mais alto nível de programação. Havia também uma classe chamada PdfTable para ser usada pelo iText internamente. Essas classes tinham muitas falhas e foram preteridas em favor de PdfPTable e PdfPCell. Eles foram removidos há muito tempo.

Ao longo dos anos, PdfPTable e PdfPCell também receberam algumas críticas dos usuários. Por exemplo: os usuários não entenderam a diferença entre o modo texto e o modo composto.

Modo de texto é usado quando você cria um PdfPCell como este:

cell = new PdfPCell(new Phrase("Cell with rowspan 2"));

Neste caso, você define propriedades como o alinhamento horizontal no nível do PdfPCell.

Modo composto entra em ação no momento em que você usa o método addElement():

cell = new PdfPCell();
cell.addElement(new Phrase("Cell 1.2"));

Neste caso, algumas propriedades definidas no nível do PdfPCell (como o alinhamento horizontal) são ignoradas. O alinhamento horizontal deve ser definido ao nível dos elementos adicionados à célula. Por exemplo: se você deseja criar uma célula na qual diferentes parágrafos precisam ter um alinhamento horizontal diferente, você alternará para o modo composto.

Se você observar a captura de tela da tabela criada com o exemplo do iText 5, notará que as células com conteúdo Cell 1.1 (adicionada no modo texto) e Cell 1.2 (adicionada no modo composto) estão alinhados de forma bastante diferente.

Em resposta às críticas sobre o alinhamento ímpar, introduzimos métodos para usar informações ascendentes e descendentes. Usamos esses métodos para as células com conteúdo Cell 2.1 (adicionado no modo de texto) e Cell 2.2 (adicionado no modo composto). Também introduzimos um preenchimento de 5 para essas células.

Agora o resultado é muito melhor.

O que corrigimos no iText 7:

Como criamos o iText 7 do zero, não tínhamos classes legadas com nomes que não pudéssemos reutilizar. Introduzimos uma nova classe Table e uma nova classe Cell.

Não há mais modo de texto e não mais modo composto. Um Cell é criado sem parâmetros ou com parâmetros que definem o rowspan e o colspan. Todo o conteúdo é adicionado da mesma forma: usando o método add().

Nossos clientes também estavam pedindo para fornecer um meio de distinguir uma margem e um preenchimento. No exemplo do iText 7, adicionamos um plano de fundo cinza para mostrar a diferença. Na célula com conteúdo Célula 2.1, definimos uma margem de 5 unidades de usuário. O preenchimento padrão é 2. Na célula com conteúdo Célula 2.2, definimos um preenchimento de 5 unidades de usuário, a margem padrão em 0.

Como você pode ver nas capturas de tela, as células são renderizadas muito bem. Não tivemos que usar métodos para definir o ascendente ou descendente. O comportamento padrão é muito mais próximo do comportamento que um desenvolvedor esperaria.

##HelloWorldTable.java (iText 5) Neste exemplo, criaremos a seguinte tabela usando o iText 5:

Precisamos das classes PdfPTable e PdfPCell para conseguir isso:

public void createPdf(String dest) throws IOException, DocumentException {
    Document document = new Document();
    PdfWriter.getInstance(document, new FileOutputStream(dest));
    document.open();
    PdfPTable table = new PdfPTable(3);
    PdfPCell cell = new PdfPCell(new Phrase("Cell with colspan 3"));
    cell.setColspan(3);
    cell.setHorizontalAlignment(Element.ALIGN_CENTER);
    table.addCell(cell);
    cell = new PdfPCell(new Phrase("Cell with rowspan 2"));
    cell.setRowspan(2);
    cell.setVerticalAlignment(Element.ALIGN_MIDDLE);
    table.addCell(cell);
    table.addCell("Cell 1.1");
    cell = new PdfPCell();
    cell.addElement(new Phrase("Cell 1.2"));
    table.addCell(cell);
    cell = new PdfPCell(new Phrase("Cell 2.1"));
    cell.setPadding(5);
    cell.setUseAscender(true);
    cell.setUseDescender(true);
    cell.setHorizontalAlignment(Element.ALIGN_CENTER);
    table.addCell(cell);
    cell = new PdfPCell();
    cell.setPadding(5);
    cell.setUseAscender(true);
    cell.setUseDescender(true);
    Paragraph p = new Paragraph("Cell 2.2");
    p.setAlignment(Element.ALIGN_CENTER);
    cell.addElement(p);
    table.addCell(cell);
    document.add(table);
    document.close();
}

Fonte: developers.itextpdf.com

##HelloWorldTable.java (iText 7) Neste exemplo, criaremos a seguinte tabela usando o iText 7:

Vamos precisar das classes Table e Cell para conseguir isso:

public void createPdf(String dest) throws IOException {
    PdfDocument pdf = new PdfDocument(new PdfWriter(dest));
    try (Document document = new Document(pdf)) {
        Table table = new Table(3);
        Cell cell = new Cell(1, 3)
            .setTextAlignment(TextAlignment.CENTER)
            .add("Cell with colspan 3");
        table.addCell(cell);
        cell = new Cell(2, 1)
            .add("Cell with rowspan 2")
            .setVerticalAlignment(VerticalAlignment.MIDDLE);
        table.addCell(cell);
        table.addCell("Cell 1.1");
        table.addCell(new Cell().add("Cell 1.2"));
        table.addCell(new Cell()
            .add("Cell 2.1")
            .setBackgroundColor(Color.LIGHT_GRAY)
            .setMargin(5));
        table.addCell(new Cell()
            .add("Cell 1.2")
            .setBackgroundColor(Color.LIGHT_GRAY)
            .setPadding(5));
        document.add(table);
    }
}

Fonte: developers.itextpdf.com e [iText 7: Building Blocks](http:// pages.itextpdf.com/iText-7-abc.html) tutorial.
