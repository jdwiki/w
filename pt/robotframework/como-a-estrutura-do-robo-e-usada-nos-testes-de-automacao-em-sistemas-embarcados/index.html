A estrutura do robô é amplamente utilizada em testes de automação de produtos incorporados. Vamos pegar um produto Embedded como exemplo e ver como automatizar os casos de teste usando o Robot Framework.

Abreviação:

  • RPS - Fonte de alimentação remota
  • RF - Trabalho de estrutura do robô

Teste de fonte de alimentação remota #

Simulação de fonte de alimentação remota #

Como não temos um hardware de fonte de alimentação remota real, vamos simulá-lo usando o programa python.

Ideia básica sobre RPS #

  • Na verdade, a fonte de alimentação remota possui um servidor http.
  • O usuário pode enviar comandos para ligar/desligar a fonte de alimentação usando solicitação http.

Vamos simular uma fonte de alimentação remota usando o seguinte programa rps-server.py.

from flask import Flask, request
from flask_httpauth import HTTPBasicAuth


app = Flask(__name__)
auth = HTTPBasicAuth()

users = {
        'admin': '12345678'
}
app.url_map.strict_slashes = False

PINS = ['P60', 'P61', 'P62', 'P63']

PINS_STATUS = {'P60':'0', 'P61': '0', 'P62':'0', 'P63':'0'}

@auth.get_password
def get_pw(username):
        if username in users:
                return users.get(username)
        return None

@app.route('/')
@auth.login_required
def index():
        return "Hello, %s!" % auth.username()

def get_html_string():
        html_str = '<html>P60={}P61={}P62={}P63={}</html>'.format(PINS_STATUS['P60'],
                                                PINS_STATUS['P61'],
                                            PINS_STATUS['P62'],
                                            PINS_STATUS['P63'])
        return html_str

def parse_cmd_args(args):
        global current_status
        if str(args['CMD']) == 'SetPower':
                for key in args:
                        if key in PINS:
                                PINS_STATUS[key] = str(args[key])
                            
                return get_html_string()
    
        if str(args['CMD']) == 'GetPower':
                return get_html_string()
   

    @app.route('/SetCmd', methods=['GET','POST'])
    def rps():
        if request.method=="GET":
                args=request.args.to_dict()
                ret = parse_cmd_args(args)
                return ret

O código acima simula o servidor http para controlar a fonte de alimentação remota.

Como executar o servidor RPS? #

$ export FLASK_APP=rps-server.py
$ flask run

Como enviar comandos para o servidor rps? #

A seguir estão os dois comandos usados ​​para controlar o RPS

  1. Definir Potência
  2. Obtenha poder

Por padrão, o servidor estará escutando na porta 5000.

As portas de alimentação são,

  1. P60
  2. P61
  3. P62
  4. P64

Os estados dos portos são,

  1. LIGADO - 1
  2. DESLIGADO - 0

Requisitos #

Os requisitos para construir uma fonte de alimentação remota são

  1. A fonte de alimentação remota deve ser capaz de LIGAR/DESLIGAR remotamente
  2. O status da fonte de alimentação remota pode ser acessado remotamente.

Derivando casos de teste #

Casos de teste derivados do requisito

  1. Ligue a fonte de alimentação 2 remotamente.
  2. Verifique se a fonte de alimentação 2 está ligada.
  3. Desligue a fonte de alimentação 2 remotamente.
  4. Verifique se a fonte de alimentação 2 está desligada.

Testes manuais #

  • Execute o servidor rps.
  • Para ativar a porta 3, abra um navegador e forneça o seguinte URI

http://admin:12345678@localhost:5000/SetCmd?CMD=SetPower&P62=1

  • Para obter o status de todas as portas

http://admin:12345678@localhost:5000/SetCmd?CMD=GetPower

Biblioteca de testes de escrita #

Precisamos escrever uma biblioteca de teste em python para enviar comandos http usando a solicitação http. Mais tarde, usaremos esta biblioteca como palavras-chave no trabalho de estrutura do robô.

comandos.py #

Vamos usar a biblioteca de comandos.py para enviar SetPower e GetPower.

import requests
import re


class commands(object):

    ROBOT_LIBRARY_SCOPE = 'GLOBAL'
    def __init__(self, ip='localhost:5000'):
        self.ip_address = ip
        self.query = {}
        self.user = 'admin'
        self.passw = '12345678'

    def form_query(self, state, cmd, port):
        port = self.get_port_no(port)
        self.query = {port: state}
        return self.query

    def get_port_no(self, port_no):
        port = 'P6' + str(port_no)
        return port

    def clean_html(self, data):
        exp = re.compile('<.*?>')
        text = re.sub(exp, "", data)
        return text.rstrip()

    def send_cmds(self, cmd, port=None, state=None):
        url = 'http://{}:{}@{}/SetCmd?CMD={}'\
              .format(self.user,
                      self.passw,
                      self.ip_address,
                      cmd)
        print url
        if cmd == 'SetPower':
            self.form_query(state, cmd, port)
            self.req = requests.get(url, params=self.query)
            return True
        elif cmd == 'GetPower':
            self.req = requests.get(url)
            data = self.clean_html(self.req.text)
            return data
        else:
            return False

        return self.req.text

Documentação de palavras-chave do Python #

  1. send_cmds(cmd, port=None, state=None) é a função que vamos usar.
  2. Ao usar esta função na palavra-chave Robot, não há necessidade de se preocupar com _, ou minúsculas ou maiúsculas no nome da função.

A função Python ficará assim enquanto estiver usando como palavra-chave,

Send Cmds       cmd   port  state

Palavras-chave de teste de redação #

Vamos usar Send Cmds como palavra-chave python no conjunto de testes.

  • Os comandos de envio RPS usam os seguintes quatro argumentos para definir a energia

    • command = SetPower
    • port = 2
    • state = 1 for ON / 0 for off Quando chamamos esse comando, ele liga/desliga a fonte de alimentação

  • RPS get power retornará o status de todas as portas da fonte de alimentação

    *** Keywords *** RPS send commands [Arguments] ${command} ${port} ${state} ${output}= Send cmds ${command} ${port} ${state} [return] ${output}

    RPS get Power [Arguments] ${command} ${output}= Send cmds ${command} [return] ${output}}

Algoritmo para testar a fonte de alimentação #

  1. Defina a energia para uma porta
  2. Verifique o status do cmd
  3. Obtenha o status da porta e verifique se está ON/OFF
