Instalação ou configuração #

Janelas #

  1. Verifique se você tem o JDK correto. Você pode verificá-lo abrindo o prompt de comando (pressione tecla windows e escreva cmd). No prompt de comando digite javac -version, isso mostrará a versão atual do JDK instalada em sua correspondência ou um erro* se o Java estiver ausente. Se o JDK não estiver disponível ou a versão for inferior a 1.8, baixe o Java SE Development Kit 8.

  2. Baixe o Android Studio mais recente.

  3. Inicie o arquivo .exe baixado.

  4. Siga o assistente para instalar o Android Studio

  5. Após a conclusão da instalação, abra o Android Studio a partir do atalho que foi feito na área de trabalho durante o processo de instalação.

  6. Quando você abre o Android Studio pela primeira vez, ele pode solicitar que você copie suas configurações anteriores, mas como é sua primeira vez, você pode simplesmente optar por não copiar nada.

  7. Em seguida, o Android Studio solicita o download das APIs necessárias para desenvolver aplicativos Android. Faça o download deles, depois de baixar o IDE será aberto e você poderá escrever seu primeiro aplicativo.

Mac #

  1. Verifique se você tem o JDK correto. Você pode verificar abrindo o terminal (pressione command + espaço e escreva terminal). Na linha de comando digite javac -version, isso mostrará a versão atual do JDK instalada em sua correspondência ou um erro* se o Java estiver ausente. Se o JDK não estiver disponível ou a versão for inferior a 1.8, baixe o Java SE Development Kit 8. **

  2. Baixe o Android Studio mais recente.

  3. Inicie o arquivo .dmg baixado.

  4. Arraste e solte o Android Studio na pasta Aplicativos e inicie o Android Studio.

  5. Abra o Android Studio.

  6. Quando você abre o Android Studio pela primeira vez, ele pode solicitar que você copie suas configurações anteriores, mas como é sua primeira vez, você pode simplesmente optar por não copiar nada.

  7. Em seguida, o Android Studio solicita o download das APIs necessárias para desenvolver aplicativos Android. Faça o download deles, depois de baixar o IDE será aberto e você poderá escrever seu primeiro aplicativo.

#Linux

  1. Verifique se você tem o JDK correto. Você pode verificar abrindo o terminal (pressione comando + espaço e escreva terminal). Na linha de comando digite javac -version, isso mostrará a versão atual do JDK instalada em sua correspondência ou um erro* se o Java estiver ausente. Se o JDK não estiver disponível ou a versão for inferior a 1.8, baixe o Java SE Development Kit 8. **

  2. Baixe o Android Studio mais recente.

  3. Descompacte/extraia o Android Studio em uma pasta específica.

  4. Abra o terminal e siga o caminho onde você extraiu o Android Studio. (Então, use o comando cd para entrar na pasta do Android Studio.) Depois de entrar, precisamos entrar na pasta bin, então, novamente, use o comando cd bin e digite.

  5. Agora precisamos alterar o mod do nosso arquivo necessário, ou seja, studio.sh, para isso digite o comando sudo chmod 777 -R studio.sh , pressione enter e escreva sua senha (se houver) e digite. (Também você pode ver a lista de arquivos presentes dentro do bin pelo comando ls.).

  6. Após alterar o mod, basta executar o arquivo .studio.sh para isso digite o comando ./studio.sh

  7. Quando você executa o comando acima, o Android Studio inicia pela primeira vez, ele pode solicitar que você copie suas configurações anteriores, mas como é sua primeira vez, você pode simplesmente optar por não copiar nada.

  8. Em seguida, o Android Studio solicita o download das APIs necessárias para desenvolver aplicativos Android. Faça o download deles, depois de baixar o IDE será aberto e você poderá escrever seu primeiro aplicativo.

Você pode encontrar “não é possível executar a ferramenta mksdcard SDK” quando estiver instalando o Android Studio em um sistema operacional Ubuntu de 64 bits porque o Studio requer cerca de 32 binários. Para superar esse erro, termine e feche o Android Studio e vá para o terminal e execute sudo apt-get install lib32z1 lib32ncurses5 lib32bz2-1.0 lib32stdc++6 . Depois que a instalação desses binários estiver concluída, volte para a etapa 6 e retome o processo de instalação.

Notas #

  • Se você já instalou o JDK e ainda está recebendo, certifique-se de ter definido JAVA_HOME em suas Variáveis ​​do Sistema. Você pode verificar esta resposta sobre como configurar um.

  • No entanto, existem problemas de estabilidade conhecidos no Android Studio no Mac ao usar o JDK 1.8. Até que esses problemas sejam resolvidos, você pode melhorar a estabilidade fazendo downgrade do seu JDK para uma versão mais antiga (mas não inferior ao JDK 1.6).

  • Enquanto o download do Android Studio for concluído, verifique qual versão do JDK você possui: abra uma linha de comando e digite javac -version. Se o JDK não estiver disponível ou a versão for inferior a 1.8, baixe o [Java SE Development Kit 8](http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads- 2133151.html).

  • Se você estiver executando uma versão de 64 bits do Ubuntu, precisará instalar algumas bibliotecas de 32 bits com os seguintes comandos:

    sudo dpkg --add-architecture i386
    sudo apt-get update
    sudo apt-get install lib32z1 lib32ncurses5 libbz2-1.0:i386 lib32stdc++6

  • Se você estiver executando o Fedora de 64 bits, o comando é:

    sudo yum install zlib.i686 ncurses-libs.i686 bzip2-libs.i686

Problema com o download #

  • Se você enfrentar qualquer problema como a mensagem de que seu sistema não é compatível com o Android Studio (pode acontecer quando você estiver usando um navegador da Web diferente do Chrome), baixe o Android Studio em: http://tools.android.com/

Visualize diferentes tamanhos de tela (dispositivos) e orientações #

1. Visualize Diferentes Dispositivos #

Há um painel de visualização à direita do estúdio Android. Neste painel há um botão com o nome do dispositivo com o qual você está visualizando a UI do seu aplicativo assim .

Clique no pequeno indicador suspenso disso e um painel flutuante aparecerá com todos os dispositivos predefinidos. Você pode selecionar qualquer um deles para visualizar seu aplicativo UI com diferentes dispositivos mencionando seus tamanhos de tela.
