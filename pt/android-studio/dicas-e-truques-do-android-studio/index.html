Emulador para teste #

Se não houver necessidades específicas, escolha imagens do sistema x86_64 com APIs do Google para criar o emulador.

Ele funciona muito mais rápido que o armeabi-v7a em computadores baseados em Intel x86 x64.

Existem algumas bibliotecas SDK compiladas e projetadas com arquitetura ARM.

Se você tentar instalá-los em emuladores baseados em Intel, receberá uma mensagem de erro dizendo

“INSTALL_FAILED_NO_MATCHING_ABIS”

Você pode optar por um emulador de terceiros como [Genymotion] 1 ou emulador de Android autônomo do Visual Studio nessas ocasiões.

Modelo ao vivo personalizado #

Exemplo : #

Para atingir esse objetivo, você deve adicionar um modelo dinâmico personalizado: #

  1. Abra Configurações [Ctrl + Alt + S ]
  2. Digite “Modelos ao vivo” na barra de pesquisa superior esquerda.
  3. Clique em + para adicionar um “Grupo de Modelos” e digite um nome de grupo (por exemplo: MeuModelo) para continuar

4. Escolha seu grupo de modelos personalizados (por exemplo: MyTemplate), clique em + e escolha “Live Template”. 5. Vou usar “dizer” por exemplo: >say -> startActivity(new Intent($from$.this,$to$.class)); 6. Digite a chave que deseja simplificar em “Abbreviation” (por exemplo: diga), e digite a instrução no “Texto do modelo” (por exemplo:startActivity(new Intent($from$.this,$to$.class ));)

7. Clique no botão “definir” na parte inferior esquerda e escolha a situação (por exemplo: Java) 8. Clique em “Editar variáveis”, defina a expressão. (veja o detalhe: Diálogo Editar Variáveis ​​do Modelo)

9. Clique em “OK” e “Aplicar”. E tente digitar “dizer” no seu editor.

Use estilos de código personalizados, compartilhe com outros membros da equipe e formate automaticamente com atalho #

É possível ter seus próprios estilos de código personalizados, compartilhá-los com outros membros da equipe e usar um atalho para formatar automaticamente o código em um arquivo.

Para criar seu próprio estilo de código personalizado, acesse: Preferências -> Editor -> Estilo de Código

Existem algumas configurações gerais de estilo de código aqui. Você também pode selecionar o idioma (Java para Android) e definir os parâmetros como achar melhor. Existem muitas configurações.

Depois de fazer isso, salve seu estilo de código. Isso é apenas para manter a segurança.

Depois de salvá-lo, selecione “Copiar para projeto” na caixa de diálogo “Gerenciar”. Você será perguntado se deseja mudar para este esquema criado. Responda sim.

Feito isso, feche a caixa de diálogo de preferências de estilo de código.

Agora, verifique se suas configurações de estilo de código foram salvas em: .idea/codeStyleSettings.xml

Certifique-se de que esse arquivo não seja ignorado em seu sistema de controle de versão para que você possa compartilhá-lo com seus colegas de equipe.

Depois que seus colegas de equipe tiverem esse arquivo, eles também deverão ter as mesmas configurações.

Agora, ao editar um arquivo, você pode formatar o código selecionando:

Código -> Reformatar Código

Atalhos para reformatar código (retirado de this resposta - veja a resposta para obter detalhes sobre como resolver problemas com atalhos no Ubuntu):

Ganhar

Ctrl + Alt + L

Linux:

Ctrl + Shift + Alt + L

Mac:

Opção + Comando + L

Quando você executa uma reformatação de código, uma pequena caixa de diálogo deve aparecer informando sobre as alterações feitas. Você pode clicar em “Mostrar” nesta caixa de diálogo para abrir a “Diálogo de reformatar arquivo”.

Você também pode abrir esta caixa de diálogo a partir do menu Código e seu atalho correspondente.

Esteja avisado que “Somente o texto alterado do VCS” nem sempre funciona dependendo de como o código foi editado (pode ignorar uma regra se parte do código não tiver sido editada).

Você também pode selecionar o texto e mostrar a caixa de diálogo do código de reformatação para formatar apenas o texto selecionado.
