que #

Para determinar onde em seu sistema existe um executável em seu caminho, use o comando which:

$ which python
$

Se não houver resposta, esse executável não existe em seu caminho. O sistema simplesmente retornará um novo prompt sem uma mensagem de erro. Se o executável existir no seu caminho, ele mostrará o diretório onde ele realmente existe:

$ which ls
/bin/ls

Isso pode ser útil para determinar por que o comportamento não corresponde à expectativa, garantindo que você esteja executando a versão do executável que você acha que está. Por exemplo, se você tiver o Python 2 e o Python 3 instalados, ambos podem ser executados digitando python no terminal - mas o executável que está sendo executado pode ser diferente do esperado. Como mostrado acima, este comando funcionará para qualquer comando unix padrão, que é suportado por executáveis ​​individuais.

pwd - imprime o diretório de trabalho #

$> pwd
/home/myUserHome
$> cd ..
$> pwd
/home

imprimirá o caminho atual para o console.

comandos de manipulação de arquivos #

Lista de comandos que serão introduzidos aqui:

ls     #view contents of a directory
touch  #create new file
mkdir  #create new directory
cp     #copy contents of one file to another
mv     #move file from one location to another
rm     #delete a file or directory

ls exemplos

jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     test.cpp

mostra o diretório atual

jennifer@my_computer:~/Desktop$ ls c++\ projects
DNA_analysis.cpp        encryption.cpp  pool_game.cpp

mostra o diretório “projetos c++”. Os caracteres de espaço nos nomes dos arquivos são digitados como “".

exemplo de toque

jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     test.cpp
jennifer@my_computer:~/Desktop$ touch ruby_test.rb
jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     ruby_test.rb    test.cpp

exemplo mkdir

jennifer@my_computer:~/Desktop$ mkdir ruby
jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     ruby    ruby_test.rb    test.cpp
jennifer@my_computer:~/Desktop$ cd ruby
jennifer@my_computer:~/Desktop/ruby$ ls
<nothing>
jennifer@my_computer:~/Desktop/ruby

Na verdade, ele não imprime <nada>. É apenas como estou representando que não produz nada

exemplos de cp

jennifer@my_computer:~/Desktop/ruby$ cd ..
jennifer@my_computer:~/Desktop$ cp test.cpp c++_test.cpp
jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby_test.rb
test.cpp

Isto é quando o último argumento para cp, neste caso “c++_test.cpp” não é um diretório existente. cp criará um arquivo chamado “c++_test.cpp”, com conteúdo idêntico ao de “test.cpp”. Se c++_test.cpp já existisse, o cp teria deletado o que estava anteriormente antes de copiar o conteúdo de “test.cpp”.

jennifer@my_comptuer:~/Desktop$ ls ruby
<nothing>
jennifer@my_computer:~/Desktop$ cp ruby_test.rb ruby
jennifer@my_computer:~/Desktop$ ls ruby
ruby_test.rb

Isto é o que acontece quando o último argumento para cp, neste caso “ruby”, é um diretório. cp cria um arquivo com o mesmo nome de “ruby_test.rb”, mas no diretório “ruby”.

exemplos de MV

jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby_test.rb
test.cpp
jennifer@my_computer:~/Desktop$ mv ruby_test.rb ruby\ test.rb
jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby test.rb
test.cpp

Isto é o que acontece quando o último argumento para mv, neste caso “ruby test.rb”, não é um diretório existente. O arquivo “ruby_test.rb” foi renomeado para “ruby test.rb”. Se “ruby test.rb” já existisse, ele teria sido sobrescrito Observe, novamente, que os espaços são precedidos por um ‘'.

jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby test.rb
test.cpp
jennifer@my_computer:~/Desktop$ ls c++\ projects
DNA_analysis.cpp        encryption.cpp  pool_game.cpp
jennifer@my_computer:~/Desktop$ mv test.cpp c++\ projects
jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby test.rb
jennifer@my_computer:~/Desktop$ ls c++\ projects
DNA_analysis.cpp        encryption.cpp  pool_game.cpp   test.cpp

Isto é o que acontece quando mv é um diretório que já existia. O arquivo “test.cpp” é movido para o diretório “projetos c++”.

exemplos de rm

jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby test.rb
jennifer@my_computer:~/Desktop$ rm c++_test.cpp
jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     ruby    ruby test.rb

c++_test.cpp foi excluído

jennifer@my_computer:~/Desktop$ rm c++\ projects
rm: cannot remove 'c++ projects': Is a directory
jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     ruby    ruby test.rb

rm tem um requisito extra para deletar diretórios

jennifer@my_computer:~/Desktop$ rm -rf c++\ projects
jennifer@my_computer:~/Desktop$ ls
Research Paper.docx     ruby    ruby test.rb

-rf deve ser adicionado para excluir um diretório.

Para saber mais sobre ls, digite o comando ls --help. Para touch, digite touch --help. Da mesma forma com todos os 6 comandos mencionados aqui. Isso imprime uma explicação detalhada de uso sem criar ou excluir nada.

comando cd, diretórios explicados #

michael@who-cares:~$

O símbolo ~ após o who-cares: é o diretório atual. ~ na verdade significa o diretório pessoal da pessoa. Neste caso, é /home/michael.

michael@who-cares:~$ cd Downloads
michael@who-cares:~/Downloads$

Procura por Downloads no diretório atual, então o torna o diretório atual.

michael@who-cares:~/Downlaods$ cd /var
michael@who-cares:/var$

Como este diretório começou com um /, isso significa procurar no diretório raiz pelo diretório var. Para quem vem do Windows, o diretório raiz é equivalente a C:\. Diretórios que começam com / são chamados de “diretórios absolutos” e diretórios que não são chamados de “diretórios relativos”

michael@who-cares:/var cd lib/dbus
michael@who-cares:/var/lib/dbus$

O / no meio significa fazer cd lib e uma vez feito cd dbus em um comando.

michael@who-cares:/var/lib/dbus$ cd .
michael@who-cares:/var/lib/dbus$

. na verdade significa “o diretório atual”. O comando cd . é basicamente inútil, mas . é útil para outras coisas.

michael@who-cares:/var/lib/dbus$ cd ..
michael@who-cares:/var/lib$

.. na verdade significa “o pai do diretório atual”. Como tal, cd .. significa “navegar um diretório para cima”.

michael@who-cares:/var/lib$ cd ../log/apt
michael@who-cares:/var/log/apt$

. e .. também podem fazer parte da cadeia /. Além disso, não há limite para quanto tempo pode ser.

michael@who-cares:/var/log/apt$ cd /dev/bus
michael@who-cares:/dev/bus$

A cadeia / pode até existir quando o diretório inicia na raiz.

michael@who-cares:/dev/bus$ cd /
michael@who-cares:/$

cd / leva você ao diretório raiz. Eu me pergunto o que acontece se você digitar cd .. aqui… (não se preocupe. É seguro)

michael@who-cares:/$ cd home
michael@who-cares:/home$ cd michael
michael@who-cares:~$

Cada usuário tem um diretório para suas coisas dentro do diretório inicial. Se o diretório atual estiver sob o diretório home, essa parte do nome, neste caso /home/michael, será substituída por ~.

michael@who-cares:~$ cd sys
michael@who-cares:/sys$ cd ~/Desktop
michael@who-cares:~/Desktop$ cd ~/..
michael@who-cares:/home$

~ também pode fazer parte da cadeia /. Pode até estar na mesma cadeia que ... Se o diretório começa com ~, é um diretório absoluto como se começasse com /.

Última coisa a tentar: digite cd sem diretório depois.

Comandos básicos do Unix #

$pwd

Exibe o diretório de trabalho atual.

$who

Exibe todos os usuários logados.

$who am i

Mostra o nome de usuário do usuário atual.

$date

Exibe a data atual do sistema

$which <command>

Mostra o caminho do comando especificado. Por exemplo, “$which pwd” mostrará o caminho do comando ‘pwd’.

$file <file_name>

Mostra o tipo do arquivo especificado (arquivo regular, diretório ou outros arquivos)

$cal

Exibe o calendário do mês atual.

$bc

Mostra o cálculo matemático entre dois inteiros de floats. Por exemplo, “$bc 2+3” retornará a soma aritmética de 3 e 5.

$ls

Lista o conteúdo do diretório.

  • $ls -l : listas em formato longo.

  • $ls -c : saída de várias colunas.

  • $ls -f : Lista o tipo de arquivo.

  • $ls -r : Listagem recursiva de todos os subdiretórios encontrados.

  • $ls -a : Exibe todos os arquivos, incluindo arquivos ocultos.

  • $ls -i : Lista todos os arquivos junto com seu número de I-Node.

    $grep [options] <input_file_names> Imprime linhas que contêm uma correspondência para um padrão.

  • $grep -i : executa correspondência insensível a maiúsculas e minúsculas

  • $grep -v : Imprime todas as linhas que não contêm o regex

  • $grep -r : Pesquisa recursivamente nos subdiretórios listados e imprime os nomes dos arquivos com a ocorrência do padrão

  • $grep -I : Excluir arquivos binários

pushd/popd (armazene o diretório atual na pilha e vá para dest / pop prev dir e vá até ele) #

$ pwd
/home/bob/somedir1/somedir2/somedir3

$ pushd /home/bob/otherdir1/otherdir2
/home/bob/otherdir1/otherdir2 /home/bob/somedir1/somedir2/somedir3

$ popd
/home/bob/somedir1/somedir2/somedir3

$ pushd /usr
/usr /home/bob/somedir1/somedir2/somedir3

$ pushd /var
/var /usr /home/bob/somedir1/somedir2/somedir3

$ popd
/usr /home/bob/somedir1/somedir2/somedir3

$ pwd
/usr

$ popd
/home/bob/somedir1/somedir2/somedir3

$ pwd
/home/bob/somedir1/somedir2/somedir3
