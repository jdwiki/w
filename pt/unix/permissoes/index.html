No unix cada arquivo tem certas permissões como ler, escrever e executar. Um usuário pode manipular as permissões de um arquivo usando o comando ‘chmod’.

No UNIX, existem três permissões usadas para conceder um certo nível de acesso a um arquivo ou pasta.

Para arquivos:

  • Ler: Permite que o usuário/grupo/outros leiam um arquivo.
  • Escrever: Permite que o usuário/grupo/outros modifiquem um arquivo.
  • Execute: Permite que o usuário/grupo/outros executem (ou executem) um arquivo.

Estes são ligeiramente alterados para diretórios:

  • Ler: Permite que o usuário/grupo/outros listem os nomes dos arquivos em um diretório.
  • Escrever: Permite que o usuário/grupo/outros criem, excluam e renomeiem arquivos em um diretório.
  • Executar: Permite que o usuário/grupo/outros acessem os metadados e o conteúdo do arquivo para um diretório.

Essas permissões podem ser representadas usando as letras “r” para leitura, “w” para gravação e “x” para execução. Eles também podem ser representados numericamente: 4 para leitura, 2 para gravação e 1 para execução.

Alterar as permissões de um arquivo #

> chmod 644 example.txt
> ls -l example.txt
-rw-r--r--  1 owner ogroup 57 Jul  3 10:13  example.txt

O comando acima altera as permissões do arquivo para permitir que o proprietário do arquivo leia e grave em um arquivo. Também permite que os usuários do grupo do proprietário e outros usuários do sistema leiam o arquivo.

Entendendo as permissões #

Digamos que há um arquivo que gostaríamos de executar, um script bash chamado add.sh, por exemplo. Digitar ./add.sh no entanto, gera um erro de permissão. Obter as permissões é um processo simples.

Para determinar as permissões de um arquivo, digite:

ls -l filename, ou, no nosso caso, ls -l ./add.sh

Isso imprime o seguinte no console:

-r--r--r-- 1 nome de usuário nome do grupo 0 4 de janeiro 12:00 add.sh

Vamos parar e entender o que isso significa. Existem três tipos diferentes de permissões: proprietário, grupo, outros. Permissões distintas se aplicam a cada tipo de permissão.

Há também três ações de permissão, que descrevem mais amplamente o que exatamente um usuário pode fazer em um arquivo. São eles: (ler: r, escrever: w, executar: x).

Então, volte para aquela sequência de traços e r’s. Cada grupo de permissão tem três habilidades potenciais. Os grupos são listados na ordem owner-group-others e as ações como read-write-execute.

Mas espere, isso significa que há um caractere extra no início da string. Este é realmente o caractere descritor de arquivo. Podemos ver que há um - lá, mas existem outros caracteres para coisas como diretórios(d), sockets(s), link simbólico(l) etc.

Isso nos deixa essencialmente com esta informação: um arquivo onde proprietário, grupo e outros têm permissões de leitura. Nenhuma outra permissão concedida.

Vamos alterar isso para permitir que o proprietário também escreva e execute o arquivo. Nota: Dependendo das permissões, pode ser necessário acrescentar sudo a este comando.

chmod 744 add.sh
ls -l add.sh

Imprime

-rwxr--r-- 1 username groupname 0 Month time add.sh

Agora, o proprietário do arquivo pode executar o arquivo digitando

./add.sh

Cálculo CHMOD #

Cálculo CHMOD

CHMOD é binário. _ / _ _ _ / _ _ _ / _ _ _ = _/4+2+1/4+2+1/4+2+1 = 777 = _/rwx/rwx/rwx = 777 Portanto _rwx = _/4+2+1 = 7

D / _ _ _ / _ _ _ / _ _ _ (‘D’ = diretório, outro uso é L = Link)

Então, por exemplo _rwxr_xr_x = _/rwx/rx/r_x = 755

CHOWN #

Para alterar own:group você usa o comando chown user:group

por exemplo. chown proprietário:grupo ou se proprietário e grupo são os mesmos, você pode usar chown proprietário: (porque linus assume que proprietário:grupo é o mesmo).
