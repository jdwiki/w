Verificando uma string quanto a caracteres indesejados #

Como desenvolvedor, você frequentemente lida com strings que não são criadas por seu próprio código.

Eles geralmente são fornecidos por bibliotecas de terceiros, sistemas externos ou até mesmo usuários finais. A validação de strings de proveniência incerta é considerada uma das características da programação defensiva e, na maioria dos casos, você desejará rejeitar a entrada de strings que não atende às suas expectativas.

Um caso bastante comum é quando você deseja permitir apenas caracteres alfanuméricos em uma string de entrada, então usaremos isso como exemplo. Em Java simples, os dois métodos a seguir servem ao mesmo propósito:

public static boolean isAlphanumeric(String s) {
    for (char c : s.toCharArray()) {
        if (!Character.isLetterOrDigit(c)) {
            return false;
        }
    }

    return true;
}

public static boolean isAlphanumeric(String s) {
    return s.matches("^[0­-9a­-zA­-Z]*$");
}

A primeira versão converte a string em uma matriz de caracteres e, em seguida, usa o método estático isLetterOrDigit da classe Character para determinar se os caracteres contidos na matriz são alfanuméricos ou não. Essa abordagem é previsível e legível, embora um pouco detalhada.

A segunda versão usa uma expressão regular para atingir o mesmo propósito. É mais conciso, mas pode ser um tanto enigmático para desenvolvedores com pouco ou nenhum conhecimento de expressões regulares.

Guava apresenta a classe CharMatcher para lidar com esses tipos de situações. Nosso teste alfanumérico, usando Guava, ficaria assim:

import static com.google.common.base.CharMatcher.javaLetterOrDigit;

/* ... */

public static boolean isAlphanumeric(String s) {
    return javaLetterOrDigit().matchesAllOf(s);
}

O corpo do método contém apenas uma linha, mas na verdade há muita coisa acontecendo aqui, então vamos detalhar um pouco mais.

Se você der uma olhada na API da classe CharMatcher do Guava, você notará que ela implementa a interface Predicate<Character>. Se você criar uma classe que implemente Predicate<Character> você mesmo, pode ser algo assim:

import com.google.common.base.Predicate;

public class AlphanumericPredicate implements Predicate<Character> {
    @Override
    public boolean apply(Character c) {
        return Character.isLetterOrDigit(c);
    }
}

No Guava, como em várias outras linguagens de programação e bibliotecas que atendem a um estilo funcional de programação, um predicado é uma construção que avalia uma determinada entrada como verdadeira ou falsa. Na interface Predicate<T> do Guava, isso é evidenciado pela presença do único método boolean apply(T t). A classe CharMatcher é construída sobre este conceito e avaliará um caractere ou sequência de caracteres para verificar se eles correspondem ou não aos critérios estabelecidos pela instância CharMatcher usada.

Atualmente, o Guava fornece os seguintes correspondentes de caracteres predefinidos:

Matcher Descrição
qualquer() Corresponde a qualquer caractere.
nenhum() Não corresponde a nenhum caractere.
javaDigit() Corresponde a dígitos, de acordo com a definição Java.
javaUpperCase() Corresponde a qualquer caractere maiúsculo, de acordo com a definição de Java.
javaLowerCase() Corresponde a qualquer caractere minúsculo, de acordo com a definição do Java.
javaLetter() Corresponde a qualquer letra, de acordo com a definição de Java.
javaLetterOrDigit() Corresponde a qualquer letra ou dígito, de acordo com a definição de Java.
javaIsoControl() Corresponde a qualquer caractere de controle ISO, de acordo com a definição do Java.
ascii() Corresponde a qualquer caractere no conjunto de caracteres ASCII.
invisível() Corresponde a caracteres que não são visíveis, de acordo com o padrão Unicode.
digit() Corresponde a qualquer dígito, de acordo com a especificação Unicode.
espaço em branco() Corresponde a qualquer caractere de espaço em branco, de acordo com a especificação Unicode.
breakingWhitespace() Corresponde a qualquer caractere de espaço em branco quebrado, de acordo com a especificação unicode.
singleWidth() Corresponde a qualquer caractere de largura única.

Se você leu a tabela acima, sem dúvida notou a quantidade de definição e especificação envolvida na determinação de quais caracteres pertencem a uma determinada categoria. A abordagem do Guava, até agora, tem sido fornecer wrappers CharMatcher para várias categorias de caracteres definidas pelo Java, e você pode consultar a API da classe Character do Java para obter mais informações sobre essas categorias. Por outro lado, o Guava tenta fornecer várias instâncias CharMatcher que estão alinhadas com a especificação Unicode atual. Para mais detalhes, consulte a documentação da API CharMatcher.

Voltando ao nosso exemplo de verificação de caracteres indesejados em uma string, os métodos CharMatcher a seguir fornecem os recursos necessários para verificar se o uso de caracteres de uma determinada string atende aos seus requisitos:

  • boolean matchesNoneOf(CharSequence sequence) Retorna true se nenhum dos caracteres na string do argumento corresponder à instância CharMatcher.

  • boolean matchesAnyOf(CharSequence sequence) Retorna true se pelo menos um caractere na string do argumento corresponder à instância CharMatcher.

  • boolean matchesAllOf(CharSequence sequence) Retorna true se todos os caracteres na string do argumento corresponderem à instância CharMatcher.

Encontrando e contando caracteres em uma string #

Para ajudá-lo a encontrar e contar caracteres em uma string, CharMatcher fornece os seguintes métodos:

  • int indexIn(Sequência de caracteres) Retorna o índice do primeiro caractere que corresponde à instância CharMatcher. Retorna -1 se nenhum caractere corresponder.

  • int indexIn(Sequência de caracteres, int início) Retorna o índice do primeiro caractere após a posição inicial especificada que corresponde à instância CharMatcher. Retorna -1 se nenhum caractere corresponder.

  • int lastIndexIn(sequência CharSequence) Retorna o índice do último caractere que corresponde à instância CharMatcher. Retorna -1 se nenhum caractere corresponder.

  • int countIn(Sequência de caracteres) Retorna o número de caracteres que correspondem à instância CharMatcher.

Usando esses métodos, aqui está um aplicativo de console simples chamado NonAsciiFinder que recebe uma string como argumento de entrada. Primeiro, ele imprime o número total de caracteres não ASCII contidos na string. Subsequentemente, ele imprime a representação Unicode de cada caractere não ASCII que encontra. Aqui está o código:

import com.google.common.base.CharMatcher;

public class NonAsciiFinder {
    private static final CharMatcher NON_ASCII = CharMatcher.ascii().negate();

    public static void main(String[] args) {
        String input = args[0];
        int nonAsciiCount = NON_ASCII.countIn(input);

        echo("Non-ASCII characters found: %d", nonAsciiCount);

        if (nonAsciiCount > 0) {
            int position = -­1;
            char character = 0;

            while (position != NON_ASCII.lastIndexIn(input)) {
                position = NON_ASCII.indexIn(input, position + 1);
                character = input.charAt(position);
                
                echo("%s => \\u%04x", character, (int) character);
            }
        }
    }

    private static void echo(String s, Object... args) {
        System.out.println(String.format(s, args));
    }
}

Observe no exemplo acima como você pode simplesmente inverter um CharMatcher chamando seu método negate. Da mesma forma, o CharMatcher abaixo corresponde a todos os caracteres de largura dupla e é criado negando o CharMatcher predefinido para caracteres de largura simples.

final static CharMatcher DOUBLE_WIDTH = CharMatcher.singleWidth().negate();

A execução do aplicativo NonAsciiFinder produz a seguinte saída:

$> java NonAsciiFinder "Maître Corbeau, sur un arbre perché"
Non­-ASCII characters found: 2
î => \u00ee
é => \u00e9

$> java NonAsciiFinder "古池や蛙飛び込む水の音"
Non­ASCII characters found: 11
古 => \u53e4
池 => \u6c60
や => \u3084
蛙 => \u86d9
飛 => \u98db
び => \u3073
込 => \u8fbc
む => \u3080
水 => \u6c34
の => \u306e
音 => \u97f3

Removendo caracteres indesejados de uma string #

O exemplo [Verificando se há caracteres indesejados em uma string](https://www.wikiod.com/pt/guava/cordas#Verificando uma string quanto a caracteres indesejados), descreve como testar e rejeitar strings que não atendem a determinados critérios. Obviamente, nem sempre é possível rejeitar a entrada diretamente, e às vezes você só precisa se contentar com o que recebe. Nesses casos, um desenvolvedor cauteloso tentará limpar as strings fornecidas para remover quaisquer caracteres que possam atrapalhar o processamento adicional.

Para remover, aparar e substituir caracteres indesejados, a arma de escolha será novamente a classe CharMatcher da Guava.

Removendo caracteres #

Os dois métodos CharMatcher de interesse nesta seção são:

  • String reterFrom(sequência CharSequence) Retorna uma string contendo todos os caracteres que corresponderam à instância CharMatcher.

  • String removeFrom(sequência CharSequence) Retorna uma string contendo todos os caracteres que não corresponderam à instância CharMatcher.

Como exemplo, usaremos CharMatcher.digit(), uma instância predefinida de CharMatcher que, sem surpresa, só corresponde a dígitos.

String rock = "1, 2, 3 o'clock, 4 o'clock rock!";

CharMatcher.digit().retainFrom(rock); // "1234"
CharMatcher.digit().removeFrom(rock); // ", , o'clock, o'clock rock!"
CharMatcher.digit().negate().removeFrom(rock); // "1234"

A última linha neste exemplo ilustra que removeFrom é na verdade a operação inversa de retainFrom. Invocar retainFrom em um CharMatcher tem o mesmo efeito que invocar removeFrom em uma versão negada desse CharMatcher.

Aparar caracteres iniciais e finais #

A remoção de caracteres à esquerda e à direita é uma operação muito comum, usada com mais frequência para cortar espaços em branco de strings. O CharMatcher da Guava oferece estes métodos de corte:

  • String trimLeadingFrom(sequência CharSequence) Remove todos os caracteres iniciais que correspondem à instância CharMatcher.

  • String trimTrailingFrom(sequência CharSequence) Remove todos os caracteres à direita que correspondem à instância CharMatcher.

  • String trimFrom(sequência CharSequence) Remove todos os caracteres iniciais e finais que correspondem à instância CharMatcher.

Quando usado com CharMatcher.whitespace(), esses métodos cuidarão efetivamente de todas as suas necessidades de corte de espaço em branco:

CharMatcher.whitespace().trimFrom("   Too much space   "); // returns "Too much space"

Substituindo caracteres #

Muitas vezes, os aplicativos substituirão os caracteres que não são permitidos em uma determinada situação por um caractere de espaço reservado. Para substituir caracteres em uma string, a API do CharMatcher fornece os seguintes métodos:

  • String replaceFrom(Sequência de caracteres, substituição de caracteres) Substitui todas as ocorrências de caracteres que correspondem à instância CharMatcher pelo caractere de substituição fornecido.

  • String replaceFrom(sequência CharSequence, substituição CharSequence) Substitui todas as ocorrências de caracteres que correspondem à instância CharMatcher pela sequência de caracteres de substituição fornecida (string).

  • String colapsoFrom(Sequência de caracteres, substituição de caracteres) Substitui grupos de caracteres consecutivos que correspondem à instância CharMatcher por uma única instância do caractere de substituição fornecido.

  • String trimAndCollapseFrom(Sequência de caracteres, substituição de caracteres) Comporta-se da mesma forma que collapseFrom, mas os grupos correspondentes no início e no final são removidos em vez de substituídos.

Vejamos um exemplo que demonstra como o comportamento desses métodos difere. Digamos que estamos criando um aplicativo que permite ao usuário especificar nomes de arquivos de saída. Para limpar a entrada fornecida pelo usuário, criamos uma instância CharMatcher que é uma combinação do espaço em branco predefinido CharMatcher e um CharMatcher personalizado que especifica um conjunto de caracteres que preferimos evitar em nossos nomes de arquivos.

CharMatcher illegal = CharMatcher.whitespace().or(CharMatcher.anyOf("<>:|?*\"/\\"));

Agora, se invocarmos os métodos de substituição discutidos da seguinte forma em um nome de arquivo que precisa urgentemente de limpeza:

String filename = "<A::12> first draft???";

System.out.println(illegal.replaceFrom(filename, '_'));
System.out.println(illegal.collapseFrom(filename, '_'));
System.out.println(illegal.trimAndCollapseFrom(filename, '_'));

Veremos a saída abaixo em nosso console.

_A__12___first_draft___
_A_12_first_draft_
A_12_first_draft

Dividindo uma string em uma lista #

Para dividir strings, o Guava apresenta a classe Splitter.

Por que não usar os recursos de divisão do Java? #

Como regra, o Guava não duplica a funcionalidade que está prontamente disponível em Java. Por que então precisamos de uma classe Splitter adicional? Os métodos de divisão na classe String do Java não nos fornecem todas as mecânicas de divisão de strings que precisaremos?

A maneira mais fácil de responder a essa pergunta é com alguns exemplos. Primeiro, vamos lidar com a seguinte dupla de pistoleiros:

String gunslingers = "Wyatt Earp+Doc Holliday";

Para tentar separar o lendário homem da lei e seu amigo dentista, podemos tentar o seguinte:

String[] result = gunslingers.split("+"); // wrong

Em tempo de execução, no entanto, somos confrontados com a seguinte exceção:

Exception in thread "main" java.util.regex.PatternSyntaxException:
Dangling meta character '+' near index 0

Após um facepalm involuntário, somos rápidos em lembrar que o método split do String recebe uma expressão regular como argumento, e que o caractere + é usado como quantificador em expressões regulares. A solução é então escapar do caractere + ou colocá-lo em uma classe de caracteres.

String[] result = gunslingers.split("\\+");
String[] result = gunslingers.split("[+]");

Tendo resolvido esse problema com sucesso, passamos para os três mosqueteiros.

String musketeers = ",Porthos , Athos ,Aramis,";

A vírgula não tem nenhum significado especial em expressões regulares, então vamos contar os mosqueteiros aplicando o método String.split() e obtendo o comprimento do array resultante.

System.out.println(musketeers.split(",").length);

O que produz o seguinte resultado no console:

4

Quatro? Dado o fato de que a string contém uma vírgula inicial e uma vírgula final, um resultado de cinco estaria dentro das expectativas normais, mas quatro? Como se vê, o comportamento do método split do Java é preservar o início, mas descartar strings vazias no final, de modo que o conteúdo real do array seja ["", "Porthos", "Athos", "Aramis"] .

Como não precisamos de nenhuma string vazia, inicial ou final, vamos filtrá-las com um loop:

for (String musketeer : musketeers.split(",")) {
    if (!musketeer.isEmpty()) {
        System.out.println(musketeer);
    }
}

Isso nos dá a seguinte saída:

Porthos 
 Athos
Aramis

Como você pode ver na saída acima, os espaços extras antes e depois dos separadores de vírgula foram preservados na saída. Para contornar isso, podemos cortar os espaços desnecessários, que finalmente produzirão a saída desejada:

for (String musketeer : musketeers.split(",")) {
    if(!musketeer.isEmpty()) {
        System.out.println(musketeer.trim());
    }
}

(Como alternativa, também podemos adaptar a expressão regular para incluir espaços em branco ao redor dos separadores de vírgulas. No entanto, lembre-se de que os espaços iniciais antes da primeira entrada ou espaços finais após a última entrada ainda seriam preservados.)

Depois de ler os exemplos acima, não podemos deixar de concluir que dividir strings com Java é um pouco irritante, na melhor das hipóteses.

Dividindo cordas com goiaba #

A melhor maneira de demonstrar como o Guava transforma a divisão de strings em uma experiência relativamente indolor, é tratar as mesmas duas strings novamente, mas desta vez usando a classe Splitter do Guava.

List<String> gunslingers = Splitter.on('+')
        .splitToList("Wyatt Earp+Doc Holliday");

List<String> musketeers = Splitter.on(",")
        .omitEmptyStrings()
        .trimResults()
        .splitToList(",Porthos , Athos ,Aramis,");

Como você pode ver no código acima, Splitter expõe uma API fluente e permite que você crie instâncias por meio de uma série de métodos de fábrica estáticos:

  • Divisor estático on(separador de caracteres) Permite especificar o separador como um caractere.

  • Divisor estático em (separador de string) Permite especificar o separador como uma string.

  • static Splitter on(CharMatcher separatorMatcher) Permite especificar o separador como um Guava CharMatcher.

  • Divisor estático on(Pattern separatorPattern) Permite especificar o separador como uma expressão regular Java Padrão.

  • Divisor estático onPattern(String separatorPattern) Permite especificar o separador como uma string de expressão regular.

Além desses métodos de fábrica baseados em separadores, há também um método static Splitter fixedLength(int length) para criar instâncias Splitter que dividem strings em partes do comprimento especificado.

Após a criação da instância Splitter, vários modificadores podem ser aplicados:

  • Divisor omitEmptyStrings() Instrui o Splitter a excluir strings vazias dos resultados.

  • Divisor trimResults() Instrui o Splitter a cortar os resultados usando o espaço em branco predefinido CharMatcher.

  • Splitter trimResults(Trimmer CharMatcher) Instrui o Splitter a cortar os resultados usando o CharMatcher especificado.

Depois de criar (e opcionalmente modificar) um Splitter, ele pode ser invocado em uma sequência de caracteres invocando seu método split, que retornará um objeto do tipo Iterable<String>, ou seu método splitToList, que retornará um objeto (imutável) do tipo List<String>.

Você pode se perguntar em quais casos seria benéfico usar o método split (que retorna um Iterable) em vez do método splitToList (que retorna o tipo List mais comumente usado). A resposta curta para isso é: você provavelmente quer usar o método split apenas para processar strings muito grandes. A resposta um pouco mais longa é que, como o método split retorna um Iterable, as operações de divisão podem ser avaliadas lentamente (no tempo de iteração), eliminando assim a necessidade de manter todo o resultado da operação de divisão na memória.
