Pronto para produção #

Criando projeto de aplicativo

Ao criar um aplicativo Android pronto para a loja de aplicativos, é importante ao usar ionic start adicionamos os sinalizadores --appname|-a e --id|-i que é usado para o google play identificar seu aplicativo de outros aplicativos.

Se você estiver iniciando um novo projeto de aplicativo móvel, poderá usar o exemplo cli abaixo.

$ ionic start --v2 -a "App Example" -i "com.example.app" -t "tabs"

1. Arquivo de configuração do aplicativo

se você quiser definir esta informação dentro de um aplicativo existente, você pode modificar config.xml. Eu recomendo aqueles que usaram o comando acima para modificar o config.xml também.

Confirme/edite os atributos widget id, name, description e author.

Exemplo:

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<widget id="com.example.app" version="1.0.0" xmlns="http://www.w3.org/ns/widgets" xmlns:cdv="http://cordova.apache.org/ns/1.0">
  <name>Example App</name>
  <description>Example app for stackoverflow users</description>
  <author email="admin@example.com" href="http://example.com/">Your name or team</author>
  ...
</widget>

2. ícone e tela inicial

Ambos os tipos de arquivo suportados por ícone e imagem inicial são png, psd ou ai e devem ter um nome de arquivo que corresponda ao que é icon ou splash e colocado no diretório de recursos na raiz do seu projeto. As dimensões mínimas da imagem do ícone devem ser de 192 x 192 px e não devem ter cantos arredondados. e a tela inicial é muito mais complicada, então clique aqui para ler mais. No entanto, as dimensões mínimas devem ser 2208 x 2208 px.

se você tiver um arquivo de ícone para gerar, use este comando ionic resources --icon se você tiver um arquivo splash para gerar, use este comando ionic resources --splash

3. Aplicativo de produção de construção

Antes de criar seu aplicativo de produção, remova todos os dados de log confidenciais.

Para criar uma versão de lançamento com todas as otimizações padrão em vigor, use a tag –release & –prod

compilação iônica android --release --prod

Para obter uma lista completa de otimizações disponíveis, você pode visitar o @ionic/app-scripts repository

4. Criar chave privada

Agora, precisamos assinar o APK não assinado (android-release-unsigned.apk) e executar um utilitário de alinhamento nele para otimizá-lo e prepará-lo para a loja de aplicativos. Se você já tiver uma chave de assinatura, ignore estas etapas e use-a.

Em seguida, localize seu arquivo APK não assinado android-release-unsigned.apk dentro do diretório do projeto/platforms/android/build/outputs/apk/ e use Comando keytools que será usado para assinar nosso arquivo apk. Você pode usar o exemplo abaixo:

$ keytool -genkey -v -keystore my-release-key.keystore -alias androidKey -keyalg RSA -keysize 2048 -validity 10000

você pode encontrar my-release-key.keystore em seu diretório atual.

Vamos gerar nossa chave privada usando o comando keytool que vem com o JDK. Se esta ferramenta não for encontrada, consulte o guia de instalação:

Primeiro, você será solicitado a criar uma senha para o keystore. Em seguida, responda o restante das perguntas das boas ferramentas e quando tudo estiver pronto, você deverá ter um arquivo chamado my-release-key.keystore criado no diretório atual.

Nota: Certifique-se de salvar este arquivo em algum lugar seguro, se você perdê-lo, não poderá enviar atualizações para seu aplicativo!

5. Assinar APK

Para assinar o APK não assinado, execute a ferramenta jarsigner que também está incluída no JDK:

$ jarsigner -verbose -sigalg SHA1withRSA -digestalg SHA1 -keystore my-release-key.keystore HelloWorld-release-unsigned.apk alias_name

Isso assina o apk no lugar. Por fim, precisamos executar a ferramenta de alinhamento zip para otimizar o APK. A ferramenta zipalign pode ser encontrada em /path/to/Android/sdk/build-tools/VERSION/zipalign.

$ zipalign -v 4 HelloWorld-release-unsigned.apk HelloWorld.apk

Agora temos nosso binário de lançamento final chamado HelloWorld.apk e podemos lançá-lo na Google Play Store para todo o mundo aproveitar!

Publique seu aplicativo na Google Play Store. Agora que temos nosso APK de lançamento pronto para a Google Play Store, podemos criar uma listagem da Play Store e fazer upload de nosso APK. Para começar, você precisará visitar o Console do desenvolvedor da Google Play Store e criar uma nova conta de desenvolvedor. Vai custar $ 25 uma taxa de tempo.

Depois de ter uma conta de desenvolvedor, você pode clicar em “Publicar um aplicativo Android no Google Play” e seguir as instruções na tela.
