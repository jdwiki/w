Usando serviços

  • Compartilhe informações entre diferentes páginas

Nesta página

  • Compartilhe informações entre diferentes páginas

Uma coisa muito importante sobre o uso de serviços compartilhados é que eles devem ser incluídos no array providers do componente mais alto onde devem ser compartilhados.

Por que é que? Bem, vamos supor que incluímos a referência MyService no array providers de cada Component. Algo como:

@Component({
  templateUrl:"page1.html",
  providers: [MyService]
})

E

@Component({
  templateUrl:"page2.html",
  providers: [MyService]
})

Dessa forma será criada uma nova instância do serviço para cada componente, de modo que a instância em que uma página salvará os dados será diferente da instância usada para obter os dados. Então isso não vai funcionar.

Para fazer com que o aplicativo inteiro use a mesma instância (fazendo o serviço funcionar como um serviço singleton), podemos adicionar sua referência no App Component assim:

@Component({
  template: '<ion-nav [root]="rootPage"></ion-nav>',
  providers: [MyService]
})

Você também pode adicionar a referência MyService no ionicBootstrap(MyApp, [MyService]); mas de acordo com guias de estilo Angular2

Fornecer serviços para o injetor Angular 2 no nível mais alto componente onde serão compartilhados.

Por quê? O injetor Angular 2 é hierárquico.

Por quê? Ao fornecer o serviço a um componente de nível superior, esse a instância é compartilhada e disponível para todos os componentes filhos desse topo componente de nível.

Por quê? Isso é ideal quando um serviço está compartilhando métodos ou estado.

Por quê? Isso não é ideal quando dois componentes diferentes precisam de instâncias de um serviço. Nesse cenário, seria melhor fornecer o serviço no nível do componente que precisa do novo e separado instância.

E

Funcionará. Não é apenas uma prática recomendada. O provedor de bootstrap a opção destina-se a configurar e substituir o próprio Angular serviços pré-registrados, como suporte de roteamento.

… o App Component seria a melhor escolha.

Compartilhe informações entre diferentes páginas #

Um dos exemplos mais fáceis de usar serviços compartilhados é quando queremos armazenar alguns dados de uma determinada página do nosso aplicativo e, em seguida, obter esses dados novamente, mas de outra página.

Uma opção poderia ser enviar esses dados como parâmetro (por exemplo, se uma página chama a outra), mas se quisermos usar esses dados de uma parte completamente diferente do aplicativo, essa não parece ser a melhor maneira de fazer isto. É quando os serviços compartilhados entram em ação.

Neste exemplo, vamos usar um serviço simples chamado MyService que tem apenas dois métodos simples: saveMessage() para armazenar uma string e getMessage() para obtê-la novamente. Este código faz parte deste plunker de trabalho onde você pode vê-lo em ação.

import {Injectable} from '@angular/core';

@Injectable()
export class MyService { 
  
  private message: string;
  
  constructor(){ }
  
  public saveMessage(theMessage: string): void {
    this.message = theMessage;
  }
  
  public getMessage(): string {
    return this.message;
  }
}

Então, quando queremos armazenar uma nova mensagem, podemos simplesmente usar o método saveMessage(theMessageWeWantToSave); da instância MyService (chamada apenas service).

import { Component } from "@angular/core";
import { MyService } from 'service.ts';

@Component({
  templateUrl:"page1.html"
})
export class Page1 {
  
  message: string;
  
  // ... 
   
  public saveSecretMessage(): void {
    this.service.saveMessage(this.message);
  }      
}

Da mesma forma, quando queremos obter esses dados, podemos usar o método getMessage() da instância do serviço assim:

import { Component } from "@angular/core";
import { MyService } from 'service.ts';

@Component({
  templateUrl:"page2.html"
})
export class Page2 {
  
  enteredMessage: string;
  
  constructor(private service: MyService) {
    this.enteredMessage = this.service.getMessage();
  }
    
  // ...
}

Por favor, lembre-se de verificar a seção Observações para ver onde a referência para o serviço MyService deve ser incluída e por quê.
