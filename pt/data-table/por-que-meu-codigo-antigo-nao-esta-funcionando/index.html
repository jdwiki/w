Nesta página

  • exclusivo e duplicado não funciona mais em data.table com chave

O pacote data.table passou por várias mudanças e inovações ao longo do tempo. Aqui estão algumas armadilhas em potencial que podem ajudar os usuários a analisar o código legado ou revisar postagens antigas do blog.

exclusivo e duplicado não funciona mais em data.table com chave #

Isto é para quem está migrando para data.table >= 1.9.8

Você tem um conjunto de dados de donos e nomes de animais de estimação, mas suspeita que alguns dados repetidos foram capturados.

library(data.table)
DT <- data.table(pet = c("dog","dog","cat","dog"),
                 owner = c("Alice","Bob","Charlie","Alice"),
                 entry.date = c("31/12/2015","31/12/2015","14/2/2016","14/2/2016"),
                 key = "owner")


> tables()
     NAME NROW NCOL MB COLS                 KEY  
[1,] DT      4    3  1 pet,owner,entry.date owner
Total: 1MB

Lembre-se de digitar uma tabela para classificá-la. Alice foi inserida duas vezes.

> DT
   pet   owner entry.date
1: dog   Alice 31/12/2015
2: dog   Alice  14/2/2016
3: dog     Bob 31/12/2015
4: cat Charlie  14/2/2016

Digamos que você tenha usado unique para se livrar de duplicatas em seus dados com base na chave, usando a data de captura de dados mais recente definindo deLast para TRUE.

clean.DT <- unique(DT, fromLast = TRUE)

> tables()
    NAME     NROW NCOL MB COLS                 KEY  
[1,] clean.DT    3    3  1 pet,owner,entry.date owner
[2,] DT          4    3  1 pet,owner,entry.date owner
Total: 2MB

A duplicata de Alice foi removida.

clean.DT <- unique(DT, fromLast = TRUE)

> tables()
     NAME     NROW NCOL MB COLS                 KEY  
[1,] clean.DT    4    3  1 pet,owner,entry.date owner
[2,] DT          4    3  1 pet,owner,entry.date owner

Isso não funciona. Ainda 4 linhas!

Fixar #

Use o parâmetro by= que não é mais padrão para sua chave, mas para todas as colunas.

clean.DT <- unique(DT, by = key(DT), fromLast = TRUE)

Agora está tudo bem.

> clean.DT
   pet   owner entry.date
1: dog   Alice  14/2/2016
2: dog     Bob 31/12/2015
3: cat Charlie  14/2/2016

Detalhes e correção temporária #

Veja item 1 nas notas de lançamento do NEWS para detalhes:

Alterações na v1.9.8 (em CRAN 25 de novembro de 2016)

MUDANÇAS POTENCIALMENTE QUEBRADAS

  1. Por padrão, todas as colunas agora são usadas por unique(), duplicated() e métodos data.table uniqueN(), #1284 e #1841. Para restaurar o antigo comportamento: options(datatable.old.unique.by.key=TRUE). Em 1 ano isso opção para restaurar o padrão antigo será preterida com aviso. Dentro 2 anos a opção será removida. Por favor, passe explicitamente by=key(DT) para maior clareza. Somente o código que depende do padrão é afetado. 266 Os pacotes CRAN e Bioconductor usando data.table foram verificados antes liberação. 9 precisaram mudar e foram notificados. Qualquer linha de código sem cobertura de teste será perdido por essas verificações. Algum pacotes que não estão no CRAN ou no Bioconductor não foram verificados.

Assim, você pode usar as opções como uma solução temporária até que seu código seja corrigido.

options(datatable.old.unique.by.key=TRUE)
