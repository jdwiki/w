Instalação e configuração #

Instale a versão estável do CRAN:

install.packages("data.table")

Ou a versão de desenvolvimento do github:

install.packages("data.table", type = "source", 
  repos = "http://Rdatatable.github.io/data.table")

Para reverter de devel para CRAN, a versão atual deve primeiro ser removida:

remove.packages("data.table")
install.packages("data.table")

[Visite o site para] 1 instruções completas de instalação e os números da versão mais recente.

Usando o pacote #

Normalmente você vai querer carregar o pacote e todas as suas funções com uma linha como

library(data.table)

Se você precisar apenas de uma ou duas funções, você pode se referir a elas como data.table::fread.

Sintaxe e recursos #

Sintaxe básica #

A sintaxe DT[where, select|update|do, by] é usada para trabalhar com colunas de um data.table.

  • A parte “onde” é o argumento i
  • A parte “select|update|do” é o argumento j

Esses dois argumentos geralmente são passados ​​por posição em vez de por nome.

Uma sequência de passos pode ser encadeada como DT[...][...].

Atalhos, funções especiais e símbolos especiais dentro de DT[...] #

Função ou símbolo Notas
.() em vários argumentos, substitui list()
J() em i, substitui list()
:= em j, uma função usada para adicionar ou modificar colunas
.N em i, o número total de linhas
em j, o número de linhas em um grupo
.I em j, o vetor de números de linha na tabela (filtrado por i)
.SD em j, o subconjunto atual dos dados
selecionados pelo argumento .SDcols
.GRP em j, o índice atual do subconjunto dos dados
.BY em j, a lista de valores por para o subconjunto atual de dados
V1, V2, ... nomes padrão para colunas sem nome criadas em j

Junta dentro de DT[...] #

Notação Notas
DT1[DT2, em, j] juntar duas tabelas
i.* prefixo especial nas colunas do DT2 após a junção
by=.EACHI opção especial disponível apenas com uma associação
DT1[!DT2, ligado, j] anti-junção de duas tabelas
DT1[DT2, on, roll, j] juntar duas tabelas, rolando na última coluna em on=

Remodelagem, empilhamento e divisão #

Notação Notas
melt(DT, id.vars, medida.vars) transforme para formato longo
para várias colunas, use measure.vars = patterns(...)
dcast(DT, fórmula) transformar para formato largo
rbind(DT1, DT2, ...) empilhar dados enumerados.tabelas
rbindlist(DT_list, idcol) empilhar uma lista de data.tables
split(DT, por) dividir um data.table em uma lista

Algumas outras funções especializadas para data.tables #

Função(ões) Notas
foverlaps junções sobrepostas
mesclar outra maneira de juntar duas tabelas
conjunto outra maneira de adicionar ou modificar colunas
fintersect, fsetdiff,
funion, fsetequal,
unique, duplicated, anyDuplicated		 operações de teoria de conjuntos com linhas como elementos
CJ o produto cartesiano de vetores
únicoN o número de linhas distintas
rowidv(DT, colunas) ID da linha (1 a .N) dentro de cada grupo determinado por colunas
rleidv(DT, colunas) ID do grupo (1 a .GRP) dentro de cada grupo determinado por execuções de colunas
shift(DT, n) aplicar um operador de deslocamento para cada coluna
setorder, setcolorder,
setnames, setkey, setindex,
setattr		 modificar atributos e ordenar por referência

Começando e encontrando ajuda #

O wiki oficial do pacote tem alguns materiais essenciais:

  • Como um novo usuário, você vai querer conferir as vignettes, FAQ e folha de dicas.

  • Antes de fazer uma pergunta - aqui no StackOverflow ou em qualquer outro lugar - leia a página de suporte.

Para ajuda em funções individuais, a sintaxe é help("fread") ou ?fread. Se o pacote não foi carregado, use o nome completo como ?data.table::fread.
