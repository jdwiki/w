Adicionando e modificando colunas

A vinheta oficial, “Semântica de referência”, é a melhor introdução a este tópico.

Um lembrete: a sintaxe DT[where, select|update|do, by] é usada para trabalhar com colunas de um data.table.

  • A parte “onde” é o argumento i
  • A parte “select|update|do” é o argumento j

Esses dois argumentos geralmente são passados ​​por posição em vez de por nome.

Todas as modificações nas colunas podem ser feitas em j. Além disso, a função set está disponível para este uso.

Editando valores #

# example data
DT = as.data.table(mtcars, keep.rownames = TRUE)

Editando uma coluna #

Use o operador := dentro de j para criar novas colunas ou modificar as existentes:

DT[, mpg_sq := mpg^2]

Editando em um subconjunto de linhas #

Use o argumento i para subconjunto de linhas “onde” as edições devem ser feitas:

DT[1:3, newvar := "Hello"]

Como em um data.frame, podemos subconjuntos usando números de linha ou testes lógicos. Também é possível usar [um “join” em i ao modificar][need_a_link].

Removendo uma coluna #

Remova as colunas definindo como NULL:

DT[, mpg_sq := NULL]

Observe que não atribuímos o resultado <-, pois o DT foi modificado no local.

Editando várias colunas #

Adicione várias colunas usando o formato multivariado do operador :=:

DT[, `:=`(mpg_sq = mpg^2, wt_sqrt = sqrt(wt))]
# or 
DT[, c("mpg_sq", "wt_sqrt") := .(mpg^2, sqrt(wt))]

A sintaxe .() é usada quando o lado direito de LHS := RHS é uma lista de colunas.

Editando várias colunas sequencialmente dependentes #

Se as colunas são dependentes e devem ser definidas em sequência, algumas maneiras de fazer isso são:

DT[, c("mpg_sq", "mpg2_hp") := .(temp1 <- mpg^2, temp1/hp)]
# or
DT[, c("mpg_sq", "mpg2_hp") := {temp1 = mpg^2; .(temp1, temp1/hp)}]

Editando colunas por nomes determinados dinamicamente #

Para nomes de coluna determinados dinamicamente, use parênteses:

vn = "mpg_sq"
DT[, (vn) := mpg^2]

Usando set #

As colunas também podem ser modificadas com set para uma pequena redução na sobrecarga, embora isso raramente seja necessário:

set(DT, j = "hp_over_wt", v = mtcars$hp/mtcars$wt)

Reordenando colunas #

# example data
DT = as.data.table(mtcars, keep.rownames = TRUE)

Para reorganizar a ordem das colunas, use setcolorder. Por exemplo, para revertê-los

setcolorder(DT, rev(names(DT)))

Isso não custa quase nada em termos de desempenho, pois é apenas permutar a lista de ponteiros de coluna no data.table.

Renomeando colunas #

# example data
DT = as.data.table(mtcars, keep.rownames = TRUE)

Para renomear uma coluna (mantendo seus dados iguais), não há necessidade de copiar os dados para uma coluna com um novo nome e excluir o antigo. Em vez disso, podemos usar

setnames(DT, "mpg_sq", "mpq_squared")

para modificar a coluna original por referência.

Modificando níveis de fator e outros atributos de coluna #

# example data
DT = data.table(iris)

Para modificar os níveis dos fatores por referência, use setattr:

setattr(DT$Species, "levels", c("set", "ver", "vir")
# or
DT[, setattr(Species, "levels", c("set", "ver", "vir"))]

A segunda opção pode imprimir o resultado na tela.

Com setattr, evitamos a cópia normalmente incorrida ao fazer levels(x) <- lvls, mas também irá pular algumas verificações, então é importante ter cuidado para atribuir um vetor de níveis válido.
