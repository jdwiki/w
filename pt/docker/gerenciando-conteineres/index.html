Sintaxe #

  • docker rm [OPÇÕES] CONTAINER [CONTAINER…]
  • anexo docker [OPÇÕES] CONTAINER
  • docker exec [OPÇÕES] CONTAINER COMMAND [ARG…]
  • docker ps [OPÇÕES]
  • logs docker [OPÇÕES] CONTAINER
  • docker inspecionar [OPÇÕES] CONTAINER|IMAGE [CONTAINER|IMAGE…]
  • Nos exemplos acima, sempre que container é um parâmetro do comando docker, ele é mencionado como <container> ou container id ou <CONTAINER_NAME>. Em todos esses lugares, você pode passar um nome de contêiner ou id de contêiner para especificar um contêiner.

Listando contêineres #

$ docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS                     NAMES
2bc9b1988080        redis               "docker-entrypoint.sh"   2 weeks ago         Up 2 hours          0.0.0.0:6379->6379/tcp    elephant-redis
817879be2230        postgres            "/docker-entrypoint.s"   2 weeks ago         Up 2 hours          0.0.0.0:65432->5432/tcp   pt-postgres

docker ps por si só imprime contêineres atualmente em execução. Para visualizar todos os contêineres (incluindo os parados), use o sinalizador -a:

$ docker ps -a
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS                    PORTS                     NAMES
9cc69f11a0f7        docker/whalesay     "ls /"                   26 hours ago        Exited (0) 26 hours ago                             berserk_wozniak
2bc9b1988080        redis               "docker-entrypoint.sh"   2 weeks ago         Up 2 hours                0.0.0.0:6379->6379/tcp    elephant-redis
817879be2230        postgres            "/docker-entrypoint.s"   2 weeks ago         Up 2 hours                0.0.0.0:65432->5432/tcp   pt-postgres

Para listar contêineres com um status específico, use a opção de linha de comando -f para filtrar os resultados. Aqui está um exemplo de listagem de todos os contêineres que saíram:

$ docker ps -a -f status=exited
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS                    PORTS                     NAMES
9cc69f11a0f7        docker/whalesay     "ls /"                   26 hours ago        Exited (0) 26 hours ago

Também é possível listar apenas os IDs do Container com a opção -q. Isso torna muito fácil operar no resultado com outros utilitários Unix (como grep e awk):

$ docker ps -aq
9cc69f11a0f7
2bc9b1988080
817879be2230

Ao lançar um container com docker run --name mycontainer1 você dá um nome específico e não um nome aleatório (na forma mood_famous, como nostalgic_stallman), e pode ser fácil encontrá-los com tal comando

docker ps -f name=mycontainer1

Remover, excluir e limpar contêineres #

docker rm pode ser usado para remover um contêiner específico como este:

docker rm <container name or id>

Para remover todos os contêineres, você pode usar esta expressão:

docker rm $(docker ps -qa)

Por padrão, o docker não excluirá um contêiner em execução. Qualquer contêiner em execução produzirá uma mensagem de aviso e não será excluído. Todos os outros contêineres serão excluídos.

Alternativamente, você pode usar xargs:

docker ps -aq -f status=exited | xargs -r docker rm

Onde docker ps -aq -f status=exited retornará uma lista de IDs de contêineres com status “Exited”.

Atenção: Todos os exemplos acima removerão somente containers ‘parados’.

Para remover um container, independentemente de estar ou não parado, você pode usar o sinalizador force -f:

docker rm -f <container name or id>

Para remover todos os contêineres, independentemente do estado:

docker rm -f $(docker ps -qa)

Se você deseja remover apenas contêineres com status “morto”:

docker rm $(docker ps --all -q -f status=dead)

Se você deseja remover apenas contêineres com status exited:

docker rm $(docker ps --all -q -f status=exited)

Essas são todas as permutações de filtros usados ​​ao [listar contêineres][1].

Para remover contêineres indesejados e imagens pendentes que usam espaço após a [versão 1.3][2], use o seguinte (semelhante à ferramenta Unix df):

$ docker system df

Para remover todos os dados não utilizados:

$ docker system prune

[1]: https://www.wikiod.com/pt/docker/gerenciando-conteineres#Como listar contêineres [2]: https://blog.docker.com/2017/01/whats-new-in-docker-1-13/

Referenciando contêineres #

Os comandos do Docker que levam o nome de um contêiner aceitam três formas diferentes:

Tipo Exemplo
UUID completo 9cc69f11a0f76073e87f25cb6eaf0e079fbfbd1bc47c063bcd25ed3722a8cc4a
UUID curto 9cc69f11a0f7
Nome berserk_wozniak

Use docker ps para visualizar esses valores para os contêineres em seu sistema.

O UUID é gerado pelo Docker e não pode ser modificado. Você pode fornecer um nome para o contêiner ao iniciá-lo docker run --name <given name> <image>. O Docker gerará um nome aleatório para o contêiner se você não especificar um no momento de iniciar o contêiner.

NOTA: O valor do UUID (ou um UUID ‘curto’) pode ter qualquer comprimento, desde que o valor fornecido seja exclusivo para um contêiner

Conecte-se a uma instância em execução como daemon #

Existem duas maneiras de conseguir isso, a primeira e mais conhecida é a seguinte:

docker attach --sig-proxy=false <container>

Este literalmente anexa seu bash ao contêiner bash, o que significa que, se você tiver um script em execução, verá o resultado.

Para desanexar, basta digitar: Ctl-P Ctl-Q

Mas se você precisar de uma maneira mais amigável e poder criar novas instâncias do bash, basta executar o seguinte comando:

docker exec -it <container> bash

Encontrando um container específico #

docker ps --filter name=myapp_1

Copiando arquivo de/para contêineres #

do contêiner ao host

docker cp CONTAINER_NAME:PATH_IN_CONTAINER PATH_IN_HOST

do host ao contêiner

docker cp PATH_IN_HOST CONTAINER_NAME:PATH_IN_CONTAINER

Se eu usar jess/transmissão de

https://hub.docker.com/r/jess/transmission/builds/bsn7eqxrkzrhxazcuytbmzp/

, os arquivos no contêiner estão em /transmission/download

e meu diretório atual no host é /home/$USER/abc, depois

docker cp transmit_id_or_name:/transmission/download .

Vou copiar os arquivos para

/home/$USER/abc/transmission/download

você não pode, usando o docker cp copiar apenas um arquivo, você copia a árvore de diretórios e os arquivos

Iniciando e parando contêineres #

Para interromper um contêiner em execução:

docker stop <container> [<container>...]

Isso enviará ao processo principal no contêiner um SIGTERM, seguido por um SIGKILL se não parar dentro do período de carência. O nome de cada contêiner é impresso quando ele para.

Para iniciar um contêiner que está parado:

docker start <container> [<container>...]

Isso iniciará cada contêiner passado em segundo plano; o nome de cada container é impresso assim que começa. Para iniciar o container em primeiro plano, passe o sinalizador -a (--attach).

Listar contêineres com formato personalizado #

docker ps --format 'table {{.ID}}\t{{.Names}}\t{{.Status}}'

Encontre o IP do contêiner #

Para descobrir o endereço IP do seu contêiner, use:

docker inspect <container id> | grep IPAddress

ou use o docker inspecionar

docker inspect --format '{{ .NetworkSettings.IPAddress }}' ${CID}

Reiniciando o contêiner docker #

docker restart <container> [<container>...]

Opção –time : segundos para aguardar a parada antes de matar o contêiner (padrão 10)

docker restart <container> --time 10

Execute o comando em um contêiner docker já existente #

docker exec -it <container id> /bin/bash

É comum fazer login em um container já em execução para fazer alguns testes rápidos ou ver o que a aplicação está fazendo. Muitas vezes denota práticas ruins de uso de contêiner devido a logs e arquivos alterados devem ser colocados em volumes. Este exemplo nos permite fazer login no container. Isso supõe que /bin/bash está disponível no contêiner, pode ser /bin/sh ou qualquer outra coisa.

docker exec <container id> tar -czvf /tmp/backup.tgz /data
docker cp <container id>:/tmp/backup.tgz .

Este exemplo arquiva o conteúdo do diretório de dados em um arquivo tar. Então com docker cp você pode recuperá-lo.

Registros de contêiner #

Usage:  docker logs [OPTIONS] CONTAINER

Fetch the logs of a container

  -f, --follow=false        Follow log output
  --help=false              Print usage
  --since=                  Show logs since timestamp
  -t, --timestamps=false    Show timestamps
  --tail=all                Number of lines to show from the end of the logs

$ docker ps
CONTAINER ID    IMAGE    COMMAND                 CREATED     STATUS       PORTS

$ docker logs ff9716dda6cb

Remova, exclua e limpe os volumes do docker #

Os volumes do Docker não são removidos automaticamente quando um contêiner é interrompido. Para remover volumes associados ao interromper um contêiner:

docker rm -v <container id or name>

Se o sinalizador -v não for especificado, o volume permanecerá no disco como um ‘volume pendente’. Para excluir todos os volumes pendentes:

docker volume rm $(docker volume ls -qf dangling=true)

O filtro docker volume ls -qf dangling=true retornará uma lista de nomes de volumes docker, incluindo os não marcados, que não estão anexados a um contêiner.

Alternativamente, você pode usar xargs:

docker volume ls -f dangling=true -q | xargs --no-run-if-empty docker volume rm

Exporte e importe sistemas de arquivos de contêiner do Docker #

É possível salvar o conteúdo do sistema de arquivos de um contêiner do Docker em um arquivo tarball. Isso é útil para mover sistemas de arquivos de contêiner para hosts diferentes, por exemplo, se um contêiner de banco de dados tiver alterações importantes e não for possível replicar essas alterações em outro lugar. Observe que é preferível criar um contêiner totalmente novo a partir de uma imagem atualizada usando um comando docker run ou um arquivo docker-compose.yml, em vez de exportar e mover o sistema de arquivos de um contêiner. Parte do poder do Docker é a auditabilidade e a responsabilidade de seu estilo declarativo de criação de imagens e contêineres. Ao usar docker export e docker import, esse poder é subjugado devido à ofuscação de alterações feitas dentro do sistema de arquivos de um contêiner a partir de seu estado original.

docker export -o redis.tar redis

O comando acima criará uma imagem vazia e exportará o sistema de arquivos do contêiner redis para esta imagem vazia. Para importar de um arquivo tarball, use:

docker import ./redis.tar redis-imported:3.0.7

Este comando irá criar a imagem redis-imported:3.0.7, a partir da qual os containers podem ser criados. Também é possível criar alterações na importação, bem como definir uma mensagem de commit:

docker import -c="ENV DEBUG true" -m="enable debug mode" ./redis.tar redis-changed

As diretivas do Dockerfile disponíveis para uso com a opção de linha de comando -c são CMD, ENTRYPOINT, ENV, EXPOSE, ONBUILD, USER, VOLUME, WORKDIR.
