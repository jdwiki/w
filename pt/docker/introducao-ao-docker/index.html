Instalando o Docker no Windows #

Requisitos: versão de 64 bits do Windows 7 ou superior em uma máquina que suporte a Tecnologia de Virtualização de Hardware e esteja habilitada.

Embora o binário docker possa ser executado nativamente no Windows, para criar e hospedar contêineres, você precisa executar uma máquina virtual Linux na caixa.

Desde a versão 1.12, você não precisa ter uma VM separada para ser instalada, pois o Docker pode usar a funcionalidade nativa do Hyper-V do Windows para iniciar uma pequena máquina Linux para atuar como back-end.

Para instalar o docker siga os seguintes passos:

  1. Vá para Docker para Windows
  2. Baixe e execute o instalador.
  3. Continue com o instalador com as opções padrão e insira as credenciais da sua conta quando solicitado.

Verifique aqui para mais informações sobre a instalação.

Até a versão 1.11, a melhor maneira de executar essa VM Linux é instalar o Docker Toolbox, que instala o Docker, o VirtualBox e a máquina convidada do Linux.

Para instalar a caixa de ferramentas do docker, siga as etapas a seguir:

  1. Acesse Docker Toolbox
  2. Clique no link para Windows e execute o instalador.
  3. Continue com o instalador com as opções padrão e insira as credenciais da sua conta quando solicitado.

Isso instalará os binários do Docker em Arquivos de programas e atualizará qualquer instalação existente do Virtual Box. Verifique aqui para obter mais informações sobre a instalação.

Para verificar a instalação:

  1. Inicie o Docker a partir do menu Iniciar se ainda não tiver sido iniciado, e certifique-se de que esteja rodando. Em seguida, abra qualquer terminal (‘cmd’ ou PowerShell)
  1. Na área de trabalho, localize o ícone do Docker Toolbox. Clique no ícone para iniciar um terminal do Docker Toolbox.

  1. Quando o terminal estiver aberto, digite

    docker run hello-world

  2. Se tudo estiver bem, isso deve imprimir uma mensagem de boas-vindas, verificando se a instalação foi bem-sucedida.

Desde a versão 1.12, você não precisa ter uma VM separada para ser instalada, pois o Docker pode usar a funcionalidade nativa Hypervisor.framework do OSX para iniciar uma pequena máquina Linux para atuar como backend.

Para instalar o docker siga os seguintes passos:

  1. Vá para Docker para Mac
  2. Baixe e execute o instalador.
  3. Continue com o instalador com as opções padrão e insira as credenciais da sua conta quando solicitado.

Verifique aqui para mais informações sobre a instalação.

Até a versão 1.11, a melhor maneira de executar essa VM Linux é instalar o Docker Toolbox, que instala o Docker, o VirtualBox e a máquina convidada do Linux.

Para instalar a caixa de ferramentas do docker, siga as etapas a seguir:

  1. Acesse Docker Toolbox
  2. Clique no link para Mac e execute o instalador.
  3. Continue com o instalador com as opções padrão e insira as credenciais da sua conta quando solicitado.

Isso instalará os binários do Docker em /usr/local/bin e atualizará qualquer instalação do Virtual Box existente. Verifique aqui para obter mais informações sobre a instalação.

Para verificar a instalação:

Instalando o docker no Ubuntu Linux #

O Docker é compatível com as seguintes versões de 64 bits do Ubuntu Linux:

*Ubuntu Xenial 16.04 (LTS)

  • Wily grátis 15.10
  • Ubuntu Trusty 14.04 (LTS)
  • Ubuntu Preciso 12.04 (LTS)

Algumas notas:

As instruções a seguir envolvem a instalação usando apenas pacotes Docker, e isso garante a obtenção da versão oficial mais recente do Docker. Se você precisar instalar apenas usando pacotes Gerenciado pelo Ubuntu, consulte a documentação do Ubuntu (não recomendado de outra forma por razões óbvias).

Ubuntu Utopic 14.10 e 15.04 existem no repositório APT do Docker, mas não são mais oficialmente suportados devido a problemas de segurança conhecidos.

Pré-requisitos

  • O Docker funciona apenas em uma instalação de 64 bits do Linux.
  • Docker requer kernel Linux versão 3.10 ou superior (Exceto para Ubuntu Precise 12.04, que requer versão 3.13 ou superior). Os kernels anteriores à versão 3.10 não possuem alguns dos recursos necessários para executar contêineres do Docker e contêm bugs conhecidos que causam perda de dados e frequentemente entram em pânico sob certas condições. Verifique a versão atual do kernel com o comando uname -r. Verifique este post se você precisar atualizar seu kernel Ubuntu Precise (12.04 LTS) rolando mais para baixo. Consulte este WikiHow post para obter a versão mais recente para outras instalações do Ubuntu.

Atualizar fontes APT

Isso precisa ser feito para acessar os pacotes do repositório do Docker.

  1. Faça login em sua máquina como um usuário com privilégios sudo ou root.

  2. Abra uma janela de terminal.

  3. Atualize as informações do pacote, certifique-se de que o APT funcione com o método https e que os certificados de CA estejam instalados.

    $ sudo apt-get update $ sudo apt-get install
    apt-transport-https
    ca-certificates
    curl
    software-properties-common

  4. Adicione a chave GPG oficial do Docker:

     $ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

Verifique se a impressão digital da chave é 9DC8 5822 9FC7 DD38 854A E2D8 8D81 803C 0EBF CD88.

    $ sudo apt-key fingerprint 0EBFCD88

        
        pub   4096R/0EBFCD88 2017-02-22
              Key fingerprint = 9DC8 5822 9FC7 DD38 854A  E2D8 8D81 803C 0EBF CD88
        uid                  Docker Release (CE deb) <docker@docker.com>
        sub   4096R/F273FCD8 2017-02-22
  1. Encontre a entrada na tabela abaixo que corresponde à sua versão do Ubuntu. Isso determina onde o APT procurará os pacotes do Docker. Quando possível, execute uma edição de suporte de longo prazo (LTS) do Ubuntu.
Versão do Ubuntu Repositório
Preciso 12.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-precise main
Trusty 14.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-trusty main
Wily 15.10 deb https://apt.dockerproject.org/repo ubuntu-wily main
Xenial 16.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-xenial main

Observação: o Docker não fornece pacotes para todas as arquiteturas. Artefatos binários são construídos todas as noites e você pode baixá-los em https://master.dockerproject.org. Para instalar o docker em um sistema multi-arquitetura, adicione uma cláusula [arch=...] à entrada. Consulte wiki do Debian Multiarch para detalhes.

  1. Execute o seguinte comando, substituindo a entrada do seu sistema operacional pelo espaço reservado <REPO>.

    $ echo “” | sudo tee /etc/apt/sources.list.d/docker.list

  2. Atualize o índice do pacote APT executando sudo apt-get update.

  3. Verifique se o APT está puxando do repositório correto.

Quando você executa o comando a seguir, uma entrada é retornada para cada versão do Docker disponível para instalação. Cada entrada deve ter a URL https://apt.dockerproject.org/repo/. A versão atualmente instalada está marcada com ***. Veja a saída do exemplo abaixo.

$ apt-cache policy docker-engine

  docker-engine:
    Installed: 1.12.2-0~trusty
    Candidate: 1.12.2-0~trusty
    Version table:
   *** 1.12.2-0~trusty 0
          500 https://apt.dockerproject.org/repo/ ubuntu-trusty/main amd64 Packages
          100 /var/lib/dpkg/status
       1.12.1-0~trusty 0
          500 https://apt.dockerproject.org/repo/ ubuntu-trusty/main amd64 Packages
       1.12.0-0~trusty 0
          500 https://apt.dockerproject.org/repo/ ubuntu-trusty/main amd64 Packages

A partir de agora, quando você executar o apt-get upgrade, o APT será extraído do novo repositório.

Pré-requisitos por versão do Ubuntu

Para Ubuntu Trusty (14.04) , Wily (15.10) e Xenial (16.04) , instale os pacotes de kernel linux-image-extra-*, que permitem que você use o driver de armazenamento aufs.

Para instalar os pacotes linux-image-extra-*:

  1. Abra um terminal no seu host Ubuntu.

  2. Atualize seu gerenciador de pacotes com o comando sudo apt-get update.

  3. Instale os pacotes recomendados.

    $ sudo apt-get install linux-image-extra-$(uname -r) linux-image-extra-virtual

  4. Prossiga para a instalação do Docker

Para o Ubuntu Precise (12.04 LTS), o Docker requer a versão do kernel 3.13. Se a versão do seu kernel for anterior a 3.13, você deve atualizá-la. Consulte esta tabela para ver quais pacotes são necessários para seu ambiente:

Pacote Descrição
linux-image-generic-lts-trusty Imagem genérica do kernel Linux. Este kernel tem o AUFS embutido. Isso é necessário para executar o Docker.
linux-headers-generic-lts-trusty Permite pacotes como ZFS e adições de convidados do VirtualBox que dependem deles. Se você não instalou os cabeçalhos para o kernel existente, você pode pular esses cabeçalhos para o kernel trusty. Se não tiver certeza, inclua este pacote por segurança.
xserver-xorg-lts-trusty Opcional em ambientes não gráficos sem Unity/Xorg. Obrigatório ao executar o Docker em uma máquina com ambiente gráfico.
ligbl1-mesa-glx-lts-trusty Para saber mais sobre os motivos desses pacotes, leia as instruções de instalação para kernels com backport, especificamente o LTS Enablement Stack. Consulte a nota 5 em cada versão.

Para atualizar seu kernel e instalar os pacotes adicionais, faça o seguinte:

  1. Abra um terminal em seu host Ubuntu.

  2. Atualize seu gerenciador de pacotes com o comando sudo apt-get update.

  3. Instale os pacotes obrigatórios e opcionais.

    $ sudo apt-get install linux-image-generic-lts-trusty

  4. Repita esta etapa para outros pacotes que você precisa instalar.

  5. Reinicie seu host para usar o kernel atualizado usando o comando sudo reboot.

  6. Após a reinicialização, vá em frente e instale o Docker.

Instale a versão mais recente

Certifique-se de atender aos pré-requisitos, só então siga os passos abaixo.

Observação: para sistemas de produção, é recomendável que você instale uma versão específica para não atualizar acidentalmente o Docker. Você deve planejar as atualizações dos sistemas de produção com cuidado.

  1. Faça login na sua instalação do Ubuntu como um usuário com privilégios sudo. (Possivelmente executando sudo -su).

  2. Atualize seu índice de pacotes APT executando sudo apt-get update.

  3. Instale o Docker Community Edition com o comando sudo apt-get install docker-ce.

  4. Inicie o daemon docker com o comando sudo service docker start.

  5. Verifique se o docker está instalado corretamente executando a imagem hello-world.

     $ sudo docker run hello-world

Este comando baixa uma imagem de teste e a executa em um contêiner. Quando o contêiner é executado, ele imprime uma mensagem informativa e sai.

Gerenciar o Docker como um usuário não root

Se você não quiser usar sudo ao usar o comando docker, crie um grupo Unix chamado docker e adicione usuários a ele. Quando o daemon docker é iniciado, ele torna a propriedade do soquete Unix de leitura/gravação pelo grupo docker.

Para criar o grupo docker e adicionar seu usuário:

  1. Faça login no Ubuntu como um usuário com privilégios sudo.

  2. Crie o grupo docker com o comando sudo groupadd docker.

  3. Adicione seu usuário ao grupo docker.

     $ sudo usermod -aG docker $USER

  4. Saia e faça login novamente para que sua associação ao grupo seja reavaliada.

  5. Verifique se você pode comandos docker sem permissão sudo.

     $ docker run hello-world

Se isso falhar, você verá um erro:

     Cannot connect to the Docker daemon. Is 'docker daemon' running on this host?

Verifique se a variável de ambiente DOCKER_HOST está definida para seu shell.

    $ env | grep DOCKER_HOST

Se estiver definido, o comando acima retornará um resultado. Em caso afirmativo, desmarque-o.

    $ unset DOCKER_HOST

Você pode precisar editar seu ambiente em arquivos como ~/.bashrc ou ~/.profile para evitar que a variável DOCKER_HOST seja configurada incorretamente.

Instalando o Docker no Ubuntu #

Requisitos: O Docker pode ser instalado em qualquer Linux com um kernel de pelo menos a versão 3.10. O Docker é compatível com as seguintes versões de 64 bits do Ubuntu Linux:

*Ubuntu Xenial 16.04 (LTS)

  • Wily grátis 15.10
  • Ubuntu Trusty 14.04 (LTS)
  • Ubuntu Preciso 12.04 (LTS)

Instalação fácil

Observação: a instalação do Docker a partir do repositório padrão do Ubuntu instalará uma versão antiga do Docker.

Para instalar a versão mais recente do Docker usando o repositório do Docker, use curl para pegar e executar o script de instalação fornecido pelo Docker:

$ curl -sSL https://get.docker.com/ | sh

Alternativamente, wget pode ser usado para instalar o Docker:

$ wget -qO- https://get.docker.com/ | sh

O Docker agora será instalado.

Instalação Manual

Se, no entanto, executar o script de instalação não for uma opção, as instruções a seguir podem ser usadas para instalar manualmente a versão mais recente do Docker do repositório oficial.

$ sudo apt-get update
$ sudo apt-get install apt-transport-https ca-certificates

Adicione a chave GPG:

$ sudo apt-key adv --keyserver hkp://p80.pool.sks-keyservers.net:80 \
  --recv-keys 58118E89F3A912897C070ADBF76221572C52609D

Em seguida, abra o arquivo /etc/apt/sources.list.d/docker.list em seu editor favorito. Se o arquivo não existir, crie-o. Remova todas as entradas existentes. Então, dependendo da sua versão, adicione a seguinte linha:

  • Ubuntu Precise 12.04 (LTS):

    deb https://apt.dockerproject.org/repo ubuntu-precise main

  • Ubuntu Trusty 14.04 (LTS)

    deb https://apt.dockerproject.org/repo ubuntu-trusty main

  • Wily gratuito 15.10

    deb https://apt.dockerproject.org/repo ubuntu-wily main

-Ubuntu Xenial 16.04 (LTS)

`deb https://apt.dockerproject.org/repo ubuntu-xenial main`

Salve o arquivo e saia, atualize seu índice de pacotes, desinstale qualquer versão instalada do Docker e verifique se o apt está puxando do repositório correto:

$ sudo apt-get update
$ sudo apt-get purge lxc-docker
$ sudo apt-cache policy docker-engine

Dependendo da sua versão do Ubuntu, alguns pré-requisitos podem ser necessários:

  • Ubuntu Xenial 16.04 (LTS), Ubuntu Wily 15.10, Ubuntu Trusty 14.04 (LTS)

    sudo apt-get update && sudo apt-get install linux-image-extra-$(uname -r)

  • Ubuntu Precise 12.04 (LTS)

    This version of Ubuntu requires kernel version 3.13. You may need to install additional packages depending on your environment:

    linux-image-generic-lts-trusty

    Generic Linux kernel image. This kernel has AUFS built in. This is required to run Docker.

    linux-headers-generic-lts-trusty

    Allows packages such as ZFS and VirtualBox guest additions which depend on them. If you didn’t install the headers for your existing kernel, then you can skip these headers for the trusty kernel. If you’re unsure, you should include this package for safety.

    xserver-xorg-lts-trusty

    libgl1-mesa-glx-lts-trusty

    These two packages are optional in non-graphical environments without Unity/Xorg. Required when running Docker on machine with a graphical environment.

    To learn more about the reasons for these packages, read the installation instructions for backported kernels, specifically the LTS Enablement Stack — refer to note 5 under each version.

    Install the required packages then reboot the host:

    $ sudo apt-get install linux-image-generic-lts-trusty

    $ sudo reboot

Finalmente, atualize o índice do pacote apt e instale o Docker:

$ sudo apt-get update
$ sudo apt-get install docker-engine

Inicie o daemon:

$ sudo service docker start

Agora verifique se o docker está funcionando corretamente iniciando uma imagem de teste:

$ sudo docker run hello-world

Este comando deve imprimir uma mensagem de boas-vindas verificando se a instalação foi bem-sucedida.

Crie um contêiner docker no Google Cloud #

Você pode usar o docker, sem usar o daemon do docker (mecanismo), usando provedores de nuvem. Neste exemplo, você deve ter um gcloud (util do Google Cloud), conectado à sua conta

docker-machine create --driver google --google-project `your-project-name` google-machine-type f1-large fm02

Este exemplo criará uma nova instância no console do Google Cloud. Usando o tempo de máquina f1-large

  5. Encontre a entrada na tabela abaixo que corresponde à sua versão do Ubuntu. Isso determina onde o APT procurará os pacotes do Docker. Quando possível, execute uma edição de suporte de longo prazo (LTS) do Ubuntu.

Versão do Ubuntu Repositório

Wily 15.10 deb https://apt.dockerproject.org/repo ubuntu-wily main

Observação: o Docker não fornece pacotes para todas as arquiteturas. Artefatos binários são construídos todas as noites e você pode baixá-los em https://master.dockerproject.org. Para instalar o docker em um sistema multi-arquitetura, adicione uma cláusula [arch=...] à entrada. Consulte wiki do Debian Multiarch para detalhes.

  1. Execute o seguinte comando, substituindo a entrada do seu sistema operacional pelo espaço reservado <REPO>.

    $ echo “” | sudo tee /etc/apt/sources.list.d/docker.list

  2. Atualize o índice do pacote APT executando sudo apt-get update.

  3. Verifique se o APT está puxando do repositório correto.

Quando você executa o comando a seguir, uma entrada é retornada para cada versão do Docker disponível para instalação. Cada entrada deve ter a URL https://apt.dockerproject.org/repo/. A versão atualmente instalada está marcada com ***. Veja a saída do exemplo abaixo.

Pré-requisitos por versão do Ubuntu

Para Ubuntu Trusty (14.04) , Wily (15.10) e Xenial (16.04) , instale os pacotes de kernel linux-image-extra-*, que permitem que você use o driver de armazenamento aufs.

Para instalar os pacotes linux-image-extra-*:

  1. Abra um terminal no seu host Ubuntu.

  2. Atualize seu gerenciador de pacotes com o comando sudo apt-get update.

  3. Instale os pacotes recomendados.

    $ sudo apt-get install linux-image-extra-$(uname -r) linux-image-extra-virtual

  4. Prossiga para a instalação do Docker

Para o Ubuntu Precise (12.04 LTS), o Docker requer a versão do kernel 3.13. Se a versão do seu kernel for anterior a 3.13, você deve atualizá-la. Consulte esta tabela para ver quais pacotes são necessários para seu ambiente:

Pacote Descrição
linux-image-generic-lts-trusty Imagem genérica do kernel Linux. Este kernel tem o AUFS embutido. Isso é necessário para executar o Docker.
linux-headers-generic-lts-trusty Permite pacotes como ZFS e adições de convidados do VirtualBox que dependem deles. Se você não instalou os cabeçalhos para o kernel existente, você pode pular esses cabeçalhos para o kernel trusty. Se não tiver certeza, inclua este pacote por segurança.
xserver-xorg-lts-trusty Opcional em ambientes não gráficos sem Unity/Xorg. Obrigatório ao executar o Docker em uma máquina com ambiente gráfico.
ligbl1-mesa-glx-lts-trusty Para saber mais sobre os motivos desses pacotes, leia as instruções de instalação para kernels com backport, especificamente o LTS Enablement Stack. Consulte a nota 5 em cada versão.

Para atualizar seu kernel e instalar os pacotes adicionais, faça o seguinte:

  1. Abra um terminal em seu host Ubuntu.

  2. Atualize seu gerenciador de pacotes com o comando sudo apt-get update.

  3. Instale os pacotes obrigatórios e opcionais.

    $ sudo apt-get install linux-image-generic-lts-trusty

  4. Repita esta etapa para outros pacotes que você precisa instalar.

  5. Reinicie seu host para usar o kernel atualizado usando o comando sudo reboot.

  6. Após a reinicialização, vá em frente e instale o Docker.

Instale a versão mais recente

Certifique-se de atender aos pré-requisitos, só então siga os passos abaixo.

Observação: para sistemas de produção, é recomendável que você instale uma versão específica para não atualizar acidentalmente o Docker. Você deve planejar as atualizações dos sistemas de produção com cuidado.

  1. Faça login na sua instalação do Ubuntu como um usuário com privilégios sudo. (Possivelmente executando sudo -su).

  2. Atualize seu índice de pacotes APT executando sudo apt-get update.

  3. Instale o Docker com o comando sudo apt-get install docker-engine.

  4. Inicie o daemon docker com o comando sudo service docker start.

  5. Verifique se o docker está instalado corretamente executando a imagem hello-world.

     $ sudo docker run hello-world

Este comando baixa uma imagem de teste e a executa em um contêiner. Quando o contêiner é executado, ele imprime uma mensagem informativa e sai.

Gerenciar o Docker como um usuário não root

Se você não quiser usar sudo ao usar o comando docker, crie um grupo Unix chamado docker e adicione usuários a ele. Quando o daemon docker é iniciado, ele torna a propriedade do soquete Unix de leitura/gravação pelo grupo docker.

Para criar o grupo docker e adicionar seu usuário:

  1. Faça login no Ubuntu como um usuário com privilégios sudo.

  2. Crie o grupo docker com o comando sudo groupadd docker.

  3. Adicione seu usuário ao grupo docker.

     $ sudo usermod -aG docker $USER

  4. Saia e faça login novamente para que sua associação ao grupo seja reavaliada.

  5. Verifique se você pode comandos docker sem permissão sudo.

     $ docker run hello-world

Se isso falhar, você verá um erro:

     Cannot connect to the Docker daemon. Is 'docker daemon' running on this host?

Verifique se a variável de ambiente DOCKER_HOST está definida para seu shell.

    $ env | grep DOCKER_HOST

Se estiver definido, o comando acima retornará um resultado. Em caso afirmativo, desmarque-o.

    $ unset DOCKER_HOST

Você pode precisar editar seu ambiente em arquivos como ~/.bashrc ou ~/.profile para evitar que a variável DOCKER_HOST seja configurada incorretamente.

Instalando o Docker-ce OU Docker-ee no CentOS #

Docker anunciou as seguintes edições:

-Docker-ee (Enterprise Edition) junto com Docker-ce (Community Edition) e Docker (Commercial Support)

Este documento irá ajudá-lo com as etapas de instalação da edição Docker-ee e Docker-ce no CentOS

Instalação do Docker-ce

A seguir estão as etapas para instalar a edição docker-ce

  1. Instale o yum-utils, que fornece o utilitário yum-config-manager:

    $ sudo yum install -y yum-utils

  2. Use o seguinte comando para configurar o repositório estável:

$ sudo yum-config-manager \
    --add-repo \
    https://download.docker.com/linux/centos/docker-ce.repo

  1. Opcional: Ative o repositório de borda. Este repositório está incluído no arquivo docker.repo acima, mas está desabilitado por padrão. Você pode habilitá-lo junto com o repositório estável.

     $ sudo yum-config-manager --enable docker-ce-edge
  • Você pode desabilitar o repositório de borda executando o comando yum-config-manager com o sinalizador --disable. Para reativá-lo, use o sinalizador --enable. O comando a seguir desabilita o repositório de borda. 
     $ sudo yum-config-manager --disable docker-ce-edge

  1. Atualize o índice do pacote yum.

     $ sudo yum makecache fast

  2. Instale o docker-ce usando o seguinte comando:

     $ sudo yum install docker-ce-17.03.0.ce

  3. Confirme a impressão digital do Docker-ce

    060A 61C5 1B55 8A7F 742B 77AA C52F EB6B 621E 9F35

    If you want to install some other version of docker-ce you can use following command:

    $ sudo yum install docker-ce-VERSION

Especifique o número VERSION

  1. Se tudo correu bem, o docker-ce agora está instalado em seu sistema, use o seguinte comando para iniciar:

     $ sudo systemctl start docker

  2. Teste a instalação do docker:

     $ sudo docker run hello-world

    you should get following message:

     Hello from Docker! 
 This message shows that your installation appears to be working correctly.

-Instalação do Docker-ee (Enterprise Edition)

Para Enterprise Edition (EE), seria necessário se inscrever para obter seu <DOCKER-EE-URL>.

  1. Para se inscrever, acesse https://cloud.docker.com/. Insira seus dados e confirme seu ID de e-mail. Após a confirmação, você receberá um <DOCKER-EE-URL>, que pode ser visto em seu painel após clicar em configuração.

  2. Remova quaisquer repositórios Docker existentes de /etc/yum.repos.d/

  3. Armazene a URL do repositório do Docker EE em uma variável yum em /etc/yum/vars/. Substitua <DOCKER-EE-URL> com o URL que você anotou na primeira etapa.

     $ sudo sh -c 'echo "<DOCKER-EE-URL>" > /etc/yum/vars/dockerurl'

  4. Instale o yum-utils, que fornece o utilitário yum-config-manager:

     $ sudo yum install -y yum-utils

  5. Use o seguinte comando para adicionar o repositório estável:

     $ sudo yum-config-manager \
 --add-repo \
 <DOCKER-EE-URL>/docker-ee.repo

  6. Atualize o índice do pacote yum.

     $ sudo yum makecache fast

  7. Instale o docker-ee

     sudo yum install docker-ee

  8. Você pode iniciar o docker-ee usando o seguinte comando:

    $ sudo systemctl start docker
