Sintaxe #

  • docker volume create –name <volume_name> # Cria um volume chamado <volume_name>
  • docker run -v <volume_name>:<mount_point> -d crramirez/limesurvey:latest # Monta o volume <volume_name> no diretório <mount_point> no contêiner

Parâmetros #

Parâmetro Detalhes
–name <nome_volume> Especifique o nome do volume a ser criado
-v <volume_name>:<mount_point> Especifique onde o volume nomeado será montado no contêiner

A persistência é criada em contêineres docker usando volumes. O Docker tem muitas maneiras de lidar com volumes. Volumes nomeados são muito convenientes por:

  • Eles persistem mesmo quando o contêiner é removido usando a opção -v.
  • A única maneira de excluir um volume nomeado é fazer uma chamada explícita para o volume docker rm
  • Os volumes nomeados podem ser compartilhados entre contêineres sem vinculação ou opção –volumes-from.
  • Eles não têm problemas de permissão que os volumes montados no host têm.
  • Eles podem ser manipulados usando o comando docker volume.

Faça backup de um conteúdo de volume nomeado #

Precisamos criar um container para montar o volume. Em seguida, arquive-o e baixe o arquivo para o nosso host.

Vamos criar primeiro um volume de dados com alguns dados:

docker volume create --name=data
echo "Hello World" |  docker run -i --rm=true -v data:/data ubuntu:trusty tee /data/hello.txt

Vamos fazer backup dos dados:

docker run -d --name backup -v data:/data ubuntu:trusty tar -czvf /tmp/data.tgz /data
docker cp backup:/tmp/data.tgz data.tgz
docker rm -fv backup

Vamos testar:

tar -xzvf data.tgz
cat data/hello.txt

Persistência com volumes nomeados #

A persistência é criada em contêineres docker usando volumes. Vamos criar um container Limesurvey e persistir o banco de dados, conteúdo carregado e configuração em um volume nomeado:

docker volume create --name mysql
docker volume create --name upload

docker run -d --name limesurvey -v mysql:/var/lib/mysql -v upload:/app/upload -p 80:80 crramirez/limesurvey:latest
