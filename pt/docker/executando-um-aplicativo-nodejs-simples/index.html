Executando um aplicativo Node.js básico dentro de um contêiner #

O exemplo que vou discutir pressupõe que você tenha uma instalação do Docker que funcione em seu sistema e um entendimento básico de como trabalhar com Node.js . Se você está ciente de como deve trabalhar com o Docker , deve ficar evidente que o framework Node.js não precisa ser instalado em seu sistema, ao invés disso, estaríamos usando a versão latest da imagem node disponível no Docker. Portanto, se necessário, você pode baixar a imagem antecipadamente com o comando docker pull node. (O comando puxa automaticamente a versão mais recente da imagem node do docker.)

  1. Prossiga para criar um diretório onde todos os arquivos do aplicativo de trabalho residiriam. Crie um arquivo package.json neste diretório que descreva seu aplicativo, bem como as dependências. Seu arquivo package.json deve ficar assim:

     {
   "name": "docker_web_app",
   "version": "1.0.0",
   "description": "Node.js on Docker",
   "author": "First Last <first.last@example.com>",
   "main": "server.js",
   "scripts": {
     "start": "node server.js"
   },
   "dependencies": {
     "express": "^4.13.3"
   }
 }

  1. Se precisarmos trabalhar com Node.js, normalmente criamos um arquivo server que define uma aplicação web. Neste caso usamos o framework Express.js (versão 4.13.3 em diante). Um arquivo server.js básico seria algo assim:

     var express = require('express');
 var PORT = 8080;
 var app = express();
 app.get('/', function (req, res) {
   res.send('Hello world\n');
 });

 app.listen(PORT);
 console.log('Running on http://localhost:' + PORT);

  2. Para aqueles familiarizados com o Docker, você já deve ter encontrado um Dockerfile. Um Dockerfile é um arquivo de texto que contém todos os comandos necessários para criar uma imagem personalizada sob medida para seu aplicativo.

Crie um arquivo de texto vazio chamado Dockerfile no diretório atual. O método para criar um é simples no Windows. No Linux, você pode querer executar touch Dockerfile no diretório que contém todos os arquivos necessários para sua aplicação. Abra o Dockerfile com qualquer editor de texto e adicione as seguintes linhas:

FROM node:latest
RUN mkdir -p /usr/src/my_first_app
WORKDIR /usr/src/my_first_app
COPY package.json /usr/src/my_first_app/
RUN npm install
COPY . /usr/src/my_first_app
EXPOSE 8080

  • FROM node:latest instrui o daemon do Docker de qual imagem queremos construir. Neste caso, usamos a versão latest da imagem oficial do Docker node disponível no Docker Hub.

  • Dentro desta imagem, criamos um diretório de trabalho que contém todos os arquivos necessários e instruímos o daemon a definir esse diretório como o diretório de trabalho desejado para nossa aplicação. Para isso adicionamos

      RUN mkdir -p /usr/src/my_first_app
  WORKDIR /usr/src/my_first_app

  • Em seguida, instalamos as dependências do aplicativo movendo primeiro o arquivo package.json (que especifica as informações do aplicativo, incluindo as dependências) para o diretório de trabalho /usr/src/my_first_app na imagem. Fazemos isso por

      COPY package.json /usr/src/my_first_app/
  RUN npm install

  • Em seguida, digitamos COPY . /usr/src/my_first_app para adicionar todos os arquivos do aplicativo e código fonte ao diretório de trabalho na imagem.

  • Em seguida, usamos a diretiva EXPOSE para instruir o daemon a tornar a porta 8080 do contêiner resultante visível (através de um mapeamento de contêiner para host), já que o aplicativo se vincula à porta 8080.

  • Na última etapa, instruímos o daemon a executar o comando node server.js dentro da imagem executando o comando básico npm start. Usamos a diretiva CMD para isso, que recebe os comandos como argumentos.

      CMD [ "npm", "start" ]

  1. Em seguida, criamos um arquivo .dockerignore no mesmo diretório que o Dockerfile para evitar que nossa cópia de node_modules e logs usados ​​por nossa instalação do sistema Node.js sejam copiados para a imagem do Docker. O arquivo .dockerignore deve ter o seguinte conteúdo:

     node_modules
 npm-debug.log

  2. Construa sua imagem #

Navegue até o diretório que contém o Dockerfile e execute o comando a seguir para criar a imagem do Docker. O sinalizador -t permite marcar sua imagem para que seja mais fácil encontrá-la mais tarde usando o comando docker images:

    $ docker build -t <your username>/node-web-app .

Sua imagem agora será listada pelo Docker. Visualize as imagens usando o comando abaixo:

$ docker images

REPOSITORY                      TAG        ID              CREATED
node                            latest     539c0211cd76    10 minutes ago
<your username>/node-web-app    latest     d64d3505b0d2    1 minute ago

  1. Executando a imagem #

Agora podemos executar a imagem que acabamos de criar usando o conteúdo do aplicativo, a imagem base do node e o Dockerfile. Agora vamos executar nossa imagem recém-criada <seu nome de usuário>/node-web-app. Fornecer a opção -d para o comando docker run executa o contêiner no modo desanexado, para que o contêiner seja executado em segundo plano. O sinalizador -p redireciona uma porta pública para uma porta privada dentro do contêiner. Execute a imagem que você criou anteriormente usando este comando:

$ docker run -p 49160:8080 -d <your username>/node-web-app

  1. Imprima a saída de seu aplicativo executando docker ps em seu terminal. A saída deve ser algo como isto.

     CONTAINER ID        IMAGE                         COMMAND             CREATED             STATUS              PORTS                     NAMES
 7b701693b294      <your username>/node-web-app   "npm start"         20 minutes ago       Up 48 seconds       0.0.0.0:49160->8080/tcp   loving_goldstine

Obtenha a saída do aplicativo digitando docker logs <CONTAINER ID>. Neste caso é docker logs 7b701693b294.

Saída: Executando em http://localhost:8080

  1. Da saída docker ps, o mapeamento de porta obtido é 0.0.0.0:49160->8080/tcp. Portanto, o Docker mapeou a porta 8080 dentro do contêiner para a porta 49160 na máquina host. No navegador, agora podemos inserir localhost:49160.

Também podemos chamar nosso aplicativo usando curl:

$ curl -i localhost:49160

HTTP/1.1 200 OK
X-Powered-By: Express
Content-Type: text/html; charset=utf-8
Content-Length: 12
Date: Sun, 08 Jan 2017 14:00:12 GMT
Connection: keep-alive

Hello world
