Modos Docker --net (ponte, hots, contêiner mapeado e nenhum).

  • Modo Bridge, Modo Host e Modo Container Mapeado

Nesta página

  • Modo Bridge, Modo Host e Modo Container Mapeado

Começando

Modo Ponte É um padrão e anexado à ponte docker0. Coloque o container em um namespace de rede completamente separado.

Modo Host Quando o contêiner é apenas um processo em execução em um host, anexaremos o contêiner à NIC do host.

Modo Contêiner Mapeado Esse modo basicamente mapeia um novo contêiner em uma pilha de rede de contêineres existente. Também é chamado de ‘contêiner em modo contêiner’.

Nenhum Ele diz ao docker para colocar o contêiner em sua própria pilha de rede sem configuração

Modo Bridge, Modo Host e Modo Container Mapeado #

Modo Ponte

$ docker run –d –-name my_app -p 10000:80 image_name

Observe que não precisamos especificar –net=bridge porque esse é o modo de trabalho padrão para o docker. Isso permite executar vários contêineres no mesmo host sem nenhuma atribuição de porta dinâmica. Portanto, o modo BRIDGE evita o conflito de portas e é seguro, pois cada contêiner está executando seu próprio namespace de rede privada.

Modo Host

$ docker run –d –-name my_app –net=host image_name

Como ele usa o namespace da rede do host, não há necessidade de configuração especial, mas pode levar a problemas de segurança.

Modo Contêiner Mapeado

Esse modo basicamente mapeia um novo contêiner em uma pilha de rede de contêineres existente. Isso implica que recursos de rede, como endereço IP e mapeamentos de porta do primeiro contêiner, serão compartilhados pelo segundo contêiner. Isso também é chamado de modo ‘contêiner em contêiner’. Suponha que você tenha dois contêineres como web_container_1 e web_container_2 e executaremos web_container_2 no modo de contêiner mapeado. Vamos primeiro fazer o download do web_container_1 e executá-lo no modo desanexado com o seguinte comando,

$ docker run -d --name web1 -p 80:80 USERNAME/web_container_1

Depois de baixado, vamos dar uma olhada e ter certeza de que está funcionando. Aqui, apenas mapeamos uma porta em um contêiner que está sendo executado no modo de ponte padrão. Agora, vamos executar um segundo contêiner no modo de contêiner mapeado. Faremos isso com este comando.

$ docker run -d --name web2 --net=container:web1 USERNAME/web_container_2

Agora, se você simplesmente obtiver as informações da interface em ambos os recipientes, obterá a mesma configuração de rede. Isso realmente inclui o modo HOST que mapeia com informações exatas do host. O primeiro contêiner foi executado no modo de ponte padrão e o segundo contêiner está sendo executado no modo de contêiner mapeado. Podemos obter resultados muito semelhantes iniciando o primeiro contêiner no modo de host e o segundo contêiner no modo de contêiner mapeado.
