Sintaxe #

  • imagens do docker [OPÇÕES] [REPOSITÓRIO[:TAG]]
  • docker inspecionar [OPÇÕES] CONTAINER|IMAGE [CONTAINER|IMAGE…]
  • docker pull [OPÇÕES] NOME[:TAG|@DIGEST]
  • docker rmi [OPÇÕES] IMAGEM [IMAGEM…]
  • tag docker [OPTIONS] IMAGE[:TAG] [REGISTRYHOST/][USERNAME/]NAME[:TAG]

Listando imagens baixadas localmente #

$ docker images
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
hello-world         latest              693bce725149        6 days ago          967 B
postgres            9.5                 0f3af79d8673        10 weeks ago        265.7 MB
postgres            latest              0f3af79d8673        10 weeks ago        265.7 MB

Buscando uma imagem do Docker Hub #

Normalmente, as imagens são extraídas automaticamente do Docker Hub. O Docker tentará extrair qualquer imagem do Docker Hub que ainda não exista no host do Docker. Por exemplo, usar docker run ubuntu quando a imagem ubuntu ainda não estiver no host do Docker fará com que o Docker inicie um pull da imagem ubuntu mais recente. É possível extrair uma imagem separadamente usando docker pull para buscar ou atualizar manualmente uma imagem do Docker Hub.

docker pull ubuntu
docker pull ubuntu:14.04

Existem opções adicionais para extrair de um registro de imagem diferente ou extrair uma versão específica de uma imagem. A indicação de um registro alternativo é feita usando o nome completo da imagem e a versão opcional. Por exemplo, o comando a seguir tentará extrair a imagem ubuntu:14.04 do registro registry.example.com:

docker pull registry.example.com/username/ubuntu:14.04

Imagens de referência #

Os comandos do Docker que levam o nome de uma imagem aceitam quatro formas diferentes:

Tipo Exemplo
ID curto 693bce725149
Nome hello-world (padrão para tag :latest)
Nome+tag hello-world:latest
Resumo hello-world@sha256:e52be8ffeeb1f374f440893189cd32f44cb166650e7ab185fa7735b7dc48d619

Observação: você só pode se referir a uma imagem por seu resumo se essa imagem foi originalmente extraída usando esse resumo. Para ver o resumo de uma imagem (se estiver disponível), execute docker images --digests.

Removendo imagens #

O comando docker rmi é usado para remover imagens:

docker rmi <image name>

O nome completo da imagem deve ser usado para remover uma imagem. A menos que a imagem tenha sido marcada para remover o nome do registro, ela precisa ser especificada. Por exemplo:

docker rmi registry.example.com/username/myAppImage:1.3.5

Também é possível remover imagens pelo ID:

docker rmi 693bce725149

Por conveniência, é possível remover imagens pelo ID da imagem especificando apenas os primeiros caracteres do ID da imagem, desde que a substring especificada não seja ambígua:

docker rmi 693

Observação: As imagens podem ser removidas mesmo se houver contêineres existentes que usem essa imagem; docker rmi simplesmente “desmarca” a imagem.

Se nenhum contêiner estiver usando uma imagem, ela será coletada como lixo. Se um contêiner usar uma imagem, a imagem será coletada como lixo assim que todos os contêineres que a utilizam forem removidos. Por exemplo:

$ docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED                  STATUS                     PORTS               NAMES
5483657ee07b        hello-world         "/hello"            Less than a second ago   Exited (0) 2 seconds ago                       small_elion

$ docker rmi hello-world
Untagged: hello-world:latest

$ docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED                  STATUS                      PORTS               NAMES
5483657ee07b        693bce725149        "/hello"            Less than a second ago   Exited (0) 12 seconds ago                       small_elion

Remover todas as imagens sem contêineres iniciados

Para remover todas as imagens locais que não possuem contêineres iniciados, você pode fornecer uma lista das imagens como parâmetro:

docker rmi $(docker images -qa)

Remover todas as imagens

Se você deseja remover imagens, independentemente de terem ou não um contêiner iniciado, use o sinalizador de força (-f):

docker rmi -f $(docker images -qa)

Remover imagens pendentes

Se uma imagem não estiver marcada e não estiver sendo usada por nenhum contêiner, ela está ‘pendurada’ e pode ser removida assim:

docker images -q --no-trunc -f dangling=true | xargs -r docker rmi

Pesquise imagens no Docker Hub #

Você pode pesquisar imagens no Docker Hub usando o comando search:

docker search <term>

Por exemplo:

$ docker search nginx
NAME                      DESCRIPTION                                     STARS     OFFICIAL   AUTOMATED
nginx                     Official build of Nginx.                        3565      [OK]
jwilder/nginx-proxy       Automated Nginx reverse proxy for docker c...   717                  [OK]
richarvey/nginx-php-fpm   Container running Nginx + PHP-FPM capable ...   232                  [OK]
...

Inspecionando imagens #

docker inspect <image>

A saída está no formato JSON. Você pode usar o utilitário de linha de comando jq para analisar e imprimir apenas as chaves desejadas.

docker inspect <image> | jq -r '.[0].Author'

O comando acima mostrará o nome do autor das imagens.

Marcando imagens #

Marcar uma imagem é útil para acompanhar as diferentes versões da imagem:

docker tag ubuntu:latest registry.example.com/username/ubuntu:latest

Outro exemplo de marcação:

docker tag myApp:1.4.2 myApp:latest
docker tag myApp:1.4.2 registry.example.com/company/myApp:1.4.2

Salvando e carregando imagens do Docker #

docker save -o ubuntu.latest.tar ubuntu:latest

Este comando salvará a imagem ubuntu:latest como um arquivo tarball no diretório atual com o nome ubuntu.latest.tar. Este arquivo tarball pode então ser movido para outro host, por exemplo, usando rsync, ou arquivado no armazenamento.

Depois que o tarball for movido, o seguinte comando criará uma imagem do arquivo:

docker load -i /tmp/ubuntu.latest.tar

Agora é possível criar containers a partir da imagem ubuntu:latest como de costume.
