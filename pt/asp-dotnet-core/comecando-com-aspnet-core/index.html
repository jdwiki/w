Instalação e configuração #

Instalando o Visual Studio #

Se você não tiver o Visual Studio instalado, você pode baixar o Visual Studio Community Edition gratuito aqui. Se você já o tiver instalado, poderá prosseguir para a próxima etapa.

Criando um aplicativo ASP.NET Core MVC. #

  1. Abra o Visual Studio.
  2. Selecione Arquivo > Novo Projeto.
  3. Selecione Web no idioma de sua escolha na seção Modelos à esquerda.
  4. Escolha um tipo de projeto preferido na caixa de diálogo.
  5. Opcional: escolha um .NET Framework que você gostaria de segmentar
  6. Nomeie seu projeto e indique se deseja criar uma Solução para o projeto.
  7. Clique em OK para criar o projeto.

Você será apresentado a outra caixa de diálogo para selecionar o modelo que deseja usar para o projeto:

Cada uma das descrições é autoexplicativa. Para este primeiro projeto, selecione Aplicativo Web, que conterá todas as configurações padrão, autenticação e algum conteúdo existente.

Como este é um aplicativo de introdução e não requer nenhuma segurança ou autenticação, você pode alterar a opção de autenticação para Sem autenticação no lado direito da caixa de diálogo e clicar em OK para criar o projeto.

Você deve então ver o novo projeto dentro do Solution Explorer:

Pressione a tecla F5 para executar o aplicativo e iniciar uma sessão de depuração, que iniciará o aplicativo em seu navegador padrão:

Agora você pode ver que seu projeto está funcionando localmente e pronto como ponto de partida para você construir seu aplicativo.

API da Web ASP.NET Core mínima com ASP.NET Core MVC #

Com o ASP.NET Core 1.0, o MVC e a estrutura de API da Web foram mesclados em uma estrutura chamada ASP.NET Core MVC. Isso é bom, já que o MVC e a API da Web compartilham muitas funcionalidades, mas sempre houve diferenças sutis e duplicação de código.

No entanto, a fusão desses dois no quadro um também tornou mais difícil distinguir um do outro. Por exemplo, o Microsoft.AspNet.WebApi representa o framework Web API 5.x.x, não o novo. Mas, quando você inclui Microsoft.AspNetCore.Mvc (versão 1.0.0), você obtém o pacote completo. Isso conterá todos os recursos prontos para uso que a estrutura MVC oferece. Como Razor, auxiliares de marcação e vinculação de modelo.

Quando você quer apenas construir uma API Web, não precisamos de todos esses recursos. Então, como construímos uma API Web minimalista? A resposta é: Microsoft.AspNetCore.Mvc.Core. No novo mundo, o MVC é dividido em vários pacotes e este pacote contém apenas os componentes principais do framework MVC, como roteamento e autorização.

Para este exemplo, vamos criar uma API MVC mínima. Incluindo um formatador JSON e CORS. Crie um aplicativo Web ASP.NET Core 1.0 vazio e adicione estes pacotes ao seu project.json:

"Microsoft.AspNetCore.Mvc.Core": "1.0.0",
"Microsoft.AspNetCore.Mvc.Cors": "1.0.0",
"Microsoft.AspNetCore.Mvc.Formatters.Json": "1.0.0"

Agora podemos registrar o MVC usando AddMvcCore() na classe de inicialização:

public void ConfigureServices(IServiceCollection services)
{
    services.AddMvcCore()
            .AddCors()
            .AddJsonFormatters();
}

AddMvcCore retorna uma instância IMvcCoreBuilder que permite a construção adicional. A configuração do middleware é a mesma de sempre:

public void Configure(IApplicationBuilder app)
{
    app.UseCors(policy =>
    {
        policy.AllowAnyOrigin();
    });
    app.UseMvc();
}

Controladores #

A ‘antiga’ API da Web vem com sua própria classe base de controlador: ApiController. No novo mundo não existe tal coisa, apenas a classe padrão Controller. Infelizmente, essa é uma classe base bastante grande e está vinculada à vinculação de modelo, visualizações e JSON.NET.

Felizmente, no novo framework, as classes de controlador não precisam derivar de Controller para serem escolhidas pelo mecanismo de roteamento. Apenas anexar o nome com Controller é suficiente. Isso nos permite construir nossa própria classe base do controlador. Vamos chamá-lo de ApiController, apenas pelos velhos tempos:

/// <summary>
/// Base class for an API controller.
/// </summary>
[Controller]
public abstract class ApiController
{
    [ActionContext]
    public ActionContext ActionContext { get; set; }

    public HttpContext HttpContext => ActionContext?.HttpContext;

    public HttpRequest Request => ActionContext?.HttpContext?.Request;

    public HttpResponse Response => ActionContext?.HttpContext?.Response;

    public IServiceProvider Resolver => ActionContext?.HttpContext?.RequestServices;
}

O atributo [Controller] indica que o tipo ou qualquer tipo derivado é considerado um controlador pelo mecanismo de descoberta de controlador padrão. O atributo [ActionContext] especifica que a propriedade deve ser definida com o ActionContext atual quando o MVC cria o controlador. O ActionContext fornece informações sobre a solicitação atual.

ASP.NET Core MVC também oferece uma classe ControllerBase que fornece uma classe base do controlador apenas sem suporte para visualizações. Ainda é muito maior que o nosso. Use-o se achar conveniente.

Conclusão #

Agora podemos criar uma API Web mínima usando a nova estrutura ASP.NET Core MVC. A estrutura modular do pacote nos permite apenas puxar os pacotes que precisamos e criar um aplicativo simples e enxuto.

Crie um novo projeto a partir da linha de comando #

É possível criar um novo projeto ASP.NET Core inteiramente a partir da linha de comando usando o comando dotnet.

dotnet new web
dotnet restore
dotnet run

dotnet new web cria um novo projeto web “vazio”. O parâmetro web informa à ferramenta dotnet para usar o modelo ASP.NET Core Empty. Use dotnet new -all para mostrar todos os modelos disponíveis atualmente instalados. Outros modelos-chave incluem console, classlib, mvc e xunit.

Uma vez que o modelo tenha sido desenvolvido, você pode restaurar os pacotes necessários para executar o projeto (dotnet restore), e compilá-lo e iniciá-lo (dotnet run).

Assim que o projeto estiver em execução, ele estará disponível na porta padrão: http://localhost:5000

Configure a variável de ambiente no ASP.NET Core [Windows] #

=> Postagem original <=

O ASP.NET Core usa a variável de ambiente ASPNETCORE_ENVIRONMENT para determinar o ambiente atual. Por padrão, se você executar seu aplicativo sem definir esse valor, ele será automaticamente padronizado para o ambiente Production.

> dotnet run
Project TestApp (.NETCoreApp,Version=v1.0) was previously compiled. Skipping compilation.

Hosting environment: Production  
Content root path: C:\Projects\TestApp  
Now listening on: http://localhost:5000  
Application started. Press Ctrl+C to shut down.

Definindo a variável de ambiente no Windows

Na linha de comando

Você pode definir facilmente uma variável de ambiente em um prompt de comando usando o comando setx.exe incluído no Windows. Você pode usá-lo para definir facilmente uma variável de usuário:

>setx ASPNETCORE_ENVIRONMENT "Development"

SUCCESS: Specified value was saved.

Observe que a variável de ambiente não está definida na janela aberta atual. Você precisará abrir um novo prompt de comando para ver o ambiente atualizado. Também é possível definir variáveis ​​de sistema (em vez de apenas variáveis ​​de usuário) se você abrir um prompt de comando administrativo e adicionar a opção /M:

>setx ASPNETCORE_ENVIRONMENT "Development" /M

SUCCESS: Specified value was saved.

Usando o PowerShell Como alternativa, você pode usar o PowerShell para definir a variável. No PowerShell, assim como as variáveis ​​normais de usuário e sistema, você também pode criar uma variável temporária usando o comando $Env::

$Env:ASPNETCORE_ENVIRONMENT = "Development"

A variável criada dura apenas a duração de sua sessão do PowerShell - assim que você fecha a janela, o ambiente volta ao valor padrão.

Como alternativa, você pode definir as variáveis ​​de ambiente do usuário ou do sistema diretamente. Esse método não altera as variáveis ​​de ambiente na sessão atual, portanto, você precisará abrir uma nova janela do PowerShell para ver suas alterações. Como antes, alterar as variáveis ​​do sistema (Máquina) exigirá acesso administrativo

[Environment]::SetEnvironmentVariable("ASPNETCORE_ENVIRONMENT", "Development", "User")
[Environment]::SetEnvironmentVariable("ASPNETCORE_ENVIRONMENT", "Development", "Machine")

Usando o painel de controle do Windows Se você não é fã do prompt de comando, pode atualizar facilmente suas variáveis ​​usando o mouse! Clique no botão do menu Iniciar do Windows (ou pressione a tecla Windows), procure por variáveis ​​de ambiente e escolha Editar variáveis ​​de ambiente para sua conta:

Selecionar esta opção irá abrir a janela Propriedades do Sistema

Clique em Variáveis ​​de Ambiente para visualizar a lista de variáveis ​​de ambiente atuais em seu sistema.

Supondo que você ainda não tenha uma variável chamada ASPNETCORE_ENVIRONMENT, clique no botão Novo… e adicione uma nova variável de ambiente de conta: Clique em OK para salvar todas as suas alterações. Você precisará reabrir qualquer janela de comando para garantir que as novas variáveis ​​de ambiente sejam carregadas.
