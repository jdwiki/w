Parâmetros #

Nome Informações
asp-ação O nome do método de ação para o qual o formulário deve ser postado
controlador asp O nome do controlador onde o método de ação especificado em asp-action existe
asp-route-* Valores de rota personalizados que você deseja adicionar como querystring ao valor do atributo de ação do formulário. Substitua 8 pelo nome da querystring que você deseja

Form Tag Helper - Com atributos de rota personalizados #

<form asp-action="create" 
      asp-controller="Home" 
      asp-route-returnurl="dashboard" 
      asp-route-from="google">
     <!--Your form elements goes here--> 
</form>

Isso irá gerar a marcação abaixo

<form action="/Home/create?returnurl=dashboard&amp;from=google" method="post">
    <!--Your form elements goes here--> 
</form>

Form Tag Helper - Exemplo básico #

<form asp-action="create" asp-controller="Home">
     <!--Your form elements goes here--> 
</form>

Auxiliar de tag de entrada #

Supondo que sua visão seja fortemente tipada para um modelo de visão como

public class CreateProduct
{
   public string Name { set; get; }
}

E você está passando um objeto disso para a visão do seu método de ação.

@model CreateProduct
<form asp-action="create" asp-controller="Home" >

    <input type="text" asp-for="Name"/>   
    <input type="submit"/>

</form>

Isso gerará a marcação abaixo.

<form action="/Home/create" method="post"> 

    <input type="text" id="Name" name="Name" value="" />
    <input type="submit"/>
    <input name="__RequestVerificationToken" type="hidden" value="ThisWillBeAUniqueToken" />

</form>

Se desejar que o campo de entrada seja renderizado com um valor padrão, você pode definir o valor da propriedade Name de seu modelo de exibição no método de ação.

public IActionResult Create()
{
  var vm = new CreateProduct { Name="IPhone"};
  return View(vm);
}

Envio de formulário e vinculação de modelo

A vinculação de modelo funcionará bem se você usar CreateProduct como seu parâmetro de método de ação HttpPost/um parâmetro chamado name

Selecione o auxiliar de tags #

Supondo que sua visão seja fortemente tipada para um modelo de visão como este

public class CreateProduct
{       
   public IEnumerable<SelectListItem> Categories { set; get; }
   public int SelectedCategory { set; get; }
}

E no seu método de ação GET, você está criando um objeto dessa view model, configurando a propriedade Categories e enviando para a view

public IActionResult Create()
{
    var vm = new CreateProduct();
    vm.Categories = new List<SelectListItem>
    {
        new SelectListItem {Text = "Books", Value = "1"},
        new SelectListItem {Text = "Furniture", Value = "2"}
    };
    return View(vm);
}

e na sua opinião

@model CreateProduct

<form asp-action="create" asp-controller="Home">
    <select asp-for="SelectedCategory" asp-items="@Model.Categories">
        <option>Select one</option>
    </select>
    <input type="submit"/>
</form>

Isso renderizará a marcação abaixo (incluídas apenas as partes relevantes do formulário/campos)

<form action="/Home/create" method="post">  
    <select data-val="true" id="SelectedCategory" name="SelectedCategory">
        <option>Select one</option>
        <option value="1">Shyju</option>
        <option value="2">Sean</option>
    </select>
    <input type="submit"/>
</form>

Obtendo o valor da lista suspensa selecionado no envio do formulário

Você pode usar o mesmo modelo de exibição que seu parâmetro de método de ação HttpPost

[HttpPost]
public ActionResult Create(CreateProduct model)
{
  //check model.SelectedCategory value
  / /to do : return something
}

Defina uma opção como a selecionada

Se você deseja definir uma opção como a opção selecionada, basta definir o valor da propriedade SelectedCategory.

public IActionResult Create()
{
    var vm = new CreateProduct();
    vm.Categories = new List<SelectListItem>
    {
        new SelectListItem {Text = "Books", Value = "1"},
        new SelectListItem {Text = "Furniture", Value = "2"},
        new SelectListItem {Text = "Music", Value = "3"}
    };
    vm.SelectedCategory = 2;
    return View(vm);
}

Renderização de um menu suspenso/ListBox de seleção múltipla

Se você deseja renderizar um menu suspenso de seleção múltipla, basta alterar sua propriedade de modelo de exibição que você usa para o atributo asp-for em sua exibição para um tipo de matriz.

public class CreateProduct
{       
   public IEnumerable<SelectListItem> Categories { set; get; }
   public int[] SelectedCategories { set; get; }
}

Na visão

@model CreateProduct

<form asp-action="create" asp-controller="Home" >
    <select asp-for="SelectedCategories" asp-items="@Model.Categories">
        <option>Select one</option>
    </select>
    <input type="submit"/>
</form>

Isso irá gerar o elemento SELECT com o atributo multiple

<form action="/Home/create" method="post">
     <select id="SelectedCategories" multiple="multiple" name="SelectedCategories">
        <option>Select one</option>
        <option value="1">Shyju</option>
       <option value="2">Sean</option>
     </select>
    <input type="submit"/>
</form>

Ajudante de tags personalizadas #

Você pode criar seus próprios auxiliares de tag implementando ITagHelper ou derivando da classe de conveniência TagHelper.

  • A convenção padrão é direcionar uma tag html que corresponda ao nome do auxiliar sem o sufixo TagHelper opcional. Por exemplo, WidgetTagHelper terá como alvo uma tag <widget>.
  • O atributo [HtmlTargetElement] pode ser usado para controlar ainda mais a tag que está sendo segmentada
  • Qualquer propriedade pública da classe pode receber um valor como um atributo na marcação razor. Por exemplo, uma propriedade pública public string Title {get; set;} pode receber um valor como <widget title="my title">
  • Por padrão, os auxiliares de tag traduzem nomes e propriedades de classe C# com maiúsculas em Pascal para auxiliares de tag em minúsculas kebab. Por exemplo, se você omitir o uso de [HtmlTargetElement] e o nome da classe for WidgetBoxTagHelper, no Razor você escreverá <widget-box></widget-box>.
  • Process e ProcessAsync contêm a lógica de renderização. Ambos recebem um parâmetro context com informações sobre a tag atual sendo renderizada e um parâmetro output usado para personalizar o resultado renderizado.

Qualquer assembly que contenha auxiliares de tag personalizados precisa ser adicionado ao arquivo _ViewImports.cshtml (Observe que é o assembly que está sendo registrado, não o namespace):

@addTagHelper *, MyAssembly

Amostra de ajudante de tags personalizadas de widgets #

O exemplo a seguir cria um auxiliar de tag de widget personalizado que terá como alvo a marcação do Razor como:

<widget-box title="My Title">This is my content: @ViewData["Message"]</widget-box>

Que será renderizado como:

<div class="widget-box">
    <div class="widget-header">My Title</div>
    <div class="widget-body">This is my content: some message</div>
</div>

O código necessário para criar um auxiliar de tag é o seguinte:

[HtmlTargetElement("widget-box")]
public class WidgetTagHelper : TagHelper
{
    public string Title { get; set; }

    public override async Task ProcessAsync(TagHelperContext context, TagHelperOutput output)
    {
        var outerTag = new TagBuilder("div");
        outerTag.Attributes.Add("class", output.TagName);
        output.MergeAttributes(outerTag);
        output.TagName = outerTag.TagName;

        //Create the header
        var header = new TagBuilder("div");
        header.Attributes.Add("class", "widget-header");
        header.InnerHtml.Append(this.Title);
        output.PreContent.SetHtmlContent(header);

        //Create the body and replace original tag helper content
        var body = new TagBuilder("div");
        body.Attributes.Add("class", "widget-body");
        var originalContents = await output.GetChildContentAsync();
        body.InnerHtml.Append(originalContents.GetContent());
        output.Content.SetHtmlContent(body);
    }
}

Auxiliar de etiqueta de etiqueta #

Label Tag Helper pode ser usado para renderizar ’label’ para uma propriedade de modelo. Ele substitui o método Html.LabelFor nas versões anteriores do MVC.

Digamos que você tenha um modelo:

public class FormViewModel
{
    public string Name { get; set; }
}

Na visualização você pode usar o elemento HTML label e o auxiliar de tag asp-for:

<form>
    <label asp-for="Name"></label>
    <input asp-for="Name" type="text" />
</form>

Isso é equivalente ao seguinte código em versões anteriores do MVC:

<form>
    @Html.LabelFor(x => x.Name)
    @Html.TextBoxFor(x => x.Name)
</form>

Ambos os snippets de código acima renderizam o mesmo HTML:

<form>
    <label for="Name">Name</label>
    <input name="Name" id="Name" type="text" value="">
</form>

Auxiliar de marcação de âncora #

O auxiliar de marca de âncora é usado para gerar atributos href para vincular a uma determinada ação do controlador ou rota MVC. Exemplo básico

<a asp-controller="Products" asp-action="Index">Login</a>

Às vezes, precisamos especificar parâmetros adicionais para a ação do controlador à qual você está vinculando. Podemos especificar valores para esses parâmetros adicionando atributos com o prefixo asp-route-.

<a asp-controller="Products" asp-action="Details" asp-route-id="@Model.ProductId">
   View Details
</a>
