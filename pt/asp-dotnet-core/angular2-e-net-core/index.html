Erros esperados ao gerar componentes Angular 2 no projeto .NET Core (versão 0.8.3) #

Ao gerar novos componentes Angular 2 em um projeto .NET Core, você pode encontrar os seguintes erros (a partir da versão 0.8.3):

Error locating module for declaration
        SilentError: No module files found

OU

No app module found. Please add your new Class to your component. 
        Identical ClientApp/app/app.module.ts

[SOLUÇÃO]

  1. Renomeie app.module.client.ts para app.client.module.ts

  2. Abra app.client.module.ts: coloque 3 pontos “…” antes da declaração e coloque a declaração entre colchetes.

    For example: [...sharedConfig.declarations, <MyComponent>]

  3. Abra boot-client.ts: atualize sua importação para usar o novo app.client.module reference.

    For example: import { AppModule } from './app/app.client.module';

  4. Agora tente gerar o novo componente: ng g component my-component

[EXPLICAÇÃO]

A Angular CLI procura um arquivo chamado app.module.ts em seu projeto e tenta encontrar uma referência para a propriedade de declarações para importar o componente. Deve ser uma matriz (como o sharedConfig.declarations é), mas as alterações não são aplicadas
