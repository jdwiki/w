O núcleo Asp.net fornece abstrações de configuração. Eles permitem que você carregue as definições de configuração de várias fontes e crie um modelo de configuração final que pode ser consumido pelo seu aplicativo.

Sintaxe #

  • IConfiguração
  • string this[string key] { get; definir; }
  • IEnumerable<IConfigurationSection> GetChildren();
  • IConfigurationSection GetSection(string key);

Começando #

Neste exemplo, descreveremos o que acontece quando você monta um novo projeto.

Em primeiro lugar, as seguintes dependências serão adicionadas ao seu projeto (atualmente arquivo project.json):

"Microsoft.Extensions.Configuration.EnvironmentVariables": "1.0.0",
"Microsoft.Extensions.Configuration.Json": "1.0.0",

Ele também criará um construtor em seu arquivo Startup.cs que será responsável por construir a configuração usando a API fluente ConfigurationBuilder:

  1. Primeiro cria um novo ConfigurationBuilder.
  2. Em seguida, ele define um caminho base que será usado para calcular o caminho absoluto de outros arquivos
  3. Ele adiciona um appsettings.json opcional ao construtor de configuração e monitora suas alterações
  4. Ele adiciona um arquivo de configuração appsettings.environementName.json relacionado ao ambiente opcional
  5. Em seguida, adiciona variáveis ​​de ambiente.
public Startup(IHostingEnvironment env)
{
    var builder = new ConfigurationBuilder()
        .SetBasePath(env.ContentRootPath)
        .AddJsonFile("appsettings.json", optional: true, reloadOnChange: true)
        .AddJsonFile($"appsettings.{env.EnvironmentName}.json", optional: true)
        .AddEnvironmentVariables();

    Configuration = builder.Build();
}

Se uma mesma configuração for definida em várias fontes, a última fonte adicionada vencerá e seu valor será selecionado.

A configuração pode então ser consumida usando a propriedade do indexador. O caractere de dois pontos : serve como delimitador de caminho.

Configuration["AzureLogger:ConnectionString"]

Isso procurará um valor de configuração ConnectionString em uma seção AzureLogger.

Trabalhar com variáveis ​​de ambiente #

Você pode obter a configuração de variáveis ​​de ambiente chamando .AddEnvironmentVariables() em seu ConfigurationBuilder.

Ele carregará variáveis ​​de ambiente prefixadas com APPSETTING_ Ele então usará dois pontos : como o separador de caminho de chave.

Isso significa que: seguintes configurações de ambiente:

APPSETTING_Security:Authentication:UserName = a_user_name
APPSETTING_Security:Authentication:Password = a_user_password

Será o equivalente a este json:

{
    "Security" : {
       "Authentication" : {
           "UserName" : "a_user_name",
           "Password" : "a_user_password" 
        } 
    }
}

** Observe que o Serviço do Azure transmitirá as configurações como variáveis ​​de ambiente. O prefixo será definido para você de forma transparente. Portanto, para fazer o mesmo no Azure, basta definir duas configurações de aplicativo na folha AppSettings:

Security:Authentication:UserName         a_user_name
Security:Authentication:Password         a_user_password

Modelo e configuração da opção #

Ao lidar com grandes conjuntos de configuração de valor, pode se tornar bastante inconveniente carregá-los de uma só vez.

O modelo de opção que vem com asp.net oferece uma maneira conveniente de mapear uma section para um dotnet poco: Por exemplo, pode-se hidratar StorageOptions diretamente de uma seção de configuração b adicionando o pacote Microsoft.Extensions.Options.ConfigurationExtensions e chamando o método de extensão Configure<TOptions>(IConfiguration config).

services.Configure<StorageOptions>(Configuration.GetSection("Storage"));

Fonte de configuração na memória #

Você também pode obter a configuração de um objeto na memória, como um Dictionary<string,string>

.AddInMemoryCollection(new Dictionary<string, string>
{
    ["akey"] = "a value"
})

Isso pode ser útil em cenários de integração/teste de unidade.

Acessando a configuração usando injeção de dependência #

A abordagem recomendada seria evitar fazer isso e usar IOptions<TOptions> e IServiceCollection.Configure<TOptions>.

Dito isso, ainda é bastante simples tornar o aplicativo IConfigurationRoot disponível em toda a extensão.

No construtor Startup.cs você deve ter o seguinte código para construir a configuração,

 Configuration = builder.Build();

Aqui Configuration é uma instância de IConfigurationRoot, E adicione esta instância como um Singleton à coleção de serviços no método ConfigureServices , Startup.cs ,

 public void ConfigureServices(IServiceCollection services)
 {
    services.AddSingleton<IConfigurationRoot>(provider => Configuration);

Por exemplo, agora você pode acessar a configuração em um Controller/Service

  public MyController(IConfigurationRoot config){
      var setting1= config.GetValue<string>("Setting1")
  }
