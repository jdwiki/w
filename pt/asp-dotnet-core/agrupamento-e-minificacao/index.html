Extensão de Bundler e Minifier #

O Visual Studio também apresenta um Bundler and Minifier Extension disponível que é capaz de lidar com esse processo para você. A extensão permite selecionar e agrupar facilmente os arquivos necessários sem escrever uma linha de código.

Construindo seus pacotes #

Depois de instalar a extensão, você seleciona todos os arquivos específicos que deseja incluir em um bundle e usa a opção Bundle and Miniify Files da extensão :

Isso solicitará que você nomeie seu pacote e escolha um local para salvá-lo. Você notará um novo arquivo em seu projeto chamado bundleconfig.json que se parece com o seguinte:

[
  {
    "outputFileName": "wwwroot/app/bundle.js",
    "inputFiles": [
      "wwwroot/lib/jquery/dist/jquery.js",
      "wwwroot/lib/bootstrap/dist/js/bootstrap.js",
      "wwwroot/lib/jquery-validation/dist/jquery.validate.js",
      "wwwroot/lib/jquery-validation-unobtrusive/jquery.validate.unobtrusive.js" 
    ]
  }
]

OBSERVAÇÃO: a ordem em que os arquivos são selecionados determinará a ordene que eles apareçam dentro do pacote, então se você tiver algum dependências, certifique-se de levar isso em consideração.

Reduzindo seus pacotes #

Agora, a etapa anterior simplesmente agrupará seus arquivos, se você quiser reduzir o pacote, precisará indicar isso no arquivo bundleconfig.json. Basta adicionar um bloco minify como o seguinte ao seu pacote existente para indicar que você deseja minificá-lo:

[
  {
    "outputFileName": "wwwroot/app/bundle.js",
    "inputFiles": [
      "wwwroot/lib/jquery/dist/jquery.js",
      "wwwroot/lib/bootstrap/dist/js/bootstrap.js",
      "wwwroot/lib/jquery-validation/dist/jquery.validate.js",
      "wwwroot/lib/jquery-validation-unobtrusive/jquery.validate.unobtrusive.js" 
    ],
    "minify": {
      "enabled": true
    }
  }
]

Automatize seus pacotes #

Por fim, se quiser automatizar esse processo, você pode agendar uma tarefa para ser executada sempre que seu aplicativo for criado para garantir que seus pacotes reflitam quaisquer alterações em seu aplicativo.

Para fazer isso, você precisará fazer o seguinte:

  • Abra o Task Runner Explorer (via Ferramentas > Task Runner Explorer).
  • Clique com o botão direito do mouse na opção Atualizar todos os arquivos abaixo de bundleconfig.json.
  • Selecione sua ligação preferida no menu de contexto Ligações.

Depois de fazer isso, seus pacotes devem ser atualizados automaticamente na etapa preferida que você selecionou.

O comando dotnet bundle #

A versão ASP.NET Core RTM introduziu o BundlerMinifier.Core, uma nova ferramenta de Agrupamento e Minificação que pode ser facilmente integrada a aplicativos ASP.NET Core existentes e não requer extensões externas ou arquivos de script.

Usando BundlerMinifier.Core #

Para usar esta ferramenta, basta adicionar uma referência a BundlerMinifier.Core dentro da seção tools de seu arquivo project.json existente como visto abaixo:

"tools": {
  "BundlerMinifier.Core": "2.0.238",
  "Microsoft.AspNetCore.Razor.Tools": "1.0.0-preview2-final",
  "Microsoft.AspNetCore.Server.IISIntegration.Tools": "1.0.0-preview2-final"
}

Configurando seus pacotes #

Após adicionar a ferramenta, você precisará adicionar um arquivo bundleconfig.json em seu projeto que será usado para configurar os arquivos que você deseja incluir em seus bundles. Uma configuração mínima pode ser vista abaixo:

[
  {
    "outputFileName": "wwwroot/css/site.min.css",
    "inputFiles": [
      "wwwroot/css/site.css"
    ]
  },
  {
    "outputFileName": "wwwroot/js/site.min.js",
    "inputFiles": [
      "wwwroot/js/site.js"
    ],
    "minify": {
      "enabled": true,
      "renameLocals": true
    },
    "sourceMap": false
  },
  {
    "outputFileName": "wwwroot/js/semantic.validation.min.js",
    "inputFiles": [
      "wwwroot/js/semantic.validation.js"
    ],
    "minify": {
      "enabled": true,
      "renameLocals": true
    }
  }
]

Criando/atualizando pacotes #

Depois que seus pacotes forem configurados, você poderá agrupar e reduzir seus arquivos existentes por meio do seguinte comando:

dotnet bundle

Agrupamento automatizado #

O processo de Agrupamento e Minificação pode ser automatizado como parte do processo de compilação, adicionando o comando dotnet bundle na seção de pré-compilação de seu arquivo project.json existente:

"scripts": {
  "precompile": [
    "dotnet bundle"
  ]
}

Comandos Disponíveis #

Você pode ver uma lista de todos os comandos disponíveis e suas descrições abaixo:

  • dotnet bundle - Executa o comando bundle usando o arquivo bundleconfig.json para agrupar e reduzir seus arquivos especificados.
  • dotnet bundle clean - Limpa todos os arquivos de saída existentes do disco.
  • dotnet bundle watch - Cria um watchers que executará automaticamente o dotnet bundle sempre que um arquivo de entrada existente da configuração bundleconfig.json agrupar seus arquivos.
  • ajuda do pacote dotnet - Exibe todas as opções de ajuda disponíveis e instruções para usar a interface de linha de comando.

Grunt and Gulp #

Em aplicativos ASP.NET Core, você agrupa e reduz os recursos do lado do cliente durante o tempo de design usando ferramentas de terceiros, como Gulp e [Grunt](http://gruntjs .com/). Ao usar o agrupamento e a minificação em tempo de design, os arquivos reduzidos são criados antes da implantação do aplicativo. Agrupar e reduzir antes da implantação oferece a vantagem de reduzir a carga do servidor. No entanto, é importante reconhecer que o agrupamento e a minificação em tempo de design aumentam a complexidade da compilação e só funcionam com arquivos estáticos.

Isso é feito no ASP.NET Core configurando o Gulp por meio de um arquivo gulpfile.js dentro do seu projeto:

// Defining dependencies
var gulp = require("gulp"),  
    rimraf = require("rimraf"),
    concat = require("gulp-concat"),
    cssmin = require("gulp-cssmin"),
    uglify = require("gulp-uglify");

// Define web root
var webroot = "./wwwroot/"

// Defining paths
var paths = {  
    js: webroot + "js/**/*.js",
    minJs: webroot + "js/**/*.min.js",
    css: webroot + "css/**/*.css",
    minCss: webroot + "css/**/*.min.css",
    concatJsDest: webroot + "js/site.min.js",
    concatCssDest: webroot + "css/site.min.css"
};

// Bundling (via concat()) and minifying (via uglify()) Javascript
gulp.task("min:js", function () {  
    return gulp.src([paths.js, "!" + paths.minJs], { base: "." })
        .pipe(concat(paths.concatJsDest))
        .pipe(uglify())
        .pipe(gulp.dest("."));
});

// Bundling (via concat()) and minifying (via cssmin()) Javascript
gulp.task("min:css", function () {  
    return gulp.src([paths.css, "!" + paths.minCss])
        .pipe(concat(paths.concatCssDest))
        .pipe(cssmin())
        .pipe(gulp.dest("."));
});

Essa abordagem agrupará e reduzirá adequadamente seus arquivos Javascript e CSS existentes, respectivamente, de acordo com os diretórios e padrões de globbing usados.
