Instalação ou Configuração #

O compilador padrão DMD da linguagem de programação D pode ser executado em todas as principais plataformas. Para instalar o DMD, veja aqui. Para instalar por linha de comando, você pode executar o comando (encontrado no site D):

curl -fsS https://dlang.org/install.sh | bash -s dmd

Gerenciadores de Pacotes #

Arch Linux #

pacman -S dlang

Chocolate #

choco install dmd

Gento #

layman -f -a dlang

OSX Homebrew #

brew install dmd

Debian/Ubuntu #

A instalação em distribuições Debian/Ubuntu precisa que o repositório APT seja adicionado à lista de fontes.

wget http://master.dl.sourceforge.net/project/d-apt/files/d-apt.list -O /etc/apt/sources.list.d/d-apt.list
wget -qO - https://dlang.org/d-keyring.gpg | sudo apt-key add -
apt-get update
apt-get install dmd-bin

Outros compiladores #

LDC é um compilador de D que usa o frontend do compilador DMD oficial e o LLVM como backend.

GDC é um compilador de D que usa o backend GCC para gerar código.

IDE #

Para facilitar a vida, você também pode instalar um IDE (Ambiente de Desenvolvimento Integrado). O D-Language Wiki tem uma lista de IDEs e plugins disponíveis para todas as plataformas aqui.

Olá Mundo #

import std.stdio;

// Let's get going!
void main()
{
    writeln("Hello World!");
}

Para compilar e executar, salve este texto como um arquivo chamado main.d. Na linha de comando execute dmd main.d para compilar o programa. Finalmente, execute ./main para executar o programa em um shell bash ou você pode clicar no executável nas janelas.

Ler valores de uma string #

import std.format;

void main() {
    string s = "Name Surname 18";
    string name, surname;
    int age;
    formattedRead(s, "%s %s %s", &name, &surname, &age);
    // %s selects a format based on the corresponding argument's type
}

A documentação oficial para as strings de formato pode ser encontrada em: https://dlang.org/phobos/std_format.html#std.format

Olá Mundo! #

Para criar o programa de impressão clássico “Hello, world”, crie um arquivo chamado hello.d com um editor de texto contendo o seguinte código:

import std.stdio;

void main() {
    writeln("Hello, World!");    //writeln() automatically adds a newline (\n) to the output
}

Explicação: #

import std.stdio

Esta linha diz ao compilador que as funções definidas no módulo da Biblioteca Padrão std.stdio serão usadas. Qualquer módulo pode ser importado, desde que o compilador saiba onde procurá-los. Muitas funções são fornecidas como parte da enorme Biblioteca Padrão do D.

void main() {

Esta linha declara a função main, retornando void. Note que diferentemente de C e C++, D permite que main seja do tipo void. A função main é especial por ser o ponto de entrada do programa, ou seja, é aqui que começa a execução do programa. Algumas notas sobre funções em geral:

  • O nome de uma função pode ser qualquer coisa que comece com uma letra e seja composta por letras, dígitos e sublinhados.

  • Os parâmetros esperados serão uma lista separada por vírgulas de nomes de variáveis ​​e seus tipos de dados.

  • O valor que a função deve retornar pode ser qualquer tipo de dados existente e deve corresponder ao tipo de expressão usado na instrução de retorno dentro da função.

As chaves { … } são usadas em pares para indicar onde um bloco de código começa e termina. Eles podem ser usados ​​de várias maneiras, mas neste caso eles indicam onde a função começa e termina.

writeln("Hello, World!");

writeln é uma função declarada em std.stdio que escreve seus agruments em stdout. Neste caso, seu argumento é "Hello, World", que será escrito no console. Vários caracteres de formato, semelhantes aos usados ​​pelo printf do C, podem ser usados, como \n, \r, etc.

Toda instrução precisa ser encerrada por um ponto e vírgula.

Comentários são usados ​​para indicar algo para a pessoa que está lendo o código e são tratados como um espaço em branco pelo compilador. No código acima, este é um comentário:

//writeln() automatically adds a newline (\n) to the output

Esses são pedaços de código que são ignorados pelo compilador. Existem três maneiras diferentes de comentar em D :

  1. // - Comente todo o texto na mesma linha, após o //
  2. /* texto do comentário */ - Estes são úteis para comentários de várias linhas
  3. /+ texto do comentário + - Estes também são comentários de várias linhas

Eles são muito úteis para transmitir o que uma função/parte de código está fazendo para um colega desenvolvedor.

Compilando e executando o programa #

Para executar este programa, o código deve ser compilado em um executável. Isso pode ser feito com a ajuda do compilador.

Para compilar usando DMD, o compilador D de referência, abra um terminal, navegue até o local do arquivo hello.d que você criou e execute:

dmd olá.d

Se nenhum erro for encontrado, o compilador produzirá um executável com o nome do arquivo de origem. Isso agora pode ser executado digitando

./olá

Após a execução, o programa imprimirá Hello, World!, seguido por uma nova linha.
