Um recurso de pepino #

O Cucumber usa sintaxe Gherkin para descrever os comportamentos do seu software em linguagem natural estruturada.

Como tal, o Cucumber não é um framework de teste (um mal-entendido comum), mas um framework de documentação do sistema, não muito diferente de outros como o Use Case Scenario. O mal-entendido comum se deve ao fato de que a documentação do Cucumber pode ser automatizada para garantir que reflita o comportamento real do sistema.

Um conjunto de documentação do Cucumber é composto de Features, cada um descrevendo um recurso do seu software, escrito em Gherkin e hospedado em seu próprio arquivo. Ao organizar esses arquivos em uma estrutura de diretórios, você pode agrupar e organizar recursos:

  • banco/
  • retirada.recurso
  • recurso atm.
  • empréstimo pessoal.recurso
  • negociação/
  • portfólio.recurso
  • recurso intradiário
  • hipoteca/
  • avaliação.recurso
  • contabilidade.recurso

Cada Feature é um arquivo de texto simples composto por uma seção introdutória opcional, não estruturada e puramente informativa e um ou mais Scenarios, cada um representando uma condição de uso ou caso de uso.

Exemplo:

Feature: Documentation
As a StackOverflow user or visitor
I want to access the documentation section
    
    Scenario: search documentation on Stack Overflow
        Given I am on StackOverflow
        And I go to the Documentation section
        When I search for "cucumber"
        And I follow the link to "cucumber"
        Then I should see documentation for "cucumber"

Cada linha que começa com Given, When, And, But ou Then é chamada de Step. Qualquer passo pode começar com qualquer uma dessas palavras, independentemente da ordem, mas é convencional usá-las da maneira mais natural possível.

Os recursos também podem ser organizados por meio de Tags, anotações que o editor pode colocar em um Feature ou em um Scenario para categorizá-lo ainda mais.

A Executabilidade de um Recurso é alcançada via código glue que pode ser escrito em muitas linguagens diferentes (Java, Ruby, Scala, C/C++): cada Step é comparado com o código Glue para identificar Step Definitions (comumente abreviado para StepDef) por meio de expressões regulares.

Cada Step pode ter apenas uma Step Definition associada.

Quando um Feature é executado, cada Scenario de composição é executado, significando que cada StepDef que corresponde aos Steps em cada Scenario é executado.

Uma definição de etapa do Cucumber em Ruby #

Em features/step_definitions/documentation.rb:

When /^I go to the "([^"]+)" documentation$/ do |section|
  path_part =
    case section
      when "Documentation"
        "documentation"
      else
        raise "Unknown documentation section: #{section}"
    end
  visit "/documentation/#{path_part}/topics"
end

Then /^I should see the "([^"]+) documentation"$/ do |section|
  expect(page).to have_css('h2.doctag_title a', text: section)
end

Essas etapas exercitam um aplicativo da web. Eles são tão simples quanto podem ser enquanto ainda são práticos.

Cada etapa começa com uma palavra-chave Gherkin, que em um arquivo de definição de etapa é um método que registra uma etapa com o Cucumber. O método de definição de etapas usa uma expressão regular, que corresponde a uma linha em um cenário, e um bloco, que é executado quando o cenário chega a uma linha correspondente. Os grupos de captura na expressão regular são passados ​​para o bloco como parâmetros de bloco.

A etapa When tem um exemplo simples e em linha de ir de uma referência legível para uma página (“Documentação”) para um URL. As suítes do Real Cucumber geralmente colocam essa lógica em um método separado. O método visit é fornecido pelo Capybara. A capivara não é obrigada a usar o pepino, embora seja muito usada com ele. visit diz ao navegador controlado pelo Capybara para visitar o URL fornecido.

A etapa Then mostra como o conteúdo de uma página pode ser testado. expect/to é fornecido pelo RSpec (novamente, não é exigido pelo Cucumber, mas muito comumente usado com ele). have_css é fornecido pela Capivara. A expectativa é que o seletor CSS fornecido corresponda a um elemento na página que contém o texto fornecido. Observe que essa expectativa falharia se a solicitação do navegador falhasse.

Para obter mais exemplos, consulte o tópico “Definição da etapa”.

Instalação pura do Ruby #

Para instalar o Cucumber para uso com Ruby basta usar o comando

gem install cucumber

Alternativamente, se você estiver usando o bundler, você pode adicionar a seguinte linha ao seu Gemfile

gem 'cucumber'

E então execute o bundler

bundle install

[Acho que isso pertence ao seu próprio tópico, Instalação. Eu criei esse tópico e copiei esse exemplo lá. Quando esse tópico for aprovado, vou movê-lo para lá e excluir a cópia.]
