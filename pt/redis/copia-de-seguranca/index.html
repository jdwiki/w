O backup de uma instância remota do Redis pode ser feito com replicação. Isso é útil se você deseja tirar um instantâneo de um conjunto de dados antes de atualizar, excluir ou alterar um banco de dados Redis.

Backup de uma instância remota do Redis para uma instância local #

Na máquina em que você deseja fazer o backup, vá para a CLI do Redis:

redis-cli

Senha? #

Se o seu Redis DB mestre (aquele que você deseja replicar) tiver uma senha:

config set masterauth <password>

Iniciar replicação #

Execute o seguinte para iniciar a replicação:

SLAVEOF <host> <port>

Para verificar se a replicação está em andamento, execute:

INFO replication

E você deve ver uma saída como esta:

# Replication
role:slave
master_host:some-host.compute-1.amazonaws.com
master_port:6519
master_link_status:up
master_last_io_seconds_ago:3
master_sync_in_progress:0
slave_repl_offset:35492914
slave_priority:100
slave_read_only:1
connected_slaves:0
master_repl_offset:0
repl_backlog_active:0
repl_backlog_size:1048576
repl_backlog_first_byte_offset:0
repl_backlog_histlen:0

Observe que o master_link_status deve ser up.

Verificando o progresso da sincronização #

Quando a sincronização for concluída, a replicação de INFO deverá mostrar:

master_sync_in_progress:0

Para verificar se o conjunto de dados foi sincronizado, você pode comparar o tamanho do banco de dados:

DBSIZE

Salvando um dump de dados no disco #

Para salvar o banco de dados em disco de forma assíncrona:

BGSAVE
CONFIG GET dir

Então você deve encontrar um arquivo dump.rdb no diretório listado pelo comando config.

Parando a replicação #

Você pode interromper a replicação com:

SLAVEOF NO ONE
