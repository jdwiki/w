Existem mais de dez bibliotecas de cliente diferentes para usar com o Redis em Java. Um dos clientes mais populares é o Jedis.

Outras informações:

  • Clientes Java Redis
  • Jedis Github Repository
  • Documentação Jedi/Wiki

Conectando-se ao Redis #

Usando uma piscina

A maioria dos códigos desejará se conectar ao Redis usando um pool de objetos de conexão compartilhados. A conexão ao Redis usando um pool envolve dois blocos de código diferentes. No momento da inicialização, seu aplicativo precisa criar o pool de conexões:

    JedisPoolConfig poolCfg = new JedisPoolConfig();
    poolCfg.setMaxTotal(3);

    pool = new JedisPool(poolCfg, hostname, port, 500, password, false);

O JedisPoolConfig fornece opções para ajustar o pool.

À medida que seu aplicativo processa sua carga de trabalho, você precisará obter uma conexão do pool compartilhado usando o seguinte código:

    try (Jedis jedis = pool.getResource()) {

        ...
    }

A melhor prática é obter o objeto de conexão Jedis do pool dentro de um bloco try-with-resources.

Sem piscinas

Em alguns casos, como um aplicativo simples ou um teste de integração, você pode não querer lidar com pools compartilhados e, em vez disso, criar o objeto de conexão Jedis diretamente. Isso pode ser feito com o seguinte código:

try (Jedis jedis = new Jedis(hostname, port)) {
    jedis.connect();
    jedis.auth(password);
    jedis.select(db);

    . . .
}

Novamente, a melhor prática é criar o objeto cliente Jedis dentro de um bloco try-with-resources.

Obtendo Jedis #

A biblioteca Jedis geralmente é adicionada ao projeto Java usando um sistema de gerenciamento de dependência embutido no ambiente de construção do projeto. Dois sistemas de construção Java populares são Maven e Gradle.

Usando Gradle

Para adicionar a biblioteca Jedis a um projeto Gradle, você precisará configurar um repositório e adicionar uma dependência. O snippet a seguir mostra como adicionar a versão 2.9.0 da biblioteca Jedis a um projeto Gradle.

repositories {
    mavenCentral()
}

dependencies {
    compile 'redis.clients:jedis:2.9.0'
}

Usando Maven

Para adicionar Jedis a um projeto Maven, você precisa adicionar uma dependência à sua lista de dependências e fornecer as coordenadas da biblioteca. O snippet a seguir seria adicionado ao seu arquivo pom.xml:

<dependencies>
    <dependency>
        <groupId>redis.clients</groupId>
        <artifactId>jedis</artifactId>
        <version>2.9.0</version>
    </dependency>
</dependencies>

Executando comandos básicos Get/Set #

Depois de estabelecer uma conexão com o Redis, você pode obter e definir valores usando o objeto de conexão Jedis:

Pegue

String value = jedis.get(myKey);

Definir

jedis.put(myKey, "some value");

Executando Comandos #

Para executar um comando Redis usando Jedis, você faz chamadas de método no objeto Jedis que você criou a partir do pool. Jedis expõe comandos Redis como chamadas de método, alguns exemplos são:

- String get(String key) 
- Long geoadd(String key, double longitude, double latitude, String member)
- List<String> hmget(String key, String... fields)
- Long hsetnx(String key, String field, String value)

Se você quisesse definir o valor de uma chave String no Redis, usaria um bloco de código semelhante a:

try (Jedis jedis = pool.getResource()) {

   String myKey = "users:20";
   String myValue = "active";

   jedis.set(myKey, myValue);
}
