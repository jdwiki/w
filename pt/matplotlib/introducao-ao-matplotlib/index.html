Sintaxe Imperativa vs. Orientada a Objetos #

O Matplotlib suporta tanto a sintaxe orientada a objetos quanto a imperativa para plotagem. A sintaxe imperativa é intencionalmente projetada para ser muito próxima da sintaxe do Matlab.

A sintaxe imperativa (às vezes chamada de sintaxe de ‘máquina de estado’) emite uma sequência de comandos que atuam na figura ou eixo mais recente (como Matlab). A sintaxe orientada a objetos, por outro lado, atua explicitamente sobre os objetos (figura, eixo, etc.) de interesse. Um ponto-chave no zen of Python afirma que explícito é melhor que implícito, então a sintaxe orientada a objetos é mais pythônica. No entanto, a sintaxe imperativa é conveniente para novos convertidos do Matlab e para escrever scripts de plotagem pequenos e “descartáveis”. Abaixo está um exemplo dos dois estilos diferentes.

import matplotlib.pyplot as plt
import numpy as np

t = np.arange(0, 2, 0.01)
y = np.sin(4 * np.pi * t)

# Imperative syntax
plt.figure(1)
plt.clf()
plt.plot(t, y)
plt.xlabel('Time (s)')
plt.ylabel('Amplitude (V)')
plt.title('Sine Wave')
plt.grid(True)

# Object oriented syntax
fig = plt.figure(2)
fig.clf()
ax = fig.add_subplot(1,1,1)
ax.plot(t, y)
ax.set_xlabel('Time (s)')
ax.set_ylabel('Amplitude (V)')
ax.set_title('Sine Wave')
ax.grid(True)

Instalação e configuração #

Existem várias maneiras de instalar o matplotlib, algumas das quais dependerão do sistema que você está usando. Se você tiver sorte, poderá usar um gerenciador de pacotes para instalar facilmente o módulo matplotlib e suas dependências.

Janelas #

Em máquinas Windows, você pode tentar usar o gerenciador de pacotes pip para instalar o matplotlib. Veja aqui para informações sobre como configurar o pip em um ambiente Windows.

#OS X

É recomendado que você use o gerenciador de pacotes pip para instalar o matplotlib. Se você precisar instalar algumas das bibliotecas não Python em seu sistema (por exemplo, libfreetype), considere usar homebrew.

Se você não puder usar o pip por qualquer motivo, tente instalar de source.

#Linux

Idealmente, o gerenciador de pacotes do sistema ou pip deve ser usado para instalar o matplotlib, instalando o pacote python-matplotlib ou executando pip install matplotlib.

Se isso não for possível (por exemplo, você não tem privilégios sudo na máquina que está usando), então você pode instalar a partir de source usando a opção --user : python setup.py install --user. Normalmente, isso instalará o matplotlib em ~/.local.

Debian/Free #

sudo apt-get install python-matplotlib

Fedora/Chapéu Vermelho #

sudo yum install python-matplotlib

Solução de problemas #

Consulte o [site do matplotlib][resolução de problemas] para obter conselhos sobre como corrigir um arquivo matplotlib.

Matrizes bidimensionais (2D) #

Exiba uma matriz bidimensional (2D) nos eixos.

import numpy as np
from matplotlib.pyplot import imshow, show, colorbar

image = np.random.rand(4,4)
imshow(image)
colorbar()
show()

Personalizando um gráfico matplotlib #

import pylab as plt
import numpy as np

plt.style.use('ggplot')

fig = plt.figure(1)
ax = plt.gca()

# make some testing data
x = np.linspace( 0, np.pi, 1000 )
test_f = lambda x: np.sin(x)*3 + np.cos(2*x)

# plot the test data
ax.plot( x, test_f(x) , lw = 2)

# set the axis labels
ax.set_xlabel(r'$x$', fontsize=14, labelpad=10)
ax.set_ylabel(r'$f(x)$', fontsize=14, labelpad=25, rotation=0)

# set axis limits
ax.set_xlim(0,np.pi)

plt.draw()


# Customize the plot
ax.grid(1, ls='--', color='#777777', alpha=0.5, lw=1)
ax.tick_params(labelsize=12, length=0)
ax.set_axis_bgcolor('w')
# add a legend
leg = plt.legend( ['text'], loc=1 )
fr = leg.get_frame()
fr.set_facecolor('w')
fr.set_alpha(.7)
plt.draw()
