As funções de string executam operações em valores de string e retornam valores numéricos ou de string.

Usando funções de string, você pode, por exemplo, combinar dados, extrair uma substring, comparar strings ou converter uma string para todos os caracteres maiúsculos ou minúsculos.

Sintaxe #

  • CONCAT ( string_value1, string_value2 [, string_valueN ] )
  • LTRIM (expressão_caractere )
  • RTRIM (expressão_caractere )
  • SUBSTRING (expressão, início, comprimento)
  • ASCII (expressão_caractere)
  • REPLICATE ( string_expression ,integer_expression )
  • REVERSE ( string_expression )
  • SUPERIOR (expressão_caractere)
  • TRIM (string [caracteres FROM])
  • STRING_SPLIT ( string , separador )
  • STUFF ( character_expression , start , length , replaceWith_expression )
  • REPLACE ( string_expression , string_pattern , string_replacement )

Referência de funções de string para MySQL

Concatenar #

No SQL (ANSI/ISO padrão), o operador para concatenação de strings é ||. Esta sintaxe é suportada por todos os principais bancos de dados, exceto SQL Server:

SELECT 'Hello' || 'World' || '!'; --returns HelloWorld!

Muitos bancos de dados suportam uma função CONCAT para unir strings:

SELECT CONCAT('Hello', 'World'); --returns 'HelloWorld'

Alguns bancos de dados suportam o uso de CONCAT para unir mais de duas strings (o Oracle não suporta):

SELECT CONCAT('Hello', 'World', '!'); --returns 'HelloWorld!'

Em alguns bancos de dados, os tipos não string devem ser convertidos ou convertidos:

SELECT CONCAT('Foo', CAST(42 AS VARCHAR(5)), 'Bar'); --returns 'Foo42Bar'

Alguns bancos de dados (por exemplo, Oracle) realizam conversões implícitas sem perdas. Por exemplo, um CONCAT em um CLOB e um NCLOB produz um NCLOB. Um CONCAT em um número e um varchar2 resulta em um varchar2, etc.:

SELECT CONCAT(CONCAT('Foo', 42), 'Bar') FROM dual; --returns Foo42Bar

Alguns bancos de dados podem usar o operador + não padrão (mas na maioria, + funciona apenas para números):

SELECT 'Foo' + CAST(42 AS VARCHAR(5)) + 'Bar';

No SQL Server < 2012, onde CONCAT não é suportado, + é a única maneira de unir strings.

Comprimento #

Servidor SQL

O LEN não conta o espaço à direita.

SELECT LEN('Hello') -- returns 5

SELECT LEN('Hello '); -- returns 5

O DATALENGTH conta o espaço à direita.

SELECT DATALENGTH('Hello') -- returns 5

SELECT DATALENGTH('Hello '); -- returns 6

Deve-se notar, porém, que DATALENGTH retorna o comprimento da representação de byte subjacente da string, que depende, por exemplo, do charset usado para armazenar a string.

DECLARE @str varchar(100) = 'Hello ' --varchar is usually an ASCII string, occupying 1 byte per char
SELECT DATALENGTH(@str) -- returns 6

DECLARE @nstr nvarchar(100) = 'Hello ' --nvarchar is a unicode string, occupying 2 bytes per char
SELECT DATALENGTH(@nstr) -- returns 12

Oráculo

Sintaxe: Comprimento ( char )

Exemplos:

SELECT Length('Bible') FROM dual; --Returns 5
SELECT Length('righteousness') FROM dual; --Returns 13
SELECT Length(NULL) FROM dual; --Returns NULL

Veja também: ComprimentoB, ComprimentoC, Comprimento2, Comprimento4

Apara espaços vazios #

Trim é usado para remover o espaço de gravação no início ou no final da seleção

No MSSQL não existe um único TRIM()

SELECT LTRIM('  Hello  ') --returns 'Hello  '
SELECT RTRIM('  Hello  ') --returns '  Hello'
SELECT LTRIM(RTRIM('  Hello  ')) --returns 'Hello'

MySql e Oracle

SELECT TRIM('  Hello  ') --returns 'Hello'

Maiúsculas e minúsculas #

SELECT UPPER('HelloWorld') --returns 'HELLOWORLD'
SELECT LOWER('HelloWorld') --returns 'helloworld'

Dividir #

Divide uma expressão de string usando um separador de caracteres. Observe que STRING_SPLIT() é uma função com valor de tabela.

SELECT value FROM STRING_SPLIT('Lorem ipsum dolor sit amet.', ' ');

Resultado:

value
-----
Lorem
ipsum
dolor
sit
amet.

Substituir #

Sintaxe:

REPLACE( String para pesquisar , String para pesquisar e substituir , String para colocar na string original )

Exemplo:

SELECT REPLACE( 'Peter Steve Tom', 'Steve', 'Billy' ) --Return Values: Peter Billy Tom

Substring #

A sintaxe é: SUBSTRING ( string_expression, start, length ). Observe que as strings SQL são indexadas em 1.

SELECT SUBSTRING('Hello', 1, 2) --returns 'He'
SELECT SUBSTRING('Hello', 3, 3) --returns 'llo'

Isso é frequentemente usado em conjunto com a função LEN() para obter os últimos caracteres n de uma string de comprimento desconhecido.

DECLARE @str1 VARCHAR(10) = 'Hello', @str2 VARCHAR(10) = 'FooBarBaz';
SELECT SUBSTRING(@str1, LEN(@str1) - 2, 3) --returns 'llo'
SELECT SUBSTRING(@str2, LEN(@str2) - 2, 3) --returns 'Baz'

Material #

Coloque uma string em outra, substituindo 0 ou mais caracteres em uma determinada posição.

Nota: a posição start é indexada em 1 (você começa a indexar em 1, não em 0).

Sintaxe:

STUFF ( character_expression , start , length , replaceWith_expression )

Exemplo:

SELECT STUFF('FooBarBaz', 4, 3, 'Hello') --returns 'FooHelloBaz'

REGEXP #

Verifica se uma string corresponde a uma expressão regular (definida por outra string).

SELECT 'bedded' REGEXP '[a-f]' -- returns True

SELECT 'beam' REGEXP '[a-f]' -- returns False

ESQUERDA DIREITA #

A sintaxe é:
LEFT ( expressão-string , integer ) RIGHT ( string-expression , integer )

SELECT LEFT('Hello',2)  --return He  
SELECT RIGHT('Hello',2) --return lo

O Oracle SQL não possui funções LEFT e RIGHT. Eles podem ser emulados com SUBSTR e LENGTH.
SUBSTR ( expressão-string, 1, integer )
SUBSTR ( string-expression, length(string-expression)-integer+1, integer)

SELECT SUBSTR('Hello',1,2)  --return He  
SELECT SUBSTR('Hello',LENGTH('Hello')-2+1,2) --return lo

MARCHA RÉ #

A sintaxe é: REVERSE ( expressão de string )

SELECT REVERSE('Hello') --returns olleH

REPLICAR #

A função REPLICATE concatena uma string consigo mesma um número especificado de vezes.

A sintaxe é: REPLICATE ( string-expression , integer )

SELECT REPLICATE ('Hello',4) --returns 'HelloHelloHelloHello'

Substitua a função na consulta SQL Select and Update #

A função Replace no SQL é usada para atualizar o conteúdo de uma string. A chamada de função é REPLACE( ) para MySQL, Oracle e SQL Server.

A sintaxe da função Substituir é:

REPLACE (str, find, repl)

O exemplo a seguir substitui as ocorrências de South por Southern na tabela Employees:

Nome Endereço
Tiago Sul de Nova York
João Sul de Boston
Miguel Sul de San Diego

Selecione a Declaração:

Se aplicarmos a seguinte função Replace:

SELECT 
    FirstName, 
    REPLACE (Address, 'South', 'Southern') Address
FROM Employees 
ORDER BY FirstName

Resultado:

Nome Endereço
Tiago Sul de Nova York
João Sul de Boston
Miguel Sul de San Diego

Atualizar Declaração:

Podemos usar uma função de substituição para fazer alterações permanentes em nossa tabela por meio da seguinte abordagem.

Update Employees 
Set city = (Address, 'South', 'Southern');

Uma abordagem mais comum é usar isso em conjunto com uma cláusula WHERE como esta:

Update Employees 
Set Address = (Address, 'South', 'Southern')
Where Address LIKE 'South%';

PARSENAME #

BANCO DE DADOS: SQL Server

A função PARSENAME retorna a parte específica de uma determinada string (nome do objeto). nome do objeto pode conter string como nome do objeto, nome do proprietário, nome do banco de dados e nome do servidor.

Mais detalhes MSDN:PARSENAME

Sintaxe

PARSENAME('NameOfStringToParse',PartIndex)

Exemplo

Para obter o nome do objeto, use o índice da parte 1

SELECT PARSENAME('ServerName.DatabaseName.SchemaName.ObjectName',1)  // returns `ObjectName`
SELECT PARSENAME('[1012-1111].SchoolDatabase.school.Student',1)     // returns `Student`

Para obter o nome do esquema, use o índice da parte 2

SELECT PARSENAME('ServerName.DatabaseName.SchemaName.ObjectName',2)  // returns `SchemaName`
SELECT PARSENAME('[1012-1111].SchoolDatabase.school.Student',2)     // returns `school`

Para obter o nome do banco de dados, use o índice da parte 3

SELECT PARSENAME('ServerName.DatabaseName.SchemaName.ObjectName',3) // returns `DatabaseName`
SELECT PARSENAME('[1012-1111].SchoolDatabase.school.Student',3)    // returns `SchoolDatabase`

Para obter o nome do servidor, use o índice da parte 4

SELECT PARSENAME('ServerName.DatabaseName.SchemaName.ObjectName',4)  // returns `ServerName`
SELECT PARSENAME('[1012-1111].SchoolDatabase.school.Student',4)     // returns `[1012-1111]`

PARSENAME retornará nulo se a parte especificada não estiver presente na string do nome do objeto fornecido

INSTR #

Retorna o índice da primeira ocorrência de uma substring (zero se não for encontrada)

Sintaxe: INSTR (string, substring)

SELECT INSTR('FooBarBar', 'Bar') -- return 4
SELECT INSTR('FooBarBar', 'Xar') -- return 0
