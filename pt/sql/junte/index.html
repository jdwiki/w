JOIN é um método de combinar (unir) informações de duas tabelas. O resultado é um conjunto costurado de colunas de ambas as tabelas, definido pelo tipo de junção (INNER/OUTER/CROSS e LEFT/RIGHT/FULL, explicado abaixo) e critérios de junção (como as linhas de ambas as tabelas se relacionam).

Uma tabela pode ser unida a si mesma ou a qualquer outra tabela. Se as informações de mais de duas tabelas precisarem ser acessadas, várias junções podem ser especificadas em uma cláusula FROM.

Sintaxe #

  • [ { INTERNO | { { ESQUERDA | DIREITA | FULL } [ OUTER ] } } ] JOIN

As junções, como o próprio nome sugere, são uma maneira de consultar dados de várias tabelas de maneira conjunta, com as linhas exibindo colunas retiradas de mais de uma tabela.

Auto-ingresso #

Uma tabela pode ser unida a si mesma, com diferentes linhas correspondendo umas às outras por alguma condição. Neste caso de uso, os aliases devem ser usados ​​para distinguir as duas ocorrências da tabela.

No exemplo abaixo, para cada Funcionário na tabela Funcionários do banco de dados de exemplo, é retornado um registro contendo o primeiro nome do funcionário junto com o primeiro nome correspondente do gerente do funcionário. Como os gerentes também são funcionários, a tabela é unida a ela mesma:

SELECT 
    e.FName AS "Employee", 
    m.FName AS "Manager"
FROM   
    Employees e
JOIN   
    Employees m 
    ON e.ManagerId = m.Id

Essa consulta retornará os seguintes dados:

Funcionário Gerente
João Tiago
Miguel Tiago
Jonathas João

Diferenças entre junções internas/externas #

SQL tem vários tipos de junção para especificar se linhas (não) correspondentes são incluídas no resultado: INNER JOIN, LEFT OUTER JOIN, RIGHT OUTER JOIN e FULL OUTER JOIN (o INNER e palavras-chave OUTER são opcionais). A figura abaixo destaca as diferenças entre esses tipos de junção: a área azul representa os resultados retornados pela junção e a área branca representa os resultados que a junção não retornará.

Apresentação de imagens SQL de junção cruzada (referência):

Abaixo estão exemplos de esta resposta.

Por exemplo, existem duas tabelas como abaixo:

A    B
-    -
1    3
2    4
3    5
4    6

Observe que (1,2) são exclusivos de A, (3,4) são comuns e (5,6) são exclusivos de B.

Junção Interna #

Uma junção interna usando uma das consultas equivalentes fornece a interseção das duas tabelas, ou seja, as duas linhas que elas têm em comum:

select * from a INNER JOIN b on a.a = b.b;
select a.*,b.* from a,b where a.a = b.b;

a | b
--+--
3 | 3
4 | 4

Junção externa esquerda #

Uma junção externa esquerda fornecerá todas as linhas em A, mais todas as linhas comuns em B:

select * from a LEFT OUTER JOIN b on a.a = b.b;

a |  b
--+-----
1 | null
2 | null
3 |    3
4 |    4

Junção externa direita #

Da mesma forma, uma junção externa direita fornecerá todas as linhas em B, além de quaisquer linhas comuns em A:

select * from a RIGHT OUTER JOIN b on a.a = b.b;

a    |  b
-----+----
3    |  3
4    |  4
null |  5
null |  6

Junção externa completa #

Uma junção externa completa fornecerá a união de A e B, ou seja, todas as linhas em A e todas as linhas em B. Se algo em A não tiver um datum correspondente em B, a parte B será nula e vice-versa.

select * from a FULL OUTER JOIN b on a.a = b.b;

 a   |  b
-----+-----
   1 | null
   2 | null
   3 |    3
   4 |    4
null |    6
null |    5

JOIN Terminologia: Interno, Externo, Semi, Anti… #

Digamos que temos duas tabelas (A e B) e algumas de suas linhas correspondem (em relação à condição JOIN fornecida, seja ela qual for no caso específico):

Podemos usar vários tipos de junção para incluir ou excluir linhas correspondentes ou não correspondentes de ambos os lados e nomear corretamente a junção escolhendo os termos correspondentes no diagrama acima.

Os exemplos abaixo usam os seguintes dados de teste:

CREATE TABLE A (
    X varchar(255) PRIMARY KEY
);

CREATE TABLE B (
    Y varchar(255) PRIMARY KEY
);

INSERT INTO A VALUES
    ('Amy'),
    ('John'),
    ('Lisa'),
    ('Marco'),
    ('Phil');

INSERT INTO B VALUES
    ('Lisa'),
    ('Marco'),
    ('Phil'),
    ('Tim'),
    ('Vincent');

Junção interna #

Combina as linhas esquerda e direita que correspondem.

SELECT * FROM A JOIN B ON X = Y;

X      Y
------ -----
Lisa   Lisa
Marco  Marco
Phil   Phil

Junção Externa Esquerda #

Às vezes abreviado para “junção à esquerda”. Combina as linhas esquerda e direita que correspondem e inclui linhas esquerdas não correspondentes.

SELECT * FROM A LEFT JOIN B ON X = Y;

X      Y
-----  -----
Amy    NULL
John   NULL
Lisa   Lisa
Marco  Marco
Phil   Phil

Junção Externa Direita #

Às vezes abreviado para “junção direita”. Combina as linhas esquerda e direita que correspondem e inclui linhas direitas não correspondentes.

SELECT * FROM A RIGHT JOIN B ON X = Y;

X      Y
-----  -------
Lisa   Lisa
Marco  Marco
Phil   Phil
NULL   Tim
NULL   Vincent

Junção Externa Completa #

Às vezes abreviado para “junção completa”. União da junção externa esquerda e direita.

SELECT * FROM A FULL JOIN B ON X = Y;

X      Y
-----  -------
Amy    NULL
John   NULL
Lisa   Lisa
Marco  Marco
Phil   Phil
NULL   Tim
NULL   Vincent

Semi Junção Esquerda #

Inclui linhas da esquerda que correspondem às linhas da direita.

SELECT * FROM A WHERE X IN (SELECT Y FROM B);

X
-----
Lisa
Marco
Phil

Semi Junção Direita #

Inclui linhas direitas que correspondem às linhas esquerdas.

SELECT * FROM B WHERE Y IN (SELECT X FROM A);

Y
-----
Lisa
Marco
Phil

Como você pode ver, não há uma sintaxe IN dedicada para a semi-junção esquerda vs. direita - nós alcançamos o efeito simplesmente trocando as posições da tabela dentro do texto SQL.

Anti Semi Junção Esquerda #

Inclui as linhas da esquerda que não correspondem às linhas da direita.

![Left Anti Semi Join]]8

SELECT * FROM A WHERE X NOT IN (SELECT Y FROM B);

X
----
Amy
John

AVISO: Tenha cuidado se estiver usando NOT IN em uma coluna NULL! Mais detalhes aqui.

Anti Semi Junção Direita #

Inclui as linhas direitas que não correspondem às linhas esquerdas.

SELECT * FROM B WHERE Y NOT IN (SELECT X FROM A);

Y
-------
Tim
Vincent

Como você pode ver, não há sintaxe NOT IN dedicada para a anti-semi join esquerda vs. direita - nós alcançamos o efeito simplesmente trocando as posições da tabela dentro do texto SQL.

A junção cruzada é equivalente a uma junção interna com condição de junção que sempre corresponde, portanto, a consulta a seguir teria retornado o mesmo resultado:

SELECT * FROM A JOIN B ON 1 = 1;

Auto-inscrição #

Isso simplesmente denota uma tabela se unindo a ela mesma. Uma auto-junção pode ser qualquer um dos tipos de junção discutidos acima. Por exemplo, esta é uma auto-junção interna:

SELECT * FROM A A1 JOIN A A2 ON LEN(A1.X) < LEN(A2.X);

X     X
----  -----
Amy   John
Amy   Lisa
Amy   Marco
John  Marco
Lisa  Marco
Phil  Marco
Amy   Phil

Junção Externa Esquerda #

Uma Junção Externa Esquerda (também conhecida como Junção Esquerda ou Junção Externa) é uma Junção que garante que todas as linhas da tabela esquerda sejam representadas; se não existir nenhuma linha correspondente da tabela correta, seus campos correspondentes serão NULL.

O exemplo a seguir selecionará todos os departamentos e o primeiro nome dos funcionários que trabalham nesse departamento. Os departamentos sem funcionários ainda são retornados nos resultados, mas terão NULL para o nome do funcionário:

SELECT          Departments.Name, Employees.FName
FROM            Departments 
LEFT OUTER JOIN Employees 
ON              Departments.Id = Employees.DepartmentId

Isso retornaria o seguinte do banco de dados de exemplo:

Junção Implícita #

As junções também podem ser realizadas tendo várias tabelas na cláusula from, separadas por vírgulas , e definindo o relacionamento entre elas na cláusula where. Essa técnica é chamada de Junção Implícita (já que na verdade não contém uma cláusula join).

Todos os RDBMSs o suportam, mas a sintaxe geralmente é desaconselhada. As razões pelas quais é uma má ideia usar essa sintaxe são:

  • É possível obter junções cruzadas acidentais que retornam resultados incorretos, especialmente se você tiver muitas junções na consulta.
  • Se você pretendia uma junção cruzada, então não está claro na sintaxe (escreva CROSS JOIN em vez disso), e é provável que alguém o altere durante a manutenção.

O exemplo a seguir selecionará os nomes dos funcionários e o nome dos departamentos para os quais eles trabalham:

SELECT e.FName, d.Name
FROM   Employee e, Departments d
WHERE  e.DeptartmentId = d.Id

Isso retornaria o seguinte do banco de dados de exemplo:

e.FName d.Nome
Tiago RH
João RH
Ricardo Vendas

Junção interna explícita básica #

Uma junção básica (também chamada de “junção interna”) consulta dados de duas tabelas, com seu relacionamento definido em uma cláusula join.

O exemplo a seguir selecionará os nomes dos funcionários (FName) da tabela Employees e o nome do departamento para o qual trabalham (Name) da tabela Departments:

SELECT Employees.FName, Departments.Name
FROM   Employees
JOIN   Departments
ON Employees.DepartmentId = Departments.Id

Isso retornaria o seguinte do banco de dados de exemplo:

Employees.FName Departamentos.Nome
James RH
João RH
Ricardo Vendas

CROSS JOIN #

A junção cruzada faz um produto cartesiano dos dois membros, um produto cartesiano significa que cada linha de uma tabela é combinada com cada linha da segunda tabela na junção. Por exemplo, se TABLEA tiver 20 linhas e TABLEB tiver 20 linhas, o resultado seria 20*20 = 400 linhas de saída.

Usando exemplo de banco de dados

SELECT d.Name, e.FName
FROM   Departments d
CROSS JOIN Employees e;

Que retorna:

d.Nome e.FName
RH James
RH John
HR Michael
RH Jonathon

Entrando em uma Subconsulta #

A associação de uma subconsulta é frequentemente usada quando você deseja obter dados agregados de uma tabela filho/detalhes e exibi-los junto com os registros da tabela pai/cabeçalho. Por exemplo, você pode querer obter uma contagem de registros filho, uma média de alguma coluna numérica em registros filho ou a linha superior ou inferior com base em uma data ou campo numérico. Este exemplo usa aliases, o que torna as consultas mais fáceis de ler quando você tem várias tabelas envolvidas. Veja como é uma junção de subconsulta bastante típica. Nesse caso, estamos recuperando todas as linhas da tabela pai Purchase Orders e recuperando apenas a primeira linha de cada registro pai da tabela filha PurchaseOrderLineItems.

SELECT po.Id, po.PODate, po.VendorName, po.Status, item.ItemNo, 
  item.Description, item.Cost, item.Price
FROM PurchaseOrders po
LEFT JOIN 
     (
       SELECT l.PurchaseOrderId, l.ItemNo, l.Description, l.Cost, l.Price, Min(l.id) as Id 
       FROM PurchaseOrderLineItems l
       GROUP BY l.PurchaseOrderId, l.ItemNo, l.Description, l.Cost, l.Price
     ) AS item ON item.PurchaseOrderId = po.Id

CROSS APPLY & LATERAL JOIN #

Um tipo de JOIN muito interessante é o LATERAL JOIN (novo no PostgreSQL 9.3+),
que também é conhecido como CROSS APPLY/OUTER APPLY no SQL-Server e Oracle.

A ideia básica é que uma função com valor de tabela (ou subconsulta em linha) seja aplicada para cada linha que você unir.

Isso possibilita, por exemplo, unir apenas a primeira entrada correspondente em outra tabela.
A diferença entre uma junção normal e uma lateral está no fato de que você pode usar uma coluna que você juntou anteriormente na subconsulta que você “CROSS APPLY”.

Sintaxe:

PostgreSQL 9.3+

esquerda | direito | interno JOIN LATERAL

Servidor SQL:

CRUZ | EXTERIOR APLICAR

INNER JOIN LATERAL é o mesmo que CROSS APPLY
e LEFT JOIN LATERAL é o mesmo que OUTER APPLY

Exemplo de uso (PostgreSQL 9.3+):

SELECT * FROM T_Contacts 

--LEFT JOIN T_MAP_Contacts_Ref_OrganisationalUnit ON MAP_CTCOU_CT_UID = T_Contacts.CT_UID AND MAP_CTCOU_SoftDeleteStatus = 1 
--WHERE T_MAP_Contacts_Ref_OrganisationalUnit.MAP_CTCOU_UID IS NULL -- 989


LEFT JOIN LATERAL 
(
    SELECT 
         --MAP_CTCOU_UID    
         MAP_CTCOU_CT_UID   
        ,MAP_CTCOU_COU_UID  
        ,MAP_CTCOU_DateFrom 
        ,MAP_CTCOU_DateTo   
   FROM T_MAP_Contacts_Ref_OrganisationalUnit 
   WHERE MAP_CTCOU_SoftDeleteStatus = 1 
   AND MAP_CTCOU_CT_UID = T_Contacts.CT_UID 

    /*  
    AND 
    ( 
        (__in_DateFrom <= T_MAP_Contacts_Ref_OrganisationalUnit.MAP_KTKOE_DateTo) 
        AND 
        (__in_DateTo >= T_MAP_Contacts_Ref_OrganisationalUnit.MAP_KTKOE_DateFrom) 
    ) 
    */
   ORDER BY MAP_CTCOU_DateFrom 
   LIMIT 1 
) AS FirstOE

E para SQL-Server

SELECT * FROM T_Contacts 

--LEFT JOIN T_MAP_Contacts_Ref_OrganisationalUnit ON MAP_CTCOU_CT_UID = T_Contacts.CT_UID AND MAP_CTCOU_SoftDeleteStatus = 1 
--WHERE T_MAP_Contacts_Ref_OrganisationalUnit.MAP_CTCOU_UID IS NULL -- 989

-- CROSS APPLY -- = INNER JOIN 
OUTER APPLY    -- = LEFT JOIN 
(
    SELECT TOP 1 
         --MAP_CTCOU_UID    
         MAP_CTCOU_CT_UID   
        ,MAP_CTCOU_COU_UID  
        ,MAP_CTCOU_DateFrom 
        ,MAP_CTCOU_DateTo   
   FROM T_MAP_Contacts_Ref_OrganisationalUnit 
   WHERE MAP_CTCOU_SoftDeleteStatus = 1 
   AND MAP_CTCOU_CT_UID = T_Contacts.CT_UID 

    /*  
    AND 
    ( 
        (@in_DateFrom <= T_MAP_Contacts_Ref_OrganisationalUnit.MAP_KTKOE_DateTo) 
        AND 
        (@in_DateTo >= T_MAP_Contacts_Ref_OrganisationalUnit.MAP_KTKOE_DateFrom) 
    ) 
    */
   ORDER BY MAP_CTCOU_DateFrom 
) AS FirstOE

PARTICIPAÇÃO COMPLETA #

Um tipo de JOIN menos conhecido é o FULL JOIN.
(Nota: FULL JOIN não é suportado pelo MySQL conforme 2016)

Um FULL OUTER JOIN retorna todas as linhas da tabela à esquerda e todas as linhas da tabela à direita.

Se houver linhas na tabela à esquerda que não tenham correspondências na tabela à direita, ou se houver linhas na tabela à direita que não tenham correspondências na tabela à esquerda, essas linhas também serão listadas.

Exemplo 1 :

SELECT * FROM Table1

FULL JOIN Table2 
     ON 1 = 2

Exemplo 2:

SELECT 
     COALESCE(T_Budget.Year, tYear.Year) AS RPT_BudgetInYear 
    ,COALESCE(T_Budget.Value, 0.0) AS RPT_Value 
FROM T_Budget 

FULL JOIN tfu_RPT_All_CreateYearInterval(@budget_year_from, @budget_year_to) AS tYear 
      ON tYear.Year = T_Budget.Year

Observe que se você estiver usando soft-deletes, você terá que verificar o status de soft-delete novamente na cláusula WHERE (porque FULL JOIN se comporta como uma UNION);
É fácil ignorar esse pequeno fato, já que você coloca AP_SoftDeleteStatus = 1 na cláusula join.

Além disso, se você estiver fazendo um FULL JOIN, normalmente terá que permitir NULL na cláusula WHERE; esquecer de permitir NULL em um valor terá os mesmos efeitos que uma junção INNER, que é algo que você não deseja se estiver fazendo uma FULL JOIN.

Exemplo:

SELECT 
     T_AccountPlan.AP_UID
    ,T_AccountPlan.AP_Code
    ,T_AccountPlan.AP_Lang_EN
    ,T_BudgetPositions.BUP_Budget
    ,T_BudgetPositions.BUP_UID 
    ,T_BudgetPositions.BUP_Jahr
FROM T_BudgetPositions    

FULL JOIN T_AccountPlan
    ON T_AccountPlan.AP_UID = T_BudgetPositions.BUP_AP_UID 
    AND T_AccountPlan.AP_SoftDeleteStatus = 1 

WHERE (1=1) 
AND (T_BudgetPositions.BUP_SoftDeleteStatus = 1 OR T_BudgetPositions.BUP_SoftDeleteStatus IS NULL) 
AND (T_AccountPlan.AP_SoftDeleteStatus = 1 OR T_AccountPlan.AP_SoftDeleteStatus IS NULL)

JOINs recursivos #

Junções recursivas são frequentemente usadas para obter dados pai-filho. No SQL, eles são implementados com [expressões de tabela comuns] recursivas (https://www.wikiod.com/pt/sql/expressoes-de-tabela-comuns), por exemplo:

WITH RECURSIVE MyDescendants AS (
    SELECT Name
    FROM People
    WHERE Name = 'John Doe'

    UNION ALL

    SELECT People.Name
    FROM People
    JOIN MyDescendants ON People.Name = MyDescendants.Parent
)
SELECT * FROM MyDescendants;
