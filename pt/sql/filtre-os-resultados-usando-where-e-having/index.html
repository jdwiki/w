Sintaxe #

  • SELECT nome_da_coluna FROM nome_da_tabela WHERE valor do operador nome_da_coluna
  • SELECT nome_da_coluna, função_agregada (nome_da_coluna) FROM nome_da_tabela GROUP BY nome_da_coluna HAVING valor do operador de função_agregada (nome_da_coluna)

Use BETWEEN para filtrar resultados #

Os exemplos a seguir usam os bancos de dados de amostra Item Sales e Customers.

Nota: O operador BETWEEN é inclusivo.

Usando o operador BETWEEN com números:

SELECT * From ItemSales
WHERE Quantity BETWEEN 10 AND 17

Esta consulta retornará todos os registros ItemSales que possuem uma quantidade maior ou igual a 10 e menor ou igual a 17. Os resultados serão parecidos com:

ID Data de venda ItemId Quantidade Preço
1 01-07-2013 100 10 34,5
4 23-07-2013 100 15 34,5
5 24-07-2013 145 10 34,5

Usando o operador BETWEEN com valores de data:

SELECT * From ItemSales
WHERE SaleDate BETWEEN '2013-07-11' AND '2013-05-24'

Essa consulta retornará todos os registros ItemSales com uma SaleDate maior ou igual a 11 de julho de 2013 e menor ou igual a 24 de maio de 2013.

ID Data de venda ItemId Quantidade Preço
3 2013-07-11 100 20 34,5
4 23-07-2013 100 15 34,5
5 24-07-2013 145 10 34,5

Ao comparar valores de data e hora em vez de datas, pode ser necessário converter os valores de data e hora em valores de data ou adicionar ou subtrair 24 horas para obter os resultados corretos.

Usando o operador BETWEEN com valores de texto:

SELECT Id, FName, LName FROM Customers
WHERE LName BETWEEN 'D' AND 'L';

Exemplo ao vivo: violino SQL

Esta consulta retornará todos os clientes cujo nome esteja alfabeticamente entre as letras ‘D’ e ‘L’. Neste caso, os Clientes #1 e #3 serão devolvidos. O cliente nº 2, cujo nome começa com ‘M’, não será incluído.

ID FNome LNome
1 Guilherme Jones
3 Ricardo Davis

Use HAVING com funções agregadas #

Ao contrário da cláusula WHERE, HAVING pode ser usado com funções agregadas.

Uma função agregada é uma função em que os valores de várias linhas são agrupados como entrada em determinados critérios para formar um único valor de significado ou medida mais significativo (Wikipedia).

Funções agregadas comuns incluem COUNT(), SUM(), MIN() e MAX().

Este exemplo usa a Car Table dos bancos de dados de exemplo.

SELECT CustomerId, COUNT(Id) AS [Number of Cars]
FROM Cars
GROUP BY CustomerId
HAVING COUNT(Id) > 1

Essa consulta retornará a contagem de CustomerId e Number of Cars de qualquer cliente que tenha mais de um carro. Nesse caso, o único cliente que possui mais de um carro é o Cliente nº 1.

Os resultados serão parecidos com:

ID do cliente Número de carros
1 2

Cláusula WHERE com valores NULL/NOT NULL #

SELECT *
FROM Employees
WHERE ManagerId IS NULL

Esta instrução retornará todos os registros Employee onde o valor da coluna ManagerId for NULL.

O resultado será:

Id    FName    LName    PhoneNumber    ManagerId    DepartmentId
1     James    Smith    1234567890     NULL         1

SELECT *
FROM Employees
WHERE ManagerId IS NOT NULL

Esta instrução retornará todos os registros Employee onde o valor do ManagerId não é NULL.

O resultado será:

Id    FName       LName     PhoneNumber    ManagerId    DepartmentId
2     John        Johnson   2468101214     1            1
3     Michael     Williams  1357911131     1            2
4     Johnathon   Smith     1212121212     2            1

Observação: A mesma consulta não retornará resultados se você alterar a cláusula WHERE para WHERE ManagerId = NULL ou WHERE ManagerId <> NULL.

1: https://www.wikiod.com/pt/sql/bancos-de-dados-e-tabelas-de-exemplo#Banco de dados da loja de automóveis

A cláusula WHERE só retorna as linhas que correspondem aos seus critérios #

O Steam tem uma seção de jogos abaixo de US $ 10 em sua página da loja. Em algum lugar no coração de seus sistemas, provavelmente há uma consulta que se parece com:

SELECT * 
FROM Items 
WHERE Price < 10

E e OU #

Use IN para retornar linhas com um valor contido em uma lista #

Este exemplo usa a Car Table dos bancos de dados de exemplo.

SELECT *
FROM Cars
WHERE TotalCost IN (100, 200, 300)

Esta consulta retornará o carro nº 2 que custa 200 e o carro nº 3 que custa 100. Observe que isso é equivalente a usar várias cláusulas com OR, por exemplo:

SELECT *
FROM Cars
WHERE TotalCost = 100 OR TotalCost = 200 OR TotalCost = 300

Use LIKE para encontrar strings e substrings correspondentes #

Consulte documentação completa sobre o operador LIKE.

Este exemplo usa a [Tabela de funcionários](https://www.wikiod.com/pt/sql/bancos-de-dados-e-tabelas-de-exemplo#Banco de dados da loja de automóveis) dos bancos de dados de exemplo.

SELECT *
FROM Employees
WHERE FName LIKE 'John'

Esta consulta retornará apenas o Funcionário nº 1 cujo primeiro nome corresponde exatamente a ‘João’.

SELECT *
FROM Employees
WHERE FName like 'John%'

Adicionar % permite que você procure por uma substring:

  • John% - retornará qualquer Funcionário cujo nome comece com ‘John’, seguido por qualquer quantidade de caracteres
  • %John - retornará qualquer Funcionário cujo nome termine com ‘John’, seguido por qualquer quantidade de caracteres
  • %John% - retornará qualquer Funcionário cujo nome contenha ‘John’ em qualquer lugar dentro do valor

Nesse caso, a consulta retornará o Funcionário nº 2 cujo nome é ‘John’, bem como o Funcionário nº 4 cujo nome é ‘Johnathon’.

Use HAVING para verificar várias condições em um grupo #

Tabela de pedidos

ID do cliente ProductId Quantidade Preço
1 2 5 100
1 3 2 200
1 4 1 500
2 1 4 50
3 5 6 700

Para verificar os clientes que encomendaram ambos - ProductID 2 e 3, HAVING pode ser usado

 select customerId
 from orders
 where productID in (2,3)
 group by customerId
 having count(distinct productID) = 2

Valor de retorno:

customerId
1

A consulta seleciona apenas registros com os productIDs em questão e com a cláusula HAVING verifica se há grupos com 2 productIds e não apenas um.

Outra possibilidade seria

 select customerId
 from orders
 group by customerId
 having sum(case when productID = 2 then 1 else 0 end) > 0
    and sum(case when productID = 3 then 1 else 0 end) > 0

Esta consulta seleciona apenas grupos com pelo menos um registro com productID 2 e pelo menos um com productID 3.

Onde EXISTE #

Selecionará registros em TableName que tenham registros correspondentes em TableName1.

SELECT * FROM TableName t WHERE EXISTS (
    SELECT 1 FROM TableName1 t1 where t.Id = t1.Id)
