Sintaxe #

  • Function([DISTINCT] expressão) -DISTINCT é um parâmetro opcional
  • AVG ( expressão [ ALL | DISTINCT ] )
  • COUNT( { [ALL | DISTINCT ] expressão ] | * } )
  • AGRUPAMENTO(<expressão_coluna>)
  • MAX ( expressão [ ALL | DISTINCT ] )
  • MIN ( expressão [ ALL | DISTINCT ] )
  • SUM ( [ ALL | DISTINCT ] expressão ) -VAR (expressão [ALL | DISTINCT]) OVER ( [ partition_by_clause ] order_by_clause ) -VARP (expressão [ALL | DISTINCT]) OVER ( [ partition_by_clause ] order_by_clause
  • STDEV (expressão [ALL | DISTINCT]) OVER ( [ partition_by_clause ] order_by_clause ) -STDEVP (expressão [ALL | DISTINCT]) OVER ( [ partition_by_clause ] order_by_clause )

No gerenciamento de banco de dados, uma função agregada é uma função em que os valores de várias linhas são agrupados como entrada em determinados critérios para formar um único valor de significado ou medida mais significativo, como um conjunto, uma bolsa ou uma lista.

MIN        returns the smallest value in a given column
MAX        returns the largest value in a given column
SUM        returns the sum of the numeric values in a given column
AVG        returns the average value of a given column
COUNT      returns the total number of values in a given column
COUNT(*)   returns the number of rows in a table
GROUPING   Is a column or an expression that contains a column in a GROUP BY clause.
STDEV      returns the statistical standard deviation of all values in the specified expression.
STDEVP     returns the statistical standard deviation for the population for all values in the specified expression.
VAR        returns the statistical variance of all values in the specified expression. may be followed by the OVER clause.
VARP       returns the statistical variance for the population for all values in the specified expression.

Funções agregadas são usadas para calcular em uma “coluna retornada de dados numéricos” da sua instrução SELECT. Eles basicamente resumem os resultados de uma determinada coluna de dados selecionados. - SQLCourse2.com

Todas as funções agregadas ignoram valores NULL.

Concatenação de listas #

Crédito parcial para this SO resposta.

A concatenação de lista agrega uma coluna ou expressão combinando os valores em uma única string para cada grupo. Uma string para delimitar cada valor (em branco ou uma vírgula quando omitida) e a ordem dos valores no resultado podem ser especificadas. Embora não faça parte do padrão SQL, todos os principais fornecedores de banco de dados relacional o suportam à sua maneira.

MySQL #

SELECT ColumnA
     , GROUP_CONCAT(ColumnB ORDER BY ColumnB SEPARATOR ',') AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

Oracle e DB2 #

SELECT ColumnA
     , LISTAGG(ColumnB, ',') WITHIN GROUP (ORDER BY ColumnB) AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

PostgreSQL #

SELECT ColumnA
     , STRING_AGG(ColumnB, ',' ORDER BY ColumnB) AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

Servidor SQL #

SQL Server 2016 e anteriores #

(CTE incluído para incentivar o princípio DRY)

  WITH CTE_TableName AS (
       SELECT ColumnA, ColumnB
         FROM TableName)
SELECT t0.ColumnA
     , STUFF((
       SELECT ',' + t1.ColumnB
         FROM CTE_TableName t1
        WHERE t1.ColumnA = t0.ColumnA
        ORDER BY t1.ColumnB
          FOR XML PATH('')), 1, 1, '') AS ColumnBs
  FROM CTE_TableName t0
 GROUP BY t0.ColumnA
 ORDER BY ColumnA;

SQL Server 2017 e SQL Azure #

SELECT ColumnA
     , STRING_AGG(ColumnB, ',') WITHIN GROUP (ORDER BY ColumnB) AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

SQLite #

sem encomenda:

SELECT ColumnA
     , GROUP_CONCAT(ColumnB, ',') AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

o pedido requer uma subconsulta ou CTE:

  WITH CTE_TableName AS (
       SELECT ColumnA, ColumnB
         FROM TableName
        ORDER BY ColumnA, ColumnB)
SELECT ColumnA
     , GROUP_CONCAT(ColumnB, ',') AS ColumnBs
  FROM CTE_TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

SOMA #

A função Sum soma o valor de todas as linhas do grupo. Se a cláusula group by for omitida, somará todas as linhas.

select sum(salary) TotalSalary
from employees;
TotalSalário
2500 
select DepartmentId, sum(salary) TotalSalary
from employees
group by DepartmentId;
ID de departamento TotalSalário
1 2000
2 500

AVG() #

A função agregada AVG() retorna a média de uma determinada expressão, geralmente valores numéricos em uma coluna. Suponha que temos uma tabela contendo o cálculo anual da população em cidades ao redor do mundo. Os registros para a cidade de Nova York são semelhantes aos abaixo:

TABELA DE EXEMPLO #

cidade_nome população ano
Cidade de Nova York 8.550.405 2015
Cidade de Nova York … …
Cidade de Nova York 8.000.906 2005

Para selecionar a população média da cidade de Nova York, EUA, a partir de uma tabela contendo nomes de cidades, medições populacionais e anos de medição dos últimos dez anos:

CONSULTA #

select city_name, AVG(population) avg_population
from city_population
where city_name = 'NEW YORK CITY';

Observe como o ano de medição está ausente da consulta, pois a média da população está sendo calculada ao longo do tempo.

RESULTADOS #

cidade_nome avg_population
Cidade de Nova York 8.250.754

Nota: A função AVG() converterá valores em tipos numéricos. Isso é especialmente importante ter em mente ao trabalhar com datas.

Contar #

Você pode contar o número de linhas:

SELECT count(*) TotalRows
FROM employees;
TotalRows
4

Ou conte os funcionários por departamento:

SELECT DepartmentId, count(*) NumEmployees
FROM employees
GROUP BY DepartmentId;
ID de departamento NumEmployees
1 3
2 1

Você pode contar sobre uma coluna/expressão com o efeito que não contará os valores NULL:

SELECT count(ManagerId) mgr
FROM EMPLOYEES;
gerente
3

(Há uma coluna managerID de valor nulo)

Você também pode usar DISTINCT dentro de outra função, como COUNT, para encontrar apenas os membros DISTINCT do conjunto para realizar a operação.

Por exemplo:

 SELECT COUNT(ContinentCode) AllCount
 ,      COUNT(DISTINCT ContinentCode) SingleCount
 FROM Countries;

Retornará valores diferentes. O SingleCount contará apenas continentes individuais uma vez, enquanto o AllCount incluirá duplicatas.

Máx. #

Encontre o valor máximo da coluna:

select max(age) from employee;

O exemplo acima retornará o maior valor para a coluna age da tabela employee.

Sintaxe:

SELECT MAX(column_name) FROM table_name;

Min #

Encontre o menor valor da coluna:

 select min(age) from employee;

O exemplo acima retornará o menor valor para a coluna age da tabela employee.

Sintaxe:

 SELECT MIN(column_name) FROM table_name;
