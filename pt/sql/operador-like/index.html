Sintaxe #

  • Wild Card com % : SELECT * FROM [table] WHERE [column_name] Curtir ‘%Valor%’

Wild Card com _ : SELECT * FROM [table] WHERE [column_name] Curtir ‘V_n%’

Wild Card com [charlist] : SELECT * FROM [table] WHERE [column_name] Curtir ‘V[abc]n%’

A condição LIKE na cláusula WHERE é usada para pesquisar valores de coluna que correspondam ao padrão fornecido. Os padrões são formados usando os dois caracteres curinga a seguir

  • % (Símbolo de porcentagem) - Usado para representar zero ou mais caracteres
  • _ (Sublinhado) - Usado para representar um único caractere

Corresponder ao padrão aberto #

O curinga % anexado ao início ou fim (ou ambos) de uma string permitirá que 0 ou mais de qualquer caractere antes do início ou após o final do padrão corresponda.

Usar ‘%’ no meio permitirá que 0 ou mais caracteres entre as duas partes do padrão correspondam.

Vamos usar esta Tabela de Funcionários:

Correspondência de caractere único #

Para ampliar as seleções de uma instrução de linguagem de consulta estruturada (SQL-SELECT), podem ser usados ​​caracteres curinga, o sinal de porcentagem (%) e o sublinhado (_).

O caractere _ (sublinhado) pode ser usado como curinga para qualquer caractere único em uma correspondência de padrão.

Encontre todos os funcionários cujo Fname comece com ‘j’ e termine com ’n’ e tenha exatamente 3 caracteres no Fname.

SELECT * FROM Employees WHERE FName LIKE 'j_n'

O caractere _ (sublinhado) também pode ser usado mais de uma vez como curinga para combinar padrões.

Por exemplo, esse padrão corresponderia a “jon”, “jan”, “jen”, etc.

Esses nomes não serão mostrados “jn”,“john”,“jordan”, “justin”, “jason”, “julian”, “jillian”, “joann” porque em nossa consulta um sublinhado é usado e pode pular exatamente um caractere, então o resultado deve ser de 3 caracteres Fname.

Por exemplo, esse padrão corresponderia a “LaSt”, “LoSt”, “HaLt”, etc.

SELECT * FROM Employees WHERE FName LIKE '_A_T'

Instrução ESCAPE na consulta LIKE #

Se você implementar uma pesquisa de texto como consulta LIKE, você normalmente faz assim:

SELECT * 
FROM T_Whatever 
WHERE SomeField LIKE CONCAT('%', @in_SearchText, '%')

No entanto, (além do fato de que você não deve necessariamente usar LIKE quando você pode usar fulltext-search) isso cria um problema quando alguém insere texto como “50%” ou “a_b”.

Então (em vez de mudar para pesquisa de texto completo), você pode resolver esse problema usando a instrução LIKE-escape:

SELECT * 
FROM T_Whatever 
WHERE SomeField LIKE CONCAT('%', @in_SearchText, '%') ESCAPE '\'

Isso significa que \ agora será tratado como caractere ESCAPE. Isso significa que agora você pode apenas acrescentar \ a cada caractere na string que você pesquisar, e os resultados começarão a ser corretos, mesmo quando o usuário inserir um caractere especial como % ou _.

por exemplo.

string stringToSearch = "abc_def 50%";
string newString = "";
foreach(char c in stringToSearch) 
     newString += @"\" + c;
 
sqlCmd.Parameters.Add("@in_SearchText", newString); 
// instead of sqlCmd.Parameters.Add("@in_SearchText", stringToSearch);

Observação: O algoritmo acima é apenas para fins de demonstração. Não funcionará nos casos em que 1 grafema consiste em vários caracteres (utf-8). por exemplo. string stringToSearch = "Les Mise\u0301rables"; Você precisará fazer isso para cada grafema, não para cada caractere. Você não deve usar o algoritmo acima se estiver lidando com idiomas asiáticos/leste-asiáticos/sul-asiáticos. Ou melhor, se você quer o código correto para começar, você deve fazer isso para cada graphemeCluster.

Veja também https://stackoverflow.com/questions/1009689/reversestring-a-c-sharp-interview-question/36312251#36312251

Corresponder por intervalo ou conjunto #

Corresponde a qualquer caractere único dentro do intervalo especificado (por exemplo: [a-f]) ou conjunto (por exemplo: [abcdef]).

Este padrão de intervalo corresponderia a “gary”, mas não a “mary”:

SELECT * FROM Employees WHERE FName LIKE '[a-g]ary'

Este padrão definido corresponderia a “mary”, mas não a “gary”:

SELECT * FROM Employees WHERE Fname LIKE '[lmnop]ary'

O intervalo ou conjunto também pode ser negado anexando o acento circunflexo ^ antes do intervalo ou conjunto:

Este padrão de intervalo não corresponderá a “gary”, mas corresponderá a “mary”:

SELECT * FROM Employees WHERE FName LIKE '[^a-g]ary'

Este padrão definido não corresponderá a “mary”, mas corresponderá a “gary”:

SELECT * FROM Employees WHERE Fname LIKE '[^lmnop]ary'

Pesquise um intervalo de caracteres #

A instrução a seguir corresponde a todos os registros com FName que começam com uma letra de A a F de [Employees][1] Table.

SELECT * FROM Employees WHERE FName LIKE '[A-F]%'

[1]: https://www.wikiod.com/pt/sql/bancos-de-dados-e-tabelas-de-exemplo#Banco de dados da loja de automóveis

Caracteres curinga #

caracteres curinga são usados ​​com o operador SQL LIKE. Os curingas SQL são usados ​​para pesquisar dados em uma tabela.

Os curingas em SQL são:%, _, [charlist], [^charlist]

% - Um substituto para zero ou mais caracteres

   Eg:  //selects all customers with a City starting with "Lo"
        SELECT * FROM Customers
        WHERE City LIKE 'Lo%';

       //selects all customers with a City containing the pattern "es"
      SELECT * FROM Customers
       WHERE City LIKE '%es%';

_ - Um substituto para um único caractere

Eg://selects all customers with a City starting with any character, followed by "erlin"
SELECT * FROM Customers
WHERE City LIKE '_erlin';

[charlist] - Conjuntos e intervalos de caracteres para corresponder

Eg://selects all customers with a City starting with "a", "d", or "l"
SELECT * FROM Customers
WHERE City LIKE '[adl]%';

//selects all customers with a City starting with "a", "d", or "l"
SELECT * FROM Customers
WHERE City LIKE '[a-c]%';

[^charlist] - Corresponde apenas a um caractere NÃO especificado entre colchetes

Eg://selects all customers with a City starting with a character that is not "a", "p", or "l"
SELECT * FROM Customers
WHERE City LIKE '[^apl]%';

or

SELECT * FROM Customers
WHERE City NOT LIKE '[apl]%' and city like '_%';

Combine QUALQUER contra TODOS #

Corresponder a qualquer:
Deve corresponder a pelo menos uma string. Neste exemplo, o tipo de produto deve ser ’eletrônicos’, ’livros’ ou ‘vídeo’.

SELECT *
FROM   purchase_table
WHERE  product_type LIKE ANY ('electronics', 'books', 'video');

Corresponder a todos (deve atender a todos os requisitos).
Neste exemplo, tanto ‘reino unido’ e ’london’ e ’estrada oriental’ (incluindo variações) devem ser correspondidos.

SELECT *
FROM   customer_table
WHERE  full_address LIKE ALL ('%united kingdom%', '%london%', '%eastern road%');

Seleção negativa:
Use ALL para excluir todos os itens.
Este exemplo gera todos os resultados em que o tipo de produto não é ’eletrônicos’, ’livros’ e não ‘vídeo’.

SELECT *
FROM   customer_table
WHERE  product_type NOT LIKE ALL ('electronics', 'books', 'video');
