A instrução CREATE TABLE é usada para criar uma nova tabela no banco de dados. Uma definição de tabela consiste em uma lista de colunas, seus tipos e quaisquer restrições de integridade.

Sintaxe #

  • CREATE TABLE tableName( [ColumnName1] [datatype1] [, [ColumnName2] [datatype2] …] )

Parâmetros #

Parâmetro Detalhes
tableName O nome da tabela
colunas Contém uma ’enumeração’ de todas as colunas que a tabela possui. Consulte Criar uma nova tabela para obter mais detalhes.

Os nomes das tabelas devem ser exclusivos.

Criar uma nova tabela #

Uma tabela básica de “Empregados”, contendo um ID e o nome e sobrenome do funcionário junto com seu número de telefone, pode ser criada usando

CREATE TABLE Employees(
    Id int identity(1,1) primary key not null,
    FName varchar(20) not null,
    LName varchar(20) not null,
    PhoneNumber varchar(10) not null
);

Este exemplo é específico para Transact-SQL

CREATE TABLE cria uma nova tabela no banco de dados, seguida pelo nome da tabela, Employees

Isso é seguido pela lista de nomes de colunas e suas propriedades, como o ID

Id int identity(1,1) not null
Valor Significado
Id o nome da coluna.
int é o tipo de dados.
identidade(1,1) indica que a coluna terá valores gerados automaticamente começando em 1 e incrementando em 1 para cada nova linha.
chave primária indica que todos os valores nesta coluna terão valores únicos
não nulo afirma que esta coluna não pode ter valores nulos

Criar tabela a partir da seleção #

Você pode querer criar uma duplicata de uma tabela:

CREATE TABLE ClonedEmployees AS SELECT * FROM Employees;

Você pode usar qualquer um dos outros recursos de uma instrução SELECT para modificar os dados antes de passá-los para a nova tabela. As colunas da nova tabela são criadas automaticamente de acordo com as linhas selecionadas.

CREATE TABLE ModifiedEmployees AS
SELECT Id, CONCAT(FName," ",LName) AS FullName FROM Employees
WHERE Id > 10;

CRIAR TABELA COM CHAVE ESTRANGEIRA #

Abaixo você pode encontrar a tabela Employees com uma referência à tabela Cities.

CREATE TABLE Cities(
    CityID INT IDENTITY(1,1) NOT NULL,
    Name VARCHAR(20) NOT NULL,
    Zip VARCHAR(10) NOT NULL
);

CREATE TABLE Employees(
    EmployeeID INT IDENTITY (1,1) NOT NULL,
    FirstName VARCHAR(20) NOT NULL,
    LastName VARCHAR(20) NOT NULL,
    PhoneNumber VARCHAR(10) NOT NULL,
    CityID INT FOREIGN KEY REFERENCES Cities(CityID)
);

Aqui você pode encontrar um diagrama de banco de dados.

A coluna CityID da tabela Employees fará referência à coluna CityID da tabela Cities. Abaixo você pode encontrar a sintaxe para fazer isso.

CityID INT FOREIGN KEY REFERENCES Cities(CityID)
Valor Significado
CidadeID Nome da coluna
int tipo de coluna
CHAVE ESTRANGEIRA Torna a chave estrangeira (opcional)
REFERÊNCIAS
Cidades(CityID)		 Faz a referência
à tabela Cities coluna CityID

Importante: você não pode fazer referência a uma tabela que não existe no banco de dados. Seja fonte para fazer primeiro a tabela Cities e depois a tabela Employees. Se você fizer isso vice-versa, ele lançará um erro.

Duplicar uma tabela #

Para duplicar uma tabela, basta fazer o seguinte:

CREATE TABLE newtable LIKE oldtable; 
INSERT newtable SELECT * FROM oldtable;

Crie uma tabela temporária ou na memória #

PostgreSQL e SQLite #

Para criar uma tabela temporária local para a sessão:

CREATE TEMP TABLE MyTable(...);

Servidor SQL #

Para criar uma tabela temporária local para a sessão:

CREATE TABLE #TempPhysical(...);

Para criar uma tabela temporária visível para todos:

CREATE TABLE ##TempPhysicalVisibleToEveryone(...);

Para criar uma tabela na memória:

DECLARE @TempMemory TABLE(...);
