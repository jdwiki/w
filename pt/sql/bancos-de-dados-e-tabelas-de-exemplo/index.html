Banco de dados da loja de automóveis #

No exemplo a seguir - Banco de dados para uma loja de automóveis, temos uma lista de departamentos, funcionários, clientes e carros de clientes. Estamos usando chaves estrangeiras para criar relacionamentos entre as várias tabelas.

Exemplo ao vivo: violino SQL

Relacionamentos entre tabelas #

  • Cada Departamento pode ter 0 ou mais Colaboradores
  • Cada Funcionário pode ter 0 ou 1 Gerente
  • Cada Cliente pode ter 0 ou mais Carros

Departamentos #

ID Nome
1 RH
2 Vendas
3 Tecnologia

Instruções SQL para criar a tabela:

CREATE TABLE Departments (
    Id INT NOT NULL AUTO_INCREMENT,
    Name VARCHAR(25) NOT NULL,
    PRIMARY KEY(Id)
);

INSERT INTO Departments
    ([Id], [Name])
VALUES
    (1, 'HR'),
    (2, 'Sales'),
    (3, 'Tech')
;

Instruções SQL para criar a tabela:

CREATE TABLE Employees (
    Id INT NOT NULL AUTO_INCREMENT,
    FName VARCHAR(35) NOT NULL,
    LName VARCHAR(35) NOT NULL,
    PhoneNumber VARCHAR(11),
    ManagerId INT,
    DepartmentId INT NOT NULL,
    Salary INT NOT NULL,
    HireDate DATETIME NOT NULL,
    PRIMARY KEY(Id),
    FOREIGN KEY (ManagerId) REFERENCES Employees(Id),
    FOREIGN KEY (DepartmentId) REFERENCES Departments(Id)
);

    (4, 'Johnathon', 'Smith', 1212121212, '2', 1, 500, '24-07-2016')
;

CREATE TABLE Cars (
    Id INT NOT NULL AUTO_INCREMENT,

Banco de dados da biblioteca #

Neste banco de dados de exemplo para uma biblioteca, temos as tabelas Authors, Books e BooksAuthors.

Exemplo ao vivo: violino SQL

Autores e Livros são conhecidos como tabelas base, pois contêm definição de coluna e dados para as entidades reais no modelo relacional. BooksAuthors é conhecido como tabela de relacionamento, pois essa tabela define o relacionamento entre as tabelas Books e Authors.

Relacionamentos entre tabelas #

  • Cada autor pode ter 1 ou mais livros
  • Cada livro pode ter 1 ou mais autores

CREATE TABLE Authors (
    Id INT NOT NULL AUTO_INCREMENT,

Exemplos #

Ver todos os autores (ver exemplo ao vivo):

SELECT * FROM Authors;

Ver todos os títulos de livros (ver exemplo ao vivo):

SELECT * FROM Books;

Veja todos os livros e seus autores (ver exemplo ao vivo):

SELECT
  ba.AuthorId,
  a.Name AuthorName,
  ba.BookId,
  b.Title BookTitle
FROM BooksAuthors ba
  INNER JOIN Authors a ON a.id = ba.authorid
  INNER JOIN Books b ON b.id = ba.bookid
;

Tabela de países #

Neste exemplo, temos uma tabela Países. Uma tabela para países tem muitos usos, especialmente em aplicações financeiras envolvendo moedas e taxas de câmbio.

Exemplo ao vivo: violino SQL

Alguns aplicativos de software de dados de mercado, como Bloomberg e Reuters, exigem que você forneça à API um código de país de 2 ou 3 caracteres junto com o código da moeda. Portanto, esta tabela de exemplo tem a coluna de código ISO de 2 caracteres e as colunas de código ISO3 de 3 caracteres.

Países #

(ver tabela)

| ID | ISO | ISO3 | ISONumérico | PaísName | Capitais | Código Continente | Código da Moeda | |:—|:—-|:—–|:———–|:————–|:- ———-|:————–|:————-| | 1 | UA | UA | 36 | Austrália | Camberra | CO | USD | | 2 | DE | DEU | 276 | Germany | Berlin | EU | EUR | | 2 | EM | IND | 356 | Índia | Nova Deli | AS | INR | | 3 | O | LAO | 418 | laos | Vienciana | AS | LAGO | | 4 | EUA | EUA | 840 | Estados Unidos | Washington | NA | USD | | 5 | ZW | PAÍS | 716 | Zimbábue | Harare | AF | ZWL |

SQL para criar a tabela:

CREATE TABLE Countries (
    Id INT NOT NULL AUTO_INCREMENT,
    ISO VARCHAR(2) NOT NULL,
    ISO3 VARCHAR(3) NOT NULL,
    ISONumeric INT NOT NULL,
    CountryName VARCHAR(64) NOT NULL,
    Capital VARCHAR(64) NOT NULL,
    ContinentCode VARCHAR(2) NOT NULL,
    CurrencyCode VARCHAR(3) NOT NULL,
    PRIMARY KEY(Id)
)
;

INSERT INTO Countries
    (ISO, ISO3, ISONumeric, CountryName, Capital, ContinentCode, CurrencyCode)
