A instrução SELECT está no centro da maioria das consultas SQL. Ele define qual conjunto de resultados deve ser retornado pela consulta e é quase sempre usado em conjunto com a cláusula FROM, que define quais partes do banco de dados devem ser consultadas.

Sintaxe #

  • SELECIONE [DISTINTO] [coluna1] [, [coluna2] … ] DE [tabela] [ ONDE condição ] [ AGRUPAR POR [coluna1] [, [coluna2] … ]

[ TENDO [coluna1] [, [coluna2] … ]

[ ENCOMENDA POR ASC | DESC]

SELECT determina quais dados de colunas devem ser retornados e em qual ordem DE uma determinada tabela (desde que correspondam aos outros requisitos em sua consulta especificamente - onde e com filtros e junções).

SELECT Name, SerialNumber
FROM ArmyInfo

retornará apenas resultados das colunas Name e Serial Number, mas não da coluna chamada Rank, por exemplo

SELECT *
FROM ArmyInfo

indica que todas as colunas serão retornadas. No entanto, observe que é uma prática ruim SELECT *, pois você está literalmente retornando todas as colunas de uma tabela.

Usando o caractere curinga para selecionar todas as colunas em uma consulta. #

Considere um banco de dados com as duas tabelas a seguir.

Tabela de funcionários:

ID FNome LNome DeptId
1 Tiago Smith 3
2 João Johnson 4

Tabela de departamentos:

ID Nome
1 Vendas
2 Marketing
3 Finanças
4 TI

Instrução de seleção simples #

* é o caractere curinga usado para selecionar todas as colunas disponíveis em uma tabela.

Quando usado como um substituto para nomes de colunas explícitos, ele retorna todas as colunas em todas as tabelas nas quais uma consulta está selecionando FROM. Este efeito se aplica a todas as tabelas que a consulta acessa por meio de suas cláusulas JOIN.

Considere a seguinte consulta:

SELECT * FROM Employees

Ele retornará todos os campos de todas as linhas da tabela Employees:

ID FNome LNome DeptId
1 Tiago Smith 3
2 João Johnson 4

Notação de ponto #

Para selecionar todos os valores de uma tabela específica, o caractere curinga pode ser aplicado à tabela com notação de ponto.

Considere a seguinte consulta:

SELECT 
    Employees.*, 
    Departments.Name
FROM 
    Employees
JOIN 
    Departments 
    ON Departments.Id = Employees.DeptId

Isso retornará um conjunto de dados com todos os campos na tabela Employee, seguidos apenas pelo campo Name na tabela Departments:

ID FNome LNome DeptId Nome
1 Tiago Smith 3 Finanças
2 João Johnson 4 TI

Avisos contra o uso

É geralmente recomendado que o uso de * seja evitado no código de produção sempre que possível, pois pode causar vários problemas em potencial, incluindo:

  1. Excesso de E/S, carga de rede, uso de memória e assim por diante, devido ao mecanismo de banco de dados ler dados desnecessários e transmiti-los ao código front-end. Isso é particularmente uma preocupação quando pode haver campos grandes, como aqueles usados ​​para armazenar notas longas ou arquivos anexados.
  2. Carga de E/S adicional adicional se o banco de dados precisar enviar resultados internos para o disco como parte do processamento de uma consulta mais complexa do que SELECT <columns> FROM <table>.
  3. Processamento extra (e/ou ainda mais IO) se algumas das colunas desnecessárias forem:
    • computed columns in databases that support them
    • in the case of selecting from a view, columns from a table/view that the query optimiser could otherwise optimise out
  4. O potencial de erros inesperados se as colunas forem adicionadas a tabelas e exibições posteriormente, resultando em nomes de coluna ambíguos. Por exemplo SELECT * FROM orders JOIN people ON people.id = orders.personid ORDER BY displayname - se uma coluna chamada displayname for adicionada à tabela de pedidos para permitir que os usuários deem nomes significativos aos seus pedidos para referência futura, então o nome da coluna aparecerá duas vezes na saída para que a cláusula ORDER BY seja ambígua, o que pode causar erros (“nome da coluna ambígua” nas versões recentes do MS SQL Server), e se não neste exemplo, o código do aplicativo pode começar a exibir a ordem name onde o nome da pessoa é pretendido porque a nova coluna é a primeira desse nome retornado e assim por diante.

Quando você pode usar *, tendo em mente o aviso acima? #

Embora seja melhor evitar no código de produção, usar * é bom como abreviação ao realizar consultas manuais no banco de dados para investigação ou trabalho de protótipo.

Às vezes, as decisões de design em seu aplicativo o tornam inevitável (em tais circunstâncias, prefira tablealias.* em vez de apenas * quando possível).

Ao usar EXISTS, como SELECT A.col1, A.Col2 FROM A WHERE EXISTS (SELECT * FROM B onde A.ID = B.A_ID), não estamos retornando nenhum dado de B. Assim, uma junção é desnecessário, e o mecanismo sabe que nenhum valor de B deve ser retornado, portanto, nenhum impacto no desempenho para usar *. Da mesma forma, COUNT(*) é bom, pois também não retorna nenhuma das colunas, portanto, só precisa ler e processar aquelas que são usadas para fins de filtragem.

SELECT usando aliases de coluna #

Os aliases de coluna são usados ​​principalmente para encurtar o código e tornar os nomes das colunas mais legíveis.

O código fica mais curto à medida que nomes de tabelas longos e identificação desnecessária de colunas (por exemplo, pode haver 2 IDs na tabela, mas apenas um é usado na instrução) podem ser evitados. Junto com aliases de tabela, isso permite que você use nomes descritivos mais longos em sua estrutura de banco de dados, mantendo as consultas sobre essa estrutura concisas.

Além disso, eles às vezes são necessários, por exemplo, em visualizações, para nomear as saídas computadas.

Todas as versões do SQL #

Os aliases podem ser criados em todas as versões do SQL usando aspas duplas (").

SELECT 
    FName AS "First Name", 
    MName AS "Middle Name",
    LName AS "Last Name"
FROM Employees

Diferentes versões do SQL #

Você pode usar aspas simples ('), aspas duplas (") e colchetes ([]) para criar um alias no Microsoft SQL Server.

SELECT 
    FName AS "First Name", 
    MName AS 'Middle Name',
    LName AS [Last Name]
FROM Employees

Ambos resultarão em:

Nome Nome do meio Sobrenome
Tiago João Smith
João Tiago Johnson
Miguel Marco Williams

Esta instrução retornará as colunas FName e LName com um determinado nome (um alias). Isso é feito usando o operador AS seguido pelo alias, ou simplesmente escrevendo alias diretamente após o nome da coluna. Isso significa que a consulta a seguir tem o mesmo resultado que a anterior.

SELECT 
    FName "First Name", 
    MName "Middle Name",
    LName "Last Name"
FROM Employees
Nome Nome do meio Sobrenome
Tiago João Smith
João Tiago Johnson
Miguel Marco Williams

No entanto, a versão explícita (ou seja, usando o operador AS) é mais legível.

Se o alias tiver uma única palavra que não seja uma palavra reservada, podemos escrevê-lo sem aspas simples, aspas duplas ou colchetes:

SELECT 
    FName AS FirstName, 
    LName AS LastName
FROM Employees
Nome Sobrenome
Tiago Smith
João Johnson
Miguel Williams

Uma outra variação disponível no MS SQL Server, entre outras, é <alias> = <column-or-calculation>, por exemplo:

SELECT FullName = FirstName + ' ' + LastName, 
       Addr1    = FullStreetAddress,
       Addr2    = TownName
FROM CustomerDetails

que é equivalente a:

SELECT FirstName + ' ' + LastName As FullName
       FullStreetAddress          As Addr1,
       TownName                   As Addr2
FROM CustomerDetails

Ambos resultarão em:

NomeCompleto Addr1 Addr2
James Smith 123 AnyStreet TownVille
John Johnson 668 Minha Estrada Qualquer cidade
Michael Williams 999 Dr. de ponta Williamsburgh

Alguns acham que usar = em vez de As é mais fácil de ler, embora muitos recomendem contra este formato, principalmente porque não é padrão e não é amplamente suportado por todos os bancos de dados. Pode causar confusão com outros usos do caractere =.

Todas as versões do SQL #

Além disso, se você precisar usar palavras reservadas, você pode usar colchetes ou aspas para escapar:

SELECT
    FName as "SELECT",
    MName as "FROM",
    LName as "WHERE"
FROM Employees

Diferentes versões do SQL #

Da mesma forma, você pode escapar palavras-chave no MSSQL com todas as abordagens diferentes:

SELECT 
    FName AS "SELECT", 
    MName AS 'FROM',
    LName AS [WHERE]
FROM Employees
SELECIONAR DE ONDE
Tiago João Smith
João Tiago Johnson
Miguel Marco Williams

Além disso, um alias de coluna pode ser usado em qualquer uma das cláusulas finais da mesma consulta, como um ORDER BY:

SELECT 
    FName AS FirstName, 
    LName AS LastName
FROM 
    Employees 
ORDER BY 
    LastName DESC

No entanto, você não pode usar

SELECT
    FName AS SELECT,
    LName AS FROM
FROM 
    Employees
ORDER BY 
    LastName DESC

Para criar um alias a partir dessas palavras reservadas (SELECT e FROM).

Isso causará vários erros na execução.

Selecione Colunas Individuais #

SELECT 
    PhoneNumber, 
    Email, 
    PreferredContact 
FROM Customers

Esta instrução retornará as colunas PhoneNumber, Email e PreferredContact de todas as linhas da tabela Customers. Além disso, as colunas serão retornadas na sequência em que aparecem na cláusula SELECT.

O resultado será:

TelefoneNúmero E-mail Contato preferencial
3347927472 william.jones@example.com TELEFONE
2137921892 dmiller@example.net O EMAIL
NULO richard0123@example.com O EMAIL

Se várias tabelas forem unidas, você poderá selecionar colunas de tabelas específicas especificando o nome da tabela antes do nome da coluna: [table_name].[column_name]

SELECT 
    Customers.PhoneNumber, 
    Customers.Email, 
    Customers.PreferredContact,
    Orders.Id AS OrderId
FROM 
    Customers
LEFT JOIN 
    Orders ON Orders.CustomerId = Customers.Id

*AS OrderId significa que o campo Id da tabela Orders será retornado como uma coluna chamada OrderId. Consulte [selecionando com alias de coluna](https://www.wikiod.com/pt/sql/selecionar#SELECT usando aliases de coluna) para obter mais informações.

Para evitar o uso de nomes de tabela longos, você pode usar aliases de tabela. Isso reduz a dificuldade de escrever nomes de tabela longos para cada campo selecionado nas junções. Se você estiver executando uma autojunção (uma junção entre duas instâncias da mesma tabela), deverá usar aliases de tabela para distinguir suas tabelas. Podemos escrever um alias de tabela como Clientes c ou Clientes AS c. Aqui c funciona como um alias para Customers e podemos selecionar, digamos Email assim: c.Email.

SELECT 
    c.PhoneNumber, 
    c.Email, 
    c.PreferredContact,
    o.Id AS OrderId
FROM 
    Customers c
LEFT JOIN 
    Orders o ON o.CustomerId = c.Id

Selecionando o número especificado de registros #

O padrão SQL 2008 define a cláusula FETCH FIRST para limitar o número de registros retornados.

SELECT Id, ProductName, UnitPrice, Package 
FROM Product 
ORDER BY UnitPrice DESC
FETCH FIRST 10 ROWS ONLY

Este padrão é suportado apenas em versões recentes de alguns RDMSs. A sintaxe não padrão específica do fornecedor é fornecida em outros sistemas. Progress OpenEdge 11.x também suporta a sintaxe FETCH FIRST <n> ROWS ONLY.

Além disso, OFFSET <m> ROWS antes de FETCH FIRST <n> ROWS ONLY permite pular linhas antes de buscar linhas.

SELECT Id, ProductName, UnitPrice, Package 
FROM Product 
ORDER BY UnitPrice DESC
OFFSET 5 ROWS
FETCH FIRST 10 ROWS ONLY

A seguinte consulta é suportada em SQL Server e MS Access:

SELECT TOP 10 Id, ProductName, UnitPrice, Package
FROM Product
ORDER BY UnitPrice DESC

Para fazer o mesmo em MySQL ou PostgreSQL a palavra-chave LIMIT deve ser usada:

SELECT Id, ProductName, UnitPrice, Package
FROM Product
ORDER BY UnitPrice DESC
LIMIT 10

No Oracle o mesmo pode ser feito com ROWNUM:

SELECT Id, ProductName, UnitPrice, Package
FROM Product
WHERE ROWNUM <= 10
ORDER BY UnitPrice DESC

Id    ProductName               UnitPrice             Package
38    Côte de Blaye             263.50                12 - 75 cl bottles
29    Thüringer Rostbratwurst   123.79                50 bags x 30 sausgs.
9    Mishi Kobe Niku            97.00                 18 - 500 g pkgs.
20    Sir Rodney's Marmalade    81.00                 30 gift boxes
18    Carnarvon Tigers          62.50                 16 kg pkg.
59    Raclette Courdavault      55.00                 5 kg pkg.

Nuances do fornecedor:

É importante notar que o TOP no Microsoft SQL opera após a cláusula WHERE e retornará o número especificado de resultados se existirem em qualquer lugar da tabela, enquanto o ROWNUM funciona como parte da cláusula WHERE se outras condições não existirem no número especificado de linhas no início da tabela, você obterá zero resultados quando houver outras a serem encontradas.

Selecionando com Condição #

A sintaxe básica da cláusula SELECT com WHERE é:

SELECT column1, column2, columnN
FROM table_name
WHERE [condition]

A [condição] pode ser qualquer expressão SQL, especificada usando comparação ou operadores lógicos como >, <, =, <>, >=, <=, LIKE, NOT, IN, BETWEEN etc.

A instrução a seguir retorna todas as colunas da tabela ‘Cars’ onde a coluna de status é ‘READY’:

SELECT * FROM Cars WHERE status = 'READY'

Consulte WHERE and HAVING para obter mais exemplos.

Selecionando com CASE #

Quando os resultados precisam ter alguma lógica aplicada ‘on the fly’, pode-se usar a instrução CASE para implementá-la.

SELECT CASE WHEN Col1 < 50 THEN 'under' ELSE 'over' END threshold
FROM TableName

também pode ser acorrentado

SELECT 
    CASE WHEN Col1 < 50 THEN 'under' 
         WHEN Col1 > 50 AND Col1 <100 THEN 'between' 
         ELSE 'over' 
    END threshold
FROM TableName

também se pode ter CASE dentro de outra instrução CASE

SELECT 
    CASE WHEN Col1 < 50 THEN 'under' 
         ELSE 
            CASE WHEN Col1 > 50 AND Col1 <100 THEN Col1 
            ELSE 'over' END 
    END threshold
FROM TableName

Selecione as colunas que são nomeadas com palavras-chave reservadas #

Quando um nome de coluna corresponde a uma palavra-chave reservada, o SQL padrão exige que você o coloque entre aspas duplas:

SELECT 
    "ORDER",
    ID 
FROM ORDERS

Observe que isso diferencia maiúsculas de minúsculas no nome da coluna.

Alguns SGBDs possuem formas proprietárias de citar nomes. Por exemplo, o SQL Server usa colchetes para essa finalidade:

SELECT 
    [Order],
    ID 
FROM ORDERS

enquanto o MySQL (e MariaDB) por padrão usam acentos graves:

SELECT 
    `Order`,
    id 
FROM orders

Selecionando com alias de tabela #

SELECT e.Fname, e.LName 
FROM Employees e

A tabela Employees recebe o alias ’e’ diretamente após o nome da tabela. Isso ajuda a remover a ambiguidade em cenários em que várias tabelas têm o mesmo nome de campo e você precisa especificar de qual tabela deseja retornar dados.

SELECT e.Fname, e.LName, m.Fname AS ManagerFirstName 
FROM Employees e 
    JOIN Managers m ON e.ManagerId = m.Id

Observe que, depois de definir um alias, você não poderá mais usar o nome da tabela canônica. ou seja,

SELECT e.Fname, Employees.LName, m.Fname AS ManagerFirstName 
FROM Employees e 
JOIN Managers m ON e.ManagerId = m.Id

lançaria um erro.

Vale a pena notar que os aliases de tabela – mais formalmente ‘variáveis ​​de intervalo’ – foram introduzidos na linguagem SQL para resolver o problema de colunas duplicadas causadas por INNER JOIN. O padrão SQL de 1992 corrigiu essa falha de design anterior introduzindo o NATURAL JOIN (implementado no mySQL, PostgreSQL e Oracle, mas ainda não no SQL Server), cujo resultado nunca tem nomes de coluna duplicados. O exemplo acima é interessante porque as tabelas são unidas em colunas com nomes diferentes (Id e ManagerId), mas não devem ser unidas nas colunas com o mesmo nome (LName, FName), exigindo a renomeação das colunas a serem executadas antes da junção:

SELECT Fname, LName, ManagerFirstName 
FROM Employees
     NATURAL JOIN
     ( SELECT Id AS ManagerId, Fname AS ManagerFirstName
       FROM Managers ) m;

Observe que, embora uma variável de alias/intervalo deva ser declarada para a tabela derivada (caso contrário, o SQL lançará um erro), nunca faz sentido usá-la na consulta.

Seleção com resultados ordenados #

SELECT * FROM Employees ORDER BY LName

Esta instrução retornará todas as colunas da tabela Employees.

ID FNome LNome Número de telefone
2 João Johnson 2468101214
1 Tiago Smith 1234567890
3 Miguel Williams 1357911131

1: https://www.wikiod.com/pt/sql/bancos-de-dados-e-tabelas-de-exemplo#Banco de dados da loja de automóveis

SELECT * FROM Employees ORDER BY LName DESC

Ou

 SELECT * FROM Employees ORDER BY LName ASC

Essa instrução altera a direção de classificação.

Pode-se também especificar várias colunas de classificação. Por exemplo:

SELECT * FROM Employees ORDER BY LName ASC, FName ASC

Este exemplo classificará os resultados primeiro por LName e, em seguida, para registros que tenham o mesmo LName, classificará por FName. Isso lhe dará um resultado semelhante ao que você encontraria em uma lista telefônica.

Para evitar redigitar o nome da coluna na cláusula ORDER BY, é possível usar o número da coluna. Observe que os números das colunas começam em 1.

SELECT Id, FName, LName, PhoneNumber FROM Employees ORDER BY 3

Você também pode incorporar uma instrução CASE na cláusula ORDER BY.

SELECT Id, FName, LName, PhoneNumber FROM Employees ORDER BY CASE WHEN LName='Jones` THEN 0 ELSE 1 END ASC

Isso classificará seus resultados para ter todos os registros com o LName de “Jones” no topo.

Selecionando com funções agregadas #

Média #

A função agregada AVG() retornará a média dos valores selecionados. #

SELECT AVG(Salary) FROM Employees

Funções agregadas também podem ser combinadas com a cláusula where. #

SELECT AVG(Salary) FROM Employees where DepartmentId = 1

Funções agregadas também podem ser combinadas com cláusula group by. #

Se o funcionário for categorizado com vários departamentos e quisermos encontrar o salário médio para cada departamento, podemos usar a seguinte consulta.

SELECT AVG(Salary) FROM Employees GROUP BY DepartmentId

Mínimo #

A função agregada MIN() retornará o mínimo de valores selecionados. #

SELECT MIN(Salary) FROM Employees

Máximo #

A função agregada MAX() retornará o máximo de valores selecionados. #

SELECT MAX(Salary) FROM Employees

Contar #

A função agregada COUNT() retornará a contagem de valores selecionados. #

SELECT Count(*) FROM Employees

Também pode ser combinado com where condições para obter a contagem de linhas que satisfazem condições específicas. #

SELECT Count(*) FROM Employees where ManagerId IS NOT NULL

Colunas específicas também podem ser especificadas para obter o número de valores na coluna. Observe que os valores NULL não são contados. #

Select Count(ManagerId) from Employees

Count também pode ser combinado com a palavra-chave distinct para uma contagem distinta. #

Select Count(DISTINCT DepartmentId) from Employees

Soma #

A função agregada SUM() retorna a soma dos valores selecionados para todas as linhas. #

SELECT SUM(Salary) FROM Employees

Selecionando sem Bloquear a tabela #

Às vezes, quando as tabelas são usadas principalmente (ou apenas) para leituras, a indexação não ajuda mais e cada bit conta, pode-se usar seleções sem LOCK para melhorar o desempenho.

servidor SQL

SELECT * FROM TableName WITH (nolock)

MySQL

SET SESSION TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;
SELECT * FROM TableName;
SET SESSION TRANSACTION ISOLATION LEVEL REPEATABLE READ;

Oráculo

SET TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;
SELECT * FROM TableName;

DB2

SELECT * FROM TableName WITH UR;

onde UR significa “leitura não confirmada”.

Se usado em uma tabela que tenha modificações de registro em andamento, pode ter resultados imprevisíveis.

Selecione com condição de vários valores da coluna #

SELECT * FROM Cars  WHERE status IN ( 'Waiting', 'Working' )

Isso é semanticamente equivalente a

SELECT * FROM Cars  WHERE ( status = 'Waiting' OR status = 'Working' )

ou seja, value IN ( <value list> ) é uma abreviação para disjunção (lógico OR).

Obter resultado agregado para grupos de linhas #

Contando linhas com base em um valor de coluna específico:

SELECT category, COUNT(*) AS item_count
FROM item
GROUP BY category;

Obtendo renda média por departamento:

SELECT department, AVG(income)
FROM employees
GROUP BY department;

O importante é selecionar apenas as colunas especificadas na cláusula GROUP BY ou usadas com [funções agregadas](http://stackoverflow .com/documentation/sql/222/select/6470/selecting-with-aggregate-functions).

A cláusula WHERE também pode ser usada com GROUP BY, mas WHERE filtra os registros antes de qualquer agrupamento ser feito:

SELECT department, AVG(income)
FROM employees
WHERE department <> 'ACCOUNTING'
GROUP BY department;

Se você precisar filtrar os resultados após o agrupamento ter sido feito, por exemplo, para ver apenas departamentos cuja renda média é maior que 1000, você precisa usar o [HAVING](https://www.wikiod.com/pt/docs/sql /636/filter-results-using-where-and-have):

SELECT department, AVG(income)
FROM employees
WHERE department <> 'ACCOUNTING'
GROUP BY department
HAVING avg(income) > 1000;

Selecione linhas de várias tabelas #

SELECT *
FROM
    table1,
    table2

SELECT
    table1.column1,
    table1.column2,
    table2.column1
FROM
    table1,
    table2

Isso é chamado de produto cruzado em SQL é o mesmo que produto cruzado em conjuntos

Essas instruções retornam as colunas selecionadas de várias tabelas em uma consulta.

Não há relação específica entre as colunas retornadas de cada tabela.

Selecionando com null #

SELECT Name FROM Customers WHERE PhoneNumber IS NULL

A seleção com nulos usa uma sintaxe diferente. Não use =, use IS NULL ou IS NOT NULL.

SELECT DISTINCT ContinentCode

Esta consulta retornará todos os valores DISTINCT (exclusivos, diferentes) da coluna ContinentCode da tabela Countries
