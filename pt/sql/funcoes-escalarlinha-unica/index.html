O SQL fornece várias funções escalares integradas. Cada função escalar recebe um valor como entrada e retorna um valor como saída para cada linha em um conjunto de resultados.

Você usa funções escalares sempre que uma expressão é permitida em uma instrução T-SQL.

Sintaxe #

  • CAST ( expressão AS data_type [ ( comprimento ) ] )
  • CONVERT ( data_type [ ( length ) ] , expression [ , style ] )
  • PARSE ( string_value AS data_type [ USING culture ] )
  • DATENAME ( datepart , date )
  • GETDATA()
  • DATEDIFF(datepart, startdate, enddate)
  • DATEADD (datepart , number , date )
  • ESCOLHER ( índice, val_1, val_2 [, val_n ] )
  • IIF (boolean_expression, true_value, false_value )
  • SIGN ( expressão_numérica )
  • POWER ( float_expression , y )

As funções escalares ou de linha única são usadas para operar cada linha de dados no conjunto de resultados, em oposição às funções agregadas que operam em todo o conjunto de resultados.

Existem dez tipos de funções escalares.

  1. As funções de configuração fornecem informações sobre a configuração da instância SQL atual.
  2. As funções de conversão convertem os dados no tipo de dados correto para uma determinada operação. Por exemplo, esses tipos de funções podem reformatar informações convertendo uma string em uma data ou número para permitir que dois tipos diferentes sejam comparados.
  3. As funções de data e hora manipulam campos contendo valores de data e hora. Eles podem retornar valores numéricos, de data ou de string. Por exemplo, você pode usar uma função para recuperar o dia da semana ou ano atual ou para recuperar apenas o ano da data.

Os valores retornados pelas funções de data e hora dependem da data e hora definidas para o sistema operacional do computador que executa a instância SQL. 4. Função lógica que realiza operações usando operadores lógicos. Ele avalia um conjunto de condições e retorna um único resultado. 5. As funções matemáticas realizam operações matemáticas, ou cálculos, em expressões numéricas. Esse tipo de função retorna um único valor numérico. 6. As funções de metadados recuperam informações sobre um banco de dados especificado, como seu nome e objetos de banco de dados. 7. As funções de segurança fornecem informações que podem ser usadas para gerenciar a segurança de um banco de dados, como informações sobre usuários e funções do banco de dados. 8. Funções de string executam operações em valores de string e retornam valores numéricos ou de string.

Usando funções de string, você pode, por exemplo, combinar dados, extrair uma substring, comparar strings ou converter uma string para todos os caracteres maiúsculos ou minúsculos. 9. As funções do sistema executam operações e retornam informações sobre valores, objetos e configurações para a instância SQL atual 10. As funções estatísticas do sistema fornecem várias estatísticas sobre a instância SQL atual – por exemplo, para que você possa monitorar os níveis de desempenho atuais do sistema.

Modificações de personagens #

Funções de modificação de caracteres incluem a conversão de caracteres para caracteres maiúsculos ou minúsculos, conversão de números para números formatados, manipulação de caracteres, etc.

A função lower(char) converte o parâmetro de caractere fornecido em caracteres minúsculos.

SELECT customer_id, lower(customer_last_name) FROM customer;

retornaria o sobrenome do cliente alterado de “SMITH” para “smith”.

Data e hora #

No SQL, você usa tipos de dados de data e hora para armazenar informações de calendário. Esses tipos de dados incluem hora, data, smalldatetime, datetime, datetime2 e datetimeoffset. Cada tipo de dados tem um formato específico.

Tipo de dados Formato
tempo hh:mm:ss[.nnnnnnn]
data AAAA-MM-DD
smalldatetime AAAA-MM-DD hh:mm:ss
datahora AAAA-MM-DD hh:mm:ss[.nnn]
datahora2 AAAA-MM-DD hh:mm:ss[.nnnnnnn]
deslocamento de data e hora AAAA-MM-DD hh:mm:ss[.nnnnnnn] [+/-]hh:mm

A função DATENAME retorna o nome ou valor de uma parte específica da data.

SELECT DATENAME (weekday,'2017-01-14') as Datename
nome do arquivo
Sábado

Você usa a função GETDATE para determinar a data e hora atuais do computador que está executando a instância SQL atual. Esta função não inclui a diferença de fuso horário.

SELECT GETDATE() as Systemdate
Data do sistema
14-01-2017 11:11:47.7230728

A função DATEDIFF retorna a diferença entre duas datas.

Na sintaxe, datepart é o parâmetro que especifica qual parte da data você deseja usar para calcular a diferença. A parte da data pode ser ano, mês, semana, dia, hora, minuto, segundo ou milissegundo. Em seguida, você especifica a data de início no parâmetro startdate e a data de término no parâmetro enddate para a qual deseja encontrar a diferença.

SELECT SalesOrderID, DATEDIFF(day, OrderDate, ShipDate) 
AS 'Processing time'
FROM Sales.SalesOrderHeader
SalesOrderID Tempo de processamento
43659 7
43660 7
43661 7
43662 7

A função DATEADD permite adicionar um intervalo a parte de uma data específica.

SELECT DATEADD (day, 20, '2017-01-14') AS Added20MoreDays
Adicionado20MaisDias
03-02-2017 00:00:00.000

Função de configuração e conversão #

Um exemplo de uma função de configuração em SQL é a função @@SERVERNAME. Essa função fornece o nome do servidor local que está executando o SQL.

SELECT @@SERVERNAME AS 'Server'
Servidor
SQL064

No SQL, a maioria das conversões de dados ocorre de forma implícita, sem qualquer intervenção do usuário.

Para realizar quaisquer conversões que não possam ser concluídas implicitamente, você pode usar as funções CAST ou CONVERT.

A sintaxe da função CAST é mais simples que a sintaxe da função CONVERT, mas é limitada no que pode fazer.

Aqui, usamos as funções CAST e CONVERT para converter o tipo de dados datetime para o tipo de dados varchar.

A função CAST sempre usa a configuração de estilo padrão. Por exemplo, ele representará datas e horas usando o formato AAAA-MM-DD.

A função CONVERT usa o estilo de data e hora que você especificar. Neste caso, 3 especifica o formato de data dd/mm/aa.

USE AdventureWorks2012
GO
SELECT FirstName + ' ' + LastName + ' was hired on ' +
       CAST(HireDate AS varchar(20)) AS 'Cast',
       FirstName + ' ' + LastName + ' was hired on ' +
       CONVERT(varchar, HireDate, 3) AS 'Convert'
FROM Person.Person AS p
JOIN HumanResources.Employee AS e
ON p.BusinessEntityID = e.BusinessEntityID
GO
Elenco Converter
David Hamiltion foi contratado em 04/02/2003 David Hamiltion foi contratado em 02/04/03

Outro exemplo de uma função de conversão é a função PARSE. Esta função converte uma string em um tipo de dados especificado.

Na sintaxe da função, você especifica a string que deve ser convertida, a palavra-chave AS e, em seguida, o tipo de dados necessário. Opcionalmente, você também pode especificar a cultura na qual o valor da string deve ser formatado. Se você não especificar isso, o idioma da sessão será usado.

Se o valor da string não puder ser convertido em um formato numérico, de data ou hora, isso resultará em um erro. Você precisará então usar CAST ou CONVERT para a conversão.

SELECT PARSE('Monday, 13 August 2012' AS datetime2 USING 'en-US') AS 'Date in English'
Data em inglês
2012-08-13 00:00:00.0000000

Função lógica e matemática #

SQL tem duas funções lógicas – CHOOSE e IIF. #

A função CHOOSE retorna um item de uma lista de valores, com base em sua posição na lista. Esta posição é especificada pelo índice.

Na sintaxe, o parâmetro index especifica o item e é um número inteiro ou inteiro. O parâmetro val_1 … val_n identifica a lista de valores.

SELECT CHOOSE(2, 'Human Resources', 'Sales', 'Admin', 'Marketing' ) AS Result;
Resultado
Vendas

Neste exemplo, você usa a função CHOOSE para retornar a segunda entrada em uma lista de departamentos.

A função IIF retorna um de dois valores, com base em uma condição específica. Se a condição for verdadeira, ela retornará o valor verdadeiro. Caso contrário, retornará um valor falso.

Na sintaxe, o parâmetro boolean_expression especifica a expressão booleana. O parâmetro true_value especifica o valor que deve ser retornado se a boolean_expression for avaliada como true e o parâmetro false_value especifica o valor que deve ser retornado se a boolean_expression for avaliada como false.

SELECT BusinessEntityID, SalesYTD, 
       IIF(SalesYTD > 200000, 'Bonus', 'No Bonus') AS 'Bonus?'
FROM Sales.SalesPerson
GO
BusinessEntityID VendasYTD Bônus?
274 559697.5639 Bônus
275 3763178.1787 Bônus
285 172524.4512 Sem Bônus
Neste exemplo, você usa a função IIF para retornar um dos dois valores. Se as vendas acumuladas do ano de um vendedor estiverem acima de 200.000, essa pessoa será elegível para um bônus. Valores abaixo de 200.000 significam que os funcionários não se qualificam para bônus.

SQL inclui várias funções matemáticas que você pode usar para realizar cálculos em valores de entrada e retornar resultados numéricos. #

Um exemplo é a função SIGN, que retorna um valor indicando o sinal de uma expressão. O valor de -1 indica uma expressão negativa, o valor de +1 indica uma expressão positiva e 0 indica zero.

SELECT SIGN(-20) AS 'Sign'
Assinar
-1

No exemplo, a entrada é um número negativo, portanto, o painel Resultados lista o resultado -1.

Outra função matemática é a função POWER. Esta função fornece o valor de uma expressão elevada a uma potência especificada.

Na sintaxe, o parâmetro float_expression especifica a expressão e o parâmetro y especifica a potência à qual você deseja aumentar a expressão.

SELECT POWER(50, 3) AS Result
Resultado
125000
