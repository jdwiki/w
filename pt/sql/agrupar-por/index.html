Os resultados de uma consulta SELECT podem ser agrupados por uma ou mais colunas usando a instrução GROUP BY: todos os resultados com o mesmo valor nas colunas agrupadas são agregados. Isso gera uma tabela de resultados parciais, em vez de um resultado. GROUP BY pode ser usado em conjunto com funções de agregação usando a instrução HAVING para definir como as colunas não agrupadas são agregadas.

Sintaxe #

  • AGRUPAR POR {       expressão-coluna     | ROLLUP ( <group_by_expression> [ ,…n ] )     | CUBO ( <grupo_por_expressão> [ ,…n ] )     | AGRUPANDO CONJUNTOS ( [ ,…n ] )     | () –calcula o total geral } [ ,…n ]

  • <group_by_expression> ::=       expressão-coluna     | ( expressão-coluna [ ,…n ] )

  • <conjunto_grupo> ::=       () –calcula o total geral     | <grouping_set_item>     | ( <grouping_set_item> [ ,…n ] )

  • <grouping_set_item> ::=       <group_by_expression>     | ROLLUP ( <group_by_expression> [ ,…n ] )     | CUBO ( <grupo_por_expressão> [ ,…n ] )

Exemplo básico de GROUP BY #

Pode ser mais fácil se você pensar em GROUP BY como “para cada” por uma questão de explicação. A consulta abaixo:

SELECT EmpID, SUM (MonthlySalary) 
FROM Employee
GROUP BY EmpID

está dizendo:

“Dê-me a soma de MonthlySalary para cada EmpID”

Então, se sua tabela ficou assim:

+-----+-------------+
|EmpID|MonthlySalary|
+-----+-------------+
|1    |200          |
+-----+-------------+
|2    |300          |
+-----+-------------+

Resultado:

+-+---+
|1|200|
+-+---+
|2|300|
+-+---+

A soma não parece fazer nada porque a soma de um número é esse número. Por outro lado, se ficou assim:

+-----+-------------+
|EmpID|MonthlySalary|
+-----+-------------+
|1    |200          |
+-----+-------------+
|1    |300          |
+-----+-------------+
|2    |300          |
+-----+-------------+

Resultado:

+-+---+
|1|500|
+-+---+
|2|300|
+-+---+

Então seria porque existem dois EmpID 1 para somar.

USE GROUP BY para COUNT o número de linhas para cada entrada exclusiva em uma determinada coluna #

Digamos que você queira gerar contagens ou subtotais para um determinado valor em uma coluna.

Dada esta tabela, “Westerosians”:

Nome GreatHouseAllegience
Aria Austero
Brincos Lannister
Myrcella Lannister
Yara Greyjoy
Catelyn Austero
Sansa Austero

Sem GROUP BY, COUNT simplesmente retornará um número total de linhas:

SELECT Count(*) Number_of_Westerosians
FROM Westerosians

retorna…

Number_of_Westerosians
6

Mas adicionando GROUP BY, podemos COUNT os usuários para cada valor em uma determinada coluna, para retornar o número de pessoas em uma determinada Great House, digamos:

SELECT GreatHouseAllegience House, Count(*) Number_of_Westerosians
FROM Westerosians
GROUP BY GreatHouseAllegience

retorna…

Casa Number_of_Westerosians
Austero 3
Greyjoy 1
Lannister 2

É comum combinar GROUP BY com ORDER BY para classificar os resultados por categoria maior ou menor:

SELECT GreatHouseAllegience House, Count(*) Number_of_Westerosians
FROM Westerosians
GROUP BY GreatHouseAllegience
ORDER BY Number_of_Westerosians Desc

retorna…

Casa Number_of_Westerosians
Austero 3
Lannister 2
Greyjoy 1

Filtre os resultados do GROUP BY usando uma cláusula HAVING #

Uma cláusula HAVING filtra os resultados de uma expressão GROUP BY. Observação: os exemplos a seguir estão usando o banco de dados de exemplo [Biblioteca][1].

Exemplos:

Retorna todos os autores que escreveram mais de um livro ([exemplo ao vivo][3]).

SELECT
  a.Id,
  a.Name,
  COUNT(*) BooksWritten
FROM BooksAuthors ba
  INNER JOIN Authors a ON a.id = ba.authorid
GROUP BY
  a.Id,
  a.Name
HAVING COUNT(*) > 1    -- equals to HAVING BooksWritten > 1
;

Devolva todos os livros que tenham mais de três autores ([exemplo ao vivo][4]).

SELECT
  b.Id,
  b.Title,
  COUNT(*) NumberOfAuthors
FROM BooksAuthors ba
  INNER JOIN Books b ON b.id = ba.bookid
GROUP BY
  b.Id,
  b.Title
HAVING COUNT(*) > 3    -- equals to HAVING NumberOfAuthors > 3
;

[1]: https://www.wikiod.com/pt/sql/bancos-de-dados-e-tabelas-de-exemplo#Banco de dados da biblioteca [3]: http://sqlfiddle.com/#!9/7c06f/7 [4]: http://sqlfiddle.com/#!9/7c06f/9

Agregação ROLAP (Data Mining) #

Descrição #

O padrão SQL fornece dois operadores agregados adicionais. Estes usam o valor polimórfico “ALL” para denotar o conjunto de todos os valores que um atributo pode assumir. Os dois operadores são:

  • with data cube que fornece todas as combinações possíveis além dos atributos de argumento da cláusula.
  • com roll up que fornece os agregados obtidos considerando os atributos em ordem da esquerda para a direita comparados como eles são listados no argumento da cláusula.

Versões padrão do SQL que suportam esses recursos: 1999,2003,2006,2008,2011.

Exemplos #

Considere esta tabela:

Comida Marca Total_quantia
Colar Marca1 100
Massas Marca2 250
Pizza Marca2 300

Com cubo #

select Food,Brand,Total_amount
from Table
group by Food,Brand,Total_amount with cube
Comida Marca Total_quantia
Colar Marca1 100
Massas Marca2 250
Massas TODOS 350
Pizza Marca2 300
Pizza TODOS 300
TODOS Marca1 100
TODOS Marca2 550
TODOS TODOS 650

Com enrolar #

select Food,Brand,Total_amount
from Table
group by Food,Brand,Total_amount with roll up
Comida Marca Total_quantia
Colar Marca1 100
Massas Marca2 250
Pizza Marca2 300
Massas TODOS 350
Pizza TODOS 300
TODOS TODOS 650
