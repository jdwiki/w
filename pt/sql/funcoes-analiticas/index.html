Você usa funções analíticas para determinar valores com base em grupos de valores. Por exemplo, você pode usar esse tipo de função para determinar totais, porcentagens ou o resultado principal em um grupo.

Sintaxe #

  1. FIRST_VALUE ( expressão_escalar ) OVER ( [ partition_by_clause ] order_by_clause [ rows_range_clause ] )
  2. LAST_VALUE ( expressão_escalar ) OVER ( [ partition_by_clause ] order_by_clause [ rows_range_clause ] )
  3. LAG (expressão_escalar [,deslocamento] [,padrão]) OVER ( [ partition_by_clause ] order_by_clause )
  4. LEAD ( expressão_escalar [ ,deslocamento ] , [padrão ]) OVER ( [ partition_by_clause ] order_by_clause )
  5. PERCENT_RANK( ) OVER ( [ partition_by_clause ] order_by_clause )
  6. CUME_DIST( ) OVER ( [ partition_by_clause ] order_by_clause )
  7. PERCENTILE_DISC ( numeric_literal ) WITHIN GROUP ( ORDER BY order_by_expression [ ASC | DESC ] ) OVER ( [ <partition_by_clause> ] )
  8. PERCENTILE_CONT ( numeric_literal ) WITHIN GROUP ( ORDER BY order_by_expression [ ASC | DESC ] ) OVER ( [ <partition_by_clause> ] )

LAG e LEAD #

A função LAG fornece dados em linhas antes da linha atual no mesmo conjunto de resultados. Por exemplo, em uma instrução SELECT, você pode comparar valores na linha atual com valores em uma linha anterior.

Você usa uma expressão escalar para especificar os valores que devem ser comparados. O parâmetro offset é o número de linhas antes da linha atual que será usada na comparação. Se você não especificar o número de linhas, o valor padrão de uma linha será usado.

O parâmetro default especifica o valor que deve ser retornado quando a expressão em offset tiver um valor NULL. Se você não especificar um valor, um valor de NULL será retornado.

A função LEAD fornece dados em linhas após a linha atual no conjunto de linhas. Por exemplo, em uma instrução SELECT, você pode comparar valores na linha atual com valores na linha seguinte.

Você especifica os valores que devem ser comparados usando uma expressão escalar. O parâmetro offset é o número de linhas após a linha atual que será usada na comparação.

Você especifica o valor que deve ser retornado quando a expressão no deslocamento tiver um valor NULL usando o parâmetro padrão. Se você não especificar esses parâmetros, o padrão de uma linha será usado e um valor de NULL será retornado.

SELECT BusinessEntityID, SalesYTD,
       LEAD(SalesYTD, 1, 0) OVER(ORDER BY BusinessEntityID) AS "Lead value",
       LAG(SalesYTD, 1, 0) OVER(ORDER BY BusinessEntityID) AS "Lag value"
FROM SalesPerson;

Este exemplo usa as funções LEAD e LAG para comparar os valores de vendas de cada funcionário até o momento com os dos funcionários listados acima e abaixo, com registros ordenados com base na coluna BusinessEntityID.

PERCENTILE_DISC e PERCENTILE_CONT #

A função PERCENTILE_DISC lista o valor da primeira entrada onde a distribuição cumulativa é maior que o percentil que você fornece usando o parâmetro numeric_literal.

Os valores são agrupados por conjunto de linhas ou partição, conforme especificado pela cláusula WITHIN GROUP.

A função PERCENTILE_CONT é semelhante à função PERCENTILE_DISC, mas retorna a média da soma da primeira entrada correspondente e da próxima entrada.

SELECT BusinessEntityID, JobTitle, SickLeaveHours,
       CUME_DIST() OVER(PARTITION BY JobTitle ORDER BY SickLeaveHours ASC)
       AS "Cumulative Distribution",
       PERCENTILE_DISC(0.5) WITHIN GROUP(ORDER BY SickLeaveHours)
          OVER(PARTITION BY JobTitle) AS "Percentile Discreet"
FROM Employee;

Para encontrar o valor exato da linha que corresponde ou excede o percentil 0,5, você passa o percentil como o literal numérico na função PERCENTILE_DISC. A coluna Percentil Discreto em um conjunto de resultados lista o valor da linha na qual a distribuição cumulativa é maior que o percentil especificado.

BusinessEntityID JobTitle SickLeaveHours Distribuição Cumulativa Percentil Discreto
272 Especialista em Aplicação 55 0,25 56
268 Especialista em Aplicação 56 0,75 56
269 Especialista em Aplicação 56 0,75 56
267 Especialista em Aplicação 57 1 56

Para basear o cálculo em um conjunto de valores, você usa a função PERCENTILE_CONT. A coluna “Percentil Contínuo” nos resultados lista o valor médio da soma do valor do resultado e o próximo valor correspondente mais alto.

SELECT BusinessEntityID, JobTitle, SickLeaveHours,
       CUME_DIST() OVER(PARTITION BY JobTitle ORDER BY SickLeaveHours ASC)
       AS "Cumulative Distribution",
       PERCENTILE_DISC(0.5) WITHIN GROUP(ORDER BY SickLeaveHours) 
          OVER(PARTITION BY JobTitle) AS "Percentile Discreet",
       PERCENTILE_CONT(0.5) WITHIN GROUP(ORDER BY SickLeaveHours) 
          OVER(PARTITION BY JobTitle) AS "Percentile Continuous"
FROM Employee;

FIRST_VALUE #

Você usa a função FIRST_VALUE para determinar o primeiro valor em um conjunto de resultados ordenado, que você identifica usando uma expressão escalar.

SELECT StateProvinceID, Name, TaxRate,
       FIRST_VALUE(StateProvinceID)
        OVER(ORDER BY TaxRate ASC) AS FirstValue
FROM SalesTaxRate;

Neste exemplo, a função FIRST_VALUE é usada para retornar o ID do estado ou província com a taxa de imposto mais baixa. A cláusula OVER é usada para ordenar as taxas de imposto para obter a taxa mais baixa.

LAST_VALUE #

A função LAST_VALUE fornece o último valor em um conjunto de resultados ordenado, que você especifica usando uma expressão escalar.

SELECT TerritoryID, StartDate, BusinessentityID,
       LAST_VALUE(BusinessentityID) 
        OVER(ORDER BY TerritoryID) AS LastValue
FROM SalesTerritoryHistory;

Este exemplo usa a função LAST_VALUE para retornar o último valor de cada conjunto de linhas nos valores ordenados.

PERCENT_RANK e CUME_DIST #

A função PERCENT_RANK calcula a classificação de uma linha em relação ao conjunto de linhas. A porcentagem é baseada no número de linhas no grupo que possuem um valor menor que a linha atual.

O primeiro valor no conjunto de resultados sempre tem uma classificação percentual de zero. O valor para o valor mais alto – ou último – no conjunto é sempre um.

A função CUME_DIST calcula a posição relativa de um valor especificado em um grupo de valores, determinando a porcentagem de valores menores ou iguais a esse valor. Isso é chamado de distribuição cumulativa.

SELECT BusinessEntityID, JobTitle, SickLeaveHours,
PERCENT_RANK() OVER(PARTITION BY JobTitle ORDER BY SickLeaveHours DESC)
       AS "Percent Rank",
CUME_DIST() OVER(PARTITION BY JobTitle ORDER BY SickLeaveHours DESC)
       AS "Cumulative Distribution"
FROM Employee;

Neste exemplo, você usa uma cláusula ORDER para particionar – ou agrupar – as linhas recuperadas pela instrução SELECT com base nos cargos dos funcionários, com os resultados em cada grupo classificados com base no número de horas de licença médica que os funcionários usou.

A função PERCENT_RANK classifica as entradas dentro de cada grupo. Para cada entrada, ele retorna a porcentagem de entradas no mesmo grupo que possuem valores menores.

A função CUME_DIST é semelhante, exceto que ela retorna a porcentagem de valores menores ou iguais ao valor atual.
