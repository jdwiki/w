Páginas do GitHub #

O GitHub oferece hospedagem ilimitada para usuários ou organizações e site do projeto. Ambos Jekyll e arquivos estáticos estão disponíveis.

Aqui estão os passos para hospedar seu blog Jekyll no Github.

Configurar #

Site de usuários ou organizações #

  1. Crie um repositório chamado username.github.io, onde username é seu nome de usuário (ou nome da organização) no GitHub.
  2. Clone o repositório em seu computador: 
    $ git clone https://github.com/username/username.github.io
  3. Entre na pasta do projeto, bootstrap, crie e depure seu site: 
    $ cd username.github.io
$ bundle install
$ bundle exec jekyll serve
  4. Confirme e envie o repositório: 
    $ git add --all
$ git commit -m "Initial commit"
$ git push -u origin master

Agora você deve poder acessar username.github.io para ver seu blog.

Projetar site #

O site do projeto pode ser habilitado em todos os repositórios, incluindo repositórios privados.

  1. Ative o site do projeto.

    Go to Settings-GitHub Pages-Sources, choose a source to switch on GitHub Pages for the repository.

  2. Construir site

    You may build a Jekyll site from scratch or use Theme Chooser to find a theme for your project site.

  3. Edite o conteúdo

  4. Comprometa-se

Agora você deve poder acessar username.github.io/your-project para ver o site do seu projeto.

Domínios personalizados #

  1. Abra Configurações->GitHub Pages->Domínio personalizado, adicione seu domínio personalizado.
  2. Crie um arquivo CNAME: 
    $ cd username.github.io
$ echo "example.com" > CNAME
  3. Comprometa-se e empurre 
    $ git commit -m "Add CNAME" CNAME
$ git push -u origin master

Restrições #

Plugins #

O Jekyll possui um sistema de plugins com ganchos que permitem criar conteúdo gerado personalizado específico para o seu site. No entanto, o GitHub Pages permite apenas uma lista branca de plugins por motivos de segurança.

Aqui está a lista branca:

  • Mapa do site Jekyll
  • Jekyll SEO Tag
  • github-metadados
  • Jekyll Feed
  • Redirecionamento Jekyll de
  • Yemojis
  • Menções Jekyll

Para evitar a inconsistência com o GitHub Pages, você pode usar --safe para servir em local.

Você ainda pode usar todos os plugins publicando seu site gerado no GitHub Pages, convertendo o site localmente e enviando os arquivos estáticos gerados para o seu repositório GitHub em vez dos arquivos de origem Jekyll.

Mecanismo de redução #

Desde 01/05/2016, o GitHub Pages suporta apenas kramdown como mecanismo Markdown.

Consulte https://github.com/blog/2100-github-pages-now-faster-and-simpler-with-jekyll-3-0 para obter mais detalhes.

Fontes #

O GitHub permite que você defina as fontes do Jekyll para o branch master, branch gh-pages ou pasta /docs no branch master.

Um tutorial completo está disponível em https://pages.github.com/

Máquina local #

Para fins de teste, você pode hospedar seu blog em sua máquina local. Depois de configurar e fazer qualquer alteração, o Jekyll pode servir o blog para http://localhost:4000. Na linha de comando no diretório raiz do projeto, execute:

$ bundle exec jekyll serve

A parte bundle exec é opcional, mas se você usar Bundler, isso garante que as dependências de gem estejam atualizadas. Para um [loop Edit-Build-Test] mais rápido (http://www.joelonsoftware.com/articles/fog0000000023.html), use a opção --draft para construir artigos do diretório _drafts:

$ bundle exec jekyll serve --draft --detach

Usar --detach coloca o processo em segundo plano para que o prompt de comando possa ser usado para outra coisa.

(A partir da versão 2.4, a opção --watch está habilitada por padrão. Se, por acaso, você tiver uma versão mais antiga, precisará adicionar essa opção para que as alterações sejam monitoradas.)

Você também pode definir o diretório de destino para um diretório em um servidor web diferente, como Apache, nginx ou Lighttpd:

$ jekyll build --destination /path/to/web_server/root

Hospedagem CloudCannon (e CMS) #

CloudCannon oferece hospedagem e um CMS para aplicativos Jekyll. Aqui estão as etapas para hospedar seu aplicativo Jekyll no CloudCannon (http://cloudcannon.com).

Sem controle de versão:

  • Crie seu blog localmente usando alguns arquivos locais e jekyll serve
  • Crie uma conta CloudCannon e crie um novo site
  • Arraste seu site para o ‘Navegador de arquivos’ no CloudCannon

Com controle de versão:

  • Crie um repositório no Github ou Bitbucket
  • Crie seu blog localmente usando alguns arquivos locais e jekyll serve
  • Crie uma conta CloudCannon e crie um novo site
  • Conecte seu repositório Github ou Bitbucket

Observe que o CloudCannon não é totalmente gratuito. No plano gratuito, você pode usar o CloudCannon como um CMS gráfico, mas precisará de hospedagem externa. No plano pago a hospedagem também está inclusa.
