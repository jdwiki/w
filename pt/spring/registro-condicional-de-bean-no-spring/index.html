Registro condicional de bean no Spring

  • Registra beans apenas quando uma propriedade ou valor é especificado
  • Anotações de condição

  • Registra beans apenas quando uma propriedade ou valor é especificado
  • Anotações de condição

Ponto importante a ser observado ao usar a condição

  • A classe de condição é referida como classe direta (não como bean de mola) então ela não pode usar a injeção de propriedade @Value, ou seja, nenhum outro feijão de mola pode ser injetado dentro dela.
  • De documentos java - As condições devem seguir as mesmas restrições que BeanFactoryPostProcessor e tomar cuidado para nunca interagir com instâncias de bean. As restrições mencionadas aqui são Um BeanFactoryPostProcessor pode interagir e modificar definições de bean, mas nunca instâncias de bean. Fazer isso pode causar a instanciação prematura do bean, violando o contêiner e causando efeitos colaterais indesejados.

Registra beans apenas quando uma propriedade ou valor é especificado #

Um bean de mola pode ser configurado de forma que ele registre somente se tiver um valor específico ou uma propriedade especificada for atendida. Para fazer isso, implemente [Condition.matches][1] para verificar a propriedade/valor:

public class PropertyCondition implements Condition {
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        return context.getEnvironment().getProperty("propertyName") != null;
        // optionally check the property value
    }
}

Na configuração Java, use a implementação acima como condição para registrar o bean. Observe o uso da anotação [@Conditional][2].

@Configuration
public class MyAppConfig {

    @Bean
    @Conditional(PropertyCondition.class)
    public MyBean myBean() {
      return new MyBean();
    }
}

Em PropertyCondition, qualquer quantidade de condições pode ser avaliada. No entanto, é aconselhável separar a implementação para cada condição para mantê-las fracamente acopladas. Por exemplo:

@Configuration
public class MyAppConfig {

    @Bean
    @Conditional({PropertyCondition.class, SomeOtherCondition.class})
    public MyBean myBean() {
      return new MyBean();
    }
}

[1]: http://docs.spring.io/spring/docs/current/javadoc-api/org/springframework/context/annotation/Condition.html#matches-org.springframework.context.annotation.ConditionContext-org. springframework.core.type.AnnotatedTypeMetadata- [2]: http://docs.spring.io/spring/docs/current/javadoc-api/org/springframework/context/annotation/Conditional.html

Anotações de condição #

Exceto a anotação principal @conditional, há um conjunto de anotações semelhantes para serem usadas em diferentes casos.

Condições de classe #

As anotações @ConditionalOnClass e @ConditionalOnMissingClass permitem que a configuração seja incluída com base na presença ou ausência de classes específicas.

Por exemplo. quando OObjectDatabaseTx.class é adicionado às dependências e não há bean OrientWebConfigurer criamos o configurador.

@Bean
@ConditionalOnWebApplication
@ConditionalOnClass(OObjectDatabaseTx.class)
@ConditionalOnMissingBean(OrientWebConfigurer.class)
public OrientWebConfigurer orientWebConfigurer() {
    return new OrientWebConfigurer();
}

Condições do feijão #

As anotações @ConditionalOnBean e @ConditionalOnMissingBean permitem que um bean seja incluído com base na presença ou ausência de beans específicos. Você pode usar o atributo value para especificar beans por tipo ou name para especificar beans por nome. O atributo search permite limitar a hierarquia ApplicationContext que deve ser considerada ao pesquisar beans.

Veja o exemplo acima quando verificamos se não há bean definido.

Condições da propriedade #

A anotação @ConditionalOnProperty permite que a configuração seja incluída com base em uma propriedade Spring Environment. Use os atributos prefix e name para especificar a propriedade que deve ser verificada. Por padrão, qualquer propriedade que exista e não seja igual a false será correspondida. Você também pode criar verificações mais avançadas usando os atributos haveValue e matchIfMissing.

@ConditionalOnProperty(value='somebean.enabled', matchIfMissing = true, havingValue="yes")
@Bean 
public SomeBean someBean(){
}

Condições do recurso #

A anotação @ConditionalOnResource permite que a configuração seja incluída somente quando um recurso específico estiver presente.

@ConditionalOnResource(resources = "classpath:init-db.sql")

Condições do aplicativo da web #

As anotações @ConditionalOnWebApplication e @ConditionalOnNotWebApplication permitem que a configuração seja incluída dependendo se o aplicativo é um ‘aplicativo web’.

@Configuration
@ConditionalOnWebApplication
public class MyWebMvcAutoConfiguration {...}

Condições de expressão SpEL #

A anotação @ConditionalOnExpression permite que a configuração seja incluída com base no resultado de uma expressão SpEL.

@ConditionalOnExpression("${rest.security.enabled}==false")
