Novo projeto com o cliente de API do Cloud Resource Manager para .NET

  • Começando

Usaremos Bibliotecas de cliente de API do Google para .NET para este exemplo.

Existem outras bibliotecas. Consulte o Google Client Libraries Explained para obter detalhes.

Usaremos a API Cloud Resource Manager para [Criar e gerenciar projetos](https://cloud.google.com/resource-manager/docs/ criar-gerenciar-projetos).

Vamos começar.

Juntando tudo…

Você deve ter dois arquivos. O primeiro arquivo é chamado de packages.config ou project.json.

Vamos nomear o segundo arquivo como Program.cs. Todo o código que foi incluído nas seções acima pode ser colado em um único método principal:

using System;
using System.Collections.Generic;
using System.Threading.Tasks;

using Google.Apis.Auth.OAuth2;
using Google.Apis.CloudResourceManager.v1;
using Google.Apis.Services;

using Data = Google.Apis.CloudResourceManager.v1.Data;

namespace OurFirstProject
{
    public class Program
    {
        private const string projectId = [YOUR-PROJECT-ID];
        private const string applicationName = "Test";

        public static void Main(string[] args)
        {

            var scopes = new String[] {
                CloudResourceManagerService.Scope.CloudPlatform
            };

            GoogleCredential credential = Task.Run(
                () => GoogleCredential.GetApplicationDefaultAsync()
            ).Result;

            if (credential.IsCreateScopedRequired)
            {
                credential = credential.CreateScoped(scopes);
            }

            CloudResourceManagerService service = new CloudResourceManagerService(
                new BaseClientService.Initializer()
                {
                    HttpClientInitializer = credential,
                    ApplicationName = applicationName
                }
            );

            Console.WriteLine("1. Create Project");
            Data.Operation operation1 = service.Projects.Create(
                new Data.Project()
                {
                    ProjectId = projectId,
                }
            ).Execute();

            Console.Write("2. Awaiting Operation Completion");
            Data.Operation operation2;
            do
            {
                operation2 = service.Operations.Get(operation1.Name).Execute();
                Console.WriteLine(operation2.Done.ToString());
                System.Threading.Thread.Sleep(1000);
            } while (operation2.Done != true);

            Console.WriteLine();
            Console.WriteLine("Enter to continue");
            Console.ReadLine();

            Console.WriteLine("3. Deleting Project");
            var operation3 = service.Projects.Delete(projectId).Execute();
        }
    }
}

Se você estiver usando o Windows e o Visual Studio, “Iniciar”

Se você estiver usando Linux, primeiro restaure os pacotes e execute o aplicativo

dotnet restore
dotnet run

Começando #

O código funcionará no Windows e no Linux.

Ele é executado em .NET Core ou .NET Standard. A única diferença é que, se você deseja rodar no .NET Core, você deve usar project.json. Se você deseja rodar no .NET Standard, você deve usar packages.config.

A API Client Library for Cloud Resource Manager API está disponível no nuget:

https://www.nuget.org/packages/Google.Apis.CloudResourceManager.v1/

A versão escrita é: 1.22.0.809

Faremos isso de 2 maneiras:

  • Windows com .NET Standard; e
  • Linux com .NET Core.

Janelas

Se você estiver usando o Visual Studio, crie um novo “Aplicativo de console” “Visual C#”. Caso contrário, crie um diretório para o projeto e crie um arquivo nele chamado packages.config. packages.config é para .NET Standard, mas estamos usando .NET Standard com Windows. Você só pode executar o .NET Standard no Windows. Substitua o conteúdo de packages.config por:

﻿<?xml version="1.0" encoding="utf-8"?>
<packages>
  <package id="Google.Apis" version="1.22.0"
    targetFramework="net452" />
  <package id="Google.Apis.Auth" version="1.22.0"
    targetFramework="net452" />
  <package id="Google.Apis.CloudResourceManager.v1" version="1.22.0.809" 
    targetFramework="net452" />
  <package id="Google.Apis.Core" version="1.22.0"
    targetFramework="net452" />
</packages>

Linux

Crie um diretório para o projeto e crie um arquivo nele chamado project.json. project.json é para .NET Core, mas estamos usando .NET Core com Linux neste exemplo. Você pode executar o .NET Core no Linux ou no Windows. Substitua o conteúdo de project.json por:

{
  "version": "1.0.0-*",
  "buildOptions": {
    "debugType": "portable",
    "emitEntryPoint": true
  },
  "dependencies": {},
  "frameworks": {
    "netcoreapp1.1": {
      "dependencies": {
        "Microsoft.NETCore.App": {
          "type": "platform",
          "version": "1.1.0"
        },
        "Google.Apis.CloudResourceManager.v1":"1.22.0.809"
      },
      "imports": "dnxcore50"
    }
  }
}

Credenciais padrão do aplicativo #

Não vou repetir tudo aqui: “Application Default Credentials fornece uma maneira simples de obter credenciais de autorização para usar a chamada de APIs do Google. "

Se você puder usar credenciais padrão do aplicativo, use.

Há uma etapa extra que você precisará executar antes de usar as credenciais padrão do aplicativo como sua identidade ao chamar APIs de sua máquina:

gcloud auth application-default login [yourname@gmail.com]

Veja por que você preferirá usar as credenciais padrão do aplicativo:

var scopes = new String[] {
    CloudResourceManagerService.Scope.CloudPlatform
};

GoogleCredential credential = Task.Run(
    () => GoogleCredential.GetApplicationDefaultAsync()
).Result;

if (credential.IsCreateScopedRequired)
{
    credential = credential.CreateScoped(scopes);
}

…Esse é todo o código que você precisa para autorizar chamadas para (qualquer) API do Google Cloud!

Usaremos o objeto credential na próxima etapa para fazer chamadas no serviço do Google…

Chamando qualquer método (!) em qualquer serviço do Google (!) #

Depois de se familiarizar com o código para chamar um método em um serviço do Google, você poderá inferir como chamar qualquer método em qualquer serviço do Google.

Primeiro, fazemos uma conexão com o serviço usando o objeto credential instanciado no exemplo anterior:

CloudResourceManagerService service = new CloudResourceManagerService(
    new BaseClientService.Initializer()
    {
        HttpClientInitializer = credential,
        ApplicationName = "Our First Google API Client"
    }
);

Então podemos chamar métodos fornecidos pelo serviço. Quais métodos estão disponíveis?

https://cloud.google.com/resource-manager/docs/apis

Qual é a solicitação REST subjacente para Projects.Create?

https://cloud.google.com/resource-manager/reference/rest/v1/projects/create

OK… Vamos escrever o código.

O código espera um valor de string para projectId. IDs de projeto são identificadores exclusivos. Eu recomendo que você use um sistema para nomear seus projetos para ajudá-lo a identificá-los.

Projects.Create espera um objeto Data.Project. Este objeto é uma propriedade obrigatória, o ID do projeto, que é tudo o que forneceremos, mas também podemos fornecer um nome do projeto, rótulos, detalhes do pai do projeto etc.

Data.Operation operation1 = service.Projects.Create(
    new Data.Project()
    {
        ProjectId = projectId,
    }
).Execute();

A criação do projeto é tratada de forma assíncrona. Recebemos um objeto Operation que devemos pesquisar para determinar quando o projeto é criado. As operações têm uma propriedade Name que identifica exclusivamente a operação. A próxima seção do código pesquisa a plataforma “Já terminamos?”. O projeto será criado quando nossa nova operação incluir uma propriedade Done que seja True.

Data.Operation operation2;
do
{
    operation2 = service.Operations.Get(operation1.Name).Execute();
    System.Threading.Thread.Sleep(1000);
} while (operation2.Done != true);

Para completar, e espero que daqui a muitos anos, após um uso muito feliz de seu projeto, talvez seja necessário excluí-lo. Simplesmente chamamos Projects.Delete e fornecemos nosso ID de projeto. Isso também retorna uma operação e devemos realmente pesquisar essa operação também até que ela seja concluída definitivamente. Nosso projeto será então excluído.

var operation3 = service.Projects.Delete(projectId).Execute();

É isso!
