Como conectar o Google Cloud SQL com aplicativos (como o Google App Engine) e ferramentas como (mySQL workbench)? #

Neste documento, veremos como criar uma instância do Google Cloud SQL e conectá-las em seu aplicativo Google App Engine e na ferramenta de administração MySQL Workbench.

Google Cloud SQL:

O Google Cloud SQL é um serviço de banco de dados totalmente gerenciado que facilita a configuração, manutenção, gerenciamento e administração de seus bancos de dados MySQL relacionais na nuvem.

O Google Cloud SQL fornece um banco de dados relacional que você pode usar com seu aplicativo do App Engine. Cloud SQL é um banco de dados MySQL que vive na nuvem do Google.

referir:

https://cloud.google.com/sql/

https://cloud.google.com/sql/docs/

Criando instâncias SQL:

Uma instância do Google Cloud SQL é um banco de dados MySQL hospedado na nuvem do Google.

  1. Acesse a página Instâncias do Cloud SQL no Google Cloud Platform Console (https://console.cloud.google.com/sql/instances) and Click Create instance.

  2. Clique em Escolher primeira geração, digite um nome e escolha um nível para the instance and Click Create.

  3. Após a inicialização da instância, selecione a instância para open it.

  4. Em Controle de acesso > Usuários, clique em Criar conta de usuário e crie um user with name root and specify a password (root_password). This creates the MySQL user ‘root’@’%’.

  5. Em Bancos de Dados, clique em Novo Banco de Dados e crie um banco de dados com um DataBase name (DataBase_Name)

Banco de trabalho MySQL:

MySQL Workbench é uma ferramenta visual unificada para arquitetos de banco de dados, desenvolvedores e DBAs. MySQL Workbench fornece modelagem de dados, desenvolvimento de SQL e ferramentas de administração abrangentes para configuração de servidor, administração de usuários, backup e muito mais.

consulte http://www.mysql.com/products/workbench/

Agora veremos como se conectar ao banco de dados da instância do Google Cloud SQL com o MySQL Workbench.

Configurando o acesso

  1. Acesse a página Instâncias do Cloud SQL no Google Cloud Platform Console and select the instance.

  2. Em Controle de acesso > Endereço IP, clique em Solicitar endereço IPv4 e copie it(Instance_IPv4_address). It is needed to connect your Google Cloud SQL instance database with Admin tools like MySQL Workbench.

    note: You will be charged for IPv4 address @ $0.01 each hour the instance is inactive and $0.1 each hour the instance is active

  3. Google ’endereço IP’ para encontrar seu endereço IP público

  4. Em Controle de acesso > Autorização > Redes autorizadas, clique em Adicionar network and enter your IP address.

  5. Em Controle de acesso > Usuários, crie um usuário com nome de usuário (userName), password (password) and the option ‘Allow any host selected’. It is recommended to use a seperate user account to access from WorkBench

Conectando

  1. Na tela inicial do MySQL Workbench, clique em Nova Conexão.

  2. Na janela Configurar Nova Conexão, forneça um Nome de Conexão, Hostname and Username

  3. Clique em Testar conexão. Será solicitada uma senha.

  4. Uma vez que a conexão MySQL seja bem sucedida, clique em OK e clique em the saved connection to open SQL Editor

Google App Engine:

O Google App Engine é uma plataforma para criar aplicativos da Web escaláveis ​​e back-ends móveis. O App Engine dimensionará seu aplicativo automaticamente.

consulte https://cloud.google.com/appengine

Agora veremos como configurar uma conexão entre um aplicativo do App Engine e uma instância do Cloud SQL.

Configurando o acesso

  1. Acesse a página Instâncias do Cloud SQL no Google Cloud Platform Console and select the instance.

  2. Em Controle de acesso > Autorização > App Engine autorizado applications, click Add application ID and enter the application ID. Click Done and Save.

  3. Em Visão geral > Propriedades, copie o ‘Nome da conexão da instância’ (Instance_Connection_Name)

  4. Em seu projeto de aplicativo da web do Google, war/WEB-INF/appengine-web.xml add, true

Amostra de código:

Um exemplo para o Google App Engine - Ambiente padrão Java

public static Connection connect() throws ClassNotFoundException, SQLException {
    String url = null;

    {
        if (SystemProperty.environment.value() == SystemProperty.Environment.Value.Production) {

            // Connecting from App Engine.
            Class.forName(Messages.getString("com.mysql.jdbc.GoogleDriver")); 
            url = Messages.getString("jdbc:google:mysql://{{Instance_Connection_Name}}/{{DataBase_Name}}?user=root&password={{root_password}}");
        } else {
            // Connecting from an external network or localhost
            Class.forName(Messages.getString("com.mysql.jdbc.Driver"));
            url = Messages.getString("jdbc:mysql://{{Instance_IPv4_address}}:3306/{{DataBase_Name}}?user={{userName}}&password={{password}}");
        }

        Connection conn = DriverManager.getConnection(url);

        return conn;
    }
}
