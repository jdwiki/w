Função fatorial usando correspondência de padrões #

let rec factorial n = match n with
| 0 | 1 -> 1
| n -> n * (factorial (n - 1))

Essa função corresponde aos valores 0 e 1 e os mapeia para o caso base de nossa definição recursiva. Em seguida, todos os outros números são mapeados para a chamada recursiva dessa função.

Avaliação de expressões booleanas #

Definimos o tipo de expressões booleanas cujos átomos são identificados por strings como

type expr =
| Atom of string
| Not of expr
| And of expr * expr
| Or of expr * expr

e podemos avaliar essas expressões usando um oracle : string -> bool dando os valores dos átomos que encontramos da seguinte forma:

let rec eval oracle = function
| Atom(name) -> oracle name
| Not(expr) -> not(eval oracle expr)
| And(expr1, expr2) -> (eval oracle expr1) && (eval oracle expr2)
| Or(expr1, expr2)  -> (eval oracle expr1) || (eval oracle expr2)

Veja como a função é clara e fácil de ler. Graças ao uso correto da correspondência de padrões, um programador lendo esta função precisa de pouco tempo para garantir que ela seja implementada corretamente.

Forma normal de negação: correspondência de padrões profunda #

A correspondência de padrões permite desconstruir valores complexos e não é de forma alguma limitada ao nível “mais externo” da representação de um valor. Para ilustrar isso, implementamos a função transformando uma expressão booleana em uma expressão booleana onde todas as negações estão apenas em átomos, a chamada forma normal de negação e um predicado reconhecendo expressões nesta forma:

Definimos o tipo de expressões booleanas cujos átomos são identificados por strings como

type expr =
| Atom of string
| Not of expr
| And of expr * expr
| Or of expr * expr

Vamos primeiro definir um predicado reconhecendo expressões na forma normal de negação:

let rec is_nnf = function
| (Atom(_) | Not(Atom(_))) -> true
| Not(_) -> false
| (And(expr1, expr2) | Or(expr1, expr2)) -> is_nnf expr1 && is_nnf expr2

Como você vê, é possível combinar com padrões aninhados como Not(Atom(_)). Agora implementamos uma função mapeando uma expressão booleana para uma expressão booleana equivalente na forma normal de negação:

let rec nnf = function
| (Atom(_) | Not(Atom(_))) as expr -> expr
| Not(And(expr1, expr2)) -> Or(nnf(Not(expr1)),nnf(Not(expr2)))
| Not(Or(expr1, expr2)) -> And(nnf(Not(expr1)),nnf(Not(expr2)))
| And(expr1, expr2) -> And(nnf expr1, nnf expr2)
| Or(expr1, expr2) -> Or(nnf expr1, nnf expr2)
| Not(Not(expr)) -> nnf expr

Esta segunda função faz ainda mais usos de padrões aninhados. Finalmente podemos testar nosso código no nível superior na negação de uma implicação:

# let impl a b =
Or(Not(a), b);;
  val impl : expr -> expr -> expr = <fun>
# let expr = Not(impl (Atom "A") (Atom "B"));;
val expr : expr = Not (Or (Not (Atom "A"), Atom "B"))
# nnf expr;;
- : expr = And (Atom "A", Not (Atom "B"))
# is_nnf (nnf expr);;
- : bool = true

O sistema do tipo OCaml é capaz de verificar a exaustividade de uma correspondência de padrões. Por exemplo, se omitirmos o caso Not(Or(expr1, expr2)) na função nnf, o compilador emitirá um aviso:

| And(expr1, expr2) -> And(nnf expr1, nnf expr2)
| Or(expr1, expr2) -> Or(nnf expr1, nnf expr2)
| Not(Not(expr)) -> nnf expr;;

Esse recurso fornece um maior nível de segurança e correção em programas que são aceitos pelo compilador. No entanto, tem outros usos e pode, por exemplo, ser usado em programação exploratória. É muito divertido escrever uma conversão para uma forma normal, começando com versões brutas da função que trata dos casos fáceis e usando exemplos de casos não correspondidos fornecidos pelo compilador para refinar o tratamento.

Campos de registro correspondentes #

A correspondência de padrões pode ser usada para desconstruir registros. Ilustramos isso com um tipo de registro representando locais em um arquivo de texto, por exemplo o código-fonte de um programa.

type location = {
  filename : string;
  line: int;
  column: int;
  offset: int;
}

Um valor x do tipo location pode ser desconstruído assim:

let { filename; line; column; offset; } = x

Uma sintaxe semelhante pode ser usada para definir funções, por exemplo, uma função para imprimir locais:

let print_location { filename; line; column; offset; } =
  Printf.printf "%s: %d: %d" filename line column

ou alternativamente

let print_location = function { filename; line; column; offset; } ->
  Printf.printf "%s: %d: %d" filename line column

Os padrões correspondentes aos registros não precisam mencionar todos os campos de um registro. Como a função não usa o campo offset, podemos deixá-lo de fora:

let print_location { filename; line; column; } =
  Printf.printf "%s: %d: %d" filename line column

Quando o registro é definido em um módulo, basta qualificar o primeiro campo que ocorre no padrão:

module Location =
struct
  type t = {
      filename : string;
      line: int;
      column: int;
      offset: int;
    }
end

let print_location { Location.filename; line; column; } =
  Printf.printf "%s: %d: %d" filename line column

Processamento de lista recursiva com correspondência de padrões #

Aqui demonstramos como processar listas recursivamente usando a sintaxe de correspondência de padrões do OCaml.

let rec map f lst =
  match lst with
  | [] -> []
  | hd::tl -> (f hd)::(map f tl)

Nesse caso, o padrão [] corresponde à lista vazia, enquanto hd::tl corresponde a qualquer lista que tenha pelo menos um elemento e atribuirá o primeiro elemento da lista a hd e o restante da lista (pode estar vazia) para tl.

Observe que hd::tl é um padrão muito geral e corresponderá a qualquer lista que não esteja vazia. Também podemos escrever padrões que correspondam a listas com um número específico de elementos:

(* Return the last element of a list. Fails if the list is empty. *)
let rec last lst =
  match lst with
  | [] -> failwith "Empty list"
  | [x] -> x (* Equivalent to x::[], [x] matches a list with only one element *)
  | hd::tl -> last tl

(* The second to last element of a list. *)
let rec second_to_last lst =
  match lst with
  | [] -> failwith "Empty list"
  | x::[] -> failwith "Singleton list"
  | fst::snd::[] -> snd
  | hd::tl -> second_to_last tl

Além disso, o OCaml suporta correspondência de padrões nos elementos das próprias listas. Podemos ser mais específicos sobre a estrutura dos elementos dentro de uma lista, e o OCaml inferirá o tipo de função correto:

(* Assuming a list of tuples, return a list with first element of each tuple. *)
let rec first_elements lst =
  match lst with
  | [] -> []
  | (a, b)::tl -> a::(first_elements tl)
(* val first_elements : ('a * 'b) list -> 'a list = <fun> *)

Ao combinar esses padrões, podemos processar qualquer lista arbitrariamente complexa.

Definindo uma função usando correspondência de padrões #

A palavra-chave function pode ser usada para iniciar a correspondência de padrões no último argumento de uma função. Por exemplo, podemos escrever uma função chamada sum, que calcula a soma de uma lista de inteiros, desta forma

let rec sum = function
  | [] -> 0
  | h::t -> h + sum t
;;

val sum : int list -> int = <fun>
