Instalação ou Configuração #

Instalando o OPAM #

OPAM é um gerenciador de pacotes para OCaml. Ele constrói e gerencia versões do compilador e bibliotecas OCaml para você facilmente.

A maneira mais fácil de instalar o OPAM em seu sistema operacional é usar um gerenciador de pacotes para seu sistema. por exemplo, apt-get, yum ou homebrew.

Instruções de instalação do Mac OSX #

Atualize as fórmulas homebrew e instale o OPAM.

brew update
brew install opam

Instruções de instalação do Ubuntu #

add-apt-repository ppa:avsm/ppa
apt-get update
apt-get install ocaml opam

Compilando da fonte #

wget http://caml.inria.fr/pub/distrib/ocaml-4.03/ocaml-4.03.0.tar.gz
tar xf ocaml-4.03.0.tar.gz
cd ocaml-4.03.0
./configure -prefix installation_path
make world.opt
make install

Inicializando o OPAM #

Depois de instalar o OPAM, execute opam init e siga as instruções.

Uma vez feito, você poderá executar o interpretador OCaml do seu shell.

$ ocaml
        OCaml version 4.03.0

#

Seu primeiro programa em OCaml #

Agora que a distribuição OCaml está disponível em seu sistema operacional favorito, podemos criar seu primeiro programa em OCaml: o Hello World!

Temos diferentes maneiras de lançar um programa OCaml.

O REPL (nível superior) #

Você pode executar seu código interativamente com o toplevel. Com o OCaml toplevel, você pode escrever e executar código OCaml, como um shell UNIX. Depois, o toplevel verifica o tipo do seu código imediatamente. Assim, você pode testar de forma rápida e fácil algumas partes do código sem compilação e execução.

Você pode iniciar o toplevel com o comando ocaml. Então, você pode escrever uma frase OCaml terminada por ;; que é avaliada imediatamente. O toplevel exibe o tipo e o valor da sua expressão logo após:

# "Hello Worlds!";;
- : string = "Hello Worlds!"

Também é possível iniciar o toplevel em seu arquivo. Você pode ver esta explicação sobre isso.

Para facilitar sua entrada no toplevel, você pode usar uma ferramenta como ledit ou rlwrap que fornece alguns recursos (como histórico de entrada):

 $ ledit ocaml

 $ rlwrap ocaml

Compilação para o bytecode #

Temos dois compiladores diferentes, um que compila para bytecode e outro que compila para código nativo. O primeiro é o mesmo que o bytecode da máquina virtual do Java. Assim, o bytecode é menos eficiente, mas mais portátil.

Temos alguns arquivos de extensão usados ​​pelos compiladores OCaml:

extensão definição
.ml O código-fonte (como .c em C)
.mli A interface (como .h em C)
.cmo Código fonte compilado por ocamlc em bytecode
.cmi Código de interface compilado por ocamlc
.cmx e .o Código fonte compilado por ocamlopt em código nativo
.cma Biblioteca (bucket de alguns *.cmo) em bytecode
.cmxa e .a Biblioteca em código nativo
.cmxs Biblioteca em código nativo (para carregar dinamicamente)

O compilador bytecode é ocamlc.

Você tem diferentes opções comuns:

  • -c: para compilar um arquivo fonte sem o processo de vinculação (para produzir um executável). Assim, o comando ocaml -c foo.ml produz um arquivo .cmo. Ao contrário do C em que o arquivo de cabeçalho não precisa ser compilado, é necessário no OCaml compilar o arquivo .mli: ocaml -c foo.mli.

Você precisa compilar a interface primeiro. Quando você compila o arquivo de origem posteriormente, o OCaml tenta verificar se a implementação corresponde à interface.

O arquivo .mli não é obrigatório. Se você compilar um arquivo .ml sem um arquivo .mli, o OCaml produzirá um arquivo .cmi automaticamente.

  • -o: para compilar alguns arquivos .cmo para um executável. Por exemplo: ocamlc -o programa foo.cmo bar.cmo. Esses arquivos precisam ser organizados pelas dependências para as quais o primeiro arquivo não tem dependência.

  • -I: para indicar outro diretório onde o compilador possa encontrar os arquivos necessários para a compilação (como a interface ou o código fonte). É o mesmo que o -I de um compilador C.

Temos muitas outras opções. Você pode ver o [manual][manual] para mais informações.

Então, você pode escrever o hello.ml agora, e compilar este arquivo com ocamlc -o hello hello.ml para produzir um programa bytecode:

let () = print_endline "Hello World!"

O let() = ... é a primeira entrada do seu programa (como o main em C). Depois, usamos a função print_endline (fornecida pela biblioteca padrão) com o argumento "Hello World!" para imprimir Hello Worlds com uma nova linha na saída padrão.

Após a compilação, você tem o arquivo .cmo e o arquivo .cmi produzidos automaticamente pelo compilador e seu programa hello. Você pode abrir seu programa e, no topo deste arquivo, você pode ver:

#!/usr/local/bin/ocamlrun

Isso significa que seu programa precisa do programa ocamlrun (fornecido pela distribuição) para executar o bytecode (como a JVM).

Compilação para o código nativo #

Temos um outro compilador que produz código nativo. O compilador é: ocamlopt. No entanto, o executável resultante não pode funcionar na maioria das outras arquiteturas.

ocamlopt usa as mesmas opções que ocamlc para que você possa executar ocamlopt -o hello hello.ml. Depois, você pode ver um arquivo .cmx e um arquivo .o.

Finalmente, a partir do seu programa bytecode/código nativo, você pode executar:

$ ./hello
Hello World!
$

Instalação no Windows (nativo) #

Premissa #

Estas instruções mostram um procedimento para instalar binários nativos do OCaml no Windows. Se o seu sistema operacional for Windows 10 (Insider Preview) build 14316 ou posterior, você também pode instalar o OCaml através do Bash no Ubuntu no Windows. Nesse caso, siga as instruções para instalar o OCaml no Ubuntu.

Instale OCaml e Opam #

Baixe distribuição oficial OCaml. Ele contém compiladores OCaml e gerenciador de pacotes Opam. Suponha que você tenha instalado o software em C:/OCaml. Para ter certeza de que instalou corretamente o OCaml, abra cmd.exe e digite ocaml.

Se você vir a mensagem 'ocaml' não é reconhecido como um comando interno ou externo, programa operável ou arquivo de lote, você precisa adicionar C:/OCaml/bin ao seu caminho (variável de ambiente).

Adicionar binários OCaml ao caminho #

em Painel de controle > Sistema e segurança > Sistema > Configurações avançadas do sistema (à esquerda) > Variáveis ​​de ambiente e, em seguida, selecione Caminho na guia Variável do sistema e, em seguida, Editar.

Adicione C:/OCaml/bin; à lista.

Instalar Cygwin #

Sem Cygwin você não pode usar Opam. Na verdade, se você tentar abrir o Opam digitando opam em cmd.exe ele mostra uma mensagem: Fatal error: exception Unix.Unix_error(20, "create_process", "cygcheck").

Baixe Cygwin e inicie o instalador. Certifique-se de verificar os seguintes pacotes:

  • autofabricar
  • difícil
  • libreadline
  • faço
  • m4
  • mingw64-x86_64-gcc-core
  • mingw64-x86_64-gmp
  • mingw64-x86_64-openssl
  • mingw64-x86_64-pkg-config
  • mingw64-x86_64-sqlite3
  • correção
  • rl wrap
  • descompactar
  • wget

Suponha que você tenha instalado o software em C:/cygwin (C:/cygwin64 para versão de 64 bits). Abra cmd e digite wget (ou um dos executáveis ​​presentes em C:/cygwin/bin) para verificar se você pode usar os executáveis ​​Cygwin. Se o executável não abrir, adicione C:/cygwin/bin ao seu caminho (variável de ambiente).

Configurar Opam #

Abra cmd.exe e digite opam init para configurar o Opam.

Em seguida, instale ocamlfind (parte do compilador OCaml) com

opam install ocamlfind
opam config env

Verifique se o ocamlfind está instalado digitando-o em cmd.exe.

O comando opam config env é usado para adicionar o diretório executável do opam ao caminho do ambiente. Se após o logout você não conseguir mais acessar ocamlfind, você pode adicioná-lo manualmente adicionando ao caminho a seguinte linha: C:/Users/<seu usuário>/Documents/.opam/system/bin/.

Instalando pacotes #

Os pacotes são instalados através do Opam com o comando opam install xyz onde xyz é o nome do pacote.

Instalar UTop #

Tente executar o comando opam install utop. Se você não tiver erros, digitar utop abrirá o executável.

Se você ver a mensagem

[ERROR] The compilation of zed failed at "ocaml setup.ml -build".

você tem que instalar manualmente os pacotes únicos. Tente novamente digitando:

opam install zed
opam install lambda-term
opam install utop

Tanto o lambda-term quanto o utop podem não ser instalados. Consulte a seção Solução de problemas.

Instalando o Núcleo #

Você pode instalar o pacote core com opam install core. Na versão de 64 bits do Windows (e Cygwin de 64 bits), você verá o seguinte erro:

[ERROR] core is not available because your system doesn't comply with os != "win32" & ocaml-version = "4.02.3".

Solução de problemas: não é possível criar um arquivo normal #

Se o pacote com o nome xyz.10.1 falhar na instalação (onde xyz é o nome do pacote e 10.1 sua versão) com a seguinte mensagem:

install: cannot create regular file '/cygdrive/c/Users/<your user>/Documents/.opam/system/bin/<something>': File exists

Você tem que entrar neste diretório:

C:\Users\<your user>\Documents\.opam\repo\default\packages\<xyz>\<xyz.10.1>\files

e exclua o arquivo xyz.10.1.install.

Solução de problemas: não é possível carregar a biblioteca compartilhada #

Se você tentar abrir algum pacote do Opam (por exemplo: utop) e vir este erro:

Fatal error: cannot load shared library dlllwt-unix_stubs
Reason: The specified module could not be found.

Execute opam config env novamente e tente reabrir o executável.
