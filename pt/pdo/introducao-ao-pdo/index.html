Instalação ou Configuração #

PDO é um comando de conexão de banco de dados universal em PHP, suporta 12 tipos de banco de dados diferentes, por exemplo, MySQL, MongoDB, NoSQL. Um grande bônus sobre o PDO é que ele calcula seu código para suportar o tipo de banco de dados, então você não precisa fazer nenhuma chance ao mudar para outro sistema de banco de dados.

Conexão ao banco de dados com PDO:

A parte da conexão parece estranha, mas precisamos lidar com isso. O resto do PDO é simples e útil, também ajuda a tornar a parte segura ainda mais fácil.

$connection = new PDO("mysql:host=localhost;dbname=myDatabase, username, password);

A conexão PDO é ordenada da seguinte forma:

PDO(database type:host=host;dbname=database name, root, password** );**

Seleção segura do banco de dados usando PDO:

// We use a array to hold the data about whats the :var is in normal $var
$params = array(
    ':username' => '$username', 
    ':email' => $mail, 
);

// Prepare the SQL and using named secure parameters ":username"
$pdo->prepare('SELECT * FROM users WHERE username = :username AND email = :email');

// Execute the $params and send them to the $pdo->prepare 
$pdo->execute($params);

O código que você acabou de ler, é injeção de SQL de agentes protegidos

Modo de erro do atributo de configuração do PDO #

PDO::setAttribute define um atributo no identificador de banco de dados. A descrição de setAttribute é:

public bool PDO::setAttribute ( int $attribute , mixed $value )

PDO::ATTR_ERRMODE: Este atributo é usado para relatórios de erros. Pode ter um dos seguintes valores.

  • PDO::ERRMODE_SILENT: Se o ATTR_ERRMODE não estiver definido no código, ERRMODE_SILENT é o valor padrão do atributo ATTR_ERRMODE. Isto define códigos de erro. No modo silencioso, se houver erro no SQL, PDO não lançará exceções; O PDO não emitirá avisos; vai simplesmente retorna falso. O valor de PDO::ERRMODE_SILENT é 0. O script será executado sem gerar nenhum erro ou aviso.
  • PDO::ERRMODE_WARNING: Este valor gera E_WARNING. No modo de aviso, se houver um erro no SQL, o PDO emitirá avisos, mas o script continuar correndo. O valor de PDO::ERRMODE_WARNING é 1. O script irá execute com a geração de aviso sobre o erro.
  • PDO::ERRMODE_EXCEPTION: Este valor lança exceções. Em exceção modo, se houver um erro no SQL, o PDO lançará exceções e script irá parar de ser executado. O valor de PDO::ERRMODE_EXCEPTION é 2. O valor script irá parar de executar gerando o erro que lança o exceção.

Exemplo: Agora vamos ver os vários valores do atributo ATTR_ERRMODE com alguns exemplos. Para fazer isso, crie um banco de dados chamado learn_project_db e insira uma tabela chamada user_table dentro dele. O seguinte trecho de SQL pode ser usado para isso:

DROP DATABASE IF EXISTS `learn_project_db`;
CREATE DATABASE `learn_project_db`;
USE `learn_project_db`;
CREATE TABLE `user_table` (
  `user_email` varchar(50) PRIMARY KEY,
  `user_password` varchar(50) NOT NULL
);
INSERT INTO `user_table` (`user_email`, `user_password`) VALUES
('test1@example.com', '123'),
('test2@example.com', '1234'),
('test3@example.com', '12345');

Primeiramente vamos verificar o que acontecerá se não definirmos o ATTR_ERRMODE e teremos erro na consulta SQL. Crie um arquivo PHP chamado default.php e tente isto:

<?php
    $server = "localhost";
    $db_username = "root";
    $db_password = "";
    $db_name = "learn_project_db";
    $conn = new PDO("mysql:host=$server;dbname=$db_name",$db_username,$db_password);
    $sql_query = "SELECT * FROM wrong_user_table";
    $stmt = $conn->prepare($sql_query);
    $stmt->execute();
    $result_set = $stmt->fetchAll();
    var_dump($result_set);
    /*Get the current error mode of PDO*/
    $current_error_mode = $conn->getAttribute(PDO::ATTR_ERRMODE);
    echo "<br>";
    echo "Value of PDO::ATTR_ERRMODE: ".$current_error_mode; 
?>

Observe que o nome da tabela é wrong_user_table na consulta que não está definida no banco de dados que criamos anteriormente. Mas, como não configuramos o ATTR_ERRMODE, ele executará o script sem lançar nenhuma exceção ou emitir nenhum aviso. Ele produzirá uma matriz vazia como conjunto de resultados. Devemos notar também que o valor de PDO::ATTR_ERRMODE é 0.

Agora, vamos verificar o que acontecerá se definirmos o ATTR_ERRMODE com o valor PDO::ERRMODE_WARNING e teremos erro na consulta SQL. Crie um arquivo PHP chamado warning.php e tente isto:

<?php
    $server = "localhost";
    $db_username = "root";
    $db_password = "";
    $db_name = "learn_project_db";
    $conn = new PDO("mysql:host=$server;dbname=$db_name",$db_username,$db_password);
    $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_WARNING);    
    $sql_query = "SELECT * FROM wrong_user_table";
    $stmt = $conn->prepare($sql_query);
    $stmt->execute();
    $result_set = $stmt->fetchAll();
    var_dump($result_set);
    /*Get the current error mode of PDO*/
    $current_error_mode = $conn->getAttribute(PDO::ATTR_ERRMODE);
    echo "<br>";
    echo "Value of PDO::ATTR_ERRMODE: ".$current_error_mode; 
?>

A saída de warning.php é:

Warning: PDOStatement::execute(): SQLSTATE[42S02]: Base table or view not found: 1146 Table 'learn_project_db.wrong_user_table' doesn't exist in E:\xampp\htdocs\oop\db.php on line 10
array(0) { } 
Value of PDO::ATTR_ERRMODE: 1

Desta vez, quando definirmos o valor ATTR_ERRMODE com o valor PDO::ERRMODE_WARNING, ele mostrará uma mensagem de aviso. O script é executado com sucesso e mostra um array vazio como saída com o valor de PDO::ATTR_ERRMODE de 1.

Por fim, vamos verificar o que acontecerá se definirmos o ATTR_ERRMODE com o valor PDO::ERRMODE_EXCEPTION e teremos erro na consulta SQL. Crie um arquivo PHP chamado error.php e tente isto:

<?php
    $server = "localhost";
    $db_username = "root";
    $db_password = "";
    $db_name = "learn_project_db";
    $conn = new PDO("mysql:host=$server;dbname=$db_name",$db_username,$db_password);
    $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);    
    $sql_query = "SELECT * FROM wrong_user_table";
    $stmt = $conn->prepare($sql_query);
    $stmt->execute();
    $result_set = $stmt->fetchAll();
    var_dump($result_set);
    /*Get the current error mode of PDO*/
    $current_error_mode = $conn->getAttribute(PDO::ATTR_ERRMODE);
    echo "<br>";
    echo "Value of PDO::ATTR_ERRMODE: ".$current_error_mode; 
?>

A saída de error.php é:

Fatal error: Uncaught exception 'PDOException' with message 'SQLSTATE[42S02]: Base table or view not found: 1146 Table 'learn_project_db.wrong_user_table' doesn't exist' in E:\xampp\htdocs\oop\db.php:10 Stack trace: #0 E:\xampp\htdocs\oop\db.php(10): PDOStatement->execute() #1 {main} thrown in E:\xampp\htdocs\oop\db.php on line 10

Desta vez, quando definimos o valor ATTR_ERRMODE com o valor PDO::ERRMODE_EXCEPTION ele lançará PDOException que gera um erro fatal. O script parará de ser executado depois que a exceção for lançada. Essa é a abordagem mais comum para lidar com erros relacionados a consultas de banco de dados. Na maioria das vezes, definimos o atributo ATTR_ERRMODE com este valor para tratar qualquer exceção que possa residir na consulta SQL.
