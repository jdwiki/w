Sintaxe #

  • 400 invalidParameter Indica que um parâmetro de solicitação possui um valor inválido.

  • 400 badRequest Indica que a consulta era inválida. Por exemplo, o ID do pai estava ausente ou a combinação de dimensões ou métricas solicitadas não era válida.

  • 403 insuficientePermissions Indica que o usuário não possui permissões suficientes para a entidade especificada na consulta.

  • 403 dailyLimitExceeded Indica que o usuário excedeu a cota diária (por projeto ou por vista (perfil)).

  • 403 usageLimits.userRateLimitExceededUnreg Indica que o aplicativo precisa ser registrado no Google API Console.

  • 403 userRateLimitExceeded Indica que o limite de taxa do usuário foi excedido. O limite de taxa máxima é de 10 qps por endereço IP.

  • 403 rateLimitExceeded Indica que os limites de taxa global ou geral do projeto foram excedidos. Tente novamente usando o recuo exponencial. Você precisa diminuir a taxa na qual você está enviando as solicitações.

  • 403 quotaExceeded Indica que as 10 solicitações simultâneas por vista da propriedade (perfil) na API de relatórios principais foram alcançadas.

  • 429 RESOURCE_EXHAUSTED AnalyticsDefaultGroupCLIENT_PROJECT-1d Indica que a cota de Solicitações por dia por projeto foi esgotada.

  • 429 RESOURCE_EXHAUSTED AnalyticsDefaultGroupCLIENT_PROJECT-100s Indica que a cota de solicitações por 100 segundos por projeto foi esgotada.

  • 429 RESOURCE_EXHAUSTED AnalyticsDefaultGroupUSER-100s Indica que as solicitações por 100 segundos por usuário por cota de projeto foram esgotadas.

  • 429 RESOURCE_EXHAUSTED DiscoveryGroupCLIENT_PROJECT-100s Indica que a cota de solicitações de descoberta por 100 segundos foi esgotada.

  • 500 internalServerError Ocorreu um erro de servidor interno inesperado. Não repita esta consulta mais de uma vez.

  • 503 backendError O servidor retornou um erro. Não repita esta consulta mais de uma vez.

Parâmetros #

Nome Descrição
domínio Localização do erro ex: global
razão Motivo do erro
mensagem Uma mensagem explicando o erro e uma possível solução.
localTipo O tipo de localização do erro ex: parâmetro
localização A localização real do erro

Se uma solicitação for bem-sucedida, a API retornará um código de status HTTP 200 junto com os dados solicitados no corpo da resposta.

Se ocorrer um erro com uma solicitação, a API retornará um código de status HTTP e o motivo da resposta com base no tipo de erro. Além disso, o corpo da resposta contém uma descrição detalhada do que causou o erro.

400 parâmetro inválido #

É importante ler a resposta de erro retornada pelo servidor da API do Google Analytics. Em muitos casos, eles podem dizer exatamente o que está errado.

400 invalidParameter

{
 "error": {
  "errors": [
   {
    "domain": "global",
    "reason": "invalidParameter",
    "message": "Invalid value '-1' for max-results. Value must be within the range: [1, 1000]",
    "locationType": "parameter",
    "location": "max-results"
   }
  ],
  "code": 400,
  "message": "Invalid value '-1' for max-results. Value must be within the range: [1, 1000]"
 }
}

Neste caso a mensagem:

“message”: “Valor inválido ‘-1’ para resultados máximos. O valor deve estar dentro do intervalo: [1, 1000]”,

Está nos dizendo que enviamos um valor -1 para o parâmetro max-results que não é válido, só podemos enviar um valor de 1-1000.
