Sintaxe #

  • OBTER https://accounts.google.com/o/oauth2/auth?client_id={clientid}&redirect_uri=urn:ietf:wg:oauth:2.0:oob&scope={Scopes}&response_type=code
  • POSTAR https://accounts.google.com/o/oauth2/token code={Código da chamada acima}&client_id={ClientId}&client_secret={ClientSecret}&redirect_uri=urn:ietf:wg:oauth:2.0:oob&grant_type=authorization_code
  • POSTAR https://accounts.google.com/o/oauth2/token client_id={ClientId}&client_secret={ClientSecret}&refresh_token={Da chamada acima}&grant_type=refresh_token

Parâmetros #

Parâmetro Descrição
Identificação do cliente Do console Google Developer identifica seu projeto ou aplicativo
segredo Do console Google Developer identifica seu projeto ou aplicativo
redirecionamento_uri Do Google Developer local para onde a autenticação deve ser retornada. No caso de aplicações nativas urn:ietf:wg:oauth:2.0:oob pode ser usado para denotar localhost

Para acessar qualquer API do Google, você precisa se identificar como desenvolvedor e identificar seu projeto. Fazemos isso criando um novo projeto no Google Developers console.

Ao criar seu projeto, se você deseja acessar as APIs do Google Analytics, deve habilitar as APIs que pretende acessar.

  • API de relatórios: acesso à API de relatórios do Google Analytics v4.
  • API de análise: acesso a todo o resto.

Agora você deve decidir como deseja acessar os dados.

Com o Google Data existem dois tipos de Dados Públicos e Privados.

  • os dados públicos não são de propriedade de um usuário. A API de metadados é uma API pública, você não precisa estar logado para acessar esses dados.
  • A API de relatórios contém dados do Google Analytics de usuários, são privados, você não pode visualizá-los, a menos que o usuário tenha concedido permissão para acessá-los.

Se você estiver acessando apenas dados públicos, tudo o que você precisa fazer é criar uma chave de API pública e poderá acessar a API em questão. Se você for acessar dados de usuários privados, precisará criar credenciais Oauth2 ou credenciais de conta de serviço.

Autorização de autorização2

Para acessar os dados privados do usuário, devemos ter permissão do proprietário dos dados para acessá-los. Oauth2 nos permite solicitar esse acesso de um usuário.

Você provavelmente já viu antes Oauth2 antes.

O Aplicativo “Google Analytics Windows” está solicitando acesso para visualizar os usuários “Google Analytics Data”

  1. Janelas do Google Analytics é o nome do projeto que foi criado no Google Developer Console.
  2. Os dados do Google Analytics são o escopo das permissões que solicitamos.

Alcance Precisamos dizer ao usuário o que pretendemos fazer a API do Google Analytics tem dois escopos que você pode usar.

  1. https://www.googleapis.com/auth/analytics.readonly
  2. https://www.googleapis.com/auth/analytics

É melhor solicitar apenas os escopos que você precisa. Se você estiver lendo apenas os dados de um usuário, precisará solicitar apenas o escopo somente leitura.

Contas de serviço de autorização

Contas de serviço são diferentes porque são pré-aprovadas. Se você criar credenciais de conta de serviço, você, como desenvolvedor, poderá pegar o e-mail da conta de serviço e adicioná-lo como usuário em sua conta do Google Analytics No nível da conta, isso concederá à conta de serviço acesso aos dados. Você não precisará abrir a janela de autenticação e solicitar acesso. A conta de serviço terá acesso aos dados enquanto for usuário da conta do Google Analytics.

Conclusão

A autenticação é necessária para acessar a maioria dos dados expostos pela API do Google Analytics.

Você não pode usar login/login e senha do cliente para acessar qualquer Google API a partir de maio de 2015. Você deve usar a autenticação aberta.

Oauth2 C# #

O exemplo usa a biblioteca cliente Dotnet de APIs do Google.

PM> Pacote de instalação Google.Apis.AnalyticsReporting.v4

    /// <summary>
    /// This method requests Authentcation from a user using Oauth2.  
    /// Credentials are stored in System.Environment.SpecialFolder.Personal
    /// Documentation https://developers.google.com/accounts/docs/OAuth2
    /// </summary>
    /// <param name="clientSecretJson">Path to the client secret json file from Google Developers console.</param>
    /// <param name="userName">Identifying string for the user who is being authentcated.</param>
    /// <returns>DriveService used to make requests against the Drive API</returns>
    public static AnalyticsReportingService AuthenticateOauth(string clientSecretJson, string userName)
    {
        try
        {
            if (string.IsNullOrEmpty(userName))
                throw new Exception("userName is required.");
            if (string.IsNullOrEmpty(clientSecretJson))
                throw new Exception("clientSecretJson is required.");
            if (!File.Exists(clientSecretJson))
                throw new Exception("clientSecretJson file does not exist.");
            
            // These are the scopes of permissions you need.
            string[] scopes = new string[] { AnalyticsReportingService.Scope.AnalyticsReadonly };   // View your Google Analytics Data

            UserCredential credential;
            using (var stream = new FileStream(clientSecretJson, FileMode.Open, FileAccess.Read))
            {
                string credPath = System.Environment.GetFolderPath(System.Environment.SpecialFolder.Personal);
                credPath = Path.Combine(credPath, ".credentials/", System.Reflection.Assembly.GetExecutingAssembly().GetName().Name);

                // Requesting Authentication or loading previously stored authentication for userName
                credential = GoogleWebAuthorizationBroker.AuthorizeAsync(GoogleClientSecrets.Load(stream).Secrets,
                                                                         scopes,
                                                                         userName,
                                                                         CancellationToken.None,
                                                                         new FileDataStore(credPath, true)).Result;
            }

            // Create Drive API service.
            return new AnalyticsReportingService(new BaseClientService.Initializer()
            {
                HttpClientInitializer = credential,
                ApplicationName = string.Format("{0} Authentication", System.Reflection.Assembly.GetExecutingAssembly().GetName().Name),
            });
        }
        catch (Exception ex)
        {
            Console.WriteLine("Create Oauth2 DriveService failed" + ex.Message);
            throw new Exception("CreateOauth2DriveFailed", ex);
        }
    }

 Public Shared Function getServiceInitializer() As BaseClientService
    Dim serviceAccountCredentialFilePath = "Path to Json service account key file"                    REM from Google Developers console
    Dim myKeyEMail = "XXX@developer.gserviceaccount.com"   REM from Google Developers console
    Dim scope = Google.Apis.AnalyticsReporting.v4.AnalyticsReportingService.Scope.AnalyticsReadonly


    Try

        Dim credential
        Using stream As New FileStream(serviceAccountCredentialFilePath, FileMode.Open, FileAccess.Read)

            credential = GoogleCredential.FromStream(stream).CreateScoped(scope)

        End Using

        Dim Initializer As New BaseClientService.Initializer()
        Initializer.HttpClientInitializer = credential
        Initializer.ApplicationName = "SRJCGMail"

        Dim service As New AnalyticsReportingService(Initializer)
        Return service

    Catch ex As Exception
        Console.WriteLine(ex.Message)
        Return Nothing
    End Try

End Function
