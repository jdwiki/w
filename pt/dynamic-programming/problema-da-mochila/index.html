Problema da mochila

  • 0-1 Problema na mochila

Nesta página

  • 0-1 Problema na mochila

O problema da mochila ou problema da mochila é um problema em otimização combinatória. Dado um conjunto de itens, cada um com um peso e um valor, determine o número de cada item a ser incluído em uma coleção de modo que o peso total seja menor ou igual a um determinado limite e o valor total seja o maior possível. Seu nome deriva do problema enfrentado por alguém que é limitado por uma mochila de tamanho fixo e deve enchê-la com os itens mais valiosos.

O problema geralmente surge na alocação de recursos onde há restrições financeiras e é estudado em áreas como combinatória, ciência da computação. .org/wiki/Computer_science), teoria da complexidade, criptografia, matemática aplicada e esportes de fantasia diários.

O problema da mochila tem sido estudado por mais de um século, com os primeiros trabalhos datando de 1897. O nome “problema da mochila” remonta aos primeiros trabalhos do matemático [Tobias Dantzig](https://en.wikipedia.org /wiki/Tobias_Dantzig) (1884-1956) e refere-se ao problema comum de embalar seus itens mais valiosos ou úteis sem sobrecarregar sua bagagem.

0-1 Problema na mochila #

Suponha que lhe perguntem, dado o peso total que você pode carregar em sua mochila e alguns itens com seus pesos e valores, como você pode levar esses itens de tal forma que a soma de seus valores seja máxima, mas a soma de seus pesos não exceder o peso total que você pode carregar? O 0-1 indica que você escolhe o item ou não. Também temos uma quantidade de cada item. Isso significa que você não pode dividir o item. Se não for um problema da mochila 0-1, isso significa que se você pudesse ter dividido os itens, há uma solução gananciosa para isso, que é chamada de problema da mochila fracionária.

Vamos, por enquanto, nos concentrar em nosso problema em mãos. Por exemplo, digamos que temos uma capacidade de mochila de 7. Temos 4 itens. Seus pesos e valores são:

+----------+---+---+---+---+
|   Item   | 1 | 2 | 3 | 4 |
+----------+---+---+---+---+
|  Weight  | 1 | 3 | 4 | 5 |
+----------+---+---+---+---+
|   Value  | 1 | 4 | 5 | 7 |
+----------+---+---+---+---+

Um método de força bruta seria pegar todas as combinações possíveis de itens. Então podemos calcular seus pesos totais e excluí-los que excedem a capacidade de nossa mochila e descobrir aquele que nos dá valor máximo. Para 4 itens, precisaremos verificar (4! - 1) = 23 combinações possíveis (excluindo uma sem itens). Isso é bastante complicado quando o número de itens aumenta. Aqui, alguns aspectos que podemos notar, ou seja:

  • Podemos pegar itens menores e calcular o valor máximo que podemos obter usando esses itens e combiná-los. Assim, nosso problema pode ser dividido em subproblemas.
  • Se calcularmos as combinações para o item {1,2}, podemos usá-lo quando calcularmos {1, 2, 3}.
  • Se minimizarmos o peso e maximizarmos o valor, podemos descobrir a nossa solução ideal.

Por esses motivos, usaremos programação dinâmica para resolver nosso problema. Nossa estratégia será: sempre que um novo item chegar, verificaremos se podemos ou não escolher o item e novamente escolheremos os itens que nos dão valor máximo. Agora, se escolhermos o item, nosso valor será o valor do item, mais o valor que conseguirmos subtraindo o valor da nossa capacidade e o máximo que conseguirmos com o peso restante. Se não escolhermos o item, escolheremos os itens que nos dão valor máximo sem incluir o item. Vamos tentar entender com nosso exemplo:

Pegaremos um array 2D tabela, onde o número de colunas será o valor máximo que podemos obter pegando os itens + 1. E o número de linhas será igual ao número de itens. Nosso array ficará assim:

+-------+--------+---+---+---+---+---+---+---+---+
| Value | Weight | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
+-------+--------+---+---+---+---+---+---+---+---+
|   1   |    1   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   4   |    3   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   5   |    4   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   7   |    5   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+

Incorporamos o peso e o valor de cada item ao array para nossa conveniência. Lembre-se que eles não fazem parte do array, são apenas para fins de cálculo, você não precisa armazenar esses valores no array table.

Nossa primeira coluna é preenchida com 0. Isso significa que se nossa capacidade máxima for 0, não importa o item que tenhamos, já que não podemos escolher nenhum item, nosso valor máximo será 0. Vamos começar na Tabela[0][1]. Quando estamos preenchendo a Tabela[1][1], estamos nos perguntando se nossa capacidade máxima era 1 e tivéssemos apenas o primeiro item, qual seria nosso valor máximo? O melhor que podemos fazer é 1, escolhendo o item. Para Tabela[0][2], isso significa que se nossa capacidade máxima for 2 e tivermos apenas o primeiro item, o valor máximo que podemos obter é 1. Isso será o mesmo para Tabela[0][3], Tabela[0][4], Tabela[0][5], Tabela [0][6] e Tabela[0][7]. Isso porque temos apenas um item, o que nos dá valor 1. Como temos apenas 1 quantidade de cada item, não importa como aumentemos a capacidade de nossa mochila, de um item, 1 é o melhor valor que podemos fazer. Então nosso array ficará assim:

+-------+--------+---+---+---+---+---+---+---+---+
| Value | Weight | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
+-------+--------+---+---+---+---+---+---+---+---+
|   1   |    1   | 0 | 1 | 1 | 1 | 1 | 1 | 1 | 1 |
+-------+--------+---+---+---+---+---+---+---+---+
|   4   |    3   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   5   |    4   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   7   |    5   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+

Continuando, para Tabela[1][1], estamos nos perguntando se tínhamos os itens 1 e 2 e se a capacidade máxima de nossa mochila era **1 **, qual é o valor máximo que podemos obter? Se pegarmos os itens 1 e 2, o peso total será 4, o que excederá nossa capacidade atual de mochila. Portanto, o item 2 não pode ser selecionado. Agora, qual é o melhor que podemos fazer sem pegar o item 2? O valor de cima, que é Tabela[0][1] que contém o valor máximo que podemos obter se tivéssemos capacidade máxima 1 e não escolhemos o segundo item. Para Tabela[1][2], como 2 é menor que peso[2], que é o peso do segundo item, não podemos pegar o item . Assim, podemos estabelecer que, se o peso do item atual for maior que nossa capacidade máxima, não podemos levar esse item. Nesse caso, simplesmente pegaremos o valor de cima, que representa o valor máximo que podemos pegar excluindo o item.

if weight[i] > j
    Table[i][j] := Table[i-1][j]
end if

Agora, para Tabela[1][3], já que nossa capacidade máxima é igual ao nosso peso atual, temos duas opções.

  • Escolhemos o item e somamos seu valor com o valor máximo que podemos obter de outros itens restantes após pegar este item.
  • Podemos excluir este item.

Entre as duas opções, escolheremos aquela da qual podemos obter o valor máximo. Se selecionarmos o item, obteremos: valor para este item + valor máximo do restante dos itens após pegar este item = 4 + 0 = 4. Obtemos 4(valor do item) de nossa matriz peso e o 0(valor máximo que podemos obter do restante dos itens após pegar este item) vem **1 ** um passo acima e 3 um passo atrás. Isso é Tabela[0][0]. Por quê? Se pegarmos o item, nossa capacidade restante será 3 - 3 = 0 e o item restante será o primeiro item. Bem, se você se lembra, Table[0][0] armazena o valor máximo que podemos obter se nossa capacidade fosse 0 e tivéssemos apenas o primeiro item. Agora, se não selecionarmos o item, o valor máximo que podemos obter vem de 1 etapa acima, ou seja, 1. Agora pegamos o máximo desses dois valores (4, 1) e definimos Table[1][2] = 4. Para Tabela[1][4], como 4, a capacidade máxima da mochila é maior que 3, o peso do nosso item atual, temos novamente duas opções. Tomamos max(Peso[2] + Tabela[0][4-Peso[2]], Tabela[0][4]) = max (Peso[2] + Tabela[0][1], Tabela[0][4]) = max(4 + 1, 1) = 5.

if weight[i] <= j
    w := weight[i]
    Table[i][j] := max(w + Table[i][j-w], Table[i-1][j])
end if

Usando essas duas fórmulas, podemos preencher todo o array Table. Nosso array ficará assim:

+-------+--------+---+---+---+---+---+---+---+---+
| Value | Weight | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
+-------+--------+---+---+---+---+---+---+---+---+
|   1   |    1   | 0 | 1 | 1 | 1 | 1 | 1 | 1 | 1 |
+-------+--------+---+---+---+---+---+---+---+---+
|   4   |    3   | 0 | 1 | 1 | 4 | 5 | 5 | 5 | 5 |
+-------+--------+---+---+---+---+---+---+---+---+
|   5   |    4   | 0 | 1 | 1 | 4 | 5 | 6 | 6 | 9 |
+-------+--------+---+---+---+---+---+---+---+---+
|   7   |    5   | 0 | 1 | 1 | 4 | 5 | 7 | 8 | 9 |
+-------+--------+---+---+---+---+---+---+---+---+

Aqui, o último valor que inserimos em nosso array, Table[3][7] contém nosso valor máximo necessário. Este é o valor máximo que podemos obter se tivéssemos 4 itens e nossa capacidade máxima da mochila fosse 7.

Aqui uma coisa devemos lembrar que, mesmo para a primeira fila, o peso pode ser maior que a capacidade da mochila. Precisaremos manter outra restrição para verificar o valor enquanto preenchemos a primeira linha. Ou podemos simplesmente pegar outra linha e definir todos os valores da primeira linha como 0. O pseudo-código ficaria assim:

Procedure Knapsack(Weight, Value, maxCapacity):
n := Item.size - 1
Table[n+1][maxCapacity+1]
for i from 0 to n
    Table[i][0] := 0
end for
for j from 1 to maxCapacity
    if j >= Weight[0]
        Table[0][j] := Weight[0]
    else
        Table[0][j] := 0
    end if
end for
for i from 1 to n
    for j from 1 to maxCapacity
        if Weight[i] >= j                                            //can't pick the item
            Table[i][j] := Table[i-1][j]        
        else                                                        //can pick the item
            w := Weight[i]
            Table[i][j] := max(w + Table[i-1][j-w], Table[i-1][j])
        end if
    end for
end for
Return Table[n][maxCapacity]

A complexidade de tempo deste algoritmo é O(n*maxCapacity), onde n é o número de itens e maxCapacity é a capacidade máxima da nossa mochila.

Até agora, encontramos o valor máximo que podemos obter do nosso exemplo. Uma pergunta ainda permanece. Quais são os itens reais? Vamos refazer os valores em nosso array Table para descobrir os itens que pegamos. Seguiremos duas estratégias:

  • Para qualquer item, se o valor for proveniente da posição acima, não levamos o item atual. Vamos 1 passo acima.
  • Caso o valor não seja proveniente da posição acima, levamos o item. Então vamos 1 passo acima e x passos para trás onde x é o peso do item atual.

O pseudo-código será:

Procedure printItems(Table, maxCapacity, Value):
i := Item.size - 1
j := maxCapacity
while j is not equal to 0
    if Table[i][j] is equal to Table[i-1][j]
        i := i - 1
    else
        Print: i
        j := j - weight[i]
        i := i - 1
    end if
end while
