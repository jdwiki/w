Multiplicação da Cadeia de Matrizes

  • Solução recursiva

Nesta página

  • Solução recursiva

Solução recursiva #

Multiplicação da cadeia de matrizes é um problema de otimização que pode ser resolvido usando programação dinâmica. Dada uma sequência de matrizes, o objetivo é encontrar a maneira mais eficiente de multiplicar essas matrizes. O problema não é realmente realizar as multiplicações, mas apenas decidir a sequência das multiplicações de matrizes envolvidas.

Digamos que temos duas matrizes A1 e A2 de dimensão m * n e p * q . Pelas regras da multiplicação de matrizes, sabemos que,

  1. Podemos multiplicar A1 e A2 se e somente se n = p. Isso significa que o número de colunas de A1 deve ser igual ao número de linhas de A2.
  2. Se a primeira condição for satisfeita e multiplicarmos A1 e A2, obteremos uma nova matriz, vamos chamar é A3 de dimensão m * q.
  3. Para multiplicar A1 e A2, precisamos fazer algumas multiplicações de escala. O número total de multiplicações de escala que precisamos realizar é (m * n * q) ou (m * p * q ).
  4. A1 * A2 não é igual a A2 * A 1.

Se tivermos três matrizes A1, A2 e A3 com dimensão ** m** * n, n * p e p * q respectivamente, então A4 = A1 * A2 * A3 terá dimensão de m * q. Agora podemos realizar esta multiplicação de matrizes de duas maneiras:

  • Primeiro multiplicamos A1 e A2 e, com o resultado, multiplicamos *A3 *. Ou seja: (A1 * A2) * A3.
  • Primeiro, multiplicamos A2 e A3 e, com o resultado, multiplicamos *A1 *. Ou seja: A1 * (A2 * A3).

Você pode notar que a ordem da multiplicação permanece a mesma, ou seja, não multiplicamos (A1 * A3) * ** A2** porque pode não ser válido. Apenas alteramos o parêntese para multiplicar um conjunto antes de multiplicá-lo pelo restante. Como colocamos esses parênteses são importantes. Por quê? Digamos que a dimensão de 3 matrizes A1, A2 e A3 são 10 * 100, 100 * 5, 5 * 50. Então,

  1. Para (A1 * A2) * A3, o número total de multiplicações de escala são: (10 * 100 * 5) + (10 * 5 * 50) = 7500 vezes.
  2. Para A1 * (A2 * A3), o número total de multiplicações de escala são: (100 * 5 * 50) + (10 * 100 * 50) = 75000 vezes.

Para o 2º tipo, o número de multiplicação do scaler é 10 vezes o número do 1º tipo! Portanto, se você puder inventar uma maneira de descobrir a orientação correta dos parênteses necessária para minimizar a multiplicação total do scaler, isso reduziria o tempo e a memória necessários para a multiplicação de matrizes. É aqui que a multiplicação da cadeia de matrizes é útil. Aqui, não estaremos preocupados com a multiplicação real de matrizes, apenas descobriremos a ordem correta dos parênteses para que o número de multiplicação do escalonador seja minimizado. Teremos matrizes A1, A2, A3 …. …. An e descobriremos o número mínimo de multiplicações de escala necessárias para multiplicá-las. Vamos supor que as dimensões dadas são válidas, ou seja, satisfaz nosso primeiro requisito para a multiplicação de matrizes.

Usaremos o método de dividir e conquistar para resolver esse problema. A programação dinâmica é necessária devido a subproblemas comuns. Por exemplo: para n = 5, temos 5 matrizes A1, *A2 *, A3, A4 e A5. Queremos descobrir o número mínimo de multiplicação necessário para realizar esta multiplicação de matrizes A1 * A2 * A 3 * A4 * A5. Das muitas maneiras, vamos nos concentrar em uma: (A1 * A2) * (A3</ sub> * A4 * A5).

Para este, descobriremos Aesquerda = A1 * A2 . Adireito = A3 * A4 * A5< /sub>. Então descobriremos Aresposta = Aesquerda * Adireita. O número total de multiplicações de escalador necessárias para descobrir Aresposta: = O número total de multiplicações de escalador necessárias para determinar Aesquerda + O total número de multiplicações de escalador necessárias para determinar Adireita + O número total de multiplicações de escalador necessárias para determinar Aesquerda * A direito.

O último termo, O número total de multiplicações de escala necessárias para determinar Aesquerda * Adireita pode ser escrito como: O número de linhas em Aesquerda * o número de colunas na Aesquerda * o número de colunas na *Adireita *. (De acordo com a 2ª regra da multiplicação de matrizes)

Mas poderíamos definir os parênteses de outras maneiras também. Por exemplo:

  • A1 * (A2 * A3 * A 4 * A5)
  • (A1 * A2 * A3) * (A< sub>4 * A5)
  • (A1 * A2 * A3 * A 4) * A5 etc.

Para cada um dos casos possíveis, determinaremos o número de multiplicação do escalonador necessário para descobrir Aesquerda e Adireita e, em seguida, para Aesquerda * Adireita. Se você tem uma ideia geral sobre recursão, já entendeu como realizar essa tarefa. Nosso algoritmo será:

- Set parenthesis in all possible ways.
- Recursively solve for the smaller parts.
- Find out the total number of scaler multiplication by merging left and right.
- Of all possible ways, choose the best one.

Por que isso é problema de programação dinâmica? Para determinar (A1 * A2 * A3), se você já calculado (A1 * A2), será útil neste caso.

Para determinar o estado dessa recursão, podemos ver que para resolver cada caso, precisaremos conhecer o intervalo de matrizes com que estamos trabalhando. Precisamos de um início e um fim. Como estamos usando dividir e conquistar, nosso caso base terá menos de 2 matrizes (begin >= end), onde não precisamos multiplicar. Teremos 2 arrays: row e column. linha[i] e coluna[i] armazenarão o número de linhas e colunas da matriz Ai. Teremos um array dp[n][n] para armazenar os valores já calculados e inicializá-lo com -1, onde -1 representa o valor não calculado ainda. dp[i][j] representa o número de multiplicações do escalonador necessárias para multiplicar Ai, *Ai+1 *, …..,Aj inclusive. O pseudo-código ficará assim:

Procedure matrixChain(begin, end):
if begin >= end
    Return 0
else if dp[begin][end] is not equal to -1
    Return dp[begin][end]
end if
answer := infinity
for mid from begin to end
    operation_for_left := matrixChain(begin, mid)
    operation_for_right := matrixChain(mid+1, right)
    operation_for_left_and_right := row[begin] * column[mid] * column[end]
    total := operation_for_left + operation_for_right + operation_for_left_and_right
    answer := min(total, answer)
end for
dp[begin][end] := answer
Return dp[begin][end]

Complexidade:

O valor de begin e end pode variar de 1 a n. Existem n2 estados diferentes. Para cada estado, o loop interno será executado n vezes. Complexidade de tempo total: O(n³) e complexidade de memória: O(n²).
