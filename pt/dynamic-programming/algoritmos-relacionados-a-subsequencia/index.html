Subsequência crescente mais longa #

A tarefa é encontrar o comprimento da subsequência mais longa em uma determinada matriz de inteiros, de modo que todos os elementos da subsequência sejam classificados em ordem crescente. Por exemplo, o comprimento da subsequência crescente mais longa (LIS) para {15, 27, 14, 38, 26, 55, 46, 65, 85} é 6 e a subsequência crescente mais longa é * *{15, 27, 38, 55, 65, 85}**. Novamente para {3, 4, -1, 0, 6, 2, 3} o comprimento do LIS é 4 e a subsequência é {-1, 0, 2, 3}. Usaremos programação dinâmica para resolver este problema.

Vamos pegar o segundo exemplo: Item = {3, 4, -1, 0, 6, 2, 3}. Começaremos pegando um array dp do mesmo tamanho da nossa sequência. dp[i] representa o comprimento do LIS se incluirmos o iº item de nossa sequência original. No início, sabemos que para cada item, pelo menos, a subsequência crescente mais longa tem comprimento 1. Isso é considerando o próprio elemento único. Portanto, inicializaremos o array dp com 1. Teremos duas variáveis ​​i e j. Inicialmente i será 1 e j será 0. Nosso array ficará assim:

           3     4    -1     0     6     2     3
+-------+-----+-----+-----+-----+-----+-----+-----+
| Index |  0  |  1  |  2  |  3  |  4  |  5  |  6  |
+-------+-----+-----+-----+-----+-----+-----+-----+
| Value |  1  |  1  |  1  |  1  |  1  |  1  |  1  |
+-------+-----+-----+-----+-----+-----+-----+-----+
           j     i

O número acima do array representa o elemento correspondente da nossa sequência. Nossa estratégia será:

if Item[i] > Item[j]
    dp[i] := dp[j] + 1

Isso significa que se o elemento em i for maior que o elemento em j, o comprimento do LIS que contém o elemento em j aumentará em comprimento 1 se incluirmos o elemento em eu com ele. Em nosso exemplo, para i = 1 e j = 0, Item[i] é maior que Item[j] . Então dp[i] = dp[j] + 1. Nosso array ficará assim:

           3     4    -1     0     6     2     3
+-------+-----+-----+-----+-----+-----+-----+-----+

| Value |  1  |  2  |  1  |  1  |  1  |  1  |  1  |
+-------+-----+-----+-----+-----+-----+-----+-----+
           j     i

Em cada etapa, aumentaremos j até i e, em seguida, redefiniremos j para 0 e incrementaremos i. Por enquanto, j atingiu i, então incrementamos i para 2.

Também podemos notar que dp[i] = dp[j] + 1 nos fornecerá 3, o que significa que se pegarmos o LIS para Item [j] e adicionar Item[i] com ele, obteremos um LIS melhor{3,4,6} do que antes de {3,6}. Então definimos dp[i] = dp[j] + 1. Então incrementamos j.

+-------+-----+-----+-----+-----+-----+-----+-----+

Para i = 4 e j = 2, Item[i] é maior que Item[j]. Mas, para este caso, se definirmos dp[i] = dp[j] + 1, obteremos 2, que é {-1, 6} não é o melhor{3,4,6} que podemos fazer usando Item[i]. Então descartamos este. Adicionaremos uma condição à nossa estratégia, ou seja:

if Item[i]>Item[j] and dp[i]<dp[j] + 1
    dp[i] := dp[j] + 1

Incrementamos j em 1. Seguindo essa estratégia, se completarmos nosso array dp, ele ficará assim:

           3     4    -1     0     6     2     3
+-------+-----+-----+-----+-----+-----+-----+-----+
| Index |  0  |  1  |  2  |  3  |  4  |  5  |  6  |
+-------+-----+-----+-----+-----+-----+-----+-----+
| Value |  1  |  2  |  1  |  2  |  3  |  3  |  4  |
+-------+-----+-----+-----+-----+-----+-----+-----+
                                         j     i

Agora vamos percorrer esta matriz e descobrir o valor máximo, que será o comprimento do LIS. Nosso pseudo-código será:

Procedure LISLength(Item):
n := Item.length
dp[] := new Array(n)
for i from 0  to n
    dp[i] := 1
end for
for i from 1 to n
    for j from 0 to i
        if Item[i]>Item[j] and dp[i]<dp[j] + 1
            dp[i] := dp[j] + 1
        end if
    end for
end for
l := -1
for i from 0 to n
    l := max(l, dp[i])
end for
Return l

A complexidade de tempo deste algoritmo é O(n²) onde n é o comprimento da sequência.

Para descobrir a sequência original, precisamos iterar para trás e combiná-la com nosso comprimento. O procedimento é:

Procedure LIS(Item, dp, maxLength):
i := Item.length
while dp[i] is not equal to maxLength
    i := i - 1
end while
s = new Stack()
s.push(Item[i])
maxLength := maxLength - 1
current := Item[i]
while maxLength is not equal to 0
    i := i-1
    if dp[i] := maxLength and Item[i] < current
        current := Item[i]
        s.push(current)
        maxLength := maxLength - 1
    end if
end while
while s is not empty
    x := s.pop
    Print(s)
end while

A complexidade de tempo deste algoritmo é: O(n).

Subsequência Palindrômica mais longa #

Dada uma string, qual é a subsequência palindrômica mais longa (LPS) dela? Vamos pegar uma string agbdba. O LPS desta string é abdba de comprimento 5. Lembre-se, já que estamos procurando por subsequência, os caracteres não precisam ser contínuos na string original. A substring palindrômica mais longa da sequência seria bdb de comprimento 3. Mas vamos nos concentrar na subsequência aqui. Vamos usar programação dinâmica para resolver este problema.

Primeiro, vamos pegar uma matriz 2D da mesma dimensão da nossa sequência original. Para nosso exemplo: S = "agbdba", usaremos o array dp[6][6]. Aqui, dp[i][j] representa o comprimento do LPS que podemos fazer se considerarmos os caracteres de s[i] a s[j]. Por exemplo. se nossa string fosse aa, dp[0][1] armazenaria 2. Agora vamos considerar diferentes comprimentos de nossa corda e descobrir o maior comprimento possível que podemos fazer com ela.

Comprimento = 1:
Aqui, estamos considerando apenas 1 caractere por vez. Então, se tivéssemos uma string de comprimento 1, qual é o LPS que podemos ter? Claro que a resposta é 1. Como armazená-lo? dp[i][j] em que i é igual a j representa uma string de comprimento 1. Portanto, definiremos dp[0][0], dp[1][1], dp[2][2], dp [3][3], dp[4][4], dp[5][5] a 1. Nosso array ficará assim:

+---+---+---+---+---+---+---+
|   | 0 | 1 | 2 | 3 | 4 | 5 |
+---+---+---+---+---+---+---+
| 0 | 1 |   |   |   |   |   |
+---+---+---+---+---+---+---+
| 1 |   | 1 |   |   |   |   |
+---+---+---+---+---+---+---+
| 2 |   |   | 1 |   |   |   |
+---+---+---+---+---+---+---+
| 3 |   |   |   | 1 |   |   |
+---+---+---+---+---+---+---+
| 4 |   |   |   |   | 1 |   |
+---+---+---+---+---+---+---+
| 5 |   |   |   |   |   | 1 |
+---+---+---+---+---+---+---+

Comprimento = 2:
Desta vez, consideraremos strings de comprimento 2. Agora, considerando strings de comprimento 2, o comprimento máximo de LPS pode ser 2 se e somente se os dois caracteres da string forem iguais. Então nossa estratégia será:

j := i + Length - 1
if s[i] is equal to s[j]
    dp[i][j] := 2
else
    dp[i][j] := 1

Se preenchermos nosso array seguindo a estratégia para Length = 2, obteremos:

+---+---+---+---+---+---+---+
|   | 0 | 1 | 2 | 3 | 4 | 5 |
+---+---+---+---+---+---+---+
| 0 | 1 | 1 |   |   |   |   |
+---+---+---+---+---+---+---+
| 1 |   | 1 | 1 |   |   |   |
+---+---+---+---+---+---+---+
| 2 |   |   | 1 | 1 |   |   |
+---+---+---+---+---+---+---+
| 3 |   |   |   | 1 | 1 |   |
+---+---+---+---+---+---+---+
| 4 |   |   |   |   | 1 | 1 |
+---+---+---+---+---+---+---+
| 5 |   |   |   |   |   | 1 |
+---+---+---+---+---+---+---+

Comprimento = 3:
Agora estamos analisando 3 caracteres de cada vez para nossa string original. A partir de agora, o LPS que podemos fazer a partir de nossa string será determinado por:

  • Se o primeiro e o último caractere corresponderem, teremos pelo menos 2 itens dos quais podemos fazer o LPS + se excluirmos o primeiro e o último caractere, qualquer que seja o melhor que pudermos fazer com a string restante.
  • Se o primeiro e o último caractere não corresponderem, o LPS que podemos fazer virá da exclusão do primeiro caractere ou do último caractere, que já calculamos.

Para fazer verão,

j := i + Length - 1
if s[i] is equal to s[j]
    dp[i][j] := 2 + dp[i+1][j-1]
else
    dp[i][j] := max(dp[i+1][j], dp[i][j-1])
end if

Se preenchermos o array dp de Length = 3 a Length = 6, obteremos:

+---+---+---+---+---+---+---+
|   | 0 | 1 | 2 | 3 | 4 | 5 |
+---+---+---+---+---+---+---+
| 0 | 1 | 1 | 1 | 1 | 3 | 5 |
+---+---+---+---+---+---+---+
| 1 |   | 1 | 1 | 1 | 3 | 3 |
+---+---+---+---+---+---+---+
| 2 |   |   | 1 | 1 | 3 | 3 |
+---+---+---+---+---+---+---+
| 3 |   |   |   | 1 | 1 | 1 |
+---+---+---+---+---+---+---+
| 4 |   |   |   |   | 1 | 1 |
+---+---+---+---+---+---+---+
| 5 |   |   |   |   |   | 1 |
+---+---+---+---+---+---+---+

Este é nosso array dp obrigatório e dp[0][5] conterá o comprimento do LPS. Nosso procedimento ficará assim:

Procedure LPSLength(S):
n := S.length
dp[n][n]
for i from 0 to n
    dp[i][i] := 1
end for
for i from 0 to (n-2)
    if S[i] := S[i+1]
        dp[i][i+1] := 2
    else
        dp[i][i+1] := 1
    end if
end for
Length := 3
while Length <= n
    for i from 0 to (n - Length)
        j := i + Length - 1
        if S[i] is equal to s[j]
            dp[i][j] := 2 + dp[i+1][j-1]
        else
            dp[i][j] := max(dp[i+1][j], dp[i][j-1])
        end if
    end for
Length := Length + 1
end while
Return dp[0][n-1]

A complexidade de tempo desse algoritmo é O(n²), onde n é o comprimento de nossa string fornecida. O problema da subsequência palindrômica mais longa está intimamente relacionado à subsequência comum mais longa. Se tomarmos a segunda string como o inverso da primeira string e calcularmos o comprimento e imprimirmos o resultado, essa será a subsequência palindrômica mais longa da string dada. A complexidade desse algoritmo também é O(n²).

Subsequência comum mais longa #

Uma das implementações mais importantes da Programação Dinâmica é descobrir a Longest Common Subsequence. Vamos definir algumas das terminologias básicas primeiro.

Subsequência:

Uma subsequência é uma sequência que pode ser derivada de outra sequência pela exclusão de alguns elementos sem alterar a ordem dos elementos restantes. Digamos que temos uma string ABC. Se apagarmos zero ou um ou mais de um caractere desta string, obtemos a subsequência dessa string. Portanto, as subsequências da string ABC serão {“A”, “B”, “C”, “AB”, **“AC” **, “BC”, “ABC”, " “}. Mesmo se removermos todos os caracteres, a string vazia também será uma subsequência. Para descobrir a subsequência, para cada caractere em uma string, temos duas opções - ou pegamos o caractere ou não. Portanto, se o comprimento da string for n, haverá 2n subsequências dessa string.

Mais longa subsequência comum:

Como o nome sugere, de todas as subsequências comuns entre duas strings, a subsequência comum mais longa (LCS) é aquela com o comprimento máximo. Por exemplo: as subsequências comuns entre “HELLOM” e “HMLD” são “H”, “HL”, “HM” etc. Aqui * “HLL”* é a subsequência comum mais longa que tem comprimento 3.

Método de Força Bruta:

Podemos gerar todas as subsequências de duas strings usando backtracking. Então podemos compará-los para descobrir as subsequências comuns. Depois precisamos descobrir aquele com o comprimento máximo. Já vimos que existem 2n subsequências de uma string de comprimento n. Levaria anos para resolver o problema se nosso n cruzasse 20-25.

Método de programação dinâmica:

Vamos abordar nosso método com um exemplo. Suponha que temos duas strings abcdaf e acbcf. Vamos denotar isso com s1 e s2. Portanto, a subsequência comum mais longa dessas duas strings será “abcf”, que tem comprimento 4. Mais uma vez, lembro que as subsequências não precisam ser contínuas na string. Para construir “abcf”, ignoramos “da” em s1 e “c” em s2. Como descobrimos isso usando a Programação Dinâmica?

Começaremos com uma tabela (um array 2D) com todos os caracteres de s1 em uma linha e todos os caracteres de s2 na coluna. Aqui a tabela é indexada em 0 e colocamos os caracteres de 1 em diante. Percorreremos a tabela da esquerda para a direita para cada linha. Nossa tabela ficará assim:

A coluna 0 representa a subsequência vazia de s1. Da mesma forma, a linha 0 representa a subsequência vazia de s2. Se pegarmos uma subsequência vazia de uma string e tentarmos combiná-la com outra string, não importa o tamanho da segunda subsequência, a subsequência comum terá comprimento 0. Assim, podemos preencher as 0-th linhas e 0-th colunas com 0’s. Nós temos:

Vamos começar. Quando estamos preenchendo a Tabela[1][1], nos perguntamos: se tivéssemos uma string a e outra a e nada mais, qual seria a mais longa subsequência comum aqui? O comprimento do LCS aqui será 1. Agora vamos ver Tabela[1][2]. Temos a string ab e a string a. O comprimento do LCS será 1. Como você pode ver, o restante dos valores também será 1 para a primeira linha, pois considera apenas a string a com abcd, abcda, abcdaf. Assim nossa tabela ficará assim:

Para a linha 2, que agora incluirá c. Para Tabela[2][1] temos ac de um lado e a do outro. Então o comprimento do LCS é 1. De onde tiramos esse 1? A partir do topo, o que denota o LCS a entre duas substrings. Então, o que estamos dizendo é que, se s1[2] e s2[1] não forem iguais, o comprimento do LCS será o máximo do comprimento do LCS no *top *, ou à esquerda. Pegar o comprimento do LCS na parte superior indica que não pegamos o caractere atual de s2. Da mesma forma, tomar o comprimento do LCS à esquerda indica que não pegamos o caractere atual de s1 para criar o LCS. Nós temos:

if s2[i] is not equal to s1[j]

Continuando, para Table[2][2] temos a string ab e ac. Como c e b não são iguais, colocamos o máximo da parte superior ou esquerda aqui. Nesse caso, é novamente 1. Depois disso, para Tabela[2][3] temos a string abc e ac. Desta vez, os valores atuais da linha e da coluna são os mesmos. Agora o comprimento do LCS será igual ao comprimento máximo do LCS até agora + 1. Como obtemos o comprimento máximo do LCS até agora? Verificamos o valor diagonal, que representa a melhor correspondência entre ab e a. A partir desse estado, para os valores atuais, adicionamos mais um caractere a s1 e s2 que por acaso eram os mesmos. Portanto, o comprimento do LCS obviamente aumentará. Colocaremos 1 + 1 = 2 em Tabela[2][3]. Nós temos,

O comprimento do LCS entre s1 e s2 será Table[5][6] = 4. Aqui, 5 e 6 são o comprimento de s2 e s1, respectivamente. Nosso pseudo-código será:

Procedure LCSlength(s1, s2):
Table[0][0] = 0
for i from 1 to s1.length
    Table[0][i] = 0
endfor
for i from 1 to s2.length
    Table[i][0] = 0
endfor
for i from 1 to s2.length
    for j from 1 to s1.length
        if s2[i] equals to s1[j]
            Table[i][j] = Table[i-1][j-1] + 1
        else
            Table[i][j] = max(Table[i-1][j], Table[i][j-1])
        endif
    endfor
endfor
Return Table[s2.length][s1.length]

A complexidade de tempo para este algoritmo é: O(mn) onde m e n denotam o comprimento de cada string.

Como descobrimos a subsequência comum mais longa? Começaremos do canto inferior direito. Iremos verificar de onde vem o valor. Se o valor estiver vindo da diagonal, ou seja, se Table[i-1][j-1] for igual a Table[i][j] - 1, empurramos s2 [i] ou s1[j] (ambos são iguais) e se movem na diagonal. Se o valor vem de cima, significa que se Table[i-1][j] for igual a Table[i][j], vamos para o topo. Se o valor vier da esquerda, significa que se Table[i][j-1] for igual a Table[i][j], nos movemos para a esquerda. Quando chegamos à coluna mais à esquerda ou mais alta, nossa busca termina. Em seguida, retiramos os valores da pilha e os imprimimos. O pseudo-código:

Procedure PrintLCS(LCSlength, s1, s2)
temp := LCSlength
S = stack()
i := s2.length
j := s1.length
while i is not equal to 0 and j is not equal to 0
     if Table[i-1][j-1] == Table[i][j] - 1 and s1[j]==s2[i]
        S.push(s1[j])   //or S.push(s2[i])
        i := i - 1
        j := j - 1
    else if Table[i-1][j] == Table[i][j]
        i := i-1
    else
        j := j-1
    endif
endwhile
while S is not empty
    print(S.pop)
endwhile

Ponto a ser observado: se Table[i-1][j] e Table[i][j-1] forem iguais a Table[i][j] e * Tabela[i-1][j-1]* não é igual a Tabela[i][j] - 1, pode haver dois LCS para aquele momento. Este pseudo-código não considera esta situação. Você terá que resolver isso recursivamente para encontrar vários LCSs.

A complexidade de tempo para este algoritmo é: O(max(m, n)).
