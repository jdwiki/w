Algoritmo Floyd-Warshall #

Cobertura Mínima de Vértices #

Cobertura mínima de vértices é um problema clássico de grafos. Digamos que em uma cidade temos algumas estradas conectando alguns pontos. Vamos representar as estradas usando arestas e os pontos usando nós. Vamos pegar dois gráficos de exemplo:

Queremos colocar vigias em alguns pontos. Um vigia pode vigiar todas as estradas conectadas ao ponto. O problema é: qual é o número mínimo de vigias necessários para cobrir todas as estradas? Se colocarmos vigias no nó A, B, H, I e J, podemos cobrir todas as estradas.

Esta é a nossa solução ideal. Precisamos de pelo menos 5 vigias para guardar toda a cidade. Como determinar isso?

Primeiramente, precisamos entender que este é um problema NP-difícil, ou seja, o problema não tem solução em tempo polinomial. Mas se o grafo fosse uma Árvore, isso significa que se ele tivesse (n-1) nós onde n é o número de arestas e não há ciclo no grafo, podemos resolver usando programação dinâmica.

Para construir uma solução de DP, precisamos seguir duas estratégias:

  1. Se não houver vigia em um nó, todos os nós conectados a ele devem ter um vigia, ou todas as estradas não serão cobertas. Se u e v estiverem conectados e u não tiver um vigia, v deverá ter um vigia.
  2. Se houver um vigia em um nó, um nó diferente conectado a ele pode ou não ter um vigia. Isso significa que não é necessário ter um vigia, mas pode ser benéfico. Se u e v estiverem conectados e u tiver um vigia, verificaremos e descobriremos qual é benéfico para nós:
    • Setting watchman in v.
    • Not setting watchman in v.

Vamos definir uma função recursiva com estado sendo o nó atual em que estamos e se ele tem um vigia ou não. Aqui:

F(u,1) = Currently we're in 'u' node and there is a watchman in this node.
F(u,0) = Currently we're in 'u' node and there is no watchman in this node.

A função retornará o número de vigias nos nós restantes.

Vamos pegar uma árvore de exemplo:

Podemos dizer facilmente que se não colocarmos vigia no nó-A, teremos que colocar vigias no nó-B, C e D. Podemos deduzir:

F(A,0) = F(B,1) + F(C,1) + F(D,1) + 0

Ele nos retorna o número de vigias necessários se não colocarmos vigia no nó-A. Adicionamos 0 no final porque não configuramos nenhum vigia em nosso nó atual.

Agora F(A,1) significa que colocamos watchman em node-A. Para isso, podemos definir vigias em todos os nós conectados ou não. Pegaremos aquele que nos fornecer o número mínimo de vigias.

F(A,1) = min(F(B,0), F(B,1) + min(F(C,0), F(C,1)) + min(F(D,0), F(D,1)) + 1

Verificamos configurando e não configurando watchman em cada nó e obtendo o valor ideal.

Uma coisa que devemos ter cuidado é que, uma vez que vamos para o nó filho, nunca olharemos para o nó pai. No exemplo acima, passamos para B de A, então parent[B] = A. Portanto, não voltaremos para A de B.

Para determinar o caso base, podemos notar que, se de um nó não pudermos ir para nenhum outro novo nó, retornaremos 1 se houver um vigia em nosso nó atual, 0 se não houver vigia em nosso nó atual.

É melhor ter uma lista de adjacências para nossa árvore. Deixe a lista ser denotada por aresta. Teremos um array dp[n][2], onde n denota o número de nós para armazenar os valores calculados e inicializá-lo com -1. Também teremos um array parent[n] para denotar a relação pai e filho entre os nós. Nosso pseudo-código ficará assim:

Procedure f(u, isGuarded):
if edge[u].size is equal to 0                    //node doesn't have any new edge
    Return isGuarded
else if dp[u][isGuarded] is not equal to -1      //already calculated
    Return dp[u][isGuarded]
end if
sum := 0
for i from i to edge[u].size
    v := edge[u][i]
    if v is not equal to parent[u]               //not a parent
        parent[v] := u
        if isGuarded equals to 0                 //not guarded, must set a watchman
            sum := sum + f(v,1)
        else                                     //guarded, check both
            sum := sum + min(f(v,1), f(v,0)
        end if
    end if
end for
dp[u][isGuarded] := sum + isGuarded
Return dp[u][isGuarded]

Se denotarmos node-A como root, chamaremos a função por: min(f(A,1), f(A,0)). Isso significa que também verificaremos se é ideal definir watchman no nó raiz ou não. Esta é a nossa solução DP. Este problema também pode ser resolvido usando algoritmo de correspondência máxima ou fluxo máximo.
