Array como lista #

O array é um dos tipos básicos de variáveis ​​do Perl. Ele contém uma lista, que é uma sequência ordenada de zero ou mais escalares. A matriz é a variável que contém (e fornece acesso a) os dados da lista, conforme documentado em perldata.

Você pode atribuir uma lista a uma matriz:

my @foo = ( 4, 5, 6 );

Você pode usar uma matriz onde quer que uma lista seja esperada:

join '-', ( 4, 5, 6 );
join '-', @foo;

Alguns operadores trabalham apenas com arrays, pois alteram a lista que um array contém:

shift @array;
unshift @array, ( 1, 2, 3 );
pop @array;
push @array, ( 7, 8, 9 );

Atribuindo uma lista a um hash #

As listas também podem ser atribuídas a variáveis ​​de hash. Ao criar uma lista que será atribuída a uma variável hash, é recomendável usar a fat comma => entre chaves e valores para mostrar sua relação:

my %hash = ( foo => 42, bar => 43, baz => 44 );

O => é realmente apenas uma vírgula especial que cita automaticamente o operando à sua esquerda. Então, você poderia usar vírgulas normais, mas a relação não é tão clara:

my %hash = ( 'foo', 42, 'bar', 43, 'baz', 44 );

Você também pode usar strings entre aspas para o operando esquerdo da vírgula =>, o que é especialmente útil para chaves contendo espaços.

my %hash = ( 'foo bar' => 42, 'baz qux' => 43 );

Para obter detalhes, consulte Operador de vírgula em perldoc perlop.

Listas podem ser passadas para sub-rotinas #

Para passar a lista para uma sub-rotina, você especifica o nome da sub-rotina e então fornece a lista para ela:

test_subroutine( 'item1', 'item2' );
test_subroutine  'item1', 'item2';     # same

Internamente, o Perl cria aliases para esses argumentos e os coloca no array @_ que está disponível na sub-rotina:

@_ =  ( 'item1', 'item2' ); # Done internally by perl

Você acessa argumentos de sub-rotina assim:

sub test_subroutine {
    print $_[0]; # item1
    print $_[1]; # item2
}

Aliasing permite alterar o valor original do argumento passado para a sub-rotina:

sub test_subroutine {
    $_[0] +=  2;
}

my $x =  7;
test_subroutine( $x );
print $x; # 9

Para evitar alterações inadvertidas de valores originais passados ​​para sua sub-rotina, você deve copiá-los:

sub test_subroutine {
    my( $copy_arg1, $copy_arg2 ) =  @_;
    $copy_arg1 += 2;
}

my $x =  7;
test_subroutine $x; # in this case $copy_arg2 will have `undef` value
print $x; # 7

Para testar quantos argumentos foram passados ​​para a sub-rotina, verifique o tamanho de @_

sub test_subroutine {
    print scalar @_, ' argument(s) passed into subroutine';
}

Se você passar argumentos de array para uma sub-rotina, todos eles serão achatados:

my @x =  ( 1, 2, 3 );
my @y =  qw/ a b c /; # ( 'a', 'b', 'c' )
test_some_subroutine @x, 'hi', @y; # 7 argument(s) passed into subroutine
# @_ =  ( 1, 2, 3, 'hi', 'a', 'b', 'c' ) # Done internally for this call

Se seu test_some_subroutine contiver a instrução $_[4] = 'd', para a chamada acima ela fará com que $y[0] tenha valor d depois:

print "@y"; # d b c

Retorna a lista da sub-rotina #

Você pode, é claro, retornar listas de subs:

sub foo {
    my @list1 =  ( 1, 2, 3 );
    my @list2 =  ( 4, 5 );

    return    ( @list1, @list2 );
}

my @list =  foo();
print @list;          # 12345

Mas não é a maneira recomendada de fazer isso, a menos que você saiba o que está fazendo.

Embora isso esteja correto quando o resultado está no contexto LIST, no contexto SCALAR as coisas não são claras. Vamos dar uma olhada na próxima linha:

print scalar foo();  # 2

Por que 2? O que está acontecendo?

  1. Como foo() foi avaliado no contexto SCALAR, esta lista ( @list1, @list2 ) também foi avaliada no contexto SCALAR
  2. No contexto SCALAR, LIST retorna seu último elemento. Aqui está @list2
  3. Novamente no contexto SCALAR, array @list2 retorna o número de seus elementos. Aqui é 2.

Na maioria dos casos, a estratégia correta retornará referências a estruturas de dados. Então, no nosso caso, devemos fazer o seguinte:

 return    ( \@list1, \@list2 );

Então o chamador faz algo assim para receber os dois arrayrefs retornados:

 my ($list1, $list2) = foo(...);

Hash como lista #

No contexto da lista, o hash é achatado.

my @bar =  ( %hash, %hash );

O array @bar é inicializado por uma lista de dois hashes %hash

  • ambos %hash são achatados
  • nova lista é criada a partir de itens achatados
  • O array @bar é inicializado por essa lista

É garantido que os pares de valores-chave vão juntos. As chaves são sempre indexadas pares, valores - ímpares. Não é garantido que os pares de valores-chave sejam sempre nivelados na mesma ordem:

my %hash =  ( a => 1, b => 2 );
print %hash; # Maybe 'a1b2' or 'b2a1'

Usando arrayref para passar array para sub #

O arrayref para @foo é \@foo. Isso é útil se você precisar passar um array e outras coisas para uma sub-rotina. Passar @foo é como passar vários escalares. Mas passar \@foo é um único escalar. Dentro da sub-rotina:

xyz(\@foo, 123);
...
sub xyz {
    my ($arr, $etc) = @_;
