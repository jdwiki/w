Um aviso sobre a codificação de nome de arquivo #

Vale a pena mencionar que a codificação de nome de arquivo não é apenas específica da plataforma, mas também específica do sistema de arquivos.

Nunca é totalmente seguro assumir (mas geralmente é) que só porque você pode codificar e gravar em um determinado nome de arquivo, quando você tentar abrir esse mesmo nome de arquivo para leitura, ele ainda será chamado da mesma coisa.

Por exemplo, se você escrever em um sistema de arquivos como FAT16 que não suporta unicode, seus nomes de arquivos podem ser traduzidos silenciosamente para formas compatíveis com ASCII.

Mas é ainda menos seguro assumir que um arquivo que você pode criar, ler e escrever por nomeação explícita será chamado da mesma forma quando consultado por outras chamadas, por exemplo, readdir pode retornar bytes diferentes para o seu nome de arquivo do que você especificou para open.

Em alguns sistemas como o VAX, você nem sempre pode assumir que readdir retornará o mesmo nome de arquivo que você especificou com open para nomes de arquivo tão simples quanto foo.bar, porque o nome de arquivo extensions pode ser desmembrado pelo SO.

Além disso, no UNIX, há um conjunto muito liberal de caracteres legais para nomes de arquivos que o sistema operacional permite, excluindo apenas / e \0, enquanto no Windows, existem intervalos específicos de caracteres que são proibidos em nomes de arquivos e causar erros.

Tenha muita cautela aqui, evite truques extravagantes com nomes de arquivos se você tiver uma escolha, e sempre faça testes para garantir que quaisquer truques extravagantes que você faça sejam consistentes.

Tenha o dobro de cuidado se estiver escrevendo código destinado a ser executado em plataformas fora de seu controle, como se estiver escrevendo código destinado a CPAN, e assuma pelo menos 5% de seu usuário base ficará presa usando alguma tecnologia antiga ou quebrada, seja por escolha, por acidente ou por poderes fora de seu controle, e que estes conspirarão para criar bugs para eles.

:codificação(utf8) vs :utf8 #

Como o UTF-8 é um dos formatos internos para representação de strings em Perl, a etapa de codificação/decodificação pode ser ignorada. Em vez de :encoding(utf-8), você pode simplesmente usar :utf8, se seus dados já estiverem em UTF-8. :utf8 pode ser usado com segurança com fluxos de saída, enquanto que para fluxos de entrada pode ser perigoso, porque causa inconsistência interna quando você tem seqüências de bytes inválidas. Além disso, usar :utf8 para entrada pode resultar em quebras de segurança, então o uso de :encoding(utf-8) é aconselhável.

Mais detalhes: Qual é a diferença entre :encoding e :utf8

UTF-8 vs utf8 vs UTF8 #

A partir do Perl v5.8.7, "UTF-8" (com traço) significa UTF-8 em sua forma estrita e consciente da segurança, enquanto "utf8" significa UTF-8 em sua forma liberal e solta.

Por exemplo, "utf8" pode ser usado para pontos de código que não existem em Unicode, como 0xFFFFFFFF. Da mesma forma, sequências de bytes UTF-8 inválidas como "\x{FE}\x{83}\x{BF}\x{BF}\x{BF}\x{BF}\x{BF}" irão decodificar em um ponto de código Unicode inválido (mas Perl válido) (0xFFFFFFFF) ao usar "utf8", enquanto a codificação "UTF-8" não permitiria a decodificação para pontos de código fora do intervalo de Unicode válido e forneceria uma caractere de substituição (0xFFFD) em vez disso.

Como os nomes de codificação não diferenciam maiúsculas de minúsculas, "UTF8" é o mesmo que "utf8" (ou seja, variante não restrita).

O pragma utf8: usando Unicode em suas fontes #

O pragma utf8 indica que o código fonte será interpretado como UTF-8. Claro, isso só funcionará se o seu editor de texto também estiver salvando a fonte como codificado em UTF-8.

Agora, literais de string podem conter caracteres Unicode arbitrários; os identificadores também podem conter Unicode, mas apenas caracteres semelhantes a palavras (consulte perldata e [perlrecharclass](http://perldoc.perl.org/ perlrecharclass.html#backslash-sequences) para obter mais informações):

use utf8;
my $var1 = '§я§©😄';      # works fine
my $я = 4;                # works since я is a word (matches \w) character
my $p§2 = 3;              # does not work since § is not a word character.
say "ya" if $var1 =~ /я§/; # works fine (prints "ya")

Observação: Ao imprimir texto no terminal, certifique-se de que ele seja compatível com UTF-8.*

Pode haver relações complexas e contra-intuitivas entre a codificação de saída e de origem. Executando em um terminal UTF-8, você pode descobrir que adicionar o pragma utf8 parece quebrar as coisas:

$ perl -e 'print "Møøse\n"'
Møøse
$ perl -Mutf8 -e 'print "Møøse\n"'
M��se
$ perl -Mutf8 -CO -e 'print "Møøse\n"'
Møøse

No primeiro caso, Perl trata a string como bytes brutos e os imprime assim. Como esses bytes são UTF-8 válidos, eles parecem corretos, embora o Perl não saiba realmente quais caracteres eles são (por exemplo, length("Møøse") retornará 7, não 5). Depois de adicionar -Mutf8, Perl decodifica corretamente a fonte UTF-8 para caracteres, mas a saída está no modo Latin-1 por padrão e imprimir Latin-1 em um terminal UTF-8 não funciona. Somente quando você alternar STDOUT para UTF-8 usando -CO a saída estará correta.

use utf8 não afeta a codificação de E/S padrão nem os identificadores de arquivo!

Criar nomes de arquivos #

Os exemplos a seguir usam a codificação UTF-8 para representar nomes de arquivos (e nomes de diretórios) no disco. Se você quiser usar outra codificação, você deve usar Encode::encode(...).

use v5.14;
# Make Perl recognize UTF-8 encoded characters in literal strings.
# For this to work: Make sure your text-editor is using UTF-8, so
# that bytes on disk are really UTF-8 encoded.
use utf8;

# Ensure that possible error messages printed to screen are converted to UTF-8.
# For this to work: Check that your terminal emulator is using UTF-8.
binmode STDOUT, ':utf8';
binmode STDERR, ':utf8';
    
my $filename = 'æ€'; # $filename is now an internally UTF-8 encoded string. 

# Note: in the following it is assumed that $filename has the internal UTF-8
#  flag set, if $filename is pure ASCII, it will also work since its encoding
#  overlaps with UTF-8. However, if it has another encoding like extended ASCII,
#  $filename will be written with that encoding and not UTF-8. 
# Note: it is not necessary to encode $filename as UTF-8 here
#  since Perl is using UTF-8 as its internal encoding of $filename already

# Example1  -- using open()
open ( my $fh, '>', $filename ) or die "Could not open '$filename': $!";
close $fh;

# Example2 -- using qx() and touch
qx{touch $filename};

# Example3 -- using system() and touch
system 'touch', $filename;

# Example4 -- using File::Touch
use File::Touch;
eval { touch( $filename ) }; die "Could not create file '$filename': $!" if $@;

Ler nomes de arquivos #

Perl não tenta decodificar nomes de arquivos retornados por funções ou módulos internos. Tais strings que representam nomes de arquivos devem sempre ser decodificadas explicitamente, para que Perl as reconheça como Unicode.

use v5.14;
use Encode qw(decode_utf8);

# Ensure that possible error messages printed to screen are converted to UTF-8.
# For this to work: Check that you terminal emulator is using UTF-8.
binmode STDOUT, ':utf8';
binmode STDERR, ':utf8';

# Example1  -- using readdir()
my $dir = '.';
opendir(my $dh, $dir) or die "Could not open directory '$dir': $!";
while (my $filename = decode_utf8(readdir $dh)) {
    # Do something with $filename
}
close $dh;

# Example2  -- using getcwd()
use Cwd qw(getcwd);
my $dir = decode_utf8( getcwd() );

# Example3  -- using abs2rel()
use File::Spec;
use utf8;
my $base = 'ø';
my $path = "$base/b/æ";
my $relpath = decode_utf8( File::Spec->abs2rel( $path, $base ) );
# Note: If you omit $base, you need to encode $path first:
use Encode qw(encode_utf8);
my $relpath = decode_utf8( File::Spec->abs2rel( encode_utf8( $path ) ) );

# Example4  -- using File::Find::Rule (part1 matching a filename)
use File::Find::Rule;
use utf8;
use Encode qw(encode_utf8);
my $filename = 'æ';
# File::Find::Rule needs $filename to be encoded
my @files = File::Find::Rule->new->name( encode_utf8($filename) )->in('.');
$_ = decode_utf8( $_ ) for @files;

# Example5  -- using File::Find::Rule (part2 matching a regular expression)
use File::Find::Rule;
use utf8;
my $pat = '[æ].$'; # Unicode pattern
# Note: In this case:  File::Find::Rule->new->name( qr/$pat/ )->in('.')
#  will not work since $pat is Unicode and filenames are bytes
#  Also encoding $pat first will not work correctly
my @files;
File::Find::Rule->new->exec( sub { wanted( $pat, \@files ) } )->in('.');
$_ = decode_utf8( $_ ) for @files;
sub wanted {
    my ( $pat, $files ) = @_;
    my $name = decode_utf8( $_ );
    my $full_name = decode_utf8( $File::Find::name );
    push @$files, $full_name if $name =~ /$pat/;
}

Nota: se você está preocupado com UTF-8 inválido nos nomes dos arquivos, o uso de decode_utf8( ... ) nos exemplos acima provavelmente deve ser substituído por decode( 'utf-8', ... ). Isso ocorre porque decode_utf8( ... ) é um sinônimo para decode( 'utf8', ... ) e há uma diferença entre as codificações utf-8 e utf8 (veja Observações abaixo para obter mais informações) onde utf-8 é mais rigoroso sobre o que é aceitável do que utf8.

Interruptores de linha de comando para one-liners #

Ativa o pragma utf8 #

Para habilitar o pragma utf8 em uma linha, o interpretador perl deve ser chamado com a opção -Mutf8:

perl -Mutf8 -E 'my $人 = "human"; say $人'

Manipulação Unicode com switch -C #

O sinalizador de linha de comando -C permite controlar recursos Unicode. Ele pode ser seguido por uma lista de letras de opção.

E/S padrão #

  • I - STDIN estará em UTF-8

  • O - STDOUT estará em UTF-8

  • E - STDERR estará em UTF-8

  • S - abreviação de IOE, fluxos de E/S padrão estarão em UTF-8

    echo “Ματαιότης ματαιοτήτων” | perl -CS -Mutf8 -nE ‘say “ok” if /Ματαιότης/’

Argumentos do script #

  • A - trata @ARGV como uma matriz de strings codificadas UTF-8

    perl -CA -Mutf8 -E ‘my $arg = shift; say “anteater” if $arg eq “муравьед”’ муравьед

Camada PerlIO padrão #

  • i - UTF-8 é a camada PerlIO padrão para fluxos de entrada

  • o - UTF-8 é a camada PerlIO padrão para fluxos de saída

  • D - abreviação de io

    perl -CD -Mutf8 -e ‘open my $fh, “>”, “utf8.txt” or die $!; print $fh “개미 조심해”’

Os switches -M e -C podem ser combinados:

perl -CASD -Mutf8 -E 'say "Ματαιότης ματαιοτήτων\n"';

E/S padrão #

A codificação a ser usada para os manipuladores de arquivos de E/S padrão (STDIN, STDOUT e STDERR), pode ser definida separadamente para cada manipulador usando [binmode](http://perldoc.perl.org /functions/binmode.html):

binmode STDIN, ':encoding(utf-8)';
binmode STDOUT, ':utf8';
binmode STDERR, ':utf8';

Nota: ao ler, em geral, preferiria :encoding(utf-8) sobre :utf8, consulte Comentários para mais informações .

Alternativamente, você pode usar o pragma open.

# Setup such that all subsequently opened input streams will use ':encoding(utf-8)'
# and all subsequently opened output streams will use ':utf8'
# by default
use open (IN => ':encoding(utf-8)', OUT => ':utf8');
# Make the (already opened) standard file handles inherit the setting 
# given by the IO settings for the open pragma
use open ( :std );
# Now, STDIN has been converted to ':encoding(utf-8)', and 
# STDOUT and STDERR have ':utf8'

Alternativamente, para definir todos os identificadores de arquivo (tanto os que ainda não foram abertos quanto os padrão) para usar :encoding(utf-8):

use open qw( :encoding(utf-8) :std );

Manipuladores de arquivos #

Configurando a codificação com open() #

Ao abrir um arquivo de texto, você pode especificar sua codificação explicitamente com um open() de três argumentos. Este en-/decoder anexado a um identificador de arquivo é chamado de “camada de E/S”:

my $filename = '/path/to/file';
open my $fh, '<:encoding(utf-8)', $filename or die "Failed to open $filename: $!";

Veja Comentários para uma discussão das diferenças entre :utf8 e :encoding(utf-8).

Configurando a codificação com binmode() #

Alternativamente, você pode usar binmode() para definir a codificação para o identificador de arquivo individual:

my $filename = '/path/to/file';
open my $fh, '<', $filename or die "Failed to open $filename: $!";
binmode $fh, ':encoding(utf-8)';

#pragma aberto Para evitar definir a codificação para cada identificador de arquivo separadamente, você pode usar o pragma open para definir uma camada de E/S padrão usada por todas as chamadas subsequentes para o função open() e operadores semelhantes dentro do escopo lexical deste pragma:

# Set input streams to ':encoding(utf-8)' and output streams to ':utf8'
use open (IN => ':encoding(utf-8)', OUT => ':utf8');
# Or to set all input and output streams to ':encoding(utf-8)'
use open ':encoding(utf-8)';

Configurando a codificação com linha de comando -C sinalizador #

Finalmente, também é possível executar o interpretador perl com um sinalizador -CD que aplica UTF-8 como a camada de E/S padrão. No entanto, essa opção deve ser evitada, pois depende do comportamento específico do usuário que não pode ser previsto nem controlado.

Manipulando UTF-8 inválido #

Lendo UTF-8 inválido #

Ao ler dados codificados em UTF-8, é importante estar ciente de que os dados codificados em UTF-8 podem ser inválidos ou malformados. Esses dados geralmente não devem ser aceitos pelo seu programa (a menos que você saiba o que está fazendo). Ao encontrar dados malformados inesperadamente, diferentes ações podem ser consideradas:

  • Imprimir stacktrace ou mensagem de erro e abortar o programa normalmente, ou
  • Insira um caractere de substituição no local onde a sequência de bytes malformada apareceu, imprima uma mensagem de aviso para STDERR e continue lendo, pois nada aconteceu.

Por padrão, Perl irá avisar sobre falhas de codificação, mas não irá abortar seu programa. Você pode fazer seu programa abortar tornando os avisos UTF-8 fatais, mas esteja ciente das advertências em Avisos Fatais.

O exemplo a seguir grava 3 bytes na codificação ISO 8859-1 no disco. Em seguida, ele tenta ler os bytes novamente como dados codificados em UTF-8. Um dos bytes, 0xE5, é uma sequência de um byte UTF-8 inválida:

use strict;
use warnings;
use warnings FATAL => 'utf8';

binmode STDOUT, ':utf8';
binmode STDERR, ':utf8';
my $bytes = "\x{61}\x{E5}\x{61}";  # 3 bytes in iso 8859-1: aåa
my $fn = 'test.txt';
open ( my $fh, '>:raw', $fn ) or die "Could not open file '$fn': $!";
print $fh $bytes;
close $fh;
open ( $fh, "<:encoding(utf-8)", $fn ) or die "Could not open file '$fn': $!";
my $str = do { local $/; <$fh> };
close $fh;
print "Read string: '$str'\n";

O programa será abortado com um aviso fatal:

utf8 "\xE5" does not map to Unicode at ./test.pl line 10.

A linha 10 é aqui a penúltima linha, e o erro ocorre na parte da linha com <$fh> ao tentar ler uma linha do arquivo.

Se você não tornar os avisos fatais no programa acima, o Perl ainda imprimirá o aviso. No entanto, neste caso, ele tentará se recuperar do byte malformado 0xE5 inserindo os quatro caracteres \xE5 no fluxo e, em seguida, continuará com o próximo byte. Como resultado, o programa imprimirá:

Read string: 'a\xE5a'
