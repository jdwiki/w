Instalando módulos manualmente #

Se você não tem permissões para instalar módulos perl, você ainda pode instalá-los manualmente, indicando um caminho personalizado onde você tem permissões de escrita.

Punho, baixe e descompacte o arquivo do módulo:

wget module.tar.gz
tar -xzf module.tar.gz
cd module

Então, se a distribuição do módulo contiver um arquivo Makefile.PL, execute:

perl Makefile.PL INSTALL_BASE=$HOME/perl
make
make test
make install

ou se você tiver um arquivo Build.PL em vez de um Makefile.PL:

perl Build.PL --install_base $HOME/perl
perl Build
perl Build test
perl Build install

Você também deve incluir o caminho do módulo na variável de ambiente PERL5LIB para usá-lo em seu código:

export PERL5LIB=$HOME/perl

cpanminus, o substituto leve e sem configuração para o cpan #

Uso

Para instalar um módulo (assumindo que o cpanm já esteja instalado):

cpanm Data::Section

O cpanm (“cpanminus”) se esforça para ser menos detalhado que o cpan, mas ainda captura todas as informações de instalação em um arquivo de log caso seja necessário. Ele também lida com muitas “perguntas interativas” para você, enquanto o cpan não.

cpanm também é popular para instalar dependências de um projeto, por exemplo, GitHub. O uso típico é primeiro cd na raiz do projeto, depois executar

cpanm --installdeps .

Com --installdeps ele irá:

  1. Verifique e instale as dependências configure_requires de qualquer
    • META.json
    • META.yml (if META.json is missing)
  2. Construa o projeto (equivalente a perl Build.PL), gerando arquivos MYMETA
  3. Digitalize e instale requer dependências de qualquer
    • MYMETA.json
    • MYMETA.yml (if MYMETA.json is missing)

Para especificar o arquivo ‘some.cpanfile’, contendo as dependências, execute:

cpanm --installdeps --cpanfile some.cpanfile .

Instalação do cpanm

Existem várias maneiras de instalá-lo. Aqui está a instalação via cpan:

cpan App::cpanminus

Configuração do cpanm

Não existe nenhum arquivo de configuração para cpanm. Em vez disso, ele conta com as seguintes variáveis ​​de ambiente para sua configuração:

  • PERL_CPANM_OPT (Opções gerais de linha de comando cpanm)
    • export PERL_CPANM_OPT="--prompt" # in .bashrc, to enable prompting, e.g.
    • setenv PERL_CPANM_OPT "--prompt" # in .tcshrc
  • PERL_MM_OPT (ExtUtils::MakeMaker opções de linha de comando, afeta o destino de instalação do módulo)
  • PERL_MB_OPT (Module::Build command line options, afeta o destino de instalação do módulo)

Execute o Perl CPAN no seu terminal (Mac e Linux) ou prompt de comando (Windows) #

Linha de comando #

Você pode usar cpan para instalar módulos diretamente da linha de comando:

cpan install DBI

Isso seria seguido por possivelmente muitas páginas de saída descrevendo exatamente o que está fazendo para instalar o módulo. Dependendo dos módulos que estão sendo instalados, ele pode pausar e fazer perguntas.

Shell interativo #

Você também pode inserir um “shell” assim:

perl -MCPAN -e "shell"

Ele irá produzir saída como abaixo:

Terminal does not support AddHistory.

cpan shell -- CPAN exploration and modules installation (v2.00)
Enter 'h' for help.

cpan[1]>

Então você pode instalar os módulos que você quer pelo comando fácil install <module>.

Exemplo: cpan[1]> instalar DBI

Depois de instalar com sucesso, digite exit para sair.
