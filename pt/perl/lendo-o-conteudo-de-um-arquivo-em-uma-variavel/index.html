Caminho::Pequeno #

Usando o idioma de [The Manual Way](https://www.wikiod.com/pt/perl/lendo-o-conteudo-de-um-arquivo-em-uma-variavel#A maneira manual) várias vezes em um script logo fica tedioso, então você pode querer experimentar um módulo.

use Path::Tiny;
my $contents = path($filename)->slurp;

Você pode passar uma opção binmode se precisar de controle sobre codificações de arquivos, terminações de linha etc. - veja man perlio:

my $contents = path($filename)->slurp( {binmode => ":encoding(UTF-8)"} );

Path::Tiny também tem muitas outras funções para lidar com arquivos seja uma boa escolha.

A maneira manual #

open my $fh, '<', $filename
    or die "Could not open $filename for reading: $!";
my $contents = do { local $/; <$fh> };

Depois de abrir o arquivo (leia man perlio se você quiser ler codificações de arquivos específicos em vez de bytes brutos), o truque está no bloco do: <$fh>, o identificador do arquivo em um operador diamante, retorna um único registro do arquivo. A variável “input record separator” $/ especifica o que é um “registro”—por padrão, ele é definido como um caractere de nova linha, portanto, “um registro” significa “uma única linha”. Como $/ é uma variável global, local faz duas coisas: cria uma cópia local temporária de $/ que desaparecerá no final do bloco e fornece o valor (não) undef (o “valor” que Perl dá a variáveis ​​não inicializadas). Quando o separador de registro de entrada tiver esse valor (não), o operador de diamante retornará o arquivo inteiro. (Ele considera o arquivo inteiro como uma única linha.)

Usando do, você pode até abrir manualmente um arquivo. Para leitura repetida de arquivos,

sub readfile { do { local(@ARGV,$/) = $_[0]; <> } }
my $content = readfile($filename);

pode ser usado. Aqui, outra variável global (@ARGV) é localizada para simular o mesmo processo usado ao iniciar um script perl com parâmetros. $/ ainda é undef, já que o array na frente dele “come” todos os argumentos recebidos. Em seguida, o operador diamante <> entrega novamente um registro definido por $/ (o arquivo inteiro) e retorna do bloco do, que por sua vez retorna do sub.

O sub não tem tratamento de erros explícito, o que é uma má prática! Se ocorrer um erro durante a leitura do arquivo, você receberá undef como valor de retorno, em oposição a uma string vazia de um arquivo vazio.

Outra desvantagem do último código é o fato de que você não pode usar PerlIO para diferentes codificações de arquivo – você sempre obtém bytes brutos.

Arquivo::Slurp #

Não use. Embora já exista há muito tempo e ainda seja o módulo que a maioria dos programadores sugere, [está quebrado e provavelmente não será corrigido](http://blogs.perl.org/users/leon_timmermans/2015/08/fileslurp -é-quebrado-e-errado.html).

Arquivo::Slurper #

Este é um módulo minimalista que apenas transforma arquivos em variáveis, nada mais.

use File::Slurper 'read_text';
my $contents = read_text($filename);

read_text() usa dois parâmetros opcionais para especificar a codificação do arquivo e se os finais de linha devem ser traduzidos entre os padrões unixish LF ou DOSish CRLF:

my $contents = read_text($filename, 'UTF-8', 1);

Colocando um arquivo em uma variável de array #

open(my $fh, '<', "/some/path") or die $!;
my @ary = <$fh>;

Quando avaliado em contexto de lista, o operador diamante retorna uma lista composta por todas as linhas do arquivo (neste caso, atribuir o resultado a um array fornece contexto de lista). O terminador de linha é retido e pode ser removido mastigando:

chomp(@ary); #removes line terminators from all the array elements.

Slurp arquivo em uma linha #

O separador de registro de entrada pode ser especificado com a opção -0 (zero, não capital O). Recebe um número octal ou hexadecimal como valor. Qualquer valor 0400 ou superior fará com que o Perl sugue os arquivos, mas por convenção, o valor usado para este propósito é 0777.

perl -0777 -e 'my $file = <>; print length($file)' input.txt

Indo mais longe com o minimalismo, especificar a opção -n faz com que o Perl leia automaticamente cada linha (no nosso caso — o arquivo inteiro) na variável $_.

perl -0777 -ne 'print length($_)' input.txt
