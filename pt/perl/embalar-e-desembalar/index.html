Convertendo manualmente estruturas C em sintaxe de pacote #

Se você estiver lidando com APIs C Binárias do Perl Code, através das funções syscall, ioctl ou fcntl, você precisa saber como construir memória de uma maneira compatível com C.

Por exemplo, se você estivesse lidando com alguma função que esperasse um timespec, você procuraria em /usr/include/time.h e encontraria:

struct timespec
{
    __time_t tv_sec;            /* Seconds.  */
    __syscall_slong_t tv_nsec;  /* Nanoseconds.  */
};

Você faz uma dança com cpp para descobrir o que isso realmente significa:

cpp -E /usr/include/time.h -o /dev/stdout | grep __time_t
# typedef long int __time_t;
cpp -E /usr/include/time.h -o /dev/stdout | grep __syscall_slong_t
# typedef long int __syscall_slong_t

Então é um int (assinado)

echo 'void main(){ printf("%#lx\n", sizeof(__syscall_slong_t)); }' | 
  gcc -x c -include stdio.h -include time.h - -o /tmp/a.out && /tmp/a.out
# 0x8

E leva 8 bytes. Então, 64 bits assinado int. E eu estou em um processador de 64 bits. =)

Perldoc pack diz

q Um valor quad com sinal (64 bits).

Então, para empacotar um timespec:

sub packtime {
    my ( $config ) = @_; 
    return pack 'qq', @{$config}{qw( tv_sec tv_nsec )};
}

E para descompactar um timespec:

sub unpacktime {
   my ( $buf ) = @_;
   my $out = {};
   @{$out}{qw( tv_sec tv_nsec )} = unpack 'qq', $buf;
   return $out;
}

Agora você pode simplesmente usar essas funções.

my $timespec = packtime({ tv_sec => 0, tv_nsec => 0 });
syscall(  ..., $timespec ); # some syscall that reads timespec

later ...
syscall( ..., $timespec ); # some syscall that writes timespec
print Dumper( unpacktime( $timespec ));

Construindo um cabeçalho IPv4 #

Às vezes você tem que lidar com estruturas definidas em termos de tipos de dados C de Perl. Uma dessas aplicações é a criação de pacotes de rede brutos, caso você queira fazer algo mais sofisticado do que o que a API de soquete regular tem a oferecer. É exatamente para isso que pack() (e unpack() é claro) existe.

A parte obrigatória de um cabeçalho IP tem 20 octetos (também conhecido como “bytes”). Como você pode ver por trás deste link, os endereços IP de origem e destino compõem os dois últimos valores de 32 bits no cabeçalho. Entre os outros campos estão alguns com 16 bits, alguns com 8 bits e alguns pedaços menores entre 2 e 13 bits.

Assumindo que temos as seguintes variáveis ​​para colocar em nosso cabeçalho:

my ($dscp, $ecn, $length,
    $id, $flags, $frag_off,
    $ttl, $proto,
    $src_ip,
    $dst_ip);

Observe que estão faltando três campos do cabeçalho:

  • A versão é sempre 4 (afinal é IPv4)
  • IHL é 5 em nosso exemplo, pois não temos um campo options; comprimento é especificado em unidades de 4 octetos, então 20 octetos dá um comprimento de 5.
  • A soma de verificação pode ser deixada em 0. Na verdade, teríamos que calculá-la, mas o código para fazer isso não nos interessa aqui.

Poderíamos tentar usar operações de bits para construir, por exemplo, os primeiros 32 bits:

my $hdr = 4 << 28 | 5 << 24 | $dscp << 18 | $ecn << 16 | $length;

Essa abordagem só funciona até o tamanho de um inteiro, que geralmente é de 64 bits, mas pode ser tão baixo quanto 32. Pior, depende do [endianness] da CPU (https://en.wikipedia.org/wiki/Endianness ) para que funcione em algumas CPUs e falhe em outras. Vamos tentar pack():

my $hdr = pack('H2B8n', '45', sprintf("%06b%02b", $dscp, $ecn), $length);

O modelo primeiro especifica H2, uma string hexadecimal de 2 caracteres, nybble alto primeiro. O argumento correspondente ao pacote é “45”—versão 4, comprimento 5. O próximo modelo é B8, uma string de 8 bits, ordem de bits decrescente dentro de cada byte. Precisamos usar strings de bits para controlar o layout em pedaços menores que um nybble (4 bits), então o sprintf() é usado para construir tal string de bits a partir de 6 bits de $dscp e 2 de $ecn . O último é n, um valor não assinado de 16 bits em Network Byte Order, ou seja, sempre big-endian, não importa qual seja o formato inteiro nativo da sua CPU, e é preenchido a partir de $length.

Esses são os primeiros 32 bits do cabeçalho. O resto pode ser construído de forma semelhante:

Modelo Argumento Observações
n $id
B16 sprintf("%03b%013b", $flags, $frag_off) O mesmo que DSCP/ECN
C2 $ttl, $proto Dois octetos não assinados consecutivos
n 0 / $checksum x pode ser usado para inserir um byte nulo, mas n nos permite especificar um argumento se escolhermos calcular uma soma de verificação
N2 $src_ip, $dst_ip use a4a4 para empacotar o resultado de duas chamadas gethostbyname() como já está em Network Byte Order!

Portanto, a chamada completa para empacotar um cabeçalho IPv4 seria:

my $hdr = pack('H2B8n2B16C2nN2',
    '45', sprintf("%06b%02b", $dscp, $ecn), $length,
    $id, sprintf("%03b%013b", $flags, $frag_off),
    $ttl, $proto, 0,
    $src_ip, $dst_ip
);
