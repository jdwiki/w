Definindo classes em Perl moderno #

Embora disponível, definir uma classe [do zero](https://www.wikiod.com/pt/perl/perl-orientado-a-objetos#Definindo classes) não é recomendado no Perl moderno. Use um dos sistemas OO auxiliares que fornecem mais recursos e conveniência. Entre esses sistemas estão:

  • Moose - inspirado no design Perl 6 OO

  • Class::Accessor - uma alternativa leve ao Moose

  • Class::Tiny - construtor de classes verdadeiramente mínimo

Alce

package Foo;
use Moose;

has bar => (is => 'ro');                 # a read-only property
has baz => (is => 'rw', isa => 'Bool');  # a read-write boolean property

sub qux {
    my $self = shift;
    my $barIsBaz = $self->bar eq 'baz';  # property getter
    $self->baz($barIsBaz);               # property setter
}

Class::Accessor (sintaxe Moose)

package Foo;
use Class::Accessor 'antlers';

has bar => (is => 'ro');                 # a read-only property
has baz => (is => 'rw', isa => 'Bool');  # a read-write property (only 'is' supported, the type is ignored)

Class::Accessor (sintaxe nativa)

package Foo;
use base qw(Class::Accessor);

Foo->mk_accessors(qw(bar baz));  # some read-write properties
Foo->mk_accessors(qw(qux));      # a read-only property

Classe::Pequeno

package Foo;
use Class::Tiny qw(bar baz);  # just props

Criando objetos #

Ao contrário de muitas outras linguagens, Perl não possui construtores que alocam memória para seus objetos. Em vez disso, deve-se escrever um método de classe que crie uma estrutura de dados e a preencha com dados (você pode conhecê-lo como o padrão de projeto Factory Method).

package Point;
use strict;

sub new {
    my ($class, $x, $y) = @_;
    my $self = { x => $x, y => $y }; # store object data in a hash
    bless $self, $class;             # bind the hash to the class
    return $self;
}

Este método pode ser usado da seguinte forma:

my $point = Point->new(1, 2.5);

Sempre que o operador de seta -> é usado com métodos, seu operando esquerdo é anexado à lista de argumentos fornecida. Portanto, @_ em new conterá valores ('Point', 1, 2.5).

Não há nada de especial no nome novo. Você pode chamar os métodos de fábrica como preferir.

Não há nada de especial em hashes. Você poderia fazer o mesmo da seguinte maneira:

package Point;
use strict;

sub new {
    my ($class, @coord) = @_;
    my $self = \@coord;
    bless $self, $class;
    return $self;
}

Em geral, qualquer referência pode ser um objeto, até mesmo uma referência escalar. Mas, na maioria das vezes, os hashes são a maneira mais conveniente de representar dados de objetos.

Definindo classes #

Em geral, as classes em Perl são apenas pacotes. Eles podem conter dados e métodos, como pacotes usuais.

package Point;
use strict;

my $CANVAS_SIZE = [1000, 1000];

sub new {
    ...
}

sub polar_coordinates {
    ...
}

1;

É importante notar que as variáveis ​​declaradas em um pacote são variáveis ​​de classe, não variáveis ​​de objeto (instância). A alteração de uma variável de nível de pacote afeta todos os objetos da classe. Como armazenar dados específicos de objetos, veja em “Criando Objetos”.

O que torna os pacotes de classe específicos é o operador de seta ->. Pode ser usado depois de uma palavra simples:

Point->new(...);

ou depois de uma variável escalar (geralmente segurando uma referência):

my @polar = $point->polar_coordinates;

O que está à esquerda da seta é anexado à lista de argumentos fornecida do método. Por exemplo, após a chamada

Point->new(1, 2);

array @_ em new conterá três argumentos: ('Point', 1, 2).

Os pacotes que representam classes devem levar essa convenção em consideração e esperar que todos os seus métodos tenham um argumento extra.

Resolução de herança e métodos #

Para tornar uma classe uma subclasse de outra classe, use o pragma parent:

package Point;
use strict;
...
1;

package Point2D;
use strict;
use parent qw(Point);
...
1;

package Point3D;
use strict;
use parent qw(Point);
...
1;

Perl permite herança múltipla:

package Point2D;
use strict;
use parent qw(Point PlanarObject);
...
1;

A herança tem tudo a ver com a resolução de qual método deve ser chamado em uma situação específica. Como Perl puro não prescreve nenhuma regra sobre a estrutura de dados usada para armazenar dados de objetos, a herança não tem nada a ver com isso.

Considere a seguinte hierarquia de classes:

package GeometryObject;
use strict;

sub transpose { ...}

1;

package Point;
use strict;
use parent qw(GeometryObject);

sub new { ... };

1;

package PlanarObject;
use strict;
use parent qw(GeometryObject);

sub transpose { ... }

1;

package Point2D;
use strict;
use parent qw(Point PlanarObject);

sub new { ... }

sub polar_coordinates { ... }

1;

A resolução do método funciona da seguinte forma:

  1. O ponto inicial é definido pelo operando esquerdo do operador de seta.

  • Se for uma palavra vazia:

       Point2D->new(...);

    …or a scalar variable holding a string:

       my $class = 'Point2D';
   $class->new(...);

    …then the starting point is the package with the corresponding name (Point2D in both examples).

  • Se o operando esquerdo for uma variável escalar contendo uma referência blessed:

       my $point = {...};
   bless $point, 'Point2D'; # typically, it is encapsulated into class methods
   my @coord = $point->polar_coordinates;

    then the starting point is the class of the reference (again, Point2D). The arrow operator cannot be used to call methods for unblessed references.

  1. Se o ponto inicial contém o método requerido, ele é simplesmente chamado.

Assim, como Point2D::new existe,

   Point2D->new(...);

vai simplesmente chamá-lo.

  1. Se o ponto de partida não contiver o método necessário, a pesquisa em profundidade nas classes pai será executada. No exemplo acima, a ordem de busca será a seguinte:
  • Point2D
  • Point (primeiro pai de Point2D)
  • GeometryObject (pai de Point)
  • PlanarObject (segundo pai de Point2D)

Por exemplo, no código a seguir:

   my $point = Point2D->new(...);
   $point->transpose(...);

o método que será chamado é GeometryObject::transpose, mesmo que seja sobrescrito em PlanarObject::transpose.

  1. Você pode definir o ponto de partida explicitamente.

No exemplo anterior, você pode chamar explicitamente PlanarObject::transpose assim:

   my $point = Point2D->new(...);
   $point->PlanarObject::transpose(...);
  1. De maneira semelhante, SUPER:: executa a pesquisa de métodos nas classes pai da classe atual.

Por exemplo,

   package Point2D;
   use strict;
   use parent qw(Point PlanarObject);

   sub new {
       (my $class, $x, $y) = @_;
       my $self = $class->SUPER::new;
       ...
   }

   1;

irá chamar Point::new no decorrer da execução Point2D::new.

Métodos de classe e objeto #

Em Perl, a diferença entre métodos de classe (estático) e de objeto (instância) não é tão forte quanto em algumas outras linguagens, mas ainda existe.

O operando esquerdo do operador de seta -> torna-se o primeiro argumento do método a ser chamado. Pode ser uma string:

# the first argument of new is string 'Point' in both cases
Point->new(...);

my $class = 'Point';
$class->new(...);

ou uma referência de objeto:

# reference contained in $point is the first argument of polar_coordinates
my $point = Point->new(...);
my @coord = $point->polar_coordinates;

Métodos de classe são apenas aqueles que esperam que seu primeiro argumento seja uma string, e métodos de objeto são aqueles que esperam que seu primeiro argumento seja uma referência de objeto.

Métodos de classe normalmente não fazem nada com seu primeiro argumento, que é apenas um nome da classe. Geralmente, ele é usado apenas pelo próprio Perl para resolução de métodos. Portanto, um método de classe típico também pode ser chamado para um objeto:

my $width = Point->canvas_width;

my $point = Point->new(...);
my $width = $point->canvas_width;

Embora essa sintaxe seja permitida, geralmente é enganosa, portanto, é melhor evitá-la.

Os métodos de objeto recebem uma referência de objeto como o primeiro argumento, para que possam endereçar os dados do objeto (diferentemente dos métodos de classe):

package Point;
use strict;

sub polar_coordinates {
    my ($point) = @_;
    my $x = $point->{x};
    my $y = $point->{y};
    return (sqrt($x * $x + $y * $y), atan2($y, $x));
}

1;

O mesmo método pode rastrear ambos os casos: quando é chamado como um método de classe ou objeto:

sub universal_method {
    my $self = shift;
    if (ref $self) {
        # object logic
        ...
    }
    else {
        # class logic
        ...
    }
}

Funções #

Um papel em Perl é essencialmente

  • um conjunto de métodos e atributos que
  • injetado em uma classe diretamente.

Uma função fornece uma funcionalidade que pode ser composta em (ou aplicada a) qualquer classe (que se diz consumir a função). Uma função não pode ser herdada, mas pode ser consumida por outra função.

Uma função também pode requerer consumir classes para implementar alguns métodos em vez de implementar os próprios métodos (assim como interfaces em Java ou C#).

Perl não possui suporte embutido para funções, mas existem classes CPAN que fornecem tal suporte.

Moose::Role

package Chatty;
use Moose::Role;

requires 'introduce';  # a method consuming classes must implement

sub greet {            # a method already implemented in the role
    print "Hi!\n";
}


package Parrot;
use Moose;

with 'Chatty';

sub introduce {
    print "I'm Buddy.\n";
}

Role::Tiny

Use se o seu sistema OO não fornecer suporte para funções (por exemplo, Class::Accessor ou Class::Tiny). Não suporta atributos.

package Chatty;
use Role::Tiny;

requires 'introduce';  # a method consuming classes must implement

sub greet {            # a method already implemented in the role
    print "Hi!\n";
}

package Parrot;
use Class::Tiny;
use Role::Tiny::With;

with 'Chatty';

sub introduce {
    print "I'm Buddy.\n";
}
