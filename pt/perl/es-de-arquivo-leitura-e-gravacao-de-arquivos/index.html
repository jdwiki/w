Parâmetros #

Modo Explicação
> Escrever (truncar). Substituirá os arquivos existentes. Cria um novo arquivo se nenhum arquivo foi encontrado
>> Escrever (anexar). Não substituirá os arquivos, mas acrescentará novo conteúdo no final. Também criará um arquivo se usado para abrir um arquivo não existente.
< Ler. Abre o arquivo no modo somente leitura.
+< Ler escrever. Não criará ou truncará o arquivo.
+> Ler/Gravar (trunc). Irá criar e truncar o arquivo.
+>> Ler/Gravar (anexar). Irá criar mas não truncar o arquivo.

chomp é frequentemente usado ao ler um arquivo. Por padrão, ele corta o caractere de nova linha, embora para sua funcionalidade completa consulte o perldocs.

Cuidado com a diferença entre caracteres e bytes: Nem todas as codificações - especialmente UTF-8 - usam caracteres de 1 byte. Embora isso seja tratado com perfeição pelo PerlIO, há uma armadilha potencial digna de nota:

  • read usa characters para seus parâmetros length e offset
  • seek e tell sempre usam bytes para posicionamento

Portanto, não use aritmética com base nesses valores mistos. Em vez disso, use, por exemplo, Encode::encode('utf8',$value_by_read) para obter os octetos(bytes) de um resultado read, cuja contagem você pode usar com tell e seek.

Abrindo um FileHandle para leitura #

Abrindo arquivos de texto ASCII genéricos #

open my $filehandle, '<', $name_of_file or die "Can't open $name_of_file, $!";

Este é o idioma básico para E/S de arquivo “padrão” e torna $filehandle um fluxo de entrada legível de bytes, filtrado por um decodificador padrão específico do sistema, que pode ser definido localmente com o open pragma

Perl em si não lida com erros na abertura de arquivos, então você tem que lidar com eles verificando a condição de saída de open. $! é preenchido com a mensagem de erro que causou a falha de abertura.

No Windows, o decodificador padrão é um filtro “CRLF”, que mapeia quaisquer sequências “\r\n” na entrada para “\n”

Abrindo arquivos binários #

open my $filehandle, '<:raw', 'path/to/file' or die "Can't open $name_of_file, $!";

Isso especifica que Perl não deve executar uma tradução CRLF no Windows.

Abrindo arquivos de texto UTF8 #

open my $filehandle, '<:raw:encoding(utf-8)', 'path/to/file' 
   or die "Can't open $name_of_file, $!";

Isso especifica que o Perl deve evitar a tradução CRLF e, em seguida, decodificar os bytes resultantes em strings de caracteres (implementado internamente como arrays de inteiros que podem exceder 255), em vez de strings de bytes

1: https://www.wikiod.com/pt/perl/es-de-arquivo-leitura-e-gravacao-de-arquivos#Definindo a codificação padrão para IO

Lendo de um arquivo #

my $filename = '/path/to/file';

open my $fh, '<', $filename or die "Failed to open file: $filename"; 
    
# You can then either read the file one line at a time...
while(chomp(my $line = <$fh>)) {
    print $line . "\n";
}

# ...or read whole file into an array in one go
chomp(my @fileArray = <$fh>);

Se você sabe que seu arquivo de entrada é UTF-8, você pode especificar a codificação:

open my $fh, '<:encoding(utf8)', $filename or die "Failed to open file: $filename";

Após terminar a leitura do arquivo, o filehandle deve ser fechado:

close $fh or warn "close failed: $!";

Consulte também: Ler um arquivo em uma variável

Outra maneira mais rápida de ler um arquivo é usar File::Slurper Module. Isso é útil se você trabalha com muitos arquivos.

use File::Slurper;
my $file = read_text("path/to/file"); # utf8 without CRLF transforms by default
print $file; #Contains the file body

Veja também: [Ler um arquivo com slurp]

Grava em um arquivo #

Este código abre um arquivo para escrita. Retorna um erro se o arquivo não pôde ser aberto. Também fecha o arquivo no final.

#!/usr/bin/perl
use strict;
use warnings;
use open qw( :encoding(UTF-8) :std ); # Make UTF-8 default encoding

# Open "output.txt" for writing (">") and from now on, refer to it as the variable $fh.
open(my $fh, ">", "output.txt")
# In case the action failed, print error message and quit.
or die "Can't open > output.txt: $!";

Agora temos um arquivo aberto pronto para escrita que acessamos através de $fh (esta variável é chamada de filehandle). Em seguida, podemos direcionar a saída para esse arquivo usando o operador print:

# Print "Hello" to $fh ("output.txt").
print $fh "Hello";
# Don't forget to close the file once we're done!
close $fh or warn "Close failed: $!";

O operador open tem uma variável escalar ($fh neste caso) como seu primeiro parâmetro. Uma vez que é definido no operador open, é tratado como um filehandle. O segundo parâmetro ">" (maior que) define que o arquivo é aberto para escrita. O último parâmetro é o caminho do arquivo para gravar os dados.

Para escrever os dados no arquivo, o operador print é usado junto com o filehandle. Observe que no operador print não há vírgula entre o filehandle e a própria instrução, apenas espaços em branco.

“use autodie” e você não precisará verificar falhas de abertura/fechamento de arquivos #

autodie permite que você trabalhe com arquivos sem ter que verificar explicitamente por falhas de abertura/fechamento. #

Desde Perl 5.10.1, o pragma autodie está disponível no núcleo Perl. Quando usado, o Perl verificará automaticamente os erros ao abrir e fechar arquivos.

Aqui está um exemplo no qual todas as linhas de um arquivo são lidas e então gravadas no final de um arquivo de log.

use 5.010;    # 5.010 and later enable "say", which prints arguments, then a newline
use strict;   # require declaring variables (avoid silent errors due to typos)
use warnings; # enable helpful syntax-related warnings
use open qw( :encoding(UTF-8) :std ); # Make UTF-8 default encoding
use autodie;  # Automatically handle errors in opening and closing files

open(my $fh_in, '<', "input.txt"); # check for failure is automatic

# open a file for appending (i.e. using ">>")
open( my $fh_log, '>>', "output.log"); # check for failure is automatic

while (my $line = readline $fh_in) # also works: while (my $line = <$fh_in>)
{
     # remove newline
     chomp $line;

     # write to log file
     say $fh_log $line or die "failed to print '$line'"; # autodie doesn't check print
}

# Close the file handles (check for failure is automatic)
close $fh_in;
close $fh_log;

A propósito, você deve sempre checar tecnicamente as instruções print. Muitas pessoas não, mas perl (o interpretador Perl) não faz isso automaticamente e nem autodie.

Retroceder um identificador de arquivo #

Às vezes é necessário voltar atrás após a leitura.

# identify current position in file, in case the first line isn't a comment
my $current_pos = tell; 

while (my $line = readline $fh)
{
    if ($line =~ /$START_OF_COMMENT_LINE/)
    {
        push @names, get_name_from_comment($line);
    }
    else {
        last; # break out of the while loop
    }
    $current_pos = tell; # keep track of current position, in case we need to rewind the next line read
}

# Step back a line so that it can be processed later as the first data line
seek $fh, $current_pos, 0;

Lendo e gravando arquivos compactados com gzip #

Escrevendo um arquivo gzipado #

Para escrever um arquivo gzip, use o módulo IO::Compress::Gzip e crie um filehandle criando uma nova instância de IO::Compress::Gzip para o arquivo de saída desejado:

use strict;
use warnings;
use open qw( :encoding(UTF-8) :std ); # Make UTF-8 default encoding

use IO::Compress::Gzip;

my $fh_out = IO::Compress::Gzip->new("hello.txt.gz");

print $fh_out "Hello World!\n";

close $fh_out;

use IO::Compress::Gzip;

Lendo de um arquivo gzipado #

Para ler de um arquivo gzipado, use o módulo IO::Uncompress::Gunzip e então crie um filehandle criando uma nova instância de IO::Uncompress::Gunzip para o arquivo de entrada:

#!/bin/env perl
use strict;
use warnings;
use open qw( :encoding(UTF-8) :std ); # Make UTF-8 default encoding

use IO::Uncompress::Gunzip;

my $fh_in = IO::Uncompress::Gunzip->new("hello.txt.gz");

my $line = readline $fh_in;

print $line;

Configurando a codificação padrão para IO #

# encode/decode UTF-8 for files and standard input/output
use open qw( :encoding(UTF-8) :std );

Este pragma altera o modo padrão de leitura e escrita de texto (arquivos, entrada padrão, saída padrão e erro padrão) para UTF-8, que normalmente é o que você deseja ao escrever novos aplicativos.

ASCII é um subconjunto de UTF-8, portanto, não se espera que isso cause problemas com arquivos ASCII legados e ajudará a proteger você contra a corrupção acidental de arquivos que pode ocorrer ao tratar arquivos UTF-8 como ASCII.

No entanto, é importante que você saiba qual é a codificação de seus arquivos com a qual está lidando e trate-os de acordo. (Razões pelas quais não devemos ignorar o Unicode.) Para um tratamento mais aprofundado do Unicode, consulte o tópico Perl Unicode.

Lendo e gravando em um arquivo #

Antes de ler e escrever arquivos de texto, você deve saber qual codificação usar. Consulte a Documentação Perl Unicode para obter mais detalhes sobre codificação. Aqui mostramos a configuração de UTF-8 como a codificação e decodificação padrão para a função open. Isso é feito usando o pragma open próximo ao topo do seu código (logo após use strict; e use warnings; seriam apropriados):

use strict;
use warnings;
use open qw( :encoding(UTF-8) :std );

A função open cria um filehandle que é usado para ler e/ou gravar em um arquivo. A função open tem a assinatura

open(FILEHANDLE, MODE, FILEPATH) e retorna um valor falso se a operação falhar. A descrição do erro é então armazenada em $!.

Leitura

#!/usr/bin/perl
use strict;
use warnings;
use open qw( :encoding(UTF-8) :std ); # Make UTF-8 default encoding

my $file_path = "/path/to/file";
open(my $file_handle, '<', $file_path) or die "Could not open file! $!";

while(my $row = <$file_handle>) {
    print chomp($row), "\n";
}

close $file_handle 
      or warn "Close failed!";

Escrita

#!/usr/bin/perl
use strict;
use warnings;
use open qw( :encoding(UTF-8) :std ); # Make UTF-8 default encoding

my $file_path = "/path/to/file";
open(my $file_handle, '>', $file_path) or die "Could not open file! $!";

print $file_handle "Writing to a file";

close $file_handle 
      or warn "Close failed!";

Lendo trechos

Abrir e ler arquivos grandes pode levar algum tempo e recursos. Se apenas uma pequena parte do conteúdo for necessária, pode ser uma boa ideia ler o conteúdo em pedaços usando a função read que tem a assinatura

read(FILEHANDLE, SCALAR, LENGTH, OFFSET)

FILEHANDLE deve ser um handle de arquivo aberto, SCALAR manterá os dados lidos após a operação. LENGTH especifica o número de caracteres a serem lidos a partir de OFFSET. A função retorna o número de caracteres lidos, 0 se o final do arquivo foi atingido e undef em caso de erro.

read($file_handle, $data, 16, 0);

Lê 16 caracteres do início do arquivo em $data.
