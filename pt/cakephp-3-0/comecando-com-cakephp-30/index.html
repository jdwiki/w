Crie o primeiro ‘Hello World!’ aplicação com CakePHP 3.x (migrações de tabelas de banco de dados. Parte 2) #

Você pode facilmente criar tabelas para seu banco de dados ou descartá-las se desejar. Se você deseja fazer isso, você deve aprender a escrever Migrations para o banco de dados desejado.

Os arquivos de migração devem estar localizados na pasta config/Migrations. Os nomes dos arquivos podem estar nos seguintes formatos:

  • AAAAMMDDHHIISS_(Criar|Alterar|Excluir)AdministratorsTable.php

  • (1-9){1,}_(Create|Alter|Delete)AdministratorsTable.php

    _tableName = '"Administrators"'; $this->_tablePrefix = 'administrators'; } public function up() { Log::info("Trying to create {$this->_tableName} table"); $administratorsTable = $this->table($this->_tablePrefix); if ($administratorsTable->exists()) { return Log::warning("Table {$this->_tableName} already exists"); } $administratorsTable ->addPrimaryKey('id') ->addColumn('username', 'char', [ 'length' => 25, 'null' => false ]) ->addColumn('password', 'char', [ 'length' => 255, 'null' => false ]) ->addColumn('email', 'char', [ 'length' => 50, 'null' => false ]) ->addColumn('first_name', 'char', [ 'length' => 50, 'null' => false ]) ->addColumn('last_name', 'char', [ 'length' => 50, 'null' => false ]) ->addColumn('avatar', 'char', [ 'length' => 255, 'default' => '/img/no-avatar.png' ]) ->addColumn('active', 'boolean', [ 'default' => 0 ]) ->addTimestamps() ->create(); return Log::notice("Table {$this->_tableName} has been created"); } public function down() { if ($this->table($this->_tablePrefix)->exists()) { $this->table($this->_tablePrefix)->drop(); return Log::info("Table {$this->_tableName} has been dropped"); } return Log::warning("Table {$this->_tableName} does not exists"); } }

Se você deseja executar a migração, você precisa executar o próximo comando:

bin/cake migrations migrate para criar tabela(-s).

Se você quiser reverter:

bin/cake migrations rollback - reverterá a última migração, onde existe a função drop()

bin/cake migrations (-t|--target) all - reverterá todas as migrações, onde existe a função drop()

Instalação do Cakephp 3.X #

Requisitos:

  • Extensão PHP 5.6.0 ou superior mbstring PHP (o padrão funciona em WAMP/XAMPP, se não, habilite-o)
  • intl PHP extension (Disponível em WAMP/XAMPP, você deve habilitá-lo de php.ini)
  • O CakePHP será executado em uma variedade de servidores web como nginx,
  • LightHTTPD ou Microsoft IIS.

Antes de começar você deve certificar-se de que sua versão do PHP está atualizada:

php -v

Use o Composer para instalar o Cakephp 3 Framework,

Composer é um método oficialmente suportado para instalação, então baixe o compositor em Compositor (Windows/Linux/Mac)

Execute isso para instalar o cakephp,

php composer.phar create-project --prefer-dist cakephp/app my_app_name

Assim que o Composer terminar de baixar o esqueleto do aplicativo e a biblioteca principal do CakePHP, você deverá ter um aplicativo CakePHP funcional instalado via Composer. Certifique-se de manter os arquivos composer.json e composer.lock com o restante do código-fonte.

Ou siga esta maneira mais fácil de instalar o cakephp

Siga os passos abaixo,

  1. vá para este Repositório Git
  2. Baixe o cakeDC/forno para fácil instalação
  3. Extraia o arquivo zip dentro do LOCALHOST
  4. Dê permissão 777 para essa pasta
  5. Execute o arquivo forno.php (disponível dentro da pasta)
  6. Uma página será aberta com muitas opções, escolha a opção de acordo com sua preferência
  7. Clique na imagem e sente-se. Seu projeto será instalado em alguns minutos

executar:

bin/cake server

Por padrão, sem nenhum argumento fornecido, isso servirá seu aplicativo em http://localhost:8765/.

dispare isso no seu navegador, http://example.com/ ou http://localhost:8765/. Neste ponto, você verá a página inicial padrão do CakePHP e uma mensagem informando o status de sua conexão de banco de dados atual e você está pronto para sua primeira aplicação.

Para mais detalhes sobre instalação e configuração, siga Instalação do Cakephp 3.X

Crie o primeiro ‘Hello World!’ aplicação com CakePHP 3.x (Introdução. Parte 1) #

O CakePHP 3.x tem a habilidade de bake controllers, models, views e outros objetos definidos pelo framework.

Nota : Se você tem alguma experiência com o framework Laravel, o componente artisan é similar ao bake.

A aplicação bake está localizada na pasta bin; a seguir estão alguns dos comandos disponíveis:

  • bin/cake bake shell %shellName% - para assar ShellClass
  • bin/cake bake controller %controllerName% - para assar a classe do controlador
  • bin/cake bake model %modelName% - para assar Model + Entity Class
  • bin/cake bake view %viewName% - para assar View template
  • bin/cake bake all %className% - para preparar Controller, Model + Entity, View para desenvolvedor.

Nota: Você não poderá ‘assar modelo’ se não tiver tabelas em seu banco de dados

Nota : Se você bake all componentes, você obterá Controllers com ações CRUD predefinidas.

Crie o primeiro ‘Hello World!’ aplicação com CakePHP 3.x (Controllers, Response, View. Parte 3) #

Quer criar um controlador? Existem 2 maneiras de criá-lo:

  • Manualmente (você será forçado a criar manualmente o arquivo Controller em src/Controller)
  • Baked (Executando o comando bin/cake bake controller %controllerName% da CLI)

Se você quiser criá-lo manualmente, vá para a pasta src/Controller e crie o arquivo que segue o próximo padrão:

([A-Z]{1}[a-z]{1,})Controller.php

Nesse controlador, você deve definir namspace, que será usado:

<?php
    namespace App\Controller;

Então você deve nomeá-lo como nome do arquivo, ex. Controlador Admini:

use App\Controller\AppController;

class AdminController extends AppController{}

Dentro desta classe, você deve criar seu primeiro método, ex. login:

public function login(){}

Se você digitar em seu navegador : http://{{project-name}}/admin/login ele lançará um erro de modelo ausente. Como resolver este problema?

Você precisa criar no arquivo src/Template/Admin/ login.ctp.

Nota : *.ctp curinga - é o arquivo Cake Template, que está sendo usado para passar/renderizar dados que você configura através do controller.

Nesse arquivo apenas digite ‘Hello World!’ onde você quiser, atualize a página com erro de modelo e você obterá seu mundo, que o cumprimenta!

Instalando o CakePHP 3.4 no CentOS 7 com PHP 7 e SELinux habilitados #

Isto é o que eu fiz para instalar o CakePHP em um CentOS 7 mínimo instalado

  • Instalou um CentOS-7-x86_64-Minimal-1611.iso no VirtualBox, duas interfaces de rede: primeiro NAT, segundo Host-Only

  • defina ONBOOT=yes em /etc/sysconfig/network-scripts/ifcfg-enp0s3

  • reinício

  • atualização yum

  • yum install net-tools (para obter ifconfig e netstat)

  • yum instalar wget

  • yum instala yum-utils

  • wget -q http://rpms.remirepo.net/enterprise/remi-release-7.rpm

  • wget -q https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm

  • rpm -Uvh remi-release-7.rpm epel-release-latest-7.noarch.rpm

  • yum-config-manager –enable remi-php71

  • yum instalar php

  • systemctl habilita httpd

  • systemctl inicia httpd

  • firewall-cmd –permanent –zone=public –add-service=http

  • firewall-cmd –permanent –zone=public –add-service=https

  • firewall-cmd –reload

  • yum instale httpd mariadb-server mariadb php phpmyadmin

  • systemctl iniciar mariadb

  • systemctl habilita mariadb

  • systemctl reiniciar httpd

  • yum install php-mbstring php-intl

  • mysql_secure_installation

  • curl -s https://getcomposer.org/installer | php

  • cd /var/www/html/

  • php composer.phar create-project –prefer-dist cakephp/app MyApp

  • chown apache: -R MyApp/

  • Criar banco de dados:

    mysql -u root -p #

    Enter password:

    mysql> CREATE DATABASE mydb; mysql> GRANT ALL ON mydb.* to ‘myuser’@’localhost’ IDENTIFIED BY ‘password’; mysql> FLUSH PRIVILEGES; mysql> quit

  • Crie o arquivo /etc/httpd/conf.d/my_app.conf com o conteúdo:

    <VirtualHost *:80> ServerAdmin root@localhost ServerName cakephp.myapp.net DocumentRoot /var/www/html/MyApp <Directory /var/www/html/MyApp> Allowoverride All

  • cd /var/www/html/secure_logging; chcon -Rv –type=httpd_user_content_rw_t tmp

  • toque em /.autorelabel; reinício

  • No meu host eu edito /etc/hosts e entro na linha (192.168.56.101 é o host-only-ip-address da minha VM) 192.168.56.101 cakephp.myapp.net

  • Navegue para http://cakephp.myapp.net/

  • ToDo: Editar arquivo de conexão de banco de dados.
