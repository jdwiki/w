Espaço do usuário e espaço do kernel

  • Alternando entre o espaço do usuário e o espaço do kernel
  • Examinar a alocação de tempo de CPU entre o espaço do usuário e o espaço do kernel

Nesta página

  • Alternando entre o espaço do usuário e o espaço do kernel
  • Examinar a alocação de tempo de CPU entre o espaço do usuário e o espaço do kernel

A RAM pode ser dividida em duas partes. O espaço do kernel e o espaço do usuário. O kernel é executado no espaço do kernel, que nenhum outro programa pode acessar. Os programas do usuário devem ser executados no espaço do usuário. O espaço do usuário é uma forma de sand-boxing, onde os programas do usuário só podem acessar a memória que lhes foi alocada para que não atrapalhem outros programas e o kernel. Para usar o recurso do sistema, um programa usa chamadas do sistema para acessar uma determinada parte do espaço do kernel e voltar ao espaço do usuário quando a chamada retornar.

Alternando entre o espaço do usuário e o espaço do kernel #

Kernel gerencia os recursos do sistema operacional. O programa do usuário só pode acessar esses recursos fazendo chamadas de sistema para o kernel. A chamada do sistema é semelhante a uma API do kernel, que em termo, executa as tarefas do kernel que seu programa precisa.

str = "something" // run on user space
x = x + 1 // run on user space
file.write(str) // switch to kernel space
y = x + 4 // switch back to user space

Examinar a alocação de tempo de CPU entre o espaço do usuário e o espaço do kernel #

Use o comando top para examinar a alocação de tempo de CPU entre o espaço do usuário e o espaço do kernel.

Explicação:

  1. 24,8 us (espaço do usuário): 24,8% do tempo de CPU é gasto no processo do usuário.
  2. 0,5 sy (sistema): 0,5% do tempo de CPU é gasto no espaço do kernel.
  3. ni (gentileza): a proporção do tempo de CPU gasto em processos de baixa prioridade.
  4. id (idle): a proporção do tempo de CPU gasto em processos ociosos (durante o período de tempo, a CPU não pode fazer nada).
  5. wa (wait): a proporção do tempo de CPU gasto na espera de E/S (durante o período de tempo, a CPU não pode fazer nada).
  6. hi (interrupção de hardware): a proporção do tempo de CPU gasto na resposta à interrupção de hardware.
  7. si (interrupção de software): a proporção do tempo de CPU gasto na resposta à interrupção de software.
  8. st (tempo roubado): a proporção do tempo de CPU roubado pela máquina virtual.

Tempo de CPU do exame gasto em um processo #

Use o comando de tempo.

time ./perl-timeout-example 100.100.100

We could not ping the desired address!

real    0m5.0013s
user    0m0.004s
sys     0m0.008s
  • Real: Tempo total do início ao fim da chamada, incluindo o tempo de CPU gasto em outros processos.
  • Usuário: A quantidade de tempo de CPU gasto no espaço do usuário.
  • Sys: A quantidade de tempo de CPU gasto no espaço do kernel.

Normalmente, usuário + sys é o tempo total de CPU gasto em tal processo. É menor que o real. Mas na CPU multicore, user + sys é o tempo total de CPU gasto neste processo. Pode ser grande do que real.
