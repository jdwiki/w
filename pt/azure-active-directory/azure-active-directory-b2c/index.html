O Azure AD B2C é uma solução de gerenciamento de identidade na nuvem para seus aplicativos Web e móveis. É um serviço global altamente disponível que pode ser dimensionado para centenas de milhões de identidades.

Azure AD B2C - aplicativo de amostra Angularjs (Web e Mobile) #

Esta amostra demonstra o uso do AD B2C para proteger um aplicativo móvel e web baseado em AngularJS.

Consulte https://github.com/NewtonJoshua/Azure-ADB2C-Angularjs-sample

Azure AD B2C #

Azure AD B2C é uma solução de gerenciamento de identidade na nuvem para seus aplicativos móveis e da Web. É um serviço global altamente disponível que pode ser dimensionado para centenas de milhões de identidades.

Aplicativo da Web - Hello.js #

A implementação do aplicativo Web usa Hello.js que executa o gerenciamento de identidade com o Azure AD B2C . Hello.js é um SDK JavaScript do lado do cliente para autenticação com serviços da Web OAuth2 e consulta de APIs REST.

Aplicativo móvel - plug-in ADAL #

A implementação do aplicativo móvel usa ADAL Cordova Plugin Patch para B2C. Esta é uma versão cortada do plug-in Active Directory Authentication Library (ADAL) para aplicativos Apache Cordova, cordova-plugin-ms-adal que funciona com o Azure AD B2C. O plug-in cordova-plugin-ms-adal original fornece funcionalidade de autenticação fácil de usar para seus aplicativos Apache Cordova, aproveitando o Active Directory.

Decodificar JWT #

jwtHelper de angular-jwt cuidará de ajudá-lo a decodificar o token (JWT) e verificar sua data de validade. Os JSON Web Tokens são um método aberto padrão da indústria RFC 7519 para representar declarações de forma segura entre duas partes.

  1. Clone ou baixe este repositório

  2. Instale as dependências

    npm install

    bower install

2. Configuração do anúncio: #

Configuração ADAL-B2C

  1. Criar um diretório Azure AD B2C

    Note the Domain name, it’ll be used as the tenantName.

  2. Registre seu aplicativo Follow the instructions to create an application and enable both Web App and Native client. Refer Register a web application and Register a mobile/native application

    Enter the Reply URL as http://localhost:8100 or any port from wher you’ll be serving your app.

    In Application Claims, select Email Addresses too.

    Note the Application ID . It’ll be used as the clientId.

  3. [Crie uma política de inscrição ou entrada](https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-reference-policies#create-a -política de inscrição ou inscrição)

    Note the name of the policy. It’ll be used as policy.

  4. [Criar uma política de redefinição de senha](https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-reference-policies#create-a-password-reset- política)

    Note the name of the policy. It’ll be used as the password-reset-policy

3. Configurações do AD: #

Em settings.value.js, insira os seguintes valores

  • tenantName: nome de domínio da etapa 2.1
  • clientId: ID do aplicativo da etapa 2.2
  • policy: nome da política da etapa 2.3

5. Personalize a interface de usuário do Azure AD B2C #

A tela de login do Azure AD B2C pode ser personalizada para se adequar à nossa marca. Consulte Personalizando a interface do usuário

Neste exemplo, temos duas telas de interface do usuário personalizadas,

  • AD B2C Entrar e página de inscrição: adCustomPages/unified.html
  • Página de redefinição de senha AD B2C: adCustomPages/resetpassword.html

Em adCustomPages/unified.html, nas linhas 442 e 445, insira seu tenantName, password-reset-policy e clientId

As páginas devem ser carregadas em um blob e seu URL deve ser referido nas políticas do Azure AD B2C.

  • Crie uma conta de armazenamento conforme mencionado em [Carregar o conteúdo de exemplo para o Armazenamento de Blobs do Azure](https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-reference-ui -customization-helper-tool#upload-the-sample-content-to-azure-blob-storage)
  • Carregue as páginas do AD de amostra no contêiner e anote seu URL.
  • Para a conta de armazenamento do serviço Blob criada, crie uma regra CORS com ‘*’ como ORIGENS PERMITIDAS. Selecione tudo em MÉTODOS PERMITIDOS. Digite * para CABEÇALHOS PERMITIDOS e CABEÇALHOS EXPOSTOS também.
  • Personalize sua política

Agora, em seu aplicativo, você pode ver a interface do usuário personalizada.
