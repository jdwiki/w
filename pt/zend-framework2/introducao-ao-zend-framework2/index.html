Um simples Olá Mundo #

Na linha de comando, entre no diretório em que deseja criar o projeto e digite: composer create-project zendframework/skeleton-application helloWorldTest. Durante a instalação, você será perguntado se deseja uma instalação mínima: Digamos sim no momento, estamos apenas testando.

Para simplificar, usaremos o servidor PHP CLI integrado. Na linha de comando, entre no diretório raiz do seu projeto (helloWorldTest), então execute: php -S 0.0.0.0:8080 -t public/ public/index.php. Agora, abra seu navegador da web e vá para http://localhost/, você deverá ver a página de boas-vindas do ZF2 Skeleton Application.

Se você fizer isso, agora vamos configurar uma nova página. Em module/Application/config/module.config.php você pode ver que uma rota dinâmica já está configurada para a subpasta do aplicativo:

return [
    'router' => [
        'routes' => [
            'home' => [
                ...
            ],
            'application' => [
                'type'    => Segment::class,
                'options' => [
                    'route'    => '/application[/:action]',
                    'defaults' => [
                        'controller'    => Controller\IndexController::class,
                        'action'        => 'index',
                    ],
                ],
            ],
        ],
    ],

Defina uma nova ação “helloWorldAction()” em module/Aplicaiton/src/Controller/IndexController.php:

class IndexController extends AbstractActionController
{
    public function indexAction()
    {
        ...
    }
    
    public function helloWorldAction()
    {
        return new ViewModel();
    }
}

Por fim, crie o arquivo de visualização module/Application/view/application/index/hello-world.phtml com o seguinte conteúdo:

<?php
echo "Hello World !";

Agora, vá para http://localhost/application/hello-world e diga oi para ZF2 !

Instalação ou Configuração #

Instruções detalhadas sobre como configurar ou instalar o Zend Framework 2. Existem várias maneiras de instalar o framework. Abaixo estão alguns deles:

Usando o Composer - Maneira recomendada #

Supondo que o composer esteja instalado na caixa de destino.

Para instalar um aplicativo MVC esqueleto, execute em seu terminal para criar um novo projeto zend framework 2 no local especificado:

php composer.phar create-project -sdev \
    --repository-url="https://packages.zendframework.com" \
    zendframework/skeleton-application path/to/install

para instalar manualmente um ZF2 mínimo (Zend MVC + seu punhado de dependências), execute em sua linha de comando:

composer require zendframework/zend-mvc

ou para um ZF2 completod (+64 módulos):

composer require zendframework/zendframework`

Observe que a primeira opção executa um instalador que fornecerá um aplicativo totalmente funcional junto com a estrutura de diretórios de aplicativos usual. Outras opções permitem que você construa todo o aplicativo do zero, pois ele simplesmente fornece módulos ZF2 para construir.

Usando submódulos do Git #

Execute o comando abaixo para clonar o zf2 e suas dependências recursivamente do Github:

git clone git://github.com/zendframework/ZendSkeletonApplication.git --recursive

Configuração do servidor HTTP #

Um aplicativo da Web típico requer a execução de um serviço HTTP escutando uma porta dedicada (geralmente :80) para passar solicitações recebidas para o aplicativo, processar e fornecer a saída (resposta) de volta.

Nota: Você também pode escrever aplicativos com reconhecimento de console no Zend Framework 2 sem a necessidade de um servidor HTTP.

OPÇÃO 1 - Servidor CLI PHP #

A maneira mais simples de começar se você estiver usando o PHP 5.4 ou superior é iniciar o cli-server interno do PHP no diretório raiz.

Vá para o diretório do projeto e execute:

php -S 0.0.0.0:8080 -t public/ public/index.php`.

Isso iniciará o servidor cli integrado na porta 8080 e o vinculará a todas as interfaces de rede.

OPÇÃO 2 - Um servidor HTTP personalizado #

Configure um host virtual no Apache ou Microsoft IIS Server ou Nginx e passe as solicitações HTTP de entrada para o aplicativo.

Como criar uma fábrica #

Quando uma classe precisa ser fornecida com dependências rígidas, a prática recomendada é usar um padrão de injeção de construtor em que essas dependências são injetadas usando uma fábrica.

Vamos supor que MyClass seja fortemente dependente de um valor $dependency que precisa ser resolvido a partir da configuração do aplicativo.

<?php
namespace Application\Folder;

use Zend\ServiceManager\FactoryInterface;
use Zend\ServiceManager\ServiceLocatorInterface;

class MyClass
{
    protected $dependency;

    public function __construct($dependency)
    { 
        $this->dependency = $dependency;
    }
}

Para injetar essa dependência é criada uma classe de fábrica. Esta fábrica resolverá a dependência da configuração e injetará o valor da configuração na construção da classe e retornará o resultado:

<?php
namespace Application\Factory;

use Zend\ServiceManager\FactoryInterface;
use Zend\ServiceManager\ServiceLocatorInterface;

class MyClassFactory implements FactoryInterface
{
    public function createService(ServiceLocatorInterface $serviceLocator)
    { 
        $config = $servicelocator->get('Config');
        $dependency = $config['dependency'];
        $myClass = new MyClass($dependency);
        return $myClass;
    }
}

Agora que a classe de fábrica foi criada, ela deve ser registrada dentro da configuração do gerenciador de serviços no arquivo de configuração do módulo module.config.php sob as fábricas de chaves. É uma boa prática usar os mesmos nomes para a classe e a fábrica para que seja fácil encontrá-los na árvore de pastas do projeto:

<?php

namespace Application;

return array(
    //...
    'service_manager' => [
        'factories' => [
            'Application\Folder\MyClass' => 'Application\Factory\MyClassFactory'
        ]
    ],
    //...
);

Alternativamente, as constantes de nome de classe podem ser usadas para registrá-las:

<?php

namespace Application;

use Application\Folder\MyClass;
use Application\Factory\MyClassFactory;

return array(
    //...
    'service_manager' => [
        'factories' => [
            MyClass::class => MyClassFactory::class'
        ]
    ],
    //...
);

Agora a classe pode ser coletada no gerenciador de serviço usando a chave que usamos ao registrar a fábrica para essa classe:

 $serviceManager->get('Application\Folder\MyClass');

ou

 $serviceManager->get(MyClass::class);

O gerenciador de serviço encontrará, coletará e executará a fábrica e, em seguida, retornará sua instância de classe com a dependência injetada.
