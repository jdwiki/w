Sintaxe #

  1. Função functionName(argumentVariable As dataType, argumentVariable2 As dataType, Opcional argumentVariable3 As dataType) As functionReturnDataType Declaração básica de uma função. Toda função precisa de um nome, mas não precisa receber argumentos. Pode levar 0 argumentos ou pode levar um determinado número de argumentos. Você também pode declarar um argumento como opcional (o que significa que não importa se você o fornece ao chamar a função). É uma prática recomendada fornecer o tipo de variável para cada argumento e, da mesma forma, retornar o tipo de dados que a própria função retornará.

  2. functionName = theVariableOrValueBeingReturned Se você vem de outras linguagens de programação, pode estar acostumado com a palavra-chave Return. Isso não é usado no VBA - em vez disso, usamos o nome da função. Você pode configurá-lo para o conteúdo de uma variável ou para algum valor fornecido diretamente. Observe que se você definiu um tipo de dados para o retorno da função, a variável ou os dados que você está fornecendo desta vez devem ser desse tipo de dados.

  3. Função Final Obrigatoriedade. Significa o fim do bloco de código Function e, portanto, deve estar no final. O VBE geralmente fornece isso automaticamente quando você cria uma nova função.

Uma Função Definida pelo Usuário (também conhecida como UDF) refere-se a uma função específica da tarefa que foi criada pelo usuário. Ele pode ser chamado como uma função de planilha (ex: =SUM(...)) ou usado para retornar um valor para um processo em execução em um procedimento Sub. Uma UDF retorna um valor, normalmente de informações passadas para ela como um ou mais parâmetros.

Pode ser criado por:

  1. usando VBA.
  2. usando a API Excel C - Criando um XLL que exporta funções compiladas para o Excel.
  3. usando a interface COM.

Permitir referências de coluna completas sem penalidade #

É mais fácil implementar algumas UDFs na planilha se referências de coluna completas puderem ser passadas como parâmetros. No entanto, devido à natureza explícita da codificação, qualquer loop envolvendo esses intervalos pode estar processando centenas de milhares de células completamente vazias. Isso reduz seu projeto VBA (e pasta de trabalho) a uma bagunça congelada enquanto não valores desnecessários são processados.

Percorrer as células de uma planilha é um dos métodos mais lentos de realizar uma tarefa, mas às vezes é inevitável. Reduzir o trabalho realizado ao que é realmente necessário faz todo o sentido.

A solução é truncar as referências de coluna completa ou linha completa para a propriedade Worksheet.UsedRange com o método Intersect. O exemplo a seguir replicará vagamente a função SUMIF nativa de uma planilha para que o criteria_range também seja redimensionado para se adequar ao sum_range, pois cada valor no sum_range deve ser acompanhado por um valor no criteria_range.

O Application.Caller para uma UDF usada em uma planilha é a célula na qual ela reside. A propriedade .Parent da célula é a planilha. Isso será usado para definir o .UsedRange.

Em uma folha de código do módulo:

Option Explicit

Function udfMySumIf(rngA As Range, rngB As Range, _
                    Optional crit As Variant = "yes")
    Dim c As Long, ttl As Double
    
    With Application.Caller.Parent
        Set rngA = Intersect(rngA, .UsedRange)
        Set rngB = rngB.Resize(rngA.Rows.Count, rngA.Columns.Count)
    End With
    
    For c = 1 To rngA.Cells.Count
        If IsNumeric(rngA.Cells(c).Value2) Then
            If LCase(rngB(c).Value2) = LCase(crit) Then
                ttl = ttl + rngA.Cells(c).Value2
            End If
        End If
    Next c
    
    udfMySumIf = ttl

End Function

Sintaxe:
        =udfMySumIf(*sum_range*, *criteria_range*, [*criteria*])

Embora este seja um exemplo bastante simplista, ele demonstra adequadamente a passagem de duas referências de coluna completas (1.048.576 linhas cada), mas processando apenas 15 linhas de dados e critérios.

UDF - Olá Mundo #

  1. Abra o Excel
  2. Abra o Editor do Visual Basic (consulte https://www.wikiod.com/pt/excel-vba/introducao-ao-excel-vba#Abrindo o Editor do Visual Basic (VBE) )
  3. Adicione um novo módulo clicando em Inserir –> Módulo :

  1. Copie e cole o seguinte código no novo módulo:

    Public Function Hello() As String ‘Note: the output of the function is simply the function’s name Hello = “Hello, World !” End Function

Obter :

  1. Volte para sua pasta de trabalho e digite “=Hello()” em uma célula para ver o “Hello World”.

Contagem de valores exclusivos no intervalo #

Function countUnique(r As range) As Long
    'Application.Volatile False ' optional
    Set r = Intersect(r, r.Worksheet.UsedRange) ' optional if you pass entire rows or columns to the function
    Dim c As New Collection, v
    On Error Resume Next   ' to ignore the Run-time error 457: "This key is already associated with an element of this collection".
    For Each v In r.Value  ' remove .Value for ranges with more than one Areas
        c.Add 0, v & ""
    Next
    c.Remove "" ' optional to exclude blank values from the count
    countUnique = c.Count
End Function
