Quando usar ActiveWorkbook e ThisWorkbook #

É uma prática recomendada do VBA sempre especificar a qual pasta de trabalho seu código VBA se refere. Se esta especificação for omitida, o VBA assume que o código é direcionado para a pasta de trabalho ativa no momento (ActiveWorkbook).

'--- the currently active workbook (and worksheet) is implied
Range("A1").value = 3.1415
Cells(1, 1).value = 3.1415

No entanto, quando várias pastas de trabalho são abertas ao mesmo tempo - particularmente e especialmente quando o código VBA está sendo executado a partir de um suplemento do Excel - as referências ao ‘ActiveWorkbook’ podem ser confusas ou mal direcionadas. Por exemplo, um suplemento com uma UDF que verifica a hora do dia e a compara com um valor armazenado em uma das planilhas do suplemento (que normalmente não são prontamente visíveis para o usuário) terá que identificar explicitamente qual pasta de trabalho é sendo referenciado. Em nosso exemplo, nossa pasta de trabalho aberta (e ativa) tem uma fórmula na célula A1 =EarlyOrLate() e NÃO tem nenhum VBA escrito para essa pasta de trabalho ativa. Em nosso suplemento, temos a seguinte Função Definida pelo Usuário (UDF):

Public Function EarlyOrLate() As String
    If Hour(Now) > ThisWorkbook.Sheets("WatchTime").Range("A1") Then
        EarlyOrLate = "It's Late!"
    Else
        EarlyOrLate = "It's Early!"
    End If
End Function

O código para a UDF é escrito e armazenado no suplemento do Excel instalado. Ele usa dados armazenados em uma planilha no suplemento chamado “WatchTime”. Se a UDF tivesse usado ActiveWorkbook em vez de ThisWorkbook, então ela nunca seria capaz de garantir qual pasta de trabalho se destinava.

Alterando o número padrão de planilhas em uma nova pasta de trabalho #

O número “padrão de fábrica” ​​de planilhas criadas em uma nova pasta de trabalho do Excel geralmente é definido como três. Seu código VBA pode definir explicitamente o número de planilhas em uma nova pasta de trabalho.

'--- save the current Excel global setting
With Application
    Dim oldSheetsCount As Integer
    oldSheetsCount = .SheetsInNewWorkbook
    Dim myNewWB As Workbook
    .SheetsInNewWorkbook = 1
    Set myNewWB = .Workbooks.Add
    '--- restore the previous setting
    .SheetsInNewWorkbook = oldsheetcount
End With

Pastas de trabalho do aplicativo #

Em muitos aplicativos do Excel, o código VBA executa ações direcionadas à pasta de trabalho na qual está contido. Você salva essa pasta de trabalho com uma extensão “.xlsm” e as macros VBA se concentram apenas nas planilhas e nos dados nela contidos. No entanto, muitas vezes você precisa combinar ou mesclar dados de outras pastas de trabalho ou gravar alguns de seus dados em uma pasta de trabalho separada. Abrir, fechar, salvar, criar e excluir outras pastas de trabalho é uma necessidade comum para muitos aplicativos VBA.

A qualquer momento no Editor VBA, você pode visualizar e acessar todas e quaisquer pastas de trabalho atualmente abertas por essa instância do Excel usando a propriedade Pastas de trabalho do objeto Aplicativo. A Documentação do MSDN explica com referências.

Abrindo uma (nova) pasta de trabalho, mesmo que já esteja aberta #

Se você deseja acessar uma pasta de trabalho que já está aberta, obter a atribuição da coleção Workbooks é simples:

dim myWB as Workbook
Set myWB = Workbooks("UsuallyFullPathnameOfWorkbook.xlsx")

Se você deseja criar uma nova pasta de trabalho, use o objeto de coleção Workbooks para Add uma nova entrada.

Dim myNewWB as Workbook
Set myNewWB = Workbooks.Add

Há momentos em que você não pode ou (ou se importa) se a pasta de trabalho que você precisa já está aberta ou não, ou se possível não existe. A função de exemplo mostra como sempre retornar um objeto de pasta de trabalho válido.

Option Explicit
Function GetWorkbook(ByVal wbFilename As String) As Workbook
    '--- returns a workbook object for the given filename, including checks
    '    for when the workbook is already open, exists but not open, or
    '    does not yet exist (and must be created)
    '    ***  wbFilename must be a fully specified pathname
    Dim folderFile As String
    Dim returnedWB As Workbook
    
    '--- check if the file exists in the directory location
    folderFile = File(wbFilename)
    If folderFile = "" Then
        '--- the workbook doesn't exist, so create it
        Dim pos1 As Integer
        Dim fileExt As String
        Dim fileFormatNum As Long
        '--- in order to save the workbook correctly, we need to infer which workbook
        '    type the user intended from the file extension
        pos1 = InStrRev(sFullName, ".", , vbTextCompare)
        fileExt = Right(sFullName, Len(sFullName) - pos1)
        Select Case fileExt
            Case "xlsx"
                fileFormatNum = 51
            Case "xlsm"
                fileFormatNum = 52
            Case "xls"
                fileFormatNum = 56
            Case "xlsb"
                fileFormatNum = 50
            Case Else
                Err.Raise vbObjectError + 1000, "GetWorkbook function", _
                         "The file type you've requested (file extension) is not recognized. " & _
                         "Please use a known extension: xlsx, xlsm, xls, or xlsb."
        End Select
        Set returnedWB = Workbooks.Add
        Application.DisplayAlerts = False
        returnedWB.SaveAs filename:=wbFilename, FileFormat:=fileFormatNum
        Application.DisplayAlerts = True
        Set GetWorkbook = returnedWB
    Else
        '--- the workbook exists in the directory, so check to see if
        '    it's already open or not
        On Error Resume Next
        Set returnedWB = Workbooks(sFile)
        If returnedWB Is Nothing Then
            Set returnedWB = Workbooks.Open(sFullName)
        End If
    End If
End Function

Salvando uma pasta de trabalho sem perguntar ao usuário #

Muitas vezes, salvar novos dados em uma pasta de trabalho existente usando o VBA causará uma pergunta pop-up informando que o arquivo já existe.

Para evitar essa pergunta pop-up, você deve suprimir esses tipos de alertas.

Application.DisplayAlerts = False        'disable user prompt to overwrite file
myWB.SaveAs FileName:="NewOrExistingFilename.xlsx"
Application.DisplayAlerts = True         're-enable user prompt to overwrite file
