Referências qualificadas #

Ao se referir a uma planilha, um intervalo ou células individuais, é importante qualificar totalmente a referência.

Por exemplo:

ThisWorkbook.Worksheets("Sheet1").Range(Cells(1, 2), Cells(2, 3)).Copy

Não é totalmente qualificado: As referências Cells não possuem uma pasta de trabalho e planilha associada a elas. Sem uma referência explícita, Cells se refere ao ActiveSheet por padrão. Portanto, este código falhará (produzindo resultados incorretos) se uma planilha diferente de Sheet1 for a ActiveSheet atual.

A maneira mais fácil de corrigir isso é usar uma instrução With da seguinte forma:

With ThisWorkbook.Worksheets("Sheet1")
    .Range(.Cells(1, 2), .Cells(2, 3)).Copy
End With

Como alternativa, você pode usar uma variável de planilha. (Esse provavelmente será o método preferido se seu código precisar fazer referência a várias planilhas, como copiar dados de uma planilha para outra.)

Dim ws1 As Worksheet
Set ws1 = ThisWorkbook.Worksheets("Sheet1")
ws1.Range(ws1.Cells(1, 2), ws1.Cells(2, 3)).Copy

Outro problema frequente é fazer referência à coleção de planilhas sem qualificar a pasta de trabalho. Por exemplo:

Worksheets("Sheet1").Copy

A planilha Planilha1 não está totalmente qualificada e não possui uma pasta de trabalho. Isso pode falhar se várias pastas de trabalho forem referenciadas no código. Em vez disso, use um dos seguintes:

ThisWorkbook.Worksheets("Sheet1")       '<--ThisWorkbook refers to the workbook containing 
                                        'the running VBA code
Workbooks("Book1").Worksheets("Sheet1") '<--Where Book1 is the workbook containing Sheet1

No entanto, evite usar o seguinte:

ActiveWorkbook.Worksheets("Sheet1")     '<--Valid, but if another workbook is activated
                                        'the reference will be changed

Da mesma forma para objetos range, se não for explicitamente qualificado, o range fará referência à planilha atualmente ativa:

Range("a1")

É o mesmo que:

ActiveSheet.Range("a1")

Excluindo linhas ou colunas em um loop #

Se você deseja excluir linhas (ou colunas) em um loop, você deve sempre fazer um loop começando no final do intervalo e voltar a cada etapa. No caso de usar o código:

Dim i As Long
With Workbooks("Book1").Worksheets("Sheet1")
    For i = 1 To 4
        If IsEmpty(.Cells(i, 1)) Then .Rows(i).Delete
    Next i
End With

Você vai perder algumas linhas. Por exemplo, se o código excluir a linha 3, a linha 4 se tornará a linha 3. No entanto, a variável i mudará para 4. Portanto, neste caso, o código perderá uma linha e verificará outra, que não estava no intervalo anteriormente .

O código correto seria

Dim i As Long
With Workbooks("Book1").Worksheets("Sheet1")
    For i = 4 To 1 Step -1
        If IsEmpty(.Cells(i, 1)) Then .Rows(i).Delete
    Next i
End With

ActiveWorkbook vs. ThisWorkbook #

ActiveWorkbook e ThisWorkbook às vezes são usados ​​de forma intercambiável por novos usuários do VBA sem entender completamente a que cada objeto se relaciona, isso pode causar um comportamento indesejado em tempo de execução. Ambos os objetos pertencem ao Application Object

O objeto ‘ActiveWorkbook’ refere-se à pasta de trabalho que está atualmente na exibição superior do objeto de aplicativo do Excel no momento da execução. (por exemplo, a pasta de trabalho que você pode ver e interagir no ponto em que este objeto é referenciado)

Sub ActiveWorkbookExample()

'// Let's assume that 'Other Workbook.xlsx' has "Bar" written in A1.

    ActiveWorkbook.ActiveSheet.Range("A1").Value = "Foo"
    Debug.Print ActiveWorkbook.ActiveSheet.Range("A1").Value '// Prints "Foo"

    Workbooks.Open("C:\Users\BloggsJ\Other Workbook.xlsx")
    Debug.Print ActiveWorkbook.ActiveSheet.Range("A1").Value '// Prints "Bar"

    Workbooks.Add 1
    Debug.Print ActiveWorkbook.ActiveSheet.Range("A1").Value '// Prints nothing

End Sub

O objeto ThisWorkbook refere-se à pasta de trabalho na qual o código pertence no momento em que está sendo executado.

Sub ThisWorkbookExample()

'// Let's assume to begin that this code is in the same workbook that is currently active

    ActiveWorkbook.Sheet1.Range("A1").Value = "Foo"
    Workbooks.Add 1
    ActiveWorkbook.ActiveSheet.Range("A1").Value = "Bar"

    Debug.Print ActiveWorkbook.ActiveSheet.Range("A1").Value '// Prints "Bar"
    Debug.Print ThisWorkbook.Sheet1.Range("A1").Value '// Prints "Foo"

End Sub

Interface de documento único versus interfaces de vários documentos #

Esteja ciente de que o Microsoft Excel 2013 (e superior) usa Documento Único Interface (SDI) e que o Excel 2010 (e abaixo) usa vários documentos Interfaces (MDI).

Isso implica que, para o Excel 2013 (SDI), cada pasta de trabalho em uma única instância do Excel contém sua própria interface do usuário da faixa de opções:

Por outro lado, para o Excel 2010, cada pasta de trabalho em uma única instância do Excel utilizou uma interface de usuário de faixa de opções comum (MDI):

Isso levanta alguns problemas importantes se você deseja migrar um código VBA (2010 <->2013) que interage com a Faixa de Opções.

Um procedimento deve ser criado para atualizar os controles da interface do usuário da faixa de opções no mesmo estado em todas as pastas de trabalho do Excel 2013 e superior.

Observe que :

  1. Todos os métodos, eventos e propriedades de janela em nível de aplicativo do Excel permanecem inalterados. (Application.ActiveWindow, Application.Windows … )
  2. No Excel 2013 e superior (SDI), todos os métodos, eventos e propriedades da janela no nível da pasta de trabalho agora operam na janela de nível superior. É possível recuperar o identificador desta janela de nível superior com Application.Hwnd

Para obter mais detalhes, consulte a fonte deste exemplo: MSDN.
