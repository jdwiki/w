Uso básico da API #

Ele também pode ser instalado baixando o código fonte e colocando-o em um diretório do seu projeto. No entanto, há muitos benefícios em usar o composer.

require '/path/to/lib/Twig/Autoloader.php';
Twig_Autoloader::register();

$loader = new Twig_Loader_Filesystem('/path/to/templates');

$options = array(
    'strict_variables' => false,
    'debug' => false,
    'cache'=> false
);

$twig = new Twig_Environment($loader, $options);

Ao criar uma nova instância Twig_Environment, você pode passar um array de opções como segundo argumento do construtor. Aqui está uma lista das opções disponíveis:

  • debug (boolean, padrão false)

Quando definido como true, os templates gerados possuem um método __toString() que você pode usar para exibir os nós gerados.

  • charset (string, padrão utf-8)

O conjunto de caracteres usado pelos modelos.

  • base_template_class (string, padrão Twig_Template)

A classe de modelo base a ser usada para modelos gerados.

  • cache (string ou false, padrão false)

Um caminho absoluto onde armazenar os templates compilados, ou false para desabilitar o cache (que é o padrão).

  • auto_reload (boolean, padrão herdado de debug)

Ao desenvolver com Twig, é útil recompilar o template sempre que o código-fonte for alterado. Se você não fornecer um valor para o opção auto_reload, ela será determinada automaticamente com base na valor de depuração.

  • strict_variables (boolean, padrão false)

Se definido como false, o Twig ignorará silenciosamente variáveis ​​inválidas (variáveis ​​e ou atributos/métodos que não existem) e substituirá eles com um valor nulo. Quando definido como true, Twig lança uma exceção em vez disso.

  • autoescape (string ou boolean, defaulttrue)

Se definido como verdadeiro, o escape automático de HTML será ativado por padrão para todos os modelos.

A partir do Twig 1.8, você pode definir a estratégia de escape a ser usada (html, js, false para desabilitar).

A partir do Twig 1.9, você pode definir a estratégia de escape a ser usada (css, url, html_attr ou um retorno de chamada PHP que leva o modelo “nome do arquivo” e deve retornar a estratégia de escape a ser usada – o retorno de chamada não pode ser um nome de função para evitar colisão com estratégias de escape internas).

A partir do Twig 1.17, a estratégia de escape do nome do arquivo determina a estratégia de escape a ser usada para um modelo baseado no nome do arquivo do modelo extensão (esta estratégia não incorre em nenhuma sobrecarga em tempo de execução, pois o escape automático é feito em tempo de compilação.)

  • otimizações (inteiro, padrão -1)

Um sinalizador que indica quais otimizações aplicar:
definido como -1 para habilitar todas as otimizações
defina o 0 para desabilitar todas as otimizações

Guia oficial de instalação do Twig

Uma extensão Twig PHP (escrita em C) também pode ser compilada e instalada, e o pacote PHP automaticamente tirará vantagem disso para otimizar algumas rotinas comuns.

O que é o Twig? #

Twig é uma linguagem de modelagem que compila para código PHP otimizado. É usado principalmente para saída de HTML, mas também pode ser usado para saída de qualquer outro formato baseado em texto. É um componente autônomo que pode ser facilmente integrado a qualquer projeto PHP.

Ele fornece muitos recursos excelentes:

  • Autoescaping para HTML (ajuda a evitar XSS)
  • Sintaxe projetada com templates em mente (baseado em templates Django)
  • Herança de modelos
  • Macros

Documentação oficial de modelagem do Twig

Exemplo da sintaxe do Twig:

{% extends "base.html" %}

{% block sidebar %}
  {{ parent() }}
  <span>Sidebar content specific to this page</span>
{% endblock sidebar %}

{% block body %}
  <p>Select an item:</p>
  <ul>
    {% for item in list %}
      <li><a href="/items/{{ item.id }}">{{ item.name }}</a>
    {% else %}
      <li>No items yet.
    {% endfor %}
  </ul>
{% endblock body %}

Introdução #

Se você tiver alguma exposição a outras linguagens de modelo baseadas em texto, como Smarty, Django ou Jinja, você deve se sentir em casa com Twig . É amigável ao designer e ao desenvolvedor, aderindo aos princípios do PHP e adicionando funcionalidades úteis para ambientes de modelagem.

Os principais recursos são…

  • Rápido: Twig compila templates para código PHP simples e otimizado. A sobrecarga em comparação com o código PHP normal foi reduzida ao mínimo.

  • Seguro: Twig tem um modo sandbox para avaliar modelos não confiáveis código. Isso permite que o Twig seja usado como uma linguagem de modelo para aplicativos em que os usuários podem modificar o design do modelo.

  • Flexível: Twig é alimentado por um lexer e um parser flexíveis. este permite que o desenvolvedor defina suas próprias tags personalizadas e filtros, e para criar seu próprio DSL.

Twig é usado por muitos projetos de código aberto como Symfony, Drupal, eZPublish e muitos frameworks também têm suporte para isso, como Slim, [Yii] 9, Laravel, Codeigniter,silex e Kohana — só para citar alguns.

Instalação

A maneira recomendada de instalar o Twig é via Composer:

Para usuários do php 5.x

composer require "twig/twig:~1.0"

Para usuários do php 7.x

composer require "twig/twig:~2.0"
