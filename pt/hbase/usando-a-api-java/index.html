Sintaxe #

  • HBaseConfiguration.create(); //Cria um arquivo de configuração

  • Configuration.set(chave de string, valor de string); //Adiciona uma chave na configuração

  • ConnectionFactory.createConnection(configuração HBaseConfiguration); //Conecta-se ao HBase

  • Connection.getAdmin(); //Instancia um novo Admin

  • new HTableDescriptor(Table.valueOf(String tableName));; //Cria um descritor de tabela

  • HTableDescriptor.addFamily(new HColumnDescriptor(String familyName)); //Adiciona uma família ao descritor da tabela

  • Admin.createTable(descritor HTableDescriptor); //Cria uma tabela conforme descrito no descritor

  • Admin.deleteTable(TableName.valueOf(String tableName)); //Deleta uma tabela

  • Connection.getTable(TableName.valueOf(String tableName)); //Obter um objeto de tabela

  • new Get(Bytes.toBytes(String row_key)); //Cria um novo Get

  • table.get(Get get) //Retorna um resultado

  • new Put(String row_key); //Cria um novo Put

  • table.put(Put put); //Inserir a(s) linha(s)

  • Novo Scan(); //Cria uma nova varredura

  • table.getScanner(Scan scan); //Retorna um ResultScanner

  • new Delete(Bytes.toBytes(String row_key)); //Cria um novo Delete

  • table.delete(Delete delete); //Deleta uma linha da tabela

Parâmetros #

Parâmetro Valores Possíveis
CompararOp CompareOp.EQUAL , CompareOp.GREATER , CompareOp.GREATER_OR_EQUAL , CompareOp.LESS , CompareOp.LESS_OR_EQUAL , CompareOp.NOT_EQUAL , CompareOp.NO_OP (sem operação)

Este tópico mostra vários exemplos de como usar a API Java para HBase. Neste tópico, você aprenderá a criar e excluir uma tabela, inserir, consultar e excluir linhas de uma tabela, mas também usar os filtros Scans.

Você notará que muitos métodos dessa API usam Bytes como parâmetros, por exemplo, o nome columnFamily, isso se deve à implementação do HBase. Para fins de otimização, em vez de armazenar os valores como String, Integer ou qualquer outro, ele armazena uma lista de Bytes, por isso você precisa analisar todos esses valores como Bytes. Para fazer isso, o método mais fácil é usar Bytes.toBytes(algo).

Por favor, sinta-se à vontade para observar se você ver algum erro ou mal-entendido.

Consultando HBase, Get, Put, Delete e Scans #

No HBase, você pode usar 4 tipos de operações

  • Get: recupera uma linha
  • Put : insere uma ou mais linhas
  • Excluir: excluir uma linha
  • Scan: recupera várias linhas

Se você simplesmente deseja recuperar uma linha, dada sua row_key, você pode usar o objeto Get:

Get get = new Get(Bytes.toBytes("my_row_key"));
Table table = this.connection.getTable(TableName.valueOf("myTable"));
Result r = table.get(get);
byte[] value = r.getValue(Bytes.toBytes(columnFamily), Bytes.toBytes("myColumn"));
String valueStr = Bytes.toString(value);
System.out.println("Get result :" + valueStr);

Aqui só pegamos o valor da coluna que queremos, se você quiser recuperar toda a coluna, use o atributo rawCell do objeto Get:

Get get = new Get(Bytes.toBytes(rowKey));
Table table = this.connection.getTable(TableName.valueOf(tableName));
Result r = table.get(get);
System.out.println("GET result :");
    for (Cell c : r.rawCells()) {
        System.out.println("Family : " + new String(CellUtil.cloneFamily(c)));
        System.out.println("Column Qualifier : " + new String(CellUtil.cloneQualifier(c)));
        System.out.println("Value : " + new String(CellUtil.cloneValue(c)));
        System.out.println("----------");
    }

Bem, agora podemos recuperar dados de nossa tabela, linha por linha, mas como colocar alguns? Você usa o objeto Put:

Put put = new Put("my_row_key");
put.addColumn(Bytes.toBytes("myFamily"), Bytes.toBytes("myColumn"), Bytes.toBytes("awesomeValue");
//Add as many columns as you want

Table table = connection.getTable(TableName.valueOf("myTable");
table.put(put);

NB : Table.put também pode receber no parâmetro uma lista de puts, que é, quando você deseja adicionar muitas linhas, muito mais eficiente do que put by put.

Tudo bem agora, eu posso colocar algumas linhas e recuperar algumas do meu HBase, mas e se eu quiser obter várias linhas e se eu não souber minhas row_keys ?

Capitão aqui! Você pode usar o objeto Scan:

Uma varredura basicamente procura todas as linhas e as recupera, você pode adicionar vários parâmetros a ela, como filtros e linha inicial/final, mas veremos isso em [outro exemplo](https://www.wikiod.com/pt/hbase/usando-a-api-java /using-the-java-api/15541/using-the-scan-filters#t=201607281342145028956).

Se você quiser varrer todos os valores de coluna da sua tabela, dada uma coluna, use as seguintes linhas:

Table table = this.connection.getTable(TableName.valueOf("myTable"));
Scan scan = new Scan();
scan.addColumn(Bytes.toBytes("myFamily"), Bytes.toBytes("myColumn"));
ResultScanner rs = table.getScanner(scan);
    try {
        for (Result r = rs.next(); r != null; r = rs.next()) {
            byte[] value = r.getValue(Bytes.toBytes("myFamily"), Bytes.toBytes("myCOlumn"));
            String valueStr = Bytes.toString(value);
            System.out.println("row key "+new String(r.getRow()));
            System.out.println("Scan result :" + valueStr);
        }
    } finally {
        rs.close(); // always close the ResultScanner!
   }

Eu realmente quero insistir no fato de que você deve sempre fechar o ResultScanner (a mesma coisa que qualquer ResultSet de um banco de dados, a propósito)

Quase pronto ! Agora vamos aprender como excluir uma linha. Você tem um objeto Delete para isso:

Table table = this.connection.getTable(TableName.valueOf("myTable"));
Delete d = new Delete(Bytes.toBytes("my_weird_key"));
table.delete(d);
System.out.prinln("Row " + row_key + " from table " + tableName + " deleted");

Uma última coisa: antes de executar qualquer uma das operações, sempre verifique se a tabela existe, ou você receberá uma exceção.

Isso é tudo por enquanto, você pode gerenciar seus dados no HBase com este exemplo.

Criando e excluindo tabelas #

No HBase, os dados são armazenados em tabelas com colunas. As colunas são reagrupadas em famílias de colunas, que podem ser por exemplo “pessoais” ou “profissionais”, cada uma delas contendo informações específicas.

Para criar uma tabela, você precisa usar o Admin Object, crie-o usando:

Admin admin = connection.getAdmin();

Depois de ter esse administrador, você pode começar a criar tabelas. Antes de tudo, certifique-se de que esta tabela já não exista com a linha

admin.tableExists(TableName.valueOf("myTable);

Este método retornará true se a tabela existir. Depois de verificar isso, você pode criar sua tabela usando as linhas

HTableDescriptor descriptor = new HTableDescriptor(TableName.valueOf("myTable"));
descriptor.addFamily(new HColumnDescriptor("myFamily"));
admin.createTable(descriptor);

Você precisa definir pelo menos uma família para a tabela, e o livro de referência do HBase recomenda não obter mais de 3 famílias de colunas, caso contrário você perderá desempenho.

Parabéns! Sua tabela foi criada!

Se você precisar excluir sua tabela, você pode usar

this.admin.disableTable(TableName.valueOf(tableName));
this.admin.deleteTable(TableName.valueOf(tableName));

Certifique-se de sempre desabilitar a tabela primeiro!

Agora você sabe como gerenciar tabelas no HBase.

Usando os filtros de varredura #

Basicamente, o objeto Scan recupera todas as linhas da tabela, mas e se você quiser recuperar apenas as linhas em que o valor de uma determinada coluna é igual a algo? Deixe-me apresentar os Filtros, eles funcionam como o WHERE no SQL.

Antes de começar a usar os filtros, se você sabe como suas row_keys são armazenadas, você pode definir uma linha inicial e uma final para seu Scan, o que otimizará sua consulta.

No HBase, row_keys são armazenados na ordem lexicográfica, mas você ainda pode usar o salting para alterar a forma como ele é armazenado, não vou explicar o salting neste tópico, levaria muito tempo e esse não é o ponto.

Vamos voltar aos nossos limites de linha, você tem dois métodos para usar para definir a linha inicial e final

Scan scan = new Scan();
scan.setStartRow(Bytes.toBytes("row_10"));
scan.setStopRow(Bytes.toBytes("row_42"));

Isso mudará o comportamento do seu scanner para buscar todas as linhas entre “row_10” e “row_42”.

NB : Como na maioria dos métodos “sub” (por exemplo, substring), o startRow é inclusivo e o stopRow é exclusivo.

Agora que podemos vincular nosso Scan, devemos agora adicionar alguns filtros aos nossos scans, existem muitos deles, mas veremos aqui os mais importantes.

  • Se você quiser recuperar todas as linhas com uma row_key começando por um determinado padrão

Use o RowPrefixFilter:

Scan scan = new Scan();
scan.setRowPrefixFilter(Bytes.toBytes("hello"));

Com este código, sua varredura apenas recuperará as linhas com uma row_key começando por “hello”.

  • Se você deseja recuperar todas as linhas em que o valor de uma determinada coluna é igual a algo

Use o SingleColumnValueFilter:

Scan scan = new Scan();
SingleColumnValueFilter filter = new SingleColumnValueFilter(Bytes.toBytes("myFamily"),Bytes.toBytes("myColumn"), CompareOp.EQUAL, Bytes.toBytes("42"));
scan.setFilter(filter);

Com este código, você obterá todas as linhas onde o valor da coluna myColumn é igual a 42. Você tem valores diferentes para CompareOp que são explicados na seção Parâmetros.

-Bom, mas e se eu quiser usar expressões regulares

Use o filtro RegexStringComparator:

Scan scan = new Scan();
RegexStringComparator comparator = new RegexStringComparator(".hello.");
SingleColumnValueFilter filter = new SingleColumnValueFilter(Bytes.toBytes("myFamily"),Bytes.toBytes("myColumn"), CompareOp.EQUAL, comparator);
scan.setFilter(filter);

E você obterá todas as linhas em que a coluna myColumn contém hello.

Observe também que o método Scan.setFilter() também pode receber uma lista de Filter como parâmetros

Conectando-se ao HBase #

Se você deseja se conectar a um servidor HBase, primeiro você precisa ter certeza de que o IP do servidor está em seu arquivo /etc/hosts, por exemplo, adicione a linha

255.255.255.255    hbase

Então você pode usar a API Java para se conectar ao zookeeper, você só precisa especificar a porta do cliente e o endereço do zookeeper

Configuration config =  HBaseConfiguration.create();
config.set("hbase.zookeeper.quorum", "hbase");
config.set("hbase.zookeeper.property.clientPort","2181");

Depois de configurar a conexão, você pode testá-la usando

HBaseAdmin.checkHBaseAvailable(config);

Se você tiver um problema com a configuração do HBase, uma exceção será lançada.

Finalmente para se conectar ao servidor, basta usar

Connection connection = ConnectionFactory.createConnection(config);
