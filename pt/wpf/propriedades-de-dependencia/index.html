Propriedades de dependência são um tipo de propriedade que estende uma propriedade CLR. Enquanto uma propriedade CLR é lida diretamente de um membro de sua classe, uma propriedade de dependência será resolvida dinamicamente ao chamar o método GetValue() que seu objeto ganha por meio de herança da classe base DependencyObject.

Esta seção detalhará as Propriedades de Dependência e explicará seu uso conceitualmente e por meio de exemplos de código.

Sintaxe #

  • DependencyProperty.Register(nome da string, Type propertyType, Type ownerType)
  • DependencyProperty.Register(string name, Type propertyType, Type ownerType, PropertyMetadata typeMetadata)
  • DependencyProperty.Register(string name, Type propertyType, Type ownerType, PropertyMetadata typeMetadata, ValidateValueCallback validateValueCallback)
  • DependencyProperty.RegisterAttached(string name, Type propertyType, Type ownerType)
  • DependencyProperty.RegisterAttached(string name, Type propertyType, Type ownerType, PropertyMetadata typeMetadata)
  • DependencyProperty.RegisterAttached(string name, Type propertyType, Type ownerType, PropertyMetadata typeMetadata, ValidateValueCallback validateValueCallback)
  • DependencyProperty.RegisterReadOnly(string name, Type propertyType, Type ownerType, PropertyMetadata typeMetadata)
  • DependencyProperty.RegisterReadOnly(string name, Type propertyType, Type ownerType, PropertyMetadata typeMetadata, ValidateValueCallback validateValueCallback)
  • DependencyProperty.RegisterAttachedReadOnly(nome da string, Tipo propertyType, Tipo ownerType, Tipo PropertyMetadataMetadata)
  • DependencyProperty.RegisterAttachedReadOnly(nome da string, Type propertyType, Type ownerType, PropertyMetadata typeMetadata, ValidateValueCallback validateValueCallback)

Parâmetros #

Parâmetro Detalhes
nome A representação String do nome da propriedade
propertyType O “Tipo” da propriedade, por exemplo, typeof(int)
proprietárioTipo O Type da classe na qual a propriedade está sendo definida, por exemplo typeof(MyControl) ou typeof(MyAttachedProperties).
typeMetadados Instância de System.Windows.PropertyMetadata (ou uma de suas subclasses) que define valores padrão, retornos de chamada de propriedade alterados, FrameworkPropertyMetadata permite definir opções de vinculação como System.Windows.Data.BindingMode.TwoWay.
validValueCallback Retorno de chamada personalizado que retorna true se o novo valor da propriedade for válido, caso contrário, false.

Propriedades de dependência padrão #

Quando usar #

Praticamente todos os controles WPF fazem uso intenso de propriedades de dependência. Uma propriedade de dependência permite o uso de muitos recursos do WPF que não são possíveis apenas com propriedades CLR padrão, incluindo, entre outros, suporte a estilos, animações, vinculação de dados, herança de valor e notificações de alteração.

A propriedade TextBox.Text é um exemplo simples de onde uma propriedade de dependência padrão é necessária. Aqui, a vinculação de dados não seria possível se Text fosse uma propriedade CLR padrão.

<TextBox Text="{Binding FirstName}" />

Como definir #

Propriedades de dependência só podem ser definidas em classes derivadas de DependencyObject, como FrameworkElement, Control, etc.

Uma das maneiras mais rápidas de criar uma propriedade de dependência padrão sem ter que lembrar a sintaxe é usar o trecho “propdp” digitando propdp e depois pressionando Tab. Um snippet de código será inserido e poderá ser modificado para atender às suas necessidades:

public class MyControl : Control
{
    public int MyProperty
    {
        get { return (int)GetValue(MyPropertyProperty); }
        set { SetValue(MyPropertyProperty, value); }
    }

    // Using a DependencyProperty as the backing store for MyProperty.
    // This enables animation, styling, binding, etc...
    public static readonly DependencyProperty MyPropertyProperty =
        DependencyProperty.Register("MyProperty", typeof(int), typeof(MyControl),
            new PropertyMetadata(0));
}

Você deve Tab pelas diferentes partes do trecho de código para fazer as alterações necessárias, incluindo a atualização do nome da propriedade, tipo de propriedade, tipo de classe que contém e o valor padrão.

Convenções importantes #

Existem algumas convenções/regras importantes a serem seguidas aqui:

  1. Crie uma propriedade CLR para a propriedade de dependência. Essa propriedade é usada no code-behind do seu objeto ou por outros consumidores. Ele deve invocar GetValue e SetValue para que os consumidores não precisem.

  2. Nomeie a propriedade de dependência corretamente. O campo DependencyProperty deve ser public static readonly. Deve ter um nome que corresponda ao nome da propriedade CLR e termine com “Propriedade”, por exemplo Texto e TextProperty.

  3. Não adicione lógica adicional ao setter da propriedade CLR. O sistema de propriedades de dependência (e XAML especificamente) não faz uso da propriedade CLR. Se você quiser realizar uma ação quando o valor da propriedade mudar, você deve fornecer um retorno de chamada via PropertyMetadata:

     public static readonly DependencyProperty MyPropertyProperty =
     DependencyProperty.Register("MyProperty", typeof(int), typeof(MyControl),
         new PropertyMetadata(0, MyPropertyChangedHandler));

 private static void MyPropertyChangedHandler(DependencyObject sender, DependencyPropertyChangedEventArgs args)
 {
     // Use args.OldValue and args.NewValue here as needed.
     // sender is the object whose property changed.
     // Some unboxing required.
 }

Modo de ligação #

Para eliminar a necessidade de especificar Mode=TwoWay em ligações (semelhante ao comportamento de TextBox.Text), atualize o código para usar FrameworkPropertyMetadata em vez de PropertyMetadata e especifique o sinalizador apropriado:

public static readonly DependencyProperty MyPropertyProperty =
    DependencyProperty.Register("MyProperty", typeof(int), typeof(MyControl), 
        new FrameworkPropertyMetadata(0, FrameworkPropertyMetadataOptions.BindsTwoWayByDefault));

Propriedades de dependência anexadas #

Quando usar #

Uma propriedade anexada é uma propriedade de dependência que pode ser aplicada a qualquer DependencyObject para aprimorar o comportamento de vários controles ou serviços que estão cientes da existência da propriedade.

Alguns casos de uso para propriedades anexadas incluem:

  1. Ter um elemento pai iterando por meio de seus filhos e agir sobre os filhos de uma certa maneira. Por exemplo, o controle Grid usa as propriedades anexadas Grid.Row, Grid.Column, Grid.RowSpan e Grid.ColumnSpan para organizar elementos em linhas e colunas.
  2. Adicionando visuais a controles existentes com modelos personalizados, por exemplo, adicionando marcas d’água a caixas de texto vazias em todo o aplicativo sem ter que subclassificar TextBox.
  3. Fornecer um serviço ou recurso genérico para alguns ou todos os controles existentes, por exemplo. ToolTipService ou FocusManager. Estes são comumente chamados de comportamentos anexados.
  4. Quando a herança para baixo da árvore visual é necessária, por exemplo. semelhante ao comportamento de DataContext.

Isso demonstra ainda mais o que está acontecendo no caso de uso Grid:

<Grid>
    <Grid.ColumnDefinitions>
        <ColumnDefinition />
        <ColumnDefinition />
    </Grid.ColumnDefinitions>

    <Label Grid.Column="0" Content="Your Name:" />
    <TextBox Grid.Column="1" Text="{Binding FirstName}" />
</Grid>

Grid.Column não é uma propriedade que existe em Label ou TextBox. Em vez disso, o controle Grid examina seus elementos filhos e os organiza de acordo com os valores das propriedades anexadas.

Como definir #

Continuaremos a usar Grid para este exemplo. A definição de Grid.Column é mostrada abaixo, mas o DependencyPropertyChangedEventHandler é excluído por questões de brevidade.

public static readonly DependencyProperty RowProperty =
    DependencyProperty.RegisterAttached("Row", typeof(int), typeof(Grid),
        new FrameworkPropertyMetadata(0, ...));

public static void SetRow(UIElement element, int value)
{
    if (element == null)
        throw new ArgumentNullException("element");

    element.SetValue(RowProperty, value);
}

public static int GetRow(UIElement element)
{
    if (element == null)
        throw new ArgumentNullException("element");

    return ((int)element.GetValue(RowProperty));
}

Como as propriedades anexadas podem ser anexadas a uma ampla variedade de itens, elas não podem ser implementadas como propriedades CLR. Em vez disso, um par de métodos estáticos é introduzido.

Assim, em contraste com as propriedades de dependência padrão, as propriedades anexadas também podem ser definidas em classes que não são derivadas de DependencyObject.

As mesmas convenções de nomenclatura que se aplicam às propriedades de dependência regulares também se aplicam aqui: a propriedade de dependência RowProperty tem os métodos correspondentes GetRow e SetRow.

Ressalvas #

Conforme documentado no MSDN:

Embora a herança de valor de propriedade possa parecer funcionar para propriedades de dependência não anexadas, o comportamento de herança para uma propriedade não anexada por meio de certos limites de elemento na árvore de tempo de execução é indefinido. Sempre use RegisterAttached para registrar propriedades nas quais você especifica Inherits nos metadados.

Propriedades de dependência somente leitura #

Quando usar #

Uma propriedade de dependência somente leitura é semelhante a uma propriedade de dependência normal, mas é estruturada para não permitir que seu valor seja definido fora do controle. Isso funciona bem se você tiver uma propriedade puramente informativa para os consumidores, por exemplo, IsMouseOver ou IsKeyboardFocusWithin.

Como definir #

Assim como as propriedades de dependência padrão, uma propriedade de dependência somente leitura deve ser definida em uma classe derivada de DependencyObject.

public class MyControl : Control
{
    private static readonly DependencyPropertyKey MyPropertyPropertyKey = 
        DependencyProperty.RegisterReadOnly("MyProperty", typeof(int), typeof(MyControl),
            new FrameworkPropertyMetadata(0));

    public static readonly DependencyProperty MyPropertyProperty = MyPropertyPropertyKey.DependencyProperty;

    public int MyProperty
    {
        get { return (int)GetValue(MyPropertyProperty); }
        private set { SetValue(MyPropertyPropertyKey, value); }
    }
}

As mesmas convenções que se aplicam às propriedades de dependência regulares também se aplicam aqui, mas com duas diferenças principais:

  1. A DependencyProperty é originada de uma DependencyPropertyKey privada.
  2. O configurador de propriedade CLR é protected ou private em vez de public.

Observe que o setter passa MyPropertyPropertyKey e não MyPropertyProperty para o método SetValue. Como a propriedade foi definida como somente leitura, qualquer tentativa de usar SetValue na propriedade deve ser usada com sobrecarga que recebe DependencyPropertyKey; caso contrário, um InvalidOperationException será lançado.
