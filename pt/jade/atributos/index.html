&atributos #

Pronunciado “and attribute”, a sintaxe &attributes pode ser usada para explodir um objeto em atributos de um elemento.

Código: #

div#foo(data-bar="foo")&attributes({'data-foo': 'bar'})

Resultado: #

<div id="foo" data-bar="foo" data-foo="bar"></div>

O objeto não precisa ser um literal de objeto. Também pode ser apenas uma variável que tenha um objeto como seu valor (consulte também Mixin Attributes)

Código: #

- var attributes = {'data-foo': 'bar'};
div#foo(data-bar="foo")&attributes(attributes)

Resultado: #

<div id="foo" data-bar="foo" data-foo="bar"></div>

Atributos normais #

Etiqueta HTML: #

Os atributos de tag são semelhantes ao html, porém seus valores são apenas JavaScript normal.

a(href='google.com') Google
a(class='button', href='google.com') Google

Resultado: #

<a href="google.com">Google</a><a href="google.com" class="button">Google</a>

Variável: #

Todas as expressões normais de JavaScript também funcionam bem:

- var authenticated = true
body(class=authenticated ? 'authed' : 'anon')

Resultado: #

<body class="authed"></body>

Muitos atributos #

Se você tiver muitos atributos, também poderá espalhá-los em várias linhas:

input(
  type='checkbox'
  name='agreement'
  checked
)

Resultado: #

<input type="checkbox" name="agreement" selected="checked"/>

Atributos sem escape #

Por padrão, todos os atributos são escapados (substituindo caracteres especiais por sequências de escape) para evitar ataques como scripts entre sites. Se você precisar usar caracteres especiais, você pode usar != em vez de =.

Código: #

div(escaped="<code>")
div(unescaped!="<code>")

Resultado: #

<div escaped="&lt;code&gt;"></div>
<div unescaped="<code>"></div>

Atributos booleanos #

Atributos booleanos são espelhados por Jade e aceitam bools, também conhecidos como true ou false. Quando nenhum valor é especificado, true é assumido.

Código: #

input(type='checkbox', checked)
input(type='checkbox', checked=true)
input(type='checkbox', checked=false)
input(type='checkbox', checked=true.toString())

Resultado: #

<input type="checkbox" checked="checked"/>
<input type="checkbox" checked="checked"/>
<input type="checkbox"/>
<input type="checkbox" checked="true"/>

Se o doctype for html jade sabe que não deve espelhar o atributo e usa o estilo conciso (compreendido por todos os navegadores).

Código: #

doctype html
input(type='checkbox', checked)
input(type='checkbox', checked=true)
input(type='checkbox', checked=false)
input(type='checkbox', checked=true && 'checked')

Resultado: #

<!DOCTYPE html>
<input type="checkbox" checked>
<input type="checkbox" checked>
<input type="checkbox">
<input type="checkbox" checked="checked">

Atributos de estilo #

O atributo style pode ser uma string (como qualquer atributo normal), mas também pode ser um objeto, o que é útil quando partes do estilo são geradas por JavaScript.

Código: #

a(style={color: 'red', background: 'green'})

Resultado: #

<a style="color:red;background:green"></a>

Atributos de classe #

O atributo class pode ser uma string (como qualquer atributo normal), mas também pode ser um array de nomes de classes, o que é útil quando gerado a partir de JavaScript.

Código: #

- var classes = ['foo', 'bar', 'baz']
a(class=classes)
//- the class attribute may also be repeated to merge arrays
a.bing(class=classes class=['bing'])

Resultado: #

<a class="foo bar baz"></a><a class="bing foo bar baz bing"></a>

Também pode ser um objeto mapeando nomes de classe para valores verdadeiros ou falsos, o que é útil para aplicar classes condicionais

Código: #

- var currentUrl = '/about'
a(class={active: currentUrl === '/'} href='/') Home
a(class={active: currentUrl === '/about'} href='/about'

Resultado: #

<a href="/">Página inicial</a><a href="/about" class="active">Sobre</a>

Literal de Classe #

As classes podem ser definidas usando uma sintaxe .classname:

Código: #

a.botão

Resultado: #

<a class="button"></a>

Como div’s são uma escolha tão comum de tag, é o padrão se você omitir o nome da tag:

Código: #

.conteúdo

Resultado: #

<div class="content"></div>

Literal de ID #

Os IDs podem ser definidos usando uma sintaxe #idname:

Código: #

a#link principal

Resultado: #

<a id="main-link"></a>

Como div’s são uma escolha tão comum de tag, é o padrão se você omitir o nome da tag:

Código: #

#conteúdo

Resultado: #

<div id="content"></div>
