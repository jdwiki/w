Sintaxe #

Pug (o nome antigo é Jade) é uma sintaxe limpa e sensível a espaços em branco para escrever HTML. Aqui está um exemplo simples:

doctype html
html(lang="en")
  head
    title= pageTitle
    script(type='text/javascript').
      if (foo) bar(1 + 5)
  body
    h1 Pug - node template engine
    #container.col
      if youAreUsingPug
        p You are amazing
      else
        p Get on it!
      p.
        Pug is a terse and simple templating language with a
        strong focus on performance and powerful features.

Produz a seguinte saída como HTML

<!DOCTYPE html>
<html lang="en">
  <head>
    <title>Pug</title>
    <script type="text/javascript">
      if (foo) bar(1 + 5)
    </script>
  </head>
  <body>
    <h1>Pug - node template engine</h1>
    <div id="container" class="col">
      <p>You are amazing</p>
      <p>Pug is a terse and simple templating language with a strong focus on performance and powerful features.</p>
    </div>
  </body>
</html>

Aqui estão as regras para renderizar o código Pug para HTML:

  1. Ao recuar o texto, a árvore HTML será construída. o recuo pode ser usado com espaços ou tabulações. Isso não pode ser misturado!
  2. As tags HTML são escritas sem < e >. Atributos são lugares entre colchetes.
  3. O comentário pode ser feito com // ou <!-- -->. Comentários com //- não são visíveis no HTML renderizado.
  4. Com #{ } será gerado um modelo oferecido: #{header} #{user.username}.
  5. O # (hashtag) sem chaves será um elemento div criado com o texto como ID. Exemplo #myID será renderizado como <div id="myID"></div>.
  6. Com um . (ponto) será um div gerado com um atributo de classe. Exemplo: .myClass será renderizado como <div class="myClass"></div>
  7. Com =  (sinal de igualdade seguido de um espaço), uma variável será recuperada. Exemplo: h1= título
  8. Um != (diferente de) recuperou uma variável sem escape.
  9. Um - (hífen) permite que você escreva JavaScript. Exemplo: - console.log("foo");
  10. O link para um arquivo externo pode ser o seguinte: script(src="/js/chat.js")
  11. O script embutido pode usar este script..
  12. Uma diretiva para adicionar o layout básico: extends ../layout.
  13. Em layout.pug acontece a inserção usando block content
  14. O uso de parciais pode ser feito de duas maneiras:
    1. by partial: != partial(template file name/options).
    2. By include: include ../includes/footer
  15. O inverso de include é extend. Isso permite que uma página “html block parts” seja enviada para uma página de layout, por exemplo: extend layout
  16. A concatenação acontece com o caractere + (mais) ou # (hashtag). O plus é usado no código JavaScript. A hashtag em HTML e renderiza o conteúdo: `p O nome é: #{myName}

Usando pug com Node.js #

var pug = require('pug');

// compile
var fn = pug.compile('string of pug', options);
var html = fn(locals);

// render
var html = pug.render('string of pug', merge(options, locals));

// renderFile
var html = pug.renderFile('filename.pug', merge(options, locals));

Opções

  • nome do arquivo Usado em exceções e obrigatório ao usar inclusões
  • compileDebug Quando false nenhuma instrumentação de depuração é compilada
  • pretty Adicionar espaço em branco de recuo bonito à saída (falso por padrão)

Instalação ou Configuração #

Antes de iniciar o código com o Pug, você precisa ter alguns pré-requisitos.

Você precisará instalar:

  • NodeJS com NPM
  • ExpressJS (opcional)

Após instalar o NodeJS, você pode verificar em seu terminal a instalação correta fazendo:

$ nó -v

Se for bem sucedido, imprimirá o número da versão do Node.

Para instalar o Pug em seu projeto, a forma preferida e fácil é através do NPM (Node Package Manager). Se você estiver familiarizado com isso, basta executar esta linha de código no seu Terminal:

$ npm instalar pug

Se você quiser instalar globalmente, você pode digitar:

$ npm instala pug-cli -g

e correr com

$ pug --help
