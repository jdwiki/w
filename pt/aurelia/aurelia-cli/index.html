Adicionando Bootstrap a um aplicativo CLI #

Uma biblioteca CSS/Javascript comumente usada é Bootstrap. Para instalá-lo em seu aplicativo Aurelia CLI, primeiro você precisa instalá-lo usando Npm.

npm install bootstrap --save

Como o Bootstrap tem uma dependência rígida do jQuery, precisamos ter certeza de que também temos o jQuery instalado:

npm install jquery --save

Agora, em seu IDE/editor de código preferido, abra o seguinte arquivo no diretório do seu projeto: aurelia_project/aurelia.json e role para baixo até a seção build.bundles no final do arquivo. Vamos querer adicionar Bootstrap a um dos pacotes. Para este exemplo, adicionaremos jQuery e Bootstrap ao pacote do fornecedor.

"jquery",
{
  "name": "bootstrap",
  "path": "../node_modules/bootstrap/dist",
  "main": "js/bootstrap.min",
  "deps": ["jquery"],
  "exports": "$",
  "resources": [
    "css/bootstrap.css"
  ]
}

Isso tornará o Bootstrap acessível em sua aplicação e importável via import 'bootstrap' em seu código (esta é a propriedade name definida acima). Observe a referência a jquery na seção "deps": [] de nossa definição de bootstrap. Também especificamos que queremos agrupar o CSS do Bootstrap em nosso arquivo principal vendor-bundle.js usando a propriedade "resources":[]

Por último e não menos importante, para usar nossa dependência Bootstrap recém-adicionada, queremos primeiro importar a biblioteca Bootstrap dentro do nosso arquivo main.js colocando o seguinte no início do arquivo.

import 'bootstrap';

Queremos incluir o CSS do Bootstrap de um lugar onde ele seja globalmente acessível para toda a aplicação, então dentro de app.html coloque o seguinte entre as tags <template></template>.

<require from="bootstrap/css/bootstrap.css"></require>

Adicionando Lodash a um aplicativo CLI #

Uma biblioteca de utilitários comumente usada é Lodash. Para instalá-lo em seu aplicativo Aurelia CLI, primeiro você precisa instalá-lo usando Npm.

npm instala lodash --save

Agora, em seu IDE/editor de código preferido, abra o seguinte arquivo no diretório do projeto: aurelia_project/aurelia.json e role para baixo até a seção build.bundles no final do arquivo. Queremos adicionar o Lodash a um dos pacotes. Para este exemplo, adicionaremos o Lodash ao pacote do fornecedor.

Como o Lodash segue as convenções do CommonJS e exporta um arquivo concatenado, podemos registrá-lo facilmente adicionando o seguinte objeto ao array de dependências:

{
  "name": "lodash",
  "path": "../node_modules/lodash/lodash.min"
}

Observe que omitimos o sufixo do arquivo .js, pois eles serão anexados automaticamente ao resolver a dependência.

Agora podemos usar o Lodash simplesmente importando-o, usando a sintaxe de importação do ES6:

importar _ de 'lodash';

Instalando o plug-in Aurelia-I18N #

Para obter o I18N Plugin oficial em seu projeto CLI, precisamos instalá-lo seguindo as próximas etapas.

Primeiro você deseja instalar o plugin via npm:

npm instalar aurelia-i18n

Como o Aurelia-I18N é suportado pelo i18next, você deve instalá-lo e um plug-in de back-end de sua escolha. Como exemplo, vamos aproveitar o backend i18next-xhr:

npm instalar i18next i18next-xhr-backend

Depois disso, precisamos informar ao nosso aplicativo CLI sobre as novas dependências. Para fazer isso, vamos abrir o arquivo aurelia_project/aurelia.json e rolar para baixo até a seção chamada dependencies. Lá, adicione as três entradas a seguir:

{
  "name": "i18next",
  "path": "../node_modules/i18next/dist/umd",
  "main": "i18next"
},
{
  "name": "aurelia-i18n",
  "path": "../node_modules/aurelia-i18n/dist/amd",
  "main": "aurelia-i18n"
},
{
  "name": "i18next-xhr-backend",
  "path": "../node_modules/i18next-xhr-backend/dist/umd",
  "main": "i18nextXHRBackend"
}

Ótimo, agora seguindo o Guia Aurelia-I18N oficial, criamos uma pasta na raiz do seu aplicativo chamada locales.

Você precisa colocar a pasta na raiz (no mesmo nível que src), pois esta é a raiz hospedada do seu aplicativo

Lá, adicione subpastas para cada idioma que você gostaria de oferecer suporte, por exemplo, en e de para o idioma inglês e alemão.

Dentro de cada uma dessas pastas, crie um arquivo chamado translation.json com suas chaves e valores de tradução. Siga o guia oficial para obter informações detalhadas.

Por último, mas não menos importante, é hora de conectar o plug-in dentro do seu aplicativo. Portanto, vá para o seu arquivo src/main.js e configure-o da seguinte forma.

/**********************************************/
/          add the necessary imports           /
/**********************************************/
import environment from './environment';
import Backend from 'i18next-xhr-backend';

//Configure Bluebird Promises.
//Note: You may want to use environment-specific configuration.
Promise.config({
  warnings: {
    wForgottenReturn: false
  }
});

export function configure(aurelia) {
  aurelia.use
    .standardConfiguration()
    .feature('resources');

  if (environment.debug) {
    aurelia.use.developmentLogging();
  }

  if (environment.testing) {
    aurelia.use.plugin('aurelia-testing');
  }

  /**********************************************/
  /             register the plugin              /
  /**********************************************/
  aurelia.use.plugin('aurelia-i18n', (instance) => {
    // register backend plugin
    instance.i18next.use(Backend);

    // adapt options to your needs (see http://i18next.com/docs/options/)
    // make sure to return the promise of the setup method, in order to guarantee proper loading
    return instance.setup({
      backend: {                                  // <-- configure backend settings
        loadPath: './locales/{{lng}}/{{ns}}.json', // <-- XHR settings for where to get the files from
      },
      lng : 'de',
      attributes : ['t','i18n'],
      fallbackLng : 'en',
      debug : false
    });
  });

  aurelia.start().then(() => aurelia.setRoot());
}

Adicionando a configuração do Aurelia a um aplicativo CLI #

Adicionar a configuração aurelia a um aplicativo cli às vezes produz um erro de compilação. Isso é causado por uma dependência ausente, então simplesmente adicionamos a dependência ao pacote de compilação.

Tente o seguinte:

<pré><código> npm instala deep-extend –save npm install aurelia-configuration –save

Agora adicione o seguinte ao arquivo aurelia.json na matriz de dependências:

<pré><código> { “name”: “deep-extend”, “path”: “../node_modules/deep-extend/lib/deep-extend” }, { “name”: “aurelia-configuration”, “path”: “../node_modules/aurelia-configuration/dist/amd”, “main”: “index” }
