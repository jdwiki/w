Instalação ou Configuração #

Download #

Use o site Firebird para baixar o “pacote do servidor” correto para seu sistema. Primeiro, selecione a versão do Firebird que deseja instalar. Em seguida, selecione o instalador apropriado para o seu sistema. Por exemplo, para quase qualquer versão do Windows 32 bits, você selecionaria em kits de 32 bits a opção com “instalador executável do Windows recomendado para usuários iniciantes”.

Instalando #

Execute o instalador e siga as instruções. Para usuários iniciantes, você provavelmente não precisará alterar nenhuma configuração no instalador.

Qual é a versão do ODS e como recuperá-la? #

A versão ODS (estrutura em disco) é um número que representa a versão da estrutura de layout de dados de baixo nível (ODS) do banco de dados. Quando um novo recurso é adicionado ao Firebird, ele pode ou não exigir que a estrutura das páginas do banco de dados ou tabelas do sistema (metadados do banco de dados) seja alterada. Se isso acontecer, a versão do ODS deve aumentar.

Este número é verificado na conexão, para que o servidor tenha certeza de que pode ’entender’ a estrutura do banco de dados. Por exemplo, quando você tenta se conectar com um servidor 1.0 a um banco de dados criado com Firebird 2.0, você receberá um erro porque o servidor 1.0 não é capaz de lidar com esse ODS - simplesmente porque existem campos cujo significado ele não entende.

O Firebird 2.5 pode abrir bancos de dados com ODS do Interbase 5, Interbase 6 e Firebird 0.9 a 2.5. No entanto, o Firebird 3.0 foi um começo limpo em relação à compatibilidade com versões anteriores e não pode mais abrir bancos de dados com versões ODS de versões anteriores do Firebird.

A versão do ODS, conforme relatado pelas ferramentas do usuário, mostra com qual versão do servidor o banco de dados foi criado, por exemplo:

-------------------------------------------------------------
Database created with version:                    ODS version:
InterBase® 5                                                9
InterBase® 5.5, 5.6                                        9.1
InterBase® 6    /   Firebird 1.0                          10.0
InterBase® 6.5  /   Firebird 1.5                          10.1
InterBase® 7    /   Firebird 2.0                            11
InterBase® 7.1  /   Firebird 2.1                          11.1
InterBase® 7.5  /   Firebird 2.5                          11.2
InterBase® 2007 /   Firebird 3.0                            12
InterBase® 2009                                             13
InterBase® XE                                             15.0

NOTA 1: Quando a mesma versão do ODS é relatada para algumas versões do Interbase e Firebird, isso NÃO significa que o ODS é o mesmo, portanto, NÃO significa compatibilidade entre os limites IB/FB! Exceto para Firebird 0.9 e 1.0 e Interbase 6.0 que eram quase compatíveis. Naquela época, esperava-se que o Interbase fosse mantido em código aberto e reutilizasse o código do projeto Firebird. No entanto, com o Interbase 6.5 isso mudou. O que isso significa praticamente aqui, é que enquanto alguns bancos de dados Interbase/Yaffil/Firebird podem relatar ter a mesma versão (número) do ODS, a própria estrutura (o próprio ODS) deles estava ficando cada vez mais diferente. O IB 7 não abriria o banco de dados FB 2 e vice-versa - eles têm diferentes formatos internos (ODS), enquanto os dois projetos separados agora dão a eles o mesmo número de versão. A versão ODS pode ser a mesma entre algumas versões IB e FB, mas a própria ODS (exceto IB6.0) não seria!

NOTA 2: com o Firebird versão 1.5 foi introduzida a versão de 64 bits do servidor. Bancos de dados criados com versões de 64 bits e 32 bits do Firebird 1.5 estão reportando a versão 10.1 do ODS, mas seus ODSs reais são um pouco diferentes e não podem abrir bancos de dados um do outro. Começando com o FB 2.0 que foi corrigido e ambas as versões x86 e x64 do Firebird Server podem abrir bancos de dados criados um pelo outro.

Para recuperar a versão do ODS você pode usar a API do Firebird, ou simplesmente usar a ferramenta que a lê para você.

Se você só tem acesso à linha de comando, você pode usar a ferramenta de linha de comando gstat do Firebird (localizada no diretório bin). Sua opção -h gera as informações da página de cabeçalho, que contém o ODS:

gstat –h database_file_name

Usuário e senha aqui desnecessários, pois gstat com opção –h apenas lê parte física do banco de dados (página de cabeçalho, número 0).

Se o gstat não entender as informações de leitura, ele mostrará a mensagem correspondente – o que esperava e o que encontrou.

Se você tiver apenas uma conexão remota com o servidor e puder fazer login no banco de dados, mas não tiver acesso ao arquivo do banco de dados em si, a partir do Firebird 2.1 você também poderá consultar o ODS por comandos SQL regulares usando as Tabelas de Monitoramento.

     select MON$ODS_MAJOR, MON$ODS_MINOR from MON$DATABASE
