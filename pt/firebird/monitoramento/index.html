Com Firebird 2.1 e bancos de dados com ODS 11.1 (e superior) Firebird introduz a capacidade de monitorar a atividade do lado do servidor que acontece dentro de um banco de dados específico.

O monitoramento completo do banco de dados está disponível para o SYSDBA e o proprietário do banco de dados. Os usuários comuns estão restritos apenas às informações sobre seus próprios anexos—outros anexos são invisíveis para eles.

Tabelas de monitoramento disponíveis desde Firebird 2.1 e ODS 11.1:

  • MON$DATABASE (banco de dados conectado)
  • MON$ATTACHMENTS (anexos conectados)
  • MON$TRANSACTIONS (transações iniciadas)
  • MON$STATEMENTS (declarações preparadas)
  • MON$CALL_STACK (pilha de chamadas de solicitações PSQL ativas)
  • MON$IO_STATS (estatísticas de E/S)
  • MON$RECORD_STATS (estatísticas em nível de registro)

Referência: Notas de versão do Firebird 2.1

Com Firebird 2.5 e bancos de dados com ODS 11.2 (e superior) Firebird adiciona as seguintes novas tabelas de monitores

  • MON$MEMORY_USAGE (uso de memória atual)
  • MON$CONTEXT_VARIABLES (variáveis ​​de contexto conhecidas)

Referência: Notas de versão do Firebird 2.5

Informações sobre as conexões de banco de dados

SELECT
       a.mon$attachment_id as Attachment_ID,
       a.mon$server_pid as Server_PID, 
       case a.mon$state 
          when 1 then 'active'
          when 0 then 'idle'
       end as State, 
       a.mon$attachment_name as Database_Name, 
       a.mon$user as User_Name, 
       a.mon$role as Role_Name, 
       a.mon$remote_protocol as Remote_Protocol, 
       a.mon$remote_address as  Remote_Address, 
       a.mon$remote_pid as Remote_PID, 
       cs.rdb$character_set_name as Connection_Character_Set, 
       a.mon$timestamp as Established_At,
       case a.mon$garbage_collection 
          when 1 then 'allowed'
          when 0 then 'not allowed'
       end as Garbage_Collection, 
       a.mon$remote_process as Remote_Process, 
       a.mon$stat_id as Statistics_ID
    FROM
       mon$attachments a, rdb$character_sets cs
    where 
       (a.mon$character_set_id = cs.rdb$character_set_id)

Exemplos mais específicos

Informações sobre os clientes conectados.

SELECT
   a.mon$remote_protocol as Remote_Protocol,
   a.mon$remote_address as  Remote_Address,
   a.mon$remote_pid as Remote_PID,
   a.mon$timestamp as Established_At,
   a.mon$remote_process as Remote_Process
FROM
   mon$attachments a

Recupere PIDs de todos os processos do servidor que carregam a CPU no momento (interessante com uma Arquitetura de Servidor Clássica)

SELECT
   MON$SERVER_PID
FROM
   MON$ATTACHMENTS
WHERE
   MON$STATE = 1

Recuperar informações sobre os usuários conectados, estações de trabalho e aplicativos cliente

SELECT
   mon$attachment_name as Database_Name,
   mon$user as User_Name,
   mon$role as Role_Name,
   mon$remote_process as Client_Application,
   mon$remote_address as Client_IP,
   mon$remote_pid as Client_Application_PID
FROM
   mon$attachments
