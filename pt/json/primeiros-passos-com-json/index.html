Matriz JSON #

Uma matriz JSON é uma coleção ordenada de valores. Ele é cercado por colchetes, ou seja, [], e os valores são delimitados por vírgulas:

{ "colors" : [ "red", "green", "blue" ] }

Arrays JSON também podem conter qualquer elemento JSON válido, incluindo objetos, como neste exemplo de um array com 2 objetos (retirado do documento RFC):

[
  {
     "precision": "zip",
     "Latitude":  37.7668,
     "Longitude": -122.3959,
     "Address":   "",
     "City":      "SAN FRANCISCO",
     "State":     "CA",
     "Zip":       "94107",
     "Country":   "US"
  },
  {
     "precision": "zip",
     "Latitude":  37.371991,
     "Longitude": -122.026020,
     "Address":   "",
     "City":      "SUNNYVALE",
     "State":     "CA",
     "Zip":       "94085",
     "Country":   "US"
  }
]

Eles também podem conter elementos com tipos mistos, por exemplo:

[
    "red",
    51,
    true,
    null,
    {
        "state": "complete"
    }
]

Um erro comum ao escrever arrays JSON (e objetos) é deixar uma vírgula à direita após o último elemento. Este é um padrão comum em muitas linguagens, mas infelizmente não é válido em JSON. Por exemplo, a seguinte matriz é inválida:

[
    1,
    2,
]

Para tornar isso válido, você precisaria remover a vírgula após o último elemento, transformando-o em:

[
    1,
    2
]

Objeto JSON #

Um objeto JSON é cercado por chaves e contém pares chave-valor.

{ "key1": "value1", "key2": "value2", ... }

As chaves devem ser strings válidas, portanto, entre aspas duplas. Os valores podem ser objetos JSON, números, strings, arrays ou um dos seguintes ’nomes literais’: false, null ou true. Em um par chave-valor, a chave é separada do valor por dois pontos. Vários pares de valores-chave são separados por vírgulas.

A ordem nos objetos não é importante. O seguinte objeto JSON é, portanto, equivalente ao acima:

{ "key2": "value2", "key1": "value1", ... }

Para resumir, este é um exemplo de um objeto JSON válido:

{
  "image": {
    "width":  800,
    "height": 600,
    "title":  "View from 15th Floor",
    "thumbnail": {
      "url": "http://www.example.com/image/481989943",
      "height": 125,
      "width":  100
    },
    "visible": true,
    "ids": [116, 943, 234, 38793]
  }
}

Exemplos comuns de objetos JSON, com equivalentes de objetos (Java) relacionados #

Ao longo deste exemplo, assume-se que o objeto ‘root’ que está sendo serializado para JSON é uma instância da seguinte classe:

public class MyJson {
}

 

Exemplo 1 : Um exemplo de uma instância de MyJson, como está:

{}

ou seja, como a classe não possui campos, apenas colchetes são serializados. As chaves são os delimitadores comuns para representar um objeto. Observe também como o objeto raiz não é serializado como um par chave-valor. Isso também é verdade para tipos simples (String, números, arrays) quando não são campos de um objeto (externo).

 

Exemplo 2: Vamos adicionar alguns campos ao MyJson e inicializá-los com alguns valores:

// another class, useful to show how objects are serialized when inside other objects
public class MyOtherJson {} 

// an enriched version of our test class
public class MyJson {
  String myString = "my string";
  int myInt = 5;
  double[] myArrayOfDoubles = new double[] { 3.14, 2.72 };
  MyOtherJson objectInObject = new MyOtherJson();    
}

Esta é a representação JSON relacionada:

{
  "myString" : "my string",
  "myInt" : 5,
  "myArrayOfDoubles" : [ 3.14, 2.72 ],
  "objectInObject" : {}
}

Observe como todos os campos são serializados em uma estrutura de valor-chave, onde a chave é o nome do campo que contém o valor. Os delimitadores comuns para matrizes são colchetes. Observe também que cada par chave-valor é seguido por uma vírgula, exceto o último par.

Editando JSON manualmente #

JSON é um formato muito estrito (consulte http://json.org). Isso facilita a análise e a escrita para máquinas, mas surpreende os humanos quando um erro imperceptível quebra o documento.

Problemas comuns #

Vírgula à direita #

Ao contrário da maioria das linguagens de programação, você não tem permissão para adicionar uma vírgula à direita:

{
  a: 1,
  b: 2,
  c: 3
}

Adicionar uma vírgula após 3 produzirá um erro de sintaxe.

O mesmo problema existe para matrizes:

[ 
  1,
  2
]

Você deve tomar cuidado extra se precisar reordenar os itens.

Faltando vírgula #

{
  a: 1,
  b: 2,
  c: 3
  d: 4
}

Como as vírgulas à direita não são permitidas, é fácil esquecer de anexar uma antes de adicionar um novo valor (neste caso, após 3).

Comentários #

O JSON não permite comentários, pois é um formato de intercâmbio de dados. Este ainda é um tópico quente, embora sem respostas claras além de não usá-los.

Existem várias soluções alternativas:

  • Use comentários de estilo C e remova-os antes de passá-los para o analisador
  • Incorporar comentários nos dados
{
  "//": "comment",
  "data": 1
}
  • Incorporar comentários e sobrescrevê-los com dados
{
  "data": "comment",
  "data": 1
}

A segunda entrada data substituirá o comentário na maioria dos analisadores.

Soluções #

Para facilitar a gravação de JSON, use um IDE que verifique erros de sintaxe e forneça cores de sintaxe. Plugins estão disponíveis para a maioria dos editores.

Ao desenvolver aplicativos e ferramentas, use JSON internamente e como um protocolo, mas tente não expô-lo em locais onde um humano provavelmente seria obrigado a editá-lo manualmente (exceto para depuração).

Avalie outros formatos mais adequados para esse uso, como:

  • Hjson, pode ser facilmente convertido de e para JSON
  • TOML, semelhante aos arquivos INI
  • YAML, mais recursos, mas ao custo de maior complexidade

Regras de sintaxe JSON #

A sintaxe JSON (JavaScript Object Notation) é baseada em um subconjunto de JavaScript (consulte também json.org).

Uma expressão JSON válida pode ser um dos seguintes tipos de dados

  • tipos de dados simples: String, Number, Boolean, Null
  • tipos de dados compostos: Valor, Objeto, Matriz

Tipos de dados simples #

Uma string JSON deve ser colocada entre aspas duplas e pode conter zero ou mais caracteres Unicode; escapes de barra invertida são permitidos. Os números JSON aceitos estão em notação E. Boolean é um dos true, false. Null é a palavra reservada null.

Tipo de dados Exemplos de JSON válido
### Cadeia "maçã"
"Maçã"
"\u00c4pfel\n"
""
### Número 3
1.4
-1.5e3
### Booleano verdadeiro
falso
### Nulo nulo

Tipos de dados compostos #

Valor #

Um valor JSON pode ser: String, Number, Boolean, Null, Object, Array.

Objeto #

Um objeto JSON é uma coleção não ordenada separada por vírgulas de pares nome:valor entre chaves onde nome é uma String e valor um valor JSON.

Variedade #

Uma matriz JSON é uma coleção ordenada de valores JSON.

Exemplo de uma matriz JSON:

["home", "wooden"]

Exemplos de objetos JSON:

{
    "id": 1,
    "name": "A wooden door",
    "price": 12.50,
    "tags": ["home", "wooden"]
}

[
  1,
  2,
  [3, 4, 5, 6],
  {
    "id": 1,
    "name": "A wooden door",
    "price": 12.50,
    "tags": ["home", "wooden"]
  }
]

Racional para Array vs Object (ou seja, quando usar o quê) #

As matrizes JSON representam uma coleção de objetos. Em JS, há um monte de funções de coleção como slice, pop, push. Os objetos têm apenas mais dados brutos.

Um JSONArray é uma sequência ordenada de valores. Seu formato de texto externo é uma string entre colchetes com vírgulas separando os valores.

Um JSONObject é uma coleção não ordenada de pares nome/valor. Sua forma externa é uma string envolta em chaves com dois pontos entre os nomes e valores e vírgulas entre os valores e nomes.

Objeto - chave e valor, Array - numerais, strings, booleanos. Quando você usa isso ou aquilo?

Você pode pensar em Arrays como “é um/um” e Objetos como “tem um”. Vamos usar “Fruit” como exemplo. Cada item na matriz de frutas é um tipo de fruta.

array fruit : [laranja, manga, banana]

Arrays podem conter objetos, strings, números, arrays, mas vamos lidar apenas com objetos e arrays.

array fruit : [orange:[], manga:{}, banana:{}]

. Você pode ver que laranja também é uma matriz. Isso implica que qualquer item que vai para laranja é um tipo de laranja, digamos: bitter_orange, mandarin, sweet_orange.

para o objeto fruta, qualquer item nele é um atributo da fruta. assim o fruto tem

object fruit :{seed:{}, endocarp:{},flesh:{}}

Isso também implica que qualquer coisa dentro do objeto semente deve ser propriedade da semente, digamos: cor, ..

JSON é principalmente uma linguagem que permite serializar objetos javascript em strings. Portanto, ao desserializar uma string JSON, você deve obter uma estrutura de objeto javascript. Se o seu json desseriar em um objeto que armazena 100 objetos chamados object1 para object100, isso será muito inconveniente. A maioria dos desserializadores espera que você tenha objetos conhecidos e arrays de objetos conhecidos para que eles possam converter as strings na estrutura de objeto real na linguagem que você está usando. Também esta é uma pergunta que a filosofia do design orientado a objetos responderia a você.
