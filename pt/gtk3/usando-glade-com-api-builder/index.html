[C++] usando Gtk::Builder em gtkmm #

Visão geral #

O Gtk+ suporta um fluxo de trabalho onde a tarefa de design da interface do usuário e a tarefa de programação são dissociadas. Embora os elementos da interface do usuário, como botões, menus, layout, etc., possam ser adicionados diretamente do código, essa abordagem não apenas atrapalha o código, mas também dificulta a alteração da interface do usuário para qualquer pessoa, exceto para o programador. Além disso, alguns elementos da interface são usados ​​apenas para manter a estrutura do layout e não precisam participar da lógica, adicioná-los a partir do código apenas o torna mais longo. Em vez disso, o Glade pode ser usado para gerar a descrição da interface do usuário como XML e a API Gtk+ Builder pode ser usada para carregar a interface do usuário e operar nela.

Fluxo de trabalho #

  1. Crie elementos de interface do usuário em Glade usando arrastar e soltar. Glade gera arquivo XML que contém descrições de UI. Isso também pode ser feito manualmente escrevendo a sintaxe XML adequada e salvando-a com uma extensão .glade.
  2. Carregue a interface do usuário do arquivo diretamente com 
     auto ui = Gtk::Builder::create_from_file("ui.glade");
  3. Acesse elementos individuais da interface do usuário 
     // when element doesn't need to be added to another UI element
 auto ui_elem = Glib::RefPtr<Gtk::Button>::cast_dynamic(
     ui->get_object("button_UI_id")
 );
 // when element needs to be added to another widget
 Gtk::Button *btn = nullptr;
 ui->get_widget<Gtk::Button>("button_UI_id", btn);

Exemplo #

simples.glade

<?xml version="1.0" encoding="UTF-8"?>
<!-- Generated with glade 3.20.0 -->
<interface>
  <requires lib="gtk+" version="3.20"/>
  <object class="GtkBox" id="cont">
    <property name="width_request">200</property>
    <property name="height_request">200</property>
    <property name="visible">True</property>
    <property name="can_focus">False</property>
    <property name="orientation">vertical</property>
    <child>
      <object class="GtkLabel" id="display_label">
        <property name="visible">True</property>
        <property name="can_focus">False</property>
        <property name="wrap">True</property>
        <attributes>
          <attribute name="weight" value="bold"/>
          <attribute name="scale" value="5"/>
          <attribute name="foreground" value="#a4a400000000"/>
        </attributes>
      </object>
      <packing>
        <property name="expand">True</property>
        <property name="fill">True</property>
        <property name="position">0</property>
      </packing>
    </child>
    <child>
      <object class="GtkButton" id="display_button">
        <property name="label" translatable="yes">Display Message</property>
        <property name="visible">True</property>
        <property name="can_focus">True</property>
        <property name="receives_default">True</property>
      </object>
      <packing>
        <property name="expand">False</property>
        <property name="fill">True</property>
        <property name="position">1</property>
      </packing>
    </child>
  </object>
</interface>

simples.cpp

#include <gtkmm/application.h>
#include <gtkmm/applicationwindow.h>
#include <gtkmm/button.h>
#include <gtkmm/label.h>
#include <gtkmm/box.h>
#include <gtkmm/builder.h>

class HelloWindow : public Gtk::ApplicationWindow {
    Gtk::Box *cont;
    Glib::RefPtr<Gtk::Label> display_label;
    Glib::RefPtr<Gtk::Button> display_btn;
    Glib::RefPtr<Gtk::Builder> ui;
public:
    HelloWindow()
    : ui{Gtk::Builder::create_from_file("simple.glade")} {
        if(ui) {
            ui->get_widget<Gtk::Box>("cont", cont);
            display_label = Glib::RefPtr<Gtk::Label>::cast_dynamic(
                ui->get_object("display_label")
            );
            display_btn = Glib::RefPtr<Gtk::Button>::cast_dynamic(
                ui->get_object("display_button")
            );
            if(cont && display_label && display_btn) {
                display_btn->signal_clicked().connect(
                [this]() {
                    display_label->set_text("Hello World");
                });
                add(*cont);
            }
        }
        set_title("Simple Gtk::Builder demo");
        set_default_size(400, 400);
        show_all();
    }
};

int main(int argc, char *argv[]) {
    auto app = Gtk::Application::create(
        argc, argv, 
        "org.gtkmm.example.HelloApp"
    );
    HelloWindow hw;
    return app->run(hw);
}

Usando Gio::Resource #

Carregar UIs diretamente de arquivos .glade é rápido e fácil. Mas quando o aplicativo é empacotado, as descrições da interface do usuário, ícones e outras imagens podem ser reunidas em pacotes de recursos. Primeiro, uma descrição do recurso precisa ser criada como arquivo XML.

recursos.xml

<gresources>
    <gresource prefix="/unique/prefix/">
        <file>icon.png</file>
        <!-- text files such as XML can be compressed to save memory -->
        <file compressed="true">ui.glade</file>
    </gresource>
</gresources>

Em seguida, crie um arquivo .gresource separado ou um arquivo .c contendo recursos como dados de string a serem vinculados como parte do aplicativo.

# generates separate resource file
glib-compile-resources --target=ui.gresource resources.xml
# generates .c file
glib-compile-resources --generate-source resources.xml

Em seguida, a partir do código do aplicativo, carregue o pacote de recursos

// from separate file
auto resource_bundle = Gio::Resource::create_from_file("ui.gresource");
// from stream of bytes in .c file
auto resource_bundle = Glib:wrap(draw_resource_get_resource());
resource_bundle.register_global();

Do pacote de recursos, carregue os elementos da interface do usuário

auto ui = Gtk::Builder::create_from_resource("/unique/prefix/ui.glade");
