  • Configure o Gitlab CI para permitir a clonagem de outros repositórios privados

O GitLab CI executa trabalhos de compilação com base em um check-in .gitlab-ci.yml. Os trabalhos são executados em um servidor remoto em seu próprio contêiner docker.

O próprio servidor CI é configurado com um config.toml.

  • Uma compilação falhará se alguma linha em um trabalho retornar um código de saída != 0.

Instalação do corredor #

Debian, Ubuntu e CentOS #

  1. Adicione o repositório oficial

Debian/Ubuntu

curl -L https://packages.gitlab.com/install/repositories/runner/gitlab-ci-multi-runner/script.deb.sh | sudo bash

CentOS

curl -L https://packages.gitlab.com/install/repositories/runner/gitlab-ci-multi-runner/script.rpm.sh | sudo bash
  1. Instale o pacote gitlab-ci-multi-runner

Debian/Ubuntu

sudo apt-get install gitlab-ci-multi-runner

CentOS

sudo yum install gitlab-ci-multi-runner

  1. Registre o corredor

    sudo gitlab-ci-multi-runner register

  • Digite o URL do seu GitLab CI. Deve ficar assim http://example.com/ci

  • Digite o token de registro. Se este é um runner específico do projeto, você pode encontrar o token em Configurações do projeto -> Corredores. Se for um corredor compartilhado, vá para Área de administração -> Corredores e encontre o token de registro lá.

  • Agora dê ao seu corredor um nome descritivo.

  • Selecione o executor que você deseja usar. Os executores válidos são: shell (estes podem ser configurados posteriormente para usar sh ou bash),docker,docker-ssh,ssh,parallels,virtualbox,docker+machine ou docker -ssh+máquina. Para informações mais detalhadas sobre executores, consulte a documentação oficial.

Janelas #

  1. Baixe o binário do runner e coloque-o em algum lugar apropriado em seu sistema.

  2. Abra um prompt de comando como Administrador

  3. Registre o corredor

    register

  • Digite o URL do seu GitLab CI. Deve ficar assim http://example.com/ci

  • Digite o token de registro. Se este é um runner específico do projeto, você pode encontrar o token em Configurações do projeto -> Corredores. Se for um corredor compartilhado, vá para Área de administração -> Corredores e encontre o token de registro lá.

  • Agora dê ao seu corredor um nome descritivo.

  • Selecione o executor que você deseja usar. Os executores válidos são: shell(Pode ser configurado posteriormente para usar cmd ou powershell),ssh,parallels ou virtualbox. Para informações mais detalhadas sobre executores, consulte a documentação oficial.

  1. (Opcional) Registrar o executor como serviço

    install –user –password

  2. Inicie o corredor

    start

Configuração do corredor #

O local de configuração do seu runner é:

Debian/Ubuntu/CentOS

/etc/gitlab-runner/config.toml se executado como root

~/.gitlab-runner/config.toml se executado como não root

janelas

config.toml onde seu binário está localizado

Um config.toml mínimo pode ser assim:

concurrent = 1
[[runners]]
  name = "ExampleRunner"
  url = "https://example.com/ci"
  token = "f3058595ca4b2d217726466b1feed9"
  executor = "shell"
  shell = "bash"

Para configuração avançada, verifique a documentação oficial.

Configure o Gitlab CI para permitir a clonagem de outros repositórios privados #

Alguns projetos como o GoLang podem precisar clonar outros repositórios GitLab dependentes durante a compilação. Para fazer isso funcionar, você pode adicionar uma Deploy Key aos repositórios dependentes e colocar a chave privada (sem senha) no repositório de origem.

Crie e faça check-in de uma chave SSH dentro do repositório Git que dependa de algum outro repositório durante o tempo de compilação:

ssh-keygen -t rsa -b 4096 -C "My CI Deploykey"

# In the following promt name the key "deploykey" and leave the passphrase empty
Generating public/private rsa key pair.
Enter file in which to save the key (/home/user/.ssh/id_rsa): deploykey
Enter passphrase (empty for no passphrase):
Enter same passphrase again:
Your identification has been saved in deploykey.
Your public key has been saved in deploykey.pub.

# check-in both files

Use o deploykey.pub para configurar uma deploykey no repositório dependente. Você pode encontrar uma página Deploykey nas configurações do projeto GitLab.

Agora adicione o seguinte a você .gitlab-ci.yml

before_script:
  # Git and SSH setup to clone private repos
  # Needs the deploykey file to be installed in all dependent repositories
  - git config --global url."git@gitlab.com:".insteadOf "https://gitlab.com/"
  # Add gitlab to known_hosts
  - mkdir -p ~/.ssh && chmod 700 ~/.ssh
  - ssh-keyscan -H gitlab.com >> ~/.ssh/known_hosts
  # Start the ssh agent and add the deploykey
  - chmod 400 deploykey
  - eval $(ssh-agent -s)
  - ssh-add deploykey

Agora qualquer chamada para git clone dentro de sua compilação deve funcionar. Mesmo que seja através de outras ferramentas como go get, govendor sync, ou o que você estiver usando.
