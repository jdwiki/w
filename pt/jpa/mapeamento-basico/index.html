Parâmetros #

Anotação Detalhes
@Id Marca campo/coluna como a chave da entidade
@Basic Marca o campo solicitado para mapeado como um tipo basic. Isso é aplicável a tipos primitivos e seus wrappers, String, Date e Calendar. A anotação é realmente opcional se nenhum parâmetro for fornecido, mas um bom estilo ditaria para tornar suas intenções explícitas.
@Transient Campos marcados como transientes não são considerados para persistência, assim como a palavra-chave transient para serialização.

Sempre precisa haver um construtor padrão, ou seja, o sem parâmetros. No exemplo básico, não havia construtor especificado, então Java adicionou um; mas se você adicionar um construtor com argumentos, certifique-se de adicionar também o construtor sem parâmetros.

Uma entidade muito simples #

@Entity
class Note {
    @Id
    Integer id;
 
    @Basic
    String note;

    @Basic
    int count;
}

Getters, setters etc. são omitidos por questões de brevidade, mas não são necessários para JPA de qualquer maneira.

Esta classe Java seria mapeada para a seguinte tabela (dependendo do seu banco de dados, aqui dado em um possível mapeamento Postgres):

CREATE TABLE Note (
  id integer NOT NULL,
  note text,
  count integer NOT NULL
)

Os provedores JPA podem ser usados ​​para gerar o DDL e provavelmente produzirão DDL diferente do mostrado aqui, mas desde que os tipos sejam compatíveis, isso não causará problemas em tempo de execução. É melhor não confiar na geração automática de DDL.

Omitindo campo do mapeamento #

@Entity
class Note {
    @Id
    Integer id;
 
    @Basic
    String note;

    @Transient
    String parsedNote;

    String readParsedNote() {
        if (parsedNote == null) { /* initialize from note */ }
        return parsedNote;
    }
}

Se sua classe precisar de campos que não devem ser gravados no banco de dados, marque-os como @Transient. Após a leitura do banco de dados, o campo será null.

Mapeamento de hora e data #

Hora e data vêm em vários tipos diferentes em Java: Os agora históricos Date e Calendar, e os mais recentes LocalDate e LocalDateTime. E Timestamp, Instant, ZonedLocalDateTime e os tipos Joda-time. No lado do banco de dados, temos time, date e timestamp (time e date), possivelmente com ou sem fuso horário.

Data e hora antes do Java 8 #

O mapeamento default para os tipos pré-Java-8 java.util.Date, java.util.Calendar e java.sql.Timestamp é timestamp em SQL; para java.sql.Date é date.

@Entity
class Times {
    @Id
    private Integer id;

    @Basic
    private Timestamp timestamp;

    @Basic
    private java.sql.Date sqldate;

    @Basic
    private java.util.Date utildate;

    @Basic
    private Calendar calendar;
}

Isso será mapeado perfeitamente para a tabela a seguir:

CREATE TABLE times (
    id integer not null,
    timestamp timestamp,
    sqldate date,
    utildate timestamp,
    calendar timestamp
)

Essa pode não ser a intenção. Por exemplo, muitas vezes um Java Date ou Calendar é usado para representar apenas a data (para data de nascimento). Para alterar o mapeamento padrão, ou apenas para torná-lo explícito, você pode usar a anotação @Temporal.

@Entity
class Times {
    @Id
    private Integer id;

    @Temporal(TemporalType.TIME)
    private Date date;

    @Temporal(TemporalType.DATE)
    private Calendar calendar;
}

A tabela SQL equivalente é:

CREATE TABLE times (
    id integer not null,
    date time,
    calendar date
)

Nota 1: O tipo especificado com @Temporal influencia a geração de DDL; mas você também pode ter uma coluna do tipo date map para Date apenas com a anotação @Basic.

Nota 2: Calendar não pode persistir somente time.

Data e hora com Java 8 #

O JPA 2.1 não define suporte para tipos java.time fornecidos no Java 8. A maioria das implementações do JPA 2.1 oferece suporte para esses tipos, embora sejam extensões de fornecedores estritamente falando.

Para o DataNucleus, esses tipos funcionam imediatamente e oferecem uma ampla gama de possibilidades de mapeamento, acopladas à anotação @Temporal.

Para o Hibernate, se estiver usando o Hibernate 5.2+, eles devem funcionar imediatamente, apenas usando a anotação @Basic. Se estiver usando o Hibernate 5.0-5.1 você precisa adicionar a dependência org.hibernate:hibernate-java8. Os mapeamentos fornecidos são

  • LocalDate para date
  • Instant, LocalDateTime e ZonedDateTime para timestamp

Uma alternativa neutra de fornecedor também seria definir um JPA 2.1 AttributeConverter para qualquer tipo Java 8 java.time que deve ser persistido.

Entidade com ID gerenciado por sequência #

Aqui temos uma classe e queremos que o campo de identidade (userUid) tenha seu valor gerado através de uma SEQUENCE no banco de dados. Essa SEQUÊNCIA é chamada de USER_UID_SEQ e pode ser criada por um DBA ou pode ser criada pelo provedor JPA.

@Entity
@Table(name="USER")
public class User implements Serializable {
    private static final long serialVersionUID = 1L;

    @Id
    @SequenceGenerator(name="USER_UID_GENERATOR", sequenceName="USER_UID_SEQ")
    @GeneratedValue(strategy=GenerationType.SEQUENCE, generator="USER_UID_GENERATOR")
    private Long userUid;

    @Basic
    private String userName;
}
