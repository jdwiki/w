Crie seu próprio plugin Gradle Scala #

Depois de passar pelo exemplo de Configuração Básica, você pode acabar repetindo a maior parte dele em cada projeto Scala Gradle. Cheira a código clichê…

E se, ao invés de aplicar o [plugin Scala] oferecido pelo Gradle, você pudesse aplicar o seu próprio plug-in Scala, que seria responsável por lidar com toda a sua lógica de compilação comum, estendendo, ao mesmo tempo, o plug-in já existente.

Este exemplo vai transformar a lógica de construção anterior em um plugin Gradle reutilizável.

Felizmente, no Gradle, você pode facilmente escrever plugins personalizados com a ajuda da API do Gradle, conforme descrito na [documentação]. Como linguagem de implementação, você pode usar o próprio Scala ou até mesmo Java. No entanto, a maioria dos exemplos que você pode encontrar ao longo dos documentos são escritos em Groovy. Se você precisar de mais exemplos de código ou quiser entender o que está por trás do plugin Scala, por exemplo, você pode verificar o gradle github repo.

Escrevendo o plug-in #

Requisitos #

O plug-in personalizado adicionará a seguinte funcionalidade quando aplicado a um projeto:

  • um objeto de propriedade scalaVersion, que terá duas propriedades padrão substituíveis
    • major = “2.12”
    • minor = “0”
  • uma função withScalaVersion, que se aplica a um nome de dependência, adicionará a versão scala major para garantir a compatibilidade binária (o operador sbt %% pode tocar uma campainha, caso contrário, vá [aqui] antes de continuar)
  • uma tarefa createDirs para criar a árvore de diretórios necessária, exatamente como no exemplo anterior

Diretrizes de implementação #

  1. crie um novo projeto gradle e adicione o seguinte a build.gradle
apply plugin: 'scala'
apply plugin: 'maven'

repositories {
    mavenLocal()
    mavenCentral()
}

dependencies {
    compile gradleApi()
    compile "org.scala-lang:scala-library:2.12.0"
}

Notas:

  • a implementação do plugin é escrita em Scala, portanto, precisamos do Plugin Gradle Scala
  • para usar o plugin de outros projetos, o plugin Gradle Maven é usado; isso adiciona a tarefa install usada para salvar o jar do projeto no repositório local do Maven
  • compile gradleApi() adiciona o gradle-api-<gradle_version>.jar ao caminho de classe
  1. crie uma nova classe Scala para a implementação do plugin
package com.btesila.gradle.plugins 
 
import org.gradle.api.{Plugin, Project}  
 
class ScalaCustomPlugin extends Plugin[Project] {
    override def apply(project: Project): Unit = {
        project.getPlugins.apply("scala")
    }
}

Notas:

  • para implementar um Plugin, basta estender o traço Plugin do tipo Project e substituir o método apply
  • dentro do método apply, você tem acesso à instância Project à qual o plug-in é aplicado e pode usá-lo para adicionar lógica de compilação a ele
  • este plugin não faz nada além de aplicar o plugin Gradle Scala já existente
  1. adicione a propriedade do objeto scalaVersion

Primeiramente, criamos uma classe ScalaVersion, que conterá as duas propriedades da versão

class ScalaVersion {
  var major: String = "2.12"
  var minor: String = "0"
}

Uma coisa legal sobre os plugins Gradle é o fato de que você sempre pode adicionar ou substituir propriedades específicas. Um plugin recebe este tipo de entrada do usuário através do ExtensionContainer anexado a uma instância gradle Project. Para mais detalhes, confira [este] para fora. Ao adicionar o seguinte ao método apply, estamos basicamente fazendo isso:

  • se não houver uma propriedade scalaVersion definida no projeto, adicionamos uma com os valores padrão
  • caso contrário, obtemos o existente como instância de ScalaVersion, para usá-lo ainda mais
var scalaVersion = new ScalaVersion
if (!project.getExtensions.getExtraProperties.has("scalaVersion"))
   project.getExtensions.getExtraProperties.set("scalaVersion", scalaVersion)
senão
   scalaVersion = project.getExtensions.getExtraProperties.get("scalaVersion").asInstanceOf[ScalaVersion]

This is equivalent to writing the following to the build file of the project that applies the plugin:

extensão {
    scalaVersion.major = "2.12"
    scalaVersion.minor = "0"

}
  1. add the scala-lang library to the project dependencies, using the scalaVersion
project.getDependencies.add("compile", s"org.scala-lang:scala-library:${scalaVersion.major}.${scalaVersion.minor}")

This is equivalent to writing the following to the build file of the project that applies the plugin:

compilar "org.scala-lang:scala-library:2.12.0"
  1. add the withScalaVersion function
val withScalaVersion = (lib: String) => {
    val libComp = lib.split(":")
    libComp.update(1, s"${libComp(1)}_${scalaVersion.major}")
    libComp.mkString(":")
}
project.getExtensions.getExtraProperties.set("withScalaVersion", withScalaVersion)
  1. finally, create the createDirs task and add it to the project
    Implement a Gradle task by extending DefaultTask:
class CreateDirs estende DefaultTask {
@TaskAction
def createDirs(): Unidade = {
    val sourceSetContainer = this.getProject.getConvention.getPlugin(classOf[JavaPluginConvention]).getSourceSets

    sourceSetContainer forEach { sourceSet =>
      sourceSet.getAllSource.getSrcDirs.forEach(file => if (!file.getName.contains("java")) file.mkdirs())
    }
  }
}

Note: the SourceSetContainer has information about all source directories present in the project. What the Gradle Scala Plugin does, is to add the extra source sets to the Java ones, as you can see in theplugin docs.

Add the createDir task to the project by appending this to the apply method:

project.getTasks.create("createDirs", classOf[CreateDirs])

In the end, your ScalaCustomPlugin class should look like this:

class ScalaCustomPlugin estende Plugin[Projeto] {
override def apply(project: Project): Unit = {
    project.getPlugins.apply("scala")

    var scalaVersion = new ScalaVersion
    if (!project.getExtensions.getExtraProperties.has("scalaVersion"))
      project.getExtensions.getExtraProperties.set("scalaVersion", scalaVersion)
    else
      scalaVersion = project.getExtensions.getExtraProperties.get("scalaVersion").asInstanceOf[ScalaVersion]

    project.getDependencies.add("compile", s"org.scala-lang:scala-library:${scalaVersion.major}.${scalaVersion.minor}")

    val withScalaVersion = (lib: String) => {
      val libComp = lib.split(":")
      libComp.update(1, s"${libComp(1)}_${scalaVersion.major}")
      libComp.mkString(":")
    }
    project.getExtensions.getExtraProperties.set("withScalaVersion", withScalaVersion)

    project.getTasks.create("createDirs", classOf[CreateDirs])
  }
}

Installing the plugin project to the local Maven repository #

This is done really easy by running gradle install
You can check the installation by going to local repository directory, usually found at ~/.m2/repository

How does Gradle find our new plugin? #

Each Gradle plugin has an id which is used in the apply statement. For instance, by writing the following to the build file, it translates to a trigger to Gradle to find and apply the plugin with id scala.

aplicar plugin: 'scala'

In the same way, we would like to apply our new plugin in the following way,

aplicar plugin: "com.btesila.scala.plugin"

meaning that our plugin will have the com.btesila.scala.plugin id.

In order to set this id, add the following file:
src/main/resources/META-INF/gradle-plugin/com.btesil.scala.plugin.properties

classe de implementação=com.btesila.gradle.plugins.ScalaCustomPlugin

Afterwards, run again gradle install.

Using the plugin #

  1. create a new empty Gradle project and add the following to the build file
script de construção {
    repositories {
        mavenLocal()
        mavenCentral()
    }

    dependencies {
        //modify this path to match the installed plugin project in your local repository
        classpath 'com.btesila:working-with-gradle:1.0-SNAPSHOT'
    }
}

repositórios {
    mavenLocal()
    mavenCentral()
}


aplicar plugin: "com.btesila.scala.plugin"
  1. run gradle createDirs - you should now have all the source directories generated
  2. override the scala version by adding this to the build file:
extensão {
    scalaVersion.major = "2.11"
    scalaVersion.minor = "8"

}
println(project.ext.scalaVersion.major)
println(project.ext.scalaVersion.minor)
  1. add a dependency library that is binary compatible with the Scala version
dependências {
    compile withScalaVersion("com.typesafe.scala-logging:scala-logging:3.5.0")
}

That’s it! You can now use this plugin across all your projects without repeating the same old boilerplate.

Basic Setup #

  1. Create a file named SCALA_PROJECT/build.gradle with these contents:

    group 'scala_gradle'
version '1.0-SNAPSHOT'

apply plugin: 'scala'

repositories {
    jcenter()
    mavenCentral()
    maven {
        url "https://repo.typesafe.com/typesafe/maven-releases"
    }
}

dependencies {
    compile group: 'org.scala-lang', name: 'scala-library', version: '2.10.6'
}


task  "create-dirs" << {
    sourceSets*.scala.srcDirs*.each { it.mkdirs() }
    sourceSets*.resources.srcDirs*.each { it.mkdirs() }
}

  2. Run gradle tasks to see available tasks.

  3. Run gradle create-dirs to create a src/scala, src/resources directory.

  4. Run gradle build to build the project and download dependencies.
