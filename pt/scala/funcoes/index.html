Scala tem funções de primeira classe.

Diferença entre funções e métodos: #

Uma função não é um método em Scala: funções são um valor e podem ser atribuídas como tal. Métodos (criados usando def), por outro lado, devem pertencer a uma classe, traço ou objeto.

  • As funções são compiladas para uma classe que estende uma característica (como Function1) em tempo de compilação e são instanciadas em um valor em tempo de execução. Os métodos, por outro lado, são membros de sua classe, traço ou objeto, e não existem fora disso.
  • Um método pode ser convertido em uma função, mas uma função não pode ser convertida em um método.
  • Métodos podem ter parametrização de tipo, enquanto funções não.
  • Métodos podem ter valores padrão de parâmetros, enquanto funções não podem.

Funções anônimas #

Funções anônimas são funções definidas, mas não atribuídas a um nome.

A seguir está uma função anônima que recebe dois inteiros e retorna a soma.

(x: Int, y: Int) => x + y

A expressão resultante pode ser atribuída a um val:

val sum = (x: Int, y: Int) => x + y

Funções anônimas são usadas principalmente como argumentos para outras funções. Por exemplo, a função map em uma coleção espera outra função como argumento:

// Returns Seq("FOO", "BAR", "QUX")
Seq("Foo", "Bar", "Qux").map((x: String) => x.toUpperCase)

Os tipos dos argumentos da função anônima podem ser omitidos: os tipos são inferidos automaticamente:

Seq("Foo", "Bar", "Qux").map((x) => x.toUpperCase)

Se houver apenas um argumento, os parênteses em torno desse argumento podem ser omitidos:

Seq("Foo", "Bar", "Qux").map(x => x.toUpperCase)

Abreviação de sublinhados #

Existe uma sintaxe ainda mais curta que não requer nomes para os argumentos. O trecho acima pode ser escrito:

Seq("Foo", "Bar", "Qux").map(_.toUpperCase)

_ representa os argumentos da função anônima posicionalmente. Com uma função anônima que possui vários parâmetros, cada ocorrência de _ fará referência a um argumento diferente. Por exemplo, as duas expressões a seguir são equivalentes:

// Returns "FooBarQux" in both cases
Seq("Foo", "Bar", "Qux").reduce((s1, s2) => s1 + s2)
Seq("Foo", "Bar", "Qux").reduce(_ + _)

Ao usar esta abreviação, qualquer argumento representado pelo posicional _ só pode ser referenciado uma única vez e na mesma ordem.

Funções anônimas sem parâmetros #

Para criar um valor para uma função anônima que não aceita parâmetros, deixe a lista de parâmetros em branco:

val sayHello = () => println("hello")

Composição #

A composição de funções permite que duas funções operem e sejam vistas como uma única função. Expressa em termos matemáticos, dada uma função f(x) e uma função g(x), a função h(x) = f(g(x)).

Quando uma função é compilada, ela é compilada para um tipo relacionado a Function1. Scala fornece dois métodos na implementação Function1 relacionados à composição: andThen e compose. O método compose se encaixa com a definição matemática acima da seguinte forma:

val f: B => C = ...
val g: A => B = ...

val h: A => C = f compose g

O andThen (pense em h(x) = g(f(x))) tem uma sensação mais ‘DSL-like’:

val f: A => B = ...
val g: B => C = ...

val h: A => C = f andThen g

Uma nova função anônima é alocada com que é fechada sobre f e g. Esta função está vinculada à nova função h em ambos os casos.

def andThen(g: B => C): A => C = new (A => C){
  def apply(x: A) = g(self(x))
}

Se f ou g funcionar através de um efeito colateral, então chamar h fará com que todos os efeitos colaterais de f e g aconteçam na ordem. O mesmo vale para qualquer mudança de estado mutável.

Relação com PartialFunctions #

trait PartialFunction[-A, +B] extends (A => B)

Cada argumento único PartialFunction também é um Function1. Isso é contra-intuitivo em um sentido matemático formal, mas se encaixa melhor no design orientado a objetos. Por esta razão, Function1 não precisa fornecer um método true isDefinedAt constante.

Para definir uma função parcial (que também é uma função), use a seguinte sintaxe:

{ case i: Int => i + 1 } // or equivalently { case i: Int ⇒ i + 1 }

Para mais detalhes, dê uma olhada em PartialFunctions.
