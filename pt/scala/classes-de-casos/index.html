Sintaxe #

  • case class Foo() // Classes case sem parâmetros devem ter uma lista vazia
  • case class Foo(a1: A1, …, aN: AN) // Cria uma classe case com os campos a1 … aN
  • case object Bar // Cria uma classe case singleton

Artefatos de código gerados #

O modificador case faz com que o compilador Scala gere automaticamente um código clichê comum para a classe. Implementar este código manualmente é tedioso e uma fonte de erros. A seguinte definição de classe de caso:

case class Person(name: String, age: Int)

… terá o seguinte código gerado automaticamente:

class Person(val name: String, val age: Int)
  extends Product with Serializable
{
  def copy(name: String = this.name, age: Int = this.age): Person =
    new Person(name, age)

  def productArity: Int = 2

  def productElement(i: Int): Any = i match {
    case 0 => name
    case 1 => age
    case _ => throw new IndexOutOfBoundsException(i.toString)
  }

  def productIterator: Iterator[Any] =
    scala.runtime.ScalaRunTime.typedProductIterator(this)

  def productPrefix: String = "Person"

  def canEqual(obj: Any): Boolean = obj.isInstanceOf[Person]

  override def hashCode(): Int = scala.runtime.ScalaRunTime._hashCode(this)

  override def equals(obj: Any): Boolean = this.eq(obj) || obj match {
    case that: Person => this.name == that.name && this.age == that.age
    case _ => false
  }

  override def toString: String =
    scala.runtime.ScalaRunTime._toString(this)
}

O modificador case também gera um objeto complementar:

object Person extends AbstractFunction2[String, Int, Person] with Serializable {
  def apply(name: String, age: Int): Person = new Person(name, age)

  def unapply(p: Person): Option[(String, Int)] =
    if(p == null) None else Some((p.name, p.age))
}

Quando aplicado a um objeto, o modificador case tem efeitos semelhantes (embora menos dramáticos). Aqui os principais ganhos são uma implementação toString e um valor hashCode que é consistente em todos os processos. Observe que os objetos case (corretamente) usam igualdade de referência:

object Foo extends Product with Serializable {
  def productArity: Int = 0

  def productIterator: Iterator[Any] =
    scala.runtime.ScalaRunTime.typedProductIterator(this)

  def productElement(i: Int): Any =
    throw new IndexOutOfBoundsException(i.toString)

  def productPrefix: String = "Foo"

  def canEqual(obj: Any): Boolean = obj.isInstanceOf[this.type]

  override def hashCode(): Int = 70822 // "Foo".hashCode()

  override def toString: String = "Foo"
}

Ainda é possível implementar manualmente métodos que de outra forma seriam fornecidos pelo modificador case tanto na própria classe quanto em seu objeto complementar.

Noções básicas de classe de caso #

Em comparação com as classes regulares – a notação de classes de caso oferece vários benefícios:

  • Todos os argumentos do construtor são públicos e podem ser acessados ​​em objetos inicializados (normalmente este não é o caso, como demonstrado aqui):

    case class Dog1(age: Int)
val x = Dog1(18)
println(x.age) // 18 (success!)

class Dog2(age: Int)
val x = new Dog2(18)
println(x.age) // Error: "value age is not a member of Dog2"

  • Ele fornece uma implementação para os seguintes métodos: toString, equals, hashCode (baseado em propriedades), copy, apply e unapply:

    case class Dog(age: Int)
val d1 = Dog(10)
val d2 = d1.copy(age = 15)

  • Ele fornece um mecanismo conveniente para correspondência de padrões:

    sealed trait Animal // `sealed` modifier allows inheritance within current build-unit only
case class Dog(age: Int) extends Animal
case class Cat(owner: String) extends Animal
val x: Animal = Dog(18)
x match {
    case Dog(x) => println(s"It's a $x years old dog.")
    case Cat(x) => println(s"This cat belongs to $x.")
}

Igualdade de Classe de Caso #

Um recurso fornecido gratuitamente pelas classes case é um método equals gerado automaticamente que verifica a igualdade de valor de todos os campos de membros individuais em vez de apenas verificar a igualdade de referência dos objetos.

Com aulas normais:

class Foo(val i: Int)
val a = new Foo(3)
val b = new Foo(3)
println(a == b)// "false" because they are different objects

Com classes de casos:

case class Foo(i: Int)
val a = Foo(3)
val b = Foo(3)
println(a == b)// "true" because their members have the same value

Classes de caso e imutabilidade #

O compilador Scala prefixa cada argumento na lista de parâmetros por padrão com val. Isso significa que, por padrão, as classes case são imutáveis. Cada parâmetro recebe um método de acesso, mas não há métodos de mutação. Por exemplo:

case class Foo(i: Int)

val fooInstance = Foo(1)
val j = fooInstance.i       // get
fooInstance.i = 2           // compile-time exception (mutation: reassignment to val)

Declarar um parâmetro em uma classe case como var substitui o comportamento padrão e torna a classe case mutável:

case class Bar(var i: Int)

val barInstance = Bar(1)
val j = barInstance.i       // get
barInstance.i = 2           // set

Outra instância em que uma classe case é ‘mutável’ é quando o valor na classe case é mutável:

import scala.collection._

case class Bar(m: mutable.Map[Int, Int])

val barInstance = Bar(mutable.Map(1 -> 2))
barInstance.m.update(1, 3)                  // mutate m
barInstance                                 // Bar(Map(1 -> 3)

Observe que a ‘mutação’ que está ocorrendo aqui está no mapa para o qual m aponta, não para m em si. Assim, se algum outro objeto tivesse m como membro, ele também veria a mudança. Observe como no exemplo a seguir, alterar instanceA também altera instanceB:

import scala.collection.mutable

case class Bar(m: mutable.Map[Int, Int])

val m = mutable.Map(1 ->2)
val barInstanceA = Bar(m)
val barInstanceB = Bar(m)
barInstanceA.m.update(1,3)
barInstanceA  // Bar = Bar(Map(1 -> 3))
barInstanceB  // Bar = Bar(Map(1 -> 3))
m  // scala.collection.mutable.Map[Int,Int] = Map(1 -> 3)

Criar uma cópia de um objeto com determinadas alterações #

As classes Case fornecem um método copy que cria um novo objeto que compartilha os mesmos campos que o antigo, com algumas alterações.

Podemos usar esse recurso para criar um novo objeto a partir de um anterior que tenha algumas das mesmas características. Esta classe case simples para demonstrar esse recurso:

case class Person(firstName: String, lastName: String, grade: String, subject: String)
val putu = Person("Putu", "Kevin", "A1", "Math")
val mark = putu.copy(firstName = "Ketut", lastName = "Mark")
// mark: People = People(Ketut,Mark,A1,Math)

Neste exemplo podemos ver que os dois objetos compartilham características semelhantes (grade = A1, subject = Math), exceto onde eles foram especificados na cópia (firstName e lastName).

Classes de caso de elemento único para segurança de tipo #

Para alcançar a segurança de tipos, às vezes queremos evitar o uso de tipos primitivos em nosso domínio. Por exemplo, imagine uma Pessoa com um nome. Normalmente, codificamos o name como uma String. No entanto, não seria difícil misturar uma String representando o name de uma Person com uma String representando uma mensagem de erro:

def logError(message: ErrorMessage): Unit = ???
case class Person(name: String)
val maybeName: Either[String, String] = ??? // Left is error, Right is name
maybeName.foreach(logError) // But that won't stop me from logging the name as an error!

Para evitar tais armadilhas, você pode codificar os dados assim:

case class PersonName(value: String)
case class ErrorMessage(value: String)
case class Person(name: PersonName)

e agora nosso código não compilará se misturarmos PersonName com ErrorMessage, ou mesmo uma String comum.

val maybeName: Either[ErrorMessage, PersonName] = ???
maybeName.foreach(reportError) // ERROR: tried to pass PersonName; ErrorMessage expected
maybeName.swap.foreach(reportError) // OK

Mas isso incorre em uma pequena sobrecarga de tempo de execução, pois agora temos que box/unbox Strings para/de seus contêineres PersonName. Para evitar isso, pode-se criar as classes de valor PersonName e ErrorMessage:

case class PersonName(val value: String) extends AnyVal
case class ErrorMessage(val value: String) extends AnyVal
