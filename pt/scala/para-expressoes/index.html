Sintaxe #

  • para {clauses} corpo
  • para {clauses} corpo de rendimento
  • para (cláusulas) corpo
  • para (cláusulas) corpo de rendimento

Parâmetros #

Parâmetro Detalhes
para Palavra-chave necessária para usar um loop for/compreensão
cláusulas A iteração e os filtros sobre os quais o for funciona.
rendimento Use isso se quiser criar ou ‘render’ uma coleção. Usar yield fará com que o tipo de retorno do for seja uma coleção em vez de Unit.
corpo O corpo da expressão for, executado em cada iteração.

Monádico para compreensões #

Se você tiver vários objetos do tipo monadic, podemos obter combinações dos valores usando um ‘para compreensão’:

for {
   x <- Option(1)
   y <- Option("b")
   z <- List(3, 4)
} {
    // Now we can use the x, y, z variables
    println(x, y, z)
    x  // the last expression is *not* the output of the block in this case!
}

// This prints
// (1, "b", 3)
// (1, "b", 4)

O tipo de retorno deste bloco é Unit.

Se os objetos forem do mesmo tipo monádico M (por exemplo, Option), então usar yield retornará um objeto do tipo M em vez de Unit.

val a = for {
   x <- Option(1)
   y <- Option("b")
} yield {
    // Now we can use the x, y variables
    println(x, y)
    // whatever is at the end of the block is the output
    (7 * x, y)
}

// This prints:
// (1, "b")
// The val `a` is set:
// a: Option[(Int, String)] = Some((7,b))

Observe que a palavra-chave yield não pode ser usada no exemplo original, onde há uma mistura de tipos monádicos (Option e List). Tentar fazer isso resultará em um erro de incompatibilidade de tipo em tempo de compilação.

Aninhado para Loop #

Isso mostra como você pode iterar em várias variáveis:

for {
  x <- 1 to 2
  y <- 'a' to 'd'
} println("(" + x + "," + y + ")")

(Observe que to aqui é um método de operador infixo que retorna um intervalo inclusivo. Veja a definição aqui.)

Isso cria a saída:

(1,a)
(1,b)
(1,c)
(1,d)
(2,a)
(2,b)
(2,c)
(2,d)

Observe que esta é uma expressão equivalente, usando parênteses em vez de colchetes:

for (
  x <- 1 to 2
  y <- 'a' to 'd'
) println("(" + x + "," + y + ")")

Para obter todas as combinações em um único vetor, podemos yield o resultado e defini-lo como um val:

val a = for {
  x <- 1 to 2
  y <- 'a' to 'd'
} yield "(%s,%s)".format(x, y)
// a: scala.collection.immutable.IndexedSeq[String] = Vector((1,a), (1,b), (1,c), (1,d), (2,a), (2,b), (2,c), (2,d))

Iterar por meio de coleções usando um loop for #

Isso demonstra como imprimir cada elemento de um mapa

val map = Map(1 -> "a", 2 -> "b")
for (number <- map) println(number) // prints (1,a), (2,b)
for ((key, value) <- map) println(value) // prints a, b

Isso demonstra como imprimir cada elemento de uma lista

val list = List(1,2,3)
for(number <- list) println(number) // prints 1, 2, 3

Desaçucar para compreensões #

As compreensões for em Scala são apenas açúcar sintático. Essas compreensões são implementadas usando os métodos withFilter, foreach, flatMap e map de seus tipos de assunto. Por esta razão, apenas os tipos que possuem esses métodos definidos podem ser utilizados em uma compreensão for.

Uma compreensão for da seguinte forma, com padrões pN, geradores gN e condições cN:

for(p0 <- x0 if g0; p1 <- g1 if c1) { ??? }

… irá remover o açúcar para chamadas aninhadas usando withFilter e foreach:

g0.withFilter({ case p0 => c0  case _ => false }).foreach({
  case p0 => g1.withFilter({ case p1 => c1  case _ => false }).foreach({
    case p1 => ???
  })
})

Considerando que uma expressão for/yield da seguinte forma:

for(p0 <- g0 if c0; p1 <- g1 if c1) yield ???

… irá remover o açúcar para chamadas aninhadas usando withFilter e flatMap ou map:

g0.withFilter({ case p0 => c0  case _ => false }).flatMap({
  case p0 => g1.withFilter({ case p1 => c1  case _ => false }).map({
    case p1 => ???
  })
})

(Observe que map é usado na compreensão mais interna e flatMap é usado em toda compreensão externa.)

Uma compreensão for pode ser aplicada a qualquer tipo de implementação dos métodos requeridos pela representação sem açúcar. Não há restrições sobre os tipos de retorno desses métodos, desde que sejam compostos.

Básico para Loop #

for (x <- 1 to 10)
  println("Iteration number " + x)

Isso demonstra a iteração de uma variável, x, de 1 a 10 e fazendo algo com esse valor. O tipo de retorno desta compreensão for é Unit.

Básico para compreensão #

Isso demonstra um filtro em um loop for e o uso de yield para criar uma ‘compreensão de sequência’:

for ( x <- 1 to 10 if x % 2 == 0)
  yield x

A saída para isso é:

scala.collection.immutable.IndexedSeq[Int] = Vector(2, 4, 6, 8, 10)

Um para compreensão é útil quando você precisa criar uma nova coleção com base na iteração e seus filtros.
