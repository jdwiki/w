Sintaxe #

  • class Some[+T](valor: T) estende Option[T]

  • objeto Nenhum estende Opção[Nada]

  • Opção[T](valor: T)

    Constructor to create either a Some(value) or None as appropriate for the value provided.

Usando Option em vez de Null #

Em Java (e outras linguagens), usar null é uma maneira comum de indicar que não há valor anexado a uma variável de referência. Em Scala, usar Option é preferível a usar null. Option envolve valores que podem ser null.

None é uma subclasse de Option envolvendo uma referência nula. Some é uma subclasse de Option que envolve uma referência não nula.

Envolver uma referência é fácil:

val nothing = Option(null) // None
val something = Option("Aren't options cool?") // Some("Aren't options cool?")

Este é um código típico ao chamar uma biblioteca Java que pode retornar uma referência nula:

val resource = Option(JavaLib.getResource())
// if null, then resource = None
// else resource = Some(resource)

Se getResource() retornar um valor null, resource será um objeto None. Caso contrário, será um objeto Some(resource). A maneira preferida de lidar com uma Option é usar funções de ordem superior disponíveis no tipo Option. Por exemplo, se você quiser verificar se seu valor não é None (semelhante a verificar se value == null), você usaria a função isDefined:

val resource: Option[Resource] = Option(JavaLib.getResource())
if (resource.isDefined) {  // resource is `Some(_)` type
  val r: Resource = resource.get
  r.connect()
}

Da mesma forma, para verificar uma referência null, você pode fazer isso:

val resource: Option[Resource] = Option(JavaLib.getResource())
if (resource.isEmpty) { // resource is `None` type.
  System.out.println("Resource is empty! Cannot connect.")
}

É preferível que você trate a execução condicional no valor encapsulado de uma Option (sem usar o método ’excepcional’ Option.get) tratando a Option como uma mônada e usando foreach:

val resource: Option[Resource] = Option(JavaLib.getResource())
resource foreach (r => r.connect())
// if r is defined, then r.connect() is run
// if r is empty, then it does nothing

Se uma instância Resource for necessária (versus uma instância Option[Resource]), você ainda poderá usar Option para se proteger contra valores nulos. Aqui o método getOrElse fornece um valor padrão:

lazy val defaultResource = new Resource()
val resource: Resource = Option(JavaLib.getResource()).getOrElse(defaultResource)

O código Java não lidará prontamente com a Option do Scala, então ao passar valores para o código Java é uma boa forma de desempacotar uma Option, passando null ou um padrão sensato quando apropriado:

val resource: Option[Resource] = ???
JavaLib.sendResource(resource.orNull)
JavaLib.sendResource(resource.getOrElse(defaultResource)) //

Fundamentos #

Uma Option é uma estrutura de dados que contém um valor único ou nenhum valor. Uma Option pode ser pensada como coleções de zero ou um elemento.

Option é uma classe abstrata com dois filhos: Some e None.

Some contém um único valor e None não contém nenhum valor.

Option é útil em expressões que usariam null para representar a falta de um valor concreto. Isso protege contra um NullPointerException e permite a composição de muitas expressões que podem não retornar um valor usando combinadores como Map, FlatMap, etc.

Exemplo com Mapa #

val countries = Map(
  "USA" -> "Washington",
  "UK" -> "London",
  "Germany" -> "Berlin",
  "Netherlands" -> "Amsterdam",
  "Japan" -> "Tokyo"
)
    
println(countries.get("USA")) // Some(Washington)
println(countries.get("France")) // None
println(countries.get("USA").get) // Washington
println(countries.get("France").get) // Error: NoSuchElementException
println(countries.get("USA").getOrElse("Nope")) // Washington
println(countries.get("France").getOrElse("Nope")) // Nope

Option[A] é selado e, portanto, não pode ser estendido. Portanto, sua semântica é estável e confiável.

Opções como coleções #

Options têm algumas funções úteis de ordem superior que podem ser facilmente compreendidas visualizando opções como collections com zero ou um item - onde None se comporta como a coleção vazia, e Some(x) se comporta como uma coleção com um item único, x.

val option: Option[String] = ???

option.map(_.trim) // None if option is None, Some(s.trim) if Some(s)
option.foreach(println) // prints the string if it exists, does nothing otherwise
option.forall(_.length > 4) // true if None or if Some(s) and s.length > 4
option.exists(_.length > 4) // true if Some(s) and s.length > 4
option.toList // returns an actual list

Opções para compreensão #

Options tem um método flatMap. Isso significa que eles podem ser usados ​​em um para compreensão. Desta forma podemos levantar funções regulares para trabalhar em Options sem ter que redefini-las.

val firstOption: Option[Int] = Option(1)
val secondOption: Option[Int] = Option(2)

val myResult = for {
  firstValue <- firstOption
  secondValue <- secondOption
} yield firstValue + secondValue
// myResult: Option[Int] = Some(3)

Quando um dos valores for ‘Nenhum’, o resultado final do cálculo também será ‘Nenhum’.

val firstOption: Option[Int] = Option(1)
val secondOption: Option[Int] = None

val myResult = for {
  firstValue <- firstOption
  secondValue <- secondOption
} yield firstValue + secondValue
// myResult: Option[Int] = None

Nota: este padrão se estende de forma mais geral para conceitos chamados Monads. (Mais informações devem estar disponíveis nas páginas relativas a compreensões e Mônadass)

Em geral não é possível misturar diferentes mônadas em uma para compreensão. Mas como Option pode ser facilmente convertido em Iterable, podemos facilmente misturar Options e Iterables chamando o método .toIterable.

val option: Option[Int] = Option(1)
val iterable: Iterable[Int] = Iterable(2, 3, 4, 5)

// does NOT compile since we cannot mix Monads in a for comprehension
// val myResult = for {
//   optionValue <- option
//   iterableValue <- iterable
//} yield optionValue + iterableValue

// It does compile when adding a .toIterable on the option
val myResult = for {
  optionValue <- option.toIterable
  iterableValue <- iterable
} yield optionValue + iterableValue
// myResult: Iterable[Int] = List(2, 3, 4, 5)

Uma pequena nota: se tivéssemos definido nosso para compreensão ao contrário, o para compreensão seria compilado, pois nossa opção seria convertida implicitamente. Por essa razão é útil sempre adicionar este .toIterable (ou função correspondente dependendo de qual coleção você está usando) para consistência.
