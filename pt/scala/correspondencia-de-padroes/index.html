Sintaxe #

  • seletor de correspondência parcialFunction
  • selector match {lista de alternativas de caso) // Esta é a forma mais comum do acima

Parâmetros #

Parâmetro Detalhes
seletor A expressão cujo valor está sendo correspondido ao padrão.
alternativas uma lista de alternativas delimitadas por case.

Correspondência de padrões simples #

Este exemplo mostra como combinar uma entrada com vários valores:

def f(x: Int): String = x match {
  case 1 => "One"
  case 2 => "Two"
  case _ => "Unknown!"
}

f(2)  // "Two"
f(3)  // "Unknown!"

Demonstração ao vivo

Nota: _ é o caso fall through ou default, mas não é obrigatório.

def g(x: Int): String = x match {
  case 1 => "One"
  case 2 => "Two"
}

g(1)  // "One"
g(3)  // throws a MatchError

Para evitar lançar uma exceção, é uma melhor prática de programação funcional lidar com o caso padrão (case _ => <do something>). Observe que a correspondência sobre uma classe de caso pode ajudar o compilador a produzir um aviso se um caso estiver ausente. O mesmo vale para tipos definidos pelo usuário que estendem uma característica selada. Se a correspondência for total, um caso padrão pode não ser necessário

Também é possível fazer a correspondência com valores que não são definidos em linha. Estes devem ser identificadores estáveis, que são obtidos usando um nome em maiúsculas ou incluindo acentos graves.

Com One e two definidos em outro lugar, ou passados ​​como parâmetros de função:

val One: Int = 1
val two: Int = 2

Eles podem ser combinados da seguinte maneira:

def g(x: Int): String = x match {
  case One => "One"
  case `two` => "Two"
}

Ao contrário de outras linguagens de programação como Java, por exemplo, não há falhas. Se um bloco case corresponder a uma entrada, ele será executado e a correspondência será finalizada. Portanto, o caso menos específico deve ser o último bloco de casos.

def f(x: Int): String = x match {
  case _ => "Default"
  case 1 => "One"
}

f(5) // "Default"
f(1) // "Default"

Correspondência de padrões em uma sequência #

Para verificar um número preciso de elementos na coleção

def f(ints: Seq[Int]): String = ints match {
  case Seq() =>
      "The Seq is empty !"
  case Seq(first) =>
      s"The seq has exactly one element : $first"
  case Seq(first, second) =>
      s"The seq has exactly two elements : $first, $second"
  case  s @ Seq(_, _, _) => 
      s"s is a Seq of length three and looks like ${s}"  // Note individual elements are not bound to their own names.
  case s: Seq[Int] if s.length == 4 =>
      s"s is a Seq of Ints of exactly length 4"  // Again, individual elements are not bound to their own names.
  case _ =>
      "No match was found!"
}

Demonstração ao vivo

Para extrair o(s) primeiro(s) elemento(s) e manter o restante como uma coleção:

def f(ints: Seq[Int]): String = ints match {
  case Seq(first, second, tail @ _*) =>
      s"The seq has at least two elements : $first, $second. The rest of the Seq is $tail"
  case Seq(first, tail @ _*) =>
      s"The seq has at least one element : $first. The rest of the Seq is $tail"
  // alternative syntax
  // here of course this one will never match since it checks
  // for the same thing as the one above
  case first +: tail =>
      s"The seq has at least one element : $first. The rest of the Seq is $tail"
  case _ =>
      "The seq didn't match any of the above, so it must be empty"
}

Em geral, qualquer forma que possa ser usada para construir uma sequência pode ser usada para combinar padrões com uma sequência existente.

Observe que ao usar Nil e :: funcionará quando o padrão corresponder a uma Sequência, ele a converterá em uma Lista e poderá ter resultados inesperados. Restrinja-se a Seq( ...) e +: para evitar isso.

Observe que ao usar :: não funcionará para WrappedArray, Vector etc, veja:

scala> def f(ints:Seq[Int]) = ints match {
     | case h :: t => h
     | case _ => "No match"
     | }
f: (ints: Seq[Int])Any

scala> f(Array(1,2))
res0: Any = No match

E com +:

scala> def g(ints:Seq[Int]) = ints match {
     | case h+:t => h
     | case _ => "No match"
     | }
g: (ints: Seq[Int])Any

scala> g(Array(1,2).toSeq)
res4: Any = 1

Guardas (se expressões) #

As instruções case podem ser combinadas com expressões if para fornecer lógica extra ao combinar padrões.

def checkSign(x: Int): String = {
    x match {
      case a if a < 0 => s"$a is a negative number"
      case b if b > 0 => s"$b is a positive number"
      case c => s"$c neither positive nor negative"
    }
}

É importante garantir que seus guardas não criem uma correspondência não exaustiva (o compilador geralmente não vai pegar isso):

def f(x: Option[Int]) = x match {
    case Some(i) if i % 2 == 0 => doSomething(i)
    case None    => doSomethingIfNone
}

Isso lança um MatchError em números ímpares. Você deve considerar todos os casos ou usar um caso de correspondência curinga:

def f(x: Option[Int]) = x match {
    case Some(i) if i % 2 == 0 => doSomething(i)
    case _ => doSomethingIfNoneOrOdd
}

Correspondência de padrões com Regex #

val emailRegex: Regex = "(.+)@(.+)\\.(.+)".r

"name@example.com" match {
  case emailRegex(userName, domain, topDomain) => println(s"Hi $userName from $domain")
  case _ => println(s"This is not a valid email.")
}

Neste exemplo, o regex tenta corresponder ao endereço de e-mail fornecido. Se isso acontecer, o userName e o domain serão extraídos e impressos. topDomain também é extraído, mas nada é feito com ele neste exemplo. Chamar .r em uma String str é equivalente a new Regex(str). A função r está disponível através de uma conversão implícita.

Correspondência de padrões com identificador estável #

Na correspondência de padrões padrão, o identificador usado ocultará qualquer identificador no escopo delimitador. Às vezes, é necessário corresponder à variável do escopo delimitador.

A função de exemplo a seguir usa um caractere e uma lista de tuplas e retorna uma nova lista de tuplas. Se o caractere existia como o primeiro elemento em uma das tuplas, o segundo elemento é incrementado. Se ainda não existir na lista, uma nova tupla será criada.

def tabulate(char: Char, tab: List[(Char, Int)]): List[(Char, Int)] = tab match {
  case Nil => List((char, 1))
  case (`char`, count) :: tail => (char, count + 1) :: tail
  case head :: tail => head :: tabulate(char, tail)
}

O exemplo acima demonstra a correspondência de padrões onde a entrada do método, char, é mantida ’estável’ na correspondência de padrões: ou seja, se você chamar tabulate('x', ...), a primeira instrução case seria interpretada Como:

case('x', count) => ...

Scala interpretará qualquer variável demarcada com uma marca como um identificador estável: também interpretará qualquer variável que comece com letra maiúscula da mesma maneira.

Correspondência em uma opção #

Se você estiver correspondendo em um tipo Option:

def f(x: Option[Int]) = x match {
    case Some(i) => doSomething(i)
    case None    => doSomethingIfNone
}

Isso é funcionalmente equivalente a usar fold ou map/getOrElse:

def g(x: Option[Int]) = x.fold(doSomethingIfNone)(doSomething)
def h(x: Option[Int]) = x.map(doSomething).getOrElse(doSomethingIfNone)

Características seladas de correspondência de padrões #

Ao combinar um padrão com um objeto cujo tipo é uma característica selada, Scala verificará em tempo de compilação se todos os casos são ‘casados ​​exaustivamente’:

sealed trait Shape
case class Square(height: Int, width: Int) extends Shape
case class Circle(radius: Int) extends Shape
case object Point extends Shape


def matchShape(shape: Shape): String = shape match {
    case Square(height, width) => "It's a square"
    case Circle(radius)        => "It's a circle"
    //no case for Point because it would cause a compiler warning.
}

Se uma nova classe case para Shape for adicionada posteriormente, todas as instruções match em Shape começarão a lançar um aviso do compilador. Isso facilita a refatoração completa: o compilador alertará o desenvolvedor sobre todo o código que precisa ser atualizado.

Pasta padrão (@) #

O sinal @ liga uma variável a um nome durante uma correspondência de padrão. A variável vinculada pode ser todo o objeto correspondente ou parte do objeto correspondente:

sealed trait Shape
case class Rectangle(height: Int, width: Int) extends Shape
case class Circle(radius: Int) extends Shape
case object Point extends Shape

(Circle(5): Shape) match {
  case Rectangle(h, w) => s"rectangle, $h x $w."
  case Circle(r) if r > 9 => s"large circle"
  case c @ Circle(_) => s"small circle: ${c.radius}"  // Whole matched object is bound to c
  case Point => "point"
}

> res0: String = círculo pequeno: 5

O identificador vinculado pode ser usado em filtros condicionais. Desta forma:

case Circle(r) if r > 9 => s"large circle"

pode ser escrito como:

case c @ Circle(_) if c.radius > 9 => s"large circle"

O nome pode ser vinculado a apenas uma parte do padrão correspondente:

Seq(Some(1), Some(2), None) match {
  // Only the first element of the matched sequence is bound to the name 'c'
  case Seq(c @ Some(1), _*) => head
  case _ => None
}

> res0: Option[Int] = Some(1)

Correspondência de padrões com classes de caso #

Cada classe case define um extrator que pode ser usado para capturar os membros da classe case durante a correspondência de padrões:

case class Student(name: String, email: String)

def matchStudent1(student: Student): String = student match {
    case Student(name, email) => s"$name has the following email: $email" // extract name and email
}

Todas as regras normais de correspondência de padrões se aplicam - você pode usar guardas e expressões constantes para controlar a correspondência:

def matchStudent2(student: Student): String = student match {
    case Student("Paul", _) => "Matched Paul" // Only match students named Paul, ignore email
    case Student(name, _) if name == "Paul" => "Matched Paul" // Use a guard to match students named Paul, ignore email
    case s if s.name == "Paul" => "Matched Paul" // Don't use extractor; use a guard to match students named Paul, ignore email
    case Student("Joe", email) => s"Joe has email $email" // Match students named Joe, capture their email
    case Student(name, email) if name == "Joe" => s"Joe has email $email" // use a guard to match students named Joe, capture their email
    case Student(name, email) => s"$name has email $email." // Match all students, capture name and email 
}

Tipos de correspondência de padrões #

A correspondência de padrões também pode ser usada para verificar o tipo de uma instância, em vez de usar isInstanceOf[B]:

val anyRef: AnyRef = ""
                                                  
anyRef match {
  case _: Number       => "It is a number"
  case _: String       => "It is a string"
  case _: CharSequence => "It is a char sequence"
}
//> res0: String = It is a string

A ordem dos casos é importante:

anyRef match {
  case _: Number       => "It is a number"
  case _: CharSequence => "It is a char sequence"
  case _: String       => "It is a string"
}
//> res1: String = It is a char sequence

Desta forma, é semelhante a uma declaração ‘switch’ clássica, sem a funcionalidade de fall-through. No entanto, você também pode combinar padrões e ’extrair’ valores do tipo em questão. Por exemplo:

case class Foo(s: String)
case class Bar(s: String)
case class Woo(s: String, i: Int)

def matcher(g: Any):String = {
  g match {
    case Bar(s) => s + " is classy!" 
    case Foo(_) => "Someone is wicked smart!"
    case Woo(s, _) => s + " is adventerous!"
    case _ => "What are we talking about?"
  }
}

print(matcher(Foo("Diana")))  // prints 'Diana is classy!'
print(matcher(Bar("Hadas")))  // prints 'Someone is wicked smart!'
print(matcher(Woo("Beth", 27)))   // prints 'Beth is adventerous!'
print(matcher(Option("Katie")))  // prints 'What are we talking about?'

Observe que no caso Foo e Woo usamos o sublinhado (_) para ‘corresponder a uma variável não vinculada’. Isso quer dizer que o valor (neste caso Hadas e 27, respectivamente) não está vinculado a um nome e, portanto, não está disponível no manipulador para esse caso. Isso é uma abreviação útil para corresponder a ‘qualquer’ valor sem se preocupar com qual é esse valor.

Pattern Matching compilado como tablewitch ou lookupswitch #

A anotação @switch informa ao compilador que a instrução match pode ser substituída por uma única instrução tableswitch no nível de bytecode. Esta é uma otimização menor que pode remover comparações desnecessárias e cargas variáveis ​​durante o tempo de execução.

A anotação @switch funciona apenas para correspondências com constantes literais e identificadores final val. Se a correspondência de padrão não puder ser compilada como um tableswitch/lookupswitch, o compilador emitirá um aviso.

import annotation.switch

def suffix(i: Int) = (i: @switch) match {
  case 1 => "st"
  case 2 => "nd"
  case 3 => "rd"
  case _ => "th"
}

Os resultados são os mesmos de uma correspondência de padrão normal:

scala> suffix(2)
res1: String = "2nd"

scala> suffix(4)
res2: String = "4th"

Da Documentação Scala (2.8+) – @switch:

Uma anotação a ser aplicada a uma expressão de correspondência. Se presente, o o compilador verificará se a correspondência foi compilada para um tablewitch ou lookupswitch, e emitirá um erro se, em vez disso, compilar em um série de expressões condicionais.

Da especificação Java:

  • tableswitch: “Acessar tabela de saltos por índice e salto”
  • lookupswitch: “Acessar tabela de saltos por combinação de teclas e salto”

Correspondência de vários padrões de uma só vez #

O | pode ser usado para que uma instrução case única corresponda a várias entradas para produzir o mesmo resultado:

def f(str: String): String = str match {
  case "foo" | "bar" => "Matched!"
  case _ => "No match."
}

f("foo")  // res0: String = Matched!
f("bar")  // res1: String = Matched!
f("fubar")  // res2: String = No match.

Observe que, embora a correspondência de valores funcione bem, a seguinte correspondência de tipos causará problemas:

sealed class FooBar
case class Foo(s: String) extends FooBar
case class Bar(s: String) extends FooBar

val d = Foo("Diana")
val h = Bar("Hadas")

// This matcher WILL NOT work.
def matcher(g: FooBar):String = {
  g match {
    case Foo(s) | Bar(s) => print(s)  // Won't work: s cannot be resolved
    case Foo(_) | Bar(_) => _         // Won't work: _ is an unbound placeholder
    case _ => "Could not match"
  }
}

Se no último caso (com _) você não precisa do valor da variável unbound e só quer fazer outra coisa, tudo bem:

def matcher(g: FooBar):String = {
  g match {
    case Foo(_) | Bar(_) => "Is either Foo or Bar."  // Works fine
    case _ => "Could not match"
  }
}

Caso contrário, você fica dividindo seus casos:

def matcher(g: FooBar):String = {
  g match {
    case Foo(s) => s 
    case Bar(s) => s
    case _ => "Could not match"
  }
}

Correspondência de padrões em tuplas #

Dada a seguinte Lista de tuplas:

val pastries = List(("Chocolate Cupcake", 2.50), 
                    ("Vanilla Cupcake", 2.25),
                    ("Plain Muffin", 3.25))

A correspondência de padrões pode ser usada para lidar com cada elemento de maneira diferente:

pastries foreach { pastry =>
  pastry match {
    case ("Plain Muffin", price) => println(s"Buying muffin for $price")
    case p if p._1 contains "Cupcake" => println(s"Buying cupcake for ${p._2}")
    case _ => println("We don't sell that pastry")
  }
}

O primeiro caso mostra como combinar com uma string específica e obter o preço correspondente. O segundo caso mostra o uso de if e extração de tupla para corresponder aos elementos da tupla.
