Sintaxe #

  • val extractor(extractedValue1, _ /* ignorou o segundo valor extraído */) = valueToBeExtracted
  • valueToBeExtracted match { case extractor(extractedValue1, _) => ???}
  • val (tuple1, tuple2, tuple3) = tupleWith3Elements
  • object Foo { def unapply(foo: Foo): Option[String] = Some(foo.x); }

Extratores de classe de caso #

Uma classe de caso é uma classe com muitos códigos clichê padrão incluídos automaticamente. Um benefício disso é que o Scala facilita o uso de extratores com classes case.

case class Person(name: String, age: Int)  // Define the case class
val p = Person("Paola", 42)  // Instantiate a value with the case class type

val Person(n, a) = p  // Extract values n and a
// n: String = Paola
// a: Int = 42

Nesta conjuntura, tanto n quanto a são vals no programa e podem ser acessados ​​como tal: dizem que eles foram ’extraídos’ de p. Continuando:

val p2 = Person("Angela", 1337)

val List(Person(n1, a1), Person(_, a2)) = List(p, p2)
// n1: String = Paola
// a1: Int = 42
// a2: Int = 1337

Aqui vemos duas coisas importantes:

  • A extração pode acontecer em níveis ‘profundos’: propriedades de objetos aninhados podem ser extraídas.
  • Nem todos os elementos precisam ser extraídos. O caractere curinga _ indica que esse valor específico pode ser qualquer coisa e é ignorado. Nenhum val é criado.

Em particular, isso pode facilitar a correspondência em coleções:

val ls = List(p1, p2, p3)  // List of Person objects
ls.map(person => person match {
  case Person(n, a) => println("%s is %d years old".format(n, a))
})

Aqui, temos um código que usa o extrator para verificar explicitamente se person é um objeto Person e imediatamente retira as variáveis ​​que nos interessam: n e a.

Extratores de Tuplas #

x e y são extraídos da tupla:

val (x, y) = (1337, 42)
// x: Int = 1337
// y: Int = 42

Para ignorar um valor use _:

val (_, y: Int) = (1337, 42)
// y: Int = 42

Para desembalar um extrator:

val myTuple = (1337, 42)
myTuple._1  // res0: Int = 1337
myTuple._2  // res1: Int = 42

Observe que as tuplas têm um comprimento máximo de 22 e, portanto, ._1 a ._22 funcionarão (assumindo que a tupla tenha pelo menos esse tamanho).

Os extratores de tupla podem ser usados ​​para fornecer argumentos simbólicos para funções literais:

val persons = List("A." -> "Lovelace", "G." -> "Hopper")
val names = List("Lovelace, A.", "Hopper, G.")

assert {
  names ==
    (persons map { name =>
      s"${name._2}, ${name._1}"
    })
}

assert {
  names ==
    (persons map { case (given, surname) =>
      s"$surname, $given"
    })
}

Extratores de Regex #

Uma expressão regular com partes agrupadas pode ser usada como extrator:

scala> val address = """(.+):(\d+)""".r
address: scala.util.matching.Regex = (.+):(\d+)

scala> val address(host, port) = "some.domain.org:8080"
host: String = some.domain.org
port: String = 8080

Observe que quando não for correspondido, um MatchError será lançado em tempo de execução:

scala> val address(host, port) = "something not a host and port"
scala.MatchError: something not a host and port (of class java.lang.String)

Notação infixa do extrator #

Se uma classe case tem exatamente dois valores, seu extrator pode ser usado em notação infixa.

case class Pair(a: String, b: String)
val p: Pair = Pair("hello", "world")
val x Pair y = p
//x: String = hello
//y: String = world

Qualquer extrator que retorne uma tupla de 2 pode funcionar dessa maneira.

object Foo {
    def unapply(s: String): Option[(Int, Int)] = Some((s.length, 5))
}
val a Foo b = "hello world!"
//a: Int = 12
//b: Int = 5

Desaplicar - Extratores personalizados #

Uma extração personalizada pode ser escrita implementando o método unapply e retornando um valor do tipo Option:

class Foo(val x: String)

object Foo {
  def unapply(foo: Foo): Option[String] = Some(foo.x)
}

new Foo("42") match {
  case Foo(x) => x
}
// "42"

O tipo de retorno de unapply pode ser algo diferente de Option, desde que o tipo retornado forneça os métodos get e isEmpty. Neste exemplo, Bar é definido com esses métodos, e unapply retorna uma instância de Bar:

class Bar(val x: String) {
  def get = x
  def isEmpty = false
}

object Bar {
  def unapply(bar: Bar): Bar = bar
}

new Bar("1337") match {
  case Bar(x) => x
}
// "1337"

O tipo de retorno de unapply também pode ser um Boolean, que é um caso especial que não possui os requisitos get e isEmpty acima. No entanto, observe neste exemplo que DivisibleByTwo é um objeto, não uma classe, e não recebe um parâmetro (e, portanto, esse parâmetro não pode ser vinculado):

object DivisibleByTwo {
  def unapply(num: Int): Boolean = num % 2 == 0
}

4 match {                        
  case DivisibleByTwo() => "yes" 
  case _ => "no"
}
// yes

3 match {
  case DivisibleByTwo() => "yes"
  case _ => "no"
}
// no

Lembre-se que unapply vai no objeto companheiro de uma classe, não na classe. O exemplo acima ficará claro se você entender essa distinção.

Extratores transformadores #

O comportamento do extrator pode ser usado para derivar valores arbitrários de sua entrada. Isso pode ser útil em cenários em que você deseja atuar nos resultados de uma transformação caso a transformação seja bem-sucedida.

Considere como exemplo os vários formatos de nome de usuário utilizáveis ​​em um ambiente Windows:

object UserPrincipalName {
  def unapply(str: String): Option[(String, String)] = str.split('@') match {
    case Array(u, d) if u.length > 0 && d.length > 0 => Some((u, d))
    case _ => None
  }        
}

object DownLevelLogonName {
  def unapply(str: String): Option[(String, String)] = str.split('\\') match {
    case Array(d, u) if u.length > 0 && d.length > 0 => Some((d, u))
    case _ => None
  }
}

def getDomain(str: String): Option[String] = str match {
  case UserPrincipalName(_, domain) => Some(domain)
  case DownLevelLogonName(domain, _) => Some(domain)
  case _ => None
}

De fato, é possível criar um extrator exibindo ambos os comportamentos, ampliando os tipos que podem corresponder:

object UserPrincipalName {
  def unapply(obj: Any): Option[(String, String)] = obj match {
    case upn: UserPrincipalName => Some((upn.username, upn.domain))
    case str: String => str.split('@') match {
      case Array(u, d) if u.length > 0 && d.length > 0 => Some((u, d))
      case _ => None
    }
    case _ => None
  }        
}

Em geral, extratores são simplesmente uma reformulação conveniente do padrão Option, aplicado a métodos com nomes como tryParse:

UserPrincipalName.unapply("user@domain") match {
  case Some((u, d)) => ???
  case None => ???
}
