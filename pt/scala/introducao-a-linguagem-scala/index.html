Hello World estendendo App #

object HelloWorld extends App {
  println("Hello, world!")
}

Demonstração ao vivo

Ao estender o App trait, você pode evitar definir um método main explícito . Todo o corpo do objeto HelloWorld é tratado como “o método principal”.

Inicialização atrasada #

De acordo com a documentação oficial, o App usa um recurso chamado Inicialização Atrasada. Isso significa que os campos do objeto são inicializados depois que o método main é chamado.

Inicialização atrasada #

De acordo com a documentação oficial, App usa um recurso chamado Delayed Initialization. Isso significa que os campos do objeto são inicializados depois que o método main é chamado.

DelayedInit agora está obsoleto para uso geral, mas ainda é suportado para App como um caso especial. O suporte continuará até que um recurso de substituição seja decidido e implementado.

Para acessar argumentos de linha de comando ao estender App, use this.args:

object HelloWorld extends App {
  println("Hello World!")
  for {
    arg <- this.args
  } println(s"Arg=$arg")
}

Ao usar App, o corpo do objeto será executado como o método main, não há necessidade de substituir main.

Hello World Definindo um método ‘principal’ #

Coloque este código em um arquivo chamado HelloWorld.scala:

object Hello {
  def main(args: Array[String]): Unit = {
    println("Hello World!")
  }
}

Demonstração ao vivo

Para compilá-lo para bytecode executável pela JVM:

$ scalac HelloWorld.scala

Para executá-lo:

$ scala Hello

Quando o runtime Scala carrega o programa, ele procura um objeto chamado Hello com um método main. O método main é o ponto de entrada do programa e é executado.

Note que, diferentemente do Java, Scala não tem nenhum requisito de nomear objetos ou classes após o arquivo em que estão. Em vez disso, o parâmetro Hello passado no comando scala Hello refere-se ao objeto a ser procurado que contém o método a ser executado. É perfeitamente possível ter vários objetos com métodos principais no mesmo arquivo.scala`.

O array args conterá os argumentos de linha de comando fornecidos ao programa, se houver. Por exemplo, podemos modificar o programa assim:

object HelloWorld {
  def main(args: Array[String]): Unit = {
    println("Hello World!")
    for {
      arg <- args
    } println(s"Arg=$arg")
  }
}

Compile-o:

$ scalac HelloWorld.scala

E depois execute-o:

$ scala HelloWorld 1 2 3
Hello World!
Arg=1
Arg=2
Arg=3

Olá Mundo como um script #

Scala pode ser usado como uma linguagem de script. Para demonstrar, crie HelloWorld.scala com o seguinte conteúdo:

println("Hello")

Execute-o com o interpretador de linha de comando (o $ é o prompt de linha de comando):

$ scala HelloWorld.scala
Hello

Se você omitir .scala (como se você simplesmente digitasse scala HelloWorld), o executor procurará um arquivo .class compilado com bytecode em vez de compilar e executar o script.

Observação: Se scala for usado como linguagem de script, nenhum pacote poderá ser definido.

Em sistemas operacionais que utilizam bash ou terminais shell semelhantes, os scripts Scala podem ser executados usando um ‘preâmbulo shell’. Crie um arquivo chamado HelloWorld.sh e coloque o seguinte como seu conteúdo:

#!/bin/sh
exec scala "$0" "$@"
!#
println("Hello")

As partes entre #! e !# são o ‘preâmbulo do shell’ e são interpretadas como um script bash. O resto é Scala.

Depois de salvar o arquivo acima, você deve conceder permissões ’executáveis’. No shell você pode fazer isso:

$ chmod a+x HelloWorld.sh

(Observe que isso dá permissão a todos: leia sobre chmod para aprender como configurá-lo para conjuntos de usuários mais específicos.)

Agora você pode executar o script assim:

$ ./HelloWorld.sh

Usando o Scala REPL #

Quando você executa scala em um terminal sem parâmetros adicionais, ele abre um interpretador REPL (Read-Eval-Print Loop):

nford:~ $ scala
Welcome to Scala 2.11.8 (Java HotSpot(TM) 64-Bit Server VM, Java 1.8.0_66).
Type in expressions for evaluation. Or try :help.

scala>

O REPL permite que você execute Scala em forma de planilha: o contexto de execução é preservado e você pode experimentar comandos manualmente sem ter que construir um programa inteiro. Por exemplo, digitando val poema = "As halcyons seremos" ficaria assim:

scala> val poem = "As halcyons we shall be"
poem: String = As halcyons we shall be

Agora podemos imprimir nosso val:

scala> print(poem)
As halcyons we shall be

Observe que val é imutável e não pode ser substituído:

scala> poem = "Brooding on the open sea"
<console>:12: error: reassignment to val
       poem = "Brooding on the open sea"

Mas no REPL você pode redefinir um val (o que causaria um erro em um programa Scala normal, se fosse feito no mesmo escopo):

scala> val poem = "Brooding on the open sea"
poem: String = Brooding on the open sea

Para o restante de sua sessão REPL, esta variável recém-definida irá sombrear a variável definida anteriormente. REPLs são úteis para ver rapidamente como os objetos ou outros códigos funcionam. Todos os recursos do Scala estão disponíveis: você pode definir funções, classes, métodos, etc.
