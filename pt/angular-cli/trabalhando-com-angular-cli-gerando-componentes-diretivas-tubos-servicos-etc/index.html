A ferramenta angular-cli pode ajudá-lo a montar diferentes partes de um aplicativo angular (componentes, diretivas, tubos, serviços, classes, guardas, interfaces, enumerações e módulos).

Sintaxe #

  • ng gerar [componente | directiva | serviço | tubo | aula | enum | interface | guarda | módulo] [nome] [sinalizadores…]
  • ng g [c | d | s | p | cl | e | eu | g | m] [nome] [bandeiras…]

Parâmetros #

Parâmetro Descrição
componente ou c Usado para gerar o componente
diretiva ou d Usado para gerar diretivas
serviço ou s Usado para gerar serviços
pipe ou p Usado para gerar tubos
class ou cl Usado para gerar classes
enum ou e Usado para gerar enums
interfaces ou i Usado para gerar interfaces
guarda ou g Usado para gerar guardas
módulo ou m Usado para gerar módulos
--flat ou -f Usado para habilitar/desabilitar a criação de diretórios
--inline-template ou -it Usado para ativar/desativar modelos html embutidos em componentes
--inline-style ou -is Usado para habilitar/desabilitar estilos embutidos em componentes
--prefix ou -p Usado para desativar ou alterar o prefixo
--spec ou -s Usado para habilitar/desabilitar a criação de arquivos .spec
--skip-import Usado para pular a importação do módulo
--app ou -a Usado para especificar o nome do aplicativo a ser usado
--module ou -m Usado para especificar o módulo declarante
--view-encapsulation ou -ve Usado para especificar a estratégia de encapsulamento de visão em componentes
--change-detection ou -cd Usado para especificar a estratégia de detecção de alterações em componentes
--routing ou -r Usado para especificar se o arquivo do módulo de roteamento deve ser gerado

Uso de “Gerar comando” #

Você pode usar o comando ng generate ou ng g para gerar blocos de construção Angular (componentes, serviços, pipes, etc.).

Você pode encontrar todos os planos possíveis na tabela abaixo:

Andaime Uso Encurtado
Componente de gerar componente nome-do-componente of g c component-name
Directiva ng gere diretiva nome-diretiva ng g d nome-diretiva
Tubulação ng gerar pipe-name ng g p pipe-name
Serviço ng gere serviço service-name ng g s service-name
Classe ng gerar classe class-name ng g cl class-name
Guarda ng gerar guard guard-name ng g g guard-name
Interface ng gere interface nome-da-interface ng g i interface-name
Enum ng gere enum enum-name ng g e enum-name
Módulo de gerar módulo nome-do-módulo of g m module-name

Então, por exemplo, se você executar ng generate component user-list - o angular-cli irá:

  • crie o diretório user-list na pasta src/app ou pasta onde você executou o comando.
  • dentro desse diretório gere 4 arquivos (user-list.component.ts, user-list.component.html, user-list.component.css e user-list.component.spec.ts)
  • adicione user-list como uma declaração no decorador @NgModule do módulo mais próximo.

Gerando serviços #

Para adicionar um serviço com o nome UserService, execute:

  $ ng g s user

  installing service
    create src/app/user.service.spec.ts
    create src/app/user.service.ts
  • Para evitar a criação de arquivos .spec adicione o sinalizador --spec false ou -sp false
  $ ng g s user --spec false
  
  installing service
    create src/app/user.service.ts
  • Para habilitar a criação de pastas, adicione o sinalizador --flat false ou -f false
  $ ng g s user --flat false

  installing service
    create src/app/user/user.service.spec.ts
    create src/app/user/user.service.ts

Você também pode combinar os sinalizadores listados acima. Por exemplo, para criar apenas o arquivo user.service.ts dentro da pasta user sem o arquivo .spec use o seguinte comando.

  $ ng g s user -f false -sp false

  installing service
    create src/app/user/user.service.ts

Todos os sinalizadores generate service:

Descrição Bandeira Encurtado Valor padrão
Ativar a criação de pastas --flat false -f falso verdadeiro
Impedir a criação de arquivos .spec --spec false -sp falso verdadeiro

Gerando módulos #

Para adicionar um módulo chamado GuestModule, execute:

  $ ng g m guest

  installing module
    create src/app/guest/guest.module.ts
  • Para habilitar a criação de arquivos .spec adicione o sinalizador --spec ou -sp
  $ ng g m guest --spec

  installing module
    create src/app/guest/guest.module.spec.ts
    create src/app/guest/guest.module.ts
  • Para habilitar o roteamento, adicione o sinalizador --routing ou -r
  $ ng g m guest --routing

  installing module
    create src/app/guest/guest-routing.module.ts
    create src/app/guest/guest.module.ts

Você também pode combinar os sinalizadores listados acima. Por exemplo, para criar um módulo com roteamento e especificações, use o comando a seguir.

  $ ng g m guest -sp -r
  
  installing module
    create src/app/guest/guest-routing.module.ts
    create src/app/guest/guest.module.spec.ts
    create src/app/guest/guest.module.ts

Todos os sinalizadores generate module:

Descrição Bandeira Encurtado Valor padrão
Habilite a criação de arquivos .spec --spec -sp falso
Ativar roteamento --routing -r falso

Gerando componentes #

Para adicionar um componente com um seletor [prefix]-user-list, execute:

  $ ng g c user-list
  
  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts

O prefixo evita colisões de nomes de elementos com componentes em outros aplicativos e com elementos HTML nativos. Assim, por exemplo, se o prefixo for app - o componente gerado terá o seletor app-user-list.

  • Para evitar o uso de prefixo, adicione o sinalizador --prefix false ou -p false
  $ ng g c user-list --prefix false

import { Component } from '@angular/core';

@Component({
  selector: 'user-list',
  templateUrl: './user-list.component.html',
  styleUrls: ['./user-list.component.css']
})
export class UserListComponent {}
  • Para evitar a criação de arquivos .spec adicione o sinalizador --spec false ou -sp false
  $ ng g c user-list --spec false

  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Para usar modelos html embutidos em vez de modelos externos, adicione o sinalizador --inline-template ou -it
  $ ng g c user-list --inline-template

  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Para usar estilos inline em vez de estilos externos, adicione o sinalizador --inline-style ou -is
  $ ng g c user-list --inline-style

  installing component
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Para evitar a criação de pastas, adicione o sinalizador --flat ou -f
  $ ng g c user-list --flat
  
  installing component
    create src/app/user-list.component.css
    create src/app/user-list.component.html
    create src/app/user-list.component.spec.ts
    create src/app/user-list.component.ts
    update src/app/app.module.ts

Você também pode combinar os sinalizadores listados acima. Por exemplo, para criar apenas o arquivo .component.ts sem os arquivos e a pasta .css, .html, .spec use o seguinte comando.

  $ ng g c user-list -f -it -is -sp false
  
  installing component
    create src/app/user-list.component.ts
    update src/app/app.module.ts

Todos os sinalizadores generate component:

Descrição Bandeira Encurtado Valor padrão
Impedir a criação de pastas --flat -f falso
Impedir o uso de prefixo --prefix false -p falso verdadeiro
Impedir a criação de arquivos .spec --spec false -sp falso verdadeiro
Ativar modelos html embutidos --inline-template -it falso
Ativar estilos embutidos --inline-style -é falso

Gerando diretivas #

Para adicionar uma diretiva com um seletor [prefix]Highlight, execute:

  $ ng g d highlight

  installing directive
    create src/app/highlight.directive.spec.ts
    create src/app/highlight.directive.ts
    update src/app/app.module.ts
  • Para evitar o uso de prefixo, adicione o sinalizador --prefix false ou -p false
  $ ng g d highlight --prefix false

import { Directive } from '@angular/core';

@Directive({
  selector: '[highlight]'
})
export class HighlightDirective {}
  • Para evitar a criação de arquivos .spec adicione o sinalizador --spec false ou -sp false
  $ ng g d highlight --spec false

  installing directive
    create src/app/highlight.directive.ts
    update src/app/app.module.ts
  • Para habilitar a criação de pastas, adicione o sinalizador --flat false ou -f false
  $ ng g d highlight --flat false
  
  installing directive
    create src/app/highlight/highlight.directive.spec.ts
    create src/app/highlight/highlight.directive.ts
    update src/app/app.module.ts

Você também pode combinar os sinalizadores listados acima. Por exemplo, para criar apenas o arquivo highlight.directive.ts dentro da pasta highlight sem o arquivo .spec use o seguinte comando.

  $ ng g d highlight -f false -sp false

  installing directive
    create src/app/highlight/highlight.directive.ts
    update src/app/app.module.ts

Todos os sinalizadores de gerar diretiva:

Descrição Bandeira Encurtado Valor padrão
Ativar a criação de pastas --flat false -f falso verdadeiro
Impedir o uso de prefixo --prefix false -p falso verdadeiro
Impedir a criação de arquivos .spec --spec false -sp falso verdadeiro
