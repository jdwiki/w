LeftAndMain é mais uma API de nível inferior e muitas vezes não é necessária devido à existência de ModelAdmin. No entanto, se você quiser criar uma interface de usuário personalizada que não exija necessariamente a funcionalidade de ModelAdmin no painel de administração do seu módulo, então LeftAndMain é onde você deseja começar.

1. Introdução #

Este guia destina-se a ajudá-lo a criar sua própria interface de usuário subclassificando a classe LeftAndMain.

Ao final deste guia, você terá criado sua primeira interface Hello World no Painel de Administração.

Requisitos #

Este guia requer que você tenha pelo menos a versão 3.* do framework AND CMS, mas inferior à versão 4.*.

Se você quiser usar este guia, precisará trocar quaisquer referências de classe pelo Fully Quality Class Name (FQCN), conforme definido no guia de atualização do SS4.

Preparação #

tl;dr Ignore as etapas a seguir e simplesmente crie a estrutura abaixo delas

  1. Crie uma pasta com o nome de qualquer coisa que você escolher no diretório raiz do seu projeto SilverStripe, para este exemplo usaremos /helloworld/ e criaremos um arquivo vazio dentro dessa pasta chamado _config.php. Um _config.php no mínimo é necessário em cada diretório de módulo para que o SilverStripe detecte sua existência.

  2. Dentro de sua nova pasta, crie uma subpasta chamada exatamente /code/ e dentro dessa pasta, para fins de organização; crie outra pasta chamada /admin/

  3. Crie /helloworld/code/admin/HelloWorldLeftAndMain.php e coloque o seguinte código nele por enquanto.

     class HelloWorldLeftAndMain extends LeftAndMain {

 }

  4. Crie o arquivo de modelo que será usado com esta classe chamado /helloworld/templates/Includes/HelloWorldLeftAndMain.ss

Estrutura #

/framework/
/cms/
/helloworld/
    + _config.php
    + /code/
        + /admin/
            + /HelloWorldLeftAndMain.php
    + /templates/
        + /Includes/
            + /HelloWorldLeftAndMain_Content.ss

2. Configurando HelloWorldLeftAndMain.php #

Se você ainda não o fez, basta iniciar este arquivo com:

class HelloWorldLeftAndMain extends LeftAndMain {

}

Configurar #

A primeira coisa que você deve fazer é definir o $url_segment que será usado para acessar a interface, e o título ($menu_title) que aparecerá no menu de navegação lateral do painel de administração:

private static $url_segment = 'helloworld';
private static $menu_title  = 'Hello World';

As seguintes variáveis ​​de configuração são opcionais e não são usadas neste guia:

private static $menu_icon   = 'helloworld/path/to/my/icon.png';
private static $url_rule    = '/$Action/$ID/$OtherID';

Adicionando folhas de estilo e Javascript #

LeftAndMain permite que você substitua o método init em seu pai, podemos usar isso para exigir arquivos específicos para nossa interface. Sem dúvida, você deve sempre precisar de uma folha de estilo CSS que estilize os elementos da sua interface de usuário.

Como dica, é recomendável nunca confiar nas classes CSS fornecidas pelo CMS, pois elas estão sujeitas a alterações sem aviso prévio e, posteriormente, destruirão a aparência e o comportamento de sua interface do usuário (por exemplo, 3.* a 4.* viu uma reforma completa da interface, portanto, qualquer classe CSS que você confiou em 3.* precisa ser reestilizada para conversão para 4.*)

Então vamos adicionar nosso arquivo helloworld/css/styles.css:

public function init() {
    parent::init();

    Requirements::css('helloworld/css/styles.css');
    //Requirements::javascript('helloworld/javascript/script.min.js');
}

Não precisamos de nenhuma funcionalidade Javascript para este exemplo, mas no exemplo acima incluí como se poderia adicionar um arquivo Javascript usando a classe Requirements.

Depois disso, você pode adotar o que está acostumado ao lidar com Page_Controller como $allowed_actions etc com uma diferença notável, no entanto

Você NÃO PODE substituir index().

Em vez disso, index() é assumido como HelloWorldLeftAndMain_Content.ss e, a partir daí, é necessário lidar com a exibição de índices por meio de funções de modelo (veja o exemplo abaixo)

Código Completo #

class HelloWorldLeftAndMain extends LeftAndMain {
    private static $url_segment = 'helloworld';
    private static $menu_title  = 'Hello World';
    private static $allowed_actions = array(
        'some_action'
    );

    public function init() {
        parent::init();

        Requirements::css('helloworld/css/styles.css');
        //Requirements::javascript('helloworld/javascript/script.min.js');
    }

    public function Hello($who=null) {
        if (!$who) {
            $who = 'World';
        }

        return "Hello " . htmlentities($who);
    } 
}

3. O modelo (HelloWorldLeftAndMain_Content.ss) #

A estrutura esperada deste modelo pode ser um pouco complicada, mas tudo se resume a isso:

  1. Existem 3 seções dignas de nota para este guia: #

    • .north
    • .center
    • .south

  2. Deve ser encapsulado inteiramente dentro de um elemento que tenha o atributo data-pjax-fragment="Content". Isso é para que as chamadas AJAX geradas a partir do menu lateral saibam onde está o “Conteúdo” para que ele possa exibi-lo adequadamente:

<div class="cms-content center $BaseCSSClasses" data-layout-type="border" data-pjax-fragment="Content">

</div>

Não entrarei em detalhes sobre a funcionalidade do modelo, incluí comentários quando relevantes, mas você não deve ler este guia se não entender a sintaxe do modelo para SilverStripe

Código Completo #

A única coisa de baixo; que você deve esperar sair já estilizado é o <% include CMSBreadcrumbs %> tudo o mais que você deve providenciar no arquivo CSS que foi incluído anteriormente

<div class="cms-content center $BaseCSSClasses" data-layout-type="border" data-pjax-fragment="Content">
    <%-- This will add the breadcrumb that you see on every other menu item --%>
    <div class="cms-content-header north">
        <div class="cms-content-header-info">
            <% include CMSBreadcrumbs %>
        </div>
    </div>
    
    <div class="center">
        <%-- Our function in HelloWorldLeftAndMain.php --%>
        $Hello('USER');
        <%-- ^ outputs "Hello USER" --%>
    </div>

    <div class='south'>
        Some footer-worthy content
    </div>
</div>

Agora tudo o que resta a fazer é /dev/build e ?flush=1 então você pode conferir nosso pequeno módulo inútil no Painel de Administração!
